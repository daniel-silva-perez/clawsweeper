## Summary

Keep this issue open. The repository policy for openclaw/clawhub only allows automated close proposals for pull requests that current main certainly implements, not issues. Substantively, current main has partial support for optional env metadata through `metadata.openclaw.envVars` entries with `required: false`, but the skill-format docs still direct publishers to `requires.env`, the CLI schema copy does not expose `envVars`, and draft PR #1859 is the active implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and use PR #1859 as the implementation candidate. The best path is to document `metadata.openclaw.envVars` entries with `required: false`, align the CLI schema copy and security/scanner guidance, and keep regression coverage showing optional env vars do not need to be declared under `requires.env`. If maintainers prefer `requires.optionalEnv`, implement that spelling consistently across schema, parser, scanner, docs, CLI, API, and UI.
