## Summary

Keep this issue open. The repository cleanup policy for openclaw/clawhub only permits auto-close proposals for pull requests that are certainly implemented on current main, not issues. Substantively, current main only partially supports optional environment metadata: shared schema/parser/UI paths recognize `envVars` with `required: false`, but the public skill format docs still direct authors to `requires.env`, the CLI package schema copy lacks `envVars`, and draft PR #1859 is the open implementation candidate linked to this issue.

## Best Possible Solution

Keep this issue open and use PR #1859 as the implementation candidate. The best path is to document `metadata.openclaw.envVars` entries with `required: false`, align the CLI schema copy and scanner/security prompt, and keep regression coverage showing optional env vars do not need to be declared under `requires.env`. If maintainers prefer `requires.optionalEnv`, implement that spelling consistently across schema, parser, scanner, docs, CLI, API, and UI.
