## Summary

Keep open. Current main only partially supports optional skill env vars: shared parsing/schema/UI paths understand `envVars` with `required: false`, but the public skill-format docs still describe `requires.env`, the CLI schema copy does not expose `envVars`, and the linked draft implementation remains open. The ClawHub cleanup policy for this pass also keeps issues open.

## What This Changes

The issue asks ClawHub to provide a semantically correct way to declare optional skill environment variables and clarify scanner/docs behavior around them.

## Best Possible Solution

Align on one supported optional-env declaration, likely `metadata.openclaw.envVars` entries with `required: false` unless maintainers choose `requires.optionalEnv`; document it, add it to the CLI schema copy, keep scanner/security-prompt behavior consistent, and land regression coverage through #1859 or an equivalent maintainer-approved change.
