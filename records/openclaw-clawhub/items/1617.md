## Summary

Keep open. Current main only partially addresses optional skill environment variables: shared parsing/schema/UI paths understand `envVars` with `required: false`, but the public skill-format docs still tell authors to use `requires.env`, the CLI schema copy does not expose `envVars`, and the linked draft implementation remains open.

## What This Changes

The issue asks ClawHub to clarify or support optional environment variables in skill metadata without requiring authors to declare them as required under `requires.env`.

## Best Possible Solution

Align on one supported optional-env declaration, likely `metadata.openclaw.envVars` entries with `required: false` unless maintainers choose `requires.optionalEnv`; document it in the skill-format docs, add it to the CLI schema copy, keep scanner/security-prompt behavior consistent, and land regression coverage through #1859 or an equivalent maintainer-approved change.
