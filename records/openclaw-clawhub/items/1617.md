## Summary

Keep this issue open. Repository policy for openclaw/clawhub only permits auto-closing PRs that are definitely implemented on main, and this is an issue. Substantively, current main has partial support for optional env declarations via `envVars[].required: false`, but the public skill-format docs still document only `requires.env`, the security prompt/install surfaces are not fully aligned, and the packaged CLI schema copy does not expose `envVars`.

## Best Possible Solution

Keep this issue open and resolve it by choosing one canonical optional-env spelling. The lowest-risk path is to document `metadata.openclaw.envVars` / `metadata.clawdis.envVars` entries with `required: false`, update all schema/API/CLI copies that expose skill metadata, teach the security prompt and install/setup surfaces to distinguish optional from required env vars, and add focused tests showing that an optional variable such as `NWC_URL` is displayed as optional and does not trigger an undeclared-env warning. If maintainers prefer `requires.optionalEnv`, implement that consistently across schema, parser, scanner, docs, UI, and tests instead.
