## Summary

Keep open. Current main only partially supports optional skill environment variables: shared parsing, schema, moderation, and UI paths understand `envVars` with `required: false`, but the public skill-format docs still center `requires.env`, the CLI schema copy lacks `envVars`, and the linked draft PR remains open. The ClawHub cleanup policy for this pass also keeps issues open.

## What This Changes

The issue asks ClawHub to clarify or support a semantically correct way for skills to declare optional environment variables instead of forcing optional credentials into `requires.env`.

## Best Possible Solution

Align on one supported optional-env declaration, likely `metadata.openclaw.envVars` entries with `required: false` unless maintainers choose `requires.optionalEnv`; document it, add it to the CLI schema copy, keep scanner/security-prompt behavior consistent, and land regression coverage through #1859 or an equivalent maintainer-approved change.
