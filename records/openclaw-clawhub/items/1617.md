## Summary

Keep this issue open. Repository policy for openclaw/clawhub only permits automated close proposals for pull requests that are certainly implemented on current main, not issues. Substantively, current main has partial optional-env support through `envVars` entries with `required: false`, but the public skill-format docs and CLI schema copy are still not aligned, and open draft PR #1859 is the active implementation candidate with closing syntax.

## Best Possible Solution

Keep this issue open and use PR #1859 as the implementation candidate. The best path is to document `metadata.openclaw.envVars` entries with `required: false`, align the CLI schema copy and scanner/security guidance, and keep regression coverage showing optional env vars do not need to be declared under `requires.env`. If maintainers prefer the proposed `requires.optionalEnv` spelling, implement it consistently across schema, parser, scanner, docs, CLI, API, and UI.
