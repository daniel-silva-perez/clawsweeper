## Summary

Keep open. This is an open ClawHub issue, and the repo-specific cleanup policy only allows automated closure for pull requests that are certainly implemented on current main. The optional-env request is also only partially resolved: current main supports `envVars` with `required: false` in shared schema/parser/UI paths, but the public skill-format docs, broader spec docs, CLI schema copy, and scanner prompt wording remain inconsistent. Draft PR #1859 is the linked open implementation candidate.

## Best Possible Solution

Keep this issue open while PR #1859, or an equivalent maintainer-approved change, aligns the docs, CLI schema copy, scanner/security prompt, and regression coverage around optional environment declarations. The likely lowest-impact path is to document `metadata.openclaw.envVars` entries with `required: false`; if maintainers prefer `requires.optionalEnv`, that spelling should be implemented consistently across schema, parser, scanner, docs, CLI, API, and UI.
