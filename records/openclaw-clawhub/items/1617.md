## Summary

Keep this issue open. ClawHub cleanup policy keeps issues open unless they are PRs certainly implemented on main, and current main still only partially resolves the optional-env-variable story: shared metadata/parser/UI paths understand `envVars` with `required: false`, but the public skill-format docs and CLI schema copy remain inconsistent, and draft PR #1859 is the open implementation candidate for this issue.

## What This Changes

The issue asks ClawHub to clarify how skills should declare optional environment variables without incorrectly marking them as required under `requires.env`.

## Best Possible Solution

Keep this issue open while maintainers review and either land PR #1859 or merge an equivalent change that documents `metadata.openclaw.envVars` with `required: false`, aligns the CLI schema copy, updates scanner/security-prompt wording, and adds regression coverage. If maintainers prefer the proposed `requires.optionalEnv` spelling instead, that should be implemented consistently across schema, parser, scanner, docs, CLI, API, and UI.
