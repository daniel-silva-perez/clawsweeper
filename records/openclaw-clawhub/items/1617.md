## Summary

Keep this issue open. Current main has partial optional-env support through `envVars` entries with `required: false`, but the public skill-format docs still document only `requires.env`, the CLI schema copy does not yet expose `envVars`, and an open draft PR (#1859) explicitly references this issue with closing syntax. ClawHub policy also says issues should not be auto-closed by this review workflow.

## Best Possible Solution

Keep this issue open and use PR #1859 as the implementation candidate. The best path is to document the canonical `metadata.openclaw.envVars` shape with `required: false`, align the CLI schema copy and scanner/security guidance, and keep regression coverage showing optional env vars do not have to be declared under `requires.env`. If maintainers prefer `requires.optionalEnv`, implement that spelling consistently across schema, parser, scanner, docs, CLI, API, and UI instead.
