## Summary

Keep open. The report is concrete, but it targets OpenClaw runtime configuration, model picker role persistence, and Web UI/WebSocket synchronization, while this checkout contains the ClawHub registry, package catalog, and adjacent CLI sync/config discovery surface. The ClawHub cleanup policy for this pass also keeps issues open unless they are PRs already implemented on current main.

## What This Changes

The issue reports that selecting the intended fallback model before the intended primary model in OpenClaw's `/model` multi-select writes the model array in checkbox order, causing primary/fallback priority and Web UI sync to misalign.

## Best Possible Solution

Transfer or recreate the report in the OpenClaw runtime/configuration repository, then fix the configure/Web UI flow so primary and fallback model roles are explicit or reconstructed deterministically from final role assignments, with regression coverage proving checkbox order cannot determine priority.
