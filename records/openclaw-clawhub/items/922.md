## Summary

Keep this issue open. The repository-specific cleanup policy only permits auto-closing ClawHub PRs that are certainly implemented on current main, and this item is an issue. The current source also supports the reported product/support gap: skill slugs are still stored behind a global slug index, soft-delete does not free the slug, publishing under a different publisher is rejected as a slug collision, and the available reclaim path is admin/support tooling rather than a self-service personal-to-company reclaim flow.

## Best Possible Solution

Keep this issue open for maintainer support. A maintainer should inspect the live pixverse user, skill, and publisher records, resolve the immediate ownership request if appropriate, and then either document the expected deleted-slug behavior or implement a publisher-aware reclaim/transfer flow that safely moves a soft-deleted personal slug to an authorized company publisher.
