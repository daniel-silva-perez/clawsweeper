## Summary

Current main still leaves this as a maintainer support/product issue. Skill slugs are globally indexed, soft deletion preserves the existing skill row and owner publisher, publishing as a different publisher is rejected, and the available reclaim/transfer flows are user-centric or personal-publisher based rather than a company-publisher reuse path.

## What This Changes

The issue asks why a deleted personal-account skill slug, `pixverse`, cannot be reused from a company account and what recovery path is available.

## Best Possible Solution

Keep this issue open until a maintainer inspects the live `pixverse` skill and publisher records, then either performs an audited support action or scopes a publisher-aware reclaim/transfer path that fits the org ownership roadmap.
