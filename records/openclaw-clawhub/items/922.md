## Summary

Keep this issue open. The ClawHub policy only permits automated closure for PRs that are certainly implemented on current main, and current main still leaves this as a maintainer support case: soft-deleted skill slugs remain globally occupied, publishing under a different company publisher is rejected, and the admin reclaim path is user-handle based rather than a self-service personal-to-company slug reuse flow.

## What This Changes

The issue asks how to reuse the deleted `pixverse` skill slug from a personal account under a company account.

## Best Possible Solution

Keep this open for maintainer support. A maintainer should inspect the live `pixverse` records and, if appropriate, use or extend an admin/support path to move, restore, reclaim, or otherwise resolve the slug for the company publisher with audit trail intact. Longer term, document deleted-slug behavior and design a publisher-aware reclaim or transfer flow that covers personal-to-company ownership without weakening authorization.
