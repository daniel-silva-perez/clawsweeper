## Summary

Keep this issue open. It is an open ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main also still supports the reported support gap: skill slugs are stored behind a global `by_slug` index, soft delete keeps the slug row, publishing to a different publisher is rejected as a collision, and the available reclaim tooling is admin/user-handle oriented rather than a self-service personal-to-company publisher transfer flow.

## Best Possible Solution

Keep this issue open for maintainer support. A maintainer should inspect the live `pixverse` skill, personal user, company publisher, memberships, and audit records, resolve the immediate ownership request if appropriate, and then either document the deleted-slug behavior or implement a publisher-aware reclaim/transfer flow that can safely move a soft-deleted personal slug to an authorized company publisher.
