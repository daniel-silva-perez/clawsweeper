## Summary

Keep open. This is an open ClawHub issue, and repository policy only permits automated closure for pull requests that current main certainly implements. Current main still leaves the reported personal-to-company slug reuse blocked: skill slugs are globally indexed, soft deletion does not free or transfer the slug, publishing rejects a slug already owned by a different publisher, and the available reclaim/transfer flows are user-handle or user-to-user oriented rather than a company-publisher reclaim path.

## What This Changes

The reporter asks how to reuse the deleted personal-account `pixverse` skill slug from a company account.

## Best Possible Solution

A maintainer should resolve the live `pixverse` ownership collision with an audited support action or a deliberately scoped publisher-aware reclaim/transfer path, then document deleted-slug behavior and align it with the planned scoped org slug model.
