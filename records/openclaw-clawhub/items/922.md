## Summary

Keep this issue open. The ClawHub cleanup policy only permits automated closure for PRs that are certainly implemented on current main, and this item is an issue. The current code still supports the reported support gap: skill slugs are globally keyed, soft deletion keeps the slug row, publish to a different owner publisher is rejected when the slug already exists, and the available reclaim/transfer paths are admin or user-transfer oriented rather than a self-service personal-to-company slug reuse flow.

## Best Possible Solution

Keep this open for maintainer support. A maintainer should inspect the live pixverse records and, if appropriate, use an admin/support path to move or reclaim the slug for the company publisher. Longer term, either document deleted-slug behavior clearly or design a safe publisher-aware reclaim/transfer flow that covers personal publisher to org publisher without weakening authorization or auditability.
