## What This Changes

The issue asks maintainers to re-review whether the `ai-replace-me-check` skill was incorrectly flagged by ClawHub/OpenClaw security assessment despite being a read-only SOP extraction and documentation skill.

## Best Possible Solution

Maintainers should decide whether the live `requires-sensitive-credentials` tag is an expected conservative capability signal or a false positive from negated API-key wording. If it is a false positive, the end state should be a narrowly calibrated capability matcher with regression tests for negated and affirmative credential requirements, plus the appropriate production rederive/backfill or manual correction for this skill.
