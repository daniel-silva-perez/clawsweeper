## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy for this lane does not auto-close issues. Substantively, the live skill is not publicly moderated and the scanners are clean, but the public scan still reports `hasWarnings:true` with `capabilityTags:["requires-sensitive-credentials"]`; current main still derives that tag from raw credential regex matches without negation handling, so the appeal needs maintainer security/moderation judgment.

## What This Changes

The issue asks maintainers to re-review the `ai-replace-me-check` skill because the reporter says it is a read-only SOP extraction/documentation skill that does not request credentials, execute commands, or upload data.

## Best Possible Solution

A maintainer should decide whether `requires-sensitive-credentials` is expected for this skill. If it is a false positive, the end state should be conservative scanner calibration with tests for both negated and positive credential requirements, followed by the appropriate production rescan/backfill or manual tag correction for the affected version.
