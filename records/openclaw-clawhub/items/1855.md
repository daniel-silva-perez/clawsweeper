## Summary

Keep open. Current main still does not implement this PR's central behavior: `clawhub inspect` accepts the skill detail API moderation field in the schema, but omits it from JSON output and never prints a plain-text skill moderation summary. The PR remains a useful draft implementation candidate for the related open CLI security request, #1483.

## Best Possible Solution

Keep this PR open as the implementation candidate for #1483. The best path is to review and land this PR or an equivalent change that passes `skillResult.moderation` through inspect JSON, prints a concise plain-text moderation summary, and keeps the version security schema aligned with the v1 version response.
