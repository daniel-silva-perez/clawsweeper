## Summary

Keep open. Current main still does not implement this PR's central behavior: `clawhub inspect` accepts skill-detail moderation in the API schema, but does not include it in inspect JSON output or print a plain-text skill-level moderation summary. The PR remains a relevant draft implementation candidate for #1483.

## Best Possible Solution

Keep this PR open and review/land it, or an equivalent change, as the implementation path for #1483. The fix should pass `skillResult.moderation` through inspect JSON, print a concise skill-level moderation summary in normal inspect output without requiring a version fetch, type version-level `security` metadata in the CLI schema, and keep focused regression tests.
