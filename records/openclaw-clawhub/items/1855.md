## Summary

Keep open. Current main still does not implement this PR's intended CLI behavior: `clawhub inspect <slug> --json` omits the API's skill-level `moderation` block, and ordinary `clawhub inspect <slug>` only prints version-level security when a version fetch occurs. The PR remains a focused draft implementation path for the open #1483 CLI security visibility request.

## What This Changes

The PR updates `clawhub inspect` to print skill-level moderation in text output, preserve `moderation` in JSON output, type version-level `security` metadata, and add inspect regression tests.

## Best Possible Solution

Keep this draft PR open for maintainer review, or land an equivalent change that passes `skillResult.moderation` through inspect JSON, prints a concise skill-level moderation summary in ordinary `clawhub inspect <slug>`, types version-level `security` in the CLI schema, and includes focused regression tests for both plain and JSON output.
