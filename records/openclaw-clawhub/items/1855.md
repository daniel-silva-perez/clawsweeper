## Summary

Keep this PR open. Current main still does not implement the PR's intended behavior: `clawhub inspect` can parse skill-level moderation from the detail API schema, and the backend returns it, but inspect JSON omits it and plain inspect only prints version-level `security` after a version fetch. The PR remains a relevant draft implementation candidate for #1483.

## Best Possible Solution

Keep this PR open and review or land it, or an equivalent change, as the implementation path for #1483. The final change should pass `skillResult.moderation` through inspect JSON, print a concise plain-text skill moderation summary without requiring a version fetch, keep version `security` typed in the CLI schema, and preserve focused regression coverage.
