## Summary

Keep open. Current main still does not implement the PR’s central behavior: `clawhub inspect` JSON omits the v1 skill-detail `moderation` block, and ordinary plain output only prints version-level `security` when a version fetch is requested. The backend and CLI schema already expose skill-level moderation, so this draft remains a focused implementation path for #1483. The diff is limited to CLI inspect output, CLI schema typing, and tests; I did not find a concrete blocking bug or supply-chain/security regression in the proposed patch surface.

## What This Changes

The PR passes skill-level moderation through `clawhub inspect`, prints moderation details in plain output, preserves `moderation` in JSON output, types version-level `security`, and adds inspect regression tests.

## Best Possible Solution

Land this PR or an equivalent change after rebase so `clawhub inspect <slug>` exposes the skill-level moderation returned by the v1 detail API in both JSON and ordinary text output, while keeping backend visibility rules as the source of truth and retaining focused regression coverage.
