## Summary

Keep open. Current main at 7bef2a0 still does not implement this PR's intended CLI inspect behavior: ordinary inspect output does not print skill-level moderation, inspect JSON omits the top-level moderation object returned by the v1 skill detail API, and the CLI version response schema still does not type version.security. The PR remains a focused draft implementation path for #1483 and needs maintainer review rather than auto-close.

## What This Changes

The PR adds skill-level moderation display to `clawhub inspect`, includes moderation in inspect JSON output, types version security metadata in the CLI schema, and adds focused inspect regression tests.

## Best Possible Solution

Keep this draft PR open for maintainer review, or land an equivalent change that passes `skillResult.moderation` through inspect JSON, prints a concise skill-level moderation summary in ordinary `clawhub inspect <slug>`, types version-level `security` in the CLI schema, and includes focused regression tests for both plain and JSON output without changing backend moderation policy.
