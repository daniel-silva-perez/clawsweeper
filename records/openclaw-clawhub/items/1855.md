## Summary

Keep open. Current main has backend/API support for skill-level moderation, and the CLI response schema accepts the skill detail moderation block, but `clawhub inspect` still omits that data from JSON output and has no plain-text moderation summary. This draft PR remains useful work toward open #1483 rather than something already implemented on main.

## Best Possible Solution

Keep this PR open as the implementation candidate for #1483. The best path is to review and land this PR or an equivalent change that passes `skillResult.moderation` through inspect JSON, prints a concise plain-text moderation summary without requiring a version fetch, keeps the local CLI schemas aligned with the v1 responses, and considers whether `package inspect` needs a follow-up for the same user-facing security-status request.

## Risks / Open Questions

- Current main still drops skill-level moderation from `clawhub inspect --json`, so scripts cannot filter on clean/suspicious/malicious status through inspect.
- Plain `clawhub inspect <slug>` still requires a separate version fetch to show any security-like output, and that output is version-level rather than the skill-level moderation state requested in #1483.
- The PR is draft and unmerged, so maintainers should still review the exact output contract and ensure it only surfaces moderation details already returned by the public or authenticated API response.
