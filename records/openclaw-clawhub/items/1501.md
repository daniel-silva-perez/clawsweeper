## Summary

Keep this issue open. ClawHub cleanup policy keeps issues open in this pass, and current main only resolves the CLI payload symptom for personal-user transfers; the backend transfer model is still centered on `fromUserId`/`toUserId`, so a transfer to an org or other non-user publisher remains unresolved and is covered by open #1675.

## What This Changes

The issue requests that `clawhub transfer request baidu-netdisk-storage BaiduNetdiskAIBot --message "to baidu skill"` create an ownership transfer instead of failing with `toUserHandle required`.

## Best Possible Solution

For a personal-user recipient, maintainers should verify the reporter can retry with the current published CLI/backend and then close manually. For an org or non-user publisher recipient, the durable fix belongs in #1675: model transfer targets as publishers, enforce membership-based acceptance, and preserve existing skill stats and canonical owner URLs.
