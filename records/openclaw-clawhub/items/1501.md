## Summary

Keep this issue open. Current main appears to fix the direct user-to-user CLI/API payload path, but this is an open ClawHub issue and the repository cleanup policy only allows auto-close for PRs. There is also a real remaining product boundary if the target handle is an org/publisher rather than a personal user publisher, which is tracked by open related work in #1675.

## Best Possible Solution

Keep this issue open for maintainer triage. Maintainers should verify the reporter's installed CLI and current deployed backend behavior for the exact command, ask them to retry on `clawhub` 0.11.0 or newer if this was the user-to-user payload bug, and add a parser-level CLI regression for the reported invocation. If the destination is an org/publisher account, route the product work through #1675 by migrating transfers from user ids to publisher target ids with membership-based acceptance.
