## Summary

Keep open. Current main still contains the broad standalone `swap` crypto matcher and lacks the focused regressions this draft PR adds, so the PR is an active implementation candidate for the related false-positive report rather than work already implemented on main. A security pass found the proposed diff limited to deterministic capability-tag matching and unit tests, with no dependency, workflow, secret, permission, lockfile, release, or artifact-execution changes.

## What This Changes

The PR removes the standalone crypto `swap` regex, adds contextual swap regexes, and adds regression tests for Google Fonts `display=swap` and token-swap wording.

## Best Possible Solution

Capability tagging should ignore generic `display=swap` text while still tagging unambiguous crypto/token swap content, with regression coverage for both cases; any historical tag cleanup should be handled separately if needed.
