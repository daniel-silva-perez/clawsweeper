## Summary

Keep open. The current source package has moved to `clawhub` and includes `undici`, but the live legacy `clawdhub@latest` npm package still resolves to `0.3.0`, whose metadata omits `undici` while its published build imports it. This needs maintainer registry/release handling, and ClawHub policy keeps issues open for maintainer judgment.

## What This Changes

The issue requests adding `undici` to the published `clawdhub` package dependencies and releasing a patched version so fresh npm/npx installs do not fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

A maintainer should resolve the legacy npm package state by publishing or retagging a stable patched `clawdhub` release, or by deprecating `clawdhub@0.3.0` with clear migration guidance to the current `clawhub` package, then close duplicate reports against the chosen canonical item.
