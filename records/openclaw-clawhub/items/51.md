## Summary

Keep open. Current `main` and the renamed `clawhub` npm package declare `undici`, but the reported legacy npm package state still appears unresolved: `clawdhub@latest` points to `0.3.0`, that metadata omits `undici`, and the published built HTTP client imports it. The ClawHub repository policy also keeps issues open unless a maintainer handles them explicitly.

## What This Changes

The issue reports that fresh installs of `clawdhub@0.3.0` fail because the published CLI imports `undici` from `dist/http.js` without declaring `undici` in package dependencies.

## Best Possible Solution

A maintainer should resolve the legacy npm package state by choosing a canonical report, publishing or retagging a stable patched `clawdhub` release, or deprecating `clawdhub@0.3.0` with clear migration guidance to the current `clawhub` package.
