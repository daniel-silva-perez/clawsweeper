## Summary

Keep this issue open. Current `main` and the renamed `clawhub` npm package include `undici`, but the reported legacy package still appears unresolved: live npm metadata has `clawdhub@latest` pointing at `0.3.0`, whose metadata omits `undici` while the published `dist/http.js` imports it. The repository policy also keeps ClawHub issues open unless a maintainer handles them explicitly.

## What This Changes

The issue requests adding the missing `undici` runtime dependency to the published legacy `clawdhub@0.3.0` package and shipping a patched release so fresh global/npx installs do not fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

Keep this open for maintainer npm registry/release triage. Pick a canonical report across #23, #33, and this issue, then either publish a stable patched `clawdhub` release, move the `clawdhub` `latest` dist-tag to a patched version, or deprecate `clawdhub@0.3.0` with clear migration guidance to `clawhub`.
