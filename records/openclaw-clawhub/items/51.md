## Summary

Keep this issue open. Current main and the renamed `clawhub` npm package include `undici`, but the reported legacy npm path remains affected: `clawdhub@latest` still resolves to `0.3.0`, whose metadata omits `undici` while its published `dist/http.js` imports it. Repository cleanup policy for openclaw/clawhub also says issues should stay open unless they are PRs certainly implemented on main.

## Best Possible Solution

Keep this issue open for maintainer npm registry/release triage. Pick a canonical report across #23, #33, and this issue, then either publish a stable patched `clawdhub` release, move or deprecate the legacy `clawdhub` package with clear migration guidance to `clawhub`, or otherwise update npm dist-tag/deprecation metadata so `clawdhub@latest` no longer points users at the undeclared-`undici` `0.3.0` package.
