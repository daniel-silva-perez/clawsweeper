## Summary

Keep this issue open. Current main and the renamed `clawhub` npm package include `undici`, but the reported legacy npm package remains unresolved: live registry metadata still has `clawdhub@latest` pointing to `0.3.0`, whose package metadata omits `undici` while the published `dist/http.js` imports it. ClawHub cleanup policy also says issues should stay open unless they are PRs certainly implemented on main.

## What This Changes

The issue asks maintainers to add `undici` to the legacy `clawdhub@0.3.0` npm dependencies and publish, retag, or otherwise replace the broken latest release so fresh global or npx installs no longer fail with `ERR_MODULE_NOT_FOUND`.
