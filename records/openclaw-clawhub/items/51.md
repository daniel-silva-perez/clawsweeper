## Summary

Keep open. The report is still valid for the legacy `clawdhub` npm package: current source has moved to `clawhub` and declares `undici`, but live `clawdhub@latest` still resolves to `0.3.0`, whose metadata omits `undici` while its published build imports it. ClawHub policy also keeps issues open unless maintainers handle them explicitly.

## What This Changes

The issue requests adding `undici` to the published `clawdhub` CLI package dependencies and publishing or retagging a patched release so clean npm, npx, and pnpm installs do not fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

A maintainer should resolve the legacy npm package state by publishing or retagging a stable patched `clawdhub` release, or by deprecating `clawdhub@0.3.0` with clear migration guidance to the current `clawhub` package, then close the duplicate reports against the chosen canonical item.
