## Summary

Keep this issue open. The ClawHub cleanup policy for this run keeps issues open, and current main still leaves the named owner-publisher skill digest helper as a same-invocation pagination loop. Main does include a useful mitigation that schedules package and skill digest rebuilds as separate background mutations, but the cursor-batched hardening requested by the failure mode is not present, and related #1182 remains the likely consolidation target.

## Best Possible Solution

Keep this issue open or have maintainers consolidate it under #1182. The robust fix should make owner-publisher package and skill digest sync cursor-batched: each internal mutation should accept a cursor, call .paginate() at most once, process that page, and schedule the next invocation with continueCursor. Add regression coverage for an isDone: false skill page.
