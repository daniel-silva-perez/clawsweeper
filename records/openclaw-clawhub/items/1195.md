## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and current main still contains the reported Convex limit shape: owner-publisher package and skill digest rebuilds can call `.paginate()` repeatedly inside one internal mutation, while the scheduled internal mutations have no continuation-cursor arguments. The related reports remain open, so this should be fixed or explicitly consolidated by maintainers rather than closed.

## What This Changes

The issue reports `clawhub publish` failing with Convex's single-paginated-query error in `syncSkillSearchDigestsForOwnerPublisherId` during publisher setup.

## Best Possible Solution

Land one backend fix that processes owner-publisher package and skill digest rebuilds one indexed page per internal mutation invocation, then self-schedules with `continueCursor` until complete, with focused tests for multi-page package and skill paths.

## Repair Work Prompt

Fix the Convex pagination-limit bug behind publish/import failures in owner-publisher digest rebuilds. In `convex/functions.ts`, change both `syncPackageSearchDigestsForOwnerPublisherId` and `syncSkillSearchDigestsForOwnerPublisherId` so each internal mutation invocation performs at most one `.paginate()` call, processes that page, and self-schedules the same internal mutation with `continueCursor` until `isDone`. Extend the internal mutation args and `scheduleOwnerPublisherDigestSync` payloads to carry an optional cursor and keep using the existing `by_owner_publisher` indexes. Update `convex/functions.test.ts` with focused tests proving package and skill multi-page paths schedule continuation work and do not loop multiple pages in one invocation. Keep the fix scoped to owner-publisher digest scheduling; do not replace pagination with unbounded `.collect()`, do not refactor unrelated triggers, and do not change public API behavior. Related refs: #1195, #1182, #1171, #1197, #1205, #1176, #1184, #1175, #1201.
