## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and current main still contains the reported Convex limit shape: owner-publisher digest rebuilds can call `.paginate()` repeatedly inside one internal mutation, while the scheduled jobs have no continuation-cursor arguments. The related reports remain open, so this should be fixed or deliberately consolidated by maintainers rather than closed.

## What This Changes

Publishing a skill fails for affected accounts because the owner-publisher skill digest rebuild can hit Convex's multiple-paginated-query limit during the publish/publisher update path.

## Best Possible Solution

Land a backend fix that processes owner-publisher package and skill digest rebuilds one indexed page per internal mutation invocation, then self-schedules with `continueCursor` until complete, with focused tests for multi-page package and skill paths.

## Repair Work Prompt

Fix the Convex multiple-paginated-query failure in publish/import paths for owner-publisher digest rebuilds. In `convex/functions.ts`, update the owner-publisher package and skill digest internal mutations/helpers so each invocation performs at most one `.paginate()` call, processes that page, and if `!isDone` schedules the same internal mutation with `continueCursor` and the same `ownerPublisherId`. Preserve the existing indexed queries, missing-table behavior, and scheduler-absence behavior. Do not replace pagination with `.collect()` or any unbounded read. Add focused coverage in `convex/functions.test.ts` proving initial scheduling, multi-page package and skill continuation scheduling, and no reschedule when the page is done. Mention the related cluster in the PR summary and keep the change scoped to this backend path unless tests expose the same required adjustment in a directly shared helper.
