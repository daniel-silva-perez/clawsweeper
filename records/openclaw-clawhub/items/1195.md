## Repair Work Prompt

Fix the publish/import Convex failure where owner-publisher digest sync can run multiple `.paginate()` calls in one mutation. In `convex/functions.ts`, update `syncPackageSearchDigestsForOwnerPublisherIdInternal` and `syncSkillSearchDigestsForOwnerPublisherIdInternal` and their helper flow so each invocation processes only one indexed `by_owner_publisher` page, then self-schedules the same internal mutation with `continueCursor` when `!isDone`. Preserve missing-table handling and existing indexed queries. Add focused tests in `convex/functions.test.ts` for multi-page package and skill digest sync, cursor propagation, and the initial scheduler call. Do not replace this with `.collect()`, full-table scans, or unrelated CLI/UI changes. No changelog entry is expected unless maintainers require one for this bugfix.
