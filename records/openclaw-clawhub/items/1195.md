## Summary

Keep open. ClawHub cleanup policy keeps issues open unless they are PRs already implemented on current main, and the current code still has the reported Convex failure shape for owner-publisher digest rebuilds: the skill and package helpers can run multiple `.paginate()` calls in one mutation invocation, with related reports still open around #1182.

## What This Changes

The issue reports `clawhub publish` failing with a Convex multiple-paginated-query error in `syncSkillSearchDigestsForOwnerPublisherId`.

## Best Possible Solution

Keep this issue open unless maintainers explicitly consolidate it under #1182. The concrete fix should make owner-publisher package and skill digest sync cursor-batched: each internal mutation should accept an optional cursor, perform at most one `.paginate()` call, process that page, and schedule itself with `continueCursor` when `isDone` is false, with focused tests for both skill and package multi-page paths.

## Repair Work Prompt

Fix the owner-publisher digest sync Convex pagination bug tracked by #1182 and related reports. In `convex/functions.ts`, refactor the owner-publisher package and skill digest internal mutations so each invocation performs at most one `.paginate()` call, accepts an optional continuation cursor, processes only that page, and schedules itself with `continueCursor` when `isDone` is false. Do not replace this with unbounded `.collect()`. Keep package and skill jobs separate. Prefer the existing self-scheduling style used elsewhere, such as `convex/lib/batching.ts`, if it fits. Add focused tests in `convex/functions.test.ts` for both skill and package owner-publisher multi-page paths: first page `isDone: false`, one paginate call per invocation, continuation job scheduled with cursor, and existing initial scheduling/no-scheduler behavior preserved. Also inspect `syncPackageSearchDigest`/`getPreferredFallbackPackageRelease` so the package owner-publisher batch does not accidentally perform a second paginated query inside the same mutation. No unrelated refactors or release metadata changes.
