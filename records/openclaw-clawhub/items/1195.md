## Summary

Keep this issue open. The repository policy for openclaw/clawhub says this cleanup pass should only auto-close pull requests that are certainly implemented on current main, and this item is an issue. Source review also shows current main still has the named owner-publisher skill digest helper looping over `.paginate()` inside one mutation without a cursor continuation API, so the reported failure mode is not conclusively fixed. Related #1182 remains open and looks like the best canonical consolidation target if maintainers want one.

## Best Possible Solution

Keep this issue open unless maintainers explicitly consolidate it under #1182. The robust fix should make owner-publisher package and skill digest sync cursor-batched: each internal mutation should accept a cursor, call `.paginate()` at most once, process that page, and schedule the next invocation with `continueCursor`. Add regression coverage for an `isDone: false` skill page.
