## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy says ClawHub issues should remain open in this pass. Current main has adjacent moderation overrides, official/highlighted/redaction-approved badge plumbing, trusted-publisher machinery, and docs saying skills are not paid, but it does not implement a paid expedited human security-review pipeline, certificate, or distinct reviewed-and-approved skill badge.

## Best Possible Solution

Keep this issue open for maintainer product and security judgment. A complete implementation would define whether paid expedited review is desired, then add a request/payment/status workflow, reviewer assignment and audit trail, version-scoped review records, public certificate or badge semantics, and user-facing docs explaining the review scope and limits.

## Risks / Open Questions

- A paid review badge needs maintainer product, security, and legal policy decisions so users understand exactly what the badge guarantees and what it does not.
- The design should be coordinated with #192 plus existing official, trusted-publisher, redaction-approved, and moderation labels to avoid overlapping or contradictory trust signals.
- Any certification should likely be version-scoped and audit-backed, because a skill can publish new versions after review.
