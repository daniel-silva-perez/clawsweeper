## Summary

Keep open. The ClawHub cleanup policy does not auto-close issues, and current main still leaves a concrete Convex hardening gap: owner-publisher package and skill digest rebuilds were split into separate scheduled mutations, but each scheduled mutation can still run more than one `.paginate()` call when an owner has more than one 100-item page.

## What This Changes

The issue reports `clawhub publish` failing for authenticated CLI users with a Convex multiple-paginated-query error in the personal publisher and search digest sync path.

## Best Possible Solution

Implement cursor-based continuation for owner-publisher package and skill digest rebuilds: each scheduled internal mutation should process one indexed page, update digests for that page, and reschedule itself with `continueCursor` until complete. After deployment verification, consolidate the related publish-pagination reports against the shipped fix.
