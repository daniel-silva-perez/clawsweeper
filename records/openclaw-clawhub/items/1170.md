## Summary

Keep open. This is an open ClawHub issue, and the supplied cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Current main partially mitigates the original inline trigger failure by scheduling package and skill owner digest rebuilds separately, but each scheduled helper still loops over `.paginate()` pages inside one mutation, so large owner catalogs can still hit Convex's single-paginated-query limit.

## Best Possible Solution

Keep this issue open and land a narrow Convex hardening PR that changes the owner-publisher package and skill digest rebuilds to process one paginated page per scheduled internal mutation, then reschedule with the returned cursor until complete. Preserve the existing separate package/skill scheduling mitigation, add regression tests for multi-page continuation, and consolidate the related publish-pagination reports after deployment verification.
