## Summary

Keep open. This is an open ClawHub issue, and the supplied repository cleanup policy only permits automated closure for pull requests that are certainly implemented on current main. Current main mitigates the original inline publisher-trigger conflict by scheduling package and skill digest rebuilds separately, but the owner-publisher digest helpers still contain multi-page `.paginate()` loops, so a large-owner failure mode remains and related publish-pagination reports are still open.

## Best Possible Solution

Keep this issue open and land a narrow Convex hardening PR that makes owner-publisher package and skill digest sync process one page per scheduled internal mutation, then reschedule with the returned cursor until complete. Preserve the existing separate package/skill scheduling mitigation, add regression tests for multi-page pagination, deploy, and then consolidate related publish-pagination reports after production verification.
