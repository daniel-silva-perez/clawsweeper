## Summary

Keep open. This is an open ClawHub issue, and the repo-specific cleanup policy only allows auto-closing pull requests that current main certainly implements. Current main mitigates the original publisher-trigger shape by scheduling package and skill digest rebuilds separately, but each scheduled internal mutation still loops over `.paginate()` until completion, so a multi-page owner catalog can still hit Convex's single-pagination limit.

## What This Changes

The issue requests fixing CLI publish failures caused by Convex's `multiple paginated queries` error in the backend publisher digest sync path.

## Best Possible Solution

Implement cursor-based continuation for owner-publisher package and skill digest rebuilds: each scheduled internal mutation should process one indexed page, update digests for that page, and reschedule itself with the returned cursor until complete, then use that shipped fix to consolidate the duplicate publish-blocking reports.
