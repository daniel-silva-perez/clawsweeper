## Summary

Keep open. ClawHub cleanup policy keeps open issues open, and current main still has a concrete residual pagination failure path: the publish flow reaches personal publisher creation and publisher digest scheduling, while each scheduled owner-publisher digest mutation can still loop over `.paginate()` pages inside one Convex mutation.

## What This Changes

The issue asks to fix authenticated `clawhub publish` failures caused by Convex's single-pagination constraint in publisher digest sync during personal publisher creation.

## Best Possible Solution

Keep this issue open until the backend digest rebuilds process one owner-publisher page per internal mutation and self-reschedule with the returned cursor, with regression coverage and deployment verification before consolidating the related publish-pagination reports.

## Repair Work Prompt

Fix the owner-publisher digest sync pagination bug reported by the CLI publish failures. In `convex/functions.ts`, change `syncPackageSearchDigestsForOwnerPublisherIdInternal` and `syncSkillSearchDigestsForOwnerPublisherIdInternal` so each internal mutation processes exactly one indexed `by_owner_publisher` page, then schedules itself with the returned cursor until `isDone`. Preserve the current package/skill split from `scheduleOwnerPublisherDigestSync`. Do not replace pagination with unbounded `.collect()`, do not change CLI publish semantics or API request/response schemas, and avoid unrelated refactors. Add regression coverage in `convex/functions.test.ts` for a multi-page owner catalog: first invocation processes one page, reschedules with `continueCursor`, and does not call `.paginate()` a second time in the same mutation. Regenerate Convex types if internal function args change. No changelog entry is expected unless the repo has a nearby convention; include the related issue refs and commands run in the PR summary.
