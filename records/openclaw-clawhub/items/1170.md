## Summary

Keep open. This is an open ClawHub issue, and the repo-specific cleanup policy only allows auto-closing pull requests that are certainly implemented on current main. Current main likely mitigates the original publisher-trigger failure by scheduling package and skill digest rebuilds separately, but each scheduled owner-publisher digest mutation still loops over `.paginate()` pages in one Convex mutation, so a large owner catalog can still hit the same Convex single-pagination limit.

## What This Changes

CLI publish should not fail with Convex's multiple-paginated-query error while ensuring a personal publisher and syncing owner-publisher search digests.

## Best Possible Solution

Keep this issue open and land a narrow Convex hardening PR that processes exactly one owner-publisher package or skill digest page per scheduled internal mutation, then reschedules with the returned cursor until complete. Preserve indexed `by_owner_publisher` reads, do not replace pagination with unbounded `.collect()`, add multi-page regression coverage, and consolidate the related publish-pagination reports after deployment verification.
