## Summary

Keep open. This is an open ClawHub issue, and the repo-specific cleanup policy only allows auto-closing pull requests that current main certainly implements. Source inspection also shows a remaining Convex pagination hardening gap: owner-publisher digest rebuilds are scheduled separately, but each scheduled mutation can still loop through multiple `.paginate()` calls for owners with more than one page of packages or skills.

## What This Changes

The issue reports `clawhub publish` failing with Convex's multiple-paginated-queries error during the personal-publisher publish path.

## Best Possible Solution

Implement cursor-based continuation for owner-publisher package and skill digest rebuilds: each scheduled internal mutation should process one indexed page, update digests for that page, and reschedule itself with `continueCursor` until complete. After deployment verification, consolidate the duplicate publish-pagination reports against the shipped fix.
