## Summary

Keep open. This is an open ClawHub issue, and the repo-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Current main partially mitigates the original publisher-trigger shape by scheduling package and skill digest rebuilds separately, but each scheduled mutation still loops through `.paginate()` pages in one Convex mutation, so owners with more than one page of packages or skills can still hit the same single-paginated-query Convex limit.

## What This Changes

The issue reports `clawhub publish` failing after login with Convex's single-paginated-query error while creating or syncing publisher search digests.

## Best Possible Solution

Keep this issue open and land a narrow Convex hardening PR that processes exactly one owner-publisher package or skill digest page per scheduled internal mutation, then reschedules with the returned cursor until complete. Preserve the current indexed `by_owner_publisher` reads, avoid replacing pagination with unbounded `collect()`, add regression coverage for multi-page continuation, and consolidate the related publish-pagination reports after deployment verification.
