## Summary

Keep open. This ClawHub cleanup lane only auto-closes PRs that are already implemented on main, and the live production skill still reports a suspicious moderation verdict and is excluded by the non-suspicious search filter. Current main has the APIs and moderator override tooling needed to handle the appeal, but the remaining decision is human moderation/security judgment, not repository cleanup.

## What This Changes

The reporter requests manual review and clearing of the suspicious flag on the published `vcf-loginsight-sddc-errors` skill, which uses `curl` and `jq` with `LOGINSIGHT_HOST` and `LOGINSIGHT_API_TOKEN` to query VMware Aria Operations for Logs.

## Best Possible Solution

A ClawHub moderator should review the live `SKILL.md`, VirusTotal/ClawScan evidence, and publisher context, then either apply the existing clean manual override with an audit note or ask the publisher to remove or justify `curl -k`, tighten host/token handling, publish a safer version, and request a rescan.

## Security Review

Status: needs_attention

Summary: This non-PR issue is security-sensitive because the requested action would clear a live suspicious flag on a token-bearing shell-command skill.

Concerns:

- **[medium] Review TLS bypass before clearing the flag:**
  - body: The published skill sends `LOGINSIGHT_API_TOKEN` as a bearer token while using `curl -k`, which disables TLS certificate verification and can expose the token on misconfigured or untrusted networks.
  - confidence: 0.94
- **[low] Review dynamic host token forwarding:**
  - body: The skill sends the bearer token to the host supplied by `LOGINSIGHT_HOST`. That may be expected for user-owned VMware environments, but staff should verify the trust model before marking the artifact clean.
  - confidence: 0.84
