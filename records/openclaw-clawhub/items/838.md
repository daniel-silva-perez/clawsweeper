## Summary

Keep open. This is a ClawHub issue, and the supplied repository policy only permits auto-closing ClawHub pull requests that are definitely implemented on current main. The report also remains substantively valid: current main can still show the top suspicious banner from skill-level moderation while VT and OpenClaw scan cards are clean, and production currently reports web-search-pro v2.1.4 as suspicious because of static `suspicious.potential_exfiltration` findings.

## Best Possible Solution

Keep this issue open for maintainer moderation and UI follow-up. Maintainers should decide whether the current `suspicious.potential_exfiltration` finding for web-search-pro is valid; if valid, expose the static reason near the top banner or make the scan-card presentation clearly include static analysis as the unresolved source. If it is a false positive, clear it through the moderation override/resync path or broaden scanner calibration with focused regression coverage. If desired, consolidate this class of banner-confusion reports with related open reports such as #450, #733, or #768.
