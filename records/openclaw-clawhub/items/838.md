## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are definitely implemented on main. Substantively, current main still supports the reported mismatch: the top banner is keyed off skill-level moderation state, while the scan cards are keyed off latest-version VT/OpenClaw scan fields. Production also still reports web-search-pro as suspicious for `suspicious.potential_exfiltration`, so this needs maintainer moderation/UI follow-up rather than cleanup closure.

## Best Possible Solution

Keep this issue open for maintainer moderation review. Maintainers should decide whether the current `suspicious.potential_exfiltration` finding for web-search-pro is valid. If valid, the skill page should surface or link the unresolved moderation reason near the banner when scan cards look benign; if it is a false positive, clear it through the existing moderator override/resync/rescan path. If the team wants one tracker for this broader confusing-banner class, consolidate with the related open reports such as #450, #733, or #768.
