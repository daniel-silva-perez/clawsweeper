## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. The report also remains substantively valid enough for maintainer follow-up: current main still derives the top banner from skill-level moderation flags while the visible scanner rows come from latest-version scan fields, and production currently reports web-search-pro v2.1.4 as suspicious because of `suspicious.potential_exfiltration`.

## Best Possible Solution

Keep this open for maintainer moderation and UI follow-up. Maintainers should decide whether the current `suspicious.potential_exfiltration` finding for web-search-pro v2.1.4 is valid. If valid, surface the static-analysis reason clearly near the top banner or in the security summary so the page does not appear contradictory. If it is a false positive, clear it through the moderator override/rescan path or adjust scanner calibration with focused regression coverage.
