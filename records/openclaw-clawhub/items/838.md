## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are definitely implemented on main. Substantively, current main and production still support the reported confusing state: the top banner is driven by skill-level moderation, while scan cards are driven by latest-version scan fields, and production still reports web-search-pro as suspicious for `suspicious.potential_exfiltration`.

## Best Possible Solution

Keep this issue open for maintainer moderation/UI follow-up. Maintainers should decide whether the current `suspicious.potential_exfiltration` finding for web-search-pro is valid. If valid, the skill page should make that unresolved moderation source visible near the banner; if it is a false positive, clear it through the existing moderator override or rescan path. If the team wants one tracker for this broader banner confusion class, consolidate with related open reports such as #450, #733, or #768.
