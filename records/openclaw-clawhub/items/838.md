## Summary

Keep open. The ClawHub cleanup policy only permits auto-closing pull requests that are certainly implemented on current main, and this is an issue. Substantively, current main and the live API still show the reported split: the top banner is driven by skill-level moderation state, while the scan rows are latest-version scanner data; web-search-pro now shows moderation suspicious for `suspicious.potential_exfiltration` with VT and ClawScan clean/benign and static analysis suspicious.

## Best Possible Solution

Keep this issue open for maintainer moderation and security UI follow-up. Maintainers should decide whether the `suspicious.potential_exfiltration` finding for web-search-pro is valid. If it is valid, surface the reason/source near the top banner or security summary so the page does not read as contradictory. If it is a false positive, clear it through the manual override/rescan path or adjust the static scanner calibration with focused regression coverage for this pattern and the related #450/#733/#768 false-positive cluster.
