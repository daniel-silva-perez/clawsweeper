## Summary

Keep open. Current main appears to address the core code-level GitHub rename path by using the immutable GitHub provider ID and syncing mutable profile fields after sign-in, but ClawHub cleanup policy keeps issues open and the related production/account-state follow-up in #1109 is still open.

## What This Changes

The issue requests that GitHub OAuth sign-in continue to work after a user changes their GitHub username by keying accounts on the immutable GitHub user ID rather than the mutable login name.

## Best Possible Solution

The desired end state is production sign-in keyed by GitHub numeric provider ID, automatic sync of mutable GitHub login/avatar/profile fields for active accounts, and a maintainer-owned recovery/consolidation path for any existing account rows whose authAccounts or deleted/deactivated state still prevents reauth.
