## Summary

Keep open. Current main appears to cover the code-level GitHub rename path by using GitHub's immutable numeric ID and syncing mutable profile/publisher fields, but the ClawHub cleanup policy keeps issues open and the related open report #1109 still needs production/account-state verification.

## What This Changes

The issue requests that GitHub sign-in continue to recognize the same account after a GitHub username rename by using the immutable GitHub user ID instead of the mutable username.

## Best Possible Solution

Production should consistently authenticate GitHub users by numeric providerAccountId, refresh mutable GitHub login/avatar/profile and personal-publisher fields for active accounts, and provide a maintainer-owned recovery or consolidation path for stale, deleted/deactivated, duplicate, or verifier/session state that still blocks reauth.
