## Summary

Keep open. Current main covers the code-level stable GitHub ID and profile-sync path, and the closed unmerged #179 proposal is now effectively represented in main, but this is a ClawHub issue and the review policy keeps issues open. The related #1109 report is also still open for production/account-state verification of the same username-change login symptom.

## What This Changes

The issue requests that GitHub OAuth sign-in keep recognizing the same account after a GitHub username change by keying lookup on the immutable GitHub user ID and updating mutable username fields.

## Best Possible Solution

Production should consistently authenticate GitHub users by numeric providerAccountId, refresh mutable GitHub login/avatar/profile fields for active accounts, and provide a maintainer-owned recovery or consolidation path for stale, deleted/deactivated, or duplicate auth/user rows that still block reauth.
