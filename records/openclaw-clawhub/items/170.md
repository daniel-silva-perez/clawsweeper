## Summary

Keep open. Current main covers the core code-level rename path by mapping GitHub OAuth to the immutable numeric profile ID and syncing mutable GitHub profile fields after sign-in, but this is an open ClawHub issue and the cleanup policy only permits auto-closing PRs. The related open report #1109 also keeps the production/account-state side unresolved.

## What This Changes

The issue reports GitHub OAuth sign-in failing after a GitHub username rename and asks ClawHub to identify accounts by immutable GitHub user ID while treating the username as mutable profile data.

## Best Possible Solution

Production should consistently authenticate GitHub users by numeric providerAccountId, refresh mutable GitHub login/avatar/profile fields for active accounts, and provide a maintainer-owned recovery or consolidation path for stale, deleted/deactivated, duplicate, or verifier/session state that still blocks reauth.
