## Summary

Keep open. Current main covers the core code path for GitHub renames by using GitHub's numeric OAuth ID and syncing mutable profile fields, but this is an open ClawHub issue and the supplied cleanup policy only permits auto-closing PRs. The remaining symptom is also still represented by open related report #1109, so this needs maintainer-owned production/account-state triage rather than conservative closure.

## What This Changes

The issue requests that GitHub sign-in continue working after a user changes their GitHub username by keying account lookup on the immutable GitHub user ID and refreshing mutable username/profile fields.

## Best Possible Solution

Production should authenticate GitHub users by numeric providerAccountId, refresh mutable GitHub login/avatar/profile and personal-publisher fields for active accounts, and provide a maintainer-owned recovery or consolidation path for stale, deleted/deactivated, duplicate, or verifier/session state that still blocks reauth.
