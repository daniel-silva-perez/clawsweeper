## Summary

Keep open. Current main appears to cover the core code-level GitHub rename path by using the immutable GitHub provider ID and syncing mutable profile fields, but ClawHub cleanup policy keeps issues open and the related open report #1109 still needs production/account-state verification.

## What This Changes

The issue reports that GitHub OAuth sign-in stops working after a user changes their GitHub username and asks ClawHub to key account lookup on the immutable GitHub user ID while treating the login as mutable profile data.

## Best Possible Solution

Production should consistently authenticate GitHub users by numeric providerAccountId, refresh mutable GitHub login/avatar/profile fields for active accounts, and provide a maintainer-owned recovery or consolidation path for stale, deleted/deactivated, duplicate, or verifier/session state that still blocks reauth.
