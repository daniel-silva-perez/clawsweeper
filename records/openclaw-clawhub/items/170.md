## Summary

Keep this issue open. Current main appears to implement the core code-level GitHub rename handling by using the immutable GitHub provider ID and syncing mutable profile fields after sign-in, but the ClawHub cleanup policy for this pass keeps issues open and the related production/account-state follow-up in #1109 remains open.

## What This Changes

After a GitHub username change, ClawHub should continue recognizing the same GitHub account by immutable GitHub user ID and update mutable username/profile data instead of breaking sign-in.

## Best Possible Solution

Keep this issue open for maintainer follow-up. Maintainers should verify production is deployed with the numeric-provider-ID OAuth/profile-sync code, inspect the affected authAccounts.providerAccountId and users.name/handle/githubProfileSyncedAt/deletedAt/deactivatedAt/purgedAt rows, then either close manually as implemented or consolidate remaining production/account-state work under #1109.
