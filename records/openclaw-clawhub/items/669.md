## Summary

Keep open. Repository policy for this ClawHub pass only permits auto-closing PRs that are certainly implemented on current main, and this item is an issue. Substantively, current main fixes the CLI publish consent problem, but the moderation complaint still needs maintainer/security judgment because clean VirusTotal plus clean ClawScan only suppresses allowlisted static findings, while non-allowlisted findings such as potential exfiltration remain suspicious by design.

## What This Changes

The issue reports that several published skills displayed a blocked or malicious banner despite benign VirusTotal and OpenClaw scan results, and separately notes that CLI v0.7.0 omitted `acceptLicenseTerms` from publish payloads.

## Best Possible Solution

Keep the CLI consent fix, then make an explicit maintainer/security policy decision for skills with clean external scanners but non-allowlisted static findings: document why the warning remains, apply staff manual overrides where review clears specific skills, improve author-facing rescan/appeal guidance, or narrowly expand the allowlist with regression tests and a production backfill/rescan plan.
