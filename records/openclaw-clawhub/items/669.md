## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy for this pass only permits auto-closing PRs that current main certainly implements. The CLI publish consent problem is fixed, but the moderation complaint remains active: current main intentionally keeps non-allowlisted static findings such as `suspicious.potential_exfiltration` suspicious even when VirusTotal and ClawScan/LLM are clean, and the affected public scan endpoints still show that state.

## What This Changes

The issue asks ClawHub to fix CLI publishes that omit `acceptLicenseTerms` and to stop showing blocked or suspicious skill states when VirusTotal and OpenClaw/ClawScan results are benign.

## Best Possible Solution

Keep this issue open for maintainer/security triage. Treat the CLI `acceptLicenseTerms` publish failure as fixed on current main and the `v0.12.0` CLI tag, then make an explicit policy call for clean VT plus clean ClawScan/LLM scans with non-allowlisted static findings such as `suspicious.potential_exfiltration`: document why the warning remains, apply staff manual overrides where appropriate, add author-facing rescan/appeal guidance, or narrowly adjust the allowlist with regression tests and a production backfill/rescan plan.
