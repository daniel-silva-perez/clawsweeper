Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main fixes the CLI `acceptLicenseTerms` publish payload problem, but the moderation concern is still real enough for maintainer policy review: four named skills are no longer malware-blocked in the public API, but they remain suspicious for `suspicious.potential_exfiltration`.

## Best Possible Solution

Keep this issue open for maintainer triage. Treat the CLI `acceptLicenseTerms` failure as fixed in current main/current CLI, then decide the remaining moderation policy: whether `suspicious.potential_exfiltration` should remain warning/suppression-worthy, become externally clearable in narrow benign cases, be handled through manual overrides/rescans, or be documented with clearer author guidance.
