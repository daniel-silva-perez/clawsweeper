## Summary

Keep open. The CLI acceptLicenseTerms part is fixed on current main/current CLI, and the v1 publish API now accepts legacy clients that omit the field. The separate moderation false-positive complaint is not fully resolved in source: current main still preserves most static findings even when VirusTotal and OpenClaw/LLM statuses are clean or benign, the blocked banner is still driven by blocked.malware, and the checkout has no evidence that the named production skill rows were manually reviewed or unblocked. The ClawHub cleanup policy also says open issues should stay open; only pull requests that are certainly implemented on main are eligible for auto-close.

## Best Possible Solution

Keep this issue open for maintainer triage. Treat the CLI acceptLicenseTerms failure as fixed in current CLI/current main, then separately verify the named production skill rows and decide whether moderation policy, scanner calibration, manual override/rescan tooling, backfills, or author-facing scan guidance should change. If maintainers want one tracker for the remaining moderation UX and false-positive class, consolidate with related open reports such as #708 and #1021.
