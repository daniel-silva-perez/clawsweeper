## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and the remaining user-visible problem is not fully implemented away: the CLI consent defect is fixed, but the affected hosted skills can still be flagged as suspicious when VirusTotal and ClawScan are clean because current moderation intentionally preserves non-allowlisted static findings.

## What This Changes

The issue reports that skills published with CLI v0.7.0 missed `acceptLicenseTerms` and that several benign-scanned skills were shown as blocked or malicious on ClawHub.

## Best Possible Solution

Keep the CLI consent compatibility, then make an explicit maintainer/security decision for clean VT and ClawScan results with non-allowlisted static findings: document why the warning remains, apply staff clean overrides where review clears specific skills, or expand the allowlist with targeted regression tests and a production rescan/backfill plan.
