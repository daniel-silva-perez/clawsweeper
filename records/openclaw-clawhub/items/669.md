## Summary

Keep open. The CLI publish-consent part is fixed on current main, and legacy v1 publish payloads without the field are accepted, but the issue also reports production skills showing the blocked-malicious banner despite benign scanner results. Current main still intentionally lets non-allowlisted static suspicious or malicious findings drive moderation even when external scanners are clean, and related false-positive/actionability reports #708 and #1021 remain open. Repository policy for this ClawHub cleanup run also says issues should remain open rather than auto-closed.

## Best Possible Solution

Keep this issue open for maintainer triage. Treat the CLI acceptLicenseTerms failure as already fixed in current CLI/API behavior, then separately verify the affected skill rows and decide whether moderation policy, static-scan allowlisting, manual override/rescan tooling, backfills, or author-facing scan guidance should change. If maintainers want one canonical tracker, consolidate this with the still-open related false-positive/actionability reports such as #708 and #1021.
