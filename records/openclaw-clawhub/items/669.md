## Summary

Keep open. The ClawHub cleanup policy for this pass only permits auto-closing PRs that are certainly implemented on current main, not open issues. Substantively, the CLI `acceptLicenseTerms` publish bug is fixed on main and in the `v0.12.0` tag, but the moderation complaint remains live: current code intentionally keeps non-allowlisted static findings such as `suspicious.potential_exfiltration` suspicious even when VirusTotal and OpenClaw/LLM scanners are clean, and the affected live skill scan endpoints still show that state.

## What This Changes

The issue reports that skills published with CLI v0.7.0 fail without `acceptLicenseTerms` and, after local patching, still display blocked or suspicious banners despite benign VirusTotal and OpenClaw scan results.

## Best Possible Solution

Keep this issue open for maintainer/security triage. Treat the CLI `acceptLicenseTerms` publish failure as fixed on current main and the `v0.12.0` tag, then make an explicit policy call for clean VT + clean OpenClaw/LLM scans with non-allowlisted static findings such as `suspicious.potential_exfiltration`: document why the warning remains, add manual review/override guidance, narrowly adjust the allowlist with tests, or consolidate the work with #328, #708, and #1021.
