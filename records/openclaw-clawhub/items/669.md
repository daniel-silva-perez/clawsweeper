## Summary

Keep open. This is an open ClawHub issue, and the supplied repository policy only permits auto-closing ClawHub pull requests that are definitely implemented on current main. Substantively, current main fixes the CLI `acceptLicenseTerms` publish failure and tolerates legacy clients that omit the field, but the moderation false-positive class is still not fully resolved: current production no longer reports the named skills as malware-blocked, yet four named skills remain suspicious for `suspicious.potential_exfiltration`, which current code and tests intentionally preserve even when external scanners are clean.

## Best Possible Solution

Keep this issue open for maintainer triage. Treat the CLI `acceptLicenseTerms` failure as fixed in current CLI/current main, then decide the remaining moderation policy: whether `suspicious.potential_exfiltration` should remain scanner-blocking, become externally clearable in narrow benign cases, be handled through manual overrides/rescans, or be documented with clearer author guidance. If maintainers want one canonical tracker for this class, consolidate with related open false-positive and scanner-guidance reports such as #328, #708, and #1021.
