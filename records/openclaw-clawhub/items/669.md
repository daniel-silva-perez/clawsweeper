## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy for this review only permits auto-closing ClawHub pull requests that are certainly implemented on current main. The CLI `acceptLicenseTerms` problem is fixed on current main, but the remaining moderation behavior is still live: the affected skills still report clean VirusTotal/OpenClaw scans while retaining a `suspicious.potential_exfiltration` static finding, and current main intentionally keeps non-allowlisted static findings suspicious despite clean external scanners.

## Best Possible Solution

Keep this issue open for maintainer triage. Treat the CLI `acceptLicenseTerms` publish failure as fixed on current main, then decide the moderation policy for benign VirusTotal/OpenClaw results combined with non-allowlisted static findings such as `suspicious.potential_exfiltration`: keep the warning, document it, offer manual review/override guidance, adjust the allowlist narrowly, or consolidate the policy with #328, #708, and #1021.
