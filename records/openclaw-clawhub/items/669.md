## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and the remaining report is still a live moderation/security appeal: current main fixes the CLI license-consent publish defect, but clean VirusTotal and ClawScan results do not automatically clear non-allowlisted static findings such as potential exfiltration.

## What This Changes

The issue reports that `clawhub publish` omitted `acceptLicenseTerms` and that several published skills showed blocked or malicious banners despite benign VirusTotal and OpenClaw scan results.

## Best Possible Solution

Keep the CLI consent compatibility, then make an explicit maintainer/security decision for clean external scanners plus non-allowlisted static findings: document why the warning remains, apply staff clean overrides where review clears specific skills, or expand the allowlist with targeted regression tests and a production rescan/backfill plan.
