## Summary

Keep open. Current main still lacks native per-skill pinning in the CLI, lockfile schema, update guard, list output, and docs. The provided GitHub context also shows open PR #1806 explicitly targeting this issue and still unmerged, so this issue should remain open until that PR or an equivalent maintainer-approved implementation lands.

## Best Possible Solution

Keep this issue open and review/land PR #1806, or an equivalent maintainer-approved patch, that adds `clawhub pin` and `clawhub unpin`, persists pin metadata, shows pin state in `list`, blocks direct pinned updates, skips and reports pinned entries during `update --all`, and honors pins even when `--force` is supplied. Once that PR merges, GitHub can close this issue through the linked closing reference.
