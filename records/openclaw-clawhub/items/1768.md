## Summary

Keep open. Current main still lacks native per-skill pinning in the ClawHub CLI, and open PR #1806 is the active linked implementation candidate for this exact behavior, so the issue should stay open pending maintainer review of that PR rather than being closed or duplicated.

## What This Changes

Request native CLI support for pinning installed skills so `update`, `update --force`, and `update --all` cannot replace operator-frozen skill bytes until the skill is explicitly unpinned.

## Best Possible Solution

Land PR #1806 or an equivalent maintainer-approved patch that adds pin/unpin commands, persists optional pin metadata, shows pin state in `list`, rejects direct updates for pinned skills, skips and reports pinned entries for `update --all`, and honors pins even with `--force`.
