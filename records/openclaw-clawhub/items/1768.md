## Summary

Keep open. Current main does not implement native per-skill pinning: the CLI has `update`/`list` but no `pin` or `unpin`, the update path has no pinned-skill guard, and the lockfile schema only stores `version` plus `installedAt`. The provided GitHub context shows open PR #1806 explicitly targeting this request and still unmerged, so it should be reviewed or closed by maintainers rather than pre-closing this issue.

## Best Possible Solution

Keep this issue open and review/land PR #1806, or an equivalent maintainer-approved patch, that adds native `pin`/`unpin`, persists pin metadata in the lockfile, shows pin state in `list`, blocks direct pinned updates, and skips/reports pinned entries during `update --all` while honoring pins even with `--force`. After that PR merges, the linked closing reference can close this issue normally.
