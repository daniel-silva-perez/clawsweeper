## Summary

Keep open. Current main still does not implement native per-skill pinning, and open PR #1806 is already the active implementation candidate with closing syntax for this issue, so the right cleanup path is maintainer review of that PR rather than closing this issue or queueing duplicate automation.

## What This Changes

The issue requests native ClawHub CLI per-skill pinning that persists pin metadata, shows pin state in list output, blocks direct updates, and skips pinned skills during update --all even when --force is used.

## Best Possible Solution

Land #1806 or an equivalent maintainer-approved patch that adds pin/unpin commands, persists optional pin metadata, shows pinned state in list output, rejects direct updates for pinned skills, skips and reports pinned entries for update --all, honors pins under --force, and updates the focused CLI/schema tests and docs.
