## Summary

Keep open. Current main still lacks native per-skill pinning in the ClawHub CLI, and the provided GitHub context shows open, unmerged PR #1806 specifically targeting this behavior with closing syntax. Because this is an issue in openclaw/clawhub and an active implementation PR is already paired with it, the right cleanup path is maintainer review of PR #1806 or an equivalent patch, not auto-close or a duplicate repair PR.

## What This Changes

The issue requests native ClawHub CLI support for per-skill pinning so pinned installed skills cannot be updated by direct update, --force, or --all until explicitly unpinned.

## Best Possible Solution

Keep this issue open and review/land PR #1806, or an equivalent maintainer-approved patch, that adds clawhub pin/unpin, persists pin metadata, shows pin state in list, blocks direct updates for pinned skills, skips and reports pinned entries during update --all, and honors pins even when --force is supplied. After that work merges, GitHub can close this issue through the linked implementation.
