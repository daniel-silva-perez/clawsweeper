## Summary

Keep this issue open. The ClawHub cleanup policy for this run only auto-closes PRs that are certainly implemented on main, and this is an issue. Current main also still has the reported auth-vs-anonymous behavior: authenticated downloads use a user bucket, while anonymous downloads use an IP bucket that can fall back to `ip:unknown` when no trusted client IP header is available.

## Best Possible Solution

Keep this issue open for maintainer or ops follow-up. Verify which client-IP headers reach Convex for anonymous `https://clawhub.ai/api/v1/packages/.../download` requests through the production proxy path, preserve a real trusted client IP or enable `TRUST_FORWARDED_IPS` only behind a trusted proxy, and use the existing `rate_limit_denied` logging fields (`ipSource`, `hasClientIp`) to distinguish real per-IP throttling from `ip:unknown` collapse. If the OpenClaw install client still hides `Retry-After` or rate-limit budget details, track that client-side display work separately while keeping this production false-positive report open.
