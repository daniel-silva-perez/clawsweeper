## Summary

Keep this issue open. The repository cleanup policy for openclaw/clawhub only allows auto-closing pull requests that are certainly implemented on current main, and this is an open contributor issue. Substantively, current main still preserves the reported behavior split: authenticated download requests use a per-user bucket and skip the shared IP bucket, while anonymous downloads use an IP bucket that can fall back to `ip:unknown` when no trusted client IP header is available.

## Best Possible Solution

Keep this open for maintainer or ops follow-up. Use the existing `rate_limit_denied` log fields (`ipSource`, `hasClientIp`, auth state, and bucket kind) to verify live anonymous package downloads on clawhub.ai, then adjust proxy/header configuration or anonymous bucket behavior if requests collapse to `ip:unknown`. Track the OpenClaw install-client retry-header display gap separately if that client still hides `Retry-After` and rate-limit metadata.
