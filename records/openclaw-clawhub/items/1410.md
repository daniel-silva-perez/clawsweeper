## Summary

Keep open. Current main still has the anonymous/authenticated rate-limit split described in the report, and the ClawHub cleanup policy for this review pass does not auto-close issues; the remaining decision depends on production proxy/header logs and abuse-protection tradeoffs.

## What This Changes

The issue asks maintainers to investigate anonymous plugin installs that can return 429 while the same install succeeds immediately after login, likely due to anonymous download requests sharing an IP or unknown bucket.

## Best Possible Solution

Keep the issue open until maintainers inspect production rate-limit logs and proxy/header configuration, then either fix real client-IP attribution or deliberately adjust anonymous download bucketing with an abuse-protection review. Any remaining client-facing retry-message gap should be split to the exact client implementation path.
