## Summary

Keep this issue open. The ClawHub cleanup policy only allows auto-closing pull requests that are certainly implemented on current main, and this is an open contributor issue. Substantively, current main still preserves the reported split: authenticated downloads consume a per-user bucket, while anonymous downloads consume an IP bucket that can fall back to `ip:unknown` if no trusted client IP header reaches Convex.

## Best Possible Solution

Keep this open for maintainer or ops follow-up. Use the existing `rate_limit_denied` log fields (`ipSource`, `hasClientIp`, auth state, and bucket kind) to verify whether anonymous package downloads on clawhub.ai receive a real client IP or collapse to `ip:unknown`, then adjust proxy/header configuration or anonymous bucket behavior as needed. Track any remaining OpenClaw install-client retry-header display gap separately if that client still hides `Retry-After` and rate-limit metadata.
