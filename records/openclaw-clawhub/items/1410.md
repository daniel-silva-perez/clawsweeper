## Summary

Keep open. This is an open ClawHub contributor issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main also still preserves the reported split: authenticated downloads use a user bucket, while anonymous downloads use an IP bucket that can fall back to `ip:unknown` when no trusted client IP header reaches the Convex rate-limit layer.

## Best Possible Solution

Keep this open for maintainer or ops follow-up. Use production `rate_limit_denied` logs for `/api/v1/packages/*/download` and `/api/v1/download` to check `ipSource`, `hasClientIp`, auth state, bucket kind, and whether anonymous requests are reaching Convex with a real client IP. If anonymous traffic is collapsing to `ip:unknown` or a shared proxy IP, fix the proxy/header configuration or adjust anonymous bucket behavior; track the OpenClaw install-client retry-header display gap separately if that client still hides `Retry-After` and rate-limit metadata.
