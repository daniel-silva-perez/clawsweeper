## Summary

Keep this issue open. The ClawHub cleanup policy for this run only auto-closes pull requests that are certainly implemented on current main, and this item is an issue. Current main also still matches the reported failure mode: authenticated downloads use a per-user bucket, while anonymous downloads use an IP bucket that can fall back to `ip:unknown` when no trusted client IP header is available.

## Best Possible Solution

Keep this issue open for maintainer or ops follow-up. Verify which client-IP headers reach Convex for anonymous `https://clawhub.ai/api/v1/packages/.../download` and `/api/v1/download` requests, use the existing `rate_limit_denied` fields (`ipSource`, `hasClientIp`) to tell real per-IP throttling from `ip:unknown` collapse, then adjust trusted proxy/header configuration or anonymous bucket behavior as needed. Track any remaining OpenClaw plugin-install retry-budget display work separately if that client still hides `Retry-After` and rate-limit headers.
