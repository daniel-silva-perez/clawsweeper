## Summary

Keep this open. This is an open ClawHub contributor issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main also still preserves the reported behavior split: valid Bearer-token downloads use a per-user bucket, while anonymous downloads use a per-IP bucket that falls back to `ip:unknown` when no trusted client IP header reaches the rate-limit layer.

## Best Possible Solution

Keep this open for maintainer and ops follow-up. Check production `rate_limit_denied` logs for `/api/v1/packages/*/download` and `/api/v1/download`, especially `ipSource`, `hasClientIp`, auth state, and bucket kind. If anonymous traffic is collapsing to `ip:unknown` or a shared proxy IP, fix the deployed proxy/header configuration or adjust anonymous download bucket behavior. Track the OpenClaw install-client retry-header display gap separately if that client still hides `Retry-After` and rate-limit metadata.
