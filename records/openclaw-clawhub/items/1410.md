## Summary

Keep this issue open. The ClawHub cleanup policy for this run only auto-closes pull requests that are certainly implemented on current main, and this item is an issue. Substantively, current main still has the reported auth-vs-anonymous shape: authenticated downloads use a per-user bucket, while anonymous downloads use a client-IP bucket that can fall back to `ip:unknown` if no trusted IP header is available.

## Best Possible Solution

Keep this issue open for maintainer or ops follow-up. Verify which client-IP headers reach Convex for anonymous `https://clawhub.ai/api/v1/packages/.../download` and `/api/v1/download` requests, use the existing `rate_limit_denied` fields (`ipSource`, `hasClientIp`) to distinguish real per-IP throttling from `ip:unknown` collapse, and adjust trusted proxy/header configuration or anonymous bucket behavior accordingly. If the OpenClaw plugin install client still hides retry budget details, track that client-side display work separately while preserving this production false-positive report.
