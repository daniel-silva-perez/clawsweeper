## Summary

Keep this issue open. The repository-specific cleanup rule for openclaw/clawhub only permits auto-closing PRs that are certainly implemented on main, and this item is an issue. The current main code also still matches the reported diagnostic shape: authenticated package downloads use a user bucket, anonymous downloads use an IP bucket, and missing trusted client-IP headers collapse to `ip:unknown`.

## Best Possible Solution

Keep this open for maintainer or ops follow-up. Verify which client-IP headers reach Convex for anonymous `https://clawhub.ai/api/v1/packages/.../download` requests, ensure the production proxy preserves a trusted real client IP or intentionally enables `TRUST_FORWARDED_IPS` only behind a trusted proxy, and use the existing `rate_limit_denied` logging fields (`ipSource`, `hasClientIp`) to distinguish real per-IP throttling from `ip:unknown` collapse. If OpenClaw still hides `Retry-After` or budget details on plugin installs, track that client-side display work separately.
