## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only allows auto-closing ClawHub pull requests that are certainly implemented on current main. Current main has real malicious-skill mitigations, but the strict malicious-distribution question remains unsettled: malware-blocked skill pages and file reads are still available for transparency, the detail page still renders install command/prompt UI, and AgentGuard from #75 is useful prior art but was closed unmerged rather than shipped in core.

## Best Possible Solution

Keep this issue open for maintainer-owned security policy work. Maintainers should decide whether malware-blocked content should stay publicly inspectable for transparency or be blocked/redacted consistently across detail pages, readme/file APIs, downloads, and CLI surfaces. If strict distribution prevention is desired, add malware-blocked checks or redaction to raw file/readme paths and replace install prompts on blocked pages with a clear blocked state, while preserving safe scan metadata for review. Treat AgentGuard from #75 as prior art to evaluate rather than shipped protection.
