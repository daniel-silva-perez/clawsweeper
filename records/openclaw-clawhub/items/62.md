## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main has real malicious-skill mitigations, but the malicious-distribution policy is not fully settled: malware-blocked skill pages and raw files remain intentionally visible for transparency, the page still renders install-command UI, and AgentGuard from #75 is closed unmerged prior art rather than shipped core behavior.

## Best Possible Solution

Keep this issue open for maintainer-owned security policy work. Decide whether malware-blocked content should remain publicly transparent or be blocked/redacted consistently across page, readme/action, raw file API, download, and CLI surfaces. If distribution prevention is the desired policy, add malware-blocked checks/redaction to raw file and readme access paths and replace install prompts on blocked pages with a clear blocked state, while preserving safe scan metadata for transparency. Treat AgentGuard from #75 as prior art to evaluate rather than shipped protection.
