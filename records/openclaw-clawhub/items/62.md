## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy only permits auto-closing ClawHub pull requests that current main certainly implements. Current main has meaningful static-scan and download-block mitigations for malicious skill uploads, but end-to-end malicious-distribution prevention is still unresolved because malware-blocked skill detail/file surfaces remain readable and the public skill UI can still render install prompts.

## What This Changes

The issue requests mitigation for malicious code distribution through ClawHub, citing a published skill as an example.

## Best Possible Solution

Keep this open for maintainer-owned security policy work. Maintainers should decide whether malware-blocked skill content should remain publicly inspectable for transparency or be blocked/redacted consistently across detail pages, readme/file APIs, downloads, package skill fallback paths, and CLI/install surfaces. If strict distribution prevention is desired, add malware-blocked checks or redaction to raw/readme/file paths and replace install prompts on blocked pages with a clear blocked state while preserving safe scan metadata for review. Treat AgentGuard from PR #75 as prior art to evaluate, not shipped protection.
