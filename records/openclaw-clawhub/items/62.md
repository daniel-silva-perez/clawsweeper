## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Substantively, current main has meaningful malicious-skill mitigations, but the original malicious-distribution policy question is still unresolved because install/download paths are blocked while malware-blocked pages and file reads remain intentionally visible for transparency.

## Best Possible Solution

Keep this issue open for maintainer-owned security policy work. Maintainers should decide whether malware-blocked content should remain publicly transparent or be blocked/redacted consistently across raw file, readme/action, API, page viewer, download, and CLI surfaces. If distribution prevention is the desired policy, add malware-blocked checks to canReadSkillVersionFiles, /api/v1/skills/{slug}/file, and related viewer/API paths while preserving redacted scan metadata for transparency; evaluate AgentGuard from #75 as prior art rather than treating it as shipped.
