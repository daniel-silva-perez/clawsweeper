## Summary

Keep open. This is a ClawHub issue, and the repository-specific review policy for this run only permits auto-closing ClawHub pull requests that are certainly implemented on main. The current main branch has substantial malicious-skill mitigations, but the original distribution concern is not fully resolved because malware-blocked skill pages and raw skill file reads remain intentionally public for transparency.

## Best Possible Solution

Keep this issue open for maintainer-owned security policy work. Maintainers should decide whether malware-blocked skill content should remain publicly readable for transparency or be blocked/redacted across raw file, install, and API surfaces. If distribution prevention is the goal, add malware checks to `/api/v1/skills/{slug}/file` and any remaining install/file affordances while preserving scan metadata for public transparency, and separately evaluate AgentGuard or another scanner integration as a reviewed security workflow.
