## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main has meaningful malicious-skill mitigations, but the reported malicious-distribution policy is not fully settled because malware-blocked skill pages and raw files remain intentionally visible for transparency, and AgentGuard from the related PR is closed unmerged prior art rather than shipped core behavior.

## Best Possible Solution

Keep this issue open for maintainer-owned security policy work. Decide whether malware-blocked content should remain publicly transparent or be blocked/redacted consistently across page, readme/action, raw file API, download, and CLI surfaces. If distribution prevention is the desired policy, add malware-blocked checks/redaction to raw file and readme access paths and hide or replace install prompts on blocked pages, while preserving safe scan metadata for transparency. Treat AgentGuard from #75 as prior art to evaluate rather than shipped protection.
