## Summary

Keep open. This is an open ClawHub issue, and this cleanup pass only allows automated closure for PRs that current main certainly implements. Current main has meaningful malicious-skill mitigations, but the distribution policy is still unresolved because malware-blocked detail/file surfaces remain publicly inspectable for transparency and the detail UI still renders install prompts.

## What This Changes

The issue asks ClawHub to address the risk of malicious skill code being distributed through public skill listings.

## Best Possible Solution

Maintainers should define the intended security posture for malware-blocked skills, then enforce it consistently across skill detail pages, install prompts, readme/file APIs, ZIP/package download fallbacks, and CLI/install surfaces while preserving an explicit staff or redacted transparency path. If scanner integration is desired, evaluate AgentGuard from #75 as prior art and coordinate with the broader open security report in #662.
