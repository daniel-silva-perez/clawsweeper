## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review only permits auto-closing pull requests that are certainly implemented on current main. Current main has substantial malware mitigations, but the raw file/page transparency behavior still leaves the original malicious-distribution policy question unresolved. AgentGuard from PR #75 is useful prior art, but that PR was closed unmerged and AgentGuard is not integrated on main.

## Best Possible Solution

Keep this issue open for maintainer-owned security policy work. Maintainers should decide whether malware-blocked skill content should remain publicly readable for transparency or be blocked/redacted across raw file, API, install, and page/file viewer surfaces. If distribution prevention is the goal, add explicit malware checks to `/api/v1/skills/{slug}/file`, `skills.getReadme`, `skills.getFileText`, and any remaining raw/install-adjacent surfaces while preserving public redacted scan metadata; evaluate AgentGuard or another scanner as a separate reviewed integration.
