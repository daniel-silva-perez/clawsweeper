## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy for openclaw/clawhub only allows auto-closing pull requests that are certainly implemented on current main. Current main has meaningful malicious-skill mitigations, but the strict malicious-distribution policy remains unresolved: malware-blocked skills are intentionally visible for transparency, raw file reads remain available on some skill surfaces, and the AgentGuard work raised in the discussion was closed unmerged rather than shipped in core.

## Best Possible Solution

Keep this issue open for maintainer-owned security policy work. Maintainers should decide whether malware-blocked content should remain publicly inspectable for transparency or be blocked/redacted consistently across detail pages, readme/file APIs, downloads, and CLI surfaces. If strict distribution prevention is desired, add malware-blocked checks or redaction to raw file/readme paths and replace install prompts on blocked pages with a clear blocked state, while preserving safe scan metadata for review. Treat AgentGuard from #75 as prior art to evaluate rather than shipped protection.
