## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy only permits auto-closing ClawHub pull requests that current main certainly implements. Substantively, current main has meaningful malicious-skill mitigations, but the broader distribution policy is still unresolved because malware-blocked skill detail and file surfaces remain intentionally readable for transparency while the public UI still renders install prompts. AgentGuard from #75 is useful prior art, but that PR closed unmerged and is not shipped protection.

## What This Changes

The issue reports a malicious code distribution risk on ClawHub using a concrete published skill URL as the example.

## Best Possible Solution

Maintainers should decide the intended security posture for malware-blocked skills, then enforce it consistently across skill detail pages, install prompts, readme/file APIs, ZIP/package download fallbacks, and CLI/install surfaces while preserving a deliberate staff or redacted transparency path for review. If a scanner integration is desired, evaluate AgentGuard from #75 as prior art rather than treating it as already shipped.
