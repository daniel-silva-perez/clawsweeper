## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy for this pass only permits auto-closing ClawHub pull requests that current main certainly implements. Current main has meaningful malicious-skill mitigations, but the broader distribution policy remains unresolved because malware-blocked skill detail/file surfaces are intentionally still readable for transparency and the public UI still renders install prompts.

## What This Changes

The issue reports a malicious-code distribution risk from a ClawHub-hosted skill and asks for stronger protection against malicious skill distribution.

## Best Possible Solution

Maintainers should decide the intended posture for malware-blocked skills, then enforce it consistently across skill detail pages, install prompts, readme/file APIs, ZIP/package download fallbacks, and CLI/install surfaces while preserving an explicit staff or redacted transparency path. If scanner integration is desired, AgentGuard from #75 should be evaluated as prior art rather than treated as shipped protection.
