## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy for this repository only permits auto-closing pull requests that current main certainly implements. Current main has meaningful malicious-skill mitigations, but the broader distribution posture remains unresolved because malware-blocked detail and raw-file surfaces are deliberately still readable for transparency and the public detail UI still renders install prompts; AgentGuard from #75 is useful prior art, but that PR closed unmerged and is not shipped protection.

## What This Changes

The issue reports a ClawHub malicious-code distribution risk, illustrated by a published polymarketagent skill page.

## Best Possible Solution

Maintainers should define the intended security posture for malware-blocked skills, then enforce it consistently across skill detail pages, install prompts, readme/file APIs, ZIP/package download fallbacks, and CLI/install surfaces while preserving an explicit staff or redacted transparency path. If scanner integration is desired, evaluate AgentGuard from #75 as prior art rather than treating it as existing protection.
