## Summary

Keep open. This is an open ClawHub issue, and the repository policy for this cleanup pass only permits auto-closing ClawHub pull requests that current main certainly implements. Current main has meaningful static-scan and ZIP-download mitigations for malicious skill uploads, but the broader malicious-distribution policy is still unresolved because malware-blocked skill detail and file surfaces remain intentionally readable for transparency and the public skill UI still renders install prompts.

## What This Changes

The issue reports that ClawHub can distribute malicious skill content, with an example hosted skill URL and a related AgentGuard proposal raised in discussion.

## Best Possible Solution

Keep this open for maintainer-owned security policy. Maintainers should decide whether malware-blocked skill content should remain publicly inspectable for transparency or be blocked/redacted consistently across detail pages, readme/file APIs, downloads, package skill fallback paths, and CLI/install surfaces. If strict distribution prevention is desired, add malware-blocked checks or redaction to raw/readme/file paths and replace install prompts on blocked pages with a clear blocked state while preserving safe scan metadata for review. Treat AgentGuard from #75 as prior art to evaluate, not shipped protection.
