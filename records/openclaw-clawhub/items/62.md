## Summary

Keep open. This is an open ClawHub security issue, and the repository policy for this cleanup pass only permits auto-closing ClawHub PRs that current main certainly implements. Substantively, current main has useful malicious-skill mitigations, but strict malicious-distribution prevention is still unresolved: malware-blocked skills can remain publicly inspectable through detail/raw-file paths, and the public skill page still renders install prompts while downloads are blocked.

## Best Possible Solution

Keep this issue open for maintainer-owned security policy work. Maintainers should decide whether malware-blocked skill content should remain publicly inspectable for transparency or be blocked/redacted consistently across detail pages, readme/file APIs, downloads, package skill fallback paths, and CLI/install surfaces. If strict distribution prevention is desired, add malware-blocked checks or redaction to raw/readme/file paths and replace install prompts on blocked pages with a clear blocked state while preserving safe scan metadata for review. Treat AgentGuard from PR #75 as prior art to evaluate, not shipped protection.
