## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Substantively, current main has meaningful malware mitigations, but the original malicious-distribution policy question is not fully resolved: installs and zip downloads are blocked, while malware-blocked skill pages/files remain intentionally readable for transparency and the raw file API serves stored text without a malware-blocked guard.

## Best Possible Solution

Keep this issue open for maintainer-owned security policy work. Maintainers should decide whether malware-blocked content should remain publicly transparent or be blocked/redacted consistently across raw file, readme/action, API, page viewer, download, and CLI install/update surfaces. If distribution prevention is the desired policy, add malware-blocked checks to canReadSkillVersionFiles, /api/v1/skills/{slug}/file, and related viewer/API paths while keeping redacted scan metadata visible; evaluate AgentGuard from PR #75 as prior art rather than treating it as shipped.
