## Summary

Keep open under the ClawHub cleanup policy for issues. The narrow CLI bug is already fixed on current main: both `search` and `explore` read the optional saved token, pass it to `apiRequest`, and have regression coverage. The only plausible remaining question is product/API scope: whether authenticated search/explore should also return owner-visible hidden/private catalog entries rather than only public catalog results.

## What This Changes

The issue asks for `clawhub search` and `clawhub explore` to include the saved authentication token when calling the backend, matching other authenticated CLI commands.

## Best Possible Solution

Keep the current CLI token forwarding and regression tests. A maintainer should manually close this as fixed for the CLI bug, or split a focused backend/API issue if authenticated search/explore should expose owner-visible non-public catalog entries.
