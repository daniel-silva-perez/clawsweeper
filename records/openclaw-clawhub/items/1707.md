## Summary

Keep open under the ClawHub cleanup policy for issues. The reported CLI token-forwarding bug is already implemented and covered by tests on current main, and related merged PR #608 plus the changelog point to the same fix; the only remaining question is whether authenticated search/explore should also change backend visibility semantics.

## What This Changes

The issue asks for `clawhub search` and `clawhub explore` to include the saved authentication token in backend requests, matching other authenticated CLI commands.

## Best Possible Solution

Keep the current CLI token forwarding and regression tests. A maintainer should manually close this as fixed for the CLI bug, or split a focused backend/API issue if authenticated search/explore should expose owner-visible non-public catalog entries.
