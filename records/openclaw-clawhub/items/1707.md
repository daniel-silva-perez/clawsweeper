## Summary

Keep open under the ClawHub cleanup policy: this is an issue, and the policy only allows implemented-on-main auto-closes for pull requests. Substantively, current main already forwards the saved optional token from both CLI commands into apiRequest and has focused tests, but the backend search/list handlers still use public catalog paths after auth-aware rate limiting, so any broader owner-visible/private-content expectation needs maintainer product/API judgment.

## What This Changes

The issue reports that `clawhub search` and `clawhub explore` did not include the saved authentication token when calling the registry APIs.

## Best Possible Solution

A maintainer should resolve the remaining scope explicitly: close this issue manually if CLI-side token forwarding was the intended fix, or split/open a narrower backend API issue if authenticated `search` and `explore` should return owner-visible non-public content.
