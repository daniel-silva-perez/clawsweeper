## Summary

Keep open. Current main fixes the narrow CLI bug by forwarding the optional saved token from both `search` and `explore`, but ClawHub cleanup policy does not auto-close issues. A maintainer still needs to decide whether the report is fully resolved or whether authenticated owner-visible search/list behavior should be split into a separate backend/API item.

## What This Changes

The issue reports that `clawhub search` and `clawhub explore` did not include the saved authentication token when calling the registry.

## Best Possible Solution

Keep the current CLI token forwarding. A maintainer should either close this manually as fixed for the CLI bug, or open a separate focused backend/API issue defining authenticated search/explore visibility semantics.
