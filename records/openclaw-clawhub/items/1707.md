## Summary

Keep this issue open under the ClawHub cleanup policy, which does not auto-close issues. Substantively, current main fixes the CLI-side token omission: both `cmdSearch` and `cmdExplore` read the optional auth token, pass it into `apiRequest`, and tests assert that behavior. One remaining product question is whether authenticated search/explore should expose owner-visible private or hidden content, because the backend search/list handlers still use public catalog paths.

## Best Possible Solution

A maintainer should manually resolve this issue: close it as fixed if the CLI-side token forwarding is the intended scope, or split a narrower backend/product issue if authenticated search/explore should include owner-visible private or hidden content.
