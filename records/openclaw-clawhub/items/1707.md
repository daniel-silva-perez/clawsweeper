## Summary

Keep open. ClawHub cleanup policy keeps issues open, and although current main fixes the narrow CLI token-forwarding bug with tests, the remaining backend visibility question still needs maintainer disposition rather than automated closure.

## What This Changes

The issue reports that `clawhub search` and `clawhub explore` should send the saved auth token to the backend like other authenticated CLI commands.

## Best Possible Solution

Leave the current CLI token forwarding in place, then have a maintainer explicitly decide whether authenticated search/explore should remain public-catalog reads with authenticated rate-limit buckets or gain owner-visible private/hidden result semantics.
