## Summary

Keep open. Current main does not implement this PR's publisher-declared `category: security` moderation behavior, and the remaining decision is security-sensitive moderation policy rather than an already-shipped mainline change.

This PR adds a publisher-declared skill category system for moderation, contextualizes selected security-tool static scan findings into `info.security_context.*`, passes category context to LLM evaluation, updates security/skill-format docs and tests, and adds a `package-lock.json`.

## Best Possible Solution

Keep this PR open for maintainer/security review.
