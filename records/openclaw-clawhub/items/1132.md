## Summary

Keep open. Current main does not implement the moderation-category behavior, and ClawHub cleanup policy only closes PRs when their intended change is definitely already on main; the remaining work also needs maintainer/security review because it changes how publisher-declared metadata affects moderation severity.

## What This Changes

This PR adds a publisher-declared `category: security` moderation context that rewrites selected static-scan `suspicious.*` codes to `info.security_context.*`, passes category context to the LLM evaluator, updates docs/tests, and adds `package-lock.json`.

## Best Possible Solution

If maintainers want this capability, land a rebased Bun-only implementation that explicitly settles the category trust model and LLM-unavailable behavior, integrates with the current `v2.4.2` moderation snapshot/verdict paths, preserves conservative safety-floor handling, and updates focused tests plus user-facing docs.
