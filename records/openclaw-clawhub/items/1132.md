## Summary

Keep open. ## Best Possible Solution

Keep this PR open for maintainer review rather than auto-closing it. If maintainers want this feature, the PR should be rebased onto current moderation engine v2.4.2, remove or justify the `package-lock.json`, make an explicit fail-open/fail-closed decision for LLM-unavailable security-category skills, preserve conservative handling for unknown reason-code prefixes, and add current-main tests/docs for publisher-declared security-category behavior.
