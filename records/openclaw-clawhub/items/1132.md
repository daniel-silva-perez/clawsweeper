## Summary

Keep open. Current main does not implement this PR's core moderation behavior: publisher-declared `category: security` context that rewrites selected static scanner reason codes to non-escalating `info.security_context.*` codes and passes that category into LLM evaluation.

## Best Possible Solution

Keep this PR open for maintainer review. If maintainers want this category system, rebase it onto current moderation engine v2.4.2, remove or justify the `package-lock.json`, make an explicit fail-open/fail-closed decision for LLM-unavailable cases, preserve conservative handling for unknown reason-code prefixes, and add current-main tests/docs for security-category behavior.
