## Summary

Keep open. Current main still does not implement this PR's publisher-declared `category: security` moderation path; it has browse UI categories and recent VT/static-scan calibration, but no moderation category extraction, `info.security_context.*` reason codes, LLM category context, or docs for skill metadata categories. The PR also remains paired with open issue #1131 and related open PR #1133, and its security-sensitive fail-open behavior still needs maintainer judgment.

## Best Possible Solution

Keep this PR open for maintainer review. If maintainers want this category system, rebase it onto current moderation engine v2.4.2, remove or explicitly justify `package-lock.json`, make a documented fail-open versus fail-closed decision for LLM-unavailable cases, preserve conservative handling for unknown reason-code prefixes, align category metadata precedence with the parser, and update current-main tests/docs for security-category behavior.
