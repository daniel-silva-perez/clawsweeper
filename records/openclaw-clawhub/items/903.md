## Summary

Keep open. Current main still has the global-stats live recount paths and single-key rate-limit counter behavior that this PR is intended to change, so it is not implemented on main. The PR also changes rate-limit enforcement semantics, and the aggregate-cap review concern makes this a maintainer/security-sensitive review item rather than an auto-close candidate.

## What This Changes

The PR removes the global stats live recount fallback and deprecated manual mutation, updates the seeding docs, and shards rate-limit counters while using cross-shard status for remaining-rate-limit headers.
