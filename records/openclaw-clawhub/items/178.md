## Clownfish Work Prompt

Fix the ClawHub CLI/server fingerprint mismatch reported in this issue and related reports #169 and #937. On current main, skill downloads include a generated root `_meta.json`, while publish/resolve fingerprints are computed from stored version files; local update/sync fingerprints include root JSON files unless ignored. Implement a narrow fix so a fresh install or `clawhub update --all --force` followed immediately by `clawhub update --all` resolves as up to date when files are otherwise unchanged. Prefer excluding only ClawHub-generated root `_meta.json` from managed install/update/sync fingerprinting, or otherwise make download/resolve hash the same logical file set. Preserve authored JSON files and publish-time fingerprints; do not blanket-ignore all JSON or all `_meta.json` in arbitrary publish source unless that is explicitly required by the existing product contract. Add regression tests for the fresh install/force-update then update path and for the helper behavior. Keep the change focused; do not add unrelated CLI, package, or release changes.
