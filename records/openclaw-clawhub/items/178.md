## Summary

Keep this issue open. The repository-specific cleanup policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on current main, and this item is an issue. Substantively, current main still has the reported mismatch path: skill downloads inject `_meta.json`, local fingerprinting includes ordinary `.json` files unless ignored, and resolve compares against version fingerprints built from published files that do not include generated download metadata.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The likely fix is to make client and server fingerprint inputs match, either by excluding generated ClawHub metadata such as `_meta.json` from managed-install fingerprints or by removing generated metadata from downloaded skill archives, then add a regression test for install or force-update followed immediately by update reporting the skill as up to date.
