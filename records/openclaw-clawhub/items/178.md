## Summary

Keep open. Repository policy for this ClawHub review pass only permits auto-closing PRs that current main definitely implements, and the reported fingerprint mismatch still appears valid on current main: downloaded skill ZIPs include generated root `_meta.json`, local update hashing includes root JSON files unless ignored, and server resolve compares against publish-time fingerprints built from the authored version files without that generated metadata.

## What This Changes

The issue requests fixing `clawhub update` so freshly installed or force-updated skills resolve against the server fingerprint instead of reporting false local changes.

## Best Possible Solution

Keep this issue open and land one focused fix coordinated with #169 and #937. The safest path is to make downloaded and published fingerprints cover the same logical file set, likely by excluding only ClawHub-generated root `_meta.json` from managed install/update/sync fingerprinting or by aligning packaging and resolve consistently, while preserving normal authored JSON files in fingerprints and adding regression coverage for install or force-update followed by update reporting up to date.
