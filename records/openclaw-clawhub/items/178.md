## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the reported mismatch still appears valid on current main: downloaded skill ZIPs get a generated root `_meta.json`, the CLI fingerprints local JSON files unless ignored, and the server resolve path compares against fingerprints derived from published/version files without that generated metadata. The jeanmonet comment's `.clawhubignore` workaround aligns with the current code, and related open reports #169 and #937 should be handled by one focused fix.

## Best Possible Solution

Keep this issue open and fix the client/server fingerprint input mismatch in one focused PR coordinated with #169 and #937. The safest path is to exclude only ClawHub-generated root `_meta.json` from managed install/update/sync fingerprints, or otherwise change packaging/resolve so the server and client hash the same logical skill files, while preserving normal authored JSON in fingerprints and adding regression coverage for fresh install or force-update followed by update reporting up to date.
