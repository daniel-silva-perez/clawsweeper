## Clownfish Work Prompt

Fix the ClawHub skill update fingerprint mismatch reported here and in related #169/#937. Read `convex/_generated/ai/guidelines.md` first if touching Convex code. Current behavior: `/api/v1/download` skill zips include generated root `_meta.json`; `clawhub update` and `clawhub sync` compute local fingerprints with `listTextFiles()`, which includes JSON files and does not exclude `_meta.json`; `/api/v1/resolve` matches publish-time fingerprints that do not include that generated metadata, causing false `local changes (no match)` immediately after install or force-update.

Expected boundary: make managed skill fingerprints compare the same logical skill files on both client and server. Prefer a small, explicit handling of ClawHub-generated root `_meta.json` in the CLI fingerprint helpers (`listTextFiles` and `hashSkillZip` if remote zip fallback should stay comparable). Do not ignore arbitrary JSON files or unrelated metadata directories. If root `_meta.json` is treated as a reserved generated path, add or update tests/docs accordingly and consider whether publish should reject/reserve that path instead of silently hashing user-supplied content that downloads overwrite.

Add regression coverage showing a downloaded/managed skill containing generated `_meta.json` still resolves as up to date after install or force-update. Include sync coverage if changing shared fingerprint helpers. Keep the PR focused on fingerprint parity; do not change moderation, force-update prompts, registry auth, package plugin downloads, or unrelated CLI behavior. No changelog entry is required unless maintainers expect one for CLI bug fixes.
