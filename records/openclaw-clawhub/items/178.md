## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that current main definitely implements. Substantively, the fingerprint mismatch still appears valid on current main: downloaded ZIPs include generated root `_meta.json`, local update fingerprinting includes root JSON files unless ignored, and server resolve compares against fingerprints from published version files without that generated metadata.

## What This Changes

The issue reports that `clawhub update --all` falsely reports `local changes (no match)` immediately after install or force-update because local fingerprints do not match the server resolve index.

## Best Possible Solution

Keep this issue open and land one focused fix coordinated with #169 and #937. The safest path is to make downloaded and published fingerprints cover the same logical file set, likely by excluding only ClawHub-generated root `_meta.json` from managed install/update/sync fingerprints or by changing packaging/resolve consistently, while preserving normal authored JSON in fingerprints and adding regression coverage for fresh install or force-update followed by update reporting up to date.
