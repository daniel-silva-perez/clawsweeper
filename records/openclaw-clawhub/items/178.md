## Summary

Keep this issue open. The ClawHub cleanup policy only allows auto-closing PRs that are certainly implemented on current main, and this item is an issue. Substantively, current main still shows the reported mismatch: update hashes local JSON files, downloaded archives still add generated `_meta.json`, and the resolve endpoint compares against fingerprints built from published version files rather than generated download metadata.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The likely fix is to make client and server fingerprint inputs match, either by excluding generated ClawHub metadata such as `_meta.json` from managed-install fingerprints or by not adding generated metadata to downloaded skill archives, then add a regression test for install or force-update followed immediately by update reporting up to date. Maintainers may also choose to consolidate this with open related #169.
