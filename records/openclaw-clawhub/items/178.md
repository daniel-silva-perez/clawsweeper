## Summary

Keep open. The ClawHub cleanup policy for this pass keeps issues open, and current main still appears to have the reported mismatch: downloaded skills can contain generated root `_meta.json`, local update hashing includes ordinary JSON files unless ignored, and server resolve compares against publish/version fingerprints built from authored version files only.

## What This Changes

Fix `clawhub update` fingerprint matching so freshly installed or force-updated skills resolve as known registry versions instead of falsely reporting local changes.

## Best Possible Solution

Align the downloaded/local and published/resolve fingerprint inputs so ClawHub-generated root `_meta.json` does not create false local-change warnings, while preserving integrity checks for authored files and adding regression coverage for install or force-update followed by update reporting up to date.
