## Summary

Keep this issue open. The ClawHub cleanup policy says issues should remain open, and current main still shows the reported fingerprint mismatch path: skill downloads include generated `_meta.json`, local hashing includes JSON files unless ignored manually, and server-side publish/resolve fingerprints are based on published version files rather than the generated download metadata.

## Best Possible Solution

Keep this issue open and fix the core fingerprint mismatch. The likely maintainer path is to either stop adding generated `_meta.json` to skill download archives, or make local fingerprinting ignore generated ClawHub metadata for managed installs, then add a regression test for install or force-update followed immediately by update reporting the skill as up to date. The related open #169 can remain the broader duplicate/canonical tracking item if maintainers prefer consolidating reports manually.
