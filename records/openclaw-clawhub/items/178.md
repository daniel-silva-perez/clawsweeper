## Summary

Keep this issue open. The repository-specific ClawHub cleanup policy only permits auto-closing pull requests that are certainly implemented on main, not issues. Substantively, current main still appears vulnerable to the reported mismatch: skill downloads inject `_meta.json`, local fingerprinting includes JSON files and does not ignore `_meta.json` by default, while publish/resolve fingerprints are based on published `version.files`. The `.clawhubignore` workaround from the discussion is valid and useful, but it is not applied by the core install/update flow.

## Best Possible Solution

Keep this issue open and fix the core fingerprint mismatch. The likely maintainer path is to either stop injecting generated `_meta.json` into skill download archives, or make local fingerprinting ignore generated ClawHub metadata for managed installs, then add a regression test covering install or force-update followed immediately by update reporting up to date.
