## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. The underlying bug path still appears present: update fingerprints local text files, root JSON files are included unless ignored, downloaded skill ZIPs still add generated `_meta.json`, and server resolve compares against published-version fingerprints/files that do not include that generated download metadata.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The likely fix is to make client and server fingerprint inputs match, either by excluding generated ClawHub metadata such as `_meta.json` from managed-install fingerprints or by changing download/package metadata behavior, then add a regression test for install or force-update followed immediately by update reporting up to date. Maintainers may also consolidate this with open related #169 if they want one canonical tracker.
