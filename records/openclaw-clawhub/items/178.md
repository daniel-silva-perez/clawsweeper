## Summary

Keep open. The ClawHub cleanup policy for this pass keeps issues open, and the current main code still matches the reported mismatch: update hashes local extracted files, downloads add a generated root `_meta.json`, and resolve compares against fingerprints built from published version files. The `_meta.json` / `.clawhubignore` workaround described in the discussion is consistent with the current implementation.

## What This Changes

Fix `clawhub update --all` so freshly installed or force-updated skills resolve as known versions instead of warning `local changes (no match)` from a fingerprint mismatch.

## Best Possible Solution

Make the CLI and registry compare the same canonical skill file set for update integrity checks, treating generated ClawHub download metadata consistently while preserving checks for authored JSON and other text files. Add regression coverage for install or force-update followed by update reporting up to date.
