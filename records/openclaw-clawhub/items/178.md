## Summary

Keep open. The ClawHub cleanup policy for this pass keeps issues open, and current main still appears to have the reported fingerprint mismatch: the downloaded install tree can include generated root `_meta.json`, local update hashing includes JSON files unless ignored, and server resolve compares against published/version fingerprints that do not include that generated metadata.

## What This Changes

Fix `clawhub update --all` so a clean installed or force-updated skill is recognized as a known version instead of reporting `local changes (no match)`.

## Best Possible Solution

Align the downloaded/local and published/resolve fingerprint inputs so ClawHub-generated root `_meta.json` does not create false local-change warnings, while preserving integrity checks for authored files and adding regression coverage for install or force-update followed by update reporting up to date.

Current main downloads skills with generated root `_meta.json`, local update hashing includes JSON files, and `/api/v1/resolve` compares against publish/version fingerprints that do not include that generated metadata. Align the compared file set so ClawHub-generated root `_meta.json` does not cause a mismatch, while preserving integrity coverage for authored files; do not ignore all JSON files or broad `_meta.json` paths without a reasoned test.
