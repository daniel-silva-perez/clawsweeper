## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Substantively, current main still supports the reported false `local changes (no match)` path: update fingerprints local text files, local JSON files are included unless ignored, downloaded skill archives still add generated `_meta.json`, and server resolve compares against fingerprints stored or recomputed from published version files rather than generated download metadata.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The likely fix is to make client and server fingerprint inputs match, either by excluding generated ClawHub metadata such as `_meta.json` from managed-install fingerprints or by not adding generated metadata to downloaded skill archives, then add a regression test for install or force-update followed immediately by update reporting up to date. Maintainers may also choose to consolidate this with open related #169.
