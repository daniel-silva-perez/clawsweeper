## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only allows auto-closing ClawHub pull requests that are certainly implemented on current main. The bug also still appears valid on current main: skill downloads inject `_meta.json`, local update fingerprinting includes JSON files unless ignored, and server resolution compares against publish-time fingerprints that do not include generated download metadata.

## Best Possible Solution

Keep this issue open and fix the fingerprint input mismatch in one focused CLI/backend PR, preferably coordinated with open related #169. The implementation should make client and server fingerprints cover the same logical skill files, most likely by excluding generated ClawHub metadata such as `_meta.json` from managed-install/update fingerprints or by changing the generated metadata packaging behavior, then add a regression test for install or force-update followed immediately by update reporting up to date.
