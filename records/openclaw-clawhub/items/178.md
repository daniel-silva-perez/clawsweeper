## Summary

Keep open. The ClawHub cleanup policy for this pass keeps issues open, and current main still appears to have the reported mismatch: downloaded skills include generated root `_meta.json`, local update hashing includes ordinary JSON files unless ignored, and server resolve compares against publish/version file fingerprints that do not include that generated metadata.

## What This Changes

The issue requests that `clawhub update` recognize freshly installed or force-updated skills as matching published fingerprints instead of warning about `local changes (no match)`.

## Best Possible Solution

Align downloaded/local and published/resolve fingerprint inputs so ClawHub-generated root `_meta.json` does not create false local-change warnings, while preserving integrity checks for authored files and adding regression coverage for install or force-update followed by update reporting up to date.
