## Summary

Keep open. Current main addresses much of the original report with higher auth-aware quotas, Retry-After/RateLimit headers, CLI 429 guidance, and authenticated install/download requests, but repository policy for ClawHub issues requires keeping issues open rather than auto-closing them. Substantively, related install-rate-limit reports are still open and the public OpenAPI snapshot still advertises stale quotas without 429/header response metadata.

## What This Changes

The issue requests less aggressive ClawHub CLI/API rate limits, visible Retry-After/rate-limit metadata, clearer CLI errors, and published rate-limit policy documentation.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to verify production 429 behavior and headers for anonymous and authenticated install/download requests, inspect rate_limit_denied logs for client-IP attribution and bucket selection, reconcile related open reports #390, #589, #654, and #1410, then update public/api/v1/openapi.json so machine-readable API metadata matches current rate-limit code and docs.
