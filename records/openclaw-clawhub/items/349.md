## Summary

Keep this issue open. The ClawHub-specific cleanup policy says issues should remain open and only PRs may be auto-closed as implemented on current main. Current main has substantial fixes for the original complaint: higher auth-aware limits, rate-limit headers, Retry-After handling, CLI error enrichment, and regression tests. However, related install/rate-limit reports remain open, anonymous requests can still collapse to an unknown/shared IP bucket when client-IP headers are unavailable, and the static OpenAPI description still advertises older limits.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to verify production anonymous install/download behavior and client-IP header propagation, fix or document any remaining anonymous attribution failure, update the static OpenAPI description to match the implementation, then consolidate this broad issue with the narrower open rate-limit/install reports if maintainers prefer.
