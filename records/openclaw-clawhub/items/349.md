## Summary

Keep this issue open. Current main has substantial rate-limit fixes for the original complaint, but this is a ClawHub issue and the provided cleanup policy only permits auto-closing PRs as implemented. There are also still unresolved signals: related install/rate-limit reports remain open, anonymous requests can still collapse to an `ip:unknown` bucket when client IP headers are unavailable, and the static OpenAPI description still advertises old limits.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely path is to verify production anonymous download/install behavior, especially Cloudflare/Vercel/Convex client-IP header propagation, then either fix the anonymous attribution path or consolidate this broad issue with the narrower open install/rate-limit reports. Also update the static OpenAPI rate-limit description so all public docs match the shipped limiter.
