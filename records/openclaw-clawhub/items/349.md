## Summary

Keep open. Current main has substantially improved the original rate-limit behavior with higher auth-aware quotas, rate-limit headers, CLI retry guidance, and authenticated install/download requests, but ClawHub cleanup policy keeps issues open and there are still related live reports plus stale machine-readable OpenAPI metadata.

## What This Changes

The issue requests less aggressive ClawHub CLI/API rate limits, visible retry/quota/reset headers and CLI errors, and a published rate-limit policy for normal publish-install iteration.

## Best Possible Solution

Keep the current auth-aware limiter and CLI 429 UX, then reconcile production behavior and documentation by verifying real anonymous/authenticated 429 headers, checking rate_limit_denied logs for IP attribution and bucket selection, and updating OpenAPI so public machine-readable metadata matches current code and docs.
