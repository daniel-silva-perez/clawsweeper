Keep open. Current main substantially addresses the original rate-limit complaint with auth-aware quotas, retry/quota headers, CLI 429 guidance, authenticated install requests, docs, tests, and a v0.12.0 changelog entry, but this is an openclaw/clawhub issue and the supplied cleanup policy only auto-closes PRs that are certainly implemented on main. The issue requests less aggressive ClawHub CLI/API rate limits plus visible retry/quota headers, clearer CLI errors, and a published rate-limit policy.
