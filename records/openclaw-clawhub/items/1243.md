## Summary

Keep this issue open. The openclaw/clawhub cleanup policy only permits automated closure for pull requests that are certainly implemented on current main, and this item is an issue. Current main also still has a plausible search-recall gap: skill tags are stored in the lightweight search digest, but the lexical matching paths used by skills search only check display name, slug, and summary, so explicit tag terms like phone or voice are not guaranteed to surface a skill when vector recall misses it.

## Best Possible Solution

Keep this issue open, or have maintainers manually consolidate it with #1254. The likely fix belongs in convex/search.ts and the skillSearchDigest schema/indexing path: add bounded lexical recall over normalized searchable fields that include tags and capabilityTags, preserve existing nonSuspiciousOnly behavior, and add focused tests for tag queries such as phone and voice.
