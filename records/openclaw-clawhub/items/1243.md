## Summary

Keep this open. ClawHub cleanup policy does not auto-close issues, and current main still has a credible search-recall gap: tag data is stored in the lightweight search digest, but the skill search matching paths only compare display name, slug, and summary, and fallback can be skipped once vector-hydrated primary matches fill the limit. Related #1254 remains the broader lexical/vector recall issue, while #1326 tracks the suspicious-label aspect for the specific skill.

## What This Changes

Searches for `phone` or `voice` should surface `batthis/amber-voice-assistant` based on its tags and prominent title/summary text.

## Best Possible Solution

Land a bounded search-recall change, preferably coordinated with #1254, that treats normalized skill tags and capability tags as lexical candidates while preserving existing moderation filters. Add regression coverage for `phone` and `voice` tag queries, and leave the suspicious-flag decision to #1326.
