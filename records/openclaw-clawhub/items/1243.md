## Summary

Keep this issue open. The repository-specific policy for openclaw/clawhub only permits automated close recommendations for PRs that are certainly implemented on current main, and this item is an issue. Substantively, current main still has a search-recall gap for tag-only terms: tags are stored on the search digest, but both vector-hydrated exact filtering and lexical fallback compare query tokens only against display name, slug, and summary.

## Best Possible Solution

Keep this issue open, or have maintainers manually consolidate it with open issue #1254. The likely fix belongs in convex/search.ts plus the search digest schema/indexing strategy: add a bounded lexical candidate path over normalized searchable fields that includes tags and capabilityTags, preserve highlightedOnly, capabilityTag, and nonSuspiciousOnly behavior, and add focused tests for tag queries such as phone and voice against a skill like amber-voice-assistant.
