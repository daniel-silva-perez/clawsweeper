## Summary

Keep this open. The current main branch still has a credible skill-search recall gap: tag fields are present in the lightweight search digest, but both vector-hydrated exact matching and lexical fallback only match display name, slug, and summary, and fallback can be skipped after enough primary matches. The ClawHub cleanup policy also does not auto-close issues, while related #1254 and #1326 remain open for the broader recall and suspicious-label parts.

## What This Changes

The issue asks ClawHub search to surface a published skill for `phone` and `voice` queries when those terms are present in the skill tags and metadata.

## Best Possible Solution

Land a bounded search-recall fix, preferably coordinated with #1254, that treats normalized skill tag names and capability tags as lexical candidates while preserving existing moderation filters. Add regression coverage for `phone` and `voice` tag queries, and leave suspicious-label review to #1326.
