## Clownfish Work Prompt

Fix ClawHub skill search recall for user tag queries. Current main stores `skillSearchDigest.tags` and `capabilityTags`, but `searchSkills` and `lexicalFallbackSkills` only pass displayName, slug, and summary into `matchesExactTokens`, so a skill tagged `phone` or `voice` may not be found when vector recall misses it. Implement a focused fix in the existing search path that treats normalized skill tag names, including hyphen/underscore tokenization such as `ai-phone`, `phone-screening`, and `outbound_calls`, as lexical match inputs. Coordinate the shape with open #1254's direct lexical candidate work, but keep the PR narrow: do not rewrite ranking wholesale, do not introduce unbounded Convex scans, and preserve `nonSuspiciousOnly`, `highlightedOnly`, `capabilityTag`, soft-delete, and moderation filtering. Add regression coverage in `convex/search.test.ts` for `phone` and `voice` queries matching tag-only or tag-primary skills, plus a hyphenated tag case. No changelog is required unless maintainers normally include one for search behavior fixes.
