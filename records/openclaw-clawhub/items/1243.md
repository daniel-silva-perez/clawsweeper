## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open in this pass, and current main still has a plausible tag-search recall gap: skill digests store `tags` and `capabilityTags`, but both vector-hydrated exact matching and fallback lexical matching only check display name, slug, and summary. Related #1254 remains open and is a good consolidation target, but it has not superseded or fixed this report.

## Best Possible Solution

Keep this issue open, or manually consolidate it into open #1254 with explicit scope for tag and capability-tag recall. A focused fix should add deterministic lexical candidates over normalized searchable skill fields including display name, slug, summary, tags, and capabilityTags; preserve highlightedOnly/nonSuspiciousOnly/capabilityTag behavior; avoid unbounded scans; and add tests for `phone` and `voice` tag queries.
