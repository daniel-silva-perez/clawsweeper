## Summary

Keep this issue open. ClawHub cleanup policy does not allow automated closure of issues, and current main still has a plausible tag-search recall gap: tags are stored in the search digest, but skill lexical matching still checks only display name, slug, and summary. Related #1254 remains open and may be a good maintainer consolidation target, but it has not replaced or fixed this issue yet.

## Best Possible Solution

Keep this issue open, or have maintainers manually consolidate it into #1254 with explicit scope for tag and capability-tag recall. A good fix likely belongs in convex/search.ts plus the skillSearchDigest schema: add a bounded indexed lexical candidate source over normalized searchable fields that includes display name, slug, summary, tags, and capabilityTags; preserve highlightedOnly, nonSuspiciousOnly, and capabilityTag filtering; and add focused tests for phone and voice tag queries.
