Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy for this review pass only permits auto-closing PRs that current main definitely implements. Current main has meaningful VirusTotal pending-scan recovery, rescan support, and a generic pending UI/API state, but it still does not expose the owner-facing queue position, worker heartbeat, retry timing, or state-transition visibility requested in the issue discussion.

Keep this issue open for maintainer follow-up. If maintainers decide the named production record being unhidden is sufficient, they can manually close it; otherwise the durable fix is an owner-visible scan status surface for pending skills that safely exposes state transitions, last check time, check count, retry or next-poll timing, stale reason, and appropriate queue/worker health without leaking sensitive moderation internals.
