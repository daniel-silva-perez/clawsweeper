## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy for this review only permits auto-closing pull requests that are certainly implemented on current main. The named production skill now reports clean scan results, and current main has pending-scan polling/stale handling, but owner-visible queue position, worker heartbeat, retry timing, and state-transition status are still not exposed.

## Best Possible Solution

Keep this issue open for maintainer follow-up. Maintainers can manually close if the cleaned production record is considered sufficient; otherwise the durable fix is to add an owner-visible scan status surface for pending skills that reports scan state transitions, last check time, retry schedule or next poll window, queue age/position or worker health, and includes tests for pending, stale, clean, suspicious, and malicious scan states.
