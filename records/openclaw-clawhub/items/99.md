## Summary

Keep open. The ClawHub cleanup policy keeps issues open for maintainer judgment, and current main does not prove the reported login/token verification failure is fixed: browser auth still hands a token to the CLI, the CLI verifies `/api/v1/whoami` before persisting it, token-auth failures still collapse to `Unauthorized`, and related Unauthorized reports remain open while the redirect-specific PR was closed unmerged.

## What This Changes

The issue requests fixing ClawHub CLI login so browser auth and `clawhub login --token` store a usable token instead of failing with `Unauthorized` after web authentication succeeds.

## Best Possible Solution

Confirm the production auth path with fresh browser-created and settings-created CLI tokens, make the CLI persist and verify the canonical registry consistently, add focused regression coverage for host/token behavior, and consolidate the related Unauthorized reports under one maintainer-owned tracker or fix.
