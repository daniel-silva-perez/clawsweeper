## Summary

Keep open. The ClawHub cleanup policy keeps issues open for maintainer judgment, and current main does not prove the reported login/token verification failure is fixed: the CLI still verifies `/api/v1/whoami` before writing the token, token-auth failures still surface as plain `Unauthorized`, the redirect-focused PR was closed unmerged, and current live host behavior no longer matches the original workaround closely enough for an automatic code change.

## What This Changes

The issue reports `clawhub login` completing browser authentication but the CLI failing with `Unauthorized` before storing/verifying the token, including direct `--token` attempts.

## Best Possible Solution

Confirm the production auth path with fresh browser-created and settings-created CLI tokens, decide the canonical host/registry behavior, make the CLI verify and persist that registry consistently, add focused regression coverage for host/token behavior, and consolidate the related Unauthorized reports under one maintainer-owned tracker or fix.
