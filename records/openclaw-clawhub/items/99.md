## Summary

Keep this issue open. The repository-specific cleanup policy for openclaw/clawhub only allows automated closure for PRs that are certainly implemented on main, not open issues. Current main also does not prove the reported Unauthorized-after-browser-login path is fixed: CLI defaults and discovery still point at https://clawhub.ai, login still verifies via /api/v1/whoami before saving the token, and the related www workaround PR was closed unmerged while related Unauthorized reports remain open.

## Best Possible Solution

Keep this issue open for maintainer follow-up alongside #72, #100, #115, and #702. The most useful next step is a current smoke test of `clawhub login`, `clawhub login --token`, and `clawhub whoami` against production, followed by a targeted fix and regression tests around registry discovery/canonical host behavior, callback registry propagation, Authorization behavior across redirects, token lookup, and clearer Unauthorized diagnostics. Preserve the existing https://www.clawhub.ai workaround comment as useful operational context until the production path is verified fixed.
