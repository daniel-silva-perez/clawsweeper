## Summary

Keep open. Repository policy keeps ClawHub issues open for maintainer judgment, and current main does not prove this login failure is resolved: the CLI still verifies `/api/v1/whoami` before persisting tokens, auth failures still surface as `Unauthorized`, the earlier redirect-fix PR was closed unmerged, and live host behavior now differs from the original workaround theory.

## What This Changes

The issue reports that `clawhub login` and `clawhub login --token` can complete browser or web-token creation but fail CLI token verification with `Unauthorized` before the token is stored.

## Best Possible Solution

Confirm the production auth path with fresh browser-created and settings-created CLI tokens, ensure the CLI verifies and stores the canonical registry consistently, add focused regression coverage for host/token behavior, and consolidate related Unauthorized reports under one maintainer-owned tracker or fix.
