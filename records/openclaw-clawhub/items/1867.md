Keep this PR open. Current main still has only token paste and loopback browser CLI auth; it has no `--device` flag, no device auth module, no `/cli/device/*` HTTP routes, and docs still describe `clawhub login --token` as the headless path. The PR is also paired with still-open #613 via closing syntax, so it remains an implementation candidate rather than an obsolete PR.
