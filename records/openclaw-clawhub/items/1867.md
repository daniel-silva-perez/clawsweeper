## Summary

Keep open: current main does not implement device-flow login, and this PR is still a useful implementation candidate for open #613, but it should not merge as-is because it exposes a new CLI auth path before the required server endpoints, verification UI, and auth-security controls exist. The earlier double JSON-read review comment appears addressed by the PR's follow-up commit, so the remaining blocker is end-to-end product/security completion rather than that client bug.

## What This Changes

The PR adds a ClawHub CLI `--device` login option, a new device-code request/polling helper with tests, and client calls to `/cli/device/code` and `/cli/device/token`.

## Best Possible Solution

Land or supersede this with an end-to-end device-flow auth implementation covering the CLI flag, Convex/server endpoints, verification UI, secure one-time device-code storage, expiry and rate limiting, token creation, docs, tests, and auth-security review; otherwise keep the flag hidden until those pieces are ready.
