## Summary

Keep open. Current main does not implement device-flow auth, and this PR is still an implementation candidate for the open headless-auth request rather than something already covered on main. The previously reported double JSON-read bug was fixed in the PR's second commit, but the branch still adds only the CLI-side flag/polling helper while the required server endpoints and verification UI are absent, so it needs maintainer and auth-security review before merge.

## What This Changes

The PR adds a `--device` login option to the ClawHub CLI, a device-code request/polling helper, and unit tests for the helper.

## Best Possible Solution

Land or supersede this with an end-to-end device-flow auth implementation covering the CLI flag, Convex/server endpoints, verification UI, secure one-time device-code storage, expiry/rate limiting, token creation, docs, tests, and auth-security review; otherwise close this PR and #613 together with a clear product decision.
