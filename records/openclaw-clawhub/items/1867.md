## Summary

Current main does not implement the PR’s device-flow CLI auth. The CLI still supports browser-loopback login and token paste only, the server route/API surface has no device-code or device-token endpoints, and the related feature issue #613 remains open with this PR as the implementation candidate.

## Best Possible Solution

Keep this PR open for maintainer review as the implementation path for #613. If the feature is desired, land a complete device-flow auth implementation that includes the CLI flag/client polling, server endpoints, verification UI, docs, and focused tests; otherwise close or supersede both the PR and paired issue together with a clear product decision.
