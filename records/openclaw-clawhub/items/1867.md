## Summary

Keep open. Current main does not implement device-flow login, and this PR is not obsolete; however, the proposed patch exposes a visible `--device` auth path while the required server endpoints, verification UI, token issuance controls, and docs are still absent. The earlier double JSON-read review comment appears resolved by the follow-up commit, so the remaining blocker is end-to-end auth completeness and security review.

## What This Changes

The PR adds a ClawHub CLI `--device` login option, a client-side device-code request/polling helper, and unit tests for headless authentication.

## Best Possible Solution

Land or supersede this with an end-to-end device-flow auth implementation covering the CLI flag, Convex/server endpoints, verification UI, secure one-time device-code storage, expiry and rate limiting, token creation, docs, tests, and maintainer auth-security review; keep the flag hidden until those pieces are ready.
