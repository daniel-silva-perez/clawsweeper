## Summary

This PR should stay open because current main still lacks device-flow CLI support and the PR is the open implementation path for the linked headless-auth feature request. The latest PR commit appears to address the previously reported double JSON-read bug, but the branch remains client-only: the server endpoints and verification UI it depends on are not present on main or in the provided PR file list, so it needs maintainer product and auth-security review before merge.

## What This Changes

The PR adds a `--device` option to both CLI login commands, a `deviceAuth` helper for requesting and polling device-flow tokens, and unit tests for the new client-side flow.

## Best Possible Solution

Land or supersede this with an end-to-end device-flow implementation that includes the CLI flow, server-side code/token endpoints, verification UI, documentation, tests, and auth-security review; otherwise keep the linked feature request open until maintainers decide not to support this auth mode.
