## Summary

Keep this issue open. The repository-specific ClawHub cleanup policy does not auto-close issues, and the reported production outage still reproduces: both the canonical site route and the direct Convex auth route return HTTP 500 for /api/auth/signin/github. Current main still configures GitHub OAuth with empty-string fallbacks for missing AUTH_GITHUB_ID/AUTH_GITHUB_SECRET, the auth routes are mounted from that config, and browser-based CLI token creation still depends on successful GitHub sign-in. The paired same-author implementation candidate PR #1720 and related production callback issue #1581 are both open, so this issue should remain visible for maintainer follow-up.

## Best Possible Solution

Keep this issue open while maintainers verify the production Convex OAuth environment variables and GitHub OAuth app callback settings, then review and land PR #1720 or replace it with an equivalent source-side missing-env validation and test/CI coverage. After deployment, run production smoke checks for /api/auth/signin/github and the browser-based clawhub login token flow.
