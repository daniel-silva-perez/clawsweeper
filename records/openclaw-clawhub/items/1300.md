## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open, and current main still contains the reported double JSON serialization paths for transfer request, skill rename, and skill merge. The related fix PRs cited in the discussion were closed without merging, and the current tests still assert string payloads rather than the plain-object contract.

## Best Possible Solution

Keep this issue open and land a scoped CLI fix: pass plain objects to apiRequest for transfer request, skill rename, and skill merge; update the command tests to assert object payload shapes; and consider hardening packages/clawhub/src/http.ts so pre-serialized string bodies are passed through defensively in both Node and Bun/curl paths.
