## Repair Work Prompt

Fix the CLI double JSON serialization bug reported here. In `packages/clawhub/src/cli/commands/transfer.ts` and `packages/clawhub/src/cli/commands/ownership.ts`, stop passing `JSON.stringify(...)` as the `apiRequest` body for transfer request, skill rename, and skill merge; pass plain objects consistent with existing working callers such as moderation commands. Update `transfer.test.ts` and `ownership.test.ts` to assert object body shape rather than string substrings. If you harden `packages/clawhub/src/http.ts` to tolerate already-serialized string bodies, cover both the Node fetch path and the Bun/curl path in tests. Use #973, #1251, and #1365 as context, with #1251 as the broadest prior implementation reference. Do not change server endpoint semantics, auth behavior, unrelated CLI commands, or generated files. No docs/changelog entry is expected for this narrow CLI bug fix unless the repo has an existing unreleased CLI bugfix convention nearby.
