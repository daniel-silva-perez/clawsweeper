## Repair Work Prompt

Fix the double JSON serialization bug for `clawhub transfer request`, `clawhub skill rename`, and `clawhub skill merge`. In `packages/clawhub/src/cli/commands/transfer.ts` and `packages/clawhub/src/cli/commands/ownership.ts`, pass plain object bodies to `apiRequest` instead of pre-stringifying. Update the command tests to assert object payload shape. If you also harden `packages/clawhub/src/http.ts` to tolerate string bodies, cover both the Node fetch path and the Bun/curl path. Related refs: openclaw/clawhub#1300, #973, #1251, #1365. Do not change API route names, server ownership semantics, unrelated CLI commands, or generated files. No changelog is expected for this narrow CLI bug unless maintainers request one.
