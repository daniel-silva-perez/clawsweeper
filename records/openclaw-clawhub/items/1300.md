## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open, and current main still contains the reported double-serialization path for transfer request, skill rename, and skill merge. The supplied related PR context shows prior fix attempts (#973, #1251, #1365), but they were closed without merging, and the current code/tests still preserve the buggy string-body contract.

## Best Possible Solution

Keep this issue open and land a scoped CLI fix: pass plain objects to apiRequest for transfer request, skill rename, and skill merge; update command tests to assert object payloads; and consider hardening packages/clawhub/src/http.ts so already-serialized string bodies pass through defensively in both Node and Bun/curl paths.
