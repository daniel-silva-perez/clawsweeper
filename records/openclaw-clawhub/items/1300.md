## Summary

Keep open. Current main still double-serializes the POST bodies for transfer request, skill rename, and skill merge: the commands pass JSON strings while both HTTP runtime paths stringify `args.body` again, and the server handlers then read object fields. Related PRs #973, #1251, and #1365 were closed without merging, and the ClawHub cleanup policy keeps issues open.

## What This Changes

The issue asks to fix CLI transfer request, skill rename, and skill merge so they pass plain JSON objects to `apiRequest` instead of pre-stringified bodies.

## Best Possible Solution

Land a focused CLI fix that sends plain object bodies from the affected commands, updates command tests to assert object payloads, and optionally hardens the shared HTTP client against pre-serialized bodies with coverage for both Node fetch and Bun/curl paths.
