## Repair Work Prompt

Fix the ClawHub CLI double JSON serialization bug for transfer request, skill rename, and skill merge. In `packages/clawhub/src/cli/commands/transfer.ts` and `packages/clawhub/src/cli/commands/ownership.ts`, pass plain object bodies to `apiRequest` instead of `JSON.stringify(...)`. Update `packages/clawhub/src/cli/commands/transfer.test.ts` and `packages/clawhub/src/cli/commands/ownership.test.ts` to assert object payloads rather than string substrings. Consider hardening `packages/clawhub/src/http.ts` so already-serialized string bodies are not double-encoded, but cover both Node fetch and Bun/curl paths if you do. Use closed PR #1251 as implementation context, but keep the PR focused and do not change unrelated CLI behavior, auth, registry discovery, or server ownership semantics. Include a concise changelog or release-note entry only if the repo has a current CLI changelog pattern for bug fixes.
