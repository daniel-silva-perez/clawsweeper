## Summary

Keep this issue open. Current main still contains the reported double JSON serialization path: transfer request, skill rename, and skill merge pass pre-stringified JSON to `apiRequest`, and both HTTP runtime paths stringify the body again before the server reads object fields. The related fix PRs #973, #1251, and #1365 are closed unmerged, so there is no merged fix yet.

## Best Possible Solution

Keep this issue open and land a focused CLI fix: pass plain object bodies from transfer request, skill rename, and skill merge; update command tests to assert object payloads; and, if hardening `apiRequest`, apply the same pre-serialized-string behavior to both Node and Bun/curl paths with focused HTTP tests.
