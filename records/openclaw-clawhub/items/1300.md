## Summary

Keep this issue open. The ClawHub policy only permits auto-closing PRs, and current main still contains the reported double-serialization path for transfer request, skill rename, and skill merge: the command handlers pass JSON strings to apiRequest, while both Node and Bun/curl HTTP paths stringify args.body again.

## Best Possible Solution

Keep this issue open and land a focused CLI fix: pass plain object bodies from transfer request, skill rename, and skill merge; update command tests to assert object payloads; and consider narrowly hardening the shared HTTP helper so pre-serialized JSON strings are not double-encoded in either runtime path.
