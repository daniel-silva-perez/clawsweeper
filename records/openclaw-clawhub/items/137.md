## Summary

Keep open. ClawHub policy for this cleanup pass does not auto-close issues, and current main still has a credible publish-discovery failure path where ordinary publishes can store a hidden pending-scan state that public search, listing, and unauthenticated detail paths suppress.

## What This Changes

A successfully published skill is reported as absent from CLI search, explore, inspect, and the web owner dashboard despite publish confirmation.

## Best Possible Solution

Separate version review from skill-level public availability: keep the last approved latest version discoverable while a new version scans, make first-time pending skills clearly owner-visible with actionable status, and ensure scan finalization updates moderation state, skillSearchDigest, embedding/search visibility, CLI inspect/search, and web dashboard state consistently.
