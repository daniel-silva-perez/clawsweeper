## Summary

Keep open. The repository policy for this cleanup pass keeps ClawHub issues open, and current main still supports the reported public-discovery failure mode: ordinary publishes can be stored as hidden while scan work is pending, and public search/list/detail paths suppress hidden skills. The owner dashboard path appears improved on current main, but that does not resolve search, explore, or unauthenticated inspect visibility.

## What This Changes

The issue reports that a successfully published skill is not discoverable through CLI search, explore, inspect, or the web dashboard after publish returns success.

## Best Possible Solution

Keep the last approved latest version discoverable while a new version scans, make first-time pending skills clearly visible to their owner with actionable status, and ensure scan finalization consistently updates moderation state, skillSearchDigest, search/list/detail hydration, CLI inspect/search/explore, and dashboard state.
