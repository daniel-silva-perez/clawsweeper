## Summary

Keep this issue open. The ClawHub cleanup policy for this pass keeps issues open, and current main still has a valid path matching the report: ordinary publishes can be written as hidden with `pending.scan`, public search/explore/inspect surfaces filter non-active skills, and the search digest mirrors that moderation state. Owner-facing dashboard/status handling has improved, but the public discoverability problem while scans/indexing are pending is not fully resolved.

## What This Changes

The issue requests that a successfully published skill become discoverable via CLI search, explore, inspect, and the web dashboard instead of disappearing after publish.

## Best Possible Solution

Keep this open for maintainer triage. The best direction is to separate version-review state from skill-level public visibility: keep an already-approved latest version discoverable while a new version scans, keep first-time unapproved skills owner-visible with a clear pending status, and ensure `skillSearchDigest` plus embedding/search state are updated or backfilled when scans finalize. Coordinate with #423, #960, and #1743 rather than closing this report independently.
