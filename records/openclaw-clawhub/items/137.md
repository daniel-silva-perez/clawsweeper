## Summary

Keep this issue open. The ClawHub cleanup policy does not auto-close issues, and current main still has a credible publish-discovery failure path: ordinary publishes can set the skill row to hidden while scans are pending, and the public search, listing, and unauthenticated detail paths filter hidden skills out. Owner-facing dashboard/status handling has improved, but the public discoverability problem reported here remains tied to the active #423, #960, and #1743 cluster.

## What This Changes

The issue reports that a successfully published skill should appear in CLI search, explore, inspect, and the web dashboard instead of disappearing after publish confirmation.

## Best Possible Solution

Separate version review from skill-level public availability: keep the last approved latest version discoverable while a new version scans, make first-time pending skills clearly owner-visible with actionable status, and ensure scan finalization updates moderation state, skillSearchDigest, and embedding/search visibility consistently across CLI and web surfaces.
