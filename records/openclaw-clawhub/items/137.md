## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open in this pass, and current main still maps to the reported discovery failure: ordinary publishes can be marked `hidden` with `pending.scan`, search/explore/inspect public paths filter non-active skills, and digest syncing preserves that moderation state. Current main does improve owner-facing visibility, but it does not fully resolve public discoverability while scans/indexing are pending.

## Best Possible Solution

Keep this issue open for maintainer triage. The best direction is to separate version-review state from skill-level public visibility: keep an already-approved latest version discoverable while a new version scans, keep new unapproved skills owner-visible with clear pending status, and ensure `skillSearchDigest`/embedding state is updated or backfilled when scans finalize. Coordinate with related open reports #423, #960, and #1743 rather than closing this report independently.
