## Summary

Keep open. The ClawHub cleanup policy does not auto-close issues, and current main still has a credible publish-discovery failure path: ordinary non-trusted publishes can write a hidden pending-scan state to the skill row, while public search, list, and unauthenticated detail paths filter hidden skills out until scan finalization or moderation changes that state.

## What This Changes

The issue reports that a successfully published skill is not discoverable through CLI search, explore, inspect, or the web dashboard.

## Best Possible Solution

Separate version review from skill-level public availability: keep the last approved latest version discoverable while a new version scans, make first-time pending skills clearly owner-visible with actionable status, and ensure scan finalization updates moderation state, skillSearchDigest, embedding/search visibility, CLI inspect/search, and web dashboard state consistently.
