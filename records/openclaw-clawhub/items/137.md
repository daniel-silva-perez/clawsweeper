## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still contains a credible publish-discovery failure path: successful publishes can store a hidden pending-scan moderation state, while public search, list, and unauthenticated detail surfaces suppress hidden skills until scan finalization or moderation changes that state.

## What This Changes

A published skill returns success but is missing from CLI search, explore, inspect, and the web dashboard owner view.

## Best Possible Solution

Separate version review from skill-level public availability: keep the last approved latest version discoverable while a new version scans, make first-time pending skills clearly visible to their owner with actionable status, and ensure scan finalization consistently updates moderation state, skillSearchDigest, search/list/detail hydration, CLI inspect/search/explore, and dashboard state.
