## Summary

Keep this issue open. The repository cleanup policy keeps ClawHub issues open, and current main still has a valid public-discovery failure mode: ordinary publishes can set a skill to hidden while scans are pending, while search, explore/listing, and unauthenticated inspect/detail paths filter hidden skills out. Owner-facing dashboard/status handling has improved, but the core public discoverability problem is not fully resolved.

## What This Changes

The issue requests that successfully published skills become discoverable through CLI search/explore/inspect and the web dashboard instead of remaining invisible after publish.

## Best Possible Solution

Keep this open for maintainer triage. The best direction is to separate version-review state from skill-level public visibility: keep an already-approved latest version discoverable while a new version scans, keep first-time unapproved skills owner-visible with an explicit pending status, and ensure skillSearchDigest plus embedding/search state are updated or backfilled when scans finalize. Coordinate this with #423, #960, and #1743 rather than closing reports independently.
