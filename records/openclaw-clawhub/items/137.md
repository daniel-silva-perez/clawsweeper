## Summary

Keep this issue open. The repository-specific policy only permits automated closure for ClawHub pull requests that are certainly implemented on current main, and this item is an issue. Substantively, the reported problem is still plausible on main: successful publishes can remain hidden pending moderation/scans, while public search, explore, and inspect paths intentionally drop hidden, removed, soft-deleted, or malware-blocked skills. The discussion also has a post-fix recurrence and related open reports #423, #960, and #1743.

## Best Possible Solution

Keep this issue open, or explicitly fold it into a canonical maintainer-owned tracking issue for published skills missing from search. The concrete path is to audit the affected production slugs, repair stale digest/embedding/scan rows, decide whether public discovery should preserve the last approved version while a new version is under review, and make dashboard/inspect responses explain every hidden or blocked state to the owner.
