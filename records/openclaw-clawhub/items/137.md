## Summary

Keep this issue open. ClawHub cleanup policy keeps issues open, and current main still has a credible publish-discovery failure path: ordinary publishes can write a hidden pending-scan state onto the skill row, while public search/list/detail hydration filters hidden skills out until scan finalization or manual moderation changes that state.

## What This Changes

Published skills should be discoverable through CLI search, explore, inspect, and the web dashboard immediately after a successful publish, or show a clear owner-visible pending state instead of disappearing.

## Best Possible Solution

Separate version review from skill-level public availability: keep the last approved latest version discoverable while a new version scans, make first-time pending skills clearly owner-visible with actionable status, and ensure scan finalization updates moderation state, skillSearchDigest, embedding/search visibility, CLI inspect/search, and web dashboard state consistently.
