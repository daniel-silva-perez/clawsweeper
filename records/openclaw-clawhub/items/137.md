## Summary

Keep this issue open under the ClawHub cleanup policy: it is an issue, and this pass only auto-closes PRs that are certainly implemented on main. Substantively, current main has improved parts of the flow, but the original failure mode still maps to real product behavior: ordinary publishes are hidden while scans are pending, and hidden skills are filtered out of public search, explore/list, and unauthenticated inspect paths.

## Best Possible Solution

Keep this open for maintainer triage. The best product/code direction is to separate version-review state from public skill visibility: keep an already-approved version discoverable while a new version scans, keep new unapproved skills owner-visible with clear pending status, and ensure search digest/vector state is updated or backfilled when scans finalize.
