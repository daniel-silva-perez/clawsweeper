## Summary

Keep open. The ClawHub cleanup policy for this pass keeps issues open, and current main only appears to mitigate the reported double-pagination trigger by scheduling package and skill digest syncs separately; the named helper and its package sibling still contain paginated loops, so production verification is still needed before resolving this publish/import cluster.

## What This Changes

Publishing or importing a skill fails with Convex's multiple-paginated-query error in `syncSkillSearchDigestsForOwnerPublisherId`, blocking updates for affected publishers.

## Best Possible Solution

Keep the issue open until maintainers verify the deployed scheduler mitigation has stopped the Convex failures across CLI publish, upload publish, and GitHub import. If failures remain for multi-page owner publishers, implement a narrow self-scheduling cursor continuation for both owner-publisher digest helpers and consolidate the duplicate publish/import reports around that fix.
