Keep open. This is an open ClawHub issue, and the repository cleanup policy for this pass only permits auto-closing PRs that are certainly implemented on current main. ## What This Changes

Publishing or importing skill versions should not fail with Convex's multiple-paginated-query error from `syncSkillSearchDigestsForOwnerPublisherId`.

## Best Possible Solution

Keep this issue open for maintainer follow-up.
