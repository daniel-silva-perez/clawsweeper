## Summary

Keep open. The repository cleanup policy for ClawHub issues does not allow auto-closing issues, and current main still has the plausible bug path described in the existing ClawSweeper comment: public skill detail data can expose duplicate/fork metadata for a referenced skill without verifying that referenced skill is still public.

## Best Possible Solution

Keep this issue open for maintainer handling. The likely code fix is to make convex/skills.ts suppress forkOf and canonical response metadata unless the referenced skill passes the same public visibility rules as normal skill detail pages, then add a regression test for a duplicate target with softDeletedAt set. If the visible production case is stale data, a moderator can also clear the row through the existing duplicate-clear path.
