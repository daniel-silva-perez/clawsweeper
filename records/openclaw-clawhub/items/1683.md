## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy only allows auto-closing PRs that are definitely implemented on main. Current main also still has a plausible path where a public skill detail response can return duplicate/fork metadata for a referenced skill document without checking whether that referenced skill is still public.

## Best Possible Solution

Keep this issue open for maintainer handling. The likely fix is to make convex/skills.ts suppress forkOf and canonical response metadata unless the referenced skill passes the same public visibility rules as normal skill detail pages, then add a focused regression test for a duplicate target with softDeletedAt set. If the current production row is stale data, a moderator can also clear forkOf/canonicalSkillId for global-search through the existing duplicate-clear path.
