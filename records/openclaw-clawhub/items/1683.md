## Summary

Keep open. This is a ClawHub issue, and the repo policy says this cleanup flow should keep ClawHub issues open. The current public skill detail path still appears to expose duplicate/canonical metadata for referenced skills without applying the same public visibility guard used for normal skill pages, matching the deleted apexsearch duplicate attribution report.

## Best Possible Solution

Keep this issue open and land a narrow fix in convex/skills.ts: for public getBySlug, only return forkOf and canonical metadata when the referenced skill and owner pass the same public visibility rules as normal skill detail pages. Add a regression test in convex/skills.public.test.ts for a duplicate target that is soft-deleted or otherwise non-public. If the live data should also be cleaned, clear the affected duplicate relationship through the existing moderator setDuplicate path.
