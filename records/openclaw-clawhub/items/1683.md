## Repair Work Prompt

Fix the public ClawHub duplicate/canonical reference visibility bug reported for the global-search skill page. In current main, `convex/skills.ts:getBySlug` returns `forkOf` and `canonical` metadata for referenced skills fetched by id even when the referenced skill is deleted, hidden, removed, or otherwise not public. Update the public detail path so these references are returned only when the referenced skill passes the normal public visibility rules and its owner/publisher is public. Preserve staff/management behavior, especially `getBySlugForStaff`, so maintainers can still inspect relationships. Add focused regression tests in `convex/skills.public.test.ts` for a public skill whose `forkOf`/`canonicalSkillId` points at a soft-deleted or moderation-hidden target, expecting public `forkOf`/`canonical` to be null. Prefer a small helper if it avoids duplicating visibility logic. Do not edit seeded skill content, published skill files, or production data; note in the PR that maintainers may still choose to clear the existing production relationship for global-search after deploy.
