## Summary

Keep open. This is a ClawHub issue, and the repository policy says ClawHub issues should not be auto-closed by this cleanup flow. The current main branch also still has a plausible code path for the reported problem: public skill detail responses can return duplicate/fork metadata for a referenced skill without checking that the referenced skill is still public.

## Best Possible Solution

Keep this issue open for maintainer handling. The likely code fix is to make convex/skills.ts suppress forkOf and canonical response metadata unless the referenced skill passes the same public visibility rules as normal skill detail pages, then add a regression test for a duplicate target with softDeletedAt set. If the current production case is only stale data, a moderator should also clear the row through the existing duplicate-clear path.
