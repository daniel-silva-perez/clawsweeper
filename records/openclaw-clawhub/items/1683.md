## Summary

Keep open. This is a ClawHub issue, and the supplied repository policy says this cleanup flow should keep issues open. Current main also still has a plausible unresolved product/code path: public skill detail data can expose duplicate/fork metadata for a referenced skill without checking whether that referenced skill is still public, which matches the reported deleted `apexsearch` duplicate attribution.

## Best Possible Solution

Keep this issue open for maintainer handling. The likely fix belongs in `convex/skills.ts`: only return `forkOf` and `canonical` metadata when the referenced skill and owner pass the same public visibility rules as normal skill detail pages, then add a regression test in `convex/skills.public.test.ts` for a duplicate target with `softDeletedAt` or non-active moderation state. If the live report is also stale production data, clear the affected row through the existing moderator duplicate-clear path.
