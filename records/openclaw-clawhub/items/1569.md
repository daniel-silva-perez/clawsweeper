## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open, and current main still has a user-to-user skill transfer model: the CLI/API send `toUserHandle`, the backend resolves only users or personal user publishers, the transfer table stores `fromUserId`/`toUserId`, and accepting a transfer moves ownership to the recipient's personal publisher rather than an organization publisher. Related #1431 and #1675 remain open, and #1635 is an open, unmerged implementation candidate for the broader org-aware ownership-transfer work.

## Best Possible Solution

Keep this issue open and resolve it through review and landing of #1635 or a maintainer-selected successor. The fix should make skill ownership transfers target publishers/organizations rather than only users, preserve or migrate existing user-to-user transfer state safely, update Convex mutations/queries, HTTP API, CLI schema/UX, docs, authorization rules, audit logs, and tests for personal-to-org, org-to-user, org-to-org, and existing user-to-user flows. After that lands on main, this issue plus #1431 and #1675 can be closed as implemented or superseded by the merged change.
