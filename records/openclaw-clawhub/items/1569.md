## Summary

Keep this issue open. ClawHub cleanup policy only permits automated closes for pull requests that current main certainly implements, and current main still models skill ownership transfers as user-to-user rather than publisher/org-targeted. The reported `User not found` path still matches the code: transfer requests require `toUserHandle`, org publishers are excluded by the recipient resolver, transfer rows store user IDs, and acceptance assigns the recipient's personal publisher. Related #1431 remains open, and #1635 is an open, unmerged implementation candidate rather than shipped main behavior.

## Best Possible Solution

Keep this issue open and resolve it through #1635 or a maintainer-selected successor. The durable fix should make transfers publisher-targeted rather than user-only, update the Convex schema and mutations, HTTP API, CLI contract, docs, authorization rules, and tests, with explicit coverage for personal-to-org, org-to-user, and org-to-org flows. Once that lands on main, this issue and #1431 can be closed as implemented or superseded by the merged PR.
