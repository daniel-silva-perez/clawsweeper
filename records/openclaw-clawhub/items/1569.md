## Summary

Keep this issue open. ClawHub cleanup policy only permits auto-closing pull requests that are certainly implemented on main, not issues. Substantively, current main still treats skill ownership transfers as user-to-user: the CLI/API send `toUserHandle`, the backend resolves only active users or personal user publishers, and the transfer table stores `fromUserId`/`toUserId` rather than publisher targets. The related open report #1431 and open PR #1635 confirm this is still unresolved product work.

## Best Possible Solution

Keep this issue open and use #1635, or a maintainer-selected successor, to implement publisher-target ownership transfers in core. The likely path is to add source/target publisher fields to transfer records, accept org handles in the API and CLI, enforce owner/admin acceptance rules for org publishers, and cover user-to-org, org-to-user, and org-to-org transfers in tests. After that lands, close this issue and #1431 as implemented or consolidate them under the canonical tracker.
