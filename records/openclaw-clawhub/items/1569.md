## Summary

Keep open. Current main still implements skill ownership transfers as user-to-user only, and ClawHub cleanup policy keeps issues open unless they are PRs certainly implemented on main. Related #1431 and #1675 remain open, and #1635 is an open, unmerged implementation candidate for org-aware transfers.

## What This Changes

Allow skills owned by a personal account to be transferred or re-homed to an organization so org members can publish and update them.

## Best Possible Solution

Keep this issue open and resolve it through review and landing of #1635 or a maintainer-selected successor. The fix should make skill ownership transfers target publishers/organizations rather than only users, preserve existing user-to-user behavior and pending transfer state safely, and update Convex schema/mutations/queries, HTTP API, CLI UX/schema, docs, authorization, audit logs, and tests for personal-to-org, org-to-user, org-to-org, and current user-to-user flows.
