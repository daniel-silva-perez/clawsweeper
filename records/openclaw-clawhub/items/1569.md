## Summary

Keep open. This is an open ClawHub issue, and the repository policy for this review only permits auto-closing ClawHub PRs that are certainly implemented on main. Substantively, current main still implements skill transfers as user-to-user: the API and CLI accept `toUserHandle`, recipient resolution rejects org publishers, transfer records store `fromUserId`/`toUserId`, and the org migration plan still lists publisher-target transfers as future work. The related #1431 remains open and #1635 is an open implementation candidate, not a merged fix.

## Best Possible Solution

Keep this issue open and let #1635, or a maintainer-selected successor, implement publisher-target ownership transfers in core. The likely product path is to add source/target publisher fields, update the API and CLI to accept org publisher handles, enforce owner/admin acceptance rules for org publishers, and test user-to-org, org-to-user, and org-to-org transfers. After that merges, close this issue and #1431 as implemented or consolidate them under the canonical tracker.
