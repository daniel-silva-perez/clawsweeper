## Summary

Keep this issue open. Repository policy for openclaw/clawhub only permits automated close proposals for pull requests that are certainly implemented on current main, and this item is an open issue. The current source still confirms the reported gap: skill transfers target users, organization publishers are not accepted as recipients, and the docs still describe publisher-target transfers as planned work. The related #1431 is the same remaining product gap, and #1635 is an open implementation candidate rather than merged main.

## Best Possible Solution

Keep this issue open and resolve it through #1635 or a maintainer-selected successor. The durable fix should make skill transfer targets publisher-based rather than user-only, update the Convex schema and mutations, HTTP API, CLI schemas/commands, docs, authorization rules, and tests, with explicit coverage for personal-to-org, org-to-user, and org-to-org flows.
