## Summary

Keep this issue open. It is an open ClawHub issue, and the repository cleanup policy only permits automated closes for pull requests that current main certainly implements. Substantively, current main still models skill ownership transfers as user-to-user: the request path accepts `toUserHandle`, organization publishers are excluded by the recipient resolver, transfer rows store `fromUserId` and `toUserId`, and acceptance assigns the recipient user's personal publisher. Related #1431 remains open, and #1635 is an open, unmerged implementation candidate rather than shipped main behavior.

## Best Possible Solution

Keep this issue open and resolve it through #1635 or a maintainer-selected successor. The durable fix should make ownership transfers publisher-targeted rather than user-only, update Convex schema and mutations, HTTP API, CLI contract, docs, authorization rules, and tests, with explicit coverage for personal-to-org, org-to-user, and org-to-org flows. After that lands on main, this issue and #1431 can close as implemented or superseded by the merged PR.
