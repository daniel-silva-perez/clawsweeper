## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy keeps issues open; current main also still implements skill transfers as user-to-user only, so the requested personal-to-organization transfer path is not implemented. Related #1431 and #1675 remain open, and #1635 is an open, unmerged implementation candidate for publisher/org-aware transfers.

## What This Changes

The issue asks for a way to transfer existing personal skills to an organization so organization members can publish and update them.

## Best Possible Solution

Implement a publisher-targeted ownership transfer model that supports personal-to-org, org-to-user, and org-to-org transfers while preserving existing user-to-user behavior, aliases, stats, audit logs, and install URLs where possible; #1635 or a maintainer-selected successor is the right place to consolidate that work.
