## Summary

Keep this issue open. The reporter's request is still not implemented on current main: skill transfer remains user-to-user, the CLI/API contract still sends `toUserHandle`, the backend resolver only accepts users or personal user publishers, and accepting a transfer moves ownership to the recipient's personal publisher. Related #1431 and #1675 remain open, and #1635 is an open, unmerged implementation candidate for org-aware ownership transfers.

## Best Possible Solution

Keep this issue open and resolve it through review and landing of #1635 or a maintainer-selected successor. The fix should make skill ownership transfers target publishers/organizations rather than only users, preserve or migrate existing user-to-user transfer state safely, and update Convex mutations/queries, HTTP API, CLI schema/UX, docs, authorization rules, audit logs, and tests for personal-to-org, org-to-user, org-to-org, and existing user-to-user flows. After that lands on main, this issue plus #1431 and #1675 can be closed as implemented or superseded by the merged change.
