## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open, and current main still does not support transferring a skill from a personal publisher to an organization publisher. The transfer path remains user-recipient based, the schema stores `fromUserId`/`toUserId`, and related #1431 and #1675 remain open while #1635 is an open, unmerged implementation candidate.

## Best Possible Solution

Keep this issue open and resolve it through #1635 or a maintainer-selected successor. The durable fix should make skill ownership transfers target publishers/orgs rather than only users, update the Convex schema and mutations, HTTP API, CLI contract, docs, authorization rules, and tests, and cover personal-to-org, org-to-user, and org-to-org flows. After that lands on main, this issue plus #1431/#1675 can close as implemented or superseded by the merged change.
