## Summary

Keep this issue open. ClawHub cleanup policy only permits automated close proposals for pull requests that current main certainly implements, and this is an open issue. Substantively, current main still models skill ownership transfers as user-to-user: requests require `toUserHandle`, organization publishers are excluded by recipient resolution, the transfer table stores user IDs, and acceptance assigns the recipient's personal publisher. The related #1431 remains open, and #1635 is an open implementation candidate rather than merged main.

## Best Possible Solution

Keep this issue open and resolve it through review and landing of #1635 or a maintainer-selected successor. The durable fix should make transfer targets publisher-based, update the Convex schema and mutations, HTTP API, CLI contract, docs, authorization rules, and tests, with explicit coverage for personal-to-org, org-to-user, and org-to-org flows.
