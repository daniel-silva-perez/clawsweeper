Keep open. This is a ClawHub issue, and the repository policy says issues are not auto-closed. Current main also still leaves the core reported work unfinished: folder picking is hardened, but publish-plugin has no package changelog preview/history action and no stale auto-preview overwrite protection. ## Best Possible Solution

Keep this issue open while maintainers review PR #1287 or land an equivalent patch on main. The implementation should preserve reliable plugin directory upload, add a package generateChangelogPreview action based on packageReleases history with package-specific fallback text, wire publish-plugin to request/cancel previews without overwriting manual edits, and add regression coverage for the async race and package-history changelog behavior.
