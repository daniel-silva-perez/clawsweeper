## Summary

Keep this issue open. The ClawHub cleanup policy only allows auto-closing PRs that are certainly implemented on current main, and this item is an issue. Current main also still lacks GitHub issue forms, issue-template config, or support auto-labeling for the requested intake buckets.

## Best Possible Solution

Keep this issue open for maintainer/product handling. A good implementation would add .github/ISSUE_TEMPLATE YAML forms and config.yml for the requested support buckets, standardize the label taxonomy, use form-level labels where appropriate, add a narrowly scoped triage workflow only if static issue-form labels are insufficient, and point users to maintainer-approved canonical support threads before opening duplicates.
