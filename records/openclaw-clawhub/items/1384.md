## Summary

Keep this issue open. The repository-specific ClawHub cleanup policy allows auto-closing only PRs that are certainly implemented on current main, and this is an issue. Current main also still lacks `.github/ISSUE_TEMPLATE` issue forms or broad support-intake labeling; the new rescan guidance workflow only handles a narrow labeled rescan path.

## Best Possible Solution

Keep this issue open for maintainer/product follow-up. The best path is to approve the public support categories and label taxonomy, add `.github/ISSUE_TEMPLATE` YAML forms plus `config.yml`, use static form-level labels where possible, point forms at maintainer-approved canonical threads, and only add a small `issues: write` triage workflow if issue-form labels are insufficient.
