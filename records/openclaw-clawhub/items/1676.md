## Summary

Keep this issue open. Current main has substantial automatic GitHub profile and handle sync by immutable GitHub numeric ID, and the synced handle should flow into personal publisher and transfer-target lookup paths. However, this is a ClawHub issue, and the repository cleanup policy only allows auto-closing PRs that are certainly implemented on main. The remaining choices around explicit rename, old-handle reservation or aliases, profile-handle redirects, deployment status, and org ownership still need maintainer product judgment.

## What This Changes

The issue requests a supported way for ClawHub user handles and owner URLs to follow GitHub username changes, with safe handling for redirects, reservations, and transfer target lookup.

## Best Possible Solution

Leave this issue open for maintainer judgment. If maintainers accept the current automatic sync behavior as the product answer, verify the relevant deployment and document the behavior before manually closing. Otherwise, define and implement the remaining handle-rename policy: explicit authenticated rename versus automatic sync, old profile-handle alias or reservation semantics, canonical skill URL redirect guarantees, API/CLI/docs coverage, and coordination with org/publisher ownership in #1675.
