## Summary

Keep open. Current main now covers the core automatic GitHub rename path for derived personal handles, including login-triggered sync by immutable GitHub numeric ID, claim/conflict checks, personal publisher updates, transfer lookup by synced publisher handle, and canonical owner redirects. However, this is an issue in openclaw/clawhub, and the supplied cleanup policy only permits auto-closing PRs; the remaining choices around opt-in versus automatic rename, old-handle alias/reservation semantics, public URL behavior, docs/API coverage, and org ownership coordination still need maintainer judgment.

## What This Changes

The issue asks ClawHub to handle GitHub username changes so profile handles, owner URLs, whoami, and transfer-target lookup no longer remain stuck on the original GitHub login.

## Best Possible Solution

Maintainers should decide and document the handle-rename policy, then either bless the current automatic derived-handle sync as the intended behavior and fill any missing alias/reservation/docs gaps, or specify an explicit authenticated rename UX/API with tests and coordination with publisher/org ownership.
