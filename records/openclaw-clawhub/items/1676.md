## Summary

Keep this issue open. Current main substantially covers the derived personal-handle sync path after a GitHub username change, including login-triggered profile refresh, immutable GitHub numeric-ID lookup, claim/conflict checks, personal publisher synchronization, transfer lookup through personal publisher handles, and canonical owner redirects. However, this is an open ClawHub issue, and the repository cleanup policy only permits auto-closing implemented PRs; the remaining opt-in rename, old-handle alias/reservation, docs/API/CLI, and org-ownership semantics still need maintainer product judgment.

## What This Changes

The issue requests a supported way for ClawHub handles, profile/skill owner URLs, and transfer targets to follow GitHub username changes without deleting and recreating the account.

## Best Possible Solution

Keep the automatic sync path if that is the intended default, then define the remaining handle-rename product contract: explicit opt-in versus automatic rename, old-handle alias/reservation behavior, profile and skill URL redirects, API/CLI/docs coverage, and how this interacts with publisher/org ownership.
