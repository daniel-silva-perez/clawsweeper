## Summary

Keep this issue open. Current main has substantial GitHub username/profile auto-sync support, including login-triggered background sync by immutable GitHub numeric ID, derived-handle updates, personal-publisher refresh, and skill owner redirects. But the ClawHub cleanup policy only permits automated closure for pull requests that are certainly implemented on main; issues stay open for maintainer judgment, especially because the requested opt-in rename, old-handle reservation/profile redirect, and broader publisher/org ownership details are still product decisions.

## Best Possible Solution

Leave this issue open for maintainer judgment. If maintainers accept the current auto-sync behavior as the product answer, they can close it manually after confirming production has v0.11.0-era sync behavior; otherwise this issue should define the remaining handle-rename flow: opt-in versus automatic policy, old-handle reservation and profile redirects, API/docs coverage, and coordination with publisher/org ownership work in #1675.
