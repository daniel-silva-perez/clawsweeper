## Summary

Keep this issue open. Current main contains a substantive auto-sync implementation for GitHub username changes, including handle updates for derived handles and canonical skill URL redirects, but the repository-specific ClawHub cleanup policy only permits automated closure of PRs that are certainly implemented on main, not issues. Maintainer judgment is still needed on the remaining product details such as opt-in rename UX, old-handle reservation, profile redirects, and deployment status.

## Best Possible Solution

Leave this issue open for maintainer judgment under the ClawHub cleanup policy. If the current auto-sync behavior is deployed and accepted as the product answer, a maintainer can manually close it as implemented; otherwise the follow-up should define a first-class handle rename/sync flow covering opt-in UX, old-handle reservation or alias records, profile and skill redirects, transfer-target lookup behavior, API/docs updates, and regression coverage, while coordinating broader publisher/org ownership work with #1675.
