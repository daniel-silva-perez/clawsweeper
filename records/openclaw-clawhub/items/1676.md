## Summary

Keep this issue open. Current main has substantial GitHub username/profile auto-sync support, including login-triggered sync by immutable GitHub numeric ID, derived-handle updates when claimable, and personal-publisher handle sync. But this is an issue, and the openclaw/clawhub cleanup policy only permits automated closure for PRs that are definitely implemented on main. The remaining policy questions around explicit versus automatic rename, old-handle aliases/reservations, and org/publisher ownership should stay with maintainers.

## Best Possible Solution

Leave this issue open for maintainer judgment. If maintainers accept the current automatic sync behavior as the product answer, they can close it manually after confirming production deployment. Otherwise, define the remaining handle-rename policy: explicit versus automatic rename, old-handle profile aliases and reservations, skill URL alias semantics, API/CLI/docs coverage, and coordination with publisher/org ownership work in #1675.
