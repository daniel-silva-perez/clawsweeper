## Summary

Keep this issue open. Current main separates package and skill digest rebuild scheduling, which mitigates the original two-paginated-query publisher-trigger path, but each owner-publisher digest helper can still call `.paginate()` more than once inside a single internal mutation for larger owner catalogs. The related open publish/profile reports and the closed, unmerged fix PR make this a concrete backend cleanup cluster rather than a cleanup-close candidate.

## What This Changes

The issue asks for first-time publisher CLI publishing and settings/profile saves to stop failing with Convex's multiple-paginated-query error in `ensurePersonalPublisherForUser` and owner-publisher digest sync.

## Best Possible Solution

Refactor owner-publisher digest rebuilds into cursor-carrying scheduled pages so each internal mutation performs at most one indexed paginated read, then add regression tests for multi-page package and skill owners and consolidate duplicate reports after the backend fix is deployed.

## Repair Work Prompt

Fix the Convex owner-publisher digest sync pagination bug reported by this issue and related publish/profile reports. In `convex/functions.ts`, refactor `syncPackageSearchDigestsForOwnerPublisherId` and `syncSkillSearchDigestsForOwnerPublisherId` plus their internal mutation wrappers so each internal mutation invocation performs at most one indexed `.paginate()` call, accepts/carries a continuation cursor, processes that page, and schedules the next page with `ctx.scheduler.runAfter(0, ...)` until `isDone`. Preserve the existing `by_owner_publisher` indexes and keep package and skill syncs scheduled separately. Do not replace the owner scans with unbounded `.collect()` and do not broaden this into unrelated publish, profile, search, or moderation changes. Add focused regression tests in `convex/functions.test.ts` for multi-page package and skill owners and for cursor continuation. Related refs: this issue, #1170, #1175, #1188, #1201, and closed unmerged PR #1183. Regenerate Convex generated files if function arg validators change.
