## What This Changes

The issue reports first-time publisher failures where CLI publish and settings/profile mutations hit Convex server errors through `ensurePersonalPublisherForUser` and owner-publisher digest sync.

## Best Possible Solution

Refactor owner-publisher digest rebuilds into cursor-carrying scheduled page mutations so each internal mutation performs at most one indexed paginated read, self-schedules until complete, and has regression coverage for multi-page package and skill owners.

## Repair Work Prompt

Fix the owner-publisher digest sync Convex pagination bug for publish/profile flows. In current main, `convex/functions.ts` schedules package and skill digest rebuilds separately, but `syncPackageSearchDigestsForOwnerPublisherId` and `syncSkillSearchDigestsForOwnerPublisherId` still loop `.paginate()` inside a single internal mutation. Replace that with cursor-carrying scheduled pages for both package and skill owner-publisher rebuilds, preserving the existing `by_owner_publisher` indexed reads and existing single package/skill trigger behavior. Do not switch to unbounded `.collect()` and do not change the CLI API or public v1 publish route. Add regression coverage in `convex/functions.test.ts` for multi-page package and skill owners: each internal mutation invocation should issue at most one paginated read, process that page, and schedule itself with `continueCursor` until done. Related refs: #1170, #1175, #1188, #1201, and closed unmerged PR #1183. No changelog entry is expected unless the repo has a current backend bugfix changelog convention.
