## Summary

Keep this issue open. ClawHub issue cleanup policy keeps issues open, and current main only partially mitigates the originally reported first-time publisher path: publisher-trigger digest work is now scheduled into separate package and skill mutations, but each owner-publisher digest helper can still call `.paginate()` multiple times in one internal mutation for owners with more than one page of packages or skills.

## What This Changes

Fix first-time publisher CLI publish and Web settings/profile failures caused by Convex multiple-pagination errors around `ensurePersonalPublisherForUser` and owner-publisher digest sync.

## Best Possible Solution

Refactor owner-publisher digest rebuilds into cursor-carrying scheduled pages so each internal mutation performs at most one indexed paginated read, schedules follow-up work until complete, and has regression coverage for multi-page package and skill owners.

## Repair Work Prompt

Fix the Convex owner-publisher digest rebuild pagination bug reported by first-time publisher/profile-save failures. In `convex/functions.ts`, change `syncPackageSearchDigestsForOwnerPublisherIdInternal` and `syncSkillSearchDigestsForOwnerPublisherIdInternal` so each internal mutation performs at most one indexed `.paginate()` call, accepts/carries a continuation cursor in validated args, processes one page, and self-schedules the next page with `ctx.scheduler.runAfter(0, ...)` until done. Preserve the existing split between package and skill digest jobs and keep queries on the existing `by_owner_publisher` indexes. Do not replace these paths with unbounded `.collect()`. Add regression coverage in `convex/functions.test.ts` for multi-page package and skill owners, including assertions that one invocation paginates once and schedules continuation. Also cover that `scheduleOwnerPublisherDigestSync` starts both first-page jobs. Avoid unrelated schema or product changes. Related refs: #1170, #1175, #1179, #1188, #1201, #49, #614, and closed unmerged PR #1183. Run `bun run test -- convex/functions.test.ts`, `bun run test -- convex/users.test.ts convex/httpApiV1.handlers.test.ts`, `bun run lint`, `bun run format:check`, and `bunx convex codegen`; include a short changelog/PR note explaining the pagination repair and why `.collect()` was not used.
