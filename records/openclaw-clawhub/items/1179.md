## Summary

Keep this issue open. Current main partially mitigates the originally reported first-time publisher failure by scheduling package and skill digest syncs as separate background mutations, but the owner-publisher digest helpers still loop over `.paginate()` inside a single internal mutation and can still hit Convex's single-pagination constraint for larger owner catalogs. The related open publish/profile reports and the closed, unmerged fix PR make this a concrete backend cleanup cluster.

## What This Changes

A first-time publisher reports that CLI publish and web settings/profile saves fail with Convex server errors through `ensurePersonalPublisherForUser` and owner-publisher search digest sync.

## Best Possible Solution

Update the owner-publisher digest rebuilds so each internal mutation performs one indexed paginated read, carries the continuation cursor through validated args, self-schedules follow-up work until complete, and adds regression coverage for multi-page package and skill owners before consolidating duplicate reports.
