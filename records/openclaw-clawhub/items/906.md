## Summary

Keep this issue open. ClawHub cleanup policy does not auto-close open issues, and current main still does not prove the reported production Chrome OAuth callback/session failure is fixed: the app has callback/error handling, but GitHub callback routing, cookie/session creation, active-user resolution, and related account-state failures remain unresolved across linked reports.

## What This Changes

The issue reports that GitHub OAuth authorization on clawhub.ai succeeds, but the callback returns the user to the upload/login flow without an active ClawHub session in Chrome.

## Best Possible Solution

Keep this issue open and triage it with the OAuth/session cluster. Maintainers should reproduce GitHub OAuth login on https://clawhub.ai/upload in a clean Chrome profile, capture HAR plus cookies/localStorage/sessionStorage before and after callback, inspect Convex Auth logs for PKCE verifier, callback exchange, session creation, and account-state failures, verify production OAuth/callback URL/env configuration, then land a focused code/config/dependency fix with coverage that exercises callback session creation rather than only mocked or preloaded auth state.
