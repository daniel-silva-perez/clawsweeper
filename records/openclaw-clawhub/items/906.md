## Summary

Keep this issue open. It reports a production Chrome GitHub OAuth callback/session failure, related OAuth/session reports are still open, and current main contains callback/error handling but does not prove that successful GitHub authorization establishes a production session. ClawHub cleanup policy also keeps issues open unless maintainers handle them manually.

## What This Changes

The issue requests that GitHub OAuth callback/login on clawhub.ai create an active session after successful authorization instead of returning the user to the upload/login flow still signed out.

## Best Possible Solution

Keep this issue open and triage it with #647 and #1552 as the GitHub OAuth callback/session cluster. Maintainers should reproduce GitHub OAuth login on https://clawhub.ai/upload in a clean Chrome profile, capture HAR plus cookies/localStorage/sessionStorage before and after callback, inspect Convex Auth logs for PKCE verifier, callback exchange, session creation, and account-state failures, verify production OAuth/callback URL/env configuration, then land a focused code/config/dependency fix with coverage that exercises callback session creation rather than only mocked or preloaded auth state.
