## Summary

Keep open. This is an open ClawHub issue, and the repository policy only permits automated closure for PRs that are certainly implemented on main. The report is also still actionable: it has reproduction steps, a confirming comment, and current main only shows partial OAuth callback/error handling, not an end-to-end fix for Chrome session persistence or the stuck sign-in state.

## Best Possible Solution

Keep this issue open and triage it with #647 as the likely generic OAuth callback/session tracker while keeping deleted-account cases like #1244 separate. Maintainers should reproduce production GitHub OAuth in Chrome with cookies/storage/HAR and Convex auth logs, then add an end-to-end or smoke path that verifies callback session creation and prevents the sign-in button from getting stuck.
