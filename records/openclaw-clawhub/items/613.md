The issue requests GitHub-style device-flow authentication for the ClawHub CLI so headless agents can authenticate without a localhost browser callback.

## Risks / Open Questions

- Device-flow auth is token issuance infrastructure and needs maintainer/security review for device-code entropy, expiry, polling interval/backoff, denial states, replay resistance, user binding, rate limiting, and auditability.
- The linked PR is described as client-side implementation with server-side endpoints and a verification page still required, so a client-only change would not fully solve the reported headless-auth problem.
- The current token-paste workaround still requires a browser-authenticated user to create or obtain the first token, which does not satisfy the requested autonomous/headless login flow.
