## Summary

Keep this issue open. The repository cleanup policy for openclaw/clawhub only allows auto-closing pull requests that are certainly implemented on main, and this is an open issue. The current main branch also still lacks the requested device-code authentication flow: login is either loopback browser auth or manual token paste.

## Best Possible Solution

Keep this issue open for maintainer prioritization. The likely implementation belongs in the first-party ClawHub CLI, web app, and Convex backend: create short-lived device authorization records, print a verification URL and user code, let an authenticated browser approve or deny the request, have the CLI poll with backoff/expiry/slow-down handling, mint and store a scoped ClawHub API token on success, and cover success, denial, expiry, polling, rate-limit, and docs behavior.
