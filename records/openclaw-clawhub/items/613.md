## Summary

Keep this issue open. Current main still does not implement a GitHub-style device-code or CLI/web polling authentication flow, and the ClawHub review policy only permits auto-closing PRs that are certainly implemented on main, not issues.

## Best Possible Solution

Keep this issue open for maintainer prioritization. A complete solution likely belongs in first-party ClawHub CLI and web/auth code: add a device-code or CLI/web pairing flow that prints a verification URL and user code, lets an authenticated browser approve it, polls from the CLI with backoff and expiry/denial handling, then stores a scoped ClawHub API token. Cover the CLI, web route, Convex token state, abuse-rate-limit behavior, docs, and tests for success, expiry, denial, and polling edge cases.
