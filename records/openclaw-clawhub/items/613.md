## Summary

Keep this issue open. Current main still lacks device-flow auth, and the issue is paired with open, unmerged PR #1867 using closing syntax. The ClawHub cleanup policy also says issues stay open; only PRs that are certainly implemented on main are candidates for auto-close.

## Best Possible Solution

Keep this issue open and review #1867 as the current implementation candidate. A complete solution likely belongs in core CLI plus ClawHub server: add `clawhub login --device` and `clawhub auth login --device`, device-code issuance and polling endpoints, a verification UI, compatible token storage, and tests for success, pending, slow_down/backoff, expiry, denial, and persistence.
