## Summary

Keep open. Current main still only supports token paste or loopback browser login, and the linked unmerged PR #1867 uses closing syntax for this issue, so the work should remain tracked until that PR or a replacement lands.

## What This Changes

Add GitHub-style device-flow authentication so headless agents can run `clawhub login` without a localhost browser OAuth callback.

## Best Possible Solution

Land PR #1867 or a replacement that implements the full flow across CLI, shared route constants, server-side device-code issuance, polling/token exchange, authenticated verification UI, secure token persistence, rate limiting, and tests for success, pending/backoff, expiry, denial, replay, and persistence behavior.
