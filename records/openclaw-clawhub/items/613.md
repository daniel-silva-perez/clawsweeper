## Summary

Keep open. Current main still only supports pasted API tokens or the browser loopback login, and linked PR #1867 is open, unmerged, and uses closing syntax for this issue, so the issue should stay open until that implementation or a replacement lands.

## What This Changes

Request device-code authentication for the ClawHub CLI so headless agents can log in without a local browser or localhost callback.

## Best Possible Solution

Keep this issue open and use linked #1867 as the current implementation candidate or replacement starting point. A complete fix should add CLI `--device` support, shared route constants, server-side device-code issuance and polling endpoints, an authenticated verification page, secure API-token creation/storage, rate limiting, and tests for success, pending, slow_down/backoff, expiry, denial, replay, and persistence.

- The current token-paste workaround still requires a browser-authenticated user to create or obtain the first token, which does not satisfy the requested headless agent flow.
