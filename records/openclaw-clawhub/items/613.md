## Summary

Keep open. Current main still lacks device-flow authentication, and the provided GitHub context shows open, unmerged PR #1867 uses closing syntax for this issue, so the issue should remain open until that implementation or a replacement lands. The ClawHub cleanup policy also says issues should stay open rather than be auto-closed.

## What This Changes

Add GitHub/RFC 8628-style device-flow authentication to the ClawHub CLI so headless agents can log in without a localhost browser callback.

## Best Possible Solution

Keep this issue open and use PR #1867 as the current implementation candidate or replacement starting point. A complete end state should add CLI `--device` support plus server-side device-code issuance, polling/token exchange endpoints, an authenticated verification page, secure API-token creation/storage, rate limiting, and coverage for success, pending, slow_down/backoff, expiry, denial, replay, and persistence behavior.
