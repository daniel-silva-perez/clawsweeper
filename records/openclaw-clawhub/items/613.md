## Summary

Keep open. Current main still lacks device-flow authentication, and the provided GitHub context shows linked PR #1867 is open, unmerged, and uses closing syntax for this issue, so this should remain open until that implementation or a replacement lands.

## What This Changes

Add GitHub-style device-flow authentication to `clawhub login` / `clawhub auth login` so headless agents can authenticate without a localhost browser redirect.

## Best Possible Solution

Keep this issue open and use #1867 as the current implementation candidate or replacement starting point. A complete solution should add CLI `--device` support, shared route constants, server-side device-code issuance and polling endpoints, an authenticated verification page, secure API-token creation/storage, rate limiting, and coverage for success, pending, slow_down/backoff, expiry, denial, replay, and persistence.
