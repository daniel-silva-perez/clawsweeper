## Summary

Keep this issue open. Current main still implements only browser loopback login plus manual token paste for headless use, no device-code CLI flag or server endpoints exist, and PR #1867 is open/unmerged as the implementation candidate that references this issue for closure. The ClawHub cleanup policy also says issues should stay open unless the item is a PR already implemented on main.

## Best Possible Solution

Keep this issue open and review PR #1867 as the current implementation candidate. A complete solution should add `clawhub login --device` / `clawhub auth login --device`, server-side device-code issuance and token polling endpoints, a verification UI, storage-compatible token handling, and tests for success, authorization_pending, slow_down/backoff, expiry, denial, and token persistence.
