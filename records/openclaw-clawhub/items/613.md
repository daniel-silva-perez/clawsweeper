## Summary

Keep this issue open. Current main still only supports loopback browser login or pre-created token paste for the CLI, while the requested first-time device-code/pairing flow for headless agents is not present. The repository policy for this ClawHub review path also says issues should stay open; only PRs that are definitely implemented on main are candidates for auto-close.

## Best Possible Solution

Keep this issue open for maintainer prioritization. The likely path is a first-party ClawHub CLI plus web/Convex implementation: generate short-lived device authorization records, print a verification URL and user code, let an authenticated browser approve or deny the request, have the CLI poll with backoff/expiry/slow-down handling, mint and store a scoped ClawHub API token on success, and document/test success, denial, expiry, polling, and abuse-control behavior.
