## Summary

Keep open. Current main still only supports token-paste or browser loopback CLI auth, and this issue is also paired with open, unmerged PR #1867. For ClawHub cleanup, issues should remain open unless a maintainer resolves them; here the requested device-flow auth is still missing and security-sensitive.

## What This Changes

The issue requests GitHub-style device flow authentication so headless ClawHub CLI agents can log in without a localhost browser callback.

## Best Possible Solution

Keep this issue open and use PR #1867 as the current implementation candidate or replacement starting point. A complete end state should add CLI `--device` support plus server-side device-code issuance, polling/token exchange endpoints, an authenticated verification page, secure API-token creation/storage, rate limiting, and coverage for success, pending, slow_down/backoff, expiry, denial, replay, and persistence behavior.
