## Summary

Keep open. Current main still does not implement a skill title/display-name length policy, and the ClawHub cleanup policy for this review pass keeps open issues for maintainer follow-up. The schema/API/CLI/web publish/import paths accept display names as plain strings and only reject empty names; the card UI visually clamps long titles but does not enforce a product limit.

## Best Possible Solution

Keep this issue open for maintainer product follow-up. Maintainers should define the public skill title/display-name policy first, including the maximum length and handling for existing long names, then implement it consistently across Convex validation, web publish, GitHub import, shared CLI/API schemas, docs, and listing/detail display behavior.
