## Summary

Keep this issue open. Current main still lacks a product-level maximum for skill titles/display names, and the ClawHub review policy only allows auto-closing PRs that are certainly implemented on main, not open issues.

## Best Possible Solution

Keep this issue open for maintainer product judgment and implementation. A conservative fix would add a shared display-name limit constant, frontend validation and input `maxLength`, backend enforcement in the publish and GitHub import paths, focused tests for upload/Convex publishing behavior, and public docs for the chosen limit.
