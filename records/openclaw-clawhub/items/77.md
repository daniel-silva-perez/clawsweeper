## Summary

Keep open. Current main still has no skill title/display-name length limit, and the ClawHub cleanup policy for this review pass keeps issues open unless maintainers handle them directly. The remaining work is real but needs a maintainer decision on the exact maximum length and how to treat already-published long names.

## What This Changes

Request a short, concise title/display-name limit for published skills.

## Best Possible Solution

Define the public skill display-name policy first, then enforce the chosen limit consistently in shared schema/publish validation, web publish, GitHub import, CLI/API payload handling, OpenAPI/docs, and tests, with explicit handling for existing long names.
