## Summary

Keep open. Current main still treats skill display names as unrestricted strings apart from trimming and non-empty checks, and the ClawHub cleanup policy says issues should remain open for maintainer handling. The report is valid, but choosing the maximum title length and legacy handling for existing long titles needs product judgment before implementation.

## What This Changes

Request that published skill titles/display names be kept short by adding a length limit.

## Best Possible Solution

Define the public skill display-name policy first, then enforce the chosen limit consistently in backend publish validation, web publish/import UI, GitHub import, CLI/API schemas, docs, and tests, with explicit treatment for already-published long names.
