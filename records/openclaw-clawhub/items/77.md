## Summary

Keep open. The requested title/display-name limit is still not implemented on current main, and the ClawHub cleanup policy for this pass keeps issues open unless they are PRs already implemented on main. The remaining work needs a maintainer product decision on the maximum length and how to handle existing long names before a focused implementation can land.

## What This Changes

Add or define a short maximum length policy for skill titles/display names.

## Best Possible Solution

Define the public skill title/display-name policy first, then enforce it consistently across Convex publish validation, web publish, GitHub import, CLI/API schemas, OpenAPI/docs, and tests, with explicit handling for already-published long names.
