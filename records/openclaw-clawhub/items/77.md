## Summary

Keep open. Current main still does not define or enforce a maximum length for skill titles/display names, and the ClawHub cleanup policy for this pass keeps issues open unless they are PRs already implemented on main. The request is valid but needs maintainer product judgment first because no concrete maximum length or existing-skill handling policy is specified.

## What This Changes

The issue requests a maximum/shortness policy for skill titles so long display names do not appear in ClawHub listings.

## Best Possible Solution

Keep this issue open for maintainer product follow-up. Maintainers should define the public skill title/display-name policy first, including the maximum length and handling for existing long names, then implement it consistently across Convex publish validation, web publish, GitHub import, CLI/API schemas, docs, and listing/detail display behavior.
