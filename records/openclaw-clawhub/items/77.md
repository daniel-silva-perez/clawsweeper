## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing pull requests that are certainly implemented on main. Current main does not implement a concrete short-title policy: skill display names are accepted as plain strings, web/import/publish paths only require a non-empty value, and listing cards rely on two-line display clamping rather than a stored title limit.

## Best Possible Solution

Keep this issue open for maintainer product follow-up. Maintainers should decide the concrete public title policy first, including the maximum length and how to handle existing long display names. Then implement that policy consistently across Convex schema/argument validation, publish and GitHub import forms, shared package/CLI schemas, docs, and listing/detail display behavior.
