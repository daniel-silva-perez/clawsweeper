## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy for this pass keeps issues open unless they are PRs already implemented on main. Current main also still lacks a maximum length or public policy for skill titles/display names across storage, backend publishing, web publishing, GitHub import, and CLI/API validation.

## What This Changes

Request a maximum-length policy and enforcement so skill titles/display names stay short and concise.

## Best Possible Solution

Keep this issue open for maintainer product follow-up. Maintainers should define the public skill title/display-name policy first, including the maximum length and handling for existing long names, then implement it consistently across Convex publish validation, web publish, GitHub import, CLI/API schemas, docs, and listing/detail display behavior.
