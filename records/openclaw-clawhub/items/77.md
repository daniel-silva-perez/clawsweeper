## Summary

Keep this issue open. The ClawHub cleanup policy only permits auto-closing pull requests that are certainly implemented on main, and this is an open contributor issue. The underlying request also still appears valid: current main has no shared skill title/display-name length limit in the publish UI, GitHub import UI, backend publish path, schema/API payload validation, CLI publish path, docs, or focused tests.

## Best Possible Solution

Keep this issue open for maintainer product judgment and implementation. A conservative fix would add a shared skill display-name limit constant and validator, apply it in frontend publish/import flows and backend/API/CLI publish validation, add focused tests, document the chosen limit, and decide how existing long display names should be handled.
