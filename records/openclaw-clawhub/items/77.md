## Summary

Keep open. Current main still does not enforce a skill title/display-name length policy, and the ClawHub cleanup policy for this pass keeps issues open for maintainer follow-up. The schema, backend publish helper, web publish UI, GitHub import path, and CLI publish path all accept display names as plain strings and only reject empty values.

## Best Possible Solution

Keep this issue open for maintainer product follow-up. Maintainers should define the public skill title/display-name policy first, including maximum length and handling for existing long names, then implement it consistently across Convex publish validation, web publish, GitHub import, CLI/API schemas, docs, and listing/detail display behavior.
