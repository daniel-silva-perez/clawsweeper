## Summary

Keep this issue open. Under the ClawHub cleanup policy, open issues are not auto-close candidates, and current main still does not implement a skill title/display-name length limit in the frontend, backend publish paths, schema, CLI/API payload schema, or docs.

## Best Possible Solution

Keep this issue open for maintainer product judgment and implementation. A conservative fix would add a shared skill display-name limit constant and validator, apply it in the publish form, GitHub import flow, backend publish/import/API paths, and CLI request schema, add focused tests, document the chosen limit, and decide how to handle existing long display names.
