## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits automated closure for pull requests that are certainly implemented on main. Current main also still does not implement a concrete title/display-name limit: skill names are plain strings across schema/API/shared schemas, publish/import paths only require a non-empty value, and the card UI only clamps display to two lines.

## Best Possible Solution

Keep this issue open for maintainer product follow-up. Maintainers should decide the public skill-title/display-name policy first, including maximum length and handling for existing long names, then implement it consistently across Convex schema/action validation, publish and GitHub import flows, shared CLI/API schemas, docs, and card/detail display behavior.
