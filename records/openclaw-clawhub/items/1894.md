## Summary

Keep this issue open. The ClawHub cleanup policy for this pass keeps issues open, and current main makes the reported ordering plausible rather than resolved: standalone slug-token matches receive stronger lexical boosts than display-name-only matches, while downloads are only a small log-scaled popularity prior. Existing docs mention the broad ranking ingredients and rename aliases, but they do not answer the reporter's practical discoverability and rename side-effect questions.

## What This Changes

The issue asks for clarification or documentation about why high-download map-related skills rank below lower-download standalone slug-token matches, and whether renaming slugs affects discoverability, redirects, installs, stats, or indexing.

## Best Possible Solution

A maintainer should answer the reporter directly, decide whether the current map versus amap ranking is intended, and then either document discoverability guidance covering slug-token effects, downloads, moderation filters, and rename aliases, or adjust ranking weights and tests if the current ordering is not desired.
