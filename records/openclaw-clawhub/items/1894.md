## Summary

Keep this issue open. Current main makes the reported ordering plausible rather than resolved: standalone slug-token matches get stronger lexical boosts than display-name-only matches, while downloads are only a small log-scaled prior. The docs mention broad ranking ingredients and rename aliases, but they do not yet answer the publisher-facing discoverability and slug-rename side-effect questions, and the ClawHub cleanup policy keeps issues open in this pass.

## What This Changes

The issue asks ClawHub to explain why high-download AMap skills rank low for the query "map", whether standalone slug tokens materially affect ranking, and what side effects slug renames have.

## Best Possible Solution

A maintainer should answer the publisher directly, decide whether the current map-versus-amap ordering is intended, then either document discoverability guidance for slug tokens, downloads, moderation filters, and rename aliases, or adjust the ranking weights and tests if the current behavior is not desired.
