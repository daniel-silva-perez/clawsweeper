## Summary

Keep this issue open. The ClawHub cleanup policy for this pass keeps issues open, and current main supports the reporter's observation: standalone slug-token matches get stronger lexical boosts than display-name-only matches, while downloads are only a small log-scaled popularity prior. The existing docs describe the broad ranking ingredients and rename aliases, but they do not yet answer the practical discoverability and slug-rename side-effect questions well enough for publishers.

## What This Changes

The issue asks why high-download map-related skills rank below lower-download standalone slug-token matches, whether slug tokens materially affect search visibility, and what side effects slug renames have for aliases, installs, stats, and search indexing.

## Best Possible Solution

A maintainer should answer the publisher directly, decide whether the current slug-token versus display-name weighting is intended, then either document discoverability guidance covering slug tokens, downloads, moderation filters, and rename aliases, or open a focused ranking change with tests if the current ordering is not desired.
