## Summary

Keep this open. The ClawHub cleanup policy keeps issues open in this pass, and current main makes the reported ranking behavior plausible: standalone slug-token matches receive a stronger lexical boost than display-name-only matches, while downloads are a small log-scaled prior. The docs mention the broad ranking ingredients and slug aliases, but they do not fully answer the discoverability and rename side-effect questions.

## What This Changes

The issue asks why a high-download skill ranks low for the query `map`, whether standalone slug tokens affect ranking, and what side effects slug renames have.

## Best Possible Solution

Keep this open for maintainer/product follow-up. The best path is to answer the reporter directly and decide whether the current `map`/`amap` ordering is intended; then either document discoverability guidance covering slug token effects, download popularity, moderation filters, and rename aliases, or adjust ranking weights/tests for this scenario if the current ordering is not desired.
