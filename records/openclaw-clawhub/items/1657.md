## Summary

Keep this PR open. Current main does not implement the dependency-registry phantom-package scanner, and this is a security-sensitive moderation change that should be reviewed by maintainers rather than auto-closed.

## What This Changes

The PR adds a Convex publish-time dependency registry scanner for bundled skill dependency files, stores and caches registry results, injects phantom-package moderation findings, schedules scans after publish, and adjusts static finding UI rendering for line-zero findings.

## Best Possible Solution

Keep this PR open for maintainer/security review. The best outcome is to either land a corrected scanner or replace it with a successor that is integrated into publish/moderation, has tests for dependency-file parser edge cases and registry error handling, coordinates with openclaw/trust#29, and has an acceptable retry/backfill and UI/API reporting story.
