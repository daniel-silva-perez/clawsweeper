## Summary

Keep open. Current main does not implement phantom dependency detection for bundled skill dependency files, and the repository policy only supports auto-closing ClawHub PRs when the intended change is definitely implemented on main. This PR also remains security-sensitive because it would add external registry lookups and auto-inject critical moderation findings, with review history already identifying correctness risks that need maintainer review.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a corrected dependency-registry scanner or replace it with a new PR that implements the same supply-chain mitigation in the Convex publish/moderation pipeline, with schema/cache support and focused tests for scoped npm packages, transient registry failures, npm aliases and URL/workspace specs, optional dependencies, pyproject formats, vendored dependency files, and indexed retry/backfill behavior. Coordinate the final product/security decision with the linked openclaw/trust report.
