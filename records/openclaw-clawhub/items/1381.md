## Summary

Keep this issue open. Current main supports deleting published skills through owner CLI/API soft-delete and admin-only hard delete, but this is a ClawHub issue, and the repository policy for this review only allows auto-closing implemented PRs. A maintainer should answer the user directly or improve dashboard/docs discoverability.

## Best Possible Solution

A maintainer should leave the issue open long enough to answer with the supported workflow: use `clawhub delete <slug> --yes` to soft-delete a published skill, `clawhub undelete <slug> --yes` to restore it, or the equivalent v1 API endpoints. If self-service web deletion is intended, add delete/restore actions to the publisher dashboard or skill management UI and document the distinction between owner soft-delete and admin-only hard delete.
