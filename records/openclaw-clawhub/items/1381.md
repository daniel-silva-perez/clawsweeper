## Summary

Current main supports deleting published skills via owner/moderator/admin soft-delete and restore through the CLI and v1 API, with hard delete reserved for admins. This should still stay open because the repository-specific ClawHub policy for this review only allows auto-closing PRs that are implemented on main; issues need maintainer follow-up.

## Best Possible Solution

Keep this issue open for maintainer follow-up. A maintainer should answer with the supported workflow: `clawhub delete <slug> --yes` soft-deletes a published skill, `clawhub undelete <slug> --yes` restores it, and the v1 API endpoints provide the same behavior. If self-service web deletion is intended, add owner delete/restore controls to the dashboard or skill owner tools and document the distinction between owner soft-delete and admin-only hard delete.
