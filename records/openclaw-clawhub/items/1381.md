## Summary

Keep this issue open. Current main already supports owner deletion of published skills through CLI/API soft-delete and restore, with admin-only hard delete, but the ClawHub review policy only permits auto-closing PRs that are implemented on main. This is an issue asking a user-facing how-to question, so a maintainer should answer it or improve dashboard/docs discoverability.

## Best Possible Solution

A maintainer should keep this issue open long enough to answer with the supported workflow: `clawhub delete <slug> --yes` soft-deletes a published skill, `clawhub undelete <slug> --yes` restores it, and the equivalent v1 API endpoints are available. If web self-service is intended, add owner delete/restore controls to the publisher dashboard or skill owner tools and document the distinction between owner soft-delete and admin-only hard delete.
