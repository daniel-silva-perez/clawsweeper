## Summary

Keep this open. The current main branch already supports deleting published skills through a documented CLI/API soft-delete workflow, with admin-only hard delete in management surfaces, but the repository policy for openclaw/clawhub says issues should stay open unless they are implemented PRs. This issue is also a user support/discoverability question that a maintainer can answer directly.

## Best Possible Solution

A maintainer should reply with the supported workflow: owners can soft-delete a published skill with `clawhub delete <slug>` or `DELETE /api/v1/skills/{slug}`, restore with `clawhub undelete <slug>` or `POST /api/v1/skills/{slug}/undelete`, and request maintainer/admin help for permanent hard deletion. If the product should support this in the web UI, add an owner-facing delete/restore action on the dashboard or skill page and document the soft-delete vs hard-delete distinction more prominently.
