## Summary

Keep this issue open. Current main has a documented soft-delete workflow through the ClawHub CLI/API and admin-only hard delete, but the repository policy for this review requires keeping ClawHub issues open unless they are PRs already implemented on main. A maintainer should answer the user directly or improve discoverability if the intended workflow is CLI/API-only.

## Best Possible Solution

A maintainer should answer this issue with the supported workflow: owners can soft-delete a published skill with `clawhub delete <slug>` or `DELETE /api/v1/skills/{slug}`, restore with `clawhub undelete <slug>`, and request/admin-only hard deletion if permanent removal is needed. If self-service deletion should be available in the web UI, add an owner-facing action on the dashboard or skill page and document the distinction between soft delete and admin hard delete.
