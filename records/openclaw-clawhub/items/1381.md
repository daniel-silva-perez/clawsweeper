Keep this issue open under the repository-specific ClawHub cleanup policy. Current main already supports deleting published skills through documented CLI and v1 API soft-delete/restore paths, but this item is an issue, and the provided policy says ClawHub issues should stay open for maintainer follow-up rather than be auto-closed.

## Best Possible Solution

Keep the issue open for maintainer follow-up. A maintainer should answer with the supported workflow: `clawhub delete <slug> --yes` soft-deletes a published skill, `clawhub undelete <slug> --yes` restores it, and the v1 API endpoints provide the same behavior.
