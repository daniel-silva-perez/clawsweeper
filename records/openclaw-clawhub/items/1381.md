## Summary

Keep this issue open. Current main supports deleting published skills through a documented owner soft-delete workflow in the ClawHub CLI/API, with admin-only hard delete, but the repository policy for this review says ClawHub issues should remain open unless they are implemented PRs. A maintainer should answer the user directly or improve web UI/docs discoverability.

## Best Possible Solution

A maintainer should answer with the supported workflow: owners can soft-delete a published skill with `clawhub delete <slug>` or `DELETE /api/v1/skills/{slug}`, restore with `clawhub undelete <slug>` or `POST /api/v1/skills/{slug}/undelete`, and request/admin-only hard deletion if permanent removal is required. If web self-service is intended, add a delete/restore action to the owner dashboard or skill management surface and document the distinction between soft delete and hard delete.
