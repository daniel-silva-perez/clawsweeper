## Summary

Keep this issue open. Current main still documents and implements lowercase/plural skill readme compatibility across docs, CLI discovery/publish, browser upload, GitHub import, server publish validation, and readme resolution, so uppercase-only enforcement is not implemented. For this repository, cleanup policy also says issues stay open unless maintainers resolve them; the remaining work is a breaking format-policy and migration decision, not an auto-close candidate.

## What This Changes

The issue requests enforcing canonical uppercase `SKILL.md` and `PROTOCOL.md` package filenames so duplicate case variants cannot be accepted or checked out incorrectly on case-insensitive filesystems.

## Best Possible Solution

Maintainers should define the canonical casing policy for skill package files, decide whether `PROTOCOL.md` belongs in the package contract, then update docs and validation consistently across CLI scan/sync/publish, browser upload, GitHub import, server publish, stored readme resolution, and any content cleanup plan.
