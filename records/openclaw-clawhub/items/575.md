## Summary

Keep this issue open. Current main still documents and implements lowercase and plural skill readme compatibility across docs, CLI discovery/publish, browser upload, GitHub import, server publish validation, and stored readme resolution, so uppercase-only enforcement is not implemented. The repository cleanup policy also says ClawHub issues should remain open unless maintainers resolve them.

## What This Changes

Request to require exact uppercase `SKILL.md` and `PROTOCOL.md` filenames and reject lowercase or duplicate case variants to avoid case-insensitive filesystem conflicts.

## Best Possible Solution

Maintainers should decide the canonical casing policy for skill package files, define whether `PROTOCOL.md` belongs in the package contract, then update docs and validation consistently across CLI scan/sync/publish, browser upload, GitHub import, server publish, stored readme resolution, and any external content cleanup plan.
