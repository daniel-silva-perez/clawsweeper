## Summary

Keep this issue open. The repository policy for ClawHub issues does not permit auto-closing issues, and current main still documents and implements compatibility with lowercase/plural skill marker filenames rather than enforcing uppercase-only SKILL.md or any PROTOCOL.md filename policy.

## Best Possible Solution

Keep this issue open for maintainer follow-up. A complete fix should define the canonical filename policy, add case-insensitive duplicate detection and uppercase validation across CLI scan/sync, CLI publish, browser upload, GitHub import, server/API publish validation, docs, and tests, and decide how to migrate or grandfather existing lowercase/plural marker files and any PROTOCOL.md convention.
