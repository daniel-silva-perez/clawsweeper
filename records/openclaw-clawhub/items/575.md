## Summary

Keep this issue open. ClawHub policy keeps issues open for maintainer judgment, and current main still intentionally accepts lowercase/plural skill marker variants in docs, CLI scan/publish, browser upload, GitHub import, and server-side publish validation. Exact uppercase-only enforcement and case-insensitive duplicate rejection remain unimplemented, and the PROTOCOL.md part needs a policy decision because current main does not define a protocol-file validation surface.

## Best Possible Solution

Keep this issue open for maintainer product follow-up. A complete solution should decide whether ClawHub is moving to uppercase-only marker filenames, define any PROTOCOL.md convention, add exact-case and case-insensitive duplicate detection across CLI scan/sync, CLI publish, browser upload, GitHub import, and server/API publish validation, update docs/tests, and define a migration or grandfathering path for existing lowercase/plural marker files and existing openclaw/skills content.
