## Summary

Keep this issue open. Current main still documents and implements lowercase and plural skill readme compatibility across docs, CLI discovery/publish, browser upload, GitHub import, server-side publish validation, and stored readme resolution. The requested uppercase-only rule is a breaking package-format policy and migration decision, and this repository's cleanup policy keeps ClawHub issues open unless maintainers resolve them.

## What This Changes

Request to enforce canonical uppercase `SKILL.md` and `PROTOCOL.md` file names and prevent mixed-case duplicates that break case-insensitive checkouts.

## Best Possible Solution

Maintainers should define the canonical casing policy for skill package files, decide whether `PROTOCOL.md` belongs in the package contract, then update docs and validation consistently across CLI scan/sync/publish, browser upload, GitHub import, server publish, stored readme resolution, and any content cleanup plan.
