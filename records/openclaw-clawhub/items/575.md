## Summary

Keep this issue open. ClawHub cleanup policy keeps issues open, and current main still documents and implements lowercase `skill.md` plus `skills.md` compatibility across scan, publish, browser upload, GitHub import, and readme-resolution paths; `PROTOCOL.md` has no current skill-package validation surface to enforce.

## What This Changes

Request to enforce exact uppercase `SKILL.md` and `PROTOCOL.md` names and reject lowercase or case-insensitive duplicate marker files that break Windows checkouts.

## Best Possible Solution

Keep this open for maintainer product follow-up. Decide whether ClawHub should intentionally break lowercase marker compatibility, define whether `PROTOCOL.md` is part of the skill package contract, then implement exact-case and case-insensitive duplicate detection across CLI scan/sync, CLI publish, browser upload, GitHub import, server/API publish validation, package readme resolution, docs, and tests, with a migration plan for existing lowercase or duplicate content.
