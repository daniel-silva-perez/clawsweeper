## Summary

Keep open. This is a ClawHub issue, and the cleanup policy only permits auto-closing ClawHub PRs when their intended change is already implemented on main. Independently, current main still documents and accepts lowercase/plural skill markers, and there is no visible PROTOCOL.md filename validation path.

## Best Possible Solution

Keep this issue open for maintainer follow-up. A complete fix should define the canonical filename policy, add case-insensitive duplicate detection and uppercase validation across CLI scan/sync, CLI publish, browser upload, GitHub import, server/API publish validation, docs, and tests, and decide how to migrate or grandfather existing lowercase/plural marker files.
