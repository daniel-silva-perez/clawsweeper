## Summary

Keep open. This is a ClawHub issue, and the repository policy only allows auto-closing pull requests that are certainly implemented on main. Current main also does not implement the requested uppercase-only enforcement: the docs, CLI scan/publish paths, browser upload flow, GitHub import, and server publish/readback logic still accept lowercase `skill.md` and/or `skills.md`; I did not find a `PROTOCOL.md` filename validation surface.

## Best Possible Solution

Keep this issue open for maintainer decision and implementation. A complete fix would add case-insensitive duplicate detection plus canonical uppercase validation across CLI scanning/sync, CLI publish, browser upload, GitHub import, API/server publish validation, readback behavior, docs, and any migration or cleanup path for already-published content; it should also explicitly decide whether legacy `skill.md` and `skills.md` compatibility remains supported.
