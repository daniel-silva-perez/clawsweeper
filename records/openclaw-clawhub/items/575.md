## Summary

Keep open. This is a ClawHub issue, and the repository policy for this cleanup pass says ClawHub issues should remain open. The requested uppercase-only enforcement is also still not implemented on current main: docs and multiple publish/import/read paths continue to accept lowercase or plural skill marker variants, and I found no current PROTOCOL.md filename validation surface.

## Best Possible Solution

Keep this issue open for maintainer decision and implementation. A complete fix would define the canonical filename policy, add case-insensitive duplicate detection and uppercase validation across CLI scan/sync, CLI publish, browser upload, GitHub import, server/API publish validation, readback behavior, docs, and tests, and include a migration or compatibility plan for already-published lowercase or plural marker files.
