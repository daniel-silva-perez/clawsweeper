## Summary

Keep this issue open. Current main still explicitly permits lowercase and plural skill readme variants in docs, CLI scan/publish, browser upload, GitHub import, and server-side publish validation; the requested uppercase-only policy is not implemented and would be a breaking compatibility decision. ClawHub cleanup policy for this repository also says issues should remain open unless maintainers resolve them.

## What This Changes

The issue requests enforcing uppercase `SKILL.md` and `PROTOCOL.md` names and rejecting conflicting uppercase/lowercase variants that break case-insensitive filesystems.

## Best Possible Solution

Maintainers should define the canonical casing policy for skill package files, decide whether `PROTOCOL.md` is part of the package contract, then update docs and validation consistently across CLI scan/sync/publish, browser upload, GitHub import, server publish, stored readme resolution, and any content cleanup plan.
