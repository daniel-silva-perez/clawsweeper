## Summary

Keep open. This is an issue in openclaw/clawhub, and the repository policy only allows auto-closing PRs that are certainly implemented on main. Current main also does not implement the requested uppercase enforcement: docs and multiple publish/import/read paths still explicitly accept lowercase `skill.md` and/or `skills.md`, and no `PROTOCOL.md` filename validation path was found.

## Best Possible Solution

Keep this issue open for maintainer decision and implementation. A complete fix would add case-insensitive duplicate detection and canonical uppercase validation across CLI scanning/sync, CLI publish, browser upload, GitHub import, API/server publish validation, readback behavior, docs, and any cleanup/backfill path for already-published content; it should also explicitly decide whether legacy `skill.md`/`skills.md` compatibility remains supported.
