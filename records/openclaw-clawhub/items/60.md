## Summary

Keep open. Current `main` does not contain the reported legacy `molt.bot` installer or its `cleanup_npm_clawdbot_paths` function, while the maintained ClawHub CLI install path is a separate package/API flow; combined with the repository policy to keep ClawHub issues open, this needs maintainer support-boundary triage rather than cleanup closure.

## What This Changes

The issue reports an npm install failure when running the legacy `curl -fsSL https://molt.bot/install.sh | zsh` installer, where zsh errors on an unmatched Homebrew Node 25.5.0 `.clawdbot-*` cleanup glob.

## Best Possible Solution

Resolve the installer support boundary explicitly: if the legacy hosted installer remains supported, make its source traceable and harden the zsh cleanup glob for unmatched Homebrew npm paths and supported shells; if it is retired, update or redirect user-facing install guidance toward the maintained OpenClaw/ClawHub commands and state the supported Node range.
