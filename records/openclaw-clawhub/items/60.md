## Summary

Keep open. ClawHub cleanup policy says issues should stay open, and the evidence points to a support-boundary problem rather than a source-level fix in current `main`: the reported legacy `molt.bot` zsh cleanup function is absent from this repo, current ClawHub install paths are separate, and the live legacy URL now serves an OpenClaw bash installer outside the checked-out source tree.

## What This Changes

The issue reports a curl-piped legacy installer failing under zsh during npm cleanup because an unmatched Homebrew Node 25 glob causes `no matches found`.

## Best Possible Solution

Resolve the support boundary explicitly: if the legacy `molt.bot` installer remains supported, track or expose its owning source and harden cleanup globs for the documented shells and supported Node versions; if it is retired, steer users to the maintained OpenClaw/ClawHub install commands and document that the old zsh-piped path is unsupported.
