## Summary

Keep open. The report targets a legacy hosted `molt.bot` curl-piped installer that is not present in current `main`; the maintained ClawHub CLI install path is a separate registry/API package flow, and ClawHub cleanup policy keeps issues open unless a PR is certainly implemented on main.

## What This Changes

The issue reports npm installation failing during `curl -fsSL https://molt.bot/install.sh | zsh` because an unmatched Homebrew npm cleanup glob aborts in zsh.

## Best Possible Solution

Resolve the installer support boundary explicitly: if the legacy hosted installer remains supported, make its source traceable and harden the zsh cleanup glob for unmatched Homebrew npm paths and supported shells; if it is retired, redirect or update user-facing install guidance to the maintained OpenClaw/ClawHub commands and state the supported Node range.
