## Summary

Keep open. The report is concrete, but the failing `molt.bot` zsh cleanup function is not present in current ClawHub source, and current main points users at the maintained `clawhub` CLI install path. The live legacy URL now serves a separate OpenClaw bash installer, so this needs maintainer triage on whether that hosted legacy installer URL is still supported from this repository or should be redirected/documented elsewhere.

## Best Possible Solution

Keep this issue open for maintainer triage. Decide whether `https://molt.bot/install.sh` remains a supported ClawHub/OpenClaw user path. If yes, identify and fix or verify the owning hosted installer source for zsh/bash compatibility and current Node support; if no, steer users to the supported ClawHub/OpenClaw install commands and close manually with that support-boundary explanation.
