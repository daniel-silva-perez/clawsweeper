## Summary

Keep open. The report is concrete and current main has a maintained `clawhub` CLI install path, but the failing `molt.bot` zsh cleanup function is not in this repository and the live legacy URL now serves a different OpenClaw bash installer, so this needs maintainer support-boundary triage rather than an autonomous source cleanup.

## What This Changes

The issue reports that `curl -fsSL https://molt.bot/install.sh | zsh` fails during npm install with a zsh no-match error in `cleanup_npm_clawdbot_paths` for a Homebrew Node 25.5.0 global npm path.

## Best Possible Solution

Keep this issue open for maintainer triage. Maintainers should decide whether the legacy `molt.bot` installer remains supported; if it does, find and fix or verify the owning hosted installer source for zsh/bash cleanup behavior and supported Node versions, and if it does not, steer users to the supported `clawhub`/`openclaw skills install` paths and close manually with that support-boundary explanation.
