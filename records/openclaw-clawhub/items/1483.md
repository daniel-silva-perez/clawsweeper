## Summary

Keep open. Current main still does not expose skill-level moderation/security in `clawhub inspect` plain or JSON output, even though the CLI schema and backend API already carry that data; the issue is also paired with open draft PR #1855 using closing syntax, so it should remain open until that PR is reviewed and resolved.

## What This Changes

Request to expose a skill's security/moderation status from the ClawHub CLI, especially through inspect output so users and scripts can prefer benign skills before installation.

## Best Possible Solution

Review, land, or close #1855 as the focused inspect implementation with plain and JSON skill-level moderation output plus regression tests; after that, split any remaining benign-only filtering or package inspect scan output into narrower CLI follow-ups.
