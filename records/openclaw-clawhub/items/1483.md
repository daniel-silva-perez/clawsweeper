Keep open. Current main still does not expose skill-level moderation/security in `clawhub inspect` text or JSON output, and the issue is already paired with open draft PR #1855 using closing syntax. The repository cleanup policy also keeps ClawHub issues open rather than auto-closing them.

## What This Changes

The issue requests CLI visibility into a skill's security/moderation status so users and scripts can identify benign skills before install.

## Best Possible Solution

Land or revise a focused CLI change that preserves skill moderation in `clawhub inspect --json` and prints a concise moderation/security summary in normal inspect output, then split any benign-only install or search filtering into a narrower follow-up if needed.
