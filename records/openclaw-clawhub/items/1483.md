## Summary

Keep this issue open. Current main partially addresses the request through version-level security output and install-time malware/suspicious checks, but it does not clearly provide the requested plain CLI security visibility or benign-only install/filter workflow. The ClawHub review policy also says issues should remain open for maintainer triage.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely fix belongs in the ClawHub CLI: consistently show skill-level moderation/security in plain `clawhub inspect` text and JSON, expose package scan status/details in `clawhub package inspect`, and add or document a benign/non-suspicious-only workflow for search/explore/install selection.
