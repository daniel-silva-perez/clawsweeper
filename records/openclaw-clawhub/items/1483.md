## Summary

Keep this issue open. Current main still does not fully satisfy the request: the skill detail API and CLI schema know about skill-level moderation, and install/update have partial safety checks, but `clawhub inspect` still omits the top-level moderation block from JSON and plain inspect only prints version-level security when a version response is fetched. PR #1855 is an open draft implementation candidate that explicitly references this issue, so the best next step is to review that PR rather than close this tracker.

## Best Possible Solution

Keep this issue open for maintainer triage. Review and either land or close PR #1855 as the focused implementation for `clawhub inspect` text and JSON moderation visibility. If it lands, decide whether benign-only install/search filtering should be split into a narrower follow-up that wires existing moderation data into CLI selection flows.
