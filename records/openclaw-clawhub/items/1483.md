## Summary

Keep this issue open. Repository policy for openclaw/clawhub says issues are not auto-closed, and current main still does not fully satisfy the request: the skill detail API schema accepts skill-level moderation data, install/update partially guard against suspicious or malware-blocked skills, but `clawhub inspect` still omits the top-level `moderation` block from JSON output and plain inspect only prints version-level `security` when a version response is fetched. An open draft implementation candidate, PR #1855, explicitly references this issue with closing syntax, so this issue should stay open until that PR is reviewed, merged, or closed.

## Best Possible Solution

Keep this issue open for maintainer triage. Review and either land or close PR #1855 as the implementation candidate for `clawhub inspect` text and JSON moderation visibility. If it lands, decide whether benign-only install/search filtering should be included in the same resolution or split into a focused follow-up that wires existing non-suspicious API support into CLI search/explore or install selection flows.
