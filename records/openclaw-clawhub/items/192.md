## Summary

Keep open. ClawHub cleanup policy only permits auto-closing pull requests that are certainly implemented on current main, and this is an issue. Substantively, current main still lacks a separate verified-publisher identity signal for skills, public owner/API exposure, skill card/list/detail UI, and publisher-aware security presentation; open PR #1502 remains the direct implementation candidate to review, land, or supersede.

## What This Changes

Add verified publisher identity and badges so skill UI and security warnings can distinguish legitimate verified publishers from unknown publishers.

## Best Possible Solution

Keep this issue open and have maintainers review, land, or supersede PR #1502. The implementation should model verifiedPublisher separately from trustedPublisher or use an equivalent verification record, expose the signal through public owner/API shapes, render it on skill cards, browse/profile lists, and skill detail headers, and define how verified identity affects security presentation without weakening suspicious or malicious findings.
