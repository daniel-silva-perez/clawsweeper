## Summary

Keep this PR open. Current main only partially overlaps with the proposed stats semantics cleanup: SkillHeader uses a Download icon for downloads, but it still renders stars first, keeps current/all-time installs as separate History rows, and does not include the proposed installs tooltip utility or tests. Shared skill, soul, and list stat surfaces still use Package for downloads and older ordering/style. The related draft PR #1854 is narrower download-icon work and does not supersede this broader PR.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with a clean scoped update against current main. The best implementation would consistently apply semantic download/user icons and downloads-first ordering across SkillHeader, SkillStats, SoulStats, SkillListItem, and related public list surfaces; keep both current and all-time install information visible or accessibly exposed; add focused regression tests for install edge cases; coordinate the narrower download-icon work with #1854; and avoid unrelated backend, moderation, security, or formatting churn.
