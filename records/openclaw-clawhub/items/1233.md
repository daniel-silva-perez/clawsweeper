## Summary

Keep this PR open. Current main still does not implement the PR’s intended skill/soul stat semantics cleanup: the public stat UI still uses package icons for downloads, star-first ordering remains in the shared stat components, the skill header still shows separate visible current/all-time install stats rather than the proposed centralized tooltip/accessibility pattern, and the proposed installs tooltip utility/tests are absent. Related PR #1854 overlaps only with download-icon cleanup and is still open, so it does not make this broader PR implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review or a scoped replacement. The best path is to rebase/update it against current main, apply semantic download/user icons and downloads-first ordering consistently across SkillHeader, SkillStats, SoulStats, and any shared public skill list surfaces, preserve both current and all-time install information in visible or accessible DOM, add focused regression tests for install display edge cases, and avoid unrelated backend or formatting churn. Coordinate the narrower download-icon work with #1854 rather than closing this broader PR as implemented.
