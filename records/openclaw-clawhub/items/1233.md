## Summary

Keep this PR open. Current main only partially overlaps with the proposed change: the skill detail header has a Download icon now, but it still orders stars before downloads, keeps current/all-time installs as separate History stats, and does not include the proposed installs tooltip utility or tests. Shared skill, soul, and public list stats still use Package for downloads and keep the old star/download ordering. The related draft PR #1854 is narrower and does not supersede the broader stats semantics cleanup.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with a clean scoped update against current main. The best path is to consistently apply semantic download/user icons and downloads-first ordering across SkillHeader, SkillStats, SoulStats, SkillListItem, and related public list surfaces; keep both current and all-time install information visible or accessibly exposed; add focused regression tests for install edge cases; coordinate the narrower download-icon work with #1854; and avoid unrelated backend or formatting churn.
