## Summary

Keep this PR open. Current main still does not implement the PR's intended skill/soul stats semantics cleanup: the skill header and shared stat components still use Package icons for downloads, keep stars before downloads in triplet displays, show current/all-time installs as separate visible stats rather than a centralized tooltip/accessibility pattern, and do not contain the proposed installs tooltip utility or tests. Related PR #1854 is open and narrower, so it does not make this broader PR obsolete or implemented.

## Best Possible Solution

Keep this PR open for maintainer review or a scoped replacement. The best path is to rebase/update it against current main, apply semantic download/user icons and downloads-first ordering consistently across SkillHeader, SkillStats, SoulStats, and public skill list surfaces, preserve both current and all-time install information in visible or accessible DOM, add focused regression tests for install display edge cases, and avoid unrelated backend or formatting churn. Coordinate the narrower download-icon work with #1854 rather than closing this broader PR as implemented.
