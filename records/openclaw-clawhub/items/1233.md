## Summary

Keep this PR open. Current main only partially overlaps the proposed stats-semantics cleanup: the skill header now uses a Download icon for downloads, but the shared skill/soul stats rows and skill list still use Package for downloads, retain stars-first ordering, and the proposed installs tooltip utility and edge-case tests are absent. The PR also overlaps with draft PR #1854 and has an unresolved accessibility concern around hiding current installs behind a native hover tooltip, so this needs maintainer review rather than auto-close.

## What This Changes

The PR branch updates skill and soul stat UI semantics, adding clearer download/user icon usage, downloads-first ordering, an installs tooltip utility, and focused regression tests for install-count edge cases.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with a clean scoped update against current main. The best implementation would consistently apply semantic download/user icons and downloads-first ordering across SkillHeader, SkillStats, SoulStats, SkillListItem, and related list surfaces; keep both current and all-time install information visible or accessibly exposed; add focused regression tests for install edge cases; coordinate the narrower download-icon work with #1854; and avoid unrelated backend, moderation, security, or formatting churn.
