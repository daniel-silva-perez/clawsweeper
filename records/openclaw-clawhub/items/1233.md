Keep this PR open. Current main only partially overlaps with the proposed stats semantics cleanup: SkillHeader now uses a Download icon for downloads, but it still renders stars first, keeps current/all-time installs as separate History rows, and has no installs tooltip utility or focused tests. Shared SkillStats, SoulStats, and SkillListItem surfaces still use Package for downloads and older ordering/style. The related #1854 is a narrower download-icon PR and does not supersede this broader PR.

Keep this PR open for maintainer review, or replace it with a clean scoped update against current main.
