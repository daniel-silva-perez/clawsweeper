## Summary

Keep this PR open. Current main still does not implement the PR's intended skill/soul stats semantics cleanup: the skill header and shared stat components still use Package icons for downloads, keep star-first displays in triplet contexts, show current/all-time installs as separate visible stats, and do not include the proposed installs tooltip utility or tests. The related open draft PR #1854 is narrower and does not supersede this broader cleanup.

## Best Possible Solution

Keep this PR open for maintainer review or a scoped replacement. The best path is to rebase/update it against current main, apply semantic download/user icons and downloads-first ordering consistently across SkillHeader, SkillStats, SoulStats, and any public list surfaces maintainers want covered, preserve both current and all-time install information in visible or accessible DOM, add focused regression tests for install display edge cases, and avoid unrelated backend or formatting churn. Coordinate the narrower download-icon work with #1854 rather than closing this broader PR as implemented.
