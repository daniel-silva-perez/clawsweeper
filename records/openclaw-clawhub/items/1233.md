## Summary

Keep this PR open. Current main still does not implement the PR’s intended stats semantics cleanup: skill and soul stat components still use the package icon for downloads, show stars before downloads in shared triplet lines, and do not include the proposed installs tooltip utility or focused install-display regression tests. The related open draft PR #1854 is narrower and does not supersede this broader work.

## Best Possible Solution

Keep this PR open for maintainer review or replace it with a scoped update against current main. The best path is to apply semantic download/user icons and downloads-first ordering consistently across SkillHeader, SkillStats, SoulStats, and public list surfaces; keep both current and all-time install information available in visible or accessible DOM; add focused regression tests for install display edge cases; coordinate the narrower download-icon work with #1854; and avoid unrelated backend or formatting churn.
