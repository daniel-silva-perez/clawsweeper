## Summary

Keep this PR open. Current main at 3701733797866be20db584cab84f4ba3572ad7ff still has the older skill/soul stats UI: Package icons remain for downloads, metric order is still stars before downloads, current/all-time installs are separate visible header stats rather than a centralized tooltip/accessibility pattern, and the installsTooltip utility/tests are absent. A newer related draft PR #1854 overlaps only with the download icon cleanup and is not merged, so it does not make this broader PR obsolete or implemented.

## Best Possible Solution

Keep this PR open for maintainer review or replacement by a scoped successor. The best implementation path is to update it against current main, apply semantic download/user icons and downloads-first ordering consistently across SkillHeader, SkillStats, and SoulStats, preserve both current and all-time install information in accessible DOM or an ARIA-backed tooltip pattern, add focused regression tests for install display edge cases, and keep unrelated backend or formatting churn out of the final diff. Coordinate with #1854 for the narrower download-icon overlap.
