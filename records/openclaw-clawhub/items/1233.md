## Summary

Keep this PR open. Current main does not fully implement the intended stats-semantics change, and the proposed patch still has an accessibility regression around hiding current install counts behind a native title tooltip, so it is not an obsolete PR that can be closed as implemented.

## What This Changes

The PR branch updates SkillHeader, SkillStats, and SoulStats stat presentation, adds an installsTooltip formatter and tests, and adds skill detail coverage for current-only installs.

## Best Possible Solution

Land a clean, rebased stats-semantics update that consistently applies semantic download/install icons and the chosen ordering across public skill and soul stat surfaces, keeps both current and all-time install counts visible or exposed through an accessible tooltip pattern, preserves current layout behavior, adds focused regression coverage for install edge cases, and coordinates with #1854.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Expose current installs without relying on title:** `src/components/SkillHeader.tsx:224`
  - body: When all-time installs are positive, this renders only the all-time installs text and moves the current install count into the native title tooltip. Native title tooltips are not reliably reachable on touch devices or keyboard-only paths, so the current install metric regresses from visible text on main to hover-only detail. Keep both values visible or use an accessible tooltip pattern with proper ARIA wiring.
  - confidence: 0.88
