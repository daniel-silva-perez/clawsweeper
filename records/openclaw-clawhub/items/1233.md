## Summary

Keep open. Current main only partially overlaps the intended stats-semantics change, and the PR branch still has a concrete accessibility regression around current install counts being available only through a native title tooltip. The branch also overlaps the narrower open draft #1854, so this needs maintainer review/rebase rather than cleanup closure.

## What This Changes

The PR updates skill detail, shared skill stats, and soul stats UI to use clearer download/install icon semantics and ordering, adds an installs tooltip formatter, and adds focused installs display tests.

## Best Possible Solution

Land a clean, rebased stats-semantics update that consistently applies semantic download/install icons and the chosen ordering across public skill and soul stat surfaces, keeps both current and all-time install counts visible or available through an accessible ARIA tooltip pattern, preserves current main layout behavior, adds focused regression coverage for install-count edge cases, and coordinates with #1854.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Expose current installs outside native title:** `src/components/SkillHeader.tsx:226`
  - body: When `installsAllTime > 0`, the visible text shows only the all-time install count and puts `installsCurrent` inside the native `title` tooltip. Native title tooltips are not reliably reachable on touch devices or keyboard-only paths, so this regresses the current install metric from visible text on main to hover-only detail. Keep both values visible or use an accessible tooltip pattern with proper ARIA wiring.
  - confidence: 0.88
