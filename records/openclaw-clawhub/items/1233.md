## Summary

Keep this PR open. Current main only partially overlaps with the PR: the skill detail header now uses a download icon and visibly shows current/all-time installs, but the shared skill and soul stat components still use the package icon, keep stars before downloads in triplet lines, and lack the proposed installs tooltip utility and focused tooltip tests. The related draft PR #1854 is narrower and does not supersede the broader skill/soul stats semantics cleanup.

## Best Possible Solution

Keep this PR open for maintainer review or replace it with a clean, scoped update against current main. The best path is to apply semantic download/user icons and downloads-first ordering consistently across SkillHeader, SkillStats, SoulStats, SkillListItem, and related public list surfaces; keep both current and all-time install information visible or accessibly exposed; add focused regression tests for install edge cases; coordinate the narrower download-icon work with #1854; and avoid unrelated backend or formatting churn.
