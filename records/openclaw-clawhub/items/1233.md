## Summary

Keep this PR open. Current main only partially overlaps the intended stats-semantics cleanup, and the PR still has an actionable accessibility regression around hiding one install metric behind a native title tooltip. The final diff is limited to UI/test utility files, so I did not find a security or supply-chain concern, but the branch needs maintainer review and coordination with the narrower draft #1854 rather than cleanup closure.

## What This Changes

The PR branch updates skill detail, shared skill stats, and soul stats UI to use clearer download/install icon semantics and ordering, adds an installs tooltip formatter, and adds focused installs display tests.

## Best Possible Solution

Land a clean, rebased stats-semantics update that consistently applies semantic download/install icons and the chosen ordering across the public skill/soul surfaces, keeps both current and all-time install counts visible or exposed through an accessible tooltip pattern, preserves current layout classes from main, adds focused regression coverage for install-count edge cases, and either subsumes or coordinates with #1854.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Expose current installs without relying on title:** `src/components/SkillHeader.tsx:224`
  - body: When `installsAllTime > 0`, the rendered text shows only the all-time install count while `installsCurrent` is available only through the native `title` tooltip. Native title tooltips are not reliably reachable on touch devices or keyboard-only paths, so the current install metric regresses from visible text on main to inaccessible hover-only detail. Keep both values visible or use an accessible tooltip pattern with proper ARIA wiring.
  - confidence: 0.88
