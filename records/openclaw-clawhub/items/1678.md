## Summary

Keep this issue open. Repository policy for openclaw/clawhub only allows auto-closing pull requests that are certainly implemented on main, and this item is an issue. Current main also still appears to leave the reported behavior unresolved: org publishing is explicit via `--owner`, not inferred from package scope or source repo, and GitHub shorthand sources are fetched before `--source-path` is applied.

## Best Possible Solution

Keep this issue open and implement a focused package-publish fix: apply `--source-path` to GitHub shorthand sources before archive extraction, add regression coverage for `owner/repo@ref --source-path plugins/foo`, and decide/document org ownership behavior. If scoped package names or `--source-repo` should imply org ownership, add that inference with publisher membership checks; otherwise make `--owner <org>` the supported flow and update CLI help/docs accordingly.
