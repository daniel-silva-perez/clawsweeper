## Summary

Keep open. Repository policy for openclaw/clawhub keeps issues out of auto-close, and current main still leaves the reported package-publish problem unresolved enough for maintainer follow-up: local-folder publish only sends an org owner when `--owner` is explicit, while `owner/repo@ref --source-path` still fetches the GitHub archive before the source-path override can affect package discovery.

## Best Possible Solution

Keep this open for maintainer follow-up. The narrow code fix is to apply `--source-path` to GitHub shorthand sources before `fetchGitHubSource` and add regression coverage for `clawhub package publish owner/repo@ref --source-path plugins/foo`. Separately, maintainers should decide whether org namespace selection should be explicit via `--owner <org>` or inferred from scoped package/source repo data with membership checks, then update CLI help/docs to match the chosen contract.
