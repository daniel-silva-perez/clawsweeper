## Summary

Keep this issue open. Repository policy keeps ClawHub issues out of auto-close, and current main still leaves both reported package-publish paths unresolved: local folder publishing only sends an org owner when `--owner` is explicit, while GitHub shorthand sources are fetched before `--source-path` is applied to package discovery.

## Best Possible Solution

Keep this open and route it to package-publish follow-up. The narrow implementation piece is to apply `--source-path` to GitHub shorthand sources before `fetchGitHubSource` and add regression coverage for `owner/repo@ref --source-path plugins/foo`; maintainers should separately decide whether org namespace selection should stay explicit via `--owner <org>` or infer from scoped package/source repo data with membership checks.
