## Summary

Keep this issue open. Repository policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on main, and this item is an issue. The current main code also still leaves the reported package-publish behavior actionable: local-folder publishes only include an org owner when `--owner` is explicitly supplied, while GitHub shorthand sources are fetched before `--source-path` can affect archive extraction.

## Best Possible Solution

Keep this issue open and make a focused CLI/package-publish fix. Apply `--source-path` to GitHub shorthand sources before archive extraction, add regression coverage for `clawhub package publish owner/repo@ref --source-path plugins/foo`, and decide the org ownership contract: either infer org ownership from scoped package names or source repo with membership checks, or document `--owner <org>` as the required supported flow with clearer CLI help.
