## Summary

Keep this issue open. Repository policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on main, and this item is an issue. The current main code also still leaves the reported package-publish behavior actionable: org ownership is explicit through `--owner`, not inferred from a scoped package name or `--source-repo`, and `--source-path` is applied after GitHub shorthand sources are fetched, so it affects provenance metadata rather than which subdirectory is extracted.

## Best Possible Solution

Keep this issue open and make a focused package-publish fix. Apply `--source-path` to GitHub shorthand sources before archive extraction, add regression coverage for `clawhub package publish owner/repo@ref --source-path plugins/foo`, and decide the org ownership contract: either infer `ownerHandle` from scoped package names or source repo with membership checks, or make `--owner <org>` the documented supported flow with clearer CLI help and docs.
