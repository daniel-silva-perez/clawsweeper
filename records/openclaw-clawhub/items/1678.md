## Summary

Keep this issue open. Current main still leaves the reported package-publish behavior actionable: local folder publishing only sends an org owner when `--owner` is explicit, while GitHub shorthand sources are fetched before `--source-path` is applied. The repository policy for openclaw/clawhub also keeps issues open unless a maintainer handles them explicitly.

## Best Possible Solution

Keep this issue open and route it to a focused package-publish follow-up. The best implementation path is to apply `--source-path` to GitHub shorthand sources before `fetchGitHubSource`, add regression coverage for `owner/repo@ref --source-path plugins/foo`, and have maintainers choose the org publishing contract: either document and improve the explicit `--owner <org>` flow, or implement validated owner inference from scoped package/source repo data with membership checks.
