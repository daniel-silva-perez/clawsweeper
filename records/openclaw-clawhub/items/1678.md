## Summary

Keep open. Repository policy for openclaw/clawhub keeps issues open in this cleanup pass, and current main still leaves the reported package-publish workflow unresolved: GitHub shorthand sources are fetched before `--source-path` can affect the subdirectory, while org owner routing is still explicit via `--owner` rather than inferred from package scope or source repo.

## Best Possible Solution

Keep this issue open. A narrow code change should apply `--source-path` to GitHub shorthand sources before `fetchGitHubSource`, with regression coverage for `clawhub package publish owner/repo@ref --source-path plugins/foo`. Separately, maintainers should decide whether org namespace selection remains explicit via `--owner <org>` or becomes inferred from scoped package/source repo metadata, then update CLI help, docs, and backend/CLI tests to match that contract.
