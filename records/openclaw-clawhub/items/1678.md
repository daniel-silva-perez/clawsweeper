## Summary

Keep this issue open. The repository-specific review policy for openclaw/clawhub only permits auto-closing PRs that are certainly implemented on main, and this item is an issue. Current main also still leaves the reported package-publish behavior actionable: org owner selection is explicit-only, and `--source-path` is used for provenance after GitHub shorthand sources are fetched rather than being folded into the archive subdirectory selection before package discovery.

## Best Possible Solution

Keep this issue open and make a focused CLI/package-publish fix: apply `--source-path` to GitHub shorthand sources before archive extraction and package discovery, add regression coverage for `clawhub package publish owner/repo@ref --source-path plugins/foo`, and clarify org publishing by either implementing safe owner inference from scoped package/source repo data with membership checks or documenting `--owner <org>` as the supported flow with better help and errors.
