## Summary

Keep open. Repository policy for openclaw/clawhub keeps issues out of auto-close, and current main still leaves the reported package publish gaps unresolved: GitHub shorthand sources are fetched before `--source-path` can change the fetched subdirectory, and local-folder publishing only sends an owner when `--owner` is explicitly provided.

## Best Possible Solution

Keep this open for maintainer follow-up. A narrow implementation should apply `--source-path` to GitHub shorthand sources before `fetchGitHubSource` and add regression coverage for `clawhub package publish owner/repo@ref --source-path plugins/foo`. Separately, maintainers should decide whether org namespace selection should remain explicit via `--owner <org>` or be inferred from scoped package/source repo data, then update CLI help and docs to match that contract.
