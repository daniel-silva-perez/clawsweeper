## Summary

Keep open. Repository policy for openclaw/clawhub keeps issues open, and current main still leaves the package-publish behavior actionable: org ownership is explicit-only via --owner, while GitHub shorthand sources are fetched before --source-path can select a monorepo subdirectory.

## Best Possible Solution

Keep this issue open and make a focused package-publish fix: apply --source-path to GitHub shorthand sources before archive extraction/package discovery, add regression coverage for clawhub package publish owner/repo@ref --source-path plugins/foo, and clarify org publishing by either implementing safe owner inference from package scope/source repo with membership checks or documenting --owner <org> as the supported flow with better help and error text.
