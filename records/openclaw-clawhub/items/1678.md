## Summary

Keep this issue open. The repository policy for openclaw/clawhub keeps issues open, and current main still leaves both reported package-publish behaviors actionable: local folder publishing only sends an owner when `--owner` is explicit, while GitHub shorthand fetches the repo before `--source-path` can select a monorepo subdirectory.

## Best Possible Solution

Keep this issue open and make a focused CLI/package-publish fix: apply `--source-path` to GitHub shorthand sources before archive extraction, add regression coverage for `clawhub package publish owner/repo@ref --source-path plugins/foo`, and clarify org publishing by either implementing safe owner inference from scoped package/source repo data with membership checks or improving the documented `--owner <org>` flow and errors.
