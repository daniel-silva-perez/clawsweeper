## Summary

Keep open. Repository policy keeps ClawHub issues out of auto-close, and current main still leaves the reported CLI publish path unresolved: GitHub shorthand sources are fetched before `--source-path` is applied, while org ownership is only sent when the caller provides an explicit `--owner` value.

## What This Changes

The issue requests a working `clawhub package publish` flow for publishing a monorepo plugin subdirectory under an organization namespace.

## Best Possible Solution

Apply `--source-path` to resolved GitHub shorthand/URL sources before archive fetch, add a regression for monorepo subdirectory publishing, and make the org namespace contract explicit in CLI docs/help: either require `--owner <org>` with server-side membership checks or separately design safe org inference.
