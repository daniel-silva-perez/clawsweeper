## Summary

Keep this issue open. ClawHub review policy does not allow auto-closing issues, and current main still leaves the reported publish behavior actionable: org ownership is explicit-only through `--owner`, while GitHub shorthand sources fetch the repo root before `--source-path` can select a monorepo subdirectory.

## Best Possible Solution

Keep this issue open and make a focused CLI/package-publish fix: apply `--source-path` to GitHub shorthand sources before archive extraction, add regression coverage for `clawhub package publish owner/repo@ref --source-path plugins/foo`, and clarify whether org publishing should be inferred from scoped package/source repo data or remain an explicit `--owner <org>` contract with better help and errors.
