## Summary

Keep this issue open. The ClawHub review policy only permits auto-closing PRs that are definitely implemented on main, and this item is an issue. The current main code also still leaves the reported package-publish behavior actionable: local-folder publishes only send an org owner when `--owner` is supplied, while `owner/repo@ref --source-path ...` does not apply the requested subdirectory before fetching the GitHub archive.

## Best Possible Solution

Keep this issue open and make a focused CLI/package-publish fix. Apply `--source-path` to GitHub shorthand sources before archive extraction, add regression coverage for `clawhub package publish owner/repo@ref --source-path plugins/foo`, and decide the org ownership contract: either infer org ownership from scoped package names or source repo with membership checks, or document `--owner <org>` as the required supported flow with clearer CLI help/errors.
