## Summary

Keep open. This is a ClawHub issue, and the review policy only permits auto-closing ClawHub pull requests that are certainly implemented on main. Current main also still leaves the reported behavior unresolved: package publish only sends an owner handle when `--owner` is explicit, and `--source-path` is not applied before fetching/extracting a GitHub shorthand source.

## Best Possible Solution

Keep this issue open and implement a focused package-publish fix: apply `--source-path` to GitHub shorthand sources before archive extraction, add regression coverage for `owner/repo@ref --source-path plugins/foo`, and make an explicit product decision on org ownership inference. If scoped package names should imply org ownership, add inference with membership checks; otherwise document `--owner <org>` as the supported flow and update stale admin-only docs/help text.
