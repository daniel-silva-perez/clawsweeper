## Summary

Keep open. Repository policy for this ClawHub review run allows auto-closing only PRs that are certainly implemented on main, and this is an issue. Substantively, current main has partial support for the reported metadata shape, but it does not prove the false-positive path is fixed end to end: `metadata.openclaw` is parsed into normalized `parsed.clawdis` and used by the LLM prompt/UI, while the static moderation env collector still only reads direct `requires/env/primaryEnv` fields from raw frontmatter/metadata and does not descend into nested `metadata.openclaw` or consume normalized `parsed.clawdis`.

## Best Possible Solution

Keep this issue open for maintainer handling. The likely fix is to make static moderation consume normalized `parsed.clawdis`, or explicitly collect declarations from `metadata.openclaw`, `metadata.clawdis`, and `metadata.clawdbot`; add a regression test using the exact inline JSON `metadata.openclaw.requires.env` shape from the report; then rescan or manually clear the affected published versions. Maintainers can also decide whether to consolidate this with open related issue #1053 as the broader metadata-mismatch thread.
