## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub PRs that are definitely implemented on current main. Substantively, current main documents and parses `metadata.openclaw.requires.env`, but the static publish scan still receives raw frontmatter/metadata and its env collector only checks direct fields, while the live affected DeepRead records remain suspicious.

## Best Possible Solution

Keep this issue open. The best path is to make static publish moderation consume the same normalized parsed Clawdis/OpenClaw metadata used by display and LLM context, or explicitly collect env declarations from `metadata.openclaw`, `metadata.clawdis`, and `metadata.clawdbot`; add a regression test using the inline JSON frontmatter from this report. Then rescan/backfill or manually review the affected published versions, while leaving broader package-level metadata ingestion to #1053.
