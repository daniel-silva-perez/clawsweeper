## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that current main definitely implements. Substantively, current main still has a static moderation gap: `metadata.openclaw.requires.env` is documented and normalized into `parsed.clawdis`, but the static publish scan still sends only raw frontmatter/metadata to the env collector, which reads only direct env fields.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The likely code fix is to make static moderation consume normalized `parsed.clawdis`, or explicitly collect env declarations from `metadata.openclaw`, `metadata.clawdis`, and `metadata.clawdbot`; if `package.json` or `metadata.json` env declarations are intended to be supported for skills, add/document that ingestion path too. Add a regression test for the inline JSON `metadata.openclaw.requires.env` shape from the report, then rescan or clear affected published versions.
