## Summary

Keep open. The repository policy for this ClawHub pass only permits auto-closing pull requests, not issues. Substantively, current main still appears to miss the reported path: nested `metadata.openclaw.requires.env` is documented and normalized by the parser, but the static publish scan only receives raw frontmatter/metadata and its env collector only reads direct env fields.

## Best Possible Solution

Keep this issue open and land a narrow backend fix after maintainer review. Static moderation should consume normalized parsed Clawdis metadata, or the env collector should explicitly descend into `metadata.openclaw`, `metadata.clawdis`, and `metadata.clawdbot`; add a regression test using the inline JSON frontmatter shape from this report. Decide package-level `metadata.json` / `package.json` support with #1053, then rescan or manually review affected published versions.
