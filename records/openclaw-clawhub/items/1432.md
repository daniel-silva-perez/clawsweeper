## Summary

Keep open. This is an openclaw/clawhub issue, and the repository-specific cleanup policy for this review allows auto-closing only PRs that are certainly implemented on main. The source also does not prove the reported false-positive path is fixed end to end: current main parses `metadata.openclaw` into `parsed.clawdis`, but the static moderation env collector still only reads direct `requires/env/primaryEnv` fields from `frontmatter` and raw `metadata`, not nested `metadata.openclaw` or normalized `parsed.clawdis`.

## Best Possible Solution

Keep this issue open for maintainer handling. The likely code path is to make static moderation consume normalized `parsed.clawdis`, or explicitly collect env declarations from `metadata.openclaw`, `metadata.clawdis`, and `metadata.clawdbot`; add a regression test using the exact inline JSON `metadata.openclaw.requires.env` shape from the report; then rescan/backfill the affected published versions and clear confirmed false positives through the existing moderation override path. Maintainers may also choose to consolidate this with the open related metadata-mismatch reports, but this automated review should not close it.
