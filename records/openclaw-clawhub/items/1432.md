## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the reported false-positive path still appears valid: current main documents and parses nested `metadata.openclaw.requires.env`, but the static moderation collector used during publish and static rescan still only sees direct env declaration fields. The affected live DeepRead skills remain marked suspicious while their hosted `SKILL.md` files declare `DEEPREAD_API_KEY` in the documented nested shape.

## What This Changes

The issue asks ClawHub to extract `openclaw.requires.env` and `primaryEnv` from published skill metadata so correctly-declared env requirements do not trigger suspicious registry-metadata mismatches.

## Best Possible Solution

Make static publish and static rescan moderation consume the same normalized OpenClaw runtime metadata used by parsing, display, and LLM context; add regression coverage for inline JSON `metadata.openclaw.requires.env` and `primaryEnv`; then rescan or manually review the affected DeepRead versions while leaving broader `metadata.json` ingestion to #1053.
