## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that current main definitely implements. Substantively, current main still has a split path: `metadata.openclaw.requires.env` is documented and parsed into normalized Clawdis metadata, but the static moderation env collector still reads only direct raw `frontmatter`/`metadata` fields and does not consume normalized `parsed.clawdis` or descend into `metadata.openclaw`. Related env-metadata false-positive issue #1053 is also still open.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The likely fix is to make static moderation consume normalized `parsed.clawdis`, or explicitly collect declarations from `metadata.openclaw`, `metadata.clawdis`, and `metadata.clawdbot`; add a regression test using the inline JSON `metadata.openclaw.requires.env` shape from the report; then rescan or manually clear the affected published versions. Maintainers may also consolidate this with open related issue #1053 if they want one canonical metadata-mismatch thread.
