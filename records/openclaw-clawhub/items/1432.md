## Summary

Keep this issue open. The repository-specific cleanup policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on main, and this item is an issue. Substantively, current main still has a static moderation gap for nested `metadata.openclaw.requires.env`: the parser normalizes it into `parsed.clawdis`, but the static env allowlist is built from raw direct metadata fields only.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The likely fix is to make static moderation consume normalized `parsed.clawdis`, or explicitly collect declarations from `metadata.openclaw`, `metadata.clawdis`, and `metadata.clawdbot`; if package.json or metadata.json env declarations are supported skill metadata, add that extraction path too. Add regression coverage for the inline JSON `metadata.openclaw.requires.env` shape from the report, then rescan or manually clear affected published versions.
