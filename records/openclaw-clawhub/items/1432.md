## Summary

Keep open. Repository policy for this ClawHub cleanup pass does not allow auto-closing open issues, and the substantive bug still appears valid: current main parses `metadata.openclaw` for stored/display/LLM metadata, but static publish moderation still collects declared env vars only from direct top-level fields, while the affected DeepRead skills remain suspicious on the live API.

## What This Changes

Registry metadata and moderation should extract `metadata.openclaw.requires.env` and `primaryEnv` from uploaded skill files so correctly declared env requirements do not cause false suspicious flags.

## Best Possible Solution

Keep this issue open. The best path is to make static publish moderation consume the same normalized parsed Clawdis/OpenClaw metadata used by display and LLM context, or explicitly collect env declarations from `metadata.openclaw`, `metadata.clawdis`, and `metadata.clawdbot`; add a regression test using the inline JSON frontmatter from this report; then rescan/backfill or manually review the affected DeepRead versions while leaving broader package-level metadata ingestion to #1053.
