Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing pull requests that current main definitely implements. ## What This Changes

The issue requests that public skill detail page content fit within the browser/container width instead of overflowing horizontally.

## Best Possible Solution

Add a deterministic desktop and mobile regression for skill-detail README/content overflow using the reported pages or an equivalent fixture, then tighten only the skill-detail/markdown layout so content wraps or scrolls in the intended inner container without document-level horizontal overflow or clipping. Close manually after the targeted test or production verification proves the reported case is fixed.
