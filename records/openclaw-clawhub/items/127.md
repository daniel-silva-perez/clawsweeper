## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy for this run only permits auto-closing pull requests that current main definitely implements. Current main has generic skill-detail and markdown overflow mitigations, but there is still no targeted proof for the three reported pages or the desktop screenshot case.

## What This Changes

The issue reports horizontal overflow on public skill detail pages and asks for the content container to fit and respond to browser width.

## Best Possible Solution

Add a deterministic desktop and mobile regression for skill-detail README/content overflow using the reported pages or an equivalent fixture, then tighten only the skill-detail/markdown layout so content wraps or scrolls inside the intended inner container without document-level horizontal overflow or clipping.
