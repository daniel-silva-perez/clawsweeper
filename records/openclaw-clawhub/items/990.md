## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy says ClawHub issues should remain open unless the item is a PR certainly implemented on current main. Substantively, current main exposes suspicious/moderation state through the v1 JSON API and checked-in API docs, but the requested static skill-page head metadata is still missing.

## Best Possible Solution

Keep this issue open for maintainer product follow-up. A concrete path is to pass public moderationInfo from the skill page loader into the route head data, extend buildSkillMeta or the route head output with stable clawhub:suspicious metadata plus a public verdict/reason summary where appropriate, add route/head tests for flagged and clean skill pages, and document when agents should use static metadata versus the existing v1 detail and moderation APIs. Coordinate with #731 only for deeper scanner-finding transparency; it does not replace this static metadata request.
