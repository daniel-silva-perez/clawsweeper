## Summary

Keep open. Current main already exposes public suspicious/moderation state through the JSON API and UI, but the static machine-readable skill-page metadata requested for agent/web_fetch use is still missing; the ClawHub cleanup policy also keeps issues open for maintainer handling.

## What This Changes

Expose a machine-readable suspicious/moderation flag in skill page static HTML and API responses so agents can warn about or filter flagged skills.

## Best Possible Solution

Add public static head metadata for skill moderation using loaderData.initialData.result.moderationInfo, keep the existing v1 moderation response as the API source of truth, add focused head/helper/API tests for flagged and clean skills, and document how agents should consume the static metadata versus the JSON endpoints.
