## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy for this run only permits auto-closing PRs that are certainly implemented on main. Substantively, current main exposes suspicious state in the public API and UI, but the requested machine-readable static skill-page metadata for HTML/web-fetch consumers is still absent.

## Best Possible Solution

Keep this issue open for maintainer product handling. A concrete path is to carry public moderationInfo from fetchSkillPageData into the route loader/head metadata path, add a stable head tag such as clawhub:suspicious plus a public verdict/reason summary when available, cover it in route/OG tests, and update API/security docs to direct agents to the existing detail, moderation, and scan endpoints for structured checks.
