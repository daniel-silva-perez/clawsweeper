## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy for this review run only permits auto-closing pull requests that current main definitely implements. Substantively, current main now exposes suspicious/moderation state through the JSON API and related docs/tests, but the requested static HTML/meta-tag path for agents that fetch skill pages is still not implemented.

## Best Possible Solution

Keep this issue open for maintainer product handling. A good implementation path is to pass public moderationInfo from the skill page data loader into the route head metadata path, extend buildSkillMeta or the route head output with stable clawhub:suspicious metadata and a public verdict/reason summary where appropriate, add tests for flagged and clean skill pages, and document when agents should use static metadata versus the existing detail, moderation, and scan APIs.
