## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy for this run only permits auto-closing pull requests that are certainly implemented on main. Current main exposes suspicious/moderation state through public API surfaces and docs, but the requested machine-readable static skill-page HTML metadata for web-fetch consumers still appears absent.

## Best Possible Solution

Keep this issue open for maintainer product handling. A concrete implementation path is to carry public moderation state from fetchSkillPageData into the route loader/head metadata path, extend buildSkillMeta or the route head to emit a stable tag such as `clawhub:suspicious` plus public verdict/reason summary when available, and update docs to point agents at the existing detail, moderation, and scan APIs for structured checks.
