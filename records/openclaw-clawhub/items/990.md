## Summary

Keep open. Current main already exposes public suspicious/moderation state through the JSON API and UI, but the requested static machine-readable skill-page metadata for agent/web_fetch use is still missing; ClawHub cleanup policy also keeps issues open for maintainer handling.

## What This Changes

Expose suspicious/moderation status in static skill-page metadata and API responses so agents can detect flagged skills programmatically.

## Best Possible Solution

Add stable public HTML head metadata for skill moderation using loaderData.initialData.result.moderationInfo, keep the existing v1 moderation response as the JSON source of truth, add focused route-head/helper/API tests for flagged and clean skills, and document how agents should consume static metadata versus API endpoints.
