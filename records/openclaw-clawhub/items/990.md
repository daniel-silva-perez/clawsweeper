## Summary

Keep open. This is a ClawHub issue, and the repo cleanup policy only permits auto-closing PRs that current main certainly implements. Current main exposes suspicious/moderation state through the public v1 API and docs, but the requested machine-readable static skill-page metadata is still missing.

## Best Possible Solution

Keep this issue open for maintainer product/security follow-up. A concrete path is to pass public moderationInfo from the skill page loader into the route head data, emit stable static metadata such as clawhub:suspicious and possibly a conservative public verdict/reason summary, add route/head tests for flagged and clean skill pages, and document when agents should use static metadata versus the existing v1 detail and moderation APIs. Coordinate with #731 only for deeper scanner-finding transparency; it does not replace this static metadata request.
