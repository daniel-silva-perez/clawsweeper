## Summary

Review evidence is sufficient for a keep-open decision: current main exposes suspicious/moderation state through the JSON API and UI, but the requested static machine-readable skill-page metadata is still missing, and ClawHub cleanup policy does not auto-close open issues.

## What This Changes

The issue requests exposing suspicious/moderation status in static skill-page metadata and API responses so agents can detect flagged skills programmatically.

## Best Possible Solution

Keep this issue open for maintainer security/product follow-up. A good implementation would pass public moderationInfo from the skill page loader into the route head, emit stable static metadata such as clawhub:suspicious and only-approved public verdict/reason summary fields, add route/head and og helper tests for flagged and clean pages, and document when agents should use static metadata versus the existing v1 detail and moderation APIs. Coordinate with #731 for deeper scanner findings, but do not treat it as a replacement for this static metadata request.
