## Summary

Keep open. Current main already exposes public suspicious/moderation state through the JSON API and UI, but the static machine-readable skill-page metadata requested for agent/web_fetch use is still missing; repository cleanup policy also does not auto-close open ClawHub issues.

## What This Changes

Expose each skill's suspicious/moderation state in static HTML metadata and API responses so agents can programmatically warn about or filter flagged skills.

## Best Possible Solution

Keep this issue open for maintainer security/product follow-up. A good implementation would pass public moderationInfo from the skill page loader into the route head, emit stable static metadata for suspicious/moderation verdict using only public fields, add route-head and metadata helper tests for flagged and clean skills, and document when agents should use static metadata versus the existing v1 detail and moderation APIs. Coordinate with #731 for deeper scanner-finding transparency, but do not treat it as a replacement.
