## Summary

Keep open. This is an open ClawHub issue, and the supplied repository policy only permits auto-close for pull requests that are certainly implemented on current main. Substantively, current main exposes suspicious/moderation state through the v1 JSON API and docs, but the requested static skill-page head metadata for static fetchers is still missing. Related #731 covers deeper scanner-finding transparency, not this static metadata contract.

## Best Possible Solution

Keep this issue open for maintainer product/security follow-up. A concrete path is to pass public moderationInfo from the skill page loader into the route head data, extend buildSkillMeta or the route head output with stable clawhub:suspicious metadata plus a conservative public verdict/reason summary where appropriate, add route/head tests for flagged and clean skill pages, and document when agents should use static metadata versus the existing v1 detail and moderation APIs. Coordinate with #731 only for deeper scanner-finding transparency; it does not replace this static metadata request.
