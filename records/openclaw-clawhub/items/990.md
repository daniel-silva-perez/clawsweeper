## Summary

Keep open. This is a ClawHub issue, and the supplied cleanup policy for this repository only allows auto-closing pull requests that are certainly implemented on main. Substantively, current main exposes suspicious state through the JSON API and documents it, but the requested stable machine-readable static skill-page metadata is still missing from the route head metadata path.

## Best Possible Solution

Keep this issue open for maintainer product handling. A concrete implementation path is to carry public moderationInfo from fetchSkillPageData into the route loader/head metadata path, extend buildSkillMeta or route head output with a stable clawhub:suspicious tag and a public verdict/reason summary where appropriate, add route-head tests for flagged and clean skills, and update docs to tell agents when to use static metadata versus the existing detail, moderation, and scan APIs.
