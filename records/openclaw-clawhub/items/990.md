## Summary

Keep open. Current main covers the API half of the request with public moderation fields, but the skill page head still does not emit stable machine-readable suspicious/moderation metadata for static fetchers. The ClawHub cleanup policy also says open issues should not be auto-closed in this pass.

## What This Changes

Expose suspicious/moderation status in static skill-page metadata and API responses so agents can detect and avoid flagged skills programmatically.

## Best Possible Solution

Keep the issue open and add stable static metadata to skill pages using the public moderationInfo already loaded for the route, while preserving the existing API moderation response and documenting when agents should rely on page metadata versus the v1 detail/moderation endpoints.
