## Summary

Keep open. Current main covers the JSON API and UI parts of the request, but the static skill page head still lacks machine-readable moderation metadata for HTML-only agent fetches; ClawHub cleanup policy also keeps issues open for maintainer handling.

## What This Changes

Expose suspicious/moderation status in static skill-page HTML and API responses so agents can detect flagged skills programmatically.

## Best Possible Solution

Add stable public HTML head metadata for already-public skill moderation state, keep the v1 JSON moderation response as the API source of truth, add focused tests for flagged and clean skill pages, and document how agents should consume static metadata versus API endpoints.
