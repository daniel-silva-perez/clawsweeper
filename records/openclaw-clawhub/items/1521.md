## Summary

Keep this open. This is a ClawHub issue, so the repository cleanup policy does not allow automated closure here, and current main still lets exhausted OpenAI embedding failures abort skill publishing before version insertion. The reported public skill alias now resolves to a clean latest skill response, but the source-side publish path remains brittle and the historical flagged-version/removal part still needs maintainer/operator handling.

## Best Possible Solution

Keep this issue open. The best source fix is to make skill publishing resilient when embedding generation exhausts retries, ideally by recording a missing/placeholder embedding state and scheduling embedding generation or repair asynchronously without blocking version insertion. Maintainers should separately inspect the OpenAI request ID and decide whether any historical flagged versions need a rescan, manual override, or removal under ClawHub moderation policy.
