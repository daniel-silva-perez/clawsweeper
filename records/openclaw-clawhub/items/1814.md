## Summary

Keep open. This is a ClawHub issue, and the review policy for this repository says open issues should remain open; only PRs that are certainly implemented on current main are auto-close candidates. Substantively, current main still leaves the reported REST/CLI gap for org-owned skill version publishes, and open PR #1840 is the active implementation candidate for this issue.

## Best Possible Solution

Keep this issue open and review/land PR #1840 or an equivalent implementation. The fix should add ownerHandle/--owner support for skill publishing, preserve it in JSON and multipart REST payloads, resolve the requested publisher server-side with at least publisher role, pass ownerPublisherId into publishVersionForUser, and keep regression coverage for schema parsing, CLI payload generation, and both REST publish formats.
