## Summary

Keep open. This is a ClawHub issue, and the cleanup policy keeps issues open; current main also still lacks the requested REST/CLI owner selector for org-owned skill publishes. The linked open PR #1840 is the right implementation candidate to review rather than closing this early.

## What This Changes

Allow authenticated REST and CLI skill publishing to target an org owner so org members can publish new versions of org-owned skills from CI.

## Best Possible Solution

Land PR #1840 or an equivalent focused fix that threads ownerHandle/--owner through the shared publish schema, CLI multipart payload, REST JSON and multipart parsing, resolves the target publisher server-side with publisher-role authorization, passes ownerPublisherId into publishVersionForUser, and adds regression coverage for schema parsing, CLI payload generation, and both REST publish formats.
