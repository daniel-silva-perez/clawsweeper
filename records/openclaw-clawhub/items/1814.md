## Summary

Keep open. This is an open issue in openclaw/clawhub, where the cleanup policy only permits auto-closing pull requests that current main definitely implements. Substantively, current main still lacks an owner target in the REST/CLI skill publish path, and open PR #1840 is the implementation candidate with closing syntax.

## Best Possible Solution

Keep this issue open and review/land PR #1840 or an equivalent fix. The implementation should add an ownerHandle/--owner target for skill publishing, preserve it for both JSON and multipart REST payloads, resolve the target publisher server-side with at least publisher role, pass ownerPublisherId into publishVersionForUser, and cover schema parsing, CLI payload generation, and REST publish regression tests.
