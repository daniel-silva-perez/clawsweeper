## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that current main definitely implements. Substantively, current main still lacks an owner target in the skill REST/CLI publish path, while open PR #1840 is the active implementation candidate linked from the discussion.

## Best Possible Solution

Keep this issue open and review/land PR #1840 or an equivalent fix. The fix should add an ownerHandle/--owner target to skill publishing, preserve it for JSON and multipart REST payloads, resolve the target publisher server-side with at least publisher role, pass ownerPublisherId into publishVersionForUser, and cover schema parsing, CLI payload generation, and both REST publish formats.
