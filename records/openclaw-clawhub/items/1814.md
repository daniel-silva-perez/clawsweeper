## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review only allows auto-closing pull requests that are certainly implemented on current main. Substantively, current main still leaves the reported REST/CLI gap: the web publish action is owner-aware, but the REST skill publish schemas/parsers/handlers and skill CLI do not carry or resolve an org owner target. Open PR #1840 is the active implementation candidate and should be reviewed or replaced before this issue closes.

## Best Possible Solution

Keep this issue open and review/land PR #1840 or an equivalent fix. The implementation should add ownerHandle/--owner support for skill publishing, preserve it in JSON and multipart REST payloads, resolve the target publisher server-side with at least publisher role, pass ownerPublisherId into publishVersionForUser, and add regression coverage for schema parsing, CLI payload generation, and both REST publish formats.
