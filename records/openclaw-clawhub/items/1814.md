## Summary

Keep open. Repository policy for openclaw/clawhub keeps issues open, and the requested REST/CLI owner targeting for org-owned skill publishes is still missing on current main. Open PR #1840 is already the paired implementation candidate with closing syntax, so the right next step is maintainer review and landing that PR or an equivalent fix.

## What This Changes

Allow authenticated REST and CLI skill publishing to target an org publisher, so org members with publisher access can publish new versions of org-owned skills from CI.

## Best Possible Solution

Keep this issue open and review or land PR #1840, or an equivalent focused fix. The implementation should add ownerHandle/--owner support to the shared skill publish schema and CLI payload, preserve owner targeting in JSON and multipart REST publish, resolve the publisher server-side with at least publisher role, pass ownerPublisherId into publishVersionForUser, and cover schema, CLI, JSON REST, and multipart REST regression tests.
