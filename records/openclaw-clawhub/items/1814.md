## Summary

Keep open. This is a ClawHub issue, so the cleanup policy does not allow auto-closing it, and current main still lacks the requested owner selector in the REST/CLI skill publish path. The linked PR #1840 is the open closing implementation candidate and should be reviewed or replaced rather than closing this issue early.

## What This Changes

Add authenticated REST/CLI support for publishing new versions of org-owned skills by selecting an owner publisher, so CI can update org-owned skills without falling back to the actor's personal publisher.

## Best Possible Solution

Review and land PR #1840 or an equivalent focused fix that threads ownerHandle/--owner through the skill publish schema, CLI multipart payload, REST JSON and multipart parsing, resolves the publisher server-side with publisher role, passes ownerPublisherId into publishVersionForUser, and adds regression coverage for schema parsing, CLI payload generation, and both REST publish formats.
