## Summary

Keep this issue open. Current main can represent and hide a soft-deleted skill version from public reads/downloads once `skillVersions.softDeletedAt` is set, but there is still no owner-facing API, CLI, or UI workflow to hide/delete/restore one published version while keeping the skill live. The ClawHub cleanup policy for this pass also keeps issues open; only PRs are auto-close candidates when current main certainly implements the intended change.

## Best Possible Solution

Keep this open. Maintainers should define the per-version lifecycle policy, then implement authenticated owner/staff API routes and CLI or UI controls that resolve slug plus version, authorize owners/org members/staff, patch `skillVersions.softDeletedAt`, write audit logs, and consistently handle latest/tag/download/file/readme/cache semantics. Coordinate with #600 for incident handling while keeping #1370 separate for package/plugin lifecycle controls.
