## Summary

Keep this issue open. The ClawHub cleanup policy for this run keeps issues open, and current main still has only the data/read-side pieces for hidden skill versions, not an owner-facing API, CLI, or UI workflow to hide/delete/restore one published skill version. The related open incident request #600 reinforces that the missing workflow remains actionable.

## Best Possible Solution

Keep this issue open and implement a first-class per-version lifecycle workflow: add an authenticated mutation/API that resolves skill slug plus version, authorizes owners/org members/moderators, patches skillVersions.softDeletedAt with audit logging, handles latest-version and tag constraints explicitly, updates docs, adds CLI commands such as clawhub version hide/delete/unhide/undelete, and covers version list/detail/file/download/resolve behavior in focused tests. Coordinate with the related #600 incident so the feature also supports sensitive historical-version takedowns.
