## Summary

Keep this issue open. Current main can represent and enforce per-version soft deletion once `skillVersions.softDeletedAt` is set, but it still lacks an owner-facing CLI, API, or UI workflow to hide/delete/restore one published skill version while keeping the skill live. ClawHub cleanup policy also says issues should stay open; only PRs are eligible for auto-close when certainly implemented on main.

## Best Possible Solution

Keep this issue open. Implement a first-class per-version skill lifecycle path after maintainer policy review: authenticated v1 API routes and CLI/UI controls that resolve slug plus version, authorize owners/org members/staff, patch `skillVersions.softDeletedAt`, write audit logs, handle latest/tag/download/cache semantics consistently, and add focused tests and docs. Coordinate with #600 for incident handling while keeping #1370 as the separate package/plugin lifecycle track.
