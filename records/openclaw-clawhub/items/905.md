## Summary

Keep this issue open. The ClawHub cleanup policy only permits auto-closing PRs that are certainly implemented on main, and current main still lacks an owner-facing API, CLI, or UI workflow to hide/delete/restore one published skill version while keeping the skill live.

## Best Possible Solution

Keep this issue open. Implement a first-class per-version skill lifecycle path after maintainer policy review: authenticated v1 API routes and CLI/UI controls that resolve slug plus version, authorize owners/org members/staff, patch skillVersions.softDeletedAt with audit logs, handle latest/tag/download/cache behavior consistently, and add focused tests and docs. Coordinate with #600 for incident handling while keeping #1370 as the separate plugin-package lifecycle track.
