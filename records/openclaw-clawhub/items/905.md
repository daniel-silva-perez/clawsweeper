## Summary

Keep this issue open. Current main has internal version soft-delete state and public readers/downloads honor it, but the requested owner-facing workflow to hide, delete, or restore one published skill version is still missing from the CLI, HTTP API, docs, and visible management surfaces; ClawHub cleanup policy also keeps issues open for maintainer handling.

## What This Changes

Add owner-facing controls to hide/delete and restore an individual published skill version while keeping the skill slug live.

## Best Possible Solution

Define a version-scoped lifecycle policy, then implement authenticated owner/org-member/staff controls that resolve slug plus version, update skillVersions.softDeletedAt, write version-specific audit logs, and keep public version lists, detail/file/download endpoints, tag/latest behavior, cache invalidation, docs, and restore semantics consistent.
