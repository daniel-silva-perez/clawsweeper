## Summary

Keep this issue open. The repository policy for openclaw/clawhub only permits auto-closing PRs that are certainly implemented on main, and this is an open issue. A fresh read of current main also confirms the requested owner-facing per-version hide/delete/restore workflow is still missing: skill version rows have soft-delete state and public reads honor it, but the API, CLI, docs, and UI expose only whole-skill lifecycle controls.

## Best Possible Solution

Keep this issue open and implement a first-class per-version lifecycle workflow for skills. Add authenticated API and/or CLI commands that resolve slug plus version, authorize owners/org members/staff, patch skillVersions.softDeletedAt with audit logging, handle latest-version and tag constraints deliberately, preserve internal audit history, and add focused tests for version list/detail/file/download/resolve behavior. Coordinate policy with #600 while keeping #1370 as the separate package lifecycle track.
