## Summary

Keep this issue open. Current main has the internal data shape and public filtering needed to hide an individual skill version, but it still lacks the requested owner-facing workflow in the CLI, HTTP API, docs, or management UI. The ClawHub cleanup policy also says issues should stay open for maintainer handling rather than auto-close.

## What This Changes

The issue requests owner-facing controls to hide, delete, unhide, or undelete one published skill version while keeping the skill itself live.

## Best Possible Solution

Define a version-scoped lifecycle policy, then add authenticated owner/org-member/staff controls that resolve slug plus version, update skillVersions.softDeletedAt, write version-specific audit logs, and keep public version lists, detail/file/download endpoints, tag/latest behavior, cache invalidation, docs, and restore semantics consistent.
