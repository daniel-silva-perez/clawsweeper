## Summary

Keep this issue open. Current main already models version-level soft deletion and public readers/downloads honor that state, but there is still no owner-facing CLI, HTTP API, or UI workflow to hide/delete/restore one published skill version while leaving the skill live; ClawHub cleanup policy also keeps issues open for maintainer handling.

## What This Changes

Add owner-facing controls to hide, delete, unhide, or undelete a single published skill version without deleting the whole skill slug.

## Best Possible Solution

Define a version-scoped lifecycle policy, then add authenticated owner/org-member/staff controls that resolve slug plus version, patch skillVersions.softDeletedAt, write version-specific audit logs, and keep public version lists, detail/file/download endpoints, tag/latest behavior, cache invalidation, and restore semantics consistent.
