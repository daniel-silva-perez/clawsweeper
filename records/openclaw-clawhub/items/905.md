## Summary

Keep this issue open. The repository cleanup policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on main, and this item is an issue. Independently, current main still lacks the requested owner-facing API, CLI, or UI workflow to hide/delete/restore a single published skill version, even though the data model and public read paths already understand soft-deleted versions.

## Best Possible Solution

Keep this issue open. Implement a first-class per-version skill lifecycle path: authenticated v1 API routes and CLI commands that resolve slug plus version, authorize owners/org members/staff, patch skillVersions.softDeletedAt with audit logs, handle latest/tag/cache/download behavior consistently, and add focused tests and docs. Coordinate with #600 for the security-incident use case while leaving #1370 as the separate package lifecycle track.
