## Summary

Keep this issue open. The requested owner-facing per-version lifecycle workflow is still missing on current main: the data model can soft-delete a skill version and public readers already honor that state, but the exposed CLI, HTTP API, docs, and management controls remain skill-scoped. Repository policy for openclaw/clawhub also says issues should stay open for maintainer handling rather than auto-close.

## What This Changes

The issue requests owner-facing controls to hide, delete, restore, or unhide a single published skill version while leaving the skill slug live.

## Best Possible Solution

Define a version-scoped lifecycle policy, then add authenticated owner/org-member/staff controls that resolve slug plus version, update skillVersions.softDeletedAt, write version-specific audit logs, and keep public version lists, detail/file/download endpoints, tag/latest behavior, cache invalidation, docs, and restore semantics consistent.
