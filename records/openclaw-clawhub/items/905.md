## Summary

Keep this issue open. It is an open ClawHub issue, and the repository cleanup policy for this run only proposes auto-closing PRs that are already implemented on main. Current main also still lacks the requested owner-facing per-version hide/delete/restore workflow: skill version rows have a soft-delete field and public reads honor it, but the API, CLI, docs, and UI expose only whole-skill lifecycle controls.

## Best Possible Solution

Keep this issue open and implement a first-class per-version lifecycle workflow for skills: add an authenticated mutation/API that resolves slug plus version, authorizes owners/org members/moderators, patches skillVersions.softDeletedAt with audit logging, handles latest-version and tag constraints explicitly, updates CLI commands and docs, and covers version list/detail/file/download/resolve behavior in focused tests. Coordinate the policy with #600, and keep #1370 as the separate package-lifecycle track.
