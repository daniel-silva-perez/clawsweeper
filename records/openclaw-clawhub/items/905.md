## Summary

Keep this issue open. Current main can represent and honor a version-level soft delete, but there is still no owner-facing API, CLI, or UI workflow to hide/delete/restore one published skill version while leaving the skill live. #600 is adjacent operational evidence for the need, and #1370 covers plugin-package lifecycle controls rather than superseding this skill-version request.

## What This Changes

Add an owner-facing workflow to hide/delete and restore a single published skill version without deleting the whole skill.

## Best Possible Solution

Keep this open for maintainer product and policy follow-up. Define the per-version lifecycle rules first, then implement authenticated owner/org-member/staff controls that resolve slug plus version, authorize the actor, patch `skillVersions.softDeletedAt`, write audit logs, and consistently handle latest/tag/download/file/readme/cache behavior. Coordinate with #600 for operational incident needs, while keeping #1370 scoped to plugin package lifecycle controls.
