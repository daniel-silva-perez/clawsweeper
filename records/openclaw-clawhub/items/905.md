## Summary

Keep this issue open. Current main already has version-level soft-delete state and public readers honor it, but there is still no owner-facing CLI, HTTP API, or UI path to hide/delete/restore one skill version while leaving the skill live; ClawHub cleanup policy also keeps issues open rather than auto-closing them.

## What This Changes

The issue requests owner-facing CLI or API controls to hide, delete, unhide, or undelete an individual published skill version while keeping the skill itself live.

## Best Possible Solution

Define a version-scoped lifecycle policy, then add authenticated owner/org-member/staff controls that resolve slug plus version, patch `skillVersions.softDeletedAt`, write audit logs, and keep public version lists, detail/file/download endpoints, tag/latest behavior, and restore semantics consistent.
