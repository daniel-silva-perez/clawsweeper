## Summary

Keep open. The ClawHub cleanup policy only permits auto-closing pull requests, not issues, and current main still only partially addresses the report: normalized SKILL.md runtime metadata is parsed and stored, but static scanner env declaration collection still reads shallow metadata and the public v1 skill metadata response still exposes only platform fields.

## What This Changes

The issue asks ClawHub to stop false suspicious scan results when skills declare required environment variables or install dependencies in SKILL.md metadata that the registry/scanner does not consistently store or expose.

## Best Possible Solution

Keep this issue open and consolidate it with #1053. A maintainer should define the supported registry metadata contract, then route normalized SKILL.md/runtime metadata consistently through static scanning, LLM prompt inputs, UI/setup prompts, public API responses, and tests for metadata.openclaw.requires.env, envVars, dependency declarations including pip, and install specs.
