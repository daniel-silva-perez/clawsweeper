## Summary

Keep open. Repository policy keeps open ClawHub issues out of auto-close scope, and current main only partially addresses the report: metadata.openclaw is parsed, stored, documented, and used in some UI/LLM paths, but the static scanner still reads env declarations shallowly and the public v1 metadata response still omits env/install/dependency declarations.

## What This Changes

The issue asks ClawHub to stop flagging skills as suspicious when SKILL.md metadata declares env vars or install requirements that registry metadata and scanner inputs do not reflect.

## Best Possible Solution

Define the supported normalized runtime metadata contract for metadata.openclaw and aliases, then route the same normalized fields through static scanning, LLM prompt inputs, public API responses, UI/setup prompts, docs, and tests for requires.env, envVars, install specs, and dependency declarations.
