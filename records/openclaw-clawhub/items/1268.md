## Summary

Keep open. ClawHub cleanup policy keeps open issues out of auto-close scope, and the report is still substantively valid: current main parses and stores normalized runtime metadata from `metadata.openclaw`, but the static env scanner still reads declarations shallowly and the public v1 skill metadata response still omits env/install/dependency declarations.

## What This Changes

The issue asks ClawHub to make registry metadata and security scanning honor SKILL.md runtime metadata for required env vars and install specs so valid skills are not flagged as suspicious for metadata mismatches.

## Best Possible Solution

Define a supported normalized runtime metadata contract for OpenClaw skill metadata and route that same normalized shape through static scanning, LLM prompt inputs, public API responses, UI/setup prompts, docs, and tests for `requires.env`, `envVars`, install specs, and dependency declarations.
