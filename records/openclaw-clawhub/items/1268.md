## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the substantive review found only a partial main-branch resolution: publish now parses `metadata.openclaw` and the LLM prompt can include env/install data, but the static scanner and public v1 registry metadata path still do not clearly expose or consume the same normalized fields.

## Best Possible Solution

Keep this issue open and consolidate it with the related open report #1053. A maintainer should decide the supported registry contract, then make normalized SKILL.md runtime metadata the single source used by scanner summaries, static scan declared-env handling, UI/API metadata, and setup prompts, with focused tests for `metadata.openclaw.requires.env`, detailed `envVars`, and install specs.
