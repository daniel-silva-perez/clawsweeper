## Summary

Keep open. The repository policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on main, and this item is an issue. Substantively, current main has partial support: publish parsing now reads metadata.openclaw and stores parsed clawdis metadata, but the public skill API still exposes only os/systems from that metadata, and the static scanner's declared-env collection does not inspect nested metadata.openclaw.requires.env directly. That leaves the reporter-visible registry/scanner mismatch plausible until maintainers finish the scanner/API/backfill path and verify the affected production skill has been republished or rescanned.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The likely fix is to make the static scanner consume the normalized parsed clawdis metadata or explicitly traverse metadata.openclaw, expose the intended env/install registry metadata where scanners and users expect it, add regression tests for metadata.openclaw.requires.env and install specs, then backfill/republish/rescan the affected skill version before closing.
