## Summary

Keep open. Repository policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on current main, and this item is an issue. Substantively, current main has internal parsing and persistence for metadata.openclaw env/install data, but there are still unresolved reporter-visible gaps: static scan collection does not descend into nested metadata.openclaw when collecting declared env vars, public API v1 only exposes os/systems metadata, and the checkout does not prove the affected published skill was backfilled, republished, or rescanned.

## Best Possible Solution

Keep this issue open. The best maintainer path is to make the static scanner collect declared env vars from normalized parsed.clawdis or by descending into metadata.openclaw/clawdis/clawdbot, decide whether public API v1 should expose requires.env/envVars/install/dependencies, then backfill or republish and rescan the affected skill version before closing.
