## Summary

Keep open. This is an open ClawHub issue, and the repository policy for this cleanup lane keeps issues out of auto-close scope. The report also remains substantively valid: current main documents, parses, stores, and uses some normalized runtime metadata, but the static scanner still reads env declarations shallowly and the public v1 metadata response still exposes only OS/Nix fields rather than env/install/dependency declarations.

## What This Changes

The issue asks ClawHub to stop flagging skills as suspicious when required environment variables or install steps are declared in SKILL.md metadata but absent from registry-level metadata.

## Best Possible Solution

Define one supported normalized runtime metadata contract for metadata.openclaw and aliases, then route those fields consistently through static scanning, LLM prompt inputs, public API responses, UI/setup prompts, docs, and tests for requires.env, envVars, install specs, and dependency declarations.
