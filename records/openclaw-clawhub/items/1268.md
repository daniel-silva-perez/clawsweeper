## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and current main still only partially addresses the report: SKILL.md runtime metadata is parsed and stored, but the static scanner declaration collection still does not normalize nested metadata.openclaw metadata before comparing env usage, and the public v1 metadata response still exposes only platform fields.

## What This Changes

The issue requests that ClawHub registry and security metadata for published skills include SKILL.md or metadata.json env/install declarations so legitimate requirements are not flagged as suspicious.

## Best Possible Solution

Keep this issue open and consolidate it with #1053. A maintainer should define the supported registry metadata contract, then route normalized SKILL.md/runtime metadata consistently through static scanning, LLM prompt inputs, UI/setup prompts, public API responses, docs, and tests for metadata.openclaw.requires.env, envVars, dependencies including pip, and install specs. The rollout should also decide whether existing flagged versions need a rescan or moderation cleanup path.
