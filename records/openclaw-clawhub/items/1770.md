## Summary

Keep open. The repository policy for openclaw/clawhub says issues should remain open, and the current main branch still treats the requested slug cleanup as production admin handling rather than a completed self-service flow. Soft-deleted skill rows can still occupy the global slug, rename is still blocked by an existing different skill row, and the hard-delete/reclaim paths are admin-only.

## Best Possible Solution

Keep this issue open for maintainer support. A maintainer/admin should verify that the reporter controls both the old personal publisher and the target org publisher, then use the production admin path to remove or reclaim the old soft-deleted slug so creativault/cv-creator-scraper can rename to creator-scraper-cv. If this pattern recurs, add a product follow-up for a self-service transfer/reclaim flow when the same authenticated user controls both publishers.
