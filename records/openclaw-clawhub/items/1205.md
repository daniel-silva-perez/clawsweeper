## Summary

Keep this issue open. Although the reporter confirmed the immediate production publish recovered after an apparent backend hotfix, current main still has the same Convex failure shape: the owner-publisher package and skill digest rebuild helpers can call `.paginate()` repeatedly inside one internal mutation, and the internal mutations do not accept cursor continuation state. The related publish/import pagination reports remain open, so this should be handled as one focused backend fix rather than closed as resolved.

## What This Changes

The issue reports that `clawhub publish` can fail when owner-publisher digest sync calls Convex pagination more than once inside a single mutation.

## Best Possible Solution

Land one focused Convex backend fix: convert the owner-publisher package and skill digest internal mutations to process one page per invocation, accept cursor and batch-size arguments, and self-schedule continuation when `isDone` is false. Preserve the publisher-trigger scheduling split, add multi-page tests for both helpers, then consolidate the related publish/import pagination reports after the fix lands.

In `convex/functions.ts`, change `syncPackageSearchDigestsForOwnerPublisherIdInternal` and `syncSkillSearchDigestsForOwnerPublisherIdInternal` so each invocation performs at most one `.paginate()` call, accepts cursor and optional batch-size args, processes that page, and uses `ctx.scheduler.runAfter(0, ...)` to schedule the same internal mutation with the continuation cursor when `isDone` is false. Preserve the existing publisher-trigger split that schedules package and skill digest rebuilds separately. Keep indexed queries; do not replace pagination with unbounded `.collect()`. Add focused multi-page tests in `convex/functions.test.ts` for both package and skill digest helpers proving the first invocation schedules continuation and does not call paginate more than once. Update generated Convex files only if required by codegen. Do not change unrelated publish, auth, package parsing, or UI behavior.
