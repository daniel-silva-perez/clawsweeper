## Summary

Keep open. The immediate production publish path may have recovered, but current main still contains the backend failure shape behind this report: the owner-publisher package and skill digest sync helpers can call `.paginate()` repeatedly in one internal mutation, and the scheduled internal mutations do not accept cursor continuation state. Related publish/import pagination reports remain open, so this should be handled as one focused backend fix rather than swept closed.

## What This Changes

`clawhub publish` should not fail when owner-publisher digest syncing encounters Convex's single-paginated-query-per-function limit.

## Best Possible Solution

Implement cursor-based, one-page-per-invocation owner-publisher digest rebuilds for both package and skill search digests. Keep the publisher-trigger scheduling split, pass cursor and batch-size arguments through the internal mutations, self-schedule continuation when `isDone` is false, and add tests that simulate multi-page package and skill rebuilds.

## Repair Work Prompt

Fix the owner-publisher digest pagination failure reported by the clustered publish/import issues. In `convex/functions.ts`, change the owner-publisher package and skill digest internal mutations so each invocation performs at most one `.paginate()` call, accepts cursor and optional batch-size arguments, processes that page, and uses `ctx.scheduler.runAfter(0, ...)` to schedule continuation when `isDone` is false. Preserve the existing publisher-trigger scheduling split and do not replace pagination with unbounded `.collect()`. Cover both `syncPackageSearchDigestsForOwnerPublisherId` and `syncSkillSearchDigestsForOwnerPublisherId`; keep owner-user digest behavior out of scope unless tests prove it shares the same transaction-limit bug. Add focused multi-page tests in `convex/functions.test.ts` for package and skill digest rebuilds that assert continuation scheduling and one page per invocation. Add an Unreleased changelog entry for the backend publish pagination fix. Regenerate Convex generated files only if the function references/types require it, and do not change CLI command behavior.
