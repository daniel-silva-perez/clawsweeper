## Repair Work Prompt

Fix the owner-publisher digest rebuild pagination bug reported by this issue and related publish/import reports. In `convex/functions.ts`, change both `syncPackageSearchDigestsForOwnerPublisherId` and `syncSkillSearchDigestsForOwnerPublisherId` so each internal mutation processes at most one `.paginate()` page per invocation. Add validated cursor and batch-size arguments to the internal mutations, pass initial state from `scheduleOwnerPublisherDigestSync`, and self-schedule continuation when `isDone` is false. Preserve the current split between package and skill background jobs and do not move this back into the publisher trigger transaction. Add focused tests in `convex/functions.test.ts` that simulate multi-page package and skill rebuilds and assert continuation scheduling. Do not replace pagination with unbounded `.collect()`. Mention the related publish/import reports in the PR summary, but do not edit unrelated CLI or publisher authorization behavior.
