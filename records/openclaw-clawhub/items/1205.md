## Summary

Keep this issue open. The reporter confirmed their immediate publish recovered after a backend hotfix, but current main still contains the same Convex failure shape: the owner-publisher package and skill digest rebuild helpers can call `.paginate()` repeatedly inside one internal mutation invocation. ClawHub cleanup policy also keeps open issues open unless they are PRs already implemented on main.

## Best Possible Solution

Keep this issue open and land one focused Convex backend fix: convert the owner-publisher package and skill digest internal mutations to process one page per invocation, accept cursor and batch-size arguments, and self-schedule continuation when `isDone` is false. Preserve the existing publisher-trigger scheduling split, add multi-page tests for both helpers, then deploy and consolidate the duplicate publish/import pagination reports.
