## Summary

Keep open. Repository policy does not permit auto-closing ClawHub issues, and current main still leaves a real account/moderation recovery gap: banned/deactivated users can be blocked at auth, and publisher moderation holds can hide all owned skills and future publishes without a current-main API/UI to clear the hold.

## What This Changes

Requests unblocking PHY041's ClawHub account, restoring or allowing republication of roughly 42 skills, and documenting safe batch publishing rates after a suspected ClawHavoc moderation false positive.

## Best Possible Solution

Keep this issue open for maintainer/security triage. Staff should inspect production records for PHY041, determine whether the account is banned, deactivated, under publisher moderation, or auth-linked incorrectly, restore access through the existing admin unban path if it is a deletedAt-based ban, and review or land an audited moderation-hold recovery path such as #1133 for requiresModerationAt cases. A maintainer response should also document safe batch publishing rates and a clear appeal path for false positives.
