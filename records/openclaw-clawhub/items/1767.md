## Summary

Keep open. Repository policy for openclaw/clawhub only allows auto-closing pull requests that are certainly implemented on current main, and this is an issue. Substantively, current main still has publish dry-run, hosted publish-time static scans, and published-release rescan commands, but no local plugin lint/scanner CLI that validates manifests, SKILL.md metadata, env declarations, and code findings before publish with severity-coded JSON and exit behavior.

## What This Changes

Request a local ClawHub/OpenClaw plugin lint or scanner CLI that validates plugin manifests, SKILL.md metadata, environment declarations, and static code risks before publishing, with severity-coded output and CI-friendly JSON.

## Best Possible Solution

Keep this issue open for maintainer-scoped design. The best path is a local scanner/lint command in the ClawHub CLI that reuses shared package schema validation and static moderation logic, defines a stable severity and exit-code contract, supports --json, and documents which findings block publish versus remain advisory.
