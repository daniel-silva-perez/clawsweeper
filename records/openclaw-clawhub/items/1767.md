## Summary

Keep open. ClawHub policy keeps issues open, and the feature gap is still real: current main has package publish dry-run, server-side publish scanning, and package rescan requests, but no local plugin lint/scanner command that reports manifest, SKILL.md, and code findings before publishing. The @mem0/openclaw-mem0 examples in the report are concrete and align with checks that currently live partly in hosted/static moderation paths.

## What This Changes

Add a local plugin scanner package or ClawHub CLI command that validates plugin manifests, SKILL.md metadata, source-code heuristics, JSON output, severity levels, and nonzero exits before publish.

## Best Possible Solution

Define a maintainer-approved local ClawHub scanner that reuses shared manifest/schema validation and deterministic static moderation logic, adds missing plugin/SKILL metadata checks, supports human and JSON output with stable severity codes, and documents how local results relate to hosted publish-time scanning.
