## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only allows auto-closing ClawHub pull requests that are certainly implemented on current main. Substantively, current main still has package publish dry-run and server-side static scan persistence, but no local plugin scanner/lint CLI that validates manifests/SKILL.md/code before publish with structured findings and scanner-style exit behavior.

## Best Possible Solution

Keep this issue open. The best path is a maintainer-scoped local scanner, likely under the ClawHub package CLI, that reuses shared package schema validation and static moderation logic, validates local package manifests and SKILL.md metadata, supports --json, emits severity-coded findings, and exits nonzero for blocking publish errors without changing hosted moderation semantics.
