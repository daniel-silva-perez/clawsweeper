## Summary

Keep open. Repository policy keeps ClawHub issues open, and the requested local pre-publish plugin scanner is still missing on current main: the CLI has package publish dry-run and hosted rescan support, but no local lint/scan command that reports manifest, SKILL.md, and source-code findings before publishing.

## What This Changes

The issue requests a local plugin scanner CLI or publish dry-run mode that validates plugin manifests, SKILL.md metadata, code security heuristics, JSON output, severities, and exit codes before publication.

## Best Possible Solution

Add a maintainer-scoped ClawHub CLI command or package that reuses shared manifest validation and deterministic static moderation helpers, reports severity-coded findings in human and JSON modes, exits nonzero for blocking findings, and documents parity or known gaps with hosted publish-time scanning.
