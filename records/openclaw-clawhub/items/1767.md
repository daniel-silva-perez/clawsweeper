## Summary

Keep open. This is a ClawHub issue, and the repository-specific policy only permits auto-closing ClawHub pull requests that are certainly implemented on main. Substantively, current main still lacks the requested local plugin scanner/linter: the CLI has package publish dry-run output and limited publish-plan validation, while ClawHub's static scan runs in the server publish path rather than as a local pre-publish command.

## Best Possible Solution

Keep this issue open. The best implementation path is to add a local `clawhub package lint <source>` command, or an explicit stronger pre-publish dry-run mode, under `packages/clawhub` that reuses shared schema and scan logic where practical. It should report severity-coded manifest, SKILL.md, config schema, env metadata, provider/UI hint, and code/security findings, support `--json`, and exit non-zero for blocking errors.
