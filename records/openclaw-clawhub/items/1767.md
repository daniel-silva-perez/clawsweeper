## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main also still lacks the requested local plugin/package scanner CLI; it has publish dry-run planning and server-side static scanning after upload, but not a local pre-publish lint surface with severity output, JSON findings, and nonzero validation behavior.

## Best Possible Solution

Keep this issue open. The best path is to add a local ClawHub CLI command such as `clawhub package lint <source>` or a true `package publish --dry-run --pre-publish` mode that reuses shared package schema and static-scan logic, supports `--json`, emits severity-coded findings, and exits nonzero for blocking publish errors.
