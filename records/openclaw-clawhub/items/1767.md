## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on main. Current main still does not provide the requested local plugin scanner/linter CLI; `package publish --dry-run` only prints a publish plan, while richer static scan logic remains in the backend publish path.

## Best Possible Solution

Keep this issue open. The best path is to add a local ClawHub CLI command such as `clawhub package lint <source>` or an explicit pre-publish/dry-run scanner mode that reuses shared package schema and static scan logic, supports `--json`, emits severity-coded findings, and exits non-zero for blocking publish errors.
