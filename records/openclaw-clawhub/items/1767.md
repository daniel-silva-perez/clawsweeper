## Summary

Keep open. This is a ClawHub issue, and the cleanup policy only permits auto-closing ClawHub pull requests that current main definitely implements. Current main still lacks the requested local plugin scanner CLI: it has package publish dry-run planning and server-side static scan persistence, but no local lint/pre-publish command with structured findings, severity handling, and scanner-style exit behavior.

## Best Possible Solution

Keep this issue open. The best path is a maintainer-scoped local scanner, probably `clawhub package lint <source>` or an explicit pre-publish mode, that reuses shared package schema and static moderation logic, validates local manifests/SKILL.md metadata, supports `--json`, emits severity-coded findings, and exits nonzero for blocking publish errors without changing hosted moderation semantics.
