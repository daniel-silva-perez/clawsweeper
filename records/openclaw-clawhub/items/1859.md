## Summary

Keep this draft PR open. Current main has partial optional-env support in the shared schema, parser, UI, and static moderation path, but it does not fully implement the PR’s intended cleanup: public skill-format docs still describe only `requires.env`, the CLI-local schema copy still omits `envVars`, and the security evaluator requirements block still does not enumerate optional env declarations.

## Best Possible Solution

Keep this draft PR open as the implementation candidate for the linked optional-env issue. The best path is to land one consistent optional environment variable policy across `docs/skill-format.md`, the CLI-local schema in `packages/clawhub`, security prompt text, scanner behavior, and focused regression tests, using `metadata.openclaw.envVars` entries with `required: false` unless maintainers choose a different canonical field.
