## Summary

Keep this PR open. Current main only partially supports optional skill env metadata and does not include the PR's docs, CLI schema copy, security prompt, and regression-test alignment, so it is not obsolete or already implemented on main.

## What This Changes

The PR documents optional skill environment variables, adds `envVars` metadata support to the CLI schema copy, updates security evaluator wording/user-message formatting, and adds schema/moderation regression tests.

## Best Possible Solution

Land or replace this PR after maintainer review so the public skill-format docs, CLI schema copy, scanner/security prompt, and tests consistently describe `metadata.openclaw.envVars` entries with `required: false` as the supported optional environment-variable declaration, unless maintainers choose a different spelling.

## Review Findings

Full review comments:

- **[P2] Cover the documented nested envVars shape:** `convex/lib/moderationEngine.test.ts:124-125`
  - body: The new regression passes `metadata.envVars` at the top level, but the docs added in this PR tell authors to declare optional variables under `metadata.openclaw.envVars`. The publish path passes raw frontmatter metadata to the static scanner, and `collectDeclaredEnvNames` only reads top-level `envVars`/`requires.env`, so a skill following the new docs can still be flagged for undeclared env access. Please update the scanner and regression to cover `metadata: { openclaw: { envVars: [...] } }`.
