## Summary

Keep open. ClawHub policy keeps issues open unless they are PRs already implemented on main, and current main still has a credible clean-install fingerprint mismatch path: update hashes extracted local files, asks the registry to resolve that hash, and refuses with the reported message when no match is returned.

## What This Changes

The issue requests that freshly installed or force-updated skills pass `clawhub update` integrity checks instead of being reported as `local changes (no match)`.

## Best Possible Solution

Define one canonical skill-content fingerprint contract shared by publish, download, install, update, sync, and resolve. Fresh installs and force-updates should immediately resolve to their installed version, while real local edits still block non-forced updates.
