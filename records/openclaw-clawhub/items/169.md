## Summary

Keep open. Current main still has the clean-install/update fingerprint mismatch path: the CLI hashes local extracted text files, including root JSON files such as generated `_meta.json`, then asks `/api/v1/resolve` for a match; the backend publish/resolve fingerprints are based on stored version files and do not add the generated download metadata. The repository policy also says ClawHub issues should remain open rather than be auto-closed.

## Best Possible Solution

Keep this issue open and land one focused fix PR for the CLI/backend fingerprint contract. The best fix is to define one canonical fingerprint for installed skill contents: either consistently exclude generated root `_meta.json` from local/download fingerprints, or include generated metadata consistently in publish-time stored fingerprints, resolve, install, update, and sync. Add regression coverage proving a fresh install or force-update followed immediately by update is treated as clean, while a real user edit is still detected.
