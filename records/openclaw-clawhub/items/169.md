## Repair Work Prompt

Fix the ClawHub CLI/backend fingerprint contract so a freshly installed or immediately force-updated skill no longer reports `local changes (no match)` on `clawhub update`, while genuine local edits still block non-forced updates. Treat related reports #178 and #937 as the same bug cluster. Focus on a shared canonical content fingerprint across publish, download, install, update, sync, and resolve. Likely surfaces are `packages/clawhub/src/skills.ts`, `packages/clawhub/src/cli/commands/skills.ts`, `packages/clawhub/src/cli/commands/sync.ts`, `convex/downloads.ts`, `convex/lib/skillZip.ts`, `convex/skills.ts`, and their tests. Avoid broad ignores for all JSON or generated-looking files; either exclude only generated root `_meta.json` from the installed-content fingerprint contract or stop installing download-only metadata as skill content. Add regression tests proving clean install/force-update resolves, and local edits still produce the warning. Update docs only if user-facing behavior or troubleshooting text changes; do not make unrelated release or formatting changes.
