## Summary

This issue should stay open. The ClawHub cleanup policy for this review says ClawHub issues are not auto-closed, and current main still appears to preserve the reported clean-install/update false positive through a generated `_meta.json` fingerprint mismatch. Related open reports #178 and #937 describe the same user-visible behavior.

## Best Possible Solution

Keep this issue open until the CLI/backend fingerprint contract is fixed. The likely minimal fix is to exclude generated `_meta.json` from local skill fingerprinting, or to include generated download metadata consistently in server-side stored fingerprints across publish, download, install, and resolve. Add a regression test that installs or hashes a downloaded skill zip containing `_meta.json` and then verifies update resolves it as clean instead of reporting `local changes (no match)`. Maintainers may also choose one canonical issue among #178, #937, and this issue after the fix lands.
