## Summary

Keep open. Repository policy keeps ClawHub issues open, and current main still shows a credible clean-install/update fingerprint mismatch: update resolves a local fingerprint, downloaded skill zips can add generated root `_meta.json`, and the resolver compares against published version files/fingerprints that do not include that generated metadata.

## What This Changes

Make `clawhub update` recognize freshly installed or force-updated unmodified skills instead of reporting `local changes (no match)`.

## Best Possible Solution

Keep this issue open and land one focused PR that defines a canonical skill-content fingerprint across install, update, sync, downloads, and resolve. The likely fix is to ignore only generated root `_meta.json` in local/downloaded-zip fingerprint calculations, or stop generated metadata from entering installed skill contents, while preserving detection for real user-authored JSON edits. Add regressions for fresh install and force-update followed immediately by update, plus a real local edit still being detected, and coordinate with #178 and #937.
