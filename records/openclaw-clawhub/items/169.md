## Summary

Keep this issue open. ClawHub cleanup policy only permits auto-closing PRs as implemented, and current main still has a credible clean-install/update fingerprint mismatch path matching the report: the CLI refuses when resolve returns no match, while installed download contents can include generated metadata that publish-time fingerprints do not include.

## Best Possible Solution

Fix the CLI/backend fingerprint contract so freshly installed and force-updated skills resolve as clean. A conservative implementation would either exclude generated `_meta.json` from local and zip fingerprints or include it consistently in publish-time stored fingerprints, resolve, install, and update. Add a regression test for install-or-force-update followed immediately by update, then consolidate this issue with #178 and #937 once the fix is merged.
