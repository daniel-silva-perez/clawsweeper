## Summary

Keep open. Current main still has an actionable clean-install/update fingerprint mismatch path: downloaded skill archives include generated root `_meta.json`, local hashing includes root JSON files, but publish-time and resolve-time fingerprints are computed from stored version files that do not include that generated metadata. The ClawHub policy also keeps issues open rather than auto-closing them.

## Best Possible Solution

Keep this issue open and land one focused fix PR for the CLI/backend fingerprint contract. The best fix is either to exclude generated root `_meta.json` consistently from local and downloaded-zip fingerprints, or to include generated metadata consistently in publish-time stored fingerprints, resolve, install, update, and sync. Add a regression test proving fresh install or force-update followed immediately by update resolves as clean, then close this issue together with #178 and #937 after the fix ships.
