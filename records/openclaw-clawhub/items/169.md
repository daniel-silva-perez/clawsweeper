## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has an actionable clean-install/update fingerprint mismatch path: update refuses when resolve returns no match, while downloaded skill archives can include generated `_meta.json` that publish-time and resolve-time fingerprints do not include.

## Best Possible Solution

Keep this issue open. Fix the CLI/backend fingerprint contract so a freshly installed or force-updated skill resolves as clean, either by excluding generated `_meta.json` from local and zip fingerprints or by including generated metadata consistently in publish-time stored fingerprints, resolve, install, and update. Add a regression test for install or force-update immediately followed by update, then consolidate with #178 and #937 after merge.
