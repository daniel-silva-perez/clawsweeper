## Summary

Keep this issue open. Repository policy for openclaw/clawhub keeps issues open, and current main still contains the reported clean-install/update failure path: downloaded skills can include generated `_meta.json`, local fingerprinting includes root JSON files, while publish/resolve fingerprints are based on stored published files. Related open reports #178 and #937 track the same fingerprint mismatch symptoms.

## Best Possible Solution

Keep this issue open for maintainer follow-up. Fix the CLI/backend fingerprint contract by either excluding generated `_meta.json` from local and zip fingerprints, or including generated download metadata consistently in publish-time stored fingerprints, resolve, install, and update. Add a regression test for update after a fresh install, then choose one canonical thread among this issue, #178, and #937 after the fix lands.
