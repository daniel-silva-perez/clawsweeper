## Summary

This issue should stay open. The ClawHub cleanup policy for this run says issues are not auto-closed, and current main still appears to preserve the reported clean-install/update false positive: skill downloads include generated `_meta.json`, install extracts it, local update hashing includes JSON files, while backend version fingerprints are computed from published version files rather than generated download metadata. Related reports #178 and #937 were also kept open for the same remaining behavior.

## Best Possible Solution

Fix the CLI/backend fingerprint contract so generated download metadata is handled consistently. The likely minimal path is to exclude generated `_meta.json` from local skill fingerprinting, or alternatively make it part of stored server fingerprints in the same way for publish, download, install, and resolve. Add a regression test that installs a downloaded skill zip containing `_meta.json` and immediately runs update without getting `local changes (no match)`.
