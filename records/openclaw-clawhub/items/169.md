## Summary

Keep this issue open. ClawHub cleanup policy says issues stay open, and current main still has a credible clean-install/update fingerprint mismatch path: update hashes installed local files and refuses when resolve returns no match, while downloaded skill zips include generated `_meta.json` that publish-time fingerprints are not clearly storing or resolving against consistently.

## Best Possible Solution

Fix the CLI/backend fingerprint contract so a freshly installed or force-updated skill resolves as clean. A conservative path is to exclude generated `_meta.json` from local and zip fingerprints, or include generated download metadata consistently in publish-time stored fingerprints, resolve, install, and update. Add a regression test for update immediately after fresh install or force update, then consolidate this issue with #178 and #937 after the fix is merged.
