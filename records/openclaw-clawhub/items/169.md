## Summary

Keep open. Repository policy keeps ClawHub issues open, and current main still has a credible fingerprint mismatch path: downloaded installs can contain generated root `_meta.json`, local update hashing includes root JSON files, and server resolve matches against published version file fingerprints rather than download-generated metadata.

## What This Changes

Fix `clawhub update` so freshly installed or force-updated skills match registry fingerprints instead of reporting `local changes (no match)`.

## Best Possible Solution

Define one canonical skill-content fingerprint contract shared by publish, download, install, update, sync, and resolve. Fresh installs and force-updates should immediately resolve to the installed version, while real local edits still block non-forced updates.
