## Summary

Keep open. This is an issue, and the ClawHub cleanup policy for this review only allows auto-closing PRs that are certainly implemented on current main. The current source also confirms the requested Mapbox package-name transfer/removal remains a maintainer/admin action: package names are globally keyed by normalizedName, cross-publisher reuse is rejected, package soft-delete exists, and the reclaim/transfer tooling currently covers skill slugs rather than package ownership transfer.

## Best Possible Solution

Keep this issue open for maintainer handling. A maintainer should verify the reporter’s authority for the Mapbox GitHub organization or a mapbox.com identity, inspect the current production owner of the `mapbox` package, then either remove/soft-delete the squatted package so Mapbox can publish, perform an audited production transfer, or add a narrowly scoped package reclaim/transfer workflow. Longer term, the package/publisher namespace docs should explicitly cover official-publisher and trademark reclaim cases.
