## Summary

Keep open. This is a ClawHub issue, and the repository policy only permits auto-closing PRs when current main definitely implements the intended change. Substantively, current main still leaves the Mapbox package-name transfer/removal as maintainer/admin work: packages are looked up by global normalizedName, cross-publisher reuse is rejected, and the available reclaim tooling is for skill slugs rather than package ownership transfer.

## Best Possible Solution

Keep this issue open for explicit maintainer handling. A maintainer should verify the reporter's Mapbox authority, inspect the production owner of the `mapbox` package, then either perform an audited transfer/hard removal or add a narrow package reclaim workflow that covers trademark and official-publisher cases. Longer term, package reserved-name and transfer policy should be documented alongside the publisher namespace work.
