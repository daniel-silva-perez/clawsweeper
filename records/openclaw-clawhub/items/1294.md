## Summary

Keep this issue open. The request is a concrete trademark/name-ownership claim from an external publisher, and the ClawHub cleanup policy for this repository does not auto-close issues. Current main still has a global package-name collision model and no source-backed arbitrary package-name transfer/reclaim flow for a verified external organization, though admins do have adjacent tools such as package soft-delete and org publisher ensure/migration paths.

## Best Possible Solution

Keep this issue open for maintainer handling. A maintainer should verify the reporter's authority for Mapbox through the linked Mapbox repository, GitHub organization, or a mapbox.com identity; inspect the production owner of the mapbox package; then either transfer ownership to a verified Mapbox publisher, soft-delete/remove the squatted package so Mapbox can publish, or add a narrowly scoped audited package-name reclaim path. Longer term, document official-publisher and trademark/name-reclaim policy alongside the package publisher namespace migration.
