## Summary

Keep this issue open. Repository policy for openclaw/clawhub says issues should remain open unless a strong non-implemented close reason applies, and current main still intentionally blocks reauth for deleted/deactivated users rather than clearing or reusing the GitHub OAuth binding. The provided timeline also shows several related open reports for the same post-deletion GitHub sign-in loop, so this remains an active account lifecycle/support problem.

## Best Possible Solution

Maintainers should decide the desired product policy for deleted-account reauth. If same-GitHub re-signup should be allowed, add a safe unlink/retire path for the GitHub authAccounts binding during account deletion, provide an admin/support mutation for existing stuck accounts, and cover the flow with auth tests. If deletion must remain irreversible, keep the current code path but handle affected users through support/admin tooling and make the UI/docs explicit enough that the account lifecycle is understood before deletion.
