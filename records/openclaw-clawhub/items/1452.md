## Summary

Keep this issue open. Current main still intentionally treats user-initiated deletion as irreversible: OAuth sign-in for a deactivated/deleted user is blocked, legacy deleted users are migrated to deactivated state and blocked, and the admin unban path refuses permanently deleted accounts. The request also matches a larger open account-recovery cluster, and the ClawHub cleanup policy says open issues should stay open unless maintainers handle them manually.

## What This Changes

Allow a user who deleted their ClawHub account to sign in again with the same GitHub identity by clearing the OAuth binding or restoring/unblocking the deleted account.

## Best Possible Solution

Maintainers should decide whether ClawHub account deletion remains irreversible. If re-registration should be supported, add an explicit, audited recovery/unlink flow that clears or rekeys the GitHub auth binding only for user-initiated deletions, keeps banned users blocked, updates docs/UI copy, and adds Convex tests for same-GitHub re-signup and admin/operator recovery.
