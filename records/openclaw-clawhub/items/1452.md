## Summary

Keep this issue open. Current main does not implement the requested GitHub OAuth unlink, account restore, or same-GitHub re-signup path after user-initiated deletion. The issue is also a ClawHub issue rather than a PR, and the repository policy says issues should remain open unless a strong non-implemented close reason applies.

## Best Possible Solution

Maintainers should decide the desired deletion policy. If same-GitHub re-signup should be allowed, update the deletion/auth flow to safely unlink or retire the GitHub authAccounts binding, add support/admin tooling for already-stuck accounts, and cover the path with Convex auth tests. If deletion must remain irreversible, keep the code behavior but handle this as an account-support case and make the UI/docs explicit before manually closing.
