## Summary

Keep open. The repository cleanup policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on main, and this is an issue. Current main still blocks deleted/deactivated GitHub OAuth reauth, documents user deletion as irreversible, and has no restore or OAuth-unlink path for this support case, so this needs maintainer product/security handling rather than cleanup closure.

## What This Changes

The reporter asks ClawHub to remove the stale GitHub OAuth binding or restore a deleted account so the same GitHub account can sign in again.

## Best Possible Solution

Define an explicit, audited account-recovery or re-registration flow that can unlink or rekey only user-initiated deletions, keeps banned and abuse-disabled accounts blocked, updates UI/docs to match the chosen policy, and adds Convex tests for same-GitHub re-signup, operator recovery, and banned-account rejection.
