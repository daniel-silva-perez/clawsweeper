## Summary

Keep this issue open. Current main still intentionally blocks OAuth sign-in for deleted/deactivated users, account deletion does not clear the GitHub auth binding, and the request belongs to an unresolved account-recovery cluster that needs maintainer product/security direction rather than cleanup closure.

## What This Changes

The issue asks maintainers to remove or restore the stale GitHub OAuth binding for a deleted ClawHub account so the same GitHub account can sign in again.

## Best Possible Solution

Maintainers should decide whether user-initiated deletion remains irreversible. If re-registration should be supported, add an explicit audited recovery or unlink flow that clears or rekeys the GitHub identity only for eligible self-deleted accounts, keeps banned accounts blocked, and updates docs, UI copy, and Convex tests.
