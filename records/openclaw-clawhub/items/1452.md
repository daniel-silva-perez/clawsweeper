## Summary

Keep open. This is an issue in openclaw/clawhub, where the cleanup policy only permits auto-closing pull requests that are certainly implemented on main; independently, current main still intentionally blocks OAuth sign-in for deleted/deactivated accounts and has no restore or GitHub-binding unlink flow for this support case.

## What This Changes

The issue asks ClawHub maintainers to clear a stale GitHub OAuth binding or restore a deleted account so the same GitHub identity can sign in again.

## Best Possible Solution

Define an explicit, audited account recovery or re-registration policy: allow unlink/rekey only for user-initiated deletions if maintainers choose to support it, keep banned and abuse-disabled users blocked, update UI/docs to match the policy, and add Convex tests for same-GitHub re-signup, operator recovery, and banned-account rejection.
