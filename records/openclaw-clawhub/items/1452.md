## Summary

Keep this issue open. Current main still intentionally rejects GitHub OAuth re-entry for deleted/deactivated users, self-service deletion does not unlink the Convex Auth GitHub account binding, and the open related report cluster indicates this remains a live account-recovery/support problem. ClawHub cleanup policy also says issues should remain open unless there is a strong close reason; this one still needs maintainer product/security judgment.

## Best Possible Solution

Keep this open for maintainer product/support follow-up. Maintainers should decide whether same-GitHub re-signup after self-service deletion is allowed; if yes, add a guarded authAccounts unlink, retired-binding, or restoration path plus admin/support tooling for already-stuck accounts, with explicit protection for banned/moderated users. If deletion remains irreversible, add clearer user-facing support guidance and operator tooling instead of leaving users in a confusing OAuth loop/block.
