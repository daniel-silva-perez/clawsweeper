## Summary

Keep this issue open. It is an open ClawHub issue, and the repository cleanup policy says ClawHub issues should remain open unless a strong non-implemented close reason applies. Current main still preserves the reported behavior: self-service deletion deactivates/purges the user, OAuth completion rejects deleted/deactivated users, there is no current authAccounts unlink or restore path, and docs/UI/admin behavior describe user-initiated deletion as irreversible.

## Best Possible Solution

Keep this open for maintainer product/support follow-up. Maintainers should decide whether same-GitHub re-signup after self-service deletion is allowed. If yes, add a guarded authAccounts unlink/retired-binding or restoration path, plus an admin/support repair mutation for already-stuck accounts, and cover the OAuth/deleted-account flow with tests. If deletion must remain irreversible, keep the blocking behavior but add clear support tooling and user-facing guidance for affected accounts.
