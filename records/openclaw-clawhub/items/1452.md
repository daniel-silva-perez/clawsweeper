## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing pull requests that current main certainly implements. Current main still rejects deleted/deactivated GitHub OAuth reauth, documents user deletion as irreversible, and has no restore or OAuth-binding unlink flow for this support case.

## What This Changes

The issue asks ClawHub maintainers to remove a stale GitHub OAuth binding or restore a deleted account so the same GitHub account can sign in again.

## Best Possible Solution

Define and ship an explicit, audited account lifecycle policy: either keep deletion irreversible with a documented support-run rekey/unlink process, or add a controlled recovery/re-registration flow that distinguishes self-deletion from moderation bans and updates Convex auth code, docs, UI copy, and tests together.
