## Summary

Keep this issue open. It is a ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub PRs that are certainly implemented on main. Current main also still matches the reported post-deletion GitHub OAuth failure mode: account deletion marks the user permanently deactivated, OAuth completion rejects deleted/deactivated users, auth account bindings remain separate, and tests/docs codify the blocked reauth behavior.

## Best Possible Solution

Keep this issue open for maintainer product/support follow-up. Maintainers should decide whether same-GitHub re-signup after self-service deletion should be allowed. If yes, add a safe authAccounts unlink or retired-binding path during account deletion, provide an admin/support repair mutation for already-stuck accounts, and cover the OAuth/deleted-account flow with tests. If deletion must remain irreversible, keep the blocking code but add clear support tooling and user-facing guidance for affected accounts.
