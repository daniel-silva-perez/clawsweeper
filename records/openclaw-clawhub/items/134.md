## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a credible update-disappearance path: ordinary non-trusted skill updates can write a hidden pending-scan status onto the existing skill row, while public detail, search, catalog, and API paths filter hidden skills out.

## What This Changes

The issue reports that publishing an updated skill returns success or a server error while the skill disappears from inspect, search, and public listings.

## Best Possible Solution

Implement an explicit update-moderation model. If clean active skills should remain discoverable, keep the previous approved public version visible while the new upload scans, and make owner/status UX plus API/search responses reflect that pending state; if full quarantine is intentional, document and align that behavior across CLI, web, API, dashboard, and recovery flows.
