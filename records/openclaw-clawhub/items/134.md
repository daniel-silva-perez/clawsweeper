## Summary

Keep this issue open. The repository policy for this ClawHub cleanup pass keeps issues open, and current main still appears to have the reported active-skill update disappearance path: non-trusted clean updates can patch an existing skill back to hidden pending scan, while public detail, search, API, and CLI inspect paths filter hidden skills out or return not found/pending responses.

## Best Possible Solution

Keep this issue open for maintainer decision. If active skills should stay public during clean update scans, update insertVersion/publishVersionForUser so existing active skills preserve active moderation status unless the new version is static-malicious, quality-quarantined, blocked, or the publisher is under moderation; then add regression coverage for a non-trusted publisher updating an active skill. If quarantine-on-update is intentional, document that policy and align web UI, CLI inspect/search, API responses, dashboard visibility, and production repair workflows so a successful publish does not look like data loss.
