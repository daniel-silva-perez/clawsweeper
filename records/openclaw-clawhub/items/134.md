## Summary

Keep this issue open. The ClawHub cleanup policy for this pass keeps issues open, and current main still has the core disappearance path: a normal non-trusted publisher updating an existing active skill can have the skill patched back to hidden pending scan, while public search/detail/API/CLI inspect paths filter hidden skills out. PR #136 mitigated owner visibility, but PR #274, the focused preserve-active-status fix, was closed unmerged.

## Best Possible Solution

Keep this issue open for maintainer decision. If active skills should remain public while clean updates are scanned, update publishVersionForUser/insertVersion so existing active skills preserve active moderation status unless the update is statically malicious, malware-blocked, quality-quarantined, removed, or the publisher is under moderation; add regression tests for non-trusted active-skill updates and owner/public visibility. If quarantine-on-update is intentional, document that policy and align CLI, web UI, API responses, dashboard status, and repair workflows.
