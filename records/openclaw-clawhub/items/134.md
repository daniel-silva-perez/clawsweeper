## Summary

Keep open. The repository cleanup policy for openclaw/clawhub keeps issues open, and current main still has a credible update-disappearance path: an existing active skill update by an ordinary non-trusted publisher can be patched to hidden pending scan, while public detail, API, and search paths filter hidden skills out.

## What This Changes

The issue reports that publishing a skill update succeeds or passes validation but leaves the skill hidden from search, inspect, API/detail pages, and sometimes the owner view with a `skills:publishVersion` server error.

## Best Possible Solution

Define and implement the update moderation policy for already-active skills. If clean active skills should not disappear, keep the previous approved public listing visible while the new version scans or otherwise preserve public visibility under explicit safety gates; if quarantine-on-update is intentional, make that state consistent and visible across CLI, web detail/dashboard, API, search, and recovery flows.
