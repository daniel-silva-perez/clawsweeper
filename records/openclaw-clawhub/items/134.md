## Summary

Keep open. The ClawHub cleanup policy for this pass keeps open issues open, and current main still contains the substantive disappearance path for ordinary non-trusted publishers: publishing an update computes a hidden pending-scan moderation state, patches that state onto the skill row, and public search/API/detail helpers filter hidden skills out. Current main has mitigations for owners and scan completion, but it does not prove the reported public visibility loss is fully resolved.

## What This Changes

Publishing an update to an existing ClawHub skill can make the skill disappear from public search, inspect/API, and detail surfaces because the skill row is moved to hidden pending-scan moderation state.

## Best Possible Solution

Keep this issue open for maintainer security/product decision. If active existing skills should remain public during clean updates, change publishVersionForUser/insertVersion so active skills either preserve active public visibility or keep the previous approved version public while the new version scans, except for static-malicious, quality-quarantined, manually locked, removed, or publisher-under-moderation cases. Add regression coverage for web detail, public search, v1 API/CLI inspect, owner pending status, and scan-completion behavior. If quarantine-on-update is intentional, document that policy and align the CLI, web UI, API responses, dashboard state, and recovery/rescan workflow.
