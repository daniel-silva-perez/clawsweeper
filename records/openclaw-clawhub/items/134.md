## Summary

Keep open. ClawHub cleanup policy only permits auto-closing PRs, and current main still has a credible update-disappearance path: ordinary non-trusted skill updates can rewrite an existing skill to hidden pending scan, while public detail/search/API surfaces filter hidden skills out.

## What This Changes

The issue reports that publishing a skill update can return success or a server error while the skill disappears from search, inspect, and public listings.

## Best Possible Solution

Define and implement the intended moderation policy for updates to already-active skills. If clean active skills should not disappear, keep the previous approved public version visible while the new version scans, or otherwise preserve public visibility only when security and quality gates make that safe; if quarantine-on-update is intentional, document it and align CLI, web, API, dashboard, and recovery flows.
