## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open, and current main still has a plausible active-skill update disappearance path: non-trusted updates can be patched to hidden pending-scan state, while public detail, search, inspect, and catalog surfaces filter hidden skills out.

## Best Possible Solution

Keep this issue open for maintainer decision. If already-active skills should remain public while clean updates are scanned, preserve active moderation status for existing active skills in insertVersion/publishVersionForUser and add regression coverage for a non-trusted publisher updating an active skill. If hiding during scan is intentional, document that policy and align CLI, web UI, API messages, owner visibility, and production repair workflows around it.
