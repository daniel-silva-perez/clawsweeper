## Summary

Keep this issue open. The ClawHub review policy only allows auto-close for PRs that are certainly implemented on current main, and current main still does not strongly prove the reported publish/update disappearance is fixed for non-trusted publishers.

## Best Possible Solution

Keep this issue open. Maintainers should decide the intended update moderation policy, then either land a corrected #274-style fix that preserves active visibility for legitimate existing skills unless the new version is malicious/quality-blocked/manual-moderated, or document and implement the hidden-pending behavior consistently across web, CLI inspect/search, and API responses. Add regression coverage for publishing an update to an already-active skill by a non-trusted publisher.
