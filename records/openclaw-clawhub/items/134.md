## Summary

Keep open. This is an open ClawHub issue, and the repository policy for this cleanup lane only permits auto-closing ClawHub pull requests that current main certainly implements. Substantively, current main still has a credible reproduction path for the reported disappearance: ordinary non-trusted skill updates can be written back as hidden pending scan, and the public detail/search/API/CLI surfaces filter hidden skills out or return an owner-only pending message.

## What This Changes

Publishing a skill update can make an existing public skill disappear from search, inspect, and web detail/API surfaces while `skills:publishVersion` reports success or a server error.

## Best Possible Solution

Keep this issue open for maintainer security/product decision. If active existing skills should remain public during clean updates, adjust `publishVersionForUser`/`insertVersion` so an already-active skill either preserves public visibility during the new-version scan or keeps the previous approved version public, while still hiding static-malicious, quality-quarantined, manually locked, removed, or publisher-under-moderation cases. If quarantine-on-update is intentional, document that policy and align CLI, web UI, API responses, dashboard state, and recovery/rescan workflow.
