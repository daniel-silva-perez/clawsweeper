## Summary

Keep open. ClawHub cleanup policy keeps issues open in this pass, and current main still has the reported disappearance path: an ordinary non-trusted publisher updating an existing skill can have the skill patched to hidden pending scan, while public search/detail/API/inspect paths filter hidden skills out. Related PR #136 only mitigated owner visibility, and the focused preserve-active-status PR #274 was closed unmerged.

## Best Possible Solution

Keep this issue open for maintainer security/product decision. If active skills should remain public while clean updates are scanned, update publishVersionForUser/insertVersion so existing active skills preserve active public visibility unless the update is statically malicious, quality-quarantined, malware-blocked, removed, manually locked, or the publisher is under moderation; add regression tests for public search/detail/API/CLI visibility. If quarantine-on-update is intentional, document the policy and align CLI, web UI, API responses, dashboard status, and repair workflows.
