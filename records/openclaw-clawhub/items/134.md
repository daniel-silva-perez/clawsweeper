## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a credible update-disappearance path for ordinary non-trusted publishers: an existing active skill update can be patched to hidden while public detail, search, catalog, and API surfaces filter hidden skills out.

## What This Changes

The issue asks to fix skill publishing/updating so an apparently successful publish or web publish error does not make an existing skill disappear from search, inspect, direct URLs, or the public API.

## Best Possible Solution

Define and implement an explicit update-moderation model. If clean active skills should remain public, keep the previous approved public listing visible while the new version scans and expose clear owner/API/dashboard status; if quarantine-on-update is intentional, make that state explicit and consistent across CLI, web, search, API, and recovery flows.
