## Summary

Keep open. This is a ClawHub issue, and the supplied cleanup policy only permits auto-closing pull requests; independently, current main still has a credible update-disappearance path for ordinary non-trusted publishers because an existing active skill update can be patched to hidden and then filtered out of public detail, search, catalog, and API surfaces.

## What This Changes

The issue reports that publishing an update to an existing skill succeeds or partially succeeds but makes the skill disappear from inspect, search, and web/API surfaces.

## Best Possible Solution

Define and implement the update moderation policy for already-active skills. If clean active skills should not disappear, keep the previous approved public listing visible while the new version scans or preserve public visibility under explicit safety gates; if quarantine-on-update is intentional, make that state consistent and visible across CLI, web detail/dashboard, API, search, and recovery flows.
