## Summary

Keep this issue open. The repository cleanup policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on current main, and this item is an issue. Substantively, current main still has a plausible active-skill update disappearance path: ordinary non-trusted publishes can patch an existing skill to hidden pending scan, while public detail, search, API, and CLI inspect paths filter hidden skills out or report not found. Related #136 improved owner visibility for pending skills, but #274, the explicit preserve-active-status fix, was closed unmerged.

## Best Possible Solution

Keep this issue open for maintainer decision. If already-active skills should remain public while clean updates are scanned, preserve active moderation status for existing active skills in insertVersion/publishVersionForUser and add regression coverage for a non-trusted publisher updating an active skill. If quarantine-on-update is intentional, document that policy and align CLI, web UI, API responses, owner dashboard visibility, and production repair workflows so a successful publish does not look like the skill was lost.
