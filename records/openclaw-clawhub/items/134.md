## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy for this pass only permits auto-closing PRs that are certainly implemented on current main. Substantively, current main still has the reported disappearance path: ordinary non-trusted publishers can update an existing active skill, `insertVersion` patches the skill to `moderationStatus: "hidden"`, and public search/API/detail helpers filter hidden skills out for non-owners. Related PR #136 only made pending skills visible to owners, while PR #274 directly targeted preserving active moderation status for updates but was closed unmerged.

## Best Possible Solution

Keep this issue open for maintainer security/product decision. If active skills should remain public while clean updates are scanned, update publishVersionForUser/insertVersion so existing active skills preserve public visibility, or keep the previously approved version public, unless the update is statically malicious, quality-quarantined, malware-blocked, manually locked, removed, or the publisher is under moderation. Add regression tests for public search/detail/API/CLI visibility. If quarantine-on-update is intentional, document that policy and align CLI, web UI, API responses, dashboard status, and recovery/rescan workflows.
