## Summary

Keep this issue open. The repository cleanup policy for openclaw/clawhub only permits automated close recommendations for pull requests that are certainly implemented on current main, and this item is an issue. Substantively, current main still has a plausible update-disappearance path: non-trusted skill updates can be patched to hidden pending-scan moderation state, while public search, list, catalog, and inspect surfaces filter hidden skills out. The current code adds owner-facing pending-state messaging, but that is only a partial mitigation and does not prove the reported public visibility regression is fixed.

## Best Possible Solution

Keep this issue open for maintainer decision. If already-active skills should remain public while clean updates are scanned, implement a preserve-active update path in insertVersion/publishVersionForUser and add regression coverage for a non-trusted publisher updating an active skill. If hiding during scan is intentional, document that behavior and make the CLI, web UI, API responses, owner visibility, and production repair workflow consistent.
