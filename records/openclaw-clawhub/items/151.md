## Summary

Keeping this issue open. The report describes a concrete Anthropic/OpenClaw runtime message serialization bug, but this ClawHub repository is a registry/catalog app and the provided repository policy says ClawHub issues should remain open unless they are PRs already implemented on main.

## Best Possible Solution

Keep this issue open for maintainer triage, then transfer or recreate it in the OpenClaw core/runtime tracker. The likely fix belongs in provider message conversion or session replay: when one assistant turn emits multiple tool_use blocks, coalesce all corresponding tool_result blocks into the immediately following Anthropic user message, or constrain Anthropic-compatible providers to one tool call per assistant turn, with a regression test based on the reporter's JSONL shape.
