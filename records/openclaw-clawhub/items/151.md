## What This Changes

The issue reports that OpenClaw persists multiple Anthropic tool results as separate messages after one assistant message with multiple tool calls, causing Anthropic protocol rejection.

## Best Possible Solution

Move the runtime-specific report to the OpenClaw tracker and fix the Anthropic message conversion so multiple tool results for one assistant tool-use turn are coalesced into the immediately following user/tool-result message, or constrain Anthropic-compatible providers to one tool call per turn, with a regression test based on the reported JSONL shape.
