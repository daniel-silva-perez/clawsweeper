## Summary

Keep open. The report is concrete and plausibly valid, but this ClawHub checkout is a registry/catalog app and does not contain the Anthropic/OpenClaw runtime transcript serialization path needed to verify or fix the bug here. Repository policy for this target also keeps issues open; only PRs that are certainly implemented on main are eligible for auto-close.

## What This Changes

The issue reports that OpenClaw persists multiple tool results from one assistant tool-use turn as separate messages, causing Anthropic-compatible providers to reject the transcript.

## Best Possible Solution

Route the report to the OpenClaw runtime/provider tracker and fix the Anthropic transcript conversion there so all tool results for one assistant tool-use turn are grouped into the immediately following message, with a regression test based on the reported JSONL shape.
