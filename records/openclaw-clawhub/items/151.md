## Summary

Keep open. The report is concrete and plausibly valid, but the current ClawHub repository is a registry/catalog app and does not contain the OpenClaw Anthropic runtime transcript serializer needed to verify or fix this provider protocol bug here; repository policy also keeps ClawHub issues open unless a PR is certainly implemented on main.

## What This Changes

Report that OpenClaw writes separate toolResult messages after one assistant multi-tool response, causing Anthropic to reject the transcript unless all corresponding tool_result blocks are grouped into the immediately following message or multiple tool calls are prevented.

## Best Possible Solution

Route the report to the OpenClaw runtime/provider tracker and fix the transcript conversion there so every tool_result block for one assistant multi-tool turn is emitted in the immediately following message, with a regression test based on the reported JSONL shape.
