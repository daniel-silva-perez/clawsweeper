## Summary

Keep this issue open. Repository policy for ClawHub keeps issues open for maintainer judgment, and current main still lacks a dedicated false-positive or suspicious-skill appeal intake path or broad automatic labeling/filtering for those reports.

## What This Changes

The issue requests a separate intake path or automatic labeling for false-positive and suspicious-skill appeals so product bugs are easier to filter from the issue tracker.

## Best Possible Solution

Keep this open for maintainer workflow judgment. Maintainers should choose the official appeal intake path, then update the suspicious-skill owner banner and add either GitHub issue templates/label automation or a ClawHub-native appeal/support flow so false-positive and suspicious-skill appeals are clearly separated from product bugs.
