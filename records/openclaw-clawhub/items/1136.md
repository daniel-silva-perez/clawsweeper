## Summary

Keep open. This is a contributor issue, and the supplied ClawHub policy says issues should remain open unless a maintainer decides otherwise; current main also still lacks a dedicated false-positive or suspicious-skill appeal intake, GitHub issue template, broad appeal-label automation, or ClawHub-native appeal flow. The existing rescan guidance is narrower than the requested issue separation.

## What This Changes

The issue requests a separate intake or automatic labeling path for false-positive and suspicious-skill appeal reports so product issues are easier to find.

## Best Possible Solution

Maintainers should choose the official appeal intake path, then update the suspicious-skill owner banner and either add GitHub issue-template/label automation or build a ClawHub-native appeal/support flow that separates appeals from product bugs without conflating them with owner rescans.
