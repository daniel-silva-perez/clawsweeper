## Summary

Keep open. This is a contributor issue, and ClawHub cleanup policy keeps issues open for maintainer judgment; current main still routes suspicious-skill appeals into the general issue list and does not provide a dedicated appeal channel or broad false-positive labeling/filtering workflow.

## What This Changes

Request a separate false-positive or suspicious-skill appeal intake path, or automatic labeling/filtering, so product issues are easier to find.

## Best Possible Solution

Keep this open for maintainer workflow judgment. Maintainers should choose the official appeal intake path, then update the suspicious-skill owner banner and add either GitHub issue templates/label automation or a ClawHub-native appeal/support flow so false-positive and suspicious-skill appeals are clearly separated from product bugs.
