## Summary

This should stay open. Current main still sends suspicious-skill owners to the general GitHub issue list and only has narrow rescan-guidance automation; it does not provide the dedicated appeal channel, issue template, label routing, or ClawHub-native appeal flow requested here. The ClawHub cleanup policy also says issues should remain open for maintainer judgment.

## What This Changes

The issue requests a separate intake or filtering mechanism for false-positive, suspicious-skill appeal, and related reports so product issues are easier to find.

## Best Possible Solution

Maintainers should choose an official appeal intake path, then implement that separation consistently by updating the suspicious-skill owner banner and adding either GitHub issue templates/label automation or a ClawHub-native appeal/support workflow.
