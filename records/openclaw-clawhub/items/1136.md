## Summary

Keep open. Current main still sends suspicious-skill false-positive appeals to the general GitHub issue list, and the existing rescan/report/filter surfaces do not create a separate appeal intake or broad false-positive issue filter. This also matches the existing ClawSweeper review comment and the ClawHub cleanup policy for contributor issues: leave it for maintainer workflow judgment.

## What This Changes

Request a separate false-positive and suspicious-skill appeal intake or labeling/filtering workflow so those reports do not obscure product issues.

## Best Possible Solution

Pick the official appeal intake path, update the suspicious-skill owner banner to point there, and add either GitHub issue-template/label automation or a ClawHub-native appeal/support flow so false-positive appeals are clearly separated from product bugs.
