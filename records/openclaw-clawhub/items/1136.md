## Summary

This issue should stay open. The current ClawHub review policy says issues are not auto-closed, and current main still does not provide the requested separate appeal intake or broad automatic labeling/filtering for false-positive and suspicious-skill appeal reports. Existing tools cover reporting, owner rescans, moderator overrides, and a narrow rescan-guidance workflow, but they do not solve the issue-triage problem described here.

## Best Possible Solution

Keep this open for maintainer workflow judgment. Maintainers should choose the official appeal intake path, then update the suspicious-skill owner banner and either GitHub issue templates/label automation or a ClawHub-native support/appeal flow so false-positive and suspicious-skill appeals are clearly separated from product bugs. The existing rescan-guidance workflow can be reused if maintainers choose a GitHub-label path, but it is not broad enough by itself.
