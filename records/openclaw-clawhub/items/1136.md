## Summary

This should stay open. The ClawHub cleanup policy keeps issues open for maintainer judgment, and current main still routes suspicious-skill appeals into general GitHub issues rather than a dedicated appeal intake, issue template, label automation, or ClawHub-native appeal flow.

## What This Changes

The issue requests a separate intake or filtering mechanism for false-positive and suspicious-skill appeal reports so product issues are easier to find.

## Best Possible Solution

Maintainers should choose the official appeal intake path, then implement that separation consistently by updating the suspicious-skill owner banner and adding either GitHub issue templates/label automation or a ClawHub-native appeal/support flow.
