## Summary

Keep this issue open. Current main has owner rescan and moderator override tools, plus a newer manually-triggered rescan-guidance label workflow, but it still does not provide a separate appeal venue or broad automatic labeling for false-positive/suspicious-skill appeal issues. ClawHub policy for this review also says issues should remain open unless maintainers handle them directly.

## Best Possible Solution

Keep this open for maintainer workflow judgment. Maintainers should choose the official appeal intake path, then update the suspicious-skill banner, docs, and either GitHub issue templates/label automation or a ClawHub-native appeal/support flow so false-positive and suspicious-skill appeals are clearly separated from product issues. The existing rescan-guidance workflow can be reused if maintainers choose a GitHub-label path, but it is not broad enough by itself.
