## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing pull requests that current main certainly implements. The live failure still reproduces, and current main still has a route/loader path that can render an owner/slug SSR shell with null metadata when the public skill lookup returns no result.

## What This Changes

The issue requests repairing or reindexing `mastrophot/near-phishing-detector` so its public skill page exposes normal display name, summary, and version metadata instead of null SSR data.

## Best Possible Solution

Resolve the production inconsistency explicitly: if the skill should be public, repair or reindex the production record so the public API and owner/slug SSR loader return populated metadata; if it should remain unavailable, make the owner/slug route render a clear not-found or moderated-state response instead of a successful null-metadata shell, with regression coverage for both API and SSR behavior.
