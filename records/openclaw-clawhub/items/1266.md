## Summary

Keep open. This is an issue, and the ClawHub cleanup policy for this pass only permits auto-closing pull requests that current main definitely implements. Substantively, the reported false-positive class is still present: current main stores and displays a VirusTotal Code Insight-derived suspicious status, while AV engine counts are only cached for moderation suppression and are not exposed as a first-class public API/UI detection ratio.

## What This Changes

The issue requests that VirusTotal suspicious/benign status be based on actual AV engine detection counts, or at least display the 0/N detection ratio, when VirusTotal heuristic metadata says suspicious but engines report 0 detections.

## Best Possible Solution

Define a canonical VirusTotal policy that separates AV engine detections from Code Insight heuristic findings, stores and exposes engineStats or a computed malicious+suspicious/total ratio, keeps heuristic analysis visible as context, updates UI copy accordingly, and provides a rescan/backfill or manual override path for affected existing skill versions.
