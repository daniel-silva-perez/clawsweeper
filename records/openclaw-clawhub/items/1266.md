## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a real product gap: backend moderation has a targeted suppression for clean-engine VirusTotal Code Insight false positives, but the VirusTotal ingestion, public scan API, and UI still derive visible VT status from stored Code Insight/status fields and do not expose a 0/N AV engine detection ratio.

## What This Changes

The issue asks ClawHub to base VirusTotal suspicious/benign presentation on AV engine detection ratio, or at least show the 0/63 detections context, when VirusTotal Code Insight heuristic tags conflict with clean engine results.

## Best Possible Solution

Keep this open for maintainer security-product review. Define a canonical VirusTotal policy that separates AV engine detections from Code Insight heuristic findings, expose engineStats or a computed detection ratio in the public API and UI, and add a rescan/backfill or moderation reconciliation path for existing affected skill versions.
