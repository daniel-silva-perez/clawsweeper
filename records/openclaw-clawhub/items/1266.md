## Summary

Keep open. This is an issue, and the ClawHub cleanup policy for this pass only permits auto-closing pull requests that current main definitely implements. The report is also still substantively valid: current main stores and displays a VirusTotal Code Insight-derived suspicious status, while AV engine counts are only cached as auxiliary data and are not exposed as a first-class public API/UI detection ratio.

## What This Changes

The reporter asks ClawHub to base or explain VirusTotal suspicious/benign status using the AV engine detection ratio, such as 0/63 detections, instead of treating heuristic Code Insight tags as the primary visible verdict.

## Best Possible Solution

Define a canonical VirusTotal policy that separates AV engine detections from Code Insight heuristic findings, stores and exposes engineStats or a computed malicious+suspicious/total ratio, keeps heuristic analysis visible as context, updates UI copy accordingly, and provides a rescan/backfill or manual override path for affected existing skill versions.
