## Summary

Keep open. This is an open ClawHub issue, and the supplied cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main has a strong backend mitigation for uncorroborated VirusTotal Code Insight suspicious results with clean AV engine stats, clean static scan, and clean LLM scan, but the named production skill still reports `suspicious.vt_suspicious`, and the current UI/API still do not expose a 0/N VirusTotal detection ratio.

## Best Possible Solution

Keep this issue open for maintainer/product-security follow-up. The best path is to deploy and verify the current Code Insight calibration, recompute or backfill the named skill and related false-positive reports, then add `engineStats` or a normalized detection ratio to the public scan API and Security Scan UI so Code Insight advisories are clearly separated from AV engine detections. A maintainer can close manually once the production skill and UI/API behavior are verified.
