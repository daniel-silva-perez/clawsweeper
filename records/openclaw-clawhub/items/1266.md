Keep open. This is a ClawHub issue, and the repository cleanup policy for this run only permits auto-closing ClawHub pull requests that current main definitely implements. Substantively, current main has a partial moderation mitigation for uncorroborated VirusTotal Code Insight suspicious verdicts, but the public scan/API/UI path can still surface VirusTotal as Suspicious without exposing the engine detection count distinction requested here.

## Best Possible Solution

Keep this issue open for maintainer security-product review. The likely fix is to define a canonical VirusTotal display/moderation policy that separates AV engine detections from Code Insight/heuristic findings, exposes engine detection counts in the API and UI, and includes a backfill or rescan path for existing vtAnalysis rows before closing the false-positive cluster.
