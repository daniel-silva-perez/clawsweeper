## Summary

Keep open. This is an open ClawHub issue, and the repository policy for this review run only allows auto-close for ClawHub PRs that current main definitely implements. Substantively, current main contains a partial backend mitigation for uncorroborated VirusTotal Code Insight suspicious verdicts, but the public scan/API/UI path can still show VirusTotal as Suspicious without exposing the engine detection count distinction requested here.

## Best Possible Solution

Keep this issue open for maintainer security-product review. The best path is to define a canonical VirusTotal policy that separates AV engine detections from Code Insight/heuristic findings, expose the engine detection ratio in the public API and UI, and add a rescan/backfill path to clear or relabel affected existing rows once the policy is chosen.
