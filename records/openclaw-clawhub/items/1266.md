## Summary

Keep open. Repository policy for this ClawHub cleanup pass keeps issues open, and the report is still substantively live: current main has a narrow moderation suppression for uncorroborated VirusTotal Code Insight results, but VirusTotal ingestion, public security snapshots, and UI rows still derive visible VirusTotal status from stored Code Insight status/verdict rather than exposing or basing the display on an AV engine detection ratio.

## What This Changes

The issue requests changing the VirusTotal security verdict/display so a skill with zero AV engine detections is not shown as red Suspicious solely because of VirusTotal Code Insight or heuristic metadata, or at least showing the detection ratio.

## Best Possible Solution

Define a canonical VirusTotal policy that separates AV engine detections from Code Insight heuristic findings, exposes engineStats or a computed 0/N detection ratio in the public API and UI, keeps useful heuristic context visible without overstating it as an AV detection, and provides a rescan/backfill or moderation reconciliation path for affected skill versions.
