## Summary

Keep open. Repository policy for this ClawHub cleanup pass keeps issues open, and the underlying false-positive class is only partially mitigated on current main: moderation can suppress an uncorroborated VirusTotal Code Insight suspicious result when local and LLM scans are clean, but public scan/API/UI surfaces still present the stored VirusTotal Code Insight status without a first-class AV detection ratio.

## What This Changes

The issue asks ClawHub to stop showing a suspicious security warning for a skill when VirusTotal AV engines report 0/63 detections, or at least expose the detection ratio beside the status.

## Best Possible Solution

Define a canonical VirusTotal policy that separates AV engine detections from Code Insight heuristics, exposes engineStats or a computed malicious+suspicious/total ratio in the public API and UI, keeps Code Insight visible as contextual analysis, and includes a rescan/backfill or manual override path for already-scanned affected skills.
