## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Provide a stable server snapshot:** `src/i18n/useI18n.ts:6-10`
  - body: `useSyncExternalStore` is used without a server snapshot in an SSR app. Returning users with `zh-CN` in browser storage can hydrate from a different locale than the English server render, affecting `<html lang>` and shell strings. Provide a stable `getServerSnapshot` or another SSR-visible locale source.
  - confidence: 0.86
- **[P2] Keep head metadata in sync with locale:** `src/routes/__root.tsx:16-20`
  - body: The route `head()` reads the module-level `t()` singleton, but it is not subscribed to the i18n store. Switching language from the header can update body strings and `<html lang>` while title, description, OG, and Twitter metadata remain in the initial locale.
  - confidence: 0.82
- **[P3] Guard unexpected empty zh-CN translations:** `src/i18n/__tests__/locale-parity.test.ts:56-63`
  - body: The parity test only checks English leaf values for non-empty strings while `zh-CN` intentionally contains `footer.an: ""`. Add an explicit allowlist and fail on any other empty zh-CN value so accidental blank translations do not silently render nothing.
  - confidence: 0.78
