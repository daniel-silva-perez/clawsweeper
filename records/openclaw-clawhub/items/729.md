## Summary

Keep this PR open. Current main does not contain the proposed i18n infrastructure or zh-CN UI support: there is no src/i18n tree or i18n symbols, the root shell still emits English-only metadata and <html lang="en">, and shared header/navigation/browse UI remains hardcoded English. The PR remains an implementation candidate for open issue #606, but it is broad, stale against current main, and has unresolved review comments around SSR hydration, non-reactive head metadata, empty-string translation contracts, and remaining hardcoded strings.

## What This Changes

The PR branch adds a zero-dependency frontend i18n layer with typed translation keys, en and zh-CN locale files, a header language switcher, locale-aware shell metadata, broad UI string extraction, and a locale parity Vitest test.

## Best Possible Solution

Keep the PR open for maintainer review as the likely implementation path for #606. If ClawHub wants core i18n, rebase it on current main, update it for current publish/plugin/search/navigation surfaces, resolve the SSR/head/empty-string/hardcoded-string review comments, and run the normal frontend/typecheck validation. If core i18n is not desired, close it only after an explicit maintainer product decision and update #606 accordingly.
