## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Provide a server snapshot for i18n state:** `src/i18n/useI18n.ts:6-10`
  - body: `useSyncExternalStore` is used without a `getServerSnapshot`. In this SSR app, the server renders the singleton as English, but the first client render can initialize from `localStorage` or `navigator` as `zh-CN`, causing `<html lang>` and translated shell content to hydrate with a different value for returning users.
  - confidence: 0.86
- **[P2] Keep head metadata in sync with locale changes:** `src/routes/__root.tsx:16-20`
  - body: The route `head()` reads the module-level `t()` singleton, but it is not subscribed to the i18n store. Switching language from the header can update body strings and `<html lang>`, while title, description, OG, and Twitter metadata remain in the initial locale, so the PR does not fully deliver locale-aware head metadata.
  - confidence: 0.82
- **[P3] Guard non-empty zh-CN translations explicitly:** `src/i18n/__tests__/locale-parity.test.ts:56-63`
  - body: The parity test only checks English leaf values for non-empty strings, while `zh-CN` intentionally contains an empty `footer.an`. That leaves accidental empty zh-CN translations undetected; add an allowlist for intentional blanks and fail on any other empty value.
  - confidence: 0.78
