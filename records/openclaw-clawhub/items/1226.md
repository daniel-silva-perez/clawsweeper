## Summary

Keep this issue open. Repository policy for openclaw/clawhub keeps issues open, and current main still has a plausible path-resolution bug for `clawhub package publish .`: package source paths are resolved against the CLI workdir, while that workdir can fall back away from the caller's plugin repository. The related comment asking for plugin search/install/list CLI commands is useful adjacent product feedback, but it is separate from the publish failure.

## Best Possible Solution

Keep this issue open for maintainer triage. Add a regression test for `clawhub package publish . --family code-plugin` from a plugin repository while the configured/global workdir points elsewhere, then either resolve explicit local publish paths relative to the caller's cwd or make the required `--workdir` behavior clear and enforced. Track the plugin search/install/list CLI workflow as a separate product item if maintainers want that terminal experience.
