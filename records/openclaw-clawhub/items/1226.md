## Summary

Keep open. Repository policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on main, and this item is an issue. Current main also still leaves the reported false missing-manifest path plausible: `package publish` resolves the source against the resolved global workdir, while `.` can differ from the caller's plugin repository, and the exact `openclaw.plugin.json required` error remains reachable for code-plugin publishes.

## Best Possible Solution

Keep this issue open for maintainer triage. Add a regression test for `clawhub package publish . --family code-plugin` from a plugin repository while the configured global workdir points elsewhere, then either resolve explicit local publish paths relative to the caller's cwd or clearly document and enforce the required `--workdir` behavior. Track any remaining plugin install/list terminal workflow as a separate CLI enhancement, noting that `package explore` and `package inspect` already cover browsing/inspection.
