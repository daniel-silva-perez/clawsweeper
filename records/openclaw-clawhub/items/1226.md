## Summary

Keep open. The ClawHub cleanup policy does not auto-close issues, and current main still leaves the reported false manifest error plausible because relative package sources are resolved under the CLI's resolved global workdir, which can differ from the plugin directory where the user ran the command.

## What This Changes

The issue reports that `clawhub package publish . --family code-plugin` rejects a local code-plugin folder with `openclaw.plugin.json required` even though the manifest exists at the package root.

## Best Possible Solution

Implement a focused CLI fix so local package publish sources resolve against the caller's intended package directory when no explicit workdir override is supplied, while preserving explicit workdir/env behavior and GitHub source precedence. Add regression coverage for configured-workspace versus plugin-cwd behavior and keep compatibility metadata validation as the next clear failure when applicable.
