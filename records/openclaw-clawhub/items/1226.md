## Summary

Keep this issue open. The repository policy keeps open ClawHub issues out of auto-close, and current main still has a plausible package-publish path bug: local package sources are resolved against the resolved CLI workdir, while that workdir can fall back to a configured Clawdbot/OpenClaw workspace instead of the caller's plugin repository. That can make `clawhub package publish . --family code-plugin` inspect the wrong folder and throw `openclaw.plugin.json required` even when the manifest exists in the shell cwd.

## Best Possible Solution

Keep this open and fix the CLI package-publish local source path behavior. Add a regression test for `clawhub package publish . --family code-plugin` from a plugin repository when the global/default workdir would otherwise point elsewhere, then adjust local package source resolution or workdir-marker detection so the manifest in the caller's plugin repo is used. Track plugin search/install/list CLI work separately from this bug.
