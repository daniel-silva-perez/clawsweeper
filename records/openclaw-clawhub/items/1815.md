## Security Review

Status: needs_attention

Summary: This is a security-sensitive hosted-skill moderation appeal, and the live artifact still has suspicious scanner verdicts plus host-modifying scripts that need human review.

Concerns:

- **[medium] Live artifact still has suspicious scanner verdicts:**
  - body: The versioned public scan reports the requested v1.0.1 artifact as matching current moderation with `isSuspicious: true`; VirusTotal and LLM scanners are both normalized to suspicious while the static scan is clean, so clearing the flag is a moderator/security decision rather than repository cleanup.
  - confidence: 0.96
- **[medium] Served scripts make persistent host changes:**
  - body: The published setup scripts append environment variables to `~/.bashrc`, disable Testcontainers Ryuk cleanup, run pip installs, start long-lived services, and download/execute a WireMock JAR. Those behaviors may be legitimate for testing, but they are exactly the kind of host-modifying actions that should be reviewed before an override.
  - confidence: 0.92
- **[medium] Security documentation overstates ephemerality:**
  - body: The published `SECURITY.md` says test containers are automatically cleaned up and no persistent data or services remain, but the served scripts disable automatic cleanup and persist shell-profile configuration. A moderator should account for that mismatch when deciding whether to clear the flag or request a corrected upload.
  - confidence: 0.9
