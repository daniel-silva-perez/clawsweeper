## Summary

Keep this issue open. It is an open ClawHub issue, and the supplied repository policy only permits automated closure for PRs that current main certainly implements. Substantively, current main has exact slug search for active public skills, but a newly uploaded skill can still be absent from public search when moderation, pending scan, quality quarantine, suspicious filtering, owner state, or production digest/index state prevents `toPublicSkill` from returning it. Related open reports #137, #382, and #423 should be triaged as the same publish-to-search lifecycle cluster.

## Best Possible Solution

Keep this open for maintainer triage with #137, #382, and #423. Inspect the production `cloudq` skill’s `moderationStatus`, `moderationReason`, scan fields, owner/publisher state, latest version, embedding, and `skillSearchDigest` row; then decide whether the product should keep previous approved versions searchable during scans, show owner-facing search/status guidance, or adjust indexing so active approved skills are always discoverable by exact slug and relevant terms.
