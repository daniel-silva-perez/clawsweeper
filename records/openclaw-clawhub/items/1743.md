## Summary

Keep this issue open. Repository policy for openclaw/clawhub only allows automated cleanup closure for pull requests, not issues. Substantively, current main has an exact-slug search path, but public search still deliberately drops non-active skills, and the report matches an active cluster of open search/indexing reports (#137, #382, #423).

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to inspect the production `cloudq` record and scan/moderation state, consolidate with the open search-indexing cluster if appropriate, and make the publish-to-search lifecycle explicit: approved active skills should be searchable by exact slug and browse/list surfaces, while pending or hidden skills should show clear owner-facing status and remediation guidance.
