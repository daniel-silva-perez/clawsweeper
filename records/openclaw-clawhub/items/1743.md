Keep open. The ClawHub cleanup policy for this repository does not auto-close issues, and source review does not prove the reporter's production `cloudq` search failure is resolved. Current main has exact full-slug search for public active skills, but that path can still return nothing when the production row is hidden by moderation or scan state, soft deletion, owner resolution, suspicious filtering, or stale/missing search digest data; related open reports show this publish-to-discovery area is still active.

## What This Changes

The reporter asks why an uploaded skill with slug `cloudq` cannot be found by searching for `cloudq` in ClawHub.

## Best Possible Solution

Triage `cloudq` in production alongside the related publish-to-search reports, verify the skill row, latest version, scan/moderation fields, owner record, embedding and `skillSearchDigest` state, then repair indexing for approved active skills or improve owner-facing status and remediation when a skill is intentionally hidden.
