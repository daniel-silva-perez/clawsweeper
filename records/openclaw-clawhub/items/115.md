## Summary

Keep this issue open. The ClawHub cleanup policy only permits auto-closing pull requests that are certainly implemented on main, and the functional review still does not prove that the reported successful-login-then-Unauthorized flow is fixed in production. Current main has the expected CLI and API bearer-token plumbing, but two users reported the failure and related auth/publish Unauthorized reports remain open without a maintainer-confirmed fix or release note.

## Best Possible Solution

Keep this issue open for maintainer triage. Reproduce with the current published CLI and production API, verify that both browser-created and manually-created tokens work for /api/v1/whoami and POST /api/v1/skills, and add focused logging or regression coverage around registry discovery, config path selection, token hashing/lookup, user linkage, and active-user checks. If the real failure is an account state or publish permission precondition, return that specific error instead of a generic Unauthorized.
