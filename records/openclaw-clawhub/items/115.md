## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing PRs that are definitely implemented on current main. Current main contains the expected CLI token verification, token storage, bearer-token publish path, browser token minting, and Convex token-auth handlers, but the issue discussion and related open reports still describe production Unauthorized failures, so maintainer production triage is still needed.

## What This Changes

The issue requests that a successful `clawhub auth login` and `whoami` allow `clawhub publish` instead of returning `Unauthorized`.

## Best Possible Solution

Make browser-created tokens and Settings-created tokens work reliably for non-maintainer accounts against production `https://clawhub.ai` across `login`, `whoami`, and `publish`; then add or strengthen regression coverage around token storage, registry normalization, redirect/header preservation, token hash/user lookup, and publish authorization, and consolidate duplicate Unauthorized reports once production is verified.
