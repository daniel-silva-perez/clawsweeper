## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy keeps issues open unless they are PRs that are definitely implemented on current main. Current main has the expected CLI token verification, token storage, bearer publish path, browser token minting, and Convex token-auth handlers, but the issue discussion plus related open reports still describe production Unauthorized failures, so source inspection alone does not prove the user-visible problem is fixed.

## What This Changes

The issue reports that `clawhub auth login` and `whoami` can succeed, but `clawhub publish` immediately returns `Unauthorized` for the same account/token.

## Best Possible Solution

Make browser-created tokens and Settings-created tokens work reliably for non-maintainer accounts against production `https://clawhub.ai`, covering `login`, `whoami`, and `publish`; then add regression coverage for token storage, registry normalization, redirect/header preservation, token hash/user lookup, and publish authorization, or consolidate the remaining Unauthorized reports under one canonical issue.
