## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy keeps issues open unless they are PRs that are definitely implemented on current main. Current main has the expected CLI token verification, bearer publish path, browser token minting, and Convex token-auth handlers, but multiple users and related open reports still describe production Unauthorized failures, so source inspection does not prove the deployed flow is fixed.

## What This Changes

Publishing should succeed after a successful `clawhub auth login` / `whoami` flow instead of returning `Unauthorized`.

## Best Possible Solution

Keep this issue open for maintainer triage.
