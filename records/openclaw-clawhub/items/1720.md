## Summary

Keep open. Current main does not implement this PR’s intended auth fail-fast behavior: `convex/auth.ts` still falls back to empty GitHub OAuth credentials, CI test/coverage envs still lack the proposed stub `AUTH_GITHUB_*` values, and `.env.test` is not present. Under the ClawHub policy, open PRs should only be auto-closed when their intended change is already implemented on main, so this PR remains a live implementation candidate for the linked auth outage in #1717.

## Best Possible Solution

Keep this PR open and review/land an auth fix, either this fail-fast implementation with its test/CI setup or an equivalent runtime validation that produces actionable diagnostics. Separately, the production Convex deployment still needs valid `AUTH_GITHUB_ID` and `AUTH_GITHUB_SECRET` configured to resolve the live OAuth outage tracked in #1717.
