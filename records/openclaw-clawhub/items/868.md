## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open unless maintainers handle them, and current main still has the reported failure mode: `clawhub sync` publishes selected skills sequentially and directly awaits `cmdPublish`, so a slug-conflict publish rejection aborts before later selected skills can upload.

## Best Possible Solution

Keep this issue open and fix the CLI sync workflow. `cmdSync` should isolate `cmdPublish` failures per selected skill, continue later selected uploads, preserve the backend slug-conflict message and existing-owner URL, summarize successes and failures, and return a failing exit status when any selected upload failed. Add a focused regression test where one selected publish rejects with the slug-taken error while a later selected skill still publishes.
