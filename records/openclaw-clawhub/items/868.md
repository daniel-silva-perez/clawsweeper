## Summary

Keep this issue open. ClawHub policy keeps issues open for maintainer follow-up, and current main still has the reported failure mode: `clawhub sync` awaits each selected `cmdPublish` call directly, `cmdPublish` rethrows upload errors, and the backend still returns slug conflicts as publish failures. A conflict in one selected skill can still abort later uploads.

## Best Possible Solution

Keep this issue open. The likely fix belongs in `packages/clawhub/src/cli/commands/sync.ts`: catch `cmdPublish` failures per selected skill, continue later uploads, preserve the backend slug-conflict message and existing-owner URL, summarize successes and failures at the end, and return a failing exit status when any upload failed. Add a regression test where one selected publish rejects with the slug-taken error while a later selected skill still publishes.
