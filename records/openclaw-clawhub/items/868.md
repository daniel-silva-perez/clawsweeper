## Summary

Keep this issue open. Repository policy for openclaw/clawhub keeps issues open, and current main still has the reported batch-abort failure mode: sync awaits each publish directly, publish rethrows registry upload errors, and the backend still reports cross-owner slug conflicts as publish failures.

## Best Possible Solution

Keep this issue open. The likely fix belongs in packages/clawhub/src/cli/commands/sync.ts: isolate cmdPublish failures per selected skill, continue later uploads, preserve the backend slug-conflict message and existing-owner URL, summarize successes and failures at the end, and return a failing exit status when any upload failed. Add a regression test where one selected publish rejects with the slug-taken error while a later selected skill still publishes.
