## Risks / Open Questions

- External GitHub profile URLs should be derived from verified provider identity or synced GitHub login data, not from editable ClawHub handles alone.
- Adding external anchors inside current card-level links, especially SkillListItem's wrapping Link, can create invalid nested links or interfere with internal navigation.
- Repository/source links should remain tied to trusted skill/package metadata such as clawdis.links.repository, package sourceRepo, or verification sourceRepo rather than guessed from owner and slug strings.
- If GitHub profile data is denormalized into search/list owner data, the change needs a cursor-based backfill and updates to every owner-sync path that maintains the digest.
