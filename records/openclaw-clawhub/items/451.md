## Summary

Keep this issue open. The ClawHub cleanup policy for this review only permits auto-closing pull requests that are certainly implemented on current main, and this item is an issue. A fresh source review also confirms current main still renders ClawHub-internal profile/owner links or plain owner text, while GitHub profile links beside usernames and in the user profile header are not implemented; repository links are only partially surfaced on detail/verification surfaces.

## Best Possible Solution

Keep this issue open and implement a focused UI/data change: store or derive a verified public GitHub profile URL/login from the authenticated GitHub account, expose it through the public user/publisher projections, render external GitHub links beside owner/user names and in profile headers, and surface repository/source links consistently from skill metadata without breaking existing internal card/list navigation. Add targeted component/query tests for personal users, org publishers, and missing/renamed GitHub data.
