## Summary

Keep this issue open. The repository policy for openclaw/clawhub only allows auto-closing PRs that are certainly implemented on main, and this item is an issue. Current main still does not add external GitHub profile links beside usernames or in the user profile header. Repository links are only partially surfaced when skill metadata provides them, so the broader request remains unresolved.

## Best Possible Solution

Implement a focused ClawHub UI/data change: expose a verified GitHub login or profile URL in safe public user/publisher data, add external GitHub links beside owner badges and in profile headers, and surface repository/source links consistently from skill metadata without breaking card/list navigation.
