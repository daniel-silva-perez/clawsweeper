Keep this issue open. The ClawHub cleanup policy only allows auto-closing pull requests that are certainly implemented on main, and this item is an issue. Current main also still does not add external GitHub profile links beside user names or in the user profile header; repository/source links are only partially surfaced when skill/package metadata provides them.

## Best Possible Solution

Keep this open and implement a focused ClawHub UI/data change: expose a verified public GitHub login or profile URL for users/publishers, render external GitHub links beside owner/user names and in profile headers, and surface repository/source links consistently from skill metadata without breaking card/list navigation.
