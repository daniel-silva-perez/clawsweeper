## Summary

Keep this issue open. Under the provided ClawHub cleanup policy, issues are not eligible for auto-close, and current main still does not implement the requested external GitHub profile links beside usernames or in the user profile header. Repository links are only partially covered where skill metadata includes `clawdis.links.repository`; the broader user/profile link request remains open.

## Best Possible Solution

Keep this issue open and implement a focused core UI/data change: expose a verified GitHub login or profile URL in safe public user/publisher data, add external GitHub links beside owner badges and profile headers, and surface repository/source links consistently from skill metadata without breaking card or list navigation.
