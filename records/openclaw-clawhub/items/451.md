## Summary

Keep open. This is an issue in openclaw/clawhub, and the repository policy only allows auto-closing PRs that are certainly implemented on main. Current main also still does not add GitHub profile links beside usernames or make the profile page name/handle link to GitHub; repository links are only conditionally shown from skill metadata on the detail page.

## Best Possible Solution

Keep this issue open and implement a focused core UI change: expose a verified GitHub profile URL/login in the public user/publisher data where appropriate, add external GitHub links next to owner badges and on profile headers, and surface repository/source links more clearly from skill metadata without breaking the existing card/list navigation.
