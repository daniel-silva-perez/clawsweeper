## Summary

Keep this issue open. The openclaw/clawhub cleanup policy only allows auto-closing pull requests that are certainly implemented on main, and this item is an issue. Current main also still does not implement the requested GitHub user/profile-link behavior: profile headers render plain text, UserBadge links to ClawHub profile/org routes, and browse/list owner text does not expose a separate GitHub profile link. Repository links exist only when supplied through Clawdis link metadata, so the broader project/source-link request is not fully resolved either.

## Best Possible Solution

Keep this issue open and implement a focused core UI/data change: expose a verified GitHub profile URL or login in the public user/publisher data where appropriate, add external GitHub links next to owner badges and profile headers, and surface repository/source links consistently from skill metadata without breaking card/list navigation.
