## Summary

Keep open. This is a ClawHub issue, and the repository policy only permits auto-closing PRs that current main definitely implements. Current main still lacks verified GitHub profile URLs in public user/publisher data and in the requested profile/owner UI; repository links exist only on some trusted metadata-driven surfaces, so they do not fully satisfy the username/profile request.

## What This Changes

Request to add external GitHub profile links next to usernames and on public user pages, plus project repository links where available.

## Best Possible Solution

The desired end state is for public owner/profile views to offer internal ClawHub navigation plus a clearly separate external GitHub link when the account was verified through GitHub, while source/repository links remain based on trusted package or skill metadata.
