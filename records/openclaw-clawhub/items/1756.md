## Summary

Keep this issue open. Repository policy for this ClawHub review run only permits auto-closing pull requests that are certainly implemented on main, and current main still has a plausible path for the reported behavior: search load-more changes the request limit, replaces the displayed search result set, and backend relevance ranking derives candidate expansion from that limit.

## Best Possible Solution

Keep this issue open. The best fix is to make relevance search pagination stable, either with a server-side cursor over a fixed ranked result set or by decoupling candidate expansion from the display limit so a load-more request cannot insert newly discovered higher-scoring items above already-rendered entries. Add a regression test comparing the top relevance results for the initial 25-result request with a larger scroll/load-more request.
