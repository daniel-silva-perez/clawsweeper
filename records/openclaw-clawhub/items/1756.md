## Summary

Keep open. Repository policy for this ClawHub review only permits auto-closing PRs that are certainly implemented on main, and this item is an issue. The source still shows the relevant search flow: first search render uses a 25-item limit, load-more increases that limit, and the backend relevance candidate pool/slice still depends on the requested limit, with no regression test proving top-result stability across the scroll-triggered larger request.

## Best Possible Solution

Keep this issue open and fix the search contract. The likely path is server-side pagination/cursors over a stable ranked result set, or decoupling candidate expansion from display limit so scroll-triggered load-more cannot insert newly discovered higher-scoring skills above already-rendered top results. Add a regression test covering `/skills?q=image`-style initial render, load-more, and stable or deliberately surfaced ordering changes.
