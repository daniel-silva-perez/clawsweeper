## Summary

Keep open. ClawHub policy keeps issues open, and current main still has a credible source-level path for the reported scroll-triggered reorder: the UI increases the search limit and replaces the full result set, while backend relevance ranking can evaluate a different candidate pool for the larger request before slicing results.

## What This Changes

The issue reports that `/skills` relevance search initially omits high-relevance skills and inserts them near the top only after the user scrolls enough to trigger load-more.

## Best Possible Solution

Make `/skills` relevance pagination stable so the first page for a query remains a prefix of later larger/page requests. A durable fix should either page over a stable ranked result set or decouple recall and candidate expansion from the display limit, with regression coverage comparing initial and larger requests.
