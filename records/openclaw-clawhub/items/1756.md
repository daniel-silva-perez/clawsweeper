## Summary

Keep open. Repository policy for this ClawHub cleanup pass only permits auto-closing pull requests that are certainly implemented on main, and this item is an issue. Substantively, current main still supports the reported behavior: `/skills` search starts with a 25-result request, scroll/load-more increases the requested limit, the UI replaces the whole search result set, and backend relevance search can compute a broader candidate set for the larger limit.

## Best Possible Solution

Keep this issue open. The best fix is to make relevance search pagination stable, either with a server-side cursor over a fixed ranked result set or by decoupling candidate expansion from the display limit so load-more cannot discover higher-scoring results above already-rendered entries. Add regression coverage comparing the top relevance results from the initial 25-result request against a larger scroll/load-more request.
