## Summary

Keep open. This is a ClawHub issue, and the repository policy for this cleanup pass only permits auto-closing ClawHub pull requests that are certainly implemented on main. Current main also still has a plausible implementation path for the reported behavior: the /skills UI replaces the whole search result set after scroll/load-more increases the requested limit, while backend relevance search derives candidate expansion and final slicing from that limit.

## Best Possible Solution

Keep this issue open. The best fix is to make relevance search pagination stable, either by using a server-side cursor over a fixed ranked result set or by decoupling candidate expansion from the display limit so load-more cannot discover higher-scoring entries above already-rendered results. Add a regression test that compares the top results for the initial 25-result request against a larger request for the same query.
