## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a credible source-level path for the reported behavior: scrolling can increase the active relevance search limit and replace the visible result array, while backend ranking can evaluate a different candidate pool for the larger request before slicing results.

## What This Changes

The issue reports that `/skills` relevance search initially renders one ordering, then inserts higher-relevance skills near the top after the user scrolls and load-more runs.

## Best Possible Solution

Make `/skills` relevance pagination stable so the first rendered page for a query remains a prefix of later larger/page requests. A durable fix should decouple recall/candidate expansion from the display limit or page over a stable ranked result set, with backend and UI regression coverage for the initial 25-result request versus the 50-result load-more request.
