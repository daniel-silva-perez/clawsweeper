## Summary

Keep this draft PR open. Current main does not implement the PR's intended fix: `cmdPublish` still uses `listTextFiles(folder)` directly, so a broad ignore rule can hide the required root `SKILL.md`/`SKILLS.md` before the publish command checks for it. The PR remains a focused implementation candidate for the linked open bug, not obsolete work already present on main.

## Best Possible Solution

Keep this draft PR open for maintainer review. The likely path is to land a focused CLI fix that preserves only the required root manifest during publish, decide whether explicit `.clawhubignore` entries should be allowed to suppress that manifest, and then close the linked bug after merge and release.
