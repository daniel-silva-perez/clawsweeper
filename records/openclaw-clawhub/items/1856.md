## Summary

Keep open. Current main still publishes exactly the files returned by `listTextFiles(folder)`, and `listTextFiles` applies ignore rules before publish validation, so a broad `*.md` ignore can still hide root `SKILL.md`/`SKILLS.md`. This draft PR remains a focused implementation candidate for #1509 and is not implemented on main.

## What This Changes

This PR changes the ClawHub CLI publish command to re-add a root `SKILL.md`/`SKILLS.md` manifest when broad ignore rules excluded it, and adds a publish regression test for a `.gitignore` containing `*.md`.

## Best Possible Solution

Keep this draft PR open for maintainer review. The likely path is to land or adapt the localized `cmdPublish` root-manifest fallback, preserve shared `listTextFiles` ignore semantics, add the broad markdown-ignore regression test, and close #1509 only after the fix merges and ships.
