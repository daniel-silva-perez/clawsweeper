## Summary

Keep this PR open. Current main still publishes only the files returned by `listTextFiles`, and that discovery path honors `.gitignore`/`.clawhubignore` before `cmdPublish` validates that a root `SKILL.md` or `SKILLS.md` exists. The PR remains a focused draft implementation candidate for the linked open bug rather than work already present on main.

## Best Possible Solution

Keep this draft PR open for maintainer review. The likely path is to land or adapt the localized `cmdPublish` fallback, preserve normal `listTextFiles` ignore semantics, keep the broad-markdown-ignore regression test, and close the linked bug only after the fix merges and ships.

There is no fallback that re-reads a root manifest from disk.
