## Summary

Keep open. Current main still has the failure path that this draft PR addresses: `clawhub publish` validates only the files returned by the ignore-aware text-file walker, so a broad ignore rule can still hide the required root manifest. The proposed branch remains a focused implementation path for the linked publish bug, and I did not find a concrete blocking correctness or supply-chain issue in the diff.

## What This Changes

The PR updates `clawhub publish` to add back a root `SKILL.md` or `SKILLS.md` manifest when normal text-file discovery misses it due to ignore rules, with a regression test for `.gitignore` `*.md`.

## Best Possible Solution

Land a focused CLI publish fix that preserves the required root manifest despite broad ignore patterns while still rejecting plugin folders and avoiding upload of arbitrary ignored files, then use the regression test to guard the linked publish flow.
