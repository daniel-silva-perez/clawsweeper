## Summary

Keep this draft PR open. Current main still does not implement the PR's intended root-manifest recovery: `cmdPublish` still depends directly on `listTextFiles(folder)`, and `listTextFiles` applies ignore rules before files reach publish validation, so a broad markdown ignore can still remove the required root manifest. The PR remains a focused implementation candidate for the linked #1509 bug.

## Best Possible Solution

Keep this draft PR open for maintainer review. The likely path is to land or adapt the localized `cmdPublish` root-manifest fallback, preserve generic `listTextFiles` ignore semantics, add the broad markdown ignore regression test, decide explicit ignore policy for the required manifest, and close #1509 only after the fix merges and ships.
