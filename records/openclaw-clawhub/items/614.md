## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still leaves the reported server-error path plausible: the public profile route calls users.getByHandle, which can still hit .unique() lookups on handle indexes that are not schema-enforced unique. Related #1179 is adjacent and open, but it does not cover the public profile lookup failure.

## What This Changes

The issue reports that https://clawhub.ai/u/jisuapi renders a Convex users:getByHandle server error, suspected to stem from a same-name GitHub account conflict.

## Best Possible Solution

Verify the reported production request ID and the jisuapi rows across users, publishers, and authAccounts, then either repair/merge the conflicting identity data or add a deterministic, privacy-safe duplicate-tolerant handle lookup with regression coverage for duplicate users.handle and publishers.by_handle cases.
