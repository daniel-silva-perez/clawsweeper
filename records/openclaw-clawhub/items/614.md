## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a plausible remaining failure path: /u/$handle calls users.getByHandle, which resolves handles with Convex .unique() over ordinary handle indexes, so duplicate or conflicting production identity rows can still throw the reported server-error class. Related #1179 is open and adjacent, but it does not supersede this public profile lookup issue.

## What This Changes

The issue reports that the public profile URL for handle jisuapi returns a Convex users:getByHandle server error, suspected to involve a conflicting GitHub account or duplicate handle data.

## Best Possible Solution

Keep this issue open for maintainer production verification of the reported request ID and handle across users, publishers, and auth account rows. The best fix is either targeted data cleanup/merge for duplicate identity or publisher rows, or a small backend change that makes public handle lookup duplicate-tolerant in a deterministic, privacy-safe way, with regression tests for duplicate users.handle and publishers.by_handle cases and coordination with #1179.
