## Summary

Keep open. The repository cleanup policy keeps ClawHub issues open, and current main still leaves the reported profile server-error path plausible: the public profile route calls users.getByHandle, which delegates into .unique() lookups on handle indexes that are not schema-enforced unique. Related #1179 is adjacent and open, but it covers first-time publisher failures rather than this public profile lookup.

## What This Changes

The issue asks ClawHub to fix a public user profile URL that reports a Convex users:getByHandle server error, likely from conflicting GitHub or handle identity data.

## Best Possible Solution

Verify the production request ID and jisuapi identity rows across users, publishers, and auth-linked data, then either repair the conflicting records or add deterministic, privacy-safe duplicate-tolerant handle resolution with regression coverage for duplicate users.handle and publishers.by_handle cases.
