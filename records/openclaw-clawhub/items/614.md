## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still leaves the reported public profile server-error path plausible: the profile route calls `users.getByHandle`, which delegates to `.unique()` handle lookups on ordinary Convex indexes without duplicate-tolerant handling.

## What This Changes

The issue reports that `https://clawhub.ai/u/jisuapi` returns a Convex `users:getByHandle` server error, possibly due to a same-name GitHub account or handle collision.

## Best Possible Solution

Verify the production request ID and `jisuapi` identity rows across users, publishers, and auth-linked data, then either repair or merge the conflicting records, or add deterministic, privacy-safe duplicate-tolerant handle resolution with regression tests for duplicate `users.handle` and `publishers.by_handle` cases.
