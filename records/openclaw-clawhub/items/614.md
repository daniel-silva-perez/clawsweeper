## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still leaves the reported public profile server-error path plausible: the `/u/$handle` route calls `users.getByHandle`, which delegates into duplicate-sensitive `.unique()` handle lookups on ordinary Convex indexes. Related #1179 is adjacent identity/publisher work, but it tracks first-time publish/profile-save failures rather than this public profile lookup failure.

## What This Changes

The issue reports that `https://clawhub.ai/u/jisuapi` fails with a Convex `users:getByHandle` server error, likely around a GitHub handle/account collision.

## Best Possible Solution

Verify the production request ID and `jisuapi` identity rows across users, publishers, and auth-linked data, then either repair or merge the conflicting records, or add deterministic, privacy-safe duplicate-tolerant handle resolution with regression tests for duplicate `users.handle` and `publishers.by_handle` cases.
