## Summary

Keep open. Repository policy for openclaw/clawhub says issues should remain open in this cleanup pass, and current main still matches the reported failure mode: /skills uses the embedding-first search action, exact slug injection only covers exact slug-like matches, lexical fallback is bounded to recent skillSearchDigest windows, and the skill digest schema has no normalized indexed display-name prefix path. The tokenizer itself is case-insensitive, so the observed ScienceClaw/scienceclaw difference is most likely a candidate-retrieval gap rather than token matching.

## Best Possible Solution

Keep this issue open and either fix it directly or consolidate it with #1254. The likely code path is to add denormalized normalized skill name/slug fields to skillSearchDigest, backfill them, add indexed active/non-suspicious prefix lookups, query that direct lexical path before vector search, merge/rank with vector results, and add regression tests proving lowercase and camel-case ScienceClaw queries return the same direct-name matches.
