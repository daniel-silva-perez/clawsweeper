## Summary

Keep open. The issue is still substantively valid: a live API check still returns fewer ScienceClaw results for lowercase `scienceclaw` than for `ScienceClaw`, and current main still lacks an indexed normalized skill slug/display-name prefix lookup. ClawHub cleanup policy also keeps issues open; the existing open #1256 is the better implementation path to review or refresh.

## What This Changes

The issue requests stable, case-insensitive `/skills` search so lowercase and camel-case direct skill name queries return the same matching ScienceClaw skills.

## Best Possible Solution

Land #1256 or an equivalent current-main patch that adds normalized slug/display-name fields to `skillSearchDigest`, indexed active/non-suspicious prefix lookup before vector ranking, a cursor-safe backfill, and regression tests covering both #1254 and the ScienceClaw lowercase/camel-case result set.
