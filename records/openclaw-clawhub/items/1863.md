## Summary

Keep open. This is a ClawHub issue, and the supplied repository policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. The current source also still supports the reported bug: /skills query mode calls the embedding-first search action, direct lexical recall is limited to exact slugs plus bounded recent digest scans, and skillSearchDigest has no normalized indexed display-name prefix path. Related #1254 remains open and tracks the same direct name/slug recall gap.

## Best Possible Solution

Keep this issue open and either fix it directly or consolidate it with open #1254. The likely implementation is to denormalize normalized skill name and slug fields onto skillSearchDigest, backfill them, add indexed active/non-suspicious prefix lookup, merge those direct lexical candidates before vector-ranked results, and add regression coverage proving scienceclaw and ScienceClaw return the same ScienceClaw-prefixed result set.
