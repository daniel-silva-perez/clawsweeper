Keep open. This is a ClawHub issue, and the cleanup policy only permits auto-closing ClawHub pull requests that current main certainly implements. Substantively, current main still lacks a direct normalized skill slug/display-name prefix recall path: skill search is exact-slug plus vector recall plus a bounded recent-digest lexical fallback, while open PR #1256 already proposes the same normalized indexed prefix fix for related #1254.

Keep this issue open and consolidate it with the existing search-prefix work: review, refresh, and land PR #1256 or an equivalent current-main patch that adds an indexed normalized direct lexical path for skill slug/display-name prefix matches before vector recall gates the result set.
