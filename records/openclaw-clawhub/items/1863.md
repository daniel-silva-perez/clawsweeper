## Summary

Keep this issue open. The ClawHub cleanup policy for this repository only permits auto-closing pull requests that are certainly implemented on current main, not issues. Source review also supports this as a plausible live recall bug: /skills routes query searches through the embedding-first skill search action, and current main still lacks an indexed normalized skill display-name prefix lookup that would guarantee case-insensitive direct name recall.

## Best Possible Solution

Keep open and fix or consolidate with #1254. The likely implementation is to add denormalized normalized skill name/search fields to skillSearchDigest, index them for case-insensitive prefix lookup, query that direct lexical path before vector search, merge and rank with vector results, and add regression coverage proving scienceclaw and ScienceClaw return the same ScienceClaw-prefixed result set.
