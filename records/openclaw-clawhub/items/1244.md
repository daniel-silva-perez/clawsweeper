## Summary

Keep this issue open. It is a ClawHub issue, and the repository cleanup policy only permits auto-close for PRs that are certainly implemented on current main. Current main still treats user-initiated account deletion as irreversible deactivation: the deletion mutation writes deactivated/purged state, OAuth completion rejects deleted/deactivated users, active-user helpers exclude them, tests assert blocked reauthentication, and docs/UI warn that deletion cannot be undone. That means the reporter's request for purge/re-registration support is still a real account lifecycle/support item rather than completed cleanup.

## Best Possible Solution

Keep this open for maintainer triage of the account deletion and GitHub OAuth lifecycle. Maintainers should decide whether self-deleted accounts are permanently blocked or whether same-GitHub re-registration should be supported. If recovery is intended, add a safe backend/admin repair path to unlink or supersede the old Convex Auth binding, cover it with auth/account lifecycle tests, and provide a process for already-stuck users. If irreversible deletion is intended, make the hosted login/support flow clearly surface that state and document the support policy.
