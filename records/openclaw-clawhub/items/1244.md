## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open unless a strong close reason applies, and current main still implements user-initiated account deletion as permanent deactivation: deletion writes deactivated/purged state, OAuth sign-in rejects deactivated users, active-user helpers exclude deleted/deactivated users, and tests/docs assert that deleted accounts cannot be restored by logging in again. The user request is a real account lifecycle/support question, not completed cleanup.

## Best Possible Solution

Keep this open for maintainer triage of the account deletion and GitHub OAuth lifecycle. Maintainers should decide whether self-deleted accounts are permanently blocked or whether same-GitHub re-registration should be supported. If recovery is intended, add a safe backend/admin support path to unlink or supersede the old Convex Auth binding, cover it with auth/account lifecycle tests, and provide a repair process for already-stuck users. If irreversible deletion is intended, make the hosted login/support flow clearly surface that state and document the support policy.
