## Summary

Keep this issue open. Current main still treats user-initiated account deletion as terminal deactivation: OAuth completion rejects deactivated users, active-user helpers exclude those records, admin unban is not a restore path, and docs/UI/tests now describe deletion as permanent. The code has better auth-error surfacing, but it does not implement the requested purge, identity unlink, restore, or clean re-registration flow, and ClawHub policy keeps issues open for maintainer handling.

## What This Changes

The issue requests recovery or purge support for a GitHub OAuth account that cannot re-register after user-initiated ClawHub account deletion.

## Best Possible Solution

Keep this open for maintainer support plus product/security handling. Maintainers should verify the reporter controls the GitHub identity, inspect the production Convex Auth authAccounts binding and linked users record, then decide whether deletion remains irreversible or whether ClawHub should support an audited restore, GitHub identity unlink, or clean re-registration flow. If recovery is approved, implement it in the Convex auth/users/admin path with tests and docs; if not, make the terminal state and support route explicit in web and CLI login flows.
