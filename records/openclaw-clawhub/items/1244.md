## Summary

Keep this open. Current main still intentionally treats user-initiated ClawHub account deletion as terminal, rejects subsequent GitHub OAuth sign-in for deleted/deactivated users, and has no self-service purge, restore, unlink, or clean re-registration path. The remaining action is production account support plus account-lifecycle/security policy, and the ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

The reporter requests a purge or recovery path for GitHub user lnastri25 after deleting a ClawHub account because GitHub OAuth completes but ClawHub never establishes a session.

## Best Possible Solution

Maintainers should decide the account lifecycle policy, then either add an audited recovery or re-registration path that safely restores/unlinks the GitHub identity binding with support tooling, tests, and docs, or keep deletion irreversible while clearly surfacing the terminal state and support route in web and CLI login flows.
