## Summary

Keep this open. Current main still treats user-initiated ClawHub account deletion as terminal during GitHub OAuth, and this is an open ClawHub issue where repository policy only permits cleanup auto-close for pull requests that are certainly implemented on main. The remaining action is production account support plus account-lifecycle/security policy, not a conservative source cleanup close.

## What This Changes

The issue asks maintainers to recover a GitHub OAuth login loop after account deletion by purging or unlinking the reporter's stale ClawHub/GitHub identity record so they can re-register.

## Best Possible Solution

Maintainers should decide the account lifecycle policy, then either add an audited recovery or re-registration path that safely restores or unlinks the GitHub identity binding with tests and support tooling, or keep deletion irreversible while making the terminal state and support route clear in web and CLI login flows.
