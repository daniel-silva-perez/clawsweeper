## Summary

Keep this issue open. The repository-specific cleanup policy for openclaw/clawhub does not allow auto-closing issues, and current main still treats user-initiated deletion as irreversible instead of providing the requested purge, unlink, or same-GitHub re-registration path. The current code improves the failure message, but it still blocks OAuth completion for deleted/deactivated users.

## Best Possible Solution

Keep this issue open for maintainer triage of the account-deletion and GitHub OAuth lifecycle. Maintainers should decide whether self-deleted accounts are permanently blocked or whether same-GitHub re-registration/recovery is supported. If recovery is intended, add a safe backend/admin path to unlink or supersede the old Convex Auth binding, cover it with account lifecycle tests, and document the support process. If irreversible deletion is intended, make the hosted login flow and support docs clearly surface that state.
