## Summary

Keep this open. Current main still treats user-initiated ClawHub account deletion as terminal: deletion writes deactivation fields, OAuth completion rejects deleted/deactivated users, active-user helpers exclude those records, and docs/UI describe the state as irreversible. Because this is an open ClawHub issue and the remaining action involves production account support plus account-lifecycle/security policy, it should not be auto-closed by conservative cleanup.

## What This Changes

The issue asks maintainers to purge or otherwise recover a deleted ClawHub account record so the same GitHub user can register and sign in again after account deletion.

## Best Possible Solution

Maintainers should decide the account lifecycle policy, then either add an audited recovery or re-registration path that safely restores or unlinks the GitHub identity binding with tests and support tooling, or keep deletion irreversible while making the terminal state and support route clear in web and CLI login flows.
