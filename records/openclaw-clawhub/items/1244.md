## Summary

Keep this issue open. Current main still treats user-initiated account deletion as irreversible deactivation: deletion writes deactivated/purged state, OAuth completion rejects deleted/deactivated users instead of creating a fresh account for the same GitHub identity, active-user helpers exclude those users, and tests/docs assert that behavior. The supplied GitHub context also shows several related deleted-account OAuth reports still open, so this needs maintainer product/support triage rather than cleanup closure.

## Best Possible Solution

Keep this open for maintainer triage of the deleted-account and GitHub OAuth lifecycle. Maintainers should decide whether self-deleted accounts are permanently blocked or whether same-GitHub re-registration/recovery is supported. If recovery is intended, add a safe audited backend/admin repair path to unlink or supersede the old Convex Auth binding, cover it with auth/account lifecycle tests, and document a support process for already-stuck users. If irreversible deletion is intended, make the hosted login/support flow clearly surface that state and document where affected users should request manual help.
