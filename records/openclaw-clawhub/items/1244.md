## Summary

Keep this issue open. The repository-specific cleanup policy says openclaw/clawhub issues should not be auto-closed by this pass, and current main still intentionally blocks reauth after user-initiated deletion rather than implementing a purge, unlink, or same-GitHub re-registration path.

## Best Possible Solution

Keep this issue open for maintainer triage of the account-deletion and GitHub OAuth lifecycle. Maintainers should decide whether self-deleted accounts are permanently blocked or whether same-GitHub re-registration/recovery is supported. If recovery is intended, add a safe backend/admin support path to unlink or supersede the old Convex Auth binding, cover it with auth/account lifecycle tests, and document the repair process for already-stuck users. If irreversible deletion is intended, make the hosted login/support flow clearly surface that state and document the support policy.
