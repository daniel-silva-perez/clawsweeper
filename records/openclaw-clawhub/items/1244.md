## Summary

Keep open. Current main still treats user-initiated ClawHub account deletion as terminal: deletion writes deactivation/purge state, OAuth completion rejects deleted or deactivated users, active-user helpers exclude those records, and tests/docs assert the blocked reauth behavior. Because this is an open ClawHub issue and the remaining action involves production account support plus account-lifecycle/security policy, conservative cleanup should not close it.

## What This Changes

The reporter asks maintainers to purge or reset the ClawHub record for GitHub user lnastri25 so GitHub OAuth can create an active session after a prior account deletion.

## Best Possible Solution

Maintainers should decide the account lifecycle policy, then either add an audited recovery or re-registration path that safely restores or unlinks the GitHub identity binding with support tooling, tests, and docs, or keep deletion irreversible while making the terminal state and support route clear in web and CLI login flows.
