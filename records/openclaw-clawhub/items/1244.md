## Summary

Keep this issue open. The ClawHub cleanup policy only permits auto-closing PRs that are certainly implemented on current main, and this is an issue. Substantively, current main still treats user-initiated account deletion as irreversible: account deletion writes deactivation/purge state, OAuth completion rejects deactivated users, tests assert rejection, and docs say deleted accounts cannot be restored by logging in again. The requested purge/re-registration path for the same GitHub identity is still a maintainer support/product decision, not implemented cleanup.

## Best Possible Solution

Keep this open for maintainer triage of the account deletion and GitHub OAuth lifecycle. Maintainers should decide whether self-deleted accounts are permanently blocked or whether same-GitHub re-registration should be supported. If recovery is intended, add a safe backend/admin support path to unlink or supersede the old Convex Auth binding, cover it with auth/account lifecycle tests, and provide a repair process for already-stuck users. If irreversible deletion is intended, make the hosted login/support flow clearly surface that state and document the support policy.
