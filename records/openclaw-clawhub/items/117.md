## Summary

Current main has backend, HTTP API, docs, and CLI support for owner skill soft-delete/restore, but the signed-in web app still does not expose a discoverable owner delete/restore workflow. Because this is an issue in openclaw/clawhub, repository policy also requires keeping it open rather than auto-closing issues.

## Best Possible Solution

Keep this issue open and add a signed-in owner delete/restore workflow in the web app. The implementation should reuse existing soft-delete semantics, derive identity server-side, authorize direct owners and publisher-member ownership consistently with the current settings page, expose a safe confirmation UI from skill settings and/or dashboard row actions, and provide an owner-visible restore path without adding hard-delete behavior.
