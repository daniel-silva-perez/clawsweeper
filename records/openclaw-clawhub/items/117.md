## Summary

Keep this issue open. Current main has backend, HTTP API, CLI, docs, and tests for skill soft-delete/restore, but the signed-in web app still does not expose a discoverable owner delete/restore workflow, and this repository policy keeps ClawHub issues open for maintainer follow-up.

## What This Changes

Skill authors should be able to delete or restore their own published ClawHub skills.

## Best Possible Solution

Keep this issue open and add a signed-in owner delete/restore workflow in the web app. Reuse the existing soft-delete semantics, derive identity server-side, authorize direct owners and publisher-member ownership consistently with the current settings page, expose a confirmation UI from skill settings and/or dashboard row actions, and provide an owner-visible restore path without adding hard-delete behavior.

## Repair Work Prompt

Implement a signed-in web owner delete/restore workflow for skills. Reuse existing soft-delete semantics; do not add hard-delete behavior or broaden moderator/admin tooling. Add or adapt Convex functions so identity is derived server-side and authorization permits the direct owner plus publisher members consistently with SkillDetailPage/SkillOwnershipPanel ownership checks.
