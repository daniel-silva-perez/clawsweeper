## Summary

Keep this issue open. Current main has owner soft-delete/restore primitives in the backend, HTTP API, CLI, docs, and tests, but the signed-in web app still lacks a discoverable owner delete/restore workflow and the dashboard cannot surface soft-deleted skills for restoration.

## What This Changes

Users want skill authors to be able to delete or restore their own published skills from ClawHub.

## Best Possible Solution

Add a signed-in owner web workflow that reuses the existing soft-delete semantics, offers confirmation for deletion, lets owners find and restore soft-deleted skills, and keeps API/CLI behavior consistent.
