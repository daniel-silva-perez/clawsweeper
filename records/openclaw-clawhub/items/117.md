## Summary

Keep this issue open. Current main has partial coverage through owner-authorized soft-delete/restore in the API and ClawHub CLI, plus staff management controls, but the ordinary signed-in web app still lacks a discoverable owner delete/restore workflow. The repository policy for openclaw/clawhub also says issues should stay open; only pull requests that are certainly implemented on main are auto-close candidates.

## Best Possible Solution

Keep this issue open and add a signed-in owner delete/restore workflow in the web app, likely in the skill owner panel and dashboard row actions. Reuse the existing owner-authorized soft-delete logic through a public owner mutation/action that derives identity server-side, and add an include-soft-deleted owner list or restore view so authors can recover hidden skills. Keep the API/CLI workaround documented until the UI exists.
