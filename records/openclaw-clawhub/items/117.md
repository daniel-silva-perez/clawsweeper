## Summary

Keep this issue open. Current main has soft-delete/restore support through the backend, HTTP API, CLI, docs, and tests, but the signed-in web app still lacks a discoverable owner delete/restore workflow and normal owner lists filter deleted skills away.

## What This Changes

Skill authors should be able to delete or remove their own published ClawHub skills.

## Best Possible Solution

Keep this issue open and add a signed-in owner delete/restore workflow in the web app. Reuse existing soft-delete semantics, derive identity server-side, authorize direct owners and publisher-member ownership consistently, expose confirmation UI from skill settings and/or dashboard row actions, and provide an owner-visible restore path without adding hard-delete behavior.
