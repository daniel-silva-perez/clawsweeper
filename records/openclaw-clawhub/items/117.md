## Summary

Keep this issue open. Current main supports owner/moderator/admin soft-delete through the v1 API and CLI, but the signed-in web app still does not provide a discoverable owner delete or restore workflow, and ClawHub cleanup policy says issues are not auto-close candidates in this pass.

## Best Possible Solution

Keep this issue open and add a signed-in owner delete/restore workflow in the web app. The fix should reuse the existing soft-delete semantics, authorize from server-side identity, support both direct owner and publisher-member ownership consistently with the settings page, expose a safe confirmation UI from skill settings and/or dashboard row actions, and provide an owner-visible restore path for soft-deleted skills without changing hard-delete policy.
