## Summary

Keep this issue open. Current main has owner-authorized soft-delete/restore through the HTTP API and clawhub CLI, but the ordinary signed-in web UI still does not expose a direct owner delete/restore workflow. The ClawHub cleanup policy for this review also keeps issues open; only pull requests that are certainly implemented on main are close candidates.

## Best Possible Solution

Keep the issue open and add a discoverable signed-in owner soft-delete/restore workflow in the web app, likely in the skill owner panel and dashboard row actions. Reuse the existing owner-authorized soft-delete logic through a public owner mutation/action that derives identity server-side, and add an include-soft-deleted owner list path so restore is possible from the dashboard. Keep the CLI/API workaround documented until the UI exists.
