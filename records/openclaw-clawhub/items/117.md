## Summary

Keep this issue open. Current main supports owner/moderator/admin skill soft-delete and restore through the v1 API and CLI, but the signed-in web app still does not expose a discoverable owner delete/restore workflow. The repository policy for ClawHub also says issues are not auto-close candidates in this pass.

## Best Possible Solution

Keep this issue open and add a signed-in owner delete/restore workflow in the web app. The fix should reuse the existing soft-delete semantics, derive identity server-side, authorize both direct owner and publisher-member ownership consistently with SkillDetailPage settings access, expose a safe confirmation UI from skill settings and/or dashboard row actions, and provide an owner-visible restore path for soft-deleted skills without introducing hard-delete behavior.
