## Summary

Keep this issue open. Current main has owner-authorized API/CLI soft-delete and undelete, but the signed-in web app still lacks a discoverable owner delete/restore workflow; the ClawHub cleanup policy also says issues are not auto-close candidates in this pass.

## Best Possible Solution

Keep this issue open and add a signed-in owner delete/restore workflow in the web app, likely from the skill settings panel and dashboard row actions. Reuse server-side identity for authorization, call the existing owner-authorized soft-delete behavior, and add an owner-visible include-deleted or restore view so authors can recover hidden skills. Keep the API/CLI workaround documented until the UI exists.
