## Summary

Keep this issue open. Current main has owner-authorized soft-delete and restore through the API/CLI, but the signed-in web app still lacks a discoverable owner delete/restore workflow, and the repository cleanup policy for openclaw/clawhub keeps issues open during this pass.

## Best Possible Solution

Keep this issue open and add a signed-in owner delete/restore workflow in the web app, likely from the skill owner tools panel and dashboard row actions. Reuse the existing owner-authorized soft-delete logic through a public owner mutation that derives identity server-side, and add an owner-visible include-deleted or restore view so authors can recover hidden skills. Keep the API/CLI workaround documented until the UI exists.
