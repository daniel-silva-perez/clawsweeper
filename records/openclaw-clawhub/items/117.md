## Summary

Keep this issue open. Current main supports owner soft-delete and restore through backend, HTTP API, CLI, docs, and tests, but the signed-in web app still lacks a discoverable owner delete/restore workflow, and the owner dashboard/list path filters soft-deleted skills before they can be restored there. The issue also remains active through follow-up user comments and related open requests for skill removal.

## What This Changes

The issue requests that ClawHub skill authors be able to delete their own published skills.

## Best Possible Solution

Add a signed-in owner web workflow that reuses the existing soft-delete semantics, confirms deletion, lets owners find and restore soft-deleted skills, and keeps API/CLI behavior consistent.
