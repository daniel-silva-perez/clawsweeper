## Summary

Keep this issue open. Current main has owner-authorized API and CLI soft-delete/restore paths, plus staff moderation delete/restore UI, but the ordinary owner-facing web UI still does not expose a direct delete/restore action. The ClawHub cleanup policy also says issues are not auto-close candidates in this workflow.

## Best Possible Solution

Keep this issue open and add a discoverable owner-facing delete/restore path, likely in the skill owner panel and dashboard row actions. Back it with the existing soft-delete flow or a public owner mutation, and keep the CLI/API workaround documented until the web UI exists.
