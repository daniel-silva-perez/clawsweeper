## Summary

Keep this issue open. Current main has owner-authorized API/CLI soft-delete and restore paths, plus staff moderation delete/restore UI, but the owner-facing web surfaces still do not expose a direct delete/restore action for a user’s own skill. Repository policy for openclaw/clawhub also says issues should remain open in this cleanup workflow.

## Best Possible Solution

Keep this issue open and add a discoverable owner-facing delete/restore path, likely in the skill owner panel and dashboard row actions. Back it with the existing soft-delete flow or a public owner mutation, and keep the API/CLI workaround documented until the web UI exists.
