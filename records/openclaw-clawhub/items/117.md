## Summary

Keep this issue open. Current main already has backend, HTTP API, docs, and CLI soft-delete/restore support, but the signed-in web app still does not expose a discoverable owner delete/restore workflow; ClawHub policy also says issues should not be auto-closed by this review.

## Best Possible Solution

Keep this issue open and add a signed-in owner delete/restore workflow in the web app. Reuse the existing soft-delete semantics, derive identity server-side, authorize direct owners and publisher-member ownership consistently with the current settings page, expose a confirmation UI from skill settings and/or dashboard row actions, and provide an owner-visible restore path without adding hard-delete behavior.
