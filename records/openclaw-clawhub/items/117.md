## Summary

Keep this issue open. Current main has backend, HTTP API, CLI, docs, and tests for owner soft-delete/restore, but the signed-in web app still does not expose a discoverable owner delete/restore workflow and the dashboard filters soft-deleted skills before owners can restore them.

## What This Changes

The issue asks for skill authors to be able to delete their own published skills.

## Best Possible Solution

Add a signed-in web delete/restore workflow that reuses existing soft-delete semantics, derives identity server-side, authorizes direct owners and publisher members consistently, exposes confirmation UI from settings and/or dashboard row actions, and lets owners find and restore soft-deleted skills.
