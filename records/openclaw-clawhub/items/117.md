## Summary

Keep this issue open. Current main has backend, HTTP API, CLI, docs, and tests for owner soft-delete/restore, but the signed-in web app still lacks a discoverable owner delete/restore workflow, and owner list queries filter soft-deleted skills before they can be restored from the dashboard.

## What This Changes

Skill authors should be able to delete or remove their own published ClawHub skills.

## Best Possible Solution

Add a signed-in owner web workflow that reuses existing soft-delete semantics, confirms deletion, lets owners find and restore soft-deleted skills, and keeps existing API/CLI behavior consistent.
