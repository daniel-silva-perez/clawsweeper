## Summary

Keep this issue open. Current main supports owner soft-delete/restore through backend, HTTP API, CLI, docs, and tests, but the signed-in web app still has no discoverable owner delete/restore workflow and the owner dashboard filters soft-deleted skills out before rendering them.

## What This Changes

Users are asking for skill authors to be able to delete their own published ClawHub skills.

## Best Possible Solution

Add a signed-in web delete/restore workflow that reuses existing soft-delete semantics, derives identity server-side, authorizes direct owners and publisher members consistently, exposes confirmation UI from settings and/or dashboard row actions, and lets owners find and restore soft-deleted skills.
