## Summary

Keep this issue open. The repository-specific cleanup policy for this review only permits auto-closing ClawHub pull requests that are certainly implemented on main, and this item is an issue. Substantively, current main still has a real owner-facing diagnostics gap: quality-hidden skills are intentionally excluded from public search, but `clawhub inspect` does not expose the moderation reason or the dedicated moderation endpoint that would explain `quality.low` quarantine to the publisher.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely product fix is to expose existing owner-visible moderation state through `clawhub inspect` and the authenticated skill detail UI, including `quality.low` or hidden-status recovery guidance, while coordinating the stale rendered SKILL.md/tag symptoms with open issue #1407. Public search should continue excluding genuinely hidden skills unless maintainers deliberately change that moderation policy.
