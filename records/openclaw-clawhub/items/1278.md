## Summary

Keep this issue open. The ClawHub cleanup policy for this repository only allows automated closure for pull requests that are definitely implemented on main, and this item is an issue. Substantively, current main still supports the reported moderation/search invisibility behavior: quality-hidden skills are excluded from public search, while the CLI inspect path still does not surface the owner-visible moderation reason well enough for publishers to diagnose why their skill disappeared. The stale tag/SKILL.md part remains related to open issue #1407 rather than making this report closeable.

## Best Possible Solution

Keep this issue open for maintainer triage. The practical path is to add owner-facing visibility diagnostics to clawhub inspect and/or the authenticated skill detail view, including hidden status, quality.low or other moderation reason, and concrete recovery guidance. The stale tag/rendered SKILL.md portion should be coordinated with open issue #1407 while preserving public search filtering for hidden skills.
