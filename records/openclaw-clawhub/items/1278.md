## Summary

Keep this issue open. The ClawHub cleanup policy for this repository only allows automated closure for pull requests that are definitely implemented on main, and this item is an issue. The current code also still has a real owner-facing diagnostic gap around quality-hidden skills: public search properly drops hidden skills, but `clawhub inspect` does not surface the underlying moderation status or legacy `quality.low` reason in its normal output.

## Best Possible Solution

Keep this issue open for maintainer triage. The best product path is to make publisher-facing visibility diagnostics explicit in `clawhub inspect` and/or authenticated skill detail views, including hidden status, `quality.low` or other moderation reasons, and recovery steps. Keep public search excluding hidden skills, and coordinate the stale tag/rendered SKILL.md portion with #1407.
