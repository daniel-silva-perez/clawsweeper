## Summary

Keep this issue open. The repo-specific cleanup policy only permits auto-closing ClawHub pull requests, not issues, and current main still preserves the core behavior the reporter identified: quality-hidden skills are excluded from public search. Owner-facing diagnostics have improved in the dashboard and API moderation endpoint, but `clawhub inspect` still does not present the hidden reason/recovery path in its normal output, and the stale rendered content/tags symptom is still covered by open #1407.

## Best Possible Solution

Keep this issue open for maintainer triage. The strongest path is to add owner-authenticated diagnostics to `clawhub inspect` and any corresponding API/schema output: show when a caller-owned skill is hidden, surface `quality.low` or the relevant moderation reason, and include concrete recovery guidance while preserving public search filtering. Coordinate the stale rendered SKILL.md/tags symptom with #1407.
