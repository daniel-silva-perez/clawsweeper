## Summary

Keep open. Current main still treats user-initiated deletion as irreversible and rejects later GitHub OAuth sign-in for the same deleted/deactivated user record. This is also an open ClawHub issue, and the repository cleanup policy only allows auto-closing PRs that are certainly implemented on main.

## What This Changes

The issue asks maintainers to restore, hard-delete, or unbind a deleted ClawHub account's GitHub identity so the same GitHub user can log in and publish skills again.

## Best Possible Solution

Define the intended account lifecycle explicitly: either keep deletion irreversible with a documented support-only recovery process, or add a narrow audited restore/unbind flow that verifies GitHub identity ownership, preserves moderation protections, and covers web plus CLI login behavior with tests.
