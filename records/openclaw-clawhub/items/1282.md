## Summary

Keep open. Current main still implements irreversible self-deletion and rejects the next GitHub OAuth sign-in for the same linked account, so the reporter’s requested restore/unbind path remains unresolved. This is also a ClawHub issue, and the repository cleanup policy reserves auto-close for PRs that are already implemented on main.

## What This Changes

The issue asks maintainers to unblock or restore a GitHub-linked ClawHub account after user-initiated account deletion prevents re-registration or login.

## Best Possible Solution

Define the intended account lifecycle explicitly: either keep deletion irreversible with a documented support-only recovery process, or add a narrow audited restore/unbind flow that verifies GitHub identity ownership, preserves moderation protections, and covers web plus CLI login behavior with tests.
