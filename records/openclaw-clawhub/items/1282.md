## Summary

Keep open. Current main still implements user-initiated deletion as a permanent deactivation, rejects later GitHub OAuth sign-in for that linked user, and has no restore, hard-delete, or GitHub auth unbind path. The ClawHub cleanup policy also says open issues should stay open unless they are PRs already implemented on main, and several related account-deletion login reports remain open.

## What This Changes

The issue asks maintainers to unbind, hard-delete, or restore a GitHub-linked ClawHub account after self-deletion so the same GitHub profile can log in again.

## Best Possible Solution

Define the account lifecycle explicitly: either keep deletion irreversible with a documented support-only recovery process, or add a narrow audited restore/unbind flow that verifies GitHub identity ownership, preserves moderation protections, and covers web plus CLI login behavior with tests.
