## Summary

Keep this issue open. The ClawHub cleanup policy for this repository keeps issues open, and current main still matches the reported account lifecycle problem: user deletion is stored as irreversible deactivation, OAuth completion rejects deactivated accounts, active-user helpers reject deactivated users, and the admin unban path explicitly excludes permanently deleted accounts. Related deleted-account/account-block reports #1620 and #1674 are also still open.

## Best Possible Solution

Keep this open for maintainer support and product-policy handling. The best path is either a verified production recovery/unbind/hard-delete procedure for this GitHub identity, or a safe documented restoration/re-registration flow that updates the user record, Convex Auth GitHub binding, publishers, retained skills, API tokens, telemetry, audit logs, and ban protections consistently. Coordinate with related open reports #1620 and #1674.
