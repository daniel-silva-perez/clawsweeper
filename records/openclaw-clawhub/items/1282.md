## Summary

Keep open. The reported account-recovery problem still matches current main: self-deletion deactivates the user, later GitHub OAuth is rejected, and there is no restore, re-registration, or auth unbind path. ClawHub cleanup policy also keeps issues open unless they are PRs already implemented on main.

## What This Changes

A deleted-account user asks ClawHub to unbind or hard-delete their GitHub identity so they can re-register or log in again to publish and sync skills.

## Best Possible Solution

Define the intended account lifecycle explicitly: keep deletion irreversible with a documented support-only recovery process, or add a narrow audited restore/unbind flow that verifies GitHub identity ownership, preserves moderation protections, and covers web plus CLI login behavior with tests.
