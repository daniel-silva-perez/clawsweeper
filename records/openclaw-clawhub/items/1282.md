## Summary

Keep this issue open. The current main branch still intentionally treats user-initiated account deletion as irreversible: deletion writes `deactivatedAt`/`purgedAt`, GitHub OAuth reauth rejects deleted/deactivated users, active-session helpers exclude those users, admin unban refuses permanently deleted accounts, and the documented restore API is for restoring skills from backup rather than unbinding or restoring the account identity. ClawHub cleanup policy also says open issues should stay open unless maintainers handle them manually.

## What This Changes

The issue requests manual unbinding, hard deletion, or restoration of a deleted ClawHub account so the same GitHub identity can register or log in again.

## Best Possible Solution

Keep this open for maintainer support and product/security triage. Maintainers should verify the reporter controls the GitHub identity, inspect production Convex Auth `authAccounts` and the linked user record, then choose an approved support action or explicitly confirm that deleted accounts cannot be restored or re-registered. If a code change is approved, it should define a narrow recovery or unbind flow that preserves ban protections and covers web and CLI login behavior with tests.
