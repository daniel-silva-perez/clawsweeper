## Summary

Keep this issue open. Current main still treats user-initiated deletion as irreversible deactivation, OAuth sign-in rejects deleted/deactivated users, active-user helpers exclude those records, and there is no discovered re-registration/unbind/restore path. The ClawHub cleanup policy also keeps issues open; this belongs to maintainer support and product/security handling, especially because related deleted-account login reports remain open.

## Best Possible Solution

Keep this open for maintainer support and product/security handling. Verify control of the GitHub identity and inspect the production Convex Auth binding/user record, then either perform an approved recovery/unbind/hard-delete operation or document that deleted accounts cannot be restored. Any code change should be coordinated with the related deleted-account login cluster and preserve ban protections.
