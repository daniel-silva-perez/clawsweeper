## Summary

Keep this issue open. The repository-specific ClawHub cleanup policy only allows automated closure for pull requests that are certainly implemented on current main, and this item is an issue. Substantively, current main still treats user-initiated deletion as irreversible deactivation that blocks GitHub reauth/re-registration; I did not find a user or admin path that unbinds the GitHub identity or restores the account login. Related deleted-account/login-block reports such as #1620 and #1674 remain open.

## Best Possible Solution

Keep this open for maintainer support and product-policy handling. The best path is either a verified production recovery/unbind/hard-delete procedure for the reporter's GitHub identity, or a safe documented restoration/re-registration flow that updates the user record, Convex Auth GitHub binding, retained skills/publishers, API tokens, audit logs, and ban protections consistently. Coordinate with related open deleted-account and account-block reports such as #1620 and #1674 rather than closing this thread as resolved.
