## Summary

Keep open. This is a ClawHub issue requesting hosted cleanup for the `xl-weekly-report-email` skill, not a PR. Current main supports deleting non-`latest` tags through owner/moderator UI and mutation paths, but I did not find a current selected historical-version deletion workflow exposed in source or docs. The repository policy for this review also says ClawHub issues should stay open; only PRs that are certainly implemented on main are auto-close candidates.

## Best Possible Solution

Keep this issue open for maintainer or service-operator handling. Remove any non-`latest` `v1.8.1` tags through the existing owner/moderator tag-management path, then handle the historical-version cleanup with a controlled hosted-data/admin process. If this should become self-service or staff tooling, add a dedicated selected-version soft-delete/admin workflow with audit logging and safeguards for latest pointers, tags, storage, fingerprints, downloads, and stats.
