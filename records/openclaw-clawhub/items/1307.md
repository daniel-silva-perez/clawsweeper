Keep open. This is an open ClawHub issue requesting hosted data cleanup for one published skill. Current main supports owner/moderator deletion of non-latest tags, including historical tags, but I did not find a selective historical skill-version deletion workflow exposed in source or docs. Repository policy for ClawHub also says to keep issues open and only auto-close PRs that are certainly implemented on main.

## Best Possible Solution

Keep this issue open for maintainer or service-operator handling. Remove the requested non-latest v1.8.1 tags through the existing owner/moderator tag-management path, and either perform the historical version cleanup as a controlled hosted-data operation or add an explicit selected-version soft-delete/admin workflow that safely updates tags, latest pointers, storage/fingerprint/search state, stats, and audit logs.
