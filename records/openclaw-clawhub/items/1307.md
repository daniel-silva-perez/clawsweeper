## Summary

Keep this issue open. It asks for hosted cleanup of one published skill: deleting historical versions before the current latest and removing v1.8.1 tags. Current main supports owner/moderator tag deletion and admin whole-skill hard deletion, but I did not find a selected historical-version deletion workflow, and the repository policy says ClawHub issues should stay open rather than be auto-closed.

## Best Possible Solution

Keep this issue open for maintainer or service-operator handling. Remove the requested non-latest v1.8.1 tags through the existing tag-management path, and either perform the historical version cleanup through a controlled hosted-data/admin process or add a dedicated selected-version soft-delete workflow if maintainers want this to be repeatable in product code.
