## Summary

Keep open. Repository policy keeps ClawHub issues open during cleanup, and current main still does not conclusively prove the iOS/Safari mobile overflow is fixed: the detail page has responsive safeguards, but runtime/detail rows still include plausible long unwrapped values and the mobile overflow e2e coverage can skip when its live fixture is unavailable.

## What This Changes

The issue reports that the skill detail page is broken on mobile iOS/Safari, with an attachment showing layout overflow.

## Best Possible Solution

Harden the skill detail mobile layout so long summaries, stat/runtime metadata rows, install commands, links, tabs, and README content cannot increase document scrollWidth on iOS/Safari or mobile Chrome, then back it with deterministic mobile overflow coverage that cannot skip solely because a live API fixture is absent.
