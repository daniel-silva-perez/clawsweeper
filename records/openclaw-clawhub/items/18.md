## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still leaves a credible mobile overflow path: the active skill-detail view has responsive constraints, but runtime/install/config values still render through generic flex rows without strong wrapping safeguards, while the mobile overflow test can skip when its live fixture is unavailable.

## What This Changes

The issue reports that the skill detail page is broken in iOS/Safari mobile view.

## Best Possible Solution

Harden the skill detail mobile layout so long summaries, stat/runtime metadata rows, install commands, links, tabs, README content, code blocks, and tables cannot increase document scrollWidth on iOS/Safari or mobile Chrome, then cover that with deterministic mobile overflow tests that do not depend solely on a live production API fixture.
