## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still leaves a credible mobile overflow path on the skill detail page: several responsive safeguards exist, but runtime/install metadata still flows through weakly wrapped flex rows and the only mobile detail overflow e2e test can skip when its live fixture is unavailable.

## What This Changes

The issue requests fixing broken mobile rendering of the skill detail view on iOS/Safari.

## Best Possible Solution

Harden the skill detail mobile layout so long summaries, stat/runtime metadata rows, install commands, links, tabs, README content, code blocks, and tables cannot increase document scrollWidth on iOS/Safari or mobile Chrome, then cover that behavior with deterministic mobile overflow tests that do not depend solely on a live production API fixture.
