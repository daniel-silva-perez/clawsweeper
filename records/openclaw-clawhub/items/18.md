## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits automated implemented-on-main closure for pull requests. The current skill-detail path has responsive safeguards, but the reported iOS/Safari mobile overflow is not conclusively resolved because long runtime/install values still flow through weakly wrapped rows and the mobile overflow test can skip when its live fixture is unavailable.

## What This Changes

The issue requests fixing the broken iOS/Safari mobile layout on the skill detail page.

## Best Possible Solution

Harden the skill detail mobile layout so long summaries, stat/runtime metadata rows, install commands, links, tabs, README content, code blocks, and tables cannot increase document scrollWidth on iOS/Safari or mobile Chrome, then cover that with deterministic mobile overflow tests that do not depend solely on a live production API fixture.
