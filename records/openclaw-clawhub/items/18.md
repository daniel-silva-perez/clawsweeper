## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy only permits automated implemented-on-main closure for pull requests. Current main has responsive safeguards, but the original iOS/Safari overflow is not conclusively fixed: runtime requirement/detail rows still include plausible long unwrapped values, and the mobile detail overflow test can skip when its API fixture is unavailable.

## What This Changes

The issue reports that the skill detail page is broken on iOS/Safari mobile, with screenshot evidence of detail-page content extending past the viewport.

## Best Possible Solution

Harden the skill detail page so long summaries, metadata rows, runtime requirement values, install command snippets, tabs, and README content cannot increase document scrollWidth on iOS/Safari or mobile Chrome, with coverage that does not depend on a production API fixture being present.
