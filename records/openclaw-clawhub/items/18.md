## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits automated implemented-on-main closure for pull requests. Current main has several responsive safeguards, but the iOS/Safari skill-detail overflow is not conclusively resolved because long runtime/install/detail values still flow through weakly wrapped rows and the mobile overflow e2e test can skip when its live API fixture is unavailable.

## What This Changes

The issue reports a broken mobile skill detail view on iOS/Safari, with an attached screenshot showing the detail page layout failing on a narrow viewport.

## Best Possible Solution

Harden the skill detail mobile layout so long summaries, stat/runtime metadata rows, install commands, links, tabs, README content, code blocks, and tables cannot increase document scrollWidth on iOS/Safari or mobile Chrome, then cover that with deterministic mobile overflow tests that do not depend solely on a live production API fixture.
