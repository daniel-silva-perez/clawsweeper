## Summary

Keep open. This is a ClawHub issue, and the supplied repository cleanup policy only permits auto-closing pull requests that current main certainly implements. Substantively, current main recognizes declared env vars, but still intentionally flags file-read plus network-send behavior as suspicious potential exfiltration; the live ClawHub API also still reports the latest mimo-voice-assistant release as suspicious.

## Best Possible Solution

Keep this issue open for maintainer moderation. If the current published skill is benign, apply the existing manual clean override with an audit note. If maintainers want fewer future TTS/STT false positives, add a narrow scanner-policy refinement for declared fixed-provider audio processing while preserving warnings for broad env access, configurable endpoints, unexpected file reads, and exfiltration-like behavior.
