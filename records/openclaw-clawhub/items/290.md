## Summary

Keep open. Current main still does not support PowerShell skill uploads: `.ps1`, `.psm1`, and `.psd1` are absent from the text-file allowlists, browser validation still rejects unknown extensions without a text MIME type, and server publish validation still rejects non-text files. The ClawHub cleanup policy also says issues should remain open; only PRs that are certainly implemented on main should be proposed for auto-close.

## Best Possible Solution

Keep this issue open and implement explicit, security-aware PowerShell publish support. Add intended extensions such as .ps1 and likely .psm1/.psd1 to the shared and CLI allowlists, update content-type normalization, browser and server validation tests, package discovery/archive/GitHub import behavior, static scan coverage, and docs, then coordinate resolution with related #897.

## Risks / Open Questions

- PowerShell files are executable scripts, so support should include an explicit security review and static publish-scan coverage for risky patterns such as encoded commands, download-and-execute behavior, and credential or data exfiltration flows.
- The allowlist is mirrored across shared schema, generated/dist package files, browser upload helpers, server publish validation, CLI discovery, archive/GitHub import behavior, tests, and docs; a partial fix could leave inconsistent publish behavior.
- Related #897 remains open for the same limitation, so maintainers should decide whether to keep both reports linked or make one canonical before closing either manually.
