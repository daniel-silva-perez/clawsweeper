## Summary

Keep open. This is a ClawHub issue, and the supplied cleanup policy says ClawHub issues should remain open. Independently, current main still rejects PowerShell uploads: .ps1/.psm1/.psd1 are not in the shared text extension allowlist, browser publish validation rejects unknown extensions without text MIME types, server publish validation rejects non-text files, and CLI/GitHub import paths also gate on the same allowlist. Related #897 remains open for the same user-visible limitation.

## Best Possible Solution

Keep this issue open and implement explicit, security-aware PowerShell publish support. Add intended extensions such as .ps1 and likely .psm1/.psd1 to the shared schema and CLI allowlists, update content-type normalization and generated package artifacts as needed, cover browser/server/CLI/archive/GitHub import behavior with tests, extend static publish scan coverage for PowerShell-specific risk patterns, update docs, and coordinate resolution with related #897.
