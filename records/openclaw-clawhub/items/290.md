## Summary

Keep open. Current main still does not intentionally support `.ps1` files for skill creation: the shared text-extension allowlist and CLI mirror omit `ps1`, browser/archive/server/CLI/GitHub-import paths are still extension-gated, and the static scanner does not classify PowerShell as code. The related open report #897 describes the same user problem and links the `win-geolocate` workaround where PowerShell content had to be embedded in `SKILL.md`. For ClawHub issues, this review should not auto-close; this needs maintainer policy/security review before implementation.

## Best Possible Solution

Keep this issue open for maintainer policy/security review. If maintainers approve PowerShell helper files as text-based skill content, update the shared schema allowlist and CLI schema mirror, normalize `.ps1` to `text/plain`, update browser/archive/Convex/CLI/GitHub import handling and tests, extend static moderation scanner coverage for PowerShell where appropriate, update docs, and consolidate with #897.
