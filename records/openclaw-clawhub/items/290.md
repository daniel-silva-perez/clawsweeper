## Summary

Keep open. Current main still does not intentionally support `.ps1` PowerShell files across the skill upload/publish surfaces, and ClawHub repository policy says issues should remain open unless they are PRs already implemented on main. The related open report #897 adds useful context with the `win-geolocate` workaround of embedding PowerShell in `SKILL.md`, so this should be consolidated with that maintainer policy/security review rather than auto-closed.

## What This Changes

Allow `.ps1` PowerShell script files to be uploaded and published as supporting skill files.

## Best Possible Solution

Keep this issue open for maintainer policy/security review and consolidate the outcome with #897. If maintainers approve `.ps1` as text-based skill content, update the shared and mirrored schema allowlists, MIME normalization, browser upload validation, archive expansion, Convex publish/API validation, CLI package publishing, GitHub import handling, static moderation coverage, and focused tests/docs together.
