## Summary

Keep open. Current main still does not intentionally support `.ps1` PowerShell files: the shared and CLI text-extension allowlists omit `ps1`, browser/archive/server/GitHub-import paths are extension-gated, and the static scanner does not classify PowerShell as code. ClawHub repository policy also says issues should stay open; this one needs maintainer policy/security review and should be consolidated with the related open report #897, which documents the `win-geolocate` workaround of embedding PowerShell in `SKILL.md`.

## What This Changes

Request to allow `.ps1` PowerShell files to be uploaded as supporting skill files during new skill creation.

## Best Possible Solution

Keep this issue open for maintainer policy/security review. If maintainers approve `.ps1` as text-based skill content, update the shared schema allowlist and generated/mirrored schema outputs, normalize `.ps1` to `text/plain`, update browser direct upload, archive expansion, Convex publish/API validation, CLI publish, and GitHub import handling, extend static moderation coverage for PowerShell, update docs, and consolidate the outcome with #897.
