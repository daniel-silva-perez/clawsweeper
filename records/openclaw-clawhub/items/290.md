## Summary

Keep open. Current main still lacks explicit PowerShell upload support: .ps1/.psm1/.psd1 are absent from the shared text extension allowlists, browser and Convex publish validation still depend on those allowlists or text MIME types, and CLI/GitHub import paths would skip PowerShell files. Repository policy for this review also keeps ClawHub issues open; related #897 remains open for the same user-visible limitation.

## Best Possible Solution

Keep this issue open and implement security-aware PowerShell publish support across the shared schema and generated artifacts, browser/server validation, CLI folder and zip paths, GitHub import, docs, and tests. Add static moderation coverage for PowerShell-specific execution and exfiltration patterns, then coordinate resolution with related #897 so one canonical thread tracks any remaining edge cases.
