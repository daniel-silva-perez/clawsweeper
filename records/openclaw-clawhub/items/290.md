## Summary

Keep open. Current main still does not intentionally support `.ps1` files: the shared and CLI extension allowlists omit PowerShell, and the browser upload, archive expansion, server publish, CLI publish/hash, and GitHub import paths all remain gated by those text-file helpers. ClawHub cleanup policy also keeps issues open unless they are PRs already implemented on main, and related open report #897 should be handled with the same maintainer policy decision.

## What This Changes

The issue requests allowing `.ps1` PowerShell files during skill creation/upload.

## Best Possible Solution

Maintainers should decide whether PowerShell scripts are allowed as publishable text skill content. If approved, support should be added consistently across shared and CLI schema allowlists, MIME normalization, browser validation, archive expansion, Convex publish/API validation, GitHub import, moderation expectations, tests, and user-facing docs.
