## Summary

Keep this issue open. The ClawHub review policy keeps issues open, and current main still lacks explicit PowerShell upload support: the shared text-file extension allowlists omit `.ps1`/`.psm1`/`.psd1`, web and server publish validation depend on that allowlist unless the MIME type is already text, and CLI file discovery skips unknown extensions. Related #897 remains open and reports the same user problem plus the SKILL.md workaround.

## Best Possible Solution

Keep this issue open and implement explicit, security-aware PowerShell publish support. Add intended PowerShell extensions such as `.ps1` and likely `.psm1`/`.psd1` to the shared schema allowlists and content-type normalization, update web upload and server publish validation tests, update CLI/package discovery tests, document supported extensions, and extend static moderation scanning for PowerShell-specific dangerous execution and obfuscation patterns. Coordinate with related #897 rather than auto-closing either ClawHub issue before a fix lands.
