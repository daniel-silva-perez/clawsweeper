## Summary

Keep open. Current main still does not intentionally support `.ps1` skill helper files across the shared extension allowlist, browser validation, archive expansion, Convex publish validation, CLI discovery, or GitHub import. The related open report #897 describes the same user problem and links a real workaround where PowerShell content had to be embedded in `SKILL.md`; adding `.ps1` also needs maintainer policy/security review because the static code scanner currently does not classify `.ps1` as a code extension.

## Best Possible Solution

Keep this issue open for maintainer policy/security review. If maintainers approve PowerShell helper files as text-based skill content, update the shared schema allowlist and CLI schema mirror, normalize `.ps1` content type to `text/plain`, update web/archive/Convex/CLI/GitHub import handling, extend static moderation scanner coverage/tests for PowerShell where appropriate, update docs, and then consolidate with the related open report #897.
