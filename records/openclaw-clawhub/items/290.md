## Summary

Keep open. Current main still omits `.ps1` from the shared text-file allowlists and the browser, archive, Convex publish, and GitHub import paths remain extension/MIME gated; ClawHub cleanup policy also says issues should stay open unless the item is a PR already implemented on main. Related open report #897 adds useful user context and a workaround, so this should be handled as a maintainer policy/security decision rather than auto-closed.

## What This Changes

Allow `.ps1` PowerShell scripts to be uploaded and published as part of ClawHub skills.

## Best Possible Solution

Maintainers should decide whether PowerShell scripts are allowed as publishable text skill content. If approved, support should be added consistently across shared and CLI schema allowlists, MIME normalization, browser validation, archive expansion, Convex publish/API validation, GitHub import, moderation expectations, tests, and user-facing docs.
