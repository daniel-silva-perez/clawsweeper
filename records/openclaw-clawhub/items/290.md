## Summary

This issue should stay open. Current main still does not intentionally accept `.ps1` as a text-file extension for skill uploads, and the related open report #897 confirms the same user-facing problem with a real affected skill/workaround.

## Best Possible Solution

Keep this issue open as the canonical PowerShell upload support request. If maintainers approve `.ps1` as an accepted text-based skill helper format, add it to the shared and CLI-mirrored text-file allowlists, ensure content-type normalization treats it as text/plain, update tests/docs, and verify web upload, archive expansion, Convex publish validation, CLI discovery/publish, and GitHub import are consistent.
