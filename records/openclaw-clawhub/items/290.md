## Summary

Keep open. Current main still does not intentionally support `.ps1` PowerShell files across the skill upload, publish, CLI, and GitHub import paths, and ClawHub cleanup policy keeps issues open unless they are PRs already implemented on main. The related open report #897 adds useful user context and a real workaround, so this should be handled as a maintainer policy/security decision rather than auto-closed.

## What This Changes

The issue requests allowing PowerShell `.ps1` scripts to be uploaded and published as supporting skill files.

## Best Possible Solution

Maintainers should decide whether PowerShell scripts are allowed as text-based skill content. If approved, implement support consistently across shared and CLI schema allowlists, MIME normalization, browser upload validation, archive expansion, Convex publish/API validation, CLI publish/hash behavior, GitHub import, moderation/security prompts, tests, and docs, while consolidating the outcome with #897.
