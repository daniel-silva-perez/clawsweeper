## Summary

Keep open. This is a ClawHub issue, and the supplied repository policy says issues should remain open in this cleanup workflow. Independently, current main still does not support PowerShell skill uploads: .ps1/.psm1/.psd1 are absent from the text-file allowlists, browser validation still rejects unknown extensions without a text MIME type, and server publish validation still rejects non-text files.

## Best Possible Solution

Keep this issue open and implement explicit, security-aware PowerShell publish support. Add intended extensions such as .ps1 and likely .psm1/.psd1 to the shared and CLI allowlists, update content-type normalization, browser and server validation tests, package discovery/archive/GitHub import behavior, static scan coverage, and docs, then coordinate resolution with related #897.

## Risks / Open Questions

- PowerShell files are executable scripts, so support should include an explicit security pass and static publish-scan coverage for risky patterns such as encoded commands, download-and-execute behavior, credential access, and data exfiltration.
- The allowlist behavior is mirrored across shared schema, browser upload helpers, server publish validation, CLI discovery, archive/GitHub import, tests, and docs; a partial fix could leave inconsistent publish behavior.
- Related #897 remains open for the same user-visible limitation, so maintainers should decide whether to make one report canonical before manually closing either issue.
