## Summary

Keep open. Current main still does not intentionally support `.ps1` skill files: the shared and CLI text-file allowlists omit PowerShell, and the browser upload, archive expansion, Convex publish, raw file serving, CLI skill collection, and GitHub import paths are all gated through those text-file helpers. Because this is an issue in `openclaw/clawhub`, repository cleanup policy keeps it open, and related open report #897 adds current user context and a workaround.

## What This Changes

Allow PowerShell `.ps1` files to be uploaded and published as skill files.

## Best Possible Solution

Maintainers should decide whether PowerShell scripts are allowed as publishable text skill content. If approved, support should be added consistently across shared and CLI schema allowlists, MIME normalization, browser validation, archive expansion, Convex publish/API validation, GitHub import, static moderation expectations, tests, and user-facing docs.
