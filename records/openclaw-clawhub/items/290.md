## Summary

Current main still does not intentionally accept `.ps1` skill helper files. The shared schema allowlist, browser validation, archive expansion, Convex publish validation, CLI discovery, and GitHub import paths remain extension-gated and omit PowerShell. Because this is an open ClawHub issue, it should stay open for maintainer judgment rather than auto-close.

## Best Possible Solution

Keep this issue open for maintainer policy review. If maintainers approve PowerShell helper files as text-based skill content, implement a coordinated `.ps1` allowlist change across the shared schema and CLI mirror, normalize `.ps1` content type to `text/plain`, update web/archive/Convex/CLI/GitHub import tests and docs, then close or consolidate the sibling report #897.
