## Summary

Keep this issue open. Current main still does not explicitly support PowerShell skill files, and ClawHub policy says issues should remain open rather than be auto-closed by this cleanup pass.

## Best Possible Solution

Keep this issue open and implement explicit, security-aware PowerShell publish support. Add intended extensions such as .ps1 and likely .psm1/.psd1 to the shared allowlists and content-type normalization, update web and server validation tests, update CLI/package discovery behavior, document supported extensions, and coordinate with related #897 before maintainers close either report.

## Risks / Open Questions

- PowerShell files are executable scripts, so support should be added with explicit security review, static moderation coverage, and tests for encoded commands, download-and-execute patterns, and exfiltration-like behavior.
- There are multiple allowlist and upload surfaces: shared schema, package CLI discovery, browser upload helpers, server publish validation, API file serving, GitHub import, tests, and docs. A partial fix could leave inconsistent behavior.
