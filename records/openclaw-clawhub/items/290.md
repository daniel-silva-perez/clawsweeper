## Summary

Keep open. Current main still does not intentionally accept `.ps1` as a skill file type: the shared and CLI extension allowlists omit PowerShell, and the browser upload, archive expansion, Convex publish, GitHub import, raw file, and CLI collection paths remain gated through those text-file helpers. ClawHub cleanup policy also keeps issues open unless they are PRs already implemented on main, and related open report #897 adds useful user context and a current workaround.

## What This Changes

Allow PowerShell `.ps1` scripts to be uploaded and published as skill bundle files.

## Best Possible Solution

Maintainers should decide whether PowerShell scripts are allowed as publishable text skill content. If approved, support should be added consistently across shared and CLI schema allowlists, MIME normalization, browser validation, archive expansion, Convex publish/API validation, GitHub import, moderation expectations, tests, and user-facing docs.
