## Summary

Keep open. The ClawHub policy keeps issues open, and current main still has a credible source-level false-positive path: update hashes extracted local files, downloads inject generated `_meta.json`, and registry resolution compares against publish-time `version.files` fingerprints that do not include that generated metadata.

## What This Changes

The issue reports that `clawhub update --all` reports `local changes (no match)` immediately after a forced update even when installed skill content matches the registry.

## Best Possible Solution

Keep this issue open and fix the CLI/backend integrity comparison so a fresh install or `clawhub update --all --force` followed immediately by `clawhub update --all` compares the same canonical published file set as the registry. The narrow fix is to exclude generated download/install metadata such as root `_meta.json` from local and zip fingerprinting, or stop injecting/extracting that metadata for skill installs, while preserving integrity coverage for real published text files. Add regression tests and cluster the fix with #169 and #178.
