## Summary

Keep open. This is an open ClawHub issue, which the repository cleanup policy keeps open, and current main still has a credible false-positive path: installed skill fingerprints include generated `_meta.json`, while registry resolution compares against published version file fingerprints that do not include that generated download metadata.

## Best Possible Solution

Keep this issue open and fix the CLI/backend integrity comparison so a fresh install or immediate `clawhub update --all --force` followed by `clawhub update --all` compares the same canonical file set as the registry. The likely implementation is to exclude generated install/download metadata such as root `_meta.json` from local and zip fingerprinting while continuing to ignore `.clawhub/` and `.clawdhub/`, preserving hashes for user-authored JSON, and adding regression tests for install, force update, immediate clean update, and sync/hash helpers. Maintainers may consolidate this with #169 and #178 if they want one canonical tracker.
