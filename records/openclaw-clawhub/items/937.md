Keep this issue open. The repository policy for openclaw/clawhub only permits auto-closing PRs that are certainly implemented on current main, and this item is an issue. The underlying report also still has a credible source-backed path: current update integrity checks compare a local text-file fingerprint against published version fingerprints, while downloaded skill archives can inject generated `_meta.json` that becomes part of the local file set.

Keep this issue open and fix the CLI/backend integrity comparison so fresh installs and immediate `update --all --force` followed by `update --all` compare the same canonical file set as registry fingerprints.
