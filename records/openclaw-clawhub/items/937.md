## Summary

Keep this issue open. The repository-specific cleanup policy keeps ClawHub issues open, and current main still has a credible false-positive path for immediate `clawhub update --all`: local update fingerprints installed text files, downloaded skill archives can include generated `_meta.json`, while registry hash resolution compares against published version file fingerprints that do not include generated download metadata.

## Best Possible Solution

Keep this issue open and fix the CLI/backend integrity comparison so a fresh install or immediate `update --all --force` followed by `update --all` compares the same canonical file set as the registry fingerprint. The likely fix is to exclude generated install/download metadata such as `_meta.json` from local and zip fingerprinting, keep `.clawhub/origin.json` ignored, and add regression coverage for fresh install, force update, and immediate clean update. Maintainers may consolidate this with #169 if they want one canonical tracker.
