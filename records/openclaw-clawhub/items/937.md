## Repair Work Prompt

Fix the ClawHub CLI/backend update-integrity false positive reported here and in related reports #169 and #178. The expected boundary is one focused change that makes fingerprints comparable across installed local files, downloaded skill zips, and registry resolve/publish data. Exclude only generated install/download metadata such as root `_meta.json`, or stop injecting/extracting that metadata for skill installs, while preserving fingerprint coverage for real published text files including user-authored JSON. Add regression coverage for fresh install or force update followed immediately by normal update, and for real local edits still producing a local-change warning. Do not broadly ignore all JSON files, do not change unrelated sync UX, and do not refactor unrelated Convex publishing or moderation behavior. Add a CLI changelog/release-note entry only if the repo convention requires one for CLI bug fixes.
