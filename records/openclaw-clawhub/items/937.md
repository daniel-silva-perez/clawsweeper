## Summary

Keep this issue open. The ClawHub cleanup policy says issues should remain open, and current main still shows a plausible false-positive path: `update` hashes local files and reports `local changes (no match)` when the registry cannot resolve that fingerprint, while downloaded skill zips still inject generated `_meta.json` and local JSON files are included in fingerprints. Related #169 remains open and appears to track the same remaining bug, but that is not a reason to auto-close this issue under the repository policy.

## Best Possible Solution

Keep this issue open and fix the CLI integrity comparison so fresh installs and immediate force-updates compare the same canonical file set as registry fingerprints. The likely implementation is to exclude generated install/download metadata such as `_meta.json` from local fingerprinting, keep `.clawhub/origin.json` ignored, and add regression coverage for install, `update --all --force`, and immediate `update --all` reporting clean/up-to-date. Maintainers may also consolidate this with #169 if they want one canonical tracker.
