## Summary

Keep this issue open. The repository policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on main, and this item is an issue. Substantively, current main still has a credible false-positive path: update hashes local text files, JSON files are included in that hash set, downloads inject generated `_meta.json`, and registry resolution compares against published version fingerprints rather than that generated download metadata.

## Best Possible Solution

Keep this issue open and fix the CLI/backend integrity comparison so fresh installs and immediate `update --all --force` followed by `update --all` compare the same canonical file set as registry fingerprints. The likely fix is to exclude generated install/download metadata such as `_meta.json` from local and zip fingerprinting, keep `.clawhub/origin.json` ignored, and add regression coverage for fresh install, force update, and immediate clean update. Maintainers can consolidate with #169 if they want one canonical tracker.
