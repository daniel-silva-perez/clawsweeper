## Repair Work Prompt

Fix the ClawHub CLI/backend update-integrity false positive tracked by this issue and related reports #169 and #178. After a fresh install or `clawhub update --force`, a normal `clawhub update` / `clawhub update --all` should not report `local changes (no match)` when the published skill files are unchanged. Keep the change narrowly scoped: make local update hashing, downloaded zip contents, and registry resolve compare the same canonical published file set. Exclude only generated install/download metadata such as root `_meta.json`, or stop injecting/extracting that generated metadata for skill installs. Do not broadly ignore user-authored root JSON files; published JSON must still affect fingerprints and local edit detection. Add regression coverage for force-update followed immediately by normal update, generated metadata exclusion, and real local edit detection. Include a changelog entry only if the existing release workflow expects one for CLI bug fixes.
