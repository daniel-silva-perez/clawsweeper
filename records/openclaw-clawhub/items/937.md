## Summary

Keep this issue open. The ClawHub cleanup policy says issues should remain open, and current main still contains a credible false-positive path: update fingerprints local text files, refuses on an unresolved fingerprint, download zips still inject generated `_meta.json`, and registry resolution compares against published version files rather than generated download metadata. Related #169 remains open and may be a good canonical tracker, but it is not a reason to auto-close this issue under the repository policy.

## Best Possible Solution

Keep this issue open and fix the CLI integrity comparison so fresh installs and immediate `update --all --force` followed by `update --all` compare the same canonical file set as registry fingerprints. The likely code path is to exclude generated install/download metadata such as `_meta.json` from local and zip fingerprinting, keep `.clawhub/origin.json` ignored, and add regression coverage for fresh install, force update, and immediate clean update. Maintainers can consolidate with #169 if they want one canonical tracker.
