## Summary

Keep open. This is an open ClawHub issue, and repository policy keeps ClawHub issues open. Current main still has a credible false-positive path: update computes a local fingerprint from installed text files, installed downloads can include generated root `_meta.json`, and registry resolution compares against stored/published version file fingerprints that do not include that generated metadata.

## Best Possible Solution

Keep this issue open and fix the CLI/backend integrity comparison so a fresh install or immediate `clawhub update --all --force` followed by `clawhub update --all` compares the same canonical file set as the registry. The narrow fix is likely to exclude generated install/download metadata such as root `_meta.json` from local and zip fingerprinting while continuing to ignore `.clawhub/` and `.clawdhub/`, preserving hashes for user-authored JSON, and adding regression tests. Maintainers can consolidate this with #169 and #178 if they want one canonical tracker.
