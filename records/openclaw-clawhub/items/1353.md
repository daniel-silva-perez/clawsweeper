## Summary

Keep open. The WeWork duplicate still reproduces in production search, and current main only de-duplicates exact package family/name matches, not alias-equivalent or historical package records. ClawHub cleanup policy also keeps issues open unless a maintainer closes them manually.

## What This Changes

The issue asks for plugin catalog and search results to collapse historical or alias-equivalent plugin package records so WeWork appears as one card while legacy locators continue to resolve.

## Best Possible Solution

Define a maintainer-approved canonical package and alias model for plugin records, preserve compatibility for legacy detail/install locators, collapse browse and search on that canonical identity, and add WeWork-style regression coverage for both backend and frontend catalog paths.
