## Summary

Keep open. This is a ClawHub issue, and the repository policy only permits automated closing for PRs that are certainly implemented on current main. Current main also still lacks catalog-level canonical package/alias de-duplication for duplicate plugin cards such as `openclaw-wework` and `@openclaw/wework`; recent related work constrained plugin endpoints to plugin families, but did not collapse historical package aliases or runtime-equivalent records.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The best path is to decide the canonical package/alias model for historical plugin package records, preserve legacy detail/install locator compatibility, collapse browse/search by canonical package identity or runtime plugin ID, add WeWork-style regression tests, and run any required cursor-based backfill or moderation workflow for existing duplicate records.
