## Summary

Keep open. This is a ClawHub issue, and the cleanup policy for this repository only permits auto-closing pull requests that are certainly implemented on current main. The current code also still does not implement canonical alias collapse for plugin package records such as `openclaw-wework` and `@openclaw/wework`; recent #1842 work constrained catalog queries to plugin families but de-duplicates only exact package family/name or package id.

## Best Possible Solution

Keep this issue open. Implement package-level canonical identity handling for plugin browse/search: add or use an explicit canonical package/alias mapping, choose the canonical locator, collapse list and search results by canonical package identity or runtime plugin id, preserve compatibility for existing detail/install locators, add regression tests for `openclaw-wework` versus `@openclaw/wework`, and run a cursor-based data migration/backfill for historical duplicate records.
