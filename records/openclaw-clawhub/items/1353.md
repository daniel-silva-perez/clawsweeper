## Summary

Keep open. The reported duplicate WeWork cards are still observable in production, and current main still only de-duplicates plugin catalog/search rows by exact package family and package name rather than by a canonical alias or plugin lineage. ClawHub policy also says open issues should stay open for maintainer handling unless explicitly resolved outside this cleanup lane.

## What This Changes

The issue asks for plugin catalog and search results to collapse historical alias-equivalent package records, such as the two WeWork package names, into one canonical plugin card.

## Best Possible Solution

Define an explicit canonical package/alias model for plugin records, preserve compatibility for legacy detail and install locators, collapse browse and search on that canonical identity, and add WeWork-style regressions for backend and frontend catalog paths.
