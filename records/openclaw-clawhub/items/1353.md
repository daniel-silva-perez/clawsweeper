## Summary

Keep open. This is a ClawHub issue, and current main still does not provide a canonical alias/runtime collapse for historical plugin package records; the merged related work narrows plugin catalog queries but only de-dupes exact family/name matches.

## What This Changes

The issue reports that plugin catalog browse/search returns duplicate cards for historical alias-equivalent plugin package records, such as `openclaw-wework` and `@openclaw/wework`.

## Best Possible Solution

Keep this issue open for maintainer follow-up. Decide the canonical package/alias model for historical plugin package records, preserve legacy package detail/install locators where needed, collapse browse/search by the approved canonical identity, add WeWork-style regressions for `/api/v1/plugins` and `/api/v1/plugins/search`, and run any needed data cleanup or cursor-based backfill.
