## Summary

Keep open. This is a ClawHub issue, so the repo policy does not allow automated closure of issues, and the underlying duplicate is still present: production plugin search still returns both `@openclaw/wework` and `openclaw-wework`. Current main constrains plugin catalog endpoints to plugin families, but it still does not collapse historical alias-equivalent package records into one canonical catalog card.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The best path is to decide the canonical package/alias model for historical plugin package records, preserve legacy detail and install locators, collapse browse/search at the API layer by canonical package identity or approved alias mapping, add a WeWork-style regression for `/api/v1/plugins` and `/api/v1/plugins/search`, and run any required cursor-based backfill or moderation workflow for existing duplicate records.
