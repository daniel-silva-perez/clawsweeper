## Summary

Keep open. This is a ClawHub issue, and the supplied repository policy for this cleanup pass says ClawHub issues should remain open even when reviewed. The current main branch also still appears to lack canonical package/alias de-duplication for plugin catalog browse/search: recent #1842 constrained the plugin catalog to plugin families, but result collapse is still by exact family/name or package id rather than a canonical package identity that would merge `openclaw-wework` and `@openclaw/wework`.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The likely implementation path is to add or reuse a canonical package/alias mapping, choose the canonical locator, collapse plugin list/search results by canonical package identity or runtime plugin id, preserve legacy detail/install locator compatibility, add regression tests for `openclaw-wework` versus `@openclaw/wework`, and run a cursor-based migration/backfill for historical duplicate package records.
