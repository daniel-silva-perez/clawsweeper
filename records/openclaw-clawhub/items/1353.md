## Summary

Keep open. This is a ClawHub issue, and the cleanup policy for this repo only permits auto-closing PRs that are certainly implemented on main. Current main also still lacks canonical package identity collapse for alias-equivalent plugin records such as `openclaw-wework` and `@openclaw/wework`; recent #1842 work constrained the plugin catalog to plugin families but did not solve historical alias de-duplication.

## Best Possible Solution

Keep this issue open and implement package-level canonical identity handling for plugin browse/search. The likely path is to add or use a canonical package/alias mapping, choose the canonical locator, collapse list/search results by canonical package identity or runtime plugin id, preserve existing detail/install locator compatibility, add regression tests for `openclaw-wework` versus `@openclaw/wework`, and run a data migration/backfill for historical duplicates.
