Current main has the right stable GitHub numeric-ID mapping and a post-login profile sync path for renamed accounts, but it does not prove the reported production OAuth verifier/session mismatch or requested account relink is resolved. ## Best Possible Solution

ClawHub should preserve sign-in across GitHub username changes using the immutable GitHub provider id, and maintainers should either repair this user's production auth linkage or add a focused recovery/admin path if production triage shows a recurring verifier or duplicate-account failure.
