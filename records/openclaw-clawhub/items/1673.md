## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy for this repository only permits auto-closing pull requests that current main definitely implements. The underlying report also still appears valid: current main stores package/plugin manifests but does not feed the MongoDB env/install declarations into the package LLM review prompt, and production still shows the latest package release as suspicious for the same metadata mismatch.

## Best Possible Solution

Keep this issue open. A maintainer should update the package review pipeline so extractedPackageJson/extractedPluginManifest metadata, including providerAuthEnvVars, setup env vars, primary credential/config hints, and openclaw.install.npmSpec, is normalized into the static scan metadata and LLM SkillEvalContext. Then rerun or backfill static/LLM scans for @kansodata/kansodata-mongodb-plugin 0.1.2 and 0.1.3, and add a regression test for a MongoDB code plugin that declares MONGODB_URI in openclaw.plugin.json/package metadata.
