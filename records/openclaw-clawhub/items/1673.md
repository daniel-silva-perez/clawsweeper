## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub PRs that are certainly implemented on current main. The underlying report also still looks current: main stores package/plugin manifests, but the package static scan and LLM review path do not normalize the MongoDB env/install declarations into the metadata fields that the security prompt renders.

## Best Possible Solution

Keep this issue open. A maintainer should normalize extractedPackageJson and extractedPluginManifest into the package review metadata used by static scanning and SkillEvalContext, including providerAuthEnvVars, setup env vars, primary credential/config hints, and openclaw.install.npmSpec. Then rerun or backfill scans for the affected 0.1.2 and 0.1.3 releases and add a regression test for a MongoDB code plugin that declares MONGODB_URI through openclaw.plugin.json/package metadata.
