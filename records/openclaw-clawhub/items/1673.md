## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are definitely implemented on current main. The report is also still substantiated: production package-version data for the affected package still reports suspicious LLM analysis caused by env/install metadata mismatch, while the npm package and plugin manifest declare MongoDB credentials and npm install metadata.

## Best Possible Solution

Keep this issue open. A maintainer should normalize extractedPackageJson and extractedPluginManifest into the package review metadata used by static scan and SkillEvalContext: map providerAuthEnvVars/setup.envVars/configSchema credential hints into requires.env, primaryEnv, and envVars, and map packageJson.openclaw.install.npmSpec into an install spec. Then rerun or backfill package reviews for @kansodata/kansodata-mongodb-plugin 0.1.2 and 0.1.3, and add a regression test for a MongoDB code plugin that declares MONGODB_URI through package/plugin metadata. Related moderation appeals such as #345 can help validate whether the fix covers the broader pattern.
