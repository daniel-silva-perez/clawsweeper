## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are definitely implemented on current main. Substantively, the reported metadata mismatch is still live: production ClawHub scan output for the package flags the latest release as suspicious because the LLM review sees no required env vars, no primary credential, and no install spec, even though the published package and plugin manifest declare MongoDB env/config metadata.

## Best Possible Solution

Keep this issue open. A maintainer should inspect the production packageRelease records for @kansodata/kansodata-mongodb-plugin 0.1.2 and 0.1.3, compare extractedPackageJson/extractedPluginManifest against the stored files, then fix package review metadata assembly so code-plugin env credentials, providerAuthEnvVars/setup env vars, primary credential/config hints, and openclaw.install metadata are passed into static and LLM review. After that, rerun or backfill scans for the affected releases and add a regression test for a MongoDB plugin that declares MONGODB_URI in openclaw.plugin.json/package metadata.
