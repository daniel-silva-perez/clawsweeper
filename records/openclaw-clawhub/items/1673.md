## Summary

Keep open. This is an open issue in openclaw/clawhub, and the repository-specific policy only permits auto-closing PRs that are certainly implemented on main. Substantively, current main and the live package record still show a plausible package-review metadata gap: package LLM review is not fed extracted package/plugin metadata, and the live v0.1.2 record remains LLM-suspicious for the no-env/no-install metadata mismatch.

## Best Possible Solution

Keep this issue open. A maintainer should compare the production package release record against the npm v0.1.2 tarball, decide whether code-plugin credentials should come from README, configSchema, package.json openclaw, or explicit manifest env fields, then update package review metadata assembly so packageJson.openclaw.install and relevant plugin/config credential metadata are included in static/LLM review. After that, rerun or backfill the LLM review for this release and add a regression test for a code plugin that declares/uses a MongoDB credential and npm install metadata.
