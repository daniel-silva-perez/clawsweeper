## Summary

Keep open. This is an open ClawHub issue, and the repository policy only permits auto-closing ClawHub pull requests that are certainly implemented on main. The report also remains substantiated: the live package API still reports suspicious LLM analysis for 0.1.2 and 0.1.3 because the review context says no env vars / no primary credential / instruction-only, while the published npm package and plugin manifest declare MongoDB credential and npm install metadata.

## Best Possible Solution

Keep this issue open for maintainer follow-up. Normalize extractedPackageJson and extractedPluginManifest into package review metadata used by static scan and SkillEvalContext: map providerAuthEnvVars/setup.envVars/configSchema credential hints into requires.env, primaryEnv, and envVars, and map packageJson.openclaw.install.npmSpec into an install spec. Then rerun or backfill LLM/static reviews for @kansodata/kansodata-mongodb-plugin 0.1.2 and 0.1.3 and add a regression test for a MongoDB code plugin that declares MONGODB_URI through package/plugin metadata.
