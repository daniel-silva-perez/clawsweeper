## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy says to keep issues open while only auto-closing pull requests that are certainly implemented on current main. The source is also only partially aligned with the request: direct CLI/API publish now sends the requested slug and backend publish rejects exact slug/alias collisions, but GitHub import still suggests suffix-style slugs, docs still document suffixing, browse/search do not suppress duplicates or forks by default, and the malware-clone discussion still needs maintainer/security triage beyond source inspection.

## Best Possible Solution

Keep this issue open for maintainer and security triage. Preserve the current direct-publish exact-slug collision rejection and static malware scanning, then decide and implement the remaining policy: whether GitHub import should fail rather than suggest suffixes, whether public discovery should hide duplicate/fork rows by default, how production duplicate/malware rows should be cleaned up, and whether owner-scoped slugs or explicit fork metadata need a schema/API migration.
