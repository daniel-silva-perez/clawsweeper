## Summary

Keep open. Current main has strong collision and malware-scanning protections, but this is an issue and the repository policy keeps ClawHub issues open; the remaining concern also spans production registry cleanup, security triage, and product policy for whether duplicate/fork entries should be hidden by default.

## What This Changes

The issue requests that ClawHub stop or clean up duplicate skill slugs with random suffixes so registry discovery is not polluted by duplicate or malicious-looking skill clones.

## Best Possible Solution

The desired end state is a registry where exact slug collisions fail, malicious or test clone families are remediated in production, canonical/fork metadata drives clear discovery defaults, and users can intentionally opt into viewing forks or duplicates when that is useful.
