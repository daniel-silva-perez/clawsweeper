## Summary

Keep open. This is an issue, and the supplied openclaw/clawhub cleanup policy only permits auto-closing pull requests that are certainly implemented on main. Current main has important mitigations for exact-slug publishing collisions and the malware-clone concern raised in discussion, but the broader registry cleanup/product work is not fully resolved: GitHub import still suggests suffixed slugs, docs describe that behavior, search/listing does not default to canonical-only results, and source review cannot verify production takedowns or account moderation.

## Best Possible Solution

Keep this issue open for maintainer and security triage. Preserve the exact-slug collision rejection and static malware detection already present on main, then split or resolve the remaining work: production duplicate/malware cleanup, whether GitHub import should stop suggesting suffixed slugs or require explicit fork/duplicate metadata, whether discovery should hide duplicate/fork rows by default, and whether owner-scoped slug uniqueness needs a schema/API migration.
