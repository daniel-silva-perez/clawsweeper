## Summary

Keep this issue open. Current main has important partial mitigations: exact slug collisions fail instead of auto-suffixing, publish-time static scanning blocks the reported base64/curl-to-shell pattern, and owner/staff duplicate tools exist. The central remaining problem is still broader than a narrow code cleanup: random-suffix clone pollution, canonical/fork search policy, existing registry remediation, and the security response for already-published malicious clones need maintainer handling. ClawHub repository policy for this review lane also keeps issues open; only PRs that are certainly implemented on main are eligible for auto-close.
