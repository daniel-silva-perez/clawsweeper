## Summary

Keep open. Current main rejects exact slug collisions and has scanner coverage for the malware pattern raised in the discussion, but this is a ClawHub issue and the repository policy only allows auto-close for pull requests. The remaining work is live registry/security cleanup plus product policy for canonical/fork visibility, and a current production search still returns duplicate-looking variants.

## What This Changes

The issue asks ClawHub to stop publishing duplicate skills with random slug suffixes, clean up duplicate or malicious clone families, and improve canonical/fork handling in discovery.

## Best Possible Solution

Keep this as maintainer-led registry cleanup: preserve the exact-collision protections, audit and remediate abusive or test clone families in production, decide the default canonical/fork discovery policy, and document how users can intentionally view forks or duplicates.
