## Summary

Keep this issue open. The ClawHub issue policy requires keeping issues open, and the current main code still has the broad scanner/tagging behavior behind the report: static scan reads every uploaded text file, `child_process` plus exec/spawn becomes `suspicious.dangerous_exec`, and capability chips are derived from broad regexes over all uploaded text. Live ClawHub data also shows the historical v0.17.4 scan still has the six reported chips and `suspicious.dangerous_exec`; the latest listing has no capability chips but is still suspicious for other scanner reasons, so maintainer follow-up is still useful.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The best path is to review the x0x listing/version, use existing moderator tools to remove false capability tags and apply a manual clean override if appropriate, then harden the code with focused tests: distinguish cryptographic identity/signing from cryptocurrency wallets/transactions, and either scope static analysis to install-relevant files or add a carefully bounded false-positive rule for language bindings. If persisted records are corrected, sync the affected skill/version/search digest data as part of the moderation cleanup.
