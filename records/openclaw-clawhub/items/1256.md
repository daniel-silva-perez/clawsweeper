## Summary

Keep this PR open. Current main still lacks the PR's intended indexed direct lexical prefix lookup for skill slug/display-name recall, and the PR is paired with same-author open issue #1254 via closing syntax. The later `kling` -> `klingai` and `scienceclaw`/`ScienceClaw` reports point at the same unresolved recall class rather than making this PR obsolete.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with an equivalent maintainer patch that adds indexed direct prefix lookup for skill slug/display name, normalized digest fields, migration-safe backfill, and regression coverage for the Midscene case plus the reported `kling` -> `klingai` and case-stability examples. Because it is paired with open issue #1254, the issue should close only after an implementation merges.
