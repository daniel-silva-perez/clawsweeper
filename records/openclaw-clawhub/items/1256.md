## What This Changes

The PR adds normalized slug/display-name fields and indexes to skill search digests, a direct lexical prefix lookup merged into skill search, maintenance backfill updates, and regression tests for prefix recall misses.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve capability-tag filtering in direct lookup:** `convex/search.ts:233-238`
  - body: The new direct prefix path is merged into search results, but this call only forwards highlightedOnly and nonSuspiciousOnly. The current search flow filters exact slug, vector, and fallback results by capabilityTag, so a direct slug/name hit with the wrong tag can leak into tagged /skills searches and can fill the limit before the filtered fallback runs. Pass capabilityTag through and apply the same matchesCapabilityTag check in directLexicalSkills.
  - confidence: 0.91
- **[P2] Keep fallback candidates after direct prefix matches:** `convex/search.ts:247-249`
  - body: primaryMatches now includes direct prefix candidates, and fallbackMatches is skipped once that combined list reaches limit. directLexicalSkills reads only capped prefix slices from normalized indexes, so broad prefixes can fill the limit with a subset of prefix rows and suppress the existing recent-digest fallback candidates that would otherwise be ranked with the final result set. Run the fallback based on exact/vector sufficiency, or merge fallback before the final ranking decision.
  - confidence: 0.76
