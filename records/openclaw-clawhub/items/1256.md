## Summary

Keep open. Current main still lacks the direct indexed prefix-recall path, and the PR remains an active unmerged implementation for a valid search recall gap. The branch also still has concrete correctness issues to address before merge, especially preserving current capability-tag filtering and avoiding result loss when direct prefix slices hit the limit.

## What This Changes

The branch adds normalized slug/display-name fields and indexes to skill search digests, a direct lexical prefix lookup merged into skill search, a digest backfill update, and regression tests for prefix recall misses.

## Best Possible Solution

Rebase this PR or land an equivalent patch that adds indexed normalized slug/display-name prefix recall while preserving all existing filters, keeping the recent-digest fallback in the final candidate pool, handling pre-backfill rollout safely, and covering Midscene, kling/klingai, ScienceClaw casing, acronym casing, and tagged-search regressions before closing the linked search-recall reports.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve capability-tag filtering in direct lookup:** `convex/search.ts:233-241`
  - body: Current main filters exact slug, vector, and fallback skill results by capabilityTag. The new direct prefix call only forwards highlightedOnly and nonSuspiciousOnly, and directLexicalSkills has no capabilityTag filter, so after rebasing a direct slug/name hit from the wrong tag can leak into tagged searches and occupy result slots. Thread capabilityTag through directLexicalSkills and apply the same matchesCapabilityTag check before returning entries.
  - confidence: 0.91
- **[P2] Run fallback before cutting off broad prefix results:** `convex/search.ts:250-260`
  - body: primaryMatches now includes direct prefix candidates, and fallbackMatches is skipped once that combined list reaches limit. directLexicalSkills reads capped prefix slices ordered by indexed fields before final scoring, so broad prefixes can fill the limit with a subset of prefix rows and suppress existing recent-digest fallback candidates that would otherwise compete in the final ranked set.
  - confidence: 0.78
- **[P2] Cover legacy slug rows during normalized backfill:** `convex/search.ts:379-388`
  - body: normalizedSlug is optional and filled by a separate backfill, but direct slug-prefix recall only reads normalizedSlug indexes. Existing digest rows without normalizedSlug will still miss slug-prefix queries during rollout unless they happen to appear in the bounded recent fallback window. Add a legacy slug-prefix source or make the rollout/backfill path atomic for this behavior.
  - confidence: 0.82
- **[P3] Handle acronym casing in legacy name fallback:** `convex/search.ts:74-80`
  - body: legacyDisplayNamePrefixes probes raw, lowercase-normalized, and title-cased values against the case-sensitive displayName indexes. During the pre-backfill window, a lowercase query like gpt still will not match a legacy displayName starting with GPT, leaving acronym-prefixed rows invisible unless they happen to be in the recent fallback scan.
  - confidence: 0.74
