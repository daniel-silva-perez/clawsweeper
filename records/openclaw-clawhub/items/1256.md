## Summary

Keep open. Current main still lacks the direct lexical prefix lookup and normalized digest fields, while this PR remains an active but stale implementation with blocking review issues around preserving current filters, keeping fallback candidates in the final pool, and safe rollout for pre-backfill rows.

## What This Changes

The PR adds normalized slug/display-name fields and indexes to skill search digests, a direct lexical prefix lookup merged into skill search, maintenance backfill updates, and regression tests for prefix recall misses.

## Best Possible Solution

Land this PR or an equivalent patch after rebasing against current main, preserving all existing search filters, keeping direct-prefix and fallback candidates in the final ranked pool, handling pre-backfill slug/name rollout safely, and covering Midscene, kling/klingai, ScienceClaw casing, acronym casing, and tagged-search behavior before closing the linked search-recall reports.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve capability-tag filtering in direct lookup:** `convex/search.ts:233-241`
  - body: Current main filters exact slug, vector, and fallback skill results by `capabilityTag`, but the new direct prefix path neither accepts nor forwards that filter. After rebasing, direct slug/name hits from the wrong tag can leak into tagged skill searches and occupy result slots, so pass `capabilityTag` through and apply the same `matchesCapabilityTag` check in `directLexicalSkills`.
  - confidence: 0.91
- **[P2] Keep fallback candidates after direct prefix matches:** `convex/search.ts:250-260`
  - body: `primaryMatches` now includes direct prefix candidates, and fallback is skipped once that combined list reaches `limit`. Because `directLexicalSkills` reads capped prefix slices from indexes before final scoring, broad prefixes can suppress recent-digest fallback candidates that should still compete in the final ranked set.
  - confidence: 0.78
- **[P2] Cover legacy slug rows before normalized backfill:** `convex/search.ts:379-388`
  - body: `normalizedSlug` is optional and filled by a separate backfill, but slug-prefix recall only reads the normalized-slug indexes. Existing digest rows without `normalizedSlug` can still miss slug-prefix queries during rollout unless they happen to be in the bounded recent fallback window, so add a legacy slug-prefix source or make rollout/backfill atomic for this behavior.
  - confidence: 0.82
- **[P3] Handle acronym casing in legacy name fallback:** `convex/search.ts:74-80`
  - body: During the `normalizedDisplayName` backfill window, the legacy display-name scans are case-sensitive and only try raw, lowercase, and title-case prefixes. A lowercase query like `gpt` still will not match legacy rows whose display name starts with `GPT`, leaving acronym-prefixed skills invisible unless recent fallback catches them.
  - confidence: 0.74
