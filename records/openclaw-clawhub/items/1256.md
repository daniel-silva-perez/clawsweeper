## Summary

Keep open. Current main still lacks the direct normalized prefix lookup this PR is trying to add, and the PR remains the active implementation for a valid public search recall gap, but it needs rebase/conflict handling and correctness fixes before merge.

## What This Changes

The PR adds normalized slug/display-name digest fields and indexes, a direct lexical prefix lookup path merged into skill search, backfill updates, and regression tests for prefix recall misses.

## Best Possible Solution

Rebase this PR or land an equivalent patch on current main that adds indexed normalized slug/display-name prefix recall, preserves all existing filters and fallback ranking behavior, includes a safe digest backfill path, and covers Midscene, kling/klingai, ScienceClaw casing, legacy rollout, and tagged-search regressions before closing the linked search-recall reports.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve capability-tag filtering in direct lookup:** `convex/search.ts:236-241`
  - body: Current main filters exact slug, vector, and fallback results by capabilityTag. The new direct prefix call only forwards highlightedOnly and nonSuspiciousOnly, so after rebasing a slug/name hit from the wrong tag can leak into tagged skill searches and fill result slots. Thread capabilityTag through directLexicalSkills and apply the same filter before returning entries.
  - confidence: 0.88
- **[P2] Keep fallback candidates after direct prefix matches:** `convex/search.ts:250-260`
  - body: fallbackMatches is skipped once primaryMatches includes enough direct prefix hits, but directLexicalSkills reads capped prefix slices ordered by indexed fields before final scoring. Broad prefixes can therefore suppress existing recent-digest fallback candidates that would otherwise compete in the final ranked set. Base the fallback decision on exact/vector sufficiency or merge fallback before the final ranking cutoff.
  - confidence: 0.78
- **[P2] Query legacy slugs before normalized backfill completes:** `convex/search.ts:379-388`
  - body: normalizedSlug is optional and filled by a separate backfill, but direct slug-prefix recall only reads normalizedSlug indexes. Existing digest rows without normalizedSlug will still miss prefix slug queries during rollout unless they happen to be in the bounded fallback window. Add a legacy slug-prefix source or make the rollout/backfill path atomic for this behavior.
  - confidence: 0.82
- **[P3] Handle acronym casing in legacy name-prefix fallback:** `convex/search.ts:74-80`
  - body: legacyDisplayNamePrefixes emits raw, lowercase-normalized, and title-cased variants. During the pre-backfill window, a lowercase query like gpt still will not probe a case-sensitive displayName prefix such as GPT, so acronym-prefixed legacy rows can remain invisible. Add an uppercase/acronym variant or avoid depending on the case-sensitive legacy name index for this fallback.
  - confidence: 0.74
