## Summary

Keep this PR open. Current main still does not implement the PR's intended direct indexed lexical prefix lookup for skill slug/display-name recall, and the PR remains paired with open issue #1254 via closing syntax. The later `kling` -> `klingai` discussion also matches the same unresolved prefix-recall gap. Security review found no supply-chain-sensitive file changes in the provided PR file list.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with an equivalent maintainer patch that adds indexed direct prefix lookup for skill slug/display name, normalized digest fields, migration-safe backfill, and regression coverage for the Midscene case plus the reported `kling` -> `klingai` prefix case. Because this PR is paired with open issue #1254 using closing syntax, the issue should close only after an implementation merges.
