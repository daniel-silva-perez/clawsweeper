Keep open. This is an open ClawHub issue, and the cleanup policy only allows auto-closing ClawHub PRs that current main definitely implements. ## What This Changes

The issue asks maintainers to review and unflag the `unraidclaw` listing, which the reporter describes as a false positive caused by normal plugin distribution files, user-configured LAN HTTP/TLS/API-key behavior, and legitimate Unraid management tools.

## Best Possible Solution

Keep this open for maintainer moderation and listing follow-up. Coordinate with #1371: if maintainers agree `unraidclaw` should be a plugin rather than a skill, retire/delete the skill listing and guide the publisher to the code-plugin package path; if the skill listing should remain, review the package/source, apply the audited clean manual override if safe, and consider a narrow scanner change with tests so benign `uiHints.serverUrl.placeholder` LAN URLs are not treated as untrusted install-source URLs.
