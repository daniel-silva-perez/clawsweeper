## Summary

Keep open. This is an open ClawHub issue, and the repository policy for openclaw/clawhub only permits auto-closing pull requests that current main certainly implements. Substantively, the live ClawHub API still reports unraidclaw latest version 0.1.9 as suspicious for `suspicious.install_untrusted_source`, and current main still has the broad manifest raw-IP rule that produces that reason. The right next step is maintainer moderation review, not cleanup closure.

## Best Possible Solution

Keep this issue open for maintainer moderation review. A moderator should inspect the current unraidclaw package/source and, if reviewed safe, apply the existing manual clean override with an audit note. If the trigger is a LAN-IP placeholder or configuration field in `openclaw.plugin.json` rather than an install source, narrow `scanManifestFile` with focused tests so benign manifest metadata is not treated the same as untrusted install-source URLs.
