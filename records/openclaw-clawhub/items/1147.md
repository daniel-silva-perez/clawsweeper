## Summary

Keep open. This is an open ClawHub issue, and the supplied repository policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Substantively, the appeal is still actionable: current main still flags raw-IP URLs found anywhere in manifest files as `suspicious.install_untrusted_source`, and the live ClawHub record for `unraidclaw` still reports that exact suspicious verdict with evidence in `openclaw.plugin.json`.

## Best Possible Solution

Keep this issue open for maintainer moderation review. A moderator should inspect the current `unraidclaw` source/package and, if reviewed safe, apply the existing manual clean override with an audit note. If LAN-IP placeholders are acceptable in plugin UI hints, narrow the manifest scanner so placeholder metadata is not treated as an install source, or ask the publisher to change the placeholder format.
