## Summary

Keep open. This is an open ClawHub issue, and the repository policy for this review keeps issues open. Substantively, current main has a narrow clean-engine VirusTotal Code Insight suppression path, but the live public API still reports the reported HostGuard v1.0.0 hash as VT suspicious and does not expose the AV engine counts or stored scanner discriminator needed for users to see the 0/64 distinction directly.

## What This Changes

The issue requests removal of the suspicious warning for HostGuard v1.0.0 or clearer UI/API distinction between zero AV-engine detections and a VirusTotal AI heuristic flag.

## Best Possible Solution

Keep this issue open for maintainer moderation/product follow-up. Verify HostGuard v1.0.0's production vtAnalysis for the reported hash, including stored scanner and engineStats; if it is a clean-engine Code Insight-only case, trigger a rescan/backfill or apply an audited clean manual override. Separately decide whether the public scan API and UI should expose VT scanner and AV engineStats so users can distinguish zero AV detections from AI heuristic findings without weakening actual suspicious/malicious protections.
