## Summary

Keep open. Repository policy for ClawHub issues does not permit cleanup auto-closing issues, and the reported historical HostGuard v1.0.0 scan still reproduces on the public API: it returns a suspicious security status for the exact reported hash while the latest HostGuard version is clean. Current main has a narrow clean-engine VirusTotal Code Insight suppression path, but production/history still needs maintainer verification or rescan/backfill/manual override, and the public API/UI still do not expose the scanner discriminator or AV engine counts needed to show the requested 0/64 distinction directly.

## What This Changes

The issue asks ClawHub to remove or clarify a HostGuard warning where VirusTotal AV engines show 0/64 detections but a VirusTotal AI Code Insights heuristic appears to drive a suspicious classification.

## Best Possible Solution

Keep this issue open for maintainer moderation/product follow-up. Verify HostGuard v1.0.0's production vtAnalysis for the reported hash, including stored scanner and engineStats; if it is a clean-engine Code Insight-only case, trigger a rescan/backfill or apply an audited clean manual override. Separately decide whether the public scan API and UI should expose VT scanner and AV engineStats so users can distinguish zero AV detections from AI heuristic findings without weakening actual suspicious/malicious protections.
