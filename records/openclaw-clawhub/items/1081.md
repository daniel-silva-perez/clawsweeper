Keep open. ClawHub cleanup policy only permits auto-closing pull requests that are certainly implemented on main, and this is an open issue. The substance also still needs maintainer moderation follow-up: current main has targeted VirusTotal Code Insight false-positive calibration, but the live HostGuard v1.0.0 scan still reports a suspicious VirusTotal security status while the latest HostGuard version is clean, and the public API/UI do not expose scanner identity or AV engine counts directly enough to satisfy the requested 0/64 distinction.

## What This Changes

The issue asks ClawHub to clear HostGuard v1.0.0's suspicious warning or distinguish zero AV-engine detections from a VirusTotal AI heuristic finding.

## Best Possible Solution

Leave the issue open for maintainer moderation. Separately, decide whether the public scan API and security UI should expose scanner type and AV engine counts without weakening real suspicious or malicious protections.
