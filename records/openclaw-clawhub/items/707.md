## Summary

Keep open. Current main still does not document, normalize, moderate, serialize, or surface SKILL.md lifecycle hooks; existing `hooks` references are for package/plugin capability metadata or webhooks, not skill frontmatter hooks. The report is also security-sensitive and needs maintainer policy decisions before implementation.

## What This Changes

The issue requests SKILL.md lifecycle hook documentation, hook metadata parsing, security analysis, API/UI transparency, and install-time visibility for skills that declare automatic hooks.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy, then implement it consistently across the skill format docs, shared schema, parser and any backfill path, public API serialization, skill detail/install UI, static and LLM moderation context, and warnings for high-privilege events or hook types.
