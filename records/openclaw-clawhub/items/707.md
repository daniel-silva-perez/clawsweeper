## Summary

Keep open. Current main still does not document or normalize SKILL.md lifecycle hooks, does not run hook-specific security checks, and does not expose hook metadata on skill API/UI surfaces. The existing `hooks` support in the repository is for package/plugin capability metadata, which is a separate surface from SKILL.md frontmatter hooks. This is also an issue, and the ClawHub cleanup policy only auto-closes PRs that are certainly implemented on main.

## What This Changes

Requests first-class documentation, metadata parsing, security analysis, and registry/UI surfacing for SKILL.md frontmatter lifecycle hooks.

## Best Possible Solution

Land a maintainer-approved hook contract first, then implement it consistently across the skill format docs, shared schema, parser/backfill path, public API, skill detail UI, static and LLM moderation context, and install/inspect warnings for high-privilege events and hook types.
