## Summary

Keep open. Current main still lacks the requested SKILL.md lifecycle hook documentation, normalized metadata, hook-specific moderation, public serialization, and skill page/install transparency; the existing hook references are for package/plugin capabilities, a different surface. The ClawHub cleanup policy also keeps issues open unless a close reason is strongly supported, and this request needs maintainer security/product policy decisions before implementation.

## What This Changes

Document SKILL.md lifecycle hooks, add hook-aware security analysis, and surface hook metadata in ClawHub API/UI before install.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy first, then implement it consistently across the skill format docs, shared schema, parser/backfill path, public API, skill detail/install UI, static moderation, LLM moderation context, and warnings for high-privilege events or hook types.
