## Summary

Keep this issue open. It is a ClawHub issue, and the supplied repository policy says to keep issues open unless the item is a PR already implemented on main. A current-main shell check also shows the requested SKILL.md hooks documentation, typed skill metadata, parser promotion, hook-aware security validation, API surface, and skill detail disclosure are still missing.

## Best Possible Solution

Keep this issue open. The likely path is to add a typed hooks declaration to the skill metadata contract and docs, parse it from SKILL.md frontmatter, validate lifecycle event names, hook types, command targets and missing files, flag privileged or blocking events, expose hook declarations through the public API and skill detail UI, and add focused parser, scanner, API, and UI tests. Treat #129 as related broader security context rather than the canonical hook-specific tracker.
