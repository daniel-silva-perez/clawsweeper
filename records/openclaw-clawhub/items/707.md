## Summary

Keep this issue open. It is a ClawHub issue, and the supplied repository policy only allows auto-close for pull requests that are certainly implemented on main. A current-main review also shows the requested hook-specific skill format docs, typed metadata contract, parser promotion, static security checks, public API surface, and skill detail UI disclosure remain unimplemented.

## Best Possible Solution

Keep the issue open. The likely implementation path is to add a typed hooks declaration to the skill metadata contract and docs, parse it from SKILL.md frontmatter, validate lifecycle event names, hook types, command targets and missing files, flag privileged/blocking events, expose hook declarations through the public API and skill detail UI, and add focused parser, scanner, API, and UI tests. Treat #129 as related broader security context rather than the canonical hook-specific tracker.
