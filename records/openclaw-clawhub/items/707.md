## Summary

Keep open. Current main does not implement the requested SKILL.md lifecycle hook documentation, schema/parser normalization, hook-specific moderation, public API serialization, or skill-page/install transparency; the only existing `hooks` surface found is package/plugin capability metadata, which is separate. The ClawHub cleanup policy also keeps issues open and only auto-closes PRs that are certainly implemented on main.

## What This Changes

This issue asks ClawHub to document Claude Code SKILL.md frontmatter hooks, add hook-focused security analysis, and show hook lifecycle metadata to users before install.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy first, then implement it consistently across the skill format docs, shared schema, parser/backfill path, public API, skill detail/install UI, static moderation, LLM moderation context, and warnings for high-privilege events or hook types.
