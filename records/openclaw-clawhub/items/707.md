## Summary

This issue should remain open. The repository policy for openclaw/clawhub keeps issues out of auto-close scope, and current main still lacks the requested SKILL.md lifecycle hooks documentation, typed metadata contract, parser support, hook-aware security checks, API exposure, and skill detail UI disclosure.

## Best Possible Solution

Keep this issue open. The likely implementation path is to add a typed SKILL.md hooks declaration to the skill metadata contract and public skill-format documentation, parse and validate it from frontmatter, check lifecycle event names, hook types, command targets and missing files, flag privileged or blocking events in moderation, expose hook declarations through the public API, surface them on skill detail pages, and add focused parser, scanner, API, and UI tests. Treat #129 as related security background rather than the hook-specific tracker.
