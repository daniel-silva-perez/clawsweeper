The likely implementation path is to add a typed hooks declaration to the skill metadata contract and docs, parse it from SKILL.md frontmatter, validate lifecycle event names, hook types, command target existence, privileged/blocking events, and missing files, then expose hook declarations through the public API and skill detail UI with focused parser, scanner, API, and UI tests.
