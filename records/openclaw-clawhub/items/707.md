## Summary

Keep open. Current main still lacks first-class SKILL.md lifecycle hook documentation, normalized hook metadata, hook-specific security analysis, public API serialization, and skill page surfacing; the only current hooks capability I found is for plugin/package metadata, which is a separate surface.

## What This Changes

Request to document SKILL.md `hooks`, add hook-specific security checks, and surface hook declarations in registry/API/UI before install.

## Best Possible Solution

Keep this issue open for maintainer security/product review. Define the accepted SKILL.md hook metadata contract and risk policy first, then update docs, schema/parser/backfill/API serialization, skill detail UI, static moderation checks, LLM scan context, and install/inspect warnings or restrictions for high-privilege events and hook types.
