## Summary

Keep this issue open. The repository policy for openclaw/clawhub says issues should not be auto-closed by this workflow, and current main still lacks the requested SKILL.md lifecycle hooks documentation, typed metadata contract, parser support, hook-aware security checks, API exposure, and skill detail UI disclosure.

## Best Possible Solution

Keep this issue open. The likely implementation path is to add a typed SKILL.md hooks declaration to the skill metadata contract and docs, parse it from frontmatter, validate lifecycle event names, hook types, command targets and missing files, flag privileged or blocking events, expose hook declarations through the public API, surface them on skill detail pages, and add focused parser, scanner, API, and UI tests. Treat #129 as related security context rather than the canonical hook-specific tracker.
