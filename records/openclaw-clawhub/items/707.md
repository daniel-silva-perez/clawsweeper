## Summary

Keep open. Current main does not implement first-class SKILL.md lifecycle hook documentation, normalized hook metadata, hook-specific security checks, public serialization, or skill-page surfacing; the existing `hooks` references are for plugin/package capability metadata, which is a separate surface. The repository policy for this ClawHub review also keeps issues open rather than auto-closing them.

## What This Changes

Document SKILL.md lifecycle hooks, parse and surface normalized hook metadata, and add hook-specific security analysis and install/detail transparency for skills.

## Best Possible Solution

Define the accepted hook metadata model and risk policy first, then implement it across docs, shared schema, parser/backfill, public API, skill detail UI, static and LLM moderation context, and install/inspect warnings for high-privilege events and hook types.
