## Summary

Keep this PR open. Current main still omits `r` from both text-file extension whitelists, so the intended `.r`/`.R` publishing/import behavior is not implemented. The provided discussion and local inspection agree that the PR is a small additive whitelist change, but it has not landed on main.

## Best Possible Solution

Keep this PR open for maintainer review and merge the additive whitelist change in both schema copies, ideally with focused regression tests proving `.r` and uppercase `.R` are accepted through the shared schema and the CLI/web/GitHub import filtering paths. After merge, release the schema/CLI package so affected R skills can republish with their source files included.
