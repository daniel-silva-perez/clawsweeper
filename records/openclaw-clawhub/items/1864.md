## Summary

Current main still does not implement this PR's intended change. Both text-file whitelists omit `r`, and the CLI, web upload/archive, and GitHub import paths still lowercase extensions and require `TEXT_FILE_EXTENSION_SET` membership, so `.r` and `.R` sources remain excluded. The provided PR diff is narrow and I did not see security-sensitive changes, but it should stay open for maintainer review and merge.

## Best Possible Solution

Keep this PR open for maintainer review and merge the additive `r` whitelist entry in both schema copies, preferably with focused regression coverage for lowercase `.r` and uppercase `.R` through the shared schema helper and at least one filtering path. Before release, make sure the generated/published artifacts used by the schema package and CLI are rebuilt so the fix actually ships.
