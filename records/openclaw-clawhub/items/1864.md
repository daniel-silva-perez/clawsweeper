## Summary

Keep this PR open. Current main still omits `r` from the text-file whitelists, and the proposed patch is directionally right but incomplete because it updates `packages/schema/src` without updating the exported `packages/schema/dist` files that `clawhub-schema/textFiles` resolves to for web and Convex callers.

## What This Changes

The PR adds `r` to the two source text-file whitelist arrays so `.r` and `.R` files can be treated as publishable text files.

## Best Possible Solution

Update and merge the whitelist change with the exported schema dist files kept in sync, add targeted `.r`/`.R` regression coverage across the shared schema and CLI path, then run the relevant schema, CLI, lint, and typecheck validation before release.
