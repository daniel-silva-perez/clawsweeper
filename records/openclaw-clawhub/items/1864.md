## Summary

Current main still does not implement this PR's intended change. Both text-file extension whitelists omit `r`, and the CLI, web upload, archive, and GitHub import paths still lowercase extensions and require `TEXT_FILE_EXTENSION_SET` membership, so `.r` and `.R` sources remain excluded. The PR diff is narrow and does not introduce CI, dependency, script, secret, or release-path changes, so this should stay open for maintainer review and merge.

## Best Possible Solution

Keep this PR open for maintainer review and merge the additive `r` whitelist entry in both schema copies. Add focused regression coverage for both lowercase `.r` and uppercase `.R` through the shared schema helper and at least one upload/import filtering path, then release the schema/CLI package so affected R skills can republish with source files included.
