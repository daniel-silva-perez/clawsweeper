## Summary

Current main still omits `r` from both text-file extension whitelists, and the publish/import/upload paths still depend on those whitelists, so this PR is not obsolete and should remain open for maintainer review. Adds `r` to the shared and CLI-local text-file extension whitelists so `.r` and `.R` files are treated as text during skill publish, import, and upload flows.
