## Summary

Current main still renders the dashboard deployment drift warning when the frontend build SHA and Convex APP_BUILD_SHA are both present and different, and openclaw/clawhub policy keeps issues open for maintainer handling rather than auto-closing them.

## What This Changes

The issue reports a persistent red dashboard deployment version drift warning because the frontend expected SHA differs from the Convex-reported SHA, even though the dashboard still functions.

## Best Possible Solution

Keep this issue open for maintainer triage. First verify the live production frontend build SHA and Convex appMeta.getDeploymentInfo().appBuildSha values; if the mismatch is unintended, run the manual full production deploy from the same main SHA or fix the production stamping/deploy configuration so both deployables carry the same SHA. If maintainers decide the warning is too noisy during otherwise healthy operation, intentionally adjust DeploymentDriftBanner severity/policy and update the drift tests and deploy docs together.
