## Summary

Keep this issue open. The ClawHub cleanup policy only allows auto-closing PRs that are certainly implemented on main, and this is an issue. Current main still contains the drift check and banner behavior described in the report: the frontend build SHA is compared with Convex APP_BUILD_SHA and a warning is rendered when both normalized values differ.

## Best Possible Solution

Keep this issue open for maintainer triage. Verify the current production VITE_APP_BUILD_SHA and Convex APP_BUILD_SHA; if they differ unintentionally, run the manual full deploy path so frontend and backend are stamped from the same commit. If the warning is considered too noisy during otherwise healthy operation, adjust the DeploymentDriftBanner policy or severity in source with updated tests.
