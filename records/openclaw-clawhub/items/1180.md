## Summary

Keep this issue open. The repository cleanup policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on current main, and this item is an issue. Current main also still contains the reported behavior: the frontend build SHA is compared with Convex's APP_BUILD_SHA, and the dashboard-level banner renders an alert when both are present and different. The related #618 match is about CLI sync version reporting, not dashboard deployment-SHA drift, so it is not a canonical duplicate.

## Best Possible Solution

Keep this issue open for maintainer triage. Verify production VITE_APP_BUILD_SHA and Convex APP_BUILD_SHA; if they differ unintentionally, run the manual full deploy path so frontend and backend are stamped from the same commit. If the warning is considered too noisy during healthy operation, adjust DeploymentDriftBanner policy or severity and update the drift/banner tests.
