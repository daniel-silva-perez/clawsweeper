## Summary

Keep open. Current main still contains the deployment-drift alert path when frontend and Convex build SHAs are both present and unequal, while the live production check now shows a different operational state: Convex reports a current backend SHA, but the public frontend runtime env asset does not expose VITE_APP_BUILD_SHA. That makes this a maintainer deploy-stamping and warning-policy triage item, and the ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

Resolve or intentionally adjust the dashboard deployment drift warning caused by frontend and Convex build SHA mismatch.

## Best Possible Solution

Keep a clear production deployment contract: coordinated production deploys should stamp the same main SHA into Convex and the frontend, and the dashboard should alert only when that drift is actionable. If partial deploys are expected, adjust the banner severity/copy together with the drift tests and deploy docs; if the missing frontend stamp is accidental, fix the Vercel env/deploy configuration instead of weakening the signal.
