## Summary

Keep open. Current main still intentionally detects and renders the deployment drift alert when the frontend build SHA and Convex APP_BUILD_SHA differ, and the ClawHub review policy keeps issues open for maintainer handling. The related #618 cross-reference is not a duplicate because it tracks CLI sync version reporting, while this report is about frontend/backend deployment SHA drift.

## What This Changes

The issue requests resolving a persistent dashboard deployment drift warning where the frontend expects one build SHA but Convex reports a different SHA while the dashboard otherwise works.

## Best Possible Solution

Production deployments should either keep the frontend and Convex backend stamped to the same main SHA so the banner disappears during normal operation, or deliberately change the banner policy with matching tests and deploy docs if maintainers accept partial-deploy drift as non-alerting.
