## Summary

Keep open. Current main still renders the dashboard alert whenever the frontend build SHA and Convex APP_BUILD_SHA are both present and different, and the repository policy for ClawHub issues is to keep issues open for maintainer handling rather than auto-close them.

## What This Changes

The issue reports a persistent red dashboard deployment drift warning where the frontend expected build SHA differs from the Convex-reported build SHA even though the dashboard still works.

## Best Possible Solution

Keep this issue open for maintainer triage. First verify the live production frontend VITE_APP_BUILD_SHA and Convex appMeta.getDeploymentInfo().appBuildSha values; if the mismatch is unintended, run the manual full production deploy from the same main SHA or fix production stamping/deploy configuration so both deployables carry the same SHA. If maintainers decide the warning is too noisy during otherwise healthy operation, intentionally adjust DeploymentDriftBanner severity or policy and update the drift tests and deploy docs together.
