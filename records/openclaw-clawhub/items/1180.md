## Summary

Keep this issue open. Current main still implements the reported dashboard deployment-drift warning: the app compares the frontend build SHA with the Convex-reported APP_BUILD_SHA and renders an alert when both are present and different. The related #618 cross-reference is a separate CLI sync version-reporting issue, not a canonical fix for this dashboard warning.

## Best Possible Solution

Keep this issue open for maintainer triage. Verify the production VITE_APP_BUILD_SHA and Convex APP_BUILD_SHA values; if they differ unintentionally, run the manual full deploy path so frontend and backend are stamped from the same commit. If the warning is considered too noisy during otherwise healthy operation, adjust DeploymentDriftBanner policy or severity and update the drift/banner tests and deploy docs accordingly.
