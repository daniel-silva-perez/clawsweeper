## Summary

Keep this issue open. Current main has meaningful mitigations: valid API-token requests use per-user buckets, the CLI passes optional tokens and respects Retry-After, and docs describe auth-aware limits. But anonymous search/detail/download/install paths still consume per-IP buckets, client-IP fallback can collapse to unknown/shared buckets, related reports #390 and #1410 remain open, and the ClawHub cleanup policy keeps issues open rather than auto-closing them.

## Best Possible Solution

Keep this issue open for maintainer triage and consolidate with #390 and #1410 if desired. The best path is to choose one concrete direction: make CLI authentication the default onboarding path, adjust anonymous shared-IP limits, verify production client-IP forwarding, improve 429 diagnostics across install paths, or design official mirror/offline registry support.
