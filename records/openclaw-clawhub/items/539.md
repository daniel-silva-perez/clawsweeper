## Summary

Keep this issue open. Current main has meaningful mitigations for authenticated/API-token callers and retry-aware 429 handling, but anonymous search, detail, and download/install flows still use per-IP rate-limit buckets. The repository cleanup policy for openclaw/clawhub also keeps issues open; only clearly implemented PRs are eligible for auto-close.

## Best Possible Solution

Keep this issue open for maintainer triage and consolidate with #390 and #1410 if desired. The best path is to choose one concrete direction: make CLI authentication the default onboarding path, adjust anonymous shared-IP limits, verify production client-IP forwarding, improve 429 diagnostics across install paths, or design official mirror/offline registry support.
