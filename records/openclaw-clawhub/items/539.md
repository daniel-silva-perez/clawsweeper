## Summary

Keep this issue open. Current main has useful mitigations for authenticated CLI/API-token users and retry-aware 429 handling, but anonymous search, package metadata, file, and download paths are still per-IP limited. The ClawHub cleanup policy also says issues should stay open under this workflow; only clearly implemented PRs are eligible for auto-close.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to consolidate with #390 and #1410 if desired, then choose a concrete backend/product fix: make CLI authentication the default onboarding path, raise or reshape anonymous shared-IP limits, verify production client-IP forwarding, improve 429 diagnostics across all install paths, or design official mirror/offline registry support.
