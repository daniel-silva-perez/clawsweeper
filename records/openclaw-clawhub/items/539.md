## Summary

Keep this issue open. Current main partially mitigates the report through authenticated per-user buckets and Retry-After-aware CLI retries, and commenters already identified `clawhub login` as an immediate workaround, but anonymous search/install/download traffic from VPS or shared egress IPs is still IP-enforced. The repository policy for openclaw/clawhub also says issues should stay open; only PRs may be auto-closed when certainly implemented on main.

## Best Possible Solution

Keep this issue open for maintainer triage. The concrete path is to consolidate with #390 and #1410 if desired, then choose a product/backend fix: make CLI authentication the default onboarding path, raise or reshape anonymous shared-IP limits, verify production client-IP forwarding, improve 429 diagnostics across install paths, or design official mirror/offline registry support.
