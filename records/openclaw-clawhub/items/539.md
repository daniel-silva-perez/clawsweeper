## Summary

Keep this issue open. Current main has meaningful mitigations for authenticated requests and CLI retry behavior, but anonymous search/install/download traffic is still IP-bucketed, related shared-IP 429 reports remain open, and the repository policy only allows auto-close for PRs that are certainly implemented on main, not open issues.

## What This Changes

The issue requests relief for ClawHub registry API rate limiting that blocks search and installation from VPS/shared-IP environments, including better authentication, anonymous limits, caching/offline access, or mirror support.

## Best Possible Solution

Keep this issue open for maintainer triage and consolidate with #390 and #1410 if desired. The best path is to choose one explicit direction: make CLI authentication the default onboarding path, adjust anonymous shared-IP limits, verify production client-IP forwarding, improve 429 diagnostics across install paths, or design official mirror/offline registry support.
