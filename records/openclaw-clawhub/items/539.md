## Summary

Keep open. Current main contains important mitigations for authenticated users and better 429 retry behavior, but the anonymous/shared-IP search, install, and download failure mode is still plausible because anonymous registry traffic remains IP-bucketed and production client-IP attribution is still an operational question. The target repository policy also says ClawHub issues should remain open for maintainer handling; only PRs that are certainly implemented on main are eligible for auto-close.

## What This Changes

The issue requests relief from ClawHub registry API rate limiting that blocks VPS/shared-IP search, install, and download flows, including rate-limit policy, authentication, caching/offline, and mirror options.

## Best Possible Solution

Define one explicit shared-IP registry strategy: verify production client-IP attribution, keep authenticated requests on per-user buckets, decide the intended anonymous read/download policy, choose whether search/install should guide users into login by default, and document any mirror/offline registry path with matching backend and CLI coverage.
