## Summary

Keep open. Current main contains meaningful mitigations from merged auth-aware rate limiting and improved 429 retry handling, but the reported anonymous/shared-IP search and install failure mode remains plausible because anonymous registry traffic is still IP-bucketed and production client-IP attribution is still part of the question. The target repository policy also keeps ClawHub issues open for maintainer handling rather than auto-closing them.

## What This Changes

The issue requests changes to prevent ClawHub registry search and installation from being blocked by strict IP-based rate limits on VPS or shared-IP environments.

## Best Possible Solution

Define and ship one shared-IP registry strategy: verify production client-IP attribution, keep authenticated requests on per-user buckets, choose the intended anonymous read/download policy, make login guidance or onboarding explicit where appropriate, and document any mirror/offline registry path with matching backend, CLI, and docs coverage.
