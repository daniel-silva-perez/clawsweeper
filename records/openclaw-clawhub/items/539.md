## Summary

Keep this issue open. The ClawHub review policy says issues are not eligible for automated close, and current main only partially mitigates the report: authenticated CLI/API requests use per-user buckets and Retry-After handling exists, but anonymous search/download/install still use per-IP buckets and related rate-limit reports remain open.

## Best Possible Solution

Keep this issue open for maintainer triage. The most concrete path is to consolidate it with #390 and #1410 if maintainers want one canonical tracker, then decide whether to change default CLI onboarding to login, raise anonymous shared-IP headroom, harden production IP forwarding, improve anonymous install diagnostics, or document/build mirror/offline support.
