## Summary

Keep open. This is an issue, and the repository policy for openclaw/clawhub says issues are not eligible for automated close. Current main also only partially mitigates the report: authenticated requests use per-user buckets and CLI retry/token handling exists, but anonymous VPS/shared-IP search and install paths still use per-IP buckets and related reports remain open.

## Best Possible Solution

Keep this issue open for maintainer triage. The concrete path is to either consolidate it manually into #1410 and #390, or make a product/backend decision such as default CLI login, higher anonymous shared-IP limits, production IP-forwarding fixes, better anonymous install diagnostics, or documented mirror/offline support.

- Default CLI authentication, higher anonymous headroom, mirror/offline support, and any production policy changes are product decisions not fully implemented by current main.
