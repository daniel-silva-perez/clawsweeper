Keep open. Current main has partial mitigations for authenticated users and better 429 retry behavior, but the reported anonymous/shared-IP search and install failure mode is still plausible, related reports remain open, and this repository policy keeps ClawHub issues open for maintainer handling rather than auto-closing them.

## What This Changes

The issue asks ClawHub to reduce VPS/shared-IP rate-limit failures that block search and installation, with possible fixes around quotas, authentication, caching/offline access, and mirror support.

## Best Possible Solution

Define one explicit shared-IP rate-limit strategy: verify production client-IP attribution, keep authenticated requests on per-user buckets, decide whether anonymous search/install/download should get higher limits or be guided into login by default, and document any mirror/offline registry path separately with matching backend, CLI, and docs tests.
