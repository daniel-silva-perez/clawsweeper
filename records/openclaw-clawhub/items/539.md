## Summary

Keep open. Current main has meaningful mitigations for authenticated requests and better 429 retry handling, but anonymous search/install/download traffic still uses shared IP buckets and the production client-IP attribution question is still unresolved. The target-repository policy also says ClawHub issues should remain open for maintainer handling unless maintainers explicitly close them.

## What This Changes

The issue requests changes to reduce or avoid ClawHub registry API rate limiting that blocks search and install/download operations from VPS or shared-IP environments.

## Best Possible Solution

Define one explicit shared-IP rate-limit strategy: verify production client-IP attribution, keep authenticated requests on per-user buckets, decide whether anonymous search/install/download should get higher limits or default-login guidance, and document any mirror/offline registry path separately with backend, CLI, and docs coverage.
