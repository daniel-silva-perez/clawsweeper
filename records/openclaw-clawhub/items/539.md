## Summary

Keep this issue open. Current main has partial mitigations for authenticated ClawHub CLI users and retry-aware 429 handling, but anonymous search/package/download paths are still IP-bucketed and can still fail for VPS/shared-egress users. The repository policy for openclaw/clawhub also says issues should stay open; only PRs may be auto-closed when current main definitely implements them.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to consolidate with #390 and #1410 if desired, then choose a product/backend fix: make CLI authentication the default onboarding path, raise or reshape anonymous shared-IP limits, verify production client-IP forwarding, improve 429 diagnostics across all install paths, or design official mirror/offline registry support.
