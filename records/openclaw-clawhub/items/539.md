## Summary

Keep open. Current main has meaningful mitigations for authenticated users and CLI retry guidance, but anonymous search/install/download traffic is still enforced by IP bucket, client-IP attribution remains deployment-sensitive, and related shared-IP 429 reports remain open. The ClawHub review policy also keeps issues open unless a maintainer handles them explicitly.

## What This Changes

The issue asks ClawHub to stop blocking VPS/shared-IP users with registry API 429s during search and install by changing rate limits, authentication, caching, mirror support, or related deployment behavior.

## Best Possible Solution

Consolidate this with #390 and #1410, then choose one explicit path: verify production client-IP forwarding, adjust anonymous read/download buckets, make authentication the default install/search onboarding path, improve install-path 429 diagnostics, or define supported mirror/offline registry behavior. The chosen path should land with focused backend/CLI tests and docs that match production behavior.
