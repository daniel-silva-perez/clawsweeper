Keep this issue open. Current main has meaningful mitigations for authenticated/API-token callers and retry-aware 429 handling, but anonymous search, package/skill detail, and download/install flows still use per-IP rate-limit buckets. The repository cleanup policy for openclaw/clawhub also keeps issues open; only clearly implemented PRs are eligible for auto-close.
