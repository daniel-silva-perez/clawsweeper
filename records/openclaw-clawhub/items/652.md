## Summary

Keep open. Current main still creates skill backup commits without passing a Git author/committer identity, and ClawHub cleanup policy keeps issues open unless maintainers resolve them explicitly. The remaining work needs a maintainer attribution/privacy decision before a focused backend change.

## What This Changes

The issue requests that skill addition/update commits in openclaw/skills use the owner’s GitHub account details as the Git commit author instead of the ClawHub bot identity.

## Best Possible Solution

Keep this open for maintainer attribution policy. Once maintainers decide actor-vs-owner behavior and a privacy-safe email source, implement a future-only backend change that builds a GitHub author object from existing GitHub identity data and passes it through both publish-time and bulk backup paths into both skill and metadata create-commit payloads.
