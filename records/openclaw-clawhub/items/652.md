## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open, and current main still does not pass publishing-user GitHub author metadata into the skill backup commits.

## Best Possible Solution

Keep this issue open. Implement future-only commit attribution in the GitHub backup path by carrying the publishing user's GitHub identity from publish/page sync into backupSkillToGitHub, constructing a privacy-safe commit author object, deciding the committer policy explicitly, and adding focused tests that assert the GitHub commit payloads include the intended author metadata.
