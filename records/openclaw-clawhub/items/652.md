## Summary

Keep this issue open. It is an issue in openclaw/clawhub, and the repository cleanup policy for this run only permits auto-closing pull requests that are certainly implemented on main. A fresh shell check against current main also matches the prior ClawSweeper review: the GitHub backup commit creation path still does not pass a publishing user's GitHub identity or set author/committer metadata in the GitHub create-commit payloads.

## Best Possible Solution

Keep this issue open and implement future-only commit attribution in the GitHub backup path. Carry the publishing user's GitHub identity through publish-time and bulk sync backups, construct a privacy-safe Git author object, choose explicit committer behavior, include those fields in both GitHub create-commit payloads, and add focused tests for the emitted GitHub payload.
