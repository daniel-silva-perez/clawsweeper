## Summary

Keep this issue open. Current main still does not pass GitHub author identity into skill backup commits, and repository policy for ClawHub keeps issues open unless a merged PR is certainly implemented on main.

## Best Possible Solution

Keep this issue open. The best path is a maintainer decision on attribution semantics, then a future-only implementation that carries a privacy-safe GitHub author object through publish-time and bulk backup paths, adds it to both skill backup create-commit payloads, and covers the emitted payloads with focused tests. Existing backup history should stay untouched unless maintainers explicitly choose otherwise.
