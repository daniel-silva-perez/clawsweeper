## Summary

Keep this issue open. Current main still creates skill backup commits without passing a GitHub user author identity, and ClawHub cleanup policy keeps issues open unless a PR is certainly implemented on main. The remaining work is a maintainer attribution/privacy decision followed by a focused backend change.

## What This Changes

Use the skill owner's GitHub account identity as the Git author for openclaw/skills backup commits instead of the ClawHub bot/app identity.

## Best Possible Solution

Keep this issue open for maintainer attribution policy. Once maintainers decide actor-vs-owner behavior and a privacy-safe email source, implement a future-only backend change that builds a GitHub author object from existing GitHub identity data and passes it through both publish-time and bulk backup paths into both skill and metadata create-commit payloads.
