## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open, and current main still does not pass the publishing user's GitHub author metadata into skill backup commits.

## Best Possible Solution

Keep this issue open. Implement future-only commit attribution in the GitHub backup path by carrying the publishing user's GitHub identity into backupSkillToGitHub, constructing a privacy-safe commit author object, deciding committer behavior explicitly, and adding focused tests that assert the GitHub commit payload includes the intended author metadata.
