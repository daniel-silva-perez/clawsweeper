## Summary

Keep this issue open. ClawHub cleanup policy for this run only permits auto-closing pull requests that are certainly implemented on main, and this item is an issue. Current main also still does not implement the requested behavior: the GitHub backup publish and sync paths pass only an owner handle and never set GitHub commit author or committer metadata in the create-commit payloads.

## Best Possible Solution

Keep this issue open and implement future-only commit attribution in the GitHub backup path. Carry the publishing user's GitHub identity through publish-time and bulk sync backups, derive a privacy-safe Git author object, choose explicit committer behavior, include those fields in both GitHub create-commit payloads, and add focused tests for the emitted GitHub payload.
