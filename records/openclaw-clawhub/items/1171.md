## Clownfish Work Prompt

Fix the ClawHub publish/import Convex pagination issue for owner-publisher digest rebuilds.

Observable problem: reports in this cluster show `clawhub publish` and GitHub Import + publish failing with `This query or mutation function ran multiple paginated queries` in `syncSkillSearchDigestsForOwnerPublisherId` / `ensurePersonalPublisherForUser`. Current main schedules package and skill digest rebuilds separately, but `syncPackageSearchDigestsForOwnerPublisherId` and `syncSkillSearchDigestsForOwnerPublisherId` still loop over `.paginate()` inside a single internal mutation invocation.

Expected fix boundary: update `convex/functions.ts` so owner-publisher package and skill digest rebuilds process one page per Convex mutation invocation and schedule the next page with the cursor when needed. Prefer the existing cursor-batched pattern used in maintenance/backfill code. Do not replace the loops with unbounded `.collect()` unless you add a clear bounded limit and justification; avoid broad refactors of unrelated catalog/search queries. Add focused tests in `convex/functions.test.ts` that prove multi-page owner-publisher sync schedules the next batch instead of calling `.paginate()` repeatedly in the same invocation.

Likely files: `convex/functions.ts`, `convex/functions.test.ts`; if internal function signatures change, regenerate `convex/_generated/api.d.ts` and `convex/_generated/api.js` with Convex codegen.
