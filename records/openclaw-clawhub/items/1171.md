## Summary

Keep this issue open. ClawHub cleanup policy keeps issues open in this pass, and current main only partially mitigates the reported Convex pagination failure: the publisher trigger now schedules package and skill digest rebuilds separately, but each owner-publisher digest helper still loops over `.paginate()` inside one internal mutation invocation, so larger owner scopes can still hit Convex's single-paginated-query limit.

## Best Possible Solution

Keep this issue open and land one focused backend fix that converts the owner-publisher package and skill digest rebuilds in `convex/functions.ts` into cursor-batched internal mutations with at most one `.paginate()` call per Convex function invocation. Add regression tests for multi-page package and skill owner-publisher sync, then manually consolidate or close the related publish/import pagination reports after production verification.
