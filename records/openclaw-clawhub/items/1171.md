## Summary

Keep open. This is an open ClawHub issue, and the current main branch still has a concrete residual Convex pagination failure path: the publisher-trigger path now schedules package and skill digest sync separately, but each scheduled internal mutation can still execute `.paginate()` repeatedly for multi-page owners.

## What This Changes

The issue requests fixing GitHub import and publish failures caused by Convex's multiple-paginated-query limit in owner-publisher digest sync after a successful repo/skill detection flow.

## Best Possible Solution

Refactor owner-publisher digest rebuilds so each Convex function invocation performs at most one `.paginate()` call, processes that page, and schedules the next cursor until complete; keep the digest behavior unchanged and add regression tests for multi-page package and skill owners.
