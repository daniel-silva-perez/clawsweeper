## Summary

Keep open. This is an open ClawHub issue, and current main still has the reported failure shape: GitHub import wraps publish failures, publish still enters personal publisher setup, and owner-publisher digest sync can still loop over `.paginate()` inside one internal mutation for multi-page owners. The closed unmerged PR #1183 is useful context but did not land, and current tests only prove the package/skill sync split, not cursor continuation.

## What This Changes

The issue asks to fix GitHub import and publish failures caused by Convex's multiple-paginated-query limit after repository and SKILL.md detection succeeds.

## Best Possible Solution

Land a focused Convex backend fix that keeps the indexed owner queries, processes at most one paginated page per digest-sync mutation, self-schedules continuation mutations with `continueCursor`, preserves existing digest semantics and missing-table fallbacks, and adds multi-page regression coverage before maintainers close the duplicate publish-pagination reports after deployment.
