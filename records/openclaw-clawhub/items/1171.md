Audit the sibling owner-user package digest helper for the same pattern and either fix it consistently or explain why it is outside this PR. Use related refs #1175, #1176, #1182, #1184, #1195, #1197, #1201, #1205, and closed PR #1183 as context, but do not replace the loops with unbounded `.collect()` unless maintainers explicitly choose that tradeoff.
