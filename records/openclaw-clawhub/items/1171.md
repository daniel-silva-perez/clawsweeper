## Summary

Keep open. ClawHub cleanup policy keeps open issues open, and current main still has a concrete residual Convex pagination failure path: owner-publisher package and skill digest sync are scheduled into separate internal mutations, but each scheduled mutation can still execute `.paginate()` repeatedly for multi-page owners.

## What This Changes

Fix ClawHub GitHub import and publish so successful GitHub repo detection is followed by a successful publish instead of a Convex multiple-paginated-query failure in owner-publisher digest sync.

## Best Possible Solution

Keep this issue open and land one focused backend fix in `convex/functions.ts`: refactor owner-publisher package and skill digest rebuilds into cursor-batched internal mutations where each Convex function invocation performs at most one `.paginate()` call, processes that page, and schedules the next cursor until complete. Add regression tests for multi-page owner-publisher package and skill sync, then consolidate the sibling pagination-error reports after the fix ships.
