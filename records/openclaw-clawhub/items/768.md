## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that current main definitely implements. Substantively, current main has a narrow VirusTotal Code Insight false-positive suppression path and audited manual override tooling, but it does not implement the requested broad suppression whenever ClawScan is benign/high-confidence or a known-author whitelist. The live public scan for gateway-watchdog-lite v1.0.2 still shows VirusTotal suspicious versus ClawScan benign/high confidence, so this remains a maintainer/security moderation decision.

## What This Changes

Request a path to suppress the VirusTotal Suspicious warning for gateway-watchdog-lite when OpenClaw/ClawScan reports Benign with high confidence, or to whitelist manually reviewed known-author skills.

## Best Possible Solution

Keep this open for maintainer/security follow-up. Review gateway-watchdog-lite v1.0.2 and its current VirusTotal details; if maintainers agree it is a false positive, use the existing audited manual override path or define a narrow, tested policy for this exact false-positive class. Avoid a broad ClawScan-overrides-VT rule or known-author whitelist without explicit security criteria, and coordinate any stale-banner behavior with the related #838 thread.
