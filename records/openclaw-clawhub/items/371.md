Keeping this open. ClawHub cleanup policy keeps issues open, and current main still lacks a tailored false-positive appeal path: flagged skill pages, scan summaries, scanner detail pages, rescan UI, and the generic report dialog do not provide an owner-facing appeal link with safe skill and scanner context.

## What This Changes

The issue requests an owner-facing false-positive appeal link or button on flagged skill pages that routes to a security contact or templated issue.

## Best Possible Solution

Add a focused owner-facing false-positive appeal affordance on suspicious skill detail and security surfaces that includes safe context such as owner, slug, version, canonical URL, and scanner statuses, while keeping appeal, rescan, generic report, and staff override workflows distinct.

## Repair Work Prompt

Implement a focused false-positive appeal affordance for suspicious skill pages. Add an owner-facing appeal link or button on the suspicious skill banner and/or DetailSecuritySummary when a skill is flagged suspicious and the viewer can manage it. Prefer a private mailto to the existing security contact with a prefilled subject/body containing safe context: owner handle, skill slug, latest version, canonical skill URL, and high-level scanner statuses. Do not include raw scanner findings in a public URL. Do not reuse the generic report mutation/dialog, and do not change scanner verdict logic, rescan limits, moderation overrides, schema, or backend security policy. Update focused tests for the skill detail page and DetailSecuritySummary to assert the appeal affordance appears for owners of suspicious skills and remains distinct from Rescan and generic Report flows. Keep copy concise and avoid promising automatic clearance. Related refs: #329, #392, #579, #1669, #448, #645, #646, #656, #899.
