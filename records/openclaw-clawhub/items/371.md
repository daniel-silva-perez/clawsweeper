## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and current main still does not provide the requested false-positive appeal action: flagged skill pages link owners to the generic GitHub issues list, while the scanner summary/detail and report/rescan flows do not generate a tailored appeal email or prefilled issue with skill and scanner context.

## What This Changes

Add a visible false-positive appeal link or button on VirusTotal-flagged skill pages, preferably to email security@openclaw.ai or open a templated issue.

## Best Possible Solution

Add a focused appeal affordance on suspicious skill detail/security surfaces that carries safe context such as owner, slug, version, canonical URL, scanner verdicts, and scanner-detail links, while keeping rescan, report, and staff override flows distinct.
