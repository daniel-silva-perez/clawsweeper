## Summary

Keep open. This is an open ClawHub issue, and repository policy only permits cleanup auto-closing ClawHub PRs that are definitely implemented on main. Substantively, current main still has a scanner-visible env metadata gap: publish normalizes SKILL.md frontmatter, root metadata.json is only uploaded as a file, and the LLM registry summary prints required env vars from requires.env without folding in envVars or metadata.json env declarations.

## What This Changes

The issue asks ClawHub to make scanner-visible registry metadata recognize required environment variables declared in metadata.json or supported SKILL.md metadata, and to clarify uv/install handling so legitimate skills are not flagged suspicious for metadata mismatches.

## Best Possible Solution

Keep this open for maintainer product/security review. The best path is to choose one canonical scanner-visible env metadata contract, normalize required env names across requires.env, primaryEnv, envVars, and any explicitly supported root metadata.json shape, then update publish/parser/schema/docs/security-prompt tests plus the rescan/manual-override plan for already-flagged versions and related reports.
