## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a scanner-visible metadata gap for env declarations that are not normalized into the LLM registry summary, especially root metadata.json env declarations and detailed envVars-only metadata.

## Best Possible Solution

Keep this issue open and have maintainers choose the canonical scanner-visible metadata contract. The likely fix is to centralize required-env normalization across requires.env, primaryEnv, required envVars, and any explicitly supported metadata.json shape; update the LLM scanner summary/prompt and parser/publish tests; document whether root metadata.json is supported; and plan rescans for affected versions if historical verdicts should change.
