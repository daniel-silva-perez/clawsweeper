## Summary

Keep open. ClawHub cleanup policy only auto-closes PRs that current main definitely implements, and this is an issue. Substantively, current main still has a scanner-visible env metadata gap: publish parsing is centered on SKILL.md frontmatter, root metadata.json is only uploaded as a file, and the LLM registry summary renders required env vars from requires.env rather than normalizing primaryEnv, required envVars, or the reported metadata.json env shape.

## Best Possible Solution

Keep this issue open and have maintainers choose the canonical scanner-visible metadata contract. A likely implementation is to centralize required-env normalization across requires.env, primaryEnv, required envVars, and any explicitly supported root metadata.json shape; update assembleEvalUserMessage, publish/parser tests, CLI/schema expectations if needed, docs, and the rescan/manual-override plan for affected versions.
