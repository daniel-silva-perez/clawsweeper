## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still leaves the reported scanner-visible gap possible: root `metadata.json` env declarations are uploaded as files but not normalized into parsed registry metadata, while the LLM summary still renders required env vars from `requires.env` only.

## What This Changes

The issue asks ClawHub to expose declared environment variables from skill metadata to the registry/security scanner so legitimate secrets are not flagged as mismatches, and to clarify `uv` installer handling.

## Best Possible Solution

Define one canonical scanner-visible env metadata contract, then normalize required env names consistently across supported sources at publish/backfill time. The LLM registry summary, static moderation, UI/install prompts, docs, and owner rescan path should all consume the same normalized required-env set, with a clear plan for affected existing versions.
