## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy only permits auto-closing PRs that are certainly implemented on main. Substantively, current main still appears to derive registry/LLM env metadata from SKILL.md frontmatter or metadata.openclaw-style frontmatter, not standalone metadata.json env declarations like the report describes.

## Best Possible Solution

Keep this issue open for maintainer triage. The best product path is to decide whether standalone metadata.json env declarations are supported; if yes, parse uploaded metadata.json during publish/backfill into the same parsed.clawdis requires.env/envVars shape consumed by latestVersionSummary, detail UI, static scan, and the LLM prompt, with tests covering metadata.json plus SKILL.md frontmatter. If standalone metadata.json is not supported, update docs and CLI/server validation to make SKILL.md frontmatter support explicit. The secondary uv installer concern appears already documented and schema-supported.
