## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Substantively, current main still has only partial frontmatter support: SKILL.md `metadata.openclaw` and top-level frontmatter env declarations can be normalized, but I did not find a complete standalone `metadata.json`/`package.json` env-declaration path into the parsed registry metadata used by static scan, LLM scan, latest-version summaries, UI/API install prompts, or backfill/rescan. The related open reports #1432 and #1667 describe the same registry/env mismatch pattern, so this should remain available for maintainer triage.

## Best Possible Solution

Keep this issue open for maintainer triage. Decide whether standalone metadata.json/package.json env declarations are supported skill metadata sources. If yes, parse and merge them during publish into the same normalized parsed.clawdis requires.env/envVars shape used by latestVersionSummary, UI/API install prompts, static scan, and LLM prompt, then add regression tests plus a backfill/rescan path for affected versions. If no, update docs and CLI/server validation so publishers are told to declare env requirements in SKILL.md frontmatter only. Treat uv as already supported unless scanner calibration still warns on it.
