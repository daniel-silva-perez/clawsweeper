## Summary

Keep open. ClawHub cleanup policy keeps open issues out of auto-close scope, and current main still has the reported scanner-visible metadata gap: publish stores frontmatter-derived metadata, root `metadata.json` stays as uploaded file content, and the LLM prompt’s required-env summary is still based only on `requires.env`.

## What This Changes

The issue requests that ClawHub recognize declared environment variables, especially root `metadata.json` env declarations, so legitimate env use is not flagged as a registry mismatch, with a secondary clarification around `pip`/`uv` install metadata.

## Best Possible Solution

Define one scanner-visible env metadata contract, normalize required env names from supported sources into stored parsed metadata, make the LLM registry summary display that same normalized set, document the supported shape, add parser/publish/schema/security-prompt tests, and rescan or manually re-review the affected false-positive versions.
