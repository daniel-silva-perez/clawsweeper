## Summary

Keep this issue open. ClawHub policy for this cleanup pass only auto-closes PRs, and current main still has a real metadata-contract gap: SKILL.md frontmatter is documented and partially parsed, but root metadata.json env declarations are not normalized into registry metadata, and the LLM prompt still summarizes required env vars only from requires.env.

## Best Possible Solution

Keep the issue open and make a maintainer decision on the canonical metadata contract. The clean implementation path is to normalize supported env/config declarations consistently across SKILL.md frontmatter, detailed envVars, and any accepted root metadata.json format; derive scanner-visible required env vars from both requires.env and envVars where required is not false; expose the same requirement summary in public/API metadata where intended; add parser, publish, API, prompt, and moderation tests; then decide whether affected skill versions need rescan/backfill. If metadata.json is intentionally unsupported, document that clearly and adjust the scanner prompt so it does not treat that file as registry metadata.
