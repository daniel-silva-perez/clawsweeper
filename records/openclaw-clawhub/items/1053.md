## Summary

Keep open. Repository policy for openclaw/clawhub does not permit auto-closing issues, and current main still does not prove standalone metadata.json env declarations are ingested into the registry/scanner metadata path. The uv secondary concern appears documented and schema-supported, but the metadata.json mismatch remains a plausible platform bug if maintainers intend to support that file format.

## Best Possible Solution

Keep this issue open for maintainer triage. Decide whether standalone metadata.json env declarations are supported; if yes, parse uploaded metadata.json during publish/backfill into the same parsed.clawdis requires.env/envVars shape consumed by latestVersionSummary, the detail UI/API, static scan, and the LLM prompt, with regression tests for metadata.json plus SKILL.md frontmatter. If not supported, update docs and CLI/server validation so publishers get a clear error or warning directing them to SKILL.md frontmatter. Treat the uv point as already documented/schema-supported unless scanner wording still needs a small follow-up.
