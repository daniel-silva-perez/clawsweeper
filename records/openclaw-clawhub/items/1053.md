## Summary

Keep open. ClawHub cleanup policy does not auto-close open issues, and current main still leaves a scanner-visible env metadata gap: root metadata.json env declarations are uploaded as files but not normalized into parsed registry metadata, while the LLM summary prints required env vars from requires.env only.

## What This Changes

The issue asks ClawHub to make environment-variable declarations from metadata.json or supported SKILL.md metadata visible to the registry and security scanner so legitimate required secrets are not flagged as mismatches.

## Best Possible Solution

Define one canonical scanner-visible env metadata contract, then normalize required env names consistently across requires.env, primaryEnv, required envVars, and any explicitly supported root metadata.json shape. The LLM registry summary, UI/install prompts, static moderation, docs, and tests should all consume the same normalized set, with a clear plan for affected already-published versions.
