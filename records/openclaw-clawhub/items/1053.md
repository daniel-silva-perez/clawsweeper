## Summary

Keep open. ClawHub cleanup policy only permits auto-closing PRs, not issues, and current main still leaves a real scanner/registry metadata gap: root metadata.json env declarations are not normalized into registry metadata, and the LLM scanner summary still reports required env vars only from requires.env even though detailed envVars and uploaded file contents may be present.

## Best Possible Solution

Keep this issue open and have maintainers choose the canonical metadata contract for scanner-visible env requirements. The likely implementation should normalize required env vars consistently from requires.env and required envVars, decide explicitly whether root metadata.json env is supported or documented as unsupported, update the LLM scanner prompt/API metadata if intended, add parser/publish/prompt/moderation tests, and plan rescans for affected versions if historical suspicious verdicts should be corrected.
