## Summary

Keep open. ClawHub cleanup policy does not allow auto-closing open issues in this lane, and the current main code still leaves the reported scanner-visible mismatch possible because publish normalizes SKILL.md frontmatter but not a separate root metadata.json env object, while the LLM registry summary still renders required env vars from requires.env only.

## What This Changes

The issue asks ClawHub to make declared skill environment variables visible to registry-level security scans, especially declarations in metadata.json, and to clarify scanner handling of supported install kinds such as uv.

## Best Possible Solution

Define one canonical scanner-visible env contract, then make publish/backfill/security scan paths normalize required names consistently across requires.env, primaryEnv, envVars, and any explicitly supported root metadata.json shape. The LLM registry summary should show the same normalized required env set that UI/install prompts and static moderation use, and affected versions in related reports should have a clear rescan or moderation-review path.
