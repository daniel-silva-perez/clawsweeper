## Summary

Keep this ClawHub issue open. The cleanup policy for this repo does not auto-close open issues, and current main still shows a real metadata gap: SKILL.md frontmatter can populate `requires.env`, but `metadata.json` declarations are not ingested into the registry/prompt summary, and `envVars` are not emitted as required env vars in the LLM prompt.

## Best Possible Solution

Keep this issue open and implement a focused metadata normalization fix: parse supported env/config declarations from uploaded `metadata.json` or explicitly reject/document that file as non-registry metadata; normalize detailed `envVars` into scanner-visible required envs when `required: true`; include env/config requirements in the LLM prompt and public registry/API metadata where appropriate; add publish/parser/prompt tests covering `metadata.json`, `metadata.openclaw.requires.env`, `envVars`, and the uv/pip distinction; then rescan or backfill affected skill versions.

## Risks / Open Questions

- Existing suspicious flags may need a backfill/rescan after metadata extraction is fixed; changing only the docs or prompt may not clear already-published skills.

- The secondary pip/uv concern should be clarified separately: current install specs support `uv` but not `pip`, while dependency metadata supports `pip` as a dependency type.
