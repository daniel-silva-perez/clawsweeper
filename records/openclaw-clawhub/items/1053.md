## Summary

Keep open. The repository-specific policy for openclaw/clawhub keeps issues open, and current main still does not prove standalone metadata.json env declarations are ingested into the normalized registry/scanner metadata path. The secondary uv installer point appears documented and schema-supported, but the env metadata mismatch remains a plausible platform bug if maintainers intend metadata.json to be supported.

## Best Possible Solution

Keep this issue open for maintainer triage. Decide whether standalone metadata.json env declarations are a supported source. If yes, parse uploaded metadata.json during publish and backfill into the same parsed.clawdis requires.env/envVars shape consumed by latestVersionSummary, the public API/UI, static scan, and LLM prompt, then add regression tests for metadata.json plus SKILL.md frontmatter. If no, update docs and CLI/server validation so publishers get a clear warning to put env declarations in SKILL.md frontmatter. Treat the uv installer point as already supported unless scanner output still needs a focused prompt/calibration fix.
