## Summary

Keep open. This is a ClawHub issue, and the repo-specific cleanup policy only permits auto-closing PRs that are certainly implemented on main. Current main still derives registry and LLM env metadata from SKILL.md frontmatter or metadata.openclaw-style frontmatter, not standalone metadata.json env declarations like the report describes. Related reports #1432 and #1667 also remain open; the secondary uv installer concern appears documented and schema-supported.

## Best Possible Solution

Keep this issue open for maintainer triage. Decide whether standalone metadata.json env declarations are supported; if yes, parse uploaded metadata.json during publish/backfill into the same parsed.clawdis requires.env/envVars shape consumed by latestVersionSummary, detail UI, static scan, and the LLM prompt, with regression tests for metadata.json plus SKILL.md frontmatter. If standalone metadata.json is not supported, update docs and CLI/server validation to make SKILL.md frontmatter support explicit. The uv installer concern can be handled as already documented/schema-supported unless scanner prompt wording still needs tuning.
