## Summary

Keep open. The ClawHub cleanup policy says issues should remain open, and current main still only normalizes env declarations from SKILL.md frontmatter-shaped metadata. I did not find an end-to-end standalone metadata.json/package.json env ingestion path into parsed registry metadata, static scan declarations, LLM registry summary, latest-version summaries, or install/setup UI.

## Best Possible Solution

Keep this issue open for maintainer triage. Decide whether standalone metadata.json/package.json env declarations are supported skill metadata sources. If yes, parse and merge them during publish into the same normalized parsed.clawdis requires.env/envVars shape used by latestVersionSummary, UI/API install prompts, static scan, and LLM prompt, then add regression tests plus a backfill/rescan path. If no, update docs and CLI/server validation so publishers are told to declare env requirements in SKILL.md frontmatter only. Treat uv as supported unless a specific scanner warning still needs calibration.
