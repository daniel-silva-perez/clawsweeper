## Summary

Keep this issue open. It is an issue in openclaw/clawhub, and the repository-specific cleanup policy only permits auto-closing PRs that are certainly implemented on main. Current main also still only partially supports the requested owner-facing plugin package lifecycle: package soft-delete and a DELETE API exist, but owner restore/undelete, hide/unhide, CLI commands, dashboard controls, docs, and per-version de-list semantics remain incomplete.

## Best Possible Solution

Keep this issue open and finish plugin package lifecycle support end to end: define package-level versus release-level semantics, add package restore/undelete and optional version de-list backend/API paths, expose owner-facing CLI commands and dashboard controls, include a deleted/hidden package recovery view, and document the rollback workflow for plugin publishers. Coordinate version-specific behavior with #905 and validate it against concrete cleanup cases like #1744.
