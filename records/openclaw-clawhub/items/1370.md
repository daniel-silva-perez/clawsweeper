## Summary

Keep open. Current main still only provides partial package lifecycle plumbing: package and release soft-delete fields plus a package-wide DELETE API exist, but owner-facing package hide/undelete controls are missing from the CLI, site UI, restore API, version-level workflow, and docs. The supplied ClawHub cleanup policy also says issues stay open; only PRs are eligible for implemented-on-main auto-close.

## What This Changes

Add owner-facing hide/delete/undelete controls for ClawHub plugin packages, including package or version lifecycle workflows in site UI, CLI, API, and docs.

## Best Possible Solution

Keep this issue open and have maintainers define package-level versus release-level lifecycle semantics first. Then implement the chosen owner/admin workflow end to end across Convex package mutations/API, clawhub CLI commands, owner dashboard or management controls, tests, and public docs, coordinating version-specific behavior with #905 and stale plugin cleanup needs from #1744.
