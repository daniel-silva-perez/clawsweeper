## Summary

Keep this issue open. Current main has partial package lifecycle plumbing, including package/release soft-delete fields and a package DELETE API, but it still lacks the requested owner-facing hide/delete/undelete workflow for plugin packages across CLI, site UI, restore paths, version-level controls, and docs. ClawHub cleanup policy also keeps issues open; only PRs are eligible for implemented-on-main auto-close.

## Best Possible Solution

Keep this issue open and have maintainers define package-level versus release-level lifecycle semantics first. Then implement the chosen owner/admin workflow end to end across Convex package mutations/API, clawhub CLI commands, owner dashboard or management controls, tests, and public docs, while coordinating version-specific behavior with #905 and stale plugin cleanup needs from #1744.
