## Summary

Keep open. The supplied ClawHub cleanup policy keeps issues open, and current main still has only partial plugin package lifecycle support: package/release soft-delete state and a package-wide DELETE API exist, but owner-facing package hide, restore, and version-level withdrawal workflows are still missing.

## What This Changes

The issue requests owner/admin hide, delete, undelete, and version-delisting controls for ClawHub plugin packages across the website, CLI, API, and docs.

## Best Possible Solution

Define the package and package-release lifecycle model, then implement an owner/admin workflow across Convex mutations, HTTP routes, clawhub CLI commands, owner dashboard or package management UI, tests, and public docs, coordinating version-specific behavior with #905 and stale plugin cleanup needs from #1744.
