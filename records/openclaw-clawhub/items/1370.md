## Summary

Keep this issue open. Current main has partial package lifecycle plumbing, including package/release soft-delete fields and a package DELETE API, but it does not yet provide the requested owner-facing plugin package hide/delete/undelete workflow across restore semantics, CLI, dashboard UI, or docs. The ClawHub cleanup policy also says issues stay open; only PRs may be auto-closed as implemented on main.

## Best Possible Solution

Keep this issue open and have maintainers define package-level versus release-level lifecycle semantics first. Then implement the chosen workflow end to end across Convex package mutations/API, clawhub CLI commands, publisher dashboard controls including a recovery path, tests, and public docs, while coordinating version-specific behavior with #905 and stale plugin cleanup needs from #1744.
