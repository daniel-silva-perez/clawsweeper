## Summary

Keep this issue open. The ClawHub cleanup policy only allows auto-closing PRs, not issues, and current main still only partially covers the requested owner-facing plugin package lifecycle. Package-level soft delete and a DELETE API exist, but restore/undelete, hide/unhide, package CLI commands, dashboard controls, docs, and version-level de-list semantics are still missing or need maintainer product decisions.

## Best Possible Solution

Keep this issue open and have maintainers define the package-level versus release-level lifecycle semantics, then implement the chosen workflow end to end across Convex backend/API, clawhub CLI, dashboard controls including a recovery view, and public docs. Coordinate the version-specific part with #905 and validate against concrete stale plugin cleanup needs like #1744.
