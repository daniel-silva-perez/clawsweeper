## Summary

Keep open. The repository policy for this ClawHub cleanup pass keeps issues open, and current main still only has partial package lifecycle support: package/release soft-delete storage and a package-wide DELETE API exist, but owner-facing package hide/delete/undelete controls are still missing from the CLI, owner UI, restore API, version-level workflow, and docs.

## What This Changes

The issue requests owner/admin hide, delete, undelete, and optional version-level delist controls for ClawHub plugin packages, analogous to existing skill lifecycle controls.

## Best Possible Solution

Define the package and package-release lifecycle model, then implement an owner/admin workflow end to end across Convex mutations, HTTP routes, clawhub CLI commands, dashboard or plugin management UI, tests, and public docs, coordinating version-specific semantics with #905 and stale plugin cleanup needs from #1744.
