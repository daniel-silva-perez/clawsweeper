## Summary

Keep this issue open. The repository-specific policy says openclaw/clawhub issues should not be auto-closed, and current main still only has partial package lifecycle support: package soft-delete storage and a DELETE API exist, but owner-facing restore/undelete, hide/unhide, CLI/dashboard controls, docs, and package-release delisting remain unfinished.

## Best Possible Solution

Keep this issue open and finish plugin package lifecycle support end to end: define package-level versus release-level semantics, add package restore/undelete and optional package-release delist backend/API paths, expose owner-facing CLI commands and dashboard controls, include a deleted/hidden package recovery view, and document the rollback workflow for plugin publishers. Coordinate the version-specific behavior with open #905 and validate it against cleanup cases like #1744.
