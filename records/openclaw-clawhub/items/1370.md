## Summary

Keep this issue open. The repository cleanup policy says ClawHub issues should not be auto-closed, and current main still only partially covers the requested owner-facing plugin package lifecycle: package soft-delete and a DELETE API exist, but package restore/undelete, hide/unhide CLI commands, dashboard controls, docs, and owner-facing per-version de-listing are still missing. The related open cleanup/version-management items #905 and #1744 reinforce that the remaining work is still product-relevant.

## Best Possible Solution

Keep this issue open and finish plugin package lifecycle support end to end: define package-level versus release-level semantics, add package restore/undelete and optional version de-list backend/API paths, expose owner-facing CLI commands and dashboard controls, include a deleted/hidden package recovery view, and document the rollback workflow for plugin publishers. Coordinate per-version behavior with #905 and validate it against cleanup cases like #1744.
