## Summary

Keep open. Current main defaults CLI discovery and registry traffic to `https://clawhub.ai`, and the canonical live API is returning skills, but the reported legacy hostname still returns Vercel `DEPLOYMENT_NOT_FOUND`; for ClawHub issues, that remaining user-visible failure needs maintainer follow-up rather than auto-close.

## What This Changes

The issue reports that `clawhub explore` stopped working because it can use the legacy `registry.clawhub.ai` host, which now returns Vercel `DEPLOYMENT_NOT_FOUND`, while `clawhub search` still works.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment for backward compatibility, then consider a narrow CLI compatibility change that treats cached non-explicit `registry.clawhub.ai` values as legacy while preserving intentional custom registry overrides.
