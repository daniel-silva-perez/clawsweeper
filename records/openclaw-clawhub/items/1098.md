## Clownfish Work Prompt

Fix stale cached registry handling for `clawhub explore` after the `registry.clawhub.ai` deployment was removed. In `packages/clawhub/src/cli/registry.ts`, treat the hostname `registry.clawhub.ai` as a legacy cached registry host so resolver fallback does not reuse it when discovery fails and `getRegistry(..., { cache: true })` rewrites it after successful discovery. Add focused tests in `packages/clawhub/src/cli/registry.test.ts` for a cached `https://registry.clawhub.ai` value: one case where discovery succeeds and the cache is rewritten to `https://clawhub.ai`, and one case where discovery fails and resolution falls back to `DEFAULT_REGISTRY` instead of the stale cache. Preserve explicit custom override semantics for `--registry`, `CLAWHUB_REGISTRY`, and `CLAWDHUB_REGISTRY`; do not broaden the PR into release workflow, hosted Vercel config, or unrelated CLI behavior. No changelog is required unless the repo has a current CLI patch-note convention for this area.
