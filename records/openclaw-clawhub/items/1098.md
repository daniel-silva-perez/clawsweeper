## Summary

Keep this issue open. Current main has repaired the normal default path by resolving the CLI registry to the canonical https://clawhub.ai endpoint, and that live endpoint is returning skill data. However, ClawHub cleanup policy keeps issues open for maintainer triage, and the original stale host is still a real edge case: explicit or cached registry.clawhub.ai is not treated as legacy and still returns Vercel DEPLOYMENT_NOT_FOUND.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely code path is to add registry.clawhub.ai to the CLI legacy registry migration logic, add a regression test for explicit/cached stale registry behavior, and add a short recovery note for users who configured the dead host. Alternatively, maintainers can explicitly decide the dead host is unsupported when manually configured and close after documenting that stance.
