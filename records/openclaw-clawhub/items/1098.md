## Summary

Keep this issue open. Current main fixes the normal default CLI path by using `https://clawhub.ai`, but it does not migrate `registry.clawhub.ai` when a user has that stale host in explicit CLI/env config or cached global config. ClawHub cleanup policy also says issues should remain open for maintainer triage rather than auto-close.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely code fix is to add `registry.clawhub.ai` to the CLI legacy registry migration path, add regression tests for explicit CLI, env, and cached stale registry values, and document the recovery path or unsupported status for users who manually configured the old host. Maintainers should also verify whether the reported empty `https://clawhub.ai/api/v1/skills` response still reproduces in production.
