## Summary

Keep this issue open. Current main has repaired the normal default CLI path by using the canonical `https://clawhub.ai` registry, but the stale `registry.clawhub.ai` host is still not migrated for explicit or cached registry configuration. ClawHub cleanup policy also says issues should remain open for maintainer triage rather than auto-close.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely fix is to add `registry.clawhub.ai` to the CLI legacy registry migration path, add regression coverage for explicit and cached stale registry values, and document a recovery path for affected users. If maintainers choose not to support manually configured stale registry URLs, that decision should be documented before closing.
