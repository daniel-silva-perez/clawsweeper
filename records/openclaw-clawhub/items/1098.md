## Summary

Current main and the live canonical API no longer show the exact empty-list behavior for `https://clawhub.ai/api/v1/skills`, and the current CLI resolves the default registry through `https://clawhub.ai` rather than hard-coding `registry.clawhub.ai`. However, this is an issue, and the repository policy says to keep ClawHub issues open. There is also still a real legacy/ops wrinkle: `https://registry.clawhub.ai` still returns Vercel `DEPLOYMENT_NOT_FOUND`, so users on older CLI versions or explicit registry settings can still hit the reported failure.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The best outcome is to either restore/redirect `registry.clawhub.ai` for backwards compatibility, or confirm that `clawhub@0.12.0` plus the current well-known config is the supported migration path and reply with upgrade/config cleanup guidance. If maintainers want an additional repo-side hardening change, consider treating `registry.clawhub.ai` as a legacy registry host in `packages/clawhub/src/cli/registry.ts` with a focused regression test.
