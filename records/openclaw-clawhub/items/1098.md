## Summary

Keep this issue open. Current main and the current CLI release default discovery to `https://clawhub.ai`, and the canonical API is returning skills again, but the reported legacy host `registry.clawhub.ai` still returns Vercel `DEPLOYMENT_NOT_FOUND`, so older clients or explicit registry configurations can still reproduce the failure. Repository policy also limits auto-closing ClawHub issues.

## What This Changes

Investigate a report that `clawhub explore` stopped working because clients may contact `registry.clawhub.ai`, which returns Vercel `DEPLOYMENT_NOT_FOUND`, while search still works.

## Best Possible Solution

Keep the issue open for maintainer follow-up. The best outcome is to restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment for backward compatibility, or explicitly document and announce the supported migration path to current CLI defaults; a small follow-up can also add `registry.clawhub.ai` to CLI legacy-registry handling with focused tests.
