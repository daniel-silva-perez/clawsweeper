## Summary

Keep open. Current main now defaults discovery and CLI registry traffic to the canonical `https://clawhub.ai`, and the live canonical API is returning skills, but the reported legacy hostname `registry.clawhub.ai` still returns Vercel `DEPLOYMENT_NOT_FOUND`. Because this is a ClawHub issue and the user-visible legacy-host failure remains, it should stay open for maintainer follow-up rather than be auto-closed.

## What This Changes

The issue reports that `clawhub explore` stopped working because the registry host `registry.clawhub.ai` returns Vercel `DEPLOYMENT_NOT_FOUND`, while `clawhub search` still works.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment for backward compatibility, then consider a narrow CLI compatibility patch that treats cached non-explicit `registry.clawhub.ai` values as legacy while preserving intentional custom registry overrides.
