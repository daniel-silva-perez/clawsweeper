## Summary

Keep this issue open under ClawHub cleanup policy. Current main and the published CLI now default to https://clawhub.ai, and the live canonical skills API returns items, but the dead legacy host is still not classified as legacy in CLI registry migration and explicit/env stale registry values still point users at Vercel DEPLOYMENT_NOT_FOUND.

## Best Possible Solution

Keep this issue open for maintainer triage. The practical cleanup is a narrow CLI hardening PR: treat `registry.clawhub.ai` as a legacy cached registry host, add regression coverage for stale cached values and cache rewrite behavior, and leave explicit custom registry override semantics either unchanged or documented. After that ships, maintainers can decide whether the issue should be manually closed as resolved for current CLI users.
