## Summary

Keep this issue open. Current main routes the default CLI registry to the canonical https://clawhub.ai endpoint and the live canonical skills API is returning skill data, but this is a ClawHub issue and the provided cleanup policy says issues should remain open. There is also a remaining edge: explicit or cached use of https://registry.clawhub.ai still reaches a Vercel DEPLOYMENT_NOT_FOUND response and current registry migration logic does not classify that host as legacy.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely fix is to add registry.clawhub.ai to the CLI legacy registry migration path, add a regression test for that host, and document how users with explicit or cached stale registry config should recover. Maintainers could alternatively decide the manually configured dead host is unsupported, but that should be an explicit product/support call before closing.
