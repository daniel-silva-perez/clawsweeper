## Summary

Keep open. Current main defaults discovery and CLI requests to https://clawhub.ai and the canonical API is returning skills, but the originally reported legacy host still returns Vercel DEPLOYMENT_NOT_FOUND. ClawHub policy also keeps issues open rather than auto-closing them from this review lane.

## What This Changes

Request to make `clawhub explore` work again for users whose CLI resolves the registry to the legacy `registry.clawhub.ai` host, which currently returns Vercel DEPLOYMENT_NOT_FOUND.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The best outcome is to restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment for backward compatibility, and optionally add a focused CLI hardening patch that treats cached `registry.clawhub.ai` values as legacy while documenting the supported migration path to `https://clawhub.ai`.
