## Summary

Keep open. Current main mitigates new default CLI discovery by pointing at `https://clawhub.ai`, and the canonical API is live, but the reported legacy hostname `registry.clawhub.ai` still returns Vercel `DEPLOYMENT_NOT_FOUND`. Because this is a ClawHub issue and the user-visible legacy-host failure remains, it needs maintainer follow-up rather than auto-close.

## What This Changes

The issue reports that `clawhub explore` stopped working because it resolves to the legacy `registry.clawhub.ai` host, which returns Vercel `DEPLOYMENT_NOT_FOUND`, while `clawhub search` still works.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment for backward compatibility, then consider a narrow CLI compatibility patch that treats cached non-explicit `registry.clawhub.ai` values as legacy while preserving intentional custom registry overrides.
