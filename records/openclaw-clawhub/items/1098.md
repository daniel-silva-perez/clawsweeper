## Summary

Keep open. Current main mitigates new default CLI discovery by pointing at `https://clawhub.ai`, and the canonical API is live, but the reported legacy hostname still returns Vercel `DEPLOYMENT_NOT_FOUND`; because this is a ClawHub issue, repository policy also requires maintainer follow-up rather than auto-close.

## What This Changes

Restore `clawhub explore` for users whose CLI or config still resolves the registry to `https://registry.clawhub.ai`, which currently fails with Vercel `DEPLOYMENT_NOT_FOUND`.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment for backward compatibility, then consider a narrow CLI compatibility patch that treats cached non-explicit `registry.clawhub.ai` values as legacy while preserving intentional custom registry overrides.
