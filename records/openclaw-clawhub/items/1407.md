## Summary

Keep open. ClawHub cleanup policy does not auto-close issues in this sweep, and the live public API still shows the reported mismatch: latest metadata has advanced to 0.4.0, but the served SKILL.md hash/content matches the original Gmail-only file across later versions. Current main's source path stores and reads version-specific file blobs, so this needs production data/log inspection before a safe code fix is clear.

## What This Changes

The issue asks for skill pages to render the latest published SKILL.md content after new versions of auth0-token-vault are published.

## Best Possible Solution

Confirm the production skillVersions.files storage IDs, file hashes, upload request logs, latestVersionId/tags state, and page/API cache behavior for the affected publishes; then repair any bad rows and add a regression test or validation guard at the specific failing boundary.
