## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and current main still lacks a deterministic CLI/API path for exact-slug or slug-prefix skill enumeration; the available paths are ranked relevance search, installed-local listing, and bounded registry browsing.

## What This Changes

The issue requests registry-oriented ClawHub CLI discovery, such as prefix/exact search or a hub list mode, so users can enumerate matching skill slugs instead of relying on ranked search.

## Best Possible Solution

Add a narrow registry discovery surface that preserves ranked search while exposing exact slug lookup and slug-prefix enumeration through indexed, cursor-paginated backend queries, public v1 API support, CLI JSON/human output, and focused docs/tests.
