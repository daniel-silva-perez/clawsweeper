## Repair Work Prompt

Implement registry-oriented skill discovery for this issue. Preserve existing ranked `clawhub search <query>` behavior, but add deterministic exact-slug and slug-prefix enumeration for registry skills through an indexed Convex path, public v1 API support, CLI flags or hub-list mode, JSON output, docs, and tests. Prefer an indexed, cursor-paginated query over `skillSearchDigest` or another lightweight projection; do not use `.filter().collect()` or full-table scans. Consider adding an active slug index such as `skillSearchDigest.by_active_slug` if needed. Keep the change scoped to skill discovery and do not alter package publishing, install/update behavior, auth, moderation, or unrelated catalog semantics. Update docs for the chosen CLI shape and include examples for exact slug and prefix enumeration. Related context: current search has exact/prefix scoring but is still bounded relevance search; package catalog family=skill is a possible building block but lacks prefix/exact filters.
