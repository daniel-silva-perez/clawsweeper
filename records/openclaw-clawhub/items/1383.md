## Summary

Keep this issue open. The repository policy for openclaw/clawhub only permits auto-closing PRs that are certainly implemented on main, and current main still lacks the requested reliable CLI/API path for exact or prefix-based registry skill enumeration.

## Best Possible Solution

Keep this issue open. The best implementation would add an indexed backend discovery path for exact slug and slug-prefix queries, expose it through CLI flags such as `search --exact`, `search --prefix`, or a registry-backed `skills list --source hub --cursor --json` mode, and update API/CLI tests plus public docs.
