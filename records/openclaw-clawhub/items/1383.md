## Summary

Keep open. This is a ClawHub issue, and the supplied repository policy says ClawHub issues should remain open unless they are PRs certainly implemented on main. Current main also still lacks the requested registry-oriented prefix/exact/list discovery path: search is a ranked /api/v1/search wrapper, list is local lockfile-only, explore lists registry pages but has no prefix/exact or cursor CLI surface, and the public API docs/handlers expose no slug-prefix enumeration parameter.

## Best Possible Solution

Keep the issue open. The likely implementation path is an indexed slug-oriented registry discovery API plus CLI support, such as search --exact, search --prefix, or a registry-backed list/explore mode with cursor pagination and JSON output. The backend should avoid ranked/vector search for enumeration and use indexed slug constraints, with CLI tests and docs updates.
