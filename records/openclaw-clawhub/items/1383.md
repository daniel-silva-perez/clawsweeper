## Summary

Keep open. The repository-specific policy for ClawHub issues does not allow automated closure here, and current main still lacks the requested registry-oriented prefix/exact skill discovery path. CLI search remains a ranked `/api/v1/search` wrapper, local `list` is installed-lockfile only, `explore`/package browsing can list registry entries but do not provide prefix/exact filtering or a CLI cursor, and the public skill API handlers/docs expose only ranked search and paginated sorting filters.

## Best Possible Solution

Keep the issue open. A good implementation path is to add an indexed registry-discovery API for exact slug and slug-prefix enumeration, then expose it through CLI flags such as `clawhub search --exact`, `clawhub search --prefix`, or a registry-backed `explore`/`list` mode with `--cursor` and `--json`, plus focused CLI/API tests and docs updates.
