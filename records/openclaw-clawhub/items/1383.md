## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy says ClawHub issues should remain open unless they are pull requests certainly implemented on main. Current main still does not provide reliable CLI registry enumeration by slug prefix or exact slug: `search` remains a ranked `/api/v1/search` wrapper, `list` is local-lockfile only, `explore` can browse registry pages but has no prefix/exact/cursor flags, and the v1 skill APIs document only ranked search plus paginated sort filters.

## Best Possible Solution

Keep the issue open. A strong implementation would add an indexed registry discovery path for exact slug and slug-prefix queries, expose it through CLI flags such as `search --exact`, `search --prefix`, or a registry-backed `explore`/`list --source hub --cursor --json` mode, and update CLI/API tests plus public docs.
