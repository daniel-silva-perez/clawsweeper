## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Current main still has ranked skill search and bounded skill browsing, but no reliable CLI/API path for exact slug or slug-prefix registry enumeration.

## Best Possible Solution

Keep this issue open and implement a narrow indexed registry discovery path: exact slug lookup and slug-prefix enumeration for public hub skills, exposed through CLI flags or a hub-list mode with JSON/cursor support, plus API/CLI docs and focused tests. The fix should preserve ranked search semantics for normal `search <query...>` and avoid full-table scans.
