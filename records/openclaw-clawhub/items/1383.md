## Summary

Keep open. This is a ClawHub issue, and the supplied repository policy says ClawHub issues should remain open unless they are PRs certainly implemented on main. Current main also does not implement the requested registry-oriented prefix/exact discovery path: CLI search remains a ranked `/api/v1/search` wrapper, `list` is local lockfile-only, `explore` has registry listing but no prefix/exact filtering or CLI cursor flag, and the public API docs/handlers do not expose slug prefix or exact enumeration parameters.

## Best Possible Solution

Keep the issue open. The likely product path is to add an indexed registry-discovery API and CLI surface, such as `clawhub search --exact`, `clawhub search --prefix`, or a registry-backed `list`/`explore` mode with cursor pagination and JSON output. The backend should avoid ranked search for enumeration and use a slug-oriented indexed query, with CLI tests and docs updates.
