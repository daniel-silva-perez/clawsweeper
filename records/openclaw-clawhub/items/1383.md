## Summary

Keep open. This is an open ClawHub issue, and the repository-specific policy says ClawHub issues should remain open unless they are pull requests certainly implemented on main. Current main still does not provide reliable registry enumeration by slug prefix or exact slug: skill search is a ranked q/limit API wrapper, skill list is local-lockfile only, and registry explore/list APIs do not expose slug-prefix or exact-slug discovery filters.

## Best Possible Solution

Keep this issue open. The best implementation would add an indexed registry discovery path for skill exact-slug and slug-prefix queries, expose it through CLI flags such as search --exact, search --prefix, or a registry-backed list/explore --source hub --cursor --json mode, and update CLI/API tests plus public docs. The package catalog’s direct prefix-search pattern can inform the design, but the skill CLI/API surface still needs its own complete enumeration path.
