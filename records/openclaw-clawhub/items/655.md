## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the report still matches current behavior: the live Task-Specialist skill remains suspicious from `suspicious.dynamic_code_execution` while VT and LLM/OpenClaw scan signals are clean/benign. Current main still applies the dynamic-code matcher to raw shell file contents, so comments containing `eval()` can trigger the same reason code.

## What This Changes

The issue asks ClawHub to investigate and clear a suspected false positive where Task-Specialist is flagged as suspicious despite clean VirusTotal and OpenClaw scanner results.

## Best Possible Solution

Implement comment-aware static scanning for shell files so inert comments do not trigger `suspicious.dynamic_code_execution`, while real executable dynamic code remains suspicious. Add focused regression coverage, bump `MODERATION_ENGINE_VERSION`, and clear Task-Specialist through a rescan/backfill or approved staff override after deployment.
