## Summary

Keep open. Repository policy for this ClawHub sweep keeps issues open, and the underlying report is still valid: the live Task-Specialist skill remains suspicious from `suspicious.dynamic_code_execution` even though the current VT and LLM/OpenClaw scanner signals are clean/benign. Current main still applies the dynamic-code matcher to raw shell file contents, so a shell comment containing `eval()` can still trigger this reason code.

## What This Changes

The issue asks ClawHub to investigate and clear a suspicious moderation flag for the Task-Specialist skill when VirusTotal and OpenClaw scanner results are clean or benign.

## Best Possible Solution

Implement comment-aware static scanning for shell files so inert comments do not trigger `suspicious.dynamic_code_execution`, while real executable dynamic code remains suspicious. Add focused regression coverage, bump `MODERATION_ENGINE_VERSION`, and clear Task-Specialist through a rescan/backfill or staff-approved override after deployment.
