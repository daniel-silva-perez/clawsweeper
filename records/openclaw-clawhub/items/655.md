## Summary

Keep open. The ClawHub cleanup policy keeps open issues open, and the substantive report still reproduces: the live Task-Specialist version is flagged for `suspicious.dynamic_code_execution` even though VT and LLM/OpenClaw scanner signals are clean/benign. Current main still scans `.sh` files with a raw `eval(` matcher and only suppresses one allowlisted static code when external scanners are clean, so this is not implemented or obsolete.

## What This Changes

The issue asks maintainers to investigate and clear a likely false-positive ClawHub suspicious flag for the Task-Specialist skill after clean external security scan results.

## Best Possible Solution

Keep the issue open for security/moderation triage. The desired end state is a comment-aware static scan for shell files, focused regression coverage for inert shell comments versus real dynamic execution, a `MODERATION_ENGINE_VERSION` bump, and a production rescan/backfill or approved staff clean override for Task-Specialist once maintainers are satisfied the flag is false positive.
