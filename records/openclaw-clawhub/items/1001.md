## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy for this repository only permits auto-closing pull requests that are definitely implemented on current main. The reported behavior also still appears present: the skill README renderer does not rewrite relative Markdown links, while the owner/slug route has no packaged-file child route; related #1681 tracks the broader misrouting risk and draft PR #1858 is already the active implementation candidate.

## What This Changes

Relative links from a skill's rendered SKILL.md, such as references/system-design.md, should open the packaged file in the web viewer instead of navigating to a 404 or unrelated owner path.

## Best Possible Solution

Land or replace draft PR #1858 with a focused fix that rewrites safe relative SKILL.md links to the existing raw file API, adds regression coverage for normal reference files and security edge cases, and keeps this issue linked with #1681 until the fix is merged.
