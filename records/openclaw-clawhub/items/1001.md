## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on main. The reported bug also still appears present: README-relative links are rendered as ordinary links, while bundled files are only available through the Files tab or raw file API.

## Best Possible Solution

Keep this issue open for maintainer follow-up. Implement a safe relative-link resolver for skill README rendering, likely in SkillDetailTabs or a shared MarkdownPreview path, that recognizes files in latestFiles and routes them to an in-app file viewer or /api/v1/skills/{slug}/file?path=...; add regression coverage for references/system-design.md, URL encoding, query/hash preservation, case-insensitive matches, missing files, and blocked unsafe schemes.
