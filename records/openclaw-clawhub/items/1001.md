## Summary

Keep open. This is an open ClawHub issue, and the sweep policy only permits auto-closing pull requests that current main definitely implements. The reported behavior also still appears present: rendered SKILL.md links are not rewritten to a skill-scoped file URL, while the owner/slug route has no packaged-file child route. Related #1681 tracks the broader same-domain misrouting/security form, and draft PR #1858 is already the active implementation candidate.

## What This Changes

The issue asks for relative links in rendered SKILL.md content, such as `references/system-design.md`, to open packaged skill files on the web instead of navigating to a 404 page route.

## Best Possible Solution

Land or revise the existing focused relative-link fix so safe SKILL.md relative links resolve through `/api/v1/skills/{slug}/file?path=...`, with regression coverage for normal packaged files, anchors, external schemes, and traversal rejection.
