## Summary

Keep open. This is a ClawHub issue, and the repository-specific review policy says issues should not be auto-closed. Current main still does not resolve SKILL.md relative links to packaged files: the README renderer has no custom anchor or URL resolver, the public skill route only covers owner/slug, and the existing file viewer/API surfaces are not wired to markdown links.

## Best Possible Solution

Keep this issue open. Implement a safe relative-link resolver for skill README rendering that recognizes latest-version packaged files and routes them either to an in-app rendered file viewer or to /api/v1/skills/{slug}/file?path=. Add regression coverage for references/system-design.md, URL encoding, query/hash preservation, case handling, and blocking unsafe schemes such as javascript:.
