## Summary

Keep open. Repository policy for ClawHub issues does not permit auto-closing this issue, and current main still renders SKILL.md markdown without a resolver for bundled relative file links. Related #1681 and draft PR #1858 remain the active fix path rather than proof that this issue is implemented.

## What This Changes

The issue requests that links from a skill README to bundled files such as `references/system-design.md` open a viewable web page instead of navigating to a 404 route.

## Best Possible Solution

Keep this issue open and resolve it through #1681 and draft PR #1858, or an equivalent focused PR. The best fix is a README-link resolver for skill detail pages that rewrites safe relative bundled-file links to the skill file API or an in-app bundled-file viewer, preserves allowed external/hash/mail/tel links, rejects traversal and unsafe schemes, handles nested paths plus URL encoding/query/hash behavior, and adds regression tests around the link helper and `SkillDetailTabs`.
