## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on main. The underlying bug also still appears present on current main: SKILL.md is rendered without a relative file-link resolver, while bundle files are available only through the Files tab or raw file API.

## Best Possible Solution

Keep this issue open for maintainer follow-up. Implement a safe relative-link resolver for skill README rendering, likely in SkillDetailTabs or a shared MarkdownPreview path, that recognizes files in the latest version bundle and routes them to an in-app file viewer or /api/v1/skills/{slug}/file?path=...; add regression tests for references/system-design.md, URL encoding, query/hash preservation, case-insensitive matches, missing files, and blocked unsafe schemes.
