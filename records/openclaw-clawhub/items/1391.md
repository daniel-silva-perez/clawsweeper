## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Substantively, current main still has only partial `.git/` filtering and no shared exclusion for `__pycache__/`, `*.pyc`, or `*.pyo` across the skill/package upload, publish normalization, static scan, and VirusTotal ZIP paths.

## Best Possible Solution

Keep this issue open. Implement a shared artifact exclusion helper used by CLI skill/package collection, web folder/archive expansion, HTTP/API publish normalization, server-side release persistence, static scan input construction, and VirusTotal ZIP construction. Exclude `.git/`, `__pycache__/`, `*.pyc`, and `*.pyo`; preserve existing `.clawhubignore` and documented skill `.gitignore` behavior; add focused tests for CLI, web, API, static scan, and VT payload paths.
