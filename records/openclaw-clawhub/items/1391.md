## Summary

Keep open. This is an open ClawHub issue, and this review lane only auto-closes ClawHub PRs that are certainly implemented on current main. The source review also shows the reported behavior is still only partially addressed: package/CLI paths ignore `.git/`, but web/API/GitHub skill publishing still lacks shared `.git/`, `__pycache__/`, `.pyc`, or `.pyo` cleanup before storage and moderation.

## Best Possible Solution

Keep this issue open and land a maintainer-approved artifact cleanup change. The best implementation is a shared publish-artifact ignore helper that excludes `.git/`, `__pycache__/`, `.pyc`, and `.pyo` before files are stored or passed into static/LLM moderation, with coverage across web skill upload, API JSON/multipart publish, GitHub import, CLI skill publish, and package upload paths where applicable. Update skill-format or publish docs if this becomes a supported publish contract.
