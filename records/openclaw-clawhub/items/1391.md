## Summary

Keep open. This is an open ClawHub issue, and the supplied repository policy only permits auto-closing ClawHub pull requests that are certainly implemented on main. Current main also still lacks a shared built-in exclusion for Python cache artifacts, and server-side package normalization still does not enforce the same artifact ignore policy used by some client helpers.

## Best Possible Solution

Keep this issue open and implement a shared artifact exclusion policy used by CLI collection, web upload/archive expansion, JSON and multipart publish normalization, server-side release persistence, and scan payload construction. Include .git/, __pycache__/, *.pyc, and *.pyo; preserve .clawhubignore behavior; add focused tests for CLI, web, HTTP/API, multipart, and VirusTotal/static-scan payload paths; and document .clawhubignore as the interim workaround.
