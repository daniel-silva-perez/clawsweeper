## Summary

Keep open. ClawHub cleanup policy keeps issues open in this lane, and current main still only partially handles the reported artifact false positives: package and CLI paths ignore `.git/`, but skill upload, multipart API, GitHub import, and backend publish scan paths still rely on macOS-junk filtering before storage and scanning. Related scanner false-positive issue #1826 remains open and is not a merged fix or canonical replacement for this artifact-cleanup request.

## What This Changes

The issue asks ClawHub to stop security-scan false positives caused by standard VCS and Python generated artifacts such as `.git/`, `__pycache__/`, `.pyc`, and `.pyo` in skill uploads.

## Best Possible Solution

Add a small shared publish-artifact cleanup helper used before persistence and static/LLM moderation across skill publish entry points, covering only well-known generated/VCS artifacts such as `.git/`, `__pycache__/`, `.pyc`, and `.pyo` while preserving normal source-file scanning and existing macOS-junk handling.
