## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on main. Substantively, current main still lacks a shared artifact-ignore policy for `__pycache__/`, `*.pyc`, and `*.pyo`; `.git/` is filtered in normal CLI/web client paths, but package JSON and multipart publish paths can still persist arbitrary artifact paths before VirusTotal scanning.

## Best Possible Solution

Keep the issue open. Implement a shared publish-artifact ignore policy used by CLI package collection, web upload filtering, multipart/API publish normalization, server-side release persistence, and the scan zip path. Include `__pycache__/`, `*.pyc`, and `*.pyo`, preserve existing `.git/` behavior, add focused tests for each publish surface, and document `.clawhubignore` as the current workaround until the built-in exclusions ship.
