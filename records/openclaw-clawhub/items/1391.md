## Summary

Keep open. Repository policy keeps ClawHub issues open in this cleanup lane, and current main still only partially handles the report: package and CLI publishing ignore some VCS artifacts, but skill upload/API/GitHub import paths still only filter macOS junk before storage and scan input.

## What This Changes

The issue asks ClawHub to exclude standard Git and Python generated artifacts such as `.git/`, `__pycache__/`, `.pyc`, and `.pyo` from skill publishing and security-scan inputs.

## Best Possible Solution

Introduce a shared publish-artifact cleanup helper used by skill upload expansion, multipart skill API parsing, GitHub import, and backend publish filtering before storage and static/LLM moderation. It should cover `.git/`, `__pycache__/`, `.pyc`, and `.pyo` while preserving normal source-file scanning.
