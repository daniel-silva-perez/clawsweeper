## Summary

Keep this issue open. Current main still does not implement the requested built-in exclusion for `__pycache__/` or Python bytecode, and the ClawHub cleanup policy only allows auto-closing PRs that are certainly implemented on main, not issues. `.git/` is covered in the normal CLI and web package-filtering paths, but server-side package normalization and multipart/API publish paths do not provide a shared artifact ignore policy before files are persisted and later zipped for VirusTotal scanning.

## Best Possible Solution

Keep the issue open and implement a shared package artifact ignore policy used by CLI collection, web upload filtering, multipart/API publish normalization, and server-side release persistence. Include `__pycache__/`, `*.pyc`, and `*.pyo`, preserve the existing `.git/` behavior, and add focused tests for CLI local publish, web helper filtering, multipart package publish, JSON/API publish normalization, and the scan payload path. Document `.clawhubignore` as the current workaround until the built-in exclusions ship.
