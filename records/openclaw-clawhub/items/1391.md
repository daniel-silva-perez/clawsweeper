## Summary

Keep open. Repository policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on main, and this item is an issue. Substantively, current main still lacks a shared built-in exclusion for `__pycache__/`, `*.pyc`, or `*.pyo`; `.git/` is filtered in normal CLI/web package upload paths but not enforced by server-side publish normalization before package files are persisted and zipped for scanning.

## Best Possible Solution

Keep this issue open and implement a shared package artifact ignore policy used by CLI collection, web upload filtering, multipart/API publish normalization, server-side release persistence, and scan payload construction. Include `.git/`, `__pycache__/`, `*.pyc`, and `*.pyo`, preserve existing `.clawhubignore` behavior, add focused tests across CLI, web, JSON/API, multipart, and VirusTotal payload paths, and document `.clawhubignore` as the current workaround until the built-in exclusions ship.
