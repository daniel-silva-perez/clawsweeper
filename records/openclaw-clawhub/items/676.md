Keep open. Repository policy for openclaw/clawhub says issues should remain open during this cleanup, and current main still contains the reported import-publish failure path: GitHub import wraps publish errors, publish runs the GitHub account-age gate, and that gate still throws the exact GitHub API rate-limit message for 403/429 responses.

## Best Possible Solution

Keep this issue open for maintainer triage, likely alongside open #1222. Verify production Convex logs and `GITHUB_TOKEN` configuration, then improve `requireGitHubAccountAge` to distinguish true GitHub rate limits from other 403 responses, expose useful retry/reset detail when available, and reduce repeated external lookups by caching or syncing `githubCreatedAt` earlier.
