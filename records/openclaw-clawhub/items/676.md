## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy for this repository only allows auto-closing PRs that are certainly implemented on current main. Current main also still contains the reported failure path: GitHub import wraps publish failures in the exact reported message shape, publish runs the GitHub account-age gate by default, and that gate still maps GitHub 403/429 responses to the rate-limit error when `githubCreatedAt` is missing.

## Best Possible Solution

Keep this issue open for maintainer triage and consolidate it with #1222 and, if it is the same account-age gate failure, #654. The best path is to inspect production Convex failure logs and `GITHUB_TOKEN` configuration, then either fix deployment configuration or make the account-age lookup more robust by distinguishing true GitHub rate limits from other 403 responses, surfacing retry/reset details where available, and reducing critical-path lookups by caching `githubCreatedAt` earlier without weakening the account-age gate.
