## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy says ClawHub issues should remain open. Source review also shows current main still contains the reported failure path: GitHub import calls the publish pipeline, publish runs the GitHub account-age gate, and that gate still turns GitHub 403/429 responses into the exact rate-limit error that import wraps with the reported message.

## Best Possible Solution

Keep this issue open for maintainer triage, likely together with open #1222. Verify production Convex GITHUB_TOKEN configuration and Convex logs, then improve requireGitHubAccountAge to distinguish true rate limits from other 403 responses, surface retry/reset detail when available, and reduce repeated GitHub lookups by caching or syncing githubCreatedAt earlier.
