Keep open. ClawHub cleanup policy keeps issues open, and current main still has a reachable import-to-publish path that can emit the reported GitHub API rate-limit message when the GitHub account-age lookup receives a 403 or 429 from GitHub.

## What This Changes

Publishing a skill through GitHub import fails with a GitHub API rate-limit error that does not resolve after waiting.

## Best Possible Solution

Keep this issue in the publish/import rate-limit cluster, verify production Convex failures and GITHUB_TOKEN configuration first, then either correct deployment configuration or refine the GitHub account lookup so true rate limits have actionable retry/reset detail and uncached first-publish lookups are reduced without weakening the abuse gate.
