## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Current main also still has the reported GitHub import publish failure path: import wraps publish errors in the exact user-facing message shape, publish still runs the GitHub account-age gate, and the gate still maps GitHub 403/429 responses to the reported rate-limit error.

## Best Possible Solution

Keep this issue open for maintainer triage, likely alongside #1222. Verify production Convex GITHUB_TOKEN and failure logs, distinguish true GitHub rate limits from other 403 responses, surface retry/reset details where available, and reduce critical-path GitHub lookups by syncing, caching, or backfilling githubCreatedAt earlier in auth/profile flows.
