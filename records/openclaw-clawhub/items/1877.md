## Summary

Keep open. ClawHub cleanup policy only permits auto-closing pull requests, and the reported regression still appears real: current main preserves and exposes the clean staff override, but the public sidebar and scanner detail route still render raw latest-version scan statuses without applying that override.

## What This Changes

The issue requests that a staff-cleared skill version continue to show the moderated clean status consistently after a ClawHub rescan/update instead of showing raw suspicious scanner cards.

## Best Possible Solution

Keep this issue open and have maintainers confirm the public semantics for staff-cleared scan findings. The likely code path is to pass moderationInfo or a derived staffCleared state into DetailSecuritySummary and the skill scanner detail route, show a consistent staff-cleared verdict/message to non-staff viewers, preserve raw scanner evidence for staff/owner or transparent detail views as intended, and add focused React tests alongside existing backend moderation coverage.
