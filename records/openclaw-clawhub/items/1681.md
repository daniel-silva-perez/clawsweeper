## Summary

Keep open. This is an issue in openclaw/clawhub, and the repository-specific cleanup policy only allows automated close proposals for pull requests that are certainly implemented on current main. Substantively, current main still appears to render skill README markdown links without resolving relative paths into the skill file API, while draft PR #1858 remains the linked implementation candidate.

## Best Possible Solution

Keep this issue open and review/land PR #1858 or an equivalent fix. The likely product path is to resolve relative README links in the skill renderer to /api/v1/skills/{slug}/file?path=... while preserving external URLs, mailto/tel links, and hash anchors, rejecting traversal, and adding regression coverage for rendered README links.
