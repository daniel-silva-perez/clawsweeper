## Summary

Keep this issue open. Current main still appears affected, and an open draft PR (#1858) is already linked as the implementation candidate. The ClawHub cleanup policy also says issues should remain open in this review path.

## Best Possible Solution

Keep open and review/land PR #1858 or an equivalent fix. The narrow product path is to rewrite relative Markdown links in SkillDetailTabs to /api/v1/skills/{slug}/file?path=... while preserving external, mailto/tel, and hash-only links and rejecting traversal, with tests for relative links and rendered README anchors. A route-based fix would need both trailing-slash canonicalization and a real skill-file catch-all route.
