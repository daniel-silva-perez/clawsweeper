## Summary

Keep open. Current main still renders skill README links without a relative-link resolver, keeps skill pages on non-directory /$owner/$slug URLs, and the open draft implementation PR #1858 is unmerged. Repository policy for ClawHub issues also requires keeping this issue open until a fix lands.

## What This Changes

Request that relative Markdown links in skill documentation resolve within the skill's own files instead of escaping to owner-level routes or unrelated skills.

## Best Possible Solution

Keep this issue open and review or land PR #1858, or an equivalent fix on current main. The fix should rewrite safe relative README links to /api/v1/skills/{slug}/file?path=..., preserve external URLs, mailto/tel links, and hash anchors, reject traversal or malformed relative paths, and add regression coverage for both the resolver and SkillDetailTabs rendering. Treat #1001 as the same fix cluster.
