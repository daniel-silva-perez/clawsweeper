## What This Changes

The issue asks ClawHub to keep relative links in skill README content inside the current skill namespace instead of allowing browser URL resolution to escape to sibling routes.

## Best Possible Solution

Land PR #1858 or an equivalent focused fix that rewrites safe relative README links to the skill file API, rejects traversal or malformed relative paths, preserves non-relative links, and covers both the resolver and SkillDetailTabs rendering with regression tests.
