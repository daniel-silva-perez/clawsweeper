## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review path only permits auto-closing ClawHub pull requests that are certainly implemented on main. Substantively, current main still renders skill README links as authored on the slashless `/$owner/$slug` page, with no trailing-slash canonicalization, no README relative-link rewrite, and no owner/slug catch-all file route.

## Best Possible Solution

Keep open and fix this in core. The lowest-blast-radius path is to rewrite relative README anchor and image targets in the skill detail markdown renderer to the existing `/api/v1/skills/{slug}/file?path=...` endpoint while preserving external, hash, and mailto links, with regression tests for nested relative paths from `SKILL.md`. A broader route-model fix could add trailing-slash canonicalization plus a real `/$owner/$slug/*` skill-file route.
