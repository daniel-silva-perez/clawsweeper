## Summary

Keep open. This is an open ClawHub issue, and the repository policy only permits automated close proposals for ClawHub PRs that are certainly implemented on current main. Substantively, current main still renders skill README markdown links without a skill-file resolver, the skill route and canonical URL remain non-directory owner/slug URLs, and draft PR #1858 is an open unmerged implementation candidate that targets this issue.

## Best Possible Solution

Keep this issue open and review or land PR #1858, or an equivalent current-main fix. The fix should rewrite safe relative README links to `/api/v1/skills/{slug}/file?path=...`, preserve external URLs, mailto/tel links, and hash anchors, reject traversal or malformed relative paths, and add regression coverage for both the resolver and rendered SkillDetailTabs output. Related issue #1001 should be considered part of the same fix cluster.
