## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub PRs that current main certainly implements. Current main still renders README Markdown links without a relative-link rewrite or trailing-slash route shape, and draft PR #1858 is the open implementation candidate with closing syntax.

## What This Changes

The issue requests that relative links in skill documentation resolve within the viewed skill's file namespace instead of escaping to owner-level or unrelated ClawHub routes.

## Best Possible Solution

Land PR #1858 or an equivalent focused fix that resolves safe relative README links to the skill file API, rejects traversal or malformed paths, preserves non-relative links, and covers both the resolver and SkillDetailTabs rendering with regression tests; then verify whether related #1001 is covered by the same change.
