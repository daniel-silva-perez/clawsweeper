## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits automated close proposals for pull requests that are certainly implemented on current main. Substantively, current main still renders skill README markdown without resolving relative links into the skill file API, and the linked implementation candidate, PR #1858, is still open and unmerged.

## Best Possible Solution

Keep this issue open and review or land PR #1858, or an equivalent current-main fix that rewrites skill README relative links to /api/v1/skills/{slug}/file?path=... while preserving external URLs, mailto/tel links, and hash anchors, rejecting traversal, and adding regression coverage for rendered README links.
