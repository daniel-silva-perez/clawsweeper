## Summary

Keep this issue open. The repository-specific ClawHub cleanup policy only permits automated closure for PRs that are certainly implemented on current main, and this item is an issue. Source review also does not prove the deleted-account OAuth/session report is fully resolved: current main intentionally deactivates and purges deleted users, blocks reauthentication, returns null for inactive users, and can therefore leave the UI in a signed-out state unless the callback error path is surfaced. Related deleted-account login-loop issue #886 is still open.

## Best Possible Solution

Keep this issue open for maintainer triage. Coordinate it with #886, verify the production OAuth callback/session state for a deleted account, decide the intended permanent-deletion versus purge/re-registration behavior, and add focused regression coverage so deleted-account login either shows a clear irreversible-deletion error or follows a documented recovery path.
