## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open, and current main does not implement the reporter's expected post-deletion re-registration/session behavior. Source review shows user-initiated deletion is treated as irreversible deactivation, OAuth completion rejects deleted/deactivated users, active-user helpers hide them from session/profile queries, and related OAuth/session reports remain open.

## Best Possible Solution

A maintainer should decide the intended product behavior for GitHub identities after user-initiated deletion: keep deletion irreversible but surface a clear OAuth error/support path, add an explicit admin recovery/purge workflow, or allow re-registration after a well-defined tombstone migration. The implementation would likely touch `convex/auth.ts`, `convex/users.ts`, `convex/lib/access.ts`, frontend auth error handling, and regression tests, and should be coordinated with related open reports #886, #906, and #647.
