## Summary

Keep open. Current main treats user-initiated account deletion as an irreversible inactive state, so GitHub OAuth reauth is intentionally blocked and active-session helpers return no user. Because this is an issue, and because changing deletion/re-registration semantics needs maintainer account-lifecycle judgment, it is not eligible for conservative cleanup closure.

## What This Changes

The issue requests that a GitHub OAuth login after deleting a ClawHub account create a usable session and load the user profile instead of leaving the UI logged out.

## Best Possible Solution

Maintainers should explicitly choose the intended post-deletion lifecycle, then implement and document that path: permanent blocking with clear support messaging, a verified administrative purge workflow, or a deliberate tombstone/re-registration flow with tests preserving deletion and security invariants.
