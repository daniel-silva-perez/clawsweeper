## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy says ClawHub issues are not eligible for automated closure; only PRs that are certainly implemented on current main are. Source review also does not prove the reported account-deletion OAuth/session path is fixed: current main intentionally blocks deleted/deactivated users from reauthenticating and can return no active user for the UI after a cookie exists.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to verify the production OAuth flow for a deleted account, decide the intended recovery policy, and then either surface the deleted-account rejection reliably in the UI or implement a supported purge/re-registration path with end-to-end auth coverage.
