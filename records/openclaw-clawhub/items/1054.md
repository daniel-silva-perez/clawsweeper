## Summary

Keep this issue open. ClawHub cleanup policy only permits auto-closing pull requests that are certainly implemented on current main, and current main still does not create an active user session after user-initiated account deletion. The source treats deletion as irreversible deactivation, blocks OAuth reauth for deleted/deactivated users, and returns null from profile/session helpers for inactive users. Related OAuth/deleted-account reports remain open, so this needs maintainer account-lifecycle judgment rather than conservative cleanup.

## Best Possible Solution

Keep this issue open for maintainer account-lifecycle triage. Maintainers should decide whether deleted GitHub identities should remain permanently blocked with clearer OAuth/support UX, be administratively purged for re-registration, or be allowed through a deliberate tombstone/re-registration migration. Any implementation should coordinate with related reports #886, #906, #647, #1244, and #1552 and likely touch `convex/auth.ts`, `convex/users.ts`, `convex/lib/access.ts`, frontend auth error handling, docs, and regression tests.
