## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main treats this exact post-deletion OAuth path as an intentional irreversible-deletion block rather than a repaired session-creation bug. The related login-loop reports are still open, so this belongs in maintainer account-lifecycle triage.

## What This Changes

The issue requests that GitHub OAuth login after deleting a ClawHub account create an authenticated session and load the user's profile instead of leaving the UI logged out.

## Best Possible Solution

Maintainers should decide the formal deleted-account lifecycle: keep permanent blocking with clearer user/support UX, provide an explicit admin purge/re-enable path, or implement a deliberate tombstone/re-registration flow with matching Convex auth, user, publisher, docs, and regression-test updates.
