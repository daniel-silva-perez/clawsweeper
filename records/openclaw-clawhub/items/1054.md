## Summary

Keep this issue open. The repository cleanup policy for openclaw/clawhub does not auto-close issues, and current main shows this is not a simple missing session write: account deletion intentionally moves users into an inactive, irreversible state that blocks OAuth reauth and hides the user from active-session helpers. The remaining question is a maintainer account-lifecycle decision, especially because several related OAuth-loop reports are still open.

## What This Changes

The issue reports that after deleting a ClawHub account, GitHub OAuth completes and sets a cookie, but no active app user session or profile is loaded afterward.

## Best Possible Solution

Maintainers should choose and document the intended post-deletion lifecycle: keep permanent blocking with clear support UX, provide an administrative purge path for verified users, or implement a deliberate tombstone/re-registration flow with tests and docs that preserve deletion and security invariants.
