## Summary

Keep this issue open. The repository cleanup policy for openclaw/clawhub only permits automated closure for PRs that are certainly implemented on current main, and this is an issue. Current main also intentionally treats user-initiated deletion as irreversible: GitHub OAuth can complete, but deleted/deactivated users are blocked during the Convex Auth callback and hidden from active-session helpers, so the remaining work is a maintainer account-lifecycle decision rather than conservative cleanup.

## What This Changes

The issue requests that GitHub OAuth login after account deletion create a usable authenticated ClawHub session/profile instead of returning the UI to the logged-out state.

## Best Possible Solution

Keep this open for maintainer account-lifecycle triage. Maintainers should decide whether deleted GitHub identities remain permanently blocked with clearer support UX, can be administratively purged for re-registration, or should be allowed through a deliberate tombstone/re-registration migration coordinated with #647, #886, #906, #1244, and #1552.
