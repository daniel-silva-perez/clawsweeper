## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy says ClawHub issues should stay open; only PRs that are certainly implemented on main are eligible for automated closure. Current main also does not prove the reported production symptom is resolved: the code intentionally blocks deleted/deactivated-account reauthentication, returns no active user document for those accounts, and the UI can remain in the logged-out state when `me` is null.

## Best Possible Solution

Keep this issue open for maintainer triage. Maintainers should verify the production GitHub OAuth flow for a deleted account, decide whether deleted users should stay permanently blocked or be allowed to re-register after purge, and then either surface the deleted-account rejection reliably in the UI or implement the intended recovery/re-registration path with end-to-end auth coverage.
