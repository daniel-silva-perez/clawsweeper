## Summary

This issue should stay open. The ClawHub cleanup policy says open issues are not eligible for automated closure, and source review also does not prove the deleted-account OAuth/session path is fixed. Current main intentionally keeps user-initiated deleted accounts inactive and blocks reauthentication, while the UI can still end up without a resolved user document after auth state exists.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to verify the production OAuth callback/session behavior for a deleted account, confirm the intended recovery policy, then either surface the permanent-deletion rejection reliably in the UI or implement a supported purge/re-registration path with focused auth and UI coverage.
