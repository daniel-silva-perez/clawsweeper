## Summary

Keep this issue open. The ClawHub cleanup policy only allows automated closure for PRs that are certainly implemented on current main, and this item is an open issue. Source review also does not prove the deleted-account OAuth/session problem is resolved: current main records user-initiated deletion as deactivated/purged, blocks reauthentication for that account, returns null from users.me for deactivated users, and renders the signed-out header when no active user document is available. The closely related deleted-account login-loop report #886 is still open, so maintainers should decide the intended account recovery or error path.

## Best Possible Solution

Keep this issue open for maintainer triage. Verify production OAuth behavior for a deleted account, coordinate with #886, decide the intended permanent-deletion versus purge/re-registration path, and add focused auth/UI regression coverage for that chosen behavior.
