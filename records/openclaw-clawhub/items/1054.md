## Summary

Keep this issue open. ClawHub cleanup policy only permits automated closure for PRs that are certainly implemented on current main, and this is an issue. Current main also still treats user-initiated account deletion as irreversible: OAuth reauth for deleted/deactivated users is blocked, `users.me` returns null for inactive users, and the header remains in the signed-out state when no active user profile resolves. Related OAuth/deleted-account reports remain open, so this needs maintainer account-lifecycle judgment rather than cleanup closure.

## Best Possible Solution

Keep this issue open for maintainer account-lifecycle triage. Maintainers should decide whether deleted GitHub identities remain permanently blocked with clearer OAuth/support UX, can be administratively purged for re-registration, or should be allowed through a deliberate tombstone/re-registration migration. Coordinate with related open reports #647, #886, #906, #1244, and #1552 before implementation.
