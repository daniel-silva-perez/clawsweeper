## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review keeps issues open. The current main branch has proxy-related CLI mitigations and troubleshooting docs, but the reported `fetch failed` path still can surface as a bare low-level transport error and multiple users confirmed the symptom.

## Best Possible Solution

Keep this issue open for maintainer follow-up. Ask affected users to retest on the latest `clawhub` CLI and provide CLI version, install method, `node -v`, runtime, macOS version, proxy/VPN/corporate TLS details, `HTTPS_PROXY`/`HTTP_PROXY`/`NO_PROXY` values, and whether `curl -i https://clawhub.ai/api/v1/search?q=test` works from the same machine. If reproducible, improve `packages/clawhub/src/http.ts` and `packages/clawhub/src/cli/ui.ts` to unwrap fetch causes and print actionable proxy/TLS/DNS diagnostics.
