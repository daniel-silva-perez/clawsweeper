## Summary

Keep this issue open. The repository-specific policy says ClawHub issues should not be auto-closed by this workflow, and the current CLI still has an actionable core error-reporting path: `search` and token login send failures through `formatError`, which currently returns only `error.message`, so affected users can still see a bare `fetch failed`. Proxy support and docs exist, but the discussion includes multiple affected users and no confirmed retest showing the current CLI resolves all cases.

## Best Possible Solution

Keep this issue open for maintainer follow-up. Ask affected users to retest with the latest `clawhub` CLI and provide CLI version, install method, `node -v`, runtime, macOS version, proxy/VPN/corporate TLS details, `HTTPS_PROXY`/`HTTP_PROXY`/`NO_PROXY` values, and whether `curl -i https://clawhub.ai/api/v1/search?q=test` works from the same machine. If reproducible, improve `packages/clawhub/src/http.ts` and `packages/clawhub/src/cli/ui.ts` to unwrap fetch causes and print actionable diagnostics, with regression tests around the CLI-visible error text.
