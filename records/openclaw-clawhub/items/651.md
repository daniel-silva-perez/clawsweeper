## Summary

Keep open. This is a ClawHub issue, and the repository-specific review policy says this workflow should only auto-close pull requests that are certainly implemented on main. The current CLI has partial mitigations for proxy and Node transport failures, but the reported user-facing `fetch failed` path can still surface as a bare low-level error and multiple users confirmed the symptom.

## Best Possible Solution

Keep this issue open for maintainer follow-up. Ask affected users to retest on the latest `clawhub` CLI and provide CLI version, install method, `node -v`, runtime, macOS version, proxy/VPN/corporate TLS details, `HTTPS_PROXY`/`HTTP_PROXY`/`NO_PROXY` values, and whether `curl -i https://clawhub.ai/api/v1/search?q=test` works from the same machine. If reproducible, improve `packages/clawhub/src/http.ts` and `packages/clawhub/src/cli/ui.ts` to unwrap fetch causes and print actionable proxy/TLS/DNS diagnostics with regression tests.
