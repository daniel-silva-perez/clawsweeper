## Clownfish Work Prompt

Implement a focused ClawHub CLI transport-error diagnostics improvement for the current `fetch failed` search/login issue. Current main already has proxy support, but `cmdSearch` and token login call `apiRequest` and then `spinner.fail(formatError(error))`, while `formatError` only returns `error.message`. Update the shared CLI/HTTP error formatting path, likely `packages/clawhub/src/cli/ui.ts` and/or `packages/clawhub/src/http.ts`, so DNS, TLS, proxy, timeout, and Bun/curl transport failures include actionable hints such as checking proxy env vars and trying `curl -i https://clawhub.ai/api/v1/search?q=test`. Preserve existing HTTP status/rate-limit messages and retry behavior. Redact Authorization headers, API tokens, query tokens, and credentials embedded in proxy URLs or env vars. Add focused Vitest coverage for Node fetch errors with `cause`/Undici-style codes, timeouts, Bun/curl failures, and redaction. Do not change registry URL composition, auth config persistence, retry counts, or released changelog entries; only add an unreleased CLI note if the repo has an active unreleased section.
