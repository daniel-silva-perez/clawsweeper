## Repair Work Prompt

Fix the ClawHub CLI transport-error UX reported in this issue. Current main has proxy env support from #363, but `clawhub search` and token login can still print raw `fetch failed` or curl stderr because commands call `formatError(error)` and the HTTP layer rethrows transport errors unchanged. Add focused, centralized diagnostics in the shared CLI HTTP/error formatting boundary so DNS, TLS/certificate, proxy, timeout/abort, and curl failures produce actionable, redacted messages. Preserve existing successful request behavior, HTTP status errors, retry/rate-limit messages, timeout semantics, and Bun curl fallback behavior. Add tests for representative Node fetch causes and curl failures, plus command-level coverage where useful for search/login. Redact Authorization headers, API tokens, token query parameters, and credentials embedded in proxy URLs or env-derived messages. Add a concise CHANGELOG.md CLI bullet for improved transport diagnostics. Do not broaden this into unrelated networking rewrites, registry behavior changes, or credential handling changes.
