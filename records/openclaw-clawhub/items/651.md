## Summary

Keep this issue open. It reports a real CLI `fetch failed` path with multiple user confirmations, current main includes related proxy/Node mitigations but still can surface a bare low-level fetch message, and the repository policy for ClawHub says issues should not be auto-closed by this workflow.

## Best Possible Solution

Keep this issue open for maintainer follow-up. Ask affected users to retest with the latest CLI and provide CLI version, install method, `node -v`, macOS version, proxy/VPN/corporate TLS details, `HTTPS_PROXY`/`HTTP_PROXY`/`NO_PROXY` values, and whether `curl -i https://clawhub.ai/api/v1/search?q=test` succeeds from the same machine. If reproducible, improve `packages/clawhub/src/http.ts` and/or `packages/clawhub/src/cli/ui.ts` to unwrap fetch causes and print proxy/TLS/DNS diagnostics instead of only `fetch failed`.
