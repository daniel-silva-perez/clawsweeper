## Summary

Keep this issue open. It is an open ClawHub issue, and the repository-specific policy for this review says ClawHub issues should not be auto-closed. The current CLI does include proxy support and troubleshooting docs, but the `search` and token-login paths still surface transport failures through `formatError`, which only returns `error.message`, so affected users can still see a bare `fetch failed` without actionable diagnostics.

## Best Possible Solution

Keep this issue open for maintainer follow-up. Ask affected users to retest with the latest `clawhub` CLI and provide CLI version, install method, `node -v`, runtime, macOS version, proxy/VPN/corporate TLS details, `HTTPS_PROXY`/`HTTP_PROXY`/`NO_PROXY` values, and whether `curl -i https://clawhub.ai/api/v1/search?q=test` works from the same machine. If reproducible, improve `packages/clawhub/src/http.ts` and `packages/clawhub/src/cli/ui.ts` to unwrap fetch causes and print actionable diagnostics, with regression tests around CLI-visible error text.
