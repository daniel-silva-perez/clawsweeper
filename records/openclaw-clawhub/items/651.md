## Repair Work Prompt

Fix the ClawHub CLI's generic `fetch failed` UX for transport-level failures. Add centralized, redacted diagnostics in the shared HTTP/error formatting path so `clawhub search`, `clawhub login --token`, and other API-backed commands display actionable guidance for DNS, TLS/certificate, proxy, timeout/abort, and Bun curl failures while preserving existing HTTP status, rate-limit, retry, timeout, and successful request behavior. Do not treat every `fetch failed` as a proxy issue. Do not expose Authorization headers, API tokens, token query parameters, or credentials embedded in proxy URLs/environment variables. Keep the change scoped to CLI HTTP/error handling and focused tests; update troubleshooting docs only if the CLI-visible guidance changes. Related refs: this issue and merged #363 proxy support.
