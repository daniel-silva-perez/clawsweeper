## Summary

Keep open: current main still lacks the PR's intended soul-search parity changes, and the PR head still has two concrete correctness/performance issues before it should merge. The existing ClawSweeper comments are consistent with the current code and PR-head inspection.

## What This Changes

The PR branch changes `convex/search.ts` and `convex/search.test.ts` to add soul lexical fallback, merge/deduplication, shared boosted scoring, incremental soul embedding hydration, and related tests.

## Best Possible Solution

Land an updated soul-search parity change on current main that follows the current `searchSkills` flow: embedding failures fall through to lexical fallback, soul fallback uses `souls.by_active_updated` without `.filter()`, vector and fallback results are deduplicated, scoring is shared, hydration is incremental, and focused tests cover both success and embedding-failure paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Use the active soul index for fallback:** `convex/search.ts:533-535`
  - body: `lexicalFallbackSouls` scans `souls.by_updated` and then filters `softDeletedAt`, so the soft-delete predicate is not part of the index scan. Current main already defines `souls.by_active_updated`; query that index with `q.eq("softDeletedAt", undefined)` and update the test double accordingly.
  - confidence: 0.93
- **[P2] Run soul fallback after embedding failures:** `convex/search.ts:453-460`
  - body: The new lexical fallback only runs after vector search succeeds. If `generateEmbedding` throws, `searchSouls` still returns `[]` before reaching `lexicalFallbackSouls`, so exact slug/name searches fail during embedding outages while skill search falls back correctly.
  - confidence: 0.88
