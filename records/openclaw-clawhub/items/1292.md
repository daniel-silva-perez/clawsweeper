## Summary

Keep open: current main does not implement the PR's soul-search parity work, and the PR head still has a Convex indexed-query regression plus an incomplete fallback path. The change is useful and narrow, but it needs a rebase and targeted fixes before merge.

## What This Changes

The PR branch modifies `convex/search.ts` and `convex/search.test.ts` to add soul lexical fallback, merge/deduplication, shared boosted scoring, incremental soul embedding hydration, and related tests.

## Best Possible Solution

Land an updated soul-search parity change on current main that follows the current `searchSkills` pattern: embedding failures fall through to lexical fallback, soul fallback uses `souls.by_active_updated` without `.filter()`, vector and fallback matches are deduplicated, scoring is shared, hydration is incremental, and focused tests cover the success and failure paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Use the active soul index for fallback:** `convex/search.ts:533-535`
  - body: `lexicalFallbackSouls` scans `souls.by_updated` and then filters `softDeletedAt`, so Convex has to read rows that the existing `souls.by_active_updated` index can exclude. This fallback runs from search, so it can add avoidable read bandwidth and hit limits as the table grows. Query `by_active_updated` with `q.eq("softDeletedAt", undefined)` and update the test double to expect that index.
  - confidence: 0.93
- **[P2] Run soul fallback after embedding failures:** `convex/search.ts:453-460`
  - body: The new lexical fallback only runs after vector search succeeds. If `generateEmbedding` throws, `searchSouls` still returns `[]` before reaching `lexicalFallbackSouls`, so exact slug/name searches fail during embedding outages while skill search falls back correctly. Mirror the nullable-vector flow from `searchSkills` and add a regression test for this path.
  - confidence: 0.88
