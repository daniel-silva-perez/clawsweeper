## Summary

Keep open: current main does not implement the PR intent, and the PR head still has two merge-blocking soul-search issues around embedding-failure fallback and indexed fallback querying. The branch is useful but needs a focused repair before merge.

## What This Changes

The PR branch updates `convex/search.ts` and `convex/search.test.ts` to add soul lexical fallback, merge/deduplication, boosted scoring, incremental soul embedding hydration, and related tests.

## Best Possible Solution

Land an updated soul-search parity change that mirrors the current `searchSkills` flow: embedding failures fall through to lexical fallback, fallback uses `souls.by_active_updated`, vector and fallback matches are deduplicated, shared lexical/popularity scoring is applied, hydration remains incremental, and tests cover both success and embedding-failure paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use the active soul index for fallback:** `convex/search.ts:533-535`
  - body: `lexicalFallbackSouls` scans `souls.by_updated` and then filters `softDeletedAt`, so search fallback can read rows that the existing `souls.by_active_updated` index can exclude. Query `by_active_updated` with `q.eq("softDeletedAt", undefined)` and update the test double accordingly.
  - confidence: 0.93
- **[P2] Run soul fallback after embedding failures:** `convex/search.ts:399-403`
  - body: The new lexical fallback only runs after vector search succeeds. If `generateEmbedding` throws, `searchSouls` still returns `[]`, so exact slug/name searches fail during embedding outages while skill search falls back correctly. Mirror the nullable-vector flow from `searchSkills` and add a regression test.
  - confidence: 0.88
