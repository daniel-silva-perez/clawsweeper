## Summary

Keep open. ClawHub cleanup policy keeps issues open, this issue is paired with open implementation PRs, and current main still lacks the requested declared security-tool category/context path; the existing mitigation is a narrower external-scanner suppression and manual-review path.

## What This Changes

The issue requests a moderation category system so declared security tools can have expected scanner/audit patterns contextualized by both the static scanner and LLM evaluator without suppressing hard malicious signals.

## Best Possible Solution

Resolve this through maintainer review of PR #1132 and PR #1133, or a replacement design that parses declared security-tool metadata, contextualizes only allowlisted suspicious reason codes, preserves malicious and other hard safety floors, passes category and false-category guidance to the LLM evaluator, documents the behavior, tests the moderation boundaries, and provides a recovery path for already affected publishers.
