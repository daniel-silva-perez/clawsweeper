## Summary

Keep open. ClawHub cleanup policy keeps issues open, current main still lacks the requested declared security-tool category/context path, and the issue is paired with open implementation PRs that maintainers should review before this can be resolved.

## What This Changes

The issue requests a moderation category system so security-tool skills can declare their purpose and have coherent scanner patterns evaluated differently from malicious or incoherent skills.

## Best Possible Solution

Resolve this through maintainer review of PR #1132 and PR #1133, or a replacement design that parses declared security-tool metadata, contextualizes only allowlisted suspicious reason codes, preserves malicious and non-contextual safety floors, passes category and false-category guidance to the LLM evaluator, documents the behavior, and includes recovery for affected publishers.
