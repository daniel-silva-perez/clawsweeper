## Summary

Keep this issue open. Current main still lacks the requested declared-category moderation context for security tools, and the provided GitHub context shows an open implementation PR paired with this issue.

## Best Possible Solution

Keep this issue open and have maintainers review PR #1132 and PR #1133, or replace them with a narrower maintained design. The likely fix belongs in the static scanner reason-code pipeline, moderation snapshot/verdict logic, LLM evaluation context, skill-format/security docs, and an admin recovery path for existing moderation holds, with tests proving malicious signals are never suppressed by a security category.
