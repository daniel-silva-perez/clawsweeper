## Summary

Current `main` still does not implement the requested security-tool category/context mechanism, and this issue is paired with open implementation PRs #1132 and #1133. A later narrow mitigation only appears to demote selected static findings after clean external scanners, which does not solve the declared category, static-context, LLM-context, or recovery path requested here. ClawHub cleanup policy also keeps issues open rather than auto-closing them.

## Best Possible Solution

Keep this issue open and have maintainers review #1132 and #1133, or replace them with a narrower maintained design. The implementation should parse declared security-tool category metadata, contextualize only safe static reason codes, preserve hard malicious/suspicious safety floors, pass category context to the LLM with false-category guidance, add regression tests for legitimate security tools and category abuse, document the category, and decide whether an admin recovery path belongs in the same rollout.
