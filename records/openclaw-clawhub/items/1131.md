## Summary

Keep open. ClawHub cleanup policy keeps issues open, the issue is paired with open implementation PRs #1132 and #1133, and current main still lacks the requested declared security-tool category/context mechanism.

## What This Changes

Request a skill category system for security tools so moderation can contextualize expected scanner/audit patterns, preserve malicious safety floors, pass category context to the LLM evaluator, and provide a recovery path for affected publishers.

## Best Possible Solution

Keep this issue open and have maintainers review #1132 and #1133, or replace them with a maintained design. The fix should parse declared security-tool category metadata, contextualize only safe static reason codes, preserve malicious and non-contextual safety floors, pass category context to the LLM with false-category guidance, add regression tests/docs, and decide whether moderation-hold recovery ships in the same rollout.
