## Summary

Keep open. ClawHub cleanup policy keeps issues open, the report is paired with open implementation PRs, and current main still lacks the requested security-tool category/context mechanism; the existing mitigation is only a narrow external-scanner suppression path.

## What This Changes

Request category-aware moderation for security skills, including static-scan contextualization, LLM category context, malicious-code safety floors, and moderation-hold recovery for affected publishers.

## Best Possible Solution

Resolve this through maintainer review of PR #1132 and PR #1133, or a replacement design that parses declared security-tool metadata, contextualizes only allowlisted suspicious reason codes, preserves malicious safety floors, passes category context and false-category guidance to the LLM evaluator, documents the behavior, tests the moderation boundaries, and provides an admin recovery path for already affected publishers.
