## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main still matches the reported behavior: user-initiated deletion marks the user as deactivated/purged, GitHub OAuth completion rejects that identity, and the tests/docs explicitly preserve the irreversible-deletion path rather than allowing re-registration or restore.

## Best Possible Solution

A maintainer should keep this open long enough to respond to the reporter and decide the product policy. If same-GitHub re-registration should be supported, add an explicit admin/support path to unlink or reactivate deleted GitHub identities, update the deletion warning/docs, and add tests for re-registration plus banned/deactivated boundary cases. If irreversible deletion remains intentional, resolve manually as account support after helping the user rather than through automated cleanup.
