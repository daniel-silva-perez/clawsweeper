## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only allows auto-closing ClawHub pull requests that are certainly implemented on main. Current main also confirms the reported GitHub sign-in failure after user-initiated deletion is expected under the existing product policy, not an implemented re-registration or restore path.

## Best Possible Solution

A maintainer should keep this open long enough to answer the user and decide the product policy. If re-registration with the same GitHub account should be allowed, implement an explicit Convex Auth support path to unlink/reactivate a deleted GitHub identity, update the account-deletion warning and docs, and add tests for same-GitHub re-registration plus banned/deactivated boundary cases. If the irreversible-deletion policy is intentional, this should be resolved manually as a support/policy response rather than by automated cleanup.
