## Summary

Keep this issue open. This is a ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. The current code also still matches the reported failure mode: account deletion marks the user permanently deactivated, GitHub OAuth rejects deactivated/deleted users, and the admin unban path explicitly cannot restore permanently deleted accounts.

## Best Possible Solution

Keep this issue open for maintainer support and product/security policy handling. If same-GitHub recovery should be supported, add an audited admin/support path to reactivate or unlink deleted GitHub identities, update deletion warnings/docs, and add tests for re-registration plus banned/deactivated boundary cases. If irreversible deletion remains intentional, maintainers should reply with the supported manual account-resolution process.
