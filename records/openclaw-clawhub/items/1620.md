## Summary

Keep this issue open. ClawHub cleanup policy only permits auto-closing PRs that are certainly implemented on current main, and the current source still matches the reported failure: self-deletion records an irreversible deactivation, GitHub OAuth reauth rejects deactivated users, and the admin unban path does not recover permanently deleted accounts.

## What This Changes

A user cannot sign in or re-register with GitHub after accidentally deleting their ClawHub account.

## Best Possible Solution

Keep this issue open for maintainer support and product/security policy handling. If same-GitHub recovery should be supported, add an audited admin/support path to reactivate or unlink deleted GitHub identities for self-deleted accounts, preserve ban/moderation boundaries, update deletion warnings/docs, and add tests for re-registration plus banned/deactivated cases. If irreversible deletion remains intentional, maintainers should reply with the supported manual account-resolution process.
