## Summary

Keep open. Repository policy for openclaw/clawhub only permits auto-closing pull requests that current main certainly implements, and this is an issue. Current main also still matches the reported lockout: self-deletion permanently deactivates the user, OAuth completion rejects that user on reauth, and the admin unban path explicitly excludes permanently deleted accounts.

## What This Changes

The issue requests help restoring or re-registering a GitHub-linked ClawHub account after accidental ClawHub account deletion prevents sign-in.

## Best Possible Solution

Define a maintainer-owned account recovery outcome: either keep irreversible deletion as policy and document the supported support response clearly, or add an audited admin/support flow that can reactivate or unlink self-deleted GitHub identities while preserving ban and moderation boundaries.
