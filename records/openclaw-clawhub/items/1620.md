## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and current main still matches the reported lockout: self-deletion creates an irreversible deactivated user record, OAuth completion rejects that record on reauth, and the admin unban path is limited to legacy/banned deletedAt users rather than permanently deleted accounts.

## What This Changes

The issue requests help restoring access or re-registering with GitHub after the reporter accidentally deleted their ClawHub account.

## Best Possible Solution

Define a maintainer-owned account recovery outcome: either keep irreversible deletion as policy and document the supported support response clearly, or add an audited admin/support flow that can reactivate or unlink self-deleted GitHub identities while preserving bans, moderation holds, published content ownership, token revocation, and deletion warnings.
