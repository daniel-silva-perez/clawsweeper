## Summary

Keep this issue open. The ClawHub cleanup policy only permits auto-closing pull requests that are certainly implemented on current main, and this item is an issue. Current main also still matches the reported account recovery problem: self-service deletion permanently deactivates/purges the user, GitHub OAuth rejects deleted or deactivated users, admin unban explicitly refuses permanently deleted accounts, and docs/UI describe deletion as irreversible rather than offering self-service re-registration or restore.

## Best Possible Solution

Keep this issue open for maintainer support and product-policy handling. If same-GitHub recovery should be supported, add an explicit admin/support path to reactivate or unlink deleted GitHub identities, update deletion warnings/docs, and add tests for re-registration plus banned/deactivated boundary cases. If irreversible deletion remains intentional, maintainers should respond with the supported manual account-resolution process rather than closing through automated cleanup.
