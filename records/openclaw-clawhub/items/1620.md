## Summary

Keep this issue open. Repository policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on current main, and this item is an issue. Current main also still matches the reported failure mode: self-service deletion permanently deactivates the user, GitHub OAuth completion rejects deleted/deactivated users, tests assert that behavior, and docs/UI describe deletion as irreversible rather than offering self-service re-registration or restore.

## Best Possible Solution

Keep this issue open for maintainer support and product-policy handling. If same-GitHub recovery should be supported, add an explicit admin/support path to reactivate or unlink deleted GitHub identities, update deletion warnings/docs, and add tests for re-registration plus banned/deactivated boundary cases. If irreversible deletion remains intentional, maintainers should respond with the supported manual account-resolution process instead of closing through automated cleanup.
