## Summary

Keep this issue open. It is a ClawHub issue, and the repository-specific cleanup policy says ClawHub issues should not be auto-closed. Current main also intentionally blocks GitHub sign-in for deactivated/deleted accounts rather than implementing the reporter's requested re-registration or recovery path, so this needs maintainer support or product-policy judgment.

## Best Possible Solution

A maintainer should respond to the user support case with the current deletion policy and decide whether to keep deletion irreversible, improve the delete-account warning and sign-in error copy, or design an explicit re-registration/reactivation/unlink path. If re-registration with the same GitHub account should be allowed, the likely implementation area is Convex Auth account handling around deleted users, not the normal sign-in button.
