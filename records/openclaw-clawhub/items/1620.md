## Summary

Keep open. This is a ClawHub issue, and the repository-specific policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. The current code also confirms the reported problem is still a live support/product-policy issue: user deletion writes irreversible deactivation state and GitHub OAuth completion rejects the same identity rather than restoring, unlinking, or allowing re-registration.

## Best Possible Solution

A maintainer should keep this open long enough to respond to the user and choose the product policy. If same-GitHub re-registration should be allowed, implement an explicit Convex Auth support/admin path to unlink or reactivate a deleted GitHub identity, update the account-deletion warning/docs, and add tests for re-registration plus banned/deactivated boundary cases. If irreversible deletion is intentional, resolve this manually as a support/policy response after helping the reporter, not through automated cleanup.
