Current main has version-level soft-delete primitives and public endpoints that can block a soft-deleted version, but I did not find a first-class maintainer or owner workflow to hide only `tencent-doc-update-watch@0.1.0`, purge external caches, or enumerate exact historical per-version downloads.

Keep this issue open for maintainer/security handling.
