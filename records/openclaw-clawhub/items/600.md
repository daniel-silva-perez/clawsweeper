## Summary

Keep this issue open. It is a sensitive production incident request for one historical skill version, and the repository policy for ClawHub issues says only pull requests should be proposed for auto-close when current main definitely implements them. Current main has version-level soft-delete fields and public download/detail handlers that honor them, but there is still no exposed owner CLI/API path to hide only one skill version, and the requested private handling, cache confirmation, and download/install-record answer still require maintainer/operator follow-up.

## Best Possible Solution

Keep this issue open for private maintainer incident handling. Maintainers should contact the reporter privately, verify the leaked artifact, mark only `tencent-doc-update-watch@0.1.0` unavailable in production using the existing version-level `softDeletedAt` capability or by adding a staff/owner single-version hide path first, ensure tags/latest do not resolve to that version, confirm or purge any external caches, and explain what can and cannot be enumerated from production download/install data. Keep #905 as the broader self-service UX follow-up, but do not use it as a substitute for this incident response.
