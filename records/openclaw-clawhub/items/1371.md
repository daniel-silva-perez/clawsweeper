## Summary

Keep open. This is a specific hosted-production listing deletion request, and current main cannot prove the `unraidclaw` production record has been deleted or corrected. The source tree has generic owner/staff soft-delete support and a separate code-plugin package flow, but the live ClawHub APIs still serve `unraidclaw` as a skill-family listing, so this needs maintainer/admin follow-up rather than a source cleanup PR.

## What This Changes

The issue asks maintainers to delete the hosted `emaspa/unraidclaw` ClawHub skill listing because UnraidClaw is an npm/OpenClaw plugin and the skill listing leads users to install it through the wrong installer.

## Best Possible Solution

Keep this open for maintainer/admin follow-up. Verify requester ownership, then have the owner delete/hide the skill listing through the existing `DELETE /api/v1/skills/unraidclaw` or CLI path, or have staff perform the same production action. If UnraidClaw should stay discoverable on ClawHub, publish it through the package flow as `family: "code-plugin"` after the mistaken skill-family record is removed or otherwise reconciled. Close manually only after the public page/API no longer serve the mistaken skill listing, or after a corrected code-plugin listing replaces it.
