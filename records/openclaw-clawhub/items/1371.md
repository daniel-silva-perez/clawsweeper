## Summary

Keep open. The repository-specific policy for this review keeps ClawHub issues open, and this request is a live registry/admin cleanup for a mistaken skill listing rather than a source-tree change. Current main has the owner/moderator soft-delete flow, rejects plugin folders in the skill publish command, and has a separate code-plugin package catalog/install path, but the checkout contains no `unraidclaw` listing and cannot prove the hosted listing has been removed. The reporter also provided useful plugin context, including the npm/GitHub package and the upstream install failure thread.

## Best Possible Solution

Keep this issue open for maintainer/admin handling: verify ownership of the hosted `unraidclaw` listing, then have the owner run the supported soft-delete flow (`clawhub delete unraidclaw --yes` or the DELETE skill API) or have a moderator/admin hide it. If the author wants catalog visibility afterward, publish the project through the package catalog with `clawhub package publish <source> --family code-plugin`, then close this manually once production removal is verified.
