## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy says issues should remain open while only PRs that are certainly implemented on current main are auto-close candidates. The current source also leaves both reported concerns actionable: the skill-page header still uses compact unlabeled glyphs, and search has exact full-slug handling plus bounded lexical fallback but no reporter-specific regression coverage for `yescan` recalling `yescan-scan-universal`.

## Best Possible Solution

Keep this issue open for maintainer triage. The practical path is to add explicit labels, tooltips, or accessible text for the skill header version and download/package indicators, then investigate the specific `yescan` recall failure against production data: skill visibility, owner state, moderation flags, search digest row, embedding map, deployment version, and fallback candidate coverage. Add a focused regression test for a short slug-token query such as `yescan` matching a multi-token slug like `yescan-scan-universal`.
