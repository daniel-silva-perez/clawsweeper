Current main partially addresses the search side with exact-slug and bounded lexical fallback paths, but the reported header glyphs still lack clear top-level labels/tooltips and the specific short query `yescan` is not proven resolved against production data.

## Best Possible Solution

A focused regression test should prove that a short slug-token query such as `yescan` recalls a multi-token slug like `yescan-scan-universal`.
