## Summary

Keep open. This is a ClawHub issue, and the repository-specific policy for this review says issues should remain open; only PRs that are certainly implemented on current main are auto-close candidates. Substantively, current main still leaves the reported UX/search questions actionable: the skill header uses compact glyphs without direct labels/tooltips, and search has exact-slug plus bounded lexical fallback paths but no reporter-specific regression coverage for `yescan` recalling `yescan-scan-universal`.

## Best Possible Solution

Keep this issue open for maintainer triage. The practical product path is to add explicit labels/tooltips or accessible text for the skill-header version and download/package glyphs, then investigate the specific `yescan` recall failure against production search data: skill visibility, owner state, moderation flags, search digest row, embedding map, deployment version, and whether bounded lexical fallback is excluding older matching slugs. Add a focused regression test for a short slug-token query such as `yescan` matching a multi-token slug like `yescan-scan-universal`.
