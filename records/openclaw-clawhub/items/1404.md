## Summary

Keep open. The repository cleanup policy for openclaw/clawhub keeps issues open, and current main still leaves both reported concerns actionable: the skill header uses compact, unlabeled version/download indicators, and short slug-token search recall for a live skill still needs production data/index investigation. Related open work exists (#1854 for the package/download icon and #1256 for direct prefix recall), but neither is merged into current main and this issue also includes the specific yescan production case.

## Best Possible Solution

Keep this issue open for maintainer triage. The UI side should be handled by landing or superseding the open package/download icon work and adding labels, tooltips, or accessible text for the header version/download indicators. The search side should investigate the specific live yescan skill across visibility, moderation, digest, embedding, deployment, and fallback coverage, then land a direct indexed lexical prefix path or equivalent fix with a regression test for a short query like `yescan` matching `yescan-scan-universal`.
