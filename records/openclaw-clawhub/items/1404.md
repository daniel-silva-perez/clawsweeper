## Summary

Keep open. This is a ClawHub issue, and the repository policy for this cleanup pass only permits auto-closing pull requests that are certainly implemented on current main. Substantively, current main only partially addresses the icon confusion and does not prove the specific `yescan` search recall case is fixed: skill/search list stats still use an unlabeled package icon for downloads, the detail header still uses compact `v...` and package/version indicators, and short slug-token recall still depends on vector recall or bounded digest fallback rather than a direct indexed prefix lookup.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The UI side should either land/supersede the stat icon consistency work and add labels, tooltips, or accessible text for compact version/download indicators across detail, card, and list surfaces. The search side should investigate the live `yescan-scan-universal` record across visibility, moderation, digest, embedding, and deployment state, then land a direct indexed lexical prefix path or equivalent with a regression test for `yescan` matching that slug.
