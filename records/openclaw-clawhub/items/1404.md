## Summary

Keep open. This is a ClawHub issue, and the repository-specific policy for this review says to keep issues open. Current main explains the icon meanings in source and has exact-slug plus lexical fallback search paths, but the UI still lacks clear labels/tooltips for the reported glyphs and the reporter's specific `yescan` recall problem is not clearly resolved.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely product path is to add explicit labels/tooltips or accessible text for the top skill-header version and download/package glyphs, then investigate `yescan` production recall by checking the skill's search digest, embedding map, moderation/suspicious flags, owner state, and deployed search version. Add a regression test for a short slug-token query such as `yescan` matching a multi-token slug like `yescan-scan-universal`.
