## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy for this pass keeps issues open for maintainer handling. Current main appears to address the likely original shared-IP/authenticated install failure mode and the poor 429 UX: install sends the saved auth token, the download endpoint applies an auth-aware limiter, valid token requests use a user bucket rather than a shared IP bucket, and 429 responses include retry/budget guidance. The reporter has not confirmed behavior on the current CLI, while broader rate-limit reports remain open, so maintainers should ask for a current reproduction before manually closing or consolidating it.

## What This Changes

The issue requests that `clawhub install` stop failing immediately with an opaque `Rate limit exceeded` error for authenticated public skill installs, or at least return structured retry/quota guidance.

## Best Possible Solution

Keep this issue open for maintainer triage. Ask the reporter to retry with the current `clawhub` CLI, include `clawhub --version`, confirm `clawhub whoami` succeeds in the same shell/session, and paste the current 429 output plus relevant headers if it still fails. If it no longer reproduces, maintainers can manually close it as fixed by #412/#421 or consolidate it under the broader open rate-limit reports; if it still reproduces, treat it as an install-specific saved-token, token-forwarding, or production limiter configuration bug.
