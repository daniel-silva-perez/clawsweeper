## Summary

Keep this issue open. Current main still has a credible publish-update path where an ordinary non-trusted publisher updates an already-active skill, the skill row is rewritten to a hidden moderation state, and public search/list/detail paths filter it out; ClawHub cleanup policy also keeps issues open unless a maintainer resolves them directly.

## What This Changes

The issue reports that an already published skill disappeared from search/listings and produced a web UI publish server error after an update, with slug ownership and API continuity as the main concern.

## Best Possible Solution

Preserve slug ownership and public continuity for active skills by serving the last approved public version while a new upload is pending scan or review, while still hiding static-malicious, quality-quarantined, user-moderated, removed, manually locked, and malware-blocked content with clear owner-visible diagnostics and regression coverage across publish, search/list/detail, API reads, and scanner promotion.
