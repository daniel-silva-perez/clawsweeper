## Summary

Keep this issue open. The repository policy for this cleanup pass keeps ClawHub issues open, and current main still has the reported failure path: an ordinary publisher updating an existing active skill can have `moderationStatus` recomputed to `hidden`, that hidden status is mirrored into `skillSearchDigest`, and public search/list/detail surfaces filter hidden skills out. The discussion shows multiple affected publishers, related #134 remains open, and closed-unmerged #274 is the closest focused fix candidate.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The best fix is to preserve public visibility and slug ownership when an already-active skill publishes a clean update, while still hiding static-malicious uploads, quality-quarantined releases, removed/manual moderation locks, and publishers under moderation. Coordinate with open #134 and the closed-unmerged #274 approach, then add regression coverage for active-skill updates, owner-visible moderation state, and public search/list/detail visibility.
