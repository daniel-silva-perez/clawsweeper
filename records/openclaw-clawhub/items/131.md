## Summary

Keep this open. Current main still has a credible publish-update path where an ordinary non-trusted skill update writes `hidden` onto the skill row, and the public search/list/detail hydration path filters hidden skills out; for this repository, issues are not auto-closed by cleanup even when a fix direction is clear.

## What This Changes

The issue asks ClawHub to restore search/listing visibility and preserve slug ownership after a previously active skill disappeared following a web UI update that returned a server error.

## Best Possible Solution

Preserve slug ownership and public continuity for already-active skills during clean updates by keeping the last approved public version visible while the new upload is pending scan/review, while still hiding static-malicious, quality-quarantined, user-moderated, removed, manually locked, and malware-blocked skills; add owner-visible moderation diagnostics and regression coverage across publish, search/list/detail, API reads, and scanner promotion.
