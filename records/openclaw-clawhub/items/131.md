## Summary

Keep this issue open. The ClawHub cleanup policy for this review keeps issues open, and current main still has source-level evidence for the reported publish/update visibility failure: an existing active skill from an untrusted publisher can be patched to `hidden` during publish, while public search/list paths drop non-active skills.

## Best Possible Solution

Keep this issue open. Fix the skill publish update path so publishing a clean new version of an already-active skill preserves public visibility while scanner review runs, except for malicious static findings, quality quarantine, removed/manual moderation locks, or publishers under moderation. Add regression coverage for active skill updates, public search/list/detail visibility, owner-visible moderation state, and slug ownership preservation; coordinate with #134 and the closed-unmerged #274 context.
