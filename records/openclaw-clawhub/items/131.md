## Summary

Keep open. Current main still contains a credible publish-update path where an ordinary non-trusted publisher can update an already-active skill and have the skill row rewritten to a hidden moderation state, after which public search/list/detail paths filter it out; this is also a ClawHub issue, so cleanup policy should not auto-close it.

## What This Changes

The issue reports that an existing published skill disappeared from search/listings after a web UI update returned a server error, with the reporter asking for the slug and ownership to remain preserved.

## Best Possible Solution

Preserve slug ownership and public continuity for active skills by serving the last approved public version while a new upload is pending scan or review, while still hiding static-malicious, quality-quarantined, user-moderated, removed, manually locked, and malware-blocked content with clear owner-visible diagnostics.
