## Summary

Keep this issue open. ClawHub cleanup policy keeps issues open in this pass, and current main still supports the reported failure mode: an existing active skill update by an ordinary publisher can write `moderationStatus: "hidden"`, that hidden state is synchronized into the search digest, and public search/list/detail helpers filter hidden skills out. The discussion also shows multiple affected publishers and a closed-unmerged focused fix candidate in #274, while related #134 remains open.

## Best Possible Solution

Keep this issue open for maintainer follow-up. Fix the existing-skill publish path so clean updates to already-active skills preserve public search/list/detail visibility and slug ownership while review runs, except for static-malicious uploads, quality quarantine, removed/manual moderation locks, or publishers under moderation. Add regression coverage for active skill updates, owner-visible moderation state, public search/list/detail visibility, and coordinate with #134 and the closed-unmerged #274 approach.
