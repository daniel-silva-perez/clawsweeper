## Summary

Keep this issue open. The repository policy for openclaw/clawhub only permits auto-closing PRs that current main definitely implements, and this item is an issue. Substantively, current main still matches the reported failure mode: an ordinary publisher updating an existing active skill can have `moderationStatus` recomputed to `hidden`, that hidden state is mirrored into `skillSearchDigest`, and public catalog/search paths filter hidden skills out. Related #134 remains open, and the targeted #274 fix was closed without merging.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The best fix is to define the moderation policy for updates explicitly, then preserve public visibility and slug ownership when an already-active skill publishes a clean update, while still hiding static-malicious uploads, quality-quarantined releases, manual moderation locks, removed skills, and publishers under moderation. Coordinate with open #134 and the closed-unmerged #274 approach, then add regression coverage for active-skill updates, owner-visible moderation state, and public search/list/detail visibility.
