## Summary

Keep this issue open. The repository policy for openclaw/clawhub only permits automated close recommendations for pull requests that current main certainly implements, and this item is an issue. A source pass on current main also still supports the reported failure path: an existing active skill update by an ordinary non-trusted publisher can be patched to hidden during pending review, and public search/list/detail paths filter hidden skills out.

## Best Possible Solution

Keep this issue open. Fix the publish update path so a clean new version of an already-active skill preserves public search/list/detail visibility while review runs, except for static-malicious uploads, quality quarantine, removed/manual moderation locks, or publishers under moderation. Add regression coverage for active skill updates, search/list/detail visibility, owner-visible moderation state, and slug ownership preservation; coordinate with #134 and decide whether to revive or replace the closed-unmerged #274 approach.
