## Summary

Keep this issue open. Current main still has a credible path where an ordinary non-trusted publisher updates an already-active skill and the skill row is rewritten to a hidden moderation state, after which public search/list/detail hydration filters it out; this also needs maintainer moderation/security judgment rather than conservative cleanup closure.

## What This Changes

The issue reports that an already-published skill disappeared from search/listings after a web UI update returned a server error, with slug ownership and API continuity concerns.

## Best Possible Solution

Preserve slug ownership and public continuity for active skills by serving the last approved public version while a new upload is pending scan or review, while still hiding static-malicious, quality-quarantined, user-moderated, removed, manually locked, and malware-blocked content with owner-visible diagnostics.
