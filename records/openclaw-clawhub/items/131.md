## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open, and current main still matches the reported disappearance path: ordinary non-trusted publishes compute a hidden skill-level moderation status, write it onto existing skills during update, and public search/list/detail hydration filters hidden skills out. Related #134 remains open for the same publish/disappear symptom, #274 proposed the direct preservation fix but was closed unmerged, and merged #130 is the VT quarantine context that made updates hidden pending review.

## What This Changes

The issue requests fixing skill updates that can return a server error and leave an already-published skill missing from public search/listings while preserving the original slug/account ownership.

## Best Possible Solution

Keep this open for maintainer moderation-policy follow-up. The strongest product direction is to preserve slug ownership and public continuity for already-active skills during clean ordinary updates, ideally by keeping the last approved public version visible while a new version is pending scan, while still hiding malicious, quality-quarantined, under-moderation, removed, and manually locked skills; add regression coverage for publish updates, public search/list/detail, owner-visible moderation state, and scanner promotion after clean review.
