## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open, and current main still has the reported failure mode: publishing an update for an existing active skill by an ordinary publisher can recompute the skill-level moderation state to hidden, that state is mirrored into the search digest, and public search/listing surfaces filter hidden skills out. The discussion shows multiple affected publishers and slug-preservation concerns, related #134 remains open, and closed-unmerged #274 is the closest focused fix attempt.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The best fix is to preserve public visibility and slug ownership when an already-active skill publishes a clean update, while still hiding static-malicious uploads, quality-quarantined releases, manual moderation locks, removed skills, and publishers under moderation. Coordinate with open #134 and the closed-unmerged #274 approach, then add regression coverage for active-skill updates, owner-visible moderation state, and public search/list/detail visibility.
