## Summary

Keep this open. Current main still has a credible publish-update path where an ordinary non-trusted skill update writes a hidden moderation state onto the existing skill row, and public search/list/detail hydration filters that row out; ClawHub cleanup policy also keeps issues open unless a maintainer resolves them directly.

## What This Changes

The issue asks for an existing published skill to remain findable and tied to its owner/slug after a publish update fails or moves it out of public search/listings.

## Best Possible Solution

Keep slug ownership intact and preserve public continuity by serving the last approved version while a new version is pending scan/review, while still hiding static-malicious, quality-quarantined, user-moderated, removed, manually locked, and malware-blocked skills; add clear owner-facing diagnostics and regression coverage across publish, search/list/detail, API reads, and scanner promotion.
