## Summary

Keep this issue open. ClawHub cleanup policy only permits auto-closing PRs, and current main still matches the reported user-visible failure: an ordinary publisher updating an existing active skill can have the skill-level moderation status recomputed to hidden, which then filters the skill out of public search/listing paths.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The best path is to define the moderation policy for updates explicitly, then preserve visibility and slug ownership for already-active skills that publish clean updates while still hiding static-malicious uploads, quality-quarantined releases, publishers under moderation, removed skills, and manual moderation locks. Coordinate with open #134 and the closed-unmerged #274 approach, then add regression coverage for publish updates, search/list/detail visibility, and owner-visible moderation state.
