## Summary

Keep this issue open. Current main still contains a credible publish-update path where an ordinary non-trusted publisher can update an already-active skill and have the skill row rewritten to a hidden moderation state; the public search/list/detail paths then filter that row out. Because this is a ClawHub issue and the remaining fix crosses moderation/security policy, it should stay with maintainers rather than be closed by cleanup.

## What This Changes

The issue reports that a previously published skill disappeared from search and listings after a web UI update/publish attempt, with a server error and concern about preserving the existing slug/account ownership.

## Best Possible Solution

Preserve slug ownership and public continuity for active skills by serving the last approved public version while a new upload is pending scan or review, while still hiding static-malicious, quality-quarantined, user-moderated, removed, manually locked, and malware-blocked content with owner-visible diagnostics.
