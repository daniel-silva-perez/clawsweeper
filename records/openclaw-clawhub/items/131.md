## Summary

Keep this issue open. ClawHub review policy keeps issues open, and current main still supports the reported failure mode: a non-trusted update to an existing active skill can write `moderationStatus: "hidden"`, and hidden skills are filtered out of public search and listings.

## Best Possible Solution

Keep this issue open. Fix the existing-skill publish path so clean updates to already-active skills preserve public search/list/detail visibility and slug ownership while review runs, except for static-malicious uploads, quality quarantine, removed/manual moderation locks, or publishers under moderation. Add regression coverage for active skill updates, owner-visible moderation state, public search/list/detail visibility, and coordinate with #134 and the closed-unmerged #274 approach.
