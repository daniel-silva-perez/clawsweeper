## Summary

Keep open. This is a ClawHub issue, and the repository-specific review policy says ClawHub issues should not be auto-closed. Current main also still shows upload/publish validation through a centralized Validation panel plus a generic submit reminder, not inline beneath each input or as a popup as requested.

## Best Possible Solution

Keep this issue open for a focused publish/upload UX change: derive field-specific validation state, render helper/error text beneath the affected slug, display name, version, tags, license, and file upload controls, wire aria-invalid and aria-describedby or alert behavior, and update upload route tests to assert the new placement and accessibility behavior.
