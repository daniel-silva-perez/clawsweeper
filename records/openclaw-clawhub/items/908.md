## Summary

Keep open. This is a ClawHub issue, and the review policy for this pass says to keep ClawHub issues open. A shell check of current main also shows the requested UX is still not fully implemented: validation issues are still centralized in the publish form’s Validation card plus a generic submit reminder, not shown inline under the relevant inputs or as a validation popup.

## Best Possible Solution

Keep this issue open for a focused upload-page UX improvement: map validation failures to field-level helper or error text beneath Slug, Display name, Version, Tags, file upload, and license controls; add accessible aria-describedby or role=alert behavior where appropriate; keep the aggregate Validation card only as a secondary checklist; and update upload route tests to assert the new visible placement.
