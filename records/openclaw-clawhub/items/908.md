## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review pass says to keep ClawHub issues open. The current upload flow redirects to the publish-skill page, where validation exists but remains centralized in a Validation card plus a generic submit reminder; it does not implement the requested inline-under-input or popup/toast validation treatment.

## Best Possible Solution

Keep this issue open for a focused upload-page UX improvement: map validation failures to field-level helper/error text beneath Slug, Display name, Version, Tags, file upload, and license controls; add accessible aria-describedby or role=alert behavior where appropriate; keep the aggregate Validation card only as a secondary checklist; and update upload route tests to assert the new visible placement.
