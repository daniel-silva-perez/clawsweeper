## Summary

Keep open. This is a ClawHub issue, and the repository-specific review policy only allows auto-close for pull requests that are certainly implemented on main. Current main also still uses a centralized publish/upload validation panel plus a generic submit reminder, without inline-under-input or popup validation treatment for the affected fields.

## Best Possible Solution

Keep this issue open for a focused publish/upload UX improvement: derive field-specific validation state, render concise error/help text near each invalid input or an accessible popup, wire aria-invalid and aria-describedby or alert behavior, and update upload route tests to assert the placement and accessibility behavior.
