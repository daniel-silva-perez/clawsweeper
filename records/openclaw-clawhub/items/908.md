The requested upload/publish validation UX also still appears unresolved: current main routes /upload to /publish-skill, and that page still renders aggregate validation feedback instead of field-local errors under inputs or a popup.

## Best Possible Solution

Keep this issue open for a focused publish/upload UX fix: derive field-specific validation state, render concise error/help text near each invalid control or in an accessible popup, wire aria-invalid and aria-describedby or alert semantics, and add route tests that assert the placement and accessibility behavior.
