## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review pass says ClawHub issues should remain open. The current main branch still renders upload validation as an aggregate Validation card plus a generic submit reminder, not inline under each affected input or as a validation popup.

## Best Possible Solution

Keep this issue open for a focused upload-page UX change: map each validation failure to field-level helper/error text beneath Slug, Display name, Version, Tags, license, and file upload controls; add accessible aria-describedby or role=alert behavior; keep the aggregate Validation card as a secondary checklist if useful; and update upload route tests to assert the new placement.
