## Summary

Keep this issue open. Current main still has the broad Crypto capability matcher that treats standalone `swap` as crypto, publish/backfill paths still scan uploaded text files, and draft PR #1857 is the open implementation candidate for this exact false-positive path.

## Best Possible Solution

Keep this issue open and review or land #1857, or an equivalent maintainer patch: tighten Crypto matching so generic CSS/web text like `display=swap` does not trigger Crypto, preserve positive detection for real DeFi/token swap language, add regression tests, and decide whether already-published capability tags need a backfill or manual correction. A small UI/API/docs improvement exposing matched capability evidence would also help future reporters diagnose labels without guessing.
