## Summary

Keep this issue open. Current main still has the broad Crypto capability matcher the reporter is asking about, uploaded text files still feed capability derivation, and the linked draft PR #1857 is an open implementation candidate rather than a merged fix. The ClawHub cleanup policy also keeps issues open; only pull requests definitely implemented on main are eligible for auto-close.

## Best Possible Solution

Keep this issue open and review/land #1857 or an equivalent maintainer patch: tighten Crypto matching so generic web/CSS text like `display=swap` does not trigger Crypto, preserve positive detection for real DeFi/token swap language, add regression tests, and decide whether already-published capability tags need a backfill or manual correction. A small docs/UI improvement that exposes matched capability evidence would also help future reporters diagnose labels without guessing.
