## Summary

Keep this issue open. Current main still treats standalone `swap` as a Crypto capability trigger, publish/backfill paths still persist capability tags from uploaded text files, and the provided GitHub context shows open draft PR #1857 directly targets this false-positive path with `Fixes #1524`. ClawHub review policy also keeps issues open unless the linked implementation has landed.

## Best Possible Solution

Keep this issue open and review or land #1857, or an equivalent maintainer patch: tighten Crypto matching so generic CSS/web text like `display=swap` does not trigger Crypto, preserve positive detection for real DeFi/token swap language, add regression tests, and decide whether existing persisted tags need a targeted backfill or manual correction. A small UI/API/docs improvement exposing matched capability evidence would also help future reporters diagnose labels without guessing.
