## Summary

Keep this issue open. It is an issue in openclaw/clawhub, where the cleanup policy only allows auto-closing pull requests that are certainly implemented on main, and the current code still supports the reported concern: the Crypto signal is generated by broad deterministic regexes over all text files, while public docs/UI do not expose trigger evidence.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The likely product fix is to tighten `convex/lib/skillCapabilityTags.ts` so generic terms like `swap` require crypto context or exclude known CSS/font-display patterns, add focused regression coverage, and document capability-signal criteria or expose matched evidence in the author/security UI.
