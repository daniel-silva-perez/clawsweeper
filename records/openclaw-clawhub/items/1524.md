## Summary

Keep this issue open. Current main still has a broad standalone `swap` Crypto trigger, publish/backfill still derive capability tags from uploaded text files, and the linked draft PR #1857 directly targets this false-positive path but is open and unmerged. ClawHub review policy also keeps issues open until the implementation lands.

## What This Changes

The issue asks why a published skill is receiving the Crypto capability signal despite no crypto-related functionality, and whether there is an actionable way to remove that label.

## Best Possible Solution

Keep this issue open and review or land #1857, or an equivalent maintainer patch: tighten Crypto matching so generic CSS/web text like `display=swap` does not trigger Crypto, preserve positive detection for real DeFi/token swap language, add regression tests, and decide whether existing persisted tags need targeted correction. A small UI/API improvement exposing matched capability evidence would also help answer future diagnostic questions.
