Kept open. This is an open ClawHub issue, and repository cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main still lacks the requested static moderation rule for destructive troubleshooting `rm -rf` guidance without an explicit confirmation gate, while PR #1599 remains open and unmerged as the implementation candidate.

The best implementation is a focused static markdown rule that flags troubleshooting/reinstall `rm -rf` commands against sensitive absolute or home-style paths, suppresses only when explicit user confirmation is required before deletion, adds a dedicated reason code and scan-version bump, and includes regression tests for the reported `stt-simple` pattern plus safe cleanup examples.
