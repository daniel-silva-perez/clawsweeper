## Summary

Keep open. This is an issue, and the ClawHub cleanup policy for this run only allows auto-closing PRs that are definitely implemented on main. Substantively, current main has an internal publish-time anti-spam/documentation quality gate plus security/moderation signals, but it does not expose the requested public, multi-dimensional skill quality scoring model for maintenance, dependency hygiene, alternatives, claim-vs-reality, or comparative quality.

## Best Possible Solution

Keep this open for maintainer product triage. A concrete path is to define a public quality-signal model separate from security/moderation verdicts, choose fair measurable dimensions, add schema/API/search fields and backfill strategy, then render those signals on browse/detail pages if maintainers want them. Treat #288 as adjacent security-signal work, not a replacement for this quality-scoring discussion.
