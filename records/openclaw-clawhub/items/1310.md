## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy only allows auto-closing ClawHub pull requests that are certainly implemented on current main. Current main has security/moderation scan surfaces and a narrow publish-time content quality gate, but it does not implement the public, multi-dimensional skill quality scoring model requested here.

## Best Possible Solution

Keep this open for maintainer product triage. A good path would define a public quality-signal model separate from security/moderation verdicts, choose measurable dimensions such as documentation completeness, maintenance freshness, dependency hygiene, prompt/code structure, and claim-vs-reality checks, then add schema/API/search fields with a backfill plan and render the chosen signals on browse/detail pages. Treat #288 as related security-signal work, not a replacement for this broader quality proposal.
