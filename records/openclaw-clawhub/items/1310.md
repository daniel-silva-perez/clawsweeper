## Summary

Keep open. This is an issue, and the ClawHub cleanup policy only allows auto-closing pull requests that are certainly implemented on current main. Current main has security scan surfaces plus a publish-time content quality gate, but it does not implement the requested public, multidimensional skill quality scoring model; the existing ClawSweeper comment is consistent with this review.

## What This Changes

The issue proposes public quality signals for ClawHub skills, separate from security verdicts, covering documentation, claim accuracy, maintenance, dependency hygiene, alternatives, and prompt/code structure.

## Best Possible Solution

Define a public quality-signal model distinct from security/moderation verdicts, with agreed dimensions, schema/API/search support, backfill strategy, and UI presentation on browse/detail surfaces; keep #288 as related security-signal context rather than the canonical tracker for this broader quality work.
