## Summary

Keep open. The cleanup policy for this repository keeps issues open, and current main still lacks the requested public, multidimensional skill quality-scoring model; existing code has a narrow publish-time quality gate plus security/moderation scanners, not public quality ranking, filtering, or comparative trust dimensions.

## What This Changes

The issue proposes public multidimensional quality signals for ClawHub skills based on an external top-50 skill analysis, covering documentation, claim-vs-reality, complexity, maintenance, dependency hygiene, and alternatives.

## Best Possible Solution

Define a public quality-signal model distinct from security and moderation verdicts, then add agreed schema/API/search support, backfill or recomputation strategy, and UI presentation on browse/detail surfaces while keeping #288 as related security-signal context rather than the canonical tracker for broader quality scoring.
