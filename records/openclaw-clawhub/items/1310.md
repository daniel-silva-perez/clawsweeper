## Summary

Keep open. Repository policy for openclaw/clawhub only permits auto-closing PRs that are certainly implemented on current main, and this is an issue. Independently, current main has security scan surfaces and a publish-time content quality gate, but it does not implement the public, multidimensional skill quality scoring model requested here.

## What This Changes

Request discussion and possible implementation of public, multidimensional quality signals for ClawHub skills, covering documentation, claim accuracy, complexity, maintenance, dependencies, alternatives, and prompt/code structure separately from security verdicts.

## Best Possible Solution

Keep this open for maintainer product triage. A good path would define a public quality-signal model separate from security/moderation verdicts, choose measurable dimensions, add indexed schema/API/search fields with a backfill plan, and render the selected signals on browse/detail pages. Treat #288 as related security-signal work, not a replacement for this broader quality proposal.
