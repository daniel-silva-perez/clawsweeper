## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy only allows auto-closing ClawHub pull requests that are certainly implemented on current main. Current main also does not implement the requested public, multi-dimensional skill quality scoring model: it has publish-time content quality checks and security/moderation scan surfaces, but not public maintenance, dependency hygiene, documentation completeness, alternatives, or claim-vs-reality quality signals for skills.

## Best Possible Solution

Keep this open for maintainer product triage. A good implementation would define a public quality-signal model separate from security/moderation verdicts, choose measurable dimensions such as documentation completeness, maintenance freshness, dependency hygiene, prompt/code structure, and claim-vs-reality checks, then add schema/API/search fields with a backfill plan and render those signals on browse/detail pages if maintainers want them. Treat #288 as related security-signal work, not as a replacement for this issue.
