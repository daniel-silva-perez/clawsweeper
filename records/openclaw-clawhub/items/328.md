## Summary

Keep open. ClawHub cleanup policy keeps issues open in this lane, and the live named skill still reproduces the mixed scanner state: VirusTotal is suspicious while ClawScan is benign. Current main explains how that state is produced, but does not yet settle the publisher-facing false-positive or scanner-disagreement policy.

## What This Changes

The issue asks why a published skill can show both benign and suspicious scanner labels, and how publishers can avoid or clear those tags for API/query-only skills.

## Best Possible Solution

Keep the conservative scanner behavior until maintainers define the false-positive policy, then add clear publisher guidance for scanner disagreements, expose the appropriate rescan or review path, and only change moderation logic for a narrowly defined safe signal class with tests.
