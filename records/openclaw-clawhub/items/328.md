## Summary

Keep open. Repository cleanup policy keeps ClawHub issues open, and the report still matches current behavior: the named skill has a suspicious VirusTotal result alongside a benign ClawScan result, while current main has not settled the false-positive or scanner-disagreement policy for publishers.

## What This Changes

The issue asks why a skill can show Benign from ClawScan while VirusTotal marks it Suspicious, and how publishers can avoid or resolve those tags.

## Best Possible Solution

Maintain the conservative scanner behavior until maintainers define a false-positive policy, then add clear publisher guidance for scanner disagreements, expose the appropriate rescan or review path, and only change moderation logic for a narrowly defined safe signal class with tests.
