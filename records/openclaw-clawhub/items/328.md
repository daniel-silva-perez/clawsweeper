Keep open. ClawHub cleanup policy keeps issues open in this lane, and the report still matches current behavior: the named skill’s live scan has a suspicious VirusTotal result while ClawScan is benign, and current main still leaves the policy choice to maintainers rather than providing a resolved publisher guidance or false-positive clearing path.

## What This Changes

The issue asks why a skill can show a benign ClawScan/OpenClaw result while VirusTotal reports suspicious, and how publishers can avoid or resolve that mixed scanner state.

## Best Possible Solution

Keep the conservative scanner behavior until maintainers define the policy, then add publisher-facing guidance explaining why VirusTotal and ClawScan can disagree, how to request rescans or false-positive review, and which skill patterns reduce flags; use staff clean overrides for confirmed false positives and only change moderation logic with narrow tests for a specific safe scanner-signal class.
