## Summary

Keep open. This is an open ClawHub issue, and the repository policy for this review only permits auto-closing PRs that are certainly implemented on current main. Substantively, the cited skill still reproduces the reported split: the public API reports the skill as suspicious from VirusTotal while the OpenClaw LLM scanner reports benign/clean. Current main intentionally folds VT suspicious results into the skill moderation snapshot, and tests preserve that behavior, so the remaining work is maintainer product/security judgment and publisher-facing remediation guidance rather than cleanup closure.

## Best Possible Solution

Leave this issue open for maintainer product/security judgment. A good resolution would add owner-facing documentation and UI/API copy explaining why VirusTotal and OpenClaw scanner verdicts can disagree, what metadata declarations help reduce false positives, and how publishers can request review. If maintainers determine this specific instruction-only advisor skill is acceptable, use the existing audited manual override for that skill and consider narrowly tuning VT/static/LLM merge behavior without weakening malicious handling.
