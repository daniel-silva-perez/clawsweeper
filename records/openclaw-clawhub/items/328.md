## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing PRs that are certainly implemented on current main. Substantively, current main still has a real product/security question here: it preserves VirusTotal suspicious signals except for a narrow uncorroborated VT Code Insight exception, while the UI/API expose scanner disagreement and appeal paths without fully answering how publishers can avoid or resolve benign/suspicious split tags.

## Best Possible Solution

Leave this issue open for maintainer product/security judgment. A good resolution would add owner-facing UI/API/docs guidance explaining why VirusTotal and OpenClaw verdicts can disagree, what metadata or declarations help reduce false positives, and how publishers can request review. If maintainers decide the cited skill is acceptable, use the existing audited manual override path or narrowly tune the moderation engine without weakening malicious handling.
