## Summary

Keep this issue open. The repository-specific policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on main, and this item is an issue. A source pass also shows the reported behavior is not resolved as a user recovery flow: deleted/deactivated GitHub-linked accounts are intentionally blocked from re-sign-in, and skill publishing still surfaces the exact GitHub API rate-limit error from the GitHub account-age gate. Related account-lock reports #504, #886, and #1674 are still open in the provided context, and the prior ClawSweeper review already reached the same keep-open conclusion.

## Best Possible Solution

Keep this issue open for maintainer triage. The practical path is to decide the product policy for deleted GitHub-linked accounts, then either document the irreversible/manual-support path more clearly or implement controlled recreation/relinking. Separately, improve the publish flow around GitHub 403/429 account-age lookups with clearer retry/recovery guidance or backend configuration checks for GITHUB_TOKEN.
