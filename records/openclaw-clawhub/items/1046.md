## Summary

Keep open. The repository cleanup policy keeps ClawHub issues open, and current main still matches both reported surfaces: deleted/deactivated GitHub-linked accounts are rejected on reauth, and publish still surfaces GitHub 403/429 from the account-age lookup as the reported rate-limit error. This needs maintainer product/security judgment, not conservative cleanup closure.

## What This Changes

The issue requests clearer recovery for a GitHub API rate-limit failure during skill publish and a working sign-in or recreation path after deleting a GitHub-linked ClawHub account.

## Best Possible Solution

Maintainers should decide the intended deleted-account policy, then either keep irreversible deletion and improve user-facing recovery messaging or add a controlled relink/admin recovery path with audit and abuse protections. Publish should separately provide clearer 403/429 recovery guidance around the GitHub account-age lookup, including Retry-After/backoff handling or deployment checks for GITHUB_TOKEN.
