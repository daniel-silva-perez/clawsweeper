## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only allows auto-closing ClawHub PRs that are certainly implemented on current main. The current code also still matches both reported surfaces: deleted/deactivated GitHub-linked accounts are rejected on reauth, and publish still throws the reported GitHub 403/429 rate-limit error from the account-age gate.

## Best Possible Solution

Keep this issue open for maintainer triage. Maintainers should decide whether self-deleted GitHub-linked accounts are intentionally irreversible or should have a controlled recreation, relink, or admin recovery path with audit and abuse protections, then align UI/docs with that policy. Separately, improve the publish flow around GitHub 403/429 account-age lookups with clearer retry guidance, Retry-After/backoff handling, deployment checks for GITHUB_TOKEN, or a narrower canonical follow-up for publish-rate-limit recovery.
