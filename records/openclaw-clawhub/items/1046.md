## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Current main also still preserves the two reported paths: self-deleted GitHub-linked accounts are rejected on sign-in rather than recreated, and publishing can still surface the GitHub API rate-limit error from the account-age gate. Related open reports #886 and #1674 confirm this is still an active account/publish access area rather than a resolved cleanup item.

## Best Possible Solution

Keep this issue open for maintainer triage. Decide whether self-deleted GitHub-linked accounts are intentionally irreversible or should have a controlled recreation/relinking/admin recovery path, then document or implement that path. Separately, improve the publish flow around GitHub 403/429 account-age lookups with clearer retry guidance, backoff, or backend configuration checks for GITHUB_TOKEN; consider splitting the account-recovery and publish-rate-limit work if maintainers want separate tracking.
