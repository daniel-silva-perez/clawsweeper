## Summary

Keep open. The cleanup policy for this repository keeps issues open, and current main still matches both reported surfaces: deleted/deactivated GitHub-linked accounts are blocked on reauth, while publish still surfaces GitHub 403/429 from the account-age gate as the reported rate-limit error.

## What This Changes

The issue reports skill publishing failing after validation with a GitHub API rate-limit error and GitHub sign-in not recreating or restoring a self-deleted account.

## Best Possible Solution

Maintainers should choose an explicit policy for self-deleted GitHub-linked accounts, then either keep deletion irreversible with clearer recovery/admin messaging or add a controlled recreation, relink, or admin recovery path with audit and abuse protections. Separately, publish should give clearer 403/429 recovery guidance around the GitHub account-age lookup, including Retry-After/backoff handling or deployment checks for GITHUB_TOKEN.
