## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only allows auto-closing pull requests that are certainly implemented on current main. Current main also still matches the reported behavior: self-deleted GitHub-linked accounts are blocked from reauth, and publish still surfaces the GitHub 403/429 account-age lookup as the reported rate-limit error.

## What This Changes

The issue requests clearer/recoverable handling for GitHub rate-limit failures during skill publish and a clean sign-in, recreation, restoration, or admin recovery path after deleting a GitHub-linked ClawHub account.

## Best Possible Solution

Maintainers should make an explicit product/security decision for self-deleted GitHub-linked accounts, then either keep the irreversible policy and improve user-facing messaging or add a controlled recreation, relink, or admin recovery path with audit and abuse protections. Separately, publish should provide clearer 403/429 recovery guidance around the GitHub account-age lookup, including Retry-After/backoff behavior or deployment checks for GITHUB_TOKEN.
