## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy for this pass only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main also still has the reported behavior split across two surfaces: self-deleted accounts are blocked from GitHub reauth, and publishing still calls the GitHub account-age gate that can surface the reported GitHub API rate-limit error. Related account/publish-access reports #504, #886, and #1674 remain open, so this needs maintainer triage rather than cleanup closure.

## Best Possible Solution

Keep this issue open for maintainer triage. Maintainers should decide whether self-deleted GitHub-linked accounts are intentionally irreversible or should have a controlled recreation/relink/admin recovery path with audit and abuse protections, then keep docs and UI consistent with that policy. Separately, improve the publish flow around GitHub 403/429 account-age lookups with clearer retry guidance, backoff, or deployment checks for GITHUB_TOKEN, and consider splitting account recovery and publish-rate-limit handling into canonical follow-up items.
