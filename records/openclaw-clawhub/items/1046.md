## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Current main also still preserves the reported behavior: self-deleted accounts cannot sign back in, and publish can still surface the GitHub API rate-limit error from the account-age gate.

## Best Possible Solution

Keep this issue open for maintainer triage. Decide whether self-deleted GitHub-linked accounts are intentionally irreversible or should have a controlled recreation/relinking path; then either document the support/admin path more clearly or implement that recovery. Separately, improve the publish flow around GitHub 403/429 account-age lookups with clearer retry guidance, backoff, or backend configuration checks for GITHUB_TOKEN.
