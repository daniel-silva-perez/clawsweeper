## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy keeps ClawHub issues open unless they are PRs certainly implemented on main. Substantively, current main still has both reported surfaces: deleted/deactivated GitHub-linked accounts are rejected on reauth, and publish still calls the GitHub account-age gate that throws the reported 403/429 rate-limit error.

## Best Possible Solution

Keep this issue open for maintainer triage. Maintainers should decide whether self-deleted GitHub-linked accounts are intentionally irreversible or should have a controlled recreation/relink/admin recovery path with audit and abuse protections, then align UI and docs with that policy. Separately, improve the publish flow around GitHub 403/429 account-age lookups with clearer retry guidance, backoff/Retry-After handling, deployment checks for GITHUB_TOKEN, or a narrower canonical follow-up for publish-rate-limit recovery.
