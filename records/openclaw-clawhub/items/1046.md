## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main also still preserves the reported account and publish behavior: self-deleted accounts are treated as permanently deleted on reauth, while publishing still calls the GitHub account-age gate that can surface the exact GitHub API rate-limit error. Related account/publish-access reports #504, #886, and #1674 are still open, so this area still needs maintainer triage rather than cleanup closure.

## Best Possible Solution

Keep this issue open for maintainer triage. Decide whether self-deleted GitHub-linked accounts are intentionally irreversible or should have a controlled recreation/relink/admin recovery path with audit and abuse protections; document the policy clearly either way. Separately, improve the publish flow around GitHub 403/429 account-age lookups with clearer retry guidance, backoff, or deployment checks for GITHUB_TOKEN, and consider splitting the account-recovery and publish-rate-limit work if maintainers want separate tracking.
