## Summary

Keep open. The ClawHub cleanup policy for this repository keeps issues open, and current main still matches both reported surfaces: deleted/deactivated GitHub-linked accounts are blocked on reauth, and publish still surfaces GitHub 403/429 from the account-age gate as the reported rate-limit error. Related open issues #504 and #886 cover the deleted-account login loop, while #1674 overlaps broader account-blocking recovery.

## What This Changes

The issue reports that skill publishing stops on a GitHub API rate-limit error and that GitHub sign-in does not recreate or restore access after deleting the ClawHub account.

## Best Possible Solution

Maintainers should make an explicit product/security decision for self-deleted GitHub-linked accounts, then either keep the irreversible policy and improve user-facing messaging or add a controlled recreation/relink/admin recovery path with audit and abuse protections. Separately, publish should give clearer 403/429 recovery guidance around the GitHub account-age lookup, including Retry-After/backoff behavior or deployment checks for GITHUB_TOKEN.
