## Summary

Keep open. The ClawHub cleanup policy only permits auto-closing pull requests that are certainly implemented on main, and this item is an issue. Substantively, current main still matches the reported moderation conflict: `always=true` is statically mapped to `suspicious.privileged_always`, that code is not externally clearable by clean scanner results, and the visible warning banner follows the suspicious flag. A staff manual override path exists, but this specific appeal still needs maintainer moderation or a policy/docs change.

## Best Possible Solution

Keep this issue open for maintainer moderation and product review. If the skill is reviewed and acceptable, use the existing moderator manual override path for the specific listing. If the broader policy should change, update `convex/lib/moderationReasonCodes.ts` and `convex/lib/moderationEngine.ts` with tests that define when `suspicious.privileged_always` may be cleared, and document any manifest field or appeal guidance authors should use for intentional `always=true`.
