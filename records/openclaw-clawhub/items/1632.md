## Summary

Keep this issue open. Repository cleanup policy for openclaw/clawhub permits auto-closing only PRs that are definitely implemented on current main, not issues. Substantively, current main still treats `always=true` as an unconditional static suspicious finding, only allows clean VT+LLM results to suppress the credential-access static code, and renders the platform suspicious banner from `moderationInfo.isSuspicious`; there is no manifest annotation or contextual-verdict precedence rule for intentional `always=true` skills yet.

## Best Possible Solution

Keep this issue open for maintainer moderation and product/security review. For the specific Hardcore listing, staff can use the existing manual override flow if review confirms it is acceptable. For the broader platform behavior, decide whether intentional `always=true` should become an allowlisted/static-clearable reason, whether contextual analysis can override it, or whether a documented manifest justification field should be added, then update the moderation engine, scan UI, docs, and regression tests together.
