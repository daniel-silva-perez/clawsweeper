## Summary

Keep this open. The ClawHub cleanup policy says open issues should not be auto-closed in this lane, and the substantive appeal still matches current main: `always=true` is converted into `suspicious.privileged_always`, that code is not externally clearable, clean/benign external scanners only suppress the single allowlisted credential-access static code, and the skill header banner renders from the resulting suspicious moderation state.

## What This Changes

The issue asks ClawHub to let a benign high-confidence contextual analysis override, or allow authors to justify, a static `always=true` suspicious banner for the Hardcore v5.1.0 skill.

## Best Possible Solution

Keep this open for maintainer moderation and product/security review. Staff should review the Hardcore listing and use the existing manual clean override if the specific skill is acceptable; separately, maintainers should decide whether intentional `always=true` can be contextually overridden, externally clearable under strict conditions, allowlisted with a high-confidence benign verdict, or represented by a manifest justification field, then update the moderation engine, scan UI/docs, and regression tests together.
