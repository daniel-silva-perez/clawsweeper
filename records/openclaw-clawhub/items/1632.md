## Summary

This is a ClawHub issue, and the repository-specific review policy says to keep ClawHub issues open. The current main code also still supports the reporter's underlying concern: `always=true` is statically classified as `suspicious.privileged_always`, that code is not in the externally clearable allowlist, and the skill header displays the platform warning whenever the skill moderation snapshot is suspicious. This remains product/moderation policy work rather than conservative cleanup.

## Best Possible Solution

Keep this issue open for maintainer moderation/product review.
