## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open, and current main still has a real remaining Convex pagination failure shape: the owner-publisher package and skill digest internal mutations each loop over `.paginate()` until done, so owners with more than one page can still run multiple paginated queries in a single mutation invocation. The earlier related PR #1183 is closed and unmerged, and its `collect()` approach would trade this error for unbounded reads rather than the safer cursor-rescheduling pattern.

## What This Changes

The issue reports that `clawhub publish` and `clawhub sync` fail during publish preparation with a Convex multiple-paginated-query error in owner-publisher digest synchronization.

## Best Possible Solution

Keep this issue open and land one focused backend PR that makes each owner-publisher package or skill digest internal mutation process at most one indexed page, then reschedule itself with the continuation cursor until complete. Preserve the `by_owner_publisher` indexed reads, avoid `.collect()` and unbounded scans, and add tests for multi-page package and skill catalogs.

## Repair Work Prompt

Fix the owner-publisher digest pagination failure reported by the publish/sync Convex error cluster. In `convex/functions.ts`, update `syncPackageSearchDigestsForOwnerPublisherIdInternal` and `syncSkillSearchDigestsForOwnerPublisherIdInternal` so each internal mutation invocation performs at most one `.paginate()` call against the existing `by_owner_publisher` index, processes that page, and if `isDone` is false schedules the same internal mutation with `continueCursor`. Extend args/helpers as needed with an optional/null cursor. Preserve indexed reads; do not replace pagination with `.collect()` or `.filter()`, and do not introduce unbounded scans. Keep the existing separate package/skill scheduling behavior. Add focused tests in `convex/functions.test.ts` that simulate multi-page package and skill results and assert only one paginated query occurs per invocation plus continuation rescheduling. Related refs: this issue, #1170, #1171, #1175, #1183, #1184, #1197, #1205. No release metadata or unrelated CLI behavior changes; no changelog unless maintainers require one for backend bugfixes.
