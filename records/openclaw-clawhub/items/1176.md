## Summary

Keep open. Repository policy for this review keeps ClawHub issues open, and current main still has a real remaining Convex pagination failure shape: the owner-publisher package and skill digest internal mutations each loop over `.paginate()` until completion, so an owner with more than one page can still run multiple paginated queries inside one mutation. The related PR #1183 is closed and unmerged, and its `.collect()` approach would avoid this exact error by taking on unbounded read risk rather than using cursor-rescheduled work.

## What This Changes

The issue asks for `clawhub publish` and `clawhub sync` to stop failing with Convex's multiple-paginated-queries error during owner-publisher digest sync.

## Best Possible Solution

Keep this issue open and land one focused backend PR that makes each owner-publisher package or skill digest internal mutation process at most one indexed page, then reschedule itself with the continuation cursor until complete. Preserve the `by_owner_publisher` indexed reads, avoid `.collect()` and unbounded scans, and add tests for multi-page package and skill catalogs.
