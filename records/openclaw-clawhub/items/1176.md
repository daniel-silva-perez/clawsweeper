## Summary

Keep this issue open. ClawHub cleanup policy keeps issues open in this pass, and current main only partially mitigates the reported publish/sync blocker: publisher-trigger package and skill digest rebuilds are now scheduled separately, but each owner-publisher digest helper can still run repeated `.paginate()` calls inside one internal mutation for owners with more than one page of packages or skills.

## Best Possible Solution

Keep this issue open and land a narrow backend fix for the related pagination cluster: process at most one owner-publisher package or skill page per internal mutation, then reschedule the same internal mutation with the continuation cursor until done. Preserve the `by_owner_publisher` indexed reads, avoid unbounded `collect()`, and add tests for multi-page rescheduling.
