## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has the reported Convex failure shape: CLI sync delegates to publish, publish reaches the server-side skill publish path, and the owner-publisher package/skill digest internal mutations can still call `.paginate()` repeatedly inside one mutation. The related fix PR #1183 is closed and unmerged, and its `.collect()` approach would avoid the immediate error by taking on unbounded read risk.

## What This Changes

The issue requests fixing CLI publish/sync failures caused by a Convex “multiple paginated queries” error during owner-publisher skill digest sync.

## Best Possible Solution

Implement bounded, cursor-rescheduled owner-publisher digest rebuilds so each internal mutation performs at most one indexed page of package or skill work, preserves the `by_owner_publisher` index usage, and has tests that exercise catalogs larger than one page.

## Repair Work Prompt

Fix the ClawHub publish/sync Convex pagination failure tracked by #1176 and related reports #1170, #1171, #1175, #1184, #1197, and #1205. In `convex/functions.ts`, change the owner-publisher package and skill digest internal mutations so each invocation performs at most one indexed `.paginate()` call over `by_owner_publisher`, processes that page, and reschedules itself with the continuation cursor until complete. Add validators for nullable cursor args as needed. Preserve the current indexed reads and do not replace the loops with `.collect()` or any unbounded scan. Update/add focused tests in `convex/functions.test.ts` for multi-page package and skill catalogs and for continuation rescheduling. Keep the patch scoped to this backend path; do not change CLI behavior except where a test requires updated mocks. No changelog entry is required unless the repository has an active unreleased changelog convention for backend bug fixes.
