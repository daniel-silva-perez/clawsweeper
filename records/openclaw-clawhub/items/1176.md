## Summary

Keep this issue open. The ClawHub cleanup policy only permits auto-closing pull requests that current main definitely implements, not issues. Current main partially mitigates the reported publish/sync blocker by moving package and skill owner-publisher digest rebuilds into separate scheduled mutations, but the owner-publisher digest helpers still contain paginated loops that can run more than one `.paginate()` call inside a single internal mutation for multi-page owners.

## Best Possible Solution

Keep this issue open for maintainer handling. Verify the deployed backend and current CLI publish/sync behavior for affected accounts, consolidate the related pagination reports under a canonical issue if appropriate, and change owner-publisher digest sync to cursor-schedule one page per mutation if large owner catalogs can still trigger Convex pagination limits.
