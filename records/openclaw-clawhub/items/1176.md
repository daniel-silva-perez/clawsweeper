## Summary

Keep open. Repository policy for this ClawHub cleanup pass keeps issues open, and current main still has a real remaining Convex pagination failure shape: publish/sync reaches personal-publisher setup, publisher writes schedule owner-publisher digest sync jobs, and both owner-publisher digest helpers can call `.paginate()` more than once inside a single internal mutation when an owner has more than one page of packages or skills. The related PR #1183 is closed and unmerged, and its `.collect()` approach would avoid the exact error by introducing unbounded read risk rather than cursor-rescheduled work.

## What This Changes

The issue reports that `clawhub publish` and `clawhub sync --all` fail during preparation with Convex's multiple-paginated-queries error in owner-publisher skill digest syncing.

## Best Possible Solution

Implement cursor-rescheduled owner-publisher digest rebuilds: each internal mutation should process at most one indexed page for packages or skills, then schedule itself with the continuation cursor until complete. Preserve `by_owner_publisher` indexed reads, avoid `.collect()` and unbounded scans, and add focused tests for multi-page package and skill catalogs.
