## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has the reported Convex failure shape: publish and sync reach the backend publish path, then owner-publisher digest synchronization can run repeated `.paginate()` calls inside one internal mutation. The related fix PR #1183 is closed and unmerged, and its `.collect()` approach would trade the immediate error for unbounded read risk.

## What This Changes

The issue asks to fix `clawhub publish` and `clawhub sync --all` failures caused by Convex's multiple-paginated-query limit during owner-publisher digest synchronization.

## Best Possible Solution

Implement bounded, cursor-rescheduled owner-publisher package and skill digest rebuilds so each internal mutation performs at most one indexed page of work, preserves existing digest semantics, and has tests for catalogs larger than one page.
