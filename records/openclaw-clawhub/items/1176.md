## Summary

Keep this issue open. ClawHub cleanup policy keeps issues open, and current main only partially mitigates the reported publish/sync blocker by scheduling package and skill owner-publisher digest rebuilds separately. The remaining owner-publisher digest helpers still loop over `.paginate()` inside one internal mutation, so large owner catalogs can still hit Convex's multiple-paginated-query limit. Related #1170 and #1175 are also open, and the proposed PR #1183 was closed unmerged.

## Best Possible Solution

Keep this issue open, consolidate it with the related pagination reports if maintainers want one canonical bug, and land a narrow backend fix that processes at most one owner-publisher package or skill page per internal mutation before rescheduling with the continuation cursor. Avoid an unbounded `collect()` replacement; keep the indexed `by_owner_publisher` reads and add tests for multi-page rescheduling.
