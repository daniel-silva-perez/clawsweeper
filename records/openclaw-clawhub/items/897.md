## Summary

Keep this issue open. The ClawHub cleanup policy says issues are not auto-closed by this review path, and current main still does not explicitly support `.ps1` files in the shared text-file allowlist, browser validation, server publish validation, CLI file collection/hash behavior, or docs/tests. The issue body also gives a concrete affected skill, win-geolocate, and a workaround of embedding the PowerShell in `SKILL.md`; related #290 remains open for the same user-visible gap.

## Best Possible Solution

Keep this issue open and either consolidate it with open #290 or implement explicit, security-aware PowerShell support. A complete fix should add accepted PowerShell extensions such as `.ps1` and possibly `.psm1`/`.psd1` to the shared schema and CLI allowlists, normalize content types, update browser and server publish validation, ensure raw file serving/static scan/GitHub import/CLI archive hashing behave consistently, add tests, and update the skill-format docs.
