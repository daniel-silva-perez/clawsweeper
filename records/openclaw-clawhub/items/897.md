## Summary

Keep open. Current main still does not intentionally accept `.ps1` skill files, the issue has concrete user context plus two confirmations, and related #290 remains open. Because this is an issue in `openclaw/clawhub` and the change would add a new executable script extension to the accepted skill-file surface, it needs maintainer product/security triage before an automated fix.

## What This Changes

Allow PowerShell `.ps1` files to be uploaded and published as text-based supporting files in ClawHub skills.

## Best Possible Solution

If maintainers approve PowerShell support, add `ps1` consistently to the shared schema allowlist and the CLI-mirrored allowlist, normalize it to an appropriate text content type, update docs and tests, and validate browser upload, archive/drop handling, Convex publish, GitHub import, raw file serving, and CLI publish/discovery paths together.
