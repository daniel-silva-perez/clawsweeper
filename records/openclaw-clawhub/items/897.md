## Summary

Keep open. Current main still does not intentionally support `.ps1` skill files, and the request is concrete with additional user confirmations plus related #290 still open. Because this is a ClawHub issue and the requested change would expand the accepted skill bundle surface to another executable script format, it should get maintainer product/security review before any allowlist change.

## What This Changes

The issue requests allowing `.ps1` PowerShell files to be uploaded and published as part of ClawHub skills.

## Best Possible Solution

If maintainers approve PowerShell support, add `ps1` consistently to the shared schema allowlist and CLI-mirrored allowlist, normalize it to an appropriate text content type, update docs and focused tests, and verify browser upload, archive/drop handling, Convex publish, GitHub import, raw file serving, and CLI publish/discovery paths together.
