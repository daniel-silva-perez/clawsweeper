## Summary

Keep open. Current main still does not intentionally accept `.ps1` skill files, the issue has concrete user context plus additional confirmations, and related #290 remains open. Because this is an issue in `openclaw/clawhub` and the change would expand the accepted skill bundle surface to another executable script format, it needs maintainer product/security triage before any automated allowlist change.

## What This Changes

The issue requests allowing `.ps1` PowerShell script files to be uploaded and published as part of ClawHub skills.

## Best Possible Solution

If maintainers approve PowerShell support, add `ps1` consistently to the shared schema allowlist and the CLI-mirrored allowlist, normalize it to an appropriate text content type, update docs and focused tests, and verify browser upload, archive/drop handling, Convex publish, GitHub import, raw file serving, and CLI publish/discovery paths together.
