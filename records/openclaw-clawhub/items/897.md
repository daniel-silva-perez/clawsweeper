## Summary

Keep this issue open. The repository policy for openclaw/clawhub only allows auto-closing PRs that are definitely implemented on main, and this item is an issue. The requested PowerShell upload support also remains incomplete on current main: .ps1 is absent from the shared and CLI text-file allowlists, web validation still depends on that allowlist or a text MIME type, server publish validation still rejects non-text files, and the CLI publish/hash paths skip non-allowlisted extensions. Related issue #290 remains open for the same .ps1 upload gap, and the issue has additional user confirmation in comments.

## Best Possible Solution

Keep this issue open and either consolidate it with open related issue #290 or implement explicit PowerShell support if maintainers accept the policy tradeoff. A complete implementation should add ps1 to the relevant allowlists, normalize or guess an appropriate text content type, update web/server/archive/CLI tests, and document the allowed file behavior.
