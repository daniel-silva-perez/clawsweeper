## Summary

Current main still does not intentionally support `.ps1` skill files, and ClawHub policy keeps issues open. The report is valid, related #290 is still open, and adding a PowerShell executable script extension needs maintainer product/security triage before an automated fix PR.

## What This Changes

Allow `.ps1` PowerShell scripts to be uploaded and published as part of ClawHub skills.

## Best Possible Solution

Keep this open and triage it with #290. If maintainers approve PowerShell as an accepted text helper format, add `ps1` consistently to the shared and CLI-mirrored text-file allowlists/content-type normalization, update docs and tests, and verify browser upload, Convex publish validation, GitHub import, archive/drop handling, static scan/read paths, and CLI discovery/publish all agree.
