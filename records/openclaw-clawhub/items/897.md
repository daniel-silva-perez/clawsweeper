## Summary

Keep open. The repository policy for openclaw/clawhub says issues should not be auto-closed, and current main still does not consistently support .ps1 files in the shared allowlist, web upload validation, server publish validation, or CLI publish path.

## Best Possible Solution

Keep this issue open and either consolidate it with open related issue #290 or implement explicit PowerShell support if maintainers accept the policy tradeoff: add ps1 to the shared and CLI schema allowlists, normalize/guess a text content type, update web/server/archive/CLI tests, and document the allowed file behavior.
