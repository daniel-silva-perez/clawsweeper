## Summary

Current main still does not intentionally accept `.ps1` as a text-based skill file extension. The upload UI, server publish validation, CLI file discovery, and GitHub import path all depend on the shared extension allowlist, which includes `.py`, `.sh`, `.js`, and other text/script formats but not `.ps1`. There is an older open duplicate/sibling report, #290, so the best cleanup is to keep this issue open and route it with that canonical PowerShell support work rather than auto-close an issue in this repository.

## Best Possible Solution

Keep this issue open and handle it together with #290. If maintainers approve PowerShell as an accepted text-based skill helper format, add `ps1` to the shared and CLI-mirrored text-file allowlists, normalize/guess it as `text/plain`, update tests and docs, and verify web upload, Convex publish validation, CLI publish/discovery, and GitHub import all behave consistently.
