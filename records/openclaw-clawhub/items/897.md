## Summary

Current main still does not intentionally support `.ps1` skill files. The shared text-file extension allowlist, browser publish validation, Convex publish validation, GitHub import filtering, and CLI discovery paths remain extension-allowlist based and exclude PowerShell. Because this is an openclaw/clawhub issue, repository policy says to keep issues open; it also overlaps with open sibling #290 and needs maintainer product/security approval before adding another executable script format.

## Best Possible Solution

Keep this issue open and handle it together with #290. If maintainers approve PowerShell as an accepted text-based skill helper format, add `ps1` consistently to the shared and CLI-mirrored text-file allowlists/content-type normalization, update tests and docs, and verify browser upload, Convex publish validation, GitHub import, static scan/read paths, and CLI discovery/publish behavior match.

- Browser MIME type behavior for `.ps1` may vary; relying on MIME alone would leave inconsistent acceptance unless the extension is explicitly supported and tested.
