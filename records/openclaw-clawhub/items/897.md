## Summary

Current main still does not intentionally support `.ps1` skill files. The shared and CLI-mirrored text-file extension allowlists exclude PowerShell, and browser upload, Convex publish validation, GitHub import, archive/drop handling, and CLI discovery all depend on that allowlist. Because this is an open ClawHub issue, repository policy keeps it open; it should be handled with open sibling #290 after maintainer product/security review.

## Best Possible Solution

Keep this issue open and triage it with #290. If maintainers approve PowerShell as an accepted text-based helper format, add `ps1` consistently to the shared and CLI-mirrored text-file allowlists/content-type normalization, update docs and tests, and verify browser upload, Convex publish validation, GitHub import, static scan/read paths, archive/drop handling, and CLI discovery/publish all agree.
