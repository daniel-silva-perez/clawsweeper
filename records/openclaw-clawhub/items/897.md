## Summary

Keep open. This is a ClawHub issue, and the target-repo cleanup policy says issues are not auto-closed by this review path. Current main also still lacks explicit PowerShell upload support: .ps1 is absent from the shared text-file allowlist, browser validation, server publish validation, CLI file collection/hash paths, and docs. Related #290 is open for the same user-visible gap.

## Best Possible Solution

Keep this issue open and either consolidate it with open #290 or implement explicit, security-aware PowerShell support if maintainers accept the policy tradeoff. A complete fix should add intended extensions such as .ps1, and likely .psm1/.psd1, to the shared and CLI allowlists; normalize content types; update browser and server publish validation; cover CLI archive/hash and GitHub import/static scan behavior; add tests; and update the skill-format docs.
