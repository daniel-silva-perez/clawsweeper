## Summary

Keep open. Current main still does not intentionally support `.ps1` skill files by extension, the issue has concrete user impact plus additional confirmations, and related #290 remains open. Because this is a ClawHub issue and the requested change expands the accepted skill-bundle surface to another executable script format, it needs maintainer product/security review before an automated allowlist change.

## What This Changes

Allow `.ps1` PowerShell script files to be uploaded and published as supporting files in ClawHub skills.

## Best Possible Solution

If maintainers approve PowerShell support, make `.ps1` a first-class text skill-file type across the shared schema and CLI mirror, normalize it to an appropriate text content type, update focused tests and docs, and verify browser upload, archive/drop expansion, Convex publish, GitHub import, raw file serving, and CLI publish/discovery behavior together. If maintainers reject the expansion, document the intentional exclusion and recommended workaround.
