## Summary

Keep open. Current main still does not intentionally support `.ps1` skill files by extension, the issue has concrete user impact plus confirmations, and related #290 remains open. Because this is an issue in `openclaw/clawhub` and PowerShell support would expand the accepted executable script surface for published skill bundles, it needs maintainer product/security approval before an allowlist change.

## What This Changes

Allow `.ps1` PowerShell scripts to be uploaded and published as skill files instead of forcing authors to embed them in `SKILL.md`.

## Best Possible Solution

If maintainers approve PowerShell support, add `ps1` consistently to the shared schema allowlist and CLI-mirrored allowlist, normalize it to an appropriate text content type, update focused tests and docs, and verify browser upload, archive/drop expansion, Convex publish, GitHub import, raw file serving, and CLI publish/discovery behavior together. If maintainers reject it, document the intentional exclusion and recommended workaround.
