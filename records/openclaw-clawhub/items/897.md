## Summary

Keep open. This is a ClawHub issue, and the repository policy says issues should not be auto-closed; current main also still does not consistently support .ps1 files across the shared allowlist, web upload validation, server publish validation, CLI publish path, or docs. Related issue #290 remains open for the same PowerShell upload gap.

## Best Possible Solution

Keep this issue open and either consolidate it with open related issue #290 or implement explicit PowerShell support if maintainers accept the policy tradeoff: add ps1 to the shared and CLI allowlists, normalize/guess an appropriate text content type, update web/server/archive/CLI tests, and document the allowed file behavior.
