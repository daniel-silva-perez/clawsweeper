## Summary

Keep this issue open. It is a ClawHub issue, so the provided repository policy does not allow auto-closing it; current main also still lacks consistent .ps1 support across the shared allowlist, browser upload validation, server publish validation, CLI publish path, and docs. Related issue #290 remains open for the same PowerShell upload gap.

## Best Possible Solution

Keep this issue open and either consolidate it with open related issue #290 or implement explicit PowerShell support if maintainers accept the policy tradeoff: add ps1 to the shared and CLI allowlists, normalize or guess an appropriate text content type, update web/server/archive/CLI tests, and document the allowed file behavior.
