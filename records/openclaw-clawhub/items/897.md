## Summary

Keep this issue open. The repository policy for openclaw/clawhub only allows auto-closing PRs that are definitely implemented on main, and this is an issue. The requested PowerShell upload support also still appears incomplete on current main across the shared allowlist, browser validation, server validation, CLI publish path, tests, and docs. Related issue #290 remains open for the same .ps1 upload gap.

## Best Possible Solution

Keep this issue open and either consolidate it with open related issue #290 or implement explicit PowerShell support if maintainers accept the policy tradeoff: add ps1 to the shared and CLI allowlists, normalize or guess an appropriate text content type, update web/server/archive/CLI tests, and document the allowed file behavior.
