## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that current main definitely implements. Substantively, current main still does not show the reported default-skill slugs as source-level skills or aliases: the CLI installs by resolving the exact slug through `/api/v1/skills/<slug>`, the API returns `404 Skill not found` when no public skill or alias resolves, and the reported names are absent from checked-in seed/alias content.

## Best Possible Solution

Keep this issue open for maintainer triage. Maintainers should decide whether these OpenClaw default skill names should be published or restored as official ClawHub skills, mapped through explicit safe aliases to maintained canonical skills, or removed/renamed in OpenClaw setup and install documentation so the CLI no longer directs users to nonexistent slugs.
