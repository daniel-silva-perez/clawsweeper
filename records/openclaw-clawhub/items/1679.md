## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy says ClawHub issues are not auto-closed by this workflow. The current code also still supports the reported failure mode: install resolves the literal slug through the skill detail API, the API returns 404 when no public skill or explicit alias resolves, and checked-in source/docs do not define the reported default-skill names. A live read-only API check still returned 404 for `coding-agent`, `himayala`, `summarize`, and `wacli`; `himalaya` now resolves, which suggests at least one reported name may be a spelling/default-list mismatch.

Keep this issue open for maintainer triage. Avoid adding skill content directly to the source tree; use the registry/publish path or an explicit migration/alias operation if these are official defaults.
