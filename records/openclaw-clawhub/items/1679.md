## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy for this pass only permits auto-closing pull requests that current main definitely implements. The report also still matches current behavior: CLI install resolves the literal slug through the v1 skill detail API, the backend only accepts an existing skill or explicit alias, and live checks still return 404 for the reported slugs except the likely typo correction `himalaya`, which resolves.

## What This Changes

The issue reports that installing several expected default skill slugs through the ClawHub CLI fails because `/api/v1/skills/<slug>` returns `404 Skill not found`.

## Best Possible Solution

Maintainers should decide whether these names are intended official OpenClaw defaults. If they are, restore them through the ClawHub publishing or alias-management path; if not, update the OpenClaw setup/default-skill guidance so users are not told to install nonexistent slugs.
