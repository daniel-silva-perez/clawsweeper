## Summary

Keep this issue open. Repository policy for openclaw/clawhub only permits auto-closing PRs that are certainly implemented on main, not issues. Substantively, current main still resolves skill installs through GET /api/v1/skills/<slug>, only supports direct skill rows or explicit slug aliases, and returns 404 when neither exists. The live ClawHub API still returns 404 for the reported slugs: coding-agent, himayala, summarize, and wacli.

## Best Possible Solution

Keep this issue open for maintainer triage. Maintainers should decide whether these default OpenClaw skill names should be published or restored as verified ClawHub skills, mapped through explicit safe aliases to maintained canonical skills, or removed/renamed in OpenClaw setup and install documentation so the CLI no longer points users at nonexistent slugs.
