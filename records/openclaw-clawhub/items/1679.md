## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that current main certainly implements. The current code and live production API still preserve the reported behavior: install resolves the literal slug through `/api/v1/skills/<slug>`, missing public skill or alias records return `Skill not found`, and the reported default skill slugs are not present in the checked-in source or live API responses.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to decide whether these OpenClaw default skill names should be published/restored as official ClawHub skills, mapped through explicit maintained aliases, or removed/renamed in OpenClaw setup and install documentation so the CLI no longer points users at nonexistent slugs.
