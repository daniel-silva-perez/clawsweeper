## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only allows auto-closing ClawHub pull requests that are certainly implemented on current main, not issues. The current source still resolves install metadata through /api/v1/skills/<slug> and returns 404 when a requested slug has no public skill row or explicit alias, so this remains a maintainer triage item.

## Best Possible Solution

Keep this issue open for maintainer triage. Maintainers should decide whether these default OpenClaw skill names should be published or restored as official ClawHub skills, mapped through explicit safe aliases to canonical maintained skills, or removed/renamed in OpenClaw setup and install documentation so the CLI no longer points users at nonexistent slugs.
