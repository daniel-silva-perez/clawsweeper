## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy for this run only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Substantively, current main still resolves installs through GET /api/v1/skills/<slug>; a slug with no public skill row or explicit alias returns 404 "Skill not found", and the reported default-skill names are not represented as built-in source/docs/seed slugs or aliases in the checkout.

## Best Possible Solution

Keep this issue open for maintainer triage. Maintainers should decide whether these default OpenClaw skill names should be published or restored as official ClawHub skills, mapped through explicit safe aliases to canonical maintained skills, or removed/renamed in OpenClaw setup and install documentation so the CLI no longer points users at nonexistent slugs.
