## Summary

Keep this open. Repository policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on current main, and this item is an issue. Substantively, current main still resolves `clawhub install <slug>` through `GET /api/v1/skills/<slug>` and returns 404 when no public skill row or explicit alias exists; the reported default-skill names are not present as checked-in built-in slugs, seed entries, docs examples, or alias mappings.

## Best Possible Solution

Keep this issue open for maintainer triage. Maintainers should decide whether these OpenClaw default-skill names should be published or restored as official ClawHub skills, mapped through explicit safe aliases to canonical maintained skills, or removed/renamed in OpenClaw setup and install documentation so the CLI no longer points users at nonexistent slugs.
