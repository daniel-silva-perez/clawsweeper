## Summary

Keep this PR open. Current main still implements `clawhub list` as a lockfile-only command, and the PR’s intended behavior of listing manually installed, untracked skill directories is not present. The related cleanup PR #1419 is closed unmerged and does not supersede this work.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase it onto the current `packages/clawhub` tree, implement `clawhub list` by combining lockfile entries with valid untracked skill directories under `opts.dir`, update CLI help and docs, add focused tests for manual skills and filesystem-error handling, and resolve the scan-error handling concern before merge.
