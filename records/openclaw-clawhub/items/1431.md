## Summary

Keep this issue open. The ClawHub cleanup policy for this repository only allows auto-closing pull requests that are certainly implemented on current main, and this item is an issue. Current main also still matches the reported bug: the publish UI can submit an organization owner handle for an existing skill, but the backend rejects an existing skill when its stored `ownerPublisherId` differs from the selected publisher, and the transfer path remains user-to-user rather than org/publisher-targeted.

## Best Possible Solution

Keep this issue open for maintainer/product handling. The likely fix is either an explicit authorized publisher-change path in the publish/version flow, or an org-aware transfer flow that accepts publisher targets, updates `skills.ownerPublisherId` and `skillSlugAliases` consistently, and exposes the behavior through the web UI, HTTP API, CLI, tests, and docs. The closed unmerged PRs #1562, #1388, #1603, and #1604 are useful implementation references; #1569 should be considered during consolidation.
