## Summary

Keep this issue open. The ClawHub cleanup policy for this pass only permits auto-closing pull requests that are certainly implemented on current main, and this item is an issue. Current main also still lacks the requested personal-to-organization owner change for an existing skill: the publish path still treats a different existing `ownerPublisherId` as a slug collision, and the transfer path remains user-to-user rather than publisher/org-targeted.

## Best Possible Solution

Keep this issue open for maintainer/product handling. The likely fix is either an explicit authorized publisher-change path in the existing skill publish/version flow, or a dedicated org-aware skill transfer flow that accepts publisher targets, updates skill ownership and slug aliases consistently, and exposes the behavior through the UI, HTTP API, CLI, and docs. The closed unmerged PRs #1562, #1388, #1603, and #1604 remain useful implementation references, and #1569 is the active related user-facing report.
