## Summary

Keep open. The repository policy for this ClawHub pass only permits auto-closing pull requests, and current main still has the reported gap: the publish UI can submit an organization owner for an existing skill, but the backend rejects a different existing `ownerPublisherId` as a slug collision. The dedicated transfer path is still user-to-user/personal-publisher based and does not provide org-targeted transfer fields.

## Best Possible Solution

Keep this issue open and route it with #1569 and open PR #1635. The best fix is a maintainer-approved publisher-change or ownership-transfer flow that supports org targets consistently across `skills.ownerPublisherId`, `skillSlugAliases`, publish/edit UI, HTTP API, CLI, docs, and tests. If ownership moves should not happen through republish, the UI should stop offering that path for existing skills and direct users to the transfer flow instead.
