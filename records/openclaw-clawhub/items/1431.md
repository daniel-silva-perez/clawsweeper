## Summary

Keep this issue open. The repository-specific cleanup policy for openclaw/clawhub only permits auto-closing pull requests that current main definitely implements, and this item is an issue. Current main also still matches the reported bug: the publish UI can submit a different selected owner, but `insertVersion` rejects an existing skill whose `ownerPublisherId` differs from the requested publisher as a slug collision. The alternate transfer path remains user-to-user, not publisher/org-targeted.

## Best Possible Solution

Keep this issue open for maintainer/product follow-up. A robust fix would add an authorized publisher-change path in the publish/version flow or an org-aware transfer flow that accepts publisher targets, updates `skills.ownerPublisherId` and `skillSlugAliases` consistently, and covers the web UI, HTTP API, CLI, tests, and docs. Review #1562, #1388, #1603, and #1604 as implementation references, and consider consolidating with #1569 if maintainers want one canonical tracking issue.
