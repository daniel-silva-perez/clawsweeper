## Summary

Keep open. This is an open ClawHub issue, and the repository-specific policy for this review only permits automated closure for pull requests that are certainly implemented on current main. A source pass also confirms current main still rejects the reported personal-to-org republish path and still lacks an org-targeted transfer path.

## Best Possible Solution

Keep this issue open for maintainer/product handling. A robust fix should add an authorized publisher-change path in the publish/version flow or an org-aware transfer flow that accepts publisher targets, updates `skills.ownerPublisherId` and `skillSlugAliases` consistently, and covers the web UI, HTTP API, CLI, tests, and docs. Review #1562, #1388, #1603, and #1604 as implementation references, and consider consolidating with #1569 if maintainers want one canonical tracking issue.
