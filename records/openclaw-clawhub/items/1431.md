## Summary

Keep open. Current main still allows the publish UI to submit a different owner publisher for an existing skill, but `insertVersion` rejects an existing skill whose stored `ownerPublisherId` differs from the selected publisher. The dedicated transfer path also remains user-to-user/personal-publisher based, and the related open PR #1635 is the active maintainer review path rather than a reason to close this issue.

## What This Changes

Allow an existing skill owner who belongs to an organization to move that skill from a personal publisher to the organization publisher, either through republish or a dedicated transfer flow.

## Best Possible Solution

Keep this issue open and route it with #1569 and open PR #1635. The best fix is a maintainer-approved publisher-target transfer flow that supports org targets consistently across the backend schema, aliases, HTTP API, CLI, publish/edit UI, docs, and regression tests; alternatively, if republish must not transfer ownership, the publish UI should stop offering owner changes for existing skills and point users to transfer.
