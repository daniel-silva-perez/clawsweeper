## Summary

Keep open. The ClawHub cleanup policy for this pass keeps issues open, and current main still matches the reported failure path: the publish UI can submit a different owner for an existing skill, while the backend rejects an existing skill whose stored publisher differs from the selected publisher. The dedicated transfer path is still user-to-user and personal-publisher based, so org ownership transfer remains unresolved; related org-transfer PRs are closed unmerged and #1569 remains open.

## What This Changes

The issue requests a way to move an existing skill from a personal publisher to an organization publisher without hitting the slug-collision error.

## Best Possible Solution

Land a maintainer-approved publisher-target ownership-transfer design that updates skills, slug aliases, public URLs, HTTP API, CLI behavior, docs, and tests consistently. If republish is not intended to move ownership, the existing-skill publish UI should stop offering owner changes and route users to the supported transfer flow.
