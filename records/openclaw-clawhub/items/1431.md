## Summary

Keep open. ClawHub cleanup policy for this pass does not auto-close issues, and current main still matches the reported failure path: the publish form can submit an organization owner for an existing skill, but the backend rejects the existing slug when the stored publisher differs from the selected publisher. The dedicated transfer path also remains user-to-user and personal-publisher based, while related org-transfer PRs are closed unmerged and #1569 remains open.

## What This Changes

The issue asks for an existing skill to be movable from a personal publisher to an organization publisher, either through republish or a dedicated ownership-transfer flow.

## Best Possible Solution

Land a maintainer-approved publisher-target ownership-transfer design that updates skills, slug aliases, public URLs, HTTP API behavior, CLI behavior, docs, and tests consistently. If republish is not intended to move ownership, the existing-skill publish UI should avoid offering owner changes and route users to the supported transfer flow.
