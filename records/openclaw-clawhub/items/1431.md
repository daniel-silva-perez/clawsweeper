## Summary

Keep this issue open. The repository policy for this ClawHub cleanup pass only permits auto-closing pull requests that are certainly implemented on current main, and this item is an issue. Substantively, current main still does not support changing an existing skill from a personal publisher to an organization publisher: the publish path still treats a different existing `ownerPublisherId` as a slug collision, and the transfer path remains user-to-user instead of publisher/org-targeted.

## Best Possible Solution

Keep this issue open for maintainer/product handling. The likely fix is either an explicit authorized publisher-change path in the existing skill publish/version flow, or an org-aware skill transfer flow that accepts publisher targets, updates `skills.ownerPublisherId` and `skillSlugAliases` consistently, and exposes the behavior through the web UI, HTTP API, CLI, tests, and docs. The closed unmerged PRs #1562, #1388, #1603, and #1604 are useful implementation references; #1569 is the active related user-facing report to consider during consolidation.
