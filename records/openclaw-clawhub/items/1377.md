## Summary

Keep open. The report remains valid against current main because ClawHub still models published skill licensing as a platform-wide MIT-0/no-attribution contract, and this repository’s cleanup policy keeps ClawHub issues open for maintainer judgment rather than auto-closing them.

## What This Changes

The issue asks ClawHub to show the named skill's actual MIT license instead of MIT-0/no-attribution and to check whether the mismatch affects other published skills.

## Best Possible Solution

ClawHub should have one coherent license model across upload acceptance, Convex storage, public API schemas, listing UI, CLI output, docs, migrations or backfills, and tests so published listings cannot contradict the actual license contract.
