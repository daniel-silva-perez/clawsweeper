## Summary

Keep this issue open. Current main still does not support displaying standard MIT for an individual skill: skill publishing, storage validators, public API types, UI display, CLI display, and docs all treat MIT-0 as the platform-wide skill license. The repository cleanup policy for openclaw/clawhub also says issues should remain open in this workflow; only PRs can be proposed for auto-close when implemented on main.

## Best Possible Solution

Maintain this as an open maintainer decision. Either add real per-skill SPDX license support by parsing/validating package or manifest license metadata, widening schema/API/client types, migrating existing versions, and updating UI/CLI/docs/tests; or explicitly reaffirm the platform-wide MIT-0 publishing policy and address publisher consent/documentation for skills whose upstream sources use standard MIT.
