## Summary

Keep this issue open. The openclaw/clawhub cleanup policy only permits auto-closing PRs that are certainly implemented on main, and current main still implements a platform-wide MIT-0 license path rather than preserving or displaying the reporter's upstream MIT license metadata.

## Best Possible Solution

Keep this open for maintainer product/legal judgment. Maintainers should either explicitly reaffirm and document ClawHub's platform-wide MIT-0 publishing policy with handling for conflicting upstream licenses, or implement per-skill SPDX license support across parsing, storage, API responses, UI, CLI, docs, migration/backfill, and tests.
