## Summary

Keep open. Current main still hardcodes and publishes skill licenses as MIT-0/no-attribution across shared schema, Convex storage, publish flow, public surfaces, docs, CLI, and tests, so the reported MIT versus MIT-0 mismatch remains valid. Under the ClawHub cleanup policy this issue should not be auto-closed, and the fix needs maintainer/legal product direction rather than a narrow automated patch.

## What This Changes

The issue asks ClawHub to display the `openclaw-workspace-sync` skill as standard MIT rather than MIT-0 and to check whether the same license mismatch affects other skills.

## Best Possible Solution

Maintainers should choose one license model and implement it consistently across publish acceptance, Convex storage, shared/API schemas, listing UI, CLI output, docs, migrations or backfills, and tests so published skill listings cannot disagree with the actual license contract.
