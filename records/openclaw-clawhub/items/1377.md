## Summary

Keep this open under the ClawHub cleanup policy for issues, and because current main still implements a platform-wide MIT-0/no-attribution license path rather than showing per-skill MIT metadata. The report appears valid, but the appropriate fix requires maintainer product/legal judgment across schema, publish, API, UI, CLI, docs, migration/backfill, and tests.

## What This Changes

The issue asks ClawHub to display the standard MIT license, not MIT-0/no-attribution, for a published skill whose upstream package metadata and LICENSE file are MIT.

## Best Possible Solution

Keep this open for maintainer judgment. Maintainers should either reaffirm ClawHub's platform-wide MIT-0 publishing policy and clearly document or reject conflicting upstream package metadata, or implement per-skill SPDX license support consistently across parsing, shared schemas, Convex storage/API responses, listing/upload UI, CLI/docs, migration/backfill, and tests.
