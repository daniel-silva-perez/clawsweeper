## Summary

Keep this issue open. Current main still hard-codes ClawHub skill licensing as MIT-0 across schema, publish storage, API/UI/CLI display, and docs, while the reporter provided concrete upstream MIT evidence for the affected skill. The ClawHub cleanup policy also keeps issues open unless maintainer action resolves them.

## Best Possible Solution

Keep this issue open for maintainer judgment. Either explicitly reaffirm ClawHub’s platform-wide MIT-0 publishing policy and address publisher consent for upstream MIT skills, or implement real per-skill SPDX license support by parsing license metadata, widening schemas and validators, migrating/backfilling stored versions, and updating API, UI, CLI, docs, and tests.
