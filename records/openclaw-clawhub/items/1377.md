## Summary

Keep this issue open. Current main still treats published skill licensing as a platform-wide MIT-0 policy, not per-skill SPDX metadata, and the repository policy for openclaw/clawhub says this review workflow should keep issues open even when cleanup evidence is strong.

## Best Possible Solution

Maintain this as an open maintainer decision. Either add real per-skill SPDX license support by parsing and preserving package or manifest license metadata, widening schema/API/client types, migrating existing versions, and updating UI/CLI/docs/tests; or explicitly reaffirm the platform-wide MIT-0 publishing policy and address publisher consent/documentation for skills whose upstream sources use standard MIT.
