## Summary

Keep this issue open. Current main still forces MIT-0 as the platform skill license across shared schemas, Convex storage/publishing, public API types, UI display, CLI behavior, and docs; it does not preserve or display standard MIT for an individual skill. Also, the openclaw/clawhub cleanup policy only proposes auto-close for PRs that are certainly implemented on main, not issues.

## Best Possible Solution

Keep this open for maintainer judgment. Either add real per-skill SPDX license support by parsing package/manifest/license metadata, widening storage and API schemas, migrating/backfilling existing records, and updating UI/CLI/docs/tests; or explicitly reaffirm the platform-wide MIT-0 publishing policy and address publisher consent/documentation for skills whose upstream sources use standard MIT.
