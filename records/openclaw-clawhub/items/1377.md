## Summary

Keep this issue open. The ClawHub policy for this review run only allows auto-closing PRs that are certainly implemented on main, and this is an issue. Current main also still models and displays published skill licenses as the platform-wide MIT-0 license, so the reported MIT-versus-MIT-0 mismatch remains a real maintainer decision rather than resolved cleanup.

## Best Possible Solution

Keep this issue open for maintainer judgment. Maintainers should either explicitly reaffirm and document ClawHub's platform-wide MIT-0 publishing policy, including how conflicting upstream MIT licenses are handled, or implement per-skill SPDX license support across parsing, storage, API responses, UI, CLI/docs, migration/backfill, and tests.
