## Summary

Keep open. Current main still treats skill licenses as a platform-wide MIT-0/no-attribution contract, so the reported MIT versus MIT-0 mismatch remains unresolved, and the repository cleanup policy keeps ClawHub issues open rather than auto-closing them.

## What This Changes

The issue requests that a ClawHub skill listing show standard MIT, not MIT-0, when the upstream package metadata and LICENSE file specify MIT.

## Best Possible Solution

Maintainers should choose one license model and implement it consistently across upload acceptance, Convex storage, public API schemas, listing UI, CLI output, docs, migrations or backfills, and tests so published skill listings cannot disagree with the actual license contract.
