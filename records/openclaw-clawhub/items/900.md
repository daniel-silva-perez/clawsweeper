## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. The reported account lifecycle problem also remains unresolved: current main intentionally blocks sign-in after user-initiated account deletion, and related reports #504, #907, and #917 are still open.

## Best Possible Solution

Keep this issue open and consolidate it with #504, #907, #917, or a dedicated deleted-account lifecycle tracking issue. Maintainers should either preserve irreversible deletion and improve the OAuth/CLI/settings error path, or implement a safe recreate/relink/admin recovery flow with tests across auth, user access checks, token creation, settings UI, and docs.
