## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing PRs that are certainly implemented on main. Current main still intentionally blocks re-login after user-initiated account deletion, so the reported account lifecycle problem remains a product/security decision rather than implemented cleanup.

## Best Possible Solution

Keep this issue open for maintainer triage, preferably consolidated with #504 and the other deleted-account re-login reports. Maintainers should decide whether deletion remains permanently blocking as documented, or whether ClawHub should support a safe recreate/relink or admin recovery flow. If the product decision is to allow re-login, likely changes belong in convex/auth.ts, convex/users.ts, convex/lib/access.ts, the settings and CLI token flows, and matching docs/tests; if deletion remains irreversible, the UI should make the blocked state and support path clearer after OAuth returns.
