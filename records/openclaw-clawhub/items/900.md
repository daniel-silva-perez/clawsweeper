## Summary

Keep open. Current main still treats self-deleted/deactivated accounts as inactive and blocks the same GitHub identity from becoming an active user again; the documented behavior is explicitly irreversible, while the reported desired recovery/relink behavior remains unresolved and related reports are still open.

## What This Changes

The issue requests that a GitHub identity tied to a deleted ClawHub account can sign back in, recreate or relink the account, and access CLI tokens instead of staying in a sign-in loop.

## Best Possible Solution

End with one consistent deleted-account policy enforced across OAuth callback handling, active-user gates, token creation, settings/header/CLI UI, and docs, with regression tests for both self-deleted re-login and CLI auth behavior.
