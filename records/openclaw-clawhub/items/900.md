## Summary

Keep open. Current main still blocks self-deleted/deactivated GitHub identities from becoming active users again, which matches the OAuth loop, persistent sign-in button, and CLI-token lockout. This is an open ClawHub issue, so the cleanup policy would keep it open anyway, and the remaining fix needs maintainer product/security policy before code changes.

## What This Changes

Allow a user who deleted their ClawHub account to sign in with GitHub again and regain access to their account and CLI token creation instead of remaining in a logged-out/sign-in loop.

## Best Possible Solution

Make one explicit deleted-account policy and implement it consistently across OAuth callback handling, active-user gates, token creation, settings/header/CLI UI, management recovery paths, audit logging, and docs, with regression tests for self-deleted re-login and CLI auth behavior.
