## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Substantively, current main still treats user deletion as irreversible: OAuth reauth for deleted/deactivated users is blocked, deleted users are excluded from active-user gates, token creation requires an active user, and the docs/tests assert that deleted accounts cannot be restored by logging in again. Related account-lifecycle reports #504, #907, and #917 are also still open.

## Best Possible Solution

Keep this issue open and consolidate it with #504, #907, #917, or a dedicated deleted-account lifecycle tracker. Maintainers should choose one policy: preserve irreversible deletion and improve the OAuth/CLI/settings error path, or implement a safe recreate, relink, admin recovery, or GitHub identity unlink flow with tests across auth callbacks, active-user gates, token creation, settings UI, and docs.
