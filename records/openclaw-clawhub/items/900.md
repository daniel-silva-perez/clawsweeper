## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits automated closure for PRs that are certainly implemented on main. Current main still intentionally blocks re-login after user-initiated account deletion, so the reported OAuth/account lifecycle problem remains unresolved.

## Best Possible Solution

Keep this issue open and consolidate it with #504/#907 or a dedicated deleted-account lifecycle tracking issue. Maintainers should decide whether deletion remains permanently blocking as documented, or implement a safe recreate/relink/admin recovery flow with tests across `convex/auth.ts`, `convex/users.ts`, `convex/lib/access.ts`, CLI token creation, settings UI, and docs.
