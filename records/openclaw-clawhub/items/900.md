## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. The current code also still rejects sign-in for deleted/deactivated users, blocks token creation without an active user, documents deletion as irreversible, and related account-lifecycle reports #504, #907, and #917 remain open.

## Best Possible Solution

Keep this issue open and consolidate it with #504, #907, #917, or a dedicated deleted-account lifecycle tracker. Maintainers should either preserve irreversible deletion and improve the OAuth/CLI/settings error path, or implement a safe recreate, relink, admin recovery, or GitHub identity unlink flow with tests across auth callbacks, active-user gates, token creation, settings UI, and docs.
