## Summary

Keep open. This is an open ClawHub issue, and the repository policy only permits auto-closing ClawHub PRs that are certainly implemented on main. Current main still intentionally treats user-initiated deletion as irreversible: account deletion deactivates/purges the user, OAuth reauth rejects deactivated accounts, active-user helpers hide the user from UI/token flows, and docs/tests assert that deleted accounts cannot be restored by logging in again. The related #504 remains open for the same account lifecycle problem, so consolidation is a maintainer triage path rather than an auto-close reason.

## Best Possible Solution

Keep this issue open for maintainer triage, preferably consolidated with #504 and the other deleted-account re-login reports. Maintainers should decide whether account deletion remains permanently blocking as documented, or whether ClawHub should support a safe recreate/relink or admin recovery flow. If the product decision is to allow re-login, likely changes belong in convex/auth.ts, convex/users.ts, active-user access helpers, settings/CLI token flows, and matching docs/tests.
