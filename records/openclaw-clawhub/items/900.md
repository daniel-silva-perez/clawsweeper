## Summary

Keep open. Current main still treats user-initiated account deletion as irreversible and blocks deleted/deactivated OAuth identities from becoming active users again, so the reported GitHub re-login loop remains a real account-lifecycle gap. ClawHub cleanup policy also keeps issues open unless maintainers explicitly decide otherwise.

## What This Changes

The issue requests that a user who deleted their ClawHub account can sign in with GitHub again, recover or recreate an active account, and access CLI token features instead of remaining in a logged-out sign-in loop.

## Best Possible Solution

Make an explicit maintainer decision for deleted-account identity handling, then implement that policy consistently: either keep deletion irreversible with clearer OAuth/settings/CLI recovery messaging and support handling, or add a deliberate restore/relink/recreate flow with security checks and regression coverage across OAuth callback handling, active-user gates, token creation, settings UI, CLI auth, and docs.
