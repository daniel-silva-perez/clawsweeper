## Summary

Keep open. Current main still blocks deleted/deactivated users from OAuth reauth and active-user paths, so the reported deleted-account GitHub sign-in loop remains a real account-lifecycle gap; ClawHub cleanup policy also keeps issues open unless maintainers decide otherwise.

## What This Changes

A deleted ClawHub account linked to GitHub should be able to sign back in, relink, unlink, or recreate access so the user can reach settings and CLI tokens instead of remaining in a sign-in loop.

## Best Possible Solution

Keep this open and consolidate it with #504, #907, and #917 under one maintainer account-lifecycle decision: either preserve irreversible deletion with clearer OAuth/CLI/settings recovery messaging and support handling, or design a deliberate restore/relink/unlink/recreate flow with tests across OAuth callback handling, active-user gates, token creation, settings UI, CLI auth, and security docs.
