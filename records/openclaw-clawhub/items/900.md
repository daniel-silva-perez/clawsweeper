## Summary

Keep this issue open. ClawHub cleanup policy only permits auto-closing PRs that current main definitely implements, and current main still intentionally blocks user-initiated deleted accounts from OAuth reauth, active-user access, and CLI/API token creation. Related open reports #504, #907, and #917 should be consolidated under an explicit account-lifecycle policy decision.

## Best Possible Solution

Keep this issue open and consolidate it with #504, #907, and #917 or designate one canonical deleted-account lifecycle tracker. Maintainers should explicitly choose the product/security policy: preserve irreversible deletion and improve OAuth/CLI/settings error handling, or add a deliberate recreate, relink, admin recovery, or GitHub identity unlink flow with tests across auth callback handling, active-user gates, token creation, settings UI, CLI auth, and security docs.
