## Summary

Keep open. Current main still blocks deleted/deactivated GitHub identities from becoming active users again, so the reported re-login loop and CLI-token lockout remain a real account-lifecycle gap. The ClawHub cleanup policy also keeps issues open unless maintainers explicitly decide otherwise, and the related reports remain open rather than merged or superseded.

## What This Changes

The issue requests that GitHub OAuth after user-initiated account deletion re-create, re-link, or restore the account so the user can log in and create CLI tokens instead of returning to an active sign-in button.

## Best Possible Solution

Make an explicit product/security decision for deleted-account identity handling, then implement that decision consistently: either preserve irreversible deletion with clearer OAuth/settings/CLI recovery messaging and support routing, or add a deliberate restore/relink/recreate flow with auditability and regression coverage across OAuth callback handling, active-user gates, token creation, settings UI, CLI auth, and docs.
