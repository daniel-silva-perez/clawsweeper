## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and current main still matches the reported deleted-account lockout: self-service deletion marks the user deactivated/purged, OAuth reauth rejects deleted/deactivated users, active-user gates return no user, CLI/settings token creation requires an active user, and the header/CLI UI can continue presenting sign-in when `me` is null. Related open reports #504, #907, and #917 should be consolidated under an explicit account-lifecycle policy decision.

## What This Changes

The issue requests that a user who deleted their ClawHub account can sign back in with GitHub or otherwise recover/re-link access, instead of ending in an OAuth loop with the sign-in button still available.

## Best Possible Solution

Keep this open and consolidate it with #504, #907, and #917 under one account-lifecycle decision. Maintainers should explicitly choose between preserving irreversible deletion with clearer OAuth/CLI/settings recovery messaging and support handling, or adding a deliberate restore/relink/unlink/recreate flow with tests for OAuth callback handling, active-user gates, token creation, settings UI, CLI auth, and security docs.
