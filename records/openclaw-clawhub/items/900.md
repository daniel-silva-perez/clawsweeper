## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that current main definitely implements. Current main still preserves irreversible account deletion semantics: OAuth sign-in for deactivated or legacy deleted users is blocked, active-user gates reject deleted/deactivated users, API token creation requires an active user, and docs/tests explicitly say deleted accounts cannot be restored by logging in again. Related account-lifecycle reports #504, #907, and #917 remain open, so this needs consolidation or an explicit maintainer policy decision rather than cleanup closure.

## Best Possible Solution

Keep this issue open and consolidate it with #504, #907, #917, or a dedicated deleted-account lifecycle tracker. Maintainers should choose one policy: preserve irreversible deletion and improve the OAuth, CLI, and settings error path, or implement a safe recreate, relink, admin recovery, or GitHub identity unlink flow with tests across auth callbacks, active-user gates, token creation, settings UI, and docs.
