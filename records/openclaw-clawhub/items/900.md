## Summary

Keep open. Current main still blocks self-deleted/deactivated GitHub identities from becoming active users again, and ClawHub cleanup policy keeps issues open unless maintainers explicitly decide otherwise. The remaining work is a product/security policy decision, not a narrow autonomous code cleanup.

## What This Changes

The reporter asks for GitHub OAuth sign-in to re-create, relink, or restore a self-deleted ClawHub account so account access and CLI tokens work again.

## Best Possible Solution

End with one explicit deleted-account policy enforced across OAuth callback handling, active-user gates, token creation, settings/header/CLI UI, and docs, with regression tests for both self-deleted web re-login and CLI auth behavior.
