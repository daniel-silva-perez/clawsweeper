## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only permits auto-closing PRs that are certainly implemented on main. Current main still intentionally blocks re-login after user-initiated account deletion: deletion sets deactivated/purged fields, OAuth sign-in rejects deactivated users, active-user helpers hide them from UI/API-token flows, and docs/tests describe deletion as irreversible rather than supporting GitHub relink/recreate.

## Best Possible Solution

Keep this issue open for maintainer triage, preferably consolidated with #504 and the other deleted-account re-login reports. Maintainers should decide whether account deletion remains permanently blocking as documented, or whether ClawHub should support a safe recreate/relink or admin recovery flow. If the product decision is to allow re-login, likely changes belong in convex/auth.ts, convex/users.ts, active-user access helpers, settings/CLI token flows, and matching docs/tests.
