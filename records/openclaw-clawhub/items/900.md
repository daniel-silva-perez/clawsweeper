## Summary

Keep this issue open. The repository-specific cleanup policy for openclaw/clawhub only permits auto-closing pull requests that current main definitely implements, and this item is an issue. Substantively, current main still enforces irreversible account deletion: deleted/deactivated users are blocked during OAuth reauth, active-user gates reject them, token creation requires an active user, and docs/tests assert that deleted accounts cannot be restored by logging in again.

## Best Possible Solution

Keep this issue open and consolidate it with #504, #907, and #917, or create a single dedicated deleted-account lifecycle tracker. Maintainers should choose the intended policy: preserve irreversible deletion and improve the OAuth/CLI/settings error path, or add a deliberate recreate, relink, admin recovery, or GitHub identity unlink flow with tests across auth callback handling, active-user gates, token creation, settings UI, CLI auth, and security docs.
