## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on main. Current main has useful partial scanner transparency, including persisted structured findings, a scan endpoint, docs, and owner/staff moderation evidence, but it still lacks the requested local `clawhub scan <path> --format json` workflow and a complete findings contract with remediation and blocking/informational status.

## Best Possible Solution

Keep this issue open. A complete fix should define one stable findings contract and wire it through API, CLI, UI, docs, and tests: rule/category, severity, file/line range, redacted snippet, remediation guidance, and whether each finding blocks publish/install or is informational. The existing `staticScan.findings`, `/moderation`, and `/scan` surfaces are good foundations, but `clawhub scan <path> --format json` and the full remediation/blocking contract still need implementation.
