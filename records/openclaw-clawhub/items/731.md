## Summary

Keep open. Repository policy for openclaw/clawhub issues says not to auto-close issues in this cleanup pass, and the substantive request is still only partially implemented on main. Current main has structured moderation/static-scan storage, owner/staff moderation evidence, API docs, tests, and some UI display for static findings, but it does not provide the requested local `clawhub scan <path> --format json` workflow or a complete remediation-oriented CLI/API surface with rule, snippet, block status, and fix guidance.

## Best Possible Solution

Keep this issue open. A complete fix should add a stable owner-accessible findings contract and CLI workflow such as `clawhub scan <path> --format json`, exposing rule ID/category, severity, file/line, redacted snippet, remediation guidance, and whether each finding blocks publish/install or is informational, with UI/CLI/docs/tests updated around that contract.
