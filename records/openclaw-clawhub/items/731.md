## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy for this pass only permits auto-closing PRs that are certainly implemented on main. Current main partially addresses scanner transparency with stored structured findings and owner/staff moderation evidence, but it still lacks a local `clawhub scan <path> --format json` workflow and does not expose a complete rule/snippet/remediation/blocking contract through the scan API, CLI, docs, or UI.

## Best Possible Solution

Keep this issue open. A complete fix should define a stable owner-accessible findings contract and a local workflow such as `clawhub scan <path> --format json`, exposing rule ID/category, severity, file/line, redacted snippet, remediation guidance, and whether each finding blocks publish/install or is informational. The API, CLI, UI, docs, and tests should share that contract while preserving public evidence redaction.
