## Summary

This should stay open. Repository policy for openclaw/clawhub keeps issues open, and current main only partially satisfies the request: static scanner findings are now structured in backend/UI/API surfaces, but the contract still lacks remediation guidance, explicit blocking-versus-informational semantics per finding, and a local `clawhub scan <path> --format json` workflow.

## Best Possible Solution

Keep this issue open for a maintainer-owned scanner findings contract: define rule/category, severity, file and line range, redacted evidence policy, remediation guidance, and blocking versus informational semantics, then wire that contract through Convex schema/API, owner/staff redaction, web UI, docs, tests, and a local `clawhub scan <path> --format json` pre-publish workflow.
