## Summary

Keep this open. Current main has structured static-scan findings, but the contract is still only partial and does not expose the full actionable scanner output requested here; #990 is related narrower visibility work, not a replacement. The ClawHub cleanup policy also says issues should remain open for maintainer follow-up.

## What This Changes

Request to expose actionable scanner findings for suspicious skills, including rule/category, severity, file and line range, redacted snippet, remediation guidance, and blocking semantics, with an optional local CLI scan command.

## Best Possible Solution

Define a stable scanner-finding contract with rule/category, severity, file plus line range, redacted evidence policy, remediation guidance, and blocking semantics, then wire it consistently through persistence, API responses, skill UI, CLI publish output or a local scan command, docs, and tests.
