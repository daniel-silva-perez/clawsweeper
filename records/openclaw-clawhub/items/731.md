## Summary

Keep this issue open. Current main has moved from an opaque suspicious flag to structured moderation/static-scan evidence, but it still does not expose the full actionable scanner contract requested here, and the ClawHub review policy keeps issues open for maintainer follow-up.

## What This Changes

Request to expose actionable security scanner findings, including rule/category, severity, file and line range, redacted snippets, remediation guidance, blocking semantics, and optionally a local JSON scan command.

## Best Possible Solution

Keep this issue open for maintainer-scoped scanner transparency work: define a stable finding contract for rule/category, severity, file plus line range, redacted evidence/snippet policy, remediation guidance, and blocking semantics; then wire it through Convex persistence, API redaction, skill UI, CLI publish output or a local scan command, docs, and tests. Treat #990 as a related narrower visibility issue, not a replacement.
