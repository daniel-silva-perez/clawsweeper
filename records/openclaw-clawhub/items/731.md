## Summary

Keep this issue open. Current main has structured static/moderation findings, API evidence redaction, UI rows, and owner rescan surfaces, but it still does not expose the full actionable scanner contract requested here, and the ClawHub cleanup policy says open issues should not be auto-closed.

## What This Changes

Expose actionable scanner findings for flagged skills, including rule/category, severity, file/line evidence, snippets with redaction, remediation guidance, blocking semantics, and optionally a local CLI scan command.

## Best Possible Solution

Keep this issue open for maintainer-scoped scanner transparency work: define a stable finding contract covering rule/category, severity, file plus line range, redacted evidence policy, remediation guidance, and per-finding blocking semantics; then wire it through Convex persistence, API owner/staff/public redaction, skill UI, CLI publish output or a local scan command, docs, and tests. Treat #990 as a related narrower visibility issue, not a replacement.
