## Summary

Keep open. The ClawHub cleanup policy says issues should not be auto-closed in this pass, and current main still only partially addresses the request: structured scan/moderation evidence exists, but there is no local `clawhub scan <path> --format json` workflow and no complete remediation-oriented findings contract with rule, snippet, blocking status, and fix guidance exposed through the CLI/API/UI.

## Best Possible Solution

Keep this issue open. A complete fix should add a stable owner-accessible findings contract plus a local workflow such as `clawhub scan <path> --format json`, exposing rule ID/category, severity, file/line, redacted snippet, remediation guidance, and whether each finding blocks publish/install or is informational. The API, CLI, UI, docs, and tests should all use that same contract.
