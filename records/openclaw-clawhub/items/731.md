## Summary

Keep open. Current main has partial structured scanner evidence, but it still does not satisfy the actionable rule/file/line-range/remediation/blocking contract requested here, and the ClawHub cleanup policy for this run keeps issues open unless they are PRs already implemented on main.

## What This Changes

The issue requests actionable security scanner findings for flagged skills, including rule/category, severity, file and line range, redacted snippet evidence, remediation guidance, blocking semantics, and optionally a local CLI scan command.

## Best Possible Solution

Define a stable scanner-finding contract with rule/category, severity, file plus line range, redacted evidence policy, remediation guidance, and blocking semantics, then wire it consistently through persistence, API responses, skill UI, CLI publish output or a local scan command, docs, and tests.
