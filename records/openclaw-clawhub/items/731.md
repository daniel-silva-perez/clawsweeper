## Summary

Keep open. Current main exposes partial structured static-scan findings, but it still lacks the full actionable scanner contract requested here, and the ClawHub cleanup policy for this run keeps issues open for maintainer follow-up. The related #990 is narrower suspicious-flag visibility work, not a replacement for rule/file/line-range/remediation/blocking semantics.

## What This Changes

The issue asks ClawHub to expose actionable security scanner findings for flagged skills, including rule/category, severity, file and line details, redacted snippets, remediation guidance, blocking semantics, and optionally a local CLI scan command.

## Best Possible Solution

Define a stable scanner-finding contract with rule/category, severity, file plus line range, redacted evidence policy, remediation guidance, and blocking semantics, then wire it consistently through persistence, API responses, skill UI, CLI publish output or a local scan command, docs, and tests.
