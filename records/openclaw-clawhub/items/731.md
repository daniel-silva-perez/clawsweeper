## Summary

Keep open. Current main exposes some structured static-scan findings, but it still does not provide the full actionable scanner contract requested here, and the ClawHub cleanup policy keeps issues open for maintainer follow-up rather than auto-closing them.

## What This Changes

The issue requests human- and machine-readable scanner findings with rule/category, severity, file and line range, matched evidence, remediation guidance, blocking semantics, and optionally a local `clawhub scan <path> --format json` command.

## Best Possible Solution

Define a stable scanner-finding contract with rule/category, severity, file plus line range, redacted evidence policy, remediation guidance, and blocking semantics, then wire it consistently through persistence, API responses, skill UI, CLI publish output or a local scan command, docs, and tests.
