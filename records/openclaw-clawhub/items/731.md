## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy for this review only permits auto-closing PRs that are certainly implemented on main. Current main partially improves scanner transparency with stored findings and owner/staff moderation evidence, but it still does not provide the requested local `clawhub scan <path> --format json` workflow or a complete API/CLI contract for rule, file/line, snippet, remediation, and blocking status.

## Best Possible Solution

Keep this issue open. A complete fix should define one stable findings contract and wire it through API, CLI, UI, docs, and tests: rule/category, severity, file/line, redacted snippet, remediation guidance, and whether each finding blocks publish/install or is informational. The existing persisted `staticScan.findings` and owner/staff moderation evidence endpoint are good foundations, but the local `clawhub scan <path> --format json` workflow and full contract still need implementation.
