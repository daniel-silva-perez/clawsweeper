## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy for this review run only permits auto-closing ClawHub pull requests that are certainly implemented on main. Substantively, current main has partial scanner transparency through persisted static findings, moderation evidence, API docs, and UI guidance, but it still lacks the requested stable per-finding remediation/blocking contract and local `clawhub scan <path> --format json` workflow.

## Best Possible Solution

Keep this issue open. A complete fix should define one stable scanner findings contract and wire it through API, CLI, UI, docs, and tests: rule/category, severity, file/line range, redacted snippet, remediation guidance, and whether each finding blocks publish/install or is informational. Add a local `clawhub scan <path> --format json` workflow for pre-publish remediation, and preserve the current owner/staff-only raw evidence boundary.
