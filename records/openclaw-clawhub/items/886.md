## Summary

Keep this issue open. Current main still treats user-initiated account deletion as terminal during GitHub OAuth, and the related deleted-account login reports remain open; ClawHub cleanup policy also keeps issues open for maintainer handling unless they are PRs already implemented on main.

## What This Changes

The issue asks maintainers to purge, restore, or otherwise recover a deleted GitHub-linked ClawHub account so the same GitHub user can sign in or re-register instead of looping back to Sign In.

## Best Possible Solution

Define one canonical lifecycle policy for deleted GitHub-linked accounts, then implement the approved recovery behavior across the Convex auth callback, account deletion/admin tooling, UI error messaging, docs, and tests for deleted, deactivated, banned, and legacy deletedAt users.
