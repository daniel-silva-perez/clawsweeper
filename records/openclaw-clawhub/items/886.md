## Summary

Keep this open. Current main still treats self-deleted ClawHub accounts as terminal during GitHub OAuth, so the reported authorize-then-return-to-signed-out behavior is still explained by source and tests. This is also an open ClawHub issue, and the cleanup policy for this repository keeps issues open for maintainer handling rather than auto-closing them.

## What This Changes

The issue asks for account recovery or backend cleanup so a user who deleted their ClawHub account can sign in again with the same GitHub account instead of looping back to the signed-out state.

## Best Possible Solution

Define one canonical lifecycle policy for deleted GitHub-linked accounts, then implement the approved recovery behavior across the Convex auth callback, account deletion or admin tooling, UI error messaging, docs, and regression tests for deleted, deactivated, banned, and legacy deletedAt users.
