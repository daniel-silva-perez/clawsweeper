## Summary

Keep this issue open. The repository-specific cleanup policy does not allow auto-closing ClawHub issues, and current main still treats user-initiated deletion as irreversible rather than providing a purge, relink, restore, or same-GitHub re-registration path.

## Best Possible Solution

Keep this open for maintainer support and product-policy handling. The concrete path is either an admin-controlled purge/relink/recovery flow for self-deleted GitHub identities, with tests around Convex Auth bindings and ban invariants, or clearer public/support documentation plus manual production cleanup for affected users if deletion is intentionally permanent.
