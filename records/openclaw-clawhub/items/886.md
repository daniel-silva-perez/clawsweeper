## Summary

Keep this issue open. Current main still implements account deletion as an irreversible deactivation, the OAuth callback rejects deleted/deactivated users, tests assert that behavior, and related open reports show the same user-visible post-deletion GitHub sign-in failure. The repository cleanup policy for openclaw/clawhub also keeps issues open unless a strong non-PR close reason applies.

## Best Possible Solution

Keep this issue open for maintainer support and product-policy handling. The concrete path is either an admin-controlled purge/relink/recovery flow for self-deleted GitHub identities with tests around Convex Auth bindings and ban/deactivation invariants, or clearer support documentation plus manual production cleanup if permanent deletion is intended.
