## Summary

Keep this issue open. Current main still intentionally blocks deleted/deactivated ClawHub accounts during GitHub OAuth instead of purging, restoring, or re-linking the account record; related deleted-account OAuth reports remain open, and the ClawHub cleanup policy keeps issues open unless maintainers handle them manually.

## What This Changes

The issue asks maintainers to resolve a GitHub OAuth sign-in loop after account deletion by purging, restoring, or otherwise re-registering the old GitHub-linked account record.

## Best Possible Solution

Define one canonical account lifecycle policy for deleted GitHub-linked accounts, then implement the approved recovery behavior across the Convex auth callback, account deletion/admin tooling, UI error messaging, docs, and tests for deleted, deactivated, banned, and legacy deletedAt users.
