## Summary

Keep open. Current main still treats self-deleted ClawHub accounts as terminal during GitHub OAuth, so the reported authorize-then-return-to-signed-out loop remains valid. Because this is an open ClawHub issue involving account lifecycle and auth policy, it needs maintainer handling rather than cleanup closure.

## What This Changes

The issue asks ClawHub to recover or purge old account records so a GitHub user can sign in again after self-deleting a ClawHub account.

## Best Possible Solution

Define one canonical lifecycle policy for deleted GitHub-linked accounts, then implement the approved recovery behavior across the Convex auth callback, account deletion or admin tooling, user-facing auth errors, docs, and regression tests for deleted, deactivated, banned, and legacy deletedAt users.
