## Summary

Keep open. This is an open ClawHub issue, and the repository-specific policy says ClawHub issues should remain open rather than auto-closed. Current main still treats self-deletion as irreversible: account deletion sets deactivated/purged state, OAuth completion rejects deactivated users, admin unban explicitly refuses permanently deleted accounts, and docs/tests assert that deleted accounts cannot sign in again.

## Best Possible Solution

Keep this issue open for maintainer support and product-policy handling. A concrete path would be an admin-controlled purge/relink/recovery flow for self-deleted GitHub identities, with tests around Convex Auth bindings and ban invariants; if deletion is intentionally permanent, maintainers should document the support path clearly and handle the affected production accounts manually.

## Risks / Open Questions

- Any purge, relink, or recovery flow is auth-sensitive and must preserve the distinction between self-deleted accounts, moderator bans, and malware auto-bans.
- The named production accounts may still need maintainer support even if the intended product policy remains irreversible deletion.
- Closing as implemented would misstate the current behavior: main improves messaging around deleted accounts, but it does not satisfy the requested ability to re-register with the same GitHub identity.
