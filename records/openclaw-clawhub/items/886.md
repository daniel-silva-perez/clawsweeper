## Summary

Keep open. This is a ClawHub issue, and the provided repository policy says ClawHub issues should remain open rather than being auto-closed. Current main also still does not implement the requested purge/re-register path after self-deletion: deletion marks the account deactivated/purged, OAuth completion rejects deactivated users, tests/docs encode that behavior as irreversible, and admin unban does not restore deactivated accounts.

## Best Possible Solution

Keep this issue open for maintainer support and product-policy handling. The concrete path is either a controlled admin purge/relink/recovery flow for affected GitHub identities, or clearer UX/support documentation if user-initiated deletion is intentionally permanent. Any implementation should preserve ban/deactivation security invariants and handle Convex Auth GitHub bindings deliberately.
