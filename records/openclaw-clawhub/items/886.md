## Summary

Keep this issue open. The repository-specific cleanup policy for openclaw/clawhub keeps issues open unless a strong non-PR close reason applies, and current main still blocks sign-in after user-initiated deletion instead of offering the requested purge, relink, restore, or same-GitHub re-registration path.

## Best Possible Solution

Keep this issue open for maintainer support and product-policy handling. The concrete path is either an admin-controlled purge/relink/recovery flow for self-deleted GitHub identities, with tests around Convex Auth bindings and ban/deactivation invariants, or clearer public/support documentation plus manual production cleanup if permanent deletion is the intended behavior.
