## Summary

Keep this issue open. Current main still treats self-deleted ClawHub accounts as terminal during GitHub OAuth, so the reported authorize-then-return-to-signed-out loop remains explainable from the code and tests. This is also an open ClawHub issue, and the repository cleanup policy keeps issues open for maintainer handling rather than auto-closing them.

## What This Changes

The issue requests recovery, purge, or re-registration support for GitHub-linked ClawHub accounts that cannot sign in after self-deletion.

## Best Possible Solution

Define one canonical account lifecycle policy for deleted GitHub-linked accounts, then implement the approved recovery behavior across the Convex auth callback, account deletion or admin tooling, UI error messaging, docs, and regression tests for deleted, deactivated, banned, and legacy deletedAt users.
