## Summary

Keep this open. Current main still blocks sign-in for deleted/deactivated ClawHub accounts instead of re-registering or purging the GitHub-linked record, and several related deleted-account OAuth reports remain open. The ClawHub cleanup policy also keeps issues open unless a merged PR clearly implements the fix.

## What This Changes

The issue requests support or product handling for users who deleted a ClawHub account and then remain logged out after GitHub OAuth instead of being able to re-register.

## Best Possible Solution

Keep this open and route it to maintainers for an explicit account recovery policy. Decide whether deleted GitHub-linked accounts should be restorable, hard-purgeable by support, or allowed to re-register, then implement that in the Convex Auth callback, account deletion/admin tooling, UI messaging, and tests covering deleted, deactivated, banned, and legacy deletedAt users.
