## Summary

Keep this issue open. Current main still intentionally treats user-initiated account deletion as irreversible: deletion writes deactivated/purged state, OAuth completion blocks deleted/deactivated users, tests and docs assert that deleted accounts cannot sign in again, and related deleted-account login reports remain open. This needs maintainer product/security handling rather than cleanup closure.

## Best Possible Solution

Keep this open and consolidate it with the related deleted-account OAuth loop reports. Maintainers should decide whether ClawHub deletion remains an irreversible block or gains a verified recovery/re-registration path; if recovery is allowed, implement it explicitly in the Convex Auth/account lifecycle and admin support tooling, with tests and docs covering deleted, banned, and legacy soft-deleted users.
