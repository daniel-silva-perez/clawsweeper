## Summary

Keep this open. Current main still intentionally blocks sign-in for deleted/deactivated ClawHub accounts, and the issue discussion plus related open reports show this remains a live account recovery/support problem rather than cleanup. The repo policy also keeps ClawHub issues open unless there is a merged PR implementation to close against.

## What This Changes

The issue asks for a way to purge, restore, or re-register a GitHub-linked ClawHub account after the user deleted it and then entered an OAuth login loop.

## Best Possible Solution

Keep this open and route it to maintainers for an explicit account recovery policy. The best path is to decide whether deleted GitHub-linked accounts should be restorable, hard-purgeable by support, or allowed to re-register, then implement that in the Convex Auth callback, account deletion/admin tooling, and tests covering deleted, banned, and legacy `deletedAt` users. If deletion remains irreversible, the sign-in UI should surface that clearly and support should handle affected existing accounts outside automated cleanup.
