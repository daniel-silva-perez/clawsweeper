## Summary

Keep this open. This is a ClawHub issue, and the supplied repository policy only allows automated closure for ClawHub PRs that are certainly implemented on current main. Source review also supports the reported failure shape: profile lookup returns null for deleted/deactivated users, the profile page renders “User not found.” for that null result, sign-in blocks deleted/deactivated accounts, and the admin restore/reclaim HTTP paths still depend on an active handle lookup. The related account-recovery cluster and PR #1133 remain open, so this needs maintainer support rather than cleanup closure.

Keep this open for maintainer support.
