## Summary

Keep open. This is an open ClawHub issue, and the repository-specific policy only allows auto-closing ClawHub pull requests that are certainly implemented on main. Current main also still matches the reported failure mode: inactive/deleted/deactivated users resolve to null, the profile page renders “User not found,” OAuth blocks inactive account states, and the existing restore/reclaim HTTP helpers require an active public user lookup before they can repair skills or slugs.

## Best Possible Solution

Keep this issue open for maintainer support. A maintainer should inspect production records for samiralibabic, restore or relink the rightful active user/auth/publisher state if appropriate, confirm ownership of the listed skills, clean up any duplicate or orphaned entries, and decide whether to land or supersede #1133 for moderation-hold false-positive recovery.
