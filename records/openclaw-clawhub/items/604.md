## Summary

Keep open. This is an open ClawHub issue, and the provided repository policy only permits auto-closing ClawHub PRs that are certainly implemented on main. Current main also still matches the reported account recovery shape: inactive/deleted users resolve to null for public profiles, deleted/deactivated sign-in is blocked, and the restore/reclaim HTTP helpers require an active public user lookup before they can repair skills or slugs.

## Best Possible Solution

Keep this open for maintainer support. A maintainer should inspect production records for samiralibabic, restore or relink the rightful active user/auth/publisher state if appropriate, confirm ownership of the listed skills, clean up duplicate or orphaned entries, and decide whether #1133 or an equivalent admin recovery path should land for moderation-hold false positives.
