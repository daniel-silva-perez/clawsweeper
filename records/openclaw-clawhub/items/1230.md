## Summary

Keep open because current main does not definitely implement this PR's intended change. The PR still represents useful work for the open theme-toggle bug, but it is stale against the current theme/header implementation and now needs maintainer review rather than auto-close.

## What This Changes

The PR changes the header theme ToggleGroup, runtime theme storage fallback, and regression tests to restore system/light/dark selection by click and keyboard while ignoring the legacy `clawdhub-theme` key.

## Best Possible Solution

The final implementation should make the header theme selector work consistently for pointer, keyboard, cycle-button, runtime storage, and pre-hydration first paint, with an explicit legacy-key policy and regression coverage for each supported interaction path.
