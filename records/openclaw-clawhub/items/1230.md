## Summary

Current main does not definitely implement this PR's intended change. The desktop header still uses the root ToggleGroup onValueChange path without the PR's per-item pointer handling/value-change coordination, the legacy clawdhub-theme fallback remains active in runtime code and the early boot script, and tests still assert the legacy fallback behavior. Under the ClawHub policy, this open PR should stay open unless main has clearly shipped the same intended behavior.

## Best Possible Solution

Keep this PR open for maintainer review, or supersede it with a merged implementation that restores desktop theme switching across pointer and keyboard/value-change paths and makes a consistent product decision about the legacy clawdhub-theme fallback across src/components/Header.tsx, src/lib/theme.ts, src/routes/__root.tsx, and tests. Because the related issue remains open and this PR is an implementation candidate, the cleanup path is to land or explicitly supersede the implementation before closing it.
