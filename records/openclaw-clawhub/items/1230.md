## Summary

Current main does not definitely implement this PR's intended change. The header currently has a single light/dark cycle button rather than direct system/light/dark selection, the legacy `clawdhub-theme` fallback still exists in both runtime and pre-hydration theme paths, and the paired #1228 remains open. #1234 is a related alternative, but it was closed unmerged, so it does not supersede this PR.

## Best Possible Solution

Keep this PR open for maintainer review, rebase, or explicit supersession. The maintainer path should decide whether the header should restore direct system/light/dark selection or keep the current cycle button, then land one focused fix that handles both runtime and pre-hydration legacy storage behavior, adds regression coverage for the chosen interaction model, and closes #1228 after merge.
