## Summary

Current main does not definitely implement this PR's intended change. The header still uses the root Radix ToggleGroup value-change path without the PR's pointer/value-change coordination, legacy `clawdhub-theme` fallback remains active in runtime and boot-time theme selection, and tests still expect that legacy fallback instead of the PR's first-visit `system` default. The related same-author issue remains open, so this PR should stay available for maintainer review or explicit supersession.

## Best Possible Solution

Keep this PR open for maintainer review, or explicitly supersede it with a merged implementation that restores the desktop header theme controls across pointer and keyboard/value-change paths and makes a consistent product decision about the `clawdhub-theme` legacy fallback in `Header.tsx`, `theme.ts`, the root boot script, and tests. Because the related same-author issue remains open, the cleanup path is to land, revise, or intentionally close both halves with maintainer judgment.
