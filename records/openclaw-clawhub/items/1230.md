## Summary

Keep this PR open. Current main does not clearly implement the PR’s intended change: the header still lacks the PR’s per-item pointer handlers and direct click/keyboard regression coverage, the runtime legacy `clawdhub-theme` fallback remains, and the root pre-hydration script still honors the same legacy key. The PR is also stale against the rewritten theme/header surface and has a concrete rebase concern around keeping runtime and first-paint theme resolution consistent.

## What This Changes

The PR changes the header theme ToggleGroup, runtime theme storage fallback, and regression tests to restore system/light/dark selection by click and keyboard while ignoring the legacy `clawdhub-theme` key.

## Best Possible Solution

The final implementation should make the header theme selector work consistently for pointer, keyboard, cycle-button, runtime storage, and pre-hydration first paint, with an explicit legacy-key policy and regression coverage for each supported interaction path.
