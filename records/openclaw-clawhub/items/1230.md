## Summary

Current main does not already implement this branch’s intended changes: the header segmented theme control still differs from the PR, the legacy theme key is still honored, and the linked bug remains open while the alternative PR is closed unmerged. This should stay open for maintainer judgment because the branch is stale against rewritten theme/header code and the legacy-storage policy needs to be applied consistently across hydration paths.

## What This Changes

The branch updates the header theme ToggleGroup, runtime theme storage fallback, and header/theme regression tests to restore system/light/dark switching by pointer and keyboard while ignoring the legacy `clawdhub-theme` key.

## Best Possible Solution

The final implementation should make the header theme selector work consistently for pointer, keyboard, cycle-button, runtime storage, and pre-hydration first paint, with an explicit legacy-key policy and regression coverage for each supported interaction path.
