## Summary

Keep this issue open. It is an open ClawHub issue, and the repository-specific cleanup policy says ClawHub issues should remain open unless maintainers handle them manually. Current main still treats user-initiated account deletion as irreversible: deletion writes permanent deactivation/purge state, OAuth reauthorization rejects deleted/deactivated users, admin unban explicitly refuses permanently deleted accounts, and docs/tests codify that deleted accounts cannot be restored by logging in again. The frontend now surfaces clearer auth errors, but that does not satisfy the requested same-GitHub re-registration or admin reset path.

## Best Possible Solution

Keep this issue open for maintainer product/support triage. Maintainers should decide whether user-initiated deletion is permanently irreversible or whether same-GitHub re-registration should be supported. If recovery is intended, add a backend path that safely unlinks or supersedes the old Convex Auth identity for self-deleted users only, preserves ban and ownership protections, revokes old tokens, and covers web OAuth plus CLI/API-token flows with tests. If deletion remains irreversible, improve the login/support UX so affected users receive a clear deleted-account outcome and know whether manual cleanup is possible.
