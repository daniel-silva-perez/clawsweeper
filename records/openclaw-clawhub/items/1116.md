## Summary

Keep open. Current main still treats user deletion as an irreversible account state: OAuth reauthorization is rejected for deleted/deactivated users, account deletion keeps an inactive purged user row, and tests/docs assert that deleted accounts cannot be restored by logging in again. The report is also clustered with multiple open user reports, and the ClawHub cleanup policy for this repository keeps issues open for maintainer resolution rather than auto-closing them.

## What This Changes

The issue asks ClawHub to allow or repair sign-in after a user deletes a ClawHub account and later re-authorizes the same GitHub identity.

## Best Possible Solution

Maintainers should define and document the deleted-account lifecycle, then implement the chosen path: either a controlled re-registration/admin reset flow, or an explicit irreversible-deletion error and support workflow. The implementation should preserve token revocation, profile/contact purging, published-skill behavior, and ban/autoban safeguards with regression coverage for active, deleted, legacy-deleted, banned, and reauthorized GitHub identities.
