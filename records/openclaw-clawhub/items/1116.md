## Summary

Keep this issue open. The ClawHub cleanup policy does not auto-close issues, and current main still intentionally treats user-initiated account deletion as permanent deactivation: deletion keeps a deactivated user row, OAuth completion rejects deleted/deactivated users, active-user helpers hide those rows, and tests/docs assert that deleted accounts cannot be restored by logging in again. The reporter and commenters are describing a real account-lifecycle/support decision rather than a cleanup-close candidate.

## Best Possible Solution

Keep this issue open and consolidate it with the related deleted-account sign-in reports. Maintainers should decide whether deleted GitHub identities should support self-service re-registration, admin-only hard purge/reset, or remain permanently blocked with clearer UX and support guidance, then implement that policy in the Convex auth/account lifecycle path with tests and docs.
