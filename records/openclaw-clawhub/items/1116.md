## Summary

Keep this issue open. The repository-specific cleanup policy says ClawHub issues are not auto-close candidates, and current main still treats user-initiated deletion as irreversible rather than supporting same-GitHub re-registration, unlink/relink, or admin reset.

## Best Possible Solution

Keep this open for maintainer product and support triage. Decide whether account deletion is permanently irreversible or whether same-GitHub re-registration should be supported. If recovery is intended, add a backend flow that safely unlinks or supersedes the old Convex Auth identity for self-deleted users only, revokes old tokens, preserves ownership/audit protections, and covers web OAuth plus CLI/API-token flows with tests. If deletion remains irreversible, improve login/support docs and hosted UX so affected users know whether manual cleanup is possible.

## Risks / Open Questions

- A recovery path must distinguish self-deleted accounts from banned or moderator-deactivated accounts so OAuth rebinding cannot bypass moderation.
- Any reset or re-registration flow needs to preserve publisher handles, skill ownership, audit history, API-token revocation, and CLI/API-token invariants.
- If irreversible deletion remains intended, hosted login and support documentation still need an actionable outcome for affected users rather than only a generic blocked-sign-in state.
