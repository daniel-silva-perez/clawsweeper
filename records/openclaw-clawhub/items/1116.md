## Summary

Keep this issue open. Repository policy for openclaw/clawhub only allows auto-closing PRs that are certainly implemented on current main, and this item is an issue. Current main also still intentionally blocks deleted/deactivated account reauthorization instead of providing the requested same-GitHub re-registration, reset, unlink, or restore flow.

## Best Possible Solution

Keep this open for maintainer product and support triage. Maintainers should decide whether user-initiated deletion is permanently irreversible or whether same-GitHub re-registration/recovery should be supported. If recovery is intended, add a backend flow that safely unlinks or supersedes only self-deleted Convex Auth identities, revokes old sessions/tokens, preserves ownership/audit protections, and covers OAuth plus CLI/API-token paths with tests. If deletion remains irreversible, improve hosted login/support UX and public docs so affected users get an actionable deleted-account message and support path.
