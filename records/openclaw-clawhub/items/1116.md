## Summary

Keep this issue open. Current main still blocks OAuth completion for deleted/deactivated ClawHub users, tests assert that deleted-account reauth is rejected, and the repo policy for ClawHub issues does not allow auto-closing non-PRs. The remaining work is a maintainer account-lifecycle decision: allow self-service re-registration, provide an admin reset/purge path, or keep deletion irreversible with clearer UX/support guidance.

## What This Changes

The issue requests that a GitHub identity previously tied to a deleted ClawHub account be able to authorize and create or restore a usable account instead of looping or remaining unauthorized.

## Best Possible Solution

Keep this open and consolidate it with the related deleted-account sign-in reports. Maintainers should decide the intended policy for deleted GitHub identities, then implement that policy in the Convex auth/account lifecycle path with tests, docs, and any needed support/admin workflow while preserving ban/autoban protections.
