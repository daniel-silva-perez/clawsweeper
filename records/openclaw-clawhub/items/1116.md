## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open, and current main still treats user-initiated deletion as irreversible: OAuth completion rejects deleted/deactivated users instead of providing the requested same-GitHub re-registration, reset, unlink, or restore flow. Current main appears to surface a clearer deleted-account error, but that does not solve the account recovery problem reported by multiple users.

## Best Possible Solution

Keep this open for maintainer product and support triage. Decide whether deletion is permanently irreversible or whether same-GitHub re-registration should be supported. If recovery is intended, add a backend flow that safely unlinks or supersedes only self-deleted Convex Auth identities, revokes old tokens, preserves ownership/audit protections, and covers OAuth plus CLI/API-token paths with tests. If deletion remains irreversible, improve hosted login/support UX and docs so affected users know whether manual cleanup is possible.

## Risks / Open Questions

- If irreversible deletion remains the intended product policy, hosted login and support documentation still need an actionable path for affected users instead of leaving them to infer the account state from a sign-in failure.
- Multiple independent reports in the issue discussion and related items suggest this is not a one-off user support case.
