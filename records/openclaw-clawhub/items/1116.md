## Summary

Keep this issue open. Under the supplied openclaw/clawhub cleanup policy, open issues are not auto-close candidates, and current main still treats user-initiated account deletion as irreversible rather than supporting same-GitHub re-registration or an admin reset path. The frontend now maps deleted-account auth failures to a clearer message, but the requested recovery flow remains unimplemented.

## Best Possible Solution

Keep this open for maintainer product and support triage. Maintainers should decide whether account deletion is permanently irreversible or whether same-GitHub re-registration should be supported. If recovery is intended, add a backend flow that safely unlinks or supersedes the old Convex Auth identity for self-deleted users only, preserves moderation and ownership protections, revokes old tokens, and covers web OAuth plus CLI/API-token flows with tests. If deletion remains irreversible, improve login/support UX so affected users get a clear deleted-account outcome and know whether manual cleanup is possible.
