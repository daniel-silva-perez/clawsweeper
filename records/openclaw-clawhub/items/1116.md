## Summary

Keep this issue open. Current main still treats deleted ClawHub accounts as inactive identities that cannot self-service re-register through GitHub OAuth, and repository policy for ClawHub only permits automated cleanup closure for PRs that are certainly implemented on main. The issue also has multiple confirming reports and related open issues, so the remaining work is a real account-lifecycle/support policy decision rather than a stale or implemented cleanup case.

## What This Changes

The issue requests that a user who deleted their ClawHub account be able to re-authorize with the same GitHub account, or have the stale/soft-deleted backend state reset so login succeeds.

## Best Possible Solution

Keep this issue open and consolidate it with the related deleted-account sign-in reports. Maintainers should decide whether deleted GitHub identities should support self-service re-registration, admin-only hard purge/reset, or permanent blocking with clearer UX/support guidance, then implement that policy in the Convex auth/account lifecycle path with tests and docs.
