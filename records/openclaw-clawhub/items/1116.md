## Summary

Keep this issue open. ClawHub cleanup policy keeps issues open unless they are PRs that are certainly implemented on current main, and the source review also shows the requested recovery/re-registration behavior is not implemented: current main intentionally treats user-initiated deletion as permanent deactivation, rejects OAuth completion for deactivated users, and documents deletion as irreversible. Related open reports describe the same post-deletion GitHub sign-in loop.

## Best Possible Solution

Keep this issue open and consolidate it with the related deleted-account sign-in reports. Maintainers should decide whether deleted GitHub identities should support self-service re-registration, admin-only hard purge/reset, or remain permanently blocked with clearer UX and support guidance, then implement that decision in the Convex auth/account lifecycle path with tests.
