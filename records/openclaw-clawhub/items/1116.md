## Summary

Keep open. This is an open ClawHub issue, and the repository policy says ClawHub issues should remain open unless maintainers handle them manually. Current main still intentionally treats user-initiated deletion as irreversible: deletion deactivates/purges the user, OAuth reauth blocks deleted/deactivated users, and docs/tests state deleted accounts cannot be restored by logging in again. The frontend now surfaces a clearer deleted-account message, but that does not satisfy the requested same-GitHub re-registration or admin reset path.

## Best Possible Solution

Keep this issue open for maintainer product/support triage. Maintainers should decide whether user-initiated deletion is truly irreversible or whether same-GitHub re-registration should be supported. If recovery is intended, add a backend path that safely unlinks or supersedes the old Convex Auth identity for self-deleted users only, preserves ban and ownership protections, revokes old tokens, and covers web OAuth plus CLI/API-token flows with tests. If irreversible deletion remains policy, improve the OAuth UI and support path so affected users get a durable deleted-account message and know whether manual cleanup is possible.

## Risks / Open Questions
