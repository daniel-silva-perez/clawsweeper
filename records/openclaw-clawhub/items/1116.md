## Summary

Keeping this open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on main. Current main also still intentionally blocks GitHub OAuth sign-in for deleted/deactivated users rather than providing the requested re-registration, reset, unlink, or restore path.

## Best Possible Solution

Keep this open for maintainer product and support triage. Decide whether user-initiated account deletion is permanently irreversible or whether same-GitHub re-registration should be supported. If recovery is intended, add a backend flow that safely unlinks or supersedes the old Convex Auth identity for self-deleted users only, revokes old tokens, preserves ownership/audit protections, and covers OAuth plus CLI/API-token flows with tests. If deletion remains irreversible, improve hosted login and support documentation so affected users know whether manual cleanup is possible.
