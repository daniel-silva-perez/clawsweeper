## Summary

Keep open. Current main still treats user-initiated deletion as irreversible: the OAuth callback rejects deleted/deactivated users, account deletion keeps an inactive user row, and tests/docs assert that logging in again must not restore the account. The report is also clustered with multiple open user reports, and the repository cleanup policy for ClawHub issues does not auto-close issues.

## What This Changes

The issue requests a way for a user who deleted their ClawHub account to re-authorize with the same GitHub identity and get a usable account/session again.

## Best Possible Solution

Maintainers should define and document the deleted-account lifecycle, then either implement a controlled re-registration/admin reset flow or keep deletion irreversible while showing a clear post-OAuth error and support path. Any change should preserve profile/contact purging, token revocation, and ban/autoban protections with regression coverage for active, deleted, legacy-deleted, banned, and reauthorized GitHub identities.
