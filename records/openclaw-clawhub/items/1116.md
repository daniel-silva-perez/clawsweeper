## Summary

Keep this issue open. It is an open ClawHub issue, and the repository-specific cleanup policy only allows auto-closing PRs that are certainly implemented on main. Current main still treats user-initiated deletion as irreversible: deletion sets deactivated/purged state, OAuth reauthorization rejects deleted or deactivated users, and the admin unban path does not restore permanently deleted accounts. The frontend now surfaces clearer auth errors, but it does not implement the requested same-GitHub re-registration or support reset path.

## Best Possible Solution

Keep this open for maintainer product and support triage. Maintainers should decide whether user-initiated deletion is permanently irreversible or whether same-GitHub re-registration should be supported. If recovery is intended, add a backend path that safely unlinks or supersedes the old Convex Auth identity for self-deleted users only, preserves ban and ownership protections, revokes old tokens, and covers web OAuth plus CLI/API-token flows with tests. If deletion remains irreversible, improve the login/support UX so affected users get a clear deleted-account outcome and know whether manual cleanup is possible.
