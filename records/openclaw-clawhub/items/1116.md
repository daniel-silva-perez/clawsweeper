## Summary

Keep open. Current main still treats deleted ClawHub accounts as permanently deactivated and blocks the next GitHub OAuth sign-in instead of recreating or relinking the account. The issue is valid, clustered with several related reports, and ClawHub cleanup policy keeps issues open for maintainer resolution rather than auto-closing them.

## What This Changes

The issue requests a way for a user who deleted their ClawHub account to sign in again with the same GitHub identity, or for maintainers to purge/reset the old account state.

## Best Possible Solution

Maintainers should choose and document the intended deleted-account lifecycle: either support a controlled re-registration/admin reset flow, or keep deletion irreversible while surfacing a clear post-OAuth error and support path. Any implementation should preserve token revocation, profile/contact purging, and ban/autoban protections, with regression coverage for active, deleted, legacy-deleted, banned, and reauthorized GitHub identities.
