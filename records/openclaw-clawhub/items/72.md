Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing pull requests that current main certainly implements. The issue reports that Settings-created `clh_` API tokens fail `clawhub login --token --no-browser` with `Unauthorized`, with discussion pointing at registry host redirects as a possible cause.

If it still fails, the code change should target the confirmed failing layer with focused coverage around registry discovery/defaults, redirect/auth behavior, `/api/v1/whoami` token lookup, browser callback registry propagation, and user/publisher state.
