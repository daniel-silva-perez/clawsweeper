## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing PRs that current main certainly implements. The current source still verifies token login through registry discovery/defaults and `/api/v1/whoami`, while the original `www` workaround no longer matches the live redirect direction observed during review, so this needs maintainer production verification with a fresh Settings-created token.

## What This Changes

The issue reports that fresh Settings-created `clh_` API tokens fail `clawhub login --token <token> --no-browser` with `Unauthorized`, with discussion pointing to registry host redirects stripping the Authorization header.

## Best Possible Solution

End state should be a verified production auth path where a fresh Settings-created `clh_` token works with the default CLI registry for token login, `whoami`, and a minimal publish or sync smoke path. If it still fails, the fix should target the specific failing layer with focused tests around registry discovery/defaults, redirect/auth behavior, `/api/v1/whoami` token lookup, browser callback registry propagation, and publisher/user state.
