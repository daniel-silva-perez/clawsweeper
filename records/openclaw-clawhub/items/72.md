## Summary

Keep open. This is an issue, and the openclaw/clawhub cleanup policy only permits auto-closing PRs that current main certainly implements. Substantively, current main still routes token login through the default registry and `/api/v1/whoami`, the original registry-host diagnosis has changed in production, and the issue still needs maintainer verification with a fresh Settings-created token before it can be closed.

## What This Changes

The issue reports that API tokens created in the ClawHub web settings fail `clawhub login --token <token> --no-browser` with `Unauthorized`, with discussion pointing at registry host redirects stripping bearer auth and also mentioning browser callback failures.

## Best Possible Solution

End state should be a verified production auth path where a fresh Settings-created `clh_` token works with the default CLI registry for login, `whoami`, and a minimal publish/sync smoke path. If it still fails, the fix should target the specific failing layer with focused tests around registry discovery, redirect/auth preservation, `/api/v1/whoami` token lookup, browser callback registry propagation, and publisher/user state.
