## Summary

Keep this issue open. The repository-specific cleanup policy for openclaw/clawhub says this workflow should keep issues open, and current main still does not prove the original production `clawhub login --token` failure is fixed end to end for fresh UI-created tokens. Source review shows token verification plumbing exists, but the CLI and well-known discovery still default to `https://clawhub.ai`, while the issue discussion identified `https://www.clawhub.ai` as the working workaround and the related fix PRs were closed unmerged.
