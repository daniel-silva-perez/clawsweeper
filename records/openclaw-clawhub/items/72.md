## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing PRs that current main certainly implements. Substantively, current main still verifies `clawhub login --token` through the default registry and `/api/v1/whoami`, while the original `www` workaround no longer matches current production redirects, so this needs maintainer production verification with a fresh Settings-created token rather than cleanup closure.

## What This Changes

Fresh ClawHub Settings API tokens should authenticate `clawhub login --token <token> --no-browser` without returning Unauthorized.

## Best Possible Solution

End state should be a verified production auth path where a fresh Settings-created `clh_` token works with the default CLI registry for token login, `whoami`, and a minimal publish or sync smoke path. If it still fails, the fix should target the specific failing layer with focused tests around registry discovery/defaults, redirect/auth preservation, `/api/v1/whoami` token lookup, browser callback registry propagation, and publisher/user state.
