## Summary

Keep open. Repository policy for openclaw/clawhub only allows auto-closing pull requests that are certainly implemented on main, and this is an issue. Substantively, the live voc-amazon-reviews listing is still flagged as suspicious: the current version scan reports static and VirusTotal clean, but LLM suspicious because the skill uses VOC_API_KEY while its SKILL.md metadata does not declare that env var.

## Best Possible Solution

Keep the issue open for maintainer/moderator handling. The concrete path is for the publisher to republish with metadata.openclaw.requires.env including VOC_API_KEY and primaryEnv set to VOC_API_KEY, then rescan; or for staff to review the current version and apply a manual clean override if the metadata mismatch is acceptable.
