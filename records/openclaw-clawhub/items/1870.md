## Summary

Keep this draft PR open. Current main does not implement the requested admin-help hiding: the CLI still exposes `ban-user` and `set-role` as ordinary visible commands, and `/api/v1/whoami` plus the published CLI schemas do not return a user role. Also, the PR author is a repo member, so this workflow should not auto-close it even if it looked resolved.

## Best Possible Solution

Keep this PR open for normal maintainer review. The core implementation direction appears relevant: add role to the whoami API/schema, hide admin CLI commands only for help rendering when the stored token is not admin, and keep execution-time authorization on the existing server endpoints. Review should also decide whether to split the unrelated dashboard/dev-workflow changes and scrutinize the seeded token fixture.
