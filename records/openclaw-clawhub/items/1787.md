## Summary

Keep this issue open. It is an open ClawHub issue, and the repository policy only permits auto-closing pull requests that are certainly implemented on current main. Substantively, current main still hard-blocks non-interactive installs for suspicious skills without `--force`, `trustedPublisher` only bypasses pending-scan auto-hide for skills rather than suspicious verdicts, and the live API still reports multiple listed adlai88 skills as suspicious.

## Best Possible Solution

Keep this open for maintainer moderation review. Staff can use the existing audited per-skill manual override path for verified false positives now. If maintainers want durable publisher-level clearance for legitimate trading SDK skills, define explicit reviewed-publisher semantics with audit logging, tests, and security limits rather than treating the current pending-scan `trustedPublisher` flag as sufficient.
