Keep this issue open. Current main still formats an unmatched registry latest as `UPDATE <latest> -> <semver-bumped publish target>`, while `inspect` prints the actual registry latest version. This remains a narrow CLI reporting bug and is suitable for a focused fix PR.

Keep this issue open and fix the CLI reporting path. `sync` should remain an upload-local-skills command, but when `match` is null and `latestVersion` exists, the report should distinguish the actual registry latest version from the proposed publish version, with focused tests covering that case.
