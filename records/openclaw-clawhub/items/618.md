## Summary

Keep open. Current main still has the reported CLI presentation bug: unmatched local skill content is classified as an update and displayed with a semver-bumped publish target, while `inspect` reports the actual registry latest. The related dashboard drift issue is a separate UI path and does not supersede this CLI reporting issue.

## What This Changes

The issue asks for `clawhub sync` to display actual registry latest versions instead of showing proposed semver-bumped upload versions as if they already exist.

## Best Possible Solution

Update the CLI sync report and interactive selection label so unmatched local content is described as local changes relative to the registry latest, with any proposed publish version clearly labeled as upload metadata rather than the registry latest. Add regression coverage for the `latestVersion` present plus `match: null` case and leave the separate dashboard drift warning to #1180.

Keep the existing upload workflow and semver bump behavior intact, but change the planning report and interactive selection label so `latestVersion` with `match: null` does not present the bumped publish target as an existing registry latest version.
