## Clownfish Work Prompt

Fix the `clawhub sync` version reporting bug from openclaw/clawhub#618. Current main classifies a local fingerprint mismatch as `status: "update"`, then `formatActionableStatus` displays `UPDATE <registry latest> -> <semver-bumped publish target>`, which users read as an available registry update version. Keep the publish behavior and `resolvePublishMeta` bump logic intact, but change the human-facing sync report and multiselect label so the registry latest version and proposed publish version are not conflated. Add or update tests in `packages/clawhub/src/cli/commands/sync.test.ts` for a candidate with `match: null` and `latestVersion: { version: "1.0.0" }`, asserting the output does not advertise `1.0.1` as an existing registry version and still publishes with the bumped target when selected. Likely implementation is in `packages/clawhub/src/cli/commands/syncHelpers.ts`; avoid broad CLI refactors, registry API changes, or changes to update/install commands. No changelog is expected unless the repo's CLI release workflow requires one for user-facing CLI output fixes.
