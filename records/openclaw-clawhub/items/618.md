## Summary

Keep this issue open. The ClawHub review policy supplied for this repository only permits auto-closing pull requests that are certainly implemented on main, and this item is an issue. The current CLI code still appears to have the reported misleading `clawhub sync` display: sync reads the registry latest version, classifies unmatched local fingerprints as updates, then prints a semver-bumped publish target as the right-hand update version, while `inspect` prints the actual latest registry version.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely fix is to change `clawhub sync` output so it clearly separates the actual registry latest version from the proposed publish version, or to adjust update classification when installed content is already at the registry latest. Add a focused CLI test covering the `latest -> bumped publish target` display case.
