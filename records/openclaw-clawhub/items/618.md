## Clownfish Work Prompt

Fix the CLI reporting bug where `clawhub sync` displays a semver-bumped proposed publish version as though it were an available registry latest version. Keep `sync` as an upload-local-skills command and preserve existing publish version bump behavior in `resolvePublishMeta`. Update the actionable report path so that when a candidate has `status: "update"`, `matchVersion` is null, and `latestVersion` exists, the output distinguishes the actual registry latest version from the proposed publish version. Add focused tests for that case, including the prompt/actionable line formatting and the command output path. Do not change the registry resolve API contract, package publishing behavior, auth, telemetry, or unrelated dashboard deployment-drift code. Mention in the PR that #1180 is unrelated dashboard drift, not the CLI sync report.
