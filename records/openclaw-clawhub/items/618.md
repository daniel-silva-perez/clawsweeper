## Summary

Keep open. Current main still has the reported CLI display bug: `sync` preserves the real registry latest version internally, but formats unmatched local content as an `UPDATE` from latest to the next semver bump, which can read like a non-existent registry version. The ClawHub cleanup policy also keeps issues open unless a maintainer handles them.

## What This Changes

The issue asks for `clawhub sync` to show actual latest registry versions for installed skills instead of higher non-existent versions.

## Best Possible Solution

Update the CLI sync report and interactive upload-selection label so unmatched local content is described as local changes relative to the actual registry latest, with the bumped value clearly labeled only as the proposed upload version. Add regression coverage for the `latestVersion` present plus `match: null` case, and leave the dashboard drift warning to #1180.
