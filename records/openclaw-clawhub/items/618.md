## Summary

Keep this issue open. The ClawHub cleanup policy says issues should stay open, and current main still shows the same plausible mismatch: `sync` formats update rows with the next publish version while `inspect` prints the registry latest version.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely fix is to change sync output so it distinguishes registry latest from proposed publish version, for example by wording update rows as a local-change publish plan, or to adjust classification if unchanged installed skills are being detected as updates. Add a focused CLI test for the reported `latest -> bumped target` display case.
