## Summary

Keep this ClawHub issue open. Current main still formats `clawhub sync` update rows as the next publish target version, while `clawhub inspect` prints the latest registry version, so the reported mismatch remains plausible. The supplied ClawHub cleanup policy also says this workflow should keep issues open and only auto-close PRs that are certainly implemented on main.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely fix is to change sync output wording so update rows clearly distinguish `registry latest` from `proposed publish version`, or to adjust sync/update classification if installed unmodified skills are being misdetected as changed. If maintainers decide the behavior is intentional, CLI docs and labels should make clear that `sync` is a publish workflow, not an available-update report.
