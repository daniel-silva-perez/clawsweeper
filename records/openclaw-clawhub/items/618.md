## Summary

Keep this open. This is a ClawHub issue, and the supplied repository policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main also still appears to match the reported CLI behavior: sync treats an unmatched local fingerprint as an update, then displays a semver-bumped publish target rather than an existing registry version, while inspect prints the registry latest version.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely fix is to change clawhub sync output so it separates actual registry latest from proposed publish version, or to adjust update classification when installed content is already at the registry latest but the fingerprint differs. Add a focused CLI test for the misleading latest -> bumped publish target display case.
