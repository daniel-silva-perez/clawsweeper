## Summary

Keep this issue open. ClawHub cleanup policy only permits auto-closing PRs that are definitely implemented on main, not issues. Independently, current main still has the reported CLI wording problem: sync formats an update as registry latest version -> bumped publish target, while inspect prints the registry latest version directly.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely product/code fix is to change sync output so it clearly separates the actual registry latest version from the proposed publish version, or to adjust update classification if unchanged installed skills are being marked as local changes. Add a focused CLI test for the `latest -> bumped target` display case.
