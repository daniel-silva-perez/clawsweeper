## Summary

Keep open. Current main still has the reported CLI presentation bug: when local skill content does not match any registry version but a registry latest exists, `clawhub sync` reports a semver-bumped publish target as an `UPDATE` version. The related dashboard drift report is a separate frontend warning path and does not supersede this CLI sync issue.

## What This Changes

The issue requests that `clawhub sync` show actual registry latest versions for installed skills instead of displaying bumped publish targets as available updates.

## Best Possible Solution

Update the sync planning output and interactive upload-selection label so unmatched local content is described as local changes relative to the registry latest, while any bumped version is clearly labeled as the proposed upload version. Add regression coverage for `latestVersion` present with `match: null` and keep the dashboard drift warning tracked separately in #1180.
