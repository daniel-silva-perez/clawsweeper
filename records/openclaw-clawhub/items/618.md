## Summary

Keep this open. This is an issue, and the supplied ClawHub review policy only permits auto-closing pull requests that are certainly implemented on current main. The current CLI code also still appears to match the reported behavior: `sync` classifies unmatched local fingerprints as updates, then displays a semver-bumped publish target as `UPDATE latest -> next`, while `inspect` prints the actual registry latest version.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely fix is to change `clawhub sync` output so it clearly separates the actual registry latest version from the proposed publish version, or to adjust update classification when installed content is already at the registry latest but the fingerprint does not match. Add a focused CLI test covering the misleading `latest -> bumped publish target` display case.
