## Summary

Current main still appears to have the reported CLI presentation bug: the sync path treats unmatched local content as an update and formats the right-hand version as the proposed publish bump, while inspect displays the actual registry latest. Because this is an open issue in `openclaw/clawhub`, repo policy also says not to auto-close it.

## What This Changes

The issue asks for `clawhub sync` to report actual registry latest versions for installed skills instead of showing semver-bumped versions that do not exist in the registry.

## Best Possible Solution

The CLI should present unmatched local content as local changes relative to the registry latest and label any bumped value as a proposed upload version, with regression tests for the `latestVersion` present plus `match: null` case. The dashboard drift warning should remain tracked separately in #1180.
