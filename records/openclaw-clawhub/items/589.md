## Summary

Keep open. Current main still leaves the reported install failure reachable: anonymous downloads consume an IP or `unknown` download bucket, while CLI install only authenticates when a token is already saved. The discussion’s `clawhub login`, `clawhub login --token ...`, `pnpm dlx`, VPN-node, and skills.sh workarounds are useful, but the remaining fix spans production IP attribution, rate-limit policy, and install UX, and the ClawHub cleanup policy keeps issues open.

## What This Changes

The issue reports that `npx clawhub@latest install sonoscli` fails with `Rate limit exceeded` across several networks and asks for the install/rate-limit behavior to be fixed or made clearer.

## Best Possible Solution

Keep this issue open while maintainers consolidate it with #1410 for anonymous-install attribution and #349 for rate-limit policy/visibility. The desired end state is reliable anonymous bucket attribution in production, a clear product decision on whether installs should prompt for or require login before rate-limited downloads, and install-specific 429 guidance that points users to `clawhub login` or `clawhub login --token ...` when authentication would move them to a user bucket.
