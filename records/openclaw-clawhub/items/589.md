## Summary

Keep open. This is an open ClawHub issue, and the supplied repository policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main has meaningful rate-limit improvements, but the reported install UX is still not fully resolved: anonymous downloads remain IP-limited, missing client IPs still fall into an `unknown` bucket, `clawhub install` only sends auth when a saved token exists, and the web UI still advertises `npx clawhub@latest install ...`. The discussion also shows users relying on workarounds such as `clawhub login`, `clawhub login --token`, `pnpm dlx`, VPN/IP changes, and skills.sh, while related open #1410 tracks the narrower anonymous-install false-positive 429 path.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to either manually consolidate it into open #1410 if maintainers want one canonical anonymous-install 429 thread, or make a concrete install UX fix: prompt or require login before rate-limited downloads, surface a specific `clawhub login` / `clawhub login --token` hint on 429, and update web install guidance if `npx clawhub@latest install ...` remains prone to anonymous rate-limit confusion.
