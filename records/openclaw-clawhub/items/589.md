## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy for this repository keeps ClawHub issues open unless a maintainer handles them directly. Current main still has the relevant anonymous install failure path: download requests are rate-limited before ZIP delivery, authenticated requests only move to a user bucket when a valid token is present, anonymous requests fall back to an IP or `unknown` bucket, and `clawhub install` only sends auth when a saved token already exists. The discussion contains useful login/token workarounds, but related issue #1410 remains open with a narrower reproduction of anonymous installs hitting 429 while authenticated installs succeed.

## Best Possible Solution

Keep this issue open and consolidate the remaining investigation with #1410 as the canonical anonymous-install 429 thread. The best path is to verify production client-IP forwarding and anonymous bucket attribution, decide whether installs should prompt for or require login before rate-limited downloads, and add a clear install-specific 429 hint that points users to `clawhub login` or `clawhub login --token ...` when authentication would move them to the user bucket.
