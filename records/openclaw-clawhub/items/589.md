## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy for this pass keeps ClawHub issues open. The current main branch still has the anonymous install failure path: downloads are rate-limited before ZIP delivery, unauthenticated requests use an IP or `unknown` bucket, and `clawhub install` only sends auth when a token already exists. The skill detail UI now prefers `openclaw skills install ...`, but the underlying anonymous-download 429 behavior remains and is also tracked by open related issue #1410.

## Best Possible Solution

Keep this issue open and likely consolidate the remaining work with #1410 as the canonical anonymous-install 429 thread. The best path is to verify production client-IP forwarding and bucket attribution, decide whether installs should prompt for or require login before rate-limited downloads, add an install-specific 429 hint for token/login use, and keep the updated skill-detail install surface that avoids the old `npx clawhub@latest install ...` guidance.
