## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Substantively, current main still allows anonymous `clawhub install` attempts, still applies the anonymous IP download bucket before serving skill ZIPs, and still advertises the `npx clawhub@latest install ...` flow that the report calls out. Related #1410 remains open as a narrower anonymous false-positive 429 thread.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is either to manually consolidate it into #1410 as the canonical anonymous-install 429 thread, or to implement a concrete install UX fix: surface a specific login/token hint on 429, prompt or require login before rate-limited downloads when appropriate, update web install guidance if `npx clawhub@latest install ...` remains prone to anonymous rate-limit confusion, and verify production proxy/client-IP handling for anonymous buckets.
