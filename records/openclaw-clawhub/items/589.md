## Summary

Keep open. This is an open ClawHub issue, and this cleanup pass only auto-closes ClawHub pull requests that are certainly implemented on current main. Current main still has the relevant behavior: anonymous downloads are rate-limited by IP, missing client IP collapses to an `unknown` bucket, install uses an auth token only if one already exists, and the public install surface still recommends `npx clawhub@latest install ...`. Related #1410 remains open for the narrower anonymous-install 429 path.

## Best Possible Solution

Keep this issue open for maintainer triage and likely consolidation with #1410 as the canonical anonymous-install 429 thread. The best path is to verify production client-IP forwarding and anonymous bucket attribution, decide whether installs should prompt or require login before rate-limited downloads, add an install-specific 429 hint for login/token use, and update public install guidance so users do not discover the workaround only through issue comments.
