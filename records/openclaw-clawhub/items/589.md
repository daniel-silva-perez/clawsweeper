## Summary

Keep open. The repository cleanup policy for ClawHub issues does not allow auto-closing this issue, and current main still leaves the reported failure mode reachable through anonymous download rate limiting plus optional CLI authentication. The discussion’s login, token, `pnpm dlx`, VPN-node, and skills.sh workarounds are useful, but the remaining fix needs maintainer handling across production IP attribution, rate-limit policy, and install UX.

## What This Changes

The issue reports `npx clawhub@latest install <skill>` failing with `Rate limit exceeded` across multiple networks and asks for a fix or clearer install guidance.

## Best Possible Solution

Keep this issue open while maintainers consolidate it with #1410 for anonymous-install attribution and #349 for rate-limit policy/visibility. The desired end state is reliable anonymous bucket attribution in production, a clear product decision on whether installs should prompt for or require login before rate-limited downloads, and install-specific 429 guidance that points users to `clawhub login` or `clawhub login --token ...` when authentication would move them to a user bucket.
