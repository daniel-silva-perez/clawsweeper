## Summary

Keep open. The repository cleanup policy keeps ClawHub issues open, and current main still supports the reported failure mode: anonymous ZIP downloads are rate-limited by IP or an `unknown` fallback bucket, while `clawhub install` only authenticates when a token is already saved. The login/token, `pnpm dlx`, and skills.sh workarounds in the discussion are useful, but related #1410 and #349 remain open for narrower maintainer triage.

## What This Changes

The issue reports that `npx clawhub@latest install <skill>` can fail with `Rate limit exceeded`, with users asking for a fix or clearer install/login guidance.

## Best Possible Solution

Keep this issue open while maintainers consolidate it with #1410 for the anonymous-install 429 reproduction and #349 for rate-limit policy/visibility. The desired end state is reliable anonymous bucket attribution in production, a clear decision on whether installs should prompt for or require login before rate-limited downloads, and install-specific 429 guidance that points users to `clawhub login` or `clawhub login --token ...` when authentication would move them to a user bucket.
