## Summary

Keep this issue open. It is a ClawHub issue, and the supplied repository policy only allows auto-closing pull requests that are certainly implemented on current main. Current main also shows partial rate-limit mitigation, not a complete fix for the reported install UX: anonymous installs are still IP-limited, `clawhub install` uses optional auth instead of requiring or prompting for login, and the web install surface still shows `npx clawhub@latest install ...` commands.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is either to manually consolidate it into open #1410 after maintainer review, or to make a concrete install UX fix: require or strongly prompt login before rate-limited downloads, surface a specific `clawhub login` / token-login hint on 429, and update the web install command guidance if `npx clawhub@latest install ...` remains prone to confusing anonymous rate limits.
