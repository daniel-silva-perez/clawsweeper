## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits automated closure for pull requests that are certainly implemented on current main. Current main still has an anonymous download rate-limit path for skill installs, CLI install only uses a token when one already exists, and related #1410 remains open for the narrower anonymous-install false-positive 429 behavior.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to consolidate it manually with #1410 if that is the canonical anonymous-install 429 thread, then decide the fix boundary: verify production client-IP forwarding for anonymous buckets, decide whether install should prompt or require login before rate-limited downloads, add a login/token-specific hint for install 429s, and update public install guidance so users do not discover the workaround only through issue comments.
