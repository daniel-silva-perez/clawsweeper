## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy for this repository keeps issues open unless maintainers handle them directly. Current main still has the anonymous install 429 path: downloads are rate-limited before ZIP delivery, anonymous requests use an IP or `unknown` bucket, and `clawhub install` only sends auth when a saved token already exists. The login/token and skills.sh workarounds in the discussion are useful, but related #1410 remains open with a narrower anonymous-install reproduction and #349 remains open for rate-limit policy/visibility.

## What This Changes

Users report `npx clawhub@latest install <skill>` and related ClawHub install flows failing with `Rate limit exceeded`, and ask for a fix or clearer login/install guidance.

## Best Possible Solution

Keep this issue open and consolidate maintainer triage with #1410 as the canonical anonymous-install 429 reproduction, while referencing #349 for broader rate-limit visibility and policy. The best path is to verify production client-IP forwarding and anonymous bucket attribution, decide whether installs should prompt for or require login before rate-limited downloads, and add install-specific 429 guidance that points users to `clawhub login` or `clawhub login --token ...` when authentication would move them to a user bucket.
