## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Current main has useful rate-limit mitigations, but the reported install UX is not fully resolved: anonymous downloads are still IP-limited, `clawhub install` uses auth only when a saved token exists, and the web install surface still shows `npx clawhub@latest install ...` commands. Related open #1410 also tracks the narrower anonymous-install 429 false-positive path.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is either to manually consolidate it into open #1410 after maintainer review, or to make a concrete install UX fix: require or strongly prompt login before rate-limited downloads, surface a specific `clawhub login` / token-login hint on 429, and update the web install command guidance if `npx clawhub@latest install ...` remains prone to confusing anonymous rate limits.
