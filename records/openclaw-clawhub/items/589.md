## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Current main still permits anonymous `clawhub install`, still rate-limits anonymous downloads by IP/unknown bucket, and still advertises `npx clawhub@latest install ...`; related #1410 remains open as a narrower anonymous false-positive 429 report.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to either manually consolidate it into #1410 as the canonical anonymous-install 429 thread, or implement a concrete install UX/backend fix: surface a login/token hint on install 429s, decide whether skill installs should prompt or require login before rate-limited downloads, update the public install guidance if `npx clawhub@latest install ...` remains confusing, and verify production client-IP forwarding for anonymous buckets.
