## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing PRs that are certainly implemented on main. Current main still intentionally blocks same-GitHub sign-in after account deletion, so the requested re-signup behavior remains a product/security decision rather than implemented cleanup.

## Best Possible Solution

Keep this issue open and triage it with #504. Maintainers should choose the product policy: either keep deletion permanently irreversible with clearer sign-in/support messaging, or implement a secure same-GitHub re-registration/relink flow that retires the old auth binding without allowing banned accounts to bypass moderation. Add regression coverage for delete-account followed by same-GitHub sign-in once that policy is chosen.
