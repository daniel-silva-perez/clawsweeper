## Summary

Keep open. Current main still rejects same-GitHub sign-in after user-initiated account deletion, and ClawHub cleanup policy keeps issues open unless maintainers explicitly decide otherwise. The remaining work is product- and security-sensitive because it changes deletion, OAuth binding, and abuse-prevention semantics.

## What This Changes

Allow users who deleted a previous ClawHub account to sign up or sign in again with the same GitHub account instead of landing in a refresh/sign-in loop.

## Best Possible Solution

Maintainers should decide and document the account deletion policy. If re-signup is allowed, implement a secure same-GitHub recreate or relink flow that preserves ban protections, publisher/slug ownership, and token revocation invariants; if deletion remains irreversible, make the OAuth result and support path explicit enough that users are not left in a loop.
