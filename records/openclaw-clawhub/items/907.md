## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing PRs that are certainly implemented on current main. The current code/docs still intentionally treat account deletion as irreversible and block same-GitHub reauth rather than recreating or relinking the account, so the requested re-signup behavior remains unresolved product/security work.

## Best Possible Solution

Keep this issue open and triage it with #504 and #900. Maintainers should decide the product policy: either keep account deletion irreversible and improve the sign-in/support message, or implement a secure same-GitHub re-registration or relinking flow that retires the old auth binding without allowing banned accounts to bypass moderation. Add regression coverage for delete-account followed by same-GitHub sign-in once the policy is chosen.
