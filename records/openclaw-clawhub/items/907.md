## Summary

Keep this issue open. The ClawHub cleanup policy only permits auto-close for PRs that are definitely implemented on main, and current main still treats user deletion as irreversible: deleted/deactivated users are rejected during GitHub OAuth reauth rather than re-signed up or relinked.

## Best Possible Solution

Keep this open and triage it with #504, #900, #1116, and #1452. Maintainers should choose the account-deletion policy explicitly: either keep deletion irreversible and make the OAuth callback/sign-in UI consistently show the deleted-account support message, or implement a secure same-GitHub re-registration/relinking flow that retires the old auth binding while preserving ban and auto-ban protections. Add regression coverage for delete-account followed by same-GitHub sign-in once that policy is chosen.
