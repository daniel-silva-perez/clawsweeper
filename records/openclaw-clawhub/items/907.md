## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits automated closure for PRs that current main certainly implements. The current main branch still intentionally blocks sign-in for deleted/deactivated users rather than allowing same-GitHub re-signup, while related reports #504 and #900 remain open with the same user-facing OAuth loop.

## Best Possible Solution

Keep this issue open and triage it with #504 and #900. Maintainers should choose the product policy explicitly: either keep user deletion irreversible and improve the sign-in/support messaging, or implement a secure same-GitHub re-registration/relinking flow that retires the old auth binding without letting banned or auto-banned accounts regain access. Add regression coverage for delete-account followed by same-GitHub sign-in once that policy is chosen.
