## Summary

Keep this issue open. Current main still treats user-initiated deletion as irreversible and blocks same-GitHub sign-in after deletion; recent code improves the auth error path, but it does not implement re-signup or OAuth binding reset. ClawHub cleanup policy also says issues should stay open unless maintainers decide otherwise.

## What This Changes

Allow a user who deleted a ClawHub account to sign up or sign in again with the same GitHub account instead of returning to a sign-in loop.

## Best Possible Solution

Keep this issue open and triage it with #504, #900, #1116, and #1452. Maintainers should explicitly choose between keeping deletion irreversible with consistent OAuth/support messaging, or implementing a secure same-GitHub re-registration/relinking path that retires or updates the old auth binding while preserving ban and auto-ban protections, then add regression coverage for delete-account followed by same-GitHub sign-in.
