## Summary

Keep this issue open. Current main still rejects same-GitHub sign-in after user-initiated account deletion, and the ClawHub cleanup policy says issues should remain open unless maintainers explicitly decide otherwise. The current implementation improves the auth error surface but does not provide re-signup, OAuth binding removal, or secure relinking.

## What This Changes

The issue requests that a user who deleted their ClawHub account be able to sign up or sign in again with the same GitHub account instead of getting stuck in a refresh/sign-in loop.

## Best Possible Solution

Maintainers should choose and document the intended deletion policy. If same-GitHub re-signup is allowed, implement a secure re-registration or relinking path that updates the old auth binding and deleted user state without weakening ban protections; otherwise, keep irreversible deletion and make the OAuth/support message explicit enough that users are not left in a loop.
