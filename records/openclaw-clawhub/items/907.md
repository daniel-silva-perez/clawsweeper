## Summary

Keep this issue open. Current main still intentionally blocks sign-in for users whose account was deleted/deactivated, and the docs/settings UI describe user deletion as irreversible. The frontend now surfaces a clearer auth error, but it does not implement same-GitHub re-signup or OAuth binding reset. ClawHub cleanup policy also says issues should stay open unless maintainers decide otherwise.

## What This Changes

The issue requests that a user who deleted their ClawHub account be able to sign up or sign in again with the same GitHub account instead of getting stuck in the OAuth/sign-in loop.

## Best Possible Solution

Keep this issue open and triage it with #504, #900, #1116, and #1452. Maintainers should choose the account-deletion policy explicitly: either keep deletion irreversible and make OAuth/sign-in UI consistently show the deleted-account support message, or implement a secure same-GitHub re-registration/relinking path that retires or updates the old auth binding while preserving ban and auto-ban protections. Add regression coverage for delete-account followed by same-GitHub sign-in after the policy is chosen.
