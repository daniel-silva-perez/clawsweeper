## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only allows auto-closing ClawHub PRs that are certainly implemented on main. Current main still implements account deletion as an irreversible deactivation/purge path, OAuth still rejects deleted/deactivated users instead of allowing same-GitHub re-signup, and related #504 remains open for the same account lifecycle problem.

## Best Possible Solution

Keep this issue open and triage it with #504. Maintainers should decide whether deletion remains permanently irreversible with clearer login/support messaging, or implement a safe same-GitHub re-registration/relink flow that retires the old auth binding without restoring banned accounts. Add regression coverage for delete-account followed by same-GitHub sign-in once the product policy is chosen.
