## Summary

Keep this issue open. The ClawHub repository policy supplied for this review only permits automated closure for PRs that current main certainly implements, not issues. The current main behavior still intentionally rejects sign-in for deleted/deactivated users, and related user reports #504, #900, and #1116 remain open with the same deleted-account GitHub OAuth loop/re-signup problem.

## Best Possible Solution

Keep this issue open and triage it together with #504, #900, and #1116. Maintainers should choose the product policy explicitly: either keep user deletion irreversible and improve the OAuth callback/sign-in UI so users see a clear support message instead of a refresh loop, or implement a secure same-GitHub re-registration/relinking flow that retires the old auth binding without letting banned or auto-banned accounts regain access. Add regression coverage for delete-account followed by same-GitHub sign-in once that policy is chosen.
