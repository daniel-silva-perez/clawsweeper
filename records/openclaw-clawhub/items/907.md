## Summary

Keep this issue open. It is an issue, and the ClawHub cleanup policy only permits automatic implemented-on-main closure for PRs. Current main also still blocks deleted/deactivated users from same-GitHub sign-in while documenting user deletion as irreversible; related reports #504, #900, #1116, and #1452 remain open for the same deleted-account OAuth loop/re-signup problem.

## Best Possible Solution

Keep this issue open and triage it with #504, #900, #1116, and #1452. Maintainers should choose the account-deletion policy explicitly: either keep deletion irreversible and make the OAuth callback/sign-in UI consistently show the deleted-account support message, or implement a secure same-GitHub re-registration/relinking flow that retires the old auth binding while preserving ban and auto-ban blocks. Add regression coverage for delete-account followed by same-GitHub sign-in once that policy is chosen.
