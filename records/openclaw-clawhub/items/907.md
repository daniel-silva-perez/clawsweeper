## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only allows auto-closing ClawHub PRs that are certainly implemented on main. Current main still treats user-initiated account deletion as irreversible: deletion deactivates and purges the user, the OAuth callback rejects deleted/deactivated users, and docs/UI/tests describe blocked reauth rather than same-GitHub re-signup.

## Best Possible Solution

Keep this issue open and triage it with #504. Maintainers should choose between preserving irreversible deletion with clearer login/support messaging, or implementing a safe same-GitHub re-registration/relink flow that retires the old auth binding without restoring banned accounts. Add regression coverage for delete-account followed by same-GitHub sign-in once that product policy is chosen.
