## Summary

Keep this issue open. Current main still blocks same-GitHub sign-in after user-initiated account deletion, and the ClawHub cleanup policy keeps issues open unless maintainers explicitly decide otherwise. The recent code improves the auth error surface but does not implement re-signup, OAuth binding removal, or account relinking.

## What This Changes

The issue asks for a deleted ClawHub account to be able to sign up or sign in again with the same GitHub account instead of returning to a stuck sign-in loop.

## Best Possible Solution

Maintainers should choose and document the intended deletion policy. If re-signup is allowed, implement a secure same-GitHub re-registration or relinking path that updates the old auth binding and deleted user state without weakening ban protections; otherwise, keep irreversible deletion and make the OAuth/support message explicit enough that users are not left in a loop.
