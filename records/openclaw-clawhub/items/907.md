## Summary

Keep open. Current main still treats user-initiated deletion as irreversible, blocks deleted/deactivated users during GitHub OAuth completion, and only surfaces a user-facing error instead of implementing same-GitHub re-signup or relinking. Because this is an issue in openclaw/clawhub, the repository cleanup policy does not auto-close it, and the remaining fix needs maintainer product and security judgment.

## What This Changes

Allow a user to sign up or sign in again with the same GitHub account after deleting their previous ClawHub account instead of getting stuck in a refresh/OAuth loop.

## Best Possible Solution

Maintainers should decide and document the account deletion policy. If re-signup is allowed, implement a secure same-GitHub recreate/relink flow that preserves abuse protections and ownership invariants; if deletion stays irreversible, make the OAuth result and support path explicit enough that users are not left in a loop.
