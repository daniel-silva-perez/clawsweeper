## Summary

Keep this issue open. Current main still treats user-initiated deletion as irreversible and blocks same-GitHub reauth, while the ClawHub cleanup policy only permits automated closure for PRs that are certainly implemented on main, not issues. Related reports #504, #900, #1116, and #1452 remain open for the same deleted-account OAuth loop/re-signup problem.

## Best Possible Solution

Keep this issue open and triage it with #504, #900, #1116, and #1452. Maintainers should choose the product policy explicitly: either keep deletion irreversible and ensure the OAuth callback/sign-in UI always shows the deleted-account support message instead of a refresh loop, or implement a secure same-GitHub re-registration/relinking flow that retires the old auth binding without restoring banned or auto-banned access. Add regression coverage for delete-account followed by same-GitHub sign-in once that policy is chosen.
