## Summary

Keep this issue open. It is an open ClawHub issue, and the repository-specific cleanup policy only permits automated closure of pull requests that are certainly implemented on current main. The underlying request is still live: current main treats user deletion as irreversible, blocks deleted/deactivated GitHub reauth, and does not provide an account restore or GitHub identity unlink flow.

## Best Possible Solution

Keep this open for maintainer support and product/security triage. Maintainers should either document and communicate a strict irreversible-deletion policy, or add a safe admin/support flow to restore, unlink, or supersede deleted GitHub-linked accounts with tests covering OAuth reauth, authAccounts linkage, token revocation, handles, publishers, skills, bans, and existing deleted-account records.
