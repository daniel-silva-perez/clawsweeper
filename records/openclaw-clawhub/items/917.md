## Summary

Keep this issue open. It is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing PRs that are certainly implemented on main. Current main still treats user-initiated deletion as irreversible: deletion deactivates and purges the user record, OAuth reauth blocks deleted/deactivated users, admin unban rejects permanently deleted accounts, and the docs say deleted accounts cannot be restored by logging in again. The issue discussion also includes a second affected user, with related account-lifecycle reports #504, #1452, and #1620 still open.

## Best Possible Solution

Keep this open for maintainer support and product/security triage. Maintainers should either confirm and document an irreversible-deletion support policy with a clear contact path, or add a safe backend/admin flow to restore, unlink, or supersede deleted GitHub-linked accounts, with tests covering web OAuth, CLI/API tokens, banned-account separation, handles, publishers, skills, and existing deleted-account records.
