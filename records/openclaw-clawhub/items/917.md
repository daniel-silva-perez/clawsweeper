## Summary

Keep this issue open. It is an open ClawHub issue, and the repository-specific cleanup policy only permits automated closure of pull requests that are certainly implemented on current main. The underlying request is also still live: current main treats user deletion as irreversible, blocks deleted/deactivated GitHub reauth, and has no source-level account restoration or GitHub identity unlink flow.

## Best Possible Solution

Keep this open for maintainer support and product/security triage. Maintainers should either handle the affected accounts operationally under the current irreversible-deletion policy, or design a safe admin/support flow to restore, unlink, or supersede deleted GitHub-linked accounts with tests covering OAuth reauth, authAccounts linkage, token revocation, handles, publishers, skills, bans, and existing deleted-account records.

## Risks / Open Questions

- Changing the current behavior would conflict with existing docs and UI that describe deletion as irreversible, so it needs explicit product/security review.
- Related open reports show this is a repeated support gap, so closing this issue would drop affected-user context.
