## Summary

Keep this issue open. It is an open ClawHub issue, and the repository-specific cleanup policy only permits automated closure for pull requests that are certainly implemented on current main. Current main still treats user-initiated deletion as irreversible deactivation, blocks GitHub OAuth reauthentication for deleted/deactivated users, and has no self-service account restoration or GitHub identity unlink flow.

## What This Changes

The issue requests restoration of an accidentally deleted ClawHub account or unlinking the associated GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Keep this open for maintainer support and product/security triage. Maintainers should either handle the affected production accounts operationally under the current irreversible-deletion policy, or design an audited admin/support flow to restore, unlink, or supersede deleted GitHub-linked accounts with tests covering OAuth reauth, authAccounts linkage, token revocation, handles, publishers, skills, bans, and existing deleted-account records.
