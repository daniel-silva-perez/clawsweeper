## Summary

Keep this issue open. The ClawHub cleanup policy only permits automated closure for PRs that are certainly implemented on current main, and this item is an issue. Current main also still treats user-initiated account deletion as irreversible: deletion deactivates and purges the user, OAuth reauth rejects deleted/deactivated users, tests and docs assert that behavior, and I did not find an account restoration or GitHub identity unlink path. The discussion includes another affected user plus related open reports #900 and #1620, so this needs maintainer support/product/security triage rather than cleanup closure.

## Best Possible Solution

Keep this open for maintainer support and product/security triage. Maintainers should either confirm and document an irreversible-deletion support policy with a clear support path, or add a safe backend/admin flow to restore, unlink, or supersede deleted GitHub-linked accounts. That work should include tests for OAuth reauth, authAccounts identity linkage, API token revocation, banned-account separation, handles, publishers, skills, and existing deleted-account records, and should be triaged together with #900 and #1620.
