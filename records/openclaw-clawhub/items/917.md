The repository cleanup policy for openclaw/clawhub only permits automated closure for pull requests that current main certainly implements, and this is an open issue; current main also still treats user-initiated account deletion as irreversible deactivation with no restore or GitHub identity unlink path.

## What This Changes

The issue requests restoration of an accidentally deleted ClawHub account or unlinking of the reporter’s GitHub identity so they can re-register.

## Best Possible Solution

Define an audited maintainer/admin recovery process for deleted GitHub-linked accounts, or document and perform an operational unlink/restore path for affected users, while preserving ban protections and covering existing deleted-account records in tests.
