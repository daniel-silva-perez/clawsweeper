## Summary

Keep this issue open. The ClawHub cleanup policy keeps open issues open, and current main still has the reported account lifecycle gap: deleted accounts are marked as permanently deactivated, GitHub OAuth sign-in is rejected for those records, and there is no self-service or admin restore/unlink path for the GitHub identity.

## What This Changes

The issue asks maintainers to restore an accidentally deleted ClawHub account or unlink the reporter's GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Maintain an explicit recovery policy for deleted GitHub-linked users: either handle affected production accounts operationally under the current irreversible-deletion model, or design an audited admin/support flow that safely restores or unlinks identities while preserving account ownership and moderation invariants.
