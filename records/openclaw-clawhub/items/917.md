## Summary

Keep this issue open. The repository policy for openclaw/clawhub only allows auto-closing PRs that are certainly implemented on main, and this is an issue. Current main also still treats account deletion as irreversible and does not provide account restoration or GitHub identity unlink/re-registration.

## Best Possible Solution

Keep this open for maintainer support and product/security judgment. If deletion is intended to remain irreversible, maintainers should reply with that policy and any private support path. If same-GitHub re-registration should be allowed, add a safe backend/admin flow around Convex Auth authAccounts that can unlink or supersede a deleted account without restoring banned users, with tests for web OAuth, CLI/API token behavior, handles, publishers, and existing deleted-account records.
