## Summary

Keep this issue open. Repository policy for openclaw/clawhub only permits automated closure of PRs that are certainly implemented on current main, and this item is an issue. The substantive request is also still unresolved: current main treats self-deletion as irreversible, rejects deleted/deactivated GitHub reauth, has tests and docs asserting that behavior, and the nearby admin restore/reclaim endpoints restore skills or slugs rather than restoring a user account or unlinking a GitHub identity.

## Best Possible Solution

Keep this open for maintainer support and product/security triage. Maintainers should either confirm and document a clear irreversible-deletion support policy, or add a safe backend/admin flow to restore, unlink, or supersede deleted GitHub-linked accounts, with tests around OAuth reauth, authAccounts linkage, token revocation, handles, publishers, skills, banned-account separation, and existing deleted-account records.
