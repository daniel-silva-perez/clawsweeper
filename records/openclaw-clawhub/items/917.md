## Summary

Keep this issue open. It is an open ClawHub issue, and the repository-specific cleanup policy only permits automated closure of pull requests that are certainly implemented on current main. The user request is also still live in current main: self-deletion writes irreversible deactivation state, GitHub OAuth reauth is rejected, admin unban refuses permanently deleted accounts, and there is no source-level account restoration or GitHub identity unlink flow.

## Best Possible Solution

Keep this open for maintainer support and product/security triage. Maintainers should either handle the affected accounts operationally under the current irreversible-deletion policy, or design a safe audited admin/support flow to restore, unlink, or supersede deleted GitHub-linked accounts with tests covering OAuth reauth, authAccounts linkage, token revocation, handles, publishers, skills, bans, and existing deleted-account records. Related reports should likely be consolidated around one maintainer-owned support/product decision rather than auto-closed.
