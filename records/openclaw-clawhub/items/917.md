## Summary

Keep this open. This is an open ClawHub issue, and the cleanup policy only permits automated closure for pull requests that current main certainly implements; current main also still blocks GitHub reauthentication for permanently deleted users and has no account restore or GitHub identity unlink path for the requested recovery.

## What This Changes

The issue asks maintainers to restore an accidentally deleted ClawHub account or unlink its GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Define an audited support/admin recovery path for deleted GitHub-linked accounts, or document and perform an operational unlink/restore process for affected users, while preserving ban protections and covering existing deleted-account records in tests.
