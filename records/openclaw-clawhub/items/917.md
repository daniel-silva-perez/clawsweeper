## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing PRs that are certainly implemented on current main. The current source still treats user-initiated account deletion as irreversible: it deactivates/purges the user, revokes tokens, blocks OAuth reauth, exposes no account restoration or GitHub identity unlink flow, and admin unban explicitly refuses permanently deleted accounts. The discussion also has a second affected user and a related open report, so this needs maintainer support/product/security triage rather than cleanup closure.

## Best Possible Solution

Keep this open for maintainer support and product/security triage. Maintainers should either confirm and document an irreversible-deletion support policy with a clear contact path, or add a safe backend/admin flow to restore, unlink, or supersede deleted GitHub-linked accounts, with tests covering OAuth reauth, CLI/API tokens, banned-account separation, handles, publishers, skills, and existing deleted-account records. Related open reports such as #1620 should be triaged together.
