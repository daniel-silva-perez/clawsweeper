## Summary

Keep open. This is an open ClawHub issue for a specific hosted-skill moderation appeal, and the repository policy for this cleanup pass only permits auto-closing ClawHub pull requests that are certainly implemented on main. Current main has generic appeal visibility, moderation detail APIs, and moderator override tooling, but the live Weixin skill is still flagged and its latest hosted SKILL.md sends QR payload data to a third-party QR image service, which needs maintainer/security review rather than source-only cleanup.

## Best Possible Solution

Keep this issue open for maintainer/security review of the published Weixin skill, preferably against the latest hosted version 5.0.1. If the QR-service behavior is acceptable, staff can use the existing manual override flow with an audit note and leave a resolution comment; if not, ask the author to publish a safer version, such as local/on-device QR image generation that does not send QR payload data to an external service, then rescan or review again.
