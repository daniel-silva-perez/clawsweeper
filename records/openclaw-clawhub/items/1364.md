## Summary

Keep this issue open. Current main still contains the reported `transfer request` body serialization bug: the CLI maps the positional handle to `toUserHandle`, but passes a JSON string into an HTTP helper that serializes bodies again before sending them.

## Best Possible Solution

Keep this issue open and land a focused CLI fix: pass a plain object body from `cmdTransferRequest` to `apiRequest`, update the transfer test to assert an object payload, add or adjust HTTP-layer coverage so double-serialization is caught, and audit the adjacent rename/merge ownership commands that use the same pre-serialized body pattern.
