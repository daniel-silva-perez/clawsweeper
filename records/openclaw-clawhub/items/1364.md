## Summary

Keep open. The reporter's root-cause comment still matches current main: the transfer request command maps the positional handle to `toUserHandle`, but pre-serializes the body before handing it to an HTTP client that serializes POST bodies again. The repository cleanup policy for ClawHub also says issues should stay open, and this one still appears to be a live CLI bug rather than cleanup-closeable.

## Best Possible Solution

Keep this issue open. Fix `packages/clawhub/src/cli/commands/transfer.ts` to pass a plain object body to `apiRequest`, add a regression test that verifies the final POST payload is a JSON object containing `toUserHandle`, and audit the adjacent ownership rename/merge commands that still pre-serialize request bodies.
