## Summary

Keep open. Current main still contains the reported CLI request serialization bug, and repository policy keeps ClawHub issues open for maintainer follow-up; the transfer command still sends a JSON string body that the HTTP helper serializes again before the API checks `toUserHandle`.

## What This Changes

The issue reports that `clawhub transfer request <slug> <handle>` fails because the positional handle is not delivered to the API as the `toUserHandle` JSON field.

## Best Possible Solution

Land a focused CLI repair that sends plain object bodies from transfer, rename, and merge command handlers, updates the command tests to assert object bodies, and leaves the HTTP helper responsible for final JSON serialization.

## Repair Work Prompt

Fix the CLI request-body serialization bug reported for `clawhub transfer request <slug> <handle>`. In `packages/clawhub/src/cli/commands/transfer.ts`, make `cmdTransferRequest` pass a plain object body with `toUserHandle` and `message` to `apiRequest`, not a pre-serialized JSON string. Audit and fix the same command-layer pattern in `packages/clawhub/src/cli/commands/ownership.ts` for rename and merge. Update `packages/clawhub/src/cli/commands/transfer.test.ts` and `packages/clawhub/src/cli/commands/ownership.test.ts` so they assert object bodies rather than serialized strings. Keep `packages/clawhub/src/http.ts` as the single JSON serialization boundary; do not change the Convex API to accept double-encoded strings. Add a short `CHANGELOG.md` entry under `Unreleased` / `Fixes` for the CLI transfer/ownership request body fix. Avoid unrelated refactors or UI changes.
