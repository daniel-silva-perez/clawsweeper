## Summary

Keep open. This is a ClawHub issue, and the repo policy only permits auto-closing PRs that are certainly implemented on main. The underlying bug also still appears present on current main: the transfer command passes a pre-stringified body to an HTTP client that stringifies POST bodies again, while the backend expects a JSON object with `toUserHandle`.

## Best Possible Solution

Keep this issue open. Fix `packages/clawhub/src/cli/commands/transfer.ts` to pass a plain object body to `apiRequest`, add a regression test that verifies the final POST payload is a JSON object containing `toUserHandle`, and audit adjacent ownership POST commands that still pre-serialize request bodies.
