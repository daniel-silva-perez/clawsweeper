## Summary

Keep this issue open. Current main still contains the reported transfer request bug: the CLI maps the positional handle to `toUserHandle`, but pre-serializes the body before passing it to `apiRequest`, and both HTTP transport paths serialize `args.body` again. ClawHub policy also says open issues should not be auto-closed by this cleanup workflow.

## Best Possible Solution

Keep this issue open. Fix `packages/clawhub/src/cli/commands/transfer.ts` by passing a plain object body to `apiRequest`, add a regression test that fails if the transfer request body is pre-serialized or double-serialized through the HTTP layer, and audit adjacent ownership commands that still use `body: JSON.stringify(...)`.
