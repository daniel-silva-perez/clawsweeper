## Summary

Keep open. Repository policy for openclaw/clawhub only permits auto-closing pull requests that are definitely implemented on current main, not issues. Substantively, current main still treats `child_process` exec/spawn usage as `suspicious.dangerous_exec`, does not include that code in the externally-clearable static allowlist, and the related demotion proposal was closed unmerged, so this remains a maintainer/moderator decision for the hosted Bc Calc listing rather than conservative cleanup.

## What This Changes

The issue asks whether Bc Calc should be flagged as suspicious when its stated purpose is to shell out to a dedicated calculator command.

## Best Possible Solution

Keep this issue open for maintainer/moderator follow-up. If the hosted Bc Calc implementation is acceptable, clear it through the audited manual override or rescan workflow. If maintainers want a broader false-positive policy change, open a narrow scanner-policy PR with regression tests that preserves default `child_process` detection and explicitly defines when a declared single-command shell wrapper may be treated as benign.
