## Summary

Keep open for maintainer/operator follow-up. Current main still exposes owner-facing deletion as soft-delete, while hard-delete is admin-only and destructive; under the ClawHub cleanup policy, issues should not be auto-closed here.

## What This Changes

The issue asks maintainers to hard-delete versions 1.0.0, 1.0.1, and 1.0.2 of JasonStarlight/phosor-ai so the owner can republish from a clean v1.0.0.

## Best Possible Solution

A maintainer should verify the requester controls the production owner account and target skill, then either run an audited admin purge with an explicit scope or design a documented owner reset/version-purge workflow if ClawHub wants to support this regularly.
