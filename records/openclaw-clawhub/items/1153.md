This issue should remain open for maintainer/operator handling. Current main still treats owner-facing CLI/API deletion as soft-delete, existing version rows continue to block republishing the same version, and hard deletion is a destructive production/admin concern rather than a safe automated code cleanup.

## What This Changes

The requester asks maintainers to hard-delete all published versions of `JasonStarlight/phosor-ai` so the skill can be republished from a clean `v1.0.0`.

## Best Possible Solution

Keep this open for maintainer/operator handling. A maintainer should verify the requester controls the production owner account and target skill, decide the exact purge scope, then either run the existing admin hard-delete path or design an audited owner reset/version-purge workflow if this request pattern should become supported.
