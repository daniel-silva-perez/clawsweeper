## Summary

Keep open. This is a live production purge request, not a source cleanup item: current main still exposes owner-facing deletion as soft-delete, while the existing hard-delete path is admin-only and destructive. The related broader hard-delete/version-purge policy request #1167 is also still open, so this should stay with maintainers/operators for ownership verification and purge-scope judgment.

## What This Changes

The issue asks maintainers to hard-delete versions 1.0.0, 1.0.1, and 1.0.2 of `JasonStarlight/phosor-ai` so it can be republished from a clean v1.0.0.

A maintainer should verify the requester controls the production owner account and target skill, decide the exact purge scope, then either run the existing admin hard-delete flow or design an audited owner reset/version-purge workflow that handles namespace reuse and retention consistently with #1167.
