## Summary

Keep open for maintainer/operator handling. Current main still exposes owner-facing deletion as a soft-delete, historical version rows still block republishing the same version, and the hard-delete path is admin-only/destructive rather than a safe automated cleanup for an issue request.

## What This Changes

The issue asks maintainers to hard-delete all stored versions of `JasonStarlight/phosor-ai` so the requester can republish from a clean `v1.0.0`.

## Best Possible Solution

A maintainer should verify the requester controls the production owner account and target skill, decide the destructive purge scope explicitly, then use the existing admin hard-delete path or design an audited owner reset/version-purge workflow if ClawHub wants to support this request pattern regularly.
