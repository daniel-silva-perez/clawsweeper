## Summary

Keep open. This is an open ClawHub issue, and the repository policy for this review only permits auto-close proposals for pull requests that are certainly implemented on current main. Substantively, current main still matches the reporter's problem: the owner-facing CLI/API delete path is a soft delete, existing skillVersions rows continue to block reusing the same semver, and the hard-delete path is admin/operator-controlled production cleanup rather than a self-service reset.

## Best Possible Solution

Keep this issue open for maintainer/operator handling. A maintainer should verify requester ownership in production, inspect the skill, version, fingerprint, embedding, file-storage, and audit implications for the named versions, then run the existing admin hard-delete path or add a narrower audited owner reset/version-purge workflow if these clean-republish requests are expected to recur. Related open cleanup request #1167 can be triaged alongside this one, but it is not a reason to auto-close this issue under the ClawHub policy.
