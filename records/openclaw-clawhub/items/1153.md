## Summary

Keep open. This is a ClawHub issue, and the supplied repository policy only permits auto-close for pull requests that current main definitely implements. Current main also still leaves the owner-facing delete path as soft delete, while hard delete is admin-only and production-destructive.

## Best Possible Solution

Keep this issue open for maintainer/operator handling. A maintainer should verify the requester controls the production owner account and target skill, decide whether storage blobs also need deletion, then either run the existing admin hard-delete path or define an audited owner reset/version-purge workflow if these requests are expected to recur. Triage #1167 at the same time for a consistent policy.
