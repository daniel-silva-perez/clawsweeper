## Summary

Keep open. This is an issue, and the supplied ClawHub policy only permits auto-close for pull requests that are certainly implemented on current main. Current main still exposes owner CLI/API deletion as soft-delete, while destructive hard-delete is admin-only and requires production ownership/purge-scope verification.

## What This Changes

The issue requests hard-deleting all versions (1.0.0, 1.0.1, 1.0.2) of JasonStarlight/phosor-ai so the owner can republish from a clean v1.0.0.

## Best Possible Solution

Keep this open for maintainer/operator handling. A maintainer should verify that the requester controls the production owner account and target skill, decide the purge scope for version rows, files/storage, backups, audit logs, and slug reservation, then either run the existing admin hard-delete path or define an audited owner reset/version-purge workflow if this request class will recur. Triage #1167 at the same time for policy consistency.
