## Summary

Keep open for maintainer/operator handling. Current main still exposes owner-facing deletion as a soft-delete, historical version rows still block republishing the same version, and the available hard-delete path is admin-only/destructive rather than a safe automated cleanup for an issue request.

## What This Changes

The requester wants all published versions 1.0.0, 1.0.1, and 1.0.2 of JasonStarlight/phosor-ai hard-deleted so they can republish v1.0.0 from a clean slate.

## Best Possible Solution

Production data should be purged only after maintainer verification and an explicit scope decision, either by using the existing audited admin hard-delete path or by designing a documented owner reset/version-purge workflow if ClawHub wants to support this pattern regularly.
