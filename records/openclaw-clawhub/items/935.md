## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy only auto-closes ClawHub PRs that current main definitely implements. Substantively, the live cashbook 1.3.0 record is still flagged suspicious: VT and static scans are clean, but the active LLM moderation finding remains because the published package references a missing export script and makes local-only privacy claims while using screenshot OCR/image tooling.

## What This Changes

The issue asks maintainers to review and clear the suspicious install warning for the published `cashbook` skill version 1.3.0 as an alleged VirusTotal false positive.

## Best Possible Solution

The published cashbook package should be corrected to either include `scripts/export.py` or remove all export references, and it should clarify whether screenshot OCR can involve external processing despite the local-only wording. After a fresh scan, staff can clear the verdict with the moderator override path and an audit note if the remaining finding is confirmed as a false positive.
