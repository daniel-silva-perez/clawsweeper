## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only auto-closes ClawHub pull requests that current main definitely implements. Substantively, the live cashbook 1.3.0 record is still suspicious: static and VirusTotal scans are clean, but the active ClawScan/LLM result remains suspicious because the package references a missing export script and makes a local-only claim while using screenshot OCR/image tooling.

## What This Changes

The issue asks maintainers to review and clear the suspicious install flag for cashbook v1.3.0 as a VirusTotal false positive.

## Best Possible Solution

Keep this issue open for maintainer/security moderation follow-up. The publisher should publish a corrected cashbook package that either includes scripts/export.py or removes export references, and clarifies whether screenshot OCR can involve external image processing despite the local-only wording. After that, staff can request or wait for a fresh scan; if the remaining finding is confirmed as a false positive, a moderator can clear it with setSkillManualOverride and an audit note.
