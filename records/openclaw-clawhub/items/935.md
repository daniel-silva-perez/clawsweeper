Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only allows auto-closing PRs that are certainly implemented on current main. Substantively, the live ClawHub API still reports cashbook 1.3.0 as suspicious: VirusTotal and static scans are clean, but the current LLM moderation verdict remains suspicious because of a missing referenced export script and a local-only/privacy claim that conflicts with screenshot OCR instructions. A small follow-up should also make the CLI warning identify the actual scanner/reason instead of always naming VirusTotal Code Insight.
