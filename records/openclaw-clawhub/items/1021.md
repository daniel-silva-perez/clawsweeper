## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Substantively, current main has added scanner detail pages and owner/API rescan paths, but the author-facing product gap remains: the main security UI still presents ClawScan/OpenClaw scanner failures as a bare Error, public docs do not explain the Error state or rescan workflow, and moderation still requires explicit clean external scanner signals before suppressing allowlisted static findings. The related false-positive/rescan workflow issue #669 is also still open.

## Best Possible Solution

Keep this issue open for maintainer/security triage. The best path is to add clear author-facing UI and public docs for what ClawScan/OpenClaw Error means, when authors should wait versus request a rescan, what information is safe to expose from stored scanner error summaries, and how moderation should distinguish scanner-side failures from actual suspicious evidence. Coordinate the rescan and false-positive handling with #669.
