## Summary

Keep open. This is a ClawHub issue, and the cleanup policy only permits auto-closing ClawHub PRs that current main definitely implements. The underlying product gap also remains on current main: OpenClaw scanner errors still render as a bare Error, stored LLM error summaries are hidden from the scan panel, API behavior still treats LLM scanner failures as overall security errors, and docs do not explain what authors should do next.

## Best Possible Solution

Keep this issue open for maintainer triage. Add scan-page and public docs guidance explaining what OpenClaw `Error` means, expose a safe scanner failure explanation or stored summary to authors, document whether authors should wait, change the skill, or request a rescan, and define moderation behavior that separates scanner-side failures from actual suspicious evidence. Coordinate with the broader false-positive/rescan workflow in #669.
