## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy for this lane only permits auto-closing ClawHub pull requests that current main certainly implements. Substantively, current main still leaves ClawScan `Error` mostly unexplained: the main skill security panel shows a bare error badge, hides stored LLM error summaries, public docs do not define the error/remediation path, and the scanner-side failure vs static-findings moderation policy still needs maintainer/security judgment. The related false-positive thread #669 remains open and should be coordinated with this issue.

## Best Possible Solution

Keep this issue open for maintainer/security triage. The best path is to define safe author-facing ClawScan `Error` semantics, add sanitized guidance to the skill security UI and scanner detail page, document owner/API/CLI rescan and review paths, and make an explicit policy decision for scanner-side LLM failures when static findings also exist. Coordinate with #669 so false-positive and rescan handling stay consistent.
