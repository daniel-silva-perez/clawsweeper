## Summary

Keep open. ClawHub policy only permits auto-closing PRs that are certainly implemented on main, and current main still does not make ClawScan/OpenClaw `Error` actionable for skill authors: the UI shows a bare destructive Error state, primary skill-page details hide stored error summaries, docs do not explain remediation, and scanner-error moderation semantics still need security/product judgment alongside #669.

## What This Changes

The issue requests actionable documentation and UI/policy guidance for OpenClaw/ClawScan Error scan states that can block skill installs.

## Best Possible Solution

Keep this open for maintainer/security triage. Define safe author-facing ClawScan/OpenClaw `Error` semantics, add sanitized guidance to the skill security summary and scanner detail page, document web/API/CLI rescan and appeal paths, and make an explicit moderation policy for scanner-side failures when static findings exist. Coordinate the outcome with #669 so false-positive and rescan handling stay consistent.
