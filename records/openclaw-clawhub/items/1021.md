## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main has partial rescan support and scanner detail pages, but the author-facing gap remains: ClawScan/OpenClaw scanner failures still surface as a terse Error, the main security panel hides stored error summaries, public docs do not explain the Error state or rescan workflow, and moderation still requires explicit clean external scanner results before suppressing allowlisted static findings. The related false-positive/rescan workflow report #669 remains open in the provided context.

## Best Possible Solution

Keep this issue open for maintainer/security triage. The best path is to add clear author-facing UI and public docs for what ClawScan/OpenClaw Error means, when authors should wait versus request a rescan, what information can safely be exposed from stored scanner error summaries, and how moderation should distinguish scanner-side failures from suspicious evidence. Coordinate the rescan and false-positive handling with #669.
