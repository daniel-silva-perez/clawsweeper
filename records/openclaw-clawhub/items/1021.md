## Summary

Keep open. This is an open ClawHub issue, and the repository policy for this review only permits auto-closing ClawHub pull requests that are certainly implemented on current main. The product gap also remains on current main: the scan panel still shows OpenClaw scanner failures as a bare Error, hides stored LLM error summaries, and docs do not explain the author workflow for scanner-side failures or rescans. The related false-positive/rescan issue #669 is also still open.

## Best Possible Solution

Keep this issue open for maintainer triage. Add scan-page and public docs guidance explaining what OpenClaw Error means, expose a safe scanner failure explanation or stored summary to authors, document whether authors should wait, change the skill, or request a rescan, and define moderation behavior that separates scanner-side failures from actual suspicious evidence. Coordinate the false-positive/rescan workflow with #669.
