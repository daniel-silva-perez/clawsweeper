## Summary

Keep open. This is an open ClawHub issue, and the repository policy for this review only permits auto-closing ClawHub pull requests that are certainly implemented on current main. The product gap also remains: current main still renders OpenClaw scanner failures as a bare Error, hides stored scanner error summaries from the scan panel, normalizes LLM scanner failures to overall API error status, and lacks author-facing docs for what to do next.

## Best Possible Solution

Keep this issue open for maintainer triage. Add scan-page and public docs guidance explaining what OpenClaw Error means, expose a safe scanner failure explanation or stored summary to authors, document whether authors should wait, change the skill, or request a rescan, and define moderation behavior that separates scanner-side failures from actual suspicious evidence. Coordinate the author-facing false-positive/rescan workflow with #669.
