## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy for this lane only allows auto-closing ClawHub pull requests that current main certainly implements. Current main has partial scanner detail pages and owner/API rescan support, but ClawScan error states still render as a terse Error, the primary skill security panel hides stored error summaries for error states, public docs do not explain the Error state or remediation path, and scanner-side failure versus static suspicious findings still needs security/product judgment. The related false-positive/moderation thread #669 remains open and should be coordinated with this issue.

## Best Possible Solution

Keep this issue open for maintainer/security triage. The best path is to add safe author-facing ClawScan Error semantics in the skill security UI and public docs, surface a sanitized stored error summary or next-step guidance when appropriate, document owner/API/CLI rescan paths, and make an explicit moderation decision for scanner-side LLM failures versus static suspicious findings. Coordinate the policy and copy with #669 so false-positive and rescan handling stay consistent.
