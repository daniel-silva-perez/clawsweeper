## Summary

Keep open. This is an open issue, and the repository cleanup policy for ClawHub only allows automatic closure for pull requests that current main definitely implements. Substantively, current main still shows ClawScan/OpenClaw `Error` as a bare failure state without a clear author-facing contract, remediation path, or settled moderation behavior for scanner-side failures.

## What This Changes

The issue asks ClawHub to explain the OpenClaw/ClawScan `Error` scan status, provide author remediation and rescan guidance, and decide whether scanner-side failures should mark skills suspicious or block installs.

## Best Possible Solution

Define a clear ClawScan/OpenClaw `Error` contract, expose sanitized failure details and next steps on the skill page and scanner detail page, document web/API/CLI rescan and appeal paths, and explicitly decide how scanner-side failures interact with static suspicious findings, coordinated with #669.
