## Summary

Current main still does not implement the admin moderation-hold recovery path proposed here. The hold placement path remains active, `unbanUser` does not clear `requiresModerationAt` or `requiresModerationReason`, and main has no `liftModerationHold` mutation, moderation-lift restore batch, `user.moderation.lift` audit action, or docs for lifting holds. Since this open PR's intended change is not already shipped on main, it should stay open for maintainer review.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a corrected implementation, or an equivalent admin-only recovery path, that clears user moderation hold fields, restores only eligible `user.moderation` skills, preserves malicious/manual/ban states, records an audit trail, handles batch races, keeps pending scan work in the pipeline, and documents the Convex CLI operation.
