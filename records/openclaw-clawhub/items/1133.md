## Summary

Current main still does not implement the PR's intended moderation-hold recovery path. The repository continues to set and enforce user moderation holds via `requiresModerationAt` / `requiresModerationReason`, but there is no `liftModerationHold` API, no `user.moderation.lift` audit path, no restoration batch for `user.moderation` skills, and no docs for lifting holds. Because this is an open PR and ClawHub policy only permits auto-closing PRs when current main definitely implements the change, this PR should stay open for maintainer review rather than be closed.

## Best Possible Solution

Keep this PR open for maintainer review, or land a corrected equivalent implementation that clears `requiresModerationAt` and `requiresModerationReason`, restores only skills that should become visible after the relevant user-level hold is lifted, preserves malicious/manual/ban states, keeps unresolved scan states in the scan pipeline, records accurate audit data across batches, handles hide/restore races, and documents the admin Convex CLI operation.
