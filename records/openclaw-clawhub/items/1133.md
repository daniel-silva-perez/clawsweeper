## Summary

Keep this PR open. Current main still has the moderation-hold creation and enforcement path, but no equivalent `liftModerationHold` recovery API that clears `requiresModerationAt` / `requiresModerationReason` and restores eligible `user.moderation` skills. Because this is a ClawHub PR and main does not already implement the intended change, it should remain an active maintainer-review candidate.

## Best Possible Solution

Keep this PR open for maintainer review, or land a corrected equivalent implementation that clears `requiresModerationAt` and `requiresModerationReason`, restores only skills that should become visible after the specific user-level hold is lifted, preserves malicious/manual/ban states, keeps unresolved scans in the scan pipeline, records accurate audit data across batches, handles hide/restore races, and documents the admin Convex CLI operation.
