## Summary

Keep this PR open. Current main still does not implement the proposed moderation-hold recovery path: holds are placed and affect future publishes, `unbanUser` does not clear the hold fields, and there is no `liftModerationHold` mutation, moderation-lift restore batch, audit action, docs section, or tests on main. Under the ClawHub policy, an open PR should only be auto-closed when main already implements its intended change.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a corrected equivalent admin-only recovery flow that clears moderation-hold fields, restores only eligible `user.moderation` skills, preserves malicious/manual/ban states, keeps unresolved scan work in the moderation pipeline, records an audit trail, handles batch races, and documents the Convex CLI operation.
