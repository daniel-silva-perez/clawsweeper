## Summary

Keep this PR open. Current main still lacks the proposed moderation-hold recovery API: it can place a user under moderation, hide owned skills, and keep future publishes hidden, but there is no `liftModerationHold` mutation or equivalent path that clears `requiresModerationAt` / `requiresModerationReason` and restores eligible skills. Because this is a ClawHub PR and main does not already implement the intended change, it is an active implementation candidate rather than cleanup-close material.

## Best Possible Solution

Keep this PR open for maintainer review, or land a corrected equivalent implementation that clears `requiresModerationAt` and `requiresModerationReason`, restores only skills hidden by the relevant user-level hold, preserves malicious/manual/ban states, keeps unresolved scans in the queue, records accurate audit data across batches, handles hide/restore races, and documents the admin Convex CLI operation.
