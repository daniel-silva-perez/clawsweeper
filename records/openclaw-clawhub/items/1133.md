The existing code still sets requiresModerationAt/requiresModerationReason, hides owned skills, and keeps future publishes hidden while the hold is present; the only nearby recovery flow is unban handling, which does not clear moderation-hold fields. Because this is an open ClawHub PR and main does not already implement the intended change, it should stay open for maintainer review or a corrected equivalent implementation.

## Best Possible Solution

Keep this PR open for maintainer review, or land a corrected equivalent admin-only recovery flow that clears requiresModerationAt and requiresModerationReason, restores only eligible skills hidden by the current user-level hold, preserves malicious/manual/ban states, keeps unresolved scans in the moderation pipeline, records accurate audit data across batches, handles hide/restore races, and documents the Convex CLI operation.
