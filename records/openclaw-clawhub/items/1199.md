## Summary

Keep this issue open. The report is concrete, ClawHub cleanup policy keeps issues open, and current main still routes the Settings Save button through a Convex mutation that patches the user and then awaits personal-publisher synchronization in the same transaction. The synchronization helper still has handle-conflict paths that can throw after the profile patch, so the observable server-error save failure remains plausible and not implemented away.

## What This Changes

The issue requests that personal Settings page display name and bio changes save successfully instead of failing with a Convex server error.

## Best Possible Solution

Persist personal displayName and bio updates even when recoverable personal-publisher synchronization conflicts occur, while leaving publishing, ownership, and true data-integrity failures strict and covered by focused Convex tests.
