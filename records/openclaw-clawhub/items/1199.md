## Summary

Keep this issue open. The ClawHub cleanup policy for this review pass only permits auto-closing pull requests that are certainly implemented on current main, and this item is an open issue. The current source also still matches the reported Settings save failure surface: the UI directly awaits the profile mutation, and that mutation still couples the profile save to personal publisher synchronization that can throw a server-side Convex error.

## Best Possible Solution

Keep this open for maintainer triage. Reproduce with the reporter's account or Convex Request ID, then harden convex/users.ts and convex/lib/publishers.ts so personal display name/bio saves either succeed independently of personal publisher synchronization or return a clear handled error. Add focused tests for the Settings personal save path, publisher conflict/error behavior, and the UI error state.
