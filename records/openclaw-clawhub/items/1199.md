## Summary

Keep this issue open. The ClawHub cleanup policy only permits auto-closing pull requests that are certainly implemented on current main, and this item is an open issue. The current source also still matches the reported failure surface: the Settings form directly awaits `users.updateProfile`, and that mutation still synchronizes the personal publisher in the same transaction through branches that can throw a Convex server error for handle conflicts or publisher-state problems.

## Best Possible Solution

Keep this open for maintainer triage. Reproduce with the reporter's account or Convex Request ID, then harden `convex/users.ts` and `convex/lib/publishers.ts` so personal display name/bio saves either succeed independently of personal publisher synchronization or return a clear handled error. Add focused coverage for the Settings personal save success path, publisher conflict/error behavior, and the UI error state.
