## Summary

Keep open. Repository policy for openclaw/clawhub says automated cleanup should keep issues open, and this is a concrete Settings Save failure report rather than a PR. Substantively, current main still routes the personal settings form through users.updateProfile, which patches the user and then runs personal publisher synchronization that can throw a server-side handle-conflict ConvexError; the Settings UI still has no focused save/error regression coverage.

## Best Possible Solution

Keep this issue open for maintainer triage. Reproduce against the reported account or Convex Request ID, then harden convex/users.ts and convex/lib/publishers.ts so personal display name/bio saves either complete successfully or return a clear handled error when personal publisher synchronization hits a handle conflict. Add focused regression coverage for the Settings Save success path, the conflict/error path, and UI error handling in src/routes/settings.tsx.
