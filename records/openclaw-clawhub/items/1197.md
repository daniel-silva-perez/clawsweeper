## Summary

Keep this issue open. The repository policy for openclaw/clawhub only permits automated closure for pull requests that are certainly implemented on current main, and this item is an issue. Source review shows current main partially mitigates the reported first-publish failure by scheduling owner-publisher package and skill digest syncs as separate background mutations, but the named skill digest helper still loops over `.paginate()` inside a single internal mutation for multi-page owners. The related duplicate candidate #1195 is also still open.

## Best Possible Solution

Keep this issue open for maintainer handling under the ClawHub policy. The practical fix is to verify production is running the scheduled digest-sync mitigation, decide whether #1195 should be the canonical duplicate, and change the owner-publisher digest rebuild helpers to process one cursor page per scheduled mutation invocation if multi-page owners can still hit the Convex pagination rule.
