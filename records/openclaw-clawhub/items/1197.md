## Summary

Keep open. Repository policy keeps ClawHub issues open, and current main still contains the reported Convex failure shape: publish can create or update a personal publisher, publisher writes schedule owner-publisher digest sync, and the package and skill owner-publisher digest helpers still loop over `.paginate()` inside one internal mutation. Related reports such as #1195, #1171, and #1184 remain open and should be consolidated after one backend fix lands.

## What This Changes

Publishing a skill can fail on first publish with Convex's “multiple paginated queries” error in the owner-publisher skill search digest sync path.

## Best Possible Solution

Implement cursor-rescheduled owner-publisher digest rebuilds so each internal mutation processes at most one indexed page, schedules a continuation with the returned cursor when needed, and covers both package and skill digest rebuilds with regression tests before consolidating the duplicate publish-failure reports.

## Repair Work Prompt

Fix the Convex owner-publisher digest rebuild pagination bug behind the publish/import failures reported in this issue and related reports #1195, #1171, #1184, #1176, #1175, #1182, #1201, and #1205. In `convex/functions.ts`, refactor `syncPackageSearchDigestsForOwnerPublisherIdInternal` and `syncSkillSearchDigestsForOwnerPublisherIdInternal` so each mutation invocation performs at most one `.paginate()` call, processes that indexed page, and schedules a continuation with the returned cursor when `isDone` is false. Cover both package and skill owner-publisher digest rebuilds. Do not replace the loops with unbounded `.collect()`, do not remove the existing indexed `by_owner_publisher` query shape, and keep unrelated publish behavior unchanged. Add focused regression tests in `convex/functions.test.ts` that simulate multi-page package and skill owner-publisher digest rebuilds and assert continuation scheduling instead of repeated pagination in one invocation. Run the targeted test and the repo checks listed in the validation plan.
