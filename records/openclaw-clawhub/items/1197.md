## Summary

Keep this issue open. The ClawHub cleanup policy for this repository only permits automated closure for pull requests that are certainly implemented on current main, and this item is an issue. The current code also only partially mitigates the reported first-publish failure: publisher-trigger digest work is scheduled into separate mutations, but the named skill owner-publisher digest helper still loops over `.paginate()` inside one internal mutation invocation.

## Best Possible Solution

Keep this issue open for maintainer handling. The code fix should convert the owner-publisher package and skill digest rebuild helpers into cursor-based self-scheduling batches that perform at most one `.paginate()` call per mutation invocation, then decide whether #1195 should be the canonical duplicate for the remaining publish-pagination work.
