## Summary

Keep this issue open. ClawHub cleanup policy keeps issues open in this pass, and current main still has the Convex pagination failure shape: the publish path can create or repair a personal publisher, that publisher write schedules owner-publisher digest sync work, and the package/skill owner-publisher digest helpers still loop over `.paginate()` inside one internal mutation invocation.

## Best Possible Solution

Keep this issue open and land one narrow backend fix for the publish-pagination cluster. Convert the package and skill owner-publisher digest sync internal mutations to cursor-based self-scheduling batches that perform at most one `.paginate()` per invocation, preserve indexed `by_owner_publisher` reads, add multi-page regression tests, and then let maintainers close duplicate reports such as #1195 after the fix is merged and deployed.
