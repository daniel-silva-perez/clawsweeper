## Summary

Keep this issue open. Current main still has the reported Convex failure shape: the publish path can ensure or repair a personal publisher, publisher writes schedule owner-publisher digest sync work, and both owner-publisher digest helpers still loop over `.paginate()` inside one internal mutation invocation. The existing scheduling split helps small owner catalogs but does not fix owners with more than one page of packages or skills.

## Best Possible Solution

Keep this issue open and land one narrow backend fix for the publish-pagination cluster. Convert the package and skill owner-publisher digest sync internal mutations to cursor-based self-scheduling batches that perform at most one `.paginate()` per invocation, preserve indexed `by_owner_publisher` reads, add multi-page regression tests, and then close duplicate reports after the fix is merged and deployed.
