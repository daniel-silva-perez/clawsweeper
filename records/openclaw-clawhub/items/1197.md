## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy keeps issues open unless they are PRs already implemented on current main. Substantively, current main still matches the reported failure shape: first publish can create or ensure a personal publisher, publisher writes schedule owner-publisher digest sync jobs, and both owner-publisher digest helpers still loop over `.paginate()` inside one internal mutation invocation.

## What This Changes

The issue reports first publish failing with a Convex error because owner-publisher digest sync can run multiple paginated queries in one function invocation.

## Best Possible Solution

Land one narrow backend fix for the publish-pagination cluster: make owner-publisher package and skill digest sync internal mutations process at most one paginated page per invocation, pass continuation cursors through self-scheduled follow-up mutations, preserve indexed `by_owner_publisher` reads, add multi-page regression tests, then consolidate duplicate reports such as #1195 and #1171 after the fix ships.

## Repair Work Prompt

Fix the ClawHub publish-pagination cluster reported here and in related reports. Current main still lets `syncPackageSearchDigestsForOwnerPublisherId` and `syncSkillSearchDigestsForOwnerPublisherId` loop over `.paginate()` inside one internal mutation, which can trigger Convex's multiple paginated queries error during first publish/publisher setup. Update the owner-publisher package and skill digest sync internal mutations in `convex/functions.ts` so each invocation performs at most one paginated query, accepts/passes a continuation cursor, and self-schedules the next page with `ctx.scheduler.runAfter` when `isDone` is false. Preserve the indexed `by_owner_publisher` reads and do not replace pagination with unbounded `.collect()`. Add focused regression tests in `convex/functions.test.ts` covering multi-page package and skill owner-publisher digest sync and the scheduled continuation args. If Convex generated API/types change, regenerate and commit only the necessary generated files. Do not broaden the fix into unrelated publish, search ranking, package catalog, or publisher refactors.
