## Repair Work Prompt

Fix the ClawHub publish/import Convex pagination cluster without broad refactors. Current main still has `syncPackageSearchDigestsForOwnerPublisherId` and `syncSkillSearchDigestsForOwnerPublisherId` looping over `.paginate()` inside one internal mutation, which can trigger Convex's “multiple paginated queries” error on first publish when publisher creation schedules owner-publisher digest sync jobs. Update the owner-publisher package and skill digest sync internals to process at most one paginated page per invocation, pass a continuation cursor through self-scheduled follow-up mutations, and preserve the indexed `by_owner_publisher` reads. Add focused multi-page regression tests in `convex/functions.test.ts` proving only one `.paginate()` call happens per invocation and that follow-up work is scheduled with the cursor. Inspect sibling digest helpers for the same trigger-visible pattern, but keep the PR narrowly scoped to the publish-pagination failure. Do not replace pagination with unbounded `.collect()`, do not change unrelated publish behavior, and do not make UI changes. If Convex function signatures change, regenerate and commit the relevant `convex/_generated` updates. No changelog entry is required unless the repository has an existing unreleased changelog section for backend bug fixes; otherwise document the fix in the PR summary.
