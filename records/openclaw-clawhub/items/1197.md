Keep this issue open. The ClawHub cleanup policy for this repository only permits automated closure for pull requests that are certainly implemented on current main, and this item is an issue. Current main also still contains the remaining failure shape: owner-publisher package and skill digest rebuild helpers each loop over `.paginate()` inside one internal mutation invocation, even though publisher trigger work is now scheduled into separate package and skill background mutations.

## Best Possible Solution

Keep this issue open for maintainer handling. Convert `syncPackageSearchDigestsForOwnerPublisherId` and `syncSkillSearchDigestsForOwnerPublisherId` into cursor-based self-scheduling batches that perform at most one `.paginate()` call per mutation invocation, add regression tests for multi-page owners, and then decide whether #1195 should be the canonical duplicate for the remaining publish-pagination work.
