## Summary

Keep this issue open. The repository cleanup policy for openclaw/clawhub keeps issues open in this pass, and current main still has the remaining Convex failure shape: the owner-publisher package and skill digest sync helpers loop over `.paginate()` inside one internal mutation invocation. The publisher trigger now schedules package and skill digest syncs as separate background mutations, which mitigates the inline trigger path, but it does not make multi-page owner rebuilds single-pagination-safe.

## Best Possible Solution

Keep this issue open and fix the remaining Convex pagination bug in the owner-publisher digest rebuild path. Convert the package and skill owner-publisher digest sync internal mutations to cursor-based self-scheduling batches that perform at most one `.paginate()` per invocation, add regression tests for multi-page owners, and then use #1195 or another maintainer-chosen issue as the canonical duplicate target for the cluster.
