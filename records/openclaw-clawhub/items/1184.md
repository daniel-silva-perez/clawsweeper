## Summary

Keep this issue open. ClawHub cleanup policy keeps issues open in this pass, and current main still leaves the owner-publisher package and skill digest helpers able to run repeated `.paginate()` calls inside one internal mutation invocation. The existing scheduling change mitigates the original same-trigger double-pagination path for small owners, but it does not prove the multi-page owner case is fixed.

## Best Possible Solution

Keep this issue open and land one focused Convex fix for the duplicate cluster. The durable implementation is to make owner-publisher package and skill digest sync process one indexed page per scheduled internal mutation, pass cursor and batch size through the internal args, reschedule while `isDone` is false, and add regression tests proving the multi-page path never performs multiple `.paginate()` calls in one invocation. After that merges and deploys, maintainers can consolidate or close the duplicate reports against the canonical fix.
