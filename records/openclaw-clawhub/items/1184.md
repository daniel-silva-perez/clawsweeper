## Summary

Keep this issue open. It is an open ClawHub issue, and the repository cleanup policy only permits auto-closing PRs that are certainly implemented on current main. Substantively, the reported Convex failure shape still exists: the owner-publisher skill digest rebuild can call `.paginate()` repeatedly inside one internal mutation invocation, and the sibling package digest rebuild has the same pattern.

## Best Possible Solution

Keep this open and land a focused Convex fix for the owner digest rebuild cluster: make the owner-publisher package and skill digest internal mutations cursor-aware, process exactly one paginated page per invocation, and schedule a continuation with `continueCursor` when needed. Audit the owner-user package sync sibling as part of the same PR because it has the same loop shape.
