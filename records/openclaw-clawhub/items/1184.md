## Summary

Keep open. Current main still has the reported Convex failure shape: `syncSkillSearchDigestsForOwnerPublisherId` can call `.paginate()` repeatedly inside one internal mutation, and the owner-publisher package digest helper has the same pattern. The existing scheduler split is only partial mitigation because the internal mutations do not accept or schedule continuation cursors.

## What This Changes

Publishing via the CLI can fail because owner-publisher skill digest sync may run multiple Convex paginated queries inside one function invocation.

## Best Possible Solution

Keep this open and land one focused backend fix that makes owner-publisher package and skill digest rebuilds cursor-batched: each internal mutation should perform at most one indexed `.paginate()` call, process that page, and schedule the same internal mutation with `continueCursor` when more rows remain. Preserve missing-table fallbacks and digest semantics, add multi-page tests for both helpers, then consolidate the related reports after the fix merges.
