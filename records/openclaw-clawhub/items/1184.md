## Summary

Keep open. Current main still contains the reported Convex limit violation shape: the owner-publisher skill digest helper can execute repeated `.paginate()` calls inside one internal mutation, and the scheduled wrapper does not carry cursor state. The package digest sibling has the same remaining pattern, so this is a valid narrow backend bug rather than cleanup that can be closed.

## What This Changes

Publishing through the ClawHub CLI fails when owner-publisher search digest sync runs multiple paginated Convex queries in one function invocation.

## Best Possible Solution

Refactor owner-publisher package and skill digest rebuilds so each scheduled internal mutation performs at most one indexed `.paginate()` call, processes that page, and reschedules itself with `continueCursor` until complete while preserving missing-table fallback behavior and digest semantics.

## Repair Work Prompt

Fix the Convex multiple-pagination publish failure for owner-publisher digest sync. In `convex/functions.ts`, refactor both `syncPackageSearchDigestsForOwnerPublisherId` and `syncSkillSearchDigestsForOwnerPublisherId` plus their internal scheduled wrappers so each internal mutation performs at most one indexed `.paginate()` call, processes that page, and reschedules itself with `continueCursor` until complete. Preserve existing missing-table fallbacks and digest update semantics. Cover both package and skill owner-publisher digest paths; do not replace pagination with an unbounded full-table or owner-wide `collect()` unless maintainers explicitly approve that tradeoff. Add focused tests in `convex/functions.test.ts` that prove only one page is processed per invocation and unfinished pages schedule continuation work. Keep the change scoped to this backend bug; no CLI UX, docs, schema, or unrelated search refactors unless required by the implementation.
