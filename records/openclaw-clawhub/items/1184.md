## Summary

Keep this issue open. The ClawHub cleanup policy does not auto-close issues, and current main only partially mitigates the reported publish failure: publisher changes now schedule package and skill digest syncs as separate background mutations, but the named skill helper still loops over `.paginate()` inside one internal mutation invocation.

## Best Possible Solution

Keep this issue open for maintainer triage. The durable fix is to refactor the owner-publisher package and skill digest sync internals to accept cursor and batch-size arguments, process one page per scheduled internal mutation, reschedule until complete, and add regression coverage for multi-page publisher-backed publish/digest sync behavior. After that lands, maintainers can manually close or consolidate the related duplicate reports.
