## Summary

Keep open. This is an open ClawHub issue, and current main still contains the reported Convex limit violation shape: owner-publisher digest rebuilds can execute repeated `.paginate()` calls inside one internal mutation, while the scheduled internal wrappers do not carry continuation cursor state.

## What This Changes

Publishing via the ClawHub CLI can fail with Convex's multiple-paginated-query error during owner-publisher skill digest synchronization.

## Best Possible Solution

Make owner-publisher package and skill digest rebuilds cursor-batched: each scheduled internal mutation should perform at most one indexed `.paginate()` call, process that page, and reschedule itself with `continueCursor` until complete while preserving missing-table fallbacks and digest semantics.
