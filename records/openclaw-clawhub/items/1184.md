## Summary

Keep this issue open. The repository cleanup policy for openclaw/clawhub says issues are not eligible for auto-close in this pass, and current main still leaves the named owner-publisher skill digest helper looping over `.paginate()` calls within one internal mutation invocation. Current main does include a useful mitigation by scheduling package and skill publisher digest syncs separately, but that does not prove the multi-page owner case is fixed.

## Best Possible Solution

Keep this issue open for maintainer triage or consolidation with the canonical duplicate. The durable fix is to refactor the owner-publisher package and skill digest sync internals to accept cursor and batch size, process one page per scheduled internal mutation, reschedule until complete, and add regression coverage proving multi-page publisher digest sync never runs multiple paginated queries in a single Convex function invocation.
