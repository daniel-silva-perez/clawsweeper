## Summary

Keep open. This is an open ClawHub issue, and current main still matches the reported Convex failure shape: the owner-publisher skill digest helper can call `.paginate()` repeatedly inside one internal mutation, the package sibling has the same remaining pattern, and the scheduler split does not pass or schedule continuation cursors. Existing focused tests verify only that package and skill rebuilds are split into two background mutations, not that each invocation processes a single page and reschedules.

## What This Changes

Publishing through the CLI can fail because `syncSkillSearchDigestsForOwnerPublisherId` loops over paginated Convex queries in one function invocation.

## Best Possible Solution

Implement cursor-batched owner-publisher digest rebuilds for both skill and package search digests so each scheduled mutation processes one indexed page, preserves existing digest and missing-table behavior, and self-schedules with `continueCursor` until complete.

In `convex/functions.ts`, make both `syncPackageSearchDigestsForOwnerPublisherIdInternal` and `syncSkillSearchDigestsForOwnerPublisherIdInternal` cursor-batched: each internal mutation invocation should perform at most one indexed `.paginate()` call, process that page, and schedule the same internal mutation with `continueCursor` when more rows remain. Preserve current digest semantics, indexes, missing-table fallbacks, and the existing scheduler entry point. Do not replace the loops with unbounded `.collect()`, do not broaden the publish flow, and do not change unrelated package/skill behavior.
