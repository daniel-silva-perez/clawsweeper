## Summary

Keep open. Current main still contains the reported Convex failure shape: owner-publisher skill digest sync can call `.paginate()` repeatedly inside one internal mutation, and the scheduled wrapper has no continuation cursor. The package digest sibling has the same remaining pattern, so the issue is valid and should be fixed rather than closed or consolidated yet.

## What This Changes

Publishing via the CLI can fail when owner-publisher digest rebuilds perform multiple paginated Convex queries in one function invocation.

## Best Possible Solution

Make owner-publisher package and skill digest rebuilds cursor-batched: each scheduled internal mutation should execute at most one indexed `.paginate()` call, process that page, and reschedule itself with `continueCursor` until complete while preserving missing-table fallbacks and digest semantics.
