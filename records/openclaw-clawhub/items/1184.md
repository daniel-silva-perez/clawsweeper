## Summary

Keep this issue open. Current main still leaves the reported Convex failure shape: the owner-publisher skill digest sync helper can call `.paginate()` repeatedly inside one internal mutation invocation, and the sibling package helper has the same issue. The scheduling split runs package and skill digest sync in separate background mutations, but it does not pass a cursor or batch size through those mutations, so multi-page owner catalogs can still hit Convex's single-paginated-query limit. Because this is an open ClawHub issue, repository policy also says not to auto-close it in this cleanup pass.

## Best Possible Solution

Keep this open and land one focused Convex fix for the publish/import pagination cluster: make the owner-publisher package and skill digest sync internal mutations process exactly one indexed page per invocation, accept and pass `cursor` and `batchSize`, reschedule themselves with `continueCursor` while `isDone` is false, and add regression tests proving the multi-page path never calls `.paginate()` more than once in a single mutation invocation.
