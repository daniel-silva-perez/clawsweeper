## Summary

Keep this issue open. Current main has owner/moderator/admin soft-delete, transfer, admin reclaim, owner-change, and hard-delete tooling, but the reporter’s request is still a concrete production ownership/support case: they say the original GitHub sub-account was deleted, so ordinary owner self-service may not work. The repository policy for this ClawHub review also says to keep issues open and only auto-close PRs that are already implemented on main.

After the production registry state is resolved, a maintainer can close this support request manually.
