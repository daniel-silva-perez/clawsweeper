## Summary

Keep this open. This is a concrete production registry deletion/ownership support request for four Gate skills, and the openclaw/clawhub cleanup policy only permits auto-closing PRs that are certainly implemented on main. Current main has CLI/API owner-or-staff soft-delete plus admin transfer/reclaim/hard-delete paths, but the thread has no confirmation that the current Gate account can delete skills owned by the deleted sub-account or that staff completed the cleanup.

## Best Possible Solution

Keep this issue open for maintainer/support follow-up. A maintainer should confirm the current production state with Gate, verify the exact slugs and owners for the four named skills, then either guide the authorized owner through `clawhub delete <slug>` or use the existing staff soft-delete, ownership change, reclaim, or hard-delete paths as appropriate. Close manually only after the cleanup or transfer is confirmed.
