## Summary

Keep open. The ClawHub cleanup policy for this review only allows auto-closing pull requests that are certainly implemented on current main, and this item is an issue. Substantively, current main still rejects package names that collide with skill slugs, and the scan/backfill code paths do not prove the hosted `claw-pay` plugin v0.2.5 was resynced or deleted.

## Best Possible Solution

Keep this issue open for maintainer handling. The best path is to resync or manually update the VirusTotal result for the stale `claw-pay` plugin v0.2.5, or soft-delete the obsolete plugin record if the replacement `claw-pay-plugin` should be canonical. Separately, maintainers should make an explicit product decision on whether the global package namespace should support a same-owner skill/plugin migration or override flow, coordinating with #1744 and #1272.
