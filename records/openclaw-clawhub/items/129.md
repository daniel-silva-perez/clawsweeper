## Summary

Keep open. The repository policy for this cleanup pass keeps ClawHub issues open, and the underlying mirror-persistence security problem is still observable: current main filters removed/hidden skills out of future backup pages and has a periodic pruning path, but deletion, report auto-hide, scanner-hidden, and ban flows do not deterministically remove the already-written GitHub mirror entry.

## What This Changes

The issue asks ClawHub to prevent malicious or removed skills from persisting in the public openclaw/skills GitHub mirror used by downstream marketplaces.

## Best Possible Solution

Keep this open until ClawHub has a deterministic mirror-removal or tombstone policy triggered by skill deletion, hidden/removed moderation, scanner-malicious blocking, report auto-hide, and user bans, with focused tests and a production cleanup plan for known stale malicious mirror entries such as bird-js.
