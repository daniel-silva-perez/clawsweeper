## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy for this pass only permits auto-closing ClawHub pull requests that current main certainly implements. The substantive mirror-persistence problem also still reproduces: current main filters hidden/deleted skills out of future GitHub backup pages and has a periodic pruning path, but it does not deterministically purge the GitHub mirror when a skill is hidden, deleted, scanner-blocked, report-hidden, or affected by a user ban.

## What This Changes

The issue asks ClawHub to prevent malicious or removed skills from persisting in the public openclaw/skills GitHub mirror where downstream marketplaces can continue to display and install them.

## Best Possible Solution

Keep this open until ClawHub has a deterministic mirror-removal or tombstone policy for skills that are deleted, hidden as malicious, auto-hidden by reports, scanner-blocked, or affected by user bans, plus focused tests and a production cleanup plan for known stale malicious mirror entries such as bird-js.
