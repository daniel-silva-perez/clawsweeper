## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and the security concern remains reproducible: ClawHub no longer serves the reported skill through its public APIs, but the public GitHub mirror still serves the mirrored SKILL.md. Current main filters hidden/deleted skills out of future backup pages and has batch pruning, but it does not deterministically remove already-mirrored content when a skill is deleted, hidden by reports/scanners, or hidden through user bans.

## What This Changes

The issue asks ClawHub to prevent removed malicious skills from persisting in the public openclaw/skills GitHub mirror that downstream marketplaces may index or install from.

## Best Possible Solution

Keep this open until ClawHub has a deterministic mirror-removal or tombstone policy triggered by skill deletion, hidden/removed moderation, scanner-malicious blocking, report auto-hide, and user bans, with focused tests and a production cleanup plan for known stale malicious mirror entries such as bird-js.
