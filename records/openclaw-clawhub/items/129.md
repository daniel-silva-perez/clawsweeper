## Summary

Keep open. The repository cleanup policy keeps ClawHub issues open, and the security concern is still observable: ClawHub no longer serves the reported skill through its public APIs, but the public GitHub mirror still serves the mirrored SKILL.md. Current main has filtering and batch pruning, but removed or hidden skills are not deterministically removed from the mirror at each deletion or moderation transition.

## What This Changes

The issue asks ClawHub to remove or stop persistently mirroring malicious or deleted skills in the openclaw/skills GitHub archive so downstream marketplaces cannot continue distributing them.

## Best Possible Solution

Keep this open until ClawHub has a deterministic mirror-removal or tombstone policy triggered by skill deletion, hidden/removed moderation, scanner-malicious blocking, report auto-hide, and user bans, with focused tests and a production cleanup plan for known stale malicious mirror entries such as bird-js.
