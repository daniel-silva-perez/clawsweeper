## Summary

Keep open. This is an open ClawHub issue, and the repository policy for this cleanup pass only permits auto-closing ClawHub pull requests that current main certainly implements. Substantively, the reported mirror-persistence problem still reproduces: ClawHub no longer serves the example skill through its public API, but the openclaw/skills GitHub mirror still serves the malicious SKILL.md, and current main still relies on periodic pruning rather than deterministic purge from delete, moderation, scanner, or ban transitions.

## What This Changes

The issue requests that malicious or removed skills no longer persist in the public openclaw/skills GitHub archive and downstream marketplaces after ClawHub deletion or moderation.

## Best Possible Solution

Keep this issue open for maintainer/security follow-up. The best path is deterministic mirror cleanup: purge or tombstone GitHub mirror entries when skills are deleted, hidden as malicious, auto-hidden by reports, or affected by user bans/autobans; add focused tests for direct mirror deletion and prune behavior; verify production github-backup-sync/prune health; manually remove known stale malicious entries such as bird-js; and coordinate downstream cache or advisory handling before closing.
