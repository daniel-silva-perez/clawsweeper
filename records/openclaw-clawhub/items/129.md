## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that current main certainly implements. Substantively, the reported mirror-persistence problem still reproduces: ClawHub returns 404 for the example skill, but the openclaw/skills GitHub mirror still serves its SKILL.md, and current main only has periodic pruning plus an unwired direct-delete helper rather than deterministic purge on delete, moderation, report auto-hide, scanner malicious verdicts, or user bans.

## What This Changes

Request deterministic cleanup or retirement of openclaw/skills mirror entries for skills removed or hidden as malicious on ClawHub, so downstream marketplaces cannot keep serving them as installable content.

## Best Possible Solution

Keep this issue open for maintainer/security follow-up. The best path is deterministic mirror cleanup: purge or tombstone GitHub mirror entries when skills are deleted, hidden as malicious, auto-hidden by reports, or affected by user bans/autobans; add focused tests around direct mirror deletion and prune behavior; verify production github-backup-sync/prune health; manually remove known stale malicious entries such as bird-js; and coordinate downstream cache/advisory handling before closing.
