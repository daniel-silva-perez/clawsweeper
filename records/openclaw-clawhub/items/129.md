## Summary

Keep open. Repository policy for this cleanup pass keeps ClawHub issues open, and the underlying security problem is still observable: ClawHub no longer serves the example skill through the public skill API/file API, but the GitHub mirror still serves the mirrored SKILL.md. Current main prevents hidden/deleted skills from being included in future backup pages and has periodic pruning, but removal is not deterministically wired into the delete, auto-hide, scanner-hidden, or ban transitions.

## What This Changes

The issue requests that skills removed or marked malicious in ClawHub no longer persist in the public openclaw/skills GitHub mirror used by downstream marketplaces.

## Best Possible Solution

Keep this open until ClawHub has a deterministic mirror-removal or tombstone policy triggered by skill deletion, hidden/removed moderation, scanner-malicious blocking, report auto-hide, and user bans, plus focused tests and a production cleanup plan for known stale malicious mirror entries such as bird-js.
