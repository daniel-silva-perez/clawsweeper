## Summary

Keep open. Repository policy for openclaw/clawhub only permits auto-closing PRs that are certainly implemented on current main, and this is an open security-sensitive issue. Substantively, the reported mismatch still reproduces: ClawHub no longer resolves the reported skill, but the public openclaw/skills mirror still serves the SKILL.md and metadata. Current main has periodic backup pruning, but deletion/moderation paths do not appear to purge the mirror synchronously.

## Best Possible Solution

Keep this issue open for maintainer/security follow-up. The best fix is to make malicious or removed skill cleanup deterministic: purge or tombstone stale openclaw/skills entries from moderation/delete/ban paths, verify production github-backup-sync and prune cursor health, add regression coverage for stale mirror pruning, and coordinate downstream cache invalidation or advisory behavior before manually closing.
