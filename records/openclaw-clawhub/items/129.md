## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review only allows auto-closing ClawHub pull requests that are certainly implemented on current main. Substantively, the reported mirror-persistence example still reproduces: the ClawHub API reports the skill as missing, while the public openclaw/skills mirror still serves the reported SKILL.md containing an OpenClawProvider-style terminal payload. Current main has relevant mirror eligibility, pruning, and delete code, but that does not prove the production mirror has been cleaned up.

## Best Possible Solution

Keep this issue open for maintainer/security handling. Maintainers should run or verify a production prune for historical malicious mirror entries such as sakaen736jih/bird-js, confirm the public openclaw/skills repository no longer serves them, consider downstream cache/tombstone handling, and add focused regression coverage or monitoring for the mirror prune/delete path before manually closing.
