## Summary

Keep open. This is an open ClawHub issue, and the repository policy only permits auto-closing ClawHub pull requests that current main certainly implements. Substantively, the reported mirror-persistence problem still reproduces: ClawHub no longer serves the example skill through its public API, but the openclaw/skills mirror still serves the malicious SKILL.md, and current main still depends on periodic pruning rather than deterministic purge from deletion, moderation, scanner, or ban paths.

## Best Possible Solution

Keep this issue open for maintainer/security follow-up. The best path is deterministic mirror cleanup: purge or tombstone GitHub mirror entries when skills are deleted, hidden as malicious, auto-hidden by reports, or affected by user bans/autobans; add focused tests for direct mirror deletion and prune behavior; verify production github-backup-sync/prune health; manually remove known stale malicious entries; and coordinate downstream cache or advisory handling before closing.
