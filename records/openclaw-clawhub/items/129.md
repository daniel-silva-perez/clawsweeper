## Summary

Keep open. This is a ClawHub issue, and the repository policy only allows auto-closing ClawHub pull requests that are certainly implemented on current main. The reported mismatch is also still observable: ClawHub returns 404 for `bird-js`, but the public `openclaw/skills` mirror still serves the archived malicious `SKILL.md` and `_meta.json`.

## Best Possible Solution

Keep this issue open for maintainer/security follow-up. Maintainers should remove or tombstone the stale malicious mirror entry, verify the production `github-backup-sync` job and credentials are healthy, consider directly scheduling backup deletion from moderation/delete/ban paths instead of relying only on periodic prune, add regression coverage around stale mirror pruning, and confirm downstream cache behavior before manually closing.
