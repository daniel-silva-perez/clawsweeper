## Summary

Keep open. This is an open ClawHub security-sensitive issue, and the repository policy only permits auto-closing ClawHub PRs that current main certainly implements. Substantively, the reported mismatch still reproduces: ClawHub no longer serves the reported skill through its public API, but the public GitHub mirror still serves the skill files, and current main relies on periodic mirror pruning rather than deterministic purge from delete/moderation/ban paths.

## Best Possible Solution

Keep this issue open for maintainer/security follow-up. The best path is to make malicious or removed skill cleanup deterministic: purge or tombstone openclaw/skills entries from moderation delete, report auto-hide, scanner malicious, autoban, and ban paths; add focused tests around mirror deletion/prune behavior; verify production github-backup-sync/prune health; manually remove known stale malicious entries; and coordinate downstream cache or advisory handling before closing.
