## Summary

Keep open. This is an open ClawHub issue, and the repository policy for this cleanup pass only allows auto-closing pull requests that current main certainly implements. The security report also remains substantively valid: the example skill is no longer served by ClawHub's public skill/file APIs, but the GitHub mirror still serves the mirrored SKILL.md, and current main relies on future filtering plus periodic pruning rather than deterministic mirror removal at every deletion or moderation transition.

## What This Changes

The issue asks ClawHub to remove malicious or deleted skills from the public openclaw/skills mirror, or stop using that mirror, so downstream marketplaces cannot continue distributing removed malware.

## Best Possible Solution

Keep this open until ClawHub has a deterministic mirror-removal or tombstone policy triggered by skill deletion, hidden/removed moderation, scanner-malicious blocking, report auto-hide, and user bans, with focused tests and a production cleanup plan for known stale malicious mirror entries such as bird-js.
