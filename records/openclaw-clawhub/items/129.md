## Summary

Keep open. This is a ClawHub issue, and the repository policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Substantively, the reported production symptom is still observable: ClawHub’s API returns 404 for `bird-js`, but the public `openclaw/skills` mirror still serves `skills/sakaen736jih/bird-js`, so downstream consumers can still index it.

## Best Possible Solution

Keep this issue open for maintainer/security follow-up. Maintainers should remove or tombstone the historical malicious entry from `openclaw/skills`, verify the scheduled prune job is healthy in production, add/extend regression coverage for stale mirror pruning if needed, and confirm downstream cache behavior before manually closing.
