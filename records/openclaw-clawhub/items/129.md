## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. The reported mismatch also still reproduces on April 29, 2026: ClawHub returns 404 for the reported skill, while the public openclaw/skills mirror still returns 200 and contains the reported OpenClawProvider/base64 curl-to-bash pattern.

## Best Possible Solution

Keep this issue open for maintainer/security follow-up. The best fix is to make malicious or removed skill cleanup deterministic: remove or tombstone stale openclaw/skills entries from moderation/delete/ban paths, verify the production `github-backup-sync` job and prune cursor health, add regression coverage around stale mirror pruning, and coordinate downstream cache invalidation or advisory behavior before manually closing.
