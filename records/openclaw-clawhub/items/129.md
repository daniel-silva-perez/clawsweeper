## Risks / Open Questions

- The public openclaw/skills mirror still serves the reported malicious SKILL.md and _meta.json while ClawHub’s own public API returns 404, so downstream marketplaces or installers that index the mirror can continue exposing it.
- Current main relies on a periodic backup sync/prune job for mirror cleanup. If production credentials, environment target, cursor state, batch progress, or GitHub write permissions are unhealthy, stale malicious entries can persist indefinitely.
- Moderation hide/delete paths do not directly schedule mirror deletion, so high-risk malicious removals are not immediately tombstoned in the GitHub mirror.
- Even after the mirror entry is removed, downstream caches may continue serving stale copies unless maintainers publish a tombstone/advisory or coordinate cache invalidation.
