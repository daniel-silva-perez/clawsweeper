## Summary

Keep open. Current main still renders package icons beside download totals in shared public skill stat/list components, while the public detail header only partially has the desired Download/Package split. The issue is also paired with open draft PR #1854, so it should remain open until that PR lands or is superseded.

## What This Changes

The issue requests consistent download-count iconography across ClawHub UI surfaces, preferably using a download/down-arrow icon for downloads and reserving package/box icons for version or package concepts.

## Best Possible Solution

Review and land or supersede PR #1854. The end state should use a download-specific lucide icon for public skill download metrics in `SkillStatsTripletLine`, `SkillMetricsRow`, and `SkillListItem`, while preserving `Package` for version/package concepts and avoiding stat or layout changes.
