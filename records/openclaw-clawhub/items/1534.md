## Summary

Keep open. Current main only partially resolves the reported inconsistency: the public skill detail header uses a Download icon for downloads, but shared public stat rows and public list items still render Package beside download counts. The issue also has an open draft PR (#1854) that is the active fix path, and ClawHub policy keeps issues open rather than auto-closing them.

## Best Possible Solution

Keep this issue open and review, land, or supersede #1854. The narrow code direction is to use a download-specific lucide icon such as Download or ArrowDownToLine for public skill download metrics in SkillStatsTripletLine, SkillMetricsRow, and SkillListItem, while keeping Package for versions/package concepts. Maintainers can separately decide whether the same convention should apply to SoulStats.
