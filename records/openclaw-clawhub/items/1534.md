## Summary

Keep open. Current main only partially resolves the icon inconsistency: the public skill detail header now uses a download icon, but shared public stat rows and public list items still render a package icon beside download counts. Also, #1854 is an open draft PR with closing syntax for this issue, so the issue should remain open until that PR is reviewed, landed, or superseded.

## Best Possible Solution

Keep this issue open and review, land, or supersede #1854. The narrow fix is to use a download-specific lucide icon such as Download or ArrowDownToLine for public skill download metrics in SkillStatsTripletLine, SkillMetricsRow, and SkillListItem, while keeping Package for versions/package concepts and avoiding stat calculation or layout behavior changes. Maintainers can separately decide whether SoulStats should follow the same convention.
