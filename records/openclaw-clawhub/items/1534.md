## Summary

Current main still has the reported UI inconsistency: dashboard download counts use a down-arrow icon, while public skill surfaces still render the package icon beside download counts. This is an issue, and the ClawHub cleanup policy keeps issues open rather than auto-closing them from this workflow.

## Best Possible Solution

Keep this issue open. Land a small core UI fix that standardizes download-count icons across dashboard and public skill surfaces, preferably using ArrowDownToLine or Download for downloads and reserving Package for versions/packages. PR #1550 can be used as prior art, but it needs to be revived or superseded because it was closed unmerged.
