## Summary

Keep open. Current main only partially addresses the inconsistency: the public skill detail header uses `Download` for downloads, but shared public stat rows and list-style public skill rows still render `Package` next to download counts. The issue is also linked to open draft PR #1854, which explicitly targets this fix, so the best path is to review, land, or supersede that PR rather than close the issue now.

## Best Possible Solution

Keep this issue open and review, land, or supersede PR #1854. The narrow fix is to use a download-specific lucide icon such as `Download` or `ArrowDownToLine` for public skill download metrics in `SkillStatsTripletLine`, `SkillMetricsRow`, and `SkillListItem`, while keeping `Package` for versions/package concepts and avoiding stat or layout changes.
