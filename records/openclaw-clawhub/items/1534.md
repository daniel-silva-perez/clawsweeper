## Summary

Keeping this issue open. Current main has partially addressed the public skill detail header by using a Download icon there, and the dashboard no longer renders the old download/version stat row, but shared public skill stats and list items still render Package for download counts. There is also an open draft PR (#1854) with closing syntax for this issue, so the right path is to review, land, or supersede that PR first.

## Best Possible Solution

Keep this issue open and review, land, or supersede #1854. The narrow product/code direction is to use a download-specific lucide icon such as Download or ArrowDownToLine for public skill download metrics in shared stat/list surfaces, while keeping Package for versions/package concepts. Maintainers may also decide whether the same icon convention should apply to SoulStats for cross-resource consistency.
