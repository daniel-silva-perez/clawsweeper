## Summary

Keep open. Current main does not implement post-install/update active-session event injection, and this ClawHub cleanup policy keeps issues open for maintainer handling. The request is concrete, but the remaining decision is a product and security boundary around allowing newly installed or updated skill content to influence an active OpenClaw session.

## What This Changes

Request to have `clawhub install` and `clawhub update` notify the active OpenClaw session to read the installed or updated `SKILL.md` and run setup/onboarding instructions.

## Best Possible Solution

Maintainers should define the supported install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with focused CLI/session tests and docs, or document the inspectable copy-prompt flow as the intended setup path.
