## Summary

Keep open. Current main does not implement post-install/update active-session event injection, and the ClawHub cleanup policy keeps issues open for maintainer handling. The request is concrete, but it crosses a product and security boundary around letting newly installed or updated skill content influence an active OpenClaw session.

## What This Changes

The issue requests that ClawHub install/update notify the active OpenClaw session so the agent reads the installed skill's SKILL.md and performs onboarding or setup steps.

## Best Possible Solution

Define the install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with CLI/session tests and docs, or document the current inspectable copy-prompt flow as the supported setup path.
