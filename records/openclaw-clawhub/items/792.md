## Summary

Keeping this issue open. Repository policy for openclaw/clawhub only allows automatic closure for PRs that are certainly implemented on main, and this item is an issue. Current main also still does not appear to implement the requested post-install/update active-session system event: the CLI install and update commands remain package-management flows, and the web install helper still relies on an explicit user-copied prompt rather than automatic session injection.

## Best Possible Solution

Keep this issue open for maintainer product/API triage. The best path is to define a safe, explicit active-session notification contract between ClawHub and OpenClaw, then implement post-success install/update signaling with tests and docs covering install, update, non-interactive CLI use, and user-consent boundaries.
