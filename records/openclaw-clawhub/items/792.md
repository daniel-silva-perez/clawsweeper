## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and this ClawHub cleanup policy keeps issues open for maintainer handling. The request is concrete, but it crosses a product and security boundary around newly installed or updated skill content influencing an active agent session.

## What This Changes

The issue requests an active-session system event after `clawhub install` or `clawhub update` so the agent reads the installed skill's `SKILL.md` and performs onboarding or update instructions.

## Best Possible Solution

The desired end state is an explicit install/update handoff contract: either a user-consented OpenClaw session path with security boundaries, focused CLI/session tests, and docs, or a documented decision that the inspectable copy-prompt flow is the supported setup path.
