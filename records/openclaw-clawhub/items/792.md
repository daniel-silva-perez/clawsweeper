## Summary

Keep open. Current main does not implement post-install/update active-session event injection: `clawhub install` and `clawhub update` still resolve/download/extract skill files and write metadata, while the current OpenClaw setup UX is an explicit copyable prompt surface whose approved spec rules out direct handoff or automatic execution. This is also an issue, and the supplied ClawHub review policy says issues should remain open for maintainer handling.

## What This Changes

Request to emit an active-session system event after `clawhub install` or `clawhub update` so OpenClaw reads the installed or updated `SKILL.md` and performs setup/onboarding instructions.

## Best Possible Solution

Keep this open for maintainer product/security/API triage. The best path is to decide whether ClawHub should ever notify an active OpenClaw session after CLI install/update; if yes, define an explicit user-consented session handoff contract and implement it with CLI/session tests, docs, and security-boundary coverage. Maintainers may also decide the current inspectable copyable-prompt UX is the intended boundary and document that decision.

## Risks / Open Questions

- Automatic active-session injection would allow newly downloaded `SKILL.md` instructions to influence an active agent after install/update, so the consent and trust boundary needs maintainer review before implementation.
- The ClawHub CLI currently has no visible active OpenClaw session or handoff API, so a real implementation may require an OpenClaw-side contract rather than a narrow ClawHub-only patch.
- The issue mentions `postUpdate` hooks in skill metadata, but the current skill metadata schema/docs do not define `postInstall` or `postUpdate`; maintainers should clarify whether the desired contract is hook execution, SKILL.md setup guidance, or both.
