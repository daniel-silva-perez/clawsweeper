## Summary

Keep open. Current main does not implement active-session event injection after ClawHub CLI install/update, and for this repository the cleanup policy keeps issues open for maintainer handling. The remaining question is a product/security/API boundary: whether ClawHub should be allowed to notify or influence an active OpenClaw session after installing updated skill content.

## What This Changes

The issue requests that `clawhub install` and `clawhub update` inject a system event into the active agent session so OpenClaw reads the installed skill's `SKILL.md` and performs setup/onboarding instructions.

## Best Possible Solution

Define the intended install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with CLI/session tests and docs, or document the current inspectable copyable-prompt flow as the supported setup path.
