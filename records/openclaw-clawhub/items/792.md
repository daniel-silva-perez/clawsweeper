## Summary

Keep this issue open. The repository policy for openclaw/clawhub keeps issues open, and current main still does not implement post-install/update active-session event injection. The CLI install/update paths perform package resolution, extraction, origin/lock metadata writes, and success output; the related web UX is an explicit Copy Prompt flow, with the approved design explicitly excluding direct handoff or automatic execution into OpenClaw.

## Best Possible Solution

Keep this issue open for maintainer product/API triage. The strongest path is to define an explicit, user-consented ClawHub-to-OpenClaw session notification contract, then implement post-success install/update signaling with tests and docs covering install, update, non-interactive CLI use, and security boundaries. Maintainers may also decide the current explicit Copy Prompt UX is the intended boundary, but that should be a deliberate product decision.
