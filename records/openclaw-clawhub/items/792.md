## Summary

This is a ClawHub issue, and the repository-specific review policy only permits auto-close for PRs that are certainly implemented on main. The requested post-install/update active-session event also still appears unimplemented in current main: install/update are CLI package-management flows that download, extract, record metadata, and print status without a host/session event contract or onboarding trigger.

## Best Possible Solution

Keep this issue open for maintainer product/API triage. The best path is to define a safe active-session notification contract between ClawHub and OpenClaw, then update install/update or the host integration layer to emit a post-success event that points the agent at the installed `SKILL.md`, with tests and docs covering install, update, non-interactive CLI use, and user-consent boundaries.

## Risks / Open Questions

- The feature crosses from ClawHub package management into active OpenClaw session orchestration, so implementation likely needs an explicit host/session event contract rather than only a CLI message change.
- Automatically causing an agent to read and act on newly installed skill instructions has security and consent implications, especially for setup commands, plugin configuration changes, and restarts triggered from third-party skill content.
