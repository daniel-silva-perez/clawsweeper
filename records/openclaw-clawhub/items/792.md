## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session after ClawHub CLI install/update, and this repository’s cleanup policy keeps issues open for maintainer handling. The request is concrete, but it needs product and security judgment because it would let newly installed or updated skill content influence an active agent session.

## What This Changes

Request to notify an active OpenClaw session after `clawhub install` or `clawhub update` so the agent reads the installed skill’s `SKILL.md` and performs setup or onboarding work.

## Best Possible Solution

Maintainers should define the supported install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with focused CLI/session tests and docs, or document the inspectable copy-prompt flow as the intended setup path.
