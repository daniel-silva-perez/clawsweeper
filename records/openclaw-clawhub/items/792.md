## Summary

Keep this issue open. Current main still does not inject a post-install/update system event into an active OpenClaw session, and repository policy for openclaw/clawhub says issues should remain open unless maintainer-handled. The existing web prompt flow is related but intentionally copy-only, not automatic handoff or execution.

## Best Possible Solution

Keep this open for maintainer product/API triage. The best path is to define an explicit, user-consented ClawHub-to-OpenClaw session notification contract, decide whether automatic post-install/update signaling is allowed, then implement the accepted behavior with CLI tests, docs, and security-boundary coverage. Maintainers may also decide the current copyable prompt UX is the intended boundary, but that should be an explicit product decision.
