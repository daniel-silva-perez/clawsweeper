## Summary

Keep open. Current main does not implement post-install/update active-session event injection, and the ClawHub review policy says issues should remain open for maintainer handling; the remaining decision is a product/security/API boundary around whether the CLI may notify an active OpenClaw session.

## What This Changes

The issue requests that `clawhub install` and `clawhub update` inject a system event into the active agent session so the newly installed or updated skill’s `SKILL.md` onboarding instructions are read and acted on.

## Best Possible Solution

Keep this open for maintainer product/security/API triage. Maintainers should decide whether ClawHub may notify an active OpenClaw session after install/update; if yes, define a user-consented session handoff contract with CLI/session tests, docs, and security-boundary coverage. If not, document the current inspectable copyable-prompt UX as the intended boundary.
