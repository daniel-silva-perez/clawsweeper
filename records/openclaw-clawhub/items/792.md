## Summary

Keep this issue open. Repository policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on main, and this item is an issue. Current main also still lacks the requested post-install/update active-session system event: the CLI installs or updates files and metadata, while the web UI provides an explicit copy-paste OpenClaw prompt flow rather than automatic session injection.

## Best Possible Solution

Keep this issue open for maintainer product/API triage. The strongest path is to define an explicit, user-consented ClawHub-to-OpenClaw session notification contract, then implement post-success install/update signaling with tests and docs covering install, update, non-interactive CLI use, and security boundaries. Maintainers may also decide the current explicit Copy Prompt UX is the intended boundary, but that should be a product decision rather than an automated close.
