## Risks / Open Questions

- The PR’s proposed backend authorizes only the skill owner user or moderators, while the UI uses `canManage`, which also includes publisher members; publisher-owned skills would show an editor to users who cannot save.
- The proposed `summary: v.string()` mutation has no application-level maximum length, trimming, or normalization, leaving a public write path for oversized or low-quality metadata.
- The proposed frontend renders a form inside a paragraph element, which is invalid HTML and can cause browser parsing, layout, or hydration issues.
- The proposed mutation patches only the `skills` row; current main denormalizes summary into `skillSearchDigest`, so search/catalog views may become stale without a digest update.
- No tests are shown in the PR file list for owner, publisher member, moderator, validation failure, or digest synchronization behavior.
