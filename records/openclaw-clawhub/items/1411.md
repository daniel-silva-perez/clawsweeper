## Summary

Keep open. Current main still does not implement owner-editable skill summaries, and this PR remains a plausible implementation path with unresolved functional and validation feedback. ClawHub cleanup policy only closes PRs when current main definitely implements the intended change, which is not true here.

## Best Possible Solution

Keep this PR open for contributor or maintainer revision if the feature is still wanted. The best path is to add a validated updateSummary mutation, align authorization with the existing owner/publisher-member/moderator management model, normalize and cap summary length, update skillSearchDigest and any other denormalized search/catalog surfaces, fix the invalid JSX nesting, and add focused tests for owner, publisher member, moderator, validation, and digest-sync behavior. It should also resolve or coordinate with related issue #1345.

## Risks / Open Questions

- Functional risk in the PR diff: publisher members can be treated as managers in the current UI model, but the proposed backend mutation only authorizes ownerUserId or moderators.
- Validation and abuse risk in the PR diff: the proposed public mutation accepts an unrestricted v.string() summary, with no trimming or application-level length cap.
- UI risk in the PR diff: the proposed edit form is rendered inside a paragraph element, which is invalid HTML and can cause browser parsing or hydration issues.
- Data consistency risk in the PR diff: summary is denormalized into skillSearchDigest, but the proposed mutation only patches the skills document.
- Security review pass: the provided changed files are limited to Convex skill code, SkillDetailPage, SkillHeader, and CSS; no CI, dependency, lockfile, release, package publishing, secret, or downloaded-artifact changes are shown.
