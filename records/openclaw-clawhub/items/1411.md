## Summary

Keep open. Current main still has a read-only skill summary and no summary update mutation, so the PR is not implemented on main and does not qualify for ClawHub cleanup closure. The proposed patch is useful but not ready to merge because it adds a public summary write path without aligned publisher authorization, moderation recomputation, or server-side length validation, and the inline form uses invalid HTML nesting.

## What This Changes

The PR adds a Convex `updateSummary` mutation, detail-page React state, inline skill-header editing controls, and CSS so a skill summary can be edited from the skill detail page.

## Best Possible Solution

Land a revised implementation that treats summary editing as a validated metadata update: define whether edits override or are replaced by future publishes, authorize the same direct-owner, publisher-member, and moderator actors exposed by the UI, normalize and cap the value consistently with publish-time summaries, recompute or explicitly reset moderation state, and cover the detail UI plus Convex mutation behavior with focused tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep moderation state in sync with edited summaries:** `convex/skills.ts:4814-4818`
  - body: The mutation changes public summary text but only patches `summary` and `updatedAt`. Because the publish path derives moderation flags and `isSuspicious` from the summary that is persisted, edited summaries can leave safety state describing the old text until another publish or scan updates it.
  - confidence: 0.9
- **[P1] Authorize publisher-owned skill managers:** `convex/skills.ts:4808-4812`
  - body: The edit button is shown through `canManage`, which includes members of `skill.ownerPublisherId`, but `updateSummary` only allows the direct `ownerUserId` or moderators. Publisher members can open the editor and will always fail when saving publisher-owned skills.
  - confidence: 0.93
- **[P2] Move the summary edit form out of the paragraph:** `src/components/SkillHeader.tsx:211-214`
  - body: When edit mode is active, the component renders a `<form>` and block children inside `<p className="section-subtitle">`. Browsers implicitly close paragraphs around flow content, which can cause hydration or layout issues; use a non-paragraph wrapper for both states.
  - confidence: 0.95
- **[P2] Cap editable summary length on the server:** `convex/skills.ts:4800-4803`
  - body: The public mutation accepts an unrestricted string and stores it on public skill metadata. Publish-time summaries are normalized to a bounded length, so this path should enforce a server-side limit and matching textarea `maxLength` to avoid response bloat and document-size pressure.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The PR adds a public metadata write path that can make public searchable text diverge from moderation state and accepts unbounded input.

Concerns:

- **[medium] Edited summaries bypass moderation recomputation:** `convex/skills.ts:4814`
  - body: `updateSummary` changes the public summary but does not recompute moderation flags, `isSuspicious`, or related moderation summary fields, even though publish-time code derives those fields from the persisted summary.
  - confidence: 0.9

  - body: The mutation accepts any Convex string and stores it on public skill metadata surfaces. A server-side cap is needed to limit document-size and response-bloat pressure from malformed or hostile clients.
