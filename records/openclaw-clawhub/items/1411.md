## Summary

Keep open. Current main still does not implement owner-editable skill summaries: the skill detail UI renders the summary as static text, the detail page wires only tag/report/readme mutations, and the backend still has no updateSummary mutation. The PR is also paired with open related issue #1345 by the same author and has unresolved review feedback, so it should remain available for maintainer review or revision.

## Best Possible Solution

Keep the PR open and revise it if the feature is still wanted: add a validated updateSummary mutation, align authorization with the existing owner/publisher/moderator management model, enforce a summary length limit and normalization, update any denormalized digest/search data when the skill summary changes, fix the invalid HTML nesting in the inline editor, and add focused tests for owner, publisher member, moderator, validation, and digest-sync behavior.
