## Summary

Keep this issue open. The ClawHub source tree now has meaningful registry-side mitigations for the reported malicious-skill pattern, including static detection, hidden/malware moderation state, tests, and CLI install/update gates. However this is a ClawHub issue, and the repository-specific cleanup policy permits auto-closing only PRs that are certainly implemented on main. Substantively, the broader report also asks for OpenClaw runtime installer sandboxing/approval and keychain-backed credential storage, which are not fully resolved in this repository.

## Best Possible Solution

Keep this issue open for maintainer security triage. Preserve the shipped ClawHub registry mitigations, then split or link the remaining work into explicit follow-ups: ClawHub static scanning/abuse-response hardening here, and OpenClaw core installer sandboxing, script approval, and keychain-backed credential storage in the core repository if still applicable.
