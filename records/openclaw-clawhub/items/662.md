## Summary

Keep open. This is an open ClawHub security issue, and the repository cleanup policy for this pass only permits auto-closing ClawHub pull requests that current main definitely implements. Current main has strong registry-side mitigations for the reported obfuscated install-prompt payload family, but the broader report still needs maintainer security triage because it spans OpenClaw runtime sandboxing/approval, local credential storage, outbound network policy, and registry distribution policy.

## What This Changes

The issue requests mitigation for malicious skills that can run shell commands during installation, exfiltrate local API keys, and bypass ClawHub/static review controls.

## Best Possible Solution

Keep this open for maintainer security triage. Preserve the registry mitigations already on main, then split the remaining work explicitly: ClawHub registry hardening and moderation policy in this repository, and OpenClaw core issues for sandboxed prerequisite execution, script approval, outbound network controls, and keychain-backed credential storage.
