Keep open. This is an open security issue, and the repository cleanup policy for openclaw/clawhub does not auto-close issues in this pass. Current main has meaningful ClawHub-side registry mitigations for the reported obfuscated installer payload pattern, but the report still spans OpenClaw runtime sandboxing, install approval UX, credential storage, outbound network controls, and production moderation policy.

## What This Changes

The issue requests security hardening against malicious third-party skills that use installation instructions to execute shell commands and exfiltrate local OpenClaw credentials.
