## Summary

Keep open. This is an open ClawHub issue, and the supplied repository cleanup policy only permits auto-closing ClawHub pull requests that current main certainly implements. Current main has meaningful registry-side mitigations for the reported obfuscated install-prompt pattern, but the issue also asks for OpenClaw runtime installer sandboxing, explicit script approval, outbound-network controls, and keychain-backed local credential storage, none of which are proven resolved in this ClawHub source tree.

## Best Possible Solution

Keep this issue open for maintainer security triage. Preserve the ClawHub registry mitigations already on main, then split or link the remaining work explicitly: ClawHub scanning/moderation hardening in this repository, and OpenClaw core tasks for runtime installer sandboxing, prerequisite-script approval, outbound-network policy, and keychain-backed local credential storage.
