## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main has strong registry-side mitigations for the reported obfuscated install-prompt pattern, but the report also asks for OpenClaw runtime installer sandboxing, script approval, outbound-network controls, and keychain-backed local credential storage, none of which are proven resolved in this ClawHub source tree.

## Best Possible Solution

Keep this issue open for maintainer security triage. Preserve the ClawHub registry mitigations already on main, then explicitly split or link remaining work: ClawHub scanning/moderation hardening in this repository, and OpenClaw core tasks for runtime installer sandboxing, script approval, outbound-network policy, and keychain-backed credential storage.
