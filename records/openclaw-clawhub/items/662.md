## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy for this pass only permits auto-closing pull requests that current main definitely implements. Current main has meaningful registry-side mitigations for the reported obfuscated install-prompt payload family, but the broader report still spans security-sensitive OpenClaw runtime sandboxing, user approval, local credential storage, outbound network controls, and production moderation policy.

## What This Changes

The issue reports that malicious third-party skills can use SKILL.md prerequisite or install instructions to run shell commands and exfiltrate local OpenClaw credentials, and asks for sandboxing, explicit script approval, keychain-backed secrets, static analysis, and network restrictions.

## Best Possible Solution

Keep this open for maintainer security triage. Preserve the ClawHub registry hardening already on main, then split the remaining work explicitly between ClawHub moderation/scanning/distribution policy in this repository and OpenClaw core issues for runtime sandboxing, prerequisite approval UX, keychain-backed credential storage, and outbound network controls.
