## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Current main has substantially newer CLI/browser/backend auth code than the reported CLI v0.7.0 path, but I did not find proof that the production Unauthorized path has been verified fixed for affected users, and related auth/publish reports remain open.

## What This Changes

The issue reports that `clawhub login` succeeds through browser OAuth but later CLI commands such as `whoami`, `sync`, and `publish` return `Unauthorized`, possibly because the token is not stored, not verified, or not preserved across `clawhub.ai` / `www.clawhub.ai` registry handling.

## Best Possible Solution

Keep this issue open for maintainer or CLI-owner production verification using the current released CLI and a fresh token. If production is fixed, close manually with the exact CLI/npm release and fixing commit; if it still fails, make a focused auth fix around registry discovery/canonicalization, loopback token delivery, config writes, bearer preservation, backend token lookup, and user/publisher resolution, with regression coverage for `whoami`, `sync`, and `publish`.
