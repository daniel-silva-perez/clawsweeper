## Summary

Keep open. Repository policy for this cleanup pass keeps ClawHub issues open, and current main only has partial building blocks: dependency metadata, query-driven search, skill install/version lock state, and install metadata display. It does not yet implement task-context skill recommendations, skill conflict/tool-name metadata for skills, or production guidance for skill layering and version pinning.

## What This Changes

This issue asks how ClawHub should help agents manage growing skill sets, including task-context skill recommendations, dependency/conflict metadata, and production version-pinning guidance.

## Best Possible Solution

Keep this open for maintainer/product triage. The best path is to split it into narrower follow-ups if maintainers agree: skill conflict/tool-name metadata for skills, task-context recommendation/search UX, and production docs for skill layering plus version pinning, while keeping the current dependency metadata, query search, install metadata, and lockfile behavior.
