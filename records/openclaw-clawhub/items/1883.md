## Summary

Keep this issue open. It is a newly opened ClawHub product/community issue about skill accumulation, recommender behavior, conflict metadata, and production version policy. Current main has related building blocks, including skill dependency declarations, query-driven search, visible install metadata, explicit CLI version targeting, and lockfile state, but it does not fully implement the requested recommender/conflict/version-governance surface. The ClawHub cleanup policy for this pass also keeps issues open and only proposes auto-close for PRs that are certainly implemented on main.

## Best Possible Solution

Keep the issue open for maintainer/product triage. The best path is to split it into narrower follow-ups if maintainers agree: a skill metadata proposal for conflicts/tool names/priority semantics, a task-context recommender or search UX proposal, and a docs/CLI policy page for production skill layering and version pinning. Current main should keep the existing dependency, search, install metadata, version-target install/update, and lockfile behavior while that product boundary is decided.
