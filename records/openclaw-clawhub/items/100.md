## Summary

Keep open. Current main still defaults and discovers the apex host, while live production now redirects www to the apex host rather than the apex to www, so the original root cause needs maintainer verification with a fresh production token. The related www-default PR was closed unmerged, and #99 remains open for adjacent Unauthorized login failures.

## Best Possible Solution

Keep this issue open for maintainer handling. A maintainer should verify `clawhub login --token <fresh production token>` and `clawhub whoami` against the default registry, then compare explicit `CLAWHUB_REGISTRY=https://clawhub.ai` and `CLAWHUB_REGISTRY=https://www.clawhub.ai`. If auth still fails, fix production routing so authenticated API requests never cross origins, or intentionally change the canonical host across CLI defaults, discovery metadata, docs, tests, and release notes together. Coordinate any remaining Unauthorized-login work with #99 and do not revive #101 without rechecking current production routing.
