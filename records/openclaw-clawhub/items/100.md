## Summary

Keep open. Current main still uses `https://clawhub.ai` as the CLI/site registry identity and the live non-www API endpoint now returns a direct 401 rather than a redirect for a dummy Authorization request, so this needs maintainer verification with a real token rather than an automated close. ClawHub policy also keeps issues open unless a maintainer closes them.

## What This Changes

The issue requests fixing CLI authentication failures caused by authenticated API calls to `clawhub.ai` being redirected to `www.clawhub.ai` and losing the Authorization header.

## Best Possible Solution

A maintainer should verify the current published CLI against production with a real token. If the redirect is gone, close this as externally resolved or fold any remaining auth work into #99; if a real-token redirect still occurs, make one narrow canonical-domain fix across CLI defaults, discovery JSON, tests, and docs.
