Keep open. This is an issue, and the openclaw/clawhub cleanup policy says issues are not auto-closed by this workflow. Current main also has not implemented the proposed default-to-www change: CLI defaults, discovery metadata, tests/docs, and the token verification path still use `https://clawhub.ai`. A live invalid-token check on April 29, 2026 no longer shows the originally reported apex-to-www redirect, but without a valid production token it does not prove the authenticated CLI flow is fixed. Related PR #101 was closed unmerged, while #99 remains open for broader Unauthorized login symptoms.

Keep this issue open for maintainer handling. A maintainer should verify the live token login and `whoami` flow with a fresh production API token against the default apex registry. If it still fails, fix production routing so authenticated API requests are not cross-origin redirected, or intentionally switch CLI defaults, discovery metadata, docs, and tests to the chosen canonical host. If production is now verified fixed, maintainers can manually close this as obsolete or consolidate any remaining Unauthorized-login work with #99.
