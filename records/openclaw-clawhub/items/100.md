## Summary

Keep open. The repository cleanup policy for openclaw/clawhub says issues are not auto-closed by this workflow, and current main has not implemented the proposed switch to `https://www.clawhub.ai`. Live routing has changed since the original report: the apex API now returns 401 directly for an invalid token, while `www.clawhub.ai` redirects back to the apex host, so the remaining question needs maintainer verification with a valid production token rather than an automatic close or blind code PR.

## Best Possible Solution

Keep this issue open for maintainer handling. A maintainer should verify `clawhub login --token <fresh production token>` and `clawhub whoami` against the default apex registry. If it still fails, fix production routing so authenticated API requests are not cross-origin redirected, or intentionally switch CLI defaults, discovery metadata, docs, tests, and deployment/canonical URL guidance to the chosen host together. If production is now verified fixed, maintainers can manually close this as obsolete or consolidate remaining Unauthorized-login work with #99.
