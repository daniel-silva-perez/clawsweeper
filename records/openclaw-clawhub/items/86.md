## Summary

Keep open. This is an open ClawHub issue, and the repository-specific policy only permits automated closure for pull requests that are certainly implemented on current main. Current main also still lacks a first-class shareable cron/automation-template content type: the product scope, schema, public API routes, CLI, and UI routes are all scoped to skills, SOULs, code plugins, bundle plugins, and package catalog browsing/publishing rather than previewable cron configs or one-click cron installation.

## Best Possible Solution

Keep this issue open for maintainer product triage. The next concrete step is to decide whether shareable cron jobs should be a new ClawHub content type, a new package family with cron manifest metadata, or examples embedded in skills/bundle plugins; then specify schema, preview format, publish/browse/detail APIs, CLI behavior, and the OpenClaw install/runtime contract.
