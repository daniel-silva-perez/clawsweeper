## Summary

Keep open. This is a ClawHub issue, and the repository-specific policy only allows automated closure for pull requests that are certainly implemented on current main. Substantively, current main still has no shareable cron/automation-template content type: docs, schema, API routes, CLI commands, and UI routes remain scoped to skills, SOULs, code plugins, bundle plugins, and package catalog flows. The only related item in the supplied context, #1112, is an open cron form layout bug and does not implement or supersede this product request.

## Best Possible Solution

Keep this issue open for maintainer product triage. The next step is to decide whether shareable cron jobs should be a new ClawHub content type, a package family with cron manifest metadata, or examples embedded in skills/bundle plugins, then specify schema, preview format, publish/browse/detail APIs, CLI behavior, moderation/security checks, and the OpenClaw install/runtime contract.
