## Summary

Keep open. This is an open ClawHub issue, and the repository-specific policy only permits automated closure for pull requests that are certainly implemented on current main. Substantively, current main still lacks a shareable cron or automation-template marketplace: product docs, schema, route constants, HTTP routes, CLI docs, and UI routes are scoped to skills, SOULs, packages, code plugins, and bundle plugins.

## Best Possible Solution

Keep this issue open for maintainer product triage. The next concrete step is to decide whether shareable cron jobs should be a new ClawHub content type, package/bundle plugin metadata tied to an OpenClaw cron install API, or examples embedded in skills; then specify schema, preview format, publish/browse/detail APIs, CLI behavior, and install UX.
