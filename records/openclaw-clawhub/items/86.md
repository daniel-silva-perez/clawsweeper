## Summary

Keep open. The repository policy for this cleanup pass keeps ClawHub issues open, and current main still does not implement a first-class cron or automation-template catalog. The existing product, schema, API, UI, and CLI surfaces cover skills, code plugins, and bundle plugins; the linked cron form report is a separate dashboard UI bug, not a superseding implementation path.

## What This Changes

The issue requests a browsable, shareable catalog of cron job or automation templates with descriptions, cron config preview, one-click install, and categories.

## Best Possible Solution

Keep this open until maintainers choose the cron-template product model. If approved, implement the agreed schema/API/UI/CLI/docs path plus an OpenClaw runtime install contract with explicit preview, consent, trust labels, and moderation semantics.

## Risks / Open Questions

- A shareable scheduled-job marketplace spans product model, schema/API, browse/detail UI, CLI support, moderation, and OpenClaw runtime installation behavior rather than a narrow bug fix.
- One-click installation of scheduled jobs needs explicit trust, consent, preview, and review semantics because templates can trigger future actions after installation.
- The existing skill/plugin catalog could be reused as a substrate, but maintainers still need to choose whether cron templates are a new package family, a skill/category convention, a bundle-plugin pattern, or an OpenClaw runtime/plugin feature.
