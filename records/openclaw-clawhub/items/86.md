## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only permits auto-closing PRs that are certainly implemented on current main. The requested shareable cron/automation-template marketplace is also not implemented on current main: existing product, schema, API, package family, CLI, and UI surfaces are oriented around skills, souls, and plugin/package catalogs, with only internal maintenance crons and skill category keywords touching cron/automation language.

## Best Possible Solution

Keep this issue open for maintainer product triage. The next useful step is to decide whether shareable cron jobs belong as a new ClawHub content type, as package/bundle plugins tied to an OpenClaw cron install API, or as examples/metadata inside skills; then specify the schema, preview format, publish/install API, CLI flow, and browse UI before implementation.
