## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and current main still does not implement a shareable cron-job or automation-template catalog. The existing schema, API, CLI, and plugin UI surfaces are limited to skills, code plugins, and bundle plugins; the related Cron form report is a separate dashboard layout issue, not a superseding implementation path.

## What This Changes

Requests a shareable cron-job or automation-template catalog with browsing, descriptions, config preview, categories, and one-click install into OpenClaw.

## Best Possible Solution

Keep this open until maintainers choose the cron-template product model. If approved, implement the agreed schema/API/UI/CLI/docs path plus an OpenClaw runtime install contract with explicit preview, consent, trust labels, and moderation semantics.

## Risks / Open Questions

- A shareable scheduled-job marketplace spans product model, schema/API, browse/detail UI, CLI behavior, moderation, and OpenClaw runtime installation semantics rather than a narrow code fix.
- One-click installation of scheduled jobs needs explicit preview, consent, trust labeling, and review semantics because templates can trigger future actions after installation.
- The existing skill/plugin catalog might be reused as a substrate, but maintainers still need to choose whether cron templates are a new package family, a skill/category convention, a bundle-plugin pattern, or an OpenClaw runtime feature.
