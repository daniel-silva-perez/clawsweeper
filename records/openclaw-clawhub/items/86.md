## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and current main still does not implement a shareable cron-job or automation-template catalog. The existing catalog/schema/API/CLI/UI surfaces are limited to skills, code plugins, and bundle plugins, while the related Cron form report is a separate dashboard layout bug rather than a superseding implementation path.

## What This Changes

The issue requests a ClawHub Cron Jobs or Automations catalog where users can browse and share cron job templates, preview cron config, and install them into an OpenClaw instance.

## Best Possible Solution

Keep this open until maintainers choose the cron-template product model. If approved, implement the agreed schema/API/UI/CLI/docs path plus an OpenClaw runtime install contract with explicit preview, consent, trust labels, and moderation semantics.
