## Summary

Keep open. ClawHub review policy keeps issues open, and current main does not implement a first-class catalog for shareable cron job templates with preview and one-click install. The closest overlap is skill/category discovery and the newer package/plugin catalog work, but cron templates still need maintainer product and security design.

## Best Possible Solution

Keep this issue open for maintainer product judgment. Decide whether cron templates belong as a new package family, a skill capability/tag, a bundle plugin format, or an OpenClaw runtime/plugin feature, then specify manifest shape, preview/install semantics, trust labels, security prompts, moderation, and API/UI support before implementation.

## Risks / Open Questions

- A cron-template marketplace spans product model, schema, API, browse/detail UI, moderation, and OpenClaw runtime install behavior rather than one narrow fix.
- One-click installation of scheduled jobs has explicit consent and trust implications because templates can trigger future actions after install.
- The current package/plugin catalog may be a good substrate, but maintainers still need to decide whether cron templates are a package family, a skill capability, a bundle plugin pattern, or an OpenClaw runtime feature first.
