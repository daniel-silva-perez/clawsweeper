## Summary

Keep open. This is an open ClawHub issue, and the repository policy says issues stay open unless maintainers explicitly resolve them. Current main still does not implement a first-class catalog for shareable cron job templates with preview and one-click install. The closest existing surfaces are skill automation categorization and the package/plugin catalog, but cron templates remain a product and security design decision.

## Best Possible Solution

Keep this issue open for maintainer product judgment. Decide whether cron templates belong as a new package family, a skill capability/tag, a bundle plugin format, or an OpenClaw runtime/plugin feature; then define manifest shape, preview/install semantics, trust labels, security prompts, moderation, API/UI support, and runtime integration before implementation.

## Risks / Open Questions

- A cron-template marketplace spans product model, schema, API, browse/detail UI, moderation, and OpenClaw runtime install behavior rather than one narrow code change.
- One-click installation of scheduled jobs has explicit consent and trust implications because templates can trigger future actions after install.
- The current package/plugin catalog may be a useful substrate, but maintainers still need to decide whether cron templates are a package family, a skill capability, a bundle plugin pattern, or an OpenClaw runtime feature.
