## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Substantively, the reported hosted skill still returns `suspicious.prompt_injection_instructions`, the public SKILL.md still contains the `system prompt:` wording that current main flags, and current main intentionally does not clear that reason code merely because external scanners are clean.

## Best Possible Solution

Keep this issue open for moderator or maintainer review. A moderator should inspect the `suspicious.prompt_injection_instructions` evidence for the reported skill and apply the existing clean manual override if it is benign. If legitimate memory/setup skills commonly need to say `system prompt:`, maintainers should make a narrow policy change in `convex/lib/moderationEngine.ts` or `convex/lib/moderationReasonCodes.ts`, update `src/components/SkillSecurityScanResults.tsx` if user-facing guidance changes, and add focused moderation tests.
