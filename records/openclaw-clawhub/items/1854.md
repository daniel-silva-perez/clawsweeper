## Summary

Keep this PR open. Current main still has the inconsistency this PR is meant to fix: public skill download counts use the package icon in the skill header, skill list rows, and shared stats components, while the dashboard uses `ArrowDownToLine` for downloads and keeps `Package` for versions. The PR is narrow and not security-sensitive, but it is not implemented on main, so ClawHub cleanup policy should not auto-close it.

## Best Possible Solution

Keep this draft PR open for maintainer review. The best path is to land the narrow icon swap, or explicitly fold the same public download-icon change into #1233, so `SkillHeader`, `SkillListItem`, and `SkillStats` use `ArrowDownToLine` for download counts while `Package` remains reserved for package/version concepts.
