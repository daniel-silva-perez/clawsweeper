## Summary

Keep this issue open. It is a ClawHub issue, and the repo cleanup policy only proposes auto-closing PRs that are certainly implemented on main. Current main still treats user-initiated deletion as irreversible: OAuth completion blocks deactivated/deleted users, deletion purges and deactivates the user record, admin unban explicitly excludes permanently deleted accounts, and docs/UI still describe deletion as non-recoverable. Recent auth UI work may surface a clearer error, but it does not provide the requested purge, recovery, GitHub identity unlink/relink, or clean re-registration path.

## Best Possible Solution

Keep this open for ClawHub account lifecycle triage. A durable fix should define and document the intended post-deletion state machine, then add a safe admin or self-service path to purge, restore, unlink, or relink GitHub OAuth identities after deletion, or make the irreversible policy surface a clear terminal error plus support route instead of a refresh loop.
