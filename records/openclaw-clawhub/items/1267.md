## Summary

Keep this issue open. Current main still intentionally treats user-initiated account deletion as irreversible and blocks OAuth sign-in for deleted/deactivated users; recent UI work may surface a clearer auth error, but it does not provide the requested purge, recovery, unlink/relink, or clean re-registration path.

## Best Possible Solution

Keep this open for ClawHub account lifecycle triage. A durable fix should define and document the intended post-deletion state machine, then add a safe admin or self-service path to purge, restore, unlink, or relink GitHub OAuth identities after deletion, or else make the irreversible policy surface a clear terminal error with a support route instead of a confusing loop.
