## Summary

Keep this open. Current main still treats user-initiated ClawHub account deletion as irreversible deactivation: OAuth completion rejects deleted/deactivated users, account deletion writes deactivated/purged state, active-user helpers exclude those records, admin unban excludes permanently deleted accounts, and docs/UI describe deletion as permanent/non-recoverable. This is also an open ClawHub issue, and the repository cleanup policy only permits auto-close for PRs that are certainly implemented on main.

## Best Possible Solution

Keep this open for ClawHub account lifecycle triage. Maintainers should define the post-deletion state machine and then implement one explicit path: safe admin/self-service purge, restore, unlink, or relink for GitHub identities after deletion, or a clearer terminal error plus support route if deletion must remain irreversible. The related login-loop reports should be handled as one account lifecycle cluster with docs and tests updated together.
