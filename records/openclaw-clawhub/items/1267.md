## Summary

Keep this open. Current main still has no self-service recovery, purge/recreate, unlink/relink, or support-admin restore path for a user-initiated deleted ClawHub account; the code intentionally records permanent deactivation and rejects later GitHub OAuth sign-in. The related reports remain open, and the repository policy for ClawHub issues also favors maintainer handling rather than auto-closing issues.

## What This Changes

The issue requests recovery or server-side cleanup for a GitHub-linked ClawHub account that enters a login refresh loop after deleting a ClawHub account and switching GitHub accounts.

## Best Possible Solution

Define one supported post-deletion lifecycle for GitHub-linked ClawHub users, then implement it consistently across OAuth callbacks, user records, admin/support tooling, user-facing errors, docs, and tests: restore, purge/recreate, unlink/relink, or an explicitly terminal state with a clear support route.
