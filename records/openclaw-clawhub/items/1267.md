## Summary

Keep this open. Current main still treats user-initiated account deletion as an irreversible deactivation state, blocks later GitHub OAuth sign-in, and offers no self-service recovery, purge/recreate, unlink/relink, or admin restore path for deleted accounts. The related login-loop reports are still open, and the repository policy for ClawHub issues keeps issues open for maintainer handling.

## What This Changes

The issue requests server-side recovery or cleanup so a user who deleted one GitHub-linked ClawHub account can sign in or re-register with a different GitHub account instead of looping back to logged-out state.

## Best Possible Solution

Define one supported post-deletion lifecycle for GitHub-linked ClawHub users, then implement it consistently across OAuth callbacks, auth-account binding, user records, admin/support tooling, user-facing error messages, docs, and tests.
