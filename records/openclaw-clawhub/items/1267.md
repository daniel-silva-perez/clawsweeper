## Summary

Keep this issue open. It is a ClawHub issue, and the supplied repository policy says issues should not be auto-closed; only PRs that are certainly implemented on current main are eligible. Substantively, current main still treats user-initiated deletion as irreversible and blocks deleted/deactivated users at OAuth completion, so the requested account purge/recovery/GitHub unlink or relink path is not implemented.

## Best Possible Solution

Keep this open for maintainer triage as ClawHub account lifecycle work. A durable fix should add a safe documented admin or self-service path to purge, restore, unlink, or relink GitHub OAuth identities after account deletion, or make the irreversible-deletion policy and OAuth failure path explicit enough that affected users get a clear terminal error and support route.
