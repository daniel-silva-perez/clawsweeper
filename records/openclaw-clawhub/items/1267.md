## Summary

Keep this issue open. The ClawHub cleanup policy only permits auto-closing PRs that are certainly implemented on main, and this is an issue. Substantively, current main still treats user-initiated account deletion as irreversible and blocks deleted/deactivated users from GitHub reauth rather than providing the requested purge, recovery, unlink, or relink path.

## Best Possible Solution

Keep this open for maintainer triage as ClawHub account lifecycle work. A durable fix should add a safe documented admin or self-service path to purge, restore, unlink, or relink GitHub OAuth identities after account deletion, or explicitly document and surface the irreversible-deletion failure mode with a clear support path instead of a login loop.
