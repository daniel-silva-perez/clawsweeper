## Summary

Keep this issue open. It is an open ClawHub issue, and the repository policy only permits implemented-on-main auto-close for PRs in this repository. Current main still treats deleted/deactivated ClawHub accounts as terminal for OAuth reauthentication: the auth hook rejects deleted/deactivated users, account deletion writes deactivated and purged state, active-session helpers exclude those users, admin recovery explicitly does not restore permanently deleted accounts, tests assert the block, and related deleted-account login-loop reports remain open.

## Best Possible Solution

Keep this open for ClawHub account lifecycle triage. Maintainers should define the intended post-deletion state machine, then implement either a safe admin/self-service purge, restore, unlink, or relink flow for GitHub identities after deletion, or a clearer terminal error with a support route if deletion must remain irreversible. The related open login-loop reports should be handled as one account lifecycle cluster.
