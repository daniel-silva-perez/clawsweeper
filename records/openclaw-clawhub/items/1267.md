## Summary

Keep this open. Current main still treats user-initiated ClawHub deletion as terminal deactivation: OAuth completion rejects deactivated users, access helpers exclude deleted/deactivated records, admin unban cannot restore permanently deleted accounts, and docs/settings UI describe deletion as irreversible. The related discussion and open reports show this is a broader account lifecycle/recovery issue, not conservative cleanup closure.

## What This Changes

The issue requests a ClawHub account lifecycle fix or server-side cleanup so a user can sign in with GitHub after deleting a prior ClawHub account and switching GitHub accounts.

## Best Possible Solution

Keep this open for ClawHub account lifecycle triage. Maintainers should define the post-deletion state machine, then implement one explicit path: safe admin/self-service purge, restore, unlink, or relink for GitHub identities after deletion, or a clear terminal error plus support route if deletion must remain irreversible. Handle the related login-loop reports as one cluster with docs and tests updated together.
