## Summary

Keep open. Current main still matches the reported failure mode rather than resolving it: user-initiated deletion is treated as a terminal deactivated/purged state, OAuth completion rejects that state, and active-user lookup returns no user afterward. The related account-deletion login-loop reports remain open, and the ClawHub cleanup policy does not auto-close issues for this kind of maintainer/product decision.

## What This Changes

The issue asks for a way to recover, purge, or relink a ClawHub account after deletion so GitHub OAuth sign-in no longer loops or returns to a logged-out state.

## Best Possible Solution

Define one supported post-deletion lifecycle for GitHub-linked ClawHub users, then implement it consistently across OAuth callbacks, user/auth account records, admin or support tooling, user-facing errors, documentation, and tests: restore, purge/recreate, unlink/relink, or an explicitly terminal state with a clear support route.
