## Summary

This ClawHub issue should stay open. The repository-specific cleanup policy for openclaw/clawhub only proposes auto-closing PRs that are certainly implemented on main, and current main still treats user-initiated deletion as irreversible rather than providing the requested purge, recovery, GitHub identity unlink/relink, or clean re-registration path. Related account lifecycle reports remain open, including #1620 and #1674.

## Best Possible Solution

Keep this open for ClawHub account lifecycle triage. A durable fix should add a safe documented admin or self-service path to purge, restore, unlink, or relink GitHub OAuth identities after account deletion, or make the irreversible-deletion policy and support route explicit enough that affected users get a clear terminal error and an account-support path instead of a refresh loop.
