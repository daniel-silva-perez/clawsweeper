## Summary

Keep open. This is an open ClawHub issue, and the repository policy for this review keeps issues open. Separately, current main still has source-level evidence matching the report: publish can set the skill-level moderation state to hidden for non-trusted publishers, and public search/list hydration rejects non-active skills rather than preserving a last-approved public version during review.

## Best Possible Solution

Implement a version-aware review/search path for skills: when an existing approved skill publishes a new version, keep the last approved public version, digest, and searchable embedding visible until the new version scan clears, or skip re-review when the content fingerprint and scanner inputs are unchanged. Add regression coverage for a non-trusted publisher updating an already-approved skill and verify public search still returns the skill during pending review.
