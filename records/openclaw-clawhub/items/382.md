## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open, and current main still matches the reported behavior: ordinary publishes schedule fresh scans, can reset the skill row to hidden pending review, sync that state into the search digest, and public search hydration filters hidden skills out.

## What This Changes

Already-approved skills should remain searchable while a newly published version is under VT/LLM review, and unchanged content should avoid redundant review when that is safe.

## Best Possible Solution

Keep this issue open and design a version-aware moderation/search path: keep the last approved public skill projection, digest, and searchable embedding visible while a newly published version is pending VT/LLM review, then promote the new projection only after review clears it. Add regression coverage for ordinary publishers republishing an already-approved skill and for unchanged-content review reuse if maintainers choose to implement that optimization.
