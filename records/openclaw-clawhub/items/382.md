## Summary

Keep open. This is an open ClawHub issue, and the review policy for this repository only permits automated closure for pull requests that are certainly implemented on main. A source pass on current main also confirms the reported behavior is still plausible: publishing can write a hidden skill-level moderation state, and public search hydration rejects hidden skills instead of preserving a last-approved searchable projection during pending review.

## Best Possible Solution

Implement a version-aware review/search path for skills. When an existing approved skill publishes a new version, keep the last approved public version, digest, and searchable embedding visible until the new version scan clears, or skip re-review when the scanner inputs/content fingerprint are unchanged. Add regression coverage for a non-trusted publisher updating an already-approved skill and verify public search still returns the skill during pending review.
