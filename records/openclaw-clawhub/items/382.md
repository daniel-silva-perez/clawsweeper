## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open in this pass, and current main still matches the reported behavior: each publish schedules VT and LLM review for the new version, ordinary publishers can have the skill-level moderation status reset to hidden while review is pending, the search digest mirrors that hidden state, and public search hydration filters hidden skills out. Related open report #1743 appears to describe the same user-visible search disappearance area.

## Best Possible Solution

Keep this issue open and design a version-aware moderation/search path. The likely direction is to preserve the last approved public skill projection, digest, and searchable embedding while a newly published version is pending VT/LLM review, then promote the new latest public projection only after review clears it. If unchanged-content reuse is added, compare the exact scanner/review inputs and add regression coverage for ordinary publishers republishing already-approved skills.
