## Summary

Keep this issue open. The repository-specific cleanup policy says ClawHub issues should not be auto-closed, and current main still supports the reported behavior: publishing can set the skill-level moderation state to hidden while scans/review are pending, the search digest mirrors that current hidden state, and public search/exact-slug hydration filters hidden skills instead of serving a last-approved public projection.

## Best Possible Solution

Keep this issue open and implement a version-aware public search/moderation path: preserve the last approved public skill projection, digest, and searchable embedding while a newly published version is pending scan/review, or skip re-review when the scanner inputs/content fingerprint are unchanged. Add regression coverage for an already-approved skill updated by a non-trusted publisher so public search still returns it during pending review.
