## Summary

Keep open. This is a ClawHub issue, so the cleanup policy does not auto-close it, and current main still has a credible path where an ordinary republish rewrites the skill to hidden during review and public search filters it out.

## What This Changes

The issue reports that publishing a new version temporarily removes an already-approved skill from search while the new version is under review.

## Best Possible Solution

Implement a version-aware public projection for skills: keep the last approved skill card, digest, and searchable embedding visible while a newly published version is pending review, then atomically promote the new projection only after moderation and scanning clear it.
