## Summary

Keep this issue open. The ClawHub cleanup policy only permits automated closure of pull requests that are certainly implemented on current main, and this item is an issue. The current source also still supports the reported behavior: publishing a version can set the skill-level moderation state to hidden while review/scan is pending, and public search hydration filters hidden skills instead of preserving the last approved searchable projection.

## Best Possible Solution

Keep this issue open and implement a version-aware review/search flow: preserve the last approved public skill projection, digest, and embedding while a newly published version is pending scan/review, or skip re-review when scanner inputs/content fingerprints are unchanged. Add regression coverage for a non-trusted publisher updating an already-approved skill and verify public search still returns the skill during pending review.
