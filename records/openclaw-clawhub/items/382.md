## Summary

Keep this issue open. The ClawHub cleanup policy for this run only permits automated closure for PRs that current main certainly implements, and this item is an issue. Current main also still supports the reported behavior: a publish can set the skill-level moderation status to hidden, the search digest mirrors that current hidden state, and public search hydration filters hidden skills instead of preserving the last approved searchable version.

## Best Possible Solution

Keep this issue open and implement a version-aware moderation/search flow: preserve the last approved public skill projection, digest, and embedding while a newly published version is pending scan/review, or skip re-review when scanner inputs/content fingerprints are unchanged. Add regression coverage for an already-approved skill updated by a non-trusted publisher so public search still returns the skill during pending review.
