## Summary

Keep open. The repository-specific cleanup policy says open ClawHub issues should not be auto-closed, and current main still supports the reported failure mode: publishing a new version can write `hidden` onto the skill record, the digest mirrors that skill-level moderation state, and search/list/catalog hydration all gate through `toPublicSkill`, which rejects hidden skills instead of keeping a last-approved public projection searchable.

## Best Possible Solution

Keep this issue open and implement a version-aware review/search path: when an already-approved skill publishes a new version, keep the last approved public skill projection, digest, and embedding searchable until the new scan clears, or skip re-review when the scanner inputs/content fingerprint are unchanged. Add regression coverage for a non-trusted publisher updating an already-approved skill and verify public search still returns the skill during pending review.
