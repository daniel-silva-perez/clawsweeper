## Summary

Keep this issue open. The repository-specific cleanup policy keeps ClawHub issues open, and current main still supports the reported behavior: each publish schedules scan/review work, ordinary publishers can have the skill-level moderation state set to hidden during that pending window, and public search hydrates/filter-checks the current skill/digest rather than a last-approved public projection. Related open report #1743 points to the same searchable-after-upload failure mode.

## Best Possible Solution

Keep this issue open and design a version-aware public search/moderation path: retain the last approved public skill projection, digest, and searchable embedding while a newly published version is pending review, and only expose the new latest version after scan/review clears it. If unchanged-content fingerprint reuse is added, it should compare the exact scanner inputs and preserve regression coverage for already-approved skills updated by non-trusted publishers.
