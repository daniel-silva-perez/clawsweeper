## Summary

Keep open. This is an issue, and the ClawHub cleanup policy for this pass does not auto-close issues; independently, current main still supports the reported behavior where an ordinary republish can hide the skill row during pending scan/review and public search then filters it out.

## What This Changes

The issue reports that republishing an already approved skill hides it from ClawHub search while the new version undergoes review/scanning, and asks to keep the last approved version indexed until the new scan completes.

## Best Possible Solution

Implement a version-aware public projection for skills: keep the last approved skill card, digest, and searchable embedding visible while a newly published version is pending review, then atomically promote the new projection only after moderation/scanning clears it. Add regression coverage for republishing an approved skill and handle unchanged-content reuse only if maintainers approve a safe fingerprint policy.
