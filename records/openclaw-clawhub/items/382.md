## Summary

Keep this issue open. The ClawHub cleanup policy keeps issues open, and current main still matches the reported behavior: an ordinary publish schedules new scans, writes a hidden pending-review moderation state onto the skill row, syncs that state into the search digest, and public search/list hydration filters hidden skills out instead of preserving the last approved public projection.

## What This Changes

The issue asks ClawHub to keep an already-approved skill searchable while a newly published version is pending review/scanning, instead of dropping the skill from search during each publish.

## Best Possible Solution

Design a version-aware moderation/search path that keeps the last approved public skill card, digest, and searchable embedding visible while a newly published version is pending VT/LLM/quality review, then promotes the new public projection only after review clears it. Add regression coverage for republishing an already-approved skill and, if maintainers want it, unchanged-content review reuse.
