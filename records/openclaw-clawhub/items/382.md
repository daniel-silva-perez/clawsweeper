## Summary

Keep this issue open. Current main still matches the reported behavior: every skill publish schedules scan/review work, ordinary publishers can have the skill-level moderation status reset to hidden while that work is pending, and public search/listing paths filter the current skill or digest through `toPublicSkill`, which rejects hidden skills. The related open report #1743 describes the same public discovery failure mode, so this should remain tracked rather than be swept closed.

## Best Possible Solution

Keep this issue open and design a version-aware public search/moderation path. The likely direction is to retain the last approved public skill projection, digest, and searchable embedding while a newly published version is pending review, then promote the new latest version only after scan/review clears it. If unchanged-content reuse is added, compare the exact scanner inputs and add regression coverage for already-approved skills republished by non-trusted publishers.
