## Summary

Keep open. This is a ClawHub issue, and the repository policy only allows auto-closing ClawHub PRs that are certainly implemented on main. Current main has trust-channel, trusted-publisher, and package-scan primitives, but it still does not provide a documented community verified-publisher path or a source-backed operational answer for code-plugin scans that remain pending after 24+ hours. Related #1526 is also open with overlapping verified-publisher and scan-classification questions.

## Best Possible Solution

Keep this issue open for maintainer/product follow-up. The best resolution is to publish or document ClawHub's publisher trust model, clearly distinguish package-level GitHub Actions trusted publishing from marketplace verified/official status, define whether a community verified-publisher path will exist, and document scan-status expectations or escalation steps when code-plugin scans remain pending. Maintainers may also choose to consolidate the policy discussion with #1526 if that becomes the canonical verified-publisher thread.
