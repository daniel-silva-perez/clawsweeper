## Summary

Keep open. Current main has verification tiers, trusted-publisher gates, and scheduled package scanner work, but this issue asks for a maintainer trust-policy answer plus production investigation of named code-plugin releases whose scans reportedly stayed pending; source review alone cannot resolve either question, and the ClawHub cleanup policy keeps issues open.

## What This Changes

The issue asks whether active community code-plugin publishers can get verified publisher status and why several named code plugins remained in pending scan state after more than 24 hours.

## Best Possible Solution

Maintainers should publish the current community verified/trusted publisher policy, including whether it is available or intentionally cohort-limited, and inspect production scanner state for the named plugins. If that inspection finds a queue, configuration, backfill, or scan-status finalization bug, track and fix that as a narrower source issue.
