## Summary

Keep open. Current main does not implement the PR’s intended deploy-smoke hardening: the deploy workflow still omits the prod HTTP smoke script and publish/import UI smoke spec, and the current prod HTTP smoke helper still has no 429 retry behavior. The proposed patch is directionally correct, but it touches production deploy CI and still has a small unreachable-code cleanup noted in review.

## What This Changes

The PR adds a production HTTP smoke step to the deploy workflow, expands production UI smoke coverage to publish/import workflows, adds bounded 429 retry handling and skill-detail memoization to prod HTTP smoke, and updates the signed-out upload assertion for the /publish-skill redirect.

## Best Possible Solution

Land a cleaned-up version of this PR that wires the existing prod HTTP smoke script into deploy CI, expands production UI smoke coverage to publish/import workflows, keeps bounded 429 retry behavior and skill-detail memoization without unreachable fallback code, and preserves the current deploy secret and permission boundaries.
