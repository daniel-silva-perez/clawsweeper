## Summary

Keep this PR open. Current main still does not implement the PR's central production smoke hardening: the deploy workflow does not run the production HTTP smoke step, does not include the signed-out publish/import Playwright file in the production UI smoke command, and the HTTP smoke test still lacks the PR's retry/memoization hardening. The PR also remains useful because current app behavior redirects `/upload` to `/publish-skill`, while the current signed-out test still asserts the old upload gate text.

## Best Possible Solution

Keep this PR open for maintainer review. The useful path is to rebase or merge the smoke-coverage changes into main, remove the unreachable post-loop code in the proposed retry helper, and keep the security-sensitive CI change limited to running the existing production HTTP smoke before the production UI smoke plus adding signed-out publish/import coverage.
