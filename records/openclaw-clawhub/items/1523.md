## Summary

Keep open: current main does not already contain this deploy-smoke hardening, and the proposed patch is not merge-ready because the publish-entry smoke assertions do not match the signed-out gate text currently rendered by main. The workflow direction is useful, but this is production deploy CI and needs maintainer review before merge.

## What This Changes

This PR adds a production HTTP smoke step to the deploy workflow, expands production UI smoke coverage to publish/import workflows, adds bounded 429 retry handling and skill-detail memoization to prod HTTP smoke, and updates the signed-out upload redirect assertion.

## Best Possible Solution

Land a corrected version that wires the existing prod HTTP smoke script into deploy CI, expands production UI smoke to publish/import workflows, keeps bounded 429 retry behavior and skill-detail memoization, aligns the publish/import smoke assertions with the rendered gate text, removes the unreachable retry fallback, and preserves current deploy secret and action boundaries.

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

  - body: The updated upload assertion expects `Sign in to publish a skill.` with a trailing period, but `/publish-skill` currently renders `Sign in to publish a skill` without one. The suite also has the same punctuation mismatch for the import gate, and this PR now adds that suite to production deploy smoke, so the deploy smoke can fail even when the app is healthy.   - confidence: 0.92
- **[P3] Remove unreachable retry fallback:** `e2e/prod-http-smoke.e2e.test.ts:81-85`

  - confidence: 0.94
