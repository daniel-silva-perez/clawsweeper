## Summary

Keep open. Current main still lacks the PR’s deploy smoke wiring and 429 retry hardening, and the PR branch has a blocking smoke-test mismatch because the newly deployed publish/import suite expects trailing periods that the signed-out routes do not render.

## What This Changes

The PR adds production HTTP smoke to the deploy workflow, expands production UI smoke to publish/import entry coverage, adds bounded 429 retry handling and skill-detail memoization to prod HTTP smoke, and updates the upload redirect assertion.

## Best Possible Solution

Land a corrected version that wires prod HTTP smoke into deploy CI, adds publish/import UI coverage after aligning the signed-out assertions or route copy, keeps bounded 429 retry behavior and skill-detail memoization, removes the unreachable retry fallback, and preserves the existing deploy secret and action boundaries.
