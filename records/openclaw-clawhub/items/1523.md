## Summary

Current main does not implement this PR’s intended production smoke hardening. The deploy workflow still omits the production HTTP smoke and signed-out publish/import UI smoke, the HTTP smoke test lacks the retry/memoization changes, and the upload gate test still asserts the old text even though /upload redirects to /publish-skill.

## Best Possible Solution

Keep this PR open for maintainer review. The likely path is to rebase or merge the smoke-coverage changes into main, add the HTTP smoke before UI smoke in deploy.yml, include publish-entry-workflows in production UI smoke, update the upload redirect assertion, and address the unreachable fetchWithRetry cleanup noted in review.
