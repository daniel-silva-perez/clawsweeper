## Summary

Keep this issue open. The repository-specific review policy for ClawHub only permits auto-closing PRs that are certainly implemented on main, and this issue still describes a real mismatch: current main continues to advertise and route the SOUL.md/SoulHub experience through onlycrabs.ai, while the live domain check reached Namecheap/Parking behavior rather than the ClawHub app.

## Best Possible Solution

Keep this issue open for maintainer triage. The best fix is either to connect onlycrabs.ai to the production ClawHub/SoulHub app and add host smoke coverage, or to update README/docs/source defaults/tests together so onlycrabs.ai is not presented as a live SOUL.md registry until the domain is connected.
