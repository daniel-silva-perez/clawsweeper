## Summary

Keep this issue open. The ClawHub review policy for this sweep only permits auto-closing PRs that are certainly implemented on main, and the reported mismatch still exists: current main advertises onlycrabs.ai as the SOUL.md/SoulHub host, while a live check of https://onlycrabs.ai timed out instead of serving the ClawHub app. The issue discussion also has a +1 screenshot and a prior ClawSweeper comment reaching the same keep-open conclusion.

## Best Possible Solution

Keep this issue open for maintainer triage. The best fix is either to connect onlycrabs.ai to the production ClawHub/SoulHub app and add host smoke coverage, or to update README/docs/source defaults/tests together so onlycrabs.ai is not presented as a live SOUL.md registry until the domain is connected.
