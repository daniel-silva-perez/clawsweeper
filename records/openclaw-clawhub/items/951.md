## Summary

Keep open. Current main still advertises onlycrabs.ai as the SOUL.md/SoulHub registry in README/docs, runtime defaults, and tests, while a live check on 2026-04-29 still shows onlycrabs.ai is not connected to the ClawHub/SoulHub app. ClawHub repository policy also says issues should stay open for maintainer follow-up rather than auto-close.

## What This Changes

The issue asks to remove or fix README/user-facing references to onlycrabs.ai because the advertised SOUL.md registry domain resolves to parking behavior instead of a working registry.

## Best Possible Solution

Keep this issue open for maintainer triage. Maintainers should decide whether onlycrabs.ai should be connected to production ClawHub/SoulHub; if yes, configure the external domain/DNS/Vercel/env path and add a smoke check. If not, update README, docs, site defaults, tests, and user-facing SoulHub copy together so onlycrabs.ai is not presented as a live registry until it is connected.
