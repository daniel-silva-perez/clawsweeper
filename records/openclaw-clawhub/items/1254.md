## Summary

Keep this issue open. Current main still uses exact-slug injection, vector recall, and a bounded recent-digest lexical fallback; it does not have the requested indexed normalized skill slug/display-name prefix recall path. There is also an open unmerged implementation candidate, PR #1256, whose body uses closing syntax for this issue, so the best path is to review and land that PR or an equivalent fix rather than close the report early.

## Best Possible Solution

Keep this issue open and review PR #1256, or land an equivalent focused implementation. The best fix is to add normalized slug and display-name fields to `skillSearchDigest`, populate them on writes, backfill existing rows safely, add active/non-suspicious prefix indexes, merge direct prefix candidates into `searchSkills` before the bounded recent fallback, and add regression tests for Midscene-style display-name prefix, partial slug, space-separated name queries, and case-insensitive stability.
