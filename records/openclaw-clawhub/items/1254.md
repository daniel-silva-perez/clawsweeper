## Summary

This issue should stay open. The repository review policy for openclaw/clawhub only allows auto-closing pull requests that are certainly implemented on main, not issues. The current main branch also still lacks the requested indexed normalized slug/display-name prefix recall path; it has exact-slug preservation plus bounded recent lexical fallback instead. An open implementation candidate, PR #1256, explicitly closes this issue and is still unmerged in the provided GitHub context.

## Best Possible Solution

Keep this issue open and review PR #1256 as the implementation candidate. The likely maintainer path is to add indexed normalized lexical fields for skill slug and display name on skillSearchDigest, backfill them safely, merge direct prefix candidates into searchSkills before the bounded recent fallback, and add regression tests for Midscene-style display-name prefix, partial slug, and space-separated direct-name queries.
