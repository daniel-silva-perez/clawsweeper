## Summary

Keep open. Current main still lacks the requested indexed normalized slug/display-name prefix recall path, and this issue is already paired with open PR #1256, which should be reviewed or replaced by an equivalent focused fix before closing.

## What This Changes

The issue requests reliable direct slug/display-name prefix recall for skill search when vector search does not retrieve matching skills.

## Best Possible Solution

Keep this issue open and review/land PR #1256, or land an equivalent focused implementation that adds normalized slug/display-name digest fields, active/non-suspicious prefix indexes, digest writes, a cursor backfill, direct prefix candidate merging in searchSkills, and regression tests for Midscene-style name prefixes, partial slug queries, space-separated names, and case-insensitive queries.
