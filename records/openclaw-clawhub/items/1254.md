## Summary

Keep this issue open. The repository policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on main, not issues. Independently, current main still has exact-slug preservation plus bounded recent lexical fallback, but it does not implement the requested indexed normalized display-name/slug prefix recall path. PR #1256 remains the open implementation candidate for this issue.

## Best Possible Solution

Keep this issue open and review PR #1256, or land an equivalent implementation that adds indexed normalized lexical fields for skill slug/displayName on skillSearchDigest, safely backfills them, merges direct prefix candidates into searchSkills before bounded recent fallback, and adds regression tests for Midscene-style display-name prefixes, partial slugs, space-separated direct-name queries, and case-stable prefix recall.
