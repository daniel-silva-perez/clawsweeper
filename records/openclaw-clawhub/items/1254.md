## Summary

Keep this issue open. Repository policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on main, not issues; separately, current main still lacks the requested indexed normalized skill slug/display-name prefix recall path. The existing implementation has exact slug lookup and bounded recent digest fallback, while the open implementation candidate PR #1256 remains unmerged.

## Best Possible Solution

Keep this issue open and review PR #1256, or land an equivalent implementation. The maintainable path is to add normalized slug and display-name fields to skillSearchDigest, populate them on writes, backfill existing rows safely, add active/non-suspicious prefix indexes, merge direct prefix candidates into searchSkills before the bounded recent fallback, and add regression tests for Midscene-style display-name prefix, partial slug, space-separated name queries, and case-insensitive stability.
