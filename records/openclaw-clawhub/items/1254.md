## Summary

Keep open. Current main still has exact slug injection and a bounded recent-digest lexical fallback, but it does not add the requested indexed normalized slug/display-name prefix recall path. The issue is also already paired with open PR #1256 using closing syntax, so maintainer follow-up should happen there or in an equivalent replacement PR.

## What This Changes

The issue asks search to surface direct display-name and slug prefix matches, such as Midscene and Gaggiuino, even when vector search does not recall those skills.

## Best Possible Solution

Land or replace PR #1256 with a focused Convex search change that adds normalized slug/display-name fields to `skillSearchDigest`, active/non-suspicious index-backed prefix lookup, digest writes and cursor backfill, and regression tests for Midscene, Gaggiuino, and ScienceClaw-style case-insensitive prefix and space-separated queries.
