## Summary

Keep open. This is a ClawHub issue, so repository cleanup policy does not auto-close it, and the substantive behavior is only partially fixed: exact-slug search now returns `openclaw-checkpoint`, but the shorter `checkpoint` query still misses the skill. The remaining indexed direct slug/name token or prefix recall gap is already described by the open #1254.

## What This Changes

The issue requests that the published `openclaw-checkpoint` skill appear in ClawHub CLI/API search for both its exact slug and `checkpoint` keyword queries.

## Best Possible Solution

Keep this issue open as concrete reproduction data for #1254. Implement an indexed direct lexical candidate layer for normalized skill slug/display-name exact, token, and prefix matches, merge those candidates ahead of vector/recent fallback ranking, and preserve exact slug behavior plus existing filters.
