## Summary

Keep open. This is an open ClawHub issue, and the supplied repository cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on main. Current main appears to cover exact slug lookup, but the broader keyword/prefix miss remains plausible and is already represented by open related issue #1254.

## Best Possible Solution

Keep this issue open as a concrete repro linked to #1254. The maintainable fix is to add an indexed direct lexical candidate path over normalized slug and display-name fields on skillSearchDigest, merge those candidates into searchSkills ahead of vector/recent fallback ranking, and verify both the CLI and /api/v1/search for exact slug plus keyword/prefix queries such as openclaw-checkpoint and checkpoint.
