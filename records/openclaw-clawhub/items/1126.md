## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main has an indexed exact-slug path that should help `openclaw-checkpoint`, but the broader `checkpoint` token/prefix failure remains plausible because fallback recall still depends on vector results plus bounded recent digest scans. Open #1254 is the better canonical tracker for that remaining search-recall bug and already cites this report.

## Best Possible Solution

Keep this issue open and treat it as a concrete reproduction under #1254. The best code path is to add an indexed direct lexical candidate layer for normalized slug/display-name tokens or equivalent prefix keys, merge those candidates ahead of vector/recent fallback ranking, preserve existing filters, and verify both `/api/v1/search` and `clawhub search` for exact slug and token/prefix queries such as `openclaw-checkpoint`, `checkpoint`, `Midscene`, and `midscene-computer-automation`.
