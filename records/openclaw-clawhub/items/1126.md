## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the search bug is still only partially resolved: production now returns `openclaw-checkpoint` for the exact slug, but the keyword query `checkpoint` still misses it. Current main still has exact-slug injection plus bounded recent-digest fallback scans, but not the indexed normalized slug/display-name prefix candidate path tracked by #1254 and proposed in open PR #1256.

## What This Changes

The issue requests that the published `openclaw-checkpoint` skill appear in ClawHub CLI/API search results for its exact slug and for keyword queries such as `checkpoint`.

## Best Possible Solution

Land or supersede #1256/#1254 with an indexed direct lexical candidate path for normalized skill slug and display-name exact, token, and prefix matches, merged ahead of vector/recent fallback ranking while preserving current filters.
