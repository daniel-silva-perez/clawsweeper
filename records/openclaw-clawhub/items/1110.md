## Summary

Keep this PR open. Current main still does not provide the proposed GET /api/v1/stars endpoint or clawhub list-stars command; the existing web highlights page uses a Convex query directly, not the authenticated HTTP API/CLI surface proposed here. The PR is still potentially useful, but it needs maintainer review, a current-main rebase/path correction, and at least the review findings below addressed before merge.

## What This Changes

The PR adds an authenticated GET /api/v1/stars endpoint, a list-stars CLI command, response schemas, docs, and tests for listing a user's starred skills.

## Best Possible Solution

If maintainers want this feature, land a rebased current-main implementation that registers the exact GET /api/v1/stars route, returns a bounded { items: publicSkill[] } response with normalized tags and public-skill filtering, wires list-stars into packages/clawhub, and covers HTTP/schema/CLI/docs behavior.
