## Summary

Current main does not implement this PR’s intended GET /api/v1/stars endpoint or clawhub list-stars CLI command. The existing web highlights page proves the product concept exists, but it uses a Convex query and is not equivalent to the requested public HTTP/CLI surface. The PR branch remains an implementation candidate with useful review follow-up rather than work that has already shipped on main.

## Best Possible Solution

Keep this PR open for maintainer review. If the feature is desired, land a corrected implementation that registers GET /api/v1/stars, returns { items: publicSkill[] } for the authenticated API-token user with bounded pagination and public visibility filtering, adds the shared response schema, wires clawhub list-stars, and keeps CLI/docs/tests aligned. If maintainers do not want this public API/CLI surface, close the PR manually with that product decision.
