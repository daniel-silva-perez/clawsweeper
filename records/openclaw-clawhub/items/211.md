## Summary

Keep open. This is an openclaw/clawhub issue, and the repository policy for this review only permits auto-closing ClawHub PRs that are certainly implemented on main. Current main also still has unresolved false-positive paths: test and fixture source can be included in both LLM and static moderation scans, suspicious verdicts still gate non-forced CLI install/update flows, and the directly related test-file exclusion PR was closed unmerged.

## Best Possible Solution

Keep this issue open as the product/security tracking item. The best path is to define how scanners should treat security-skill tests and fixtures, add regression tests for .test.*, .spec.*, fixture, and test-directory handling in both LLM and static scan paths, preserve scanning for runtime files, and provide a clearer false-positive review or non-disruptive override path for VirusTotal results so automated installs and updates do not have to rely on --force.
