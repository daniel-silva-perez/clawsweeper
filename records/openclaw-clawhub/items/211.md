## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review only allows auto-closing PRs that are definitely implemented on main. Current main also still shows unresolved scanner and CLI-gating behavior for the reported false-positive class: the broad auth/crypto keyword false positives were narrowed, but test/fixture source is still fed to LLM/static scans and suspicious verdicts still block non-forced automated install/update paths.

## Best Possible Solution

Keep this issue open as the product/security tracking item. The best path is to define explicit scanner semantics for security-skill test fixtures, add direct tests for LLM and static scan treatment of common `.test.*`, `.spec.*`, fixture, and test-directory files, preserve detection in runtime/production files, and provide a clear false-positive review or override path for VirusTotal results that does not require every automated install/update job to use `--force`. Related #270 was closed unmerged, #273 only partially narrowed keyword false positives, and #288 can complement this with behavioral analysis but should not replace fixing these false-positive paths.
