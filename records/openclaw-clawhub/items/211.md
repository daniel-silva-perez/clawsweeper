## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only allows auto-closing ClawHub pull requests that current main certainly implements. Substantively, current main still sends test/spec/fixture-looking source files through both ClawScan LLM evaluation and the deterministic static scan, and the CLI still blocks or skips suspicious skills without `--force`, so the reported false-positive path for security skills is not resolved.

## Best Possible Solution

Keep this issue open as the product/security tracking item. Maintainers should define scanner semantics for test/spec/fixture/mocks paths, apply the shared behavior across ClawScan LLM and deterministic static moderation, preserve scanning for runtime files, and improve review/override plus CLI install/update behavior so known false positives do not unnecessarily block legitimate security skills.
