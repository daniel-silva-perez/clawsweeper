## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy for this review pass only permits auto-closing ClawHub PRs that current main certainly implements. Substantively, current main still lacks explicit test/spec/fixture/mock semantics in both ClawScan input collection and deterministic static moderation, while CLI install/update still gates suspicious skills behind prompt or --force. Related PR #270 proposed the test-file exclusion but closed unmerged; merged PR #273 only narrowed some keyword false positives.

## Best Possible Solution

Keep this open for product/security maintainer judgment. The best path is to define explicit scanner semantics for test/spec/fixture/mock/example content, apply that policy consistently across LLM and deterministic static moderation, preserve scanning for runtime files and any install/hook-executed files, and ensure trusted/manual false-positive clearance updates the moderation state consumed by API and CLI install/update paths.
