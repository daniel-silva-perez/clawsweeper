## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only allows auto-closing ClawHub pull requests that current main certainly implements. Substantively, current main still feeds test/spec/fixture-looking source into ClawScan LLM evaluation and static moderation, and the linked openclaw-sec latest scan still reports a suspicious LLM result even though the scan text acknowledges the dangerous strings are expected security-test/example inputs.

## Best Possible Solution

Keep this open as the product/security tracking issue. Maintainers should define explicit scanner semantics for test/spec/fixture/mock/example content, apply that shared policy across LLM evaluation and deterministic static moderation, preserve scanning for runtime or hook-installed files, and make CLI install/update behavior reflect trusted review or cleared false positives without weakening real malicious-skill detection.
