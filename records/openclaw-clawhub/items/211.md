## Summary

Keep open. ClawHub cleanup policy keeps open issues open in this lane, and the underlying false-positive problem still has unresolved scanner semantics: current main still sends all non-SKILL files to the LLM review, statically scans all text files without test/spec/fixture/mock treatment, and CLI install/update still gates suspicious skills unless a human confirms or passes --force.

## What This Changes

Security skills with malicious-looking test or fixture strings are being flagged by ClawHub security scanning and then blocked or skipped during install/update unless overridden.

## Best Possible Solution

Keep this open for product/security maintainer judgment. The best path is to define explicit scanner semantics for test/spec/fixture/mock/example content, apply that policy consistently across LLM and deterministic static moderation, preserve scanning for runtime files and install/hook-executed files, and ensure reviewed false-positive clearance updates the moderation state consumed by API and CLI install/update paths.
