## Summary

Keep open. Repository cleanup policy keeps ClawHub issues open, and the underlying product question is still real: the current scanner paths do not define test/spec/fixture semantics, while suspicious results still affect automated CLI install/update flows.

## What This Changes

The issue asks ClawHub to stop security-audit false positives that flag defensive security skills for malicious-looking test strings and then block automated install/update.

## Best Possible Solution

Define and document scanner semantics for test/spec/fixture/mock/example content, apply them consistently to LLM and deterministic static scans, keep runtime and install-executed files in scope, and ensure staff-reviewed false-positive clearance is reflected in API and CLI install/update behavior.
