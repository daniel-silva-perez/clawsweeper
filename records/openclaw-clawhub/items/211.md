## Summary

Keep open. This is an open ClawHub issue, and the repo cleanup policy only permits auto-closing ClawHub PRs that current main certainly implements. Substantively, current main still sends all non-SKILL files to ClawScan and scans all text-like uploaded files with static moderation, without explicit test/spec/fixture semantics; CLI install/update still blocks suspicious skills without --force. Related PR #270 proposed a test-file exclusion but closed unmerged, while merged PR #273 only addressed narrower OAuth/static keyword false positives.

## Best Possible Solution

Keep this open for product/security maintainer judgment. The best path is to define explicit scanner semantics for test/spec/fixture/mock/example content, apply that policy consistently across ClawScan and deterministic static moderation, preserve scanning for runtime files and install/hook-executed files, and make manual or trusted-review clearance flow through the public moderation state so CLI install/update behavior reflects cleared false positives without weakening malicious-skill detection.

## Risks / Open Questions
