## Summary

Keep open. ClawHub cleanup policy keeps issues open in this lane, and the substantive false-positive path still exists on current main: LLM evaluation and static moderation both inspect non-SKILL text files without test/spec/fixture/mock semantics, while CLI install/update still gates suspicious skills unless the user confirms or passes force. Related PR #270 closed unmerged, and merged PR #273 only narrowed broad keyword false positives.

## What This Changes

The issue asks ClawHub to stop security-audit false positives that flag legitimate security skills containing malicious-looking test strings and prevent bot install/update flows.

## Best Possible Solution

Define and document scanner semantics for test/spec/fixture/mock/example content, apply them consistently across LLM and deterministic static scans, keep runtime and install-executed files in scope, and ensure staff-reviewed false-positive clearance propagates to API and CLI install/update behavior.
