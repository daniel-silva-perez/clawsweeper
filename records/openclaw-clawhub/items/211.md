## Summary

Keep open. ClawHub cleanup policy keeps open issues open in this lane, and the underlying false-positive problem remains unresolved: current main still feeds non-SKILL files into the LLM review, statically scans all text files without test/spec/fixture/mock semantics, and CLI install/update still gates suspicious skills unless the user confirms or passes --force. Related PR #270 proposed a test-file exclusion but closed unmerged, while merged PR #273 only partially reduced broad keyword false positives.

## What This Changes

The issue asks ClawHub to remove or redesign the security audit behavior that flags security skills when their test files contain intentionally malicious-looking strings used for validation.

## Best Possible Solution

Define explicit scanner semantics for test/spec/fixture/mock/example content, apply them consistently across LLM and deterministic static moderation, preserve scanning for runtime files and install/hook-executed files, and ensure staff-reviewed false-positive clearance updates the moderation state consumed by API and CLI install/update paths.
