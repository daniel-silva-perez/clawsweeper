## Summary

Keep open. ClawHub cleanup policy keeps issues open in this lane, and current main still has the reported false-positive path: LLM evaluation and static scanning include non-SKILL text/code files without test/spec/fixture semantics, while CLI install/update still blocks or skips suspicious skills unless a user confirms or passes force. Related PR #270 proposed a direct test-file exclusion but closed unmerged; merged PR #273 only narrowed broader keyword false positives.

## What This Changes

Security skills with malicious-looking test strings are being flagged by ClawHub security auditing, which can block bot installs or updates; the reporter asks to remove or change that audit behavior.

## Best Possible Solution

Define and document scanner semantics for test/spec/fixture/mock/example content, apply them consistently to LLM and deterministic static scans, keep runtime and install-executed files in scope, and ensure staff-reviewed false-positive clearance is reflected in API and CLI install/update behavior.
