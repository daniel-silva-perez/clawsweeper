## Summary

Keep open. The repository-specific cleanup policy says ClawHub issues should stay open, and the current main branch does not fully solve the reported false-positive path: LLM and static moderation still ingest test/spec/fixture-looking source files, and non-forced CLI install/update still gates suspicious skills. PR #273 reduced one broad keyword false-positive class, but PR #270, which targeted test-file exclusion for this report, was closed unmerged.

## Best Possible Solution

Keep this issue open as the product/security tracking item. Maintainers should define how security scanners treat test/spec/fixture/mocks files, implement shared behavior across LLM and static scan paths with regression coverage for .test.*, .spec.*, __tests__, fixtures, mocks, and test directories, preserve scanning for runtime files, and improve manual review/override plus CLI install/update behavior so known false positives do not unnecessarily block legitimate security skills.

- The user-visible problem spans LLM evaluation, deterministic static scanning, VirusTotal aggregation, manual override/review, and CLI install/update behavior; fixing only one scanner may leave security skills blocked.
