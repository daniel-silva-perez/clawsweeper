## Summary

Keep this PR open. Current main still does not implement the PR's primary digest-based GitHub backup pagination: `getGitHubBackupPageInternal` continues to page through full `skills` documents, join `skillVersions` and `users`, and include version `files` during page construction. Main does contain the later fast-forward retry work, but that only covers one commit from the PR and does not satisfy the performance change in the title and body.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. The best path is to revise the digest-based scan so `getGitHubBackupPageInternal` pages over `skillSearchDigest`, preserves the existing public eligibility semantics, handles empty owner handles with the current user-id fallback behavior, revalidates or refreshes live owner state before choosing backup paths, fetches version files lazily only after detecting a GitHub version mismatch, and retains the fast-forward retry behavior already present on main.
