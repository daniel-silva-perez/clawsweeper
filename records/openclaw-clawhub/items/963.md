## Summary

Keep this PR open. Current main still does not implement the PR‚Äôs central change: `getGitHubBackupPageInternal` still paginates full `skills` documents, joins `skillVersions` and `users` per row, and returns version `files` in the page item so the sync action can pass them directly to `backupSkillToGitHub`. Main contains the GitHub fast-forward retry behavior from the PR‚Äôs final commit theme, but that is only partial overlap and does not close the digest-pagination/lazy-file-fetch performance work.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. The best path is to revise the digest-based backup scan so `getGitHubBackupPageInternal` pages over `skillSearchDigest`, preserves handle-less owner fallback behavior, revalidates or refreshes owner state where needed, fetches version files lazily only after a GitHub version mismatch, and keeps the fast-forward retry behavior already present on main.
