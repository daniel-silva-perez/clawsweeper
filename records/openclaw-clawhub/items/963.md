## Summary

Keep this PR open. Current main still does not implement the PR’s central digest-based GitHub backup scan or lazy version-file fetch: the backup page still paginates full skills, joins versions and users while building the page, and passes eager files to the Node backup action. Part of the PR’s adjacent fast-forward retry behavior is already present on main, but that does not cover the main performance change, and the PR still has review concerns around owner fallback and live owner revalidation for a path that writes GitHub backup content.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. The best path is to revise the digest-based scan so getGitHubBackupPageInternal pages skillSearchDigest, fetches version files lazily only after a GitHub version mismatch, preserves the current handle-less owner fallback, revalidates or keeps fresh live owner state for renamed/deactivated/deleted owners, and retains the current fast-forward retry behavior in GitHub writes/deletes.
