## Summary

Keep this PR open. Current main still does not implement the PR’s central digest-pagination and lazy-file-fetch change: the GitHub backup cron page still scans full `skills` documents, reads `skillVersions` and `users` for each eligible row, and passes `version.files` eagerly into the Node backup action. Main has partial overlap only in the GitHub fast-forward retry behavior, which does not solve the read-amplification problem this PR targets.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. The best path is to revise the digest-based backup scan so it pages over `skillSearchDigest`, fetches version files lazily only after a GitHub version mismatch, preserves the current handle-less owner fallback, refreshes or revalidates live owner state where needed, and retains the fast-forward retry behavior already present on main.
