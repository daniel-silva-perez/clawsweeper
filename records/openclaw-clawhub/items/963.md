## Summary

Keep open. Current main still uses full `skills` pagination with owner/version joins and eager version files, so the main performance goal is not implemented; only the fast-forward retry part is already present. The PR targets a real hot-path cost, but the review thread identifies owner fallback and stale denormalized owner-state concerns that need maintainer or contributor follow-up before merge.

## What This Changes

The PR changes GitHub backup sync to paginate `skillSearchDigest`, avoid owner/version joins during the scan phase, fetch version files lazily after a GitHub version mismatch, update backup tests, remove the missing-owner stat, and retry GitHub backup pushes on fast-forward conflicts.

## Best Possible Solution

Land a revised digest-based backup scan that reads `skillSearchDigest` for the cheap scan, keeps correct owner fallback and live owner eligibility semantics or refreshes digest owner metadata reliably, fetches version files only after a GitHub version mismatch, handles stale cursors visibly, preserves the existing fast-forward retry behavior, and proves the change with focused tests plus non-production Convex validation.
