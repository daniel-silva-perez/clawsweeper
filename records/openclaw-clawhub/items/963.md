## Summary

Keep this PR open. Current main still does not implement the PR’s main digest-based GitHub backup pagination: the backup page continues to paginate full `skills` documents, join `skillVersions` and `users`, and return `version.files` during page construction. Main appears to include the later fast-forward retry behavior from the PR, but that is only a partial overlap and does not satisfy the performance change described by the title/body.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. The best path is to revise the digest-based backup scan so `getGitHubBackupPageInternal` pages over `skillSearchDigest`, preserves current public eligibility and handle-less owner fallback behavior, revalidates or refreshes owner state where needed, fetches version files lazily only after a GitHub version mismatch, and keeps the fast-forward retry behavior already present on main.
