## Summary

Keep this PR open. Current main still does not implement the PR's central digest-based GitHub backup pagination or lazy version-file loading: the backup page still scans full `skills` documents, joins live `skillVersions` and `users` records while constructing each page item, and sends eager `files` into the Node backup action. Main does include the adjacent fast-forward retry behavior in `convex/lib/githubBackup.ts`, but that is not the core performance change this PR proposes.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. The best path is to revise the digest-based backup scan so `getGitHubBackupPageInternal` pages `skillSearchDigest`, fetches version files lazily only after a GitHub version mismatch, preserves the current owner-id fallback for owners without handles, and either revalidates live owner state or keeps skill digest owner metadata fresh when users rename, deactivate, or delete accounts. The already-present fast-forward retry behavior should be retained.
