## Summary

Keep this PR open. Current main still does not implement the PR’s central digest-pagination and lazy-file-fetch change: the GitHub backup page still paginates full `skills` documents, joins `skillVersions` and `users` during page construction, and carries `version.files` eagerly into the Node backup action. The PR also has unresolved owner/path correctness concerns from review that matter because this code writes GitHub backup paths from owner and slug data.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. The best path is to revise the digest-based backup scan so it pages over `skillSearchDigest`, fetches version files lazily only after a GitHub version mismatch, preserves the current handle-less owner fallback, refreshes or revalidates live owner state where needed, and keeps the fast-forward retry behavior already present on current main.
