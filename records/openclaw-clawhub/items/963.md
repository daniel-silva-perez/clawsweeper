## Summary

Keep this PR open. Current main still does not implement the PR's central digest-based GitHub backup scan or lazy version-file fetch: the backup page still paginates full `skills` documents, joins `skillVersions` and `users` while constructing the page, and passes eager `files` through to the Node backup action. Main does contain the PR's adjacent fast-forward retry behavior in `convex/lib/githubBackup.ts`, but that is not the main performance change. The PR also still has meaningful review concerns around handle-less owner fallback and stale/live owner revalidation for a path that writes GitHub backup content.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. The best path is to revise the digest-based scan so `getGitHubBackupPageInternal` pages `skillSearchDigest`, fetches version files lazily only after a GitHub version mismatch, preserves the current fallback for owners without handles, revalidates or keeps fresh live owner state for renamed/deactivated/deleted owners, and retains the fast-forward retry behavior already present on main.
