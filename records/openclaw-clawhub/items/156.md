## Summary

Keep this issue open. The ClawHub review policy only allows auto-closing pull requests that are certainly implemented on main, and this item is an issue. Current main records download and star events, but the visible skill-document counters are intentionally batch-synced on a 6-hour cron, so the reporter's visible-metrics concern remains a valid maintainer/product triage item.

## Best Possible Solution

Keep this issue open for maintainer triage. The concrete path is to verify production behavior end to end, then decide whether to keep the 6-hour visible-counter delay, reduce the skill-doc-stat-sync cadence, expose a fresher read model for visible counters, or document the delay and hourly download dedupe behavior.
