## Summary

Keep this issue open. Current main records download and star events, but the user-visible counters still depend on a delayed skill-document sync path, and the openclaw/clawhub cleanup policy says issues are not eligible for automatic close.

## Best Possible Solution

Keep this issue open for maintainer triage. Verify production cron health and event backlog, then decide whether to make visible counters read from a fresher source, shorten or trigger skill-doc stat sync with reactive-load safeguards, or document the expected delay and hourly download dedupe behavior. A corrected replacement for the closed PR #176 would be the likely code path if faster visible updates are desired.
