## Summary

Keep this issue open. The ClawHub review policy only allows auto-closing pull requests that are certainly implemented on main, and this item is an issue. Current main records download and star events, but visible public counters are still delayed by the 6-hour skill document sync, so the reported metrics concern remains a valid maintainer triage item.

## Best Possible Solution

Keep this issue open for maintainer triage. Verify production behavior end to end, then decide whether to shorten the skill-doc-stat-sync cadence, update visible counters through a fresher read model, or document the 6-hour counter delay and hourly download dedupe behavior.
