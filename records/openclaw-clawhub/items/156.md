## Summary

Keep this issue open. Current main records download and star events and eventually patches skill counters, but the user-visible counters still depend on the delayed skill-document sync path. The linked responsiveness PR #176 was closed unmerged, and the ClawHub cleanup policy says issues are not eligible for automatic close.

## Best Possible Solution

Keep this issue open for maintainer triage. Verify production cron health and any skillStatEvents backlog, then decide whether visible counters should read from a fresher source, whether skill-doc stat sync should run sooner with reactive-load safeguards, or whether the product should document the expected delay and hourly download dedupe behavior. A corrected replacement for the closed PR #176 is the likely code path if faster visible updates are desired.
