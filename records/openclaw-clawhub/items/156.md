## Summary

Keep this issue open. Current main records download and star events, but the user-visible counters still read denormalized skill document fields that are only patched by the 6-hour skill-doc stat sync. The related responsiveness PR #176 was closed unmerged, and the ClawHub review policy for this pass says open issues should not be auto-closed.

## Best Possible Solution

Keep this open for maintainer triage. Verify production cron health and any skillStatEvents backlog, then choose the intended freshness model: faster skill-doc stat sync with load safeguards, a fresher read model for visible counters, optimistic UI for local star actions, or explicit documentation of the 6-hour sync and hourly download dedupe behavior. A corrected replacement for the closed #176 is the likely path if faster visible updates are desired.
