## Summary

Keep this issue open. Current main records download and star events, but visible skill counters still read denormalized skill-document stats that are only flushed by the scheduled skill-doc sync, so the responsiveness complaint remains plausible. The related cadence PR was closed unmerged, and ClawHub cleanup policy keeps issues open unless maintainers decide otherwise.

## What This Changes

Report that starring skills and downloading skills from the CLI or website do not update the visible download/star metrics on the skill page.

## Best Possible Solution

Keep this open for maintainer triage. Verify production cron health and skillStatEvents backlog, then choose the intended freshness model: safer/faster skill-doc sync cadence, a lightweight live read or overlay for visible counters, optimistic UI for star toggles, or explicit docs/UI copy for the 6-hour visible counter latency and hourly download dedupe.
