## Summary

Keep this issue open. Current main records download and star events, but the user-visible skill counters still depend on skill-document stats that are only flushed by the scheduled skill-doc sync, so the responsiveness complaint remains plausible and needs maintainer product/performance judgment.

## What This Changes

Download and star metrics should update correctly and responsively after CLI/download-button downloads and star toggles.

## Best Possible Solution

Keep this open for maintainer triage. Verify production cron health and skillStatEvents backlog, then choose the intended freshness model: a safer/faster skill-doc sync cadence, a lightweight live read model for visible counters, optimistic UI for local star toggles, or explicit docs for 6-hour visible counter latency and hourly download dedupe.
