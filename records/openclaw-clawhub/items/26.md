## Summary

Keep this issue open. The repository cleanup policy for ClawHub keeps issues open, and current main still does not clearly implement the related PR #27 behavior that would relax multi-token search matching. The active skill search path still filters vector and fallback candidates through a matcher requiring every query token to prefix-match.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely path is to recover or request the exact failing query, decide whether skill search should require all query tokens or accept semantically relevant partial token matches, then update convex/lib/searchText.ts with matching coverage in convex/lib/searchText.test.ts and search/UI or API regression tests. If strict all-token matching is intentional now, a maintainer should close this manually with that product decision.
