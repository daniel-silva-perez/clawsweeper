## Summary

Keep open. Current main still implements only static, VirusTotal, and LLM/ClawScan security signals; there is no behavioral/Oathe scanner in schema, publish/rescan lifecycle, cron polling, security UI, scanner detail routes, public scan snapshot, or browse/search sorting/filtering. The Oathe team comment and closed unmerged PR #562 are useful implementation context, but the feature is not present on main and ClawHub issues are not auto-close candidates under the repository policy.

## What This Changes

The issue requests a behavioral sandbox security signal, with Oathe-style scores and reports, alongside VirusTotal and LLM/static scanner data on skill detail, browse/search, and API surfaces.

## Best Possible Solution

Define a reviewed, opt-in behavioral scanner provider contract, then implement persisted scan results, publish/rescan submission, bounded indexed polling, public API exposure, skill detail/report UI, and indexed browse/search sort or filter fields once the data model and trust semantics are settled.
