## Summary

Keep open. This is an issue, and the ClawHub cleanup policy only allows auto-closing PRs that are certainly implemented on current main. Current main still has static, VirusTotal, OpenClaw LLM, and capability signals, but no Oathe/behavioral sandbox persistence, publish trigger, polling lifecycle, public API fields, skill-detail UI row/badge, or browse/search sort/filter. The Oathe comment and closed-unmerged PR #562 are useful implementation context, but the feature is not present on main.

## Best Possible Solution

Keep this open for maintainer product and security review. The concrete path is to revive or replace PR #562 with a reviewed, opt-in behavioral scan integration that persists provider results on skill versions, safely submits and polls scans, exposes score/verdict/report data in the skill detail UI and public API, and adds browse/search sort or filter support once the provider contract and data model are stable.
