## Summary

Keep open. Current main still lacks an Oathe or generic behavioral sandbox scanner across storage, publish/rescan lifecycle, API, detail UI, and browse/search sort or filter support; the Oathe team comment and closed, unmerged PR #562 are useful context but do not make the feature present on main, and ClawHub issues are not auto-close candidates under the repository policy.

## What This Changes

Add a behavioral sandbox security signal, proposed as Oathe, alongside VirusTotal and ClawScan, with full report links plus browse/search sorting and filtering support.

## Best Possible Solution

Keep this open for maintainer product and security review. The best path is to revive or replace PR #562 with a scoped, opt-in behavioral scanner provider contract: persist behavioral scan results on skill versions, safely submit/rescan on publish, poll with bounded indexed crons, expose score/verdict/report data through the public scan API and skill detail UI, then add indexed browse/search sort or filter support once the data model is settled.
