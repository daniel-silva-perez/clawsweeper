## Summary

Keep this issue open. It is a ClawHub issue, and the repository-specific cleanup policy says ClawHub issues are not eligible for auto-close. Current main also still lacks the requested Oathe/behavioral security signal: there is no persisted behavioral scan field, no publish trigger, no poller, no UI row/badge, no API scanner entry, and no browse/search sort or filter for behavioral scores.

## Best Possible Solution

Keep this issue open for maintainer product and security judgment. The practical path is to revive or replace #562 with a reviewed implementation that persists behavioral scan results on skill versions, gates provider calls behind explicit configuration, safely triggers and polls scans, exposes score/verdict/report details in the skill detail UI and public API, and only then adds browse/search sort and filter support.
