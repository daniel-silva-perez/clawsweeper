## Summary

Keep open. Current main still implements static, VirusTotal, and LLM/ClawScan security signals only; the requested behavioral/Oathe signal is absent from storage, publish/rescan lifecycle, cron polling, public security snapshots, scanner pages, and browse/search sort/filter fields. The Oathe team comment and closed unmerged PR #562 are useful context, but this is an open ClawHub issue and needs maintainer security/product judgment before implementation.

## What This Changes

Add a behavioral/Oathe sandbox security signal alongside VirusTotal and LLM/ClawScan, with persisted scores, report links, skill-detail display, and browse/search filtering or sorting.

## Best Possible Solution

Define a reviewed, opt-in behavioral scanner provider contract, then implement persisted scan results, publish/rescan submission, bounded indexed polling, public API exposure, skill detail/report UI, and indexed browse/search sort or filter fields once trust semantics are settled.
