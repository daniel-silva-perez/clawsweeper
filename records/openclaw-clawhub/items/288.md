## Summary

Keep open. This is a ClawHub issue, and the cleanup policy for this repo only permits auto-closing pull requests that are certainly implemented on main. Current main still has VT, ClawScan/LLM, and static analysis surfaces, but no Oathe or behavioral sandbox scanner schema, publish trigger, cron poller, API snapshot field, skill detail row, or browse/search sort/filter support. The Oathe team did confirm their docs/endpoints are live in the thread, and PR #562 was a concrete implementation attempt, but that PR was closed unmerged.

## Best Possible Solution

Keep this open for maintainer product and security review. The likely path is to revive or replace PR #562 with a narrowly scoped, opt-in behavioral scanner integration: persist provider results on skill versions, submit scans on publish/rescan, poll pending results safely, expose score/verdict/report data through the API and skill detail UI, then add browse/search sort or filter support only after the provider contract and indexed data model are settled.
