## Summary

Keep this issue open. The ClawHub cleanup policy for this review says issues stay open, and current main still has no Oathe or behavioral sandbox scanner model, publish trigger, cron poller, public scan payload, skill-detail row, scanner route, or browse/search sort/filter support. The Oathe team comment and closed unmerged PR #562 are useful implementation context, but they do not make the feature present on main.

## Best Possible Solution

Keep this open for maintainer product and security review. The best path is to revive or replace PR #562 with a scoped, opt-in behavioral scanner integration: define the provider contract, persist behavioral scan results on skill versions, trigger submit/rescan safely, poll results with bounded crons, expose score/verdict/report data through the public scan API and skill detail UI, then add indexed browse/search sort or filter support once the data model is settled.

## Risks / Open Questions

- A behavioral scanner integration is security-sensitive: it may send skill bundles or metadata to a third-party provider and needs explicit provider, retention, secret, failure, and opt-in policy decisions.
- Browse/search sorting or filtering by behavioral score needs denormalized, indexed fields; adding it without a schema/index plan risks expensive Convex reads.
- Closed PR #562 is useful prior art but was not merged; any revival should be re-reviewed, especially for provider calls, secrets, cron behavior, and any CI/workflow changes.
