## Summary

Keep open. Current main still has only static, VirusTotal, and LLM/ClawScan security signals, with no behavioral/Oathe scanner in storage, scan lifecycle, cron polling, public API snapshots, scanner routes, or browse/search controls; repository policy also keeps ClawHub issues open for maintainer judgment.

## What This Changes

The issue requests adding a behavioral sandbox security signal, using Oathe-style runtime scan scores and reports, alongside the existing VirusTotal and LLM/static security signals.

## Best Possible Solution

Define a reviewed, opt-in behavioral scanner provider contract, then implement persisted scan results, publish/rescan submission, bounded indexed polling, public API exposure, skill detail/report UI, and indexed browse/search sort or filter fields once the data model and trust semantics are settled.
