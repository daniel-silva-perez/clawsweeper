## Summary

Keep open. Current main still lacks the requested behavioral/Oathe scanner across storage, publish/rescan lifecycle, cron polling, public API snapshots, skill detail UI, scanner routes, and browse/search controls; under this repository policy, ClawHub issues are not auto-close candidates.

## What This Changes

The issue requests a behavioral security analysis signal, ideally Oathe-backed, displayed and queryable alongside VirusTotal and ClawScan/LLM scan results.

## Best Possible Solution

Define a reviewed, opt-in behavioral scanner provider contract, then implement persisted scan results, publish/rescan submission, bounded indexed polling, public API exposure, skill detail/report UI, and indexed browse/search sort or filter fields once the data model and trust semantics are settled.
