## Summary

Keep open. This is a ClawHub issue, and the cleanup policy for this review run keeps issues open unless maintainers decide otherwise. Current main also still lacks the requested behavioral/Oathe scanner storage, publish trigger, cron polling, public API fields, skill-detail UI row, and browse/search sort or filter support. The Oathe comment and closed-unmerged PR #562 are useful prior context, but the feature is not implemented on main.

## Best Possible Solution

Keep this open for maintainer product and security review. The concrete path is to revive or replace PR #562 with a reviewed, opt-in behavioral scan integration that persists provider results on skill versions, safely submits and polls scans, exposes score/verdict/report data in the skill detail UI and public API, and adds browse/search sort or filter support once the provider contract and data model are stable.
