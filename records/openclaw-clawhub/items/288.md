## Summary

Keep this open. ClawHub policy only allows auto-close for PRs that are definitely implemented on main, and this item is an issue. Independently, current main still lacks the requested behavioral/Oathe security signal: no schema field, scanner action, cron poller, API scanner entry, or skill-detail UI row exists for it. The Oathe team comment and closed-unmerged #562 are useful integration context, but they do not make the feature present on main.

## Best Possible Solution

Keep this issue open for maintainer product and security judgment. The concrete path is to revive or replace #562 with a reviewed implementation that persists behavioral scan results on skill versions, gates provider calls behind explicit configuration, safely triggers and polls scans, exposes verdict/score/report details in the skill detail UI and public API, and adds browse/search sort and filter support once the data model is stable.
