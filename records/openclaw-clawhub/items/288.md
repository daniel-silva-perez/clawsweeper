## Summary

Keep this issue open. The ClawHub review policy says issues are not eligible for auto-close, and current main still does not implement the requested Oathe/behavioral security signal. The Oathe team did respond that their docs/API are live, and PR #562 attempted this integration, but that PR was closed unmerged.

## Best Possible Solution

Keep this issue open for maintainer product and security judgment. The practical path is to revive or replace PR #562 with a reviewed design that persists behavioral scan results on skill versions, gates provider calls behind explicit configuration, safely triggers and polls scans, exposes score/verdict/report details in the skill detail UI and API, and only then adds browse/search sort and filter support.
