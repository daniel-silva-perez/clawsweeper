## Summary

Keep this open. The ClawHub cleanup policy for this review only permits auto-closing PRs that are certainly implemented on main, and this item is an issue. Independently, current main still lacks the requested Oathe/behavioral sandbox signal in persistence, scan lifecycle, API output, skill-detail UI, and browse/search sort or filter controls. ## Best Possible Solution

Keep this issue open for maintainer product and security judgment. The practical path is to revive or replace PR #562 with a reviewed, opt-in implementation that persists behavioral scan results on skill versions, safely submits and polls provider scans, exposes score/verdict/report data in the skill detail UI and public API, and adds browse/search sort or filter support once the data model is stable.
