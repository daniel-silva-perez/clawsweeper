## Summary

Keep open. This is a ClawHub issue, and the cleanup policy for this repository only permits auto-closing pull requests that are certainly implemented on current main. Current main still lacks an Oathe or behavioral sandbox scanner data model, publish trigger, cron poller, API scanner entry, skill detail row, scanner route, or browse/search sort/filter support. The Oathe team’s comment and closed PR #562 are useful implementation context, but the PR was not merged and the feature is absent from main.

## Best Possible Solution

Keep this open for maintainer product and security review. The best path is to revive or replace PR #562 with a narrowly scoped, opt-in behavioral scanner integration: define the provider contract, persist behavioral results on skill versions, submit on publish/rescan, poll safely, expose score/verdict/report data through the API and skill detail UI, and add browse/search sort or filter support only after the indexed data model is settled.
