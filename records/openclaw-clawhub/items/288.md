## Summary

Keep open. Current main still lacks the requested behavioral/Oathe scanner across storage, publish/rescan lifecycle, cron polling, public scan snapshots, skill detail UI, scanner routes, and browse/search sort or filter support; as a ClawHub issue, it is also not an auto-close candidate under the repository policy.

## What This Changes

Add a behavioral sandbox security signal, suggested as Oathe, alongside existing static, VirusTotal, and ClawScan/LLM signals with detail UI, reports, and browse/search sorting or filtering.

## Best Possible Solution

Define a reviewed, opt-in behavioral scanner provider contract, then implement persisted scan results, publish/rescan submission, bounded indexed polling, public API/CLI exposure, skill detail/report UI, and indexed browse/search sort or filter fields once trust semantics are settled.
