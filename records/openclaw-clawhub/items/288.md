## Summary

Keep open. Current main still lacks an Oathe or generic behavioral sandbox scanner in storage, publish/rescan lifecycle, API exposure, skill detail UI, and browse/search sort or filter support. The Oathe docs and the closed, unmerged PR #562 are useful implementation context, but they do not make the feature present on main, and ClawHub issues are not auto-close candidates under the repository policy.

## What This Changes

Request to add an Oathe-backed behavioral sandbox security signal with skill detail badges/report links plus browse/search sort and filter support alongside VirusTotal and ClawScan.

## Best Possible Solution

Define a reviewed, opt-in behavioral scanner provider contract, then implement persisted scan results, publish/rescan submission, bounded indexed polling, public scan/detail API exposure, skill detail UI/report links, and indexed browse/search sort or filter fields once the data model is settled.
