Keep open. ClawHub cleanup policy keeps open issues open, and the substantive request is not implemented: current main and the live API still do not treat top-level `required_credentials` as supported credential metadata for this skill.

## What This Changes

The issue asks ClawHub to clear the `buff-roundup` suspicious flag by recognizing credentials and permissions declared in top-level `SKILL.md` frontmatter.

## Best Possible Solution

Keep this open for maintainer/security moderation review. Maintainers should decide whether top-level `required_credentials` and `permissions` are supported ClawHub aliases; if yes, implement that consistently through the shared schema, SKILL.md parser, static declaration collector, LLM prompt context, UI/API/docs/tests, and a rescan/backfill path. If not, ask the publisher to republish with the documented `metadata.openclaw.requires.env`, `primaryEnv`, and/or `envVars` shape, then rescan or make an explicit manual moderation decision for `buff-roundup`.
