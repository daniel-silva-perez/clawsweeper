## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that current main definitely implements. Substantively, current main still does not support top-level `required_credentials` as registry metadata or as a static moderation declaration source, and the live `buff-roundup` listing remains suspicious with `metadata: null` while its published SKILL.md uses `required_credentials`.

## Best Possible Solution

Keep this open for maintainer/security moderation review. The concrete path is to decide whether top-level `required_credentials` should become a supported alias for credential declarations and wire it through schema, parser, static moderation, prompt/docs, tests, and rescanning; otherwise ask the publisher to republish using the documented `metadata.openclaw.requires.env`, `primaryEnv`, and/or `envVars` shape, then rescan `buff-roundup` or apply the existing clean override if staff verifies the false positive.
