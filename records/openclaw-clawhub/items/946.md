## Summary

Keep open. ClawHub cleanup policy keeps open issues open, and the substantive request is still unresolved: current main does not treat top-level `required_credentials` or `permissions` as supported credential metadata, while the live `buff-roundup` API still shows `metadata: null` and a suspicious moderation verdict.

## What This Changes

The reporter asks ClawHub to read `buff-roundup`'s top-level `SKILL.md` `required_credentials` and `permissions` declarations as registry/scanner metadata so the suspicious flag can be cleared.

## Best Possible Solution

Maintainers should make an explicit security/product decision: either require the publisher to republish using the documented `metadata.openclaw.requires.env`, `primaryEnv`, and/or `envVars` shape and then rescan or manually review `buff-roundup`, or add first-class alias support consistently through schema, parsing, static scan declaration collection, LLM prompt context, UI/API/docs/tests, and a backfill/rescan path.
