The ClawHub cleanup policy for this review keeps open issues open, and the substantive request is still unresolved: current main does not parse the reporter's top-level `required_credentials` or `permissions` frontmatter into ClawHub credential metadata, and the live API still reports the skill as suspicious with `metadata: null`.

## What This Changes

The issue asks ClawHub to read `required_credentials` and `permissions` from `buff-roundup`'s SKILL.md frontmatter and clear the suspicious moderation flag.

## Best Possible Solution

Maintainers should decide whether top-level `required_credentials` and `permissions` are supported ClawHub aliases. If yes, implement them consistently in the shared schema, SKILL.md parser, static declaration collector, LLM prompt context, UI/API/docs/tests, and a rescan/backfill path; if not, ask the publisher to republish with the documented `metadata.openclaw.requires.env`, `primaryEnv`, or `envVars` shape and then make an explicit moderation decision for the skill.
