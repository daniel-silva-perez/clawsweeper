## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy supplied for this review only allows auto-closing pull requests that current main definitely implements. Substantively, current main still does not parse the reporter's top-level `required_credentials` and `permissions` frontmatter into ClawHub credential metadata, so the moderation appeal remains a maintainer/security policy decision rather than completed source behavior.

## What This Changes

The issue asks ClawHub to stop flagging the `buff-roundup` skill as suspicious by recognizing credentials and permissions declared in top-level SKILL.md frontmatter.

## Best Possible Solution

Maintainers should decide whether top-level `required_credentials` and `permissions` are supported ClawHub aliases. If yes, implement them consistently in the shared schema, SKILL.md parser, static declaration collector, LLM prompt context, UI/API/docs/tests, and a rescan/backfill path; if not, ask the publisher to republish with the documented `metadata.openclaw.requires.env`, `primaryEnv`, or `envVars` shape and then make an explicit moderation decision for the skill.
