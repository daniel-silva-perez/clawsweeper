## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that current main definitely implements. Substantively, current main still does not accept top-level `required_credentials` as registry/runtime metadata or as a static moderation declaration source, and the live `buff-roundup` listing remains flagged as suspicious with API `metadata: null` while its SKILL.md uses `required_credentials`. The prior ClawSweeper comment reached the same conclusion, and a fresh pass over current main did not find an implemented alias or shipped cleanup path for this report.

## Best Possible Solution

Keep this open for maintainer/security moderation review. The concrete path is to decide whether top-level `required_credentials` should become a supported alias for credential declarations, or ask the publisher to republish using the documented `metadata.openclaw.requires.env`, `primaryEnv`, and/or `envVars` shape; after that, rescan `buff-roundup` or apply the existing staff clean override if the skill is verified as a false positive.
