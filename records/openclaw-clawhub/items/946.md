## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that current main certainly implements. Substantively, current main still does not parse top-level `required_credentials` into registry metadata or static moderation declarations, and the live `buff-roundup` API response still reports `metadata: null` with `moderation.isSuspicious: true`.

## Best Possible Solution

Keep this open for maintainer/security moderation review. The concrete path is to decide whether top-level `required_credentials` should become a supported alias for credential declarations and, if so, wire it through schema, parser, static moderation, LLM prompt context, docs, tests, and a rescan path. If maintainers do not want that alias, ask the publisher to republish with the documented `metadata.openclaw.requires.env`, `primaryEnv`, and/or `envVars` shape, then rescan or apply an appropriate manual moderation decision for `buff-roundup`.
