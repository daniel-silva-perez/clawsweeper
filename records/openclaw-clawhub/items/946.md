## Summary

Keep open. This is an open ClawHub issue, and the supplied repository policy only allows auto-closing pull requests that current main definitely implements. Current main also still does not parse the reporter's top-level `required_credentials` or `permissions` frontmatter into registry or scanner metadata, so the appeal remains an unresolved schema and moderation policy question.

## What This Changes

The reporter asks ClawHub to clear the `buff-roundup` suspicious flag and treat SKILL.md top-level `required_credentials` and `permissions` declarations as scanner/registry metadata.

## Best Possible Solution

Maintainers should decide whether top-level `required_credentials` and `permissions` are supported ClawHub manifest aliases. If yes, implement them consistently across the shared schema, SKILL.md parser, static declaration collector, security prompt context, UI/API/docs/tests, and a rescan/backfill path; if not, ask the publisher to republish with the documented `metadata.openclaw.requires.env`, `primaryEnv`, or `envVars` shape and then make an explicit moderation decision for `buff-roundup`.
