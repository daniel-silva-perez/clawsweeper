## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that current main certainly implements. The substantive report also remains unresolved: current main parses and documents `metadata.openclaw.requires.env`, `primaryEnv`, `envVars`, `env`, and top-level `requires.env`, but not the reporter's top-level `required_credentials` shape.

## What This Changes

The issue asks ClawHub's scanner and registry metadata to treat top-level `required_credentials` in SKILL.md frontmatter as declared sensitive credentials so the `buff-roundup` suspicious flag can be reconsidered.

## Best Possible Solution

Keep this open for maintainer/security moderation review. Maintainers should decide whether top-level `required_credentials` is a supported ClawHub alias. If yes, implement it consistently through the shared schema, SKILL.md parser, static moderation declaration collector, moderation prompt context, docs, tests, and a rescan/backfill path. If no, ask the publisher to republish with the documented `metadata.openclaw.requires.env`, `primaryEnv`, and/or `envVars` shape, then rescan or make an explicit manual moderation decision for `buff-roundup`.
