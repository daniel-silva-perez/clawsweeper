## Summary

Keep open. Current source and the modern `clawhub` npm package declare `undici`, but the reported legacy `clawdhub@latest` path still resolves to the broken 0.3.0 package without that dependency, so the user-visible failure remains a maintainer-owned distribution issue.

## What This Changes

The issue asks for the legacy `clawdhub` package to include `undici` so Node.js installs via `npx`, npm, or pnpm do not fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

A maintainer should make the legacy package path unambiguous by publishing or promoting a fixed stable `clawdhub`, deprecating or migrating the broken legacy package toward `clawhub`, or otherwise ensuring legacy install commands no longer resolve to a package missing `undici`; then close this and related reports after a clean Node smoke test.
