## Summary

Keep open. Current source and the modern `clawhub` package include `undici`, but the reported legacy `clawdhub@latest` path still resolves to a published package missing that dependency, so the user-visible failure remains a distribution issue requiring maintainer action.

## What This Changes

The issue reports that `clawdhub@0.3.0` and `clawdhub@latest` crash on Node.js because the published runtime imports `undici` without declaring it as a dependency.

## Best Possible Solution

A maintainer should make the legacy package path unambiguous by publishing or promoting a fixed stable `clawdhub`, deprecating or migrating the broken legacy package toward `clawhub`, or otherwise ensuring legacy install commands no longer resolve to a package missing `undici`; then close this and related reports after a clean Node smoke test.
