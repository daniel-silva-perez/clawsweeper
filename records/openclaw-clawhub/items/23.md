## Summary

Keep this issue open. Current main and the modern `clawhub` npm package declare `undici`, but npm registry metadata still shows the legacy `clawdhub@latest` path resolving to `0.3.0` without `undici`, so the reported Node.js failure remains reachable. ClawHub review policy also keeps issues open unless they are explicit PR implemented-on-main cases.

## Best Possible Solution

Keep this open for maintainer npm-distribution handling. A maintainer should decide the legacy `clawdhub` package policy, then publish or retag a fixed legacy package, deprecate/document the broken legacy package as unsupported with migration guidance to `clawhub`, or otherwise ensure affected install paths no longer resolve to `clawdhub@0.3.0` without `undici`; then close this together with #33 and #51 as appropriate.
