## Summary

Keep this issue open. Current main and the modern `clawhub` npm package declare `undici`, but the reported legacy `clawdhub@latest` install path still resolves to `0.3.0` without `undici`, and the repository policy for this review keeps ClawHub issues open unless maintainer action resolves them.

## What This Changes

The issue reports that fresh Node installs of legacy `clawdhub@0.3.0` fail because the runtime imports `undici` without declaring it as a dependency, and asks for the dependency to be added and released.

## Best Possible Solution

A maintainer should decide and apply the legacy package distribution path: publish or retag a fixed stable `clawdhub`, deprecate/migrate `clawdhub@0.3.0` toward `clawhub`, or otherwise ensure legacy install commands no longer resolve to a package missing `undici`, then close the related reports together after a clean Node smoke test.
