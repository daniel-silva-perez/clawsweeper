## Summary

Keep this issue open. Current main and the modern `clawhub` npm package declare `undici`, but the reported failure is still reachable through the legacy `clawdhub@latest` package: current npm registry metadata still points `clawdhub@latest` to `0.3.0`, whose dependencies omit `undici`. ClawHub review policy also keeps issues open unless they are explicit PR implemented-on-main cases.

## Best Possible Solution

A maintainer should make an explicit legacy-package distribution decision: publish or retag a fixed stable `clawdhub` package, deprecate/document `clawdhub@0.3.0` with migration guidance to `clawhub`, or otherwise ensure affected install paths no longer resolve to a package missing `undici`; then close this together with #33 and #51 if a clean Node install smoke test passes.
