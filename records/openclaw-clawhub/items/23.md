## Summary

Keep this issue open. Current main and the current `clawhub` npm package include the `undici` runtime dependency, but the originally reported legacy install path still resolves to `clawdhub@0.3.0` on npm, whose latest metadata does not declare `undici`. ClawHub cleanup policy also says issues should stay open for maintainer handling rather than auto-close.

## Best Possible Solution

Keep this open for maintainer npm-distribution handling. A maintainer should decide the legacy `clawdhub` package policy, then publish or retag a fixed legacy package, deprecate/document the broken legacy package as unsupported, or otherwise make sure affected install paths no longer resolve to `clawdhub@0.3.0` without `undici`; then close this together with #33 and #51.
