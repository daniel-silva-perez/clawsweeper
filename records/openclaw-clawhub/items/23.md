## Summary

Keep this issue open. Current main and the modern `clawhub` npm package include `undici`, but the reported legacy install path is still unresolved because `clawdhub@latest` still points to `0.3.0`, whose registry dependencies omit `undici`. ClawHub cleanup policy also keeps issues open unless they are explicit PR implemented-on-main cases.

## What This Changes

The issue requests fixing the CLI package distribution so Node.js installs of the legacy `clawdhub` command do not fail with `ERR_MODULE_NOT_FOUND` for `undici`.

## Best Possible Solution

A maintainer should make an explicit legacy-package distribution decision: publish or retag a fixed stable `clawdhub` package, deprecate `clawdhub@0.3.0` with migration guidance to `clawhub`, or otherwise ensure affected install paths no longer resolve to a package missing `undici`; then run a clean Node install smoke test and close this together with #33 and #51 if the legacy path is resolved.
