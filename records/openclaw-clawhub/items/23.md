## Summary

Keep open. Current source and the modern `clawhub` npm package declare `undici`, but the reported legacy `clawdhub@latest` install path still resolves to `0.3.0` without that dependency, so the user-visible failure is not resolved for the exact package path in the report. ClawHub repository policy also keeps issues open for maintainer handling rather than auto-closing source-fixed issues.

## What This Changes

The issue requests that the CLI package declare `undici` so Node.js installs do not fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

A maintainer should make the legacy package path unambiguous: publish or promote a fixed stable `clawdhub`, deprecate or migrate the broken legacy package toward `clawhub`, or otherwise ensure legacy install commands no longer resolve to a package missing `undici`; then close this and related reports after a clean Node smoke test.
