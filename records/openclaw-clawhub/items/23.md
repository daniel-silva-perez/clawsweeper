## Summary

Keep open. Current `main` and the modern `clawhub` package declare `undici`, but the reported legacy `clawdhub@latest` path still resolves to `0.3.0` without that dependency, and ClawHub review policy keeps issues open for maintainer handling rather than auto-closing them as implemented.

## What This Changes

The issue reports that the legacy `clawdhub` npm package crashes under Node because runtime code imports `undici` without declaring it as a dependency.

## Best Possible Solution

A maintainer should make the legacy package distribution path unambiguous: publish or promote a fixed stable `clawdhub`, deprecate/migrate the broken legacy package toward `clawhub`, or otherwise ensure legacy install commands no longer resolve to a package missing `undici`, then close the related reports after a clean Node smoke test.
