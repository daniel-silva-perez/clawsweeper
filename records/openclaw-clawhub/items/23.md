## Summary

Keep open. Current source and the modern `clawhub` npm package declare `undici`, but the reported legacy `clawdhub@latest` path still resolves to `0.3.0` without that dependency; for this repository, issues should stay open for maintainer handling unless the remaining user-visible problem is resolved.

## What This Changes

The issue asks for the legacy `clawdhub` npm package to include `undici` so Node.js npx/npm installs do not fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

A maintainer should make the legacy package path unambiguous: publish or promote a fixed stable `clawdhub`, deprecate or migrate the broken legacy package toward `clawhub`, or otherwise ensure legacy install commands no longer resolve to a package missing `undici`; then close this and related reports after a clean Node smoke test.
