## Summary

Keep this issue open. The external `apple-notes-native` skill is already published and current ClawHub main has the registry, metadata, install, API, and security-review surfaces needed for community distribution. The remaining request is a maintainer product/security decision about whether OpenClaw should recommend or replace an integrated memo-based Apple Notes path, and ClawHub cleanup policy keeps issues open rather than auto-closing them.

## Best Possible Solution

Keep this issue open for maintainer product/security review. The published `apple-notes-native` skill can remain the community path on ClawHub; maintainers should separately decide whether to document it as a recommended alternative or open/land an OpenClaw core/default-skill change after evaluating reliability, permissions, setup friction, and support burden.
