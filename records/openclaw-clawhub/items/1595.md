## Summary

Keep open. The linked `apple-notes-native` skill is already published externally and current ClawHub main has the registry, metadata, public API, and install surfaces needed for a macOS osascript-based skill. The unresolved request is maintainer product/security judgment about whether that community skill should replace or be recommended over an integrated memo-based Apple Notes path, and ClawHub cleanup policy keeps issues open unless they are PRs already implemented on main.

## Best Possible Solution

Keep this issue open for maintainer product/security review. The published `apple-notes-native` ClawHub skill can remain the community-distributed path now; maintainers can separately decide whether to document it as a recommended alternative or open/track an OpenClaw core change to replace the memo-based Apple Notes default after evaluating reliability, setup friction, and macOS Automation permissions.
