## Summary

Keep open. This is an open ClawHub issue, and this review lane only auto-closes PRs that current main definitely implements. Substantively, current main still flags `child_process` plus spawn/exec patterns as `suspicious.dangerous_exec`, does not recognize `HOOK.md` `spawns_process` metadata or `openclaw:allow` comments, and preserves non-allowlisted static findings even when external scanners are clean.

## What This Changes

The issue asks ClawHub to stop flagging intentional, fully declared hook `child_process.spawn` usage as suspicious, either by documenting the correct suppression path or by recognizing `HOOK.md` `spawns_process: true` metadata.

## Best Possible Solution

Keep this issue open for maintainer/security triage. The safest path is either to document that intentional `child_process` usage remains a surfaced static-analysis finding handled by owner rescan/manual moderation, or to design a narrow allowlist that validates trusted hook metadata, exact executable/script paths, literal argv/no shell interpretation, constrained environment inheritance, and scanner/API/UI/docs behavior together.
