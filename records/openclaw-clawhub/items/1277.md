## Summary

Keep open. ClawHub cleanup policy keeps open issues open, and the substantive request is still unresolved: current main still emits `suspicious.dangerous_exec` for `child_process` execution and has no documented or implemented `HOOK.md`, `spawns_process`, or inline allow mechanism for intentional hook process spawns.

## What This Changes

The issue asks for documented or implemented suppression of intentional `child_process.spawn` usage in hook files, especially via declared `HOOK.md` metadata such as `spawns_process: true`.

## Best Possible Solution

Define an explicit security policy for declared process spawns: either document `suspicious.dangerous_exec` as an always-surfaced manual-review finding, or implement a narrow trusted allowlist with validation across hook metadata, exact executable/script paths, argv construction, environment isolation, owner rescan behavior, API/UI presentation, docs, and regression tests.
