## Summary

Keep open. This is an open ClawHub issue, and this repository cleanup policy only permits auto-closing pull requests that current main definitely implements. The substantive request is also still live: current main still flags `child_process` execution as `suspicious.dangerous_exec`, has no recognized `HOOK.md`/`spawns_process`/inline allow mechanism, and only clears a different static-scan code when external scanners are clean.

## What This Changes

The issue requests a documented or implemented suppression path so declared, intentional `child_process.spawn` usage in hook files is not surfaced as an unresolved security-scanner finding.

## Best Possible Solution

Define an explicit scanner policy for declared process spawns: either document `suspicious.dangerous_exec` as an always-surfaced manual-review finding, or implement a narrow trusted allowlist that validates hook metadata, exact executable/script paths, literal argv/no shell interpretation, constrained environment inheritance, owner rescan behavior, API/UI presentation, docs, and regression tests.
