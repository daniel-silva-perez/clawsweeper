## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Substantively, current main still emits `suspicious.dangerous_exec` for `child_process` spawn/exec usage, does not recognize `HOOK.md`, `spawns_process`, or `openclaw:allow` as scanner suppressions, and the live `claw-diplomat` listing still reports the exact flagged hook file as suspicious.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is either to document that intentional process spawns remain review findings handled through staff manual override, or to implement a narrow scanner allowlist that validates an exact executable/script path, literal arguments without shell interpretation, constrained environment inheritance, trusted hook metadata, tests in publish/moderation coverage, and user-facing security docs.
