## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Substantively, current main still treats child_process spawn/exec as suspicious, does not recognize HOOK.md, spawns_process, or openclaw:allow as scanner suppressions, and tests preserve dangerous_exec as a surfaced finding.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is either to document that intentional process spawns remain review findings handled through manual moderation, or implement a narrow allowlist that validates the exact executable/script path, literal arguments without shell interpretation, constrained environment inheritance, and trusted hook metadata, with regression tests in the moderation/package publish coverage and public docs for the supported declaration mechanism.
