## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy only permits auto-closing PRs that are certainly implemented on current main. Substantively, current main still flags child_process spawn/exec patterns as suspicious, does not recognize HOOK.md, spawns_process, or openclaw:allow as suppression signals, and the live claw-diplomat listing still reports the dangerous_exec finding.

## Best Possible Solution

Keep this issue open for explicit maintainer/security triage. The best path is either to document that intentional child_process usage remains a review finding handled through staff manual override, or to implement a narrow allowlist that validates trusted hook metadata, exact executable/script paths, literal arguments without shell interpretation, constrained environment inheritance, and coverage in publish/moderation tests plus user-facing security docs.
