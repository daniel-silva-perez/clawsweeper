## Summary

Keep open. The repository-specific cleanup policy for openclaw/clawhub only permits auto-closing pull requests that are definitely implemented on current main, and this item is an issue. Substantively, current main still treats `child_process` spawn/exec usage as `suspicious.dangerous_exec`; I did not find scanner support for `HOOK.md`, `spawns_process`, or `openclaw:allow` suppressions.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is either to document that intentional process spawns remain staff-reviewed findings, or implement a narrow scanner allowlist that verifies exact executable/script paths, literal arguments without shell interpretation, constrained environment inheritance, and trusted hook metadata, with coverage in moderation/package-publish tests and user-facing docs.
