## Summary

Keep open. ClawHub cleanup policy keeps open issues open, and the substantive request is still unresolved on current main: `child_process` spawn usage still becomes `suspicious.dangerous_exec`, while the requested inline and `HOOK.md` suppression signals are not implemented or documented.

## What This Changes

The issue asks ClawHub to document or support a safe suppression path for intentional `child_process` spawns declared in hook metadata.

## Best Possible Solution

Define an explicit scanner policy for declared process spawns: either document `suspicious.dangerous_exec` as an intentional manual-review finding, or implement a narrow trusted allowlist with validation, UI/API behavior, owner rescan behavior, docs, and regression tests.
