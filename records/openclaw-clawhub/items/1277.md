## Summary

Keep open. This is an open ClawHub issue, and the repository policy for this review lane only permits auto-closing ClawHub pull requests that current main certainly implements. Current main still treats child_process spawn/exec usage as a static suspicious finding, does not implement HOOK.md/spawns_process/openclaw:allow suppression, and preserves non-allowlisted static findings even when external scanners are clean.

## Best Possible Solution

Keep this issue open for explicit maintainer/security triage. The best path is either to document that intentional child_process usage remains a review finding handled through staff/manual override and rescan workflows, or design a narrow allowlist that validates trusted hook metadata, exact executable/script paths, literal argv without shell interpretation, constrained environment inheritance, and updates tests/docs/API/UI behavior around the resulting scanner status.
