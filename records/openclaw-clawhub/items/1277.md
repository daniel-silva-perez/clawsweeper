## Summary

Keep open. This is an open ClawHub issue, and this review lane only auto-closes ClawHub pull requests that current main certainly implements. Substantively, current main still treats `child_process` plus spawn/exec usage as `suspicious.dangerous_exec`, does not recognize `HOOK.md` `spawns_process` or `openclaw:allow` suppressions, and only suppresses a narrow allowlisted static code when external scanners are clean.

## Best Possible Solution

Keep this issue open for explicit maintainer/security triage. The best path is either to document that intentional `child_process` usage remains a surfaced review finding handled through owner rescan/manual moderation workflows, or to design a narrow allowlist that validates trusted hook metadata, exact executable/script paths, literal argv without shell interpretation, constrained environment inheritance, and updates scanner tests, docs, API/UI behavior, and rescan guidance together.
