## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Substantively, current main still flags child_process spawn/exec usage as suspicious, does not document or implement the requested HOOK.md/spawns_process/openclaw:allow suppression path, and only allows a separate staff manual-override path for reviewed findings.

## Best Possible Solution

Keep this issue open for explicit maintainer/security triage. The best path is either to document that intentional child_process usage remains a review finding handled through staff manual override, or to design a narrow allowlist that validates trusted hook metadata, exact executable/script paths, literal argv without shell interpretation, constrained environment inheritance, and publish/moderation tests plus user-facing security docs.
