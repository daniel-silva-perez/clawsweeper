## Summary

Keep this issue open. Current main and the maintained `clawhub` npm package include `undici`, but the exact reported legacy `clawdhub@latest` install path still resolves to `clawdhub@0.3.0`, whose published dependency metadata omits `undici`.

## What This Changes

The issue requests that the legacy `clawdhub` CLI package include `undici` as a runtime dependency so `pnpm dlx`/`npx` installs do not crash with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

Keep this issue open until a maintainer with npm ownership either publishes or promotes a fixed stable `clawdhub` release whose dependency metadata includes `undici`, or intentionally deprecates/redirects the legacy `clawdhub` package with clear guidance to use `clawhub`. After that, consolidate #23 and #51.
