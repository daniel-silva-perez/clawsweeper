## Summary

This issue should stay open. Current main has the dependency in the rebranded `clawhub` CLI, but the reported legacy package path is still a release/distribution problem: `clawdhub@latest` still resolves to `0.3.0`, whose published dependency metadata omits `undici`. Related reports #23 and #51 remain open for the same failure, and closed PR #57 documented that the remaining action was publishing/promoting a fixed stable legacy release rather than a normal source-code patch.

## Best Possible Solution

Keep this issue open until a maintainer resolves the legacy package path: publish/promote a fixed stable `clawdhub` release that includes `undici`, or intentionally deprecate/redirect `clawdhub` with clear guidance to use the maintained `clawhub` package. After that, consolidate or close the same-cluster reports #23 and #51.
