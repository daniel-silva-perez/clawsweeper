## Summary

Keep open. Current main and the maintained `clawhub` package include `undici`, but the reported `clawdhub@latest` install path still resolves to legacy `clawdhub@0.3.0` without that dependency, so the user-visible failure remains and needs maintainer npm/package handling.

## What This Changes

The issue requests adding `undici` to the CLI package dependencies so `pnpm dlx` or `npx clawdhub@latest` no longer fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

End with one stable user path: either publish or promote a fixed stable `clawdhub` release whose metadata includes `undici`, or intentionally deprecate/redirect the legacy package with clear guidance to use `clawhub`, then consolidate the duplicate reports.
