## Summary

Keep open. Current main and the maintained `clawhub` npm package include `undici`, but the exact reported `clawdhub@latest` path still resolves to `0.3.0` without that runtime dependency, so the user-visible failure remains for legacy installs and needs maintainer npm/package handling.

## What This Changes

The issue requests that the published `clawdhub` CLI dependency metadata include `undici` so `pnpm dlx` and `npx` installs do not crash with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

End with one stable user path: either publish or promote a fixed stable `clawdhub` release whose dependency metadata includes `undici`, or intentionally deprecate/redirect the legacy package with clear guidance to use `clawhub`, then consolidate the duplicate reports.
