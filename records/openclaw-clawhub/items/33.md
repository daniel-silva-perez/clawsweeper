## Summary

Keep open. Current main and the rebranded npm package `clawhub@latest` include `undici`, but the exact failing legacy command path still appears affected: npm registry metadata shows `clawdhub@latest` is still `0.3.0` without `undici`, while only `clawdhub@0.3.1-beta.1` has the dependency. Related reports #23 and #51 describe the same remaining legacy-package failure, and #57 was closed unmerged after documenting that the remaining step was publishing/promoting a fixed stable release.

## Best Possible Solution

Keep this open until a maintainer resolves the legacy package path: either publish/promote a fixed stable `clawdhub` release that includes `undici`, or intentionally deprecate/redirect `clawdhub` with clear guidance to use `clawhub`. After that, consolidate the same-cluster reports #23 and #51.
