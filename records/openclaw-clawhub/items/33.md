## Summary

Keep open. Current `main` and the maintained `clawhub` npm package include `undici`, but the exact reported `clawdhub@latest` path still resolves to `0.3.0` without that dependency, so the user-visible failure remains unresolved for legacy installs. Under the repository policy for ClawHub issues, this should stay open for maintainer handling rather than be auto-closed.

## What This Changes

The issue asks for the CLI package dependency metadata to include `undici` so `pnpm dlx clawdhub@latest ...` and `npx clawdhub@latest ...` do not crash with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

End with one stable user path: either publish or promote a fixed stable `clawdhub` release whose dependency metadata includes `undici`, or intentionally deprecate/redirect the legacy package with clear guidance to use `clawhub`, then consolidate the duplicate reports.
