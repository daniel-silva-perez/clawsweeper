## Summary

Keep open. Current main and the maintained `clawhub` npm package have the runtime dependency, but the exact reported `clawdhub@latest` package still resolves to `0.3.0` without `undici`, so the user-visible failure is not resolved for that install path.

## What This Changes

The issue asks for `undici` to be declared as a runtime dependency so `pnpm dlx` and `npx` installs of `clawdhub@latest` do not crash with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

End with one stable user path: either publish or promote a fixed stable `clawdhub` release whose dependency metadata includes `undici`, or intentionally deprecate/redirect the legacy package with clear guidance to use `clawhub`, then consolidate #23 and #51.
