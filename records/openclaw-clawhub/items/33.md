## Summary

Keep open. Current main and the maintained `clawhub` package include `undici`, but the exact reported `clawdhub@latest` path still resolves to the legacy `clawdhub@0.3.0` package without that dependency, so the user-visible failure remains unresolved and needs maintainer npm/package handling rather than auto-close.

## What This Changes

The issue asks for the CLI package metadata used by `pnpm dlx clawdhub@latest` and `npx clawdhub@latest` to include the runtime `undici` dependency so installs do not crash with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

End with one stable user path: either publish or promote a fixed stable `clawdhub` release whose dependency metadata includes `undici`, or intentionally deprecate/redirect the legacy package with clear guidance to use `clawhub`, then consolidate the duplicate reports.
