## Summary

Keep open. Current main and the maintained `clawhub` npm package include `undici`, but the exact reported `clawdhub@latest` path still resolves to legacy `clawdhub@0.3.0` whose published dependency metadata omits `undici`; repository policy also says ClawHub issues should stay open unless a maintainer handles them.

## What This Changes

Add `undici` as a runtime dependency for the legacy `clawdhub` CLI package so `pnpm dlx` and `npx` installs do not crash with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

Keep this issue open until a maintainer with npm ownership either publishes or promotes a fixed stable `clawdhub` release whose dependency metadata includes `undici`, or intentionally deprecates/redirects the legacy `clawdhub` package with clear guidance to use `clawhub`. After that, consolidate #23 and #51.
