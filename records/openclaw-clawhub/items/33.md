## Summary

Keep open. Current main and the maintained `clawhub` npm package include `undici`, but the exact reported `clawdhub@latest` path still resolves to `0.3.0` without that dependency; the remaining fix is npm package or dist-tag administration rather than a normal source-only patch. citeturn0view0

## What This Changes

The issue asks for the legacy `clawdhub` CLI package to declare `undici` so `pnpm dlx` and `npx clawdhub@latest install ...` do not crash with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

End with one stable user path: either publish or promote a fixed stable `clawdhub` release whose dependency metadata includes `undici`, or intentionally deprecate/redirect the legacy package with clear guidance to use `clawhub`, then consolidate the duplicate reports.
