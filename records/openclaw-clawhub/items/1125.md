## Summary

Keep this PR open. Current main still does not enforce metadata.os during clawhub install, and the PR is not currently safe to merge because it checks raw Node platform values, validates latest-version metadata before requested-version resolution, and adds metadata to the CLI schema as required rather than optional.

## Best Possible Solution

Keep the PR open and revise it rather than closing it as obsolete. A mergeable version should parse metadata as optional and nullable, normalize host platform aliases to the documented metadata vocabulary, check OS requirements for the resolved version when --version is used, make an explicit maintainer-approved decision about whether --force bypasses OS mismatch, and add focused cmdInstall tests for allowed, rejected, forced, missing-metadata, alias, and requested-version cases.

## Risks / Open Questions

- Closing this PR would discard a still-missing install-time OS compatibility feature; current main has no equivalent guard.
- The PR as provided compares raw node:os platform values against documented metadata values, so macOS and Windows skills can be rejected incorrectly unless aliases are normalized.
- The proposed check uses latest-version metadata before resolving --version, so it can allow or reject the wrong release when OS requirements differ by version.
- Adding metadata as a required CLI response field can break API responses that omit the key instead of returning null.
- Changing --force to bypass OS mismatch is install policy, not only plumbing, and should be explicitly accepted or removed before merge.
- Security pass found no new CI/dependency/secret surface, but the install-gating change still needs tests to avoid misleading compatibility bypasses.
