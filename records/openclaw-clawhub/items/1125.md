## Summary

Keep this PR open. Current main still does not implement the intended `clawhub install` OS compatibility check, and the PR has unresolved functional review concerns around schema optionality, OS vocabulary normalization, and version-specific metadata.

## Best Possible Solution

Keep the PR open and revise it. The maintainer path is to add optional CLI parsing for API `metadata`, normalize host platform aliases to the documented metadata vocabulary, decide whether `--force` should bypass OS mismatch separately from suspicious-skill handling, validate OS requirements against the resolved version when `--version` is used, and add focused `cmdInstall` tests for allowed, rejected, forced, missing-metadata, and version-specific metadata cases.

## Risks / Open Questions

- Merging the PR as-is may reject supported macOS/Windows installs if raw Node platform names are compared to documented metadata values like `macos` and `windows`.
- Making `metadata` required in the CLI response schema could break installs for API responses or older fixtures that omit the field.
- Checking only latest-version metadata can allow or reject the wrong install when `--version` targets an older release with different OS requirements.
