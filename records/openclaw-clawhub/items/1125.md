Keep this PR open. Current main still does not enforce `metadata.os` during `clawhub install`, and the PR remains useful but needs revision before merge: optional metadata parsing, OS alias normalization, requested-version metadata handling, tests, and maintainer policy on `--force`.

Keep this PR open and revise it. ## Risks / Open Questions

- Merging the PR as-is would compare Node platform values such as `darwin`/`win32` against documented metadata values such as `macos`/`windows`, incorrectly blocking supported macOS and Windows installs.
- The proposed check runs before requested-version resolution and would use latest-version metadata for `--version` installs unless the API/schema are extended to expose metadata for the resolved version.
- The PR schema makes `metadata` required rather than optional/nullable, which is brittle for older or compatible registries that omit the field even when no OS restriction exists.
- Changing `--force` to bypass OS incompatibility is a product/security policy decision because it turns a compatibility guard into an overrideable warning path.
