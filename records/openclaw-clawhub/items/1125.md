## Summary

Keep this PR open. Current main still lacks the requested install-time OS compatibility check, and the proposed branch needs revision before merge because it would compare Node platform names directly against the documented metadata vocabulary, treats metadata as required in the CLI schema, and validates latest-version metadata before resolving a requested version.

## What This Changes

The PR adds CLI response parsing for skill metadata, checks `metadata.os` during `clawhub install`, and expands the `--force` help text to say it bypasses OS mismatch.

## Best Possible Solution

Implement a tested install-time compatibility guard that reads optional/nullable platform metadata for the resolved version, normalizes documented OS aliases to host platform values, and makes the `--force` behavior explicit in CLI semantics and help text.

## Risks / Open Questions

- Merging the PR as-is could falsely block documented macOS and Windows skills because Node reports `darwin` and `win32`, while skill metadata examples use `macos` and `windows`.
- The PR schema makes `metadata` required instead of optional/nullable, which can reject skill detail responses that omit the key.
- The proposed check runs before requested-version resolution and would validate latest-version metadata rather than the version installed with `--version`.
- The `--force` bypass for OS incompatibility changes install-safety semantics and should get explicit maintainer approval.
- No focused install tests currently cover compatible hosts, incompatible hosts, OS aliases, requested-version metadata, or force behavior.
