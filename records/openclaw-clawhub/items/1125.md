## Summary

Keep this PR open. Current main still does not enforce `metadata.os` during `clawhub install`, and the proposed PR is not merge-ready because it checks raw Node platform values, checks latest-version metadata before requested-version resolution, and adds CLI response metadata as required rather than optional.

## Best Possible Solution

Keep this PR open and revise it rather than closing it as obsolete. A mergeable version should parse `metadata` as optional and nullable, normalize host platform aliases to the documented metadata vocabulary, check OS requirements for the resolved version when `--version` is used, make a maintainer-approved decision about whether `--force` bypasses OS mismatch, and add focused `cmdInstall` tests for allowed, rejected, forced, missing-metadata, alias, and requested-version cases.
