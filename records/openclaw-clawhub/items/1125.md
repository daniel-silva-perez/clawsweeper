## Summary

Keep this PR open. Current main still does not enforce `metadata.os` during `clawhub install`, and the PR remains useful but not merge-ready because it needs optional metadata parsing, documented OS alias normalization, requested-version metadata handling, tests, and a maintainer decision on whether `--force` should bypass OS incompatibility.

## Best Possible Solution

Keep this PR open and revise it. A mergeable version should parse skill metadata as optional/nullable, normalize host platform aliases to the documented metadata vocabulary, expose or retrieve metadata for the resolved version when `--version` is used, make a maintainer-approved decision about `--force` and OS mismatch, and add focused `cmdInstall` tests for allowed, rejected, forced, missing-metadata, alias, and requested-version cases.

## Risks / Open Questions

- Merging the PR as-is would likely reject documented macOS and Windows metadata values because Node reports `darwin` and `win32` while skill metadata uses values such as `macos` and `windows`.

- Making `metadata` required in the CLI schema would risk parse failures for responses that omit the field unless it is optional and nullable.
- Changing `--force` to bypass OS mismatch is a product-policy decision, not only an implementation detail.
