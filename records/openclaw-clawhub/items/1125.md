## Summary

Keep this PR open. Current main still does not implement install-time OS compatibility checks, and the PR is not obsolete, but the supplied change is not safe to auto-close as implemented because it needs schema, version-resolution, platform-normalization, policy, and test follow-up.

## Best Possible Solution

Keep the PR open and revise it. A mergeable path should parse metadata as optional and nullable, normalize host platform aliases to the documented metadata vocabulary, validate OS metadata for the resolved version when --version is used, explicitly decide whether --force bypasses OS mismatch, and add focused cmdInstall tests for allowed, rejected, forced, missing-metadata, and requested-version cases.

## Risks / Open Questions

- Current main lacks the requested install-time OS compatibility behavior, so closing would incorrectly discard still-useful work.
- The PR as supplied can incorrectly reject supported macOS or Windows installs because Node reports darwin/win32 while ClawHub metadata is documented as macos/windows-style values.
- Checking latest-version metadata before resolving --version can allow or reject the wrong release when OS requirements differ by version.
- Making metadata required in the CLI response schema could break installs against responses that omit metadata.
- The --force bypass semantics for OS mismatch are security/product policy and need maintainer decision plus tests.
