## Summary

Keep this PR open. Current main still does not implement install-time OS compatibility checks, and the PR remains useful but needs revision before it is safe to merge.

## Best Possible Solution

Keep the PR open and revise it. A mergeable path should parse metadata as optional and nullable, normalize host platform aliases to the documented metadata vocabulary, validate OS requirements for the resolved version when --version is used, explicitly decide whether --force bypasses OS mismatch, and add focused cmdInstall tests for allowed, rejected, forced, missing-metadata, and requested-version cases.

## Risks / Open Questions

- Closing this PR would discard a still-missing install-time OS compatibility feature; current main has no equivalent guard.
- The PR as supplied compares raw Node platform() values against documented metadata values, so macOS and Windows skills can be rejected incorrectly unless aliases are normalized.
- The proposed check runs before resolving --version-specific metadata, so it can allow or reject the wrong release when OS requirements differ across versions.
- Adding metadata as a required CLI response field can break older or custom registry responses that omit the key instead of returning null.
- Changing --force to bypass OS mismatch is an install policy decision and needs explicit maintainer agreement plus tests.
