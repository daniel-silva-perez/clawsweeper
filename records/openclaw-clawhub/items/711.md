## Summary

Keep open. Current main has not implemented the PR’s intended backend optimization: lexical fallback still reads active digest windows and filters suspicious rows after the read. The PR remains useful but needs maintainer review because main has since moved this path to `skillSearchDigest`, and the branch also includes a broader fixed-200 client-side search pagination change.

## What This Changes

The PR changes lexical fallback search to use a nonsuspicious Convex index when requested, updates related tests, and changes `/skills` search pagination to fetch a fixed larger result set for client-side paging.

## Best Possible Solution

Land a narrow rebase or replacement that updates current main’s `skillSearchDigest` fallback to select the nonsuspicious updated and created indexes with `softDeletedAt = undefined` and `isSuspicious = false` when requested, preserves the default search semantics, adds index-routing tests, and handles the fixed-200 pagination idea as a separate maintainer decision.
