## Summary

Current main does not definitely implement this PR’s intended change. `lexicalFallbackSkills` now reads lightweight `skillSearchDigest` rows instead of full `skills` documents, but it still uses the active updated/created indexes and filters `nonSuspiciousOnly` in JavaScript rather than selecting the non-suspicious indexes at the database level. Under the ClawHub PR policy, this should remain open for maintainer review or rebase rather than be auto-closed.

## Best Possible Solution

Keep this PR open. The best path is to rebase or rewrite it against current main so `lexicalFallbackSkills` uses `skillSearchDigest` non-suspicious indexes for both updated and created scan windows when `nonSuspiciousOnly` is set, while preserving the current digest-based fallback, highlighted/capability filtering, exact slug handling, and search pagination semantics.
