## Summary

Current main still does not implement this PR's central optimization. `lexicalFallbackSkills` now scans `skillSearchDigest`, but it still uses the active updated/created indexes and filters suspicious rows in JavaScript when `nonSuspiciousOnly` is set. The PR should stay open for a focused rebase or rewrite against the digest-based current implementation.

## Best Possible Solution

Keep this PR open and rebase or rewrite it narrowly against current main. The backend change should make `lexicalFallbackSkills` choose `skillSearchDigest` nonsuspicious updated/created indexes when `nonSuspiciousOnly` is set, while preserving the current digest hydration, exact slug behavior, capability and highlighted filters, and updated/created scan windows. Add tests that assert nonsuspicious digest index selection and that suspicious rows still pass through when the flag is unset. Treat the client-side fixed-200 pagination change as a separate maintainer decision unless it is explicitly desired.
