## Summary

Current main still does not implement this PR's central optimization. `lexicalFallbackSkills` now reads lightweight `skillSearchDigest` rows, but when `nonSuspiciousOnly` is set it still scans `by_active_updated` and `by_active_created`, then filters suspicious rows in JavaScript. The PR remains useful, but it needs a rebase or rewrite against the current digest-based fallback shape, and the separate fixed-200 client pagination change needs maintainer review.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is a narrow rebase or rewrite against current main: make `lexicalFallbackSkills` choose the `skillSearchDigest` nonsuspicious updated and created indexes with `softDeletedAt` and `isSuspicious=false` when `nonSuspiciousOnly` is set, while preserving the current digest-based behavior. Split or explicitly re-review the fixed-200 client pagination change rather than treating it as part of the backend index optimization.
