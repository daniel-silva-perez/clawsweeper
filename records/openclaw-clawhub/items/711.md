## Summary

Keep open. Current main still does not implement the PR’s intended nonsuspicious-index routing for lexical fallback search, and the open PR also contains a broader pagination change that can truncate broad search results, so this needs maintainer review rather than cleanup closure.

## What This Changes

The PR updates lexical fallback search to route nonsuspicious scans through indexed filtering, adds related tests, and changes `/skills` search pagination to fetch a fixed 200 results for client-side paging.

## Best Possible Solution

Land a narrow update against current main’s `skillSearchDigest` fallback that selects `by_nonsuspicious_updated` and `by_nonsuspicious_created` with the active+nonsuspicious prefix when requested, preserves default search semantics, and treats the fixed-200 pagination idea as a separate product/performance decision.
