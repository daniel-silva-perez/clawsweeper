## Summary

Current main does not implement this PR's central optimization. `lexicalFallbackSkills` now uses `skillSearchDigest`, but it still reads the active updated and created digest windows and filters suspicious rows in JavaScript when `nonSuspiciousOnly` is set. The PR remains useful, but it needs a rebase or rewrite against the current digest-based fallback path, plus maintainer review of the separate fixed-200 search pagination change.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is a narrow rebase or rewrite against current main: make `lexicalFallbackSkills` choose `skillSearchDigest` `by_nonsuspicious_updated` and `by_nonsuspicious_created` with `softDeletedAt=undefined` and `isSuspicious=false` when `nonSuspiciousOnly` is set, update the lexical fallback tests to assert that routing, and split or explicitly re-review the fixed-200 client pagination behavior.
