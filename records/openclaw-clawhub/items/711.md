## Summary

Keep this PR open. Current main has moved lexical fallback to skillSearchDigest rows, but it still reads the active updated/created indexes and filters nonSuspiciousOnly after the read, so the PR’s intended database-level non-suspicious index optimization is not implemented on main.

## Best Possible Solution

Keep this PR open and rebase or rewrite it against current main. The focused path is to make lexicalFallbackSkills choose skillSearchDigest by_nonsuspicious_updated and by_nonsuspicious_created when nonSuspiciousOnly is set, preserve digest hydration, exact slug behavior, capability/highlight filtering, and current pagination semantics unless maintainers intentionally approve the broader client-side pagination change. Add tests that assert the non-suspicious digest indexes are selected and that the default path still permits suspicious rows when the flag is unset.
