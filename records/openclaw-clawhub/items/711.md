## Summary

This PR should stay open for maintainer review. Current main still does not implement the requested nonsuspicious-index routing in lexical fallback, and the branch needs a rebase to the current digest-based search path plus an explicit decision on the broader fixed-200 client pagination change.

## What This Changes

The PR changes lexical fallback search to use a nonsuspicious index and streamed scan, updates search tests, and switches `/skills` search load-more to fetch a fixed 200 results and paginate client-side.

## Best Possible Solution

Land a narrow update against current main’s `skillSearchDigest` fallback that selects `by_nonsuspicious_updated` and `by_nonsuspicious_created` with the active+nonsuspicious prefix when requested, preserves existing default search semantics, and handles the client-side fixed-200 pagination idea as a separate explicit review decision.
