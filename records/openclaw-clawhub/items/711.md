## Summary

Keep open. Current main does not implement the PR’s core backend optimization: lexical fallback search still scans active digest indexes and filters suspicious rows after reading them. The PR remains relevant, but it needs maintainer review because main has moved to a two-window `skillSearchDigest` fallback and the branch also includes a broader fixed-200 client-side pagination change.

## What This Changes

The PR changes lexical fallback search to select a nonsuspicious index for filtered scans, updates related search tests, and changes `/skills` search pagination to fetch a fixed larger result set for client-side paging.

## Best Possible Solution

Land a narrow update against current main’s `skillSearchDigest` fallback that selects `by_nonsuspicious_updated` and `by_nonsuspicious_created` with the active+nonsuspicious prefix when requested, preserves existing default search semantics, and adds index-routing tests; handle the client-side fixed-200 pagination idea as a separate explicit review decision.
