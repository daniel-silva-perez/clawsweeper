## Summary

Keep open: current main does not implement the PR's intended nonsuspicious index routing for lexical fallback, and the proposed branch still contains a blocking pagination regression that needs maintainer review before merge.

## What This Changes

The PR changes lexical fallback search to use nonsuspicious index routing and streamed scanning, updates search tests, and switches /skills search pagination to a fixed larger fetch with client-side paging.

## Best Possible Solution

Land a narrow update against current main's skillSearchDigest fallback that routes both updated and created scans through by_nonsuspicious_* indexes with the active+nonsuspicious prefix when requested, preserves existing default search semantics, and treats the fixed-200 pagination idea as a separate explicit product/performance decision.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Remove the fixed search result cap:** `src/routes/skills/-useSkillsBrowseModel.ts:111`
  - body: Hard-coding search requests to `limit: 200` means broad searches can never load results beyond the first 200 server matches. The current load-more flow increases the server limit, so replacing it with cached client-side paging silently truncates valid results once those 200 items are exhausted.
