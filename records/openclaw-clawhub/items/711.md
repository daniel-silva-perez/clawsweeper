## Summary

Current main does not implement this PR’s core optimization. `lexicalFallbackSkills` has moved to `skillSearchDigest`, but it still scans active digest indexes and filters suspicious results after reading rows instead of selecting nonsuspicious indexes when `nonSuspiciousOnly` is set. The PR also includes broader client-side search pagination behavior that remains unmerged and has review risk, so it should stay open for maintainer follow-up or a focused rewrite.

## Best Possible Solution

Keep this PR open and rebase or rewrite it against current main. The focused path is to update `lexicalFallbackSkills` to choose the `skillSearchDigest` nonsuspicious indexes when `nonSuspiciousOnly` is set, preserve current digest hydration, exact slug handling, highlighted and capability filters, updated/created scan windows, and existing pagination semantics unless maintainers separately accept the client-side pagination change. Add tests for nonsuspicious digest index selection and for suspicious rows passing through when the flag is unset.
