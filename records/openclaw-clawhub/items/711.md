## Summary

Current main still does not implement this PR's central optimization. `lexicalFallbackSkills` now uses `skillSearchDigest`, but it still reads active updated/created digest windows and filters suspicious rows after the read when `nonSuspiciousOnly` is set. The PR should stay open for a narrow rebase or rewrite against the current digest-based implementation.

## Best Possible Solution

Keep this PR open and rebase or rewrite it narrowly against current main. The backend change should make `lexicalFallbackSkills` choose the `skillSearchDigest` nonsuspicious updated/created indexes with the `softDeletedAt` and `isSuspicious=false` equality prefix when `nonSuspiciousOnly` is set, while preserving exact slug injection, digest owner hydration, capability filtering, highlighted filtering, and the current updated/created scan windows. Add tests that assert nonsuspicious digest index routing and that suspicious rows still pass through when the flag is unset. Treat the client-side fixed-200 pagination change as a separate maintainer decision unless it is explicitly desired.
