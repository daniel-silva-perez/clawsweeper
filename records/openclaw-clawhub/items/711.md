## Summary

Keep this PR open. Current main has optimized lexical fallback to read `skillSearchDigest` rows instead of full `skills` docs, but it still uses the active updated/created digest indexes and applies `nonSuspiciousOnly` after reading, so the PR’s title-level nonsuspicious index optimization is not implemented on main.

## Best Possible Solution

Keep this PR open and rebase or rewrite it against current main. The focused maintainer path is to make `lexicalFallbackSkills` select `skillSearchDigest` `by_nonsuspicious_updated` and `by_nonsuspicious_created` when `nonSuspiciousOnly` is set, preserve the current digest hydration, exact slug behavior, highlighted/capability filtering, updated/created scan windows, and current search pagination semantics unless maintainers explicitly accept the broader client-side pagination change. Add tests that assert the nonsuspicious digest indexes are selected and that the default path still permits suspicious rows when the flag is unset.
