## Summary

Current main does not implement this PR's core optimization. `lexicalFallbackSkills` has moved to `skillSearchDigest`, but it still reads active updated/created digest windows and applies the `nonSuspiciousOnly` suspicious filter after rows are read. The PR also carries broader search pagination changes and branch-specific review history, so it should stay open for a focused rebase or rewrite against current main.

## Best Possible Solution

Keep this PR open and rebase or rewrite it narrowly against current main. The best implementation is to update `lexicalFallbackSkills` to choose `skillSearchDigest` nonsuspicious updated/created indexes when `nonSuspiciousOnly` is set, preserve exact slug handling, digest hydration, capability/highlighted filters, updated/created scan windows, and current search pagination semantics unless maintainers separately approve the client-side pagination change. Add tests that assert nonsuspicious digest index selection and that suspicious rows still pass through when the flag is unset.
