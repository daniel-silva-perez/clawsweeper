## Summary

Current main does not implement the PR’s central backend change: lexical fallback still reads active digest indexes and filters suspicious rows after the read. The PR should stay open for maintainer review because it also carries a broader fixed-200 client-side pagination change that is not just the backend index optimization.

## What This Changes

The PR changes lexical fallback search to use a nonsuspicious index when requested, adjusts fallback candidate scanning, updates search tests, and changes search pagination to fetch a fixed larger result set for client-side paging.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a narrow rebase or replacement against current main that routes lexicalFallbackSkills through skillSearchDigest by_nonsuspicious_updated and by_nonsuspicious_created with softDeletedAt=undefined and isSuspicious=false when nonSuspiciousOnly is set, preserves default suspicious passthrough, updates the lexical fallback tests, and separately decides whether the fixed-200 client pagination behavior belongs in this PR.
