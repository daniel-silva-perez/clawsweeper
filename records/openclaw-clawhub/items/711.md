## Summary

Keep this PR open. Current main still does not implement the central backend optimization: lexical fallback scans `skillSearchDigest` through active updated/created indexes and filters suspicious rows after reading them. The PR also includes a separate fixed-200 client search pagination behavior that remains a maintainer/product review decision.

## What This Changes

The PR changes lexical fallback search to use nonsuspicious indexes when requested, adjusts fallback candidate scanning, updates search tests, and changes frontend search pagination to fetch a fixed larger result set and paginate client-side.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is a narrow rebase or replacement against current main: make `lexicalFallbackSkills` choose `skillSearchDigest` `by_nonsuspicious_updated` and `by_nonsuspicious_created` with `softDeletedAt=undefined` and `isSuspicious=false` when `nonSuspiciousOnly` is set, update the lexical fallback tests to assert that routing and default suspicious passthrough, and split or explicitly re-review the fixed-200 client pagination behavior.
