## Summary

Keep this PR open. Current main has improved lexical fallback by moving it to `skillSearchDigest`, but it has not implemented the PR’s central change: selecting nonsuspicious digest indexes for `lexicalFallbackSkills` when `nonSuspiciousOnly` is set. The PR is also stale against newer fallback behavior and includes a broader client-side search pagination change that is not on main.

## Best Possible Solution

Keep this PR open and rebase or rewrite it against current main. The focused maintainer path is to make `lexicalFallbackSkills` choose `skillSearchDigest` `by_nonsuspicious_updated` and `by_nonsuspicious_created` when `nonSuspiciousOnly` is set, preserve current digest hydration, exact slug behavior, highlighted/capability filtering, updated/created scan windows, and existing search pagination semantics unless maintainers explicitly accept the broader client-side pagination change. Add tests that assert the nonsuspicious digest indexes are selected and that suspicious rows still pass through when the flag is unset.
