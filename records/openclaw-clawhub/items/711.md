## Summary

Keep open: current main still lacks the requested DB-level nonsuspicious routing in lexical fallback search, and the PR is not ready as-is because it is stale relative to the current digest-based fallback and carries a blocking search-pagination regression.

## What This Changes

The branch changes lexical fallback search to use a nonsuspicious index and streamed scan when requested, updates search tests, and changes `/skills` search load-more to fetch 200 results once and paginate client-side.

## Best Possible Solution

Land a narrow update against current main’s `skillSearchDigest` fallback that selects `by_nonsuspicious_updated` and `by_nonsuspicious_created` with the active+nonsuspicious equality prefix when requested, preserves default search semantics, adds index-routing tests, and handles the fixed-200 pagination idea as a separate maintainer decision.

## Review Findings

- **[P2] Remove the fixed search result cap:** `src/routes/skills/-useSkillsBrowseModel.ts:111`
