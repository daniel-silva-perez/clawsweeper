## Summary

Keep this issue open. Current main still has the reported behavior: `clawhub publish` only uploads files returned by the extension allowlist collector, extensionless text executables are skipped before upload, `.tsv` is still absent from the shared text-file allowlist, and no skipped-file warning is emitted. Repository policy for openclaw/clawhub also says issues should stay open rather than be auto-closed.

## Best Possible Solution

Keep this issue open and land a focused fix: add `.tsv` to the shared text-file extension lists, include extensionless text files in CLI skill publish using a conservative binary sniff, warn authors about skipped non-text files, and add regression tests for extensionless scripts, `.tsv` files, binary skips, warning behavior, and any touched upload/import validation surfaces.
