## Summary

Keep this open. Repository policy for openclaw/clawhub says issues should not be auto-closed, and current main still has the reported CLI publish behavior: publish only uploads files returned by listTextFiles, listTextFiles filters solely by the shared extension allowlist, and .tsv plus extensionless text files are still excluded without any warning path.

## Best Possible Solution

Keep this issue open and fix the ClawHub skill publish collection path: add .tsv to the shared text-file allowlist, include extensionless text files using a conservative binary sniff, emit clear warnings for skipped files, and add focused regression tests for extensionless scripts, .tsv files, binary skips, and warning behavior.

## Risks / Open Questions

- Authors can still publish successful-looking skill versions that omit required extensionless wrapper scripts or .tsv configuration files, leaving installed skills broken.
- Adding only .tsv would not cover extensionless text executables; the collector needs a conservative text/binary sniff and skipped-file warnings.
- Related upload/import/hash paths also rely on extension allowlists in places, so the fix should check consistency across CLI publish, sync/fingerprint, GitHub import, and web upload behavior.
