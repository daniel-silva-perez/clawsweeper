## Summary

Keep open. This is an open ClawHub issue, and the repository policy says issues should not be auto-closed. The current main branch also still matches the reported bug: `clawhub publish` only uploads files returned by `listTextFiles`, and that collector still filters out extensionless paths and `.tsv` files without an author-facing warning.

## Best Possible Solution

Keep this issue open and fix the skill publish collection path. Add `tsv` to the shared text-file allowlist, include extensionless text files using a conservative binary sniff, emit clear publish warnings for skipped files, and add focused tests covering extensionless scripts, `.tsv` files, unsupported binary files, and skipped-file warnings. Then review sibling web upload, GitHub import, zip hashing, and sync paths for consistent behavior.

## Risks / Open Questions

- Authors can still publish successfully while required extensionless scripts or `.tsv` config/registry files are silently omitted, leaving installed skills broken.
- Adding only `tsv` would not address extensionless executable wrappers; the collector needs a conservative text/binary sniff or another explicit inclusion policy.
- Related upload/import/hash paths also use extension allowlists, so maintainers should decide whether the same policy applies there for consistency.
