## Summary

Keep this issue open. Repository policy for openclaw/clawhub only permits auto-close for pull requests that are certainly implemented on current main, and current main still has the reported behavior: CLI publish uploads only files returned by the extension allowlist collector, `.tsv` is absent from the shared allowlist, extensionless text files are skipped, and there is no skipped-file warning path.

## Best Possible Solution

Keep this issue open and fix the ClawHub skill publish collection path: add .tsv to the shared text-file allowlist, include extensionless text files using a conservative binary sniff, emit clear warnings for skipped files during clawhub publish, and add focused regression tests for extensionless scripts, .tsv files, binary skips, and warning behavior across the CLI and related upload/import validation surfaces.
