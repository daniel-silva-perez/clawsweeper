## Clownfish Work Prompt

Implement a focused fix for the current ClawHub issue where `clawhub publish` silently omits extensionless text files and `.tsv` files. Keep the change scoped: do not add broad binary publishing support.

Expected behavior:
- `.tsv` files such as `config/endpoints.tsv` are included by skill publish and accepted anywhere shared text-file validation applies.
- Extensionless text scripts such as `bin/gurkerl` or `bin/openclaw-kraken` are included when they are text.
- Binary or unsupported files remain excluded, but the CLI warns authors about skipped non-ignored files so a publish cannot silently omit required assets.
- Ignored files/directories from built-in ignores, `.gitignore`, `.clawhubignore`, and legacy ClawHub ignore paths should stay ignored without noisy warnings.

Likely implementation boundary:
- Update both `packages/clawhub/src/schema/textFiles.ts` and `packages/schema/src/textFiles.ts` plus committed schema dist/type output as appropriate.
- Refactor `packages/clawhub/src/skills.ts` so `listTextFiles` and zip hashing use a shared text-file decision helper: allow known text extensions, allow extensionless files only after a conservative binary sniff, and reject true binaries.
- Surface skipped-file data to `packages/clawhub/src/cli/commands/publish.ts` and print a concise warning before upload when non-ignored files are skipped.
- Check whether `convex/lib/skills.ts`, `convex/lib/skillPublish.ts`, `convex/lib/githubImport.ts`, `src/lib/uploadUtils.ts`, `src/routes/upload/-utils.ts`, and `src/lib/uploadFiles.ts` need matching tests or minor updates so accepted file rules remain consistent.
- Add regression tests for extensionless script inclusion, `.tsv` inclusion, binary skip behavior, skipped-file warning behavior, and fingerprint/sync consistency. Do not change unrelated package-publish behavior.

Changelog expectation: add a short fix note if the repo convention expects user-visible CLI fixes in `CHANGELOG.md`.
