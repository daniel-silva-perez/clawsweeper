## Summary

Keep this issue open. Current main still silently filters skill publish files through extension-only allowlists: extensionless text scripts and `.tsv` files are still omitted before upload/archive import paths can inspect content, and the ClawHub policy keeps issues open rather than auto-closing them.

## What This Changes

The issue requests that `clawhub publish` include extensionless text executables and `.tsv` files instead of silently dropping them through fixed extension allowlists.

## Best Possible Solution

Implement a focused publish-file classification fix: add `.tsv` to the shared and bundled CLI text allowlists, add a conservative text-content fallback for extensionless files in CLI and archive/web import paths, emit clear CLI publish warnings for unsupported skipped files, keep binary files rejected, and add regression tests for extensionless bin scripts and `.tsv` files.
