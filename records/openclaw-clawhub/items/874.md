## Summary

Keep this issue open. Current main still matches the reported publish bug: CLI and archive/GitHub import paths decide text eligibility from extensions before any byte/content fallback, `.tsv` is absent from the text allowlists, and omitted files are not surfaced to the publisher.

## What This Changes

The issue asks for skill publish to include `.tsv` files and extensionless text executables, and to warn when files are skipped during publish.

## Best Possible Solution

Adopt one conservative text-file classification policy across skill publish entry points: add `.tsv` to the allowlists, classify extensionless text files by content without admitting binaries, warn authors about skipped files, and add regression tests for CLI publish, archive/web upload, GitHub import, and server validation.
