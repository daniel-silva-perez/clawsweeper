## Summary

Keep this issue open. Repository policy for openclaw/clawhub keeps issues open, and current main still has the reported behavior: CLI publish only uploads files returned by the extension allowlist collector, extensionless paths are skipped, `.tsv` is not in the shared text-file allowlist, and there is no skipped-file warning in the publish path.

## Best Possible Solution

Keep this issue open and fix the ClawHub skill publish collection path: add .tsv to the shared text-file allowlist, include extensionless text files using a conservative binary sniff, emit clear warnings for skipped files during clawhub publish, and add focused regression tests for extensionless scripts, .tsv files, binary skips, and warning behavior across the CLI and related upload/import validation surfaces.
