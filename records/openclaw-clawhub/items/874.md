## Summary

Keep open. Current main still matches the reported publish-file classification bug: `.tsv` is not allowlisted, extensionless files are skipped before content inspection in CLI and archive/GitHub import paths, and omitted files are not surfaced to the publisher. The ClawHub cleanup policy also keeps issues open rather than auto-closing them, while this one is narrow enough for a focused repair PR.

## What This Changes

`clawhub publish` should include `.tsv` files and extensionless text files instead of silently dropping them, and should warn authors about skipped files.

## Best Possible Solution

Adopt one conservative text-file classification policy across skill publish entry points: add `.tsv`, classify extensionless text files by content without admitting binaries, surface skipped-file warnings to authors, and cover CLI publish, archive/web upload, GitHub import, and server validation with regression tests.

## Repair Work Prompt

Fix the skill publish/import bug where `.tsv` files and extensionless text files are silently omitted. Implement a shared conservative text-file classification approach across CLI file discovery/hash paths, shared and bundled schema allowlists, web archive upload, GitHub import selection, and Convex publish validation. Add `.tsv` at minimum; only add other extensions such as `conf`, `properties`, or `dat` if covered by tests and consistent with the text-only policy. For extensionless files, inspect bytes before accepting them as text, reject binary-like content such as null bytes, assign a safe text content type, and keep path sanitization, `.git`/`node_modules`/`.clawhub` ignores, macOS junk filtering, size limits, and binary rejection intact. Surface author-facing warnings for skipped files in CLI publish/sync and web archive upload where practical without warning on ignored paths. Add regression tests for a `bin/openclaw-kraken`-style extensionless script, `config/endpoints.tsv`, and a binary extensionless file. Update changelog/docs only if the repo convention expects a user-facing CLI publish behavior note; do not change plugin publishing or release workflows.
