## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review keeps issues open. Independently, current main still matches the reported bug: CLI skill publish collects only files whose final dotted suffix is in the shared text extension allowlist, so extensionless scripts and `.tsv` files are excluded before upload and there is no author-facing warning.

## Best Possible Solution

Keep this issue open and fix the skill publish collection path. A conservative implementation should add `tsv` to the shared text extension allowlist, include extensionless files only when they pass a text/binary sniff, emit clear warnings for skipped files during publish, and add focused tests for extensionless scripts, `.tsv` files, unsupported binary files, and the warning behavior. Related collectors such as zip hashing, web upload filtering, and GitHub import should be checked for consistency with the chosen text-file policy.
