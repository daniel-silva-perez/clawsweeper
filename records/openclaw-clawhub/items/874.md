## Summary

Keep this issue open. Current main still matches the report: CLI skill publish filters files by extension before reading bytes, the shared text extension allowlist still lacks tsv, and archive/web helpers also reject extensionless paths in several flows. ClawHub review policy also keeps issues open, but this one is independently valid and narrow enough for a focused fix PR.

## Best Possible Solution

Implement a focused publish-file classification fix: add tsv to the shared/CLI text allowlists, add a conservative text-content fallback for extensionless files in CLI and archive/web expansion paths, emit clear warnings for unsupported skipped files during CLI publish, and add regression tests for extensionless bin scripts plus .tsv files without broadening binary upload acceptance.
