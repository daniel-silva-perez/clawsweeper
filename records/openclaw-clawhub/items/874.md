## Clownfish Work Prompt

Fix the publish-file classification bug reported in https://github.com/openclaw/clawhub/issues/874. Current behavior: clawhub publish silently drops extensionless text files such as bin/gurkerl/bin/openclaw-kraken and .tsv files because CLI and archive/web helpers filter by extension before reading bytes. Implement a focused fix only in the publish/upload text-file classification surface. Add tsv to shared and CLI text extension allowlists, update content-type guessing/normalization as needed, add a conservative byte/content sniff fallback for extensionless files in CLI listTextFiles/hashSkillZip and web/archive expansion paths, and emit a clear CLI publish warning/report for unsupported skipped files. Do not accept binary files broadly: reject null bytes and obvious binary content, keep existing mac-junk and ignore behavior, and do not change package/plugin binary archive behavior beyond this skill publish boundary. Add regression tests for extensionless bin scripts, .tsv files, binary extensionless rejection, and skipped-file warnings. Update committed generated/dist schema outputs only if this repo's package workflow requires them for source changes. No unrelated refactors or repo-wide formatting.
