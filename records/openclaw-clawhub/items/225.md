## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that current main certainly implements. Current main partially addresses the request with skill publishing, ignore-file handling, persisted static scan results, and acceptable-usage docs, but the requested skill lint command, skill publish dry-run, and shared pre-upload suspicious-scan feedback are still missing.

## What This Changes

Request a better skill-publishing onboarding flow with a CLI linter/upload tooling, clearer .git and ignore-file behavior, pre-upload suspicious warnings, abuse-policy clarity, and remediation guidance for flagged skills.

## Best Possible Solution

Build a shared skill preflight layer that both CLI and browser upload can call: local/schema/path/text/static checks where feasible, a `clawhub skill lint <path>` command, a `clawhub skill publish --dry-run` mode, remediation-oriented suspicious findings, explicit ignore-file behavior, and user-facing links or copy for acceptable-usage enforcement.
