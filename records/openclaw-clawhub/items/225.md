## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main partially addresses the request with CLI skill publishing, CLI-side ignore handling, server-side static scan persistence, and acceptable-usage docs, but it still lacks a `skill lint` command, `skill publish --dry-run`, and shared pre-upload suspicious-scan feedback for browser and CLI uploads.

## Best Possible Solution

Keep this issue open. The best path is a maintainer-approved shared skill preflight layer used by both CLI and web upload: `clawhub skill lint <path>`, `clawhub skill publish --dry-run`, reusable schema/path/text/static-scan checks before final publish/storage where feasible, suspicious findings shown before final publish, explicit browser `.git`/ignore-file behavior, and links from upload errors to acceptable-usage and remediation guidance.
