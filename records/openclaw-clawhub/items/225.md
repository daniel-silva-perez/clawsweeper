## Summary

Keep open. Repository cleanup policy for openclaw/clawhub keeps issues open, and current main only partially addresses the request: skill publishing, ignore handling, server-side static scan persistence, and acceptable-usage copy exist, but there is still no direct skill lint command, no direct `skill publish --dry-run`, and no shared pre-upload suspicious-scan feedback across browser and CLI flows.

## What This Changes

The issue requests better skill onboarding and upload management through linting, CLI upload tooling, pre-upload suspicious warnings, clearer abuse guidance, and a contributor-friendly moderation path.

## Best Possible Solution

Build a shared skill preflight layer that both CLI and browser upload can call: local schema/path/text/static checks where feasible, a `clawhub skill lint <path>` command, a direct `clawhub skill publish --dry-run` mode, remediation-oriented suspicious findings, explicit ignore-file behavior, and user-facing acceptable-usage enforcement copy or links.
