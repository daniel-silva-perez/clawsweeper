## Summary

Keep this issue open. The current main branch has shipped useful pieces of the requested surface, including CLI skill publishing, sync dry-run, plugin/package publish dry-run, `.git` pruning for CLI skill file collection, acceptable-usage docs, and server-side static moderation. The central onboarding/upload-management request is still not complete for skills: there is no `clawhub skill lint` or `clawhub skill publish --dry-run`, the web wizard only performs basic metadata/file validation, and static suspicious findings are produced during server-side publish after files have already been uploaded to storage. The openclaw/clawhub review policy also says ClawHub issues should be kept open rather than auto-closed.

## Best Possible Solution

Keep this issue open. The best path is a shared skill preflight layer used by both CLI and web: add `clawhub skill lint <path>` and `clawhub skill publish --dry-run`, reuse the same schema/path/text/static-scan checks as publish, show static findings in the web wizard before final upload/publish, explicitly ignore or explain `.git` handling for browser uploads, and link acceptable-usage guidance from the upload flow with clear remediation language for suspicious findings and repeated uploads.
