## Summary

Keep open. The repository policy for ClawHub issues does not allow automatic closure here, and current main only partially addresses the request: skill publish/sync and ignore handling exist, but there is still no direct skill lint command, no `skill publish --dry-run`, and no shared pre-upload suspicious-scan feedback across browser and CLI flows.

## What This Changes

The issue requests better skill author onboarding with a linter or CLI upload tool, pre-upload suspicious warnings, clearer ignore and abuse-policy guidance, and a contributor-friendly remediation/sandbox flow.

## Best Possible Solution

Build a shared skill preflight layer that both CLI and browser upload can call: local schema/path/text/static checks where feasible, a `clawhub skill lint <path>` command, a direct `clawhub skill publish --dry-run` mode, remediation-oriented suspicious findings, explicit ignore-file behavior, and user-facing acceptable-usage enforcement copy or links.
