Current main partially addresses the report with CLI skill publishing, `.git`/ignore handling, persisted static scan results, owner rescan/security surfaces, and acceptable-usage policy text, but it still lacks the central requested preflight layer: a dedicated `skill lint` command, `skill publish --dry-run`, and shared suspicious-scan feedback before browser/CLI upload completion.
