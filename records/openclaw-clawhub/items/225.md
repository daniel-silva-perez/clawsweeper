## Summary

Keep open. This is a ClawHub issue, and the review policy says ClawHub issues should not be auto-closed. Current main has useful partial fixes, but the core skill onboarding request is still incomplete: there is no skill lint command, no skill publish dry-run, and suspicious findings are still generated server-side after uploaded files are already in storage.

## Best Possible Solution

Keep this issue open. The best path is a shared skill preflight layer used by both CLI and web: add `clawhub skill lint <path>` and `clawhub skill publish --dry-run`, reuse the same schema/path/text/static-scan checks as publish, show static findings before final web upload/publish, document .git handling for browser uploads, and link acceptable-usage guidance from the upload flow with clear remediation language for suspicious findings and repeated uploads.
