## Summary

Keep open. Repository policy for this ClawHub review keeps issues open, and the substantive request is still only partially implemented on current main: CLI skill publishing exists and has some local file filtering, but there is still no skill lint command or skill publish dry-run, and suspicious/static-scan findings are still produced after files have been uploaded or stored rather than as a pre-upload warning.

## Best Possible Solution

Keep this issue open. The best product path is a shared skill preflight layer used by CLI and web: add `clawhub skill lint <path>` and `clawhub skill publish --dry-run`, reuse publish schema/path/text/static-scan checks before final upload, show suspicious findings before final web upload/storage, document `.git` handling clearly for browser uploads, and link acceptable-usage/remediation guidance from the upload flow.
