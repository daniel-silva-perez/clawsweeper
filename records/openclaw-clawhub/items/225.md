## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main also only partially addresses the requested onboarding/upload workflow: CLI skill publishing exists and CLI-side ignore handling is documented, but there is still no `skill lint`, no `skill publish --dry-run`, no shared pre-upload suspicious-scan/preflight path, and browser folder upload does not mirror or explain CLI `.git`/ignore-file behavior.

## Best Possible Solution

Keep this issue open. The best path is to add a shared skill preflight layer used by both CLI and web upload: `clawhub skill lint <path>`, `clawhub skill publish --dry-run`, reusable schema/path/text/static-scan checks before final upload/storage where possible, suspicious findings shown before final publish, explicit browser `.git`/ignore-file behavior, and links from the upload flow to acceptable-usage/remediation guidance.
