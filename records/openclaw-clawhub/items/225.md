## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that current main certainly implements. Current main partially addresses onboarding with skill publishing, ignore handling, server-side static scan persistence, and acceptable-usage docs, but the requested skill lint command, skill publish dry-run, and shared pre-upload suspicious-scan feedback are still missing.

## What This Changes

Request to improve skill publishing onboarding with CLI lint/upload tooling, pre-upload suspicious scan feedback, clearer .git/ignore handling, and clearer abuse/sandboxing guidance.

## Best Possible Solution

Build a shared skill preflight layer that both CLI and browser upload can call: local schema/path/text/static checks where feasible, a `clawhub skill lint <path>` command, a `clawhub skill publish --dry-run` mode, remediation-oriented suspicious findings, explicit ignore-file behavior, and user-facing acceptable-usage enforcement copy or links.
