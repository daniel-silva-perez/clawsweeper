## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main partially addresses the request with CLI skill publishing, CLI-side ignore handling, persisted static scan results, owner/admin rescan affordances, and acceptable-usage documentation, but it still lacks a dedicated `skill lint` command, `skill publish --dry-run`, and shared pre-upload suspicious-scan feedback across browser and CLI upload flows.

## Best Possible Solution

Keep this issue open for maintainer scoping. The best path is a maintainer-approved shared skill preflight layer used by both CLI and web upload: `clawhub skill lint <path>`, `clawhub skill publish --dry-run`, reusable schema/path/text/static-scan checks before final publish/storage where feasible, suspicious findings shown in a remediation-oriented way before final publish, explicit browser `.git`/ignore-file behavior, and upload/rescan guidance linked to the public acceptable-usage policy.
