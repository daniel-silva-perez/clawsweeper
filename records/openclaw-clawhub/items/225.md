## Summary

Keep open. This is an open ClawHub issue, and the repository-specific policy for this cleanup run only permits auto-closing ClawHub pull requests that are certainly implemented on main. Current main also still only partially addresses the request: CLI skill publishing and ignore handling exist, and abuse policy docs are clearer, but there is still no `skill lint`, no per-skill publish dry run, no shared preflight suspicious-scan path before upload, and browser upload handling does not mirror CLI ignore behavior clearly.

## Best Possible Solution

Keep this issue open. The best path is a shared skill preflight layer used by both CLI and web upload: add `clawhub skill lint <path>` and `clawhub skill publish --dry-run`, reuse schema/path/text/static-scan checks before final upload/storage where possible, show suspicious findings before final publish, document `.git` and ignore-file behavior for browser uploads, and link acceptable-usage/remediation guidance from the upload flow.
