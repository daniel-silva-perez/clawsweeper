Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that current main certainly implements. Current main partially addresses the report with CLI skill publishing, .git/.gitignore/.clawhubignore handling, persisted static scan results, and acceptable-usage docs, but it still lacks the central requested preflight layer: a dedicated skill lint command, skill publish dry-run mode, and shared suspicious-scan feedback before browser/CLI upload completion.

Keep this issue open for maintainer scoping. The best product/code path is a shared skill preflight layer used by both CLI and browser upload: clawhub skill lint <path>, clawhub skill publish --dry-run, reusable schema/path/text/static-scan checks before final publish/storage where feasible, remediation-oriented suspicious findings, explicit .git/ignore-file behavior, and links to acceptable-usage policy text.
