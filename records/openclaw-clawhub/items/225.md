## Summary

Keep open. This is an open ClawHub issue, and the repo-specific cleanup policy only permits auto-closing ClawHub pull requests that current main certainly implements. Current main partially addresses the report with CLI skill publishing, CLI-side .git/.gitignore/.clawhubignore handling, persisted static scan results, and acceptable-usage guidance, but it still lacks the requested shared pre-upload/preflight layer: a skill lint command, skill publish dry-run mode, and suspicious-scan feedback before browser or CLI upload completion.

## What This Changes

Request clearer onboarding and skill upload management with CLI lint/upload tooling, pre-upload suspicious warnings, explicit .git/ignore behavior, and clearer abuse/sandbox guidance.

## Best Possible Solution

Keep this issue open for maintainer scoping. The best product/code path is a shared skill preflight layer used by both CLI and browser upload: clawhub skill lint <path>, clawhub skill publish --dry-run, reusable schema/path/text/static-scan checks before final publish/storage where feasible, remediation-oriented suspicious findings, explicit .git/ignore-file behavior, and links to acceptable-usage policy text.
