## Summary

Keep open. Repository cleanup policy keeps ClawHub issues open, and current main only partially addresses the request: skill CLI publish/sync, ignore handling, server-side static scan persistence, and acceptable-usage docs exist, but there is still no skill lint command, no direct `skill publish --dry-run`, and no shared pre-upload suspicious-scan feedback for the browser and CLI flows.

## What This Changes

The issue asks for clearer skill-upload onboarding, a skill linter or CLI upload tooling, earlier suspicious-skill feedback, clearer abuse guidance, and more collaborative handling of flagged modules.

## Best Possible Solution

Build a shared skill preflight layer that both CLI and browser upload can call: local schema/path/text/static checks where feasible, a `clawhub skill lint <path>` command, a direct `clawhub skill publish --dry-run` mode, remediation-oriented suspicious findings, explicit ignore-file behavior, and user-facing acceptable-usage enforcement copy or links.
