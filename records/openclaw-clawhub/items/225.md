## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing ClawHub pull requests that current main certainly implements. Current main partially addresses the request with CLI skill publishing, CLI ignore handling, persisted static scan results, and acceptable-usage docs, but it still lacks the requested skill lint command, skill publish dry-run, and shared pre-upload suspicious-scan feedback for browser and CLI uploads.

## What This Changes

The issue requests clearer skill onboarding and upload management, especially CLI lint/upload tooling, pre-upload suspicious feedback, .git handling clarity, abuse policy clarity, and remediation-oriented handling for flagged skills.

## Best Possible Solution

Keep this issue open for maintainer scoping. The best product/code path is a shared skill preflight layer used by both CLI and browser upload: a `clawhub skill lint <path>` command, a `clawhub skill publish --dry-run` mode, reusable schema/path/text/static-scan checks before final publish/storage where feasible, remediation-oriented suspicious findings, explicit .git/.gitignore/.clawhubignore behavior, and clear links or copy for acceptable-usage enforcement.
