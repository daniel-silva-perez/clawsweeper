## Summary

Keep open. Repository-specific policy keeps ClawHub issues open, and current main only partially addresses the request: CLI skill publishing and sync exist, ignore rules are honored, and abuse policy docs are clearer, but there is still no skill lint command, no per-skill publish dry-run, and no suspicious-scan preflight before web or CLI upload completes.

## Best Possible Solution

Keep this issue open. The best path is a shared skill preflight layer used by both CLI and web upload: add `clawhub skill lint <path>` and `clawhub skill publish --dry-run`, reuse schema/path/text/static-scan checks before final upload/storage, show suspicious findings before final publish, document `.git` and ignore-file behavior for browser uploads, and link acceptable-usage/remediation guidance from the upload flow.
