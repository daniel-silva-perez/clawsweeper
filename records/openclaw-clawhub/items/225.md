## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that are certainly implemented on current main. Current main partially addresses the request with CLI skill publishing, CLI-side ignore handling, server-side static scan, and acceptable-usage documentation, but it still does not provide a skill lint command, `clawhub skill publish --dry-run`, or shared pre-upload suspicious-scan feedback for both CLI and browser uploads.

## Best Possible Solution

Keep this issue open. The best path is to add a shared skill preflight layer used by both CLI and web upload: `clawhub skill lint <path>`, `clawhub skill publish --dry-run`, reusable schema/path/text/static-scan checks before final upload/storage where possible, suspicious findings shown before final publish, explicit browser `.git`/ignore-file behavior, and links from the upload flow to acceptable-usage and remediation guidance.
