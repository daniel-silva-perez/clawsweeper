## Summary

Keep open. This is an issue, and the repository policy for openclaw/clawhub only permits conservative auto-close for pull requests that are certainly implemented on main. The current code also makes the report ambiguous: `clawhub install` handles one slug without invoking the local scanner, while the full-scan behavior is in `clawhub sync` and tied to documented full-snapshot telemetry and removal detection.

## What This Changes

Request to make skill installation scan only the newly installed skill instead of rescanning all installed skills, with full scans left as an optional/manual operation.

## Best Possible Solution

Keep the issue open for maintainer triage. First reproduce the slow workflow and version; if the target is `sync`, design cached fingerprints or incremental parsing that preserves full-root snapshot telemetry and removal detection, with an explicit manual full-rescan path.
