The ClawHub cleanup policy keeps issues open in this review lane, and the source review shows the report is not yet a narrow implementation task: current `clawhub install` installs one slug without invoking the local skill scanner, while the full scan happens in `clawhub sync` and is coupled to full-snapshot telemetry and removal detection.

## What This Changes

The issue requests optimizing ClawHub skill installation so adding one skill scans only the new skill instead of rescanning all installed skills.

## Best Possible Solution

Keep the issue open for maintainer triage. First reproduce the exact slow workflow and CLI version; if the target is `sync`, design cached fingerprints or incremental parsing that preserves full-root snapshot telemetry and removal detection, with an explicit manual full-rescan path.
