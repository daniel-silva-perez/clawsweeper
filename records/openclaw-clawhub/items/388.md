## Summary

Keep open. Repository policy for openclaw/clawhub only permits auto-closing pull requests that current main certainly implements, and this item is an issue. Substantively, current CLI install is already single-slug and does not call the local skill scanner; the full local scan is in `clawhub sync`, where documented telemetry and removal detection currently depend on full snapshots.

## What This Changes

Request to optimize skill installation so installing one new skill scans only that newly installed skill instead of rescanning all installed skills.

## Best Possible Solution

Keep this open for maintainer triage. Ask the reporter to confirm whether the slowdown was observed in `clawhub install`, `clawhub sync`, the ClawHub web/OpenClaw host installer, or an older CLI version. If the target is `sync`, design a cached-fingerprint or incremental scan path that preserves full-snapshot telemetry and removal detection, with an explicit manual full-rescan option.
