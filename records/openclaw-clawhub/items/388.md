## Summary

Keep open. Repository policy for openclaw/clawhub keeps issues open during this cleanup pass, and the substantive report is still ambiguous: current `clawhub install` installs one slug without invoking the local scanner, while the full-scan behavior appears in `clawhub sync` and is tied to documented telemetry/removal semantics.

## What This Changes

Optimize skill installation so adding one new skill scans only that skill instead of rescanning every installed skill.

## Best Possible Solution

Keep this open for maintainer triage. First identify and reproduce the slow workflow; if `sync` is the target, design cached fingerprints or incremental parsing that preserves full-root snapshot telemetry and removal detection, with an explicit manual full-rescan path.
