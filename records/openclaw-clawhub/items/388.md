## Summary

Keep open. The repository-specific review policy for openclaw/clawhub allows auto-close only for pull requests that current main definitely implements, not for issues. Substantively, current main’s ClawHub CLI `install` path is already single-slug and does not call the local root scanner; the scan-all behavior I found is in `clawhub sync`, where docs also describe a full snapshot for telemetry/removal detection. This needs maintainer clarification about whether the slow surface is sync, the ClawHub web prompt/OpenClaw host installer, or an older CLI version.

## Best Possible Solution

Keep this issue open for maintainer triage. Ask the reporter whether the slowdown was observed in `clawhub install`, `clawhub sync`, the ClawHub web copy/prompt flow, or an OpenClaw host-side `skills install` implementation. If it is sync, design a narrower performance change around cached fingerprints, incremental root scanning, or an explicit manual full-rescan mode while preserving telemetry/removal semantics.
