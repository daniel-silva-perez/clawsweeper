Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that current main certainly implements. Current main appears to keep `clawhub install <slug>` targeted to one skill while full local skill-folder scanning belongs to `clawhub sync`, but maintainer triage is still needed to confirm the reporter's actual CLI version and slow path.

## Best Possible Solution

Keep this open for maintainer triage. If the report targets current `clawhub install`, add a focused regression test that mocks the scan helpers and proves install does not call them, then let a maintainer close manually. If the slow path is actually `clawhub sync`, an older CLI, or downstream OpenClaw installation behavior, rescope the issue to that flow and consider an incremental install/sync state update while preserving full scans for manual sync and repair.
