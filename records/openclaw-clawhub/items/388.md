## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy for this lane only permits auto-closing ClawHub pull requests that current main certainly implements. Substantively, current CLI `install` is already single-slug and does not call the local skill scanner; the full scan behavior exists in `clawhub sync`, where docs describe full-snapshot telemetry/removal semantics that need maintainer design before an incremental-only change.

## What This Changes

Request to optimize skill installation so adding a new skill scans only that new skill instead of rescanning all installed skills, with full scans left as optional/manual behavior.

## Best Possible Solution

Keep this issue open for maintainer triage. Ask the reporter to confirm whether the slowdown was observed in `clawhub install`, `clawhub sync`, the ClawHub web/OpenClaw host installer, or an older CLI version. If the target is `sync`, design a narrow cached-fingerprint or incremental scan change that preserves full-snapshot telemetry and removal detection, with an explicit manual full-rescan path if needed.
