## Summary

Keep open. The repository cleanup policy for openclaw/clawhub only permits auto-closing pull requests that current main definitely implements, and this item is an issue. The code also shows the reported surface is ambiguous: current `clawhub install` installs one slug without invoking the local skill scanner, while `clawhub sync` still performs a full local scan that is tied to documented full-snapshot telemetry and removal detection.

## What This Changes

The issue asks to optimize skill installation so adding a new skill scans only that newly installed skill instead of rescanning all installed skills.

## Best Possible Solution

Keep the issue open for maintainer triage. First confirm the command and version that are slow; if the target is `sync`, design cached fingerprints or incremental parsing that preserves full-root snapshot telemetry and removal detection, with an explicit manual full-rescan path.
