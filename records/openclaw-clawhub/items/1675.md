## Summary

Keep this open. Current main supports org publishers and web owner selection for new skill publishes, but the reported no-loss transfer of an existing personal skill to an org is still modeled as user-to-user transfer; the ClawHub cleanup policy also keeps issues open.

## What This Changes

Support org ownership for skills, especially transferring existing personal skills to organization publishers without losing stats or canonical URLs.

## Best Possible Solution

Implement a first-class publisher-target skill transfer flow that resolves org handles, authorizes through publisher membership roles, updates skills and slug aliases without losing stats, preserves compatibility for current clients, and adds regression coverage for user-to-org, org-to-user, and org-to-org transfers.
