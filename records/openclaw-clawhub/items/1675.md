## Summary

Keep this issue open. ClawHub cleanup policy says issues should not be auto-closed by this workflow, and current main still has the reported product gap: skill transfer records and APIs are user-targeted, org publisher handles are excluded by transfer recipient resolution, and publish-as-org support does not provide a no-loss re-home path for an already personal-owned skill.

## Best Possible Solution

Keep this issue open for core implementation. The best path is to migrate skill transfers from user-recipient records to publisher-target records, resolve org handles as valid destinations, authorize acceptance by owner/admin roles on the target publisher, update skills and slug aliases without losing stats or redirects, and add coverage for user-to-org, org-to-user, and org-to-org transfers. Existing publish-as-org support can remain, but it does not replace a safe re-home flow for already published skills.
