## Summary

Keep this open. Current main supports org publishers for new skill publishes through the web flow, but existing skill transfers are still modeled as user-to-user transfers and cannot resolve org publisher handles as transfer targets.

## What This Changes

The issue requests a no-loss path to move an existing personal-owned skill under an org owner, either via transfer-to-org, publish-as-org, or re-homing on publish.

## Best Possible Solution

Implement publisher-target ownership transfers in core: store source and target publisher IDs, resolve org handles through publishers, authorize request/cancel from the current owner publisher, authorize accept/reject by owner/admin on the target publisher per docs/orgs.md, update skills and slug aliases without resetting stats or versions, preserve existing user-to-user API compatibility where possible, and add regression coverage for user-to-org, org-to-user, and org-to-org transfers.

## Repair Work Prompt

Implement org-capable skill ownership transfers for openclaw/clawhub. Current main only supports user-to-user transfers via skillOwnershipTransfers.fromUserId/toUserId and toUserHandle. Change the transfer model and handlers so transfer targets are publishers: user publisher -> org publisher, org publisher -> user publisher, and org publisher -> org publisher. Follow docs/orgs.md for acceptance semantics: actors with owner or admin on the target publisher may accept. Preserve existing user-to-user transfer clients where reasonable, including accepting the existing toUserHandle payload as an alias for a publisher handle. Do not reset skill stats, versions, or download data. Update skills and skillSlugAliases ownership consistently and preserve existing slug alias/redirect behavior. Add or update tests covering user-to-org, org-to-user, org-to-org, compatibility for user-to-user transfers, HTTP handler behavior, and CLI/schema response changes. Keep the change scoped to transfer ownership; do not redesign publishing, org membership, or unrelated skill ownership code.
