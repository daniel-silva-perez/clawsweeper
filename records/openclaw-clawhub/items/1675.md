## Summary

Keep this issue open. The repository policy for openclaw/clawhub only allows auto-closing pull requests that are certainly implemented on main, and this item is an open issue. The current main branch also still has a real product gap: skill ownership transfer is modeled as user-to-user, while org publishers are not valid transfer recipients.

## Best Possible Solution

Keep this issue open for core implementation. The most direct path is to migrate ownership transfer from user-recipient records to publisher-target records, resolve org handles as valid transfer destinations, authorize acceptance by org owners/admins or publisher-role members according to maintainer policy, update skill and slug-alias ownership without losing stats or redirects, and add coverage for user-to-org, org-to-user, and org-to-org transfers. If maintainers choose publish-as-org instead, the public HTTP API and skill CLI need `ownerHandle` support plus an explicit safe re-home flow for existing skills.
