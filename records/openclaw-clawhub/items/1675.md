## Summary

Keep open. Repository policy for openclaw/clawhub keeps issues open, and current main still does not implement the reported existing-skill path from personal ownership to org ownership. Main has partial publisher support, including a web owner selector for new web publishes, but the skill transfer schema/API/CLI remain user-recipient based and the skill CLI/public publish API still do not expose a skill owner handle for org publishing.

## Best Possible Solution

Keep this issue open and implement a publisher-targeted ownership path. The likely maintainable path is to migrate skillOwnershipTransfers from user IDs to source/target publisher IDs, resolve transfer targets by publisher handle, require owner/admin role on the target publisher for acceptance, update skill and alias ownerPublisherId without losing stats, and add CLI/API/docs/tests. If maintainers choose publish-as-org instead, skill CLI and public HTTP publish should accept ownerHandle and there should still be an explicit migration path for existing personal-publisher skills.
