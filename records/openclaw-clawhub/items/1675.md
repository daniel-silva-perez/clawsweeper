## Summary

Keep this issue open. Current main has partial org publisher support, including a web publish owner selector, but the no-loss personal-to-org skill transfer path remains missing: skill transfers are still modeled around `fromUserId`/`toUserId`, transfer request lookup rejects org publishers, accept always assigns the recipient's personal publisher, and the skill CLI publish path still has no `--owner` payload for skills. ClawHub cleanup policy also says issues are not auto-closed here; only PRs that are certainly implemented on main are eligible.

## What This Changes

The issue requests support for organization-owned skills and a no-loss path to transfer an existing personal skill to an organization owner.

## Best Possible Solution

Implement a core publisher-targeted skill transfer flow: resolve user and org publisher handles, authorize by publisher membership, preserve stats and aliases, keep old user-to-user clients compatible, update CLI/API schemas, and add regression coverage for personal-to-org, org-to-user, and org-to-org transfers.
