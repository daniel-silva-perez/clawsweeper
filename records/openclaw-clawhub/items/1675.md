## Summary

Keep this issue open. Repository policy for openclaw/clawhub only permits automated close proposals for pull requests that current main certainly implements, and this item is an issue. The substantive gap also remains on current main: skill transfers are still user-to-user, org publisher handles are excluded from transfer recipient lookup, transfer acceptance assigns the recipient user's personal publisher, and the org docs still describe publisher-target transfers as future work.

## Best Possible Solution

Keep this issue open for core implementation. The best path is to migrate skill ownership transfers from user-recipient records to publisher-target records, resolve org handles as valid destinations, authorize acceptance by owner/admin roles on the target org publisher, preserve stats and aliases during ownership changes, and add coverage for user-to-org, org-to-user, and org-to-org transfer flows. Existing publish-as-org support can stay, but it does not replace a safe re-home flow for already published skills.
