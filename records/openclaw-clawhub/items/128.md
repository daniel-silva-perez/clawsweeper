## Summary

Current main still lacks `clawhub update --dry-run`. The update command only registers `--all`, `--version`, and `--force`, and `cmdUpdate` still flows into filesystem mutation, zip download/extraction, origin metadata writes, and lockfile writes. This is a narrow, valid CLI enhancement and is a good fix-PR candidate.

## Best Possible Solution

Keep this issue open and implement a focused CLI enhancement: add `--dry-run` to `clawhub update`, share the existing update resolution checks, print a useful preview for single-skill and `--all` flows, and guarantee no filesystem, origin metadata, download/extract, or lockfile mutation in dry-run mode. Use the existing sync/package publish dry-run patterns as precedent without changing those commands.

## Clownfish Work Prompt

Implement `--dry-run` for `clawhub update` only. Add the CLI option in `packages/clawhub/src/cli.ts` and extend `cmdUpdate` options in `packages/clawhub/src/cli/commands/skills.ts`. Print a concise preview that distinguishes would-update, already-up-to-date/already-at-version, skipped/blocked, suspicious, and local-change cases. Add focused tests in `packages/clawhub/src/cli/commands/skills.test.ts` using existing mocks to prove dry-run resolves update state but performs no filesystem/download/lockfile writes. Do not touch package publish or sync dry-run behavior, generated files, or dist output.
