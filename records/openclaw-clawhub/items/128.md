## Summary

Keep open. Current main still lacks `clawhub update --dry-run`, and ClawHub cleanup policy keeps issues open unless another close rule applies; this remains a narrow, valid CLI enhancement and is a good repair-PR candidate.

## What This Changes

Request to add a preview-only `--dry-run` mode to `clawhub update` that reports pending skill updates without changing installed skills or lockfile state.

## Best Possible Solution

Keep this issue open and implement a focused CLI enhancement: add `--dry-run` to `clawhub update`, reuse existing metadata/fingerprint/version resolution, print a clear preview for single-skill and `--all` flows, and guarantee dry-run mode performs no installed-skill, origin metadata, download/extraction, or lockfile mutation. Update CLI docs and add focused tests around no side effects.

## Repair Work Prompt

Implement a focused ClawHub CLI enhancement for `clawhub update --dry-run`. Add flag registration in `packages/clawhub/src/cli.ts` and a `dryRun` option path in `packages/clawhub/src/cli/commands/skills.ts`. Reuse the current metadata, moderation, local fingerprint, and version-resolution checks, but in dry-run mode print a preview for single-skill and `--all` flows showing would-update, already-up-to-date, skipped/blocked/not-found/local-change statuses. Update `docs/cli.md` and add an Unreleased changelog entry for the CLI-visible feature. Do not change package publish or sync dry-run behavior except as a style reference.
