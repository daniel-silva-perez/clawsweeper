## Summary

Current main still does not implement `clawhub update --dry-run`, and the ClawHub cleanup policy says issues should remain open. The update command only exposes `--all`, `--version`, and `--force`; `cmdUpdate` still performs directory replacement, origin metadata writes, and lockfile writes on the update path with no dry-run planning branch.

## Best Possible Solution

Keep this issue open and implement the feature in the ClawHub CLI core: add `--dry-run` to `clawhub update`, share a planning path for single-skill and `--all` flows, print would-update/up-to-date/skipped/local-change states, guarantee no directory/origin/lockfile writes during dry-run, add focused no-write tests, and document the flag with the existing update options.
