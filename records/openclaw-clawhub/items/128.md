## Summary

Current main still does not implement `clawhub update --dry-run`. The update command only exposes `--all`, `--version`, and `--force`, and `cmdUpdate` still proceeds through lockfile normalization plus filesystem/download/origin/lockfile writes. Because this is an issue in `openclaw/clawhub`, repository policy keeps it open, and it is a narrow, valid fix-PR candidate.

## Best Possible Solution

Keep this issue open and implement a focused CLI enhancement: add `--dry-run` to `clawhub update`, reuse the existing update resolution checks, print a useful preview for single-skill and `--all` flows, and guarantee dry-run mode performs no filesystem, origin metadata, download/extract, or lockfile mutation. Use the existing sync and package publish dry-run paths as precedents, without changing those commands.
