## Summary

Keep open. Current main still does not implement a dry-run mode for `clawhub update`; the command wiring, implementation, docs, and tests all cover only the mutating update path. The request is narrow and valid, and the repository policy keeps issues open rather than auto-closing them.

## What This Changes

Add a `--dry-run` flag to `clawhub update` so users can preview available skill updates without changing installed files or lock metadata.

## Best Possible Solution

Add an update planning path that reuses the existing registry, moderation, local fingerprint, version resolution, and semver checks, prints clear would-update/up-to-date/skipped output for single-skill and `--all` modes, and exits before any directory, download, extraction, origin metadata, or lockfile write occurs. Document the flag and cover no-mutation behavior in focused CLI tests.
