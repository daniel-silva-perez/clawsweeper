## Summary

Current main still does not implement `clawhub update --dry-run`, and this is an issue, so the ClawHub cleanup policy does not allow auto-closing it. The update command only registers `--all`, `--version`, and `--force`; `cmdUpdate` has no dry-run option or planning branch and still performs filesystem and lockfile writes on the update path.

## Best Possible Solution

Implement this in the ClawHub CLI core: add `--dry-run` to `clawhub update`, share a planning path for single-skill and `--all` flows, print would-update/up-to-date/skipped/local-change states, guarantee no directory, origin metadata, or lockfile writes during dry-run, add focused no-write tests, and document the flag with the existing update options.
