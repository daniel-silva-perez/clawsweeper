## Summary

Keep open. Current main still lacks a dry-run mode on the CLI update command; the implementation continues into mutating update work and the docs/tests only cover the existing behavior. The request is narrow and valid, and repository policy keeps issues open rather than auto-closing them.

## What This Changes

The issue requests a `clawhub update --dry-run` mode that previews available skill updates without changing installed files, origin metadata, or lockfile state.

## Best Possible Solution

Add a planning path for update that reuses the existing registry, moderation, local fingerprint, version resolution, and semver checks, prints clear would-update/up-to-date/skipped output for single-skill and `--all` modes, and exits before any directory, download, origin metadata, or lockfile write occurs. Document the flag and cover the no-mutation behavior in focused CLI tests.

## Repair Work Prompt

Implement `--dry-run` for `clawhub update` in the ClawHub CLI. Register the Commander flag in `packages/clawhub/src/cli.ts` and extend `cmdUpdate` in `packages/clawhub/src/cli/commands/skills.ts`. The dry-run path should compute the same targets as normal update for both a single slug and `--all`, reuse the existing registry, moderation, local fingerprint, resolve, and semver checks, print clear preview output for would-update, already-up-to-date, skipped, suspicious/blocked, and local-change cases, and return before any mutating work. In dry-run mode do not call `rm`, `downloadZip`, `extractZipToDir`, `writeSkillOrigin`, or `writeLockfile`, including the current fingerprint-based lockfile reconciliation. Add focused tests in `packages/clawhub/src/cli/commands/skills.test.ts` for single-skill and `--all` dry-run behavior and update `docs/cli.md`. Do not change unrelated package publish or sync dry-run behavior. Documentation expectation: update CLI docs; no broad changelog change unless maintainers request one.
