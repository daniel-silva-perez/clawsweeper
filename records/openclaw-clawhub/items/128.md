## Summary

Current main still does not implement `clawhub update --dry-run`. The update command exposes `--all`, `--version`, and `--force` only, while `cmdUpdate` still performs filesystem and lockfile mutations when an update is needed. The prior ClawSweeper comment reached the same conclusion, and the newer checkout shows the gap remains actionable CLI work. Under the ClawHub repository policy, issues are kept open rather than auto-closed.

## Best Possible Solution

Keep this issue open and implement the feature in the ClawHub CLI core: add `--dry-run` to `clawhub update`, extend `cmdUpdate` with a no-write plan for single-skill and `--all` flows, report would-update/up-to-date/skipped states clearly, add tests proving no filesystem or lockfile mutation occurs, and document the flag in `docs/cli.md` plus `packages/clawhub/README.md`.
