## Summary

Keep this issue open. Current main still does not implement `clawhub update --dry-run`; the update command only exposes `--all`, `--version`, and `--force`, and the implementation still goes through the mutating update path. Under the ClawHub cleanup policy, issues stay open unless maintainers decide otherwise, so this remains actionable CLI core work.

## Best Possible Solution

Implement the feature in the ClawHub CLI core: register `--dry-run` on `clawhub update`, add a planning path shared by single-skill and `--all` flows, report would-update/up-to-date/skipped/local-change states, guarantee no filesystem or lockfile writes during dry-run, add focused tests for the no-write behavior, and document the flag with the existing update options.
