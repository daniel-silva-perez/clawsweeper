## Summary

Current main still does not implement `clawhub update --dry-run`. The update command exposes only `--all`, `--version`, and `--force`, and `cmdUpdate` still performs filesystem and lockfile mutations when an update is needed. Existing dry-run support is limited to package publishing and sync. Under the ClawHub repository policy, issues are kept open unless they meet a close-only exception, so this remains actionable CLI work.

## Best Possible Solution

Keep this issue open and implement the feature in the ClawHub CLI core: add `--dry-run` to the update command, extend `cmdUpdate` with a no-write plan for single-skill and `--all` flows, report would-update/up-to-date/skipped states clearly, add tests proving no filesystem or lockfile mutation occurs, and document the flag in `docs/cli.md` and `packages/clawhub/README.md`.
