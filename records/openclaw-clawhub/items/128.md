## Summary

Keep open. Current main still lacks a dry-run mode for the CLI update command, and the existing dry-run support applies to package publish and sync rather than skill updates. The request is narrow, valid, and not superseded by the related items in the provided context.

## What This Changes

Add a `--dry-run` flag to `clawhub update` that reports pending skill updates without modifying installed files or lock state.

## Best Possible Solution

Add an update planning path that reuses the existing registry, moderation, local fingerprint, version resolution, and semver checks, then prints clear would-update, up-to-date, and skipped output for single-skill and `--all` modes before any directory, download, extraction, origin metadata, or lockfile write occurs.
