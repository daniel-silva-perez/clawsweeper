## Summary

This issue should stay open. Current main still does not implement `clawhub update --dry-run`: the update command has no flag, `cmdUpdate` has no dry-run option or planning-only branch, and the normal update path still writes installed skill files, origin metadata, and the lockfile. Under the ClawHub cleanup policy, issues are not auto-closed here.

## Best Possible Solution

Implement this in the ClawHub CLI core: add `--dry-run` to `clawhub update`, share a planning path for single-skill and `--all` flows, print would-update/up-to-date/skipped/local-change states, guarantee no directory, origin metadata, or lockfile writes during dry-run, add focused no-write tests, and document the flag with the existing update options.
