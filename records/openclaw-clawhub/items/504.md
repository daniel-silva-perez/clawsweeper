## Summary

Keep open. Current main still treats user-initiated account deletion as permanent and blocks OAuth reauth for deleted/deactivated users, while the issue discussion and related open reports show multiple affected users asking for recovery, unlinking, or clean re-registration. This is an open ClawHub issue, and the remaining work is security-sensitive identity/account policy rather than conservative cleanup.

## What This Changes

Allow users who deleted their ClawHub account to sign in with GitHub again, restoring, relinking, or recreating account access so web login and CLI token creation work.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery triage item. The desired end state is a maintainer-approved recovery policy implemented coherently across web OAuth, CLI token creation, account deletion, auth account binding, audit/operator flows, user-facing errors, docs, and regression tests.
