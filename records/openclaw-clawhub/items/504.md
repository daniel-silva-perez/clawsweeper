## Summary

Keep open. Current main still deliberately blocks sign-in for deactivated/deleted accounts, the issue has multiple confirmed affected users and related open duplicates, and ClawHub cleanup policy does not auto-close issues. The remaining decision is a security- and privacy-sensitive account recovery policy, not a conservative cleanup close.

## What This Changes

The issue requests that users who deleted a ClawHub account can sign in with GitHub again, either by relinking/recreating the account or otherwise recovering access to CLI tokens and account functions.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery triage item. The desired end state is a maintainer-approved account recovery policy implemented consistently across web OAuth, CLI token creation, account deletion, auth account binding, audit/operator flows, user-facing errors, docs, and regression tests.
