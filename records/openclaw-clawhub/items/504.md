## Summary

Keep this issue open. The repository cleanup policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on current main, and this item is an issue. Current main also still matches the reported deleted-account OAuth failure mode: deletion marks the user permanently deactivated, GitHub OAuth rejects deactivated users, tests assert blocked reauthentication, and docs/UI describe deletion as irreversible rather than recreating or relinking the account.

## Best Possible Solution

Keep this open for maintainer product and support triage. Maintainers should decide whether user-initiated deletion is intentionally irreversible or whether GitHub identities may be safely restored/unlinked for re-registration. If irreversible, web OAuth and CLI login should show a durable deleted-account/support path instead of a confusing loop. If re-registration is intended, add an audited backend recovery path that supersedes or unlinks the old auth identity, preserves bans and ownership protections, revokes old tokens, and covers both web OAuth and CLI login with tests.
