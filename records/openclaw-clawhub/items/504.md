## Summary

Keep this issue open. The ClawHub cleanup policy only allows auto-closing pull requests that are certainly implemented on current main, and this item is an issue. Current main has a clearer auth-error surface than the original sign-in loop report, but it still intentionally treats user-deleted accounts as unrecoverable: deletion writes deactivatedAt/purgedAt, GitHub OAuth rejects deactivated users, tests assert blocked reauthentication, and docs/UI describe deletion as irreversible.

## Best Possible Solution

Keep this open for maintainer product and support triage. Maintainers should decide whether self-deleted GitHub-linked accounts are permanently unrecoverable or whether ClawHub should support an audited restore, unlink, or re-registration flow. If deletion remains irreversible, web OAuth and CLI login should consistently show a durable deleted-account/support path. If recovery is intended, implement a backend/admin path that preserves ban and ownership protections, keeps token revocation safe, records audit history, and covers web OAuth plus CLI login with tests.
