## Summary

Keep this issue open. It is a ClawHub issue, and the repository cleanup policy only permits auto-closing PRs that are certainly implemented on current main. Current main also still matches the reported backend account lifecycle problem: account deletion writes permanent deactivation state, GitHub OAuth rejects deleted/deactivated users, tests assert blocked reauthentication, and docs/UI state deletion is permanent rather than recreating or relinking the account.

## Best Possible Solution

Keep this open for maintainer product and support triage. Maintainers should decide whether user-initiated account deletion is intentionally irreversible or whether GitHub identities may be safely restored/unlinked for re-registration. If irreversible, web OAuth and CLI login should show a durable deleted-account/support path. If re-registration is intended, add an audited backend recovery path that supersedes or unlinks the old auth identity, revokes old tokens, preserves bans and ownership protections, and covers web OAuth plus CLI login with tests.
