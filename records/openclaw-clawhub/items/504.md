## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy says ClawHub issues should remain open unless a strong non-implemented close reason applies. Current main does not implement the reporter’s requested post-deletion GitHub re-link/recreate behavior: deleted accounts are treated as permanently deactivated, OAuth reauth is blocked, and API-token/CLI auth rejects deactivated users. The discussion and related issues show multiple recent reporters with the same concrete account-deletion login loop, so this needs maintainer product/support handling rather than cleanup closure.

## Best Possible Solution

Keep this issue open for maintainer product and support triage. Maintainers should decide whether deleted accounts are truly irreversible or whether GitHub identities may be safely unlinked after deletion. If irreversible, the web and CLI login flows should show a durable deleted-account message and document the support policy. If re-registration should be supported, add a backend path that safely supersedes or unlinks the old Convex Auth identity, preserves ban and ownership protections, revokes old tokens, and covers web OAuth plus CLI login with tests. Given the number of open duplicates, maintainers may also choose a canonical tracking issue and link the others to it after deciding the policy.
