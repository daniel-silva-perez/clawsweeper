## Summary

Keep this issue open. Current main does not implement the requested re-link, restore, or re-registration behavior after self-deletion. Instead, the backend intentionally marks self-deleted users as permanently deactivated, rejects GitHub OAuth completion for those records, and documents deletion as irreversible. The newer code does improve the user-facing auth error, but that only changes the surface from a silent loop toward a deleted-account message; it does not solve the affected users’ account recovery request. Repository cleanup policy for openclaw/clawhub also says issues should stay open unless there is a strong allowed close reason, and this is an issue with multiple related open reports.

## Best Possible Solution

Keep this open for maintainer product, security, and support triage. Maintainers should decide whether self-deleted GitHub-linked accounts are permanently unrecoverable or whether ClawHub should support an audited restore, identity unlink, or re-registration flow. If deletion remains irreversible, web OAuth and CLI login should consistently show a durable deleted-account/support path. If recovery is intended, implement a backend/admin flow that preserves ban and ownership protections, keeps token revocation safe, records audit history, and covers both web OAuth and CLI login with tests.
