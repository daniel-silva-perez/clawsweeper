## Summary

Keep open. Current main still deliberately treats user-initiated deletion as irreversible and rejects GitHub OAuth reauth for deleted/deactivated users, while the issue discussion shows multiple affected users. Repository policy for openclaw/clawhub keeps issues open unless maintainers explicitly resolve them, and the remaining work is an account recovery/security policy decision rather than conservative cleanup.

## What This Changes

The issue requests a way for users who deleted their ClawHub account to sign in again with the same GitHub identity and regain access to account features such as CLI tokens.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery triage item. The desired end state is a maintainer-approved recovery policy implemented consistently across web OAuth, CLI token creation, account deletion, auth account binding, audit/operator flows, user-facing errors, docs, and regression tests.
