Keep open. Current main still intentionally rejects OAuth reauth for deleted/deactivated accounts, the issue has multiple affected-user confirmations and related open reports, and the ClawHub cleanup policy keeps issues open unless maintainers explicitly handle them.

## What This Changes

Deleted ClawHub users want GitHub OAuth to restore, relink, or recreate their account so web login and CLI token creation work again.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery triage item. The desired end state is an explicit maintainer-approved account recovery policy implemented across web OAuth, CLI token creation, authAccounts/providerAccountId handling, deleted/banned state preservation, audit logs, and user-facing docs/tests.
