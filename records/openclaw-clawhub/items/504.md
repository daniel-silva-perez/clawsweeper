## Summary

Keep open. Current main intentionally blocks GitHub OAuth reauth for deactivated/self-deleted users and documents account deletion as irreversible, so it does not implement the requested restore, unlink, or clean re-registration flow. ClawHub issue policy also keeps issues open unless maintainers handle them explicitly, and the affected-user comments plus related open reports make this a live account-recovery triage item.

## What This Changes

The issue requests a recovery path for GitHub OAuth users who deleted their ClawHub account and can no longer sign back in or create CLI tokens.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery triage item. Maintainers should decide whether self-deleted accounts are permanently unrecoverable or whether ClawHub should support an audited restore, GitHub identity unlink, or clean re-registration flow; the implementation should cover web OAuth, CLI login/token creation, revoked token handling, ban preservation, authAccounts/providerAccountId behavior, audit logging, and docs/tests.
