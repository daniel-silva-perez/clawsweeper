## Summary

Keep this issue open. It is a ClawHub issue, and the repository-specific cleanup policy says not to auto-close ClawHub issues. Current main also does not implement the requested post-deletion GitHub re-link or account recreation path: user deletion sets `deactivatedAt`, OAuth blocks deactivated/deleted users, docs describe deletion as irreversible, and CLI/API-token auth rejects deactivated users.

## Best Possible Solution

Keep this open for maintainer product and support triage. Maintainers should decide whether user-initiated deletion is truly irreversible or whether GitHub identities may be safely unlinked/re-registered. If irreversible, the web and CLI flows should show a durable deleted-account message and the public docs should make the support policy clear. If re-registration should be supported, add a backend path that safely supersedes or unlinks the old Convex Auth identity, preserves ban and ownership protections, revokes old tokens, and covers web OAuth plus CLI login with tests.
