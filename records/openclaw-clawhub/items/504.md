## Summary

Keep this issue open. Current main still intentionally blocks GitHub OAuth for self-deleted/deactivated accounts and only improves error surfacing; it does not implement the requested restore, identity unlink, or clean re-registration path. ClawHub cleanup policy also says issues should stay open unless there is a strong allowed close reason, and this issue has multiple affected-user comments plus related open reports.

## Best Possible Solution

Keep this issue open as the canonical maintainer triage item for deleted-account GitHub OAuth recovery. Maintainers should decide whether self-deleted accounts are permanently unrecoverable or whether ClawHub should support an audited restore, GitHub identity unlink, or clean re-registration flow. If deletion remains irreversible, web OAuth and CLI login should consistently show a durable support path; if recovery is intended, implement it in the Convex auth/users/admin path with tests for web OAuth, CLI auth, token revocation, ban preservation, and audit logging.
