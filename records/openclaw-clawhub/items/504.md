## Summary

Keep open. Current main still intentionally blocks GitHub OAuth sign-in for deleted/deactivated ClawHub accounts, and the issue discussion shows multiple affected users with the same post-deletion login loop. This is a ClawHub issue, so cleanup policy keeps it open; the remaining work is account recovery policy rather than a conservative close.

## What This Changes

The issue asks ClawHub to let a user recover, re-link, or cleanly re-register with GitHub after deleting a ClawHub account so web login and CLI token access work again.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery triage item. The desired end state is a maintainer-approved policy implemented consistently across web OAuth, CLI token creation, account deletion, auth account binding or unlinking, operator flows, user-facing errors, docs, and regression tests.
