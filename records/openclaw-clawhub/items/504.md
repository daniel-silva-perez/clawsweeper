## Summary

Keep this issue open. The repository-specific cleanup policy for openclaw/clawhub only permits auto-closing pull requests that are certainly implemented on current main, and this item is an issue. Source review also shows the reported account lifecycle problem remains live: deleting an account writes permanent deactivation state, GitHub OAuth completion rejects deleted/deactivated users, and tests/docs/UI codify deletion as irreversible rather than offering re-registration or restore.

## Best Possible Solution

Keep this open for maintainer product and support triage. Maintainers should decide whether user-initiated deletion is intentionally irreversible or whether GitHub identities may be safely unlinked/re-registered. If irreversible, web OAuth and CLI login should show a durable deleted-account message and a public support policy. If re-registration should be supported, add a backend path that safely supersedes or unlinks the old Convex Auth identity, revokes old tokens, preserves ban and ownership protections, and covers web OAuth plus CLI login with tests.
