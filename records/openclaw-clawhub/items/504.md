## Summary

Keep this issue open. It is a ClawHub issue, and the repository-specific cleanup policy says only pull requests that are certainly implemented on main should be proposed for auto-close. Current main still treats user-initiated account deletion as a deactivation path that blocks future GitHub OAuth sign-in for the same identity, so the reported login loop/re-registration problem remains a live product/support issue rather than implemented cleanup.

## Best Possible Solution

Keep this open for maintainer product and support triage. Maintainers should decide whether user-initiated deletion is truly irreversible or whether GitHub identities may be safely unlinked/re-registered. If deletion is irreversible, the web and CLI login flows should show a durable deleted-account message and public support policy. If re-registration should be supported, add a backend path that safely supersedes or unlinks the old Convex Auth identity, preserves ban and ownership protections, revokes old tokens, and covers web OAuth plus CLI login with tests.
