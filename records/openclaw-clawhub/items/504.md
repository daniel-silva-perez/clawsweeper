## Summary

Keep this issue open. The openclaw/clawhub cleanup policy only permits auto-closing pull requests that are certainly implemented on current main, and this item is an issue. Current main also still matches the reported account lifecycle problem: deletion writes irreversible deactivation state, GitHub OAuth completion rejects deleted/deactivated users, docs/UI say deletion is permanent, and tests assert blocked reauthentication rather than re-registration or restore.

## Best Possible Solution

Keep this open for maintainer product and support triage. Maintainers should decide whether user-initiated account deletion is intentionally irreversible or whether GitHub identities may be safely restored/unlinked for re-registration. If irreversible, web OAuth and CLI login should show a durable deleted-account/support message. If re-registration is intended, add a backend recovery path that supersedes or unlinks the old auth identity, revokes old tokens, preserves bans and ownership protections, and covers web OAuth plus CLI login with tests.
