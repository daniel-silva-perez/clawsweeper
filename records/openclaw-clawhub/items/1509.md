## Summary

Keep this issue open. Current main still has a plausible CLI failure path for `SKILL.md required`: `cmdPublish` only validates files returned by `listTextFiles`, and `listTextFiles` applies ignore rules before reading markdown files. Open draft PR #1856 is already the focused implementation candidate with closing syntax for this issue, so the right next step is maintainer review and merge of that PR or an equivalent fix.

## Best Possible Solution

Keep this issue open while maintainers review and land PR #1856 or an equivalent CLI fix. The fix should preserve a root `SKILL.md` or `SKILLS.md` manifest for `clawhub publish` when broad ignore rules hide markdown files, keep other ignored markdown excluded, verify the Windows/Node 24 report and related #48 behavior, then ship a CLI release. Until then, `https://clawhub.ai/publish-skill` is the useful manual upload workaround.
