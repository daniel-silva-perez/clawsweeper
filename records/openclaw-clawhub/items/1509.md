## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing pull requests that current main certainly implements. Current main still has a plausible `clawhub publish` path where ignore rules can remove the root `SKILL.md` before validation, and draft PR #1856 is the active unmerged implementation candidate.

## Best Possible Solution

Keep this issue open while maintainers review and land PR #1856 or an equivalent CLI fix. The likely code path is to ensure `clawhub publish` always preserves a root `SKILL.md` or `SKILLS.md` manifest even when ignore rules exclude other markdown files, add regression coverage for broad ignore patterns, verify the Windows/Node 24 reproduction and related #48, then release the CLI fix. Until then, `https://clawhub.ai/publish-skill` is the best manual upload workaround.
