## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits automated closure for pull requests that are certainly implemented on current main. Current main still has the CLI-side `SKILL.md required` failure surface: publish validates only the files returned by `listTextFiles`, while file discovery applies ignore rules before the manifest check. Open draft PR #1856 is the active implementation candidate and uses closing syntax for this issue, so the right maintainer path is to review and land or replace that PR before closing the report.

## Best Possible Solution

Keep this issue open while maintainers review and either land PR #1856 or replace it with an equivalent CLI fix. The likely product path is to preserve a root `SKILL.md` or `SKILLS.md` during `clawhub publish` even when ignore rules hide markdown files, add a regression test for that case, and point affected users to `https://clawhub.ai/publish-skill` as the manual upload workaround until a fixed CLI release ships.
