## What This Changes

The issue requests a fix for `clawhub publish` failing with `SKILL.md required` when publishing a new skill even though a root `SKILL.md` exists.

## Best Possible Solution

Review, land, or supersede #1856 with a CLI fix that always includes an existing root `SKILL.md` or `SKILLS.md` manifest during publish while preserving ignore behavior for non-required files, then close this issue after the fix is merged and available.
