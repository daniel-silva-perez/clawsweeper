## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy for this repo keeps issues open. Current main also still has the reported CLI failure path: `clawhub publish` validates only files returned by `listTextFiles`, while ignore rules can exclude the root manifest before that validation. Draft PR #1856 is the active unmerged implementation candidate and uses closing syntax for this issue, so the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open while maintainers review and land PR #1856 or an equivalent CLI fix. The fix should ensure `clawhub publish` preserves a root `SKILL.md` or `SKILLS.md` manifest even when ignore rules exclude other markdown files, keep other ignored markdown excluded, verify the Windows/Node 24 and related #48 paths, then ship a CLI release. Until then, `https://clawhub.ai/publish-skill` is the useful manual upload workaround.
