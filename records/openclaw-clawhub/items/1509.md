## Summary

Keep open. Repository policy for this ClawHub cleanup lane keeps issues open, and current main still has a plausible CLI publish failure path: `cmdPublish` requires the root manifest to appear in `listTextFiles()`, while `listTextFiles()` applies ignore rules before reading files. Open draft PR #1856 is already the focused implementation candidate for preserving a root `SKILL.md` or `SKILLS.md` during publish.

## What This Changes

The issue requests a fix for `clawhub publish` failing with `SKILL.md required` when publishing new skills even though the root skill manifest exists.

## Best Possible Solution

Keep this issue open while maintainers review and land PR #1856 or an equivalent CLI fix. The fix should preserve an existing root `SKILL.md` or `SKILLS.md` manifest for `clawhub publish` when broad ignore rules hide markdown files, keep other ignored markdown excluded, add regression coverage, and ship a CLI release. Until then, `https://clawhub.ai/publish-skill` is the practical manual upload workaround.
