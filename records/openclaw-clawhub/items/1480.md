## Summary

Keep open. Current main still does not implement dashboard skill pagination, and this open PR is the implementation path rather than something to auto-close. The latest patch is narrowed to pagination, but it still needs a publisher-scoping fix in the new paginated query before merge.

## What This Changes

The PR adds active owner/publisher skill indexes, a `skills.listDashboardPaginated` Convex query, `/dashboard` `usePaginatedQuery` Load More UI, and focused dashboard pagination tests.

## Best Possible Solution

Keep the pagination-only direction, but make the personal-publisher paginated path preserve publisher scoping while still handling legacy rows deliberately, then land after CI confirms tests, typecheck, lint, and build.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Scope personal publisher pagination to that publisher:** `convex/skills.ts:2361-2362`
  - body: When `ownerPublisherId` points to the caller's personal publisher, this branch paginates by `ownerUserId` only. Published org skills also carry the same `ownerUserId`, so the personal publisher page can include skills from unrelated publishers, including hidden dashboard-only rows. Preserve the old list path's `!ownerPublisherId || ownerPublisherId === requestedPublisherId` scoping when adding pagination.
  - confidence: 0.86

## Security Review
