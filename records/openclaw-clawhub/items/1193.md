## Best Possible Solution

End state should be that API-token users always resolve to the correct GitHub-backed handle and have an active personal publisher before publish. Maintainers should first verify the reporter’s production account rows, then add a focused backend fix if the state is systemic, with regression coverage for null/wrong handle, missing personal publisher, CLI whoami/login, and web/CLI publish preflight.
