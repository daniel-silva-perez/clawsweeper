## Summary

Keep open. Current main still permits the reported API-token identity path to return a missing handle as `unknown`, and publishing still depends on resolving or creating a personal publisher. The related digest pagination path appears improved, but it does not prove the reporter's wrong/null handle and live publisher state are repaired. ## What This Changes

The issue asks for ClawHub login and API-token identity to resolve to the user's GitHub handle, ensure a personal publisher exists, and allow skill publishing instead of returning `unknown` or `Personal publisher not found`.

## Best Possible Solution

The product should guarantee that every authenticated API-token user has a canonical GitHub handle and an active personal publisher before login, whoami, or publish can appear successful, with an ops repair for existing bad rows and regression coverage for null or wrong-handle users.
