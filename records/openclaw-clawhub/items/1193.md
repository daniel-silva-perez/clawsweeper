## Summary

Keep open. Current main still allows a valid API-token `whoami` response to contain no handle, and publish still has a `Personal publisher not found` guard if personal-publisher resolution returns null. The digest pagination path appears improved by scheduled digest sync, but that does not prove the reporter's live account handle or publisher rows are repaired; ClawHub cleanup policy also keeps issues open for maintainer handling.

## What This Changes

The issue reports that ClawHub CLI login can succeed while `whoami` returns an unknown or empty handle, after which web or CLI publish fails because the account has no personal publisher.

## Best Possible Solution

ClawHub should guarantee that every authenticated API-token user has a canonical GitHub handle and an active personal publisher before login, `whoami`, or publish can appear successful, then repair existing bad production rows and add regression coverage for null or wrong-handle users across CLI and web publish flows.
