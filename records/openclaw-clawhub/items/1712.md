## Summary

Keep open. Current main still exposes package stats but has no package download/install writer, and skill install counters are still driven by authenticated sync telemetry rather than immediate skill-install activity. The prior ClawSweeper comment reached the same keep-open outcome, and ClawHub policy keeps issues open unless they are PRs definitely implemented on main.

## What This Changes

The issue asks for ClawHub package download/install counters and skill install counters to increment from real package fetches and community skill installs instead of remaining zero.

## Best Possible Solution

Define the stats semantics, then add a ClawHub-owned package stats pipeline for archive downloads and any deliberate install signal. For skills, either keep and document sync telemetry as the install source or add explicit install reporting with opt-out and privacy behavior preserved.
