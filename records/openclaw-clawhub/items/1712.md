## Summary

Keep open. Current main still returns package stats but only initializes package counters and bumps `versions`; the package download endpoint does not record `downloads` or `installs`. Skill install counters are implemented through authenticated `clawhub sync` telemetry, while `clawhub install` only downloads/extracts the skill and writes local origin/lockfile data, so the issue’s immediate-install expectation remains a product/privacy decision rather than implemented behavior. ClawHub review policy also keeps issues open and only auto-closes PRs that are certainly implemented on main.

## What This Changes

Package download/install stats and skill install counters should increment from real package fetch/install and community skill install activity instead of remaining zero.

## Best Possible Solution

Keep this issue open and split the follow-up into two maintainer decisions: add a package stats pipeline for ClawHub package archive downloads and any ClawHub-controlled package install signal, then decide whether skill install counters should remain sync-time telemetry or also count immediate CLI installs, with privacy docs and focused tests updated either way.

## Risks / Open Questions

- The package install signal may partly live in OpenClaw plugin install flows outside this repository, so maintainers should separate ClawHub-controlled archive downloads from external install telemetry.
