## Summary

Current main still lacks a package download/install stats pipeline, and skill install counters are still based on authenticated `clawhub sync` telemetry rather than immediate CLI install activity. Repository policy for `openclaw/clawhub` also keeps issues open and only auto-closes definitely implemented PRs.

## Best Possible Solution

Keep this issue open. The best path is to split the decision: implement a focused package stats pipeline for package archive downloads and any ClawHub-controlled package install signal, then make an explicit maintainer decision on whether skill installs should remain sync-time telemetry or also be counted at install time with clear docs and privacy behavior.

## Risks / Open Questions

- Package authors and users still see misleading zero package `downloads` and `installs` because the package archive download path succeeds without recording a package stat event.
- Counting skill installs at immediate install time could conflict with the documented opt-in/opt-out `clawhub sync` telemetry model, so that part needs maintainer product and privacy judgment.
- Package install counts may require an OpenClaw-side plugin installer event or API contract outside this repository, since this repo currently exposes a package download endpoint and ClawHub CLI browse/inspect/publish commands, not the `openclaw plugins install` implementation.
