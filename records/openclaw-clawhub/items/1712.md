## Summary

Keep this issue open. Current main still exposes stored package stats in the package detail API, but the package download path does not record package download/install stats, and skill install counters still depend on authenticated `clawhub sync` telemetry rather than immediate `clawhub skill install` activity. The ClawHub cleanup policy also keeps all issues open; only definitely-implemented PRs are auto-close candidates.

## Best Possible Solution

Keep this issue open and implement a focused stats pipeline: add package download/install stat recording with tests around `/api/v1/packages/{name}/download` and the package/plugin install path, then decide whether skill installs should count at `clawhub skill install` time or keep the documented `clawhub sync` telemetry model and make that clearer in API/CLI docs.
