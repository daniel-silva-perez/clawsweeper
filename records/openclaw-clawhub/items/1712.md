## Summary

Keep this issue open. Current main exposes package stats and has skill telemetry/stat-event machinery, but the package download path still returns archives without incrementing package download/install counters, and skill installs are counted only through explicit CLI sync telemetry rather than immediately from install/add commands.

## Best Possible Solution

Keep this issue open and implement a focused stats pipeline: add package download/install stat recording with tests around `/api/v1/packages/{name}/download` and the package/plugin install client path, then decide whether skill installs should report immediately on install/add or whether docs/CLI UX should make the existing `clawhub sync` telemetry requirement explicit.
