## Summary

Keep open. Current main still exposes package stats but only initializes them and bumps package version counts; there is no package download/install writer on the package archive path. Skill installs are counted from authenticated `clawhub sync` telemetry, while the CLI `skill install` path downloads and writes local files without sending an immediate install event. The prior ClawSweeper comment reached the same conclusion, and the ClawHub cleanup policy keeps issues open unless a PR is definitely implemented on main.

## What This Changes

Request to make package download/install/star stats and skill install counters increment from real package fetch/install and skill add activity instead of remaining at zero.

## Best Possible Solution

Define the intended stat semantics first, then add a package stats pipeline for ClawHub-controlled archive downloads and any explicit install signal the platform owns. For skills, either keep and document sync-time telemetry as the install counter source, or add a deliberate immediate-install reporting path that preserves opt-out and privacy behavior.
