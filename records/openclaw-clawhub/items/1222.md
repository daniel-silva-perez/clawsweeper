## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Substantively, current main still contains the publish-time GitHub account-age lookup that throws the reported rate-limit error, and normal skill publishing plus user-authenticated package/plugin publishing still route through it.

## Best Possible Solution

Keep this open, ideally consolidate maintainer work with #676, and fix the publish gate across skill, import, and package/plugin publishing. A robust fix should ensure production GitHub token coverage, avoid repeated uncached GitHub lookups through caching/backfill, respect retry metadata where useful, and improve user-facing errors so operators can distinguish configuration pressure from transient GitHub limits.
