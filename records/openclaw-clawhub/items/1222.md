## Summary

Keep open. The repository cleanup policy for `openclaw/clawhub` keeps issues open, and current `main` still contains the reported publish failure path: normal skill publish calls the GitHub account-age gate, users without cached `githubCreatedAt` trigger a GitHub API lookup, and 403/429 responses map to the exact rate-limit error. The related Publish plugin comment is consistent with manual package publish using the same gate.

## What This Changes

The issue reports that publishing a skill fails with a GitHub API rate-limit error during upload.

## Best Possible Solution

Keep the account-age gate, but remove avoidable publish-time dependence on unauthenticated GitHub API calls: verify production `GITHUB_TOKEN` and 403/429 logs, then persist or backfill GitHub `created_at` for users missing `githubCreatedAt` if maintainers choose a code mitigation.
