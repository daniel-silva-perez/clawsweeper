## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still contains a live path where skill publish can fail with the reported GitHub API rate-limit error when `githubCreatedAt` is missing; the Publish plugin comment is consistent with manual package publish using the same GitHub account-age gate.

## What This Changes

The issue reports skill upload/publish failing with `GitHub API rate limit exceeded` and asks for investigation.

## Best Possible Solution

Keep the account-age gate, configure authenticated GitHub lookups in production, and reduce publish-time API dependence by recording or backfilling `githubCreatedAt` for users missing it; if quota is exhausted, surface a clearer retry/ops path without bypassing the anti-abuse check.
