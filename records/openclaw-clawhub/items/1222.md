## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and current main still contains the reported failure path: skill publish calls the GitHub account-age gate, missing cached `githubCreatedAt` triggers a GitHub API lookup, and 403/429 responses surface the exact rate-limit error. The Publish plugin comment is consistent with package publish using the same gate.

## What This Changes

The issue reports that publishing a skill fails with a GitHub API rate-limit error during upload.

## Best Possible Solution

Keep the account-age gate, verify production `GITHUB_TOKEN` and GitHub 403/429 logs, then remove avoidable publish-time quota dependence by recording or backfilling `githubCreatedAt` for affected users and surfacing a clearer retry/ops path when quota is exhausted.
