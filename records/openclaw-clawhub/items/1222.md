## Summary

Keep open. ClawHub cleanup policy keeps open issues open, and current main still contains the reported failure path: skill publish reaches the GitHub account-age gate, which fetches GitHub `created_at` when `githubCreatedAt` is missing and maps GitHub 403/429 to the exact rate-limit error. The comment about the Publish plugin hitting the same problem is consistent with manual package publish using the same gate.

## What This Changes

The issue reports that publishing a new skill fails with a GitHub API rate-limit error during upload/publish.

## Best Possible Solution

Keep this issue open for maintainer follow-up. First inspect production Convex `GITHUB_TOKEN` configuration and GitHub 403/429 logs around `requireGitHubAccountAge`; then decide whether to add a code mitigation such as persisting GitHub `created_at` during sign-in/profile sync and/or running a controlled backfill for users missing `githubCreatedAt`, while preserving the account-age gate. Use #676 as the closest related publish/import thread if maintainers consolidate.
