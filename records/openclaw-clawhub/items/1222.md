## Summary

Keep open. Current main still contains the reported GitHub API rate-limit failure path for skill publish, and the ClawHub cleanup policy only permits auto-closing PRs, not open issues. The same account-age gate is shared by web publish, API/CLI publish, GitHub import, and manual package/Publish plugin publish, matching the reporter context and the later Publish plugin confirmation.

## Best Possible Solution

Keep this issue open and consolidate it with the broader publish-gate rate-limit work in #676. The best fix should preserve the 14-day anti-abuse gate, avoid repeated uncached GitHub lookups when githubCreatedAt is missing, backfill or capture githubCreatedAt earlier where safe, make production GITHUB_TOKEN coverage observable, and improve errors so deployment misconfiguration and transient GitHub limits are distinguishable across web, CLI/API, GitHub import, and package/Publish plugin publish.
