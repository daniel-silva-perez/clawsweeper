## Summary

Keep open. This is a ClawHub issue, and current main still contains the reported publish-time failure path: skill publish calls the GitHub account-age gate, users missing cached `githubCreatedAt` trigger a GitHub API lookup, and 403/429 responses throw the exact rate-limit message. The Publish plugin comment is consistent with package publish using the same gate.

## What This Changes

Fix publish/upload so skill or package publishing does not fail with `GitHub API rate limit exceeded` during the GitHub account-age lookup.

## Best Possible Solution

Keep the account-age gate, verify production `GITHUB_TOKEN` and GitHub 403/429 logs, then remove avoidable publish-time quota dependence by recording or backfilling `githubCreatedAt` for affected users and surfacing a clearer retry or ops path when quota is exhausted.
