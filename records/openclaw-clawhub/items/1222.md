## Summary

Keep open. Current main still has the reported GitHub API rate-limit failure path on web skill publish, API/CLI publish, GitHub import, and manual package/Publish plugin publish. The issue is also an open ClawHub issue, and this cleanup policy only auto-closes PRs that current main certainly implements.

## Best Possible Solution

Keep this issue open and handle it with the broader GitHub publish-gate rate-limit work, especially #676. The best fix should preserve the 14-day anti-abuse gate while reducing repeated uncached GitHub lookups, caching or backfilling githubCreatedAt earlier where safe, ensuring production GITHUB_TOKEN coverage, and improving user/operator-facing errors so transient GitHub limits and deployment misconfiguration are distinguishable across web, CLI/API, GitHub import, and manual package publish.
