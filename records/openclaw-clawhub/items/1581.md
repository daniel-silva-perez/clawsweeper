Keep open. This is an open ClawHub issue, and the cleanup policy only permits auto-closing pull requests that current main certainly implements. ## What This Changes

The issue reports that production GitHub OAuth redirects to 127.0.0.1 instead of the production Convex callback URL after authorization.

## Best Possible Solution

Restore production GitHub OAuth so clawhub.ai starts GitHub authorization successfully and completes on the production Convex callback without any localhost or 127.0.0.1 target, then add or extend production smoke coverage for HTTP 500 sign-in starts and localhost callback regressions.
