## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy says ClawHub issues should stay open unless maintainers handle them; only PRs that are certainly implemented on main are eligible for auto-close. Substantively, Overlay Market remains a live hosted-skill moderation appeal: the public API still reports the latest version as suspicious for `suspicious.potential_exfiltration`, while current main provides staff override tooling but no publisher-side annotation or cache-read exception that would clear this automatically.

## Best Possible Solution

Keep this issue open for maintainer/moderator review of the Overlay Market listing. If the appeal is accepted, use the existing manual override flow with an audit note. If maintainers want publisher-side annotations, add a narrow scanner enhancement for benign cache-directory reads, with regression tests that preserve detection of real file-content exfiltration. Separately, guide the publisher to declare env vars in the documented `metadata.openclaw.requires.env` shape so env/RPC usage is machine-readable.
