## Summary

Keep open. This is an issue in openclaw/clawhub, and the review policy only permits auto-closing PRs as implemented for this repository. The technical evidence also does not support an implemented-on-main close: current main intentionally blocks deleted/deactivated accounts from signing in again, while this issue is about affected users being unable to re-authenticate after soft deletion.

## Best Possible Solution

Keep this issue open for maintainer product and support triage. The best path is to decide the intended behavior for self-deleted accounts: either keep deletion irreversible and make the UI/docs/support path unmistakable for affected users, or implement a deliberate reactivation flow that safely matches the GitHub identity/provider account and includes negative tests for wrong-identity reactivation.
