## Summary

Keep this issue open. The repository policy for openclaw/clawhub says issues should not be auto-closed, and current main still intentionally blocks sign-in for self-deleted accounts instead of restoring them. The remaining decision is product/support: either make permanent deletion and blocked login unmistakable, or implement a safe restoration path for the same GitHub identity.

## Best Possible Solution

Keep this open for maintainer triage. Maintainers should decide whether self-deleted accounts are permanently non-restorable; then either improve deletion/login/support messaging around that policy, or implement a safe account restoration flow keyed to the same GitHub identity with regression tests for wrong-identity restoration.
