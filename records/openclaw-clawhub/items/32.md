## Summary

Keep this issue open. Current main no longer restores self-deleted accounts; it intentionally blocks deleted/deactivated users on OAuth sign-in, migrates legacy deletedAt users into irreversible deactivation state, and documents account deletion as permanent. Because this is an open ClawHub issue and the remaining choice is product/security policy rather than a narrow confirmed implementation gap, it needs maintainer triage instead of automated closure.

## Best Possible Solution

Keep this issue open for explicit maintainer triage. Maintainers should decide whether deleted accounts are intentionally non-restorable; then either close manually with clear production support/messaging guidance, or implement a scoped same-provider reactivation flow with regression tests for self-deleted, deactivated, banned, legacy deletedAt, and mismatched existingUserId cases.
