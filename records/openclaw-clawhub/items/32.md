## Summary

Kept open. Current main intentionally treats self-deleted/deactivated accounts as non-restorable: OAuth sign-in is blocked with a permanent-deletion message, deletion clears account/profile state, and docs/UI describe deletion as permanent. That means the original user problem is not implemented as a restoration flow; it has become a maintainer product/security decision. Repository policy for openclaw/clawhub also says this workflow should keep issues open and reserve auto-close for PRs that are definitely implemented on main.

## Best Possible Solution

Keep this issue open for explicit maintainer triage. Maintainers should decide whether deleted accounts are intentionally non-restorable; then either close manually after verifying production messaging/support guidance, or implement a scoped restoration/reactivation flow keyed to the same provider identity with regression tests for self-deleted, deactivated, banned, and legacy deletedAt users.
