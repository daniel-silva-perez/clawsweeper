## Summary

Keep this issue open. Repository policy says ClawHub issues should not be auto-closed, and current main still treats self-deleted accounts as irreversible rather than restoring the same GitHub identity on re-authentication.

## Best Possible Solution

Keep this open for maintainer triage. Maintainers should decide whether self-deleted accounts are permanently non-restorable; then either improve deletion/login/support messaging around that policy, or implement a safe restoration flow keyed to the same GitHub identity with regression tests for wrong-identity restoration.
