## Summary

Keep this issue open. The provided discussion has multiple confirmations of the login failure after account deletion, and current main now intentionally treats user-initiated deletion as irreversible rather than restoring the same GitHub identity on re-authentication. Because this is an open ClawHub issue, repository policy also says this workflow should not auto-close it; maintainers need to decide whether the irreversible-deletion policy is correct and whether production login messaging is adequate.

## Best Possible Solution

Keep this issue open for maintainer triage. Maintainers should decide whether self-deleted accounts are intentionally non-restorable; then either close manually after verifying production messaging/support guidance, or implement a carefully scoped restoration flow keyed to the same GitHub identity with banned/deactivated-account regression tests.
