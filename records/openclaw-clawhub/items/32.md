## Summary

Keep this open. Current main now treats user-initiated deletion as permanent and blocks deleted/deactivated accounts during OAuth, while the report and multiple follow-up comments describe users expecting re-authentication after self-deletion. For this ClawHub issue, the remaining choice is account-recovery policy and security handling rather than conservative cleanup closure.

## What This Changes

Users who self-deleted their ClawHub account report that GitHub OAuth completes but returns them to the home page with a sign-in button instead of restoring access or clearly resolving the account state.

## Best Possible Solution

Maintainers should decide whether user-initiated deletion is intentionally non-restorable. If yes, keep the permanent-deletion implementation and improve OAuth, settings, and support messaging; if no, implement a scoped same-provider reactivation or account-recreation flow with regression coverage for deleted, deactivated, banned/autobanned, legacy deletedAt, fresh-login, and mismatched-identity cases.
