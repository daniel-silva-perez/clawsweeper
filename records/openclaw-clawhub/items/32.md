## Summary

Keep open. ClawHub policy says issues should not be auto-closed by this workflow, and current main intentionally treats self-deleted accounts as permanently deleted rather than restoring the same GitHub identity on re-authentication.

## Best Possible Solution

Keep this open for maintainer triage. Maintainers should decide whether self-deleted accounts are intentionally non-restorable; then either close manually after confirming clear production messaging/support guidance, or implement a carefully scoped restoration flow keyed to the same GitHub identity with ban/deactivated-account regression tests.
