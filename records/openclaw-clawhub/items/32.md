## Summary

Keep this ClawHub issue open. Current main intentionally treats user-initiated account deletion as permanent and blocks deleted/deactivated users during OAuth sign-in, while the issue discussion shows multiple real users expected to re-authenticate after soft deletion; resolving that mismatch needs maintainer product/security triage rather than automated closure.

## What This Changes

Users who self-deleted a ClawHub account report that GitHub OAuth succeeds but returns them to the site still signed out, preventing re-authentication and upload with the same GitHub account.

## Best Possible Solution

Keep this issue open for explicit maintainer triage. Maintainers should decide whether user-initiated account deletion is intentionally non-restorable; then either close manually with clear production support and UX guidance, or implement a scoped same-provider reactivation flow with regression coverage for self-deleted, deactivated, banned/autobanned, legacy deletedAt, and mismatched existingUserId cases.
