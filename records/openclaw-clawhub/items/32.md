## Summary

Keep this open. Current main now encodes permanent account deletion and surfaces deleted-account auth errors, but the report and follow-up comments describe users expecting re-authentication after self-deletion; for ClawHub issues, this should stay with maintainers for product/security policy rather than cleanup closure.

## What This Changes

Users who delete their ClawHub account cannot sign in again with the same GitHub account; OAuth returns them to a signed-out state instead of restoring the account or clearly resolving the account state.

## Best Possible Solution

Maintainers should decide whether user-initiated deletion is intentionally non-restorable. If yes, keep the permanent-deletion implementation and improve OAuth/settings/support messaging; if no, implement a scoped same-provider reactivation or account-recreation flow with regression coverage for self-deleted, deactivated, banned/autobanned, legacy deletedAt, fresh-login existingUserId null, and mismatched identity cases.
