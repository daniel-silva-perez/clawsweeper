## Summary

Keep this open. Current main intentionally blocks deleted/deactivated users during OAuth and surfaces a permanent-deletion error, while the issue discussion describes multiple users expecting re-authentication after self-deletion. Because this is a ClawHub issue and the remaining decision is account recovery versus irreversible deletion, it needs maintainer product/security triage rather than cleanup closure.

## What This Changes

Users who deleted their ClawHub account cannot sign back in with the same GitHub OAuth account and need either recovery behavior or clearer deletion-policy handling.

## Best Possible Solution

Maintainers should decide whether user-initiated deletion is intentionally non-restorable. If yes, keep the current permanent-deletion behavior and improve OAuth/settings/support messaging; if no, implement a scoped same-provider reactivation or account-recreation flow with regression coverage across deleted, banned, legacy, fresh-login, and mismatched-identity cases.
