## Summary

Keep this issue open. The repository policy for openclaw/clawhub does not allow auto-closing issues, and the current main branch still intentionally blocks re-authentication for user-deleted accounts rather than restoring them. The remaining work is a maintainer product/support decision: make permanent deletion and blocked login unmistakable, or implement a safe restoration path.

## Best Possible Solution

Keep this open for maintainer triage. Maintainers should decide whether self-deleted accounts are permanently non-restorable; then either improve deletion/login/support messaging around that policy, or implement a safe account restoration flow keyed to the same GitHub identity with regression tests for wrong-identity restoration.
