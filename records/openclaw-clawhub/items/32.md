## Summary

Keep this open. Current main intentionally blocks deleted/deactivated users during OAuth and surfaces a deleted-account error, while the report and follow-up comments describe users expecting to re-authenticate after self-deletion. Because this is a ClawHub issue and the remaining choice is account recovery versus irreversible deletion, it needs maintainer product/security triage rather than cleanup closure.

## What This Changes

Users who soft-delete their ClawHub account should be able to sign back in with the same GitHub account instead of returning to the homepage with the sign-in button.

## Best Possible Solution

Maintainers should decide whether user-initiated deletion is intentionally non-restorable. If yes, keep the permanent-deletion implementation and improve OAuth/settings/support messaging; if no, implement a scoped same-provider reactivation or account-recreation flow with regression coverage for self-deleted, deactivated, banned/autobanned, legacy deletedAt, fresh-login existingUserId null, and mismatched existingUserId cases.
