## Summary

Keep this open. Current main intentionally blocks deleted/deactivated users during OAuth and now surfaces deleted-account errors, but the issue discussion asks for re-authentication after self-deletion and includes several affected users. Because this is a ClawHub issue and the remaining decision is account recovery versus irreversible deletion, it should stay with maintainers for product/security triage rather than cleanup closure.

## What This Changes

Users who soft-deleted their ClawHub account report that OAuth completes but returns them to the signed-out homepage and prevents re-authentication with the same GitHub account.

## Best Possible Solution

Maintainers should decide whether user-initiated deletion is intentionally non-restorable. If yes, keep the permanent-deletion implementation and improve OAuth/settings/support messaging; if no, implement a scoped same-provider reactivation or account-recreation flow with regression coverage for self-deleted, deactivated, banned/autobanned, legacy deletedAt, fresh-login existingUserId null, and mismatched existingUserId cases.
