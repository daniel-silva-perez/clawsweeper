## Summary

Keep this open. Current main no longer restores self-deleted users; it intentionally blocks deleted/deactivated accounts during OAuth, while the discussion shows several users expected re-authentication after soft deletion. Because this is an open issue in ClawHub and the remaining choice affects account recovery, bans, and deleted-user data boundaries, it needs maintainer product/security triage rather than cleanup closure.

## What This Changes

Users who soft-deleted their ClawHub account cannot sign back in with the same GitHub account after completing OAuth.

## Best Possible Solution

Maintainers should decide whether user-initiated deletion is intentionally non-restorable. If yes, keep the permanent-deletion implementation and improve the OAuth/settings/support messaging; if no, implement a scoped same-provider reactivation or account-recreation flow with regression coverage for self-deleted, deactivated, banned/autobanned, legacy deletedAt, fresh-login existingUserId null, and mismatched existingUserId cases.
