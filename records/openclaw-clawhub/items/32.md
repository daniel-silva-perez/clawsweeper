## Summary

Keep this issue open. ClawHub repository policy says issues should not be auto-closed, and current main still intentionally blocks sign-in for self-deleted accounts rather than restoring them. The remaining maintainer decision is whether permanent deletion is the intended product policy, or whether ClawHub should add a safe restoration path for the same GitHub identity.

## Best Possible Solution

Keep this open for maintainer triage. Maintainers should decide whether self-deleted accounts are permanently non-restorable; then either improve deletion/login/support messaging around that policy, or implement a safe restoration flow keyed to the same GitHub identity with regression tests for wrong-identity restoration.
