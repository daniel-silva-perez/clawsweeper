## Summary

Keep this issue open. The repository policy for openclaw/clawhub only permits auto-closing PRs that are certainly implemented on current main, and this item is an issue. The technical review also shows current main intentionally blocks re-authentication after user-initiated deletion, so the reported soft-deleted-account login problem is not implemented as a restoration flow.

## Best Possible Solution

Keep this issue open for maintainer product and support triage. Maintainers should decide whether self-deleted accounts are intentionally non-restorable, then either make the deletion/login UX and support path unmistakable or implement a deliberate restoration flow that safely matches the GitHub identity/provider account and adds regression tests.
