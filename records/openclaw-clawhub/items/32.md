## Summary

Keep this issue open. Current main intentionally treats user-initiated account deletion as irreversible and blocks deleted/deactivated users during OAuth sign-in, while the issue discussion shows multiple affected users expected to re-authenticate after soft deletion; that mismatch needs maintainer product/security triage rather than cleanup closure.

## What This Changes

The issue reports that users who soft-delete their ClawHub account cannot sign back in with the same GitHub account and are returned to the homepage with a sign-in button.

## Best Possible Solution

Keep this open for explicit maintainer triage. Maintainers should decide whether user-initiated deletion is intentionally non-restorable; then either close manually with clear support/UX guidance, or implement a scoped same-provider reactivation or account-recreation flow with regression coverage for self-deleted, deactivated, banned/autobanned, legacy deletedAt, fresh-login existingUserId null, and mismatched existingUserId cases.
