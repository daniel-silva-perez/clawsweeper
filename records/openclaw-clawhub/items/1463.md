## Summary

Keep open. ClawHub cleanup policy keeps open issues out of auto-close scope, and the substantive appeal is still unresolved: the live latest `email-suite-imap-smtp` scan is still suspicious on version 1.2.6, with static env/file-read findings and an LLM concern that the registry metadata does not declare the IMAP/SMTP credentials. Current main has metadata-backed env-var suppression plus rescan/manual override paths, but it does not prove this production moderation appeal is resolved.

## What This Changes

The issue asks ClawHub to stop treating declared IMAP/SMTP credential environment variables sent to user-configured mail servers as suspicious exfiltration.

## Best Possible Solution

Resolve this as a maintainer moderation decision: review the latest `email-suite-imap-smtp` artifact together with #1519, require canonical `metadata.openclaw.requires.env` and `primaryEnv` declarations for IMAP/SMTP credentials if they are missing, then rescan or apply a documented clean manual override if the bundle is safe. Only add a focused scanner improvement if a metadata-backed legitimate mail-client flow still produces the static false positive.
