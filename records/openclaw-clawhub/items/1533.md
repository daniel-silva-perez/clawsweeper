## Summary

Keep open. Current main has not implemented the shared helper, and the proposed refactor still needs maintainer judgment because it changes `skills.ts` and `souls.ts` non-integer limit handling from rounding to truncation. The security review did not find a supply-chain concern in the PR surface.

## What This Changes

The PR adds a shared Convex `clampInt` helper in `convex/lib/math.ts` with tests and replaces 11 local helpers across Convex modules with imports from it.

## Best Possible Solution

Keep the deduplication direction, but land it only after preserving existing call-site semantics or explicitly standardizing the integer coercion contract with focused tests for public pagination and maintenance batch-size behavior.
