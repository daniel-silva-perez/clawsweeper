## Summary

Keep open. Repository policy keeps ClawHub issues open, and current main still has a real user-visible gap: backend moderation can suppress uncorroborated VirusTotal Code Insight suspicious results, but the skill UI and v1 scan/security snapshots still derive VirusTotal display status from raw stored `vtAnalysis.status` or `verdict`.

## What This Changes

The issue requests that VirusTotal 0-detection Code Insight results display as clean/undetected instead of suspicious and stop showing the skill warning banner.

## Best Possible Solution

Keep the issue open and implement a central VirusTotal display/status normalization path shared by skill UI, scanner detail pages, and v1 security snapshots. Code Insight-only suspicious results with zero malicious and zero suspicious AV engine counts should display as clean/undetected when ClawScan and static analysis are clean, while preserving Code Insight text as context and preserving warnings for any corroborating AV-engine, static-scan, or ClawScan suspicious/malicious signal.
