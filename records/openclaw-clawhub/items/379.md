## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still leaves the reported false-positive class partly unresolved: backend moderation can suppress an uncorroborated VirusTotal Code Insight suspicious result, but the skill UI, scanner detail page, and v1 security snapshot still derive visible VirusTotal status from the stored raw VT status/verdict.

## What This Changes

The issue requests that skills with zero VirusTotal AV detections and clean OpenClaw analysis show a clean or undetected VirusTotal badge instead of a suspicious badge/banner driven by Code Insight text alone.

## Best Possible Solution

Add one shared VirusTotal display/status normalization path used by the skill UI, scanner detail pages, and v1 security snapshots. It should treat Code Insight-only suspicious results with zero malicious and zero suspicious AV engine counts as clean or undetected only when local scanners are clean, while keeping Code Insight text visible as context and preserving stronger warnings when any scanner corroborates risk.

## Repair Work Prompt

Implement a focused fix for VirusTotal Code Insight-only false positives on ClawHub skill security display/API paths. Current backend moderation already suppresses VT suspicious reason codes when `vtAnalysis.scanner === 'code_insight'`, AV engine `malicious` and `suspicious` counts are both zero, ClawScan is clean/benign, and static scan is clean or absent. Reuse that rule, or extract an equivalent shared helper, so skill UI badges, detail security summaries, scanner detail pages, and v1 skill security snapshots normalize the displayed VT status to clean/undetected for that narrow case.

Expected boundary: update skill security display/API normalization only; do not change VirusTotal fetching, rescan scheduling, manual moderation behavior, package/plugin behavior unless the same shared display helper already covers it safely, or unrelated UI styling. Keep Code Insight text visible as context on the scanner detail page, but do not show the warning badge/banner solely because of Code Insight text when engine counts and local scanners are clean. Preserve suspicious/malicious output whenever AV engine counts are positive, engine stats are missing, static scan is suspicious/malicious, ClawScan is suspicious/malicious/error, malware/manual moderation signals exist, or another scanner corroborates risk.

Add regression coverage for the UI badge/panel, detail security summary, scanner page if practical, and v1 security snapshot. No changelog entry is expected for this narrow bug fix unless maintainers request one.
