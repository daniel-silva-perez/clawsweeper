## Summary

Keep open. This is an issue in openclaw/clawhub, and the repository policy for this review only permits auto-closing PRs that are definitely implemented on main. Substantively, current main still has skill VirusTotal paths that use Code Insight verdicts as the stored VT status and moderation input before considering AV engine stats, so the reported false-positive behavior is not clearly fixed.

## Best Possible Solution

Keep this issue open for maintainer work. The likely fix belongs in the skill VirusTotal scan/status pipeline: derive the user-facing VirusTotal badge and VT moderation contribution from AV engine malicious/suspicious counts when they are zero, surface Code Insight as explanatory context instead of a detection verdict, add a regression test for 'Code Insight suspicious plus 0 engine detections', and backfill/re-sync already-flagged skills if the stored status is stale.
