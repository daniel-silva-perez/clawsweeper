## Summary

Keep open. Repository policy keeps ClawHub issues open, and current main still has a real display/API gap: backend moderation suppresses one uncorroborated VirusTotal Code Insight false-positive case, but skill UI and v1 security snapshots still derive VirusTotal status from raw stored `vtAnalysis.status` or `verdict`.

## What This Changes

The issue asks ClawHub to show VirusTotal clean/undetected, and remove the suspicious warning banner, when AV engine detections are 0 and OpenClaw scanning is benign instead of treating Code Insight-only text as suspicious.

## Best Possible Solution

Add one shared VirusTotal display/status normalization path for skill UI, scanner detail pages, and v1 security snapshots. It should treat Code Insight-only suspicious results with zero malicious and zero suspicious AV engine counts as clean/undetected only when ClawScan and static analysis are clean, while keeping Code Insight text visible as context and preserving stronger warnings whenever another scanner corroborates risk.

## Repair Work Prompt

Implement a central VirusTotal display/status normalization path for the ClawHub skill security surfaces. A VirusTotal Code Insight `suspicious` result should display/report as clean or undetected only when the stored VT analysis indicates Code Insight, AV engine stats have zero `malicious` and zero `suspicious`, ClawScan/LLM is clean or benign, and static analysis is clean or absent. Preserve the raw Code Insight analysis/source as context. Do not downgrade any case with nonzero AV malicious/suspicious counts, static suspicious/malicious findings, ClawScan suspicious/malicious, malicious VT status, or manual moderation signals. Share the helper across `SecurityScanResults`, `DetailSecuritySummary`, `SecurityScannerPage`, and `buildSkillSecuritySnapshot` rather than duplicating ad hoc checks. Add regression tests for the UI badge/detail behavior and v1 API security snapshot; keep the existing backend moderation tests passing. No unrelated styling, routing, scan policy, or persistence changes.
