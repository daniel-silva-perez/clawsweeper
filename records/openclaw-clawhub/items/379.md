## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Current main contains backend mitigation for uncorroborated VirusTotal Code Insight suspicion, but the user-visible VirusTotal badge/API status can still be driven by stored `vtAnalysis.status`, so the reported badge behavior is not clearly resolved end to end.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The best product fix is to derive the VirusTotal badge and public security snapshot primarily from AV engine malicious/suspicious counts when Code Insight is the only suspicious signal, present Code Insight as separate context, and reconcile or rescan affected stored analyses so existing clean-engine skills no longer show a misleading suspicious badge or warning banner.
