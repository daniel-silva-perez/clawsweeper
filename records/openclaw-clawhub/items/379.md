## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Current main has a backend moderation mitigation for uncorroborated VirusTotal Code Insight false positives, but the user-visible scan badge/API path can still render from stored Code Insight-derived VT status, so the reported behavior is not clearly resolved end to end.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The best product fix is to derive the user-facing VirusTotal badge and public security snapshot from AV malicious/suspicious engine counts when Code Insight is the only suspicious signal, show Code Insight as separate context, and reconcile or rescan existing affected analyses so clean-engine skills no longer show a misleading suspicious badge or warning banner.
