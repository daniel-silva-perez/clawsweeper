## Summary

Keep open. This is an issue, and the ClawHub cleanup policy only permits auto-closing PRs that are certainly implemented on main; current main also still has a real display/API gap where backend moderation can suppress uncorroborated VirusTotal Code Insight suspicious results, but visible security surfaces still read the raw stored VirusTotal status or verdict.

## What This Changes

The issue requests that VirusTotal scans with zero malicious or suspicious engine detections display as clean or undetected instead of showing a suspicious badge and warning banner.

## Best Possible Solution

Add a shared VirusTotal display/status normalization helper used by skill scan badges, scanner detail pages, and v1 security snapshots. It should reuse the backend suppression semantics for Code Insight-only suspicious results with zero malicious and zero suspicious engine counts, keep Code Insight text visible as context, and preserve stronger warnings whenever any scanner corroborates risk.
