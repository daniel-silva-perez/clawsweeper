## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that current main definitely implements. Substantively, current main has backend moderation suppression for uncorroborated VirusTotal Code Insight suspicious results with clean AV engine counts, but the user-facing badge, scanner detail pages, and public scan API can still display or report VirusTotal as suspicious from the stored Code Insight status/verdict rather than clean engine counts.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The best fix is to make the user-facing VirusTotal badge, scanner detail pages, and public scan API derive the visible VT status from AV malicious/suspicious engine counts when Code Insight is the only suspicious signal, keep Code Insight analysis as separate explanatory context, and reconcile existing affected analyses so clean-engine skills no longer show a misleading suspicious badge or warning banner.
