## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing ClawHub pull requests that current main definitely implements. Current main has backend moderation mitigation for uncorroborated VirusTotal Code Insight false positives, but the displayed VirusTotal badge and public scan API can still derive a suspicious status from stored Code Insight status/verdict rather than clean AV engine counts.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The best fix is to make the user-facing VirusTotal badge and public scan API derive the visible VT status from AV malicious/suspicious engine counts when Code Insight is the only suspicious signal, keep Code Insight text as separate context, and reconcile existing affected analyses so clean-engine skills no longer display a misleading suspicious badge or warning banner.
