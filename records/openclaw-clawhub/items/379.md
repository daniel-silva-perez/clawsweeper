## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Current main includes a backend moderation mitigation for uncorroborated VirusTotal Code Insight false positives, but the user-visible VirusTotal badge and public API still derive status from stored Code Insight VT status/verdict rather than clean engine counts, so the reported badge behavior is not fully resolved.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The best fix is to derive the user-facing VirusTotal badge and public API security snapshot from AV malicious/suspicious engine counts when Code Insight is the only suspicious signal, show Code Insight as separate context, and reconcile or rescan existing affected analyses so clean-engine skills no longer display a misleading suspicious VT badge or warning banner.
