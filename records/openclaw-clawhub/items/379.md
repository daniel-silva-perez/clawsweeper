## Summary

Keep open. Repository policy for openclaw/clawhub only permits auto-closing PRs that are certainly implemented on main, and this item is an issue. Substantively, current main has a recent backend moderation fix for uncorroborated VirusTotal Code Insight suspicious results with clean AV engine stats, but the exact user-visible VT badge request is not fully resolved: the UI still renders `vtAnalysis.status === "suspicious"` as a Suspicious VirusTotal badge.

## Best Possible Solution

Keep this issue open. The best product fix is to separate VirusTotal engine detection status from Code Insight commentary in the stored model and UI: show the VT badge from AV engine malicious/suspicious counts, keep Code Insight as explanatory context, retain backend suppression for uncorroborated Code Insight suspicious results, and run a cursor-based reconciliation/rescan for already-flagged skills.
