## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has an unresolved display/API gap: moderation can now suppress the uncorroborated VirusTotal Code Insight false-positive class, but visible skill/security surfaces still derive VirusTotal status from the raw stored VT status or verdict.

## What This Changes

The issue asks ClawHub to show a clean or undetected VirusTotal badge, and no suspicious warning banner, when VirusTotal has zero malicious or suspicious engine detections.

## Best Possible Solution

Add a shared VirusTotal display/status normalization helper used by the skill UI, scanner detail pages, and v1 security snapshots. It should reuse the current backend suppression semantics for Code Insight-only suspicious results with zero malicious and zero suspicious engine counts, keep Code Insight text visible as context, and leave stronger warnings intact whenever another scanner corroborates risk.

## Repair Work Prompt

Fix the VirusTotal Code Insight false-positive display/API gap for the current issue and related reports (#438, #442, #443, #447, #448, #450, #454, #455, #459, #1314). Add one shared normalization path for VirusTotal display/security status that treats Code Insight-only suspicious results with `engineStats.malicious === 0` and `engineStats.suspicious === 0` as clean/undetected only when ClawScan is clean/benign and static scan is clean or absent. Reuse the backend semantics from `convex/lib/moderationEngine.ts` rather than inventing a broader moderation policy. Apply it to `src/components/SkillSecurityScanResults.tsx`, `src/components/DetailSecuritySummary.tsx`, `src/components/SecurityScannerPage.tsx`, and `convex/httpApiV1/skillsV1.ts`. Preserve warnings for malicious VT results, any nonzero malicious/suspicious AV engine count, non-clean ClawScan, non-clean static scan, manual moderation flags, and other corroborating signals. Keep Code Insight analysis text visible as context, but do not let the suppressed status drive a suspicious badge/security snapshot. Add focused regression coverage in the existing component and HTTP API tests, and keep the existing moderation engine tests passing. No changelog entry is required unless maintainers prefer one for this user-visible false-positive fix; do not change VirusTotal upload/polling behavior or rescan quotas.
