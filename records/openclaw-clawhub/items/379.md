## Summary

Keep open. This is a ClawHub issue, and the repo-specific cleanup policy only permits auto-closing PRs that are definitely implemented on main. Current main has backend mitigation for uncorroborated VirusTotal Code Insight suspicious results, but the user-visible VirusTotal badge still renders directly from stored `vtAnalysis.status`, so the core badge/display complaint is not clearly resolved.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The best fix is to make the VirusTotal badge and warning banner derive their primary verdict from AV engine malicious/suspicious counts, show Code Insight text as separate context, and backfill or rescan existing affected skill rows so stale suspicious state clears.
