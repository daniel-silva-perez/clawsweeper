## Summary

Keep open. Current main only partially addresses the false-positive class: backend moderation can suppress uncorroborated VirusTotal Code Insight suspicious results, but the visible skill/security UI and v1 API security snapshot still derive VirusTotal status directly from stored `vtAnalysis.status` or `verdict`. Repository policy also keeps ClawHub issues open unless they are PRs already implemented on main.

## What This Changes

The issue requests that VirusTotal zero-detection Code Insight false positives render as clean or undetected instead of Suspicious and stop showing the suspicious warning banner.

## Best Possible Solution

Add a shared VirusTotal display/status normalization path for skill UI, scanner detail pages, and v1 security snapshots. It should reuse the current backend suppression semantics for Code Insight-only suspicious results with zero malicious and zero suspicious engine counts, keep Code Insight text visible as context, and preserve stronger warnings whenever another scanner corroborates risk.

## Repair Work Prompt

Fix the VirusTotal Code Insight false-positive display/API gap for this issue and the related clean-VT false-positive cluster. Current backend moderation already suppresses uncorroborated `code_insight` suspicious results when ClawScan is clean/benign, static scan is clean or absent, and `vtAnalysis.engineStats.malicious` and `.suspicious` are both zero. Carry that same rule into the visible VirusTotal status used by `SecurityScanResults`, `DetailSecuritySummary`, `SecurityScannerPage`, and the v1 skill security snapshot. Preserve Code Insight text as context and do not change VirusTotal ingestion. Do not suppress suspicious/malicious display when any AV engine, static scan, ClawScan, malicious verdict, or manual moderation signal corroborates risk. Add regression tests for the zero-engine Code Insight case and for a neighboring corroborated suspicious case. Avoid broad moderation-policy rewrites or unrelated UI changes. Include a short changelog note only if the repo has an established user-visible changelog flow for this surface.
