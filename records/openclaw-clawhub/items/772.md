Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing PRs that are certainly implemented on current main. Substantively, current main still has the same plausible disabled-button path: `/cli/auth` disables the sign-in button while Convex auth is loading, and that loading state comes directly from `useConvexAuth()`.
