## Summary

Keep this issue open. The repository policy for openclaw/clawhub only allows automated closure for pull requests that are certainly implemented on current main, and this item is an issue. A source pass also shows the reported failure mode is still plausible: the CLI opens the hosted `/cli/auth` route, that route disables the GitHub sign-in button while Convex auth is loading, and that loading state depends on the Convex client configured from `VITE_CONVEX_URL`.

## Best Possible Solution

Keep this issue open for maintainer runtime verification. Check the deployed `/cli/auth` flow and production Convex/Vercel configuration; if the WebSocket 307 still occurs, fix the deployment or runtime URL path and decouple the unauthenticated GitHub sign-in action from indefinite Convex auth loading, with a regression test for the stuck-loading pre-OAuth state. Close manually only after a verified production fix ships.
