## Summary

Keep open. Current main still leaves the reported owner/slug ambiguity unresolved in the CLI and public v1 search/install contract, and the repository policy for this sweep says ClawHub issues should remain open. The web detail page has a partial owner-qualified prompt mitigation, but CLI search, CLI install, API schema, docs, and the visible CLI command still use bare slugs.

## Best Possible Solution

Keep this issue open for a maintainer-owned scoped identity design: expose owner identity in the v1 search API/schema, show owner or owner/slug in `clawhub search`, decide whether `clawhub install` should accept owner-qualified targets while preserving bare-slug compatibility, resolve owner-qualified input without unsafe nested filesystem writes, and align the skill detail command card, docs, and tests with that contract.
