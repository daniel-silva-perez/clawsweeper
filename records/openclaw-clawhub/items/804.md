## Summary

Keep open. Current main still leaves the reported CLI discovery ambiguity in place: public skill search and `clawhub search` omit owner identity, while install accepts only bare slugs. The web prompt surface partially helps on skill detail pages, but it does not solve the reporter's CLI search/install confusion.

## What This Changes

The issue requests clearer owner-qualified skill identity in ClawHub search/install flows, such as owner/slug syntax, website install guidance, or an owner column in `clawhub search`.

## Best Possible Solution

Expose owner identity additively in the public v1 skill search response, update shared and CLI-local schemas, render owner information in `clawhub search`, and document the response/output change. Keep full owner-qualified install syntax as a separate maintainer decision unless explicitly brought into scope.
