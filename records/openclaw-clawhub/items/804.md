## Summary

Keep open. Current main still has the reported owner/slug ambiguity in the CLI and public v1 search contract: search results omit owner identity, and install still accepts only bare slugs. The web skill-detail prompt surface partially mitigates this by building owner-qualified prompt targets, but it does not solve CLI search discovery; ClawHub policy also keeps issues open for maintainer follow-up.

## What This Changes

The issue asks ClawHub to make duplicate skill slugs less ambiguous by showing owner identity in search/install surfaces or supporting owner-qualified install locators.

## Best Possible Solution

Expose owner identity in the public v1 search response and schema, then render that owner information in `clawhub search` so users can distinguish `steipete/weather` from other `weather` results. Treat full owner-qualified `clawhub install owner/slug` support as a separate design unless maintainers explicitly include it.
