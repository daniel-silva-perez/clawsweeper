## Summary

Keep open. Current main still does not resolve the reported owner/slug ambiguity for CLI search or install. The web skill page has a partial owner-qualified prompt path, and internal search results already carry owner identity, but the public v1 search response, CLI schema, CLI search output, docs, and install path still center on bare slugs.

## Best Possible Solution

Keep this issue open and land a narrow owner-disambiguation PR: expose owner identity in the v1 search response, update both CLI schemas, print owner or owner/slug in clawhub search, and refresh CLI/API docs and tests. Treat owner-qualified install locators as a follow-up product/security design unless maintainers explicitly scope it into the same work.
