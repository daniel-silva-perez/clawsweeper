## Summary

Keep this open. The repository policy for this review keeps ClawHub issues open, and current main still preserves the reported owner/slug ambiguity in CLI search, CLI install, the v1 skill search API, download filenames, and docs. The web skill page has a partial owner-aware prompt mitigation, but its visible ClawHub CLI command still uses the bare slug.

## Best Possible Solution

Keep this issue open and implement a scoped owner/slug identity design: expose owner handle or owner id in the v1 skill search API and CLI schema, display owner or owner/slug in clawhub search, decide whether clawhub install should accept owner/slug while preserving slug-only compatibility, and align download filenames, docs, and skill-page commands with that contract.
