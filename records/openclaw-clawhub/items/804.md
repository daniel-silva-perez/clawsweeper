## Summary

This is an open ClawHub issue, and the repository policy for this review only allows auto-close proposals for PRs that are certainly implemented on main. Current main also still appears to preserve the reported ambiguity: CLI search output is slug-only, CLI install normalizes to bare slugs and rejects owner-qualified input, the public search response lacks owner identity, and download filenames are still slug/version based. A prior ClawSweeper comment already captured the same remaining gap, so the item should stay open for maintainer/product follow-up rather than be closed.

## Best Possible Solution

Keep this issue open and implement a scoped owner/slug identity design: expose owner handle or id in the v1 search API and CLI schema, display owner or owner/slug in clawhub search, decide whether clawhub install should accept owner/slug or @owner/slug while preserving slug-only compatibility, and align download filenames, docs, and skill-page commands with that contract.
