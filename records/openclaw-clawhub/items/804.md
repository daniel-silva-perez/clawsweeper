## Summary

Keep open. Current main still leaves the reported CLI discovery ambiguity in place: skill search output and the public v1 search contract omit owner identity, while install still accepts only bare slugs. The web skill-detail prompt path partially mitigates owner-qualified identity, but it does not solve the reporter’s `clawhub search` and `clawhub install` confusion.

## What This Changes

The issue asks for ClawHub CLI and/or web install surfaces to disambiguate skills by owner, such as showing owner information in search results or supporting clearer owner/slug install syntax.

## Best Possible Solution

Expose owner identity additively in the public v1 skill search response, update both shared and CLI-local schemas, render owner information in `clawhub search`, and document the response/output change. Keep full owner-qualified install syntax as a separate maintainer decision unless explicitly brought into scope.
