## Summary

Keep open. Current main still leaves the reported CLI discovery ambiguity unresolved: the public search response and CLI output omit owner identity, while install continues to accept only bare slugs. The web skill-detail prompt path now partly mitigates owner-qualified identity, but it does not address the reporter's `clawhub search` result ambiguity.

## What This Changes

The issue requests clearer owner-qualified skill identity in ClawHub search/install flows so users can distinguish same-or-similar skill slugs by publisher.

## Best Possible Solution

Expose owner identity additively in the public v1 search response, update shared and CLI-local schemas plus API docs, and render owner information in `clawhub search` so users can distinguish entries like `steipete/weather`; leave full owner-qualified install locator support for a separate maintainer decision unless scope is explicitly expanded.
