## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review only allows auto-closing PRs that are already implemented on main. Current main also still only partially addresses the report: the web UI can surface owner context, but CLI search, the public search API, CLI install, docs, and download filenames remain slug-only.

## Best Possible Solution

Keep this issue open and implement a scoped identity clarification: expose ownerHandle or owner in /api/v1/search, update the CLI search schema and output to show owner/slug, decide whether clawhub install should accept owner/slug or another owner-qualified form while preserving slug-only compatibility, and consider owner-qualified download filenames or stronger web copy for the exact canonical skill page.
