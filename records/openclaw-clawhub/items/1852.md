## Summary

Keep open. This is a ClawHub issue, and the repository policy only permits auto-closing ClawHub PRs that are certainly implemented on main. Current main has registry/package trust controls, capability metadata, scans, trusted publishing, provenance fields, and moderation audit logs, but it does not implement the requested PAP-style runtime principal hierarchy, transport authorization, context minimization, signed action receipts, or multi-agent trust boundary.

## What This Changes

Request to integrate PAP as an optional trust layer for skill execution and inter-instance communication, with declared context needs, enforced principal hierarchy, signed/auditable actions, and pre-execution authorization.

## Best Possible Solution

Keep open for maintainer security/product triage. The next useful step is to decide whether PAP belongs in OpenClaw runtime enforcement, ClawHub package metadata, CLI install policy, or a new extension API, then write a small design that defines principals, context requests, authorization receipts, audit storage, and how ClawHub-published skills/plugins declare or consume those boundaries.
