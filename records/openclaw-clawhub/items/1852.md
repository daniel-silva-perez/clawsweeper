## Summary

Keep open. This is a ClawHub issue, and the repository policy only allows auto-closing ClawHub PRs that are certainly implemented on main. Current main has registry/package trust controls, capability metadata, scanning, moderation, provenance, and audit surfaces, but it does not implement the requested PAP-style runtime principal hierarchy, transport authorization, context minimization, signed action receipts, or multi-agent trust boundary.

## What This Changes

The issue requests an optional PAP-style trust layer for skill execution and inter-instance communication, with declared context needs, principal enforcement, signed/auditable actions, and pre-execution authorization.

## Best Possible Solution

Maintainers should keep this open long enough to scope a security/product design: decide which parts belong in OpenClaw runtime enforcement versus ClawHub package metadata and install policy, then define any minimal ClawHub schema/API changes for declaring context needs, trust requirements, or audit receipt compatibility.
