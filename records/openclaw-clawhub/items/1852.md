## Summary

Keep open. Current main has registry/package trust controls, capability metadata, verification/scanning, and administrative audit logs, but not the requested PAP-style runtime authorization boundary for multi-agent or skill execution. The ClawHub cleanup policy also says issues should stay open; only PRs that are certainly implemented on main are eligible for auto-close.

## What This Changes

The issue requests an optional PAP-based trust layer for multi-agent skill execution and inter-instance communication, with context boundaries, authorization, signed actions, and auditability.

## Best Possible Solution

Keep the issue open for security/product scoping. The desired end state should clearly assign the boundary: either ClawHub records and exposes a narrow PAP-adjacent registry/receipt surface, OpenClaw runtime owns enforcement entirely, or the project documents why PAP integration is out of scope.
