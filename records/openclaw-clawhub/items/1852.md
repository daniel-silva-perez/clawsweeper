## Summary

Keep open. This is a ClawHub issue, and the provided repository policy only permits auto-closing ClawHub pull requests that are certainly implemented on main. Substantively, current main has package catalog trust metadata, trusted-publisher/OIDC publishing, scan status, verification, install warnings, and admin/package audit logs, but it does not implement the PAP/principal/transport-layer authorization model requested for multi-agent execution and context boundaries.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to decide whether PAP-style trust belongs in OpenClaw runtime/security architecture, ClawHub package metadata/signing, or a split design, then define the smallest core authorization and audit boundary needed before registry or plugin integration work proceeds.
