## Summary

Keep open. This is a ClawHub issue, and the repository-specific policy only permits auto-closing pull requests that are certainly implemented on current main. The current code has registry/package trust, capability, verification, and trusted-publisher controls, but it does not implement the requested PAP-style execution-time principal, transport authorization, context minimization, delegation, or audit receipt model.

## Best Possible Solution

Keep this issue open for maintainer product/security triage. The best path is to decide whether PAP-style trust belongs in OpenClaw runtime architecture, ClawHub package metadata/signing, or a split design, then define the smallest principal/context/audit boundary before implementation proceeds.
