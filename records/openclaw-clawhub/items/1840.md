## Summary

Keep this PR open. Current main at b3c42b661b5e30c57e7756899ea50920ec9ef294 still lacks the intended REST/CLI support for org-owned skill publishes: the skill publish schema and parser do not preserve ownerHandle, the V1 publish handler calls publishVersionForUser directly, and the skill CLI does not expose --owner. The PR remains the direct implementation candidate for #1814, and its diff is focused on the API/CLI/schema/test surface without unrelated CI, dependency, release, or workflow changes.

## Best Possible Solution

Review and land this PR, or an equivalent patch, so skill publish payloads accept ownerHandle in schema source and generated artifacts, JSON and multipart REST paths preserve it, the REST handler resolves the requested publisher with minimumRole: "publisher", the skill CLI exposes --owner <handle>, and regression tests cover schema parsing, CLI payload generation, JSON API publishing, and multipart API publishing. Keep #1814 open until this PR or an equivalent fix merges.
