## Summary

Keep open. Current main still does not implement this PR’s intended REST/CLI org-owned skill publish support, and the linked #1814 remains open. The PR is still a live implementation candidate rather than obsolete work.

## Best Possible Solution

Review and land this PR, or an equivalent patch, so `/api/v1/skills` accepts optional `ownerHandle`, preserves it through JSON and multipart parsing, resolves the target publisher server-side with `minimumRole: "publisher"`, passes `ownerPublisherId` into `publishVersionForUser`, and exposes the same capability through both `clawhub publish --owner` and `clawhub skill publish --owner`. Keep #1814 open until that implementation merges.
