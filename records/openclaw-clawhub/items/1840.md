## Summary

Keep this PR open. Current main does not yet implement the PR’s intended REST/CLI support for publishing new versions of org-owned skills: skill publish payload parsing drops ownerHandle, the V1 skill publish handler calls publishVersionForUser without resolving a requested publisher, and the skill CLI publish commands still have no --owner option.

## Best Possible Solution

Review and land this PR, or an equivalent patch, so skill publish payloads accept ownerHandle in source and generated schema artifacts, JSON and multipart REST paths preserve it, the REST handler resolves the requested publisher with minimumRole: "publisher", the skill CLI exposes --owner <handle>, and regression tests cover schema parsing, CLI payload generation, JSON API publishing, and multipart API publishing. Keep #1814 open until this PR or an equivalent fix merges.
