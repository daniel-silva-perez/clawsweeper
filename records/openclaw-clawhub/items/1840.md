## Summary

Keep open. Current main still lacks the REST/CLI org-owned skill publishing path this PR adds, so this PR remains a live implementation candidate for the open #1814 gap. Security review found no unrelated workflow, dependency, lockfile, secret, or package-publishing changes in the provided diff; the ownership-sensitive part should still be reviewed to preserve server-side publisher-role authorization.

## Best Possible Solution

Review and land this PR, or an equivalent patch, so `/api/v1/skills` accepts optional `ownerHandle`, preserves it through JSON and multipart payload parsing, resolves the target publisher server-side with `minimumRole: "publisher"`, passes `ownerPublisherId` into `publishVersionForUser`, and exposes the same capability through both `clawhub publish --owner` and `clawhub skill publish --owner`. Keep #1814 open until the implementation merges.
