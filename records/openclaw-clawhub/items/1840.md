## Summary

Keep this PR open. Current main still does not implement the PR’s intended REST/CLI support for org-owned skill publishes: skill publish payloads do not accept or preserve ownerHandle, the V1 skill publish handler still publishes under the API token actor’s default publisher, and the skill CLI has no --owner flag. The PR directly addresses the still-open linked issue #1814 and needs maintainer review rather than cleanup closure.

## What This Changes

This PR adds optional ownerHandle support to the skill publish schema, JSON and multipart REST publish paths, server-side publish-target resolution, CLI publish flags and payload generation, generated schema artifacts, and regression tests.

## Best Possible Solution

Review and land this PR, or an equivalent focused patch, so skill publish payloads accept ownerHandle in schema source and generated artifacts, JSON and multipart REST paths preserve it, the REST handler resolves the requested publisher with minimumRole: "publisher", the skill CLI exposes --owner <handle>, and regression tests cover schema parsing, CLI payload generation, JSON API publishing, and multipart API publishing. Keep #1814 open until this PR or an equivalent fix merges.
