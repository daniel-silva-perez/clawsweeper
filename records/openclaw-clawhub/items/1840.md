## Summary

Keep this PR open. Current main at 7bef2a0b65d40960c4ff7087d0acaacde64b0110 still does not implement the PR’s intended REST/CLI support for org-owned skill publishes: the V1 skill publish schema and parsers omit ownerHandle, the V1 skill publish handler calls publishVersionForUser directly without resolving a requested publisher, and both skill publish CLI commands lack --owner. The PR is still the direct implementation candidate for open issue #1814 and needs maintainer review rather than cleanup closure.

## What This Changes

The PR adds optional ownerHandle support to REST/CLI skill publishing so org-owned skills can publish new versions through API/CI with server-side publisher authorization.

## Best Possible Solution

Review and land this PR, or an equivalent focused patch, so skill publish payloads accept ownerHandle in schema source and generated artifacts, JSON and multipart REST paths preserve it, the REST handler resolves the requested publisher with minimumRole: "publisher", the skill CLI exposes --owner <handle>, and regression tests cover schema parsing, CLI payload generation, JSON API publishing, and multipart API publishing. Keep #1814 open until this PR or an equivalent fix merges.
