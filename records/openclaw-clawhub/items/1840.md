## Summary

Keep this PR open. Current main still does not implement the PR's intended REST/CLI path for publishing new versions of org-owned skills: the skill API publish handler calls `publishVersionForUser` with the API-token actor's default publisher, the shared skill publish parser/schema do not preserve `ownerHandle`, and the skill CLI publish commands have no `--owner` option. The PR remains a live implementation candidate for the open org-owned skill publishing issue it references.

## Best Possible Solution

Review and land this PR, or an equivalent patch, so `/api/v1/skills` accepts optional `ownerHandle`, resolves the requested publisher server-side with publisher-role authorization, passes the resulting `ownerPublisherId` to `publishVersionForUser`, and exposes the same capability through both `clawhub publish --owner` and `clawhub skill publish --owner`. Keep the linked issue open until the implementation merges.
