## Summary

Keep open. Current main still lacks the REST/CLI org-owned skill publishing support this PR adds for open #1814: the v1 publish endpoint drops any owner target, the publish schema has no ownerHandle field, and the CLI skill publish commands have no --owner option. The provided PR diff is directly relevant and does not show unrelated CI, dependency, lockfile, secret, or release-publishing changes, but the owner-selection auth path still deserves maintainer review before merge.

## Best Possible Solution

Review and land this PR, or an equivalent patch, so /api/v1/skills accepts optional ownerHandle for both JSON and multipart publishes, resolves the target publisher server-side with minimumRole: "publisher", passes ownerPublisherId into publishVersionForUser, and exposes the same capability through both clawhub publish --owner and clawhub skill publish --owner. Keep #1814 open until the implementation merges.
