## Summary

Keep open. Current main still has the reported VirusTotal presentation mismatch: scan records and moderation can distinguish VirusTotal Code Insight from AV engine detections, but the public skill security snapshot and frontend scanner rows still flatten that into a plain VirusTotal suspicious status without scanner or engine-count context. ClawHub policy also keeps issues open unless a maintainer resolves them or a fix lands.

## What This Changes

The issue requests that a skill whose VirusTotal report says no vendors flagged the file not display a misleading Suspicious VirusTotal tag.

## Best Possible Solution

Expose the VT scanner source and AV engine counts through the relevant skill security API and frontend shapes, label VirusTotal Code Insight separately from AV engine detections, and render zero-engine Code Insight suspicious results without implying that VirusTotal vendors flagged the file.
