## Summary

Keep this issue open. Current main has backend handling for uncorroborated VirusTotal Code Insight findings, but the user-visible scan UI and API surfaces still do not consistently carry or render `scanner` and `engineStats`, so the reported mismatch can still occur.

## Best Possible Solution

Keep this issue open and fix the scan presentation path. The likely implementation is to carry `scanner` and `engineStats` through the frontend/API VT analysis shapes, treat `scanner: "code_insight"` as the canonical Code Insight marker, render Code Insight separately or explicitly annotate it as AI analysis rather than AV vendor detections, show clean AV engine counts when relevant, preserve suspicious/malicious engine hits, and add regression coverage for Code Insight suspicious with zero malicious/suspicious AV engines.
