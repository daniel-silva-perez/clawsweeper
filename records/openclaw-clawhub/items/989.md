## Summary

Keep open. ClawHub policy keeps issues open, and current main still has the reported VirusTotal presentation mismatch: backend data can distinguish VirusTotal Code Insight from AV engine stats, but the user-facing scan components and API security snapshot still collapse the result into a plain VirusTotal suspicious status without scanner or engine-count context.

## What This Changes

The issue reports that a skill page shows a Suspicious VirusTotal tag even though the linked VirusTotal report says no security vendors flagged the file.

## Best Possible Solution

Keep this issue open and land a narrow fix that carries scanner and engineStats through the frontend/API VT analysis shapes, labels VirusTotal Code Insight separately from AV engine detections, shows malicious/suspicious engine counts, preserves true AV detections, and adds regression tests for Code Insight suspicious with zero and nonzero AV detections.
