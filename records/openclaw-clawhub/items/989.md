## Summary

Keep open. ClawHub review policy does not auto-close issues, and current main still leaves the reported VirusTotal wording/data mismatch plausible: recent commits mitigate backend moderation for uncorroborated Code Insight suspicious results, but the UI still renders `vtAnalysis.status` as a VirusTotal row and only detects Code Insight by `source`, while writers persist `scanner: 'code_insight'` with provider-specific `source` values.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely fix is to update `SecurityScanResults` and its `VtAnalysis` type to use `scanner` and `engineStats`, distinguish VirusTotal AV engine detections from VirusTotal Code Insight, render Code Insight as a separate or clearly annotated signal, normalize/backfill existing scan records as needed, and add a regression component test for clean AV stats plus Code Insight suspicious.
