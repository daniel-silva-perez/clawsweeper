## Repair Work Prompt

Fix the VirusTotal presentation mismatch reported in openclaw/clawhub#989. Current backend `vtAnalysis` can include `scanner: "code_insight"` and `engineStats`, but the frontend/API shapes drop that context and render a plain VirusTotal Suspicious tag even when AV engine malicious/suspicious counts are zero. Update the shared VT analysis shapes used by the skill detail UI, scanner detail page, HTTP API security snapshot, and public schema packages as needed to carry `scanner` and `engineStats`. Render VirusTotal Code Insight separately from AV engine detections: for Code Insight suspicious with `engineStats.malicious === 0` and `engineStats.suspicious === 0`, avoid implying AV vendors flagged the file; show clear Code Insight labeling and engine counts. Preserve true AV detections when malicious/suspicious engine counts are nonzero. Add regression tests for Code Insight suspicious with zero AV detections and with nonzero AV detections. Do not change VT scan scheduling, moderation visibility semantics, or unrelated scanner UI.
