## Summary

Keep this issue open. Current main has backend handling for VirusTotal Code Insight and AV engine stats, but the frontend and API presentation paths still mostly render `vtAnalysis.status` or `verdict` as a plain VirusTotal verdict and do not consistently expose `scanner` or `engineStats`, so the reported mismatch can still happen.

## Best Possible Solution

Keep this issue open and fix the scan presentation path. Carry `scanner` and `engineStats` through the public/frontend VT analysis shapes, treat `scanner: "code_insight"` as the canonical Code Insight marker, render Code Insight separately or clearly annotate it as AI analysis rather than AV vendor detections, show clean AV engine counts when relevant, preserve suspicious/malicious engine hits, and add regression coverage for Code Insight suspicious with zero and nonzero AV detections.
