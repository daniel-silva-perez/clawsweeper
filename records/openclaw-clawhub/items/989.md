## Repair Work Prompt

Fix the VirusTotal Code Insight presentation mismatch reported by https://github.com/openclaw/clawhub/issues/989. Current backend scan data stores `vtAnalysis.scanner` and `vtAnalysis.engineStats`, and moderation already suppresses uncorroborated Code Insight-only suspicious results, but the frontend/API still present a plain VirusTotal suspicious status. Implement a narrow fix that carries `scanner` and `engineStats` through the relevant frontend/API VT analysis shapes, labels VirusTotal Code Insight separately from AV engine detections, shows malicious/suspicious engine counts where useful, and preserves true AV malicious/suspicious detections. Add regression tests for Code Insight suspicious with zero AV detections and with nonzero AV detections. Do not change unrelated moderation policy, install gating, release workflows, or broad UI layout.
