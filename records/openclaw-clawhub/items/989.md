## Summary

Keep open. ClawHub policy keeps issues open, and current main still has the reported VirusTotal presentation mismatch: backend scan records can distinguish VirusTotal Code Insight from AV engine stats, but the skill security API and frontend still flatten that into a plain VirusTotal suspicious status without scanner or engine-count context.

## What This Changes

The issue reports that a skill page shows a Suspicious VirusTotal tag even though the linked VirusTotal report says no security vendors flagged the file.

## Best Possible Solution

Expose the VT scanner source and AV engine counts through the relevant skill security API and frontend shapes, label VirusTotal Code Insight separately from AV engine detections, and render zero-engine Code Insight suspicious results without implying that VirusTotal vendors flagged the file.
