## Summary

Keep this issue open. Current main has backend moderation improvements for uncorroborated VirusTotal Code Insight findings, but the user-visible scan UI still renders the VT-derived status as a VirusTotal row and does not consume the normalized `scanner` or `engineStats` fields needed to distinguish Code Insight from AV vendor detections. Repository policy for this ClawHub run also says issues remain open; only PRs that are certainly implemented on main are auto-close candidates.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The likely fix is to extend the frontend/API VT analysis shape to carry `scanner` and `engineStats`, treat `scanner: "code_insight"` as the canonical Code Insight signal, render Code Insight separately or clearly annotate it as AI analysis rather than AV vendor detections, show or summarize clean AV engine stats when relevant, handle legacy source-only records defensively, and add a UI/API regression test for Code Insight suspicious with zero malicious/suspicious AV engines.
