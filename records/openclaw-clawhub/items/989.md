## Summary

Keep open. Repository policy for this ClawHub review run says issues should remain open, and current main still leaves the reported VirusTotal wording mismatch plausible: backend records Code Insight scanner metadata and clean AV engine stats, but the UI still renders the status as a VirusTotal row and only detects Code Insight from the provider source string.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely fix belongs in SecurityScanResults and its VtAnalysis/security data shape: pass through scanner and engineStats, treat scanner: "code_insight" as the canonical Code Insight signal, render Code Insight separately or clearly annotate it as an AI analysis rather than AV vendor detections, handle legacy cached records defensively, and add a regression test for suspicious Code Insight with clean AV engine stats.
