Keep open. ClawHub policy keeps issues open, and current main still appears to have the reported VirusTotal presentation mismatch: backend scan records can distinguish VirusTotal Code Insight from AV engine stats, but the UI and skill security API still expose a generic VirusTotal suspicious status without scanner or engine-count context.

## What This Changes

The issue reports a ClawHub skill showing a Suspicious VirusTotal tag while the linked VirusTotal report says no security vendors flagged the file.

## Best Possible Solution

Expose VT scanner source and AV engine counts through the relevant frontend/API shapes, label VirusTotal Code Insight separately from AV engine detections, and render zero-engine Code Insight suspicious results without implying that VirusTotal vendors flagged the file.
