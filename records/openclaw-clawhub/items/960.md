## Summary

Keep this issue open. The repository-specific cleanup policy for ClawHub issues does not allow automated closure here, and current main still has credible search/indexing gaps for published skills: publish inserts embeddings but does not visibly upsert the search digest, while search relies on vector recall, exact full-slug injection, and bounded recent digest scans.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The likely fix path is to add publish-time skillSearchDigest upsert/repair and a backfill for affected skills, land durable indexed normalized slug/display-name prefix recall such as the #1254 direction, then verify christine-srp/amazon-seller-research appears in CLI/API and website search before manual closure.
