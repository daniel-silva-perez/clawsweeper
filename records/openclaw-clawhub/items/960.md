## Summary

Keep open. Current main still lacks durable indexed slug/display-name prefix recall for skills, so a published skill can remain inspectable by direct slug while being missed by CLI and website search when vector recall and the bounded recent fallback do not surface it. The open #1254 tracker is the clearest canonical path for the code-level fix, and ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

A published skill that is accessible by direct URL and CLI inspect should appear in CLI and website search results for matching slug, display-name, and topic queries after publish.

## Best Possible Solution

Implement the #1254-style backend fix: add normalized slug and display-name prefix lookup fields to `skillSearchDigest`, keep them synced from skill publish/update paths, backfill existing rows cursor-wise, query those indexes before or alongside vector results, and then verify or repair the affected production skill rows.
