## Summary

Keep open. This is an open ClawHub issue, and the repo-specific cleanup policy only permits auto-closing pull requests. Substantively, current main has a mitigation that schedules package and skill owner digest sync as separate internal mutations, but the scheduled helpers still loop through paginated pages inside one mutation invocation, so large publishers can still hit Convex's single-paginated-query limit.

## Best Possible Solution

Keep this issue open and land one narrow Convex hardening PR. The best fix is to keep the current separate package and skill scheduled mutations, but add cursor-aware batching so each internal mutation invocation performs at most one paginated query, processes that page, and self-schedules a continuation when needed. Add regression tests for multi-page package and skill owner sync, then verify CLI publish, web upload, and import publishing after deployment. Related reports #1173, #1175, #1179, #1182, and #1205 can then be consolidated or closed by maintainers.
