## Summary

Keep open. Current main mitigates the originally reported two-paginated-query publisher trigger path by scheduling package and skill digest rebuilds as separate internal mutations, with regression coverage. However, this is an open ClawHub issue, and the supplied repository policy only permits automated closure for pull requests that are certainly implemented on main. There is also a remaining large-owner hardening question because each scheduled owner-scoped helper can still loop over paginated results.

## Best Possible Solution

Keep this issue open for maintainer handling. Maintainers should verify the scheduled digest-sync mitigation is deployed and unblocks CLI and web publishing for first-time and existing publishers, then manually close or consolidate it with related reports such as #1175, #1182, and #1205 if production is fixed. If large-owner publishes can still fail, replace the owner-scoped paginated loops with bounded self-scheduling batch mutations so each invocation performs at most one paginated query.
