## Summary

Keep open. ClawHub cleanup policy keeps open issues open, and current main still has a concrete residual Convex pagination failure path: publish/profile flows still touch `ensurePersonalPublisherForUser`, publisher changes schedule owner digest sync, and each owner digest sync helper can still call `.paginate()` repeatedly inside one internal mutation for multi-page owners.

## What This Changes

The issue reports that CLI and web publishing fail because the backend publish/profile path can call multiple Convex paginated queries from `ensurePersonalPublisherForUser` through owner digest sync.

## Best Possible Solution

Keep this issue open as the canonical publish-pagination bug and land one focused Convex hardening PR. Preserve indexed owner queries and background scheduling, but make each owner digest sync mutation process at most one paginated page per invocation and self-schedule continuations with `continueCursor`; add regression tests for multi-page package and skill owner sync, then close or link the duplicate reports after the fix merges and deploys.

## Repair Work Prompt

Fix the Convex publish/profile pagination failure reported here and in the related publish-pagination cluster (#1170, #1171, #1173, #1175, #1179, #1182, #1205). Current main schedules `syncPackageSearchDigestsForOwnerPublisherIdInternal` and `syncSkillSearchDigestsForOwnerPublisherIdInternal` from publisher changes, but each helper loops `.paginate({ cursor, numItems: 100 })` inside one mutation. Refactor `convex/functions.ts` so each owner digest sync mutation performs at most one paginated query per invocation and schedules a continuation with `continueCursor` when more rows remain. Preserve indexed `by_owner_publisher` / `by_owner` queries and existing digest update behavior. Do not replace the loops with unbounded `.collect()`. Audit `syncPackageSearchDigestsForOwnerUserId` for the same pattern and apply the same one-page continuation approach if needed. Add focused regression tests in `convex/functions.test.ts` for multi-page publisher package sync, multi-page publisher skill sync, and continuation scheduling without a second paginate call in the same invocation. Keep public API/CLI behavior unchanged; run Convex codegen if internal function args change.
