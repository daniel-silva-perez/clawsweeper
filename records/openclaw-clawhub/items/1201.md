## Summary

Keep this issue open. The current main branch appears to mitigate the reported publish-blocking Convex error by moving publisher package and skill digest rebuilds into separate scheduled internal mutations, with regression coverage. However, the repository-specific ClawHub cleanup policy only permits automated closure for pull requests that are certainly implemented on main, and this item is an issue. There is also still residual hardening work possible because each scheduled digest helper can run more than one `.paginate()` call when an owner has more than 100 matching records.

## Best Possible Solution

Keep this issue open for maintainer handling. The practical path is to verify that the scheduled digest-sync mitigation is deployed and unblocks CLI and web publishing for both first-time and existing publishers, then manually close or consolidate this with related reports such as #1175, #1182, and #1205 if production is fixed. If large-owner publishes can still fail, replace the remaining owner-scoped paginated loops with bounded self-scheduling batch mutations.
