## Summary

Keep open. ClawHub cleanup policy does not auto-close open issues, and source inspection shows a remaining backend pagination failure path: publisher writes schedule owner digest mutations, but each scheduled mutation can still loop over `.paginate()` in one Convex invocation for large owners. The related fix PR #1202 was closed unmerged, so there is no shipped fix to close this against.

## What This Changes

The issue requests fixing CLI and web skill publish failures caused by Convex pagination errors during personal publisher and search digest synchronization.

## Best Possible Solution

Land a focused Convex backend hardening PR that keeps indexed owner queries, processes at most one paginated page per digest-sync invocation, self-schedules continuations with `continueCursor`, and adds regression coverage for multi-page package and skill owner sync before maintainers close the duplicate publish-pagination reports after deployment.
