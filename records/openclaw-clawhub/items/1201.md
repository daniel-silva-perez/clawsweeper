## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits automated closure for pull requests that are certainly implemented on current main. Substantively, current main has a partial mitigation that schedules package and skill owner digest sync as separate internal mutations, but each scheduled helper still loops through paginated pages inside one mutation invocation, so large publishers can still hit Convex's single-paginated-query limit.

## Best Possible Solution

Keep this issue open and land one focused Convex hardening PR. Preserve the current split background package/skill scheduling, but make each owner digest sync internal mutation process at most one paginated page, write that page's digests, and self-schedule a continuation with continueCursor when more rows remain. Add regression coverage for multi-page package and skill owner sync, then verify CLI publish, web upload, and GitHub import publishing after deployment. Related reports #1173, #1175, #1179, #1182, and #1205 can then be consolidated or closed by maintainers.
