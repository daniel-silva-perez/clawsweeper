Keep open. The repository policy for ClawHub issues does not allow auto-closing this issue, and current main still has a concrete residual Convex pagination path: publish and profile flows still ensure the personal publisher, publisher writes still schedule owner digest sync, and the scheduled digest mutations still loop over `.paginate()` until done.

## What This Changes

The issue reports that CLI and web publish/profile flows fail with Convex's multiple-paginated-query error when `ensurePersonalPublisherForUser` triggers owner digest synchronization.

## Best Possible Solution

Land a focused Convex backend hardening PR that keeps the indexed owner queries, processes at most one paginated page per digest-sync mutation, self-schedules continuations with `continueCursor`, and adds regression coverage for multi-page package and skill owner sync before maintainers close the duplicate publish-pagination reports after deployment.
