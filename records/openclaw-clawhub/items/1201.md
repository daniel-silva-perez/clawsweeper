## Clownfish Work Prompt

Implement bounded continuation for publisher owner digest sync pagination. Current main schedules `syncPackageSearchDigestsForOwnerPublisherIdInternal` and `syncSkillSearchDigestsForOwnerPublisherIdInternal` separately, but each helper loops over `.paginate()` until done. Refactor so each internal mutation invocation performs at most one paginated query for one owner and schedules itself with the continuation cursor when `isDone` is false. Keep package and skill sync as separate jobs, keep the optional `ctx.scheduler` guard, use the existing `by_owner_publisher` indexes, and do not switch to unbounded `.collect()`. Add focused tests in `convex/functions.test.ts` proving no invocation calls `paginate` more than once and that package and skill continuations are scheduled with the next cursor. Run codegen if internal function args change. Do not touch unrelated UI, docs, or release metadata.
