## Summary

Keep this issue open. Current main appears to mitigate the originally reported Convex error by scheduling package and skill owner-publisher digest rebuilds as separate internal mutations, with regression coverage. However, the supplied openclaw/clawhub cleanup policy says issues should remain open and only pull requests may be auto-closed when certainly implemented on main. There is also a residual hardening question because each scheduled digest helper can still execute more than one paginated query if an owner has more than 100 matching records.

## Best Possible Solution

Keep this issue open for maintainer handling. Maintainers should verify the scheduled digest-sync mitigation is deployed and unblocks CLI and web publishing for first-time and existing publishers, then manually close or consolidate this with related reports such as #1175, #1182, and #1205 if production is fixed. If large-owner publishes can still fail, replace the remaining owner-scoped paginated loops with bounded self-scheduling batch mutations so each invocation performs at most one paginated query.
