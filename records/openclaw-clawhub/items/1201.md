Preserve the scheduling split, but make owner digest sync process at most one page per mutation invocation and self-schedule a continuation with `continueCursor` when more rows remain. Cover publisher-owned package sync and publisher-owned skill sync; also audit the sibling owner-user package sync and package-release fallback pagination helpers so the same single-pagination limit is not left on adjacent trigger paths. Do not replace these paths with unbounded `.collect()` unless maintainers explicitly choose that tradeoff. Add focused regression tests in `convex/functions.test.ts` that simulate multi-page owners, assert one paginated query per invocation, and assert continuation scheduling. Keep changes scoped to Convex functions/tests and generated API output only if codegen requires it.
