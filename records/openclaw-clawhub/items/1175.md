## Summary

Keep this issue open. Current main still contains the reported Convex failure shape: CLI/Web publish reaches `insertVersion`, `insertVersion` still calls `ensurePersonalPublisherForUser`, publisher writes still schedule owner-publisher digest rebuilds, and each owner-publisher package/skill rebuild can still call `.paginate()` more than once inside a single internal mutation. ClawHub cleanup policy also keeps issues open in this pass unless they are PRs already implemented on main.

## What This Changes

Fix the publish/update path so owner-publisher digest sync does not trigger Convex's multiple-paginated-query error during `ensurePersonalPublisherForUser`.

## Best Possible Solution

Land one focused Convex backend PR that makes the owner-publisher package and skill digest rebuild internal mutations accept a cursor, process at most one paginated page per invocation, and self-schedule a continuation with `continueCursor` when `isDone` is false. Add regression tests for multi-page owner rebuilds, then consolidate the duplicate publish-failure issue cluster after the fix is merged and deployed.

Do not replace the loops with unbounded `collect()`.
