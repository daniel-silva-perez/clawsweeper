## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still contains a concrete Convex pagination hazard in the owner-publisher digest rebuild path: publisher writes schedule background internal mutations, but each scheduled mutation can still loop over `.paginate()` multiple times without accepting or scheduling a continuation cursor. The original synchronous trigger failure appears partly mitigated by background scheduling, but the source-backed multi-page failure shape remains unresolved and related publish-pagination reports remain open.

## What This Changes

The issue reports that `clawhub publish` fails while ensuring a personal publisher because owner-publisher digest sync runs multiple Convex paginated queries in one function.

## Best Possible Solution

Keep the issue open until owner-publisher package and skill digest rebuilds process at most one paginated page per Convex function invocation, persist continuation through scheduled follow-up mutations, and then consolidate the duplicate publish/import pagination reports after deployment.

Fix the ClawHub publish/import pagination failure reported in this issue and the related cluster. In `convex/functions.ts`, change owner-publisher package and skill digest rebuilds so each Convex internal mutation performs at most one `.paginate()` call per invocation, accepts a nullable continuation cursor, processes that page, and schedules a follow-up invocation when `isDone` is false. Keep the fix scoped to the backend digest sync/publish path and avoid unrelated UI, CLI, release, or schema churn unless codegen is required.
