## Summary

Keep this issue open. Current main still leaves the reported Convex failure shape in the owner-publisher digest rebuild helpers: the publish path reaches `ensurePersonalPublisherForUser`, publisher writes schedule digest rebuild mutations, and each owner-publisher digest helper can still loop over `.paginate()` more than once inside a single Convex mutation invocation. ClawHub cleanup policy also keeps open issues open in this pass; this is a strong focused fix-PR candidate.

## Best Possible Solution

Land one focused Convex backend PR for the publish-pagination cluster: make the owner-publisher package and skill digest rebuild internal mutations process at most one paginated page per invocation, carry the cursor in validated args, and schedule the same mutation with `continueCursor` when `isDone` is false. Add regression tests for multi-page owner rebuilds, avoid unbounded `.collect()`, then deploy and verify an updated-skill publish succeeds before consolidating duplicate reports.
