## Summary

Keep open. ClawHub cleanup policy keeps issues open unless maintainers resolve them, and current main still has a concrete Convex pagination hazard in the owner-publisher digest rebuild helpers: each scheduled internal mutation can loop over `.paginate()` multiple times without accepting or scheduling a continuation cursor. Current main does appear to have moved publisher-trigger rebuilds into background mutations, which may mitigate the original synchronous publish failure, but the cursorless multi-page rebuild path remains unresolved and related publish-pagination reports are still open.

## What This Changes

Fix `clawhub publish` so creating or updating a personal publisher does not fail from Convex pagination errors in owner-publisher digest rebuilds.

## Best Possible Solution

Keep the issue open until the Convex backend processes owner-publisher package and skill digest rebuilds one page per invocation with continuation scheduling, then deploy the fix and consolidate the duplicate publish-pagination reports.
