## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still contains the source-backed Convex hazard behind this report: owner-publisher package and skill digest rebuilds can execute repeated `.paginate()` calls inside one internal mutation, while the scheduled internal mutations accept only `ownerPublisherId` and no continuation cursor. Related publish-pagination reports remain open, and the prior fix PR was closed unmerged.

## What This Changes

Fix publish failures where creating or updating a personal publisher triggers Convex owner-publisher digest sync work that can violate Convex's single paginated-query-per-function limit.

## Best Possible Solution

Keep this open until the Convex backend processes owner-publisher package and skill digest rebuilds one page per invocation with continuation scheduling, then deploy the fix and consolidate the duplicate publish-pagination reports.
