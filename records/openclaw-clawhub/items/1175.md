## What This Changes

The issue reports `clawhub publish` failing because `ensurePersonalPublisherForUser` can trigger owner-publisher digest rebuilds that run multiple Convex paginated queries in one mutation.

## Best Possible Solution

Land a focused Convex backend fix that makes each owner-publisher digest rebuild process one paginated page per mutation invocation, carry continuation cursors through validated args, and self-schedule follow-up work until complete; after deploy, consolidate the duplicate publish/import failure reports.

## Repair Work Prompt

Fix the ClawHub publish/import Convex pagination failure reported across #1170, #1171, #1173, #1175, #1176, #1179, #1182, #1195, #1197, #1201, and #1205. In `convex/functions.ts`, update `syncPackageSearchDigestsForOwnerPublisherIdInternal` and `syncSkillSearchDigestsForOwnerPublisherIdInternal` so each invocation performs at most one `.paginate()` call, accepts a nullable cursor argument, processes that page, and schedules itself with `continueCursor` when `isDone` is false. Keep the indexed `by_owner_publisher` queries and avoid replacing this with broad `.collect()` reads. Add focused regression tests in `convex/functions.test.ts` that prove multi-page owner-publisher package and skill digest rebuilds schedule continuations and do not run multiple paginated queries in a single mutation. Do not change unrelated publish, scanner, or UI behavior. Include a short changelog/PR note that this unblocks first-time and existing publishers hitting the Convex single-pagination limit.
