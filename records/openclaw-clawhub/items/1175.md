## Summary

Keep open. ClawHub cleanup policy keeps issues open unless a maintainer consolidates them or a fix lands, and current main still contains the reported Convex failure shape: publish reaches `ensurePersonalPublisherForUser`, publisher writes schedule owner-publisher digest rebuilds, and the package/skill rebuild helpers can still loop over `.paginate()` inside a single internal mutation without accepting continuation cursors. Related PR #1183 was closed unmerged, so no merged main fix supersedes this report.

## What This Changes

Publishing an updated skill version fails with a Convex multiple-paginated-queries error in `ensurePersonalPublisherForUser`, and the expected behavior is for the version publish to complete successfully.

## Best Possible Solution

Land one focused Convex backend PR that changes the owner-publisher package and skill digest rebuild internal mutations to accept a cursor, process at most one paginated page per invocation, and self-schedule a continuation with `continueCursor` when `isDone` is false. Add regression coverage for multi-page owner-publisher rebuilds, then consolidate the duplicate publish-failure issue cluster after the fix is merged and deployed.

## Repair Work Prompt

Fix the Convex publish failure where owner-publisher digest rebuilds can run multiple paginated queries in one internal mutation. In `convex/functions.ts`, update `syncPackageSearchDigestsForOwnerPublisherIdInternal` and `syncSkillSearchDigestsForOwnerPublisherIdInternal` so each invocation processes at most one `.paginate()` page for its owner publisher, accepts an optional cursor, and schedules itself with `continueCursor` when `isDone` is false. Keep the package and skill syncs as separate scheduled mutations, preserve the existing `by_owner_publisher` indexes, and do not replace the pagination with unbounded `collect()` or unrelated CLI/UI changes. Add regression tests in `convex/functions.test.ts` for multi-page package and skill owner-publisher rebuilds proving only one paginated query happens per invocation and continuations are scheduled. Include generated Convex files only if codegen changes them. Related refs: #1170, #1171, #1173, #1175, #1176, #1179, #1182, #1195, #1197, #1201, #1205, and closed unmerged PR #1183.
