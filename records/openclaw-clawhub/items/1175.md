## Summary

Keep this issue open. The ClawHub cleanup policy for this run says open issues are not eligible for automated closure, and current main only mitigates the reported publish-time double-pagination path rather than proving the production issue is fully resolved for all publisher sizes.

## Best Possible Solution

A maintainer should verify the production Convex backend includes the scheduled owner-publisher digest sync, confirm publishing an updated skill no longer fails, then manually close or consolidate this issue with the related publish-failure reports. For a complete code fix, convert the owner-publisher digest rebuild helpers to cursor-based self-scheduling or another bounded single-pagination pattern so multi-page owners cannot reproduce the same Convex limit.
