## What This Changes

The issue reports that CLI skill publishing fails with `SKILL.md required` even when a valid `SKILL.md` exists in the source skill directory.

## Best Possible Solution

Skill publish should resolve user-entered relative source paths against the caller's current directory when that resolves to a skill folder, while preserving absolute paths and existing configured-workdir behavior for install, update, and sync flows. Add focused regression coverage for a cwd skill folder with a different configured Clawdbot/OpenClaw workdir.

## Repair Work Prompt

Preserve absolute path behavior and avoid broad workdir semantic changes for install, update, sync, and configured workspace flows. Add focused regression coverage for a cwd skill folder with a different configured/global workdir, plus any small helper tests needed for the chosen precedence rule. Do not change registry API behavior, auth, package publishing, or moderation/search code. Include a short changelog or release-note entry only if the repository has an established place for CLI bugfix notes.
