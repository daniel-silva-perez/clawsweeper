## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing PRs that current main certainly implements. Substantively, current main still has the plausible failure path described in the report: `skill publish .` resolves the explicit folder argument through the global workdir, which can come from Clawdbot/OpenClaw config instead of the caller's skill directory, then throws `SKILL.md required` when that resolved folder lacks a root skill file. Related #1509 appears to track the same user-visible bug.

## Best Possible Solution

Keep this issue open and land a focused CLI fix. Explicit `clawhub skill publish .` and legacy `clawhub publish .` should resolve the source folder against the caller's actual cwd, or publish should otherwise clearly separate source-path resolution from global workdir fallback. Add regression coverage where a configured Clawdbot/OpenClaw workspace differs from cwd and the cwd skill folder contains `SKILL.md`; handle related #1509 in the same PR or mark one report canonical.
