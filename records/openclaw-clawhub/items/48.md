## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and current main still has a plausible path to the reported error: `publish .` is resolved under the global workdir, which can be a configured Clawdbot/OpenClaw workspace instead of the caller's skill directory containing `SKILL.md`. Related #1509 reports the same user-visible publish failure and should be handled with the same focused CLI fix.

## What This Changes

The issue reports that the CLI publish command errors with `SKILL.md required` even when publishing a directory that contains a valid `SKILL.md`.

## Best Possible Solution

Publish should resolve user-entered relative source paths against the caller's cwd when that cwd contains the requested skill folder or `SKILL.md`, while preserving absolute paths and configured workspace behavior for install/sync flows. Add a regression test for a cwd skill folder with a different configured Clawdbot/OpenClaw workdir.
