Current main still has a plausible failure path: `skill publish .` resolves the source path against the resolved global workdir, and that workdir can come from Clawdbot/OpenClaw config instead of the shell directory containing `SKILL.md`. ## Clownfish Work Prompt

Fix the ClawHub CLI skill publish path semantics for explicit relative source paths. Current `cmdPublish` resolves `folderArg` against `opts.workdir`; when `resolveWorkdir()` falls back to a configured Clawdbot/OpenClaw workspace, `clawhub skill publish .` run from a skill folder can scan the configured workspace and fail with `SKILL.md required`. Update the command plumbing so explicit publish source paths such as `.` are resolved relative to the caller's actual `process.cwd()` unless an explicit absolute path is supplied. Preserve existing behavior for absolute paths and for `sync`, which passes absolute skill folders. Add regression coverage where configured default workspace differs from the process cwd and cwd contains a valid `SKILL.md`. Cover related #1509 with the same fix. Do not change package publish semantics, server APIs, auth behavior, or unrelated workspace discovery. No changelog is required unless the repo has an existing CLI release-note convention for this fix.
