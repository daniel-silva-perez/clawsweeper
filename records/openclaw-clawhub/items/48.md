## Summary

Keep this issue open. The repository-specific cleanup policy for openclaw/clawhub keeps issues open, and current main still has a plausible code path for the reported `SKILL.md required` failure: `publish .` is resolved against the CLI workdir, while the workdir can be a configured Clawdbot/OpenClaw workspace rather than the shell directory that contains `SKILL.md`. Related #1509 is open and overlapping, but it is not clearly marked as the canonical replacement.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely fix is to resolve explicit publish source paths relative to the caller's actual cwd for user-facing publish commands, or make the workdir fallback behavior explicit, then add a regression test where a configured default workspace differs from cwd and `clawhub skill publish .` runs from a directory containing `SKILL.md`. Maintainers can also decide whether #1509 should become the canonical tracker.
