## Summary

Keep this issue open. The repository-specific cleanup policy says openclaw/clawhub issues should stay open unless they are pull requests implemented on current main, and the current CLI still has a plausible path to the reported `SKILL.md required` error when `publish .` is resolved against the configured ClawHub/OpenClaw workdir instead of the shell cwd containing `SKILL.md`. Related #1509 is open and overlapping, but not clearly resolved or marked as the canonical replacement.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely fix is to resolve explicit publish source paths relative to the caller's actual cwd for user-facing publish commands, or make the workdir fallback behavior explicit, then add a regression test where a configured default workspace differs from cwd and `clawhub skill publish .` runs from a directory containing `SKILL.md`. Maintainers can also decide whether #1509 should become the canonical tracker.
