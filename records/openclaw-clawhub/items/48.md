## Summary

Keep open. This is an open ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on current main. Substantively, current main still has a plausible failure path: explicit `clawhub skill publish .` / legacy `clawhub publish .` source paths are resolved relative to the resolved global workdir, which can come from Clawdbot/OpenClaw config instead of the shell directory containing `SKILL.md`. Related #1509 appears to report the same remaining user-visible bug.

## Best Possible Solution

Keep this issue open and land a focused CLI fix. Explicit `clawhub skill publish .` and legacy `clawhub publish .` source paths should resolve against the caller's actual cwd, or publish should clearly distinguish source-path resolution from global `--workdir` behavior. Add regression coverage where configured Clawdbot/OpenClaw workdir differs from cwd and the cwd skill folder contains `SKILL.md`; handle #1509 in the same PR or mark one report canonical.
