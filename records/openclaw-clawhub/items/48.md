## What This Changes

The issue reports that CLI skill publishing fails with `SKILL.md required` even when the caller's target directory contains a valid `SKILL.md`.

## Best Possible Solution

Skill publish should resolve user-entered relative source paths against the caller's current directory when that resolves to a skill folder, while preserving absolute paths and existing configured-workdir behavior for install, update, and sync flows. Add focused regression coverage for a cwd skill folder with a different configured Clawdbot/OpenClaw workdir.

## Repair Work Prompt

Fix the CLI skill publish path-resolution bug reported here and in #1509. `clawhub skill publish .` and legacy `clawhub publish .` should inspect the caller's current directory when the user supplies a relative source path that resolves to a skill folder containing `SKILL.md`, even if the global workdir resolves to a configured Clawdbot/OpenClaw workspace. Preserve absolute path behavior and avoid broad changes to install, update, sync, or configured workspace semantics. Add focused regression coverage for publishing from a cwd skill folder while the resolved global workdir points elsewhere. Also keep existing plugin-folder detection and `SKILL.md required` validation behavior for genuinely invalid folders. Do not change registry/API payload shape or unrelated publish moderation behavior.
