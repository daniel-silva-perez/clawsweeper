## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and current main still has a credible path to the reported failure: publish resolves a relative source path against the resolved global workdir, which can differ from the caller's directory containing `SKILL.md`. Related #1509 reports the same user-visible publish error and should be handled with the same focused CLI fix.

## What This Changes

Fix CLI skill publishing so `clawhub publish .` / `clawhub skill publish .` recognizes an existing `SKILL.md` in the caller's skill directory instead of failing with `SKILL.md required`.

## Best Possible Solution

Publish should resolve user-entered relative source paths against the caller's cwd when that cwd contains the requested skill folder or `SKILL.md`, while preserving absolute paths and existing configured-workdir behavior for install/update/sync flows. Add focused regression coverage for a cwd skill folder with a different configured Clawdbot/OpenClaw workdir.
