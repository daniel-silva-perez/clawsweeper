## Summary

Keep this issue open. Repository policy for openclaw/clawhub only permits automated closure for pull requests that are definitely implemented on current main, and this item is an issue. The current CLI also still has a plausible path to the reported `SKILL.md required` error: `publish .` resolves `.` against the resolved ClawHub workdir, which can be a configured Clawdbot/OpenClaw workspace instead of the shell cwd containing `SKILL.md`.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely fix is to resolve explicit publish source paths relative to the caller's actual cwd for user-facing publish commands, or make the workdir fallback behavior explicit, then add a regression test where a configured default workspace differs from cwd and `clawhub skill publish .` runs from a directory containing `SKILL.md`. Maintainers can also decide whether #1509 should become the canonical tracker.
