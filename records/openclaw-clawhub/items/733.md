## Summary

Keep open. This is an open ClawHub issue, and this cleanup lane only auto-closes ClawHub PRs that are certainly implemented on current main. Substantively, the original VT-only stale reason appears resolved, but both live skills still remain suspicious because current main intentionally preserves non-allowlisted static `suspicious.potential_exfiltration` findings even when VT and LLM/OpenClaw are clean; the related allowlist PR was closed unmerged as a policy decision.

## What This Changes

The issue requests a rescan or manual clearing of stale suspicious flags for `ai-image-prompts` and `nano-banana-pro-prompts-recommend` when VirusTotal and OpenClaw scan results are benign.

## Best Possible Solution

Maintainers should manually review the two production skills and either clear/rescan them as false positives or approve a narrow moderation-policy change that demotes selected static findings only when VT and LLM/OpenClaw are explicitly clean, with regression coverage and a production backfill/rescan plan.
