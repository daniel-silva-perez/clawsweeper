## Summary

Keep open. This is a ClawHub issue, and the supplied cleanup policy only permits auto-closing ClawHub pull requests that current main definitely implements. Substantively, both named skills still report a live suspicious moderation state for `suspicious.potential_exfiltration`; current main clears uncorroborated VT Code Insight suspicious states, but intentionally preserves non-allowlisted static findings even when VT and LLM/OpenClaw are clean. The closed #1688 PR is useful prior art for the policy change, but it was not merged.

## Best Possible Solution

Keep this issue open for maintainer moderation follow-up. The practical path is either a staff manual override/recompute for the two named skills after review, or a deliberate policy change based on the #1688 prior art that demotes `suspicious.potential_exfiltration` only under explicit clean VT and LLM/OpenClaw verdicts, with moderation engine behavior and regression tests updated together.
