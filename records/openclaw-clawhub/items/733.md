## Summary

Keep open. Repository policy for openclaw/clawhub says issues should not be auto-closed, and the named production skills still show `isSuspicious: true`. Current main contains the older VT stale-flag cleanup from #418, but it does not clear the remaining static `suspicious.potential_exfiltration` reason when VT and LLM are clean; the related PR #1688 proposed that narrower demotion but was closed unmerged.

## Best Possible Solution

Keep this issue open for maintainer moderation action. The practical paths are either a staff manual override/recompute for the two named skills if their `scripts/setup.js` findings are judged false positives, or a narrower policy change that safely demotes `suspicious.potential_exfiltration` only when VT and OpenClaw/LLM are both explicitly clean, with backend snapshot logic, scan/API/UI behavior, and regression tests updated together.
