## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are definitely implemented on current main. Substantively, both named production skills still report `isSuspicious: true`; current main has the stale VT Code Insight cleanup path, but it intentionally does not externally clear `suspicious.potential_exfiltration` when VT and OpenClaw/LLM are clean. The related #1688 policy-change PR was closed unmerged.

## Best Possible Solution

Keep this issue open for maintainer moderation action. The practical path is either a staff manual override/recompute for the two named skills if their static findings are reviewed as false positives, or a deliberate policy change that demotes `suspicious.potential_exfiltration` only when VT and OpenClaw/LLM are both explicitly clean, with backend snapshot logic, API/UI behavior, and regression tests updated together. The closed #1688 PR is useful prior art but is not shipped.
