## Summary

Keep open. This is an open ClawHub issue, so the cleanup policy does not permit auto-closing it; substantively, the specific user-facing scan concern is still observable because the latest version’s scan API reports `security.status: "suspicious"` from the VirusTotal/Code Insight telemetry analysis even though the main skill detail API has no public moderation warning.

## What This Changes

The issue asks ClawHub to stop treating `arbitrum-dapp-skill` as suspicious when the triggering install-script network call is documented, opt-out GoatCounter telemetry.

## Best Possible Solution

Define a maintainer-approved scanner policy for documented, opt-out install telemetry, then make the scan API/UI and any rescan/backfill path present that case as an informational note rather than a suspicious warning when corroborating signals are clean; leave genuinely unknown, obfuscated, or exfiltrating install network calls flagged.
