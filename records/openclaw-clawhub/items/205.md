## Summary

Keep open. The repository cleanup policy for ClawHub does not permit auto-closing open issues, and the reported false-positive condition is still observable in production: the skill detail API has no moderation warning, but the scan API still reports the latest arbitrum-dapp-skill version as suspicious from the VirusTotal-style scanner.

## What This Changes

The issue asks ClawHub to treat arbitrum-dapp-skill's documented, opt-out GoatCounter installer telemetry as informational rather than suspicious.

## Best Possible Solution

Define a maintainer-approved security policy for transparent, opt-out install telemetry, then update the scan API/UI and any rescan or backfill path so clean corroborating signals present this case as informational while unknown, obfuscated, or data-transmitting install calls remain suspicious.
