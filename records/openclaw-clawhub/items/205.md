## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and the reported condition is still observable: the public skill detail has no moderation warning, but the scan endpoint still reports the latest arbitrum-dapp-skill version as suspicious because of the documented GoatCounter install ping. Changing that classification is a maintainer security-policy decision, not conservative cleanup.

## What This Changes

The issue asks ClawHub to treat arbitrum-dapp-skill's documented, opt-out GoatCounter install telemetry as informational rather than suspicious.

## Best Possible Solution

Define a maintainer-approved scanner policy for transparent, opt-out install telemetry, then update scan responses, UI messaging, and any rescan/backfill or manual-review workflow so corroborated benign telemetry is presented as informational while unknown, obfuscated, or data-transmitting install calls remain suspicious.
