## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing PRs that current main definitely implements; substantively, the live scan API still reports the latest arbitrum-dapp-skill version as suspicious from the VirusTotal-style scanner even though the public skill moderation state is null.

## What This Changes

The issue asks ClawHub to treat arbitrum-dapp-skill’s documented, opt-out GoatCounter install telemetry as informational rather than a suspicious warning when other scan signals are clean.

## Best Possible Solution

Define a maintainer-approved scanner policy for transparent, opt-out install telemetry, then update the scan API/UI and any rescan or backfill path so clean corroborating signals present that case as an informational note while unknown, obfuscated, or data-transmitting install calls remain suspicious.
