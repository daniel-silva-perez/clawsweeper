## Summary

Keep this PR open. Current main still routes an active Discord `reconnect-exhausted` event to the lifecycle stop/reject path when it arrives during abort-driven stale-socket restart before `lifecycleStopping` flips. This PR is the current replacement for closed PR #58216 and remains a narrow Discord runtime/test/changelog candidate rather than something already implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review or landing. The best fix is this PR or an equivalent narrow Discord plugin change that treats abort-time live `reconnect-exhausted` events as expected shutdown, returns `continue` to avoid lifecycle rejection, keeps focused regression coverage for the live event path after abort, and preserves the original #58216 contributor credit.
