## Summary

Keep open. The Telegram group/topic silent-drop and direct status symptoms are valid and current main still lacks several proposed paths, but this PR should not merge as-is because the new fallback visibly sends Telegram group output that the current `message_tool_only` privacy contract is supposed to suppress. The branch also includes unrelated gateway/session-list and performance changes, so it needs maintainer review and likely splitting.

## What This Changes

This PR changes Telegram native `/status`, group/topic reply fallback delivery, follow-up failure replies, Telegram update queueing/latency diagnostics, session-list usage fast paths, and focused tests across Telegram, auto-reply, Gateway, and agent utility files.

## Best Possible Solution

Keep the valid Telegram work active, but split or narrow it: land a minimal maintainer-approved fix for direct Telegram `/status`, topic-preserving failure replies, and any suppressed-source recovery that is explicitly gated by config or by message-tool unavailability, while moving session-list/performance diagnostics into separate PRs.

## Reproduction Assessment

Yes. The provided comments give a high-confidence OpenClaw 2026.4.27 Telegram group/topic silent-drop reproduction, and current main source shows why group finals are suppressed under the default `message_tool_only` policy. The PR finding is also reproducible from the PR's own fallback test: a suppressed plain final is delivered visibly through Telegram.

## Solution Assessment

No. The direct status and generic follow-up error pieces are narrow, but the proposed default suppressed-source fallback overrides the current group privacy contract and the PR bundles unrelated performance/session-list changes. A safer solution is a smaller fix with an explicit opt-in or narrower fallback condition plus separate review for the unrelated changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve message-tool-only group privacy:** `extensions/telegram/src/bot-message-dispatch.ts:1087-1092`
  - body: The new Telegram suppressed-source callback sends any fallback payload with `sendPayload`, so a plain assistant final in a default group/topic is posted visibly even when `sourceReplyDeliveryMode` is `message_tool_only`.
