## Summary

Keep open. Current main still leaves ZAI vision models on the configured provider-wide base URL, and #70915 is an open, unmerged PR with closing syntax for this issue, so the issue should not be closed before that PR lands or is rejected.

## What This Changes

The issue asks OpenClaw to make the built-in image tool work with ZAI vision models such as `zai/glm-4.6v` when ZAI is configured with a Coding Plan endpoint.

## Best Possible Solution

Keep this issue open while maintainers review #70915. The desired end state is a ZAI-owned media-understanding or onboarding fix that routes ZAI vision models from Coding Plan base URLs to the same-region standard multimodal endpoint, with regression tests proving text/coding models stay on Coding Plan URLs and image models use the standard URL.
