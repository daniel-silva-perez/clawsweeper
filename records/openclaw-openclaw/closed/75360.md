## Summary

Keep open. The PR is protected by the maintainer label and the current diff still introduces two contract regressions: `web_search` no longer advertises the selected provider/runtime schema, and generation tool availability now bypasses provider readiness checks that current main relies on. The latency direction is plausible, but the patch needs a contract-preserving revision before merge.

## What This Changes

The PR caches targeted provider runtime hook resolution, defers web/media provider and model resolution from tool registration to execution, and makes typing indicators asynchronous to reduce reply cold-start latency.

## Best Possible Solution

Keep the cold-start optimization direction, but preserve existing tool contracts with lightweight provider-owned descriptors for web_search and a generation availability path that honors provider `isConfigured`, default models, active runtime providers, and plugin config without broad cold-start scans.

## Reproduction Assessment

Partially. The Telegram latency claim was not live-reproduced here, but the review findings are reproducible by comparing the PR head against current main provider schemas, current registration tests, and generation readiness tests.

## Solution Assessment

No. Targeted provider runtime lookup and async typing are reasonable, but replacing selected web_search schemas with a fixed schema and bypassing provider readiness for generation tools is not the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve provider web_search tool schemas:** `src/agents/tools/web-search.ts:95`
  - body: The PR always advertises the fixed `WebSearchSchema`, but current main gets the description and parameters from the selected provider and has a runtime-provider regression test for that contract. This drops supported fields such as Exa's `type`/`contents` and SearXNG's `categories`, so the model cannot discover provider-specific controls even though execution can still accept them.
  - confidence: 0.92
- **[P2] Keep generation availability on provider readiness:** `src/agents/tools/media-tool-shared.ts:289-292`
  - body: `hasGenerationToolAvailability` only checks explicit defaults or bundled provider IDs with generic env/auth-profile auth. Current resolution also honors each provider's `defaultModel` and `isConfigured({ cfg, agentDir })`, so plugin-configured or agentDir-ready image/video/music providers can disappear at registration even though execution would resolve them.
  - confidence: 0.88
