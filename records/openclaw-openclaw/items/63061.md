## Summary

Keep this PR open. Current main still suffixes CLI profile state under OPENCLAW_HOME, while the generic config/state resolver defaults OPENCLAW_HOME to an unsuffixed .openclaw directory. The PR at https://github.com/openclaw/openclaw/pull/63061 remains a focused implementation candidate for the CLI profile/env surface, though maintainer product judgment is still needed because existing docs and help describe named profiles as separate state directories.

## Best Possible Solution

Keep this PR open for maintainer review. If OPENCLAW_HOME is intended to define the physical state root, land a normalized version of this PR and update CLI/help/docs plus any daemon/service-path helpers that should follow the same precedence. If named profiles must remain separate even inside OPENCLAW_HOME, reject the PR and document that profile suffixing wins for the combined case.
