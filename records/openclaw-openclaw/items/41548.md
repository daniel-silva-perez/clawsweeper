## Summary

Keep this issue open. Current main has shared session routing and Gateway session-event primitives, but it still does not implement the requested unified live inbox across TUI, Telegram, and iMessage. The TUI handles only chat, side-result, and agent event families, filters those to the active session, and does not subscribe to the Gateway session message/change streams that would be needed for cross-channel activity. Related #40678 and #51691 remain open coordination points, not shipped replacements.

## Best Possible Solution

Keep this open as a core TUI/Gateway product issue unless a maintainer manually consolidates it into #40678 or #51691. The likely implementation is an opt-in TUI observer or unified inbox mode that subscribes to Gateway `sessions.subscribe` / `session.message` events for a selected agent/session scope, renders source channel/session labels, and preserves existing delivery-route privacy rules.

## Risks / Open Questions

- A naive unified inbox could leak private DM or group context across local TUI, Telegram, and iMessage surfaces. Any implementation should be opt-in, source-labeled, and preserve pairing, allowlist, delivery-route, and DM-isolation rules.
- There is real overlap with #40678 and #51691. Maintainers may want to consolidate these into one canonical tracker, but the current evidence does not support closing this issue automatically as implemented or duplicate.
