## Summary

Keep this open. Current main has Gateway session subscription primitives and shared DM session routing, but the bundled TUI still does not subscribe to `sessions.subscribe` / `sessions.messages.subscribe`, does not dispatch `session.message` / `sessions.changed`, and filters visible chat events to the active session. The supplied related context also shows same-author issue #40678 is still open, #51691 was closed back to this broader tracker, and open maintainer PR #73402 is only a narrow active-session live transcript candidate rather than the full TUI/Telegram/iMessage unified inbox request.

## Best Possible Solution

Keep this open as the broader TUI/Gateway product tracker unless maintainers manually consolidate it with #40678. Review or replace #73402 for the narrow active-session live-update gap, then design an opt-in TUI observer or mirror mode using `sessions.subscribe` and/or `sessions.messages.subscribe`, with channel/session labels and explicit privacy boundaries for Telegram, iMessage, and other surfaces.

## Risks / Open Questions

- Naive cross-surface mirroring can leak private DM, group, or channel context across local TUI, Telegram, and iMessage; this needs opt-in behavior, clear channel/session labels, and privacy-aware routing boundaries.
- Open PR #73402 appears useful for the active-session live transcript gap, but treating it as the full unified inbox request would underdeliver the broader TUI/Telegram/iMessage sync behavior.
- The existing Gateway subscription primitives are per-session; a real unified inbox or observer mode needs product decisions about which sessions to observe and how to avoid noisy or surprising cross-channel fanout.
