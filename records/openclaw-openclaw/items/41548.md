## Summary

Keep this issue open. Current main has shared session routing, identity links/docking, and Gateway session event primitives, but it still does not implement the requested unified live inbox across TUI, Telegram, and iMessage. The TUI remains scoped to the active session and does not consume the Gateway session subscription events that would show cross-channel activity live.

## Best Possible Solution

Keep this open as a core TUI/Gateway product issue unless a maintainer manually consolidates it into #40678 or #51691. The likely implementation is an opt-in TUI observer or unified inbox mode that subscribes to Gateway `sessions.subscribe` / `sessions.messages.subscribe`, renders `session.message` and `sessions.changed` activity with channel/session labels, and preserves existing delivery-route privacy rules.
