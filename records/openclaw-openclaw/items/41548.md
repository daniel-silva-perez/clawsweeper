## Summary

Keep open. Current main has Gateway session subscription primitives and shared DM-session routing, but the bundled TUI still does not consume `session.message` or `sessions.changed` events and has no opt-in observer or unified inbox mode for Telegram/iMessage activity.

## What This Changes

The issue requests unified cross-channel message visibility and reply sync across TUI, Telegram, and iMessage surfaces.

## Best Possible Solution

Keep a canonical product tracker for an opt-in TUI mirror or observer mode that consumes the existing Gateway session subscription primitives, clearly labels channel/session origin, and preserves explicit reply-routing and privacy boundaries instead of restoring unconditional cross-client broadcast behavior.

## Reproduction Assessment

Yes, for the current missing implementation path: source inspection shows the TUI never subscribes to or dispatches Gateway `session.message` / `sessions.changed` events. A live Telegram/iMessage setup would demonstrate the same gap, but the code path is already deterministic enough to confirm it.

## Solution Assessment

No for unconditional full sync as stated; the safer maintainable direction is an explicit opt-in observer/unified-inbox mode built on `sessions.subscribe` and/or `sessions.messages.subscribe`. The narrow active-session work in #73402 should be reviewed separately and not treated as the full solution.
