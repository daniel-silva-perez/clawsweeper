## Summary

Current main still does not implement the requested unified live inbox across TUI, Telegram, and iMessage. The Gateway has session-level `session.message` and `sessions.changed` event primitives, and docs describe shared gateway-backed transcripts plus deterministic channel routing, but the TUI does not subscribe to or render those session events as cross-channel activity. Related #40678 and #51691 remain open coordination points rather than shipped replacements.

## Best Possible Solution

Keep this open as a core TUI/Gateway product issue unless a maintainer explicitly consolidates it into #40678 or #51691. The likely path is an opt-in TUI unified inbox or observer mode that subscribes to Gateway `session.message`/`sessions.changed` events for a selected agent/session scope, renders source channel/session labels, and keeps existing delivery-route privacy rules intact.
