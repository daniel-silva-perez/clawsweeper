## Summary

Keep this issue open. Current main has shared session routing, identity links, WebChat/session event primitives, and Gateway transcript broadcasts, but the TUI still does not implement the requested unified live inbox across TUI, Telegram, and iMessage. The related same-author issue #40678 and related #51691 are still open, so this should not be auto-closed without maintainer consolidation.

## Best Possible Solution

Keep this open as a core TUI/Gateway product issue unless a maintainer manually consolidates it into #40678 or #51691. The likely fix is an opt-in TUI observer or unified inbox mode that subscribes to Gateway `sessions.subscribe` and/or `sessions.messages.subscribe`, renders `session.message` and `sessions.changed` activity with channel/session labels, and preserves existing routing and privacy rules.
