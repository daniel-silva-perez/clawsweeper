## Summary

Keep this open. Current main has shared session routing and Gateway session-event primitives, but the TUI still does not implement the requested unified live inbox across TUI, Telegram, and iMessage. The same-author related issue #40678 and overlapping #51691 are still open, so this needs maintainer consolidation or implementation rather than an automated close.

## Best Possible Solution

Keep this open as a core TUI/Gateway product issue unless maintainers manually consolidate it into #40678 or #51691. The likely implementation is an opt-in TUI observer or unified inbox mode that uses `sessions.subscribe` and/or `sessions.messages.subscribe`, renders `session.message` and `sessions.changed` activity with channel/session labels, and keeps the current active-session TUI behavior as the default.
