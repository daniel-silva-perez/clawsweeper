## Summary

Keep open. Current main still lacks a Baileys `messaging-history.set` subscription or reconnect-history config, while the existing WhatsApp paths either handle live `messages.upsert` events or deliberately skip stale append backlog. Related history-store and offline-reply attempts are closed or unmerged, so there is no canonical merged replacement; #56088 is useful caution that on-demand fetch can be unreliable, not a fix for reconnect sync. ([baileys.wiki](https://baileys.wiki/docs/api/type-aliases/BaileysEventMap/))

## What This Changes

Request opt-in WhatsApp reconnect history sync that captures Baileys `messaging-history.set` messages and makes missed downtime messages available to agent context/history buffers.

## Best Possible Solution

Add an opt-in WhatsApp plugin reconnect-history path: subscribe to typed `messaging-history.set`, configure/allow the needed Baileys history-sync filter, normalize and dedupe synced messages through existing WhatsApp identity/access/history helpers, populate DM/group context without auto-replying to stale backlog by default, and document the exact sync-type behavior and limitations.
