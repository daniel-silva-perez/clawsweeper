## Summary

Keep open. Current main does not implement TTS auto-emotion, and the PR is an active implementation, but the latest head still has a concrete correctness issue: ElevenLabs configs that set only `voiceSettings.similarityBoost` suppress auto-emotion even though the PR only maps `style` and `stability` and documents those as the disabling fields.

## What This Changes

The PR adds opt-in `messages.tts.autoEmotion` inference, provider-specific TTS emotion mappings, provider/config schema support, docs, generated schema updates, and focused tests across speech-core and speech providers.

## Best Possible Solution

Keep the PR open and revise the ElevenLabs explicit-setting detection so only raw provider/persona/request `voiceSettings.style` or `voiceSettings.stability` suppress auto-emotion, then add a regression test for a similarityBoost-only ElevenLabs config and rerun the targeted TTS/config/docs checks from the PR plan.

## Reproduction Assessment

Yes for the review finding: configure ElevenLabs with `autoEmotion.enabled: true` and a raw provider or persona `voiceSettings` object containing only `similarityBoost`, then synthesize happy text. The PR's predicate returns explicit=true before building the auto `style`/`stability` override, so the feature is skipped.

## Solution Assessment

No, not as-is. The shared opt-in mapping approach is maintainable if maintainers want the feature, but the ElevenLabs override guard is broader than the provider fields this PR actually owns and broader than the docs describe.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Don't treat similarityBoost as an emotion override:** `extensions/speech-core/src/tts.ts:392-394`
  - body: With ElevenLabs, `hasElevenLabsVoiceSettings` returns true when the effective config only sets `voiceSettings.similarityBoost`. That field controls speaker similarity, while this PR's auto-emotion mapping only writes `style` and `stability`; a common config that pins similarity now disables auto-emotion entirely, contradicting the docs that say only explicit style or stability should opt out.
  - confidence: 0.88
