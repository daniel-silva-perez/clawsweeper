## Summary

Keep open. Current main does not implement automatic TTS emotion mapping, and the PR is useful but not merge-ready because the auto-emotion override path ignores persona-merged provider config, so explicit persona TTS tuning can still be overwritten by inferred provider overrides.

## What This Changes

The PR adds opt-in `messages.tts.autoEmotion`, speech-core heuristic emotion inference, provider-specific mappings for Volcengine, Xiaomi, OpenAI, ElevenLabs, Microsoft, and Azure Speech, config schema/type updates, generated schema changes, and focused provider/runtime tests.

## Best Possible Solution

Land the feature only after auto-emotion respects all explicit provider settings, including persona provider bindings, and after the public TTS docs plus config help/labels are aligned for the new config key.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. The current-main gap is high-confidence from source search, and the review finding is reproducible by enabling `autoEmotion` with a persona that sets provider-specific OpenAI instructions, ElevenLabs `voiceSettings`, or prosody settings.

## Solution Assessment

No, not as-is. An opt-in speech-core implementation can be maintainable if maintainers want this feature, but the patch should preserve persona provider config before applying inferred provider overrides.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Honor persona provider overrides before auto emotion:** `extensions/speech-core/src/tts.ts:1298`
  - body: `applyAutoEmotionProviderOverride` checks only the raw root provider config, even though `providerConfig` already includes persona provider bindings from `mergeProviderConfigWithPersona`. With `autoEmotion` enabled, a persona that explicitly sets OpenAI `instructions`, ElevenLabs `voiceSettings`, or Azure/Microsoft prosody still receives inferred emotion overrides, so persona-owned speech tuning can be overwritten instead of staying authoritative.
  - confidence: 0.86
