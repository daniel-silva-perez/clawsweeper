Keep open. Current main still has the reported allowlist/execution identity mismatch, and this PR is a focused implementation candidate rather than obsolete work. ## What This Changes

The PR changes `src/infra/exec-command-resolution.ts` to test path-shaped allowlist patterns against `resolvedPath` or a distinct `resolvedRealPath`, and adds four regression tests in `src/infra/exec-allowlist-matching.test.ts`.

## Best Possible Solution

Land this matcher fix or an equivalent focused change with the regression tests, while keeping the broader realpath trust issue open or explicitly narrowing the linked issue before merge. A complete canonical executable-identity fix would need separate maintainer-scoped work across allowlist matching, safe-bin trust, skill auto-allow, durable persistence, and audit metadata.
