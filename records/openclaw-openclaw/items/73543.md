## Summary

Current main still lacks the PR's intended behavior: path-shaped exec allowlist entries are matched against `resolvedPath` only, while resolution records `resolvedRealPath` and approval-backed execution can run that real path. The PR is a plausible focused fix, but its linked bug remains broader than the matcher-only change, so this is not obsolete and should stay open for maintainer review.

## Best Possible Solution

Keep this PR open and review it as the active focused fix candidate. Either land a scoped matcher fix with regression tests and narrow the closing/changelog language so #45595 remains open for the remaining trust surfaces, or expand the implementation into a coherent realpath trust model across matcher, safe-bin trust, skill auto-allow, Allow Always persistence, and audit metadata.
