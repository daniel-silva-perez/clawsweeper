## Summary

Keep open. Current main still has the exact matcher gap this PR targets: path-shaped exec allowlist entries are matched against `resolvedPath` only, while approval-backed execution can run `resolvedRealPath`. The PR is a plausible focused fix, but it is not obsolete on main and its linked #45595 scope is broader than the matcher-only change.

## Best Possible Solution

Review this PR as a focused bug-fix candidate. Before landing, either expand the fix to define a consistent realpath trust model across matcher, safe-bin trust, skill auto-allow, Allow Always persistence, and audit metadata, or narrow the PR description/changelog so #45595 remains open for the remaining surfaces. Add or keep the targeted symlink/realpath regression tests and make the symlink-pinned approval threat-model choice explicit.
