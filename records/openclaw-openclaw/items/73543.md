## Summary

Keep this PR open. Current main still matches path-shaped exec allowlist entries only against `resolvedPath`, while execution planning can pin the executable to `resolvedRealPath`. The PR is a plausible focused fix for the allowlist-matcher part, but it is security-adjacent and appears narrower than the linked bug's broader trust/persistence scope, so it needs maintainer review rather than cleanup closure.

## Best Possible Solution

Review this PR as a focused bug-fix candidate for path-shaped `tools.exec.allowlist` matching. Before landing, either expand it to address the linked bug's remaining realpath trust/persistence surfaces, or change the closing language so the broader issue remains open for safe-bin, skill auto-allow, allow-always persistence, and audit metadata follow-up. Maintainer security review should explicitly decide whether symlink-pattern allowlists are intended to keep matching retargeted binaries.
