## Summary

Current checked-out main at ab5c8025c9d0a31873d311ae3e73f3664b0a1185 still matches path-shaped exec allowlist entries against `resolvedPath` only, while command resolution records `resolvedRealPath` and approval-backed execution can pin argv to that real path. This PR is still a live, focused implementation candidate, but it should stay open for maintainer review because the linked bug also covers broader realpath trust surfaces beyond this matcher-only change.

## Best Possible Solution

Keep this PR open as the active focused fix candidate. Either land the scoped matcher fix with regression tests while narrowing the closing/changelog language so #45595 remains open for the remaining trust surfaces, or expand the implementation into a coherent realpath trust model across matcher, safe-bin trust, skill auto-allow, Allow Always persistence, and audit metadata.
