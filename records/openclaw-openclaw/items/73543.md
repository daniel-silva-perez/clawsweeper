## Summary

Keep this PR open. Current main still has the reported exec-approval identity mismatch: the allowlist matcher uses the PATH-resolved executable path, while the approval-backed execution plan can pin argv[0] to the realpath. The PR is a plausible focused fix, but it is not implemented on main and the linked #45595 scope is broader/security-adjacent, so it needs maintainer review rather than cleanup closure.

## Best Possible Solution

Review this PR as a focused matcher fix. Before landing, either narrow the PR body/changelog so #45595 remains open for the broader trust/persistence work, or expand the implementation and tests to cover the remaining realpath identity surfaces. Maintainer security review should explicitly document the intended semantics for symlink-pattern allowlists versus realpath-pinned allowlists.
