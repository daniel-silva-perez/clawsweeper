## Summary

Current main still does not run an agentic memory flush against the previous session before manual `/new`/`/reset`, daily reset, or idle reset. Main has adjacent mitigations: reset hooks for manual commands, transcript archiving for stale rollover, the bundled mechanical `session-memory` hook, and startup loading for slugged memory files. Those do not satisfy this issue's requested pre-reset curated flush, and several open PRs already cover overlapping implementation slices, so this should stay open for maintainer consolidation.

## Best Possible Solution

Keep this issue open as the canonical product/design thread for unified pre-reset memory persistence. Maintainers should consolidate the active PRs, then choose a bounded reset-time agentic flush design for manual `/new` and `/reset`, decide daily/idle rollover semantics, and keep the mechanical `session-memory` fallback plus plugin hooks aligned with the chosen contract.
