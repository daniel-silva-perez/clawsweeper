Current main still lacks the requested `forcePruneRatio` config/runtime/schema/docs surface, and the context-pruning hook still exits during a fresh cache-TTL window before estimating context usage. The linked implementation PR is closed unmerged, and no open PR currently references this issue with closing syntax.

## What This Changes

Add an optional `agents.defaults.contextPruning.forcePruneRatio` setting so high estimated context usage can bypass cache-TTL pruning delay.
