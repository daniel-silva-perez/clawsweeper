## Summary

Keep open. Current main still only has TTL-gated context pruning: there is no `forcePruneRatio` config, schema, runtime, docs, or regression coverage, and the related implementation PR #13572 is closed unmerged.

## What This Changes

Add an optional `agents.defaults.contextPruning.forcePruneRatio` setting so high estimated context usage can bypass the context-pruning TTL gate.

## Best Possible Solution

Implement the feature as an additive context-pruning option: accept, type, generate, and document `agents.defaults.contextPruning.forcePruneRatio`; estimate context usage before the fresh-TTL early return; bypass TTL only when a configured finite threshold is met; preserve existing TTL behavior otherwise; and cover both schema acceptance and runtime bypass regressions.
