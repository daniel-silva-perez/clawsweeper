## Summary

Keep open. Current `main` still lacks the requested `forcePruneRatio` setting in runtime settings, config schema, generated schema, docs, and regression coverage; the context-pruning hook still exits during a fresh cache-TTL window before usage is estimated, and related PR #13572 is closed unmerged.

## What This Changes

Add an optional `agents.defaults.contextPruning.forcePruneRatio` setting that lets high estimated context usage bypass cache TTL and trigger context pruning immediately, with schema/docs/tests.

## Best Possible Solution

Implement the knob as an additive config field, carry it through effective settings/runtime setup, estimate usage before the cache-TTL early return, bypass TTL only when a configured finite threshold is met, preserve existing behavior when unset or below threshold, and align generated schema plus public docs.
