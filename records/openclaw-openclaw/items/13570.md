## Summary

Keep open. Current main still has only TTL-gated context pruning: the requested `forcePruneRatio` config/schema/runtime/docs surface is absent, the extension still returns inside a fresh TTL window before usage is estimated, and the paired implementation PR is closed unmerged.

## What This Changes

Add an optional `contextPruning.forcePruneRatio` setting so high estimated context usage can bypass the cache TTL and run pruning immediately.

## Best Possible Solution

Implement the feature as an additive context-pruning config knob: accept and document `agents.defaults.contextPruning.forcePruneRatio`, carry it through effective settings/runtime setup, estimate usage before the fresh-TTL early return, bypass TTL only when known usage meets the configured threshold, preserve existing TTL behavior otherwise, and cover schema plus runtime regressions.
