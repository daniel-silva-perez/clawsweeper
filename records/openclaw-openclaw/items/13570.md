## Summary

Keep this issue open. Current main still has no `contextPruning.forcePruneRatio` public config field or equivalent high-usage TTL bypass, and the context-pruning hook still exits during a fresh TTL window before pruning can inspect estimated context usage. The related PR #13572 is closed unmerged, so it remains useful reference material rather than a shipped fix.

## Best Possible Solution

Implement the additive feature in current main: add `agents.defaults.contextPruning.forcePruneRatio` to the public config type, strict Zod schema, generated schema/help, docs, and runtime settings; estimate context usage before the fresh-TTL early return; bypass the TTL only when usage is at or above the configured threshold; preserve current TTL behavior below threshold or without a known context window; and add focused regression coverage. Use #13572 as reference only because it is closed unmerged.
