## Summary

Keep this issue open. Current main still has no `contextPruning.forcePruneRatio` config field or equivalent high-usage TTL bypass, and the context-pruning hook still returns during a fresh TTL window before pruning can inspect context usage. The linked implementation PR #13572 remains closed unmerged and targeted the older `pi-extensions` path, so it is useful reference material but not a shipped fix.

## Best Possible Solution

Implement the feature in current `main`: add `agents.defaults.contextPruning.forcePruneRatio` to the public config type, strict Zod schema, generated schema/help, and docs; thread it through `computeEffectiveSettings()` and `ContextPruningRuntimeValue`; estimate context usage before the fresh-TTL early return; bypass TTL only when usage is at or above the configured threshold; preserve existing TTL behavior below threshold or when the context window is unknown; and add focused regression tests for both bypass and non-bypass paths. Use #13572 as reference only, since it is closed unmerged and targets old paths.
