## Summary

Keep open. Current main still lacks `contextPruning.forcePruneRatio` or an equivalent high-usage TTL bypass, and the context-pruning hook still exits during a fresh TTL window before estimating usage. The directly related implementation PR #13572 is closed unmerged.

## What This Changes

Add an optional `agents.defaults.contextPruning.forcePruneRatio` setting that bypasses cache-TTL pruning delay when estimated context usage is above the configured threshold, with schema/docs/runtime tests.

## Best Possible Solution

Implement the additive `agents.defaults.contextPruning.forcePruneRatio` feature in the context-pruning owner path: accept and document the config key, carry it through effective runtime settings, estimate usage before the fresh-TTL early return, bypass TTL only when known usage is at or above the threshold, preserve existing TTL behavior otherwise, and add focused schema/runtime regression coverage. Use closed PR #13572 as reference, not as a merge source.
