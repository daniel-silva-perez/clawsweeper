## Repair Work Prompt

Implement the requested `contextPruning.forcePruneRatio` enhancement for `agents.defaults.contextPruning`. Add the optional field to config/runtime types, Zod schema, generated schema/docs surface, and effective settings. In the context-pruning hook, estimate context usage before the fresh TTL early return and bypass TTL only when a finite known usage ratio is greater than or equal to the configured threshold; preserve current TTL behavior when the setting is absent, invalid, or usage cannot be estimated. Prefer reusing/refactoring the existing character-window estimation logic in `src/agents/pi-hooks/context-pruning/pruner.ts` rather than duplicating brittle logic. Add focused regression coverage for schema acceptance and for a fresh-TTL window that still prunes when over threshold, plus coverage that a fresh TTL still suppresses pruning below threshold. Do not change provider defaults or make the new setting enabled by default unless maintainers explicitly ask. Credit related PR #13572 in the replacement PR body if using it as reference work.
