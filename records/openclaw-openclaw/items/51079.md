## Summary

Keep this PR open. Current main still passes the persistent session id as the node `agent.request` run id, and the focused test still asserts that old behavior. The PR’s intended change is not implemented or superseded on main, and the provided diff is a narrow gateway runtime/test change with no visible supply-chain or secrets-handling expansion.

## Best Possible Solution

Review and land this as a focused gateway bug fix if validation passes. The best implementation is to generate a fresh `randomUUID()` for each node `agent.request` run while preserving the existing `sessionId` for session continuity, update the colocated regression test to assert that the run id is distinct from the session id, and run the targeted `src/gateway/server-node-events.test.ts` coverage plus the normal changed gate.
