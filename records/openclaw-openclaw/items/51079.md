## Summary

Keep this PR open. Current main still reuses the persistent session id as the node `agent.request` run id, and the focused test still asserts that old behavior. The PR’s intended change is not implemented or superseded on main; the provided diff is a narrow gateway runtime/test change with no visible supply-chain, secrets, workflow, dependency, or publishing surface expansion.

## Best Possible Solution

Keep this PR open and review it as a focused gateway bug fix. The best implementation is to generate a fresh `randomUUID()` for each node `agent.request` run while preserving the existing `sessionId` for session continuity, update the colocated regression test to assert the run id is distinct from the session id, and run the targeted gateway test plus the normal changed gate before landing.
