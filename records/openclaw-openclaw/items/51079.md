## Summary

Keep this PR open. Current main still passes the persistent session id as the node `agent.request` run id, and the focused regression test still asserts that old behavior. The PR is a narrow gateway runtime/test fix, and I did not see a security or supply-chain reason to close it as obsolete.

## Best Possible Solution

Review and land this as a focused gateway bug fix if validation passes. The best implementation is to generate a fresh `randomUUID()` for each node `agent.request` run while preserving the existing `sessionId` for session continuity, update the colocated regression test to assert that the run id is distinct from the session id, and run `pnpm test src/gateway/server-node-events.test.ts` plus the normal changed gate.
