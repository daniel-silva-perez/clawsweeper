## Summary

Keep this PR open. Current main still reuses the persistent session id as the node `agent.request` run id, and the focused regression test still asserts that old behavior. The PR’s intended change is not implemented or superseded on main, and the provided diff is a narrow gateway runtime/test patch with no visible supply-chain, secrets, workflow, or dependency expansion.

## Best Possible Solution

Keep this PR open and review it as a focused gateway bug fix. The best path is to generate a fresh `randomUUID()` for each node `agent.request` run while keeping the existing `sessionId` for session continuity, update the colocated test to assert the run id is distinct from the session id, and run the targeted gateway test plus the normal changed gate before landing.
