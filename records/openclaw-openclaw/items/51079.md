## Summary

Keep this PR open. Current main still sends node `agent.request` runs through `agentCommandFromIngress` with `runId: sessionId`, and the focused gateway test still asserts that old behavior. The PR is a narrow runtime/test change that matches the existing voice-ingress pattern and I did not find a security or supply-chain reason to close it as obsolete.

## Best Possible Solution

Review and land this as a focused gateway bug fix if validation passes. The best implementation is to generate a fresh `randomUUID()` for each node `agent.request` run while preserving `sessionId` for session continuity, update the colocated regression test to assert the run id differs from the session id, and then run the targeted gateway test plus the normal changed gate.
