## Summary

Keep open. Current main still does not implement inter-session progress streaming for long-running `sessions_send`: the tool exposes only fire-and-forget or wait modes, wait expiry still returns `status: "timeout"`, A2A announce remains completion-only, and the gateway has no agent-callable run-event read or subscription method. The related Task Flow and ACP parent-completion work are useful adjacent surfaces but do not let a caller observe a peer run started through `sessions_send`.

## What This Changes

Request inter-session progress streaming or subscription for long-running `sessions_send` runs so callers can distinguish accepted delivery from wait timeout and observe tool, assistant, and lifecycle progress.

## Best Possible Solution

Add an additive, permission-checked core run-event observation surface, likely backed by bounded per-run event buffering, then expose it through either a `sessions_subscribe` tool or an explicit `sessions_send` progress option. The implementation should also separate accepted delivery from wait timeout and cover long-running sends, event isolation, ACP/Codex behavior, and topic-bound delivery.
