## Summary

Keep this PR open for maintainer rework. Current main has moved thinking support to provider-owned profiles, and OpenAI/OpenAI Codex/GitHub Copilot direct xhigh support is implemented there, but the PR’s Vercel AI Gateway proxy-ref use case is still missing. The proposed core fallback is now stale, and the PR also includes unrelated plugin/web-search/channel churn with unresolved review concerns.

## Best Possible Solution

Rework the PR around the current provider-profile architecture: add a Vercel AI Gateway `resolveThinkingProfile` that recognizes trusted OpenAI/Codex upstream refs, add focused regression coverage for accepted Vercel OpenAI xhigh refs and rejected unsupported or untrusted namespaced refs, and drop or split the unrelated web-search/plugin-loader/channel test churn before review.
