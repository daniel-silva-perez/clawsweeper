## Summary

Keep this PR open for rework. Current main has moved thinking support to provider-owned profiles, and direct OpenAI/OpenAI Codex/GitHub Copilot xhigh paths are covered there, but the PR’s Vercel AI Gateway proxy-ref case is still not implemented: Vercel advertises `openai/gpt-5.4` and `openai/gpt-5.4-pro` as reasoning models but does not expose a `resolveThinkingProfile`/xhigh hook, while `sessions.patch` still validates explicit thinking levels through provider profile support. The PR’s old core fallback approach is now stale and its extra plugin/web-search/channel churn should be dropped or split before any merge.

## Best Possible Solution

Keep this PR open for rework. Rebase onto current main, drop or split unrelated web-search/plugin/channel churn, and implement the remaining useful behavior in the Vercel AI Gateway provider via a `resolveThinkingProfile` that recognizes trusted OpenAI/Codex upstream refs. Add focused regression coverage for accepted Vercel OpenAI xhigh refs and rejected unsupported or untrusted namespaced refs; if generic provider-alias support is still desired, define the trusted alias policy explicitly instead of using an unconditional trailing-id fallback.
