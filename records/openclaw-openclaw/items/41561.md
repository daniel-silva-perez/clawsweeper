## Summary

Keep this PR open. Current main has superseded the old core xhigh allowlist design with provider-owned thinking profiles, and direct OpenAI/OpenAI Codex/GitHub Copilot xhigh paths are now handled by their providers. The PR’s stated Vercel AI Gateway proxy-ref case is still not implemented, though: Vercel exposes OpenAI upstream refs in its catalog but does not advertise an xhigh thinking profile, and session patch validation still depends on that provider profile support. The PR should be reworked rather than closed or merged as-is because its core fallback approach is stale and its file list includes unrelated/stale plugin-loader and web-search discovery churn with unresolved security-sensitive review concerns.

## Best Possible Solution

Keep the PR open for rework. Rebase onto current main, drop unrelated channel/web-search/test churn, and implement the remaining useful behavior through provider-owned thinking profiles, likely a Vercel AI Gateway `resolveThinkingProfile` that recognizes trusted OpenAI/Codex upstream refs. Add focused regression coverage for accepted Vercel OpenAI xhigh refs and rejected unsupported or untrusted namespaced refs. If provider-alias support is still desired, define the trusted alias policy explicitly rather than using a broad trailing-id fallback.

## Risks / Open Questions

- Closing this PR as implemented would lose the remaining Vercel AI Gateway proxy-ref behavior; current main still does not advertise `xhigh` for Vercel OpenAI upstream refs.
- Merging this PR as-is would route xhigh through a stale core fallback instead of the provider-owned `resolveThinkingProfile` seam now used on main.
- The PR includes unrelated/stale plugin-loader and web-search discovery changes. Those are code-loading and provider-discovery surfaces, and the provided review comments identify module-load/export failures that need to be split or dropped.
- A generic trailing-model-id fallback needs a clear trusted-alias policy; otherwise it can approve xhigh for provider/model combinations that are not actually supported downstream.
