## Summary

Keep this PR open. Current main has superseded the old core allowlist with provider-owned thinking profiles and already covers direct OpenAI, OpenAI Codex, and GitHub Copilot xhigh paths, but it still does not implement the PR’s stated Vercel AI Gateway proxy-ref case. The PR should be reworked rather than merged as-is because its core fallback design is stale and its file list includes unrelated loader/web-search churn with unresolved security-sensitive review concerns.

## Best Possible Solution

Keep the PR open for rework. Rebase onto current main, drop unrelated channel/web-search/test churn, and implement the remaining useful behavior through provider-owned thinking profiles, likely a Vercel AI Gateway `resolveThinkingProfile` that recognizes trusted upstream OpenAI/Codex refs or a narrow shared helper for trusted proxy refs. Add focused regression coverage for accepted Vercel OpenAI xhigh refs and rejected unsupported or untrusted namespaced refs.
