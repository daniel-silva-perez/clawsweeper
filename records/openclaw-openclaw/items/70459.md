## Summary

Keep this draft PR open. Current main still lacks the proposed prerequisite section, but the PR is not merge-ready because its runnable Bash and PowerShell checkers fail to show the promised missing-dependency marker and its auto-install wording overstates installer behavior.

## What This Changes

The PR expands `docs/start/getting-started.md` with a prerequisite checklist, Step 0 verification snippets for macOS/Linux and Windows, per-platform install hints, and a note that `pnpm` is only needed for source builds.

## Best Possible Solution

Update this draft in place so the docs accurately describe installer-scoped remediation, use explicit command availability/status checks for both shells, and publish only after target-shell and Mintlify rendering validation.

## Reproduction Assessment

Yes for the review findings. The Bash failure reproduces with the proposed pipeline and a deliberately missing command; the PowerShell issue follows from routing through a shell command whose nonzero exit does not enter `catch`, though native Windows validation should still be completed.

## Solution Assessment

No, not as written. The docs direction is useful, but the current snippets and installer claims are not the narrowest maintainable solution until they match the actual installer contracts and shell behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Check command status before piping to head:** `docs/start/getting-started.md:46`
  - body: The fallback is attached to `head`, not the checked command. A missing command prints the shell error and the loop never emits the promised `MISSING` marker, so the diagnostic snippet gives unclear results for exactly the case it is meant to handle.
  - confidence: 0.96
- **[P2] Probe executables before using cmd:** `docs/start/getting-started.md:65`
  - body: `cmd /c` can return a nonzero status for a missing tool without entering PowerShell `catch`, so Windows users also do not get the promised `MISSING` marker. Use `Get-Command` or explicit status handling before running each command.
  - confidence: 0.88
- **[P2] Scope the auto-install claim to supported dependencies:** `docs/start/getting-started.md:38`
  - body: The docs say the installer auto-installs anything missing, but the installers only remediate specific setup pieces such as Node, Git, and source-install pnpm; bootstrap dependencies like curl/wget still have to exist. This overpromises first-run behavior.
  - confidence: 0.93
