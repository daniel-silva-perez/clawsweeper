## Summary

Keep open. Current main still does not implement the PR's bounded same-auth-profile retry for transient overloaded LLM errors. The runner rotates auth profiles, optionally backs off before rotation/fallback, then falls through to model fallback or surface-error. The PR remains a focused implementation candidate, with the related cooldown split in #49807 and broader configurable retry work still tracked by #49376.

## Best Possible Solution

Keep the PR open as an implementation candidate. Rebase it onto current main, preserve the bounded overloaded same-auth-profile retry for both prompt and assistant paths, add or keep behavioral coverage, normalize retry-sleep aborts as `AbortError`, and decide whether the hardcoded narrow retry should land before or be folded into #49376.

## Risks / Open Questions

- The PR patch still awaits the new same-profile retry `sleepWithAbort(...)` directly, matching the unresolved review concern that cancellation during retry sleep may throw a plain error instead of an `AbortError`.
- The patch was written against an older runner shape; current main has assistant failover split into `src/agents/pi-embedded-runner/run/assistant-failover.ts`, so it likely needs a rebase and integration cleanup.
- The retry policy is hardcoded in this PR, while #49376 asks for configurable LLM retry behavior; maintainers should decide whether to land this bounded fix first or fold it into the broader config design.
