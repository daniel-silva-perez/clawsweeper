## Summary

Current main at e583db63c602d8d21b4ebdcd119061571a2e40a9 still does not implement this PR's bounded same-auth-profile retry for transient overloaded LLM errors. The prompt and assistant failover paths classify overloaded errors and then rotate profiles, apply the configured overload failover backoff, fall back to another model, or surface the error. The requested retry_same_profile decision, same-profile retry policy/counter, and overloaded retry test are absent, so this PR remains a useful implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, fit bounded same-auth-profile overload retry into both the prompt path and the extracted assistant-failover module, preserve AbortError cancellation behavior, add/refresh behavioral coverage for overloaded-overloaded-success on the same auth profile, and explicitly decide how this narrow policy relates to the broader configurable retry work in #49376 and cooldown behavior from #49807.
