## Summary

Keep open. Current main still does not implement the PR's bounded same-auth-profile retry for transient overloaded LLM errors before profile rotation, model fallback, or surfacing the error. The PR remains a focused implementation candidate, while #49376 still tracks the broader configurable retry policy.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, integrate bounded same-auth-profile retry for transient overloaded errors in both prompt and assistant failover paths, preserve behavioral coverage for two overloaded attempts followed by success, normalize retry-sleep cancellation as AbortError, and decide whether the hardcoded policy should remain narrow or be folded into #49376.
