## Summary

Current main at ef58307f843eed1a7832e9c23c519591f1b7cdbd still does not implement this PR's bounded same-auth-profile retry for transient overloaded LLM errors. The prompt and assistant failover paths classify overloaded errors and then rotate profiles, apply overload failover backoff, fall back to another model, or surface the error. The requested retry_same_profile decision, same-profile retry policy/counter, and overloaded retry test are absent, so this PR remains a useful implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, fit bounded same-auth-profile overload retry into both the prompt path and the extracted assistant-failover module, preserve AbortError cancellation behavior, add or refresh behavioral coverage for overloaded-overloaded-success on the same auth profile, and explicitly decide how this narrow policy relates to the broader configurable retry work in #49376 and cooldown behavior from #49807.
