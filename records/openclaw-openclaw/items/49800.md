## Summary

Keep open. Current main still does not implement this PR's bounded same-auth-profile retry for transient overloaded LLM errors. The prompt and assistant paths still route overloaded failures through auth-profile rotation, optional overload failover backoff, model fallback, or surfaced error, and the advertised retry_same_profile observation/test are absent. The PR remains a focused implementation candidate, with the broader configurable retry policy still tracked separately by #49376.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, implement bounded same-auth-profile retry for transient overloaded errors in both prompt and assistant failover paths, preserve abort semantics, add/keep behavioral coverage for two overloaded attempts followed by success, and document how the narrow policy relates to #49376.
