## Summary

Keep this PR open. Current main still lacks operational-alert filtering in the delivery-mirror transcript resolver, so the PR is not obsolete, but the proposed patch still needs revision because its Spanish warning pattern can silently drop legitimate mirrored assistant text.

## What This Changes

The PR adds operational-alert pattern filtering to `resolveMirroredTranscriptText` and regression coverage so known backup/git alert mirrors are skipped while ordinary delivery-mirror text remains appendable.

## Best Possible Solution

Revise this PR so the destructive filter matches only documented non-user-facing operational alerts, ideally by constraining the backup/git signature or moving the suppression closer to the alert producer, while leaving broader queue replay duplicate handling to #51628.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Narrow the Spanish alert pattern to the backup signal:** `src/config/sessions/transcript-mirror.ts:35`
  - body: The new `^⚠️\s*error en\b` pattern still runs for every delivery mirror and drops any mirrored text that starts with `⚠️ Error en...`, not just the reported `backup-git` conflict alert. Because the caller turns this into `empty text`, a legitimate Spanish warning-style assistant reply would disappear from the transcript. Please constrain this match to the known backup/conflict signature and add a prefixed false-positive test.
  - confidence: 0.88
