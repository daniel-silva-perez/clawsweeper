## Summary

Keep open: current main still persists arbitrary non-empty delivery mirror text, so the PR is not obsolete or implemented elsewhere. The branch addresses a plausible residual transcript-contamination path, but it needs maintainer review because the proposed generic mirror filter can silently drop legitimate user-visible mirror text and the backup/git-specific filtering boundary may belong closer to the alert producer.

## What This Changes

The PR adds operational-alert pattern filtering to `resolveMirroredTranscriptText` plus transcript regression coverage so known backup/git alert mirrors are skipped while normal delivery mirrors remain unchanged.

## Best Possible Solution

Revise and then land a narrow, regression-tested suppression for non-user-facing operational alert mirrors, preferably constrained to the known backup/git signature or to a producer-owned marker if maintainers do not want owner-specific strings in the generic transcript mirror layer.
