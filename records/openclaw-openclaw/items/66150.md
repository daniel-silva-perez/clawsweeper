## Summary

Keep open. Current main and the latest release still persist arbitrary delivery-mirror text without operational-alert suppression, so this PR is not obsolete; however the proposed generic text filter can silently discard legitimate mirrored assistant content and needs maintainer review before merge.

## What This Changes

The PR adds operational-alert pattern filtering to `resolveMirroredTranscriptText` and transcript tests for backup/git alert suppression while preserving ordinary mirror text.

## Best Possible Solution

Revise and land a narrow, regression-tested suppression for non-user-facing operational alert mirrors, preferably constrained to the known backup-git conflict signature or to a producer-owned marker, while preserving ordinary delivery-mirror transcript content and leaving broader replay recovery work to #51628.
