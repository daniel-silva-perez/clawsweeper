## Summary

Keep this PR open. Current main and the latest shipped release still persist arbitrary delivery-mirror text without an operational-alert ingress filter, so the PR is not obsolete; however, the proposed filter still has a patch-level false-positive risk because the Spanish warning pattern is not constrained to the reported backup/git alert shape.

## What This Changes

This PR adds operational-alert pattern filtering to `resolveMirroredTranscriptText` and regression tests so backup/git alert mirrors are not persisted as delivery-mirror transcript text.

## Best Possible Solution

Revise and then land a narrow, regression-tested suppression for non-user-facing operational alert mirrors, preferably constrained to the known backup/git signature or to a producer-owned marker if maintainers do not want owner-specific strings in the generic transcript mirror layer.

- The alert strings are backup/git-owner-specific, so maintainers should decide whether suppression belongs in the generic transcript mirror resolver or closer to the alert producer.
- Broader delivery queue replay and duplicate mirror behavior remains tracked separately by #51628 and is not solved by this narrow PR.
