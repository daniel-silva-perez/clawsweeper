## Summary

Keep this PR open. Current main still mirrors arbitrary non-empty delivery text into persisted `delivery-mirror` transcript entries, and the existing replay hygiene only removes those entries later from provider replay. The PR remains a focused implementation candidate, but it needs maintainer review because the proposed filtering is content-dropping and uses backup/git-specific alert strings in a generic transcript mirror helper.

## What This Changes

The PR adds operational-alert pattern filtering to `resolveMirroredTranscriptText` and regression coverage so known stale backup/git alert mirrors are not persisted as assistant transcript text.

## Best Possible Solution

Land or revise a narrow, regression-tested suppression for non-user-facing operational alert mirrors, with maintainer agreement on whether the filter belongs in the generic transcript mirror helper or at the backup/git alert producer.
