## Summary

Keep open. Rechecking current main at 2d53b49b shows the Feishu-specific gap from the issue discussion is still present: `/activation mention` persists generic session activation state, but Feishu's pre-dispatch group mention gate still derives `requireMention` only from config before the later session activation path can affect behavior.

## What This Changes

The issue requests that Feishu group chats honor `/activation mention` by suppressing ordinary non-mention group messages after the activation mode is changed.

## Best Possible Solution

Implement the behavior in the Feishu plugin by resolving effective group activation before the Feishu hard mention gate, with persisted session `groupActivation` overriding Feishu config/defaults for the admitted group session while preserving group access, sender allowlists, pending-history, broadcast, and topic/thread behavior.
