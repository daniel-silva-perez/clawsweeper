## Summary

Keep open. Current main still matches the Feishu-specific gap described in the discussion: `/activation mention` stores generic group activation state, but Feishu decides whether to admit non-mentioned group messages from config-only `requireMention` before the later session activation state can affect behavior.

## What This Changes

The issue reports that Feishu group chats confirm `/activation mention` but continue responding to ordinary group messages that do not mention the bot.

## Best Possible Solution

Implement the behavior in the Feishu plugin by resolving effective group activation for the admitted target session before the hard mention gate, using config-derived `requireMention` as the default and persisted `/activation` state as the per-session override, with docs/tests updated to match the chosen supported surface.
