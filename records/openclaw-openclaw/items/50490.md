## Summary

Keep open. Current main still matches the reported Feishu bug: the Feishu plugin’s hard group mention gate is resolved from Feishu config before route/session state is available, while `/activation mention` only writes the generic session `groupActivation` state that is consumed later in the reply path. Hollychou924’s root-cause comment and workaround remain consistent with current source.

## What This Changes

The issue requests that Feishu group `/activation mention` make the bot ignore ordinary non-@ group messages after the command succeeds.

## Best Possible Solution

Keep this open and fix it in the Feishu plugin: resolve effective group activation for a Feishu group session as session override over Feishu config/default before the hard mention gate, preserve existing access-control and broadcast behavior, add Feishu regression coverage for `/activation mention`, and update docs/changelog if the supported Feishu surface changes.
