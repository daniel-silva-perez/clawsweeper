## Summary

Keep open. Current main still has the Feishu-specific gap described in the discussion: `/activation` stores per-session `groupActivation`, but Feishu's hard pre-dispatch mention gate derives `requireMention` from config-only policy before that session state can affect routing.

## What This Changes

The issue asks for Feishu group `/activation mention` to make the bot stop responding to ordinary unmentioned group messages after the command reports success.

## Best Possible Solution

Implement the behavior inside the Feishu plugin by resolving effective per-session group activation before the hard mention gate, using config-derived `requireMention` as the default and persisted `/activation` state as the override; if Feishu activation is not intended to be supported, the command should return an explicit unsupported response instead of a success message.
