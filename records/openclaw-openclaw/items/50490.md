## Summary

Keep open. Current main still matches the Feishu-specific gap described in the discussion: `/activation` persists group activation in session state, but Feishu's hard pre-dispatch mention gate still derives `requireMention` from config-only policy and returns before that session activation can affect routing.

## What This Changes

Feishu group chats should honor `/activation mention` by suppressing non-mentioned group messages after switching out of always-on mode.

## Best Possible Solution

Implement this inside the Feishu plugin by resolving effective group activation before the hard mention check: use config-derived `requireMention` as the default, let persisted `/activation` state override it for the target session, and add focused Feishu regression coverage plus docs/changelog clarification if the supported Feishu surface changes.

## Repair Work Prompt

Fix the Feishu group activation regression tracked by openclaw/openclaw#50490. Current main admits or drops Feishu group messages through a config-only `requireMention` gate in `extensions/feishu/src/bot.ts` before generic `sessionEntry.groupActivation` is read, so `/activation mention` can report success while non-mentioned messages still dispatch when config has `requireMention: false`.
