## Summary

Keep open. Current main still has the Feishu-specific activation gap described in the discussion: `/activation mention` persists generic session activation state, but Feishu's hard group-message admission gate still derives `requireMention` from configuration before routed session activation is consulted.

## What This Changes

Feishu group `/activation mention` should make the bot ignore ordinary unmentioned group messages after the command succeeds.

## Best Possible Solution

Implement the behavior in the Feishu plugin: after group and sender allowlist checks, resolve the effective routed group session and use persisted `groupActivation` as the per-session mention-admission override, with config-derived `requireMention` remaining the default. Add focused Feishu tests plus docs/changelog updates so command confirmation matches runtime behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: with Feishu config allowing unmentioned group messages, `/activation mention` persists `groupActivation`, but the next unmentioned group message is admitted by a config-only Feishu gate before that session state is read.

## Solution Assessment

Yes, the narrowest maintainable solution is Feishu-owned admission logic, aligned with Telegram/WhatsApp activation gating patterns. A prompt-only change or core startup migration would leave the hard Feishu gate disconnected from the persisted command state.
