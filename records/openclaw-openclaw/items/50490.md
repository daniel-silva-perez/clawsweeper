## Summary

Keep open. Current main still has the Feishu-specific gap described in the issue discussion: `/activation mention` persists generic session activation state, but Feishu's pre-dispatch group mention gate still resolves only config-derived `requireMention` before route/session state is available.

## What This Changes

The issue requests that Feishu group chats honor `/activation mention` by suppressing replies to ordinary non-mention group messages after the activation mode is changed.

## Best Possible Solution

Implement the behavior in the Feishu plugin by resolving the effective group activation before the hard mention gate, with session `groupActivation` overriding Feishu config/defaults for the admitted group session while preserving existing group access, sender allowlist, pending-history, broadcast, and topic-thread behavior.
