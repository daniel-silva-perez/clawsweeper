## Summary

Current main does not fix the reported Feishu behavior. Feishu still resolves the hard group mention gate from config-level `requireMention` before dispatch, while `/activation mention` only persists `sessionEntry.groupActivation` inside the core reply command path and turns it into prompt guidance. That means the root-cause analysis in the issue discussion still matches the current code.

## Best Possible Solution

Keep this issue open. The best fix is Feishu-owned: either look up the target group session before the hard mention gate and treat `groupActivation: "mention"` as a hard require-mention override, or add a generic plugin/session activation seam that lets channel plugins enforce session activation before dispatch. Add a Feishu regression test for `/activation mention` followed by an unmentioned group message. If maintainers instead decide `/activation` is WhatsApp-only, update command handling/docs so Feishu does not report a successful mode switch that it cannot enforce.
