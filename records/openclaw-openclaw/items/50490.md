## Summary

Keep open. Current main still matches the reported Feishu-specific gap: the Feishu group pre-dispatch gate derives mention requirements from config-only policy, while `/activation mention` only persists generic session activation that is consumed later in the reply path. The discussion already identified this same split and provided a config workaround, and no linked implementation PR is present in the provided context.

## What This Changes

Feishu group chats should honor `/activation mention` by no longer responding to ordinary non-mentioned messages after the command reports success.

## Best Possible Solution

Implement Feishu's effective group activation at the plugin boundary: keep config-derived `requireMention` as the default, apply persisted per-session `/activation` state as the override before the hard mention gate, and update Feishu/group docs plus targeted tests to make the supported behavior explicit.
