## Summary

Keep open. Current main still matches the Feishu-specific gap: `/activation mention` persists generic session activation state, but Feishu still admits or drops group messages using config-derived `requireMention` before that session activation can affect the hard routing decision. The recent channel-turn refactor routes dispatch through the shared kernel, but Feishu supplies no preflight/admission check for `groupActivation`, so unmentioned messages still reach dispatch when config says mentions are not required.

## What This Changes

The issue requests that Feishu group chats honor `/activation mention` by requiring a bot mention after the command instead of continuing to respond to unmentioned group messages.

## Best Possible Solution

Make Feishu's effective group activation internally consistent: after group and sender allowlist checks, resolve the routed group session's persisted `/activation` mode and use it as the per-session override for mention admission, then update Feishu/group docs and tests so the command's confirmation reflects real behavior.
