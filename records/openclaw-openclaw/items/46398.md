## Summary

Keep this issue open. Current main still only shows context usage in the Control UI/WebChat when fresh session usage is at or above the high-pressure threshold, plus a separate draft token estimate for the text being typed. The requested persistent current-session usage indicator is not present yet. A maintainer-labeled open PR, #73744, directly carries this request and should be reviewed/landed or closed before this issue is resolved.

## Best Possible Solution

Keep this issue open and use it as the canonical tracker for the Control UI/WebChat persistent context usage indicator. The best path is to review and either land or close #73744; after a merge, this issue can close as implemented with the landed SHA/release provenance.
