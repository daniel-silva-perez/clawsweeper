## Summary

Keep open. Current main still has the shared same-target message-tool suppression path that can drop a distinct final reply after a file/media send, and the maintainer discussion explicitly calls that remaining path valid. Open PR #59752 appears to be the canonical fix path rather than a reason to close this issue before merge.

## What This Changes

Telegram should deliver or explicitly fail a user-visible final text reply after mixed thinking/text and same-target message-tool file/media sends instead of recording it in transcript without an outbound send.

## Best Possible Solution

Keep this issue open and resolve the remaining shared reply-payload bug by reviewing and landing or superseding #59752. The fix should narrow same-target message-tool suppression to exact duplicate text/media/caption deliveries, preserve distinct final text after file/media sends, update buildReplyPayloads and follow-up delivery, and replace tests that currently assert blanket suppression while keeping the Telegram reasoning/commentary fixes intact.
