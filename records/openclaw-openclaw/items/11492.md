## Summary

Current main still does not expose Slack `channel-info` or `channel-list` as agent-facing message actions. Slack has internal `conversations.info`/`conversations.list` use for inbound context and allowlist resolution, but discovery, dispatch, runtime helpers, public exports, tests, and Slack docs still omit the requested tool actions. The open #53199 implementation candidate links this issue but only clearly adds `channel-list` plus `search`, so this issue should stay open until both requested actions are merged or the missing `channel-info` work is tracked separately.

## Best Possible Solution

Keep this issue open. The best implementation belongs in `extensions/slack`: expose both `channel-info` and `channel-list` through `listSlackMessageActions`, translate them in `handleSlackMessageAction`, add runtime/action helpers around Slack `conversations.info` and paginated `conversations.list`, gate consistently with the existing `channelInfo` config unless maintainers intentionally add a separate `channelList` gate, then update Slack docs and focused contract/runtime tests. Review #53199 as a partial candidate, but do not close this issue unless `channel-info` is also implemented or a narrower canonical follow-up is opened for that missing half.
