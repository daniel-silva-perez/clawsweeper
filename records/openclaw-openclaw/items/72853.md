## Summary

Keep open. Current main still has the id-less cron state-collision path, while this PR is a focused implementation for the linked #72849 and I did not find a blocking correctness or security issue in the provided diff plus current-main source review.

## What This Changes

The PR generates and persists IDs for id-less persisted cron jobs, rejects outcome application with invalid job IDs, and adds cron store/timer tests plus cron docs and changelog updates.

## Best Possible Solution

Land a focused cron fix after maintainer review and CI, keeping the ID generation in the cron store load path and the outcome guard in the timer path, with the docs/changelog update retained for operator visibility.

## Reproduction Assessment

Yes. #72849 gives a high-confidence repro: multiple persisted cron jobs without `id` or `jobId`, then a run outcome on current main can compare `undefined` to `undefined` and update the first id-less row.

## Solution Assessment

Yes. The proposed load-time backfill is the narrowest maintainable fix because it makes persisted hand-authored jobs satisfy the existing `CronJob.id` contract, while the timer guard prevents any remaining invalid outcome from mutating the wrong row.
