## Summary

Keep open. Current main still has only partial edit-tool hardening: OpenClaw wraps the upstream edit tool with mismatch hints and post-write recovery, while the pinned upstream tool provides LF normalization and limited fuzzy matching. It does not implement the requested broad whitespace-collapsing fallback, explicit preflight mismatch step, `read line=` / `keyword=` positioning, or docs guidance for repeated 1-3 line edits.

## Best Possible Solution

Keep this issue open as a focused enhancement. The best path is to decide the product contract for whitespace-tolerant edit matching, implement it either upstream or in the OpenClaw wrapper with uniqueness and offset-mapping safeguards, add tests for ambiguous whitespace matches and read positioning behavior, and update public tool docs if the workflow guidance changes.
