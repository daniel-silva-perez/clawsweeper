## Summary

Keep this PR open. Current main still does not contain the proposed unified post-compaction refresh, and the linked bug remains valid against the current announce/output flow. The PR is relevant, but it needs maintainer review because it adds only a one-shot latest-assistant read, does not add a regression test, and does not implement the linked issue's suggested bounded polling for delayed post-compaction output.

## Best Possible Solution

Keep this PR open for maintainer review and likely revision. The fix belongs in the subagent announce/output layer: after child wait/output capture settles, perform a bounded post-wait refresh that prefers a delayed latest assistant reply or silent token over raw tool output before building the completion event, covering both no-child-findings and child-findings paths. Add focused regression coverage for raw Tavily-style tool JSON followed by delayed post-compaction assistant text, and preserve existing ANNOUNCE_SKIP/NO_REPLY, failed-run, timeout, and child-completion aggregation behavior.

## Risks / Open Questions

- The proposed implementation performs one immediate `readLatestAssistantReply` check, while the linked issue suggested bounded polling; it may still miss assistant output that lands shortly after the check.
