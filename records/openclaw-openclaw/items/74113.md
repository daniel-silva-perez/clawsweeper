## Summary

Keep this PR open. Current main still lacks a unified post-output refresh for the subagent announce race, and this PR is relevant to #74073, but it is not ready to close or merge as-is because the branch adds only a one-shot latest-assistant read, has no regression test, and has unresolved review feedback around NO_REPLY normalization and childCompletionFindings behavior.

## Best Possible Solution

Keep this PR open for maintainer review and revision. The fix should add a bounded post-wait refresh in the subagent announce/output layer that prefers delayed latest assistant text or a silent token over raw tool output before building the completion event, applies the existing strip/silent/ANNOUNCE_SKIP normalization, covers both no-child-findings and childCompletionFindings paths, and adds focused regression coverage while preserving failed-run, timeout, sessions_yield, and child aggregation behavior.
