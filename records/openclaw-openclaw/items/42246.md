## Summary

Keep this issue open. Current main has Telegram outbound retry/chunking and a write-ahead delivery recovery queue, but it does not implement configurable batching, aggregation, digest formatting, grouping by severity/source/type, max-message flushing, or shutdown flushing for Telegram notifications. The related open #69165 is complementary transport queue/rate-limit work, not a replacement for this digest feature.

## Best Possible Solution

Keep this issue open and implement an opt-in Telegram outbound batching/digest feature in the Telegram plugin, with schema/types/docs/tests for interval, max batch size, grouping policy, shutdown flush, error bypass behavior, and media handling. Keep #69165 separate as the transport queue/rate-limit/backoff track unless maintainers decide to design both under one shared outbound delivery abstraction.
