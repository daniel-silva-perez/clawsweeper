## Summary

Keep open. Current main does not already contain the Feishu doctor repair, and this PR is the active implementation for the linked Feishu hot-loop report, but the patch should not merge as-is because its repair matcher can remove Feishu group/broadcast session state while the PR describes scoped direct-session cleanup.

## What This Changes

The PR adds a Feishu doctor adapter, wires it into the Feishu channel registration, and adds tests for detecting corrupt Feishu state plus archiving/rebuilding Feishu runtime/session state under `openclaw doctor --fix`.

## Best Possible Solution

Keep the repair Feishu-owned through the existing channel doctor seam, but narrow the repair target to the intended direct/suspicious Feishu sessions or explicitly get maintainer approval for all Feishu session reset behavior. Add regression coverage proving Feishu group/broadcast and ACP binding sessions are preserved when only direct stale state is repaired, then run the focused Feishu doctor/channel doctor tests and changed gate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Narrow repair matching to direct Feishu sessions:** `extensions/feishu/src/doctor.ts:150-152`
  - body: The matcher accepts every `agent:<id>:feishu...` key, including current group and broadcast keys such as `agent:main:feishu:group:*`. Because the repair path deletes every collected Feishu entry once any finding triggers repair, `doctor --fix` can archive/remove healthy group session state even though the PR describes direct-session cleanup. Please restrict the matcher or repair set and add a test that preserves group/broadcast Feishu sessions.
