## Summary

Keep open. Current main still lacks the Feishu doctor repair, and this PR is the candidate implementation for #74237, but the latest head has a blocking repair-safety bug: the repair path can remove every Feishu direct session when any Feishu finding exists, including a corrupt local JSON finding unrelated to otherwise healthy session transcripts.

## What This Changes

The PR adds a Feishu channel doctor adapter, wires it into Feishu channel registration, and adds Feishu doctor tests for detecting and repairing corrupt local state plus Feishu-scoped direct session entries.

## Best Possible Solution

Revise the repair path so corrupt Feishu JSON only rebuilds the Feishu state directory, while session-store removal and artifact archiving target only the session keys referenced by session-specific findings; add a regression for corrupt state plus a healthy Feishu session, then rerun focused Feishu doctor/session tests and the changed gate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Limit repair to sessions with findings:** `extensions/feishu/src/doctor.ts:587-595`
  - body: `inspection.sessionEntries` contains every Feishu direct session key, including healthy transcripts. Building `entriesByStore` from that list means a lone `corrupt-state-json` finding causes `doctor --fix` to archive and remove all Feishu session entries and their transcript sidecars. Derive the target keys from session findings instead, and leave healthy sessions untouched when only local JSON is corrupt.
  - confidence: 0.97

## Security Review

Status: needs_attention

Summary: The diff adds no new dependencies, workflow changes, or secret handling, but its repair path can remove healthy local Feishu conversation state beyond the triggering finding.

Concerns:

- **[medium] Overbroad local session deletion:** `extensions/feishu/src/doctor.ts:587`
  - body: Because repair targets all inspected Feishu session entries, a user accepting `doctor --fix` for corrupt local JSON can lose healthy Feishu session-store entries and have healthy transcript artifacts renamed out of active use. This is a data-integrity concern for a local repair command.
  - confidence: 0.95
