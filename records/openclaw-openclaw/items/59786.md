## Summary

Keep open. Current main still does not provide a documented, cross-platform way to allow only selected `openclaw` subcommands: the public allowlist docs describe executable path/name patterns, TypeScript `argPattern` matching is still guarded as Windows-only, and the macOS companion matcher has no argument-pattern field at all. A related open PR already attempts this area, but it is broad and has substantive review concerns, so this should remain a maintainer-reviewed security/product item.

## What This Changes

The issue requests subcommand-level exec allowlist patterns so safe commands such as `openclaw cron list` can run without allowing every `openclaw` subcommand.

## Best Possible Solution

Keep the issue open and converge on a narrow, reviewed exec-approval contract for argument/subcommand matching, preferably by extending the existing `argPattern` or normalized argv binding consistently across gateway, node host, macOS companion, protocol/schema, UI/CLI, and docs. Treat broader auto-classification or new approval modes as separate product/security work unless maintainers explicitly want them bundled.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection gives a high-confidence behavior check: current main documents executable-level allowlist patterns, has Windows-only TypeScript `argPattern` behavior, and lacks macOS argument-pattern matching.

## Solution Assessment

Unclear. The requested capability is valid, but the proposed colon syntax is not proven to be the best contract; a safer solution would reuse or formalize the existing argument-pattern machinery and review the related open PR or a narrower replacement before landing anything security-sensitive.
