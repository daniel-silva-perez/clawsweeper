## Summary

Keep open: current main still does not expose or enforce provider-level fallback media trust, and open PR #57959 carries closing syntax for this work but has not merged. The issue is concrete and still security-relevant, so it should remain open until that PR is reviewed, revised, landed, or closed.

## What This Changes

Add provider-level fallback trust controls so cross-provider model retries do not forward user images or attachments unless the target provider is trusted for that data.

## Best Possible Solution

Review PR #57959 against current main and either land a revised version or close it with a replacement plan. The desired end state is a centralized fallback-boundary decision that has primary and target provider context, covers explicit and prompt-detected media consistently, exposes documented user-facing trust configuration, and has tests for same-provider and cross-provider retries.
