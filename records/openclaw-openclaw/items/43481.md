## Summary

Keep open: current main still has no provider-level media trust policy for fallback retries, explicit images are handled by a coarse retry strip, prompt-detected image/attachment references still load in the embedded fallback path, and open PR #57959 uses closing syntax for this issue. Because the remaining work is security-sensitive and already has an implementation candidate, cleanup should not close it or queue a competing repair.

## What This Changes

Add fallback media trust controls so cross-provider model retries do not send user images or attachments to a provider the user has not trusted for that data.

## Best Possible Solution

Keep this issue open until PR #57959 lands or is superseded by a centralized, schema/docs-backed fallback media trust policy that consistently covers explicit images, prompt-detected image references, attachment-style image refs, same-provider retries, and cross-provider retries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure a cross-provider model fallback and submit a prompt that references a local image or image attachment marker; current main can still load prompt-detected media for the fallback model while offering no provider-level media trust configuration.

## Solution Assessment

No current-main solution exists. PR #57959 is the paired candidate, but maintainers should decide whether its fail-closed cross-provider media stripping is the desired policy or whether the configurable trust-policy design from the issue is required before merge.
