## Summary

Keep this issue open. Current main still lacks a provider-level fallback media trust policy, still makes only a coarse all-fallback decision for explicit images, and still runs prompt-detected image loading without a fallback trust guard. It is also paired with open PR #57959, which uses closing syntax and should be reviewed or closed before this issue is resolved.

## What This Changes

The issue requests provider-level fallback trust controls so user-supplied images and attachments do not cross provider boundaries unexpectedly during model fallback.

## Best Possible Solution

Review, revise, land, or close PR #57959. The desired end state is a centralized fallback-boundary decision that has primary and target provider context, handles explicit and prompt-detected media consistently, preserves intentional same-provider/image-capable retries where appropriate, and documents any user-facing trust configuration.
