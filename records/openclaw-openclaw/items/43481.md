## Summary

Keep open for https://github.com/openclaw/openclaw/issues/43481: current main still lacks provider-level fallback media trust configuration, still strips explicit images with only a coarse fallback retry flag, and still loads prompt-detected images without fallback/provider trust context. The linked open PR #57959 carries closing syntax for this issue, so the issue should remain open until that PR is reviewed and either landed or closed.

## What This Changes

Add provider-level fallback media trust controls so user images and attachments are not unexpectedly sent across provider boundaries during model fallback.

## Best Possible Solution

Review, revise, land, or close PR #57959. The desired end state is a centralized fallback-boundary decision with primary and target provider context, consistent handling for explicit and prompt-detected media, documented user-facing trust configuration, and tests for same-provider and cross-provider retries.
