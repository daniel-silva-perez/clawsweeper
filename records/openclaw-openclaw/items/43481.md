## Summary

Keep open: current main still lacks provider-level media trust configuration for model fallback, and the open closing PR #57959 already represents the active implementation path. The issue should stay open until maintainers land, revise, or intentionally replace that PR.

## What This Changes

Add provider-level trust controls so cross-provider model fallback can avoid forwarding user images or attachments to an untrusted fallback provider.

## Best Possible Solution

Review PR #57959 against current main and land a revised version, or replace it with a centralized fallback-boundary implementation that has primary and target provider context, covers explicit and prompt-detected media consistently, documents the user-facing trust control, and includes same-provider plus cross-provider tests.
