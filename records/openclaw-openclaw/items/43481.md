## Summary

Keep open. Current main still lacks provider-level fallback media trust configuration, still strips explicit images with only a coarse `isFallbackRetry` check, and still runs prompt-detected image loading without fallback/provider trust context; the linked open PR #57959 should be reviewed or closed before this issue is resolved.

## What This Changes

Add provider-level fallback media trust controls so user images and attachments are not unexpectedly sent across provider boundaries during model fallback.

## Best Possible Solution

Review, revise, land, or close PR #57959. The desired end state is a centralized fallback-boundary decision with primary and target provider context, consistent handling for explicit and prompt-detected media, documented user-facing trust configuration, and tests covering same-provider and cross-provider retries.
