## Summary

Keep open. Current main still has the requested timing value in the health snapshot and JSON output, but the human-readable `openclaw health` renderer does not print the top-level gateway probe duration. An open closing PR already targets this exact issue, so the issue should wait for that PR or an equivalent mainline fix to land.

## What This Changes

Show the gateway probe `durationMs` in standard text output for `openclaw health` while leaving `--json` unchanged.

## Best Possible Solution

The health text renderer should print a `Gateway probe duration: <n>ms` line from `summary.durationMs` with focused regression coverage, while preserving the existing JSON payload unchanged.
