## Summary

Keep this issue open. Current `main` still builds `durationMs` into the health snapshot and preserves it in `--json`, but the human text branch for `openclaw health` does not print a gateway probe duration. The supplied GitHub context also shows open, unmerged closing PR #50250 for the same change, so the issue should stay open until that PR lands or is replaced.

## Best Possible Solution

Review and either land or supersede PR #50250. The likely minimal fix is to print `summary.durationMs` in the non-JSON `openclaw health` output, keep `--json` unchanged, and add a regression assertion in `src/commands/health.command.coverage.test.ts` or `src/commands/health.test.ts` for the text line.
