## Summary

Keep open. Current main still does not implement the requested permanent delivery-error classification for `Local media path is not under an allowed directory`, and it also does not add the proposed `message is too long` classifier or `lastErrorClass` field. Related media-path fixes reduce some upstream ways to hit this error, but they do not solve poisoned queued delivery entries that already fail with this deterministic allowlist rejection.

## Best Possible Solution

Keep this issue open. The narrow product fix belongs in `src/infra/outbound/delivery-queue-recovery.ts` with focused delivery-queue policy/recovery tests: classify the exact local-media allowlist rejection as permanent so poisoned queue entries move directly to `failed/` instead of burning retry budget. A follow-up can decide whether to add enqueue-time media validation and a persisted `lastErrorClass`; the Telegram `message is too long` case should be verified separately before classification.
