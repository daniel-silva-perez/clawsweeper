## Summary

Keep open. Current `main` still matches the report: `/status` is present in the Control UI slash-command menu, but it is not marked as an instant/local command, so the menu Enter path fills the draft instead of sending it. The issue also has an open implementation PR with closing syntax, so the right cleanup action is to let maintainers review, rebase, replace, or close that PR rather than close this issue now.

## What This Changes

Make the Control UI `/status` slash command execute after one Enter from the slash-command menu instead of requiring a second Enter.

## Best Possible Solution

Review, rebase, or replace PR #45602 with a focused Control UI fix that makes `/status` submit from the slash menu on the first Enter while preserving the intended local/Gateway status behavior and updating regression coverage for the menu-selection path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: type `/status` so the slash menu is open, press Enter, and the current selection path fills the draft because `/status` is not `executeLocal`; pressing Enter again then sends the draft through the normal send path.

## Solution Assessment

Unclear for the already-linked PR. The issue should be fixed in the Control UI slash-command selection/registry path, but simply adding `executeLocal: true` is only safe if the implementation also preserves the current Gateway-backed `/status` behavior or adds an equivalent local status executor.
