## Summary

Keep open. Current main still has the reported first-Enter selection path for `/status`: the slash menu intercepts Enter, `/status` is not marked `executeLocal`, and current regression coverage explicitly keeps `/status` on the gateway/agent command path. The issue is also paired with open PR #45602 using closing syntax, so this issue should remain open until that PR is reviewed, landed, closed, or replaced.

## Best Possible Solution

Keep this issue open and review #45602 or replace it with a narrow fix that makes selecting `/status` from the Control UI slash menu submit on the first Enter while preserving the current gateway-backed `/status` behavior. The fix should include regression coverage for the menu Enter path and should not merely flip `executeLocal` unless it also implements a correct local status handler.
