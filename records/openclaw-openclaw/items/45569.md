## Summary

Keep open. Current `main` still has the reported Control UI path: `/status` is present in the slash-command menu, but it is not marked for immediate execution, so Enter selects/fills the command instead of submitting it. The issue is also paired with open PR #45602 using closing syntax, so it should remain open until that PR is reviewed, landed, replaced, or closed.

## What This Changes

The issue requests that the web UI execute `/status` with a single Enter press instead of requiring a second Enter after selecting it from the slash-command menu.

## Best Possible Solution

Keep the issue open and finish the linked implementation path by reviewing PR #45602 or replacing it with a focused fix that makes `/status` submit on the first Enter while preserving the intended local status semantics and adding regression coverage for the menu selection path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: type `/status` with the slash menu open and press Enter; because `/status` is not `executeLocal`, `selectSlashCommand` fills the draft and the next Enter is required to send it.

## Solution Assessment

No for the one-line `executeLocal` proposal by itself. The local executor currently lacks a `status` handler, so the maintainable fix should either add complete client-side status handling or change the menu path to immediately submit gateway-backed no-argument commands safely.
