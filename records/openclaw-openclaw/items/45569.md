## Summary

Keep open. Current main still has the reported web UI path: `/status` is present in the slash menu, but it is not marked `executeLocal`, so Enter selects/fills the command instead of sending it. There is also an open unmerged PR, #45602, whose body uses closing syntax for this issue, so the issue should stay open until that PR is reviewed, landed, closed, or replaced.

## What This Changes

The issue requests that selecting `/status` from the web UI slash-command menu execute with one Enter press instead of requiring a second Enter.

## Best Possible Solution

Keep the issue open and review #45602 or replace it with a focused fix that makes selecting `/status` from the Control UI slash menu submit on the first Enter while preserving the current gateway-backed status semantics. A complete local `/status` handler is also viable, but flipping `executeLocal` alone is not enough on current main.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: type `/status` so the slash menu is open, press Enter, and `selectSlashCommand` fills the draft because `/status` is not `executeLocal`; the next Enter sends it.

## Solution Assessment

No for the one-line `executeLocal` proposal by itself. Current main would route that to the client executor, which lacks a `status` handler, so the safer fix is either menu-level immediate submission for no-argument gateway commands or a complete client-side status implementation with tests.
