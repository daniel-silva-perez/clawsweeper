## Summary

Keep open. Current main still has the reported two-Enter `/status` path in the Control UI slash menu, and an open implementation candidate (#45602) explicitly links this issue, so the issue should not be closed before that PR is reviewed, corrected, merged, or closed.

## Best Possible Solution

Keep this issue open and resolve it through the linked implementation work. The likely fix belongs in the Control UI slash-command path: either make selecting `/status` submit through the gateway command path on the first Enter, or add a real local `/status` implementation plus regression coverage so it cannot fall into `Unknown command`.
