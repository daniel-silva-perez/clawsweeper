## Summary

Keep this PR open. Current main still has Telegram topic/session routing and ACP binding primitives, but the /status path does not receive Telegram message context or render topic routing, configured binding, live binding, drift, or ACP runtime lines. The PR remains a plausible focused visibility slice for the state-visibility gap raised in review discussion.

## Best Possible Solution

Keep the PR open for maintainer review or a focused replacement. The best implementation is to pass message context and command target into /status, add a channel-owned status-lines slot below the session line, resolve configured and live Telegram topic ACP bindings through existing binding helpers, render configured vs live/drift plus ACP runtime state, and keep tests for configured binding, live binding, drift, ACP runtime metadata, and non-topic no-op behavior. Keep unrelated infra or dependency changes out of this PR.
