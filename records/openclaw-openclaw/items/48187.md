## Summary

Keep this PR open. Current main has Telegram topic/session routing and ACP binding primitives, but the `/status` path still does not receive Telegram message context or render topic routing, configured binding, live binding, drift, or ACP runtime lines. The PR is a plausible focused visibility slice rather than obsolete work.

## Best Possible Solution

Keep the PR open for maintainer review or a focused replacement. The best implementation is to pass message context and command target into `/status`, resolve configured and live Telegram topic ACP bindings through the existing binding helpers, render channel-owned topic routing lines below the session line, and keep tests for configured binding, live binding, drift, ACP runtime metadata, and non-topic no-op behavior. Avoid unrelated infra or dependency changes in the same PR.
