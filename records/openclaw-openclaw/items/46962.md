## Summary

Keep open. Current main still drops configured blockquote prefixes after soft/hard breaks and paragraph separators, so this PR targets a real Slack/Signal formatting gap. The branch should not merge as-is because its separator helper is stale against current main's token-aware loose-list paragraph handling and would regress that behavior if rebased directly.

## What This Changes

The PR adds blockquote-depth tracking in `markdownToIR` to re-emit configured blockquote prefixes after line breaks and paragraph separators, plus regression tests for continuation, hardbreak, multi-paragraph, and nested blockquotes.

## Best Possible Solution

Integrate the blockquote-depth prefixing into current main's shared Markdown IR renderer while preserving token-aware list separator behavior, then validate blockquote spacing, nested-list spacing, and the broader markdown suite before merge.
