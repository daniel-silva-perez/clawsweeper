## Summary

Keep open. Current main still drops configured blockquote prefixes after soft/hard line breaks and later paragraphs, so the user-visible bug is not implemented on main. The PR is a focused implementation, but GitHub reports it dirty against current main and it must be rebased without losing the newer loose-list paragraph separator behavior.

## What This Changes

The PR adds blockquote-depth tracking to `markdownToIR`, re-emits configured blockquote prefixes after soft/hard breaks and paragraph separators, and adds regression tests for continuation, hardbreak, multi-paragraph, and nested blockquotes.

## Best Possible Solution

The desired end state is a shared Markdown IR renderer that prefixes every emitted line inside configured blockquotes, including nested and multi-paragraph cases, while preserving current loose/tight list separator semantics and leaving channel adapters on the existing `blockquotePrefix` contract.
