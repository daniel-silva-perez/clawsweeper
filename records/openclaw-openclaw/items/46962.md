## Summary

Keep open. Current main still has the reported Markdown blockquote-prefix bug, and the PR remains relevant, but the branch is not merge-ready because its renderer hunk would drop newer loose-list paragraph handling now present on main.

The PR adds blockquote-depth tracking in `markdownToIR` plus regression tests for soft breaks, hard breaks, multi-paragraph blockquotes, and nested blockquote continuation prefixes.

## Best Possible Solution

Integrate blockquote-depth prefix restoration into the shared Markdown IR renderer while retaining the current token/open-level list paragraph separator contract, with regression coverage for prefixed blockquote continuation and loose-list spacing.
