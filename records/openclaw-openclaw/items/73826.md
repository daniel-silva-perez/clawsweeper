## Summary

Current main already implements the concrete WebChat/Control UI audio attachment path described here: assistant MEDIA audio is classified as audio, audio attachments render with an inline HTML audio player, and webchat TTS/tool audio is persisted as structured audio content. The implementation shipped in v2026.4.11 through the structured assistant media rendering work, with earlier and adjacent TTS delivery fixes also present on current main.

## Best Possible Solution

Close this issue as implemented on main and in v2026.4.11. Keep the shipped inline audio player and structured MEDIA handling; track any future autoplay behavior or local-root availability edge cases as separate focused items.
