## Summary

Keep this issue open. Current main has adjacent package/update hardening, but I do not see a source-backed fix for the reported byte-level corruption of an expected installed `dist/run-main-*.js` chunk. The current inventory and global-install verification are still path-based, so a modified expected file would pass those checks, and startup would still hit the module import/compile failure path.

## Best Possible Solution

Keep this open as a packaging/runtime integrity investigation. Maintainers should reproduce or rule out the 2026.4.5 WSL/global-install corruption, audit update/postinstall/restart/logging paths for writes under the installed package root, and either prove OpenClaw cannot mutate installed core dist chunks at runtime or add content-hash inventory plus startup/doctor verification with clear reinstall or repair guidance for corrupted expected chunks.
