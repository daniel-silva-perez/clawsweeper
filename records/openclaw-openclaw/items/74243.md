## Summary

Keep this PR open. Current main still propagates image optimization failures before Telegram/media upload, so the PR addresses a real gap, but the proposed catch-all fallback affects the shared media loader and can also catch the existing image pixel-limit guard before returning the original byte-small image. It needs maintainer review and consolidation with the other open image-optimization failure work rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to consolidate the optimizer-failure cluster around #73424, keep the useful under-cap fallback from this PR, but narrow it so pixel-limit, unsafe dimension, HEIC-conversion, and over-cap failures still fail closed. Add regression coverage for the oversized-pixel guard alongside the new optimizer-backend fallback tests.
