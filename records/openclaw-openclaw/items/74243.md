## Summary

Keep this PR open. Current `main` still propagates optimizer backend failures for already-under-cap images and still drops the underlying optimizer cause, while this PR is an active, focused consolidation candidate for the shared media optimizer-failure cluster. The security-sensitive part is behavioral rather than supply-chain: the fallback must stay constrained by byte cap, non-HEIC sources, and image input guard failures.

## What This Changes

This PR updates `src/media/web-media.ts` and media tests to return original under-cap non-HEIC image bytes when optimization fails, preserve MIME/type on fallback, rethrow guard and HEIC failures, preserve optimizer causes, and add regression coverage.

## Best Possible Solution

Review this PR as the consolidation path: if maintainers accept the fallback policy, land it with the focused media tests and changed-lane validation, then close #73424 and #73148 and supersede the narrower fallback/diagnostic PRs. Keep #38260 separate because a native libvips SIGILL crash needs crash-level investigation rather than a JavaScript fallback.
