## Summary

Keep this PR open. Current main still has the unguarded child stdin write/end path, so the cleanup reason `implemented_on_main` does not apply, but the latest PR head is not merge-ready because CI reports a diff-local test type error in the new regression test.

## What This Changes

The PR guards child-process stdin writes in `createChildAdapter` against non-writable or asynchronously failing streams and adds regression tests for non-writable stdin plus shell-wrapped missing-binary EPIPE behavior.

## Best Possible Solution

Keep the narrow supervisor-adapter fix, correct the test type typo, and land only after the targeted child adapter tests plus the changed gate are green.

## Reproduction Assessment

Yes. The report has a high-confidence reproduction path through Linux shell-wrapped child spawning with a missing CLI binary, and a read-only Node probe against current main reproduced writable stdin followed by async `EPIPE`.

## Solution Assessment

Yes directionally, but not as currently written. The fix belongs at the generic child-process stdin boundary, but the branch must change `NodeJS.errnoException` to the exported `NodeJS.ErrnoException` type before it is mergeable.
