## Summary

Current main already prevents the reported gateway crash by treating uncaught broken-pipe stdin errors such as EPIPE as benign at the OpenClaw CLI/gateway entrypoints, and that fix shipped in v2026.4.27. The PR’s local child-adapter guard is plausible, but the central user-visible failure is already covered on main.

## What This Changes

The PR changes `createChildAdapter` to guard and swallow child-process stdin write/end failures, with regression tests for non-writable stdin and EPIPE-style missing-binary failures.

## Best Possible Solution

Keep the shipped benign broken-pipe handling as the current crash fix; if maintainers want quieter local stdin handling later, track that as a narrower follow-up rather than keeping this redundant PR open.

## Reproduction Assessment

Yes. Current main still has the shell-wrapped missing-binary EPIPE shape, and a read-only Node probe reproduced EPIPE plus exit code 127 while the current benign-handler behavior kept the process alive.

## Solution Assessment

Yes. The shipped main implementation solves the user-visible gateway crash at the entrypoint error boundary; the PR’s local adapter guard is a reasonable refinement but no longer needed to fix crash safety.
