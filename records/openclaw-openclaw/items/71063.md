## Summary

Keep open. The PR is aimed at the right supervisor failure path, but the latest head still only adds a `child.stdin.writable` guard; a read-only reproduction of the Linux shell-wrapper missing-binary path shows `stdin.writable === true` before an async `EPIPE`, so the reported gateway crash remains reachable.

## What This Changes

The PR changes `createChildAdapter` to skip the initial stdin write when `child.stdin.writable` is false and adds a unit test for a stubbed non-writable stdin stream.

## Best Possible Solution

Handle the initial `params.input` write/end path as a managed stdin operation: install or otherwise consume stdin stream errors before writing, guard synchronous write/end failures, preserve normal writable stdin behavior, and add regression coverage for both the non-writable stub and the shell-wrapped async `EPIPE` missing-binary case.

## Reproduction Assessment

Yes. A read-only Node probe using OpenClaw's Linux shell-wrapper shape reproduces `child.stdin.writable === true` followed by an uncaught `EPIPE`, and the Gemini CLI backend path reaches the same supervisor input write.

## Solution Assessment

No. The PR targets the right file, but a `writable` guard is not the narrowest sufficient contract because Node streams can emit pipe errors asynchronously after the check passes; explicit stdin write/end error handling is safer.
