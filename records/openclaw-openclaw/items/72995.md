## Summary

Keep open. The report is recent and still matches current main: the managed web_search path does not pass the agent/tool AbortSignal into the web-search runtime or Gemini provider, AbortError-shaped failures are rethrown before normal tool error logging, and the existing Gemini tests do not prove the instantaneous-abort case fixed.

## What This Changes

The issue reports intermittent near-instant Gemini-backed web_search aborts with no gateway err.log entry and no effect from timeoutSeconds.

## Best Possible Solution

Add a backwards-compatible web-search execution context that carries AbortSignal and abort diagnostics through the managed tool, runtime, provider helpers, and Gemini implementation; classify/log unexpected AbortError failures safely, and cover the path with focused unit tests plus optional Gemini live proof.
