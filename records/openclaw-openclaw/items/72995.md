## Summary

Keep open. Current main still matches the reported failure class: managed `web_search` does not forward the agent tool `AbortSignal` into the web-search runtime/provider contract, Gemini search receives only the configured timeout, and `AbortError` is rethrown before the normal tool error logging path.

## What This Changes

The report asks OpenClaw to fix intermittent near-instant Gemini-backed `web_search` aborts that return `This operation was aborted` without a corresponding gateway error log entry.

## Best Possible Solution

Add a backward-compatible web-search execution context that carries `AbortSignal` and safe abort diagnostics through the managed tool wrapper, runtime, provider SDK type, provider helpers, and Gemini implementation; then log unexpected provider abort failures while preserving true user/session cancellation behavior.

## Reproduction Assessment

Unclear for the live intermittent Gemini failure: the reporter gave concrete v2026.4.23 macOS/Gemini steps, but the race is intermittent and this read-only pass did not run live Gemini. The supporting source path is high confidence: current main drops the managed tool signal and rethrows `AbortError` before normal tool logging.

## Solution Assessment

Not applicable as a proposed patch because this is an issue, not a PR. The suspected session-level abort race is plausible but not yet proven as the only trigger, so the best repair path is an additive execution-context seam plus focused abort/logging tests rather than a Gemini-only workaround.
