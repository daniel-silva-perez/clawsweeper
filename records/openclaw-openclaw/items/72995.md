## Summary

Keep open. Current main still has the reported failure class: the managed web_search wrapper and provider contract do not propagate the tool AbortSignal, the Gemini provider only receives a configured timeout, and AbortError-shaped provider failures bypass the normal tool error log path. The exact live Gemini race is not proven, but the source still leaves the silent abort path intact.

## What This Changes

The issue reports intermittent sub-second Gemini web_search aborts with no gateway.err.log entry despite a healthy Gemini API and timeout configuration having no effect.

## Best Possible Solution

Ship an additive web-search provider execution context that carries AbortSignal and safe diagnostics from the managed tool wrapper through runtime resolution and provider implementations, then add tests that distinguish true user/session cancellation from unexpected provider AbortError failures.

## Reproduction Assessment

Unclear for the intermittent live Gemini failure: the reporter gave concrete steps, but the bug is timing-dependent and no live Gemini run was performed in this read-only review. The supporting source-level path is high-confidence because current main drops the tool signal and rethrows AbortError before normal logging.

## Solution Assessment

Unclear as a final fix because the suspected session runAbortController race is plausible but not yet proven. A Gemini-only timeout tweak would not address the dropped signal/logging path; the maintainable direction is an additive execution-context seam with adapter/runtime tests.
