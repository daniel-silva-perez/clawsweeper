## Summary

Keep open. Current main still matches the report: the managed `web_search` path accepts an agent tool signal at the outer interface but does not pass it through the web-search runtime/provider contract, and `AbortError` failures are rethrown before normal tool error logging. The older #72986 thread is useful duplicate context, not shipped-fix evidence.

## What This Changes

The issue reports intermittent sub-second `This operation was aborted` failures from Gemini-backed `web_search`, unaffected by timeout config and missing from `gateway.err.log`.

## Best Possible Solution

Add a backwards-compatible web-search execution context that carries `AbortSignal` and safe abort diagnostics through the managed tool, runtime, provider helper layer, and Gemini implementation; log unexpected abort failures without leaking secrets; and cover the path with focused unit tests plus optional live Gemini proof.
