## Summary

Keep open. Current main still matches the reported failure class: the managed `web_search` path does not pass the agent tool `AbortSignal` through the web-search provider contract, and provider-raised `AbortError`s still bypass the normal tool failure logging path.

## What This Changes

The issue asks OpenClaw to fix intermittent silent `This operation was aborted` failures from Gemini-backed `web_search`, including ineffective timeout tuning and missing `gateway.err.log` diagnostics.

## Best Possible Solution

Add a backwards-compatible web-search execution context carrying `AbortSignal` and abort diagnostics through the managed tool wrapper, runtime types, provider SDK helper, and Gemini implementation; then log/classify unexpected provider aborts while preserving true user/session cancellation as cancellation.

## Reproduction Assessment

Unclear for the live intermittent Gemini abort because no live Gemini run was performed and the report is timing-dependent. The supporting failure path is source-reproducible by having a selected provider throw an `AbortError`, which the current adapter rethrows before normal tool failure logging.

## Solution Assessment

Not applicable as a proposed patch; the current implementation is not the best end state because it loses cancellation context and treats all provider `AbortError`s as silent cancellation. The narrow maintainable direction is an additive execution-context seam plus cancellation/logging tests, not a Gemini-only timeout tweak.
