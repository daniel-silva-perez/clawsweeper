## Summary

Keep open. Current main does not implement this PR's intended vendored A2UI dependency hardening: the same package manifests and lockfiles still contain the older markdown-it, Angular, ajv, fast-xml-parser, handlebars, @tootallnate/once, jws, node-forge, path-to-regexp, qs, and yaml versions that the PR proposes to update or override. The PR is security-sensitive but, based on the provided PR file list and local main inspection, it is scoped to vendored A2UI package and lock files rather than runtime logic, CI, secrets, or release scripts.

## Best Possible Solution

Keep this PR open for maintainer security review. The best path is to verify the CVE claims and lockfile provenance, run the targeted `trivy fs vendor/a2ui --scanners vuln` proof plus an A2UI bundle/build check in an appropriate environment, then either land a corrected version of this PR or supersede it with a maintainer-authored vendored A2UI lockfile update.
