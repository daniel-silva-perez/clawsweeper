## Summary

Keep open. Current `main` and the latest release still have the older vendored A2UI dependency state, so the PR is not obsolete, but the patch is incomplete because it updates the lit renderer dependency without regenerating the tracked A2UI runtime bundle that serves bundled `markdown-it`.

## What This Changes

The branch bumps vendored A2UI renderer `markdown-it`/Angular package versions and adds `pnpm.overrides` plus regenerated lockfiles for A2UI spec eval packages to address reported CVEs.

## Best Possible Solution

Keep the security-hardening direction, but update the PR so the vendored lockfiles, generated A2UI bundle, and bundle hash are consistent, then verify the exact vendored scan and an A2UI bundle/build smoke before merge.

## Reproduction Assessment

Yes. Current `main` shows the older vendored dependency and lockfile versions, and the remaining patch issue is reproducible by observing that the PR changes `vendor/a2ui/renderers/lit` while omitting the tracked `src/canvas-host/a2ui/a2ui.bundle.js` and `.bundle.hash` outputs that `pnpm canvas:a2ui:bundle` would update.

## Solution Assessment

No, not as-is. Updating the vendored manifests and lockfiles is the right core approach, but the PR should also account for the tracked generated A2UI runtime asset or explicitly prove that only release-time regeneration is intended and safe.
