## Summary

Keep this PR open. Current main still forwards wizard select and multiselect labels directly into Clack, including the searchable paths, and current tests only cover tokenized search rather than blank or whitespace-only option-label fallback. The related onboarding trim/crash work is adjacent but does not implement this prompt-adapter behavior.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is a narrow helper in src/wizard/clack-prompter.ts that normalizes blank or whitespace-only labels for select, searchable select, multiselect, and searchable multiselect to a value-derived fallback, with focused unit coverage and clarified crash/display-hardening evidence.
