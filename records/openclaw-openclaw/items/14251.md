## Summary

Keep open. Current main still manages the macOS Gateway service as a user LaunchAgent only; `status --deep` is diagnostic and the requested stop/uninstall system LaunchDaemon lifecycle behavior remains unresolved, with related LaunchDaemon scope work still requiring maintainer product/security review.

## What This Changes

The issue asks for macOS gateway stop/uninstall/status lifecycle commands to detect system-level `/Library/LaunchDaemons/ai.openclaw.gateway.plist` installations and provide safe system-scope handling or guidance.

## Best Possible Solution

Define one explicit macOS launchd user/system scope model, then align install, status, stop, uninstall, restart, docs, and regression tests so user LaunchAgents and system LaunchDaemons are detected accurately and privileged system operations either run through a safe explicit path or return precise elevated commands without false success.
