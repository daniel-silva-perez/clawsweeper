## Summary

Keep open. Current main still does not provide macOS system LaunchDaemon lifecycle management for gateway stop/uninstall: the managed Darwin service remains a user LaunchAgent, lifecycle options have no scope selector, and system-level detection is limited to diagnostic `status --deep` output. The issue discussion also keeps this launchd-scope work active alongside #51860 and #55208, so this is not conservative close material.

## What This Changes

The issue requests better macOS gateway stop/uninstall behavior for system-level `/Library/LaunchDaemons` installs, including detection, accurate status, and an explicit or guided system-scope lifecycle path.

## Best Possible Solution

Define one explicit macOS launchd scope model, then align install/status/stop/uninstall/restart, docs, and regression tests so user LaunchAgents and system LaunchDaemons are detected accurately and privileged system operations either run through a safe explicit path or return precise elevated commands without false success.
