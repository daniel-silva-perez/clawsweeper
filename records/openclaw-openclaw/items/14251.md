## Summary

Keep open. Current main still treats macOS gateway lifecycle as a user LaunchAgent: stop/uninstall have no system-scope option, the Darwin service registry targets LaunchAgent helpers only, and deep status is diagnostic rather than lifecycle control. The remaining work is valid but crosses a privileged LaunchDaemon boundary and overlaps related launchd-scope decisions in #51860 and #55208.

## What This Changes

Improve macOS gateway stop/uninstall/status lifecycle so system-level LaunchDaemon installations are detected and either safely managed or reported with privileged guidance.

## Best Possible Solution

Define one explicit macOS launchd scope contract, then update lifecycle, status, docs, and regression tests so user LaunchAgents and system LaunchDaemons are detected accurately, stop/uninstall do not report user-scope success while a matching system daemon remains active, and privileged actions require explicit safe operator intent.
