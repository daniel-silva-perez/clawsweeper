## Summary

Keep open. Current main still keeps macOS Gateway lifecycle management scoped to the per-user LaunchAgent path and GUI launchd domain; system LaunchDaemon awareness is diagnostic-only and does not let stop/uninstall manage `/Library/LaunchDaemons/ai.openclaw.gateway.plist`.

## Best Possible Solution

Keep this issue open and resolve it as part of a deliberate macOS LaunchDaemon scope design: detect `/Library/LaunchDaemons/ai.openclaw.gateway.plist` during stop/uninstall, avoid user-scope success responses while a KeepAlive system daemon remains active, provide exact safe `sudo launchctl bootout system ...` and removal guidance or explicit `--system` handling, and add focused tests/docs for user LaunchAgents versus system LaunchDaemons. Coordinate with #51860 and #55208 rather than closing this narrower lifecycle gap prematurely.
