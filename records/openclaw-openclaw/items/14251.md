## Summary

Current main still has the reported macOS system LaunchDaemon lifecycle gap. Gateway lifecycle commands remain scoped to the per-user LaunchAgent path and GUI launchd domain; system LaunchDaemon discovery exists only in diagnostic `status --deep`, not in stop/uninstall management.

## Best Possible Solution

Keep this issue open and resolve it through a deliberate macOS launchd scope design: detect `/Library/LaunchDaemons/ai.openclaw.gateway.plist` during stop/uninstall, avoid user-scope success responses while a KeepAlive system daemon remains active, provide exact safe sudo bootout/removal guidance or an explicit `--system` lifecycle path, and add focused tests/docs for user LaunchAgents versus system LaunchDaemons while coordinating with #51860 and #55208.
