## Summary

Keep open. Current main still manages the macOS gateway lifecycle as a user LaunchAgent only; stop/uninstall have no system-scope flag or LaunchDaemon target, and status --deep remains diagnostic rather than lifecycle management. The issue discussion also explicitly keeps this launchd-scope work active alongside related open LaunchDaemon requests.

## What This Changes

The issue requests macOS gateway stop/uninstall/status support for detecting and safely handling system-level LaunchDaemons in addition to user LaunchAgents.

## Best Possible Solution

Define one explicit macOS launchd user/system scope model, then update lifecycle operations to detect /Library/LaunchDaemons/ai.openclaw.gateway.plist, avoid user-scope success when a matching system daemon remains active, provide safe elevated-command guidance or an explicit system lifecycle path, and align status, docs, and regression tests with the chosen contract.
