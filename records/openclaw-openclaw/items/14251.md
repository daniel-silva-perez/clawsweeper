## Summary

Current main still does not implement the requested macOS system LaunchDaemon stop/uninstall lifecycle. Gateway service management remains per-user LaunchAgent and gui/<uid> scoped, lifecycle commands have no system target option, status --deep is only a diagnostic extra-service scan, and related LaunchDaemon scope items #51860 and #55208 are still open.

## What This Changes

Request is to make macOS gateway stop/uninstall detect system-level /Library/LaunchDaemons installs, avoid false not-loaded success, and provide safe sudo guidance or an explicit system-scope lifecycle option.

## Best Possible Solution

Keep this open and resolve it as part of the macOS launchd scope design: introduce an explicit user/system launchd model, detect /Library/LaunchDaemons/ai.openclaw.gateway.plist during stop/uninstall, avoid user-scope success when a matching system daemon is active, provide exact safe sudo guidance or an explicit system lifecycle path, and align status/docs/tests with #51860 and #55208.
