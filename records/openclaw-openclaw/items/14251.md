## Summary

Current main at f2405c830bd925bd24f91b6bac5d417c6d36e5ba still does not implement the requested macOS system LaunchDaemon stop/uninstall lifecycle. Gateway service management remains LaunchAgent and gui/<uid> scoped, lifecycle commands expose no system target flag, docs still list lifecycle commands as --json-only, and related LaunchDaemon design issues #51860 and #55208 remain the better coordination points.

## Best Possible Solution

Keep this issue open and resolve it as part of a coordinated macOS launchd scope design: detect /Library/LaunchDaemons/ai.openclaw.gateway.plist during stop/uninstall, avoid reporting user-scope success while a matching system daemon is active, provide exact safe sudo guidance or an explicit system lifecycle path, and align tests/docs with #51860 and #55208.
