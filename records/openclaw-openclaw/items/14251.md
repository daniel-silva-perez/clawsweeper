## Summary

Current main still does not let gateway stop/uninstall detect or manage a macOS system LaunchDaemon. The CLI lifecycle surface remains user LaunchAgent-scoped, while system LaunchDaemon work is only partially diagnostic and overlaps with open broader LaunchDaemon issues #51860 and #55208.

## Best Possible Solution

Keep this issue open and resolve it through a coordinated macOS launchd scope design: detect `/Library/LaunchDaemons/ai.openclaw.gateway.plist` during stop/uninstall, avoid reporting user-scope success while a matching system daemon is active, provide exact safe sudo bootout/removal guidance or an explicit system lifecycle path, and align tests/docs with the broader LaunchDaemon install/status work in #51860 and #55208.
