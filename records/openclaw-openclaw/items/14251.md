## Summary

Keep open. Current main still leaves the core LaunchDaemon lifecycle gap: macOS stop/uninstall remain scoped to the per-user LaunchAgent path and GUI launchd domain, while system LaunchDaemon handling is only a best-effort status diagnostic and the related LaunchDaemon product/security questions remain open in #51860 and #55208.

## What This Changes

The issue asks gateway lifecycle commands to detect system-level macOS LaunchDaemons and either manage them safely or provide accurate elevated-command guidance instead of reporting user LaunchAgent success.

## Best Possible Solution

Define a single macOS launchd scope model, then align lifecycle commands, status/doctor/docs, and regression tests so user LaunchAgents and system LaunchDaemons are detected accurately and system-scope operations either execute through an explicit safe path or return precise elevated commands without false success.
