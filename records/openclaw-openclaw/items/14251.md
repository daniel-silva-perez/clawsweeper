## Summary

Keep open. Current main still has only user LaunchAgent lifecycle for macOS gateway stop/uninstall; there is no system-scope option or lifecycle detection, and the only /Library/LaunchDaemons scan is status --deep diagnostics that excludes the managed gateway label. The issue discussion and open related items keep this as active launchd-scope work.

## What This Changes

The issue requests macOS gateway lifecycle support for detecting and managing or guiding users through system-level LaunchDaemons during stop, uninstall, and status flows.

## Best Possible Solution

Introduce an explicit macOS launchd user/system scope model, detect /Library/LaunchDaemons/ai.openclaw.gateway.plist during lifecycle operations, avoid user-scope success when a matching system daemon remains active, provide safe sudo guidance or an explicit system lifecycle path, and align status, docs, and tests with the related LaunchDaemon work.
