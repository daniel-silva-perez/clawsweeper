## Summary

Keep open. Current main still implements macOS gateway lifecycle as a user LaunchAgent only; the deep-status scanner is diagnostic, filters the canonical gateway label, and does not give stop/uninstall control over a matching system LaunchDaemon. The requested privileged launchd lifecycle still overlaps #51860 and #55208 and needs maintainer scope/security design rather than conservative cleanup.

## What This Changes

Request to make macOS gateway stop, uninstall, and status detect and handle system-level LaunchDaemons in addition to user LaunchAgents, with safe guidance or explicit system-scope flags.

## Best Possible Solution

Define one macOS launchd scope contract, then update lifecycle, status, docs, and regression tests so user LaunchAgents and system LaunchDaemons are detected accurately, stop/uninstall avoid false success when a system daemon remains active, and privileged actions require explicit safe operator intent.
