## Summary

Keep open. Current main still only manages the macOS gateway lifecycle as a user LaunchAgent, with no user/system launchd scope in stop or uninstall, and the related open LaunchDaemon requests keep the product and privilege contract active rather than closed or superseded.

## What This Changes

The issue asks OpenClaw to detect and safely handle macOS system-level gateway LaunchDaemons for stop, uninstall, and status flows, with accurate reporting and elevated-operation guidance.

## Best Possible Solution

Define one explicit macOS launchd user/system scope model, then align install, status, stop, uninstall, restart, docs, and regression tests so user LaunchAgents and system LaunchDaemons are detected accurately and privileged system operations either run through a safe explicit path or return precise elevated commands without false success.
