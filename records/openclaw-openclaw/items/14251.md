## Summary

Keep open. Current main still manages the macOS gateway lifecycle as a user LaunchAgent only; `status --deep` is diagnostic extra-service discovery and does not provide a managed system LaunchDaemon lifecycle path. The latest issue discussion also keeps the launchd scope decision active alongside #51860 and #55208.

## What This Changes

The issue requests macOS gateway stop, uninstall, and status behavior that detects system-level LaunchDaemons and either manages them safely or gives precise elevated-command guidance instead of reporting user LaunchAgent success.

## Best Possible Solution

Define one explicit macOS launchd user/system scope model, then update lifecycle operations to detect `/Library/LaunchDaemons/ai.openclaw.gateway.plist`, avoid user-scope success when a matching system daemon remains active, provide safe elevated-command guidance or an explicit system lifecycle path, and align status, doctor/docs, macOS app assumptions, and regression tests with that contract.
