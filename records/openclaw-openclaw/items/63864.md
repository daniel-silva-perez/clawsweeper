## Summary

Keep open. Current main still lacks configurable per-session lane concurrency, so the PR remains relevant to the session-concurrency slice of #53023, and the latest commit does address the earlier hot-reload propagation/test-reset feedback. It should not merge as-is because the branch still omits the strict config schema/docs/metadata for the new public key and also removes gateway reload and SDK/type-surface behavior unrelated to the feature.

## What This Changes

The PR branch adds an `agents.defaults.sessionLaneMaxConcurrent` resolver and queue wiring while also changing agent default types, gateway reload handling, and plugin SDK config-runtime exports.

## Best Possible Solution

Keep the feature work focused: add a settled session-lane concurrency config key across the exported types, strict schema, generated config metadata/help, docs, and focused queue/gateway tests; wire startup and hot reload through the shared queue state; preserve existing gateway reload secrets/auth behavior and SDK compatibility exports.
