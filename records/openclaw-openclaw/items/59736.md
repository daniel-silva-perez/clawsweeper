## Summary

Keep open. Current main has a root Makefile, but it only exposes `build` and has no self-documenting `help` target or common dev-task aliases, so the requested DX surface is still materially unmet.

## What This Changes

Request to add a self-documenting root Makefile or justfile for common OpenClaw development tasks and document it for contributors.

## Best Possible Solution

Add a maintained root Makefile with a self-documenting `help` target and thin aliases for the common existing `pnpm` scripts, then mention the optional Make workflow in contributor or setup docs without duplicating command semantics outside `package.json`.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a runtime bug, but yes for verification: current main can be checked with `make -n help`, which fails, and `Makefile` only contains the `build` alias.

## Solution Assessment

Is this the best way to solve the issue? Yes, a thin Makefile wrapper is a narrow maintainable DX improvement; using `just` would add a new tool expectation, so Makefile is the lower-friction option unless maintainers prefer otherwise.

## Repair Work Prompt

Implement the DX request in https://github.com/openclaw/openclaw/issues/59736 with a conservative root Makefile expansion. Keep it as thin wrappers over existing `pnpm` scripts; do not move command logic out of `package.json`, do not add a `just` dependency, and do not run broad gates locally. Add a self-documenting `help` target and common targets such as `build`, `dev`, `test`, `test-fast`, `test-e2e`, `test-channels`, `test-extensions`, `check`, `check-changed`, `lint`, `lint-fix`, `format`, `docs-list`, and other low-surprise aliases only when matching scripts already exist. Document the optional Make workflow in `CONTRIBUTING.md` or the dev setup docs. Changelog is probably not needed unless maintainers consider this user-facing enough. Validate with `make help`, `make -n build`, `make -n dev`, `make -n test`, `make -n check`, `make -n test-fast`, `make -n test-e2e`, `make -n test-channels`, `make -n test-extensions`, `git diff --check`, and the relevant docs formatter/check path for touched markdown.
