## Summary

Keep open. Current main has a root Makefile, but it is only a one-target `build` alias and does not address the requested self-documenting Makefile/justfile for common development tasks. The underlying discoverability problem remains: `package.json` currently has 350 scripts, and contributor docs still direct people to raw `pnpm` commands rather than `make help` or equivalent.

## Best Possible Solution

Keep this issue open and implement it as a focused DX change: expand the existing root Makefile into a self-documenting wrapper with `help` plus curated common targets such as `build`, `dev`, `check`, `check-changed`, `test`, `test-fast`, `test-e2e`, `test-channels`, `test-extensions`, `lint`, `lint-fix`, `format`, and `docs-list`, all delegating to current `pnpm` scripts. Update README or CONTRIBUTING to mention the optional Makefile path while keeping `pnpm` as the canonical script source.
