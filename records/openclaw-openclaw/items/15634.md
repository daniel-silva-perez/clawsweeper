## Summary

Keep this issue open. Current main has implemented many Mattermost parity pieces, including markdown-preserving delivery, reactions, threading, typing indicators, media, and interactive buttons from #19957, but the requested read receipts / presence parity still is not implemented or documented for Mattermost.

## Best Possible Solution

Keep this issue open as the Mattermost parity umbrella until read receipts / presence are implemented in the Mattermost plugin, explicitly documented as descoped, or split into a narrower canonical Mattermost issue. Any implementation should update the Mattermost client/WebSocket handling, config/types/docs, and tests for the chosen event/action/privacy contract.
