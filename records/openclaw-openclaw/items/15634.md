## Summary

Keep this issue open. Current main has implemented many Mattermost parity pieces, including bundled-plugin docs, markdown-preserving delivery, reactions, threading, typing indicators, media, and interactive buttons from #19957, but the requested read receipts / presence parity is still not implemented or documented for Mattermost.

## Best Possible Solution

Keep this issue open as the Mattermost parity umbrella until read receipts / presence are implemented, explicitly documented as descoped, or split into a narrower canonical Mattermost issue. Treat #19957 as the completed rich-interactions/buttons evidence and the existing reactions/threading/typing code as completed subscopes.
