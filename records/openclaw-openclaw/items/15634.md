## Summary

Keep this issue open. Current main has completed many Mattermost parity pieces, including markdown-aware delivery, reactions, threading, typing indicators, media, and interactive buttons from #19957, but the requested read receipts / presence parity is still not implemented or documented for Mattermost. There were no Mattermost or Mattermost-doc changes between the prior maintainer review SHA and current main, so the earlier keep-open conclusion still holds.

## Best Possible Solution

Keep this issue open as the Mattermost parity umbrella until read receipts / presence are implemented in the Mattermost plugin, explicitly documented as descoped, or split into a narrower canonical Mattermost issue. Any implementation should update the Mattermost WebSocket/client handling, config/types/docs, and tests for the chosen event/action/privacy contract.
