## Summary

Keep open. Current main still returns without creating a models.json when provider planning skips, and this issue is explicitly paired with open PR #58373, so it should remain open until that PR is reviewed, landed, replaced, or closed.

## What This Changes

A newly created non-main agent can start without its own models.json when provider planning returns an empty skip plan, causing later model resolution to fail with an Unknown model error.

## Best Possible Solution

Land a narrow fix in the models-config skip branch that prevents runtime from continuing with a missing non-main models.json, either by one-time bootstrap from the main agent without overwriting existing files or by failing early with actionable guidance. Keep the broader global sync request out of scope.

## Reproduction Assessment

Yes. The high-confidence reproduction path is a non-main agent with no models.json, empty provider resolution causing plan.action === "skip", then a runtime model lookup that can return Unknown model because the file was never created.

## Solution Assessment

Unclear until PR review. The one-time bootstrap proposed in #58373 matches the narrow requested behavior, but maintainers should review the credential-copying boundary, idempotency, and concurrency guards before landing it.
