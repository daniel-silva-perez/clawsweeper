## Summary

Keep this issue open. Current main still has the reported empty-provider skip path: `planOpenClawModelsJson` returns `skip` when provider resolution is empty, and `ensureOpenClawModelsJson` immediately returns without creating or bootstrapping `models.json`. The linked open PR #58373 uses closing syntax for this issue and is an implementation candidate, so this issue should remain open until that PR is reviewed and merged or replaced.

## Best Possible Solution

Keep this issue open and review #58373 as the canonical implementation candidate. The likely fix belongs in `src/agents/models-config.ts`: when a non-main agent has no `models.json` and provider planning returns `skip`, either bootstrap once from the main agent registry with strict non-overwrite/file-mode guards and regression coverage, or fail before runtime with actionable guidance. After the PR lands, GitHub can close this issue as implemented.
