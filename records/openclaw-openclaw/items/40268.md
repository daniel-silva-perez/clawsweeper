## Summary

Current main still rejects unknown config keys on normal config validation, load, snapshot, startup, and hot-reload paths. The repository has mitigation and repair paths through last-known-good recovery and doctor, but it does not yet implement the requested read-time strip-and-warn behavior. An open implementation candidate, #42201, is still the best path to resolve this bug.

## Best Possible Solution

Keep this issue open. The best maintainer path is to review and either land or close #42201. If it lands, close this as implemented with regression coverage for startup and hot reload accepting unknown keys while strict write/config-validate behavior and invalid known-field failures remain intact. If #42201 is abandoned, implement the same behavior directly in the config read/snapshot path, with focused tests for the root and nested misplaced-key examples from this issue.
