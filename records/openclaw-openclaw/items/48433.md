## Summary

Keep this PR open. Current main still does not implement the PR’s intended `tools.exec.allowedHosts` config field or per-host `exec-approvals.json` allowlist buckets. The existing `host=auto` routing overlaps with part of the problem space, but it is not equivalent to a concrete `host: "gateway"` default that may explicitly request sandbox at runtime with separate per-host approval buckets. The PR also remains security-sensitive because it changes command execution routing and approval persistence.

## Best Possible Solution

Keep this PR open. The best path is to rebase it onto current main’s `host=auto` routing model, decide whether maintainers still want explicit `tools.exec.allowedHosts`, and address the remaining approval-flow concerns with focused tests for gateway-default plus sandbox-request routing, per-host approval buckets, sandbox wildcard approvals, node-host compatibility, doctor migration, schema validation, and docs.
