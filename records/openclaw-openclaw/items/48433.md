## Summary

Keep this PR open. Current main still does not implement `tools.exec.allowedHosts` or per-host `exec-approvals.json` allowlist buckets, and the existing `host=auto` routing model is not equivalent to the PR’s gateway-default plus sandbox-request use case. The PR is still a plausible implementation candidate, but it changes command execution routing and approval persistence, so the unresolved security-sensitive review concerns need maintainer follow-up before merge.

## Best Possible Solution

Keep this PR open. The best path is to rebase it onto current main’s `host=auto` model, preserve the useful explicit `tools.exec.allowedHosts` and per-host approval design if maintainers still want that product shape, and address the remaining P1/P2 review items with focused tests for gateway-to-sandbox requests, approval prompts, sandbox wildcard behavior, node-host compatibility, migration/doctor behavior, and docs/schema alignment.
