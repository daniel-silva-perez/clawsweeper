## Summary

Keep this PR open. Current main still does not implement the intended `tools.exec.allowedHosts` config surface or per-host `exec-approvals.json` allowlist buckets. The existing `host=auto` routing overlaps with part of the goal, but it is not equivalent to a gateway-default agent explicitly requesting sandbox at runtime with separate approval buckets. The PR is also security-sensitive because it changes command execution routing and approval persistence, and the provided review discussion still contains concrete approval-flow concerns.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it onto current main's `host=auto` routing model, decide whether explicit `tools.exec.allowedHosts` is still wanted, and require focused tests for gateway-default plus sandbox-request routing, per-host approval buckets, sandbox wildcard approvals with `ask` preserved, node-host compatibility, doctor migration, schema validation, and docs.
