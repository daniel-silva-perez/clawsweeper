## Summary

Keep open. Current main still does not persist or replay followup queues or drain-window inbound messages across gateway restart, and the linked tracking issue #51620 remains open. This PR remains a plausible implementation candidate, but it should be revised before merge because the submitted approach still has routing/durability concerns and includes unrelated changes/artifacts.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The best path is to remove the debug log artifact, split or drop the unrelated `/model` slash-command change, and rework persistence so queued and drain-rejected inbound messages replay through existing followup or dispatch semantics with routing, session wake, untrusted user content, media/command context, queue caps, and summary state covered by focused tests. If another PR becomes the canonical implementation for #51620, close this PR as superseded then.
