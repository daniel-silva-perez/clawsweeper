## Summary

Keep open. Current main still does not implement the PR’s intended durable followup-queue or drain-window inbound-message recovery, and the linked bug report remains open. This PR is still an implementation candidate, but it needs maintainer review and likely revision because the proposed replay path, unrelated Control UI change, and committed log artifact carry real functional and security-review risk.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The best path is to remove the debug log artifact, split the unrelated /model command fix, and implement restart recovery through normal durable followup/dispatch semantics with tests for routing, wakeup, untrusted user content, media and command context, overflow summary state, malformed persisted data, and consecutive restarts. If maintainers choose #51623 or another PR as the canonical fix, close this PR as superseded only after that direction is explicit.
