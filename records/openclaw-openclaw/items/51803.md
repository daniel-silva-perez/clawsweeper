## Summary

Keep open. Current main still lacks the PR's intended durable followup-queue and drain-window inbound-message persistence, #51620 remains open, and this PR is still an implementation candidate rather than obsolete. The PR also needs maintainer review because the proposed replay path and unrelated/debug-log changes carry functional and security-review risk.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The best path is to remove the debug log artifact, split the unrelated /model command fix, and implement pending-message recovery through normal followup/dispatch semantics with durable storage, routing, wakeup, untrusted content, media/command context, queue cap/summary state, and consecutive-restart behavior covered by tests. If a different PR becomes the canonical fix for #51620, close this PR as superseded then.
