## Summary

Keep open. Current main still does not persist or replay followup queues or drain-window inbound messages across gateway restart, and the linked tracker #51620 remains open. This PR is still a plausible implementation candidate, but the supplied PR context shows it needs maintainer review/revision before merge because it replays through system-event semantics and includes unrelated/debug-log changes.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The best path is to remove the debug log artifact, split or drop the unrelated /model slash-command change, and rework pending-message recovery so followup and drain-rejected inbound messages replay through normal followup or dispatch semantics with routing, wakeup, untrusted content handling, media/command context, queue caps, summary state, validation, and consecutive-restart durability covered by focused tests. If another PR becomes the canonical fix for #51620, close this PR as superseded then.
