## Summary

Keep this PR open. Current main has added an explicit system-prompt cache boundary and already places Group Chat Context below it, but the central PR objective is not implemented: runtime/channel-sensitive Messaging and Reactions guidance still appears before stable Project Context and before the cache boundary.

## Best Possible Solution

Keep this PR or its linked issue open, then refresh the implementation against current main. Move or split volatile runtime/channel/session-specific Messaging and Reactions guidance below `SYSTEM_PROMPT_CACHE_BOUNDARY`, preserve stable Project Context in the reusable prefix, and add a current-layout regression test in `src/agents/system-prompt.test.ts`.
