## Summary

Keep this PR open: current main and the latest release still do not emit or parse a `chcp 65001>nul` bootstrap in the Windows gateway Scheduled Task wrapper, so the PR is not obsolete. The patch is useful and narrow, but it needs a branch refresh/test repair because the new exact parser expectation omits the `sourcePath` field returned by current main.

## What This Changes

The PR prepends `chcp 65001>nul` to generated Windows gateway Scheduled Task scripts, skips that bootstrap line during command read-back, and adds non-ASCII Windows path regression coverage.

## Best Possible Solution

Refresh and land this PR or an equivalent narrow replacement that emits a deterministic UTF-8 bootstrap immediately after `@echo off`, skips only that recognized bootstrap during scheduled-task command read-back, preserves `sourcePath` and environment-source behavior, and keeps the regression coverage focused on non-ASCII Windows task scripts.
