## Summary

Current main still does not implement the PR's intended Windows gateway task-script UTF-8 bootstrap. The PR remains a focused implementation candidate for the linked Chinese-username Scheduled Task failure, though it likely needs a rebase/test refresh against current parser return fields.

## Best Possible Solution

Keep this PR open for maintainer review, or land an equivalent fix in `src/daemon/schtasks.ts`: emit a deterministic UTF-8 code-page bootstrap immediately after `@echo off`, teach `readScheduledTaskCommand` to ignore that exact bootstrap during read-back, and add Chinese-path regression coverage while preserving current `sourcePath` and `environmentValueSources` behavior.
