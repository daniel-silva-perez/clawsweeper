## Summary

Keep open. Current main has restored the duplicate heartbeat suppression half, but the plugin-channel fallback gap is still present: `target: "last"` can recover a stored plugin route from session metadata, then drop it when heartbeat routing re-enters strict outbound resolution against an empty or unbootstrapped plugin registry. The related implementation attempt, #46203, is closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep this issue open. Implement the remaining heartbeat fallback so a stored plugin route recovered for `target: "last"` is preserved or bootstrapped when strict outbound resolution fails only because the plugin registry is empty or unbootstrapped. Leave the restored duplicate suppression in place and add unmocked regression coverage for `resolveHeartbeatDeliveryTarget()` with an empty active plugin registry.
