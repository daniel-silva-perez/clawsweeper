## Summary

Keep open. Current main supports a partial per-agent parameter surface through `agents.list[].params`, and that already covers `temperature` and `maxTokens`, but it does not implement the requested `generation` config block or provider-generic forwarding for `top_p`/`topP`, `top_k`/`topK`, presence/frequency penalties, or related sampling parameters. The closest canonical follow-up for sampling passthrough is open #32496, while #71350 and #59168 are adjacent broader design discussions rather than exact replacements.

## Best Possible Solution

Keep this open until maintainers choose the public config contract. The narrow path is to extend the existing `agents.list[].params` contract, or add a documented `agents.list[].generation` alias if that shape is preferred, then normalize camelCase and snake_case generation keys and forward only supported fields per provider/transport. Coordinate the sampling-field forwarding with open #32496, and link or split the broader per-agent/profile design with #71350 and #59168 if maintainers want model profiles instead of direct per-agent generation fields.
