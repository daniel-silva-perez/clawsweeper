## Summary

Keep open. Current main has a partial per-agent params surface, but it does not implement the requested generation block or the full parameter set in a provider-generic way. `agents.list[].params` can carry arbitrary keys and is merged over defaults, with documented/tested support for `temperature` and `maxTokens`; however the generic stream wrapper only forwards those two fields, while fields like `topP`/`top_p`, `topK`/`top_k`, and penalties are not covered as a general per-agent generation contract. The open related #59168 is broader and may inform the design, but it is not an exact canonical replacement for this per-agent request.

## Best Possible Solution

Keep this open until maintainers choose the public config shape. The narrow implementation path is to extend the existing `agents.list[].params` contract, or add a documented `agents.list[].generation` alias if product wants that shape, then normalize and forward common generation keys per provider with tests for at least OpenAI-compatible, Anthropic, and Ollama-style transports. If #59168 becomes the canonical design for model profiles, link this issue there explicitly and close only once the per-agent or profile-based workflow covers the same user problem.
