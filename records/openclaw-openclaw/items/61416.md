## Summary

Keep open. Current main still drops SearXNG `img_src`: the bundled SearXNG client only normalizes `url`, `title`, and `content`, and the web_search payload mapper only returns `title`, `url`, `snippet`, and `siteName`. The PR is a narrow plugin-owned additive change, and the provided diff does not touch workflows, dependencies, lockfiles, release scripts, or other code-execution surfaces.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a focused SearXNG plugin change that adds optional `img_src` normalization and includes it in the returned web_search result payload, ideally with a mapper-level regression test. No core API change appears necessary.
