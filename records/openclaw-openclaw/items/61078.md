## Summary

Keep this PR open. Current main still does not implement the requested `@tmp/...` or tolerant `tmp/...` alias to `run/tmp/`, and the PR is focused on the right agent file-tool surface. It should not be closed as implemented, but it also should not be merged as-is without resolving the branch-specific gateway-watch failure and the sandbox path-remap concerns raised in review.

## Best Possible Solution

Keep the PR open and have the branch updated against current main with a narrower, tested implementation: map `@tmp/...` and `tmp/...` to workspace `run/tmp/` in the shared file-tool path layer, avoid forwarding pre-remapped host paths into sandbox tools, preserve the canonical `path` runtime contract unless maintainers explicitly want legacy aliases, add coverage for `root=/workspace/openclaw` with `containerWorkdir=/workspace`, and rerun the focused guard tests plus the gateway watch-regression lane before maintainer review.
