## Summary

Keep this PR open. Current main still lacks the requested `@tmp/...` / tolerant `tmp/...` alias to workspace `run/tmp/`, while the PR remains a focused but unfinished change to the agent file-tool path layer with unresolved sandbox-path and validation concerns.

## Best Possible Solution

Keep the PR open and ask for a refreshed, narrower implementation: map `@tmp/...` and `tmp/...` to workspace `run/tmp/` in the shared file-tool path layer, preserve containment under `run/tmp/`, avoid forwarding pre-remapped host paths into sandbox tools, add coverage for `root=/workspace/openclaw` with `containerWorkdir=/workspace`, and rerun the focused guard tests plus the gateway watch-regression lane.
