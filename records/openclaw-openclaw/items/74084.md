## Summary

Keep this PR open. Current main still only caches evaluated bindings by the cfg WeakMap entry plus that cfg's bindings reference; it does not have the proposed bindings-reference/fingerprint fallback for rebuilt cfg objects. The PR is a live routing performance proposal with a runtime cache design that needs maintainer review and benchmark/correctness validation, not cleanup closure.

## Best Possible Solution

Keep this PR open for routing maintainer review. The best path is to validate the cache design with targeted resolve-route tests plus a small benchmark or perf trace for rebuilt cfg objects, confirm correctness under OpenClaw's config immutability expectations, then either land a refined optimization or close it manually if the tradeoff is not worth carrying.
