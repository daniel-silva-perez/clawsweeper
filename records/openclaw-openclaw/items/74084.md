## Summary

Keep this PR open. Current main still only caches evaluated binding indexes by config object identity and bindings reference, while this PR proposes a live routing hot-path optimization for rebuilt config objects. The diff is narrow and not security-sensitive, but it still needs maintainer review for the unresolved FIFO-vs-LRU note, benchmark value, and config immutability assumptions.

## What This Changes

The PR adds secondary evaluated-binding caches keyed by bindings array identity and SHA-256 binding-content fingerprints, plus resolve-route tests for rebuilt configs with identical or changed bindings.

## Best Possible Solution

Keep this PR open for routing maintainer review. The best path is to resolve the FIFO-vs-LRU review item or adjust the wording, add/keep targeted resolve-route coverage for rebuilt cfg objects, capture a small benchmark or perf trace for large binding sets, validate config immutability/content-cache assumptions, then either land a refined optimization or close it manually if the tradeoff is not worth carrying.
