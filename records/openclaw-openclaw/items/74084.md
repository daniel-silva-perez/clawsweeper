## Summary

Keep this PR open. Current main still only caches evaluated bindings by the `OpenClawConfig` object and current `bindings` reference, while this PR proposes new bindings-reference and content-fingerprint fallbacks for rebuilt configs; that is not implemented on main and the open PR still needs routing maintainer review for the eviction semantics, benchmark value, and config immutability assumptions.

## What This Changes

This PR adds secondary evaluated-bindings cache lookup by bindings array identity and SHA-256 content fingerprint, plus tests for rebuilt configs with identical or changed binding content.

## Best Possible Solution

Keep this PR open for routing maintainer review. The best path is to resolve or reword the FIFO-vs-LRU eviction behavior, add or keep focused rebuilt-cfg tests, capture a small benchmark/perf trace for large binding sets, validate config immutability assumptions, and land only after the focused routing checks and changed gate are green.
