## Summary

Keep open. Current main does not implement the PR's cross-config evaluated-bindings cache reuse, and the branch is still an open performance/cache-semantics change that needs routing maintainer review before merge.

## What This Changes

This PR adds a `node:crypto` SHA-256 binding-content fingerprint cache and a bindings-array `WeakMap` fallback in `src/routing/resolve-route.ts`, plus rebuilt-config cache tests in `src/routing/resolve-route.test.ts`.

## Best Possible Solution

Land a small routing-owned cache improvement only after the eviction semantics are made accurate, rebuilt-cfg tests remain focused, perf evidence supports the added complexity, config immutability assumptions are accepted, and the routing/changed gates are green.
