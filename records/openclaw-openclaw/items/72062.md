## Summary

Keep this issue open. Current main still has per-agent tool-name allow/deny and positive exec allowlist handling, but no typed, validated, documented, or enforced per-agent negative policy for shell wrapper or script paths. The requested behavior remains a real core exec-policy gap rather than something implemented, duplicate, stale, or safely moved to ClawHub.

## Best Possible Solution

Keep this issue open for maintainer design and implementation. A good direction is a typed and documented per-agent exec path deny policy, probably under `agents.list[].tools.exec`, that reuses existing shell wrapper/script candidate resolution, applies before approval prompts on gateway and node exec paths, makes deny win over allowlist and durable approvals, and includes docs plus regression coverage for `sh|bash <script>` and interpreter script operands.
