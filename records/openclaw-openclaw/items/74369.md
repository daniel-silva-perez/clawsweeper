## Summary

Keep this PR open. Current main does not render the proposed plain-text/autocorrect suppression attributes on the Control UI chat textarea, while the linked bug reports browser/OS rewriting before the send path sees the draft. The PR is narrow and appears aligned with the reported UI-layer problem; it needs normal maintainer review and validation rather than cleanup closure.

## What This Changes

This PR adds plain-text input attributes to the Control UI chat composer textarea and adds UI tests for those attributes plus exact numeric `chat.send` payload preservation.

## Best Possible Solution

Land a narrow Control UI composer fix that suppresses browser/OS text rewriting at the textarea, leaves gateway and send normalization untouched, and keeps regression coverage for the rendered attributes plus exact numeric message delivery.
