## Summary

Keep open. Current main still treats cron as Gateway-owned stored jobs with optional agent binding, while the strict per-agent config type and runtime schema do not define an `agents.list[].cron` contract. The prior ClawSweeper keep-open review remains consistent with the current source and docs.

## What This Changes

Request to add a declarative `cron` array to each agent config entry so scheduled tasks are owned by agents and loaded into the Gateway scheduler.

## Best Possible Solution

If accepted, add an additive core config contract for agent-owned scheduled jobs: typed config, strict schema and generated schema/help updates, startup and reload compilation into scheduler jobs, deterministic namespacing, clear precedence over stored jobs, docs, and focused tests for config validation, scheduler merge behavior, and duplicate prevention.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection provide a high-confidence current-main check: adding `cron` under an agent is not defined by the strict config schema and is not loaded by the Gateway scheduler.

## Solution Assessment

Unclear pending maintainer product review. The proposed direction is coherent, but the maintainable implementation should be an additive core config/scheduler contract because this cannot be reliably provided by a plugin without a core extension point for agent config and scheduler merge semantics.
