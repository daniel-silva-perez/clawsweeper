## Summary

Keep open. Current main still supports cron as Gateway-owned stored jobs with optional agent binding, but it does not implement declarative `agents.list[].cron` entries, startup scanning/merging from agent config, ID namespacing, or jobs.json precedence handling. The related ClawSweeper items are adjacent cron UX/metadata requests, not a canonical duplicate of this feature.

## Best Possible Solution

Keep this issue open for maintainer product/design review. A complete implementation would belong in core scheduler/config work: add typed `agents.list[].cron` schema and docs, normalize entries into cron jobs at Gateway startup, define deterministic namespacing and jobs.json precedence, and cover validation, startup merge/reload behavior, and multi-agent inheritance with tests. If maintainers prefer a plugin route, first add a supported plugin API for contributing scheduled jobs from declarative config.
