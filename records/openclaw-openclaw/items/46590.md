## Summary

Keep open. Current main still has Gateway-owned cron jobs persisted in `~/.openclaw/cron/jobs.json` with optional `agentId` binding, but it does not define, validate, document, or load an `agents.list[].cron` declarative task field. The issue is concrete, only has a stale label plus prior keep-open review context, and there is no merged implementation or canonical superseding item in the provided context.

## Best Possible Solution

Keep this issue open for maintainer product/design review. If accepted, implement an additive `agents.list[].cron` config contract that validates declarative job specs, compiles them into scheduler jobs at Gateway startup and reload, preserves `jobs.json` as the mutable CLI/Control UI store, documents precedence, prevents duplicate runs, and adds focused config, cron, and docs coverage.
