## Summary

Keep open. Current main still has Gateway-owned cron jobs persisted through the cron store with optional `agentId` binding, but it does not define, validate, document, or load an `agents.list[].cron` declarative task field. The prior ClawSweeper keep-open review remains consistent with current source and docs, and the related items in the provided context are not canonical implementations of this request.

## Best Possible Solution

Keep this open for maintainer product/design review. If accepted, implement an additive `agents.list[].cron` contract that validates declarative job specs, compiles them into scheduler jobs on Gateway startup and config reload, preserves `jobs.json` as the mutable CLI/Control UI store, defines namespacing and precedence, prevents duplicate runs, and updates config schema/help plus cron docs/tests.
