## Summary

Keep open. Current main still does not surface session `label`/`displayName` through the `/status` recent-session pipeline or the TUI status summary/statusline. Cron sessions already populate a friendly `Cron: <job name>` label, and the Control UI dropdown label path is already addressed, but the remaining `/status` and TUI statusline work is still represented by open implementation candidates #51257 and #1139, both of which explicitly close this issue. Per maintainer policy, the issue should stay open until that PR work is reviewed, landed, or closed.

## Best Possible Solution

Review and land a focused implementation such as #51257, or its fork #1139 if that becomes the active branch. The fix should carry `label` and `displayName` through the status summary/GatewayStatusSummary contracts, prefer friendly labels in CLI `/status`, TUI `/status`, and the footer/statusline, and add targeted tests for cron-labeled sessions plus fallback-to-key behavior. Once that PR merges, close this issue as implemented by the merge.
