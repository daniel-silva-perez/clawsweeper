## Summary

Current main still does not solve the remaining `/status` and TUI statusline request. Cron sessions do have a stored friendly label, and the Control UI dropdown portion was addressed by #45130, but the status summary contract and status renderers still omit `label`/`displayName` and render raw session keys. Two open PR candidates, #51257 and fork #1139, explicitly link this issue for closure, so maintainer policy says to keep this issue open until that work lands or is closed.

## Best Possible Solution

Review and land #51257, or the active fork #1139 if maintainers choose that branch. The implementation should carry `label` and `displayName` through `SessionStatus` and `GatewayStatusSummary`, render a consistent friendly fallback in CLI `openclaw status`, TUI `/status`, TUI statusline/footer, and the session picker, and add focused tests for cron-labeled sessions plus fallback-to-key behavior.
