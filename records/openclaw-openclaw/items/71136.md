## Summary

Keep open. Current main still has precedence-based token resolution paths that can return an agent/session/configured value before lower model or discovery limits, and the output-token behavior is not consistently conservative across transports.

## What This Changes

The issue requests conservative contextTokens and maxTokens resolution by using the minimum positive limit available from agent config, provider model config, model discovery, and defaults.

## Best Possible Solution

Introduce a shared positive-finite token-limit resolver that gathers candidates from agent defaults/per-agent overrides, configured provider/model metadata, runtime discovery/model metadata, and defaults, then applies the conservative minimum while preserving provider-qualified lookup semantics and documented long-context opt-ins. Use the same pattern for output maxTokens at request/transport boundaries and update tests plus docs to define the contract.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence path: configure agent/session contextTokens or provider model metadata above a lower discovered/runtime limit, and current code can return the higher value before the lower one; for maxTokens, Anthropic Messages accepts a positive requested value before model limits.

## Solution Assessment

Yes. The requested minimum-based direction is the narrow maintainable fix, but it should be implemented through normalized candidate collection rather than a bare Math.min over ambiguous model IDs.
