## Summary

Keep open. Current main still does not classify `application/msword` or `application/x-cfb` as binary in the media-understanding prompt embedding path, the latest release tag shows the same gap, and #73799 is an open unmerged implementation PR that references this issue with closing syntax.

## What This Changes

The issue requests that legacy Word/OLE `.doc` attachments with `application/x-cfb` or `application/msword` be skipped or converted instead of embedded into prompts as raw text.

## Best Possible Solution

Land #73799 or an equivalent narrow patch that treats `application/msword` and `application/x-cfb` as binary in `src/media-understanding/apply.ts`, keeps vendor `+json`/`+xml` text behavior intact, and adds focused regression tests in `src/media-understanding/apply.test.ts`; close this only after that fix reaches main.
