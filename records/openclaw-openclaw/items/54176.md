## Summary

Current `main` still lacks the requested explicit legacy Word/OLE binary MIME guard in the media-understanding prompt embedding path, and open PR #54380 remains the implementation candidate linked with closing syntax for this issue. This should stay open until that PR, or an equivalent scoped fix, lands or is intentionally rejected by a maintainer.

## Best Possible Solution

Keep this issue open and review/land #54380, or an equivalent scoped patch, that adds `application/x-cfb` and `application/msword` to the media-understanding binary MIME guard in `src/media-understanding/apply.ts` with focused coverage in `src/media-understanding/apply.test.ts`. If maintainers decide the current unsupported-MIME fallback is sufficient, close or replace #54380 first and document that judgment before closing this issue.
