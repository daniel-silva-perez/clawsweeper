## Summary

Keep this PR open. Current main still lacks the immediate ACP spawn-task transcript seed that the linked bug describes, but the branch should not merge as-is because it is stale, broad, and has a concrete false-confirmation bug around `taskDelivered`.

## What This Changes

The branch seeds ACP `sessions_spawn(runtime="acp")` transcripts with the initial task, adds provenance-aware replay dedupe and prompt-only transcript tests, and also changes several CI/labeler workflows.

## Best Possible Solution

Land a narrow, rebased ACP-only fix that seeds the initial spawn task into the child transcript before returning, reports delivery truthfully, preserves current main’s newer ACP spawn/runtime behavior, dedupes only the seeded replay, and keeps the CI/labeler workflow changes in a separate maintainer-reviewed PR.
