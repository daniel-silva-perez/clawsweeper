## Summary

Keep open: current main still does not seed ACP `sessions_spawn` task text into the child transcript before dispatch, and the discussion includes useful empirical validation that the empty-history behavior is real. This PR is still relevant, but it should not merge as-is because it reports delivery even when the seed write is skipped or fails, misses required changelog attribution, is stale against current main, and mixes the ACP fix with security-sensitive workflow/token policy changes.

## What This Changes

The PR seeds ACP `sessions_spawn` transcripts with the initial task, forwards input provenance into the first ACP dispatch, dedupes the seeded replay, adjusts prompt-only session-manager initialization, adds ACP transcript/spawn tests, updates the changelog, and changes several CI/labeler workflows.

## Best Possible Solution

Land a narrow, rebased ACP-only fix that seeds the initial spawn task into the child transcript before returning, dedupes only the seeded replay by provenance or marker, reports delivery truthfully only when the seed was written, preserves current main's newer ACP spawn behavior, and keeps CI/labeler policy changes in a separate maintainer-reviewed PR if still needed.
