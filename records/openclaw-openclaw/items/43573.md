## Best Possible Solution

Land a narrow, rebased ACP-only fix that writes the initial spawn task to the child transcript before reporting success, dedupes only the first seeded replay by marker/provenance, preserves current main’s newer ACP spawn behavior, and moves workflow policy changes to a separate maintainer-owned PR if still needed.

## Solution Assessment
