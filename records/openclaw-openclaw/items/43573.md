## Summary

Keep this PR open. Current main still lacks the PR's intended immediate ACP spawn transcript seed, so the branch is not obsolete; however, the proposed patch is not merge-ready because it can report `taskDelivered: true` even when the seed write is skipped or fails, its changelog entry misses required contributor credit, and it mixes the ACP fix with security-sensitive CI/token workflow changes.

## What This Changes

The PR seeds ACP `sessions_spawn` transcripts with the initial task, forwards input provenance into the first ACP dispatch, dedupes the seeded replay, adjusts prompt-only session-manager initialization, adds ACP transcript/spawn tests, updates the changelog, and modifies CI/labeler workflows.

## Best Possible Solution

Land a narrow, rebased ACP-only fix that writes the initial spawn task into the child transcript before returning, dedupes only the seeded replay by provenance or marker, reports seed/delivery status truthfully, preserves current main's newer ACP behavior, includes focused regression tests, and moves workflow policy changes to a separate maintainer-reviewed PR if still needed.
