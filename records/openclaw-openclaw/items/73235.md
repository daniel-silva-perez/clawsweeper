## Summary

Keep open. Current main still contains the BlueBubbles routing behavior this PR is trying to change: `chat_guid` targets are always classified as groups, short message-id resolution has no caller chat context, group inbounds without identifiers can still fall back through the sender handle path, and group reactions without identifiers still synthesize the literal `group` peer id. This PR is an active implementation candidate, not obsolete cleanup.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is to review the final head, require the BlueBubbles routing/security concerns to be addressed with focused regression tests, and then land or request changes; closing now would leave the current main behavior unfixed.
