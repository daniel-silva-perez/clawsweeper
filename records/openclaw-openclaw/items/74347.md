## What This Changes

The PR modifies the ACPX Codex wrapper generator to symlink `~/.codex/auth.json` into the isolated `codex-home` at wrapper startup and adds tests for generated wrapper contents plus symlink creation.

## Best Possible Solution

Keep this PR open for maintainer comparison with #73943. The best outcome is one canonical, security-reviewed Codex ACP auth bridge that preserves generated config isolation, decides symlink-only versus copy fallback deliberately, respects the expected Codex auth-home source, updates docs/changelog, and keeps focused ACPX regression tests.

## Risks / Open Questions

- The PR intentionally exposes host Codex `auth.json` to the managed Codex ACP adapter process; that is probably the desired fix, but it needs maintainer/security sign-off on the trust boundary.
- The branch hard-codes `os.homedir()/.codex/auth.json` and does not respect an explicit `CODEX_HOME`, so it may not cover users whose working Codex auth lives outside the default home.
- Symlink failures are silently ignored, which preserves fallback behavior but can leave the original authentication failure without a clearer OpenClaw diagnostic.
- The branch lacks docs/changelog updates that #73943 already includes, so merging this exact diff would leave user-facing ACP docs behind the behavior.
