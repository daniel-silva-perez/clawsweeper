## Summary

Keep this issue open. Current main still lacks a `bindings[].nickname` or equivalent Discord identity override for ordinary route bindings. The existing Discord webhook persona path only applies to bound thread delivery, so it does not solve one bot token serving multiple agent-bound Discord channels with distinct visible names.

## Best Possible Solution

Keep the issue open and implement the feature in the bundled Discord plugin. A conservative path is to add a documented Discord route-binding identity override, or reuse `agents.list[].identity` for route-selected agents, then extend Discord outbound delivery to use per-channel webhooks where available with safe fallback to bot-token sends. Update config types/schema/help, Discord docs, and tests covering normal channel bindings, missing webhook permission, and webhook echo suppression.

## Risks / Open Questions

- Discord bot nickname mutation may be guild-scoped rather than channel-scoped; implementation should validate Discord API behavior and likely prefer per-message webhook identity where possible.
- Extending webhook persona delivery to ordinary channel bindings needs explicit fallback and echo-suppression behavior for missing Manage Webhooks permission, media/poll/silent sends, and allowBots configurations.
