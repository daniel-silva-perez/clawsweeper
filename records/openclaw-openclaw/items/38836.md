## Summary

Keep this issue open. Current main has improved managed restart handling, restart intent, single-session restart sentinel delivery, and internal restart-aborted main-session recovery, but it still does not implement the reported Feishu-visible notification/fan-out for all in-flight Feishu sessions interrupted by a direct supervisor restart such as `systemctl --user restart openclaw-gateway.service`. The prior ClawSweeper comment correctly identified the same remaining gap, and current main still matches that assessment.

## Best Possible Solution

Keep this issue open until OpenClaw either implements generic external-channel restart interruption handling or explicitly documents direct supervisor/systemctl restarts as outside the supported no-loss boundary. The likely fix belongs in gateway restart/session recovery: persist affected delivery contexts before shutdown or while marking aborted sessions, then on startup send a Feishu-visible interruption notice or resume-and-deliver through the original route with regression coverage for Feishu or a generic channel delivery contract.
