## Summary

Keep this issue open. Current main still has restart hardening and single-session restart-sentinel delivery, but it does not implement Feishu-visible notification or fan-out for all in-flight Feishu sessions interrupted by a direct supervisor restart such as systemctl restart.

## Best Possible Solution

Keep this issue open until OpenClaw either implements generic channel-visible restart interruption handling or explicitly documents direct supervisor/systemctl restarts as outside the supported no-loss boundary. The likely fix belongs in gateway restart/session recovery: durably identify affected in-flight sessions and delivery contexts before shutdown or during startup lock cleanup, then enqueue an idempotent Feishu-visible interruption notice or resume-and-deliver through the original route, with regression coverage for Feishu or a generic channel delivery contract.
