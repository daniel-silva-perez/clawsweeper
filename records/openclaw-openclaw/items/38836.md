## Summary

Keep this issue open. Current main has restart hardening, a single restart-sentinel delivery path, and internal main-session restart recovery, but it still does not implement a Feishu-visible notification for all in-flight Feishu sessions interrupted by a direct supervisor restart such as `systemctl --user restart`. Direct SIGTERM without OpenClaw restart intent remains a stop path, startup recovery resumes marked main sessions with `deliver: false`, and the restart sentinel is scoped to one optional session/delivery context rather than a fan-out over all interrupted channel sessions.

## Best Possible Solution

Keep this issue open until OpenClaw either implements generic channel-visible restart interruption handling or explicitly documents direct supervisor/systemctl restarts as outside the supported no-loss boundary. The likely implementation belongs around gateway shutdown/startup recovery: durably identify interrupted in-flight sessions and their delivery contexts, then enqueue idempotent same-channel notices or resume-and-deliver through the original route, with coverage for Feishu or a generic channel delivery contract.
