## Summary

Keep this PR open. Current main partially addresses #60686 by constraining workspace auto-discovery to `.openclaw/extensions`, but it does not implement this PR’s narrower behavior: manifest load failures are still untyped, and `loadPluginManifestRegistry` still emits every missing-manifest failure as an error, including workspace-origin candidates. The supplied PR remains a plausible focused fix, superseding the earlier closed #60701 attempt with a typed failure reason and tests for workspace warn versus non-workspace error behavior.

## Best Possible Solution

Review and land this PR or an equivalent change that adds a typed manifest-load failure reason, downgrades only workspace-origin missing-manifest candidates to warnings, preserves errors for non-workspace and unsafe/parse/invalid failures, and keeps the focused regression tests from the PR.
