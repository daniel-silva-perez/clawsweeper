## Summary

Keep open. Current main still has the workspace missing-manifest severity gap, and the PR remains a focused fix, but its diff no longer applies cleanly to current main and needs maintainer review or rebase before landing.

## What This Changes

The PR adds typed native/bundle manifest-load failure reasons, downgrades only workspace-origin missing-manifest diagnostics to warnings, and adds registry tests for workspace versus global severity.

## Best Possible Solution

Land this PR after rebasing it onto current main, or land an equivalent change that keeps the severity downgrade limited to workspace-origin missing manifests, preserves hard errors for all other manifest failures, and keeps the focused registry regression tests.

## Reproduction Assessment

Yes. A workspace-origin candidate whose root lacks openclaw.plugin.json still flows through loadPluginManifest into an unconditional registry error on current main; the PR's proposed test shape using loadSingleCandidateRegistry is a high-confidence reproduction path, though it was not executed in this read-only review.

## Solution Assessment

Yes, conceptually. A typed failure reason is the narrow maintainable fix because it avoids brittle error-string matching while preserving error behavior for unsafe, parse, invalid, and non-workspace missing-manifest failures; the branch just needs rebase/adaptation before it can land.
