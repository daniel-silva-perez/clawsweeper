## Summary

Keep open. Current main has substantial SecretRef support, audit/configure/apply flows, and plaintext scrubbing, but it still does not implement built-in encrypted `.env` values, transparent `ENC:` decryption, master-key/keyring setup, or key rotation; this is security-sensitive product/API design rather than conservative cleanup.

## What This Changes

Request built-in encrypted-at-rest handling for sensitive configuration values such as API keys and passwords, including transparent `ENC:` storage/decryption and key management.

## Best Possible Solution

Keep this as an explicit secops/product design item: define a first-party encrypted/keyring-backed SecretRef provider or local vault, choose migration and key-rotation behavior for plaintext `.env` and auth-profile values, and document fail-fast/redaction semantics before implementation.
