## Summary

Keep open. Current main has mature SecretRef support, audit, configure/apply, and one-way plaintext scrubbing, but it still does not provide built-in encrypted-at-rest `.env` values, transparent `ENC:` decryption, a master-key/keyring setup flow, or key rotation. The request is security/product design work rather than conservative cleanup.

## What This Changes

Request built-in encrypted-at-rest storage and transparent `ENC:` decryption for sensitive `.env` configuration values such as API keys and passwords.

## Best Possible Solution

Design a first-party encrypted/keyring-backed SecretRef provider or local vault that integrates with the existing activation snapshot, audit/apply, onboarding, docs, and fail-fast semantics, then add a deliberate migration path for plaintext `.env` and auth-profile values plus key rotation behavior.
