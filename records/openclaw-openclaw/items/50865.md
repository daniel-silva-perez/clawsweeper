## Summary

Keep this issue open. Current main has a Control UI sessions list, session switching, session delete/archive behavior, and backup/export-adjacent workarounds, but I do not see a first-party WebUI path to restore a session after the plus button `/new` reset or after `sessions.delete`. Ryce's KeepMyClaw comment is useful external recovery context, but it does not make the requested OpenClaw WebUI restore flow implemented.

## Best Possible Solution

Keep the issue open and scope a core session-recovery design: add a gateway API that can safely enumerate archived reset/deleted transcript artifacts, preview them, and restore or branch them into a live session row; then wire that into the Control UI Sessions view with confirmation, docs, and tests. KeepMyClaw can remain a recommended external backup option, but it should not replace a first-party restore path if maintainers want this UX in OpenClaw.
