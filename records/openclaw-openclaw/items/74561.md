## Summary

Keep this PR open. Current main still has the diagnostics flag contract gap the PR targets: false-like `OPENCLAW_DIAGNOSTICS` values parse to an empty env flag list, then config flags are still merged in, which conflicts with the docs promise that `OPENCLAW_DIAGNOSTICS=0` disables all flags. The proposed patch is narrow, aligns the shared resolver with the documented contract, and adds focused regression coverage.

## What This Changes

The PR changes diagnostics flag parsing so false-like `OPENCLAW_DIAGNOSTICS` values override config flags to disable all diagnostics, and adds resolver and timeline regression tests.

## Best Possible Solution

Land a resolver-level fix that treats false-like diagnostics env values as explicit disable overrides, keeps blank env values as no-ops, preserves normal config-plus-env union semantics, and includes the added regression coverage for both the resolver and timeline gate.
