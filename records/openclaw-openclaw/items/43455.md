## Summary

Current main still has the reported Browserbase gap: the public browser docs recommend the static Browserbase WebSocket URL, while the browser runtime treats profile `cdpUrl` as a direct or discovered CDP endpoint and has no Browserbase Sessions API step before Playwright connects.

## What This Changes

Request to make Browserbase browser profiles create a Browserbase session and connect to its returned `connectUrl` instead of relying on a static `cdpUrl`.

## Best Possible Solution

Browserbase support should live in the browser plugin as explicit session lifecycle handling: accept Browserbase credentials or env-backed configuration, create a session through Browserbase before CDP attach, use the returned `connectUrl`, clean up the session when OpenClaw releases it, and update docs/tests while leaving generic remote CDP behavior unchanged.
