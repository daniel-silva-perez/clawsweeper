## Summary

Current main still has the reported Browserbase gap: the docs recommend the static Browserbase WebSocket URL, while the browser runtime treats profile `cdpUrl` as a direct/discovered CDP endpoint and does not create a Browserbase session before connecting. The issue is valid and should stay open for a focused browser plugin repair.

## What This Changes

The issue requests Browserbase-aware browser profile handling that creates a Browserbase session through the REST API and connects to the returned ephemeral CDP URL instead of using a static `cdpUrl`.

## Best Possible Solution

Browserbase support should live in the browser plugin as explicit session lifecycle handling: accept safe credential/config inputs, create a Browserbase session before CDP attach, connect Playwright to the returned `connectUrl`, clean up or respect Browserbase session lifetime when OpenClaw releases the connection, and update docs/tests while leaving generic remote CDP behavior unchanged.
