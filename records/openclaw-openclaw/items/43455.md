## Summary

Current main still matches the reported Browserbase breakage: OpenClaw docs still advertise the static Browserbase WebSocket URL, while the browser plugin continues to treat configured CDP URLs as direct endpoints and has no Browserbase session creation, returned connectUrl handling, or release lifecycle.

## What This Changes

The issue requests Browserbase session API support because the documented static Browserbase cdpUrl now returns 404 and Browserbase requires creating a session before connecting.

## Best Possible Solution

Update the browser plugin and user docs so Browserbase profiles create, reuse, and release Browserbase sessions through the provider API instead of relying on the obsolete static root WebSocket URL, with mocked tests for credentials, connectUrl use, failures, rate-limit messaging, and cleanup.

## Repair Work Prompt

Fix openclaw/openclaw#43455. Current docs recommend `browser.profiles.browserbase.cdpUrl = "wss://connect.browserbase.com?apiKey=..."`, but Browserbase now requires creating a session first and connecting to the returned `connectUrl`. Keep vendor-specific behavior inside the bundled browser plugin; do not move Browserbase-specific policy into core. Implement a small Browserbase remote-CDP session adapter in `extensions/browser` that detects Browserbase profiles/URLs, obtains the API key from the existing query parameter or a documented env/config path, optionally supports project id if already available or with a minimal documented field, calls `POST https://api.browserbase.com/v1/sessions`, validates/redacts the returned `connectUrl`, connects Playwright/CDP to that URL, caches/reuses the session for the normalized profile while appropriate, and releases/terminates the Browserbase session on explicit browser/profile close when supported. Preserve generic Browserless/direct-CDP behavior. Update `docs/tools/browser.md` and any config help/schema docs for the new Browserbase flow; add a single-line changelog Fixes entry with `Thanks @HanenVit` if changelog policy applies. Add mocked tests for session creation, connectUrl use, API errors, missing credentials, rate-limit messaging/redaction, and cleanup; do not require live Browserbase credentials.
