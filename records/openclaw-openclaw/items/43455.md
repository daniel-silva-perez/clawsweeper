## Summary

Current main still matches the reported Browserbase breakage. The docs still recommend the static Browserbase WebSocket URL, the bundled browser plugin still treats configured CDP URLs as direct endpoints, and I found no Browserbase session creation or returned connectUrl lifecycle support.

## Best Possible Solution

Keep this issue open. The best maintainer path is to update `docs/tools/browser.md` so it no longer recommends the obsolete static Browserbase root URL, then implement tested Browserbase session lifecycle support in `extensions/browser` or introduce a vendor-neutral remote-CDP session-provider seam. Coverage should verify session creation, returned connectUrl use, errors, rate-limit messaging, and session release/termination behavior.
