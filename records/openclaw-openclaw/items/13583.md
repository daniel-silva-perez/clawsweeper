## Summary

Keep this issue open. Current main has adjacent pre-tool blocking, final-answer revision, reply-dispatch, and outbound message-cancel hooks, but it still lacks the requested fail-closed response gate with mandatory tool-call accounting, typed validators, two-phase planning, bounded retry/fallback behavior, and audit records. The discussion usefully identifies complementary plugin work, while preserving the core gap that plugins cannot yet mechanically prevent final response delivery until required tools and validators pass.

## What This Changes

The issue requests a pre-response enforcement layer that blocks final answer delivery until configured mandatory tool calls and validator checks have run and passed.

## Best Possible Solution

Keep this open as the canonical response-gate policy-as-code design item. The desired end state is a versioned core/plugin API for mandatory tool-call accounting, typed response validators, fail-closed final-response gating, bounded retry/fallback behavior, and audit records, with domain-specific policy packs implemented as plugins or ClawHub packages.
