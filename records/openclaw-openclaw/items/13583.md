## Summary

Keep this open. Current main has adjacent hooks for pre-tool blocking, final-answer revision, reply dispatch, and outbound delivery cancellation, but it still lacks the requested fail-closed response gate with mandatory tool-call accounting, typed validators, two-phase planning, bounded retry/fallback behavior, and audit records. The issue discussion usefully identifies complementary plugin implementations while preserving the core gap: plugins can gate tools or filter/cancel output, but cannot yet mechanically prevent final response delivery until required tools and validators pass.

## What This Changes

The issue requests a pre-response enforcement layer that blocks final assistant output until configured mandatory tool calls and validator checks have run and passed.

## Best Possible Solution

Keep this as the canonical response-gate policy-as-code design item. The desired end state is a versioned core/plugin API for mandatory tool-call accounting, typed response validators, fail-closed final-response gating, bounded retry/fallback behavior, and audit records, with domain-specific policy packs implemented as plugins or ClawHub packages.
