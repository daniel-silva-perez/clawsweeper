## Summary

Keep this open. Current main has adjacent pre-tool blocking, final-answer revision, reply-dispatch, and outbound delivery cancellation hooks, but it still lacks the requested fail-closed response gate with mandatory tool-call accounting, typed validators, two-phase planning, bounded retry/fallback behavior, and audit records. The discussion is useful because linked plugin work covers complementary tool execution or output filtering paths while preserving the remaining core/plugin API gap.

## What This Changes

Request a fail-closed pre-response enforcement layer that requires configured validators or mandatory tool calls to pass before final assistant output is delivered.

## Best Possible Solution

Keep this as the canonical response-gate policy-as-code design item. The desired end state is a versioned core/plugin API for mandatory tool-call accounting, typed response validators, fail-closed final-response gating, bounded retry/fallback behavior, and audit records, with domain-specific policy packs implemented as plugins or ClawHub packages.
