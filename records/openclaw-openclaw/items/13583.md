## Summary

Current main still has adjacent policy hooks for pre-tool blocking, final-answer revision, reply dispatch, and outbound message cancellation, but it does not implement the requested fail-closed pre-response responseGate with mandatory tool-call accounting, typed validators, bounded retry/fallback behavior, two-phase planning, or audit records. The issue discussion and existing maintainer review identify this as a real core/plugin API gap, not something already solved by the current hook surfaces.

## Best Possible Solution

Keep this issue open as the canonical response-gate policy-as-code design item. The best maintainer path is a versioned core/plugin API for mandatory tool-call accounting, typed response validators, fail-closed final-response gating, bounded retry/fallback behavior, and audit records; domain-specific policy packs can then live in plugins or ClawHub.
