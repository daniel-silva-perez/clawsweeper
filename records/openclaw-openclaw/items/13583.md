## Summary

Keep this open. Current main has adjacent pre-tool blocking, final-answer revision, reply-dispatch, and outbound message cancellation hooks, but it still does not provide the requested fail-closed response gate with mandatory tool-call accounting, typed validators, two-phase planning, retry/fallback policy, and audit records.

## What This Changes

The issue requests a fail-closed pre-response enforcement layer that can require tool calls and validator checks before final assistant output is delivered.

## Best Possible Solution

Keep this as the canonical response-gate policy-as-code design item. The desired end state is a versioned core/plugin API for mandatory tool-call accounting, typed response validators, fail-closed final-response gating, bounded retry/fallback behavior, and audit records, with domain-specific policy packs implemented as plugins or ClawHub packages.
