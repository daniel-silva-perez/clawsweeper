## Summary

Keep open. Current main still lacks the proposed gateway-owned, account-scoped async resource manager for channel plugin resources; the existing surfaces are per-account start/stop hooks, opt-in lifecycle helpers, synchronous runtime-context leases, and generic host cleanup hooks. The related Feishu cleanup bug and PR remain open, and the broader ResourceManager tracker overlaps but is not clearly canonical enough to close this channel-specific design request.

## What This Changes

The issue requests a centralized PluginResourceManager-style interface so channel plugins can start, stop, and fully clean up async resources consistently before gateway restarts.

## Best Possible Solution

Maintainers should make an explicit API decision: either add a backward-compatible account-scoped async lifecycle seam to the plugin SDK/gateway with stop-accepting-new-work, drain or cancellation, timeout, and forced-dispose semantics, then prove it by migrating Feishu plus at least one other high-risk channel with contract tests; or intentionally consolidate this request into #49804 after the Feishu-specific cleanup path is resolved.
