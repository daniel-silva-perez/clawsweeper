## Summary

Keep this issue open. Current main has partial lifecycle infrastructure for channel start/stop and generic plugin host cleanup callbacks, but it does not implement the requested centralized PluginResourceManager-style contract with uniform async cleanup across channel plugins. The Feishu cleanup problem cited by the issue is still visible on main, and the linked Feishu PR remains open rather than merged.

## Best Possible Solution

Keep this open for maintainer product/design judgment. The best path is to first resolve the narrow Feishu cleanup PR/bug, then decide whether to add a documented, backward-compatible channel/plugin resource lifecycle seam with tests across at least two channel plugins, or intentionally keep cleanup extension-owned and close this later with that rationale.
