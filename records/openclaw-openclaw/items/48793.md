## Summary

Keep open. Current main still has per-channel start/stop hooks and task abort tracking, but no centralized PluginResourceManager-style async resource ownership API. The motivating Feishu HTTP server cleanup race remains visible, PR #48588 is still open and Feishu-specific, and #49804 overlaps broader lifecycle design without superseding this channel-plugin-specific SDK question.

## Best Possible Solution

Keep this open for maintainer design review. The best path is to decide whether the public plugin SDK should grow an account-scoped async resource registration/lifecycle contract, then migrate one or two high-risk channel resources such as Feishu HTTP servers as contract-test examples. Handle the immediate Feishu bug through #48588/#48183, and coordinate this narrower channel-plugin API request with #49804 rather than closing it as already implemented.
