## Summary

Keep open. Current main still has per-account channel start/stop hooks plus opt-in lifecycle helpers, but no gateway-owned account-scoped async resource manager for channel plugins. The motivating Feishu close-before-delete pattern is still present, and the related Feishu bug/PR plus broader lifecycle design tracker remain open.

## What This Changes

Request a centralized PluginResourceManager interface for consistent async start, stop, and cleanup across channel plugins.

## Best Possible Solution

Make an explicit maintainer API decision: either add a backward-compatible account-scoped async resource lifecycle seam to the plugin SDK and prove it by migrating Feishu plus at least one other high-risk channel with contract tests, or consolidate this channel-specific request into #49804 after the Feishu-specific cleanup bug is handled separately.
