## Summary

Keep open. Current main still has per-account channel start/stop hooks, opt-in lifecycle helpers, synchronous runtime-context leases, and a generic plugin host cleanup hook, but not the gateway-owned, account-scoped async resource manager proposed here. The Feishu close-before-delete pattern that motivated the request is also still present, while the related Feishu-specific bug/PR and broader resource-lifecycle design tracker remain open.

## What This Changes

The issue asks OpenClaw to add a centralized PluginResourceManager-style interface so channel plugins initialize and asynchronously clean up account-scoped resources consistently.

## Best Possible Solution

Make an explicit maintainer API decision: either add a backward-compatible account-scoped async resource lifecycle seam to the plugin SDK and prove it by migrating Feishu plus at least one other high-risk channel with contract tests, or intentionally consolidate the channel-specific request into #49804 after the Feishu-specific cleanup path is handled.
