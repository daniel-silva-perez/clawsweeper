## Summary

Keep open. Current main still does not provide the requested gateway-owned, account-scoped async resource manager for channel plugin resources; the implemented surfaces are per-account start/stop hooks, opt-in SDK lifecycle helpers, synchronous runtime-context leases, and generic plugin host cleanup hooks. The motivating Feishu close-before-delete cleanup shape is still present, while the Feishu-specific bug/PR and broader resource-lifecycle tracker remain open rather than clearly superseding this channel-specific API request.

## What This Changes

The issue requests a centralized PluginResourceManager-style async lifecycle interface for channel plugins so gateway restarts can consistently await per-account resource cleanup.

## Best Possible Solution

Maintainers should make an explicit API call: either add a backward-compatible account-scoped async lifecycle seam to the plugin SDK/gateway with drain, timeout, and forced-dispose semantics, then prove it by migrating Feishu plus at least one other high-risk channel with contract tests; or intentionally consolidate the channel-specific request into #49804 after the Feishu-specific cleanup path is resolved.
