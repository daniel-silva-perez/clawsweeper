## Summary

Keep open. Current main still uses per-account channel start/stop hooks plus opt-in SDK lifecycle helpers, not a gateway-owned async resource manager for channel plugin resources. The Feishu-specific cleanup bug and the broader resource-lifecycle design tracker are both still open, so this needs maintainer API/product review rather than conservative cleanup closure.

## What This Changes

The issue requests a centralized PluginResourceManager-style async lifecycle interface so channel plugin resources can be started, stopped, and fully cleaned up consistently by the gateway.

## Best Possible Solution

Maintainers should decide the lifecycle contract first: either add a backward-compatible, account-scoped async resource lifecycle seam in the plugin SDK/gateway and prove it by migrating Feishu plus another high-risk channel with contract tests, or explicitly consolidate the channel-specific request into #49804 while keeping the Feishu leak on #48183/#48588.

## Reproduction Assessment

Not applicable as a direct reproduction because this is a feature/design request; the source check gives high confidence that the requested API is absent. The motivating Feishu bug has a concrete path in #48183, and current code still shows close-before-delete cleanup in the Feishu monitor state and transport paths.

## Solution Assessment

Unclear until maintainer API review. A centralized PluginResourceManager may be a maintainable direction, but the safer next step is to decide whether this belongs as a channel-account SDK seam here or as part of the broader #49804 resource lifecycle design.
