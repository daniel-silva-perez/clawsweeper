## Summary

Keep open. Current main still lacks the requested gateway-owned, account-scoped async resource manager for channel plugins; the existing implementation remains a mix of per-account hooks, opt-in lifecycle helpers, and plugin-specific cleanup paths, so this is not conservative cleanup-closeable.

## What This Changes

The issue asks for a centralized PluginResourceManager-style async lifecycle interface so channel plugin resources are started, stopped, and fully cleaned up consistently across accounts.

## Best Possible Solution

Maintainers should make an explicit API decision: either add a backward-compatible, account-scoped async lifecycle seam to the plugin SDK/gateway with drain, timeout, and forced-dispose semantics, then migrate Feishu plus at least one other high-risk channel with contract tests; or intentionally consolidate this channel-specific request into #49804 after the Feishu-specific cleanup bug is handled.
