## Summary

Keep open. Current main still exposes per-account channel start/stop hooks and opt-in lifecycle helpers, not a gateway-owned async PluginResourceManager contract, and the motivating Feishu close-before-delete cleanup pattern is still present. This is a public plugin SDK/channel lifecycle design issue, so it should not be swept closed or queued as a narrow automated repair without maintainer API review.

## What This Changes

The issue requests a centralized PluginResourceManager-style async lifecycle contract for channel plugin resources so startup, stop, and teardown cleanup are consistent across channel plugins.

## Best Possible Solution

Maintainers should decide the lifecycle contract first: either add a backward-compatible, account-scoped async resource lifecycle seam in the plugin SDK/gateway and prove it by migrating Feishu plus another high-risk channel with contract tests, or explicitly consolidate the generic resource-manager design into #49804 while keeping the Feishu leak tracked by #48183/#48588.

## Reproduction Assessment

Not applicable as a direct reproduction because this is a feature/design request. The source check gives high confidence that the requested API is absent, and the motivating Feishu race remains visible in the close-before-delete code path.

## Solution Assessment

Unclear until maintainer API review. A centralized PluginResourceManager may be the right direction, but the safer next step is to decide whether the contract belongs as a channel-account SDK seam here or as part of the broader #49804 resource lifecycle design.
