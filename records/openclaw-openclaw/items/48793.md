## Summary

Keep open. Current main still lacks the requested gateway-owned, account-scoped async resource manager for channel plugin resources; the available surfaces are per-account start/stop hooks, opt-in SDK lifecycle helpers, generic plugin host cleanup callbacks, and synchronous runtime-context leases. The related Feishu cleanup bug/PR and broader async resource-lifecycle tracker remain open, with no maintainer decision consolidating this channel-specific request.

## What This Changes

The issue requests a centralized PluginResourceManager-style lifecycle interface so channel plugins can start, stop, and fully clean up async resources consistently.

## Best Possible Solution

Maintainers should make an explicit API decision: either add a backward-compatible, account-scoped async lifecycle seam to the plugin SDK/gateway with drain, timeout, and forced-dispose semantics, then migrate Feishu plus at least one other high-risk channel with contract tests; or intentionally consolidate this channel-specific request into #49804 after the Feishu-specific cleanup bug is handled.
