## Summary

Keep open. Current main has per-account channel start/stop hooks, task abort tracking, synchronous runtime-context leases, and generic plugin runtime cleanup hooks, but it still does not provide the proposed centralized account-scoped async resource manager for channel plugins. The Feishu HTTP server cleanup pattern that motivated the request is still present, and the related narrower Feishu PR plus broader lifecycle issue remain open rather than merged/shipped superseders.

## What This Changes

The issue requests a centralized PluginResourceManager-style lifecycle contract so channel plugins can consistently start, stop, and fully clean up async resources before gateway restart or teardown.

## Best Possible Solution

Make an explicit maintainer API decision: either add a backward-compatible, account-scoped async resource lifecycle seam to the plugin SDK and prove it by migrating Feishu plus one other high-risk channel with contract tests, or consolidate this channel-specific request into the broader #49804 design track after the Feishu bug is handled separately.
