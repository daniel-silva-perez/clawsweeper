## Summary

Keep open. Current main has per-account channel start/stop hooks, optional lifecycle helpers, synchronous runtime-context leases, and generic plugin lifecycle cleanup, but it still does not provide the proposed centralized account-scoped async resource manager for channel plugins. The Feishu server cleanup pattern that motivated the request is still present, and the related Feishu-specific PR plus broader lifecycle design issue remain open.

## What This Changes

Introduce a centralized `PluginResourceManager`-style interface with async `start`, `stop`, and `cleanup` semantics so channel plugin resources can be consistently tracked and drained before gateway restart.

## Best Possible Solution

Make an explicit maintainer API decision: either add a backward-compatible account-scoped async resource lifecycle seam to the plugin SDK and prove it by migrating Feishu plus at least one other high-risk channel with contract tests, or consolidate this request into #49804 after the Feishu-specific cleanup bug is handled separately.
