## Summary

Keep this issue open. Current main still has the same-target media-only suppression path: message-tool media can record target/media metadata without sent text, and final/follow-up reply delivery still suppresses all session text based on target metadata alone. The useful linked fix attempt in PR #37900 targeted the right behavior but was closed unmerged, and the current tests still do not cover same-target media-only preservation.

## Best Possible Solution

Patch buildReplyPayloads and resolveFollowupDeliveryPayloads so same-origin target suppression only drops final or follow-up session text when the message tool actually sent text to that same target. Media-only sends should still dedupe already-sent media URLs while preserving the agent's session text. Add focused regression tests for same-target media-only preservation, same-target text-send duplicate suppression, and the follow-up delivery path.

It is not gated on messagingToolSentTexts being non-empty, so same-target media-only sends can still drop the session text.
