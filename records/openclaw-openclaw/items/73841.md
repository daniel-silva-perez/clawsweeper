## Summary

Keep this PR open. Current main still has the gateway dedupe cleanup under-eviction bug, and the PR’s production change is pointed at the right code path. The regression test needs adjustment because it only overflows by one entry, which current main already handles and already evicts by timestamp. The PR also overlaps with #45979, so maintainers should choose one canonical patch before landing.

## Best Possible Solution

Keep this PR open as a plausible fix candidate, but ask for a regression test that fails on current main by overflowing `dedupe` by more than one entry. The production change should precompute the excess or the keys-to-delete before mutation, preserve timestamp-based eviction, and then maintainers should reconcile it with #45979 so only one dedupe cleanup patch lands.
