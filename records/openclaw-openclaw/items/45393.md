## Summary

Keep this PR open. Current main still has the user-facing formatter gap the PR targets: Anthropic `tool_use`/`tool_result` mismatch payloads still fall through to the generic `LLM request rejected: ...` invalid-request branch. Main does have broader dangling tool-call stripping that appears to cover much of the final-assistant-message repair path, but the PR is not obsolete, and its bundled inbound-media prompt-injection hardening is also not present on main.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is to rebase or cherry-pick the missing formatter guard and focused `formatAssistantErrorText` tests, confirm or add exact final-assistant-message repair coverage against the current broader repair logic, and split the inbound-media untrusted-data wrapper into a separate security-focused PR if maintainers want that hardening reviewed independently.
