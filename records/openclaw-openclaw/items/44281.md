## Repair Work Prompt

Implement a narrow replacement/refresh for PR 44281 on current main. Observable bug: agent replies shaped like `[[reply_to:<id>]] MEDIA:./photo.jpg` are not delivered as Telegram media because `parseReplyDirectives()` extracts media before stripping reply tags, while `splitMediaFromOutput()` only recognizes lines whose trimmed start is `MEDIA:`. Fix boundary: keep this in shared reply directive/outbound normalization plus Telegram delivery tests; do not add core Telegram IDs or broad product behavior. Preserve existing plain-text and code-block whitespace. Do not call `parseInlineDirectives(raw)` before media extraction in a way that normalizes unrelated text. Prefer a helper/path that strips reply tags only for the media extraction scan while still returning reply metadata and cleaned delivery text. If wiring `[[reply_to_current]]`, pass `currentMessageId` only from Telegram dispatch/native-command call sites that have the triggering message id. Add regression coverage for `[[reply_to:42]] MEDIA:./photo.jpg` in outbound/parser behavior and a Telegram `deliverReplies` test proving `sendPhoto` is called with `reply_to_message_id: 42`. Add/adjust tests for whitespace preservation if the helper changes directive stripping. Changelog: this is user-facing Telegram behavior, so add a single-line Fixes entry in the active changelog with `Thanks @hugh-agent` unless maintainers decide to attribute differently. Do not touch workflows, dependencies, lockfiles, package metadata, generated files, or unrelated Telegram behavior.
