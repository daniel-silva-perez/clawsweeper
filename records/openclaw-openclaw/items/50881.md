## Summary

Keep this PR open. Current main still does not implement the helper-level Slack reaction idempotency this PR targets: `already_reacted` from `reactSlackMessage`, `no_reaction` from `removeSlackReaction`, and remove-own cleanup races still propagate outside the status-reaction adapter. The PR is stale and needs rebase/tests, and #71843/#58242 overlap on the duplicate-add half, but they do not fully ship or supersede the broader remove/no_reaction work here.

## Best Possible Solution

Keep this PR open for maintainer review or contributor rebase. The best path is a current-main Slack patch that handles `already_reacted` as success in the shared add helper, handles `no_reaction` as success for explicit removal and remove-own cleanup if maintainers accept that scope, preserves current `cfg`/token/client/write-client contracts, and adds focused helper/action-runtime regression tests while coordinating with #69005, #71843, and #58242.
