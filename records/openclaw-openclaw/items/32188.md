## Summary

Keep open. Current main has adjacent Control UI, /status, /usage, sessions, model-auth, and usage-report surfaces, but it still lacks the requested persistent in-interface status area showing the active agent/model plus token, quota, and subscription context during operations. Related open requests #46398 and #71630 cover narrower slices and do not supersede the broader request.

## Best Possible Solution

Keep this open for maintainer product/UX review. The best path is to decide whether this remains the umbrella issue or is split across narrower follow-ups, then implement a persistent Control UI status surface that reuses sessions.changed/session rows, models.authStatus, and usage APIs with explicit redaction and reliability rules.
