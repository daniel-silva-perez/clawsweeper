## Summary

Keep open. Current main has adjacent CLI/chat status commands, usage dashboards, session metadata, context warnings, and model-auth quota cards, but it still does not provide the requested persistent in-interface status area for active agent/runtime model plus token, quota, subscription, or cost context during operations. The related #73744 PR is open and narrower: it addresses persistent chat context usage, not the broader agent/model/quota/subscription status surface.

## What This Changes

Add a persistent in-interface status area/status bar that shows the active agent/model plus token, quota, subscription, and related efficiency status during operations.

## Best Possible Solution

Keep this as a product/UX umbrella until maintainers decide the exact Control UI shape and whether to split it after #73744. The end state should be a persistent status surface that combines active session agent, actual runtime model, fresh context-token data, permitted response usage/cost, provider quota/subscription windows, and clear stale/unavailable/redacted states without relying on transcript messages or manual status commands.
