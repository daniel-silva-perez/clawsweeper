## Summary

Keep open. Current main still has adjacent controls, warning notices, per-message metadata, usage commands, and provider-auth overview cards, but it does not provide the requested persistent operation-time status area that combines active agent/model with token, quota, subscription, and cost context.

## What This Changes

The issue requests a persistent in-interface status area showing the active agent/model plus token, quota, subscription, or cost context during operations.

## Best Possible Solution

Keep this as a product/UX umbrella until maintainers define one Control UI status surface that composes active session agent/runtime, actual runtime model, fresh context-token usage, permitted response usage/cost, provider quota/subscription windows, and clear stale/unavailable/redacted states without relying on manual status commands.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Current-main source, tests, and docs give a high-confidence gap because existing surfaces are controls, warnings, per-message details, reports, or command workarounds rather than the requested persistent active status area.

## Solution Assessment

Unclear for implementation. The requested direction is valid, but the best path is maintainer-scoped UX/API design first, then focused implementation that coordinates the narrower topbar, persistent context, active-agent, and provider-quota work without duplicating surfaces.
