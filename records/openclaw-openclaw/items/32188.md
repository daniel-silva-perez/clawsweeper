## Summary

Keep open. Current main still has adjacent model controls, usage reports, context warnings, and session metadata, but not the requested persistent in-interface status area showing the active agent/runtime model plus token, quota, subscription, or cost context during operations.

## What This Changes

The issue requests a persistent status area in the interface that shows the active agent/model and relevant token, quota, subscription, or cost information during work.

## Best Possible Solution

Keep this as a product/UX umbrella until maintainers define the canonical Control UI placement and data contract for a persistent status surface that composes active session agent, actual runtime model, fresh context usage, response usage/cost, provider quota/subscription windows, and clear stale/unavailable/redacted states.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current-main code/docs check gives a high-confidence gap: existing surfaces are controls, command/report workarounds, per-message/draft metadata, or threshold-gated warnings rather than the requested persistent status area.

## Solution Assessment

Unclear for implementation. The requested direction is valid, but the best path is maintainer-scoped UX/API design first, then a focused implementation that composes the narrower topbar, context-usage, and provider-quota requests without duplicating them.
