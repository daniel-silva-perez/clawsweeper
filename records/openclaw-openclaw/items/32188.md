## Summary

Keep open. Current main has adjacent Control UI, session, usage, model-auth, and context-warning surfaces, but it still does not provide the requested persistent in-interface status area for active agent/model plus token, quota, and subscription context during operations. A newer open PR covers the narrower persistent context-usage slice only, so it does not supersede this broader product/UX request.

## What This Changes

The issue requests a persistent UI status area showing the active agent/model and token, quota, and subscription context during OpenClaw operations.

## Best Possible Solution

Keep this as a product/UX umbrella until maintainers decide whether to split it. The desired end state is a persistent Control UI status surface that combines session runtime metadata, fresh context-token data, model-auth/provider usage windows, and clear unavailable/redacted states without relying on transcript messages or manual /status checks.
