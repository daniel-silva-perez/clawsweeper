## Summary

Keep this enhancement open. Current main has adjacent Control UI session/model controls, a usage report page, an overview auth/usage card, `/status` and `/usage` surfaces, and live session metadata, but it still does not provide the requested persistent Control UI status area showing the active agent/model plus token, quota, and subscription context during operations.

## Best Possible Solution

Keep the enhancement open for maintainer product/UX review. The best path is a persistent Control UI status surface in chat/app chrome that reuses existing sessions.changed metadata, sessions rows, model-auth status, and usage seams for active session/agent/run state and selected provider/model, then adds privacy-aware context usage, token/cache/cost, provider quota window, auth/subscription, and fallback indicators only where reliable data exists.
