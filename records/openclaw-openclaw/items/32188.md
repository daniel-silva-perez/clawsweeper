## Summary

Keep open. Current main still has adjacent status, usage, context-warning, and model-auth surfaces, but it does not provide the requested persistent in-interface status area for the active agent/runtime model plus token, quota, subscription, or cost context during operations. Related work exists, especially #73744 and the narrower topbar/quota requests, but those do not fully close this broader product/UX request yet.

## What This Changes

Add a persistent interface status area or status bar showing the active agent/model plus token, quota, subscription, and related efficiency status during operations.

## Best Possible Solution

Keep this as a product/UX umbrella until maintainers decide how to compose the narrower active-model, persistent context, and provider quota work. The end state should be a persistent Control UI status surface that shows the active session agent, actual runtime model, fresh context-token usage, permitted response usage/cost, provider quota/subscription windows, and clear stale/unavailable/redacted states without relying on manual status commands.
