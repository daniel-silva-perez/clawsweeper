## Summary

Keep open. Current main has adjacent session/model controls, usage reports, a high-context warning, session metadata plumbing, and provider-auth overview cards, but it still lacks the requested persistent operation-time status area combining active agent/model with token, quota, subscription, and cost context. Related #73744 is still open and appears to cover only the narrower persistent context-usage subset, not this broader UX request.

## What This Changes

The issue requests an in-interface status area showing the active agent, active model, token usage or remaining quota, subscription/cost context, and related runtime status during operations.

## Best Possible Solution

Keep this as a product/UX umbrella until maintainers define one Control UI status surface that composes active session agent/runtime, actual runtime model, fresh context-token usage, permitted response usage/cost, provider quota/subscription windows, and clear stale/unavailable/redacted states without relying on manual status commands.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, tests, docs, and related-item context provide a high-confidence current-main gap because the existing surfaces are controls, warnings, reports, or command workarounds rather than the requested persistent active status area.

## Solution Assessment

Unclear for implementation. The requested direction is valid, but the best path is maintainer-scoped UX/API design first, then a focused implementation that coordinates the narrower topbar, persistent context, active-agent, and provider-quota work without duplicating surfaces.
