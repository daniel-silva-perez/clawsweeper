## Summary

Keep this issue open. Current main has adjacent session, usage, /status, /usage, context-warning, and transient run-state surfaces, but the browser Control UI still lacks the requested persistent in-interface status area showing the active agent/model plus token, quota, and subscription context during operations.

## Best Possible Solution

Keep the enhancement open for maintainer product/UX review. The likely direction is a persistent browser Control UI status surface in app/chat chrome that reuses existing session-change and usage seams to show active session/agent/run state and selected provider/model, then adds privacy-aware context usage, token/cache/cost, provider quota windows, auth/subscription state, and fallback state where reliable data exists.
