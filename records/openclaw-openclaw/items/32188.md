## Summary

Keep open. Current main still has related model controls, context warnings, per-message metadata, usage reports, and provider-auth cards, but it does not provide the requested persistent in-interface status area showing the active agent/runtime model plus token, quota, subscription, or cost context during operations. The related open PR #73744 appears to cover only persistent context usage and is not merged, so this broader product request remains unresolved.

## What This Changes

Request a persistent UI status area showing the active agent/model and useful token, quota, subscription, or cost information during operations.

## Best Possible Solution

Keep this as a product/UX umbrella until maintainers define one canonical Control UI status surface that composes active session agent/runtime, actual runtime model, fresh context-token usage, response usage/cost, provider quota/subscription windows, and clear stale/unavailable/redacted states without relying on manual status commands.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Current-main source, tests, and docs give a high-confidence gap: existing surfaces are controls, warnings, per-message details, reports, or command workarounds rather than the requested persistent active status area.

## Solution Assessment

Unclear for implementation. The requested direction is valid, but the best path is maintainer-scoped UX/API design first, then a focused implementation that coordinates the narrower topbar, persistent context, and provider-quota work without duplicating surfaces.
