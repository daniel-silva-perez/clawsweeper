## Summary

Keep this issue open. Current main has adjacent functionality for model selection, threshold-based context warnings, transient fallback/compaction notices, overview/auth usage cards, `/status`, `/usage`, and a TUI footer/status line, but the browser Control UI still lacks the requested persistent in-interface status area showing the active agent/model plus token, quota, and subscription context during operations.

## Best Possible Solution

Keep the enhancement open for maintainer product/UX review. The best path is a persistent browser Control UI status surface in the chat/application chrome that shows active agent/session/run state and selected provider/model, then adds privacy-aware context usage, token/cache/cost, provider quota windows, auth/subscription state, and fallback state where reliable data exists.
