## Summary

Keep open: current main still has no reset-command bypass in preflight compaction, so the PR is not obsolete. The contribution is plausibly aimed at a real latency path, but it needs maintainer review before merge because the proposed regex also bypasses `/reset soft`, which current code and docs define as transcript-preserving rather than a full session clear.

## What This Changes

The PR adds a reset-command early return to `runPreflightCompactionIfNeeded`, adds focused regression coverage, and adds a changelog entry for the Discord reset-command compaction timeout case.

## Best Possible Solution

Land a narrow reset-command preflight bypass only after aligning it with the real reset semantics: full `/new` and `/reset` should avoid wasted compaction before session rotation, while `/reset soft` should either keep normal compaction or get an explicit maintainer-approved rationale and coverage because it preserves the transcript.
