## Summary

Keep open: current main still lacks a reset-command preflight-compaction bypass, but the latest PR branch should not merge as-is because it drops unrelated active changelog entries and bypasses compaction for `/reset soft`, which current main treats as transcript-preserving.

## What This Changes

The PR adds a preflight compaction early return for prompts beginning with `/new` or `/reset`, adds focused regression tests, and adds an Unreleased changelog bullet.

## Best Possible Solution

Revise this PR so it preserves current changelog content and aligns the bypass with reset semantics: full `/new` and hard `/reset` can skip wasted preflight compaction, while `/reset soft` should stay on the normal compaction path unless maintainers explicitly approve different transcript-preserving behavior with coverage.
