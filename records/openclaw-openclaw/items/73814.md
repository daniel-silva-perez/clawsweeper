## What This Changes

The issue asks OpenClaw to fix the documented macOS/Linux installer one-liner so interactive installer prompts do not consume the piped shell script stream and truncate shell code.

## Best Possible Solution

Patch and publish the installer path so the hosted `openclaw.ai/install.sh` artifact and documented one-liners are consistent, all interactive reads and child onboarding commands use `/dev/tty` or an explicit noninteractive mode safely under piped execution, and a WSL/Linux smoke covers the documented interactive path.
