## Summary

Keep open. Current main still treats `$VAR` and `${VAR}` placeholders literally in shared user path resolution, so this PR addresses a real open bug linked to #53628, but the current patch should not merge as-is because missing variables collapse to empty strings and can redirect `$HOME`-based paths to root instead of using the existing home fallback.

## What This Changes

The PR adds an exported `expandEnvVars` helper to `src/infra/home-dir.ts`, calls it from `resolveHomeRelativePath` and `resolveOsHomeRelativePath`, and extends `src/infra/home-dir.test.ts` with env-placeholder path tests.

## Best Possible Solution

Land one canonical shared path-helper fix that expands `$VAR` and `${VAR}` for configured user paths, avoids silently redirecting missing placeholders to root, covers config/state/workspace/skill-install paths with focused tests, and then resolves #53628 while superseding the duplicate PR.

## Reproduction Assessment

Yes. The linked #53628 Docker/ClawHub report gives concrete steps, and current main has a source-level reproduction because non-tilde `${XDG_CONFIG_HOME}/...` input flows through `resolveHomeRelativePath` to `path.resolve(trimmed)` unchanged.

## Solution Assessment

No. Expanding in `src/infra/home-dir.ts` is the right boundary, but replacing missing variables with `""` is not the safest implementation; preserve unresolved placeholders, report them, or special-case home variables through the existing `resolveRequiredHomeDir` / `resolveRequiredOsHomeDir` fallback path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve home fallback for missing env vars:** `src/infra/home-dir.ts:115`
  - body: `expandEnvVars` currently turns any unset variable into an empty string. That makes paths like `$HOME/.openclaw` resolve to `/.openclaw` when `HOME` is absent, even though the existing home resolver can fall back to `USERPROFILE` or `os.homedir()`. Please avoid collapsing missing vars, or route home variables through the existing fallback logic before resolving the path.
  - confidence: 0.87
