## Summary

Keep open. Current main has Discord thread parent detection, thread starter context, parent model/transcript inheritance, and a manual Discord pins action, but I did not find automatic parent-channel pin fetching/injection or a thread pin-inheritance config. The request is actionable in the bundled Discord plugin and is not covered by the provided related reports.

## Best Possible Solution

Implement this in the bundled Discord plugin, likely in the thread preflight/process path: add a documented config such as channels.discord.thread.inheritParentPins, fetch parent channel pins with the existing Discord REST client when a thread parent is known, normalize/filter them through the same supplemental-context trust controls, inject them as untrusted session context or explicit metadata, and cover schema, docs, and thread-session tests.

## Risks / Open Questions

- Automatic pin injection would add Discord user-authored content to prompts, so any implementation should treat pins as untrusted context and honor existing contextVisibility/allowlist behavior.
- Fetching parent pins on every thread session start could affect latency, Discord API rate limits, and prompt token budget; a config flag, caching, limits, and tests would be needed.
