## Summary

Keep this issue open. Current main still rejects `--gateway-auth trusted-proxy` in non-interactive onboarding, while runtime config, public docs, and the interactive gateway flow already support trusted-proxy auth. Same-author draft PR #49107 is an implementation candidate, and related issue #73639 is not a shipped canonical fix.

## Best Possible Solution

Keep this issue open and implement the feature in core onboarding. Accept `--gateway-auth trusted-proxy`, add explicit non-interactive flags for `gateway.trustedProxies`, `gateway.auth.trustedProxy.userHeader`, `requiredHeaders`, `allowUsers`, `allowLoopback`, and any needed Control UI allowed origins, then update CLI docs and add focused non-interactive tests. Review or replace draft PR #49107 as the implementation candidate.
