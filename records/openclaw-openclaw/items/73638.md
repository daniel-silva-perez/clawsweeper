## Summary

Keep this issue open. Current main still rejects `--gateway-auth trusted-proxy` in non-interactive onboarding, while runtime config, startup validation, interactive onboarding, docs, and tests show trusted-proxy auth is a supported core gateway mode. The same-author draft PR #49107 is an implementation candidate, not a merged fix, and #73639 is the newer duplicate-looking report.

## Best Possible Solution

Keep this issue open and implement the feature in core onboarding, preferably by reviewing or replacing draft PR #49107. The fix should accept `--gateway-auth trusted-proxy`, add explicit non-interactive flags for `gateway.trustedProxies`, `gateway.auth.trustedProxy.userHeader`, required headers, `allowUsers`, `allowLoopback`, and Control UI allowed origins, update the public onboard docs, and add focused tests for valid config writes plus missing or unsafe security fields. Treat #73639 as the newer duplicate rather than closing this older tracking issue.
