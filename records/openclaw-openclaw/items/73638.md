## Summary

Keep this issue open. Current main still rejects `--gateway-auth trusted-proxy` in non-interactive onboarding, while runtime config, docs, interactive configuration, and tests already support trusted-proxy auth. The same-author draft PR #49107 is an implementation candidate, not a merged fix, and the duplicate-looking #73639 is newer than this issue.

## Best Possible Solution

Keep this issue open and implement the feature in core onboarding, preferably by reviewing or replacing draft PR #49107. The fix should accept `--gateway-auth trusted-proxy`, add explicit non-interactive flags for `gateway.trustedProxies`, `gateway.auth.trustedProxy.*`, and Control UI allowed origins, update `docs.openclaw.ai/cli/onboard`, and add focused tests for valid config writes plus missing or unsafe security fields. Treat #73639 as the newer duplicate rather than closing this older tracking issue.
