## Summary

Keep this PR open. Current main still does not implement `session.reset.mode = "off"`: the runtime reset mode, exported config type, Zod schema, generated schema/help, and public docs all still expose only `daily` and `idle`. The PR remains a plausible core implementation candidate, but it should be tightened before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate. The best path is a focused update against current main that adds `"off"` consistently to exported config types, Zod validation, generated schema/help, public docs, and `evaluateSessionFreshness`; adds a direct regression test showing stale daily/idle timestamps remain fresh under `off`; regenerates the config baseline; and removes or splits the unrelated Plugin SDK runtime-logger cleanup.
