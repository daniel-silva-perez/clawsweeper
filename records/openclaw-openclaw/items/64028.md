## Summary

Current main still does not implement `session.reset.mode = "off"`; the PR remains a plausible core implementation candidate, but it needs cleanup before maintainer merge. The reset scheduler, config types, Zod schema, generated schema/help, and public docs still expose only `daily` and `idle` on main. The provided PR diff adds the intended core config/runtime behavior but still needs direct regression coverage, generated baseline proof, public docs alignment, and removal or splitting of the unrelated Plugin SDK return-type cleanup.

## Best Possible Solution

Keep this PR open as the implementation candidate. The best maintainer path is a focused update against current main that adds `session.reset.mode: "off"` consistently across exported types, Zod validation, generated schema/help/docs, and `evaluateSessionFreshness`; adds a direct unit test for stale daily/idle timestamps staying fresh with off mode; regenerates and commits the config baseline hash; updates public session/config docs; and removes or splits the unrelated Plugin SDK runtime-logger cleanup.
