## Summary

Keep this issue open. Current main still models node commands as string allowlist entries only; there is no per-command input/output schema declaration in connect, node pairing, discovery, or invoke validation. The related #8287 is adjacent but broader because it asks for node-registered agent tools, while this issue is specifically a gateway command-schema protocol gap.

## Best Possible Solution

Keep this issue open for maintainer-designed additive gateway protocol work: preserve commands: string[] compatibility, add optional per-command schema metadata, persist and expose it through connect, pairing, registry/catalog, node.list/node.describe, generated clients, node-host/plugin runtime APIs, docs, and tests. Define explicit node.invoke input and output validation semantics, while treating #8287 as related but separate node-tool registration work.
