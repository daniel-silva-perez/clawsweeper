## Summary

Keep this issue open. Current main still represents node command declarations as `string[]` across the connect handshake, node pairing, runtime registry, discovery, and generated/mobile client surfaces; `node.invoke` params/results remain unknown payloads without command-specific schema validation. The related #8287 is adjacent, but it asks for node-registered agent tools and does not supersede this narrower gateway command-schema protocol request.

## Best Possible Solution

Keep open for a maintainer-designed additive gateway protocol change. Preserve `commands: string[]` compatibility, add optional per-command metadata/schema declarations, carry them through connect, node pairing request/persistence, registry/session state, `node.list`/`node.describe`, generated Swift/Kotlin clients, node-host/plugin runtime APIs, docs, and focused tests. Validation semantics for `node.invoke` params/results should be explicit.
