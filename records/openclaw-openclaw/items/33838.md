## Summary

Keep this issue open. Current main still only lets nodes declare command names, not per-command input/output schemas. `node.invoke` validates the RPC envelope and command allowlist, then forwards arbitrary params/results. The related #8287 is adjacent but broader: node-registered agent tools, not this gateway command-schema protocol gap.

## Best Possible Solution

Keep open for maintainer-designed additive gateway protocol work. Preserve `commands: string[]` compatibility, add optional per-command schema metadata, carry it through connect, node pairing request/persistence, registry/session state, `node.list`/`node.describe`, generated Swift/Kotlin clients, node-host/plugin runtime APIs, docs, and focused tests. Define explicit validation behavior for `node.invoke` params and results, and keep #8287 as a related but separate node-tool registration discussion.
