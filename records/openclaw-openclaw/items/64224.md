## Summary

Keep open. Current main still treats billing-classified failures as auth-profile-wide disabled state, so a shared LiteLLM proxy profile can remain unavailable for sibling models. The related billing-summary user-message work is present on main, but it does not change cooldown scope or add a supported zero/disable opt-out.

## What This Changes

Request model-scoped or proxy-aware billing cooldowns, or a supported opt-out, so one LiteLLM upstream model's 402 does not disable the shared provider auth profile.

## Best Possible Solution

Add a deliberate billing-cooldown scope policy for OpenAI-compatible aggregators such as LiteLLM: preserve provider-wide disables for real key/account exhaustion, but avoid blocking sibling proxy models when the failure is model/upstream-specific, or expose a documented operator opt-out with safe retry behavior. Update config schema/help/docs and regression tests for one LiteLLM profile serving two models.
