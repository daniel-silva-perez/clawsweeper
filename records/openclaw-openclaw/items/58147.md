## Summary

Keep open. Current main still treats `agents.defaults.timeFormat` as agent prompt context, while the browser Control UI and native SwiftUI WebChat timestamp paths do not receive or apply that setting. This is a real in-repo UI behavior gap, not a stale/duplicate/ClawHub cleanup candidate.

## Best Possible Solution

Keep this issue open and implement a small, generic UI-facing time-format path: resolve `agents.defaults.timeFormat` once in the gateway, expose it through a stable Control UI/WebChat surface such as `sessions.list.defaults` or the bootstrap snapshot, then thread it into browser and Swift timestamp/duration formatters with focused tests for 12h and 24h display.
