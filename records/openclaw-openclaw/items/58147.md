## Summary

Keep open: current main still has no data path for `agents.defaults.timeFormat` to affect Control UI/WebChat timestamp rendering. The gateway sessions defaults payload omits the setting, the UI type cannot carry it, and the chat timestamp formatter uses locale defaults directly.

## What This Changes

The issue requests that Control UI/WebChat session duration and timestamp display honor `agents.defaults.timeFormat`.

## Best Possible Solution

Expose a browser-safe `timeFormat` value through the sessions/defaults or equivalent Control UI payload, then apply explicit 12h/24h formatting in the chat/session timestamp renderer while preserving locale behavior for `auto`. Add focused gateway and UI tests; update docs only if maintainers want the setting documented as UI-visible too.

## Reproduction Assessment

Yes for a source-level reproduction: set `agents.defaults.timeFormat` to `24` and inspect the Control UI chat timestamp path. `sessions.list` cannot deliver that setting today, and `formatChatTimestampForDisplay` has no input that could change 12h/24h output.

## Solution Assessment

Yes, the narrow maintainable fix is to thread the existing config value through the gateway UI contract and format in browser-safe UI code. If maintainers decide `timeFormat` must remain prompt-only, the alternative is to document that boundary and open a separate UI preference request.
