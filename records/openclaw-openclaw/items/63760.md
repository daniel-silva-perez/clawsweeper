## Summary

Keep open. Current main still has the reported cost-control gap: announce relay calls use the requester session's normal agent path, while existing cheaper-model and lightweight-context controls apply to spawned child runs rather than the completion announce turn itself.

## What This Changes

Request configurable cheaper-model and light-context controls for subagent and ACP completion announce runs so result relays do not pay full parent-session model and cache costs.

## Best Possible Solution

Add a schema-backed announce policy for completion relays, likely under subagent defaults with matching per-agent inheritance, that can resolve a configured cheaper model through the normal model catalog/auth/allowlist path and optionally run the announce turn without requester history while preserving existing delivery routing and docs.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: follow any completion announce path to the gateway `agent` call and observe that it passes no model/context override, then follow agent execution to see it resolve the requester session's default or stored model and active history.

## Solution Assessment

Yes, the requested direction is the right narrow fix if implemented through existing config and model-selection machinery. A hardcoded cheap model or direct text send would be less maintainable because announce delivery still needs policy, routing, and internal-event synthesis.
