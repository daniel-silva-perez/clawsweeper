## Summary

Keep open. Current main still has the reported cost-control gap: completion announce relay calls use the requester session's normal agent path, while existing cheaper-model and lightweight-context controls apply to spawned child runs or heartbeat/plugin helper runs rather than to the announce relay itself.

## What This Changes

Issue requests configurable cheaper-model and light-context controls for subagent/ACP completion announce relay runs.

## Best Possible Solution

Add a schema-backed announce relay policy, likely under subagent defaults with matching per-agent inheritance, then wire all completion announce paths to resolve a configured model and optional lightweight/no-history context without changing child spawn semantics.
