## Summary

Keep open. Current main still has the reported cost-control gap: completion announce relay calls continue through the requester session's normal agent path, while the existing cheaper-model and light-context settings apply to spawned child runs rather than the announce turn itself.

## What This Changes

The issue requests announce-specific cost controls for subagent and ACP completion announcements, including a cheaper announce model override and a light or no-history context mode.

## Best Possible Solution

Add a schema-backed announce policy for completion relays, likely under subagent defaults with per-agent inheritance where the existing subagents config supports it. The policy should resolve a configured cheaper model through the normal catalog/auth/allowlist machinery and optionally run the announce turn with isolated or no requester history while preserving current delivery behavior and docs.
