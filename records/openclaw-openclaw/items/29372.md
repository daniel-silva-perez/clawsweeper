## Summary

Keep this issue open. Current main still lacks `agents.list[].subagents.workspace`: the per-agent subagent type and strict runtime/generated schemas omit it, docs document only model/thinking/timeout/allowlist/requireAgentId defaults, and same-agent/default native subagent spawns still inherit the requester workspace. Related workspace-inheritance fixes are present, but they do not add a configurable default-subagent workspace override.

## Best Possible Solution

Keep open for maintainer product/security review. If approved, add a backwards-compatible `agents.list[].subagents.workspace` key, normalize it like other agent workspace paths, update TypeScript/zod/generated schemas and public docs, and adjust omitted-`agentId` plus explicit same-agent native spawn workspace resolution to prefer that override while preserving existing explicit cross-agent target-workspace behavior.
