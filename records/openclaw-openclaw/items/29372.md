## Summary

Keep this issue open. Current main still does not implement `agents.list[].subagents.workspace`: the per-agent subagent config type, strict runtime schema, generated schema, and public docs omit the field, and the native spawn path still defaults omitted `agentId` to the requester agent while preserving the caller workspace for same-agent/default subagents. The related #40171 cross-agent workspace fix is present, but it does not cover the requested default subagent workspace override.

## Best Possible Solution

Keep this open for maintainer decision on a core config/runtime feature. If approved, add `agents.list[].subagents.workspace` as a backwards-compatible per-agent subagent workspace override, normalize it like other workspace paths, document it in public config/subagent docs, update TypeScript/zod/generated schemas/help, and change native spawn workspace resolution for omitted `agentId` and explicit same-agent spawns while preserving cross-agent target-workspace behavior.
