## Summary

Keep this issue open. Current main still lacks `agents.list[].subagents.workspace`: the per-agent subagent config type and strict schemas omit it, docs only describe model/thinking/allowlist/requireAgentId defaults, and default or same-agent native subagent spawns still carry the caller workspace. The related cross-agent workspace inheritance work is present, but it does not provide a configurable default-subagent workspace override.

## Best Possible Solution

Keep this open for maintainer product/security review. If approved, add a backwards-compatible `agents.list[].subagents.workspace` config key, normalize it like other agent workspace paths, update TypeScript/zod/generated schemas and public docs, and adjust omitted-`agentId` plus explicit same-agent native spawn workspace resolution to prefer that override while preserving explicit cross-agent target-workspace behavior.
