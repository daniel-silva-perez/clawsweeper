## Summary

Keep this issue open. Current main still does not implement `agents.list[].subagents.workspace`: the per-agent subagent type, strict runtime schema, generated schema/help, and public docs omit the field, and native default/same-agent subagent spawns still preserve the caller workspace. The related cross-agent workspace fix is present, but it covers explicit `agentId` target workspaces, not the requested default-subagent workspace override.

## Best Possible Solution

Keep this open for maintainer decision on a core config/runtime feature. If approved, add `agents.list[].subagents.workspace` as a backwards-compatible per-agent subagent workspace override, normalize it like other workspace paths, document it in the public subagent and config docs, update TypeScript/zod/generated schemas/help, and change native spawn workspace resolution for omitted `agentId` and explicit same-agent spawns while preserving cross-agent target-workspace behavior.
