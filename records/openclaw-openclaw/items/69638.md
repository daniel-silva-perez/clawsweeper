## Summary

Keep this PR open. Current main still rejects password-mode clients that provide only `connectAuth.token`, while this PR is a narrow implementation candidate for the linked Rabbit/Funnel compatibility bug and includes regression coverage for the reviewed edge cases.

## Best Possible Solution

Keep this PR open, rebase it onto current main, preserve the scoped `connectAuth.password ?? connectAuth.token` fallback only inside the password-mode gateway branch, keep the wrong-token and password-precedence regression coverage, and run the changed gateway gate before landing. Longer-term Rabbit onboarding should use device-specific bootstrap tokens as a separate follow-up.
