## Summary

Keep open. Current main still creates a fresh native subagent session key for each `sessions_spawn` call, exposes only `mode: "run" | "session"` without a `reuse`/TTL target, and requires channel `subagent_spawning` hooks for persistent thread-bound native sessions. Discord and Feishu register those hooks; Microsoft Teams declares thread-capable chat but has no subagent hook wiring. The related #67000 is broader active-memory warm-start/session-reuse work and is still open, but it does not fully supersede this Teams/expert-agent session reuse request.

## Best Possible Solution

Keep this issue open and decide the product direction explicitly. A conservative implementation would either add Teams-owned `subagent_spawning`/delivery/end hooks with tests and docs, or add a generic core `sessions_spawn` reuse target with TTL semantics, registry conflict handling, cleanup behavior, and cross-channel tests. If maintainers want to consolidate broader warm-session work, link this issue from #67000 but keep the Teams/expert-agent acceptance criteria visible.
