## Summary

Keep open. Current main still lacks the requested operator-facing Dreaming session/cron exclusion config, and #72913 is open, unmerged, and explicitly paired as the implementation candidate, so closing or queueing a competing repair would be premature.

## What This Changes

Request robust Dreaming transcript filtering so isolated cron and rotated cron transcripts stay out of the session corpus, plus configurable operator exclusions for session keys, cron jobs, agents, and source paths.

## Best Possible Solution

Review and land #72913, or replace it with a narrower equivalent that preserves the same end state: reliable cron transcript exclusion for live and rotated artifacts, documented operator-facing Dreaming filters, and regression coverage for strict cron-run keys, base cron keys, and archive handling.

Yes for the remaining actionable gap: current main source, manifest, config resolver, and docs show no Dreaming session/cron exclusion config. The exact production corpus leak was not live-replayed here, but the current ingestion classifier still depends on path sets and strict cron-run session keys.

## Solution Assessment

Yes: keeping this issue open while #72913 is reviewed is the best current handling. The maintainable fix belongs in memory-core config/manifest/docs plus the memory-host transcript classification helpers, not in a competing ClawSweeper branch while an explicit closing PR is open.
