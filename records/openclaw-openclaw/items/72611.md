## Summary

Keep open. Current main still lacks the requested operator-facing Dreaming session/cron exclusion config, and the explicit implementation candidate #72913 is open and unmerged, so this issue should stay paired with that PR rather than be closed or sent to a competing repair lane.

## What This Changes

The issue requests reliable Dreaming exclusion for isolated cron transcripts, including rotated/deleted transcript artifacts, plus configurable operator filters for sessions, cron jobs, agents, prefixes, or source paths.

## Best Possible Solution

Review and land #72913, or replace it with an equivalent narrow fix that adds documented Dreaming session filters, keeps cron transcript exclusion reliable for live and rotated artifacts, preserves plugin boundaries, and includes regression coverage for strict cron-run keys, base cron keys, and archive handling.

## Reproduction Assessment

Yes for the remaining actionable gap: current main source, manifest, config resolver, tests, and docs show no Dreaming session/cron exclusion config. The exact production corpus leak was not live-replayed, and current main appears to skip some archive artifacts globally, but the classifier still lacks an operator predicate and only recognizes strict cron-run session keys.

## Solution Assessment

Yes: keeping this issue open while #72913 is reviewed is the best current handling. The maintainable fix belongs in memory-core config/manifest/docs plus memory-host transcript classification helpers, not in a competing ClawSweeper repair branch while a closing PR is open.
