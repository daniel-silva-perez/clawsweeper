## Summary

Keep this PR open. Current main still does not implement the requested `sessionSaveRedirectPath` session-memory redirect contract, and the PR is an active same-author follow-up paired with open #38161 rather than obsolete cleanup. Main still writes session memory only to the default workspace `memory/` path, exposes no `postHookActions` event field, and documents default-only session-memory saves.

## Best Possible Solution

Keep this PR open. Review or decide #38161 first, then rebase and review this redirect-specific follow-up against current main with focused security coverage for `writeFileWithinRoot` containment, fail-closed behavior, symlink-aware rollback/restoration, bounded snapshot/read behavior, Windows path handling, SDK compatibility, tests, and public hooks docs. If maintainers decide not to add this core policy surface, close it only after an explicit product/security decision and point contributors to #36671 for the broader security-plugin hook discussion.
