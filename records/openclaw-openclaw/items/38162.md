## Summary

Keep this PR open. Current main still does not implement `sessionSaveRedirectPath` or the dependent `postHookActions` / session-save policy contract, and this PR remains an active same-author follow-up to open #38161 rather than obsolete cleanup. The discussion is substantive: reviewers flagged security-relevant filesystem and hook-drain issues, and the author has been actively addressing them on the PR branch, so the right next step is maintainer/security review, not closure.

## Best Possible Solution

Keep this PR open. Review or resolve #38161 first, then rebase and review this redirect-specific follow-up against current main with focus on filesystem containment, symlink rollback, Windows path handling, bounded post-hook drains, SDK compatibility, privacy logging, and the intended session-memory policy contract. If maintainers decide not to add this core policy surface, close it only after an explicit product/security decision and point contributors to the broader security-plugin hook discussion in #36671.
