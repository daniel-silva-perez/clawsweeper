## Summary

Keep this PR open. Current main still does not implement the requested `sessionSaveRedirectPath` session-memory redirect behavior, nor the supporting `postHookActions`, `blockSessionSave`, or `sessionSaveContent` hook contract. This PR is also active, security-sensitive, and paired with same-author open #38161, so it should be reviewed or explicitly rejected by maintainers rather than closed as cleanup.

## Best Possible Solution

Keep this PR open. Review or resolve #38161 first, then rebase and review this redirect-specific follow-up against current main, with focused attention on filesystem containment, symlink rollback, Windows path handling, bounded post-hook drains, SDK compatibility, privacy logging, and the intended session-memory policy contract. If maintainers decide not to add this core policy surface, close it only after an explicit product/security decision and point contributors to the broader security-plugin hook discussion in #36671.
