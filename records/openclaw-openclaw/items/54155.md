## Best Possible Solution

Keep this as the broad Gateway memory tracker until maintainers split or complete the remaining auth-profile clone and transcript-history follow-ups, then close only after targeted regression tests plus a representative long-running Gateway soak/profile show stable RSS under the reported workload shape.
