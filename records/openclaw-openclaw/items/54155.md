## Summary

Keep this issue open. Current main has fixed the session-store structuredClone contributor from #73737 and contains useful seqByRun, stale-run-context, cron-session, and task cleanup mitigations, but the broader multi-day Gateway RSS/session-accumulation problem is not proven fixed: auth-profile structuredClone paths and full-transcript materialization still remain, and maintainer discussion explicitly kept the issue open for soak/profiling proof.

## What This Changes

The issue reports unbounded Gateway RSS growth over multi-day uptime and asks for completed subagent, cron, session-store, transcript, and compaction/pruning state to plateau or be evicted instead of accumulating in process memory.

## Best Possible Solution

Keep this issue open as the broad Gateway memory tracker. The best maintainer path is to finish or explicitly split the remaining #45438 auth-profile structuredClone work, replace or bound full-transcript materialization on hot Gateway history/usage/message-sequence paths, preserve the shipped seqByRun and session/cron cleanup mitigations, and close only after targeted regression tests plus a long-running soak/profile run show RSS plateaus under the reported workload.
