## Summary

Keep this issue open. Current main has partial mitigations for session maintenance, stale run contexts, and the seqByRun contributor, but the broader Gateway RSS/session-accumulation problem is not proven fixed: the structuredClone session-store cache path and whole-file session/transcript materialization remain on current main.

## Best Possible Solution

Keep this issue open as the broad Gateway RSS/session-accumulation tracker. The maintainer path is to fix or explicitly supersede #45438, bound or replace the structuredClone session-store cache path, avoid full transcript materialization before limits on hot history/usage/message-sequence surfaces, preserve the shipped seqByRun and maintenance mitigations, and add long-running profiling or soak evidence showing Gateway RSS plateaus under cron, channel traffic, local embeddings, and subagent sessions.
