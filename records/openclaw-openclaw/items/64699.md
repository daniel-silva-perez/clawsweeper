## Summary

Keep open. Current main still rejects `sessions_send` calls that include both `sessionKey` and `label`, which matches the reported failure mode, and the latest release source still contains the same guard. PR #74009 and fork PR BingqingLyu/openclaw#2317 are active implementation paths, so this issue should remain open until a canonical fix lands.

## What This Changes

The issue reports that `sessions_send` can receive an unintended `label` alongside an intended `sessionKey`, causing the tool's mutual-exclusion validation to reject an otherwise valid send.

## Best Possible Solution

Land one canonical fix in `src/agents/tools/sessions-send-tool.ts` that treats an explicit `sessionKey` or session-id-shaped key as authoritative when redundant `label` or `agentId` hints are present, preserves label-only resolution and A2A visibility checks, updates the tool description, and adds focused regression tests in `src/agents/tools/sessions.test.ts`.
