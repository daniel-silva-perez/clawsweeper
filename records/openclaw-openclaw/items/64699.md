Keep open. Current main and the latest release still reject mixed `sessionKey` plus `label` inputs before target resolution, which matches the reported failure. PR #74009 is the active canonical fix path, but it is open and has review/security concerns to resolve before this issue can close.

## What This Changes

The issue reports that `sessions_send` calls intended to target by `sessionKey` can receive a redundant `label` and then fail the mutual-exclusion guard.

## Best Possible Solution

Land one canonical fix, likely through PR #74009 or an equivalent replacement, that makes explicit `sessionKey` or session-id-shaped input authoritative over redundant `label`/`agentId` hints, preserves label-only resolution and A2A visibility checks, avoids denied-access key disclosure, updates the tool-facing description, and adds focused regression tests.
