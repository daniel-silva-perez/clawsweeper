## Summary

Keep this issue open. Current main has Discord native-command channel policy checks and routes commands through the effective channel or binding target, but `/new` and `/reset` still have no explicit agent/mention target option or parser. `/new @AgentName` would be treated as generic reset body/model input, not a target-agent override.

## Best Possible Solution

Keep open and implement this in the bundled Discord native-command reset path: add a supported optional target-agent option or mention/name parser for `/new` and `/reset`, resolve it through configured agents, Discord routing, bindings/thread bindings, authorization, and channel policy, set `CommandTargetSessionKey` only for the selected session, and add regression tests for shared guild channels, bound sessions, blocked/disabled channels, unauthorized targets, and the no-target default behavior.
