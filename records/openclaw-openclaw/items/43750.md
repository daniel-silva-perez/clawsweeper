## Summary

Keep this issue open. Current main routes Discord and Telegram native commands through the effective channel/topic/binding target, but `/new` and `/reset` still do not have an explicit `@agent` or agent option parser. The newer Telegram production comment reinforces that this is not only a Discord UX problem, and the broader multi-agent chat UI item does not supersede this narrow command-targeting request.

## Best Possible Solution

Implement this as a focused native-command targeting feature, not as a broad UI issue: add an optional agent target for `/new` and `/reset` in Discord and Telegram command paths, resolve it through configured agents, identities/mention patterns, bindings, and channel authorization, set `CommandTargetSessionKey` only for the selected session, and add regression tests for shared channels, bound ACP/thread sessions, blocked channels, unauthorized targets, ambiguous targets, and the chosen no-target default.

## Risks / Open Questions

- No live shared-channel Discord or Telegram reproduction was run under the read-only review constraints, but the current source, docs, and tests are enough to show explicit agent targeting is still absent.
- The no-target behavior needs product direction: highest-context agent, active participants, prompt/select UI, or fail-closed are materially different policies.
- A fix must preserve channel/guild/topic authorization and binding checks so targeting another agent cannot bypass allowlists or reset sessions the sender should not control.
