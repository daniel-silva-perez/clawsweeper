## Summary

Keep open. Current main has backup archives and plugin-owned imports from other agent systems, but it does not yet provide the requested portable OpenClaw agent package/export/import/pull/run model. The linked migration PR is open and unmerged, and its discussion still has unresolved maintainer security concerns around restore/import scope.

## Best Possible Solution

Keep this issue open as the broader product/design request. The best path is to align it with the #53520/#45993 discussion: decide whether core should add safer selective backup/export/convert or a gated import flow, while using ClawHub or bundle-style plugins for distributable skills, MCP, and curated knowledge where the current plugin surface is sufficient.

## Risks / Open Questions

- Closing now would preempt unresolved maintainer judgment about whether restore/import belongs in core, backup convert, or a different user-driven flow.
- Existing plugin bundles and ClawHub can cover skills/MCP-style distribution, but they do not yet cover the requested agent identity/rules/knowledge package with `pull`/`run` semantics.
- A future solution may need both a ClawHub/bundle path for shareable stable capabilities and a separate core backup/migration path for trusted machine-to-machine moves.
