## Summary

Keep this issue open. Current main already ships Gemini Embedding 2 support, configurable output dimensions, and opt-in image/audio multimodal indexing for files under memorySearch.extraPaths, but the requested conversation-history attachment indexing workflow is still missing: session-memory export/indexing flattens transcripts to text and does not turn chat or Telegram MediaPath/MediaPaths attachments into structured embedding inputs.

## Best Possible Solution

Keep this open and narrow the remaining work to session-memory attachment indexing. Extend the session indexing path to discover eligible MediaPath/MediaPaths plus MIME metadata from chat transcripts, resolve them through existing managed-media and sandbox boundaries, create structured EmbeddingInput chunks for supported attachments, add regression coverage for chat/Telegram attachment recall, and document the distinction between file-backed extraPaths multimodal memory and conversation-history attachment indexing.
