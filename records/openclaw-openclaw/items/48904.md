## Summary

Keep this PR open. Current main has moved thinking validation toward provider-owned `resolveThinkingProfile`/`thinkingLevels`, but it still does not implement the PR’s central behavior: a custom `models.providers.*.models[*].compat.supportsXHighThinking` config opt-in, or an equivalent config/catalog path that lets custom OpenAI-compatible model refs expose `xhigh`.

## Best Possible Solution

Keep this PR open for maintainer review or contributor refresh. The best path is to adapt the custom-model xhigh opt-in to current main’s `resolveThinkingProfile` and gateway `thinkingLevels` architecture, decide whether the public API should remain `compat.supportsXHighThinking` or become a broader configured thinking-profile field, then update the strict config schema, generated config schema/help, docs, and focused tests for every affected validation and presentation surface.
