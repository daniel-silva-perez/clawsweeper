## Summary

Keep this PR open. Current main still lacks the requested generic outbound metadata route across gateway send, sendMessage, channel outbound context, delivery queue storage, and recovery replay, so the PR is not obsolete. The PR branch also is not ready to land unchanged: the live diff still does not pass metadata into createChannelHandler, hardcodes Hermes topic/bot policy in core task/agent paths, includes or overlaps an unrelated auth-profile credential-surface change, and has test hygiene concerns from review comments.

## Best Possible Solution

Keep this PR open and ask for a focused rebase or split. The best path is a generic, documented opaque outbound metadata seam that flows through sendMessage, gateway send validation, channel outbound context, queue persistence/recovery, generated client models, and targeted tests. Hermes-specific metadata construction and topic/bot selection should be configurable/plugin-owned or explicitly accepted by a maintainer, and unrelated credential-surface changes should stay in a separate PR or be dropped if already on main.
