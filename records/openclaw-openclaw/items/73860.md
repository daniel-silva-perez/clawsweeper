## Summary

Keep open. The reporter’s follow-up confirms the target lookup can work with `open_id`, but Feishu `im.message.create` still returns vendor `code: 9499` for DMs, and current `main` still drops the useful Feishu error body on the message and `feishu_chat` paths. Current source maps `user:ou_...` to `receive_id_type: open_id`, so this is no longer a cleanup-close candidate; it needs Feishu-plugin diagnostics and then a scoped root-cause fix or documentation of Feishu bot/tenant/permission constraints.

## Best Possible Solution

Keep this issue open. First, add Feishu-plugin error-body preservation for message sends and `feishu_chat` lookups so HTTP 400 exposes Feishu `code`/`msg`/`log_id`/troubleshooter details. Then use the reporter’s Feishu log IDs to determine whether `im.message.create` `9499` needs a real OpenClaw send/content fix, a setup/scope/tenant-policy doc update, or an upstream/vendor workaround.
