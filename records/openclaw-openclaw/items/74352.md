## Summary

Keep open. Current main still uses a fixed 20s client timeout for browser file-chooser arming, so the PR is not obsolete; the proposed fix is directionally useful but still has a narrow CLI timeout consistency issue and is missing the required changelog entry for a user-facing fix.

The PR changes browser file-chooser arming to use the browser action timeout budget, raises the CLI upload wrapper default timeout from 20s to 60s, and adds client tests for default and explicit file-chooser request timeouts.

## Best Possible Solution

## Solution Assessment

No, not quite as proposed. The core client change is a maintainable direction, but the CLI wrapper should use the same explicit-timeout slack model or a shared resolver, and the user-facing fix needs the required changelog entry before merge.
