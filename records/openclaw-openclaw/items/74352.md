## Summary

Keep open. Current main still has the reported 20s browser file-chooser client transport timeout and does not have timeout assertions for file-chooser arming, while the neighboring browser action path already uses the longer action-style timeout model. The PR is a narrow, relevant fix candidate, but it should stay open for maintainer review because the review discussion flags one remaining CLI consistency edge case for explicit `--timeout-ms`.

## What This Changes

The PR changes `browserArmFileChooser` to use the browser action timeout/default-plus-slack model, raises the CLI upload wrapper default from 20s to 60s, and adds client tests for default and explicit file-chooser request timeouts.

## Best Possible Solution

Land this narrow timeout alignment after addressing or explicitly accepting the CLI explicit-timeout slack edge case, with focused browser client coverage; keep the broader stale upload trigger and error-mapping work tracked separately in #38844.
