## Summary

Keep open for maintainer review. Current main still has the fixed 20s file-chooser client timeout, and the reported failure mode is plausible because the hook route resolves tab/upload state and arms Playwright before responding. The patch is narrow and does not add security-sensitive surfaces, but the CLI explicit timeout path still appears to preserve the same client/server deadline race.

## What This Changes

The PR changes browser file-chooser arming to use the browser action timeout budget, raises the CLI upload default request timeout to 60s, and adds browser client timeout assertions.

## Best Possible Solution

Land the narrow timeout alignment after padding or explicitly accepting the CLI explicit-timeout edge case, while keeping upload path guardrails and Playwright semantics unchanged and leaving the broader upload-flow reliability work in #38844.
