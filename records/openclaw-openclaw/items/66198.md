## Summary

Keep this PR open. Current main still sends Signal inbound user text through raw model-facing fields, so the security hardening goal is relevant, but the submitted patch is not merge-ready because it wraps only legacy envelope bodies and pending-history rendering rather than the active prompt path.

## Best Possible Solution

Keep this PR open, but require a rework or replacement before merge. The fix should protect Signal's actual model-facing path by using wrapped untrusted user text in BodyForAgent or structured UntrustedContext, while preserving raw CommandBody/BodyForCommands for command parsing. It should wrap pending history once or use stable marker IDs, keep generated message IDs outside untrusted blocks, and add focused Signal coverage for direct messages, group messages, debounced bodies, quote-only context, and pending group history.

## Risks / Open Questions

- Merging as-is would create a false sense of Signal prompt-injection protection because raw BodyForAgent remains the normal model prompt source.
- The submitted history wrapping would call wrapExternalContent during history formatting; current wrapExternalContent creates a fresh random marker ID per call, so unchanged history could destabilize prompt bytes and prompt caching.
- The submitted history change places generated message-id metadata inside the untrusted block instead of keeping trusted application metadata outside it.
