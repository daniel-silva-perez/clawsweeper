## Summary

Keep this issue open. Current main partially fixes blank user replay, and direct Anthropic transport has last-mile guards, but the shared replay sanitizer still has the reported assistant/toolResult/images-only/preserveSignatures gaps, including a regression test that explicitly preserves an empty text block when signatures are preserved.

## Best Possible Solution

Keep the issue open and implement a shared empty-text block helper in the transcript replay sanitizer. Apply it uniformly to toolResult content and all assistant branches, keep the existing user replay repair aligned, preserve signed thinking block order/identity, and add focused regression tests for toolResult, assistant error, images-only, preserveSignatures-with-thinking, and the already-partial user path. If filtering empties a provider-visible non-assistant content array, use a provider-safe non-empty placeholder or drop the replay message according to the existing turn semantics.
