## Summary

Keep this issue open. Current main still lacks the requested write-time Markdown code-fence lint/fix utility, configuration, per-write skip option, and integration across workspace Markdown write paths. The two related implementation PRs, #37628 and #41207, were closed unmerged, so they remain useful prior art rather than a shipped fix.

## Best Possible Solution

Keep this issue open until maintainers either implement a central Markdown code-fence lint/fix utility with sh/ts/python/json inference, change records, warn/auto-fix configuration, per-write skip support, and focused coverage across current workspace Markdown write paths, or explicitly decide that OpenClaw should not mutate Markdown at write time.
