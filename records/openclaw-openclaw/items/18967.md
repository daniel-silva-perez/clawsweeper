## Summary

Current main still does not implement the requested native parent-scoped `sessions_send` path for sub-agents. It supports completion announce-chain delivery, ACP-only `streamTo: "parent"`, and a broad config opt-in that can re-enable `sessions_send`, but none of those provide a trusted direct-parent-only send API with same-tree and no-chaining guarantees.

## Best Possible Solution

Keep this issue open for maintainer design. The best implementation is a dedicated `sessions_send_parent` tool or a `sessionKey: "parent"` resolver that derives the direct parent from trusted spawn/session metadata, rejects missing-parent, sibling, ancestor, and cross-agent targets, avoids automatic forwarding beyond that parent, and updates policy, resolver, docs, and regression tests.
