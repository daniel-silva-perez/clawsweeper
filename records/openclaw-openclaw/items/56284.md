Keep open. Current main has partial restart-intent and drain wiring, but the Windows Scheduled Task restart path still goes through service-manager `/End` plus listener termination, the CLI/daemon path does not write the restart sentinel requested by the report, and the linked implementation PR is closed unmerged. This is not safe to close as implemented without a live Windows Scheduled Task verification or a narrower merged fix.

## What This Changes

The issue requests that Windows Scheduled Task `openclaw gateway restart` wait for active tasks, preserve or recover session state, and return only after the restarted gateway is healthy.

## Best Possible Solution

Keep this issue open and make the Windows Scheduled Task restart contract explicit: CLI restart should coordinate with the running gateway to drain active tasks/runs, avoid duplicate Scheduled Task `/Run` races, preserve recoverable session state or continuation metadata, and wait for post-restart health. The implementation should add focused unit coverage around `runServiceRestart`, `restartScheduledTask`, restart intent consumption, and Windows handoff behavior, plus live Windows proof before closure.
