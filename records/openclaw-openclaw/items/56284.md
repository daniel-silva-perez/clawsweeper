## Summary

Keep open. Current main has partial restart-intent, drain, health-check, and restart-aborted-session recovery wiring, but the Windows Scheduled Task restart path still goes through service-manager termination and listener cleanup, and the CLI service restart path still does not write the durable restart sentinel requested for session continuation. The linked implementation PR is closed unmerged, while adjacent session-loss and Windows restart-race PRs remain open.

## What This Changes

The issue asks for Windows Scheduled Task `openclaw gateway restart` to drain active work, preserve restart/session recovery state, and return only after the restarted Gateway is healthy.

## Best Possible Solution

Define and implement one Windows Scheduled Task restart contract that coordinates with the running Gateway: write the appropriate restart/recovery state, signal the old process in a way that lets it drain active tasks and embedded runs, avoid duplicate `/Run` races, wait for post-restart health, and verify interrupted sessions are completed, resumed, or explicitly marked failed without silent loss.

## Reproduction Assessment

Unclear. The issue provides concrete Windows 11 Scheduled Task steps and current source shows the relevant termination path, but a high-confidence reproduction still requires a live Windows run with active work to prove whether the restart intent is consumed and the drain/recovery path completes before listener cleanup.

## Solution Assessment

Unclear. The closed direct PR is useful context, but the safest fix is likely to integrate the Scheduled Task CLI path with the existing gateway-managed restart intent, drain, sentinel, and health mechanisms rather than adding a parallel ad hoc Windows flow.
