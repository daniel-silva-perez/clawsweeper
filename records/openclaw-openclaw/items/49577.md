## Summary

Keep this issue open. Current main has liveness, timestamp, diagnostic-event, and some stale listener/process checks, but status still does not expose a generic inbound-receipt to agent-dispatch health signal; #72724 is an open unmerged PR with closing wording for this issue, so repository policy says to wait for that or an equivalent merge.

## What This Changes

The issue requests clearer degraded gateway/channel status when liveness checks look healthy but inbound messages are not reaching the agent, including stale duplicate gateway processes after restart.

## Best Possible Solution

Review and land #72724 or an equivalent generic core implementation that surfaces inbound receipt, dispatch, session-turn creation, processing outcomes, no-recent-agent-activity, and multi-listener warnings without hard-coding channel plugin internals.

## Reproduction Assessment

No. The issue has concrete operational observations, but this read-only review did not have a live Feishu gateway setup; the missing generic status telemetry is reproducible by inspecting the current status and channel issue aggregation paths.

## Solution Assessment

Yes. A generic gateway/status telemetry layer is the right maintainable direction; the safer alternative to per-channel fixes is to keep the core signal plugin-agnostic and let channel plugins add only channel-specific diagnostics.
