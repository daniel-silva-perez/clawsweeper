## Summary

Current main still does not explicitly distribute onboarding auth profiles across configured agents. Provider onboarding and Anthropic setup-token paths persist to one resolved agentDir, while runtime main-agent fallback only helps missing or OAuth-refresh cases and still lets agent-local profiles override main credentials on profile-id conflicts.

## Best Possible Solution

Keep this issue open and make an explicit maintainer decision on multi-agent auth semantics: either sync selected onboarding credentials to configured agent dirs with clear SecretRef and overwrite rules, or preserve per-agent isolation and add a first-class auth sync/repair/warning path for stale profiles that shadow main credentials. The fix should cover Anthropic setup-token/tokenRef, API-key SecretRefs, missing stores, stale conflicting non-main profiles, and docs alignment.
