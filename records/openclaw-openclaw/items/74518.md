## Summary

Keep this PR open. Current main still derives direct list paragraph depth from list-stack length, which misses loose list paragraphs inside enclosing containers such as blockquotes, and the PR is a narrow relevant fix with matching regression coverage and no blocking code-review finding.

## What This Changes

This PR records each Markdown list opening token level, uses that level when deciding whether to preserve loose-list paragraph breaks, and adds a blockquoted loose-list regression test.

## Best Possible Solution

Land the narrow Markdown IR fix after maintainer review and green changed-gate/CI proof, keeping loose-list paragraph detection tied to the actual list-opening token level with regression coverage for blockquoted lists.
