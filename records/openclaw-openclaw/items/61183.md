## Summary

Keep this PR open. Current main still does not implement the PR's intended bookkeeping fix: embedded message telemetry is committed from the stale tool_execution_start args before adjusted before_tool_call params are consumed, so hook-rewritten message text/target/media can still be mis-recorded. The PR is a plausible implementation candidate, but it needs rebase/review cleanup rather than closure.

## Best Possible Solution

Keep the PR open as an implementation candidate. The best path is to rebase it onto current main, apply the adjusted-params bookkeeping in handleToolExecutionEnd before message/cron commits, add the direct message and embedded bookkeeping regression tests, and either split or separately justify the session-transcript-repair changes.
