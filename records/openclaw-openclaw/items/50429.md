## Summary

Keep this PR open. Current main still has no plain-text fallback for non-streaming Feishu reply cards rejected by the table-count limit, and related #73651 only covers outbound sends. The branch is relevant, but its fallback should not merge as-is because it catches every card error, can retry content that was already partially delivered, and can turn suppressed block-preview payloads into normal Feishu messages.

## What This Changes

The PR wraps non-streaming Feishu reply card delivery in a try/catch, logs card-send failures, retries through the existing plain-text chunk sender, and adds two reply-dispatcher tests for card failure fallback and successful card delivery.

## Best Possible Solution

Revise the reply-dispatcher fallback so it only handles Feishu table-limit card failures, preserves the existing block payload suppression, and retries plain text only before any card chunk has been delivered or from the failed chunk onward. Align the classifier with the outbound table-limit work and add focused regression tests for the edge cases.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Avoid resending already delivered card chunks:** `extensions/feishu/src/reply-dispatcher.ts:438-460`
  - body: If `sendChunkedTextReply()` fails on chunk 2 or later, earlier card chunks have already been sent. Retrying the original full `text` as plain text duplicates the beginning of the answer and can repeat first-chunk mentions. Track whether any card chunk was delivered and only fall back before the first send, or retry from the failed chunk.
  - confidence: 0.88
- **[P2] Keep block payloads out of plain-text fallback:** `extensions/feishu/src/reply-dispatcher.ts:438-460`
  - body: `info.kind === "block"` payloads are treated as streaming-card preview content earlier in this function and are intentionally dropped when streaming cannot consume them. This catch path now sends those blocks as normal Feishu text, so a streaming startup failure can expose a partial preview before the final reply. Limit fallback to final user-visible replies or drop block payloads here.
  - confidence: 0.84
- **[P2] Rethrow non-table-limit card failures:** `extensions/feishu/src/reply-dispatcher.ts:438-460`
  - body: The fallback catches every `sendStructuredCardFeishu` error. That turns auth, permission, malformed-card, or rate-limit failures into plain-text sends and hides operational problems unrelated to Feishu's table-count limit. Classify 230099 / 11310 table-limit errors and rethrow everything else.
  - confidence: 0.8
