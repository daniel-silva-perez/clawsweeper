## Summary

Keep this PR open. The Feishu table-limit reply-drop bug is valid and current main still has no reply-dispatcher fallback, but this branch is not merge-ready: the fallback is catch-all, retries the whole reply after any chunk failure, and can turn block/preview payload failures into plain Feishu text. The related outbound work in #73651 is explicitly scoped to outbound sends and does not supersede this reply-dispatcher change.

## What This Changes

The PR wraps non-streaming Feishu reply card delivery in a try/catch, logs card-send failures, retries through the existing plain-text chunk sender, and adds two reply-dispatcher tests for card failure fallback and successful card delivery.

## Best Possible Solution

Revise the reply-dispatcher fallback to use a table-limit-specific classifier, preserve block/preview suppression, and only retry plain text when doing so cannot duplicate already delivered card chunks. Add regression tests for table-limit success, non-table rethrow, block payload handling, and multi-chunk partial-delivery behavior, coordinating with the outbound classifier from #73651 if that work lands first.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Avoid retrying the whole reply after a later card chunk fails:** `extensions/feishu/src/reply-dispatcher.ts:438-460`
  - body: If `sendChunkedTextReply()` fails on chunk 2 or later, earlier card chunks have already been sent. Retrying the original full `text` as plain text duplicates the beginning of the answer and can repeat first-chunk mentions; the fallback should only run before any card chunk is delivered or resume from the failed chunk.
  - confidence: 0.88
- **[P2] Limit the fallback to final reply payloads:** `extensions/feishu/src/reply-dispatcher.ts:438-460`
  - body: When streaming startup has failed, `info.kind === "block"` can still reach this catch path. Sending those failed card chunks through `sendMessageFeishu` can expose block/preview content as normal chat text before the final answer, so the fallback should stay limited to final user-visible replies or explicitly drop block payloads here.
  - confidence: 0.84
- **[P2] Classify table-limit errors before falling back:** `extensions/feishu/src/reply-dispatcher.ts:438-460`
  - body: The catch handles every `sendStructuredCardFeishu` failure. That hides auth, permission, malformed-card, and rate-limit problems by retrying them as plain text; only the known 230099 / 11310 table-limit rejection should degrade to text, and unrelated failures should still propagate.
  - confidence: 0.8
