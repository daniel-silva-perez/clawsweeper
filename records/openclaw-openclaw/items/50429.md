## Summary

Keep this PR open. The reported Feishu reply-dispatcher bug is still real on current main, and the branch is useful, but the proposed catch path is too broad: it retries whole multi-chunk replies after partial card delivery, can expose block payloads as plain text, and falls back for every card error instead of only the known table-limit rejection.

## What This Changes

The branch wraps non-streaming Feishu card reply delivery in a catch block, retries failed card sends through the existing plain-text chunk sender, and adds two reply-dispatcher tests.

## Best Possible Solution

Revise the reply-dispatcher fallback so it only degrades known Feishu table-limit errors, only for final user-visible replies, and only before any card chunk has been delivered or from the failed chunk onward. If #73651 lands first, reuse or align with its table-limit classifier, then add regression tests for non-table errors, block payloads, and partial multi-chunk delivery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Avoid retrying after delivered card chunks:** `extensions/feishu/src/reply-dispatcher.ts:416-443`
  - body: The catch wraps the whole card-mode sendChunkedTextReply call. If chunk 2 or later fails, earlier card chunks were already sent, and the fallback resends the entire reply as text, including duplicate first-chunk mentions. Track delivered chunks and only fall back before the first send, or resume from the failed chunk.
  - confidence: 0.88
- **[P2] Limit fallback to final reply payloads:** `extensions/feishu/src/reply-dispatcher.ts:443-448`
  - body: When streaming startup has failed, info.kind === "block" can still reach this catch path. Sending those failed card chunks through sendMessageFeishu bypasses the existing block suppression and can expose partial or preview text before the final answer. Keep this fallback limited to final payloads or explicitly drop block payloads here.
  - confidence: 0.84
- **[P2] Classify table-limit errors before fallback:** `extensions/feishu/src/reply-dispatcher.ts:436-443`
  - body: The catch handles every sendStructuredCardFeishu failure. Auth, permission, malformed-card, and rate-limit failures should still propagate; only the known 230099 / 11310 table-limit rejection should degrade to plain text.
  - confidence: 0.8
