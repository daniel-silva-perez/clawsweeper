## Summary

Keep open. Current main has adjacent Feishu inbound broadcast support for platform-delivered group events, but it still does not implement the requested send-time cross-bot fan-out for bot-authored Feishu group messages. The remaining work needs maintainer channel-architecture judgment because it touches synthetic bot-originated turns, loop limits, opt-in config, and overlap with active Feishu multi-agent delivery work.

## What This Changes

Add an opt-in Feishu cross-bot broadcast mechanism so successful bot-authored group sends are injected as synthetic message events for other registered OpenClaw Feishu bot accounts in the same group.

## Best Possible Solution

Keep this as a maintainer design item for Feishu send-time cross-bot delivery. If approved, implement the behavior in the Feishu plugin-owned outbound path after successful group sends, mark injected turns as synthetic bot-originated, enforce dedupe and bot-turn limits, expose documented opt-in config, and cover final replies plus message-tool sends in focused Feishu tests.

## Reproduction Assessment

Yes. The reporter gives a concrete multi-bot Feishu group setup, and current source provides a high-confidence code-level reproduction: outbound Feishu sends return after the API call and no current path injects a peer-bot inbound event after successful delivery.

## Solution Assessment

Unclear. Send-time fan-out is a plausible narrow solution for Feishu's platform limitation, but the best maintainable fix depends on maintainer decisions about shared channel contracts, synthetic-event metadata, loop safety, and whether any cross-channel abstraction is needed.
