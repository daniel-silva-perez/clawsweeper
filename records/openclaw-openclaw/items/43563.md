## Summary

Keep open. Current main still has only Feishu broadcast handling for platform-delivered inbound group events, not the requested send-time fan-out for bot-authored outbound Feishu messages. The adjacent merged inbound broadcast work and the related open lost-message report do not replace this product decision.

## What This Changes

The issue requests opt-in Feishu cross-bot broadcast so bot-authored group sends are injected as synthetic inbound events to peer bots, with loop guardrails and possible cross-channel generalization.

## Best Possible Solution

Keep this as the canonical maintainer design item. If maintainers approve bundled Feishu support, add an opt-in Feishu-owned post-send fan-out after successful group sends, mark injected events as synthetic bot-originated, enforce dedupe and consecutive-bot loop limits, update config schema/help/docs, and cover final replies plus message-tool sends in focused tests.
