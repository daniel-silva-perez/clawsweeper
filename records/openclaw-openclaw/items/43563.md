## Summary

Keep open. Current main has adjacent Feishu inbound broadcast support for platform-delivered group messages, but it still does not implement the requested send-time fan-out for bot-authored Feishu group sends. The remaining work needs maintainer channel-design judgment because it would introduce synthetic bot-originated turns, loop limits, dedupe rules, and opt-in config.

## What This Changes

Request send-time synthetic cross-bot broadcast for Feishu group messages authored by OpenClaw bots, with opt-in config and loop protection, to match Discord multi-bot collaboration behavior.

## Best Possible Solution

Keep this as a maintainer-owned Feishu channel design item. If approved, implement the behavior in the Feishu plugin-owned outbound path after successful group sends, mark injected turns as synthetic bot-originated, enforce dedupe and bot-turn limits, expose documented opt-in config, and cover final replies plus message-tool sends in focused Feishu tests.

## Reproduction Assessment

Yes. The reporter gives a concrete multi-bot Feishu group setup, and current source provides a high-confidence code-level reproduction: outbound Feishu sends return after the API call and no current path injects a peer-bot inbound event after successful delivery.

## Solution Assessment

Unclear. Send-time fan-out is a plausible narrow solution for Feishu's platform limitation, but the best maintainable fix depends on maintainer decisions about shared channel contracts, synthetic-event metadata, loop safety, and overlap with existing inbound broadcast behavior.
