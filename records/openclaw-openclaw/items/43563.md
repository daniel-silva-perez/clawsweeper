## Summary

Keep open. Current main still only fans out Feishu platform-delivered inbound group events through the existing broadcast path; outbound Feishu text, card, media, and final-reply sends go straight to Feishu without a post-send synthetic peer-bot event, and there is no Feishu cross-bot opt-in config or loop-limit surface.

## What This Changes

Request an opt-in Feishu cross-bot broadcast mechanism so messages sent by one Feishu bot in a shared group are synthesized as inbound events for other registered bots in that group.

## Best Possible Solution

Add a Feishu-owned, opt-in send-time fan-out after successful outbound group sends, producing clearly marked synthetic inbound events for peer accounts in the same group with origin metadata, dedupe, loop limits, config schema/help/docs, and focused tests; extract a shared channel seam only after another channel proves the same contract is needed.
