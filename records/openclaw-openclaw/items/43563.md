## Summary

Keep open. Current main still has Feishu inbound broadcast for platform-delivered group messages, but it does not implement the requested send-time synthetic fan-out for bot-authored Feishu group sends; the remaining work needs maintainer channel-design decisions around synthetic turns, loop limits, dedupe, and config.

## What This Changes

The issue requests opt-in Feishu cross-bot send-time broadcast so other OpenClaw Feishu bots in the same group can receive bot-authored group messages.

## Best Possible Solution

Keep this as a Feishu plugin-owned channel design item. If approved, implement the fan-out after successful Feishu group sends in the outbound/reply paths, mark injected turns as synthetic bot-originated, enforce dedupe and bot-turn limits, expose documented opt-in config, and add focused Feishu tests for final replies plus message-tool sends.

## Reproduction Assessment

Yes. The reporter gives a concrete multi-bot Feishu group setup, and current source provides a high-confidence code-level reproduction: outbound Feishu sends return after the API call and no current path injects a peer-bot inbound event after successful delivery.

## Solution Assessment

Unclear. Send-time fan-out is a plausible narrow solution for Feishu's platform limitation, but the best maintainable fix depends on maintainer decisions about synthetic-event metadata, loop safety, dedupe, and overlap with existing inbound broadcast behavior.
