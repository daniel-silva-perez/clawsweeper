## Summary

Keep open. Current main still does not implement the requested send-time Feishu cross-bot broadcast: the existing broadcast code fans out platform-delivered inbound events, while outbound sends still go straight to Feishu without synthesizing peer-bot inbound events. The merged Feishu broadcast work is useful adjacent context but does not solve bot-sent-message visibility.

## What This Changes

The issue requests opt-in Feishu cross-bot broadcast so a bot message sent to a group is delivered as a synthetic inbound event to other registered OpenClaw Feishu bots in the same group, with loop protection and Discord-style parity.

## Best Possible Solution

Add a Feishu-owned opt-in send-time fan-out after successful outbound group sends, producing clearly marked synthetic inbound events for peer accounts in the same group with origin metadata, dedupe, loop limits, config schema/help/docs, and focused tests; extract a generic seam only after another channel proves the same contract is needed.
