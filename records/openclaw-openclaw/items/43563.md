## Summary

Keep open. Current main has Feishu broadcast handling for platform-delivered inbound group events, but it still lacks the requested send-time fan-out for bot-authored outbound Feishu messages, as well as the proposed opt-in `crossBotBroadcast` config and consecutive-bot loop limit. The merged inbound broadcast work in #29575 and broader open lost-message report #46776 are adjacent, not replacements for this exact request.

## What This Changes

The issue requests opt-in send-time cross-bot broadcast for Feishu group messages so bot-authored outbound messages are delivered as synthetic inbound events to peer bots in the same group, with loop guardrails and possible cross-channel generalization.

## Best Possible Solution

Keep this as the canonical maintainer design item until maintainers choose the product boundary. If approved for bundled Feishu, implement an opt-in Feishu-owned send-time fan-out after successful group sends, mark injected events as synthetic bot-originated, enforce dedupe and consecutive-bot loop limits, update config schema/help/docs, and add focused tests for text, card, media, final replies, message-tool sends, and loop prevention.
