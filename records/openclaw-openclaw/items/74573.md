## Summary

Keep this issue open. Current main still has the internal `[tool calls omitted]` replay placeholder, and the outbound/user-facing sanitizer path does not strip that exact placeholder before channel delivery. Related #25592 tracks a broader messaging leak class, but this report is a narrower actionable bug with a clear fix boundary.

## What This Changes

Prevent the internal `[tool calls omitted]` replay placeholder from appearing in user-facing channel replies such as Telegram messages.

## Best Possible Solution

Add a narrow user-facing reply sanitizer or outbound projection guard that drops exact standalone `[tool calls omitted]` placeholder text before any channel adapter receives it, while preserving the internal transcript-repair behavior needed for provider replay. Cover the shared sanitizer/outbound path plus Telegram dispatch so other channels inherit the same protection.
