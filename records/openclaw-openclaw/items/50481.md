## Summary

Keep this issue open. Current main still has Slack assistant thread status support, but it does not implement `assistant.threads.setSuggestedPrompts`, a static `assistantPrompts` config surface, a dynamic message action, or Slack docs/tests for suggested prompts. The related items in the provided context are adjacent Slack/threading reports, not canonical duplicates for this request.

## Best Possible Solution

Keep the issue open and implement this in the bundled Slack plugin. A conservative first pass would add a typed `channels.slack.assistantPrompts` config with strict schema, docs, and tests, then call `client.assistant.threads.setSuggestedPrompts` after a successful assistant-thread reply when the delivered Slack thread timestamp is known. The stretch goal can add a Slack-owned message action after validating action naming, payload limits, account scoping, and test coverage.
