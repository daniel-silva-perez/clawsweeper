## Summary

Keep open. Current main still has Slack Assistant thread-status wiring but does not implement Slack suggested prompts in config, post-reply dispatch, message-action discovery, runtime action handling, or Slack docs. The request is a valid bundled Slack plugin feature, not stale cleanup or an external ClawHub-only idea.

## What This Changes

Add Slack support for `assistant.threads.setSuggestedPrompts`, with static configured prompt suggestions and a possible dynamic message action for context-aware suggestions.

## Best Possible Solution

Implement the feature inside the Slack plugin boundary: add validated prompt config, call Slack's suggested-prompts API for eligible assistant-thread replies, document the config and behavior, and add a bounded dynamic action only through the existing shared channel message-action contract.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The missing feature is high-confidence from current-main inspection: the Slack status path is present, while suggested-prompt config, helper, dispatch, dynamic action, docs, and tests are absent.

## Solution Assessment

Yes, the best maintainable direction is Slack-plugin-owned wiring with a small config schema and optional additive message action. Core should only gain the generic action-name/schema seam if the dynamic tool is implemented; Slack-specific defaults and API calls should stay in `extensions/slack`.
