## Summary

Keep open. Current main still wires Slack Assistant thread status, but it does not implement Slack suggested prompts through static config, reply finalization, dynamic message actions, Slack action runtime, config help, docs, or tests. The upstream Slack API and Node SDK surface exist, so this remains a concrete bundled Slack plugin feature rather than cleanup or external-only work.

## What This Changes

Request support in the Slack plugin for `assistant.threads.setSuggestedPrompts`, starting with static configured suggestions and optionally adding a dynamic message action for contextual suggestions.

## Best Possible Solution

Implement this inside the bundled Slack plugin boundary: add a validated global/account prompt config, call Slack's suggested-prompts API for eligible assistant-thread replies, document the behavior, and add a bounded dynamic action through the shared channel message-action contract only if maintainers want the agent-facing surface in the same change.
