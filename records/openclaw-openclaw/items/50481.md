## Summary

Keep open. Current main still has Slack Assistant thread-status wiring, but it does not implement static suggested-prompt config, post-reply suggested-prompt dispatch, a dynamic `set-suggested-prompts` message action, Slack runtime handling, or Slack docs/config help. Slack's public API and Node SDK docs confirm the requested method and payload shape exist, so this remains a valid bundled Slack plugin feature rather than cleanup or external-only work.

## What This Changes

Add Slack support for `assistant.threads.setSuggestedPrompts`, including configured default suggestions and an optional dynamic message action for context-aware prompt suggestions.

## Best Possible Solution

Implement this inside the Slack plugin boundary: add validated per-account/global prompt config, call Slack's suggested-prompts API for eligible assistant-thread replies, document the behavior, and add a narrow dynamic action only through the existing shared channel message-action contract if maintainers want the stretch goal.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The missing feature is high-confidence from current-main inspection: status support exists, while suggested-prompt config, helper, dispatch, dynamic action, docs, and tests are absent.

## Solution Assessment

Yes. The narrow maintainable path is Slack-plugin-owned wiring with a small config schema; core should only gain an additive generic action-name/schema seam if the dynamic tool surface is implemented.
