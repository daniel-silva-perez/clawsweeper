## Repair Work Prompt

Implement Slack Assistant suggested prompts for the Slack plugin. The observable gap is that Slack Assistant thread status is wired, but OpenClaw never calls `assistant.threads.setSuggestedPrompts`, has no static suggested-prompt config, and has no dynamic message action for agents to set contextual prompts.

Expected boundary:
- Keep behavior in the Slack plugin. Do not move Slack-specific policy into core except for a generic closed message-action name if a dynamic action is implemented.
- Add a backwards-compatible config shape for static prompts, such as `channels.slack.assistantPrompts` and account-level override support if consistent with Slack config patterns. Validate title/message fields and Slack's small prompt count limit.
- After eligible assistant-thread replies are delivered, call Slack `assistant.threads.setSuggestedPrompts` with `token`, `channel_id`, `thread_ts`, optional title, and prompts. Skip safely when there is no thread target or no configured prompts; log verbose failures without failing reply delivery.
- If adding the stretch dynamic action, expose a bounded `set-suggested-prompts` action through the existing channel message-action contract and Slack dispatcher/runtime. Validate params and avoid broad cross-channel API churn.
- Update Slack docs/config UI hints/generated config metadata as required, and add a user-facing changelog entry with `Thanks @leslielesleyles`.
- Do not change Slack auth scopes unless implementation proves the existing documented `assistant:write` scope is insufficient.
