## Summary

Keep this PR open. Current main still lacks the requested non-blocking high-frequency `--every <30m` cron advisory in CLI and gateway add/update paths, the paired issue #38730 remains open, and the submitted patch needs rework before merge because review comments identify concrete logger/output and coverage issues.

## Best Possible Solution

Keep this PR open or replace it with a revised implementation. The best fix is a shared high-frequency `every` schedule helper applied consistently to CLI `cron add`, CLI `cron edit --every`, gateway `cron.add`, and gateway `cron.update`; short intervals should remain valid, the advisory should be non-fatal, and focused CLI/gateway regression tests should cover both add and update paths.
