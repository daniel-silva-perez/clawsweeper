## Summary

Keep open. Current main still lacks the requested non-blocking high-frequency `--every <30m` cron advisory in CLI and gateway paths, the paired issue #38730 remains open, and this PR is a focused implementation candidate. It needs rework before merge because the supplied patch only covers add paths and review comments identify concrete logger/output issues.

## Best Possible Solution

Keep the PR open or replace it with a revised implementation. Centralize a small high-frequency `every` schedule helper and apply it consistently in CLI `cron add`, CLI `cron edit --every`, gateway `cron.add`, and gateway `cron.update`. Keep sub-30-minute schedules valid, emit/log the advisory non-fatally through the correct CLI warning/log path and `SubsystemLogger.warn(message, meta)` order, and add focused regression tests for add and update paths.
