## Summary

Keep this PR open as useful source work for the Docker whitespace bug, but do not merge it as-is. Current main still rejects whitespace in Docker config and workspace paths, while this PR is stale against the moved setup implementation and still has generated YAML quoting gaps.

## What This Changes

The PR removes Docker setup whitespace rejection for config/workspace paths, relaxes extra mount validation, and emits generated compose volume entries as single-quoted YAML scalars in the old root `docker-setup.sh`.

## Best Possible Solution

Implement the whitespace-path fix in `scripts/docker/setup.sh` with a small, reusable YAML scalar emitter that preserves spaces and backslashes while escaping apostrophes, use it for generated extra mounts and the sandbox socket overlay, and add focused Docker setup e2e coverage for config, workspace, and extra mount paths with spaces and apostrophes.

## Reproduction Assessment

Yes. A source-level reproduction is clear: on current main, any `OPENCLAW_CONFIG_DIR` or `OPENCLAW_WORKSPACE_DIR` containing a space reaches `validate_mount_path_value` and fails with the existing whitespace check before setup can proceed.

## Solution Assessment

No. The requested direction is right, but this PR is not the best current fix because it edits the old root implementation, leaves single-quoted YAML unescaped, and does not handle the sandbox socket overlay.
