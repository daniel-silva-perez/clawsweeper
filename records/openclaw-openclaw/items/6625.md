## Summary

Keep this issue open. Current main still has sub-agent run timeouts and post-timeout partial-progress recovery, but it does not implement a configurable pre-timeout grace period, a `sessions_spawn` warning parameter, or a native/ACP mechanism that warns the child agent before the hard timeout.

## Best Possible Solution

Keep this issue open for core sub-agent lifecycle work. Add a documented timeout-warning/grace setting, optionally a per-spawn `sessions_spawn` override, schedule a warning before `runTimeoutSeconds` expires, deliver it through a child-session control/message path for native and ACP runs where feasible, and cover it with focused tests and docs while preserving existing post-timeout partial-progress recovery.
