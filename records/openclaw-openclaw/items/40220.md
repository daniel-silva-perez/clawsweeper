## Summary

Keep open. Current main still serializes the raw tslog `_meta` object unchanged in file logs while adding a separate top-level `time` field with an explicit local offset, so raw JSONL can still show `_meta.date` as a UTC `Z` timestamp. The directly related implementation PR is closed, unmerged, and had no file changes, so there is no shipped or active direct fix.

## What This Changes

The issue requests consistent raw JSONL log timestamp fields so `_meta.date` no longer shows UTC while the top-level `time` field shows an offset/local timestamp.

## Best Possible Solution

Normalize the copied raw JSONL file-log payload so `_meta.date` is derived from the same original timestamp and formatted identically to top-level `time`, with focused regression coverage for raw file output. Keep the documented host-local transport policy unless maintainers explicitly decide to change it.
