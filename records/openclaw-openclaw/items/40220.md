## Summary

Keep open. Current main still serializes the original tslog `_meta.date` Date object unchanged in JSONL file logs while adding a separate top-level `time` string with a local offset, so the raw mismatch remains. The related implementation PR #48717 is closed and unmerged.

## Best Possible Solution

Keep this issue open. Patch `src/logging/logger.ts` so the JSONL file transport derives one timestamp from the original tslog metadata when present, writes it to top-level `time`, and serializes a copied `_meta.date` with the same explicit-offset value. Add focused regression coverage in `src/logging/logger-timestamp.test.ts` asserting `_meta.date === time` and neither value ends in `Z`. Treat any move from host-local log timestamps to `agents.defaults.userTimezone` as a separate maintainer product/docs decision.
