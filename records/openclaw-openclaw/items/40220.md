## Summary

Keep open. Current main still computes the file-log top-level `time` with a local offset but serializes the original log object without rewriting nested `_meta.date`, so raw JSONL can still show `_meta.date` as UTC `Z` while `time` shows local-offset time. The directly related fix PR is closed, unmerged, and has no changed files.

## What This Changes

The issue asks OpenClaw to stop emitting inconsistent raw log timestamps where `_meta.date` is UTC while the top-level `time` field is local or configured-timezone formatted.

## Best Possible Solution

Normalize `_meta.date` in the copied JSONL file-log payload so it equals the computed top-level `time` string for the same record, avoid mutating the original tslog object, preserve existing file-log timestamp policy, and add regression coverage for both fields.
