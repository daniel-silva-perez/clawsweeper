## Summary

Keep open. Current main still leaves the raw JSONL log mismatch unresolved: file logs add a local-offset top-level `time` while serializing tslog's original `_meta.date` metadata unchanged, so raw readers can still see `_meta.date` as UTC `Z`. The related PR #48717 is closed and unmerged in the provided context, so it is not a shipped fix.

## Best Possible Solution

Keep this issue open. Patch `src/logging/logger.ts` so the file-log JSON payload copies `_meta` and serializes `_meta.date` with the same explicit-offset timestamp used for top-level `time`, preferably deriving both from tslog's `_meta.date` when present. Add focused regression coverage around file-log JSONL output asserting `_meta.date === time` and the offset format. Treat switching file logs from host-local time to `agents.defaults.userTimezone` as a separate product/docs decision unless maintainers explicitly want that broader behavior here.
