## Summary

Keep this issue open. Current main has subagent completion announces, plugin spawn/end hooks, a persisted task ledger, and an in-process announce queue, but it still does not implement the central request: framework-owned immediate requester-channel spawn acknowledgements, native per-run progress lifecycle notifications, or a durable announce queue that survives gateway restarts/channel outages.

## Best Possible Solution

Keep this open as the umbrella runtime architecture request. The best implementation path is to extend the existing subagent registry, task ledger, and completion announce flow with runtime-owned lifecycle events for spawned/running/progress/completed/failed/cancelled, immediate requester-channel spawn acknowledgement, durable persisted announce retry with idempotency keys, and delivery that is not starved by the parent session lane. Coordinate with #22358 and #54488, but do not close this until the broader lifecycle delivery contract ships.
