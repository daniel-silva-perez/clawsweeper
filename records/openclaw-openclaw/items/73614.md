## Summary

Keep this PR open. Current main still has the reported logging.file tilde behavior: configured logging.file values are read directly from openclaw.json, resolveActiveLogFile returns non-rolling paths unchanged, and buildLogger passes the resulting dirname to fs.mkdirSync. The PR remains a focused implementation candidate for the open linked regression, with only a small test-helper review nit to resolve.

## Best Possible Solution

Keep this PR open as the implementation candidate for the linked logging.file regression. The best path is to address the test-helper nit by reusing resolveEffectiveHomeDir(), then land the focused logger change with the new regression tests and changelog entry once maintainer validation is green.
