## Summary

Keep this PR open. Current main still reads `logging.file` from config without tilde expansion, passes the resulting dirname to `fs.mkdirSync`, and `resolveActiveLogFile()` returns non-rolling paths verbatim. The PR remains a focused implementation candidate for #73587, with one test-helper cleanup still worth addressing before maintainer validation.

## Best Possible Solution

Keep this PR open as the implementation candidate for #73587. The best path is to update the test helper to reuse `resolveEffectiveHomeDir()`, keep the narrow `expandHomePrefix()` call inside `resolveActiveLogFile()`, and land with focused logging regression coverage plus the normal changed-gate validation.
