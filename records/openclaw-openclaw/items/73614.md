## Summary

Keep this PR open. Current main still passes configured non-rolling `logging.file` values through unchanged, then calls `fs.mkdirSync(path.dirname(activeFile), { recursive: true })`, so a `~/...` path can still reach Node filesystem APIs as a literal tilde. The PR remains a focused implementation candidate for the linked open regression, with one small test-helper nit to address before maintainer landing.

## Best Possible Solution

Keep the PR open as the implementation candidate for the linked logging regression. Update the test helper to call `resolveEffectiveHomeDir()`, then land the focused `expandHomePrefix()` change with the regression tests and changelog entry after the targeted logging checks pass.
