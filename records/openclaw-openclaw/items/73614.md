## Summary

Keep this PR open. Current main still has the reported `logging.file` tilde behavior: non-rolling log paths are returned verbatim, then their dirname is passed to `fs.mkdirSync`, so a configured `~/.openclaw/logs/gateway.log` can still reach filesystem setup as a literal `~` path. The PR remains a focused implementation candidate for the still-open linked regression, with only a small test-helper review nit noted in the discussion.

## Best Possible Solution

Keep this PR open as the implementation candidate for #73587. The best path is to align the regression test helper with `resolveEffectiveHomeDir()` if that nit is still present, then land the focused logger change with targeted logging tests and the changelog entry once normal maintainer validation is green.
