## Summary

Keep this PR open. Current main still returns non-rolling `logging.file` paths verbatim and passes the literal dirname to `fs.mkdirSync`, so the linked `~/...` gateway startup crash remains plausible. The PR is still a focused implementation candidate for #73587, with a small test-helper cleanup to address before maintainer validation.

## Best Possible Solution

Keep this PR open as the implementation candidate for #73587. The best path is to address the `effectiveHome()` test-helper nit by reusing `resolveEffectiveHomeDir()`, then land the narrow `resolveActiveLogFile()` change with focused logging regression tests and the changelog entry after maintainer validation is green.
