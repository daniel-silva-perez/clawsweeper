## Summary

Keep this PR open. Current main still leaves non-rolling `logging.file` values untouched, so a configured `~/.openclaw/logs/gateway.log` can still flow into `fs.mkdirSync` as a literal tilde path. The PR remains a focused implementation candidate for the linked regression, with one small test-helper nit to resolve before normal maintainer validation.

## Best Possible Solution

Keep this PR open as the implementation candidate for #73587. The best path is to update the test helper to reuse `resolveEffectiveHomeDir()`, then land the narrow `resolveActiveLogFile()` change with the added regression tests and changelog entry after maintainer validation is green.
