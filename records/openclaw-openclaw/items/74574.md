## Summary

Keep open. Current source and the published 2026.4.26 package both contain JSON branches for `browser open` and `browser tabs`, but the reported runtime symptom is still plausible if the browser parent `--json` option is not reaching the manage handlers after lazy command registration; I did not find a real lazy-CLI regression test proving those two commands emit JSON on stdout.

## What This Changes

The issue reports that `openclaw browser --json open` and `openclaw browser --json tabs` emit plain text on stderr instead of machine-readable JSON on stdout in 2026.4.26.

## Best Possible Solution

Add regression coverage that drives the real browser CLI registration path for `openclaw browser --json open` and `openclaw browser --json tabs`, then make the manage handlers receive the selected parent JSON option reliably and keep stdout reserved for structured JSON.
