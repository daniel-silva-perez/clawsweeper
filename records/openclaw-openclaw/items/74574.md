## Summary

Keep open. Current main has not materially changed the affected browser CLI output path since the reported release, and the implementation still only produces JSON for `open` and `tabs` when the browser parent `json` option reaches the manage handler. The report is concrete, recent, and narrow enough for a focused regression fix rather than cleanup closure.

## What This Changes

The issue reports that `openclaw browser --json open` and `openclaw browser --json tabs` in v2026.4.26 ignore JSON mode and emit human-readable output instead of parseable stdout JSON.

## Best Possible Solution

Add regression tests that drive `openclaw browser --json open` and `openclaw browser --json tabs` through the real lazy browser registration path, then preserve the selected parent JSON mode so those commands emit structured JSON to stdout while human-readable logs stay out of the machine-readable result.
