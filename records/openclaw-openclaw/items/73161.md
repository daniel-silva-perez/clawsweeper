## Summary

Keep this PR open, but do not merge it as-is. The stale subagent cancellation fix is still useful because current main still leaves missing-child subagent tasks uncancelled, but the branch is stale against current main and introduces blocking runtime packaging, security-audit, and generated-config documentation regressions.

## What This Changes

The PR changes task runtime postbuild aliasing and subagent cancellation, Discord and Telegram liveness handling, outbound payload/message-tool sanitization, failover classification, security trust-model config, `.gitignore`, and coding-agent guidance.

## Best Possible Solution

Rebase and split this into focused changes: keep the narrow stale subagent cancellation fix, drop or adapt superseded runtime and Discord liveness hunks to current main, and handle `security.trustModel` as a separate security-reviewed config/docs change with generated artifacts updated.

## Reproduction Assessment

Yes. The stale task bug has a focused unit reproduction by mocking `killSubagentRunAdmin` to return `{ found: false, killed: false }`; the PR regressions are reproducible by inspecting the generated subagent wrapper path and the missing generated config artifacts.

## Solution Assessment

No. The branch solves a real stale-task problem, but the current patch mixes unrelated changes, overlaps with newer main behavior, and introduces runtime and security regressions; the safer solution is a rebased, narrower patch set.
