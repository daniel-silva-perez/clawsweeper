Implement a narrow replacement for the stale WhatsApp reconnect-window PR. Current main still uses `extensions/whatsapp`, not `src/web`; do not reintroduce or modify `src/web/auto-reply/*`. In `extensions/whatsapp/src/auto-reply/deliver-reply.ts` and/or `extensions/whatsapp/src/outbound-media-contract.ts`, widen the auto-reply send retry window from the current 3 attempts / 500ms linear backoff to the intended reconnect-safe 5 attempts / 1000ms linear backoff. Add regression coverage that a transient text send can succeed on the fifth attempt and that retry sleeps use 1000/2000/3000/4000ms. Guard the handler to `info.kind === "final"`, preserve existing logging, and sanitize/classify the agent-visible error text so untrusted transport strings or unnecessary identifiers are not injected into agent context. Add tests in current `extensions/whatsapp` tests for the final-only system event and for no event on tool/block errors. Include a single-line user-facing changelog fix entry with proper attribution to the original report/PR participants; do not add a durable outbound queue or broad transport refactor in this PR.
