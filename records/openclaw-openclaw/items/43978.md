## Summary

Keep open. Current main still has the old WhatsApp auto-reply retry window and does not surface final WhatsApp reply delivery failures as agent-visible system events. The PR is a plausible fix candidate, but it targets removed `src/web/auto-reply/*` paths and needs to be rebased into `extensions/whatsapp`, with the Greptile final-only `onError` concern addressed.

## Best Possible Solution

Keep this PR open as a useful but stale WhatsApp reliability fix candidate. The best path is to rebase/adapt it into `extensions/whatsapp`, widen the auto-reply send retry window in the current retry helper or call site, add a final-only sanitized `onError` system event using the existing plugin SDK seam, and add regression tests for retry timing plus agent-visible final delivery failure handling. Because the change is user-facing, the final PR should include a changelog fix entry with proper attribution.
