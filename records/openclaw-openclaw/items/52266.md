## Summary

Keep this issue open. Current main partially supports WebChat session switching through a grouped header/select control and the Sessions page, but it still does not implement the requested Discord-style session sidebar or wire the WebChat New session action to create and switch to a new dashboard session. The provided discussion also shows open maintainer PR #73725 as the active replacement path for the WebChat create/switch slice, so closing before that lands would be premature.

## Best Possible Solution

Keep this issue open while maintainers review or land #73725 for the focused WebChat New Chat `sessions.create` create/switch path. After that lands, reassess the remaining sidebar/list UX in the Control UI chat/sidebar code and decide whether WebChat should also gain thread-bound subagent hooks or whether that part should stay scoped to channel-backed threads.
