## Summary

Keep this issue open. Current main partially covers session switching through a grouped chat-header/mobile select and label editing on the Sessions page, but it does not implement the requested session switching sidebar or wire the WebChat New Session action to create/switch isolated dashboard sessions. The provided GitHub context also shows open maintainer PR #73725 as the active replacement path for the WebChat create/switch slice, so closing before that lands would be premature.

## Best Possible Solution

Keep this issue open while maintainers review or land #73725 for the narrow sessions.create-backed New Chat/create-switch behavior. After that lands, reassess whether the remaining sidebar/list UX should be implemented in the Control UI chat/nav code or consolidated under the broader native conversation-management thread.
