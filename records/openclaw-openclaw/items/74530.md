## Summary

Keep open. Current main already has the shared visible-catalog policy for Gateway, Web chat, and text `/models`, but it still uses the narrower provider-auth checker that can hide runtime-valid Bedrock AWS SDK and local no-key custom providers. The PR is a plausible narrow follow-up and I did not find a blocking diff-level bug in the proposed patch.

## What This Changes

The PR adds a synchronous runtime auth availability helper, routes model provider visibility checks through it, and adds regression coverage for implicit Bedrock AWS SDK auth and local no-key custom providers.

## Best Possible Solution

Land a narrow fix that keeps default model browsing surfaces on the shared visible-catalog policy while making provider visibility honor runtime-valid auth modes such as Bedrock AWS SDK and synthetic local providers, with focused regression coverage.
