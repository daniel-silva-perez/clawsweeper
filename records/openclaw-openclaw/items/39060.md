## Summary

Keep open. Current main still lacks the Agent Files word-wrap toggle, so the PR is not obsolete, but the submitted patch has a blocking UI correctness issue: the disabled state removes the wrap class without adding an explicit nowrap state, so the editor can continue wrapping while the button appears off. The branch also predates the current nested Agent Files state plumbing and needs a refresh or narrow replacement against main.

## What This Changes

The PR branch adds Control UI Agent Files word-wrap state plumbing, a Wrap button, and stretched file-content styling across app state, render, view, and CSS files.

## Best Possible Solution

Land a narrow Control UI implementation that wires word-wrap state through the current nested AgentFilesState/AgentsProps path, adds accessible wrap and nowrap editor states with explicit CSS, preserves the markdown preview/editor behavior, avoids unrelated stylesheet churn, and covers both toggle states in focused render tests.
