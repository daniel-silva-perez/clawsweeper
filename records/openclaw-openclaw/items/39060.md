## Summary

Keep open. Current main does not have an Agent Files word-wrap toggle, so the contribution is still relevant, but this PR should not merge as-is because the off state never applies nowrap behavior and the branch is stale against the current nested Agent Files wiring.

## What This Changes

This PR adds Control UI Agent Files word-wrap state plumbing, a Wrap button, stretched Agent Files layout styles, and textarea wrap styling across app state/render, Agents view, and CSS files.

## Best Possible Solution

Land a narrow Control UI implementation on current main that stores the wrap preference in the current AgentFilesState/AgentsProps path, applies explicit wrap and nowrap textarea states, localizes the new labels/titles, preserves existing markdown preview behavior, and covers both toggle states in focused render tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. Code inspection of the PR branch is enough: rendering with agentFileWordWrap=false emits no nowrap class and the PR adds no false-state CSS, so the toggle can appear off while wrapping remains browser-default behavior.

## Solution Assessment

Is this the best way to solve the issue? No, not as patched. The feature direction is narrow and maintainable, but the implementation should be rebased onto the current nested Agent Files state flow and add explicit nowrap semantics plus locale keys before merge.
