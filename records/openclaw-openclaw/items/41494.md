## Summary

Keep this issue open. Current main still covers native Gemini thought parts, explicit `isReasoning` payloads, exact `Reasoning:` payload prefixes, and tagged `<think>/<final>` cleanup, but it does not implement the broader plain-text or multilingual Gemini reasoning-preamble stripping reported here. Open PR #60551 remains an implementation candidate, not shipped behavior.

## Best Possible Solution

Keep this issue open as the Gemini plain-text reasoning leakage tracker. Review and land PR #60551 or a replacement that adds a conservative shared outbound/message-tool sanitizer with regression tests for untagged planning prose, appended reasoning blocks, multilingual reasoning headers, tagged fragments, code/prose pass-through, and normal `/reasoning` visibility.

## Risks / Open Questions

- A broad plain-text sanitizer can false-positive legitimate prose, quoted examples, code blocks, or multilingual user-facing answers unless it is tightly scoped and regression-tested.

- Open PR #60551 may contain the right direction, but the issue should stay open until that PR or a replacement lands on main.
