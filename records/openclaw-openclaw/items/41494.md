## Summary

Current main still does not fully address the reported Gemini plain-text reasoning leak. It hides provider-marked Google thought parts and XML-style reasoning tags, but unmarked/localized reasoning preambles can still travel as normal visible text. PR #60551 remains open and unmerged and appears to be the right implementation candidate or reference point.

## Best Possible Solution

Keep this open as the Gemini/plain-text reasoning leakage tracker. Review and land PR #60551 or a replacement that adds a conservative shared sanitizer for untagged reasoning/planning preambles before outbound delivery, with regression tests for the reported Gemini examples, localized headers, tagged fragments, code/prose pass-through, normal `/reasoning` visibility, and Telegram/message-tool delivery paths.

## Evidence

Plain Gemini reasoning text without the Google thought flag remains visible.

## Risks / Open Questions

- A broad plain-text reasoning heuristic can false-positive legitimate prose, quoted examples, code blocks, or localized user-facing text unless it is conservative and well covered.
- Fixing only Telegram would leave other outbound/message-tool paths able to deliver the same plain-text reasoning leak; the shared sanitizer/outbound normalization path is the better seam.
- Provider-marked Google `thought` parts are already handled, but the reported leak shapes are unmarked text, so a provider-flag-only fix is insufficient.
