## Summary

Current main still does not fully cover the reported Gemini plain-text reasoning leak. The code hides provider-marked Google `thought` parts and strips XML-like `<think>/<final>` tags, but unmarked Gemini prose such as `Thinking Process:`, `Analysis:`, `Tankegang:`, `Analyse:`, or `Begrunnelse:` is not classified before delivery. Recent issue discussion also points to open PR #60551 as the likely fix candidate, so this should remain open for maintainer follow-up.

## Best Possible Solution

Keep this issue open as the Gemini plain-text reasoning leakage tracker. Review and land PR #60551 or a replacement that adds a conservative shared outbound/message-tool sanitizer with regression tests for untagged planning prose, appended reasoning blocks, localized headers, tagged fragments, code/prose pass-through, and normal `/reasoning` visibility.

## Risks / Open Questions

- A broad text sanitizer can false-positive legitimate prose, quoted examples, code, or localized user-facing text unless it is narrowly scoped and well tested.
- The fix likely needs coordination between shared outbound delivery, Telegram streaming, and message-tool sends so channel paths do not drift.
- No live Gemini plus Telegram roundtrip was run in this read-only review, but current source still lacks the plain-text classifier needed for the reported examples.
