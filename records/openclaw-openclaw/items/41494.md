## Summary

Keep this issue open. Current main has partial protections for native Google thought parts, explicit reasoning payloads, exact `Reasoning:` prefixes on some channel paths, and tagged `<think>/<final>` cleanup, but it still does not classify or strip the broader Gemini leak shape reported here: plain-text or localized reasoning preambles delivered as ordinary assistant text.

## Best Possible Solution

Keep this issue open as the Gemini plain-text reasoning leakage tracker. Review and land PR #60551 or a replacement that adds a conservative shared outbound/message-tool sanitizer with tests for untagged planning prose, appended reasoning blocks, multilingual reasoning headers, tagged fragments, code/prose pass-through, and normal `/reasoning` visibility.
