## Summary

Keep this issue open. Current main has several structured protections for Gemini and channel reasoning, but the broader user-reported case remains: plain-text or multilingual reasoning preambles can still be treated as normal assistant text and delivered. The discussion also points to open PR #60551 as candidate work, which is not present on current main.

## Best Possible Solution

Keep this issue open as the Gemini plain-text reasoning leakage tracker. Review and land PR #60551 or a replacement that adds a conservative shared outbound/message-tool sanitizer with tests for untagged planning prose, appended reasoning blocks, multilingual reasoning headers, tagged fragments, code/prose pass-through, and normal /reasoning visibility.
