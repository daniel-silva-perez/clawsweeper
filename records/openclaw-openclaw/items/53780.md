## Summary

Keep open. Current main still lacks a media-fetch-specific sanitizer in the outbound action and delivery paths, so the PR targets a real privacy gap linked to #20279. The proposed patch should not merge as-is because it bypasses existing `formatErrorMessage` handling for non-media errors and uses recursive cause traversal on arbitrary caught errors.

## What This Changes

The PR adds a safe `MediaFetchError` message/helper, redacts media fetch failures at attachment hydration, broadcast, and `message_sent` delivery boundaries, and adds focused sanitizer tests.

## Best Possible Solution

Land a rebased media-specific sanitizer that maps `MediaFetchError` failures to one stable channel-safe message at attachment, action-result, and hook boundaries, preserves `formatErrorMessage` for every non-media failure, uses cycle-safe cause traversal, and covers media plus non-media regression cases in tests.
