## Summary

Keep open: current main still lacks a media-specific outbound sanitizer for these action and delivery paths, so the PR is not obsolete. The branch is also dirty against main and the proposed diff has blocking regressions that should be fixed before merge: non-media failures would lose existing formatted/redacted diagnostics, and the new cause-chain helper is not cycle-safe.

## What This Changes

The PR adds a safe MediaFetchError message/helper, catches media fetch failures during outbound attachment hydration, substitutes the safe message in broadcast and message_sent failure payloads, and adds sanitizer tests.

## Best Possible Solution

Land a rebased media-specific sanitizer that maps MediaFetchError failures to one stable channel-safe message at attachment hydration, broadcast result, and message_sent hook boundaries, while preserving formatErrorMessage for all non-media failures and using cycle-safe cause traversal with focused regression tests.

  - confidence: 0.82
