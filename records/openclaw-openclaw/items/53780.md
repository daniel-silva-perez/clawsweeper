## Summary

Keep open. Current main does not implement the media-specific sanitizer, so the PR is not obsolete, but the branch should not merge as-is because the patch drops current non-media error formatting/redaction and uses recursive cause-chain traversal that can overflow on cycles.

## What This Changes

The PR adds a safe MediaFetchError message/helper, redacts media fetch failures in outbound action and delivery paths, and adds focused sanitizer tests.

## Best Possible Solution

Land a rebased media-specific sanitizer that maps MediaFetchError failures to one stable channel-safe message at outbound action and hook boundaries, preserves formatErrorMessage for every non-media failure, uses cycle-safe cause traversal, and covers attachment hydration, broadcast results, and message_sent failures in tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve redacted message_sent diagnostics:** `src/infra/outbound/deliver.ts:761-762`
  - body: For non-media failures this new rawError path bypasses current main's formatErrorMessage behavior, losing string/object formatting, cause-chain context, and sensitive-token redaction before message_sent hooks receive the error. Keep the safe media message only for isMediaFetchError(err); otherwise use the existing formatter.
  - confidence: 0.9
- **[P2] Preserve redacted broadcast diagnostics:** `src/infra/outbound/message-action-runner.ts:348-353`
  - body: The broadcast failure path has the same regression: non-media thrown strings or plain objects become "unknown error", and Error messages skip the current formatter's redaction and cause-chain handling. Only replace the text for media fetch failures and preserve existing formatting otherwise.
  - confidence: 0.88
- **[P2] Guard cause-chain traversal against cycles:** `src/media/fetch.ts:35`
  - body: isMediaFetchError is called on arbitrary caught outbound errors, but it recursively follows Error.cause without tracking seen values. A cyclic cause chain from a plugin or adapter error would overflow before the failure can be handled; use an iterative traversal or the existing cycle-safe helper pattern.
  - confidence: 0.82
