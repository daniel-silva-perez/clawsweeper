## Summary

Keep open. Current main has partial OAuth refresh improvements, including error wrapping and runtime snapshot refresh, but the central Gemini OAuth refresh complaint is still not fully resolved: automatic provider resolution still logs profile-resolution failures only at debug level, the google-gemini-cli provider still has no provider-owned refreshOAuth hook, and cooldown bypass still excludes google-gemini-cli. The issue discussion also has recent confirmations and the linked PR #42610 was closed unmerged.

## Best Possible Solution

Keep this issue open for a focused Gemini OAuth refresh fix. The likely path is to add a regression test for an expired google-gemini-cli auth profile through the embedded runtime/provider-resolution path, surface refresh failures at warn/error or user-visible status level instead of debug-only, decide whether google-gemini-cli needs a provider-owned refreshOAuth implementation, and make an explicit product decision on Gemini cooldown behavior.
