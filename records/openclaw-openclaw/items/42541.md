## Summary

Keep open. Current main still does not fully resolve the Gemini OAuth refresh report: automatic provider auth catches refresh/profile failures and logs only at debug level, provider-usage OAuth resolution still ignores refresh errors entirely, google-gemini-cli still has no provider-owned refreshOAuth hook, and Gemini remains subject to normal auth cooldown handling. The runtime snapshot/cache part is fixed, and direct profile refresh now wraps errors with a user-facing OAuth refresh message, but the central silent/low-visibility automatic refresh paths remain unresolved.

## Best Possible Solution

Keep this issue open for a focused Gemini OAuth refresh fix. The likely maintainer path is to add regression coverage for expired google-gemini-cli OAuth profiles through embedded/runtime provider resolution, surface automatic and usage-auth refresh failures at an operator-visible level, decide whether google-gemini-cli should implement provider-owned refreshOAuth or doctor guidance, and make an explicit product decision on Gemini cooldown behavior.
