## Summary

Keep open. Current main still has unresolved Gemini OAuth refresh visibility and recovery gaps: automatic profile resolution only logs refresh/profile failures at debug level, provider-usage OAuth resolution still ignores thrown refresh errors, google-gemini-cli still has no provider-owned refreshOAuth hook, and Gemini is still not excluded from auth cooldown handling. Recent issue discussion also reports the behavior still live, while the linked logging PR was closed unmerged.

## Best Possible Solution

Keep this issue open for a focused Gemini OAuth refresh fix. The likely maintainer path is to add regression coverage for expired google-gemini-cli OAuth profiles through the embedded/runtime provider-resolution path, surface automatic and usage-auth refresh failures at an operator-visible level, decide whether google-gemini-cli should implement refreshOAuth, and make an explicit product decision on Gemini cooldown behavior.
