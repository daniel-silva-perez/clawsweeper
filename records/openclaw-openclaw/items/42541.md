## Summary

Keep open. Current main still has the same unresolved Gemini OAuth refresh visibility and recovery gaps: automatic profile resolution logs refresh/profile failures only at debug level, provider-usage OAuth resolution still discards refresh errors, google-gemini-cli still has no provider-owned refreshOAuth hook, and Gemini is still subject to normal auth cooldown handling. Part of the report is fixed: refreshed stores update the runtime snapshot, and direct profile refresh now preserves cause/wraps an operator-facing error.

## Best Possible Solution

Keep this issue open for a focused Gemini OAuth refresh fix. The likely maintainer path is to add regression coverage for expired google-gemini-cli OAuth profiles through embedded/runtime provider resolution, surface automatic and usage-auth refresh failures at an operator-visible level, decide whether google-gemini-cli should implement refreshOAuth or provider-owned doctor guidance, and make an explicit product decision on Gemini cooldown behavior.
