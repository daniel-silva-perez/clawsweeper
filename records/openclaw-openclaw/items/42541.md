## Summary

Keep open. Current main has several partial OAuth refresh fixes, but the Gemini OAuth refresh report is not fully resolved: automatic provider resolution still downgrades profile-resolution failures to debug-only logging, a usage-auth path still ignores OAuth resolution errors outright, google-gemini-cli still lacks a provider-owned refreshOAuth hook, and the cooldown bypass request remains unimplemented. Recent issue discussion also reports the bug still live, and the related logging PR was closed unmerged.

## Best Possible Solution

Keep this issue open for a focused Gemini OAuth refresh fix. The likely maintainer path is to add regression coverage for an expired google-gemini-cli OAuth profile through the embedded/runtime provider-resolution path, surface refresh failures from automatic and usage-auth resolution at an operator-visible level, decide whether google-gemini-cli should own refreshOAuth, and make an explicit product decision on Gemini cooldown behavior.
