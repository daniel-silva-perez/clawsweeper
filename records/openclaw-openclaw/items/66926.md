## Summary

Keep open. Current main still applies tagged reasoning to Google/Gemini and still injects the reported text-only `<think>/<final>` contract, while the direct Google transport already has native function-call request and response plumbing. The observable bug is therefore not implemented or superseded.

## What This Changes

The issue requests fixing Google provider reasoning prompts so Google models invoke registered tools through native function calling instead of writing tool-call-shaped text inside `<final>` output.

## Best Possible Solution

Fix the Google-owned reasoning-output path so direct `google-generative-ai` models with native Gemini thinking/function calling are not instructed to express the whole turn as text-only `<think>/<final>` output. Preserve the existing native `functionDeclarations` and `functionCall` transport handling, add focused regression coverage for Google prompt composition with tools, and update provider SDK docs if the Gemini replay-family contract changes.
