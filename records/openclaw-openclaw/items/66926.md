## Summary

Keep this issue open. Current main still marks direct Google Gemini as tagged reasoning, still injects the reported <think>/<final> prompt wording into embedded agent prompts, and still has native Google function-call request/response plumbing rather than a fix that reconciles tagged text prompting with provider tool calls. The reporter's follow-up is also relevant: the direct Google transport can send tool schemas correctly, so the remaining work is specifically around Google reasoning-output/tool-call behavior, not missing basic functionDeclaration plumbing.

## Best Possible Solution

Keep this issue open and fix the Google provider reasoning-output path. The strongest maintainer path is to stop using tagged reasoning for direct `google-generative-ai` when native Gemini thinking/function calling is available, or update the tagged prompt so native function calls are explicitly outside the `<think>/<final>` text contract and must be invoked through provider function calling. Add focused regression coverage around Google `resolveReasoningOutputMode`/prompt composition with tools, while preserving the existing Google transport tests for `functionDeclarations` and `functionCall` parsing.
