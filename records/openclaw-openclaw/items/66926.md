## Summary

Keep this issue open. Current main still routes Google Gemini through tagged reasoning, still injects the reported <think>/<final> text contract, and the direct Google transport already has native function-call request/response plumbing. The requested fix is not implemented or clearly superseded.

## Best Possible Solution

Keep this issue open and fix the Google provider reasoning-output path. The likely maintainer path is to stop using tagged reasoning for direct google-generative-ai when native Gemini thinking/function calling is available, or to update the tagged prompt so native function calls are explicitly outside the <think>/<final> text contract and must be invoked through provider function calling. Add focused regression coverage around Google resolveReasoningOutputMode/prompt composition with tools, while preserving existing Google transport tests for functionDeclarations and functionCall parsing.
