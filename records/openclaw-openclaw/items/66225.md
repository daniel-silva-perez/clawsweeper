## Summary

Keep this PR open. Current main still has the narrow final-tag regexes in the streaming subscriber and user-facing sanitizer, so the self-closing `<final/>` leak described by the PR and linked #65867 is not implemented on main. The PR is a real implementation candidate, but its current scanner still needs maintainer review or hardening for strict-mode malformed tag handling before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this fix or an equivalent shared final-tag parser that strips `<final/>` and `<final />` in the streaming and sanitizer paths while preserving strict-mode fail-closed behavior for malformed tags. Add regression coverage for sanitizer stripping, streaming with enforcement off, streaming with enforcement on, slash-containing attributes, and malformed attribute text; then close #65867 as implemented and dedupe overlapping open PRs such as #48593.
