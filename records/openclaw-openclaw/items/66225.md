## Summary

Keep this PR open. Current main still has the narrow final-tag regex in both the streaming subscriber and user-facing sanitizer, so the self-closing `<final/>` leak described by this PR and linked #65867 is not implemented on main. The recent main change only preserves split `<final>` tags across streamed deltas; it does not handle `<final/>` or `<final />`. Security review found no workflow, dependency, lockfile, or install-script changes in the PR diff, but the strict final-tag parser still needs maintainer review for malformed/attribute-heavy tag handling before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this fix or an equivalent shared final-tag parser that strips `<final/>`, `<final />`, and ordinary final tags in both streaming and user-facing sanitizer paths while preserving strict-mode fail-closed behavior for malformed tags. Add focused regression coverage for sanitizer stripping, streaming with enforcement off, streaming with enforcement on, slash-containing attributes, and malformed attribute text; then close #65867 as implemented and dedupe overlapping open PRs such as #48593.
