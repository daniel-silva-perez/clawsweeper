## Summary

Keep this PR open. Current `main` still has the narrow final-tag regexes in both affected paths, so the self-closing `<final/>` leak is not implemented on main or in `v2026.4.26`. The PR is a relevant fix candidate for #65867, but it needs maintainer review/rework because the branch is currently dirty and the proposed broader strict-mode scanner still has a malformed-tag fail-closed risk called out in review.

## What This Changes

The PR broadens final-tag regexes in the embedded streaming subscriber and user-facing sanitizer to strip self-closing `<final/>` tags, adds explicit self-closing handling in strict final-tag parsing, and adds sanitizer/streaming regression tests.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this fix or an equivalent shared final-tag parser after rebasing and tightening strict-mode parsing so `<final/>`, `<final />`, ordinary closing tags, and slash-containing attributes are handled while malformed tag text still fails closed. Add focused regression coverage for sanitizer stripping, streaming with enforcement off, streaming with enforcement on, slash-containing attributes, and malformed attribute text; then close #65867 and dedupe #48593 as appropriate.
