## Summary

Keep this PR open. The linked discussion ties it to #65867, where users reported Gemini/Gemma `<final/>` leakage, and current main still has the narrow final-tag regexes in the two affected paths. The PR is a plausible focused implementation candidate; its earlier review feedback was addressed, but the latest malformed-attribute strict-mode concern still needs maintainer review or a hardened parser before landing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this fix after hardening the strict final-tag parser, or land an equivalent shared final-tag matcher/parser used by both the streaming subscriber and user-facing sanitizer. Regression coverage should include `sanitizeUserFacingText`, streaming with enforcement off, streaming with enforcement on, `<final/>`, `<final />`, slash-containing attributes, and malformed-attribute fail-closed behavior; after that ships, #65867 can close as implemented.
