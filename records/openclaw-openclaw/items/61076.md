## Summary

Keep open. Current main has partial hardening for malformed Anthropic streaming fragments, including stable error classification and a lifecycle backstop, but the Anthropic SSE parser still treats raw control-character JSON parse failures as terminal stream errors rather than recovering the long pure-text Telegram response described here. The closed related webchat/tool-call issue does not fully supersede this Telegram pure-text failure mode.

## What This Changes

Prevent embedded Anthropic streaming JSON control-character parse failures from aborting long Telegram pure-text runs and dropping the user-visible reply.

## Best Possible Solution

Add focused regression coverage for Anthropic SSE frames containing the reported raw-control-character JSON failure, long emoji/non-ASCII text, and split UTF-8 chunks; then implement a narrow Anthropic transport or embedded-runner fix that either safely recovers text deltas or guarantees a visible terminal error without silently dropping the reply.
