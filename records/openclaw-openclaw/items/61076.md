## Summary

Keep this issue open. Current main still routes embedded Anthropic Messages runs through the boundary-aware transport, and that transport still parses SSE `data:` frames with raw `JSON.parse` without scoped recovery for raw control characters in JSON strings. The report is recent and concrete, and related #68442 remains open rather than superseding this Telegram pure-text failure mode.

## Best Possible Solution

Keep this issue open. Add focused regression coverage around `parseAnthropicSseBody` and the embedded Anthropic Messages runner path for long emoji/non-ASCII text, split UTF-8 chunk boundaries, and an invalid raw-control-character-in-string SSE frame matching the reported `JSON.parse` failure. If reproduced, harden the Anthropic SSE parser or caller with scoped recovery/failure handling that preserves terminal lifecycle delivery and avoids silently dropping user-visible replies. Evaluate separately whether the same parser fix also resolves #68442 or whether that webchat/tool-call freeze needs its own UI/runtime fix.
