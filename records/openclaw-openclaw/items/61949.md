## Summary

Keep open. Current main still injects the contested `Model Aliases` prompt block, still tells the model to prefer aliases for model overrides, and builds those alias lines from every configured `agents.defaults.models` entry without primary/fallback or overlapping-provider awareness. The issue discussion describes a concrete failure mode that still fits the current prompt-to-`sessions_spawn.model` path.

## Best Possible Solution

Keep the issue open and land a focused core fix: stop telling the parent model to prefer aliases when an exact `provider/model` is specified, avoid presenting fallback or overlapping-provider aliases as interchangeable routing choices, and add regression coverage for `buildModelAliasLines`/system prompt rendering plus `sessions_spawn` model override preservation in a GitHub Copilot versus Bedrock overlap case.
