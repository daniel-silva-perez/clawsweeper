## Summary

Keep this PR open. Current main still does not implement the central behavior: gateway agent runs store a structured result, but agent.wait snapshots/responses and subagent wait normalization do not preserve or expose that result. The PR remains useful source work, and the author addressed the earlier frozen-result overwrite concern, but the branch still has blocking correctness gaps around normal `{ payloads, meta }` agent results and lifecycle-vs-dedupe races.

## What This Changes

The PR threads agent.wait result payloads through gateway dedupe/wait responses, uses result text as a fallback for subagent announce/completion delivery, and adds gateway plus subagent regression tests.

## Best Possible Solution

Port the useful parts of this PR onto current main by preserving structured agent results through the gateway wait contract, carrying that field through subagent wait normalization, extracting visible text from standard agent result envelopes, and using it only as a fallback when frozen chat-history output is absent. Cover both cached/live wait paths and lifecycle/dedupe ordering with focused tests, then document the result-bearing wait response as an intentional contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Parse agent result payload arrays:** `src/agents/subagent-announce.ts:344`
  - body: `extractAgentWaitResultText` only checks scalar-looking fields such as `result`, `output`, `content`, and `summary`, then delegates to text extraction on those values. A normal gateway agent result is `{ payloads, meta }`, so `result.payloads: [{ text: "..." }]` is ignored and the fallback still fails when chat history has not caught up.
  - confidence: 0.9
- **[P2] Wait for the dedupe result before returning:** `src/gateway/server-methods/agent.ts:772`
  - body: This response can still return without the final result when the lifecycle wait wins the race, because lifecycle snapshots do not carry `result` and the dedupe wait is aborted. Normal agent execution can emit lifecycle end before the gateway writes the dedupe payload that contains the command result, so subagent callers can still receive `result: undefined`.
  - confidence: 0.84
