## Summary

Keep open. Current main still has the underlying gap: gateway agent runs store a result payload, but agent.wait snapshots, responses, and the shared subagent wait layer do not carry that payload through. The PR is useful source work, but it is not merge-ready because it patches an older registry path and still misses normal result-envelope parsing plus a lifecycle/dedupe ordering case.

## What This Changes

The PR threads agent.wait result payloads through gateway dedupe/wait responses, uses wait-result text as a subagent announce/completion fallback when history is behind, and adds gateway plus subagent regression tests.

## Best Possible Solution

Land a current-main implementation that preserves structured gateway results through dedupe snapshots and every agent.wait response, carries the field through run-wait and the run manager, parses normal result envelopes and reply payloads, and only uses wait-result text when chat-history freezing has not already captured a reply.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Parse agent result payload envelopes:** `src/agents/subagent-announce.ts:345-347`
  - body: extractAgentWaitResultText only checks direct result/output/content/summary candidates. Normal gateway agent results are shaped like { payloads, meta }, and payload entries commonly use plain { text } rather than chat blocks with type: "text", so ordinary completions still produce no fallback text when history has not caught up.
  - confidence: 0.91
- **[P1] Port the fallback into the run manager:** `src/agents/subagent-registry.ts:1259-1264`
  - body: This adds the frozen-result fallback to the old inline registry wait path, but current main completes subagent waits through createSubagentRunManager and waitForAgentRun. Since that shared wait result still drops result, the real completion path after rebase will not receive or freeze the fallback text.
  - confidence: 0.9
- **[P2] Wait for the result-bearing dedupe snapshot:** `src/gateway/server-methods/agent.ts:772-775`
  - body: The live agent.wait response returns the first terminal snapshot and aborts the other waiter. If the lifecycle snapshot wins, it has no result, and normal agent execution can emit lifecycle end before the gateway writes the dedupe payload that carries the command result, so callers can still receive result: undefined.
  - confidence: 0.85
