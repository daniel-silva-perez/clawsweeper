## Summary

Keep open. Current main still stores agent command results but drops them from the terminal wait snapshot, the `agent.wait` response, and the shared run-wait helper; this PR addresses the right problem but needs rework because it targets an older registry path, misses normal `{ payloads, meta }` result envelopes, and can still return a lifecycle-only snapshot before the result-bearing dedupe entry is available.

## What This Changes

The PR threads `agent.wait` result payloads through gateway wait plumbing, adds subagent announce/registry fallback handling, and adds gateway plus subagent regression tests.

## Best Possible Solution

Land a current-main implementation that defines `result` in the `agent.wait` contract end to end, extracts reply text from structured result envelopes, and fills `frozenResultText` only as a fallback before completion delivery without provider-specific core behavior.

## Reproduction Assessment

Yes. The code path is reproducible by inspection: current main stores `payload.result`, then omits it from `AgentWaitTerminalSnapshot`, `agent.wait` responses, and `waitForAgentRun`; the original linked bug report also supplied logs showing the result in gateway output but `(no output)` in the parent session.

## Solution Assessment

No. The PR is directionally correct but not the best current-main fix because it patches the old registry wait path, does not parse normal `{ payloads, meta }` result envelopes, and leaves the live wait race that can return a result-less lifecycle snapshot.

  - body: `extractAgentWaitResultText` only checks direct scalar candidates and chat-style text blocks.   - confidence: 0.91

  - confidence: 0.9
- **[P2] Wait for the result-bearing dedupe snapshot:** `src/gateway/server-methods/agent.ts:772-775`
  - body: The live `agent.wait` path returns the first terminal snapshot and aborts the other waiter. If lifecycle wins first, it has no `result`, while the result-bearing dedupe entry is written afterward, so callers can still receive `result: undefined`.
  - confidence: 0.85
