## Summary

Keep open. Current main still drops `agent.wait` result payloads before subagent completion code can use them, and this PR is useful source work, but it needs changes before merge: the branch is behind the current run-manager split, misses normal `{ payloads, meta }` result envelopes, and can still return before the result-bearing dedupe snapshot is available. The earlier frozen-result overwrite concern appears addressed on the PR branch.

## What This Changes

The PR threads `agent.wait` result payloads through gateway dedupe/wait responses, uses result text as a subagent announce/completion fallback, and adds gateway plus subagent regression tests.

## Best Possible Solution

Port the useful behavior onto current main: preserve the structured agent command result in the gateway dedupe snapshot and `agent.wait` responses, ensure the live wait path does not discard the result when lifecycle finishes before dedupe, carry the result through `run-wait`, and use extracted text only as a fallback when chat-history freezing has not already captured a reply.
