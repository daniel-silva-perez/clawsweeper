## Summary

Keep this PR open. Current main still has the backend session-row metadata needed for room/channel/title/last-message/cost/lineage context, but the Control UI does not request the derived fields and does not render the richer Sessions or Chat context this PR targets. The provided review thread also contains concrete correctness feedback that should be addressed before landing, so this is still a useful implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open as the implementation candidate. The best path is to rebase it onto current main, align the Control UI `GatewaySessionRow` type with `src/gateway/session-utils.types.ts`, request `includeDerivedTitles` and `includeLastMessage`, render the session/chat context from existing gateway fields, address the unresolved review comments, drop unrelated formatting churn, and keep verification focused on the affected Control UI tests plus any targeted gateway contract tests needed for row metadata.
