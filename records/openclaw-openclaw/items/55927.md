## Summary

Keep this PR open. Current main still exposes the richer gateway session-row metadata in the backend, but the Control UI does not request the derived title/last-message fields, its local row type still lags the gateway contract, and the Sessions/Chat views still do not render the visible room/channel/title/cost/lineage context targeted by this PR.

## Best Possible Solution

Keep this PR open. The best path is to rebase it onto current main, align the Control UI session row type with the backend `GatewaySessionRow` contract, request `includeDerivedTitles` and `includeLastMessage`, render the existing gateway metadata in Sessions and Chat, address the review comments, drop unrelated formatting churn, and validate with focused Control UI tests plus any needed gateway row-contract tests.
