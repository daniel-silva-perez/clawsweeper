## Summary

Keep this PR open. Current main still hydrates Signal mention placeholders into @uuid/@phone text and then gates only through configured/derived text mention regexes, so a native Signal mention of the bot's own account identity can still miss when the regex is derived from the bot display name. The PR is a focused Signal plugin fix with relevant tests and a follow-up commit that addresses the review concern about forged/stale mention spans.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land or rework the focused Signal plugin fix after confirming the span-validated native mention detection, adding or adjusting docs for Signal native mention behavior, and deciding whether the metadata-only canDetectMention gap should be included here or tracked separately.
