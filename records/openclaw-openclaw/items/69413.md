## Summary

Keep this PR open. Current main still hydrates Signal native mention placeholders into @uuid/@phone text and gates `requireMention` only through text regexes, so the PR's bot-identity metadata detection is not implemented on main. The PR is a focused Signal plugin fix with a relevant second commit addressing the span-validation review concern, but it still needs normal maintainer review and likely a docs/scope decision before merge.

## What This Changes

The PR adds Signal native mention detection by comparing `dataMessage.mentions[].uuid/.number` against the bot account identity, ORs that into Signal group `wasMentioned`, and adds mention-gating tests for bot UUID/phone matches and non-bot mentions.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or rework the focused Signal plugin fix after confirming the span-validated native mention logic, updating `docs/channels/signal.md` for native mention behavior, and deciding whether the metadata-only `canDetectMention` case belongs in this PR or a follow-up.
