## Summary

Keep this PR open. Current main still does not implement the PR’s intended remote embedding batch HTTP timeout threading. The active implementation has moved from the PR’s old src/memory files into the memory-host SDK and provider plugins, and those current HTTP helpers still cannot accept or forward timeoutMs to fetchWithSsrFGuard. The PR remains useful context, but it needs a fresh port and review rather than cleanup closure.

## Best Possible Solution

Keep this PR open as useful but stale work. The best maintainer path is a fresh port that adds optional timeoutMs or deadline-budget support to packages/memory-host-sdk/src/host/{remote-http,post-json,batch-http,batch-upload}.ts, threads one remaining deadline through extensions/{openai,voyage,google}/embedding-batch.ts upload/create/status/error-read/download calls, budgets retry attempts, handles AbortError robustly without replaying already accepted remote batches, and adds focused regression tests for the SDK helpers and provider batch wait paths.
