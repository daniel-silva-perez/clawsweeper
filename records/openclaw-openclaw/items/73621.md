## Summary

Keep this issue open. Current main still keeps busy follow-up messages only in the active in-memory `chatQueue`, clears that queue on Control UI session changes, and only flushes queued sends from that same array. The provided GitHub context also shows open PR #73679 as an unmerged implementation candidate that links this issue for automatic closure, so the issue should not be closed before that PR is reviewed and merged or replaced.

## Best Possible Solution

Keep this issue open and review PR #73679 as the current implementation candidate. The best fix is to make queued Control UI messages session-scoped, preserve the original session queue when navigating away, restore it when returning, and add a focused regression test for switch-away/switch-back with a queued follow-up. Broader durable queue or ACK semantics can remain with #45952 and #51549.
