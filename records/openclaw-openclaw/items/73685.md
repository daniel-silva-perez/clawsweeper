## Summary

This issue should stay open. Current main still does not expose `openclaw/plugin-sdk/discord`, while npm `@openclaw/discord` latest is still `2026.3.13` and its published entry imports that missing subpath. The repo source has a newer Discord plugin entry that uses `channel-entry-contract`, but there is no release/publish evidence that users installing `@openclaw/discord` get that fixed package yet.

## Best Possible Solution

Keep this issue open until maintainers either publish a compatible `@openclaw/discord` package and verify `openclaw plugins install @openclaw/discord` resolves to it, or restore a temporary `openclaw/plugin-sdk/discord` compatibility export with a documented deprecation path and regression coverage for loading `@openclaw/discord@2026.3.13` against the current core.

## Evidence

- **Published Discord plugin still imports missing subpath:** The published `@openclaw/discord@2026.3.13` tarball entry imports both `OpenClawPluginApi` and `emptyPluginConfigSchema` from `openclaw/plugin-sdk/discord`, matching the reported failure.
  - command: `curl -Ls https://registry.npmjs.org/@openclaw/discord/-/discord-2026.3.13.tgz | tar -xOzf - package/index.ts | nl -ba | sed -n '1,40p'`
- **npm latest versions reproduce the mismatch:** npm reports `openclaw` latest as `2026.4.26`, but `@openclaw/discord` latest as `2026.3.13`; reinstalling the plugin would still select the older Discord package.
  - command: `npm view @openclaw/discord version dist-tags peerDependencies openclaw --json && npm view openclaw version dist-tags --json`
- **Current exports map has no Discord SDK subpath:** Current `package.json` exposes `./plugin-sdk`, `./plugin-sdk/core`, and many narrow generic subpaths, but no `./plugin-sdk/discord` export.
