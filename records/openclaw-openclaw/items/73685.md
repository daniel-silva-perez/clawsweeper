## Summary

Keep this issue open. Current main still lacks the `openclaw/plugin-sdk/discord` export, and open PR #73703 is an unmerged implementation candidate that references this issue with GitHub closing syntax.

## Best Possible Solution

Keep this issue open while maintainers review PR #73703. The best fix is either to land a narrow temporary `openclaw/plugin-sdk/discord` compatibility export with aligned SDK metadata and regression coverage, or publish a compatible `@openclaw/discord` package and verify `openclaw plugins install @openclaw/discord` resolves to it.
