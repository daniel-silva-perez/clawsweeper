## Summary

Keep open. This PR is member-authored and carries the protected `maintainer` label, so cleanup should not auto-close it. It also is not obsolete on current `main`: the boundary-aware embedded stream path still returns the transport stream directly, while resolved API key/auth storage and run-signal injection only happen on the provider-owned branch. The linked bug #73559 remains open and this PR is the implementation candidate for it.

## Best Possible Solution

Keep this PR open for maintainer review. The likely best path is to land a narrow fix in `src/agents/pi-embedded-runner/stream-resolution.ts` that wraps boundary-aware streams with the same resolved-key/auth-storage and run-signal handling as provider-owned streams, with regression tests for resolved bearer injection, auth-storage fallback, explicit signal precedence, and cache-boundary preservation. If maintainers prefer a different credential-propagation design, close this PR manually after updating or replacing the implementation candidate for #73559.
