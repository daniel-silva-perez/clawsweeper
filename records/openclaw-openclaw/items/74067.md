## Summary

Keep this PR open. Current main still has the compile-cache prune bug: OpenClaw enables Node's compile cache without an OpenClaw-specific subdirectory, but the postinstall prune helper removes `<base>/openclaw`. The PR directly addresses that mismatch by pruning Node-versioned cache directories under the configured and default compile-cache bases, with focused test updates. Security review found no workflow, dependency, lockfile, or downloaded-code changes; the main review points are deletion scope and per-entry error handling in the postinstall prune loop.
