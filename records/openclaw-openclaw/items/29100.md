## Summary

Keep this issue open. Current main still lacks a supported cost metadata overlay for Amazon Bedrock auto-discovered models: plugin discovery config, legacy config typing/schema, discovery cache key, and discovery conversion all omit costs, and discovered foundation/profile models still get zero-cost metadata. The prior implementation PR #31160 is closed unmerged, while #53286 remains a related open display-layer issue for aws-sdk-auth cost visibility.

## Best Possible Solution

Implement a backward-compatible Amazon Bedrock plugin pricing overlay, preferably under plugins.entries.amazon-bedrock.config.discovery.costs with legacy models.bedrockDiscovery.costs compatibility only where needed. Apply configured costs to both foundation model IDs and inference profile IDs, include normalized pricing in the discovery cache key, update manifest/schema/help/docs/tests, and coordinate with #53286 so configured Bedrock costs are displayed for aws-sdk-auth status and usage surfaces.

## Risks / Open Questions

- Adding only the discovery cost map will not fully satisfy the user-visible /status and /usage full behavior while those surfaces still hide costs for aws-sdk auth; coordinate with #53286 or fix both in one PR.
- The cost map is keyed by arbitrary model IDs, so config migration/normalization should preserve prototype-pollution protections already present in the Bedrock compat migration helper.
- Pricing must be included in the Bedrock discovery cache key in deterministic order, otherwise cached zero-cost or old-cost model definitions can be reused after config changes.
- Open PR #73959 touches adjacent Bedrock discovery config/cache behavior for per-model context-window overrides, so a repair PR may need a small rebase or schema merge if that lands first.
