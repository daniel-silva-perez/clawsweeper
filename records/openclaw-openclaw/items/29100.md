## Summary

Keep this issue open. Current main still has no supported cost metadata overlay for Amazon Bedrock auto-discovered models: the plugin discovery schema, shared Bedrock discovery type, strict legacy config schema, discovery implementation, cache key, tests, and docs all omit a costs map. The direct implementation PR #31160 is closed and unmerged, and #53286 remains a related open display-layer issue for showing configured costs under aws-sdk auth.

## Best Possible Solution

Keep this open and implement a backward-compatible pricing overlay in the Amazon Bedrock plugin, preferably under plugins.entries.amazon-bedrock.config.discovery.costs with legacy models.bedrockDiscovery compatibility only where needed. Apply configured costs to foundation model IDs and inference profile IDs, include normalized pricing in the discovery cache key, update manifest/schema/help/docs/tests, and coordinate with #53286 so configured Bedrock costs are actually shown for aws-sdk-auth usage surfaces.

## Clownfish Work Prompt

Implement cost metadata support for Amazon Bedrock auto-discovered models. Current main discovers Bedrock foundation models and inference profiles but assigns zero DEFAULT_COST and has no config overlay. Add a backward-compatible `costs` map under `plugins.entries.amazon-bedrock.config.discovery.costs`, keyed by foundation model ID and inference profile ID. Keep the surface plugin-owned; only support legacy `models.bedrockDiscovery` migration/typing where the existing compatibility path requires it, and do not introduce a broad top-level `models.costs` surface in this PR.

Expected behavior: discovery should resolve full or partial cost entries with missing fields defaulting to zero, prefer an exact inference profile ID cost when present, otherwise allow the profile to inherit the backing foundation model cost, and leave unspecified models at zero cost. Include a deterministic normalized costs representation in the Bedrock discovery cache key so different pricing configs do not share cached model definitions. Update `extensions/amazon-bedrock/openclaw.plugin.json` schema and UI hints, `src/config/types.models.ts`, `src/config/zod-schema.core.ts` if legacy validation needs it, Bedrock discovery tests, config migration tests as needed, and docs. Also coordinate #53286 by allowing configured costs to display for aws-sdk-auth Bedrock usage/status surfaces when a model cost config exists, while keeping OAuth/token subscription-style cost hiding intact. No AWS live credentials should be required; use mocked discovery clients.

Changelog: add an entry only if repo policy treats this user-facing provider config enhancement as changelog-worthy, with proper reporter attribution; otherwise document in the PR why docs/tests are sufficient. Do not change dependency versions, Bedrock request routing, auth resolution, or provider selection behavior beyond cost metadata and display.
