## Summary

Keep this issue open. Current main still does not provide a supported cost metadata overlay for auto-discovered Amazon Bedrock models: the plugin discovery config and legacy/core schemas omit any `costs` map, discovered foundation models still receive all-zero `DEFAULT_COST`, inference profiles only inherit that discovered/default cost, and the docs still show pricing only on explicit provider model entries. The direct implementation PR #31160 is closed and unmerged, while #53286 remains open for the related aws-sdk cost-display path.

## Best Possible Solution

Keep this issue open and implement a backward-compatible Bedrock pricing overlay in the Amazon Bedrock plugin, preferably under `plugins.entries.amazon-bedrock.config.discovery.costs` with legacy `models.bedrockDiscovery` compatibility only as needed. The fix should apply costs to foundation model IDs and inference profile IDs, include deterministic cache-key handling, update manifest/schema/help/docs/tests, and coordinate with #53286 so aws-sdk-auth Bedrock costs are actually visible in `/status` and `/usage full` when configured.
