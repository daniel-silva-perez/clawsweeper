## Summary

Keep open. Current main has a partial bundled plugin load-path cleanup, but it only handles paths that alias the active package root. The reported failure is a removed pnpm/npm versioned package path such as `openclaw@2026.3.28.../dist/extensions/acpx`, and I do not see current code or tests that repair that cross-version absolute path before config validation reports `plugins.load.paths: plugin path not found`.

## Best Possible Solution

Add a targeted repair path for stale packaged bundled-plugin paths from previous OpenClaw package roots. `openclaw doctor --fix` should recognize versioned npm/pnpm global paths ending in `dist/extensions/<bundled-plugin-dir>` or `dist-runtime/extensions/<bundled-plugin-dir>`, remove the redundant `plugins.load.paths` entry, and clean or refresh matching bundled install records. Gateway startup/config validation could also downgrade those recognizable bundled aliases to warnings so one obsolete path does not block startup. Add regression coverage with a pnpm-style `.pnpm/openclaw@2026.3.28.../node_modules/openclaw/dist/extensions/acpx` path against a newer current bundled root.
