## Summary

Keep open. Current main still has the reported timestamp-refresh path, and this PR is a focused implementation that follows the existing preserve-activity pattern; it needs normal maintainer/author follow-up for low-severity test portability, changelog coverage, and validation before merge.

## What This Changes

The branch adds a preserve-activity option to session-store entry updates, applies it to compaction and memory-flush metadata writes, removes compaction bookkeeping updates to `updatedAt`, and adds regression coverage for legacy daily/idle reset behavior.

## Best Possible Solution

Land a narrow version of this fix after replacing the hard-coded test temp path with `os.tmpdir()`, adding the required Unreleased changelog entry, and validating the touched session-store, compaction, and memory-flush paths. The reset policy should stay unchanged; metadata-only writers should opt into preserve-activity explicitly.

## Reproduction Assessment

Yes. The linked bug gives concrete legacy daily and idle scenarios, and current main source confirms the path: compaction/flush metadata can advance `updatedAt` while reset freshness falls back to `updatedAt` when explicit lifecycle timestamps are missing.

## Solution Assessment

Yes, with small cleanup. Reusing the existing preserve-activity merge policy only for metadata-only compaction and memory-flush writes is the narrowest maintainable fix; the safer merge path is to address the test temp root and changelog entry before validation.
