## Summary

Keep open. Current main still resolves exec approvals file and socket paths through home-backed defaults, and the local CLI, gateway RPC handlers, runtime helpers, tests, and public docs all continue to inherit or describe that behavior. The focused repair is already represented by open PR #74002, so this should stay open until that PR or a maintainer-approved equivalent lands.

## What This Changes

Exec approvals should honor the active `OPENCLAW_STATE_DIR` or profile state root instead of always storing `exec-approvals.*` under home-backed `.openclaw` paths.

## Best Possible Solution

Land #74002 or a maintainer-approved equivalent that derives the approvals JSON path, socket path, and effective-policy host reporting from the active state-dir/profile contract, preserves the legacy fallback when no override is active, and updates focused runtime/CLI tests plus public approvals docs.

## Security Review

Status: needs_attention

Summary: The issue is security-sensitive because exec approval state can still resolve to the wrong root, causing intended allowlists or prompt policy to be bypassed in relocated/profile deployments.
