## Summary

Keep open. Current main still resolves exec approvals file and socket defaults through home-backed constants instead of the active state-dir/profile contract, and the focused implementation candidate remains open rather than merged.

## What This Changes

The issue requests that exec approval file and socket storage follow the active OpenClaw state root instead of silently using home-backed storage.

## Best Possible Solution

Land #74002 or a maintainer-approved equivalent that derives the approvals JSON path, default socket path, and effective-policy default host path from the active state-dir/profile contract, preserves unset legacy fallback behavior, and updates focused CLI/gateway/runtime tests plus public approvals docs.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: with `OPENCLAW_STATE_DIR` set, current main still expands built-in home-backed approvals file/socket constants and routes CLI/gateway reads and writes through that resolver; I did not run a live write repro because this review is read-only.

## Solution Assessment

Yes. A narrow resolver-level fix that reuses the existing state-dir contract is the maintainable direction, but the implementation should explicitly handle legacy approvals-state migration or fail-closed concerns before landing.
