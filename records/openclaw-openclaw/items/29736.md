## Summary

Keep this issue open. Current main still resolves exec approvals file, socket, effective-policy display, CLI access, and gateway read/write paths through home-backed defaults, while the focused implementation candidate remains open in #74002 and has not landed.

## What This Changes

Exec approvals should store and read the approvals JSON/socket under the active OpenClaw state-dir/profile root instead of defaulting to `~/.openclaw`.

## Best Possible Solution

Land #74002 or a maintainer-chosen equivalent that derives the approvals JSON path, default socket path, and effective-policy default host path from the active state-dir/profile contract, preserves the unset legacy fallback, and updates focused CLI/gateway/runtime tests plus public approvals docs.
