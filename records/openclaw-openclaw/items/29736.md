## Summary

Keep open. Current main still resolves exec approvals file/socket defaults through home-backed constants instead of the active state-dir contract, and the focused implementation candidate remains open rather than merged.

## What This Changes

Exec approvals should read and write their file/socket under the active state dir or profile root instead of the default home state directory.

## Best Possible Solution

Land #74002 or a maintainer-approved equivalent that derives the approvals JSON path, default socket path, and effective-policy host source from the active state-dir/profile contract, preserves existing fallback behavior when no explicit state root is configured, and updates focused CLI/gateway/runtime tests plus public approvals docs.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: with `OPENCLAW_STATE_DIR` set, current main still expands the built-in home-backed approvals file/socket constants and routes CLI/gateway reads and writes through that resolver.

## Solution Assessment

Yes. A narrow resolver-level fix that reuses the existing state-dir contract is the maintainable path; it should avoid broad exec policy changes and keep legacy fallback behavior intact.
