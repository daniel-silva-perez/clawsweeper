## Summary

Keep open. Current main still resolves exec approvals defaults through home-expanded `~/.openclaw` paths instead of the active `OPENCLAW_STATE_DIR` or profile-derived state root, and the same helpers are still used by gateway approvals and node-host runtime enforcement. There is now an open implementation candidate, #74002, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and route it to the active fix lane. The best fix is for exec approvals file and socket defaults to derive from the active state root/profile semantics, likely via `resolveStateDir()`, while preserving a safe legacy fallback or migration from the existing home-backed approvals file. Review and land #74002 or an equivalent canonical PR, then update tests/docs and close this issue as implemented after merge.
