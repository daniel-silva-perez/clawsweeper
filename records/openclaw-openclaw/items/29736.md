## Summary

Keep this issue open. Current main still resolves exec approvals file and socket defaults through home-expanded `~/.openclaw` paths instead of the active `OPENCLAW_STATE_DIR` or profile-derived state root. The same resolver feeds local CLI storage, gateway approvals RPCs, node-host approvals RPCs, and `system.run` enforcement. Open PR #74002 is the focused implementation candidate, so the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and route it to #74002 or an equivalent canonical PR. The final fix should derive exec approvals file/socket defaults from the active state root/profile semantics, preserve a safe legacy fallback or migration path for existing home-backed approval files, update docs and focused regression tests, and validate local CLI, gateway, node-host, and `system.run` enforcement surfaces before closing as implemented.
