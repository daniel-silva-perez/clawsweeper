## Summary

Keep open. Current main still resolves exec approvals file and socket defaults through home-backed `~/.openclaw` constants instead of the active state-dir/profile contract, the latest release contains the same resolver, and the focused fix candidate in #74002 remains open and unmerged.

## What This Changes

The issue asks for exec approvals file and socket storage to follow the active OpenClaw state root or profile instead of falling back to home-backed `~/.openclaw` paths.

## Best Possible Solution

Land #74002 or a maintainer-approved equivalent that derives the approvals JSON path, default socket path, and effective-policy default host path from the active state-dir/profile contract, preserves unset legacy fallback behavior, and updates focused CLI/gateway/runtime tests plus public approvals docs.

## Reproduction Assessment

Yes. A high-confidence source-level repro is to set `OPENCLAW_STATE_DIR` to a non-default root and observe that the current resolver still expands `~/.openclaw/exec-approvals.*`; the issue comments also include CLI, profile, and container repros.

## Solution Assessment

Yes. The narrow maintainable fix is at the resolver/effective-policy layer using the existing state-dir contract, with focused tests and docs; creating a second automated PR would be worse while #74002 is already open.
