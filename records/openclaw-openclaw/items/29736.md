## Summary

Keep open. Current main still resolves exec approvals file/socket defaults through home-backed paths instead of the active state dir, and the latest release tag shows the same behavior. The discussion has multiple independent confirmations, including profile and container deployments; one later “seems fixed” comment is not enough to outweigh the current source evidence and the still-open focused fix PR.

## What This Changes

Request exec approvals JSON, socket, and effective-policy path resolution to honor the active OpenClaw state/profile root instead of always using `~/.openclaw` defaults.

## Best Possible Solution

Land #74002 or a maintainer-approved equivalent that derives the approvals JSON path, default socket path, and effective-policy host source from the active state-dir/profile contract, preserves existing default installs, and updates focused CLI/gateway/runtime tests plus public approvals docs.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: with `OPENCLAW_STATE_DIR` set, the current resolver still expands `~/.openclaw/exec-approvals.*`, and the read/write/gateway helpers route through that resolver.

## Solution Assessment

Yes. A narrow resolver-level fix that reuses the existing state-dir contract is the maintainable path; it should avoid broader policy semantics changes and keep the legacy home-backed fallback when no explicit state root/profile applies.
