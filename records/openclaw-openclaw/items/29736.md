## Summary

Keep open. Current main still resolves exec approvals storage and socket paths through `~/.openclaw` defaults instead of the active `OPENCLAW_STATE_DIR`/profile state root, and the latest release tag still contains the same resolver. The focused implementation candidate is open in #74002, so this should remain open until that PR or a maintainer-approved equivalent lands.

## What This Changes

Exec approvals file and socket resolution should follow the active OpenClaw state/profile root instead of always using `~/.openclaw`.

## Best Possible Solution

Land #74002 or a maintainer-approved equivalent that derives the approvals JSON path, default socket path, and effective-policy host path from the active state-dir/profile contract, preserves the legacy fallback when no override is active, and updates focused runtime, CLI, gateway, and docs coverage.
