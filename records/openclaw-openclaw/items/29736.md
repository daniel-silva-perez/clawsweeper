## Summary

Keep this issue open. Current main still resolves exec approvals file and socket defaults through home-expanded `~/.openclaw` paths instead of the active `OPENCLAW_STATE_DIR` or profile-derived state root. The gateway, node-host management, and runtime enforcement paths still use those helpers, and current docs/tests still describe the home-backed approvals store.

## Best Possible Solution

Keep this issue open and fix the core exec approvals path contract. The default file and socket should derive from the active state root/profile semantics, likely via `resolveStateDir()`, while preserving a safe legacy fallback or migration from the existing home-backed approvals file. Update CLI/gateway/node-host tests and docs to cover `OPENCLAW_STATE_DIR`, `--profile`, `OPENCLAW_HOME` compatibility, read-only home/container behavior, and symlink safety.
