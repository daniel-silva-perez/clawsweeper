## Summary

Keep open. Current main still resolves exec approvals file and socket defaults through home-backed constants instead of the active state-dir contract, and the latest release tag has the same behavior; the issue is already paired with open focused PR #74002, so it should stay open for maintainer review/landing rather than be closed or queued again.

## What This Changes

Exec approvals should store and read their JSON file and socket under the active OpenClaw state root/profile instead of always using the home-backed default state directory.

## Best Possible Solution

Land #74002 or a maintainer-approved equivalent that derives the approvals JSON path, default socket path, and effective-policy host source from the active state-dir/profile contract, preserves existing default installs, and updates focused CLI/gateway/runtime tests plus public approvals docs.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: with `OPENCLAW_STATE_DIR` set, current main still expands the built-in home-backed approvals file/socket constants, and the CLI/gateway helpers route reads and writes through that resolver.

## Solution Assessment

Yes. A narrow resolver-level fix that reuses the existing state-dir contract is the maintainable path; it should avoid broad exec policy changes and preserve the legacy fallback when no explicit state root/profile applies.
