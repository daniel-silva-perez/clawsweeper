## Summary

Keep this issue open. Current main still resolves exec approvals file/socket storage and default effective-policy host reporting through home-backed defaults, while the focused implementation candidate in #74002 remains open and unmerged.

## What This Changes

The issue asks for exec approvals file, socket, and related CLI/gateway behavior to honor the active OpenClaw state root or profile instead of always using `~/.openclaw`.

## Best Possible Solution

Land #74002 or a maintainer-chosen equivalent that derives the approvals JSON path, default socket path, and effective-policy default host path from the active state-dir/profile contract, preserves the legacy fallback when no override is active, and updates focused CLI/gateway/runtime tests plus the public approvals docs.
