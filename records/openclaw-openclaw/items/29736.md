## Summary

Keep this issue open. Current main still resolves exec approval storage, socket defaults, effective-policy display, gateway/node management, and runtime enforcement through home-backed `~/.openclaw` paths instead of the active state directory, and open PR #74002 is already the focused implementation candidate.

## What This Changes

The issue requests that exec approvals storage and socket paths honor active `OPENCLAW_STATE_DIR`/profile state-root semantics instead of defaulting to `~/.openclaw`.

## Best Possible Solution

Land #74002 or a maintainer-chosen equivalent that derives exec approvals JSON, default socket, and display paths from active state-dir/profile semantics, preserves the unset home-backed fallback, updates focused docs/tests, and validates local CLI, gateway, node-host, and `system.run` enforcement before closing this as implemented.
