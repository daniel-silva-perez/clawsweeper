## Summary

Keep open. Current main still resolves exec approvals storage and default effective-policy host reporting through home-backed defaults, and the focused implementation candidate remains open in #74002 rather than merged into main.

## What This Changes

The issue asks exec approvals file/socket paths and related policy reporting to follow the active OpenClaw state root/profile semantics instead of falling back to `~/.openclaw`.

## Best Possible Solution

Land #74002 or an equivalent maintainer-approved patch that derives the approvals JSON path, default socket path, and effective-policy default host path from the active state-dir/profile contract, preserves legacy fallback behavior when no override is active, and updates focused runtime/CLI tests plus the public approvals docs.
