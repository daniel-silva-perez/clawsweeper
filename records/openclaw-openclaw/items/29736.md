Keep this issue open. Current main still resolves the exec approvals file, socket, CLI/gateway reads and writes, and default effective-policy host-source display through home-backed defaults, and the focused fix is still open in #74002 rather than merged.

The issue requests that exec approvals storage follow the active OpenClaw state root/profile instead of silently using the default home-backed state path.
