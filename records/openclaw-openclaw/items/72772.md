## Summary

Keep open. Current main still lacks an ultrawide-aware Control UI layout policy for the shared shell, Config form surface, and Sessions/data-table surface; the related chat-width issue and PR cover only chat readability and do not supersede this broader forms/tables request.

## What This Changes

Request ultrawide-aware Control UI layouts that cap or restructure wide forms and tables instead of stretching primary content across the full viewport.

## Best Possible Solution

Define a shared Control UI responsive layout policy and apply it per surface: cap readable form/detail content, keep action controls visually attached to their sections, and handle wide tables with constrained containers, side filters/detail panels, or density-specific layouts instead of stretching every view by default.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to open the Control UI Config or Sessions view at a viewport wider than 2560px; current source shows the shell, Config layout, and shared data table still consume the available content width.

## Solution Assessment

Unclear. The requested direction is sound, but the narrowest maintainable fix is likely shared layout primitives plus targeted Config/Sessions/table changes, not a blanket max-width applied to every Control UI view.
