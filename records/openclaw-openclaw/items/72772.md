## Summary

Keep open. Current main still lacks a shared ultrawide layout policy for the Control UI shell, Config form surface, and Sessions table; the linked chat-width items address only chat readability and do not supersede this broader forms/tables request.

## What This Changes

Request ultrawide-aware Control UI layouts that cap or repurpose wide content areas so forms, action buttons, and tables remain visually scannable on 21:9 and 32:9 displays.

## Best Possible Solution

Define a shared Control UI responsive layout policy, then apply it per view: cap readable form/detail content, keep form actions visually attached to their sections, and handle wide data tables with constrained containers, useful side panels, or density-specific layouts rather than stretching every surface by default.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path is to open the Control UI Config or Sessions view at a viewport wider than 2560px; current source shows the shell, Config layout, and Sessions table still consume available width.

## Solution Assessment

Unclear. The requested direction is valid, but the maintainable fix is likely a shared layout pattern plus targeted Config, Sessions/data-table, Usage, Chat, and Quick Settings changes rather than a single CSS max-width rule.
