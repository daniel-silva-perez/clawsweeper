## Summary

Keep open. Current main still has no shared ultrawide layout policy for the Control UI shell, Config form surface, or Sessions data table; the linked chat-width issue and PR cover only chat readability and do not supersede this broader forms/tables request.

## What This Changes

Adopt ultrawide-aware Control UI layouts that cap or reflow forms and tables instead of stretching them across very wide viewports.

## Best Possible Solution

Define a shared Control UI responsive layout policy: cap readable form/detail content, keep form actions visually attached to their sections, and handle wide tables with constrained containers, useful side panels, or density-specific layouts instead of stretching every view by default.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to open the Control UI Config or Sessions view at a viewport wider than 2560px; current source shows the shell, Config layout, and Sessions table still consume the available content width.

## Solution Assessment

Unclear. The requested direction is valid, but the best maintainable fix is likely a shared layout pattern plus view-specific changes for Config, Sessions/data tables, chat, Quick Settings, and Usage rather than a single global max-width.
