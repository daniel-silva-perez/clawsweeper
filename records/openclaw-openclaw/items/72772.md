## Summary

Keep open. Current main still leaves the shared Control UI shell, Config form surface, and Sessions data table expanding across the available content column, while the related open chat-width issue and PR only cover chat readability and do not supersede the broader forms/tables ultrawide request.

## What This Changes

Issue requests ultrawide-aware Control UI layout patterns for forms and tables instead of stretching content to the full viewport.

## Best Possible Solution

Define a Control UI-wide responsive layout policy: cap readable form/detail content, keep form actions visually tied to their sections, and handle data tables intentionally with constrained containers, useful side panels, or view-specific density patterns rather than full-viewport stretching by default.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to open the Control UI Config or Sessions view at a viewport wider than 2560px; source inspection shows the shared shell, Config layout, and Sessions table still consume the available content width.

## Solution Assessment

Unclear. The requested direction is valid, but the best maintainable fix is not a blanket max-width; it should use shared layout tokens/patterns plus view-specific decisions for Config, Sessions/data tables, chat, and Quick Settings.
