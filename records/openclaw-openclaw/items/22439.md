## Summary

Keep open. Current main still lacks the requested configurable bootstrap tier behavior, and this is an active contributor PR paired with the open proposal #22438. The PR should not be closed as cleanup, but it still needs maintainer product/API review and a concrete fix for the tier-boundary bug before merge.

## What This Changes

The PR adds `agents.defaults.bootstrapTier`, tier resolution and filtering for workspace bootstrap files, extra-bootstrap hook wiring, generated config metadata/help, and focused tier tests.

## Best Possible Solution

Keep this PR and #22438 open for maintainer product/API review. If accepted, land a tier contract that can reliably distinguish root bootstrap files from hook-loaded extras, aligns generated config metadata/help/docs, and includes hook-level tests with real recognized-basename extras such as `packages/*/AGENTS.md`; if maintainers prefer an existing or alternate context-control model, close it manually after that decision.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Distinguish extra bootstrap files from root files:** `src/agents/workspace.ts:731-733`
  - body: The `standard` branch filters only on `file.name`, but hook-loaded extras are also represented by recognized basenames such as `AGENTS.md` and `TOOLS.md`. A configured extra like `packages/core/AGENTS.md` still passes the standard allowlist, so `standard` and `full` are not distinct in the real hook path and standard can inject extra prompt context contrary to the tier contract. Track extra-file provenance or compare root-relative paths, and add a hook-level test with a real loadable extra file.
  - confidence: 0.9
- **[P3] Align the minimal tier type comment with the code:** `src/config/types.agent-defaults.ts:233-235`
  - body: The public type comment says `minimal` is `AGENTS.md + TOOLS.md only`, but the PR code and generated help include `SOUL.md`, `IDENTITY.md`, and `USER.md` too. This will mislead readers of the generated TypeScript config surface; update the comment to match the actual minimal allowlist.
  - confidence: 0.83
