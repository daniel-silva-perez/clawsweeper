## Summary

Keep this PR open. Current main still has only binary bootstrap filtering, and the contributor branch is active and paired with an open proposal, but the latest head still has blocking prompt-context regressions around legacy `bootstrap-extra-files` behavior and explicit minimal-tier filtering.

## What This Changes

The PR adds `agents.defaults.bootstrapTier`, tier resolution/filtering for workspace bootstrap files, `bootstrap-extra-files` tier wiring, generated config metadata/help/types, and focused tests.

## Best Possible Solution

Keep this PR open with #22438 for maintainer API review. If accepted, revise the branch so no configured tier preserves current hook behavior, explicit minimal/standard have source-aware semantics, and the docs/generated schema/tests describe the same contract while coordinating with #74735.

## Reproduction Assessment

Yes. Static inspection is enough for the two review findings: current main keeps main-session hook extras, while PR head line 754 drops hook files under default `standard`, and PR head line 746 admits hook-sourced allowlisted files under `minimal`.

## Solution Assessment

No. Tiered bootstrap control may be a useful product direction, but this patch is not yet the best implementation because it conflates the unset legacy path with explicit `standard` and leaves explicit `minimal` source-blind.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve legacy hook extras without a tier:** `src/agents/workspace.ts:754`
  - body: The PR says no configured tier keeps existing behavior, but the hook passes `undefined`, `resolveBootstrapTier` maps that to `standard`, and this predicate excludes every `source: "hook"` file. Existing users with `bootstrap-extra-files` enabled stop receiving main-session extras unless they discover `bootstrapTier: "full"`; keep an unset legacy path separate from explicit `standard`, or document an intentional migration.
  - confidence: 0.9
- **[P2] Filter hook extras out of explicit minimal:** `src/agents/workspace.ts:746`
  - body: The `minimal` branch filters only by basename, so `bootstrapTier: "minimal"` can still inject hook-loaded `AGENTS.md`, `SOUL.md`, `IDENTITY.md`, `TOOLS.md`, or `USER.md`. Since `full` is the tier described as including extra patterns, explicit minimal should be source-aware while any desired no-override legacy behavior is handled separately.
  - confidence: 0.86
