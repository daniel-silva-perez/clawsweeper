## Summary

Keep open. Current main does not implement tiered bootstrap loading, and this active contributor PR is paired with an open proposal, but PR head still has prompt-context semantic regressions that should block merge until maintainers review the public API direction.

## What This Changes

The PR adds `agents.defaults.bootstrapTier`, tier-aware workspace bootstrap filtering, bootstrap-extra-files tier wiring, generated config metadata/help/types, and focused tests.

## Best Possible Solution

Keep this PR open with #22438 for maintainer API review. If accepted, revise the branch so unset config preserves current hook behavior, explicit tiers have source-aware semantics, and docs/generated schema/tests all describe the same contract while coordinating with #74735.

## Reproduction Assessment

Yes. Static inspection is enough for the review findings: current main keeps main-session hook extras, while PR head drops hook files under default `standard`, and PR head admits hook-sourced allowlisted files under `minimal`.

## Solution Assessment

No. Tiered bootstrap control may be useful, but this patch is not yet the best implementation because it conflates unset legacy behavior with explicit `standard` and leaves explicit `minimal` source-blind.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve legacy hook extras without a tier:** `src/agents/workspace.ts:753-754`
  - body: The PR says an unset tier is backward compatible, but the hook passes `undefined`, `resolveBootstrapTier` maps that to `standard`, and this predicate excludes every `source: "hook"` file. Existing main sessions using `bootstrap-extra-files` would stop receiving configured extras unless users discover `bootstrapTier: "full"`; keep the unset legacy path separate from explicit standard or document an intentional migration.
  - confidence: 0.9
- **[P2] Filter hook extras out of explicit minimal:** `src/agents/workspace.ts:745-746`
  - body: The `minimal` branch filters only by basename, so `bootstrapTier: "minimal"` can still inject hook-loaded `AGENTS.md`, `SOUL.md`, `IDENTITY.md`, `TOOLS.md`, or `USER.md`. Since the new help text describes `full` as the tier that includes extra patterns, explicit minimal should be source-aware while any no-override legacy behavior is handled separately.
  - confidence: 0.86
