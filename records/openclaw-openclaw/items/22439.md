## Summary

Keep this PR open, but the latest head still has merge-blocking correctness issues around the new tier contract. Current main does not implement `bootstrapTier`, and the PR is an active contributor branch paired with an open proposal, so this is not cleanup-closeable; the remaining work should happen on the PR branch with maintainer API review.

## What This Changes

The PR adds `agents.defaults.bootstrapTier`, tier resolution and filtering for workspace bootstrap files, hook wiring for `bootstrap-extra-files`, generated config metadata/help/types, and focused tests.

## Best Possible Solution

Keep the PR open with #22438 for maintainer API review. If accepted, revise the branch so explicit `minimal` and `standard` have a source-aware extra-file contract, no-config behavior preserves existing `bootstrap-extra-files` users unless maintainers explicitly choose a migration, and docs/generated schema/tests all describe the same behavior.

## Reproduction Assessment

Yes. The remaining issues are reproducible by static inspection of PR head: the minimal branch ignores hook provenance, and the no-config main path resolves to `standard` before filtering out hook-sourced extras.

## Solution Assessment

No. Tiered bootstrap control may be a reasonable direction, but this patch is not yet the best implementation because it conflates explicit tier selection with legacy no-override behavior and leaves the minimal prompt-exposure boundary source-blind.
