## Summary

Keep this PR open. Current main still has no `bootstrapTier` config or tier-aware bootstrap filtering, and the PR is paired with open proposal #22438 plus an explicit maintainer review request. The patch is not ready as-is because the intended `standard` versus `full` split does not work for real `bootstrap-extra-files` inputs: the loader only admits recognized bootstrap basenames, and the new standard filter allows those same basenames.

## What This Changes

The PR branch adds `agents.defaults.bootstrapTier`, tier resolution/filtering for workspace bootstrap files, extra-bootstrap hook wiring, generated config metadata/help, and focused tier tests.

## Best Possible Solution

Keep the PR open with #22438 for maintainer product/API review. If maintainers accept this API, fix the real extra-file tier distinction, refresh the branch against current main, and land the source, hook, schema/types, generated metadata/help/docs, and tests together; otherwise close it manually after the product decision.
