## Summary

Keep this PR open. Current main still does not implement a configurable bootstrap tier, the PR is paired with the open product proposal #22438, and prior maintainer-side review explicitly asked for human product/API review before merge. The PR also still has a concrete tier-boundary bug around hook-loaded extra bootstrap files, so it is not ready to merge as-is.

## What This Changes

The PR adds a schema-backed `agents.defaults.bootstrapTier` option, tier resolution and filtering for workspace bootstrap files, extra-bootstrap hook tier wiring, generated config metadata/help, and focused tier tests.

## Best Possible Solution

Land a maintainer-approved tier contract only after the implementation can reliably distinguish root bootstrap files from hook-loaded extra files, keeps generated schema/help/docs aligned, and has hook-level tests using real recognized-basename extras such as `packages/*/AGENTS.md`.

Full review comments:

  - body: The `standard` branch filters only on `file.name`, but hook-loaded extras are also represented by recognized basenames such as `AGENTS.md` and `TOOLS.md`. Track extra-file provenance or compare root-relative paths, and add a hook-level test with a real loadable extra file.
