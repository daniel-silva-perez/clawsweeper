## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Distinguish extra bootstrap files from root files:** `src/agents/workspace.ts:733`
  - body: The `standard` branch filters only on `file.name`, but hook-loaded extras are also represented by recognized basenames such as `AGENTS.md` and `TOOLS.md`. A configured extra like `packages/core/AGENTS.md` still passes this allowlist, so `standard` can inject extra prompt context and is not reliably distinct from `full` on the real hook path. Track extra-file provenance or compare root-relative paths, and add a hook-level test with a real loadable extra file.
  - confidence: 0.9
- **[P3] Align the minimal tier type comment with the code:** `src/config/types.agent-defaults.ts:233-235`
  - body: The public type comment says `minimal` is `AGENTS.md + TOOLS.md only`, but the PR code and generated help include `SOUL.md`, `IDENTITY.md`, and `USER.md` too. This will mislead readers of the generated TypeScript config surface; update the comment to match the actual minimal allowlist.
  - confidence: 0.83
