## Summary

Keep open. Current main does not implement tiered bootstrap loading, the PR is active and paired with the open proposal #22438, and maintainer discussion already calls for product/API review. The current branch still has two merge-blocking bootstrap filtering issues around hook-loaded prompt files and legacy hook compatibility.

## What This Changes

The PR branch adds `agents.defaults.bootstrapTier`, tier-aware workspace bootstrap filtering with root/hook provenance, hook wiring, generated config metadata/help/types, and focused tier tests.

## Best Possible Solution

Keep this PR open with #22438 for maintainer product/API review. If accepted, revise the branch so explicit `minimal` and `standard` do not admit hook-loaded extras, legacy no-override behavior remains compatible or has an explicit migration decision, and the source, hook, config metadata, docs, and tests all describe the same tier contract.

## Reproduction Assessment

Yes. The two blocking issues are reproducible by static inspection of PR head: `filterBootstrapFilesForSession` filters minimal by basename only, while no tier override resolves main sessions to `standard` and then excludes `source: "hook"` files.

## Solution Assessment

No. The tiered API may be a reasonable direction, but the current patch is not the best implementation because it conflates legacy no-override behavior with explicit `standard` and leaves minimal sessions source-blind for hook-loaded prompt files.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Exclude hook extras from minimal tier:** `src/agents/workspace.ts:746`
  - body: The minimal branch still filters only by basename. Because hook-loaded files are tagged `source: "hook"`, configured extras such as `packages/*/SOUL.md` or `packages/*/AGENTS.md` still enter default subagent/cron sessions whenever their basename is allowlisted, even though `full` is the tier meant to opt into hook extras. Add source-aware filtering for minimal and cover it through the hook path.
  - confidence: 0.87
- **[P2] Preserve legacy extras without a tier override:** `src/agents/workspace.ts:754`
  - body: With no `bootstrapTier` configured, main sessions resolve to `standard`, and `standard` now drops `source: "hook"` files. Existing users who enabled `bootstrap-extra-files` currently receive those extras in main sessions; after this change they silently disappear unless they discover `bootstrapTier: "full"`, which contradicts the backward-compatibility claim. Distinguish an explicit `standard` override from legacy no-override behavior or document a deliberate migration.
  - confidence: 0.82
