Keep this open and implement a narrow, backward-compatible state fix that distinguishes legacy/no-flush memoryFlushCompactionCount: 0 rows from recorded cycle-0 flushes, likely using memoryFlushAt or an explicit sentinel, while preserving the once-per-compaction-cycle guard.
