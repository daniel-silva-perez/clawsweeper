## Summary

Keep open: current main still treats the reported persisted 0/0 counter state as already flushed, and the two closed fix attempts did not land because they would break the legitimate cycle-0 duplicate-flush guard. The remaining fix is narrow but must distinguish legacy/no-flush rows from real cycle-0 flush records.

## What This Changes

The issue asks OpenClaw to let memory flush run for never-flushed sessions whose `compactionCount` and `memoryFlushCompactionCount` are both zero.

## Best Possible Solution

Define a backward-compatible session-state contract that distinguishes never flushed from recorded cycle-0 flushes, update the helper and runner/tests around that contract, and keep the broader pre-request context-budget guard as separate #12590/#51421 scope.
