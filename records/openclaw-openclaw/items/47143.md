## Summary

Keep open: current main still treats the reported persisted 0/0 memory-flush counter state as already flushed, so an over-threshold row with `compactionCount: 0` and `memoryFlushCompactionCount: 0` can still skip the flush. The normal new-session path now clears the marker, but legacy or explicitly persisted rows remain ambiguous and need a state-contract fix that preserves once-per-cycle deduplication.

## What This Changes

The issue asks OpenClaw to trigger `memoryFlush` for over-threshold sessions that have never compacted when both `compactionCount` and `memoryFlushCompactionCount` are `0`.

## Best Possible Solution

Define a backward-compatible session-state contract for memory-flush metadata, likely using `memoryFlushAt` or an explicit future sentinel to distinguish never-flushed or legacy 0/0 rows from recorded cycle-0 flushes, then update the helper, runner persistence, and focused tests while keeping broader pre-request context-budget work separate.
