## Summary

Keep this issue open. Current main still has the reported degraded Discord inbound path: message preflight classifies DMs only from fetched channel type, channel-info fetch failures return null, and routing then falls back to channel identity. Adjacent direct-user routing and session-key normalization help already-direct contexts but do not repair this misclassification path.

## What This Changes

Discord DMs should keep a stable direct-session key even when inbound channel-info lookup fails or times out.

## Best Possible Solution

Add a narrow Discord-owned preflight classification helper that prefers channelInfo when available, then falls back to trusted gateway payload signals for absent-guild one-to-one DM messages. It should route degraded one-to-one DMs through direct user identity and include regression coverage for fetchChannel returning null or throwing, without moving Discord-specific policy into core.

## Reproduction Assessment

Yes. A focused reproduction is an inbound Discord MESSAGE_CREATE payload with no guild_id, a normal user author, and a client whose fetchChannel returns null or throws; current preflight sets isDirectMessage false and routes through the channel path.

## Solution Assessment

Yes, the best fix is the narrow preflight classification change. More aggressive session-key normalization alone is not sufficient because the current bad path marks ChatType as channel before the normalizer can canonicalize direct keys.
