## Summary

Keep open. Current main still lacks a skill-specific runtime lifecycle hook or documented skill invocation event; the existing hook surfaces cover adjacent agent, tool, install, message, session, and gateway behavior but do not provide the requested before/after skill execution boundary. The enterprise permission-check comment reinforces that this needs maintainer API design, not a ClawHub-only skill implementation.

## What This Changes

Request adds before/after skill execution lifecycle hooks or skill invocation/success/failure events so plugins or hooks can observe, block, log, and analyze skill runs.

## Best Possible Solution

Add an additive, documented typed plugin hook surface for explicit skill slash-command invocation first, with a pre-invocation decision hook and post-invocation observation hook carrying bounded metadata such as skill name, command name, source, agent/session/run context, outcome, and redacted argument summary. Document separately that model-selected skills are prompt guidance unless a future runtime can identify a concrete execution boundary.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/API gap. The missing behavior is high-confidence from static inspection of the current hook catalog, hook runner, docs, skill command runtime, and skill metadata types.

## Solution Assessment

Unclear for the exact proposed `skill:beforeExecute`/`skill:afterExecute` shape. The maintainable direction is a typed plugin API with security-reviewed payloads and blocking semantics, rather than only adding raw script hook names to skill frontmatter.
