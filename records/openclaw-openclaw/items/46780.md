## Summary

Keep open. Current main has internal command hooks, plugin tool hooks, and skill slash-command dispatch, but it still lacks a skill-specific before/after execution hook or documented skill invocation event. The enterprise permission-check comment reinforces that this is a missing core/plugin API design point, not a ClawHub-only skill implementation.

## Best Possible Solution

Keep this issue open for maintainer API design. The likely path is an additive plugin hook surface for explicit skill command invocation first, with a blocking pre-invocation decision hook and a post-invocation observation hook carrying bounded metadata such as skill name, command name, source, agent/session/run context, outcome, and redacted args metadata. The docs should also clarify how model-selected skills differ from discrete runtime executions.

## Risks / Open Questions

- A generic skill execution hook needs careful semantics because model-selected skills are prompt instructions and may not map to a discrete runtime invocation.
- A pre-execution hook for enterprise permission checks can become security-sensitive if it exposes raw prompt text, user input, tool args, secrets, or skill-provided context without a bounded public contract.
- Relying on existing tool hooks would leave the prompt-rewrite skill command path and model-selected skill usage only partially covered.
