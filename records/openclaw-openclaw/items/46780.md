## Summary

Keep open. Current main has internal command hooks and plugin agent/tool hooks, and skills can be exposed as slash commands, but there is no skill-specific before/after lifecycle hook or documented skill execution event. The request identifies a missing core/plugin API rather than a ClawHub-only skill/plugin implementation.

## Best Possible Solution

Keep this open and design an additive plugin hook surface for skill invocation, likely around `src/auto-reply/reply/get-reply-inline-actions.ts` and the skill command/runtime surfaces. A good path would add a pre-invocation decision hook for block/approval plus a post-invocation observation hook with skill name, command name, source, agent/session/run context, success/error, and bounded args metadata, then document it in `docs/plugins/hooks.md` and `docs/tools/skills.md` with focused contract tests.
