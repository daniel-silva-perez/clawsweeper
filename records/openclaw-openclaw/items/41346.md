## Summary

Keep this issue open. Current main still has enabled-by-default cron creation paths, managed cron reconciliation that explicitly creates/enforces an enabled recurring job, default-model inheritance for isolated cron jobs without payload.model, and no evident cron consent/diff, expensive inherited-model guard, doctor dry-run estimate, or session-usage circuit breaker.

## Best Possible Solution

Implement an origin-aware cron safety contract in core: persist cron origin/owner metadata, require explicit activation or a consent/diff flow for newly registered recurring external/plugin/agent-tool jobs, show cadence/resolved model/model source/estimated call volume in doctor or update dry-runs, warn or block high-cost inherited defaults unless the job explicitly sets payload.model, and add a configurable usage/quota circuit breaker that can pause non-critical recurring cron jobs. Cover gateway cron.add, the agent cron tool, plugin hook cron access, bundled managed cron reconciliation, CLI/docs, and migration/backcompat tests.
