## Summary

Current main still has the reported non-deterministic `.skill` packaging path. `package_skill()` writes archive entries while iterating `skill_path.rglob("*")`, the current tests convert `ZipFile.namelist()` to sets so they do not assert entry order, the latest release has the same packaging behavior, and the related implementation PRs I found are closed and unmerged with no open replacement PR.

## Best Possible Solution

Keep this issue open and land a narrow skill-creator fix: collect eligible regular files after the existing filters and safety checks, compute each final archive-relative name, sort entries by that archive name, write the zip in that order, and add a regression test that asserts exact `ZipFile.namelist()` order for deliberately out-of-order nested files.
