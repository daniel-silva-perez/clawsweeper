## Summary

Keep this issue open. Current main still delivers prefix text for mixed trailing `NO_REPLY` in shared auto-reply/outbound paths, while the shipped cron work is narrower. The provided GitHub context also shows open implementation candidates #42873 and #46626, with #42873 using closing syntax for this issue, so the issue should remain open until that work is reviewed, landed, or closed.

## Best Possible Solution

Keep this issue open. Maintainers should review, rework, or land #42873 and #46626, ideally by adding one shared own-line/trailing silent-intent helper and applying it consistently across `normalizeReplyPayload`, directive/outbound delivery, gateway live/history projection, cron delivery, and subagent fallback paths, with tests preserving legitimate inline `NO_REPLY` text and media payload behavior.
