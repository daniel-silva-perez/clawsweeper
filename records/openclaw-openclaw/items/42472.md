## Summary

Keep this issue open. Current main still does not implement the requested shared post-processing rule: normal auto-reply normalization strips a trailing mixed `NO_REPLY` token and delivers the preceding text, outbound delivery still treats only exact silent payloads as silent, and the provided context shows open implementation candidates including PR #42873 with closing syntax for this issue.

## Best Possible Solution

Keep this issue open. Maintainers should review, land, or close PR #42873 and PR #46626, preferably by adding a shared own-line/trailing silent-token intent helper and applying it consistently across `normalizeReplyPayload`, outbound directive parsing, gateway live/history projection, cron/subagent paths, and runtime delivery classification, with tests preserving legitimate inline `NO_REPLY` mentions and media payload behavior.

## Evidence

- **Open closing PR candidate:** The provided GitHub context shows PR #42873 is open, unmerged, and its body says it closes this issue by adding own-line `NO_REPLY` suppression. Repository policy says an issue with an open PR using closing syntax should stay open until that PR is merged or closed.
