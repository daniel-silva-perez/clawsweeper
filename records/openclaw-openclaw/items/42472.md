## Summary

Keep open. Current main still preserves preamble text before a trailing `NO_REPLY` in the shared reply normalizer and several non-cron projection paths, while only exact-token, JSON-envelope, leading-token, and scoped cron trailing-token cases are covered. The issue is already paired with open implementation PRs, including #42873 with closing syntax, so maintainer consolidation is the right next step rather than closing or opening a separate repair lane.

## What This Changes

The issue requests suppressing the entire assistant reply when model output includes or ends with `NO_REPLY`, instead of stripping only the token and delivering earlier text.

## Best Possible Solution

Define one shared silent-intent helper for exact tokens, strict JSON envelopes, leading-token fragments, and chosen own-line/trailing-token semantics, then apply it consistently across reply normalization, outbound directive/delivery projection, gateway live/history projection, cron/subagent delivery, and runtime delivery classification with focused regression tests and docs updates.
