## Summary

Keep this PR open. Current main still does not implement the proposed durable pending-inbound store/replay path or disk-backed active-turn lifecycle, and the latest PR head remains a live but incomplete/security-sensitive implementation candidate rather than obsolete work. The open SQLite resilience issue #65650 is related, but it has not merged or clearly superseded this PR by maintainer decision.

## Best Possible Solution

Keep this PR open as an implementation candidate. Maintainers should require a production capture path plus file/replay trust-boundary hardening, or explicitly supersede it with an accepted merged restart-resilience design such as the SQLite direction in #65650.
