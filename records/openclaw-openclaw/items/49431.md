## Summary

Keep this PR open. Current main has added a narrower durable restart-continuation/session-delivery queue, but it still does not implement the PR’s core pending-inbound store, generic inbound replay during gateway drain, or disk-backed active-turn lifecycle/recovery. The related SQLite resilience issue #65650 remains open and is not a merged or maintainer-declared supersession. The PR also remains security-sensitive because its diff writes and replays local state into model-facing system events, with unresolved review concerns in the provided discussion.

## Best Possible Solution

Keep this PR open as an implementation candidate until maintainers either land a hardened version, explicitly supersede it with an accepted merged restart-resilience design such as #65650, or split the work into the current durable session-delivery queue plus a reviewed generic inbound capture and active-turn recovery contract.
