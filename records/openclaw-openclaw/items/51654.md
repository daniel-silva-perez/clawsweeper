## Summary

Keep this issue open. Current main still has no session-level env contract for ACP `sessions_spawn`: the tool schema does not expose `env`, the ACP spawn params do not carry it, `AcpInitializeSessionInput` does not accept it, and the ACP manager calls `runtime.ensureSession` without forwarding env even though the low-level runtime type has an optional env field.

## Best Possible Solution

Add a complete ACP session env contract: expose `env?: Record<string, string>` for `sessions_spawn` when `runtime: "acp"`, validate and redact it, carry it through `SpawnAcpParams` and `AcpInitializeSessionInput`, forward it into `runtime.ensureSession`, document merge/override behavior on the public ACP docs page, and add focused tests at the tool, ACP spawn, manager, and acpx-runtime boundary.
