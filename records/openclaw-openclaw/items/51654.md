## Summary

Keep this issue open. Current main has ACP session spawning and even a low-level ACP runtime `env` field, but `sessions_spawn` does not expose an `env` parameter, `SpawnAcpParams` does not carry one, and the ACP manager does not forward any per-session environment into `runtime.ensureSession`. The docs still describe host-level harness auth rather than session-level env overrides.

## Best Possible Solution

Add a real ACP session env contract: expose `env?: Record<string, string>` on `sessions_spawn` only for `runtime: "acp"`, validate string keys/values, redact it from persisted tool-call transcripts, carry it through `SpawnAcpParams` and `AcpInitializeSessionInput`, forward it into `runtime.ensureSession`, and add focused tests/docs showing merge/override behavior for acpx harness process env.
