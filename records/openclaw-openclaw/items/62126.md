## Summary

Keep this PR open. Current main already supports native Codex web search in runtime/config/docs and in the configure wizard, but the onboarding search path still goes directly to the managed-provider setup, so the PR's central onboarding improvement is not implemented on main. The branch is useful, but it overlaps later configure/web-search changes and should be rebased and revalidated before normal maintainer review.

## What This Changes

The PR extracts a shared native Codex web-search prompt helper, wires it into configure and onboarding search setup, and adds onboarding tests for accepting native Codex search or declining into managed-provider setup.

## Best Possible Solution

Update the PR onto current main, keep the shared helper or equivalent narrow onboarding wrapper, preserve the current configure wizard's deferred managed-provider resolution, and validate both accept and decline paths for Codex OAuth plus the non-Codex path.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main's onboarding `setupSearch` is a direct managed-flow alias, while configure already has the native Codex prompt; a fresh Codex OAuth onboarding run therefore cannot show the proposed prompt until this PR or equivalent code lands.

## Solution Assessment

Yes, with a rebase caveat. A shared prompt helper used by configure and onboarding is a narrow maintainable solution for the duplicated UX, but the final patch should be reconciled with current main's later configure wizard changes.
