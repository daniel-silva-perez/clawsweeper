## Summary

Keep this PR open. Current main already has native Codex web-search runtime support, docs, and a `configure --section web` prompt, but the onboarding path still imports `setupSearch` from `src/commands/onboard-search.ts`, where `setupSearch` is only an alias for the managed-provider `runSearchSetupFlow`. That means the PR’s central onboarding behavior is not implemented on main, although part of its configure behavior has landed separately. The provided PR diff is also narrowly scoped to command/onboarding TypeScript and tests, with no apparent supply-chain or CI/security-sensitive changes.

## Best Possible Solution

Keep the PR open and ask for a current-main rebase that focuses on the missing onboarding behavior: extract the native Codex web-search prompt into a shared helper used by both `configure.wizard.ts` and `onboard-search.ts`, preserve the existing configure behavior, add onboarding tests for accept and decline/fallthrough paths, and address the existing style review comments. After that, targeted command tests plus the normal changed gate should decide mergeability.
