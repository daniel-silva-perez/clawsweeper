## Summary

Keep this issue open. Current main has explicit Feishu `topic_group` support, but the reported delivery shape, a topic/open-chat message arriving as `chat_type: p2p`, is still classified as direct because routing and mention-forward logic only look at the normalized chat type. Open PR #52342 remains an unmerged implementation candidate that explicitly links this issue with closing syntax, so the issue should stay open until that PR is reviewed, merged, or replaced.

## Best Possible Solution

Keep this issue open. Review and land PR #52342 if it safely handles the reported event shape, or replace it with a focused Feishu plugin patch that derives group/topic routing for `p2p` topic/open-chat events from reliable context such as `thread_id`/`root_id` and/or verified chat metadata. Update both bot routing and mention-forward handling, and add regression tests for the p2p topic event plus real p2p DM/card-action cases.
