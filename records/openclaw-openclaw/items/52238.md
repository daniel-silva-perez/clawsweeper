## Summary

Keep this issue open. Current main has partial Feishu topic support for `chat_type: "topic_group"`, but the reported case is specifically a topic group delivered as `chat_type: "p2p"`; that still follows the direct-message path. Open PR #52342 is an implementation candidate and explicitly links this issue with closing syntax, so the issue should stay open until that PR is reviewed, merged, or replaced.

## Best Possible Solution

Review PR #52342, or replace it with a focused Feishu plugin patch that derives group/topic routing for the reported `p2p` topic-group events using reliable event context such as `thread_id`/`root_id` and/or verified chat metadata. Update both bot routing and mention-forward handling, and add regression coverage for the reported `p2p` topic-group event plus a real p2p callback/DM case.
