## Summary

Keep this PR open. Current main still lacks the PR's specific reusable PowerShell-cmdlet allowlist fallback, and the proposed branch still has a material execution-path concern: once a bare cmdlet is treated as allowlist-satisfied, the Windows node-host allowlist rewrite can drop the PowerShell host and try to spawn the cmdlet token directly.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is either to close it as superseded if exact-command durable approvals are the intended resolution for linked issue #52952, or revise the PR so bare cmdlet allowlist matches preserve the PowerShell host at execution time and add regression coverage for allowlist evaluation, allow-always persistence, and Windows node-host execution.
