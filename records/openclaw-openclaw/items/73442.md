## Summary

Keep this protected maintainer draft open. The branch adds useful bounded filesystem helpers, but current main does not already contain them and the proposed `statPathWithinRoot` has a concrete symlink-parent boundary bug that should be fixed before merge.

## What This Changes

This draft PR adds `statPathWithinRoot`, `readdirWithinRoot`, and `renamePathWithinRoot` to `src/infra/fs-safe.ts` with colocated `fs-safe` tests for stat, missing paths, directory listing, traversal rejection, and rename behavior.

## Best Possible Solution

Fix `statPathWithinRoot` to use the existing boundary canonicalization path or equivalent parent-symlink resolution while preserving only the final symlink when requested, add a regression test for an in-root symlinked parent pointing outside the root, then review the consumer migration plan separately.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is `root/link/file.txt` where `link` is a symlink to an outside directory, then calling `statPathWithinRoot({ rootDir: root, relativePath: "link/file.txt", followSymlinks: false })`; this review did not execute it because the pass is read-only.

## Solution Assessment

No. The PR direction is useful, but the current stat implementation is not the safest narrow solution because it bypasses the existing canonical boundary traversal used by the other new helpers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Canonicalize symlink parents when statting paths:** `src/infra/fs-safe.ts:371`
  - body: With `followSymlinks: false`, `fs.lstat` avoids following only the final path component; symlinks in parent directories are still traversed. Because this branch then treats the lexical `resolved` path as `realPath`, `root/link/file.txt` where `link` points outside the root can pass the root check while returning metadata for an out-of-root target. Use the boundary resolver or canonicalize parents while preserving only the final symlink.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff has no workflow, dependency, lockfile, or release-script changes, but it introduces a filesystem-boundary helper with a symlink-parent escape in the `followSymlinks: false` stat path.

Concerns:

- **[medium] Stat helper can misclassify out-of-root paths:** `src/infra/fs-safe.ts:371`
  - body: The new stat helper treats the lexical resolved path as safe when `followSymlinks` is false even though `fs.lstat` still traverses symlinked parent directories. Future callers could rely on this helper as a safe root boundary while receiving metadata for an out-of-root target.
  - confidence: 0.86
