## Summary

Keep open. Current main still has the silent gateway revocation close-failure behavior, and this PR is a focused hardening patch with passing GitHub checks, but it is still missing the required changelog entry for a user-visible gateway auth fix.

## What This Changes

The PR updates gateway WebSocket revocation paths to warn when `socket.close` fails, fall back to `socket.terminate`, thread revocation loggers through reload call sites, add unit coverage, and mark one secrets-reloader system event as explicitly trusted.

## Best Possible Solution

Add the required `CHANGELOG.md` entry, keep the focused log-and-terminate implementation, then complete maintainer/security review and merge if the final head remains clean and green.

## Reproduction Assessment

Yes. Current main exposes the issue directly in the three bare catch blocks, and the PR's test path reproduces it by injecting a throwing `socket.close` into shared-auth and device disconnect helpers.

## Solution Assessment

Yes for the code path. Logging the failed graceful close and falling back to `terminate` inside the existing revocation loops is the narrow maintainable fix; the remaining gap is release-note policy, not the revocation implementation.
