## Summary

Keep this issue open. Current main still does not implement an opt-in macOS LaunchDaemon/system-scope Gateway lifecycle. The CLI, service registry, launchd adapter, plist renderer, macOS app integration, doctor flow, docs, and tests remain centered on per-user LaunchAgent management; LaunchDaemon support is limited to detecting or documenting externally managed/custom services.

## Best Possible Solution

Keep this issue open for explicit macOS Gateway lifecycle design. The likely product path is an opt-in system-scope LaunchDaemon mode across openclaw gateway install/start/stop/restart/status/uninstall, macOS app integration, doctor/repair, docs, and regression tests, while preserving the current per-user LaunchAgent default.
