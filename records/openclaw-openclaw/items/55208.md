## Summary

Current main still does not implement a managed macOS LaunchDaemon/system-scope Gateway mode. The Gateway service lifecycle remains LaunchAgent-only on Darwin, while LaunchDaemon awareness is limited to best-effort extra-service detection and docs that explicitly describe custom LaunchDaemons as not shipped.

## Best Possible Solution

Keep this issue open for explicit macOS Gateway lifecycle design. The likely implementation is an opt-in system-scope LaunchDaemon mode across `openclaw gateway install/start/stop/restart/status/uninstall`, macOS app integration, doctor/repair, docs, and regression tests, while preserving the current per-user LaunchAgent default.
