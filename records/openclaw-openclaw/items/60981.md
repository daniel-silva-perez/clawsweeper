## Summary

Keep this PR open. Current main still has only the binary `tools.fs.workspaceOnly` filesystem guard; it does not implement the PR's central PathGuard capability: `allowedPaths` / `denyPaths`, canonical deny-wins policy checks, structured policy denial results, media/apply_patch/prompt-image parity, or monotonic spawned-subagent filesystem ceilings.

## Best Possible Solution

Keep this PR open. The best path is to request a current-main rebase and focused maintainer security/design review, then either land a narrowed core implementation or replace it with maintainer-owned work that adds canonical deny-wins `tools.fs.allowedPaths` / `tools.fs.denyPaths`, config schema/docs, structured policy-denied results, read/write/edit/apply_patch/media/prompt-image coverage, and monotonic spawned-subagent filesystem ceilings. Coordinate the remaining scope with #7722 and the broader security-profile discussion in #8719.
