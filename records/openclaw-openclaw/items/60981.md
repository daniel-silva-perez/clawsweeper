## Summary

Keep this PR open. Current main still has the narrower `tools.fs.workspaceOnly` filesystem guard, but not the PR's central PathGuard work: `allowedPaths` / `denyPaths`, deny-wins canonical path policy, structured `policy_denied` results, media/apply_patch/prompt-image parity, or spawned-subagent filesystem ceilings. The PR is an active implementation candidate for still-open security work tracked by #7722 and the broader #8719 discussion, so cleanup should not close it.

## Best Possible Solution

Keep this PR open. The best path is a current-main rebase plus focused maintainer security/design review, then either land a narrowed implementation or replace it with maintainer-owned work that adds canonical deny-wins `tools.fs.allowedPaths` / `tools.fs.denyPaths`, config schema/docs/generated schema alignment, structured policy-denied results, read/write/edit/apply_patch/image/pdf/prompt-image coverage, sandbox namespace correctness, and monotonic spawned-subagent filesystem ceilings. Coordinate the remaining scope with #7722 and #8719.
