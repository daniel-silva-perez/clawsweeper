## Summary

Keep this PR open. Current main still lacks the requested PathGuard ACL capability, and the PR remains a useful implementation candidate, but the current PR head still misses at least one security-relevant path and has generated config-schema drift that should block merge until fixed.

## What This Changes

The PR branch adds a PathGuard filesystem ACL layer for agent filesystem tools, media loaders, apply_patch, config schema, and spawned-subagent filesystem policy ceilings.

## Best Possible Solution

Keep the PR open and request focused updates: pass the full ToolFsPolicy through prompt-inline image detection/loading, regenerate and check config schema/docs artifacts, add regression coverage for those paths, then do a maintainer security review of the rebased branch. If maintainers prefer a smaller rollout, replace it with a narrowed maintainer-owned PathGuard PR that preserves the same enforcement invariants.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Forward fsPolicy into prompt image loading:** `src/agents/pi-embedded-runner/run/attempt.ts:466`
  - body: The new ceiling is passed into `createOpenClawCodingTools`, but prompt-local image loading still receives only `workspaceOnly` and `loadImageFromRef` has no `allowedPaths`/`denyPaths` input. In a vision run, an image under a denied workspace path can still be auto-loaded from the prompt, so PathGuard is not enforced consistently across local media reads.
  - confidence: 0.9
- **[P2] Regenerate the public config schema:** `src/config/types.tools.ts:299-302`
  - body: These new filesystem policy keys are added to the TypeScript/runtime config surface, but `src/config/schema.base.generated.ts` in the PR head still exposes only `tools.fs.workspaceOnly` and lacks `tools.subagents.fs` / `tools.sessions_spawn.fsPolicy`. Config schema/UI/docs consumers will not see or validate the new settings until the generated schema is updated.
  - confidence: 0.88
- **[P3] Preserve compatible glob allowlist intersections:** `src/agents/tool-fs-policy.ts:137-138`
  - body: When two allowlists are compatible but not exact-token or pure `/**` directory-prefix matches, such as `src/**/*.ts` plus `src/api/**/*.ts`, the combiner falls through to exact string intersection and returns `[]`. Because an empty allowlist means deny-all, a valid tightening can silently remove all filesystem access for spawned sessions.
  - confidence: 0.78
