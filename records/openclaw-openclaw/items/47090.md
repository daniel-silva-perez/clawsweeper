## Summary

Keep this PR open. Current main still does not implement the central session-archiving feature: there is no persistent `archivedAt` session state, no Gateway `includeArchived` or `archivedAt` patch contract, no reversible archive/unarchive UI, and no archive-summary write path. The existing archive code is transcript-file backup during delete/reset/doctor cleanup, which is different from hiding and restoring sessions while preserving history. The PR itself still looks like useful work, but it needs a rebase, focused maintainer review, tests, and separation of the unrelated Telegram `/set_topic_name` change before it can land.

## Best Possible Solution

Keep the PR open and ask for a rebase plus focused review. The best path is an additive Gateway/session contract for `SessionEntry.archivedAt`, `sessions.list includeArchived`, and `sessions.patch archivedAt`; Control UI archive/unarchive controls with an Archived filter; archived-session lookup paths that still work for exact resolution, subagent tools, announce targets, and ACP snapshots; auto-unarchive on inbound activity; focused Gateway/UI tests; and a reviewed archive-summary design. Split the Telegram command and unrelated docs tweak unless maintainers explicitly want them in this PR.
