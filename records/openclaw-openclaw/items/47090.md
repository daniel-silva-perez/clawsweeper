## Summary

Keep this PR open. Current main still does not implement session archiving, and the PR contains useful archive-stack work, but the branch is not merge-ready because the auto-unarchive path writes `null` into a `number | undefined` store field under strict TypeScript and the new archive filter still hides archived ACP sessions from exact session snapshot hydration.

## What This Changes

The PR adds session archive metadata, archive-aware Gateway list/patch behavior, Control UI archive/unarchive controls, archive summary file generation, inbound auto-unarchive handling, Swift protocol model updates, and a Telegram topic-name command.

## Best Possible Solution

Land a rebased, focused archive implementation after fixing typed unarchive persistence, making internal exact session lookups archive-aware, adding targeted Gateway/UI/ACP tests, and deciding whether the Telegram topic-name command belongs in the same change.

## Reproduction Assessment

Yes. The compile issue is reproducible by typechecking the PR branch because strict TypeScript rejects assigning `null` to `SessionEntry.archivedAt?: number`; the ACP behavior is reproducible by archiving an ACP session and then asking the ACP bridge to hydrate its session snapshot through `sessions.list` search without `includeArchived`.

## Solution Assessment

No. The feature direction is plausible, but this branch is not the best mergeable solution until the archive state type is modeled consistently and archive hiding is limited to user-facing list views or explicitly opted out of by internal exact-lookup callers.

  - body: `SessionEntry.archivedAt` is declared as `number | undefined`, but this inbound auto-unarchive path assigns `null` with `strict` TypeScript enabled. That blocks the TS build on this branch; either widen the stored type to include `null` or persist unarchive by deleting/normalizing the field before the store merge.
  - confidence: 0.91

  - body: The new default filter drops archived rows before search filtering, while ACP still hydrates session snapshots with `sessions.list({ search: sessionKey })` and no `includeArchived`. Archived ACP sessions then fall back to default presentation even though the session still exists, so exact/internal lookups need `includeArchived: true` or a lookup path that bypasses the default UI hiding filter.
  - confidence: 0.87
