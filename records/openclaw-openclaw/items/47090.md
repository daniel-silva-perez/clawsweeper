## Summary

Keep this PR open. Current main still does not implement the central session archiving behavior: no persistent archivedAt session state, no sessions.list includeArchived contract, no sessions.patch archivedAt mutation, no archive/unarchive UI controls, and no archive-summary write path. The PR remains potentially useful, but it still needs maintainer review, focused tests, and separation or explicit acceptance of the unrelated Telegram /set_topic_name work.

## Best Possible Solution

Keep this PR open and ask for a rebase plus focused maintainer review. The best path is an additive Gateway/session contract for SessionEntry.archivedAt, sessions.list includeArchived, and sessions.patch archivedAt; Control UI archive/unarchive controls with an Archived filter; archived-session lookup paths that still work for exact resolution, subagent tools, announce targets, and ACP snapshots; auto-unarchive on inbound activity; focused Gateway/UI tests; and a reviewed archive-summary design. Split the Telegram command and unrelated docs tweak unless maintainers explicitly want them in this PR.
