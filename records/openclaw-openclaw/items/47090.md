## Summary

Keep this PR open. Current main does not implement the PR’s intended session archiving feature: there is no `archivedAt` session field, no `includeArchived` sessions.list parameter, no `archivedAt` sessions.patch support, and the Control UI Sessions view still only has Global/Unknown filters plus delete, not Archive/Unarchive. Existing archive code on main is transcript-file archival for reset/delete/pruning, which is a different behavior from hiding/restoring sessions while preserving history.

## Best Possible Solution

Keep the PR open, but require a rebase onto current main and a focused maintainer review. The best path is to implement session archiving as an additive Gateway/session contract (`SessionEntry.archivedAt`, `sessions.list includeArchived`, `sessions.patch archivedAt`), wire Control UI Archive/Unarchive and Archived filter behavior, preserve lookup paths that must still resolve archived sessions, auto-unarchive on inbound activity, and add focused Gateway/UI tests. Split the Telegram `/set_topic_name` and unrelated docs tweak unless maintainers explicitly want them in the same PR.
