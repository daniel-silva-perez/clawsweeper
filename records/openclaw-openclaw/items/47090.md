## Summary

Keep this PR open, but it is not ready to merge. Current main still lacks the requested session archive contract, while this branch is stale against current main, mixes unrelated Telegram/topic-label and docs changes into the archive feature, and needs a privacy/security decision for writing chat excerpts into agent memory.

## What This Changes

The PR adds an `archivedAt` session flag, archived-session filtering and UI archive/unarchive controls, archive-summary file writing, inbound auto-unarchive behavior, protocol/client field updates, and a Telegram `/set_topic_name` command plus small docs edits.

## Best Possible Solution

Land this only as a rebased, focused session-archiving change: add a generic `archivedAt` contract through schema, store, listing, resolve/tool paths, UI state, and session events; preserve programmatic access to archived sessions where needed; cover archive/unarchive and inbound auto-unarchive with targeted tests; and split unrelated Telegram topic-label work into a separate PR. The memory-summary side effect should either be opt-in/redacted/documented or removed from the first archive implementation.
