## Summary

Keep open. Current main still lacks Gateway directory RPCs, so this PR is not obsolete, but the proposed patch is not ready as-is because it advertises new public methods without typed protocol contract follow-through and does not preserve live-directory fallback behavior for channels that expose both config-backed and live directory adapters.

## What This Changes

The PR adds `directory.self`, `directory.peers.list`, and `directory.groups.list` Gateway handlers, registers them in core handler discovery, and maps them to `operator.read`.

## Best Possible Solution

Land this as a complete Gateway protocol addition: define typed params/result schemas and validators, document the methods, add focused handler/method-discovery/scope tests, preserve cache-first plus live-on-miss directory behavior where live adapters exist, and then merge this PR or an equivalent maintainer-polished implementation.

## Reproduction Assessment

Not applicable as a feature PR, but the current-main gap is high-confidence: the repo has CLI directory support and channel adapter contracts, while Gateway handler registration, method discovery, and scope mapping for `directory.*` are absent.

## Solution Assessment

No, not as-is. The direction is maintainable, but the patch should first add the typed Gateway protocol contract and fix cache/live directory lookup semantics before it becomes the best solution.
