## Summary

Keep this issue open. Current main still has manual `contextTokens`/`contextWindow` override plumbing, but it does not implement llama.cpp runtime-context discovery: generic OpenAI-compatible discovery only reads `/models`, there is no production `/props.n_ctx` reader, and the sessions table still displays the resolved/stored context cap as its denominator.

## Best Possible Solution

Keep the issue open and fix provider/context resolution. For llama.cpp-style OpenAI-compatible providers, add a guarded discovery path that strips `/v1` when needed, reads `/props.n_ctx`, stores or resolves that as `contextTokens`, preserves `contextWindow` as native/training metadata, and keeps explicit user-configured `contextTokens` first. Add regression coverage for a server advertising `n_ctx_train: 262144` via `/v1/models` but runtime `n_ctx: 65536` via `/props`, then verify sessions show about `56k/65k`, not `56k/262k`.
