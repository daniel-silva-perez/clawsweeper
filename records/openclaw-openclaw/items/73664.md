## Summary

Keep this issue open. Current main has useful `contextTokens` plumbing and manual config override support, but it still does not implement the reported llama.cpp runtime-context discovery: there is no production `/props`/`n_ctx` reader, generic OpenAI-compatible discovery still only hits `/models`, and session displays still depend on whatever context cap was stored or resolved upstream.

## Best Possible Solution

Keep the issue open and fix the provider/context-resolution path. For llama.cpp-style OpenAI-compatible local providers, add a guarded runtime-context discovery path that reads `/props.n_ctx` and stores or resolves it as `contextTokens` while preserving `contextWindow` as native/training metadata. Preserve explicit user caps, document the precedence, and add regression coverage for a model advertising `n_ctx_train: 262144` with runtime `n_ctx: 65536` so sessions show about `56k/65k`, not `56k/262k`.
