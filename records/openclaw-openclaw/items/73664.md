## Summary

Keep this issue open. Current main still lacks llama.cpp `/props.n_ctx` discovery for self-hosted OpenAI-compatible providers, and the provided GitHub context shows #74057 is an open, unmerged PR with closing syntax that directly targets this report.

## What This Changes

Request the llamacpp provider/session display to use the actual runtime context cap (`n_ctx` or an authoritative configured cap) instead of the model training context (`n_ctx_train`) when calculating session token percentages.

## Best Possible Solution

Review and land or revise #74057, or an equivalent narrow self-hosted provider discovery change, so explicit configured caps stay authoritative while llama.cpp-compatible servers can supply the smaller loaded runtime context via `/props.n_ctx` for session budgeting.

Yes. ## Solution Assessment
