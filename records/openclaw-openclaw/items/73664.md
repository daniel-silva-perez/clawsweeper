## Summary

Keep this issue open. Current main still has the session display wired to persisted/resolved `contextTokens`, but it does not implement the reporter's two requested fixes for llama.cpp: there is no `/props`/`n_ctx` reader, and existing agent `models.json` model context values are not preserved as authoritative during the merge-mode rewrite path.

## Best Possible Solution

Keep the issue open and fix the provider/context-resolution path. The most direct product fix is to make llama.cpp/OpenAI-compatible local providers persist an effective runtime `contextTokens`: either fetch llama.cpp `/props` and use `n_ctx`, or preserve an explicit configured/runtime cap from agent `models.json` over live `/v1/models` native metadata. Add focused coverage for a model whose native/training context is 262144 but runtime cap is 65536, proving sessions display the smaller runtime denominator.
