## Summary

Current main still lacks the requested llama.cpp runtime context discovery. OpenAI-compatible self-hosted discovery reads only `/models`, current context-token resolution only uses configured metadata/cache/model rows, and the sessions table displays the resulting `contextTokens` denominator. Manual `contextTokens`/`contextWindow` config can mitigate the display, but there is still no production `/props.n_ctx` path to prevent `n_ctx_train` from being used when that larger value is what upstream discovery stored.

## Best Possible Solution

Keep this issue open and implement a guarded llama.cpp runtime-context discovery path. For llama.cpp-style OpenAI-compatible providers, read `/props.n_ctx` and store or resolve it as effective `contextTokens` while keeping `contextWindow` available for native/training metadata. Preserve explicit configured caps, document the precedence, and add regression coverage where `/v1/models` exposes `n_ctx_train: 262144` but `/props.n_ctx` is `65536`, so sessions display about `56k/65k` instead of `56k/262k`.
