## Summary

Keep this issue open because current main still lacks llama.cpp runtime-context discovery and the provided GitHub context shows an open, unmerged PR that explicitly targets this issue with closing syntax. The source still discovers self-hosted OpenAI-compatible models only through `/models`, while session display consumes stored/configured context tokens without probing `/props.n_ctx`.

## What This Changes

The issue requests that self-hosted llama.cpp provider sessions display the loaded runtime context window (`n_ctx`) instead of the model training context ceiling (`n_ctx_train`).

## Best Possible Solution

Review, revise if needed, and land PR #74057 or an equivalent guarded self-hosted provider discovery change that reads positive llama.cpp `/props.n_ctx` as `contextTokens`, handles trailing `/v1` base URLs, keeps explicit configured caps authoritative, and adds regression coverage for `/v1/models` reporting `n_ctx_train: 262144` while `/props` reports `n_ctx: 65536`.

## Reproduction Assessment

Yes. ## Solution Assessment
