## Summary

Keep this issue open. Current main still has no llama.cpp `/props`/`n_ctx` runtime-context reader, and the sessions table continues to display percentages from stored or resolved `contextTokens`. Config-level `contextTokens` can override discovery, but existing generated `models.json` model metadata is not preserved as authoritative when the same provider is regenerated from live discovery.

## Best Possible Solution

Fix this in the provider/context-resolution path, not by special-casing table rendering. For llama.cpp/OpenAI-compatible local providers, either fetch `/props` with the existing guarded provider HTTP policy and persist `n_ctx` as effective `contextTokens`, or preserve an explicit same-provider/model runtime cap from config/existing `models.json` before live `/v1/models` metadata can replace it. Add focused coverage for 262144 trained context versus 65536 runtime context and verify sessions display against 65536.
