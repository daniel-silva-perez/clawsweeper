## Summary

Keep this issue open. Current main still does not implement llama.cpp `/props.n_ctx` runtime-context discovery for self-hosted OpenAI-compatible providers, and the linked #74057 PR is still the open implementation candidate with closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to show the actual llama.cpp runtime context window in session token denominators instead of the model's trained `n_ctx_train` limit.

## Best Possible Solution

Land or revise #74057, or an equivalent narrow provider-discovery change, so explicit configured caps stay authoritative while llama.cpp-compatible servers can supply the smaller loaded runtime context via `/props.n_ctx`.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: current main has no `/props.n_ctx` path, and session display uses stored/resolved context tokens without a live llama.cpp runtime-context probe.

## Solution Assessment

Yes. Keeping this issue open while maintainers review the existing guarded `/props.n_ctx` PR is the best path; opening a duplicate ClawSweeper repair branch would add churn before that candidate is resolved.
