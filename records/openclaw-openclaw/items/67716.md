## Summary

Current main still appears to have the reported host-side scheduling gap. Bootstrap/reconcile maintenance enters the default foreground path, deferred compaction execution is still only enabled for background maintenance, and only turn-triggered maintenance is eligible for the deferred background worker. The existing ClawSweeper review comment is consistent with current main at 5828dcdb05aa.

## Best Possible Solution

Keep this issue open. The best path is a narrow host change that lets bootstrap/reconcile-created deferred compaction debt schedule or run an allowed background maintenance pass, or an additive context-engine maintenance contract for engines to request a debt-consuming follow-up or explicitly report that policy superseded debt. Add a bootstrap/reconcile regression test and coordinate the hot-cache semantics with the Lossless Claw plugin.
