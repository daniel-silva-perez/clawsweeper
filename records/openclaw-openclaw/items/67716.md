## Summary

Keep open. Current main still matches the reported OpenClaw host gap: bootstrap/reconcile maintenance runs in the foreground path, while only turn-triggered maintenance can queue the background worker that sets `allowDeferredCompactionExecution`. The April 29 comment also reports the same sticky-debt symptom on the latest release with `lossless-claw` 0.9.2, so this is neither stale nor already implemented.

## What This Changes

The issue requests that context-engine maintenance avoid leaving deferred compaction debt stranded after bootstrap/reconcile imports or hot-cache threshold crossings.

## Best Possible Solution

Add a generic host path or contract so bootstrap/reconcile-created maintenance debt can request or receive an allowed background follow-up, and make the engine outcome explicit when policy intentionally supersedes recorded debt. Keep plugin-specific hot-cache thresholds and policy inside the Lossless Claw plugin.
