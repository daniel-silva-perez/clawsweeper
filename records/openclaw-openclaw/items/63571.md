## Summary

Keep open. Current main still lacks explicit `tools.web.search.fallbacks` support, while this PR proposes a focused config/runtime/docs/test feature that is not obsolete but needs maintainer review because it changes outbound web-search retry behavior and is currently unmergeable against main.

## What This Changes

The PR adds a `tools.web.search.fallbacks` config surface and runtime retry logic so a configured primary web-search provider can fall back through ordered provider ids on failure, with docs, generated schema metadata, and runtime tests.

## Best Possible Solution

Keep this PR open for maintainer review.
