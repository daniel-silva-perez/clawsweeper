## Summary

Keep open. Current main still only implements the native Signal JSON-RPC/SSE transport, while this PR is an active, unmerged implementation candidate for bbernhard signal-cli-rest-api REST/WebSocket support. The PR should stay in maintainer review because it changes transport, config, docs, and runtime dependency behavior and still has review items that are not just a narrow autonomous repair.

## What This Changes

The PR adds a Signal native/container adapter, a bbernhard REST/WebSocket client, `channels.signal.apiMode` docs/types/tests, caller rewiring, and a direct `ws` dependency for the Signal plugin.

## Best Possible Solution

Ship a Signal plugin-local transport adapter that preserves the existing native JSON-RPC/SSE path while adding documented bbernhard REST/WebSocket mode, with config types/schema/help/docs kept in sync, raw payload logging removed or gated, focused tests for both modes, and live container smoke proof or an explicit maintainer waiver.
