## Summary

Keep open. Current main still lacks Signal container REST/WebSocket support, while this PR is the active implementation branch and has concrete pre-merge blockers: raw inbound Signal payload logging and a public `apiMode` config field that is not accepted by the strict config schema.

## What This Changes

The PR adds a Signal native/container adapter, a bbernhard REST/WebSocket client, `channels.signal.apiMode` docs/types/tests, caller rewiring, a changelog fragment, and a direct `ws` runtime dependency for the Signal plugin.

## Best Possible Solution

Ship a Signal plugin-local transport adapter that preserves native JSON-RPC/SSE behavior while adding documented bbernhard REST/WebSocket mode, with config schema/help/docs kept in sync, raw payload logging removed or redacted, focused tests for both modes, and live container smoke proof or an explicit maintainer waiver.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Remove raw Signal payload logging:** `extensions/signal/src/client-container.ts:202`
  - body: The bbernhard receive stream carries full Signal envelopes. Logging the first 200 characters of every inbound frame can write private message text and sender metadata to gateway logs during normal operation; log redacted metadata only or gate this behind an explicit debug path.
  - confidence: 0.93
- **[P2] Validate the apiMode setting:** `src/config/types.signal.ts:68`
  - body: This exposes `channels.signal.apiMode` in the public type/docs, but PR head does not add `apiMode` to the strict `SignalConfigSchema`. Users following the new docs with `apiMode: "container"` can fail config validation before adapter routing can use the setting.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The new WebSocket dependency and transport are purpose-aligned, but the receive path currently logs raw Signal payload snippets.

Concerns:

- **[medium] Raw Signal payloads are logged:** `extensions/signal/src/client-container.ts:202`
