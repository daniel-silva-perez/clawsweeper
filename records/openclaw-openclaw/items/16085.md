## Summary

Keep this PR open. Current main still implements the bundled Signal plugin against native signal-cli JSON-RPC plus SSE only, with no `apiMode`, no container adapter, no `/v2/send`, and no `/v1/receive/{account}` WebSocket path. The PR remains the canonical implementation candidate for the container REST/WebSocket support that #10240 and #73553 describe.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate for bundled Signal container REST/WebSocket support. Finish maintainer review of the adapter/container client, remove or gate raw inbound payload logging, align config schema/help/docs/tests, preserve native signal-cli JSON-RPC/SSE behavior, and require targeted Signal tests plus live bbernhard container proof or an explicit maintainer waiver before merge. Treat #73086 as adjacent native SSE timeout work, not a replacement for this PR.
