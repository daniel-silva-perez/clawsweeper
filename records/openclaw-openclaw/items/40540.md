## Summary

Keep open. Current main still performs the npm-global package replacement through the running update flow, and there is no detached Windows updater on main; the open PR #44614 remains the existing implementation candidate rather than a reason to close this issue before merge.

## What This Changes

Fix `openclaw update` on Windows npm-global installs so self-update does not fail with `EBUSY` while replacing the active OpenClaw package tree.

## Best Possible Solution

Keep this as the canonical Windows npm-global self-update `EBUSY` report until #44614 or an equivalent Windows-safe design lands. The desired end state is an updater path that releases the active CLI/Gateway process from the live package tree before npm replaces it, with regression coverage for `EBUSY`/`EPERM` activation and a real Windows update smoke path.
