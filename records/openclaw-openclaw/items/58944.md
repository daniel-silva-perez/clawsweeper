## Summary

Keep open. Current main still treats explicit `background: false` the same as an omitted `background` parameter, so the requested synchronous exec behavior is not implemented; however, the PR branch is now dirty against current main and needs a rebase or narrow replacement before it can merge safely.

## What This Changes

The PR changes exec so `background: false` disables the yield timer, updates exec parameter copy, and adds a unit test for synchronous completion with an aggressive background timeout.

## Best Possible Solution

Implement the feature on current main by making only explicit `background: false` set `yieldWindow` to null while preserving omitted `background` and `background: true` behavior, then update the current schema file, tool description, public docs, targeted tests, and changelog.

## Reproduction Assessment

Yes. A focused static reproduction is clear: on current main, call exec with `background: false` and a low `backgroundMs`; the code still builds a non-null `yieldWindow`, so the timer can background the command. The PR's proposed unit test is the right executable proof, though tests were not run during this read-only review.

## Solution Assessment

Unclear as submitted. Setting `yieldWindow = null` for explicit false is the narrow code path if maintainers accept that API contract, but this branch needs a current-main rebase and docs/changelog follow-through before it is the best mergeable solution.
