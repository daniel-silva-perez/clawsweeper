Keep this PR open. Current main still has the Telegram no-session-override path reading `agents.defaults.model` directly, while the model-list UI marks the check based on that value. The PR is a focused candidate that changes the implicated path to the bound agent's resolved default model and adds a regression test for the reported Telegram scenario.

The best path is to reuse a single `defaultModel` value in `resolveTelegramSessionState`, keep the regression test for bound-agent default selection in the Telegram model list, address the minor review comments, then run the targeted Telegram test and changed gate before merge.
