## Summary

Keep this PR open. Current main still has the reported Telegram no-session-override fallback to `agents.defaults.model`, and that fallback feeds the Telegram model-list check mark. The PR remains a focused implementation candidate with a targeted runtime change and regression test.

## Best Possible Solution

Keep this PR open and review it as the focused implementation candidate. The best fix is to compute the bound agent default once with `resolveDefaultModelForAgent({ cfg: runtimeCfg, agentId: route.agentId })`, reuse it for stored override default-provider resolution, return `${defaultModel.provider}/${defaultModel.model}` when no session override exists, and keep the regression test for a Telegram-bound agent whose explicit model differs from `agents.defaults.model`.
