## Summary

Keep this PR open. Current main still uses `agents.defaults.model` as the no-session-override fallback in Telegram model callbacks, so the bound agent's configured model is not what drives the model-list check mark. The PR remains a focused implementation candidate with a matching regression test and no security-sensitive file drift in the provided diff.

## Best Possible Solution

Keep this PR open and review it as the focused implementation candidate. The best fix is to compute the bound agent default once with `resolveDefaultModelForAgent({ cfg: runtimeCfg, agentId: route.agentId })`, reuse its provider for stored override normalization, return `${defaultModel.provider}/${defaultModel.model}` when no session override exists, and keep the regression test for a Telegram-bound agent whose explicit model differs from `agents.defaults.model`.
