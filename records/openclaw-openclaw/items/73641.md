## Summary

Keep this PR open. Current main still falls back to `agents.defaults.model` when Telegram has no stored session model override, and that value feeds the model-list keyboard check mark. The live PR remains a focused implementation candidate with one runtime change and one regression test, and I did not find security-sensitive or unrelated file changes in the PR file list.

## Best Possible Solution

Keep this PR open and review it as the focused fix. The best implementation is to compute the bound agent default with `resolveDefaultModelForAgent({ cfg: runtimeCfg, agentId: route.agentId })`, reuse it for stored-override default provider resolution, return `${defaultModel.provider}/${defaultModel.model}` on the no-override path, and keep the bound-agent regression test.
