## Summary

Keep open. Current main already fixes the main write-side clobber path by persisting configured subagent models as session overrides before the child run starts, but this PR still proposes additional fallback and precedence behavior that is not fully on main. The patch needs maintainer/author revision because both new fallback paths parse configured subagent model aliases as raw model ids, and the global-default precedence change is user-visible.

## What This Changes

The PR changes subagent model selection so gateway/reply runtime fallbacks prefer configured subagent defaults, reorders subagent default precedence, adds targeted tests, and includes small config-validation and qa-lab type-export changes.

## Best Possible Solution

Revise the PR so all subagent runtime/read fallbacks reuse the same alias-aware model resolution contract as the spawn path, decide and document the intended global subagent default precedence, and drop or split unrelated edits before running targeted tests and the changed gate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Resolve aliases in the reply-time subagent default:** `src/auto-reply/reply/directive-handling.defaults.ts:56`
  - body: `resolveSubagentConfiguredModelSelection` can return a configured alias such as `gpt`; the spawn path resolves that through the model alias index, but this helper feeds the raw value to `parseModelRef`. In the fallback case it can boot a subagent as `<defaultProvider>/gpt` and then persist that bogus runtime model instead of the configured provider/model.
  - confidence: 0.91
- **[P2] Resolve aliases in the gateway subagent fallback:** `src/gateway/session-utils.ts:1149`
  - body: This fallback has the same alias gap when `entry.model` is empty: a configured subagent alias is parsed as a bare model under the default provider instead of being resolved through the alias index. That makes the read fallback disagree with the existing spawn-time model selection contract and its alias tests.
  - confidence: 0.88
