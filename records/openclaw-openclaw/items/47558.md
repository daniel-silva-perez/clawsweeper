## Summary

Keep open: current main has fixed some of the reported docs drift, including the `streamTo` parameter and clearer thread-binding key separation, but the sub-agents docs still omit `announceTimeoutMs`, still describe depth-2 as unable to spawn despite the current `depth < maxSpawnDepth` rule, and still use depth-specific auto-archive wording. The issue also has multiple open PRs with closing syntax, so the issue should stay open until maintainers choose and land one updated canonical docs fix.

## What This Changes

The issue requests six corrections to sub-agent documentation around `announceTimeoutMs`, `streamTo`, thread-binding keys, `maxSpawnDepth`, sub-agent tool policy, and auto-archive wording.

## Best Possible Solution

Land one focused docs PR that updates `docs/tools/subagents.md` and the related config reference to current source facts: document `announceTimeoutMs` with the 120000ms default, describe spawning as allowed while `depth < maxSpawnDepth` across the 1-5 range, make auto-archive and cascade wording depth-agnostic, and clarify the current always-denied versus leaf-only sub-agent tool sets while preserving the already-correct `streamTo` and thread-binding sections.
