## Summary

Keep this issue open. Current main only partially addresses the requested sub-agent docs fixes: `streamTo` and the thread-binding split are now documented, but `announceTimeoutMs` is still absent from the sub-agents page, the depth docs still hard-code depth 2 as a leaf despite source support through `maxSpawnDepth` 5, auto-archive is still described as depth-1/depth-2 specific, and the tool-policy wording still does not enumerate the always-denied system tools. The issue is also paired with open fix PRs using closing syntax, including same-author #47935, so maintainers should review one canonical PR rather than close the issue before merge.

Keep the issue open and review one canonical PR, preferably a focused update to `docs/tools/subagents.md` that documents `announceTimeoutMs` with the current runtime default, corrects the depth table/tool-policy wording for `maxSpawnDepth` 1-5, makes auto-archive wording depth-agnostic, and verifies the current always-denied tool list from `src/agents/pi-tools.policy.ts`. After that PR merges, GitHub can close this issue as implemented.
