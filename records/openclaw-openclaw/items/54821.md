## Summary

Keep this PR open. Current main still lacks the PR's intended `thinking-events` gateway capability and dedicated recipient registry, and embedded reasoning streaming is still gated on `onReasoningStream`, so gateway WebSocket runs without that callback will not get the proposed thinking events. The linked feature issue being closed does not make this PR obsolete; this PR remains the implementation candidate unless a newer canonical PR replaces it.

## Best Possible Solution

Keep the PR open for maintainer review or supersede it with a newer canonical implementation. The best code path is to add a documented `thinking-events` gateway capability, register thinking recipients before events can emit, route thinking only through run-scoped `broadcastToConnIds`, keep node/channel subscribers from receiving raw reasoning tokens, compute deltas from raw emitted reasoning state, and cover the routing and delta behavior with gateway and embedded-subscribe tests.
