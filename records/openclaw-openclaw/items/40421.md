## Summary

Keep this PR open but do not merge it as-is. Current main does not implement waiting tips, and the PR still targets the removed legacy Telegram path while also exposing an unimplemented config option and carrying a send/delete race that prior review comments already identified. The related open processing-indicator issue is the better product-direction anchor before deciding whether this should be a minimal indicator or bilingual educational tips.

## What This Changes

The PR adds a shared waiting-tip controller and tests, `messages.waitingTips` config/schema/help/labels, and legacy Telegram dispatch wiring to send and delete bilingual tip messages while replies are being generated.

## Best Possible Solution

Settle the Telegram feedback UX against #6946, then port any accepted implementation to `extensions/telegram` with honest typed config, race-safe send/delete cleanup, focused dispatch/config regression coverage, and docs if the user-visible config ships.
