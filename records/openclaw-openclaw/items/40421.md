## Summary

Keep open: current main does not implement the waiting-tip feature, and the proposed patch is not merge-ready because it hooks the removed legacy Telegram path, has a cleanup race that can leave tip messages behind, and exposes a custom tips config option that is not wired to runtime behavior. The overlapping narrower UX request in #6946 is still open, so this needs maintainer product direction before any port or replacement work.

## What This Changes

This PR adds a shared waiting-tip controller and tests, `messages.waitingTips` config/schema/help/labels, and Telegram dispatch wiring to send and delete bilingual tips while replies are being generated.

## Best Possible Solution

Decide the Telegram waiting-feedback UX against #6946, then port any accepted implementation into `extensions/telegram` with honest typed config, race-safe send/delete cleanup, focused dispatch/config regression coverage, and docs if user-facing configuration ships.

## Reproduction Assessment

Yes. The main mismatch is reproducible by checking that `src/telegram` is absent and the active dispatcher is `extensions/telegram/src/bot-message-dispatch.ts`; the cleanup race is reproducible by letting `sendTip` resolve after `cleanup()` has set `cancelled = true`.

## Solution Assessment

No. The PR is useful source work, but it is not the best merge path as-is because it targets an obsolete path and exposes incomplete config; the safer path is a maintainer-approved Telegram plugin implementation, possibly narrowed to #6946.
