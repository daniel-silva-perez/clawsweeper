## Summary

Keep open. Current main does not implement the requested visible Telegram waiting-tip behavior, and the PR still contains useful source work, but the branch is not mergeable as-is because it hooks a removed Telegram path and the reviewed diff has concrete cleanup/config defects already noted in prior review comments.

## What This Changes

This PR adds a shared waiting-tip controller, `messages.waitingTips` config metadata, tests, and Telegram dispatch wiring to send and delete bilingual tip messages while a reply is being generated.

## Best Possible Solution

Decide the Telegram waiting-feedback UX against #6946, then port any accepted implementation into `extensions/telegram` with race-safe send/delete cleanup, honest typed config, focused dispatch/config regression coverage, and docs if user-facing configuration ships.

## Reproduction Assessment

Yes. The main mismatch is reproducible by checking that current main has no `src/telegram` directory and the active dispatcher is `extensions/telegram/src/bot-message-dispatch.ts`; the cleanup race is reproducible by letting `sendTip` resolve after `cleanup()` has set `cancelled = true`.

## Solution Assessment

No. The PR is useful source work, but it is not the best merge path as-is because it targets an obsolete path, can leave temporary tip messages behind, and exposes incomplete custom-tip config; the safer path is a maintainer-approved Telegram plugin implementation, possibly narrowed to #6946.
