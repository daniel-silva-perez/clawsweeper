## Summary

Keep open: current main does not implement the proposed Telegram bilingual waiting-tip behavior, and the PR still has concrete merge blockers around the active plugin path, temporary-message cleanup, and advertised config. It also overlaps the narrower open Telegram processing-indicator request in #6946, so maintainer product direction is needed before this should be repaired or replaced.

## What This Changes

The PR adds a shared waiting-tip controller, tests, `messages.waitingTips` config metadata, and Telegram dispatch wiring to send and later delete bilingual tip messages while a reply is being generated.

## Best Possible Solution

Decide the Telegram waiting-feedback UX against #6946, then implement any accepted behavior in `extensions/telegram` with race-safe send/delete lifecycle handling, honest config metadata, focused regression coverage, and docs if user-facing configuration ships.

## Reproduction Assessment

Yes. The integration mismatch is reproducible by checking that current main has no `src/telegram` directory and that the active dispatcher is `extensions/telegram/src/bot-message-dispatch.ts`; the cleanup race follows from the proposed `cleanup()` path when `sendTip` resolves after cancellation.

## Solution Assessment

No. The PR is useful source work, but it is not the best merge path as-is because it targets an obsolete path, can leave temporary tip messages behind, and exposes incomplete custom-tip config; a maintainer-approved Telegram plugin implementation is safer.
