## Summary

Keep this issue open. Current main still creates Feishu dynamic-agent DM bindings without an account scope, while Feishu ingress re-resolves routes with the runtime account id; named-account traffic can still miss the generated binding and fall back to agent:main. The directly related fix PR was closed unmerged, so the remaining work is not shipped.

## Best Possible Solution

Refresh the account-aware approach from PR #42886 on current main: pass the normalized Feishu runtime accountId into maybeCreateDynamicAgent, include match.accountId when checking and writing dynamic bindings for both message and comment flows, preserve legacy default-account behavior for account-less bindings, and add focused regression tests for named-account creation, missing-binding repair, default-account compatibility, and duplicate-binding avoidance.
