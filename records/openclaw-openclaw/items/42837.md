## Summary

Keep this issue open. Current main still creates Feishu dynamic-agent DM bindings without an account scope, while Feishu DM and comment ingress resolve routes using the runtime account id. For named Feishu accounts, the generated binding can miss the account-scoped route lookup and fall back to the default agent. The directly related fix PR #42886 was closed unmerged, so there is no shipped fix to close against.

## Best Possible Solution

Refresh the account-aware approach from PR #42886 on current main: pass the normalized Feishu runtime accountId into maybeCreateDynamicAgent, include match.accountId when checking and writing dynamic bindings for message and comment flows, preserve default-account compatibility for legacy account-less bindings, and add focused regression tests for named-account creation, missing-binding repair, default-account behavior, and duplicate-binding avoidance.
