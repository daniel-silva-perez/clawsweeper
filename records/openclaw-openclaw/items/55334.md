## Summary

Keep this open. Current main has bounded-growth mitigations for session stores and cron run sessions, but the core reported bloat remains: retained session rows still store full `skillsSnapshot` and `systemPromptReport` payloads inline, and fresh production comments report the same retained-row cost on v2026.4.26.

## What This Changes

The issue requests a lighter, bounded session persistence model by pruning ephemeral sessions, deduplicating large static prompt payloads, cleaning orphan artifacts, and reducing whole-store heap and lock pressure.

## Best Possible Solution

Keep this as the broad session-store bloat tracker and design a backward-compatible static-payload storage path: keep the current pruning and disk-budget safeguards, move skills snapshots and system-prompt reports out of per-session rows by reference or recomputation, compact legacy rows safely, and add coverage proving many cron/subagent/chat sessions no longer duplicate large static payloads while old sessions still load.
