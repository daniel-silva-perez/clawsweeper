## Summary

Keep this PR open. Current main still does not implement sandbox-only Python/Node research import preflight, and the PR remains a plausible core sandbox hardening change. It should be reviewed rather than closed, with security fixes for the parser bypasses already raised in review comments.

## Best Possible Solution

Keep this PR open and request changes. The best path is to rebase onto current main, add sandbox-only import-template validation after the existing direct-script preflight, use parser-backed or multiline-aware Node import extraction, handle Python bare-relative denied-name imports, preserve host exec behavior, and add focused regression tests for sandbox-vs-host behavior plus the reported bypasses.
