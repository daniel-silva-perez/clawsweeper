## Summary

Keep this PR open. Current main still has only the existing shell-syntax script preflight for Python/Node exec files, not the proposed sandbox-only research import templates, and the PR remains a plausible core sandbox hardening change. It should not be closed as implemented, but it also should not merge as-is because review comments identify bypasses in the proposed text scanner.

## Best Possible Solution

Keep this PR open and request changes. The best path is to rebase onto current main, add sandbox-only import-template validation after the existing direct script preflight, use parser-backed or multiline-aware Node import extraction, handle Python bare-relative denied-name imports, and add focused regression tests for sandbox-vs-host behavior plus the reported bypasses.
