## Summary

Keep this PR open. Current main still has only the existing Python/Node script preflight for shell-syntax leakage and ambiguous interpreter invocations; it does not implement sandbox-only research-script import templates. The PR targets a real command-execution hardening gap, but the proposed implementation has unresolved parser bypasses in the new scanner, so the best maintainer action is request changes rather than close it as obsolete.

## Best Possible Solution

Keep this PR open and request changes. The best path is to rebase onto current main, add sandbox-only import-template validation after the existing safe script preflight, harden parsing for multiline Node import/re-export syntax and whitespace-separated dynamic require/import calls, handle Python bare-relative denied-name imports, and add regression tests proving sandbox-vs-host behavior plus the identified bypasses.
