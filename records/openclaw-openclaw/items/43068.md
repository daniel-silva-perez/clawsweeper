## Summary

Keep this PR open. Current main still does not implement sandbox-only Python/Node research-script import templates, and the PR is a plausible hardening change for a real exec boundary, but the proposed text scanner has unresolved security-relevant parser bypasses that need maintainer review before merge.

## Best Possible Solution

Keep this PR open and request changes. The best path is to rebase onto current main, add sandbox-only import-template validation after the existing script preflight, use a parser or multiline-aware extraction for Node import/re-export and dynamic-call forms, handle Python bare-relative denied-name imports, and add regression tests for sandbox-vs-host behavior plus the reported bypasses.
