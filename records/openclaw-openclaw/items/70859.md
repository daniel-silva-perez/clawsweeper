## Summary

Keep this PR open, but do not merge it as-is. The status-all snapshot-path idea is valid and current main already has the setup snapshot helper, but the branch carries stale and unrelated changes that would delete current changelog content and regress setup migration, plugin-install commit handling, and cold onboarding paths.

## What This Changes

The PR routes status/setup plugin compatibility notices toward `buildPluginCompatibilitySnapshotNotices(...)` and also changes changelog content, setup wizard flows, changed-test planning, provider-runtime tests, and several timeout/test-support surfaces.

## Best Possible Solution

Rebase the branch on current main and reduce it to the remaining status-all compatibility-path optimization, with focused status tests proving equivalent notices and no diagnostics load; leave setup import/migration, plugin-install commit handling, changed-gate behavior, and changelog history intact unless each has its own reviewed PR.
