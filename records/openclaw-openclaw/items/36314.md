## Summary

Keep open. Current main has adjacent Gateway resilience mechanisms, but it still does not provide the requested coherent process watchdog, operator-configurable global recovery policies, richer HTTP health contract, or generalized degraded startup mode; open #53716 is a narrower partial proposal for #53684 rather than a complete resolution of this broader request.

## What This Changes

Request to add built-in Gateway robustness features: crash-loop watchdog supervision, global error handling and recovery strategies, detailed health reporting, and degraded startup fallback.

## Best Possible Solution

Keep this as a Gateway resilience design item and split it into maintainer-approved follow-ups: process crash-loop supervision with bounded stop/notification behavior, explicit recovery-policy hooks for runtime/provider errors, an additive health/readiness schema plan, and a defined degraded startup contract that composes with existing diagnostics and config recovery.
