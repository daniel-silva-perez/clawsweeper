## Summary

Keep this open. Current main has adjacent reliability work, including channel health restarts, hardcoded unhandled-rejection/uncaught-exception handlers, readiness probes, last-known-good config recovery, and scoped degraded plugin startup. It still does not implement the requested first-class Gateway process watchdog, configurable 5-minute restart-window auto-stop policy, user-configurable global recovery strategies, richer health counters, or general degraded startup mode.

## Best Possible Solution

Keep this open as a maintainer-owned reliability umbrella, or split it into focused issues: Gateway process watchdog config/schema/docs and supervisor behavior; global error handling policy and whether user-configurable recovery strategies are desirable; privacy-safe health/readiness counters; and explicit degraded-startup semantics that build on existing last-known-good config recovery and channel health monitoring.
