## Summary

Keep open. Current main has adjacent Gateway resilience work, including channel-level health restarts, readiness/event-loop diagnostics, stability bundles, fatal-error logging, and narrow config recovery, but it still does not implement the requested coherent process watchdog, operator-configurable recovery policies, richer `/health` contract, or generalized degraded startup mode.

## What This Changes

The issue requests built-in Gateway robustness mechanisms: a process watchdog, global error-handling and automatic recovery strategies, enhanced health checks, and degraded startup behavior.

## Best Possible Solution

Keep this as a Gateway resilience design item and split it into maintainer-approved follow-ups: process crash-loop supervision with bounded stop/notification behavior, explicit recovery-policy hooks for runtime errors, additive health-schema expansion, and a defined degraded startup mode that composes with existing diagnostics and config recovery.

## Reproduction Assessment

Not applicable as an executable bug reproduction; this is a broad feature/design request. Source, docs, and test inspection on current main give high-confidence proof that existing mechanisms only partially cover the requested surface.

## Solution Assessment

No, a single automated implementation is not the best way to solve this. The maintainable path is maintainer scoping plus review of the narrower #53716 work before splitting concrete PRs.
