## Summary

Keep open: current main has meaningful adjacent resilience work, including channel health restarts, shallow HTTP probes, diagnostics/stability bundles, and global crash logging, but it still does not implement the broader Gateway-wide watchdog, configurable recovery-policy surface, richer `/health` contract, or general degraded startup mode requested here. The related #53716 remains open and targets the narrower #53684, so it is useful context but not a merged or complete resolution.

## What This Changes

Request built-in Gateway robustness mechanisms covering a process watchdog, global error recovery policies, enhanced health checks, and degraded startup behavior.

## Best Possible Solution

Keep this as a Gateway resilience design item and split it into maintainer-approved follow-ups: process crash-loop watchdog and notification semantics, configurable recovery hooks for runtime/provider/agent errors, additive health-schema expansion, and a defined degraded startup mode that composes with diagnostics, channel health monitoring, restart sentinels, doctor, and config recovery.
