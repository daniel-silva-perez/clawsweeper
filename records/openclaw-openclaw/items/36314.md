Keep open. Current main has adjacent Gateway resilience pieces, but it still does not implement the requested coherent process watchdog, operator-configurable recovery policies, richer HTTP health contract, or generalized degraded startup mode; the related #53684/#53716 work is narrower and still open rather than a merged resolution.

## What This Changes

The issue requests built-in Gateway robustness mechanisms: a process watchdog with crash-loop limits, global error recovery policies, richer health status, and degraded startup behavior.

## Best Possible Solution

Keep this as a Gateway resilience design item and split it into maintainer-approved follow-ups: process crash-loop supervision with bounded stop/notification behavior, explicit recovery-policy hooks for runtime/provider errors, an additive health/readiness schema plan, and a defined degraded startup contract that composes with existing diagnostics and config recovery.
