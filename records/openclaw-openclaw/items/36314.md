## Summary

Keep this open. Current main has adjacent reliability work, but it still lacks the requested first-class Gateway process watchdog, configurable restart-window/auto-stop policy, user-configurable global error recovery strategies, richer process/error counters in health output, and a general degraded-startup mode.

## Best Possible Solution

Keep this open as a maintainer-owned reliability umbrella, or split it into focused issues: define a gateway.watchdog config/schema/docs surface for process restart-loop detection and auto-stop/notification; decide whether configurable global error recovery strategies are desired; enrich Gateway health/readiness with privacy-safe process/error counters; and specify degraded-startup semantics that build on existing last-known-good config recovery, plugin-local degraded startup, diagnostics, and channel health monitoring.
