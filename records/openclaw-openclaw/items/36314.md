## Summary

Keep open: current main has several adjacent resilience mechanisms, but it still does not provide the requested built-in Gateway process watchdog, configurable global recovery strategies, richer health schema, and generalized degraded startup mode as one coherent product surface.

## What This Changes

The issue requests built-in Gateway robustness mechanisms: a whole-process watchdog, configurable global error recovery, richer health checks, and degraded startup behavior.

## Best Possible Solution

Keep this open as a Gateway resilience design item and split it into maintainer-approved follow-ups: process crash-loop watchdog with bounded stop/notification semantics, explicit recovery-policy hooks for agent/provider errors, additive health-schema expansion, and a defined degraded startup mode that reuses the existing channel monitor, diagnostics recorder, global handlers, doctor, and restart-sentinel paths.
