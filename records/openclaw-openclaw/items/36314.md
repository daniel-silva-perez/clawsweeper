## Summary

Keep open: current main has several adjacent robustness mechanisms, but it does not fully implement the requested built-in Gateway process watchdog, configurable global recovery strategies, expanded health schema, and generalized degraded startup mode as one product surface.

## What This Changes

The issue requests built-in OpenClaw Gateway robustness features: watchdog restart-loop detection, global uncaught error handling with recovery strategies, richer health checks, and degraded startup behavior.

## Best Possible Solution

Keep this open as a Gateway resilience design item and split it into narrower maintainer-approved follow-ups: whole-process crash-loop watchdog with bounded stop/notification semantics, explicit recovery policy hooks for agent/provider errors, health schema additions, and a defined degraded startup mode that reuses the existing channel monitor, diagnostics recorder, global handlers, and restart-sentinel paths.
