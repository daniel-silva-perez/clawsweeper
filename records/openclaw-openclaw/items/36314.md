## Summary

Keep open. Current main has adjacent Gateway resilience work, but it still does not provide the requested coherent process-level watchdog, operator-configurable global recovery policies, richer HTTP /health contract, or generalized degraded startup mode. The related open PR #53716 targets a narrower watchdog/startup-diagnostics slice for #53684 rather than closing this broader design item.

## What This Changes

The issue requests built-in Gateway robustness features: a process watchdog with crash-loop limits, global error recovery strategies, expanded health data, and degraded startup behavior.

## Best Possible Solution

Keep this as a Gateway resilience design item and split it into maintainer-approved follow-ups: a process crash-loop watchdog with stop/notification semantics, explicit recovery-policy hooks for agent/provider/runtime errors, additive health-schema expansion, and a defined degraded startup mode that composes with the existing diagnostics, service supervision, restart sentinel, doctor, and config recovery paths.
