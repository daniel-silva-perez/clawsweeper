Keep open: current main has several adjacent resilience mechanisms, but the requested built-in Gateway process watchdog, configurable recovery policies, richer health contract, and generalized degraded startup mode are still not implemented as a coherent product surface.

## What This Changes

Request to add built-in Gateway robustness features covering a process watchdog, global error handling with recovery strategies, enhanced health checks, and degraded startup behavior.

## Best Possible Solution

Keep this as a Gateway resilience design item and split it into maintainer-approved follow-ups: a process crash-loop watchdog with bounded stop/notification semantics, explicit recovery-policy hooks for agent/provider errors, additive health-schema expansion, and a defined degraded startup mode that reuses the existing diagnostics, channel monitor, restart sentinel, doctor, and config recovery paths.
