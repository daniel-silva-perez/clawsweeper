## Summary

Keep open. Current main has adjacent Gateway resilience work, but it still lacks the requested coherent process-level watchdog, operator-configurable global recovery policies, richer HTTP health contract, and generalized degraded startup mode; the related open PR is narrower and targets #53684 rather than resolving this broader design item.

## What This Changes

## Best Possible Solution

Keep this as a Gateway resilience design item and split it into maintainer-approved follow-ups: process crash-loop watchdog with bounded stop/notification semantics, explicit recovery-policy hooks for agent/provider/runtime errors, additive health-schema expansion, and a defined degraded startup mode that composes with existing diagnostics, channel monitor, restart sentinel, doctor, and config recovery paths.
