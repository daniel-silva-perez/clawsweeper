## Summary

Keep this PR open. Current main still lacks the requested TUI flag or equivalent option, and the current ChatLog path updates one assistant component in place rather than preserving intermediate stream snapshots as separate visible entries. Main does have some boundary-text retention in the stream assembler, but that is not the same user-facing preservation mode. The linked feature issue was closed as superseded by this PR, so closing the PR now would strand the remaining implementation work.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate. The best path is to revise it so the TUI preserves meaningful intermediate snapshots or content-block boundaries without per-token log churn, deduplicates unchanged final output, drops all preserved chunks when output is intentionally suppressed, documents the history-cap tradeoff, and adds focused ChatLog/event-handler/CLI smoke coverage before maintainer merge review.
