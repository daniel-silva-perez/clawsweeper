## Summary

Keep open. This is a fresh, detailed WhatsApp/OpenRouter embedded-runtime hang report against the latest released npm build, with a second user confirming the same symptom after upgrade. Current main has adjacent WhatsApp listener recovery, prompt-memory timeout, and Discord/session-key fallback changes, but none proves this WhatsApp inbound reply stall is fixed.

## Best Possible Solution

Keep this issue open for maintainer triage. Ask affected users to retest on current main or the next npm release that includes 7950a180, d55c7ea9, and e4ff7c16, then collect debug logs around before_agent_reply, before_prompt_build, model_call_started/model_call_ended, OpenRouter request timeout/abort handling, WhatsApp reconnect, and session idle transition. If reproduced, add a focused regression or live smoke around the WhatsApp inbound dispatch boundary and whichever layer is proven to never settle.
