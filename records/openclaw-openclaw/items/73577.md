## Summary

Keep open. Current main still has the operator-visible diagnostic gap: invalid snapshots can be restored from last-known-good while the successful recovery logs and recovery notice omit the original validation path/message. The reported top-level providers shape is unsupported, and related PR #39766 is still open for a narrower models.providers credential-only schema issue, but neither resolves the recovery UX problem.

## Best Possible Solution

Preserve the original invalid snapshot issues through successful last-known-good recovery and surface a concise rejected path/message in reload, startup, and the recovery notice. Add regression tests for invalid direct edits that recover successfully but still log the original validation reason. Separately decide whether credential/baseUrl-only `models.providers.<id>` entries should be accepted, likely through the existing #39766 thread or a narrower schema/docs follow-up.
