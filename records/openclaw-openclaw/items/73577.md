## Summary

Keep open. Current main still restores last-known-good config on invalid reload without surfacing the original validation issue path/message; the adjacent provider-config schema explains why the sample can be rejected, but it does not solve the operator-visible diagnostic gap.

## Best Possible Solution

Keep this issue open. The best fix is to preserve and surface the original `snapshot.issues` during last-known-good recovery in gateway reload and startup paths, and ideally include the same path/message in the recovery notice or a doctor/status clue. Separately coordinate with #39766 if maintainers want credential-only `models.providers.<id>` overrides to validate.
