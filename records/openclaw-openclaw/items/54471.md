## Summary

Keep open. Current main still does not relay ACP status/system progress through `streamTo: "parent"`, and the PR targets a real user-facing gap, but the branch needs maintainer updates before merge because the current ACP dispatch path drops runtime `status` events before the parent relay and the added relay branch does not count relayed system text as progress for stall detection. The diff is narrowly scoped and does not introduce an obvious security-sensitive surface.

## What This Changes

The PR adds a `system_event` branch to `startAcpSpawnParentStreamRelay()` so non-empty system-event text can be emitted to the parent session with a fallback ACP context key.

## Best Possible Solution

Land an in-place update that maps the actual ACP status/system progress source into the parent relay, forwards non-empty text as untrusted parent-session system events, treats that text as progress for stall/resume state, preserves existing assistant/commentary/lifecycle behavior, and adds focused relay coverage plus a user-facing changelog fix if merged.
