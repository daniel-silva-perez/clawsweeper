## Summary

Keep open. The PR targets a real ACP parent-relay gap, but the proposed patch is not sufficient on current main: it listens for an agent-event stream named `system_event` while current ACP runtime progress is surfaced as `AcpRuntimeEvent` type `status` and is dropped in `agent-command.ts` before an agent event is emitted. It also does not count the newly relayed messages as progress for stall detection.

## What This Changes

The PR adds a `system_event` branch to `startAcpSpawnParentStreamRelay()` so non-empty event text can be emitted to the parent session with a fallback ACP context key.

## Best Possible Solution

Update the branch so ACP runtime `status` events are translated into the parent relay contract, forward non-empty text as untrusted parent-session system events, treat that text as progress for stall/resume state, preserve existing assistant/commentary/lifecycle behavior, and add focused relay coverage plus a user-facing changelog fix if merged.
