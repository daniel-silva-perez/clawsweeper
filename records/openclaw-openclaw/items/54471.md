## Summary

Keep open. The PR targets a real ACP parent-relay gap, but the patch is not sufficient because current ACP progress and clarification text is surfaced as runtime `status` events that are dropped before `onAgentEvent`, while the added relay branch only listens for an agent-event stream named `system_event`. The branch also does not count relayed system text as progress for stall/resume state.

## What This Changes

The PR adds a `system_event` branch to `startAcpSpawnParentStreamRelay()` so non-empty event text can be emitted to the parent session with a fallback ACP context key.

## Best Possible Solution

Update the branch so the actual ACP runtime status/progress source is translated into the parent relay contract, forward non-empty text as untrusted parent-session system events, treat that text as progress for stall/resume state, preserve current assistant/commentary/lifecycle behavior, and add focused regression coverage plus a user-facing changelog fix if merged.
