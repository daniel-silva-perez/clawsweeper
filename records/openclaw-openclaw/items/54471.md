## Summary

Keep open. Current main still does not implement this PR's intended ACP parent relay behavior: `startAcpSpawnParentStreamRelay()` handles assistant deltas and lifecycle events, then returns for every other stream, so a `system_event` stream would still be dropped. The PR is a focused code-only fix candidate, but it should address the existing review feedback by counting relayed system events as progress so the no-output watchdog does not emit a false stall warning.

## Best Possible Solution

Keep this PR open or ask the author to update it against current main: add explicit `system_event` handling in `startAcpSpawnParentStreamRelay()`, count non-empty system-event text as relay progress, preserve `trusted: false` delivery through the existing `emit()` helper, and add focused tests for relay delivery plus stall suppression.
