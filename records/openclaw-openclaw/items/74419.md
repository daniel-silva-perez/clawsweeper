## Summary

Keep open. Current main still only exposes and forwards OpenAI-compatible `input_type` fields for memory embeddings, while the requested Jina `task` parameter is not present in config, inline embedding requests, batch indexing, cache identity, or docs. The linked PR #74444 is open and explicitly paired with this issue, so the right cleanup action is to review that implementation rather than close the issue before it lands.

## What This Changes

Allow OpenAI-compatible memory embedding requests to configure and forward Jina v5 task-specific adapter parameters, ideally with separate query and document task settings.

## Best Possible Solution

Land a narrow typed memory embedding task configuration, forward it as Jina's `task` parameter for query, document, and batch embedding paths, include task values in cache identity, and update schema/types/docs/tests/changelog before closing this issue through the linked PR or an equivalent merged fix.
