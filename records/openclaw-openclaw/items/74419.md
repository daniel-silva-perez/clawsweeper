## Summary

Keep open. Current main still has the requested feature gap: OpenAI-compatible memory embeddings can forward `input_type`, but there is no typed `task`, `queryTask`, or `documentTask` config surface and no `task` field in the OpenAI inline or batch embedding bodies. The issue is also paired with open PR #74444, so it should remain open until that PR, or an equivalent implementation, lands.

## What This Changes

Add support for configuring and forwarding Jina Embeddings v5 task-specific adapter parameters for memory embedding query and document requests.

## Best Possible Solution

Land PR #74444 or an equivalent typed implementation that adds query/document task config, propagates it through memorySearch resolution and embedding provider options, forwards `task` in both inline and batch OpenAI-compatible requests, includes task values in cache identity, and updates schema help, docs, tests, and changelog without weakening existing `input_type` behavior.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: configure the requested task fields and current `MemorySearchSchema` has no accepted keys, while the OpenAI embedding request construction has no path that emits `task`.

## Solution Assessment

Yes, the typed query/document task direction is the narrowest maintainable path for this feature. A generic `extraBody` passthrough may still be useful later, but it is broader and should be a separate product/API decision.
