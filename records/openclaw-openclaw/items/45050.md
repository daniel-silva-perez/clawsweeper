## Summary

Keep this open. Current main parses raw Codex/provider JSON into summarized text, but the reported Feishu-safe UX is still missing: `server_error` becomes provider-shaped user copy, error payloads can survive after visible output paths, and Feishu dispatch forwards the resolved text without a generic channel fallback.

## What This Changes

Feishu should turn transient provider `server_error` failures into generic user-facing copy, keep raw provider details in diagnostics, and only retry automatically before visible output is delivered.

## Best Possible Solution

Add a focused shared external-chat error presentation policy: transient/internal provider failures should become generic user copy, raw provider details and request IDs should stay in diagnostics, and final assistant error payloads after already-visible assistant content should be suppressed or genericized. Feishu should get a dispatcher-path regression while preserving the existing raw JSON parsing improvements from related raw-leak fixes.
