## Summary

Keep open. Current main still routes new direct messages according to `session.dmScope`, but it does not reconcile, retire, archive, or clearly mark existing peer-keyed direct-session rows after switching back to `main`; the cleanup command and docs still cover generic retention maintenance only.

## What This Changes

Switching `session.dmScope` back to `main` should reconcile, retire, mark, or safely clean up stale peer-keyed direct DM session entries.

## Best Possible Solution

Add a channel-agnostic session maintenance path that detects direct-session rows inconsistent with current main-scoped DM routing, previews them in dry-run output, and preserves history while retiring, archiving, or marking them inactive only under an explicit repair/enforce path; update CLI docs and focused tests for the cleanup behavior.

## Reproduction Assessment

Yes. Seed a session store with `agent:main:main` and `agent:main:telegram:direct:123` while config has `session.dmScope: "main"`; current routing resolves new DMs to main, and current cleanup has no dmScope-specific action for the stale row.

## Solution Assessment

Yes. A generic cleanup or doctor-style audit/repair with dry-run and history preservation is the narrow maintainable direction; silent startup deletion or a Telegram-only special case would be less safe.
