## Summary

Keep this issue open. Current main routes new Telegram DMs with `session.dmScope: "main"` to the canonical main session, but it still lacks the requested reconciliation, retirement, warning, or safe cleanup path for existing peer-keyed direct-session rows such as `agent:main:telegram:direct:*`. The discussion’s proposed direction remains valid; #45505 is related state-convergence work, not a canonical fix for this DM-session cleanup gap.

## Best Possible Solution

Keep this issue open and implement a dmScope-aware session health/cleanup path. The best fix is likely in session-store maintenance or doctor/sessions cleanup: detect direct peer-keyed rows for an agent/channel when current `session.dmScope` resolves DMs to main, show a dry-run plan, then mark/archive/retire those rows so routing and session listings no longer make them look active while preserving history.
