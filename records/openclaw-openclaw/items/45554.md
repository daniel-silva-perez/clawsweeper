## Summary

Keep open. Current main still handles `session.dmScope: "main"` as forward routing to the canonical main session, but the inspected routing, inbound session metadata, cleanup command, and docs do not reconcile, retire, warn about, or safely collapse existing peer-keyed direct-session rows.

## What This Changes

The issue requests a safe dmScope-aware reconciliation or cleanup path for stale peer-keyed direct DM sessions left behind after switching `session.dmScope` back to `main`.

## Best Possible Solution

Add a generic dmScope-aware session maintenance path: dry-run should detect peer-keyed direct rows that are inconsistent with the current `main` DM routing, and enforce/repair should archive, retire, or mark them inactive while preserving transcripts and avoiding Telegram-only special cases.

## Reproduction Assessment

Yes. A focused fixture can seed `sessions.json` with both `agent:main:main` and `agent:main:telegram:direct:123` while config has `session.dmScope: "main"`; current route code will choose main for new DMs and current cleanup code has no dmScope-specific action for the stale row.

## Solution Assessment

Yes. The narrow maintainable direction is a generic sessions cleanup or doctor-style audit/repair that preserves history; silent startup deletion or a Telegram-only special case would be a less safe solution.
