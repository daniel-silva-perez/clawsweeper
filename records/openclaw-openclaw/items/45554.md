## Summary

Keep open. Current main still only routes new direct messages according to the current `session.dmScope`; it does not reconcile existing peer-keyed direct-session rows, mark them inactive, or document a safe collapse/cleanup path, and the commits after the prior review did not touch this surface.

## What This Changes

The issue requests a safe dmScope-aware reconciliation, retirement, warning, or cleanup workflow for stale peer-keyed direct DM sessions left behind after switching `session.dmScope` back to `main`.

## Best Possible Solution

Add a channel-agnostic session maintenance path that detects direct-session rows inconsistent with current main-scoped DM routing, previews them in dry-run output, and preserves history while retiring or marking them inactive when enforcement is requested.

## Reproduction Assessment

Yes. Seed a store with `agent:main:main` and `agent:main:telegram:direct:123` while config has `session.dmScope: "main"`; current routing resolves new DMs to main, and current cleanup has no dmScope-specific action for the stale row.

## Solution Assessment

Yes. A generic cleanup or doctor-style audit/repair is the narrow maintainable direction; silent startup deletion or a Telegram-only special case would be less safe.
