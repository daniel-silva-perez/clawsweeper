## Summary

Keep open. Current main still routes new DMs with `session.dmScope: "main"` to the canonical main session, but I did not find a dmScope-change reconciliation, inactive marker, warning, or official cleanup path for existing peer-keyed direct-session rows such as `agent:main:telegram:direct:*`. The stale label should not drive closure because the reporter and follow-up comments describe a still-valid session-store lifecycle gap.

## Best Possible Solution

Keep this issue open and add a dmScope-aware session health/cleanup path. The best scoped fix is a generic direct-session audit used by `openclaw sessions cleanup` and/or doctor: when current config routes DMs to `main`, detect peer-keyed direct rows for the same agent, show them in dry-run output, and provide an explicit archive/retire cleanup that preserves transcripts and trajectory artifacts instead of silently deleting history.
