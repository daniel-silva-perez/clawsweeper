## Summary

Keep this PR open. Current main still lacks a google-vertex ADC marker-stripping stream wrapper and still has auto-reply paths that return no final reply for empty payload lists, but the submitted implementation needs revision because it replaces the existing Google thinking wrapper and does not preserve several intentionally empty successful delivery paths.

## What This Changes

The PR adds a Google provider stream wrapper and tests to strip the Vertex ADC marker, plus runReplyAgent guards and a regression test that convert zero or non-renderable final payloads into a friendly error.

## Best Possible Solution

Revise this PR, or replace it with a narrow maintainer-owned patch, so google-vertex ADC marker stripping composes with the Google thinking wrapper and the empty-payload handling only fails truly terminal non-silent runs that produced no delivered or renderable output.
