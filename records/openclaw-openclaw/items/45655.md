## Summary

Keep the issue open. Current main has broader media-generation tools and now accepts more input modality labels, but the reported core gap remains: custom `models.providers` entries do not declare output modality, selection/catalog paths carry only input metadata, and configured Poe-style provider models are routed as LLM stream models rather than image/video generation providers. The requested explicit rejection/diagnostic for media-output chat selections is still actionable.

## Best Possible Solution

Keep this open and make the text-chat provider boundary explicit: document `models.providers` as a text-output LLM/session surface, add schema/runtime diagnostics for media-output declarations or known unsupported selections, and route real Poe image/video generation through the existing image/video provider plugin APIs if someone contributes that integration.
