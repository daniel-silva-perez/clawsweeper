## Summary

Keep this issue open. Current main still does not wire `plugins.entries.openai.config.stt` into browser Talk realtime sessions, and the reported `talk.realtime.session` error is consistent with the current resolver/configuration path. The existing ClawSweeper comment on this issue already identified the same gap, and this review against `69fb7455c6223682a45dd8b8fb720bc3231bfa43` confirms it remains actionable rather than implemented, stale, or cleanly closeable as a ClawHub-only request.

## Best Possible Solution

Keep the issue open for maintainer follow-up. The best fix is to clarify and validate the Talk realtime configuration contract: document that `plugins.entries.openai.config.stt` and batch `tools.media.audio` STT do not configure browser Talk realtime, point users to `talk.providers.<provider>` or `plugins.entries.voice-call.config.realtime.providers.<provider>`, and decide whether local Whisper Talk should be provided through a dedicated realtime voice plugin using the existing plugin API.
