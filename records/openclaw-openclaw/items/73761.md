## Summary

Keep this issue open. Current main still does not wire `plugins.entries.openai.config.stt` into browser Talk realtime sessions, and the reported `talk.realtime.session` error matches the current resolver path. The remaining work is actionable: clarify/validate the Talk realtime config contract and decide whether local Whisper Talk support should be a realtime voice plugin or a documented unsupported batch-STT case.

## Best Possible Solution

Keep the issue open for maintainer follow-up. The best path is to clarify and enforce the Talk realtime configuration contract: document and/or validate that `plugins.entries.openai.config.stt` and batch `tools.media.audio` STT do not configure browser Talk, improve the `talk.realtime.session` error to point users at `talk.providers.<provider>` or `plugins.entries.voice-call.config.realtime.providers.<provider>`, and, if local Whisper Talk is desired, implement it as a dedicated realtime voice provider/plugin rather than overloading the OpenAI batch STT config.
