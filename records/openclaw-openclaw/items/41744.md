## Summary

Keep this issue open. Current main still has the reported `read` image variant where trusted image content without `details.path` cannot fall back to the original tool-call args, and the provided GitHub context shows open implementation PRs #46268 and #50359 covering that same remaining bug shape.

## Best Possible Solution

Keep this issue open and review or consolidate PR #46268 and #50359. The fix should pass stored `tool_execution_start` args into embedded media extraction, fall back to `path` / `file_path` / `filePath` only when trusted image content exists and no explicit media path is present, then preserve the resolved media through the existing pending-tool-media merge and Feishu `sendMediaFeishu` path with focused regression coverage.
