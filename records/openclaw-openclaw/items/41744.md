## Summary

Keep this issue open. Current main still preserves and delivers media once a path reaches pending tool media, but it does not implement the reported missing `details.path` fallback from the original `read` tool args. The provided GitHub context also shows open implementation candidates, including #46268 with closing syntax, so the issue should remain open until a fix is reviewed and merged.

## Best Possible Solution

Keep this issue open and review or consolidate #46268 and #50359. The fix should thread stored `tool_execution_start` args into embedded media extraction, fall back to `path` / `file_path` / `filePath` only when trusted image content exists and no explicit media path is present, then preserve that media through the existing pending-tool-media merge and Feishu `sendMediaFeishu` path with focused regression tests.
