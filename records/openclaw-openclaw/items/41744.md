## Summary

Keep this issue open. Current main still lacks the trusted read-image fallback from original tool-call args, and open implementation candidates #46268 and #50359 cover the same remaining bug shape.

## Best Possible Solution

Keep this issue open and review or consolidate #46268 and #50359. The fix should thread stored tool_execution_start args into embedded media extraction, fall back to path/file_path/filePath only when trusted image content exists and no explicit media path is present, then preserve the resolved media through the existing pending-tool-media merge and Feishu sendMediaFeishu path with focused regression tests.
