## Summary

Keep open. Current main still lacks the reported fallback from trusted `read` image tool output to the original tool-call args when `details.path` is absent, and open implementation PRs still reference this work.

## Best Possible Solution

Keep this issue open and review or consolidate #46268 and #50359. The fix should thread stored `tool_execution_start` args into embedded media extraction, fall back to `path` / `file_path` / `filePath` only when trusted image content exists and no explicit media path is present, then preserve that media through the pending-tool-media merge and Feishu `sendMediaFeishu` path with focused regression tests.
