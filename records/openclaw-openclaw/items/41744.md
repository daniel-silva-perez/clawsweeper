## Summary

Keep this issue open. Current main still handles structured media, MEDIA: text, and details.path, but it does not thread the original read tool args into media extraction for the missing-details.path case described in the later issue comments. The provided context also shows open implementation candidates #46268 and #50359, with #46268 using closing syntax for this issue, so this issue should stay open until a fix is reviewed and merged.

## Best Possible Solution

Keep this issue open and review or consolidate #46268 and #50359. The implementation should pass stored tool_execution_start args into embedded media extraction, fall back to path/file_path/filePath only when trusted image content exists and no explicit media path is present, then preserve that resolved media through the existing pending-tool-media merge and Feishu sendMediaFeishu path with focused regression tests.
