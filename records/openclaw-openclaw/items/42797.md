## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: default gateway `models.list` is not narrowed to authenticated or otherwise available providers when no explicit allowlist is configured. Main has a `view: "configured"` path used by some UI callers, but default `models.list` still returns the full catalog, and the TUI/macOS gateway model pickers still call the default path. The submitted PR also remains unsafe to land as-is because its auth heuristic misses credential paths that current main supports.

## Best Possible Solution

Keep this PR open for maintainer review, but require changes before merge. The implementation should use `hasAvailableAuthForProvider` or an equivalent shared gateway-safe availability helper, preserve the full-catalog fallback for truly unconfigured installs, and add focused coverage for env keys, auth profiles, config keys, Bedrock AWS SDK/default-chain auth, synthetic local-provider auth, and the TUI/native gateway picker paths. If a broader canonical implementation lands first and covers these surfaces, close this PR as superseded then.

## Risks / Open Questions

- Landing the submitted helper as-is could silently hide valid Amazon Bedrock models for users relying on AWS SDK credentials, `AWS_PROFILE`, or Bedrock bearer-token env auth.
- Landing the submitted helper as-is could hide usable local/custom providers that rely on synthetic local auth rather than an explicit API key.
- The broader open #59693 overlaps this area, so maintainers should coordinate the protocol/config direction before merging either approach.
