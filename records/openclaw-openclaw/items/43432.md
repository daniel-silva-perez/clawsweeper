## Summary

Keep this issue open. Current main partially improves memory durability with soft pre-compaction flushes and transcript-byte forced flushes, but it does not implement the requested token hard-threshold key, priority-aware compaction strategy, or embedding cache TTL. The related #42877 overlaps the broader hard-enforcement/memory-bloat theme, but it tracks a bounded memory write tool rather than these three config and runtime behaviors.

## Best Possible Solution

Keep this issue open as a real product/config request. The best path is to design the three behaviors explicitly: a token hard-flush threshold or equivalent guarantee in the memory flush pipeline, a compaction priority contract that works for built-in and plugin compaction providers, and a TTL-aware embedding cache invalidation path wired through schema, docs, tests, and cache queries. Coordinate with #42877 only for the shared hard-enforcement memory theme; it does not supersede this issue.
