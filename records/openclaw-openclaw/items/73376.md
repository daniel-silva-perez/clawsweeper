## Summary

Keep open. The report is fresh, concrete, and now corroborated by a second user on macOS. Current main has related post-release mitigations for local gateway reachability, paired probe auth, and the fast `status` path, but I found no source, test, changelog, or release evidence proving the combined `openclaw status` plus `openclaw cron list` hang from 2026.4.26 is fixed. The linked #73381 remains useful context, but it is an `agent` sendPolicy path, while the reported `status` and `cron.list` commands still use separate gateway probe/RPC paths.

## Best Possible Solution

Keep this issue open. Reproduce against current main or the next release with `openclaw status --debug --timeout 5000`, `openclaw status --json --timeout 5000`, and `openclaw cron list --json --timeout 5000`, then add a regression around the failing CLI-to-gateway path. If the root cause is probe/auth scope, extend the current gateway reachability fixes; if it is cron delivery preview or scheduler read blocking, bound that `cron.list` path or make preview resolution degradable.
