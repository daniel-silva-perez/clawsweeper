Keep open. The report is fresh, author is not protected, and there is no protected label, but current main does not provide enough source/test/history evidence to close it. Main has plausible partial mitigations after v2026.4.26, but `cron list` still uses the Gateway RPC path plus delivery-preview resolution, and the overlapping open #73381 tracks a narrower agent sendPolicy failure rather than this full CLI hang symptom.

## Best Possible Solution

Handle the narrower #73381 sendPolicy problem separately unless investigation proves it is the same root cause.
