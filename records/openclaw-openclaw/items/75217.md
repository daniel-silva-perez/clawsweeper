## Summary

Keep open. The PR addresses the linked runtime-injection bug directionally, but the current diff still bypasses the `agent:bootstrap` hook pipeline when `skipBootstrap` is true, which can drop explicitly configured hook-provided bootstrap context. It also leaves the CLI config schema metadata without a scope hint for `skipBootstrap`, despite the report calling out that schema gap.

## What This Changes

The PR adds a `skipBootstrap` guard to runtime bootstrap-file resolution and updates agent config docs, type JSDoc, and resolver tests for existing workspace bootstrap files.

## Best Possible Solution

Keep the PR open and adjust the resolver so `skipBootstrap` suppresses the default workspace bootstrap file load while still running `agent:bootstrap` hooks against an empty baseline. If maintainers accept the expanded runtime meaning, also add `skipBootstrap` label/help metadata and regenerated schema proof so `openclaw config schema` communicates the same contract.

## Reproduction Assessment

Yes. The linked bug is reproducible by keeping workspace bootstrap files on disk, setting `agents.defaults.skipBootstrap: true`, and observing current main still resolves them through `resolveBootstrapContextForRun`; the PR regression is reproducible by registering an `agent:bootstrap` hook that appends a file and seeing the new early return skip the hook.

## Solution Assessment

No, not yet. The shared resolver is the right place to fix runtime injection, but the guard should be applied to the built-in workspace-file source rather than bypassing the hook override seam, and the config schema metadata should be aligned with the new public behavior.
