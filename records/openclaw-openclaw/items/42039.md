## Summary

Keep open. Current main still exposes `maxMissedJobsPerRestart` and `missedJobStaggerMs` only as internal `CronServiceDeps`; the public cron config type, strict zod schema, generated JSON schema/help metadata, gateway wiring, and cron docs do not accept or document these settings. The prior related implementation PR is closed and unmerged in the provided GitHub context, and the related skip-missed-runs issue is also closed, so there is no active canonical replacement in the supplied thread context.

## Best Possible Solution

Implement a narrow, documented cron catch-up configuration surface in core. Add the chosen fields to `CronConfig`, zod validation, generated schema/help/labels metadata, gateway `CronService` pass-through, and public cron/config docs; preserve current defaults and add focused tests for config acceptance, gateway passthrough, `maxMissedJobsPerRestart: 0`, stagger timing, and interrupted-job skip behavior.
