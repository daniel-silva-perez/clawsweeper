## Summary

Keep this issue open. Current main still has `maxMissedJobsPerRestart` and `missedJobStaggerMs` only as internal `CronServiceDeps` knobs; they are not exposed through public cron config, strict zod validation, generated schema/help metadata, docs, or gateway wiring. The related implementation PR #42069 is closed and unmerged, and related issue #48345 is closed, so there is no active canonical replacement to close against.

## Best Possible Solution

Implement a narrow, documented cron catch-up configuration surface. Add the selected option set to `CronConfig`, zod validation, generated schema/help metadata, gateway `CronService` wiring, and public cron/config docs; preserve existing defaults and add focused tests for config acceptance, gateway passthrough, startup catch-up limits, stagger timing, and disabled/skipped catch-up behavior.
