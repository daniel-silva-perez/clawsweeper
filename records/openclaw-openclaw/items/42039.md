## Summary

Keep open. Current main still has `maxMissedJobsPerRestart` and `missedJobStaggerMs` only as internal `CronServiceDeps` controls, while the public `CronConfig` type, strict zod config schema, generated JSON schema, gateway construction, and cron/config docs do not expose them for `openclaw.json`. The exact implementation PR #42069 is closed and unmerged; open PR #41332 is only a partial adjacent proposal (`startupStaggerMs`) and does not close this issue. Open #73643 overlaps, but this issue is the older, more specific canonical tracker.

## Best Possible Solution

Keep this issue open as the canonical, specific tracker for public cron catch-up controls. Implement a narrow core config surface after choosing the product shape: either expose `maxMissedJobsPerRestart` and `missedJobStaggerMs` directly, add a clearer global/per-job missed-run policy, or combine both. The implementation should update `CronConfig`, strict zod validation, generated schema/help metadata, gateway `CronService` deps pass-through, public docs, and regression tests for config acceptance, `maxMissedJobsPerRestart: 0`, stagger timing, and interrupted/restart behavior. Reconcile partial PR #41332 and overlapping #73643 during review.
