## Summary

Keep this issue open. Current main still has the two requested cron catch-up knobs only as internal CronService dependency fields, not as public cron config accepted by openclaw.json, generated config schema, gateway wiring, or cron docs. The related implementation PR #42069 is closed and unmerged, and the related skipMissedRuns issue #48345 is also closed, so there is no active canonical replacement visible from the provided context.

## Best Possible Solution

Implement a narrow public cron catch-up configuration surface. Add either maxMissedJobsPerRestart plus missedJobStaggerMs, or a clearer skipMissedRuns/catch-up policy, to CronConfig, zod validation, generated config metadata/schema, gateway passthrough into CronServiceDeps, and user-facing cron docs. Preserve current defaults for compatibility, define 0/disabled semantics explicitly, and add focused tests for config acceptance, gateway wiring, startup catch-up limits, stagger timing, and skipped/deferred catch-up behavior. Consider per-job control only after maintainers decide whether announcement jobs and monitoring jobs need different policy.
