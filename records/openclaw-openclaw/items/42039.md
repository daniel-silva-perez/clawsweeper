## Summary

Keep open. Current main still has `maxMissedJobsPerRestart` and `missedJobStaggerMs` only as internal `CronServiceDeps` controls; the public cron config type, strict zod schema, generated schema, gateway wiring, and docs do not expose them for `openclaw.json`. The exact implementation PR #42069 is closed and unmerged, while open PR #41332 is only a partial adjacent stagger proposal and open #73643 overlaps without superseding this older, more specific tracker.

## Best Possible Solution

Keep this issue open as the canonical specific tracker for public cron missed-job catch-up controls. A complete fix should update `CronConfig`, strict zod validation, generated schema/help metadata, gateway `CronService` dep pass-through, public cron/config docs, and regression tests for config acceptance, `maxMissedJobsPerRestart: 0`, stagger timing, and restart/interrupted-run behavior. Reconcile PR #41332 and overlapping #73643 during review.
