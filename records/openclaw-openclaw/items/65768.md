## Summary

Keep open. Current main and the latest tagged release still exclude `MINIMAX_OAUTH_TOKEN` from the MiniMax managed `web_search` credential path, while `minimax-portal` OAuth is separate provider auth. #65785 is an open, unmerged implementation candidate with closing syntax, so this issue should remain open until that PR or an equivalent fix is resolved.

## Best Possible Solution

Keep this issue open and review #65785 or an equivalent patch. If MiniMax Search accepts portal OAuth tokens, update the MiniMax web search runtime resolver, provider envVars/contract metadata, manifest UI hints, tests, and docs to include `MINIMAX_OAUTH_TOKEN`. If Search intentionally requires a separate Coding Plan credential, update the docs and missing-credential message to say OAuth chat users still need a Coding Plan key for managed `web_search`.
