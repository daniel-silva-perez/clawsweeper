## Summary

Keep open. Current main still excludes MINIMAX_OAUTH_TOKEN from MiniMax managed web_search credential resolution, provider metadata, manifest hints, tests, and docs; the latest release v2026.4.26 has the same behavior. #65785 is already an open, unmerged implementation candidate with closing syntax for this issue, so this issue should remain open until that PR or an equivalent maintainer patch lands.

## What This Changes

Allow MiniMax managed web_search to reuse MINIMAX_OAUTH_TOKEN from minimax-portal OAuth instead of requiring a separate Coding Plan API key.

## Best Possible Solution

Keep this issue open and review #65785 or an equivalent maintainer patch. If MiniMax Search accepts minimax-portal OAuth bearer tokens, update the MiniMax web-search runtime resolver, provider/contract metadata, manifest UI hint, regression tests, and public docs to include MINIMAX_OAUTH_TOKEN. If it intentionally requires a separate Coding Plan credential, update the docs and missing-credential message so OAuth chat users understand the split.
