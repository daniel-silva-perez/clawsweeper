## Summary

Keep open. Current main still has the narrow Feishu Bitable cleanup predicate that only deletes records with missing or zero fields, while this PR targets the unhandled default-empty field case and adds focused regression coverage. The PR remains a plausible bundled Feishu plugin bug fix, not cleanup material.

## Best Possible Solution

Keep this PR open as a focused bundled Feishu plugin bug fix. The best maintainer path is to review and possibly harden the empty-value predicate, keep the new regression coverage, run the targeted Feishu Bitable test lane, and merge if CI confirms the behavior.
