## Summary

Keep open. The PR addresses the real diagnostic gap from #71867, and prior review feedback about missing labels and inherited-only duplication was addressed, but the latest head still disables the existing default-store OAuth refresh prompt whenever a non-default agent has a direct auth store.

## What This Changes

The branch changes doctor auth health reporting to enumerate direct per-agent auth stores, load them in read-only mode, suffix multi-agent auth diagnostics with agent IDs, export a direct auth-store source helper, add regression tests, and add a changelog entry for #71867.

## Best Possible Solution

Keep the PR's per-agent read-only diagnostics, but separate display labeling from refresh eligibility so the `agentDir: undefined` default/runtime store retains the existing interactive OAuth refresh path while non-default direct agent-store passes remain diagnostics-only.

## Reproduction Assessment

Yes. The diff gives a high-confidence static reproduction: with `main` and `coder` direct stores, the PR includes the default store as `{ agentId: main, agentDir: undefined }`, then line 307 forces `shouldRefresh` false because an agent label is present.

## Solution Assessment

No. The requested direction is right, but the current patch is not the best fix because it ties refresh eligibility to the display agent label; the safer boundary is to allow refresh only for the default/runtime `agentDir: undefined` pass and keep subagent passes read-only.

## Review Findings
