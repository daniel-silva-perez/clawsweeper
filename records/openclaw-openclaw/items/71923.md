## Summary

Keep open. The PR addresses a real doctor-auth labeling gap and incorporates earlier feedback, but the latest head still suppresses the existing default/runtime OAuth refresh prompt whenever a direct non-default agent auth store makes the default store part of the labeled multi-agent target list.

## What This Changes

The PR changes doctor auth health reporting to inspect direct per-agent auth stores in read-only mode, suffix multi-agent auth diagnostics with agent IDs, export a direct auth-store source check, add regression tests, and add a changelog entry for the linked doctor-auth issue.

## Best Possible Solution

Keep the per-agent read-only diagnostic labeling and inherited-store de-duplication, but separate display labeling from refresh eligibility so the `agentDir: undefined` default/runtime pass keeps the existing interactive OAuth refresh path while non-default `agentDir`-scoped passes remain diagnostics-only.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep the default auth refresh path active:** `src/commands/doctor-auth.ts:307-312`
  - body: When a non-default agent has a direct auth store, the target list includes the default/runtime store as `{ agentId: defaultAgentId, agentDir: undefined }`. Because this code disables refresh for any `params.agentId`, interactive `openclaw doctor` no longer offers the existing OAuth refresh for that default store in this multi-agent setup. Gate refresh on the scoped `agentDir` instead, so only non-default per-agent diagnostics stay read-only.
  - confidence: 0.86
