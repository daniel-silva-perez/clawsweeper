## Summary

Keep open. Current main still has both unresolved surfaces: `doctor` eagerly loads/parses the full session store before its state-integrity checks, and the cleanup paths I inspected do not recognize or remove stale `sessions.json.*.tmp` atomic-write leftovers. The report is concrete and recent, so the stale label should not drive closure.

## What This Changes

`openclaw doctor` should complete on long-lived installations with large session stores and should not be destabilized by accumulated stale `sessions.json.*.tmp` files.

## Best Possible Solution

Harden the existing session diagnostics first: make `doctor` cheap and non-fatal on oversized session stores, detect and safely clean stale same-directory `sessions.json.*.tmp` files, and teach session cleanup/disk-budget maintenance to account for those temp artifacts. Track any SQLite migration separately as a larger storage design item.

## Reproduction Assessment

Unclear. The issue provides concrete store sizes and current source shows the same heavy full-store path, but I did not execute an end-to-end large-store fixture in this read-only review.

## Solution Assessment

Yes for a narrow repair, not for an immediate SQLite rewrite. The maintainable first step is targeted doctor/session-cleanup hardening around oversized stores and stale atomic-write temp files; SQLite should remain a separate design discussion if the monolithic catalog continues to be a bottleneck.

## Repair Work Prompt

Fix openclaw/openclaw#56827 with a narrow doctor/session-maintenance repair. Do not implement a SQLite backend or broad session-store redesign. In `src/commands/doctor-state-integrity.ts`, scan the configured session-store directory before any expensive full-store work, detect regular same-directory stale temp files matching the active store basename pattern such as `sessions.json.<uuid>.tmp`, report their count/bytes, and offer a safe repair path for old temp files only. Make `openclaw doctor` complete on oversized `sessions.json` files by avoiding mandatory full catalog parsing when the store is above a conservative size threshold; emit actionable guidance and keep the existing recent/missing transcript checks for normal-size stores. In `src/config/sessions/disk-budget.ts` and/or `src/commands/sessions-cleanup.ts`, include stale session-store temp files as removable maintenance candidates so disk-budget cleanup can actually free them. Add focused tests for doctor warnings/repair prompts, stale temp-file cleanup, and disk-budget behavior. Update `docs/cli/doctor.md` and `docs/cli/sessions.md` only if the user-facing behavior changes.
