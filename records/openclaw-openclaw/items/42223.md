## Summary

Keep this PR open. Current main still has the active-tab exception that makes a manually collapsed Control UI sidebar group stay visible when the active tab is inside that group, so the PR’s intended behavior is not implemented on main yet.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase it onto current main, adapt the fix to the current `showItems`/`nav-section--collapsed` render path so manual group collapse hides children even when the active tab is inside the group, preserve `navCollapsed` icon-only sidebar behavior, and add a focused Control UI browser regression test in `ui/src/ui/navigation.browser.test.ts`.
