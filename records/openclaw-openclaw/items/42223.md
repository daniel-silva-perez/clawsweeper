## Summary

Keep this PR open. Current main still forces a sidebar group containing the active tab to render its child items even when the saved group collapse state is true, so the PR’s intended behavior is not implemented on main yet. The idea remains useful, but the patch needs rebasing from the older `nav-group--collapsed` path to the current `nav-section`/`showItems` render path.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase it onto current main, adapt the fix to remove the active-tab exception from the current `showItems`/`nav-section--collapsed` path while preserving `navCollapsed` icon-only sidebar behavior, and add a focused regression test in `ui/src/ui/navigation.browser.test.ts`.
