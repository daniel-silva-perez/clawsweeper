## Summary

Keep this PR open. Current main still has the active-tab exception that prevents a manually collapsed sidebar group from hiding children when the active tab is inside that group, and the submitted PR is a focused implementation candidate that needs rebase/adaptation to the current `nav-section`/`showItems` render path.

## Best Possible Solution

Keep this PR open as the implementation path. Rebase it onto current main, adapt the fix to the current `showItems`/`nav-section--collapsed` code so manual group collapse hides children even when the active tab is inside the group, preserve the icon-only collapsed-sidebar behavior, and add a focused Control UI browser regression test in `ui/src/ui/navigation.browser.test.ts`.
