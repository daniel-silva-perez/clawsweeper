## Summary

Keep this PR open. Current main still makes an active tab force its sidebar group open even when the group collapse state is true, so the PR's intended user-visible fix is not implemented on main. The contribution remains useful, but it now needs rebasing from the older nav-group class path to the current nav-section/showItems render path and should add focused regression coverage.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase it onto current main, adapt ui/src/ui/app-render.ts so a manually collapsed group hides its children even when it contains the active tab, preserve whole-sidebar navCollapsed icon-only rendering, and add a focused regression test in ui/src/ui/navigation.browser.test.ts.
