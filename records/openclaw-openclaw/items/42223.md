## Summary

Keep this PR open. Current main still makes an active tab force its sidebar group open even when `navGroupsCollapsed` says the group is collapsed, so the user-visible bug is not implemented on main yet. The contribution remains useful, but it needs rebasing from the older `nav-group--collapsed` markup to the current `nav-section`/`showItems` path and should add focused regression coverage.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase it onto current main, adapt the fix in `ui/src/ui/app-render.ts` so a manually collapsed group hides its children even when it contains the active tab, preserve whole-sidebar `navCollapsed` behavior, and add a focused browser regression test in `ui/src/ui/navigation.browser.test.ts`.
