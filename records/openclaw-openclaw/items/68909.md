## Summary

Keep open. Current main still returns `unknown` as soon as the user systemd probe fails and its stale-listener logic has no cgroup awareness, so the PR targets a real remaining Linux system-service gap. The patch is not mergeable as-is because it can select node-host as the gateway runtime and can hide real stale gateway listeners running under unrelated service cgroups.

## What This Changes

Adds a system-bus systemd runtime probe, cgroup-based stale PID filtering, status/hint output for system-level units, and tests for the new daemon and restart-health behavior.

## Best Possible Solution

Land a rebased implementation that probes the system bus for supported system-level gateway units, keeps node units as auxiliary status context, constrains cgroup dedup to known safe OpenClaw gateway/node sibling units, and preserves current restart-health behavior and tests.

## Reproduction Assessment

Yes. The PR body gives a concrete headless Linux setup with no user bus and separate gateway/node-host systemd cgroups, and current main's source shows the user-bus failure and PID-only stale listener paths that can be mocked without a live host.

## Solution Assessment

No. The direction is right, but the current patch should be narrowed so primary runtime selection is gateway-only and cgroup suppression only applies to known safe sibling OpenClaw units, then rebased onto current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restrict primary selection to gateway units:** `src/daemon/systemd-system-probe.ts:160-164`
  - body: `pickPrimaryGatewayUnit` iterates `buildGatewayCandidates`, but that list now includes `openclaw-node-host.service` and `openclaw-node.service`. On a host where only the node-host unit is active, `gateway status` can report the gateway as running with the node PID/cgroup, masking a stopped or missing gateway. Keep node units in the auxiliary `systemUnits` list, but select the primary runtime from gateway unit names only.
  - confidence: 0.9
- **[P2] Keep foreign service cgroups reportable:** `src/cli/daemon-cli/restart-health.ts:132-135`
  - body: This treats any two different `*.service` cgroups as definitively distinct. Since port classification marks any listener command containing `openclaw` as a gateway listener, an old gateway under `user@1000.service` or a custom service would disappear from `staleGatewayPids` when the runtime is under `openclaw-host-gateway.service`. Limit the bypass to known safe OpenClaw gateway/node sibling units.
  - confidence: 0.86
