## Summary

Keep this PR open. Current main still reads the exact configured logger file in `openclaw channels logs` and returns no lines when that file is missing, while the shared rolling-log fallback exists only on the generic `logs.tail` path. The latest PR context is a focused implementation candidate with tests and no security-sensitive file changes.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land the narrow branch if CI remains green: export the shared rolling-log resolver, call it from `channelsLogsCommand`, keep the rolling and custom non-rolling regression tests, and preserve the contributor credits. If maintainers decide channel-log fallback parity with `logs.tail` is not wanted, that should be an explicit product decision rather than automated cleanup.
