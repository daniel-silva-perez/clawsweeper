## Summary

Keep open. Current main still exposes `buffer`/`filename`/`contentType`/`mimeType` on `message.send`, but the shared send path and Telegram plugin still only route URL/path-style media. The issue also has open implementation candidates (#42674 and #63578), so this should remain open until one lands or maintainers choose an explicit rejection path.

## Best Possible Solution

Keep this issue open. The best maintainer path is to land or replace #42674/#63578 with a focused fix that either hydrates inline `buffer`/`filename`/`contentType`/`mimeType` through `message.send` into Telegram media delivery, or rejects those fields explicitly for `send` with a clear error and regression coverage.
