## Summary

Keep this issue open. Current main has useful Telegram forum-topic name support and persistence, but it still exposes only the current numeric topic plus current topic name; it does not provide an agent-visible topic-name-to-topic-id resolver, topic list action, forum_topics catalog, or name-based delivery target parsing for routing to another topic by human-readable name.

## Best Possible Solution

Keep this issue open and implement the missing capability in the Telegram plugin. Reuse the persisted topic-name cache, optionally allow explicit operator aliases, expose an agent-visible topic catalog or resolver action with topic_id/title/source/freshness, route only exact unique matches, and fall back to numeric topic ids or clarification on ambiguity. Add focused tests for listing/resolution, duplicate names, stale or closed topics, persistence, and send routing, then update the public Telegram channel and message docs.
