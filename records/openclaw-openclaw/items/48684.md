## Summary

Keep open. Current main still routes the shared inbound debounce bypass through `hasControlCommand()`, which intentionally recognizes slash/text-command aliases but not bare abort triggers such as `stop`, `abort`, or `wait`. The documented out-of-band abort behavior and abort-trigger primitives still exist, but the debounced channel path has not been updated; the related fix PR was closed unmerged.

## Best Possible Solution

Keep this issue open and land a targeted fix: make `shouldDebounceTextInbound()` use `isControlCommandMessage()` or equivalent abort-aware detection, add regression tests for bare `stop` and `wait`, and audit direct debounce gates in channel-specific code so bare abort triggers bypass debounce consistently across debounced channels.
