## Summary

Keep open. Current main still has the reported debounce bug: the shared inbound debounce policy bypasses only slash/text control commands through `hasControlCommand()`, while bare abort phrases are recognized through the separate abort-aware `isControlCommandMessage()`/`isAbortTrigger()` path. The prior fixing PR #48916 is closed unmerged, and current tests still do not cover bare abort debounce bypass.

## What This Changes

The issue requests that bare abort phrases such as `stop`, `abort`, and `wait` bypass inbound message debounce immediately instead of being delayed or merged with later text.

## Best Possible Solution

Land a narrow fix that changes the shared debounce command check to the abort-aware `isControlCommandMessage()`, adds regression tests for bare `stop`, `abort`, and `wait`, and audits direct plugin debounce predicates that still call `hasControlCommand()` so documented command-bypass behavior is consistent without changing normal text/media debounce semantics.
