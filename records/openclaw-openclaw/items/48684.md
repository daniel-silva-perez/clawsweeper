## Summary

Keep open. Current main still has the reported debounce gap: the shared inbound debounce policy bypasses slash/text commands through `hasControlCommand()`, while the existing abort-aware helper and abort trigger set recognize bare `stop`, `abort`, and `wait`. The linked fixing PR #48916 is closed unmerged, and the current debounce policy test still lacks bare abort regression coverage.

## What This Changes

The issue asks for bare abort phrases such as `stop`, `abort`, and `wait` to bypass inbound channel debouncing instead of being delayed or merged as normal text.

## Best Possible Solution

Use the existing abort-aware command detection for debounce bypass, cover bare `stop`, `abort`, and `wait` with regression tests, and update plugin-local debounce predicates so documented abort behavior is consistent while preserving explicit channel exceptions such as BlueBubbles DM coalescing.
