## Summary

Current main still has the reported debounce gap: shared inbound debouncing bypasses slash/text commands through `hasControlCommand()`, while the abort-aware helper already recognizes standalone abort phrases such as `stop`, `abort`, and `wait`. The linked fix PR is closed unmerged, and debounce-boundary regression coverage is still missing.

## What This Changes

The issue asks for bare abort phrases such as `stop`, `abort`, and `wait` to bypass inbound message debouncing the same way `/stop` does.

## Best Possible Solution

Use abort-aware command detection for shared and plugin-local debounce bypasses, preserve the documented BlueBubbles coalescing opt-in, and add focused regression tests proving bare abort phrases flush immediately while normal text still debounces.
