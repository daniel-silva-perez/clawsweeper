## Summary

Keep open. Current main still has the reported debounce gap: `shouldDebounceTextInbound()` bypasses debounce only for `hasControlCommand()`, while the abort-aware helper and abort trigger tests already recognize standalone phrases such as `stop`, `abort`, and `wait`. The related fix PR #48916 is closed unmerged, so there is no canonical merged fix to rely on.

## What This Changes

The issue requests that standalone abort phrases such as `stop`, `abort`, and `wait` bypass inbound message debouncing the same way slash commands do.

## Best Possible Solution

Use abort-aware command detection for shared and plugin-local debounce bypass decisions, keep normal text/media debounce behavior unchanged, preserve BlueBubbles’ documented coalescing opt-in, and add regression tests that prove bare abort phrases flush immediately.
