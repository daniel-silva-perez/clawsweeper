## Summary

Keep open. Current main still has the reported bug: the shared inbound debounce policy bypasses only slash/text control commands via `hasControlCommand()`, while bare abort phrases such as `stop`, `abort`, and `wait` are recognized through the separate abort-aware path. The prior fix PR #48916 is closed unmerged, and current tests still do not cover bare abort debounce bypass.

## Best Possible Solution

Keep this issue open and land a narrow fix that changes the shared inbound debounce command check to the abort-aware `isControlCommandMessage()`, adds regression tests for bare `stop`, `abort`, and `wait`, and audits direct channel debouncer predicates that still use `hasControlCommand()`. Preserve normal text debouncing, media flush behavior, slash-command behavior, Telegram bot-name command normalization, and the documented BlueBubbles same-sender DM coalescing exception.

## Clownfish Work Prompt

Fix the inbound debounce abort-trigger bug tracked by https://github.com/openclaw/openclaw/issues/48684. Current `src/channels/inbound-debounce-policy.ts` uses `hasControlCommand()` inside `shouldDebounceTextInbound()`, so bare abort phrases such as `stop`, `abort`, and `wait` remain debounceable and can be merged/swallowed. Change the shared policy to use the abort-aware `isControlCommandMessage()` while preserving existing behavior for blank text, media, slash commands, Telegram bot-name command options, and `allowDebounce=false`. Add regression tests in `src/channels/inbound-debounce-policy.test.ts` for bare abort phrases (`stop`, `abort`, `wait`, and at least one multi-word phrase such as `stop run`) plus normal text staying debounceable. Preserve the documented BlueBubbles `coalesceSameSenderDms` behavior: same-sender DM control commands may still wait inside that explicit coalescing path, but bare abort triggers outside that exception should bypass debouncing consistently. Include a user-facing changelog fix entry with appropriate attribution, for example Thanks @sherly-techtree. Do not broaden command parsing beyond the existing abort-aware helper and do not change debounce timing defaults.
