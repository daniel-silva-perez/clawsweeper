## Summary

Keep open. Current main still lacks the interactive `/context viz` visualizer, so the feature is not implemented, but this PR is not merge-ready because it is dirty against current main and the proposed handler calls a backend method that the current `TuiBackend` contract does not expose.

## What This Changes

The PR adds a `/context visualize`/`/context viz` TUI overlay with context bars, sparkline history, drill-down rows, slash-command help/completions, and a gateway-client `sessions.usage` context-report fetch.

## Best Possible Solution

If maintainers want this feature, rebase it on the current `/context` selector, add a typed context-report capability across the relevant TUI backends or reuse an existing report path, render clear empty/error states, and add focused command/component coverage plus manual terminal proof before documenting the new subcommand.

## Reproduction Assessment

Yes for the blocking review issue: the PR calls `client.getContextReport(...)` from a value typed as `TuiBackend`, while current main's `TuiBackend` has no such method. The UI empty-state issue is also reproducible by a successful `sessions.usage` response with `contextWeight: null`.

## Solution Assessment

No. The visualizer direction is plausible, but the current PR is not the safest merge shape until it is rebased onto current main, the backend contract is made explicit, and valid no-report sessions stop looking like permanent loading.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Add the context report method to the TUI backend contract:** `src/tui/tui-command-handlers.ts:252`
  - body: `client` is typed as `TuiBackend`, but this PR only adds `getContextReport()` to `GatewayChatClient`. Current `TuiBackend` does not declare that method, so this call fails typechecking and the local embedded backend has no corresponding implementation. Add a typed backend method across the contract and backends, or fetch the report through an existing typed path.
  - confidence: 0.94
- **[P2] Render an empty state when no context report exists:** `src/tui/components/context-viz-overlay.ts:70-73`
  - body: `sessions.usage` can legitimately return `contextWeight: null` for a fresh or reset session. `refresh()` ignores that value and leaves `this.report` unset, so the overlay stays on `Loading context data...` after a successful request. Track a loaded-empty/error state or fall back to an estimated report so users are not left in a dead-end UI.
  - confidence: 0.88
