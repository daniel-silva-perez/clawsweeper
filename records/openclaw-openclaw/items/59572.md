## Summary

Keep open. Current main does not implement the PR's `/context visualize` or `/context viz` TUI overlay; it only opens a list/detail/json selector for bare `/context` and forwards context subcommands to the Gateway. The PR is still a plausible implementation candidate, but it needs normal review against current main and the remaining overlay behavior concerns.

## Best Possible Solution

Keep this PR open as the implementation candidate for TUI context visualization. The best path is to rebase it onto current main's existing `/context` selector behavior, keep `/context list|detail|json|help` forwarding intact, add/finish targeted tests for `/context visualize` and `/context viz`, address the null-report empty state and polling/load concerns, then have a maintainer review the TUI UX before merge.
