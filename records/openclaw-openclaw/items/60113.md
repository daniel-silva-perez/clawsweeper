## Summary

Keep open. Current main has some later Bedrock transcript hardening for empty assistant error turns, but it does not implement the issue’s requested stack capture and still leaves the shared transport text sanitizer as a direct `text.replace(...)` call with no runtime null/undefined guard. The closed related PR #60157 was not merged, so there is no canonical merged fix for this issue yet.

## Best Possible Solution

Keep this issue open and implement a focused hardening patch: preserve a redacted stack/cause from transport failures into `embedded run agent end` diagnostics, make shared transport text sanitization robust to nullish or malformed text inputs at runtime, and add regression coverage for malformed assistant/tool text blocks plus the lifecycle log metadata. The closed PR #60157 can be used as background, but the final fix should be reconciled with the newer empty-assistant replay repair already on main.
