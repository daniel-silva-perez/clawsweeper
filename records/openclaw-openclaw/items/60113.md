## Summary

Keep open. Current main has adjacent Bedrock transcript hardening, but it still does not satisfy the reported remaining problem: the shared transport sanitizer can still dereference malformed runtime text with `text.replace(...)`, and the embedded run-end lifecycle warning still records only derived message/raw-error metadata rather than a captured stack or cause.

## What This Changes

The issue asks for embedded-agent `.replace()` crashes during Bedrock/failover/tool-error runs to be handled gracefully and for `embedded run agent end` diagnostics to include stack-level failure detail.

## Best Possible Solution

Land a focused agent transport hardening patch that coerces or rejects malformed text before shared transport serialization, preserves valid string behavior, and carries a redacted, bounded stack/cause observation into embedded run-end diagnostics without undoing the existing Bedrock empty-assistant replay repair or silent-reply semantics.
