## Summary

Keep open. Current main has later Bedrock replay hardening for empty assistant error turns, but it still does not implement the issue’s requested stack capture in the `embedded run agent end` diagnostic, and the shared transport text sanitizer still assumes a string at runtime. The related fixing PR #60157 is closed and unmerged, so there is no canonical shipped fix for this issue yet.

## Best Possible Solution

Keep this issue open and implement a focused hardening patch: capture a redacted stack or cause from transport/runtime failures into the embedded run-end diagnostic, make shared transport text sanitization resilient to nullish or malformed runtime text, and add regression coverage for malformed assistant/tool text blocks plus lifecycle log metadata. Reconcile the patch with the existing #71572 empty-assistant replay repair rather than replacing that shipped behavior.
