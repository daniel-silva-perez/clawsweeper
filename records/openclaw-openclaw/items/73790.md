## Summary

Keep this PR open. Current main still has the unguarded runtime-model persistence in both the direct agent path and isolated cron path, and the PR discussion shows the follow-up commit addressed the Greptile live-switch edge case. The remaining maintainer question is product semantics: current docs intentionally keep auto fallback overrides selected via modelOverrideSource="auto", so review should confirm this PR only changes transient runtime model fields and does not conflict with that policy.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is a focused runtime-field fix, if accepted, that prevents automatic fallback provider/model values from being saved as durable runtime model fields while preserving documented auto fallback override behavior via providerOverride/modelOverride/modelOverrideSource="auto". Validate direct agent fallback, isolated cron fallback, primary-success persistence, and explicit live model switches; update docs/changelog only if the selected-model fallback policy changes.
