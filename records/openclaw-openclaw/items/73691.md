## Summary

Keep open. Current main still has the memory-core promotion path appending new deep-phase promotion sections to `MEMORY.md` without a durable disk-side size budget, compaction, or archive layer, and the linked closing PR #74088 remains separate from `main`.

## What This Changes

The report asks for dreaming/deep promotion writes to keep `MEMORY.md` bounded so bootstrap injection limits are not exceeded after long-term use.

## Best Possible Solution

Land PR #74088 after review, or replace it with an equivalent memory-core fix that enforces a durable pre-write budget for promoted sections, preserves user-authored `MEMORY.md` content, and adds regression coverage for near-limit and over-limit promotion runs.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partially: repeated promotion candidates on current main will append new promotion sections without a disk budget, which reproduces the unbounded file-growth portion. The full Gateway freeze/session-write-lock timeout remains unverified because the report includes no logs or focused freeze reproduction.

## Solution Assessment

Is this the best way to solve the issue? Yes on boundary: the narrow maintainable fix belongs in bundled `memory-core` promotion writing, not in bootstrap truncation alone. The exact policy should be settled by reviewing #74088 or an equivalent replacement before this issue closes.
