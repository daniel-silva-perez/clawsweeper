## Summary

Keep open. Current main still appends deep dreaming promotions into root `MEMORY.md` without a durable size budget, compaction step, or archive layer. Bootstrap injection is capped on current main, so the exact overflow-to-session-lock timeout path is not proven, but the central on-disk growth bug remains in the bundled `memory-core` owner surface.

## Best Possible Solution

Keep this issue open and fix it in the bundled `memory-core` owner surface. Add a durable active `MEMORY.md` budget before deep promotion writes, choose a preservation strategy for overflowed material (compaction, archive layer, or structured active/history split), surface oversized root memory in status/doctor, and add regression tests for near-limit and over-limit scheduled and manual promotion runs. Treat the Gateway freeze/session-lock timeout as a verification point that still needs logs or a focused repro.
