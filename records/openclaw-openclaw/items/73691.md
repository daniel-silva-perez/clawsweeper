## Summary

Keep open. Current main caps bootstrap injection, which weakens the exact overflow explanation, but the central product gap remains: memory-core deep/dreaming promotion still appends to root MEMORY.md without a durable file-size budget, compaction path, or archive layer.

## Best Possible Solution

Keep this issue open and fix it in memory-core: add a pre-write MEMORY.md budget check for deep promotion, choose a preserving compaction or archive layer for older promoted entries, surface oversized root memory in memory status or doctor, and add regression tests for near-limit and over-limit promotion runs. Separately verify the session-write-lock/Gateway freeze claim with logs or a focused reproduction.
