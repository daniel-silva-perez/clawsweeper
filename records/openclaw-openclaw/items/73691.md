## Summary

Keep open. Current main still has the memory-core deep promotion path appending to root MEMORY.md without a durable disk-side budget, compaction, or archive layer. PR #74088 is an open, unmerged fix candidate with closing syntax, so this issue should stay open until that PR, or an equivalent replacement, lands.

## What This Changes

The issue requests a durable bound for memory-core dreaming promotions so MEMORY.md cannot grow past bootstrap-safe size limits and wedge Gateway startup or session handling.

## Best Possible Solution

Land PR #74088 after review, or replace it with an equivalent memory-core fix that applies a durable budget before promotion writes, preserves user-authored MEMORY.md content, and adds regression coverage for near-limit and over-limit promotion runs. Gateway freeze verification can remain a follow-up if the disk-growth bug is fixed first and the lock-timeout path still lacks logs.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partially: the unbounded MEMORY.md growth is directly reproducible from the current append-only applyShortTermPromotions path and repeated dreaming promotions, but the full Gateway freeze/session-write-lock timeout still lacks logs or a focused current-main reproduction.

## Solution Assessment

Is this the best way to solve the issue? Unclear until PR #74088 is reviewed, but fixing the bundled memory-core promotion writer is the right boundary; the safest solution should bound dreaming-owned sections while preserving user-authored durable memory.
