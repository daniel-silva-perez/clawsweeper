## Summary

Keep open. Current main still has the bundled memory-core dreaming promotion path appending to `MEMORY.md` without a durable file budget, compaction step, or archive layer, and the paired fix PR is still open and unmerged.

## What This Changes

The issue asks OpenClaw to bound `MEMORY.md` growth from dreaming/deep promotion so bootstrap context stays within limits and Gateway sessions do not wedge after long-running use.

## Best Possible Solution

Land PR #74088 after review, or replace it with an equivalent memory-core fix that applies a durable budget before promotion writes, preserves user-authored `MEMORY.md` content, and adds regression coverage for near-limit and over-limit promotion runs.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partially: the unbounded `MEMORY.md` growth is directly reproducible from the current append-only `applyShortTermPromotions` path and repeated dreaming promotions, but the full Gateway freeze still lacks logs or a focused current-main reproduction.

## Solution Assessment

Is this the best way to solve the issue? Unclear until PR #74088 is reviewed, but fixing the bundled memory-core promotion writer is the right boundary; the safest solution should bound auto-promoted sections while preserving user-authored durable memory.
