## Summary

Keep this issue open. Current main still has the reported TUI session-title precedence and sanitization gap, and PR #39790 is an open, unmerged implementation candidate that explicitly references this issue with closing syntax.

## Best Possible Solution

Keep this issue open until PR #39790 or an equivalent maintainer-reviewed fix lands. The fix should pass the computed outer displayName into title derivation, strip inbound metadata before first-user-message and last-message-preview extraction, preserve subject/message/session-id fallbacks, and add regression tests for external displayName precedence plus metadata stripping.
