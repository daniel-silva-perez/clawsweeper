## Summary

Keep this issue open. Current main still has the reported session-title precedence and metadata-sanitization gap, and the provided GitHub context shows PR #39790 is still open/unmerged with explicit closing syntax for this issue.

## Best Possible Solution

Keep this issue open until PR #39790 or an equivalent maintainer-reviewed fix lands. The fix should pass the computed outer displayName into title derivation, strip inbound metadata before first-user-message and last-message-preview extraction where appropriate, preserve existing subject/message/session-id fallbacks, and add regression tests for external displayName precedence plus metadata stripping.
