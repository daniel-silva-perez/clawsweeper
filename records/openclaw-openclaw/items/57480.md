## Summary

Keep this PR open. Current main does not implement the PR’s intended behavior: `patternProperties` is still stripped for Gemini/Google paths and for explicitly configured `modelCompat.unsupportedToolSchemaKeywords`, but not by default for BytePlus/Volcengine or other non-Anthropic OpenAI-compatible providers. The PR is still a plausible implementation candidate, though it needs porting to the current refactored helper rather than the older wrapper location.

## Best Possible Solution

Port the intended fix to current main’s schema-normalization path, preferably in `normalizeToolParameterSchema()` or via BytePlus/Volcengine/OpenAI-compatible model compat defaults, then add the PR’s regression coverage for non-Anthropic stripping and Anthropic preservation. Review the scope carefully before applying a universal non-Anthropic default, and keep #57443 linked until the implementation lands.
