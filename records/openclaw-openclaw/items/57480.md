## Summary

Keep this PR open. Current main still does not implement the PR’s intended default behavior: `patternProperties` is stripped for Gemini/Google and for explicitly configured `modelCompat.unsupportedToolSchemaKeywords`, but not by default for BytePlus, Volcengine/Doubao, or other non-Anthropic OpenAI-compatible providers. The prior discussion usefully ties the problem to #57443 and BytePlus Ark 400s; this PR remains a plausible implementation candidate, but it needs porting to the current refactored helper and a maintainer scope decision.

## Best Possible Solution

Port the intended fix to the current schema-normalization path, preferably in `normalizeToolParameterSchema()` or as BytePlus/Volcengine/OpenAI-compatible model compat defaults, then add regression coverage for non-Anthropic stripping and Anthropic preservation. Keep #57443 linked until a current-main implementation lands, and review the universal-vs-provider-scoped policy before merging.
