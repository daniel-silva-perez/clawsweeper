## Summary

Keep this PR open. Current main still has the cron/main-session behavior the PR is trying to remediate: cron event text is embedded into relay-style heartbeat prompt prose, main-session systemEvent payloads are accepted without text-shape validation, and current docs/tool guidance still presents rich main-session reminder prose. The PR is not implemented on main and is not safely superseded; its no-relay and legacy-job editability review comments still need maintainer follow-up before landing.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to rebase or adopt it onto current main, replace main-session cron reminder prose with short internal wake tokens, constrain or safely degrade rich main-session systemEvent text, preserve isolated agentTurn reminder flows, keep no-relay runs strict HEARTBEAT_OK/internal-only, allow benign updates to legacy rich main-session jobs, update docs/tool guidance, and add focused regression coverage for those paths.
