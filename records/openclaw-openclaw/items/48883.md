## Summary

Current main does not implement this PR's intended cron/main-session remediation. Cron heartbeat events still build rich relay-style reminder prompts, main-session systemEvent payload text is still accepted and enqueued verbatim, and the agent/tool/docs guidance still allows rich main-session reminder text. The PR remains a plausible implementation candidate, but the open functional review comments around no-relay handling and legacy-job edits should be resolved before landing.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to rebase or adopt it onto current main, preserve isolated agentTurn reminder flows, constrain main-session cron/systemEvent text to short internal wake tokens, update the agent/tool/docs guidance, resolve the no-relay and legacy-job review comments, and land with focused regression tests for tokenized main-session cron prompts and legacy job operations.
