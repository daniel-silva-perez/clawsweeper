## Summary

Keep this issue open. Current main still does not expose `cron.defaults.delivery` or an equivalent primary cron announce delivery default. The only global cron delivery-like config is failure-notification-specific, and job creation still preserves explicit delivery or falls back to `{ mode: "announce" }` without a channel/target default. The newer #57638 is broader and still open, while #41164 was closed back to this narrower request, so this remains useful as the focused delivery-default tracker.

## Best Possible Solution

Keep this issue open as the focused product issue for `cron.defaults.delivery`. A complete fix should add the config type, strict zod schema, generated schema/help/docs, gateway/service normalization, CLI and Control UI support, plus regression tests for precedence: per-job delivery first, then `cron.defaults.delivery`, then the existing fallback behavior. Coordinate with #57638 if maintainers decide to solve broader cron defaults in one design.
