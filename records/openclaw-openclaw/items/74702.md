Keep open. The issue asks OpenClaw to restore or provide a migration path for the missing `openclaw/plugin-sdk/zalouser` subpath so `@larksuite/openclaw-lark` can load after OpenClaw 2026.4.27.

## Best Possible Solution

Restore the shipped compatibility surface, preferably as a deprecated `openclaw/plugin-sdk/zalouser` facade that forwards only the already-generic helpers needed by existing packages, then document the generic replacement and keep the SDK export/API baselines aligned.
