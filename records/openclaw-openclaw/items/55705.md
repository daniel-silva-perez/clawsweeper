## Summary

Keep this issue open. Current main still has the stale Control UI sessions cache path: `loadSessionsOnce()` requests `sessions.list` as `SessionsListResult | undefined` but only updates `state.sessionsResult` inside `if (res)`, so a successful empty/undefined response can leave the previous phantom count visible. An open linked PR, #55831, uses closing syntax for this issue, so this should stay open until that implementation is revised and merged or replaced.

## What This Changes

The issue reports that the Control UI shows a nonzero active Subagent/session count while API/CLI session listing is empty, with a possibly stale UI version string also visible.

## Best Possible Solution

Revise and land the linked UI controller fix so a successful empty/undefined `sessions.list` response clears `sessionsResult` to `null`, while preserving the current last-known-good cache behavior for generic errors and the existing missing-scope clear path. Add focused controller tests for undefined-success clearing and transient-error preservation.
