## Summary

Current main still has the reported TUI display-corruption path: assistant-visible text is routed through `sanitizeRenderableText`, which inserts literal spaces into long unprotected tokens, and the current copy-sensitive guard still does not protect hyphen-only package names. The issue also has an open unmerged closing PR (#69340), so it should remain open until that PR is reviewed and either merged or closed.

## Best Possible Solution

Keep this issue open while #69340 is reviewed. The best fix is to preserve copy-sensitive code/table content in the TUI without injecting literal display spaces, with focused regression coverage for the reported hyphenated package-name case and fenced code blocks; if #69340 lands and proves the central user problem is solved, close this issue as implemented by that PR.
