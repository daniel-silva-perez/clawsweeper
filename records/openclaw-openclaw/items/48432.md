## Summary

Keep this issue open. Current main still has the reported TUI long-token behavior for hyphen-separated code tokens, and an unmerged PR already references this issue with closing syntax, so the right cleanup path is to review and either land or close that PR rather than closing the issue early.

## Best Possible Solution

Review PR #69340 against the current TUI formatter behavior, require a focused regression test for long hyphen-separated package names in code/TUI-rendered text, and land it if the narrow sanitizer change is correct; otherwise keep this issue open and replace it with a better fix in `src/tui/tui-formatters.ts` plus coverage in `src/tui/tui-formatters.test.ts`.
