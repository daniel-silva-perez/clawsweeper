## Summary

Keep open. Current main still has the sanitizer behavior that can inject literal spaces into long hyphenated TUI tokens, and the issue is paired with open unmerged PR #69340, so it should wait for maintainer review or a replacement fix.

## What This Changes

The issue reports that TUI-rendered code and table blocks with long unbroken tokens receive literal spaces at wrap points, corrupting displayed content.

## Best Possible Solution

Preserve copy-sensitive fenced code and table content in the TUI without injecting literal display spaces, then land or replace the open PR with focused regression tests for the hyphenated package-name example and the code/table rendering boundary.

## Reproduction Assessment

Yes. A code-level reproduction against current main shows the current sanitizer turns a 33-character hyphenated package token into a token with an inserted literal space, and assistant TUI content still flows through that formatter path.

## Solution Assessment

Unclear. PR #69340 is a plausible narrow fix for the reported package-name example, but the comments also cover fenced code blocks and box-drawn tables, so the safest fix should be validated against those boundaries before closing.
