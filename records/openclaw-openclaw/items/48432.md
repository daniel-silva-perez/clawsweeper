## Summary

Current main still has the reported TUI display-corruption path: `sanitizeRenderableText` inserts literal spaces into unprotected long tokens, and the copy-sensitive guard protects underscore file-like tokens but not hyphen-only package names. The issue also has an open unmerged closing PR (#69340), so it should remain open until that PR is reviewed and either merged or closed.

## Best Possible Solution

Keep this issue open while #69340 is reviewed. The best fix is to preserve copy-sensitive code/table content in the TUI without injecting literal display spaces, with focused regression coverage for the reported hyphenated package-name case and fenced code blocks; if #69340 lands and proves the central user problem is solved, close this issue as implemented by that PR.
