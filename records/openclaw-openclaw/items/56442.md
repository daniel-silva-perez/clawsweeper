## Summary

Keep this PR open. Current main has ACP stream-to-parent relay and background-task completion delivery, but it still does not implement the PR's explicit `parentUpdates: "notify"` API, completion-only notify mode, or ACP-specific announce/internal-event source. Closing as implemented would drop a still-missing API/product decision.

## Best Possible Solution

Keep the PR open for maintainer review or replacement. Rebase the idea onto current main's ACP stream relay and background-task delivery model, then explicitly decide whether to add `parentUpdates: "notify"` or upgrade a generic completion path. Any accepted version should include schema validation, docs, protocol/source typing if needed, duplicate-delivery tests, fallback tests, and security coverage for untrusted child output.
