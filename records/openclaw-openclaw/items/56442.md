## Summary

Keep this PR open. Current main does not implement the PR's explicit `parentUpdates: "notify"` API or its ACP completion-through-parent-announce behavior. Main has ACP stream-to-parent system-event relays plus background task terminal delivery, but that is not the same as the opt-in subagent-style parent completion path proposed here.

## Best Possible Solution

Keep the PR open for maintainer review or replacement. The clean path is to rebase the idea onto current main's task-registry/background-delivery model, then decide whether ACP needs an explicit `parentUpdates: "notify"` option or whether existing ACP task completion should be upgraded to a generic announce path. Any accepted version should include schema validation, docs, protocol/source typing if needed, duplicate-delivery tests, fallback tests, and security review around untrusted child output.
