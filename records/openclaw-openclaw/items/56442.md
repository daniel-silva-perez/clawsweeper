## Summary

Keep this PR open. Current main has related ACP background-task delivery and stream-to-parent relay behavior, but it still does not implement the PR's explicit `parentUpdates: "notify"` API, the streamTo-vs-terminal-notify split, or an ACP-specific subagent announce/internal-event source. Closing as implemented would drop a still-missing product/API decision.

## Best Possible Solution

Keep the PR open for maintainer review or replacement. The best path is to rebase the idea onto current main's ACP task-registry and background-delivery model, then explicitly decide whether to add `parentUpdates: "notify"` or upgrade an existing generic completion path. Any accepted version should include tool schema validation, docs, protocol/source typing if needed, duplicate-delivery tests, fallback tests, and security review around untrusted child output.
