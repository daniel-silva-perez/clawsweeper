## Summary

Keep this PR open. Current main still does not include plugin-registered commands in the TUI slash autocomplete path: the TUI calls `getSlashCommands()`, which currently merges only built-in TUI commands plus `listChatCommands*()` results, and `listChatCommands*()` only adds core/generated commands plus optional skill commands. Plugin commands are already registered, executable, and shown in `/commands`/gateway command-list surfaces, so the PR targets a real remaining TUI-only gap. The PR is small and has no apparent supply-chain/security-sensitive changes, but it should be updated with the missing `getSlashCommands()` regression test before merge.

## Best Possible Solution

Keep this PR open as a plausible focused fix. The best path is to rebase or apply the one-file TUI autocomplete change on current main, add a colocated `src/tui/commands.test.ts` test that registers and clears a plugin command and asserts inclusion plus deduplication, then land it if the targeted TUI/plugin command tests and changed gate pass.
