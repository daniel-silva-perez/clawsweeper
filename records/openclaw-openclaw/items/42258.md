## Summary

Keep this PR open. Current main still does not implement the proposed Control UI stale-run watchdog: queued messages remain blocked by an active `chatRunId` until a matching terminal chat event or reconnect cleanup clears it. The PR is an open, unmerged implementation candidate for #42213, with related context from #42011 and actionable review comments still worth resolving.

## Best Possible Solution

Keep this PR open as the implementation candidate for #42213. Rebase it onto current main, address the review comments, keep the recovery based on the existing authenticated `agent.wait` contract, and preserve regression coverage for stale-finished recovery, active-run timeout retry, and watchdog cancellation/reset.
