## Summary

Keep this PR open. Current main at 14f140d6 still does not implement the proposed Control UI stale-run watchdog: a stale chatRunId still makes the UI busy, queued messages still stop behind that busy state, and the UI clears chatRunId only when matching terminal chat events arrive or after reconnect cleanup. The linked #42213 bug remains open, and this PR is still the implementation candidate rather than obsolete.

## Best Possible Solution

Keep this PR open as the implementation candidate for #42213. Rebase it onto current main or recreate the same behavior there, add/keep regression coverage for stale-finished recovery, active timeout retry, and cancellation/reset, address the review comments, and update the security-impact wording for the authenticated agent.wait probe.
