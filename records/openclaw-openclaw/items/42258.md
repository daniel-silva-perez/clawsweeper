## Summary

Keep this PR open. Current main still lacks the proposed Control UI stale-run watchdog, and the linked open bug remains plausible: an active `chatRunId` still makes the UI busy and prevents queued messages from flushing unless a matching terminal chat event or reconnect cleanup clears the run.

## Best Possible Solution

Keep this PR open as the implementation candidate for #42213. Rebase it onto current main, address the watchdog review comments, update the PR body to acknowledge the new authenticated `agent.wait` call, and preserve regression coverage for stale-finished recovery, active-run timeout retry, and watchdog cancellation/reset.
