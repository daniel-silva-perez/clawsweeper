## Summary

Keep open. Current main still has only replacement-style `systemPromptOverride` at the agent/default level, while the requested additive `agents.defaults.systemPrompt` / `rules`, per-agent additive merge behavior, and `$include` handling are not present. The related direct PR is closed unmerged, the duplicate issue is closed, and the open adjacent `systemPromptFile` PR does not cover the full config contract requested here.

## What This Changes

Add config-level additive agent prompt/rules fields under `agents.defaults` and per-agent entries so centrally managed rules can be injected after bootstrap files.

## Best Possible Solution

Define a single additive agent prompt contribution contract that coexists with `systemPromptOverride`, merges defaults and per-agent contributions predictably, reuses existing safe config/include handling only if approved, threads the resolved contribution through CLI, embedded, cron, subagent, and channel session creation, and updates schema/generated docs plus focused prompt tests.

## Reproduction Assessment

Not applicable as a feature request. The verification path is high-confidence source inspection: current strict schemas, docs, and runner code show only replacement-style `systemPromptOverride`, not the requested additive config prompt/rules behavior.

## Solution Assessment

Unclear until maintainers choose the API and include/security semantics. The existing `systemPromptOverride` is not the best solution for this request because it replaces the OpenClaw-built prompt instead of appending centrally managed rules.
