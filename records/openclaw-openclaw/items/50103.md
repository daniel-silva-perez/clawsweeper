## Summary

Keep this open. Current main still lacks additive `agents.defaults.systemPrompt` / `agents.defaults.rules` and per-agent equivalents; the implemented `systemPromptOverride` surface is schema-backed, documented, and wired as a full replacement for the assembled prompt. The direct implementation PR is closed unmerged, the related duplicate issue is closed, and the currently open adjacent prompt PRs do not supersede this request.

## What This Changes

Add config-driven agent prompt rules via additive `agents.defaults.systemPrompt` / `rules` and per-agent equivalents, with include support and consistent session-time injection after bootstrap content.

## Best Possible Solution

Define one additive agent prompt contribution contract alongside the existing full-replacement `systemPromptOverride`, document defaults-plus-per-agent merge order, rely on the existing `$include` mechanism where possible, and thread the resolved contribution through CLI, embedded, channel-triggered, cron, and subagent session creation with schema/generated docs and focused tests.
