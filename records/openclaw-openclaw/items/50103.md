## Summary

Keep this open. Current main still only exposes `systemPromptOverride` as a full replacement, while the requested additive defaults-plus-per-agent prompt/rules contract remains missing; the direct implementation PR is closed unmerged, and the open prompt-related PRs cover adjacent but different APIs.

## What This Changes

The issue requests config-driven additive agent prompt/rules fields under `agents.defaults` and per-agent config, including optional `$include` support and session-time injection after bootstrap files.

## Best Possible Solution

Define one additive agent prompt contribution contract alongside the existing full-replacement `systemPromptOverride`, document defaults-plus-per-agent merge order, reuse the existing `$include` resolver where appropriate, and thread the resolved contribution through CLI, embedded, channel-triggered, cron, and subagent session creation with schema/generated docs and focused tests.
