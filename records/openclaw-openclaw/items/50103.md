## Summary

Keep this issue open. Current main still lacks additive `agents.defaults.systemPrompt` / `agents.defaults.rules` and per-agent equivalents; the implemented surface is `systemPromptOverride`, which is schema-backed, documented, and wired as a full replacement for the assembled prompt. The linked implementation PR is closed unmerged, the related duplicate issue is closed, and the currently open adjacent PRs cover different prompt configuration shapes rather than superseding this request.

## What This Changes

Requests config-level additive agent system prompt/rules fields under `agents.defaults` and `agents.list[]`, including merge order and optional included rule files.

## Best Possible Solution

Define one additive agent-prompt contribution contract, preserve `systemPromptOverride` as a full-replacement escape hatch, document defaults-plus-per-agent merge order, and thread the resolved contribution through CLI, embedded, channel-triggered, cron, and subagent session creation paths with schema/help/generated docs and focused tests.
