## Summary

Keep open. Current main still lacks the requested additive `agents.defaults.systemPrompt` / `rules` and per-agent prompt contribution contract; the only agent-level config surface is `systemPromptOverride`, which replaces the built prompt. The direct implementation PR is closed unmerged, the related duplicate issue is closed, and the open adjacent prompt PRs cover different APIs rather than this full defaults-plus-per-agent rules feature.

## What This Changes

Requests config-injectable agent rules via additive `agents.defaults.systemPrompt` or `rules`, optional per-agent overrides, and `$include` support so behavior rules can apply across workspaces and session surfaces.

## Best Possible Solution

Define one additive agent prompt contribution contract that coexists with `systemPromptOverride`, document defaults-plus-per-agent merge order, reuse existing config include handling where safe, and thread the resolved contribution through all session creation paths with schema/generated-doc updates and focused tests.

## Reproduction Assessment

Not applicable as a feature request. The verification path is high-confidence source inspection: strict schemas and runner code on current main show only replacement-style `systemPromptOverride`, not the requested additive config prompt/rules behavior.

## Solution Assessment

Unclear until maintainers choose the public contract and include/security semantics. The existing `systemPromptOverride` is not the best solution for this request because it replaces the OpenClaw-built system prompt instead of appending centrally managed rules.
