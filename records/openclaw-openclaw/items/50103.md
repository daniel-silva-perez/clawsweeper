## Summary

Keep this issue open. Current main still lacks additive `agents.defaults.systemPrompt` / `agents.defaults.rules` and per-agent equivalents; it only supports `systemPromptOverride`, which replaces the assembled prompt. The linked implementation PR is closed unmerged, and the related duplicate issue is closed, so there is no active canonical tracker for the remaining feature.

## Best Possible Solution

Implement a distinct additive, schema-backed prompt contribution surface such as `agents.defaults.systemPrompt` and/or `agents.defaults.rules`, plus per-agent fields with documented merge order. Thread the resolved contribution through shared prompt-building paths after bootstrap/context injection, preserve `systemPromptOverride` as full replacement, update generated schema/help/docs and Control UI/config metadata, and add focused tests for defaults, per-agent merge behavior, blank values, include semantics, and main/channel/cron/subagent sessions.

## Risks / Open Questions

- The requested `$include` behavior still needs product and security design for path resolution, trust boundaries, and whether it should reuse existing config include semantics.
- `systemPromptOverride` should remain a full-replacement escape hatch; changing its semantics to additive would be a compatibility risk for existing configs.
- Implementation needs coverage across CLI, embedded Pi, channel-triggered runs, cron/background runs, and subagent sessions so the resolved agent-level prompt contribution is applied consistently.
