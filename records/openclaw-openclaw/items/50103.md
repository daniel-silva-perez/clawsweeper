## Summary

Keep this issue open. Current main still does not implement additive `agents.defaults.systemPrompt` / `agents.defaults.rules` or per-agent equivalents. The shipped config surface is `systemPromptOverride`, which is documented and wired as a full replacement for the assembled OpenClaw prompt. The related implementation PR is closed unmerged, and the related duplicate issue is also closed, so there is no active canonical tracker for the remaining feature.

## Best Possible Solution

Implement a distinct additive, schema-backed prompt contribution surface such as `agents.defaults.systemPrompt` and/or `agents.defaults.rules`, plus per-agent fields with documented merge order. Thread the resolved contribution through shared prompt-building paths after bootstrap/context injection, preserve `systemPromptOverride` as full replacement, update generated schema/help/docs and Control UI/config metadata, and add focused tests for defaults, per-agent merge behavior, blank values, include semantics, and main/channel/cron/subagent sessions.

## Risks / Open Questions

- The requested `$include` behavior needs explicit product and security design for path resolution, trust boundaries, and whether it reuses any existing config include semantics.
- `systemPromptOverride` should remain a full-prompt replacement escape hatch; changing it to additive would be a compatibility risk for existing configs.
- Implementation needs coverage across CLI, embedded Pi, channel-triggered runs, cron/background runs, and subagent sessions so the resolved agent-level prompt contribution is applied consistently.
