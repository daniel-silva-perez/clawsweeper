## Summary

Keep open. Current main still does not implement the PR's intended Dockerized sandbox browser fix: the sandbox browser CDP path is still loopback-only, the bridge contract still exposes only a host-local baseUrl/bridgeUrl, embedded sandbox info still injects that same URL, and non-loopback attach-only browser profiles still enter the persistent Playwright path. The PR is useful context but still needs a current-path rewrite and security review before it can land.

## Best Possible Solution

Replace or rewrite this PR on the current browser plugin path. Add an explicit advertised/container-reachable browser bridge URL through src/plugin-sdk/browser-bridge.ts and extensions/browser runtime API, keep the host-local bridge URL for gateway-side browser tools, expose advertisedBridgeUrl on SandboxBrowserContext, inject that URL from buildEmbeddedSandboxInfo(), resolve sandbox browser CDP host/port separately for Dockerized gateways, keep attach-only non-loopback sandbox profiles on the raw CDP tab-open path where appropriate, and add focused Dockerized gateway plus non-Docker regression coverage and docs.
