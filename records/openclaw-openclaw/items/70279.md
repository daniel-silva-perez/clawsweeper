## Summary

Keep this issue open. Current main has adjacent improvements, including generic `model_fallback_decision` skip/probe logs and Claude CLI auth-environment cleanup, but the concrete requests are not fully implemented: there is still no CLI-backend-specific pre-dispatch skip diagnostic, the CLI backend docs still do not document the root/systemd Claude CLI constraint or `IS_SANDBOX` workaround, and fallback session persistence still intentionally writes auto fallback overrides before retrying rather than gating on whether the configured primary was attempted.

## Best Possible Solution

Keep this open for maintainer triage. Split the work into explicit decisions: add a CLI-backend-facing skip diagnostic for pre-dispatch auth/cooldown skips, document the safe root/systemd Claude CLI operating model on the public CLI backends page, and separately decide whether auto fallback persistence should be suppressed when the primary candidate was skipped instead of attempted. Avoid adding any root bypass switch without security review.
