## Summary

Keep open. Current main has adjacent Claude CLI environment cleanup and generic model-fallback observability, but it still does not satisfy the requested CLI-backend-facing skip log, root/systemd Claude CLI documentation, or maintainer decision on fallback session persistence after primary skips. The reporter’s follow-up documents a local workaround but explicitly keeps the OpenClaw maintainer requests active.

## What This Changes

The issue asks OpenClaw to expose pre-dispatch claude-cli skips, document root/systemd Claude CLI constraints, and reconsider fallback model persistence when the primary was not attempted.

## Best Possible Solution

Split the remaining work into maintainer-approved pieces: add a CLI-backend-facing pre-dispatch skip diagnostic for auth/cooldown skips, document the safe Claude CLI root/systemd operating model on the public CLI backend page, and decide whether auto fallback persistence should be suppressed when the configured primary was skipped rather than actually attempted.

## Reproduction Assessment

Yes for triage: the issue provides concrete root+systemd steps and current main has a deterministic pre-dispatch skip path when all auth profiles for a candidate are unavailable. A live VPS/systemd run would still be useful before changing root-specific guidance.

## Solution Assessment

Unclear as one bundled fix. Logging and documentation are narrow, but changing fallback persistence and adding or endorsing any root bypass need maintainer product and security judgment first.
