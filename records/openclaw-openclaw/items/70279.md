## Summary

Keep open. Current main has some adjacent improvements for this report, including generic `model_fallback_decision` skip logs and Claude CLI environment cleanup, but the specific maintainer requests are not fully implemented: there is no CLI-backend-specific skipped diagnostic, the CLI backend docs do not document the root/systemd constraint or a safe operator path, and current fallback/session behavior intentionally persists auto fallback overrides across turns rather than gating persistence on whether the primary was actually attempted.

## Best Possible Solution

Keep this open for maintainer triage. The best path is to split the work into clear, reviewable decisions: add explicit CLI-backend skip diagnostics for pre-dispatch auth/cooldown skips, document a safe root/systemd Claude CLI operating model, and separately decide whether auto fallback persistence should be suppressed when the primary candidate was skipped rather than attempted. Avoid adding an `allowRoot` or root bypass knob without an explicit security review.
