## Summary

Keep this PR open. Current main still does not log successful Feishu reply-dispatcher deliveries for text, card, or media replies, so the PR is not obsolete. The implementation should be revised before merge because current send helpers can fall back from a failed non-thread reply to a direct send, which means a log mode derived only from `replyToMessageId` can be wrong.

## Best Possible Solution

Keep the PR open and ask for a small rebase/revision. Add success logging to the current Feishu reply-dispatcher text, card, and media paths; make the logged mode reflect the actual delivery path returned by the send helpers; and cover direct send, reply/thread send, media success, and withdrawn-target fallback logging in focused tests.
