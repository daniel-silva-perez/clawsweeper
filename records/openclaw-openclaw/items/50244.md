## Summary

Keep this PR open. Current main still lacks Feishu reply-dispatcher success logging for outbound text, card, and media deliveries, so the PR is not obsolete. The implementation candidate is focused, but it needs a rebase/revision because its logged send-vs-reply mode can be wrong when current Feishu send helpers fall back from a failed non-thread reply to a top-level direct send.

## Best Possible Solution

Keep the PR open and ask for a small rebase/revision on current main. Add success logging to the current Feishu reply-dispatcher text, card, and media paths; make the logged mode reflect the actual delivery path by returning or exposing reply-vs-direct from the send helpers; and add focused tests for direct send, reply/thread send, media success, and withdrawn-target fallback logging.
