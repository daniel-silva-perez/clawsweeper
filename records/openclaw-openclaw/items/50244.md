## Summary

Keep this PR open. Current main still lacks the Feishu reply-dispatcher success logging for outbound text, card, and media replies, so the PR is not obsolete. It still needs a rebase/revision before merge because the proposed log mode is derived from the requested reply target, while current Feishu text/card send helpers can fall back from reply to top-level direct send for withdrawn or missing non-thread reply targets.

## Best Possible Solution

Keep the PR open and ask for a small rebase/revision on current main. Add success logging to the current reply-dispatcher text/card/media paths, make the logged mode reflect the actual delivery path by exposing reply-vs-direct from the send helpers, and add focused tests for direct send, reply/thread send, media success, and withdrawn-target fallback logging.
