## Summary

Keep this PR open. Current main still does not log successful Feishu reply-dispatcher delivery results for text/card/media replies, and the PR’s approach needs revision so the logged send mode reflects the actual fallback/direct-vs-reply path.

## Best Possible Solution

Keep the PR open and ask for a small rebase/revision. Add success logging for Feishu reply-dispatcher text, card, and media sends; have the send helpers expose or return the actual delivery mode when fallback occurs; and add focused tests for direct send, reply/thread send, media success, card success, and withdrawn-target fallback logging.
