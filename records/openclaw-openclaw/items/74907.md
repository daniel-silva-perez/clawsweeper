## Summary

Keep open. The report is specific, current-release, and points at a real strict Anthropic replay boundary: current main has several replay guards, but I did not find a regression or fix that proves compacted multi-tool turns plus persisted trajectory/session state cannot still submit retained `tool_use` blocks without all matching `tool_result` blocks.

## What This Changes

Fix Anthropic multi-tool replay after compaction so retained assistant tool-use blocks stay paired with all matching tool results and repair paths do not resurrect orphan tool-use state.

## Best Possible Solution

Add a regression-driven invariant at the replay/compaction boundary: before any Anthropic payload is submitted, every retained assistant tool-use span must be followed by one user payload containing all matching tool results, or the assistant tool-use blocks must be stripped/synthesized symmetrically across session and captured replay state.
