## What This Changes

The issue reports a 2026.4.x regression where sessions with parallel Claude/Anthropic tool calls later replay as orphaned `tool_use` blocks and fail on the next user message.

## Best Possible Solution

Add a focused fix and regression around Claude CLI stdio live-session replay so a session with parallel tool calls can accept a later user message without submitting orphaned Anthropic `tool_use` blocks. Preserve the direct Anthropic HTTP converter's existing grouped `tool_result` behavior and avoid broad transcript rewrites unless the CLI replay test proves they are necessary.

Yes. The reporter gives a high-confidence path: start a fresh Claude CLI/OAuth session, trigger an assistant turn with multiple tool calls, then send any follow-up user message and observe Anthropic rejecting orphan `tool_use` IDs; current code still matches the described route.
