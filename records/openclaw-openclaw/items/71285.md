## Summary

Keep open. Current main still lacks any configurable memory-search exclusion contract, and the existing dreaming hardening only blocks promotion/self-ingestion paths; it does not stop `memory/dreaming/**` markdown from being enumerated by the built-in indexer, watched by memory sync, or included in QMD default memory collections.

## What This Changes

Request a config-driven memory-search exclusion setting so users can keep paths such as `memory/dreaming/**` out of memory search indexing and watchers.

## Best Possible Solution

Add a documented `memorySearch.ignorePaths` contract with consistent glob matching across schema/types, resolved agent config, built-in file listing, built-in watcher filtering, QMD collection/update behavior where feasible, reindex-state hashing, generated config docs, and focused regression tests. Keep `memory/dreaming/**` opt-in unless maintainers explicitly choose a default exclusion.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: create a markdown file under `memory/dreaming/light/` and follow current defaults, where built-in `listMemoryFiles` walks `memory/` recursively and QMD default collections bind `memory` with `**/*.md`.

## Solution Assessment

Yes, the requested config-based exclusion is the narrowest maintainable direction for this issue. The safer implementation detail is to use shared glob semantics and wire the setting through all search backends instead of hardcoding `dreaming` or only changing a watcher predicate.
