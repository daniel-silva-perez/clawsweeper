## Summary

Keep open. Current main still lacks a configurable memory-search exclusion contract, and generated dreaming report markdown remains eligible for both the built-in memory index walk and QMD default memory collections; the related merged dreaming work only hardened promotion/self-ingestion paths.

## What This Changes

Add a configurable `memorySearch.ignorePaths` mechanism so users can exclude `memory/dreaming/**` or other paths from memory indexing/search and file watching.

## Best Possible Solution

Add one documented `memorySearch.ignorePaths` contract with consistent glob semantics across config resolution, built-in file listing, built-in watcher filtering, QMD collection/search handling where supported, reindex-state hashing, generated config docs, and focused regression tests. Keep `memory/dreaming/**` opt-in unless maintainers explicitly choose a default exclusion.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to place a markdown file under `memory/dreaming/light/`: the built-in `listMemoryFiles` path recursively walks `memory/`, the watcher does not ignore `dreaming`, and QMD binds the default memory directory with `**/*.md`.

## Solution Assessment

Yes. A config-based ignore list is the narrow maintainable direction for this report, but it should use shared glob semantics and verified QMD behavior rather than a hardcoded `dreaming` special case or a literal-directory-only matcher.
