## Summary

Keep open. Current main still lacks a configurable `memorySearch.ignorePaths` or equivalent exclusion contract, while the builtin memory file lister and memory-core watchers can still include `memory/dreaming/**` markdown unless it happens to match one of the fixed dependency/cache directory names.

## What This Changes

Request a user-configurable memory-search exclusion setting so paths such as `memory/dreaming/**` can be excluded from indexing, search results, and file watching without hardcoding dreaming as a default exclusion.

## Best Possible Solution

Add a documented `memorySearch.ignorePaths` contract with shared path/glob matching, merge default and per-agent config consistently, apply it to builtin file enumeration and memory-core watch scope, handle QMD behavior explicitly, include it in any relevant reindex-state/config hashing, and add focused regression coverage plus generated config docs. Keep `memory/dreaming/**` opt-in unless maintainers choose a default policy.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to place a markdown file under `memory/dreaming/light/` and follow the current builtin listing path: `listMemoryFiles` recurses through `memory/`, while the watcher ignores only fixed dependency/cache directory names and there is no accepted config field to exclude the path.

## Solution Assessment

Yes. A config-based exclusion is the narrowest maintainable direction for this issue because it solves dreaming search pollution and large extra-path churn without hardcoding a product decision about every directory name; the implementation should use a shared matcher rather than divergent watcher/indexer semantics.
