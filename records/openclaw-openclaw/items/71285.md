## Summary

Keep open. Current main still lacks a configurable memory-search exclusion contract, and the merged dreaming hardening only prevents promotion/self-ingestion; it does not stop `memory/dreaming/**` markdown from being indexed, watched, or returned by memory search.

## What This Changes

The issue requests a configurable memory-search ignore/exclude path setting so dreaming artifacts, especially `memory/dreaming/**`, can be excluded from indexing, search results, and watcher traversal.

## Best Possible Solution

Add one documented memory-search exclusion contract, likely `memorySearch.ignorePaths`, with shared glob matching and consistent propagation through config types/schema, builtin file listing, memory-core watcher filtering, QMD collection/search behavior where feasible, reindex-state hashing, generated config docs, and focused regression tests. Keep `memory/dreaming/**` opt-in unless maintainers explicitly choose a default exclusion.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to create a markdown file under `memory/dreaming/light/` and follow the default memory indexing path: current `listMemoryFiles` walks `memory/` recursively, and QMD's default collection is `memory/**/*.md`; no live run was performed because this was a read-only review.

## Solution Assessment

Yes. A config-based exclusion is the narrowest maintainable direction for this report, but it should use shared glob semantics across builtin indexing, watchers, and QMD where possible rather than a narrow literal-directory matcher or a hardcoded `dreaming` special case.
