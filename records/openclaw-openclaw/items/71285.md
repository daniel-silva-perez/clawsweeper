## Summary

Keep open. Current main still lacks any `memorySearch.ignorePaths` or equivalent exclusion contract, so `memory/dreaming/**` markdown remains in the default memory search scope and the watcher path is still only filtered by hardcoded dependency/cache directory names. The related dreaming fixes address promotion/self-ingestion, not search indexing or watch scope.

## What This Changes

The issue requests a configurable memory-search exclusion setting so `memory/dreaming/**` and other directories can be excluded from memory search indexing and file watching.

## Best Possible Solution

Add a unified memory-search exclusion contract, preferably `memorySearch.ignorePaths`, with documented glob semantics and a shared matcher used by builtin indexing, builtin watchers, QMD collection/update/watch scope, and scope hashing. Preserve current dependency/cache defaults, make `memory/dreaming/**` configurable unless maintainers choose a different default, and add focused regression tests for builtin/QMD parity.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to place a markdown report under `memory/dreaming/<phase>/YYYY-MM-DD.md` and run the default memory indexing/search path: current `listMemoryFiles` recursively includes all markdown under `memory/`, and the watcher ignore set does not exclude `dreaming`.

## Solution Assessment

Yes, the requested config-based exclusion is a maintainable direction, but the prior PR should not be copied narrowly if it diverges between chokidar glob behavior and indexer matching. The safer fix is a shared ignore matcher plus QMD parity and scope-hash coverage.
