## Summary

Keep open. Current main still lacks a configurable memory-search exclusion contract, and `memory/dreaming/**` remains eligible for builtin indexing and QMD indexing/search; the existing dreaming fixes only address promotion/self-ingestion, not recall index scope or user-configurable exclusions.

## What This Changes

The issue requests a `memorySearch.ignorePaths` config option to exclude paths such as `memory/dreaming/**` from memory search indexing and file watching.

## Best Possible Solution

Add one documented `memorySearch.ignorePaths` exclusion contract with consistent glob matching across resolved config, builtin file listing, builtin watcher filtering, QMD collection/update behavior where feasible, reindex-state hashing, generated config docs, and focused regression tests. Keep `memory/dreaming/**` opt-in unless maintainers explicitly choose a default exclusion.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to create a markdown file under `memory/dreaming/light/` and follow the default indexing paths: builtin `listMemoryFiles` recursively walks `memory/`, and QMD binds `memory` with `**/*.md`.

## Solution Assessment

Yes. A config-based exclusion is the narrowest maintainable solution for this report, but it should use shared glob semantics instead of a hardcoded `dreaming` special case or a literal-directory-only matcher.
