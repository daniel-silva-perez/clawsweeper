## Summary

Keep open. Current main still lacks a `memorySearch.ignorePaths` or equivalent exclusion contract; default built-in indexing still walks `memory/**` markdown, QMD default collections still include `memory/**/*.md`, and the watcher only skips a fixed dependency/cache directory list. The related dreaming PRs are useful context, but they address promotion/self-ingestion rather than search indexing or watch scope.

## What This Changes

Issue requests a configurable memory-search exclusion setting so `memory/dreaming/**` and other directories can be omitted from memory search indexing and file watching.

## Best Possible Solution

Add a single memory-search exclusion contract, likely `memorySearch.ignorePaths`, with documented glob semantics and one shared matcher used by config resolution, builtin file listing, watcher filtering, QMD collection/update scope, and reindex scope hashing. Keep current dependency/cache defaults, add focused regression tests for `memory/dreaming/**`, and update generated config docs plus user-facing memory docs.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to create `memory/dreaming/light/YYYY-MM-DD.md` and use the default memory sync/search path: current `listMemoryFiles` recursively walks `workspace/memory` and the watcher only skips hardcoded dependency/cache directory names.

## Solution Assessment

Yes, a config-based exclusion is a maintainable way to solve this, but the safer implementation should not copy the old narrow matcher from the closed PR if it diverges from chokidar glob behavior. The fix should use shared matching semantics and cover both builtin and QMD backends.
