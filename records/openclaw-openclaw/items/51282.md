## Summary

Keep this PR open. Current main still does not implement `gateway.handshakeTimeoutMs` as a config-file key, and the PR is a plausible implementation candidate, but its diff is incomplete and stale against the current gateway timeout path.

## Best Possible Solution

Keep the PR open and request a rebase plus complete implementation against current main: add `gateway.handshakeTimeoutMs` to `GatewayConfig`, the strict Zod schema, labels/help, generated schema/docs surfaces, and focused tests for `OPENCLAW_HANDSHAKE_TIMEOUT_MS` > config value > default. Wire runtime behavior through an existing gateway startup/runtime config snapshot rather than calling `loadConfig()` per unauthenticated socket open.
