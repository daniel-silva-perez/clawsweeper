## Summary

Keep this PR open. Current main still does not implement `gateway.handshakeTimeoutMs` as a config-file key, and the PR remains a plausible focused fix, but it is stale and incomplete against the current gateway handshake/config surfaces.

## Best Possible Solution

Keep the PR open and ask for a rebase plus a complete implementation against current main: add the config field to `GatewayConfig`, the strict Zod schema, schema help, schema labels, generated schema/docs surfaces as required, and focused tests proving priority order `OPENCLAW_HANDSHAKE_TIMEOUT_MS` > `gateway.handshakeTimeoutMs` > default. The runtime should source the config value from gateway startup/runtime config state rather than reparsing config on each unauthenticated socket open.
