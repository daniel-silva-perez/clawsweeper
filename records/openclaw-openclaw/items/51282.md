## Summary

Keep this PR open. Current main still does not implement `gateway.handshakeTimeoutMs` as a config-file key, and the submitted PR remains a plausible implementation candidate, but it is incomplete and stale against the current gateway timeout path.

## Best Possible Solution

Keep the PR open and request a rebase plus complete implementation against current main: add `gateway.handshakeTimeoutMs` to `GatewayConfig`, strict Zod schema, help, labels, generated schema/docs surfaces, and focused tests for `OPENCLAW_HANDSHAKE_TIMEOUT_MS` > config value > default. Wire the runtime through an existing gateway config snapshot rather than reparsing config on each unauthenticated connection.

## Risks / Open Questions

- Porting the PR as-is would still be non-functional because `gateway.handshakeTimeoutMs` is absent from the strict Zod schema.
- Adding the help text without a matching `schema.labels.ts` entry would fail the existing help/label parity test.
- The PR's runtime change is based on `src/gateway/server-constants.ts`, but current main resolves the pre-auth timeout from `src/gateway/handshake-timeouts.ts`.
- Calling `loadConfig()` from the pre-auth WebSocket path would add synchronous config parsing/validation work to unauthenticated socket opens; a safer implementation should use gateway startup/runtime config state.
