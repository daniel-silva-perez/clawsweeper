## Summary

Keep open. Current main still lacks the requested durable per-agent inbox, stable sender idempotency, acknowledgement record, and restart-safe retry path for `sessions_send`; the existing durable queues cover outbound delivery and restart continuation, not reliable cross-session message injection.

## What This Changes

The issue requests a persistent message queue for agent-to-agent `sessions_send` delivery so messages survive timeouts, failures, and target-agent restarts until acknowledged.

## Best Possible Solution

Keep this as a core gateway/session reliability design item. The desired end state is a durable accepted-delivery record for cross-session sends, stable caller/sender idempotency, ack or terminal-failure state, startup/heartbeat draining, and regression coverage for timeout plus gateway restart behavior.
