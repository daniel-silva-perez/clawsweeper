## Summary

Keep open. Current main has a working `sessions_send` path and a durable session-delivery queue, but the requested persistent at-least-once inbox for ordinary `sessions_send` delivery is still not wired into the tool or documented as supported.

## What This Changes

Add a persistent at-least-once message queue/inbox for `sessions_send` so inter-agent messages survive timeouts or restarts and are retried until acknowledged.

## Best Possible Solution

Define and implement a core reliable A2A delivery contract: persist accepted `sessions_send` messages before dispatch, drain them on startup or heartbeat, record stable acknowledgement/idempotency state, and cover timeout, restart, and duplicate-delivery cases in the session-tool, gateway, and queue tests plus docs.
