## Summary

Keep open. Current main has durable delivery primitives for restart-continuation recovery, but ordinary `sessions_send` still starts a Gateway `agent` run directly and reports timeout/error without a persisted inbox entry, durable acknowledgement state, or retry handle. The A2A announce path also remains best-effort, so the central reliable-delivery request is still unresolved core behavior.

## What This Changes

Request adding a persistent per-agent message queue or inbox for `sessions_send` with at-least-once delivery, retries, startup/heartbeat polling, acknowledgements, and idempotency.

## Best Possible Solution

Define a core reliable A2A delivery contract: persist accepted `sessions_send` messages before dispatch, drain pending entries on startup or heartbeat, record stable acknowledgement and idempotency state, and document/test timeout, restart, duplicate-delivery, and security-boundary behavior across the session tool, Gateway agent method, delivery queue, and public docs.
