## Summary

Keep open. Current main still forwards only the raw Slack toolContext through the bundled custom invoke path, while agent-scoped mediaLocalRoots/mediaReadFile are present on the top-level channel action context. Slack upload-file then reaches the runtime without the media access it needs for agent workspace files, and the related #64632 fix PR is closed unmerged.

## What This Changes

The issue asks Slack upload-file to preserve agent-scoped mediaLocalRoots/mediaReadFile when message actions run through Slack's custom invoke path.

## Best Possible Solution

Update the bundled Slack plugin to build one augmented Slack action context from toolContext plus ctx.mediaLocalRoots/mediaReadFile, pass that context through both custom invoke and fallback runtime paths, and add focused regression coverage for upload-file via createSlackActions(options.invoke).
