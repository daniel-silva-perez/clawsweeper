## Summary

Keep open. Current main still forwards only the raw Slack toolContext through the bundled custom invoke path, while the agent-scoped mediaLocalRoots/mediaReadFile live on the top-level channel action context. That still leaves Slack upload-file without the media access required for agent workspace files, and the related #64632 fix PR is closed unmerged.

## What This Changes

The issue requests forwarding agent-scoped mediaLocalRoots and mediaReadFile through Slack message action custom invoke handling for upload-file and related media sends.

## Best Possible Solution

Fix this in the bundled Slack plugin by constructing one augmented Slack action context from toolContext plus ctx.mediaLocalRoots/mediaReadFile and passing that through both custom invoke and fallback runtime paths, with focused regression coverage for createSlackActions(options.invoke) upload-file behavior.
