## Summary

Keep open. Current main has meaningful partial coverage through SecretRefs, config API redaction, Control UI masking/reveal controls, and several log-sink redaction gates, but the central request is not complete because normal config writes and backups still persist plaintext secret-bearing JSON and some diagnostic/raw outputs still need an explicit redaction policy.

## What This Changes

Request to protect sensitive data by encrypting or otherwise avoiding plaintext config secrets, redacting OpenClaw and gateway logs, and masking sensitive Control UI fields by default with explicit reveal controls.

## Best Possible Solution

Keep this as the umbrella until maintainers split or land canonical work covering the storage strategy, backup/service artifact handling, diagnostic/raw sink policy, UI inventory, and regression tests; close it only after linked outcomes cover the original storage, logging, and display requirements.
