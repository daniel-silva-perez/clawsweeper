## Summary

Keep open. Current main has substantial partial coverage through SecretRefs, config API redaction, Control UI masking/reveal controls, and several log sink redaction gates, but it does not satisfy the full request because normal config writes and backup rotation still persist secret-bearing JSON as plaintext and some diagnostic/raw outputs still need explicit policy or redaction work.

Request support for protecting sensitive data in OpenClaw config persistence, logs/gateway logs, and built-in UI display surfaces.
