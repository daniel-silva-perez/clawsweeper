## Summary

Keep open. Current main has meaningful log redaction and Control UI masking, but the broader sensitive-data request is still unresolved because primary config, included config, auth-profile/model JSON stores, and config backups still persist secret-bearing JSON in plaintext with restrictive permissions rather than encryption or an equivalent at-rest strategy.

## What This Changes

Add support for encrypting or masking sensitive data across config files, logs/gateway logs, and Control UI display surfaces.

## Best Possible Solution

Keep this umbrella open until maintainers either split and link canonical sub-issues or land the remaining persistence strategy, backup handling, diagnostic/raw sink audit, UI inventory, and regression coverage, then close it only when those linked outcomes cover the original request.
