## Summary

Keep open. Current main has meaningful partial work for sensitive-data handling: additive SecretRefs, `openclaw secrets` audit/apply, log/transcript redaction, log tail redaction, and Control UI masking/reveal controls. The central request is still broader than that: plaintext config remains supported, ordinary config and included-config writes still serialize JSON directly to disk, and normal backup rotation still copies current config contents into `.bak` files.

## Best Possible Solution

Keep this issue open as the umbrella until maintainers either split and link canonical follow-up issues or land the remaining work: an at-rest secret strategy for config/auth artifacts/backups/service outputs, completion of the log/diagnostic sink audit, a full Control UI sensitive-field inventory, and regression coverage across persistence, logs, API snapshots, diagnostics, and UI masking.
