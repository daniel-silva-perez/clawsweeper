## Summary

Keep open. Current main already has packaged compile-cache enablement, operator-managed cache tuning docs, and a public `OPENCLAW_NO_RESPAWN` escape hatch, but it does not implement the remaining first-class prewarm command, shipped cache artifact, or postinstall priming path requested by the issue.

## What This Changes

The issue requests cold-start improvements by adding a prewarmed Node compile-cache path and a public opt-in to skip CLI self-respawn.

## Best Possible Solution

Keep the implemented compile-cache scoping and `OPENCLAW_NO_RESPAWN` behavior, then have maintainers decide whether the supported upstream shape should be a narrow explicit `openclaw warm-cache --output` command with docs and tests; treat postinstall priming and shipped cache artifacts as separate security/release decisions.

## Reproduction Assessment

Partly. ## Solution Assessment

Unclear until maintainers choose the accepted product shape. The narrowest maintainable direction appears to be an explicit opt-in warm-cache command rather than automatic postinstall priming or shipped cache artifacts.
