## Summary

Keep open. Current main partially addresses the report with automatic packaged-launcher compile-cache enablement, `NODE_COMPILE_CACHE` docs, and `OPENCLAW_NO_RESPAWN=1`, but it does not provide a first-class `openclaw warm-cache` command, shipped pre-warmed cache artifact, or postinstall cache priming. The remaining request needs maintainer product/security judgment because one proposed path changes install-time behavior and Node compile-cache invalidation.

## Best Possible Solution

Keep the issue open and have maintainers choose a narrow accepted shape. The safest likely direction is a documented, explicit `openclaw warm-cache --output <dir>` image-build helper with tests and container/VPS docs, while treating automatic postinstall prewarming or shipped cache artifacts as separate product/security decisions. The existing `OPENCLAW_NO_RESPAWN=1` env can be documented as the public respawn escape hatch unless maintainers want a CLI flag alias.
