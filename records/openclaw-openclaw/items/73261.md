## Summary

Keep open. Current main does not include this targetProvider fast path, and the PR still has a blocking request-destination integrity issue: it can accept a disk provider after provider-level checks while leaving model-level transport fields from `models.json` unvalidated.

## What This Changes

This PR adds `targetProvider`/`targetModel` options to `ensureOpenClawModelsJson`, a disk-vs-config provider short-circuit before implicit provider discovery, and a new Vitest file covering the short-circuit and cache behavior.

## Best Possible Solution

Keep the fast path only if it proves every disk field that can shape runtime requests is consistent with the current planned provider, or make the short-circuit fail closed whenever disk `models[]`, per-model `baseUrl`/`api`/headers, provider request options, or similar transport-shaping fields are present and not independently validated.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to generate `models.json`, edit the target provider's model entry to add a malicious `baseUrl` or `api` while leaving provider-level `apiKey`/`baseUrl`/`headers`/`auth` matching config, then call `ensureOpenClawModelsJson` with `targetProvider`; the PR fast path can return before the full plan rewrites the model entry.

## Solution Assessment

No. The proposed provider-level comparison is narrower than the runtime request surface; the safer fix is to validate model-level and request-shaping drift, or fall through to the full planning path when those fields are not proven safe.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject model transport drift before short-circuiting:** `src/agents/models-config.ts:375`
  - body: The short-circuit returns after comparing only provider-level `apiKey`, `baseUrl`, `headers`, and `auth`, but current runtime can route requests from `discoveredModel.api` and `discoveredModel.baseUrl` loaded from `models.json`. A disk edit that keeps the provider row matching while changing `models[].baseUrl` or `models[].api` will skip the full planning rewrite and preserve the tampered destination. Please compare the model/request transport fields that can affect requests, or fail closed and run the full plan when they are present but unvalidated.
  - confidence: 0.9
