## Summary

Keep open. The PR is useful and active, but the proposed fast path still trusts an on-disk provider when only provider-level fields match, while current runtime can take request destinations from per-model `models.json` entries. That leaves a concrete tampered `models[].baseUrl` path that the full planning path would overwrite but the new short-circuit would preserve.

## What This Changes

This PR adds a `targetProvider`/`targetModel` options type for `ensureOpenClawModelsJson`, a provider-level disk-vs-config short-circuit before implicit provider discovery, and a new test file covering the short-circuit and cache behavior.

## Best Possible Solution

Keep the fast path only if it proves every disk field that can shape runtime requests is consistent with the current planned provider, or make it fail closed whenever disk `models[]`, per-model `baseUrl`/`api`/headers, provider request options, or other transport-shaping fields are present and not independently validated.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to generate `models.json`, edit the target provider's model entry to add a malicious `baseUrl` while leaving provider-level `apiKey`/`baseUrl`/`headers`/`auth` matching config, then call `ensureOpenClawModelsJson` with `targetProvider`; the PR fast path returns before the full plan can rewrite the model entry, and the embedded runner can consume `discoveredModel.baseUrl`.

## Solution Assessment

No. The provider-level comparison is narrower than the runtime request surface; the safer approach is to validate or reject model-level/request-shaping drift before skipping the full planning path.
