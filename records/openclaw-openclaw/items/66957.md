## Summary

Keep open. Current main still invokes implicit provider discovery before it reads `models.mode`, so `replace` mode can still pay discovery cost and merge implicit providers despite the documented configured-providers-only contract. An open, unmerged PR (#73557) now explicitly targets this issue, so the right cleanup is to keep the issue open until that PR or another fix lands.

## Best Possible Solution

Keep this issue open. Review and either land or close #73557. The fix should branch on `cfg.models?.mode === "replace"` before any implicit provider discovery, use only explicitly configured providers on that path, preserve normalization and source-managed secret enforcement, and add regression coverage proving the injected implicit resolver is not called in replace mode while merge/default mode still discovers providers. Treat #63552 as related performance work unless it is updated to satisfy this replace-mode contract too.
