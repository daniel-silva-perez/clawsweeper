## Summary

Keep open. Current main still awaits implicit provider discovery before reading `models.mode`, and open PR #73557 uses closing syntax for the same bug, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for `models.mode="replace"` to skip implicit provider discovery and use only explicitly configured providers to avoid large startup delays.

## Best Possible Solution

Review and either land or close #73557. The final fix should branch on `cfg.models?.mode === "replace"` before any implicit discovery, preserve explicit-provider normalization and source-managed secret enforcement, keep merge-mode discovery behavior intact, and add regression coverage proving the implicit resolver is not called in replace mode.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: call `planOpenClawModelsJsonWithDeps()` with `cfg.models.mode` set to `"replace"` and an injected implicit resolver; current main still reaches that resolver before it reads the mode. The precise 72-second timing was not live-reproduced here.

## Solution Assessment

Yes, the intended direction in #73557 appears to be the narrow maintainable fix, but it is still unmerged. The safer boundary is to gate discovery early while leaving normalization, secret enforcement, and merge-mode provider discovery unchanged.
