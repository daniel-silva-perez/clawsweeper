## Summary

Current main still preserves the reported unmanaged Linux custom-supervisor failure mode for non-update SIGUSR1/config restarts: unknown supervisors are not detected, the generic restart helper detached-spawns, and the run loop exits the original tracked PID. Recent update.run and platform mitigations are adjacent, but they do not close this issue.

## Best Possible Solution

Keep this open and land a focused core fix: make non-update unmanaged SIGUSR1/config restarts use the in-process restart path by default when no trusted supervisor is detected, preserve `respawnGatewayProcessForUpdate()` for update.run, update doctor/docs wording, and replace tests that currently assert detached unmanaged Linux behavior with regression coverage for the safer default.
