## Summary

Keep open. Current main does not implement the requested `fallbacksFromModels` behavior, and the PR is useful, but it is not merge-ready because the new fallback source is not wired into the embedded runner's fallback-enabled gate and the branch is stale against current model timeout config support.

## What This Changes

The PR adds an opt-in `agents.defaults.model.fallbacksFromModels` config path, schema/types/docs/tests for deriving model fallbacks from `agents.defaults.models`, and a preservation fix for default model updates.

## Best Possible Solution

Rebase the branch onto current main, preserve the existing shared model `timeoutMs` schema/type behavior, and update the shared fallback-configured helper plus embedded-runner coverage so catalog-derived fallbacks participate in the same failover decisions as explicit fallback arrays.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve model timeout fields when splitting the schema:** `src/config/zod-schema.agent-model.ts:3-8`
  - body: Current main accepts `timeoutMs` in the shared agent model schema and tests `agents.defaults.imageGenerationModel.timeoutMs`. This refactor extracts `AgentModelObjectSchema` with only `primary` and `fallbacks`, so after rebasing it would reject existing timeout configs. Carry `timeoutMs: z.number().int().positive().optional()` through the shared object schema and matching TS type.
  - confidence: 0.9
- **[P1] Mark catalog-derived fallbacks as configured:** `src/agents/model-fallback.ts:390-394`
  - body: The new candidate builder can derive fallback models, but embedded Pi still decides whether to enter outer model fallback through `hasConfiguredModelFallbacks`, which only counts explicit fallback arrays. With `fallbacksFromModels: true` and no explicit `fallbacks`, prompt/assistant/retry-limit failures can surface on the primary instead of advancing to the derived catalog candidate. Update that shared gate and add coverage for this path.
  - confidence: 0.92
