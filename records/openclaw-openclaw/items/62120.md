## Summary

Keep open. The report is a concrete 2026.4.5 regression in `openclaw channels login --channel openclaw-weixin --verbose`, and current `main` still has a broad `channels` CLI pre-action plugin preload before `runChannelLogin()` reaches the channel plugin's `auth.login`. I did not find a merged current-main change or canonical related issue that definitively covers this exact pre-login hang.

## Best Possible Solution

Keep this issue open and fix the generic host CLI startup path: avoid broad `channels` plugin preloading for `channels login`/`logout`, then let `channel-auth` load only the requested or configured auth-capable channel through the existing scoped channel setup registry path before invoking `auth.login`. Preserve channel inference and external plugin compatibility with focused tests.
