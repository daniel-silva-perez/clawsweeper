## Summary

Keep open. Current main still lets `openclaw channels login --channel openclaw-weixin` inherit the parent `channels` startup policy, which broadly preloads plugins in CLI bootstrap before `runChannelLogin()` can reach the selected channel plugin's QR auth flow. The scoped channel setup loader exists, but login/logout have not been moved onto that narrow startup path yet.

## Best Possible Solution

Implement a generic host CLI fix: make `channels login` and `channels logout` avoid broad pre-action plugin preload, then ensure channel auth explicitly loads only the requested or inferred auth-capable channel through the existing scoped channel setup/plugin resolution path. Keep OpenClaw core channel-agnostic and avoid hard-coded Weixin policy.
