## Summary

Keep open. Current main still routes `openclaw channels login --channel openclaw-weixin --verbose` through the shared `channels` startup policy, which preloads plugins before `runChannelLogin()` can invoke the selected channel plugin's `auth.login`. The narrower scoped channel plugin load path exists, but the CLI pre-action path still has the pre-login blocking surface reported here, and I did not find an open PR with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and fix the generic host CLI startup path: avoid broad parent `channels` plugin preloading for `channels login` and likely `channels logout`, then let `channel-auth` load only the requested or inferred auth-capable channel through the existing scoped channel setup registry path. Preserve explicit `--channel openclaw-weixin`, single configured channel inference, external plugin compatibility, and config auto-enable behavior with focused tests.
