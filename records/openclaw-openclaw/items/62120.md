## Summary

Keep open. Current main still lets channel auth inherit the broad `channels` plugin-preload startup policy, so bootstrap can load all plugins before the selected external plugin's QR login starts. The scoped one-channel resolver exists, but `channels login` and `channels logout` still need exact startup-policy handling.

## What This Changes

The issue reports that `openclaw channels login --channel openclaw-weixin --verbose` hangs on OpenClaw 2026.4.5 before the Weixin plugin login flow can print a QR code.

## Best Possible Solution

Land a channel-agnostic host CLI fix that prevents interactive channel auth commands from doing broad pre-action plugin preload, then explicitly loads only the requested or inferred auth-capable channel through the existing scoped resolver before invoking plugin auth.
