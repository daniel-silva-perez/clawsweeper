## Summary

Keep open. Current main still lets interactive channel auth commands inherit the broad parent `channels` plugin preload during pre-action bootstrap, while the scoped resolver that reaches the selected plugin's QR login runs later. The prior host-side diagnosis in the discussion remains valid on the current tree.

## What This Changes

The issue reports that `openclaw channels login --channel openclaw-weixin --verbose` hangs on OpenClaw 2026.4.5 before the external Weixin plugin starts QR-login output.

## Best Possible Solution

Land a channel-agnostic CLI startup fix that keeps `channels login` and `channels logout` off broad pre-action plugin preload, then relies on the existing scoped auth resolver to load only the requested or inferred auth-capable channel before invoking plugin auth.
