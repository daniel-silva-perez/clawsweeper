## Summary

Keep open. Current main still lets interactive `channels login` and `channels logout` inherit the parent `channels` plugin-preload policy, so the host CLI can load all plugins during pre-action bootstrap before the selected external channel plugin reaches its QR-login code. The existing issue comments already traced the same host-side path, and the current source still matches that analysis.

## What This Changes

The issue reports that `openclaw channels login --channel openclaw-weixin` hangs on OpenClaw 2026.4.5 before the external Weixin plugin prints its QR-login prompt.

## Best Possible Solution

Add channel-agnostic startup policy for interactive channel auth so `channels login` and `channels logout` avoid broad pre-action plugin preload, then let the existing scoped auth resolver load only the requested or inferred auth-capable channel. Cover explicit external-channel login and logout startup policy with regression tests and add a user-facing changelog fix entry.

## Repair Work Prompt

Fix the host-side `channels login` / `channels logout` startup path for issue https://github.com/openclaw/openclaw/issues/62120. Current main lets both auth commands inherit parent `channels` `loadPlugins: "always"`, so pre-action bootstrap can load all plugins before `runChannelLogin()` reaches the scoped auth resolver and the external Weixin plugin QR-login code. Make the fix channel-agnostic: add exact startup policy for `channels login` and `channels logout` so they do not broadly preload plugins during pre-action bootstrap, and rely on the existing `resolveInstallableChannelPlugin()` path in `src/cli/channel-auth.ts` / `src/commands/channel-setup/channel-plugin-resolution.ts` to load only the requested or inferred auth-capable channel. Do not add Weixin-specific ids, dependency strings, defaults, migrations, or plugin implementation code to core. Add regression coverage for startup policy and pre-action/bootstrap behavior for login/logout, plus any channel-auth resolver coverage needed for explicit external/catelog-backed channels. Add a single-line user-facing changelog fix entry under the active version with `Thanks @annanygy`.
