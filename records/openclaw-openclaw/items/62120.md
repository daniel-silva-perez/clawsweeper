## Summary

Keep open. Current main still lets interactive channel auth inherit the broad parent `channels` plugin-preload policy, so CLI bootstrap can load all plugins before the selected external plugin's QR login starts.

## What This Changes

The issue reports that `openclaw channels login --channel openclaw-weixin --verbose` hangs on OpenClaw 2026.4.5 before the Weixin plugin can print its QR-code login output.

## Best Possible Solution

Land a channel-agnostic host CLI fix that prevents interactive channel auth commands from doing broad pre-action plugin preload, then explicitly loads only the requested or inferred auth-capable channel through the existing scoped resolver before invoking plugin auth.
