## Summary

Keep open. Current main still has a host-side CLI startup path that can run before `openclaw channels login --channel openclaw-weixin` reaches the plugin `auth.login` hook, and `channels login` still inherits broad plugin preloading from the generic `channels` command. I did not find a current-main fix, regression test, or release note proving the reported pre-QR Weixin login hang is resolved.

## Best Possible Solution

Keep this issue open for a focused core/plugin-boundary fix. The likely path is to make `channels login` avoid broad pre-action plugin loading, then load only the requested or configured login-capable channel through the existing scoped plugin resolution path, with a regression test proving `channels login --channel openclaw-weixin` reaches `plugin.auth.login` without unrelated plugin startup or doctor/preflight work blocking first.
