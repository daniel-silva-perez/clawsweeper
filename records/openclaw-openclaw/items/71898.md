## Summary

Keep this PR open. Current main still accepts and filters session-corpus memory search hits but then hard-codes surfaced memory-manager results as `corpus: "memory"`; the PR is a narrow relevant fix with focused regression coverage and no security-sensitive diff surface.

## What This Changes

This PR changes `memory_search` result shaping to surface `MemorySearchResult.source` as each result's `corpus` and adds a regression test for session-sourced hits.

## Best Possible Solution

Review and land this PR, or an equivalent narrow change, so `memory_search` preserves `MemorySearchResult.source` in surfaced results and keeps a focused regression test for session-sourced search hits.
