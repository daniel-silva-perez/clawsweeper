## Summary

Current main still does not implement configurable Amazon Bedrock Converse `anthropic_beta` or general `additionalModelRequestFields` passthrough. The original Claude 4.6 1M preview motivation appears less urgent because AWS now documents Claude Opus/Sonnet 4.6 Bedrock models with a 1M context window, but the broader request remains concrete and API-supported. citeturn1search0turn1search4

## Best Possible Solution

Keep this open for a Bedrock-plugin-owned implementation or explicit maintainer product decision. A good path is to add a validated config surface, likely under per-model `params` or `plugins.entries.amazon-bedrock.config`, for Bedrock Converse `additionalModelRequestFields`/`anthropic_beta`, merge it in `extensions/amazon-bedrock/register.sync.runtime.ts` with existing `onPayload` composition, and cover it with Bedrock plugin tests plus `docs/providers/bedrock.md`. Coordinate with #69491 if that PR is still active, because it touches adjacent default beta behavior but does not replace custom config.
