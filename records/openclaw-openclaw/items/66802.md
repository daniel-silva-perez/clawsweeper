## Summary

Keep this issue open. Current main still lacks a nested `agents.defaults.compaction.memoryFlush.model` override: config schemas reject it, docs do not advertise it, the memory flush plan has no model slot, and the runtime still resolves the flush model from the session run. PR #66846 is an open implementation candidate, but it is not merged in current main.

## Best Possible Solution

Keep this issue open until PR #66846 is reviewed and merged, or an equivalent implementation lands on main. The fix should add `agents.defaults.compaction.memoryFlush.model` to config types, zod/generated schema, labels/help, and docs; carry the value through the memory flush plan or runtime boundary; resolve provider/model/auth behavior consistently with `agents.defaults.compaction.model`; and add regression coverage for override and fallback behavior.
