## Summary

Keep open. Current main still hardcodes the media store limit to 5 MB and reuses that cap for sandbox staging, while the top-level media config schema and type still expose only filename preservation and TTL fields. The issue is also paired with open closing PR #40950, so it should remain open until that PR or an equivalent focused fix lands.

## What This Changes

Make the sandbox/media-store 5 MB media staging cap configurable through OpenClaw config so larger inbound files can be copied into sandboxed agent workspaces.

## Best Possible Solution

Review and land PR #40950 or replace it with an equivalent narrow fix that adds a bounded top-level media.maxBytes setting, preserves the 5 MB default when unset, threads the effective limit through media store and sandbox staging paths, updates config schema/help/docs/generated schema, and adds regression coverage for default and configured limits.
