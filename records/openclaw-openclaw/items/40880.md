## Summary

Keep open. Current main still hardcodes the 5 MB media store and sandbox staging cap, and the top-level `media` config schema/help/generated schema do not expose `media.maxBytes`; an open implementation PR already targets this issue, so it should not be closed before that path is resolved.

## What This Changes

Request to make the 5 MB media store and sandbox staging limit user-configurable, such as through a bounded top-level `media.maxBytes` setting, while preserving the current default when unset.

## Best Possible Solution

Review and land PR #40950, or replace it with an equivalent narrow fix that adds a bounded `media.maxBytes` configuration path, threads the effective limit through media store and sandbox staging, updates config schema/help/generated schema and docs, and adds regression coverage for default and configured limits.
