## Summary

Keep open. Current main still uses the fixed 5 MB media-store and sandbox staging limit, the strict top-level `media` config schema still has no `maxBytes` field, and open PR #40950 is the unmerged closing implementation path, so this issue should not be closed before that PR or an equivalent fix lands.

## What This Changes

The issue requests making the hardcoded 5 MB sandbox/media staging limit configurable through OpenClaw configuration.

## Best Possible Solution

Land PR #40950 or an equivalent narrow fix that adds a bounded top-level `media.maxBytes` config with the current 5 MB default, threads the effective value through media store, sandbox staging, and chat preflight, and updates schema/help/docs/tests without expanding into the broader image/provider limit work tracked separately by #67031.

## Reproduction Assessment

Yes. ## Solution Assessment
