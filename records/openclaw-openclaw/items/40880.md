## Summary

Keep open. Current main still hardcodes the 5 MB media store limit and still passes that constant into sandbox media staging, so the reported bug remains reproducible from source. The provided GitHub context also shows open PR #40950 uses closing syntax for this issue and proposes the narrow configurable `media.maxBytes` fix, so this issue should remain open until that PR or an equivalent implementation lands.

## Best Possible Solution

Keep this issue open and review, update, and land PR #40950 or an equivalent narrow fix. The implementation should add a bounded top-level `media.maxBytes` config field, thread it through media storage, media serving, and sandbox staging, preserve the 5 MB default when unset, update schema/help/docs, and add regression coverage for configured and default limits.
