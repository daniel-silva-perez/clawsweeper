## Summary

Keep open. Current main still hardcodes the 5 MB media-store and sandbox staging cap, the chat send preflight still rejects sandboxed non-image offloads above that same constant, and the strict top-level media config does not expose media.maxBytes. PR #40950 is already the open unmerged implementation path that closes this issue, so the issue should remain open until that PR is reviewed, landed, or closed.

## What This Changes

The issue requests a user-configurable sandbox and media-store staging size cap instead of the fixed 5 MB MEDIA_MAX_BYTES limit.

## Best Possible Solution

Land PR #40950 or an equivalent narrow fix that adds a bounded top-level media.maxBytes config with the current 5 MB default, threads the effective value through media store, sandbox staging, and chat preflight, updates schema/help/generated docs as needed, and adds regression coverage for default and configured limits.
