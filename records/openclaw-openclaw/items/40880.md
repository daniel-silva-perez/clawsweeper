## Summary

Keep open. Current main still hardcodes the 5 MB media-store and sandbox staging cap, does not expose `media.maxBytes` in the strict top-level media config, and #40950 is already the open unmerged implementation path that closes this issue.

## What This Changes

The issue requests making the hardcoded 5 MB media-store and sandbox staging limit user-configurable via `media.maxBytes` or an equivalent config key.

## Best Possible Solution

Land PR #40950 or an equivalent narrow fix that adds a bounded `media.maxBytes` config with the current 5 MB default, threads the effective limit through media store, sandbox staging, and chat preflight, updates schema/help/generated docs as needed, and adds regression coverage for default and configured limits.
