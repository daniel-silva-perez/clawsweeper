## Summary

Keep open. Current main still buffers Feishu group messages that fail `requireMention` and later prepends that pending history to a handled group turn, while the Feishu schema and docs do not expose the requested per-bot or mention-only context recording policy. The related open PR #40782 is useful adjacent mention-identity work, but it does not implement context isolation.

## What This Changes

Add Feishu group context isolation so each bot only records group messages in its context when explicitly mentioned, avoiding cross-bot history pollution.

## Best Possible Solution

Add a backwards-compatible Feishu context-recording policy, likely available globally and per group, that defaults to current pending-history behavior but can skip recording unmentioned group messages. Cover the new behavior in Feishu tests, update the public Feishu/group docs, and update generated config metadata if the schema changes.

## Reproduction Assessment

Yes. A source-level reproduction is clear: in an allowed Feishu group with `requireMention: true` and `historyLimit > 0`, send an unmentioned message, then send a mentioned message in the same group; the first message is recorded as pending history and then included in the second turn.

## Solution Assessment

Unclear until a maintainer chooses the public config contract. A backwards-compatible Feishu policy is likely the narrowest maintainable direction; silently changing `requireMention` would conflict with the current documented pending-history behavior.
