## Summary

Keep this issue open. Current main still records unmentioned Feishu group messages into pending group history when `requireMention` is enabled, then prepends that pending history to the next handled group turn. The available Feishu config/docs expose mention gating, history limits, group/session scopes, and generic context visibility, but not the requested per-bot or mention-only context recording policy.

## What This Changes

The issue requests a Feishu group configuration that prevents one bot from recording messages addressed to other bots into its conversation context.

## Best Possible Solution

Add a backwards-compatible Feishu context-recording policy, likely global plus per-group, that defaults to current behavior but can skip pending-history recording for unmentioned group messages and, once mention identity is reliable, filter messages addressed to other bots. Cover the behavior in Feishu tests and update the public Feishu/group docs and generated config metadata if the schema changes.
