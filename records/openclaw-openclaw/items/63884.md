## Summary

Keep open. Current main still buffers unmentioned Feishu group messages into pending group history when `requireMention` is enabled, then prepends that history to the next handled group turn; the requested per-bot or mention-only context recording policy is not present in code, schema, or docs.

## What This Changes

Add Feishu group context isolation so a bot records context only for messages that explicitly mention that bot, preventing multi-bot group conversations from polluting each other.

## Best Possible Solution

Add a backwards-compatible Feishu context-recording policy, likely available globally and per group, that defaults to current behavior but can skip pending-history recording for unmentioned group messages and filter messages addressed to other bots once mention identity is reliable. Cover the behavior in Feishu tests and update the public Feishu/groups docs plus generated config metadata if the schema changes.
