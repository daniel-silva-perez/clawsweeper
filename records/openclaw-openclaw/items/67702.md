## Summary

Current main still lacks the PR's git-mode same-install running-gateway guard. The PR remains relevant, but its current diff needs revision for wrapper-backed services and for restoring the gateway when an auto-stop is followed by a skipped or failed update. Security review found no supply-chain-sensitive file changes; the main risk is service availability.

## Best Possible Solution

Keep this PR open and request a revised implementation in src/cli/update-cli/update-command.ts: detect a same-install running gateway before in-place git checkout/build work, cover direct dist entrypoints and wrapper-backed service commands, and either fail closed or only stop the service when restart/restoration is guaranteed across success, skipped, and failed update paths. Add focused src/cli/update-cli.test.ts coverage for absolute, relative, wrapper, different-install, --no-restart, dirty/skipped, and failed-update scenarios.
