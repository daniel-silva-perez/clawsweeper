## Summary

Keep this PR open. Current main still omits `agentId` and `sessionId` from the embedded compaction hook context, while the PR addresses that remaining gap narrowly and leaves the already-fixed reset/direct-compaction slices alone. The broader observability trace-context request remains separate in #50291.

## What This Changes

The PR forwards existing `agentId`, `sessionId`, and `sessionKey` values through embedded before/after compaction plugin hook callbacks and updates the compaction hook wiring test to assert that context.

## Best Possible Solution
