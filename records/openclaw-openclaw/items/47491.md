## Summary

Keep this PR open. Current main still has the embedded compaction hook context gap: the embedded before/after compaction handlers forward only `sessionKey`, even though the subscription params already carry `agentId` and `sessionId` and the direct compaction helper forwards those fields. The PR is a narrow implementation of the right fix, but it needs maintainer review and rebase/adaptation to current main before validation.

## What This Changes

The PR forwards existing `agentId`, `sessionId`, and `sessionKey` values into embedded before/after compaction plugin hook callbacks and updates the compaction hook wiring tests.

## Best Possible Solution

Land the narrow embedded compaction context fix after rebase so plugin hooks receive consistent scoped `agentId` and `sessionId` context across direct and embedded compaction paths, while leaving broader observability and trace-context expansion to #50291.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: construct an embedded compaction context with `ctx.params.agentId` and `ctx.params.sessionId`, then observe that `handleCompactionStart` and `handleCompactionEnd` pass only `sessionKey` to the plugin hook runner.

## Solution Assessment

Yes, with a rebase caveat. Forwarding the already-resolved optional context fields at the two embedded hook call sites is the narrowest maintainable fix and avoids expanding the plugin API surface.
