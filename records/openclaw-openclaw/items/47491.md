## Summary

Keep open. Current main still drops `agentId` and `sessionId` from the embedded before/after compaction plugin hook contexts, while the subscription params and hook context type already carry those fields. The supplied PR diff is a narrow fix for that remaining gap and has no definite correctness or security finding, but it needs maintainer review plus rebase/adaptation to current main before merge.

## What This Changes

The PR forwards existing `agentId`, `sessionId`, and `sessionKey` values into embedded compaction plugin hook callbacks and updates the compaction hook wiring test assertions.

## Best Possible Solution

Land the narrow embedded compaction context fix after rebase so plugin hooks receive the same scoped `agentId` and `sessionId` context across direct and embedded compaction paths, while keeping broader trace/run/message correlation work in #50291.

## Reproduction Assessment

Yes. A source-level reproduction on current main is high confidence: the embedded compaction handlers receive params containing `agentId` and `sessionId`, but the hook calls forward only `sessionKey`.

## Solution Assessment

Yes, with a rebase caveat. Forwarding the existing optional context fields at the two embedded hook call sites is the narrowest maintainable fix and avoids expanding the plugin API surface.
