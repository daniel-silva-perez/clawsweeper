## Summary

Keep open. Current main still has the reported request-shape problem, and the PR is useful source work, but it targets an older label-file layout and needs to be ported onto the current shared helper before merge.

## What This Changes

This PR changes auto-topic-label and TTS summarization simple-completion calls to pass instructions through `systemPrompt`, omit `temperature` for `openai-codex-responses`, handle completion errors, and add regression tests.

## Best Possible Solution

Port the request-shape fix onto current main by updating `generateConversationLabel` and `summarizeText`, preserving non-Codex backend behavior, adding Codex-specific regression coverage in the current test suites, and keeping shared provider transport behavior unchanged unless new tests prove a generic seam is needed.
