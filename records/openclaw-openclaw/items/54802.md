## Summary

Keep this PR open. Current main still has the behavior the PR is trying to fix: both the generic conversation label path used by Telegram auto-topic labels and the direct TTS summarization path put instructions inside the user message and always pass temperature, while the Codex Responses transport derives top-level `instructions` from `context.systemPrompt`. The PR is useful but stale against current main because auto-topic labeling now delegates through `generateConversationLabel`, not the old file path in the PR.

## Best Possible Solution

Keep the PR open and adapt it to current main: move the auto-topic-label fix into `generateConversationLabel`, update `summarizeText`, preserve existing temperature behavior for non-Codex simple-completion backends, omit temperature for `openai-codex-responses`, handle `stopReason === "error"`, and add/update the current conversation-label and TTS contract tests. Address the bot review's fallback-message wording during the same pass.
