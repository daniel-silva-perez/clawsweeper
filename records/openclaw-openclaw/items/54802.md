## Summary

Keep open. The requested runtime fix is still needed on current main, but the PR branch is stale: its auto-topic-label change targets a file that current main no longer uses, while the active Telegram label path still delegates to the generic conversation-label helper with the legacy request shape.

## What This Changes

The PR changes auto topic label and TTS summarization simple-completion calls to pass instructions through `systemPrompt`, omit `temperature` for `openai-codex-responses`, handle error stop reasons, and add regression tests against the older label helper and TTS test surface.

## Best Possible Solution

Port the request-shape fix onto current main by updating `generateConversationLabel` and `summarizeText`, preserving non-Codex backend behavior, adding Codex-specific regression coverage in the current test suites, and keeping shared provider transport behavior unchanged unless new evidence proves a generic seam is needed.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure the label or TTS summary model as `openai-codex-responses` and inspect the `completeSimple` call: current main sends no `systemPrompt` and still passes `temperature`, while the pinned dependency reads Codex `instructions` from `context.systemPrompt`.

## Solution Assessment

No, not as-is. The PR’s TTS direction is aligned with the dependency contract, but the label change must be moved to the current `generateConversationLabel` implementation and the error fallback should distinguish provider errors from empty successful summaries.
