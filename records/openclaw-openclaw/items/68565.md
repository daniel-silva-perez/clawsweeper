## Summary

Keep open. The PR targets a real Anthropic replay problem, and the discussion correctly points at signed/redacted thinking preservation, but the proposed guard is too broad: it would carry opaque Anthropic thinking signatures into OpenAI Responses replay, where current main parses `thinkingSignature` as JSON and would throw before request assembly.

## What This Changes

The PR changes `transformTransportMessages` to preserve signed or redacted thinking blocks across all model transitions, adds regression tests for those replay cases, and adds an Unreleased changelog entry.

## Best Possible Solution

Add a replay-boundary policy keyed by source and target API: preserve Anthropic signed/redacted thinking verbatim only for Anthropic-compatible targets, keep OpenAI Responses JSON reasoning items on OpenAI Responses replay, and downgrade or drop incompatible thinking blocks for other targets with matrix coverage for Anthropic to Anthropic, Anthropic to OpenAI Responses, Anthropic to Google, and retry back to Anthropic.

## Reproduction Assessment

Yes. A static reproduction is enough: replay an Anthropic assistant thinking block with `thinkingSignature: "sig_opaque"` to an `openai-responses` model after the PR guard; the block survives transformation and `JSON.parse("sig_opaque")` throws in `convertResponsesMessages`.

## Solution Assessment

No. The intended direction is valid, but preserving all signed/redacted thinking blocks for every target is not the narrowest maintainable fix; the safer solution is target-compatible preservation at the replay boundary.
