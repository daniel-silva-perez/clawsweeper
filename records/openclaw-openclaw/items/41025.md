## Summary

Keep this PR open. Current main still flattens remote http/https markdown images to alt text in Control UI chat, so the PR’s intended behavior is not implemented. The PR remains a focused candidate for #40945, but it needs maintainer product/security direction because the current renderer, served CSP, docs, tests, and export-html behavior all preserve a no-remote-image-fetch posture.

## Best Possible Solution

Keep this PR open as an implementation candidate for #40945. Maintainers should make an explicit remote-media policy decision, then either revise the PR to align the renderer, CSP, docs, tests, export/share surfaces, and mitigations such as HTTPS-only, allowlisting, proxying, lazy loading, and referrerpolicy="no-referrer", or document that remote image fetches remain intentionally blocked and close focused implementation attempts against #40945 or #64607.

## Risks / Open Questions

- Merging the PR as-is would let chat or model output cause arbitrary browser image fetches if CSP is relaxed, exposing viewer IP address, user agent, and read timing to third-party hosts.
- The PR allows http:// as well as https://, which creates mixed-content and downgrade concerns for HTTPS Control UI deployments.
- Because the PR patch only adjusts renderer/tests, the current served Control UI CSP would still block remote images in real browser use unless the security policy is deliberately changed.
- The PR does not update the documented no-remote-fetch policy or the adjacent export-html renderer/tests, so product behavior would become inconsistent across surfaces.
- The emitted remote image markup in the provided patch does not include the current markdown-inline-image class unless the PR is rebased/adjusted.
