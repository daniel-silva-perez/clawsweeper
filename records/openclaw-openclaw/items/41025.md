## Summary

Keep open. Current main still does not implement this PR's intended behavior: markdown `http://`/`https://` images in Control UI chat are flattened to alt text, while data URI images render. The PR remains a focused implementation candidate for open #40945, but it needs maintainer product/security direction because the renderer-only change conflicts with the current served CSP/docs and introduces arbitrary browser remote-image fetches.

## Best Possible Solution

Keep this PR open as an implementation candidate for #40945. A complete maintainer-approved solution should align the markdown renderer, served CSP, docs, and tests around an explicit remote-media policy, likely HTTPS-only at minimum and preferably an allowlist, opt-in setting, managed-media proxy, and/or `referrerpolicy="no-referrer"`. If maintainers choose to preserve the current no-remote-fetch policy, record that decision on #40945 or #64607 before closing implementation attempts.

## Risks / Open Questions

- Merging the PR as-is would conflict with the current Control UI CSP and docs; served deployments may still block remote images unless CSP and documentation change too.
- Allowing arbitrary remote image URLs in chat/model output creates privacy risk: third-party hosts can observe viewer IP address, user agent, and read timing. The PR also allows `http://`, adding mixed-content and downgrade concerns on HTTPS deployments.
- The PR likely needs a rebase/design update against current main, including `markdown-inline-image` styling expectations, CSP tests, and the documented no-remote-fetch policy.
