## Summary

Keep open. Current main still intentionally flattens remote markdown images to alt text, while the served Control UI CSP and docs keep remote image fetches blocked. This PR remains a focused implementation candidate for open #40945, but it needs maintainer product/security direction because the renderer-only change conflicts with current CSP/docs and allows arbitrary browser remote-image fetches.

## Best Possible Solution

Keep this PR open as an implementation candidate for #40945. A complete maintainer-approved solution should align the markdown renderer, Control UI CSP, docs, and tests around an explicit remote-media policy, likely HTTPS-only at minimum and preferably an allowlist, opt-in setting, managed-media proxy, and/or referrerpolicy="no-referrer". If maintainers choose to preserve the current no-remote-fetch policy, record that decision on #40945 or #64607 before closing implementation attempts.

## Risks / Open Questions

- Merging this PR as-is would conflict with the current served Control UI CSP and docs; remote images may still be blocked in normal served Control UI unless the policy changes too.
- Allowing arbitrary remote image URLs in chat/model output creates privacy risk: third-party hosts can observe viewer IP address, user agent, and read timing. The PR also permits http:// URLs, which adds mixed-content and downgrade concerns on HTTPS deployments.
- The PR is older than current main and likely needs rebase/design updates around the current markdown-inline-image class, CSP tests, and documented no-remote-fetch policy.
