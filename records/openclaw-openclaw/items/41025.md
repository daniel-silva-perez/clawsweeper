## Summary

Keep this PR open. Current main still flattens remote http/https markdown images to alt text in the Control UI, the linked bug #40945 is still open, and the PR remains a plausible implementation candidate. It needs rebase plus an explicit product/security decision because current renderer, CSP, docs, tests, and export-html behavior still preserve a no-remote-image-fetch posture.

## Best Possible Solution

Keep this PR open as a focused implementation candidate for #40945. Maintainers should make an explicit remote-media policy decision, then either rework the PR against the current markdown-it renderer and align CSP, docs, tests, export/share surfaces, and mitigations such as HTTPS-only, allowlisting/proxying, lazy loading, and referrerpolicy="no-referrer", or document that remote image fetches remain intentionally blocked and close implementation attempts in favor of #40945 or the broader media discussion in #64607.

## Risks / Open Questions

- Merging the PR as-is would let chat/model content trigger arbitrary browser image fetches if CSP is relaxed, exposing viewer IP address, user agent, and read timing to third-party hosts.
- The PR allows http:// as well as https://, which creates mixed-content and downgrade concerns for HTTPS Control UI deployments.
- The patch only changes renderer/tests, while the current served CSP would still block remote images unless deliberately changed.
- The docs and export-html renderer/tests still document and enforce a no-remote-fetch policy, so a renderer-only merge would make behavior inconsistent across surfaces.
- The PR patch targets the older htmlEscapeRenderer shape; current main uses markdown-it md.renderer.rules.image, so it needs a rebase/rework before it can land cleanly.
