## Summary

Current main still lacks a `bailian` provider auth env candidate, `BAILIAN_API_KEY` support, or a `bailian` alias to the bundled Qwen/Alibaba provider path. This should stay open because #42978 remains an open implementation candidate that uses GitHub closing syntax for this issue.

## Best Possible Solution

Keep this issue open while #42978 remains open. Maintainers should review that PR, then either land a focused compatibility fix with regression coverage for `resolveEnvApiKey("bailian")` and `BAILIAN_API_KEY`, or close the PR and update this issue/docs with the supported `qwen`/`alibaba` configuration path and current env/SecretRef syntax.
