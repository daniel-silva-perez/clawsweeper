## Summary

Keep this open. Current main still lacks `bailian` as a provider/auth alias and does not include `BAILIAN_API_KEY` in the bundled Qwen auth env candidates. There is also an open external implementation candidate that uses GitHub closing syntax for this issue, so the right path is maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this open and review the linked implementation candidate. If maintainers want Bailian naming compatibility, land a focused Qwen/Model Studio compatibility change that adds a `bailian` provider/auth alias plus `BAILIAN_API_KEY` env candidate and regression coverage for `resolveEnvApiKey("bailian")`. If maintainers reject that alias, document the supported `qwen`/`modelstudio`/`dashscope` provider ids and current SecretRef syntax, then close with that product decision.
