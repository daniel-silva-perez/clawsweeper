## Summary

Keep this PR open. Current main still lacks equivalent plugin logging best-practice guidance, and the docs-only patch targets a real stdout/JSON concern, but it should be revised before merge because it documents `api.logger` and required logger methods as optional even though the current SDK contract only makes `debug` optional.

## Best Possible Solution

Revise this docs patch and then review/merge it: add concise plugin logging guidance to `docs/plugins/building-plugins.md` or the SDK docs, keep the warning that `console.log` can corrupt JSON stdout for commands such as `openclaw config schema`, and align the example with the current contract: `api.logger.info(...)`, `api.logger.warn(...)`, `api.logger.error(...)`, and `api.logger.debug?.(...)`.
