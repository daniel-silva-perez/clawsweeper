## Summary

Keep this PR open. Current main still does not implement the PR’s intended `resolveModel("google", "google/gemini-2.5-pro", ...)` behavior for a bare configured Google model, and the overlapping #41398 is still open rather than shipped. This PR also has unresolved review risk around over-broad normalization and provider-native nested model ids, so it should be reviewed or consolidated rather than auto-closed.

## Best Possible Solution

Keep this PR open and consolidate it with #41398. The best implementation is a narrow resolver fix that strips a duplicated active-provider prefix only when it is truly redundant for matching/transport, preserves pass-through and nested provider-native model ids such as OpenRouter and NVIDIA ids, adds the specific Google configured-model regression test, and then closes the competing PR after the chosen fix ships.
