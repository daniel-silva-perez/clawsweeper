## Summary

Keep this PR open. Current main still does not implement the intended `provider=google` plus `modelId=google/gemini-2.5-pro` resolver behavior, and the overlapping #41398 is still open/unmerged in the provided GitHub context rather than shipped. The PR is plausible but needs maintainer follow-up because review comments identify broader normalization and nested model-id risks.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is a narrow resolver, manifest, or provider-owned normalization fix that strips duplicated `google/` only when it is truly provider/model duplication, preserves OpenRouter pass-through and nested provider-native ids, and adds regression coverage for Google inline/configured resolution plus nested-id preservation. Review it alongside #41398, then close the unmerged duplicate only after one implementation lands.
