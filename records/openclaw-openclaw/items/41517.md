## Summary

Keep open. Current main still does not implement this PR's intended same-provider prefix stripping for resolveModel("google", "google/gemini-2.5-pro", ...) against a bare configured Google model, and the overlapping #41398 is still open rather than shipped. The PR is focused and plausible, but maintainer consolidation is still needed because review comments flagged normalization scope and provider-native nested-id preservation concerns.

## Best Possible Solution

Keep this PR open and consolidate it with #41398. The best implementation is a narrow resolver fix that strips a duplicated active-provider prefix only when it is truly redundant for matching and transport, preserves pass-through and provider-native nested ids, adds the Google configured-model regression test from this PR, and closes the competing implementation only after the selected fix lands.
