## Summary

Keep this PR open. Current main still does not implement the intended resolver behavior for `provider=google` with `modelId=google/gemini-2.5-pro`, and the overlapping #41398 remains open rather than a shipped replacement. This PR is still a plausible implementation candidate, but it needs revision because review comments flagged broader normalization and nested same-provider ID risks.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is a narrow current-main revision that uses the manifest/provider-owned normalization seam or an equivalent resolver guard to strip duplicated `google/` only for Google provider/model duplication, preserves OpenRouter pass-through and provider-native nested ids, and adds regression coverage for Google inline/configured resolution plus nested-id preservation. Review it alongside #41398, then close the unmerged duplicate only after one implementation lands.
