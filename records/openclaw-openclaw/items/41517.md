## Summary

Keep this PR open. Current main still does not implement the PR’s intended `provider="google"` plus `modelId="google/gemini-2.5-pro"` resolver behavior for inline/configured Google models, and the overlapping #41398 remains open rather than shipped. The PR is a plausible fix candidate, but it still needs maintainer review because the review discussion flags normalization-scope and nested-model-id risks.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is a narrow resolver or manifest-normalization fix that strips duplicated `google/` only when it is truly the active provider prefix, preserves OpenRouter and nested provider-native ids, adds regression coverage for the Google inline/configured resolution case, and resolves the overlap with #41398 after one implementation lands.
