## Summary

Keep this PR open. Current main still does not implement the intended `provider=google` + `modelId=google/gemini-2.5-pro` resolver behavior, and the overlapping #41398 is open/unmerged rather than a shipped replacement. The PR is still a plausible candidate, but the submitted patch needs revision because review comments identify broader normalization and nested same-provider model-id risks.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is a narrow resolver or manifest/provider-owned normalization fix that strips duplicated `google/` only where it is truly provider/model duplication, preserves OpenRouter pass-through and provider-native nested ids, and adds regression coverage for Google inline/configured resolution plus nested-id preservation. Review it alongside #41398, then close the unmerged duplicate only after one implementation lands.
