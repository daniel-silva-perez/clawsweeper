## Summary

Keep this issue open. Current main has runtime per-provider discovery gates for GitHub Copilot, Hugging Face, Ollama, and Bedrock, and Bedrock is documented. The remaining user-visible work is still real: public docs do not document the non-Bedrock plugin config paths, there is no typed global model-discovery kill switch requested in the discussion, and startup planning still resolves implicit providers before applying `models.mode` merge/replace handling. The related implementation PR #33365 is closed and unmerged, so it does not supersede this issue.

## Best Possible Solution

Keep this issue open. Document the shipped plugin-owned discovery config paths in public provider/config docs, especially `plugins.entries.github-copilot.config.discovery.enabled`, `plugins.entries.huggingface.config.discovery.enabled`, and `plugins.entries.ollama.config.discovery.enabled`, and clarify the deprecated `models.*Discovery.enabled` aliases while they remain supported. Then either add a typed global model-discovery disable control that is checked before `resolveImplicitProviders`, or split that global kill switch and `models.mode` interaction into a focused follow-up with tests.
