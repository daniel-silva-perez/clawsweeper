## Summary

Keep this issue open. Current main has runtime per-provider discovery toggles for GitHub Copilot, Hugging Face, Ollama, and Bedrock, and Bedrock is documented. The remaining gaps are still real: public docs do not document the non-Bedrock toggle paths, the strict models config schema has no global disable-all model discovery control requested in the discussion, and model planning still resolves implicit providers before merge/replace handling. Related PR #33365 was closed unmerged in the provided context, so it does not supersede this work.

## Best Possible Solution

Keep this issue open. Document the shipped per-provider plugin config paths in public provider/config docs, especially `plugins.entries.github-copilot.config.discovery.enabled`, `plugins.entries.huggingface.config.discovery.enabled`, and `plugins.entries.ollama.config.discovery.enabled`, and clarify legacy alias behavior if those aliases remain supported. Then either add a typed global model-discovery disable control checked before `resolveImplicitProviders`, or split that global kill switch into a focused follow-up with tests covering explicit providers and `models.mode` interaction.
