## Risks / Open Questions

- Closing this PR now would discard an active implementation candidate while current main still does not provide API 26-30 install support.
- Lowering `minSdk` from 31 to 26 expands the supported runtime surface substantially; maintainers should require a deliberate Android compatibility sweep and emulator/device proof across API 26, 29/30, 31, and current target SDK before merge.
- The PR's direct UDP DNS fallback for older Android versions is plausibly necessary when `DnsResolver` is unavailable, but it is still a network/privacy behavior change that deserves normal Android review.
