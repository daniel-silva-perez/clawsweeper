## Summary

Keep open. Current main still keeps the Android app at minSdk 31, this PR remains the active contributor implementation for API 26 support, and the discussion already includes maintainer concern that this is a broader compatibility/product decision. The latest patch also still has a concrete runtime compatibility gap around motion command advertisement below API 29.

## What This Changes

The PR lowers the Android app minSdk from 31 to 26 and adds API-level fallbacks for dynamic color, foreground service startup, gateway discovery DNS, location, notification listener access, thermal status, and motion-related UI/permission surfaces.

## Best Possible Solution

Decide the Android support floor explicitly. If API 26 support is accepted, land a comprehensive compatibility pass that gates runtime command advertisement and invocation consistently, updates Android docs to match the new floor, and collects focused emulator or device proof for install, launch, connection, discovery/manual connect, foreground notification, permissions/settings, and advertised node commands across representative old and current API levels.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate motion command availability below API 29:** `apps/android/app/src/main/java/ai/openclaw/app/tools/Features.kt:9`
  - body: The new Features.Motion gate is only wired into UI and device permission reporting. NodeRuntime still advertises motion.activity and motion.pedometer from MotionHandler sensor availability, while MotionHandler still requires ACTIVITY_RECOGNITION.
