## Summary

Keep this PR open. Current main still requires Android API 31, so the requested API 26 support is not implemented on main; however the PR still needs maintainer product-scope judgment and has a concrete motion runtime compatibility bug before it can be considered merge-ready.

## What This Changes

The PR lowers the Android app minimum SDK to API 26 and adds compatibility guards or fallbacks for theme colors, foreground service startup, DNS discovery, location, notification listener access, thermal status, and motion-related UI/permission flows.

## Best Possible Solution

Keep the PR open as an implementation candidate until maintainers decide whether OpenClaw should support API 26. If they proceed, the branch should be rebased onto current main, the motion runtime availability gap fixed, Android docs and any affected test/benchmark surfaces aligned, and emulator or device proof collected across the newly supported API range; if maintainers decide to stay API 31+, close it with that explicit product decision.
