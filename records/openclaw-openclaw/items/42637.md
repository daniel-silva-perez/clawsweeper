## Summary

Keep this PR open. Current main still drops prompt-truncated skills with only a generic `included N of M` warning, so the PR is not obsolete or implemented. The PR remains a focused implementation candidate, but because it changes model-facing skills prompt text and may expose or render local skill names, it still needs maintainer/security review before landing.

## Best Possible Solution

Keep this PR open for maintainer/security review. The best path is to land this patch or an equivalent maintainer patch that tracks omitted skills for both count-based and character-budget truncation, lists enough omitted names for discovery, preserves the final prompt budget, and adds regression coverage for tight budgets plus hostile or special-character skill names.

## Risks / Open Questions

- The PR would add local installed skill names to model-facing prompt text, which may disclose workspace or environment metadata to remote model providers unless maintainers accept or gate that behavior.
- Skill names come from metadata and need a rendering policy that preserves exact `openclaw skills info` lookup keys while neutralizing control characters, markup, quotes, commas, and prompt-injection-like text.
- The final emitted skills prompt must continue to honor `skills.limits.maxSkillsPromptChars` after remote notes, truncation warnings, omitted-name lists, and compact/full skill bodies are combined.
