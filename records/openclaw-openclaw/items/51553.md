## Summary

Keep this PR open. Current main still has the stale-cache gap this PR targets: `updateLastRoute` takes the session-store lock but reads `loadSessionStore(storePath)` without `{ skipCache: true }`, while neighboring locked writers already bypass the cache.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a current-main version of the `{ skipCache: true }` change in `updateLastRoute`, ideally with a regression test that primes the session-store cache, performs an external preserved-mtime write, calls `updateLastRoute`, and verifies unrelated external fields are not clobbered.
