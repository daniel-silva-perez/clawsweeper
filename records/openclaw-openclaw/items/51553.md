## Summary

Keep this PR open. Current main still has the stale-cache read in `updateLastRoute`: it enters the session-store lock and then calls `loadSessionStore(storePath)` without `{ skipCache: true }`, while the other lock-holding session-store writers already re-read with `skipCache: true`. The PR's one-line functional change is still useful and is not implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a current-main version of the `{ skipCache: true }` change in `updateLastRoute`, preferably with a regression test that primes the session-store cache, performs an external preserved-mtime write, calls `updateLastRoute`, and verifies unrelated external fields are not clobbered.
