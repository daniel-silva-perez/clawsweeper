## Summary

Keep open. Current main has prompt guidance and a guarded first-class gateway config tool, but the raw `exec` path still allows model-issued OpenClaw config/model/gateway self-management commands to run under `security=full` plus `ask=off` without a hard confirmation boundary.

## What This Changes

The issue requests a confirmation boundary so example OpenClaw config, model, and gateway restart commands are not automatically executed by the agent.

## Best Possible Solution

Add a hard exec-boundary policy for OpenClaw self-management commands so model-initiated `openclaw config set`, `openclaw models set`, gateway lifecycle, update, uninstall, reset, and similar commands require explicit user-approved intent when run through raw shell execution. Keep the first-class gateway tool as the preferred config/restart path and coordinate the broader destructive-command policy with #65307.
