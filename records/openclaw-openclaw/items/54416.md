## Summary

Keep this issue open. Current main still has prompt guidance and a first-class gateway config mutation guard, but the raw `exec` path still allows model-issued OpenClaw self-management commands under the documented gateway/node `security=full` plus `ask=off` policy, with no hard confirmation boundary for `openclaw config set`, `openclaw models set`, or `openclaw gateway restart`.

## What This Changes

The issue asks OpenClaw to prevent agents from automatically executing example config/model/gateway commands without explicit user confirmation.

## Best Possible Solution

Add a hard exec-boundary policy for OpenClaw self-management commands, requiring explicit user-approved intent before model-initiated `openclaw config set`, model-switch/config writes, gateway lifecycle, update, uninstall, and similar commands can run through raw shell execution. Keep the first-class gateway tool path as the preferred implementation surface, and coordinate the broader destructive-command policy with #65307.
