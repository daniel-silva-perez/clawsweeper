## Summary

Keep this issue open. Current main still relies on advisory prompt guidance and configurable exec approvals; it does not add a hard exec-boundary guard that blocks or requires confirmation for model-issued OpenClaw self-management shell commands such as `openclaw config set ...`, `openclaw models set ...`, or `openclaw gateway restart` when host exec is in the documented full/off mode.

## Best Possible Solution

Keep this open for a maintainer/security fix at the exec boundary. Add a hard runtime guard that blocks or requires explicit user approval for model-initiated OpenClaw self-management shell commands such as `openclaw config set`, `openclaw models set`, `openclaw gateway restart`, update, uninstall, and equivalent package-manager self-removal commands unless they come from an explicit user-approved config-change action. Add regression coverage for ambiguous model-switch prompts that may show example commands but must not execute them automatically, and coordinate the broader destructive-command policy with #65307.
