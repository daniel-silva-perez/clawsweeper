## Summary

Keep this issue open. Current main still relies on prompt guidance and first-class gateway-tool guards, but raw host exec can still run OpenClaw config/model/restart commands without a confirmation boundary when the effective policy is `security=full` and `ask=off`.

## What This Changes

Prevent agents from automatically executing OpenClaw config, model-switch, and gateway restart commands presented as examples unless the user explicitly approves or requests execution.

## Best Possible Solution

Add a hard exec-boundary policy for OpenClaw self-management commands so model-initiated `openclaw config set`, `openclaw models set`, gateway lifecycle, update, uninstall, and similar commands require explicit user-approved intent when run through raw shell execution. Keep the gateway tool as the preferred config/restart path and coordinate the broader destructive-command policy with #65307.
