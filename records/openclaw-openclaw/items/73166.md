## Summary

Keep this issue open. Current main still has cron timer rejection handlers that only log failures, no independent watchdog lifecycle, and an open same-author PR with closing syntax is already paired with this runtime scheduler-stall report.

## What This Changes

The issue reports that scheduled cron jobs can silently stop firing after about 2.5 days of macOS Gateway uptime while manual cron runs and a Gateway restart still work.

## Best Possible Solution

Review and land PR #73355 or an equivalent narrow cron-runtime recovery change that re-arms rejected timer ticks, adds any needed watchdog lifecycle cleanup, and includes direct regression coverage for the rejection/stalled-chain path.
