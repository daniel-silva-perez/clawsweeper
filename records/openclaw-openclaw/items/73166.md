Keep this issue open. The report is specific, current main still has timer catch handlers that only log and does not include an independent cron watchdog, and open PR #73355 is already paired with this issue as an implementation candidate that should be reviewed before any close.

## What This Changes

A timezone-aware daily cron job stopped firing automatically after about 2.5 days of macOS Gateway uptime while manual cron execution and a Gateway restart still worked.

## Best Possible Solution

Keep this issue and PR #73355 linked until the PR is reviewed, revised if needed, and merged, or land an equivalent narrow cron-runtime recovery change with direct rejection-path coverage and watchdog lifecycle cleanup.
