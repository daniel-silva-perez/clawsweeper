## Summary

Keep this issue open. Current main still has the specific TUI reconnect/backfill gap: TUI reloads history once on reconnect, reports sequence gaps without recovery, and does not subscribe to the session transcript event stream that already exists and that Control UI handles. Open PR #73402 is an active implementation candidate for this class of fix, but it is not merged into main.

## Best Possible Solution

Keep this issue open and route it to the TUI transcript/reconnect work. The best product fix is to land or supersede #73402 with safe TUI `sessions.messages.subscribe` lifecycle handling, gap/reconnect-triggered backfill, session-switch unsubscribe behavior, and focused regression tests for post-restart heartbeat replies without clobbering active runs.
