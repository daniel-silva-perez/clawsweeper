## Summary

Keep open. Current main has partial Codex OAuth and quota support, including OAuth refresh, Codex WHAM usage-window parsing, Control UI quota hints, and doctor/log guidance for permanent refresh-token failures, but it still lacks the requested native diagnostic state contract and bounded brokered reauth action.

## What This Changes

Request native Codex/ChatGPT OAuth quota diagnosis, local-time quota display in Control UI, and optional brokered host-side reauth only when true reauthentication is required.

## Best Possible Solution

Add an additive provider auth/usage diagnostic contract with states such as ok, expiring_soon, quota_wait, and reauth_required; keep existing automatic OAuth refresh and WHAM parsing; expose local-time 5-hour or primary-window and weekly Codex renewal details in Control UI; and design a narrow, auditable broker API that can run only after reauth_required is diagnosed.
