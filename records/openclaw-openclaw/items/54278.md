## Summary

Keep this issue open. Current main has useful partial Codex OAuth/usage handling, including automatic OAuth refresh, WHAM usage-window parsing, WHAM-informed cooldowns, provider usage display, and doctor/runner guidance for known permanent OAuth refresh failures. It still lacks the requested native diagnostic contract that distinguishes quota_wait from reauth_required through models.authStatus, and it has no bounded Control UI or host-side brokered reauth action.

## Best Possible Solution

Keep this open as a core/provider plus Control UI product issue. Add an additive diagnostic contract on the provider auth/usage path with states such as ok, expiring_soon, quota_wait, and reauth_required; keep the existing WHAM parsing and automatic OAuth refresh behavior; expose Codex 5-hour and weekly renewal details in local time; and design a narrow authorized broker/adapter execution API that is only available when diagnosis is reauth_required. Cover 401/403 refresh failure, 429/quota-window behavior, Control UI rendering/actions, and broker authorization boundaries with focused tests.
