## Summary

Keep open. Current main still builds the skills snapshot from the host workspace and then reuses that snapshot prompt after switching non-rw sandbox runs to the sandbox workspace, so the reported host-only skill path can still be exposed. Open #38789 remains the focused implementation candidate for the root fix; #74161 is related prompt hardening but does not remap or rebuild stale snapshot paths.

## What This Changes

Sandboxed non-main sessions should expose sandbox-readable skill paths in available-skills prompts instead of host-managed skill locations.

## Best Possible Solution

Land a root fix that rebuilds or remaps available-skills prompt locations from sandbox-readable skill entries whenever a non-rw sandbox run receives a host-built snapshot. The fix should cover both attempt and compaction paths while preserving filters, exposure metadata, remote eligibility notes, and skill env behavior; the exact-location wording in #74161 can remain defense-in-depth rather than the root repair.
