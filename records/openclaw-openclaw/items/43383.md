## Summary

Keep open. Current main still reuses the host-built skills snapshot prompt after non-rw sandbox setup switches execution to the sandbox workspace, so the synced sandbox skill copy does not guarantee that `<available_skills>` points at sandbox-readable paths.

## What This Changes

The issue reports that Docker non-main sandbox sessions expose host-only managed skill paths in `<available_skills>` even though synced copies exist inside the sandbox workspace.

## Best Possible Solution

Review and land a root fix that rebuilds or remaps available-skills prompt entries from sandbox-readable skill entries whenever a non-rw sandbox attempt or compaction run receives a host-built snapshot, while preserving normal snapshot reuse, skill allowlists, exposure metadata, remote eligibility notes, and skill env overrides.
