## Summary

Keep open. Current main still keeps a host-built skills snapshot prompt for sandboxed non-rw attempt and compaction runs, even though sandbox setup mirrors eligible skills into the sandbox workspace. The open #38789 PR remains the focused root-fix path, while #74161 is related prompt hardening and does not by itself remap stale host paths.

## What This Changes

Sandboxed non-main sessions should expose skill locations that are readable inside the sandbox workspace instead of host-only managed skill paths.

## Best Possible Solution

Review and land a root fix that rebuilds or remaps available-skills prompts from sandbox-readable skill entries whenever a non-rw sandbox run receives a host-built snapshot. The fix should cover both attempt and compaction paths, preserve filters and skill exposure metadata, and treat exact-location prompt wording as defense-in-depth rather than the core repair.
