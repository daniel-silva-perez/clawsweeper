## Summary

Keep open. Current main still has the reported sandbox skill prompt mismatch: the skills snapshot is built from the host workspace, sandbox setup syncs skills into the sandbox workspace, but attempt and compaction runs keep using the original snapshot prompt when resolved snapshot skills are present. The open #38789 PR remains the focused implementation path; #74161 is related prompt hardening but does not by itself fix unreadable locations.

## What This Changes

Sandboxed non-main sessions should expose skill locations that are readable inside the sandbox instead of host-style or `~`-prefixed paths outside the sandbox runtime.

## Best Possible Solution

Make sandboxed non-rw attempt and compaction runs rebuild or remap available-skills prompts from sandbox-readable skill entries whenever a host-built snapshot is present, including avoiding prompt-facing `~` paths in sandbox contexts. Preserve skill filters, exposure metadata, env overrides, remote eligibility notes, and empty-sandbox behavior; treat exact-location prompt wording as supplementary hardening, not the root fix.
