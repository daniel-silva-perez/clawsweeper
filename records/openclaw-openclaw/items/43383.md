## Summary

Keep open. Current main still has the host-built skill snapshot reuse path: non-rw sandbox attempt and compaction runs switch to the sandbox workspace, but reuse the original snapshot prompt and skip live sandbox skill loading when resolved snapshot skills exist. Open #38789 remains the focused root-fix PR; #74161 is related prompt hardening, not the path rebuild/remap fix.

## What This Changes

Sandboxed non-main sessions should expose skill file locations that are readable inside the sandbox instead of host-only managed skill paths.

## Best Possible Solution

Land a root fix that rebuilds or remaps available-skills prompt entries from sandbox-readable skill entries whenever a non-rw sandbox run receives a host-built snapshot, covering attempt and compaction while preserving normal snapshot reuse, skill allowlists, exposure metadata, remote eligibility notes, and skill env overrides.
