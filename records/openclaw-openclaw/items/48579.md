## Summary

Keep open. Current main supports the discussion’s main distinction: `contextPruning.mode: "off"` is respected and disables cache-TTL pruning, but compaction is a separate path. The reported hard-disable or hard-limit-only behavior is still not implemented because public config/schema still only allow `compaction.mode` values `default` and `safeguard`, and the exact Anthropic API-key + Claude Opus 4.6 + 1M-context budget scenario is not covered as a regression. Related open PR #73839 targets the separate safeguard-vs-Pi-auto-compaction conflict tracked by #73003 and does not close this issue.

## Best Possible Solution

Keep this open and narrow it into a config/runtime follow-up: add a regression for the reported Anthropic API-key + Claude Opus 4.6 1M setup, document the independence of context pruning and compaction more explicitly, verify the effective context-window and reserve-token budget used by both Pi and OpenClaw compaction paths, review #73839 for the related safeguard interlock, and make a maintainer product decision on adding `compaction.mode: "off"`/`"error"` or a documented hard-limit-only recipe.
