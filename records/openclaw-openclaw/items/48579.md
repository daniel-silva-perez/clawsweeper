## Summary

Keep open. Current main confirms the discussion’s useful distinction: `contextPruning.mode: "off"` disables the cache-TTL pruning extension, but compaction is a separate runtime path. Current config/schema still has no accepted `compaction.mode: "off"`, and the exact Anthropic API-key + Claude Opus 4.6 + 1M-context + `reserveTokensFloor: 200000` scenario is not covered as a regression. The issue remains a real config/runtime follow-up rather than a solved cleanup item.

## Best Possible Solution

Keep this open and narrow it into a config/runtime task: add regression coverage for the reported Anthropic API-key + Claude Opus 4.6 configuration, document the independence of context pruning and compaction in the user-facing config docs, verify that Anthropic 1M request settings and Pi compaction budgets use the same effective context window, and decide whether to add `compaction.mode: "off"` or document a supported hard-limit-only recipe.
