## Summary

Keep open. Current main still leaves the reported diagnostic gap: `openclaw infer embedding create` reaches the embedding provider directly, and failures bubble to the shared CLI wrapper, which prints `String(err)` instead of a cause-aware/redacted message. The Voyage path does use the guarded remote HTTP helper and current fetch-guard fallback, so part of the reporter's dispatcher audit is already addressed, but the nested `fetch` cause/code is still not surfaced and there is no regression test for this CLI failure mode.

## Best Possible Solution

Keep this issue open. The best fix is to make the `infer embedding create` error path cause-aware and redacted, preserving nested `fetch` cause messages and codes such as `UND_ERR_INVALID_ARG`, then add a regression test that injects a `TypeError("fetch failed")` with a nested Undici cause. In parallel, audit the Voyage embedding path for any custom `fetchImpl` or dispatcher mixing and run a Node 24 Voyage CLI smoke when credentials are available.
