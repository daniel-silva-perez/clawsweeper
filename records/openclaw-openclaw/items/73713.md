## Summary

Keep open. Current main still preserves the reported `infer embedding create` diagnostic gap: provider errors flow through the shared CLI wrapper, which prints `String(err)` and drops nested Undici causes. The Voyage path uses the shared guarded remote embedding helper and does not pass a custom fetch implementation, so the remaining actionable work is focused on cause-aware error preservation rather than a confirmed Voyage-specific transport bypass.

## What This Changes

The issue asks OpenClaw to make `openclaw infer embedding create --provider voyage` succeed on Node 24 when the credential works directly, and to surface nested fetch/Undici causes when embedding provider fetches fail.

## Best Possible Solution

Preserve the current guarded fetch and SSRF architecture, but make embedding provider failures cause-aware: use a shared redacting formatter in the CLI wrapper and wrap remote embedding fetch exceptions with provider context while preserving the original cause and safe Undici code details. Avoid changing Voyage auth, model defaults, or dispatcher construction unless the improved diagnostics still show a concrete transport bug.

## Reproduction Assessment

Partial. There is no live Voyage credential available here, but the diagnostic bug has a high-confidence local reproduction: a `TypeError('fetch failed')` with an `UND_ERR_INVALID_ARG` cause reaches `runtime.error(String(err))`, which prints only the top-level message.

## Solution Assessment

Yes. The narrow maintainable path is an error-preservation fix in the shared CLI/remote embedding helper path, with regression tests for nested causes; broader provider or dispatcher changes should wait for cause-rich live evidence.
