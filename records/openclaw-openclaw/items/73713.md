## Summary

Keep open. Current main still does not close the reported diagnostic gap for `openclaw infer embedding create`: the embedding command awaits the provider directly, the shared CLI wrapper still prints `String(err)`, and the remote embedding JSON helper only prefixes non-OK HTTP responses, not network/fetch exceptions with nested causes. The Voyage path does use the shared guarded remote embedding helper and does not pass a custom `fetchImpl`, so the transport suspicion was partly audited, but the issue’s requested cause-preserving failure output is still missing.

## Best Possible Solution

Keep the issue open. The best fix is to make the `infer embedding create` failure path cause-aware, either by using `formatErrorMessage` in `runCommandWithRuntime` or by adding a command-specific error handler, and to wrap remote embedding fetch exceptions with the provider `errorPrefix` while preserving the original `cause` and Undici `code`. Add regression coverage that injects `TypeError('fetch failed')` with an `UND_ERR_INVALID_ARG`-style cause, then run a live Node 24 Voyage CLI smoke with valid credentials.
