## Summary

Keep open. Current main still does not close the reported Node 24 Voyage embedding CLI failure or the diagnostic gap: the embedding CLI calls the provider directly, remote embedding fetch exceptions are not wrapped with provider/cause detail, and the shared CLI wrapper still prints String(err), which drops nested fetch causes such as Undici error codes.

## Best Possible Solution

Keep the issue open. The best fix is to make the infer embedding create failure path cause-aware, likely by using the existing formatErrorMessage/redaction path in runCommandWithRuntime or by adding a command-specific onError, and to wrap remote embedding fetch exceptions with the provider errorPrefix while preserving the original cause/code. Add a regression test that injects TypeError('fetch failed') with an UND_ERR_INVALID_ARG-style cause and audit the Voyage provider for any custom fetchImpl or dispatcher mixing before running a live Node 24 Voyage CLI smoke.
