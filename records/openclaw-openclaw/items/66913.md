## Summary

Keep open: current main still truncates the visible `memory_forget` candidate IDs to 8 characters while direct deletion validates the full UUID format, and this PR is a focused implementation with regression coverage for the visible candidate list.

## What This Changes

The PR changes `memory-lancedb` so `memory_forget` prints full candidate UUIDs in the visible list and adds a Vitest regression for that output.

## Best Possible Solution

Land the focused candidate-list fix with its regression test if maintainers accept full UUIDs as the current deletion handle, then keep any broader memory recall or safer-handle design work tracked separately in #42665.
