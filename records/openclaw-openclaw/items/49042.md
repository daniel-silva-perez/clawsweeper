## Summary

Keep open. Current main still does not implement the PR’s intended `agent_end.finalLlmOutcome` plugin contract. The existing `agent_end` hook exposes only the legacy run summary fields, and the newer `model_call_ended` telemetry is adjacent but not equivalent because it is per model call and lacks the requested terminal `agent_end` outcome object with stop reason, HTTP status, and sanitized final error metadata.

## Best Possible Solution

Keep this PR open as a plausible additive plugin API change. The best path is to rebase it onto current main’s `src/plugins/hook-types.ts` contract split and current harness emitters, keep the implementation focused on `agent_end.finalLlmOutcome`, preserve legacy `success` semantics, document `finalLlmOutcome.ok` versus `success` at https://docs.openclaw.ai/plugins/hooks, and add targeted coverage for terminal assistant errors, prompt/build failures, provider HTTP failures, provider timeouts, and compaction/runner failure precedence.
