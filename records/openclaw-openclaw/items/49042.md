## Summary

Keep open. Current main still lacks the requested `agent_end.finalLlmOutcome` contract, so the PR is not obsolete; however, the branch should not merge as-is because it weakens legacy error redaction and still misclassifies provider-call timeouts as runner outcomes.

## What This Changes

The branch adds optional `finalLlmOutcome` metadata to the `agent_end` plugin hook and embedded runner docs/tests, while also changing several unrelated test stabilization files.

## Best Possible Solution

Keep the PR open as a focused additive plugin API candidate, but require a rebase onto current main, move the contract/docs updates to the current hook files, keep legacy `agent_end.error` on the redacted formatter path, classify provider-call timeouts as provider outcomes, and split or justify the unrelated test stabilization edits.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the missing feature because `finalLlmOutcome` has no source/docs/test hits and the emitted `agent_end` payload is still legacy-only; the PR defects are reproducible by reading the helper branches or calling them with a provider-started `TimeoutError` and an error message containing a token.

## Solution Assessment

No, not as-is. The requested additive `agent_end.finalLlmOutcome` direction is reasonable, but the current branch is not the safest implementation until the redaction regression, provider-timeout source handling, stale file layout, and unrelated test edits are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep legacy agent_end errors redacted:** `src/agents/pi-embedded-runner/run/attempt.ts:3009`
  - body: This changes the existing top-level `agent_end.error` from the redacted `formatErrorMessage` path to `describeUnknownError`, which returns raw `Error.message`, strings, or JSON. Provider exceptions can include auth headers or tokens, so plugins could receive unredacted secrets through the legacy field even though the new `finalLlmOutcome.errorMessage` is sanitized.
  - confidence: 0.92
- **[P2] Classify provider-call timeouts as provider outcomes:** `src/agents/pi-embedded-runner/run/attempt.ts:1422`
  - body: After `providerCallStarted` is set, a timeout thrown by `activeSession.prompt()` still satisfies `isTimeoutError(promptError)` and takes the runner-failure branch. That reports `source: "runner"` for provider/network timeouts, so plugins cannot distinguish provider timeouts from local runner aborts.
  - confidence: 0.86
