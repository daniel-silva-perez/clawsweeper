## Summary

Keep this PR open. Current main still does not expose the requested optional `agent_end.finalLlmOutcome` contract, and the adjacent `model_call_ended` telemetry is per model call rather than a terminal agent outcome with stop reason, HTTP status, and sanitized final error metadata.

## Best Possible Solution

Keep this PR open as a focused additive plugin API candidate. Rebase onto current main, add optional `PluginHookAgentEndEvent.finalLlmOutcome` in `src/plugins/hook-types.ts`, populate it from the embedded runner and Codex harness/shared hook helper without changing legacy `success`, document `finalLlmOutcome.ok` versus `success` in `docs/plugins/hooks.md`, add targeted tests for terminal assistant errors, prompt/build failures, structured provider HTTP failures, provider timeouts, and compaction/runner precedence, and split unrelated test-only stabilization edits.
