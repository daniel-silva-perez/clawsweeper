## Summary

Keep this PR open. Current main still does not implement the central Gemma transport fix: dynamic Gemma resolution only patches reasoning, explicit provider-model normalization only normalizes model IDs, and runtime resolution still lets configured Gemma models or provider configs stay on `openai-completions`. The PR addresses a real remaining Google provider bug, but it needs revision before merge because review comments identified `google-gemini-cli`/baseUrl and provider-normalization edge cases.

## Best Possible Solution

Keep this PR open, but revise it before merge: implement Gemma native routing in the Google plugin owner code for both dynamic and explicit model configs, add assertions for `api` and compatible `baseUrl` on `google` and `google-gemini-cli` paths, and address the existing review comments. Once merged, use the landed fix to close the remaining Gemma transport reports such as #63685 if current behavior is covered.
