## Summary

Keep this issue open. Current main still builds the Skills section with unconditional read-based guidance, even though tool names are normalized earlier and plugin tools can be registered and forwarded into prompt construction. The open #63940 PR is adjacent prompt wording work, but it is unmerged and its diff still keeps the read-based loading path.

## Best Possible Solution

Keep this issue open. The best fix belongs in `src/agents/system-prompt.ts`: pass normalized available-tool information and caller casing into `buildSkillsSection`, prefer invoking a registered Skill tool when present, preserve the existing read fallback otherwise, and add focused coverage in `src/agents/system-prompt.test.ts`. The docs at `docs/concepts/system-prompt.md` should describe both paths after the behavior changes, and #63940 should be reconciled if it lands first because it touches the same prompt text.
