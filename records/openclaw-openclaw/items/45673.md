## Summary

Keep this PR open. Current main implements the basic Feishu tool precedence change, but it still lacks the PR’s final channel-origin guard: a non-Feishu session can still supply an `agentAccountId` that collides with a configured Feishu account and win over `channels.feishu.defaultAccount`. The PR discussion and later commits specifically added `messageChannel` threading to address that remaining case, so current main does not fully implement the PR’s intended behavior.

## Best Possible Solution

Keep this PR open for maintainer review or a narrow replacement patch. The desired Feishu plugin behavior is: explicit tool `accountId` first; normalized inherited account only when `ctx.messageChannel` is Feishu and the account maps to an enabled Feishu account; then configured `channels.feishu.defaultAccount`; then normal Feishu default selection. Thread `ctx.messageChannel` through wiki, drive, perm, bitable, doc/media/scopes account resolution, and add targeted tests for Feishu-origin routing, non-Feishu collision fallback, unknown-account fallback, normalized account IDs, explicit override, and configured default fallback.
