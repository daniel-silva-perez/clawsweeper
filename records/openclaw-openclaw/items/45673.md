## Summary

Keep this PR open. Current main implements the basic Feishu tool precedence change, but it still lacks the PR’s final review-driven guard that only trusts inherited account context for Feishu-origin sessions. The remaining work is scoped to the Feishu plugin and protects against cross-channel account-id collisions selecting the wrong Feishu tenant.

## Best Possible Solution

Keep this PR open for maintainer review or a narrow replacement patch. Rebase it onto current main and preserve the intended order: explicit `accountId` > normalized enabled contextual account only when `messageChannel` is Feishu > configured `defaultAccount` > normal Feishu default selection. Thread `ctx.messageChannel` through Feishu tool account resolution for wiki, drive, perm, bitable, doc/media/scopes, and add targeted tests for Feishu-origin contextual routing, non-Feishu fallback, explicit override, unknown-account fallback, normalized account IDs, and configured default fallback.
