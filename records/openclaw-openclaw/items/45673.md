## Summary

Keep this PR open. Current main has the basic Feishu tool precedence change, but it still lacks the PR discussion’s final channel-origin guard, so non-Feishu sessions with a colliding account ID can still override `channels.feishu.defaultAccount`. The PR branch’s later commits and comments specifically address that remaining behavior by threading `messageChannel` through Feishu tool resolution.

## Best Possible Solution

Keep this PR open for maintainer review, rebase, or a narrow replacement patch. The desired behavior is explicit tool `accountId` first; normalized inherited account only when `ctx.messageChannel` is Feishu and the account maps to an enabled Feishu account; then configured `channels.feishu.defaultAccount`; then normal Feishu default selection. Add tests for Feishu-origin routing, non-Feishu collision fallback, unknown-account fallback, normalized account IDs, explicit override, and configured default fallback across the shared Feishu tool-account helper and representative tools.
