## Summary

Current main still has the zero-minute Control UI chat-session refresh paths this PR targets, and the patch is a narrow consistency fix with focused regression coverage. I found no blocking correctness or security findings, so this should stay open for maintainer review and merge rather than be closed as obsolete.

## What This Changes

The PR changes three Control UI chat-session refresh callsites to pass the existing `CHAT_SESSIONS_ACTIVE_MINUTES` constant instead of `activeMinutes: 0`, and adds/updates UI tests for chat refresh and session switching.

## Best Possible Solution

Land this PR or an equivalent scoped UI fix so every current Control UI chat-session refresh path uses the existing shared active-session window while preserving the constant value, `sessions.list` contract, Dreaming filters, cron filtering, and config/schema behavior.

## Reproduction Assessment

Yes. On current main, inspecting the three refresh callsites plus `loadSessions` shows `activeMinutes: 0` is dropped before the `sessions.list` RPC, and the existing session-switch test still locks in that old zero-minute expectation.

## Solution Assessment

Yes. Reusing the existing constant at the three affected callsites is the narrowest maintainable fix; moving the constant to a smaller shared module could be a follow-up preference, but it is not required for correctness.
