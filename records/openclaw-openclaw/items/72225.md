## What This Changes

The branch changes three Control UI chat-session list refresh paths and associated UI tests so they use `CHAT_SESSIONS_ACTIVE_MINUTES` instead of `activeMinutes: 0`.

## Best Possible Solution

Land this PR or an equivalent scoped UI fix so every current Control UI chat-session refresh path uses the existing shared active-session window while preserving the 120-minute value, `sessions.list` contract, Dreaming filters, cron filtering, and config/schema behavior.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the issue: the three UI refresh paths pass `activeMinutes: 0`, and `loadSessions` drops non-positive active-minute values before calling `sessions.list`.

## Solution Assessment

Yes. Reusing the existing constant at the three affected callsites is the narrowest maintainable fix; a constant relocation could be a follow-up preference but is not required for correctness.
