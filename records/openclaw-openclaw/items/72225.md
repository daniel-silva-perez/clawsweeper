The PR changes three Control UI chat-session refresh callsites to pass `CHAT_SESSIONS_ACTIVE_MINUTES` instead of `activeMinutes: 0`, and adds/updates tests for initial chat refresh and session switching.

## Best Possible Solution

Land this PR or an equivalent scoped Control UI fix so every current chat-session refresh path uses the existing shared active-session window while preserving the constant value, `sessions.list` contract, Dreaming filters, cron filtering, and config/schema behavior.
