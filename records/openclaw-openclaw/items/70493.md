## Summary

Keep this open. Current main still has sandboxed/sub-agent browser calls routed through the sandbox bridge, while Playwright-backed navigate/act/snapshot paths can still emit the reported 501 when that runtime cannot load the Playwright module; the issue thread also has explicit maintainer direction not to close until isolated-session coverage exists.

## What This Changes

The issue requests consistent Playwright-backed browser automation for isolated sub-agent sessions so navigate, snapshot, and act work like the main session.

## Best Possible Solution

Define the browser runtime owner for isolated sessions, implement that path without weakening sandbox policy, and add a regression test that exercises navigate or snapshot through the real isolated sub-agent/sandbox bridge path without returning the Playwright-unavailable error.
