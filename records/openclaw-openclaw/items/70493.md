## Summary

This should stay open: current main still has isolated/sandbox browser calls using the sandbox bridge and the same Playwright-dependent routes, there is no isolated-session navigate/snapshot regression coverage, and the discussion includes explicit maintainer direction not to close until that coverage exists.

## What This Changes

Request browser navigate, snapshot, and act support in isolated sub-agent/sandbox gateway sessions with the same Playwright-backed capability available to main-session browser use.

## Best Possible Solution

Define the browser runtime owner for isolated sessions, implement that path without weakening sandbox policy, and add an isolated sub-agent/sandbox bridge regression test proving navigate or snapshot does not return the Playwright-unavailable error when the owning browser runtime supports Playwright.
