## Summary

Keep open. Current main still defaults sandboxed browser tools to the sandbox bridge, that bridge registers the same Playwright-gated browser routes, and there is explicit maintainer direction in the discussion not to close this until isolated-session browser coverage exists.

## What This Changes

The issue requests Playwright-backed browser automation for isolated sub-agent sessions so navigate, snapshot, and act work consistently with the main browser session.

## Best Possible Solution

Define the browser runtime owner for isolated sessions, implement that path without weakening sandbox boundaries, and add an isolated sub-agent or sandbox-bridge regression test proving navigate or snapshot no longer returns the Playwright-unavailable error when the owning runtime supports Playwright.
