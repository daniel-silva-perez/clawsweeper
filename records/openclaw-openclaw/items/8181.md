## Summary

Keep this issue open. Current main has partial explicit local embedding support: `provider: "local"` can fall back to the default EmbeddingGemma GGUF URI and resolve it through `node-llama-cpp`. The zero-config request is still not complete because `node-llama-cpp` is not installed as a normal runtime dependency, setup errors and docs still require manual install/approve/rebuild steps, and clean `provider: "auto"` still skips the default downloadable local model unless a real local file path is already configured.

## Best Possible Solution

Keep this open until maintainers either implement true turnkey local embeddings or explicitly decide that local embeddings remain a manual optional runtime. A complete fix should handle native runtime install/build or prebuilt binaries, first-use GGUF download with progress and clear failure messaging, clean `auto` versus explicit `local` semantics, docs, and regression tests.
