## Summary

Keep open. Current main has adjacent mitigations, including PDF-specific tooling, attachment offload, and binary media skipping, but the core `read` tool still has no general binary-document placeholder or DOCX/XLSX/PPTX preprocessing path, so the reported context-pollution problem is not fully solved.

## What This Changes

The issue requests detecting binary document types such as DOCX, XLSX, PDF, and PPTX before file contents enter model context, extracting useful text when possible or replacing unsupported binary content with a concise placeholder.

## Best Possible Solution

Add a MIME-aware guard to the OpenClaw `read` wrapper so known binary documents never enter model context as decoded bytes. Keep PDF analysis on the existing `pdf` tool/document-extract path, and treat Office extraction as a follow-up plugin or document-extractor capability rather than a broad core parser bundle.

## Reproduction Assessment

Yes. A focused source-level reproduction is to read a DOCX/XLSX/PPTX-like binary file through the core `read` tool: current OpenClaw only post-processes images after delegating to the base read tool, and the upstream non-image path decodes the buffer as UTF-8. I did not run a temp-file live test because this review is read-only.

## Solution Assessment

No, current main is not yet the best solution for this issue. PDF-specific tooling and attachment offload are useful adjacent work, but the maintainable fix for the reported `read` behavior is a small binary-document guard first, with richer extraction handled through the existing plugin-oriented document path.

## Repair Work Prompt

Implement a narrow fix for https://github.com/openclaw/openclaw/issues/56876. Add MIME-aware binary document preprocessing to the OpenClaw `read` tool path so non-image binary documents do not get decoded into model context. Scope: detect known binary/document formats by magic/MIME plus extension for host and sandbox reads, including PDF, DOC/DOCX, XLS/XLSX, PPT/PPTX, ZIP/OLE/octet-stream cases where the filename identifies the document type.
