## Summary

Current main still does not implement the requested security/API contract. Skills cannot declare heartbeat-callable sandboxed functions with parameter schemas, heartbeat config has no remote structured invocation surface, and heartbeat execution still builds a text prompt and sends it through the model. The earlier ClawSweeper comment reached the same result; this re-check confirms it on ef58307f843eed1a7832e9c23c519591f1b7cdbd.

## Best Possible Solution

Keep this open for maintainer product/security design. A complete fix should add a versioned metadata/API contract for declared callable functions, document and parse it, add a structured remote heartbeat fetch/parse path, validate call names and params against declared schemas, execute only through existing tool/plugin capability boundaries, and add regression tests proving unknown calls, schema failures, and prompt-injection text are not delivered to the model as free-form heartbeat instructions. ClawHub trust indicators can follow after the core contract exists.
