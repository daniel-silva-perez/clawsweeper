## Summary

Keep this PR open. Current main still queues WhatsApp gateway connect/disconnect banners into prompt-bound system events, but the PR's broad `WhatsApp gateway disconnected` filter would also suppress the only queued prompt-visible signal for terminal WhatsApp close modes such as logged-out, conflict, or max-attempt stop. The security review did not find CI, dependency, script, secret, artifact, or package-resolution changes in the PR context; the blocker is functional correctness.

## Best Possible Solution

Keep the PR open and request a narrower implementation: suppress routine connect banners and only suppress disconnect events that are proven retryable/routine, or first change the WhatsApp monitor to enqueue distinct high-signal terminal system events before prompt filtering. Add tests for both routine flaps and terminal close modes. Treat #72043 as a broader configurable system-event policy discussion, not as a replacement for this focused WhatsApp fix.
