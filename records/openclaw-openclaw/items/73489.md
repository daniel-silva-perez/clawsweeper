## Summary

Keep this PR open. Current main still has the raw Signal `node:https` archive downloader described by #54153, so the PR is not obsolete; it is the active implementation candidate. The diff is narrow and security-positive, but it needs normal review, especially around the proxy-support claim and missing focused tests for the new size cap.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for #54153. The likely best fix is this PR or a revised equivalent that moves the Signal archive download to `fetchWithSsrFGuard`, keeps the timeout and size cap, adds focused tests for the Signal download wrapper, and either proves or corrects the proxy-support behavior and changelog wording before merge.
