Keep this PR open. Current main still has the raw Signal signal-cli archive downloader that this PR is trying to replace, and the linked issue #54153 remains open. The PR is recent, unmerged, narrow, and directly targets an unresolved security hardening gap rather than being obsolete cleanup.

## Best Possible Solution

The best path is to revise or land this PR, or an equivalent patch, that moves the Signal archive download to fetchWithSsrFGuard, keeps bounded timeout and size caps, makes the proxy behavior explicit, adds focused Signal installer transport tests, and then closes #54153 after merge.
