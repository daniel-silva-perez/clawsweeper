## Summary

Keep open. Current main still lacks the requested WebChat/Control UI inline-button path: the Gateway protocol, method list, scopes, WebChat config, final chat projection, and Control UI chat model/render pipeline all omit button payloads and callback handling. The related closed #54720 appears to be the narrower duplicate, so this remains the canonical tracker rather than something to close as stale.

## What This Changes

Add WebChat/Control UI inline-button support with a WebChat capability toggle, final chat-event button payloads, a callback RPC, and Control UI rendering/click handling.

## Best Possible Solution

Keep this as the canonical tracker and implement an additive, presentation-based WebChat interactive-control path: add a deliberate `gateway.webchat` capability/config contract, carry sanitized final-only button blocks through chat events and history, render them in Control UI with clicked/disabled state, and add a scoped `chat.callback` RPC that initially routes values back into the same session. Treat webhook callback routing as a separate product/security decision.
