## Summary

Keep this issue open. Current main still lacks WebChat/Control UI inline-button support: there is no WebChat capability config, no typed button payload or callback RPC in the Gateway chat protocol, no server handler/method scope for callbacks, and the Control UI chat model/render path only normalizes text, tools, attachments, and canvas blocks.

## What This Changes

Add WebChat/Control UI inline button support, including a capability config path, final chat button payloads, a callback RPC, and Control UI rendering/routing.

## Best Possible Solution

Keep this issue open as the canonical tracker. Implement an additive WebChat/Control UI path around the existing shared `presentation` model: add a deliberate WebChat capability config, carry final presentation button blocks through chat event/history payloads, render them in Control UI with disabled/clicked state, and add a scoped `chat.callback` RPC that initially injects callback values back into the same session. Defer webhook callback routing to explicit maintainer product/security review.
