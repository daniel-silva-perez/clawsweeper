## Summary

Keep this issue open. Current main still lacks the requested WebChat/Control UI inline-button contract: no WebChat inlineButtons config, no buttons field or chat.callback RPC in the gateway chat protocol, and no Control UI normalized content or rendering path for button blocks. The related closed item #54720 appears to have been closed back to this older, more detailed tracker, so this issue remains the canonical place for the remaining work.

## Best Possible Solution

Keep this issue open as the canonical tracker. The eventual fix should use the current semantic `presentation` contract rather than adding a new provider-native `buttons` escape hatch, and should cover gateway config/protocol, final assistant message payload projection, Control UI normalization/rendering, click routing via a WebChat callback or chat-send path, docs, and regression tests.
