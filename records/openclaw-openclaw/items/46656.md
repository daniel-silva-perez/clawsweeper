## Summary

Keep open. Current main still lacks the requested WebChat/Control UI inline-button path: there is no WebChat button or presentation payload in the chat WebSocket schema, no `chat.callback` method or scope, no WebChat inline-button config, no projection of presentation/button metadata from WebChat replies, and no Control UI button content item or renderer. The closed #54720 is a narrower duplicate, while #46841 is Telegram-specific webhook routing, so this issue remains the active WebChat tracker.

## What This Changes

Add WebChat/Control UI inline button support, including a capability config path, final chat payload rendering, and a callback RPC or webhook route for button clicks.

## Best Possible Solution

Implement an additive WebChat interactive-control path based on the shared `presentation` contract: preserve sanitized final-only button/select blocks through chat events and history, render them in Control UI with clicked/disabled state, and add a scoped `chat.callback` RPC that initially routes selected values back into the same session. Treat webhook routing as a separately hardened follow-up unless maintainers explicitly include it in the first design.
