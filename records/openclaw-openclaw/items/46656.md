## Summary

Keep open. Current main still lacks the requested WebChat/Control UI inline-button path: the WebSocket chat schemas, method registry/scopes, WebChat config, final chat broadcast, and Control UI message model/rendering all omit interactive button payloads and callback handling. The closed #54720 is a narrower duplicate, so this remains the better canonical tracker.

## What This Changes

Add WebChat/Control UI inline button support, including a capability/config path, final chat payload support, callback RPC handling, and Control UI rendering for agent-authored buttons.

## Best Possible Solution

Keep this as the canonical tracker and implement an additive WebChat interactive-control path based on the shared `presentation` model: carry sanitized final-only button blocks through chat events/history, render them in Control UI with clicked/disabled state, and add a scoped `chat.callback` RPC that initially routes selected values back into the same session. Treat webhook routing as a separate product/security decision.
