## Summary

Current main still does not implement the requested on-demand HEARTBEAT.md behavior for normal non-heartbeat runs. HEARTBEAT.md remains in the fixed workspace bootstrap load list, the session cache still loads the full bootstrap set before filtering, and docs still describe normal runs injecting HEARTBEAT.md when heartbeat guidance is enabled. The related #55311 lightweight-heartbeat fix is useful but narrower than this issue.

## Best Possible Solution

Keep this issue open. Implement a run-kind-aware workspace bootstrap loader/cache path that can skip HEARTBEAT.md for default non-heartbeat runs before disk reads, session snapshot caching, hooks, and prompt injection, while preserving HEARTBEAT.md for actual heartbeat runs and the already-fixed #55311 lightweight heartbeat path. Update the public heartbeat and system-prompt docs if the normal-run contract changes.
