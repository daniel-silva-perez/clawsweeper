## Summary

Current main still does not implement strict on-demand HEARTBEAT.md loading for normal non-heartbeat runs. HEARTBEAT.md remains part of the fixed workspace bootstrap load list, is read before filtering/cache boundaries, and docs still describe normal runs injecting it when heartbeat guidance is enabled. The related #55311 lightweight-heartbeat path is fixed, but that is narrower than this request.

## Best Possible Solution

Keep this issue open. Implement a run-kind-aware workspace bootstrap loader/cache path that can skip HEARTBEAT.md for default non-heartbeat runs before disk reads, session snapshot caching, hooks, and prompt injection, while preserving HEARTBEAT.md for actual heartbeat runs and the already-fixed #55311 lightweight heartbeat behavior. Update the public heartbeat/system-prompt docs if the normal-run contract changes.
