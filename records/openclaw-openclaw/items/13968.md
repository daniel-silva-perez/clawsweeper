## Summary

Current main still does not implement strict on-demand HEARTBEAT.md loading for normal non-heartbeat runs. The `isHeartbeat`/run-kind signal reaches the embedded runner, and #55311's lightweight heartbeat path is fixed, but `HEARTBEAT.md` is still in the default workspace bootstrap loader, still goes through the session bootstrap cache before filtering, and current docs explicitly describe normal runs retaining `HEARTBEAT.md` when heartbeat guidance is enabled. The prior Codex review comment remains accurate against current main.

## Best Possible Solution

Keep this issue open. Implement a run-kind-aware bootstrap loader/cache contract that excludes `HEARTBEAT.md` for default non-heartbeat runs before disk reads, session snapshot caching, hooks, and prompt injection, while preserving `HEARTBEAT.md` for actual heartbeat runs and the already-fixed #55311 lightweight heartbeat behavior. Add focused tests for enabled default non-heartbeat omission, cache behavior, disabled cadence/prompt regressions, and heartbeat retention, then update the public heartbeat docs if the contract changes.
