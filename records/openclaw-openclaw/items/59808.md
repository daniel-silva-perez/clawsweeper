## Summary

Keep open. Current main still has the mcporter-backed QMD search path parsing the entire `mcporter call --output json` stdout with `JSON.parse(result.stdout)`, while daemon startup is invoked separately with no stdout-discard option and the regression tests only cover clean mcporter stdout. The reported first-query fallback remains plausible and is not implemented-on-main.

## Best Possible Solution

Keep the issue open and patch `extensions/memory-core/src/memory/qmd-manager.ts`: expose a `discardStdout`/`discardOutput` option for `runMcporter()` and use it for `daemon start`, then make `runQmdSearchViaMcporter()` parse the structured mcporter payload defensively when stdout contains leading mcporter log lines. Add a regression test for the first `startDaemon=true` search where mcporter emits `[mcporter] ...` before `{"results":[]}`, and verify with the supported mcporter/qmd versions.
