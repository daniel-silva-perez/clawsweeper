## Summary

Keep open. Current main still has the mcporter-backed QMD search path parsing the full `mcporter call --output json` stdout directly, while the daemon-start wrapper still cannot discard stdout; the prior Codex review remains accurate after the latest main changes.

## What This Changes

Fix the first `memory_search` after gateway restart so mcporter/QMD startup log lines cannot make vector search fall back to the builtin FTS path.

## Best Possible Solution

Patch the memory-core QMD mcporter wrapper so daemon startup output is discarded and `runQmdSearchViaMcporter()` defensively extracts the structured JSON payload when stdout contains known mcporter log lines, with a focused regression test for the first `startDaemon=true` search after restart.

## Reproduction Assessment

Yes. The current code gives a high-confidence unit reproduction path: configure QMD mcporter with `startDaemon: true`, have the mocked first mcporter call emit a `[mcporter] ...` line before `{"results":[]}`, and observe `JSON.parse(result.stdout)` throwing before the fallback manager switches to builtin search.

## Solution Assessment

Yes, a narrow memory-core fix is the best repository-side solution. Upstream mcporter should ideally keep logs on stderr, but OpenClaw should still discard daemon-start stdout and parse mcporter call output defensively so supported versions do not break memory search on startup noise.

## Repair Work Prompt

Fix issue https://github.com/openclaw/openclaw/issues/59808. Observable bug: after a gateway restart, the first `memory_search` through QMD mcporter can fall back to builtin FTS because mcporter startup/log output appears before the JSON payload and `runQmdSearchViaMcporter()` parses the whole stdout buffer. Keep the fix inside `extensions/memory-core/src/memory/qmd-manager.ts` unless a tiny host-sdk type seam is unavoidable. Expose a `discardOutput`/`discardStdout` option on the local `runMcporter()` wrapper and use it for `mcporter daemon start`; also make mcporter call parsing tolerate leading known mcporter log lines while still rejecting malformed non-JSON output. Add a regression test in `extensions/memory-core/src/memory/qmd-manager.test.ts` for `mcporter.startDaemon=true` where the first search sees `[mcporter] ...` before `{"results":[]}` and still returns QMD results without activating builtin fallback. Do not bypass the existing QMD/mcporter version fallback logic or broaden accepted arbitrary stdout. Add a user-facing changelog entry for the memory search fix with `Thanks @Silver-Aurora` if the active changelog policy requires it.
