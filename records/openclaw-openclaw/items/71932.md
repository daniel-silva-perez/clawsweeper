## Summary

Keep open. A newer comment adds a concrete Google provider config where the provider-owned model entry itself is `google/<model>`, and current `main` still has paths that can carry that prefixed id into the native Google request URL as `models/google/<model>`. Related Google prefix fixes remain open in #41398 and #41517, so this should be consolidated rather than closed as already implemented.

## What This Changes

The issue asks to fix Google Generative AI 404s when sub-agent spawn or session reset uses provider-prefixed Gemini model refs such as `google/gemini-3.1-flash-lite-preview`.

## Best Possible Solution

Keep this issue open and consolidate the Google prefix work into one narrow fix that strips only redundant active-provider prefixes before Google model matching and native transport URL construction, preserves valid nested model ids for wrapper providers, and adds regression coverage for sub-agent spawn/reset plus configured Google model entries whose ids are written as `google/<model>`.
