## Summary

Keep this PR open. Current main and the latest release still keep Slack thread participation only in a process-local dedupe cache, while Slack mention gating depends on that cache for non-mention replies in bot-participated threads. The PR remains a useful implementation candidate, but it overlaps with other persistent-store work and still needs maintainer storage/security review before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to preserve the Slack restart-survival behavior, then either migrate it to the approved shared persistent keyed-store/dedupe primitive or harden the Slack-local store with explicit storage/security sign-off. Before merge, trim unrelated churn and keep focused tests for restart survival, clear semantics, TTL expiry, corrupt or oversized persisted files, entry bounds, and storage failure behavior.
