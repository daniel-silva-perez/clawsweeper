## Summary

Keep this PR open. Current main still stores Slack thread participation only in a process-local dedupe cache, and the Slack mention-gating path still depends on that cache to permit non-mention replies in bot-participated threads. The latest shipped release has the same memory-only behavior. The PR is an active implementation candidate, but it still needs maintainer coordination with overlapping persistent-store work and a security/storage review before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to preserve the Slack restart-survival behavior, then either harden and narrow this Slack-local implementation or migrate it to the approved shared persistent-store direction. Before merge, trim unrelated churn and keep focused tests for restart survival, clear semantics, TTL expiry, corrupt or oversized persisted files, bounds, and storage hardening.
