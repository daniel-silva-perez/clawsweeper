## Summary

Keep this PR open. Current main still keeps Slack thread participation in a process-local dedupe cache with no disk hydrate/write path, while Slack mention gating still relies on that cache for non-mention thread continuation after the bot has participated. Related persistence work in #58731 and #70626 is still open and not a shipped replacement, and the contributor is actively asking for maintainer guidance rather than leaving an obsolete PR behind.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to preserve the Slack restart-survival behavior, then either migrate Slack thread participation to the shared async persistence primitive or explicitly approve a hardened Slack-local store. The final branch should remove unrelated churn outside the Slack surface and keep focused restart-survival, clear, corrupt-file, bounds, and security regression coverage.
