## Summary

Keep open. Current main still keeps Slack thread participation in memory only, so this PR targets a real restart-survival gap, but the latest patch should not merge as-is because the custom state-file hydration still follows non-regular files and the storage direction needs maintainer alignment with merged #74190 and overlapping #58731.

## What This Changes

This PR adds JSON-backed disk persistence, lazy hydration, clear/restart coverage, and a Slack API-barrel comment for the Slack thread participation cache.

## Best Possible Solution

Land one owner-approved Slack restart-survival implementation, preferably on the merged plugin state store unless maintainers explicitly choose a hardened JSON path, preserving account/channel/thread keying, 24h TTL, bounded growth, clear semantics, restart/corrupt-state coverage, and compatibility with the current synchronous Slack hot path or a deliberate async refactor.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Reject non-regular state files before hydration:** `extensions/slack/src/sent-thread-cache.ts:103-110`
  - body: `hydrateFromDisk` calls `statSync` and then `readFileSync` on the configured state path without rejecting symlinks, FIFOs, devices, or other non-regular files. A tampered local state path can block the first Slack event or read unintended local content before the cache falls back, so reject non-regular files before the size check/read or move this onto the keyed state store.
  - confidence: 0.87
- **[P3] Remove temp files when rename fails:** `extensions/slack/src/sent-thread-cache.ts:217`
  - body: `persistToDisk` writes a UUID-named temp JSON file and then renames it, but a thrown `renameSync` leaves the temp file behind while the scheduled persist path swallows the error. Wrap the rename so failed writes remove `tmpPath` and do not accumulate Slack thread identifiers in leftover temp files.
  - confidence: 0.82
- **[P3] Initialize tempDir before the directory-mode test:** `extensions/slack/src/sent-thread-cache.test.ts:311-312`
  - body: This test builds `path.join(tempDir, ...)` without calling `setup()` in the test. Running it by itself can leave `tempDir` undefined and fail before the directory-mode assertion, so create a temp dir in the test or call the shared setup helper first.
  - confidence: 0.79

## Security Review

Status: needs_attention

Summary: The diff adds local filesystem persistence for Slack thread state and still has concrete hardening gaps around non-regular state files and failed temp-file cleanup.

Concerns:

- **[medium] Hydration follows non-regular state files:** `extensions/slack/src/sent-thread-cache.ts:103`
  - body: The persisted state path is read with `statSync` and `readFileSync` without rejecting symlinks, FIFOs, or device files, which can turn local state-path tampering into a gateway event-loop block or unintended local file read before Slack message handling continues.
  - confidence: 0.87

  - confidence: 0.82
