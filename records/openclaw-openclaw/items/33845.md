## Summary

Keep this PR open. Current main still keeps Slack thread participation only in a process-local dedupe cache, and the Slack mention-gating path still depends on that cache to let bot-participated threads bypass repeated mentions. The PR is therefore not obsolete or implemented on main. Related PRs #58731 and #70626 are still open and unmerged; #70626 is a draft SDK helper with no Slack consumer migration, so it is coordination work rather than a shipped replacement.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to preserve the Slack restart-survival behavior, then either migrate Slack thread participation to the existing async persistent-dedupe/keyed-store direction or coordinate with #70626 if that becomes the approved SDK contract. The final PR should trim unrelated churn and keep focused tests for restart survival, clear semantics, TTL expiry, corrupt/oversized files, bounds, and state-file hardening.
