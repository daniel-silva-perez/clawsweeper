## Summary

Keep this PR open. The Slack restart-survival bug is real on current main, but the branch still introduces a blocking local-state hardening issue and a broken focused test; it also now needs maintainer direction because current main has a bundled-plugin keyed state store intended for this class of persistence.

## What This Changes

The PR adds JSON-backed disk persistence, lazy hydration, clear/restart tests, and an API-barrel comment for the Slack thread participation cache.

## Best Possible Solution

Land one owner-approved Slack restart-survival implementation that keeps the in-memory hot path semantics, persists account/channel/thread participation for the existing 24-hour policy, uses the shared plugin state store where feasible, and keeps restart, TTL, clear, corrupt-state, and cap behavior covered by focused Slack tests.

## Reproduction Assessment

Yes. The code path is high-confidence: record participation in a Slack thread, restart the gateway so the process-global map is empty, then send a non-mention reply in that thread; current main has no persisted state for prepare.ts to classify it as bot_thread_participant.

## Solution Assessment

No. The PR targets the right user-visible bug, but the current patch is not the best merge shape while openKeyedStore exists and while the custom filesystem path still has non-regular-file, temp-cleanup, and isolated-test defects.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Reject non-regular state files before hydration:** `extensions/slack/src/sent-thread-cache.ts:103-110`
  - body: `hydrateFromDisk` calls `statSync` and then `readFileSync` on the configured state path without rejecting symlinks, FIFOs, devices, or other non-regular files. A tampered local state path can block the first Slack event or read unintended local content before the cache falls back, so use `lstat`/`isFile` or move this persistence onto the keyed state store.
  - confidence: 0.87
- **[P3] Clean up temp files when rename fails:** `extensions/slack/src/sent-thread-cache.ts:217`
  - body: `persistToDisk` writes a UUID-named temp JSON file and then renames it, but a thrown `renameSync` leaves the temp file behind while the scheduled persist path swallows the error. Wrap the rename so failed writes remove `tmpPath` and do not accumulate Slack thread identifiers in leftover temp files.
  - confidence: 0.82
- **[P3] Initialize tempDir before the directory-mode test:** `extensions/slack/src/sent-thread-cache.test.ts:311-312`
  - body: This test builds `path.join(tempDir, ...)` without calling `setup()`. Running it by itself can leave `tempDir` undefined and fail before the directory-mode assertion, so create a temp dir in the test or call the shared setup helper first.
  - confidence: 0.79
