## Summary

Keep open. Current main still keeps Slack thread participation in memory only, so the user-facing restart problem is real, but the latest PR head still has a concrete P2 hardening bug in the new local persistence path before it should merge.

## What This Changes

This PR adds JSON-backed disk persistence, lazy hydration, clear/restart behavior, and unit coverage for the Slack thread participation cache, plus an API-barrel comment to keep test helpers private.

## Best Possible Solution

Land one owner-approved Slack restart-survival implementation using the merged plugin state store, or an explicitly approved hardened JSON path, with account/channel/thread keying, 24h TTL, bounded growth, clear semantics, and restart/corrupt-state coverage; reconcile the overlap with #58731 before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Reject non-regular state files before hydration:** `extensions/slack/src/sent-thread-cache.ts:103-110`
  - body: `hydrateFromDisk` calls `statSync` and then `readFileSync` on the configured state path. Because `statSync` follows symlinks and this code never checks `lstatSync().isFile()`, a tampered state path can block the first Slack event on a FIFO/device or read unexpected local file content before gating continues. Reject non-regular files before the size check/read, or move this persistence onto the keyed state store.
  - confidence: 0.87
- **[P3] Remove temp files when rename fails:** `extensions/slack/src/sent-thread-cache.ts:217`
  - body: `persistToDisk` writes `${filePath}.${randomUUID()}.tmp` and then renames it, but a thrown `renameSync` leaves the temp JSON file behind while the scheduled persist path swallows the error. Wrap the rename in cleanup so failed writes do not accumulate Slack thread identifiers in leftover temp files.
  - confidence: 0.82
- **[P3] Initialize tempDir in the directory-mode test:** `extensions/slack/src/sent-thread-cache.test.ts:311-312`
  - body: This test builds `path.join(tempDir, ...)` without calling `setup()` or using a `beforeEach` in this describe block. Running this case by itself can fail before it exercises the directory-mode assertion, so create a temp dir inside the test or call the shared setup helper first.
  - confidence: 0.79

## Security Review

Status: needs_attention

Summary: The diff does not add dependencies or CI execution paths, but it introduces custom local filesystem persistence with concrete local-state hardening gaps.

Concerns:

- **[medium] Hydration follows non-regular state files:** `extensions/slack/src/sent-thread-cache.ts:103`
  - body: The persisted state path is read with `statSync` and `readFileSync` without rejecting symlinks, FIFOs, or device files, which can turn local state-path tampering into a gateway event-loop block or unintended local file read before Slack message handling continues.
  - confidence: 0.87
- **[low] Failed rename leaves state temp files:** `extensions/slack/src/sent-thread-cache.ts:217`
  - body: If the final rename fails, the temp JSON file is not removed and the scheduled write swallows the error, allowing repeated failures to leave Slack thread identifiers in leftover temp files.
  - confidence: 0.82
