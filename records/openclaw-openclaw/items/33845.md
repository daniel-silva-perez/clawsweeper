## Summary

Current main still keeps Slack thread participation only in process memory, so the restart-survival bug this PR targets is not implemented on main. The PR is a useful active implementation candidate, but it should stay open for maintainer storage/security direction and coordination with overlapping Slack and SDK persistence work.

## Best Possible Solution

Keep this PR open for maintainer review. Preserve the Slack restart-survival behavior, but either migrate it to the accepted shared persistent-dedupe/persistent-keyed-store contract if maintainers choose that direction, or harden the local implementation with regular-file and directory ownership checks, TOCTOU-safe bounded reads, temp cleanup, serialization/locking semantics, and clear logging. Drop unrelated formatting-only changes, keep the focused restart/TTL/corrupt/clear/max-entry regression coverage, and coordinate the final Slack implementation with #58731 and #70626.
