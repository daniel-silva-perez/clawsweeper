## Summary

Keep open. Current main still has thinking production controls, reasoning visibility controls, display/history sanitizers, provider replay sanitizers, and a plugin hook workaround surface, but it does not expose a schema-backed mode that lets reasoning run while discarding thinking blocks from the durable session transcript and transcript APIs.

## What This Changes

Request to add a silent reasoning mode so model thinking can run without persisting thinking or redacted-thinking content into session transcripts and downstream history consumers.

## Best Possible Solution

Define a core transcript-persistence policy separate from model thinking effort and user/channel reasoning visibility. The end state should document and implement where thinking blocks are discarded from user/history/memory transcript surfaces, where provider-required replay state is preserved, and how sessions.get, ACP loadSession, compaction, context pruning, UI/config, and plugin hooks observe the policy.
