## Summary

Keep open. Current main still has thinking effort controls, reasoning visibility controls, display/history sanitizers, provider replay cleanup, and a plugin hook workaround, but no schema-backed policy that lets reasoning run while discarding thinking blocks from durable session transcripts and transcript APIs.

## What This Changes

Request a hidden/silent reasoning mode that preserves model reasoning production while omitting thinking blocks from durable session transcripts and transcript consumers.

## Best Possible Solution

Define a core transcript-persistence policy separate from model thinking effort and channel/user reasoning visibility. The end state should discard thinking blocks from user/history/memory transcript surfaces, preserve provider-required replay state where needed, and document how sessions.get, ACP loadSession, compaction, context pruning, plugin hooks, UI controls, and config observe that policy.
