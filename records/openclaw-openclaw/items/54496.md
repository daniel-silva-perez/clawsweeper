## Summary

Keep open. Current main separates thinking effort from reasoning visibility and has provider replay sanitizers, but it still lacks a core, schema-backed transcript-persistence mode that lets reasoning run while excluding thinking blocks from durable session transcripts and transcript consumers.

## What This Changes

Request a silent reasoning mode that produces model thinking while preventing `type: "thinking"` blocks from being written to session transcripts or downstream memory/history surfaces.

## Best Possible Solution

Define and document a core transcript-persistence policy separate from model thinking effort and user/channel reasoning visibility, with durable transcript/history/memory surfaces dropping thinking blocks while provider replay retains any required internal state through an explicit, tested path.
