## Summary

Current main still does not implement silent reasoning persistence. OpenClaw separates thinking production from reasoning visibility, and has provider replay sanitizers plus a plugin write hook, but there is no schema-backed mode that lets reasoning run while excluding `thinking`/`redacted_thinking` blocks from durable session transcripts and transcript APIs.

## Best Possible Solution

Keep this issue open. The best product direction is a core transcript-persistence policy separate from `thinkingLevel` model reasoning production and `reasoningLevel` user/channel visibility. Implement a documented persist/discard setting for `thinking` and `redacted_thinking` blocks, define where provider-required replay signatures or opaque reasoning state live, and cover transcript writes/reads, `sessions.get`, ACP `loadSession`, compaction/replay sanitization, memory/session-history indexing, UI/config docs, and plugin hook expectations.

## Risks / Open Questions

- A naive strip-before-write implementation could break providers that require signed or opaque reasoning state for replay, so the fix needs a clear split between provider replay state and user/history/memory transcript surfaces.
- The existing hook can support local plugin workarounds, but relying on it would leave core APIs and clients with inconsistent transcript semantics.
