## Summary

Keep open. Current main does not implement full FAL catalog discovery: the bundled FAL image and video providers expose static model arrays, and the public media provider contracts model selectable choices as `models?: string[]`. The request is valid but broad and likely needs maintainer product/API direction before implementation.

## What This Changes

The issue requests dynamic discovery and exposure of the full FAL.ai image/video model catalog instead of the current small hardcoded FAL model set.

## Best Possible Solution

Define a scoped FAL catalog strategy before coding: keep core generic, decide whether media providers need an async/discovered catalog API, and implement FAL coverage either as a maintained external ClawHub plugin or a curated bundled provider expansion with model-specific capability tests and docs.

## Reproduction Assessment

Not applicable as a bug report. We do have a high-confidence current-state check: source inspection shows the FAL image and video providers register static model arrays rather than dynamically discovering the full catalog.

## Solution Assessment

Unclear. Exposing all FAL endpoints as built-in selectable models may not be the narrowest maintainable solution because the catalog spans many modalities and schema shapes; a scoped plugin strategy or additive media catalog API should be decided first.
