## Summary

Keep open. Current main still exposes FAL media models through static provider arrays, and the inspected media-provider list paths serialize those arrays directly; full FAL catalog discovery remains unimplemented and needs maintainer product/API direction before a narrow fix PR is safe.

## What This Changes

The issue requests dynamically discovering and exposing the full FAL.ai model catalog as selectable OpenClaw image/video generation provider models.

## Best Possible Solution

Define a scoped FAL catalog strategy before implementation: keep core generic, decide whether media providers need an async/discovered catalog API, and only expose FAL endpoints after mapping them to supported OpenClaw modalities, capabilities, and request schemas.

## Reproduction Assessment

Not applicable as a bug report. We do have a high-confidence current-state check: source inspection shows FAL image/video providers register static model arrays, and the list actions serialize those arrays rather than querying a catalog.

## Solution Assessment

Unclear. Exposing every FAL endpoint inside the bundled provider may not be the narrowest maintainable solution; a schema-aware plugin-owned catalog or an additive media catalog API should be decided before coding.
