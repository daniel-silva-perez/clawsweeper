## Summary

Keep open. Current main still exposes the bundled FAL image and video providers through curated static model lists, while the request asks for dynamic full-catalog discovery across many FAL modalities; that remaining work is real but broad enough to need maintainer product/API direction before implementation.

## What This Changes

The issue requests expanding the bundled FAL provider from curated static image/video model lists to dynamic discovery and selection for the full FAL.ai model catalog.

## Best Possible Solution

Define a scoped FAL catalog strategy first: keep core generic, decide whether media providers need an additive discovered-catalog API, and implement only capability-mapped FAL coverage in the provider/plugin layer with tests and user-facing docs.

## Reproduction Assessment

Not applicable as a bug reproduction. We do have a high-confidence current-state check from source and docs: FAL image/video support is statically registered through curated model arrays, and the shared list action reports those arrays directly.

## Solution Assessment

Unclear. Exposing every FAL endpoint as selectable built-in models is not yet proven to be the narrowest maintainable solution; a filtered provider-owned catalog or a generic media catalog API may be safer than a raw 1000+ model dump.
