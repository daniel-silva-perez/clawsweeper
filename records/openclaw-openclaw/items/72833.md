## Summary

Keep open. Current main still exposes FAL media generation through static provider-owned model arrays, with only curated image/video coverage; the requested dynamic full-catalog discovery across FAL modalities remains real product/API work and is too broad for conservative cleanup closure.

## What This Changes

The issue requests dynamic discovery and exposure of the full FAL.ai model catalog instead of the currently curated FAL media model lists.

## Best Possible Solution

Define a scoped FAL catalog strategy first: keep core contracts generic, decide whether media providers need an additive discovered-catalog API, then implement a provider-owned FAL catalog with per-model capability/schema mapping, focused docs, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction. We do have a high-confidence current-state check: source shows static FAL image/video model arrays, and the shared list action returns those arrays directly.

## Solution Assessment

Unclear. Exposing every FAL endpoint as a selectable built-in media model is not yet proven to be the most maintainable solution; a filtered provider-owned catalog or a generic discovered media catalog seam may be safer.
