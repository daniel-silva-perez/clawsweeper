## Summary

Keep open. Current main still exposes FAL image/video generation through static, curated model lists, while the request asks for dynamic full-catalog discovery; the remaining work is valid but broad and likely needs maintainer product/API direction before implementation.

## What This Changes

Add dynamic FAL.ai catalog discovery so the FAL image/video generation provider can expose the vendor's broader model catalog as selectable models instead of the current static curated lists.

## Best Possible Solution

Define a scoped FAL catalog strategy first: keep core generic, decide whether media providers need an additive async/discovered catalog API, and implement FAL coverage in the provider/plugin layer with per-model capability mapping, tests, and user-facing docs.

## Reproduction Assessment

Not applicable as a bug report. We do have a high-confidence current-state check: source inspection shows static FAL image/video model arrays and static `models?: string[]` media provider contracts rather than dynamic full-catalog discovery.

## Solution Assessment

Unclear. Exposing every FAL endpoint through the bundled provider is not obviously the narrowest maintainable solution because the catalog spans multiple modalities and request schemas; a plugin-owned catalog strategy or generic media catalog API should be decided first.
