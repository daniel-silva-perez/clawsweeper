## Summary

Keep open. Current main still exposes FAL media generation through curated static model arrays, while this request asks for dynamic full-catalog discovery across mixed FAL modalities; that is real remaining work and needs maintainer product/API direction before implementation.

## What This Changes

Request to expand the bundled `fal` media provider from curated hardcoded image/video model lists to dynamically discover and expose the full FAL.ai catalog as selectable generation models.

## Best Possible Solution

Define a scoped FAL media catalog strategy first: keep core capability contracts generic, decide whether media providers need an additive discovered-catalog API, then implement capability-mapped FAL coverage in the provider/plugin layer with focused docs and tests.

## Reproduction Assessment

Not applicable as a bug reproduction. We do have a high-confidence current-state check: source and docs show static curated FAL image/video model registration, and the shared list action returns those registered arrays directly.

## Solution Assessment

Unclear. Exposing every FAL endpoint as selectable built-in media models is not yet proven to be the most maintainable path; a filtered provider-owned catalog or a generic discovered media catalog seam may be safer.
