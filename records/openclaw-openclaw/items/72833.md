## Summary

Keep open. Current main still exposes FAL image and video generation through static, curated provider model arrays, while the issue asks for dynamic full-catalog discovery across mixed FAL modalities; that remains real product/API work rather than cleanup-close material.

## What This Changes

The issue requests dynamic discovery and exposure of the full FAL.ai model catalog as selectable media-generation provider models instead of the current curated FAL model lists.

## Best Possible Solution

Define a scoped FAL media catalog strategy first, keep vendor behavior in the FAL plugin, add an additive generic discovered-catalog seam only if the current media provider contracts are insufficient, then implement capability-mapped FAL coverage with docs and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction. There is a high-confidence current-state check: inspect or call the media list surfaces on current main and they return the static `models` arrays registered by the FAL image/video providers.

## Solution Assessment

Unclear. Exposing every FAL endpoint as a selectable built-in media model is not yet proven to be the narrowest maintainable solution; a filtered provider-owned catalog with capability/schema mapping, or a small generic discovery seam, is safer.
