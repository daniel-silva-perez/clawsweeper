## Summary

Keep open. Current main still lacks a CLI-wide root-execution guard or an equivalent DigitalOcean 1-Click mitigation, and the linked open PR #67509 is the direct implementation candidate, so the issue should remain open until that PR or an equivalent maintainer-approved fix lands.

## What This Changes

Prevent DigitalOcean 1-Click users from invoking `openclaw` directly as root and corrupting state, service ownership, and gateway startup.

## Best Possible Solution

Review and land PR #67509, or replace it with an equivalent early CLI guard before config/state resolution, with an explicit override, root help/version coverage, and targeted tests; add DigitalOcean recovery or wrapper guidance if maintainers want image-specific mitigation.
