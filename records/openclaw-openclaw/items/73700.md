## Summary

Keep open. Current main at 7e41913 still has the deleted-cwd bootstrap crash path, and this PR remains the active implementation candidate for linked bug #73676. The proposed change covers `shouldLoadCliDotEnv`, but the provided diff still leaves `loadCliDotEnv` with a direct `process.cwd()` call that can throw when a state-dir `.env` causes dotenv loading to proceed.

## Best Possible Solution

Keep this PR open and amend the narrow CLI dotenv fix so both `shouldLoadCliDotEnv` and `loadCliDotEnv` tolerate a deleted current working directory. Add regression coverage for deleted cwd with no state-dir `.env` and with a state-dir `.env`, then land the PR so #73676 closes through the normal GitHub flow.
