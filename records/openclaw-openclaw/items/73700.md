## Summary

Keep open. Current main still has the deleted-cwd bootstrap crash path: `shouldLoadCliDotEnv` calls `process.cwd()` directly before command dispatch, and there is no current-main regression coverage or changelog entry for the `uv_cwd` case. This PR remains a focused implementation candidate for open bug #73676, but it should be amended because `loadCliDotEnv` also calls `process.cwd()` when a state-dir `.env` causes dotenv loading to proceed.

## Best Possible Solution

Keep this PR open and amend the narrow CLI dotenv fix so both `shouldLoadCliDotEnv` and the workspace `.env` load in `loadCliDotEnv` tolerate a deleted current working directory. Add regression coverage for deleted cwd both with no state-dir `.env` and with a state-dir `.env`, then land the PR so #73676 closes through the normal GitHub flow.
