## Summary

Keep this PR open. Current main still has the deleted-cwd CLI bootstrap crash, and the PR is a focused implementation candidate for open bug #73676, but the proposed patch only guards `shouldLoadCliDotEnv`; `loadCliDotEnv` still calls `process.cwd()` unguarded when a state-dir `.env` makes dotenv loading proceed.

## Best Possible Solution

Amend this PR so both `shouldLoadCliDotEnv` and the workspace `.env` load inside `loadCliDotEnv` tolerate `process.cwd()` throwing. Add regression coverage for deleted cwd with no state-dir `.env` and with a state-dir `.env`, then land the PR so #73676 closes through the normal GitHub flow.
