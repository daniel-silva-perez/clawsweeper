## Summary

Keep this PR open. Current main still has the deleted-cwd bootstrap crash path, and this PR remains a focused implementation candidate for open bug #73676. It likely needs a small amendment because the proposed guard only covers `shouldLoadCliDotEnv`; if the state-dir `.env` exists, `loadCliDotEnv` still calls `process.cwd()` directly and can hit the same `uv_cwd` throw.

## Best Possible Solution

Keep this PR open and amend it narrowly: guard both the cwd-local `.env` check in `shouldLoadCliDotEnv` and the workspace `.env` load in `loadCliDotEnv`, then add regression coverage for deleted cwd with and without a state-dir `.env`. After that lands, GitHub can close #73676 as implemented.
