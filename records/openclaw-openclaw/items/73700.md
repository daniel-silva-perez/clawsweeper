## Summary

Keep this PR open. Current main still has the reported bootstrap crash path: `shouldLoadCliDotEnv` calls `process.cwd()` directly before CLI command dispatch, and there is no current-main regression coverage for the deleted-cwd/`uv_cwd` case. The PR is a focused implementation candidate for open issue #73676, not obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The likely product path is to land this narrow bootstrap guard with the regression test and changelog entry, or request small revisions if maintainers prefer a shared safe-cwd helper. After merge, GitHub can close the linked bug as implemented.
