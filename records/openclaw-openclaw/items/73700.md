## Summary

Keep open. Current main still has the deleted-cwd bootstrap crash path: `shouldLoadCliDotEnv` calls `process.cwd()` directly before command dispatch, and there is no current-main regression coverage or changelog entry for the `uv_cwd` case. This PR is a focused implementation candidate for open bug #73676, but it likely needs a small follow-up because `loadCliDotEnv` also calls `process.cwd()` when a state-dir `.env` causes dotenv loading to proceed.

## Best Possible Solution

Keep this PR open. The best path is to review and land an amended narrow fix for the CLI bootstrap dotenv path, likely guarding both `shouldLoadCliDotEnv` and the workspace `.env` load inside `loadCliDotEnv`, with the regression test covering deleted cwd and state-dir `.env` behavior. After merge, GitHub can close #73676 as implemented.
