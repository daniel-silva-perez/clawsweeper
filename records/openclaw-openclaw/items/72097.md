## Summary

Keep open. Current main still builds plugin-provided tools as a run/session snapshot and plugin install paths refresh the registry plus prompt a Gateway restart, without a session-level toolset delta or turn-boundary re-resolution path; the related restart-event PR is adjacent but does not satisfy this requested active-session tool refresh.

## What This Changes

Refresh active agent session tool definitions after successful tool-providing plugin installs so newly installed plugin tools can be used without restarting the Gateway or session.

## Best Possible Solution

Add a generic runtime lifecycle path that re-resolves plugin-provided tools after a successful plugin install or registry refresh, applies the same policy filters used at normal run setup, updates active sessions only at a safe turn boundary, and invalidates any affected prompt/tool-cache state without duplicating plugin runtime loads.

## Reproduction Assessment

Yes. The issue gives a concrete install -> immediate use -> Gateway restart comparison, and current-main source inspection confirms plugin installs refresh the registry while active runs build and apply a fixed tool snapshot during session setup.

## Solution Assessment

Unclear as a patch because this issue has no implementation. The requested direction is sound, but the maintainable solution should likely be turn-boundary re-resolution/update through existing policy filters rather than arbitrary mid-stream mutation of active session state.
