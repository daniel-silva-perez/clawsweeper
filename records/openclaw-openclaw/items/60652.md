## Summary

Keep this issue open. Current main still does not advertise `upload-file` for Discord message actions, even though the Discord send path accepts file inputs through the existing `send` action and the shared action registry/schema can represent `upload-file`; the provided GitHub context also shows an open closing PR, so the issue should remain open until maintainers land or reject the canonical Discord fix.

## What This Changes

The issue asks for Discord message-tool discovery to expose a file-send action so agents can discover and use Discord attachment delivery instead of relying on hidden `send` parameters.

## Best Possible Solution

Keep the issue open and review one canonical Discord plugin fix. The accepted change should add `upload-file` to Discord's advertised message actions under the chosen gate, route or document the action through the existing `sendMessage` media path, and update Discord-specific contract/unit tests so the tool schema and dispatcher stay aligned.
