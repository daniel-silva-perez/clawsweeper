## Summary

Keep this issue open. Current main still does not advertise `upload-file` for Discord message actions: `describeDiscordMessageTool` only adds `read`, `edit`, and `delete` under the `messages` feature gate, while the Discord handler only accepts file parameters through the existing `send` action. The discussion also points to open implementation PRs with closing syntax, especially #61087 and #60808, so this should remain open until maintainers review and land or close the canonical fix.

## Best Possible Solution

Keep the issue open and review one canonical Discord fix, likely #61087 or #60808. The accepted patch should add `upload-file` to Discord's advertised message actions under the chosen feature gate, add or confirm an explicit dispatcher path for `upload-file`, and update Discord-specific contract/unit tests so agents can discover and use file-send capability.
