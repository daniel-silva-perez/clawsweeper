## Summary

Current main still does not implement the requested Memory MVP SQLite record model. The default memory-core path remains a Markdown-backed retrieval index with SQLite chunks/FTS/vector support, while memory-wiki and memory-lancedb are adjacent plugin layers rather than the durable typed SQLite source of truth described in this issue.

## Best Possible Solution

Keep this open for maintainer-owned design and implementation in memory-core plus memory-host-sdk: versioned SQLite migrations, exported record/revision/entity/status/confidence/validity types, tests for canonical current rows and non-destructive memory_revisions, write-path integration, and docs explaining how Markdown memory, public artifacts, memory-wiki, memory-lancedb, and optional memory plugins interact with the durable source-of-truth model. Coordinate the schema with #42647, #42648, and #42650 before implementation.
