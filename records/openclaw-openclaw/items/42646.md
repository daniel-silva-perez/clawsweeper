## Summary

Keep open. Current main still implements the default SQLite memory database as a Markdown/session retrieval index, not the requested typed durable SQLite source of truth with canonical memory records, revision history, normalized entities, lifecycle statuses, confidence, and validity windows.

## Best Possible Solution

Keep this open for maintainer-owned Memory MVP design and implementation. The likely path is a versioned SQLite schema and migration in memory-core or memory-host-sdk, exported record/revision/entity/status/confidence/validity types, write-path integration, tests proving one canonical current record plus non-destructive revisions, and docs explaining how Markdown memory, public artifacts, memory-wiki, memory-lancedb, and optional memory plugins interact.
