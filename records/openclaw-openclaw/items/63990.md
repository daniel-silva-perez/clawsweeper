## Summary

Keep this issue open. Current main still supports one resolved memory embedding provider/model/store per agent and a fallback path that switches provider then rebuilds the single index. It does not implement the requested first-class `indexes[]`/`activeIndex`, independent per-index ingestion status, controlled read failover between already-built indexes, or per-index status/metrics.

## Best Possible Solution

Keep this issue open for maintainer design. The likely path is an additive memory-core and public memory/plugin contract change: add backward-compatible `indexes[]`/`activeIndex` config, preserve a canonical chunk source, build per-index ingestion/backfill state, route reads to exactly one selected ready index with controlled fallback, expose per-index status/metrics, and keep existing single-index configs plus full-reindex fallback behavior working during migration.
