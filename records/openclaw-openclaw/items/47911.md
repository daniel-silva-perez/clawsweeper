## Summary

Keep open. Current main still has separate observability commands, but no `openclaw fleet status` command or equivalent single JSON surface that combines all configured agents, provider/auth health, channel health, session/task liveness, service state, and recent errors. The related #50066 and #50072 observability/signal items are closed and did not land a replacement for this request.

## Best Possible Solution

Keep this open for maintainer product/design review. The best path is to define one supported read-only observability surface, likely `openclaw fleet status --json` or an expanded `openclaw status --all --json`, that composes existing agent, session, task, auth, channel, service, and log summaries with clear redaction rules and focused tests. If maintainers prefer this outside core, first confirm the plugin/CLI APIs expose enough stable, non-secret state for a ClawHub plugin to implement it.
