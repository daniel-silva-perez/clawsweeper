## Summary

Keep this issue open. Current main has useful partial observability in `openclaw status --all`, `openclaw sessions --all-agents`, `openclaw health`, `openclaw channels status --probe`, and `openclaw models status --agent`, but it still does not provide the requested unified fleet command or one JSON payload that combines per-agent model/auth health, session/stuck state, channel health, service state, and recent errors. The related observability PR #50066 was closed unmerged, and the related signal-layer proposal #50072 was also closed, so there is no merged or canonical replacement that supersedes the remaining work.

## Best Possible Solution

Implement a first-class fleet observability surface, either as `openclaw fleet status` or a clearly documented `openclaw status --fleet`/`--all-agents` extension, with `--json`. It should compose existing agent config/session summaries, per-agent model/auth status, channel health, task/stuck-session diagnostics, gateway/service state, and redacted recent-error summaries, with tests for multi-agent output and secret redaction.

## Risks / Open Questions

- A future implementation should redact or summarize auth-profile and log data carefully, because the requested fleet view crosses credential, provider, channel, and session surfaces.
- The existing `openclaw status` fast path is intentionally read-only and lightweight, so a fleet implementation should avoid making normal status slower or more invasive.
