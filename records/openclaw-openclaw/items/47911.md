## Summary

Keep open. Current main still does not provide `openclaw fleet status` or an equivalent single JSON surface combining all configured agents, provider/auth health, channel health, session/task liveness, service state, and recent errors. The existing implementation is spread across `status`, `models status`, `channels status/logs`, `sessions --all-agents`, `tasks audit`, and `logs`, and the related #50066 and #50072 observability proposals are closed without superseding this request.

## Best Possible Solution

Keep this open for maintainer product/design review. The best path is to define one supported read-only observability surface, likely `openclaw fleet status --json` or an expanded `openclaw status --all --json`, that composes existing agent, session, task, auth, channel, service, and log summaries with clear redaction rules. If maintainers prefer this outside core, first confirm the plugin/CLI APIs expose enough stable non-secret state for a ClawHub plugin to implement it.
