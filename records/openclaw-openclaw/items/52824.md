## Summary

Keep open. Current main at 7e5d6dba still does not fully implement this draft PR: the subagent completion bound-route lookup still uses requester threadId before the chat-level `to` target, and Telegram `sendMessage ok` logs still omit `message_thread_id`. Main has absorbed some overlapping behavior for direct stale-entry Telegram topic routing and queued route preservation, so the PR should be rebased/narrowed rather than closed as implemented.

## Best Possible Solution

Keep this draft PR open for maintainer review or author rebase, narrowed to the remaining useful deltas: add/prove a regression for chat-target-first bound-route lookup in `resolveSubagentCompletionOrigin`, and decide whether to add `message_thread_id` to Telegram `sendMessage ok` logs. Avoid re-landing the already-covered direct stale-entry and queued announce fixes unless the rebase shows they are still needed.
