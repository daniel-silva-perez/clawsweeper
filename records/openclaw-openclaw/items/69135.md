## Summary

Current main still matches the reported false-positive path. `gateway probe` builds an SSH tunnel target and keeps the configured remote target unless the URLs are identical, then `buildGatewayStatusWarnings` emits `multiple_gateways` solely from `reachable.length > 1` without comparing the `self`/presence identity already captured on each probed target. No duplicate or linked implementation PR was found in the provided context or targeted GitHub issue searches, so this should stay open and is a good narrow fix-PR candidate.

## Best Possible Solution

Keep this open and fix the warning in the gateway-status owner path. The best implementation is to classify reachable targets by a conservative gateway identity key from `self`/presence when available, suppress or downgrade the `multiple_gateways` warning when all reachable targets share the same non-null identity, and keep the existing warning for distinct identities or unknown/mixed identity cases. Add focused tests for SSH tunnel plus configured remote URL hitting the same gateway and for two genuinely distinct reachable gateways.

## Clownfish Work Prompt

Fix the false-positive `openclaw gateway probe` `multiple_gateways` warning when an SSH tunnel target and configured `gateway.remote.url` reach the same logical gateway. Work in the gateway-status owner path only. Current main builds both targets in `src/commands/gateway-status/probe-run.ts` and `src/commands/gateway-status/helpers.ts`, stores `self` from `pickGatewaySelfPresence`, then `src/commands/gateway-status/output.ts` warns whenever `reachable.length > 1`. Implement a conservative identity-aware warning policy: when all reachable targets have the same non-null gateway self identity, do not emit `multiple_gateways` as though distinct gateways were found; either suppress it or replace it with a clearer informational/warning code if the existing output conventions need a record. Preserve the existing warning for genuinely distinct identities and for unknown/mixed identity cases where suppressing could hide real duplicate gateways. Add focused Vitest coverage, preferably in `src/commands/gateway-status/output.test.ts` and/or `src/commands/gateway-status.test.ts`, for SSH tunnel plus configured remote URL sharing the same `self`, and for two reachable targets with different `self` values still warning.
