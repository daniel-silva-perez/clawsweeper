## Summary

Current main still matches the reported false positive. `gateway probe` builds an SSH tunnel target and keeps the configured remote URL unless the URLs are identical, then `buildGatewayStatusWarnings` emits `multiple_gateways` from reachable target count alone. The probed target already carries `self` gateway presence, but that identity is not used to distinguish one gateway reached through two transports from two distinct gateways.

## Best Possible Solution

Keep this open and fix the gateway-status owner path. Classify reachable targets by a conservative non-null gateway identity from `self`/presence, suppress or downgrade `multiple_gateways` when all reachable targets share that identity, and keep the current warning for distinct identities or unknown/mixed identity cases. Add focused tests for SSH tunnel plus configured remote URL hitting the same gateway and for genuinely distinct reachable gateways.
