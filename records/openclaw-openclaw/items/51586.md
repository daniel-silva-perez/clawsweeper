## Summary

Current main still does not implement the requested configurable reset prompt. Bare `/new` and `/reset` now acknowledge without invoking the model, but reply-producing reset paths still select hardcoded reset prompt strings, and config schema/types/docs expose `startupContext` controls rather than `agents.defaults.sessionResetPrompt` or a per-agent equivalent.

## Best Possible Solution

Keep this issue open for maintainer product judgment. A direct implementation would add a documented, schema-validated `agents.defaults.sessionResetPrompt` plus per-agent override, resolve it before `appendCronStyleCurrentTimeLine`, and cover default/custom/empty prompt behavior with focused tests. If maintainers prefer the newer acknowledgement-only bare reset model, they should explicitly rescope this to reset tails, soft reset, or a hook/template API instead of treating it as already implemented.
