## Summary

Keep open. Current main partially changed the surface by acknowledging bare `/new` and `/reset` without invoking the model, but it does not implement the requested configurable reset prompt. The hardcoded reset prompt still exists, there is no `sessionResetPrompt` key in the default or per-agent config schema/types/docs, and the external KeepMyClaw continuity discussion does not replace a core config surface for reset prompt behavior.

## Best Possible Solution

Keep the issue open for maintainer product judgment. A direct implementation would add a documented, schema-validated `agents.defaults.sessionResetPrompt` plus per-agent override, resolve it in the reset prompt builder before the current-time append, and cover default/custom/empty prompt behavior with focused tests. If maintainers prefer the newer ack-only bare reset model, they should explicitly close or re-scope this as a reset hook/template discussion rather than auto-closing it as already implemented.
