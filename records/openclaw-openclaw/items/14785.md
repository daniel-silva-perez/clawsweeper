## Summary

Keep open. Current main still has the fixed tool-schema and prompt overhead described by the report, and the discussion treats this as the canonical multi-step performance tracker rather than a completed or superseded item.

## What This Changes

The issue requests reducing per-session tool schema token overhead through channel-aware message schema pruning, lazy or slim tool loading, shorter tool descriptions, and less duplicated prompt/tool metadata.

## Best Possible Solution

Keep this as the canonical performance tracker and sequence incremental core changes with measurement: remove or make optional duplicated Tooling text, avoid emitting tool definitions when no effective tools remain, prune high-footprint message fields by active channel and capability, shorten long descriptions where behavior remains covered, and design a compatible slim or lazy schema mode for plugins and declarative tools.
