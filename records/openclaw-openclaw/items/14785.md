## Summary

Keep this issue open. Current main has useful policy workarounds and partial scoping, but the central fixed overhead remains: broad message parameters are still merged into the schema, full schemas are still passed to the provider adapter, the system prompt still contains Tooling text, and the cron tool still embeds long always-sent guidance.

## Best Possible Solution

Keep this issue open as the canonical tool-schema overhead tracker. The best path is incremental core work: remove or make optional duplicated Tooling text, avoid schema payloads when the effective tool list is empty, prune message parameters by active channel/capability without breaking cross-channel jobs, shorten always-sent cron guidance, and design a compatible lazy or slim schema mode for plugin and declarative tools. Narrower items such as #14619, #52861, #58405, and #58434 should stay linked rather than superseding this broader tracker.
