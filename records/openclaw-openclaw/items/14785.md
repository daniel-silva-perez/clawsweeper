## Summary

Keep this open. Current main still has the fixed tool-schema overhead at the center of the report: the message tool schema is still broad, model-facing tool definitions still include full descriptions and parameters, the system prompt still emits a Tooling list, and cron still carries long always-sent guidance. The issue discussion also treats this as the canonical tracker with partial and planned follow-ups rather than a completed or superseded item.

## What This Changes

The issue requests reducing fixed per-session tool schema token overhead through channel-aware pruning, lazy or slim tool schemas, shorter descriptions, less duplicated tool text, and possible mega-tool splitting.

## Best Possible Solution

Keep this as the canonical performance tracker and sequence incremental core changes: remove or make optional duplicated Tooling text, avoid emitting tool definitions when no effective tools remain, prune high-footprint message fields by active channel and capability without breaking cross-channel cron or isolated runs, shorten long descriptions where behavior stays covered, and design a compatible slim or lazy schema mode for plugins and declarative tools.
