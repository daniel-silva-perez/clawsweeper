## What This Changes

## Best Possible Solution

Keep this as the canonical performance tracker and sequence incremental core changes: remove or make optional duplicated Tooling text, prune high-footprint message fields by active channel and capability, shorten long descriptions where docs/tests preserve behavior, avoid unnecessary tool payloads, and design a compatible slim or lazy schema mode for plugins and declarative tools.
