## Summary

Keep this issue open. Current main still has the core overhead described here: broad message schemas, full effective tools converted into model-facing definitions, Tooling text in the system prompt, and a long always-sent cron description. Existing allow/deny/profile controls and related work are mitigations, not a full fix.

## Best Possible Solution

Keep this issue open as the canonical tool-schema overhead tracker. The best path is incremental core work: remove or make optional duplicated Tooling text, avoid schema payloads when the effective tool list is empty, prune message parameters by active channel/capability without breaking cross-channel jobs, shorten always-sent cron guidance, and design a compatible lazy or slim schema mode for plugin and declarative tools. Keep narrower work such as #14619, #52861, #54732, #58405, and #58434 linked rather than treating them as superseding this broader tracker.
