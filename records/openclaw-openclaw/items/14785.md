## Summary

Keep open. Current main still has the central fixed tool-schema and prompt overhead described in the report: the message tool schema remains broad, tool definitions still carry descriptions and parameter schemas into each session, the system prompt still emits a Tooling list, and cron still embeds long guidance. The issue discussion also treats this as the canonical multi-step tracker with partial and planned follow-up work, not as completed or superseded.

## What This Changes

The issue requests reducing fixed model-context overhead from always-injected tool schemas, especially the broad message schema, cron guidance, duplicated tool list text, and non-lazy tool loading.

## Best Possible Solution

Keep this as the canonical performance tracker and sequence incremental core changes: remove or make optional duplicated Tooling text, avoid emitting definitions when no effective tools remain, prune high-footprint message fields by active channel and capability without breaking cross-channel cron or isolated runs, shorten long descriptions where behavior stays covered, and design a compatible slim or lazy schema mode for plugins and declarative tools.
