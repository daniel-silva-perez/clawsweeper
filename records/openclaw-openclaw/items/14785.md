## Summary

Keep this issue open. Current main still carries the central overhead the report describes: the message tool keeps broad cross-channel parameters, full effective tool schemas are passed into model-facing custom tool definitions, the system prompt still includes a Tooling list, and the cron tool still embeds long guidance in its always-sent description. Related work and workarounds reduce parts of the cost but do not close the canonical tracker.

## Best Possible Solution

Keep this issue open as the canonical tool-schema overhead tracker. The best path is incremental core work: remove or make optional duplicated Tooling text, avoid schema payloads when the effective tool list is empty, prune message parameters by active channel/capability while preserving cross-channel isolated/cron behavior, shorten or move cron tutorial text out of the always-sent description, and design a compatible slim/lazy schema mode for plugin and declarative tools. Keep narrower refs such as #14619, #52861, #58405, and #58434 linked rather than treating them as superseding the broader tracker.
