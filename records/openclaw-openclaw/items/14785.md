## Summary

Keep this open. Current main still carries the core fixed overhead described in the report: broad cross-channel message schema fields remain in one message tool schema, full tool descriptions and parameters still become model-facing custom tool definitions, the system prompt still has a Tooling list, and cron still embeds long guidance. The latest maintainer comment also treats this as the canonical multi-step tracker rather than a resolved duplicate.

## What This Changes

The issue requests reducing fixed per-session model context from broad tool schemas through channel-aware message schema pruning, lazy or slim tool loading, shorter tool descriptions, and removal of duplicated tool summaries.

## Best Possible Solution

Keep this as the canonical performance tracker and sequence incremental core changes: remove or make optional duplicated Tooling text, avoid emitting tool definitions when no effective tools remain, prune high-footprint message fields by active channel and capability without breaking cross-channel cron or isolated runs, shorten long descriptions where tests show safe coverage, and design a compatible slim or lazy schema mode for plugins and declarative tools.
