## Summary

Keep open. Current main still has the central fixed tool-schema and prompt overhead described in the report, and the discussion treats this as the canonical multi-step performance tracker rather than a completed or superseded item.

## What This Changes

The issue requests reducing fixed model context overhead from always-sent tool schemas through channel-aware pruning, lazy or slim schemas, shorter descriptions, and related prompt/schema compression.

## Best Possible Solution

Keep this as the canonical performance tracker and sequence incremental core changes: remove or make optional duplicated Tooling text, avoid emitting tool definitions when no effective tools remain, prune high-footprint message fields by active channel and capability without breaking cross-channel cron or isolated runs, shorten long descriptions where behavior stays covered, and design a compatible slim or lazy schema mode for plugins and declarative tools.
