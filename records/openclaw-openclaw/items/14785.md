## Summary

Keep this issue open. Current main has partial mitigations and configuration workarounds, but it still sends broad tool schemas for the effective tool set: the `message` tool still includes cross-channel parameters in scoped sessions, the adapter still forwards full parameter schemas, and the system prompt still has a Tooling/tool-list section. The discussion also explicitly treats #30359 as partial and includes a recent maintainer Codex review keeping this as the canonical tracker.

## Best Possible Solution

Keep this issue open as the canonical tool-schema overhead tracker. The best path is incremental core work: remove or make optional duplicated tool-list text, avoid schema payloads when the effective tool list is empty, prune `message` parameters by active channel/capability without breaking cross-channel jobs, keep long tool guidance out of always-sent schemas, and design a compatible lazy or slim schema mode for plugin and declarative tools. Narrower items such as #14619 and #52861 should stay linked as follow-ups rather than superseding this broader tracker.
