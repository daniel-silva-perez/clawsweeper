## Summary

Keep open. Current main still has the core fixed overhead paths described by the report: the message tool schema remains broad, full tool descriptions and parameter schemas still flow into model-facing custom tools, the system prompt still emits a Tooling section, and cron still carries long model-visible guidance. The discussion also treats this as the canonical multi-step performance tracker, with only partial/workaround-level progress so far.

## What This Changes

The issue requests reducing fixed per-session tool schema token overhead through channel-aware pruning, lazy or stubbed tool loading, shorter descriptions, moving cron tutorial text out of schemas, and possible mega-tool splitting.

## Best Possible Solution

Keep this as the canonical performance tracker and land incremental, measured core changes: remove or make optional duplicated Tooling text, avoid emitting tool definitions when no effective tools remain, prune high-footprint message fields by active channel and capability, shorten long descriptions where behavior remains covered, and design a compatible slim or lazy schema mode for plugins and declarative tools.
