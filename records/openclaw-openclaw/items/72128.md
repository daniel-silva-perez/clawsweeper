## Summary

Keep this PR open. Current main still lacks configurable `sessions_spawn` label templates, and the PR is relevant but not merge-ready because it generates fallback labels before native/ACP target resolution and omits generated config-schema, docs, and focused test follow-through.

## What This Changes

The PR adds optional `labelTemplate` fields under `agents.defaults.subagents` and `agents.list[].subagents`, then uses the template to generate a spawned-session label when `sessions_spawn.label` is omitted.

## Best Possible Solution

Complete the feature in the spawn layer that knows both requester and resolved target, preserve explicit-label precedence, define and document per-agent override semantics, keep native and ACP registry labels consistent, and update generated schema artifacts, docs, changelog, and focused regression tests.

## Reproduction Assessment

Yes. On current main, a config containing `agents.defaults.subagents.labelTemplate` is not accepted by the strict schemas and the tool path reads only an explicit `label`; on the PR branch, code inspection shows omitted-`agentId` and ACP-default-target cases expand `{agentId}` from raw input before target resolution.

## Solution Assessment

No. The PR is a useful starting point, but the maintainable fix should generate fallback labels after target resolution and keep config schema/docs/generated artifacts in sync rather than doing raw-input templating in the tool wrapper.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Use the resolved target before expanding labels:** `src/agents/tools/sessions-spawn-tool.ts:219-231`
  - body: The fallback label is expanded in the tool wrapper from `requestedAgentId`, before native spawn falls back to the requester and ACP falls back to `acp.defaultAgent`. Calls that omit `agentId` can therefore produce labels with an empty `{agentId}` and miss target-specific template behavior even though a concrete target is spawned. Move template expansion to a point that has the resolved requester/target ids.
  - confidence: 0.91
- **[P2] Regenerate the config schema artifacts:** `src/config/zod-schema.agent-defaults.ts:282`
  - body: Adding `labelTemplate` to the strict Zod schemas without updating `src/config/schema.base.generated.ts` leaves generated config-schema consumers unaware of the new key and should make the config schema check stale. Regenerate the base config schema for both default and per-agent subagent entries.
  - confidence: 0.84
