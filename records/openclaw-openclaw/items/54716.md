## Summary

Keep this PR open, but do not merge it as-is. Current main still has the reported literal per-agent session-store bug, and the PR is a useful narrow fix candidate, but the branch is stale against the current split session-store modules and still has two correctness issues in the proposed diff.

## What This Changes

The PR changes gateway session-store aggregation so literal `agents/<id>/sessions/sessions.json` paths discover sibling per-agent stores scoped to that agents root, with regression tests for the literal-path case.

## Best Possible Solution

Adapt the fix to current main in `src/config/sessions/combined-store-gateway.ts` and, if needed, `src/config/sessions/targets.ts`: literal paths matching `agents/<id>/sessions/sessions.json` should discover sibling stores under that same agents root, return the resolved literal store path for downstream file lookup, derive literal-layout agent ids from disk paths, and preserve single-store behavior for non-agent-layout literal paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Return the resolved literal store path:** `src/gateway/session-utils.ts:988`
  - body: Literal agent-layout stores now skip the single-store branch, but the multi-store return still uses the raw `storeConfig` string. For `~` or relative paths, downstream usage/transcript lookup can resolve from the wrong directory and miss files for sessions that `sessions.list` returned. Carry `resolvedLiteralStorePath` through this branch.
  - confidence: 0.84
- **[P2] Derive literal target agents from disk paths:** `src/gateway/session-utils.ts:1034-1036`
  - body: For a literal `.../agents/main/sessions/sessions.json`, target dedupe can keep a configured target for the same store path before the discovered `main` target. If another agent appears first in `agents.list`, this loop canonicalizes legacy keys from the main store under that other agent id. Use the agent id encoded in the store path or discovered directory for literal-layout targets.
  - confidence: 0.86
