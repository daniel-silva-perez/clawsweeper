## Summary

Keep this PR open. Current main still has the literal non-template `session.store` single-store fallback, so the user-visible bug from #54435 is not implemented on main. The PR remains a plausible fix candidate, but it needs to be rebased/ported to the current `src/config/sessions/combined-store-gateway.ts` implementation and should address the remaining review concern about deriving the agent id from the literal store path before canonicalization.

## Best Possible Solution

Keep this PR open and have it rebased onto current main by porting the literal agent-layout handling into `src/config/sessions/combined-store-gateway.ts`. The fix should treat literal `agents/<id>/sessions/sessions.json` paths as a scoped agent root, return the resolved literal path where downstream callers need it, derive the target agent id from the store path for canonicalization, and keep the two regression tests for sibling discovery and no default-state leakage.
