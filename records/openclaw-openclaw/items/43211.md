## Summary

Keep this PR open. Current main still builds and checks model allowlists as exact normalized provider/model keys, so the intended bug fix is not implemented on main. The PR is also not ready to close as obsolete or merge as-is because unconditional hyphen stripping would broaden authorization to a potentially distinct custom provider ID.

## Best Possible Solution

Keep this PR open for maintainer or contributor rework. The best path is to add a concrete reproduction for the runtime path that strips provider hyphens, then either fix that resolver to preserve the configured provider ID or represent the equivalence through explicit provider/plugin alias metadata. Regression coverage should prove declared aliases work while distinct providers such as `my-provider` and `myprovider` remain separately allowlisted unless explicitly linked.
