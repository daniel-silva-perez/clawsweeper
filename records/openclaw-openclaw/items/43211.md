## Summary

Current main still builds and enforces model allowlists as exact normalized provider/model keys, so this PR is not obsolete. The proposed patch remains a plausible fix candidate, but it needs rework because unconditional hyphen stripping can authorize a distinct custom provider ID.

## Best Possible Solution

Keep this PR open for maintainer or contributor rework. The best path is to add a concrete reproduction for the runtime path that strips provider hyphens, then either fix that resolver directly or model the equivalence through explicit provider/plugin alias metadata. Regression coverage should prove declared aliases work while distinct providers such as `my-provider` and `myprovider` remain separately allowlisted unless explicitly linked.

## Risks / Open Questions

- Closing this PR would drop a focused fix candidate while current main still has the exact-key allowlist behavior described in the PR.
