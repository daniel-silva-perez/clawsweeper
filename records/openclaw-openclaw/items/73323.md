## Summary

Keep this issue open. Current main has useful adjacent work for pricing opt-out, Telegram transport cleanup, readiness/event-loop diagnostics, and diagnostics export, but I did not find a source-backed fix for the reported chronic cross-subsystem Gateway degradation. The related pricing-startup PR #73486 is still open and explicitly narrower than the long-running Telegram/RPC degradation described here.

## Best Possible Solution

Keep this open for Gateway runtime triage. Review, land, or close #73486 as the pricing-startup subset, then run a long-lived Windows + Node 24/25 Gateway with Telegram enabled on current main or the next release. Compare models.pricing.enabled true versus false, collect diagnostics.stability, readiness event-loop health, ELU/event-loop delay, handle/socket counts, Telegram transport state, undici dispatcher state, and models.list/doctor.memory.status timings during stalls. Close only after the chronic Telegram/RPC degradation is proven fixed, or split the remaining non-pricing behavior into a narrower canonical follow-up.
