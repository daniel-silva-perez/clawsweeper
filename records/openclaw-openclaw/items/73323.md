## Summary

Keep open. Current main has useful mitigations and diagnostics, but it does not prove the chronic cross-subsystem Gateway degradation is fixed: pricing still has the 60s fetch path, pricing still starts from runtime services when enabled, Telegram recovery remains reactive, and the channel stop-timeout zombie behavior is still pinned by current tests.

## What This Changes

The issue asks maintainers to investigate and fix chronic Gateway runtime degradation affecting pricing fetches, Telegram polling/sendMessage, and Gateway RPC latency on Windows/Node global npm installs.

## Best Possible Solution

Keep this as the canonical broad Gateway runtime degradation tracker. Maintainers should land or replace the narrow pricing-startup work, fix the channel stop-timeout lifecycle bug, then run a long-lived Windows Gateway with Telegram enabled and pricing toggled on/off before splitting any remaining root cause into narrower follow-ups.
