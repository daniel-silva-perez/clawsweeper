## Summary

Keep this issue open. Current main has several adjacent mitigations for pricing startup, model prewarm, Telegram transport cleanup, plugin metadata churn, and stale session lanes, but I did not find a source-backed fix that covers the reported cross-subsystem long-running gateway degradation: pricing fetch 60s timeouts, Telegram getUpdates/sendMessage stalls, and slow normal Gateway RPCs. The issue is fresh and has multiple corroborating comments, including one Linux report and several Windows reproductions.

## Best Possible Solution

Keep this open for gateway runtime triage. Validate current main or the next release on a long-running Windows gateway with Telegram enabled, compare models.pricing.enabled true versus false, and collect startup trace, event-loop delay, handle/socket counts, Undici/global dispatcher state, Telegram transport state, and RPC timings during stalls. Treat #73486/#53639 as the pricing-startup subset; close or split this only after the chronic Telegram/RPC degradation is proven fixed or narrowed to a new canonical issue.
