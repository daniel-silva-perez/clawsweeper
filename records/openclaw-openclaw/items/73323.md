## Summary

Keep this issue open. It is a fresh, detailed bug report on a supported Windows + Node 24 setup, and current main does not have a source-backed fix for the cross-subsystem degradation described. Current main still uses 60s OpenRouter/LiteLLM pricing fetch timeouts, Telegram still relies on a 120s polling-stall watchdog/rebuild mitigation, and the reported slow Gateway RPCs make this broader than the related pricing-timeout PR.

## Best Possible Solution

Keep the issue open for Gateway/Telegram maintainer triage. The best path is to reproduce or instrument a long-running Windows 11 + Node 24 gateway with Telegram enabled, collect event-loop delay, handle/socket counts, undici dispatcher state, and inspector profiles during stalls, then fix the shared runtime/network degradation or add targeted isolation, timeouts, and diagnostics. Treat #72027 as adjacent pricing fail-fast work, not a canonical fix for this broader report.
