Keep open. Current main has useful mitigations, but the broad Gateway runtime degradation is still not proven fixed: pricing refresh remains in the initial runtime-service path when enabled, remote pricing still has 60s fetch timeouts, and the channel stop-timeout path still leaves stale task state pinned by current tests.

## What This Changes

The issue requests investigation and repair for chronic Gateway degradation where pricing fetches time out, Telegram polling/send operations stall or fail, and Gateway RPCs slow dramatically on Windows 11 with Node 24/25.

## Best Possible Solution

Keep this as the canonical broad Gateway runtime-degradation tracker until maintainers validate current main or the next release on a long-lived Windows Gateway with Telegram enabled. Land or replace the narrow pricing-startup and stopChannel timeout fixes, then split any remaining process-wide RPC/event-loop or Telegram transport root cause into smaller follow-up issues once isolated.

Yes. The issue provides a concrete Windows 11 + Node 24/25 + global npm + Telegram reproduction path, and multiple commenters corroborate Gateway-internal stalls while standalone fetch/curl to the same endpoints stays fast.
