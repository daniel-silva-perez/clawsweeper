## Summary

Keep this issue open. Current main contains adjacent startup mitigations, a working `models.pricing.enabled` config toggle, and older Telegram transport leak hardening, but I did not find a source-backed fix for the broader cross-subsystem gateway degradation reported here: pricing 60s timeouts, Telegram polling/send failures, slow Gateway RPCs, and stuck sessions across multiple hosts. The related pricing PR remains a narrow implementation candidate rather than a merged full resolution.

## Best Possible Solution

Keep this open as the broader Gateway runtime degradation tracker. Review or land the narrower pricing-startup PR if it is correct, then reproduce or instrument a long-running Gateway with Telegram enabled on affected hosts. Collect event-loop delay, handle/socket counts, undici dispatcher state, pricing refresh state, and RPC timings during stalls; compare with `models.pricing.enabled=false`; then either fix the shared runtime/network degradation or split a narrower canonical follow-up once the remaining cause is proven.
