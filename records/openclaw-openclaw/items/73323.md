## Summary

Keep this issue open. Current main has useful adjacent work for pricing opt-out, startup tracing, event-loop diagnostics, and Telegram transport cleanup, but I did not find a source-backed fix for the reported chronic cross-subsystem degradation: 60s pricing fetches, Telegram polling/send stalls, and slow normal Gateway RPCs after the process has been running. The report is fresh, detailed, and corroborated by multiple comments, including related #73454 and the narrower open #73486 pricing-startup PR.

## Best Possible Solution

Keep this open for Gateway runtime triage. Review or land/close #73486 as the pricing-startup subset, then run a long-lived Windows + Node 24 Gateway with Telegram enabled against current main or the next release. Compare models.pricing.enabled true versus false, collect event-loop delay/ELU, handle and socket counts, Undici dispatcher/Telegram transport state, diagnostics.stability output, and models.list/doctor.memory.status timings during stalls. Close or split only after the chronic Telegram/RPC degradation is proven fixed or narrowed to a new canonical issue.
