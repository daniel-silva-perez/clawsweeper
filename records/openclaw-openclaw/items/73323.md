## Summary

Keep this issue open. Current main has adjacent mitigation and observability work, but it does not contain a source-backed fix for the reported cross-subsystem Windows gateway degradation: pricing fetch 60s timeouts, Telegram polling/send stalls, and slow normal Gateway RPCs. The provided discussion includes multiple fresh Windows reproductions and an open, intentionally narrower pricing-startup PR, so this is still actionable maintainer triage rather than cleanup-close material.

## Best Possible Solution

Keep this open for Gateway and Telegram maintainer triage. Review or land a narrow pricing-startup mitigation such as #73486 if it is correct, but keep this issue tracking the broader chronic runtime degradation until maintainers collect Windows event-loop delay, handle/socket counts, undici dispatcher state, pricing-disabled behavior, Telegram transport diagnostics, and RPC timing during stalls, then fix the shared runtime/network cause or split a narrower canonical follow-up with evidence.
