## Summary

Current main still matches the reported problem: `generateSlugViaLLM` resolves the default/effective agent model and calls `runEmbeddedPiAgent` with `sessionKey: "temp:slug-generator"`, which maps to the queued `session:temp:slug-generator` lane. There is still no slug-specific lightweight model setting, no `completeSimple` slug path, and no early fallback/debounce. The discussion includes a concrete confirmation from @mikronn2 and the supplied context plus remote refs show #65134 remains the open implementation candidate, so this should stay open rather than be closed or queued as a separate automated fix.

## Best Possible Solution

Keep this issue open and review/land #65134 or an equivalent focused fix. The fix should move slug generation off the full embedded-runner temp session path, use a lightweight slug-specific model selection contract or an approved existing lightweight setting such as `agents.defaults.heartbeat.model`, preserve `hooks.internal.entries.session-memory.llmSlug: false`, preserve timestamp fallback behavior, and add regression coverage proving no `session:temp:slug-generator` embedded-runner lane use. If maintainers choose a new `slugModel` surface, it should include schema, help, docs, and tests.
