## Summary

Keep open. Current main still routes LLM session-memory slug generation through the default agent model and embedded-runner session/global lane machinery, with no dedicated lightweight slug model, direct simple-completion path, early fallback, or debounce. Open PR #65134 is the active implementation candidate, so this should not be queued as a separate automated fix PR.

## Best Possible Solution

Keep this issue open and review/land #65134 or an equivalent focused fix. The implementation should move slug generation off the full embedded-runner temp session path, select a lightweight slug-specific model through an approved config contract or existing lightweight-model setting, preserve `hooks.internal.entries.session-memory.llmSlug: false`, keep timestamp fallback behavior, and add regression coverage for model selection plus no `session:temp:slug-generator` embedded-runner lane use. If maintainers want the exact requested `slugModel` surface, it should be added through schema/help/docs, not only runtime code.
