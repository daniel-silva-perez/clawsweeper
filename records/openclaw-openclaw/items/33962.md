## Summary

Keep this issue open. Current main still uses the default agent effective primary model for LLM session-memory slugs and still runs slug generation through the embedded runner session/global lane machinery. The requested lightweight/dedicated slug model, direct completion path, early heuristic fallback, or debounce is not implemented yet. Open PR #65134 is the active implementation candidate, so this should not be queued as a separate automated fix PR.

## Best Possible Solution

Keep this issue open and use #65134 as the current review target, or implement an equivalent focused fix on main: move slug generation off the full embedded runner/temp session path, choose a cheap slug-specific model through a clear config contract or an approved existing lightweight-model setting, preserve `hooks.internal.entries.session-memory.llmSlug: false`, keep timestamp fallback behavior, and add regression coverage for model selection plus no embedded-runner lane use. If maintainers want the exact requested `slugModel` surface, that should be added through the normal config schema/help/docs path rather than only reusing heartbeat settings.
