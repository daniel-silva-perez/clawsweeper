## Summary

Current main still matches the reported problem: LLM slug generation resolves the effective/default agent model and runs through `runEmbeddedPiAgent` on `sessionKey: "temp:slug-generator"`, which maps to the queued `session:temp:slug-generator` lane. There is no slug-specific lightweight model setting, no `completeSimple` slug path, and fallback still waits for the LLM helper to return. The open #65134 PR remains the focused implementation candidate, so this issue should stay open for maintainer review rather than being closed or queued as a duplicate repair lane.

## Best Possible Solution

Keep this issue open and review/land #65134 or an equivalent focused fix. The best implementation should move slug generation off `runEmbeddedPiAgent` and `session:temp:slug-generator`, use `completeSimple` or an approved lightweight slug-model selection contract, preserve the existing `llmSlug: false` opt-out and timestamp fallback, and add regression coverage proving the slug path no longer queues on the embedded-runner temp session lane. If maintainers choose a new `slugModel` config surface, it should include schema/help/docs/tests.
