## Summary

Keep this PR open. Current main still warms the context-window/model metadata path for `openclaw message ...` because `message` is not in the eager-warmup skip list, and the existing regression test does not cover the message CLI case the PR adds. The PR is a focused implementation candidate rather than obsolete work.

## Best Possible Solution

Review and land this focused PR, or apply the same minimal change on main: add `message` to `SKIP_EAGER_WARMUP_PRIMARY_COMMANDS` in `src/agents/context.ts` and keep the regression assertion for `openclaw message send --channel telegram` in `src/agents/context.lookup.test.ts`. No broader product or security-sensitive change appears necessary.
