## Summary

Keep this issue open. Current main still has no WhatsApp `silentErrorReplies`, `suppressAgentErrors`, `errorPolicy`, `errorCooldownMs`, or custom agent-failure message config, and the same author has an open paired implementation PR (#59801) for WhatsApp `errorPolicy` support.

## Best Possible Solution

Keep the issue open and use #59801, or a revised maintainer-owned implementation, as the concrete path: add typed WhatsApp channel/account error policy config, schema/docs/generated metadata, runtime suppression/cooldown handling in WhatsApp auto-reply delivery, and focused tests for DM and account-scoped behavior.
