## Summary

Keep this issue open. Current main still lacks WhatsApp `silentErrorReplies`, `suppressAgentErrors`, `errorPolicy`, `errorCooldownMs`, or a custom agent-run failure message path, and the same author has an open paired implementation candidate in #59801.

## What This Changes

The issue requests a WhatsApp configuration path to suppress or customize user-visible agent-run failure replies, especially for direct WhatsApp business conversations.

## Best Possible Solution

Keep the issue open until #59801 or a revised maintainer-owned patch lands. The fix should stay WhatsApp-owned: add typed root/account error policy config, schema/generated metadata/docs, runtime suppression/cooldown handling in WhatsApp auto-reply delivery, and focused tests for direct chats and account-scoped behavior.

## Reproduction Assessment

Yes. Source and tests provide a high-confidence reproduction path: a WhatsApp direct conversation receiving a raw agent-run failure still gets the generic `/new`-style failure text, and adding the requested WhatsApp config keys is not accepted by the current strict schema.

## Solution Assessment

Yes, the narrowest maintainable direction is a WhatsApp channel/plugin implementation rather than a global core migration. #59801 has the right general boundary, but it still needs normal PR review and likely rebase/rework before it can close this issue.
