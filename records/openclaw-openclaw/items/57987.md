## Summary

Keep this PR open. Current main still implements only the older OpenRouter Anthropic system/developer `cache_control` behavior; it does not add the requested trailing user-turn marker, cap OpenRouter Anthropic markers to the last system/developer breakpoint, or backtrack to the nearest cacheable user block. The PR remains a plausible implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and review it as an implementation candidate. The best path is to land a focused version that updates the OpenRouter Anthropic marker helper/tests/docs/changelog, preserves the four-breakpoint limit, marks only cacheable blocks, and explicitly accounts for the shared plugin-SDK/GitHub Copilot use of the helper.
