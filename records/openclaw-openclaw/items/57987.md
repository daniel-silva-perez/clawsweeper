## Summary

Keep this PR open. Current main still lacks the requested OpenRouter Anthropic trailing user-turn cache marker and still documents/tests only the older system/developer marker behavior, while the PR remains a plausible focused implementation with no definite correctness finding from this read-only review.

## What This Changes

The PR changes the shared Anthropic cache marker helper and OpenRouter cache-control tests so native OpenRouter Anthropic requests mark only the last system/developer block plus the trailing user turn, with a changelog entry.

## Best Possible Solution

Land a focused version that updates the OpenRouter Anthropic marker behavior, tests, docs, and changelog while explicitly deciding whether the shared helper should also change GitHub Copilot/plugin-SDK behavior or whether the trailing user marker belongs in an OpenRouter-specific wrapper.

Yes. A source-level reproduction is clear: run an OpenRouter Anthropic payload with a system message followed by a user message through `createOpenRouterSystemCacheWrapper`; current main converts only the system content and the existing test asserts the user content stays unchanged.

## Solution Assessment

Unclear until maintainer review of the shared-helper boundary. The PR's core algorithm is a narrow and maintainable way to add the requested markers, but the safer alternative may be to keep the user-turn change OpenRouter-specific if GitHub Copilot or third-party SDK users should not inherit it.
