## Summary

Keep this PR open. Current main still lacks the PR’s intended OpenRouter Anthropic conversation-history cache marker: the wrapper only applies the existing shared system/developer marker helper, tests still assert trailing user messages remain unchanged, and docs still describe only system/developer OpenRouter cache markers. The PR remains an implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and review it as a focused implementation candidate. The best path is to land a current-main version that adds the trailing OpenRouter Anthropic user-turn marker, caps OpenRouter Anthropic requests to the intended system/developer plus user breakpoints, skips or backtracks around non-cacheable blocks, updates the targeted tests, and documents/changelogs the shipped behavior.
