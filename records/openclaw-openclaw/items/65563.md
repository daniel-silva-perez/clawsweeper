## Summary

Current main still appears affected: the TaskFlow SQLite migration copies legacy `owner_session_key` values into `owner_key` but leaves the legacy NOT NULL column in place, while new flow writes only populate `owner_key`. The webhooks `create_flow` path still creates a managed flow from the route-bound session, and the same-author implementation candidate #65767 remains open, so this issue should stay open for review/landing rather than be closed.

## Best Possible Solution

Keep this issue open and review #65767, or land an equivalent maintainer patch, that rebuilds or migrates legacy `flow_runs` tables so `owner_session_key` no longer blocks inserts, preserves existing owner data in `owner_key`, and adds focused regression coverage for `createManagedTaskFlow` or the webhooks `create_flow` path against a legacy SQLite schema.
