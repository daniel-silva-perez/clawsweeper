## Summary

Keep open. Current main still makes `tool_result_persist` and `before_message_write` synchronous-only, warns when a handler returns a Promise, and ignores the async result, so the reported governance gap remains real. This should not be auto-closed as stale or implemented.

## What This Changes

The issue requests async `Promise` return support for `tool_result_persist`, and notes the same limitation for `before_message_write`, so governance plugins can call external policy services before transcript persistence.

## Best Possible Solution

Keep the issue open for a maintainer decision on the right async pre-persistence governance seam: either make these persistence hooks awaitable with clear timeout/fail-open behavior and updated SDK/docs/tests, or introduce a new awaited tool-result governance hook before transcript append while preserving the existing sync contract.

## Reproduction Assessment

Yes. The existing `src/plugins/hooks.sync-only.test.ts` reproduces the reported behavior by registering async handlers for both hooks and asserting their Promise results are ignored.

## Solution Assessment

Unclear. Supporting async scanning is a valid goal, but simply adding `Promise` to the public type is not sufficient because the current implementation is deliberately synchronous and called inline before `appendMessage`.
