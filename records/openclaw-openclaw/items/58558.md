## Summary

Keep open. Current main still treats `tool_result_persist` and `before_message_write` as sync-only: the public handler types do not allow `Promise`, the hook runner warns and ignores promise-like returns, and tests explicitly lock that behavior. The reporter's governance-plugin use case remains a real plugin API gap rather than a ClawHub-only plugin request.

## Best Possible Solution

Keep this issue open for a focused plugin-SDK design. The best path is either an async-compatible persistence hook/variant or a documented public trusted tool-result transform API that covers external governance plugins, defines ordering relative to `after_tool_call`, bundled tool-result middleware, `tool_result_persist`, `before_message_write`, and transcript caps, and adds tests proving async redaction/blocking happens before persistence/model-visible replay.
