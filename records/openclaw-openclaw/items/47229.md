## Summary

Keep this open. Current main still accumulates Anthropic `input_json_delta` fragments into `partialJson` and parses them with `parseStreamingJson`, while the existing unsafe-integer guard remains Ollama-scoped, so Discord snowflake-sized numeric tool arguments can still be rounded before tool execution.

## What This Changes

The issue requests preserving large integer literals in Anthropic streamed `tool_use` input arguments instead of allowing JSON parsing to coerce them into imprecise JavaScript numbers.

## Best Possible Solution

Add an unsafe-integer-preserving JSON parse path for Anthropic streamed tool arguments before numeric coercion occurs, while keeping plugin boundaries clean and preserving the existing Ollama behavior.

## Reproduction Assessment

Yes. The current-main source path is clear, and a parser-level Node probe reproduces rounding for the reported Discord snowflake-sized value.

## Solution Assessment

Yes. A parser guard at the Anthropic tool-argument parse boundary is the narrow maintainable fix; monkey-patching vendored SDK or dependency internals would be more fragile.
