## Summary

Keep this open. Current main still routes Anthropic Messages streams through OpenClaw's transport, accumulates `input_json_delta` into `partialJson`, and parses the raw tool input with `parseStreamingJson`, whose dependency implementation reaches `JSON.parse`/`partial-json` without the existing unsafe-integer guard.

## What This Changes

The issue asks for Anthropic Messages streamed `tool_use.input` parsing to preserve unsafe large integer literals instead of rounding Discord-sized IDs through JavaScript number parsing.

## Best Possible Solution

Promote the unsafe-integer literal preservation into a neutral shared helper, use it for raw streamed Anthropic tool argument JSON before any `JSON.parse`-based parser sees it, and keep Ollama coverage aligned so unsafe integers become exact strings while safe integers remain numbers.

## Reproduction Assessment

Yes. A parser-level probe reproduces the rounding, and current main has a clear source path from Anthropic `input_json_delta` to `parseStreamingJson`/`JSON.parse`.

## Solution Assessment

Yes. A shared or Anthropic-local parser guard before numeric coercion is the narrow maintainable fix; monkey-patching vendored SDK or `partial-json` code would be more fragile and harder to validate.
