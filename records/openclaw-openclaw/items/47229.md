## Summary

Current main still has the reported unsafe-integer precision-loss path for Anthropic Messages tool-use input. The old Anthropic SDK `MessageStream` root cause has mostly moved to OpenClaw's boundary-aware transport, but that transport still accumulates `input_json_delta` into `partialJson` and parses it through `parseStreamingJson`, which reaches `JSON.parse`/`partial-json` without the Ollama unsafe-integer guard.

## Best Possible Solution

Keep this issue open and fix the Anthropic Messages tool-use parser so raw accumulated `input_json_delta` JSON is passed through an unsafe-integer-preserving guard before any `JSON.parse` or partial parser can round it. Prefer a neutral shared helper reused by Ollama and Anthropic, or a narrow Anthropic-local wrapper if extraction causes too much churn.
