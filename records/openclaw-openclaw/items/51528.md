## Summary

Current main still does not implement this PR’s intended MRU/custom-provider ordering for the Control UI chat model picker. The PR remains useful but stale: the active picker path is now split across `chat-model-select-state.ts` and `chat/session-controls.ts`, while the PR targets the older `app-render.helpers.ts` path. A security/supply-chain pass found no concerning infra, dependency, workflow, lockfile, or package-script changes in the provided PR file list; the remaining concern is behavioral porting correctness.

## Best Possible Solution

Keep this PR open as useful but stale work. Port the MRU/custom-provider ordering into `ui/src/ui/chat-model-select-state.ts`, persist successful non-default selections from `ui/src/ui/chat/session-controls.ts`, keep the Default option pinned, preserve missing active/default options, ignore stale recents, use `getSafeLocalStorage` with best-effort writes, store canonical resolved model refs, and add focused tests beside `chat-model-select-state.test.ts` plus the relevant chat/session-control tests.
