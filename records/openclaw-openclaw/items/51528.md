## Summary

Keep this PR open. Current main still lacks the PR’s intended behavior: the Control UI chat model picker uses the gateway catalog order directly, has no browser-local MRU storage, and does not persist successful selections as recent model refs. The PR remains useful but needs to be ported to the current UI split before it can land.

## Best Possible Solution

Keep this PR open as useful but stale work. The best path is to port the MRU/custom-provider ordering into the current `ui/src/ui/chat-model-select-state.ts` and `ui/src/ui/chat/session-controls.ts` flow, keep the Default option pinned, preserve active missing overrides, persist canonical resolved refs only after successful non-default `sessions.patch`, use `getSafeLocalStorage` with best-effort writes, and add focused tests beside `chat-model-select-state.test.ts` plus the chat/session-control tests.
