## Summary

Current main still does not implement this PR's intended recent-first and custom-provider-first Control UI chat model picker behavior. The PR remains useful, but it is stale against the current UI split: active option construction now lives in `ui/src/ui/chat-model-select-state.ts`, and the active switch path lives in `ui/src/ui/chat/session-controls.ts`.

## Best Possible Solution

Keep this PR open as useful but stale work. Port the MRU/custom-provider ordering into `ui/src/ui/chat-model-select-state.ts`, persist successful non-default selections from `ui/src/ui/chat/session-controls.ts`, keep the Default option pinned, preserve missing active/default options, ignore stale recents, use safe best-effort browser storage, store canonical resolved refs, and add focused tests beside `chat-model-select-state.test.ts` plus chat/session-control coverage.
