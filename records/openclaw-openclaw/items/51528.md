## Summary

Current main still lacks the PR’s intended behavior. The active Control UI picker builds options in catalog/current/default order with no browser-local MRU read, no custom-provider bucket, and no persistence after successful model switches. The PR remains useful, but it targets the older `app-render.helpers.ts` picker path and needs to be ported to the current `chat-model-select-state.ts` and `chat/session-controls.ts` split before it can land.

## Best Possible Solution

Keep this PR open as useful but stale work. Port the MRU/custom-provider ordering into `ui/src/ui/chat-model-select-state.ts` and persist successful non-default selections from `ui/src/ui/chat/session-controls.ts`; keep the Default option pinned, preserve missing active overrides/defaults, ignore stale recents, use `getSafeLocalStorage` with best-effort writes, store canonical resolved refs, and add focused tests beside `chat-model-select-state.test.ts` plus the relevant chat/session-control tests.
