## Summary

Keep open. Current main does not contain this PR's intended Apple Silicon utility/runtime work, but the proposed patch is not merge-ready because it regresses existing process execution and process-tree contracts, includes tests that cannot compile, and introduces a command-execution security concern.

## What This Changes

The PR branch adds a platform utility module, Apple Silicon-specific process execution/shell/process-tree changes, hybrid runtime/tensor/thread-pool helpers, docs, and a verification script.

## Best Possible Solution

Narrow the work to measured macOS ARM64 improvements that preserve existing process and shell contracts: keep Node's default execFile buffer behavior, respect user shell preferences, preserve detached-aware process-group cleanup, remove or separately justify the unintegrated hybrid/tensor/thread-pool modules, and add focused macOS ARM64 plus Linux/Intel regression tests and benchmark evidence.

## Reproduction Assessment

Yes for the blocking review findings: static inspection of the PR head shows non-compiling tests, changed maxBuffer behavior, non-Apple Unix process-group cleanup regression, and detached Apple Silicon timeout cleanup gaps. A live Apple Silicon performance reproduction was not provided, so the optimization claims themselves are not reproducible from the PR evidence.

## Solution Assessment

No. This PR is broader than the stated problem and changes core process/shell behavior without preserving current contracts; the safer solution is a narrow, benchmark-backed patch with regression tests for all supported platforms.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P1] Preserve runExec default maxBuffer capacity:** `src/process/exec.ts:115`
  - body: When callers pass an options object without maxBuffer, this now caps output at OPTIMAL_BUFFER_SIZE instead of leaving maxBuffer undefined for Node's execFile default. Existing call sites such as timeout-only runExec calls can now fail with ERR_CHILD_PROCESS_STDIO_MAXBUFFER on outputs that previously succeeded.
  - confidence: 0.96
- **[P1] Keep Unix group SIGKILL enabled off Apple Silicon:** `src/process/kill-tree.ts:81-83`
  - body: On Linux and Intel macOS, IS_APPLE_SILICON is false, so useProcessGroup becomes false and the grace-period SIGKILL branch skips the process group. That regresses killProcessTree by allowing descendants to survive after the parent pid is killed.
  - confidence: 0.95
- **[P1] Avoid detached spawns without group timeout cleanup:** `src/process/exec.ts:252-253`
  - body: Setting detached: true on Apple Silicon puts the child into a new process group, but the timeout path still calls child.kill("SIGKILL") on only the direct child. Grandchildren can survive timeouts, which is the opposite of the stated process-tree cleanup goal.
  - confidence: 0.92
- **[P2] Make dynamic-import tests async:** `src/utils/platform.test.ts:54`
  - body: This test body and several later ones use await import() inside a non-async arrow function. TypeScript rejects that syntax, so the new test file cannot compile until those callbacks are marked async or rewritten.
  - confidence: 0.99
- **[P2] Preserve thread-pool promise handlers:** `src/utils/thread-pool.ts:181-184`
  - body: run() stores the caller's resolve/reject handlers, but submitTaskToWorker immediately overwrites that activeTasks entry with no-op callbacks. If a task completion ever arrives, the Promise returned by run() will not settle for the caller.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: Needs attention: the patch changes command execution paths and adds a shell-injection sink in the new platform helper.

Concerns:

- **[medium] Avoid shelling out with raw SHELL:** `src/utils/platform.ts:166`
  - body: getShellCommand interpolates process.env.SHELL into an execSync string. A crafted SHELL value containing quotes or shell metacharacters could execute shell syntax when the helper is called; use fs.accessSync or execFile-style arguments instead.
  - confidence: 0.87
