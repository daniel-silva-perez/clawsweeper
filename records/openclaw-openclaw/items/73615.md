## Summary

Current main still appears to match the reported Telegram regression. Telegram preflight STT frames the transcript for the agent and marks the audio as already transcribed, while transcript echo is still only sent from the shared media-understanding path when that path produces new audio outputs.

## Best Possible Solution

Keep this issue open. The best fix is to add a preflight-safe echo path through a public media-runtime/helper seam or equivalent shared media-understanding integration, then invoke it once after Telegram has enough routing context and before agent processing. Add Telegram DM and mention-gated group voice-note regression tests with `tools.media.audio.echoTranscript=true` that verify framed agent context, one outbound echo, and no duplicate STT.
