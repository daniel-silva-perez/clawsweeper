## Summary

Keep this issue open. Current main still exposes only the lower-level SSRF hostname allowlist primitive, browser-specific hostname allowlist config, and a narrow Codex-native `tools.web.search.openaiCodex.allowedDomains` option. It still does not define or enforce the requested global/per-agent `allowedDomains` and `denyDomains` egress policy for OpenClaw-managed `web_fetch` or managed/provider `web_search`.

## Best Possible Solution

Keep the issue open and implement a first-class core web-tool network policy. Add typed/schema/docs support for global and per-agent allowedDomains and denyDomains, resolve it with other tool policies, enforce deny-before-allow for web_fetch through SsrFPolicy.hostnameAllowlist plus a deny check, include redirects and cache-key behavior in tests, and explicitly document what can and cannot be guaranteed for provider-native or managed web_search providers.

## Risks / Open Questions
