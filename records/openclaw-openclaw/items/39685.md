## Summary

Keep this issue open. Current main still has SSRF hostname allowlist primitives, browser-specific hostname allowlist config, and Codex-native `tools.web.search.openaiCodex.allowedDomains`, but it does not expose or enforce the requested global/per-agent `allowedDomains` plus `denyDomains` egress firewall for OpenClaw-managed `web_fetch` and `web_search`. An adjacent open PR, #67421, covers per-agent `web_fetch` SSRF overrides only and is not merged or complete for this request.

## Best Possible Solution

Keep this issue open. The best implementation is a first-class core web-tool network policy with typed/schema/docs support for global and per-agent `allowedDomains` and `denyDomains`, deny-before-allow resolution, enforcement in `web_fetch` including redirects and cache keys, explicit managed/provider `web_search` behavior, and clear docs for what provider-native search can and cannot guarantee. Review #67421 as a possible partial building block for `web_fetch`, but finish the broader policy before closing this issue.
