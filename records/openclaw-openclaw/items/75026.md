## Summary

Keep this open. Current main still has a plausible MiniMax Anthropic-compatible usage propagation gap: the Control UI Usage page aggregates persisted transcript usage, while the Anthropic transport only copies a small set of native Anthropic usage fields from `message_start` and `message_delta`; the MiniMax-specific Anthropic streaming path has no regression coverage for nonzero token usage reaching `lastAssistant.usage`/session JSONL. The closed related OpenAI-format MiniMax usage reports do not fully cover this Anthropic endpoint case.

## What This Changes

Track nonzero token usage for MiniMax Anthropic-compatible model runs in the Control UI Usage page instead of showing only message counts.

## Best Possible Solution

Fix the provider/transport side so MiniMax Anthropic-compatible streaming responses produce a nonzero assistant usage snapshot before session persistence. The UI should continue reading transcript-backed usage; it should not infer tokens from message counts or provider identity.

## Reproduction Assessment

Yes. The issue gives a concrete release, endpoint, environment, and JSONL symptom, and current source tracing shows a deterministic path where zero transport usage becomes zero persisted usage while message counts still aggregate.

## Solution Assessment

Yes, the narrow maintainable solution is to normalize MiniMax Anthropic streaming usage at the Anthropic transport or immediate usage-capture boundary and add regression coverage. A UI-only fix would hide the symptom without repairing `/usage`, session JSONL, cost summaries, and downstream compaction/status consumers.
