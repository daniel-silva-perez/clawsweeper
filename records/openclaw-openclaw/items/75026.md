## Summary

Current main still appears to have a real MiniMax Anthropic-compatible usage propagation gap: MiniMax is configured on the Anthropic Messages transport, but that transport only copies a narrow Anthropic-native usage shape before downstream session and Control UI aggregation. The issue should remain open and is narrow enough for a focused fix PR.

## What This Changes

The issue requests fixing MiniMax Anthropic-compatible token usage propagation so the Control UI Usage page shows token totals instead of only message counts.

## Best Possible Solution

Fix usage capture at the Anthropic-compatible transport or immediate subscription boundary so MiniMax streaming responses record a nonzero assistant usage snapshot before session persistence; keep the Control UI and Usage page backed by transcript/session usage rather than provider-specific UI inference.

## Reproduction Assessment

Yes. The issue provides endpoint, release, raw nonzero provider usage, and persisted zero-usage symptoms; current source tracing shows how missed transport usage remains zero through the accumulator.

## Solution Assessment

Yes. The maintainable solution is a narrow transport/subscription usage-normalization fix with MiniMax-shaped regression coverage; a Control UI-only workaround would leave session JSONL, `/usage`, status, and cost summaries wrong.
