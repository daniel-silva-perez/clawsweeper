## Summary

Keep open. Current main still has a credible MiniMax Anthropic-compatible usage propagation gap: MiniMax API-key provider config routes through `anthropic-messages`, but the Anthropic stream transport only copies native `input_tokens`/`output_tokens` fields from `message_start` and `message_delta`; downstream session and Usage page code can only display nonzero token totals after that transport snapshot exists.

## What This Changes

Fix MiniMax Anthropic-compatible token usage propagation so the Control UI Usage page shows nonzero token counts, not only message counts.

## Best Possible Solution

Fix usage capture at the Anthropic-compatible transport or immediate provider wrapper boundary so MiniMax streaming responses produce a normalized nonzero assistant usage snapshot before session persistence. Keep the Usage page transcript-backed rather than inferring token totals from message counts or provider identity.

## Reproduction Assessment

Yes. The issue includes a concrete release, endpoint, raw nonzero provider usage, and zero persisted JSONL usage; source tracing shows how a missed transport usage snapshot becomes zero/absent persisted token totals while message counts still aggregate.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize the MiniMax/Anthropic-compatible usage event shape at the transport boundary and add regression coverage; a UI-only change would hide the symptom while leaving session logs, status, and cost summaries wrong.
