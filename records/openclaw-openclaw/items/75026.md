## Summary

Keep open. Current main still has the MiniMax Anthropic-compatible usage propagation gap: the stream transport copies only native Anthropic usage fields and does not consume compatible-provider usage aliases or message_stop usage, while an open unmerged PR already references this issue with closing syntax.

## What This Changes

Ensure MiniMax Anthropic-compatible token usage propagates into persisted session logs and the Control UI Usage page instead of showing zero token counts while message counts work.

## Best Possible Solution

Review and land PR #75136, or an equivalent narrow transport-boundary fix, so MiniMax-compatible Anthropic streams normalize usage aliases and message_stop usage before assistant snapshots and transcript persistence. The Usage page should continue to consume persisted normalized session usage rather than infer tokens from provider identity or message counts.

Yes. The reporter supplied concrete endpoint/version details plus raw nonzero provider usage and zero persisted JSONL usage, and current-main source tracing shows the transport can produce that zero assistant snapshot before aggregation.

## Solution Assessment

Yes. Normalizing compatible-provider usage at the Anthropic stream transport boundary is the narrow maintainable fix; a Control UI-only workaround would leave session logs, status, cost summaries, and downstream aggregations wrong.
