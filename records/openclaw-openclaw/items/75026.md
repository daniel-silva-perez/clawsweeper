## Summary

Keep open. Current main still has the MiniMax Anthropic-compatible usage propagation gap, and open PR #75136 already references this issue with closing syntax, so this should remain open until that PR or an equivalent fix lands.

## What This Changes

MiniMax Anthropic-compatible runs should persist nonzero token usage so the Control UI Usage page shows token counts as well as message counts.

## Best Possible Solution

Review and land PR #75136 or an equivalent transport-level fix that normalizes MiniMax-compatible Anthropic stream usage from OpenAI-style aliases and late usage events before session persistence, with regression coverage at the SSE transport boundary and existing usage accumulator/session summary paths.

## Reproduction Assessment

Yes. The reporter provided a concrete release, MiniMax CN Anthropic endpoint, raw nonzero provider usage, and zero persisted JSONL usage; current-main source tracing shows how that usage can remain zero before session aggregation.

## Solution Assessment

Yes. The narrow maintainable solution is to normalize usage at the Anthropic-compatible transport or immediate provider wrapper boundary; changing only the Control UI would leave session logs, status, and cost summaries wrong.
