## Summary

Keep open. Current main still has the MiniMax Anthropic-compatible usage propagation gap, and the linked PR that explicitly closes this issue is open and unmerged, so the issue should remain open until that PR or an equivalent transport-boundary fix lands.

## What This Changes

MiniMax Anthropic-compatible token usage should be preserved from provider stream responses into session JSONL and the Control UI Usage page instead of showing zero tokens while message counts work.

## Best Possible Solution

Review, rebase, and land PR #75136, or an equivalent narrow fix in the Anthropic stream transport, so compatible-provider usage is normalized before assistant snapshots and session persistence. The Usage page should continue to consume transcript-backed usage rather than infer tokens from provider identity or message counts.

## Reproduction Assessment

Yes. The report gives exact setup details, nonzero raw provider usage, and zero persisted session usage, and current-main source tracing shows OpenAI-style aliases and `message_stop` usage can be dropped before aggregation.

## Solution Assessment

Yes. Normalizing compatible-provider usage at the Anthropic stream transport boundary is the narrow maintainable fix; a Control UI-only workaround would leave session logs, status summaries, and cost totals wrong.
