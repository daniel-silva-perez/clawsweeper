## Summary

Keep open. Current main still has the reported workspace-isolation bug, but this PR should not merge as-is because it adds an uncaught config load to the post-delivery transcript mirror path and the branch is currently dirty against main.

## What This Changes

The PR changes mirrored session transcript header creation to use a resolved agent workspace instead of `process.cwd()` and adds a regression test for per-agent cwd isolation.

## Best Possible Solution

Land one current-main patch that threads an explicit target agent workspace into mirrored transcript header creation without reloading config inside the post-delivery mirror path, preserves mirror failures as bookkeeping-only where delivery already succeeded, and coordinates any broader gateway transcript-header coverage with #56133.

## Reproduction Assessment

Yes. A focused reproduction is to create a new mirrored transcript while `process.cwd()` points at a different workspace and assert the first JSONL header records that wrong cwd on current main; the PR’s added test models that scenario.

## Solution Assessment

No. The requested direction is correct, but this implementation reloads config inside the append path instead of using already-available runtime context, and that can convert a mirror bookkeeping problem into a delivery retry/failure after the user already received the message.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Avoid reloading config from the delivery mirror path:** `src/config/sessions/transcript.ts:165-170`
  - body: `loadConfig()` can throw for invalid or unreadable config, and this append is awaited after a message has already been sent. That makes a transcript mirror failure reject delivery and may cause a retry/duplicate send; thread the existing config or resolved workspace into this path instead.
  - confidence: 0.91
