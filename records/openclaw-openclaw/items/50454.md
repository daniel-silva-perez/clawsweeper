## Summary

Keep this PR open. Current main still has the transcript-header behavior the PR targets: new mirrored transcript headers are created with `cwd: process.cwd()` and the append path does not resolve or pass an explicit target agent workspace. The PR also uses closing syntax for the same author's still-open bug report #49523, so it remains an implementation candidate rather than conservative cleanup. The submitted patch likely needs revision before merge because the review discussion flagged real reliability issues around ambient config loading and resolved-agent-id consistency.

## Best Possible Solution

Keep this PR open or supersede it with a cleaned-up fix. The best implementation is to write transcript headers with an explicit cwd derived from already-available runtime config plus the resolved target agent id, pass that context through the mirror append path instead of reloading config, forward a consistent resolved agent id through session-file resolution, and add a focused per-agent cwd isolation regression test.
