## Summary

Keep this PR open. Current main still has the gateway-side session reset machinery, but it does not expose the PR's intended plugin-facing `api.resetSession()` contract, result type, SDK export, registry wiring, or focused plugin contract tests. The PR is a plausible core/plugin-SDK seam rather than ClawHub-only work, but it still needs maintainer API review and resolution of the latest reset broadcast concern before merge.

## Best Possible Solution

Keep this PR open as the current implementation candidate. The best path is to review or update it to add `PluginResetSessionResult` and optional `api.resetSession()` to the public plugin API, wire it only in full registration mode behind `sessions.reset`, preserve exactly one `sessions.changed` broadcast for both gateway and plugin reset paths, keep any accepted config-snapshot/in-flight reset behavior, update SDK docs/baselines, and rerun the targeted plugin SDK plus gateway session tests.
