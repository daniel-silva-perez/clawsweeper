## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Enforce the loopback 10x threshold in the lockout path:** `src/gateway/auth-rate-limit.ts:214-215`
  - body: `recordFailure()` still applies the normal `maxAttempts` lockout for loopback clients. Because `check()` returns from `isExempt()` before honoring `lockedUntil` while attempts are below `maxAttempts * 10`, additional loopback failures stop being recorded and the intended 10x ceiling is never reached.
  - confidence: 0.9
- **[P2] Reject malformed webhook signatures without throwing:** `src/security/shield/webhook-dispatch.ts:63`
  - body: `verifyWebhookSignature()` compares hex string lengths before decoding, then passes decoded buffers to `timingSafeEqual()`. A same-length non-hex signature can decode to a shorter buffer and throw instead of returning `false`, turning a bad signature into an exception path.
  - confidence: 0.86
- **[P2] Refresh escalation state when circuits recover:** `src/security/shield/gateway-shield.ts:174-175`
  - body: `recordSuccess()` updates the circuit but does not recompute `this.escalation`. After an open circuit closes, `getStatus()` and `getSummary()` can keep reporting an escalated pause state until another failure reopens a circuit.
  - confidence: 0.84
- **[P2] Collapse health metrics by function before storing scores:** `src/security/shield/gateway-shield.ts:286-287`
  - body: `aggregateByFunctionAndMinute()` returns buckets per function and minute, but this loop writes each bucket into `healthScores` keyed only by function name. If buckets are not chronological, an older minute can overwrite newer health and make throttling depend on arrival order.
  - confidence: 0.78
- **[P2] Drop stale health scores when metrics leave the window:** `src/security/shield/gateway-shield.ts:301`
  - body: `computeHealthScores()` filters metrics to the recent window but never removes scores for functions absent from the new aggregate. Idle or recovered functions can keep old `CRITICAL` or `CIRCUIT_OPEN` scores, so `shouldThrottle()` may continue throttling from stale state.
  - confidence: 0.82
