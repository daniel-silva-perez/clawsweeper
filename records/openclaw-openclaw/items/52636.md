## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Enforce the loopback 10x threshold in the lockout path:** `src/gateway/auth-rate-limit.ts:214-215`
  - body: `recordFailure()` still sets the normal `maxAttempts` lockout for loopback clients, while `check()` returns from `isExempt()` before honoring that lockout while the count is below `maxAttempts * 10`. Once the normal lockout is set, further failures stop being counted, so localhost never reaches the intended 10x cap.
  - confidence: 0.9
- **[P2] Move evaluate hardening to the active browser path:** `src/browser/pw-tools-core.interactions.ts:305-307`
  - body: Current main no longer has `src/browser`; `evaluateViaPlaywright()` lives under `extensions/browser/src/browser`. Adding the blocked-pattern filter here either conflicts on rebase or leaves the active browser evaluate surface unchanged.
  - confidence: 0.88
- **[P2] Reject malformed webhook signatures without throwing:** `src/security/shield/webhook-dispatch.ts:63`
  - body: `verifyWebhookSignature()` compares hex string lengths before decoding, then passes decoded buffers to `timingSafeEqual()`. A same-length non-hex signature can decode to a shorter buffer and throw instead of returning `false`, which can turn bad input into an exception path.
  - confidence: 0.86
- **[P2] Refresh escalation state when circuits recover:** `src/security/shield/gateway-shield.ts:174-175`
  - body: `recordSuccess()` updates circuit state but never recomputes `this.escalation`. If a previously open circuit closes, `getStatus()` and `getSummary()` can keep reporting an escalated pause state until another circuit opens.
  - confidence: 0.84
- **[P2] Drop stale health scores when metrics leave the window:** `src/security/shield/gateway-shield.ts:301`
  - body: `computeHealthScores()` filters metrics to the last five minutes but only updates functions present in the new aggregate. Functions with no recent metrics keep stale `CRITICAL` or `CIRCUIT_OPEN` scores, so `shouldThrottle()` can keep throttling recovered or idle functions.
  - confidence: 0.82
- **[P2] Collapse health metrics by function before storing scores:** `src/security/shield/gateway-shield.ts:287`
  - body: `aggregateByFunctionAndMinute()` returns one bucket per function and minute, but this loop stores them by function name only. If buckets are not chronological, an older minute can overwrite newer health and make throttling depend on insertion order.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: Security review needs attention because the diff changes gateway/browser security policy and adds Kairos-attributed runtime protection code with stale paths, bypassable throttling, exception paths, and provenance gaps.

Concerns:

- **[high] Browser evaluate hardening targets an obsolete path:** `src/browser/pw-tools-core.interactions.ts:305`
  - body: The claimed evaluate mitigation is added under `src/browser`, while current main's active implementation lives under `extensions/browser`; until rebased and moved, the active evaluate surface is not hardened.
  - confidence: 0.86
- **[high] Loopback throttling remains bypassable:** `src/gateway/auth-rate-limit.ts:214`
  - body: The proposed loopback 10x cap is undercut by the normal lockout path, which stops additional failure recording before the 10x attempt threshold is reached.
  - confidence: 0.9
- **[medium] Malformed webhook signatures can throw:** `src/security/shield/webhook-dispatch.ts:63`
  - body: Webhook signature verification can throw for same-length malformed hex input instead of returning `false`, creating an exception path where verification should fail closed.
  - confidence: 0.86
- **[medium] Shield runtime state can remain stale:** `src/security/shield/gateway-shield.ts:174`
  - body: Escalation and health-score state can persist after recovery or after metrics age out, which is risky for a runtime subsystem that may pause or throttle gateway behavior.
  - confidence: 0.82
- **[low] Kairos-attributed code needs provenance review:** `src/security/shield/SHIELD.md:4`
  - body: The new Shield files state they are adapted from the Kairos Shield Protocol and by Kairos Lab, but the PR does not add local provenance or license evidence for the adapted code.
  - confidence: 0.76
