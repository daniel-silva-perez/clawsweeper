## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Enforce the loopback 10x threshold in the lockout path:** `src/gateway/auth-rate-limit.ts:194`
  - body: `recordFailure()` still applies the normal `maxAttempts` lockout to loopback entries, while `check()` returns from the relaxed loopback exemption before honoring that lockout. Once the normal lockout is set, additional failures stop being counted, so localhost never reaches the intended 10x cap.
  - confidence: 0.9
- **[P2] Move evaluate hardening to the active browser path:** `src/browser/pw-tools-core.interactions.ts:305-307`
  - body: Current main no longer has `src/browser`; `evaluateViaPlaywright()` lives under `extensions/browser/src/browser`. Adding the blocked-pattern filter here either conflicts on rebase or leaves the active browser evaluate surface unchanged.
  - confidence: 0.88
- **[P2] Reject malformed webhook signatures without throwing:** `src/security/shield/webhook-dispatch.ts:59`
  - body: `verifyWebhookSignature()` compares hex string lengths before decoding, then passes decoded buffers to `timingSafeEqual()`. A same-length non-hex signature can decode to a different buffer length and throw instead of failing closed with `false`.
  - confidence: 0.86
- **[P2] Recompute escalation when circuits recover:** `src/security/shield/gateway-shield.ts:197`
  - body: Escalation is only recalculated when a failure opens a circuit. When `recordSuccess()` moves a circuit toward recovery, `getStatus()` and `getSummary()` can keep reporting an old `PARTIAL_PAUSE` or `FULL_PAUSE` state until another circuit opens.
  - confidence: 0.84
- **[P2] Expire stale health scores without recent metrics:** `src/security/shield/gateway-shield.ts:265`
  - body: `computeHealthScores()` filters to recent metrics but never removes functions that dropped out of the window. Stale `CRITICAL` or `CIRCUIT_OPEN` scores can therefore keep throttling idle or recovered functions.
  - confidence: 0.82
- **[P2] Aggregate per-function health before updating scores:** `src/security/shield/gateway-shield.ts:287`
  - body: `aggregateByFunctionAndMinute()` returns separate buckets by function and minute, but the loop stores them by function name only. If buckets are not chronological, an older minute can overwrite newer health and make throttling depend on insertion order.
  - confidence: 0.78

  - body: The claimed evaluate mitigation is added under `src/browser`, while current main's active implementation lives under `extensions/browser`; until rebased and moved, the active evaluate surface is not hardened.
  - confidence: 0.86

  - confidence: 0.9

  - confidence: 0.86

  - body: Escalation and health-score state can persist after recovery or after metrics age out, which is risky for a runtime subsystem that may pause or throttle gateway behavior.
  - confidence: 0.82
