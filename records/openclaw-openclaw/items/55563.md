## Summary

Keep open. Current main still has the browser CLI/docs mismatch for the reported legacy `openclaw browser tabs --browser-profile remote` form, and the related compatibility PR #55567 was closed unmerged. The gateway half has later service-audit mitigation for version-managed Node paths, but there is not enough source/test proof that the reported WSL2 systemd+nvm failed-unit cycling case is fully fixed.

## Best Possible Solution

Keep this issue open and split or finish the two regressions deliberately: restore `openclaw browser tabs --browser-profile <name>` compatibility with a regression test, or update the affected WSL2 docs and changelog/migration notes if the new syntax is intentional; separately add focused proof that `doctor --fix` repairs or clearly diagnoses WSL2 systemd units pinned to nvm-managed Node paths.
