## Summary

Keep open. Current main still has the browser CLI/docs mismatch for the reported legacy `browser tabs --browser-profile` form, and the gateway doctor work is only partial mitigation for the WSL2/systemd+nvm cycling case. The related PR #55567 was closed unmerged, so there is no shipped canonical fix.

## Best Possible Solution

Keep this issue open and split or finish the two regressions deliberately: either restore `openclaw browser tabs --browser-profile <name>` with a regression test or update the affected docs/changelog if the new syntax is intended; separately add a WSL2/systemd+nvm failed-unit reproduction or unit test proving `doctor --fix` repairs or clearly diagnoses the version-managed Node service state.
