## Summary

Keep open. Current main still lacks a `heartbeat.tools` or heartbeat-scoped no-tools option. Heartbeat currently supports model, timeout, light context, isolated session, reasoning, prompt, delivery, and tool-error-warning controls, but the runner does not pass a tool-disable or tool-allowlist override into the agent turn. Existing global/per-agent tool policy can reduce tools for an agent, but it is not heartbeat-only and does not solve the reported local-model heartbeat problem.

## Best Possible Solution

Add a heartbeat-scoped tool policy to the existing heartbeat config surface, probably starting with `tools: false` and optionally an allowlist shape if maintainers want safe read-only tools. The implementation should update `HeartbeatSchema`, `AgentDefaultsConfig`, generated config metadata/docs, propagate the resolved heartbeat setting through `GetReplyOptions` into the existing `disableTools` or `toolsAllow` run seam, and add focused heartbeat-runner tests proving the model request gets no tool definitions when heartbeat tools are disabled.
