## Summary

Current main still lacks a heartbeat-scoped `tools: false`, `disableTools`, or tool allowlist option. The existing heartbeat config supports model, timeout, light context, isolated sessions, reasoning, delivery, and tool-error-warning controls, but it does not let heartbeat runs omit tool definitions while leaving normal turns unchanged.

## Best Possible Solution

Keep this issue open and add a heartbeat-scoped tool policy to the existing heartbeat config surface. A conservative implementation would add `tools: false` first, or a documented allowlist shape if maintainers want safe read-only tools, then update `HeartbeatSchema`, `AgentDefaultsConfig`, generated config metadata/help/docs, `GetReplyOptions`, and `runHeartbeatOnce` so heartbeat runs can pass `disableTools` or `toolsAllow` into the existing embedded runner seam. Add focused heartbeat-runner tests proving disabled heartbeat tools produce a model request with no tool definitions while normal user turns for the same agent keep their configured tools.
