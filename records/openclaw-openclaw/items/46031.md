## Summary

Keep open. Current main still routes implicit GitHub Copilot discovery through `resolveFirstGithubToken`, which enumerates stored Copilot profiles with `listProfilesForProvider` and selects `profileIds[0]` instead of using the order-aware auth-profile helpers. The issue is also paired with open PR #48682 using closing syntax, so the right maintainer path is to review, rework, merge, or close that PR rather than close this issue early.

## Best Possible Solution

Keep this issue open and use PR #48682 as the implementation lane if maintainers want to continue it. The reworked fix should live in `extensions/github-copilot/auth.ts`, use the order-aware plugin SDK helper or equivalent `resolveAuthProfileOrder` flow, cover both catalog discovery and the embedding caller that share `resolveFirstGithubToken`, and add focused tests for non-first `auth.order`, unresolved ordered tokenRef behavior, env-token precedence, and unchanged explicit provider config behavior.
