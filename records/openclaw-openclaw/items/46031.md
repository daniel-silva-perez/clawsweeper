## Summary

Keep open. Current main still has the reported behavior in the GitHub Copilot plugin: implicit Copilot discovery resolves stored profiles with `listProfilesForProvider(...)` and then selects `profileIds[0]`, so `auth.order` is not honored on that path. There is also an open unmerged PR, #48682, that uses closing syntax for this issue, so the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Review and land #48682, or an equivalent plugin-owned fix in `extensions/github-copilot/auth.ts`, so Copilot token resolution uses ordered usable profiles from the provider-auth SDK and falls through unresolved tokenRefs safely. Add focused tests for `auth.order` precedence and fallback in the GitHub Copilot discovery/auth path, then close this issue after the fix is merged.
