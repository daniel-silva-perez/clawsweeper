## Summary

Keep open. Current main mitigates A2A context pollution by marking `sessions_send` traffic as inter-session data, but it still executes the target turn and reply-back turns in the existing requester/target session keys, and there is no `ephemeral` A2A config or tool parameter.

## What This Changes

The issue requests an opt-in ephemeral peer-to-peer A2A mode for `sessions_send` that creates a clean temporary session for the exchange and cleans it up afterward.

## Best Possible Solution

Define a first-class A2A isolation mode in the session tool contract, then implement it in the `sessions_send` orchestration path so existing cross-agent allowlists still apply while requester/target main transcripts receive at most a summarized result or no persisted coordination turns, with docs and focused tests for cleanup and provenance.

## Reproduction Assessment

Yes. A static reproduction is high-confidence: current `sessions_send` calls gateway `agent` with the existing resolved target session key, and the A2A reply loop explicitly runs steps on the requester and target session keys; no Feishu-specific behavior is needed to see the missing ephemeral session boundary.

## Solution Assessment

Unclear. The requested direction is plausible, but the exact best fix needs maintainer product/API judgment because it changes public config/tool semantics and the persistence/audit model for cross-agent communication.
