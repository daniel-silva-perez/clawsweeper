## Summary

Keep open. Current main has agent-to-agent handling for `sessions_send`, but it still sends into the resolved existing target session and uses requester/target sessions for follow-up turns; there is no ephemeral session parameter or `tools.agentToAgent.ephemeral` config contract for the requested clean temporary exchange.

## What This Changes

Request an opt-in ephemeral agent-to-agent mode for `sessions_send` so inter-agent messages run in a temporary clean session without polluting either agent's main conversation.

## Best Possible Solution

Add a core opt-in ephemeral A2A mode only after the contract is settled: preserve existing allowlist and visibility checks, route the exchange through a transient isolated session, return or deliver the reply without writing to either main transcript, and document cleanup and retention behavior.

## Reproduction Assessment

Not applicable as a traditional bug reproduction because this is a feature request. The current behavior is high-confidence from static inspection: `sessions_send` targets `resolvedKey`, the A2A loop uses requester/target session keys, and no ephemeral option exists in tool or config schema.

## Solution Assessment

Unclear. An explicit ephemeral `sessions_send` or `tools.agentToAgent` opt-in is plausible, but the best fix needs maintainer design around API shape, cleanup, transcript retention, and reply delivery.
