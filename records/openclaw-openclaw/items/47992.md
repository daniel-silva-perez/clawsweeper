Current main still does not implement this PR's intended per-session chat draft preservation. The relevant Control UI session-switch path is now more centralized than when the PR was opened, and current main has adjacent per-session queue preservation, but `resetChatStateForSessionSwitch()` still clears `state.chatMessage` and there is no draft save/restore cache. Related #54655 remains open and unmerged, so it is useful context but not a shipped replacement.

## Best Possible Solution

Keep this PR open for maintainer review. The clean implementation should preserve drafts in the current centralized session-switch path, scope the draft key beyond bare `sessionKey` if needed, clear empty/deleted-session drafts, and add focused A -> B -> A restore plus empty-draft cleanup tests. Maintainers should decide whether to revise this focused PR or consolidate the work with #54655.
