## Summary

Keep this PR open. Current main still discards unsent Control UI chat drafts on session switch, but this branch is stale against the centralized switch helper and has concrete coverage/isolation gaps, so it needs maintainer consolidation with the overlapping open work in #54655 rather than cleanup closure.

## What This Changes

The PR adds an in-memory Control UI chat draft cache and wires save/restore calls into selected session and agent switching paths.

## Best Possible Solution

Ship one rebased Control UI change that preserves composer drafts in the centralized session-switch helper, scopes draft storage to the active gateway/session context, evicts empty and deleted-session entries, preserves existing per-session queue behavior, and adds focused tests for session dropdown, agent switch, and sidebar selection.

## Reproduction Assessment

Yes. On current main, type a draft in the Control UI composer, switch sessions through a visible path that calls switchChatSession(), and resetChatStateForSessionSwitch() clears state.chatMessage before the original draft can be restored.

## Solution Assessment

No, not as posted. The in-memory approach is a reasonable narrow boundary, but the posted branch should be rebased onto the centralized switch helper and adjusted for gateway scoping, sidebar coverage, deletion cleanup, and focused tests.

  - confidence: 0.86
