## Summary

Keep open. Current main still drops the chat composer draft on session switches, and this PR is useful but incomplete/stale: the branch misses a visible sidebar session-selection path, lacks cache cleanup for deleted sessions, and overlaps the open #54655 implementation.

## What This Changes

The PR adds an in-memory per-session chat draft cache plus save/restore helpers and wires them into Control UI session and agent switching callbacks.

## Best Possible Solution

Ship one rebased Control UI change that preserves composer drafts inside the centralized session-switch/reset helper, evicts empty or deleted-session draft entries, keeps existing queued-message preservation intact, and adds focused tests for draft restoration across session and agent switches.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve drafts from sidebar session selections:** `ui/src/ui/app-render.ts:1438`
  - body: The PR head still switches sessions in `onSessionSelect` with `state.setSessionKey(key)` and no save/restore call. Picking a session from the chat sidebar can leave the outgoing draft attached to the new session and does not restore the destination session's draft, so the proposed feature is incomplete for a visible switch path.
  - confidence: 0.86
- **[P3] Evict draft cache entries when sessions are deleted:** `ui/src/ui/app-render.helpers.ts:37`
  - body: `chatDraftCache` is module-level state with no deletion hook. Drafts for removed sessions would stay in memory until page reload; exposing a clear helper and calling it from the session deletion flow would keep the cache bounded.
  - confidence: 0.78
