## Summary

Keep this PR open, but do not merge it as-is. Current main still clears the Control UI composer draft on session switch, so the user problem is real, but the proposed patch is stale against the now-centralized switch path and introduces an under-scoped in-memory draft cache. Open #54655 overlaps the same remaining UX work, but it is also open and unmerged, so this needs maintainer coordination rather than cleanup closure.

## What This Changes

The PR adds an in-memory chat draft cache in the Control UI render helpers and wires save/restore calls into selected session and agent switching callbacks.

## Best Possible Solution

Ship one rebased Control UI change that saves and restores composer drafts through the centralized session-switch helper, scopes draft storage to the active gateway/session context, clears empty and deleted-session entries, preserves the existing per-session queue behavior, and adds focused restoration tests for session dropdown, agent switch, and sidebar selection.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Scope cached drafts to the active gateway:** `ui/src/ui/app-render.helpers.ts:37`
  - body: `chatDraftCache` is module-level and keyed only by `sessionKey`, but the Control UI can reconnect the same browser tab to another `gatewayUrl` where keys such as `main` are reused. Include the gateway/context in the cache key or clear drafts when the gateway changes so unsent text is not restored into another workspace.
  - confidence: 0.82
- **[P2] Preserve drafts from sidebar session selections:** `ui/src/ui/app-render.ts:1438`
  - body: The PR head still leaves the chat sidebar `onSessionSelect` path switching sessions without the save/restore wrapper. Selecting a session there can carry the outgoing draft into the destination session and cannot restore the destination draft.
  - confidence: 0.86
- **[P3] Prune deleted sessions from the draft cache:** `ui/src/ui/app-render.helpers.ts:37`
  - body: The module-level draft map has no clear helper or deletion hook, so drafts for removed sessions remain in memory until page reload. Wire cache eviction into the session deletion flow to keep stale draft text bounded.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: No supply-chain or code-execution issue was found, but the in-memory cache has a low-severity draft isolation problem because it is keyed only by session key.

Concerns:

- **[low] Drafts can bleed across gateway contexts:** `ui/src/ui/app-render.helpers.ts:37`
  - body: The PR stores unsent text in a module-level cache keyed only by `sessionKey`; after reconnecting the same Control UI tab to another gateway with a reused key like `main`, previously typed draft text can be restored in the wrong workspace view.
  - confidence: 0.74
