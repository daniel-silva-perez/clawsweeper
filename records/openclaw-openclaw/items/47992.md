Current main still clears the Control UI composer draft during session switches, so the requested behavior is not implemented, but this PR needs maintainer review before merge because the branch predates the current centralized `switchChatSession()` shape, leaves one PR-head session selection path uncovered, and overlaps the still-open #54655. ## What This Changes

The PR adds an in-memory per-session chat draft cache plus save/restore helpers and wires them into Control UI session and agent switching callbacks.

## Best Possible Solution

Ship one rebased Control UI change that preserves composer drafts in the centralized session-switch helper, removes stale draft entries for empty or deleted sessions, keeps the existing per-session queue behavior intact, and adds focused tests for draft restoration across session and agent switches.
