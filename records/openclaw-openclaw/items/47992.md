## Summary

Keep open. Current main still discards the composer draft during session switches, so the PR is not obsolete, but the branch as proposed has review-blocking gaps and overlaps with open PR #54655, which makes this a maintainer coordination/rebase item rather than cleanup to close.

## What This Changes

The PR adds an in-memory Control UI chat draft cache and wires save/restore calls into selected session and agent switching paths.

## Best Possible Solution

Ship one rebased Control UI change that preserves composer drafts through the centralized session-switch helper, scopes draft storage to the active gateway/session context, evicts empty and deleted-session entries, preserves existing per-session queue behavior, and adds focused tests for session dropdown, agent switch, and sidebar selection.

## Reproduction Assessment

Yes. On current main, visible session-switch callbacks flow into switchChatSession(), whose reset helper assigns state.chatMessage to an empty string; a source-level reproduction is to type a draft, switch sessions, and observe that the draft is discarded.

## Solution Assessment

No, not as posted. The in-memory approach is a reasonable narrow boundary, but the branch should be rebased onto the centralized switch helper and adjusted for gateway scoping, sidebar coverage, deletion cleanup, and focused tests.

## Review Findings

  - body: `chatDraftCache` is module-level and keyed only by `sessionKey`, but the Control UI can reconnect the same tab to another `gatewayUrl` where keys like `main` are reused.   - confidence: 0.82

  - confidence: 0.86
- **[P3] Prune deleted sessions from the draft cache:** `ui/src/ui/app-render.helpers.ts:37`
  - body: The module-level draft map has no clear helper or deletion hook, so drafts for removed sessions remain in memory until page reload. Wire cache eviction into the session deletion flow to keep stale draft text bounded.
  - confidence: 0.78
