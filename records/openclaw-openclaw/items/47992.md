## Summary

Keep this PR open. Current main still clears the Control UI composer draft during centralized session switches and does not include per-session draft storage, so the proposed behavior is not implemented on main. The branch is directionally relevant, but it predates the current centralized switch helper shape, overlaps open PR #54655, and needs maintainer review around cleanup semantics and tests before merge.

## What This Changes

The PR adds an in-memory per-session chat draft cache and wires save/restore calls into Control UI session and agent switching paths.

## Best Possible Solution

Ship a centralized Control UI draft-preservation implementation in `switchChatSession()` / `resetChatStateForSessionSwitch()`, keyed by session, with cleanup for empty or deleted sessions and focused tests that prove A -> B -> A draft restoration without weakening the existing queue preservation behavior.
