## Summary

Keep open. The requested UX fix is still valid because current main still clears the Control UI composer draft on session switches, but this PR is stale/dirty against current main, misses a visible switch path on its branch, and overlaps the open #54655 implementation, so it needs maintainer consolidation rather than cleanup closure.

## What This Changes

The PR adds an in-memory per-session chat draft cache and wires save/restore helpers into Control UI session and agent switching callbacks.

## Best Possible Solution

Ship one rebased Control UI change that preserves composer drafts in the centralized session-switch helper, removes stale entries for empty or deleted sessions, keeps the existing chatQueueBySession behavior intact, and adds focused tests for restoration across session and agent switches.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve drafts from chat session selections:** `ui/src/ui/app-render.ts:1438`
  - body: The PR head still switches sessions in onSessionSelect with state.setSessionKey(key) and no save/restore wrapper. Selecting a session from the chat sidebar can leave the outgoing draft attached to the new session and does not restore the destination session's draft, so one visible switch path is outside the proposed feature.
  - confidence: 0.86
- **[P3] Clear deleted sessions from the draft cache:** `ui/src/ui/app-render.helpers.ts:37`
  - body: chatDraftCache is module-level state with no eviction hook for deleted sessions. Drafts for removed sessions would stay in memory until page reload; exposing a clear helper and calling it from the session deletion flow would keep the cache bounded.
  - confidence: 0.78
