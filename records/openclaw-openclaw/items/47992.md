## Summary

Keep open. The underlying Control UI draft-loss bug is still present on current main, but this PR head is dirty against main, overlaps the still-open #54655, and misses a visible session-selection path on its own branch, so it needs maintainer consolidation or a rebase before merge.

## What This Changes

The PR adds an in-memory per-session chat draft cache and wires save/restore calls into Control UI session and agent switching callbacks.

## Best Possible Solution

Ship one rebased Control UI change that preserves composer drafts inside the centralized session-switch reset helper, evicts drafts for empty or deleted sessions, keeps the existing per-session queue behavior intact, and adds focused tests for restoration across session, agent, and sidebar switches.
