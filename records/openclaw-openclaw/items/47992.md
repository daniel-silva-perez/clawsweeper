## Summary

Keep open. Current main still clears the Control UI composer draft during the centralized session-switch reset and has no per-session draft cache, so the PR addresses a real remaining UX gap. It should go through maintainer review because the branch is dirty against current main, overlaps open #54655, and needs decisions about cache cleanup/scope and focused tests.

## What This Changes

The PR adds a module-level in-memory per-session chat draft cache in Control UI helpers and wires save/restore calls around session and agent switching paths in `ui/src/ui/app-render.helpers.ts` and `ui/src/ui/app-render.ts`.

## Best Possible Solution

Ship one rebased Control UI change that preserves composer drafts in the centralized session-switch helper, removes stale draft entries for empty or deleted sessions, keeps the existing per-session queue behavior intact, and adds focused helper/view tests for draft restoration across session and agent switches.
