## Summary

Keep open. Current main still clears unsent Control UI chat drafts on session switch, so the PR is not obsolete; however the branch is stale against the newer centralized switch path, overlaps the open #54655 implementation, and the proposed cache has concrete isolation/cleanup gaps before merge.

## What This Changes

The PR adds a module-level in-memory per-session chat draft cache and wires save/restore calls into Control UI session and agent switching callbacks.

## Best Possible Solution

Ship one rebased Control UI change that stores composer drafts in the centralized session-switch helper using a cache key scoped to the active gateway/session context, prunes empty or deleted-session drafts, preserves the existing per-session queue behavior, and adds focused A -> B -> A restoration tests across session, agent, and sidebar switches.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Scope cached drafts to the active gateway:** `ui/src/ui/app-render.helpers.ts:37`
  - body: `chatDraftCache` is module-level and keyed only by `sessionKey`, but the Control UI can reconnect to a different `gatewayUrl` without reloading and common keys like `main` are reused. A draft typed for one gateway can be restored into another gateway's session, so the cache key needs to include the active gateway/context or be cleared on gateway change.
  - confidence: 0.82
- **[P2] Preserve drafts from sidebar session selections:** `ui/src/ui/app-render.ts:1438`
  - body: On the PR head, `onSessionSelect` still switches sessions with `state.setSessionKey(key)` and no draft save/restore wrapper. Picking a session from the chat sidebar can carry the outgoing draft into the destination session and does not restore that destination session's cached draft.
  - confidence: 0.86
- **[P3] Clear deleted sessions from the draft cache:** `ui/src/ui/app-render.helpers.ts:37`
  - body: The module-level draft map has no clear helper or deletion hook, so drafts for removed sessions remain in memory until page reload. Add an eviction path from the session deletion flow so deleted sessions cannot keep stale drafts around.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: No supply-chain or code-execution risk was found, but the new in-memory draft cache has a low-severity draft isolation issue because it is keyed only by session key.

Concerns:

- **[low] Drafts can bleed across gateway contexts:** `ui/src/ui/app-render.helpers.ts:37`
  - body: The PR stores unsent text in a module-level cache keyed only by `sessionKey`; after reconnecting the same Control UI tab to another gateway with a reused key like `main`, previously typed draft text can be restored in the wrong context. This is local to the browser session but can expose sensitive unsent text to the wrong workspace/gateway view.
  - confidence: 0.74
