## Summary

Keep open. The original direct gateway re-entry theory is stale on current main, but the active failure mode remains: ACP parent-stream completion still lands as a session-scoped system event and depends on heartbeat wake delivery, while disabled/no-target heartbeat paths still skip before the yielded parent gets a new turn. The provided context also shows same-author PR #60402 is still the active implementation candidate, so this should stay open until that PR is reviewed, landed, replaced, or closed.

## What This Changes

The issue requests reliable yielded-parent wake/resume behavior when an ACP child completes, including deployments where heartbeat delivery is disabled or insufficient.

## Best Possible Solution

Ship or replace the open implementation with a shared yielded-parent completion delivery path that does not depend on optional heartbeat scheduling, and add regression coverage proving ACP completion wakes a yielded parent when heartbeat is disabled or the parent agent is not heartbeat-enabled.
