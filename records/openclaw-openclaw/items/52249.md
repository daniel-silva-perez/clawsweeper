## Summary

Keep open. Current main has moved past the original direct-resume theory, but the active ACP parent completion path still depends on session-scoped system events plus heartbeat wake delivery, and the same-author #60402 remains the open implementation candidate for the heartbeat-disabled gap.

## What This Changes

The issue asks for yielded parent sessions to resume automatically when ACP child work completes, without requiring a UI refresh, a manual inbound message, or optional heartbeat scheduling.

## Best Possible Solution

Land, revise, or replace #60402 with a shared yielded-parent completion delivery path that does not depend on optional heartbeat scheduling, and cover both ACP streamTo:"parent" completion and native yielded child completion with regression tests for heartbeat-disabled and split-explicit-heartbeat configurations.
