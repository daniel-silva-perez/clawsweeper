## Summary

Keep open. The original direct-resume root-cause text is stale, but current main still routes ACP parent stream completion through session-scoped system events plus heartbeat wake delivery, and the disabled/no-target-heartbeat paths still skip before a parent turn can run. The provided issue context also shows same-author PR #60402 remains open as the active implementation candidate, so this issue should stay open until that PR is reviewed, landed, replaced, or closed.

## What This Changes

The issue requests reliable ACP yielded-parent wake/resume behavior when an ACP child completes, including deployments where heartbeat delivery is disabled or insufficient.

## Best Possible Solution

Ship or replace the open implementation with a shared yielded-parent completion delivery path that does not depend on optional heartbeat scheduling, and add regression coverage proving ACP completion wakes a yielded parent when heartbeat is disabled or the parent agent is not heartbeat-enabled.
