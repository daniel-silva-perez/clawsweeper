## Summary

Keep open. Current main has moved past the original direct-resume theory, but ACP parent completion still goes through session-scoped system events plus heartbeat wake delivery, and the heartbeat-disabled gap remains tracked by the same-author open implementation PR #60402; the latest discussion also suggests the final fix may need to cover native yielded child completion, not only ACP.

## What This Changes

The issue requests a reliable yielded-parent wake path when ACP or native child completion occurs without relying on optional heartbeat delivery.

## Best Possible Solution

Add or land a shared yielded-parent completion delivery path that works without optional heartbeat scheduling, then cover ACP streamTo:"parent" completion and native child completion with heartbeat-disabled and split-heartbeat regression tests.

## Reproduction Assessment

Yes for the currently narrowed ACP path: enqueue an ACP streamTo:"parent" completion with heartbeat disabled or no wake handler, and current source leaves the system event queued without an independent parent turn. The native-child extension has credible logs in the discussion but needs its own focused regression.

## Solution Assessment

Unclear for the open paired PR as the complete solution: #60402 targets the narrow ACP heartbeat-disabled wake gap, but the best maintainable endpoint appears to be a shared yielded-parent completion primitive that also covers native child completion.
