## Summary

Keep this issue open. The original direct-resume root-cause writeup is stale, but current main still routes ACP parent completion through session-scoped system events plus heartbeat wake delivery, and current code still has disabled/no-handler/target-agent-disabled paths where those wakes do not produce a parent turn. The provided GitHub context also shows same-author PR #60402 remains open as an implementation candidate, so the paired contributor issue/PR should stay open until that PR is reviewed, landed, replaced, or closed.

## Best Possible Solution

Keep this issue open. Review, rebase, and either land or close PR #60402, or replace it with a shared yielded-parent completion wake/delivery primitive that is independent of optional heartbeat scheduling. The accepted fix should add regression coverage proving ACP child completion wakes a yielded parent when heartbeat is disabled or the parent agent is not heartbeat-enabled, and should evaluate whether the native spawn/yield report needs the same primitive.
