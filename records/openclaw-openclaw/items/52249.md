## Summary

Keep this issue open. Current main has moved past the old direct-resume root-cause theory, but ACP parent completion still depends on session-scoped system events plus heartbeat wake delivery, and that path still has disabled/no-handler/not-heartbeat-enabled skip cases. The same-author PR #60402 remains open as an implementation candidate, so this issue should stay open until that PR is reviewed, landed, replaced, or closed.

## Best Possible Solution

Keep this issue open. Review, rebase, and either land or close PR #60402, or replace it with a shared yielded-parent completion wake/delivery primitive that does not depend on optional heartbeat scheduling. The accepted fix should add regression coverage for ACP child completion waking a yielded parent when heartbeat is disabled or the parent agent is not heartbeat-enabled, and should evaluate whether native spawn/yield needs the same primitive.
