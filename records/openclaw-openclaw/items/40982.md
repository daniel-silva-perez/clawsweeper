## Summary

Keep this open. Current main and the latest release still preserve the reported 180-second resumed CLI no-output cap, the schema and resolver still require positive numeric watchdog values, and an open closing-linked PR is already the active implementation path.

## What This Changes

The issue requests raising or removing the CLI no-output watchdog defaults for fresh and resumed CLI sessions, with optional per-backend configuration to tune or disable the cap.

## Best Possible Solution

Land or revise #41022, or a successor, so shared CLI watchdog defaults no longer terminate legitimate long-running CLI turns at 180 seconds, supported disable/configuration semantics are explicit if accepted, global timeout enforcement remains intact, and regression coverage covers fresh and resumed backend resolution plus any disable path.
