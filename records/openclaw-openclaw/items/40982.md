## Summary

Keep this issue open. Current main still has the reported 180,000 ms resumed CLI no-output watchdog cap, fresh CLI runs still have a 180,000 ms floor, and an open unmerged implementation candidate (#41022) uses closing syntax for this issue.

## Best Possible Solution

Keep the issue open and review or supersede #41022. If the 30-minute default policy is acceptable, land it with focused regression coverage for resolveCliNoOutputTimeoutMs and bundled CLI backend defaults; if maintainers want a true disable mode, implement null/0 no-cap behavior deliberately across config schema, exported types, resolver, supervisor/live-session contracts, generated schema, docs, and tests.
