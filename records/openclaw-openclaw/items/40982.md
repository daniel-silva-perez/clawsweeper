## Summary

Keep open. Current main still retains the 180-second resume watchdog cap and fresh-session 180-second floor, the latest release tag preserves the same behavior, and open closing-linked #41022 remains the active implementation path rather than a reason to close this issue before merge.

## What This Changes

Raise, remove, or make configurable the 3-minute CLI no-output watchdog cap so long-running CLI-backed agent requests can continue up to the intended global/session timeout.

## Best Possible Solution

Land or revise #41022, or a narrow successor, so shared CLI watchdog defaults no longer terminate legitimate long-running CLI turns at 180 seconds; if disable/configuration semantics are accepted, make them explicit in the schema, resolver, and regression coverage while keeping global timeout enforcement intact.
