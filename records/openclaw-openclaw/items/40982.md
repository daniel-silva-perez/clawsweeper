## Summary

Keep open. Current main still has the reported 180-second resumed CLI no-output cap, the latest release tag still ships that behavior, and the linked open PR remains the active closing-syntax implementation path rather than a reason to close this issue before merge.

## What This Changes

The issue requests raising or disabling the CLI no-output watchdog cap so long-running CLI agent turns are not killed after about three minutes of silence.

## Best Possible Solution

Ship a narrow CLI watchdog update that removes the effective 180-second cap for normal long-running CLI sessions, with resolver, schema/types, tests, and public config behavior aligned; handle Monitor-specific expected-idle behavior as a scoped follow-up if the default raise does not cover it.

## Reproduction Assessment

Yes. Current main deterministically resolves resumed CLI runs to a no-output timeout capped at 180000ms, passes that timeout into the supervisor or Claude live-session timer, and the discussion includes matching production logs from Apr 29.

## Solution Assessment

Unclear. Raising the defaults to 30 minutes is a narrow maintainable first step, but the best final shape should still decide whether OpenClaw supports explicit no-cap semantics and whether Monitor-backed expected idle needs a separate exemption.
