## Summary

Keep this issue open. Current main still has the reported 180s fresh minimum and resumed-session maximum, still resolves the CLI no-output watchdog to a positive numeric timeout, and still passes that timeout into CLI process supervision. An open implementation candidate, #41022, directly targets this issue, so the maintainer path is to review and land or close that PR rather than close this issue early.

Keep this issue open and review #41022, or a maintainer-owned replacement, against current main.
