## Summary

Keep this issue open. Current main still has the reported 180,000 ms resume watchdog cap and the 180,000 ms fresh watchdog floor, the resolver always returns a bounded numeric watchdog value, and the config schema still rejects null/0 disable-style values. The issue also has an open unmerged implementation candidate, #41022, with closing syntax, so the best maintainer path is to review that PR rather than close the issue now.

## Best Possible Solution

Keep the issue open and review #41022 against current main. If the 30-minute default policy is acceptable, land it with focused regression coverage for resolveCliNoOutputTimeoutMs and default CLI backend behavior. If maintainers want the requested disable mode, implement it deliberately across config types, zod schema, resolver, supervisor/live-session contracts, and tests rather than closing the issue as fully solved by the current PR.
