## Summary

Keep this issue open. Current main still has the reported 180,000 ms fresh watchdog floor and resumed-session max, still returns a bounded numeric CLI no-output timeout, and still rejects null/0 disable-style watchdog values in config. There is also an open, unmerged implementation candidate, #41022, with closing syntax, so maintainers should review or supersede that PR before closing this issue.

## Best Possible Solution

Keep the issue open and review #41022 against current main. If the 30-minute default policy is acceptable, land it with focused regression coverage for `resolveCliNoOutputTimeoutMs` and bundled CLI backend defaults. If maintainers want the requested disable mode, implement it deliberately across config types, zod schema, resolver, supervisor/live-session contracts, and tests.
