## Summary

Keep this issue open. Current main still has the reported 180,000 ms resumed CLI no-output watchdog cap, fresh runs still have a 180,000 ms floor, the resolver still returns a bounded numeric timeout with no null/0 disable path, and open PR #41022 is an unmerged implementation candidate that marks this issue as its closing target.

## Best Possible Solution

Keep the issue open and review or supersede #41022. If maintainers accept a 30-minute default policy, land it with focused regression coverage for resolveCliNoOutputTimeoutMs and bundled CLI backend defaults. If maintainers want true disable behavior, implement null/0 no-cap support deliberately across config schema, exported types, resolver, supervisor/live-session contracts, generated schema, docs, and tests.
