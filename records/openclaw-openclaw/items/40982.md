## Summary

Keep open. Current main still has the reported 180-second resume watchdog cap and 180-second fresh-session floor, the latest release tag preserves those defaults, and open PR #41022 remains the active closing-linked implementation path rather than a reason to close this issue before merge.

## What This Changes

Request to raise, remove, or configure the CLI backend no-output watchdog so legitimate long-running CLI turns are not terminated after about 180 seconds of silence.

## Best Possible Solution

Land or revise a narrow CLI watchdog fix so legitimate long no-output CLI turns can complete, with defaults, resolver behavior, schema/types, tests, and any public config docs aligned while keeping overall timeout cleanup intact.

## Reproduction Assessment

Yes. Current main deterministically resolves resumed CLI runs to a no-output timeout capped at 180000ms, passes that timeout into the supervisor or Claude live-session timer, and recent production logs in the discussion match the exact 180-second failure.

## Solution Assessment

Unclear. Raising the defaults to 30 minutes is a narrow maintainable first step, but the full requested outcome may also need explicit disable/config semantics and possibly a separate expected-idle treatment for Monitor-backed workflows.
