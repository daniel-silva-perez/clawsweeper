## Summary

Keep this issue open. Current main still has the reported 180,000 ms resume watchdog cap and 180,000 ms fresh watchdog floor, the resolver always returns a bounded numeric timeout, and the config schema still rejects null/0 disable-style values. The issue also has an open unmerged implementation candidate, #41022, with closing syntax, so maintainer cleanup should not close this before that PR is reviewed or superseded.

## Best Possible Solution

Keep the issue open and review #41022 against current main. If the 30-minute default policy is acceptable, land it with focused regression coverage for `resolveCliNoOutputTimeoutMs` and default CLI backend behavior. If maintainers want the requested disable mode, implement it deliberately across config types, zod schema, resolver, supervisor/live-session contracts, and tests rather than treating the current code as already solved.
