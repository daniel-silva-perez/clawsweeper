## Summary

Keep open. Current main still keeps the effective 180-second resumed CLI no-output cap, the latest release tag still ships the same defaults, and open PR #41022 is the active closing-syntax implementation path rather than a reason to close this issue before merge.

## What This Changes

The issue requests raising, removing, or making configurable the CLI no-output watchdog cap that can terminate long-running resumed CLI sessions after roughly three minutes of silence.

## Best Possible Solution

Ship a narrow CLI watchdog fix that removes the practical 180-second cap for normal long-running CLI sessions, keeps plugin-exported defaults and bundled CLI backend behavior aligned, and decides whether no-cap configuration belongs in the same change or a scoped follow-up.

## Reproduction Assessment

Yes. A resumed CLI backend run with no watchdog override, an overall timeout of at least 600000ms, and no stdout deterministically resolves to a 180000ms no-output watchdog on current main; the Apr 29 production log reports that exact termination string.

## Solution Assessment

Unclear. Raising defaults to 30 minutes is a maintainable first step and is already represented by PR #41022, but the final best shape still needs maintainer judgment on explicit disable semantics and the Monitor expected-idle case.
