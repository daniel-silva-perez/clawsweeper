## Summary

Keep this issue open. Current main still has the reported 180,000 ms fresh floor and resumed-session max, the CLI resolver still always returns a bounded numeric watchdog timeout, and the config schema still rejects null/0 disable-style values. There is also an open, unmerged implementation candidate, #41022, with closing syntax, so the right maintainer action is to review or supersede that PR rather than close this issue now.

## Best Possible Solution

Keep the issue open and review #41022 against current main. If the 30-minute default policy is acceptable, land it with focused regression coverage for `resolveCliNoOutputTimeoutMs` and bundled CLI backend defaults. If maintainers want the requested disable mode, implement it deliberately across config types, zod schema, resolver, supervisor/live-session contracts, and tests.
