## Summary

Current main still does not implement the requested default Chrome OptGuide/WasmTts model-download suppression. The managed browser launcher only disables Translate/MediaRouter by default, headless mode only adds headless/GPU flags, the sandbox browser entrypoint has no matching Chromium feature disables, and the public config/docs surface still exposes only generic extraArgs or custom sandbox entrypoints. The related implementation PR #41213 is useful context, but it is closed unmerged and the requested behavior is not present on current main.

## Best Possible Solution

Keep this issue open. The best fix is a focused browser plugin and sandbox entrypoint change that centralizes the Chromium feature names, appends them to managed headless/server Chrome launches, mirrors the safe defaults in scripts/sandbox-browser-entrypoint.sh, adds launch-argument and entrypoint coverage, and updates browser/sandbox docs. PR #41213 is useful reference material, but maintainers need a fresh landed implementation.
