## Summary

Keep open: current main still lacks the requested Chrome OptGuide/WasmTts model-download suppression in the managed browser launcher and sandbox browser entrypoint, and the related implementation PR is closed and unmerged. This is narrow enough for a focused repair PR.

## What This Changes

The issue asks OpenClaw to disable Chrome Optimization Guide and WasmTts model downloads for headless/server browser launches by default or through a browser configuration option.

## Best Possible Solution

Ship a browser-plugin-owned default that suppresses Chromium background ML/optimization downloads for OpenClaw-managed headless/server and sandbox browser startup, preserves user-supplied `extraArgs`, adds focused launch-argument coverage, and documents the behavior and escape hatch.

## Reproduction Assessment

Yes for the OpenClaw-side gap: current-main launch-argument inspection shows the requested features are not disabled in managed launches or the sandbox entrypoint. I did not run a live Chrome download trace in this read-only review.

## Solution Assessment

Yes: a narrow replacement fix in the bundled browser plugin and sandbox browser entrypoint is the maintainable path. A new core migration or broad config redesign is not needed unless maintainers want an explicit opt-out setting.
