## Summary

Current main still does not implement the requested default Chrome OptGuide/WasmTts model-download suppression. Managed Chrome launch args only disable Translate/MediaRouter, headless resolution only adds headless/GPU behavior, the sandbox browser entrypoint has no matching --disable-features entry, and config/docs still expose only generic extraArgs or custom sandbox image/entrypoint workarounds. The related PR #41213 is useful reference material, but it is closed unmerged, patched the old src/browser path, and no open PR currently tracks the fix.

## Best Possible Solution

Keep this issue open and land a focused replacement fix on current main: define or centralize the disabled Chromium background ML/optimization feature list in the browser plugin, append it to buildOpenClawChromeLaunchArgs for OpenClaw-managed headless/server launches, mirror the safe defaults in scripts/sandbox-browser-entrypoint.sh, add launch-argument coverage, and update the browser/sandbox docs. Use PR #41213 as reference, but redo it on extensions/browser/src/browser/chrome.ts without unrelated generated/baseline churn.
