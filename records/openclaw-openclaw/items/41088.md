## Summary

Keep this issue open. Current main still does not implement default Chrome OptGuide/WasmTts model-download suppression, and the related implementation PR #41213 is closed unmerged. Managed Chrome launches still only disable Translate/MediaRouter, headless mode adds only headless/GPU flags, the sandbox Chromium entrypoint lacks the requested feature disables, and the config surface still only exposes generic browser.extraArgs rather than browser.disableModelDownloads or equivalent.

## Best Possible Solution

Keep this issue open. The best fix is a small browser-plugin and sandbox-entrypoint change that centralizes the OptGuide/WasmTts feature names, appends them to managed headless/server Chrome launches, mirrors the same safe defaults in scripts/sandbox-browser-entrypoint.sh, adds focused launch-argument and entrypoint coverage, and updates browser/sandbox docs. PR #41213 is useful reference material, but maintainers need a fresh landed implementation.
