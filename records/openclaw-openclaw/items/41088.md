## Summary

Current main still does not implement the requested Chrome OptGuide/WasmTts model-download suppression. The managed browser launcher still only disables Translate/MediaRouter, headless mode only adds normal headless/GPU flags, the sandbox Chromium entrypoint has no --disable-features entry for these model/TTS features, and config/docs still expose only generic browser.extraArgs as a workaround. The related PR #41213 is closed unmerged, so there is no landed implementation to close this issue against.

## Best Possible Solution

Keep this issue open. The best fix is to add a small, centralized disabled-Chrome-features list in the bundled browser plugin, include the OptGuide/WasmTts model-download suppression in managed headless/server launches and the sandbox browser entrypoint, update the public browser/sandbox docs, and add focused regression coverage for buildOpenClawChromeLaunchArgs plus a lightweight sandbox-entrypoint default check. The closed PR #41213 is useful reference material, but it is not a landed fix.
