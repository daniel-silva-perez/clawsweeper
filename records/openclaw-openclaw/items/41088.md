## Summary

Current main still does not implement the requested default Chrome OptGuide/WasmTts model-download suppression. Managed browser launches only disable Translate/MediaRouter, headless resolution only decides whether to add headless/GPU flags, the sandbox browser entrypoint has no matching disabled feature list, and the config/docs surface still exposes only generic browser.extraArgs or custom sandbox entrypoint workarounds. The related PR #41213 is useful reference material, but it is closed and unmerged.

## Best Possible Solution

Keep this issue open and land a focused replacement fix on current main: centralize the disabled Chromium background ML/optimization feature list in the browser plugin, append or merge it into buildOpenClawChromeLaunchArgs for OpenClaw-managed headless/server launches, mirror the safe defaults in scripts/sandbox-browser-entrypoint.sh, add launch-argument coverage, and update the browser/sandbox docs. Use PR #41213 as reference, but redo it against extensions/browser/src/browser/chrome.ts and the current config/docs surface.
