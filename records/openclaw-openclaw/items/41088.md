## Summary

Current main still does not implement the requested Chrome OptGuide/WasmTts model-download suppression. Managed Chrome launches only disable Translate/MediaRouter, headless mode only adds headless/GPU flags, the sandbox browser entrypoint has no OptGuide/WasmTts feature disables, and the config/docs surface still only offers generic browser.extraArgs or a custom sandbox entrypoint. The related implementation PR #41213 is useful reference material, but it is closed unmerged, so this issue should remain open.

## Best Possible Solution

Keep this issue open. The best fix is a focused browser plugin and sandbox entrypoint change that centralizes the Chromium feature names, appends them to managed headless/server Chrome launches, mirrors the safe defaults in scripts/sandbox-browser-entrypoint.sh, adds launch-argument and entrypoint coverage, and updates browser/sandbox docs. PR #41213 is useful reference material, but maintainers need a fresh landed implementation.
