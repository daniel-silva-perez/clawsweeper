## Summary

Keep this issue open. Current main still does not implement default Chrome OptGuide/WasmTts model-download suppression: managed launches only disable Translate/MediaRouter, headless mode adds only headless/GPU flags, the sandbox Chromium entrypoint lacks the requested feature disables, and the only current workaround surface is generic browser.extraArgs. The related implementation PR #41213 is closed unmerged, so there is no landed fix to close against.

## Best Possible Solution

Keep this issue open. The best fix is a small centralized browser-plugin change that appends the OptGuide/WasmTts feature names to managed headless/server Chrome launches, mirrors the safe defaults in scripts/sandbox-browser-entrypoint.sh, adds focused launch-argument and entrypoint coverage, and updates the browser/sandbox docs. PR #41213 is useful reference material, but maintainers need a fresh landed implementation.
