## Summary

Keep this open. Current main still intentionally removes the Chrome Extension Relay, rejects `driver: "extension"`, and migrates legacy configs to Chrome MCP `existing-session`; that migration does not restore the extension-specific video, chat, and auto-reattach capabilities requested here. The related #19289 browser-backend plugin interface is relevant, but it is still open and does not by itself implement this legacy relay path.

## What This Changes

Request restoration of the removed Chrome Extension Relay as an optional browser driver, including `driver: "extension"`, the port 18792 relay server, bundled extension files, and migration guidance for legacy users.

## Best Possible Solution

Maintainers should make an explicit browser strategy decision: either keep the legacy relay retired with clearer migration documentation, or add a vetted browser-backend/plugin path and a maintained optional Chrome Extension Relay that covers the missing existing-tab, video-control, chat, and auto-reattach workflows with security review and tests.

## Risks / Open Questions

- It is unresolved whether the right product direction is restoring a maintained core optional driver, extending the browser backend plugin API tracked by #19289, or documenting the relay as retired with non-equivalent migration limits.
