## Summary

Keep open. Current main still intentionally removes the Chrome Extension Relay, rejects `driver: "extension"`, and migrates legacy relay config to Chrome MCP `existing-session`; that migration does not restore the requested existing-tab extension relay, auto-reattach, video-control, and chat-automation workflows. The related browser-backend plugin interface remains open, and restoring a Chrome extension/CDP relay is security-sensitive enough to need maintainer product and security direction.

## What This Changes

Restore the removed Chrome Extension Relay as an optional browser driver with the gateway relay, bundled extension assets, and backward-compatible `driver: "extension"` configs.

## Best Possible Solution

Maintainers should make an explicit browser strategy decision: either keep the legacy extension relay retired and document the migration limits clearly, or design/land a vetted browser-backend/plugin API and maintain an optional Chrome Extension Relay implementation with security review, tests, docs, and migration behavior.
