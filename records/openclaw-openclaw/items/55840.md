## Summary

Keep open. Current main still intentionally removes the Chrome Extension Relay: the relay startup helper is a no-op, profile creation rejects `driver: "extension"`, doctor migrates legacy relay config to Chrome MCP `existing-session`, and the public docs describe existing-session limitations rather than restored relay behavior. The related browser-backend plugin/API item is relevant but broader and still open, so this needs maintainer product and security direction rather than cleanup closure.

## What This Changes

The issue requests restoring the legacy Chrome Extension Relay as an optional browser driver, including `driver: "extension"`, the integrated relay server, extension assets, and backward-compatible migration behavior.

## Best Possible Solution

Maintainers should make the browser strategy explicit: either keep the legacy relay retired and document the migration gaps clearly, or design a vetted browser-backend/plugin path and maintain an optional Chrome Extension Relay with security review, tests, docs, packaging, and legacy-config migration behavior.
