## Summary

Keep open. Current main still does not implement `tools.sessions.visibility="allowlist"` or a companion `tools.sessions.allowlist`; the core visibility type, runtime resolver, guard, config schema, generated config metadata, docs, and tests all still model only `self`, `tree`, `agent`, and `all`. The broader related #48785 remains adjacent rather than a canonical duplicate because it tracks per-capability visibility, not session-key pattern allowlisting.

## Best Possible Solution

Keep this issue open and implement it in the core session-tools visibility layer. Add `allowlist` to `SessionToolsVisibility`, add a strict `tools.sessions.allowlist: string[]` config/schema/docs/help/types surface, match glob/prefix patterns against canonical normalized session keys inside the shared visibility guard, and add focused tests covering list/history/send/status, sandbox clamping, overbroad globs, and interaction with `tools.agentToAgent`.
