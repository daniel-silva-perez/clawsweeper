## Summary

Keep this PR open. Current main still does not resolve quoted absolute Windows QMD `.cmd` entrypoints, and the PR remains a focused implementation candidate with a useful Windows reproduction. It still needs maintainer review because it changes a generic Windows spawn helper, leaves at least one runtime QMD `packageName: "qmd"` call site unchanged, and should tighten/cover the generic parser behavior before merge.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The best path is a narrow Windows-only fix that supports quoted absolute and UNC JS/EXE shim entrypoints without shell execution, keeps the source and packaged resolver copies synchronized, updates every QMD fallback package name that should use `@tobilu/qmd`, and adds portable regression coverage for the generic resolver plus QMD availability/runtime command paths.
