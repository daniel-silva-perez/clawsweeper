## Summary

Keep this PR open. Current main still has the reported Windows QMD gap: quoted absolute `.cmd` shim entrypoints are ignored, QMD fallback still uses the stale `qmd` package name, and current tests do not cover the reported `node "E:\\...\\@tobilu\\qmd\\...\\qmd.js" %*` wrapper form. The PR is a focused implementation candidate, but it needs maintainer review because it changes a public Windows process-spawn helper and may need to update one additional QMD runtime call site.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The best path is to land a narrow Windows-only fix that supports quoted absolute and UNC JS/EXE shim entrypoints without shell execution, keeps both Windows resolver copies synchronized, updates all QMD fallback package-name call sites that should use `@tobilu/qmd`, and adds regression coverage for the generic resolver plus the QMD availability/runtime path.
