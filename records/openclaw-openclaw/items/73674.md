## Summary

Keep this PR open. Current main still has the reported Windows QMD gap: both Windows `.cmd` resolver copies only unwrap `%~dp0`-relative quoted targets, while the QMD availability and runtime paths still use the stale `qmd` package fallback even though the docs install `@tobilu/qmd`. The PR is focused and the file list does not show unrelated workflow, dependency, lockfile, or publishing changes, but it touches command resolution and should get normal maintainer/security review plus targeted CI before merge.

## Best Possible Solution

Review and land a narrow resolver/QMD fix if CI is green: support quoted absolute Windows JS/EXE shim entrypoints without shell execution, keep both resolver copies aligned, update all QMD fallback package names to `@tobilu/qmd`, preserve the fail-closed unresolved-wrapper policy from #58436, and cover the plugin SDK, memory-host SDK, and memory-core runtime paths with portable tests.
