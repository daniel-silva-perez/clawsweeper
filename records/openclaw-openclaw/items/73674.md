## Summary

Keep this PR open. Current main still has the reported Windows QMD failure mode: quoted `.cmd` shim tokens are ignored unless they are `%~dp0`-relative, and the QMD fallback package name is still the stale unscoped `qmd`. The PR is a narrow active implementation for a real memory-core/Windows path, so it should get maintainer CI and security review rather than cleanup closure.

## What This Changes

The PR updates the plugin SDK and memory-host SDK Windows `.cmd` wrapper resolvers to accept quoted absolute JS/EXE entrypoints, changes QMD fallback package names to `@tobilu/qmd`, and adjusts QMD-related tests.

## Best Possible Solution

Ship a reviewed version of this narrow fix that keeps source and packaged resolver copies aligned, supports quoted absolute JS/EXE `.cmd` shim entrypoints without shell execution, uses `@tobilu/qmd` for QMD fallback resolution everywhere, and locks the behavior with focused Windows resolver and QMD runtime tests.
