## Summary

Keep this PR open. Current main still cannot resolve the reported quoted absolute QMD `.cmd` wrapper target, and the PR is a focused implementation candidate rather than obsolete cleanup. The latest patch also appears to address the earlier non-Windows test portability concern, but it still needs maintainer review because it changes command-resolution behavior and leaves one QMD package fallback call site inconsistent.

## Best Possible Solution

Keep this PR open for focused maintainer review or revision: support quoted Windows absolute and UNC JS/EXE entrypoints without shell execution, keep the duplicated SDK/package resolvers in sync, update every QMD package fallback call site that needs `@tobilu/qmd`, and land with portable regression coverage plus CI.

## Risks / Open Questions

- The PR changes command resolution before process spawn. Maintainers should review that quoted-token selection cannot choose an unrelated quoted argument before the intended executable or JavaScript entrypoint.
- The PR updates QMD package fallback in the availability probes but not the memory-core `runQmd` call site, so scoped-package package-json fallback may remain inconsistent outside the probe path.
- No tests were executed because this ClawSweeper pass was constrained to read-only inspection; PR CI or a maintainer run should cover the new unit test and Windows semantics.
