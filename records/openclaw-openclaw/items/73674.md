## Summary

Keep this PR open. Current main still does not support the PR's quoted absolute Windows QMD `.cmd` shim case, still passes the stale `qmd` package name in QMD fallback paths, and the proposed file list appears to miss the extension runtime QMD manager call site. The PR is a plausible focused implementation candidate with useful Windows repro context, but it needs maintainer review because it broadens a generic Windows spawn helper used outside QMD.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The best path is a narrow fix that supports quoted absolute and UNC Windows JS/EXE shim entrypoints without shell execution, keeps the source and packaged resolver copies synchronized, updates all QMD fallback package names that should use `@tobilu/qmd`, and adds portable regression coverage for the generic resolver plus QMD availability/runtime paths.
