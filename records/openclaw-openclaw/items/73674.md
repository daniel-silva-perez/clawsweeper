## Summary

Keep this PR open. Current main still has the Windows QMD gap the PR targets: the `.cmd` shim resolver only accepts `%~dp0`-relative quoted targets, QMD fallback package names still use `qmd`, and the runtime manager still passes that stale package name. The PR is narrow and appears relevant, but it touches Windows command resolution, so it needs normal maintainer/security review and CI rather than cleanup closure.

## What This Changes

This PR updates both Windows `.cmd` resolver copies to accept quoted absolute JS/EXE entrypoints, changes QMD fallback package names to `@tobilu/qmd`, and adds/updates QMD Windows resolver coverage.

## Best Possible Solution

Review and land this narrow resolver/QMD fix if CI and maintainer security review pass: support quoted absolute Windows JS/EXE shim entrypoints without shell execution, keep both resolver copies aligned, update all QMD fallback package names to `@tobilu/qmd`, preserve the fail-closed policy from #58436, and keep regression coverage across plugin SDK, memory-host SDK, and memory-core runtime paths.
