## Summary

Keep this PR open. Current main still lacks the PR’s intended QMD Windows `.cmd` shim behavior: both resolver copies only handle quoted `%~dp0`-relative shim targets, QMD fallback resolution still uses the stale `qmd` package name, and the runtime QMD manager has an additional unchanged `packageName: "qmd"` call site. The PR is a focused implementation candidate, but it needs maintainer review because it changes a generic Windows spawn helper used beyond QMD and appears incomplete for the extension runtime path.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The best path is a narrow fix that supports quoted Windows absolute and UNC JS/EXE shim entrypoints without shell execution, keeps the source and packaged resolver copies synchronized, updates all QMD fallback package names that should use `@tobilu/qmd`, and adds portable regression coverage for the generic resolver plus the QMD availability/runtime paths.
