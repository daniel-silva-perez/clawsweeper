## Summary

Keep this PR open. Current main still has the reported Windows QMD gap: both Windows `.cmd` shim resolvers only resolve quoted `%~dp0`-relative targets, QMD fallback lookup still uses `packageName: "qmd"`, and there is no current regression coverage for quoted absolute JavaScript entrypoints. The supplied PR is a focused implementation candidate, but it still needs maintainer review because it changes a generic Windows spawn helper and appears to miss the extension runtime QMD call site.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The best path is a narrow Windows-only fix that supports quoted absolute and UNC JS/EXE shim entrypoints without shell execution, keeps both resolver copies synchronized, updates all QMD package fallback call sites that should use `@tobilu/qmd`, and adds portable regression coverage for the generic resolver plus QMD availability/runtime paths.
