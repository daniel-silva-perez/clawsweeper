## Summary

Keep open. Current main still returns no usable transcript usage snapshot when a transcript has conversation text but no provider usage telemetry, so local/OpenAI-compatible sessions can still collapse to a missing or zero-ish context display despite the broader usage-alias fixes already present.

## What This Changes

The issue requests a conservative transcript-derived context estimate for local/OpenAI-compatible sessions when provider usage telemetry is absent or zero.

## Best Possible Solution

Add a narrow Gateway transcript fallback that preserves valid provider usage, but when a transcript has non-empty local/OpenAI-compatible conversation text and no positive usage telemetry, estimates prompt/context tokens conservatively and exposes that as a usable session status snapshot with regression coverage.

## Reproduction Assessment

Yes. A focused unit reproduction already exists in `src/gateway/session-utils.fs.test.ts`: a transcript with user and assistant content but no usage telemetry currently returns null, which explains the reported 0% context display path.

## Solution Assessment

Yes. The narrowest maintainable solution is in the transcript usage/status fallback path, not in provider adapters, because the problem is specifically the absence of telemetry after a valid transcript has already been persisted.
