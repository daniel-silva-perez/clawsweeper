## Summary

Keep open. Current main still returns no transcript usage snapshot for contentful local/OpenAI-compatible transcripts when provider usage telemetry is absent or zero, so status/session surfaces can still fall back to zero or missing context usage. The existing ClawSweeper comment is still consistent with the current checkout and the latest release tag.

## What This Changes

The issue requests a conservative transcript-derived context-usage fallback for local/OpenAI-compatible sessions when provider usage telemetry is missing or zero.

## Best Possible Solution

Implement a generic session-usage fallback that preserves valid provider usage first, then estimates prompt/context tokens from visible transcript text only when no positive usage-derived context total is available, with focused regression coverage for local/OpenAI-compatible no-telemetry transcripts.

## Reproduction Assessment

Yes. A deterministic reproduction exists in the current test fixture: a transcript with user and assistant content but no usage telemetry currently returns `null`, and the status path drops that missing snapshot before formatting context usage.

## Solution Assessment

Yes. The narrow extractor/status fallback direction is the best maintainable fix; provider transports and OpenAI-compatible usage normalization should stay authoritative whenever they report usable token telemetry.
