## Summary

Keep open. Current main still only derives status/session context from provider usage snapshots or transcript usage logs; it does not estimate context from transcript text when local/OpenAI-compatible providers omit or zero usage telemetry. The current regression test explicitly expects a transcript with user and assistant content but no usage snapshot to return null, and the latest shipped release carries the same gap.

## Best Possible Solution

Implement a narrow transcript-estimate fallback in the Gateway session usage path: keep provider-reported usage authoritative when present, but when a local/OpenAI-compatible transcript has conversation text and no positive usage telemetry, estimate prompt/context tokens conservatively from visible transcript text, mark the snapshot usable, and add regression coverage for the no-telemetry case.
