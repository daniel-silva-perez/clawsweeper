## Summary

Current main still treats multi-agent channel conversations as isolated session stores. Discord has an `allowBots` gate and a pending guild-history buffer for recent non-triggering channel messages, but that buffer is explicitly pending-only and is not shared cross-agent session history. The related Feishu report #59534 was closed as a lower-signal duplicate/context link, not as a merged canonical fix for this Discord behavior.

## Best Possible Solution

Keep this open and implement an explicit shared-channel history design if maintainers want this behavior: likely a Discord/channel-scoped, opt-in shared context buffer or bounded cross-session injection path, with clear attribution, loop prevention, and docs that distinguish pending channel history from durable per-agent session transcripts.
