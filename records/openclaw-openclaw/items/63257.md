## Summary

Keep open. Current main still has the reported Feishu startup shape: the startup bot-identity probe defaults to 30 seconds and is awaited before account transport starts, while the focused implementation PR remains open rather than merged.

## What This Changes

The issue requests eliminating a Windows gateway startup delay where Feishu API calls can time out after 30 seconds before Feishu continues.

## Best Possible Solution

Land or revise #74187 so Feishu startup uses bounded startup-safe bot-identity and WebSocket token waits, starts account transport promptly when identity lookup is slow, recovers identity in the background, and adds focused regression coverage for the Windows startup timeout path.
