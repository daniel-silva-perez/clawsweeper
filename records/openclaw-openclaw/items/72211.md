## Summary

Keep open. Current main still lacks the focused local dashboard-after-update recovery docs, and the PR documents real gateway lifecycle behavior, but the proposed runbook restarts the gateway before checking whether the dashboard is already healthy. The PR should be revised in place so restart/start is conditional after the listener and HTML checks.

## What This Changes

This PR adds a section to `docs/gateway/troubleshooting.md` for local dashboard connection failures after updates, including LaunchAgent recovery, `lsof`/`curl` diagnostics, gateway restart/start steps, and stale browser-state guidance.

## Best Possible Solution

Keep the docs-only scope, but make the flow diagnostic-first: check the listener and root dashboard URL with `lsof`/`curl`, treat returned OpenClaw HTML as a browser-state problem, and run `openclaw gateway restart` or `openclaw gateway start` only when the gateway is not listening or not serving the dashboard.

## Reproduction Assessment

Not applicable as a docs enhancement. The current-main docs gap is verifiable by inspection and search, and the PR-specific ordering bug is visible directly in the proposed diff.

## Solution Assessment
