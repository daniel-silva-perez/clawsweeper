## Summary

Keep open. Current main still lacks the proposed local-dashboard-after-update recovery runbook, so the PR is not obsolete; however, the patch should be revised before merge because it tells users to restart the gateway before first proving the gateway is actually down.

## What This Changes

The PR adds a local dashboard recovery section to `docs/gateway/troubleshooting.md` with LaunchAgent recovery, `lsof`/`curl` checks, `gateway restart`/`start`, and stale browser-state guidance.

## Best Possible Solution

Keep the docs addition, but make the flow diagnostic-first: check the listener and root dashboard URL with `lsof`/`curl`, treat returned HTML as a browser-state problem, and run `openclaw gateway restart` or `openclaw gateway start` only when the gateway is not listening or not serving the dashboard.

## Reproduction Assessment

Not applicable as a docs enhancement. The content gap is high-confidence from current-main docs inspection and targeted phrase search, and the PR-specific problem is visible directly in the proposed diff ordering.

## Solution Assessment

No, not as written. A docs-only fix is the right boundary, but the safest version makes restart/start conditional after the `lsof`/`curl` checks rather than the first action.
