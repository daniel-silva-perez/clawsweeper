## Summary

Keep open. Current main still lacks the focused local-dashboard-after-update recovery guidance, and the PR documents real gateway lifecycle behavior, but the proposed runbook is not merge-ready because it restarts the gateway before checking whether the dashboard is already being served.

## What This Changes

The PR adds a gateway troubleshooting section for local dashboard connection failures after updates, including LaunchAgent recovery, listener and curl checks, restart/start commands, and stale browser-state guidance.

## Best Possible Solution

Merge a docs-only runbook once the flow is diagnostic-first: check the listener and root dashboard response, preserve the stale-browser guidance when HTML is returned, and reserve restart/start for cases where the gateway is not listening or not serving the dashboard.

## Reproduction Assessment

Not applicable as a docs enhancement. The docs gap is verifiable from current-main docs search, and the PR-specific issue is directly visible in the proposed command ordering.

## Solution Assessment

No, not as written. A docs-only change is the right boundary, but the safest version checks lsof/curl before restart so a healthy gateway is not interrupted for a stale browser tab.
