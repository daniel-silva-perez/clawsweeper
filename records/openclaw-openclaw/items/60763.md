## Risks / Open Questions

- `bindSession=false` would intentionally clear stored route overrides. Because current dispatch falls back to the channel route and component-level `allowedUsers` is optional, shared components can change who can trigger callbacks unless the PR adds an explicit authorization rule such as required allowlists or originating-user enforcement.
- `expiresAtMs` would add a caller-controlled absolute expiry timestamp. The registry honors explicit `expiresAt` values, so the implementation should bound or normalize the maximum TTL before storing entries and modals.
- No supply-chain or CI execution expansion is visible in the provided PR file list; the review risk is behavioral/security semantics inside the bundled Discord channel, not dependency or workflow mutation.
- Closing now would drop an active implementation candidate while current main still lacks the requested Discord component fields.
