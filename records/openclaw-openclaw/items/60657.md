## Summary

Keep this issue open. Current main still formats collected busy-session followup prompts with `Queued #N` and the fixed title `[Queued messages while agent was busy]`; it does not include a total count in either the title or per-message markers. Related queue cleanup appears to have solved batching/default-mode issues, but not this narrower prompt-format request.

## Best Possible Solution

Implement this as a small auto-reply queue formatting change: include the delivered batch size in the collect title and render each item as `[Queued i/N]`, probably by extending `buildCollectPrompt` to pass `total` or by closing over `groupItems.length` in `src/auto-reply/reply/queue/drain.ts`. Update `queue.collect.test.ts` to assert the new marker shape and keep the existing routing, auth-group, retry, sender suffix, and overflow-summary behavior intact.

## Risks / Open Questions

- Changing the prompt format should preserve existing sender suffixes and authorization-group splitting, so grouped prompts count the messages actually delivered in that batch.
- The same shared collect helper is used by subagent announce queues, which still render `Queued #N`; maintainers should decide whether only inbound user-message queues or all collect-mode queues should adopt `N/total` markers.
