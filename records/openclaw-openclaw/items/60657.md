## Summary

Keep open. Current main still renders collect-mode busy-session batches with the fixed title `[Queued messages while agent was busy]` and per-item `Queued #N` markers, while tests continue to assert that old marker shape. The earlier ClawSweeper comment remains accurate against the newer main SHA.

## What This Changes

Add total-aware sequence markers to queued collect-mode user messages, such as `[Queued 1/3]`, so the model can track all messages in a batch.

## Best Possible Solution

Implement total-aware collect prompt formatting for delivered user-message batches, preserving sender suffixes, auth-group splitting, routing metadata, retry behavior, image forwarding, dedupe behavior, and overflow summary placement. If the shared helper is changed to pass total, update adjacent announce queue expectations deliberately rather than by accident.

## Reproduction Assessment

Yes. A read-only current-main check shows same-route collect-mode followups still build a prompt containing `Queued #1` and `Queued #2`, and the existing collect retry test asserts that shape; no runtime test was run because this review had to keep the checkout read-only.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix: change the collect prompt renderer or helper contract to include `[Queued i/N]` and a counted title, then update focused queue tests. A broader queue architecture change is not needed for this issue.

## Repair Work Prompt

Fix the collect-mode queued message prompt format for https://github.com/openclaw/openclaw/issues/60657. In `src/auto-reply/reply/queue/drain.ts`, change delivered busy-session collect batches from `Queued #N` to total-aware markers like `[Queued i/N]`, and include the delivered batch size in the collect title, e.g. `[Queued messages while agent was busy - 3 total]` or the existing project-preferred ASCII equivalent. Count the messages actually delivered in each auth/routing group, not the original queue before auth splits, cross-channel fallback, drops, or overflow summaries. Preserve sender suffixes such as `(from Guest)`, image forwarding, routing metadata, auth-group splitting, retry semantics, dedupe behavior, and overflow summary placement. Update `src/auto-reply/reply/queue.collect.test.ts` and related queue tests that assert the old marker shape. If you change `buildCollectPrompt` in `src/utils/queue-helpers.ts` to pass total into the renderer, update `src/agents/subagent-announce-queue.ts` and its tests deliberately, or keep the helper backward compatible and leave announce behavior unchanged. Add a user-facing changelog fix entry with `Thanks @justmaker` if the project treats this prompt-quality behavior as user-facing. Do not change queue defaults, debounce/cap/drop policy, authorization grouping, routing keys, or Feishu-specific queue behavior.
