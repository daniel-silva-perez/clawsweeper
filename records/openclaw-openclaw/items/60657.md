## Summary

Keep this issue open. Current main still renders collected busy-session followup prompts with the fixed header and per-item `Queued #N` markers, without the requested `N/total` sequence markers. The prior ClawSweeper review remains accurate against current main; the relevant queue files have not changed since that review.

## Best Possible Solution

Implement a small collect-mode prompt-format change in `src/auto-reply/reply/queue/drain.ts`: include the batch size in the header and render each delivered item as `[Queued i/N]`, preserving sender suffixes and existing grouping behavior. Update `src/auto-reply/reply/queue.collect.test.ts` for the new marker shape and decide whether the shared `buildCollectPrompt` renderer should receive `total` so subagent announce queues can stay consistent or intentionally remain unchanged.

## Risks / Open Questions

- The implementation should count messages per delivered collect batch after routing and authorization grouping, not the original pre-split queue length.
- Sender suffixes, overflow summaries, retry behavior, image payload collection, and cross-channel/thread splitting should be preserved.
- `src/agents/subagent-announce-queue.ts` also uses `Queued #N` with the shared collect helper, so maintainers should decide whether the marker change applies only to inbound user-message queues or to all collect prompts.
