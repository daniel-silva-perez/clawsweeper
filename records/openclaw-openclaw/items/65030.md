## Summary

Keep open. Current main has the merged Bedrock known-context-window lookup from #65952, but it still does not implement this PR's fallback-provenance behavior, so unknown/defaulted Bedrock models and inference profiles can still be classified as model-sourced context windows. The branch is also dirty against main and has unresolved review findings before it could merge safely.

## What This Changes

This PR adds a Bedrock context-window fallback provenance flag across discovery, model resolution, and context-window guard call sites, with tests for default and user-configured context-window behavior.

## Best Possible Solution

Rebase onto current main, keep the known-window table for mapped Bedrock models, add explicit fallback provenance only for unknown/defaulted Bedrock values, thread the field through the shared and runtime model types plus all guard call sites, include explicit-default provenance in the discovery cache key, and cover unknown foundation models, unknown inference profiles, and explicit `defaultContextWindow` in focused tests.
