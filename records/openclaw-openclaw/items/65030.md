## Summary

Keep this PR open. Current main still lacks the proposed fallback provenance, so the central Bedrock warning problem remains real; #65952 is complementary because it fixed known model windows but not unknown/default fallback classification. The branch is not merge-ready because the diff has blocking type/runtime issues and needs a careful rebase over the merged Bedrock known-window table.

## What This Changes

This PR adds a Bedrock `contextWindowIsLiteralDefault` provenance flag across discovery, model resolution, and context-window guard call sites, with tests for fallback and explicit default behavior.

## Best Possible Solution

Rebase onto current main, preserve the known Bedrock context-window table, add fallback provenance only for unknown/defaulted Bedrock values, thread the field through shared and runtime model types plus all guard call sites, include explicit-default provenance in the discovery cache key, and cover unknown foundation models, unknown inference profiles, and explicit `defaultContextWindow` in focused tests.

Full review comments:

- **[P1] Thread the provenance field into ProviderRuntimeModel:** `src/agents/pi-embedded-runner/run/setup.ts:115`
  - body: The new call sites read `contextWindowIsLiteralDefault` from `ProviderRuntimeModel`, but that type still does not declare the field. Strict TypeScript should reject the changed setup/extensions/compact reads, so add the optional field to `src/plugins/provider-runtime-model.types.ts` or avoid reading it through that type.
  - confidence: 0.91
- **[P1] Pass discovery config into profile fallback:** `extensions/amazon-bedrock/discovery.ts:290-294`
  - body: `resolveInferenceProfiles` reads `config?.defaultContextWindow` in the missing-foundation fallback, but `config` is not in scope inside that helper. That is both a type error and a runtime `ReferenceError` for unknown inference profiles, so pass the discovery config into the helper and use that parameter.
  - confidence: 0.97
- **[P2] Include default provenance in the Bedrock cache key:** `extensions/amazon-bedrock/discovery.ts:166-169`
  - body: The new flag depends on whether `defaultContextWindow` was explicitly set, but the cache key only includes the resolved numeric window. A cached implicit 32000 result can be reused for an explicit `defaultContextWindow: 32000` call, returning stale provenance until cache expiry.
  - confidence: 0.9
