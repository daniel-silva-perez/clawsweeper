## Summary

Keep this PR open. Current main still lacks explicit fallback provenance for unknown Bedrock context-window defaults, but this branch is not merge-ready: it is far behind current main and has concrete type, runtime, and cache-key correctness findings that need maintainer-guided rebase work.

## What This Changes

The PR adds an optional Bedrock context-window provenance flag through discovery, runtime context-window resolution, model-resolution call sites, and focused tests.

## Best Possible Solution

Rebase onto current main, preserve the known Bedrock context-window table, add explicit fallback provenance only for unknown/defaulted Bedrock values, thread the field through shared and runtime model types plus all guard call sites, include explicit-default provenance in the discovery cache key, and cover unknown foundation models, unknown inference profiles, explicit defaultContextWindow, and the warning path in focused tests.

## Reproduction Assessment

Yes. Source review reproduces the blocking defects: the PR diff reads an undeclared config variable in resolveInferenceProfiles, and the current pi-ai Model plus ProviderRuntimeModel types do not expose the new runtime field.

## Solution Assessment

No, not as submitted. The provenance-flag direction is plausible and complementary to #65952, but the branch needs a current-main rebase, complete type propagation, cache-key provenance, and alignment with the default-source warning behavior before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Pass discovery config into profile fallback:** `extensions/amazon-bedrock/discovery.ts:293`
  - body: When an inference profile has no matching foundation model, this new fallback reads `config?.defaultContextWindow`, but `config` is not in scope inside `resolveInferenceProfiles`. TypeScript should reject this, and JavaScript would throw a ReferenceError in the missing-foundation path this fallback is meant to handle.
  - confidence: 0.98
- **[P1] Add the provenance field to ProviderRuntimeModel:** `src/agents/pi-embedded-runner/run/setup.ts:115`
  - body: This new access is on `params.runtimeModel`, whose type is `ProviderRuntimeModel`; that type currently only adds `contextTokens`, `params`, and `requestTimeoutMs` to pi-ai's `Model<Api>`, and pi-ai's model type does not declare this property. The same type error applies to the other new runtime accesses, so the shared runtime model type needs the optional field too.
  - confidence: 0.93
- **[P2] Include default provenance in the discovery cache key:** `extensions/amazon-bedrock/discovery.ts:166-168`
  - body: The new flag depends on whether `defaultContextWindow` was omitted versus explicitly set to the same numeric value, but the cache key only includes the resolved number. With refresh caching enabled, a call using implicit 32000 can be reused for an explicit 32000 call, or the reverse, returning stale provenance until cache expiry.
  - confidence: 0.9
