## Summary

Keep this PR open. Current main still does not implement backend-synced Control UI message deletion: the UI delete action only hides a message group in browser localStorage, and the Gateway/protocol/client surfaces still have no `sessions.truncate` or equivalent sequence-based transcript mutation. The PR remains a plausible implementation candidate, but it needs focused maintainer and security review before merge.

## Best Possible Solution

Keep this PR open and review or rebase it as the implementation candidate. The best path is a focused change that adds a typed `sessions.truncate` protocol contract, a server-side sequence-based transcript mutation with safe locking/rollback and active-run cleanup, Control UI wiring that avoids local-only fallback, generated Swift model updates, and targeted regression tests for success, stale seq, active runs, webchat rejection, permissions, rollback, and race behavior.
