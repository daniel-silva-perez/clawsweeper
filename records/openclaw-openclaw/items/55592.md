## Summary

Keep this PR open. Current main still has the reported Control UI/backend divergence, and the PR is a relevant implementation candidate, but the proposed `sessions.truncate` handler has a blocking transcript-targeting bug for branched SessionManager transcripts because it maps UI sequence numbers by raw JSONL order instead of active-branch order.

## What This Changes

This PR adds a `sessions.truncate` Gateway RPC with protocol/Swift model wiring, operator scope registration, server-side transcript mutation, and Control UI delete handling that calls the backend instead of only hiding messages locally.

## Best Possible Solution

Keep the PR open and require the truncate implementation to resolve the requested UI sequence against the active SessionManager branch entry/id before choosing the raw file cut point. The final mergeable shape should include focused regression coverage for linear transcripts, branched transcripts, compaction entries, stale seq/no-op behavior, active-run cleanup, webchat rejection, file-mode preservation, rollback, and Control UI rollback behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a SessionManager transcript with an abandoned branch before the active branch: `readSessionMessages()` assigns `__openclaw.seq` from `getBranch()`, while the PR handler counts raw JSONL message lines, so the same seq can point at a different file line.

## Solution Assessment

No. The overall direction is right, but the current PR is not the best safe implementation until sequence mapping uses the same active-branch semantics as `readSessionMessages()` and only then performs the destructive transcript rewrite.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Map truncation seqs through the active branch:** `src/gateway/server-methods/sessions.ts:1387-1390`
  - body: The UI seq values come from `readSessionMessages()`, which assigns them while iterating `SessionManager.open(filePath).getBranch()` for tree transcripts. This scan counts every raw JSONL message/compaction line instead, including abandoned branch entries, so deleting seq N in a branched session can cut the wrong file line and remove unrelated history. Resolve the seq to an active-branch entry/id before choosing the raw cut line.
  - confidence: 0.9
