## Summary

Keep open for maintainer review. Current main still preserves Ctrl+C as clear/warn/exit text with no active-run hint, while this PR is a narrow text-only implementation and #44223 is an overlapping broader behavior change that would make Ctrl+C abort active runs.

## What This Changes

This PR appends a `/abort` active-run hint to the TUI Ctrl+C clear and warn status messages in `src/tui/tui.ts` without changing keybindings or abort behavior.

## Best Possible Solution

Keep one coherent TUI interaction policy: either land this narrow active-run status hint with aligned PR text and focused proof, or supersede it with #44223's Ctrl+C abort behavior and resolve the paired bug through the implementation that actually lands.

## Solution Assessment

Unclear. This patch is the narrowest maintainable fix if Ctrl+C should remain clear/exit, but #44223 may be the better product fix if maintainers want Ctrl+C to stop active runs directly.
