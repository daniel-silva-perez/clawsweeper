## Summary

Keep open. The patch misses the fast-submit window described by the linked bug: current main marks a message as pending before any chat event assigns `activeChatRunId`, while this PR only shows the hint when `activeChatRunId` is already set. The open #44223 alternative also means maintainers should choose between a text-only clarification and changing Ctrl+C to abort active runs.

## What This Changes

This PR appends a TUI Ctrl+C status hint in `src/tui/tui.ts` when `activeChatRunId` is set, without changing abort behavior or keybindings.

## Best Possible Solution

Adopt one TUI interaction policy consistently: either amend this PR so the status hint covers both pending optimistic submissions and assigned active run ids with focused TUI tests, or use #44223 if Ctrl+C should abort active runs directly.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to submit a TUI message and press Ctrl+C before the first chat event arrives: `pendingOptimisticUserMessage` is true, `activeChatRunId` is still null, and this PR's hint condition does not fire while the run can continue.

## Solution Assessment

Unclear. A text-only hint is the narrowest maintainable solution if Ctrl+C should remain clear/exit, but this diff is too narrow for the reported timing; #44223 may be the better product direction if maintainers want Ctrl+C to abort active runs.

## Review Findings
