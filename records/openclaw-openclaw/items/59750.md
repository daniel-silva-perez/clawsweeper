## Summary

Keep open. Current main still hard-prioritizes a stored session model override during normal session reuse, and the session config schema/docs do not expose a resume strategy knob; the related model-configuration cleanup report is broader and still open, not a shipped substitute for this specific request.

## What This Changes

The issue asks for a configurable resume model strategy so resumed sessions can either keep the saved session model or use the current default model.

## Best Possible Solution

Add a backward-compatible enum config for resumed-session model selection, defaulting to current `session` continuity behavior and documenting the opt-in `default` behavior alongside session lifecycle/model docs; wire it through the central model-selection path rather than provider-specific code.

## Solution Assessment

Add a configurable resumed-session model strategy so ordinary resumed sessions can either keep the saved session model override (`session`, current/default behavior) or ignore saved session model overrides and use the current configured default model (`default`).
