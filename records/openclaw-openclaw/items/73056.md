## Summary

Keep open. Current main still has the outer raw-message pre-check that can return `no real conversation messages` before Pi SDK preparation exists, and open PR #73092 already references this issue with closing syntax for the narrow fix.

## What This Changes

Manual `/compact` on long tool-heavy Discord-thread sessions should not be skipped as idle merely because the limited raw session slice lacks a recent user message.

## Best Possible Solution

Review and land PR #73092, or an equivalent narrow change, so genuinely idle sessions still skip while over-threshold manual compaction can proceed even when the limited raw slice is tool-heavy.

## Reproduction Assessment

Yes. A source-level reproduction is clear: construct a limited session slice whose recent entries are heartbeat/tool-result messages with the meaningful user ask outside the 20-entry lookback, then trigger manual compaction with high context usage; current main can hit the outer skip before SDK preparation exists.

## Solution Assessment

Yes for the proposed direction. Passing reliable observed token usage into manual `/compact` and bypassing only the no-real-conversation outer skip when already over threshold is narrower than changing the shared classifier and preserves the idle-session protection.
