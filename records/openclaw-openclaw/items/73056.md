## Summary

Keep open. Current main still has the outer raw-session compaction pre-check that can return `no real conversation messages` before Pi SDK preparation/turn-prefix context exists, and open PR #73092 already tracks the narrow fix with closing syntax. The issue should remain open until that PR is reviewed and merged or closed. https://github.com/openclaw/openclaw/issues/73056

## What This Changes

Manual `/compact` on over-threshold, tool-heavy sessions should not be skipped by the outer no-real-conversation pre-check.

## Best Possible Solution

Land #73092 or an equivalent narrow fix that preserves the idle-session skip below threshold while allowing over-threshold manual compaction to proceed when the limited raw message slice is tool-heavy.

## Reproduction Assessment

Yes. A source-level reproduction is clear: use a limited session message slice whose recent entries are tool results or boilerplate with the meaningful user turn more than 20 entries back, while the session is already over the compaction threshold; current main can return the outer skip before SDK preparation exists.

## Solution Assessment

Yes. Passing reliable observed token usage into manual compaction and bypassing only the outer idle skip at threshold is narrower than changing the shared message classifier and keeps the original idle-cron protection intact.
