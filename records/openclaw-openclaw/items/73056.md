## Summary

Keep open. Current main still has the outer raw-session compaction pre-check that can return `no real conversation messages` before Pi SDK preparation supplies turn-prefix context, and open PR #73092 is the active closing implementation candidate.

## What This Changes

The issue reports that manual `/compact` can skip over-threshold, tool-heavy Discord thread sessions because an outer idle pre-check inspects only the limited raw session messages.

## Best Possible Solution

Keep the idle-session skip for genuinely empty below-threshold sessions, but allow over-threshold manual compaction to proceed past the outer pre-check; #73092 is the current implementation path for that end state.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a limited message slice containing only boilerplate/tool results with the meaningful user turn more than 20 entries back, plus over-threshold context usage; current main returns the outer skip before SDK preparation can add prefix context.

## Solution Assessment

Yes, the threshold bypass direction is the narrowest maintainable fix visible from current code. It avoids changing the shared classifier while preserving the original idle-cron protection for below-threshold sessions.
