## Summary

Keep open. Current main still has the outer compaction pre-check that can return `no real conversation messages` from the limited raw session slice before the Pi SDK preparation exists, and an open linked PR is already tracking the narrow fix with closing syntax, so this issue should stay open until that PR is reviewed and merged or closed.

## What This Changes

Manual `/compact` should not classify a long, tool-heavy session as idle when the session is already at or above the compaction threshold.

## Best Possible Solution

Land a narrow fix that preserves the idle-session skip for genuinely empty or below-threshold sessions but prevents over-threshold manual compaction from being short-circuited by the limited raw-message slice; the active linked PR is the current review target for that end state.

## Reproduction Assessment

Yes. A source-level reproduction is clear: build a limited session slice whose recent entries are tool results or boilerplate with the meaningful user turn more than 20 entries back, then call manual compaction with high context usage; current main can hit the outer skip before the SDK preparation can supply prefix context.

## Solution Assessment

Yes for the direction. Passing reliable current token usage into manual `/compact` and bypassing the outer idle skip when the session is already over threshold is narrower than changing the shared message classifier and keeps the original idle-cron protection intact.
