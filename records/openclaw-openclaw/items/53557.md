Keep this PR open. The intended behavior is still missing from current main, but the branch targets old core memory files that no longer exist; the useful change needs to be ported to the active `extensions/memory-core` plugin implementation before it can merge.

## What This Changes

This PR changes the old core memory CLI/tool paths to show memory-search mode, fallback/provider issue details, and a more specific Ollama auto-mode unavailable hint.

## Best Possible Solution

Port the useful behavior to `extensions/memory-core`: render Mode and Provider issue in human memory status, consolidate fallback display into one row, include `providerUnavailableReason` in the `memory_search` success payload when present, add the Ollama auto-mode hint in the shared unavailable-result helper, and cover those surfaces with memory-core tests.
