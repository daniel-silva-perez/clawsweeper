## Summary

Keep this PR open. The proposed behavior is still useful, but current main has moved the active memory CLI/tool implementation into `extensions/memory-core`, while this branch edits legacy `src/cli` and `src/agents/tools` paths that are no longer present. Current main also still omits the requested human status rows, still prints fallback as separate `from` and reason lines, and still does not return `providerUnavailableReason` from successful `memory_search` results.

## What This Changes

The branch adds memory status display rows and an Ollama auto-mode unavailable-action hint in the legacy memory CLI/tool files.

## Best Possible Solution

Port the useful behavior into `extensions/memory-core`: render search mode and provider issue in human `openclaw memory status`, consolidate fallback into one readable row, include `providerUnavailableReason` in successful `memory_search` results when present, and keep the Ollama auto-mode hint specific without changing provider selection semantics.
