## Summary

Keep this PR open. Current main still does not implement the PR’s full intended behavior: the memory status command does not display search mode or provider-unavailable reason, and the memory_search success payload still omits providerUnavailableReason. The PR also needs to be rebased/ported from the old core paths to the current memory-core plugin files and should address the duplicate fallback output review before merge.

## Best Possible Solution

Port the intended visibility improvements into the current memory-core plugin files, add focused tests for CLI status mode/provider issue output and memory_search providerUnavailableReason, consolidate fallback rendering to one clear line, and keep the small Ollama/auto-mode action hint if it still matches current provider resolution behavior.
