## Summary

Keep this PR open. The requested memory-search visibility work is useful, but the branch is not mergeable as-is against current main: it changes legacy memory CLI/tool paths that are absent from the current tree, while the active implementation lives in `extensions/memory-core`. The diff also still misses the promised successful `memory_search` `providerUnavailableReason` payload and would need fallback rendering consolidated in the active status renderer.

## What This Changes

The PR adds memory status display rows and an Ollama auto-mode unavailable-action hint in legacy memory CLI/tool files.

## Best Possible Solution

Port the behavior into `extensions/memory-core`: render search mode and provider issue in human `openclaw memory status`, replace split fallback output with one readable row, include `providerUnavailableReason` in successful `memory_search` results when present, and keep the Ollama auto-mode hint specific without changing provider selection semantics.
