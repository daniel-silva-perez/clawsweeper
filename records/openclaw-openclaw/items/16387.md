## Summary

Keep this issue open. Current main still treats configured provider headers as static strings, the Fireworks plugin does not add session-affinity headers, and the newer provider turn-state hook is only an adjacent provider-owned seam rather than config-level placeholder interpolation. The two linked implementation PRs are closed unmerged, while the maintainer discussion has already narrowed the remaining work to a small runtime resolver.

## What This Changes

The issue requests dynamic interpolation for custom LLM provider headers, such as `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}`, to support Fireworks.ai prompt-cache affinity and similar tracing headers.

## Best Possible Solution

Land a narrow documented runtime resolver for configured model-provider headers at model-request dispatch, covering both `models.providers.*.headers` and `models.providers.*.request.headers` with explicit placeholders while preserving existing static-header behavior and transport policy.
