## Summary

Keep open. Current main still treats configured provider headers as literal strings, the runtime wrapper path does not carry the requested session/run template context, docs still describe static headers, and the related implementation PRs are closed unmerged while maintainer discussion frames this as remaining narrow work.

## What This Changes

Add runtime interpolation for configured custom LLM provider headers so placeholders like `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` can be used for provider-specific routing headers such as Fireworks affinity or LiteLLM/Langfuse tracing.

## Best Possible Solution

Land a narrow documented resolver for configured model-provider headers at model-request dispatch, covering both provider-level and request-level headers with explicit placeholders while preserving existing static-header behavior, SecretRef resolution, auth injection, attribution headers, retries, and provider-owned hooks.
