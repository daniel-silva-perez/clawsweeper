## Summary

Keep this issue open. Current main still treats custom provider header values as literal/static config, the Fireworks plugin does not add `x-session-affinity`, and the existing provider-plugin header hooks are provider-owned seams rather than arbitrary `models.providers.*.headers` or `request.headers` templating for custom providers. The issue also has multiple concrete use cases and two closed, unmerged implementation PRs, so the work remains valid.

## Best Possible Solution

Keep the issue open and implement a narrow, documented runtime header-template resolver for `models.providers.*.headers` and `models.providers.*.request.headers`. Resolve only explicit placeholders such as `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` at request dispatch, preserve static headers and SecretRef behavior, cover main turns, compaction, retries, and OpenAI-compatible transports, and document the Fireworks and LiteLLM/Langfuse examples.
