## Summary

Keep open. Current main still preserves configured provider headers as static strings, has no resolver for the requested placeholders, and the Fireworks plugin does not add session-affinity behavior; the recent discussion and closed unmerged PRs make this a valid narrow feature gap rather than cleanup to close.

## What This Changes

Request dynamic interpolation for configured custom-provider headers, including placeholders such as `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}`, so providers and proxies can receive session-aware routing or tracing headers.

## Best Possible Solution

Land a narrow documented dispatch-time resolver for configured model-provider headers covering both provider-level and request-level headers with explicit placeholders, while keeping static headers unchanged and avoiding Fireworks- or Langfuse-specific behavior in core.

## Reproduction Assessment

Yes. A source-level reproduction is to configure a provider header such as `x-session-affinity: {{session.key}}`; current main sanitizes and merges that value as a literal string, and the runtime wrapper path does not carry `sessionKey` or `runId` for interpolation.

## Solution Assessment

Yes. A generic resolver at model-request dispatch is the narrowest maintainable path because OpenClaw owns the session/run identity and already centralizes provider header merging there; provider-specific plugin hooks remain useful but do not solve arbitrary user-configured header templates.
