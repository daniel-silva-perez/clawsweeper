## Summary

Keep this issue open. Current main still has static configured provider headers and no config-level placeholder resolver, while the newer provider-plugin turn-state hook is only an adjacent provider-owned seam and does not satisfy arbitrary custom-provider header templates. The prior implementation PRs are closed unmerged, and the latest maintainer discussion scopes the remaining work as a narrow OpenClaw runtime feature.

## What This Changes

Support runtime interpolation in custom LLM provider header config so values such as session, agent, and run identifiers can be sent as provider request headers.

## Best Possible Solution

Land a small documented allowlisted placeholder resolver for configured model-provider headers at request dispatch time, covering models.providers.*.headers and models.providers.*.request.headers while preserving current static-header behavior and existing provider transport policy.
