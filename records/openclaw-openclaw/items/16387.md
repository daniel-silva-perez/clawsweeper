## Summary

Current main still does not implement dynamic templating for custom provider headers. Provider and request headers are sanitized and merged as literal strings, the Fireworks plugin does not add `x-session-affinity`, and the available provider-owned header hooks do not resolve arbitrary user-configured `models.providers.*.headers` or `request.headers` templates.

## Best Possible Solution

Keep the issue open and implement a narrow, documented runtime header-template resolver for `models.providers.*.headers` and `models.providers.*.request.headers`. Resolve only explicit placeholders such as `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` at request dispatch, preserve static headers and SecretRef behavior, cover main turns, compaction, retries, and OpenAI-compatible transports, and document Fireworks and LiteLLM/Langfuse examples.
