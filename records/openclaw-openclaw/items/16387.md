## Summary

Keep open. Current main still treats configured model-provider headers as literal strings, the Fireworks plugin does not add session-affinity headers, and the discussion now includes two concrete runtime-header use cases plus closed unmerged implementation attempts.

## What This Changes

Add runtime interpolation for configured model provider headers so placeholders such as `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` can produce per-session or per-run upstream headers.

## Best Possible Solution

Land a narrow documented dispatch-time resolver for configured provider headers covering both provider-level and request-level headers with explicit placeholders, while preserving static headers, SecretRef resolution, auth injection, attribution headers, retries, and provider-owned stream hooks.

## Reproduction Assessment

Yes. A source-level reproduction is to configure a header such as `x-session-affinity: {{session.key}}`; current main sanitizes and merges that value as a literal string, and the runtime wrapper path does not carry `sessionKey` or `runId` for interpolation.

## Solution Assessment

Yes. A generic resolver at model-request dispatch is the narrowest maintainable direction because OpenClaw owns the session/run identity and already centralizes configured header merging there; hardcoding Fireworks or Langfuse behavior in core would be less reusable.
