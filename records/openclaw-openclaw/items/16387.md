## Summary

Keep this issue open. Current main still treats custom model-provider headers as literal strings: provider-level headers and `request.headers` are sanitized, merged, and attached without runtime interpolation; Fireworks does not add `x-session-affinity`; and the existing provider hook surface is provider-owned rather than a resolver for arbitrary user-configured header templates. The issue discussion includes concrete Fireworks and LiteLLM/Langfuse use cases, and the related implementation PRs #16448 and #18213 are closed unmerged.

## Best Possible Solution

Implement a narrow, documented runtime header-template resolver for `models.providers.*.headers` and `models.providers.*.request.headers`. Resolve explicit placeholders such as `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` at request dispatch; preserve static headers, SecretRef handling, auth injection, precedence, retries, and compaction; and document Fireworks plus LiteLLM/Langfuse examples.
