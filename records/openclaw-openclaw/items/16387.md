## Summary

Keep this issue open. Current main still preserves configured custom-provider headers as static strings, and the newer provider-plugin turn-state hook is adjacent but does not provide the requested config-level templating for arbitrary custom providers. The existing issue discussion gives a clear OpenClaw-owned acceptance path, and the earlier implementation PRs are closed unmerged.

## What This Changes

Allow custom LLM provider header config to interpolate runtime session, agent, and run placeholders so providers such as Fireworks.ai or LiteLLM can receive per-session routing headers.

## Best Possible Solution

Land a small generic placeholder resolver for configured model-provider headers at request dispatch time, documented as an explicit allowlist of placeholders and covered across normal runs, compaction, static headers, SecretRef request headers, auth injection, attribution headers, and provider-plugin hooks.
