## Summary

Keep this issue open. Current main still treats configured custom-provider headers as literal strings, while the provider-plugin turn-state hook is only an adjacent plugin seam and does not provide config-level templating for arbitrary custom providers. The issue remains actionable because the discussion scopes a narrow OpenClaw-owned runtime feature, and the prior implementation attempts are closed unmerged.

## What This Changes

Allow custom LLM provider header config to interpolate runtime session, agent, and run identifiers for provider-side routing such as Fireworks prompt caching or LiteLLM/Langfuse aggregation.

## Best Possible Solution

Land a small documented allowlisted placeholder resolver for configured model-provider headers at request dispatch time, preserving static headers, SecretRef request headers, auth injection, attribution headers, caller/default precedence, retries, provider plugin hooks, and compaction behavior.
