## Summary

Keep open. Current main still stores, edits, searches, and renders the cron job prompt/task, but the Control UI cron dashboard job card does not visibly render the saved human-readable description requested by the issue.

## What This Changes

Request that the Web UI cron dashboard reveal a cron job's prompt/task and human-readable summary or description when the user inspects a job.

## Best Possible Solution

Add a visible Summary or Description field to the cron dashboard job card or selected-job detail area for both systemEvent and agentTurn jobs, using the existing CronJob.description value while leaving storage, RPC, payload rendering, and run-history selection semantics intact.
