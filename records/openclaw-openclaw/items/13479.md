## Summary

Keep open. Current main still carries cron descriptions through the schema and edit flow, and it renders the prompt/task payload in the Control UI job card, but the job card/detail renderer does not visibly render the saved description as the requested human-readable summary. This remains a narrow presentation gap, and prior in-thread reviews reached the same conclusion.

## What This Changes

Show a cron job's prompt/task and saved human-readable summary from the Web UI cron dashboard/detail flow.

## Best Possible Solution

Add a visible Summary or Description field to the Control UI cron job card or selected-job detail area for both systemEvent and agentTurn jobs, using the existing CronJob.description value while leaving cron storage, RPC schemas, payload rendering, and run-history selection semantics intact.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: render a CronJob with a non-empty description; current renderJob calls renderJobPayload but neither path reads or renders job.description.

## Solution Assessment

Yes. The narrowest maintainable fix is a Control UI presentation change with focused view tests; the current storage, Gateway protocol, and edit form plumbing already support the field.
