## Summary

Keep open. Current main still has the cron description persisted and editable, and it renders the actual cron payload prompt/system text, but the Control UI cron job card/detail path still does not visibly show `job.description` as the requested human-readable summary.

## What This Changes

The issue requests that clicking or inspecting a cron job in the Web UI reveal both the cron prompt/task and a human-readable summary/description.

## Best Possible Solution

Render the saved cron `description` as a visible summary in the Control UI cron job card or selected-job detail area for both `systemEvent` and `agentTurn` jobs, while leaving the existing cron storage/RPC contract and payload rendering intact.
