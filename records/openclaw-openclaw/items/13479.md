## Summary

Keep open. Current main stores, edits, searches, and renders the cron job prompt/task, but the Control UI cron dashboard still does not visibly render the saved `job.description` as the requested human-readable summary.

## What This Changes

The issue requests that the Web UI cron dashboard expose each cron job's prompt/task and human-readable summary without requiring users to inspect config files.

## Best Possible Solution

Add a visible Summary or Description field to the cron dashboard job card or selected-job detail area for both `systemEvent` and `agentTurn` jobs, using the existing `CronJob.description` value while leaving storage, RPC, payload rendering, and run-history selection semantics intact.

## Repair Work Prompt

Fix the Control UI cron dashboard so users can see each cron job's saved human-readable summary without opening config files. Render `CronJob.description` as a visible Summary or Description field in the cron job card or selected-job detail area for both `systemEvent` and `agentTurn` jobs. Keep the existing storage/RPC contract intact, keep prompt/system payload rendering intact, and preserve the current row-click behavior that selects job-scoped run history. Treat description as user-authored text; use Lit's escaped text rendering unless maintainers intentionally choose the existing sanitized markdown helper. Add focused view coverage that fails before the change for a job with `description`, and update English i18n/generated locale outputs only if a new visible label key is added. Add a user-facing changelog fix entry crediting @vokaplok if repo policy requires one for this UI change. Do not change cron execution semantics, scheduler storage, or gateway protocol schemas.
