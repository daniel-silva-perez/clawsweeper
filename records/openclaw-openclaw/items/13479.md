## Summary

Keep open. Current main still stores, edits, searches, and carries cron job descriptions through the Control UI form, and the job card renders the prompt/task payload, but the visible cron dashboard card/detail path still omits the saved description as the requested human-readable summary.

## What This Changes

Request to show each cron job's prompt/task and saved human-readable summary in the Control UI cron dashboard instead of requiring config inspection.

## Best Possible Solution

Add a visible Summary or Description field to the Control UI cron job card or selected-job detail area for both systemEvent and agentTurn jobs, using the existing CronJob.description value while leaving storage, RPC, payload rendering, and run-history selection semantics intact.

## Reproduction Assessment

Yes. A source-level reproduction is clear: render a CronJob with a non-empty description and the current renderJob path will show name, schedule, payload, chips, and actions, but not job.description.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation change with focused view tests; no core cron schema, storage, or Gateway RPC change appears necessary.
