## Summary

Keep open. Current main still stores and edits cron descriptions and renders the actual system-event or agent-turn prompt text, but the cron dashboard job card/detail renderer does not visibly display the saved description as the requested human-readable summary.

## What This Changes

The issue requests that clicking or viewing a cron job in the Web UI reveal both the job prompt/task and a human-readable summary/description so users do not need to inspect config files.

## Best Possible Solution

Add a visible Summary or Description field in the Control UI cron job card or selected-job detail area for both systemEvent and agentTurn jobs, backed by focused view/controller tests, while keeping the existing cron storage/RPC contract, payload display, and run-history selection behavior intact.
