## Summary

Keep open. Current main shows the cron job payload/prompt in the dashboard, but the saved human-readable description/summary is still not rendered in the cron job card or selected-job detail path; it only exists in the storage/API/edit/search surfaces and in an adjacent agent status panel.

## What This Changes

Request that the Web UI cron dashboard reveal each cron job's prompt/task and human-readable summary/description without requiring users to inspect config files.

## Best Possible Solution

Add a visible Summary or Description field to the cron dashboard job card or selected-job detail area for both systemEvent and agentTurn jobs, using the existing CronJob.description value while leaving cron storage, RPC, payload rendering, and run-history selection semantics intact.
