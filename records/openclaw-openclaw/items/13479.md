## Summary

Current main still does not satisfy the requested Cron dashboard detail view. The Cron job cards render system-event text or agent-turn prompt/delivery, and clicking a job still loads run history; `job.description` is persisted and editable but is not shown on the Cron dashboard as a summary/detail.

## Best Possible Solution

Implement a narrow Control UI cron presentation fix: show saved `job.description` as a job summary/detail for Cron dashboard job cards or the selected/expanded job detail area, preserve existing prompt/system payload rendering and History/run filtering, add focused tests for both `systemEvent` and `agentTurn` jobs, and update i18n/changelog only as required by new visible copy.
