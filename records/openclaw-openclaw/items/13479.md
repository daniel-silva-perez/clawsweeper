## Summary

Keep open. Current main at 66cdbccc still persists, edits, and searches cron job descriptions, and it renders prompt/system payload text, but the Control UI cron job card/detail renderer still does not display the saved description as the requested human-readable summary.

## What This Changes

The issue asks for the Web UI cron dashboard inspection path to show each cron job's prompt/task and human-readable summary.

## Best Possible Solution

The Control UI should visibly render saved job.description for both systemEvent and agentTurn cron jobs in the job card or selected-job detail area, while keeping existing payload rendering and run-history selection behavior intact.
