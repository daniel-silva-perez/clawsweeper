## Summary

Keep open. Current main still exposes and edits cron job descriptions, and the job card renders prompt/system payload text, but the Control UI job card/detail path does not visibly render the saved description as the requested human-readable summary.

## What This Changes

Show a cron job's prompt/task and human-readable summary/description when users click or inspect a job in the Web UI.

## Best Possible Solution

Render saved job.description as a visible summary in the Control UI cron job card or selected-job detail area for both systemEvent and agentTurn jobs, while leaving the existing cron storage/RPC contract and payload rendering intact.
