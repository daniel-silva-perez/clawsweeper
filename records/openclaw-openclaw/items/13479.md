## Summary

Keep open. Current main still has the cron description stored, edited, searched, and carried through the Control UI form, and it renders the cron prompt/task payload on the job card, but it does not visibly render the saved description as the requested human-readable summary.

## What This Changes

The issue asks for the Web UI cron dashboard to reveal each cron job's prompt/task and human-readable summary without requiring users to inspect config files.

## Best Possible Solution

Add a visible Summary or Description field to the Control UI cron job card or selected-job detail area for both systemEvent and agentTurn jobs, using the existing CronJob.description value while leaving storage, RPC, payload rendering, and run-history selection semantics intact.
