## Summary

Keep open. Current main still has the cron description contract, edit form, and job search support, and it already renders the prompt/task payload on the cron job card, but the visible Control UI job card/detail path still does not render the saved `job.description` as the requested human-readable summary.

## What This Changes

Request to show each cron job's prompt/task and human-readable summary or description in the Web UI cron dashboard without requiring users to inspect config files.

## Best Possible Solution

Add a visible Summary or Description field to the Control UI cron job card or selected-job detail area for both `systemEvent` and `agentTurn` jobs, using the existing `CronJob.description` value while leaving storage, RPC, payload rendering, and run-history selection semantics intact.

## Reproduction Assessment

Yes. A source-level reproduction is clear: render a `CronJob` with a non-empty `description`; current `renderJob` displays name, schedule, payload, chips, and actions, but never reads or renders `job.description`.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation change with focused view tests; no core cron schema, storage, or Gateway RPC change appears necessary.
