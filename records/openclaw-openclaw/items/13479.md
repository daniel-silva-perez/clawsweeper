## Summary

Keep open. Current main still supports storing, editing, searching, and returning cron job descriptions, and the job card already renders prompt/task payloads, but the visible Control UI cron job card/detail path still does not render `job.description` as the requested human-readable summary.

## What This Changes

Show a cron job's prompt/task and human-readable summary in the Web UI when inspecting a job from the cron dashboard.

## Best Possible Solution

Add a visible Summary or Description field to the Control UI cron job card or selected-job detail area for both `systemEvent` and `agentTurn` jobs, using the existing `CronJob.description` value while leaving storage, RPC, payload rendering, and run-history selection semantics intact.

## Reproduction Assessment

Yes. A source-level reproduction is clear: render a `CronJob` with a non-empty `description`; current `renderJob` displays name, schedule, payload, chips, and actions, but never reads or renders `job.description`.

## Solution Assessment

Yes. The narrowest maintainable fix is a Control UI presentation change with focused view tests; no core cron schema, storage, or Gateway RPC change appears necessary.
