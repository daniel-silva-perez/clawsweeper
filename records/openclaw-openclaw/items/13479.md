## Summary

Keep this issue open. Current main still shows the cron prompt/system payload inline and persists an optional job description, but clicking a cron job is still wired to run-history selection and the cron job card/detail area does not render `job.description` as the requested human-readable summary.

## Best Possible Solution

Implement this in the Control UI cron surface: keep the existing prompt/system payload visibility, add an explicit selected/expanded job detail or inline summary area that renders `job.description` when present, preserve the History control/run filtering behavior, add English i18n copy, and cover both system-event and agent-turn jobs in focused cron view tests.
