## Summary

Current main still does not implement this PR's intended behavior: `sessions.list` applies `activeMinutes` as a plain `updatedAt` cutoff, and the chat UI still uses that active-session path. The PR remains a focused candidate for the linked open regression, with the custom `mainKey` review concern addressed in its second commit. Security review found no supply-chain, CI, dependency, release, or secret-handling changes in the PR diff.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, adapt the helper to the current entry-filtering pipeline, preserve the custom `mainKey` guard from the second commit, run focused gateway validation, and land it if maintainers accept the primary-session visibility behavior; that should resolve #43737.
