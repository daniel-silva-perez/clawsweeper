## Summary

Keep this PR open. It targets a valid gateway/UI regression and contains useful regression coverage, but the branch is stale against current main: the active-session cutoff now happens on `entries` before row construction, while the PR still exempts rows at the older `sessions` filter.

## What This Changes

The PR adds a primary-agent-session helper, exempts primary agent main sessions from `activeMinutes` filtering in `listSessionsFromStore`, and adds default/custom `mainKey` regression tests.

## Best Possible Solution

Land a rebased gateway fix that applies the primary agent main-session exemption during the current `entries` active cutoff in `listSessionsFromStore`, before limit and row construction, while preserving the custom `mainKey` guard and focused regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is clear: with stale `agent:main:main`, fresh `agent:heartbeat:main`, and `sessions.list` called with `activeMinutes`, current main drops the stale main entry at `src/gateway/session-utils.ts:1701-1703` before the dropdown receives rows.

## Solution Assessment

No. The intended approach is narrow and maintainable, but this PR applies it to the old row-level filter; the safer fix is to move the same exemption to current main's pre-row `entries` filter.
