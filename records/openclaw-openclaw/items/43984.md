## Summary

Keep open. Current main still has the active-session cutoff that can drop a stale primary main session, and this contributor PR is the linked implementation attempt for #43737, but the branch is stale against the current `listSessionsFromStore` shape and needs maintainer-directed rebase or replacement before merge.

## What This Changes

The PR adds a primary-agent-session helper, exempts primary agent main sessions from `activeMinutes` filtering, and adds default/custom `mainKey` regression tests.

## Best Possible Solution

Land a rebased gateway fix that applies the primary main-session exemption during the current `entries` active cutoff in `listSessionsFromStore`, before limit and row construction, while preserving the custom `mainKey` guard and focused regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is clear: with stale `agent:main:main`, fresh `agent:heartbeat:main`, and `sessions.list` called with `activeMinutes`, current main drops the stale main entry at `src/gateway/session-utils.ts:1701` before the dropdown receives rows.

## Solution Assessment

No. The intended behavior is narrow and maintainable, but this PR applies it to the old row-level filter; the safer fix is to move the same exemption to current main's pre-row `entries` filter.
