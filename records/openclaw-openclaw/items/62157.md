## Summary

Keep this PR open. Current main still suppresses ordinary non-error text finals after a successful block stream, so the central bug is not implemented on main. The PR targets the right shared code path, but the submitted patch still needs revision for coalesced streamed chunks before it is safely mergeable.

## Best Possible Solution

Keep the PR open, but revise it before merge: preserve genuinely new post-tool final payloads after successful block streaming while retaining suppression for exact, coalesced, and directly sent block content. Update `agent-runner-payloads` tests and add a coalescing regression so the fix does not re-send already streamed chunks. The Feishu workaround in #38422 does not supersede this shared core payload-filtering gap.
