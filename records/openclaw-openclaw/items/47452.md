## Summary

Current main still does not implement the reported Kimi/Qiniu behavior for a custom `anthropic-messages` provider. The unowned Anthropic replay fallback only drops thinking blocks for Claude model IDs, so `qiniu/moonshotai/kimi-k2.5` can still replay `thinking` blocks. The related fix PR #47459 was closed unmerged, and current bundled Kimi/Moonshot paths do not cover this custom Qiniu Anthropic-compatible setup.

## What This Changes

The issue asks OpenClaw to stop replaying Anthropic `thinking` content blocks to Kimi K2.5 through a Qiniu `anthropic-messages` custom provider when the model is configured without reasoning support.

## Best Possible Solution

Keep this open for maintainer boundary review. The best fix is either a Qiniu/MoonshotAI-compatible provider plugin with a `buildReplayPolicy` that drops unsupported Kimi thinking blocks, or a documented generic custom-provider replay compatibility flag for `anthropic-messages` models. Either path should add regression coverage proving `moonshotai/kimi-k2.5` does not replay `thinking` blocks when configured as unsupported.
