## Summary

Keep open. Current main still does not implement the reported Qiniu Anthropic-compatible Kimi behavior: unowned Anthropic replay drops `thinking` blocks only for Claude model IDs, the configured `supportsReasoningEffort` flag is not an Anthropic replay compatibility switch, and the related fix PR was closed unmerged.

## What This Changes

The issue asks OpenClaw to stop sending Anthropic `thinking` content blocks to a custom Qiniu Moonshot/Kimi route when that configured model does not support reasoning blocks.

## Best Possible Solution

Add a documented provider-owned or generic replay-policy mechanism for Anthropic-compatible custom routes to declare unsupported thinking blocks, then cover `qiniu/moonshotai/kimi-k2.5` with regression tests proving historical `thinking` blocks are dropped or converted before replay.
