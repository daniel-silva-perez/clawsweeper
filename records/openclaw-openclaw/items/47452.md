## Summary

Keep open. Current main still preserves `thinking` blocks for the reported custom `qiniu` Anthropic Messages route: the unowned Anthropic replay fallback only enables `dropThinkingBlocks` for Claude model IDs, and the reported `supportsReasoningEffort` flag is not an Anthropic replay compatibility knob. The linked fix PR was closed unmerged, so there is no implemented or superseding fix to close against.

## What This Changes

The issue asks OpenClaw to stop sending Anthropic `thinking` content blocks to `qiniu/moonshotai/kimi-k2.5` when the configured model disables reasoning or does not support reasoning effort.

## Best Possible Solution

Keep this open until a maintainer chooses and implements a provider-owned or documented generic replay policy for Anthropic-compatible routes that reject `thinking` blocks, with regression coverage proving `qiniu/moonshotai/kimi-k2.5` drops or converts historical thinking blocks before replay while preserving Claude signed-thinking behavior.
