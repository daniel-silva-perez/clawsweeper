## Summary

Keep open. The checked-out main tree still preserves historical `thinking` blocks for the reported custom `qiniu` provider using `api: "anthropic-messages"`; the generic Anthropic fallback only drops them for Claude model ids, and the earlier targeted PR was closed unmerged.

## What This Changes

The issue requests that Anthropic Messages routes for Kimi-compatible providers stop replaying `thinking` content blocks when the configured model/provider does not support them.

## Best Possible Solution

Keep this open until a maintainer chooses a documented replay-policy surface for Anthropic-compatible routes that reject `thinking` blocks, then add regression coverage proving `qiniu/moonshotai/kimi-k2.5` drops or converts historical thinking blocks without regressing Claude signed-thinking replay.
