## Summary

Keep open. Current main still preserves historical `thinking` blocks for the reported custom Qiniu Anthropic Messages route because the unowned Anthropic fallback only enables `dropThinkingBlocks` for Claude model ids, and the related fix PR was closed unmerged.

## What This Changes

The issue asks OpenClaw to stop sending Anthropic `thinking` content blocks to a custom Qiniu/Moonshot Kimi model route when the configured model says reasoning is unsupported.

## Best Possible Solution

Add a documented replay-policy surface for Anthropic-compatible custom providers or a targeted provider-owned Qiniu/Moonshot path, with regression coverage proving Kimi drops or converts replayed `thinking` blocks while Claude signed-thinking replay remains preserved.
