## Summary

Keep open. Current main still does not route the reported custom `qiniu` Anthropic Messages Kimi model through a policy that drops historical `thinking` blocks, and the reported `supportsReasoningEffort: false` setting is not an Anthropic replay-policy control. The linked fix PR was closed unmerged, so there is no implemented or superseding fix to close against.

## What This Changes

The issue requests that Anthropic-compatible Kimi/Qiniu routes stop replaying unsupported `thinking` content blocks when model/provider compatibility settings disable reasoning support.

## Best Possible Solution

Keep this open until a maintainer chooses and implements a provider-owned or documented generic replay policy for Anthropic-compatible routes that reject `thinking` blocks, with regression coverage proving `qiniu/moonshotai/kimi-k2.5` drops or converts historical thinking blocks before replay while preserving Claude signed-thinking behavior.
