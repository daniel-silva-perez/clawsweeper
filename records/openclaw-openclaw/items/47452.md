## Summary

Current main still does not appear to fix the reported qiniu/custom Anthropic-compatible Kimi failure. The linked fix PR was closed unmerged, the current transcript policy only drops Anthropic thinking blocks for Claude-model fallback cases, and the Anthropic transport still emits signed same-model `thinking` content blocks.

## Best Possible Solution

Keep this issue open and add a focused replay-policy fix for Anthropic-compatible Kimi/custom-provider routes: strip or convert persisted `thinking`/`redacted_thinking` blocks when the target model or config declares that thinking content is unsupported, then cover `qiniu/moonshotai/kimi-k2.5` with a transcript-policy or transport conversion regression test.
