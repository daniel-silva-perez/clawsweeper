## Summary

Keep open. Current main still does not drop historical `thinking` blocks for the reported custom `qiniu` provider using `api: "anthropic-messages"`; the unowned Anthropic fallback only enables that behavior for Claude model ids, and the reported `supportsReasoningEffort: false` field is not an Anthropic replay policy control. The related fix PR #47459 was closed without merge, so there is no implemented or superseding fix to close against.

## What This Changes

The issue requests that OpenClaw avoid replaying Anthropic `thinking` content blocks for a custom Qiniu-hosted Kimi K2.5 model when the model is configured as non-reasoning or unsupported for reasoning effort.

## Best Possible Solution

Keep this open until a maintainer chooses and implements a documented replay policy for Anthropic-compatible routes that reject `thinking` blocks, with regression coverage proving `qiniu/moonshotai/kimi-k2.5` drops or converts historical thinking blocks while Claude signed-thinking replay remains preserved.
