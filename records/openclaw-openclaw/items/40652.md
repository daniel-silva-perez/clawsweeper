## Summary

Keep open. Current `main` still only partially satisfies the requested recovery guidance: direct Gateway startup suggests `openclaw doctor --fix` but omits the `openclaw.json.bak` fallback, while daemon start/restart preflight still uses generic `openclaw doctor`; the related implementation PRs in the provided context are closed and unmerged.

## What This Changes

Gateway invalid-config startup errors should tell operators to run `openclaw doctor --fix` and fall back to restoring `openclaw.json.bak` if repair does not unblock startup.

## Best Possible Solution

Use one shared, accurate invalid-config recovery hint for direct Gateway startup validation and daemon start/restart preflight, with tests covering both surfaces and no change to the existing recovery policy.
