## Summary

Current main still only partially addresses the requested recovery guidance. Direct gateway startup now suggests `openclaw doctor --fix`, but the thrown invalid-config error still omits an explicit `openclaw.json.bak` fallback, and daemon start/restart preflight still points at generic `openclaw doctor`. The related implementation PRs in the provided context are closed and unmerged, so there is no canonical merged fix to close against.

## Best Possible Solution

Keep this issue open and make a narrow user-facing recovery-message fix. Add a shared gateway invalid-config recovery hint used by direct startup validation and daemon start/restart preflight. The hint should mention `openclaw doctor --fix` and an explicit backup fallback such as `openclaw.json.bak` where appropriate, without obscuring current last-known-good and `.clobbered.*` behavior. Extend `src/gateway/server-startup-config.recovery.test.ts` and `src/cli/daemon-cli/lifecycle-core.config-guard.test.ts` to assert both recovery steps, and add a concise changelog entry because the text is operator-facing.
