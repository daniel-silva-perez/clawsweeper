## Summary

Keep open. Current main still only partially satisfies the requested recovery guidance: direct Gateway startup suggests `openclaw doctor --fix` but omits the `openclaw.json.bak` fallback, while daemon start/restart preflight still uses generic `openclaw doctor`; the related implementation PRs are closed and unmerged, and no open replacement PR was found.

## What This Changes

Gateway invalid-config startup errors should tell operators to run `openclaw doctor --fix` and fall back to restoring `openclaw.json.bak` if repair does not unblock startup.

## Best Possible Solution

Add one shared, accurate invalid-config recovery hint and use it for both direct Gateway startup validation and daemon start/restart preflight, preserving existing automatic last-known-good recovery while making the manual repair and backup rollback path clear and test-covered.
