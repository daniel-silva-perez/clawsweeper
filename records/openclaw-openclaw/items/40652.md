## Summary

Keep open. Current main only partially satisfies the requested recovery guidance: direct gateway startup now suggests `openclaw doctor --fix`, but it still omits the `openclaw.json.bak` fallback, while daemon start/restart preflight still says generic `openclaw doctor`. The prior implementation PRs are closed and unmerged, and no open replacement PR was found.

## What This Changes

The issue asks gateway invalid-config startup errors to suggest `openclaw doctor --fix` and a fallback restore from `openclaw.json.bak`.

## Best Possible Solution

Add one shared invalid-config recovery hint and use it for both gateway startup validation and daemon start/restart preflight, preserving existing automatic last-known-good recovery while making the manual repair and backup rollback path clear.
