## Summary

Keep open. Current main still only partially satisfies the requested recovery guidance: direct Gateway startup suggests `openclaw doctor --fix` but omits the `openclaw.json.bak` fallback, while daemon start/restart preflight still uses generic `openclaw doctor` wording; the related implementation PRs are closed and unmerged.

## What This Changes

The issue requests that Gateway invalid-config startup errors suggest `openclaw doctor --fix` and an `openclaw.json.bak` recovery fallback.

## Best Possible Solution

Centralize the invalid-config recovery hint used by direct Gateway startup validation and daemon lifecycle preflight, include `openclaw doctor --fix` plus a cautious `openclaw.json.bak` restore fallback, and cover both surfaces with focused tests without changing automatic rollback policy.

## Solution Assessment

Yes. The narrowest maintainable fix is a shared message/helper update with targeted assertions for direct startup and daemon preflight; this should stay a UX/error-text fix rather than expanding automatic config recovery behavior.
