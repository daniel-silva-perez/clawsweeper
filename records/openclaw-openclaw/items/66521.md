## Summary

Current main still does not implement this PR's intended Signal-local early group typing cue before dispatch. The latest PR head remains useful, but it should be kept open because the added pre-dispatch `onReplyStart` side effect would run before current main resolves `sendPolicy` and typing suppression.

## Best Possible Solution

Keep this PR open and rework it into a policy-aware Signal implementation. The early group typing cue should share the same `sendPolicy`, `suppressTyping`, and `typingMode` decision that dispatch already uses, then add targeted Signal tests for allowed group typing and denied or disabled sessions emitting no typing signal.
