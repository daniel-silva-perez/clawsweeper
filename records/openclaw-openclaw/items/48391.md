## Summary

Keep this PR open. Current main still lacks the PR's intended cron read-model fields and richer `cron.run` acknowledgment, so the PR is not obsolete or implemented on main.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is to land a reviewed variant that adds a separate cron read-state/read-job type, projects `list()` and `listPage()` through that read model, keeps derived fields out of writable cron update state, updates the gateway schema/tests, and either documents or renames the immediate `cron.run` in-progress acknowledgment so clients can distinguish queued acceptance from persisted running state.
