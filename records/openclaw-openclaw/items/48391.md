## Summary

Keep this PR open. Current main still does not include the cron read-model fields or richer cron.run acknowledgment proposed here, and the PR is not superseded by a shipped implementation. The remaining review issue is the queued-versus-persisted meaning of reporting a manual run as running before runningAtMs is written.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is to land a reviewed variant that adds separate cron read-state/read-job types, projects both list() and listPage() through that read model, keeps derived fields out of writable cron update state, updates gateway/client/docs/tests, and resolves or documents the queued-versus-persisted semantics of the cron.run in-progress acknowledgment.
