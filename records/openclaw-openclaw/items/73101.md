## Summary

Keep open. Current main still lacks Slack configured ACP binding support, and this PR is useful implementation work; the runtime approach matches the existing plugin-owned configured-binding pattern, but it needs the small docs/changelog follow-ups and green merge validation before landing.

## What This Changes

The PR adds Slack configured ACP binding normalization, configured-route fallback, ACP target readiness handling, stable configured session keys, regression tests, and two unrelated extension test lint cleanups.

## Best Possible Solution

Land the Slack-owned runtime fix through the existing configured-binding SDK seam, keep runtime bindings higher precedence than configured bindings, drop unavailable configured ACP targets instead of falling back to `main`, and add the missing release-note/docs coverage before merge.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: current main has no Slack `bindings` provider, the generic compiler skips channels without that provider, and Slack inbound routing never calls the configured-binding resolver.

## Solution Assessment

Yes for the runtime design: the PR keeps Slack-specific ID normalization and routing in the Slack plugin while reusing the generic configured-binding registry and readiness helper. It is incomplete as a merge-ready PR until changelog/docs and CI are addressed.
