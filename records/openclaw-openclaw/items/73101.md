## Summary

Keep this PR open. Current main still lacks Slack's configured ACP binding adapter and inbound configured-binding route/readiness path, while the PR is a focused Slack-owned implementation candidate. The supplied PR diff is limited to Slack source/tests, and the bot-review style finding appears addressed by the latest PR commit.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to land a focused Slack-owned fix that registers Slack as a configured ACP binding provider, routes inbound Slack messages through `resolveConfiguredBindingRoute` only when no runtime binding is active, gates dispatch with `ensureConfiguredBindingRouteReady`, and keeps regression coverage for stable configured ACP session keys across Slack channel/thread cases.
