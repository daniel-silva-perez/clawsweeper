## Summary

Keep this PR open. Current main still does not implement Slack configured ACP bindings: Slack has runtime conversation binding support, but no configured-binding adapter, no configured-binding fallback in inbound routing, and no configured ACP readiness/drop handling. The PR remains a focused implementation candidate, and the supplied diff surface does not show CI, dependency, lockfile, release, package-resolution, or other supply-chain-sensitive changes.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a Slack-owned fix that registers Slack as a configured ACP binding provider, routes Slack inbound messages through configured ACP bindings only when no runtime conversation binding is active, gates dispatch with `ensureConfiguredBindingRouteReady`, drops unavailable configured ACP targets instead of falling back to `main`, and keeps regression coverage for stable configured ACP session keys across Slack channel/thread cases.
