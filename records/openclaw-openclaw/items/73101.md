## Summary

Keep this PR open. Current main still lacks Slack's configured ACP binding adapter, configured-binding inbound fallback, configured ACP readiness/drop handling, and regression coverage for unsuffixed configured ACP session keys. The supplied diff is a plausible Slack-owned implementation candidate, with only unrelated test-only lint cleanup outside Slack and no observed CI, dependency, lockfile, release, or package metadata changes.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a focused Slack-owned fix that registers Slack as a configured ACP binding provider, resolves configured ACP routes only when no runtime conversation binding is active, gates configured ACP targets with `ensureConfiguredBindingRouteReady`, drops unavailable targets instead of falling back to `main`, and keeps regression coverage for stable configured ACP session keys across Slack channel/thread cases.
