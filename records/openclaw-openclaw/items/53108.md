## Summary

Keep open. Current main does not implement `capAdd`; the existing sandbox Docker surface only wires adjacent `capDrop` and `gpus` behavior. The proposal is coherent, but it changes Docker sandbox capability grants and the affected sandbox code/docs are routed to secops, so this needs maintainer/security review rather than conservative cleanup closure.

## What This Changes

The PR adds `sandbox.docker.capAdd` across config types, Zod/generated schemas, config resolution, Docker create args, docs, generated config artifacts, and focused tests, with one unrelated browser test formatting change.

## Best Possible Solution

A merged solution should expose `capAdd` as an explicit opt-in Docker sandbox config with default `[]`, preserve `capDrop: ["ALL"]` and `no-new-privileges`, carry the field through public types, validation, resolved config, config hash behavior, Docker args, generated tracked config artifacts, docs, tests, and changelog, and keep unrelated formatting out of the feature diff.
