## Summary

Current main at 1b25dcf57a9fca5d00cf275046e20a2e3ea3730c still does not expose WhatsApp message delete/revoke. The shared action name exists and the pinned Baileys source supports delete payloads, but the bundled WhatsApp plugin only advertises and routes react/poll behavior on main. Related #38723 is already closed as a duplicate, while open PR #70813 is a plausible implementation candidate that should be reviewed rather than duplicating the work.

## Best Possible Solution

Keep this open as the canonical bundled WhatsApp plugin request until #70813 or a similar focused PR lands. The best fix belongs in the WhatsApp plugin action path: advertise and accept the agreed public action name, route through the gateway-owned WhatsApp runtime, call Baileys with an own-message delete key, enforce fromMe/target authorization and group key handling, and update focused WhatsApp action/send tests plus user-facing docs/changelog.
