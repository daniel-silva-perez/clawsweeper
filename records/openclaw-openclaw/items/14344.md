## Summary

Keep open. Current main still does not expose WhatsApp message delete/revoke: discovery advertises only react/poll, the WhatsApp action adapter only supports react, the runtime rejects non-react actions, and the active listener API has no delete/unsend helper. Open PR #70813 remains the focused implementation candidate, so this issue should not be closed before that PR or an equivalent maintainer-reviewed change lands.

## What This Changes

Add a WhatsApp message delete/revoke action so agents can retract their own previously sent WhatsApp messages through the shared message tool.

## Best Possible Solution

Land #70813 or an equivalent focused WhatsApp plugin change that advertises the agreed public action, routes it through the gateway listener/runtime, builds Baileys own-message delete keys, enforces target and fromMe constraints, handles group participants correctly, and updates focused tests, docs, and changelog.
