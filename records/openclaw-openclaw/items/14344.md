## Summary

Keep open. Current main still does not expose WhatsApp delete/revoke for the message tool: WhatsApp discovery advertises only react/poll, the adapter only supports react, the runtime rejects non-react actions, and the active listener API has no delete/unsend helper. Open PR #70813 remains the focused implementation candidate, so this issue should stay open until that PR or an equivalent maintainer-reviewed WhatsApp plugin change lands.

## What This Changes

Add a WhatsApp message delete/revoke action so agents can retract their own sent WhatsApp messages through the message tool.

## Best Possible Solution

Keep this as the canonical bundled WhatsApp plugin request until #70813 or an equivalent focused PR lands. The end state should be a WhatsApp-owned action implementation that advertises the agreed public action, routes through the gateway listener/runtime, builds the Baileys own-message delete key, enforces target authorization and fromMe constraints, handles group participants correctly, and updates focused tests, docs, and changelog.
