## Summary

Keep open. Current main still does not expose WhatsApp message delete/revoke: discovery only advertises react/poll, the channel adapter only supports react, the runtime rejects non-react actions, and open PR #70813 remains the paired unmerged implementation candidate for this request.

## What This Changes

Add a WhatsApp message-tool delete/revoke action so agents can retract their own sent WhatsApp messages.

## Best Possible Solution

A focused WhatsApp plugin change should advertise the agreed delete/unsend action, route it through the gateway listener/runtime, build Baileys own-message delete keys, enforce authorization and fromMe constraints, handle group messages correctly, and update focused tests, docs, and changelog.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: WhatsApp delete is not advertised, `supportsAction` rejects non-react actions, and the runtime throws `Unsupported WhatsApp action` if a non-react action reaches it.

## Solution Assessment

Yes, the narrow WhatsApp-plugin implementation path is the right direction rather than a core change. PR #70813 appears aligned with that path, but it remains open and needs maintainer review for naming and mutation-safety choices.
