## Summary

Keep open. Current main still does not expose WhatsApp message delete/revoke: discovery advertises only WhatsApp react/poll, the channel action adapter supports only react, the runtime rejects every non-react WhatsApp action, and the active listener/send API has no delete helper. The supplied related context shows #70813 is still open and unmerged as the focused implementation candidate, while #38723 was closed as a duplicate of this request.

## What This Changes

Add a WhatsApp message delete/revoke action so agents can retract their own previously sent WhatsApp messages through the shared message tool.

## Best Possible Solution

Keep this as the canonical bundled WhatsApp plugin request until #70813 or an equivalent focused PR lands. The end state should be a WhatsApp-owned action implementation that advertises the agreed public action, routes through the gateway listener/runtime, builds the Baileys own-message delete key, enforces target authorization and fromMe constraints, handles group participants correctly, and updates focused tests, docs, and changelog.
