## Summary

Keep open. Current main still does not expose WhatsApp message delete/revoke: WhatsApp discovery advertises only react/poll, the adapter only supports react, and the runtime rejects non-react actions; open PR #70813 is the paired unmerged implementation candidate with closing syntax.

## What This Changes

Request to add a WhatsApp message delete/revoke action so agents can retract their own sent messages through the message tool.

## Best Possible Solution

A focused WhatsApp plugin change should advertise the agreed public delete/unsend action, route it through the gateway listener/runtime, build Baileys own-message delete keys, enforce authorization and fromMe constraints, handle group messages correctly, and update tests, docs, and changelog before this closes.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a WhatsApp `delete` message action is not advertised, `supportsAction` rejects non-react actions, and the runtime throws `Unsupported WhatsApp action` if a non-react action reaches it.

## Solution Assessment

Yes, the narrow WhatsApp-plugin implementation path is the best way to solve this. PR #70813 appears aligned with that path, but it remains open and needs maintainer review for naming and mutation-safety choices before this issue can close.
