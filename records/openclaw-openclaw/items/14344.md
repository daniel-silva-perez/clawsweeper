## Summary

Keep open. Current main still does not expose WhatsApp message delete/revoke: WhatsApp action discovery advertises only react/poll, the adapter only supports react, the runtime rejects non-react actions, and the listener send API has no delete/unsend/revoke helper. Open PR #70813 is paired to this request and remains the focused implementation candidate, so this issue should not be closed before that PR or an equivalent maintainer-reviewed change lands.

## What This Changes

The issue requests a WhatsApp `message` tool delete/revoke action that lets agents retract their own previously sent WhatsApp messages.

## Best Possible Solution

Keep this issue open as the canonical bundled WhatsApp plugin request until #70813 or an equivalent focused PR lands. The end state should be a WhatsApp-owned message action implementation that advertises the agreed delete/unsend action, routes through the gateway listener/runtime, builds the Baileys own-message delete key, enforces authorization and fromMe constraints, handles groups correctly, and updates focused tests, docs, and changelog.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: WhatsApp delete is not advertised, `supportsAction` rejects non-react actions, and the runtime throws `Unsupported WhatsApp action` if a non-react action reaches it.

## Solution Assessment

Yes. The narrow WhatsApp plugin implementation path is the best direction rather than a core change; #70813 appears aligned but remains open and needs maintainer review for action naming and mutation-safety details.
