## Summary

Keep open. Current main at 43da08979035503401a21a06a55e66920411ad67 still does not expose WhatsApp message delete/revoke: discovery advertises react/poll only, the WhatsApp action adapter and runtime handle react only, the active listener/send API has no delete helper, and docs still exclude WhatsApp from shared delete support. Related #38723 was closed as a duplicate of this canonical request, not as an implementation.

## Best Possible Solution

Keep this open as the canonical bundled WhatsApp plugin feature request. The best implementation is to add a WhatsApp `delete` action to discovery and supportsAction, route it through the gateway-owned WhatsApp action runtime, add an active-listener delete/revoke helper using the verified Baileys message key shape, enforce bot-owned/fromMe deletion with group participant handling, and update focused tests plus `docs/cli/message.md`, `docs/channels/whatsapp.md`, and the user-facing changelog.
