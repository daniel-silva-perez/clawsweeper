## Summary

Keep open. Current main still lacks WhatsApp message delete/revoke support, and open PR #70813 is already paired to this issue with closing syntax but has not merged, so this remains the canonical tracked request.

## What This Changes

Add a WhatsApp message-tool action that lets the bot delete or revoke its own previously sent WhatsApp messages.

## Best Possible Solution

Keep this as the canonical bundled WhatsApp plugin request until #70813 or an equivalent focused maintainer-reviewed PR lands. The desired end state is a WhatsApp-owned action implementation that advertises the agreed public action, routes through the gateway listener/runtime, builds Baileys own-message delete keys, enforces authorization and fromMe constraints, handles groups correctly, and updates focused tests, docs, and changelog.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: WhatsApp discovery does not advertise delete, the plugin adapter rejects non-react actions, and the runtime throws for any non-react WhatsApp action.

## Solution Assessment

Yes, the narrow WhatsApp plugin implementation path is the best direction rather than a core change. The paired PR appears aligned in scope, but it remains unmerged and needs maintainer review for naming and mutation-safety details.

Status: needs_attention

Summary: No patch is attached to this issue, but the requested WhatsApp delete/revoke capability is security-sensitive message mutation and needs maintainer safety review before implementation.
