## Summary

Keep open. Current main still does not expose WhatsApp message delete/revoke: WhatsApp discovery advertises react/poll only, the adapter supports only react, the runtime rejects non-react actions, and the active listener/send API has no delete or unsend helper. Open PR #70813 remains the focused implementation candidate, so this should stay open for maintainer review rather than cleanup closure.

## What This Changes

Request to add WhatsApp message delete/revoke support to the message tool so agents can retract their own sent WhatsApp messages.

## Best Possible Solution

Keep this as the canonical bundled WhatsApp plugin request until #70813 or an equivalent focused PR lands. The desired end state is a WhatsApp-owned action implementation that advertises the agreed public action, routes through the gateway listener/runtime, builds the own-message delete key, enforces target authorization and fromMe constraints, handles group participants correctly, and updates focused tests, docs, and changelog.
