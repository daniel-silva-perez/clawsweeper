## Summary

Keep this PR open. Current main still does not expose the five requested active-run steer/abort helpers from `openclaw/plugin-sdk/agent-runtime`, and the existing `agent-harness` aliases only cover part of the requested surface. The PR remains a focused implementation candidate, but it needs maintainer judgment on the SDK boundary and the cross-session `abortEmbeddedPiRun` overload before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to expose a supported active-run control surface for external channel plugins covering the five requested helpers, then decide whether that belongs in `plugin-sdk/agent-runtime`, a narrower active-run subpath, or `api.runtime.agent`. The implementation should also settle the `abortEmbeddedPiRun` overload policy and include the matching SDK docs, API baseline/export checks, and validation proof before merge.

## Risks / Open Questions

- Directly exporting `abortEmbeddedPiRun` exposes the `(undefined, { mode: "all" | "compacting" })` cross-session abort overload to external plugins; maintainers should explicitly accept, document, or wrap that capability.
- Adding these helpers to the broad `agent-runtime` barrel may be less aligned with current SDK boundary guidance than a narrow active-run control subpath or an injected `api.runtime.agent` helper.
- Because this changes a published plugin SDK surface, it should land with matching docs/API-baseline/export coverage and normal SDK validation.
