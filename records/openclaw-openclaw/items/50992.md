## Summary

Keep open. Current main still does not expose the five requested active-run steer/abort helpers from `openclaw/plugin-sdk/agent-runtime`, so this PR is not obsolete or implemented. The linked closed issue #49624 does not supersede the PR because the observable SDK gap remains, while the PR still needs maintainer judgment on the `abortEmbeddedPiRun` overload and the right SDK boundary.

## Best Possible Solution

Review this PR as a still-relevant SDK proposal. The best path is to expose a supported active-run control surface for channel plugins covering the five requested helpers, with an explicit maintainer decision on whether `abortEmbeddedPiRun` is exported whole or wrapped to single-session abort. If maintainers prefer a new narrow subpath over `plugin-sdk/agent-runtime`, ask the contributor to retarget the PR and include SDK docs/API-baseline coverage.

## Risks / Open Questions

- If this PR lands as-is, external plugins could call `abortEmbeddedPiRun(undefined, { mode: "all" | "compacting" })`; that may fit OpenClaw's trusted native-plugin model, but it should be explicitly accepted, documented, or wrapped to a single-session contract.
- Adding the helpers to the broad `agent-runtime` barrel may be less aligned with current SDK boundary guidance than a narrow dedicated subpath, so maintainers should choose the intended public contract before merge.
- Because this changes a published SDK surface, it needs the normal SDK docs/baseline/export checks and build proof before landing; this review stayed read-only and did not run tests or build commands.
