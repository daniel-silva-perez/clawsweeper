## Summary

Keep this PR open. Current main still does not redirect gateway root `/` to the configured Control UI `basePath`: with `basePath: "/openclaw"`, `pathname: "/"` returns `not-control-ui`, then the HTTP server falls through to the generic `Not Found` response. The PR remains a focused implementation candidate, and the prior review note about query-string preservation is addressed in the PR diff.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or reapply the focused gateway routing change so `GET` and `HEAD` requests to `/` redirect to `${basePath}/${search}` when `gateway.controlUi.basePath` is set, while non-read root requests still fall through for plugin or webhook handling and regression tests cover root redirect, query preservation, and write-method fallthrough.
