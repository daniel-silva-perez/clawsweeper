## Summary

Keep this PR open. Current main still does not redirect gateway root `/` to the configured Control UI `basePath`; `GET /` with `basePath: "/openclaw"` still falls through Control UI routing and reaches the generic `Not Found` response. The PR remains a narrow implementation candidate, and the review discussion shows the query-string coverage note was addressed.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or reapply the focused gateway routing change: when `gateway.controlUi.basePath` is non-empty and `pathname === "/"`, redirect read methods to `${basePath}/${search}`, preserve non-read fallthrough for plugin/webhook routes, and keep regression coverage for root redirect, query preservation, and write-method fallthrough.
