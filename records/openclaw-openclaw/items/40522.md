## Summary

Keep this PR open. Current main still falls through `GET /` and `HEAD /` when `gateway.controlUi.basePath` is set, so the user still gets the generic gateway `Not Found` instead of being redirected to the Control UI base path. The PR remains a focused implementation candidate, and the discussion shows the query-string test gap was addressed.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase or reapply the focused routing change so read methods to `/` redirect to `${basePath}/${search}` when `gateway.controlUi.basePath` is set, while non-read root requests still fall through. Keep regression tests for root redirect, query preservation, and write-method fallthrough, then run the targeted gateway routing test.
