## Summary

Keep this PR open. Current main still does not implement the PR’s intended gateway behavior: with `gateway.controlUi.basePath` set, `GET /` is classified as `not-control-ui`, then the HTTP server falls through to the generic `404 Not Found`. The PR remains a focused implementation candidate and its later diff addresses the prior query-string preservation review note.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase or reapply the focused gateway routing change so `GET` and `HEAD` requests to `/` redirect to `${basePath}/${search}` when `gateway.controlUi.basePath` is set, while non-read root requests still fall through for plugin/webhook handling and regression tests cover root redirect, query preservation, and write-method fallthrough.
