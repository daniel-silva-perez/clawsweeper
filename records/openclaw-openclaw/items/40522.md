## Summary

This PR should stay open. Current main still does not redirect gateway root `/` to the configured Control UI `basePath`; with `basePath: "/openclaw"`, `GET /` still falls through Control UI routing and will end as `404 Not Found`. The PR remains a focused implementation candidate, and the provided discussion shows the query-string review note was addressed.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase or reapply the focused gateway routing change on current main: when `gateway.controlUi.basePath` is non-empty and `pathname === "/"`, redirect `GET` and `HEAD` to `${basePath}/${search}`, preserve non-read fallthrough for plugin/webhook routes, and keep regression coverage for root redirect, query preservation, and write-method fallthrough.
