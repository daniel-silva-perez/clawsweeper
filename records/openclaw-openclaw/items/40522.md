## Summary

Keep this PR open. Current main still classifies gateway root `/` as `not-control-ui` when `gateway.controlUi.basePath` is configured, which falls through to the generic `404 Not Found`. The PR remains a focused implementation candidate, and the provided discussion shows the query-string preservation review note was addressed.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase or reapply the focused gateway routing change so `GET` and `HEAD` requests to `/` redirect to `${basePath}/${search}` when `gateway.controlUi.basePath` is set, while non-read root requests still fall through for plugin/webhook handling. Add or keep regression tests for root redirect, query preservation, and write-method fallthrough, then run the targeted gateway routing test.
