## Summary

Keep open: current main still has the reported native PDF security gap, but the PR branch is stale and its strict guarded-fetch wiring would regress proxy-only/provider-proxy deployments, so this needs maintainer revision rather than cleanup close.

## What This Changes

The branch updates native Anthropic and Gemini PDF provider calls in `src/agents/tools/pdf-native-providers.ts` to use the SSRF guard, send Gemini auth in `x-goog-api-key`, and release guarded-fetch resources.

## Best Possible Solution

Revise the native PDF provider calls to use the shared provider guarded HTTP path or an equivalent helper that preserves timeout, provider transport normalization, proxy selection, private-network policy, audit context, and cleanup while moving Gemini credentials out of the URL.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve provider proxy routing in guarded PDF fetches:** `src/agents/tools/pdf-native-providers.ts:69`
  - body: The new `withStrictGuardedFetchMode` calls force guarded fetch into pinned direct-dispatch mode. In HTTP(S)_PROXY-only deployments, native Anthropic/Gemini PDF calls can bypass the provider helper's trusted-env-proxy path and fail while other provider calls still work through the configured proxy, so this should route through the shared provider guarded HTTP helper or preserve its proxy selection semantics.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The security fix is directionally relevant, but the diff changes provider network routing in a way that can bypass operator proxy controls.

Concerns:

- **[medium] Strict guarded fetch bypasses provider proxy routing:** `src/agents/tools/pdf-native-providers.ts:69`
  - body: Using `withStrictGuardedFetchMode` creates a strict guarded fetch path instead of the shared provider HTTP helper that conditionally uses trusted env-proxy mode. That can bypass operator-managed HTTP(S)_PROXY routing and its egress controls for these security-sensitive provider requests.
  - confidence: 0.88
