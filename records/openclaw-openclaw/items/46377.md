## Summary

Keep open. The underlying native PDF security issue is still present on current main, but this PR is not merge-ready because the branch is stale against current timeout and Gemini transport behavior, and its strict guarded-fetch path can bypass the provider proxy routing that newer code preserves.

## What This Changes

The branch modifies `src/agents/tools/pdf-native-providers.ts` to route Anthropic and Gemini native PDF API calls through the SSRF guard, move Gemini API key auth into an `x-goog-api-key` header, and release guarded-fetch dispatchers.

## Best Possible Solution

Land a rebased security fix that keeps the current native PDF timeout and Gemini transport normalization, uses the SSRF guard with provider-compatible proxy mode selection and clear audit context, sends Gemini credentials in `x-goog-api-key`, and adds focused coverage for private-network blocking, header placement, timeout propagation, and transport normalization.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: both native PDF functions still use configurable raw fetch destinations, and Gemini still embeds the API key in the URL. A focused unit test can inject a private `baseUrl` and assert the guarded path blocks before fetch.

## Solution Assessment

No. The security direction is right, and the Gemini header change matches the upstream API contract, but this PR is not the best current patch because it drops current timeout behavior, skips current Gemini transport normalization, and forces strict mode instead of preserving provider-aware proxy routing.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Keep guarded PDF calls proxy-compatible:** `src/agents/tools/pdf-native-providers.ts:69-70`
  - body: Using `withStrictGuardedFetchMode` here forces native PDF calls onto the strict pinned-dispatcher path. Provider HTTP helpers now preserve env-proxy routing when `HTTP_PROXY` or `HTTPS_PROXY` is the only egress path; this patch would make these requests fail or bypass that expected proxy path. Please route through the same provider-aware guarded helper or apply the same mode selection.
  - confidence: 0.86
- **[P2] Preserve the native PDF request timeout:** `src/agents/tools/pdf-native-providers.ts:69-70`
  - body: The guarded fetch options do not pass `timeoutMs` or a signal. Current main bounds both native PDF provider requests at 120 seconds, so rebasing this patch as-is would make hung provider calls unbounded again. Please pass `NATIVE_PDF_PROVIDER_FETCH_TIMEOUT_MS` into the guard and keep the timeout assertions.
  - confidence: 0.88
- **[P2] Preserve Gemini provider transport normalization:** `src/agents/tools/pdf-native-providers.ts:154-157`
  - body: The PR head builds the Gemini URL directly from `params.baseUrl`, but current main first runs the Google transport through `normalizeProviderTransportWithPlugin`. A rebase needs to guard the already-normalized transport URL; otherwise plugin-owned base URL and request-policy behavior will be skipped for native Gemini PDF calls.
  - confidence: 0.82
