## Summary

Keep open. Current main still has the reported native PDF security gap, but this PR is not merge-ready against current main because the patch is based on older provider code and would need to preserve current timeout, Gemini transport normalization, and proxy-routing behavior while adding the SSRF/key-leak fixes.

## What This Changes

The PR wraps Anthropic and Gemini native PDF provider fetches in strict guarded fetch calls, moves the Gemini API key from the request URL to an `x-goog-api-key` header, and releases guarded dispatchers after use.

## Best Possible Solution

Land a rebased or replacement security fix that guards both native PDF provider requests, keeps Gemini credentials out of the URL, preserves the 120s request timeout and Google transport normalization, and adds focused tests for private/internal `baseUrl` blocking, Gemini header auth, timeout propagation, and proxy-compatible guarded egress.

## Reproduction Assessment

Yes. The source-level reproduction is clear: current main calls raw provider `fetch` with user-controlled `baseUrl`, and Gemini includes the key in the query string; a focused unit test can assert private-IP `baseUrl` is blocked and the key is absent from the URL after the fix.

## Solution Assessment

No, not as currently proposed. The security direction is right, but the patch should be rebased and adjusted so the guarded calls preserve current timeout, transport normalization, and proxy-routing contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Preserve env-proxy egress for guarded PDF calls:** `src/agents/tools/pdf-native-providers.ts:67-68`
  - body: Using `withStrictGuardedFetchMode` here makes the provider call ignore ordinary env proxy routing unless `OPENCLAW_PROXY_ACTIVE` is set. Current gateway/CLI bootstrap can route raw `fetch` through `EnvHttpProxyAgent`, so HTTPS_PROXY-only or proxy-only deployments can regress from working provider egress to direct or failed native PDF calls; thread the right proxy-aware guarded mode/dispatcher policy and cover it.
  - confidence: 0.82
- **[P2] Preserve the native PDF request timeout:** `src/agents/tools/pdf-native-providers.ts:66-80`
  - body: The guarded fetch call does not pass `timeoutMs` or a signal. Current main aborts both native PDF provider calls after `NATIVE_PDF_PROVIDER_FETCH_TIMEOUT_MS`; without carrying that into `fetchWithSsrFGuard`, these credentialed PDF calls can run until much longer dispatcher defaults or hang. Preserve the 120s timeout for both providers.
  - confidence: 0.86
- **[P2] Keep Gemini transport normalization:** `src/agents/tools/pdf-native-providers.ts:154-157`
  - body: The Gemini block in this stale branch rebuilds the base URL from `params.baseUrl` only. Current main first runs `normalizeProviderTransportWithPlugin`, so plugin-owned Google transport/base URL normalization would be skipped if this branch replaces that code during rebase. Keep the current transport normalization before adding the guard and header auth.
  - confidence: 0.78
