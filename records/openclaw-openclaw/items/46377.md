## Summary

Keep open. The underlying security report is still valid on current main, but this PR is not merge-ready: the branch is stale against newer native PDF timeout and Gemini transport work, and the strict guarded-fetch shape would regress proxy-only/provider-transport behavior unless it is rebased and adjusted.

## What This Changes

The PR replaces raw native Anthropic/Gemini PDF fetches with SSRF-guarded fetches, moves the Gemini API key from a query parameter to an `x-goog-api-key` header, and releases guarded-fetch dispatchers.

## Best Possible Solution

Land a rebased security fix that keeps current native PDF timeout and Gemini transport normalization, routes both provider calls through the SSRF guard with clear audit context and proxy-compatible mode selection, sends Gemini credentials in `x-goog-api-key`, and adds focused tests for private-network blocking, header placement, timeout propagation, and transport normalization.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: `anthropicAnalyzePdf` and `geminiAnalyzePdf` still call raw `fetch` using configurable base URLs, and Gemini still embeds the API key in the query string. A focused unit test can inject a private `baseUrl` and assert the guarded path blocks before fetch.

## Solution Assessment

No. The intended security direction is right, and the Gemini header change matches the current API contract, but this PR is not the best current patch because it omits main's timeout behavior, bypasses current Gemini transport normalization, and forces strict guarded-fetch mode without preserving proxy-compatible provider routing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep guarded PDF calls proxy-compatible:** `src/agents/tools/pdf-native-providers.ts:69-70`
  - body: Using `withStrictGuardedFetchMode` here forces the native PDF calls onto the strict pinned-dispatcher path. Other provider HTTP helpers now auto-upgrade to trusted env-proxy mode when `HTTP_PROXY`/`HTTPS_PROXY` is the only egress path; this patch would make Anthropic/Gemini PDF requests fail or bypass the expected proxy path in those deployments. Please route through the same provider-aware guarded helper or apply the same guarded mode selection.
  - confidence: 0.86
- **[P2] Preserve the native PDF request timeout:** `src/agents/tools/pdf-native-providers.ts:69-70`
  - body: The guarded fetch options do not pass `timeoutMs` or a signal. Current main now bounds both native PDF provider requests at 120 seconds, so rebasing this patch as-is would make hung provider calls unbounded again. Please pass `NATIVE_PDF_PROVIDER_FETCH_TIMEOUT_MS` into the guard and keep the timeout assertions.
  - confidence: 0.88
- **[P2] Preserve Gemini provider transport normalization:** `src/agents/tools/pdf-native-providers.ts:157`
  - body: The PR head builds the Gemini URL directly from `params.baseUrl`, but current main first runs the Google transport through `normalizeProviderTransportWithPlugin`. A rebase needs to guard the already-normalized transport URL; otherwise plugin-owned base URL and request-policy behavior will be skipped for native Gemini PDF calls.
  - confidence: 0.82
