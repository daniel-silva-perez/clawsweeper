## Summary

Keep this PR open. Current main still lacks the requested `openclaw doctor` warning for missing memory-search `extraPaths`, but the proposed patch should not land as-is because its path check can disagree with the active memory runtime for tilde-prefixed entries.

## What This Changes

The PR adds a `doctor` warning for configured memory-search `extraPaths` that do not exist, with tests for missing, existing, and tilde-prefixed paths.

## Best Possible Solution

Land a small doctor diagnostic that uses the same path resolver as the active memory backend, covers default and per-agent `extraPaths`, and tests existing, missing, absolute, workspace-relative, builtin, QMD, and tilde-prefix cases without changing the documented path contract unless maintainers choose to support it consistently.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Align extraPaths checks with runtime path resolution:** `src/commands/doctor-memory-search.ts:267-270`
  - body: The new helper home-expands tilde-prefixed `extraPaths`, but the builtin memory host resolves every non-absolute extra path against the workspace. With builtin memory, doctor can now report no problem because the home-expanded location exists while runtime still indexes the workspace-relative location and finds nothing. Please share the runtime resolver or branch this check by active backend semantics.
  - confidence: 0.87
