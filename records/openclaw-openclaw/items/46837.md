## Summary

Keep open. Current main still lacks this diagnostic, but the PR head is not merge-ready because its tilde handling can disagree with the builtin memory runtime and hide a stale path. The existing review discussion already identifies this blocker, so this should continue through normal PR review rather than be closed or replaced.

## What This Changes

The PR adds an `openclaw doctor` diagnostic for configured `memorySearch.extraPaths` that do not exist, plus unit coverage for missing, existing, and tilde-prefixed paths.

## Best Possible Solution

Revise the diagnostic to reuse or exactly mirror the active backend path resolver. For builtin memory, check the same workspace-relative normalization used by indexing and memory reads; keep any QMD tilde behavior backend-specific and align tests/docs with the intended path contract.

## Review Findings

- **[P2] Align extraPaths checks with builtin runtime resolution:** `src/commands/doctor-memory-search.ts:271-273`
