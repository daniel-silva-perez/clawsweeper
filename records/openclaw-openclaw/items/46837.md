## Summary

Keep this PR open. Current main still does not implement the requested `openclaw doctor` warning for missing `agents.defaults.memorySearch.extraPaths`, and this PR is a focused implementation candidate. It needs maintainer review or a follow-up patch because its latest `~` handling diverges from current runtime `extraPaths` resolution semantics.

## Best Possible Solution

Keep this PR open or replace it with a small maintainer patch that adds a doctor warning for missing `memorySearch.extraPaths`, reuses or mirrors the runtime path normalization contract, and adds focused tests for default plus per-agent paths, absolute paths, workspace-relative paths, QMD behavior, and the decided `~` semantics.
