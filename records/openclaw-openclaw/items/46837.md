## Summary

Keep open. Current main still lacks the missing-`extraPaths` doctor warning, and the proposed change addresses a real diagnostic gap, but the latest PR patch should not merge as-is because its tilde-path handling can diverge from the builtin memory runtime and produce a false health result.

## What This Changes

The PR adds an `openclaw doctor` memory-search diagnostic for missing configured `extraPaths` and extends the doctor memory-search unit tests, including tilde-prefixed path coverage.

## Best Possible Solution

Land a narrow doctor diagnostic that uses the same resolver as the active memory backend, warns for missing default and per-agent `extraPaths`, and covers existing, missing, absolute, workspace-relative, builtin, QMD, and tilde-prefix cases without changing the documented path contract accidentally.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Align extraPaths checks with runtime path resolution:** `src/commands/doctor-memory-search.ts:267-270`
  - body: The new helper home-expands `~/...` entries before checking existence, but builtin memory resolves every non-absolute `extraPaths` entry relative to the workspace. If `$HOME/notes` exists while `<workspace>/~/notes` does not, doctor reports healthy even though runtime indexes the workspace-relative path and finds nothing. Share the runtime resolver or branch this check by active backend semantics.
  - confidence: 0.87
