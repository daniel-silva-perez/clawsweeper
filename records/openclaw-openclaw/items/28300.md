## Summary

Keep open. Current main implements a narrower Control UI theming model with Claw, Knot, Dash, and one browser-local tweakcn import slot, but it does not provide the requested preset gallery, hue-based generator, gradient controls, grouped editor, or per-variable override/reset studio. The request is coherent and supported by discussion, but it needs maintainer product/design direction rather than conservative cleanup closure.

## What This Changes

Add six curated Control UI preset themes plus a Custom Theme Studio with hue-based color generation, mode controls, gradient accents, per-variable overrides, persistence, and localized UI strings.

## Best Possible Solution

Define the intended Control UI theming model first. If maintainers approve a built-in studio, implement it consistently across the closed theme contract, CSS token families, browser-local persistence, Appearance and Quick Settings UI, generated/i18n copy, public docs, and focused UI/storage regression tests.

## Reproduction Assessment

Not applicable as a feature request. Current-main source/docs/test inspection gives a high-confidence verification path: the UI exposes one tweakcn import slot and the requested preset gallery/studio surfaces are absent.

## Solution Assessment

Unclear until maintainers choose the product direction. The current tweakcn import slot is a maintainable partial answer for custom colors, but it is not the full requested built-in preset and Theme Studio system.
