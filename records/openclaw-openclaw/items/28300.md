## Summary

Keep open. Current main implements a narrower Control UI theme surface: Claw, Knot, Dash, and one browser-local tweakcn import slot. It does not implement the requested preset gallery, hue-based generator, gradient controls, grouped editor, or per-variable override/reset studio, and related preset work remains an open product/design question.

## What This Changes

Request adding a Control UI theme customization system with six curated presets and a Custom Theme Studio for generated and manually overridden theme variables.

## Best Possible Solution

Define the intended Control UI theming model first. If approved, implement it consistently across the closed theme contract, CSS token families, browser-local persistence, Appearance and Quick Settings UI, generated/i18n copy, public docs, and focused regression tests while coordinating the narrower preset discussion in #49788.
