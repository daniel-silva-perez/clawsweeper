## Summary

Keep open. Current main implements a narrower Control UI theme-family model with one browser-local tweakcn import slot; it does not provide the requested preset gallery, hue-based generator, gradient controls, grouped editor, or per-variable override/reset studio. The request is coherent, but broad enough to need maintainer product/design direction, especially with #49788 still open for a related preset decision.

## What This Changes

Add six curated Control UI preset themes and a Custom Theme Studio with hue-based color generation, dark/light controls, gradient accents, grouped color editing, per-variable overrides, persistence, and localized UI strings.

## Best Possible Solution

Define the intended Control UI theming model first, then implement the chosen model consistently across the closed theme contract, CSS token families, browser-local persistence, Appearance and Quick Settings UI, generated/i18n copy, public docs, and focused regression tests.
