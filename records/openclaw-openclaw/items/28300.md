## Summary

Keep open. Current main implements a narrower Control UI theme-family model with one browser-local tweakcn import slot, but it does not implement the requested preset gallery, hue-based generator, gradient controls, grouped editor, or per-variable override/reset studio. The request is coherent and still active, and the adjacent accessible-preset decision remains open, so this needs maintainer product/design direction rather than cleanup closure.

## What This Changes

Add six curated Control UI theme presets plus an in-app Custom Theme Studio with hue-based generation, light/dark mode, gradient accents, grouped color editing, per-variable overrides, persistence, and i18n.

## Best Possible Solution

Define the intended Control UI theming model first, including whether built-in presets and a studio should live beside or replace the current tweakcn import slot. Once approved, implement the chosen model consistently across the closed theme contract, CSS token families, browser-local persistence, Appearance, Quick Settings, generated/i18n copy, public docs, and focused UI regression tests, coordinating preset policy with #49788.
