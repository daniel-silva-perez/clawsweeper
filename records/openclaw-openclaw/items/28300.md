## Summary

Keep open. Current main provides only a narrower Control UI theme-family model with one browser-local tweakcn import slot; it does not provide the requested preset gallery, hue-based generator, gradient controls, grouped editor, or per-variable override/reset studio. The request is coherent, but broad enough to need maintainer product/design direction before implementation.

## What This Changes

Add Control UI preset themes plus a Custom Theme Studio with generated colors, mode and gradient controls, per-variable overrides, persistence, and localized UI strings.

## Best Possible Solution

Define the intended Control UI theming model first, then implement it consistently across the closed theme contract, CSS token families, browser-local persistence, Appearance and quick settings UI, generated/i18n copy, public docs, and focused tests while coordinating preset choices with #49788.
