## Summary

Keep open. Current main only provides the narrower Claw/Knot/Dash theme-family model plus one browser-local tweakcn import slot, while this issue asks for six curated presets and an in-app Custom Theme Studio with hue generation, gradients, grouped editing, and per-variable reset. The discussion is coherent and still active, and the related accessible-preset request remains open, so this needs maintainer product/design direction rather than cleanup closure.

## What This Changes

The issue requests a Control UI theme customization system with six preset themes, a hue-based custom theme studio, gradient controls, local persistence, and per-variable color overrides.

## Best Possible Solution

Define the intended Control UI theming model first, including whether preset themes and a studio belong beside or instead of the current one-slot tweakcn import. Once approved, implement that model consistently across theme types, CSS token families, local storage, Appearance, Quick Settings, i18n/generated copy, public docs, and focused UI tests, coordinating preset policy with #49788.
