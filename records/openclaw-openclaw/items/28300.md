## Summary

Keep open. Current main implements a narrower Control UI theming surface with Claw, Knot, Dash, and one browser-local tweakcn import slot; it does not provide the requested preset gallery, hue-based generator, gradient controls, grouped editor, or per-variable override/reset studio. The issue is coherent and has supporting discussion, but it needs maintainer product/design direction before implementation.

## What This Changes

Add six curated Control UI preset themes plus a Custom Theme Studio with hue-based generation, light/dark controls, gradient accents, per-variable overrides, local persistence, and localized UI strings.

## Best Possible Solution

Define the intended Control UI theming model first. If maintainers approve a built-in studio, implement it consistently across the closed theme contract, CSS token families, browser-local persistence, Appearance and Quick Settings UI, i18n generation, public docs, and focused UI/storage regression tests.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is source/docs/test inspection: the UI exposes one tweakcn import slot and the requested preset gallery/studio surfaces are absent.

## Solution Assessment

Unclear until maintainers choose the product direction. The current tweakcn import slot is a maintainable narrower solution for custom colors, but it is not the full requested built-in preset and Theme Studio system.
