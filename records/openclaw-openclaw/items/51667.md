## Summary

Keep open. Current main has partial modality metadata for audio, but the observable WhatsApp/Telegram voice-note path still runs audio through transcription/media-understanding and injects text into the agent context. The embedded agent and gateway run surfaces only forward native image content to the primary model, so the requested opt-in raw audio forwarding for omni-modal models remains unimplemented. The related open feature PR #48900 is a plausible implementation candidate, not a reason to close the issue before merge.

## Best Possible Solution

Keep this issue open and implement the feature in the shared media/agent path, likely by reviewing or evolving #48900: add an opt-in native-audio config, capability detection for audio-capable primary models, secure audio staging/content-part forwarding, STT fallback, and transcript/hash metadata so WhatsApp/Telegram voice notes can reach omni-modal models without losing auditability.
