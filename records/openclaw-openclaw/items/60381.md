## Summary

Keep open. Current main already implements the evaluate half of the request, including CLI/docs coverage and an evaluateEnabled safety gate, but it does not implement a force parameter for click. The click schema, normalizer, CLI command, and Playwright execution path all omit force, so the requested Playwright actionability-bypass escape hatch remains missing.

## Best Possible Solution

Keep this issue open and narrow the remaining implementation to click force support: add force?: boolean to the browser act schema and BrowserActRequest click type, preserve it in normalizeActRequest and legacy flattened params, thread it through executeSingleAction/clickViaPlaywright, pass it to locator.click and locator.dblclick where supported, define existing-session behavior explicitly, and add focused contract tests plus CLI/docs updates. The evaluate portion can be treated as already implemented, with #55598 as related context.
