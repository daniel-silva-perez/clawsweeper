## Summary

Keep open. Current main implements the evaluate half of the request with routing, execution, CLI/docs coverage, and an evaluateEnabled gate, but the requested Playwright click force option is still missing from the browser act schema, typed request, normalizer, CLI, and Playwright locator click path.

## What This Changes

The issue requests adding a Playwright `force` option to browser clicks and exposing an `evaluate` browser action.

## Best Possible Solution

Keep the issue open and implement only the remaining force-click capability: add the boolean option through the browser act contract, normalization, CLI, docs, and Playwright click/double-click calls, while preserving the existing evaluate implementation and its safety gate.

## Reproduction Assessment

Yes. This is a high-confidence implementation gap by source inspection: a request containing `force` would be outside the schema/type contract, dropped by `normalizeActRequest`, and never passed to `locator.click` or `locator.dblclick`.

## Solution Assessment

Yes. The narrow maintainable fix is to thread `force?: boolean` through the existing click pipeline and document existing-session behavior; replacing it with `evaluate` would not preserve real Playwright click semantics.
