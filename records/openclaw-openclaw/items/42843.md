## Summary

Keep this PR open. Current main still has the WhatsApp reconnect behavior the PR is trying to change: the default retry cap is bounded, retry exhaustion returns a stop decision, and the monitor shuts down/breaks on that stop path. The PR is still useful, but it needs to be refreshed against the current WhatsApp plugin layout instead of the obsolete src/web paths.

## Best Possible Solution

Refresh this PR against the current WhatsApp plugin. The likely maintainer path is to update extensions/whatsapp/src/reconnect.ts, extensions/whatsapp/src/connection-controller.ts, and extensions/whatsapp/src/auto-reply/monitor.ts so retryable disconnects do not permanently kill the monitor, while logged-out, session-conflict, and intentionally auth-unstable states remain explicit. Add focused tests for default unlimited retries, bounded degraded-mode retry, non-retryable closes, and auth-unstable behavior, then align web.reconnect.maxAttempts schema/help/generated metadata.
