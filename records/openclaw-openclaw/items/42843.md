## Summary

Keep this PR open. Current main still has the WhatsApp reconnect default capped at 12 attempts and still stops the plugin monitor when the positive retry cap is reached. The PR’s intent remains useful, but it needs to be refreshed from the old `src/web` layout to the current `extensions/whatsapp/src` plugin implementation.

## Best Possible Solution

Refresh this PR against `extensions/whatsapp/src`: update the plugin reconnect policy and close-decision path so retryable WhatsApp disconnects keep retrying or enter an explicit degraded retry mode, while logged-out/session-conflict/auth-unstable paths remain terminal and visible. Add focused tests for default retry semantics, bounded degraded-mode retries, terminal close statuses, and align `web.reconnect.maxAttempts` schema/help/generated docs.
