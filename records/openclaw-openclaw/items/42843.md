## Summary

Keep this PR open. Current main still does not implement the PR's intended WhatsApp reconnect behavior: the default reconnect cap remains 12 attempts, positive retry-cap exhaustion still returns a stop decision, and the monitor still shuts down and breaks after max attempts. The PR is useful but needs to be refreshed from the old src/web layout into extensions/whatsapp/src and should resolve the noted degraded-mode logging/test contract issues before merge.

## Best Possible Solution

Refresh this PR against extensions/whatsapp/src: change the plugin reconnect policy and close-decision path so retryable WhatsApp disconnects keep retrying or enter explicit degraded retry mode, while terminal logged-out/conflict/auth-unstable paths still stop visibly. Add focused tests for default retry semantics, bounded degraded-mode retries, terminal close statuses, and align web.reconnect.maxAttempts schema/help/generated docs.
