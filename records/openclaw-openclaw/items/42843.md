## Summary

Keep this PR open. The linked production reports and PR body describe a real WhatsApp listener lifecycle failure, and current main still has the bounded retry cap plus stop-on-exhaustion behavior. The PR is not merge-ready because it edits the old `src/web` WhatsApp layout; the current implementation lives under `extensions/whatsapp/src`.

## Best Possible Solution

Refresh this PR against the current WhatsApp plugin. The likely maintainer path is to update `extensions/whatsapp/src/reconnect.ts`, `extensions/whatsapp/src/connection-controller.ts`, and `extensions/whatsapp/src/auto-reply/monitor.ts` so retryable disconnects do not permanently kill the monitor, while terminal auth/session-conflict paths remain explicit. Add focused tests for default unlimited retries, bounded degraded-mode retry, non-retryable closes, and auth-unstable behavior, then align `web.reconnect.maxAttempts` schema/help/generated metadata.
