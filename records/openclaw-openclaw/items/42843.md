## Summary

Keep this PR open. Current main still has the WhatsApp reconnect default capped at 12 attempts and still shuts down/breaks the WhatsApp monitor when that cap is reached. The PR is useful but needs to be refreshed against the current WhatsApp plugin layout under extensions/whatsapp/src instead of the obsolete src/web paths.

## Best Possible Solution

Refresh this PR against extensions/whatsapp/src. The likely path is to update the plugin reconnect policy, connection-controller decision logic, and auto-reply monitor so retryable WhatsApp disconnects keep retrying or enter an explicit degraded retry mode, while logged-out/session-conflict/auth-unstable terminal paths remain bounded and visible. Add focused tests for default retry semantics, bounded degraded-mode retries, terminal close statuses, and align web.reconnect.maxAttempts schema/help/generated docs.
