## Summary

Keep this PR open. Current main still derives the Mattermost interaction signing secret from the bot token using the public HMAC key, the paired bug #64545 remains open, and this PR is the active implementation candidate. The supplied diff is focused on Mattermost code/tests/config and does not show unrelated supply-chain or CI changes, but the in-memory random fallback needs revision because send and gateway verification can happen in different processes or after restart.

## Best Possible Solution

Keep this PR and #64545 open. Revise the Mattermost fix so interaction signing uses a persistent per-account secret that is not derived from the bot token, is shared by send and gateway monitor paths across processes and restarts, supports SecretRef or credential-store persistence, updates schema/types/setup/docs, and adds regression coverage for bot-token forgery resistance plus cross-process/restart verification.
