## Summary

Keep this PR open. Current main still implements the reported Mattermost interaction signing scheme by deriving the HMAC secret from the bot token with a public constant, the paired security bug #64545 remains open, and this PR is the active implementation candidate. The supplied PR diff is focused on Mattermost code/tests/config and does not show unrelated CI, dependency, package, or supply-chain changes, but it still needs revision because the random in-memory fallback can break valid button callbacks across separate CLI/gateway processes or after restart unless a persistent configured secret exists.

## Best Possible Solution

Keep this PR and #64545 open. Revise the Mattermost fix so interaction signing uses a persistent per-account secret that is not derived from the bot token, is shared by the send and gateway monitor paths across processes and restarts, supports SecretRef or credential-store persistence, updates schema/types/setup/docs, and adds regression coverage for bot-token forgery resistance plus cross-process/restart verification.
