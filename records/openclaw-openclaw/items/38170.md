## Summary

Keep this PR open. Current main still does not implement the Telegram workflow this PR targets: Telegram does not advertise or handle `download-file`, media placeholders remain bare `<media:...>` tags, and the only current `file_id` exposure is a sidecar string used in a narrower caption/download-failure path. The PR also needs to be ported from removed pre-plugin paths into `extensions/telegram` and reworked to use the existing Telegram media transport/download helpers.

## Best Possible Solution

Keep this PR or a refreshed follow-up open, but port it to the current Telegram plugin layout. The implementation should advertise and dispatch `download-file` from `extensions/telegram/src/channel-actions.ts`, implement it in `extensions/telegram/src/action-runtime.ts` by reusing the existing Telegram media transport/download helpers, expose `file_id` in a parser-safe inbound placeholder/history contract, update exact-tag audio consumers, and add focused Telegram tests plus docs for the new action.
