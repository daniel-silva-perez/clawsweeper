## Summary

Keep this PR open. Current main at 69fb7455c6223682a45dd8b8fb720bc3231bfa43 still lacks the PR's intended Telegram workflow: no Telegram `download-file` action and no parser-safe `file_id` attribute in inbound media placeholders/history. The idea remains useful, but the submitted branch edits removed pre-plugin paths and needs to be ported to `extensions/telegram` with the existing Telegram media transport/download helpers.

## Best Possible Solution

Keep this PR or a refreshed follow-up open. Port the feature into `extensions/telegram`: advertise and dispatch `download-file` from `channel-actions.ts`, implement it in `action-runtime.ts` using the existing Telegram media transport/download helpers, preserve `file_id` in a parser-safe inbound placeholder/history contract, update exact-tag/audio consumers, and add focused Telegram tests plus docs for the new action.
