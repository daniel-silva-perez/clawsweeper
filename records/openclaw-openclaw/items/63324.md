## Summary

Keep this PR open. Current main still has the pairing allowlist stat-error behavior the PR targets: the async write path still swallows all `fs.promises.stat` failures, and the current sync file helper still treats non-`ENOENT` `statSync` failures as a missing allowlist. The PR remains useful but needs adaptation to the current helper split plus focused regression tests.

## What This Changes

The PR updates the pairing allow-from store code to rethrow non-ENOENT filesystem stat errors instead of treating unexpected failures as missing allowlist files.

## Best Possible Solution

Keep this PR open and update it for current main: preserve the async `writeAllowFromState` non-`ENOENT` rethrow in `src/pairing/pairing-store.ts`, move the sync stat-error fix to `src/pairing/allow-from-store-file.ts`, and add focused regression tests for async write stat failures and sync read stat failures. Coordinate with #63423, but do not treat that umbrella issue as a replacement for this narrow pairing fix.
