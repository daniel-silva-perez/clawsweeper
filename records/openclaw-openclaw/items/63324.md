## Summary

Keep this PR open. Current main still has the pairing allowlist stat-error behavior this PR targets: `writeAllowFromState` still swallows all async `fs.promises.stat` failures, and the current sync allow-from helper still treats non-`ENOENT` `statSync` failures as a missing allowlist. The PR remains useful, but it is stale against the current helper split and needs adaptation plus focused regression tests rather than closure.

## Best Possible Solution

Keep this PR open and update it for current main: preserve the async `writeAllowFromState` non-`ENOENT` rethrow in `src/pairing/pairing-store.ts`, move the sync stat-error fix to `src/pairing/allow-from-store-file.ts`, and add focused regression tests for async write stat failures plus sync read stat failures. Coordinate with #63423 for broader empty-catch cleanup, but do not treat that umbrella as a replacement for this pairing-specific fix.
