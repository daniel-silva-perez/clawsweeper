## Summary

Keep this PR open. Current main still only has the earlier provider-side clear-output path from #72524; it does not implement this PR's distinct follow-up: a transport-callable realtime `handleBargeIn()` seam, Google Meet `chrome.bargeIn*` config/docs, a side-channel microphone monitor, or suppression of shared loopback input during assistant playback. The related architecture issue #73850 is also open, so the best path is maintainer review of the proposed seam rather than cleanup closure.

## Best Possible Solution

Keep this draft PR available for maintainer review alongside open issue #73850. The best implementation path is to decide whether handleBargeIn is the right additive realtime voice provider contract, keep the Google Meet side-channel microphone command opt-in with clear config/docs, add any missing manifest/config metadata for chrome.bargeIn* keys, and validate the command-pair behavior with focused Google Meet/OpenAI realtime tests plus SDK/config checks before merge.
