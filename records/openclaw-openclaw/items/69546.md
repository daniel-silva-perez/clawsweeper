## Summary

Keep this issue open. Current main still routes QQBot inbound messages through the gateway startup config snapshot even though bindings are classified as no-restart config, and #73567 is an open, unmerged closing PR for the same bug.

## What This Changes

Request that QQBot apply updated account and peer bindings to inbound routing without requiring a gateway restart.

## Best Possible Solution

Keep this issue open and route maintainer attention to #73567. If that PR lands, this issue can close as implemented; if it closes unmerged, the replacement should narrowly make QQBot resolve the active runtime config for each inbound routing decision and add focused regression coverage for account-binding to peer-binding reroute without a gateway restart.
