## Summary

Keep this issue open. Current main still passes the QQBot gateway startup config snapshot into inbound routing and outbound dispatch, while top-level `bindings` are classified as no-restart config and the generic router caches evaluated bindings by config object/reference. PR #73567 is an open, unmerged implementation candidate that explicitly targets this issue, so the best cleanup action is to review/land or close that PR rather than close the issue early.

## Best Possible Solution

Keep this issue open and route review to PR #73567. If that PR lands, close this issue as implemented by the merge; if it is closed unmerged, the replacement fix should keep the scope narrow: make QQBot resolve the active runtime config per inbound routing decision and add focused regression coverage for account-only to peer-binding reroute without a gateway restart.
