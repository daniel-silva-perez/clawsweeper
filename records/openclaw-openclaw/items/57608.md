## Summary

Keep this PR open. The Google Chat auth-diagnostic half is largely covered on current main through rejection-reason logging and appPrincipal warnings, and the linked Google Chat bug is already closed, but current main still keeps Zalo's default media cap at 5 MB with tests/docs expecting 5 MB. Because one of the PR's two intended fixes is not implemented on main, this PR is not closeable as implemented.

## Best Possible Solution

Keep the PR open for maintainer review or contributor rework. The best path is to rebase and narrow it around the still-missing Zalo default if maintainers agree with the 20 MB platform limit, updating Zalo source, tests, docs, and any user-facing changelog entry. The Google Chat half should either be dropped as already covered on main or deliberately adjusted only if maintainers want `runtime.warn` semantics instead of the existing logged diagnostics.
