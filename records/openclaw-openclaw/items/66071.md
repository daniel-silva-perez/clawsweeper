## Summary

Keep open. Current main still streams OpenResponses HTTP assistant deltas without checking `phase`, so the PR targets a real remaining gateway bug, but the proposed patch should not merge as-is because hidden commentary is still accumulated into the text used for final SSE completion and can reappear as final-answer output.

## What This Changes

This PR buffers OpenResponses HTTP streaming assistant text by phase, suppresses commentary `response.output_text.delta` events, preserves phase metadata on completed output items, and adds two SSE regression tests.

## Best Possible Solution

Revise the streaming handler to track user-visible final-answer text separately from commentary text, use only visible text for `response.output_text.delta`, `response.output_text.done`, and final-answer message content, and add a mixed commentary-then-final-answer regression test while preserving phase-marked structured output where the API contract requires it.

## Reproduction Assessment

Yes. A focused SSE test can emit a `commentary` assistant delta followed by a `final_answer` delta and lifecycle end; the PR head source trace shows both chunks flow through `accumulatedText` and into final completion output.

## Solution Assessment

No. Buffering by phase is the right direction, but this patch only suppresses delta events and does not separate hidden commentary from the finalized visible text used by completion events.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep hidden commentary out of final visible text:** `src/gateway/openresponses-http.ts:963-965`
  - body: This still appends commentary chunks to `accumulatedText` before returning for `phase === "commentary"`. When a run later emits a `final_answer` chunk, lifecycle finalization uses that combined `accumulatedText` for `response.output_text.done` and the completed final-answer message, so the commentary hidden from deltas reappears at completion.
  - confidence: 0.87
