## Summary

Keep open. Current main still does not implement native DOCX text extraction for either `input_file` or `read`: DOCX is only recognized by MIME detection, while input-file defaults exclude it, the bundled document extractor is PDF-only, and the read wrapper only handles adaptive paging/image normalization. The PR discussion contains useful follow-up fixes and security hardening notes, so the best next step is maintainer review/adaptation rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best product/code direction is to adapt the DOCX extractor into the existing `document-extract` plugin and generic document-extractor routing, add the DOCX MIME to the `input_file` defaults and docs, wire `read` through a bounded DOCX-only extraction path, and preserve the PR’s regression coverage for paragraphs, tables, numeric entities, malformed archives, default MIME handling, `input_file`, and `read`.
