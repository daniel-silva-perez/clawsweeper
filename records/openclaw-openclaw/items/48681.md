## Summary

Keep this PR open. Current main still does not implement the PR’s central user-visible change: agent config accepts a single `workspace`, runtime workspace resolution/setup remains single-root, and docs still describe one active workspace rather than a shipped composite multi-workspace mode. The PR is a plausible implementation candidate, but it needs maintainer review and fixes for the symlink/filesystem edge cases called out in review.

## Best Possible Solution

Keep this PR open as the implementation candidate for multi-workspace agent support. If maintainers want the feature in core, finish the open review items, route all agent execution paths through one tested composite workspace setup seam, update config schema/generated docs, and add focused tests for symlink collision, relative/root paths, stale entries, and filesystem-scope behavior.
