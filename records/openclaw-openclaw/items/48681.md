## Summary

Keep this PR open. Current main still implements agent configuration and runtime workspace handling as a single active workspace, while this PR remains the active implementation candidate for the multi-workspace composite symlink feature. It is not obsolete or implemented on main, and the PR still needs maintainer review for filesystem behavior and coverage before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate. If maintainers want multi-workspace agents in core, finish the config type/schema/docs, route every workspace setup path through one tested composite-workspace seam, and require focused tests for collision handling, blank/relative/root paths, stale and non-symlink entries, platform symlink behavior, reverse lookup, workspace listing/backup behavior, and reply/CLI/cron/heartbeat parity.
