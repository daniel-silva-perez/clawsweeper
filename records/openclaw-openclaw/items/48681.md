## Summary

Keep this PR open. Current main still has single-workspace agent config, schema, resolution, setup, reverse lookup, and docs; the proposed `multipleWorkspaces`/composite symlink behavior is not implemented. The PR remains a plausible implementation candidate, but the discussion and file list show it still needs maintainer review around all execution paths and symlink filesystem edge cases before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate. If maintainers want multi-workspace agents in core, finish the config type/schema/generated docs, route every agent workspace setup path through one tested composite-workspace seam, update public docs, and add focused tests for collision handling, blank/relative/root paths, stale and non-symlink entries, platform symlink behavior, reverse lookup, backup/listing behavior, and reply/CLI/cron/heartbeat parity.
