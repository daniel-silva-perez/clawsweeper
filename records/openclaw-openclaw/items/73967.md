## Summary

Keep open: the PR addresses a real Discord message-tool gap on current main, but it is not merge-ready because the edit support is still applied to the old monolithic messaging runtime instead of the current split edit handler.

## What This Changes

The PR exposes a Discord `components` schema contribution for shared message send/edit actions, forwards top-level `params.components` through Discord send/edit handling, and adds focused Discord tests.

## Best Possible Solution

Rebase the PR onto current main and adapt the edit path to the split runtime: keep the object-shaped OpenClaw Discord component schema, forward top-level `components` for send and edit, add `editDiscordComponentMessage` to the messaging runtime seam, and cover schema, send bridge, and edit runtime behavior with targeted Discord tests.
