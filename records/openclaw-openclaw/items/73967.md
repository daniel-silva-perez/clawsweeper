## Summary

Current main still lacks the PR's intended Discord message-tool behavior: Discord discovery does not contribute a `components` schema, and the `send` bridge does not read top-level `params.components`. The PR remains a focused implementation candidate; the only maintainer follow-up is to review the schema shape against the existing OpenClaw Discord component spec helper and then land or request changes.

## Best Possible Solution

Keep this PR open and review it as the focused fix.
