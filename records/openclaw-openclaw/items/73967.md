## Summary

Current main does not implement this PR's intended Discord message-tool behavior. The Discord docs and lower runtime already support OpenClaw Discord component specs, but the shared `message` send bridge still ignores top-level `params.components`, and Discord action discovery does not contribute a `components` schema to the message tool. This PR remains a relevant implementation candidate and should stay open for maintainer review rather than being closed as obsolete.

## Best Possible Solution

Keep this PR open and review it as the focused fix for Discord `message(action="send", components=...)`. The best path is likely to land the forwarding change plus a schema contribution aligned with the existing OpenClaw Discord component spec helper, with targeted coverage in `extensions/discord/src/channel-actions.test.ts` and `extensions/discord/src/actions/handle-action.test.ts`.
