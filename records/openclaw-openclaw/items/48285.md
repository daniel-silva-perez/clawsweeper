## Summary

Keep this PR open. Current main still does not implement the PR's two observable changes: there is no loopback `node` role exemption in the missing-device policy, and the backend self-pairing skip still excludes `auth.mode="none"`. The PR also remains security-sensitive because it broadens gateway device/pairing behavior and has unresolved review feedback about Control UI scoping and whether the named cron/session/ACP callers actually use the node-role path.

## Best Possible Solution

Keep this PR open for maintainer review or ask the author for a rebased, narrower replacement tied to the confirmed runtime rejection path. A strong patch should add regression coverage for cron, `sessions_spawn`, and ACP/internal backend calls across the relevant auth modes, include `auth.mode="none"` if that path remains broken, and scope any loopback node-role exemption to non-Control-UI direct-local sessions with explicit security review.
