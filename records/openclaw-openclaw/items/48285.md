## Summary

Keep this PR open. Current main still does not implement the PR's intended gateway behavior: device-less node-role loopback sessions are still rejected, backend self-pairing still has no auth.mode="none" exception, and docs/tests continue to describe node clients as requiring normal device identity and pairing. The PR also remains security-sensitive because it broadens device/pairing bypass behavior and has unresolved review feedback about Control UI scoping and whether the named cron/session/ACP paths actually use node-role sessions.

## Best Possible Solution

Keep this PR open for maintainer review or ask for a rebased, narrower replacement tied to the confirmed runtime rejection path. A strong fix should prove the actual cron, sessions_spawn, and ACP caller paths, cover auth.mode="none" if that remains broken, and scope any loopback node-role exemption to non-Control-UI direct-local sessions with explicit regression tests for valid auth, failed auth, and remote rejection.
