## Summary

Keep this PR open. Current main still rejects device-less node-role connections even with valid shared auth, still limits backend self-pairing skips to direct-local gateway-client backend calls using token/password or device-token auth, and the public docs describe node clients as using normal pairing. The PR discussion remains useful: the author later narrowed the cron/bootstrap claim, and review comments identified unresolved security/scope questions around Control UI handshakes and operator-backed internal callers.

## Best Possible Solution

Keep this PR open for maintainer review, or ask for a rebased narrower replacement tied to the confirmed runtime rejection path. A strong fix should prove the actual cron, sessions_spawn, and ACP caller identities, decide explicitly whether auth.mode="none" should skip backend self-pairing, scope any node-role exemption to non-Control-UI direct-local sessions, and update gateway tests/docs for valid auth, failed auth, remote rejection, and auth.mode="none" behavior.
