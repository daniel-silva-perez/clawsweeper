## Summary

Keep this PR open. Current main still rejects device-less node-role sessions and documents/tests the shipped exception as a narrower shared-secret `gateway-client` backend path, not a general loopback node-role bypass. The PR’s later `auth.mode="none"` backend self-pairing branch is also not present on main, and the PR discussion leaves the actual cron/bootstrap rejection path partly unresolved.

## Best Possible Solution

Keep this PR open for maintainer review, or ask for a rebased narrower replacement tied to the confirmed runtime rejection path. A strong next patch should add focused regression coverage for cron, `sessions_spawn`, and ACP/internal backend calls across relevant auth modes, include `auth.mode="none"` if that path is still broken, and only add a loopback node-role device-identity exemption if maintainers explicitly accept that gateway security behavior with non-Control-UI scoping.
