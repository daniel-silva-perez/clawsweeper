## Summary

Keep this issue open. Current main still parses Feishu fetched quoted/thread message content without modeling or normalizing fetched-message `mentions`, while the live inbound event path already normalizes Feishu mention placeholders. The issue is also paired with open PR #48976 via closing syntax, so the right next step is to review, narrow, land, or close that PR rather than close this issue early.

## What This Changes

Resolve Feishu quoted/replied message content so fetched `@_user_N` placeholders are converted using Feishu mention metadata instead of being passed through raw.

## Best Possible Solution

Keep this issue open and use PR #48976 as the canonical implementation candidate. The fix should stay in the Feishu plugin: type/read fetched-message `mentions`, apply the existing `normalizeMentions()` helper after `parseFeishuMessageContent()` for `getMessageFeishu()` and thread-list parsing where applicable, and add focused regression coverage in `extensions/feishu/src/send.test.ts` for fetched text/post content containing an `@_user_N` placeholder plus matching mention metadata.
