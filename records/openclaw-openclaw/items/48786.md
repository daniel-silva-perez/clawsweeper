## Summary

Keep open. Current main still parses Feishu fetched quoted/thread message content without modeling or normalizing fetched-message mentions, while the live inbound path already normalizes Feishu mention placeholders. The issue is also paired with open PR #48976, so it should remain open until that implementation candidate is reviewed, landed, or closed.

## What This Changes

Resolve Feishu fetched quoted/replied message content so raw `@_user_N` placeholders render as `<at user_id="...">name</at>` tags or readable names, consistent with live inbound mentions.

## Best Possible Solution

Review and narrow PR #48976 or an equivalent Feishu-plugin-only patch: add a typed fetched-message `mentions` field, apply the existing `normalizeMentions()` helper after fetched content parsing for `getMessageFeishu()` and thread-list content, and add focused `extensions/feishu/src/send.test.ts` regression coverage for text/post or thread content containing an `@_user_N` placeholder plus matching mention metadata.
