## Summary

Keep open. Current main still parses Feishu fetched quoted/thread message content without modeling or normalizing fetched-message mentions, while the live inbound path already resolves Feishu mention placeholders. The issue is also paired with open PR #48976, so it should remain open until that implementation candidate is reviewed, landed, narrowed, or closed.

## What This Changes

Apply Feishu mention normalization to fetched quoted/replied message content so raw `@_user_N` placeholders render as readable mention tags or names.

## Best Possible Solution

Review and narrow PR #48976 or land an equivalent Feishu-plugin-only patch: add a typed fetched-message `mentions` field, apply the existing `normalizeMentions()` helper after fetched content parsing for quoted/root/thread messages, and add focused `extensions/feishu/src/send.test.ts` regression coverage for text/post or thread content containing an `@_user_N` placeholder plus matching mention metadata.
