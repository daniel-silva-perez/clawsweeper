## Summary

Keep this issue open. Current main still fetches Feishu quoted/replied message content through `getMessageFeishu()` without reading or normalizing the fetched message's `mentions`, while the same normalization exists only on the live inbound-message path. There is an open implementation PR, #48976, with closing syntax for this issue, so the issue should stay open until that PR is reviewed, merged, or replaced by a narrower fix.

## Best Possible Solution

Keep the issue open and fix it in the Feishu plugin owner surface with a narrow change: extend the fetched-message item type to include API `mentions`, adapt those mentions into the existing normalization format or extract a shared helper, apply it in `getMessageFeishu()` and likely `listFeishuThreadMessages()`, and add focused `send.test.ts` regression coverage for quoted text/post content containing `@_user_N`. Review #48976 as the linked implementation candidate, but narrow or replace it because the current diff contains unrelated changes.
