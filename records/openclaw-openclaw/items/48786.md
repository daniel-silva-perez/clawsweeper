## Summary

Keep open. Current main still parses Feishu fetched quoted, root, and thread message content without modeling or normalizing fetched-message mention metadata, while the live inbound path already normalizes Feishu mention placeholders. The issue is paired with open PR #48976 using closing syntax, so it should remain open until that PR or an equivalent focused fix lands.

## What This Changes

Resolve Feishu fetched quoted/replied message mention placeholders so raw `@_user_N` text in quoted context is converted to readable `<at user_id="...">name</at>`-style mentions.

## Best Possible Solution

Review and narrow PR #48976, or land an equivalent Feishu-plugin-only patch that models fetched-message `mentions`, adapts the fetched API shape into the existing mention-normalization helper, applies normalization after fetched content parsing for quoted/root/thread messages, and adds focused `extensions/feishu/src/send.test.ts` coverage for text and post/thread content containing `@_user_N` plus matching mention metadata.
