## Summary

Keep open. Current main still parses Feishu fetched quoted, root, and thread message content from raw `body.content` without modeling fetched-message mention metadata or applying the existing Feishu mention normalizer; open PR #48976 already targets the same bug, so the issue should remain open until that PR or an equivalent narrow fix lands.

## What This Changes

Resolve Feishu fetched quoted/replied message `@_user_N` placeholders by normalizing fetched quoted, root, and thread message content with Feishu mention metadata.

## Best Possible Solution

Review and narrow PR #48976, or land an equivalent Feishu-plugin-only patch that models fetched-message `mentions`, adapts the fetched API shape into the existing mention-normalization helper, applies it after fetched content parsing for quoted/root/thread messages, and adds focused `extensions/feishu/src/send.test.ts` coverage for text and post/thread content containing `@_user_N` plus matching mention metadata.
