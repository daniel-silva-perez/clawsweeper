## Summary

Keep open. Current main still parses Feishu fetched quoted/thread message content without modeling or normalizing fetched-message mentions, while the live inbound path already resolves Feishu mention placeholders. The issue is also paired with open PR #48976, so it should remain open until that implementation candidate is reviewed, narrowed, landed, or closed.

## What This Changes

The issue reports that Feishu quoted/replied message content fetched for agent context leaves `@_user_N` mention placeholders unresolved instead of converting them to readable `<at>` tags or names.

## Best Possible Solution

Review and narrow PR #48976 or land an equivalent Feishu-plugin-only patch: model fetched-message `mentions`, adapt the SDK `{id, id_type}` mention shape into the existing mention-normalization helper, apply it after fetched content parsing for quoted/root/thread messages, and add focused `extensions/feishu/src/send.test.ts` regression coverage for text and post/thread content containing `@_user_N` plus matching mention metadata.
