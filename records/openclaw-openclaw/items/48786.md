## Summary

Keep open. Current main still parses Feishu fetched quoted/root/thread message content without modeling or normalizing fetched-message mention metadata, while the live inbound path already resolves Feishu mention placeholders. Open PR #48976 is an unmerged implementation candidate with closing syntax for this issue, so the issue should stay open until that PR or an equivalent narrow fix lands.

## What This Changes

Normalize Feishu quoted/replied fetched message content so raw `@_user_N` placeholders resolve to `<at user_id="...">name</at>` tags or readable names, matching live inbound messages.

## Best Possible Solution

Review and narrow PR #48976, or land an equivalent Feishu-plugin-only patch that models fetched-message `mentions`, adapts them into the existing mention-normalization helper, applies normalization after fetched content parsing for quoted/root/thread messages, and adds focused `extensions/feishu/src/send.test.ts` regression coverage for text and post/thread content containing `@_user_N` plus matching mention metadata.
