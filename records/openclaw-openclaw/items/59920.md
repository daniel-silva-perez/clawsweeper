## Summary

Keep this PR open. Current main still does not implement the terminal-field preference this PR targets: `parseCliJsonl()` only handles Claude `type:"result"` JSONL plus generic `item.text` fallback, while the OpenAI Codex CLI backend still emits `output: "jsonl"` into that parser path. The PR remains an active, focused implementation candidate for the closed #59918 parser bug rather than obsolete work. Security review of the provided file list found no workflow, dependency, lockfile, package publishing, install-script, permission, or secrets-handling changes; the remaining risk is parser compatibility, especially the existing review concern about commentary-only successful JSONL falling back to raw JSONL.

## Best Possible Solution

Keep this PR open for normal maintainer review and landing. The best path is a narrow parser fix that prefers `task_complete.last_agent_message`, falls back to assistant content tagged `phase:"final_answer"`, preserves Claude stream-json behavior, deliberately handles commentary-only fallback semantics, and keeps focused regression coverage in `src/agents/cli-output.test.ts`.
