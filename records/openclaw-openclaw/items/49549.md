## Summary

Keep this PR open. Current main still has no WhatsApp outbound-only allowlist or equivalent `allowTo` config, and the PR remains a focused implementation candidate for the allowFrom/outbound coupling problem tracked by #25039/#25513. It is not superseded by #48387 because that open PR is narrower and current main has not shipped either approach.

## Best Possible Solution

Keep this PR open. The best path is to review it against current main, require the directory-resolution behavior to be fixed or explicitly split to a canonical follow-up, then decide whether to land the full `allowTo` config/schema/runtime/docs/tests model or supersede it with a maintainer-chosen narrower approach such as #48387.
