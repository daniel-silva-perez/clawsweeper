## Summary

Keep this PR open. Current main still lacks a WhatsApp `allowTo` or equivalent outbound-only allowlist, so the PR is not obsolete; however, the branch is not merge-ready because the directory-backed target lookup gap remains and the branch is stale against the current WhatsApp plugin layout.

## What This Changes

The PR adds a WhatsApp `allowTo` config field intended to decouple outbound target gating from inbound `allowFrom` checks across account config, outbound resolution, action authorization, implicit delivery, docs, and tests.

## Best Possible Solution

Keep the PR open and decide whether OpenClaw wants the full `allowTo` config/schema/runtime/docs/tests model or the narrower explicit-target behavior in #48387. If proceeding with `allowTo`, rebase it onto current main and include directory resolution coverage or explicitly split that limitation into a canonical follow-up before landing.

## Reproduction Assessment

Yes. The current-main source path is enough to reproduce the core issue: a direct WhatsApp target outside `allowFrom` is rejected by `resolveWhatsAppOutboundTarget`, and directory lookup currently lists peers from `allowFrom` only.

## Solution Assessment

Unclear. A separate outbound allowlist is a plausible maintainable fix for the reported coupling, but this branch is incomplete for directory-backed resolution and stale against current main, while #48387 proposes a narrower policy change that maintainers may prefer.
