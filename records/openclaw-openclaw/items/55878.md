## Summary

Keep open. The report has a concrete macOS crash stack in `WebSocketTaskBox.sendPing()`, and current `main` still contains the same checked-continuation ping bridge plus the keepalive loop that calls it on the shared Gateway WebSocket. I did not find a release note, source change, or targeted regression test proving this crash is fixed.

## Best Possible Solution

Keep this issue open. The likely fix belongs in `apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayChannel.swift`: make the `sendPing()` async bridge robust against duplicate or late callbacks, ensure keepalive shutdown cannot resume a stale continuation, and add a Swift regression test with a fake `WebSocketTasking` that invokes the ping handler more than once. After that, verify DMG launch/menu bar stability on macOS and add a changelog fix entry.
