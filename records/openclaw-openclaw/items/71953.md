## Summary

Keep open. Current main still has the standalone `copy-hook-metadata` full-build step, but the `gatewayWatch` profile and `runRuntimePostBuild` path do not copy bundled hook `HOOK.md` manifests alongside the built `dist/bundled/<hook>/handler.js` files. The PR is a narrow, relevant fix and is not implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review, or land an equivalent fix that copies bundled hook `HOOK.md` manifests as part of `runRuntimePostBuild` so `gateway:watch` and full builds share the same hook discovery artifacts. Preserve the narrow scope: runtime postbuild helper, focused script tests, and the existing changelog attribution to @jtatum.
