## Summary

Keep this issue open. Current main has meaningful prompt/context and gateway-write hardening, including post-compaction red-line refresh and protected config-path checks, but model-callable owner gateway actions can still restart, apply/patch allowed config, or run updates without a separate recent user-consent artifact.

## Best Possible Solution

Keep this open as a product/security-behavior bug. Add a recent explicit-consent mechanism for model-initiated gateway restart, config.apply, config.patch, and update.run, enforced in the owner gateway tool and preferably in the RPC/server contract too. Preserve the existing owner-only, scope, allowlist, dangerous-flag, base-hash, and rate-limit protections, and add regression coverage for long-session or post-compaction red-line preservation before sensitive control-plane actions.
