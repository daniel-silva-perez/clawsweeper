## Summary

Keep this issue open. Current main has meaningful hardening around the owner-only gateway tool, but the core requested behavior is still not enforced: model-callable gateway actions can still restart, apply/patch allowed config, or run updates without a separate recent user-consent artifact.

## Best Possible Solution

Keep this open as a product/security-behavior bug. Add a recent explicit-consent mechanism for model-initiated gateway restart, config.apply, config.patch, and update.run, enforced server-side or in the owner-only gateway tool with regression coverage. Preserve the existing owner-only/admin-scope, allowlist, dangerous-flag, base-hash, protected-path, and rate-limit protections, and add long-session/compaction coverage so user red-line instructions and bootstrap identity/memory context remain available or are re-read before sensitive control-plane actions.
