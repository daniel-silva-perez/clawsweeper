## Summary

Keep open. Current main has meaningful hardening around the agent-facing gateway tool, including owner-only exposure, admin-scoped RPCs, config mutation allowlists, dangerous-flag blocking, base-hash checks, rate limits, and trimmed config-write tool responses. It still does not enforce a fresh user-consent gate before a model-callable owner gateway tool can run `restart`, allowed `config.apply`/`config.patch`, or `update.run`, so the central bug report is not fully resolved.

## Best Possible Solution

Keep this open as a product/security-behavior bug. Add an enforceable, test-backed recent explicit-consent mechanism for model-initiated `gateway` actions `restart`, `config.apply`, `config.patch`, and `update.run`, while preserving the existing owner-only/admin-scope, config allowlist, dangerous-flag blocking, base-hash, protected-path, and rate-limit guardrails. Add regression coverage for long sessions and compaction where user red-line instructions and workspace identity/memory files remain available or are re-read before sensitive control-plane actions.
