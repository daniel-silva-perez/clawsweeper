## Summary

Keep open. Current main still has global-only internal hook registration and dispatch, and it does not expose the proposed `hooks.internal.events`, `agents.defaults.hooks`, or `agents.list[].hooks` policy surfaces. The PR remains a coherent implementation candidate, but it changes core hook execution policy and has unresolved review concerns that need maintainer judgment before merge.

## Best Possible Solution

Keep this PR open for maintainer review. If maintainers accept the capability, revise it to define strict schema/types for `hooks.internal.events`, `agents.defaults.hooks`, and `agents.list[].hooks`; apply agent-scoped policy only when an event has a real agent scope; preserve global behavior for gateway/non-agent lifecycle hooks; document exact event matching or add wildcard semantics; and add focused tests for precedence, named-hook gating, non-agent hook behavior, hot-path policy resolution, and backward compatibility. If maintainers reject the product direction, close it explicitly with that rationale.
