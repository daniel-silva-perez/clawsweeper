## Summary

Keep open. Current main still has global-only internal hook dispatch/config, so this PR's agent-scoped internal hook policy override work is not implemented or superseded. The PR is a coherent implementation candidate, but it still needs maintainer review because it changes core hook execution policy and has unresolved review concerns around exact event matching, applying agent defaults to non-agent events, and hot-path policy resolution.

## Best Possible Solution

Keep this PR open for maintainer review. If the capability is accepted, land it as a core config/runtime change after addressing the review comments: define strict schema/types for `hooks.internal.events`, `agents.defaults.hooks`, and `agents.list[].hooks`; pass config plus resolved agent identity into internal hook dispatch; apply agent overlays only to agent-scoped events; document exact event matching; and add focused tests for precedence, named hook gating, non-agent hook behavior, and backward compatibility. If maintainers reject the product direction, close it explicitly with that rationale.
