## What This Changes

The PR changes Codex app-server queued steering to return delivery success, adds a durable embedded-message helper and followup fallback in `runReplyAgent`, widens queue-handle return types, and adds regression coverage for rejected or post-completion steering.

## Best Possible Solution

Land a rebased narrow fix that makes every default Codex queued steering message either reach an active `turn/steer` successfully or fall through to the existing followup queue when the turn is already terminal, not steerable, aborted, or rejected.

## Reproduction Assessment

Yes. ## Solution Assessment

Yes, the proposed direction is the narrow maintainable fix because it makes steering delivery observable and reuses the existing followup queue for rejection. The current branch still needs rebase and reconciliation before that solution can be safely merged.
