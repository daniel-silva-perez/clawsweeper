## Summary

Keep this PR open. Current main still has the core durability problem: queued Codex app-server steering is treated as accepted before `turn/steer` is known to have succeeded, so a terminal or rejected steer can still leave no followup. The PR is a narrow implementation that appears to address that contract, but it needs maintainer review, rebase against the newer partial main fix, and the changed gate before merge.

## What This Changes

The PR makes Codex app-server queued steering return delivery success, adds an async durable embedded-message helper, awaits that helper in `runReplyAgent`, adds regression tests for terminal steering fallback, and updates one changed-test routing expectation.

## Best Possible Solution

Land a rebased narrow fix that preserves the active-run steering model while guaranteeing each queued user message is either accepted by live `turn/steer` or queued through the existing followup path when steering is terminal or rejected, and reconcile the current-main flush-after-completion test with that behavior.
