## Summary

Keep open. Current main still has no OpenRouter-owned thinking profile or legacy xhigh hook, so OpenRouter DeepSeek V4 models can resolve to the base thinking levels before the stream wrapper can forward `reasoning.effort`. The issue is also paired with open closing PR #74839, so the right cleanup path is to review or replace that PR, not close the issue early.

## What This Changes

Allow the OpenRouter provider plugin to advertise `/think xhigh` for OpenRouter DeepSeek V4 Pro and Flash so OpenClaw validation accepts the level before sending OpenRouter reasoning payloads.

## Best Possible Solution

Land a narrow OpenRouter-owned provider profile fix, or an equivalent replacement, that advertises the supported DeepSeek V4 levels and adds regression coverage for the positive V4 models plus a negative non-supported OpenRouter model before closing the issue.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: selecting `openrouter/deepseek/deepseek-v4-pro` or Flash and sending `/think xhigh` reaches thinking validation with no OpenRouter profile/compat xhigh support on current main, so the level is rejected before transport shaping.

## Solution Assessment

Yes, the requested provider-owned thinking profile is the narrow maintainable direction. The open PR should still be reviewed for scope so it does not over-advertise xhigh for OpenRouter models whose upstream route only has a generic `reasoning` boolean.
