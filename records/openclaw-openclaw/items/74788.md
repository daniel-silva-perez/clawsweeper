## What This Changes

The issue asks OpenRouter to advertise `xhigh` thinking support for DeepSeek V4 Pro and Flash models so `/think xhigh` is accepted before request transport.

## Best Possible Solution

Land a narrow OpenRouter-owned `resolveThinkingProfile` implementation, or an equivalent revised PR, with focused coverage for the two DeepSeek V4 OpenRouter IDs plus negative cases for non-reasoning and explicitly unsupported OpenRouter proxy routes.

## Reproduction Assessment

## Solution Assessment

Yes. A provider-owned thinking profile is the narrow maintainable fix because current docs and SDK contracts make plugins responsible for model-specific `/think` level sets; a core resolver change would be broader than necessary.
