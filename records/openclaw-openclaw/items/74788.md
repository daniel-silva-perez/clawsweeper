## Summary

Keep open. Current main still lacks an OpenRouter-owned thinking profile or legacy xhigh hook, so `openrouter/deepseek/deepseek-v4-pro` and Flash can still fall back to the base thinking levels before the OpenRouter stream wrapper can forward `reasoning.effort`; open closing PR #74839 remains the right implementation path to review rather than closing this issue early.

## What This Changes

The issue reports that OpenRouter DeepSeek V4 models reject `/think xhigh` because the OpenRouter plugin does not advertise an xhigh-capable thinking profile.

## Best Possible Solution

Land a narrow OpenRouter provider-profile fix, or an equivalent replacement, that advertises the supported DeepSeek V4 thinking levels and adds regression coverage for the positive V4 routes plus a negative OpenRouter route before closing the issue.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: selecting an OpenRouter DeepSeek V4 model and sending `/think xhigh` reaches thinking validation with no OpenRouter provider profile, no compat xhigh metadata, and no legacy xhigh hook, so the level is not accepted on current main.

## Solution Assessment

Yes. A provider-owned `resolveThinkingProfile` is the narrow maintainable solution because provider plugins already own model-specific thinking level sets; the safer implementation should scope xhigh to the verified DeepSeek V4 routes or equally precise capability metadata.
