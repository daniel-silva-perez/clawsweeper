Current main still lacks an OpenRouter-owned thinking profile or legacy xhigh hook, so the reported `/think xhigh` rejection path remains source-plausible; open PR #74839 already references this issue with closing syntax, so the maintainer-safe path is to review, revise, or land that PR rather than close this issue early.

## What This Changes

The issue requests OpenRouter provider thinking-level support so `/think xhigh` is accepted for `openrouter/deepseek/deepseek-v4-pro` and `openrouter/deepseek/deepseek-v4-flash`.

## Best Possible Solution

Land a narrow OpenRouter-owned thinking profile, or an equivalent replacement, that advertises the documented DeepSeek V4 OpenRouter levels and includes focused positive coverage for Pro/Flash plus negative coverage for OpenRouter routes that should not receive extended reasoning levels.

## Reproduction Assessment

Yes. The source-level path is high-confidence: selecting either reported OpenRouter DeepSeek V4 model with `/think xhigh` reaches thinking validation, where current main has no OpenRouter provider profile/hook and no compat effort metadata to add `xhigh`.

## Solution Assessment

Yes. A provider-owned `resolveThinkingProfile` is the narrow maintainable fix because the current docs and SDK contract make provider plugins responsible for model-specific `/think` level sets; changing the core resolver would be broader than necessary.
