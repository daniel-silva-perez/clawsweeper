## Summary

Keep open. Current main still leaves OpenRouter DeepSeek V4 models on the base thinking profile, so explicit `/think xhigh` can be rejected before the OpenRouter stream wrapper has a chance to send the supported reasoning payload. The related DeepSeek-provider PR is useful precedent but explicitly does not cover OpenRouter behavior.

## What This Changes

Expose `xhigh`/max thinking support for OpenRouter DeepSeek V4 model refs so the thinking directive and picker accept the levels before request construction.

## Best Possible Solution

Add an OpenRouter-owned thinking profile for `deepseek/deepseek-v4-pro` and `deepseek/deepseek-v4-flash`, keep the change inside the OpenRouter plugin boundary, and cover both positive V4 and negative non-V4 model cases so UI, session validation, directives, and transport behavior agree.
