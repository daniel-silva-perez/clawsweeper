## Summary

Keep open. Current main has useful adjacent improvements for service drift, gateway status/probe output, and Telegram duplicate-poller logging, but `openclaw doctor` still lacks the requested consolidated mixed-state diagnosis and final healthy/degraded/down verdict.

## Best Possible Solution

Keep this open and implement a doctor-owned gateway diagnosis collector that gathers service installed/loaded state, LaunchAgent supervision, runtime PID/status, RPC/read-probe result, port listener ownership, service config drift, extra gateway-like services, recent channel-specific signals such as Telegram `getUpdates` 409 conflicts, and last gateway errors. It should print one final healthy/degraded/down verdict with deterministic next steps and, if maintainers want it, a compact machine-readable diagnosis block.
