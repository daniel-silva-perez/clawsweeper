## Summary

Keep open. Current main has useful adjacent improvements for service drift, gateway status, and Telegram 409 duplicate-poller hints, but `openclaw doctor` still does not collect LaunchAgent/service/runtime/RPC/port/channel signals into a single final healthy/degraded/down diagnosis with deterministic next steps.

## Best Possible Solution

Keep this issue open and implement a doctor-owned gateway diagnosis collector. It should gather LaunchAgent/service installed and loaded state, service runtime PID/status, RPC/read-probe result, port listener ownership, service config drift, extra gateway-like services, recent Telegram `getUpdates` 409 signals, and last gateway errors, then print one final healthy/degraded/down verdict with ordered next steps and an optional compact JSON block.
