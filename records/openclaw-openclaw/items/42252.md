## Summary

Keep open. Current main still has useful adjacent work for service drift, gateway status/probe semantics, and Telegram duplicate-poller hints, but it does not implement the core request: `openclaw doctor` still lacks a consolidated mixed LaunchAgent/runtime/RPC diagnosis or final healthy/degraded/down verdict.

## Best Possible Solution

Keep this issue open and implement a doctor-owned gateway diagnosis collector. It should gather service installed/loaded state, LaunchAgent supervision, runtime PID/status, RPC/read-probe result, port listener ownership, service config drift, extra gateway-like services, recent Telegram `getUpdates` 409 signals, and last gateway errors, then print one final healthy/degraded/down verdict with ordered next steps and an optional compact JSON block.
