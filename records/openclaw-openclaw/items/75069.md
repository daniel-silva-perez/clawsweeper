## Summary

Keep open. Current main has meaningful partial mitigations for hardlink churn, dist-mirror memoization, and runtime-deps reinstall loops, but the synchronous runtime-deps/mirror preparation path and pre-HTTP Gateway staging/readiness path still exist, while maintainer discussion and open PR #75183 frame the remaining work as unresolved.

## What This Changes

The issue requests fixing a regression where bundled plugin runtime-deps and mirror preparation run synchronously and repeatedly during Gateway startup or first pi-agent tool construction, causing long event-loop stalls.

## Best Possible Solution

Keep this issue open until PR #75183 or an equivalent staged fix is resolved and packaged/global-install profiling demonstrates bounded Gateway HTTP readiness and first-agent event-loop lag for bundled runtime-deps and mirror preparation.

## Reproduction Assessment

Yes. The issue body gives a versioned npm-global reproduction, CPU attribution, and standalone mirror benchmarks, while NikolaFC adds an isolated HOME acpx-only readiness repro; current source still contains the paths those reproductions target.

## Solution Assessment

Unclear. Current main is a partial solution, but the remaining fix needs profiled maintainer choice between narrower readiness deferral/caching and broader async or cooperative mirror preparation.
