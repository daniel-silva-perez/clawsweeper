## Summary

Keep open. Current main still limits `browser upload` to standard file chooser/input mechanisms: managed/CDP profiles use Playwright file chooser or `locator.setInputFiles`, while existing-session profiles delegate to Chrome MCP `upload_file`. I found no DataTransfer, clipboard, or CDP drag-event upload fallback in the browser plugin, and the adjacent related reports do not supersede this ByteUploader/non-standard input request.

## Best Possible Solution

Keep this issue open and implement a generic browser-plugin upload fallback rather than a ByteDance hardcode. The likely home is `extensions/browser/src/browser`: preserve the existing guarded upload path, then add an explicit fallback/strategy for custom uploaders, such as DataTransfer/drop or CDP drag/paste, with path-root validation, target/ref scoping, clear success/failure reporting, docs, and synthetic tests for a custom uploader whose `files.length` remains zero after standard `setFiles`/`setInputFiles`. Coordinate with #38844 only for broader upload trigger/stale-click behavior.
