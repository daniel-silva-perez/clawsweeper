## Summary

Current main still does not implement the requested browser upload fallback for non-standard/custom uploaders. The browser plugin upload path validates files and then uses Playwright file chooser/setInputFiles or Chrome MCP upload_file; I found no DataTransfer, paste, or CDP drag-event fallback, and the related open upload issue tracks a different stale-ref/timeout problem rather than superseding this request.

## Best Possible Solution

Keep this issue open and implement a generic browser-plugin upload fallback rather than a ByteDance-specific hardcode. The likely path is to keep the current guarded standard upload flow, then add an explicit custom-uploader strategy such as DataTransfer/drop, paste, or CDP drag with target/ref scoping, path-root validation, clear success/failure reporting, docs, and tests for a custom uploader whose files.length remains zero after the standard path.
