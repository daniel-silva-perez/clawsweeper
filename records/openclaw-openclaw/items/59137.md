## Summary

Keep this PR open. Current main still does not implement the PR's central rollback recovery behavior: there is no full-session retry-pending state, no sync-state snapshot/restore around safe or unsafe reindex rollback, and successful embedding batches are still cached only after all batches complete, so a later batch failure can discard earlier cache progress. The linked discussion is useful: the PR is a narrower durability/retry-state improvement related to #44166 and the closed/superseded #55497, not something current main already ships.

## Best Possible Solution

Keep this PR open as a focused memory-core rollback/cache durability implementation candidate. The best path is to rebase it onto current main, preserve the retry-state and cache-mirroring tests, verify the current open review concern remains fixed, and review it alongside #44166/#44167 as complementary reliability work rather than closing it as already implemented.
