## Summary

Keep open. The underlying cron/session bug is real and current main still lacks the destination-session mirror, but this PR needs changes before merge because the diff can still miss documented bare custom-session targets and can mirror text that was intentionally stripped from the delivered payload.

## What This Changes

The PR changes isolated cron direct delivery to resolve a destination delivery session key, pass it into outbound delivery context and transcript mirroring, and adds a focused cron delivery regression test.

## Best Possible Solution

Land a cron-owned fix that resolves the destination to the same agent-scoped store session key used by the cron run, mirrors only the payload text/media that was actually delivered after silent-token filtering, and covers both full session keys and bare session:custom-id targets in tests.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: existing tests show bare custom sessions normalize to agent-scoped store keys, while the PR passes the raw target key to the outbound mirror path that requires a real session-store key.

## Solution Assessment

No. Using the outbound mirror path is the right direction, but this implementation should normalize the mirror key to the actual destination store key and derive mirror contents from the filtered delivery payload projection rather than raw output text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Normalize the mirror key to the stored cron session:** `src/cron/isolated-agent/delivery-dispatch.ts:585-586`
  - body: For a documented target like `session:project-alpha-monitor`, the cron run stores and uses `agent:<id>:project-alpha-monitor`, but this helper returns the raw `project-alpha-monitor`. The mirror append path looks up the session store by the key it receives, so bare custom-session cron jobs will still deliver to chat without appending to the destination transcript. Resolve the same agent-scoped key used for the run before passing it to `session` and `mirror`.
  - confidence: 0.88
- **[P2] Mirror the filtered delivered payload:** `src/cron/isolated-agent/delivery-dispatch.ts:617`
  - body: `payloadsForDelivery` strips silent-token text before sending, but the mirror writes `outputText || synthesizedText` instead. A payload that sends media after removing `NO_REPLY` text would still record that hidden text in the session transcript, making the agent remember content that was not actually delivered. Build the mirror text/media from the normalized payloads that are sent.
  - confidence: 0.82
