## Summary

Keep open. Current main still only detects nvm for bootstrap CA bundle injection, so this PR addresses a real gap; however, the proposed NVS marker misses default ~/.nvs installs and should be corrected before merge.

## What This Changes

The PR adds a version-manager marker list to `src/bootstrap/node-extra-ca-certs.ts` so bootstrap CA bundle auto-injection detects fnm, Volta, asdf, n, nodenv, nodebrew, and nvs in addition to nvm, with matching tests.

## Best Possible Solution

Land a corrected implementation that shares version-managed Node path detection across bootstrap, daemon service audit, and runtime path resolution, covers default `/.nvs/` paths, and keeps focused tests for typical paths for all supported managers.

## Reproduction Assessment

Yes. Current main can be checked directly from `isNodeVersionManagerRuntime`: any fnm/Volta/asdf/n/nodenv/nodebrew/nvs exec path returns false unless it also contains `/.nvm/`; for the PR finding, `/home/user/.nvs/node/22/x64/bin/node`.includes('/nvs/') is false.

## Solution Assessment

No, not yet. Expanding bootstrap detection is the right boundary, but copying the existing marker list preserves the NVS miss; a shared helper with corrected NVS coverage is the safer maintainable fix.
