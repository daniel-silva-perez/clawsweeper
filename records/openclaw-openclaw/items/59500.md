## Summary

Keep open. Current main still has the original bootstrap behavior: Linux CA bundle auto-injection only treats nvm-managed Node as version-managed, so this PR's intended fnm/volta/asdf/n/nodenv/nodebrew/nvs coverage is not implemented on main. The PR is focused and matches broader daemon/runtime marker logic, but the NVS marker concern from review should be resolved or explicitly scoped before landing.

## Best Possible Solution

Keep the PR open and either update it or replace it with a maintainer patch that centralizes/shared-aligns version-manager path detection for bootstrap, daemon audit, and runtime path helpers, adds regression tests for all intended managers, and resolves the default `~/.nvs` marker gap before landing.
