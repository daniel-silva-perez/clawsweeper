## Summary

Keep this PR open for maintainer review. Current main and the latest shipped canvas-host code still call `decodeURIComponent` without a malformed-encoding guard, so the reported 500 class is current; the proposed resolver-level fallback is narrow and matches the existing not-found behavior for unsafe or unresolvable paths.

## What This Changes

The PR catches `URIError` from canvas URL-path normalization, returns a resolver miss, and adds a server regression test for malformed percent-encoded canvas paths returning 404 `not found`.

## Best Possible Solution

Merge a resolver-level malformed-percent-decoding fallback with regression coverage so canvas and A2UI static-file serving treat invalid encoded paths as ordinary misses while preserving the existing root, traversal, symlink, method, and file-open safety checks.
