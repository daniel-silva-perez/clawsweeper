## Summary

Keep this PR open. Current main still lets malformed percent-encoded canvas paths throw from the shared file resolver before the normal not-found path, and the proposed diff is a narrow resolver-level fallback with matching regression coverage. I did not find a discrete correctness or security finding that should block maintainer review.

## What This Changes

The branch catches `URIError` from canvas URL path normalization in `resolveFileWithinRoot` and adds a canvas-host regression test for malformed encoded paths returning 404 `not found`.

## Best Possible Solution

Land a resolver-level malformed percent-decoding fallback with regression coverage so canvas and A2UI static-file serving treat invalid encoded paths as ordinary misses while preserving existing root, traversal, symlink, method, and safe-open behavior.
