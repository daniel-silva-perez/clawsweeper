## Summary

Keep open. Current main still does not implement the PR's intended WhatsApp per-agent ack emoji fallback: WhatsApp ack reactions still read only `channels.whatsapp.ackReaction.emoji`, while the generic identity resolver still treats account/channel ack reactions as strings. The PR remains a focused implementation candidate, but it still needs maintainer review or contributor revision for the public SDK export choice and the outstanding test lint/type concern.

## Best Possible Solution

Keep this PR open for maintainer review or contributor revision. The best path is a cleaned implementation that preserves WhatsApp's no-global-ack behavior, adds per-agent identity fallback only when WhatsApp ack config is intentionally present, covers account/channel/object edge cases with typed tests, and treats any plugin-SDK export as an intentional public contract.
