## Summary

Keep this PR open, but do not merge it as-is. The requested Slack `allowBots: "mentions"` behavior is still missing on current `main`, while the branch targets the removed `src/slack` runtime path and leaves the active Slack plugin/config metadata path incomplete.

## What This Changes

The PR widens Slack `allowBots` to accept `"mentions"`, adds mention-gated bot-message handling and tests in the old Slack runtime path, and updates core config schema/help text.

## Best Possible Solution

Ship the feature through the active Slack plugin: add a tri-state Slack bot policy in `extensions/slack`, preserve own-bot filtering, allowlists, DM behavior, control-command behavior, and `thread.requireExplicitMention`, widen current config schema/types/metadata/UI hints, document the mode, add changelog credit, and cover explicit mention, implicit-thread mention, DM, `true`, and `false` cases in active plugin tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current `main`: Slack validation/types reject `allowBots: "mentions"`, and active Slack message preparation only has a boolean bot gate with no mention-only mode.

## Solution Assessment

No. The intended behavior is useful, but this PR is not the best current implementation because it patches a removed Slack runtime path and misses active plugin metadata; the safer fix is a focused port into `extensions/slack` plus generated/public config surfaces.
