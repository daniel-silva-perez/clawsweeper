## Summary

Keep this PR open, but do not merge it as-is. The Slack `allowBots: "mentions"` feature is still missing on current `main`, while this branch targets the retired `src/slack` runtime path and only partially updates the active Slack plugin/config surfaces.

## What This Changes

This PR widens Slack `allowBots` to accept `"mentions"`, adds mention-gated bot-message handling and tests in the old Slack runtime path, and updates core config schema/help text.

## Best Possible Solution

Ship the feature through the active Slack plugin: add a tri-state Slack bot policy in `extensions/slack`, preserve own-bot filtering, allowlists, DMs, control commands, and `thread.requireExplicitMention`, update public config/schema/plugin metadata/docs/changelog, and cover explicit mention, implicit-thread mention, DM, `true`, and `false` cases in active plugin tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current `main`: Slack validation/types reject `allowBots: "mentions"`, and active Slack message preparation only has a boolean bot gate with no mention-only mode.

## Solution Assessment

No. The feature direction is sound, but this PR is not the best current implementation because it patches the retired `src/slack` path and misses active plugin metadata; the safer fix is a focused port or replacement PR against `extensions/slack` and the generated/public config surfaces.
