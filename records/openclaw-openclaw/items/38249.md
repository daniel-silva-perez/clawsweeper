## Summary

Keep this issue open. Current main still has active-schema validation, doctor best-effort legacy repair, unknown-key pruning, and a Control UI pending-change diff, but it does not implement the requested `openclaw doctor --config-diff` CLI workflow: there is no doctor flag or typed option, no version-to-version schema comparison from `meta.lastTouchedVersion`, no categorized migration report, and no `--config-diff --fix` repair path. Related open issues remain adjacent rather than superseding this CLI migration-diff request.

## Best Possible Solution

Keep this issue open until maintainers either implement or explicitly reject a CLI upgrade-diff workflow. A complete path would add `openclaw doctor --config-diff` plumbing, docs, and tests; compare the active config against the current schema and a trustworthy previous-version baseline from `meta.lastTouchedVersion`; report unknown keys, invalid enum values, removed/renamed/new keys, and migration suggestions; and reserve `--config-diff --fix` for explicit safe migrations with backups. Coordinate with #39780 for Control UI apply/diff UX and with config-regression reports such as #72872, but do not close this CLI feature request as covered by them.
