## Summary

Keep open. Current main still lacks a session-recap sanitizer, so the linked Control UI/TUI leak is still valid, but this PR is not merge-ready because the proposed helper can delete malformed leading user text and can alter non-leading user text by trimming it.

## What This Changes

The PR adds a session-recap fast path, a helper to strip leading `<session-recap>` or `<session_recap>` blocks from inbound user text, and colocated sanitizer tests.

## Best Possible Solution

Land a narrow sanitizer update that strips only complete leading `<session-recap>` and `<session_recap>` blocks in both inbound display paths, leaves malformed or mid-message literals untouched, preserves existing metadata behavior, and adds regression coverage for the two PR edge cases.

## Reproduction Assessment

Yes. A leading closed `<session-recap>` block with no standard inbound metadata sentinel bypasses both current-main sanitizer entry points; the PR-specific regressions are visible from the proposed EOF scan and no-strip slow-path return.

## Solution Assessment

No as-is. The PR is pointed at the right shared sanitizer boundary, but it should require a matching closing tag and preserve the original text when no leading recap block was removed.
