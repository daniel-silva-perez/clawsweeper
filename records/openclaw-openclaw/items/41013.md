## Summary

Keep open. Current main still does not strip leading <session-recap> blocks from the shared user-message cleanup paths used by Control UI and TUI, so this PR is not obsolete or implemented. The PR remains a focused implementation candidate for #40948, but the submitted diff has unresolved correctness review feedback around malformed blocks and whitespace preservation.

## Best Possible Solution

Keep the PR open as a focused candidate, but require a small revision before landing: add session-recap support in src/auto-reply/reply/strip-inbound-meta.ts, wire it into both full and leading-only stripping paths, only remove a well-formed leading block with a matching close tag, preserve the original string when no block is stripped, and add regression tests for well-formed, malformed/no-close, mid-message, leading blank line, and whitespace-preservation cases. Once revised and reviewed, it can close #40948.
