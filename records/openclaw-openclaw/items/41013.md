## Summary

Keep open. Current main still lacks `<session-recap>` stripping on the Control UI/TUI user-visible paths, so the PR addresses a real linked bug, but the proposed patch should not merge as-is because it can erase malformed user text and mutates non-stripped user content.

## What This Changes

The PR adds a `<session-recap>` fast-path/helper to the inbound metadata stripper and colocated tests for leading, blank-line, underscore-variant, chained-sentinel, and mid-message cases.

## Best Possible Solution

Repair and rebase the focused sanitizer change on current main so both full and leading-only cleanup strip only well-formed leading recap blocks while preserving existing metadata, timestamp, active-memory, and user-authored-content behavior; then land it as the implementation for #40948.
