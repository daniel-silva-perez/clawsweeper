## Summary

Keep open. Current main still lacks any `<session-recap>` handling in the shared inbound metadata sanitizer, and the Control UI/TUI display paths still rely on that sanitizer, so the PR targets a valid user-visible leak. The branch is not merge-ready because its helper deletes malformed leading recap text without a closing tag and its slow path trims user-authored text when no leading recap block was actually removed.

## What This Changes

The PR adds a session-recap fast path, a leading `<session-recap>`/`<session_recap>` stripping helper, and colocated sanitizer tests for user-visible inbound message rendering.

## Best Possible Solution

Land a rebased sanitizer fix on current main that strips only complete leading `<session-recap>` and `<session_recap>` blocks in both full and leading-only display paths, leaves malformed and mid-message literals untouched, preserves timestamp and active-memory stripping, and adds regression coverage for malformed/no-close and no-strip whitespace cases before resolving #40948.
