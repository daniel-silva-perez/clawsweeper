## Summary

Keep open. Current main still lacks session-recap handling on the shared user-visible metadata sanitizer path, and this PR is the right narrow implementation vehicle for the linked leak, but the proposed patch has two concrete sanitizer regressions that should block merge as-is.

## What This Changes

The PR adds a session-recap fast path, a helper to strip leading `<session-recap>` or `<session_recap>` blocks, and focused tests in `src/auto-reply/reply/strip-inbound-meta.test.ts`.

## Best Possible Solution

Land a narrow sanitizer update that strips only complete leading `<session-recap>` and `<session_recap>` blocks in both exported strip functions, leaves malformed or mid-message literals untouched, preserves existing timestamp, active-memory, and standard metadata behavior, and adds regression coverage for the two edge cases.

## Reproduction Assessment

Yes. A leading closed recap-only block reproduces the current-main leak through both sanitizer entry points, and the PR regressions reproduce by passing an unclosed leading recap opener or a mid-message recap marker through `stripInboundMetadata()`.

## Solution Assessment

No as-is. The PR uses the right shared sanitizer boundary, but it needs to require a matching closing recap tag and return the original string when no leading recap block was actually removed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Require a closing recap tag before stripping:** `src/auto-reply/reply/strip-inbound-meta.ts:135`
  - body: If a message starts with `<session-recap>` or `<session_recap>` but never closes it, the helper scans to EOF and returns `lines.slice(i)`, erasing the whole message in both display paths. Track whether a close tag was found and return the original lines when it is absent.
  - confidence: 0.96
- **[P2] Preserve text when no recap is stripped:** `src/auto-reply/reply/strip-inbound-meta.ts:173`
  - body: A non-leading recap marker still makes `hasSessionRecap` true, so when no standard sentinel exists this branch rejoins the unchanged lines and trims user-authored surrounding newlines. Return the original string when the helper did not remove a leading recap block.
  - confidence: 0.91
