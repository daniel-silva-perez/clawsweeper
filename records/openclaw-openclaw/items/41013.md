## Summary

Keep open. Current main still does not implement this PR’s intended fix: the shared inbound metadata sanitizer and the TUI leading-only sanitizer have no `session-recap` handling, and the linked UI/TUI leak remains tracked by open issue #40948. The PR is still a focused implementation candidate, but its unresolved review comments should be addressed before landing.

## Best Possible Solution

Keep this PR open as the focused implementation candidate for #40948, but require a small revision before landing: strip only well-formed leading `<session-recap>` or `<session_recap>` blocks with a matching close tag, preserve the original string when no recap block is removed, wire the behavior into both full and leading-only cleanup paths, and add regression coverage for well-formed, malformed/no-close, mid-message, leading blank-line, and whitespace-preservation cases.
