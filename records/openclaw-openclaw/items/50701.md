## Summary

Keep open. Current main has useful transcript-backed usage improvements for active, reset, and deleted transcript files with the same sessionId, but it still does not implement the requested logical-session lineage/family rollups, current-instance versus historical-lineage mode, or long-range usage presets such as 90d/1y/all in the WebUI. The related closed, unmerged #51105 CLI export proposal would help with archival workflows but does not satisfy this WebUI usage-history request.

## Best Possible Solution

Keep this open. The best product path is to add a backend concept for durable logical usage families, expose lineage-aware usage queries alongside current-instance queries, index transcript-backed summaries across sessionId rebinding, and update the Control UI with current-instance/historical-lineage controls plus 90d/1y/all/custom date range affordances. Build it on top of the existing transcript archive support so reset/deleted usage remains counted without double-counting.

## Risks / Open Questions

- A future lineage implementation needs to preserve the existing reset/deleted archive counting behavior and avoid double-counting transcripts that are both discoverable individually and members of a logical family.
- Full transcript rescans across long-lived logical surfaces could be expensive; the best fix likely needs a cached/indexed summary rather than only synchronous UI-load scanning.
- Transcripts with unclear or missing logical-session metadata should stay isolated to avoid merging unrelated sessions under the same agent/channel label.
