## Summary

Keep open. Current main already covers the task timestamp half via the shipped #71871 work, but the Gateway doctor SecretRef false-positive path still exists: doctor decides a token is missing with the synchronous auth resolver, which ignores SecretRef token values, and then warns without trying the existing async SecretRef resolver.

## What This Changes

This PR updates Gateway doctor auth health to lazily resolve a SecretRef-backed gateway.auth.token before warning and adds doctor e2e coverage around resolved SecretRefs and non-token auth modes; its original task timestamp goal is now covered by #71871.

## Best Possible Solution

Review and land the doctor-only SecretRef fix, preserving lazy loading and resolving the token only after token auth is actually required; keep the task timestamp behavior anchored to the already shipped #71871 implementation.
