## What This Changes

This PR updates Gateway doctor auth health to lazily resolve a SecretRef-backed `gateway.auth.token` before warning, and adds doctor e2e coverage for resolved SecretRefs and non-token auth modes.

## Best Possible Solution

Review and land the doctor-only SecretRef fix, preserving lazy loading and resolving the token only after token auth is actually required; leave task timestamp normalization anchored to the already-shipped #71871 implementation.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a local token-auth config with `gateway.auth.token` as a resolvable SecretRef: current doctor uses `resolveGatewayAuth`, which discards SecretRef token values, then emits the unavailable warning before any SecretRef-aware lookup.

## Solution Assessment

Yes. The latest PR diff is the narrowest maintainable solution for the remaining doctor bug because it gates SecretRef resolution behind the token-needed decision, keeps the dependency behind a dynamic import, and adds regression coverage for both resolved SecretRefs and non-token auth modes.
