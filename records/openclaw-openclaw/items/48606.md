## Summary

Keep open. Current main still returns `null` when macOS recognizes Edge's LaunchServices bundle ID but `osascript` or `defaults` cannot resolve the app path, which lets the generic macOS fallback choose Chrome first. The PR is a focused source fix for the reported behavior, but it is dirty against main and still needs regression tests before merge.

## What This Changes

The PR adds a known macOS Chromium bundle-ID-to-executable fallback in the browser plugin when dynamic default-browser path resolution fails.

## Best Possible Solution

Land a rebased browser-plugin fallback with focused macOS tests covering Edge known-path success after failed dynamic lookup and missing Edge known-path fallback to the ordinary Chrome-first candidate list.
