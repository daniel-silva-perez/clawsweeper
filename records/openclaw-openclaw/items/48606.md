## Summary

Keep open. Current main still falls through to the generic macOS browser fallback when Edge's LaunchServices bundle ID is detected but osascript/defaults resolution fails; this PR is directionally the right fix, but it is not ready to merge because the branch is dirty against base and lacks the focused regression tests already requested in review.

## What This Changes

This PR adds a known macOS browser bundle-ID fallback in the browser plugin's default Chromium detector, including system and user Applications locations when osascript/defaults cannot resolve the configured default browser.

## Best Possible Solution

Land a rebased version of this browser-plugin fallback with focused macOS default-browser tests that prove Edge remains selected when LaunchServices reports `com.microsoft.edgemac`, dynamic app lookup fails, an Edge known path exists, and Chrome is also installed; also prove the resolver still falls back to Chrome when no known Edge executable exists.
