## Summary

Keep this PR open. Current main still does not surface Feishu WebSocket/webhook lifecycle health into gateway channel status: there is no Feishu `statusSink` option, no transport lifecycle logger, and the gateway start path does not bind `ctx.setStatus` into `monitorFeishuProvider`. The PR is not implemented on main and the related Feishu health PRs in the provided context are not a maintainer-designated canonical replacement.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to update it against current main by binding `ctx.setStatus` from `extensions/feishu/src/channel.ts` through `monitorFeishuProvider`, `monitorSingleAccount`, and both transports; wire the Feishu WS logger through `createFeishuWSClient`; emit connected/disconnected/reconnect patches that satisfy the current health policy; and keep targeted Feishu transport/webhook/channel tests.
