## Summary

Keep open for maintainer canonicalization and a security-aware rebase. The requested Feishu health reporting is still useful and not implemented on current main, but this PR head is stale against current Feishu transport code: it does not pass a gateway status sink into normal startup, it predates the merged WebSocket reconnect/heartbeat repair, and it carries older webhook signature handling. Open PR #39489 overlaps the same feature area, so maintainers should choose one canonical implementation rather than auto-closing this contribution.

## What This Changes

The PR branch adds Feishu monitor status-sink plumbing, a WebSocket lifecycle logger and logger passthrough, webhook connect/disconnect status updates, and targeted Feishu monitor/client tests.

## Best Possible Solution

Land one canonical Feishu-owned health-monitor implementation that wires an account status sink from startAccount through monitorFeishuProvider, monitorSingleAccount, event handlers, and both transports; reports connection lifecycle plus inbound activity timestamps; preserves current WebSocket reconnect/heartbeat behavior and raw-body webhook signature verification; then close or rebase overlapping Feishu health PRs.
