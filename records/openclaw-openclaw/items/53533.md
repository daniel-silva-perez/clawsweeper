## Summary

Keep open for maintainer review. Current main still does not implement Feishu connection health reporting, and this PR is useful but not merge-ready because the gateway startup path still would not supply a status sink and the branch predates current Feishu WebSocket reconnect/heartbeat work. Open #39489 overlaps the same health-monitor surface, so maintainers should choose or consolidate one canonical implementation.

## What This Changes

The PR branch adds Feishu monitor status-sink plumbing, a WebSocket lifecycle logger, logger passthrough for `createFeishuWSClient`, and targeted transport/webhook status tests.

## Best Possible Solution

Land one canonical Feishu-owned health-monitor implementation that passes the gateway account status sink from `startAccount` through `monitorFeishuProvider`, `monitorSingleAccount`, and both transports; reports lifecycle and inbound activity timestamps; preserves current WebSocket reconnect/heartbeat behavior and webhook security guards; then close or rebase the overlapping Feishu health PRs.
