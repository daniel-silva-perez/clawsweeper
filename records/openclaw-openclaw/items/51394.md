## Summary

Keep open. Current main has useful lower-level browser network inspection through the control API and CLI, which overlaps the already-closed Chrome relay request in #42369, but this issue asks for the agent-facing browser tool to expose Network-style request listing/interception. The current browser tool schema and dispatcher still do not expose a network act kind, requests/responsebody actions, or a true request-interception API.

## Best Possible Solution

Keep this open and narrow the implementation path: first expose agent-facing browser tool actions that reuse the existing /requests and /response/body control routes, with schema/docs/tests; then separately design any true interception API around Playwright routing or CDP with explicit safety limits.
