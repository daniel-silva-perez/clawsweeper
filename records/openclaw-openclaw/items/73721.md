## Summary

Keep this PR open. Current main still has the strict Control UI sessionKey gate, while chat.send can resolve a raw UI session key to a canonical gateway session key and live chat events are emitted with that canonical key. The PR remains a focused implementation candidate for the linked bug, not obsolete work.

## Best Possible Solution

The likely maintainer path is to land a scoped UI fix that accepts live delta/final events for the active client runId despite raw/canonical session-key differences, ideally with the additional final-path regression test requested in review; after merge, the linked issue can close through the PR.
