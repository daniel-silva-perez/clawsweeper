## Summary

Keep this PR open. Current main still rejects Control UI chat events on strict sessionKey mismatch before considering runId, while the gateway resolves chat.send to a canonical session key and broadcasts live chat events under that canonical key. The PR is a plausible, focused implementation candidate for the linked bug rather than obsolete work.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for the linked Control UI bug. The likely path is to land a scoped UI fix that accepts live chat delta/final events for the active client runId despite raw/canonical session-key differences, with regression coverage proving unrelated runIds are still dropped; after merge, the linked issue can close through the PR's closing reference.
