## Summary

Keep this issue open. Current main has post-connect Debug and Logs tabs plus improved single-message connection errors, but it still does not implement the requested ephemeral browser-side Control UI diagnostics mode for WebSocket lifecycle, handshake, close/error history, and copyable session-only diagnostics. The only linked implementation candidate, PR #49274, is closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep this issue open as the active tracker. The best fix is a Control UI-owned ephemeral diagnostics mode integrated into GatewayBrowserClient/app-gateway that records sanitized WebSocket open/error/close/retry events, connect.challenge/connect response metadata, structured connect errors, close codes/reasons, and post-auth logs.tail snippets when available, then exposes a copyable drawer/overlay backed only by memory or session-scoped storage.

## Risks / Open Questions

- A diagnostics drawer could leak gateway tokens, device tokens, nonces, signatures, origins, or user-identifying metadata if copy/export is not carefully redacted.
- The issue explicitly asks for ephemeral storage; an implementation should avoid localStorage and IndexedDB for diagnostic content and should use memory or at most session-scoped storage.
- The existing Logs tab already covers authenticated gateway file logs, so the missing work is browser-side connection and pre-auth lifecycle diagnostics rather than another logs.tail view.
