## Summary

Keep this issue open. Current main still has post-connect Debug and Logs tabs plus improved single connection error text, but it does not implement the requested ephemeral browser-side diagnostics mode for WebSocket lifecycle, pre-auth handshake visibility, close/error history, and copyable session-only logs. The linked implementation candidate, PR #49274, is closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep this issue open as the active tracker. The best fix is a Control UI-owned ephemeral diagnostics mode integrated into GatewayBrowserClient/app-gateway that records sanitized WebSocket open/error/close/retry events, connect.challenge/connect response metadata, structured connect errors, close codes/reasons, and post-auth logs.tail snippets when available, then exposes a copyable drawer or overlay backed only by memory or session-scoped storage.

## Risks / Open Questions

- The issue explicitly asks for ephemeral storage; an implementation should avoid localStorage and IndexedDB for diagnostic content and use memory or session-scoped storage only.
- The existing Logs tab is useful after authentication, but treating logs.tail as the whole fix would still leave pre-auth connection failures opaque.
