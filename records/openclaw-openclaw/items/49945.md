## Summary

Keep this PR open. Current main still has TUI `/new` switching to a fresh isolated `tui-*` session through local `setSession(...)` only, while `/reset` uses `sessions.reset`; the linked hook-visible `/new` regression is therefore not implemented on main, and this PR remains an active implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and review it as the implementation candidate for #49918. The best fix should make TUI `/new` emit hook-visible `command:new` behavior while preserving per-client `tui-*` isolation, keep bundled/custom hook attribution correct for `event.sessionKey` and `previousSessionEntry`, avoid creating empty persisted sessions from repeated `/new`, and either document/approve the additive gateway protocol field or solve the context problem without expanding the protocol.
