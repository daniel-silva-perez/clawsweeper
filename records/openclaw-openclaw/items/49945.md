## Summary

Keep this PR open. Current main still does not implement the intended TUI `/new` hook-visible reset behavior, and this PR remains the active implementation candidate for the linked regression report. The PR also changes Gateway reset parameters and hook attribution, so it needs maintainer review rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open and review it as the implementation candidate for #49918. The best fix should make `/new` emit `command:new` through the normal hook-visible reset path while preserving per-client `tui-*` isolation, keep `event.sessionKey` and `previousSessionEntry` semantics correct for bundled hooks, avoid cluttering session history with blank sessions, and explicitly approve, document, or avoid any Gateway protocol addition.
