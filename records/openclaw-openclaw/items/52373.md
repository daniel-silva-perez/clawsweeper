## Summary

Keep this issue open. Current main still does not implement a `configUrl`/`configButton` SKILL.md metadata contract, does not expose such fields through `skills.status`, and does not render a dedicated configuration action in the Control UI Skills list. This is actionable core/dashboard work, not a ClawHub-only skill request, because external skills cannot make OpenClaw parse, validate, transport, and render new metadata without a core contract.

## Best Possible Solution

Keep the issue open and implement a small reviewed core/dashboard contract: add optional validated configuration-link metadata to SKILL.md parsing/types, include it in `skills.status` and UI types, render a safe external configuration action in the Skills list or detail, and update the skills docs plus parser/status/UI tests. External skills can then use the surface without adding skill-specific behavior to core.
