## Summary

Keep this issue open. Current main has skill `homepage`, installer, requirement, and API-key setup metadata, but it does not implement the requested `configUrl` or `configButton` contract or a Skills-list configuration button in the Web Dashboard. This is actionable core/dashboard work because external skills cannot make the Control UI parse, expose, and render new SKILL.md metadata without a core status/API/UI change.

## Best Possible Solution

Keep the issue open and implement a small reviewed core/dashboard contract: add optional validated configuration link metadata to SKILL.md parsing, include it in `skills.status` and UI types, render a safe external action in the Skills list/detail, and update docs plus parser/status/UI tests. External skills can then use the surface without adding bundled skills to core.
