## Summary

Keep this PR open. Current main still has the reported explicit `openclaw crestodian` no-TTY behavior: it logs the interactive-TTY error and returns normally, while the command wrapper only exits non-zero when an error is thrown. The PR is a focused implementation candidate for the linked open bug, and current main does not already implement it.

## Best Possible Solution

Keep this PR open as the focused implementation candidate for the linked bug. The best path is to review and land a small fix that makes explicit `openclaw crestodian` fail non-zero in no-TTY contexts while preserving `--message` and `--json`, with the added regression test or an equivalent CLI-level exit-code test.
