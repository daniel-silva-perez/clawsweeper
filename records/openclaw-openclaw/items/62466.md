## Summary

Keep open. Current main still does not gracefully handle read offsets beyond EOF: OpenClaw wraps the pinned upstream read tool, and the wrapper still forwards both explicit-limit and adaptive paging reads to the upstream `execute` path without catching the upstream EOF-offset error. The related fix PR #62491 is useful context, but it was closed unmerged and its helper/tests are not present on current main.

## Best Possible Solution

Revive or reimplement the focused approach from PR #62491 in `src/agents/pi-tools.read.ts`: catch the pinned upstream EOF-offset error around `base.execute`, parse the total line count, retry with a clamped offset or return accumulated content for mid-pagination EOF, and add colocated regression tests for no-limit, explicit-limit, far-beyond-EOF, and sandboxed read cases.
