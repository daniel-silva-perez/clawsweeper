## Summary

Keep open. Current main still delegates out-of-range read offsets to upstream `@mariozechner/pi-coding-agent`, whose pinned 0.70.6 read implementation throws the reported error, and the directly related fix PR was closed without merging. This is a valid, narrow read-tool bug suitable for a replacement fix PR.

## What This Changes

The issue asks for the read tool to handle offsets beyond EOF gracefully by truncating or auto-adjusting to a valid line instead of returning an error.

## Best Possible Solution

Add a narrow OpenClaw read-tool wrapper fix that catches only the upstream offset-beyond-EOF error, clamps to the final valid line for the initial call, preserves existing errors for unrelated failures, returns already aggregated content for mid-pagination EOF, and adds focused regression coverage for explicit-limit and adaptive read paths.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: with the pinned upstream 0.70.6 read tool, any text file read with `offset` greater than the split line count throws the exact reported error, and current main does not catch that error in either active wrapper path.

## Solution Assessment

Yes. Handling this in `src/agents/pi-tools.read.ts` is the narrowest maintainable fix because OpenClaw already wraps the upstream read tool for OpenClaw-specific behavior and can translate this strict upstream EOF error without changing broader filesystem or provider contracts.
