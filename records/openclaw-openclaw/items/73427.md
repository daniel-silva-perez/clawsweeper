## Summary

Keep this issue open. Current main still has the reported OpenAI browser Talk gaps: the Control UI CSP omits the OpenAI HTTPS origin, the WebRTC transport still posts SDP directly to OpenAI, OpenAI realtime data-channel errors and lifecycle events are dropped, and the browser-session payload still lacks explicit input audio/VAD/transcription configuration.

## What This Changes

The issue requests fixes for Control UI OpenAI Realtime Talk so CSP allows the WebRTC SDP call path and the UI/provider surface realtime audio errors, lifecycle status, and input audio configuration.

## Best Possible Solution

Fix the bundled OpenAI browser Talk path in one focused PR: allow the exact OpenAI realtime HTTPS origin or proxy the SDP exchange through the Gateway, add OpenAI data-channel error and basic lifecycle status handling, configure input audio/VAD/transcription in the browser-session minting payload where appropriate, and cover those paths with targeted CSP/provider/UI tests plus docs/changelog updates.
