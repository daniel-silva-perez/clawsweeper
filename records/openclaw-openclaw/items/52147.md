## Summary

Keep open. Current main mitigates the original 600s default repro by making the agent runtime timeout 48 hours and adding a separate model idle watchdog, but the specific misclassification remains: a non-compaction run timeout is still treated as an assistant timeout failover signal, with no active-tool timeout cause tracked.

## Best Possible Solution

Add phase-aware timeout state in the embedded runner, preferably an active tool-call count or a timeout-cause enum. When the global run timer fires during active tool execution, surface a run/tool timeout or partial-progress result without rotating auth profiles or falling back models; keep model idle/request timeouts eligible for existing failover. Add focused tests around `resolveRunFailoverDecision` and an embedded attempt/tool-timeout scenario.
