## Summary

Keep this PR open. Current main still does not implement context provenance for injected bootstrap segments, and the PR remains useful source work for the open RFC, but it needs maintainer product/API review plus a security/rebase fix before merge.

## What This Changes

The PR extends embedded context files with optional provenance metadata, tags bootstrap context entries, renders provenance comments and guidance in the agent system prompt, and adds focused tests.

## Best Possible Solution

Rebase the provenance design onto the current buildProjectContextSection and stable-vs-dynamic cache-boundary layout, runtime-normalize every rendered provenance field, and resolve the Phase 1 API shape against #54373 before merge.

## Reproduction Assessment

Yes for the review finding: a runtime caller can pass a contextFiles entry whose provenance.volatile is not a boolean via an unknown cast, and the PR renderer interpolates it into the prompt comment. For the feature request itself, reproduction is not applicable; source inspection is enough to show current main lacks the requested provenance surface.

## Solution Assessment

No, not as currently proposed on this branch. The direction matches the RFC, but the branch should first rebase onto current main's project-context helper/cache-boundary split and coerce or validate all provenance attributes before rendering.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Coerce volatile before rendering provenance tags:** `src/agents/system-prompt.ts:628`
  - body: The renderer escapes source and injectedAt, but still interpolates file.provenance.volatile directly into an HTML comment. buildAgentSystemPrompt accepts runtime contextFiles, and TypeScript types are erased, so a malformed non-boolean value can close the comment and inject unintended prompt text. Normalize it to a strict boolean string before building the tag.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The PR introduces a prompt-rendered metadata surface with one remaining runtime validation gap.

Concerns:

- **[medium] Unvalidated volatile value can break provenance comment:** `src/agents/system-prompt.ts:628`
  - body: volatile is embedded directly in the provenance HTML comment while the other fields are escaped. Runtime callers can still supply malformed data despite the TypeScript type, so the renderer should coerce or validate it before interpolation.
  - confidence: 0.78
