## Summary

Keep open: current main still does not carry context-provenance metadata through bootstrap context or render provenance comments in the prompt, and the linked RFC remains open. The PR is still relevant, but it needs maintainer review and a small prompt-integrity fix before it should merge.

## What This Changes

The PR adds optional provenance metadata to embedded context files, annotates bootstrap context entries, renders provenance comments and guidance in the agent system prompt, and adds focused tests.

## Best Possible Solution

Rebase the provenance design onto the current prompt-context helper and cache-boundary layout, validate all rendered provenance fields at runtime, and keep the linked RFC as the product/design anchor until Phase 1 is accepted or explicitly declined.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Coerce volatile before rendering provenance tags:** `src/agents/system-prompt.ts:628`
  - body: The patch escapes `source` and `injectedAt`, but still string-interpolates `file.provenance.volatile` into an HTML comment attribute. `buildAgentSystemPrompt` receives caller-supplied `contextFiles` at runtime, so a malformed non-boolean value can break the comment and inject unintended prompt text; normalize it to a strict boolean before rendering.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The PR introduces a prompt-rendering metadata surface with one remaining runtime validation gap.

Concerns:

- **[medium] Unvalidated volatile value can break provenance comment:** `src/agents/system-prompt.ts:628`
  - body: `volatile` is embedded directly in the provenance HTML comment while other fields are escaped. TypeScript marks it boolean, but runtime callers can still supply malformed data, so the renderer should coerce or validate it before interpolation.
  - confidence: 0.78
