## Summary

Keep open. The underlying Google Chat off-mode threading bug is still present on current main, but this PR should not merge as-is because it suppresses final `payload.replyToId` delivery and would drop explicit reply targets that the shared reply-threading contract intentionally preserves in `replyToMode: "off"`.

## What This Changes

The PR adds Google Chat `replyToMode: "off"` guards around inbound thread context, typing placeholder sends, final text/media sends, and adds config-resolution tests for channel-level and per-account off mode.

## Best Possible Solution

Revise or fold this work into the current Google Chat threading changes: suppress only implicit inbound `ReplyToId`/`ReplyToIdFull` and typing-placeholder thread propagation when the resolved Google Chat account has `replyToMode: "off"`, keep final delivery honoring already-filtered `payload.replyToId`, and add a focused monitor regression for off-mode DM replies plus explicit reply preservation.

## Review Findings
