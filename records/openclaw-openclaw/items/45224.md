## Summary

Keep open. The merged browser dialog-race fix covers `Page.handleJavaScriptDialog` / `No dialog is showing`, but current main still leaves generic Playwright CDP assertion rejections like the reported `CRSession._onMessage` `Error: Assertion error` outside the browser-owned handler and outside the global non-fatal classifiers.

## What This Changes

The issue asks for Playwright CDP assertion failures from browser control to be handled or isolated so they degrade browser capability instead of crashing the Gateway and terminating unrelated sessions or cron jobs.

## Best Possible Solution

Extend the browser plugin's existing unhandled-rejection lifecycle to recognize narrowly defined Playwright/CDP assertion shapes, optionally evict or close the affected CDP connection best-effort, and keep core's generic fatal unhandled-rejection policy intact. Treat browser subprocess isolation as a separate roadmap/design item.
