## Summary

Keep open. Current main still only suppresses the browser dialog-race Playwright rejection, while the reported generic `CRSession._onMessage` CDP assertion remains an unclassified `Error: Assertion error` shape that reaches the Gateway's fatal unhandled-rejection fallback. The merged dialog-race work in #73983 is useful adjacent context but does not cover this stack, and the related worker-target PR #46259 was closed unmerged.

## What This Changes

The issue asks OpenClaw Gateway to avoid process exit when Playwright raises a CDP `CRSession._onMessage` assertion from browser control.

## Best Possible Solution

The desired end state is a browser-owned stability boundary where narrowly identified Playwright CDP assertion rejections degrade or reset the affected browser connection while unrelated Gateway sessions and cron work keep running, with core's generic fatal unhandled-rejection policy preserved for unknown errors.
