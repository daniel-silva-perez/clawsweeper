## Summary

Keep this issue open. Current main has a browser-owned handler for the separate Playwright dialog race, but it intentionally does not match generic Playwright/CDP assertion failures like the reported `CRSession._onMessage` `Assertion error`; unclassified unhandled rejections still take the Gateway down, and the related #45791/#52051/#46259 chain is closed or unmerged rather than a live canonical tracker.

## What This Changes

The issue requests that Playwright CDP assertion failures from browser control be isolated or handled so they log/recover instead of crashing the Gateway and killing unrelated sessions or cron jobs.

## Best Possible Solution

Fix the remaining crash in the browser plugin by extending the existing browser-owned unhandled-rejection seam to recognize narrowly defined Playwright/CDP assertion shapes, evict or close the affected Playwright CDP connection when possible, and keep core’s generic fatal policy intact. Treat browser/CDP subprocess isolation as a separate maintainer roadmap decision.
