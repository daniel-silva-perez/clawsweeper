## Summary

Keep open. Current main still suppresses only the browser dialog-race Playwright rejection, while the reported `CRSession._onMessage` / generic CDP assertion shape still falls through the fatal Gateway unhandled-rejection path.

## What This Changes

The issue asks for Playwright CDP assertion failures from browser control to be handled or isolated so the Gateway does not crash and terminate unrelated sessions or cron jobs.

## Best Possible Solution

Extend the browser plugin's existing unhandled-rejection lifecycle to recognize narrowly defined Playwright/CDP assertion shapes, optionally evict the affected cached CDP connection best-effort, and keep core's generic fatal unhandled-rejection policy intact.
