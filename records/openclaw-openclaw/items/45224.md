## Summary

Keep open. Current main still treats a plain Playwright `CRSession._onMessage` assertion as an unknown unhandled rejection that exits the Gateway unless a registered plugin handler classifies it, and the current browser plugin handler only covers the merged dialog-race case. The reporter’s production impact and follow-up workaround notes remain relevant, while #73983 is adjacent but not sufficient for this CDP session assertion path.

## What This Changes

The issue asks OpenClaw to prevent Playwright CDP `CRSession._onMessage` assertion failures from crashing the entire Gateway process.

## Best Possible Solution

Add a browser-owned stability boundary for this narrowly identified Playwright CDP session assertion: classify direct and wrapped `CRSession._onMessage` / `CRConnection._onMessage` assertion rejections, log and evict or reset the affected browser connection, and preserve core’s fatal fallback for unknown unhandled rejections. The broader subprocess-isolation roadmap can remain a separate product decision.
