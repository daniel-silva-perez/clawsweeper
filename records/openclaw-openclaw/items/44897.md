## Summary

Keep open. Current main still has the observable hanging path because `loadChatHistory` awaits `chat.history` without a timeout for a never-settling RPC, but this PR should not land as-is because its timeout leaves the underlying Gateway request in the pending map until a response frame or socket close. The branch is also stale against newer WebChat history reconciliation guards, so a current-main-native repair is the right path.

## What This Changes

The PR changes `ui/src/ui/controllers/chat.ts` to race the initial `chat.history` load against a 10-second timeout, clear the timer, late-apply delayed successes under freshness guards, and invalidate that freshness token when sending chat messages.

## Best Possible Solution

Land a current-main-native timeout path for initial WebChat history loads that dismisses the empty loading state, explicitly cancels or removes the timed-out Gateway pending RPC, preserves the existing startup retry and request-version/session/stream/optimistic-tail guards, and adds focused fake-timer coverage for both UI state and pending cleanup.
