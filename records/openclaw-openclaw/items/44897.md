## Summary

Keep open. Current main still has the reported Control UI failure mode because `loadChatHistory` can await an unresolved `chat.history` RPC indefinitely, but this PR should not merge as-is because its timeout only races the promise and leaves the underlying browser RPC pending until a response or socket close.

## What This Changes

This PR changes `ui/src/ui/controllers/chat.ts` to wrap the initial `chat.history` load in a 10-second timeout, clear the timer, late-apply delayed successful responses under guards, and invalidate a freshness token when sending a chat message.

## Best Possible Solution

Land a current-main-native WebChat history timeout that exits the empty loading state, reports a targeted timeout error, explicitly removes or cancels timed-out browser RPCs, preserves per-state request-version/session/startup-retry/optimistic-tail behavior, and adds focused fake-timer coverage for timeout state and pending cleanup.
