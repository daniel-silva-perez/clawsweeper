## Summary

Keep open. Current main still has the unbounded initial `chat.history` wait, so the user-visible loading hang is not implemented on main, but this PR needs changes before merge because the timeout branch leaves the underlying Gateway pending RPC alive and the branch predates newer current-main history guards.

## What This Changes

The PR modifies `ui/src/ui/controllers/chat.ts` to race `chat.history` against a 10-second timeout, clear the timer, late-apply delayed successful responses under guards, and invalidate that freshness token when sending messages.

## Best Possible Solution

Land a current-main-native WebChat history timeout that exits the empty loading state, reports a targeted timeout error, explicitly removes or cancels the timed-out browser RPC, preserves the existing request-version/session/startup-retry/optimistic-tail contracts, and adds focused fake-timer coverage for timeout state and pending cleanup.
