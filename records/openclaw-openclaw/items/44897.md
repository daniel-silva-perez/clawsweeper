## Summary

Keep open. Current main still has the reported Control UI failure mode because `loadChatHistory` awaits `chat.history` without a bounded timeout, but this PR should not merge as-is: its timeout path leaves the underlying browser RPC pending and the branch is stale relative to current history request guards, startup retry, and optimistic-tail preservation.

## What This Changes

The PR modifies `ui/src/ui/controllers/chat.ts` to race the initial `chat.history` load against a 10-second timeout, clear the timer, late-apply delayed successful responses under guards, and invalidate the freshness token when sending a chat message.

## Best Possible Solution

Land a current-main-native WebChat history timeout that exits the empty loading state, reports a targeted timeout error, explicitly removes or cancels timed-out browser RPCs, preserves per-state request-version/session/startup-retry/optimistic-tail behavior, and adds focused fake-timer coverage for timeout state and pending cleanup.
