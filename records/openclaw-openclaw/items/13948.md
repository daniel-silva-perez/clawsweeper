## Summary

Keep open. Current main still enforces tool policy at whole-tool granularity, and the existing maintainer/ClawSweeper discussion keeps this as the canonical tracker for related gateway action-level requests.

## What This Changes

Request action-aware tool policy so operators can deny mutating actions such as message sends or gateway config writes while leaving read/search/diagnostic actions available.

## Best Possible Solution

Define a maintainer-approved typed action-policy contract with documented grammar, wildcard behavior, precedence across global/provider/agent/group/sandbox/subagent scopes, and per-call enforcement for multiplexed tools while keeping partially allowed tools discoverable.
