## Summary

Keep this issue open. Current main has the Control UI exec approval modal, countdown, queue indicator, security/ask/context rows, scrollable command block, and allow-once/allow-always/deny buttons, but the requested command syntax highlighting remains missing: the command is still rendered as one escaped Lit text interpolation in a monospace block.

## Best Possible Solution

Keep this issue open and finish the remaining Control UI polish in `ui/src/ui/views/exec-approval.ts`: render the sanitized command through a small safe shell-token highlighter, keep all content escaped, preserve the existing countdown/actions/context rows and scroll behavior, and add focused UI coverage for highlighted command output plus security/ask/host context labels.
