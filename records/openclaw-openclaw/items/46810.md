## Summary

Keep this issue open. Current main has a Control UI exec approval modal with a countdown, queueing, host/security/ask metadata, and allow-once/allow-always/deny buttons, but the requested command syntax highlighting is not implemented; the command is rendered as plain escaped text in a monospace block. Related closed reports cover scrollability/button reachability, not the remaining highlighting request.

## Best Possible Solution

Keep the issue open and implement the missing Control UI polish in `ui/src/ui/views/exec-approval.ts`: render shell commands through a small safe highlighter or structured command-token renderer, preserve escaping, keep the existing scrollable modal and decision buttons, and add focused UI tests for highlighted command output plus context labels.
