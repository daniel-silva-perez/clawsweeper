## Summary

Keep this issue open. Current main has the Control UI exec approval modal, countdown, queue display, context metadata, and allow-once/allow-always/deny buttons, but the requested command syntax highlighting is still absent: the command is rendered as a plain Lit text node inside a monospace block.

## Best Possible Solution

Keep the issue open and implement the remaining Control UI polish in `ui/src/ui/views/exec-approval.ts`: render the already-sanitized command through a small safe shell-token renderer or highlighter, keep all content escaped, preserve the existing scrollable modal/actions/context rows, and add focused UI coverage for highlighted command output plus security/ask/host context labels.
