## Summary

Keep this issue open. Current main has the Control UI exec approval modal, countdown, context rows, queue indicator, scrollable command block, and allow-once/allow-always/deny buttons, but the requested command syntax highlighting is still missing: the command is rendered as one plain Lit text interpolation with no tokenization or highlight classes.

## What This Changes

Request a Control UI exec approval modal that shows the full command with syntax highlighting, execution context, approve/deny actions, allow-always support, and timeout status.

## Best Possible Solution

Finish the remaining Control UI polish by adding a small escaped shell-token highlighter for the sanitized command text, styling those token spans, and covering the highlighted render path while leaving the existing modal behavior and gateway protocol unchanged.

## Reproduction Assessment

Yes. A pending exec approval in current main will render through `renderExecBody`, which displays the command as one plain `.exec-approval-command` text node; source and release-tag inspection reproduce the missing highlighting without needing a live gateway run.

## Solution Assessment

Yes. The narrowest maintainable solution is UI-only: tokenize the already-sanitized display string into escaped Lit template parts, preserve the existing countdown/actions/context rows and scroll behavior, and avoid protocol or gateway changes.
