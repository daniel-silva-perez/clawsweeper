## Summary

Current main does not yet implement this PR’s intended cleanup fix. The reset path still derives a non-topic fallback transcript path during cleanup, while topic session creation uses topic-aware filenames; the PR is also likely incomplete for restored thread sessions where `MessageThreadId` is absent and the durable thread id lives on the session key or stored entry.

## Best Possible Solution

Keep this PR open for revision. The best fix is to update the current `resetReplyRunSession` cleanup path to derive the same topic/thread id used for transcript creation, including persisted/session-key fallback where appropriate, and add focused regression tests for topic cleanup with `MessageThreadId` present plus restored/stale-entry cases where it is absent.
