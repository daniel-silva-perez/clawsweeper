## Summary

Keep this PR open. Current main forwards an explicit ACP `runTimeoutSeconds`, but it still does not implement the PR's central config fallback: `acp.defaultRunTimeoutSeconds` is absent from the public ACP config type, strict schema, generated metadata/help/labels/docs, and the ACP spawn path. The linked feature request was closed stale rather than shipped, so this PR remains a focused implementation candidate after rebase.

## Best Possible Solution

Keep this PR open and ask for a focused rebase onto current main. The maintainer path is to keep the already-shipped explicit ACP timeout forwarding, add `acp.defaultRunTimeoutSeconds` to the public config type, strict Zod schema, generated schema metadata, help/labels, and user docs, then resolve ACP timeout as explicit per-call value -> ACP config default -> `0`. Tests should cover omitted config default, explicit nonzero override, and explicit `0` overriding a nonzero config default across ACP runtime initialization and Gateway dispatch.
