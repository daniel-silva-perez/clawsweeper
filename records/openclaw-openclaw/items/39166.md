## Summary

Keep this PR open. Current main still lacks the requested ACP config-level timeout default, but the branch is not merge-ready because the resolved default is only applied to the Gateway dispatch path, while ACP turn timeouts are read from persisted runtime options, and the public config generated artifacts are not updated.

## What This Changes

The branch adds optional `acp.defaultRunTimeoutSeconds`, forwards `runTimeoutSeconds` through ACP `sessions_spawn`, resolves ACP spawn timeout as explicit value to config default to `0`, and updates config metadata plus tests.

## Best Possible Solution

Land a rebased ACP-owned config default that updates the schema/type/help/label/generated-schema/docs-baseline surfaces, resolves the timeout once as explicit input to `acp.defaultRunTimeoutSeconds` to `0`, applies that resolved value consistently to ACP runtime initialization and Gateway dispatch, preserves explicit override and explicit `0` semantics, and documents the new ACP-specific default.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main has no `acp.defaultRunTimeoutSeconds`, and the PR diff resolves the config default for dispatch while ACP turn timeout is sourced from persisted runtime options.

## Solution Assessment

No. The requested direction is narrow and maintainable, but this branch is not the best merge shape until it propagates the resolved timeout into runtime initialization and commits the generated config schema/docs baseline updates.
