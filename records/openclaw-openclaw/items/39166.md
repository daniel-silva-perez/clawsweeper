## Summary

Keep this PR open. Current main still lacks `acp.defaultRunTimeoutSeconds`, so the central feature is not implemented, but the branch is not merge-ready because the resolved ACP default is only applied to the initial Gateway dispatch, generated config artifacts are missing, and the config docs baseline would drift.

## What This Changes

The PR adds optional `acp.defaultRunTimeoutSeconds`, threads `runTimeoutSeconds` through ACP `sessions_spawn`, resolves ACP spawn timeout as explicit input to config default to `0`, and updates config metadata plus tests.

## Best Possible Solution

Land a rebased ACP-owned config default that adds the schema/type/help/label/generated-schema/docs-baseline surfaces, resolves timeout once as explicit input to `acp.defaultRunTimeoutSeconds` to `0`, applies the resolved value consistently to ACP runtime initialization and Gateway dispatch, preserves explicit override and explicit `0` semantics, and updates the ACP sessions docs for the new default.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main has no `defaultRunTimeoutSeconds` key, and the PR diff leaves ACP runtime initialization on the raw parameter path while ACP turn timeout is read from persisted runtime options.

## Solution Assessment

No. The requested direction is narrow and appropriate, but this branch is not the best merge shape until it propagates the resolved timeout into runtime initialization and commits the generated config schema/docs baseline updates.
