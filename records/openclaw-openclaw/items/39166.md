## Summary

Keep this PR open. Current main already forwards explicit ACP runTimeoutSeconds into the ACP spawn path, but it still does not implement the PR's central acp.defaultRunTimeoutSeconds config fallback. The linked feature request #38419 is closed as superseded/stale rather than shipped, so this remains a focused but stale implementation candidate for maintainer review and rebase.

## Best Possible Solution

Keep this PR open for maintainer review or a focused rebase. The remaining useful delta is to add acp.defaultRunTimeoutSeconds to AcpConfig, strict Zod schema, generated schema metadata, help/labels, and user docs, then resolve ACP timeout as explicit tool value -> ACP config default -> 0 in the ACP spawn path. Regression coverage should include omitted value using config default, explicit nonzero override, explicit 0 overriding a nonzero default, and schema acceptance/rejection for non-negative integer behavior.
