## Summary

Current main still has the reported Control UI bug: the cron form rejects `timeoutSeconds: 0` and would omit an explicit zero from the submitted payload. Backend/API paths already accept and preserve zero as no-timeout, and an open paired PR still references this issue with closing syntax, so this should remain open until that fix or an equivalent lands.

## Best Possible Solution

Review and land #41275, or an equivalent maintainer patch, updating `validateCronForm` to allow zero, preserving an explicitly entered zero in `buildCronPayload`, and changing the UI regression test so negative values fail while zero is accepted. Close this issue only after the paired fix merges.
