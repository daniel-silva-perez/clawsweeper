## Summary

Current main still has the reported Control UI bug: the cron form rejects `timeoutSeconds: 0`, the submit payload only preserves positive timeout values, and the UI test suite still encodes zero as invalid. The runtime/protocol side already accepts zero as no-timeout, so the remaining work is a UI/controller mismatch. Same-author PR #41275 remains open with closing syntax, so this issue should stay open until that PR or an equivalent complete fix lands.

## Best Possible Solution

Keep this issue open and review, update, or replace PR #41275. A complete fix should allow zero in `validateCronForm`, preserve explicit `timeoutSeconds: 0` in `buildCronPayload`, update Control UI regression tests so zero is accepted and negative values are rejected, and align timeout help/error copy with the existing gateway/runtime no-timeout contract. No core protocol change appears necessary.
