## Summary

Current main still reproduces the reported Control UI/backend contract mismatch: the cron UI rejects `timeoutSeconds: 0` during validation, drops explicit zero from the submit payload, and its regression test still expects zero to be invalid. The backend/gateway contract accepts and preserves zero as no-timeout, and same-author PR #41275 remains open with closing syntax for this issue, so this should stay open until that PR or an equivalent complete fix lands.

## Best Possible Solution

Keep this issue open and review or replace PR #41275. A complete fix should allow zero in `validateCronForm`, preserve explicit `timeoutSeconds: 0` in `buildCronPayload`, update the Control UI regression tests, and align timeout help/error text with the gateway/runtime no-timeout contract.
