## Summary

Current `main` still has the reported Control UI mismatch: the cron form rejects `timeoutSeconds: 0`, drops explicit zero from submitted payloads, and its UI regression test still expects zero to be invalid, while the gateway/runtime cron contract accepts zero as no-timeout. Same-author PR #41275 remains the open implementation candidate, so this issue should stay open until that PR or an equivalent complete fix lands.

## Best Possible Solution

Keep this issue open and review or replace PR #41275. A complete fix should allow zero in `validateCronForm`, preserve explicit `timeoutSeconds: 0` in `buildCronPayload`, update the Control UI regression tests, and align timeout help/error text with the gateway/runtime no-timeout contract.
