## Summary

Keep this PR open. Current main still has the live `openclaw channels status --probe` allowFrom truncation this PR targets, and the latest release tag still has the same two-entry display. The PR is a focused formatting fix, but it overlaps with #51179 on the config-summary path and should settle the output/privacy policy plus tests before landing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to consolidate it with #51179 or choose one canonical patch, add a shared allowlist display helper used by both the live gateway formatter and config-summary path, cover 2-entry, 3-entry, exact-limit, and overflow cases in tests, and decide whether status output should show all entries or a capped/redacted marker.
