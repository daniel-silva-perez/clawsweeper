## Summary

Keep this PR open. Current main still has the exact live `channels status --probe` truncation the PR targets, and the latest release tag still contains the same two-entry display. The patch is a plausible focused fix, but it should be reviewed with overlapping PR #51179, with tests and an explicit display/privacy policy before landing.

## Best Possible Solution

Keep this PR open for maintainer review.
