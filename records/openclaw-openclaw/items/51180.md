Keep this PR open. Current main still has the exact live `channels status --probe` truncation the PR targets, and the latest release tag still contains the same two-entry display. The patch is a plausible focused fix, but it should be reviewed together with overlapping PR #51179 and should settle the display/privacy policy before landing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to consolidate it with #51179 or choose one canonical patch, add a shared allowlist display helper used by the live formatter and summary paths, cover 2-entry, 3-entry, exact-limit, and overflow cases in tests, and decide whether status output should show all entries or a clear capped/redacted marker.
