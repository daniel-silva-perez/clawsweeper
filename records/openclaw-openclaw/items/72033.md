## Summary

Keep open. Current main still does not implement or advertise `diagnostics.pricing`, and the PR remains a focused Gateway observability change rather than cleanup-close material. ## What This Changes

The PR adds a read-scoped `diagnostics.pricing` Gateway RPC, advertises it in method discovery, registers it under `operator.read`, and adds diagnostics handler tests for empty and populated pricing-cache metadata.

## Best Possible Solution

## Solution Assessment
