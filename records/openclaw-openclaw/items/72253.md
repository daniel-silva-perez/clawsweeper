## Summary

Keep open. Current main still treats outputful structured `code 0` and `signal SIGTERM` exec completions as relayable heartbeat events, so the PR's core user-visible fix is not implemented yet; the branch remains useful but needs maintainer review, rebase, and a changelog entry before merge.

## What This Changes

The PR adds local-exec provenance to async exec completion events, filters trusted structured `code 0` and `signal SIGTERM` completions out of heartbeat user relay, adjusts ack/reasoning suppression, JSON-encodes relayable exec prompt data, and adds focused regression tests.

## Best Possible Solution

Land a rebased narrow heartbeat fix that marks only trusted local structured `code 0` and benign cleanup `signal SIGTERM` exec completions internal, preserves relay for actionable failures, records the user-facing fix in the changelog, and leaves the broader event-audience model to #69492 unless maintainers choose to fold this slice into that design.

## Reproduction Assessment

Yes. Enqueue the three structured exec completion shapes from the PR body into a heartbeat session; current main's filter, runner, and tests show outputful `code 0` and `SIGTERM` completions still take the relayable exec path while `code 1` failures are also relayable.

## Solution Assessment

Yes, with rebase and policy caveats. The PR's local-exec provenance plus per-event filtering is the narrow maintainable tactical fix, but SIGTERM suppression semantics and current-main integration need maintainer confirmation before merge.
