## Summary

Keep open. Current main implements substantial adjacent memory search, opt-in transcript indexing, usage/cost observability, startupContext, model prewarm, and BOOT.md startup hooks, but the umbrella still has meaningful remaining product/API work: an agent-facing self-evaluation metrics API, documented normal-session conversation chaining/compact-summary handoff, and explicit gateway-restart preload of core context before the first user turn.

## Best Possible Solution

Keep this open as the canonical umbrella unless a maintainer splits it into narrower issues. Preserve the shipped memory/search and observability work, then design the remaining surfaces explicitly: an agent-facing self-evaluation metrics API, documented continuation or compact-summary handoff semantics for normal sessions, and clear gateway-startup preload/prewarm behavior before first user input.
