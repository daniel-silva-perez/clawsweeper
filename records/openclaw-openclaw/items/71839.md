## Summary

Keep this PR open. Current main still has the two user-visible behaviors this PR is meant to change: Telegram draft streams receive an implicit reply target when reply threading is enabled, and the assistant prompt still tells models to prefer `[[reply_to_current]]`. The PR is therefore an active implementation candidate rather than cleanup-closeable. Security review found no workflow, dependency, lockfile, package publishing, install/release script, secret-handling, permission, or artifact-download changes in the PR diff, but the bundled QA-lab mock-provider change is a scope question for maintainer review.

## Best Possible Solution

Keep this PR open for normal maintainer review. If maintainers agree with the Telegram behavior change, land the draft-stream and prompt-guidance fix with the focused regression coverage, while splitting or clearly justifying the QA-lab mock-provider follow-up so unrelated QA behavior does not ride along unnoticed.
