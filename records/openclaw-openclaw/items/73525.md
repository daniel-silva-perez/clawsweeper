## Summary

Keep open. Current main still has the reported split: top-level `openclaw status` fetches the live `channels.status` payload only for warning overlays, then renders the visible Channels table from the separate read-only discovery path. The linked fix PR #73662 is open and uses closing syntax, so this issue should remain open until that PR is merged, closed, or superseded.

## What This Changes

Make the top-level `openclaw status` Channels table show the configured/running channel accounts that `openclaw channels status` already reports.

## Best Possible Solution

Review and land #73662, or an equivalent narrow fix, so reachable-gateway `channels.status.channelAccounts` can populate missing top-level status Channels rows while preserving the read-only discovery table as the unreachable-gateway fallback.
