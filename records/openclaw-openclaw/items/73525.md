## Summary

Keep open. Current main still keeps default text status off live channel checks and setup fallback, and the overview path returns the read-only Channels table without merging gateway account rows. The report is also paired with open closing PR #73662, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Request that `openclaw status` populate the Channels table for configured and running channels consistently with `openclaw channels status` instead of rendering headers only.

## Best Possible Solution

Keep the fast status contract while guaranteeing visible rows for configured channels: either make the read-only/setup discovery path produce generic configured-channel rows in default status, or explicitly merge reachable gateway account snapshots for missing rows with tests covering default, `--deep`, and unreachable-gateway behavior. Review or adjust PR #73662 rather than starting parallel work.

## Reproduction Assessment

Yes. The issue has concrete command steps plus an independent Discord/Telegram confirmation, and current code/tests show the same split: default status skips live channel status and setup fallback while `channels status` renders gateway `channelAccounts`.

## Solution Assessment

Unclear until PR #73662 is reviewed. Gateway-snapshot synthesis is a plausible narrow fix for reachable-gateway cases, but maintainers should confirm it also covers plain default status and preserves the documented fast read-only behavior or intentionally changes that contract.
