## Summary

Keep open. Current main still leaves plain `openclaw status` on the read-only channel table path, does not merge the live `channels.status` account payload into visible rows, and has an open closing PR (#73662) for this exact display bug.

## What This Changes

The report asks for `openclaw status` to populate its Channels table consistently with `openclaw channels status` when channels are configured and running.

## Best Possible Solution

The maintained behavior should show channel rows in the status overview whenever configuration or reachable gateway state proves enabled channels exist, while preserving the unreachable-gateway read-only fallback and richer plugin-owned summaries.
