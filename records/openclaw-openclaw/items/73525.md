## Summary

Keep open. Current main still has the reported split: `openclaw status` builds the visible Channels table from read-only channel discovery, while `openclaw channels status` renders the live gateway `channelAccounts` payload. The discussion also has an open implementation candidate, #73662, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that `openclaw status` populate its Channels table when configured/running channels are already visible through `openclaw channels status`.

## Best Possible Solution

Keep the report open and resolve it by reviewing and landing #73662 or an equivalent narrow fix that fills missing visible Channels rows from the existing gateway status snapshot while preserving the fast read-only and unreachable-gateway behavior.
