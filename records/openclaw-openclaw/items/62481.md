## Summary

Keep open. Current main still has the catalog path that can surface a native provider entry beside an OpenRouter-qualified model, and this PR is a focused implementation attempt for the still-open #62317 rather than obsolete cleanup. The patch needs maintainer review/rebase, especially because current main has evolved around `loadModelCatalog`, but the intended bug is not implemented on main.

## Best Possible Solution

Keep this PR open or consolidate it with #62323, then land a current-main model-catalog filter that removes only unconfigured shadow native entries when an aggregator-qualified entry already represents the same underlying model, while preserving explicitly configured, implicit, and merge-retained providers. Add regression coverage for the OpenRouter-prefixed model case and run the targeted model-catalog tests plus the changed gate before landing.
