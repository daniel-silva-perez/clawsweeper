## Summary

Keep open. Current main still renders the Control UI credentials gate for every disconnected state, and the linked implementation PR remains the active path rather than a reason to close this issue before merge. https://github.com/openclaw/openclaw/issues/72500

## What This Changes

The issue requests that Control UI WebChat distinguish transient reconnects from first-connect or auth failures so tab refocus does not flash the credentials gate.

## Best Possible Solution

Land PR #72522 or an equivalent narrow Control UI fix that keeps chat mounted after a prior successful session during recoverable transport reconnects, restores the credentials gate only for first connect or explicit auth/pairing/origin/device-identity failures, and covers the state transition with focused tests.

## Reproduction Assessment

Yes. The deterministic path is visible in current main: after a successful hello, a recoverable WebSocket close calls `onClose`, clears `connected`, and `renderApp` immediately selects the credential-gate branch.

## Solution Assessment

Yes. A sticky prior-successful-session or reconnecting model is the narrow maintainable fix; a timing-only debounce could hide the flash but would still conflate transient transport disconnects with first-connect and auth-failure states.
