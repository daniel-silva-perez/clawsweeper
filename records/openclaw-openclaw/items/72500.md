## Summary

Keep this issue open. Current main still renders the Control UI credentials/login gate whenever `connected` is false, still clears `connected` immediately during reconnect and close handling, and still lacks a sticky prior-successful-connection state. PR #72522 is open, unmerged, and uses closing syntax for this issue, so it should be reviewed as the implementation candidate rather than closing the issue early.

## Best Possible Solution

Keep this issue open and use PR #72522 as the active implementation candidate. The fix should preserve the chat UI after a successful session across transient reconnects while restoring the credentials gate on explicit auth or pairing failures; after that PR or an equivalent fix lands, GitHub can close this issue as implemented.
