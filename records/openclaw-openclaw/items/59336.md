## Summary

Keep this PR open. Current main still contains the behavior the PR targets: safe-bin normalization writes `undefined` keys, raw snapshot fallback compares restored raw text against materialized `snapshot.config`, and the Control UI disables Raw mode when that fallback returns `raw: null`. The linked #59330 remains open, and this PR is a plausible implementation candidate, but it still needs maintainer review because it changes exec safe-bin policy normalization and has an unresolved security-sensitive review concern.

## Best Possible Solution

Keep this PR open and review a corrected version: preserve the raw-mode fix, avoid inherited safe-bin exposure, add focused regression tests for undefined safe-bin keys and parsed-vs-materialized round-trips, and either handle or split the commenter-reported token-aware raw redaction bug. Once a corrected implementation lands, #59330 can close through the normal linked-issue flow.
