## Summary

Keep this PR open. Current main still lacks the temporary `/media/:id` anti-cache hardening this PR proposes: the active handler sets only `X-Content-Type-Options`, media tests only assert `nosniff`, and the related browser noVNC hardening is a separate route. The PR is narrow and plausible, but its patch targets an older `attachMediaRoutes` shape and needs to be ported to `createMediaRequestHandler`.

## Best Possible Solution

Keep this PR open. Port the anti-cache headers into `createMediaRequestHandler` immediately after a matched `/media/:id` route and before all validation/read/error branches, then expand `src/media/server.test.ts` and `src/media/server.outside-workspace.test.ts` to assert `Cache-Control`, `Pragma`, and `Expires` on success, HEAD, expired, invalid, oversized, missing, and outside-workspace responses while preserving TTL expiry, cleanup timing, path validation, active-content download behavior, and HEAD semantics.
