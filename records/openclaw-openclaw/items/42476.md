## Summary

Keep this PR open. Current main still does not implement the intended temporary media anti-cache hardening: the active `/media/:id` handler sets `X-Content-Type-Options` only, and the media tests do not assert `Cache-Control`, `Pragma`, or `Expires`. The PR remains a narrow, plausible security hardening change, but it needs to be rebased or ported to the current `createMediaRequestHandler` shape before it can merge.

## Best Possible Solution

Keep this PR open. Port the header setting into `createMediaRequestHandler` immediately after a matched `/media/:id` route and before validation/read/error branches, then expand `src/media/server.test.ts` to assert `Cache-Control`, `Pragma`, and `Expires` on the relevant success and error responses while preserving TTL expiry, cleanup timing, path validation, active-content download behavior, and HEAD semantics.
