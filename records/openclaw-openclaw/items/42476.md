## Summary

Keep this PR open. Current main still serves temporary `/media/:id` responses with `X-Content-Type-Options: nosniff` only, while the PR’s requested `Cache-Control`, `Pragma`, and `Expires` anti-cache headers are absent from the core media handler and tests. The change remains a focused security hardening candidate, but it needs a rebase onto the current `createMediaRequestHandler` implementation and broader response-path assertions.

## Best Possible Solution

Keep this PR open and rebase the hardening onto `src/media/server.ts` by setting `Cache-Control: no-store, private, max-age=0, must-revalidate`, `Pragma: no-cache`, and `Expires: 0` immediately after a matched `/media/:id` route and before validation/read/error branches. Update `src/media/server.test.ts` and `src/media/server.outside-workspace.test.ts` as needed so success, HEAD, missing-media, expired, invalid, oversized, and outside-workspace responses assert the headers while preserving TTL expiry, cleanup timing, path validation, active-content download behavior, and file-serving semantics.
