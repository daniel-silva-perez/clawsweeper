## Summary

Keep open. Current main still has the reported Slack cross-session blind spot: Slack DM/main sessions and Slack channel sessions remain separate, outbound Slack message-tool sends are mirrored to the destination session, and default session-tool visibility does not expose sibling channel sessions back to a DM turn. Recent maintainer review context also kept this open and treated #64427 as adjacent rather than canonical.

## What This Changes

The issue requests that a Slack agent be able to recall its own outbound channel messages when operating from a DM session.

## Best Possible Solution

Add a narrow Slack self-activity recall mechanism, such as an outbound activity ledger or scoped session-history bridge keyed by agent id, Slack team/account, destination channel/thread, and Slack message id or timestamp. It should expose only the agent's own outbound channel messages to DM/main sessions without merging full DM and channel transcripts or widening general session visibility.
