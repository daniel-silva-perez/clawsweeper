## Summary

Keep open. Current main still preserves the reported Slack blind spot: Slack DM/main sessions and channel sessions remain separate, outbound Slack message sends are mirrored to the destination session only, and default session tools do not expose sibling channel sessions back to a DM turn. The recent maintainer review in the issue also kept this open and treated #64427 as adjacent rather than canonical.

## What This Changes

The issue requests a way for a Slack DM session to recall the agent's own outbound Slack channel messages and prior channel conversations.

## Best Possible Solution

Add a narrow Slack self-activity recall mechanism, such as an outbound activity ledger or scoped session-history bridge keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp. It should expose only the agent's own outbound channel messages to DM/main sessions without merging full DM and channel transcripts or widening general session visibility.
