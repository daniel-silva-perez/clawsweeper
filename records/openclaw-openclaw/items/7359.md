## Summary

Keep open. Current main still has the reported Slack cross-session blind spot: outbound Slack channel sends are mirrored into the destination channel session, while a DM/main session remains a separate session and default session-tool visibility does not expose sibling channel sessions. The existing issue discussion also kept this open and treated #64427 as adjacent rather than a replacement.

## What This Changes

The issue asks for Slack DM sessions to let the agent recall its own outbound messages and related activity from Slack channel sessions.

## Best Possible Solution

Add a narrow Slack self-activity recall path keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp, exposing only the agent's own outbound channel sends to authorized DM/main contexts without merging full transcripts or widening generic session visibility.
