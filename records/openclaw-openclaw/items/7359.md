## Summary

Current main still appears to have the reported Slack visibility gap. Slack DMs and channel traffic resolve to different session keys, outbound `message` sends are mirrored to the destination channel session, and default session-tool visibility remains the current session tree rather than a scoped same-agent view of the agent's own Slack channel activity. The related open #64427 is adjacent DM history hydration work, not a replacement for this cross-session own-send recall request.

## Best Possible Solution

Keep this open. The best fix is a scoped Slack self-activity recall path, likely an outbound activity ledger or session-history bridge keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp. It should expose only the agent's own outbound Slack sends from DM/main context, avoid broad cross-session transcript visibility, and add regression coverage for a Slack DM/main turn that posts to a Slack channel through the message tool and later recalls that sent message.
