## Summary

Keep open. Current main still separates Slack DM/main sessions from Slack channel sessions, and message-tool outbound sends are mirrored only into the destination route session, so a DM turn still lacks a scoped view of the agent's own channel sends. The adjacent open #64427 DM-history bug does not supersede this because this issue is specifically about recalling the agent's own outbound channel activity from a DM.

## What This Changes

Request a Slack self-activity recall path so an agent in a DM can see and refer to its own messages previously sent to Slack channels.

## Best Possible Solution

Add a narrow Slack self-activity recall mechanism, such as an outbound activity ledger or scoped session-history bridge keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp. It should let DM/main sessions recall the agent's own outbound channel messages without merging full DM and channel transcripts or widening general session visibility.
