## Summary

Current main still has the reported Slack blind spot: DM/main sessions are separate from Slack channel sessions, and outbound message-tool sends are mirrored into the destination session rather than a DM-visible self-activity view. The adjacent open DM-history issue does not supersede this request because this issue is specifically about recalling the agent's own outbound channel sends from a DM.

## What This Changes

Request Slack DM sessions to recall the agent's own outbound Slack channel messages and maintain coherent activity context across DMs and channels.

## Best Possible Solution

Keep this open and design a narrow Slack self-activity recall path, such as an outbound activity ledger or scoped session-history bridge keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp. It should be visible from DM/main sessions for the agent's own outbound messages without merging Slack DM and channel transcripts or making arbitrary channel history visible.
