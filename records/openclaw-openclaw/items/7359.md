## Summary

Current main still has the reported Slack cross-session visibility gap: Slack DM/main sessions and channel sessions are separate, outbound Slack sends are mirrored only to the destination session, and default session-tool visibility does not expose sibling channel sessions back to a DM. Existing maintainer and ClawSweeper comments also kept this open and treated #64427 as adjacent rather than a canonical fix for scoped self-activity recall.

## What This Changes

Request that a Slack agent operating from a DM be able to recall and reference its own outbound messages sent to Slack channels.

## Best Possible Solution

Add a narrow Slack self-activity recall mechanism, such as an outbound activity ledger or scoped session-history bridge keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp. It should expose only the agent's own outbound channel messages to DM/main sessions without merging full DM and channel transcripts or widening general sibling-session visibility.
