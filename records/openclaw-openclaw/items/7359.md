## Summary

Keep open. Current main still preserves the reported Slack blind spot: DM/main sessions and Slack channel sessions are separate, outbound `message` sends mirror to the destination session, and the default session-history tools do not expose sibling channel sessions back to a DM requester. The recent maintainer and ClawSweeper comments correctly treat #64427 as adjacent rather than a canonical fix for this self-activity recall problem.

## What This Changes

The issue requests that a Slack DM session be able to recall the agent's own outbound messages sent to Slack channels.

## Best Possible Solution

Keep Slack DM and channel transcripts isolated, but add a scoped self-activity recall path for the agent's own outbound Slack messages keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp.

## Reproduction Assessment

Yes. At code-path level, a Slack DM/main turn that sends to a channel records the assistant text in the destination channel session, while default tree visibility prevents the DM/main session from reading that sibling session.

## Solution Assessment

No current implementation solves it yet. A broad unified Slack session is not the best fix; the narrower maintainable path is a self-activity ledger or scoped history bridge for the agent's own outbound channel messages.
