## Summary

Keep open. Current main still has the reported Slack blind spot: Slack DMs and channels resolve to different session keys, outbound channel sends are mirrored only into the destination channel session, and default session-tool visibility does not expose that sibling channel session back to the DM/main requester. The provided discussion also includes a recent maintainer review explicitly keeping the item open and treating #64427 as adjacent rather than canonical.

## What This Changes

Allow a Slack agent operating from a DM/main session to recall its own outbound messages sent to Slack channels so it can answer and follow up coherently across contexts.

## Best Possible Solution

Keep Slack DM and channel transcripts isolated, but add a scoped self-activity recall path for the agent's own outbound Slack messages keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp.

## Reproduction Assessment

Yes. At code-path level, a Slack DM/main turn sends to a channel whose assistant mirror lands in the destination channel session, while default `tree` visibility prevents the DM/main session from reading that sibling session.

## Solution Assessment

No current implementation solves this yet. A broad unified Slack session would over-merge private context; the narrower maintainable fix is a self-activity ledger or scoped history bridge for agent-authored outbound Slack channel messages.
