## Summary

Keep open. Current main still keeps Slack DM/main sessions and Slack channel sessions in separate session buckets, mirrors outbound message-tool sends into the destination session, and leaves default session-tool visibility at the current session tree, so the reported DM blind spot remains. The April 26 maintainer review also kept this open and treated #64427 as adjacent rather than canonical.

## What This Changes

The request is for Slack agents to recall their own messages and channel conversations when later operating from a DM session.

## Best Possible Solution

Keep separate Slack DM and channel sessions, but add a scoped self-activity recall path for the agent's own outbound Slack messages, keyed by agent id, team/account, destination channel/thread, and Slack message id/timestamp, without widening generic session visibility.

## Reproduction Assessment

Yes, at code-path level. A Slack DM/main turn that uses the message tool to send to a Slack channel records the outbound assistant text in the destination channel session, while default tree visibility prevents the DM session from reading that sibling session.

## Solution Assessment

No current implementation solves it yet. A broad unified Slack session would be risky; the narrower maintainable path is a self-activity ledger or scoped session-history bridge for the agent's own outbound channel messages.
