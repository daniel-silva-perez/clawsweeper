## Summary

Keep this issue open. Current main still keeps Slack DM/main sessions and Slack channel sessions separate, and message-tool sends are mirrored into the outbound destination session rather than a scoped same-agent self-activity view available from a DM. The adjacent #64427 DM-history issue does not replace this DM-to-channel self-send recall request.

## Best Possible Solution

Keep this open and design a narrow Slack self-activity recall path. The safest direction is an outbound activity ledger or scoped session-history bridge keyed by agent id, Slack account/team, destination channel/thread, and Slack message id/timestamp, exposed to DM/main sessions without merging Slack DM and channel transcripts or making arbitrary channel history visible.
