## Summary

Current main still has the reported Slack visibility gap. Slack DMs and channels resolve to different session keys, outbound message-tool sends are mirrored to the destination Slack channel session, and default session-history visibility remains the current session tree rather than a scoped same-agent view of the agent's own Slack outbound activity. The related #64427 DM-history item is adjacent, but it does not replace this DM-to-channel self-send recall request.

## Best Possible Solution

Keep this open and implement a scoped Slack self-activity recall path. The fix should let a Slack DM/main session discover the same agent's own outbound Slack channel sends without merging DM and channel sessions or making arbitrary channel transcripts visible. A small outbound activity ledger or narrowly scoped session-history bridge keyed by agent id, Slack account/team, destination channel/thread, and Slack message id/timestamp is the best product/code direction.

## Clownfish Work Prompt

Fix the Slack self-visibility gap reported in https://github.com/openclaw/openclaw/issues/7359. Current main mirrors message-tool sends to the destination Slack channel session, while a later Slack DM/main session cannot see that the same agent sent the channel message because default session-tool visibility is tree-scoped. Implement a narrow same-agent Slack outbound recall path for the agent's own sent Slack messages. Prefer a small outbound activity ledger or explicitly scoped session-history bridge keyed by agent id, Slack account/team, destination channel/thread, and Slack message id/timestamp. Do not merge Slack DM and channel session keys, do not make all same-agent channel transcripts visible by default, do not expose user-authored channel history beyond what is already allowed, and keep the fix Slack/outbound-session scoped unless a generic seam is clearly needed. Add regression coverage for a Slack DM/main session that sends a Slack channel message via the message tool and can later recall that own outbound send, plus a negative case showing unrelated channel/user history is still hidden. Update Slack docs only if a user-visible recall/tool behavior changes, and add a single-line changelog Fixes entry with Thanks @trevorallred if runtime behavior changes.
