## Summary

Keep open. Current main still has the reported Slack cross-session blind spot: DM/main sessions and Slack channel sessions remain separate, outbound message-tool sends are mirrored into the destination session only, and default session-tool visibility does not expose sibling channel sessions back to a DM turn. A recent maintainer review also explicitly kept this open and distinguished the related open DM-history issue as adjacent rather than canonical.

## What This Changes

The issue requests a way for a Slack DM session to recall the agent's own outbound Slack channel messages and channel conversations.

## Best Possible Solution

Add a narrow Slack self-activity recall mechanism, such as an outbound activity ledger or scoped session-history bridge keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp. It should expose the agent's own outbound channel messages to DM/main sessions without merging full DM and channel transcripts or widening general session visibility.
