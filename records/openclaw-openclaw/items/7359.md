## Summary

Keep open. Current main still has the reported Slack cross-session blind spot: Slack DM/main sessions and Slack channel sessions are separate, outbound message-tool sends are mirrored to the destination channel session only, and default session-tool visibility does not let the DM/main session read that sibling channel session. The existing maintainer review also kept this open and treated #64427 as adjacent rather than canonical.

## What This Changes

The issue asks OpenClaw to let a Slack DM session recall the agent's own outbound messages sent to Slack channels.

## Best Possible Solution

Keep Slack DM and channel transcripts isolated, but add a scoped self-activity recall path for the agent's own outbound Slack messages, keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp, without widening general session visibility.

## Reproduction Assessment

Yes. At code-path level, a Slack DM/main turn that sends to a channel records the assistant text in the destination channel session, while default tree visibility prevents the DM/main session from reading that sibling session.

## Solution Assessment

No current implementation solves it yet. A broad unified Slack session is not the best fix; the narrower maintainable path is a self-activity ledger or scoped history bridge for the agent's own outbound channel messages.
