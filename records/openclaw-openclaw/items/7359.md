## Summary

Keep this issue open. Current main still separates Slack DM/main sessions from Slack channel sessions, and message-tool sends are mirrored into the destination channel session only. The linked #64427 DM-history issue is adjacent, but it does not replace this issue's request for recalling the agent's own outbound channel sends from a DM.

## Best Possible Solution

Keep this open and design a narrow Slack self-activity recall path. The safest direction is an outbound activity ledger or scoped session-history bridge keyed by agent id, Slack account/team, destination channel/thread, and Slack message id/timestamp, exposed to DM/main sessions without merging Slack DM and channel transcripts or making arbitrary channel history visible.
