## Summary

Keep open. Current main still has the reported Slack cross-session visibility gap: Slack DM/main sessions and Slack channel sessions remain separate, outbound sends are mirrored only to the destination session, and default session tools do not expose sibling channel sessions back to a DM. Maintainer discussion already treated the related DM-history issue as adjacent rather than canonical for this scoped self-activity recall problem.

## What This Changes

The issue requests that Slack DM sessions be able to recall the agent's own outbound messages sent to Slack channels so the agent can answer coherently across DM and channel contexts.

## Best Possible Solution

Add a narrow Slack self-activity recall mechanism, such as an outbound activity ledger or scoped session-history bridge keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp. It should make the agent's own outbound channel messages visible from DM/main sessions without merging general DM and channel transcripts.
