## Summary

Keep this PR open. Current main still does not implement the PR’s intended tool-level post-write verification: sandbox write, sandbox edit, and non-workspace host write/edit paths can still return success after the delegated write operation resolves without checking existence, file type, or byte count. The linked bug remains a plausible target for this PR, and the related lower-level atomic-write PR does not supersede this tool-level fix.

## Best Possible Solution

Keep this PR open for maintainer review. Refresh it against current main, preserve host tilde expansion, extract shared post-write verification helpers for sandbox and host paths, add sandbox edit-path regression tests, coordinate lower-level byte-count hardening with #44662, then run the changed agents validation lane before landing.
