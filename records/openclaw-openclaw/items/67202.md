## Summary

Keep this PR open. Current main still does not implement the PR's intended tool-level post-write verification: sandbox write, sandbox edit, and non-workspace host write/edit paths can still return success after the delegated write operation resolves without checking existence, file type, or byte count. The linked bug #67136 is still open, and overlapping lower-level byte-count hardening in #44662 is also open, so it does not supersede this PR.

## Best Possible Solution

Keep this PR open for maintainer review. Refresh it against current main, preserve host tilde expansion, extract shared sandbox and host post-write verification helpers, add sandbox edit-path regression tests, coordinate lower-level byte-count hardening with #44662, then run the changed agents validation lane before landing.
