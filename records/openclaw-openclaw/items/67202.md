## Summary

Keep this PR open. Current main still lacks post-write verification in the pi-tool sandbox write, sandbox edit, and non-workspace host write/edit adapters, while the upstream write/edit tools report success after OpenClaw's injected write operation resolves. The PR targets a valid linked false-success bug, but it needs a refresh before merge because its host-write hunk is based on older code and would drop current tilde expansion behavior.

## What This Changes

The PR adds post-write existence, file-type, and byte-size checks to OpenClaw's pi-coding-agent sandbox and host write operations, plus regression coverage for write-tool verification failures.

## Best Possible Solution

Land a refreshed tool-level fix that preserves current host tilde behavior, shares sandbox and host verification helpers, adds sandbox edit-path regression coverage, and coordinates lower-level byte-count hardening with #44662.
