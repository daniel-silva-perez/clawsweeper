## Summary

Keep open. Current main preserves default-enabled Feishu streaming and already streams auto-mode card/markdown replies, but it still does not implement this PR's boolean-or-object `streaming` config, config-driven `throttleMs`, or config-driven card title/header behavior. The PR head also still has a default-disabled regression for omitted `streaming`, so it needs rework rather than cleanup closure.

## Best Possible Solution

Keep this PR open for focused rework or replacement. Add a typed Feishu streaming config resolver that accepts boolean and object forms while preserving `undefined` as default-enabled and `false` / `{ enabled: false }` as disabled; wire bounded `throttleMs` and optional validated title/header behavior through `FeishuStreamingSession`; update plugin manifest metadata and Feishu docs; and add regression tests for default, disabled, object config, custom throttle/title, and auto-mode plain-text versus card routing.
