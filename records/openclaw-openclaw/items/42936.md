## Summary

Keep open. Current main already handles Feishu streaming for auto-detected card replies, but it still does not implement the PR's central request: boolean-or-object Feishu streaming config with config-driven enabled/throttle/title behavior. The current PR head also still needs rework because it changes omitted `streaming` from default-enabled to disabled and does not update the manifest or docs for the new user-facing config shape.

## Best Possible Solution

Keep this PR open for focused rework or replacement. Preserve current `undefined` default-enabled and `false` disabled semantics, add a typed Feishu streaming config resolver for boolean and object forms, pass bounded `throttleMs` and validated optional title/header behavior through `FeishuStreamingSession`, update the plugin manifest and Feishu docs, and add regression tests for default, disabled, object config, custom throttle/title, and auto-mode plain-text versus card routing.
