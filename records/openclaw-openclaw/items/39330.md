## Summary

Keep this PR open. Current main still appends the internal unscheduled-reminder note to final auto-reply delivery when a reminder-like response has no successful cron scheduling, and the latest release has the same behavior. This PR remains a focused fix path, but it still needs revision so stripping is limited to text OpenClaw injected rather than every assistant payload ending with the same sentence; the PR security pass only found auto-reply source/test changes.

## Best Possible Solution

Keep this PR open and revise the implementation so final delivery never exposes OpenClaw's internally injected unscheduled-reminder note while preserving legitimate model-authored text. The narrowest fix is to track which payload `appendUnscheduledReminderNote` actually modified, or refactor the guard to record the unscheduled-reminder condition without appending user-visible text, then update the existing reminder-guard tests plus a regression that an unmodified payload ending with the same sentence is not stripped.
