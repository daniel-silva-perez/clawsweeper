## Summary

Keep open. Current main still appends the internal unscheduled-reminder note to reply payloads and sends those guarded payloads through final delivery, while the targeted tests still assert that the note is visible in no-cron cases. This PR remains a useful implementation path, but the remaining review concern is real: sanitization should be limited to text OpenClaw injected, not every assistant text payload ending with the same sentence.

## Best Possible Solution

Keep this PR open and revise it so the final delivery path never exposes OpenClaw's internally injected reminder-guard note while preserving legitimate model output. The narrowest path is to gate stripping to payloads that appendUnscheduledReminderNote actually modified, or to refactor the guard so it records the condition without appending user-visible text, then update the existing reminder-guard tests for the no-cron, existing-cron, unrelated-cron, disabled-cron, missing-sessionKey, and cron-store-read-failure cases.
