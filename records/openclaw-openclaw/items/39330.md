## Summary

Keep this PR open. Current main still appends and delivers the internal unscheduled-reminder note for reminder-like replies without a successful cron add, and the latest release has the same behavior. The PR remains a focused fix path, but it still needs revision because its sanitizer strips any non-error text payload ending with the same sentence, including legitimate model-authored text.

The narrow fix is to track which payload appendUnscheduledReminderNote actually modified, or refactor the guard so it records the unbacked-reminder condition without appending user-visible text to delivery payloads; coordinate with #47663 rather than mixing in a config-flag API change here.
