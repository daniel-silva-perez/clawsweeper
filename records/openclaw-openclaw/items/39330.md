## Summary

Keep open. Current main still appends the internal unscheduled-reminder note and sends guarded payloads through final delivery; the latest release has the same behavior. This PR remains a useful fix path, but it still has an unresolved review concern: stripping should be limited to text OpenClaw injected, not every assistant payload that happens to end with the exact note.

## Best Possible Solution

Keep this PR open and revise it so final delivery never exposes OpenClaw's internally injected reminder-guard note while preserving legitimate model output. The narrowest path is to gate stripping to payloads actually modified by appendUnscheduledReminderNote, or to refactor the guard so it records the condition without appending user-visible text, then update the existing reminder-guard tests for no-cron, existing-cron, unrelated-cron, disabled-cron, missing-sessionKey, and cron-store-read-failure cases.
