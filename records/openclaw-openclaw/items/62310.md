## Summary

Keep this PR open. Current main still has the reported heartbeat behavior: non-interval targeted wakes run immediately, fan-out wakes are gated only when `reason === "interval"`, there is no per-agent `lastRunMs` cooldown state, and `scheduleNext()` still schedules a zero-delay timer instead of a distinct overdue recovery path. The PR is also paired with the same-author open issue #62294 and has recent maintainer repair/merge-ready context, so it should remain as the implementation candidate rather than be closed by cleanup.

## Best Possible Solution

Keep this PR open for maintainer review and landing. If the branch validation remains acceptable, land the heartbeat-runner change with its scheduler regression tests and changelog entry, then let the paired bug report be closed by the merged implementation. The companion launchd/App Nap PR can remain a separate follow-up because current main still lacks this PR's runner-level cooldown and overdue recovery changes.
