## Summary

Keep open. Current main still lacks the heartbeat cooldown state, non-interval wake gates, and overdue-timer recovery this PR is trying to add, and the paired bug report remains open. ## What This Changes

The PR adds heartbeat last-run cooldown state, scheduler tests, and a changelog entry to throttle ordinary non-interval heartbeat wake bursts while preserving selected explicit wake paths and overdue interval recovery.

## Best Possible Solution

Keep the PR open as the canonical implementation path, update the throttle predicate and tests so explicit no-argument plugin/system wake requests remain immediate, then land the branch or an equivalent fix with the paired #62294 issue.

## Reproduction Assessment

Yes. A focused code-level reproduction is: run one heartbeat, then call api.runtime.system.requestHeartbeatNow() or requestHeartbeatNow({ coalesceMs: 0 }) inside the interval; the PR's predicate classifies the default reason as throttled and skips before runOnce.

## Solution Assessment

No, not as-is. The cooldown and overdue recovery are the right narrow direction for the reported bursts, but the PR should exempt the default requested/no-reason explicit wake or map it to manual semantics before merge.
