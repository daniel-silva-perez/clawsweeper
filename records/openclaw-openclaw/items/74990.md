## Summary

Keep this PR open. Current main still has the old Anthropic CLI onboarding label and manual-auth failure path, so the proposed feature is not obsolete, but the latest PR head still hard-codes the `claude` PATH command in the new install/login gate and breaks the existing configured CLI backend command contract.

## What This Changes

The PR relabels Anthropic CLI onboarding as a Claude subscription path, adds prompted Claude CLI detection/install/login handling, updates Anthropic docs/metadata/changelog, and adds tests for the new flow.

## Best Possible Solution

Keep the Anthropic plugin-owned onboarding direction, but amend the PR so it resolves the effective `claude-cli` backend command from `ctx.config`, uses that command for detection and login, and only offers `npm install -g @anthropic-ai/claude-code` when the effective command is the default PATH binary.

## Reproduction Assessment

Yes. Configure `agents.defaults.cliBackends.claude-cli.command` to an absolute Claude Code binary while `claude` is absent from PATH; the PR probes literal `claude --version` before migration can use the configured command, so onboarding incorrectly prompts to install or fails.

## Solution Assessment

No. The feature belongs in the Anthropic plugin, but this implementation is not the narrowest maintainable fix because it bypasses the existing CLI backend config contract instead of reusing it.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Respect the configured Claude CLI command:** `extensions/anthropic/cli-install.ts:40`
  - body: The new install/login gate always probes and runs `claude`, but OpenClaw supports `agents.defaults.cliBackends.claude-cli.command` for hosts where Claude Code lives at an absolute or custom path. With that valid config and no `claude` on PATH, onboarding prompts to install or fails instead of using the configured binary. Resolve the effective backend command from `ctx.config` for detect/login, and reserve npm install for the default PATH command.
