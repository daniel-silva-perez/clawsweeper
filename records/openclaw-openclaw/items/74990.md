## Summary

Keep this PR open. The subscription onboarding direction is useful and current main has not implemented it, but the proposed install/login gate still bypasses OpenClaw’s supported custom Claude CLI command path by probing and spawning the hard-coded PATH binary.

## What This Changes

The branch relabels Anthropic Claude CLI onboarding as a Claude subscription path, adds prompted Claude Code detect/install/login handling, updates Anthropic metadata/docs/changelog, and adds tests.

## Best Possible Solution

Amend the PR so Anthropic onboarding resolves the effective `claude-cli` backend command from `ctx.config`, uses that command for detection and login, offers `npm install -g @anthropic-ai/claude-code` only for the default PATH command, and adds a regression test for a configured absolute command without `claude` on PATH.

## Reproduction Assessment

Yes. Configure `agents.defaults.cliBackends.claude-cli.command` to an absolute Claude Code binary and remove `claude` from PATH; the PR probes `claude --version` before migration can use the configured backend command, so onboarding prompts to install or fails incorrectly.

## Solution Assessment

No. The plugin-owned subscription onboarding direction is appropriate, but the current implementation is not the best fix because it bypasses the existing custom CLI backend command contract instead of reusing it.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Respect configured Claude CLI command:** `extensions/anthropic/cli-install.ts:40`
  - body: The new install/login gate always probes and runs `claude`, but OpenClaw supports `agents.defaults.cliBackends.claude-cli.command` for hosts where Claude Code lives at an absolute or custom path. With that documented config and no `claude` on PATH, onboarding now prompts to install or fails instead of using the configured binary. Resolve the effective backend command from `ctx.config` for detect/login, and reserve npm install for the default PATH command.
