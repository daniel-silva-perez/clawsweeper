## Summary

Keep this PR open for normal review, but do not merge it as-is. The subscription onboarding direction fits the Anthropic plugin boundary and current main does not already implement the requested relabel/install/login flow, but the PR introduces a blocking regression by probing and launching the hard-coded `claude` binary instead of honoring the supported configured Claude CLI command path.

## What This Changes

The PR relabels Anthropic CLI onboarding as a Claude subscription path, adds Claude Code detection/install/login handling, and updates Anthropic metadata, docs, changelog, and tests.

## Best Possible Solution

Amend the PR so Anthropic onboarding resolves the effective `claude-cli` backend command from `ctx.config`, uses that command for detection and login, offers `npm install -g @anthropic-ai/claude-code` only for the default PATH command, and adds a regression test for a configured absolute command with no `claude` on PATH.

## Reproduction Assessment

Yes. Configure `agents.defaults.cliBackends.claude-cli.command` to an absolute Claude Code binary and run Anthropic CLI onboarding on a host where `claude` is absent from PATH; the PR probes `claude --version` before migration can use the supported configured backend command.

## Solution Assessment

No. Plugin-owned Claude subscription onboarding is the right boundary, but this implementation is not the best fix until it preserves the existing configured-command contract for detect and login.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Respect configured Claude CLI command:** `extensions/anthropic/cli-install.ts:40`
  - body: The new install/login gate always probes and runs `claude`, but OpenClaw supports `agents.defaults.cliBackends.claude-cli.command` for hosts where Claude Code lives at an absolute or custom path. Resolve the effective backend command from `ctx.config` for detect/login, and reserve npm install for the default PATH command.
