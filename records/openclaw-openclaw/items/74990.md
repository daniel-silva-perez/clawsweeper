## Summary

Keep this PR open. The author addressed the earlier `claude auth login` and changelog-credit findings, but the latest patch still hard-codes the `claude` PATH binary in the new install/login gate and runs that gate before reading existing credentials, which regresses the documented custom Claude CLI command path.

## What This Changes

The PR renames the Anthropic Claude CLI onboarding choice as a Claude subscription path, adds a Claude Code install/login helper for onboarding, updates Anthropic provider metadata/docs/changelog, and adds tests for the new wizard flow.

## Best Possible Solution

Amend this PR so the onboarding helper resolves the configured `claude-cli` backend command from `ctx.config` and uses it for detection, login prompts, and login execution; keep npm global install as the fallback for the default PATH command, and add a regression test for a configured absolute command with no `claude` on PATH.

## Reproduction Assessment

Yes. Configure `agents.defaults.cliBackends.claude-cli.command` to an absolute Claude Code binary while `claude` is absent from PATH; the PR calls `ensureClaudeCliInstalled` before reading credentials and `detectClaudeCli` probes only `claude --version`, so the wizard prompts to install or fails instead of using the configured command.

## Solution Assessment

No. The plugin-owned subscription onboarding direction is appropriate and the earlier auth-command fix now matches upstream CLI docs, but the patch must preserve OpenClaw's existing custom CLI backend command contract before merge.

## Review Findings
