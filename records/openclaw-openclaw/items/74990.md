## Summary

Keep this PR open. The Anthropic subscription onboarding direction is useful and plugin-owned, but the latest PR head still hard-codes `claude` from PATH for detection and sign-in, which breaks the existing documented `agents.defaults.cliBackends.claude-cli.command` override contract.

## What This Changes

This PR renames the Anthropic CLI auth choice, adds a Claude Code install/sign-in helper for onboarding, and updates Anthropic metadata, docs, changelog text, and tests.

## Best Possible Solution

Amend this PR so the Anthropic onboarding helper resolves the effective `claude-cli` backend command from `ctx.config` and uses it for detect and login; offer `npm install -g @anthropic-ai/claude-code` only for the default PATH command, then add a regression test for a configured absolute command with no `claude` on PATH.

## Reproduction Assessment

Yes. Configure `agents.defaults.cliBackends.claude-cli.command` to an absolute Claude Code binary while `claude` is absent from PATH; the PR enters `ensureClaudeCliInstalled` before credential migration and `detectClaudeCli` probes only `claude --version`.

## Solution Assessment

No. The plugin-owned subscription onboarding direction is appropriate, but this implementation is not the narrowest maintainable fix because it bypasses the existing custom CLI backend command contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

  - body: The new install/login gate always probes `claude` from PATH, but OpenClaw supports `agents.defaults.cliBackends.claude-cli.command` for hosts where Claude Code lives at an absolute/custom path.
