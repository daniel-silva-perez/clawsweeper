## Summary

Keep open. Current main has not implemented the requested onboarding label/install flow, and the PR branch still introduces a blocking regression for supported Claude CLI custom-command configs.

## What This Changes

The PR relabels Anthropic CLI onboarding as a Claude subscription path, adds Claude Code detect/install/login handling, and updates Anthropic metadata, docs, changelog, and tests.

## Best Possible Solution

Amend the PR so Anthropic onboarding resolves the effective Claude CLI backend command from `ctx.config`, uses that command for detect and login, offers `npm install -g @anthropic-ai/claude-code` only for the default PATH command, and adds a regression test for a configured absolute command with no `claude` on PATH.

## Reproduction Assessment

Yes. Configure `agents.defaults.cliBackends.claude-cli.command` to an absolute Claude Code binary while `claude` is absent from PATH; the PR enters `ensureClaudeCliInstalled` and probes only the hard-coded `claude --version` path before migration can use the supported backend config.

## Solution Assessment

No. The plugin-owned subscription onboarding direction is appropriate, but this implementation is not the narrowest maintainable solution because it bypasses the existing custom CLI backend command contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Respect configured Claude CLI command:** `extensions/anthropic/cli-install.ts:40`
  - body: The new install/login gate always probes and runs `claude`, but OpenClaw supports `agents.defaults.cliBackends.claude-cli.command` for hosts where Claude Code lives at an absolute or custom path. With that documented config and no `claude` on PATH, onboarding now prompts to install or fails instead of using the configured binary. Resolve the effective backend command from `ctx.config` for detect/login, and reserve npm install for the default PATH command.
  - confidence: 0.89
