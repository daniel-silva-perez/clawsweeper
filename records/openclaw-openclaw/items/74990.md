## Summary

Keep this PR open. The onboarding direction is useful, and the author already addressed the login command/changelog feedback, but the latest PR head still bypasses the supported `claude-cli.command` override and can reject a valid configured Claude binary before migration reaches credential handling.

## What This Changes

The PR renames the Anthropic Claude CLI onboarding choice as a Claude subscription path, adds an interactive Claude Code install/sign-in helper, and updates Anthropic metadata, docs, changelog text, and tests.

## Best Possible Solution

Amend the Anthropic onboarding helper so install detection and sign-in use the effective `claude-cli` backend command from the current config, while offering `npm install -g @anthropic-ai/claude-code` only when the effective command is the default PATH binary. Add a regression test for an absolute configured command with no `claude` on PATH.

## Reproduction Assessment

Yes. Configure `agents.defaults.cliBackends.claude-cli.command` to an absolute Claude Code binary while `claude` is absent from PATH; at PR head the wizard enters `ensureClaudeCliInstalled` and probes only `claude --version` before credential migration.

## Solution Assessment

No. Plugin-owned subscription onboarding is the right product boundary, but this implementation is not the narrowest maintainable solution because it bypasses the existing custom CLI backend command contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Respect configured Claude CLI command:** `extensions/anthropic/cli-install.ts:40`
  - body: The new install/login gate always probes and launches `claude` from PATH, but current OpenClaw supports `agents.defaults.cliBackends.claude-cli.command` for hosts where Claude Code lives at an absolute or custom path. With that documented config and no `claude` on PATH, onboarding now prompts to install or fails instead of using the configured binary. Resolve the effective backend command from `ctx.config` for detect/login, and reserve npm install for the default PATH command.
  - confidence: 0.88
