## Summary

Keep open. Current main still has adjacent strict-agentic, incomplete-turn, liveness, and wait-snapshot safeguards, but it does not define the requested atomic run.begin/run.exec/run.commit-or-run.abort lifecycle contract with structured abort blockers. The existing maintainer and ClawSweeper comments also kept this open after checking the same core agent/gateway surfaces.

## What This Changes

Request to add a transaction-style run lifecycle with begin, execution, and mandatory commit-or-abort terminal states so pre-action notices cannot remain orphaned.

## Best Possible Solution

Define an additive core lifecycle contract at the gateway protocol boundary, document intent, execution evidence, mandatory commit-or-abort terminal states, and structured blocker details, then wire it through AgentEvent types, runtime emitters, agent.wait, session/client/plugin-visible surfaces, and regression tests.
