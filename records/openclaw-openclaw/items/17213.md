## Summary

Keep open. Current main has adjacent strict-agentic, liveness, abort, and wait-snapshot safeguards, but it still does not define the requested public atomic run.begin/run.exec/run.commit-or-run.abort lifecycle contract with structured abort blockers; the recent maintainer and ClawSweeper comments reached the same conclusion.

## What This Changes

The issue asks OpenClaw to make announced agent actions transactional so every intent is followed by execution evidence and an explicit committed or aborted terminal state.

## Best Possible Solution

Define an additive protocol-level run lifecycle contract that distinguishes intent, execution evidence, committed completion, and aborted/blocker completion, then wire it through AgentEvent types, Gateway schemas, embedded runtime emitters, agent.wait, SDK/client normalization, session state, docs, and regression tests without breaking existing lifecycle consumers.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No concrete runtime reproduction is provided, but current-main source and docs inspection gives high-confidence proof that the requested lifecycle contract is absent.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers settle the public contract; the right direction is an additive lifecycle protocol rather than more local heuristics, but the exact state names, payloads, and rollout need human review.
