## Summary

Keep open. Current main has adjacent strict-agentic, liveness, abort, and wait-snapshot safeguards, but it still does not expose or enforce the requested atomic run.begin/run.exec/run.commit-or-run.abort contract with structured abort blockers across the public agent/gateway lifecycle. Prior maintainer and ClawSweeper comments also kept this open after checking the same core surfaces.

## What This Changes

The issue requests an atomic run lifecycle contract so announced actions always end in explicit commit or abort status with machine-readable blocker details.

## Best Possible Solution

Define an additive core lifecycle contract at the gateway protocol boundary, document intent, execution evidence, mandatory commit-or-abort terminal states, and structured blocker details, then wire it through AgentEvent types, runtime emitters, agent.wait, session/client/plugin-visible surfaces, and regression tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a concrete runtime bug repro: the report lacks examples/logs, but source inspection gives high-confidence proof that the requested lifecycle contract is absent on current main.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainer design lands; an additive protocol-level lifecycle contract is the right direction, but the exact states, compatibility behavior, and client/plugin rollout need human review before a fix PR.
