## Summary

Keep open. Current main has narrower planning-only and liveness safeguards, and the existing maintainer discussion already kept the item open; current agent events, gateway schema, lifecycle emitters, session state, and `agent.wait` still do not define the requested atomic begin/exec/commit-or-abort contract with structured abort blockers.

## What This Changes

Request a transaction-style agent run lifecycle that makes announced execution intent end in an explicit commit or abort state with machine-readable abort details.

## Best Possible Solution

Define an additive core lifecycle contract at the protocol boundary, document begin intent, execution evidence, mandatory commit or abort terminal state, and structured abort/blocker details, then wire that contract through AgentEvent types, Gateway schemas, runtime emitters, `agent.wait`, clients/plugin-visible surfaces, and regression tests.
