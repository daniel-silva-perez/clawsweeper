## Summary

Keep this issue open. Current main has narrower safeguards for strict-agentic GPT-5/OpenAI runs, lifecycle start/end/error events, and agent.wait terminal snapshots, but it still does not implement the requested shared run.begin -> run.exec -> mandatory run.commit/run.abort transaction contract with structured abort blockers across agent events, Gateway protocol, lifecycle state, and client/plugin-facing wait surfaces.

## Best Possible Solution

Keep this open as a core reliability/product-contract design item. The best path is a maintainer-approved, backward-compatible lifecycle design that defines explicit begin intent, execution evidence, mandatory commit or abort terminal state, and structured abort/blocker details across AgentEvent types, Gateway protocol/schema, emitters, agent.wait, plugin or harness-facing surfaces, clients, docs, and regression tests. Treat #40710, #55721, and #70877 as useful context, not closure evidence.
