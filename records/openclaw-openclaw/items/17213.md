## Summary

Keep this issue open. Current main has narrower reliability safeguards for plan-only or empty turns, including strict-agentic recovery for supported GPT-5 OpenAI/Codex runs, but it still does not implement the requested shared run.begin -> run.exec -> mandatory run.commit/run.abort transaction contract with structured abort blockers across AgentEvent, Gateway protocol, lifecycle state, and agent.wait surfaces. Related PR #40710 was closed unmerged, so it is useful prior art rather than shipped closure evidence.

## Best Possible Solution

Keep this issue open as a core reliability/product-contract design item. The best path is a maintainer-approved, backward-compatible lifecycle design that defines explicit begin intent, execution evidence, mandatory commit or abort terminal state, and structured abort/blocker details across AgentEvent types, Gateway protocol/schema, emitters, plugin or harness-facing surfaces, clients, docs, and regression tests. Treat #40710 and #55721 as useful context, not closure evidence.
