## Summary

Keep open. Current main has adjacent strict-agentic, liveness, abort, and wait-snapshot safeguards, but it still lacks the requested public atomic run.begin/run.exec/run.commit-or-run.abort lifecycle contract with structured abort blockers; the maintainer and prior ClawSweeper discussion also kept this open after checking the same core surfaces.

## What This Changes

The issue asks OpenClaw to guarantee that announced action intent either proceeds through execution and commit or terminates with an explicit abort and machine-readable blocker.

## Best Possible Solution

Define an additive protocol-level lifecycle contract for agent runs, document intent, execution evidence, mandatory commit-or-abort terminal states, and structured blocker details, then wire it through AgentEvent types, Gateway schemas, runtime emitters, agent.wait, session/client/plugin-visible surfaces, and regression tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a concrete runtime failure: the report has no logs or steps, but source and docs inspection gives high-confidence proof that the requested lifecycle contract is absent on current main.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainer design lands; an additive public lifecycle contract is the right direction, but the exact states, payloads, compatibility behavior, and rollout need human review.
