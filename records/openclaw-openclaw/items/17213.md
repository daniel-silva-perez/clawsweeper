## Summary

Keep open. Current main has adjacent incomplete-turn, replay, liveness, abort, and wait-snapshot safeguards, but it still lacks the requested public atomic run lifecycle contract with intent, execution evidence, and mandatory committed-or-aborted terminal states; the maintainer and ClawSweeper comments already reached the same keep-open conclusion and the current source still supports that outcome.

## What This Changes

The issue requests a core reliability contract that guarantees announced actions move through begin, execution, and mandatory commit-or-abort states so pre-action notices cannot be orphaned.

## Best Possible Solution

Define an additive protocol-level lifecycle contract that distinguishes intent, execution evidence, committed completion, and aborted/blocker completion, then wire it through AgentEvent types, Gateway schemas, embedded runtime emitters, agent.wait, SDK/client normalization, session state, docs, and regression tests without breaking existing lifecycle consumers.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No concrete runtime interruption reproduction is provided, but current-main source and docs inspection gives high-confidence proof that the requested lifecycle contract is absent.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers settle the public contract; the maintainable direction appears to be an additive lifecycle protocol rather than more local heuristics, but the exact states and rollout need human review.
