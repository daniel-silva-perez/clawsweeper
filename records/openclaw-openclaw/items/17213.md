## Summary

Keep open. Current main has useful adjacent safeguards, including a new embedded lifecycle terminal backstop and planning/liveness checks, but it still does not define the requested atomic run.begin/run.exec/mandatory run.commit-or-run.abort contract with structured abort blockers. The existing maintainer and ClawSweeper comments also treated related work as context rather than closure evidence.

## What This Changes

The issue requests an atomic run lifecycle contract so every announced execution intent reaches an explicit commit or abort state with machine-readable abort details.

## Best Possible Solution

Define an additive core lifecycle contract at the gateway protocol boundary, document intent, execution evidence, mandatory commit-or-abort terminal states, and structured abort/blocker details, then wire it through AgentEvent types, gateway schemas, runtime emitters, agent.wait, clients or plugin-visible surfaces, and regression tests.
