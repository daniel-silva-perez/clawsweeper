## Summary

Keep open. Current main has useful adjacent safeguards for plan-only turns, liveness, replay invalidation, and `agent.wait`, but it still lacks the requested atomic run lifecycle contract with typed begin, execution, mandatory commit-or-abort, and structured abort blocker semantics. The existing maintainer review in the issue also explicitly kept this open as core reliability/product-contract work.

## What This Changes

The issue requests an atomic run lifecycle contract so announced execution intents must end in an explicit commit or abort status with machine-readable blocker details.

## Best Possible Solution

Define an additive core lifecycle contract at the gateway protocol boundary, document intent, execution evidence, mandatory commit-or-abort terminal states, and structured abort/blocker details, then wire it through agent event types, gateway schemas, runtime emitters, `agent.wait`, clients or plugin-visible surfaces, and regression tests.
