## Summary

Keep open. Current main has useful adjacent safeguards around planning-only turns, liveness, replay invalidation, terminal snapshots, and agent.wait races, but it still does not define the requested atomic run.begin/run.exec/mandatory run.commit-or-run.abort contract with structured abort blockers. The issue discussion also already includes maintainer and ClawSweeper keep-open reviews treating related work as context rather than closure evidence.

## What This Changes

Request to make announced execution intent atomic by requiring execution evidence followed by an explicit commit or structured abort terminal state.

## Best Possible Solution

Define an additive core lifecycle contract at the gateway protocol boundary, document intent, execution evidence, mandatory commit-or-abort terminal states, and structured abort/blocker details, then wire it through AgentEvent types, gateway schemas, runtime emitters, agent.wait, clients or plugin-visible surfaces, and regression tests.
