## Summary

Keep this issue open. Current main has narrower safeguards for incomplete or blocked agent turns, but it still models the shared run lifecycle as generic lifecycle start/end/error events plus terminal snapshots, not the requested atomic run.begin -> run.exec -> mandatory run.commit/run.abort contract with structured abort blockers across protocol, wait, plugin/runtime, clients, docs, and tests. The existing maintainer comment already reached the same conclusion, and the related PRs are adjacent safeguards rather than a merged replacement for the requested product contract.

## What This Changes

The issue asks OpenClaw to enforce a transaction-style run lifecycle so announced action intent is always followed by execution evidence and a mandatory commit or abort with machine-readable blocker details.

## Best Possible Solution

Define a backward-compatible core lifecycle contract that is additive at the protocol boundary, documents begin intent, execution evidence, mandatory commit or abort terminal state, and structured abort/blocker details, then cover the contract through AgentEvent types, Gateway schema, agent.wait, runtime emitters, client/plugin surfaces, and docs before closing this issue.
