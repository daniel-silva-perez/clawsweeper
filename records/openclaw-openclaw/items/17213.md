## Summary

Keep this issue open. Current main has useful narrower safeguards, including strict-agentic GPT-5/OpenAI blocked-state handling and lifecycle start/end/error snapshots, but it still lacks the requested shared run.begin -> run.exec -> mandatory run.commit/run.abort transaction contract with structured abort blockers across agent events, Gateway protocol, agent.wait, plugin runtime, docs, and clients.

## What This Changes

Request a shared transaction-style run lifecycle that guarantees every announced action reaches explicit execution evidence and a mandatory commit or abort state with machine-readable blocker details.

## Best Possible Solution

Keep this open as a core reliability/product-contract design item. The best path is a maintainer-approved, backward-compatible lifecycle contract that defines explicit begin intent, execution evidence, mandatory commit or abort terminal state, structured abort/blocker details, and regression tests across AgentEvent types, Gateway schema/protocol, emitters, agent.wait, plugin/runtime wait surfaces, clients, and docs.
