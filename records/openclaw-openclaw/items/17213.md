## Summary

Keep open. Current main has adjacent safeguards for incomplete turns, liveness, and missing lifecycle terminal events, but it still lacks the requested atomic intent-to-execution contract with typed begin/exec/mandatory commit-or-abort phases and structured abort blockers. The existing maintainer and ClawSweeper discussion also treated the related work as context, not closure evidence.

## What This Changes

The issue requests a transaction-style agent run lifecycle that binds announced execution intent to actual tool execution and mandatory commit or abort terminal states with machine-readable abort blockers.

## Best Possible Solution

Define an additive core lifecycle contract at the gateway protocol boundary, document intent, execution evidence, mandatory commit-or-abort terminal states, and structured blocker details, then wire that contract through AgentEvent types, gateway schemas, runtime emitters, agent.wait, clients or plugin-visible surfaces, and regression tests.
