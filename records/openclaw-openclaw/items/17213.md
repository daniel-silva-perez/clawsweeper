## Summary

Keep open. Current main still has adjacent strict-agentic, incomplete-turn, liveness, and wait-snapshot safeguards, but it does not define the requested atomic run.begin/run.exec/run.commit-or-run.abort lifecycle contract with structured abort blockers. The public event/protocol shapes remain generic, embedded runtime emits legacy start/end/error phases, and agent.wait/session state still normalize terminal snapshots to ok/error/timeout/killed rather than transaction outcomes.

## What This Changes

The issue requests an atomic run lifecycle contract so an announced action must reach an explicit commit or abort terminal state with machine-readable abort details.

## Best Possible Solution

Define an additive core lifecycle contract at the gateway protocol boundary, document intent, execution evidence, mandatory commit-or-abort terminal states, and structured blocker details, then wire it through AgentEvent types, gateway schemas, runtime emitters, agent.wait, client/plugin-visible surfaces, and regression tests.
