## Summary

Keep open. Current main still has adjacent planning-only, liveness, replay-invalidation, and terminal snapshot safeguards, but it does not define the requested atomic run.begin -> run.exec -> mandatory run.commit/run.abort contract with structured abort blockers. The issue discussion also already includes maintainer and ClawSweeper reviews keeping this as core reliability/product-contract work.

## What This Changes

The issue requests a transaction-style run lifecycle that guarantees every announced execution intent reaches an explicit commit or abort state with machine-readable abort details.

## Best Possible Solution

Define an additive core lifecycle contract at the gateway protocol boundary, document intent, execution evidence, mandatory commit-or-abort terminal states, and structured abort/blocker details, then wire it through AgentEvent types, gateway schemas, runtime emitters, agent.wait, clients or plugin-visible surfaces, and regression tests.
