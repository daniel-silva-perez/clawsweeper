## Summary

Keep open. Current main still lacks the requested SDK/Gateway environment discovery surface, and this PR is the active implementation for the open member-authored API request in #74708. The older overlapping draft #74852 is already closed, so the next step is maintainer API review and validation, not cleanup closure.

## What This Changes

The PR adds read-only `environments.list` and `environments.status` Gateway RPCs with SDK call-through/types, `operator.read` scope classification, protocol schemas/generated Swift models, docs, changelog, and focused tests while leaving create/delete unsupported.

## Best Possible Solution

Land one canonical additive implementation after maintainer API review, with typed protocol schemas, SDK call-through, generated native models, docs, changelog, and tests, while preserving unsupported create/delete and per-run environment selection until provisioning semantics are designed.

## Reproduction Assessment

Yes. Current main has a high-confidence source-level reproduction: the SDK methods still throw explicit unsupported errors, the SDK test asserts those errors, and the Gateway neither advertises nor registers `environments.*` handlers.

## Solution Assessment

Yes, with maintainer review. The PR is a narrow additive path for the linked request, but the stable environment type/capability vocabulary and managed-candidate scope should be settled before merge.
