## Summary

Keep open. Current main still lacks SDK-facing `environments.list` and `environments.status`, and this PR is an active implementation of the maintainer-authored request in #74708. The patch looks directionally correct and CI is green, but it changes the public Gateway/App SDK protocol and overlaps the open draft implementation in #74852, so the next step is maintainer API review and branch reconciliation rather than cleanup closure.

## What This Changes

The PR adds read-only Gateway `environments.list` and `environments.status` RPCs with SDK call-through/types, operator-read scoping, protocol schemas/generated Swift models, docs, changelog, and focused tests while leaving create/delete unsupported.

## Best Possible Solution

Land one canonical additive implementation for read-only environment discovery after maintainer API review, keeping create/delete/provisioning and per-run environment selection unsupported until those semantics are designed.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: SDK `list/status` throw explicit unsupported errors, tests assert that behavior, and Gateway does not advertise or register `environments.*` handlers.

## Solution Assessment

Yes, with maintainer review. The proposed approach is narrow and additive, but the maintainer should settle the public vocabulary and reconcile the duplicate branch before merge.
