## Summary

Keep this PR open, but do not merge it as-is. Current main still does not implement the linked dmScope heartbeat routing feature, and the latest PR branch introduces blocking regressions in heartbeat session ownership, exec-event classification, and system-event trust handling.

## What This Changes

The PR adds a shared event-routing key normalizer and rewrites exec notify-on-exit plus heartbeat session selection so single-owner dmScope configurations can route internal exec/system events toward main agent sessions.

## Best Possible Solution

Implement a narrow dmScope=main, single-owner routing seam after existing canonicalization and agent ownership checks. Preserve structured untrusted exec completion events, delivery context, exec-event wake classification, no-output suppression, and add focused regression coverage for the linked exec-event and cron-event routing gap.

## Reproduction Assessment

Yes. The regressions reproduce statically from the latest diff against current main: the branch changes wake reason to exec-exit, omits trusted: false, drops structured completion text, removes canonical heartbeat ownership checks, and removes an export that existing tests import. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No. The PR takes too broad a path by replacing stable exec notification and heartbeat session contracts; the safer fix is to layer dmScope-aware normalization onto the current canonicalization, ownership, trust, and event-classification behavior.
