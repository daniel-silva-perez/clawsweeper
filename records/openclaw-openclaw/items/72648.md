## Summary

Keep open. The PR targets a real session-routing gap from #71581, but the latest branch still rewrites core exec completion and heartbeat routing in ways that break current trust, delivery-context, heartbeat-reason, and session-ownership contracts before merge.

## What This Changes

The branch adds a shared event-routing-key normalizer and rewrites exec notify-on-exit plus heartbeat session resolution so single-owner DM system events and heartbeats can route toward an agent main session.

## Best Possible Solution

Repair this branch by adding a narrow session.dmScope=main, single-owner routing seam after the existing canonicalization and ownership checks, while preserving current exec completion trust, delivery context, structured summary, background-only guard, and exec-event heartbeat classification.

## Reproduction Assessment

Yes. The regressions are reproducible by source inspection of latest head 42cced05 against current main: the changed lines remove or rename contracts that current tests and helpers depend on, including trusted:false, notifyDeliveryContext, exec-event, and canonical heartbeat session selection.

## Solution Assessment

No. The current PR is broader than the needed fix and replaces established routing/notification contracts; the safer approach is to layer dmScope-aware normalization into the existing exec-event/cron-event paths without changing unrelated completion semantics.
