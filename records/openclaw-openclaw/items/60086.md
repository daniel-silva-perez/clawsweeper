## Summary

Keep open. Current main does not implement this PR's intended behavior: stale approval IDs still surface the raw gateway failure path, and active-run follow-up enqueue still returns undefined without queued/backlog-full acknowledgements. The PR is a plausible implementation candidate, but it needs revision to preserve the existing stale-active drain recovery flagged in review.

## Best Possible Solution

Keep this PR open as a focused bug-fix candidate, but update/rebase it so the detailed enqueue result and user acknowledgements are added while preserving the current stale-active drain recovery. The best patch would add a structured enqueue result API, keep enqueueFollowupRun as the boolean compatibility wrapper, return explicit queued/backlog-full ReplyPayloads from runReplyAgent, map stale approval IDs to a recovery hint, and include regression tests for stale approval, duplicate enqueue, queue-full enqueue, accepted acknowledgement, and stale-active drain restart.
