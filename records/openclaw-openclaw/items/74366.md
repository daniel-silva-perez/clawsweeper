## Summary

Keep open. This is a protected MEMBER-authored, maintainer-labeled implementation PR, current main does not already include the slash-menu accessibility wiring, and the latest head still needs maintainer review because the new combobox role on the native textarea appears non-conforming under ARIA in HTML.

## What This Changes

The PR adds Control UI chat slash-command ARIA listbox/option IDs, active-descendant and live-status wiring on the composer, supporting CSS/tests, and a changelog entry.

## Best Possible Solution

Revise the composer accessibility pattern so the slash menu has stable, test-covered relationships without assigning an invalid role to a native textarea, then get maintainer review on the final semantics and exact-head CI before landing.
