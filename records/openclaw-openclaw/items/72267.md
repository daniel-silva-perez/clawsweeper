The PR removes the leading newline before the zsh `#compdef` directive in generated completion output.

## Best Possible Solution

Land the one-line generated zsh completion fix, preferably with a focused regression test asserting `getCompletionScript("zsh", ...)` starts with `#compdef openclaw`.

## Solution Assessment
