## What This Changes

The PR removes the leading newline before the generated zsh `#compdef` directive in `src/cli/completion-cli.ts` so zsh autoload sees it on line 1.

## Best Possible Solution

Land the one-line zsh completion template fix with an Unreleased Fixes changelog entry; a focused `startsWith("#compdef openclaw")` regression test would make the fix more durable.

## Solution Assessment
