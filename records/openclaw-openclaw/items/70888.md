## Summary

Keep open. Current main still has the no-cfg session-store early return, and this PR remains a narrow implementation for the reported persisted explicit requester depth bug. The diff looks functionally correct, with one non-blocking test hygiene finding for temp directory cleanup in the announce-delivery regression tests.

## What This Changes

This PR removes the no-cfg early return from subagent depth lookup, lets omitted cfg use the default on-disk session store path, and adds regression coverage for explicit requester keys and announce classification.

## Best Possible Solution

Land the narrow helper fix after normal validation and the small temp-dir cleanup, keeping the change scoped to generic session-depth resolution and regression tests rather than changing routing policy.

## Reproduction Assessment

Yes. The source-backed reproduction is to persist an agent:main:explicit:...:acct:default entry in the default session store and call getSubagentDepthFromSessionStore(key) without cfg; current main returns before reading the store and falls back to segment counting depth 0.

## Solution Assessment

Yes, with a small test cleanup follow-up. Removing the early cfg guard and passing params.cfg?.session?.store is the narrowest maintainable fix because resolveStorePath() already owns the default sessions.json fallback when store config is omitted.
