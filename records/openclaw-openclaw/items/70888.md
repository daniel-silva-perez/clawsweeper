## Summary

Keep this PR open for normal maintainer review. Current main still has the no-cfg early return that skips the default session-store lookup, and the PR’s helper change is a narrow fix with regression coverage; the only concrete issue I found is non-blocking temp-dir cleanup in one added test block.

## What This Changes

The PR lets subagent depth lookup read the default on-disk session store when cfg is omitted and adds regression tests for explicit acct-suffixed requester keys and announce classification.

## Best Possible Solution

Land the narrow helper fix after normal validation and the small announce-test temp-dir cleanup, keeping the behavior scoped to generic session-depth resolution rather than changing routing policy.

## Reproduction Assessment

Yes. Persist an agent:main:explicit:...:acct:default entry with spawnDepth in the default sessions.json and call getSubagentDepthFromSessionStore(key) without cfg; current main skips the store and falls back to segment counting depth 0.

## Solution Assessment

Yes, with a small test cleanup follow-up. Removing the cfg early return and using params.cfg?.session?.store is the narrowest maintainable fix because resolveStorePath() already owns the default sessions.json fallback.
