## Summary

Keep open. Current main still skips the no-cfg on-disk session store lookup, while this PR is a narrow active fix with regression coverage; the remaining maintainer work is review, validation, and a small temp-directory cleanup in one test file.

## What This Changes

The PR removes the early no-cfg return from subagent depth lookup, lets omitted config use the default session store path, and adds regression tests for explicit acct-suffixed requester keys and announce classification.

## Best Possible Solution

Land the narrow helper fix after normal validation, keeping the change scoped to generic session-depth resolution and regression tests rather than changing routing policy.

## Reproduction Assessment

Yes. Persist an agent:main:explicit:...:acct:default entry in the default session store and call getSubagentDepthFromSessionStore(key) without cfg; current main returns before reading the store and falls back to segment-count depth 0.

## Solution Assessment

Yes, with a small test cleanup. Removing the early cfg guard and passing params.cfg?.session?.store is the narrowest maintainable fix because resolveStorePath() already owns the default sessions.json fallback.
