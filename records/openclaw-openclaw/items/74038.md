## Summary

Keep open. Current main still enables the pricing bootstrap whenever `models.pricing.enabled` is not explicitly false, so `models.mode: "replace"` does not yet skip OpenRouter/LiteLLM pricing fetches. This PR is a focused implementation for #74020 with matching tests/docs and no unexplained security-sensitive changes.

## Best Possible Solution

Keep this PR open and review/land it after CI finishes. The best fix is the narrow helper-level default change already proposed here: explicit `models.pricing.enabled` wins, omitted pricing disables remote pricing bootstrap in replace mode, local configured `cost` data still works, and tests/docs/changelog stay aligned.
