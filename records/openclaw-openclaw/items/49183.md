## Summary

Keep open. Current main has adjacent global, per-agent, provider, group, and sender tool policies, but it does not implement the RFC’s operator-scope-to-agent-tool-policy bridge or a `toolsByScope` config surface.

## Best Possible Solution

Keep this RFC open for maintainer product/design follow-up. The best implementation path is a typed, documented session or scope-aware policy input in the shared tool-policy resolver, threading authenticated gateway scopes and trusted sender identity into agent/tool resolution, preserving owner-only gates, and adding schema/docs/tests that prove narrower operator scopes cannot access denied tools while admin/full-operator sessions keep intended access.
