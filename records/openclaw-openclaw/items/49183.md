## Summary

Current main has global, per-agent, provider, group, and sender tool policies, but it still does not implement the RFC’s requested operator-scope-to-agent-tool-policy bridge or a `toolsByScope` config surface. This should stay open for maintainer product/design follow-up rather than being closed as implemented, duplicate, or ClawHub work.

## Best Possible Solution

Keep this RFC open for maintainer product/design follow-up. The best path is a typed and documented session or operator-scope-aware policy input in the shared tool-policy resolver, with authenticated gateway scopes and trusted sender identity threaded into agent/tool resolution, owner-only gates preserved, and schema/docs/tests proving narrower operator scopes cannot access denied tools while admin/full-operator sessions keep intended access.
