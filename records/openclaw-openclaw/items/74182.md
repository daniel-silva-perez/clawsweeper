## Summary

Keep open. Current main still has the relevant timeout/abort retention shape, and the linked draft PR #75008 explicitly targets this issue with closing syntax, so the issue should remain open until that PR is reviewed, validated, and merged or closed.

## What This Changes

The issue asks OpenClaw to release embedded LLM run state after timeout or abort so single-process Gateway RSS does not grow unbounded during hung or failed provider runs.

## Best Possible Solution

Review and validate #75008 as the canonical fix, including its retention test and changed gate. If heap profiling still shows growth afterward, split any remaining subagent announce retry retention into a narrower follow-up instead of closing this issue early.

## Reproduction Assessment

No high-confidence deterministic reproduction is checked in on current main. The report gives a concrete multi-hour Gateway scenario, and #75008 proposes a focused WeakRef/GC unit harness, but the merged tree does not yet contain that proof.

## Solution Assessment

Unclear until #75008 is reviewed and validated. Extracting abortable out of the run scope is a narrow, plausible fix for hung provider promises retaining attempt state, but subagent announce retry retention may need separate profiling if RSS still grows.
