## Summary

Keep open. Current main still has the inline abortable wrapper inside the embedded attempt run scope, and there is no merged regression proof that timeout/abort releases hung provider promise closures. The linked draft PR #75008 targets this exact leak and uses closing syntax, so this issue should remain open until that PR is reviewed, validated, and merged or replaced.

## What This Changes

The issue reports multi-GB Gateway RSS growth after embedded LLM run timeouts or aborts, with suspected retained run state across provider promises, event listeners, active-run maps, and subagent announce retry chains.

## Best Possible Solution

Review and land or revise #75008 as the focused fix for the provider-promise retention path, including its module-scope abortable helper and retention test. After that merges, use a bounded heap/RSS harness or targeted profiling to decide whether the subagent announce retry path needs a separate follow-up.

## Reproduction Assessment

Unclear. The reporter gave a credible live reproduction pattern against OpenClaw 2026.4.5, and current main still has a plausible code path, but there is no merged deterministic current-main heap/RSS harness that reproduces the multi-hour leak.

## Solution Assessment

Yes. Keeping this issue open while the linked draft PR is reviewed is the narrowest maintainable path because the PR targets the exact inline-closure retention shape still present on main; broader RSS and subagent announce work should be follow-up only if profiling confirms it remains.
