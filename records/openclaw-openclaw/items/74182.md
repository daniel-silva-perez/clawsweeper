## Summary

Keep open. Current main still has the suspected inline abort wrapper around embedded provider prompts, the latest release still contains that shape, and an open draft PR directly references this issue as the implementation candidate, so this should not be closed before that PR lands or is replaced.

## What This Changes

The issue reports that timed-out or aborted embedded LLM runs can retain run state and grow Gateway RSS to multi-GB in long-running single-process sessions.

## Best Possible Solution

Review and land PR #75008 or an equivalent maintainer-owned replacement that extracts the abort wrapper out of the run scope, adds a hung-provider abort retention regression test, and then profiles the remaining waitForIdle and subagent announce surfaces before closing this issue.

## Reproduction Assessment

Unclear. The report provides a credible live reproduction path over several hours with provider instability, but current main does not include a deterministic heap/RSS harness; code inspection shows a plausible retained-promise path rather than a completed reproduction proof.

## Solution Assessment

Yes for keeping this open. The narrowest maintainable path is to review the existing draft fix PR and require targeted GC/retention proof, not to close this issue or start an independent repair while that PR is open.
