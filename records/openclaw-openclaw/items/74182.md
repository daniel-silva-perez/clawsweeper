## Summary

Keep open. Current main still has the inline embedded-run `abortable` helper and timeout path matching the reported retention surface, while open draft PR #75008 is the direct implementation candidate rather than a merged fix.

## What This Changes

The issue reports retained embedded LLM run state after timeout or abort, causing unbounded Gateway RSS growth during long single-process Gateway runs.

## Best Possible Solution

Review and land PR #75008 or an equivalent replacement that moves the abort wrapper out of the run scope, adds a focused hung-provider abort retention regression test, and then profiles the remaining subagent announce retry surface before closing this issue.
