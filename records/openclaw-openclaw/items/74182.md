## Summary

Keep open. The report is concrete and current main still has plausible timeout/abort retention surfaces: active-run maps are cleared and cleanup is substantial, but there is no source, test, release note, or heap/RSS reproduction proving that aborted embedded prompt promises, waitForIdle cleanup waits, provider stream state, or subagent announce retries release all captured run state.

## What This Changes

The issue requests that embedded LLM run state be fully released after timeout or abort so long-running single-process Gateway instances do not accumulate multi-GB RSS growth.

## Best Possible Solution

Keep this open and investigate with a bounded current-main reproduction harness or live heap profile. The best fix should prove that timeout/abort drains ACTIVE_EMBEDDED_RUNS and snapshots, releases session event subscriptions, settles or detaches prompt/waitForIdle/provider stream chains, bounds subagent announce retries without retaining large closures, and then adds focused regression coverage for the smallest reproducible path.
