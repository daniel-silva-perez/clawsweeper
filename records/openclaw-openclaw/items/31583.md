## Summary

Current main still appears to leave the reported host exec path unresolved for sensitive-pattern keys configured only in `skills.entries.<skill>.env`. The env override layer gathers configured values but only allowlists sensitive names declared by skill metadata (`primaryEnv` / `requires.env`), so a configured `GOG_KEYRING_PASSWORD` is still blocked before host exec can inherit it. Open PR #50960 already targets this exact root cause with closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land #50960 or an equivalent maintainer-reviewed fix. The likely code change belongs in `src/agents/skills/env-overrides.ts`: add explicitly configured `skills.entries.<skill>.env` keys to the sensitive-key allowlist before sanitization, while preserving always-blocked dangerous host/runtime env policy. Add regression coverage for undeclared configured `_PASSWORD` and `_TOKEN` keys and verify host exec inherits the applied run-scoped environment.
