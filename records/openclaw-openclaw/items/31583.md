## Summary

Current main still leaves the reported host exec path unresolved for sensitive-pattern keys configured only in `skills.entries.<skill>.env`. The skill env layer collects configured values, but only allowlists sensitive names declared by skill metadata, so `GOG_KEYRING_PASSWORD` is still blocked for the bundled `gog` skill. Open PR #50960 already references this issue with closing syntax and targets the same root cause, so this issue should remain open until that PR or an equivalent maintainer-reviewed fix lands.

## Best Possible Solution

Keep this issue open and review/land #50960 or an equivalent maintainer-reviewed fix. The likely code change belongs in `src/agents/skills/env-overrides.ts`: add explicitly configured `skills.entries.<skill>.env` keys to the sensitive-key allowlist before sanitization, while preserving always-blocked dangerous host/runtime env policy. Add regression coverage for undeclared configured `_PASSWORD` and `_TOKEN` keys and verify host exec inherits the applied run-scoped environment.
