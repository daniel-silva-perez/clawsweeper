## Summary

Current main still blocks the reported `GOG_KEYRING_PASSWORD` case. `skills.entries.<skill>.env` values are collected for injection, but sensitive-pattern names are only re-allowed when declared by skill metadata, and the bundled `gog` skill does not declare `GOG_KEYRING_PASSWORD`. An open fix PR already references this issue with closing syntax, so the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land #50960 or an equivalent maintainer-reviewed fix. The likely code change belongs in `src/agents/skills/env-overrides.ts`: add explicitly configured `skills.entries.<skill>.env` keys to the sensitive-key allowlist before sanitization, while preserving the hard block for dangerous host/runtime override variables. Add regression coverage for undeclared configured `_PASSWORD`/`_TOKEN` keys and verify host exec inherits the applied run-scoped env.
