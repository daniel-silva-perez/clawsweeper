## Summary

Current main still blocks the reported `GOG_KEYRING_PASSWORD` path. `skills.entries.<skill>.env` values are copied into pending overrides, but configured env keys are not added to the sensitive-key allowlist unless the skill metadata declares them; the bundled `gog` skill only declares `requires.bins`. An open PR, #50960, already references this issue with closing syntax and proposes the narrow sanitizer fix, so the issue should stay open until that PR or an equivalent maintainer-reviewed fix lands.

## Best Possible Solution

Keep this issue open and review/land #50960 or an equivalent maintainer-reviewed fix. The likely code change belongs in `src/agents/skills/env-overrides.ts`: add explicitly configured `skills.entries.<skill>.env` keys to the sensitive-key allowlist before sanitization, while preserving the always-blocked dangerous host/runtime env policy. Add regression coverage for undeclared configured `_PASSWORD` and `_TOKEN` keys and verify host exec inherits the applied run-scoped environment.
