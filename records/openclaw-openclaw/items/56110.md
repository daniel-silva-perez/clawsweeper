## Summary

Keep this issue open. Current main at 6d323ee73640afdf02ed9fa3aded13da23359b98 still does not recognize, seed, auto-load, document, or configure `STATE.md` as a workspace bootstrap file. The issue also has fresh substantive support from the KinthAI comment, while related #39120 and #59050 remain broader open proposals rather than shipped implementations.

## Best Possible Solution

Keep this issue open for a core workspace-bootstrap decision. A complete implementation would add `STATE.md` to the recognized bootstrap filename contract, runtime loader, optional workspace seeding/templates if desired, hook allowlist behavior, docs, and regression tests for present files, missing markers, truncation, and sub-agent filtering. If maintainers prefer configurability instead of another default file, resolve it through #39120 and document how users should configure `STATE.md`.
