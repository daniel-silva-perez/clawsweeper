## Summary

Keep this issue open. Current main still does not define, seed, auto-load, document, or allow `STATE.md` as a recognized workspace bootstrap file. The discussion is also active and substantive: the latest commenter describes a real deployed `STATE.md` pattern, while related #39120 and #59050 are broader/open follow-ups rather than shipped implementations of this default file.

## Best Possible Solution

Keep this issue open for a core workspace-bootstrap product decision. A first-class implementation would add `STATE.md` to the bootstrap filename contract in `src/agents/workspace.ts`, wire it into runtime loading and any intended setup/template behavior, define the sub-agent filtering rule, update public docs/templates, and add regression coverage for present files, missing markers, truncation, and filtering. If maintainers prefer a configurable approach, resolve it through #39120 and explicitly document how users should configure `STATE.md`.

## Risks / Open Questions

- Adding a new always-loaded default file changes prompt budget, missing-marker behavior, and workspace surface area, so it needs an explicit maintainer product decision rather than cleanup closure.
- The requested sub-agent exclusion should be specified carefully because current docs say sub-agents only inject `AGENTS.md` and `TOOLS.md`, while current code allowlists `SOUL.md`, `IDENTITY.md`, and `USER.md` for sub-agent/cron filtering too.
- The existing `bootstrap-extra-files` hook is not enough to close this request as implemented because it rejects the `STATE.md` basename today.
