## Summary

Keep this PR open. Current main still preserves cron run session keys for scoped heartbeat wakes and still accepts forced cron session keys in heartbeat resolution, so the PR's intended fix is not implemented on main. The PR is focused and has no obvious supply-chain/security-sensitive diff, but the existing P1 review concern is real: dropping only the cron session key can lose originating-agent scope and turn a targeted cron wake into an unscoped heartbeat run.

## Best Possible Solution

Keep this PR open and revise it before landing: reject cron run session keys in heartbeat forced-session resolution and scoped wake construction, while preserving the originating agent target, likely by passing `agentId` or the agent main session key instead of dropping all targeting. Add regression coverage for forced cron-session fallback, scoped wake options, non-default agents, multi-agent configs, and any scoped wake callers that can originate from cron sessions.
