## Summary

Keep open. Current main still reuses persisted skill snapshots using only the in-memory snapshot version and agent skill filter, while neither the runtime nor persisted snapshot records workspace/source-root provenance. A workspace move can therefore leave old absolute SKILL.md paths in the prompt until some unrelated invalidation rebuilds the snapshot.

## What This Changes

The issue reports that sessions.json keeps cached skill paths from an old iCloud workspace after openclaw.json is moved to a local workspace, causing ENOENT when skills are read.

## Best Possible Solution

Add durable workspace-aware skillsSnapshot invalidation: store a normalized workspace/source-root identity or stable inputs hash with SkillSnapshot and SessionSkillSnapshot, compare it before reuse in every agent entry point, and rebuild snapshots when the configured workspace or resolved skill roots change.

## Reproduction Assessment

Yes. A high-confidence reproduction is to persist a session skillsSnapshot with version 0 and an old workspace SKILL.md path, change the agent workspace to a new path, restart so the in-memory version is 0, and send on the same session; current main will reuse the snapshot because version and skillFilter still match.

## Solution Assessment

Yes, the best fix direction is to invalidate by stable snapshot inputs rather than rewriting cached paths with a migration. That keeps sessions.json repair generic and handles both workspace moves and sibling stale-cache symptoms without special-casing iCloud.

Current main reuses persisted session skillsSnapshot entries based only on version and skillFilter, so old absolute SKILL.md paths can survive an iCloud-to-local workspace move. Implement a durable, generic snapshot-input invalidation model: add workspace/source-root provenance or a stable inputs hash to SkillSnapshot and SessionSkillSnapshot, compute it from the normalized active workspace, resolved skill roots, effective skills config/agent skill filter, and other inputs that affect the skill prompt, and compare it before reusing a persisted snapshot.
