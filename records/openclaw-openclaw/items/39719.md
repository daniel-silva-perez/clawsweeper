## Summary

Keep this open. Current main still reuses persisted skill snapshots based on snapshot version and skill filter only, while the persisted/runtime snapshot shape does not record the workspace or source-root identity needed to detect an iCloud-to-local workspace move. The newer #73765 is related to the same stale-cache family, but it tracks a sibling config-cache symptom rather than fully replacing this workspace-migration report.

## Best Possible Solution

Keep this issue open and implement workspace-aware skillsSnapshot invalidation. Add normalized workspace/source-root provenance or a stable inputs hash to SkillSnapshot and SessionSkillSnapshot, compare it before reusing snapshots, invalidate when agents.defaults.workspace or per-agent workspace changes, apply the same check in agent command, auto-reply, cron, embedded/CLI consumers, and add a regression test with a stale sessions.json absolute SKILL.md path from an old iCloud workspace.
