## Summary

Keep this issue open. Current main still has no iMessage startup rowid/cursor fence, and the self-chat `is_from_me=true` path still dispatches a cache-miss message by design. Related PR #62191 is about U+FFFD/control-character echo-key normalization, not restart cache warming, and the closed #72322 discussion describes the same missing since-rowid class rather than proving this path is fixed.

## Best Possible Solution

Implement an iMessage-owned startup replay guard in `extensions/imessage`: preferably pass and persist a since-rowid cursor or record a startup max-rowid fence before `watch.subscribe`, then add regression coverage for restart-time `is_from_me=true` self-chat rows with an empty echo cache. Avoid broad `is_from_me` suppression that would break the documented real self-chat path.
