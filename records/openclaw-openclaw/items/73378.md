## Summary

Keep this PR open. Current main still does not implement the PR’s intended LINE native sticker send path: there is inbound LINE sticker context, but no `channelData.line.sticker` carrier, no `STICKER:` directive parsing, no `pushStickerMessage` helper/export, no outbound sticker dispatch, and the block reply coalescer still drops payloads with neither text nor media. The PR is also paired with the same-author open tracker #64784, while the earlier broader PR #65270 was closed unmerged in favor of this narrower replacement.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to decide whether the generic coalescer fix should land with the LINE sticker send feature or be split, then review CI/tests and merge or request changes. After a suitable implementation lands, #64784 can remain as the broader send/receive tracker or be narrowed to remaining inbound/catalog work.
