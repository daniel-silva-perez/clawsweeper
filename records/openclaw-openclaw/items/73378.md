## Summary

Keep this PR open. Current main still lacks the LINE sticker send carrier, helper/export, outbound dispatch branch, STICKER directive parser, docs update, and channelData-only coalescer bypass, so the contribution is not obsolete. The patch is useful but not merge-ready as submitted because the added line-sticker skill is not discoverable by the plugin skill loader unless the LINE manifest declares a skills root or the skill is moved/registered correctly.

## What This Changes

The PR adds LINE native sticker sending via channelData.line.sticker and a STICKER directive, exports a pushStickerMessage helper, documents sticker usage, and changes block reply coalescing to flush channelData-only payloads.

## Best Possible Solution

Keep the focused send-side approach, but make the skill exposure match the plugin skill contract before merge, then review whether the generic coalescer bypass should remain in this PR or land as a separate core fix. After this PR lands, #64784 can continue tracking inbound enrichment or catalog work that remains outside this narrowed scope.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: a LINE reply containing STICKER:446:1988 is not recognized by hasLineDirectives, and a channelData-only payload is returned before onFlush by the coalescer. The PR adds targeted tests for the intended parser, dispatch, and coalescer paths, but I did not run tests because this review is read-only.

## Solution Assessment

No, not quite as submitted. The sticker send carrier and coalescer bypass are narrow and consistent with the existing LINE channelData pattern, but the bundled skill portion is incomplete because plugin skills load from manifest-declared directories; either register/move the skill or remove that user-visible claim before merge.
