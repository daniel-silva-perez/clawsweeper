## Summary

Keep this PR open. Current main still lacks the LINE native sticker send carrier, outbound dispatch helper, STICKER directive parser, docs/skill surface, and the channelData-only coalescer bypass that this PR proposes. The PR is active, unmerged, and paired with same-author tracker #64784; the earlier broader implementation #65270 was closed unmerged in favor of this narrower replacement.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to review the low-severity directive trust concern, validate the new StickerMessage helper against the current @line/bot-sdk/LINE Messaging API contract, decide whether to split the generic coalescer fix, and then merge or request changes. After a send-side implementation lands, #64784 can continue tracking the broader inbound/catalog follow-up work.

## Risks / Open Questions

- Aisle flagged a low-severity design concern: parsing STICKER directives from model text can let prompt-influenced output trigger sticker sends. This needs maintainer product/security judgment before merge, even though the directive is line-anchored, numeric-only, and fence-aware in the PR.
- The PR intentionally combines a generic core coalescer fix with a LINE-specific send feature. Maintainers may prefer landing the coalescer bypass separately, but that is a review/splitting question rather than a reason to close the PR.
- The PR adds a LINE-specific capability flag on a shared outbound adapter type. The latest patch adds JSDoc explaining the deferred routing gate, but maintainers should still decide whether that declaration belongs now or in a follow-up.
