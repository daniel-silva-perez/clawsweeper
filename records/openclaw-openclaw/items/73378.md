## Risks / Open Questions

- The STICKER directive is parsed from agent text, so prompt-influenced output can trigger native sticker sends. The PR validates numeric IDs and relies on LINE's allowlist, but maintainers should decide whether prompt or application-level sticker allowlists are needed.
- The PR combines a generic block-reply coalescer fix with a LINE-specific feature. The coalescer change looks relevant, but maintainers may prefer landing or reviewing it separately.
- The shared supportsLineNativePayload flag is intentionally LINE-specific. The latest patch adds JSDoc, but maintainers should confirm this is the right contract shape before merge.
