## Summary

Keep this PR open. Current main still lacks the send-side LINE sticker carrier/directive/helper and the generic channelData-only coalescer bypass, but the proposed patch is not ready to merge as-is because the added plugin skill is not registered for loading and the user-facing feature/fix has no changelog entry. Related issue #64784 remains open for the broader send/receive/catalog work, while the earlier broad PR #65270 was closed unmerged in favor of this narrower replacement. https://github.com/openclaw/openclaw/pull/73378

## What This Changes

The PR adds LINE native sticker sending through `channelData.line.sticker` and a `STICKER:packageId:stickerId` directive, introduces a LINE sticker send helper and docs/skill text, and changes the block-reply coalescer to pass through channelData-only payloads with tests.

## Best Possible Solution

Address the two patch findings, then review the narrowed send-side implementation normally: keep sticker transport owned by the LINE plugin, keep the coalescer change generic if maintainers accept that scope, and leave receive-side enrichment/catalog work tracked by #64784.

## Reproduction Assessment

Yes. On current main, code inspection reproduces the missing send path: there is no `LineChannelData.sticker`, no `pushStickerMessage`, no `STICKER` parser, and channelData-only coalesced payloads still return before flush; the PR findings reproduce through manifest/docs/changelog inspection.

## Solution Assessment

No, not as-is. The proposed send path is mostly the narrow maintainable direction, but the bundled skill will not load without manifest registration and the release notes policy is unmet; after those fixes, maintainer review can decide whether to split the coalescer change.
