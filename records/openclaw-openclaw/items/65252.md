## Summary

Keep open. Current main has bundle-backed LSP startup and hover/definition/references tools, but it does not collect publishDiagnostics notifications, retain server health or performance data, or expose the requested in-editor diagnostics dashboard. The paired same-author LSP setup issue #65251 is still open, so this should be scoped with that work rather than closed independently.

## What This Changes

Request to add real-time in-editor LSP diagnostics, server health, performance metrics, troubleshooting, logs, history, and export/share capabilities.

## Best Possible Solution

Define a generic LSP diagnostics contract first: capture server lifecycle, stderr/errors, request latency, and publishDiagnostics events in the owner runtime or a plugin service, expose that through a documented plugin/gateway API, then choose whether the user-facing surface belongs in Control UI, a plugin-served UI, or an external editor integration.
