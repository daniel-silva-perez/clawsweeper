## Summary

Keep open. Current main still does not implement the requested config flag to disable tokenized dashboard URLs plus direct tailnet/IP Control UI access with no token, no HTTPS, and no pairing. The implemented alternatives remain narrower: loopback-only `gateway.auth.mode: "none"`, Tailscale Serve identity-header auth for Control UI WebSocket, and trusted-proxy auth.

## Best Possible Solution

Keep this open for maintainer product/security judgment. A safe implementation would need a narrowly named trusted-tailnet/private-ingress mode, explicit warnings, origin and HTTP API scope handling, and tests covering Control UI WebSocket, HTTP API, device identity, pairing, direct tailnet/IP, Tailscale Serve, and trusted-proxy interactions. If maintainers reject direct non-loopback no-auth as policy, close manually with that rationale and point users to Tailscale Serve identity-header auth or trusted-proxy auth.
