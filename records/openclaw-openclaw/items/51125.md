## Summary

Keep this PR open. Current main still folds fullwidth ASCII and Unicode angle-bracket homoglyphs for external-content boundary markers, but it does not normalize Cyrillic or Greek letter confusables, so the PR's central security problem is not implemented on main. The PR is a focused candidate, but it should be rebased onto the current index-mapped sanitizer and completed for the missing lowercase Cyrillic marker alphabet before merge.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase the confusable folding onto foldMarkerTextWithIndexMap, include a complete marker-alphabet confusable set including Cyrillic small te U+0442, add focused regression tests for Cyrillic/Greek uppercase and lowercase spoof markers plus surrounding-text preservation, then run the targeted external-content tests and the appropriate changed gate before merge.
