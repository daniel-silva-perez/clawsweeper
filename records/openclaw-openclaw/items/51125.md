## Summary

Keep this PR open. Current main still lacks Cyrillic/Greek confusable folding for external-content boundary markers, and the PR remains a relevant security fix candidate, but it needs rebasing onto the current index-mapped sanitizer and one remaining lowercase Cyrillic gap should be addressed before merge.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase the confusable folding onto `foldMarkerTextWithIndexMap`, include a complete marker-alphabet confusable set including Cyrillic small te U+0442, add uppercase/lowercase Cyrillic and Greek spoof regression tests plus surrounding-text preservation coverage, then run the targeted external-content tests and changed gate before merge.
