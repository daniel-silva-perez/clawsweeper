## Summary

Keep this PR open. Current main still does not normalize Cyrillic or Greek confusable letters in external-content boundary markers, so the PR is not obsolete or implemented on main. It remains a focused security fix candidate, but it needs rebasing onto the current index-mapped sanitizer and should address the remaining lowercase Cyrillic U+0442 gap before merge.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase the confusable folding onto foldMarkerTextWithIndexMap, include a complete marker-alphabet confusable set including Cyrillic small te U+0442, add uppercase/lowercase Cyrillic and Greek spoof regression tests plus surrounding-text preservation coverage, then run the targeted external-content tests and the appropriate changed gate before merge.
