## Summary

Keep open. Current `origin/main` still prints the best-effort multi-gateway advisory through the error output path, while the docs describe multiple isolated gateways as supported and the deep scan as warning/advisory diagnostics; the provided related-item context does not show a merged fix or canonical superseding item.

## What This Changes

The issue asks for the multi-gateway advisory in human CLI output to be emitted at warning or informational severity instead of the error channel.

## Best Possible Solution

Change only the non-empty `extraServices` advisory branch in `printDaemonStatus` to render as warning/info human output, preserve error output for real gateway/service/probe failures, add a focused unit test for the output channel, and credit the reporter in the active changelog fixes section.

## Solution Assessment
