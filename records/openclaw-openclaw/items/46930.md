## Summary

Keep open. Current main still emits the best-effort multi-gateway advisory through the error output path, while the docs describe the same condition as a warning/advisory for a setup that can be intentional; I did not find an open fixing PR or canonical superseding item.

## What This Changes

The issue requests downgrading the `gateway status --deep` multi-gateway advisory from error-level output to warning or info-level output.

## Best Possible Solution

Land a narrow CLI fix in `printDaemonStatus` that renders only the non-empty `extraServices` advisory as warning/info output, preserves error output for real service/config/probe failures, adds focused unit coverage for non-empty `extraServices`, and credits the reporter in the active changelog fixes section.

## Solution Assessment
