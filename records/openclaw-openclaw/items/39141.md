## Summary

Keep this issue open. Current main still lacks the prerequisite per-session activity/attention RPC and WebSocket state-change contract from #39127, and the repository only has adjacent diagnostics, task audit, channel health, and subagent recovery surfaces rather than the requested opt-in gateway-side watchdog with deduped operator alerts and optional tokenized summaries.

## Best Possible Solution

Keep this issue open until #39127, or an equivalent per-session activity/attention API with WS state-change events, lands. Then implement a small opt-in watchdog that consumes that contract, emits no-token structured alerts with configurable thresholds and dedupe/cooldowns, exposes safe UI/log/webhook/plugin delivery seams, gates optional tokenized summaries behind explicit config and rate limits, and adds focused protocol/runtime/docs tests. Delivery adapters can move to plugins where the API is sufficient, but the missing core activity seam is still the blocker.
