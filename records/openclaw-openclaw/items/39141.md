## Summary

Keep open. Current main still does not expose the prerequisite per-session activity/attention API or WebSocket state-change event from #39127, and the existing diagnostics/task/channel-health surfaces are adjacent rather than the requested opt-in gateway watchdog with deduped operator alerts and optional tokenized summaries.

## Best Possible Solution

Keep this issue open until #39127, or an equivalent per-session activity/attention API with WebSocket state-change events, lands. Then implement a small opt-in gateway watchdog that consumes that contract, emits no-token structured alerts with configurable thresholds and dedupe/cooldowns, exposes safe UI/log/webhook/plugin delivery seams, gates optional tokenized summaries behind explicit config and rate limits, and adds focused protocol/runtime/docs tests.
