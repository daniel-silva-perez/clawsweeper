## Summary

Keep open. Current main still exposes the full catalog through the Web chat configured-model path and the text `/models` command when no allowlist is configured, and the issue discussion correctly notes that fixing only the interactive picker helper would leave `/models` unresolved.

## What This Changes

The issue asks for `/models` and the Web chat model picker to default to configured or authenticated provider models instead of showing the entire built-in catalog.

## Best Possible Solution

Introduce one shared default-visible-model policy used by Gateway/Web chat, `/models`, and the interactive picker: prefer explicit allowlists and configured provider models, include auth-backed provider catalog entries when usable credentials exist, and reserve the full catalog for an explicit `all`/`view: "all"` browse path with updated docs and regression tests.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/74423. Current main shows the full built-in model catalog in Web chat and `/models` by default when the user has only configured or authenticated one or two providers. Implement a shared default-visible-model policy across the Gateway configured model view, the Web chat dropdown path, the text `/models` command, and the interactive model picker. The default should prefer `agents.defaults.models`, then explicit `models.providers.*.models`, then provider catalog entries backed by usable auth/profile/env/custom-provider credentials when any usable provider auth exists. Preserve explicit catalog discovery through `models.list` with `view: "all"` and an explicit `/models all` or `/models --all` style path. Preserve manual model entry and provider setup paths. Do not hard-code bundled provider ids in core; reuse existing auth/profile/model-selection helpers or add a generic helper. Update docs for the new default and add a user-facing changelog fix entry with reporter credit. Do not change unrelated model-selection, provider runtime, or package metadata behavior.
