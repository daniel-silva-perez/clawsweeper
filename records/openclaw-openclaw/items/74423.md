## Summary

Keep open. Current main still has catalog-wide fallback behavior for both the Gateway configured model view used by Web chat and the text `/models` provider list, so the reported auth-scoping problem is not fixed or superseded by the related model-dropdown sync work.

## What This Changes

Requests that `/models` and the Web chat model picker hide unauthenticated provider models by default and reserve full-catalog browsing for an explicit all/diagnostic path.

## Best Possible Solution

The model picker surfaces should prefer explicit `agents.defaults.models` and `models.providers.*.models`, then auth-backed provider catalogs when any usable provider auth exists, while keeping `view: "all"` or an explicit `all` command path for catalog discovery and updating docs/tests to describe the new default.
