## Summary

Keep open. The linked stale-heartbeat bug is still present on current main and latest stable, and this PR targets the right shared helper, but the patch currently rewrites any line that starts with `Current time:`. Because cron reminder content is passed through verbatim before the helper runs, that can corrupt user-authored reminder text.

## What This Changes

The PR changes `appendCronStyleCurrentTimeLine` to refresh or collapse existing line-bound `Current time:` entries and adds colocated Vitest coverage for empty input, appending, refreshing, and duplicate collapse.

## Best Possible Solution

Keep the shared-helper fix, but make the replacement recognize only OpenClaw-generated timestamp lines, or add a backward-compatible internal marker, while preserving user-authored `Current time:` content and adding a regression test for that preservation case.

## Reproduction Assessment

Yes. The original bug is reproducible by passing a helper output containing an old generated `Current time:` line back into `appendCronStyleCurrentTimeLine` with a newer `nowMs`; the PR regression is reproducible by passing reminder text with a user-authored line starting `Current time:` through the new matcher.

## Solution Assessment

No. The helper-level refresh is the right narrow boundary, but the current broad regex is not the best implementation because it cannot distinguish generated timestamp lines from user-authored cron or workspace content.
