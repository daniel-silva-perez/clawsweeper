## Summary

Keep open. The PR targets the real stale timestamp bug tracked by #44993, but it should not merge as-is because the new matcher can rewrite user-authored cron reminder content that starts with `Current time:`.

## What This Changes

The PR changes `src/agents/current-time.ts` to refresh existing line-bound `Current time:` markers and adds `src/agents/current-time.test.ts` coverage for empty input, append, refresh, and duplicate-collapse behavior.

## Best Possible Solution

Keep the shared-helper fix, but restrict refresh/collapse to OpenClaw-injected timestamp lines, or add an explicit internal marker if compatible, while preserving user-authored `Current time:` reminder text and adding regression coverage for that case.

## Reproduction Assessment

Yes. The stale bug is reproducible by calling `appendCronStyleCurrentTimeLine` once with an 08:00 `nowMs`, then passing that output back with a 10:00 `nowMs`; current main returns the old line unchanged. The PR regression is reproducible by feeding cron reminder text containing a line that starts with `Current time:` through the helper, where the broad matcher would replace that reminder line.

## Solution Assessment

No. Refreshing the helper is the right narrow boundary, but the PR's broad `^Current time:.*$` matcher is not the best implementation because it cannot distinguish injected timestamp lines from user-authored content.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Restrict replacement to injected time lines:** `src/agents/current-time.ts:35`
  - body: The matcher rewrites any line starting with `Current time:`, but cron reminder text is inserted verbatim before this helper runs. A reminder line like `Current time: check the server clock` would be overwritten with the generated timestamp, and later matching lines would be dropped, changing the user's scheduled message. Please match only OpenClaw's injected timestamp format or add a sentinel, and cover user-authored `Current time:` lines.
  - confidence: 0.92
