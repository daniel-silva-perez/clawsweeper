## Summary

Keep open. The PR targets a real stale heartbeat/cron timestamp bug that is still present on current main and latest stable, but the proposed matcher is too broad and would overwrite user-authored cron reminder lines that begin with `Current time:`.

## What This Changes

The PR changes the shared current-time helper to refresh existing line-bound `Current time:` entries and adds colocated tests for empty input, append, refresh, and duplicate-collapse cases.

## Best Possible Solution

Keep the shared-helper repair, but refresh only OpenClaw-injected timestamp lines, either by matching the generated timestamp shape or by introducing a compatible internal marker, and add regression coverage that preserves user-authored `Current time:` reminder content.

## Reproduction Assessment

Yes. The original bug is reproducible by calling `appendCronStyleCurrentTimeLine` with an 08:00 injected line and then again with a 10:00 `nowMs`; current main returns the old line. The PR regression is reproducible by passing cron reminder text with a line like `Current time: check the server clock`, which the new regex would replace or drop.

## Solution Assessment

No. Fixing the shared helper is the right boundary, but replacing every line that starts with `Current time:` is not the narrowest maintainable implementation because cron and workspace text can contain that phrase as user-authored content.
