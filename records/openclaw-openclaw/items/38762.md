## Summary

Keep open. Current main still ignores top-level `ref` when `browser action="act"` also receives an object `request`, so the reported mixed parameter shape remains unsupported. The related compatibility PR #38790 is closed unmerged, and a search found no open PR currently paired with this issue.

## Best Possible Solution

Land a narrow browser-tool compatibility normalizer: when `action: "act"` receives an object `request`, clone it and fill only missing `request.ref`, `request.targetId`, and finite numeric `request.timeoutMs` from top-level params. Preserve nested request precedence, keep fully flattened legacy behavior unchanged, and add focused browser-tool tests plus a user-facing changelog fix entry.
