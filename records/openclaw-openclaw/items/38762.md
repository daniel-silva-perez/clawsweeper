## Summary

Keep this issue open. Current main still returns the nested `request` object for `browser action="act"` before considering top-level `ref`, `targetId`, or `timeoutMs`, so the mixed call shape from the report remains unsupported. The misleading service-reachability wrapping appears improved in current browser fetch code, but the underlying parameter-position inconsistency is still present, and the related compatibility PR #38790 is closed unmerged.

## Best Possible Solution

Land a narrow browser-tool compatibility normalizer in `extensions/browser/src/browser-tool.ts`: when `action: "act"` receives an object `request`, clone it and fill only missing `request.ref`, `request.targetId`, and finite numeric `request.timeoutMs` from top-level params. Preserve nested request precedence, keep fully flattened legacy behavior unchanged, add focused tests, and include a user-facing changelog fix entry crediting the reporter and prior contributor.
