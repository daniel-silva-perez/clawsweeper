## Summary

Keep this PR open. Current main still has the strict Tailscale JSON parsing sites the PR targets, but the PR's shared `return {}` behavior would change whois/auth cache semantics and Funnel diagnostics instead of only preventing crashes.

## Best Possible Solution

Keep this PR open and request a narrower fix: preserve parse failures as errors for whois/auth and as an unknown Funnel status, only tolerating empty or noisy output where that is semantically valid. Add focused regression tests for malformed `tailscale status --json`, malformed `tailscale whois --json` cache TTL behavior, and malformed `tailscale funnel status --json` diagnostics before merge.
