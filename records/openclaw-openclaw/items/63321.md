## Summary

Keep this PR open. Current main still has the strict Tailscale JSON parsing sites the PR targets, but the proposed shared `return {}` behavior would mask malformed output as a valid empty object and alter Tailscale Serve auth cache semantics and Funnel diagnostics instead of only preventing crashes.

## What This Changes

This PR changes `src/infra/tailscale.ts` so `parsePossiblyNoisyJsonObject` catches `JSON.parse` failures and returns `{}`, and so `ensureFunnel` parses `tailscale funnel status --json` through that helper.

## Best Possible Solution

Keep this PR open and request a narrower revision: prevent user-facing crashes from malformed Tailscale output where appropriate, but preserve parse failures as errors for whois/auth cache TTL behavior and treat malformed Funnel status as unknown rather than disabled. Add focused regression tests for malformed `tailscale status --json`, malformed `tailscale whois --json` cache TTL behavior, and malformed `tailscale funnel status --json` diagnostics before merge.
