## Summary

Keep this issue open. Current main still has the compact `openclaw devices list` paired-device table and does not expose client purpose, OS/device family, last active detail, full/detail output, notes, or rename support. The useful metadata is already stored and transported, so the remaining work is a CLI/workflow presentation feature. Open PR #57449 and PR #55973 are relevant attempts, and #13870 tracks the editable human-friendly name/label part.

## Best Possible Solution

Keep this issue open until a PR lands. The best path is to review/rebase #57449 or a successor for compact Device ID/Name, Client, OS/device family, and Last active display; consider #55973 timestamp semantics; preserve `--json`; sanitize all device-controlled fields; and leave editable names/notes aligned with #13870 rather than inventing a competing device-label model.
