## Summary

Keep this issue open. Current main still stores and transports useful device metadata, but the human `openclaw devices list` output remains a compact table without client purpose, platform/device family, detail/full output, notes, or rename support. Open PR #57449 and PR #55973 are relevant implementation attempts, but neither is merged, and #13870 separately tracks editable names/labels.

## Best Possible Solution

Keep this issue open until a PR lands. The best path is to review/rebase #57449 or a successor for safe compact Device ID/Name, Client, OS, and Last active display, optionally fold in #55973 timestamp semantics, preserve existing `--json` behavior, keep device-controlled fields sanitized, and coordinate editable names/notes with #13870 rather than inventing a competing label model.
