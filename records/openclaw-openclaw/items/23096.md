## Summary

Keep open. Current main has the generic SecretRef env/file/exec transport baseline, but it still does not include a Bitwarden/Vaultwarden integration, bundled bw resolver, or secret-provider plugin seam. The recent maintainer discussion also treats this as useful follow-up work rather than something obsolete or implemented.

## Best Possible Solution

Keep this PR open as the focused Bitwarden/Vaultwarden implementation candidate. The maintainer path should be to finish or reject the manifest-first secret-provider plugin seam in #72548; if it lands, port this work into a secrets-bitwarden plugin with manifest-declared source ownership, SDK factory, docs, tests, and security review. If maintainers choose the generic exec-recipe path instead, re-scope this PR around a hardened wrapper script and docs, then update the PR body and closing references before review.
