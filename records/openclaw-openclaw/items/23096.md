## Summary

Keep open. Current main still does not ship the Bitwarden/Vaultwarden secret integration from this PR, and the recent discussion points to porting or reviewing it after the secrets-provider/plugin seam rather than closing it as implemented, superseded, or ClawHub-only.

## Best Possible Solution

Keep this PR open as the focused Bitwarden/Vaultwarden implementation candidate. The best maintainer path is to settle or land the manifest-first secret-provider plugin seam, then port this into a secrets-bitwarden plugin with manifest-declared ownership, runtime dependency handling, docs, tests, and security review. If the project instead keeps the generic exec-wrapper approach, rebase this PR and harden the wrapper/docs/tests rather than closing it as implemented.
