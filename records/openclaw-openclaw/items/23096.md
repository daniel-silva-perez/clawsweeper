## Summary

Keep open. Current main still does not ship a Bitwarden/Vaultwarden secret provider, resolver script, docs section, or manifest secret-provider plugin contract. The PR is stale and should be rebased or ported, but maintainer discussion points to it as a plausible focused follow-up after the secrets baseline and future plugin seam, not something safely closeable as implemented, superseded, or ClawHub-only.

## Best Possible Solution

Keep this PR open as the focused Bitwarden/Vaultwarden implementation candidate. The best maintainer path is to settle #72548 or an equivalent secret-provider plugin contract, then port this work into an external or bundled secrets-bitwarden plugin with a manifest-declared bw provider, runtime dependency handling, docs, tests, and a dedicated security review. If maintainers prefer the generic exec-wrapper route, rebase this PR and harden the wrapper/package/docs rather than closing it as implemented.
