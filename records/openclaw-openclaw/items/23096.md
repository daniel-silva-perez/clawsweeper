## Summary

Keep open. Current main still has the shipped generic SecretRef transport baseline only (`env`, `file`, `exec`), with no Bitwarden/Vaultwarden source, bundled `bw` resolver, Bitwarden docs, or merged secret-provider plugin seam. The PR remains a plausible implementation candidate, and recent maintainer discussion points it toward #72548 or a hardened exec-wrapper/docs path rather than treating it as obsolete.

## Best Possible Solution

Keep this PR open as the focused Bitwarden/Vaultwarden implementation candidate. The best maintainer path is to resolve #72548 first; if that plugin seam lands, port this work into a `secrets-bitwarden` plugin with `contracts.secretProviders`, SDK factory, config validation, docs, and tests. If maintainers choose to stay with generic exec recipes, rescope this PR to a hardened wrapper script plus docs, refresh the PR body/closing references, coordinate with #29110, and complete focused command-execution and error-redaction review before merge.
