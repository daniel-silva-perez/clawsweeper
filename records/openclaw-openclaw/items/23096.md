## Summary

Keep open. Current main still does not ship the Bitwarden/Vaultwarden integration from this PR, and the latest maintainer discussion points to porting it after a manifest-first secret-provider plugin seam rather than closing it as implemented, superseded, or ClawHub-only.

## Best Possible Solution

Keep this PR open as the focused Bitwarden/Vaultwarden implementation candidate. The best maintainer path is to land or reject the manifest-first secret-provider plugin seam, then either port this into a `secrets-bitwarden` plugin with manifest-declared ownership, SDK factory, docs, tests, and security review, or explicitly re-scope and harden the generic exec-wrapper docs/script approach if the project chooses to keep vendor integrations as exec recipes.
