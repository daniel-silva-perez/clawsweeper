## Summary

Keep open. Current main still has only the generic SecretRef env/file/exec transport baseline; it does not include a Bitwarden/Vaultwarden provider, bundled bw resolver, Bitwarden docs, or a secret-provider plugin contract. The PR remains a plausible implementation candidate, but it needs rebase/alignment with the in-flight secret-provider plugin seam and a focused security review before it can land.

## Best Possible Solution

Keep this PR open as the focused Bitwarden/Vaultwarden implementation candidate. The best maintainer path is to finish or reject #72548 first; if that seam lands, port this work into a secrets-bitwarden plugin with manifest-declared source ownership, SDK factory, docs, tests, and security review. If maintainers choose to stay with generic exec recipes, re-scope this PR around a hardened wrapper script and docs, update the stale body/closing references, and review PATH/env/error-redaction behavior before merge.
