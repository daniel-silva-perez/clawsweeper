## Summary

Keep this PR open. Current main has the shipped generic SecretRef env/file/exec transport baseline, but it still does not include a Bitwarden/Vaultwarden provider, bundled bw resolver, Bitwarden docs, or a manifest/SDK secret-provider plugin seam. The latest maintainer discussion also points this work toward the in-flight #72548 plugin seam instead of treating it as obsolete or implemented.

## Best Possible Solution

Keep this PR open as the focused Bitwarden/Vaultwarden implementation candidate. The best maintainer path is to finish or reject #72548 first; if that seam lands, port this work into a `secrets-bitwarden` plugin with manifest-declared source ownership, SDK factory, docs, tests, and security review. If maintainers choose to stay with generic exec recipes, re-scope this PR around a hardened wrapper script and docs, update the stale PR body and closing references, and review PATH/env/error-redaction behavior before merge.
