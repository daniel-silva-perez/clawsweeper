## Summary

Keep open. Current main still does not expose or plumb a `silent` option for cron failure alerts: the cron type, gateway validation schema, global config schema, Control UI form state, and runtime delivery path all omit it, while the outbound and Telegram message layers already know how to send silent messages.

## What This Changes

The issue requests adding a `silent` boolean to cron `failureAlert` so Telegram failure alerts can be delivered without notification sound.

## Best Possible Solution

Add a narrow, backward-compatible `silent?: boolean` cron failure-alert option across the per-job and global config contracts, validate it in the gateway schema, preserve it through Control UI create/edit flows, pass it only through the announce delivery path into outbound delivery, and update focused tests plus public docs.
