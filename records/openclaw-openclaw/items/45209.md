## Summary

Keep open. Current main and the latest release still lack a Feishu Bitable tool toggle, and the PR is the canonical focused implementation for the related feature request, but it should be refreshed and fixed before merge because adding a new `Required<FeishuToolsConfig>` field leaves at least one typed test mock incomplete.

## What This Changes

The PR adds a Feishu `tools.bitable` opt-out flag to the plugin type, schema, defaults, merged tool-config helper, and skips Bitable tool registration when disabled.

## Best Possible Solution

Land the Feishu-owned toggle after refreshing the branch, updating typed mocks/tests, and documenting the new `tools.bitable` config path so users can intentionally reduce the Feishu tool surface without changing core behavior.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the gap: the Feishu config type/schema/defaults omit `bitable`, and `registerFeishuBitableTools` registers Bitable tools for any enabled Feishu account without a category guard.

## Solution Assessment

Mostly yes. The implementation direction is the narrowest maintainable fix because it stays inside the Feishu plugin and mirrors existing tool-category guards, but the PR should update typed test mocks and add a disabled-registration regression before merge.
