## Summary

Keep open. Current main and the latest release still lack a Feishu `tools.bitable` toggle, so this PR is not obsolete and it tracks the remaining work from #45195. The branch is useful but not merge-ready because the added `bitable` field widens `Required<FeishuToolsConfig>` while an existing typed test mock still returns the old six-key object.

## What This Changes

This PR adds a Feishu `tools.bitable` config flag to the Feishu plugin type/schema/defaults/account-merge path and skips Bitable tool registration when the merged setting is disabled.
