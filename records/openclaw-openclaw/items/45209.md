## Summary

Keep open. Current main still lacks a Feishu Bitable tool toggle, and this PR is the right narrow owner-local direction, but the branch is not ready to merge because adding `bitable` to `FeishuToolsConfig` makes an existing typed Feishu test mock incomplete and the user-facing config key is not documented or covered by focused regression tests.

## What This Changes

The PR adds an opt-out `tools.bitable` flag to Feishu tool config types, schema, defaults, account merge logic, and Bitable tool registration.

## Best Possible Solution

Land this as a Feishu-owned opt-out toggle after adding the missing typed mock update, focused Bitable disabled-registration coverage, and public documentation for `channels.feishu.tools.bitable` with its default-enabled behavior.

## Reproduction Assessment

Yes. Source inspection on current main is enough: the Feishu config type/schema/defaults omit `bitable`, and `registerFeishuBitableTools` proceeds whenever an enabled Feishu account exists.

## Solution Assessment

Mostly yes. Mirroring the existing Feishu tool-category toggles inside the Feishu plugin is the narrowest maintainable approach, but the PR should include matching tests and docs before merge.
