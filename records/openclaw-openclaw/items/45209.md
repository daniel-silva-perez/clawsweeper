## Summary

Keep open. Current main still lacks the requested Feishu Bitable opt-out, and this PR is useful owner-local work, but it should not be merged as-is because the new required config key leaves an existing typed Feishu test mock incomplete and the user-facing option has no focused regression coverage or docs.

## What This Changes

The PR adds a `tools.bitable` boolean to Feishu config types, schema, defaults, account merge logic, and Bitable tool registration so Bitable tools can remain enabled by default but be skipped when disabled in config.

## Best Possible Solution

Land the Feishu-owned opt-out toggle with the same semantics as the existing tool-category toggles, plus typed mock updates, disabled-registration coverage, public Feishu docs, and a credited changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main is enough: the Feishu config type/schema/defaults omit `bitable`, and Bitable registration proceeds whenever an enabled Feishu account exists.

## Solution Assessment

Mostly yes. Mirroring the existing Feishu tool-category toggle pattern inside the Feishu plugin is the narrowest maintainable solution, but the current PR should add the missing typed test update, focused coverage, and docs before merge.
