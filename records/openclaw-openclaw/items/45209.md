## Summary

Keep open. Current main and the latest release still lack the requested Feishu `tools.bitable` toggle, and this PR remains the canonical focused implementation for the already-closed companion request. The diff matches the existing Feishu tool-category toggle pattern and I did not find a blocking correctness or security issue.

## What This Changes

This PR adds an opt-out `tools.bitable` flag to the Feishu plugin type, schema, defaults, merged tool-config helper, and Bitable tool registration path.

## Best Possible Solution

Land the focused Feishu plugin toggle after normal maintainer review, ideally with a small regression test proving `tools.bitable: false` is accepted and skips Bitable tool registration while the default remains enabled.

## Reproduction Assessment

Yes. Source inspection reproduces the underlying gap: current main has no `tools.bitable` schema/default/type support, and `registerFeishuBitableTools` registers whenever any Feishu account is enabled.

## Solution Assessment

Yes. The PR’s approach is the narrowest maintainable fix because it stays inside the Feishu plugin boundary and mirrors the existing `wiki`, `drive`, and `perm` registration guards; adding a targeted test would make the merge safer.
