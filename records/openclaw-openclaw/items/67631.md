## Summary

Keep open. Current main still lacks automatic macOS app LaunchAgent ownership detection in the CLI doctor/gateway repair path, and an open PR already references this issue with GitHub closing syntax, so the issue should stay open until that PR is merged, replaced, or closed.

## What This Changes

The issue requests preventing macOS CLI doctor/gateway repair from installing, starting, or restarting a standalone `ai.openclaw.gateway` LaunchAgent when the Mac app `ai.openclaw.mac` LaunchAgent owns the local gateway lifecycle.

## Best Possible Solution

Make macOS gateway ownership explicit: terminal doctor/gateway repair should detect an installed or loaded `ai.openclaw.mac` LaunchAgent and skip or defer standalone `ai.openclaw.gateway` install/start/restart with clear output, while the app and CLI converge on one canonical owner. The fix should include focused regression coverage for doctor repair, launchd scanning, and app attach-existing fallback plus macOS gateway/doctor docs updates.
