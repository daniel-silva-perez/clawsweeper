## Summary

Keep open. Current main still lacks macOS app LaunchAgent ownership detection in the CLI doctor/gateway repair path, and PR #73974 already references this issue with closing syntax, so this should remain open until that implementation is merged, replaced, or closed.

## What This Changes

Request that macOS app and CLI gateway repair avoid creating or restarting a duplicate standalone `ai.openclaw.gateway` LaunchAgent when the app-owned `ai.openclaw.mac` runtime is present, with docs/status behavior aligned.

## Best Possible Solution

Make macOS gateway ownership explicit: terminal doctor/gateway repair should detect an installed or loaded app LaunchAgent and skip or defer standalone gateway install/start/restart with clear output, while app and CLI docs describe the canonical owner and focused regression tests cover duplicate LaunchAgent prevention.
