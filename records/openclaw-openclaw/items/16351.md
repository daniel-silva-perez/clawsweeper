## Summary

Keep this issue open. Current main implements `openclaw config validate` and has config write protections that preserve env/SecretRef-style references, but the remaining root-command request is still unresolved: bare `openclaw config` is still documented and implemented as an entry into the guided configure wizard rather than showing help on an already configured system.

## Best Possible Solution

Keep this issue open until a maintainer either implements the remaining no-subcommand behavior or explicitly decides the wizard default is intentional. The likely implementation belongs in `src/cli/config-cli.ts` and possibly `src/commands/configure.commands.ts`: detect an already configured system for bare `openclaw config` and print config help, while preserving `openclaw config --section ...`, `config validate`, config write validation, env/SecretRef preservation, and updating `docs.openclaw.ai/cli/config` plus focused tests.
