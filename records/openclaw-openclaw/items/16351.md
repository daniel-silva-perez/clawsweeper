## Summary

Keep this issue open. Current main implements the `openclaw config validate` and secret/env-reference preservation pieces, but the remaining root-command request is still unresolved: bare `openclaw config` is still implemented and documented as guided setup instead of showing help on an already configured system.

## Best Possible Solution

Keep this issue open until a maintainer either implements the remaining no-subcommand behavior or explicitly decides the wizard default is intentional. The likely implementation belongs around `src/cli/config-cli.ts` and `src/commands/configure.commands.ts`: detect an already configured system for bare `openclaw config` and print config help, while preserving first-run setup, `--section`, `config validate`, config write validation, env/SecretRef preservation, and updating `docs.openclaw.ai/cli/config` plus focused tests.
