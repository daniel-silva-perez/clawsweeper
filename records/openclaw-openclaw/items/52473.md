## Summary

Keep open. Current main still handles explicit skill slash commands by resolving the command and rewriting the user message to “Use the <skill> skill…”, except for `command-dispatch: tool`; it does not read the matching `SKILL.md` or inject its contents before the agent run. The current system prompt and docs still describe the softer model-mediated path: list available skills, then ask the model to read the selected file.

## Best Possible Solution

Keep this issue open and implement a core deterministic activation path: when `/skill <name>` or a direct user-invocable skill command resolves to a normal SKILL.md-backed skill, load the selected `SKILL.md` through the skills subsystem and inject it into the agent run context before the user arguments, while preserving the existing `command-dispatch: tool` bypass. Add regression coverage around explicit slash invocation, direct skill aliases, authorization, prompt-visible/disable-model-invocation behavior, and transcript/provider compatibility.
