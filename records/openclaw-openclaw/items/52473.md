## Summary

Keep open. Current main still resolves explicit `/skill <name>` and direct skill slash commands, but normal SKILL.md-backed skills are converted into a regular user prompt that asks the model to use the skill. The selected `SKILL.md` is not read by the gateway or injected into the run context, and docs still describe model-mediated on-demand reading.

## Best Possible Solution

Keep this issue open and implement a deterministic core activation path for explicit skill invocations. When `/skill <name>` or a direct user-invocable skill command resolves to a normal SKILL.md-backed skill, the gateway should load that selected SKILL.md through the existing skills subsystem and inject it into the agent run context before the user args. Preserve the existing `command-dispatch: tool` bypass, and add focused regression coverage for direct aliases, `/skill <name>`, authorization, `disable-model-invocation`, transcript/provider compatibility, and docs.
