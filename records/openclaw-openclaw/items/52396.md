## Summary

Keep this issue open. Current main still has the lower-level Discord retry runner limited to `RateLimitError`/HTTP 429, current tests still assert plain network errors are not retried, and the same-author implementation candidate #52480 remains open with closing syntax.

## Best Possible Solution

Keep this issue open and review #52480 as the implementation candidate. The likely fix belongs in the Discord plugin retry classifier, with focused tests for the runner/direct send path and outbound adapter behavior, while preserving `RateLimitError.retry_after` handling and avoiding duplicate 5xx retry layers.
