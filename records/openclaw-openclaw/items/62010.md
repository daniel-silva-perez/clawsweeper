## Summary

Keep open. Current main still lacks the requested WhatsApp config, and the latest PR is useful but not merge-ready because it puts WhatsApp-specific root policy in core outbound delivery, leaves bundled channel metadata stale, and accepts `file://` root entries that downstream local-root enforcement will not honor.

## What This Changes

The PR adds WhatsApp channel/account `mediaLocalRoots` config, validates it, merges configured roots into outbound media sends, updates the config baseline hash, and adds regression coverage.

## Best Possible Solution

Keep the WhatsApp-scoped feature direction, but resolve configured roots in the WhatsApp-owned outbound/config path or a generic documented adapter seam, normalize or reject configured root URI forms consistently with local media loading, regenerate bundled channel metadata and config docs baselines, then rebase and run the targeted config/outbound checks.

## Reproduction Assessment

Yes. A focused check can configure `channels.whatsapp.mediaLocalRoots: ["file:///tmp/wa-media"]`, send a WhatsApp local media file under `/tmp/wa-media`, and observe that the root entry is not treated as `/tmp/wa-media` because only media URLs are converted from `file://`, not root entries. The stale metadata issue is also directly reproducible with the channel metadata check after adding the schema field.

## Solution Assessment

No. The feature is plausible, and the added account/default-account coverage is useful, but this PR is not the narrowest maintainable solution while it keeps WhatsApp policy in core and leaves generated metadata plus `file://` root handling incomplete.

- **[P2] Move WhatsApp root merging out of core:** `src/infra/outbound/deliver.ts:271-287`
  - body: Core outbound delivery is supposed to stay channel-agnostic, but this patch imports `createAccountListHelpers("whatsapp")` and branches on `params.channel === "whatsapp"` to merge WhatsApp config.   - confidence: 0.88
- **[P2] Normalize file URL roots before accepting them:** `src/config/zod-schema.providers-whatsapp.ts:35-40`
  - body: The schema accepts `file://` root entries and the new test marks `file:///tmp/wa-media-2` valid, but downstream local-root enforcement does not convert root entries from file URLs before comparing paths. Either reject `file://` roots here or normalize them with the same safe file-URL helper before passing them to the media loader.
  - confidence: 0.86
