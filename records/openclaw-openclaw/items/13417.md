## Summary

Current main still does not implement outbound WhatsApp Newsletter/Channel targets for the message tool. The central WhatsApp target normalizer rejects any remaining target containing `@`, so a numeric `@newsletter` JID fails before the Baileys send layer. An open maintainer-labeled PR, #73393, already uses closing syntax for this issue and carries forward the earlier closed #13424 proposal while addressing the routing/session concern, so this issue should stay open pending review of that PR.

## Best Possible Solution

Keep this issue open as the outbound WhatsApp Newsletter/Channel support tracker until #73393 is reviewed and either merged or closed. The implementation should add an explicit newsletter JID recognizer/normalizer, represent newsletter/channel routing separately from direct and group chats where needed, route sends through the active WhatsApp listener, add normalization/resolver/session/send coverage, and update the message plus WhatsApp docs. Treat #62697 as related inbound newsletter processing work, not a duplicate.
