## Summary

Keep open. Current main still rejects WhatsApp `@newsletter` targets before the outbound send path, the public target-format docs remain E.164/group-only, and #73393 is an open paired implementation PR with closing syntax for this issue, so this should not be closed before that PR is reviewed and merged or closed.

## What This Changes

Allow the WhatsApp `message` tool to accept `@newsletter` Channel JIDs as outbound targets and send through the existing WhatsApp Web/Baileys socket path, with optional newsletter-specific actions later.

## Best Possible Solution

Newsletter targets should become an explicit WhatsApp outbound target kind across normalization, target resolution, session routing, focused tests, and public docs, while keeping inbound newsletter processing tracked separately in #62697.
