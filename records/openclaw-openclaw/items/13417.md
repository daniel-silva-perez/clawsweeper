## Summary

Keep open. Current main still rejects WhatsApp `@newsletter` targets before outbound sending, the public message docs remain E.164/group-only, and open PR #73393 uses closing syntax for this outbound request but is not merged. The related #62697 is inbound newsletter processing, not the same remaining work.

## What This Changes

The issue requests outbound `message` tool support for WhatsApp Newsletter/Channel `@newsletter` JIDs, with optional newsletter-specific actions.

## Best Possible Solution

Review and either land or close #73393 as the canonical outbound fix. A complete fix should add explicit newsletter JID normalization, avoid treating newsletter targets as ordinary direct chats in session metadata, keep delivery through the active WhatsApp listener, and update focused tests plus `https://docs.openclaw.ai/cli/message` and the WhatsApp channel docs. Keep #62697 separate for inbound newsletter processing.
