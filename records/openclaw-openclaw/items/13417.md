## Summary

Current main at ab5c8025c9d0a31873d311ae3e73f3664b0a1185 still does not implement outbound WhatsApp Newsletter/Channel targets for the message tool. The WhatsApp target normalizer rejects unknown JIDs containing `@`, the resolver still reports only `<E.164|group JID>`, message/session metadata remains direct/group-only, and docs/tests do not cover `@newsletter`. An open maintainer-labeled PR, #73393, already uses closing syntax for this issue, so this should stay open until that PR is reviewed and either merged or closed.

## Best Possible Solution

Keep this issue open as the outbound WhatsApp Newsletter/Channel support tracker until #73393 is reviewed and either merged or closed. The fix should add an explicit newsletter JID recognizer/normalizer, represent newsletter/channel routing separately from direct and group chats where needed, route sends through the active WhatsApp listener, add normalization/resolver/session/send coverage, and update the message plus WhatsApp docs. Treat #62697 as related inbound newsletter processing work, not a duplicate.
