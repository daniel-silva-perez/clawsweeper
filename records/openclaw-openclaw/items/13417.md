## Summary

Keep open. Current main still rejects WhatsApp `@newsletter` targets before outbound sending, the message-tool docs and tests remain E.164/group-only, and open PR #73393 is explicitly paired with this outbound request but is not merged. The earlier #13424 source PR was closed unmerged, while #62697 tracks the separate inbound-newsletter path.

## What This Changes

Request support for WhatsApp Channel/Newsletter JIDs such as `xxxxx@newsletter` as targets for the WhatsApp message tool, with optional newsletter-specific actions later.

## Best Possible Solution

Review and either land or close #73393 as the canonical outbound fix. A complete solution should add explicit newsletter JID normalization, keep newsletter targets out of ordinary direct-chat session metadata, route delivery through the active WhatsApp listener, update focused tests, and document the target format in `https://docs.openclaw.ai/cli/message` plus the WhatsApp channel docs while leaving inbound newsletter processing to #62697.
