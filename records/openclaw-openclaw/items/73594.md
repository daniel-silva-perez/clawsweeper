## Summary

Keep open. Current main still injects OpenRouter `cache_control` markers only for Anthropic model refs, while this PR is a focused implementation candidate for Qwen commercial refs. The PR is not implemented or superseded on main, and the remaining maintainer question is whether OpenRouter's Qwen/DashScope routing and cache-control contract is stable enough to accept without live E2E proof.

## Best Possible Solution

Keep this PR open for normal maintainer review. Verify the OpenRouter/DashScope Qwen cache-control contract with upstream docs or a live E2E request, coordinate with #42961 only for shared OpenRouter cache-retention semantics if needed, then land a narrow wrapper/test/docs update if the behavior is accepted.
