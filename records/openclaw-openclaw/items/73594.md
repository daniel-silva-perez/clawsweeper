## Summary

Keep this PR open. Current main still gates OpenRouter `cache_control` injection to `anthropic/*` model ids only, so the Qwen commercial-model path proposed by this PR is not implemented or superseded on main. The patch is focused and low supply-chain risk, but it still needs maintainer review of the upstream OpenRouter/DashScope cache-control contract and ideally the E2E check listed in the PR body.

## Best Possible Solution

Keep the PR open for normal review. The best path is to verify the OpenRouter/DashScope Qwen cache-control contract with either official upstream confirmation or a live E2E request, then land a narrow tested change in the OpenRouter cache-wrapper path or move the family predicate into OpenRouter-owned provider code if maintainers want that ownership boundary tighter.
