## Summary

Keep this PR open. Current main still does not implement the PR's intended behavior: OpenRouter system-prompt cache marker injection remains gated to Anthropic model refs only, and there is no Qwen cache-family predicate or Qwen cache-control test on main. The PR is a focused implementation candidate, but the upstream OpenRouter/DashScope cache_control contract and live/E2E proof still need maintainer review before landing or closing.

## Best Possible Solution

Keep this PR open for normal maintainer review. Verify the OpenRouter/DashScope Qwen cache-control contract with upstream docs or a live E2E request, decide whether Qwen belongs in the same system-prompt marker wrapper or in a provider-owned cache policy, and coordinate with #42961 only for shared cacheRetention semantics if needed. If accepted, land the narrow predicate/test/wrapper update with docs updated to describe the supported Qwen model family.
