## Summary

Keep this PR open. Current main still gates OpenRouter cache_control injection to Anthropic model refs only, and the docs/tests describe non-Anthropic OpenRouter models as unchanged. The PR is a focused implementation candidate for Qwen commercial OpenRouter models, but it is not implemented or superseded on main and still needs maintainer verification of the OpenRouter/DashScope cache-control contract.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to verify the OpenRouter/DashScope Qwen cache-control contract with upstream documentation or a live E2E request, then land a narrow tested change in the OpenRouter cache-wrapper path, or adjust the implementation if maintainers want the family predicate owned by provider-specific code.
