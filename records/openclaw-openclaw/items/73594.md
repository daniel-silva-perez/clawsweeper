## Summary

Keep open. Current main still only injects OpenRouter `cache_control` markers for Anthropic model refs, while this PR is a focused implementation candidate for Qwen commercial-model refs. It is not implemented on main, not superseded by the related Anthropic `cacheRetention` PR, and the remaining maintainer question is the upstream OpenRouter/DashScope cache-control contract plus the PR's unchecked E2E verification.

## Best Possible Solution

Keep this PR open for normal maintainer review. Verify the OpenRouter/DashScope Qwen cache-control contract with upstream docs or a live E2E request, then land a narrow tested change in the existing OpenRouter cache-wrapper path and update the prompt-caching docs if the behavior is accepted.
