## Summary

Keep this PR open. Current main still gates OpenRouter `cache_control` injection to Anthropic model refs only, while this PR is a focused implementation candidate for closed-source Qwen commercial refs. The PR is not implemented on main, not superseded by the related OpenRouter Anthropic `cacheRetention` PR, and the remaining maintainer question is whether OpenRouter/DashScope guarantees the Qwen cache-control contract strongly enough to inject these markers unconditionally.

## Best Possible Solution

Keep this PR open for normal maintainer review. Verify the OpenRouter/DashScope Qwen prompt-caching contract with upstream documentation or a live E2E request, then land a narrow tested change in the existing OpenRouter cache-wrapper path and update the prompt-caching/OpenRouter docs if the behavior is accepted.

## Risks / Open Questions

- The PR's central behavior depends on an upstream OpenRouter/DashScope contract: these Qwen refs must be routed deterministically to a provider that accepts Anthropic-style `cache_control`. That remains unverified by current main tests/docs and the PR body leaves E2E verification unchecked.
- If the regex is too broad, or OpenRouter routing/provider support changes, OpenClaw could inject Anthropic-style cache markers into a model route that ignores or rejects them.
- No supply-chain or infrastructure regression was visible in the provided PR diff; the risk is behavioral compatibility, not CI/dependency/secrets handling.
