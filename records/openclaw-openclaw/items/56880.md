## Summary

Keep open. Current main still serializes normal tool-capable turns per session and routes active-session messages through steering/followup/interrupt behavior rather than spawning parallel turns; the only implemented quick side path is the narrower `/btw` tool-less side question. An open PR already references this issue with closing syntax, so the issue should remain open until that PR is reviewed, merged, replaced, or closed.

## What This Changes

The issue requests async per-session message handling so quick follow-up messages can get timely responses while a long agent turn is still running, with context awareness and optional interruption.

## Best Possible Solution

Keep the issue open while maintainers review the linked fast-lane PR as the narrow candidate. The durable end state should either land a bounded, tested active-session quick-reply path or explicitly design safe per-session concurrency with session isolation, transcript consistency, tool conflict handling, and rate-limit controls.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a long normal run in one Discord/session key followed by another normal message in the same session; current main routes the second message through steering/followup/interrupt policy rather than creating an independent tool-capable turn.

## Solution Assessment

Unclear. The open fast-lane PR is a plausible narrow mitigation, but full concurrent agent turns would need a deeper safety design; reviewing that PR is better than closing this issue or queueing a duplicate repair.
