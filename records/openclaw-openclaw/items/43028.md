## Summary

Keep open. Current main has cron timeouts, recurring error backoff, failure alerts, retained/pruned isolated run sessions, JSONL run history, and cron_changed plugin hooks, but it still does not implement the requested recurring-job circuit breaker/auto-pause, remove-time session cleanup option, or automatic .learnings/changelog.md writer.

## Best Possible Solution

Keep this open. The best implementation path is to add a backward-compatible recurring cron circuit-breaker config/schema/docs/test surface and enforce it in src/cron/service/timer.ts; design an explicit cleanup option across Gateway protocol, CLI, UI, and session/transcript ownership; then decide whether cron run JSONL plus cron_changed plugin hooks is the supported audit trail or whether core should also write .learnings/changelog.md.

## Risks / Open Questions

- The request bundles three separable changes, so maintainers may want to split circuit breaker, remove cleanup, and audit/changelog behavior into narrower issues.
- Delete-time cleanup needs an explicit ownership contract to avoid deleting shared, custom, base, or still-referenced sessions/transcripts.
- The changelog request needs product direction: document cron JSONL run history plus cron_changed hooks as the supported audit path, or intentionally add a core .learnings/changelog.md writer.
