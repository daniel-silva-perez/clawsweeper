## Summary

Keep this open. Current main has improved cron reliability since the reported 2026.3.8 behavior: execution timeouts, recurring error backoff, failure alerts, retained/pruned isolated run sessions, JSONL run history, and cron_changed plugin hooks. It still does not implement the requested recurring-job circuit breaker/auto-pause, remove-time session cleanup option, or automatic .learnings/changelog.md writer. The existing discussion is consistent with that: one commenter called out broader 2026.3.8 cron breakage, and the prior ClawSweeper review correctly identified remaining product work that is still missing on current main.

## Best Possible Solution

Keep this open. The best implementation path is to add a backward-compatible recurring cron circuit-breaker config/schema/docs/test surface and enforce it in src/cron/service/timer.ts; design an explicit cleanup option across Gateway protocol, CLI, UI, and session/transcript ownership; then decide whether cron.runs JSONL plus the cron_changed plugin hook is the supported audit trail or whether core should also write .learnings/changelog.md.

## Risks / Open Questions

- The issue bundles three separable changes, so maintainers may want to split circuit breaker, remove cleanup, and audit/changelog behavior into separate implementation tasks.
- Delete-time cleanup needs an explicit ownership contract to avoid deleting shared, custom, base, or still-referenced sessions/transcripts.
- The changelog request needs product direction: either document JSONL run history plus cron_changed hooks as the supported audit path, or intentionally add a core .learnings/changelog.md writer.
