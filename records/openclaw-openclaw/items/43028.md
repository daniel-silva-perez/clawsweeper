## Summary

Current main still only partially addresses the requested cron automation improvements. Cron now has execution timeouts, recurring error backoff, failure alerts, isolated run-session retention, JSONL run history, and `cron_changed` plugin hooks, but it does not implement a recurring-job circuit breaker/auto-pause threshold, a delete-time `cron rm --cleanup` session cleanup option, or automatic `.learnings/changelog.md` writes.

## Best Possible Solution

Keep this issue open. The best path is to add a backward-compatible recurring cron circuit-breaker config/schema/docs/test surface and enforce it in `src/cron/service/timer.ts`; design an explicit cleanup option across Gateway protocol, CLI, UI, and session/transcript ownership; then decide whether cron run JSONL plus `cron_changed` plugin hooks are the supported audit trail or whether core should also write `.learnings/changelog.md`.

## Risks / Open Questions

- The issue bundles three separable product changes; maintainers may want to split circuit breaker, delete-time cleanup, and audit/changelog behavior into narrower issues before implementation.
- Delete-time cleanup needs an explicit ownership contract so `cron rm --cleanup` does not delete shared, custom, base, or still-referenced session/transcript data.
- The changelog portion needs product direction: document cron JSONL run history plus `cron_changed` hooks as the supported audit trail, or intentionally add a core `.learnings/changelog.md` writer.
