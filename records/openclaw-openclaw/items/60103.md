## Summary

Keep open. Current main still has format-specific Feishu mention helpers, but no outbound normalizer for agent-authored `<at user_id="...">...</at>` or `<at id=...></at>` tags across card/text/post paths. The related open #73469 is useful context for the streaming notification side, but it does not fully supersede this cross-format normalization request.

## Best Possible Solution

Keep this open and implement the behavior in the Feishu plugin: add a tested mention-normalization helper for outbound text, structured/markdown cards, streaming-card final/update content, and post payloads, then reconcile it with #73469 so the implementation handles both rendering syntax and notification semantics.
