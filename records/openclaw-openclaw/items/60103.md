## Summary

Keep open. Current main still has Feishu format-specific mention helpers and target-driven prepending, but no outbound normalizer for agent-authored `<at user_id="...">...</at>` or `<at id=...></at>` tags across card, streaming-card, text, and post delivery. The related open #73469 is useful context for streaming notification behavior, but it is narrower than this cross-format syntax normalization request.

## What This Changes

The issue requests automatic Feishu mention syntax normalization when outbound replies move between text, card, and post message formats.

## Best Possible Solution

Implement this in the Feishu plugin, not core: add a tested outbound mention-normalization helper used by text/post, structured-card, markdown-card, and streaming-card paths, while keeping existing explicit `mentionTargets` behavior and card sanitization intact. Treat #73469 as related notification-semantics follow-up unless the same helper can safely extract mentions early enough for streaming card creation.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Feishu reply with `renderMode: "card"` or streaming-card delivery where the agent final text contains `<at user_id="ou_xxx">Name</at>`; current main passes that text into card/post content without converting it to the destination format.

## Solution Assessment

Yes, a Feishu-owned outbound normalizer is the narrowest maintainable direction. The safer implementation is a small parser/formatter helper with focused send/reply-dispatcher/streaming tests, not a core channel abstraction or a broad card-markup passthrough.
