## Summary

Keep open. Current main still does not auto-normalize agent-authored Feishu mention tags across outbound post/text, card, and streaming-card paths. The implementation only formats known `MentionTarget` objects and preserves the reply text otherwise; related #73469 is useful context but narrower because it focuses on streaming-card notification behavior, not cross-format syntax normalization.

## Best Possible Solution

Keep this open and implement the behavior in the Feishu plugin: add a tested outbound mention-normalization helper for post/text payloads, markdown/structured cards, and streaming-card update/close content, then reconcile it with #73469 so the implementation handles both rendering syntax and notification semantics.
