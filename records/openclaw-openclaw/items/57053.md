## Summary

Keep this issue open. Current main still exposes `session_status` as a formatted status card plus minimal `details` metadata, not as a structured remaining time/token budget contract for skills. The related #57705 is adjacent capacity/concurrency visibility work, but it does not supersede this request for remaining session time/token budget data.

## What This Changes

The issue requests structured remaining session time/token budget fields in `session_status` or a new lightweight budget tool.

## Best Possible Solution

Define a stable structured budget contract for skill callers, reuse the existing token/context accounting where possible, define time-budget semantics for the active run explicitly, and document/tests the chosen surface for multi-agent orchestration.

## Reproduction Assessment

Not applicable as a feature request, but source inspection gives a high-confidence current-behavior check: `session_status` returns only formatted text plus minimal metadata, so a skill cannot read the requested budget fields without parsing text.

## Solution Assessment

Unclear. Structured budget data is the right direction, but maintainers should first choose the public surface and define `remaining_seconds` semantics before implementation.
