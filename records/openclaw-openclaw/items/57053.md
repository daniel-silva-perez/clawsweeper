## Summary

Keep this issue open. Current main still exposes session usage and context information through a formatted `/status`-style text card, while `session_status` structured `details` only carries `ok`, `sessionKey`, `changedModel`, and `statusText`; there is no stable structured time/token budget field for skills to branch on. The related #57705 is adjacent concurrency/capacity work, but it does not supersede this request for remaining time/token budget data.

## What This Changes

Request to expose remaining session time/token budget as structured data via `session_status` or a new lightweight `session_budget` tool for budget-aware multi-agent orchestration.

## Best Possible Solution

Define a stable structured budget contract, either as `session_status.details.budget` or a separate `session_budget` tool, that reuses the existing status/context accounting for token budget and a documented current-run timeout source for time budget, then add focused tests and docs for skill orchestration callers.

## Reproduction Assessment

Not applicable as a feature request, but source inspection gives a high-confidence current-behavior check: `session_status` currently returns only formatted status text plus minimal metadata, so a skill cannot read the requested budget fields without parsing text.

## Solution Assessment

Unclear. Structured budget data is a maintainable direction, but the best surface and exact `remaining_seconds` semantics need maintainer/API judgment before a repair worker should implement it.
