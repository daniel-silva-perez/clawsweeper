## What This Changes

The branch changes config validation formatting and plugin/channel validation paths, and also localizes several Feishu plugin error messages and tests.

## Best Possible Solution

Land a narrow config formatter change that names offending keys for both Zod `unrecognized_keys` and AJV `additionalProperties`, with focused config tests. Keep Feishu localization and plugin allowlist policy changes in separate PRs after maintainer approval.

## Reproduction Assessment

Yes. A Feishu or plugin config with an extra property should hit the AJV `additionalProperties` path; current source shows that path still emits AJV's generic message without `params.additionalProperty`.

## Solution Assessment

No. The safest fix is a targeted formatter/test change in config validation or the JSON schema validator, while this PR bundles unrelated Feishu prompt/localization and plugin policy changes.
