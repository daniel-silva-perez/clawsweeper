## Summary

Keep this PR open. Current main still lacks the proposed `SystemEvent.audience` API, the drain path still renders all queued system events as visible `System:` lines, and the isolated cron awareness producer still enqueues a visible untrusted event. The PR is tied to the open #69492/#69208 event-leakage design discussion, so it needs maintainer review rather than conservative cleanup closure.

## What This Changes

The PR adds an optional system-event audience field, wraps internal-audience events as hidden runtime context during system-event drain, marks isolated cron awareness events internal, and adds focused regression coverage.

## Best Possible Solution

Land a maintainer-approved hidden runtime-context primitive for system events with aligned SDK/docs/tests and changed-gate proof, or redirect the branch to whatever narrower primitive maintainers choose for #69492.
