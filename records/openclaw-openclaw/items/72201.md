## Summary

Keep this PR open. Current main does not implement the proposed `SystemEvent` audience lane, and the PR is a live implementation candidate for part of open #69492/#69208 rather than obsolete work. Because it changes shared prompt/runtime-context semantics and the plugin-facing system-event option shape, it needs maintainer review before merge.

## What This Changes

This PR adds an optional `SystemEvent` audience field, wraps internal-audience system events as hidden runtime context during drain, marks isolated cron awareness events internal, and adds focused regression tests.

## Best Possible Solution

Keep this PR open for maintainer review against #69492/#69208. The best path is to decide whether the narrow hidden runtime-context lane is the accepted primitive, resolve or explicitly accept the header-sharing note, update public/plugin docs and generated API artifacts if `audience` is accepted API, then run targeted tests and the changed gate before merge.
