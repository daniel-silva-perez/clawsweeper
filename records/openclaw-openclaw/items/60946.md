## Summary

Keep open. Current main still only partially addresses the request: edit mismatch errors include bounded current file contents, but the richer model-facing diagnostics requested here are still missing for similar matches, duplicate match locations, apply_patch mismatch failures, and exec command-not-found recovery hints. The plugin path is not sufficient today because tool-result middleware is limited to bundled plugins.

## What This Changes

The issue requests richer tool error responses with contextual troubleshooting hints, similar text or duplicate-match locations, and actionable fix suggestions for edit, apply_patch, and exec failures.

## Best Possible Solution

Add a shared, bounded diagnostic helper in the core agent tool runtime and wire it into edit mismatch, apply_patch mismatch, and exec exit-127 formatting so failures include concise line-numbered candidates and retry guidance without changing tool semantics or broadening plugin APIs.

## Repair Work Prompt

Fix rich tool error messages with bounded, model-facing diagnostics for edit, apply_patch, and exec failures. Preserve existing tool semantics and security boundaries. Add or reuse a shared helper under the agent tool runtime for safe line-numbered snippets/candidate matches. For edit exact-text mismatches, keep the current bounded current-file hint but add concise common causes, nearby/similar candidate lines when safe, duplicate-match locations when the old text appears multiple times, and retry guidance. For apply_patch update failures, enrich missing context / missing expected lines with bounded nearby candidates and retry guidance. For exec exit 127, improve `Command not found` only with conservative local evidence; do not run package managers, do not download anything, and do not emit speculative install commands. Keep snippets bounded and avoid leaking excessive file content. Update focused tests for the new diagnostics. Add a user-facing changelog entry only if maintainers treat this tool UX improvement as changelog-worthy; if added, credit `Thanks @ohmanthatsgreat` and follow the active-version single-line changelog convention. Do not change plugin API scope or make external plugins able to rewrite high-trust tool output as part of this fix.
