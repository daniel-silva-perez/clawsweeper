## Summary

Keep open. Current main still only partially addresses the request: edit mismatch errors include a bounded current-file dump, but similar-match diagnostics, duplicate-location guidance, apply_patch mismatch help, and exec command-not-found recovery hints are still missing. This is also not a ClawHub-only item because external plugins cannot currently rewrite high-trust tool results before the model sees them.

## What This Changes

Request richer model-facing tool error diagnostics with troubleshooting hints, similar-match or duplicate-location context, and actionable retry suggestions.

## Best Possible Solution

Add a shared bounded diagnostic helper in the core agent tool runtime and wire it into edit mismatch, apply_patch mismatch, and exec exit-127 formatting so failures include concise candidates and retry guidance without changing tool semantics or plugin trust boundaries.

## Reproduction Assessment

Yes. Current-main source and tests give a high-confidence reproduction path: edit mismatch only appends current contents, apply_patch throws terse mismatch errors, and the exec formatter/test still expect `Command not found`. Tests were not run because this review is read-only.

## Solution Assessment

Yes. A bounded core runtime diagnostic helper is the narrow maintainable path; ClawHub or external plugins are not sufficient today because external tool-result middleware is intentionally rejected before pre-model output rewriting.
