## Summary

Current main has partial tool-error guidance, but it does not implement the broader requested rich troubleshooting behavior. Edit mismatches can append current file contents, required-parameter errors include a retry hint, and exec timeouts include actionable timeout/background guidance. However, `apply_patch` still reports missing context/expected lines without similar-match locations or fix options, `exec` command-not-found is still only `Command not found`, and the bundled tokenjuice path is opt-in output compaction rather than general error suggestions. This should stay open as a real product/core tool-runtime feature request.

## Best Possible Solution

Keep this issue open and implement the feature in the core-owned tool runtime paths, starting with `edit`, `apply_patch`, and `exec`: add structured diagnostic helpers for missing text/context, multiple matches, similar snippets with line numbers, common causes, and concrete retry guidance, then cover those cases with focused tests. If maintainers want third-party experimentation first, the plugin API would need an explicitly supported third-party tool-result middleware seam; the current middleware is bundled-only.
