## Summary

Keep open. Current main still only partially addresses the request: edit mismatch failures append bounded current file contents, but the richer model-facing diagnostics requested here are still missing for similar matches, duplicate match locations, apply_patch mismatch failures, and exec command-not-found recovery hints. This also is not a ClawHub-only item because the public plugin path cannot rewrite high-trust tool results before the model sees them.

## What This Changes

Request richer tool error responses with contextual troubleshooting hints, similar or duplicate match details, and actionable fix suggestions for edit, apply_patch, and exec failures.

## Best Possible Solution

Add a shared bounded diagnostic helper in the core agent tool runtime, then wire it into edit mismatch, apply_patch mismatch, and exec exit-127 formatting so failures include concise line-numbered candidates and retry guidance without changing tool semantics or broadening plugin trust boundaries.
