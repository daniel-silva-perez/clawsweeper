## Summary

Current main has partial tool-error guidance, but the broader requested rich troubleshooting behavior is still missing for core edit, apply_patch, and exec failure paths. The prior ClawSweeper keep-open comment remains accurate against current main.

## Best Possible Solution

Keep this issue open and implement the feature in the core tool-runtime paths. Start with edit, apply_patch, and exec: add shared diagnostic helpers for missing text/context, multiple matches, similar snippets with line numbers, common causes, and concrete retry guidance, with focused tests for each failure class. If maintainers want this to be plugin-extensible first, expose a documented third-party tool-result middleware seam before moving the behavior out of core.
