## Summary

Current main still does not implement the requested rich tool-error guidance. There is partial help for edit mismatches and exec timeouts, but apply_patch mismatch failures and exec command-not-found remain minimal, and the existing tokenjuice/tool-result middleware path is opt-in and bundled-only rather than a general solution for core tool errors.

## Best Possible Solution

Keep this issue open and implement a focused core tool-runtime improvement: add shared bounded diagnostics for edit/apply_patch missing text or ambiguous matches, line-numbered nearby snippets where safe, concrete retry guidance, and safer exec command-not-found hints. Cover edit, apply_patch, and exec with targeted regression tests; use plugin API discussion only if maintainers want third-party tool-result rewriting beyond bundled plugins.
