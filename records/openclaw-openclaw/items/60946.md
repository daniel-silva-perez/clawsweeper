## Summary

Keep open. Current main only partially helps edit mismatches and still lacks the richer model-facing diagnostics requested for similar or duplicate edit matches, apply_patch mismatch failures, and exec command-not-found recovery hints. This also should stay in core for now because external plugins cannot register the pre-model tool-result middleware needed to solve it outside the trusted runtime path.

## What This Changes

Request to enrich failed tool results with contextual troubleshooting hints, similar-match snippets, duplicate-match locations, and actionable retry guidance.

## Best Possible Solution

Add a shared bounded diagnostic helper in the core agent tool runtime, then wire it into edit mismatch, apply_patch mismatch, and exec exit-127 formatting so failures include concise line-numbered candidates and retry guidance without changing tool semantics or broadening plugin trust boundaries.
