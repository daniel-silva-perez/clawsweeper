## Summary

Current main still only partially implements the requested rich tool-error guidance. Edit mismatch failures append bounded current file contents, but there is still no line-numbered similar-match or multiple-match guidance, apply_patch context failures remain terse, and exec command-not-found errors still return only a minimal fixed string.

## What This Changes

The issue requests richer contextual tool error messages with troubleshooting hints, similar text suggestions, and actionable retry steps for edit, apply_patch, and exec failures.

## Best Possible Solution

Keep this open and implement a focused core tool-runtime improvement: shared bounded diagnostics for edit/apply_patch missing text and ambiguous matches, safe line-numbered nearby snippets plus concrete retry guidance, and conservative exec command-not-found hints. Keep #46548 as the separate user-visible chat warning/retry-confirmation policy thread unless maintainers intentionally combine the work.
