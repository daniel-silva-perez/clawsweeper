## Summary

Keep open. Current main still only has partial edit mismatch help and does not implement the richer model-facing diagnostics requested for similar edit matches, duplicate match locations, apply_patch mismatch failures, or exec command-not-found recovery hints. This is not a ClawHub-only cleanup because external plugins still cannot register tool-result middleware that rewrites high-trust tool output before the model sees it.

## What This Changes

The issue requests richer tool error messages with contextual troubleshooting hints, similar-text suggestions, and actionable fix steps for edit, apply_patch, and exec failures.

## Best Possible Solution

Add a shared bounded diagnostic helper in the core agent tool runtime and wire it into edit mismatch, apply_patch mismatch, and exec exit-127 formatting so failures include concise line-numbered candidates and retry guidance without changing tool semantics or broadening plugin trust boundaries.

Implement rich, bounded tool error diagnostics for this issue. Scope the fix to core agent tool-runtime behavior: edit exact-match mismatch failures, apply_patch update mismatch failures, and exec exit-127 command-not-found failures. Add or reuse a shared helper only if it keeps the implementation smaller and consistent. Do not change tool success semantics, sandbox/write permission behavior, plugin trust boundaries, or allow external plugins to register agent tool-result middleware as part of this fix. For edit/apply_patch mismatches, include concise line-numbered candidate/context snippets and actionable retry guidance; keep snippets bounded.
