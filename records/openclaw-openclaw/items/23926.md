## Summary

Keep open. Current main still only partially covers the request: gateway-managed skill dependency installers run the generic blocking `before_install` hook, but native ClawHub skill install/update and the Skills UI ClawHub path still copy downloaded skill folders without a pre-copy policy hook, and there is no post-install observational hook carrying audit/outcome metadata.

## Best Possible Solution

Keep this issue open as the concrete tracker for skill install lifecycle policy. The best path is to extend the existing `before_install` contract or add documented skill lifecycle hooks so native ClawHub install/update, gateway `skills.install`, and Skills UI ClawHub installs run a blocking policy point before copying files, then add a post-install observational hook with `installedAt`, `targetDir`, source, outcome, and audit metadata. Clarify separately whether the standalone `clawhub` CLI needs ClawHub-owned support outside this repo.
