## Summary

Keep open. Current main still only partially covers the requested safety point: gateway-managed skill dependency installers use the existing `before_install` plugin hook, but native ClawHub skill install/update and the Skills UI ClawHub path still copy downloaded skill folders without a blocking pre-copy policy hook. There is also still no post-install observational lifecycle hook with installedAt, targetDir, success/failure, or audit semantics.

## Best Possible Solution

Keep this issue open as the concrete tracker for skill install lifecycle hook/API work. Extend the existing `before_install` contract or add an equivalent skill lifecycle hook so native ClawHub install/update, gateway, and Skills UI paths run a blocking policy point before copying files. Add a documented post-install observational hook with installedAt, targetDir, source, outcome, and audit metadata, and clarify whether the standalone `clawhub` CLI needs separate ClawHub-owned support.

## Risks / Open Questions

- The existing `before_install` hook is partial coverage, so any fix needs to avoid creating a second, confusing pre-install contract for the same policy decision.
- Native ClawHub skill install/update currently writes files before any plugin policy point; that is a real security-sensitive gap for untrusted skill bundles.
- A post-install observational hook is a public Plugin SDK surface and needs stable payload semantics for success, failure, targetDir, source, installedAt, findings, and audit metadata.
- The standalone `clawhub` CLI install/sync path may require ClawHub-side support outside this repository, even if OpenClaw native CLI/gateway/UI coverage is fixed here.
