## Summary

Current main still does not implement `agents.defaults.responseUsage`. The usage footer remains session-scoped: config types/schema omit the key, strict `agents.defaults` validation would not accept it, Gateway defaults do not resolve it, and the reply runner reads only `SessionEntry.responseUsage` from the active session/store.

## Best Possible Solution

Keep this issue open for implementation. A conservative fix should add `agents.defaults.responseUsage` to the config type, strict schema, generated config metadata/help/docs, and new-session/default resolution; preserve explicit per-session `/usage` overrides; and add tests for `off`, `tokens`, and `full` across config validation plus reply/session-default behavior.
