## Summary

Current main still does not implement `agents.defaults.responseUsage`. The config type, strict schema, generated metadata, docs, session defaults, reply runner, and `/usage` command all show the response usage footer remains per-session only. The prior ClawSweeper comment remains accurate; the only relevant-file diff since that review is unrelated slash-command documentation.

## Best Possible Solution

Keep this issue open and implement the feature in the core config/session path. A conservative fix should add `agents.defaults.responseUsage` to the config type, strict Zod schema, generated config metadata/help/docs, new-session/default resolution, and reply-runner fallback; explicit per-session `/usage` overrides should continue to win. Add focused tests for config validation plus `off`, `tokens`, and `full` behavior across new sessions and persisted session overrides.
