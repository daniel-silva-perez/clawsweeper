## Summary

Close as implemented: current main and the shipped v2026.4.27 release no longer create an isolated bundled-runtime-deps install package without dependency entries, which addresses the npm v11/Node v25 failure described for 2026.4.24.

## What This Changes

The issue asks OpenClaw to populate the isolated bundled plugin runtime-dependency package.json with explicit dependencies so npm v11 does not crash while staging plugin dependencies.

## Best Possible Solution

Keep the manifest-backed runtime-deps install path and direct affected users on 2026.4.24 through 2026.4.26 to update to v2026.4.27 or newer.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: v2026.4.24 writes the empty generated package.json described in the report, while current main and v2026.4.27 write explicit dependencies; I did not run a live npm install because this was a read-only review.

## Solution Assessment

Yes. Populating the generated package.json from normalized install specs is the narrow maintainable fix, and current main also keeps install-spec parsing centralized instead of using ad hoc package-name splitting.
