## Summary

Keep this PR open. Current main still has the safeguard compaction bug the PR targets: the safeguard hook prefers the active session model, and embedded runtime registration stores the session model instead of resolving `agents.defaults.compaction.model`. The PR remains the active implementation candidate for the linked open bug, with no supply-chain-sensitive files in the provided diff.

## Best Possible Solution

Keep this PR open and review or land it, or land an equivalent focused patch. The fix should resolve the effective compaction provider/model before safeguard runtime registration, prefer the configured runtime compaction model when present, preserve no-override fallback to the session model, preserve provider-switch credential behavior, surface explicit override registry misses clearly, and keep focused regression coverage for runtime registration plus hook precedence.
