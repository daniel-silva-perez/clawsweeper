## Summary

Keep open. Current main still has the remaining safeguard auto-compaction bug this PR targets: the safeguard hook prefers `ctx.model` over `runtime.model`, and the embedded safeguard runtime is still seeded from the active session model rather than the resolved `agents.defaults.compaction.model`. The linked issue is open and paired with this contributor PR, so the right cleanup path is review/land this PR or an equivalent fix, not close it as obsolete.

## Best Possible Solution

Review and land this PR or an equivalent focused patch that resolves the effective compaction provider/model before registering the safeguard runtime, makes the safeguard hook prefer the configured runtime compaction model when present, preserves no-override fallback to the session model, and keeps regression coverage for both the embedded runtime registration and safeguard hook precedence. Let the linked issue close through the merge.
