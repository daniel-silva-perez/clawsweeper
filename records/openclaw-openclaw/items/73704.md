## Summary

Keep this PR open. Current main still has the safeguard auto-compaction bug this PR targets: the safeguard hook prefers the active session model over the runtime model, and the embedded safeguard runtime is still registered with the active session model rather than the resolved `agents.defaults.compaction.model`. The linked issue is open and this PR is the active implementation candidate, so it should be reviewed or amended rather than closed as obsolete.

## Best Possible Solution

Review and land this PR or an equivalent focused patch that resolves the effective compaction provider/model before registering the safeguard runtime, makes the safeguard hook prefer the configured runtime compaction model when present, preserves no-override fallback to the session model, preserves provider-switch auth-profile dropping, and keeps regression coverage for both runtime registration and hook precedence. Let the linked issue close through the merge.
