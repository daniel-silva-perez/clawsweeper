## Summary

Keep this PR open. Current main still has the safeguard compaction behavior the PR targets, and the linked bug remains open; the PR is the active implementation candidate, though it now needs rebase/CI cleanup before it can land.

## What This Changes

The PR changes safeguard auto-compaction to resolve `agents.defaults.compaction.model` before embedded runtime registration, prefer the resolved runtime model in the safeguard hook, add regression coverage, and add a changelog entry.

## Best Possible Solution

Keep this PR or an equivalent focused patch as the implementation path for #57901: resolve the effective compaction provider/model before safeguard runtime registration, prefer that runtime model in the hook, preserve no-override fallback and provider-switch auth behavior, and land after rebasing and clearing checks.

## Reproduction Assessment

Yes. On current main, configure safeguard compaction with `agents.defaults.compaction.model` different from the active session model; source inspection shows runtime registration stores the session model and the hook prefers `ctx.model`, so the configured model cannot win in that path.

## Solution Assessment

Yes, the proposed direction is the narrow maintainable fix: reuse the existing compaction target resolver, pass `modelRegistry` into runtime registration, and flip hook precedence. The branch still needs rebase and CI cleanup, but I did not find a discrete code-level bug in the final diff.
