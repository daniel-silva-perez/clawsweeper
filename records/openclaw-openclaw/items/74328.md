## Summary

Keep this issue open. The reporter’s follow-up narrows the original fs.stat-storm claim but still leaves a current-main regression signal: clean-start event-loop spikes, slow probe tail, and slow CLI parse, with accumulated runtime state able to compound into a severe gateway wedge. Current main contains mitigations for client/probe readiness and packaged runtime mirror reuse, but those do not prove the underlying gateway startup/task-maintenance/plugin activation problem is fixed, and #74209 remains an open related sibling rather than a clear duplicate.

## What This Changes

The issue requests triage and a fix for a Gateway upgrade regression where current main can wedge or show large event-loop stalls, originally observed as an fs.stat microtask storm and later narrowed to stale runtime-state amplification plus clean-start startup/CLI latency spikes.

## Best Possible Solution

Keep this open and triage it with #74209 as a related cluster: reproduce on the affected macOS setup with startup tracing and sample(1), identify which startup/plugin/task-maintenance path owns the remaining event-loop stall, then bound, defer, or yield that work without disabling legitimately configured plugins or hiding stale runtime-state repair failures.
