## Summary

Keep this issue open. The report is recent, detailed, and says the regression still reproduced on current main after the packaged runtime mirror memoization work; current main still has default-enabled startup sidecars, Bonjour startup advertisement/watchdog behavior, and bundled runtime-dependency/mirror paths that perform repeated filesystem stat/mirror checks during plugin startup. #74209 is related but not a clear duplicate because this report adds a distinct fs.stat microtask-storm sample and macOS v2026.4.22-to-v2026.4.26 comparison.

## What This Changes

The issue reports a gateway CPU regression from v2026.4.22 to v2026.4.26/current main, with a sustained fs.stat promise-resolution storm that blocks the gateway main thread.

## Best Possible Solution

Keep the issue open and triage it with #74209 as a related cluster: reproduce on macOS with OPENCLAW_GATEWAY_STARTUP_TRACE=1 plus CPU sampling across v2026.4.22, v2026.4.26, and current main; then fix the narrow root cause in plugin startup/runtime-dependency mirroring or Bonjour/default startup activation without broadly disabling legitimate configured plugins.
