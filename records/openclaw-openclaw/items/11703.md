## Summary

Keep open. Current main still stores WhatsApp `messageTimestamp` on inbound messages, but batching is keyed only by account, conversation, and sender and is flushed by the shared receive-time debounce timer. The requested timestamp-based coalescing remains a real WhatsApp-owned gap, and the existing maintainer/bot comments are still accurate against the current checkout.

## What This Changes

Request WhatsApp inbound batching to group messages by Baileys/WhatsApp server `messageTimestamp`, rather than only by receive-time debounce, so forward-plus-comment bursts arrive as one agent turn.

## Best Possible Solution

Add a WhatsApp-owned, backwards-compatible timestamp coalescing path for text-only inbound messages from the same account, conversation, and sender when their WhatsApp server timestamps are identical or within a very small documented tolerance. Keep the shared debouncer generic unless a small reusable seam is needed, and update the WhatsApp config schema, docs, and fake-timer coverage for the new behavior.
