## Summary

Keep open. Current main still preserves WhatsApp `messageTimestamp` as inbound metadata, but batching is still the shared receive-time debounce keyed by account, conversation, and sender; there is no timestamp coalescing key, timer, config surface, doc, or regression test for late Baileys delivery with identical server timestamps. The existing maintainer comment remains accurate, and this is narrow enough for a focused WhatsApp repair PR.

## What This Changes

The issue requests WhatsApp inbound batching that can group same-chat, same-sender messages by WhatsApp server `messageTimestamp` instead of only by receive-time debounce.

## Best Possible Solution

Add a WhatsApp-owned, backwards-compatible timestamp coalescing path for text-only inbound messages from the same account, conversation, and sender when WhatsApp server timestamps are identical or within a small documented tolerance. Keep the shared core debouncer generic unless a small provider-agnostic hook is needed, and document any new WhatsApp config surface.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is two text-only WhatsApp `notify` messages from the same account/chat/sender with the same `messageTimestamp`, where the second arrives after `debounceMs`; current main flushes the first via receive-time `setTimeout` before the second can join.

## Solution Assessment

Yes, if implemented as WhatsApp-owned, opt-in or otherwise backward-compatible timestamp coalescing rather than a global debounce semantic change. The current request targets a Baileys/WhatsApp delivery artifact, so the narrowest maintainable fix belongs in the WhatsApp plugin path with focused fake-timer coverage.

## Repair Work Prompt

Baileys can deliver text messages with identical WhatsApp server `messageTimestamp` several seconds apart, after the normal receive-time `debounceMs` has already flushed. Add a WhatsApp-owned, backward-compatible coalescing path for text-only inbound messages from the same account, conversation, and sender when their WhatsApp server timestamps are identical or within a small documented tolerance.
