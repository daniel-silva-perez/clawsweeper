## What This Changes

The PR adds an optional `async` boolean to `sessions_send`, maps `async: true` to fire-and-forget behavior, rejects conflicting non-zero timeouts, and updates the tool description.

## Best Possible Solution

If maintainers want this alias, keep the implementation narrow: add `async` to the `sessions_send` schema, normalize it beside `timeoutSeconds`, preserve the explicit conflict validation, move the new guidance into `describeSessionsSendTool()`, and add targeted tests for schema exposure, accepted fire-and-forget behavior, and conflicting timeout validation.

## Solution Assessment
