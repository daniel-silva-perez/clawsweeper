## Summary

Keep open. Current main still routes ordinary native `sessions_send` replies through the A2A ping-pong and announce path, while the only skip path is limited to parent-owned ACP child sessions, so the persistent native sub-agent duplicate-delivery report is still actionable.

## What This Changes

The issue requests that persistent sub-agent replies sent via `sessions_send` appear once in the parent conversation instead of being duplicated through overlapping reply, announce, and delivery-context paths.

## Best Possible Solution

Keep this issue open until the parent-owned native sub-agent `sessions_send` path has one intentional parent-visible delivery mechanism, suppresses duplicate announce/control-token traffic, and has regression coverage for persistent `runtime="subagent"`, `mode="session"` sends without weakening general A2A messaging.
