## Summary

Keep this issue open. Current main still only supports cron payloads `systemEvent` and `agentTurn`; the public protocol, CLI, macOS UI/model, runtime validation, and execution path do not accept `exec` or `lobster`, and there is no `condition`/`onCondition` contract for silent deterministic runs that only trigger an agent on match. Related #18160 is an open direct-exec subset and does not supersede the Lobster pipeline plus conditional handoff scope requested here.

## Best Possible Solution

Keep this open for maintainer design. The best path is either a broader cron executor design that covers deterministic `exec` and Lobster payloads, condition evaluation, silent no-op runs, bounded output capture, failure/delivery visibility, CLI/Control UI/docs, and protocol/runtime tests, or an explicit plugin-facing executor seam if maintainers want this to live outside core. Coordinate with #18160 rather than treating it as a full duplicate.
