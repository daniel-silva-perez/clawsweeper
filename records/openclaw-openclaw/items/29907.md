## Summary

Keep this issue open. Current main still only supports cron payloads for `systemEvent` and `agentTurn`; there is no first-class `exec` or `lobster` cron payload, no `condition`/`onCondition` contract, and no CLI/UI/docs surface for deterministic cron runs that avoid the LLM until a condition matches. Existing plugin hooks make a ClawHub implementation plausible, but scheduled shell/Lobster execution plus conditional agent handoff needs maintainer product and security design before conservative cleanup should close it.

## Best Possible Solution

Keep this open for maintainer design. The best path is either an additive native cron executor contract covering `exec` and Lobster payloads, condition evaluation, silent false-condition runs, bounded output capture, failure visibility, CLI/Control UI/docs, and protocol/runtime tests, or an explicit ClawHub/plugin plan that uses existing cron lifecycle hooks, reply short-circuiting, prompt context injection, Lobster/process-runtime helpers, and plugin-owned configuration. Coordinate with #18160 as the narrower direct-exec subset rather than treating it as a full duplicate.
