## Summary

Keep this issue open. Current main still has no first-class `exec` or `lobster` cron payload, no `condition`/`onCondition` contract, and no CLI/UI/protocol/docs path for deterministic cron runs that avoid an LLM unless a condition matches. A ClawHub/plugin implementation looks plausible using the existing Lobster plugin, cron hooks, prompt hooks, and process-runtime helpers, but choosing plugin scope versus native cron protocol/runtime support is a maintainer product and security decision. Related #18160 covers only the direct-exec subset and does not supersede the Lobster pipeline plus conditional handoff scope here.

## Best Possible Solution

Keep this open for maintainer design. The best path is either an additive native cron executor design covering `exec` and Lobster payloads, condition evaluation, silent false-condition runs, bounded output capture, failure visibility, CLI/Control UI/docs, and protocol/runtime tests, or an explicit ClawHub plugin plan that uses existing cron ticks, `before_agent_reply` short-circuiting, `before_prompt_build` context injection, the Lobster plugin, and process-runtime helpers. Coordinate with #18160 as the narrower direct-exec subset rather than treating it as a full duplicate.
