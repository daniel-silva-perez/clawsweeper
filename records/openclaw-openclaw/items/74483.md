## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Gate tag unscheduling on live plugin state:** `src/plugins/registry.ts:2506-2513`
  - body: `scheduleSessionTurn`, `sendSessionAttachment`, and `emitAgentEvent` all check `shouldCommitWorkflowSideEffect()` so stale plugin API objects cannot keep mutating host state after a registry is retired. `unscheduleSessionTurnsByTag` skips that guard and still calls into `cron.remove`, so plugin code holding an old API can remove current-generation tagged jobs for the same plugin/session after reload. Add the same live-plugin guard or pass it into the workflow helper before removing jobs.
