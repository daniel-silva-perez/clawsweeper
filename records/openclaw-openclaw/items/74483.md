Overall correctness: patch is incorrect

- **[P2] Gate tag unscheduling on live plugin state:** `src/plugins/registry.ts:2506-2513`
  - body: `scheduleSessionTurn`, `sendSessionAttachment`, and `emitAgentEvent` all check the live-plugin side-effect guard so stale API objects cannot mutate host state after registry replacement. `unscheduleSessionTurnsByTag` skips that guard and still calls into the cron-backed unscheduler, so an old plugin API can remove current-generation tagged jobs for the same plugin/session after reload.
