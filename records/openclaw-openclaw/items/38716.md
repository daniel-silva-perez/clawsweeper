## Summary

Keep open. Current main still does not implement Slack RTS/action-token search: the Slack plugin has no `assistant.search.context` or `action_token` code path, does not advertise or dispatch a Slack `search` message action, and its existing read path only uses Slack conversation history/replies APIs. The open `search.messages` PR is useful adjacent work, but it does not satisfy the requested assistant action-token RTS behavior.

## What This Changes

Add Slack Real-Time Search support by capturing Slack assistant action tokens from inbound Slack assistant/message events and exposing `assistant.search.context` through the Slack message/runtime action surface.

## Best Possible Solution

Implement the remaining work in the bundled Slack plugin with an explicit RTS contract: capture the ephemeral Slack action token from the inbound user event that authorized the query, pass it only through the relevant assistant/message context, expose a typed read-only `assistant.search.context` action, document required AI/search scopes and privacy limits, and keep it distinct from user-token `search.messages` unless maintainers deliberately consolidate them.
