## Summary

Keep open. Current main still lacks the requested Slack RTS/action-token path: there is no `assistant.search.context` call, no `action_token` propagation from Slack inbound events, and Slack does not advertise or dispatch a `search` message action. The related user-token `search.messages` work remains adjacent rather than a substitute for bot-token RTS.

## What This Changes

Request support in the Slack plugin for capturing Slack assistant action tokens from inbound events and exposing `assistant.search.context` Real-Time Search through a message action or dedicated tool.

## Best Possible Solution

Implement the remaining work in the bundled Slack plugin with an explicit RTS contract: capture the ephemeral Slack action token from the authorized inbound event, scope it to the originating team/account/user/channel/thread/request, expose a typed read-only `assistant.search.context` action, document required AI/search scopes and privacy limits, and keep it distinct from user-token `search.messages` unless maintainers deliberately consolidate the two flows.

## Reproduction Assessment

Not applicable as a feature request, but the absence is reproducible with current-main inspection: symbol search, action discovery, dispatch, runtime, and read-path checks all show no RTS/action-token search implementation.

## Solution Assessment

Unclear until maintainers settle the product and security contract. The right implementation area is the Slack plugin, but the safest path is a design-reviewed RTS action rather than folding it blindly into the open user-token `search.messages` work.

## Security Review

Status: needs_attention

Summary: No patch was reviewed, but the requested Slack RTS feature is security-sensitive because it would handle ephemeral action tokens and broad workspace search results.

Concerns:

- **[medium] Scope action tokens to the originating event:**
  - body: An implementation must avoid persisting or reusing Slack `action_token` values outside the event, team/account, user, channel/thread, and request context that authorized the RTS search.
  - confidence: 0.91
- **[medium] Define workspace search privacy boundaries:**
  - body: RTS can expose workspace-wide messages, files, channels, and users according to Slack scopes and access. The plugin needs explicit limits for result handling, storage, display, and cross-user visibility before this is safe to ship.
