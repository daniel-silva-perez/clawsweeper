## Summary

Keep this issue open. Current main still lacks the requested Slack RTS path: no assistant thread event subscriptions, no `action_token` capture/storage, no `assistant.search.context` helper, and no Slack search action exposed through the message tool. The related open PR #65691 is adjacent user-token `search.messages` work, not the assistant action-token RTS flow.

## Best Possible Solution

Keep the issue open. The best implementation path is a focused Slack plugin feature: subscribe to `assistant_thread_started` and `assistant_thread_context_changed`, capture the relevant `action_token` only for the active assistant/user/thread context, add a typed `assistant.search.context` helper with Slack RTS limits/errors handled, expose it through the chosen Slack message-tool action or a dedicated tool, and update Slack setup/docs/tests for Agents & AI Apps and required search scopes. Coordinate with #65691/#54224 so user-token `search.messages` and assistant-token RTS do not get conflated.
