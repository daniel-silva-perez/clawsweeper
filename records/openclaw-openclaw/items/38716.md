## Summary

Keep open. Current main still lacks the requested Slack RTS path: no assistant action-token extraction, no assistant.search.context call, and no Slack search message action. The related open PR #65691 is useful but covers legacy search.messages with a user token, not assistant RTS action tokens.

## What This Changes

Request Slack support for extracting assistant action tokens from assistant thread events and exposing Slack assistant.search.context Real-Time Search through a message or dedicated tool action.

## Best Possible Solution

Keep this issue open for maintainer scoping. The likely implementation belongs in the bundled Slack plugin: capture assistant action tokens from supported Slack assistant events, preserve them only as needed for the initiating thread/action, add a typed assistant.search.context runtime/message action with clear bot-token and user-token rules, document required Slack AI/search scopes, and coordinate with #65691/#54224 so legacy search.messages and RTS are not conflated.
