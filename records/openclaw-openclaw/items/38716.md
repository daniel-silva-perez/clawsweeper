## Summary

Keep open. Current main still does not implement Slack RTS: the Slack plugin has no assistant action-token extraction, no assistant.search.context runtime call, and no Slack message search action. The related search.messages work is adjacent but does not satisfy the assistant action-token RTS path.

## What This Changes

The issue requests Slack Real-Time Search support by capturing assistant action tokens from Slack assistant thread events and exposing an assistant.search.context-backed message action or dedicated tool.

## Best Possible Solution

Add the feature in the bundled Slack plugin: subscribe to the relevant Slack assistant thread events, capture action tokens only for the initiating assistant context, call assistant.search.context through a typed Slack runtime/message action, document the required AI/search scopes, and keep legacy search.messages work separate from RTS.
