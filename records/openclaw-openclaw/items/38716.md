## Summary

Keep open. Current main still lacks the Slack RTS/action-token path: there are no Slack `action_token` or `assistant.search.context` symbols, Slack message action discovery and dispatch do not expose search, and the runtime read path only calls `conversations.history`/`conversations.replies`. The related `search.messages` work in #65691 is adjacent, but it does not satisfy the requested assistant-token RTS behavior.

## What This Changes

Request support for Slack Real-Time Search by extracting assistant action tokens from Slack assistant context events and exposing `assistant.search.context` through a Slack message action or dedicated tool.

## Best Possible Solution

Implement this in the bundled Slack plugin after maintainer scoping: capture user-initiated RTS action tokens only for the relevant Slack assistant/message context, add a typed `assistant.search.context` runtime/message action, document required Slack AI/search scopes and privacy/rate-limit constraints, and keep the legacy `search.messages` path separate unless maintainers explicitly consolidate the API surface.
