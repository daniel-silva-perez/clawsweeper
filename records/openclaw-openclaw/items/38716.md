## Summary

Keep open. Current main still lacks the requested Slack RTS path: there is no `assistant.search.context` implementation, no `action_token` capture or propagation from Slack events, and no Slack `search` message action. The related Slack `search.messages` work is useful adjacent context, but it is user-token search and does not satisfy bot-token RTS via event-scoped action tokens.

## What This Changes

Request support for Slack Real-Time Search by extracting event `action_token` values and exposing `assistant.search.context` through the Slack message tool.

## Best Possible Solution

Implement this in the bundled Slack plugin behind an explicit RTS contract: capture action tokens only from authorized inbound Slack events, keep them scoped to the originating team/account/channel/thread/request, expose a typed read-only `assistant.search.context` action, and document the required AI/search scopes plus privacy and rate-limit boundaries.

  - body: RTS can return broad workspace context depending on scopes and channel visibility, so maintainers should define consent, result sharing, logging, redaction, and rate-limit behavior before exposing it to agents.
