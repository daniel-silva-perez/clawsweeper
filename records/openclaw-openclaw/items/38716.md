## Summary

Keep open. Current main still lacks the requested Slack RTS/action-token search path: there is no `assistant.search.context` implementation, no Slack `action_token` capture or propagation from inbound Slack events, and no Slack `search` message action. The related `search.messages` work is useful adjacent context, but it is user-token legacy search and does not satisfy the requested bot-token RTS behavior.

## What This Changes

The issue requests Slack Real-Time Search support by capturing event-scoped Slack assistant action tokens and exposing a Slack search action backed by `assistant.search.context`.

## Best Possible Solution

Implement this in the bundled Slack plugin with an explicit RTS contract: capture the action token from the authorized inbound Slack event, keep it scoped to the originating team/account/user/channel/thread/request, expose a typed read-only `assistant.search.context` action, and document the required AI/search scopes plus privacy and rate-limit boundaries.
