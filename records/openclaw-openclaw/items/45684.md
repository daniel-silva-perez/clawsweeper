## Summary

Keep this PR open. Current main still does not implement the Slack DM message-tool threading fix: the Slack tool context keeps only the native `D...` channel id for DMs, the auto-thread resolver still rejects `user:` targets, and the current tests still assert that `user:U123` does not inherit the active thread. The PR remains a plausible fix candidate, especially after the author's two-field follow-up, but it needs a focused rebase onto `extensions/slack` and should split the unrelated dependency/CI changes before maintainer review.

## Best Possible Solution

Keep this PR open and ask for a focused rebase: implement the Slack fix under `extensions/slack`, preserve `currentChannelId` as the native `D...` id for inferred Slack actions, add a generic current-recipient/current-target field or Slack-owned resolver context for matching `user:<id>` DM sends, update `resolveSlackAutoThreadId`, and add focused regression tests for DM message-tool media/file sends. Split or drop the unrelated dependency patch and platform-test guards.
