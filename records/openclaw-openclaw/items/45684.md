## Summary

Keep this PR open. Current main still does not implement the Slack DM message-tool threading fix: DM tool context keeps the native `D...` channel id only, Slack auto-thread resolution still rejects `user:` targets, and the current tests still assert the unfixed behavior. The PR remains a plausible fix candidate, especially after the author's two-field follow-up and the existing review comment, but it needs a focused rebase onto the current `extensions/slack` layout and should drop or split unrelated dependency/CI changes.

## Best Possible Solution

Keep this PR open for a focused maintainer review path: rebase it onto `extensions/slack`, implement the Slack-owned DM thread matching while preserving native `D...` `currentChannelId` for inferred Slack actions, add focused regression tests for DM message-tool media/file sends, and split the unrelated dependency/CI changes into separate work.
