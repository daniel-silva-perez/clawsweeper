## Summary

Keep this PR open. Current main still does not implement the Slack DM message-tool threading fix: DM context preserves only the native `D...` channel as `currentChannelId`, the Slack auto-thread resolver still rejects `user:` targets, and the tests still assert that `user:U123` does not auto-thread. The PR remains a plausible fix candidate, but it needs a focused rebase onto the current `extensions/slack` layout and should split unrelated dependency/CI test changes before maintainer review.

## Best Possible Solution

Keep this PR open and ask for a focused rebase: preserve Slack DM `currentChannelId` as the native `D...` ID for inferred Slack actions, add a generic current-recipient/current-target field or equivalent Slack-owned resolver seam for `user:<id>` matching, update `resolveSlackAutoThreadId` to thread matching DM message-tool sends, and add focused regression tests for Slack DM media/file sends. Split the unrelated dependency patch, lockfile changes, and platform-test guards into separate maintainer-reviewed work if still needed.
