## Summary

Keep this PR open. Current main still does not implement the Slack DM message-tool threading fix: Slack DM tool context keeps the native `D...` channel ID only, the Slack auto-thread resolver still rejects `user:` targets, and tests still encode that rejection. The PR remains a plausible implementation candidate, but it needs a focused rebase onto the current `extensions/slack` layout and should split unrelated dependency/CI changes before maintainer review.

## Best Possible Solution

Keep this PR open and ask for a focused rebase: preserve Slack DM `currentChannelId` as the native `D...` ID for inferred Slack actions, add a generic current-recipient/current-target field or equivalent plugin-owned seam for `user:<id>` matching, update `resolveSlackAutoThreadId` to thread matching DM message-tool sends, and add focused regression tests for Slack DM media/attachment sends. Split the `@pierre/diffs` patch, lockfile changes, and unrelated platform-test guards into a separate maintainer-reviewed PR if they are still needed.
