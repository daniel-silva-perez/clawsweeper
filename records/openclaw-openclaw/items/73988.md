## Summary

Keep this PR open. Current main still rejects external symlink targets for fixed workspace bootstrap files, so the requested bug fix is not implemented, but this patch is not merge-ready because it changes a workspace-boundary read into an arbitrary readable local-file read guarded only by a narrow OpenClaw-state denylist and overlaps the open maintainer/security-routed #73401 path.

## What This Changes

The PR adds a fallback that loads fixed top-level workspace bootstrap files through symlinks to external regular files, plus regression tests and a changelog entry.

## Best Possible Solution

Land one canonical maintainer-approved fix that preserves the narrow fixed-bootstrap scope, keeps extra/glob bootstrap files workspace-bound, and only permits external targets through an explicit and auditable trust policy such as containment, user opt-in, or a reviewed allowlist with fd-limited regular-file reads.

## Reproduction Assessment

Yes. The user bug is reproducible from the reports by creating a top-level workspace bootstrap symlink to a readable file outside the workspace and observing current main mark it missing; the PR security issue is reproducible from the patch by pointing that symlink at any readable non-OpenClaw sensitive file that is not covered by the denylist.

## Solution Assessment

No. The patch solves the functional missing-file symptom, but following arbitrary external symlink targets with only a narrow denylist is not the narrowest maintainable fix for a prompt-injection/file-read boundary; the safer path is maintainer/security review of a contained or explicitly opted-in policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep external bootstrap symlinks behind an explicit trust boundary:** `src/agents/workspace.ts:215`
  - body: The new fallback allows a top-level `AGENTS.md`/`SOUL.md` symlink to resolve to any readable local file, then only rejects a narrow set of OpenClaw state paths. A repository can point `AGENTS.md` at `~/.ssh/id_rsa` or cloud credentials and have that content loaded into bootstrap prompt context. Please require the resolved target to stay inside an approved boundary or gate external targets behind an explicit user opt-in before reading them.
  - confidence: 0.91
