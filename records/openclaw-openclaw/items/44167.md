## Summary

Keep this PR open. Current main still lacks the PR's intended memory-core behavior: embedding batch retries only classify service/rate-limit failures, do not treat transient transport failures as retryable, and do not split an exhausted multi-item batch while preserving result order. The PR is a focused implementation candidate for the still-open paired bug report #44166, and the related live-query issue #71784 does not supersede this batch reindex path.

## Best Possible Solution

Keep the PR open for maintainer review, or land an equivalent scoped memory-core fix that adds a narrow transport retry classifier, reuses the existing backoff loop, recursively splits exhausted multi-item text and structured embedding batches while preserving order, and keeps focused regression coverage in `extensions/memory-core/src/memory/manager-embedding-policy.test.ts`. Track live single-query memory search transport reliability separately under #71784.
