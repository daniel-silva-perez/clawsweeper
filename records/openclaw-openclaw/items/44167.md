## Summary

Keep this PR open. Current main still has service/rate-limit-only embedding retry classification, whole-batch retry without a transport split fallback, and no regression coverage for transient transport errors. The PR remains a focused implementation candidate for the still-open paired bug #44166; #71784 covers a related but distinct live-query path.

## Best Possible Solution

Keep this PR open for maintainer review, or land an equivalent scoped memory-core fix that adds a narrow transport retry classifier, reuses the existing backoff loop, recursively splits exhausted multi-item text and structured embedding batches while preserving order, and adds focused regression coverage in `extensions/memory-core/src/memory/manager-embedding-policy.test.ts`. Track the live single-query memory search reliability path separately under #71784.
