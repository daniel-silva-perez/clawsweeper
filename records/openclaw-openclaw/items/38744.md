## Summary

Keep this PR open. Current main still does not implement the PR's native iOS ElevenLabs realtime WebSocket STT path: iOS Talk Mode continuous and push-to-talk capture remain Apple SFSpeechRecognizer-backed. Main has adjacent ElevenLabs realtime transcription for Voice Call and browser realtime Talk gateway methods, but neither is wired into native iOS Talk Mode capture. The PR also still has unresolved reliability findings and a provider/credential-boundary decision before it is merge-ready.

## Best Possible Solution

Keep this PR open for maintainer/product review. If the feature proceeds, choose the architecture first: either rebase and harden a native iOS ElevenLabs STT client, or route iOS Talk through a provider-generic gateway realtime transcription/voice path. Before merge, address the existing WebSocket readiness, PTT commit, fallback, permission, recovery, settings persistence, docs, and focused iOS test coverage gaps.
