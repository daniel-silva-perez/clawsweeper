## Summary

Keep this PR open. Current main still lacks the requested gateway subagent binding flag at the command execution, auth probe, and slug generation embedded-run call sites, so the PR is not obsolete; however, the proposed probe test teardown should restore any pre-existing OPENAI_API_KEY instead of deleting it.

## What This Changes

This PR adds allowGatewaySubagentBinding to embedded agent runs in command execution, model auth probes, and LLM slug generation, with regression tests for those call boundaries.

## Best Possible Solution

Land the narrow flag propagation with tests after fixing the probe test environment restoration and rebasing onto current main without dropping newer tests. The production change should stay limited to the embedded-run call sites unless maintainers choose a broader runtime-default contract.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main's affected runEmbeddedPiAgent call sites omit the opt-in flag, while the runtime only late-binds gateway subagents when that flag is true; the PR's focused tests exercise the expected call arguments.

## Solution Assessment

Mostly yes for the product bug: explicit opt-in at these gateway-adjacent embedded paths matches the existing runtime contract and other call-site patterns. The test teardown is not the best implementation because it should restore the original OPENAI_API_KEY value rather than unconditionally deleting it.
