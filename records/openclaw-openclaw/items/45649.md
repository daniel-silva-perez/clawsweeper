## Summary

Keep open. Current main still preserves the main parts of the reported v2 tool-bubble problem: tool cards are collapsed by default, the app explicitly passes `autoExpandToolCalls: false`, collapsed summaries still use small muted ellipsized text, and the awkward `with` detail prefix remains in the web UI formatter.

## Best Possible Solution

Keep this issue open and fix it in the Control UI chat surface: increase tool-bubble contrast and type scale in `ui/src/styles/chat/tool-cards.css`, avoid premature summary truncation or show enough command text within the message width, add a real default-expanded or persisted auto-expand preference instead of forcing per-tool clicks, and remove the blanket `with` prefix when tool details already read naturally. Add focused jsdom coverage for expansion/default wording plus a visual regression check for the summary bubble.
