## Summary

Keep this issue open: current main still has the same Control UI tool-bubble mechanics called out in the report, including collapsed-by-default tool summaries, muted/small summary text, ellipsis truncation, a 760px tool-shell width cap, and detail wording that can still prepend connector text. The prior discussion identifies this as a compound v2 readability regression rather than a stale or already-shipped fix.

## What This Changes

The issue asks for v2 Control UI tool-message bubbles to be readable, avoid early truncation, clean up awkward wording, and reduce repeated manual expansion.

## Best Possible Solution

Update the Control UI tool-card surface so collapsed tool summaries are readable in light and dark themes, use available chat width without premature ellipsis, avoid awkward detail connector text, and provide an ergonomic expansion default or persisted preference backed by focused rendering tests and a screenshot check.

## Reproduction Assessment

Yes. A source-level reproduction is clear: render a tool card or tool message in the chat view with autoExpandToolCalls false and inspect the collapsed summary button, current CSS, and width cap; a browser screenshot should still confirm the visual severity.

## Solution Assessment

No. Current main is not the solution because it retains the collapsed, muted, truncated path; the narrow maintainable fix belongs in the Control UI chat/tool-card rendering and styles rather than core runtime or provider code.
