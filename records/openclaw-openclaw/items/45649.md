## Summary

Keep this open. Current main still exposes collapsed Control UI tool summaries by default, with small/muted ellipsized summary text, a capped tool-shell width, and the UI-specific detail formatter path that can add the awkward connector wording discussed in the issue. Related open PRs address expansion preferences, mobile overflow, or sidebar inspection slices, but they do not supersede the full collapsed-bubble readability bundle.

## What This Changes

The issue asks for Control UI v2 tool bubbles to be readable, use available width without premature truncation, avoid repeated forced expansion, and clean up awkward summary wording.

## Best Possible Solution

Land a focused Control UI repair that improves collapsed tool-summary contrast and type scale in light and dark themes, lets summaries use available chat width without premature ellipsis, removes or rewrites the awkward connector wording, and coordinates expansion defaults with the existing verbose/toggle work without changing core runtime behavior.

Yes. A high-confidence source-level reproduction exists by rendering a tool card or standalone tool row with `autoExpandToolCalls` false and inspecting the collapsed summary markup, CSS truncation rules, width cap, and UI detail formatter path on current main.

## Solution Assessment

No. Current main is not the solution because it keeps the problematic collapsed, muted, truncated path; the narrow maintainable fix belongs in Control UI rendering/styles plus a small wording adjustment, not in gateway or tool execution code.
