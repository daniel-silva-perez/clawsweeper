## Summary

Keep this open. Current main still renders Control UI tool messages through a collapsed summary button with muted/small text, ellipsis truncation, a 760px tool-shell width cap, disabled default expansion, and detail formatting that can still prepend the awkward connector text discussed in the report.

## What This Changes

Make Control UI v2 tool bubbles readable, less prematurely truncated, cleaner in detail wording, and less tedious to inspect.

## Best Possible Solution

Update the Control UI chat/tool-card rendering and CSS so collapsed tool summaries are legible in light and dark themes, use available chat width without premature ellipsis, clean up detail connector wording, and coordinate expansion behavior with the existing verbose/toggle work rather than changing core runtime behavior.
