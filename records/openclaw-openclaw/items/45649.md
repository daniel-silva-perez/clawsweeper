## Summary

Current main still does not show strong evidence that the v2 Control UI tool-bubble readability regression is fixed. The tool-result shell remains a collapsed button by default, uses small muted/truncated summary text, keeps a 760px width cap, and still has the `with ...` detail-prefix path noted in the issue discussion.

## Best Possible Solution

Keep this issue open and fix the Control UI tool-card surface: make collapsed tool bubbles readable in light and dark themes, avoid the early 760px/ellipsis cutoff when there is room, clean up the detail wording that produces awkward `with ...` copy, and make expansion ergonomic either through a clear persisted preference or a conservative default. Add focused rendering/style tests plus a browser screenshot check.
