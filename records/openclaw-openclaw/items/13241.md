## Summary

Keep open. Current main has adjacent stale cleanup, duplicate cleanup, intake templates, routing labels, and broad roadmap text, but it still does not define the requested issue lifecycle states, P0-P3 priority meanings, priority assignment rules, triage cadence, visible cycle-priority surface, or state-aware stale transitions.

## What This Changes

The issue proposes a lightweight repository triage framework with lifecycle labels, P0-P3 priorities, a regular triage cadence, visible current priorities, and state-aware stale handling.

## Best Possible Solution

Define a small repo-owned triage playbook plus matching GitHub labels, project or pinned-priorities surface, and any narrowly necessary automation that reuses the existing stale, Barnacle, issue-template, duplicate-cleanup, and roadmap pieces.

## Reproduction Assessment

Not applicable as a feature/process proposal. The current-main verification path is source, docs, workflow, and history inspection, which shows the requested framework is still absent.

## Solution Assessment
