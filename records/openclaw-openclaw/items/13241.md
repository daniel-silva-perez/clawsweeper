## Summary

Keep open. Current main has stale cleanup, Barnacle auto-response/candidate labels, path/size/beta labels, issue templates, duplicate-PR cleanup, and broad roadmap priorities, but it still does not implement the requested lightweight issue triage framework with lifecycle labels, P0-P3 priority semantics, assignment rules, cadence, a visible current-cycle priorities surface, and documented stale-state transitions.

## Best Possible Solution

Keep this open for maintainer process design. A complete solution should reuse the existing stale workflow, Barnacle cleanup labels, issue templates, duplicate cleanup, and roadmap docs, then add a minimal repo-owned triage playbook plus matching label/project surface that defines lifecycle labels, P0-P3 semantics, who assigns priority, cadence, stale-state transitions, and a visible current-cycle priorities board or pinned issue.
