## Summary

Keep open. Current main has issue templates, stale cleanup, duplicate cleanup, Barnacle routing labels, maintainer labels, and broad roadmap priorities, but it still does not implement the requested triage framework with lifecycle states, P0-P3 meanings, assignment rules, cadence, visible cycle priorities, or state-aware stale transitions; the existing maintainer-side discussion also explicitly kept this open for process follow-up.

## What This Changes

The issue proposes a lightweight issue triage and prioritization framework with lifecycle labels, priority semantics, triage cadence, visible cycle priorities, and state-aware stale handling.

## Best Possible Solution

Define a small repo-owned triage playbook and matching labels/project or pinned-priorities surface that reuses existing stale, Barnacle, issue-template, duplicate-cleanup, and roadmap pieces while explicitly documenting lifecycle states, P0-P3 meanings, who assigns priority, cadence, stale-state transitions, and visible current-cycle priorities.
