## Summary

Keep open. Current main has adjacent intake, stale-cleanup, duplicate-cleanup, routing-label, and roadmap pieces, but it still does not define the requested issue lifecycle states, P0-P3 priority meanings, priority assignment rules, triage cadence, visible cycle-priority surface, or state-aware stale transitions; the existing maintainer-side discussion also explicitly kept this open for process follow-up.

## What This Changes

The issue proposes a lightweight repository triage framework with lifecycle labels, priority levels, regular cadence, visible current-cycle priorities, and clearer stale handling.

## Best Possible Solution

Define a small repo-owned triage playbook plus matching GitHub labels/project or pinned-priorities surface that reuses the existing stale, Barnacle, issue-template, duplicate-cleanup, and roadmap pieces while documenting lifecycle states, P0-P3 meanings, who assigns priority, cadence, stale-state transitions, and current-cycle priorities.
