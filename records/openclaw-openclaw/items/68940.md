## Summary

Keep open. Current main fixes the original missing `openclaw agent --model` flag and now has a plugin-owned `codex-cli/gpt-5.5` default, but the CLI Backends quick-start still documents `openclaw agent --message "hi" --model codex-cli/gpt-5.5` without the required session selector. Both Gateway and embedded/local agent paths reject selector-free runs, so the documented quick-start remains invalid as written.

## Best Possible Solution

Keep this issue open and update `docs/gateway/cli-backends.md` so the beginner quick-start uses a command current `openclaw agent` accepts, for example adding `--agent main` or another documented selector. If maintainers want selector-free one-shot model probes, implement that runtime behavior explicitly with tests instead. While touching docs, audit nearby no-selector `openclaw agent --message ...` examples so the public CLI guidance matches the enforced session-selection contract.
