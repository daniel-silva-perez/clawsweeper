## Summary

Keep open. Current main fixed the reported missing `openclaw agent --model` flag and now has plugin-owned `codex-cli` defaults, but the CLI Backends quick-start snippet still omits the required session selector, so the documented command remains unusable as written.

## Best Possible Solution

Keep this issue open and update `docs/gateway/cli-backends.md` to use a command that current `openclaw agent` accepts, for example adding `--agent main` or another documented selector. If the intended quick start is truly selector-free, add an explicit runtime behavior change plus tests so `openclaw agent --message ... --model ...` resolves a default main session instead of throwing. Preserve the already-shipped `--model` and plugin-owned `codex-cli/gpt-5.5` implementation.
