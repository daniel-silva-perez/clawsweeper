## Summary

Keep open. Current main has fixed the original missing `openclaw agent --model` surface and now registers the bundled `codex-cli` backend, but the CLI Backends quick-start still documents a selector-free `openclaw agent --message ... --model ...` command that both the gateway and embedded agent paths reject. The linked open PR #74613 is the right active fix path, so this issue should stay open until that PR lands or is closed.

## What This Changes

The issue requests correcting the CLI Backends quick-start docs because the documented `openclaw agent --message ... --model codex-cli/...` command lacks a required session selector and failed on released CLI builds.

## Best Possible Solution

Review and land or adjust #74613 so the CLI Backends quick start uses a selector accepted by current CLI behavior, such as `--agent main`, and briefly documents why that default agent id works. If maintainers instead want selector-free quick starts, that should be a deliberate runtime change with tests rather than a docs-only cleanup.

## Reproduction Assessment

Yes. The current docs provide a selector-free command, while both the gateway and embedded source paths deterministically throw without `--to`, `--session-id`, or `--agent`; this source-level check is enough to reproduce the reported docs/runtime mismatch.

## Solution Assessment

Yes, the narrow docs fix proposed by #74613 appears to be the best current path because `--model` and `codex-cli` registration already exist on main. Changing `openclaw agent` to infer a default session would be broader runtime behavior and should only happen if maintainers explicitly want that product direction.
