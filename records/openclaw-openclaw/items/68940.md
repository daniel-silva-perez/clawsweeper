## Summary

Keep open. Current main has fixed the original `openclaw agent --model` flag and bundled `codex-cli` registration pieces, but the CLI Backends quick start still shows a selector-free `openclaw agent --message ... --model ...` command that current gateway and embedded paths reject. The linked open PR #74613 is the right active fix path, so this issue should remain open until that PR lands or is otherwise resolved.

## What This Changes

The issue requests that the CLI Backends quick-start docs be corrected so the documented `openclaw agent` command matches released/current CLI behavior and default `codex-cli` availability.

## Best Possible Solution

Review and land #74613, or an equivalent narrow docs change, so the CLI Backends quick start includes a valid selector such as `--agent main` and explains the default agent id. Keep the current explicit selector requirement unless maintainers intentionally choose a broader runtime defaulting change with tests.

## Reproduction Assessment

Yes. Current source and docs provide a high-confidence reproduction: the quick start omits a selector, while both gateway and embedded `openclaw agent` paths throw when no selector is provided. The original missing `--model` part no longer reproduces on current main.

## Solution Assessment

Yes. The linked docs-only PR is the narrowest maintainable fix for the remaining mismatch because `--model` and bundled `codex-cli` registration already exist on main; making selector-free `openclaw agent` infer `main` would be a broader runtime behavior change.
