## Summary

Keep this PR open. Current main still has the reported live `channels status --probe` truncation, the read-only channel summary helper still asserts the old two-entry behavior, and the security/privacy boundary around full `allowFrom` rendering still needs maintainer review. The current PR context is useful because later commits address both prior review findings; #51180 overlaps, but it is not a clean supersedence because it covers a narrower surface.

## Best Possible Solution

Keep this PR open for maintainer review or consolidate it with #51180. The best fix removes silent two-entry truncation from the live `formatGatewayChannelsStatusLines()` path and any summary helper that intentionally exposes `allowFrom`, adds regression coverage for three-or-more entries, and keeps full allowlists out of non-admin/redacted status responses unless maintainers explicitly choose another privacy policy.
