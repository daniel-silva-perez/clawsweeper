## Summary

Keep this PR open. Current main still silently truncates `allowFrom` to two entries in the live `channels status --probe` formatter and the channel summary helper, while the PR now addresses both earlier review findings by updating the live formatter and keeping full allowlist details out of non-sensitive status summaries. The overlapping #51180 is a competing capped-display policy, not a clear supersedence.

## What This Changes

The PR removes two-entry `allowFrom` truncation from channel status formatting and channel summaries, updates focused expectations, and gates status-summary allowlist details on sensitive/admin output.

## Best Possible Solution

Land one consolidated display-policy fix that removes silent two-entry truncation from the live `channels status --probe` path and any intentional allowlist summaries, adds regression coverage for three-or-more entries, and preserves non-admin/redacted status summaries unless maintainers explicitly broaden that exposure.

## Solution Assessment

Unclear, leaning yes. This PR is now a narrow fix for both affected formatter paths and the earlier non-admin status-summary privacy concern; the remaining maintainer decision is whether full rendering is preferred over the capped `+N more` policy proposed in #51180.
