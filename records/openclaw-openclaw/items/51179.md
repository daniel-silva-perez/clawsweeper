## Summary

Keep this PR open. Current main still has the reported `channels status --probe` two-entry `allowFrom` truncation in the live formatter and in the read-only channel summary helper, while existing tests still assert the old truncated behavior. The PR context is useful: early review correctly caught that the first commit missed the live command path and widened redacted status summaries, and later PR commits/diff address both by updating the live formatter and gating status-summary allowlist details on sensitive/admin output. #51180 overlaps, but it is a narrower competing display-policy fix rather than a clear supersedence.

## Best Possible Solution

Keep this PR open for maintainer review or consolidate it with #51180. The best fix should remove silent two-entry truncation from the live `formatGatewayChannelsStatusLines()` path and any intentional allowlist summary helper, add regression coverage for three-or-more entries, and keep non-admin/redacted status summaries from exposing full allowlists unless maintainers deliberately broaden that policy.
