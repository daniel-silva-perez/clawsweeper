## Summary

Keep this PR open. Current main still has the user-visible truncation in the live `openclaw channels status --probe` formatter, still truncates the fallback channel-summary helper, and still has the redacted-status privacy path that matters when rendering full allowlists.

## Best Possible Solution

Keep this PR open for maintainer review or consolidate it with #51180. The best implementation is a small shared formatting fix that removes silent two-entry truncation from both `formatGatewayChannelsStatusLines()` and `buildChannelSummary()`, adds regression coverage for three-or-more entries, and gates full allowlist display behind sensitive/admin status summaries.
