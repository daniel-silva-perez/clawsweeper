## Summary

Keep this PR open. Current main still has the reported user-visible truncation in the live `openclaw channels status --probe` formatter, the fallback channel summary still truncates `allowFrom`, and current tests still assert the old two-entry behavior. The PR remains useful, with the main maintainer question being how to land it alongside or instead of #51180 while preserving the non-admin privacy boundary.

## Best Possible Solution

Keep this PR open for maintainer review or consolidate it with #51180. The best fix removes silent two-entry truncation from both the live `formatGatewayChannelsStatusLines()` path and the config-only `buildChannelSummary()` path, adds regression coverage for three-or-more `allowFrom` entries, and keeps full allowlists out of non-admin health/status summaries.
