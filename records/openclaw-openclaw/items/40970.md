## Summary

Keep open. Current main still exposes a literal `YYYY-MM-DD` through resolved logger settings and `logs.tail`, while this PR branch addresses that gap with focused regression coverage. The remaining concern is an adjacent daemon-status display path, so the right next step is maintainer review rather than cleanup closure.

## What This Changes

The PR adds `YYYY-MM-DD` placeholder resolution for `logging.file` in logger settings/runtime path handling and adds regression coverage for resolved settings and log tail.

## Best Possible Solution

Land this focused logging fix after normal maintainer review, with date-placeholder resolution shared across logger settings, log tail, and daemon status or with daemon status explicitly accepted as a follow-up.
