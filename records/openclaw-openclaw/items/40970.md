## Summary

Keep open. Current main still leaves configured `logging.file` placeholders unresolved, so the PR addresses a real logging gap, but the branch is not merge-ready because the new tests are timezone-fragile and the status path still reports the literal configured placeholder.

## What This Changes

The PR resolves `YYYY-MM-DD` placeholders in configured logging file paths, updates rolling-path recognition, and adds logger settings and log-tail regression tests.

## Best Possible Solution

Land placeholder support after the PR uses host-local-safe date fixtures, keeps logger, tailing, and status path resolution consistent, and passes the targeted logging tests plus the changed gate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use a host-local date in the log-tail test:** `src/logging/log-tail.test.ts:60-63`
  - body: The logger formats dates with local `getFullYear()`/`getMonth()`/`getDate()`. With this fixed UTC instant, `America/Los_Angeles` resolves `2026-04-26`, but the test writes and expects `openclaw-2026-04-27.log`, so it fails on non-UTC hosts. Use a local-component or local-noon fixture, or derive the expected filename from the same local date contract.
  - confidence: 0.92
- **[P3] Use a host-local date in the settings test:** `src/logging/logger-settings.test.ts:54-62`
  - body: This test has the same local-date contract. `2026-03-09T08:30:00.000Z` becomes `2026-03-08` in zones such as `Pacific/Honolulu`, while the expected filename is hard-coded to March 9. Build the fake time from local components so the test matches the logger everywhere.
  - confidence: 0.84
- **[P3] Share placeholder resolution with status output:** `src/logging/logger.ts:688-697`
  - body: The new resolver is private to `logger.ts`, but daemon status still reports `resolveConfiguredLogFilePath(cliCfg)`, whose helper returns `config.logging.file` unchanged. With a placeholder config, logging and tailing can use the dated path while status still shows the literal placeholder. Share the resolver with the status helper or explicitly scope the status behavior.
  - confidence: 0.84
