## Summary

Keep this PR open. Current main still writes REM separate reports as `# REM Sleep` followed by the raw REM preview body headings at `###`, while inline daily-note output uses the same raw body under `## REM Sleep`. The latest PR context addresses the bot review by preserving inline `###` headings and lowering them only for standalone reports, so the work is still useful and not implemented on main.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for the remaining heading-depth bug. The desired path is to land a mode-specific fix in `extensions/memory-core/src/dreaming-markdown.ts` with regression coverage for both inline daily-note output and standalone REM reports. The related #72284 report should stay separate because it concerns `DREAMS.md` creation and Control UI visibility, not this per-phase report heading structure.
