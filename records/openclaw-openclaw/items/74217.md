## Summary

Keep this issue open. Current main still routes heartbeat runs through `agents.defaults.heartbeat.model`, persists the actual runtime provider/model into the shared session entry, and then treats that persisted runtime identity as the session-status card model when no explicit session override exists. The docs currently describe `isolatedSession` as a workaround for heartbeat model bleed, and related PR #51706 remains open/unmerged rather than a fix for this report.

## What This Changes

The issue asks OpenClaw to stop heartbeat/internal runs from making normal user session status show the heartbeat model as the selected or runtime model for ordinary turns.

## Best Possible Solution

Keep this open and implement a focused fix that separates user-selected/default model identity from internal heartbeat runtime identity. The safest path is either to prevent same-session heartbeat/internal runs from overwriting the `SessionEntry.modelProvider`/`model` fields used as the selected card model, or to store/display heartbeat runtime identity separately while preserving normal user-run fallback/runtime metadata. Coordinate with #51706 and add regression tests for shared-session heartbeat model overrides with no session override and empty fallback lists.
