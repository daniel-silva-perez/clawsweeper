## Summary

Keep open. Current main still builds the `resolveExtraParams` model lookup key with raw `provider/modelId` concatenation, so provider-prefixed model IDs such as `openrouter/auto` still look up `openrouter/openrouter/auto` and miss canonical per-model params. This PR is a narrow replacement for the closed, unmerged source PR #44319 and applies the existing canonical `modelKey` plus `legacyModelKey` fallback pattern with regression coverage.

## Best Possible Solution

Keep this PR open, update it against current main, and merge it if review and CI are clean. The right fix boundary is the narrow one in the PR: use `modelKey` plus `legacyModelKey` fallback inside `resolveExtraParams`, keep regression coverage for canonical `openrouter/auto` and legacy double-prefixed keys, preserve @HenryXiaoYang's source PR #44319 credit, and avoid unrelated workflow, dependency, or release-surface changes.
