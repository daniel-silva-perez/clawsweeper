## Summary

Keep open. Current main still silently filters `sessions_list` results under the effective session visibility policy: default visibility remains `tree`, denied rows are skipped without a note, and the returned tool payload is only `{ count, sessions }`. The discussion's KeepMyClaw workaround and #50621 mention are useful context for the same visibility/observability pain, but they do not supersede or fix this OpenClaw session-tool UX gap.

## Best Possible Solution

Keep the issue open and implement an additive visibility disclosure for session tools, starting with `src/agents/tools/sessions-list-tool.ts`: include the effective `tools.sessions.visibility`, whether a sandbox clamp applied, and a clear note when results are scoped. If filtered counts are added, compute them carefully without weakening the existing visibility boundary; otherwise document that counts are visible-only. Add focused tests for default `tree`, `self`, `agent`, `all`, and sandbox-clamped behavior, and update the session tool docs.
