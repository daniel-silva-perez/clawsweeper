## Summary

Keep open. The report matches current main behavior: WebChat sends `chat.send` with `deliver: false`, and the Gateway explicitly prevents WebChat clients from inheriting stored external delivery routes even on channel-scoped sessions. Changing this would reverse or qualify a deliberate cross-channel leakage safety policy, so it needs maintainer product/security judgment rather than conservative auto-closure.

## Best Possible Solution

Keep this open for maintainer judgment. The best path is to decide whether WebChat should remain local-only for safety, or whether OpenClaw should add an explicit opt-in delivery affordance for selected channel-scoped sessions that preserves #39175 safeguards. If the latter is chosen, update the WebChat UI request path, `chat.send` route resolution, regression tests, and WebChat/channel-routing docs together.
