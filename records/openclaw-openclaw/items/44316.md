## Summary

Keep this PR open. Current main still does not implement the PR's intended behavior: tool-shaped history is only hidden when the Control UI tool-calls toggle is off, live tool cards remain an intentional visible mode, and the UI text extraction path still has no SECURITY NOTICE / EXTERNAL_UNTRUSTED_CONTENT stripping helper. The PR remains useful, but it is stale against the current chat-item split and needs maintainer judgment on whether tool plumbing should be hidden even when tool cards are enabled and how narrowly external-content marker redaction should apply.

## Best Possible Solution

Keep this PR open for maintainer review or an updated contributor patch. The best path is to rebase the behavior onto ui/src/ui/chat/build-chat-items.ts and ui/src/ui/chat/message-extract.ts, decide whether raw tool plumbing should be hidden even when tool cards are enabled, scope external marker stripping to confirmed tool-result/external-content payloads, and add focused UI chat tests for the reported role and marker cases.
