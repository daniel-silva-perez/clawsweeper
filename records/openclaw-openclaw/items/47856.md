## Summary

Keep open. Current main still does not expose a public configurable local media-root allowlist for the explicit image tool: the tool resolves only workspace/default roots, the public media config schema/types have no `mediaLocalRoots`-style field, and current tests still preserve rejection for non-workspace local image paths. Managed inbound media support helps one adjacent case, but it does not satisfy the requested opt-in roots for external attachment-heavy workflows.

## What This Changes

Add a configurable image/media-understanding local media root allowlist so explicit image-tool reads can opt into trusted attachment directories without manual copying.

## Best Possible Solution

Define and document a narrow opt-in local media-root allowlist for image/media understanding, validate entries as explicit non-root local directories, route them through the shared root resolver and loadWebMedia, preserve managed inbound-media behavior, and cover both configured acceptance and existing rejection cases in schema, type, docs, and tests.
