## Summary

Keep open. Current main still has managed inbound-media handling, but the explicit image tool does not expose a public configurable local-media-root allowlist for external attachment directories; its allowlist remains default/workspace-derived, and tests still assert arbitrary non-workspace local image paths are rejected. The request is valid but security-sensitive and overlaps the open maintainer-labeled PR #39034, so it should stay with maintainer review.

## What This Changes

Request to add a public configurable local-media-root allowlist for the explicit image tool so iMessage-style attachment paths outside the workspace can be read without manual copying.

## Best Possible Solution

Define and document a narrow opt-in local media root allowlist for image/media understanding, validate roots as explicit non-filesystem-root directories, route the resolved roots through the shared media-root resolver and `loadWebMedia`, preserve managed inbound-media behavior, and add schema/type/docs/tests for both configured acceptance and existing rejection cases.
