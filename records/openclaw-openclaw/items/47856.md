## Summary

Keep open. Current main still lacks a public configurable local-media-root allowlist for the explicit image tool: the resolver ignores caller media sources, the tool passes only default/workspace roots to `loadWebMedia`, and the public `tools.media.image` schema/type contract has no `mediaLocalRoots`-style field. Managed inbound media support covers OpenClaw-owned `media/inbound` references, but it does not satisfy external attachment roots such as iMessage paths under `/Users/.../Library/Messages/Attachments`.

## What This Changes

The issue requests a configurable image/media local-root allowlist so attachment paths outside the default workspace, especially iMessage attachments, can be read by the image tool without manual copying.

## Best Possible Solution

Define and document a narrow opt-in local media-root allowlist for image/media understanding, validate entries as explicit non-root local directories, route them through the shared root resolver and `loadWebMedia`, preserve managed inbound-media behavior, and cover both configured acceptance and existing rejection cases in schema, types, docs, and tests.
