## Summary

Keep open. Current main still does not expose a public configurable local-media-root allowlist for the explicit image tool, and the resolver used by the image tool still ignores source-derived roots. The supporting managed inbound-media case is now handled, but external iMessage attachment directories remain outside the explicit image-tool allowlist unless they are copied into an allowed root.

## What This Changes

Request a configurable image/media local-root allowlist so iMessage attachment paths outside the workspace can be read by the image tool without a manual copy step.

## Best Possible Solution

Define and document a narrow opt-in local media root allowlist for image/media understanding, validate roots as explicit non-filesystem-root directories, route the resolved roots through the shared media-root resolver and loadWebMedia, preserve managed inbound-media behavior, and add schema/type/docs/tests for both configured acceptance and existing rejection cases.
