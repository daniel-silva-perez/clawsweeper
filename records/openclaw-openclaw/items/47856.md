## Summary

Keep open. Current main still does not expose a public configurable local-media-root allowlist for the explicit image tool; managed inbound-media handling covers OpenClaw-owned inbound references, but not external attachment directories from iMessage-style workflows. The open maintainer-labeled PR #39034 overlaps the same behavior, so this should stay available for maintainer follow-up rather than be closed as stale.

## What This Changes

The issue requests an `openclaw.json` option that extends the image tool's allowed local media roots so attachment-heavy workflows can pass local images without copying them into the workspace.

## Best Possible Solution

Define a narrow opt-in image/media local-root allowlist in the public config contract, validate entries as explicit non-root local directories, route the resolved roots through the shared media-root resolver and `loadWebMedia`, preserve managed inbound-media behavior, and document/test both configured acceptance and existing rejection cases.
