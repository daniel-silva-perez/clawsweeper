## Summary

Keep open. Current main still lacks a public image-tool local media root allowlist: the explicit image tool resolves only default/workspace roots, and the public media config schema/types do not expose a `mediaLocalRoots`-style field. Managed inbound OpenClaw media is now exempted from root checks, which helps the supporting `media/inbound` case, but it does not cover external iMessage attachment directories such as `/Users/.../Library/Messages/Attachments`.

## What This Changes

Request to add a configurable image/media local-root allowlist so attachment-heavy workflows can read approved local image paths without copying files into the workspace.

## Best Possible Solution

Define and document a narrow opt-in local media root allowlist for image/media understanding, validate roots as explicit non-filesystem-root paths, route them through the shared root resolver and `loadWebMedia`, preserve managed inbound-media behavior, and add schema/type/docs/tests for allowed configured roots and existing rejection behavior.
