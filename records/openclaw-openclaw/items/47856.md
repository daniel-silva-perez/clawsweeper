## Summary

Keep this issue open. Current main still has no configurable image-tool local media root allowlist for external attachment directories. The managed inbound-media case discussed in comments is partially addressed, and outbound-media mediaLocalRoots work exists elsewhere, but neither implements the requested `tools.media.image.mediaLocalRoots`-style config surface for the image tool.

## Best Possible Solution

Keep open for maintainer/security design. The likely direction is an explicit opt-in image/media local-root allowlist with strict absolute path validation, clear per-agent/global precedence, documented interaction with `tools.fs.workspaceOnly`, and preservation of the existing managed-inbound-media exemption for gateway-owned files. Review #39034 and the outbound-media fixes as context, but land a focused, documented config contract before closing this issue.
