## Summary

Keep this issue open. Current main still does not expose a configurable image-tool local media root allowlist, and the explicit image tool continues to use only workspace/default roots before calling the local media loader. The managed inbound-media exemption now covers OpenClaw-owned `media://inbound` and absolute inbound-media paths, but it does not solve external iMessage attachment directories. Related PR #39034 is relevant but remains open and maintainer-labeled, so it is not a merged implementation for this request.

## What This Changes

Request to add a configurable image-tool local media root allowlist so external iMessage attachment paths can be loaded without copying them into the workspace.

## Best Possible Solution

Add a narrowly scoped opt-in image/media local-root allowlist, likely under `tools.media.image` or a clearly documented equivalent, with strict absolute path validation, filesystem-root rejection, explicit workspaceOnly semantics, docs, schema/type/help coverage, and tests that both allow configured external roots and preserve existing rejection/default behavior.
