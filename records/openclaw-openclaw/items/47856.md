## Summary

Keep this open. Current main still lacks a public image-tool local media root allowlist, and the explicit image tool continues to pass only default/workspace roots into local media loading. Managed inbound OpenClaw media is now exempted, which addresses part of the supporting comment, but external iMessage attachment directories remain outside the accepted roots unless another path-specific fix lands.

## What This Changes

The issue requests an opt-in configuration field that lets the image tool allow additional local media roots, especially iMessage attachment directories outside the default workspace/media roots.

## Best Possible Solution

Define and document a narrow opt-in local media root allowlist for image/media understanding, route it through the shared root resolver and `loadWebMedia`, reject filesystem roots, require explicit absolute paths, preserve managed inbound-media behavior, and add schema/type/docs/tests for both allowed configured roots and existing rejection behavior.
