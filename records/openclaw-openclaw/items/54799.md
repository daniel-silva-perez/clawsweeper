## Summary

Keep this open. Current main still has skill metadata, skill status, skills CLI, cron, heartbeat task, and plugin cron surfaces, but none of them implement declarative `SKILL.md` trigger metadata, automatic scheduler reconciliation from skill metadata, event-triggered skill registration, last-triggered skill status, or a skills audit command.

## What This Changes

The issue requests optional schedule and event trigger declarations in `SKILL.md` frontmatter, with scheduler registration, lifecycle event handling, last-triggered display, and never-triggered auditing for skills.

## Best Possible Solution

Define a versioned `metadata.openclaw.triggers` contract for skills, validate schedule and event trigger forms, reconcile schedule triggers into managed cron jobs with stable ownership, define approved lifecycle events for event-triggered skills, persist last/next trigger state, and expose that state through `openclaw skills list`, `openclaw skills info`, and a skills audit command.
