## Summary

Keep this issue open. Current main still has skills metadata for eligibility, requirements, installer hints, and UI display, but not declarative schedule/event triggers, scheduler reconciliation from SKILL.md, trigger history, or a skills audit surface.

## Best Possible Solution

Keep this open for maintainer product/API design. A strong implementation would add a versioned metadata.openclaw.triggers contract for SKILL.md, validate schedule and event trigger forms, reconcile schedule triggers into managed cron jobs, define approved lifecycle events for event-triggered skills, persist trigger registration and last-triggered history, and expose it through skills status, openclaw skills list/info, and a skill audit surface such as --never-triggered. If maintainers prefer plugin ownership, first add the missing public skill metadata/status and cron reconciliation APIs.
