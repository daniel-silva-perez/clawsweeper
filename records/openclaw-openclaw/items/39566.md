## Summary

Keep this PR open. Current main still lacks the requested Telegram `topics["*"]` fallback: group and DM topic resolution only checks exact `message_thread_id` keys, docs/types still describe exact thread IDs, and current tests cover exact topic routing/overrides but not wildcard topic defaults. The contribution remains useful, but it needs a refresh against the current `extensions/telegram` plugin layout and should drop unrelated `.secrets.baseline` churn.

## Best Possible Solution

Keep the PR open and ask for a refreshed, focused patch against current main: implement an exact-topic-then-`topics["*"]` resolver in `extensions/telegram/src/bot-core.ts` for both group and DM topic config paths, ensure topic-aware routing and allowlist helpers receive that resolved config, add extension tests for wildcard defaults and exact-topic priority, update `src/config/types.telegram.ts` and the public Telegram docs, and remove the unrelated `.secrets.baseline` changes.
