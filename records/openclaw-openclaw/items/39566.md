## Summary

Keep this PR open. Current main still lacks the requested Telegram `topics["*"]` default behavior: group and DM topic config resolution only checks the exact `message_thread_id`, and the public docs/types/tests do not document or cover wildcard topic defaults. The PR remains useful, but it needs a refresh against the current Telegram plugin layout.

## Best Possible Solution

Keep the PR open and ask for a refreshed, focused patch against current main: add an exact-topic-then-`topics["*"]` resolver in the Telegram plugin for both group and DM topic config paths, update any topic-aware policy helper that should honor the same default, add extension tests for wildcard defaults and exact-topic priority, update `src/config/types.telegram.ts` plus the public Telegram docs, and drop the unrelated `.secrets.baseline` churn.
