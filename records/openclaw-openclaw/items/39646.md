## Summary

Keep this PR open. Current main still has the stale raw default-model path in the status surfaces this PR targets, while the PR itself is too broad and incomplete to merge or close as obsolete.

## Best Possible Solution

Keep this PR open or replace it with a focused maintainer patch. Centralize effective default-model resolution for `openclaw status`, `models status --json`, rich model-status output, and auth-probe candidates; validate removed configured providers without rejecting CLI-backed or plugin/synthetic providers; add regression coverage for the removed-provider case; split unrelated changes into separate PRs.
