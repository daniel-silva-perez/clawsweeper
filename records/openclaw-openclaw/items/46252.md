## Summary

Keep open. Current main is only partially archive-aware: the global daily cost scan includes reset/deleted archive transcripts, but the per-session `sessions.usage` path still resolves and scans one transcript file, so an active `<id>.jsonl` plus sibling `.jsonl.reset.*` archive remains undercounted in dashboard totals.

## What This Changes

The issue asks the cost dashboard and usage summaries to include `.jsonl.reset.*` archive transcripts alongside active `.jsonl` files so `/new` and `/reset` users see complete spend.

## Best Possible Solution

Keep the archive-aware global scan, then add a shared transcript-segment resolver for a session id and have per-session cost aggregation include every usage-counted segment for that session: active `.jsonl` plus `.jsonl.reset.*` and `.jsonl.deleted.*`, while still excluding `.jsonl.bak.*`, compaction checkpoints, trajectory artifacts, and store backups.

## Reproduction Assessment

Yes. The issue body and follow-up provide concrete active-plus-reset file totals, and current source still reproduces the path statically because `loadSessionCostSummary({ sessionId })` resolves the primary transcript and scans only that file.

## Solution Assessment

Yes for the repair direction, not for current main. The narrow maintainable fix belongs in the infra session transcript resolution/aggregation path used by `sessions.usage`, not in a UI-only adjustment, because filtered totals, exports, and gateway RPCs all depend on the same per-session summaries.

## Repair Work Prompt

Fix the remaining cost-dashboard undercount for this issue. Current main already includes `.jsonl.reset.*` and `.jsonl.deleted.*` in the global `loadCostUsageSummary()` scan, but `sessions.usage` still flows through `loadSessionCostSummary()`, which resolves one file and returns the active `<sessionId>.jsonl` when it exists, dropping sibling reset/deleted archive segments for the same session id. Implement a shared resolver in `src/infra/session-cost-usage.ts` that returns all usage-counted transcript segments for a session id, preserving agentId and explicit/custom sessionFile behavior. Aggregate those segments in `loadSessionCostSummary()` so active `.jsonl` plus `.jsonl.reset.*`/`.jsonl.deleted.*` totals are combined. Consider whether `loadSessionUsageTimeSeries()` and `loadSessionLogs()` should use the same segment order for detail views, but keep the PR focused on the usage/cost bug. Do not count `.jsonl.bak.*`, compaction checkpoints, trajectory files, or `sessions.json.bak.*` backups. Add regression coverage for active transcript plus sibling reset archive totals in `src/infra/session-cost-usage.test.ts` and, if needed, `src/gateway/server-methods/usage.sessions-usage.test.ts`. Add a user-facing changelog fix entry crediting @PabloDaVa and @billybillymc. Do not rewrite the Control UI unless the gateway contract needs a small matching change.
