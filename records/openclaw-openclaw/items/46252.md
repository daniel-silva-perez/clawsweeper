## Summary

Keep open. Current main has a real partial fix: `usage.cost` now scans active transcripts plus `.jsonl.reset.*` and `.jsonl.deleted.*` archives, excluding `.bak`. But the Control UI still uses `sessions.usage.totals` for the unfiltered headline totals, and that backend path still collapses same-session primary/archive siblings to one file. That matches the reporter’s later “partial cost” reproduction where a primary `.jsonl` and same-session `.jsonl.reset.*` archive exist together.

## Best Possible Solution

Keep this issue open and finish the remaining backend/UI consistency work: either make `sessions.usage` aggregate all usage-counted sibling transcripts for the same session id, or make the default Control UI headline totals use the already-correct `usage.cost` aggregate. Add a regression test with `sess.jsonl` plus `sess.jsonl.reset.<timestamp>` verifying the default dashboard/session totals include both while detail/log file selection remains deterministic.
