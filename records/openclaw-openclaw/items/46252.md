## Summary

Keep open: current main only partially fixes the archive accounting path. The global `usage.cost` scan is archive-aware, but `sessions.usage` and the unfiltered Control UI totals still flow through a one-file per-session summary, so an active transcript plus sibling `.jsonl.reset.*` archive remains undercounted.

## What This Changes

The issue asks the Usage dashboard and usage-cost summaries to include `.jsonl.reset.<timestamp>` archive transcripts created by `/new` or `/reset`, while avoiding double-counting backup artifacts.

## Best Possible Solution

Add a shared transcript-segment resolver for a session id and make per-session cost summaries aggregate every usage-counted segment for that session: active `.jsonl` plus `.jsonl.reset.*` and `.jsonl.deleted.*`, excluding `.jsonl.bak.*`, compaction checkpoints, trajectory artifacts, and store backups. Then update `sessions.usage` regression coverage so dashboard totals cannot drift from archive-aware daily cost totals.

## Reproduction Assessment

Yes. The issue body and follow-up comment give concrete active-plus-reset file examples, and current source preserves that failure path because the primary transcript is selected and then scanned alone.

## Solution Assessment

Yes. The narrow maintainable fix is in session transcript resolution/aggregation for the `sessions.usage` path; the global daily `usage.cost` path already uses the right filename filter, so this should not require a broad UI rewrite.
