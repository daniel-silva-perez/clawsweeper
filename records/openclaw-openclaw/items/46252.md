## Summary

Keep open. Current main includes reset/deleted archive files in the global `usage.cost` scan, but the dashboard/session path can still undercount when an active `<id>.jsonl` and sibling `<id>.jsonl.reset.*` archive coexist. `sessions.usage` dedupes to one file per session and `loadSessionCostSummary()` scans only that resolved file.

## Best Possible Solution

Keep this open and fix the remaining `sessions.usage`/dashboard path. Add a helper that resolves all usage-counted transcript segments for a session id, aggregate active `.jsonl` plus `.jsonl.reset.*` and `.jsonl.deleted.*` in `loadSessionCostSummary()` and `sessions.usage` totals, preserve the exclusion of `.jsonl.bak.*`, checkpoint, trajectory, and store backup artifacts, and add a same-session active-plus-reset regression.
