## Summary

Keep open. Current main only partially fixes archive accounting: the global daily cost scan is archive-aware, but the per-session usage path still resolves one transcript and prefers the active `<sessionId>.jsonl` when it exists, so active-plus-reset sessions can still be undercounted in dashboard totals, filters, and detail views.

## What This Changes

The issue reports that usage/cost dashboards and related summaries omit archived `.jsonl.reset.*` session transcripts after `/new` or `/reset`, causing daily spend to be understated.

## Best Possible Solution

Keep the archive-aware global scan, then add shared per-session transcript-segment discovery that returns every usage-counted segment for a session id: active `.jsonl` plus `.jsonl.reset.*` and `.jsonl.deleted.*`. Use that segment list for per-session summaries, timeseries, and logs so `sessions.usage`, dashboard filtering, exports, and detail views agree without counting `.bak` or checkpoint artifacts.

## Reproduction Assessment

Yes. The issue gives concrete active-plus-reset files and costs, and current source still reproduces the path statically because `loadSessionCostSummary({ sessionId })` resolves one active transcript and scans only that file.

## Solution Assessment

Yes for a narrow repair PR; current main is not sufficient. The maintainable fix belongs in shared infra session transcript resolution/aggregation rather than a UI-only adjustment, because gateway RPC totals and dashboard detail endpoints depend on the same summaries.
