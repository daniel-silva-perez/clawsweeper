## Summary

Current main still partially reproduces the reported undercount. The global `usage.cost` daily scan includes `.jsonl.reset.*` and `.jsonl.deleted.*` files, but the dashboard/session path still flows through `sessions.usage`, which dedupes to one transcript per session and then calls `loadSessionCostSummary()` on only that one file. If both `sess.jsonl` and `sess.jsonl.reset.<timestamp>` exist, the active file is preferred and the reset archive remains omitted from per-session and unfiltered dashboard totals.

## Best Possible Solution

Keep this issue open and fix the remaining `sessions.usage`/dashboard path. Per-session and unfiltered dashboard totals should aggregate all usage-counted transcript segments for the same session id: active `.jsonl` plus `.jsonl.reset.*` and `.jsonl.deleted.*`, while excluding `.jsonl.bak.*`, compaction checkpoints, trajectory artifacts, and store backups. Add a regression where one session has both a primary transcript and a reset archive and assert `sessions.usage` totals include both.
