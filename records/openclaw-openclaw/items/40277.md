## Summary

Keep open. Current main still does not implement this PR's intended UI behavior: live chat error events are stored only in `lastError`, persisted assistant error-only messages still have no visible text fallback, and grouped chat rendering has no distinct model-error bubble styling. The shipped #70848 runner fix is real but narrower: it propagates one class of provider failures from the embedded runner, not this PR's client-side inline rendering and history fallback work.

## Best Possible Solution

Keep this PR open for maintainer review or fold its focused UI pieces into the canonical inline-error work. The best implementation should append structured assistant error messages for live `state: "error"` events, render persisted assistant `stopReason: "error"` / `errorMessage` history entries including empty-detail fallbacks, style model failures distinctly in grouped chat, preserve structured connection-error formatting, coordinate with open #55923 and shipped #70848, and add focused controller/extraction/render tests on current main.
