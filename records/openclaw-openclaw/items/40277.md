## Summary

Keep this PR open. Current main still does not implement the PR’s central behavior: live chat error events remain hidden in state, send failures are ordinary text bubbles, persisted assistant error-only messages have no extraction/render fallback, and grouped chat has no distinct model-error label or styling. Related #55923 overlaps the live-error append path, and #69747 improves delivery of lifecycle error events, but neither supersedes this PR’s persisted-history and grouped-rendering work.

## Best Possible Solution

Keep this PR open for maintainer review or fold its focused UI pieces into the canonical inline-error work. The best path is to append structured assistant error messages for live run failures and direct send failures, preserve `formatConnectError` formatting, render persisted assistant `stopReason: "error"` / `errorMessage` history entries including empty-detail fallbacks, add distinct grouped chat error styling/labeling, coordinate with #55923, and keep focused controller/extraction/view regression tests.
