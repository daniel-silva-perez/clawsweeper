## Summary

Keep open. Current main still does not implement this PR's intended inline model-error UI behavior: live `state: "error"` events only update hidden `lastError`, direct send failures append a normal assistant text bubble, persisted assistant error-only history has no `errorMessage` fallback, and grouped chat rendering has no model-error class or label. The shipped #70848 runner fix is real but narrower than this UI/history work, and open #55923 overlaps only part of the live error-event path.

## Best Possible Solution

Keep this PR open for maintainer review or fold its focused UI pieces into the canonical inline-error work. The best implementation should append structured assistant error messages for live `state: "error"` events and direct send failures, render persisted assistant `stopReason: "error"` / `errorMessage` history entries including empty-detail fallbacks, style model failures distinctly in grouped chat, preserve current attachment/tool rendering behavior, coordinate with open #55923 and shipped #70848, and add focused controller/extraction/render regression tests.
