## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: live `state: "error"` chat events only set hidden controller state, send failures are appended as ordinary assistant text, persisted assistant error-only history has no extraction fallback, and grouped chat has no distinct model-error label or styling. The merged gateway lifecycle fix #69747 makes error events reach clients, and open #55923 overlaps the live append path, but neither fully supersedes this PR's persisted-history and grouped-rendering work.

## Best Possible Solution

Keep this PR open for maintainer review or fold its focused UI pieces into the canonical inline-error work. The best implementation path is to append structured assistant error messages for live run failures and send failures, preserve current `formatConnectError` behavior, render persisted assistant `stopReason: "error"` / `errorMessage` history entries including empty-detail fallbacks, add distinct grouped chat error styling/labeling, coordinate with #55923, and keep focused controller/extraction/view regression tests.
