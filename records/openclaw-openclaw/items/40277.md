## Summary

Keep this PR open. Current main still does not implement the PR's central UI behavior: live chat error events set hidden error state only, send failures render as ordinary assistant text, persisted assistant error-only history has no display fallback, and grouped chat has no distinct model-error label or styling. The shipped runner-side failover fix helps errors reach the client, and open #55923 overlaps the live-error path, but neither supersedes the full UI/history/rendering work here.

## Best Possible Solution

Keep this PR open for maintainer review or fold its focused UI pieces into the canonical inline-error work. The best path is to append structured assistant error messages for live run failures and direct send failures, render persisted assistant `stopReason: "error"` / `errorMessage` history entries including empty-detail fallbacks, add distinct grouped chat error styling/labeling, preserve sanitized connect-error formatting, coordinate with #55923 and the shipped runner-side fix, and keep focused controller/extraction/render regression tests.
