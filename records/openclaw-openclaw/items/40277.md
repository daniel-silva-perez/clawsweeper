## Summary

Keep open. Current main still lacks this PR's intended client-side inline model-error behavior: live `state: "error"` events only set hidden error state, persisted assistant error-only records have no visible text fallback, and grouped chat rendering has no distinct model-error bubble. The shipped #70848 runner fix is useful but narrower than this PR.

## Best Possible Solution

Keep this PR open for maintainer review or fold its focused pieces into the canonical inline-error work. The best implementation should append structured assistant error messages for live `state: "error"` events, render persisted assistant `stopReason: "error"` / `errorMessage` history entries including empty-detail fallbacks, style model failures distinctly in grouped chat, preserve current structured connect-error formatting, coordinate with #55923 and the shipped #70848 runner fix, and add focused controller/extraction/render tests on current main.
