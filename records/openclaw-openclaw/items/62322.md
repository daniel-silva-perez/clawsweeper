## Summary

Keep open. Current main still has the reported recovery-state chain: `incomplete-text` recovery returns `prefill: true`, the next reply injects and clears an abort hint from `abortedLastRun`, and session persistence overwrites the abort marker from each run result. The same reporter also has open PR #62346 for the narrow incomplete-text mitigation, so this issue should stay open while that PR is reviewed rather than being closed or routed to a separate repair job.

## What This Changes

The issue requests stabilizing aborted embedded-run recovery so partial assistant text is not fed into the next turn with only a transient abort marker.

## Best Possible Solution

Review and land PR #62346, or replace it with an equivalent maintainer patch, so `incomplete-text` recovery drops the incomplete assistant message and disables prefill with focused regression coverage. If instability remains after that ships, track the abort-state persistence redesign separately instead of mixing it into the narrow recovery fix.

## Reproduction Assessment

Yes for the source-level recovery chain: current main still returns `prefill: true` for `incomplete-text`, injects a next-turn abort hint, and then clears/overwrites the abort marker.
