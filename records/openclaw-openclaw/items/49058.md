## Summary

Keep this PR open. Current main already suppresses Mattermost DM thread-session splitting, but it still does not implement the PR's remaining intended behavior: structured replied-post context, direct-only reply-reference fetching, explicit DM reply-target preservation, root typing parent_id compatibility, direct-only typing TTL override, or websocket silence heartbeat handling. The PR remains a real Mattermost bugfix candidate, with review concerns that should be resolved before merge.

## Best Possible Solution

Keep this PR open as a Mattermost plugin bugfix candidate. The best path is to update or review it so Mattermost DMs keep one OpenClaw session while preserving structured ReplyToBody/ReplyToSender context, fetching replied posts only for direct messages, preserving explicit outbound reply targets where intended, and avoiding raw body mutation or attachment downloads. The websocket heartbeat and typing changes should either be split out or landed with focused tests and live Mattermost proof.
