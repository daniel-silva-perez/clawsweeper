## Summary

Keep this PR open. Current main already keeps Mattermost direct messages out of thread-scoped sessions, but it does not implement the PR's remaining observable behavior: structured DM reply context, replied-post fetching, explicit root typing parent_id behavior, direct-only typing TTL behavior, or websocket silence heartbeat handling. The PR discussion also shows review feedback was addressed through the current PR head, so this remains a real Mattermost bugfix candidate rather than conservative cleanup material.

## Best Possible Solution

Keep this PR open as a Mattermost plugin bugfix candidate. The best path is to review or update it so Mattermost DM replies keep a single direct session while preserving structured `ReplyToBody`/`ReplyToSender` context, preserving explicit outbound reply targets where intended, gating replied-post fetches to DMs, avoiding raw body mutation and attachment downloads, and either splitting or independently validating the websocket heartbeat and typing changes.
