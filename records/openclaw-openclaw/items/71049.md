## Summary

Keep open. Current main still creates the MS Teams personal-chat streaminfo/informative-status path unconditionally, the strict MS Teams config schema has no server-side DM stream-status opt-out, and the related Microsoft tracker still lists this issue as remaining work.

## What This Changes

The issue requests an MS Teams plugin config option to disable Copilot-style informative stream status in 1:1 DMs and use normal typing plus final-message delivery instead.

## Best Possible Solution

Expose a default-preserving MS Teams `channels.msteams` config switch that prevents personal-DM streaminfo/informative-status setup, omits the partial-reply stream callback in that mode, keeps native typing controlled by `typingIndicator`, and updates schema, generated config docs, user docs, changelog, and focused tests.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: create a personal MS Teams dispatcher with `typingIndicator: false`; current main still sends an informative update and still installs the stream partial-reply callback.

## Solution Assessment

Yes. A plugin-owned default-on config option is the narrowest maintainable fix because this is MS Teams-specific behavior already implemented in the MS Teams dispatcher, stream controller, config schema, docs, and tests.
