## Summary

Keep this open. Current main still hardwires the MS Teams personal-chat streaminfo/informative status path and has no accepted server-side config key to disable it for DMs; the related Microsoft tracker only lists the request as remaining work, not an implementation.

## What This Changes

The issue requests a server-side MS Teams configuration option to disable Copilot-style informative stream status in 1:1 DMs and fall back to native typing plus one final message.

## Best Possible Solution

Add a default-preserving MS Teams plugin config option that disables the personal-DM streaminfo/informative status transport, leaves existing behavior enabled by default, and documents the native typing plus final-message fallback path.

## Solution Assessment

Yes. A plugin-owned, default-on config option is the narrowest maintainable solution because the behavior is MS Teams-specific and does not require a core channel policy change.
