## Summary

Keep open. Current main still hardwires the MS Teams personal-chat streaminfo/informative-status path, the strict MS Teams config schema exposes no server-side DM opt-out, and the related Microsoft tracker still lists this as remaining work.

## What This Changes

Request a Microsoft Teams plugin config option to disable the Copilot-style informative stream status in personal DMs and fall back to native typing plus final-message delivery.

## Best Possible Solution

Add a default-preserving MS Teams plugin config option that disables personal-DM streaminfo/informative-status delivery, omits `onPartialReply` for that mode, keeps native typing enabled when configured, and documents the final-message fallback without changing core channel policy.

Yes. A source-level reproduction is high confidence: create a personal MS Teams dispatcher with `typingIndicator: false`; current main still calls `sendInformativeUpdate` and the existing test asserts that behavior.

## Solution Assessment

Yes. A plugin-owned, default-on config option is the narrowest maintainable direction because the behavior is MS Teams-specific and the current SDK/config surfaces already cover the affected dispatcher, schema, docs, and tests.
