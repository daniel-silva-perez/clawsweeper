## Repair Work Prompt

Fix openclaw/openclaw#71049 with a narrow MS Teams plugin-owned change. Add a default-preserving config option, preferably `channels.msteams.dmStreamStatus?: boolean` defaulting to `true`, that lets personal DMs disable the Teams streaminfo informative status/partial streaming UI and instead use native typing plus normal final message delivery. Keep the change inside the MS Teams plugin/config surface; do not add core owner-specific policy. When the option is false, personal chats should not construct/use `TeamsHttpStream`, should not attach `replyOptions.onPartialReply` for Teams streaminfo updates, and should still allow the existing native typing keepalive when `typingIndicator` is enabled so long tool chains remain deliverable. Preserve existing defaults. Update `src/config/types.msteams.ts`, `src/config/zod-schema.providers-core.ts`, generated config metadata as required by repo tooling, `extensions/msteams/src/reply-dispatcher.ts`, `extensions/msteams/src/reply-stream-controller.ts` if needed, focused tests, `docs/channels/msteams.md`, and `CHANGELOG.md` with reporter credit. Do not change unrelated Teams auth, media, reply-style, or core streaming behavior.
