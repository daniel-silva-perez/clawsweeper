## Summary

Current main still hardcodes the MS Teams personal-chat streaminfo path. The schema/type surface accepts `typingIndicator`, `blockStreaming`, and `blockStreamingCoalesce`, but no DM stream-status mode; personal conversations still construct `TeamsHttpStream`, send an informative `streaminfo` typing activity on reply start, and expose partial streaming callbacks. Existing tests explicitly assert that `typingIndicator: false` skips only the native typing keepalive while still sending the informative update, so the requested server-side opt-out remains valid.

## Best Possible Solution

Keep this issue open and implement a narrow MS Teams plugin-owned config option, defaulting to current behavior, that lets personal chats disable the informative streaminfo status and use native typing plus normal final delivery. The fix should stay inside the MS Teams plugin/config surface, avoid core owner-specific policy, and include tests for the personal-chat stream controller, typing keepalive, schema acceptance/rejection, docs, and changelog.

## Clownfish Work Prompt

Implement a narrow MS Teams plugin fix for this issue. Current personal-chat behavior always constructs `TeamsHttpStream` and sends a `streaminfo` informative typing update; `typingIndicator: false` only disables the native typing keepalive. Add a plugin-owned config option under `channels.msteams` (prefer a boolean such as `dmStreamStatus?: boolean`, default `true`) so `false` disables the DM informative stream status/partial stream card and uses native Teams typing plus normal final message delivery. Keep existing default behavior unchanged, do not change group/channel behavior, do not move owner-specific policy into core, and preserve `blockStreaming` as the separate progressive block-delivery option. Update zod schema/types, generated config metadata if required, config UI hints if appropriate, `docs/channels/msteams.md`, and `CHANGELOG.md` with `Thanks @software-ec`. Add/adjust tests proving personal chats with the new option disabled do not create `TeamsHttpStream`, do not call `sendInformativeUpdate`, still start native typing callbacks when `typingIndicator` is true, and omit stream `onPartialReply`; preserve current `typingIndicator: false` semantics unless the new option is also disabled.
