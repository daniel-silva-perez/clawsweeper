## Summary

Keep open. Current main still hardwires the MS Teams personal-chat streaminfo/informative-status path, the accepted config schema has no server-side DM opt-out, and the related Microsoft tracker still lists this feature as remaining work rather than an implementation.

## What This Changes

The issue requests a MS Teams plugin config option to disable the Copilot-style informative stream status in 1:1 DMs and use native typing plus final-message delivery instead.

## Best Possible Solution

Add a default-preserving MS Teams plugin config option that disables the personal-DM streaminfo/informative-status transport, keeps existing behavior enabled by default, and documents the native typing plus normal final-message fallback path.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: create a personal MS Teams dispatcher with `typingIndicator: false`, and current tests still expect `sendInformativeUpdate` while suppressing only the native typing keepalive.

## Solution Assessment

Yes. A plugin-owned, default-on config option is the narrowest maintainable solution because the behavior is MS Teams-specific and does not require a core channel policy change.
