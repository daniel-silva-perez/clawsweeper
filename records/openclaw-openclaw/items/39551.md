## Summary

Keep open. Current main at 230f8886c631bceff3755d2236ec0d39ecebd3af still hard-rejects `sessions_send` calls that include both `sessionKey` and `label`, which is exactly the behavior this PR is meant to fix. The latest shipped tag also still contains the guard, and the provided PR timeline shows a maintainer pushed a narrow repair to this branch and marked it merge-ready for human review.

## Best Possible Solution

Review and land this PR, or replace it only with a merged canonical patch that fixes the same behavior. The best implementation is a narrow selector precedence change: treat a non-empty `sessionKey` or accepted `sessionId` as authoritative, ignore redundant `label` and `agentId` for lookup, preserve label-only resolution and downstream visibility/A2A checks, update the tool description/changelog, and keep regression coverage proving mixed inputs use the exact target without calling `sessions.resolve`. After a canonical patch lands, close or supersede overlapping open PRs such as #41255 and #59324.
