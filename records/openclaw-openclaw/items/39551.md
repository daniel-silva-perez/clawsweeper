## Summary

Keep open. Current main still hard-rejects `sessions_send` calls that include both `sessionKey` and `label`, while this PR is an active focused implementation candidate for that exact behavior. The latest release has the same guard, and related PRs are open rather than merged replacements.

## Best Possible Solution

Review and land this PR, or replace it with a maintainer-selected canonical patch for the same behavior. The best fix is narrow: normalize a non-empty `sessionKey`, make it authoritative over redundant `label`/`agentId`, preserve label-only resolution, missing-target behavior, visibility checks, and A2A policy checks, add mixed-selector regression coverage, and decide whether the explicit `sessionId` alias should stay in this patch.
