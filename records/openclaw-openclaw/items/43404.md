## Summary

Keep open. Current main still has the original reminder-guard and compaction-counter gaps, but this PR is not ready to merge because its runner call site still defaults an absent session key to the `main` agent and can suppress the guard for unrelated isolated cron jobs.

## What This Changes

The PR updates reminder-guard cron matching to include enabled isolated cron jobs by agent id, preserves `successfulCronAdds` across compaction retry, and adds helper-level reminder-guard tests.

## Best Possible Solution

Land this PR or a narrow replacement after fixing the call site so agent-id matching is used only when the runner has reliable session identity, while preserving the compaction-counter fix and adding a `runReplyAgent` regression for omitted session keys with isolated cron jobs.

## Reproduction Assessment

Yes. Current main has a direct source-level reproduction for the original bug, and the PR regression is reproducible by invoking `runReplyAgent` without `sessionKey` while the cron store contains an enabled isolated `main` job with no `sessionKey`.

## Solution Assessment

No. The PR is directionally the right narrow fix, but the current call-site implementation is unsafe; the safer solution is to avoid defaulting missing or unreliable session identity to the `main` agent before the isolated-job fallback runs.
