## Summary

Keep open. The PR targets a real reminder-guard bug and current main still lacks both requested fixes, but the PR head still passes a defaulted `main` agent id into the existing-cron guard when the run has no reliable session key, which can suppress the warning for an unrelated run.

## What This Changes

The PR preserves `successfulCronAdds` across compaction retry, extends the reminder guard to match enabled isolated cron jobs by `agentId`, passes a derived agent id from `runReplyAgent`, and adds reminder-guard unit coverage.

## Best Possible Solution

Land this PR or an equivalent focused fix after changing the call site so the isolated-job `agentId` fallback is only used when the run has reliable agent/session identity, while keeping the `sessionTarget: "isolated"` guard and preserving the cron-add counter across compaction retry.
