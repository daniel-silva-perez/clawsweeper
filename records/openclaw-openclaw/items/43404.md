## Summary

Keep open. The PR addresses a real current-main bug, but its call-site change still introduces a reminder-guard regression when `runReplyAgent` is invoked without a reliable `sessionKey`; that should be fixed before merge rather than closing or landing as-is.

## What This Changes

This PR changes reminder-guard matching to include enabled isolated cron jobs by agent id, preserves `successfulCronAdds` across compaction retry, and adds helper-level reminder-guard regression tests.

## Best Possible Solution

Land this PR or a narrow replacement after the call site only passes an agent id when the runner has reliable session identity, while keeping the `sessionTarget: "isolated"` matcher guard, preserving `successfulCronAdds` through compaction retry, and adding a `runReplyAgent` regression for omitted `sessionKey` plus an enabled isolated `main` job.
