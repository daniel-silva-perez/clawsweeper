## Summary

Keep open. Current main still has both bugs this PR targets, so the work is not obsolete; however, the PR head still has a blocking correctness issue because the call site derives a usable `main` agent id when `sessionKey` is absent, which can suppress the reminder guard for unrelated isolated jobs.

## What This Changes

The PR removes the compaction reset of `successfulCronAdds`, extends `hasSessionRelatedCronJobs` to match enabled `sessionTarget: "isolated"` jobs by `agentId`, passes a derived agent id from `runReplyAgent`, and adds reminder-guard unit coverage.

## Best Possible Solution

Land this PR or an equivalent focused fix after updating the call site so the isolated-job `agentId` fallback is used only when the current run has reliable agent/session identity, while preserving the `sessionTarget: "isolated"` guard, keeping `successfulCronAdds` across compaction retry, and adding regression coverage for the missing-session-key isolated-job case plus the compaction counter path.
