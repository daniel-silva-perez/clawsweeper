## Summary

Keep open. The PR targets a real reminder-guard bug that current main still has, and it correctly preserves the cron-add counter plus restricts helper-level agent matching to isolated jobs. The latest patch still has a call-site regression: it always derives `agentId` from `sessionKey`, and the resolver defaults missing or malformed keys to `main`, so a run without reliable session identity can be treated as covered by an unrelated isolated main-agent cron job.

## What This Changes

The PR preserves `successfulCronAdds` across compaction retries, extends the reminder guard to match enabled isolated cron jobs by `agentId`, passes a derived agent id from `runReplyAgent`, and adds reminder-guard unit coverage.

## Best Possible Solution

Land this PR or an equivalent focused fix after making the agent-id fallback depend on reliable session identity, while keeping the isolated-job `sessionTarget` guard, preserving `successfulCronAdds` across compaction retry, and adding call-site regression coverage for the omitted-session case.
