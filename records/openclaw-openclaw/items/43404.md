## Summary

Keep open. The PR addresses a real current-main reminder-guard bug, but the proposed call-site still defaults a missing `sessionKey` to the `main` agent, which can suppress the guard note for unrelated runs when any enabled isolated main-agent cron exists.

## What This Changes

This PR updates reminder-guard cron matching to include enabled isolated jobs by agent id, preserves `successfulCronAdds` across compaction retry, and adds helper-level reminder-guard tests.

## Best Possible Solution

Land this PR or a narrow replacement after the call site only passes an agent id when the runner has reliable session identity, while keeping the `sessionTarget: "isolated"` matcher guard, preserving `successfulCronAdds` through compaction retry, and adding a `runReplyAgent` regression for omitted `sessionKey` plus an enabled isolated `main` job.
