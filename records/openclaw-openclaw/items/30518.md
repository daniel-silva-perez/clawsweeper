## Summary

Current main still does not expose manual Telegram `pin` or `unpin` message actions. Telegram has low-level pin/unpin helpers and send-time delivery pinning, but the public action map, discovery contract, runtime aliases/dispatch, config gates, tests, and Telegram docs still omit manual pin/unpin for existing messages.

## Best Possible Solution

Keep this issue open and implement manual Telegram `pin` and `unpin` in the bundled Telegram plugin. Wire the public action map and discovery, add runtime aliases and dispatch branches that call the existing helpers, decide whether to add explicit `channels.telegram.actions.pinMessage` / `unpinMessage` gates, update Telegram and CLI docs, and cover adapter discovery, runtime dispatch, helper behavior, and docs expectations with focused tests.
