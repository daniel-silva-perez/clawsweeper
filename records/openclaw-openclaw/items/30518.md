## Summary

Keep this issue open. Current main still has Telegram send-time delivery pinning and low-level Telegram pin/unpin helpers, but the bundled Telegram message action surface still does not advertise, map, or dispatch manual `pin`/`unpin` actions for existing messages.

## Best Possible Solution

Implement this in the bundled Telegram plugin: add `pin` and `unpin` to Telegram message action discovery and `TELEGRAM_MESSAGE_ACTION_MAP`, import/expose `unpinMessageTelegram` in the runtime, add aliases and runtime branches that call `pinMessageTelegram` / `unpinMessageTelegram`, decide whether to add a `channels.telegram.actions.pins` gate in the config type/schema/docs, add focused adapter/runtime tests, and update Telegram/CLI docs to distinguish manual pin/unpin from send-time delivery pinning.
