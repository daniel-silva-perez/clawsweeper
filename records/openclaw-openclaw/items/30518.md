## Summary

Keep this issue open. Current main still has shared `pin`/`unpin` action names and low-level Telegram pin/unpin helpers, but the bundled Telegram plugin does not advertise, map, gate, dispatch, test, or document manual pin/unpin actions for existing messages. Prior Telegram pin/unpin PRs were closed unmerged, so they do not supersede the remaining work.

## Best Possible Solution

Implement manual Telegram `pin` and `unpin` in the bundled Telegram plugin: wire discovery/action mapping, runtime aliases and dispatch to the existing helpers, add explicit config gating if maintainers want it, keep send-time `delivery.pin` behavior distinct, update Telegram and message CLI docs, add focused tests, and include a user-facing changelog entry crediting the reporter.

## Repair Work Prompt

Implement manual Telegram message `pin` and `unpin` actions for this issue in the bundled Telegram plugin on current main. Keep the fix plugin-owned under `extensions/telegram`; do not add `list-pins`, do not move Telegram behavior into core, and keep existing send-time `--pin` / `delivery.pin` behavior distinct from pinning an existing message.

Expected implementation boundary:
- In `extensions/telegram/src/channel-actions.ts`, advertise `pin` and `unpin` in `describeTelegramMessageTool` when enabled, and map those public actions in `TELEGRAM_MESSAGE_ACTION_MAP` to runtime action names.
- In `extensions/telegram/src/action-runtime.ts`, add aliases and dispatch branches for manual pin/unpin that read `target`/`to`/`chatId` and `messageId`, call `pinMessageTelegram` / `unpinMessageTelegram`, and return structured JSON results. Preserve clear errors for missing Telegram token, missing target, missing message id, and Telegram permission/API failures.
- Decide and implement config gating consistently. Prefer explicit `channels.telegram.actions.pinMessage` / `unpinMessage` or another maintainer-approved shape; update `src/config/types.telegram.ts`, zod schema/metadata/docs if config keys are added.
- Reuse existing helpers in `extensions/telegram/src/send.ts`; only change helper behavior if needed for manual action semantics.
- Update `docs/cli/message.md` and `docs/channels/telegram.md` to document Telegram manual pin/unpin separately from send-time `--pin`.
- Add/adjust focused tests for action discovery, runtime dispatch, helper behavior if changed, config gates, and docs expectations. Add a changelog entry under the active version because this is a user-facing feature, crediting `Thanks @dan-osss` and any implementation author per repo policy.

Related refs: this issue, #12950, #30545, #31243, #17196. Prior PRs are useful context but were closed unmerged and may reference old paths.
