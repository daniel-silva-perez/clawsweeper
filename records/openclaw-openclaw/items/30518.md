## Summary

Current main still does not expose manual Telegram `pin`/`unpin` message actions. The shared action names and low-level Telegram `pinMessageTelegram`/`unpinMessageTelegram` helpers exist, and Telegram send-time delivery pinning works, but the bundled Telegram action discovery, action map, runtime aliases, and runtime dispatch branches still omit manual `pin` and `unpin`.

## Best Possible Solution

Implement this in the bundled Telegram plugin: advertise `pin` and `unpin`, map them to Telegram runtime actions, add runtime aliases and dispatch branches that call `pinMessageTelegram` / `unpinMessageTelegram`, add appropriate action gates/config schema/docs, and cover discovery, runtime, helper, and CLI/doc behavior with focused tests.

## Clownfish Work Prompt

Implement manual Telegram message `pin` and `unpin` actions for openclaw/openclaw#30518. Keep the change scoped to the bundled Telegram plugin and directly related config/docs/tests.

Expected behavior:
- `openclaw message pin --channel telegram --target <chat> --message-id <id>` and the agent message tool action `{ action: "pin", ... }` should pin an existing Telegram message by calling `pinMessageTelegram`.
- `openclaw message unpin --channel telegram --target <chat> --message-id <id>` and the agent message tool action `{ action: "unpin", ... }` should unpin an existing Telegram message by calling `unpinMessageTelegram`.
- Preserve existing send-time `--pin` / `delivery.pin` behavior for `send`; do not conflate it with manual pin/unpin.
- Do not implement `list-pins` unless it is already required by a failing test; Telegram Bot API does not provide the same cross-channel list-pins surface.

Implementation boundary:
- Add `pin` and `unpin` to Telegram message action discovery and `TELEGRAM_MESSAGE_ACTION_MAP` in `extensions/telegram/src/channel-actions.ts`.
- Import/expose `unpinMessageTelegram` in `extensions/telegram/src/action-runtime.ts`, add runtime aliases such as `pin`, `pinMessage`, `unpin`, `unpinMessage`, and add action branches that parse `chatId`/`to` plus `messageId` and return explicit JSON success.
- Add Telegram action gates such as `pinMessage?: boolean` and `unpinMessage?: boolean` if maintainers want parity with delete/edit gates; if added, update `src/config/types.telegram.ts`, `src/config/zod-schema.providers-core.ts`, generated config metadata/checks as required, and Telegram docs.
- Keep owner-specific behavior in `extensions/telegram`; avoid core changes unless a shared contract test exposes an existing generic gap.
- Update docs to list Telegram manual pin/unpin actions and explicitly distinguish them from send-time delivery pinning.
- Changelog: add a user-facing Fixes/Changes entry only if repo convention requires one for this enhancement, with the issue reporter credited as `Thanks @dan-osss`.

Do not change unrelated Telegram send/media behavior, plugin loading, or shared message action names.
