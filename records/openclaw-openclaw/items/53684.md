## Summary

Keep open. Current main still has only adjacent gateway status/probe, service lifecycle, LaunchAgent KeepAlive, stability diagnostics, channel health restarts, and manual rescue-bot guidance; it does not implement the requested gateway-down watchdog, visible escalation/notification path, startup-error artifact, dead-man monitoring, or first-class sibling health monitoring. The issue is also paired with open implementation PR #53716, so it should remain open while that PR or a successor is reviewed. https://github.com/openclaw/openclaw/issues/53684

## What This Changes

The issue requests gateway failure recovery that detects startup/runtime outages, attempts recovery, surfaces diagnostics and notifications, supports fallback alerts, and enables multi-instance health monitoring.

## Best Possible Solution

Land a maintainer-scoped implementation that combines or deliberately splits the watchdog, startup diagnostics, notification escalation, and multi-instance monitoring pieces, with tests and docs tied to the existing gateway service, stability bundle, channel health, and multiple-gateway surfaces.
