## Summary

Keep this issue open. Current main has adjacent gateway health, restart-health checks, LaunchAgent KeepAlive, channel health monitoring, stability bundles, diagnostics export, and rescue-bot setup docs, but it still lacks the requested external gateway-down watchdog, user notification/escalation path, `startup-error.txt`-style artifact, dead-man switch, and first-class sibling health monitoring. The provided GitHub context also shows open PR #53716 with explicit closing syntax, so policy says this issue should remain open until that PR merges or is closed.

## Best Possible Solution

Keep this issue open while PR #53716 remains open and unmerged. Maintainers should review that PR or a successor against current gateway status/probe/stability, service lifecycle, LaunchAgent/systemd behavior, channel health monitoring, and multiple-gateway docs, then land a focused watchdog/startup-diagnostics/notification implementation with tests and docs or close the PR and keep this as the canonical feature request.
