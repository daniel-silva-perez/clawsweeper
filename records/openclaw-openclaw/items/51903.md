## Summary

Keep open. Current main still matches the report: the `nodes` schema exposes `node` as optional, but the `describe` branch reads it with `required: true` before calling `node.describe`, so `{ "action": "describe" }` still fails locally with `node required`. The provided GitHub context also shows same-author open PR #51926 with `Closes #51903`, so the right maintainer path is to review/land or close that PR rather than auto-close this issue first.

## Best Possible Solution

Review #51926 as the implementation candidate. A good fix should update `src/agents/tools/nodes-tool.ts` so `action: "describe"` no longer hard-requires `node` and either uses the existing default-node resolver intentionally or returns discovery data when no node is provided. Add focused coverage in `src/agents/tools/nodes-tool.test.ts` for `{ action: "describe" }` without `node`, while keeping explicit-node requirements for mutating/targeted actions such as `notify` and `invoke`.
