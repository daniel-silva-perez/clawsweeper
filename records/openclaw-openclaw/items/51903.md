## Summary

Keep open. Current main still has the schema/implementation mismatch: the `nodes` tool advertises `node` as optional, but `describe` still requires it before default-node resolution can run. PR #51926 remains open and says it closes this issue, so the issue should stay open until that PR is reviewed and landed, replaced, or closed.

## Best Possible Solution

Keep this issue open while PR #51926 remains open. The best path is to review and land that PR or replace it with a maintained fix in `src/agents/tools/nodes-tool.ts` that makes `describe` intentionally accept omitted `node` through default-node resolution or discovery output, plus focused regression coverage in `src/agents/tools/nodes-tool.test.ts`; targeted or mutating actions should keep explicit node requirements.
