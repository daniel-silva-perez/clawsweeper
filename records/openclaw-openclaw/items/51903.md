## Summary

Keep open. Current main still matches the report: the flattened `nodes` tool schema exposes `node` as optional, but the `describe` branch still reads it as required before calling `node.describe`, so `{ "action": "describe" }` still takes the `node required` path. The live GitHub data also shows same-author PR #51926 is open and says it closes this issue, so the right cleanup path is to review/land or close that implementation candidate first.

## Best Possible Solution

Keep this issue open while PR #51926 is open. The best path is to review that PR or replace it with a small maintained fix in `src/agents/tools/nodes-tool.ts` that makes `describe` intentionally accept omitted `node` through either default-node resolution or discovery output, plus a focused regression test in `src/agents/tools/nodes-tool.test.ts` for `{ action: "describe" }` without `node`. Preserve explicit `node` requirements for targeted or mutating actions such as `notify`, `notifications_action`, and `invoke`.
