## Summary

The core behavior still matches the reported bug on current `main`: `describe` remains schema-valid without `node` but fails before default-node resolution. The issue is also paired with same-author open PR #51926 using closing syntax, so this should stay open until that PR is reviewed, landed, or replaced.

## What This Changes

The issue requests that the agents `nodes` tool resolve the schema/runtime mismatch where `action: "describe"` allows an omitted `node` in the schema but rejects it at execution time.

## Best Possible Solution

Review and land PR #51926, or replace it with a maintained patch in `src/agents/tools/nodes-tool.ts` that intentionally accepts omitted `node` for the read-only `describe` action while preserving explicit-node requirements for targeted actions, plus focused regression coverage in `src/agents/tools/nodes-tool.test.ts`.
