## Summary

Keep this issue open. Current main still has the reported builtin memory-core gap: session files are reconciled only when a sync is allowed to run, but clean `session-start`/`watch` paths still skip session syncing unless there is a force, targeted file list, full reindex, or in-process dirty session event.

## Best Possible Solution

Keep the issue open and fix it in `extensions/memory-core`: add a startup or first-sync session reconciliation pass for the builtin manager that enumerates `listSessionFilesForAgent()`, compares current files against stored `sessions` file state, marks changed/missing rows dirty or directly syncs them, and add regression coverage for clean gateway restart plus missed transcript-event scenarios. The CLI/status path should also expose disk-vs-index drift clearly enough that `Dirty: no` cannot imply a verified fresh session index.

There is no session-file enumeration or comparison against existing `sessions` rows during startup.
