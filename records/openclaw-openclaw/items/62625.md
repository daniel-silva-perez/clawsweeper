## Summary

Keep open. Current main and the latest stable tag still have the reported memory-core session reconciliation gap: clean `session-start`/`watch` paths skip session syncing unless a full reindex, forced/targeted sync, or in-process dirty event is already present, so #39732 remains only a partial fix for this report.

## What This Changes

The issue asks for builtin session transcript memory indexing to catch up missed or restart-lost session JSONL changes without requiring a forced manual reindex.

## Best Possible Solution

Add a bounded memory-core session reconciliation path that enumerates `listSessionFilesForAgent()`, compares current files with stored `sessions` source state, marks changed/missing files dirty or directly syncs them, and exposes disk-vs-index drift clearly in CLI/status so `Dirty: no` does not imply verified freshness.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: after a clean gateway restart or missed transcript event, `sessionsDirty=false` and `sessionsDirtyFiles` is empty; `warmSession()` triggers `reason: "session-start"`, the gate returns false, and `syncSessionFiles()` is never called. The reporter also supplied concrete production repros on 2026.3.28 and 2026.4.12.

## Solution Assessment

Yes. A bounded reconciliation scan in the memory-core owner module is the narrow maintainable fix; simply removing the `session-start`/`watch` exclusion would be riskier because it could rescan all session transcripts on hot paths rather than only changed or missing files.
