## What This Changes

The issue reports that `openclaw-tui` consumes roughly 89-99% CPU while idle or during turns, making `openclaw chat` / `openclaw tui --local` laggy on Linux ARM64 with OpenClaw 2026.4.26.

## Best Possible Solution

Keep the issue open, gather `perf top`/V8 profile or `strace` evidence from an affected idle TUI, then fix the smallest proven loop in `src/tui` or upstream `pi-tui`. If profiling shows the Gateway/runtime-deps process instead, move that evidence to #73532 rather than closing this TUI-process report prematurely.
