## Summary

Keep this PR open. Current main still does not validate explicit OpenAI-compatible gateway agent selectors against configured agents, while this PR remains a focused implementation candidate for the bug it describes. A security pass on the provided PR file list did not show workflow, dependency, lockfile, publishing, secret, or artifact-execution changes; the remaining risks are functional review items around explicit `main` compatibility and the `knownAgentIds` contract.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The fix should live in the gateway HTTP helpers/handlers: validate explicit header and model agent selectors against configured agents for both `/v1/chat/completions` and `/v1/responses`, return a stable `400 invalid_request_error` for unknown explicit selectors, preserve implicit default routing plus explicit `main`/`openclaw/default` compatibility, clarify the `knownAgentIds` contract, and retain regression tests for header and model-selector rejection.
