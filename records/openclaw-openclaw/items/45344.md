## Summary

Keep this PR open. Current main still accepts explicit OpenAI-compatible gateway agent selectors without checking them against configured agents, and neither the chat-completions nor Responses HTTP paths maps unknown agent IDs to a 400 response. The PR remains a plausible security/behavior fix, but it needs revision for current main and the review comments about explicit default/main compatibility and helper contract semantics.

## Best Possible Solution

Keep this PR open for maintainer review or contributor revision. The best path is to rebase onto current main, validate explicit `x-openclaw-agent-id`, `x-openclaw-agent`, `openclaw/<agentId>`, `openclaw:<agentId>`, and `agent:<agentId>` selections against `listAgentIds(getRuntimeConfig())` plus the resolved default agent, return `400 invalid_request_error` for unknown explicit selectors on both `/v1/chat/completions` and `/v1/responses`, preserve implicit default routing and explicit `main` / `openclaw/default` compatibility, document or tighten the helper contract, and add regression tests for known, unknown, default, and compatibility selectors.
