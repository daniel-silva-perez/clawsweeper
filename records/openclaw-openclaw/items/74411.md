## Summary

Keep open. Current main has the managed browser download routes and client helpers, but the agent-facing browser tool still does not expose `download` or `waitfordownload`, so this PR addresses a real remaining gap rather than obsolete work. Because it expands the agent tool surface to saved browser downloads, it needs normal maintainer security/CI review instead of cleanup closure.

## What This Changes

The PR adds managed browser `download` and `waitfordownload` actions to the agent-facing browser tool, exports existing runtime helpers, documents the actions, and adds routing/timeout regression tests.

## Best Possible Solution

Review and land this PR, or a narrow replacement, if CI confirms the new agent actions route only through the existing managed download API, preserve guarded download-path handling, keep existing-session profiles unsupported, and document the agent action surface alongside the CLI/control API.
