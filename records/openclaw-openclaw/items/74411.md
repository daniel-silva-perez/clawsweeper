## Summary

Keep open. Current main still has the agent-facing browser-tool gap, and this PR appears to be a narrow, relevant implementation rather than obsolete cleanup; the remaining work is maintainer review plus CI/security validation for the new agent download capability.

## What This Changes

The PR adds browser `download` and `waitfordownload` actions to the agent-facing browser tool, exports existing download helpers, adjusts download request/proxy timeouts, documents the action surface, and adds browser tool/client routing tests.

## Best Possible Solution

Land this PR, or a narrow replacement, after maintainer approval and CI confirm that the new agent actions only route through the existing managed download API, preserve guarded download-path handling, keep existing-session profiles unsupported, and document the agent action surface beside the CLI/control API.

## Solution Assessment
