## Summary

Keep this PR open. Current main still does not implement the PR's central separation between Mattermost session routing and delivery routing for `replyToMode: "off"`: threaded inbound channel/group messages still lose their thread root before `MessageThreadId`, typing, draft preview, and final delivery. The same-author dependency #57970 is also still unshipped in current main, and the author has asked for maintainer takeover rather than withdrawing the work.

## Best Possible Solution

Keep this PR open as useful unshipped Mattermost routing work. The best path is for a maintainer to take over or fold it into a fresh branch: first resolve the unknown-chat-type dependency, then rebase the delivery-thread-id change onto current main while preserving the landed DM non-threading fix from #72659 and adding focused tests for channel/group threaded replies, typing indicators, `MessageThreadId`, tool context, and reset classification.
