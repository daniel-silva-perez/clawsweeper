## Summary

Keep this issue open. Current main supports global `agents.defaults.thinkingDefault`, but the body also asks for global `agents.defaults.reasoningDefault`; that key is still absent from the global defaults type/schema and is not consumed by reply resolution. The open #39115 PR remains the focused implementation candidate for the remaining global reasoning-default work, while #71817 is only related per-agent Telegram/Feishu handling.

## Best Possible Solution

Keep this issue open until maintainers land a complete global `agents.defaults.reasoningDefault` implementation, likely by reviewing and landing #39115, or explicitly decide that reasoning visibility should remain per-agent/session-only. A complete fix should update the defaults type, Zod schema, generated schema/help metadata, docs, reply resolution, status/current-level helpers, and regression tests for directive, session, per-agent, global, and fallback precedence. Treat #71817 as useful related per-agent channel work, not the complete fix for this issue.
