## Summary

Keep this issue open. Current main implements global `agents.defaults.thinkingDefault`, but the issue body also requests global `agents.defaults.reasoningDefault`, and that global reasoning default is still not a schema-backed or runtime-consumed config key. The open #39115 PR remains the best implementation candidate for the remaining work; #71817 is related but only covers per-agent Telegram/Feishu handling.

## Best Possible Solution

Keep this issue open until maintainers either land a complete global `agents.defaults.reasoningDefault` implementation, likely by reviewing and landing #39115, or explicitly decide that reasoning visibility should remain per-agent/session-only. A complete fix should update the defaults type, Zod schema, generated schema/help metadata, docs, reply resolution, status/current-level helpers, and regression tests for directive, session, per-agent, global, and fallback precedence. Treat #71817 as useful related per-agent channel work, not as the complete fix for this issue.
