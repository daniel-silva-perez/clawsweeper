## Summary

Keep open. Current main still has several of the sub-agent documentation gaps this PR targets, and the linked source issue #47558 remains open. The PR should be updated before merge because main has moved: `streamTo` is now documented as ACP-only, and the current announce timeout default is 120000 ms rather than the 90000 ms stated in the PR body.

## Best Possible Solution

Keep this PR open as still-relevant docs cleanup for #47558, but require a rebase/update against current main: retain the ACP-only `streamTo` wording, document `announceTimeoutMs` with the current 120000 ms default, separate global and per-channel thread-binding keys, make depth/announce/auto-archive/tool-policy prose depth-N through `maxSpawnDepth` 5, and align the tool-policy section with `src/agents/pi-tools.policy.ts`. If #56157 becomes the canonical updated patch, close this PR then as superseded with that link.
