## Summary

Keep this PR open. Current main has a generic `compat.requiresStringContent` implementation for strict Chat Completions backends, but it is opt-in and the bundled NVIDIA provider/docs do not opt NVIDIA models into it. That means current main does not definitely implement this PR's intended default NVIDIA/OpenAI-compatible normalization behavior.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to reconcile it with the shipped `compat.requiresStringContent` design: either add the compat flag to affected NVIDIA/strict provider catalog entries with NVIDIA docs and regression tests for the linked bug, or revise the wrapper to preserve current newline joining and opt-in semantics if maintainers want a generic core path. The linked issue should stay open until that focused fix lands.
