## Summary

Keep this PR open. Current main still builds OpenAI Codex Responses requests with `input: messages` plus top-level `instructions`, and a system-prompt-only context can still produce `input: []` with no local fail-fast guard. This PR remains a narrow implementation candidate for #73820, but maintainers should decide whether fail-fast is the intended behavior or whether the caller/transport should preserve the active-memory bootstrap by providing a valid input item.

## Best Possible Solution

Review this PR as the active implementation candidate for #73820. If maintainers accept fail-fast as the intended behavior, land the narrow guard and regression tests. If the desired behavior is to preserve system-only bootstrap runs with Codex OAuth, revise the patch to provide a valid Codex input item or fix the caller so the transport never sees an empty user-input list for a legitimate turn.
