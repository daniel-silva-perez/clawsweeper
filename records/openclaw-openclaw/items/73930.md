## Summary

Keep this PR open. Current main still constructs `openai-codex-responses` payloads as top-level `instructions` plus `input: messages` and has no local empty-message guard, so the PR's intended change is not implemented. The patch is narrow and security-neutral, but maintainer review should decide whether fail-fast is the desired product behavior versus synthesizing a valid input item to preserve the active-memory bootstrap case reported in #73820.

## Best Possible Solution

Review this PR as the current implementation candidate for #73820. If maintainers want fail-fast behavior, land the narrow guard, regression tests, and changelog entry after normal CI. If the desired behavior is to keep bootstrap-style runs working, revise the patch to create a valid Codex input item without moving the system prompt out of top-level `instructions`, and keep the linked issue open until that implementation merges.
