## Summary

Keep this issue open. Current main still has ACPX permission policy only at the ACPX plugin config layer, while the requested per-binding and per-agent runtime overrides are absent from the public config types, strict schemas, generated config help, configured-binding materialization, persisted binding metadata, ACP runtime ensure contract, and ACPX runtime wiring. The prior ClawSweeper comment correctly kept this open; related #63859 documents the current global plugin-config workaround and explicitly leaves per-agent scoping to this issue.

## Best Possible Solution

Keep this issue open and implement scoped ACPX permission policy as an additive ACP runtime/config contract: add `permissionMode` and `nonInteractivePermissions` to `agents.list[].runtime.acp` and `bindings[].acp` types, strict schemas, config help/labels/generated docs, configured binding specs/metadata, ACP runtime ensure input, and ACPX resolution. The precedence should be binding override, then agent runtime default, then global `plugins.entries.acpx.config`, with tests for config validation, binding materialization and restart rehydration, direct ACP session initialization, and ACPX runtime wiring.
