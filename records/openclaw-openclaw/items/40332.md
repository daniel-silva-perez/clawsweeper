## Summary

Keep open. Current main still only supports ACPX `permissionMode` and `nonInteractivePermissions` as global plugin config; the agent runtime schema, binding schema, configured binding materialization, persistent binding metadata, and ACP runtime startup contract do not carry the requested scoped permission policy.

## What This Changes

Request scoped ACP permission policy so `bindings[].acp` and `agents.list[].runtime.acp` can override the global ACPX plugin permission defaults.

## Best Possible Solution

Add an explicit scoped ACP permission contract with typed strict fields on `agents.list[].runtime.acp` and `bindings[].acp`, persist and rehydrate those fields for configured bindings, pass them through ACP manager/runtime initialization, resolve ACPX policy as binding > agent runtime > global plugin config, and update schema/help/docs plus regression tests.
