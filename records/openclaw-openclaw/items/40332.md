## Summary

Keep open. Current main still models ACPX `permissionMode` and `nonInteractivePermissions` as global plugin configuration only; the typed agent runtime config, strict ACP binding schema, configured-binding materialization, persistent binding metadata, ACP runtime startup contract, and user docs do not expose the requested scoped permission policy. The prior ClawSweeper comment remains consistent with current main, and the remaining work is security-sensitive rather than a conservative cleanup close.

## What This Changes

The issue requests scoped ACPX permission settings so binding-level and agent-runtime ACP config can override the global ACPX plugin permissionMode and nonInteractivePermissions defaults.

## Best Possible Solution

Add a typed scoped ACP permission contract on agent runtime and binding config, persist and rehydrate it for configured bindings, pass the effective policy into ACP runtime startup, resolve precedence as binding > agent runtime > global ACPX plugin config, and update schema/help/docs plus regression tests for both `permissionMode` and `nonInteractivePermissions`.
