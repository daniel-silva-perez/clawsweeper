## Summary

Keep open. Current main has meaningful mitigations for parts of this incident, but it does not prove the broader gateway starvation and stale active-work accounting report is fixed: readiness diagnostics still execute on the gateway event loop, plugin prompt hooks can time out without cancelling their underlying work, and the narrower current hot-path stall report remains open.

## What This Changes

The issue requests a core reliability fix for gateway liveness under orchestration load, including responsive `/readyz`, bounded reload deferral, inspectable active-work blockers, and safer plugin/tool hot paths.

Keep this as a core reliability tracker until maintainers split and land narrower fixes for event-loop hot paths, stale active-work/reload accounting, and longer-term control-plane isolation. The already-mitigated `logs --follow` fallback should remain handled through #74583 rather than closing this broader incident.
