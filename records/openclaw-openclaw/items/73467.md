## Summary

Keep open. Current main contains several relevant mitigations for this 2026.4.26 gateway reliability report, including readiness event-loop diagnostics, bounded default restart deferral, Active Memory hook timeout tightening, and `logs --follow` local fallback behavior, but it does not prove the central CPU-hot event-loop starvation is fixed; the narrower hot-path report #74860 is still open and has later profile evidence.

## What This Changes

Reports that OpenClaw 2026.4.26 can stall the Gateway main thread under multi-agent/plugin load, causing loopback `/readyz` timeouts, delayed Discord/WebUI/control traffic, fragile `logs --follow`, and reload deferral behind active-work accounting.

## Best Possible Solution

Keep this as the umbrella core reliability tracker until maintainers either land and validate the narrower hot-path fix from #74860 and confirm the reload/accounting/log-follow symptoms are covered, or split the remaining reload/task-accounting and control-plane-responsiveness work into dedicated canonical issues.

## Reproduction Assessment

Unclear for current main. The report has strong live operational evidence, but a high-confidence reproduction requires the same kind of loaded multi-agent Discord/WebUI setup with memory prompt hooks plus event-loop delay and CPU-profile capture rather than a simple local unit test.

## Solution Assessment

Not applicable as a patch review. The current main mitigations are useful but partial; the maintainable path is targeted hot-path reduction/offloading plus explicit reload/task-accounting validation, not closing the umbrella before live-load proof exists.
