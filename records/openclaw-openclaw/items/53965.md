## Summary

Keep open. Current main does not implement the intended fix, and this PR is not mergeable as-is: the new helper is unused by the active secrets/config write path, it omits existing parent-directory creation, and it suppresses credential-file chmod failures too broadly. The existing Greptile and ClawSweeper discussion matches the current source inspection.

## What This Changes

The PR adds `src/shared/fs-atomic-fix.ts`, defining a synchronous temp-file text writer that catches `chmodSync` failures before renaming the temp file into place.

## Best Possible Solution

Revise this PR or replace it with a focused change on the active secrets persistence path: preserve parent-directory creation, handle only the expected Windows-mounted filesystem failure where the current writer is actually called, keep unexpected credential-permission failures visible, and add targeted regression coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P1] Wire the helper into active writers:** `src/shared/fs-atomic-fix.ts:7`
  - body: This adds `writeTextFileAtomicSync`, but nothing imports or calls it. Current secrets apply still uses `writeTextFileAtomic` in `src/secrets/shared.ts`, whose `chmodSync` remains unguarded, so merging this leaves the reported Windows/Docker crash unchanged.
  - confidence: 0.98
- **[P2] Preserve parent directory creation:** `src/shared/fs-atomic-fix.ts:8-9`
  - body: The new writer builds and writes the temp file before ensuring the destination directory exists. The existing writer calls `ensureDirForFile(pathname)`, so using this helper for new auth/env files would regress to `ENOENT` when the parent directory has not been created yet.
  - confidence: 0.94
- **[P2] Narrow the chmod fallback to EPERM:** `src/shared/fs-atomic-fix.ts:13-15`
  - body: The catch suppresses every `chmodSync` failure. If this is used for auth/config files, non-Windows or non-EPERM permission hardening failures would be hidden while the write still proceeds.
  - confidence: 0.84
