## Summary

This PR should stay open but is not merge-ready. Current main still has the active synchronous secrets writer calling `fs.chmodSync` and `fs.renameSync` without the Windows/Docker fallback this PR is trying to add, while the PR patch only adds a new helper file that has no callers. Later main hardens generic JSON helpers, but that does not cover the `secrets apply` auth/env write path.

## Best Possible Solution

Request changes or supersede this PR with a focused patch on the active secrets write path: update `src/secrets/shared.ts` or route `secrets apply` through an existing hardened helper, preserve `ensureDirForFile`, handle expected Windows `EPERM`/`EEXIST` replacement behavior deliberately, keep chmod best-effort narrow and documented, and add a `src/secrets/apply.test.ts` regression that mocks the Windows failure mode.
