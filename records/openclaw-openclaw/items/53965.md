## Summary

Keep this PR open, but it is not merge-ready. Current main still leaves the active secrets apply writer on the direct `chmodSync`/`renameSync` path, while the PR only adds an uncalled helper, so merging it would not fix the intended Windows-mounted Docker volume crash.

## What This Changes

The PR adds a new `src/shared/fs-atomic-fix.ts` helper that writes text through a temp file, ignores `chmodSync` failures, and renames the temp file into place.

## Best Possible Solution

Request changes or supersede this with a focused patch on the active secrets write path. The fix should update `src/secrets/shared.ts` or route secrets apply through an existing hardened helper, preserve `ensureDirForFile`, handle expected Windows chmod/replace failures deliberately, keep unexpected credential-file permission failures visible, and add a mocked regression test for the Windows failure mode.
