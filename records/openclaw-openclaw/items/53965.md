## Summary

This PR should stay open, but it is not merge-ready. Current main still has the active secrets write path calling `chmodSync` and `renameSync` without the Windows/Docker fallback, while this PR only adds an uncalled helper file, so the intended runtime fix is absent.

## What This Changes

The PR adds a new `src/shared/fs-atomic-fix.ts` helper that writes text through a temp file, ignores `chmodSync` failures, and then renames the temp file into place.

## Best Possible Solution

Request changes or supersede this PR with a focused patch on the active secrets write path: update `src/secrets/shared.ts` or route secrets apply through an existing hardened helper, preserve `ensureDirForFile`, handle expected Windows chmod/replace failures deliberately, keep credential-file permission failures visible where they matter, and add a `src/secrets/apply.test.ts` regression for the mocked Windows failure mode.
