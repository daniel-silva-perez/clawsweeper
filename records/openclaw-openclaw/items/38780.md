This PR adds opt-in agent continuation tools and response-token fallbacks, continuation/delegate scheduling, context-pressure and compaction hooks, TaskFlow-backed delegate state, protocol/client/status/docs changes, and continuation-focused tests.

  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: Security review needs attention because the diff introduces autonomous scheduling and delegation paths while weakening delegate-drain, cancel, and build-boundary controls.

Concerns:

- **[medium] Continuation delegation is exposed outside draining runs:** `src/agents/openclaw-tools.ts:347`
  - body: continue_delegate can be exposed when continuation is enabled and the drain flag is omitted, while embedded tool construction does not forward the drain flag. For an opt-in autonomous background-work feature, this broadens the tool surface beyond the expected scheduler and drain pipeline.
  - confidence: 0.86
- **[medium] TaskFlow conflicts can replay cancelled or consumed work:** `src/auto-reply/continuation-delegate-store-taskflow.ts:98`
  - body: The TaskFlow-backed delegate store ignores applied:false results from finish and cancel status updates. In conflict cases, queued records can remain dispatchable after they were logically consumed or cancelled, weakening operator cancellation and autonomous-work bounds.
  - confidence: 0.8
- **[low] Build warning suppression weakens lazy-boundary review:** `tsdown.config.ts:63`
  - body: The diff suppresses INEFFECTIVE_DYNAMIC_IMPORT for pi-embedded.ts. This is not a dependency or secret leak, but it weakens a build-time guard around a runtime code-execution boundary changed by the PR.
  - confidence: 0.72
