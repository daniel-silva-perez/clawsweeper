## Summary

Keep open. Current main has not implemented the requested ACPX silent-exit classification, but this PR is not merge-ready: it targets stale embedded ACPX runtime code that current main replaced with the published `acpx/runtime` package, and the diff still misclassifies several non-silent/control/recovery paths as backend unavailable.

## What This Changes

The PR changes ACPX/ACP error classification so selected non-zero `acpx` exits surface as `ACP_BACKEND_UNAVAILABLE`, with runtime, manager, fixture, and error-text test updates.

## Best Possible Solution

Move the classification to the current ACPX boundary: only genuinely silent prompt/backend exits should become `ACP_BACKEND_UNAVAILABLE`, while stderr-backed failures, permission-denied exits, control commands, session initialization, and recovery paths keep their current semantics. Consume that through a published or intentionally temporary pinned `acpx` update with focused OpenClaw adapter, manager, and error-text coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the review findings: the PR diff exposes deterministic branches for prompt/control exits and ensure-session handling, and current-main tests show the requested manager fallback is not implemented. I did not execute tests because this pass was read-only.

## Solution Assessment

Is this the best way to solve the issue? No. The PR edits stale in-tree ACPX runtime code and broadens classification beyond truly silent exits; the safer path is package-level ACPX classification plus narrow OpenClaw wrapper/manager tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Guard backend-unavailable code behind truly silent prompt exits:** `extensions/acpx/src/runtime.ts:676-679`
  - body: `runTurn` now tags any non-zero prompt exit without a structured `error` event as `ACP_BACKEND_UNAVAILABLE`, but this path also covers exits that include concrete stderr. Those stderr-backed prompt/auth/model/validation failures should keep turn-failure recovery instead of routing users to `/acp doctor`.
  - confidence: 0.86
- **[P2] Preserve turn-failure codes for non-silent control exits:** `extensions/acpx/src/runtime.ts:1038-1044`
  - body: `resolveSilentAcpxControlExitErrorCode` remaps every non-permission-denied control exit from `ACP_TURN_FAILED` to `ACP_BACKEND_UNAVAILABLE` without checking whether the command wrote stderr. Status, cancel, close, mode, or config failures can therefore be mislabeled as backend outages.
  - confidence: 0.84
- **[P2] Keep ensure-session recovery path for silent ensure exits:** `extensions/acpx/src/runtime.ts:480-484`
  - body: The early rethrow skips `recoverEnsureFailure` for silent `sessions ensure` exits, so OpenClaw no longer runs the status probe and fallback reuse/recreate flow that can recover transient session-init failures.
  - confidence: 0.83
