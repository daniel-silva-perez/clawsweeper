## Summary

Keep open. The requested behavior is still relevant, but this PR should not merge as-is: current main has moved ACPX process behavior behind the published `acpx/runtime` package, while the PR diff remaps several non-silent or recoverable failures to backend-unavailable guidance.

## What This Changes

The PR changes ACPX/ACP error mapping so silent non-zero `acpx` exits surface as `ACP_BACKEND_UNAVAILABLE`, adds a manager fallback for plain `acpx exited with code 1` errors, and updates runtime, manager, and error-text tests.

## Best Possible Solution

Rework this at the current ACPX boundary so only genuinely silent prompt/backend exits become `ACP_BACKEND_UNAVAILABLE`; stderr-backed failures, permission-denied exits, control commands, session-init failures, and recovery flows should keep their existing semantics. The durable path is likely an `openclaw/acpx` runtime change, a pinned `acpx` dependency bump, and focused OpenClaw adapter/manager/error-text tests.

## Reproduction Assessment

Yes for the review findings: the provided PR diff exposes deterministic branches for non-zero prompt/control exits and ensure-session error handling. I did not run tests because this review is read-only, but current-main source and tests confirm the PR's intended manager fallback is not currently implemented.

## Solution Assessment

No. The PR targets stale local process-spawn logic and remaps failures too broadly; the safer solution is a narrower upstream ACPX error classification plus OpenClaw tests at the package wrapper and manager boundaries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Guard prompt exit remapping behind empty stderr:** `extensions/acpx/src/runtime.ts:679`
  - body: This maps any non-zero prompt exit without a structured ACP error event to `ACP_BACKEND_UNAVAILABLE`, even when stderr contains a real prompt or validation failure. Those non-silent failures should keep turn-failure recovery instead of routing users to `/acp doctor`.
  - confidence: 0.86
- **[P2] Preserve control-command failures with stderr:** `extensions/acpx/src/runtime.ts:1041-1044`
  - body: `resolveSilentAcpxControlExitErrorCode` upgrades every non-permission-denied control exit whose fallback is `ACP_TURN_FAILED` without checking whether the command wrote stderr or which control failed. Status, cancel, close, mode, or config failures can therefore be mislabeled as backend outages.
  - confidence: 0.84
- **[P2] Keep ensure-session recovery for silent init exits:** `extensions/acpx/src/runtime.ts:483-484`
  - body: The early rethrow bypasses `recoverEnsureFailure` for a silent `sessions ensure` exit with no events, so OpenClaw no longer runs the status probe and fallback reuse/recreate flow that can recover transient session-init failures.
  - confidence: 0.83
