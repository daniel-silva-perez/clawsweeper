## Summary

Keep open. Current main still does not implement the PR's intended silent ACPX exit classification, and the PR branch is stale against the current thin wrapper around the published `acpx/runtime` package. The proposed patch also has concrete P2 correctness issues around non-silent exits and session-init recovery, so this is not a conservative cleanup close.

## What This Changes

The PR changes ACPX runtime and ACP manager error mapping so non-zero `acpx` exits without structured ACP errors surface as `ACP_BACKEND_UNAVAILABLE`, with regression coverage for runtime events, manager fallback, and user-facing error text.

## Best Possible Solution

Rework this as a current-boundary ACPX fix that classifies only genuinely silent prompt/backend exits as `ACP_BACKEND_UNAVAILABLE`, while preserving permission-denied, session-init, non-silent stderr, control-command, and fresh-handle recovery behavior. The preferred durable path is an `acpx/runtime` fix plus an OpenClaw dependency bump, with OpenClaw-side adapter code only if the mapping must stay local.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Guard prompt exit remapping behind empty stderr:** `extensions/acpx/src/runtime.ts:676`
  - body: This remaps any non-zero prompt exit that lacks a structured ACP error event to `ACP_BACKEND_UNAVAILABLE`. That path can still include a concrete stderr failure from the harness, so normal prompt or validation failures would be routed to `/acp doctor` instead of turn-failure recovery. Only genuinely silent exits should be upgraded.
  - confidence: 0.86
- **[P2] Preserve control-command failures with stderr:** `extensions/acpx/src/runtime.ts:1039-1042`
  - body: `resolveSilentAcpxControlExitErrorCode` upgrades every non-permission-denied control exit whose fallback is `ACP_TURN_FAILED`, without checking whether the command wrote stderr or whether the failure came from a control path such as status, cancel, close, mode, or config. Those user/control failures can be mislabeled as backend outages.
  - confidence: 0.84
- **[P2] Keep ensure-session recovery for silent init exits:** `extensions/acpx/src/runtime.ts:481-482`
  - body: The early rethrow bypasses `recoverEnsureFailure` for a silent `sessions ensure` exit with no events, so OpenClaw no longer runs the status probe and fallback reuse/recreate flow that can recover transient session-init failures.
  - confidence: 0.83
