## Summary

Keep open for maintainer review. The underlying ACPX exit-classification problem is plausible, but this PR is dirty against current main, edits the old local process-exit loop that current main replaced with the published `acpx/runtime` package, and still introduces broad remapping regressions in the diff.

## What This Changes

The PR changes ACPX/ACP error mapping so silent non-zero `acpx` exits surface as `ACP_BACKEND_UNAVAILABLE`, adds a manager fallback for plain `acpx exited with code 1` errors, and updates runtime, manager, fixture, and error-text tests.

## Best Possible Solution

Move the runtime-exit classification to the current ACPX boundary: only genuinely silent prompt/backend exits should become `ACP_BACKEND_UNAVAILABLE`, while stderr-backed failures, permission-denied exits, control commands, session initialization, and recovery paths keep their current semantics. Consume that through a published or temporary pinned `acpx` update with focused OpenClaw adapter, manager, and error-text tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the review findings: the PR diff exposes deterministic branches for non-zero prompt/control exits and ensure-session error handling, and current-main tests show the requested manager fallback is not currently implemented. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

Is this the best way to solve the issue? No. The patch targets stale local process-spawn logic and remaps failures too broadly; the safer path is a narrower ACPX package-level classification plus OpenClaw tests at the package wrapper and manager boundaries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Guard prompt exit remapping behind empty stderr:** `extensions/acpx/src/runtime.ts:679`
  - body: `runTurn` now maps any non-zero prompt exit without a structured ACP error event to `ACP_BACKEND_UNAVAILABLE`, even when stderr contains a real prompt, auth, model, or validation failure. Those non-silent failures should keep turn-failure recovery instead of routing users to `/acp doctor`.
  - confidence: 0.86
- **[P2] Preserve control-command failures with stderr:** `extensions/acpx/src/runtime.ts:1041-1044`
  - body: `resolveSilentAcpxControlExitErrorCode` upgrades every non-permission-denied control exit whose fallback is `ACP_TURN_FAILED` without checking whether the command wrote stderr or which control failed. Status, cancel, close, mode, or config failures can therefore be mislabeled as backend outages.
  - confidence: 0.84
- **[P2] Keep ensure-session recovery for silent init exits:** `extensions/acpx/src/runtime.ts:483-484`
  - body: The early rethrow bypasses `recoverEnsureFailure` for a silent `sessions ensure` exit with no events, so OpenClaw no longer runs the status probe and fallback reuse/recreate flow that can recover transient session-init failures.
  - confidence: 0.83
