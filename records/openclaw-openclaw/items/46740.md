## Summary

Keep open. The PR targets useful ACPX error classification work, but it is not merge-ready against current main: the active ACPX wrapper now delegates runtime behavior to the published `acpx/runtime` package, while this patch changes the older in-repo process path and still has blocking P2 behavior regressions around non-silent exits, control commands, and ensure-session recovery.

## What This Changes

The PR adds ACPX/ACP error-mapping helpers and tests so silent non-zero `acpx` exits surface as `ACP_BACKEND_UNAVAILABLE` with `/acp doctor` guidance while preserving permission-denied and session-init handling.

## Best Possible Solution

Rework the fix at the current ACPX boundary so only genuinely silent prompt/backend exits become `ACP_BACKEND_UNAVAILABLE`, while stderr-backed failures, control commands, permission-denied exits, session-init errors, and fresh-handle recovery keep their existing semantics. The durable path is likely an upstream `openclaw/acpx` runtime change followed by an OpenClaw dependency bump and focused OpenClaw adapter tests only where core behavior is involved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Guard prompt exit remapping behind empty stderr:** `extensions/acpx/src/runtime.ts:679`
  - body: This sets `ACP_BACKEND_UNAVAILABLE` for any non-zero prompt exit without a structured error event, even when `stderr` contains a real prompt or validation failure that `formatAcpxExitMessage` will surface. Those non-silent failures should keep turn-failure recovery instead of routing users to `/acp doctor`.
  - confidence: 0.86
- **[P2] Preserve control-command failures with stderr:** `extensions/acpx/src/runtime.ts:1041-1044`
  - body: `resolveSilentAcpxControlExitErrorCode` upgrades every non-permission-denied control exit whose fallback is `ACP_TURN_FAILED` without checking whether the command wrote stderr or which control failed. Status, cancel, close, mode, or config failures can therefore be mislabeled as backend outages.
  - confidence: 0.84
- **[P2] Keep ensure-session recovery for silent init exits:** `extensions/acpx/src/runtime.ts:483-484`
  - body: The early rethrow bypasses `recoverEnsureFailure` for a silent `sessions ensure` exit with no events, so OpenClaw no longer runs the status probe and fallback reuse/recreate flow that can recover transient session-init failures.
  - confidence: 0.83
