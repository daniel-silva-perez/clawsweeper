## Summary

Current main implements first-class `--thread-id` support for `openclaw cron add/create` and `openclaw cron edit`, validates positive integer Telegram topic IDs, passes the value through the cron delivery contract, and documents the Telegram topic cron flow. The fix is on main in commit 76cd97289b790fdbb49792c65ff48b01384872ec and is currently listed under Unreleased, with no release tag containing it yet.

## Best Possible Solution

Close this issue as implemented on main, keep the current cron CLI `delivery.threadId` approach, and let any remaining list-display or failure-cascade concerns continue as narrower follow-up issues if maintainers want those refinements.
