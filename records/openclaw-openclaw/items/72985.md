## Summary

Keep open. Current main still leaves the cron isolated-agent fallback path accepting incomplete embedded-runner results when no classifier hook is supplied, and successful channel delivery of that error payload can still persist as `lastRunStatus: "error"` with `lastDelivered: true`.

## What This Changes

The issue asks cron isolated-agent model fallback to retry or alert on incomplete-turn fallback failures and to stop recording failed cron runs as normally delivered output.

## Best Possible Solution

Cron should classify incomplete embedded-runner outputs before accepting fallback attempts, and delivery state should distinguish user-useful assistant output from error notifications instead of overloading `lastDelivered`.

## Reproduction Assessment

Yes. A deterministic source-level reproduction can mock the isolated embedded run to return the incomplete-turn warning as an `isError` payload, let announce delivery succeed, and observe the current path returning and persisting `status: "error"` with `delivered: true`.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing embedded-runner result classifier in cron fallback and adjust delivery persistence semantics; a regex-only persistence patch would leave fallback acceptance and monitoring semantics split across layers.
