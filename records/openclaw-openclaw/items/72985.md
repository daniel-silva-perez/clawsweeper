## Summary

Keep open. Current main still has the reported source-level gap: isolated cron fallback does not pass the embedded-runner result classifier into the model fallback loop, and cron delivery persistence can still record an error result with successful delivery metadata when the delivered payload is only an error message.

## What This Changes

The issue requests that isolated-agent cron fallback stop treating incomplete-turn warning text as normal delivered cron output and avoid persisting error runs as successfully delivered content.

## Best Possible Solution

Cron should classify incomplete embedded-runner outputs before accepting fallback attempts, then persist delivery state so user-useful assistant output is distinct from any error notification or fallback warning text.

## Reproduction Assessment

Yes. A deterministic reproduction can mock an isolated embedded run to return only the incomplete-turn warning as an isError payload, let announce delivery succeed, and observe the current path returning an error run with delivered metadata.

## Solution Assessment

Yes. The narrow maintainable direction is to reuse the existing embedded-runner result classifier in the cron fallback call and adjust cron delivery persistence semantics; a regex-only lastDelivered patch would leave fallback acceptance and monitoring behavior split across layers.
