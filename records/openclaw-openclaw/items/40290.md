## Summary

Keep this PR open for maintainer handling. Current main still blocks credential-shaped `sandbox.docker.env` keys and the PR addresses a real operator-configured env-var problem, but the submitted branch is not mergeable because it weakens skill eligibility policy and drops existing sandbox env sanitizer options. The affected surfaces are Docker sandbox container creation and agent skill prompt/sync filtering for https://github.com/openclaw/openclaw/pull/40290.

## What This Changes

The PR adds a `forceAllowKeys` env-sanitizer bypass for configured Docker env keys, exports sandbox env validation, keeps `/root` in the host-path denylist, and lets explicitly listed agent skills bypass eligibility checks.

## Best Possible Solution

Revise this into a narrow sandbox env fix: allow only resolved operator-configured `sandbox.docker.env` keys to bypass the credential-name blocklist, preserve all existing sanitizer options and value validation, keep skill eligibility gates unchanged unless a separate product/security decision approves that policy change, and coordinate behavior with #44569 and #57427.
