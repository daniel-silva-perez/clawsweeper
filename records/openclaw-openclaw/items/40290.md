## Summary

Keep this PR open. Current main still blocks credential-shaped `sandbox.docker.env` keys, but the submitted patch is not safe to merge as-is because it bypasses existing skill eligibility gates and drops existing sanitizer options when adding `forceAllowKeys`.

## What This Changes

The branch adds a `forceAllowKeys` env-sanitizer bypass for configured Docker env values, exports sandbox env validation, keeps `/root` in the host-path denylist, and lets explicitly listed agent skills bypass eligibility checks.

## Best Possible Solution

Revise this into a narrow sandbox env fix: allow only resolved operator-configured `sandbox.docker.env` keys to bypass the credential-name blocklist, preserve existing sanitizer options and value validation, keep skill eligibility gates unless a separate policy change is approved, and coordinate semantics with #44569 and #57427.
