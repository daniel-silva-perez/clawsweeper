## Summary

Current main does not implement the PR’s intended partial bootstrap suppression. `ensureAgentWorkspace` still accepts only `dir` and `ensureBootstrapFiles`, writes `SOUL.md`, `IDENTITY.md`, `USER.md`, and `HEARTBEAT.md` unconditionally when bootstrap is enabled, and the agent defaults schema/types/docs expose only the broader `skipBootstrap` option. The PR is focused and not obsolete, but the provided review comments identify real merge-blocking behavior gaps: the new option would need to be propagated across all workspace bootstrap entrypoints, constrained to valid optional filenames, and adjusted so legacy setup detection still respects existing customized `USER.md`/`IDENTITY.md` files. Security review did not find supply-chain or CI-sensitive file changes in the PR diff.

## Best Possible Solution

Keep this PR open or replace it with a revised patch that implements the feature end to end: add a constrained config type/schema, document the new key, thread it through every production `ensureAgentWorkspace` caller that uses `agents.defaults`, preserve required-file creation, handle skipped-missing files without masking existing customized legacy files, and add focused tests for skipped optional files, required-file non-skipping, schema validation, and legacy migration behavior.

## Risks / Open Questions

- The PR as submitted would likely be inconsistent because common workspace bootstrap paths still ignore the new setting.
- Without schema enum validation, misspelled filenames or required filenames could be accepted but silently ignored at runtime.
- The migration-probe fallback described in the PR can regress legacy setup detection if it substitutes template text even when a skipped `USER.md` or `IDENTITY.md` file already exists and is customized.
- Suppressing optional workspace guidance files is behavior-sensitive; it should be limited to the four intended optional filenames and covered by focused workspace/config tests.
