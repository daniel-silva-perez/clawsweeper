## Summary

Keep open. Current main still does not implement `agents.defaults.skipOptionalBootstrapFiles`, but this PR is not ready to merge because maintainer review and source inspection both show incomplete config-contract updates, insufficient validation, partial entrypoint propagation, and a legacy bootstrap-completion regression.

## What This Changes

The PR adds an `agents.defaults.skipOptionalBootstrapFiles` option intended to prevent selected optional workspace bootstrap files from being created while keeping required bootstrap files active.

## Best Possible Solution

Finish the feature as a complete config contract: validate the skip list against the four optional filenames, propagate it through every workspace bootstrap path that uses `agents.defaults`, preserve existing customized USER.md/IDENTITY.md as legacy completion evidence, update generated schema/help/docs, and add focused workspace/config regression tests.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: with this PR applied, `agent-command` passes the skip list, but reply, cron, setup, and gateway paths still call `ensureAgentWorkspace` without it, and invalid skip names parse successfully under the new Zod string-array schema. I did not run tests because this review was read-only.

## Solution Assessment

No. The direction is reasonable, but the proposed patch is not the narrowest maintainable solution until it treats the option as a full public config contract and applies the same behavior across all workspace bootstrap entrypoints.

  - confidence: 0.93
