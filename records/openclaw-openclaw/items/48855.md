## Summary

Keep this issue open. Current main still treats Android as unsupported for managed Gateway service lifecycle, and the Android app/docs describe Android as a companion node that requires a Gateway on macOS, Linux, or Windows/WSL2. The linked AndroidForClaw project is useful external context/workaround, but it does not make OpenClaw's own Android/Termux Gateway support implemented or clearly move this platform request to ClawHub.

## Best Possible Solution

Keep the issue open for maintainer product/platform judgment. A complete fix would either add a supported Android/Termux Gateway mode with lifecycle handling and docs, or make an explicit docs/product decision that Android remains node-only while pointing users to maintained external options such as AndroidForClaw or a future ClawHub package.

## Risks / Open Questions

- Android Gateway hosting would need explicit platform/security design for Termux lifecycle, Android background restrictions, operator auth, port binding/exposure, and dashboard access before implementation.
- Closing this as ClawHub now would conflate a useful external Android app workaround with OpenClaw's own documented and tested platform support.
