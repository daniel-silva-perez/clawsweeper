## Summary

Keep this PR open. Current main still lacks save-time canonicalization for persisted session transcript paths, and this PR remains a focused implementation candidate, but the current diff has a functional review blocker because it canonicalizes through the compatibility resolver rather than forcing the active sessions directory. Security review did not find supply-chain or CI-sensitive changes in the PR diff.

## Best Possible Solution

Keep this PR open and request a small revision: add save-time normalization in `src/config/sessions/store.ts`, derive the canonical target directly from `path.dirname(storePath)` plus a validated `sessionId`, persist both `sessionFile` and the legacy `transcriptPath` alias if that alias remains supported, add coverage for structurally valid cross-root paths, and preserve the existing external-transcript deletion safeguards in pruning and disk-budget tests.
