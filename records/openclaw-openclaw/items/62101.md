## Summary

Keep this PR open. Current main still does not coerce bare numeric BlueBubbles allowFrom entries, so the linked BlueBubbles allowlist bug is not implemented on main. The PR is a plausible implementation candidate, but it should be revised before landing because the shared schema transform would affect other channel plugins that use numeric sender IDs.

## Best Possible Solution

Keep the PR open but request a BlueBubbles-local fix: add the digit-to-+E.164 guard in BlueBubbles target normalization and make any schema/config coercion channel-specific, or otherwise preserve numeric-ID semantics for QQBot and other plugins using the shared AllowFromListSchema. The follow-up tests in the PR are useful and should stay, with added coverage proving numeric allowFrom still works for a non-BlueBubbles channel.
