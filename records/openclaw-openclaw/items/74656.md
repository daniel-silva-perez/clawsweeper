## Summary

Keep open. The patch addresses recovery replay classification, but it does not yet solve the linked bug end to end because fresh live send failures still call `failDelivery` and remain pending for retry/backoff until a later recovery pass. The security pass is clear.

## What This Changes

The branch extends `src/infra/outbound/delivery-queue-recovery.ts` permanent-error regexes for payload-size, auth, and not-found delivery failures, adds two recovery regression tests, and adds a changelog entry.

## Best Possible Solution

Use one conservative permanent-delivery classification path for both live queue cleanup and recovery replay, or route that decision through a channel-owned classifier seam if maintainers prefer that direction. Cover policy, recovery replay, and first-live-failure behavior before merge.
