## Summary

Keep this issue open. Current main still lets the Moonshot/Kimi web_search provider return the final chat-completion text as a successful search result even when no citation or search-result evidence was collected. The docs now clarify that Kimi is an AI-synthesized answer provider, but that does not satisfy the report's requested fail-closed behavior for generic no-search answers.

## Best Possible Solution

Keep this issue open. The best fix belongs in the Moonshot plugin runtime: require evidence that native `$web_search` actually produced grounding, such as citations/search_results/tool payloads, before returning success; otherwise return a structured Kimi provider failure. Add focused unit tests for chat-only/no-citation responses and unreplayable tool calls, then update the Kimi docs if the visible behavior changes.
