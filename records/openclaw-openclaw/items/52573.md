## Summary

Current main still returns Kimi's final chat-completion text as a successful web_search payload when Moonshot stops tool-calling, including paths where no search citation or result was collected. The related docs cleanup (#52575) appears addressed, but the runtime bug report remains actionable.

## Best Possible Solution

Keep the issue open. The best fix is in the Moonshot plugin runtime: require evidence that native $web_search actually produced grounding, such as citations/search_results/tool payloads, before returning success; otherwise return a structured Kimi provider failure. Add focused unit tests for chat-only/no-citation responses and malformed/unreplayable tool calls, then update the Kimi docs if the visible behavior changes.
