## Summary

Keep this PR open. Current main still has a plausible ByteString crash path for pasted non-Latin1 API keys, but the PR's global SecretInput normalization is too broad and can mutate valid Unicode credentials outside HTTP-header usage.

## Best Possible Solution

Keep the PR open, but request a narrower patch. Preserve trim-only semantics for `normalizeSecretInputString`, `hasConfiguredSecretInput`, and non-header credentials. Apply `normalizeSecretInput` or `normalizeOptionalSecretInput` only at HTTP-header boundaries, starting with Comfy Cloud `X-API-Key` and shared provider request auth/header assembly, then add regression coverage for the Comfy pasted-U+2502 path plus a guard proving Matrix-style Unicode passwords are preserved.
