## Summary

Keep this PR open. Current main still has a plausible ByteString crash path for pasted non-Latin1 API keys in header-bound flows, but the PR's proposed global SecretInput normalization is too broad and can mutate valid Unicode credentials outside HTTP-header usage.

## Best Possible Solution

Keep the PR open and request a narrower patch. Preserve trim-only semantics for normalizeSecretInputString, hasConfiguredSecretInput, and non-header credentials. Apply normalizeSecretInput or normalizeOptionalSecretInput only where values are assembled into HTTP headers, starting with Comfy Cloud X-API-Key and shared provider request auth/header assembly. Add regression coverage for the pasted-U+2502 Comfy/header path and a guard proving Matrix-style Unicode passwords remain unchanged.
