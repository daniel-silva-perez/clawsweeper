## Summary

Current main still does not expose context window size, effective context token budget, model max output tokens, or utilization on the plugin-facing `llm_output` event. The relevant model metadata exists elsewhere, so this remains a focused additive plugin hook contract and emitter update rather than a cleanup close.

## Best Possible Solution

Keep this issue open. Implement an additive plugin hook contract update that adds optional native model context window, effective context token budget, and model max output-token fields to `PluginHookLlmOutputEvent`; populate them in the embedded Pi runner, CLI harness, and Codex app-server harness; add focused hook emission and contract tests; and update the public plugin hooks docs. Treat utilization as plugin-derived unless OpenClaw defines a reliable context-used numerator.

## Risks / Open Questions

- Field naming should be coordinated with #64327 so native model context window, effective runtime context cap, and max output-token limit do not diverge as plugin API concepts.
- A computed utilization field needs a precise numerator because provider usage totals can include output/cache/multiple-call totals and may not equal live prompt-context occupancy.
- The hook contract change should be optional and additive so existing third-party plugin handlers remain compatible.
