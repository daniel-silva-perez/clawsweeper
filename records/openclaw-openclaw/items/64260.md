## Summary

Current main still does not expose context-window size, effective context-token budget, model max output tokens, or utilization on the plugin-facing `llm_output` event. The model/runtime metadata exists nearby, so this remains a focused additive plugin hook contract and emitter update rather than a cleanup close.

## Best Possible Solution

Keep this issue open. Implement an additive, optional plugin hook contract update for `PluginHookLlmOutputEvent`, populate the fields in the embedded Pi runner, CLI harness, and Codex app-server harness, add focused hook emission/contract tests, and update the public plugin hooks docs. Prefer plugin-derived utilization unless OpenClaw defines a reliable context-used numerator.
