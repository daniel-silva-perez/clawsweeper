## Summary

Keep open. Current main still exposes only identifiers, assistant text, and usage counts on the plugin-facing `llm_output` event; native/effective context limits and max output-token metadata exist elsewhere but are not threaded through the hook contract, emitters, tests, or docs. The linked #64327 should be coordinated with the same contract shape for effective context caps.

## What This Changes

Add context-window, effective context-budget, max output-token, and optional utilization metadata to the plugin-facing `llm_output` event.

## Best Possible Solution

Define optional additive fields on `PluginHookLlmOutputEvent` for native model context window, effective runtime context-token budget, and model max output tokens; populate them consistently across embedded Pi, CLI, and Codex app-server harnesses; document them on the public hooks page; and leave utilization plugin-derived unless OpenClaw defines an exact context-used numerator.
