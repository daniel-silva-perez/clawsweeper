## Summary

Keep this PR open for maintainer review. Current main still routes every bare numeric Discord outbound target as `channel:<id>` and the Discord outbound adapter does not pass `allowFrom`; the PR is a scoped implementation for that missing allowFrom-aware DM case, with one remaining precedence question around inherited top-level `allowFrom` versus account-level legacy `dm.allowFrom`.

## What This Changes

The PR updates Discord target normalization/resolution so bare numeric IDs in allowFrom are treated as DM user targets, threads allowFrom into outbound target resolution, prefers top-level allowFrom with legacy dm.allowFrom fallback, and adds focused Discord tests.

## Best Possible Solution

Keep this PR open, resolve the account-level legacy allowFrom precedence question, then review and land the scoped Discord-plugin patch if the compatibility tradeoff is accepted. Before merge, run the focused Discord tests/format/lint checks and the appropriate changed gate for an extension runtime/test change.
