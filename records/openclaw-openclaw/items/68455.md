## Summary

Keep open. Current main still routes Feishu onboarding through the default account path, and this PR targets the right product gap but is not mergeable yet: the selected accountId is discarded before config writes, and the new prompt fires on fresh setup because fallback "default" is treated as a real existing account. The existing review discussion called out the same P1/P2 issues, and source inspection confirms them.

## What This Changes

The PR adds an interactive Feishu account ID prompt for additional onboarding attempts, exports the Feishu account-list default helper, and changes the setup adapter to normalize provided account IDs.

## Best Possible Solution

Update the Feishu plugin-owned onboarding flow so the resolved accountId is passed through finalization, edit, and new-app setup and used for all credential/security patches; keep first-run setup on top-level/default config without a new-account prompt; add focused tests for fresh setup and second-account setup.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: choose a new Feishu account ID in the PR branch, and the setup framework passes it to `finalize`, but Feishu drops it before calling the config-writing helpers, which re-resolve the default target.

## Solution Assessment

No. Prompting for a custom accountId is the right direction, but this implementation is incomplete because it does not use the selected accountId for writes and it misdetects fresh installs as having an existing account.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Propagate the selected accountId into setup writes:** `extensions/feishu/src/setup-surface.ts:643-654`
  - body: `resolveAccountIdForConfigure` can now return a custom ID, but `finalize` never consumes the `accountId` that the setup framework passes in and still calls `runNewAppFlow`/`runEditFlow` without it. Those helpers re-derive the default target, so a newly selected account is not where credentials and policy are written.
  - confidence: 0.94
- **[P2] Gate the new-account prompt on real Feishu configuration:** `extensions/feishu/src/setup-surface.ts:556-560`
  - body: `listFeishuAccountIds(cfg)` includes fallback `default` even when no Feishu account is configured, so this branch never reaches the fresh-install default path. Fresh setup will show a misleading existing-account prompt; check real Feishu configuration or configured account IDs before prompting.
  - confidence: 0.91
