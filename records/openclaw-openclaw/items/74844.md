## Repair Work Prompt

Fix the configured MCP tool exposure regression tracked by https://github.com/openclaw/openclaw/issues/74844. The observable bug is that working `mcp.servers` entries such as Gmail/Outlook are listed by `openclaw mcp list` and respond to direct MCP `tools/list`, but the normal embedded Pi provider request `tools[]` contains only built-in OpenClaw tools and no `server__tool` entries. Add a focused regression that starts from a configured stdio MCP catalog and asserts the normal embedded Pi/provider request boundary includes the materialized `server__tool` entries. Then fix the narrow gate that drops them before serialization. Preserve existing `bundle-mcp` semantics: `coding` and `messaging` include bundle MCP by default, `minimal` hides it, `tools.deny: ["bundle-mcp"]` opts out, and explicit tool allowlists must remain restrictive. Do not change unrelated plugin loading, provider auth, or MCP server transport behavior. Run targeted tests first, then the changed gate in Testbox per repo policy. Add a changelog entry only if the repair changes shipped user-facing behavior.
