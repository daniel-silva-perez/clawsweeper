## Summary

Keep open. Current main still allows `openclaw status --json` to show persisted WhatsApp direct sessions while agent-side `sessions_list` and `sessions_send` remain gated by the default `tools.sessions.visibility="tree"` policy, which can hide same-agent non-current WhatsApp DM sessions.

## Best Possible Solution

Keep this issue open. Reproduce on current main with a linked WhatsApp account, `session.dmScope=per-channel-peer`, a successful cron delivery, and agent-side `sessions_list`/`sessions_send` from the same automation context. If maintainers want on-demand automation to target existing WhatsApp DMs by default, implement a generic, policy-aware discovery or target-resolution path for deliverable same-agent channel sessions, with WhatsApp direct coverage and docs clarifying the required visibility setting. Avoid a core special case for WhatsApp ids.
