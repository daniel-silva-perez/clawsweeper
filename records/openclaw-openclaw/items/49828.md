## Summary

Keep open. Current main still renders Control UI WebChat agent/session switching as a header native select, not a persistent WhatsApp/Telegram-style sidebar list. The existing chat sidebar is for rendered markdown/canvas/tool content, and related dropdown work such as #48666 does not supersede the requested multi-agent conversation-list UX.

## Best Possible Solution

Keep this issue open for a focused Control UI chat improvement: add a persistent agent/session sidebar fed by `agents.list` and `sessions.list`, switch via the existing session-switch path, define preview/history semantics, keep a responsive fallback, and cover multi-agent WebChat behavior with UI tests and docs. If maintainers want to merge this into a broader session-sidebar tracker, do that explicitly rather than auto-closing this distinct request.
