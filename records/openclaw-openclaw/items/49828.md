## Summary

Keep this issue open. Current main still implements WebChat/Control UI agent switching as a chat-header session dropdown grouped by agent, not as a persistent WhatsApp/Telegram-style sidebar of agent conversations. The provided discussion is useful and points to a plausible frontend implementation path, but no canonical related item in the provided context fully supersedes this sidebar UX request.

## Best Possible Solution

Implement this as a focused Control UI chat feature: add a persistent agent/session sidebar fed by `agents.list` and `sessions.list` with preview fields, switch through the existing session-switch path, define whether each row is per-agent main-session switching or cached per-agent history, and add UI tests plus docs for multi-agent WebChat behavior.
