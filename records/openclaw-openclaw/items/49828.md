## Summary

Current main still renders Control UI WebChat agent/session switching as a native header dropdown grouped by session/agent. The requested persistent WhatsApp/Telegram-style sidebar agent conversation list is not implemented, and the related context only covers adjacent dropdown/panel work rather than superseding this UX request.

## Best Possible Solution

Keep this issue open for a focused Control UI chat feature: add a persistent agent/session sidebar fed by `agents.list` and `sessions.list`, switch via the existing session-switch path, define preview/history semantics, and cover the multi-agent WebChat behavior with UI tests and docs.
