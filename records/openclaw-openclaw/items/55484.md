## Summary

Keep this issue open. Current main still does not implement the requested ACP `sessions_spawn` non-thread persistent affinity contract: there is no `affinityKey`, ACP spawn modes remain `run` and thread-bound `session`, non-thread ACP spawns still create a fresh random OpenClaw session key, and the docs/tests describe `resumeSessionId` rather than cron/orchestrator-owned affinity reuse. The downstream `WuKongAI-CMU/openclaw#3` work linked in the comments remains useful prior art, but it has not landed upstream.

## Best Possible Solution

Keep the issue open and adapt or reimplement the downstream `WuKongAI-CMU/openclaw#3` contract upstream: add `mode: "persistent"` plus `affinityKey` for `sessions_spawn({ runtime: "acp" })`, restrict it to trusted cron/hook/orchestrator contexts, derive deterministic ACP session keys from requester owner identity plus backend, cwd, and affinity key, preserve existing affinity-backed sessions when later dispatch fails, and add focused source, tool-schema, docs, and regression tests.
