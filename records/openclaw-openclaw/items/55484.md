## Summary

Keep this issue open. Current main still does not provide the requested ACP `sessions_spawn` non-thread persistent affinity contract: the ACP spawn mode union is still only `run` and thread-bound `session`, there is no `affinityKey` parameter, ACP child session keys are still generated with a UUID per spawn, and docs/tests describe only the existing thread-bound persistence plus `resumeSessionId` behavior. The downstream WuKongAI-CMU/openclaw#3 implementation linked in the issue comments remains useful prior art, but it has not landed upstream.

## Best Possible Solution

Keep the issue open and review/adapt the downstream WuKongAI-CMU/openclaw#3 approach or implement the same upstream contract in `src/agents/tools/sessions-spawn-tool.ts` and `src/agents/acp-spawn.ts`: add `mode: "persistent"` plus `affinityKey` for `runtime: "acp"`, restrict it to trusted cron/hook/orchestrator contexts, derive deterministic ACP session keys scoped by requester owner/backend/cwd/affinityKey, preserve existing affinity sessions when later dispatch fails, and add focused ACP spawn/tool/docs tests.
