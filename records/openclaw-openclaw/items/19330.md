## Summary

Current main still has the reported shared inbound-media behavior. The core save path defaults to a flat `inbound` bucket, representative bundled channels still pass literal `"inbound"`, managed-media resolution rejects nested/scoped inbound paths, sandbox staging rewrites media to a flat sandbox path, and related context shows the broad issue is not superseded by the closed unmerged PR or the narrower open Feishu issue.

## Best Possible Solution

Keep this issue open as the canonical core media-isolation tracker. A maintainer-quality fix should define a backward-compatible scoped inbound-media contract, thread account or resolved agent context through bundled and plugin media saves, support scoped managed-media URI and absolute-path resolution while preserving existing flat files, preserve or intentionally map scope through sandbox staging, update docs/config help, and add regression coverage. PR #46162 is useful prior art; #60154 can cover the Feishu-specific slice but should not replace the broader issue.
