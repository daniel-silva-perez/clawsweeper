## Summary

Keep this issue open. Current main still stores representative inbound channel media in the shared `media/inbound` bucket, still rejects scoped or nested inbound managed-media references, and still stages sandbox media into a flat `media/inbound/<file>` path. The related context also points back here as the canonical broad tracker: #25712 and #67194 are closed duplicates, #46162 was a closed/unmerged implementation attempt, and #60154 is a narrower Feishu follow-up rather than a replacement.

## Best Possible Solution

Keep this issue open as the canonical core media-isolation tracker. A maintainer-quality fix should define a backward-compatible scoped inbound-media contract, thread sanitized account or resolved agent context through bundled and plugin media saves, support scoped managed-media URI and absolute-path resolution while preserving existing flat files, preserve or intentionally map scope through sandbox staging, update docs/config help, and add regression coverage. Keep #60154 as the Feishu-specific configuration slice unless the generic fix fully covers it.
