## Summary

Keep open. Current main still writes inbound channel media into the shared OpenClaw media root, several bundled channel paths still pass the literal `inbound`, managed-media URI/path resolution still rejects nested scoped inbound IDs, sandbox staging still flattens files into `media/inbound/<filename>`, and docs still describe flat inbound staging. The closed PR #46162 is useful prior art but was not merged, while #25712 and #67194 point back here as the canonical tracker.

## Best Possible Solution

Keep this issue open as core media-pipeline work. A maintainer-quality fix should define a backwards-compatible scoped inbound-media contract, thread account or resolved agent context through bundled and plugin media saves, support scoped managed-media references and absolute-path resolution, preserve existing flat `media/inbound` files, update sandbox staging and docs, and add regression coverage. PR #46162 is useful prior art but should be reopened, reviewed, or replaced before this closes.
