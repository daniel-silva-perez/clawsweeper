## Summary

Keep this issue open. Current main still writes bundled inbound media to the shared `media/inbound` bucket, managed-media resolution still rejects scoped/nested inbound IDs, sandbox staging still flattens source paths to `media/inbound/<basename>`, and the related implementation PR #46162 is closed and unmerged. The recent maintainer comment also explicitly identifies this issue as the canonical tracker, and the newer KinthAI comment adds relevant privacy-design context rather than making the issue obsolete.

## Best Possible Solution

Keep this issue open as the canonical core media isolation tracker. A maintainer-quality fix should define a backwards-compatible scoped inbound-media contract, thread account or resolved agent context through bundled and plugin media saves, support scoped managed-media URI and absolute-path resolution while preserving existing flat files, preserve or intentionally map scope through sandbox staging, update docs/config help, and add regression coverage. PR #46162 is useful prior art but should be reopened, reviewed, or replaced before this issue closes.
