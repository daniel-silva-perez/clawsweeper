## Summary

Keep this issue open. Current main still stores inbound media in the shared `media/inbound` bucket, the managed-media read path still rejects scoped or nested inbound IDs, channel callers still pass the literal `inbound` bucket, and sandbox staging still flattens files into `media/inbound/<basename>`. The prior PR #46162 is useful context, but it is closed and unmerged, so this remains the canonical tracker.

## Best Possible Solution

Keep this issue open as the canonical core media isolation tracker. A maintainer-quality fix should define a backwards-compatible scoped inbound-media contract, thread account or resolved agent context through bundled and plugin media saves, support scoped managed-media URI and absolute-path resolution while preserving existing flat files, preserve scope through sandbox staging, update docs/config help, and add regression coverage. PR #46162 should be reopened, reviewed, or replaced before this issue closes.
