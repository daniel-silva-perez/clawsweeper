## Summary

Keep this issue open. Current main still saves inbound media into the shared media root with flat `inbound` writes, the managed-media read path rejects scoped/nested inbound IDs, bundled channel callers still pass the literal `inbound` bucket, and sandbox staging still rewrites files to `media/inbound/<basename>`. The closed PR #46162 remains useful prior art, but it was not merged and current main has not implemented the requested isolation.

## Best Possible Solution

Keep this issue open as the canonical core media isolation tracker. A maintainer-quality fix should define a backwards-compatible scoped inbound-media contract, thread account or resolved agent context through bundled and plugin media saves, support scoped managed-media URI and absolute-path resolution while preserving existing flat files, preserve scope through sandbox staging, update docs/config help, and add regression coverage. PR #46162 is useful prior art but should be reopened, reviewed, or replaced before this issue closes.
