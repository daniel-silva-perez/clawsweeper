## Summary

Current main still does not implement external workspace agent auto-discovery. Agent config remains centered on explicit `agents.list`, the schema is strict and has no `agents.autoDiscover`, workspace scans are limited to configured/default agent workspaces, and plugin/bundle surfaces do not currently provide an agent-definition contribution API. TinyTb's #27706 comment is useful related zero-restart context, but it does not supersede this separate discovery request.

## Best Possible Solution

Keep this issue open and narrow it around an explicit external workspace agent-discovery contract. The best maintainer path is either a first-class `agents.autoDiscover` config surface or a documented plugin-backed agent-definition discovery API that runs before normal agent enumeration, with trusted roots, accepted filenames, schema validation, deterministic ordering, explicit precedence against `agents.list`, reload/watch behavior, docs, and regression tests. Treat #27706 as related zero-restart context, not a replacement.
