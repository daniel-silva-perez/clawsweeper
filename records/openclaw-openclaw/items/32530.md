## Summary

Current main still does not implement automatic discovery of agent configurations from external workspaces. Agent configuration, validation, gateway enumeration, workspace listing, CLI docs, and config docs are still built around explicit `agents.list` entries plus the default agent; the related hot-reload path for `agents.list` does not add trusted workspace scanning or an agent-definition plugin/API contribution seam.

## Best Possible Solution

Keep this open for maintainer design. The best path is either a first-class `agents.autoDiscover` config contract or a documented plugin-backed agent-definition contribution API that runs before normal agent enumeration. It should define trusted roots, accepted filenames, schema validation, deterministic scan order, explicit precedence versus `agents.list`, watch/hot-reload behavior, docs, and regression tests. Treat #27706 and current `agents.list` hot reload as related context, not a substitute for external workspace discovery.
