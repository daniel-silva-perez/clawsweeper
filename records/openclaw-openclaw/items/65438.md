## Summary

Keep open. Current main and the latest release still lack configurable bootstrap dynamic-file classification and configurable bootstrap file order; the related #59050 ordering request was closed as duplicate of this issue, while #67419 only tracks broader context-bloat policy work.

## What This Changes

The issue requests config-driven bootstrap file ordering and dynamic-file placement below the system-prompt cache boundary to keep volatile workspace files from invalidating the stable prompt prefix.

## Best Possible Solution

Keep this issue as the canonical tracker and design one backwards-compatible config surface that preserves current defaults, keeps `HEARTBEAT.md` dynamic by default, lets operators add dynamic basenames, optionally controls ordering, and documents deterministic fallback behavior.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: any extra file such as `SESSION_HANDOFF.md` is not in the fixed dynamic set, so current prompt assembly renders it above `OPENCLAW_CACHE_BOUNDARY` with no supported config override.

## Solution Assessment

Unclear pending maintainer API review. The right end state is likely a single documented bootstrap prompt config surface, but the exact global/per-agent precedence and interaction with the stale #72520 implementation candidate should be settled before merging code.
