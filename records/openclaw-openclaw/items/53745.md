## Summary

Keep this issue open. Current main has adjacent fixes and diagnostics for Telegram polling stalls, per-session nested lanes, channel health monitoring, and event-loop liveness warnings, but I did not find source, tests, docs, changelog, or related-item evidence proving the reported combined failure mode is fixed: nested `sessions_send` activity followed by gateway-wide silence across all Telegram pollers and health-monitor timers. The recent affected-user comment is current field signal against conservative cleanup.

## Best Possible Solution

Keep this issue open until maintainers reproduce the scenario on current main with multiple Telegram bot accounts and coordinator-to-subagent `sessions_send`, or land/link a targeted fix with a regression proving Telegram polling and health-monitor timers continue after nested completion. The most useful next evidence is a multi-account Telegram liveness/integration test plus an affected deployment’s `openclaw gateway diagnostics export` bundle.
