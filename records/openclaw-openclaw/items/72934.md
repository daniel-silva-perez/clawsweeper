## Summary

Keep open. The report is recent and concrete, and current main still has a source-backed snapshot timeout gap: the browser tool exposes `timeoutMs`, and the gateway/node proxy can honor it when supplied, but the snapshot action and host snapshot client do not carry that timeout through to `/snapshot`, and the snapshot route omits timeout propagation into its Playwright/CDP snapshot calls.

## What This Changes

Request to fix browser automation so `snapshot` returns page content or an accessibility tree after successful browser open/navigate instead of consistently timing out.

## Best Possible Solution

Keep the issue open and repair the browser plugin snapshot path so an explicit or configured/default snapshot timeout is normalized once and carried consistently from the agent tool through the host client, gateway/node proxy, `/snapshot` route, and Playwright/CDP snapshot execution, with focused regression tests.

## Reproduction Assessment

Yes for triage: the report provides a precise open-then-snapshot sequence, affected version, platform, and repeated cross-URL failure. Current main source inspection independently shows that the exposed snapshot timeout path cannot affect `/snapshot`, although live macOS reproduction was not run.

## Solution Assessment

Yes: the narrow maintainable solution is to fix timeout and cancellation plumbing inside the browser plugin snapshot path. A core gateway rewrite or browser profile/default migration would be broader than the evidence supports.

## Repair Work Prompt

Fix the browser snapshot timeout regression described in https://github.com/openclaw/openclaw/issues/72934. Current main exposes top-level `timeoutMs`, and gateway/node proxy code can honor supplied timeouts, but `executeSnapshotAction`, `browserSnapshot`, and the `/snapshot` route do not carry a snapshot timeout through consistently. Implement a narrow browser plugin repair: normalize explicit `timeoutMs` for snapshot actions, use the existing configured/default browser action timeout when no explicit timeout is supplied where appropriate, pass the effective timeout through the direct host client and node proxy request, and carry it into Playwright/CDP snapshot execution including AI, role, ARIA, labels, and existing-session paths as far as supported. Add focused regression tests for host snapshot timeout propagation, node proxy snapshot timeout propagation, and route-level snapshot helper timeouts. Do not rewrite gateway routing, browser profile migration, node pairing, or unrelated navigate/load-state behavior; related issue #47203 may share timeout plumbing but should not broaden this PR unless the same helper change naturally covers it. Add a user-facing `CHANGELOG.md` fix entry with `Thanks @wkomers-hash`.
