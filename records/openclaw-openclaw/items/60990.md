## Summary

Keep this PR open. Current main still does not implement the macOS menubar stale-activity recovery that this PR is trying to add: `WorkActivityStore` has no watchdog, stale eviction, or disconnect cleanup path, so a missed terminal job event can still leave the menu bar activity state active indefinitely. The PR remains a plausible implementation candidate, but it needs revision for the stale-eviction test/logic concerns and for quiet long-running work before it should land.

## Best Possible Solution

Keep the PR open and ask for a focused revision: preserve the macOS app-side recovery path, but make stale cleanup safe for quiet long-running work, inject a clock or test hook so eviction tests age activity explicitly, add coverage for missed terminal events and gateway disconnect cleanup, and keep the change scoped to `WorkActivityStore` plus macOS tests unless a generic gateway lifecycle fix is proven necessary.
