## Summary

Keep open. Current main still lacks the requested Anthropic-native compaction controls, Anthropic `context_management.edits` request wiring, compaction block round-trip handling, and mixed long-stable/short-trailing Anthropic cache TTL policy; related open trackers overlap but do not fully replace this issue's mixed TTL requirement.

## What This Changes

The issue requests opt-in Anthropic-native server-side compaction plus mixed prompt-cache TTL handling so stable prefix content can use 1h retention while high-churn conversation content stays at 5m.

## Best Possible Solution

Keep this issue open unless maintainers explicitly fold its mixed Anthropic cache-TTL acceptance criteria into a canonical Anthropic context-management tracker. The desired end state is a documented, opt-in Anthropic transport/cache-policy implementation that gates beta headers only when needed, injects Anthropic `context_management.edits`, preserves compaction blocks through streaming and replay, and applies long cache retention only to stable prefix material while trailing conversation remains default/short TTL.
