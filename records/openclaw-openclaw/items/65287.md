## Summary

Keep open. Current main still lacks Anthropic native compaction params, `context_management.edits` injection, required compaction beta handling, and compaction-block round-trip support. The existing Anthropic cache policy also does not meet the requested mixed TTL behavior because `cacheRetention: "long"` applies the same 1-hour marker to trailing user content. The closed companion PR #65288 was not merged, and open #10213 covers server-side compaction but not the mixed TTL acceptance criteria.

## Best Possible Solution

Keep this issue open, or explicitly merge its mixed-TTL acceptance criteria into #10213 before closing it as a duplicate. The implementation likely belongs in the Anthropic provider/transport path: add typed extra params, gate Anthropic beta headers, inject `context_management.edits`, preserve compaction blocks through history and streaming, and update the cache policy/tests so 1h applies only to the stable prefix while trailing conversation remains short/default TTL.
