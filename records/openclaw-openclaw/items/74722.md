## Summary

Keep open. Current main still lacks language-aware TTS routing: the TTS config/schema has no routing contract, `auto: "inbound"` only gates whether audio is generated, provider selection remains prefs/persona/config/default based, and STT/media context does not preserve detected language for downstream TTS routing.

## What This Changes

Add language-aware routing for automatic TTS replies so the selected TTS provider, model, and voice can follow the detected input or reply language.

## Best Possible Solution

Design a backward-compatible `messages.tts.languageRouting` contract in the public TTS config, carry optional detected STT or text language metadata through inbound context, apply the selected route before speech synthesis while preserving existing global, per-agent, channel, account, prefs, and directive precedence, then document and test fallback behavior.
