## Summary

Keep open. Current main now handles the specific `process.poll` busy-wait abort path, but it still has no framework-level tool-execution race/safety net and `sessions_send`/gateway waits remain timeout-only, so the broader runtime contract is not implemented.

## What This Changes

The issue requests a framework-level and per-tool design for abort-aware tool execution, including process polling interruption, cancellable `sessions_send` gateway calls, steer-aware yield semantics, partial-result transcript behavior, and cleanup rules.

## Best Possible Solution

Keep this as a core agent-runtime design item. The end state should be an explicit tool interruption contract covering framework fallback behavior, which tools should return partial results, how aborted tool results appear in transcript history, and how gateway-backed tools such as `sessions_send` receive cancellable request plumbing without killing resumable background work.
