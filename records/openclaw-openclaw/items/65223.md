## Summary

Keep open. Current main has abort-signal plumbing and a prompt-level abort wrapper, and the `process.poll` busy-wait example is now fixed, but the broader framework-level tool interruption contract is still missing: upstream tool execution is awaited without a race, `sessions_send` ignores the tool signal, and Gateway calls remain timeout-only.

## What This Changes

Request a core agent-runtime design and implementation for abort-aware tool execution, including framework fallback behavior, per-tool partial results, steer semantics, and cancellable gateway-backed waits.

## Best Possible Solution

Keep this as a core agent-runtime design item. The desired end state is an explicit tool interruption contract that defines framework fallback behavior, which tools should return partial results, how aborted tool results appear in transcript history, how steer-aware yielding differs from abort, and how Gateway-backed tools such as `sessions_send` receive cancellable request plumbing.
