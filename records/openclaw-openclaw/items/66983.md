## Summary

Keep this issue open. Current main still lacks a first-class web/browser node platform and still intentionally rejects device-less node-role browser sessions; the related #68497 tracks a narrower browser action bridge and does not supersede browser-tab node registration.

## What This Changes

The issue requests first-class browser-tab A2UI canvas nodes, including an explicit web node platform policy, browser-compatible node pairing/auth, and optional docs for web canvas nodes.

## Best Possible Solution

Define an explicit web canvas-node contract in the Gateway: a narrow web platform allowlist for A2UI canvas commands, a scoped browser-node enrollment/auth model for trusted proxy deployments, regression coverage for Origin and node-role auth behavior, and public docs for the supported browser canvas-node setup.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: role: "node" with no device identity reaches reject-device-required, and browser-origin node auto-approval is explicitly rejected in policy and tests.

## Solution Assessment

No, not as a blanket extension of dangerouslyDisableDeviceAuth or shared-token trust. The maintainable path is a deliberate web platform plus scoped browser-node pairing/bootstrap design reviewed against the gateway auth model.
