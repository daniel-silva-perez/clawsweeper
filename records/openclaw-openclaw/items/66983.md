## Summary

Keep this issue open. Current main still lacks first-class web canvas node support: `web` is not a recognized node platform, browser-origin node pairing remains explicitly gated, shared token auth does not admit device-less node-role sessions, and the related web action-bridge request is narrower than the platform/auth/pairing work requested here.

## Best Possible Solution

Keep this open as the canonical design and implementation request for browser tabs acting as A2UI canvas node surfaces. A good fix should add an explicit `web` node platform policy with canvas commands, define a secure browser node pairing/auth story for reverse-proxied deployments, and add tests for browser Origin handling, trusted proxy behavior, device identity or bootstrap-token semantics, command allowlisting, and node.invoke delivery. Coordinate with #68497 for the separate web action bridge, but do not treat it as a replacement for this issue.
