## Summary

Keep this PR open. Current main still does not implement the explicit `tools.exec.shell` global/per-agent config contract, and the proposed patch remains incomplete for important exec paths such as PTY supervision and gateway approval follow-up runs. The related open issue #49931 is still the better product/design anchor for the remaining work; #65271 was closed unmerged and #59998 was closed as related/duplicate context rather than an implementation on main.

## Best Possible Solution

Keep this PR open for maintainer review or explicitly consolidate it into #49931. A complete implementation should define the supported shell override contract, add typed global and per-agent config plus schema/docs/help, route the selected shell consistently through direct exec, PTY supervisor, approval follow-up runs, sandbox/node policy decisions, and add focused tests for precedence, fallback, Windows arguments, validation, approval paths, and schema/docs drift.

## Risks / Open Questions

- The proposed diff changes the command-execution shell surface but does not propagate a shell override through PTY supervision or gateway approval follow-up runs, so behavior could differ depending on execution path.
- A custom shell override interacts with allowlists, safeBins, PATH restrictions, sandbox/node routing, Windows/WSL path translation, quoting, and validation. Those policy interactions need explicit maintainer design and focused tests before merge.
- No supply-chain or CI mutation was present in the provided PR file list, but the runtime command-execution semantics are security-sensitive enough that this should not be treated as a trivial config passthrough.
