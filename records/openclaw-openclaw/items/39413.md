## Summary

Keep open. Current main still does not implement the PR's explicit `tools.exec.shell` global/per-agent config contract or an equivalent shell override for exec. The newer linked shell work is not present on current main, and the open #49931 issue remains the better product/design anchor for the remaining work. The submitted patch is useful context, but it is not merge-ready as-is because important exec paths would still use the default shell.

## Best Possible Solution

Keep this PR open for maintainer review or explicit consolidation into #49931. A complete solution should define the supported shell override contract, add typed global and per-agent config plus schema/docs/help, route the selected shell consistently through direct exec, PTY supervisor, approval follow-up runs, sandbox/node policy decisions, and add focused tests for precedence, fallback, Windows arguments, validation, approval paths, and schema/docs drift.
