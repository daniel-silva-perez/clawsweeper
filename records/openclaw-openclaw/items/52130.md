## Summary

Keep open. Current main has meaningful hardening around service restart limits, Telegram doctor SecretRef handling, browser profile isolation, and multi-state-dir warnings, but the reported `channels.telegram.retry.jitter: true` config contract is still not implemented or tested as accepted/migrated. The related fix PR #52137 was closed unmerged in the provided context, so there is no canonical merged fix to close against.

## Best Possible Solution

Keep this issue open or split it. The remaining concrete maintainer path is to either accept and coerce boolean `channels.telegram.retry.jitter` values with focused schema/regression coverage, or explicitly decide numeric-only and add doctor/config repair guidance that converts or flags the boolean value before startup. After that lands, the already-present service, doctor SecretRef, browser profile, and multi-state-dir hardening can be cited in a final implemented-on-main close review.
