## Summary

Keep open. The PR addresses a real duplicate-reply prompt cause, but its unconditional final-response guidance conflicts with current main's default `message_tool_only` delivery mode for Discord guild/channel turns, so it can make default Discord channel or autoThread replies private instead of visible.

## What This Changes

The PR changes Codex developer instructions, the bundled Discord skill, and a Codex thread-lifecycle test so normal current-conversation replies are steered to final assistant delivery instead of `message` tool sends.

## Best Possible Solution

Revise the prompt and skill wording to follow the resolved visible-reply mode: final-response guidance when automatic source delivery is active, and message-tool guidance when final replies are private/message-tool-only. Keep Discord delivery mechanics unchanged and add regression coverage for both modes.

## Reproduction Assessment

Yes. The original duplicate path has concrete Discord `autoThread: true` steps, and the PR-introduced regression is reproducible from current main's default channel delivery contract: Discord guild turns resolve to `message_tool_only`, which suppresses automatic final delivery.

## Solution Assessment

No. The proposed prompt direction is useful for automatic delivery, but the unconditional wording is not the best fix because it contradicts the current default Discord channel delivery policy; the safer fix is mode-aware wording plus tests.
