## Summary

Keep open. The PR targets a real Discord duplicate-reply prompt issue, but the proposed unconditional final-response guidance conflicts with current main's newer default that keeps Discord guild/channel final replies private unless visible replies are explicitly configured automatic.

## What This Changes

The PR changes Codex developer instructions, the bundled Discord skill, and a Codex thread-lifecycle test so normal current-conversation replies are steered to final assistant delivery instead of `message` tool sends.

## Best Possible Solution

Keep the PR open and revise the prompt/skill wording so it follows the resolved visible-reply mode: final-response guidance only when the current source conversation is actually delivered automatically, and message-tool guidance when final replies are private/message-tool-only. Add regression coverage for both automatic autoThread delivery and default private guild/channel delivery.

## Reproduction Assessment

Yes. The original duplicate path has concrete Discord `autoThread: true` steps in the PR body and #73278, and the PR-introduced regression can be traced on current main through the default `message_tool_only` channel delivery path without needing live Discord access.

## Solution Assessment

No. The proposed direction fixes a real prompt cause, but the unconditional final-response wording is not the narrowest maintainable fix on current main because it contradicts the source-reply delivery policy for default Discord channel turns.
