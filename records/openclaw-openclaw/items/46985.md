## Summary

Keep this PR open. Current main still stores cumulative WebChat stream snapshots as committed segments at tool boundaries and renders those segments plus the live stream independently, so the duplicate text problem around tool cards is still present. The PR is now focused on the UI streaming fix, includes regression coverage for repeated identical segments, and the related open #47377 is not merged or clearly designated as the canonical replacement.

## Best Possible Solution

Keep this PR open while maintainers choose the implementation path. The best product fix is a focused main-branch change that stores only the uncommitted suffix when a tool starts, dedupes the live preview against already committed stream segments, and keeps regression coverage for repeated identical segments plus multi text/tool/text cycles. If #47377 becomes the preferred replacement, close this PR only after that replacement lands or a maintainer explicitly marks it superseded.
