## Summary

Current main still has the same same-channel block-streaming delivery gap: block replies are queued on the dispatcher, but `sendBlockReply` returns `boolean` and the same-channel `dispatch-from-config` path does not await dispatcher delivery before tool execution can continue. The issue has independent WhatsApp, Telegram, Discord, and WeCom confirmations; the direct fix PRs in the timeline were closed unmerged, and the open adjacent PR #65541 covers disabled block streaming rather than this enabled `text_end` same-channel await contract.

## Best Possible Solution

Keep this issue open as the canonical core bug. The best fix is a focused core change that makes same-channel block delivery awaitable, awaits accepted block delivery before tool execution proceeds for `blockStreamingBreak: "text_end"`, preserves the existing awaited cross-channel path, updates dispatcher/ACP/plugin SDK contracts and mocks consistently, and adds a regression test proving the actual channel `deliver` promise completes before the next tool starts.

## Clownfish Work Prompt

Fix the current main bug where block streaming replies with `agents.defaults.blockStreamingBreak: "text_end"` are not delivered to the same channel before the next tool starts. Current code awaits the block pipeline callback, but the same-channel `dispatch-from-config` callback only enqueues `dispatcher.sendBlockReply(...)`; `ReplyDispatcher.sendBlockReply` returns boolean and actual channel delivery happens later on the dispatcher's `sendChain`.

Expected boundary: make same-channel accepted block delivery awaitable and await it before the `onBlockReply` callback resolves. Preserve the already-awaited cross-channel `sendPayloadAsync` path, preserve dropped/empty payload behavior (`false`), preserve dispatcher ordering/error handling, and update ACP/direct delivered accounting that only needs boolean tracking. Do not mix this with PR #65541's disabled-block-streaming fallback, #26224 message-tool batching, #42106 paragraph separator handling, or plugin-specific WeCom media parsing.
