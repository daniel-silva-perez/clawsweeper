## Summary

Keep this issue open. Current main partially supports Copilot vision-capable dynamic models and explicit generic image-model paths, but it still does not register `github-copilot` as a media-understanding image provider or include it in the automatic image fallback matrix requested by the issue. The prior ClawSweeper keep-open comment remains consistent with current `main` at 230f7122dd5cd8a37a55e671254bd9be8278f6e8.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the bundled GitHub Copilot plugin: declare `contracts.mediaUnderstandingProviders` plus `mediaUnderstandingProviderMetadata` for `github-copilot`, route image understanding through the existing generic image runtime or a Copilot-specific provider wrapper as needed, choose a safe default model/priority, add registry and auto-routing tests, and update the GitHub Copilot plus media-understanding docs. If maintainers only want explicit Copilot image use, document `agents.defaults.imageModel` or `tools.media.image.models` as the supported path instead.
