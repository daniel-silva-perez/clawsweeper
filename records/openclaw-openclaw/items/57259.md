## Summary

Keep this issue open. Current main still has partial Copilot vision support through explicit/dynamic model resolution, but it does not register `github-copilot` as a media-understanding image provider or include it in the automatic bundled image fallback matrix the issue asks for.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the bundled GitHub Copilot plugin: add `contracts.mediaUnderstandingProviders` and `mediaUnderstandingProviderMetadata` for `github-copilot`, route image understanding through the existing generic image runtime or a Copilot-specific wrapper if needed, choose a safe default model and auto priority, add registry/auto-routing coverage, and update the GitHub Copilot plus media-understanding docs. If maintainers only want explicit Copilot image use, document `agents.defaults.imageModel` or `tools.media.image.models` as the supported path instead.
