## Summary

Keep this issue open. Current main has Copilot vision-capable dynamic model metadata and explicit generic image-model paths, but it still does not register `github-copilot` as a media-understanding image provider or include it in the automatic provider fallback matrix the issue asks for.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the bundled GitHub Copilot plugin: add media-understanding image provider metadata or registration backed by the existing generic image runtime, choose a safe default/priority strategy, add focused registry and auto-routing tests, and update the media-understanding plus GitHub Copilot provider docs. If maintainers only want explicit Copilot image use, document `tools.media.image.models` or `agents.defaults.imageModel` as the supported path instead of closing this as implemented.
