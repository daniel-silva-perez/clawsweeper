## Summary

Keep this PR open. Current main still has the direct webchat/TUI sender metadata prompt leak, the patch is a narrow implementation candidate for that behavior, and the related closed/open items do not supersede it; the remaining action is maintainer review and validation.

## What This Changes

The PR adds a guard in `buildInboundUserContextPrefix` to omit sender metadata for direct webchat contexts and adds a regression test for `gateway-client` / `openclaw-tui` sender fields.

## Best Possible Solution

Land the focused prompt-context fix after maintainer review, preserving sender metadata for group and external-channel contexts while suppressing transport-only sender blocks from direct webchat/TUI turns.

## Reproduction Assessment

Yes. A direct `webchat` `TemplateContext` with `SenderId`/`SenderName` reproduces by code trace on current main: conversation info is suppressed, but the sender block still renders because it is gated only by `senderInfo?.label`.

## Solution Assessment

Yes. The PR's prompt-helper guard is the narrowest maintainable fix because it avoids removing gateway sender identity from internal context while leaving external-channel and group metadata paths intact.
