## Summary

Keep this PR open. Current main still emits the `Sender (untrusted metadata)` block for direct webchat/TUI contexts, and the PR is a narrow implementation candidate that matches the reported transport-identity leak without superseding evidence from related items.

## What This Changes

The PR adds a `buildInboundUserContextPrefix` guard to omit sender metadata for direct webchat contexts and adds a regression test for `gateway-client` / `openclaw-tui` sender fields.

## Best Possible Solution

Land the focused prompt-context fix after maintainer review, preserving sender metadata for group and external-channel contexts while suppressing transport-only sender blocks from direct webchat/TUI turns.

## Reproduction Assessment

Yes. A direct `webchat` `TemplateContext` with `SenderId`/`SenderName` reproduces by code trace on current main: conversation info is suppressed, but the sender block still renders because it is gated only by `senderInfo?.label`.

## Solution Assessment

Yes. The PR's guard is the narrowest maintainable fix in the prompt-context helper because it reuses the same direct webchat channel resolution already used for conversation-info suppression while leaving external-channel and group metadata paths intact.
