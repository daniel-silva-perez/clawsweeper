## Summary

Keep this issue open. Current main still sends Feishu streaming replies as CardKit `card_id` interactive messages, but the quoted-message and thread-history read paths still parse only inline card content and do not request or parse raw CardKit `json_card` content. The related implementation PR #55466 was closed unmerged, so it remains useful prior art rather than a shipped fix.

## Best Possible Solution

Keep this issue open. The likely fix belongs in the bundled Feishu plugin read path: request validated raw CardKit content for both `getMessageFeishu()` and `listFeishuThreadMessages()`, parse `json_card` summary/body text before the existing inline element/post fallback, and add regression coverage for quoted-message and thread-history reads of streaming CardKit cards. #55466 is useful prior art but needs revival or rework against current Feishu/Lark API behavior.
