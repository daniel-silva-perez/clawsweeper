## Summary

Keep this issue open. Current main still sends Feishu streaming replies as CardKit `card_id` interactive messages, while both quoted-message and thread-history reads still parse only inline card content and do not request raw CardKit content. The directly related PR #55466 is closed unmerged, and the overlapping open PR #56795 is useful follow-up work but not a shipped fix.

## What This Changes

The issue asks OpenClaw’s bundled Feishu plugin to read actual text from streaming CardKit interactive-card replies instead of exposing `[Interactive Card]` to other agents.

## Best Possible Solution

Land a Feishu-owned read-path fix that requests validated raw CardKit content for both `getMessageFeishu()` and `listFeishuThreadMessages()`, parses the returned card summary/body text before the existing inline-card/post fallback, and adds focused regression tests for quoted-message and thread-history reads of streaming CardKit cards. Coordinate with #56795 and #55466 so only one implementation path moves forward.
