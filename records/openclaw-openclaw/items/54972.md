## Summary

Keep open. Current main still has the reported Feishu readback gap: streaming replies are sent as CardKit `card_id` interactive messages, while quoted-message and thread-history reads do not request raw CardKit content and still parse only inline card payloads before falling back to `[Interactive Card]`. The related raw-card PR is closed unmerged, and the overlapping open PR still needs maintainer review rather than an automatic close.

## What This Changes

Make Feishu streaming CardKit replies readable by other agents when quoted or included in thread history instead of returning `[Interactive Card]`.

## Best Possible Solution

Land one Feishu-owned read-path fix that requests validated raw CardKit content for both `getMessageFeishu()` and `listFeishuThreadMessages()`, parses returned summary/body text such as `json_card` before the existing inline-card and post fallbacks, and adds focused regression tests for quoted-message and thread-history reads of streaming CardKit cards. Coordinate #56795 and the prior #55466 approach so maintainers choose one implementation path.
