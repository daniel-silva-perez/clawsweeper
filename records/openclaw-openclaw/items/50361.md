## Summary

Keep open. Current main still deduplicates Feishu final replies by raw string equality, so the PR's normalized final-text dedupe for CRLF/LF, trailing whitespace, and zero-width-character variants is not implemented or superseded. The PR is a focused Feishu plugin candidate, and its file scope does not introduce an obvious security or supply-chain concern.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or recreate the focused Feishu dispatcher change on current main: add a normalized final-text fingerprint for both delivered-final recording and duplicate-final lookup, keep the behavior owned by the Feishu plugin, and retain the regression coverage for CRLF/LF, trailing whitespace, and zero-width-character variants.
