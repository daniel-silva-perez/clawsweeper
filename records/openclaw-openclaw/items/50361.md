## Summary

Keep open. Current main still suppresses Feishu duplicate finals by raw string equality only, so this PR's normalization for CRLF/LF, trailing whitespace, and zero-width characters is not implemented on main. The PR remains a focused candidate fix rather than obsolete work.

## Best Possible Solution

Review and either land this focused Feishu dispatcher change or recreate it on current main: add a normalized final-text fingerprint for duplicate detection and recording, keep it scoped to delivered final text keys, and add the proposed regression test for CRLF/trailing-whitespace/zero-width variants. Do not close as implemented until that behavior exists on main.
