## Summary

Current main still deduplicates Feishu final replies by raw string equality, so this PR's normalized final-text dedupe is not implemented. The PR remains a focused candidate fix rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or recreate the focused Feishu dispatcher change on current main: add a normalized final-text fingerprint for both delivered-final recording and duplicate-final lookup, keep the scope inside the Feishu plugin, and add the proposed regression coverage for CRLF/LF, trailing whitespace, and zero-width-character variants.
