## Summary

Keep this draft PR open. Current main still deduplicates Feishu final replies by raw string equality, so the PR's normalized dedupe for CRLF/LF, trailing whitespace, outer whitespace, and zero-width-character variants is not implemented or superseded. The proposed diff is scoped to the Feishu plugin dispatcher and its regression test, with no observed security or supply-chain-sensitive file changes.

## Best Possible Solution

Keep this draft PR open for maintainer review. The best path is to land or recreate the focused Feishu dispatcher change on current main: add a Feishu-owned normalized final-text fingerprint for both delivered-final recording and duplicate-final lookup, keep distinct final payloads deliverable, and retain regression coverage for CRLF/LF, trailing whitespace, outer whitespace, and zero-width-character variants.
