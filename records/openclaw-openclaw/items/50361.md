## Summary

Keep open. Current main still deduplicates Feishu final replies by raw string equality, so this PR's normalization for line endings, trailing whitespace, and zero-width characters is not implemented. The PR remains a focused candidate fix rather than obsolete work.

## Best Possible Solution

Review and either land this focused Feishu dispatcher change or recreate it on current main: add a normalized final-text fingerprint for duplicate detection and recording, keep it scoped to delivered final text keys, and add the proposed regression test for CRLF, trailing-whitespace, and zero-width-character variants. Do not close as implemented until that behavior exists on main.
