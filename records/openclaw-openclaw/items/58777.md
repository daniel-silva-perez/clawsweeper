## Summary

Keep open. Current main has partial fallback/result metadata, but it does not implement the requested closed-loop effective model/provider/thinking telemetry for sub-agent pin fidelity, and related hook/router fallback semantics are still open rather than superseding this report.

## What This Changes

The issue requests that sub-agent runs expose the effective model, provider, and thinking level after fallback or router rewrites so explicit model pins can be audited.

## Best Possible Solution

Define an additive effective-run metadata contract, populate it from the final runtime provider/model/thinking after fallback and hook selection, reconcile child-session/task/lifecycle surfaces, and add regression tests for explicit sub-agent pins, fallback to another model, and unsupported-thinking re-clamping.
