## Summary

Keep open. Current main has made useful progress around sub-agent model selection and fallback persistence, but it does not implement the issue’s central requested/effective telemetry contract for model, provider, or thinking fidelity.

## Best Possible Solution

Keep this issue open and implement a narrow observability contract: add optional requested_model/requested_provider/requested_thinking and effective_model/effective_provider/effective_thinking fields to the relevant run, model usage, model call, and lifecycle events; populate them after fallback resolution; and re-derive or validate thinking when fallback changes the effective provider/model capabilities. Preserve the existing sub-agent model persistence behavior and add regression coverage for a fallback from a pinned sub-agent model.
