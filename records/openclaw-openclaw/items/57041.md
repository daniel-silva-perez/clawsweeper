## Summary

Keep this PR open. Current main has adjacent fixes for ended orchestrators with pending descendants, but it still does not implement the PR’s cleanup-pending active/completing state. An ended run with no `cleanupCompletedAt` and no descendants still formats as `done` and is not counted active, which is the remaining behavior this PR is trying to change.

## Best Possible Solution

Review and either land or request a rebase of this PR’s core behavior against current main: add a shared cleanup-pending predicate using `cleanupCompletedAt` plus `suppressAnnounceReason`, apply it consistently to subagent list/status/target resolution/active-count paths, and add regression tests proving cleanup-pending ended runs show `completing` and count active until cleanup completes.
