## Summary

Keep this PR open. Current main still uses the existing `tar.c` backup path and has no archive-time ENOENT/ENOTDIR skip handling for session files deleted during packing. The PR is a focused implementation candidate for #67417, but it still needs revision/proof around manifest consistency, tar stream semantics, and test coverage before it should land or be superseded by #72251.

## Best Possible Solution

Keep this PR open and request revision, or land an equivalent superseding fix. The maintainer path should handle archive-time ENOENT/ENOTDIR for volatile session files in `src/infra/backup-create.ts`, keep manifest/result/verify semantics consistent for omitted payloads, add a regression test that simulates a session transcript disappearing during packing, and coordinate explicitly with #67417 and #72251 before closing either implementation candidate.
