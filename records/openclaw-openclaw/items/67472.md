## Summary

Keep this PR open. Current main still uses the existing `tar.c` archive path and does not implement archive-time ENOENT/ENOTDIR skipping for session files deleted during backup creation. The PR remains a focused candidate for #67417, but the supplied review context and current code show unresolved manifest consistency, test-mock fidelity, tar stream contract, and overlap questions with #72251.

## Best Possible Solution

Keep this PR open for revision or an equivalent superseding fix. The maintainer path should implement archive-time ENOENT/ENOTDIR resilience for volatile session files in `src/infra/backup-create.ts`, keep manifest/result/verify semantics consistent for omitted payloads, add a regression test that simulates a session transcript disappearing during packing, and coordinate explicitly with #67417 and #72251 before closing either implementation path.
