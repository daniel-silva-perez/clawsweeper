## Summary

Keep this PR open, but do not merge it as-is. Current main still has an unguarded gateway `agents.delete` path, while the PR's exact-path helper is narrower than the overlap-aware CLI behavior that already shipped, so the stale branch can still delete parent/child shared workspace state and may regress the CLI contract.

## What This Changes

The PR adds an exact realpath-normalized shared-workspace lookup, applies it to CLI identity/delete and gateway agent deletion, and adds tests for preserving exact shared workspaces.

## Best Possible Solution

Share a realpath-aware bidirectional workspace-overlap helper between CLI and gateway deletion, preserve the existing CLI JSON behavior, and add gateway coverage for same, parent, child, symlink, and unique workspace cases.
