## Summary

Keep this PR open. Current main has a broader CLI fix for shared/overlapping workspaces, but the gateway/UI `agents.delete` path still unconditionally trashes the workspace, while this PR explicitly intended to protect both CLI and gateway delete paths.

## Best Possible Solution

Keep this PR open or replace it with a fresh maintainer fix that shares the current CLI overlap/symlink workspace-retention logic with the gateway `agents.delete` handler, adds a gateway regression test for shared workspaces, and keeps the existing CLI JSON/docs behavior intact.
