## Summary

Keep open. Current main protects the CLI delete path, but the gateway `agents.delete` path still trashes the workspace unconditionally when `deleteFiles` is true. The PR is useful source work, but its exact-match helper is too narrow compared with the overlap and symlink semantics already shipped for the CLI path.

## What This Changes

The PR adds exact shared-workspace detection and uses it in CLI, identity, and gateway agent deletion so exact shared workspaces are preserved while agent-specific state is removed.

## Best Possible Solution

Extract or reuse a realpath-aware workspace-overlap helper that covers same, parent, child, and symlink paths, apply it to gateway `agents.delete`, keep the shipped CLI JSON behavior intact, and add gateway regression coverage for shared, nested, parent, symlink, and unique workspace cases.
