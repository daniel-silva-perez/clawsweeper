## Summary

Keep this PR open because it targets a real remaining gateway data-loss path, but the current branch should not merge as-is: current main already has broader CLI overlap protection, while the PR’s exact-only guard would still allow parent/child overlapping workspaces to be trashed.

## What This Changes

The PR adds an exact realpath-normalized workspace lookup, uses it in CLI identity/delete and gateway agent deletion, and adds tests for exact shared-workspace preservation.

## Best Possible Solution

Move the overlap-aware, realpath-normalized workspace retention logic into a shared helper and use it from gateway `agents.delete`, while preserving the current CLI contract and adding gateway regression coverage for same, parent, child, symlink, and unique workspace cases.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is two configured agents with overlapping workspaces followed by gateway `agents.delete` with `deleteFiles` true; current main still trashes `workspaceDir`, and existing CLI tests show the expected retention cases.

## Solution Assessment

No. Exact realpath equality is too narrow for deletion safety; the maintainable fix is the current CLI’s bidirectional overlap rule shared with the gateway path.
