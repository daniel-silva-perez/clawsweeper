## Summary

Keep this PR open. Current main has the shared/overlapping workspace guard for the CLI `agents delete` path, but the gateway/UI `agents.delete` handler still moves `workspaceDir` to Trash unconditionally when `deleteFiles` is true. This PR's stated scope includes both CLI and gateway delete paths, so main has not implemented the full intended change.

Keep this PR open or replace it with a fresh targeted fix that applies the current CLI overlap/symlink workspace-retention semantics to the gateway `agents.delete` handler, adds gateway regression coverage for shared and overlapping workspaces, and leaves the shipped CLI JSON/docs behavior intact.

## Clownfish Work Prompt

Fix the remaining shared-workspace deletion gap for gateway/UI agent deletion. Current main protects the CLI `openclaw agents delete` path from trashing same, parent/child, or symlink-overlapping workspaces still used by another agent, but `src/gateway/server-methods/agents.ts` still calls `moveToTrashBestEffort(workspaceDir)` unconditionally when `deleteFiles` is true. Implement an additive, focused fix for gateway `agents.delete`: reuse or extract the current CLI overlap/symlink semantics rather than adding an exact-match-only helper, skip trashing `workspaceDir` when another configured agent overlaps it, and continue trashing agent-specific state and session transcripts. Add gateway regression coverage in `src/gateway/server-methods/agents-mutate.test.ts` for at least same-workspace retention and preferably parent/child or symlink overlap if the test seam can cover it cleanly. Do not change the shipped CLI JSON fields or docs behavior, do not touch dependencies/workflows/release scripts, and keep any gateway response changes additive if retention metadata is returned. Add a single-line changelog fix entry only if the user-facing gateway/UI behavior changes, crediting the appropriate reporter/contributor such as @gautamtayal1 and/or @Torinbob, never Codex.
