## Summary

Keep open: current main still leaves the gateway `agents.delete` path unprotected, and this PR's exact-match guard is not merge-ready because it does not preserve the broader same, parent, child, and symlink overlap semantics already shipped for the CLI delete path.

## What This Changes

The PR adds exact shared-workspace detection to agent scope helpers and uses it in CLI and gateway agent deletion so exact shared workspaces are preserved while agent-specific state is removed.

## Best Possible Solution

Land an update or narrow replacement that extracts or reuses the current CLI realpath-aware overlap logic for gateway `agents.delete`, keeps agent-specific state deletion unchanged, preserves the shipped CLI JSON behavior, and adds gateway regression coverage for same, nested, parent, symlink, and unique workspace cases.
