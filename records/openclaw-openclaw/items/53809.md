## Summary

Keep this PR open. Current main has shipped the CLI-side workspace retention fix, but the gateway `agents.delete` path still trashes the resolved workspace unconditionally, and its tests do not cover shared or overlapping workspaces. The PR is useful source work, but it needs to be updated to match the broader same/parent/child/symlink retention semantics now present in the CLI path.

## What This Changes

The PR adds an exact-workspace ownership helper, applies it to CLI delete, gateway `agents.delete`, and `agents set-identity`, and adds unit/RPC tests for exact shared-workspace preservation.

## Best Possible Solution

Update or replace the PR with a narrow gateway fix that reuses or extracts the CLI overlap/realpath workspace-retention logic before `agents.delete` trashes a workspace, adds gateway regression coverage for same, nested, parent, symlink, and unique workspace cases, and leaves the shipped CLI JSON/docs behavior intact.
