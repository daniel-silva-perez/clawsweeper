## Summary

Keep open: the gateway data-loss bug remains on current main, but this PR should not merge as-is because its exact-only workspace guard is narrower than the shipped CLI behavior and would miss parent/child workspace overlap cases.

## What This Changes

The PR adds exact shared-workspace detection, threads it through CLI identity/delete and gateway agent deletion, and adds tests for preserving same-path shared workspaces while deleting agent-specific state.

## Best Possible Solution

Extract or share a realpath-aware workspace-overlap helper, apply it to gateway `agents.delete`, preserve the existing CLI JSON contract, and add gateway coverage for same, nested, parent, symlink, and unique workspace cases.
