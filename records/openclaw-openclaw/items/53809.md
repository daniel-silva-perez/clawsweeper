## Summary

Keep open. Current main fixes the CLI side of the shared-workspace deletion bug, but the gateway `agents.delete` path still trashes the workspace unconditionally when file deletion is enabled, so this PR is not obsolete and should not be closed as implemented. It also should not merge as-is because its exact-path helper is narrower than the shipped CLI overlap behavior.

## What This Changes

The PR adds exact-workspace sharing checks to agent scope, CLI `agents delete`, gateway `agents.delete`, `agents set-identity`, and related tests so deleting one agent can preserve a workspace still used by another agent.

## Best Possible Solution

Update or replace the PR with a gateway fix that reuses or extracts the CLI overlap/realpath workspace-retention logic before `agents.delete` trashes a workspace, adds gateway regression coverage for same, nested, parent, symlink, and unique workspace cases, and preserves the shipped CLI behavior.
