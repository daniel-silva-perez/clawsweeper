## Summary

Keep this PR open, but it should not merge as-is. Current main already shipped the broader CLI workspace-retention behavior, while the gateway deletion path still lacks a guard; this PR addresses the right remaining surface but its exact-path helper misses parent/child workspace overlaps and risks regressing the shipped CLI behavior.

## What This Changes

The PR adds an exact realpath-normalized workspace lookup, uses it in CLI identity/delete and gateway agent deletion, and adds tests for exact shared-workspace preservation.

## Best Possible Solution

Move the overlap-aware, realpath-normalized workspace retention logic into a shared helper and use it from gateway `agents.delete`, while preserving the existing CLI JSON/output contract and adding gateway regression coverage for same, parent, child, symlink, and unique workspace cases.
