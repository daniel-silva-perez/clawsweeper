## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks timeout propagation through the shared remote batch HTTP helpers and provider batch upload/create/status/error-read/download paths, while the PR introduces a real fail-open timeout resolver path that can turn an exhausted supplied budget into an unbounded HTTP request.

## What This Changes

Threads the configured remote embedding batch timeout budget through memory-host SDK HTTP helpers and OpenAI, Gemini, and Voyage batch phases, with docs, changelog, SDK exports, and focused provider/SDK tests.

## Best Possible Solution

Keep the narrow timeout-budget port, but change the shared resolver/retry behavior so a supplied expired budget fails closed, preserve the #49981 credit, and land this PR or an equivalent replacement after focused SDK/provider tests and the changed gate pass.

## Reproduction Assessment

Yes. Source inspection on current main shows the configured batch timeout reaches provider `batchEmbed` but is not accepted by the shared HTTP helpers or forwarded by OpenAI/Gemini/Voyage batch HTTP calls; the PR finding is reproducible by passing a timeout function that returns `0` into `postJsonWithRetry`.

## Solution Assessment

No, not as-is. The implementation direction is the right narrow fix, but the shared timeout resolver should not convert an explicitly supplied exhausted budget into `undefined` because that makes the HTTP request unbounded.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Fail closed when a supplied timeout budget is exhausted:** `packages/memory-host-sdk/src/host/remote-http.ts:18-19`
  - body: `resolveRemoteHttpTimeoutMs` returns `undefined` for `<= 0` even when the caller supplied a budget. In `postJsonWithRetry`, that makes exhausted budgets skip the budget-aware retry loop and continue through `retryAsync`, where `postJson` resolves the same value to no `fetchWithSsrFGuard` timeout. A provider/plugin timeout function that returns `0` on expiry can hang instead of failing closed.
  - confidence: 0.87
