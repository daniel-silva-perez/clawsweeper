## Summary

Keep open. The Unicode tokenizer bug remains in current main's active `memory-core` implementation, while this PR still targets obsolete `src/memory/*` files. Current MMR still uses ASCII tokens plus CJK-only unigrams/bigrams, current FTS tokenizers omit `\p{M}` and NFC normalization, and tests do not cover the accented Latin/combining-mark/NFC cases this PR is meant to fix.

## Best Possible Solution

Port this PR's narrow Unicode/NFC tokenizer fix into `extensions/memory-core`: update MMR tokenization while preserving existing CJK unigrams/bigrams, update both FTS query tokenizer paths, add focused regression tests for accented Latin, combining-mark scripts, and NFC/NFD equivalence, then land the updated PR or an equivalent replacement with contributor credit.
