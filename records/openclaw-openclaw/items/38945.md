## Summary

Keep this PR open. Current main still has the tokenizer gaps the PR targets: MMR tokenization is ASCII plus CJK-only, FTS query construction still omits combining marks and NFC normalization, and the shared keyword-search planner has the same regex/normalization gap. The PR is useful but needs to be ported from the old `src/memory/*` paths into current `extensions/memory-core/*` while preserving newer CJK trigram/unigram behavior.

## Best Possible Solution

Update or rework this PR against current main: port the tokenizer fix into `extensions/memory-core/src/memory/mmr.ts`, `extensions/memory-core/src/memory/hybrid.ts`, and `extensions/memory-core/src/memory/manager-search.ts`; add focused regression tests for ASCII, CJK, accented Latin, Cyrillic, Arabic/Hebrew, Thai/Hindi combining marks, and NFC/NFD equivalence; then review and land the updated PR or close it only after an equivalent merged fix exists.
