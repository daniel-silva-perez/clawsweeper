## Summary

Keep this draft PR open. Current main still shows friendly catalog labels in the Control UI chat model picker, while this PR’s central change is to show exact qualified provider/model refs. The PR is not obsolete or implemented on main; it is a product-choice change that still needs maintainer review.

## Best Possible Solution

Keep the PR open for maintainer product review. If explicit refs are desired, make a focused UI display change that preserves qualified-value normalization and stale-provider coverage; if friendly labels remain the intended product behavior, close the draft with that product decision rather than as implemented-on-main.
