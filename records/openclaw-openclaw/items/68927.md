## Summary

Keep this draft PR open. Current main still intentionally renders friendly catalog aliases/names in the Control UI chat model picker, while this PR proposes the opposite product behavior: displaying exact qualified provider/model refs as labels. The related merged work in #61382 is not a superseding implementation of this PR; it is evidence of the current friendly-label direction and the PR remains a product-choice review item.

## Best Possible Solution

Keep the draft PR open for maintainer product review. If exact qualified refs are desired, make the UI label change while preserving the existing qualified-value normalization, stale-provider correction, and nested-ref tests. If friendly catalog aliases/names remain the intended product behavior, close the draft with that explicit product decision rather than as implemented-on-main.
