## Summary

Keep this PR open. Current main still does not implement cron lifecycle hooks, and the newer `cron_changed` plugin hook is observation-only rather than an abortable before/after job execution lifecycle. This PR remains an implementation candidate, but it needs explicit maintainer product and security review because it introduces dynamic script execution from cron config and persisted job data.

## Best Possible Solution

Keep this PR open as an implementation candidate. Maintainers should first decide the product boundary: either land a hardened core cron lifecycle API, or extend the plugin/API surface so plugins can support pre-run aborts and outcome-specific lifecycle dispatch. If the feature proceeds, update config schema, Gateway protocol, docs, and tests for scheduled, startup catch-up, and manual cron paths together, and resolve the remaining path/type validation concerns before merge.
