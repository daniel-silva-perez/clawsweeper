## Summary

Keep open. Current main already handles the web_fetch SSRF opt-in with policy-aware cache separation, but this PR still contains security-sensitive cache and routing gaps and current main does not yet implement the citation-redirect, remote-media, or legacy-normalizer portions well enough to close it as obsolete.

## What This Changes

The PR adds opt-in SSRF policy config for web_fetch, Gemini citation redirects, remote media, browser/schema/help surfaces, and selected channel media paths, with focused tests and generated config updates.

## Best Possible Solution

Rebase the useful source-level pieces onto current main, keep the existing cache-safe web_fetch implementation, add citationRedirect schema/types/docs and legacy-normalizer preservation, include the citation policy in Gemini cache identity, and either wire messages.remoteMedia through every outbound media path or narrow the config to explicitly supported surfaces with tests.

## Reproduction Assessment

Yes. The key blockers are reproducible by source inspection and targeted tests: current main already has a strict-after-permissive web_fetch test, while the PR diff adds policy forwarding without corresponding cache identity or complete sender wiring.

## Solution Assessment

No. The PR is directionally useful, but as proposed it is not the best fix because it broadens SSRF policy surfaces without complete cache isolation and media-path enforcement; the safer path is a narrower, cache-aware, fully wired patch after maintainer/security review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Include SSRF policy in the web_fetch cache key:** `src/agents/tools/web-fetch.ts:549`
  - body: The diff applies params.ssrfPolicy to guarded fetches, but the cache lookup still keys only by URL, extract mode, and max chars. A permissive run can populate the cache and a later stricter run can return that cached content before the SSRF guard runs, so add a stable policy fingerprint or skip caching across policies.
  - confidence: 0.93
- **[P2] Include citation redirect policy in the Gemini cache key:** `extensions/google/src/gemini-web-search-provider.ts:225`
  - body: This threads citationRedirectSsrFPolicy into redirect resolution, but Gemini search caching is still provider/query/count/model only. If a permissive citation policy resolves and caches citations, later strict calls can reuse those URLs without reapplying the stricter redirect policy.
  - confidence: 0.88
- **[P2] Wire remoteMedia policy through every media sender:** `src/config/zod-schema.session.ts:188-193`
  - body: The new messages.remoteMedia.ssrfPolicy config is global, but the branch only forwards it through a subset of senders. Paths such as Slack and Telegram still load outbound media without an SSRF override, so the accepted config silently behaves differently by channel.
  - confidence: 0.86

## Security Review
