## Summary

Keep open. Current main only covers a narrow `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange` path; it does not implement the PR's broader shared SSRF policy surface for citation redirects or outbound remote media, and the provided PR review discussion still raises security-sensitive cache/coverage questions that need maintainer review before merge or replacement.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The current implementation path should build on main's existing web_fetch RFC2544 support, then add the missing shared config/type/schema surface for citation redirects and `messages.remoteMedia`, forward policy through every outbound remote-media path and Gemini citation resolution, preserve legacy search normalization if still relevant, include policy-aware cache keys, and add focused tests plus generated config/schema updates.
