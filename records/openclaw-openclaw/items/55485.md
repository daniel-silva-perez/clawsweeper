## Summary

Keep open. Current main already implements the web_fetch SSRF opt-in more safely than this branch, including policy-aware cache separation, but it does not implement the PR's citation-redirect config, messages.remoteMedia config, or legacy normalizer preservation. The PR branch is useful source work, but it still has concrete SSRF-policy cache and incomplete media-routing blockers, so it should not be closed as implemented or merged as-is.

## What This Changes

The PR adds opt-in SSRF policy config for web_fetch, Gemini citation redirects, remote media, and related browser/schema/help/generated surfaces, then threads that policy through selected fetch, search, and channel media paths with focused tests.

## Best Possible Solution

Keep the current main web_fetch cache-safe implementation, then land the missing source-level pieces in one reviewed patch: add citationRedirect schema/types/docs and legacy-normalizer preservation, pass the policy into Gemini redirect resolution with a policy-aware cache key, and either wire messages.remoteMedia consistently through every outbound media path or narrow the config to explicitly supported surfaces with tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Include SSRF policy in the web_fetch cache key:** `src/agents/tools/web-fetch.ts:549`
  - body: The diff applies params.ssrfPolicy to guarded fetches, but the existing cache lookup still keys only by URL, extract mode, and max chars. A permissive run can populate the cache and a later stricter run can return that cached content before the SSRF guard runs, so the cache key needs a stable policy fingerprint or policy-mismatched caching must be skipped.
  - confidence: 0.93
- **[P2] Include citation redirect policy in the Gemini cache key:** `extensions/google/src/gemini-web-search-provider.ts:225`
  - body: This threads citationRedirectSsrFPolicy into redirect resolution, but Gemini search caching is still provider/query/count/model only. If a permissive citation policy resolves and caches citations, later strict calls can reuse those URLs without reapplying the stricter redirect policy.
  - confidence: 0.88
- **[P2] Wire remoteMedia policy through every media sender:** `src/config/zod-schema.session.ts:188-193`
  - body: The new messages.remoteMedia.ssrfPolicy config is global, but the branch only forwards it through a subset of senders. Paths such as Slack and Telegram still load outbound media without an SSRF override, so the accepted config silently behaves differently by channel.
  - confidence: 0.86
