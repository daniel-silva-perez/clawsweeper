## Summary

Keep open. Current main has not implemented this change, and the PR is a plausible narrow update to an existing bundled skill, but the proposed cheat sheet includes an incorrect official Obsidian CLI example for plugin enable/disable. Because VISION.md also leaves optional bundled-skill promotion as a product call, this should get normal maintainer review rather than conservative cleanup closure.

## What This Changes

The PR rewrites the bundled Obsidian skill to require the official `obsidian` binary, removes the third-party Homebrew installer metadata, and replaces the `obsidian-cli` guide with official CLI setup notes and command examples.

## Best Possible Solution

If maintainers want the bundled Obsidian baseline to move to the official CLI, fix the incorrect plugin command examples, keep the skill aligned with official Obsidian Help, and consider whether the setup section should mention the current 1.12.7+ installer requirement and cross-platform registration paths. If maintainers do not want this as the bundled default, publish the same content as a ClawHub or workspace skill override.

Not applicable as a feature/docs migration. The high-confidence review path is comparing the PR head `SKILL.md` against current main, official Obsidian CLI documentation, and the prior PR discussion.

## Solution Assessment

No, not as-is: the overall approach is narrow, but the plugin enable/disable examples use the wrong required parameter. After that correction, whether this is the best repository-level solution depends on maintainer judgment about replacing the existing bundled optional skill versus routing it through ClawHub.
