## Summary

Keep this issue open. Current main still does not implement the requested Discord bot self-profile action: the Discord config gate and strict schema only expose `presence`, Discord action discovery only advertises `set-presence`, dispatch only maps that action to `setPresence`, and the runtime only updates status/activity through the gateway. The related implementation PR #34010 is closed unmerged, and closed issue #33270 is not an open canonical tracker for the remaining nickname/avatar/self-profile work.

## Best Possible Solution

Keep this issue open. If maintainers want the capability in the bundled Discord plugin, implement a scoped `self-profile` or compatible `set-profile` action behind `channels.discord.actions.selfProfile` with bot-self-only nickname/avatar/status/activity support, explicit multi-account handling, avatar MIME/base64/10MB validation, gateway preflight for presence fields, focused Discord action/config tests, and docs. If maintainers prefer ClawHub, record that product decision and link a plugin or the specific plugin API gap needed before external implementation.
