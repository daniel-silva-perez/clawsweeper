## Summary

Current main still enforces and documents a 0-5 cap for `session.agentToAgent.maxPingPongTurns`, so this PR is not obsolete or implemented on main. It remains a focused implementation candidate for the linked #43972 request, but it needs rebase/current-surface updates, generated/help/type updates, boundary tests, and maintainer review before merge.

## Best Possible Solution

Keep this PR open as the focused implementation candidate for #43972. Rebase it onto current main, raise both the Zod schema maximum and runtime clamp to 20, update current public docs plus schema help, generated schema metadata, and TypeScript comments, fix the existing review comments, and add focused boundary tests proving 20 is accepted while values above 20 remain rejected or bounded before maintainer review.
