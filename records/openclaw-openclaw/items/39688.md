## Summary

Keep open. Current main still emits `message:received` and `message:sent` internal hooks without draining hook-produced `event.messages` back through user delivery, and open PR #41375 is still the active maintainer fix path rather than a merged implementation.

## Best Possible Solution

Keep this issue open until PR #41375 or an equivalent maintainer-approved fix lands. The fix should drain internal hook replies only where a safe reply target exists, add regression coverage for hook-produced reply delivery, update the hooks docs so `event.messages` matches actual reply-capable surfaces, and explicitly document or defer the `message:sent` auto-reply boundary.
