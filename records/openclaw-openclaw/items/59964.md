## Summary

Keep this PR open. Current main has shipped a safer last-known-good startup and hot-reload recovery path, but it does not implement the PR's full intended change: there is no `openclaw config recover` command or `recoverConfigFile` API on main, and the PR itself still has unresolved review concerns around notification routing plus a misplaced non-English proposal doc.

## Best Possible Solution

Do not close this PR as implemented yet. The best maintainer path is to either ask the author to rebase onto the current last-known-good recovery implementation and narrow the PR to the still-missing manual recovery CLI/API with tests, or close it manually after deciding that `doctor --fix` plus automatic last-known-good recovery is the intended product surface. Any revised PR should remove the proposal doc, keep notifications scoped to the triggering write context, and address the invalid-backup dry-run semantics before merge.
