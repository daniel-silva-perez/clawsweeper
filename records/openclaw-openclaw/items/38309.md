## Summary

Keep open. Current main at ab5c8025c9d0a31873d311ae3e73f3664b0a1185 still does not support SecretRef objects in sandbox.docker.env: the config schema, public sandbox type, generated schema, and runtime secret collectors remain string-only or SSH-only for sandbox secrets. The related implementation PR #38310 is closed unmerged, so it is useful context rather than a superseding fix.

## Best Possible Solution

Implement SecretInput support end-to-end for Docker sandbox env in core: accept SecretRef values in the Zod and generated schemas, type unresolved sandbox docker env as Record<string, SecretInput> while keeping the resolved Docker config string-only, collect defaults and per-agent env entries through the secrets runtime and target registry, preserve current plaintext string behavior, and add focused schema/runtime/create-arg regression tests. Use closed PR #38310 as design context, but land an equivalent reviewed implementation on main.
