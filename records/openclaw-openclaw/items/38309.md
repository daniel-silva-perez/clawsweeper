## Summary

Keep open. Current main at 7e5d6dba80016a5d3502fa6cfc5589aa58a072af still does not support SecretRef objects in sandbox.docker.env: the schema, public config type, generated schema, and secret runtime remain string-only or SSH-only for sandbox secrets. The related implementation PR #38310 is useful design context and had a positive bot review plus local author testing, but it was closed unmerged and no open PR currently supersedes this issue.

## Best Possible Solution

Keep this issue open and, after maintainer/security routing, implement SecretInput support end-to-end for Docker sandbox env: accept SecretRef values in Zod and generated schemas, type unresolved SandboxDockerSettings.env as Record<string, SecretInput> while keeping resolved SandboxDockerConfig.env as Record<string, string>, collect defaults and per-agent env entries through the secrets runtime and target registry, preserve plaintext string behavior, and add focused schema/runtime/create-arg regression tests. Closed PR #38310 is useful design context but should be refreshed against current main rather than treated as a superseding fix.
