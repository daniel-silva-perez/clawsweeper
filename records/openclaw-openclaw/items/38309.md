## Summary

Current main still does not support SecretRef objects in `sandbox.docker.env`. The Zod schema, public config type, generated schema, secret runtime collectors, and SecretRef target registry remain string-only or SSH-only for sandbox secrets. The related implementation PR #38310 was closed unmerged, so this should stay open for maintainer/security routing rather than cleanup closure.

## Best Possible Solution

Keep this issue open and route it through sandbox/secrets ownership. The best implementation is an end-to-end SecretInput addition for Docker sandbox env: accept `SecretInput` in the Zod and generated schemas, type unresolved `SandboxDockerSettings.env` as `Record<string, SecretInput>` while preserving resolved `SandboxDockerConfig.env` as `Record<string, string>`, collect `agents.defaults.sandbox.docker.env.*` and `agents.list[].sandbox.docker.env.*` through the secrets runtime and target registry, resolve before Docker create args, preserve plain string behavior, update public docs, and add focused schema/runtime/registry/merge/create-arg tests.
