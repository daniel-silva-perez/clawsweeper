## Summary

Keep open. Current main only partially addresses the LaunchAgent downtime report: restart now prefers `kickstart -k` with re-bootstrap fallback, but the generated plist still uses a one-second throttle, has no shutdown timeout, and package/update service flows still reach `stopLaunchAgent`, whose degraded fallback can boot out and leave the service unloaded.

## What This Changes

The issue asks OpenClaw to harden macOS LaunchAgent auto-update and service lifecycle behavior by changing the throttle policy, adding a shutdown timeout, and avoiding non-uninstall `bootout` paths that can leave the gateway down.

## Best Possible Solution

Land a focused macOS LaunchAgent lifecycle fix that codifies the chosen launchd restart/backoff policy, adds an explicit shutdown timeout to the generated plist and tests, and makes non-uninstall bootout paths either unnecessary or reliably recoverable during install, update, stop, restart, and failed-update recovery.

## Reproduction Assessment

Yes. The current source gives a high-confidence reproduction path for the unresolved gaps: generated plists still contain `ThrottleInterval=1` and no `ExitTimeOut`, and package updates still call the macOS stop path that can fall back to `bootout`.

## Solution Assessment

No. Current main is a partial repair, and PR #73849 is not the best complete solution as-is because it only addresses an adjacent crash-loop issue and introduces unrelated changes; the maintainable fix should handle plist policy and lifecycle recovery together.
