## Summary

Current main only partially addresses the reported LaunchAgent downtime path. The merged launchd supervision work referenced this report and improves restart/repair behavior, but the plist still emits `ThrottleInterval=1`, there is no `ExitTimeOut`, and install/update service activation still has bootout/bootstrap paths. Related #66436 tracks one throttle symptom, but it does not cover the missing shutdown timeout or the full auto-update downtime scenario.

## Best Possible Solution

Keep this open unless maintainers choose to split it. The best fix is a focused macOS LaunchAgent lifecycle patch that sets or redesigns the throttle/backoff policy, adds an appropriate `ExitTimeOut` with plist tests, and audits package-update/install activation so bootout either stays uninstall-only or has a reliable rollback/rebootstrap path. #66436 can remain the narrower throttle/log-storm tracker, but it does not fully supersede this report.

- The existing #72616-era repair behavior should be preserved while tightening the remaining update/install lifecycle paths.
