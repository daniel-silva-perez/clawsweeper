Keep this PR open. Current main still parses `openclaw ltm search --limit` with `Number.parseInt(opts.limit, 10)` and passes the result directly into `db.search`, while `MemoryDB.search` forwards that value into LanceDB `.limit(limit)`. The PR remains relevant and narrow, but it should add focused regression coverage before merge. Security review found no unexplained workflow, dependency, package, secret, install, or release-surface changes; the provided diff is limited to CLI limit parsing and a warning in `extensions/memory-lancedb/index.ts`.

No core/plugin API change is needed.
