Keep this PR open. Current main still lacks the PR's intended fallback-aware inline suppression for memory-core light/REM daily-memory blocks; the linked bug may be partly mitigated by the newer default separate storage mode, but explicit inline storage remains supported and still writes raw metadata before request-scoped narrative fallback.

## Best Possible Solution

Keep the PR open as an active memory-core bug-fix candidate. The best path is to review the current head, remove the unrelated MiniMax FAQ change, reconcile overlap with #70332, and either land a focused fallback-aware inline suppression fix with regression coverage or intentionally replace it with the same behavior in the canonical diary-hardening PR.
