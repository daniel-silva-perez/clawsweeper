## Summary

Current main still lacks the PR's fallback-aware inline suppression: light and REM build raw metadata blocks, write them to inline daily memory before narrative fallback, and the narrative API gives callers no fallback-success signal. The PR remains a relevant implementation candidate, with unrelated FAQ docs churn and markdown-removal edge cases still worth maintainer review.

## Best Possible Solution

Keep this PR open as a focused memory-core bug-fix candidate. Review the current head, split the unrelated FAQ docs change, verify the light/REM request-scoped fallback tests cover inline suppression and formatting preservation, then either land it or intentionally consolidate the same fix into the broader open #70332 thread.
