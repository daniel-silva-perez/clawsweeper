## Summary

Keep this issue open. Current main has clean sub-agent sessions and a plugin subagent runtime helper, but it does not expose a before_tool_call isolation directive, constrained quarantined execution result, or standardized trust/provenance metadata. The issue is a real core/plugin API design request, not implemented, not superseded, and not safely movable to ClawHub without new core semantics.

## Best Possible Solution

Keep the issue open as a core/plugin API design item. The likely path is an additive hook or runtime capability that builds on subagent/session infrastructure but explicitly supports constrained execution, tool/model/token/time limits, and structured trust/provenance metadata, coordinated with the still-open hook-enrichment prerequisites #48503, #48506, and #48509.
