## Summary

Keep this issue open. Current main has sub-agent sessions, static sandbox/tool-policy controls, and a plugin runtime helper for launching subagent runs, but it still lacks the requested plugin-returned isolation directive, per-operation constrained/quarantined execution, and structured trust/provenance metadata. The request is a core/plugin API and security design item, not implemented on main and not safely movable to ClawHub without new core semantics.

## Best Possible Solution

Keep the issue open as a core/plugin API design item. The best path is an additive hook/runtime capability that builds on subagent and sandbox infrastructure but explicitly supports constrained execution, clear enforcement guarantees, and structured trust/provenance metadata, coordinated with the related open hook-enrichment proposals #48503, #48506, and #48509.
