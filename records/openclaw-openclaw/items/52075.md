## Summary

Keep open. Current main documents the OpenAI-compatible HTTP endpoint mechanics and security boundary, but it still does not include the PR’s decision guidance for custom mobile clients, WebChat/Gateway Protocol alternatives, channel-plugin boundary, or stable per-conversation `user` example. The PR is docs-only and addresses an auth-sensitive gap rather than being obsolete.

## Best Possible Solution

Keep the PR open for maintainer review or rebase it into the current `docs/gateway/openai-http-api.md`. The best path is to land equivalent docs that explain when to use `/v1/chat/completions`, when to use WebChat/Gateway Protocol with pairing/device tokens, when to build a channel plugin, and how to use per-conversation stable sessions. Consider updating the example to the current preferred `openclaw/default` model alias unless `openclaw:main` is intentional.
