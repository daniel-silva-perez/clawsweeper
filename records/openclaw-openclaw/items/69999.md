## Summary

Current main does not implement the proposed macOS wallboard shell: there are no Wallboard sources/resources/tests, no `--wallboard` launch path, and the existing WebChat/Canvas surfaces serve different purposes. This PR remains the active implementation candidate, but it should stay open for maintainer review and changes around safe bundle resource lookup, WebKit private/developer settings, deprecated activation, and re-open reload behavior.

## Best Possible Solution

Keep this PR open. If maintainers want the REL-433 wallboard direction, update this PR to use safe main-bundle-first resource lookup, avoid placeholder reload on re-open, replace deprecated activation API usage, and document or remove private WebKit KVC/developer extras. Then validate on macOS with the Wallboard smoke tests, a Swift build/test pass for `apps/macos`, and a manual `--wallboard` launch with and without an external display if hardware is available.
