## Summary

Keep open. Current main still registers grammY sequentialize before the Telegram callback handler, and ordinary inline-button callbacks still use the normal chat/topic sequential key. The merged approval-lane change only covers `/approve` callback payloads; the broader callback acknowledgement timing reported here remains unresolved.

## Best Possible Solution

Keep this issue open and implement the Telegram plugin fix on current main: add a pre-sequentialize middleware in `extensions/telegram/src/bot-core.ts` that immediately acknowledges callback queries and catches/logs API failures, remove or avoid redundant handler-level acknowledgement in `extensions/telegram/src/bot-handlers.runtime.ts`, and add a regression test proving `answerCallbackQuery` fires before an ordinary same-chat callback action is released from behind a busy sequentialized turn.
