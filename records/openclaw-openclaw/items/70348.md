## Summary

Keep this PR open. Current main does not yet implement the PR's intended filtering: `readSessionMessages` still turns compaction records into synthetic `system` messages with `__openclaw.kind: "compaction"`, and the visible-history sanitizer currently drops assistant commentary/suppressed control replies but not compaction markers or pre/post-compaction prompt artifacts. The PR also has a substantive review concern because its text-prefix filter can suppress legitimate user content that happens to start with the internal marker text.

## Best Possible Solution

Keep this PR open, but revise the implementation before merge. The safest path is provenance/metadata-based filtering for synthetic/internal records, plus regression coverage that compaction markers and known internal memory artifacts are hidden while user-authored messages with matching text prefixes remain visible. Coordinate with #43567 if maintainers prefer a distinct system block instead of full hiding.
