## Summary

Keep open. Current main still projects synthetic compaction markers through `chat.history`, and the shared history sanitizer does not drop the PR's compaction, pre-compaction memory-flush, or post-compaction refresh artifacts. The PR is relevant, but the proposed text-prefix filter can over-filter legitimate user-authored messages, so it needs revision or maintainer product direction before merge.

## Best Possible Solution

Keep this PR open, but revise it before merge. The safer direction is provenance or metadata-based filtering for synthetic/internal records, plus regression coverage that hides synthetic compaction and known internal memory artifacts while preserving user-authored messages with matching text prefixes. Coordinate with #43567 if maintainers prefer distinct system-block rendering over full hiding.
