## Summary

Keep open. Current main still sends the relevant `chat.history` path through projection without dropping synthetic compaction markers or the named pre/post-compaction artifacts, so the PR is not obsolete; however, the PR's raw text-prefix filter can suppress legitimate user-authored messages and should not merge as-is.

## What This Changes

The PR adds a `chat.history` sanitizer filter for synthetic compaction markers, pre-compaction memory-flush prompts, and post-compaction context-refresh messages, plus targeted Gateway history tests.

## Best Possible Solution

Revise the PR toward structural provenance or metadata-based filtering for synthetic/internal records, and add regressions that hide generated compaction artifacts while preserving ordinary user-authored text that happens to share the marker prefixes.

## Reproduction Assessment

Yes. A source-level reproduction exists by seeding a transcript with a `type: "compaction"` entry and calling `chat.history`; current main converts it to a synthetic system message and the shared projection does not drop it. The PR-introduced false positive is also reproducible from the diff with a user message whose text begins with one of the filtered marker strings.

## Solution Assessment

No. The PR targets a real leak, but raw prefix filtering is not the narrowest maintainable fix because it lacks provenance and can hide legitimate visible messages. A safer fix should distinguish synthetic/internal records structurally, then decide with #43567 which internal messages should be hidden versus rendered as system/runtime UI.
