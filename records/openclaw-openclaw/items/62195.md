## Summary

Current main does not implement the PR’s intended cron/heartbeat preHook feature, and this open contributor PR remains the active implementation candidate rather than obsolete cleanup. It should stay open for maintainer review, especially because the latest PR discussion still contains security/design concerns around local executable spawning and scheduler blocking.

## Best Possible Solution

Keep this PR open for normal maintainer review. If OpenClaw wants this as a core scheduler feature, the best path is a current-main rebase that preserves the file/args no-shell model, enforces absolute executable paths or an explicit trust policy, redacts/truncates hook output, handles skip/error/backoff semantics for cron and heartbeat, covers add/update/clear tool schemas, and includes focused tests for main-session, isolated, heartbeat, abort, timeout, and generated protocol/docs alignment. If maintainers decide this belongs outside core, that should be an explicit product decision rather than an automated cleanup close.

## Risks / Open Questions

- The PR introduces local executable spawning in cron and heartbeat paths. Before merge, maintainers should resolve the updated security findings around requiring absolute/canonical executable paths, constraining PATH/search behavior, and documenting the trust model.
- The PR awaits hook execution inline in scheduler paths. The 300 second timeout cap may still block due cron work or heartbeat turns unless maintainers accept that behavior or add bounded concurrency/process-group cleanup/resource controls.
- The change touches public protocol schemas, cron persistence, heartbeat runtime, agent tool schemas, generated Swift models, generated config baselines, and docs. It needs a full maintainer validation path if kept as a core feature.
