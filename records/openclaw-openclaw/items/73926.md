## Summary

Keep open. The issue is still paired with open PR #73929, and current main only partly covers the reported failure class: blank visible prompt submissions are skipped before provider dispatch, but runtime-only empty prompt submissions remain an intentional tested path that needs maintainer review before this can be closed.

## What This Changes

The issue asks OpenClaw to prevent `/new` and `/reset` active-memory races from dispatching an empty prompt and empty message list to providers such as Anthropic or OpenAI-compatible adapters.

## Best Possible Solution

Resolve PR #73929 against current main: land or rework it if it closes a remaining model-visible empty-input path, or close it as superseded if the current blank prompt guard and reset transcript marker cover the intended behavior. The durable end state should be one embedded-runner boundary that blocks provider-visible empty input while preserving intentional runtime-only context.
