## Summary

Keep open. Current main still has the embedded-runner gap described in the report: the content guard treats any replay-history object as content, runtime-only submissions bypass that guard, and the runtime-only dispatch path can still submit an empty visible prompt. There is also an open linked fix PR, #73929, with closing syntax for this issue, so the right outcome is to review/land or close that PR rather than close this issue early.

## Best Possible Solution

Review and either land or close #73929. The desired fix is a conservative embedded-runner prompt-content guard that checks for model-visible replay content rather than array presence, keeps bare `/new` startup prompt behavior intact, preserves runtime-only transcript hygiene, and covers blank replay-history regressions with focused tests. Once that PR or an equivalent fix merges to main, close this issue as implemented.
