## Summary

Keep open. Current main fixes the bare-reset and blank visible replay-history subcases, and the previously linked PR is now closed as superseded, but the active-memory/runtime-only path still intentionally submits an empty visible prompt through the embedded runner. That leaves the issue’s stricter provider-visible empty-input guard not fully implemented.

## What This Changes

The issue asks OpenClaw to prevent `/new` or `/reset` active-memory races from dispatching an empty prompt and empty message list to Anthropic-compatible providers.

## Best Possible Solution

Keep this issue open until the runtime-only path has an explicit provider-safe contract. The maintainable end state is a single embedded-runner or transport boundary that never sends provider-visible empty input, while preserving hidden runtime context and active-memory flush behavior outside normal user transcript history.

## Reproduction Assessment

Yes at source level. A run with non-empty runtime context and blank `transcriptPrompt` resolves to `prompt: ""`, bypasses the blank visible-prompt skip because `runtimeOnly` is true, and current tests assert `sessionPrompt` receives an empty string.

## Solution Assessment

No for the full issue. The current main fix is the right narrow solution for blank visible user prompts, but it does not address the active-memory/runtime-only empty visible prompt path described in the report.
