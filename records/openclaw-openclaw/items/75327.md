## Summary

Keep open: the PR targets a plausible reply-delivery gap, but it is not ready to merge because the new core resolver branch is bypassed by at least the Discord path, and the requested behavior changes the documented group/channel private-reply default.

## What This Changes

The PR adds `WasMentioned` to source reply delivery mode resolution so group/channel turns become `automatic` when directly mentioned, plus unit coverage for the resolver and visibility policy.

## Best Possible Solution

Decide the product contract first. If @mentions should override tool-only delivery, implement the exception consistently at the channel pre-resolution and dispatch seams, update Discord/Slack/core coverage, and document the new exception with a changelog entry; otherwise keep the current documented default and close or rework the PR.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes: the current Discord process test already models `effectiveWasMentioned: true` and shows the delivery mode remains `message_tool_only`, and the source path explains why the PR’s resolver-only change would not affect that channel path.

## Solution Assessment

Is this the best way to solve the issue? No: the patch changes the core resolver but misses callers that pass an explicit pre-resolved mode, and it does not align the documented group/channel visible-reply contract with the new behavior.
