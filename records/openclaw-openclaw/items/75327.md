## Summary

Keep this PR open, but do not merge it as-is. The reported behavior is plausible, yet the patch only changes the core resolver and misses the current Discord path that pre-resolves `message_tool_only` before dispatch; it also changes the documented group/channel visible-reply contract without docs or changelog follow-through.

## What This Changes

The PR changes source reply delivery mode resolution and unit tests so group/channel turns with `WasMentioned: true` resolve to automatic delivery.

## Best Possible Solution

First decide whether direct @mentions should override the group/channel tool-only visible-reply default. If yes, implement the exception consistently where channel plugins pre-resolve delivery mode and where dispatch computes suppression, update Discord/Slack/core coverage, and document the exception with a changelog entry; if no, rework or close the PR against the current documented contract.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes: the current Discord process test already models a mentioned guild message and shows the delivery mode stays `message_tool_only`, and the source path explains why this PR's resolver-only change would not alter that dispatch path.

## Solution Assessment

Is this the best way to solve the issue? No: changing only `resolveSourceReplyDeliveryMode` misses callers that pass an explicit pre-resolved mode and leaves the public group/channel visible-reply docs inconsistent with the new behavior.
