## Summary

Keep open. Current main has partial mitigations for the startup side of the pricing-refresh regression, but the remaining shutdown/hung-process behavior is still source-backed: pricing fetches are not tied to Gateway lifecycle abort, and refresh completion can still reschedule background work after stop.

## What This Changes

The issue reports that OpenClaw 2026.4.24 on Windows/npm global hangs during Gateway startup when OpenRouter and LiteLLM pricing fetches time out, leaving the dashboard unreachable and Ctrl+C ineffective.

## Best Possible Solution

Keep the existing startup mitigations, then make model-pricing refresh fully lifecycle-aware: abort in-flight OpenRouter and LiteLLM fetches on Gateway stop, prevent cache mutation or refresh scheduling after stop, unref long refresh timers, and lock this with focused Gateway pricing shutdown tests.

## Reproduction Assessment

Yes. The issue gives concrete released-package steps, and current main supports a focused source-level reproduction by starting startGatewayModelPricingRefresh with hanging fetches, calling the returned stop callback, and asserting the fetches abort with no post-stop timer scheduled.

## Solution Assessment

No. Current main is a useful partial mitigation, and PR #73214's timeout/opt-out direction helps restricted networks, but the narrow lifecycle fix still belongs in the pricing-refresh stop/abort path.
