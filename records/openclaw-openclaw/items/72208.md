## Summary

Keep open. Current main has partial mitigations for the startup delay, including manifest reuse, deferred refresh, and a config opt-out, but the model-pricing refresh lifecycle still is not shutdown-aware: in-flight pricing fetches use standalone timeout signals, and stopping the gateway cannot abort them or prevent post-stop refresh scheduling.

## What This Changes

The issue requests fixing a 2026.4.24 Windows npm-global regression where `openclaw gateway` hangs after OpenRouter/LiteLLM pricing fetch timeouts, leaving the dashboard unreachable and Ctrl+C ineffective.

## Best Possible Solution

Keep the shipped startup mitigations, then make the model-pricing refresh lifecycle cancellation-aware: abort in-flight OpenRouter/LiteLLM fetches on gateway stop, combine lifecycle abort with the existing timeout, suppress refresh scheduling after stop, unref any background refresh timer, and lock the behavior with focused model-pricing/runtime shutdown tests.

## Reproduction Assessment

Yes. The issue provides concrete released-package steps, and current source supports a focused unit reproduction by injecting hanging pricing fetches, calling the returned stop function after refresh starts, and asserting the fetches are aborted and no refresh timer is scheduled after stop.

## Solution Assessment

No. The current main implementation is a useful partial startup mitigation and opt-out, but the narrow complete fix belongs in the model-pricing refresh lifecycle rather than broader gateway startup, provider defaults, or user configuration.
