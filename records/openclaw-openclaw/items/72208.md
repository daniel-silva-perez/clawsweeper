## Summary

Keep this issue open. Current main includes the startup deferral/plugin-manifest mitigation for pricing refresh, but the linked closing PR #72306 is still open and current main still lacks its Ctrl+C/event-loop cleanup: the 24-hour pricing timer is not unref'd, and in-flight LiteLLM/OpenRouter pricing fetches are not aborted on stop.

## Best Possible Solution

Keep this issue open while #72306 is open. Maintainers should review and either land #72306 or an equivalent cleanup for unref'ing the refresh timer and aborting in-flight pricing fetches on stop, or explicitly close that PR if the remaining shutdown behavior is not desired. After merge/backport or package release, verify Windows npm-global gateway startup, dashboard reachability, and Ctrl+C behavior under pricing timeouts, then close as implemented/released.
