## Summary

Keep open. Current main and v2026.4.27 contain the startup-side mitigation for model-pricing refresh work, but the source still does not make the pricing refresh lifecycle shutdown-aware, leaving a concrete gap for the reported Ctrl+C/hung-process behavior.

## What This Changes

The issue requests fixing a regression where OpenClaw 2026.4.24's globally installed Gateway hangs when OpenRouter/LiteLLM pricing fetches time out, leaving the dashboard unreachable and Ctrl+C ineffective.

## Best Possible Solution

Keep the shipped startup optimization, then make the model-pricing refresh lifecycle cancellation-aware: abort in-flight pricing fetches on stop, suppress all follow-up scheduling after stop, unref background refresh timers, and lock the behavior with focused gateway pricing/runtime shutdown tests.

## Reproduction Assessment

Yes. The issue provides concrete released-package steps, and current source supports a focused unit reproduction by starting a pricing refresh with hanging fetches, calling the returned stop function, and asserting the fetches are aborted and no timer remains scheduled.

## Solution Assessment

No. The current implementation is a useful partial startup mitigation, but the narrow complete fix belongs in the model-pricing refresh lifecycle rather than broader Gateway startup or provider behavior.

## Repair Work Prompt

Fix the remaining lifecycle gap for https://github.com/openclaw/openclaw/issues/72208. Current main already defers the initial model-pricing refresh and reuses plugin lookup metadata, but shutdown still does not cancel in-flight pricing fetches and a stopped refresh can schedule a ref'ed follow-up timer. Keep the fix scoped to Gateway model-pricing lifecycle code. Make `startGatewayModelPricingRefresh`/`refreshGatewayModelPricingCache` cancellation-aware so the returned stop function aborts in-flight OpenRouter/LiteLLM pricing fetches, prevents any post-stop `scheduleRefresh`, and ensures background refresh timers call `unref`. Preserve existing config behavior for `models.pricing.enabled`; do not disable pricing refresh wholesale, do not add provider-specific policy in core, and do not broaden plugin loading. Add focused tests in `src/gateway/model-pricing-cache.test.ts` for stopping during in-flight fetches and for no timer after stop, and update runtime/close-service tests only as needed to prove the stop seam. Add a changelog fix entry if runtime behavior changes, crediting the issue reporters/commenters such as @rzcq and @hoholidayx, and keep the entry single-line.
