## Summary

Keep open. Current main still lacks the requested generic RPM/RPD-aware fallback contract: provider config has no quota fields, auth-profile state has no minute/day counters, and fallback preflight only reacts to existing cooldowns. Current main does include shorter reactive cooldowns, model-scoped cooldowns, provider usage-window reporting, and an OpenAI Codex WHAM-specific cooldown probe, but those do not solve generic per-provider quota-aware pre-request skipping.

## What This Changes

Request generic provider RPM/RPD quota settings that compute exact quota-window cooldowns and drive fallback skipping through auth-profile `cooldownUntil` instead of only reactive stepped backoff.

## Best Possible Solution

Define a generic quota contract, persist minute/day quota state with exact reset-time calculation, integrate pre-request fallback skipping with `cooldownUntil` under the existing auth-profile store locks, and update config schema/help/docs plus focused auth-profile and model-fallback tests.
