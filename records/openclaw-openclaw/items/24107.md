## Summary

Keep open. Current main has provider usage display and a provider-specific OpenAI Codex WHAM cooldown probe, but it still does not have the requested generic RPM/RPD provider quota contract, persisted minute/day quota counters, or pre-request quota-aware skip that writes exact cooldownUntil windows. Related #38775 is open and broader, but it does not supersede this narrower fallback cooldown request.

## Best Possible Solution

Keep this open as a core auth-profile/model-fallback enhancement. The best path is maintainer agreement on a generic provider quota contract, then implementation of persisted minute/day quota state, exact reset-time calculation, pre-request candidate skipping that writes cooldownUntil, config schema/help/docs updates, and focused tests around RPM/RPD windows. Coordinate the design with #38775, but do not close this as a duplicate unless maintainers explicitly choose that issue as the canonical umbrella.
