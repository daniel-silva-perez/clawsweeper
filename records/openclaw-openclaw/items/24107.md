## Summary

Keep open. Current main still lacks the requested generic RPM/RPD-aware fallback cooldown contract: model provider config and schema have no quota fields, auth-profile usage state has no minute/day counters, and fallback preflight only reacts to already-recorded cooldowns. The existing OpenAI Codex WHAM probe is provider-specific and #38775 is broader/open rather than an agreed canonical replacement.

## What This Changes

Request generic per-provider RPM/RPD quota configuration that precomputes exact cooldownUntil windows and skips fallback candidates before making requests that would exceed those quotas.

## Best Possible Solution

Define a generic quota contract, persist minute/day quota state with exact reset-time calculation, integrate pre-request fallback skipping with cooldownUntil under existing auth-profile store locks, and update config schema/help/docs plus focused auth-profile and model-fallback tests.
