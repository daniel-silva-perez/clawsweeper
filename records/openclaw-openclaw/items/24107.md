## Summary

Keep open. Current main still lacks the requested generic RPM/RPD-aware fallback cooldown contract: provider config/schema do not expose quota fields, auth-profile usage state has no minute/day quota counters, and fallback preflight only reacts to stored cooldowns or failures. The OpenAI Codex WHAM cooldown probe is provider-specific, and related #38775 is still open rather than a merged canonical replacement.

## What This Changes

The issue requests generic per-provider RPM/RPD-aware fallback cooldowns that preflight-skip quota-exhausted providers and set exact quota-window `cooldownUntil` values.

## Best Possible Solution

Define a generic quota contract, persist minute/day quota state with exact reset-time calculation, integrate pre-request fallback skipping with `cooldownUntil` under the auth-profile store lock, and update config schema/help/docs plus focused auth-profile and model-fallback tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature request. Source inspection gives high confidence that the requested contract is absent because config/schema, usage state, and fallback preflight have no generic RPM/RPD quota path.

## Solution Assessment

Unclear until maintainer review. Quota-aware cooldowns are a plausible solution, but the maintainable API shape should be chosen before implementation because it crosses public config/schema, persisted state, and provider/plugin routing semantics.
