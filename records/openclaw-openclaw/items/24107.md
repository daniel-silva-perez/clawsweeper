## Summary

Keep open. Current main still lacks the requested generic RPM/RPD-aware fallback cooldown contract: provider config has no quota fields, auth-profile state has no minute/day quota counters, and model fallback only reacts to existing cooldown/failure state. The provided maintainer and ClawSweeper comments already reached the same conclusion, and the later current-main code does not add the missing public API or pre-request quota accounting.

## What This Changes

The issue requests provider RPM/RPD-aware model fallback cooldowns that compute exact quota reset windows instead of using generic reactive backoff after 429s or timeouts.

## Best Possible Solution

Define a generic quota contract, persist minute/day quota state with exact reset-time calculation, integrate pre-request fallback skipping with cooldownUntil under existing auth-profile store locks, and update config schema/help/docs plus focused auth-profile and model-fallback tests.
