## Summary

Keep open. Current main still lacks the requested generic provider-level RPM/RPD contract, quota-window counters, and pre-request quota skip. Existing code has generic cooldowns, model-scoped cooldowns, rate-limit profile rotation limits, and an OpenAI Codex WHAM-specific cooldown probe, but those do not implement arbitrary provider RPM/RPD enforcement.

## Best Possible Solution

Keep this open as a core model fallback/auth-profile enhancement. The best path is to agree on a validated provider quota contract, then add persisted minute/day quota state with reset timestamps, a pre-request quota skip that marks `cooldownUntil`, generated schema/help/docs updates, and focused tests for RPM/RPD reset behavior alongside existing cooldown/probe coverage.
