## Summary

Keep open. Current main still lacks the requested generic RPM/RPD-aware fallback cooldown contract: provider config/schema do not expose quota fields, auth-profile usage state has no minute/day quota counters, and model fallback only reacts to stored cooldowns or failures. The OpenAI Codex WHAM path is provider-specific, and related #38775 remains open rather than a merged canonical replacement.

## What This Changes

Request to make model fallback cooldowns respect configured per-provider RPM/RPD quotas by preflight-skipping providers and setting exact quota-window `cooldownUntil` values.

## Best Possible Solution

Define a generic quota contract, persist minute/day quota state with exact reset-time calculation, integrate pre-request fallback skipping with `cooldownUntil` under the auth-profile store lock, and update config schema/help/docs plus focused auth-profile and model-fallback tests.
