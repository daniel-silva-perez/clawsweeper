## Summary

Keep open. Current main still lacks the requested generic RPM/RPD quota contract, persisted minute/day quota counters, and pre-request quota-aware skip that writes exact cooldownUntil windows. The existing cooldown path is reactive after failures, and the OpenAI Codex WHAM probe is provider-specific rather than a provider-generic fallback quota mechanism; related #38775 is broader and open, not a clear superseding canonical tracker.

## What This Changes

Request provider-configurable RPM/RPD quotas that let model fallback compute exact cooldownUntil windows and skip quota-exhausted providers before sending a request.

## Best Possible Solution

Define a generic provider quota contract, add persisted minute/day quota state with exact reset-time calculation, integrate pre-request fallback skipping with cooldownUntil, and update config schema/help/docs plus focused auth-profile and model-fallback tests. Keep #38775 coordinated as broader balancing work unless maintainers explicitly make it canonical.
