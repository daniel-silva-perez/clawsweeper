## Summary

Keep open. Current main still lacks a generic RPM/RPD-aware fallback cooldown contract: provider config/schema do not accept quota fields, auth-profile usage state does not store minute/day quota counters, and model fallback only reacts to existing cooldowns or failures. The existing WHAM reset-window probe is scoped to OpenAI Codex, and related #38775 is still open rather than a canonical replacement.

## What This Changes

Request generic per-provider RPM/RPD quota settings that preemptively set precise auth-profile cooldown windows and skip fallback candidates before wasting requests on known quota exhaustion.

## Best Possible Solution

Define a generic quota contract, persist minute/day quota state with exact reset-time calculation, integrate pre-request fallback skipping with cooldownUntil under the auth-profile store lock, and update config schema/help/docs plus focused auth-profile and model-fallback tests.
