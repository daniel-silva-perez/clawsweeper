## Summary

Keep open. Current main still does not implement the requested generic RPM/RPD quota-aware fallback contract: provider config has no quota fields, usage state has no minute/day quota counters, and fallback only reacts to existing cooldowns or failures. The OpenAI Codex WHAM path is a provider-specific probe, not a generic per-provider quota mechanism, and related #38775 is broader and still open rather than a clear canonical replacement.

## What This Changes

Request to add per-provider RPM/RPD quota configuration, exact quota-window cooldowns, and pre-request fallback skipping that updates auth-profile cooldown state.

## Best Possible Solution

Define a generic quota contract, add persisted minute/day quota state with exact reset-time calculation, integrate lock-backed pre-request fallback skipping with cooldownUntil, and update config schema/help/docs plus focused auth-profile and model-fallback tests.
