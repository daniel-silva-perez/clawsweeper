## Summary

Keep open. Current main still does not provide the requested generic RPM/RPD-aware fallback cooldown contract: provider config has no quota fields, auth-profile state has no minute/day quota buckets, and fallback preflight only reacts to existing cooldown state. The prior maintainer and ClawSweeper comments reached the same conclusion, and the later current-main changes do not add this surface.

## What This Changes

The issue requests generic RPM/RPD-aware provider fallback cooldowns so configured quota windows can drive pre-request skipping and `cooldownUntil` instead of only reactive stepped backoff.

## Best Possible Solution

Define a generic quota contract, persist minute/day quota state with exact reset-time calculation, integrate pre-request fallback skipping with `cooldownUntil` under existing auth-profile store locks, and update config schema/help/docs plus focused auth-profile and model-fallback tests.
