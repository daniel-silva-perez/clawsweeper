## Summary

Keep open. Current main still lacks the requested generic provider-level RPM/RPD config contract, persisted quota-window counters, and pre-request quota skip. Existing behavior covers generic auth-profile cooldowns, model-scoped cooldowns, profile-rotation limits, and an OpenAI Codex-specific WHAM cooldown probe, but not arbitrary provider RPM/RPD enforcement.

## Best Possible Solution

Keep this open as a core model fallback/auth-profile enhancement. The best path is to agree on the provider quota contract first, then add persisted minute/day quota state with reset timestamps, a pre-request quota skip that marks cooldownUntil, generated schema/help/docs updates, and focused tests for RPM/RPD reset behavior alongside existing cooldown/probe coverage.
