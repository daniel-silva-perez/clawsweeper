## Summary

Keep open. Current main has model fallback plumbing and an OpenRouter free-model scan, but it does not implement the issue's requested benchmark/ranked fallback policy for the named enabled models, nor the requested config guardrail/test that would reject an unsupported `models.providers.*.models[*].input` modality such as audio for a text/image model.

## Best Possible Solution

Keep this issue open and implement it as a focused model-fallback hardening task: add a benchmark/report path for the currently enabled fallback models, document the selected latency/quality/cost ordering policy, and add schema or validation coverage that rejects unsupported `models.providers.*.models[*].input` modality declarations before deploy.
