## Summary

Keep open. Current main still lacks the benchmark-backed fallback policy and semantic model-input guardrail; the runtime preserves configured fallback order, and config validation only checks modality enum names, so stale closure would drop unresolved product and validation work.

## What This Changes

The issue requests benchmark-ranked fallback ordering for the active OpenRouter fallback chain plus a config guardrail that fails fast on unsupported `models.providers.*.models[].input` modalities.

## Best Possible Solution

Define a documented fallback policy artifact for the active model chain using repeatable benchmark criteria, then add capability-aware validation for configured model inputs using trusted provider or catalog metadata while preserving valid custom multimodal provider catalogs.
