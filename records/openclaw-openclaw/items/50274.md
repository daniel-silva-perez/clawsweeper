## Summary

Keep open. Current main has tightened fallback selection-source behavior and manifest catalog validation, but it still does not implement the requested benchmark-backed fallback ranking or a semantic guardrail that rejects a configured model input modality based on that model's actual capabilities.

## What This Changes

The issue requests benchmarking active fallback models, defining latency/quality/cost fallback ordering, adding validation for unsupported `models.providers.*.models[*].input` modalities, and documenting the resulting policy.

## Best Possible Solution

Define and document a benchmark-backed fallback policy for the active model chain, then add capability-aware validation that rejects unsupported configured input modalities while preserving valid multimodal provider catalogs.
