Keep open. Current main still preserves configured fallback order and only validates model input modality names, so the benchmark-backed fallback policy and capability-aware config guardrail remain unresolved maintainer work.

## What This Changes

Request to benchmark active fallback models, define latency/quality/cost fallback ordering policy, add capability-aware validation for model input modalities, and document the resulting policy.

## Best Possible Solution
