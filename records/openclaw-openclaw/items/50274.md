## Summary

Keep open. Current main still implements/document explicit configured fallback chains and only validates model input modality names; it does not add the requested benchmark-backed latency/quality/cost fallback policy or a capability-aware guardrail for unsupported configured model inputs. The existing issue discussion already reached the same conclusion, and the current HEAD does not change the relevant fallback, model scan, schema, catalog, or docs surfaces.

## What This Changes

The issue asks to benchmark and rank the active OpenRouter fallback models, define latency/quality/cost fallback ordering policy, add validation for unsupported `models.providers.*.models[].input` modalities, and update tests/docs.

## Best Possible Solution

Define a repeatable benchmark artifact for the named active fallback chain, document the chosen fallback-ordering policy in the model failover/models docs, then add capability-aware validation for configured model inputs using trusted provider or catalog metadata while preserving valid custom multimodal catalogs.
