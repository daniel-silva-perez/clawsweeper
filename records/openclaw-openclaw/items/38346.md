## Summary

Keep open. Current main still does not have a Telegram mode that observes or ingests all visible group messages while separately limiting agent triggers/replies to allowlisted senders. The existing contract is binary: `groupPolicy: "open"` lets all group senders through, while `groupPolicy: "allowlist"` rejects unauthorized senders before message processing. The current `ingest` hook only runs after an already-authorized message is skipped for missing mention.

## Best Possible Solution

Keep this open for bundled Telegram plugin product review and implementation. The best path is an explicit opt-in observe/ingest mode for configured Telegram groups/topics that can record or hook visible messages before trigger authorization, plus a separate sender trigger allowlist that gates agent turns, native/text commands, and visible replies. Update Telegram config schema/types/docs and add regression tests for unauthorized unmentioned ingest without reply, unauthorized mentioned no reply, and allowlisted mentioned reply.
