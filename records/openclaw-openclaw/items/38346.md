## Summary

Current main still does not provide a Telegram mode that observes or ingests all visible group messages while separately allowing only selected senders to trigger agent turns, commands, tools, or visible replies. `groupPolicy: "open"` still lets all group senders through the trigger path, while `groupPolicy: "allowlist"` rejects unauthorized senders before the existing mention-skipped ingest path can run.

## Best Possible Solution

Keep this open for bundled Telegram plugin product review. The best implementation is an explicit opt-in observe/ingest mode for configured Telegram groups/topics that can record or hook visible messages before trigger authorization, plus a separate trigger allowlist that gates agent turns, native/text commands, tool execution, and visible replies. Update Telegram config schema/types/docs and add regression coverage for unauthorized unmentioned ingest without reply, unauthorized mentioned no reply, and allowlisted mentioned reply.
