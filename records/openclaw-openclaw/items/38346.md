## Summary

Current main still does not provide a Telegram mode that observes or ingests all visible group messages while separately allowing only selected senders to trigger agent turns, commands, tools, or visible replies. `groupPolicy: "open"` still allows arbitrary group senders through the trigger path, while `groupPolicy: "allowlist"` rejects unauthorized senders before message context or the existing mention-skipped ingest hook can run.

## Best Possible Solution

Keep this open for bundled Telegram plugin product/security review. If approved, add an explicit opt-in observe/ingest-before-trigger mode for configured Telegram groups/topics, keep trigger authorization separate for agent turns, native/text commands, tool execution, and visible replies, and update Telegram config types/schema/docs plus regression tests for unauthorized unmentioned ingest without reply, unauthorized mentioned no reply, and allowlisted mentioned reply.
