## Summary

Keep this PR open. Current main still only supports remote MCP static headers for SSE/streamable-http; it does not expose `auth: "oauth"`, pass an MCP SDK `authProvider`, persist OAuth state, or retry `UnauthorizedError` with `finishAuth`. The PR is a plausible implementation candidate for the still-missing core feature, but it needs maintainer review and security hardening before merge.

## Best Possible Solution

Keep the PR open for maintainer review or ask the contributor to refresh it against current main, address the remaining OAuth callback/token-storage security issues, add schema/docs coverage for `mcp.servers.*.auth`, and include targeted tests for both streamable-http and SSE OAuth flows. If maintainers prefer a different design, open a canonical replacement PR before closing this one as superseded.
