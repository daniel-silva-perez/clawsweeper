## Summary

Keep this open. Current main still has no native exec-approvals denylist mode, denylist collection, `ask: "on-match"`, or deny-pattern evaluation branch, and the latest maintainer discussion explicitly keeps the native design open. The SafeClaw and CRE links are useful prior art, and the closed implementation PRs are relevant history, but none of that replaces a merged OpenClaw policy change.

## What This Changes

Add native exec-approvals denylist support so users can allow most host exec commands while blocking or prompting on dangerous command and argument patterns.

## Best Possible Solution

Define an additive native exec approvals policy that evaluates deny rules before broad allow decisions, supports consistent command and argument matching across gateway and node `system.run` paths, exposes configuration in CLI and Control UI, documents the security model, and coordinates overlap with #41140.
