## Summary

Keep this open. Current main still has no native exec-approvals denylist mode, denylist collection, `ask: "on-match"`, or deny-pattern evaluation branch; the SafeClaw and CRE links plus the closed PR attempts are useful prior art, but they do not replace a merged OpenClaw policy change. The latest maintainer comment also explicitly keeps the native design open for follow-up.

## What This Changes

Add denylist-style exec approval policy support so users can allow most host commands while prompting or blocking matching dangerous command patterns.

## Best Possible Solution

Define an additive native exec approvals policy that evaluates deny rules before broad allow decisions, supports consistent command and argument matching across gateway and node `system.run` paths, exposes the policy in CLI and Control UI, documents the security model, and coordinates overlap with #41140 instead of treating external plugins as the core fix.
