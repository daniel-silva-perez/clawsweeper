## Summary

Keep this issue open. Current main still has no native exec-approvals denylist mode, denylist collection, `ask: "on-match"`, or deny-match evaluation path. The SafeClaw and CRE plugin links in the discussion are useful prior art, and the related closed PRs show implementation attempts, but they do not replace a merged native OpenClaw policy change; the latest maintainer context also kept this security-sensitive design open.

## What This Changes

Request native exec-approvals support for allow-all-except-deny policies, including denylist rules, argument matching, and prompting or blocking when dangerous command patterns match.

## Best Possible Solution

Define an additive native exec approvals policy that evaluates deny rules before broad allow decisions, supports consistent command and argument matching across gateway and node `system.run` paths, exposes configuration in CLI and Control UI, documents the security model, and coordinates overlap with #41140 instead of treating external plugins as the core fix.
