## Summary

Keep this issue open. Current main still does not have first-class exec-approvals denylist support: the core policy enums, approvals file model, normalization, runtime prompt decision, node `system.run` policy, config schema, Control UI, docs, and focused tests are still built around `deny`, `allowlist`, and `full` plus allowlist entries. The SafeClaw and CRE links in the discussion are useful prior art, but they do not replace the requested OpenClaw core approval surface.

## Best Possible Solution

Keep this issue open until OpenClaw either implements first-class exec-approval denylist support across config/types/schema, approvals normalization/storage, gateway and node runtime policy, CLI, Control UI, docs, and tests, or maintainers explicitly choose a ClawHub/plugin-policy direction and track any missing pre-exec authorization hooks separately. Review #41140, #71097, #48532, and the closed implementation attempts #43222, #60684, and #72466 together before accepting another implementation PR.
