## Summary

Keep this open. Current main still models exec approvals around `deny | allowlist | full`, allowlist entries, and `off | on-miss | always`; there is no native denylist mode, denylist collection, `ask: "on-match"`, or deny-pattern evaluation branch. The SafeClaw and CRE links in the discussion are useful prior art, and the closed implementation PRs show previous attempts, but they do not replace a merged native OpenClaw policy change; a maintainer comment also explicitly kept the work open for follow-up.

## What This Changes

The issue requests native exec-approvals denylist support for allow-everything-except policies, including command-argument matching and prompt-on-match behavior.

## Best Possible Solution

Define an additive native exec approvals policy that evaluates deny rules before broad allow decisions, supports consistent command and argument matching across gateway and node `system.run` paths, exposes the policy in CLI and Control UI, documents the security model, and coordinates the overlap with #41140 instead of treating external plugins as the core fix.
