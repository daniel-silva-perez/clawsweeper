## Summary

Keep this issue open. Current main still exposes exec approvals as `deny | allowlist | full` plus allowlist entries only; there is no native denylist collection, `security: "denylist"`, `ask: "on-match"`, or deny-pattern evaluation path. The SafeClaw and CRE links in the discussion are useful prior art, and the closed implementation PRs show design attempts, but the native OpenClaw policy remains unmerged and maintainer discussion already kept the security design open.

## What This Changes

Add native exec-approvals denylist support so users can allow most host commands while prompting or blocking matching command and argument patterns.

## Best Possible Solution

Define an additive native exec approvals policy that evaluates deny rules before broad allow decisions, supports consistent command and argument matching across gateway and node `system.run` paths, exposes configuration in CLI and Control UI, documents the security model, and coordinates overlap with #41140 instead of treating external plugins as the core fix.
