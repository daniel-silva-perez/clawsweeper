## Summary

Keep this issue open. Current main still models exec approvals as `deny | allowlist | full`, with no native `denylist` mode, `denylist` array, `ask: on-match`, or cross-platform command-argument deny matching. The CRE plugin linked in the discussion is useful prior art/workaround, #41140 is a related open tracker, and the closed implementation PRs did not merge.

## Best Possible Solution

Keep this issue open for a native, additive exec-approvals deny-pattern design. A good implementation should cover config/types/schema, effective policy evaluation, gateway and node exec paths, CLI/Control UI editing, docs, and focused tests, while using CRE and the closed PRs as prior art rather than proof that current main is done. Maintainers should also decide whether #41140 becomes the canonical design tracker or remains a narrower related issue.
