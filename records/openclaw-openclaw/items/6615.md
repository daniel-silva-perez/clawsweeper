## Summary

Keep this issue open. Current main and the latest release still model exec approvals as `deny | allowlist | full`, with no `denylist` security mode, no `denylist` array, no `ask: on-match`, and no general command-argument deny matching. The CRE plugin linked in the discussion is useful prior art/workaround, and #41140 is a related open tracker, but neither means this core exec-approvals request is implemented.

## Best Possible Solution

Keep this issue open and use it, or an explicitly linked canonical follow-up such as #41140, to design a native exec-approvals deny-pattern feature. A good fix should be additive and cover the config/schema, effective policy evaluation, gateway and node exec paths, CLI/UI editing, docs, and focused tests; prior plugin/PR work should be treated as useful context rather than proof that main is done.
