## Summary

Keep this open. Current main still lacks native exec-approvals denylist policy support, and the linked external implementations plus closed PR attempts are useful prior art rather than a merged OpenClaw core policy change.

## What This Changes

The issue requests native exec-approvals denylist support, including a denylist collection, argument-aware matching, and prompt-on-match behavior for allow-everything-except policies.

## Best Possible Solution

Define an additive native exec approvals policy that evaluates deny rules before broad allow decisions, supports consistent command and argument matching across gateway and node system.run paths, exposes the policy in CLI and Control UI, documents the security model, and coordinates the overlap with #41140 instead of treating external plugins as the core fix.
