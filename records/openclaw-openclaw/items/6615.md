## Summary

Keep this open. Current main still has no native exec-approvals denylist mode, denylist collection, `ask: "on-match"`, or deny-pattern evaluation path; the linked SafeClaw and CRE implementations plus the closed implementation PRs are useful prior art, but they do not replace a merged OpenClaw core policy change. The maintainer comment in the provided timeline also explicitly keeps the native design open for follow-up.

## What This Changes

Request to add native exec-approvals denylist support with argument-aware matching and prompt-on-match behavior so broad allow policies can still block or prompt on dangerous commands.

## Best Possible Solution

Define an additive native exec approvals policy that evaluates deny rules before broad allow decisions, supports consistent command and argument matching across gateway and node `system.run` paths, exposes the policy in CLI and Control UI, documents the security model, and coordinates overlap with #41140 instead of treating external plugins as the core fix.
