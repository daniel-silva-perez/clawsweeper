## Summary

Keep this PR open. Current main has the PR's QMD `--mask` default and newer same-name/path+pattern repair logic, but it does not implement the PR's central runtime-search change: managed QMD collection names are still computed once, and missing-collection repair retries with the same captured collection list. The PR also has unresolved review concerns, so the best path is rework against current main rather than auto-close.

## Best Possible Solution

Keep this PR open for maintainer review, but ask for a rebase/rework against current main. The useful direction is a safe search-time or post-reconciliation collection selection that never fails open to unscoped QMD searches, preserves valid collections when `collection list` metadata is unavailable, respects explicit custom/external collection names, recomputes after repair, and adds regression tests for the missing-materialized-collection path plus the retry path. The already-landed `--mask` default can be dropped from the refreshed diff unless it still needs adjustment.
