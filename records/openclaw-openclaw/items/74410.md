## What This Changes

The issue reports that meridian and direct Anthropic anthropic-messages streaming can drop the visible text block after a thinking block, leaving persisted assistant content empty with zero output usage and triggering fallback retries.

## Best Possible Solution

Add a focused anthropic-messages regression that feeds a stream containing a thinking block followed by visible text, verifies the final assistant content and assistantTexts keep the text, and patches the reader so non-empty upstream text cannot collapse into an empty persisted turn before fallback logic runs.
