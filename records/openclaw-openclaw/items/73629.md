## Summary

Keep this PR open. Current main still resolves exec approval storage from the HOME-derived `~/.openclaw` default, so it does not yet implement the requested `OPENCLAW_STATE_DIR` behavior or the regression coverage. The PR is a focused implementation candidate for a plausible current bug.

## Best Possible Solution

Keep this PR open for normal maintainer review. The likely best path is to route `resolveExecApprovalsPath()` and `resolveExecApprovalsSocketPath()` through `resolveStateDir(env)`, preserve the default fallback behavior, and land focused regression coverage for `OPENCLAW_STATE_DIR`.
