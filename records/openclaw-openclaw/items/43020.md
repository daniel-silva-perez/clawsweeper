## Summary

Keep this issue open. Current main still routes Mattermost inbound preview streaming through one draft post for the whole run, disables text block streaming on that path, and does not flush, seal, or rotate the draft at tool or next-assistant boundaries. The related implementation PRs were closed unmerged, and current docs describe the single-draft behavior rather than the requested independent per-turn Mattermost posts.

## Best Possible Solution

Keep the issue open and fix it in the Mattermost plugin inbound preview-streaming path. When visible assistant text exists and a tool or next-assistant boundary arrives, finalize or seal the current draft post, rotate to a fresh draft for the next assistant segment, and suppress final payloads already delivered through preview streaming. Add a focused regression test covering first partial text, tool boundary, second partial text, and final delivery without duplicate posts.
