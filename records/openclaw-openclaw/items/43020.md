## Summary

Keep this issue open. Current main still routes Mattermost inbound preview streaming through one draft preview post for the run, explicitly disables core block streaming on that path, and only clears local partial-text state at assistant/reasoning boundaries instead of flushing, sealing, or rotating the draft. The related implementation PRs #43041 and #46330 were closed unmerged in the supplied GitHub context, so there is no merged canonical fix to close against.

## Best Possible Solution

Keep the issue open and fix the Mattermost plugin inbound preview-streaming path. At the first boundary after visible assistant text, such as a tool start or next assistant message, flush and seal the current draft preview, rotate to a fresh draft for the next visible segment, and suppress any final payloads already delivered through preview streaming. Add a focused regression test covering first partial text, tool boundary, second partial text, and final delivery without duplicate posts. The closed #43041 and #46330 discussions are useful design context but need a fresh implementation or revival before this can close.
