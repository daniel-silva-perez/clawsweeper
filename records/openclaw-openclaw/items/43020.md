## Summary

Keep this issue open. Current main still wires inbound Mattermost edit-in-place streaming to one draft preview for the whole run, disables core text block streaming, and only clears local partial-text state at assistant boundaries. The linked fix attempts #43041 and #46330 are both closed/unmerged in the provided GitHub context, so there is no merged canonical fix to close against.

## Best Possible Solution

Fix this in the Mattermost plugin inbound streaming path: on assistant/tool boundaries, flush or preserve the current draft preview, rotate to a fresh Mattermost post before the next text segment, and suppress duplicate final replies for segments already delivered through preview streaming. Add a focused Mattermost regression test for partial text, assistant/tool boundary, second partial text, and final delivery.
