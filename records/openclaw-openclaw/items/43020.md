## Summary

Keep this issue open. Current main still routes Mattermost preview streaming through one draft post for the whole inbound run, disables core block streaming on that path, and only resets local partial-text state at assistant/reasoning boundaries. It does not flush, seal, or rotate the Mattermost draft post at tool/assistant boundaries, so the requested independent per-turn Mattermost posts are still not implemented. The newer preview-finalization code improves single-final cleanup/final edit behavior but does not solve the multi-turn boundary case.

## Best Possible Solution

Fix this in the Mattermost plugin inbound preview-streaming path. When a draft has visible assistant text and a tool/next-assistant boundary arrives, finalize or seal that draft post, rotate to a fresh Mattermost draft before the next assistant segment, and suppress final payloads already delivered through preview streaming. Add a focused Mattermost regression test covering first partial text, tool boundary, second partial text, and final delivery without duplicates.
