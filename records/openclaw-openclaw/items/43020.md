## Summary

Keep this issue open. Current main still matches the reported Mattermost multi-turn preview-streaming problem: the inbound path creates one draft preview stream for the whole run, disables core block streaming, and only resets local partial-text state at assistant/reasoning boundaries. It does not flush, seal, or rotate the Mattermost preview post at tool/assistant boundaries, and the related fix PRs #43041 and #46330 are closed unmerged in the provided GitHub context.

## Best Possible Solution

Fix this in the Mattermost plugin inbound preview-streaming path. When a draft has visible assistant text, finalize or seal it at assistant/tool boundaries, rotate to a fresh Mattermost draft post before the next assistant segment, and suppress final payloads already delivered through preview streaming. Add a focused Mattermost regression test that covers first partial text, tool boundary, second partial text, and final delivery without duplicate messages.
