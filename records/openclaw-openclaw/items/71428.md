## Summary

Keep open. Current main still has host-config-only enforcement for raw conversation-content hooks, the native plugin manifest and manifest registry do not model `permissions.conversationAccess`, and open PR #72690 is already the linked implementation candidate for this issue.

## What This Changes

The issue requests an additive `openclaw.plugin.json` manifest field for plugin conversation-access metadata while keeping the existing `plugins.entries.<id>.hooks.allowConversationAccess` config path backward-compatible.

## Best Possible Solution

Review and land PR #72690 or an equivalent narrow patch that adds manifest-level conversation-access request metadata, preserves it through manifest registry/status surfaces, keeps host config as the effective operator approval gate, and updates manifest, hooks, config docs, and loader/registry tests.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a non-bundled plugin declaring `permissions.conversationAccess: true` in its manifest and registering a conversation typed hook: current main ignores that manifest field and only accepts the hook when `plugins.entries.<id>.hooks.allowConversationAccess=true` is set in host config.

## Solution Assessment

Unclear for the original RFC sketch because manifest-owned metadata should not self-grant transcript access. The safer maintainable path is manifest request/disclosure metadata plus explicit host approval, which is the direction described by open PR #72690.

## Security Review

Status: needs_attention

Summary: This issue is security-sensitive because it changes metadata and approval semantics for non-bundled plugin access to raw conversation-content hooks.

Concerns:

- **[medium] Preserve operator approval for transcript access:** `src/plugins/registry.ts:1956`
  - body: If `permissions.conversationAccess` becomes an effective grant from plugin-owned manifest data, an untrusted plugin could self-authorize access to raw conversation hooks. The contract should treat the manifest field as request/disclosure metadata or otherwise require explicit host approval with documented deny precedence.
  - confidence: 0.9
