## Summary

Keep open. Current main still implements conversation-content hook access through host config only, the native plugin manifest contract still has no `permissions.conversationAccess` field or registry/status preservation path, and open PR #72690 is already the linked implementation candidate awaiting maintainer review.

## What This Changes

Request an additive native plugin manifest field for declaring conversation-access needs while preserving the existing `plugins.entries.<id>.hooks.allowConversationAccess` host config flag for compatibility and precedence.

## Best Possible Solution

Review and land PR #72690 or an equivalent narrow patch that adds manifest-level conversation-access request metadata, preserves it through manifest registry/status surfaces, keeps explicit operator approval as the effective grant, and updates manifest, hooks, and config docs plus loader and registry tests.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a non-bundled plugin registering `llm_input`, `llm_output`, `before_agent_finalize`, or `agent_end`: current registry code and loader tests show those hooks are blocked without host config and accepted only with `plugins.entries.<id>.hooks.allowConversationAccess=true`, while manifest parsing ignores `permissions.conversationAccess`.

## Solution Assessment

Unclear for the RFC sketch as written. A plugin-owned manifest value should not grant raw conversation access by itself; the safer maintainable solution is manifest request/disclosure metadata plus explicit host approval and documented precedence.

## Security Review

Status: needs_attention

Summary: This issue is security-sensitive because it changes metadata and approval semantics for non-bundled plugin access to raw conversation-content hooks.

Concerns:

- **[medium] Preserve operator approval for transcript access:** `src/plugins/registry.ts:1956`
  - body: If `permissions.conversationAccess` becomes an effective grant from a plugin-owned manifest, an untrusted plugin could self-authorize access to raw conversation hooks. The implementation should keep the manifest as request/disclosure metadata or otherwise require explicit host approval and deny precedence.
