## Summary

Keep this PR open. Current main still does not forward `tools.exec.host`, `tools.exec.security`, or `tools.exec.ask` into the chat-bash exec tool, so the intended fix is not implemented on main. The PR also still has a real unresolved review concern: its second commit narrows `!` routing in a way that would drop valid punctuation-prefixed shell commands, so it should be updated rather than closed as obsolete or merged as-is.

## Best Possible Solution

Keep the PR open and ask for a focused update: preserve the `handleBashChatCommand` forwarding of `tools.exec.host`, `tools.exec.security`, and `tools.exec.ask`; add regression coverage proving chat-bash inherits those exec defaults; and drop or rework the markdown-image/bang-routing change so valid `!` aliases with punctuation-prefixed shell commands still route to chat-bash. A stronger final patch may use a shared exec-default resolver so chat-bash stays aligned with regular exec behavior as session/per-agent defaults evolve.
