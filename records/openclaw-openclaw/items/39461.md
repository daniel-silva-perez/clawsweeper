## Summary

Current main still does not implement the reported QQBot `QQBOT_DATA_DIR` override. The QQBot storage helper still derives persistent state from `getHomeDir()/.openclaw/qqbot`, diagnostics still recommends `QQBOT_DATA_DIR`, docs and plugin metadata still omit it, and focused tests only cover the home-derived default path.

## Best Possible Solution

Implement `QQBOT_DATA_DIR` in the QQBot plugin's central persistent-storage helper, with trimmed/non-empty env handling, subpath joining, and lazy directory creation preserved. Diagnostics should report the effective data dir and avoid recommending the override once it is already in effect. Add focused default/override tests, document the env var in the QQ Bot docs and plugin metadata, and add a user-facing changelog fix crediting the reporter.
