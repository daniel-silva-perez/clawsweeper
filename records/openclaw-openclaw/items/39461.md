## Summary

Current main still ignores `QQBOT_DATA_DIR` for QQ Bot persistent storage while diagnostics recommends it, so the reported Windows path bug remains valid and narrow enough for a focused fix PR.

## Best Possible Solution

Keep this issue open. Implement `QQBOT_DATA_DIR` in the central QQ Bot storage helper, most likely `extensions/qqbot/src/engine/utils/platform.ts:getQQBotDataPath()`, preserving subpath joining, path normalization, and lazy directory creation. Update diagnostics so an ASCII env override is reflected in `dataDir` and does not keep recommending the same override unnecessarily, add focused tests for the env override and laziness behavior, and align QQ Bot docs with the supported env var.
