## Summary

Current main still ignores `QQBOT_DATA_DIR` for QQ Bot persistent storage while the diagnostics warning recommends that same variable, so the reported Windows path bug remains valid and narrow enough for a focused fix PR.

## Best Possible Solution

Keep this issue open and implement `QQBOT_DATA_DIR` in the QQ Bot plugin's central persistent-storage helper, most likely `extensions/qqbot/src/engine/utils/platform.ts:getQQBotDataPath()`. Preserve subpath joining and lazy creation, update diagnostics so the override is reflected and no longer recommended when already set, add focused tests, and document the env var in the QQ Bot channel docs with a user-facing changelog fix entry crediting the reporter.
