## Clownfish Work Prompt

Fix the QQBot plugin bug where setting `QQBOT_DATA_DIR` does not change the persistent data directory. On current main, `extensions/qqbot/src/engine/utils/platform.ts:getQQBotDataPath()` always returns `getHomeDir()/.openclaw/qqbot`, while diagnostics recommends `QQBOT_DATA_DIR` for Windows home paths with Chinese characters or spaces.

Expected fix boundary:
- Implement `QQBOT_DATA_DIR` support in the central QQBot persistent-storage helper only, most likely `getQQBotDataPath(...subPaths)` / `getQQBotDataDir(...subPaths)`.
- Preserve subpath joining and the existing lazy-creation contract: path helpers must not create directories, creation remains in `getQQBotDataDir` or existing write paths.
- Use the env var as the data root when it is non-empty; keep default behavior unchanged when unset.
- Update diagnostics so `dataDir` reflects the effective path and the Windows warning does not recommend an override that is already in effect.
- Add focused tests for default root, env override root, subpath behavior, and laziness. Clean any env changes in test teardown.
- Update `docs/channels/qqbot.md` and plugin env metadata if maintainers want the variable discoverable through channel env vars.
- Add a user-facing changelog fix entry crediting `@jiahao-0111`.

Do not migrate, delete, or rewrite existing state under `~/.openclaw/qqbot` in this narrow fix. Do not change QQBot media roots, payload allowlist behavior, auth defaults, or unrelated channel behavior.
