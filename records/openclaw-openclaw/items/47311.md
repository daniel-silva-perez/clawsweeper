## Summary

Current main still lacks the requested WebChat server-side STT dictation path. The UI can attach audio files and start realtime Talk, and Gateway/media-understanding can process audio attachments through `tools.media.audio`, but there is no MediaRecorder/getUserMedia batch recorder that sends microphone audio to Gateway and inserts the returned transcript into the chat draft. Related #73634 is closed and #67465 covered the realtime Talk visibility mismatch, so this issue remains the useful tracker for batch server-side STT in WebChat.

## Best Possible Solution

Implement a focused Control UI/WebChat batch dictation path: record browser microphone audio with `getUserMedia` plus `MediaRecorder`, send the completed recording through an authenticated additive Gateway RPC that reuses existing media attachment limits/storage and `tools.media.audio`/`transcribeAudioFile`, return the transcript without starting an agent run, and insert or append that text into the chat draft. Keep this separate from realtime Talk and generic file attachments, then update WebChat/Control UI/audio docs and focused tests.
