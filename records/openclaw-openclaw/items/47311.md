## Summary

Keep this issue open. Current main has adjacent pieces: WebChat can upload non-video files including audio, Gateway can route non-image attachments through `chat.send` into `MediaPaths`, and audio media understanding can transcribe those attachments through `tools.media.audio`. But the specific requested product behavior is still missing: a WebChat microphone/MediaRecorder dictation path that records browser audio and returns server-side STT text to the composer/input. Current UI exposes Talk realtime voice instead, and the UI test explicitly asserts the old browser dictation button is absent.

## Best Possible Solution

Implement a Control UI/WebChat dictation path that records browser microphone audio with `getUserMedia` plus `MediaRecorder`, sends the recording through the existing authenticated `chat.send` attachment/media-understanding path or a narrow Gateway transcription RPC, and inserts the returned `tools.media.audio` transcript into the draft. Keep this distinct from realtime Talk, and update the Control UI/WebChat and audio docs so users understand browser dictation, server STT, and Talk mode as separate surfaces.
