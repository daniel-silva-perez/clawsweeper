## Summary

Keep open. Current main still closes Feishu HTTP servers and immediately deletes or clears `httpServers` without awaiting the close callback, and the webhook abort cleanup path has the same pattern. The issue is also paired with open PR #48588, which directly targets this bug, so the right cleanup path is to review, land, or replace that PR before closing this issue.

## Best Possible Solution

Keep this issue open and review/land #48588, or replace it with an equivalent maintainer-owned PR that makes Feishu HTTP server shutdown awaitable in both `monitor.state.ts` and `monitor.transport.ts`, force-closes active connections where appropriate, handles timeout/error cases, and adds focused regression coverage. Keep the broader ResourceManager work in #48793/#49804 as separate architecture follow-up.
