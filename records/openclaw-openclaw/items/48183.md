## Summary

Keep open. Current main still has the reported Feishu HTTP server cleanup race: `stopFeishuMonitorState()` calls `server.close()` and immediately deletes or clears `httpServers`, and webhook abort cleanup in `monitor.transport.ts` has the same immediate close/delete pattern. The issue is also paired with open PR #48588, which explicitly targets this bug, so the right path is to review, land, or replace that PR before closing this issue.

## Best Possible Solution

Keep this issue open and review/land #48588, or replace it with an equivalent maintainer-owned PR that makes Feishu HTTP server shutdown awaitable in both `monitor.state.ts` and `monitor.transport.ts`, force-closes active connections where appropriate, adds timeout/error handling, and adds focused regression coverage. Close this issue only after that fix merges and ships or is otherwise verified on main; keep the broader ResourceManager discussion in #48793/#49804 as separate follow-up work.
