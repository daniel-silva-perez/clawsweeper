## Summary

Keep this PR open. Current main still does not implement the PR's intended generic cleanup for ordinary prompt aborts: it only strips aborted assistant messages on the sessions_yield abort path, while non-yield RunnerAbortError paths record promptError and continue through snapshot/prompt-error handling. The linked bug #48241 remains open, and this PR is still a plausible implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Review and revise this PR rather than closing it: add a shared or clearly scoped helper that runs on the ordinary RunnerAbortError prompt path, strips only the current trailing aborted assistant message from both in-memory messages and session-file entries, adds a regression test for #48241, and resolves the duplicate-helper/_rewriteFile review comments before landing.
