## Summary

Keep this PR open. Current main still only strips aborted assistant artifacts for the sessions_yield path; ordinary RunnerAbortError prompt failures still flow into the snapshot/prompt-error path without removing trailing aborted assistant messages. The PR remains a plausible implementation candidate, but it is stale against current main and needs refactoring plus tests before merge.

## Best Possible Solution

Keep this PR open and revise it. The best path is a shared tail-cleanup helper used by both sessions_yield and ordinary RunnerAbortError prompt paths, updating activeSession.agent.state.messages plus SessionManager fileEntries/byId/leafId consistently, then adding a focused regression test for #48241 and resolving the duplicate-helper/_rewriteFile review comments before merge.
