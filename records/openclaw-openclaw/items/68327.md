## Summary

Keep this PR open. Current main overlaps with parts of the branch, including registry-backed gateway discovery, gated startup tracing, Windows UI launcher quoting, and parallel startup maintenance, but it still does not implement the PR's central secrets-runtime auth-store shortcut or the proposed async-lock timeout. The PR also still needs cleanup before it could merge because its diff includes unconditional production profiling logs and a security-sensitive Windows shell-spawn change.

## Best Possible Solution

Keep the PR open, but require a rebase and narrower implementation. The best path is to preserve current main's registry-backed discovery, gated startup trace, and existing Windows UI launcher tests; remove temporary profiling logs; split the secrets-startup loader so SecretRef snapshot resolution can avoid external auth overlays without changing OAuth refresh/recovery callers; add targeted regression tests for that split and for any async-lock timeout release/timer cleanup; then validate with the gateway startup benchmark and the appropriate changed gate.

## Risks / Open Questions

- The auth-store shortcut in the PR can regress external OAuth refresh/recovery because OAuth paths currently use loadAuthProfileStoreForSecretsRuntime outside the gateway startup secrets-preflight path.
- The PR head includes unconditional production stderr profiling logs; those should be removed or routed through the existing gated startup trace facility before merge.
- The scripts/ui.js shell-spawn change is security-sensitive because it changes cmd.exe execution behavior on Windows; any retained fix needs the existing argument sanitizer and targeted tests preserved.
- The branch is partly superseded by current main, so merging as-is risks reintroducing stale discovery/startup code or hiding the narrower remaining fix behind unrelated changes.
