## Summary

Keep open. Current main still has the backend session channel metadata and channel catalog labels, but the Control UI Sessions row type, live-update merge allowlist, and Sessions view props/rendering do not carry or render channel identity as a label/icon badge. This remains a narrow UI/data-contract fix.

## What This Changes

Request to show a channel label/icon, and optionally custom labels, in the Control UI Sessions list so channel-scoped sessions are distinguishable from truncated raw session keys.

## Best Possible Solution

Use the existing Gateway session metadata and channel catalog as the source of truth: carry channel identity through the Control UI row contract and live updates, pass channel catalog metadata into the Sessions view, and render a stable channel label/badge/icon while preserving custom labels, search, sorting, pagination, selection, and chat navigation.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: a row for a key such as `agent:main:telegram:direct:7633` reaches `renderRows` without typed `row.channel` or catalog props, so the table renders the raw key or `Agent name (telegram)` rather than a resolved Telegram/Feishu label or icon.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse existing Gateway channel metadata and the existing channels.status catalog in the UI; adding channel-specific hardcoded labels or new core channel IDs would be less maintainable.
