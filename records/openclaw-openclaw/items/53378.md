## Summary

Keep this issue open. Current main still has the Gateway-side channel metadata needed for the feature, but the Control UI Sessions contract, live-update merge allowlist, and Sessions table rendering do not carry or display channel identity; the optional custom label part is already present through the existing Label input.

## What This Changes

The issue asks for the Control UI Sessions list to show a channel name or icon next to each session entry instead of relying on raw or truncated channel-bearing session keys.

## Best Possible Solution

Carry channel identity through the Control UI Sessions row type and live merge path, resolve it through the existing channel catalog metadata, and render a stable channel label or badge in the Sessions list while preserving custom Label editing, agent identity labels, search, sort, and pagination.

Implement a narrow Control UI Sessions channel identity display for https://github.com/openclaw/openclaw/issues/53378. Current main already exposes channel metadata from the Gateway (`GatewaySessionRow`, `sessions.list`, and `sessions.changed`), but the Control UI row type omits the fields, `SESSION_EVENT_ROW_FIELDS` drops them, and `renderSessions` only shows raw keys or agent-identity-derived labels.
