## Summary

Keep this issue open. Current main still does not render a channel icon/name in the Control UI Sessions list: the table has no channel column or badge, the key cell falls back to the raw session key when agent identity is unavailable, and the Control UI type/event merge path still omits several gateway channel fields. The prior ClawSweeper comment remains accurate; the backend already exposes enough channel metadata, so this is a narrow UI/data-contract repair rather than a core channel change.

## What This Changes

Show channel identity, such as a channel name or icon, next to each Control UI session-list entry and optionally support clearer custom session labels.

## Best Possible Solution

Carry channel identity through the Control UI Sessions contract and render it in the Sessions list using existing gateway row metadata plus channel labels/meta, while preserving custom Label editing, existing agent-identity labels, sorting, filtering, and live sessions.changed updates.
