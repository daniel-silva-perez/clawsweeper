## Summary

Current main still does not implement the requested Control UI Sessions channel identity display. The Sessions table has no Channel column/badge, the key cell only substitutes an agent identity label when one exists, and the UI row/event contract still omits the gateway channel metadata that backend session rows already expose.

## Best Possible Solution

Keep this issue open. Implement the Sessions list feature in the Control UI by carrying channel, groupChannel, chatType, and origin through the UI GatewaySessionRow type and SESSION_EVENT_ROW_FIELDS; pass channelLabels/channelMeta from the existing channels snapshot into renderSessions; render a compact Channel column or key-cell badge for Telegram, Feishu, and plugin channels while preserving current Label and agent-identity behavior; add focused UI tests for Telegram/Feishu rows without agent identity plus an update-event preservation test.
