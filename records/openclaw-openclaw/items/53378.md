## Summary

Keep this issue open. Current main still has the reported Control UI gap: the Sessions table does not render channel identity, the UI row type and live merge allowlist omit the gateway channel metadata, and existing tests still assert raw-key fallback for a Telegram session without agent identity. The prior ClawSweeper comment correctly identified the same gap, and the current tree still matches that finding.

## What This Changes

The issue asks the Control UI Sessions list to show a channel name or icon for channel-backed sessions, with optional custom session labels.

## Best Possible Solution

Carry channel identity through the Control UI Sessions contract and render it in the Sessions list using gateway row metadata plus existing channel labels/meta, while preserving the current agent-identity label and user-editable Label behavior.

## Repair Work Prompt

Fix the Control UI Sessions list so channel-backed sessions show a user-friendly channel identity instead of only raw session keys. Use the existing gateway row metadata first: add channel, groupChannel, chatType, and origin to the UI GatewaySessionRow type and SESSION_EVENT_ROW_FIELDS; pass channelLabels/channelMeta from the channels snapshot into renderSessions; render a compact Channel column or key-cell badge for Telegram, Feishu, and plugin channels while preserving current Label input behavior and the existing agent-identity friendly key path. Do not move owner-specific channel defaults into core and do not rely only on parsing raw session keys except as a fallback. Add focused tests for Telegram and Feishu rows without agent identity, plugin-channel label fallback from channelMeta/channelLabels, and a sessions.changed event preserving channel metadata. Because this is user-facing UI behavior, add a single-line changelog entry in the active Fixes section with Thanks @gelibing8-rgb unless the maintainer decides the changelog is not needed.
