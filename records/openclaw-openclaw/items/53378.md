## Summary

Keep this issue open. Current main still has no Control UI Sessions channel badge/column, still falls back to raw session keys when agent identity is unavailable, and still drops channel fields from the UI row type and live merge allowlist even though the Gateway already exposes them.

## What This Changes

Show channel identity, such as Telegram or Feishu name/icon, next to each Control UI session-list entry and optionally support clearer custom session labels.

## Best Possible Solution

Carry channel identity through the Control UI Sessions contract and render it in the Sessions list using existing Gateway row metadata plus channel labels/meta, while preserving custom Label editing, existing agent-identity labels, sorting, filtering, and live sessions.changed updates.
