## Summary

Keep this PR open. Current main still does not implement the Android chat message copy/text-selection workflow: only code blocks are selectable, chat bubbles have no long-press/context-menu plumbing, and the composer/send path has no reply target state. The linked feature issue #57754 is still open, and this PR remains a plausible implementation candidate after addressing the review comments.

## Best Possible Solution

Keep this PR open and either review/update it or implement an equivalent Android UI change on main: long-press chat bubbles should expose Copy, Select Copy, Share, and Reply; reply intent should be serialized or deliberately scoped; empty/non-text messages should be handled explicitly; and the linked #57754 should close only after the implementation lands.
