## Summary

Keep open. Current main still lacks Android chat bubble copy/select/share/reply actions, and the linked feature issue remains open; the PR is useful source work but its latest patch still has a concrete reply-formatting bug for multi-line quoted messages.

## What This Changes

This PR adds long-press Android chat bubble actions for Copy, Select Copy, Share, and Reply, plus a reply preview and reply-to-text serialization in the composer.

## Best Possible Solution

Keep this PR open, update it against current main, and fix reply serialization so multi-line quoted content is represented unambiguously, either by prefixing every quoted line or by adding a structured reply payload if the Android/backend contract grows one.

## Reproduction Assessment

Yes for the review finding: reply to a message whose text contains multiple lines and send a response. The patch builds the quote from newline-joined text but prefixes only the first line, so later lines are sent as plain prompt text.

## Solution Assessment

No, not yet. The long-press bottom sheet is a plausible UI approach for the requested Android feature, but the current reply-send formatter is not the narrowest correct implementation until multi-line quote handling is fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Quote every reply line before sending:** `apps/android/app/src/main/java/ai/openclaw/app/ui/chat/ChatSheetContent.kt:165`
  - body: The formatter builds `quotedText` with embedded newlines but only emits one `> ` prefix. When a user replies to a multi-line message, every line after the first is sent as ordinary user text instead of quoted context, which can change the prompt meaning. Prefix each quoted line or serialize reply context separately before calling `sendChat`.
  - confidence: 0.88
