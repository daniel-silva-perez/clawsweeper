## Summary

Keep this draft PR open. Current main still handles node `exec.started`, `exec.finished`, and `exec.denied` by canonicalizing the session key, enqueueing a system event, and requesting an `exec-event` heartbeat wake without checking the `heartbeatIsolatedBaseSessionKey` marker. The PR is a narrow implementation candidate for a remaining heartbeat-isolated self-wake loop, and the provided diff does not show security-sensitive or supply-chain changes.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or adapt the narrow `server-node-events` guard plus the focused regression test after confirming synthetic heartbeat sessions should not re-enqueue their own node exec lifecycle events, while preserving normal node exec notifications for non-heartbeat sessions.
