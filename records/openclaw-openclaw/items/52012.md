## Summary

Keep this PR open. Current main still lacks the requested `agents.defaults.compaction.modelFallbacks` config surface and has no equivalent compaction-specific model fallback path; direct and queued compaction resolve a single compaction target and fail or retry only thinking level. The PR is still an implementation candidate, but it needs maintainer review because it touches compaction auth, runtime-auth, provider transport, checkpoints, and session lifecycle behavior.

## Best Possible Solution

Keep the PR open for maintainer review or a narrower replacement. The best implementation should add `agents.defaults.compaction.modelFallbacks` consistently across public types, strict schema, generated/help metadata, docs, direct compaction, queued/context-engine compaction, and focused tests for unset behavior, resolution/auth/runtime fallback, all-fallbacks-fail diagnostics, and non-regression of existing thinking-level fallback, checkpoint, prompt, auth-profile, and runtime-auth behavior.
