## Summary

Keep open. Current main still does not implement `agents.defaults.compaction.modelFallbacks`, so the PR is not obsolete, but the current patch rewrites a large part of direct compaction and drops several main-branch runtime guarantees around auth/transport wrapping, prompt policy, checkpoint persistence, small-context settings, and existing retry behavior.

## What This Changes

The PR adds `AgentCompactionConfig.modelFallbacks` to types/schema and rewrites embedded PI direct compaction so resolution, auth, and runtime compaction failures can retry configured fallback models.

## Best Possible Solution

Keep the useful `modelFallbacks` idea, but rework it as a narrow addition on top of current main: parse/validate the new config field, reuse the existing compaction target/auth/session/prompt/stream/checkpoint helpers, add focused tests for resolution/auth/runtime fallback order, and update the public compaction docs and generated config help as needed.

## Reproduction Assessment

Yes for the blocking review findings: a static diff against current main reproduces the regressions by showing removed wrapper, checkpoint, prompt, settings, and retry calls in the final PR head. Runtime tests would be needed before merge, but the source-level failures are high-confidence.

## Solution Assessment

No. The current PR is not the best way to solve the feature because it rewrites and regresses established compaction behavior; the safer path is a smaller fallback loop integrated into the existing current-main helpers and tests.

- **[P1] Route provider streams through the compaction wrapper:** `src/agents/pi-embedded-runner/compact.ts:959`
  - body: This assigns `providerStreamFn` directly, bypassing the current `resolveEmbeddedAgentStreamFn`/extra-param/text-transform stack that injects runtime auth, OAuth bearer state, abort signals, WebSocket transport, and provider transport settings.   - confidence: 0.93

  - body: `createPreparedEmbeddedPiSettingsManager` is now called without `contextTokenBudget`. The reserve-floor cap in `pi-settings.ts` only runs when that value is present, so small-context models can keep the default 20k reserve floor and repeatedly overflow or compact instead of stabilizing.
  - confidence: 0.9
- **[P2] Restore compaction checkpoint persistence:** `src/agents/pi-embedded-runner/compact.ts:1238-1242`
  - body: After a successful compact this path now runs post-compaction side effects and returns without capturing or persisting a compaction checkpoint. Current main uses `persistSessionCompactionCheckpoint` for `sessions.compaction` restore/list behavior, so standard direct compaction would lose fresh restore checkpoints.
  - confidence: 0.88
- **[P2] Compute fallback success metrics from the fallback session:** `src/agents/pi-embedded-runner/compact.ts:1244-1252`
  - body: When primary `compact()` fails and `fbSession.compact()` succeeds, the successful state lives on `fbSession`, but `tokensAfter`, `messageCountAfter`, diagnostics, hooks, and truncation are computed from the original failed `session`. That can report stale token counts and make follow-up compaction decisions wrong after the new fallback path succeeds.
  - confidence: 0.87
- **[P1] Preserve the no-fallback thinking retry:** `src/agents/pi-embedded-runner/compact.ts:1143-1149`
  - body: With no `modelFallbacks` configured, this branch throws the compaction error immediately. Current main catches compaction failures and retries once with `pickFallbackThinkingLevel` for reasoning-required providers, so existing configs regress even when the new option is unset.
  - confidence: 0.9
- **[P2] Build prompts through the existing policy helpers:** `src/agents/pi-embedded-runner/compact.ts:843-872`
  - body: The new prompt path hardwires `buildEmbeddedSystemPrompt` and no longer applies `resolveSystemPromptOverride`, provider system-prompt contributions, or provider prompt transforms. Agent-level prompt overrides and provider prompt hooks can be ignored during compaction.
  - confidence: 0.85
- **[P2] Pass sender ownership into action discovery:** `src/agents/pi-embedded-runner/compact.ts:785-797`
  - body: `buildEmbeddedMessageActionDiscoveryInput` no longer receives `senderIsOwner`, while current main passes it. Owner-scoped channel actions can disappear from compaction prompts for owner messages, making compaction prompt guidance diverge from normal runtime action discovery.
  - confidence: 0.82
