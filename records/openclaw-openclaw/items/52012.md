## Summary

Keep this PR open. Current main still does not implement `agents.defaults.compaction.modelFallbacks` or an equivalent compaction-specific fallback loop. Normal agent model fallback support exists, but the compaction paths still resolve one effective compaction target and fail/retry only the existing thinking-level fallback. The PR remains an implementation candidate, though its current `compact.ts` diff needs careful maintainer review around auth, runtime-auth, transport wrapping, checkpoints, and session lifecycle.

## Best Possible Solution

Keep the PR open for maintainer review, or replace it with a narrower implementation. The best path is to add `agents.defaults.compaction.modelFallbacks` consistently across exported types, strict schema, docs/config reference, direct compaction, queued/context-engine compaction, and focused tests covering unset behavior, model resolution failure, auth/runtime failure, all-fallbacks-fail diagnostics, auth-profile handling, runtime-auth exchange, existing thinking-level retry, checkpoint persistence, and prompt/transport parity.
