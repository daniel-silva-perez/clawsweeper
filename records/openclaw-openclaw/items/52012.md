## Summary

Keep this PR open. Current main still does not implement `agents.defaults.compaction.modelFallbacks` or an equivalent compaction-specific fallback path, and the linked feature issue was closed as superseded by this still-open PR. The PR is not obsolete or implemented on main; it needs maintainer review, changes, or a narrower replacement.

## Best Possible Solution

Keep the PR open but require changes or replace it with a smaller maintainer-approved PR. The implementation should add `agents.defaults.compaction.modelFallbacks` consistently to public types, strict schema, generated/help metadata, docs, and both direct and queued compaction behavior; preserve existing prompt, checkpoint, stream, auth-profile, and thinking-level fallback behavior; and add focused tests for unset behavior, resolution fallback, auth fallback, runtime compaction failure fallback, and all-fallbacks-fail diagnostics.
