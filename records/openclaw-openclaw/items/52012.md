## Summary

Keep this PR open. Current main still lacks the requested `agents.defaults.compaction.modelFallbacks` config surface and has no equivalent compaction-specific fallback loop; normal agent model fallback support exists, but the inspected compaction paths resolve one compaction target and fail or retry only thinking level. The PR is still an implementation candidate, though its current diff is broad and needs maintainer review around compaction auth, runtime-auth, transport wrapping, checkpoints, and session lifecycle.

## Best Possible Solution

Keep the PR open for maintainer review or replace it with a narrower implementation. The best path is to add `agents.defaults.compaction.modelFallbacks` consistently across exported types, strict schema, generated/help metadata, docs, direct compaction, queued/context-engine compaction, and focused tests for unset behavior, resolution/auth/runtime fallback, all-fallbacks-fail diagnostics, and non-regression of existing thinking-level fallback, checkpoints, prompt construction, auth-profile handling, and runtime-auth behavior.
