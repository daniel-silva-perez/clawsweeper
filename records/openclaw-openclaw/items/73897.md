## Summary

Keep this PR open. Current main still routes every non-Windows node shell wrapper through `/bin/sh -lc`, so Android/Termux node hosts that reach `system.run` still lack the requested shell path. The PR is a narrow active implementation candidate: it changes only the node shell helper, its focused unit test, and two docs pages; the security pass found no dependency, workflow, lockfile, secret, install-script, generated/vendor, or release-path changes.

## Best Possible Solution

Keep this PR open and review it as the active implementation candidate. The best path is to land this narrow helper/test/docs change, or an equivalent patch, after maintainer review: return the Termux shell path only for normalized `android` node-host platform metadata, preserve Windows and other POSIX behavior, keep docs scoped to Android/Termux headless node-host behavior, and validate with the focused unit test plus the normal changed gate.
