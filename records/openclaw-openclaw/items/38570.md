## Clownfish Work Prompt

Fix the unresolved sessions_send A2A requester attribution bug tracked by #38570. Current main includes requester session/channel in agent-to-agent message, reply, and announce prompts, but omits the configured requester `identity.name`, so the target agent can misidentify the sender. Use #38579 as prior context only; do not apply it wholesale because it also touched CI workflow files.

Implementation boundary:
- In `src/agents/tools/sessions-send-tool.ts`, derive the requester agent id from the effective requester session key used by the tool, resolve `resolveAgentIdentity(cfg, requesterAgentId)?.name`, trim it, and pass it as optional requesterName for A2A prompts.
- In `src/agents/tools/sessions-send-tool.a2a.ts`, carry optional requesterName through `runSessionsSendA2AFlow` into reply and announce context builders.
- In `src/agents/tools/sessions-send-helpers.ts`, add optional requesterName support to the shared session-line helper and emit `Agent 1 (requester) name: <name>.` when present.
- Preserve current prompt output when no configured identity name exists. Avoid injecting `main` for malformed or non-agent requester keys unless the requester key is already the canonical effective main-agent key by existing contract.
- Add focused regression tests in `src/agents/openclaw-tools.sessions.test.ts` for initial send, ping-pong reply, announce context, and no-name fallback.
- Add a single-line `CHANGELOG.md` Fixes entry if the project still expects user-facing bug fixes there; credit `Thanks @jamclaws` and, if reusing the prior approach, `@ayanesakura`. Do not credit Codex/OpenClaw.
- Do not change `.github/workflows/*`, package metadata, lockfiles, release files, or unrelated session routing behavior.
