## Summary

Current main still omits the requester's configured `identity.name` from `sessions_send` agent-to-agent message, reply, and announce prompt context. The related fix PR #38579 is closed unmerged, and the later UI-display issue #60911 was closed as a duplicate of this issue, so this remains the canonical open bug and is a narrow fix-PR candidate.

## Best Possible Solution

Implement the narrow sessions_send fix: derive the requester agent id from the canonical effective requester session key, resolve the configured `identity.name`, pass an optional requester name through the message, reply, and announce context builders, and emit `Agent 1 (requester) name: <name>.` only when a configured name is safely available. Add focused regression tests for initial send, reply, announce, and no-name fallback; treat #38579 as prior context, not a patch to apply wholesale.

## Clownfish Work Prompt

Fix the unresolved sessions_send A2A requester attribution bug tracked by this issue. Current main includes requester session/channel in agent-to-agent message, reply, and announce prompts, but omits the configured requester `identity.name`, so the target agent can misidentify the sender. Use #38579 as prior context only; do not apply it wholesale because it also touched CI workflow files.

Implementation boundary:
- In `src/agents/tools/sessions-send-tool.ts`, derive the requester agent id from the canonical effective requester session key used by the tool, resolve `resolveAgentIdentity(cfg, requesterAgentId)?.name`, trim it, and pass it as optional requesterName for A2A prompts.
- Avoid injecting the main/default agent name for malformed, alias, or non-agent requester keys unless the requester key is already known to be the canonical effective main-agent key by existing session-tool context contracts.
- In `src/agents/tools/sessions-send-tool.a2a.ts`, carry optional requesterName through `runSessionsSendA2AFlow` into reply and announce context builders.
- In `src/agents/tools/sessions-send-helpers.ts`, add optional requesterName support to the shared session-line helper and emit `Agent 1 (requester) name: <name>.` when present.
- Preserve current prompt output when no configured identity name exists.
- Add focused regression tests in `src/agents/openclaw-tools.sessions.test.ts` for initial send, ping-pong reply, announce context, and no-name fallback.
- Add a single-line `CHANGELOG.md` Fixes entry if user-facing bug fixes still require one; credit `Thanks @jamclaws` and, if reusing the prior approach, `@ayanesakura`. Do not credit Codex/OpenClaw.
- Do not change `.github/workflows/*`, package metadata, lockfiles, release files, or unrelated session routing behavior.
