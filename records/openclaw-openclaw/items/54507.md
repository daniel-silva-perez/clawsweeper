## Summary

Current main still does not implement a Feishu/Lark option to hide or customize the runtime footer. The footer note is built and supplied to streaming and static card replies unconditionally, while the Feishu schema, plugin manifest, docs, and tests expose no footer-specific setting. The prior ClawSweeper comment reached the same result, and the current checkout still matches that open feature gap.

## Best Possible Solution

Keep this issue open. Implement the setting in the Feishu plugin, likely as a top-level and per-account option such as `channels.feishu.showRuntimeFooter` or a small `replyFooter` mode/template. Thread it through `resolveCardNote`, streaming start/close, and static card delivery, then update the Feishu zod schema, plugin manifest metadata, public Feishu docs, and reply-dispatcher tests. Consider a shared channel presentation contract only if maintainers want the same behavior across multiple channels.
