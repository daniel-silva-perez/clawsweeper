## Summary

Current main still does not implement a Feishu/Lark option to hide or customize the runtime footer. The footer is constructed and passed into streaming and non-streaming card replies unconditionally, while the Feishu config schema, plugin manifest, docs, and tests expose/assert no footer-specific control.

## Best Possible Solution

Keep this issue open. Implement the setting in the Feishu plugin, likely as top-level and per-account `channels.feishu.showRuntimeFooter` or a small `replyFooter` mode/template. Thread it through `resolveCardNote`, streaming start/close, and non-streaming card delivery; update the Feishu zod schema, plugin manifest metadata, public Feishu docs, and reply-dispatcher tests. Consider a shared presentation contract only if maintainers want the same behavior across multiple channels.
