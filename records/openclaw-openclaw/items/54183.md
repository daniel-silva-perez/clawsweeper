## Summary

Keep this PR open for maintainer review. Current main has shipped a broader write-ahead delivery queue and WhatsApp reconnect drain, but it still does not implement the configurable `channels.whatsapp.retry` surface proposed here, and the latest maintainer discussion explicitly treats configurable send retry as adjacent rather than identical to the queue work. The PR also needs a fresh review/rebase against the current WhatsApp outbound architecture before it can land.

## What This Changes

The PR adds configurable WhatsApp outbound send retry with exponential backoff for text, media, and poll sends, plus WhatsApp config schema/generated docs updates and retry tests.

## Best Possible Solution

Decide whether OpenClaw still wants a user-facing WhatsApp immediate retry knob after the shipped durable queue work; if yes, rebase this change onto the current plugin-local outbound path, align it with the queue/reconnect-drain semantics, and document exactly which transient errors are retried versus deferred to the queue.
