## Summary

Keep this issue open. Current main still preserves the reported same-agent cross-root absolute `sessionFile` path, and the provided context shows an open same-author implementation PR with closing syntax that should be reviewed rather than closing this issue early.

## What This Changes

The issue asks OpenClaw to remap stale same-agent absolute `sessionFile` metadata from an old state root into the current agent sessions directory before transcript writes.

## Best Possible Solution

Land PR #61322 or an equivalent narrow resolver change that remaps only same-agent `agents/<agent>/sessions/<file>` old-root paths into the current sessions directory while preserving existing cross-agent and unsafe-path fallback behavior, with focused regression coverage in the session path tests.

## Reproduction Assessment

Yes. The issue provides concrete metadata and CLI steps, and the current source path statically reproduces the failure mode: the resolver preserves the stale absolute path and write-lock setup then creates that path's directory.

## Solution Assessment

Yes, the paired PR direction is the narrowest maintainable fix: handle the compatibility case in the session path resolver before persistence/write-lock code sees the path. A broader startup migration is less targeted and would not cover every runtime resolver entry point.
