## Summary

Keep this issue open. Current main still preserves structurally valid cross-root absolute sessionFile paths instead of remapping same-agent paths into the current state dir, and the provided GitHub context shows an open same-author PR with closing syntax that is the implementation candidate.

## Best Possible Solution

Keep this issue open while PR #61322 is reviewed. The best fix belongs in the session path resolver: narrowly remap stale same-agent agents/<agent>/sessions/<file>.jsonl absolute paths into the current state dir before write-lock/transcript writes, with a focused regression test that preserves existing cross-agent and unsafe-path behavior.
