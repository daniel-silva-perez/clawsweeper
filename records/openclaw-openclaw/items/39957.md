## Summary

Keep this PR open. Current main still lacks the PR's narrow suppression for `message_end` block replies on assistant `stopReason === "toolUse"` turns with structured tool-call blocks, while current provider conversion can still produce text plus `toolCall` content with `stopReason: "toolUse"`. The PR is a focused implementation candidate, not obsolete cleanup.

## Best Possible Solution

Keep this PR open and land this narrow suppression, or an equivalent current-main patch, in `handleMessageEnd`: suppress only `message_end` block replies for assistant turns with `stopReason === "toolUse"` and a structured tool-call block, while preserving normal `stop` turns, `toolUse` turns without tool-call blocks, and `text_end` block-reply behavior. Include the PR's regression cases, then run the targeted agent/Telegram tests and the appropriate changed gate before merge.
