## Summary

Keep this PR open. Current main still has the reported `message_end` block-reply path without an equivalent `stopReason === "toolUse"` plus structured tool-call suppression, while the provider conversion path can still produce unphased assistant text alongside `toolCall` blocks with `stopReason: "toolUse"`.

## Best Possible Solution

Keep the PR open and land this narrow suppression, or an equivalent current-main patch, in `handleMessageEnd`: suppress only `message_end` block replies for assistant turns with `stopReason === "toolUse"` and a structured tool-call block, while preserving normal `stop` turns, `toolUse` turns without tool-call blocks, and `text_end` block-reply behavior. Include the regression cases from the PR and rebase/squash away unrelated branch history before merge.
