## Summary

Keep this PR open. Current main still lacks the narrow `message_end` block-reply suppression for assistant `stopReason === "toolUse"` turns with structured tool-call blocks, while current provider conversion can still create that message shape and the existing tests do not cover the PR's regression case.

## Best Possible Solution

Keep this PR open and land this narrow suppression, or an equivalent current-main patch, in `handleMessageEnd`: suppress only `message_end` block replies for assistant turns with `stopReason === "toolUse"` and a structured tool-call block, while preserving normal `stop` turns, `toolUse` turns without tool-call blocks, and `text_end` block-reply behavior. Include the PR's regression cases, rebase or squash away unrelated branch history, then run the targeted agent/Telegram tests plus the appropriate changed gate before merge.
