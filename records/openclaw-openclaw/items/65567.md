## Summary

Keep this PR open. Current main and the latest release still have the unguarded `hashText(event.text)` and `emitSystemStatus(event.text)` paths, while this PR contains the narrow guards but still needs maintainer review and a focused regression test before landing.

## What This Changes

The PR changes `src/auto-reply/reply/acp-projector.ts` so `hashText` accepts `undefined` and the `usage_update` path passes `event.text ?? ""` into `emitSystemStatus`.

## Best Possible Solution

Keep this PR open as the active fix candidate for #59066. The best path is to land this branch or a superseding patch that guards both trim sites, adds a focused `acp-projector` regression test for a visible textless `usage_update`, and deliberately resolves the ACP status text contract either at the runtime type boundary or by normalizing before projection.
