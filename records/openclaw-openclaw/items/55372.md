## Summary

Current main still lacks a restart-sentinel outbound-notification opt-out. The startup path still removes the sentinel, wakes the session, and calls outbound notice delivery whenever routing resolves; the config schema/docs expose internal hook configuration but no restart-sentinel-specific notify/disable flag. This remains a concrete core gateway/config request, not a cleanup close.

## Best Possible Solution

Keep this issue open. The best implementation is a first-class gateway restart-sentinel config flag that gates only deliverRestartSentinelNotice after routing resolution, with schema/help/docs coverage and focused tests in src/gateway/server-restart-sentinel.test.ts proving wake and continuation still run when outbound notification is disabled.
