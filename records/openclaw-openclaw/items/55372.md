## Summary

Current main still does not implement a restart-sentinel notification opt-out. Startup consumes the sentinel, enqueues the wake, and still attempts outbound notice delivery whenever routing resolves. The config schema/docs expose generic internal-hook entries but no restart-sentinel-specific enabled or notify flag, and the related merged PR #53940 explicitly left config changes out of scope.

## Best Possible Solution

Keep this issue open. The narrow implementation should add a core gateway config flag near restart-sentinel startup handling that gates deliverRestartSentinelNotice only, with focused coverage in src/gateway/server-restart-sentinel.test.ts proving wake/continuation still run when notifications are disabled.
