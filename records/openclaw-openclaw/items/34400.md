## Summary

Keep open. Current main still only partially supports recursive memory search: recursive file collection exists, but nested dated memory files still miss temporal decay, dirty built-in searches still sync in the background, QMD watchers still pass glob-shaped paths to chokidar 5, and several user-facing strings still advertise flat `memory/*.md` files. The reporter also has open paired PR #72748 targeting these same gaps.

## Best Possible Solution

Keep this issue open and resolve it by reviewing and landing #72748, or an equivalent focused memory-core PR, that fixes nested dated-path temporal decay, first-search freshness after dirty sync, QMD watcher root/pattern handling under chokidar 5, and the remaining `memory/**/*.md` user-facing copy. The fix should include targeted tests for temporal decay, built-in async search sync, watcher path construction/filtering, and QMD nested-file behavior.
