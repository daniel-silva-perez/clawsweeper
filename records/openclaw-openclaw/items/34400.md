## Summary

Keep open. Current main partially supports recursive memory collection, but the issue’s runtime gaps still remain: nested dated memory paths are not recognized for temporal decay, dirty search sync is still fire-and-forget for the built-in manager, QMD watcher targets still include glob-shaped paths while the repo depends on chokidar 5, and several user-facing strings still advertise `memory/*.md`. There is also an open same-author follow-up PR, #72748, that directly targets these gaps, so this issue should stay open until that PR or an equivalent memory-core fix lands.

## Best Possible Solution

Keep this issue open and resolve it through review of #72748 or an equivalent focused memory-core PR. The fix should cover nested dated-path temporal decay, first-search freshness after dirty sync, QMD watcher root/pattern handling compatible with chokidar 5, and user-facing `memory/**/*.md` copy, with targeted memory-core tests and normal changed-surface validation before merge.
