## Summary

Current main still only partially supports recursive memory search. Recursive file collection exists, but nested dated memory paths still miss temporal decay, built-in dirty search sync is still fire-and-forget, QMD watcher paths still combine collection roots with glob patterns under chokidar 5, and user-facing copy still advertises flat memory files. The reporter also has open PR #72748 targeting these same remaining gaps, so the issue should stay open.

## Best Possible Solution

Keep this issue open and resolve it by reviewing and landing #72748, or an equivalent focused memory-core PR, that fixes nested dated-path temporal decay, first-search freshness after dirty sync, QMD watcher root/pattern handling compatible with chokidar 5, and the remaining `memory/**/*.md` user-facing copy. The fix should include targeted tests for temporal decay, built-in async search sync, watcher path construction/filtering, and QMD nested-file behavior.
