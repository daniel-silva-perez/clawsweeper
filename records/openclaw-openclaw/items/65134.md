## Summary

Keep open. Current main and the latest release still use the expensive embedded-runner slug path, so the PR addresses a real remaining problem, but the branch is not merge-ready against current main because it drops current timeout behavior, bypasses runtime auth used by the existing simple-completion helper, and dereferences a nullable model parse result.

## What This Changes

The branch replaces session-memory slug generation with `completeSimple`, prefers `agents.defaults.heartbeat.model`, and adds error-level timeout logging in model fallback and embedded-runner failover observations.

## Best Possible Solution

Land a focused slug-generator update that removes `runEmbeddedPiAgent` and temp session files, uses `completeSimple` with the current runtime-auth and model-resolution pattern, preserves `llmSlug: false`, timestamp fallback, and configured timeout behavior, and adds targeted regression coverage. Keep fallback logging as a separate, explicitly reviewed change if maintainers want it.

## Reproduction Assessment

Yes. Static inspection of current main and `v2026.4.29` reproduces the old embedded-runner slug path, and the PR defects are traceable through current source/type contracts; I did not run tests because this review was constrained to read-only inspection.

## Solution Assessment

No. The direction is appropriate, but the PR should be narrowed and rebased to current main with null-safe heartbeat model parsing, runtime-ready auth, preserved timeout semantics, and tests for no temp session/no embedded runner/heartbeat fallback behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Guard the heartbeat model parse before dereferencing:** `src/hooks/llm-slug-generator.ts:50`
  - body: `parseModelRef()` returns `null` for blank or malformed refs on current main, but this branch immediately reads `parsed.provider` and `parsed.model`. That will fail strict typechecking and can throw instead of falling back to the primary model for bad heartbeat config.
  - confidence: 0.93
- **[P2] Resolve runtime auth before calling completeSimple:** `src/hooks/llm-slug-generator.ts:83`
  - body: The existing simple-completion path uses `getRuntimeAuthForModel()` so provider plugins can exchange credentials before the request. Calling raw `getApiKeyForModel()` skips hooks such as GitHub Copilot token exchange and Microsoft Foundry Entra auth, so slug generation can fail for provider-backed models that otherwise work.
  - confidence: 0.88
- **[P2] Preserve the configured slug timeout:** `src/hooks/llm-slug-generator.ts:95`
  - body: Current main honors `agents.defaults.timeoutSeconds` for slug generation and has coverage for slow local providers. This branch hardcodes a 15-second AbortController timeout, so merging it would regress that current behavior.
  - confidence: 0.86
