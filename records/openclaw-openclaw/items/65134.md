## Summary

Keep open. Current main and the latest release still use the embedded-runner slug path, so the PR addresses a real remaining performance/congestion problem; however, the submitted branch is not ready as-is because it mixes the slug change with unrelated timeout-failover logging and lacks focused regression coverage.

## What This Changes

The PR changes session-memory LLM slug generation to call `completeSimple` with `agents.defaults.heartbeat.model` preferred over the agent primary model, and also adds louder timeout/failover logging in agent observation code.

## Best Possible Solution

Land a focused slug-generator fix that removes `runEmbeddedPiAgent` and `session:temp:slug-generator` from session-memory slug creation, uses the current simple-completion/auth/model-resolution pattern, preserves `llmSlug: false` and timestamp fallback behavior, and adds targeted regression coverage. Split the timeout-failover logging into a separate PR if maintainers still want it.
