## Summary

Keep open. Current main and the latest release still use the embedded-agent slug path, so the underlying performance issue is real, but this PR is not merge-ready because the new heartbeat-model parser dereferences a nullable `parseModelRef` result and the branch mixes the slug fix with separate loud timeout logging changes.

## What This Changes

The PR changes session-memory LLM slug generation from a temporary `runEmbeddedPiAgent` session to `completeSimple` with heartbeat-model preference, and also adds louder timeout logs to model fallback and embedded-runner failover observation.

## Best Possible Solution

Land a focused slug-generator fix that removes the temp `runEmbeddedPiAgent` session, uses the current simple-completion/model-auth pattern, guards malformed heartbeat model refs or falls back cleanly to the primary model, preserves `llmSlug: false` and timestamp fallback behavior, and adds narrow slug-helper/session-memory tests. Split the loud timeout logging into its own review if it is still desired.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Guard nullable heartbeat model refs:** `src/hooks/llm-slug-generator.ts:49-50`
  - body: `parseModelRef` returns `ModelRef | null`, but this new code dereferences `parsed.provider` and `parsed.model` unconditionally. Strict TypeScript should reject this, and a malformed heartbeat model string would throw before the intended primary-model fallback, so guard the parse result and fall back or fail cleanly.
  - confidence: 0.93
