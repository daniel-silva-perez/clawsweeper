## Summary

Keep open. Current main still lacks a centralized gateway tool-call audit sink, and this PR is a plausible implementation for the gateway-level audit request. It should not be auto-closed, but it needs maintainer/security review because the proposed audit path still misses denied tool-call attempts and the branch carries unrelated test/guardrail drift.

## What This Changes

The PR adds a gateway tool-call audit JSONL helper, threads audit context through direct `/tools/invoke` and OpenResponses-originated agent runs, and updates focused gateway/agent tests plus several unrelated test guardrail files.

## Best Possible Solution

Land a clean gateway-owned audit implementation that records both allowed and denied gateway-originated tool attempts with redacted arguments, stable correlation metadata, focused tests for `/tools/invoke` and OpenResponses, and no unrelated shared-CI cleanup in the feature branch.
