## Summary

Keep this PR open. Current main still lacks the proposed gateway tool-call audit trail: there is no `gateway-tool-audit.jsonl` helper, direct `/tools/invoke` does not append audit records, OpenResponses ingress does not pass audit context into agent runs, and existing diagnostics only emit summarized telemetry rather than a durable redacted-args JSONL audit record. The PR is not implemented on main or superseded by the related ACP audit PR, but it needs maintainer review, rebase, and scope cleanup before merge.

## Best Possible Solution

Keep the PR open, rebase it onto current main, remove unrelated test/CI drift, and review the gateway audit design directly. The best mergeable shape is a narrow gateway audit patch with durable JSONL writes, redacted effective args, clear permission behavior, non-disruptive write performance, and focused tests for `/tools/invoke` and OpenResponses ingress.
