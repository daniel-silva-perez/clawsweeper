## Summary

Keep open. Current main still lacks the requested gateway tool-call audit sink, and the PR is useful but not merge-ready because its proposed audit writes only cover allowed tool calls; denied gateway-originated attempts still return before an audit row is written.

## What This Changes

The PR adds a gateway tool-call audit JSONL helper, threads audit context through direct `/tools/invoke` and OpenResponses embedded runs, and updates focused gateway tests plus unrelated shared CI guardrail tests.

## Best Possible Solution

Land a gateway-owned audit implementation that records both allowed and denied gateway-originated tool attempts across direct `/tools/invoke` and OpenResponses, includes redacted arguments plus stable correlation/status metadata, and keeps unrelated shared-CI cleanup out of the feature branch unless it is still required by the rebase.

## Reproduction Assessment

Yes. Force `runBeforeToolCallHook` to return a blocked outcome in either `/tools/invoke` or an OpenResponses-originated embedded tool call; in the PR diff, both audit append sites are after the blocked-return path, so no audit row is produced.

## Solution Assessment

No. The proposed allowed-call audit sink is a reasonable base, but the narrow maintainable fix must also audit denied attempts with a status or reason before returning from the blocked branches.
