## Summary

Keep this issue open. Current main still handles Telegram `message_reaction` updates by authorizing/filtering them and enqueuing human-readable system-event text, but it does not expose a stable structured internal hook or typed plugin hook for downstream reaction feedback. The prior ClawSweeper comment matches current main at 17811480dae20f65b526454d75e971ae8424ec10.

## Best Possible Solution

Keep this issue open. Add an additive typed reaction seam after Telegram's existing authorization and `reactionNotifications` filtering path, preferably as a cross-channel plugin hook such as `message_reaction` and/or an internal managed-hook event such as `message:reaction`. The payload should include channel/account context, session or conversation context when available, Telegram chat id, target message id, emoji, add/remove operation, sender metadata, authorization/filtering outcome, and bounded raw metadata. Add Telegram handler tests, hook contract tests, and docs, while leaving feedback recording itself to a hook/plugin built on the seam rather than hardcoding feedback storage into the Telegram channel.
