## Summary

Keep open. Current main distinguishes Gateway client identities at connection time, including browser Control UI/WebChat and native macOS, but that metadata still does not reach the agent runtime line as trusted client fields. The remaining work is a narrow core Gateway-to-agent runtime context path, not stale cleanup.

## Best Possible Solution

Implement a narrow trusted Gateway client-context path in core: sanitize client metadata from client.connect.client, thread it through chat.send and queued/follow-up run params into RuntimeInfoInput, render deterministic fields such as client=openclaw-macos, client_mode=ui, and client_platform=macos in buildRuntimeLine, and add prompt/runtime tests plus WebChat/Control UI docs for the privacy-safe contract.
