## Summary

Keep open. Current main already distinguishes Gateway client identities at connection time, including browser Control UI/WebChat and native macOS, but that metadata still does not reach agent runtime info as trusted client fields. The WebChat chat.send path maps the connected client into sender metadata and gateway scopes; RuntimeInfoInput/buildRuntimeLine still expose channel/capability metadata only.

## Best Possible Solution

Implement a narrow trusted Gateway client-context path in core: define sanitized client metadata, populate it from client.connect.client in chat.send, propagate it through auto-reply/followup run params into RuntimeInfoInput, render fields such as client=openclaw-macos, client_mode=ui, and client_platform=macos in buildRuntimeLine, then add Gateway chat and system-prompt tests plus WebChat/Control UI docs for the privacy-safe fields.

## Risks / Open Questions

- Raw userAgent, modelIdentifier, and instanceId can be privacy-sensitive; a fix should expose only sanitized trusted fields such as client id, mode, platform, and device family.
- Runtime-line changes affect prompt bytes and cache stability, so added fields should be deterministic and covered by prompt tests.
