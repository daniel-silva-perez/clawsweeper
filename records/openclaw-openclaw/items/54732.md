## Summary

Keep open. Current main still scopes the `message` tool action enum and plugin-owned schema contributions, but the shared core schema groups are still spread into every generated message tool schema regardless of the resolved enabled action set.

## Best Possible Solution

Implement action-aware shared schema trimming in `src/agents/tools/message-tool.ts`: pass the resolved action set into the shared property builder, include poll/sticker/thread/event/moderation/channel-management/presence fields only when active actions require them, decide whether gateway override fields stay global, and update focused schema tests so basic send/react-only configurations omit unsupported shared groups while advanced actions still expose required parameters.
