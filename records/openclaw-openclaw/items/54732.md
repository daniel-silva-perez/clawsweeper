## Summary

Keep open. Current main still scopes the `message` tool action enum and plugin-owned schema contributions, but the shared core schema groups remain unconditional, so the requested enabled-channel/action-aware trimming is not implemented.

## Best Possible Solution

Implement action-aware shared schema trimming in `src/agents/tools/message-tool.ts`: pass the resolved action set into the shared property builder, include poll/sticker/thread/event/moderation/channel-management/presence fields only when active actions require them, decide whether gateway override fields remain global, and update focused message-tool schema tests so basic send/react-only configurations omit unsupported shared groups while advanced actions still expose their required parameters.

## Risks / Open Questions

- Gateway override fields (`gatewayUrl`, `gatewayToken`, `timeoutMs`) are shared execution-path parameters read by the message tool gateway helper, so trimming them needs explicit contract judgment rather than a blanket action mapping.
- A fix should use the resolved cross-channel action set, not only the current channel, so configured plugins keep the shared parameters their exposed actions require.
