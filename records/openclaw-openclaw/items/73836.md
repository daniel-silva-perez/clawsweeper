## Summary

Keep open. Current main addresses the Control UI refresh-disable and heartbeat-display slices, and has some model/tool cache mitigations for control-plane latency, but outbound delivery mirroring still stores media as text-only filename content and the Docker/WSL2 reconnect plus Telegram typing-gap symptoms are not proven fixed.

## What This Changes

The issue asks OpenClaw to stay responsive during active Docker/WSL2 agent runs, keep Telegram typing continuous, hide heartbeat noise from visible chat, and mirror outbound media in Control UI as previews instead of filenames.

## Best Possible Solution

Keep this as an open umbrella until maintainers split or finish the remaining work: preserve the landed refresh guard and heartbeat filtering, fix delivery mirroring to write previewable structured/managed media, and continue Docker/WSL2 gateway plus Telegram responsiveness profiling through #73428 or a narrower linked follow-up.

## Solution Assessment

No for closure. The current main implementation is a good narrow fix for refresh and heartbeat display, but it is not the complete solution because delivery mirror media is still text-only and the broader responsiveness symptoms remain unproven.
