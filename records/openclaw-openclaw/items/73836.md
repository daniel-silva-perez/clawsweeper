## Summary

Keep this open. Current main fixed the narrow Control UI refresh-button footgun and has heartbeat display filtering, but the delivery mirror still persists outbound media as filename text, and the broader Docker/WSL2 responsiveness plus Telegram typing-gap symptoms are not proven fixed.

## What This Changes

The issue reports a v2026.4.26 regression combining Control UI reconnect stalls during active runs, Telegram typing gaps, visible heartbeat transcript noise, and outbound media mirrored in Control UI as filenames instead of previews.

## Best Possible Solution

Keep the issue open, treat #73428 as the canonical live Docker/WebChat latency tracker for the gateway stall portion, keep the merged refresh guard, and split a focused follow-up for delivery-mirror media so successful outbound images are persisted as managed/structured attachment content that Control UI can preview. Telegram typing should be validated with a long active run after the latency path is profiled, because an event-loop stall can mask otherwise-correct keepalive logic.

## Reproduction Assessment

Partially. The issue gives a concrete Docker/Windows+WSL2 reproduction path, and the media-mirror symptom is source/test-backed on current main; the broader reconnect stall and Telegram typing gap still need live Docker/Telegram verification against current main or the next package.

## Solution Assessment

No, closing this issue now is not the best solution. The merged refresh guard is a good narrow fix for #65522, but the remaining media mirror and active-run responsiveness symptoms need separate targeted work and profiling rather than being swept as implemented.
