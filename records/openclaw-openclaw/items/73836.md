## Summary

Keep this issue open. Current main still has source-backed evidence for the media mirror symptom: outbound delivery mirroring collapses media URLs to filename text before appending a delivery-mirror transcript entry, so Control UI cannot preview the delivered media from that mirror. The active-run refresh guard is also still narrower than the linked open PR proposes, and the broader Docker/WSL2 gateway stall plus Telegram typing-gap symptoms are not proven fixed. The heartbeat transcript marker is now filtered in chat history projection, but that partial coverage is not enough to close this bundled regression.

## Best Possible Solution

Keep this open and split follow-up deliberately: review or supersede #66395 for the active-run refresh guard, continue profiling the Docker/WSL2 gateway stall alongside #73428/#72365, and land a focused delivery-mirror fix that persists successful outbound media as structured attachment/image content Control UI can preview. Keep the heartbeat projection filter, but do not treat it as a complete fix for this combined regression.
