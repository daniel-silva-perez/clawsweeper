## Summary

Keep open. Current main has fixes for the Control UI refresh-button slice and heartbeat display filtering, but the issue is broader: outbound media delivery mirrors still collapse media to filename-only assistant transcript text, and the Docker/WSL2 reconnect plus Telegram typing-gap symptoms are not proven fixed.

## What This Changes

The issue reports a v2026.4.26 regression where active agent runs can stall Control UI reconnects, interrupt Telegram typing presence, leak heartbeat noise into visible chat history, and mirror generated outbound media as filenames instead of previews.

## Best Possible Solution

Keep the already-landed refresh guard and heartbeat filtering, then split the remaining work into targeted tracks: make delivery mirrors persist previewable structured/managed media for Control UI, continue Docker/WSL2 gateway responsiveness profiling through #73428 or a narrower follow-up, and verify Telegram typing keepalive behavior during long active runs.

## Reproduction Assessment

Partially. The media mirror symptom has a high-confidence source and unit-test reproduction path on current main; the Docker/WSL2 reconnect stall and Telegram typing gap still need live reproduction or trace data from a comparable setup.

## Solution Assessment

No for closure. Current main is a good narrow solution for refresh and heartbeat-display noise, but it is not the best complete solution because filename-only media mirroring remains and the broader responsiveness symptoms are not proven resolved.
