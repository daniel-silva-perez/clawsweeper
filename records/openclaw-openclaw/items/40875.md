## Summary

Keep this PR open. Current main still lacks heartbeat-specific fallback guidance, but the PR is not ready to merge as-is because its wording overstates cross-provider failure behavior, its patch targets the old field-note layout, and its changelog entry misses required attribution.

## Best Possible Solution

Request a narrow revision: rebase onto current main, put the fallback note inside the current heartbeat `model` field block, align the wording with the shared model failover docs, state that no override and same-provider overrides use the applicable fallback chain, state that unrelated different-provider configured fallbacks are skipped unless the override is already in the configured chain, and mention that the configured primary can still be appended when no explicit fallback override is supplied. Add the required changelog attribution. Keep #23718 as the separate broader request for structured heartbeat-specific fallback configuration.
