## Summary

Keep this PR open for maintainer follow-up. Current main still lacks heartbeat-specific fallback guidance, so the docs gap is real, but the submitted docs text is stale against the current ParamField layout and overstates cross-provider behavior by saying the heartbeat errors without trying other models.

## Best Possible Solution

Request a narrow revision rather than closing: add the fallback note inside the current `heartbeat.model` `ParamField`, align it with `docs/concepts/model-failover.md`, state that no override and same-provider overrides follow the standard fallback chain, state that different-provider overrides skip unrelated configured fallbacks unless the override is already in the configured fallback chain, and mention that the configured primary can still be tried when no explicit fallback override is supplied. Fix or remove the changelog entry. Keep #23718 separate as the broader feature request for structured heartbeat fallback configuration.
