## Summary

Keep this PR open. Current main still has the heartbeat docs gap, but this PR is not ready to close or merge as-is: its docs wording overstates the cross-provider failure behavior, its patch targets the old list-style field layout instead of the current ParamField block, and its changelog entry misses the required attribution.

## Best Possible Solution

Request a narrow revision: rebase onto current main, put the fallback note inside the current heartbeat `model` ParamField, align the wording with `docs/concepts/model-failover.md`, state that no override and same-provider overrides use the applicable fallback chain, state that unrelated different-provider fallbacks are skipped unless the override is already in the configured chain, and mention that the configured primary can still be appended when no explicit fallback override is supplied. Keep #23718 as the separate broader feature request for structured heartbeat-specific fallback configuration.
