## Summary

Keep this PR open. Current main still lacks heartbeat-specific fallback guidance, so the intended docs fix is not implemented on main; however the PR is not ready to close or merge as-is because its wording omits the configured-primary retry case, its patch targets the old list-style heartbeat field layout, and its changelog entry misses required attribution.

## Best Possible Solution

Request a narrow revision: rebase onto current main, place the note inside the current heartbeat `model` ParamField, align the wording with `docs/concepts/model-failover.md`, state that no override and same-provider overrides use the applicable fallback chain, state that unrelated different-provider configured fallbacks are skipped unless the override is already in the configured chain, mention that the configured primary can still be appended when no explicit fallback override is supplied, and add the required changelog attribution. Keep #23718 separate because it was the broader structured heartbeat fallback configuration request, not this docs-only correction.
