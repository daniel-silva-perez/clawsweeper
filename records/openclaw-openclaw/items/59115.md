## Summary

Keep this PR open. Current main still does not implement the Slack forwarded/shared attachment text path for thread history: `resolveSlackThreadHistory` only retains messages with `text` or `files`, does not carry `attachments`, and therefore still drops forwarded-only Slack thread messages. The PR is a plausible fix, but it needs a rebase onto the current `thread.ts` layout and should address the remaining capped-rendering predicate review note before maintainer merge review.

## Best Possible Solution

Keep the PR open as useful unfinished work. The best path is to rebase it onto current main, implement the forwarded attachment text/fallback merge in `extensions/slack/src/monitor/thread.ts` (or a small shared helper that avoids a media/thread cycle), add the thread-history regression tests in the current test layout, preserve existing file placeholder behavior, and handle the capped-rendering predicate edge case before running the targeted Slack plugin tests and changed gate.
