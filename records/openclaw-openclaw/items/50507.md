## Summary

Keep this PR open. Current main still does not implement native outbound WhatsApp location messages through the message tool, and the PR remains a plausible implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep the PR in normal review and ask for a focused revision: add a supported generic location-send contract or plugin action seam, expose location in WhatsApp message-tool discovery, wire production WhatsApp outbound through the current outbound-base/channel-outbound path and Baileys send API, add a gateway location RPC with validation and idempotency, cover message-tool/core/gateway/WhatsApp tests, and avoid logging raw coordinates.
