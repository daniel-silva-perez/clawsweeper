## Summary

Current main at 891c7d9f1c1f5e0f52935fadfeacf695b0ef1eb8 still does not implement native outbound WhatsApp location pins through the shared message tool. This PR remains a plausible implementation candidate for the closed #2214 request and the earlier closed, unmerged #2298 attempt, but it needs maintainer review and revision before merge.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is a focused revision that adds a supported location-send contract/discovery path, wires the production WhatsApp outbound-base and ActiveWebListener/Baileys send path, adds a gateway location RPC with validation and idempotency, covers message-tool/core/gateway/WhatsApp tests, and avoids logging raw coordinates.
