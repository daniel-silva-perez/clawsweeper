## Summary

Keep open. The docs-only change is directionally useful, but two recovery steps do not match current gateway behavior: approving a node-pair request updates durable pairing state, not an already-connected node session, and rejecting a request does not issue a replacement token.

## What This Changes

The PR updates docs/gateway/pairing.md with a Troubleshooting section for dropped commands, missing pending requests, and stale tokens, while normalizing a few smart quotes.

## Best Possible Solution

Merge a docs-only update after the troubleshooting section says command upgrades require reconnecting or restarting the node host after approval, and after stale-token guidance points operators to create a new request or remove and re-pair instead of reject/re-approve.

## Reproduction Assessment

Yes. The docs gap is visible on current main, and the proposed wording defects are reproducible by tracing connect-time reconciliation, approval persistence, node status cataloging, and existing gateway tests.

## Solution Assessment

No. Adding troubleshooting next to the pairing docs is the right location, but the current PR wording is not the narrowest maintainable fix until it matches live session and token-rotation behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Document reconnect before retrying approved commands:** `docs/gateway/pairing.md:209-211`
  - body: The new troubleshooting step tells operators to retry after approval, but approval only updates the pairing store. A connected node keeps the command list assigned at connect time, so upgraded commands remain unavailable until the node host reconnects or restarts.
  - confidence: 0.88
- **[P3] Fix stale-token recovery wording:** `docs/gateway/pairing.md:221-222`
  - body: Rejecting a pending node-pair request removes it and does not issue a token. For a stale or lost token after re-pair, the docs should tell operators to make the node create a new request, or remove and re-pair, then approve that new request.
  - confidence: 0.76
