## Summary

Keep this issue open. Current main still has the Usage dashboard split described in the report: the UI requests `sessions.usage` and `usage.cost` separately, the header uses session usage totals, and unscoped `usage.cost` still reads one default-agent transcript directory instead of aggregating over the configured gateway agent scope. Open PR #70205 remains the active implementation candidate, while prior PR #42063 is closed unmerged.

## Best Possible Solution

Keep this issue open and review or land #70205, or an equivalent gateway/UI fix. The fix should make unscoped `usage.cost` aggregate over the same configured-agent scope as `sessions.usage`, preserve explicit agent scoping, include the resolved scope in cache keys, keep costs in USD, and add regression coverage proving header totals and daily bars agree for multi-agent date ranges.
