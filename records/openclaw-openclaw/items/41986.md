## Summary

Keep open. Current main still has the reported Usage dashboard split: the header uses `sessions.usage` totals while the daily chart uses `usage.cost`, and unscoped `usage.cost` still scans only the default agent transcript scope instead of the configured agent set. Open PR #70205 remains the implementation candidate; #42063 was closed unmerged.

## Best Possible Solution

Keep this issue open and review or land #70205, or an equivalent gateway/UI fix. The fix should make unscoped `usage.cost` aggregate over the same configured gateway agent set as `sessions.usage`, preserve explicit agent scoping, include the resolved scope in cache keys, keep cost units as USD, and add regression coverage proving header totals and daily bars agree for multi-agent date ranges.
