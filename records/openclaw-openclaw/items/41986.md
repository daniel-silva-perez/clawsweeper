## Summary

Keep open. Current main still has the reported Usage dashboard split: the UI renders header totals from `sessions.usage` while the daily chart comes from `usage.cost`, and current gateway code still resolves unscoped `usage.cost` through the default agent transcript directory instead of the configured multi-agent scope used by `sessions.usage`. Open PR #70205 remains the active implementation candidate; earlier PR #42063 is closed unmerged.

## Best Possible Solution

Keep this issue open and review or land #70205, or an equivalent gateway/UI fix. The fix should make unscoped `usage.cost` aggregate over the same resolved configured-agent scope as `sessions.usage`, preserve explicit agent scoping if added, include that scope in cache keys, keep costs in USD, and add regression coverage proving header totals and daily bars agree for multi-agent date ranges.
