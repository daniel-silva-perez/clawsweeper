## Summary

Keep open. Current main still has the Usage dashboard split: the header totals come from `sessions.usage`, while the daily cost chart comes from `usage.cost`; `sessions.usage` discovers configured agents, but unscoped `usage.cost` still loads one default agent transcript directory and caches only by date range. Open PR #70205 remains the active implementation candidate in the provided GitHub context, while earlier PR #42063 is closed unmerged.

## Best Possible Solution

Keep this issue open and review or land #70205, or an equivalent gateway/UI fix. The fix should make unscoped `usage.cost` aggregate over the same configured-agent scope as `sessions.usage`, preserve explicit agent scoping, include the resolved scope in cache keys, keep costs in USD, and add regression coverage proving header totals and daily bars agree for multi-agent date ranges.
