## Summary

Keep open. Current main still has the Usage dashboard split reported here: the UI loads header totals from `sessions.usage` and daily chart data from `usage.cost`, while the gateway still resolves unscoped `usage.cost` through the default agent transcript directory instead of the configured multi-agent set. Open PR #70205 remains the active implementation candidate and #42063 was closed unmerged.

## Best Possible Solution

Keep this issue open and review or land #70205, or an equivalent gateway/UI fix. The fix should make unscoped `usage.cost` aggregate over the same configured gateway agent set as `sessions.usage`, preserve explicit agent scoping, include the resolved scope in cache keys, keep costs in USD, and add regression coverage proving header totals and daily bars agree for multi-agent date ranges.
