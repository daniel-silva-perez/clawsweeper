## Summary

Keep open. Current main still has the Usage dashboard split described in the report: the UI requests `sessions.usage` and `usage.cost` separately, the header uses session totals, and unscoped `usage.cost` still resolves a single default-agent transcript directory instead of the configured-agent scope used by `sessions.usage`. Open PR #70205 remains the active implementation candidate; #42063 was closed unmerged.

## Best Possible Solution

Keep this open and review/land #70205, or an equivalent gateway/UI fix. The fix should make unscoped `usage.cost` aggregate over the same configured-agent scope as `sessions.usage`, preserve explicit agent scoping, include the resolved scope in cache keys, keep costs in USD, and add regression coverage proving header totals and daily bars agree for multi-agent date ranges.
