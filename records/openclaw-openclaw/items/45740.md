## Summary

Keep this issue open. Current main still injects raw GitHub issue bodies and review-comment JSON into the `gh-issues` child-agent prompt, the skill does not request sandboxing or a scoped tool policy for those spawned agents, and the prior/linked fixes are either closed unmerged or only partial prompt-level warnings outside current main.

## Best Possible Solution

Keep this issue open and fix it in the `gh-issues` workflow/runtime path: stop interpolating raw GitHub issue bodies and review comments into child-agent prompts, pass that content through a code-backed untrusted-content wrapper or attachment/tool-parameter style path, add malicious issue/comment regression tests, and add scoped spawn-time sandbox/tool controls for unattended cron runs without breaking notification delivery.

## Risks / Open Questions

- A prompt-only fence would be a partial mitigation, but the issue discussion correctly notes it can be bypassed by marker/tag injection unless interpolation is code-backed and sanitized.
- Restricting child tools globally could break existing `--notify-channel` behavior; the fix should be scoped to this workflow or implemented as spawn-time capability control.
- This should be framed as unattended automation hardening unless a concrete auth/policy/sandbox boundary bypass is demonstrated.
