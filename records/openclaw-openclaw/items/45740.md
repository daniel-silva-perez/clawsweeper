## Summary

Keep this open. Current main still has the reported gh-issues hardening gap: raw issue bodies and raw review-comment JSON are embedded in spawned child-agent task prompts, while the available runtime controls are sandbox selection and config-level subagent tool policy rather than a per-spawn untrusted-content or tool-scope boundary. The discussion and linked PRs are useful partial-mitigation context, but no fix is merged into OpenClaw main and the issue remains a maintainer/security-hardening decision rather than a stale cleanup close.

## What This Changes

The issue requests hardening the gh-issues skill and subagent runtime so untrusted GitHub issue bodies and review comments are not injected verbatim into tool-enabled child-agent prompts, especially in unattended cron mode.

## Best Possible Solution

Define a maintainer-approved first mitigation boundary: either escape and wrap gh-issues issue/comment content as untrusted data before child task construction or route it through a structured summary/data channel, then separately decide whether unattended gh-issues spawns should require sandboxed target agents or need first-class per-spawn tool profiles. Preserve notification behavior and add regression coverage for hostile issue/comment content and any sessions_spawn policy changes.

## Reproduction Assessment

Yes. A static current-main reproduction is high confidence: the gh-issues skill template instructs the orchestrator to substitute `{body}` and `{json_array_of_actionable_comments}` directly into child task strings, so hostile delimiters or instructions in those fields are passed verbatim. There is not yet a current-main PoC showing the model crosses a security boundary after receiving that content.

## Solution Assessment

Unclear as a final implementation because there is no merged proposed patch. The linked prompt-warning/fence approaches are useful first layers but are not the strongest maintainable fix for the runtime concern; a safer direction is structured untrusted-data handling plus an explicit decision on sandbox/tool-scope defaults for unattended subagents.
