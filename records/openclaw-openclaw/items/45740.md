## Summary

Current main still has the reported gh-issues hardening gap: raw issue bodies and review-comment JSON are interpolated into child-agent prompts, while sessions_spawn exposes sandbox selection but no per-call tool allow/deny scope. The prior linked PRs are useful partial mitigation context, but they are not merged into main and do not resolve the runtime-level concern described in the issue.

## Best Possible Solution

Keep this issue open for maintainer/security-hardening judgment. The best path is to decide the intended first mitigation boundary: escape and wrap gh-issues issue/comment bodies as untrusted data before child prompt construction, and separately decide whether sessions_spawn needs per-call tool profiles or safer unattended-mode defaults such as sandbox-required children. Treat the linked prompt-warning PRs as useful context, not as resolving the runtime hardening gap unless maintainers explicitly accept that narrower scope.
