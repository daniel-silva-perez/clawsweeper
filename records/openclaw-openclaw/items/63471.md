## Summary

Keep this open: current main has improved adjacent Docker setup paths, but the reported Dockge-style paste-a-Compose-file-plus-.env workflow is still not implemented. The default Compose service still bypasses the Dockerfile CMD escape hatch and starts the gateway without `--allow-unconfigured`, while the gateway still exits when no persisted `gateway.mode=local` exists.

## What This Changes

The issue requests a Dockge-friendly Docker Compose flow where a pasted Compose YAML plus `.env` can start OpenClaw without extra CLI setup steps.

## Best Possible Solution

Add a first-class Docker Compose bootstrap path for Dockge-style installs that idempotently creates a minimal persisted local gateway config, provisions or requires durable auth, seeds safe Control UI origins for the published local port, preserves the existing non-loopback auth/origin guardrails, and documents the flow on the Docker and ClawDock pages.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: fresh Compose with no mounted `openclaw.json` runs the Compose-defined gateway command without `--allow-unconfigured`, reaches `getGatewayStartGuardErrors`, and exits with the reported missing-config message before server startup.

## Solution Assessment

No current solution is complete. The setup script path is reasonable for scripted installs, but it still requires pre-start CLI work; simply adding `--allow-unconfigured` to Compose would not be the best fix because auth and Control UI origin guardrails still need a durable bootstrap story.
