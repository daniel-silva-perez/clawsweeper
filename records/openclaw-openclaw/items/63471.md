## Summary

Keep open. Current main still has only partial Docker friendliness: the raw Docker image can start with `--allow-unconfigured`, and the setup script can bootstrap config, but the bundled Compose file still does not provide the Dockge-style paste Compose plus .env and start workflow described in this issue.

## Best Possible Solution

Keep this open and implement a first-class Dockge/Compose bootstrap path. The likely direction is a Compose-compatible entrypoint or documented template that can initialize a minimal persisted local gateway config, generate or require a gateway token, seed safe Control UI origins for the published port, provide workable config/workspace mount defaults, and preserve the current non-loopback auth guardrails.

## Risks / Open Questions

- A naive fix that only adds `--allow-unconfigured` to the LAN-bound Compose command could weaken the existing auth and Control UI origin startup guardrails unless token generation and origin seeding are handled deliberately.
- The bundled Compose file still expects host path variables for config/workspace mounts, so a Dockge-friendly template likely needs explicit safe defaults or named-volume behavior in addition to gateway config bootstrap.
- ClawDock helper docs exist and are useful, but they still describe a helper/scripted flow rather than a generic Dockge paste-and-start Compose flow.
