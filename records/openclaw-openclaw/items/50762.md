## Summary

Keep open. Current main has partial building blocks: non-blocking Control UI chat sends, inline live tool cards, a generic chat sidebar, and backend `command_output` events. It still does not implement the requested dedicated right-side live exec/log panel, and the Control UI agent-event handler currently ignores non-`tool` streams such as `command_output`. Related #74018 was closed into broader #12831 while preserving this narrower live exec/log split; draft #60814 remains open and unmerged.

## Best Possible Solution

Keep this issue open as the narrower live exec/log split. The best path is to coordinate with #12831 and #60814, then add a dedicated right-side Control UI panel backed by existing `command_output`, `item`, and `approval` events, with command status, duration, exit code, stop/copy/full-log controls, privacy-preserving output handling, focused UI/gateway tests, and docs. If stdout/stderr separation is required, extend the event contract additively rather than scraping raw tool payloads.
