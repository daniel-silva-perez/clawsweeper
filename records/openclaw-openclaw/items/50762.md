## Summary

Keep open. Current main has backend command-output plumbing, Control UI tool-event subscription, inline live tool cards, and a generic sidebar, but it does not implement the requested dedicated right-side live exec/log panel. The related broader tracker #12831 is still open and draft #60814 is unmerged, so this remains product/UI work rather than a cleanup close.

## What This Changes

Request a Control UI split view that keeps chat responsive while live exec output, status, duration, exit code, and controls appear in a separate log panel.

## Best Possible Solution

Implement a dedicated Control UI activity/exec panel backed by existing agent events, preserving a clean chat transcript while showing live command output, status, duration, exit code, controls, and privacy-preserving summaries. Coordinate the scope with #12831 and #60814 before coding so the UI and protocol shape do not diverge.
