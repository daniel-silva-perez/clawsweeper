## Summary

Keep open. Current main has related pieces: non-blocking chat sends, inline live tool cards, a reusable chat side panel, and backend exec command-output events. It does not yet implement the requested dedicated Control UI live exec/log panel: the chat UI ignores the `command_output` stream that carries exec output/status metadata, and the existing side panel is a static markdown/tool-detail viewer opened from chat cards rather than a live command log surface. The provided related reports cover exec notification suppression or system-event leakage, not this split live-log UX.

## Best Possible Solution

Keep this issue open as a Control UI product enhancement. The likely implementation path is to add a dedicated right-side exec/log panel wired to existing `command_output`, `item`, and approval events, integrate it with the current chat split container, and add command status, duration, exit code, stop/copy/full-log controls, plus tests/docs. If stdout/stderr separation is required, extend the command-output event contract additively rather than relying on raw tool payloads.
