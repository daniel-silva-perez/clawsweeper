## Summary

Keep open. Current main has useful pieces: Control UI subscribes to agent events, renders live tool cards inline, has a generic markdown/sidebar viewer, and backend exec runs emit `command_output` events. It still lacks the requested dedicated right-side live exec/log panel that streams command output separately from chat. Related #12831 is an open broader tracker and draft #60814 is unmerged, so the narrow exec/log split is not implemented yet.

## What This Changes

Request a Control UI split view with normal chat on the left and a right-side live exec/log panel that streams command output with status, duration, exit code, and controls.

## Best Possible Solution

Keep this issue open as the narrower Control UI live exec/log split. The best path is to coordinate it with #12831 and draft #60814, then add a dedicated right-side panel backed by existing `command_output`, `item`, and `approval` events, with command status, duration, exit code, stop/copy/full-log controls, responsive layout, privacy-preserving output handling, focused UI/gateway tests, and docs. If stdout/stderr separation is required, extend the event contract additively.
