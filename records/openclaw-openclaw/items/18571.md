## Summary

Keep open. Current main still has a usable `message_sending` pre-send hook, but it does not expose the requested deterministic verification context on the shared outbound path: no propagated `sessionKey`, no `runId`, and no bounded transcript/tool-call evidence view. The direct reply path also remains only partially correlated and does not provide run or tool-history context.

## Best Possible Solution

Keep this issue open. The best path is an additive plugin hook contract that propagates `sessionKey` and a real `runId` into `message_sending` and `message_sent` where available, adds a bounded documented evidence view for recent transcript/tool-call/result data, and covers shared outbound delivery plus direct reply paths with regression tests.
