## Summary

Keep open. Current main still has the `message_sending` pre-delivery hook and optional typed `sessionKey`/`runId` fields, but the shared outbound delivery path still calls `runMessageSending` with only routing context, `OutboundSessionContext` cannot carry `runId`, and the sending event has no bounded transcript/tool-call/result evidence payload for deterministic verification. Related open PRs #73706, #44011, and #63007 overlap with the session identity subset, but none is merged and none covers the full runId plus evidence request.

## Best Possible Solution

Keep this issue open. Maintainers should first resolve or supersede the overlapping session identity PRs with one canonical additive hook-context change, then make an explicit plugin-hook contract decision for `runId` and a bounded recent transcript/tool-call/result evidence view that works across shared outbound delivery and direct reply paths, with docs and regression coverage.
