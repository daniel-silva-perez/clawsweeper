## Summary

Keep open. Current main has the `message_sending` pre-delivery hook and typed `sessionKey`/`runId` fields, but the shared outbound pre-send path still invokes `runMessageSending` with only `channelId`, `accountId`, and `conversationId`. `OutboundSessionContext` still has no `runId`, and `message_sending` has no bounded transcript/tool-call/result evidence payload for deterministic verification. Related open PRs #73706, #44011, and #63007 overlap with the session identity subset, but none is merged and none covers the full runId plus tool-evidence request.

## Best Possible Solution

Keep this issue open. First review, land, or supersede the overlapping session identity PRs with one canonical additive hook-context change. Then make a maintainer-reviewed plugin-hook contract decision for `runId` and a bounded recent transcript/tool-call/result evidence view that covers shared outbound delivery and direct reply paths, with docs at https://docs.openclaw.ai/plugins/hooks and regression tests for propagation and bounds.
