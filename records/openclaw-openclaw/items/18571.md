## Summary

Keep open. Current main has a usable message_sending hook and typed correlation fields, but the shared outbound pre-send path still does not pass sessionKey or runId into the hook context, OutboundSessionContext has no runId carrier, and there is no bounded transcript/tool-call evidence view for deterministic verification. Open PR #73706 appears to address the sessionKey subset, but it is still open and does not cover the full runId/tool-history request.

## Best Possible Solution

Keep this issue open. Review and either land or supersede #73706 for the sessionKey subset, then make an additive plugin-hook contract decision for runId plus a bounded recent transcript/tool-call/result evidence view. The implementation should cover shared outbound delivery and direct reply paths, update docs, and add regression tests for sessionKey/runId propagation and evidence bounds.
