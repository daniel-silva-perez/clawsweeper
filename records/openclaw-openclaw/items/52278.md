## Summary

Keep this PR open. Current main still has only the narrower single-leaf orphan repair path, while this PR targets multi-leaf recovery, repeated-turn preservation, raw-prompt retry dedupe, and recovered prompt images. The work is useful, and merged PR #65478 only partially overlaps it, but the PR should not merge as-is because recovered/current images are deduped before the shared prompt image sanitizer canonicalizes payloads.

## What This Changes

The PR adds embedded-runner orphaned user-turn recovery that walks trailing user leaves, carries recovered text and image payloads into the next prompt, dedupes retry echoes against effective and raw prompts, and adds focused helper tests.

## Best Possible Solution

Extend the embedded runner's current repair strategy to recover all trailing orphaned user leaves, preserve repeated real turns, dedupe only true newest retry echoes against effective and raw prompts, carry recovered images in chronological order, and canonicalize image dedupe after prompt image sanitization.
