## Summary

Keep this PR open. Current main still has only the narrower single-leaf orphaned-user repair, while this PR targets multi-leaf recovery, repeated prompt text, raw-prompt retry dedupe, and recovered images; however the PR is dirty against current main and still has a blocking image-dedupe correctness issue before it should merge.

## What This Changes

The PR branch modifies embedded-runner attempt logic and helper tests to walk trailing orphaned user messages, carry recovered text and images into the next prompt, and dedupe retry echoes while preserving repeated real turns.

## Best Possible Solution

Rebase or replace the PR on current main, keep the multi-leaf orphan recovery behavior, and move recovered/current image dedupe to the canonical post-sanitization image set so retries cannot double-attach one logical image. The final patch should preserve current image ordering contracts and add focused tests for sanitized duplicate recovery.

## Reproduction Assessment

Yes for the blocking PR defect: a retry can pass the current unsanitized image again while the orphaned leaf contains a normalized copy from the prior attempt. The PR's raw mimeType+data dedupe does not catch that case before the prompt image sanitizer canonicalizes payloads.

## Solution Assessment

No. The overall recovery direction is useful, but the proposed implementation is not the best mergeable fix until image dedupe uses the same canonical post-sanitize representation and the branch is adapted to current main.
