## Summary

Keep open. Current main still lacks the requested read-only auth profile listing surface, and the linked open PR #73462 explicitly uses closing syntax for this issue, so the right path is normal review/landing of that PR or an equivalent replacement before closing.

## What This Changes

The issue requests a script-friendly `openclaw auth list` command that lists configured auth profiles instead of hanging, returning silent output, or requiring users to read `auth-profiles.json` directly.

## Best Possible Solution

Review and land PR #73462, or an equivalent narrow fix, so `openclaw auth list` and `openclaw models auth list` read local auth profile metadata, provide script-friendly output modes, document the commands, and never print raw API keys, tokens, or OAuth material.

## Reproduction Assessment

Yes. Static inspection of current main gives a high-confidence reproduction path: the top-level `auth` command is not registered, and `models auth` has no `list` subcommand; the issue discussion also confirms a recent release reports `auth` as unknown.

## Solution Assessment

Yes. A narrow read-only listing command with redacted text/JSON/plain output is the most maintainable direction; because #73462 is open and unmerged, this issue should remain open until that implementation is accepted or replaced.
