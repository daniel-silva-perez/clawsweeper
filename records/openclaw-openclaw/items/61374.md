## Summary

Keep this PR open. Current main at c0ec58f4b6906ad06fd4c6f265d43be053b2d193 still lacks the PR's two central behavior changes: Slack block actions enqueue system events without waking the target session, and thread block actions do not pass threadTs into system-event session-key resolution. Related PR #67751 is open but unmerged and appears to cover the wake side differently, not the thread-routing side in this PR.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a narrow, tested fix that wakes the owning session only after a non-duplicate Slack interaction system event is enqueued, routes Slack interaction system events through threadTs using the existing resolveThreadSessionKeys helper, and folds in the minor duplicate-import cleanup noted by the bot review if this patch is used.
