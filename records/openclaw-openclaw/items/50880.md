## Summary

Keep open. Current main still does not satisfy the documented steer behavior: current-run injection is still gated on streaming, active embedded handles still reject non-streaming phases, active steer mode still falls through to followup, and the Discord inbound path still serializes same-key work before reply handling can attempt steering. The related focused maintainer PR #73365 is open in the supplied context and its remote PR head still exists, so this should wait for review, landing, or replacement rather than be closed now.

## What This Changes

The issue requests that `messages.queue.mode: "steer"` inject inbound messages into the active agent run at tool/model boundaries, with diagnostics when steering falls back to followup.

## Best Possible Solution

Ship a lifecycle-aware steer path that lets eligible same-session messages reach the embedded steering queue while a run is active, emits useful diagnostics for rejected steer attempts, preserves deliberate followup fallback, and updates tests/docs so the injection boundary is consistent.
