## Summary

Keep open. Current main still has the same core failure shape: steering is only attempted when the active run reports streaming, non-streaming active embedded handles are rejected, active `steer` falls back to followup, and Discord same-session inbound work is still serialized before reply handling can attempt injection. The focused related PR #73365 is open and unmerged, so this should wait for maintainer review rather than close as implemented.

## What This Changes

The issue reports that `messages.queue.mode: "steer"` falls back to followup instead of injecting inbound messages into the active turn at tool-call boundaries, especially for Discord.

## Best Possible Solution

Review and land or replace the focused lifecycle-aware steering fix in #73365: eligible same-session messages should reach the embedded steering queue while a run is active, rejected steer attempts should emit useful diagnostics, deliberate followup fallback should remain intact, and docs/tests should match the actual injection boundary.
