## Summary

Keep open. Current main still does not satisfy the steer contract: current-run injection remains gated on streaming, active non-streaming embedded handles are rejected, active steer resolves to followup, and Discord same-session work is still serialized before reply handling can attempt steering. The focused related fix PR #73365 is open and unmerged, so this should not be closed as implemented.

## What This Changes

The issue requests that `messages.queue.mode: "steer"` inject inbound messages into active agent runs instead of silently falling back to followup behavior, with diagnostics when steering is rejected.

## Best Possible Solution

The desired end state is lifecycle-aware steer delivery: eligible same-session messages should reach the embedded steering queue while a run is active, rejected steer attempts should emit useful diagnostics, followup fallback should remain deliberate, and docs/tests should describe the actual injection boundary consistently.
