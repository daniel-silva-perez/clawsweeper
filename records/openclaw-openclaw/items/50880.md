## Summary

Keep this issue open. Current main still documents `steer` as immediate active-run injection, but the reply path and embedded-run queue helper both gate injection on `isStreaming`, active `steer` mode still falls back to followup, and Discord still serializes inbound jobs by the routed session key before the reply pipeline can attempt steering. Open PR #73365 is the right follow-up thread, but it is unmerged and does not make current main fixed.

## What This Changes

`messages.queue.mode: "steer"` should inject inbound messages into an active agent turn at tool-call boundaries and log rejected steer attempts instead of silently degrading to followup behavior.

## Best Possible Solution

Keep this issue open and route it to #73365 or a narrow replacement. The fix should make active steer injection lifecycle-aware in core, add diagnostics for rejected steer attempts, and regression-test the Discord same-session inbound path so `messages.queue.mode: "steer"` can reach `queueEmbeddedPiMessage` while a run is active. After the fix merges and ships, close with release provenance.
