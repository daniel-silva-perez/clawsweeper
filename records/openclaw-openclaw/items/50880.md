## Summary

Keep open. Current main still gates steer injection on `isStreaming`, then enqueues active `steer` messages as followups when that gate is not met. The Discord path also still serializes inbound jobs by session key before they reach the reply pipeline. Open PR #73365 is an unmerged implementation candidate, not proof that current main has fixed the reported behavior.

## Best Possible Solution

Keep this issue open and use #73365 or a narrow replacement as the canonical implementation path. The fix should make steer injection lifecycle-aware, avoid silently degrading active steer messages to followup, and include focused regression coverage for the core gate plus a Discord inbound worker scenario or live proof that same-session Discord messages can reach `queueEmbeddedPiMessage` during an active run.
