## Summary

Keep open. Current main still leaves the reported availability failure reachable through raw gateway LAN and tailnet interface enumeration, while PR #71950 is already the open closing-syntax implementation candidate for the remaining fix.

## What This Changes

The issue reports that restricted sandbox network interface enumeration throws `uv_interface_addresses` during Gateway startup, crashing the Gateway and disconnecting the TUI.

## Best Possible Solution

Keep the existing Bonjour-specific mitigation, then review and land PR #71950 or an equivalent narrow change so startup-critical LAN and tailnet discovery paths use the safe interface reader while preserving the raw reader contract for callers that intentionally need propagation.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by mocking `os.networkInterfaces()` to throw and calling `pickPrimaryLanIPv4()` or `listTailnetAddresses()`; current tests still assert that both paths throw.

## Solution Assessment

Yes. The narrow linked-PR direction is the best maintainable fix: harden best-effort gateway/tailnet discovery callers without globally changing `readNetworkInterfaces()` semantics.
