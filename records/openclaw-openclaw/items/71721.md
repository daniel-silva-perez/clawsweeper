## Summary

Keep open. Current main still leaves the reported availability failure reachable through the raw gateway LAN and tailnet interface enumeration paths, and an open linked PR already tracks the narrow fix candidate.

## What This Changes

The issue asks OpenClaw Gateway to avoid crashing in Nemoclaw/OpenShell sandboxes when `os.networkInterfaces()` fails with a `uv_interface_addresses` SystemError.

## Best Possible Solution

Keep the Bonjour-specific mitigation, then review and land PR #71950 or an equivalent narrow change so startup-critical LAN and tailnet discovery use the safe interface reader while preserving the raw reader contract for callers that intentionally need propagation.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by mocking `os.networkInterfaces()` to throw and then calling `pickPrimaryLanIPv4()` or `listTailnetAddresses()`; current tests still assert both paths throw.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix: harden best-effort gateway and tailnet discovery callers without globally changing `readNetworkInterfaces()` semantics.
