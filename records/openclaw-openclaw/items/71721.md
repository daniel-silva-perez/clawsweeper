## Summary

Keep open. Current main still propagates `os.networkInterfaces()` failures through the gateway LAN and tailnet discovery helpers, and the current tests explicitly assert throwing behavior for the reported `uv_interface_addresses` failure. PR #71950 is relevant but remains open and unmerged in the provided context, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review, land, or close PR #71950. The best fix is a narrow gateway/infra patch that routes the LAN and tailnet discovery helpers through `safeNetworkInterfaces()` and updates the gateway and tailnet tests to assert graceful fallback instead of thrown errors.
