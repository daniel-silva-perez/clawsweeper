## Summary

Keep open. Current main still has throwing `os.networkInterfaces()` paths in the gateway LAN and tailnet helpers, while the linked open PR is the active implementation candidate for this issue; the Bonjour/ciao path is partially hardened but does not cover all current gateway discovery and tailnet callers.

## What This Changes

The issue requests that OpenClaw gateway degrade gracefully when sandboxed `os.networkInterfaces()` fails with `uv_interface_addresses`, so the gateway and TUI connection do not crash.

## Best Possible Solution

Review and land PR #71950, or an equivalent narrow fix, so startup-critical gateway LAN and tailnet discovery paths use the existing safe network-interface helper and degrade to missing-address fallbacks while preserving the raw reader for callers that intentionally want propagation.

## Reproduction Assessment

Yes. A high-confidence unit reproduction exists by mocking `os.networkInterfaces()` to throw and then calling `pickPrimaryLanIPv4()` or `listTailnetAddresses()`; current tests still assert those helpers throw.

## Solution Assessment

Yes. The narrowest maintainable direction is to reuse `safeNetworkInterfaces()` in the gateway/tailnet callers that should be best-effort, rather than changing the raw reader contract globally.
