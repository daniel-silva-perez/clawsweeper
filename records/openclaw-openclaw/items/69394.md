## Summary

Current main still preserves text before a trailing `NO_REPLY` in the non-streaming auto-reply normalizer, and the current tests explicitly assert that behavior. This PR is therefore not obsolete, but it should not be auto-closed or auto-merged: it has an unresolved review finding around the `stripped.length < text.length` heuristic, and related open PRs propose different product semantics for the same #66701 leak.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to pick one canonical fix for #66701 across #66755, #42873, and this PR, then implement trailing-token detection without the trim-length false positive, add regression coverage for both reasoning-preamble suppression and non-trailing `NO_REPLY` mentions with trailing whitespace, and close the duplicate PRs after the chosen fix lands.
