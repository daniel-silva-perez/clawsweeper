## Summary

Current main still strips a trailing `NO_REPLY` from mixed text and delivers the remaining preamble in the auto-reply path, so this PR is not implemented on main. Keep it open for maintainer review: the reported leak is real, but the proposed trim-length heuristic can suppress legitimate non-trailing `NO_REPLY` mentions and the related open PRs make different product choices about the #30916/#19537 substantive-reply contract.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose one canonical fix across #66701, #42472, #42873, #66755, #73153, and this PR; implement an explicit trailing-token, own-line, or reasoning-prefaced predicate rather than trim-length inference; intentionally preserve or change the #30916/#19537 substantive-reply contract; add regression tests for reasoning-preamble suppression plus non-trailing `NO_REPLY` mentions with trailing whitespace; then close duplicate PRs after the chosen fix lands.
