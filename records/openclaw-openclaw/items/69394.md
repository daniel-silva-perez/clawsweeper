## Summary

Current main still strips a trailing NO_REPLY token from mixed text and delivers the remaining preamble in the auto-reply normalizer, so this PR is not implemented or obsolete. Keep it open for maintainer review because the reported leak is real, but the proposed trim-length heuristic can suppress legitimate non-trailing NO_REPLY mentions with trailing whitespace, and related open PRs represent different policy choices for the #30916/#19537 substantive-reply contract.

## What This Changes

The PR changes auto-reply payload normalization to suppress an entire reply when stripping detects a trailing NO_REPLY token, and updates normalizeReplyPayload tests to expect full suppression for trailing-token mixed text.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose one canonical fix across #66701, #42472, #42873, #66755, and this PR; implement an explicit trailing-token, own-line, or reasoning-prefixed predicate rather than trim-length inference; deliberately preserve or change the #30916/#19537 substantive-reply contract; add regression tests for reasoning-preamble suppression plus non-trailing NO_REPLY mentions with trailing whitespace; then close duplicate PRs after the chosen fix lands.
