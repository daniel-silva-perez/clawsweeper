## Summary

Keep open. Current main has local/process-bound cron scheduling, in-process command lanes, SQLite/file-backed task ledgers, and Task Flow state, but it does not implement the requested Redis/BullMQ-style distributed job orchestration, shared worker claiming, dead-letter handling, or horizontal worker coordination. The existing maintainer-facing comment explicitly keeps this as architecture input, and the related #13613 covers only the narrower multi-gateway cron coordination slice.

## Best Possible Solution

Keep open for maintainer architecture review. Decide whether OpenClaw should add a generic distributed scheduler/job backend abstraction, possibly plugin-capable, with shared queue/state, worker claiming, retries/backoff, dead-letter semantics, cron support, and observability; otherwise document Redis/BullMQ-style orchestration as an external deployment or plugin pattern while preserving the ScalyClaw prototype as prior art. Coordinate the cron HA portion with #13613.
