## Summary

Keep this open. Current main still has local/process-bound cron, an in-process command queue, SQLite-backed task ledgers, and Task Flow state, but it does not implement the requested Redis/BullMQ-style distributed job orchestration, shared worker claiming, dead-letter handling, or horizontal worker coordination. The prior issue comment and related #13613 both support keeping this as maintainer architecture input rather than conservative cleanup.

## Best Possible Solution

Keep this open for maintainer architecture review. The best path is to decide whether OpenClaw should add a generic distributed scheduler/job backend abstraction, possibly plugin-capable, with shared queue/state, worker claiming, retries/backoff, failure/dead-letter semantics, cron support, and observability; otherwise document Redis/BullMQ-style orchestration as an external deployment pattern while preserving the ScalyClaw prototype as prior art. Coordinate the narrower cron HA portion with #13613.
