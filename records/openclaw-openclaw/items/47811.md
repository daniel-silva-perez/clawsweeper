## Summary

Keep this issue open. Current main supports named profile homes, isolated multi-gateway docs, and config-mismatch diagnostics, but it still does not provide the requested authoritative multi-profile target report or define/fail loudly on custom-profile casing for macOS service targeting.

## Best Possible Solution

Keep this open and narrow the follow-up to core CLI/gateway behavior: define canonical custom-profile casing or fail loudly on ambiguous casing, make status/restart/agent surfaces print the resolved profile, config path, state directory, service label/process boundary, port/probe target, and session store path, and add macOS LaunchAgent tests for multiple named homes and case-mismatched profile names.
