## Summary

Keep this issue open. Current main has partially improved the multi-profile story with per-profile state/config paths, profile-specific service names, config mismatch diagnostics, and multi-gateway docs, but the central request is not fully implemented: profile casing is still preserved for custom profiles, status output is not yet an authoritative profile/service-boundary report, and no canonical duplicate or merged fix was found in the provided related context.

## Best Possible Solution

Keep open and narrow the maintainer path to the remaining core behavior: define canonical profile-name casing or fail loudly on ambiguous casing, add explicit resolved profile/config/state/service-label/session-store output to status/restart/agent surfaces, and add tests for macOS LaunchAgent targeting across multiple ~/.openclaw-* profiles. Existing docs and config mismatch diagnostics should be treated as the baseline, not as a complete fix.
