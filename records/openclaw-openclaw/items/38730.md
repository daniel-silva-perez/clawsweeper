## Summary

Keep open. Current main has cron run-session retention/reaper mitigation and session cleanup UX, but the requested creation/update guardrail for high-frequency cron schedules is still absent across the CLI add/edit paths, Gateway add/update handlers, Control UI add/edit flow, and agent cron-tool add/update flow. The same-author implementation PR #38731 remains open and explicitly links this issue for closure, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and treat #38731 as the active implementation candidate. The best fix is a small shared high-frequency every-schedule warning helper applied consistently to CLI add/edit, Gateway add/update, Control UI add/edit, and agent-tool-created cron jobs while keeping short intervals valid and non-breaking. After #38731 or an equivalent fix lands, close this as implemented with the landed SHA and release provenance.
