## Summary

Keep this PR open. Current main still resolves Telegram reply-target media solely from `reply_to_message` plus `hasInboundMedia`, so a human reply to a bot-sent photo can still fetch and inject the bot's own media. The PR remains the active implementation candidate for #57278, and the provided branch context shows the earlier numeric-ID review concern was addressed with regression tests.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent patch on main that adds a numeric bot-ID self-authorship guard before Telegram reply-media fetches, preserves user-authored and other-bot reply-media behavior, includes debounce-path regression coverage, and carries the changelog credit for #57278.
