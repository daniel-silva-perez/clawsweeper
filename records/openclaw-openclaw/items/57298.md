## Summary

Keep this PR open. Current main still resolves Telegram reply-target media without checking whether the replied-to media message was authored by the bot, while the PR branch adds the missing numeric bot-id guard, regression coverage, and changelog credit for the linked Telegram bug.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent patch on main that adds the numeric bot-id self-authorship guard before Telegram reply-media fetches, preserves user-authored and other-bot reply-media behavior, keeps the debounce-path regression coverage, and carries the changelog credit for #57278.
