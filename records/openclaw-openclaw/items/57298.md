## Summary

Keep this PR open. Current main still resolves and fetches Telegram reply-target media without checking whether the replied-to message was authored by the bot, so the PR’s central fix is not implemented on main. The PR branch now has a narrow numeric bot-id guard, regression tests, and changelog credit; the earlier review concern about `undefined === undefined` was addressed by the branch repair. Security pass: the current diff touches only Telegram runtime logic, Telegram tests, and CHANGELOG.md, with no CI, dependency, workflow, package-resolution, lifecycle-script, secret, or artifact-download changes.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent patch on main that adds a numeric bot-id self-authorship guard before Telegram reply-media fetches, preserves normal user-authored reply-media behavior, keeps the debounce-path regression coverage, and includes the changelog credit for the linked Telegram bug.
