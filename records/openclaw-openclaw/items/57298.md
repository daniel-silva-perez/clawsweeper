## Summary

Keep this PR open. Current main still resolves Telegram reply-target media from `reply_to_message` after only checking `hasInboundMedia`, so a user reply to the bot's own photo can still fetch and inject that bot-authored media. The PR remains the active implementation candidate for #57278, and the provided discussion shows the earlier numeric-ID review concern was addressed on the branch with validation.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent patch on main that adds a numeric bot-ID self-authorship guard before Telegram reply-media fetches, preserves user-authored and different-bot reply-media behavior, includes debounce-path regression coverage, and carries the changelog credit for #57278.
