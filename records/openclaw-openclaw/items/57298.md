## Summary

Current main still downloads Telegram reply-target media whenever the replied-to message has inbound media; it does not skip media authored by the bot itself. This PR remains useful as the implementation candidate for #57278, and the provided current branch context shows the earlier numeric-ID review concern was addressed with regression coverage.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent patch on main that adds the numeric bot-ID self-authorship guard before Telegram reply-media fetches, preserves user-authored and other-bot reply-media behavior, keeps debounce-path regression coverage, and carries the changelog credit for #57278.
