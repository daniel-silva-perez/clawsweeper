## Summary

Keep open. Current main still remaps tar entry paths during backup creation but does not rewrite or avoid tar hardlink targets, and `backup verify` still validates only entry paths. The related symlink duplicate-path fix and broader backup exclusion/redesign items do not cover this hardlink `linkpath` safety gap.

## What This Changes

Ensure backup archives created and verified by OpenClaw do not contain tar hardlink targets with traversal segments that fail common extraction tools.

## Best Possible Solution

Backup creation should emit restore-safe hardlink representation after OpenClaw's archive path remapping, and backup verification should reject unsafe tar link targets before reporting success. Keep configurable excludes and broader backup redesign in their separate tracks.

## Repair Work Prompt

Fix the backup hardlink target safety bug reported in #54242. Current `openclaw backup create` remaps tar `entry.path` only, so hardlink entries can retain `linkpath` values containing `..`; current `openclaw backup verify` reads only `entry.path`, so `--verify` can pass archives that macOS bsdtar rejects during broad extraction. Keep the fix narrowly in backup archive creation/verification and tests. Prefer a restore-safe representation: either dereference/avoid hardlink tar entries for backup archives or rewrite hardlink `linkpath` consistently with the remapped archive paths so it cannot be absolute, backslash-based, or traversal-based. Extend verification to inspect tar link targets for `Link`/hardlink entries and reject unsafe link targets. Do not implement configurable excludes or the broader backup redesign from #40786/#67990, and do not change the `tar` dependency unless a maintainer explicitly approves. Add regression tests with a hardlinked payload whose generated link target would otherwise contain `..`; include create-time and verify-time coverage. Docs/changelog: update backup docs only if user-visible semantics change; add a changelog fix entry with `Thanks @amzzzzzzz` if this ships as a user-facing bug fix.
