## Summary

This PR can close because the core streaming/backpressure concern is already satisfied on current main and in the shipped v2026.4.26 release: OpenClaw uses the pinned tar file-mode API, and tar@7.5.13 implements that path by piping a Pack stream to a WriteStream with error propagation. The PR is also stale against the current backup implementation layout and contains recovery guidance for flags that main does not register.

## What This Changes

The PR rewrites backup archive creation to pipe tar's no-file stream into a write stream, adds progress and OOM handling, and adds backup tests for stream and OOM failures.

## Best Possible Solution

Close this stale explicit-stream rewrite and keep the shipped tar file-mode plus atomic temp-file implementation. Any remaining large-installation UX, such as exclude patterns, smarter defaults, or progress policy, should stay in the separate #42273/#44288 path rather than this branch.
