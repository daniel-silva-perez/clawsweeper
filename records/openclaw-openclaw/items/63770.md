## Summary

Current main still ignores explicit `deleteAfterRun: true` for successful `every` and `cron` jobs, and an open PR with closing syntax already targets the bug, so this issue should stay open until that PR is reviewed and either merged or rejected.

## What This Changes

The issue requests that `deleteAfterRun: true` delete successful `every` and `cron` schedule jobs after their first run instead of only applying to `at` schedules.

## Best Possible Solution

Keep this issue open and review PR #64375 as the canonical implementation candidate. If maintainers agree that explicit `deleteAfterRun: true` should self-delete `every` and `cron` jobs after a successful run, land the narrow guard removal with the regression tests and a user-facing changelog entry; otherwise close or revise the PR and record the product decision here.
