## Summary

Keep this PR open. Current main at 189c91eae6e1f4f280179961bca2c4153cf42e86 still does not implement the PR’s intended shared human-day semantics: there is no human-day helper, daily session reset remains gateway-host-local, session-memory artifacts use event time plus host/TZ rather than agents.defaults.userTimezone or transcript activity time, and usage buckets plus /usage cost still derive day labels separately. The PR discussion also shows active review on real edge cases, with author follow-up fixes rather than an obsolete or superseded change.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. The best path is to review and either land this shared human-day helper approach or a narrower successor that gives session reset, session-memory artifacts, memory flush paths, and usage summaries one documented daily-boundary contract while preserving UTC/protocol timestamps and caller-requested usage date modes.
