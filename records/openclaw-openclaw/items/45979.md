## Summary

Keep open. Current main still has the mutable `params.dedupe.size - DEDUPE_MAX` loop bound, and the PR’s one-line stabilization is a correct fix for the under-eviction bug. It should go through maintainer review because #73841 overlaps the same gateway loop and adds focused regression coverage.

## What This Changes

This PR changes `src/gateway/server-maintenance.ts` to compute the dedupe cache overflow count once before deleting timestamp-sorted entries.

## Best Possible Solution

Land one canonical gateway maintenance fix that uses a stable overflow count, keeps timestamp-based oldest-entry eviction, and includes a focused regression test proving multi-entry dedupe overflow returns to `DEDUPE_MAX`.
