## Summary

Keep this PR open. Current main still has the mutable dedupe cleanup loop bound, while this PR makes the narrow source change needed to prevent under-eviction; the remaining decision is maintainer choice between this minimal patch and overlapping #73841, preferably with regression coverage.

## What This Changes

The PR changes `src/gateway/server-maintenance.ts` to compute the dedupe cache overflow count once before deleting timestamp-sorted entries.

## Best Possible Solution

Land one canonical gateway maintenance fix that uses a stable overflow count, preserves timestamp-based oldest-entry eviction, and adds a focused regression test where overflow greater than one returns exactly to `DEDUPE_MAX`.

## Reproduction Assessment

Yes. The current-main loop condition is visible in `src/gateway/server-maintenance.ts`, and a direct walkthrough of a dedupe map with more than one excess entry shows each deletion reduces the remaining loop bound before all excess entries are removed.

## Solution Assessment

Yes for this PR’s code change: precomputing `excess` is the narrow maintainable correction. The stronger landing path is to pair it with a regression test or choose #73841 after strengthening its test so it fails against the current multi-entry under-eviction bug.
