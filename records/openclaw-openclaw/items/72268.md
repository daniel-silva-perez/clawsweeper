## Summary

Keep open. Current main still has the nested approval metadata bug, and the PR targets the right parser path, but the current branch is not merge-ready because it still broadens approval-status parsing on generic command output and carries unrelated generated SDK hash drift.

## What This Changes

The PR replaces exec denied/finished regex parsing with a balanced-parentheses parser, adds nested metadata parser and followup regression tests, and modifies the generated plugin SDK API hash file.

## Best Possible Solution

Land a narrowed version that keeps the balanced metadata parser and nested denied/finished regression coverage, removes unrelated generated baseline churn, rebases cleanly, and constrains approval-wrapper parsing to trusted exec lifecycle data or a structured exec status rather than arbitrary command output.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: the gateway emits `approval-timeout (allowlist-miss)`, the parser regex cannot match nested metadata, and followup routing depends on that parser to choose the denied branch.

## Solution Assessment

No, not as currently proposed. The parser direction fixes the observable bug, but the merge-ready solution should first remove the unrelated baseline hash change and close the generic-output spoofing gap.
