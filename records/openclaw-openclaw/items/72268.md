## Summary

Keep this PR open. Current main still has the nested-parentheses exec approval parsing bug, and the PR is a focused implementation candidate, but it should not merge as-is until the spoofable approval-wrapper parsing concern and unrelated generated baseline hash change are resolved.

## What This Changes

The PR replaces exec approval denied/finished regex parsing with a balanced-parentheses parser, adds nested metadata parser and followup regression tests, and currently also changes the generated plugin SDK API hash file.

## Best Possible Solution

Land a narrowed version that keeps the balanced metadata parser and nested denied/finished regression tests, removes unrelated generated baseline churn, and scopes approval-wrapper parsing to trusted exec lifecycle data or structured exec status before broadening accepted strings from generic output.

## Reproduction Assessment

Yes. Current main has a direct source-level reproduction: the existing denied and finished regexes both fail nested metadata, while the gateway emits `approval-timeout (allowlist-miss)` in denied followups.

## Solution Assessment

No, not as currently proposed. The parser direction fixes the reported bug, but the merge-ready fix should also remove the unrelated baseline hash change and resolve the spoofable generic-output parsing path.
