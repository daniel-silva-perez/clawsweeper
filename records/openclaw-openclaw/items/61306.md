## Summary

Keep this PR open. Current main does not implement the Claw mission-control backbone: there is no `src/claw` subsystem, no Claw shared mission types, no Claw gateway RPC handlers, no Control UI Claw tab/controller/view, no `docs/claw` spec nav entry, and no Claw smoke workflow. The PR is also an XL dirty-candidate with unresolved review and security-sensitive concerns, so the right cleanup path is maintainer review/splitting rather than auto-close.

## Best Possible Solution

Keep the PR open for maintainer judgment, but require it to be split or narrowed before serious merge consideration: separate Claw specs, mission service/config/gateway RPC, Control UI, browser hardening, and CI/test stabilization; drop unrelated memory-core/Zalo/locale churn; then resolve the P1/security review comments and rerun org-owned validation on the exact head.
