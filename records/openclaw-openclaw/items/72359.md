## Summary

Keep open. The separated single-shot Active Memory request is not implemented on current main: the plugin config and manifest have no `mode` or `single-shot` option, the runtime still always builds an embedded memory sub-agent via `runEmbeddedPiAgent`, and the docs still describe Active Memory as a blocking memory sub-agent with tuning knobs rather than a one-search/one-completion path.

## Best Possible Solution

Keep this open as the narrow opt-in Active Memory single-shot mode request. The best fix is to add a default-preserving config mode, keep the current embedded-agent loop as the default, and implement a `single-shot` branch inside the Active Memory plugin that performs one bounded memory search plus one bounded simple completion, then parses `NONE` or a capped summary with docs and regression tests. If maintainers want one umbrella tracker, explicitly consolidate this into #65309 rather than closing it as already done.

## Clownfish Work Prompt

Implement the current issue as an opt-in Active Memory `single-shot` mode without changing the default embedded-agent behavior. Add a config field such as `plugins.entries.active-memory.config.mode` with default `agent` or equivalent existing behavior and opt-in `single-shot`. In single-shot mode, reuse the existing query construction, run one active memory search through the current memory search manager with the configured qmd search mode, build a compact prompt from the latest message/query plus hits, call the existing simple-completion helper once with the configured Active Memory model resolution, and parse `NONE` or a summary capped by `maxSummaryChars`. Preserve cache/status/debug/log behavior where practical, respect `timeoutMs` with abort handling, do not expose tools or call `runEmbeddedPiAgent` in single-shot mode, and do not change the existing multi-turn path for recall-heavy/contextual use cases. Update `extensions/active-memory/openclaw.plugin.json`, docs, and focused tests for config validation, mode defaulting, single-shot success, `NONE`, timeout/fail-open, and proof that default mode still calls `runEmbeddedPiAgent`. Changelog only if maintainers consider this user-facing enough for the active version; if added, follow repository attribution rules and do not credit automation.
