## Summary

Keep open. Current main still has a plausible cron-ok/no-artifacts path: memory-core writes dreaming reports/events inside per-workspace processing, but catches workspace failures, logs them, and still returns a handled processed result. The docs and writer code confirm artifacts/events are expected, while the REM harness only proves eligible input, not successful mutation.

## What This Changes

The issue reports that the memory-core managed dreaming cron reports successful runs with eligible REM/deep material but writes no new DREAMS.md, per-phase dreaming reports, or memory.dream.completed events after the last known-good run.

## Best Possible Solution

Make the memory-core managed dreaming sweep enforce an observable success invariant: with enabled phases and eligible workspaces, expected report files and memory.dream.completed events should be written, and all-workspace artifact-write failure should fail or become clearly diagnosable instead of looking like a successful processed cron run.
