## Summary

Keep open. Current main still has a plausible cron-ok/no-artifacts path: memory-core expects phase reports and memory.dream.completed events, but per-workspace failures are caught and the managed cron handler still returns a processed result. The prior discussion correctly distinguishes this from the older 6-8 ms no-op report and from the adjacent diary-only/backfill reports.

## What This Changes

The issue asks memory-core dreaming cron runs with eligible REM/deep material to write DREAMS.md, per-phase dreaming reports, and memory.dream.completed events instead of reporting success with no visible artifacts.

Make the memory-core managed dreaming sweep enforce an observable success invariant: when enabled phases and eligible workspaces run, expected report files and memory.dream.completed events are written, and an all-workspace artifact-write failure fails or surfaces a clearly diagnosable cron result instead of looking processed.
