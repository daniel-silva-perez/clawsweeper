## Summary

Keep open: current main still implements the opposite behavior and documents it as a Control UI remote-fetch hardening policy. Because open PR #41025 is still paired with this issue and a correct fix would need a renderer, CSP, docs, and security-policy decision, this is not safe maintainer cleanup to close.

## What This Changes

Allow Control UI chat markdown image syntax with remote http/https URLs to render inline instead of flattening those images to alt text.

## Best Possible Solution

Keep this open until maintainers decide the Control UI remote-media policy. If remote images are allowed, implement the behavior across the renderer, sanitizer expectations, CSP, tests, docs, and an explicit safety model; otherwise close or redirect the request to the broader inline-media policy discussion.
