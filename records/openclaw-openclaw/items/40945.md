## Summary

Keep open. Current main still blocks remote markdown images in the renderer, tests, CSP, docs, and changelog-backed hardening policy, and open PR #41025 still targets this issue with closing syntax. A renderer-only change would not be enough while the Control UI CSP and documented no-remote-fetch policy remain in place.

## What This Changes

Allow Control UI chat markdown to render safe remote http/https image URLs inline instead of flattening them to alt text.

## Best Possible Solution

Keep this issue open while maintainers decide the Control UI remote-media policy. A complete fix should update the markdown renderer, CSP, tests, and public docs together, likely through an explicit opt-in, allowlist, managed-media proxy, or a clear decision to keep remote markdown images blocked.
