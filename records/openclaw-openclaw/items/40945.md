## Summary

Keep open: current main still deliberately flattens remote markdown images and blocks remote image fetches through Control UI CSP/docs, while open PR #41025 still references this issue with closing syntax. The remaining decision is a product/security policy change, not an already-implemented cleanup.

## What This Changes

The issue requests that Control UI chat markdown render remote http/https image URLs inline instead of degrading them to alt text.

## Best Possible Solution

Keep this open until maintainers decide the Control UI remote-media policy. A complete implementation should align the renderer, sanitizer expectations, CSP, tests, docs, and likely an explicit opt-in, allowlist, or managed-media proxy if remote images are allowed.
