## Summary

Keep this issue open. Current main still flattens remote markdown images to alt text in Control UI, tests and docs preserve that behavior as a deliberate no-remote-fetch policy, and the provided GitHub context shows open PR #41025 uses closing syntax for this issue. That PR is an implementation candidate, not a reason to close before merge.

## Best Possible Solution

Keep the issue open and review PR #41025 or a replacement implementation after the product/security policy is decided. A complete fix should update the markdown renderer, Control UI CSP, docs, and regression tests together, likely with an explicit opt-in, allowlist, managed-media proxy, or a clear maintainer decision to keep remote markdown images blocked. The broader inline-media request in #64607 can remain the umbrella for media previews, but this focused markdown-image issue still has an open implementation candidate.
