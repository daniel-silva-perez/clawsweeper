## Summary

Keep open. Current main still intentionally flattens remote Control UI markdown images to alt text, tests and docs preserve that no-remote-fetch behavior, and the Control UI CSP still excludes remote http(s) image loads. The supplied GitHub context also shows open PR #41025 uses closing syntax for this issue, so this issue should remain open until that implementation candidate is reviewed and merged or closed.

## Best Possible Solution

Keep this issue open and resolve the product/security policy first, then review PR #41025 or a replacement implementation. A complete fix would need to address the renderer, CSP, docs, tests, and privacy model together, likely through an explicit opt-in, allowlist, managed-media proxy, or a deliberate decision to keep remote markdown images blocked. The broader open media-preview request #64607 can track larger inline media UX, but it does not by itself close this focused bug while PR #41025 remains open.
