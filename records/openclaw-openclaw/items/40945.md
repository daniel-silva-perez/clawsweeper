## Summary

Keep open. Current main still flattens remote Control UI markdown images to alt text, and the surrounding tests, CSP, docs, and changelog describe that as deliberate no-remote-fetch hardening. Open PR #41025 directly targets this issue with closing syntax, so the next step is maintainer product/security review rather than cleanup closure.

## What This Changes

Control UI chat should render inline remote http/https markdown image URLs instead of flattening them to alt text.

## Best Possible Solution

Define the Control UI remote-media policy first. If remote images should be supported, implement it coherently across renderer, sanitizer expectations, CSP, docs, and tests with a reviewed control model such as opt-in, allowlist, or managed proxy; otherwise keep the shipped hardening and close this issue after maintainer review.

## Reproduction Assessment

Yes. The static reproduction is high-confidence: `toSanitizedMarkdownHtml("![Alt text](https://example.com/img.png)")` reaches the renderer branch that returns escaped alt text, and the current colocated test asserts no img output for that case.

## Solution Assessment

Unclear. The requested renderer-only direction is not the best standalone fix while current CSP, docs, and tests intentionally block remote image fetches; the maintainable solution needs an explicit remote-media policy decision.
