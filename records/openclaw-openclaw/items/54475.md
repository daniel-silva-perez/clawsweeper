## Summary

Keep this PR open. Current main still lacks the pre-hello close tolerance in both the one-shot gateway call wrapper and the GatewayClient connect path, and the same-author related PR #54774 is also open, so this should not be auto-closed as implemented or superseded.

## Best Possible Solution

Review this PR together with #54774 and decide which same-author PR should be canonical. The implementation path should stay in `src/gateway/call.ts` and `src/gateway/client.ts`, use the structured close-error approach from the PR, and keep focused regression tests in `src/gateway/call.test.ts` and `src/gateway/client.test.ts`; broader cron retry work can remain separate unless maintainers choose to consolidate it under #49109.
