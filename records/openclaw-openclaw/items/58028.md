## Summary

Keep this issue open. Current main at 230f7122dd5cd8a37a55e671254bd9be8278f6e8 still does not provide a top-level `/resume` command, recent-session picker, `/resume last`, or arbitrary previous JSONL session restore. Existing restore support is limited to compaction checkpoints, and the directly related `/resume` PR #36481 was closed unmerged while #51889 remains an open implementation candidate for archived transcript restore.

## Best Possible Solution

Keep the issue open and either review/land a focused implementation candidate or build a first-party session resume flow in the core chat command surface. The likely path is a `/resume` command registered under `src/auto-reply`, backed by gateway session-store/archive support that can list recent reset/archived sessions, select by id/index or `last`, safely restore the selected transcript, and update docs/tests for the user-facing behavior.
