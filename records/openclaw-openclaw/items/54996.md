## Summary

Keep this issue open. Current main partially addresses the problem by estimating transcript tokens for preflight compaction, but the shared user-facing token snapshot still has no estimated/source state and still suppresses or reports unknown context when provider usage is missing, stale, or all-zero.

## What This Changes

Request local context-token estimation from stored session JSONL when provider usage is unavailable, show it as estimated in status/usage surfaces, and use it for compaction threshold decisions.

## Best Possible Solution

Add a single context-usage snapshot contract that can carry fresh provider totals or estimated transcript-derived totals with an explicit source/estimated flag, preserve fresh provider snapshots as authoritative, and render the estimate consistently in /status, /usage, session rows, and context notices while letting compaction use it with conservative headroom.
