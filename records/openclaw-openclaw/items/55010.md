## Summary

Keep open. Current main still contains the incorrect Plugin SDK migration table entry: it documents `mapAllowlistResolutionInputs` under a nonexistent `plugin-sdk/allowlist-resolution` subpath. The SDK source, package export map, generated entrypoint inventory, and contract test all show the helper belongs to `openclaw/plugin-sdk/allow-from`, so this PR remains a valid docs correction rather than obsolete work.

## Best Possible Solution

Review and merge or manually apply the one-line migration-guide correction so the public Plugin SDK migration docs match the existing `openclaw/plugin-sdk/allow-from` export contract. No core code or security-sensitive change is needed.
