## Summary

Keep this PR open. Current main only has a small slice of the intended agent-tools localization: effective tool source badges already use `agentTools.*`, but the PR’s central scope, the top-level agent tools/skills panel header, bulk buttons, save/loading states, filter UI, counts, and skill callouts, still contains hardcoded English. The PR is therefore not implemented on main and still represents useful work, though it needs a rebase/regeneration pass against the current locale pipeline.

## Best Possible Solution

Keep this PR open and review it as an implementation candidate. The best path is to rebase on current main, move the remaining top-level agent tools/skills panel strings through `t(...)`, add the English source keys, regenerate all shipped locales through the UI i18n pipeline, and rerun the targeted UI build/translation test after resolving any conflicts.
