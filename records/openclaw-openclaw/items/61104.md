## Summary

Keep this PR open. Current main only implements the badge-label slice of the requested agent tools/skills localization, while the panel header, bulk buttons, save states, skill callouts, filter, counts, and empty state still contain hardcoded English; the PR remains useful but needs a rebase against the current generated-locale pipeline.

## What This Changes

The PR replaces hardcoded top-level agent tools and agent skills panel labels with `agentTools.*` translation lookups and adds locale entries for the then-supported English, German, Spanish, Portuguese, Simplified Chinese, and Traditional Chinese bundles.

## Best Possible Solution

Rebase the PR on current main, keep the existing badge key naming where possible, move the remaining top-level agent tools/skills strings through English source keys, regenerate every shipped locale and `.i18n` metadata with the UI i18n pipeline, then validate the targeted translation test and UI build.

## Reproduction Assessment

Yes. Static inspection of current main gives a high-confidence reproduction path: switch the Control UI away from English, open the agent tools/skills panel, and the literals shown at `ui/src/ui/views/agents-panels-tools-skills.ts:340` and `:709` will still render in English.

## Solution Assessment

No as-is. The localization direction is right, and the earlier bot-reported missing-key issue appears fixed in the PR branch, but the current best solution needs a rebase plus generated full-locale refresh before this can be considered merge-ready.
