## What This Changes

The issue reports that the Signal channel should keep consuming the long-lived `signal-cli` `/api/v1/events` SSE stream and update inbound activity instead of reconnecting after a 10s timeout.

## Best Possible Solution

Land one narrow Signal plugin fix that disables the short deadline only for the long-lived receive SSE stream, keeps bounded RPC/check deadlines intact, adds regression coverage for idle SSE behavior, and verifies inbound activity status after a received event.
