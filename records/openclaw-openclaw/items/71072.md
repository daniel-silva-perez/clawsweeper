## Summary

Keep open. Current main preserves the final empty-result semantics for `onlyPluginIds: []`, but it still creates the loader/cache context, checks cache, creates the registry/runtime proxy, runs plugin discovery, and loads the manifest registry before scoped filtering. The PR's O(1) empty-scope fast-path is not implemented on main, and the proposed diff needs rebasing onto the current `PluginLoaderCacheState` reentry API.

## Best Possible Solution

Keep this PR open and review it as a narrow performance optimization. The best implementation is to rebase onto current main, add an explicit empty-scope early return in `loadOpenClawPlugins` using the current `PluginLoaderCacheState` reentry API, preserve activation-side-effect clearing for activating loads, return `createEmptyPluginRegistry()`, and add focused loader coverage that proves `onlyPluginIds: []` skips discovery and manifest registry loading. No changelog entry is needed unless maintainers want to call out startup performance in release notes.
