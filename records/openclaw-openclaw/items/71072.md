## Summary

Keep open. Current main preserves the empty-scope semantics, but it still reaches the empty result after cache handling, registry creation, plugin discovery, and manifest registry loading, so the PR's O(1) fast-path is not implemented on main.

## Best Possible Solution

Keep the PR open and review it as a narrow performance optimization. The best fix is to rebase it onto current main, implement the empty explicit-scope early return using the current PluginLoaderCacheState/reentry API, preserve activation side-effect clearing, and add targeted loader coverage proving discovery and manifest registry loading are skipped for onlyPluginIds: [].
