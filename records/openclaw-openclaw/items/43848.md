## Summary

Keep this open: current main covers the Telegram forum-topic `threadId` preservation part, but the `ANNOUNCE_SKIP` visible-delivery fallback and requester transcript pollution paths still appear present in the A2A flow.

## What This Changes

The issue reports that Telegram forum-topic `sessions_send` delivery can lose target `threadId`, suppress visible topic handoffs on `ANNOUNCE_SKIP`, and echo topic-to-topic replies into the requester transcript.

## Best Possible Solution

Keep the shipped `threadId` preservation, then land a coordinated fix that adds visible Telegram topic fallback after `ANNOUNCE_SKIP` when a real target reply exists and prevents same-agent Telegram topic-to-topic handoffs from persisting target replies as fresh requester-side `inter_session` turns.
