## Summary

Keep open. Current main preserves Telegram forum-topic thread routing for the announce send, but the visible-delivery fallback after ANNOUNCE_SKIP and the requester-side inter_session ping-pong pollution path still remain in the A2A flow.

## What This Changes

The issue reports that Telegram forum-topic sessions_send handoffs can lose thread delivery, suppress a visible target-topic post after ANNOUNCE_SKIP, and echo target replies into the requester transcript.

## Best Possible Solution

Keep the shipped threadId preservation, then land one coordinated A2A fix that reuses a real target reply for visible Telegram topic delivery when ANNOUNCE_SKIP would otherwise suppress it and prevents same-agent topic-to-topic handoffs from being persisted as fresh requester-side inter_session turns.
