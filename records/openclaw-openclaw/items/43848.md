## Summary

Keep open. Current main has the Telegram threadId pass-through and inter-session provenance mitigations, but the remaining observable problems are still present: ANNOUNCE_SKIP suppresses visible A2A delivery, and normal peer A2A reply steps can still run through the requester session. The remaining work overlaps open #57697 and open #49223, so this needs maintainer coordination rather than cleanup closure.

## What This Changes

The issue asks for Telegram forum-topic sessions_send handoffs to preserve target thread delivery, keep visible target replies when announce returns ANNOUNCE_SKIP, and avoid topic-to-topic requester transcript pollution.

## Best Possible Solution

Keep the shipped threadId and provenance fixes, then define and land one coordinated A2A delivery policy for visible reply fallback after skip tokens and requester-side peer handoffs, with Telegram topic coverage plus shared channel-agnostic tests.
