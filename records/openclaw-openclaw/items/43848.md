## Summary

Keep open. Current main fixes the Telegram forum-topic `threadId` slice and mitigates bare inter-session transcript injection, but the A2A announce path still drops visible delivery on `ANNOUNCE_SKIP` and still runs normal peer ping-pong; the remaining behavior overlaps open #57697 and open #49223.

## What This Changes

The issue requests that `sessions_send` preserve Telegram forum-topic delivery, avoid losing visible handoffs when announce returns `ANNOUNCE_SKIP`, and prevent topic-to-topic requester transcript pollution.

## Best Possible Solution

Keep the shipped `threadId` and provenance mitigations, then land a coordinated A2A change that defines when skip tokens suppress delivery, when a real target reply should be delivered visibly, and when same-agent topic-to-topic handoffs should avoid requester-side injected turns.
