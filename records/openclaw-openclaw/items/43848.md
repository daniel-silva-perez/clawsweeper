## Summary

Keep open. Current main has already fixed the Telegram forum-topic `threadId` loss and has added an inter-session `isUser=false` mitigation, but the A2A path still suppresses visible delivery when the announce step returns `ANNOUNCE_SKIP` and still runs requester-side ping-pong turns for the topic-to-topic flow described here.

## What This Changes

The issue requests that `sessions_send` preserve Telegram forum-topic delivery, keep a visible target-topic post when `ANNOUNCE_SKIP` follows a real target reply, and avoid polluting requester transcripts during topic-to-topic handoffs.

## Best Possible Solution

Keep the shipped `threadId` preservation, then land one coordinated A2A fix that defines the skip-token boundary, reuses a real target reply for visible Telegram topic delivery when appropriate, and prevents same-agent topic-to-topic handoffs from contaminating requester context.
