## Summary

Keep open. Current main preserves Telegram forum-topic `threadId` for `sessions_send` announce delivery and now marks inter-session prompts as `isUser=false`, but it still suppresses visible delivery when the announce step returns `ANNOUNCE_SKIP` and still runs requester-side A2A ping-pong for normal peer sessions. The remaining behavior overlaps open #57697 and open #49223, so this needs maintainer coordination rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to preserve Telegram forum-topic delivery for `sessions_send`, keep visible handoffs when a real target reply is followed by `ANNOUNCE_SKIP`, and avoid injecting same-agent topic handoff traffic into the requester transcript.

## Best Possible Solution

Keep the shipped `threadId` preservation and inter-session provenance mitigation, then land one coordinated A2A change that defines the skip-token boundary, uses a real target reply for visible Telegram topic delivery when appropriate, and prevents same-agent topic-to-topic handoffs from writing unwanted requester-side turns.
