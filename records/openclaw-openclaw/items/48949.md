## Summary

Current main still leaves the reported Feishu tenant_access_token/proxy failure unresolved. Feishu sends use Lark.Client with a wrapper around the SDK default axios instance for timeout/User-Agent only; there is no HTTP proxy bypass, NO_PROXY-aware policy, config/docs surface, or HTTP_PROXY/NO_PROXY regression coverage for tenant token/API calls. The related #26397 fix is adjacent WebSocket proxy work, not an HTTP SDK-token fix, and #57536 is duplicate symptom context rather than implementation evidence.

## Best Possible Solution

Keep this issue open and fix it in the Feishu plugin. Add an explicit HTTP proxy policy in extensions/feishu/src/client.ts for the Lark.Client httpInstance used by token and API calls, preserving timeout/User-Agent behavior and defining whether ambient env proxying, NO_PROXY bypasses, and any explicit channel/account proxy setting are supported. Add focused tests for HTTP_PROXY/HTTPS_PROXY with NO_PROXY on SDK HTTP calls and update Feishu troubleshooting/config docs with the supported workaround or new setting.
