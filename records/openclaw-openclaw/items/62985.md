## Summary

Keep open. Current main still rejects Telegram account-level `agentId` in strict account config, while docs route Telegram accounts through top-level `bindings[]`; #63380 is open, unmerged, and explicitly paired as the implementation candidate for this issue.

## What This Changes

The issue reports an upgrade regression where Telegram multi-account configs using `channels.telegram.accounts.<id>.agentId` fail validation after moving from 2026.4.5 to 2026.4.8.

## Best Possible Solution

Review #63380 as the active implementation candidate. The durable outcome should either restore backward-compatible `channels.telegram.accounts.<id>.agentId` with routing semantics and regression coverage, or deliberately keep `bindings[]` as the only supported account-routing path and improve upgrade docs or doctor output for this migration.
