## Summary

Keep open. Current main still rejects Telegram account-level `agentId` because the strict account schema and exported account type do not include that field, while the documented supported routing path is top-level `bindings[]`. An open unmerged PR, #63380, explicitly uses closing syntax for this issue, so this issue should remain open until maintainers review, land, or reject that implementation candidate.

## What This Changes

The issue reports that Telegram multi-account configs using `channels.telegram.accounts.<id>.agentId` are rejected after upgrading from 2026.4.5 to 2026.4.8.

## Best Possible Solution

Review #63380 as the active implementation candidate. The durable fix should either accept and correctly route or migrate legacy `channels.telegram.accounts.<id>.agentId`, with regression coverage for the reported multi-account config, or explicitly keep `bindings[]` as the only supported path and improve upgrade docs/doctor diagnostics so users can find the migration.
