## Summary

Keep this issue open. Current main still does not accept or route `channels.telegram.accounts.<id>.agentId`, and PR #63380 remains open with closing syntax for this issue, so it is the active implementation candidate. The workaround comment correctly points users to top-level `bindings[]`, but that does not by itself resolve the reported upgrade regression for old account-level configs.

## Best Possible Solution

Keep this issue open and review PR #63380 as the implementation candidate. Maintainers should either land a focused backward-compatibility fix that accepts `channels.telegram.accounts.<id>.agentId`, routes it only when no higher-priority binding or topic override applies, and adds regression coverage for the reported multi-account config, or explicitly choose `bindings[]` as the only supported path and improve upgrade guidance or doctor diagnostics for old configs.
