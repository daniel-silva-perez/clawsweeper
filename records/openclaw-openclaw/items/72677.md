## Summary

Keep open. Current main still lacks the cron ghost-run warning, configurable threshold, and run-log warning persistence that this PR is meant to add, so the PR is not implemented on main. It should remain in maintainer review because the branch also mixes non-cron plugin SDK/package E2E changes and the related deeper payload-handoff report (#73189) remains a separate product decision point.

## What This Changes

The PR adds a configurable warning for fast ok main-session systemEvent cron handoffs using wakeMode next-heartbeat, persists warning codes in cron run logs, updates cron config schema/help/tests, and also changes package E2E helper plus provider SDK validation surfaces.

## Best Possible Solution
