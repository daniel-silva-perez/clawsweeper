## Summary

Keep this PR open. The prior ClawSweeper review still matches current main at 189c91e: Xiaomi image understanding is mostly covered by the generic image-capable provider path, but Xiaomi video understanding is not implemented, and video support is part of the PR's stated purpose. The PR is useful but needs a rebase/rework for the current plugin entrypoint, manifest contract metadata, SDK helper exports, and provider HTTP policy path.

## Best Possible Solution

Keep the PR open and ask for a rebase/rework. The best path is a current Xiaomi media-understanding provider in the Xiaomi bundled plugin that declares manifest mediaUnderstandingProviders metadata, registers through the modern plugin entrypoint, chooses mimo-v2-omni for Xiaomi image/video media understanding instead of the text-only default, uses the shared OpenAI-compatible video helpers plus provider-http policy helpers, and adds focused tests for Xiaomi default image and video routing.
