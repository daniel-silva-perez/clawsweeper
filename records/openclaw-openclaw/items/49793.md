## Summary

Keep this PR open. Current main still does not implement Xiaomi video media understanding, while the PR remains a relevant contribution but is stale against the current manifest-first provider shape, SDK exports, provider HTTP policy helpers, and test expectations.

## Best Possible Solution

Keep this PR open and ask for a rework on current main. The best patch should add a Xiaomi media-understanding provider under extensions/xiaomi, declare contracts.mediaUnderstandingProviders and mediaUnderstandingProviderMetadata with image/video defaults to mimo-v2-omni, use the shared OpenAI-compatible video helpers plus openclaw/plugin-sdk/provider-http policy helpers, and add focused Xiaomi tests for registration, defaults, request shape, and transport-policy forwarding.
