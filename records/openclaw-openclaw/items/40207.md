## Summary

Keep open. Current main still routes macOS LaunchAgent stdout/stderr paths through the OpenClaw state directory, and the issue has two independent confirmations that launchd exits with EX_CONFIG when that path is symlinked or external. Related #46153 describes a distinct launchd startup hang and does not supersede this log-path crash.

## What This Changes

Request is to make macOS Gateway LaunchAgent log paths launchd-safe when the OpenClaw state directory is symlinked or on an external volume.

## Best Possible Solution

OpenClaw should generate macOS LaunchAgent StandardOutPath and StandardErrorPath values that do not depend on a symlinked OpenClaw state directory, then update status hints, diagnostics, tests, docs, and changelog entries to match the chosen LaunchAgent log destination.

## Reproduction Assessment

Yes. The issue gives concrete macOS reproduction steps with a symlinked home .openclaw, openclaw gateway install, and launchd exit-code checks; current source inspection shows the generated plist still uses state-dir log paths.

## Solution Assessment

Yes. A focused macOS LaunchAgent log-path fix is the narrowest maintainable solution; broad Gateway runtime changes or general state-dir migration are not needed for this report.

## Repair Work Prompt

Fix the macOS Gateway LaunchAgent log-path crash tracked by https://github.com/openclaw/openclaw/issues/40207. Current main writes LaunchAgent StandardOutPath/StandardErrorPath under resolveGatewayStateDir(env)/logs, which fails under launchd when ~/.openclaw or OPENCLAW_STATE_DIR is symlinked to an external/non-standard volume. Make the macOS LaunchAgent stdout/stderr destination launchd-safe, preferably a local user log path such as ~/Library/Logs/openclaw while preserving normal OpenClaw state/config storage and non-macOS restart-log behavior. Do not conflate this with #46153's separate launchd module-evaluation hang or #30401's logging.file tilde-expansion path.

Expected scope: update the resolver or add a LaunchAgent-specific resolver in src/daemon/restart-logs.ts, use it from src/daemon/launchd.ts install and restart-rewrite paths, keep src/daemon/launchd-plist.ts generic unless needed, and update runtime/status/diagnosis readers so hints and log tails point to the right macOS LaunchAgent stdout/stderr files. Update focused tests and docs; add a user-facing changelog Fixes entry crediting @dhruvkelawala and relevant confirmation commenters such as @crackmac/@rnrnstar2. Avoid broad Gateway startup refactors, state-dir migrations, dependency changes, or changes to launchd auth/env handling.
