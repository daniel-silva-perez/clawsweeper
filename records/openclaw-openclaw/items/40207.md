## Summary

Keep open. Current main still generates macOS LaunchAgent stdout/stderr paths from the OpenClaw state directory during install and restart rewrite, and the latest release contains the same behavior. The issue has two independent confirmations plus a working workaround, so the reported launchd `EX_CONFIG` path remains actionable.

## What This Changes

The issue asks OpenClaw to stop writing macOS LaunchAgent `StandardOutPath` and `StandardErrorPath` under symlinked or external-volume OpenClaw state directories that can make launchd exit with `EX_CONFIG`.

## Best Possible Solution

Add a macOS launchd-specific supervisor log path that does not depend on the OpenClaw state directory symlink, such as `$HOME/Library/Logs/openclaw`, or canonicalize the plist stdout/stderr targets before writing. Keep state/config directory semantics intact, preserve non-macOS restart-log behavior, and align status hints plus docs with the new macOS LaunchAgent log destination.
