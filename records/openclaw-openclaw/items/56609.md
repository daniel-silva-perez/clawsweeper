## Summary

Keep open. Current main still uses `.jsonl.reset.*` and `.jsonl.deleted.*` transcript archive names, and session maintenance can remove those archive artifacts; the related usage/search visibility fixes do not satisfy this issue's retention and naming request.

## What This Changes

The issue requests replacing session transcript `.deleted.*` and `.reset.*` archive naming with a safer `.archived.*` convention or archive directory, and avoiding hard deletion of session files.

## Best Possible Solution

Maintainers should define the transcript retention contract first, then implement it compatibly: keep recognizing legacy archive names, write the chosen new archive convention going forward, make any destructive cleanup explicit and documented, and update tests/docs across reset, delete, usage, memory indexing, restore, and maintenance surfaces.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists: reset passes `reason: "reset"`, delete passes `reason: "deleted"`, the helper appends that reason into the filename, and tests assert those suffixes while maintenance tests assert old archives are removed.

## Solution Assessment

Unclear. The requested direction is plausible for data preservation, but it is not the narrowest safe mechanical fix until maintainers choose the intended retention and disk-budget policy.
