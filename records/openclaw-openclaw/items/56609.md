## Summary

Keep open. Current main still writes `.jsonl.reset.*` and `.jsonl.deleted.*` transcript archives, and documented maintenance paths can remove old archive artifacts; the related archive visibility fixes help usage/search/indexing but do not satisfy the naming and never-delete retention request.

## What This Changes

The issue requests a safer permanent archive convention for session JSONL files, replacing `.deleted.*` and `.reset.*` naming with `.archived.*` or a `sessions/archive/` store and avoiding hard deletion of session transcripts.

## Best Possible Solution

Maintainers should first set the intended transcript retention contract, then implement it compatibly: keep recognizing legacy `.deleted.*` and `.reset.*` files everywhere, write the chosen `.archived.*` or archive-directory convention going forward, make any hard deletion explicit and clearly documented, and update tests/docs around reset, delete, usage, memory indexing, history, and maintenance.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists: `sessions.reset` passes `reason: "reset"`, `sessions.delete` passes `reason: "deleted"`, the archive helper appends that reason to the filename, and tests assert those suffixes while maintenance tests assert old archives are removed.

## Solution Assessment

Unclear. The requested direction is plausible for data preservation, but it is not a narrow mechanical fix because it changes persistent file naming, compatibility expectations, and the current retention/disk-budget cleanup contract.
