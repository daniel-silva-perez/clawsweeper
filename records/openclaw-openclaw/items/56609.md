## Summary

Keep open. Current main still writes `.jsonl.reset.*` for `/new` and `/reset`, writes `.jsonl.deleted.*` for session deletion, and contains documented maintenance paths that can remove archived session artifacts. The related archive-indexing fixes addressed search/usage visibility for the legacy suffixes, but they did not resolve this naming and retention policy request.

## What This Changes

Request to stop using `.deleted.*` and `.reset.*` session transcript artifact names, prefer archival naming or an archive directory, and avoid deleting session transcript files.

## Best Possible Solution

Decide the intended retention contract first, then implement a backwards-compatible archive convention: keep reading and indexing legacy `.deleted.*`/`.reset.*` files, write the chosen `.archived.*` or `sessions/archive/` form going forward, and update maintenance defaults/docs/tests so destructive cleanup is explicit and clearly named.

## Reproduction Assessment

Yes. A code-level reproduction is high-confidence: current tests and source show `sessions.reset` creates `.jsonl.reset.*`, `sessions.delete` creates `.jsonl.deleted.*`, and enforced maintenance can remove old archived transcript artifacts.

## Solution Assessment

Unclear. The requested direction is plausible, but the best fix needs maintainer policy judgment because it changes persistent file naming, backwards compatibility expectations, and the existing session-retention cleanup contract.
