## Summary

Keep this PR open. Current main still does not implement the requested global `agents.defaults.reasoningDefault` config surface: the global defaults schema/type/generated metadata omit the key, docs describe only per-agent `agents.list[].reasoningDefault`, and reply/status resolution still ignore a global reasoning default. The PR remains an active implementation candidate with recent contributor follow-up and prior maintainer discussion saying intended behavior was still being evaluated.

## Best Possible Solution

Keep this PR open for maintainer review. A mergeable path should settle the product/security policy, then implement `agents.defaults.reasoningDefault` consistently across config schema/types/generated metadata/docs, reply directive resolution, current-level status, direct `/status`, status text, and focused precedence tests. If maintainers reject the feature, close it with that explicit product decision rather than as stale or implemented.
