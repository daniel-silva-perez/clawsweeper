## Summary

Keep this PR open. Current main at 6d323ee73640afdf02ed9fa3aded13da23359b98 still does not implement the requested `agents.defaults.reasoningDefault` surface: the global defaults schema/type omit the key, generated metadata and docs only expose per-agent `agents.list[].reasoningDefault`, and reply/status resolution paths still read per-agent reasoning defaults but not the global default. The PR is active, addresses a closed companion feature request, and prior maintainer discussion explicitly left the feature open while intended behavior is evaluated.

## Best Possible Solution

Keep this PR open for maintainer review. A mergeable path should settle the product and authorization policy, then implement `agents.defaults.reasoningDefault` consistently across `AgentDefaultsSchema`, `AgentDefaultsConfig`, generated config metadata/help/labels, public docs, reply directive resolution, current-level status, direct `/status`, and status text. It should preserve explicit `off` over model-derived reasoning defaults and include focused tests for directive, session, per-agent, global, model-default, authorized sender, and unauthorized sender precedence. If maintainers reject the feature, close it with that explicit product decision rather than as stale or implemented.
