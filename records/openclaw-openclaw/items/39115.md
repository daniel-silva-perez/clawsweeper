## Summary

Keep this PR open. Current main at 17811480dae20f65b526454d75e971ae8424ec10 still does not implement the requested global `agents.defaults.reasoningDefault` behavior: the strict defaults schema and exported defaults type omit the key, reply/status resolution only honors session or per-agent reasoning defaults, and docs still describe `reasoningDefault` as per-agent only. The PR also has recent contributor follow-up and prior maintainer product-evaluation signal, so it remains an active implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open for maintainer review. A mergeable solution should add `agents.defaults.reasoningDefault` consistently across schema, exported types, generated schema/help metadata, docs, reply resolution, current-level status, direct status, and resolved agent defaults; preserve explicit `off` over model-derived reasoning defaults; and include tests for directive, session, per-agent, global, model-default precedence, plus sender authorization. If maintainers reject the product behavior, close it with that explicit product decision rather than as stale or implemented.
