## Summary

Keep this PR open. Current main still does not implement the intended global `agents.defaults.reasoningDefault` config surface: the global defaults schema/type/generated metadata omit it, docs only describe per-agent `agents.list[].reasoningDefault`, and reply/status resolution still read only session or per-agent reasoning defaults. The PR remains an active implementation candidate, with recent contributor follow-up and prior maintainer discussion saying the intended behavior was still being evaluated.

## Best Possible Solution

Keep this PR open for maintainer review. A mergeable path should settle the product/security policy, then implement `agents.defaults.reasoningDefault` consistently across `AgentDefaultsSchema`, `AgentDefaultsConfig`, generated config metadata/help/labels, public docs, reply directive resolution, current-level status, direct `/status`, and status text. It should preserve explicit `off` over model-derived defaults and include focused tests for directive, session, per-agent, global, model-default, authorized sender, and unauthorized sender precedence. If maintainers reject the feature, close it with that explicit product decision rather than as stale or implemented.
