## Summary

Keep this PR open. Current main still does not implement the requested global `agents.defaults.reasoningDefault` contract: the global schema/type/generated metadata omit the key, docs only describe per-agent reasoning defaults, and reply/status resolution still falls through session/per-agent state before `off` or model defaults. The PR remains the active implementation candidate, but the latest PR branch also has a real security-policy question around persisted session reasoning levels for unauthorized senders that needs maintainer handling before merge.

## Best Possible Solution

Keep this PR open for maintainer review. If the product/security policy is accepted, land a version that consistently adds `agents.defaults.reasoningDefault` across schema, exported types, generated config metadata, docs, reply directive resolution, current-level status, direct `/status`, status text, and focused precedence/authorization tests. Before merge, maintainers should explicitly resolve the persisted-session reasoning authorization question, either by gating session reasoning for unauthorized senders or documenting why that state is trusted for the affected session scopes.
