## Summary

Keep open. Current main still routes compaction through one resolved auth profile, while normal agent turns have ordered same-provider auth-profile candidates, cooldown skipping, and profile advancement. The discussion’s idempotency point remains important because compaction can mutate transcripts, hooks, checkpoints, and session state, so this needs maintainer design review before implementation.

## What This Changes

Compaction should retry or fail over across eligible auth profiles for the same provider when the selected compaction profile is rate-limited or unavailable.

## Best Possible Solution

Add shared compaction retry orchestration that captures one transcript/checkpoint snapshot, orders eligible same-provider auth profiles with the normal auth semantics, retries only failover-classified failures, updates cooldown state consistently, and commits a successful summary plus side effects at most once across direct and context-engine compaction paths.
