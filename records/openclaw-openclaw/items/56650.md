## Summary

Keep this PR open. Current main still lacks the generic `models auth paste-token --token`/stdin/env workflow and still routes GitHub Copilot runtime auth through the token-exchange helper, so the intended change is not implemented on main. The branch is also stale against current main's moved Copilot token helper and newer non-interactive Copilot onboarding path, and it needs auth/security owner review before merge.

## What This Changes

The PR adds a non-interactive token path to `models auth paste-token` and adds GitHub Copilot PAT detection that bypasses token exchange and uses the enterprise Copilot API, with related unit tests.

## Best Possible Solution

Rework this PR against current main: keep the existing non-interactive onboarding path, decide whether a generic paste-token non-interactive surface is still wanted, put PAT bypass logic in the active shared Copilot token helper used by the plugin runtime, preserve current provider-specific token validation, and add focused tests/docs for the accepted automation paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path for the missing `paste-token --token` support and for current Copilot PATs still entering the `/copilot_internal/v2/token` exchange path; the live GitHub 403 itself was not re-run with a real PAT.

## Solution Assessment

Unclear as-is. The PR targets the right user problem, but the best current-main solution needs a rebase into `src/plugin-sdk/provider-auth.ts`/`extensions/github-copilot` and should use the existing runtime request-header seam or existing Copilot dynamic headers with tests.
