## Summary

This PR should stay open because current main still lacks the requested generic `paste-token --token`/stdin/env path and still sends GitHub tokens through the Copilot exchange resolver, but the proposed patch is not ready to merge against current main because it targets stale Copilot auth seams and needs credential-handling review.

## What This Changes

The PR adds non-interactive token inputs to `models auth paste-token` and adds GitHub Copilot PAT detection that skips token exchange and uses the enterprise Copilot API.

## Best Possible Solution

Rebase and rework the change on current main: decide whether generic `paste-token` should gain a non-interactive input surface in addition to onboarding, preserve provider-specific token validation for every source, implement any Copilot PAT direct-auth behavior in the active shared resolver/facade used by the plugin runtime, and use the existing runtime request/header seams for Copilot request metadata.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current `paste-token` has no registered `--token` option and current Copilot runtime auth still calls the token exchange resolver for every GitHub token; only the reporter's live 403 was not re-run.

## Solution Assessment

No. The PR addresses the right automation problem, but as proposed it is not the best current-main fix because it targets an older Copilot token helper and does not route all paste-token inputs through the existing validation path.
