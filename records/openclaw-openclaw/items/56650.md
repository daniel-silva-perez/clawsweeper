## Summary

Keep this PR open for maintainer review: the user problem is still real on current main, but the branch is stale against the active auth architecture and introduces blocking correctness/security review items before it can merge.

## What This Changes

The PR adds non-interactive token sources to `models auth paste-token` and adds GitHub Copilot PAT detection that skips Copilot token exchange and uses the enterprise Copilot API.

## Best Possible Solution

Rebase and rework the branch against current main: decide whether generic `paste-token` should gain a non-interactive surface in addition to onboarding, route every token source through the same provider-specific validation, implement any Copilot PAT bypass in the active shared resolver/facade used by the plugin runtime, use the current runtime request-header or Copilot dynamic-header paths where needed, and add focused tests/docs for the accepted automation path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path for missing `paste-token --token` support and for PATs still entering the current `/copilot_internal/v2/token` exchange path; only the live GitHub 403 was not re-run.

## Solution Assessment

No, not as currently proposed. The PR addresses the right user problem, but the patch needs a current-main rebase and should target the active provider-auth/Copilot runtime seams while preserving existing validation and credential-safety behavior.
