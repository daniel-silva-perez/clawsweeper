## Summary

Keep this PR open. Current main has already moved the affected subagent gateway calls away from the old 10s floor by using 60s control-call timeouts and a run-timeout-aware child-agent request timeout, but it does not implement the PR’s intended configurable `agents.defaults.subagents.gatewayTimeoutMs` setting. Because this is a PR and the specific intended config surface is still absent, it is not safe to close as implemented.

## Best Possible Solution

Keep the PR open for maintainer review, but do not merge it as-is. If a configurable subagent gateway timeout is still desired after current main’s 60s/run-timeout-aware mitigation, rebase the PR onto current main, preserve the existing 60s default floor and run-timeout-aware spawn behavior, add `gatewayTimeoutMs` to types/Zod/generated schema/docs, and add focused tests for default, configured-higher, and configured-lower-than-default behavior.
