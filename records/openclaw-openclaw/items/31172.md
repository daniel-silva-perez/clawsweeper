## Summary

Keep this PR open. Current main still does not implement configurable Feishu streaming-card throttling: the runtime uses a fixed 160 ms constant, the dispatcher does not read or pass a config value, and the schema, manifest, docs, and tests omit `streamingThrottleMs`. The PR is a plausible focused fix, but it needs Feishu maintainer review because its 100 ms default no longer matches current main's fixed 160 ms behavior and the diff does not update manifest/docs/tests.

## Best Possible Solution

Keep this PR open for Feishu plugin-owner review. A mergeable version should add `streamingThrottleMs` to the shared Feishu config schema and plugin manifest metadata, preserve top-level/account merge semantics, pass the resolved value into `FeishuStreamingSession`, document it on https://docs.openclaw.ai/channels/feishu, and add tests for default behavior plus configured throttle plumbing. Maintainers should decide whether the default follows current main's 160 ms behavior or the PR's older 100 ms compatibility argument.
