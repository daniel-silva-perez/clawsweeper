## Summary

Keep this PR open. Current main at 43da08979035503401a21a06a55e66920411ad67 still does not implement a configurable Feishu `streamingThrottleMs`: the streaming session uses a fixed 160 ms throttle, the dispatcher does not read or pass a config value, the strict Feishu schemas omit the key, and the manifest/docs/tests do not expose it. The linked #31136 was stale-closed rather than shipped, and the PR author recently reiterated that the setting is needed for Feishu channel stability. The PR is a plausible implementation candidate, but not merge-ready as-is because current main now has deliberate 160 ms throttle behavior and the PR still needs manifest/docs/test coverage plus a default decision.

## Best Possible Solution

Keep this PR open for Feishu plugin-owner review. A mergeable patch should add `streamingThrottleMs` to the shared Feishu config schema and manifest metadata, preserve top-level/account merge semantics, pass the resolved value into `FeishuStreamingSession`, document the option on the public Feishu channel page, and add tests for default behavior plus configured throttle plumbing. Maintainers should explicitly choose whether the default follows current main's 160 ms behavior or the PR's older 100 ms compatibility argument.
