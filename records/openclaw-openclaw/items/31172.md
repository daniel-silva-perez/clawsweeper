## Summary

Keep this PR open. Current main still does not implement a configurable Feishu `streamingThrottleMs`: the streaming-card runtime uses a fixed 160 ms throttle, the dispatcher does not read or pass a config value, the strict Feishu config schemas and plugin manifest omit the key, and the public Feishu docs/tests only cover the existing fixed behavior. The PR is plausible but needs Feishu-owner review because its 100 ms default no longer matches current main's deliberate 160 ms throttle behavior, and it still needs schema, manifest, docs, and regression coverage before it is mergeable.

## Best Possible Solution

Keep this PR open for Feishu plugin-owner review. A mergeable version should add `streamingThrottleMs` to the shared Feishu config schema and manifest metadata, preserve top-level/account merge semantics, pass the resolved value into `FeishuStreamingSession`, document it on the public Feishu channel page, and add tests for default behavior plus configured throttle plumbing. Maintainers should explicitly choose whether the default follows current main's 160 ms behavior or the PR's older 100 ms compatibility argument.
