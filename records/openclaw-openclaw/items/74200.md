## Summary

Keep this PR open. Current main still performs the wholesale `targetWorkspace/skills` wipe that the PR is trying to change, so the PR is not obsolete. However, the proposed non-destructive sync changes a security-sensitive skills staging boundary and needs maintainer review because preserving target directories outside the current manifest can leave stale or no-longer-allowed skills in sandbox workspaces after filters, config gates, or agent allowlists change.

## Best Possible Solution

Keep this PR open for maintainer review. A safe implementation should prevent destructive wipes on transient or failed source loads while still making the target reflect the effective allowed skills set. Consider an owned-sync manifest or marker strategy, full-tree freshness/copy semantics, and regression tests for partial load, filter removal, agent allowlist changes, disabled/config-gated skills, remote eligibility changes, user-created target directories if intentionally supported, and deep nested skill assets. Review it alongside #50590 and #48034 because the same sandbox skill staging contract is involved.
