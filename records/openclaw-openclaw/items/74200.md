## Summary

Keep this PR open. Current main still removes the entire sandbox target skills directory before copying eligible skills, so the PR is not obsolete; however, the proposed incremental sync is not merge-ready because it can retain deleted files and preserve stale or no-longer-eligible skill directories inside sandbox workspaces.

## What This Changes

The PR changes `syncSkillsToWorkspace` from wholesale target skills directory deletion to incremental per-skill syncing with mtime-based skip logic, plus regression tests for preserving existing target skills and recopying changed skills.

## Best Possible Solution

Keep this PR open for maintainer review. A safe fix should prevent wholesale target deletion on transient source-load failures while still giving sandbox workspaces a clear owned-sync contract for eligible skills, including cleanup semantics for removed, disabled, filtered, or no-longer-eligible skills. An owned manifest or marker-based strategy plus full-tree copy freshness is likely safer than a newest-mtime heuristic.
