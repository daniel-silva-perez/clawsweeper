## Summary

Keep this issue open. Current main still accepts `channels.googlechat.replyToMode: "off"`, but Google Chat can still attach implicit thread IDs through the typing-placeholder path and routed outbound `sendText`/`sendMedia` sends. The related open PR #67055 is useful implementation context, but it is not merged and appears monitor-focused, so main does not yet solve all affected send paths.

## Best Possible Solution

Keep this issue open until a Google Chat fix lands on main. The fix should gate implicit Google Chat thread propagation on the resolved account `replyToMode` across inbound context, typing placeholder creation, monitor text/media delivery, and routed outbound `sendText`/`sendMedia`, while preserving intentional explicit thread targets. PR #67055 or an equivalent maintainer patch is the likely review path, with focused regression tests for off-mode typing, text, media, and routed outbound sends.
