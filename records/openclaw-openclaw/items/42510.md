## Summary

Keep this issue open. Current main still accepts `channels.googlechat.replyToMode: "off"`, but Google Chat can still propagate thread IDs through the inbound typing placeholder path and through routed outbound text/media sends. The related open PR #67055 is useful implementation context, but the fix is not on main and should be reviewed against all affected send paths before this issue closes.

## Best Possible Solution

Keep this issue open until a Google Chat fix lands on main. The fix should gate implicit Google Chat thread propagation on the resolved `replyToMode` across inbound context, typing placeholder creation, final text/media delivery, and routed outbound `sendText`/`sendMedia`, while preserving intentional explicit thread targets. PR #67055 or an equivalent maintainer patch is the likely review path, with focused regression tests for off-mode typing, text, media, and routed outbound sends.
