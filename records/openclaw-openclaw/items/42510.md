## Summary

Keep this issue open. Current main still accepts Google Chat `replyToMode`, but the Google Chat monitor and outbound adapter still carry thread identifiers through paths that do not consult the resolved `replyToMode: "off"`. The older fix PRs are closed unmerged, and the open related PR is still an implementation candidate rather than shipped behavior.

## Best Possible Solution

Keep this issue open until a Google Chat plugin fix lands on main. The fix should honor `replyToMode: "off"` across inbound context, typing placeholders, final text/media delivery, and routed outbound `sendText`/`sendMedia`, while preserving deliberate explicit thread targets. Review #67055 or land an equivalent maintainer patch with focused regression coverage for typing, text, media, and routed outbound paths.
