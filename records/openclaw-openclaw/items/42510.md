## Summary

Keep this issue open. Current main still accepts `channels.googlechat.replyToMode: "off"`, but Google Chat continues to propagate inbound thread IDs through the monitor typing path and routed outbound text/media sends. The related open PR #67055 is still an implementation candidate, not shipped behavior.

## Best Possible Solution

Keep this issue open until a Google Chat fix lands on main. The fix should gate implicit Google Chat thread propagation on the resolved `replyToMode` across inbound context, typing placeholder creation, final text/media delivery, and routed outbound `sendText`/`sendMedia`, with focused regression tests for off-mode typing, text, media, and routed outbound paths. PR #67055 or an equivalent maintainer patch is the likely review path.

## Risks / Open Questions

- Closing this issue now would leave Google Chat users with threaded replies despite `replyToMode: "off"` in the inbound typing-placeholder path and routed outbound text/media path.
- A monitor-only fix could miss routed outbound `sendText` and `sendMedia`; an outbound-only fix could miss the typing-placeholder update path.
- The eventual fix should preserve intentional explicit Google Chat thread targets while suppressing implicit inbound thread propagation when `replyToMode` is `off`.
