## Summary

Keep this issue open. Current main still lacks a SKILL.md or plugin-skill metadata contract for per-skill model routing, and the linked implementation candidate #58142 is still open rather than merged. The issue discussion also has fresh concrete validation from InvestorClaw, so this is not stale cleanup material.

## Best Possible Solution

Keep this issue open as the canonical feature request. Review, revise, land, or supersede #58142 with a documented SKILL.md and plugin-skill metadata contract, alias/allowlist enforcement through the existing model-selection APIs, explicit precedence and fallback behavior, and focused parser, snapshot, prompt, slash-invocation, runtime-routing, and security tests.
