## Summary

Keep this issue open. Current main still does not implement a SKILL.md or plugin-skill metadata contract for per-skill model routing, and the related implementation candidate #58142 is still open rather than merged. The fresh InvestorClaw and KinthAI comments provide concrete production motivation, so this is actionable feature work, not stale cleanup.

## Best Possible Solution

Keep this issue open as the canonical feature request. Review, revise, land, or supersede #58142 with a documented SKILL.md and plugin-skill metadata contract, model alias and allowlist enforcement through the existing model-selection APIs, explicit precedence and fallback behavior, and focused parser, snapshot, prompt, slash-invocation, runtime-routing, and security tests.
