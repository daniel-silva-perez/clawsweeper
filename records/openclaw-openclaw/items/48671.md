## Summary

Keep this PR open. Current main still builds the Control UI chat session dropdown from persisted session rows plus the active session key only, so configured agents with no stored session rows are still missing. The provided thread context also shows recent maintainer repair activity on this branch and `clownfish:merge-ready`, making this PR an active implementation candidate rather than cleanup to close.

## Best Possible Solution

Keep this PR open and proceed with maintainer review or merge of the repaired branch. The right fix belongs in the Control UI session dropdown path: combine `sessions.list` rows with `agents.list`, synthesize normalized main-session keys through the existing session-key helpers, avoid global-scope per-agent switch targets, and keep the added regression tests.
