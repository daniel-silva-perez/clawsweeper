## Summary

Keep open. Current main still builds the WebChat session dropdown from stored session rows plus the active session key, so configured agents without stored sessions are not synthesized into the dropdown. The PR remains the active implementation candidate, with related open tracker context and maintainer repair pushes on the branch.

## Best Possible Solution

Keep this PR open for maintainer review or landing. The best implementation should update `resolveSessionOptionGroups` to merge `sessions.list` rows with `agents.list`, synthesize normalized per-agent main-session keys using the configured `mainKey`, avoid per-agent switch targets for global-scope sessions, normalize dedupe and row lookup, and keep focused regression tests plus changelog credit.
