## Summary

Keep this PR open. Current main still builds the WebChat session dropdown from stored session rows plus the active session key; it does not synthesize options from `agents.list`, so configured agents without session rows remain missing. The related tracker #41933 is still open, prior PR #41955 is closed unmerged, and this PR remains the active implementation candidate with maintainer repair pushes in the provided context.

## Best Possible Solution

Keep this PR open for maintainer review or landing. The fix should live in `ui/src/ui/chat/session-controls.ts`: merge `sessions.list` rows with `agents.list`, synthesize normalized per-agent main-session keys using the configured `mainKey`, skip per-agent switch targets for global-scope sessions, normalize dedupe and row lookup, and keep focused regression tests plus changelog credit.
