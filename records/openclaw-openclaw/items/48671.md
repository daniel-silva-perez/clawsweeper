## Summary

Keep this PR open. Current main at 0dcab4e34760cac5d9cd02649d4b3a5c2f7e5712 still builds the chat session dropdown from stored session rows plus the active session key only; it does not synthesize options from `agents.list`, and the PR remains the active implementation candidate for the configured-agent dropdown bug.

## Best Possible Solution

Keep this PR open and continue maintainer review or merge of the repaired branch. The best fix is in the Control UI session dropdown path: combine `sessions.list` rows with `agents.list`, synthesize normalized per-agent main-session keys via the existing session-key helpers, avoid per-agent switch targets for global-scope sessions, and keep the added regression tests.
