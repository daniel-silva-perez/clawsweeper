## Summary

Keep open for maintainer review. Current main has landed a few narrower overlapping fixes, but this broad PR still contains unmerged hardening work and is not safe to close as implemented; the proposed patch also introduces concrete regressions in media API compatibility, log streaming, session archive lookup freshness, local shell semantics, and SecretRef exec environment handling.

## What This Changes

The PR branch applies broad security, memory-safety, and performance hardening across media loading, subprocess/SecretRef execution, SQL schema construction, global cache bounds, channel bridge event handling, stream writing, PDF extraction, and related tests.

## Best Possible Solution

Refresh the work as smaller owner-scoped PRs or a maintainer-curated replacement: keep the focused fixes already on main, preserve existing TUI shell, log streaming, media API, SecretRef exec, and session archive contracts, and port only still-valid hardening with targeted tests and security review.

## Reproduction Assessment

Yes. The blocking regressions are reproducible from source and focused tests: `localRoots: "any"` remains in current callers/tests, `stream.write(false)` can be mocked to show log truncation, TUI `!` commands depend on shell semantics, and session archive tests rely on fresh directory reads after reset/deleted transcript churn.

## Solution Assessment

No. This broad stale patch is not the best way to solve the hardening work because it changes several established contracts at once; narrower current-main patches with explicit compatibility and security tests would be safer and easier to land.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve `localRoots: "any"` compatibility:** `src/media/web-media.ts:35`
  - body: This removes `"any"` from the `loadWebMedia` option type, but current main still has tests and production callers, including WhatsApp outbound media, that pass `localRoots: "any"` with a `readFile` override. Preserve the compatibility type or migrate every caller in the same PR, otherwise the affected lanes will fail typecheck or lose the intended bypass behavior.
  - confidence: 0.9
- **[P1] Treat backpressure separately from writer failure:** `src/terminal/stream-writer.ts:49-50`
  - body: `stream.write()` returning false is normal backpressure, not a closed stream. Since `openclaw logs --follow` exits its output loop when the safe writer returns false, this change can truncate healthy high-volume log streams instead of waiting for drain or preserving the old success semantics.
  - confidence: 0.93
- **[P2] Read session directories fresh for archive fallback:** `src/infra/session-cost-usage.ts:344`
  - body: `resolveExistingUsageSessionFile` uses this listing to fall back from a missing active JSONL to reset/deleted archives. Caching the directory for five seconds can return a stale active filename after rotation or deletion, causing usage detail and log lookups to miss until the TTL expires.
  - confidence: 0.86
- **[P2] Preserve local shell command semantics:** `src/tui/tui-local-shell.ts:116-117`
  - body: Local `!` commands are documented and implemented as fresh shell commands after explicit in-session approval. Parsing into argv with `shell: false` treats pipes, redirects, globs, and `$VAR` expansion as literal arguments, so existing operator workflows silently misbehave.
  - confidence: 0.82
