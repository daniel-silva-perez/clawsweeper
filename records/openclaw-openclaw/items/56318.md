## Summary

Keep open and do not merge as-is. The discussion already identified meaningful hardening ideas, and current main has independently landed at least the cost-usage cache bound, but the PR is stale against current main and still introduces blocking type/behavior regressions across media loading, log streaming, session archive lookup, and local TUI shell semantics.

## What This Changes

The PR branch applies broad security, memory, and performance hardening across media loading, subprocess/secret resolver execution, SQL schema construction, global cache bounds, channel bridge event handling, stream writing, PDF extraction, and related tests.

## Best Possible Solution

Refresh this as smaller, owner-scoped PRs or a maintainer-curated replacement: preserve existing TUI, log streaming, media API, secret resolver, and session archive contracts; keep the narrow hardening already on main; and port only still-valid fixes with focused regression tests.

## Reproduction Assessment

Yes. The blocking paths are reproducible from source: current callers still pass localRoots: "any" to loadWebMedia, log callers stop when the writer returns false, TUI shell operators depend on shell: true, and session archive fallback depends on a fresh directory listing after transcript rotation or deletion.

## Solution Assessment

No. A stale bundled hardening PR is not the best solution because it changes several established contracts at once; smaller current-main fixes with explicit tests and security review would be safer and easier to validate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep localRoots any callers compiling:** `src/media/web-media.ts:35`
  - body: This narrows loadWebMedia's option type to remove localRoots: "any", but current helpers and the WhatsApp runtime still pass that value into loadWebMedia. The PR needs to either preserve the compatibility type or migrate all call sites, otherwise the extension/core typecheck will fail.
  - confidence: 0.9
- **[P1] Do not stop log output on backpressure:** `src/terminal/stream-writer.ts:49-50`
  - body: stream.write() returning false is normal backpressure, not a terminal failure. Since openclaw logs exits when the writer returns false, this change can truncate healthy high-volume log streams instead of waiting for drain.
  - confidence: 0.93
- **[P2] Read session directories fresh when resolving archives:** `src/infra/session-cost-usage.ts:344`
  - body: resolveExistingUsageSessionFile uses this listing to fall back from a missing primary JSONL to reset/deleted archives. Caching it for 5 seconds can return a stale primary path after rotation or reset, making detail/log lookups miss until the TTL expires.
  - confidence: 0.86
- **[P2] Preserve local shell command semantics:** `src/tui/tui-local-shell.ts:116-117`
  - body: Local ! commands are documented and implemented as approved non-interactive shell commands. Parsing into argv with shell: false passes pipes, redirects, globs, and $VAR expansion as literal arguments, so existing operator workflows silently misbehave.
  - confidence: 0.82
