## Summary

Keep open for maintainer review. Current main already contains narrower fixes for some slices of this broad hardening PR, but not the full intended change; the PR also introduces concrete regressions in log streaming, session archive resolution, and local TUI shell behavior, so it should not be merged as-is.

## What This Changes

The PR branch applies broad security, memory, and performance hardening across subprocess handling, secret resolver environment construction, media fetching and path access, SQL schema construction, global cache bounds, channel bridge event handling, stream writing, PDF extraction, and related tests.

## Best Possible Solution

Refresh this into smaller owner-scoped PRs or a maintainer-curated replacement: keep the narrower fixes already on main, drop or repair the regressions, preserve documented TUI/log/session behavior, and add focused regression tests for each remaining hardening claim before any broad gate.

## Reproduction Assessment

Yes for the blocking findings. A Writable stream that returns false while still healthy reproduces the log truncation path through `src/cli/logs-cli.ts`, and the session-cache issue can be reproduced by resolving a session, rotating/deleting its primary JSONL, then resolving again within the proposed 5s cache TTL.

## Solution Assessment

No, this is not the best way to solve the issue as-is. The maintainable path is to split the valid hardening ideas into focused, current-main-based changes and preserve existing contracts instead of merging a stale bundled diff with regressions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Do not stop log output on backpressure:** `src/terminal/stream-writer.ts:49-50`
  - body: `stream.write()` returning `false` only means the writable buffer hit its high-water mark and should drain later, but `src/cli/logs-cli.ts` treats any false writer result as a terminal stop. With this change, high-volume `openclaw logs --follow` output can return early and truncate healthy streams.
  - confidence: 0.93
- **[P2] Read session directories fresh when resolving archives:** `src/infra/session-cost-usage.ts:344`
  - body: `resolveExistingUsageSessionFile` uses this directory listing to fall back from a missing primary JSONL to reset/deleted archives. Caching it for 5 seconds can return a stale primary path after session rotation/reset, causing usage and log loaders to return `null` until the TTL expires.
  - confidence: 0.86
- **[P2] Preserve local shell command semantics:** `src/tui/tui-local-shell.ts:116-117`
  - body: Local `!` commands are documented and implemented as non-interactive shell commands after approval. Parsing them into argv with `shell: false` passes pipes, globs, redirects, and `$VAR` expansion as literal arguments, so existing operator workflows silently misbehave.
  - confidence: 0.82

## Security Review
