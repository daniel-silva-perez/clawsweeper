## Summary

Keep open. Current main still gates user-facing model fallback and fallback-cleared reply notices on verbose mode, the requested independent opt-in config is absent from the config and docs surface, and existing e2e coverage still asserts verbose-off suppression. The linked implementation PR #69399 is open and unmerged with closing syntax, so this issue should remain open until that PR lands or maintainers choose a different implementation shape.

## What This Changes

Add a default-false opt-in config that allows model fallback and fallback-cleared notices to appear in normal, non-verbose assistant replies.

## Best Possible Solution

Review #69399 and either land it or request a revised implementation that adds a default-false agent defaults option, updates config types/schema/help/docs, and lets `runReplyAgent` prepend fallback and fallback-cleared notices when verbose mode is enabled or the opt-in is true while preserving once-per-active-fallback behavior.
