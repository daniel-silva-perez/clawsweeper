## Summary

Keep this PR open. Current main still classifies only numeric `sessions.json.bak.<digits>` files as session archive artifacts, while disk-budget cleanup relies on that helper to decide which files can be removed. The PR’s intended change is still useful and is not implemented by current main.

## Best Possible Solution

Keep the PR open and review or land this focused fix, or apply the equivalent change on current main: classify all non-empty `sessions.json.bak.*` names as store backup artifacts, keep the match scoped to the exact store-backup prefix, and add regression cases for the incident backup naming patterns before running the targeted session artifact tests and the normal changed gate.
