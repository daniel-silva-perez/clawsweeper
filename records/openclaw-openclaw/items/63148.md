## Summary

Keep this PR open. Current main still does not implement Feishu DM topic session isolation, reply-in-thread routing, or per-topic sequential keys, while the PR remains an active focused implementation candidate linked to the closed feature gap #35478.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to review or land a rebased Feishu plugin change that scopes DM `root_id` / `thread_id` messages to topic-specific peers, sets direct-topic `parentPeer`, keeps replies inside the originating topic, partitions sequential and debounce keys by DM topic, preserves main-surface DM behavior, and covers root-id, thread-id-only, dynamic-agent, and debounce cases in Feishu tests.
