## Summary

Keep this PR open. Current main still routes Feishu DM topic messages through the flat sender session and flat per-chat queue, so the PR is not obsolete, but the patch needs at least one correctness fix before merge because its sequential-key change treats `topic_group` events as non-group DM topics.

## What This Changes

The PR scopes Feishu DM messages with `root_id` or `thread_id` to topic-specific direct peers, replies in-thread, partitions sequential/debounce keys by topic, and adds Feishu regression tests.

## Best Possible Solution

Revise this PR so DM topic scoping is added without changing `topic_group` group serialization semantics, then review or land it with targeted Feishu tests for DM root-id, thread-id-only, dynamic-agent inheritance, sequential queue keys, and debounce partitioning.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Treat topic_group as a group queue key:** `extensions/feishu/src/sequential-key.ts:19`
  - body: `topic_group` is a Feishu group chat type in this plugin, but this new check only excludes `chat_type === "group"`. A topic-group message with `root_id` or `thread_id` will now get a per-topic queue key even when `handleFeishuMessage` still routes it to a flat group or group-sender session, allowing concurrent dispatches to the same session and reintroducing ordering/context races. Use `isFeishuGroupChatType` or explicitly exclude `topic_group` here before applying DM-topic queue keys.
  - confidence: 0.9
