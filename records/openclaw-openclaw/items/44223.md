## Summary

Keep open. Current main still implements and documents Ctrl+C in the TUI as clear/warn/exit only, with Esc and `/abort` as the active-run abort paths. This PR is a focused TUI behavior change that is not obsolete on main, and related #38501/#38502 show the remaining maintainer decision: make Ctrl+C abort active runs, or keep the current keybinding and improve the active-run hint.

## Best Possible Solution

Keep this PR open for maintainer review. The best outcome is to decide the TUI Ctrl+C contract: either land this PR or an equivalent focused patch that makes Ctrl+C abort active runs and updates `docs/web/tui.md`, or explicitly keep Ctrl+C as clear/exit and resolve the user confusion through the narrower active-run hint path in #38501/#38502. If landing abort behavior, keep the fix scoped to `src/tui/tui.ts`, `src/tui/tui-session-actions.ts`, `src/tui/tui-event-handlers.ts`, and focused regression tests.
