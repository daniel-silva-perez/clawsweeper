## Summary

Keep this PR open. Current main still does not surface Telegram `Sticker.fileId` in the agent-visible inbound conversation-info prompt, and the PR still needs maintainer review because the proposed `sticker_file_id` prompt key does not match the current Telegram send tool inputs.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a narrow fix that surfaces a replayable Telegram sticker handle in per-turn conversation info while aligning the exposed key with the accepted send contract, either by using `fileId`/`stickerId` in the prompt context or by adding a documented `sticker_file_id` alias to the shared schema and Telegram runtime. Keep regression coverage for stable-system-metadata exclusion, direct webchat omission, and Telegram inclusion.
