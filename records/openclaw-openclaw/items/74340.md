## Summary

Keep this PR open. Current main still has the raw `latestRawText` fallback, and the PR is a relevant narrow fix, but the patch is incomplete because an outcome-less frozen-result refresh path can still capture and later announce raw tool output.

## What This Changes

The PR adds a success-status guard to subagent announce output selection and a regression test so successful tool-only histories no longer use raw toolResult text as completion output.

## Best Possible Solution

Revise the PR so raw toolResult fallback is suppressed for successful completion captures across both direct announce and frozen-result refresh paths, while preserving timeout/error partial-progress behavior, adding regression coverage for the refresh/capture path, and updating the subagents docs to describe the outcome-specific fallback contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Cover outcome-less frozen-result refreshes:** `src/agents/subagent-announce-output.ts:306-308`
  - body: The new raw-output suppression only runs when `outcome?.status === "ok"`, but `refreshFrozenResultFromSession` later calls `captureSubagentCompletionReply(sessionKey)` without an outcome and writes the captured value into `frozenResultText`. A successful tool-only or compaction history refreshed through that path can still fall through to `snapshot.latestRawText` and be announced, so the guard needs to cover the refresh/capture path too.
