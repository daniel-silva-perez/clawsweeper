## Summary

Keep this PR open. Current main still has the raw `latestRawText` fallback, so the proposed behavior is not already implemented; the branch is small and reviewable, but maintainers should reconcile the success-only semantics with the documented fallback and the related auto-compaction work before merge.

## What This Changes

The PR adds a success-only guard in subagent announce output selection so successful tool-only histories return no result instead of raw tool text, plus a regression test for that case.

## Best Possible Solution

Land the narrow success-path suppression if maintainers agree raw tool output is not a successful completion result, preserve timeout/error partial-progress behavior, update `docs/tools/subagents.md` to describe the outcome-specific fallback, and keep the broader auto-compaction race tracked through #74073 and the competing open fixes.
