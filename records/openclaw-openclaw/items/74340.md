## Summary

Keep open. The PR targets a real subagent announce leak, but the proposed guard is incomplete: it only suppresses raw tool text when an ok outcome is passed, while the lifecycle refresh/capture path can still call the same output reader without an outcome and store raw tool output as the frozen completion result.

## What This Changes

The PR adds an ok-status guard before the subagent output selector falls back to raw tool text, plus a regression test for successful tool-only histories.

## Best Possible Solution

Revise or supersede the PR with one canonical fix that prevents raw tool/toolResult fallback for successful completion captures across direct announce, initial freeze, and refresh paths, while preserving timeout/error partial-progress behavior and updating subagent docs/tests to match the contract.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a successful tool-only history: the PR's direct `readSubagentOutput(..., { status: "ok" })` test covers one path, and the remaining failing path is the outcome-less `refreshFrozenResultFromSession` capture that can still select `snapshot.latestRawText`.

## Solution Assessment

No. The PR is a useful partial fix, but the narrow `outcome?.status === "ok"` guard is not the best complete solution because successful refresh/capture calls can still arrive without that outcome.
