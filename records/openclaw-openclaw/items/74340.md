## Summary

Keep open. The patch is a useful partial fix, but it is not merge-ready because the raw tool-result fallback remains reachable through the frozen-result refresh path identified in the related production report and PR discussion.

## What This Changes

The PR adds a success-status guard in subagent announce output selection and a regression test for successful tool-only histories.

## Best Possible Solution

Suppress raw toolResult fallback for successful completion captures across direct announce, completion freeze, and frozen-result refresh paths while preserving timeout/error partial-progress behavior; then add coverage for the refresh path and update the subagents docs to describe the outcome-specific fallback contract.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Cover outcome-less frozen-result refreshes:** `src/agents/subagent-announce-output.ts:306-308`
