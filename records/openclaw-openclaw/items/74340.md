## Summary

Keep this PR open for maintainer review. Current main still has the raw tool-result fallback that the PR targets, but the proposed branch changes a documented and tested completion-mode contract without updating the existing e2e coverage or docs, and it cuts against the prior #58035 discussion about preserving exec output in parent completion events.

## What This Changes

The PR changes subagent announce output selection so successful tool-only histories no longer return latest raw tool/toolResult text, and adds a focused regression test for that case.

## Best Possible Solution

Decide the intended announce contract by outcome. If successful tool-only runs should no longer surface raw tool output, land a narrow selector change with matching focused and e2e tests plus docs updates; if sanitized tool fallback remains intended for successful runs, close or revise this branch while keeping the timeout/error safeguards intact.
