## Summary

Keep open. Current main still does not implement WhatsApp missed-message backfill after reconnect: inbound handling remains driven by Baileys upsert events, stale append history is skipped, full socket history sync is disabled, and reconnect handling only drains pending outbound deliveries. The BlueBubbles catch-up item is useful related context but is channel-specific and does not close this WhatsApp gap.

## What This Changes

Request bounded WhatsApp recovery for messages missed while the Web connection is down, preferably by replaying eligible monitored-chat history after reconnect or warning when backfill is unavailable.

## Best Possible Solution

Implement bounded reconnect recovery inside the WhatsApp plugin: persist last-seen state per account and eligible chat, validate the Baileys history mechanism for monitored chats, cap lookback and replay volume, dedupe by message ID before enqueueing, and emit a clear warning when provider backfill is unavailable or not trustworthy.
