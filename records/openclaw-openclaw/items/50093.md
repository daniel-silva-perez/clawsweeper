## Summary

Keep open. Current main still lacks the requested WhatsApp reconnect backfill: inbound handling only consumes Baileys `messages.upsert` events, stale `append` history older than the 60-second connect grace window is skipped, full history sync is disabled, and reconnect handling drains pending outbound deliveries without fetching or replaying missed inbound group messages. The related BlueBubbles catch-up item is useful analog evidence, but it is channel-specific and does not supersede this WhatsApp gap.

## What This Changes

The issue requests bounded WhatsApp missed-message backfill after reconnect, especially for monitored group chats that receive messages while the WhatsApp Web/Baileys connection is down.

## Best Possible Solution

Implement bounded reconnect recovery inside the WhatsApp plugin: persist last-seen state per account and eligible chat, validate a Baileys history mechanism for monitored chats, cap lookback and replay volume, dedupe by message ID before enqueueing, and emit a clear warning when provider backfill cannot be trusted or is unavailable.
