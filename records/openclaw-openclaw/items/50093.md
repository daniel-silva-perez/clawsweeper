## Summary

Keep open. Current main still lacks the requested WhatsApp reconnect backfill: the plugin processes only Baileys inbound upsert events, intentionally skips stale append/history messages outside a 60-second grace window, and reconnect handling drains outbound deliveries without any per-chat cursor, on-demand history request, or explicit missed-message warning.

## What This Changes

Request WhatsApp Web reconnect recovery that backfills messages missed during outage windows for active monitored chats, especially groups.

## Best Possible Solution

Implement this inside the WhatsApp plugin as bounded reconnect recovery: persist last-seen state per account/chat, validate and use the Baileys/WhatsApp Web history mechanism for eligible monitored chats, cap lookback and per-run volume, dedupe by message ID before enqueueing, and emit a clear gap warning when provider backfill cannot be trusted.
