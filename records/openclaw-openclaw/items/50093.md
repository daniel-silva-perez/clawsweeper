## Summary

Keep open. Current main still does not implement the requested WhatsApp missed-message backfill after reconnect: the WhatsApp plugin consumes live Baileys `messages.upsert` events, skips stale `append` history older than the 60-second connect grace window, creates the socket with full history sync disabled, and reconnect handling only drains pending outbound deliveries. The BlueBubbles catchup work is useful related evidence, but it is channel-specific and does not supersede this WhatsApp gap.

## What This Changes

Request bounded WhatsApp reconnect recovery so messages sent to monitored chats during a WhatsApp Web outage are replayed after reconnect, or at least surfaced with a clear missed-message warning.

## Best Possible Solution

Implement WhatsApp-plugin-owned reconnect recovery: persist last-seen state per account and eligible chat, use a validated Baileys history mechanism for monitored chats only, cap lookback and per-run volume, dedupe by message ID before enqueueing, avoid unintended read/receipt side effects, and emit an explicit warning when provider backfill cannot be trusted or is unavailable.
