## Summary

Keep open. Current main still only consumes Baileys live `messages.upsert` events, skips `append` history older than a 60-second connect grace window, disables full socket history sync, and reconnect handling drains outbound deliveries without a per-chat cursor, on-demand history request, or missed-message warning. The linked BlueBubbles catch-up item is useful related evidence, but it is channel-specific and does not supersede this WhatsApp gap.

## What This Changes

Add WhatsApp reconnect backfill so messages sent to monitored chats during a WhatsApp Web outage are recovered after reconnection.

## Best Possible Solution

Implement bounded reconnect recovery inside the WhatsApp plugin: persist last-seen state per account and eligible chat, validate a Baileys history mechanism for monitored chats, cap lookback and replay volume, dedupe by message ID before enqueueing, and emit a clear warning when provider backfill cannot be trusted or is unavailable.
