## Summary

Keep open. Current main still consumes WhatsApp inbound messages from Baileys events only, with a narrow 60-second `append` grace path, and does not persist per-chat cursors or run a reconnect-time missed-message backfill for monitored chats.

## What This Changes

Request bounded WhatsApp missed-message recovery after reconnect, ideally by tracking per-chat last-seen state and fetching/replaying messages sent during the outage window.

## Best Possible Solution

Keep this open and design the fix inside the WhatsApp plugin: persist last-seen state per account/chat, validate the Baileys/WhatsApp Web history mechanism for a bounded reconnect gap, replay only eligible monitored chats within a capped window, dedupe by message ID before enqueueing, and log a clear gap warning when reliable provider backfill is unavailable.
