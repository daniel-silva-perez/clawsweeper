## Summary

Keep open. Current main still does not implement WhatsApp missed-message backfill after reconnection: the WhatsApp plugin consumes live Baileys `messages.upsert` events, deliberately skips stale `append` history outside a 60-second grace window, disables full history sync, and reconnect handling only drains pending outbound sends. The related BlueBubbles catchup work is implemented for a different channel and does not supersede this WhatsApp gap.

## What This Changes

Add WhatsApp reconnect recovery that backfills missed inbound messages for monitored chats after a WhatsApp Web outage.

## Best Possible Solution

Implement a WhatsApp-plugin-owned reconnect recovery path: persist last-seen state per account and eligible chat, request bounded provider history for monitored chats only, dedupe by message ID before enqueueing, cap lookback and per-run volume, avoid unintended read/receipt side effects, and emit a clear warning when provider backfill is unavailable or untrusted.
