## Summary

Keep open. Current main still does not implement the requested `agent:turn:end` internal hook, and this PR remains the focused implementation attempt. It should not be closed as implemented because ACP turn completion is still stats-only on main and in the latest release. The PR also has concrete review issues to address before merge: the advertised failure `errorCode` is missing from the hook payload, and the proposed dispatch uses the unbounded fire-and-forget helper even though a bounded helper exists.

## Best Possible Solution

Keep this PR open and update it in place or with a narrow replacement commit. The implementation should emit `agent:turn:end` from `AcpSessionManager.recordTurnCompletion()` on both success and failure, pass `sessionKey`, `success`, `durationMs`, and optional `errorCode`, use bounded non-blocking dispatch unless a maintainer documents a deliberate exception, and keep tests focused on the type guard plus manager-level success/failure emission. Do not broaden this into `agent:turn:start`, new config, or a larger lifecycle refactor.
