## Summary

Keep open. Current main still lacks the requested ACP turn-completion internal hook, and this PR is the focused implementation attempt, but it is not merge-ready because the new payload drops the advertised failure code and dispatches a per-turn hook through the unbounded helper.

## What This Changes

This PR adds an `agent:turn:end` internal hook from ACP turn completion, adds the hook context/type guard, and adds focused success/failure emission tests.

## Best Possible Solution

Land a narrow ACP-manager internal hook that fires once per completed turn, preserves failure classification in the payload, uses bounded non-blocking dispatch, and has focused tests for success, failure, type guarding, and stalled-hook safety expectations.

## Reproduction Assessment

Yes. Current main can be verified by the exact symbol search plus inspection of `AcpSessionManager.recordTurnCompletion()`; the PR reproduction path is to register an internal `agent:turn:end` listener and run successful and failing ACP turns, which the proposed manager tests target.

## Solution Assessment

No, not as-is. The hook point is the right narrow seam, but the proposed contract should include optional `errorCode` on failures and dispatch through `fireAndForgetBoundedHook(() => triggerInternalHook(...))` or have an explicit maintainer-approved exception.
