## Summary

Keep open. Current main still lacks the requested `agent:turn:end` internal hook, and this PR is the focused implementation attempt, but the proposed patch is not merge-ready because it drops the advertised failure `errorCode`, uses unbounded hook dispatch on a per-turn path, and omits the required changelog entry.

## What This Changes

This PR adds an `agent:turn:end` internal hook emitted from ACP turn completion, defines its hook context/type guard, and adds focused success/failure emission tests.

## Best Possible Solution

Land a narrow ACP-manager internal hook that fires once per completed turn, preserves the failure classification in the payload, uses bounded non-blocking dispatch, and documents or changelogs the new hook surface according to maintainer policy.

## Reproduction Assessment

Yes. Current-main absence is reproducible with the exact symbol search plus inspection of `AcpSessionManager.recordTurnCompletion()`; the PR behavior can be exercised by registering an `agent:turn:end` listener and running successful and failing ACP turns, which the proposed manager tests target.

## Solution Assessment

No, not as-is. The hook point is the right narrow seam, but the contract should include optional `errorCode` on failures and dispatch should use `fireAndForgetBoundedHook(() => triggerInternalHook(...))` or have an explicit maintainer-approved exception.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Propagate the turn failure code:** `src/hooks/internal-hooks.ts:171-178`
  - body: `recordTurnCompletion` receives `errorCode` for failed turns, but the new hook context only exposes `success` and `durationMs`. Consumers of `success: false` lose the ACP failure classification even though the PR advertises optional `errorCode` and the call site already has it.
  - confidence: 0.91
- **[P2] Use bounded dispatch for turn-end hooks:** `src/acp/control-plane/manager.core.ts:1685-1694`
  - body: The new call uses `fireAndForgetHook(triggerInternalHook(...))`, so a listener that never settles can leave the hook promise pending indefinitely on a per-turn path. `triggerInternalHook` awaits handlers sequentially, and `fireAndForgetBoundedHook(() => ...)` already provides timeout, concurrency, and queue limits for this pattern.
  - confidence: 0.9
