## Summary

Keep open. Current main still does not implement the PR's intended `agent:turn:end` internal hook, and this PR remains the focused implementation attempt rather than obsolete cleanup. The PR is not merge-ready as-is because the proposed payload omits the advertised failure `errorCode`, and the hook dispatch path uses unbounded fire-and-forget execution even though current main has a bounded helper for similar asynchronous hook work.

## Best Possible Solution

Keep this PR open for maintainer review or update it in place. The best implementation path is to emit `agent:turn:end` from `AcpSessionManager.recordTurnCompletion()` on both success and failure, include `sessionKey`, `success`, `durationMs`, and optional `errorCode`, use bounded non-blocking hook dispatch or document a deliberate exception, cover success/failure manager emission plus the type guard, and update the internal hooks docs if the event becomes operator-facing behavior.
