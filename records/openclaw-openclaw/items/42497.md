## Summary

Keep open. Current main still lacks the proposed in-process local dispatch path, so this PR is not obsolete, but the branch is not mergeable as-is because its local branch bypasses current gateway token, scope, timeout, and JSON-frame semantics.

## What This Changes

The PR adds a local gateway dispatch registry, registers it from gateway startup, makes embedded agent gateway-tool calls prefer in-process dispatch except explicit URL and `expectFinal` cases, and adds unit/integration coverage.

## Best Possible Solution

Keep the in-process dispatch direction, but implement it as an explicit gateway request adapter that preserves WebSocket-equivalent behavior: resolved token intent, least-privilege or explicit scopes, default-deny unknown methods, timeout handling, `expectFinal` support or deliberate fallback, and JSON-frame-equivalent params normalization.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve gateway scopes before local dispatch:** `src/agents/tools/gateway.ts:151-158`
  - body: The local branch returns before `callGatewayTool` computes explicit or least-privilege scopes. `dispatchGatewayMethod` then runs under the ambient request client or a synthetic write-scoped client, so embedded admin cron calls, dynamic scope overrides, and default-deny behavior no longer match the WebSocket path.
  - confidence: 0.9
- **[P1] Honor token and timeout options locally:** `src/agents/tools/gateway.ts:151-158`
  - body: The guard skips local dispatch only for explicit URLs and `expectFinal`, but calls with `gatewayToken` or `timeoutMs` still bypass `resolveGatewayOptions` and the WebSocket request timer. That silently ignores explicit auth intent and can leave embedded tool operations unbounded.
  - confidence: 0.88
- **[P2] Normalize params before dispatching locally:** `src/agents/tools/gateway.ts:155-157`
  - body: The local path forwards the live `params` object directly, while the WebSocket path sends a JSON frame. Optional properties set to `undefined` can therefore reach TypeBox validators locally even though they would be omitted over WebSocket.
  - confidence: 0.82

## Security Review

Concerns:

- **[high] Local dispatch bypasses auth and scope resolution:** `src/agents/tools/gateway.ts:151`
  - body: `callGatewayTool` can return from local dispatch before resolving explicit gateway tokens, least-privilege method scopes, or explicit scope overrides, so embedded calls may execute under ambient or synthetic gateway identity rather than the caller's intended authorization contract.
  - confidence: 0.88
