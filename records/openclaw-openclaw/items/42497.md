## Summary

Keep this PR open. The underlying WebSocket self-contention problem is still real and current main still routes `callGatewayTool` through the WebSocket transport, but the proposed early in-process return changes existing gateway authorization, timeout, and serialization semantics in ways that should block merge until the design is reworked.

## What This Changes

The PR adds a gateway local-dispatch registry, registers `dispatchGatewayMethod` during gateway startup, and makes `callGatewayTool` prefer in-process dispatch for embedded non-`expectFinal` calls, with focused tests.

## Best Possible Solution

Rework this into an explicit in-process gateway request adapter that first resolves the normal gateway options and caller scopes, constructs an equivalent authorized client, applies bounded timeout behavior, performs JSON-frame-equivalent params normalization, and handles final-response semantics deliberately, with regression tests for token, timeout, scopes, params, and cron/session tool calls.

## Reproduction Assessment

Yes. A focused test can register a local dispatcher, call `callGatewayTool` with `gatewayToken`, `timeoutMs`, explicit scopes, or params containing optional `undefined` fields, and verify the PR path returns locally before the current WebSocket contract is applied.

## Solution Assessment

No. The registry direction is plausible, but the early-return implementation is not the best fix because it bypasses the existing transport/auth contract instead of adapting the in-process path to match it.

  - confidence: 0.88

  - body: The local path forwards the live `params` object, while the WebSocket path JSON-serializes the request frame. Optional keys set to `undefined` can therefore reach validators locally even though they are omitted over WebSocket, making embedded calls fail that currently succeed.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces an in-process gateway control-plane path without preserving the existing authorization, credential, and timeout contract.

Concerns:

- **[high] Local dispatch bypasses caller authorization intent:** `src/agents/tools/gateway.ts:152`
  - body: The early local return can execute a gateway method before resolving explicit tokens, least-privilege scopes, or explicit scope overrides, so embedded calls may run under ambient or synthetic credentials rather than the caller's intended authorization contract.
  - confidence: 0.88
- **[medium] Local dispatch skips bounded request timeout:** `src/agents/tools/gateway.ts:152`
  - body: Because `timeoutMs` is not applied on the local path, a stuck in-process handler can block an active agent turn instead of failing under the timeout behavior used by the WebSocket request path.
  - confidence: 0.84
