## Summary

Keep this PR open. Current main still lacks the intended in-process embedded gateway-tool dispatch path, and the cron tool still reaches gateway methods through the WebSocket-backed helper. The PR remains a plausible fix candidate for the linked WS self-contention bug, but it needs a current-main rebase/adaptation before it is safe to merge.

## Best Possible Solution

Keep this PR open and ask for a current-main rebase or adapted patch. The maintainer-ready fix should add embedded in-process dispatch only when it is semantically equivalent to current callGatewayTool, preserving URL/token overrides, timeouts, expectFinal, explicit scope overrides, least-privilege/default-deny method scopes, and JSON-normalized params. Add focused regression tests for embedded cron status/list/add/update/remove/run/runs plus URL, token, timeout, expectFinal, explicit-scope, default-scope, and undefined/optional parameter cases before merge.
