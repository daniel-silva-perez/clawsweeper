## Summary

Keep this PR open. Current main still has the memory-lancedb contamination path the PR targets: auto-capture passes raw user message text through shouldCapture and stores that same raw text, while auto-recall fetches the top three LanceDB results and injects them without envelope-sludge filtering. The older #67475 is closed and unmerged, so it does not supersede this focused reimplementation. Security/supply-chain pass: the supplied PR file list is limited to the memory-lancedb implementation and tests, with no workflow, dependency, lockfile, package publishing, install script, generated/vendor, or secret-handling changes; however, the Aisle review on the current PR head still flags a sentinel-leading-whitespace bypass that should be resolved or explicitly accepted before landing.

## Best Possible Solution

Keep this PR open for maintainer review rather than closing it as obsolete. The best path is to land a narrow memory-lancedb fix, either from this PR or a maintainer revision, that sanitizes auto-captured user text before embedding/storage, filters contaminated recall candidates before prompt injection, keeps sentinel matching aligned with src/auto-reply/reply/strip-inbound-meta.ts, strips recalled media annotations before prompt/media-reference parsing, and preserves focused regression tests. Resolve the current Aisle sentinel-whitespace finding before merge. #67475 can remain closed because this PR is the focused successor.
