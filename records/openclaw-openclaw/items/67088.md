## Summary

Keep open. Current main still short-circuits macOS dashboard browser opening when any `SSH_*` variable is present and no `DISPLAY`/`WAYLAND_DISPLAY` exists, then prints the reported `No GUI detected` SSH hint. The issue also has an open implementation candidate, PR #67115, that uses closing syntax, so the issue should not be closed before that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review or replace PR #67115 with an equivalent narrow fix in `src/infra/browser-open.ts`: make the SSH/no-display early return Linux-only, or otherwise exclude `darwin`, so macOS can fall through to `/usr/bin/open`. Add regression tests for Darwin with stale `SSH_*` variables and no display resolving to `open`, plus Linux preserving `ssh-no-display`.
