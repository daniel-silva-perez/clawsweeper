## Summary

Keep open. Current main still returns `ssh-no-display` on macOS when any `SSH_*` variable is set and no `DISPLAY`/`WAYLAND_DISPLAY` exists, before the Darwin `open` path can run. The provided GitHub context also shows open PR #67115 references this issue with closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review or replace PR #67115 with a narrow fix in `src/infra/browser-open.ts`: make the SSH/no-display early return Linux-only or otherwise exclude `darwin`, then add regression tests for Darwin with stale `SSH_*` variables resolving to `open` and Linux preserving `ssh-no-display`.

## Evidence

On macOS this runs before the later `platform === "darwin"` branch can resolve `open`.
