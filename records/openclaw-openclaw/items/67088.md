## Summary

Keep open. Current main still contains the reported macOS false headless path, and the issue is paired with open unmerged PR #67115 that claims to fix it, so cleanup closure before merge would be premature.

## What This Changes

The issue asks for `openclaw dashboard` on macOS to ignore stale `SSH_*` variables from Tailscale SSH or reused shells so it opens the Control UI instead of showing the headless SSH tunnel hint.

## Best Possible Solution

Keep this issue open and review or replace PR #67115 with a narrow change in `src/infra/browser-open.ts` that makes the SSH/no-display early return Linux-only or otherwise excludes Darwin, plus focused tests for Darwin with stale `SSH_*` variables resolving to `open` and Linux preserving `ssh-no-display`.
