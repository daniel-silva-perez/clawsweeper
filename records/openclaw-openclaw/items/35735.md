## Risks / Open Questions

- The PR's non-dry-run path can trigger unrelated session maintenance through `updateSessionStore`, so dry-run and actual deletion may diverge and users may lose sessions not matched by `rm` or `clear --older-than`.
- Transcript deletion failures are swallowed while the path is still reported as removed, which can mislead users after a destructive operation.
- The JSON output shape is unstable between `rm` and `clear` because `key` is omitted for `clear` instead of serialized as `null`.
- The PR adds user-facing destructive CLI commands without updating `https://docs.openclaw.ai/cli/sessions`.
- Bulk `clear` semantics need maintainer review against existing Gateway behavior that rejects deleting the main session.
