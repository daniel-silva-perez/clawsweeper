## Summary

Keep this PR open. Current main still does not provide `openclaw sessions rm` or `openclaw sessions clear`, so the PR is not obsolete or implemented on main. It remains useful, but the destructive deletion path needs maintainer review because existing review comments identify correctness and safety issues in the proposed implementation.

## Best Possible Solution

Keep this PR open for maintainer review or a rebased successor. The best implementation is a focused `openclaw sessions rm <key>` and `openclaw sessions clear` CLI that avoids unrelated maintenance pruning during targeted deletion, safely archives or removes only orphaned transcript/sidecar artifacts, returns stable JSON, documents the public CLI at `https://docs.openclaw.ai/cli/sessions`, and adds targeted command/registration tests. If maintainers reject this command shape, replace it with a narrower canonical design issue rather than closing this useful contribution as obsolete.
