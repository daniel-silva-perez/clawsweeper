## Summary

Keep open: this PR is still aimed at a real missing Signal feature, but the latest head is not merge-ready. The branch adds the requested Signal reply/reaction plumbing, yet the actual current head still misses authorNumber in quote visibility resolution and does not preserve quote authors for Signal group reply sends, despite older review-thread replies describing fixes that are not present in the latest eight-file diff.

## What This Changes

The PR branch adds Signal inbound ReplyToId propagation, reaction-triggered heartbeat wakeups, outbound quote-timestamp/quote-author handling for replyToId, first-chunk quoting in Signal send paths, a new send-quote helper, a heartbeat SDK export, and a status-json test expectation update.

## Best Possible Solution

Update the branch so Signal quote sender resolution accepts authorNumber, then restore or reimplement the quote-author propagation needed for group replies while preserving the existing visibility/remap guards. Add focused Signal regression tests for authorNumber-only group quotes, group quoted replies, chunked/media first-message quote behavior, and custom mediaReadFile payload sends before the PR is considered merge-ready.

Thread the visible/current quoted sender author through the reply payload and send calls, with the same visibility and remap guards as the quote context.
