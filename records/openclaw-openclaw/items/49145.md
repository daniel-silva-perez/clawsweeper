## Summary

Keep this PR open: current main still lacks the requested Signal ReplyToId, reaction wake, and native quote parameter behavior, so the branch is not obsolete. The current PR head still has a concrete sendPayload media regression because it drops the custom mediaReadFile reader on the new media payload path.

## What This Changes

The PR updates the Signal plugin to carry inbound quote ReplyToId metadata, wake heartbeat processing for reaction notifications, add outbound quote-timestamp/quote-author support, keep quotes on the first outbound chunk only, add a lightweight quote helper, export requestHeartbeatNow through the heartbeat SDK subpath, and relax one status-json test expectation.

## Best Possible Solution

Keep the feature branch alive, fix the media reader propagation gap with focused Signal outbound coverage, then have maintainers do normal review of the Signal quote/reaction behavior against the plugin boundary and SDK export change.
