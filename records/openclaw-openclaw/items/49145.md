## Summary

Keep open. Current main still lacks the intended Signal reply/quote behavior, so this PR is not obsolete, but the latest PR head still has blocking correctness issues: authorNumber-only quoted senders can be excluded by the visibility gate, and Signal group replies do not carry the quote author needed to emit native quote metadata.

## What This Changes

The PR branch adds Signal inbound ReplyToId propagation, reaction-triggered heartbeat wakeups, outbound Signal quote parameters for replies, first-chunk quote handling in Signal send paths, a heartbeat SDK export, and one status JSON test expectation update.

## Best Possible Solution

Update the branch so Signal quote sender resolution accepts authorNumber, then carry a verified quote author through the inbound reply payload and Signal outbound send paths for group replies while preserving the existing visibility, remap, and first-chunk guards. Add focused regression coverage for authorNumber-only group quotes, group quoted replies, chunked/media first-message quote behavior, and payload media sends before merge.

## Reproduction Assessment

Yes. A high-confidence static reproduction is an allowlisted Signal group quote payload with id/text/authorNumber but no author/authorUuid: the PR type accepts authorNumber, but the resolver ignores it, so quoteDecision.include can be false and ReplyToId is dropped. A second path is a live group reply with replyToId: the PR passes only replyToId, group targets cannot infer quoteAuthor, and send.ts emits no quote fields.

## Solution Assessment

No. The feature direction is useful and current main lacks it, but this PR is not yet the best mergeable fix because it forwards replyToId without the author metadata Signal requires for group quote replies. The safer solution is to plumb quoteAuthor as a guarded piece of reply metadata rather than infer it from group targets.
