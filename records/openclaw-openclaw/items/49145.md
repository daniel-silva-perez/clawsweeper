## Summary

Keep this PR open. Current main still lacks the Signal reply/quote support the PR is trying to add, but the latest PR diff is not mergeable because it still loses authorNumber-only inbound quote context and cannot emit Signal quote metadata for group replies that only carry replyToId.

## What This Changes

The PR branch adds Signal inbound ReplyToId propagation, reaction-triggered heartbeat wakeups, outbound Signal quote parameters for replies, first-chunk quote handling in Signal send paths, a heartbeat SDK export, and one status JSON test expectation update.

## Best Possible Solution

Update the branch so Signal quote sender resolution accepts authorNumber, then carry a verified quote author through the inbound reply payload and all Signal outbound send paths that need quote-timestamp/quote-author, while preserving visibility policy, remap guards, and first-chunk-only behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction is an allowlisted Signal group quote payload with id/text/authorNumber but no author/authorUuid; the PR type accepts authorNumber, but the visibility resolver ignores it, so ReplyToId can be dropped. A second path is a Signal group reply with replyToId only: group targets cannot infer quoteAuthor, so send.ts emits no quote fields.

## Solution Assessment

No. The feature direction is useful and current main does not implement it, but this PR is not yet the best mergeable fix because it forwards replyToId without the author metadata Signal needs for group quoted replies.
