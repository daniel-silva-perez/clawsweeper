## Summary

Keep this PR open. The latest branch addresses the prior mediaReadFile review finding, but the Signal quote-context implementation still misses an upstream quote author shape that can suppress the newly added ReplyToId path in group allowlist scenarios.

## What This Changes

The PR branch adds Signal inbound quote context fields, reaction-triggered heartbeat wakeups, outbound Signal quote parameters for replies, first-message quote handling for chunked/media payloads, a focused heartbeat SDK export, and a status-json test expectation update.

## Best Possible Solution

Update the branch so Signal quote sender resolution accepts authorNumber as a source-number fallback, then add focused regression coverage for authorNumber-only quoted group messages under allowlist visibility before merging the otherwise narrow Signal changes.
