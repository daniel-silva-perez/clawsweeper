## Summary

Keep open. Current main does not supersede this Signal reply-context work, and the latest PR head still has a concrete P2 correctness issue: authorNumber-only quoted senders are not included in the quote visibility resolver, so group allowlist mode can drop the new reply context even though signal-cli provides the sender number.

## What This Changes

The PR adds Signal inbound ReplyToId quote context, reaction-triggered heartbeat wakeups, outbound Signal quote parameters for replies, first-chunk-only quote handling for chunked/media payloads, a focused heartbeat SDK export, and a status-json test expectation adjustment.

## Best Possible Solution

Update the branch so Signal quote sender resolution treats authorNumber as a source-number fallback, keep ReplyToId under the same visibility policy, add a focused authorNumber-only group allowlist regression test, and then rerun the Signal targeted tests plus the changed gate before merge.
