## Summary

Keep this PR open. Current main still lacks the proposed ACP follow-up orchestration page and accepted-note wording, and the shipped docs/runtime/tests still describe direct routing of bound ACP follow-up text to the ACP session. The PR is narrow and has no apparent supply-chain or secrets risk, but it documents desired semantics ahead of the open runtime/product issue #69669, so it needs ACP maintainer review rather than cleanup closure.

## What This Changes

This PR adds an ACP follow-up orchestration docs page and changes the ACP spawn accepted note to say thread-bound follow-ups go through the parent assistant unless exact relay is requested.

## Best Possible Solution

Keep this PR open for ACP maintainer review alongside #69669. Maintainers should decide whether to intentionally document the desired parent-orchestrated semantics before the runtime change, or revise this docs-only PR to match current direct-routing behavior and leave the runtime policy change tracked in #69669.
