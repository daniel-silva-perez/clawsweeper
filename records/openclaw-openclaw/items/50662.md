## Summary

Keep open. Current main still does not implement the PR's two intended behavior changes: restart-health probes still rely on process environment gateway credentials, and rate-limit failover still performs the default same-provider profile rotation before model fallback. Current main has adjacent mitigation and configuration, but not the PR's combined behavior.

## Best Possible Solution

Keep this PR open for maintainer review. If the behavior is still desired, rebase it onto current main, wire restart-health through the config-aware probe-auth resolver with explicit credential-precedence handling, and update the current failover-policy implementation plus docs/tests if rate limits should skip profile rotation by default. Otherwise, maintainers should close it explicitly as a rejected policy change rather than as already implemented.
