## Summary

Keep open. Current main still does not implement the PR's combined behavior: restart-health reachability probes still use only `OPENCLAW_GATEWAY_TOKEN` / `OPENCLAW_GATEWAY_PASSWORD` from the process environment, and rate-limit failover still defaults to one same-provider auth-profile rotation before model fallback. Current main has adjacent mitigation and configuration, including local 1008 auth-close reachability handling and `auth.cooldowns.rateLimitedProfileRotations: 0`, but that is not the PR's default immediate-fallback policy or config-backed restart probe auth path.

## Best Possible Solution

Keep the PR open for maintainer review. If the behavior is still desired, rebase it onto current main, implement the rate-limit default-policy change in `src/agents/pi-embedded-runner/run/failover-policy.ts` and `src/agents/pi-embedded-runner/run/assistant-failover.ts`, update tests/docs that currently assert one rotation by default, and either wire restart-health through the config-aware probe-auth resolver or explicitly document that the 1008 auth-close reachability allowlist is the intended restart-health behavior.
