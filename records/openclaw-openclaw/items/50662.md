## Summary

Keep open. Current main does not implement the PR's combined behavior: restart health still does not use the config-aware probe-auth resolver, and rate-limit failover still rotates one same-provider auth profile by default before model fallback. Current main has related improvements, including a local auth-close reachability allowlist and a configurable `rateLimitedProfileRotations: 0` escape hatch, but those do not make the PR obsolete.

## Best Possible Solution

Keep the PR open for maintainer review. If the behavior is still desired, rebase it onto current main, implement the rate-limit decision in `src/agents/pi-embedded-runner/run/failover-policy.ts` and `src/agents/pi-embedded-runner/run/assistant-failover.ts`, update the tests that currently assert rotation-first behavior, and either wire restart-health through the config-aware probe-auth resolver or document why the existing local auth-close reachability allowlist is the intended restart-health behavior.
