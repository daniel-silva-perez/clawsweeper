## Summary

Keep this PR open. Current main still does not implement the PR's intended change: `logging.redactSensitive` is accepted as only `"off" | "tools"` in the config schema, generated schema, TypeScript type, help quality expectations, and docs. The linked bug report is closed, but the checked-out main branch does not accept `"all"`, and the PR still needs maintainer/product review because current docs now describe `"tools"` as covering log and transcript sinks.

## Best Possible Solution

Keep this PR open for maintainer/product decision. If OpenClaw wants compatibility with configs that used `logging.redactSensitive: "all"`, update the PR against current main to accept `"all"` in schema/types/generated schema/help quality expectations, make the `all` vs `tools` behavior explicit, and add validation/redaction/security-audit tests. If the intended contract is only `off | tools`, close it manually with that rationale and consider a doctor/config-validation remediation for users who copied `"all"`.
