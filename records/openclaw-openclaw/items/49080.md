## Summary

Current main implements the requested safe refresh path: live session bootstrap context is reloaded from workspace files on later turns instead of being held as a long-lived session snapshot, and the fix is shipped in v2026.4.26.

## Best Possible Solution

Keep the shipped per-turn bootstrap refresh behavior. If maintainers want stronger policy guarantees for opt-in bootstrap-skipping modes, track that as a narrower follow-up about durable-state enforcement rather than keeping this default live-session stale-context report open.
