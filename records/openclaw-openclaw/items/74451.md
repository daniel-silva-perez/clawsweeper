## Summary

Keep open. Current `main` has a partial suppression for the unsupported OpenAI Codex OAuth mini route, but the configured inline-model path still resolves rows with an `api` before checking suppression, which matches the stale configured model-list path described by the reporter.

## What This Changes

The issue reports that ChatGPT-backed OpenAI Codex sessions can switch to `openai-codex/gpt-5.4-mini` and then repeatedly fail assistant turns instead of rejecting, reverting, or falling back safely.

## Best Possible Solution

OpenAI Codex model capability gating should reject suppressed OAuth models before any assistant turn reaches the vendor transport, including built-in catalog rows, discovered rows, configured provider model rows, live session overrides, and fallback candidates. Stale overrides should revert or fall through to a supported configured fallback with the existing clear diagnostic.
