## Summary

Keep this PR open. Current main still lacks the PR's outer guards in the BlueBubbles chat helper: `markBlueBubblesChatRead` and `sendBlueBubblesTyping` delegate directly to `sendBlueBubblesChatEndpointRequest`, which only checks the cached Private API status after resolving a client. The PR is not implemented on main, and the related open #69932 overlaps typing call sites but does not cover the mark-read/helper-level behavior here.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either land a small helper-level guard for `markBlueBubblesChatRead` and `sendBlueBubblesTyping`, or fold the mark-read/helper guard into the broader BlueBubbles Private API fallback work in #69932. Before merge, confirm whether the proposed `=== false` outer guard handles the live null-cache race described in the PR body or whether the fix should also defer optional typing/read calls until Private API status is known.
