## Summary

Keep this PR open. Current main still rejects unsigned Feishu webhook requests before parsing `url_verification` challenges, so the callback setup bug remains; the PR is useful but is dirty against current main and needs to be ported around later Feishu webhook hardening before merge.

## What This Changes

The PR adds Feishu webhook authorization helpers that accept `url_verification` challenges when the normal signature is valid or the configured verification token matches the plaintext/encrypted challenge payload, plus e2e coverage for unsigned plaintext and encrypted challenges.

## Best Possible Solution

Port the token-verified `url_verification` exception onto the current Feishu webhook transport while preserving current pre-auth body limits, the required `encryptKey`, fail-closed non-challenge signature checks, and existing security tests; land it as the fix for #58905.
