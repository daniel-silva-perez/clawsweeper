Keep this PR open. Current main still rejects `nonInteractivePermissions: "allow"` in the ACPX plugin schema and manifest, the docs still describe only `fail`/`deny`, and the linked PR is still the focused implementation candidate for the closed user report. The PR should not be cleanup-closed as implemented or superseded; it needs maintainer review/changes, especially around security flagging for the new auto-approval mode.
