## Summary

Keep this PR open. Current main still does not implement progressive disclosure for long markdown messages in the Control UI, and the submitted PR remains potentially useful but needs maintainer review/revision rather than cleanup closure.

## Best Possible Solution

Keep this PR open for focused Web UI maintainer review or ask the contributor for a revision. The best implementation should preserve cheap fallback behavior for very large/streaming content, avoid eagerly parsing and mounting hundreds of thousands of characters, keep expansion state in Lit/reactive state keyed to stable message identity, place controls outside masked content, handle streaming-to-complete transitions predictably, and add regression coverage for long chat messages.
