## Summary

Keep this PR open. Current main still lacks progressive disclosure for long Control UI markdown messages, and the PR remains potentially useful Web UI work, but its current head still needs maintainer review for long-message performance and toggle/layout correctness before it could merge.

## Best Possible Solution

Keep this PR open for focused Web UI maintainer review or contributor revision. The best implementation should preserve the cheap fallback for very large or actively streaming content, avoid parsing and mounting hundreds of thousands of characters by default, key expansion state to stable message identity in reactive state, keep controls outside masked content, handle streaming completion predictably, and add long-message regression tests and docs if behavior ships.
