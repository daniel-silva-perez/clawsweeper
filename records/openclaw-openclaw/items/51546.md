## Summary

Keep this PR open. Current main still does not implement progressive disclosure for long Control UI markdown messages, and the PR remains potentially useful but needs Web UI maintainer review and revision before it could merge.

## Best Possible Solution

Keep this PR open for focused Web UI maintainer review or contributor revision. The best implementation should preserve the cheap fallback for very large or streaming content, avoid eagerly parsing and mounting hundreds of thousands of characters, keep expansion state in reactive state keyed to stable message identity, place controls outside masked content, handle streaming-to-complete transitions predictably, and add regression coverage for long chat messages.
