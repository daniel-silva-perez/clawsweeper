## Summary

Keep this PR open. Current main has not implemented the requested switch to bge-m3 for the built-in local memory embedding default: the default is still embeddinggemma, the current test asserts that contract, and the public memory config docs still name embeddinggemma. The PR patch is stale because the implementation moved from `src/memory/embeddings.ts` into `packages/memory-host-sdk`, but the underlying product change remains unresolved rather than obsolete. The Greptile comment also identified a real docs follow-up that still applies to any refreshed version of this change.

## Best Possible Solution

Keep the PR open as an implementation candidate, but ask for a refreshed patch against the current memory-host-sdk implementation: update `packages/memory-host-sdk/src/host/embedding-defaults.ts`, replace the embeddinggemma regression assertion, update the memory docs/reference surfaces, and include maintainer-ready evidence for the bge-m3 default tradeoff plus reindex/dimension behavior. The security review should stay focused on the new default remote GGUF artifact and resource footprint.
