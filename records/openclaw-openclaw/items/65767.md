Keep this PR open. Current main still has the TaskFlow SQLite owner migration gap that the PR targets, the linked bug #65563 is still open, and the PR is a focused active fix candidate with regression coverage and no supply-chain-sensitive file changes.

This PR refactors the TaskFlow SQLite schema migration to backfill `owner_key` from legacy `owner_session_key`, rebuild `flow_runs` without the legacy column, and add regression tests for full legacy and partially migrated schemas.
