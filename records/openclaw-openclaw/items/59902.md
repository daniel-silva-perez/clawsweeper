## Summary

Keep open. Current main still makes the `/cron` History button an in-page state update only: it selects/loads the job runs, while the Run history card remains a normal card lower in the same page. I did not find scroll/focus behavior, a drawer/modal, or a canonical duplicate in the provided related-item context. This remains a valid recent Control UI UX enhancement.

## Best Possible Solution

Keep this issue open and implement the UX in the Control UI cron surface: give the Run history section a stable target and, after the History action loads runs, scroll/focus the section or replace the in-page behavior with a dedicated history panel/modal. Add a focused view test that proves clicking History provides visible navigation feedback, not only `onLoadRuns`.
