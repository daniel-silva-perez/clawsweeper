## Summary

Keep this PR open. Current main still lacks the requested service-marker stripping, but the proposed prefix block is too broad for the current product contract because it would also strip a documented doctor safety control from exec-spawned CLI commands. The linked bug discussion also leaves clean process exit unproven, so this needs maintainer review before merge.

## What This Changes

The PR updates the host exec environment security policy to block `OPENCLAW_SERVICE_*` and `OPENCLAW_SYSTEMD_UNIT`, regenerates the macOS Swift policy, and adds a sanitizer regression test that preserves `OPENCLAW_SHELL=exec` while removing gateway runtime markers.

## Best Possible Solution

Update the fix to remove only the internal runtime markers or explicitly preserve documented service controls such as `OPENCLAW_SERVICE_REPAIR_POLICY`, rebase onto the current derived policy/generator shape, and add regression coverage for marker stripping, repair-policy preservation, and the reported CLI command completing cleanly.

## Review Findings

Full review comments:

- **[P2] Avoid blocking documented service repair policy:** `src/infra/host-env-security-policy.json:140`
  - body: Adding `OPENCLAW_SERVICE_` to the blocked inherited prefixes also strips documented `OPENCLAW_SERVICE_REPAIR_POLICY=external`.
