## Summary

Keep open. Current main has meaningful adjacent fixes for the v2026.4.26 startup-stall class, including nonblocking bounded primary-model prewarm and OpenAI Codex model/auth hardening, but this issue reports a broader Debian/Codex slowdown with auth refresh failures, unknown-model fallback, gateway timeouts, and near-unusable responsiveness. The linked PR context says the prewarm fix may help only if this is the same ordering problem, so there is not enough evidence to close this as fully implemented.

## What This Changes

The issue reports a severe Debian npm-global Codex setup regression after upgrading from 2026.4.2 to 2026.4.26, with slow startup, laggy responsiveness, model fallback, Codex OAuth refresh errors, and gateway timeout logs.

## Best Possible Solution

Keep the issue open until maintainers can retest a current build on the affected Debian/Codex setup or obtain a fresh startup trace. If the slowdown remains, split the fix by measured phase: gateway prewarm/cache work in `src/gateway/server-startup-post-attach.ts`, Codex model catalog handling in `extensions/openai`, and OAuth refresh recovery in `src/agents/auth-profiles` only if the refreshed logs still implicate those paths.
