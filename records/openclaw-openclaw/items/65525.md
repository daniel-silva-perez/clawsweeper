## Summary

Keep this issue open. Current main still does not auto-propagate gateway `NODE_OPTIONS` or proxy env into MCP stdio children: stdio launch config derives `env` from the server config, and the transport spawns with MCP SDK safe defaults plus explicit server env only. An open unmerged PR (#65574) explicitly targets this issue, so the best maintainer path is to review or replace that PR rather than close the issue now.

## Best Possible Solution

Keep the issue open and review #65574 as the implementation candidate, or replace it with a narrower current-main fix that auto-propagates safe proxy env vars to MCP stdio children while preserving the `NODE_OPTIONS` startup-safety policy unless maintainers explicitly decide otherwise. The fix should include MCP stdio unit coverage and docs clarifying inherited versus configured env.
