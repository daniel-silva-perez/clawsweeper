## Summary

Keep this open. Current main still does not propagate gateway proxy or NODE_OPTIONS state into OpenClaw-launched MCP stdio children, and an open PR (#65574) explicitly references this issue with a proposed fix but is not merged. The issue should stay open pending review or replacement of that PR, especially because the NODE_OPTIONS part crosses the stdio startup-safety boundary.

## What This Changes

The issue requests that MCP stdio child processes inherit gateway proxy-related environment variables and NODE_OPTIONS, or that the required explicit configuration be documented.

## Best Possible Solution

Review, update, and land #65574 or a replacement that implements an explicit MCP stdio env propagation contract: proxy variables should inherit from the trusted gateway environment with user env precedence, NODE_OPTIONS handling should be deliberately allowed, restricted, or documented, and tests/docs should cover the final security boundary.

## Reproduction Assessment

Yes. A focused unit test can set parent process.env HTTP_PROXY/HTTPS_PROXY/NO_PROXY/NODE_OPTIONS, resolve and start a stdio MCP server, and assert the spawn env; current main omits those parent values unless provided through server env, and server env currently filters NODE_OPTIONS.

## Solution Assessment

Unclear until maintainer review of the NODE_OPTIONS boundary. The narrow code location is the stdio MCP launch/env assembly, but the safest implementation should use a vetted inherited-env helper rather than blindly copying process.env.
