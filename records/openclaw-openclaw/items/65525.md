## Summary

Current main still does not auto-propagate gateway proxy variables or NODE_OPTIONS into MCP stdio child processes. The stdio resolver forwards only configured server env, the transport spawns with MCP SDK safe defaults plus that env, and an open linked PR is still the implementation candidate, so this issue should stay open until that PR is reviewed, merged, or replaced.

## What This Changes

MCP stdio child processes should inherit gateway proxy-related environment variables and NODE_OPTIONS, or the configuration/docs should make explicit per-server env setup required.

## Best Possible Solution

Keep this issue open and review/land PR #65574 or replace it with a current-main PR that auto-propagates safe proxy env vars as low-priority defaults, preserves explicit per-server env precedence, makes an explicit NODE_OPTIONS policy decision, and adds MCP stdio tests plus docs clarifying inherited versus configured env.
