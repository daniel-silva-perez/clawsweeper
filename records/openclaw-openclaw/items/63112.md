## Summary

Keep open. Current main still lacks the cron CLI warning, the linked #63107 problem remains valid, and this PR is an active, focused implementation that should go through normal maintainer review and validation rather than be closed or replaced.

## What This Changes

The PR adds a shared shell-command detector, non-blocking stderr warnings in cron add/edit for main-session systemEvent payloads that look like shell commands, and focused CLI/shared tests.

## Best Possible Solution

Land a narrow cron CLI fix that warns before the gateway RPC for both add and edit, covers existing-main edits when --session is omitted, preserves current cron runtime semantics, and includes regression tests for matching and non-matching payloads.
