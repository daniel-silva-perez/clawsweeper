## Summary

Current main and the v2026.4.26 release already implement the requested `plugins.entries.memory-core.config.dreaming.model` override. The memory-core manifest accepts the field, the dreaming config resolver propagates it into phase execution config, Dream Diary narrative subagent runs receive that model, and tests cover both override forwarding and fallback to the session default model when the configured model is unavailable.

## Best Possible Solution

Close this as implemented by the shipped `dreaming.model` Dream Diary subagent override in v2026.4.26. Broader dreaming session cleanup or batching concerns should remain in the broader related tracking item rather than keeping this focused config-schema request open.
