## Summary

## What This Changes

This PR changes Discord explicit outbound target parsing to return the normalized target string instead of the bare parsed ID and adds regression tests for explicit DM and bare numeric recipients.

## Best Possible Solution

Close this PR as already implemented and shipped; keep the current normalized-target behavior on main, and use #62024 only for any remaining user reports that still reproduce on `v2026.4.26` or newer.

Yes. The source-level reproduction is to parse an explicit Discord `user:<id>` target through `discordPlugin.messaging.parseExplicitTarget`; current main returns `to: "user:<id>"`, and the regression test covers that path plus mention, channel, and bare numeric defaults.

## Solution Assessment
