## Summary

Keep open. Current main has partial loop-time context-engine assembly, but it is only wired for active engines with `ownsCompaction === true`; non-owning engines still use the generic tool-result guard plus the single pre-prompt assembly. The requested public `transform_context`-style hook is still absent from the plugin hook surface and docs.

## What This Changes

Request a per-LLM-call context transform or `ContextEngine.assemble` invocation so context engines can rewrite the full message array during multi-call tool loops.

## Best Possible Solution

Define a stable, documented per-model-call context rewrite contract in the plugin/context-engine API, preserve existing context-engine compatibility, and add regression coverage for multi-tool-call loops across owning and non-owning engines.
