## Summary

Keep open. Current main has partial per-loop context-engine support, but it does not fully implement the requested API/behavior: `assemble` is re-run inside the tool loop only for context engines with `info.ownsCompaction === true`, while non-owning engines still get the generic tool-result guard and the one pre-prompt assembly. The proposed public `transform_context` hook from the related closed, unmerged PR is also absent from current main.

## Best Possible Solution

Keep this issue open for a maintainer/product decision. A conservative implementation would add a documented per-LLM-call context transform API, or broaden the current context-engine loop assembly contract beyond `ownsCompaction === true`, with tests proving the hook fires before each model request in a multi-tool-call loop and documenting compatibility with existing `assemble` semantics.
