## Summary

Keep open. Current main still does not provide the requested public per-LLM-call full-message rewrite hook: `assemble` runs once in the pre-prompt attempt setup, and the only loop-time `assemble` wrapper is internal and limited to context engines with `ownsCompaction: true`. The related transform-context PR was closed unmerged, so the remaining work is a plugin/context-engine API decision rather than cleanup that can be closed.

## What This Changes

The issue requests a per-LLM-call context rewrite or assembly surface so context engines or plugins can inspect and modify full message history inside multi-call agentic loops.

## Best Possible Solution

Define a stable, documented per-model-call context rewrite contract for plugin/context-engine authors, preserve existing `ContextEngine.assemble` compatibility unless maintainers explicitly choose a semantics change, and add regression coverage for owning engines, non-owning engines, and the public hook surface across multi-tool-call loops.

## Reproduction Assessment

Yes for the code-level API gap: current main shows single pre-prompt assembly, an internal loop wrapper only for `ownsCompaction` engines, and no public `transform_context` hook. A live multi-call LLM run would add end-to-end proof, but the source and focused tests are enough to reproduce the missing surface.

## Solution Assessment

Unclear. The additive hook direction from PR #57585 appears safer than changing `assemble` frequency because it can preserve existing context-engine semantics, but hook ordering, message validation, prompt-content access, and budget-enforcement guarantees need maintainer API review before implementation.
