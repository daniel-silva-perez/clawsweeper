## Summary

Keep open. Current main still lacks the requested public per-LLM-call full-message rewrite surface: `assemble` runs in pre-prompt attempt setup, and the loop-time wrapper only applies to context engines with `ownsCompaction: true`; the related `transform_context` PR is closed unmerged.

## What This Changes

The issue requests that `ContextEngine.assemble` or an equivalent hook run before each LLM call in an agentic tool loop rather than only once per user prompt.

## Best Possible Solution

Define a stable per-model-call context rewrite contract in the plugin/context-engine API, preferably additive unless maintainers intentionally change `assemble` semantics, then document hook ordering and add regression coverage for owning engines, non-owning engines, and multi-tool-call loops.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: current main shows pre-prompt assembly, an internal owns-compaction-only loop wrapper, and no public `transform_context` hook; a live LLM run would only add end-to-end confirmation.

## Solution Assessment

Unclear. The additive hook direction from PR #57585 appears safer than changing `assemble` frequency, but maintainers should first decide prompt-content access, hook ordering, message validation, and budget-enforcement guarantees.
