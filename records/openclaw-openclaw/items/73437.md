## Summary

Keep open. Current main still only partially covers loop-time context assembly: owning context engines are wrapped through an internal `transformContext` loop hook, but non-owning engines keep the generic tool-result guard and the single pre-prompt `assemble` path. The requested public per-LLM-call context rewrite hook is still absent, and the related implementation PR was closed unmerged.

## What This Changes

The issue requests that `ContextEngine.assemble()` or an equivalent `transform_context` hook run before every LLM call inside the agentic loop rather than only once before `activeSession.prompt()`.

## Best Possible Solution

Define a stable public per-model-call context rewrite contract for plugin/context-engine authors, reconcile the context-engine docs with runtime behavior, preserve compatibility for existing engines, and add focused regression coverage for owning engines, non-owning engines, and the new public hook or lifecycle semantics.
