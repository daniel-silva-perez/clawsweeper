## Summary

Keep open. Current main still only partially satisfies the request: attempt-level context-engine assembly runs before the agentic loop starts, and the loop-time assembly wrapper is gated to context engines that own compaction. There is still no public `transform_context` hook or equivalent full-message rewrite API for every LLM call boundary, and the related implementation PR was closed unmerged.

## What This Changes

The issue asks for context-engine assembly, or an equivalent `transform_context` hook, to run at each LLM call boundary inside the agentic loop so context engines/plugins can rewrite full message context during tool loops.

## Best Possible Solution

Define a stable, documented per-model-call context rewrite contract for the plugin/context-engine API, preserve existing `ContextEngine` compatibility, and add regression coverage for multi-tool-call loops across owning engines, non-owning engines, and the public hook surface.

## Reproduction Assessment

Yes for the code-level behavior. Current main shows attempt-level assembly before `activeSession.prompt(...)`, an internal loop wrapper only for `ownsCompaction` engines, and no public `transform_context` hook; focused tests are enough to reproduce the API gap without a live LLM run.

## Solution Assessment

Unclear. The additive hook direction from the related closed PR looks safer than changing `assemble` frequency, but hook ordering, prompt-content access, budget enforcement, and compatibility need maintainer API review first.
