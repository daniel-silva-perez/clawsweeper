## Summary

Keep open for normal maintainer review and CI before merge. Current main includes the browser-local tweakcn importer but the Control UI CSP still blocks the import fetch; this PR is a narrow, appropriate CSP allowlist update with no blocking review findings.

## What This Changes

The PR adds `https://tweakcn.com` to the Control UI CSP `connect-src` directive and asserts that header in the Control UI HTTP security-header test.

## Best Possible Solution

Land this exact-host `connect-src` allowlist, or an equivalent change, after CI is green. Broader user-configurable CSP behavior should remain separate under #47565 rather than blocking this built-in tweakcn importer fix.

## Reproduction Assessment

Yes. Current main fetches `https://tweakcn.com/r/themes/<id>` from the browser importer while serving a CSP whose `connect-src` lacks that origin; the live endpoint responds with permissive CORS, so the browser CSP denial is the high-confidence reproduction path.

## Solution Assessment

Yes. Adding only `https://tweakcn.com` to `connect-src` is the narrowest maintainable fix for the built-in importer because it does not relax script, style, image, dependency, or workflow surfaces.
