## Summary

Keep open. Current main has partial safeguards for timeout and reasoning-only empty turns, but it still permits dispatches that complete with no queued final reply and no visible delivery, and it intentionally preserves documented silent-reply behavior for `NO_REPLY`, groups/channels, and tool-only delivery modes.

## What This Changes

The issue asks OpenClaw to notify users whenever an agent run finishes with no replyable payload, including fallback error text and an opt-in `dispatch.requireReply`-style policy.

## Best Possible Solution

Define an explicit opt-in require-reply policy at dispatch or surface scope that can force a visible fallback for operators who want it while preserving current silent defaults. Keep the narrower provider/tool-result empty-final repair tracked separately through #46080 unless maintainers decide to fold that case into the broader policy.
