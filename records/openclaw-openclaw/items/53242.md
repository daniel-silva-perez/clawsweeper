## Summary

Keep open. Current main still truncates default chat.history text fields before the larger byte-budget checks, and the Control UI/WebChat request path does not pass a larger maxChars override. The provided GitHub context also shows same-author PR #53243 remains open and unmerged with a closing keyword for this issue, so the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review or land PR #53243, or an equivalent gateway fix. The maintainer path should raise or derive the default chat-history text cap from the byte budget, preserve long assistant text that fits the configured history budget, add a default-path regression test for Control UI/WebChat, handle UTF-8-heavy text byte-aware, and update config help/generated docs to match the shipped behavior.
