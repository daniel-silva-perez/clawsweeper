## Summary

Keep open. Current main still truncates `chat.history` text fields on the default Control UI/WebChat path before the larger byte-budget checks, and the same-author implementation candidate #53243 remains open and unmerged with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review or land PR #53243, or an equivalent gateway fix. The fix should raise or derive the default display text cap from the byte budget, keep long assistant messages that fit the configured history budget, handle UTF-8-heavy text without dropping readable content, and add a default-path regression test for the Control UI/WebChat request flow.
