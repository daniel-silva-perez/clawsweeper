## Summary

Keep this PR open. Current main has shipped the shared inbound `message_sending` before-delivery path, but WhatsApp auto-reply delivery still does not emit plugin `message_sent` or internal `message:sent` hooks, which is unique remaining work from this PR.

## Best Possible Solution

Keep this PR open, rebase it onto current main, and narrow it around WhatsApp auto-reply sent-hook parity. The implementation should reuse or extract the canonical sent-hook emitter where possible, emit plugin `message_sent` and internal `message:sent` only from accurate delivered or failed outcomes, and add focused WhatsApp tests for cancellation, rewrites, blank content, media fallback, chunking, and delivery errors.
