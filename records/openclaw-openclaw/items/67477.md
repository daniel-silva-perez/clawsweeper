## Summary

Keep this PR open. Current main implements the shared inbound `message_sending` before-delivery path, but WhatsApp auto-reply delivery still does not emit plugin `message_sent` or internal `message:sent` hooks, and it does not pass the session/account/conversation/group metadata that this PR adds for sent-hook parity.

## Best Possible Solution

Keep this PR open, rebase it onto current main, and narrow it around WhatsApp auto-reply `message_sent` / `message:sent` parity. The implementation should reuse or extract the canonical sent-hook emitter where practical, pass accurate session/account/conversation/group metadata, report success only for actual delivered outcomes, and add focused WhatsApp tests for cancellation, rewrites, blank content, chunking, media fallback, and delivery errors.
