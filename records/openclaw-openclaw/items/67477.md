## Summary

Keep this PR open. Current main only covers the shared `message_sending` before-delivery hook path; WhatsApp auto-reply delivery still lacks plugin `message_sent` and internal `message:sent` emission, so the PR is not obsolete. The branch should not merge as-is because its local hook handling can mark canceled or rewritten replies as successfully sent with the wrong remembered text.

## What This Changes

The PR adds WhatsApp auto-reply `message_sending` and `message_sent` hook handling, threads session/account/conversation metadata into delivery, exposes `getGlobalHookRunner` from `hook-runtime`, and adds focused delivery tests.

## Best Possible Solution

Rebase and narrow the PR around the still-missing WhatsApp sent-hook parity: keep current main’s shared `message_sending` `beforeDeliver` behavior, emit plugin `message_sent` and internal `message:sent` from actual WhatsApp delivery outcomes, and make the delivered/canceled/rewritten text flow back into dispatcher state and echo/history tracking. Tests should cover cancel, rewrite, blank content, chunking, media fallback, and delivery errors.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Propagate hook cancellations to the dispatcher:** `extensions/whatsapp/src/auto-reply/deliver-reply.ts:92-93`
  - body: When `message_sending` returns `cancel`, this function returns normally. The caller then sets `didSendReply = true` and remembers the original text, so canceled WhatsApp replies are recorded as sent and can pollute echo suppression/history.
  - confidence: 0.94
- **[P1] Return rewritten content before remembering sent text:** `extensions/whatsapp/src/auto-reply/deliver-reply.ts:95-96`
  - body: A hook rewrite only updates the local `replyResult`; `dispatchWhatsAppBufferedReply` still calls `rememberSentText(payload.text, ...)` with the pre-hook payload. That records text that was not delivered and can break self-echo suppression for rewritten replies.
  - confidence: 0.92
- **[P2] Avoid success hooks when no message is delivered:** `extensions/whatsapp/src/auto-reply/deliver-reply.ts:309`
  - body: If a hook rewrites a text-only reply to an empty string, there are no chunks and no media, but the media path falls through and calls `emitMessageSentHooks(true)`. Hook consumers would see a successful send even though WhatsApp received nothing.
  - confidence: 0.88
- **[P2] Include sending-hook errors in failure telemetry:** `extensions/whatsapp/src/auto-reply/deliver-reply.ts:77`
  - body: `runMessageSending` executes before the `try` block that emits failed `message_sent`/`message:sent` events. A plugin error in that hook aborts delivery without the failure signal this PR is trying to add.
  - confidence: 0.86
