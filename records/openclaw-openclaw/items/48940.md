## Summary

Keep this PR open. Current main still does not implement the central change: a gateway-owned, node-backed ACP runtime with durable gateway state, ACP worker node commands/events, durable replay projection, and startup recovery. The PR remains substantial unmerged product work, not conservative cleanup material.

## Best Possible Solution

Keep this PR open unless a maintainer manually closes or replaces it. The best path is a rebased, reviewable split: durable gateway ACP store/protocol first, node command ingress and worker bridge second, durable projection/replay plus startup recovery third. Resolve or explicitly disposition the P1 review findings, perform a focused security review of the expanded node command/event surface, and then run the ACP/gateway/node targeted tests plus the appropriate changed gate before landing.
