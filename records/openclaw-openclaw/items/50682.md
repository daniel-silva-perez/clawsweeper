## Summary

Keep this PR open. Current main at 6d323ee73640afdf02ed9fa3aded13da23359b98 still does not implement Telegram `unknownGroupAction`, `unknownGroupMessage`, `unknownGroupCooldownMs`, warning cooldowns, or automatic leave behavior. The PR remains a plausible Telegram plugin feature candidate, but the review discussion identifies real correctness issues that should be resolved before merge.

## Best Possible Solution

Keep this PR open for maintainer review rather than closing it as obsolete. If accepted, land the feature inside the Telegram plugin with public config type/schema/docs coverage, a shared unknown-group handler for normal messages and native commands, precise unknown-vs-configured group classification, `buildTelegramThreadParams` for forum warnings, bounded cooldown state, validation or diagnostics for `warn` without a message, and targeted tests for ignore/warn/leave, wildcard groups, native commands, callbacks, and General-topic forums.

## Risks / Open Questions

- Merging as-is could warn or leave wildcard-configured Telegram groups if `group-policy-allowlist-empty` is treated as an unknown-group block.
