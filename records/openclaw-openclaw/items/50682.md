## Summary

Keep this PR open. Current main at 230f7122dd5cd8a37a55e671254bd9be8278f6e8 still does not implement Telegram unknown-group actions, warning messages, cooldowns, or leave behavior. The PR remains a plausible Telegram-plugin implementation candidate, but the supplied review discussion identifies correctness issues that should be resolved before merge, especially around `leaveChat` classification and native command handling.

## Best Possible Solution

Keep this PR open for maintainer review. If accepted, implement the feature inside the Telegram plugin with public config type/schema/docs coverage, a shared unknown-group handler used by normal messages and native commands, precise unknown-vs-configured group classification, `buildTelegramThreadParams` for forum warnings, bounded cooldown state, validation or startup warning for `warn` without a message, and targeted tests for ignore/warn/leave, wildcard groups, commands, callbacks, and General-topic forums.

## Risks / Open Questions

- Merging as-is could make the bot warn or leave configured wildcard-enabled groups if `group-policy-allowlist-empty` is treated as an unknown-group block.
- Slash/native command messages can remain outside the unknown-group action path unless the command authorization path shares the same handling.
- Telegram forum General-topic warnings need the existing thread-param helper behavior; sending `message_thread_id=1` directly can fail before a leave action.
- The PR's cooldown tracker and `warn` without message behavior need bounded state or validation/startup diagnostics before this is safe operationally.
