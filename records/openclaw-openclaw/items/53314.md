## Summary

Keep open. Current main does not implement this PR's intended fix: the descendant gate still returns plain `false`, the lifecycle caller still treats that as an ordinary failed announce, and cleanup still re-counts descendants before deciding whether to defer or increment retry state. The PR is a plausible focused bug fix, but the latest review context also flags remaining edge cases that should be resolved before merge.

## Best Possible Solution

Keep this PR open and ask for a revision or maintainer-owned equivalent fix. The best path is to carry an explicit pending-descendants result through `runSubagentAnnounceFlow` into lifecycle cleanup, defer without incrementing `announceRetryCount`, preserve completion hard-expiry behavior, mirror normal finalize-error recovery, and add a regression test where the inner descendant gate blocks but the outer cleanup re-count sees zero.
