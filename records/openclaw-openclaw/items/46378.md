## Summary

Current main still routes first-run onboarding through the Clack-backed interactive security confirmation on Windows, with no Windows-specific or plain line-input fallback found. Native Windows CLI setup is still documented as supported with caveats, and the open Microsoft/Windows tracker #74163 still lists this issue unchecked, so there is not enough evidence to close it as stale, fixed, duplicate, or out of scope.

## Best Possible Solution

Keep this open for Windows maintainer reproduction. The best fix is to reproduce `openclaw onboard` or first-run setup on Windows 11 in the affected terminal, then either add a narrowly scoped Windows/plain-input fallback for the security confirmation or document a specific unsupported terminal path, while preserving the explicit `acceptRisk` acknowledgement. The open Microsoft/Windows tracker #74163 can carry prioritization, but this issue should stay open until a concrete fix or documented support decision lands.
