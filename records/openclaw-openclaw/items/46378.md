## What This Changes

The issue reports that Windows 11 installation/onboarding freezes at the security acknowledgement prompt after installing via the shell installer path.

## Best Possible Solution

Reproduce first-run onboarding on Windows 11 in the affected terminal path, then add the narrowest prompt-layer fallback or installer/docs guidance that keeps the security acknowledgement explicit. If the failure is limited to an unsupported install.sh-on-native-Windows shell path, clarify install.ps1 or WSL2 guidance instead of changing all prompt behavior.

## Solution Assessment

Unclear. A Windows/plain-input fallback in the onboarding prompter may be the best fix if Clack keypress handling is confirmed broken, but installer guidance is safer if the failure is specific to an unsupported shell path.
