## Summary

Keep this PR open. Current main still does not expose or enforce the requested Feishu `replyToMode` config at top-level, account, or group scope. The PR discussion remains relevant: the author addressed the initial `first`-mode review, but a later review still flags a streaming-card path that may double-consume reply targets, so this is useful reviewable work rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer or contributor follow-up. The best path is a rebased Feishu-owned implementation that adds `replyToMode` at the stated top-level, per-account, and per-group scopes; applies one policy across streaming cards, text/card chunks, media sends, and typing behavior; updates manifest/generated metadata and docs; and adds regression coverage for `off`, `first`, and `all`.
