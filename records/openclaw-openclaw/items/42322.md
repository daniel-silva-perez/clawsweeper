## Summary

Keep this PR open. Current main still lacks the Feishu `replyToMode` public config surface and runtime policy (`off` / `first` / `all`) requested by the PR. Main has related Feishu reply-targeting work, but it does not implement this PR’s intended change, and there is no merged or open canonical replacement in the provided context.

## Best Possible Solution

Keep this PR open for maintainer or contributor follow-up. The best path is a rebased Feishu-owned implementation that adds `replyToMode` at the stated top-level, per-account, and per-group scopes; defines precedence with `replyInThread`, topic sessions, `reply_target_message_id`, and `suppress_reply_target`; applies one policy across streaming, text/card chunks, media sends, and typing behavior; updates manifest/generated metadata/docs; and adds regression coverage for `off`, `first`, and `all`.
