## Summary

Keep this PR open. Current main at 0f3a9d812b40cbc8e545abcb80a1f12a93946cf6 still does not implement the Feishu `replyToMode` config surface or runtime policy, and this PR remains a focused implementation candidate rather than obsolete cleanup. The PR discussion is useful: the author addressed the initial unimplemented `first` review, but a later review still flags streamed-card plus media/text inconsistency that should be resolved before merge. Security pass found no supply-chain-sensitive files in the provided three-file diff, but the behavior needs docs, generated metadata, manifest, and regression coverage before landing.

## Best Possible Solution

Keep this PR open for maintainer or contributor follow-up. The best path is a rebased Feishu-owned implementation that adds `replyToMode` at top-level, per-account, and per-group scopes; applies one reply-target policy across streaming cards, text/card chunks, media sends, and typing behavior; updates manifest/generated metadata and public Feishu docs; and adds regression tests for `off`, `first`, `all`, and precedence with existing `replyInThread` behavior.
