## Summary

Keep this PR open. Current main at e476523082a6fff3a0c3048077d2f0e52413fbf2 still does not expose or enforce Feishu `replyToMode`, and the PR remains a focused implementation candidate rather than obsolete cleanup. It still needs maintainer review, a rebase, generated metadata/docs, and regression coverage, especially around streamed-card plus media/text delivery for `replyToMode: "first"`.

## Best Possible Solution

Keep this PR open for maintainer or contributor follow-up. The best path is a rebased Feishu-owned implementation that adds `replyToMode` at top-level, per-account, and per-group scopes; applies one reply-target policy across streaming cards, text/card chunks, media sends, and typing behavior; updates manifest/generated metadata and docs; and adds regression tests for `off`, `first`, and `all`.
