## Summary

Keep this PR open. Current main has related Feishu reply-threading work, but it still does not implement this PR's stated public `replyToMode` config surface or the `off` / `first` / `all` runtime policy for Feishu. The PR remains a focused contributor implementation candidate and needs maintainer review or a rebased replacement rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer or contributor follow-up. The best path is a rebased Feishu-owned implementation that intentionally adds `replyToMode` at the documented top-level, per-account, and per-group scopes; defines precedence against `replyInThread`, topic sessions, `reply_target_message_id`, and `suppress_reply_target`; applies one reply-target policy across streaming, text/card chunks, media sends, and typing behavior; updates manifest/generated metadata/docs; and adds regression coverage for `off`, `first`, and `all`.
