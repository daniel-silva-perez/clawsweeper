Keep this PR open. Current main still does not implement the core `rescueWatchdog` cron payload, protocol schema, runner, or gateway cron hook, and the open related watchdog/docs PRs do not supersede this core service-layer proposal.

## What This Changes

This PR adds daemon/service hardening, abortable managed-service restart paths, a new isolated `rescueWatchdog` cron payload and runner, gateway cron wiring, probe support, and focused tests for the rescue watchdog flow.

## Best Possible Solution

Rebase this PR or replace it with a narrower core PR that preserves the same essential behavior: an isolated `rescueWatchdog` payload, validated cron storage/schema support, a bounded watchdog runner, gateway cron wiring, service-repair behavior, and focused tests.
