## Summary

Keep open. Current main still derives Gateway pricing bootstrap enablement only from `models.pricing.enabled`, so `models.mode: "replace"` alone still reaches the startup pricing refresh path. The linked implementation candidate #74038 is open and unmerged, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Gateway startup should skip OpenRouter/LiteLLM pricing fetches when `models.mode` is `replace`, unless pricing is explicitly opted back in.

## Best Possible Solution

Land or replace the focused #74038-style fix at the shared pricing enablement boundary: derive bootstrap enablement from both `models.pricing.enabled` and `models.mode`, keep explicit configured model costs usable, preserve the chosen explicit opt-in semantics, and update regression tests plus schema/help/docs together.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with `models.mode: "replace"` and no `models.pricing.enabled: false`, the helper still returns true, so startup and refresh can proceed into OpenRouter/LiteLLM pricing fetches for configured external refs; no tests were run because this review was required to stay read-only.

## Solution Assessment

Yes. The best fix is the shared pricing enablement/default helper used by startup and refresh, not a one-off guard in only one caller; #74038 appears to target that boundary and should be reviewed, rebased, landed, or replaced with equivalent coverage.
