## Summary

Keep open. Current main still lets replace-mode configs enter the gateway pricing bootstrap path, and #74038 is already open as the linked implementation candidate, so this issue should remain open until that PR lands or maintainers choose an equivalent narrow fix.

## What This Changes

The issue asks for Gateway startup to skip OpenRouter/LiteLLM pricing catalog fetches when `models.mode` is `replace` unless pricing is explicitly opted back in.

## Best Possible Solution

Review and land #74038 or an equivalent focused fix that derives gateway pricing enablement from both `models.pricing.enabled` and `models.mode`, defaults pricing bootstrap off for replace mode, preserves an explicit true opt-in if accepted, and updates regression tests plus config/help/docs surfaces together.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with `models.mode: "replace"` and no `models.pricing.enabled: false`, current main's helper still returns true, so startup and refresh can proceed to OpenRouter/LiteLLM catalog fetches for configured external refs. I did not run tests because this review was required to stay read-only.

## Solution Assessment

Yes. The narrowest maintainable solution is to fix the shared pricing enablement/default boundary so both gateway startup and cache refresh observe replace mode; #74038 appears to target that path and should be reviewed rather than opening another repair lane.
