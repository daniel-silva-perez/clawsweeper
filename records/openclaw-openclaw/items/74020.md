## Summary

Keep open. Current main still enables the gateway pricing bootstrap solely from `models.pricing.enabled`, so `models.mode: "replace"` by itself can still reach the OpenRouter/LiteLLM pricing fetch path; the open linked implementation PR should be reviewed or replaced before this issue is closed.

## What This Changes

The issue requests that Gateway startup skip remote pricing catalog fetches when `models.mode` is `replace` and models are manually configured.

## Best Possible Solution

Land #74038 or an equivalent focused gateway/config change that derives pricing-bootstrap enablement from both `models.pricing.enabled` and `models.mode`, updates startup/cache regression tests, and keeps schema help plus user docs aligned with the chosen explicit-override semantics.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with `models.mode: "replace"` and no `models.pricing.enabled: false`, the current helper still returns true, allowing startup and refresh to proceed into remote pricing fetches for external refs. No tests were run because this review was required to stay read-only.

## Solution Assessment

Yes. The narrow maintainable fix belongs at the shared pricing enablement boundary used by startup and refresh, not as a one-off guard in a single caller; #74038 appears to target that boundary and should be reviewed or replaced with an equivalent patch.
