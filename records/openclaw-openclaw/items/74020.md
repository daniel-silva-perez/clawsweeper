## Summary

Keep open. Current main still derives gateway pricing enablement only from `models.pricing.enabled`, so `models.mode: "replace"` still reaches the startup pricing bootstrap path; the linked implementation candidate #74038 is open and unmerged, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that gateway startup skip remote OpenRouter and LiteLLM model-pricing fetches when `models.mode` is `replace`.

## Best Possible Solution

The end state should be a focused gateway/config change that derives pricing bootstrap enablement from both `models.pricing.enabled` and `models.mode`, defaults remote pricing bootstrap off for replace mode, preserves whichever explicit opt-in semantics maintainers choose, and updates regression tests plus schema/help/docs together.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with `models.mode: "replace"` and no `models.pricing.enabled: false`, current main's helper still returns true, so startup and refresh can proceed into OpenRouter/LiteLLM pricing fetches for configured external refs; no tests were run because this review was required to stay read-only.

## Solution Assessment

Yes. The best fix is at the shared pricing enablement/default boundary used by startup and refresh, not a one-off guard in only one caller; #74038 appears to target that boundary and should be reviewed or replaced with an equivalent focused fix.
