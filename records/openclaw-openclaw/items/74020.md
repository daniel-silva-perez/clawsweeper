## Summary

Keep open. Current main still derives Gateway pricing bootstrap enablement only from `models.pricing.enabled`, so `models.mode: "replace"` can still load the pricing refresh path and fetch OpenRouter/LiteLLM catalogs; open PR #74038 is already linked with closing syntax and should be reviewed or replaced before this issue closes.

## What This Changes

Request that Gateway startup skip remote OpenRouter/LiteLLM pricing fetches when `models.mode` is `replace`, while preserving manually configured model pricing.

## Best Possible Solution

Review and land PR #74038 or an equivalent narrow change that makes omitted `models.pricing.enabled` default to disabled under `models.mode: "replace"` and enabled otherwise, preserves explicit `enabled: true`, updates schema/help/docs/changelog surfaces, and adds regression coverage for both startup import and cache refresh behavior.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: call `startGatewayRuntimeServices` or `refreshGatewayModelPricingCache` with `models.mode: "replace"` and no `models.pricing.enabled: false`; the pricing helper still evaluates enabled and the remote catalog fetch path remains reachable.

## Solution Assessment

Yes, the linked PR direction appears to be the narrowest maintainable solution if maintainers accept the explicit opt-in boundary. It keeps the existing pricing toggle as the override and changes only the default derived from replace mode, rather than changing provider/catalog resolution semantics more broadly.
