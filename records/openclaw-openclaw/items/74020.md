## Summary

Keep open. Current main still enables the gateway pricing bootstrap solely from `models.pricing.enabled`, so `models.mode: "replace"` does not skip the OpenRouter/LiteLLM pricing fetch path. PR #74038 is open with closing syntax for this issue, so the issue should remain open until that PR lands or maintainers choose a different fix.

## Best Possible Solution

Keep this issue open and review PR #74038. If maintainers accept that boundary, land it so `models.mode: "replace"` disables remote pricing bootstrap by default while preserving explicit pricing opt-in behavior; otherwise close that PR and implement the same narrow guard with regression coverage in the startup and direct refresh paths plus config-help/docs updates.
