## Summary

Keep this PR open for maintainer review. Current main still has the reported strict-agentic retry gaps, and the PR is a focused implementation with regression coverage; I did not find a blocking correctness or security issue in the proposed diff.

## What This Changes

The PR broadens strict-agentic planning-only retry prompt and narration heuristics, tightens the retry instruction text, adds focused runner regression tests, and removes an unnecessary Codex plugin test cast.

## Best Possible Solution

Land this PR or an equivalent narrow runner-helper fix after maintainer review and validation, keeping the behavior scoped to strict-agentic-compatible runs and preserving the new negative tests for casual chatter and pass declinations.

## Reproduction Assessment

Yes. The helper-level reproduction is high confidence from current main source: the reported imperative prompt fails the current actionable prompt regexes, and present-continuous assistant text fails the planning-only detector because no active-narration branch exists.

## Solution Assessment

Yes, subject to normal validation. Updating the central resolver heuristics with colocated regression tests is the narrowest maintainable fix for these two strict-agentic gaps; a broader policy rewrite would be riskier and needs separate product review.
