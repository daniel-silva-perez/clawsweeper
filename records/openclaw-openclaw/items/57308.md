## Summary

Keep this PR open. Current main still has the reported Slack liveness gap: block actions and modal lifecycle handlers enqueue session-scoped system events but do not request a heartbeat wake, and the existing tests only assert enqueue behavior. The PR is the canonical rebased follow-up to #56766, has an active maintainer update preserving the contributor path, and its patch is narrowly aligned with the root cause from #28047.

## Best Possible Solution

Keep this PR as the canonical implementation candidate, rebase or update it against latest main if needed, then review and land it with the included regression coverage. The product fix belongs in the Slack plugin interaction handlers: after a system event is actually queued, request an exec-event heartbeat for the same routed sessionKey, and keep the SDK docs/export updates aligned with that public helper surface.
