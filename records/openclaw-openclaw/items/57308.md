## Summary

Keep this PR open. Current main still has the Slack interaction liveness gap: block actions and modal lifecycle handlers enqueue session-scoped system events but do not request a heartbeat wake, while this PR adds the missing guarded wake calls and regression coverage. The PR is open, unmerged, and has a maintainer comment saying ProjectClownfish pushed a narrow repair so this contributor path can stay canonical.

## Best Possible Solution

Keep this PR open as the canonical contributor-path implementation candidate, update it against current main, reconcile the overlapping #61374 block-action/thread-routing work, then land the Slack plugin fix with regression coverage. The implementation should wake the same routed session only after enqueueSystemEvent reports that the interaction event was actually queued, and keep the heartbeat-runtime SDK surface and docs aligned.
