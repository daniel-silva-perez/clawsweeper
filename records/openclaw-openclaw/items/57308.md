## Summary

Keep this PR open. Current main at 955b4df0939f90b4b6de5e7e6f3149dd125e1e50 still enqueues Slack block-action and modal lifecycle system events without requesting a heartbeat wake for the routed session, so the PR's central liveness fix is not implemented. The PR also remains the maintained contributor-path candidate in the discussion, while #61374 overlaps only part of the block-action work and does not cover modal lifecycle wakes.

## Best Possible Solution

Keep this PR open as the canonical contributor-path candidate. Refresh it against current main, reconcile the overlapping #61374 block-action/thread-routing changes, then land a focused implementation that wakes the same routed Slack session only when enqueueSystemEvent returns true, with regression coverage for block actions, modal submissions, and modal closes plus aligned Plugin SDK exports/docs.
