## Summary

Keep this PR open. Current main still persists `systemSent: true` inside `ensureSkillSnapshot()` before the LLM call, and I did not find a production post-success `systemSent` writer in the normal reply or followup paths. The PR is not obsolete or implemented on main.

## Best Possible Solution

Keep this PR open. The best maintainer path is to separate skill snapshot persistence from delivered-system-prompt state, remove the pre-response `systemSent` write from `ensureSkillSnapshot()`, and persist `systemSent: true` through a shared post-success helper used by normal replies and queued/final followups. Add targeted regression coverage for failed first call, successful retry with `isNewSession=false`, payloadless or `NO_REPLY` success, messaging-tool-only success, and followup success.
