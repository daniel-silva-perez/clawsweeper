## Summary

Keep this PR open. Current main still persists `systemSent: true` inside `ensureSkillSnapshot()` before the LLM run, and the latest release tag has the same behavior, so this PR is not obsolete. The PR is a focused fix for the session corruption path from #41462, but it still needs maintainer review for the successful queued/final followup paths and the exact success predicate before landing.

## Best Possible Solution

Keep this PR open. The best path is a corrected core-session fix: remove `systemSent` from the `ensureSkillSnapshot()` disk write, add a shared post-success persistence helper used by normal replies and followup/final queued completions, persist on a direct successful model-response or outbound-side-effect signal, skip provider/meta-error outcomes, and add focused regression tests for failed first call, successful retry with `isNewSession=false`, payloadless success, messaging-tool success, and queued/final followup success.
