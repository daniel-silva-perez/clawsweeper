## Summary

Keep this PR open. Current main still persists `systemSent: true` inside `ensureSkillSnapshot()` before the model run, and the normal reply plus followup success paths still do not have an equivalent post-success `systemSent` persistence helper. The latest PR update appears relevant, but review comments still identify follow-up needed before landing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a shared post-success `systemSent` persistence helper that removes the pre-run disk write from `ensureSkillSnapshot()`, writes best-effort after successful normal and followup model runs, handles silent and messaging-tool successes, skips errors/aborts, uses the active session entry after resets, and keeps regression coverage for failed first calls and successful retries.
