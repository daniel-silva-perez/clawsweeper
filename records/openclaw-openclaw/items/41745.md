## Summary

Keep this PR open. Current main still persists `systemSent: true` inside `ensureSkillSnapshot()` before the model run, and there is no production post-success `systemSent` writer in the reply or followup runner paths. The PR remains a focused implementation candidate for #41462, with remaining maintainer review needed around successful payloadless/messaging-tool/final followup completions before landing.

## Best Possible Solution

Keep this PR open. The best path is to separate skill snapshot persistence from delivered-system-prompt state, then persist `systemSent: true` through a shared post-success helper used by normal replies, silent or payloadless completions, messaging-tool completions, and queued/final followup completions, while skipping provider/meta-error outcomes. Add targeted regression coverage for failed first call, successful retry with `isNewSession=false`, payloadless success, messaging-tool success, and queued/final followup success.
