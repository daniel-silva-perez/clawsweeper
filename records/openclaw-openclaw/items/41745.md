## Summary

Keep this PR open. Current main still persists `systemSent: true` inside `ensureSkillSnapshot()` before the model run, and I found no equivalent post-success `systemSent` persistence path in the normal reply or followup runners. The PR is still a focused candidate for the linked #41462 session-state corruption path, but the latest review comments identify follow-up needed before landing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a shared post-success `systemSent` persistence helper that removes the premature write from `ensureSkillSnapshot()`, applies consistently to normal replies and queued/final followups, logs and continues on store-write failures, uses the active session entry after resets, and keeps regression coverage for failed first calls, successful retries, silent successes, messaging-tool sends, and followup success.
