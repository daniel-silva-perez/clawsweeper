## Summary

Keep this PR open. Current main still persists `systemSent: true` during `ensureSkillSnapshot()` before the embedded model call can succeed, and I did not find the shared post-success persistence path that the PR is adding. The latest PR context also shows the author responded to prior review concerns by covering successful silent runs, messaging-tool sends, and queued/final followups, so the work is still relevant rather than obsolete.

## Best Possible Solution

Keep this PR open and review the latest head for merge. The best implementation direction is the PR's current shape: separate skill snapshot persistence from delivered-system-prompt state, keep `ensureSkillSnapshot()` returning an in-memory first-turn signal without writing `systemSent`, and persist `systemSent: true` through a shared post-success helper used by both normal replies and queued/final followups, with regression coverage for failed first calls, successful retries, silent successes, messaging-tool sends, and followup success.
