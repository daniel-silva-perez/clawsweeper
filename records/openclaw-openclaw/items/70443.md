## Summary

Keep open. Current main still lacks a `--offline` path for Docker setup reruns: `scripts/docker/setup.sh` still builds `openclaw:local` or pulls a non-local `OPENCLAW_IMAGE` before continuing, and sandbox setup still builds `openclaw-sandbox:bookworm-slim` when enabled. The related `OPENCLAW_DOCKER_PLATFORM` items are separate cross-platform build work, not a duplicate or superseding offline rerun implementation.

## Best Possible Solution

Keep this issue open and implement the narrow `--offline` flag in `scripts/docker/setup.sh`. The flag should be Bash 3.2-compatible, skip main build/pull only after `docker image inspect "$OPENCLAW_IMAGE"` succeeds, fail clearly with `docker load` guidance when the main image is missing, warn rather than fail for a missing optional sandbox image, and leave the rest of setup unchanged. Add focused coverage in `src/docker-setup.e2e.test.ts`, document the flag in `docs/install/docker.md`, and add a changelog entry because this is user-facing Docker setup behavior.
