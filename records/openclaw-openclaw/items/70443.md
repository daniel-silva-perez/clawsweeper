## Summary

Keep open. Current main still lacks a `--offline` Docker setup path: `scripts/docker/setup.sh` accepts no argv, always builds `openclaw:local` or pulls a non-local `OPENCLAW_IMAGE` before the rest of setup, and still builds the optional sandbox image when sandbox is enabled. The current Docker tests and docs cover adjacent setup behavior such as `OPENCLAW_SKIP_ONBOARDING`, but not preloaded-image offline reruns; the related platform/buildx items do not supersede this request.

## Best Possible Solution

Implement the narrow `--offline` flag in `scripts/docker/setup.sh`: parse it in a Bash 3.2-compatible way, skip main build/pull only when `docker image inspect "$IMAGE_NAME"` succeeds, fail clearly with `docker load` guidance when the main image is missing, warn rather than fail for the optional sandbox image, and leave the rest of setup unchanged. Add focused `src/docker-setup.e2e.test.ts` coverage, document the flag at `https://docs.openclaw.ai/install/docker`, and add a user-facing changelog entry with reporter attribution.

## Clownfish Work Prompt

Implement the Docker setup offline rerun feature requested by openclaw/openclaw#70443. Add a Bash 3.2-compatible `--offline` flag to `scripts/docker/setup.sh`; do not use associative arrays. In offline mode, skip the main image build/pull only after `docker image inspect "$IMAGE_NAME"` succeeds. If the main image is missing, fail with a clear message explaining that operators should preload it with `docker load`. For the optional sandbox image `openclaw-sandbox:bookworm-slim`, skip the build in offline mode and warn rather than fail when the image is missing, preserving the current non-offline warning tone. Leave every other setup step intact: env validation, `upsert_env`, data-directory chown, onboarding or `OPENCLAW_SKIP_ONBOARDING`, `sync_gateway_config`, compose startup, sandbox overlay/config behavior, and default behavior when `--offline` is not passed. Add focused e2e-stub coverage in `src/docker-setup.e2e.test.ts` for offline success with local image inspect, missing main image failure with docker-load guidance, missing optional sandbox image warning, and default non-offline build/pull behavior. Update `docs/install/docker.md` with the user-facing flag and add one `CHANGELOG.md` Unreleased Changes entry with `Thanks @safrano9999`. Do not fold in unrelated env sourcing/runtime-image work from #47479 or #61203, and do not change default setup behavior without the flag.
