## Summary

Keep open. Current main still has no `--offline` Docker setup mode, and the setup script still runs the build/pull branch before the rest of onboarding/config synchronization; the reporter's linked out-of-repo workaround is useful supporting context but is not implemented in OpenClaw.

## What This Changes

The request asks for an explicit `--offline` flag for Docker setup reruns that use a locally preloaded image while preserving the rest of the setup routine.

## Best Possible Solution

Add a narrow, explicit `--offline` mode in `scripts/docker/setup.sh` that verifies the configured image is already present, skips only build/pull work, keeps all later setup steps intact, documents the airgapped rerun flow, and covers it with focused Docker setup e2e tests plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: run the current setup script on a host without registry/build network access after preloading an image, and the script still enters the build or pull branch before the rest of setup can run.

## Solution Assessment

Yes, the requested explicit opt-in flag is the narrowest maintainable direction. The implementation should adapt the workaround into a guarded `--offline` path rather than changing the default online build/pull behavior.
