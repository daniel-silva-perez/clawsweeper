Current main still does not propagate ambient gateway proxy env or NODE_OPTIONS into MCP stdio children; it only forwards configured server env after the current stdio safety filter. The PR addresses a real production failure linked from #65525, but it is stale against current main's `toMcpEnvRecord` safety behavior and needs maintainer judgment on any ambient `NODE_OPTIONS` propagation.
