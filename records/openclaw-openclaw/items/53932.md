## Summary

Current main still does not implement the requested Slack upload opt-out. Slack uploads continue through the optimized outbound media path, Slack config/docs expose only the media size cap, and the related implementation PR is closed and unmerged, so there is no implemented or superseding item to close against.

## Best Possible Solution

Keep this issue open. Implement a Slack-owned config key that defaults to current optimized behavior, wire it through Slack send/upload into the outbound media loader, and add regression coverage proving PNG uploads can preserve original bytes/MIME when disabled while the optimized default remains unchanged. Update Slack config schema/docs with the new setting.
