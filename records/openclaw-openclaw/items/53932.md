## Summary

Current main (5828dcdb05aad587bff831467e7fa51f73312ff8) still does not provide a Slack setting or sender option to preserve original image bytes on outbound Slack uploads. The useful related implementation attempt, PR #53988, is closed and unmerged, so it does not supersede the remaining work.

## Best Possible Solution

Keep this issue open. Implement the opt-out in the bundled Slack plugin with a documented Slack config key that defaults to current optimized behavior, wire it through Slack send/upload via an additive outbound-media option or deliberate raw-loader path, and add regression coverage proving PNG uploads can preserve original bytes/MIME when disabled while the optimized default remains unchanged.
