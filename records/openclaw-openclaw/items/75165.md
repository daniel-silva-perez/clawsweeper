## Summary

Keep open. Current main does not implement composable A2A termination or GSAR, so the PR is not obsolete, but the current head is not merge-ready because check-test-types and check-lint have concrete PR-introduced failures, and the live MaxIterations test still expects the wrong exit reason.

## What This Changes

This PR adds a TerminationCondition algebra and GSAR scorer under src/agents, integrates an optional termination hook into runSessionsSendA2AFlow, adds unit/live tests, and allowlists the new GSAR module from deadcode.

## Best Possible Solution

Fix the localized PR-head lint, type, and live-test expectation defects first, then route the API decision to agents maintainers: a small internal termination seam may be reasonable, while GSAR should land only with an agreed core/plugin/SDK boundary and matching docs or changelog if exposed to users.

## Reproduction Assessment

Not applicable for the feature request itself because this PR introduces a new capability. Yes for the review findings: check-run annotations reproduce the lint/type blockers, and reading MaxIterations.check with runLiveLoop shows the live-test exitReason assertion mismatch.

## Solution Assessment

No, the current PR is not the best merge-ready solution. The A2A hook is plausible, but the branch must be CI-clean and the GSAR core placement should be explicitly approved before merging a production module that is only imported by tests.
