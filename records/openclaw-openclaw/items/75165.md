## Summary

Keep open. Current main does not implement the termination algebra or GSAR helpers, and the PR is a new XL agents API/product surface that needs maintainer review. The diff also introduces a concrete correctness bug in `ReplyPattern` when callers use stateful regexes.

## What This Changes

The PR adds a composable `TerminationCondition` abstraction, GSAR groundedness scoring helpers, an optional A2A loop termination hook, and related agent tests.

## Best Possible Solution

Keep the PR open for maintainer API review. If this direction is accepted, first fix stateful regex handling, then keep the initial surface narrow and internal unless maintainers decide GSAR should become a documented core or SDK contract with a real scorer/evidence integration.

## Reproduction Assessment

Yes for the review finding: the PR's `ReplyPattern` directly retests the same `RegExp`, and a Node probe shows `/DONE/g` alternates true then false on identical text. For the broader feature, reproduction is not applicable because current main simply lacks this new API.

## Solution Assessment

  - body: `ReplyPattern` keeps the caller's `RegExp` instance and calls `.test()` directly. Reset `lastIndex` or test with a stateless clone.
