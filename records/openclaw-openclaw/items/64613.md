## Summary

Keep open. Current main covers heartbeat/runtime-envelope and active-memory tag-block display hygiene, but it still lacks a shared `chat.history` / session-history projection rule for the reported plain memory action prompt shape such as `## Memory system — ACTION REQUIRED` and auto-recall `memory_search` instructions.

## What This Changes

The issue asks for WebChat and Control UI history to hide internal memory and heartbeat injection blocks before returning `chat.history` to the UI.

## Best Possible Solution

Keep the issue open and fix the shared visible-history projection so known memory-system injection blocks are treated as internal content across `chat.history` and session-history SSE, while preserving normal user messages that merely discuss or quote those phrases. Longer term, memory plugins should rely on hidden `prependContext` or explicit display-hidden metadata instead of persisted user-visible prompt text.
