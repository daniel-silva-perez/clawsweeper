## Summary

Keep open. Current main has shared display hygiene for heartbeat prompts, runtime envelopes, directives, tool-call XML, and silent control replies, but it still lacks a `chat.history` / session-history projection rule for the reported plain memory action prompt shape beginning `## Memory system — ACTION REQUIRED`. The issue is recent, non-maintainer-authored, unlabelled, and appears narrow enough for a focused repair PR.

## What This Changes

The issue asks for `chat.history` and WebChat-visible session history to hide internal memory/heartbeat system-injection prompt blocks instead of rendering them to users.

## Best Possible Solution

Add a shared visible-history projection rule for known internal memory-system action prompt blocks in `src/gateway/chat-display-projection.ts`, so both `chat.history` and session-history SSE hide them while preserving normal user messages that merely discuss or quote those phrases. Longer term, memory integrations should use hidden `prependContext` or explicit display-hidden metadata instead of persisted user-visible prompt text.
