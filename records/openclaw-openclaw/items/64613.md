## Summary

Keep open. Current main has shared display projection for `chat.history` and session-history SSE, and it already hides heartbeat/runtime-envelope and bundled active-memory tag-block shapes, but it still has no rule for the reported plain `## Memory system — ACTION REQUIRED` / auto-recall `memory_search` prompt shape. The issue is open, non-maintainer-authored, unlabeled, and I found no open PR mentioning it.

## What This Changes

The issue requests that WebChat and Control UI history hide internal memory-system and heartbeat prompt-injection blocks from `chat.history` responses.

## Best Possible Solution

Add a shared visible-history projection rule for known internal memory-system action prompt blocks so both `chat.history` and session-history SSE hide them, while keeping transcript persistence unchanged and preserving normal user text that merely quotes those phrases.

## Reproduction Assessment

Yes. A source-level reproduction is a raw user-role transcript message whose content starts with `## Memory system — ACTION REQUIRED` and contains the auto-recall `memory_search` instruction; current `projectChatDisplayMessages(...)` has no predicate for that shape and would pass it through. I did not run Vitest because this review was read-only.

## Solution Assessment

Yes, the shared projection boundary is the best narrow fix because `chat.history` and session-history SSE already converge there. A WebChat-only workaround would leave other history consumers exposed, and changing transcript persistence is larger than needed for this report.
