## Summary

Current main still has the formatter bug: punctuation-wrapped Home Assistant-style entity IDs are classified using the raw token, then split with literal spaces. There is also an open unmerged implementation candidate, PR #39565, linked to this issue, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep the issue open and review PR #39565 or land an equivalent focused fix. The fix should classify the punctuation-stripped candidate for file-like copy-sensitive tokens, preserve bare and punctuation-wrapped dotted Home Assistant entity IDs, add regression tests for those shapes, and keep the existing narrow-terminal chunking behavior for generic long prose.
