## Summary

Current main still has the formatter behavior described here, and an open implementation candidate (#39565) uses GitHub closing syntax for this issue. The issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep the issue open and review/land PR #39565 or an equivalent focused fix. The implementation should classify the punctuation-stripped candidate for file-like copy-sensitive tokens, preserve bare and punctuation-wrapped dotted Home Assistant-style entity IDs, add regression tests for those shapes, and retain existing narrow-terminal chunking for generic long prose.
