## Summary

Keep open. Current main still lacks the requested per-session working-context capsule and only has partial breadcrumbs such as transcript header cwd, ACP cwd metadata, and spawnedWorkspaceDir. The discussion adds concrete WSL/cloud/Docker compaction-loss examples, and the related #71135 PR is still an open draft storage primitive rather than a shipped status/compaction/new-flow integration.

## What This Changes

Request to persist a structured per-session working context across compact/new/subagent/temp-clone flows, expose it in status, and clarify workspace/task-repo/runtime-cwd terminology in docs.

## Best Possible Solution

Keep this as the product/API tracker. Land #71135 or an equivalent reviewed storage primitive if maintainers accept the shape, then add follow-up integration that updates the capsule from run/exec/subagent/ACP paths, surfaces cwd/repo/branch/sandbox/temp-clone/push-target state in status, and documents the distinction between semantic continuity and working-state continuity.

## Reproduction Assessment

Not applicable as a failing-test reproduction; this is a feature/API gap. There is a high-confidence inspection path: current main has no workingContext fields or status output for repo/branch/temp-clone/push-target state, and the docs still describe semantic compaction rather than persisted working state.

## Solution Assessment

Unclear for implementation because this issue is a tracker, not a patch. The staged approach in #71135 looks like a plausible first storage seam, but it is explicitly incomplete until runtime updates, status rendering, compaction/new-flow handling, and docs are added.
