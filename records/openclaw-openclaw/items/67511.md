## Summary

Keep open. Current main still lacks the requested per-session structured workingContext capsule, /status exposure, and docs terminology cleanup. Existing cwd/workspace breadcrumbs are useful but partial, and the related #71135 implementation candidate is open, draft, and explicitly storage-only in the provided GitHub context.

## Best Possible Solution

Keep this issue open as the product/API tracker. Review #71135 or a successor as a storage primitive, then add runtime updates from exec/repo-affecting tools, ACP and subagent paths, expose cwd/repo/branch/sandbox/temp-clone/push-target state in /status/session_status, and update the workspace plus compaction docs to distinguish semantic continuity from working-state continuity.
