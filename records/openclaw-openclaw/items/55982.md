## Summary

Keep this PR open. Current main still does not implement the proposed `skipWhenIdle` cron option, and this PR remains the implementation candidate rather than obsolete. The PR’s second commit addresses the earlier Greptile findings for one-shot `at` jobs and `idleMs >= 1000`, but later review comments identify real remaining blockers: the branch does not persist the new field in cron create/patch paths and does not wire a production last-inbound activity source into Gateway cron construction.

## Best Possible Solution

Keep this PR open for review. Ask the author to update the branch by persisting `skipWhenIdle` in `src/cron/service/jobs.ts`, wiring a real main-session last-inbound activity source through `src/gateway/server-cron.ts`, retaining the `at`-job guard and `idleMs >= 1000` validation, adding docs for the new CLI/API surface, and extending tests to cover persistence, Gateway wiring behavior, recurring main-session skips, and one-shot schedules.
