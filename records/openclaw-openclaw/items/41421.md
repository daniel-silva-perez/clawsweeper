## Summary

Keep open. The requested command is still absent from current main, and this PR is useful direction, but the patch is not currently mergeable: it imports channel internals that no longer exist on main and duplicates only part of the embedded runtime prompt assembly path.

## What This Changes

The PR branch adds an `openclaw agents view-system-prompt [agentId]` CLI subcommand that assembles and prints a local system prompt with model/channel overrides and stderr metadata.

## Best Possible Solution

Land a rebased implementation that shares a runtime prompt-preview helper with embedded runs, or extracts the reusable prompt assembly pieces first, then adds focused CLI docs and parity tests for model override, channel prompt capabilities, workspace cwd, heartbeat policy, provider contributions, and plugin transforms.

## Reproduction Assessment

Yes. Static inspection is enough: current main has no command, and the PR diff shows missing imports plus a prompt assembly path that diverges from `runEmbeddedAttempt`.

## Solution Assessment

No. The command direction is useful, but duplicating runtime prompt assembly directly inside a CLI handler is not the narrowest maintainable solution unless it is kept in parity with the real runtime builder.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Use current channel prompt seams:** `src/commands/agents.commands.view-system-prompt.ts:37-39`
  - body: These imports target `src/signal` and `src/telegram` modules that no longer exist on current main, so the branch will fail to typecheck after rebase. Use the generic channel/plugin prompt helpers instead.
  - confidence: 0.97
- **[P2] Collect runtime channel prompt capabilities:** `src/commands/agents.commands.view-system-prompt.ts:84-86`
  - body: This uses only `resolveChannelCapabilities`, but runtime calls `collectRuntimeChannelCapabilities`, which also includes plugin-owned prompt capabilities. Channel previews can omit capabilities that the real prompt includes.
  - confidence: 0.9
- **[P2] Use the agent workspace as cwd:** `src/commands/agents.commands.view-system-prompt.ts:107-112`
  - body: Passing `process.cwd()` makes repo/runtime fields depend on the shell launch directory. Embedded runtime passes the effective agent workspace as `cwd`, so this can print a different prompt than a real run.
  - confidence: 0.92
- **[P2] Preserve runtime workspace notes:** `src/commands/agents.commands.view-system-prompt.ts:136-140`
  - body: Injecting `Loaded context files: ...` through `workspaceNotes` adds prompt text production does not send and misses the runtime workspace reminders production may send. The file contents already flow through `contextFiles`.
  - confidence: 0.87
- **[P2] Gate heartbeat guidance like runtime:** `src/commands/agents.commands.view-system-prompt.ts:159-162`
  - body: This injects heartbeat guidance for the default agent unconditionally, but embedded runtime also checks trigger and heartbeat policy before adding it. Normal-run previews can include instructions the agent would not receive.
  - confidence: 0.88
- **[P2] Apply provider prompt overlays before printing:** `src/commands/agents.commands.view-system-prompt.ts:170-193`
  - body: The command prints the direct `buildEmbeddedSystemPrompt` result, while runtime resolves provider prompt contributions and applies provider/plugin system-prompt transforms. Provider overlays can be missing from the displayed prompt.
  - confidence: 0.91
