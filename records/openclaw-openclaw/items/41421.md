## Summary

Keep this PR open. Current main still lacks the requested system-prompt preview command, and the related feature issue was closed with this PR as the canonical implementation candidate; however, the branch is stale against current prompt assembly and channel/provider plugin seams, so it needs rebase and rework before merge.

## What This Changes

The PR adds an `openclaw agents view-system-prompt [agentId]` CLI command, exports its handler, assembles an embedded system prompt with optional model/channel overrides, and prints the prompt plus metadata.

## Best Possible Solution

Keep this PR as the implementation thread, but rebase it onto current main and drive the command through the same generic runtime prompt-resolution seams used by embedded runs, with focused tests and docs for prompt fidelity, model/channel overrides, workspace handling, and sensitive local output.

Full review comments:

- **[P1] Use plugin prompt seams for channel guidance:** `src/commands/agents.commands.view-system-prompt.ts:29-31`
  - body: This command imports `../signal/*` and `../telegram/*`, but current main no longer has `src/signal` or `src/telegram` modules. The branch will fail to typecheck after rebase and also bypasses the generic channel prompt APIs now used by runtime prompt assembly.
  - confidence: 0.95
- **[P2] Build prompt params from the agent workspace:** `src/commands/agents.commands.view-system-prompt.ts:99`
  - body: Passing `process.cwd()` into `buildSystemPromptParams` makes repo-root/runtime rendering depend on the shell launch directory. Embedded runtime passes the effective agent workspace, so this preview can show a different Runtime section when invoked outside that workspace.
  - confidence: 0.9
- **[P2] Mirror runtime bootstrap notes instead of adding a file list:** `src/commands/agents.commands.view-system-prompt.ts:119-123`
  - body: `workspaceNotes` is used here to inject `Loaded context files: ...`, but runtime uses that field for BOOTSTRAP/embedded-mode reminders and passes actual context files separately. This adds prompt text production does not send and misses prompt text production does send.
  - confidence: 0.86
- **[P2] Include provider prompt contributions in the preview:** `src/commands/agents.commands.view-system-prompt.ts:147-170`
  - body: The preview calls `buildEmbeddedSystemPrompt` without resolving provider prompt contributions or applying provider prompt transforms. Current runtime includes those sections, so provider-owned prompt guidance can be absent from the displayed assembled prompt.
  - confidence: 0.88
