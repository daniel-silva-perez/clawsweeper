Current main still does not implement `openclaw agents view-system-prompt`, and the related feature issue was closed in favor of this open PR as the canonical implementation candidate. The PR should stay open for maintainer review/rework because current prompt assembly has moved through runtime helper and plugin prompt surfaces that this PR needs to align with.

## Best Possible Solution

Keep this PR open. The best path is to rebase it onto current main, share or extract prompt-preview assembly from the runtime path where feasible, route channel prompt capabilities/reactions/hints through the generic plugin prompt APIs, avoid extension-owned deep imports from core, and add focused tests for default agent selection, `--model`, normalized `--channel`, workspace cwd/repo-root behavior, sandbox/no-sandbox behavior, provider prompt contributions, effective tool metadata, and stderr counts.
