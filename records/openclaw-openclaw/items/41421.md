## Summary

Keep open. Current main still does not provide the requested full system-prompt CLI dump, but this PR is not ready to merge because it duplicates runtime prompt assembly, imports obsolete channel internals, and misses current provider/plugin prompt paths.

## What This Changes

The PR adds an `openclaw agents view-system-prompt [agentId]` CLI subcommand, exports its handler, assembles a prompt with optional model/channel overrides, and prints prompt metadata.

## Best Possible Solution

Land a rebased implementation that uses a shared runtime prompt-preview assembly path, keeps channel-specific guidance behind plugin prompt APIs, applies provider prompt contributions/transforms, and adds focused CLI docs plus tests for workspace selection, model overrides, channel guidance, heartbeat behavior, and provider overlays.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use generic channel prompt APIs:** `src/commands/agents.commands.view-system-prompt.ts:29-31`
  - body: The new command imports `../signal/*` and `../telegram/*`, but current main no longer has those core modules. A rebase will fail typecheck, and this should use the generic channel/plugin prompt seams used by runtime assembly.
  - confidence: 0.96
- **[P2] Apply provider prompt overlays before printing:** `src/commands/agents.commands.view-system-prompt.ts:147-170`
  - body: The command prints the direct `buildEmbeddedSystemPrompt` result, while runtime first resolves provider prompt contributions and then applies `transformProviderSystemPrompt`. Provider overlays and plugin text transforms can be missing from the displayed assembled prompt.
  - confidence: 0.9
- **[P2] Use the agent workspace as cwd:** `src/commands/agents.commands.view-system-prompt.ts:99`
  - body: Passing `process.cwd()` into `buildSystemPromptParams` makes repo/runtime fields depend on the shell launch directory. Embedded runtime passes the effective agent workspace for both `workspaceDir` and `cwd`, so this can preview a different prompt than a real run.
  - confidence: 0.91
- **[P2] Preserve runtime workspace notes:** `src/commands/agents.commands.view-system-prompt.ts:119-123`
  - body: Injecting `Loaded context files: ...` through `workspaceNotes` adds prompt text production does not send and misses the BOOTSTRAP/runtime reminders production does send. The file contents already flow through `contextFiles`, so this reduces preview fidelity.
  - confidence: 0.87
- **[P2] Gate heartbeat guidance like runtime triggers:** `src/commands/agents.commands.view-system-prompt.ts:136-138`
  - body: This injects the default-agent heartbeat prompt unconditionally, but current embedded runtime only includes heartbeat guidance when the heartbeat trigger and config policy allow it. Normal-run previews can therefore include instructions the agent would not receive.
  - confidence: 0.86
