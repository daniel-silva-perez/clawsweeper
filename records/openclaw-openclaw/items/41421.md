## Summary

Keep open. Current main does not contain the requested command, and the PR remains useful source work, but its latest diff is not merge-ready: it imports channel internals no longer present on main and diverges from the current runtime prompt assembly path. The linked feature issue #41417 is closed, but this PR has not merged or been superseded by an implementation on main.

## What This Changes

The PR branch registers `openclaw agents view-system-prompt [agentId]`, adds a command handler that assembles and prints a local embedded system prompt with optional model/channel overrides, and exports the handler from the agents command barrel.

## Best Possible Solution

Land a rebased implementation that shares or closely mirrors the runtime prompt-preview assembly path, uses generic channel/plugin prompt APIs, applies provider prompt contributions and transforms, and adds focused CLI docs plus parity tests for model, channel, workspace, heartbeat, provider, and sandbox-aware cases.

## Reproduction Assessment

Yes. Static comparison of the PR diff against current main reproduces the blocking issues: the branch imports modules absent from current main and assembles a narrower prompt than `runEmbeddedAttempt` uses for real runs.

## Solution Assessment

No. The command direction is useful, but duplicating runtime prompt assembly inside a CLI handler is not the narrowest maintainable solution unless the shared runtime helper is extracted or the preview path is kept in strict parity with it.

  - body: The new command imports `../signal/*` and `../telegram/*`, but those core modules do not exist on current main. A rebase will fail typecheck; this should use the generic channel/plugin prompt helpers used by runtime assembly.

  - body: This prints the direct `buildEmbeddedSystemPrompt` result, while runtime resolves provider prompt contributions and then applies provider/plugin system-prompt transforms. Provider overlays and text replacements can be missing from the displayed prompt.
  - confidence: 0.91
- **[P2] Use the agent workspace as cwd:** `src/commands/agents.commands.view-system-prompt.ts:101`
  - body: Passing `process.cwd()` into `buildSystemPromptParams` makes repo/runtime fields depend on the shell launch directory. Embedded runtime passes the effective agent workspace for both `workspaceDir` and `cwd`, so this can preview a different prompt than a real run.
  - confidence: 0.92
- **[P2] Preserve runtime workspace notes:** `src/commands/agents.commands.view-system-prompt.ts:130-134`
  - body: Injecting `Loaded context files: ...` through `workspaceNotes` adds prompt text production does not send and misses the runtime BOOTSTRAP/local-mode reminders production does send. The file contents already flow through `contextFiles`, so this reduces preview fidelity.
  - confidence: 0.87
- **[P2] Gate heartbeat guidance like runtime:** `src/commands/agents.commands.view-system-prompt.ts:153-156`
  - body: This injects the default-agent heartbeat prompt whenever the agent is default, but current embedded runtime only includes heartbeat guidance when the heartbeat trigger and config policy allow it. Normal-run previews can therefore include instructions the agent would not receive.
  - confidence: 0.88
