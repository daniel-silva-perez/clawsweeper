## Summary

Keep open. Current main still does not provide a full system-prompt preview command, so the PR is not obsolete, but the branch is not merge-ready because it no longer matches current prompt/channel/provider runtime seams and would diverge from real embedded prompt assembly.

## What This Changes

The PR adds an `openclaw agents view-system-prompt [agentId]` CLI subcommand, exports a new handler, assembles a local embedded system prompt with optional model/channel overrides, and prints prompt metadata.

## Best Possible Solution

Land a rebased implementation that drives preview rendering through shared prompt-resolution helpers used by embedded runtime, keeps channel-specific behavior behind plugin prompt APIs, includes provider prompt contributions/transforms, and adds focused CLI/runtime docs plus tests for model overrides, channel guidance, workspace selection, and provider overlays.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use plugin prompt seams for channel guidance:** `src/commands/agents.commands.view-system-prompt.ts:29-31`
  - body: This command imports `../signal/*` and `../telegram/*`, but current main no longer has those core modules. After rebase the branch will fail typecheck, and the direct imports bypass the plugin-owned channel prompt APIs now used by runtime assembly.
  - confidence: 0.95
- **[P2] Build prompt params from the agent workspace:** `src/commands/agents.commands.view-system-prompt.ts:99`
  - body: Passing `process.cwd()` into `buildSystemPromptParams` makes repo-root/runtime rendering depend on the shell launch directory. Embedded runtime passes the effective agent workspace as both `workspaceDir` and `cwd`, so this can display a different Runtime section than a real run.
  - confidence: 0.9
- **[P2] Preserve runtime workspace notes:** `src/commands/agents.commands.view-system-prompt.ts:119-123`
  - body: Using `workspaceNotes` for `Loaded context files: ...` adds prompt text production does not send and misses the BOOTSTRAP/embedded-mode reminders production does send. The file contents already flow through `contextFiles`, so this reduces preview fidelity.
  - confidence: 0.86
- **[P2] Include provider prompt contributions:** `src/commands/agents.commands.view-system-prompt.ts:147-170`
  - body: Runtime resolves provider prompt contributions, passes them into `buildEmbeddedSystemPrompt`, and then applies provider prompt transforms. This preview calls the builder directly, so OpenAI/Codex overlays and plugin text transforms can be absent from the displayed assembled prompt.
  - confidence: 0.88
