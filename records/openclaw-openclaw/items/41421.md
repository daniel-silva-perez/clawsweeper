## Summary

Keep open. Current main does not implement this command, so this contributor PR is still the implementation thread, but the proposed branch is not merge-ready because it duplicates prompt assembly and now diverges from the current runtime/plugin prompt seams.

## What This Changes

The PR adds an `openclaw agents view-system-prompt [agentId]` command, exports its handler, assembles an embedded system prompt with optional model/channel overrides, and prints prompt metadata.

## Best Possible Solution

Land a rebased implementation that drives prompt preview through shared runtime prompt-resolution seams, avoids channel-specific core imports, and adds focused tests/docs for prompt fidelity, model/channel overrides, workspace handling, provider contributions, and sensitive local output.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use plugin prompt seams for channel guidance:** `src/commands/agents.commands.view-system-prompt.ts:29-31`
  - body: This command imports `../signal/*` and `../telegram/*`, but current main no longer has those core modules. The branch will fail typecheck after rebase and also bypasses the plugin-owned channel prompt APIs used by runtime assembly.
  - confidence: 0.95
- **[P2] Build prompt params from the agent workspace:** `src/commands/agents.commands.view-system-prompt.ts:99`
  - body: Passing `process.cwd()` into `buildSystemPromptParams` makes preview rendering depend on the shell launch directory. Embedded runtime uses the effective agent workspace as `cwd`, so this can show a different Runtime section from a real run.
  - confidence: 0.9
- **[P2] Mirror runtime bootstrap notes instead of adding a file list:** `src/commands/agents.commands.view-system-prompt.ts:119-123`
  - body: Using `workspaceNotes` for `Loaded context files: ...` adds text production does not send and misses the BOOTSTRAP/embedded-mode reminders that production does send; actual context files should flow through the `contextFiles` field.
  - confidence: 0.86
- **[P2] Include provider prompt contributions in the preview:** `src/commands/agents.commands.view-system-prompt.ts:147-170`
  - body: The preview builds the prompt without resolving provider prompt contributions or applying provider prompt transforms. Current runtime includes both, so provider-owned guidance can be absent from the displayed assembled prompt.
  - confidence: 0.88
