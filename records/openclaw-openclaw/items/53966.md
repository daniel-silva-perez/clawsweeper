## Summary

Keep this PR open. Current main still loses structured Slack API error details, and the PR is a focused implementation, but the latest diff is not merge-ready because the new formatter bypasses the existing redaction path used by several replaced call sites.

## What This Changes

Adds a Slack-local `formatSlackError()` helper with unit tests and routes Slack API error logging/probe/scopes call sites through it to include SDK error codes, missing-scope details, retry timing, and HTTP status fields.

## Best Possible Solution

Land a Slack-plugin-local formatter after it preserves existing redaction behavior while adding Slack SDK structured fields; keep the broader delivery queue preservation work tracked separately through #62391 or a generic core error serialization seam.
