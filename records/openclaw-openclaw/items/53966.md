## Summary

Keep this PR open for maintainer review. Current main still lacks the Slack-specific structured error formatter, and the latest PR head addresses the earlier retryAfter nesting review, but the patch should not merge until it preserves the existing token-redaction behavior on formatted Slack error output.

## What This Changes

Adds a Slack-local `formatSlackError()` helper with unit tests and routes Slack probe, scopes, stream preview, reaction, slash reply, startup allowlist resolution, and thread-resolution error paths through it to include Slack SDK code, scope, retry, and HTTP status metadata.

## Best Possible Solution

Land a Slack-plugin-local formatter that appends the structured Slack SDK fields and then redacts the final formatted string, with a regression test using Slack-shaped token data. Keep broader delivery-queue error serialization work tracked separately through #62391 or a generic core serialization seam.
