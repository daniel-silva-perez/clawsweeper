## Summary

Keep this PR open for maintainer review. Current main and the latest release still preserve DBus/user-bus failures as errors from `isSystemdServiceEnabled`, while the PR changes the generic service loaded-state contract to return `false`; that may help one install probe path, but it is broader than current main's narrower install-probe handling and affects lifecycle callers such as uninstall.

## What This Changes

The PR adds `isSystemctlBusUnavailable(detail)` to `isSystemdServiceEnabled` so DBus/user-bus unavailable is treated like a disabled or missing systemd unit and returns `false`.

## Best Possible Solution

Keep this PR open for maintainer review. If the behavior is desired, constrain non-fatal DBus handling to the install/status probe surface or otherwise preserve lifecycle error semantics, then add focused tests for `src/daemon/systemd.test.ts` plus configure/uninstall coverage before merge; if maintainers decide loaded-state DBus transport failures should remain errors, close the PR after that explicit decision.
