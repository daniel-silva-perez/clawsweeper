## Summary

Keep open. Current main does not implement the PR’s requested function-level behavior: exhausted DBus/user-bus failures in `isSystemdServiceEnabled` still throw, while current main already has narrower install-probe handling that treats those errors as non-fatal only for install flows. The proposed branch is a real, small implementation, but it changes the generic service-loaded contract used by lifecycle callers, so this needs maintainer review rather than cleanup closure.

## What This Changes

This PR changes `isSystemdServiceEnabled` so `systemctl is-enabled` DBus/user-bus unavailable failures return `false` instead of throwing.

## Best Possible Solution

Keep the scoped install-probe recovery unless maintainers explicitly decide that the cross-platform `GatewayService.isLoaded` contract should represent exhausted DBus transport failures as not loaded. If that broader contract is accepted, update the lifecycle callers and tests together so install, status, reset, and uninstall behavior stay intentional.
