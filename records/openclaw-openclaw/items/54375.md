## Summary

Keep this PR open for maintainer review. Current main still preserves DBus/user-bus failures as errors in `isSystemdServiceEnabled`, while the PR would convert that transport failure into `false`; that is a real behavior change for service lifecycle callers, not something already implemented on main. The diff is source-only and not a supply-chain concern, but it needs a maintainer call and focused tests before merge.

## Best Possible Solution

Keep this PR open for maintainer review. If the behavior is desired, update the PR with focused systemd tests and either preserve lifecycle error semantics or narrow the non-fatal handling to the install/status surface that needs it; otherwise close it after maintainer confirmation that DBus transport failures should remain errors for loaded-state checks.
