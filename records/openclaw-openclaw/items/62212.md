## Summary

Current main still does not implement the issue's requested Vertex AI ADC OAuth path. It recognizes `google-vertex` and can synthesize a `gcp-vertex-credentials` marker when ADC files/env are present, but the Google chat transport does not exchange that marker for a real Google access token and the bundled Google provider still owns `google-vertex` through hook aliases with only API-key auth. The open Google GenAI SDK PR is an active implementation candidate, not evidence that this issue is fixed on main.

## Best Possible Solution

Keep this issue open and use it as concrete bug evidence for the Vertex AI auth work. The best fix is in the bundled Google provider: add a provider-owned runtime auth path that turns ADC into a short-lived bearer token for Vertex requests, route requests through the correct Vertex project/location endpoints, and add regression coverage or a gated live test for the ADC/service-account path. If maintainers choose the new `google-genai` provider direction from #65023 instead, close this issue only after that PR or an equivalent implementation lands on main and the user-visible setup path is documented.

## Evidence
