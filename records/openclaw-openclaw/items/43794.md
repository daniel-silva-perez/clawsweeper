## Summary

Keep this open. Current main still lacks first-class encrypted-at-rest credential storage, OS keychain-backed OpenClaw credential persistence, or a master-password store; the implemented SecretRef path is useful but remains opt-in externalization while plaintext credentials continue to be accepted and documented as a high residual risk.

## What This Changes

The issue requests first-class encryption at rest for OpenClaw credentials, using a system keychain or master password while preserving backward compatibility.

## Best Possible Solution

Design and ship a core secure credential storage layer for OpenClaw-owned credentials, with OS keychain and/or master-password encrypted storage, backward-compatible migration from plaintext openclaw.json and auth-profiles.json, SecretRef compatibility, token rotation/OAuth refresh handling, docs updates, and focused regression coverage.
