## Summary

Keep this issue open. Current main still does not provide first-class credential encryption at rest, a native OpenClaw keychain-backed credential store, or a master-password-backed encrypted store. The current mitigation is SecretRefs plus auditing/scrubbing and file permissions; plaintext credentials remain valid, opt-in SecretRefs only cover supported surfaces, and the threat model still tracks plaintext token theft as a high residual risk.

## Best Possible Solution

Keep open as core security work. Design and ship a first-class secure credential storage layer for OpenClaw-owned credentials, with OS keychain and/or encrypted-at-rest master-password/provider support, backward-compatible migration from plaintext openclaw.json and auth-profiles.json, SecretRef compatibility, OAuth and rotating credential handling, security docs updates, and focused regression coverage.
