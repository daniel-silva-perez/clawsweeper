## Summary

Keep this issue open. Current main still does not implement first-class OpenClaw credential encryption at rest, a native keychain-backed credential store, or a master-password-backed store. It has useful SecretRef mitigation and audit/apply tooling, but SecretRefs remain opt-in externalization, plaintext credentials remain valid, and the security docs/threat model still describe plaintext token theft as a high residual risk. The prior ClawSweeper review and discussion remain consistent with current source.

## Best Possible Solution

Keep open as core security work. Design and ship a first-class secure credential storage layer for OpenClaw-owned credentials, with OS keychain and/or encrypted-at-rest master-password/provider support, backward-compatible migration from plaintext openclaw.json and auth-profiles.json, SecretRef compatibility, OAuth/rotation handling, security/docs updates, and focused regression coverage.
