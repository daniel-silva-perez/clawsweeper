## Summary

Keep open. Current main still sends only `Accept: application/json` when the Kilocode plugin discovers models, so the PR’s intended BYOK-auth discovery behavior is not implemented. The related core path mentioned in the PR has since been removed as an orphaned wrapper, leaving the plugin path as the relevant surface.

## What This Changes

The PR adds `KILOCODE_API_KEY` and `KILOCODE_ORGANIZATION_ID` headers to Kilocode plugin model discovery, adds fetch-header tests, and updates the raw-fetch allowlist line.

## Best Possible Solution

Land this narrow plugin-path fix, or an equivalent maintainer-adjusted version, after confirming the Kilo Gateway header contract and deciding whether the organization-id env var needs docs or manifest metadata. The removed core wrapper should stay out of scope.
