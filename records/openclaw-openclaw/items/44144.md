## Summary

Keep this PR open. Current main still contains the model-usage TODO that this docs change targets, and the Python summarizer already supports the portable `--input` workflow. The PR is useful, but it still needs the review-feedback wording fix so Linux/other-platform guidance is clearly framed as manual script usage outside normal macOS-gated skill eligibility.

## Best Possible Solution

Revise and keep the focused docs improvement: replace the TODO with a platform note, explicitly say macOS live reads use CodexBar installed via Homebrew and available as `codexbar` on PATH, and say Linux/other platforms can run the Python summarizer manually outside normal OpenClaw skill eligibility with `--input` and exported CodexBar JSON unless maintainers intentionally broaden the skill metadata.
