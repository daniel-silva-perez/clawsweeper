## Summary

Keep this PR open. Current main still has the exact docs gap the PR tries to fix, and the underlying script already supports the portable `--input` path. The PR should be revised before merge because review comments correctly flagged that the proposed cross-platform wording conflicts with the current skill metadata gates and has an ambiguous macOS install reference.

## Best Possible Solution

Revise and keep the focused docs improvement: replace the TODO with an explicit platform note, say macOS live reads use CodexBar installed via the existing Homebrew metadata and available on PATH, and say Linux/other platforms can run the bundled Python script manually with --input using exported CodexBar JSON unless maintainers choose to broaden skill metadata eligibility. No runtime or security-sensitive change is needed for this PR.
