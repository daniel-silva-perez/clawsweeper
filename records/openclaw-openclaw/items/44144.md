## Summary

Keep this PR open. Current main still has the model-usage documentation TODO that the PR targets, and the bundled script already supports the portable `--input` path. The PR is not implemented on main, but it should be revised before merge to make clear that non-macOS usage is manual script usage outside the OpenClaw skill eligibility path unless the metadata is intentionally changed.

## Best Possible Solution

Revise and keep the focused docs improvement: replace the TODO with a platform note, say macOS live reads use CodexBar installed via Homebrew and available as `codexbar` on PATH, and say Linux/other platforms can run the bundled Python script manually outside OpenClaw skills with `--input` and exported CodexBar JSON unless maintainers intentionally broaden the skill metadata.
