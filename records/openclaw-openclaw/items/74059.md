## Summary

Keep this PR open. Current `main` still has the reported explicit `openclaw crestodian` no-TTY behavior: `runCrestodian()` prints the interactive-TTY error and returns normally, while the command wrapper only exits non-zero when the action throws. The PR is a narrow implementation candidate for #73646, though maintainers should choose a canonical fix because #73928 appears to target the same remaining work.

Keep this PR open for maintainer review, or close only after maintainers choose a different canonical PR such as #73928. The final fix should make the explicit `openclaw crestodian` no-TTY path exit non-zero, preserve `--message`, `--json`, and interactive TUI behavior, and include focused regression coverage for stdin/stdout non-TTY cases.
