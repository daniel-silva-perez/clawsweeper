## Summary

Keep open: current main still preserves the WebChat/UI safety boundary that prevents Control UI turns from inheriting and pushing to the stored Feishu route, so the reported cross-surface delivery loss remains unresolved. The existing tests intentionally assert that UI/WebChat sends stay internal even when an external route exists, so this needs maintainer product/security direction rather than cleanup closure.

## What This Changes

The issue requests that a Feishu-started conversation continue delivering assistant replies back to Feishu after the user interrupts or continues the same session from the Web Control UI.

## Best Possible Solution

Define an explicit cross-surface delivery contract for Feishu-started sessions, then implement a permissioned route-inheritance or opt-in mirroring path that preserves the current WebChat safety guards and adds regression coverage for Feishu main-session delivery after a Control UI follow-up.
