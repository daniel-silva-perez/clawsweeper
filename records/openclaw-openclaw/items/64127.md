## Summary

Keep open. Current main still does not implement the PR's intended quota-exhaustion circuit breaker: `quota_exhausted` is absent from the reason unions, periodic usage-window errors still classify as `rate_limit`, the long disabled lane only covers `billing` and `auth_permanent`, and there is no `agent:provider_tripped` hook on main. The PR is now narrowly split to the circuit-breaker scope, while #72557 tracks the separate TUI/status-provider API work.

## Best Possible Solution

Keep this PR open for focused maintainer review. The best path is to land a corrected quota-exhaustion circuit breaker in the agent failover/auth-profile path: add aligned reason unions, classify periodic quota exhaustion including 429 payload precedence, include the reason in unavailable scoring, route it through the disabled backoff lane with a quota-appropriate probe policy, add the `agent:provider_tripped` hook type/guard if maintainers want that API, and update tests/docs. Keep #72557 as the separate TUI/status-provider API track.
