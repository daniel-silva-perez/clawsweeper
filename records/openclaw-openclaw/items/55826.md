## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: tool-policy pipeline filtering has no INFO audit of removed tools or rule labels, sandbox blocked-tool handling only formats user-facing guidance, and existing tool logs remain start/end DEBUG entries without matched policy attribution. The PR is still a plausible implementation candidate for #55801, but its current sandbox logging approach needs rework because placing audit emission in the error-formatting path can duplicate audit records.

## Best Possible Solution

Keep the PR open. Rebase it over current main, keep the deterministic pipeline removal audit, move sandbox blocked-tool audit emission to a single enforcement point or add explicit per-run/tool dedupe, and add tests that prove both rule attribution and no duplicate sandbox audit lines. After that, review/land it as the implementation for #55801.
