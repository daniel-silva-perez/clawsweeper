## Summary

Keep this issue open. Current main still has the backend lifecycle gap described by the reporter: after an embedded timeout, the runner can abort and unsubscribe the pi session without `subscribeEmbeddedPiSession.unsubscribe()` synthesizing a terminal lifecycle event. Gateway/Control UI still clear the Stop state from terminal chat events, and the live closing PR #73105 is still open and unmerged.

## Best Possible Solution

Keep this issue open and review #73105 as the direct backend fix candidate. The best fix is to make embedded subscription teardown guarantee exactly one terminal lifecycle/chat finalization event when `agent_end` is missed, with regression coverage for unsubscribe-before-agent_end and idempotency when `agent_end` arrives normally. Treat #42258 as a separate UI recovery mitigation unless maintainers intentionally choose that path.
