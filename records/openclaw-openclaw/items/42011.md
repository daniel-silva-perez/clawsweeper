## Summary

Keep this issue open. Current main still has the reported backend lifecycle gap: the embedded timeout path aborts the active session and later unsubscribes, but `subscribeEmbeddedPiSession.unsubscribe()` does not synthesize a terminal lifecycle event if `agent_end` is missed. Gateway/Control UI still clear the Stop state from terminal chat events, and #73105 remains open and unmerged with closing syntax for this issue, so the maintainer path is to review and land or close that PR before closing this issue.

## Best Possible Solution

Keep this issue open and review #73105 as the direct backend fix candidate. The best fix is to make the embedded subscription teardown path guarantee exactly one terminal lifecycle/chat finalization event when `agent_end` is missed, with regression coverage for unsubscribe-before-agent_end and idempotency when `agent_end` arrives normally. Treat #42258 as a separate UI recovery mitigation unless maintainers intentionally choose that path.
