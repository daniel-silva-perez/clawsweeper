## Summary

Keep open. Current main still preserves the embedded timeout lifecycle gap described by the reporter, and the open closing PR remains the direct implementation candidate rather than a reason to close the issue before merge.

## What This Changes

The issue reports that Control UI chat can remain in a running Stop state after an embedded run times out or aborts without a terminal lifecycle event reaching the chat path.

## Best Possible Solution

Land PR #73105 or an equivalent backend fix that makes subscribeEmbeddedPiSession teardown guarantee exactly one terminal lifecycle/chat finalization event when agent_end is missed, with regression coverage for unsubscribe-before-agent_end and idempotency when agent_end arrives normally. Treat PR #42258 as a possible UI recovery layer, not the primary backend contract fix.

## Reproduction Assessment

Yes, at the source/seam level: trigger an embedded timeout or abort after onAgentRunStart, let unsubscribe remove the pi-session listener before agent_end is delivered, and the current code has no fallback terminal lifecycle emission. A focused unit test around subscribeEmbeddedPiSession.unsubscribe() before agent_end should reproduce the missing terminal event without needing a live Windows run.

## Solution Assessment

Yes, the backend teardown guarantee proposed by PR #73105 is the narrowest maintainable fix because it repairs the lifecycle contract at the subscription seam used by all callers. A UI watchdog can reduce symptoms, but it should not be the canonical fix for a missing terminal event.
