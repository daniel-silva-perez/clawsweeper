## Summary

Keep this issue open. Current main at 6d323ee73640afdf02ed9fa3aded13da23359b98 still has the backend lifecycle gap described in the report: the embedded timeout path aborts the active session fire-and-forget, teardown unsubscribes without synthesizing a terminal lifecycle event, and Gateway/Control UI still depend on terminal chat events to clear the running state. The provided GitHub context also shows #73105 is open, unmerged, and uses closing syntax for this issue, so the right maintainer path is to review/land or close that implementation candidate before closing this issue.

## Best Possible Solution

Keep this issue open and review #73105 as the direct backend fix candidate. The best fix is to make the embedded subscription teardown path guarantee exactly one terminal lifecycle/chat finalization event when agent_end is missed, with regression coverage for unsubscribe-before-agent_end and idempotency when agent_end arrives normally. Treat #42258 as a separate UI recovery mitigation unless maintainers choose that path intentionally.
