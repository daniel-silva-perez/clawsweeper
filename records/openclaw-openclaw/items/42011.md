## Summary

Keep this issue open. Current main still has the reported lifecycle gap: the embedded timeout path aborts the active session fire-and-forget, the runner later unsubscribes, and the subscription teardown does not synthesize a terminal lifecycle event when `agent_end` was missed. The provided GitHub context also shows #73105 is open, unmerged, and uses closing syntax for this issue, so the right maintainer path is to review/land or close that PR before closing this issue.

## Best Possible Solution

Keep this issue open and review #73105 as the direct backend implementation candidate. The best fix is to make `subscribeEmbeddedPiSession.unsubscribe()` or an equivalent subscription-owned seam guarantee exactly one terminal lifecycle/chat state before teardown when `agent_end` was missed, with tests proving timeout/abort teardown and normal `agent_end` idempotency. Close this issue only after that PR or an equivalent fix lands on main.
