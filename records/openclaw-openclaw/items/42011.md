## Summary

Keep this issue open. Current main still has the timeout/abort teardown path that can unsubscribe the embedded session without synthesizing a terminal lifecycle event, and #73105 is an open, unmerged implementation candidate that explicitly targets this issue.

## Best Possible Solution

Keep this issue open and review #73105 as the direct implementation candidate. The likely best fix is a backend contract in subscribeEmbeddedPiSession.unsubscribe(): before removing the listener, emit exactly one terminal lifecycle/chat state when agent_end was missed, with regression coverage for timeout/abort teardown and idempotency with handleAgentEnd.
