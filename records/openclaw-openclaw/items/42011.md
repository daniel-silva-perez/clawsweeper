## Summary

Keep this issue open. Current main still lacks the subscription-teardown fallback terminal lifecycle event, and the provided GitHub context shows #73105 is open, unmerged, and explicitly targets this bug with closing syntax. The Control UI chat path still clears the Stop state only after terminal chat events, while started chat.send runs still rely on lifecycle end/error rather than broadcasting their own fallback final event.

## Best Possible Solution

Keep this issue open and review #73105 as the direct implementation candidate. The best fix is a backend contract in subscribeEmbeddedPiSession.unsubscribe(): emit exactly one terminal lifecycle/chat state during subscription teardown when agent_end was missed, before removing the listener, with regression coverage for timeout/abort teardown. If #73105 is closed without merging, keep this issue as the canonical tracker for an equivalent backend lifecycle guarantee.
