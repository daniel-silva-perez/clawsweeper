## Summary

Keep open. Current main still hard-codes the Control UI chat message group width and does not expose an upgrade-safe config, bootstrap, UI state, CSS variable, or documented setting for this layout behavior. The prior ClawSweeper comment matches the current recheck against acae48b790fac67c3f46afae310bbba837f5e678.

## What This Changes

The issue requests an upgrade-safe Control UI setting, preferably in openclaw.json, to customize chat message max-width instead of re-patching bundled CSS after updates.

## Best Possible Solution

Add a narrowly validated Control UI chat layout option, likely gateway.controlUi.chatMessageMaxWidth, carry it through the typed config/schema/help and bootstrap JSON into UI state, set a root/shell CSS custom property, keep the existing min(900px, 68%) fallback, and document/test the behavior without introducing general custom CSS injection.
