## Summary

Keep open. Current main still has the reported raw `infer model run --model` dispatch behavior, and this PR is an active focused fix candidate, but the patch should not land as-is because its new canonicalization helper misses supported auth-profile suffixes and skips lowercase variants of mixed-case catalog model ids; maintainers also need to choose between this PR and the open alternative #74945.

## What This Changes

The PR adds a CLI helper that canonicalizes mixed-case explicit `infer model run --model` refs through the model catalog before local and gateway dispatch, plus regression tests for local/gateway canonicalization and no-match preservation.

## Best Possible Solution

Land one catalog-backed fix for the linked bug that canonicalizes known catalog matches for both local and gateway `infer model run` dispatch, preserves exact canonical mixed-case ids and auth-profile suffixes, leaves unmatched custom refs untouched, and covers those cases in `src/cli/capability-cli.test.ts`; close or supersede the other open fix after that choice is made.

## Reproduction Assessment

Yes. The linked bug gives concrete `openclaw infer model run --model anthropic/CLAUDE-OPUS-4-7 --prompt ...` steps, and current main source confirms both local and gateway dispatch still use the raw override value before this PR.

## Solution Assessment

No, not as written. Catalog-backed canonicalization is the right narrow direction, but the helper should split trailing auth-profile suffixes and run the case-insensitive catalog lookup for all full provider/model refs, including all-lowercase variants of mixed-case canonical catalog ids.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Split auth profile suffixes before catalog lookup:** `src/cli/capability-cli.ts:566`
  - body: Local model refs support a trailing `@profile`: `resolveSimpleCompletionSelectionForAgent` strips it and uses it as `profileId`. This helper passes the suffix into `findModelInCatalog`, so `anthropic/CLAUDE-OPUS-4-7@work` will not match the catalog and will still dispatch the uppercase model. Split with `splitTrailingAuthProfile` before lookup and reattach the suffix to the canonical ref.
  - confidence: 0.88
- **[P2] Canonicalize lowercase mismatches too:** `src/cli/capability-cli.ts:561`
  - body: The guard skips catalog lookup whenever the full ref is already lowercase. Bundled catalogs include canonical mixed-case ids such as `deepseek-ai/DeepSeek-V3.2` and `Qwen/Qwen3-32B`, so a lowercased user variant is still a case-only catalog mismatch but would be forwarded raw. Run the catalog lookup for full provider/model refs and return the catalog id when it differs, regardless of whether the input contains uppercase.
  - confidence: 0.84
