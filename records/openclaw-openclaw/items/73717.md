## Summary

Keep this PR open. Current main still does not implement the intended `infer model run --model` canonicalization: the local path passes the raw override into model preparation, the gateway path splits and forwards the raw provider/model strings, and current tests do not cover mixed-case catalog refs. The PR remains a focused implementation candidate for the still-open linked bug #73715, and the provided diff does not show unrelated security-sensitive or supply-chain changes.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #73715. Review and land this PR, or an equivalent fix, that canonicalizes only unique case-insensitive catalog matches before both local and gateway `infer model run` dispatch while preserving custom or ambiguous mixed-case refs; keep the local and gateway regression tests with no-match preservation coverage.
