## Summary

Keep this PR open. Current main still does not implement the intended `infer model run --model` canonicalization: the local path passes the raw override into model preparation, the gateway path splits and forwards raw provider/model strings, and current tests do not cover mixed-case catalog refs. The PR is still open, unmerged, and remains a focused implementation candidate for open bug #73715. The latest PR diff is security-narrow: it only touches `src/cli/capability-cli.ts` and `src/cli/capability-cli.test.ts`, with no workflow, dependency, lockfile, package, release, install, or secrets-handling changes.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #73715. Review and land this PR, or an equivalent fix, that canonicalizes only unique case-insensitive catalog matches before both local and gateway `infer model run` dispatch while preserving custom or ambiguous mixed-case refs, with local and gateway regression tests.
