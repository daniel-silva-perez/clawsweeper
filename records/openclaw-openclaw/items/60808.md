## Summary

Keep this PR open. Current main still does not advertise Discord `upload-file`, still has no Discord message-action branch for `action === "upload-file"`, and would still reject that action as unsupported. The PR remains a relevant candidate for linked bug #60652, though maintainers should reconcile it with sibling upload-file PRs and require focused dispatch/runtime tests before landing.

## Best Possible Solution

Keep this PR open as a plausible implementation candidate for #60652. The best path is to choose one canonical Discord upload-file PR, ensure it adds explicit discovery plus dispatch/runtime tests around filePath/path/media, contentless file sends, `silent`, and `actions.messages` gating, then land that implementation and close the linked bug.
