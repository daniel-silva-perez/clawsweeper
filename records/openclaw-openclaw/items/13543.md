## Summary

Keep this issue open. Current main still has session/agent-level sandbox placement plus sandbox-only tool availability filtering, but it does not provide a per-tool execution placement policy that can force web_fetch/browser into a sandbox while leaving exec/message/cron/gateway on host in the same session. The existing maintainer discussion also explicitly says to keep this open and warns against conflating tool filtering, browser target selection, and elevated exec escape behavior with the requested feature.

## Best Possible Solution

Keep this open as a core sandbox/tool-routing enhancement. The best path is a documented, schema-backed per-tool execution placement policy that can route selected tools such as web_fetch and browser through a sandbox backend while selected host tools such as exec, message, cron, and gateway remain on host in the same session, with tests that distinguish placement from tools.sandbox.tools filtering and tools.elevated exec escape behavior.
