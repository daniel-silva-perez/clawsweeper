## Summary

Keep this issue open. Current main still provides session/agent-level sandbox selection, sandbox-only tool allow/deny filtering, and browser target selection, but it does not implement the requested per-tool execution placement policy that can force web_fetch/browser into a sandbox while keeping exec/message/cron/gateway host-routed in the same session.

## What This Changes

The issue requests a schema-backed per-tool sandbox placement policy so selected tools can run in a sandbox while other tools remain host-routed in the same session.

## Best Possible Solution

Keep this open as a core sandbox/tool-routing enhancement. The best path is a documented, schema-backed per-tool execution placement policy that can route selected tools such as web_fetch and browser through a sandbox backend while selected host tools such as exec, message, cron, and gateway remain on host in the same session, with regression tests that distinguish placement from tools.sandbox.tools filtering and tools.elevated exec escape behavior.
