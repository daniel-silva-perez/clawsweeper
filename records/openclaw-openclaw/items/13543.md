## Summary

Keep this issue open. Current main still has session/agent-level sandbox placement, sandbox-only tool availability filtering, sandbox browser configuration, and elevated exec escape behavior, but not the requested schema-backed per-tool execution placement policy for routing web_fetch/browser into a sandbox while leaving trusted tools host-routed in the same session.

## What This Changes

Add configurable per-tool sandbox execution placement so selected tools such as web_fetch and browser always run in a sandbox while exec, message, cron, and gateway can remain host-routed.

## Best Possible Solution

Add a documented, schema-backed core tool placement contract that can sandbox selected tools such as web_fetch and browser while leaving selected trusted tools host-routed in the same session, with regression tests proving the behavior is separate from tools.sandbox.tools filtering, sandbox browser settings, and tools.elevated exec behavior.
