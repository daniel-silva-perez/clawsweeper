## Summary

Keep this issue open. Current main still has session/agent-level sandbox selection, sandbox-only tool allow/deny filtering, browser sandbox settings, and elevated exec escape behavior, but not the requested schema-backed per-tool execution placement that can force `web_fetch` or `browser` into a sandbox while keeping `exec`, `message`, `cron`, and `gateway` host-routed in the same session. The existing maintainer-side Codex review also explicitly kept the issue open for product/security follow-up.

## What This Changes

Add configurable per-tool sandbox execution so selected untrusted tools can be isolated while trusted tools remain host-routed in the same session.

## Best Possible Solution

Add a documented, schema-backed core tool placement contract that can sandbox selected tools such as `web_fetch` and `browser` while leaving selected trusted tools host-routed in the same session, with regression tests proving the behavior is separate from `tools.sandbox.tools` filtering and `tools.elevated` exec escape behavior.
