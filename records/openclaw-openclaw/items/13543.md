## Summary

Keep this open. Current main still provides session/agent-level sandbox placement, sandbox-only tool availability filtering, browser target controls, and elevated exec escape behavior, but it does not implement the requested schema-backed per-tool execution placement policy. The existing maintainer discussion also explicitly treats related Docker sandbox work as partial only and keeps this as security-sensitive architecture work.

## What This Changes

The issue requests selective per-tool sandbox execution so untrusted tools such as `web_fetch` and `browser` can be forced into a sandbox while trusted tools such as `exec`, `message`, `cron`, and `gateway` remain host-routed in the same session.

## Best Possible Solution

Add a documented, schema-backed core tool placement contract that can sandbox selected tools such as `web_fetch` and `browser` while leaving selected trusted tools host-routed in the same session, with regression tests proving placement remains separate from tool filtering, browser target selection, and elevated exec behavior.
