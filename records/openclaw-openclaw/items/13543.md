## Summary

Keep this issue open. Current main still has session/agent-level sandbox modes, sandbox-only tool allow/deny filtering, browser target selection, and elevated exec escape behavior, but not the requested per-tool execution placement policy that can force web_fetch/browser into a sandbox while keeping exec/message/cron/gateway host-routed in the same session.

## What This Changes

Request configuration for per-tool sandbox execution independent of the session sandbox mode, so selected external-content tools can always run isolated while trusted tools stay on the host.

## Best Possible Solution

Add a documented, schema-backed core tool placement contract that can sandbox selected tools such as web_fetch and browser while leaving selected trusted tools host-routed in the same session, with regression tests proving the behavior is separate from tools.sandbox.tools filtering and tools.elevated exec behavior.
