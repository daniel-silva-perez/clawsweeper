## Summary

Keep this issue open. Current main still exposes session/agent-level sandbox selection, sandbox tool allow/deny filtering, browser target selection, and exec elevated escape behavior, but not the requested schema-backed per-tool placement policy that can force web_fetch/browser into a sandbox while keeping exec/message/cron/gateway host-routed in the same session.

## What This Changes

The issue requests configurable per-tool sandbox placement so selected untrusted-content tools run in a sandbox while other tools remain on the host in the same session.

## Best Possible Solution

Add a documented, schema-backed tool placement contract in core sandbox/tool routing that can sandbox selected tools such as web_fetch and browser while leaving selected trusted tools host-routed in the same session, with tests proving placement remains separate from tool filtering and elevated exec.
