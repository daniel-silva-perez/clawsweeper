## Summary

Keep open. Current main still has session/agent-level sandbox placement plus sandbox-only tool availability filtering, browser target controls, and elevated exec escape behavior, but not the requested schema-backed per-tool execution placement policy; the issue discussion also treats this as still-open security architecture work.

## What This Changes

The issue requests configurable per-tool sandbox execution so selected tools such as `web_fetch` and `browser` can be forced into a sandbox while trusted tools such as `exec`, `message`, `cron`, and `gateway` remain host-routed in the same session.

## Best Possible Solution

Add a documented, schema-backed core tool placement contract that can sandbox selected tools such as `web_fetch` and `browser` while leaving selected trusted tools host-routed in the same session, with regression tests proving placement remains separate from tool filtering, browser target selection, and elevated exec behavior.
