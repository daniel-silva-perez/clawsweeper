## Summary

Keep this open. Current main still has session/agent-level sandbox placement plus sandbox-only tool availability filtering, browser target selection, and elevated exec escape behavior, but not the requested schema-backed per-tool execution placement policy.

## What This Changes

Add per-tool sandbox execution placement so tools such as web_fetch and browser can be forced into a sandbox while trusted tools such as exec, message, cron, and gateway remain host-routed in the same session.

## Best Possible Solution

Add a documented, schema-backed core tool placement contract that can sandbox selected tools such as web_fetch and browser while leaving selected trusted tools host-routed in the same session, with regression tests proving it is separate from sandbox tool filtering, browser target selection, and elevated exec behavior.
