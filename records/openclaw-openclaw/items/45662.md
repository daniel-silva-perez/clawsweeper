## Risks / Open Questions

- A process-local active-client Map would not fix proactive IRC sends that execute from a separate CLI process unless sends are routed through the gateway or another monitor-owned process.
- Publishing a monitor client as active immediately after connect can still race channel membership because current readiness resolves after JOIN commands are sent, not after confirmed membership.
- The PR still needs targeted regression tests for reuse, no-monitor fallback, identity-safe cleanup, and JOIN readiness before it is safe to land.
