## Summary

Keep this PR open. Current main still does not implement the monitor-client reuse proposed here: proactive IRC sends without an already-provided ready client still create a transient connection, send PRIVMSG, and quit. The same-author #47234 stack is open and includes this transport fix plus broader routing work, so it is not a shipped supersession. The PR also still has real maintainer review questions around process boundaries, JOIN readiness, and missing regression tests.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a focused IRC transport fix that reuses a verified joined monitor or gateway-owned connection when available, preserves the no-monitor fallback intentionally, handles cross-process proactive sends explicitly, and adds targeted tests for reuse, fallback, identity-safe cleanup, and JOIN readiness. Coordinate with #47234 so this smaller IRC delivery fix can land separately from the broader groupScope routing proposal if maintainers want the bug fix first.
