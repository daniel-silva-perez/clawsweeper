Treat this PR and #74245 as source work. Implement a narrow DeepSeek provider policy fix in `extensions/deepseek`.

Observable bug: sparse DeepSeek model rows can be materialized with `DEFAULT_CONTEXT_TOKENS` and zero cost because current main lacks a DeepSeek `provider-policy-api.ts` artifact. Add a single-line user-facing changelog fix entry crediting `@moonlightb`.

Do not change package dependencies, lockfiles, CI workflows, release scripts, provider pricing outside the manifest source of truth, or unrelated providers.
