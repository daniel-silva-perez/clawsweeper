## Summary

Keep open. Current main has fallback transition state, lifecycle events, `/status` visibility, WebUI indicators, and verbose-mode reply notices, but it still does not provide the requested default-on per-session/topic switch and recovery notifications or a dedicated disable/config knob.

## What This Changes

The issue requests built-in, default-on notifications to the affected session/topic when model fallback activates and when the primary model recovers, with a config option to disable them.

## Best Possible Solution

Define a fallback notification policy separate from verbose mode: default-on activation and recovery notices delivered to the affected session/topic, a documented disable knob, privacy-safe reason formatting, throttling/coalescing rules, and tests proving verbose-off chat delivery plus one-time recovery clearing.
