## Summary

Keep this issue open. Current main has useful fallback lifecycle plumbing, per-session fallback state, `/status` visibility, and verbose-mode visible switch/clear notices, but it still does not implement the requested default-on bidirectional per-session/topic notifications with a disable knob. The earlier auto-announce request is already closed as superseded by this broader item, while the linked recovery-probe request remains open for the underlying recovery semantics.

## What This Changes

Request default-on user-visible notifications to the affected session/topic when model fallback activates and when the primary model recovers, with configuration to disable the notices.

## Best Possible Solution

Keep this open and define a product-level fallback notification policy separate from verbose mode: default-on switch and recovery notices delivered to the affected session/topic, a documented disable/config knob, privacy-safe reason formatting, and tests proving verbose-off chat delivery plus recovery clearing. Reuse the existing fallback transition and lifecycle plumbing instead of treating `/status` visibility or verbose notices as the final user-facing behavior.
