## Summary

Keep open. Current main still tracks fallback transitions and exposes them through lifecycle events, verbose-mode reply notices, /status, and Control UI indicators, but it does not implement the requested default-on per-session/topic fallback and recovery chat notifications when verbose mode is off, and there is no dedicated disable/config knob.

## What This Changes

The issue requests default-on notifications to the affected session/topic when model fallback activates and when the primary model recovers, with a way to disable them.

## Best Possible Solution

Define a product-owned fallback notification policy separate from verbose mode: default-on activation and recovery notices delivered to the affected session/topic, a documented disable knob, privacy-safe reason formatting, throttling/coalescing rules, and tests proving verbose-off chat delivery plus one-time recovery clearing.
