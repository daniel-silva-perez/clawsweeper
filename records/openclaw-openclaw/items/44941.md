## Summary

Keep open. Current main has fallback transition state, lifecycle events, `/status` visibility, and verbose-mode chat notices, but it still does not implement the requested default-on per-session/topic fallback and recovery notifications or a dedicated disable/config knob.

## What This Changes

The issue requests default-on notifications to the affected session/topic when model fallback activates and when the primary model recovers, with a config option to disable them.

## Best Possible Solution

Define a core fallback notification policy separate from verbose mode: default-on switch and recovery notices delivered to the affected session/topic, a documented disable knob, privacy-safe reason formatting, throttling/coalescing, and tests proving verbose-off chat delivery plus recovery clearing.
