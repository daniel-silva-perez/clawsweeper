## Summary

Keep this issue open. Current main has fallback lifecycle events, per-session fallback state, `/status` visibility, and verbose-mode switch/recovery notices, but it still does not provide the requested default-on per-session/topic chat notifications or a dedicated disable/config knob. The related auto-announce request #30245 is already closed as superseded by this broader issue, while #44935 remains open for the underlying recovery-probe semantics.

## What This Changes

Add default-on per-session/topic notifications when model fallback activates and when the primary model recovers, with a config option to disable them.

## Best Possible Solution

Define a product-level fallback notification policy separate from verbose mode: default-on switch and recovery notices delivered to the affected session/topic, a documented disable/config knob, privacy-safe reason formatting, throttling/coalescing, and tests for verbose-off chat delivery plus recovery clearing. Reuse the existing fallback transition and lifecycle plumbing, and coordinate the recovery notice semantics with #44935.
