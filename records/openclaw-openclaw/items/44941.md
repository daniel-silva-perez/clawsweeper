## Summary

Keep open. Current main has fallback lifecycle tracking, verbose-mode reply notices, /status visibility, and Control UI indicators, but it still does not provide the requested default-on fallback activation and recovery notices in the affected chat/session when verbose mode is off, nor a dedicated disable knob.

## What This Changes

Request default-on bidirectional model fallback notifications for the affected session/topic, including activation, recovery, and a config switch to disable them.

## Best Possible Solution

Define a product-owned fallback notification policy that sends one-time activation and recovery notices to the affected conversation by default, uses privacy-safe reason formatting, has documented throttling and recovery semantics, and exposes a stable config switch to disable or tune the behavior.
