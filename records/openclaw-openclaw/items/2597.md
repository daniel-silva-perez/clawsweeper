## Summary

Current main still does not implement the requested passive agent-visible `context=X%` field in the system-prompt Runtime line. Existing `/status`, `/context`, hook metadata, post-compaction context refresh, and the community `openclaw-workspace-sync` plugin are useful mitigations, but they do not provide the per-turn Runtime-line primitive proposed here. The latest maintainer-facing issue comment also explicitly kept this issue open.

## Best Possible Solution

Keep this issue open until OpenClaw either implements bounded context-usage plumbing from the latest reliable session token snapshot into embedded-runner params, `RuntimeInfoInput`, and `buildRuntimeLine()` with focused tests/docs, or a maintainer explicitly decides that existing status commands, UI warnings, context/memory plugins, and `openclaw-workspace-sync` are the intended product boundary.
