## Summary

Keep open. Current main still lacks the requested native gateway-node file transfer surface, and the issue is paired with open PR #67495 as the active same-author implementation path. Because the feature grants remote filesystem read/write through paired nodes, it needs maintainer and security review before this tracking issue can close.

## What This Changes

Add bidirectional gateway-to-node file transfer via new `file.read`/`file.write` node commands plus `file_pull`/`file_push` actions on the `nodes` agent tool, with optional CLI helpers.

## Best Possible Solution

Keep this as the tracking issue until maintainers accept PR #67495 or an equivalent implementation covering the node RPC contract, gateway command policy, `nodes` agent tool and CLI/user-facing surfaces, node-side support or explicit follow-up tracking, tests, docs, and clear size/security limits.
