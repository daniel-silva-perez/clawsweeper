## Summary

Keep open. Current main still lacks native gateway-node file transfer in the `nodes` tool, shared node command constants, gateway command policy, headless node-host command advertising/handling, and CLI docs. The issue is also actively paired with open PR #67495 by the same author, so this should remain as the tracking issue until that PR or an equivalent security-reviewed implementation lands.

## What This Changes

The issue requests bidirectional gateway-node file transfer via `file.read`/`file.write` node commands plus `file_pull`/`file_push` actions on the `nodes` agent tool, with optional CLI helpers and size/security limits.

## Best Possible Solution

Keep this issue as the tracking item until maintainers merge PR #67495 or an equivalent implementation that covers the node RPC contract, gateway command policy, `nodes` agent tool and CLI/docs surfaces, node-side support or explicit follow-up tracking, focused tests, and size/security limits.
