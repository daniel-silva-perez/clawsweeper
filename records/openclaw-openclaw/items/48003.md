## Summary

Current main still matches the reported failure shape: steer injection is attempted only while `isStreaming` is true, the embedded/reply queue helpers reject active non-streaming handles, and Discord inbound jobs still serialize same-session work by queue key before the reply pipeline can steer. Open PR #73365 is the right active implementation lane, but it is unmerged and reviewers should confirm it covers both the lifecycle gate and the Discord delivery path before this issue closes.

## Best Possible Solution

Keep this issue open and review, land, or extend #73365 as the canonical implementation path. The final fix should make steer injection lifecycle-aware for active non-streaming runs, preserve compaction/startup/teardown safeguards, and prove that Discord same-session inbound messages can reach the steer path during an active main-session turn or deliberately track that delivery gap separately.
