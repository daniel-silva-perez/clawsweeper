## Summary

Current main at 9bb1e59447a5fdf2c0164672df6c748188b26442 still matches the reported failure shape: steer injection is gated on streaming state, active non-streaming embedded/reply runs reject queued messages, and Discord inbound work is still serialized by the same session key before it can reach the reply pipeline. Open maintainer PR #73365 is the active implementation lane and remains unmerged, so this issue should stay open until that work lands or is replaced.

## Best Possible Solution

Keep this issue open and use #73365 as the canonical implementation path. The final fix should make steer injection lifecycle-aware for active non-streaming embedded and reply runs, preserve compaction/startup/teardown safeguards, and either prove Discord same-session inbound messages can reach the steer path during an active main-session turn or split that delivery gap into a clear follow-up.
