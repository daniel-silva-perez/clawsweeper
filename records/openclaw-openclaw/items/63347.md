## Summary

Keep this PR open. Current main still does not implement `channels.msteams.webhook.host`: the MS Teams runtime reads only the webhook port/path, the strict config schema rejects a host field, and the docs still list only port/path. The PR is therefore not obsolete, but it should be reviewed/rebased and address the existing doc/logging review comments before merge.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Rebase it onto current main, preserve the backwards-compatible default when `host` is omitted, add `host` to the MS Teams schema/types/runtime/docs/tests, and address the existing review comments by keeping the quick-start externally reachable and logging the actual or explicitly unspecified bind address.
