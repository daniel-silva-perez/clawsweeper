## Summary

Keep this PR open. Current main still does not support `channels.msteams.webhook.host`, and the PR is a narrow implementation candidate, but it still needs the existing review blockers addressed before merge: avoid making loopback the baseline quick-start config and avoid logging `0.0.0.0` for an omitted host when Node listens on the unspecified address.

## What This Changes

The PR adds an optional `channels.msteams.webhook.host` config field, forwards it to the MS Teams Express webhook listener, updates bind logging, docs, and lifecycle tests.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Update it to preserve default listen behavior when `host` is omitted, pass a trimmed configured host to the MS Teams webhook listener, document loopback as an optional hardened deployment setting rather than the baseline quick start, log the actual or explicitly unspecified bind address, and refresh generated config metadata if the channel config check reports drift.
