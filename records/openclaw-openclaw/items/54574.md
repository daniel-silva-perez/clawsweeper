## Summary

Keep this PR open. Current main still does not implement `dangerouslyAllowHostNetwork`: host Docker networking is blocked unconditionally, the public config/schema surfaces do not expose the requested flag, and tests/docs still assert or describe host networking as blocked. The linked feature issue is closed, but this PR is still the visible implementation candidate rather than work already shipped on main.

## Best Possible Solution

Keep this PR open as the implementation candidate. A maintainer should decide whether the host-network break-glass escape hatch is acceptable; if yes, update the current sandbox Docker config/type/schema/validation/audit/docs/tests together, rebase onto main, and run the appropriate changed sandbox/security gates before merge.
