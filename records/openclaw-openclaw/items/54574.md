## Summary

Keep open. Current main still blocks Docker `network: "host"` unconditionally and has no `dangerouslyAllowHostNetwork` config, schema, docs, or test coverage. The PR is a coherent implementation candidate for the linked feature request, but it deliberately broadens sandbox networking and needs maintainer/secops review rather than cleanup closure.

## What This Changes

This PR adds a Docker sandbox `dangerouslyAllowHostNetwork` boolean and threads it through network validation, config/schema/help/types, Docker create-args behavior, security-audit remediation, and targeted tests.

## Best Possible Solution

Land a maintainer-approved version that keeps host networking blocked by default while adding a clearly named Docker-only break-glass flag across runtime validation, config resolution, public schema/help/docs, Docker create args, audit messaging, and focused sandbox/config/security tests.
