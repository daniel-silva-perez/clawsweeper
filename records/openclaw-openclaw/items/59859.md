## Summary

Keep this PR open. Current main still does not include a native Linux companion app, the public platform docs still describe Linux companion apps as planned, and the gateway client ID/handshake allowlists still omit a Linux native UI client. This PR remains an active implementation candidate for maintainer request #75, with alternative implementation track #61576 also open, so cleanup closure would be premature.

## Best Possible Solution

Keep this PR open and route it through maintainer product/security review for #75. Maintainers should decide whether this C/GTK implementation, the Rust/GTK alternative in #61576, or a smaller staged Linux companion path should become canonical, then require the chosen path to align gateway client IDs and handshake behavior, define packaging/support scope, resolve review findings, and land with focused protocol/app tests.
