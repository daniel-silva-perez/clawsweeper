## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve schedule variants when editing cron jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `schedule_kind = "cron"` and serializes the textbox as `expr`. Existing jobs may use `kind: "every"` or `kind: "at"`, so saving an unrelated field can fail validation or change when the job runs. Carry the original schedule variant through the dialog or block editing unsupported variants.
  - confidence: 0.9
- **[P2] Resolve session logs from session IDs:** `apps/linux/src/section_sessions.c:265-266`
  - body: This probes `<storeDir>/<session-key>/transcript.jsonl`, but the gateway resolves transcript files from `sessionId` and optional `sessionFile` under the sessions directory. Normal local transcripts will be missed and the Log action falls back to the dashboard even when a local file exists.
  - confidence: 0.87
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: The URL builder interpolates `config->host` directly into `scheme://host:port`. For an IPv6 literal like `::1`, this produces malformed URLs such as `http://::1:18789/`, breaking health checks, WebSocket connection, and dashboard handoff on valid IPv6/custom-bind setups.
  - confidence: 0.84
- **[P2] Resolve cron logs with the session transcript layout:** `apps/linux/src/section_cron.c:186-190`
  - body: The cron Log action derives `<dirname(cronStore)>/sessions/<sessionKey>/transcript.jsonl`, but cron stores and session transcripts use different layouts and transcripts are resolved from session IDs/files under agent session directories. This usually misses existing local transcripts before falling back to the dashboard.
  - confidence: 0.82
