## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always writes `patch.schedule` as `{ kind: "cron", expr: ... }`. Current cron jobs may have `kind: "every"` or `kind: "at"`, so saving an existing non-cron job after changing unrelated fields can fail validation or change when it runs. Carry the original schedule variant through the dialog or block editing unsupported variants.
  - confidence: 0.9
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:265-266`
  - body: This builds the local transcript path as `<storeDir>/<session-key>/transcript.jsonl`, but the gateway stores and resolves transcripts as direct session-id files under the sessions directory, and the PR parser already exposes `session_id`. As written, the Log action usually misses existing local transcripts and falls back to the dashboard.
  - confidence: 0.87
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: The URL builder interpolates `config->host` directly into `scheme://host:port`. For an IPv6 literal such as `::1`, that produces malformed URLs like `http://::1:18789`, breaking health checks, WebSocket connection, and dashboard handoff on valid IPv6/custom-bind setups. Normalize IPv6 literals to `[host]` before formatting these URLs.
  - confidence: 0.84
- **[P2] Resolve cron logs from session transcript paths:** `apps/linux/src/section_cron.c:188-190`
  - body: The cron Log action derives `<dirname(cronStore)>/sessions/<sessionKey>/transcript.jsonl`, but cron stores and session transcripts use different layouts, and transcripts are resolved from session IDs/files under agent session directories. This usually misses existing local transcripts before falling back to the dashboard.
  - confidence: 0.82
- **[P2] Preserve session-scoped cron targets:** `apps/linux/src/test_seams.c:42-47`
  - body: Valid cron targets include `session:<id>`, but the helper maps any unrecognized target to the first combo option, which later saves back as `isolated`. Editing a job with a session-scoped target can silently retarget future runs even when the user changes only unrelated fields.
  - confidence: 0.8
