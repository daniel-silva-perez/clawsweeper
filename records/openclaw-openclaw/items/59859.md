## Summary

Keep this PR open. Current main still does not ship a native Linux companion app, the public docs still describe Linux companion apps as planned, and the gateway protocol/server paths still do not recognize an `openclaw-linux` native UI client. The PR is an active implementation candidate for open maintainer request #75, with open alternative track #61576, so cleanup closure would be premature.

## Best Possible Solution

Keep this PR open and route it through maintainer product/security review for #75. The practical path is to choose whether this C/GTK implementation, the alternative #61576 Rust/GTK implementation, or a smaller staged Linux companion should become canonical, then require the chosen path to align gateway client IDs and handshake behavior, resolve open review/security findings, define packaging/support scope, and land with focused protocol and app tests.
