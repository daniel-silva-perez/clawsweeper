## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:293-296`
  - body: The edit save path always writes `patch.schedule` as `{ kind: "cron", expr: ... }`. Current cron jobs may have `kind: "every"` or `kind: "at"`, so saving an existing non-cron job after changing unrelated fields rewrites its schedule shape and can fail validation or change when the job runs. Carry the original schedule kind/fields through the dialog or prevent editing unsupported variants.
  - confidence: 0.9
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:265-266`
  - body: This builds the local transcript path as `<storeDir>/<session-key>/transcript.jsonl`, but the gateway stores/resolves transcripts as direct session-id files under the sessions directory, and the PR parser already exposes `session_id`. As written, the Log action usually misses existing local transcripts and falls back to the dashboard, which breaks offline/no-dashboard access.
  - confidence: 0.87
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: The URL builders interpolate `config->host` directly into `scheme://host:port`. For an IPv6 literal such as `::1`, that produces malformed URLs like `http://::1:18789`, breaking health checks, WebSocket connection, and dashboard handoff on valid IPv6/custom-bind setups. Normalize IPv6 literals to `[host]` before formatting these URLs.
  - confidence: 0.84
