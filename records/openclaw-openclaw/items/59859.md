## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `schedule_kind = "cron"` and serializes the textbox as `expr`. Existing jobs can validly use `kind: "every"` or `kind: "at"`, so saving an unrelated field rewrites the schedule shape and can fail validation or change when the job runs. Carry the original variant through the dialog or block editing unsupported variants.
  - confidence: 0.9
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:265-266`
  - body: This builds the local transcript path as `<store_dir>/<session-key>/transcript.jsonl`, but the gateway resolves transcript files from `sessionId` and optional `sessionFile`. Normal local transcripts will be missed, so the Log action falls back to the dashboard even when a local file exists.
  - confidence: 0.87
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: The URL builder interpolates `config->host` directly into `scheme://host:port`. For an IPv6 literal like `::1`, this produces malformed URLs such as `http://::1:18789/`, breaking health checks, WebSocket connection, and dashboard handoff on valid IPv6/custom-bind setups.
  - confidence: 0.84
- **[P2] Resolve cron logs with the session transcript layout:** `apps/linux/src/section_cron.c:188-190`
  - body: The cron Log action derives `<dirname(cronStore)>/sessions/<sessionKey>/transcript.jsonl`, but cron stores and session transcripts use different layouts and transcripts are resolved from session IDs/files under agent session directories. This usually misses existing local transcripts before falling back to the dashboard.
  - confidence: 0.82
- **[P2] Keep session-scoped cron targets intact:** `apps/linux/src/test_seams.c:42-47`
  - body: The reverse mapping sends unknown targets, including valid `session:<id>` targets, to index 0; saving then maps that index back to `isolated`. Editing a session-scoped cron job can silently retarget it away from the intended session, so preserve unsupported targets or block edits that cannot round-trip.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: Security review needs attention because the PR introduces a new native operator client with credential persistence, device identity, gateway control, SSH tunneling, and exec-approval authority.

Concerns:

- **[medium] Review native token and device identity storage:** `apps/linux/src/device_auth_store.c:222`
  - body: The PR persists operator device tokens and an Ed25519 private device seed in new Linux state files. The code uses restrictive file modes and zeroing in several paths, but this is a new supported operator credential store and should receive explicit secops review before merge.
  - confidence: 0.84
- **[medium] Review native exec approval handling:** `apps/linux/src/exec_approval_prompter.c:88`
  - body: The PR subscribes to `exec.approval.requested` events and resolves allow/deny decisions through a new native desktop path. Because these approvals authorize host command execution policy, the surface should be reviewed against the existing security ownership model before shipping.
  - confidence: 0.82
- **[medium] Review remote gateway SSH tunnel policy:** `apps/linux/src/remote_tunnel_command.c:15`
  - body: The Linux companion adds SSH local-forwarding for remote mode and builds a first-class tunnel command surface. The latest branch includes argv-smuggling guards, but the host-key, identity, and remote-control trust model should be explicitly reviewed before enabling this in a supported app.
  - confidence: 0.76
