## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `schedule.kind` as `cron` with `expr`, but current jobs can use `kind: "every"` or `kind: "at"`. Saving one of those jobs after changing unrelated fields can fail validation or silently change when it runs; carry the original schedule variant through the dialog or block unsupported edits.
  - confidence: 0.9
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:265-266`
  - body: This probes `<store_dir>/<session-key>/transcript.jsonl`, while the gateway resolves transcript files from `sessionId` and optional `sessionFile` under the sessions directory. The Log action will usually miss existing local transcripts and fall back to the dashboard, breaking offline or no-dashboard access.
  - confidence: 0.87
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: The URL builder formats `scheme://host:port` with `config->host` directly. For an IPv6 literal such as `::1`, this produces `http://::1:18789/`, which is not a valid URI and breaks health, dashboard, and handoff paths on IPv6/custom-bind setups.
  - confidence: 0.84
