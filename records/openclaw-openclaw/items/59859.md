## Summary

Keep this PR open. Current main still does not ship a native Linux companion app, the public platform docs still describe Linux companion apps as planned, and the gateway protocol/handshake surfaces still lack an `openclaw-linux` native UI client. The PR remains an active implementation candidate for the open maintainer request in #75, with #61576 as an alternative track rather than a merged or canonical replacement.

## Best Possible Solution

Keep this PR open and route it through maintainer product/security review for #75. Maintainers should decide whether this C/GTK implementation, the Rust/GTK alternative in #61576, or a smaller staged Linux companion should become canonical, then require the chosen path to align gateway client IDs and native UI handshake behavior, resolve review findings, define packaging/support scope, and land with focused protocol/app tests.
