## Summary

Keep open. Current main still enforces `tools.sessions.visibility` defaulting to `tree`, still returns the same terse `sessions_send` visibility error, and I found docs/changelog coverage for the security default but no current migration warning, doctor check, or clearer OC-07-style diagnostic for isolated cron workflows. The related #18967 is adjacent, but it tracks a new parent-scoped send capability rather than the upgrade/UX mitigation requested here.

## Best Possible Solution

Keep the security default, but add a narrow UX mitigation: improve the `sessions_send` forbidden message with security-default context and a docs link, document isolated-cron migration options, and consider a doctor/config audit that flags jobs likely relying on cross-agent `sessions_send`. If parent-only delivery is desired, track that separately through #18967 rather than using broad `visibility=all` as the only ergonomic path.
