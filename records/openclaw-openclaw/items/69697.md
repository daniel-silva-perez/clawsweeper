## Summary

Keep this PR open. Current main still has the runtime hole the PR targets: `stableStringify(undefined)` returns the JavaScript value `undefined`, `stableStringifyFallback` returns it as though it were a string, and `digestStable` then passes it to `createHash().update(...)`, which throws `ERR_INVALID_ARG_TYPE`. The proposed patch is narrow and touches only `src/agents/tool-loop-detection.ts`; the main gap is that it should add regression coverage for undefined tool params and undefined non-Error hash formatting before landing.

## Best Possible Solution

Keep this PR open and review it as a valid narrow bug fix. The best path is to add or request focused regression coverage in `src/agents/tool-loop-detection.test.ts` for `hashToolCall("tool", undefined)` and the undefined error-formatting path, then land the minimal serialization guard if the targeted agent test lane passes.
