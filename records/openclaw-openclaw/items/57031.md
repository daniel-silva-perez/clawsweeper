## Summary

Keep open. Current main still suppresses QueuedFileWriter setup and append failures without an operator-visible warning, and there is an open unmerged implementation PR that explicitly targets this issue. The reporter’s follow-up usefully narrows this to a low-severity diagnostic improvement rather than a crash/data-loss bug.

## Best Possible Solution

Keep this issue open until PR #57089 is reviewed. If the PR is still the right shape, land a scoped fix that preserves best-effort non-crashing writes while emitting a rate-limited diagnostic or warning after repeated failures, with tests for failure counting and reset-on-success. If maintainers reject that approach, close the PR with rationale and update this issue with the preferred diagnostic policy.
