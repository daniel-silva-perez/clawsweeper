## Summary

Keep this issue open. Current main still ships the Google Gemini CLI OAuth path, and the relevant auth flow can still produce the reported failure class: the desktop path opens the browser before starting the localhost callback wait helper, and the post-callback token exchange still uses a 10s guarded fetch path whose AbortError can bubble through the provider's generic OAuth help. Multiple comments report the same auth failure, and related #48879 is a separate post-login model/API 404 issue rather than a superseding auth-flow fix.

## Best Possible Solution

Keep this issue open for the Google plugin owner. Reproduce Gemini CLI OAuth on current main or the latest release, then make the callback listener ready before browser handoff and improve token-exchange timeout/network/account diagnostics; add focused regression tests for listener readiness and AbortError handling.
