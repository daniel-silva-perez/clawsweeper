## Summary

Keep this issue open. Current main still contains the same bundled Google Gemini CLI OAuth onboarding path, fixed localhost:8085 callback, browser-before-listener ordering, and narrow listen-error-only manual fallback. The issue discussion has multiple matching reports, including a note that direct Gemini CLI works while OpenClaw has no localhost:8085 listener, and I did not find a current-main source, test, docs, release, or related-item signal that fixes or supersedes that reported regression.

## Best Possible Solution

Keep this issue open for the Google plugin owner. The best path is to reproduce Gemini CLI OAuth on current main or the latest release with a real macOS/Homebrew setup, then harden `extensions/google/oauth.ts` and the shared callback helper so the localhost listener lifecycle is explicit before browser handoff, manual fallback is available for no-listener, timeout, and AbortError cases, and a focused regression test covers the desktop callback flow.
