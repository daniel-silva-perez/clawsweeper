## Summary

Keep this issue open. Current main still ships the bundled Google Gemini CLI OAuth path, still advertises automatic localhost:8085 callback capture, and the desktop flow still opens the browser before invoking the helper that creates the local callback server. The discussion includes multiple matching reports, including a useful note that direct Gemini CLI works while OpenClaw has no listener on localhost:8085, and I did not find a current-main source/test/docs change that fixes that callback-listener race or supersedes this report.

## Best Possible Solution

Keep this issue open for the Google plugin owner. Reproduce Gemini CLI OAuth on current main or the latest release with a real macOS/Homebrew setup, then harden `extensions/google/oauth.ts` and the shared callback helper so the localhost listener is ready before browser handoff or manual mode is available immediately. Add a focused regression test for the desktop callback flow, including timeout/no-listener behavior.
