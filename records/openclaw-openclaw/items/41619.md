## Summary

Keep this issue open. Current main still ships the Gemini CLI OAuth path, and the reported failure remains plausible: the browser handoff still happens before the localhost callback helper starts listening, the token exchange still uses the guarded 10s fetch path that can surface AbortError, and the provider still collapses OAuth failures into the same generic Gemini CLI access help. The prior ClawSweeper keep-open review remains materially accurate for the current SHA.

## Best Possible Solution

Keep this open for the Google plugin/auth owner. Reproduce Gemini CLI OAuth on current main or the latest release, then make the localhost callback listener ready before browser handoff and improve token-exchange timeout/network/account diagnostics. Add focused regression coverage for listener readiness before openUrl and for AbortError handling in the OAuth exchange path.
