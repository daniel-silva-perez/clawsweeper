## Summary

Keep this issue open. The current main branch still has the reported bundled Google Gemini CLI OAuth path, still advertises localhost:8085 automatic callback capture, and still opens the browser before awaiting the callback helper that starts the local listener. The issue discussion includes multiple matching reports, including a useful note that direct Gemini CLI works while OpenClaw has no localhost:8085 listener, and I did not find a current-main fix, focused local desktop callback regression test, or canonical superseding issue.

## Best Possible Solution

Keep this issue open for the Google plugin owner. Reproduce Gemini CLI OAuth on current main or the latest release with a real macOS/Homebrew setup, then harden extensions/google/oauth.ts and the shared callback helper so the localhost listener is ready before browser handoff or manual mode is available immediately, broaden fallback for timeout/AbortError/no-listener cases, and add a focused regression test for the desktop callback flow.
