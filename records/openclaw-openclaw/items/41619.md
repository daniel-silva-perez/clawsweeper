## Summary

Keep this issue open. Current main still contains the reported bundled Google Gemini CLI OAuth path, still advertises automatic localhost:8085 capture, opens the browser before awaiting the local callback helper, and only falls back to manual mode for listen/port errors. The discussion also has multiple matching reports, including a useful note that direct Gemini CLI works while OpenClaw has no localhost:8085 listener; I did not find a current-main implementation, test, docs, release, or related canonical issue that fixes or supersedes that regression.

## Best Possible Solution

Keep this issue open for the Google plugin owner. The best path is to reproduce Gemini CLI OAuth on current main or the latest release with a real macOS/Homebrew setup, then harden `extensions/google/oauth.ts` and the shared callback helper so the localhost listener lifecycle is explicit before browser handoff, manual fallback is available for no-listener, timeout, and AbortError cases, and a focused regression test covers the desktop callback flow.
