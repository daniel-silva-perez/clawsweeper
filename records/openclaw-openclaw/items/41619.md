## Summary

Keep this issue open. Current main still ships the Google Gemini CLI OAuth path, and the reported failure class is still plausible: the desktop flow opens the browser before starting the localhost callback listener, token exchange still goes through a 10s guarded fetch path that can surface AbortError, and the provider still wraps all OAuth failures with the same generic OAuth help shown in the report. The previous ClawSweeper keep-open review remains materially accurate on current main; the relevant files have no diff from that reviewed SHA to e583db63.

## Best Possible Solution

Keep this open for the Google plugin/auth owner. Reproduce Gemini CLI OAuth on current main or the latest release, then make the callback listener ready before browser handoff and improve token-exchange timeout/network/account diagnostics. Add focused regression coverage for listener readiness before `openUrl` and for AbortError handling in the OAuth exchange path.
