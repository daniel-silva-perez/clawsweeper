## Summary

Keep this PR open. Current main still has the same-root realpath containment behavior, so the managed-skill symlink warning problem is not implemented on main. The PR is a plausible narrower fix than related #45912, but it touches a security-sensitive skill path boundary, has no regression tests, and needs a rebase against the current loader shape before maintainer review.

## Best Possible Solution

Keep this PR open for maintainer review or consolidation. The best path is a rebased, tested managed-symlink fix that accepts symlinks from `~/.openclaw/skills` into explicitly trusted non-bundled skill roots such as `~/.agents/skills`, while preserving external symlink rejection and avoiding bundled-skill policy bypasses. Coordinate with #45912 rather than closing this PR as implemented or obsolete.
