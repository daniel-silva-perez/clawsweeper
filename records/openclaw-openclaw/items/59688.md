## Summary

Keep this PR open. Current main still enforces same-root realpath containment for skills, so the reported managed-symlink warning is not implemented, and the related broader PR remains open rather than a shipped or clearly canonical replacement.

## What This Changes

The PR adds an allowlist parameter to the skill path containment helper so managed-skill symlinks can resolve into other known skill scan roots without warning.

## Best Possible Solution

Land one rebased, tested skill-loader change that accepts managed symlinks into explicitly trusted non-bundled scan roots, preserves warnings for unknown escapes, and keeps bundled-source allowlist semantics intact.

## Reproduction Assessment

Yes. A focused fixture with a managed skill entry symlinked to a separately loaded agents-skill root would still reach `resolveContainedSkillPath`, fail the active-root realpath check, and warn on current main.

## Solution Assessment

Unclear. The narrower managed-root allowlist direction is plausible, but the submitted diff is stale and untested; the safer path is a rebased source-preserving fix coordinated with the broader open symlink PR.
