## Summary

Keep this PR open. Current main still does not retry Telegram HTTP 421 / Misdirected Request responses in either the default retry regex or the strict Telegram send path, and this PR remains a focused implementation candidate for the closed linked bug #48892.

## Best Possible Solution

Keep this PR open for maintainer review and either land it or an equivalent fix. The fix should add HTTP 421 / Misdirected Request handling to both the default retry path and the strict Telegram non-idempotent send predicate, with regression tests for both paths and a docs update if the behavior ships.
