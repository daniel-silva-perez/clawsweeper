## Summary

Keep this PR open. Current main still does not implement the PR's server-side dual-bind behavior: specific tailnet/custom bind hosts resolve to only that host, while local gateway callers still target loopback. The PR remains a plausible focused fix candidate, but it overlaps with open client-side host-resolution work in #50751 and changes documented tailnet behavior, so it needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose either this server-side dual-bind model or the client-side host-resolution model in #50751, then land one coherent fix with specific-IP regression tests, docs/security expectation updates, and targeted live proof for gateway.bind=tailnet and gateway.bind=custom internal calls.
