## Summary

Keep this PR open. Current main still does not implement the PR's intended server-side dual-bind behavior: non-loopback gateway listen hosts remain single-homed, while local gateway calls still target loopback. The PR is a plausible focused fix candidate, but it changes documented tailnet/locality expectations and overlaps open #50751's client-side host-resolution approach, so maintainers should choose one coherent path.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose either this server-side dual-bind model or #50751's client-side host-resolution model, then land one coherent fix with regression coverage for tailnet/custom bind internal calls, updated docs/security expectations, and targeted live proof for CLI management plus `sessions_spawn` paths.
