## Summary

Current main still does not implement this PR's intended server-side dual-bind behavior. Non-loopback gateway binds remain single-host listener targets, while local gateway calls still resolve to loopback, so this PR remains a plausible implementation candidate rather than obsolete. The work also overlaps open #50751's client-side host-resolution approach, so maintainers should choose one coherent model before landing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose either this server-side dual-bind model or #50751's client-side host-resolution model, then land one coherent fix with regression coverage for tailnet/custom bind internal calls, docs/security expectation updates, and targeted live proof for CLI management plus `sessions_spawn`.
