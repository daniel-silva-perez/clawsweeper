## Summary

Current main still does not implement this PR's intended server-side dual-bind behavior. Non-loopback gateway binds remain single-homed while local gateway calls still target loopback, so this PR is still a plausible implementation candidate and should not be closed as obsolete.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose either this server-side dual-bind model or #50751's client-side host-resolution model, then land one coherent fix with regression coverage for tailnet/custom bind internal calls, updated docs/security expectations, and targeted live proof for CLI management plus `sessions_spawn`.
