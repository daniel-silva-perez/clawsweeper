## Summary

Keep this PR open. Current main at 16906780fd2c481308ec4b80f96eb3b93d0add46 still does not implement the PR's server-side dual-bind behavior: specific tailnet/custom bind hosts still resolve to only that host, while local gateway callers still target loopback. The PR remains a focused candidate, but it changes documented tailnet behavior and overlaps with open client-side host-resolution work in #50751, so maintainer review is still needed.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose either the server-side dual-bind model in this PR or the client-side host-resolution model in #50751, then land one coherent fix with specific-IP regression tests, docs/security expectation updates, and targeted live proof for gateway.bind=tailnet and gateway.bind=custom internal calls.
