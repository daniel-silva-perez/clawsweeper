## Summary

Keep this issue open. Current main still does not implement Slack user-level `message.im` subscriptions or user-token identity routing for monitor DMs; `userToken` remains a supplemental read/write fallback, while Slack monitor startup, self-filtering, identity lookup, pairing replies, and normal reply delivery stay bot-token centered.

## Best Possible Solution

Implement this in the bundled Slack plugin: add and document the required Slack `user_events` manifest subscription for `message.im`, resolve the configured user-token identity during monitor startup, distinguish bot-level and user-level DM events, use the xoxp token only for eligible user-level replies when `userTokenReadOnly` is false, prevent self-loops, and add regression coverage for manifest output, event routing, token selection, docs, and reply delivery.
