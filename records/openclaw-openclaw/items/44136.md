## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: GitHub Copilot runtime auth always returns a token-derived baseUrl, and the embedded auth controller applies any prepared baseUrl to both runtimeModel and effectiveModel, so an explicit user-configured Copilot baseUrl can still be overwritten during initial auth or refresh. The PR is useful, but its patch is stale against the current provider/runtime-auth architecture and needs to be ported before it can merge.

## Best Possible Solution

Keep this PR open as useful contributor context, but port the behavior to the current GitHub Copilot provider/runtime-auth flow. The fix should preserve explicit custom Copilot baseUrl values, still use token-derived Business/Enterprise endpoints when the configured endpoint is absent/default or known token-derived, keep runtimeModel and effectiveModel synchronized, avoid eager Copilot runtime imports, and add regression coverage for initial auth, refresh, context-window-capped effectiveModel, and auth profile rotation.
