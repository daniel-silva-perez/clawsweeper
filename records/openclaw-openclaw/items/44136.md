## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: GitHub Copilot runtime auth always returns a token-derived baseUrl and the embedded auth controller always applies that prepared baseUrl to both runtimeModel and effectiveModel, so an explicit user-configured Copilot baseUrl can still be overwritten during auth or refresh. The submitted patch also targets the older runner shape/import path and needs to be ported to the current plugin/auth-controller flow before it is mergeable.

## Best Possible Solution

Keep this PR open as useful contributor context, but port the behavior to the current GitHub Copilot provider/runtime-auth flow. The best fix should preserve explicit custom Copilot baseUrl values, still apply token-derived Business/Enterprise endpoints when the configured endpoint is absent/default or known token-derived, keep runtimeModel and effectiveModel synchronized, avoid eager Copilot runtime imports, and add regression coverage for custom proxy, default-to-token endpoint, context-window-capped effectiveModel, refresh, and auth profile rotation.
