## Summary

Keep this issue open. Current main has some sanitization and a Telegram-local delivery-error policy, but the requested shared suppress/custom/owner-only policy for provider billing/rate-limit failures is still not implemented, and WhatsApp has no matching config surface.

## Best Possible Solution

Keep this open and implement a shared outbound provider/runtime error-reply policy before channel delivery. Classify billing, rate-limit, auth, and generic runtime failures, then allow per channel/account/group/topic/DM settings for normal reply, silent suppression, custom text, and owner-only routing. Wire it through WhatsApp and Telegram schemas, docs, and tests while preserving full diagnostics in logs/operator surfaces.
