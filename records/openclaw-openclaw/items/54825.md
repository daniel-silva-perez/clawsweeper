## Summary

Keep this issue open. Current main improves sanitization and generic group silence, but billing and rate-limit provider failures still become user-visible final replies, WhatsApp has no suppress/custom/owner-only error policy, and Telegram’s error policy is not wired as a shared outbound provider-error policy for the final reply path.

## Best Possible Solution

Keep this open and implement a shared outbound provider/runtime error-reply policy before channel delivery. Classify billing, rate-limit, auth, and generic runtime failures, then allow per channel/account/group/topic/DM settings for normal reply, silent suppression, custom text, and owner-only routing. Add WhatsApp and Telegram schema, docs, and tests while preserving full diagnostics in logs/operator surfaces.
