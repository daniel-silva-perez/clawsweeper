## Summary

Keep this issue open. Current main does not implement the reported Open WebUI visible-history sync: `chat.send` still updates OpenClaw transcript/event state only, there is no `chat-open-webui-sync.ts` helper or `webui.db`/`chat_message` write path, and the Open WebUI Docker smoke only verifies the normal Open WebUI -> OpenClaw API chat flow. The issue discussion includes a concrete local fix direction, but that work is not present on main and no linked implementation PR is currently attached to the issue.

## Best Possible Solution

Keep the issue open for maintainer/product design. The likely path is an explicit, supported Open WebUI history-sync integration that runs after successful external `chat.send` for WebUI-backed UUID sessions, distinguishes `webchat-ui` from `openclaw-control-ui` by client id, and includes regression coverage for both transcript context and Open WebUI-visible history. If maintainers do not want core to own Open WebUI DB writes, split the work into a documented plugin/API surface plus a ClawHub Open WebUI sync plugin rather than closing the bug as already fixed.
