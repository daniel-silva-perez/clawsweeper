## Summary

Keep this issue open. Current main still writes Control UI chat sends into OpenClaw transcript/broadcast paths, but there is no Open WebUI visible-history mirror, no `chat-open-webui-sync` helper, and no code that updates Open WebUI `webui.db`, `chat.chat`, or `chat_message` after an external Control UI turn.

## Best Possible Solution

Keep this open for maintainer follow-up. The best fix is an explicit supported sync path after successful external `chat.send` into WebUI-backed UUID sessions, using client id to skip only native `webchat-ui` traffic while allowing `openclaw-control-ui`. If maintainers do not want core to write Open WebUI storage directly, define a small plugin/API seam for post-turn external-history sync and add regression coverage for both OpenClaw transcript context and Open WebUI visible history.
