## Summary

Keep open. Current main still keeps the memory capability registry in module-local state while Memory Wiki bridge/status read artifacts through that same module-local path, and bundled plugin runtime mirroring still makes the reported split-singleton failure plausible. The same author has open draft PR #75064 for this bug, so the issue should stay open while that paired implementation is reviewed and validated.

## What This Changes

Fix Memory Wiki bridge/status so public artifacts registered by memory-core remain visible when bundled plugins execute through the managed plugin-runtime-deps mirror.

## Best Possible Solution

Land or replace #75064 with a narrow shared or host-owned memory capability registry that all mirrored runtime module instances observe, while preserving the existing plugin SDK API and adding regression coverage for Memory Wiki bridge import and status artifact counts.
