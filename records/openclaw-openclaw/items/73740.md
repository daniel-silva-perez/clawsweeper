## Summary

Keep open. Current main still lacks native per-peer WhatsApp outbound-only control: direct-chat config only supports prompt metadata, inbound DM admission is governed by dmPolicy/allowFrom/pairing, admitted inbound messages continue into session/reply dispatch, and outbound authorization still reuses allowFrom rather than a separate outbound-only permission surface.

## Best Possible Solution

Keep this issue open and implement the feature in the bundled WhatsApp plugin/config surface: add a direction-specific per-direct-peer inbound policy, enforce it immediately after WhatsApp DM admission/normalization and before read receipts, enqueueing, session metadata, hooks, agent execution, or fallback replies, keep outbound sends authorized for notification-only peers, and update the WhatsApp types, strict schema, generated config metadata/help, public WhatsApp docs, and focused tests for an outbound-allowed peer whose inbound messages are ignored.
