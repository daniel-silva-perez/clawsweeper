## Summary

Keep this PR open. Current main still lacks the requested Slack `channel-list` and `search` message actions, but this branch is not mergeable as-is: it has runtime/config-boundary blockers, returns search data outside the shared formatter contract, can advertise search for a non-invokable default account, silently truncates channel discovery, and is stale against current main's plugin-contract layout.

## What This Changes

The PR adds Slack `channel-list` and `search` message actions, Slack action config/schema entries, generated config metadata, and a channel-plugin contract expectation update.

## Best Possible Solution

Keep the feature in the Slack plugin, rebase it onto current main, and land a single coherent implementation: `channel-list` should expose pagination or truncation metadata, `search` should use the resolved account/config runtime path and return the shared search `results.messages` envelope, and the search work should be reconciled with #65691 rather than split into incompatible paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Pass resolved Slack config into search helpers:** `extensions/slack/src/actions.ts:334`
  - body: `resolveUserToken()` calls `loadConfig()` from inside a Slack action helper. Current main passes resolved `cfg` through `handleSlackAction()` and the runtime-action guard rejects ambient `loadConfig()` calls in action paths, so this should use the already-resolved config/account data instead.
  - confidence: 0.94
- **[P2] Return search hits under the results envelope:** `extensions/slack/src/action-runtime.ts:494`
  - body: The shared formatter renders search rows from `payload.results.messages`, but this returns Slack hits as top-level `messages`/`total`. Successful Slack searches will fall back to the generic summary instead of the expected search-results table.
  - confidence: 0.9
- **[P2] Advertise search only for the invokable account:** `extensions/slack/src/message-actions.ts:55-63`
  - body: When discovery is not scoped to an `accountId`, this advertises `search` if any Slack account has a user token and search enabled. Invocation without `accountId` resolves the default account, so a non-default search-capable account can expose an action the default invocation cannot run.
  - confidence: 0.84
- **[P2] Report channel-list truncation to callers:** `extensions/slack/src/actions.ts:302`
  - body: `channel-list` stops at the default 500-channel cap but returns only `channels`. Larger workspaces will silently omit conversations, leaving callers unable to tell whether discovery is complete or needs another page/cursor.
  - confidence: 0.82
