## Summary

Keep this PR open for maintainer judgment rather than closing as implemented. Current main still does not implement the PR's Slack `channel-list` or Slack `search` actions: Slack discovery only advertises existing send/read/pin/member/emoji actions, the Slack dispatch path does not route `channel-list` or `search`, and the Slack runtime has no `listSlackChannels` or `searchSlackMessages` export. The search half is partly superseded by open PR #65691, but that does not cover the channel-list work tracked by open issue #11492, so current main has not made this PR obsolete end to end.

## Best Possible Solution

Keep this PR open for now, but do not land it as-is. The clean path is to let #65691 carry the Slack search work if maintainers accept that implementation, then rebase or split this PR down to the remaining Slack channel-list/channel-info work from #11492 using the current `extensions/slack` lazy runtime, `actions.runtime.ts` export, config schema/generated metadata, docs, and focused dispatch/runtime tests. If the contributor does not rebase, a maintainer can later close this as superseded once search lands and a current channel-list PR or canonical issue remains for the rest.

## Risks / Open Questions

- The PR is stale against current main: its head is not mergeable, and its raw branch still uses older Slack/action-runtime patterns that differ from the current lazy `actions.runtime.ts` boundary.
- The search portion overlaps with open PR #65691; landing both without reconciliation would likely duplicate or conflict around Slack search action names, CLI options, and result formatting.
- The PR's final patch returns Slack search as top-level `messages`/`total`, while current CLI search formatting expects `payload.results` for search actions, so the normal text output path likely still needs adjustment.
- Slack search is a broader user-token capability. It needs explicit gating, clear docs, and focused tests to avoid advertising a non-invokable or over-broad action.
- The channel-list implementation caps results at 500 without returning cursor/truncation metadata, which can still make channel discovery silently incomplete in larger workspaces.
