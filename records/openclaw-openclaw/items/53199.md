## Summary

Keep this PR open, but it is not mergeable as written. The Slack channel-list/search feature is still missing from current main and the PR addresses a real parity gap, but the current branch introduces a compile/runtime config defect plus search formatting and multi-account discovery issues that should be fixed before merge.

## What This Changes

The PR adds Slack `channel-list` and `search` message actions, wires them through Slack action runtime/dispatch/discovery, and refreshes config metadata for the new channel-list gate.

## Best Possible Solution

Keep the Slack channel discovery/search work in the Slack plugin, but rebase it onto current main and make the runtime contract coherent: pass resolved config through instead of loading config inside action helpers, return Slack search in the shared search result shape or update the formatter, make search discovery account-scoped/default-invokable, and expose truncation/pagination metadata for channel-list.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Pass cfg through instead of calling loadConfig:** `extensions/slack/src/actions.ts:338`
  - body: `resolveUserToken()` calls `loadConfig()` from this runtime action helper, but the symbol is not imported in this file, so the branch will fail extension typecheck. Even with an import, this reintroduces an ambient config read in a runtime action path that the repo guard rejects; use the already-resolved `opts.cfg`/account data instead.
  - confidence: 0.94
- **[P2] Return Slack search hits through the formatter envelope:** `extensions/slack/src/action-runtime.ts:493`
  - body: The CLI search formatter renders search tables only from `payload.results.messages`, but this returns Slack search data as top-level `messages`/`total`. Successful Slack searches therefore fall through to the generic summary view in normal text output, breaking the expected search UX parity with Discord.
  - confidence: 0.9
- **[P2] Only advertise search for the invokable account:** `extensions/slack/src/message-actions.ts:58-66`
  - body: When discovery is not scoped to an `accountId`, this advertises `search` if any Slack account has a user token and search enabled. Invocation without `accountId` resolves the default account, so a workspace where only a non-default account is search-capable will list an action that immediately fails unless the caller guesses the account.
  - confidence: 0.84
- **[P2] Report channel-list truncation:** `extensions/slack/src/actions.ts:284`
  - body: `channel-list` now stops after the default 500-channel cap, but the runtime returns only `channels` and drops any cursor or `hasMore` signal. Larger workspaces will silently omit conversations, which makes channel discovery look complete when it is not.
  - confidence: 0.82
