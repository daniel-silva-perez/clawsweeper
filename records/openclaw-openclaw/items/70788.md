## Summary

Keep this issue open. Current main still generates the Windows Startup-folder login item as `OpenClaw Gateway.cmd` with `start "" /min cmd.exe /d /c <gateway.cmd>`, and the startup-fallback test explicitly asserts that command. There is no `.vbs`/`wscript` sidecar launcher or uninstall cleanup for one on this path, so the reported login-time terminal flash remains plausible and narrowly actionable.

## Best Possible Solution

Keep this issue open and fix the core Windows daemon startup fallback: generate a truly hidden Startup-folder launcher while keeping the existing `.cmd` path/name for detection, add sidecar cleanup on uninstall, update startup-fallback tests, and add a user-facing changelog fix entry.
