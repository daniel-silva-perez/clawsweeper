## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/70788. The observable bug is that the Windows Scheduled Task fallback login item in `src/daemon/schtasks.ts` writes `OpenClaw Gateway.cmd` with `start "" /min cmd.exe /d /c`, which can flash a terminal window at login. Keep the existing `OpenClaw Gateway.cmd` path/name so service detection continues to work, but delegate login startup through a no-window launcher or equivalent hidden mechanism with robust Windows path quoting and no line-break injection. If using a sidecar such as `OpenClaw Gateway.vbs`, write it beside the `.cmd` during fallback install and remove it during uninstall. Update `src/daemon/schtasks.startup-fallback.test.ts` to assert the hidden launcher shape, sidecar write if applicable, and sidecar cleanup. Do not change the normal Scheduled Task path, task-script command semantics, plugin surfaces, or broad Windows service policy. Add a user-facing `CHANGELOG.md` fix entry crediting `Thanks @gthgomez` unless maintainers decide this is internal-only. Validate with the targeted startup-fallback test, targeted oxfmt check, and `pnpm check:changed` via Blacksmith Testbox before handoff.
