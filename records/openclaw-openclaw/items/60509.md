## Summary

Keep open. Current main still does not expose the incoming user message, prompt, or query on the internal `agent:bootstrap` context; the adjacent `before_prompt_build` plugin hook gets the prompt later, but that does not let internal hook authors rewrite `bootstrapFiles` before workspace bootstrap context is injected.

## What This Changes

The issue requests adding the incoming user message to the `agent:bootstrap` hook context so hooks can perform query-conditioned compression of workspace bootstrap files.

## Best Possible Solution

Add a backward-compatible optional prompt field to the internal `agent:bootstrap` context, thread it only where the current user prompt is available, keep it undefined for non-user-triggered runs, and document that `before_prompt_build` remains the later prompt-injection surface while `agent:bootstrap` is for mutating bootstrap files before injection.

## Reproduction Assessment

Yes. The API gap is reproducible by source inspection on current main: the bootstrap hook context and call chain contain no query field, while the user prompt is only passed to later prompt-build plugin hooks.

## Solution Assessment

Yes. The narrow maintainable solution is to extend the existing bootstrap context contract rather than moving query-conditioned compression into core or overloading `before_prompt_build` for pre-injection file mutation.

## Repair Work Prompt

Implement the hook-context API gap from https://github.com/openclaw/openclaw/issues/60509. Add a backward-compatible optional current-user-prompt field to the internal `agent:bootstrap` context, preferably `userMessage?: string` unless nearby naming strongly favors `prompt?: string`. Thread the value from both embedded and CLI run preparation where the current user prompt is available, through `resolveBootstrapContextForRun`, `resolveBootstrapFilesForRun`, and `applyBootstrapHookOverrides`, so hooks can mutate `bootstrapFiles` before context files are built. Keep the field undefined for heartbeat/cron/startup/raw model probe/no-user-prompt paths and do not change `before_prompt_build` or Active Memory behavior. Update docs for `agent:bootstrap` and add focused tests around `applyBootstrapHookOverrides`, `resolveBootstrapFilesForRun`/`resolveBootstrapContextForRun`, and the embedded/CLI call paths that prove the prompt reaches the bootstrap hook before injection. Because this is user-facing hook API behavior, add a single-line changelog entry with `Thanks @MoritzGan`. Do not add core SMELT logic or a new retrieval subsystem.
