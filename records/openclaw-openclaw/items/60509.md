## Summary

Keep open. Current main still does not expose the incoming user message on the `agent:bootstrap` internal hook context. The newer plugin `before_prompt_build` path does expose `prompt` and `messages`, and Active Memory uses that for query-conditioned recall, but that is only a partial adjacent surface: it cannot mutate the existing `bootstrapFiles` array before workspace bootstrap context is injected, which is the API gap requested here.

## Best Possible Solution

Add a backward-compatible optional query field to the internal `agent:bootstrap` context, likely `userMessage?: string` or a more explicitly named `prompt?: string`, and thread it from both embedded and CLI prompt assembly paths where the current turn prompt is available. Document undefined behavior for heartbeat/startup/non-user-triggered runs and add focused tests around `applyBootstrapHookOverrides` plus the run path that builds bootstrap context.
