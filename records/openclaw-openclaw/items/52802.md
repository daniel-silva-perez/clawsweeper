## Summary

Keep this issue open. Current main still routes OpenAI Responses and Chat Completions streaming through the OpenAI Node SDK, current dependencies resolve to openai 6.34.0, and upstream openai-node v6.34.0 still flushes event-only SSE frames and calls JSON.parse on empty sse.data.

## Best Possible Solution

Keep the issue open and fix the OpenAI transport layer. Either upgrade to an OpenAI SDK release that demonstrably skips empty SSE data, or add an OpenClaw-owned SSE guard/Response transform before the SDK stream is parsed. Add regression coverage in `src/agents/openai-transport-stream.test.ts` for event-only flush, event-only followed by valid data, and CRLF delimiters on the Responses and Chat Completions SSE paths where feasible.
