## Summary

Keep open. Current main still exposes OpenAI Responses and Chat Completions streaming through openai-node 6.34.0, and that SDK version still parses event-only SSE frames with empty data via JSON.parse(sse.data). I found no OpenClaw-side SSE frame guard or regression coverage for this case.

## Best Possible Solution

Keep this issue open. The best fix is to either upgrade to an OpenAI SDK release that demonstrably skips empty SSE data, or add an OpenClaw-owned guard before the SDK parses SSE on the direct Responses, Chat Completions, and WebSocket HTTP fallback paths. Add focused regression coverage for event-only SSE, event-only followed by valid data, and CRLF delimiter variants.
