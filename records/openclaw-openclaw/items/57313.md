## Summary

Keep this open. Current main still exposes only the effective provider/model in `PluginHookAgentContext`; it does not expose persisted session override values or `modelOverrideSource` to `before_model_resolve`, so the routing-plugin use case from the issue remains unsolved. The prior ClawSweeper comment matches the current code after rechecking main at `230f7122dd5cd8a37a55e671254bd9be8278f6e8`.

## Best Possible Solution

Keep the issue open and implement an additive plugin SDK contract: plumb session-scoped provider/model override data from prepared session state into `PluginHookAgentContext` for `before_model_resolve`, include `modelOverrideSource` or a user-pinned boolean so plugins can distinguish user choices from auto fallback, and add focused hook wiring tests plus docs under `docs/plugins/hooks.md`.
