## Summary

Keep this issue open. Current main exposes the effective provider/model to plugin hooks, but it does not expose the persisted session override values or their source to `before_model_resolve`, so a routing plugin still cannot reliably tell a user-pinned `/model` choice from defaults or auto fallback state.

## Best Possible Solution

Keep the issue open and implement an additive SDK contract: plumb session-scoped model/provider override data from the prepared session state into `PluginHookAgentContext` for `before_model_resolve`, include enough source information to distinguish `modelOverrideSource: "user"` from `"auto"` or legacy user state, and cover it with hook wiring tests plus plugin hook docs.
