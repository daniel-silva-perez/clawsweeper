## Summary

Keep open. Current main has manual ClawHub skill updates and tracked plugin updates, and the core package auto-updater can indirectly run plugin sync only when an OpenClaw package update is available; it does not implement the requested opt-in background update path for installed skills and plugins. The request is valid but supply-chain sensitive, so it needs maintainer product/security design rather than an automatic cleanup close.

## What This Changes

The issue requests opt-in automatic updates for installed ClawHub skills and OpenClaw plugins, either on gateway start or on a configurable interval.

## Best Possible Solution

Design an explicit opt-in updater that reuses the existing tracked skill and plugin update helpers while adding clear consent, cadence, pinning/allowlist behavior, failure reporting, rollback or audit metadata, and docs for the security model before enabling background installs of third-party code.
