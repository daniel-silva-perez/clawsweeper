## Summary

Keep open. Current main still exposes manual update paths for ClawHub-installed skills and tracked plugins, and the existing gateway auto-updater is scoped to OpenClaw package updates with plugin sync only after a core update. The requested background updater remains valid but needs maintainer product and supply-chain design before implementation.

## What This Changes

The issue requests an opt-in mechanism to automatically update installed ClawHub skills and plugins on gateway start or a configurable interval.

## Best Possible Solution

Ship a maintainer-approved core update policy that reuses existing tracked skill/plugin update helpers but adds explicit operator consent, cadence controls, pinning or allowlist semantics, failure reporting, rollback guidance, audit metadata, and user-facing security documentation before any background install of third-party code.
