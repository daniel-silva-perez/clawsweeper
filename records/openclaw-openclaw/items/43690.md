## Summary

Keep this issue open. Current main still routes Feishu table-heavy replies and outbound messages through interactive-card send paths without a table-limit/plain-text fallback, while the plain-text path is the only path that applies markdown table conversion. The provided GitHub context also shows open implementation candidates with closing syntax, especially #50429, so the issue should remain open until a fix lands or maintainers consolidate the overlapping Feishu PRs.

## Best Possible Solution

Keep this issue open until a Feishu plugin fix lands. The best path is to review and land #50429 or a superseding consolidated fix that detects Feishu card table-limit failures in both reply and outbound card paths, retries that message through sendMessageFeishu so existing markdown table conversion applies, preserves unrelated card errors, and adds focused regression tests. #70651 is relevant for outbound behavior, while #42809 is a related native-table rendering enhancement rather than a complete fix for silent delivery failure.
