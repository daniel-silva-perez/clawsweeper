## What This Changes

The issue requests Feishu fallback handling when markdown-table-heavy replies or outbound messages exceed Feishu interactive-card table limits and disappear instead of being delivered.

## Best Possible Solution

A consolidated Feishu plugin fix should classify CardKit/table-limit failures such as 230099/11310, retry affected reply-card, streaming-card fallback, and outbound card sends through the existing plain-text/table-conversion path or another bounded safe renderer, preserve thread and mention routing, rethrow unrelated card failures, and add focused regression coverage for each delivery surface.

## Solution Assessment
