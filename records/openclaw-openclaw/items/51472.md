## Summary

Keep open. Current main still does not implement this PR's intended Feishu workspace artifact SDK exports, and the PR is stale against the current SDK boundary. It remains potentially useful as product input, but should not be merged unchanged because current docs and guardrails favor generic SDK surfaces over channel-branded helper subpaths, and the PR discussion already raised security-sensitive path-containment concerns.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but replace the branch with a current-main implementation if the product need remains. The best path is a plugin-neutral workspace artifact helper or documented recipe built from `api.runtime.agent.resolveAgentWorkspaceDir` plus `openclaw/plugin-sdk/file-access-runtime`, with safe filename normalization, pinned/root-scoped writes, current SDK entrypoint classification, docs, and contract tests updated together.
