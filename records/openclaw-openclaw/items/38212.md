## Summary

Keep open. Current main has moved toward a canonical provider-plus-runtime split and doctor migration for legacy `codex-cli/*` refs, but I did not find a source or test proof that the session-memory slug helper can no longer pass raw `codex-cli/gpt-5.4` into embedded model resolution and hit the reported `Unknown model` path.

## Best Possible Solution

Keep this issue open until maintainers choose and prove one explicit product path: either normalize legacy `codex-cli/*` helper runs before embedded model resolution with session-memory slug-generator regression coverage, or enforce/migrate/document that raw `codex-cli/*` primary refs are unsupported for embedded PI helpers and must use canonical provider refs plus `agentRuntime`/CLI backend configuration.
