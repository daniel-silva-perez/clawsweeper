## Summary

Keep open. The PR targets a real current-main gap in Feishu and WhatsApp terminal QR guidance, but it is not ready to merge because it expands the public Plugin SDK `runtime-env` surface without updating the tracked API baseline.

## What This Changes

The PR adds WSL-gated MS Gothic guidance before Feishu and WhatsApp terminal QR output and re-exports `isWSLEnv` from `openclaw/plugin-sdk/runtime-env`.

## Best Possible Solution

Keep the PR active, but make it a small channel-owned change with an explicit SDK decision: either keep WSL detection private to the channel/runtime code, or intentionally expose `isWSLEnv` with the generated API baseline and any needed SDK docs/test alignment.

## Reproduction Assessment

Yes for the source-level issue: current main's Feishu and WhatsApp terminal QR paths have no WSL/font guidance, and the searched channel docs do not document the workaround. A live WSL terminal-font reproduction was not performed.

## Solution Assessment

No as submitted. The WSL-gated hint is a reasonable narrow direction, but the public SDK export needs baseline alignment or should be avoided with a narrower implementation.
