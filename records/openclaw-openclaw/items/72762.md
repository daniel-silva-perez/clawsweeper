## Summary

Keep this PR open. Current main still lacks WSL/MS Gothic guidance in the Feishu and WhatsApp terminal QR-code login paths, and the channel docs do not document this failure mode. The latest provided diff targets the right channel-owned surfaces and appears to address the earlier unconditional-message review by gating on `isWSLEnv()`, but it remains a runtime/API change because it adds a public `openclaw/plugin-sdk/runtime-env` export and has no targeted WSL/non-WSL QR-output tests.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is a small channel-owned update that adds WSL-only terminal QR guidance for Feishu and WhatsApp, deliberately decides whether `isWSLEnv` belongs in the public plugin SDK or should remain behind a narrower/local helper, and adds targeted tests for WSL and non-WSL output before the usual changed gate.
