## Summary

Keep open. Current main does not provide strong close evidence for the reported `openclaw config` hang: bare `config` is intentionally routed into the interactive configure wizard, the exact reported no-response point is still ambiguous, and no current code/docs/history evidence shows this issue was fixed or superseded. The related ClawSweeper matches are config RPC response/redaction items, not this CLI startup/hang report.

## Best Possible Solution

Keep this issue open and either request a sharper reproduction or fix the likely CLI boundary directly. Maintainers should verify `node scripts/run-node.mjs config` on macOS with the reporter's install style, including invalid-config and non-TTY cases. If the intended behavior is that bare `config` is an alias for `configure`, align its startup policy with `configure` or add an explicit prompt/help fallback, then add a regression test for the bare `config` preaction path.
