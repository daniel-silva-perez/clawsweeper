## Summary

Keep this issue open. Current main still does not provide strong close evidence for the reported `openclaw config` no-response regression: bare `config` intentionally enters the interactive configure wizard, but the startup/preaction boundary still differs from `configure`, and there is no inspected regression test or linked merged fix for the exact hang/no-response symptom.

## Best Possible Solution

Keep this open and reproduce `node scripts/run-node.mjs config` and `openclaw config` on macOS with the reporter's install style, including invalid-config and non-TTY cases. If bare `config` is intended to be a full alias for `configure`, align its startup policy with `configure` or add an explicit prompt/help fallback, then add a regression test for the bare `config` preaction path.
