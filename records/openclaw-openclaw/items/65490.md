## Summary

Keep this issue open. Current main has related partial improvements for fallback visibility, session/status display, cron syntax, cron delivery previews, and cron local-provider preflight, but it still does not implement the central production-operations asks: startup primary-model reachability that blocks or marks the gateway degraded before accepting turns, a compact agent-readable model/fallback/last-error diagnostic surface, or cron add dry-run/preflight that fails closed on unresolved announce delivery before persistence.

## Best Possible Solution

Keep this open and narrow the maintainer path: add a configurable startup/provider preflight that live-probes the configured primary model and either blocks startup or marks readiness degraded; add a safe agent-visible diagnostics RPC/tool exposing selected model, active responding model, fallback state, and last provider/fallback error category/code; and add cron add dry-run/preflight plus fail-closed announce target validation before job persistence. Treat #56852, #58303, #63900, #56996, #57045, and #57046 as related completed subwork rather than a substitute for the remaining work.
