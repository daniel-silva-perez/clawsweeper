## Summary

Keep open. Current main has useful partial improvements around fallback visibility, status output, and cron delivery previews, but the central production-operations requests remain unsatisfied: startup primary-model reachability is still warning-only/background, diagnostics are not a structured agent-readable model/fallback/last-provider-error surface, and cron add still persists jobs before delivery route preflight/dry-run validation.

## What This Changes

The issue requests production-grade execution discipline for OpenClaw agents: startup primary-model reachability checks, agent-visible runtime diagnostics, stricter cron add delivery validation/dry-run support, and clearer pre-context error propagation.

## Best Possible Solution

Keep this open as the production execution-discipline tracker, or split it into canonical follow-ups for a configurable startup provider reachability/degraded-mode policy, a safe structured runtime diagnostics RPC or agent tool, and cron add dry-run/preflight delivery validation that resolves announce delivery before persistence.
