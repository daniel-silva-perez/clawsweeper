## Summary

Keep open. Current main has useful related fallback/status/cron improvements, but the central production-operations gap remains: primary-model startup reachability is still non-blocking, the agent-visible diagnostics surface is not a structured model/fallback/last-provider-error contract, and cron creation still lacks a dry-run/preflight delivery-resolution gate before persistence.

## What This Changes

The issue requests production-operability improvements for startup provider health checks, runtime model/fallback diagnostics, cron creation validation/dry-run behavior, and clearer pre-context error reporting.

## Best Possible Solution

Keep this open as the production execution-discipline tracker, or split it into canonical follow-ups for a configurable startup provider preflight/degraded-mode policy, a safe structured runtime diagnostics RPC or agent tool, and cron add dry-run/preflight delivery validation that fails closed before persistence.
