## Summary

Keep open. Current main still lacks the install-drift troubleshooting entry this PR adds, and maintainer discussion says the change remains useful but should be expanded so `unknown method: browser.request` is not treated as gateway drift when the root cause is restrictive plugin configuration.

## Best Possible Solution

Keep the PR open and ask for a refresh against current main: retain the CLI/Gateway install-drift troubleshooting entry, add a sibling `browser.enabled=true` plus restrictive `plugins.entries` or `plugins.allow` mismatch signature, and scope `openclaw gateway install --force` or `openclaw doctor` to the version/install-drift branch only.
