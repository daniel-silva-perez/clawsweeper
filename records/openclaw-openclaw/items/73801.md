## Summary

Keep open. Current main has generic Active Memory timeout guards, but it still does not prove the reported Cerebras gpt-oss-120b reasoning-model path is fixed or that the gateway CPU/health failure recovers cleanly. The docs still recommend cerebras/gpt-oss-120b for Active Memory, the bundled Cerebras catalog marks that model as a reasoning OpenAI-completions model, and the current tests cover mocked timeout behavior rather than a Cerebras reasoning stream that pins gateway CPU.

## Best Possible Solution

Keep this open as a focused Active Memory/Cerebras regression. The best fix is to reproduce or simulate a Cerebras gpt-oss-120b OpenAI-completions stream that emits reasoning/tool-call side-channel content, prove Active Memory aborts the provider/session work at the configured timeout, add a regression that checks gateway health/admin responsiveness after timeout, and update the Active Memory docs if cerebras/gpt-oss-120b is not safe as the recommended fast recall model.
