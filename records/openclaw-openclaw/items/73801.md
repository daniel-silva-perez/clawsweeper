## Summary

Keep open. Current main has generic Active Memory timeout guards, but it does not prove this Cerebras/gpt-oss-120b path is fixed: the docs still recommend that model for Active Memory, the Cerebras catalog marks it as a reasoning OpenAI-completions model, and the current tests only cover mocked timeout behavior rather than the reported reasoning-model abort/CPU/health failure. The open Active Memory timeout trackers overlap, but they do not fully supersede this issue's Cerebras-docs-recommended model plus gateway CPU-pinning evidence.

## Best Possible Solution

Keep this issue open as a focused Active Memory/Cerebras regression. The best fix is to reproduce or simulate gpt-oss-120b reasoning-only/tool-call streaming in the Active Memory embedded path, ensure timeout aborts actually cancel the provider stream and session work, preserve gateway health/admin responsiveness during and after timeout, and update the Active Memory docs if cerebras/gpt-oss-120b is not currently safe as the recommended fast recall model.
