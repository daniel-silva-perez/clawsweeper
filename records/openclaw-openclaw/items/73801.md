## Summary

Keep open. Current main has useful generic Active Memory timeout mitigations, including a circuit breaker after repeated timeouts, but it does not prove the reported Cerebras gpt-oss-120b reasoning-response path no longer stalls, aborts cleanly, or preserves gateway health. The docs still recommend the affected model for Active Memory, so this remains a focused regression rather than a cleanup close.

## What This Changes

Fix Active Memory so cerebras/gpt-oss-120b either returns recall or fails non-disruptively within the intended budget without pinning gateway CPU, and adjust docs if that model is not safe for recall.

## Best Possible Solution

Keep this as a focused Active Memory/Cerebras regression until a maintainer can reproduce or simulate gpt-oss-120b reasoning/tool-call streaming in the embedded recall path, prove provider aborts stop the underlying work, assert gateway health remains responsive during and after timeout, and update the Active Memory recommendation if the model remains unsafe for recall.

## Reproduction Assessment

Yes for the reporter path: the issue includes concrete config, steps, logs, direct API controls, and post-disable recovery observations. This review did not live-run Cerebras, so the high-confidence path is to rerun that exact config or add a provider-harness simulation of a reasoning-only/tool-call stream that ignores abort.

## Solution Assessment

Unclear. The current circuit breaker is a useful mitigation for repeated timeouts, but the best fix still needs to prove clean abort and gateway responsiveness for the Cerebras reasoning-model path rather than only skipping later recalls.
