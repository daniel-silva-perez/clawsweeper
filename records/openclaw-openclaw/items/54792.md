## Summary

Keep open. Current main still has only a runtime/startup backstop and documentation for `hooks.enabled=true` requiring `hooks.token`; the shared schema/config-write validation requested by this issue is still missing.

## Best Possible Solution

Keep this issue open. Add a shared cross-field config validation rule in the raw validation path used by `validateConfigObjectRaw` and the plugin-aware wrappers so `hooks.enabled=true` is rejected unless `hooks.token` normalizes to a non-empty string. That should make `config.set`, `config.apply`, `config.patch`, `writeConfigFile`, CLI write paths, and plugin/runtime config writes fail before persistence. Keep `resolveHooksConfig` as the startup/reload backstop, and add lookup/UI dependency metadata only if clients are expected to preflight this before edits.
