## Summary

Close as superseded. Current main still lacks the requested shared write-time/schema dependency validation, but the same remaining hooks.enabled/hooks.token work is already tracked by open PR #54831; the reporter's narrower duplicate #54804 was also closed in favor of that PR.

## What This Changes

Request config/schema validation to warn or block when enabling hooks.enabled without the required hooks.token, ideally as a reusable dependency rule for feature flags.

## Best Possible Solution

Use #54831 as the canonical implementation target: land a tested shared config validation rule that rejects hooks.enabled=true without a non-empty hooks.token across config.patch, config.set/apply, writeConfigFile, and plugin-aware validation, while keeping resolveHooksConfig as the runtime backstop. Treat schema dependency metadata as a follow-up product/API decision if maintainers want clients to preflight dependencies from config.schema.lookup.
