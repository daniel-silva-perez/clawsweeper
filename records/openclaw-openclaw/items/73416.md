## Summary

Keep open. The guard widening is directionally correct for the Google Vertex and Gemini CLI bypass, but the current PR head has a concrete CI blocker: the added test declares an async generator with no yield, and `check-lint` fails on that line.

The PR widens the Google thinking payload wrapper guard to include `google-generative-ai`, `google-vertex`, and `google-gemini-cli`, and adds wrapper regression tests for those API shapes plus non-Google passthrough.

## Best Possible Solution

Fix the test helper lint violation, keep the guard widening focused on the three native Google API shapes, then land after green CI. If #38327 still covers gaxios or OpenAI-compatible Vertex failures, adjust the closing relationship or split those remaining root causes into a separate tracked item.
