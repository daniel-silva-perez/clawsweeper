## Summary

Keep this PR open. Current main still starts the optional model-pricing refresh during initial Gateway runtime setup, before the ready/post-attach scheduled-services path the PR targets. The live PR is an active, focused implementation candidate for the linked Windows startup degradation report, and current main does not already implement its intended lifecycle change.

## Best Possible Solution

Review and land or adapt this PR if CI and maintainer review pass. The best product direction is to keep pricing enabled by default but start the optional pricing refresh only after Gateway post-attach work and sidecars/channels have reached the ready path, store the returned stop handle on runtime state, preserve plugin lookup metadata, and leave #73323 open for the broader Windows runtime/network investigation.
