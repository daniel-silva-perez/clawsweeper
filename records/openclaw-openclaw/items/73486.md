## Summary

Keep this PR open. Current main still starts the optional model-pricing refresh during initial Gateway runtime setup, before the ready/post-attach scheduled-services path this PR targets. The PR remains an active, focused implementation candidate for #73323, and the author follow-up says the force-pushed head addressed the deferred-sidecars and early-stop coverage review feedback.

## Best Possible Solution

Review and land or adapt this PR if maintainer review and CI pass. The best product direction is to keep pricing enabled by default, but start the optional pricing refresh only after post-attach work and sidecars/channels reach the ready path, store the returned stop handle on runtime state, preserve plugin lookup metadata, and leave #73323 open for the broader Windows runtime/network investigation.
