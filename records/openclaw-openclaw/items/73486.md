## Summary

Keep this PR open. Current main still starts the optional model-pricing refresh from initial Gateway runtime setup, before the ready/post-attach sidecar path this PR targets. The force-pushed PR head is an active, focused implementation candidate for #73323, and it appears to address the earlier Copilot deferred-sidecars concern and Greptile early-stop coverage gap. Security pass: the visible diff is limited to gateway lifecycle code and tests, with no dependency, workflow, release, secret, or supply-chain surface changes.

## Best Possible Solution

Review and land or adapt this PR if maintainer review and CI pass. The best product direction is to keep pricing enabled by default, but start the optional pricing refresh only after post-attach work and sidecars/channels reach the ready path, store the returned stop handle on runtime state, preserve plugin lookup metadata, and leave #73323 open for the broader Windows runtime/network investigation.
