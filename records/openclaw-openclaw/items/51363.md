## Summary

Keep open. Current main still derives Docker sandbox and sandbox-browser container names from the sandbox scope key only, so co-hosted instances with the same default agent id still collide on names like `openclaw-sbx-agent-main-f331f052`. The related fix PR #51380 proposed the right shape but is closed unmerged, and no open PR currently references this issue with closing syntax.

## Best Possible Solution

Keep this issue open and implement the core Docker sandbox fix in `src/agents/sandbox/docker.ts` and `src/agents/sandbox/browser.ts`: include a stable instance discriminator such as the resolved OpenClaw state/workspace root or HOME-derived identity in the slug input for agent/shared scopes, keep session-scope behavior stable where appropriate, add regression tests proving two different HOME/workspace roots with the same default `main` agent produce different Docker and browser container names, and document the one-time recreate/orphaned-name behavior.
