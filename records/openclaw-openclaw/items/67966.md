## Summary

Current main still lacks the requested local-managed Playwright navigation interception path. Local-managed browser profiles continue to report `usesPersistentPlaywright: false`, so `openTab` only reaches `createPageViaPlaywright` for remote CDP profiles and strict SSRF mode still fails closed before raw CDP tab creation can inspect redirect hops.

## What This Changes

Enable Playwright-backed navigation interception for local-managed browser mode so strict browser SSRF policy can inspect redirect hops consistently with remote CDP mode.

## Best Possible Solution

Implement a targeted browser plugin change that uses the existing Playwright-backed `createPageViaPlaywright` and `gotoPageWithNavigationGuard` path for local-managed profiles when strict SSRF policy requires redirect-hop inspection, while preserving Chrome MCP existing-session behavior and raw CDP fallbacks for safe cases such as non-strict navigation or `about:blank`. Add regression tests for local-managed strict-mode IP-literal navigation and private redirect blocking, and update browser docs only if the profile capability matrix or operator-facing behavior changes.
