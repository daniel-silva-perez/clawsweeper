## Summary

Keep this PR open. The underlying transcript-redaction gap is real on current main, but the PR head still has merge-blocking correctness and security/API-boundary issues around config passthrough, blocked guard writes, and public SDK exposure.

## What This Changes

This PR wraps raw session transcript append paths in the shared guard, adds update-mode and config passthrough, changes the Codex transcript mirror, exports the guard through the plugin SDK runtime barrel, and adds redaction/idempotency/update tests.

## Best Possible Solution

Land a corrected, maintainer-reviewed security fix that applies the shared transcript guard to every raw transcript append path, preserves existing transcript update semantics, threads the active OpenClawConfig through production callers, handles guard-blocked writes explicitly, and keeps any new SDK surface narrow or clearly documented/versioned.

## Reproduction Assessment

Yes. Current main shows raw SessionManager append paths, and the PR-head regressions can be reproduced with a before_message_write hook that blocks an assistant write or with custom logging.redactPatterns on delivery mirror paths whose callers still omit the active config.

## Solution Assessment

No. The direction is right, but the current PR is not yet the best maintainable fix because it leaves active config out of production mirror callers, reports success for blocked writes, and promotes internal guard plumbing through a public SDK barrel.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Thread config through delivery mirror callers:** `src/config/sessions/transcript.ts:168`
  - body: Adding `config` to the transcript helper is not enough while production delivery mirror callers still omit `params.cfg` / `params.ctx.cfg`. Those mirrored transcript writes will keep using default redaction, so custom `logging.redactPatterns` and `logging.redactSensitive="off"` are ignored for delivered-message transcript persistence.
  - confidence: 0.88
- **[P2] Handle blocked guard appends before returning success:** `src/config/sessions/transcript.ts:290-291`
  - body: After wrapping this helper with `guardSessionManager`, `appendMessage` can return `undefined` when a `before_message_write` hook blocks the message. Returning `{ ok: true, messageId }` here can therefore report a successful transcript write with no persisted entry and no valid id.
  - confidence: 0.87
- **[P2] Keep transcript guard out of the public runtime barrel:** `src/plugin-sdk/agent-harness-runtime.ts:129`
  - body: Exporting `guardSessionManager` from `openclaw/plugin-sdk/agent-harness-runtime` promotes mutable SessionManager redaction/update plumbing as a third-party Plugin SDK contract. Use a narrow bundled/internal seam, or document and version this explicitly before syncing the public baseline.
  - confidence: 0.8
