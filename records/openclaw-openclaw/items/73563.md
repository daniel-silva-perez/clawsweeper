## Summary

Keep this PR open. Current main still has the transcript redaction gaps the PR targets, but the latest patch needs maintainer/security review before merge because it leaves production delivery-mirror config threading incomplete, can report success with an undefined message id when the guard blocks a write, and promotes the internal guard through the public Plugin SDK harness barrel.

## What This Changes

The PR wraps raw session transcript append paths in core, gateway, CLI/ACP, and the bundled Codex app-server mirror with the shared transcript guard, adds update-mode handling, threads config through some callers, updates SDK baseline hashes, and adds transcript redaction/event tests.

## Best Possible Solution

Land a corrected, maintainer-reviewed patch that applies the existing transcript guard to every raw transcript append path, handles blocked guarded appends explicitly, threads available OpenClawConfig through all production callers that can honor redaction settings, preserves single-update/idempotency semantics, and keeps any guard SDK exposure intentionally narrow and documented.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Handle blocked guarded appends:** `src/config/sessions/transcript.ts:289-290`
  - body: After this change, `sessionManager.appendMessage` is the guarded wrapper, and the existing guard returns `undefined` when `before_message_write` blocks a message. `appendExactAssistantMessageToSessionTranscript` still returns `{ ok: true, messageId }`, so a blocked delivery mirror write can report success with an undefined `messageId` and no persisted transcript entry. Please treat the undefined result as a blocked/failed append or widen the result contract and cover it with a regression test.
  - confidence: 0.88
- **[P2] Thread config through delivery mirror callers:** `src/config/sessions/transcript.ts:287`
  - body: The new guard only honors `logging.redactSensitive: "off"` when `params.config` is supplied, but the production delivery mirror callers in outbound delivery still call `appendAssistantMessageToSessionTranscript` without passing their available config. That means delivery mirrored transcript writes will keep using default redaction even when the operator disabled it, while the new regression test only exercises a direct helper call. Please pass config through the delivery mirror call sites as part of this PR.
  - confidence: 0.84
- **[P2] Keep the guard off the public SDK barrel:** `src/plugin-sdk/agent-harness-runtime.ts:128`
  - body: Exporting `guardSessionManager` from `openclaw/plugin-sdk/agent-harness-runtime` promotes an internal SessionManager wrapper into a public Plugin SDK surface, which the SDK boundary rules say to avoid for implementation conveniences. The PR body also says this was moved to an internal subpath, but the latest diff still changes the broad public barrel and baseline hash. Please use a narrow/internal subpath or another intentional bundled-extension seam and align the generated entrypoint/API metadata accordingly.
  - confidence: 0.8
