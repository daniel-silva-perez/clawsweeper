## Summary

Keep this PR open. Current main still has the exact transcript append paths writing through raw SessionManager instances, while the shared guard already exists and docs say persisted session transcript text should be redacted. The same-author tracking issue #73565 is still open and this PR is an implementation candidate, not obsolete cleanup.

## Best Possible Solution

Keep the PR open as a security fix candidate. Review and refine the scoped guard-at-append approach, make sure custom redactPatterns apply on every new guarded path, confirm the SDK export is the narrowest supported seam for the Codex mirror, retain regression tests for the affected sinks, then land this PR or a maintainer-owned equivalent to close #73565 while leaving the broader #64046 umbrella for the remaining redaction surfaces.
