## Summary

Keep open. Current main still has multiple transcript persistence paths that append raw messages without the shared transcript guard, and this PR is an active security-sensitive implementation paired with open #73565. The PR wraps delivery-mirror, CLI/ACP transcript persistence, gateway injection, and Codex mirror transcript append paths with `guardSessionManager`, threads config/update-mode context, adjusts idempotency/event behavior, exposes the guard through the agent harness runtime SDK barrel, and adds redaction/event regression tests.

Land a corrected, maintainer-reviewed fix that applies the existing transcript guard to every raw transcript append path, threads available agent/session/config context, preserves idempotency and single-event semantics, and keeps any SDK exposure intentionally scoped and documented.
