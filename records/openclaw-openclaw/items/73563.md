## Summary

Keep this PR open. Current main still has the unguarded transcript append paths this PR targets, and the PR is an active implementation candidate for the same-author open security issue #73565 under the broader redaction umbrella #64046. The provided final head also appears to address the early review findings about missing sessionKey and blocked-message idempotency handling, so this is not obsolete cleanup.

## Best Possible Solution

Keep this PR open as the focused fix candidate. Review and land this PR or a maintainer-owned equivalent that wraps all four raw appendMessage paths with guardSessionManager using the correct agentId, sessionKey, config, transcript-update, and idempotency semantics; then use it to close #73565 while leaving any broader sensitive-data work under #64046.
