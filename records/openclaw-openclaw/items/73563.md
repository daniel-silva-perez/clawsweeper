## Summary

Keep this PR open. Current main still has the exact unguarded session transcript append paths this PR targets, while the shared guard already performs before-message hooks and transcript redaction when used. The PR is an active implementation candidate for the still-open same-author security issue #73565, not obsolete cleanup.

## Best Possible Solution

Keep this PR open as the focused security fix candidate. Review and refine the guard-at-append approach, make sure all four transcript sinks pass the right agentId/sessionKey and redaction config semantics, preserve idempotency and transcript update behavior, choose the narrowest acceptable SDK seam for the Codex mirror, then land this PR or a maintainer-owned equivalent to close #73565 while leaving #64046 for the broader redaction umbrella.

## Risks / Open Questions

- Closing this PR now would leave current main with documented persisted-transcript redaction still bypassed by four appendMessage paths.
- The PR expands the public plugin SDK barrel by exporting guardSessionManager; that is probably preferable to a deep core import for the Codex plugin, but it still needs explicit SDK boundary and API-baseline review.
- Several new guard calls in the PR do not pass OpenClawConfig, while guardSessionManager only honors custom redactPatterns/off mode when config is provided; maintainers should verify whether default-only redaction is acceptable or plumb config into these paths.
