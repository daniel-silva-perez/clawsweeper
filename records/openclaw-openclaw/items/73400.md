## Summary

Keep this PR open. Current main still has the reported silent-reply classification gap, and the PR is a useful focused fix attempt, but the latest branch still needs maintainer review because it trusts the generic thread-suffix parser as enough proof of an internal thread and still leaves group/channel-thread ordering inconsistent with the stated invariant.

## What This Changes

The PR imports `parseThreadSessionSuffix`, classifies parsed thread-suffixed silent-reply session keys as internal, adds direct/cron thread regression tests, and adds a changelog entry.

## Best Possible Solution

Land a refined narrow fix that treats trusted generated internal thread sessions as non-rewriting/internal without letting arbitrary caller-supplied direct session keys opt out of direct silent-reply rewrites, with tests for direct-thread, cron-thread, group/channel-thread, and malformed or user-shaped suffix cases.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Validate thread provenance before returning internal:** `src/shared/silent-reply-policy.ts:81-83`
  - body: The new branch returns `internal` whenever `parseThreadSessionSuffix()` yields a `threadId`, but that parser accepts any non-empty final `:thread:` tail and `sessions.create` can preserve caller-shaped keys. A direct session key ending in `:thread:x` can therefore bypass the direct silent-reply rewrite path; gate this on a trusted generated thread shape or explicit internal-session metadata before returning `internal`.
  - confidence: 0.86
- **[P3] Classify trusted threads before group policy:** `src/shared/silent-reply-policy.ts:72-73`
  - body: The PR says threaded sessions should be internal regardless of parent surface, but the group/channel branch still runs before the thread check. With configured group policies that differ from internal policies, a `...:group:...:thread:...` key can still take group behavior, so either move the trusted-thread predicate above group/channel or narrow and test the documented behavior.
  - confidence: 0.72
