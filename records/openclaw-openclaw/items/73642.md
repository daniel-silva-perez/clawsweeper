## Summary

Keep this PR open. Current main does not implement the requested `openclaw infer model run --thinking <level>` override: the model-run command has no `--thinking` option, the local lean completion path only forwards `maxTokens`, and the Gateway model-run request does not include `thinking`. The PR is a plausible focused implementation candidate, with one existing bot review concern about keeping the valid-levels list aligned with `ThinkLevel`.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is to review and land a small CLI change that normalizes `--thinking <level>`, forwards it as `reasoning` on the local lean completion path and `thinking` on the Gateway raw model-run path, updates `docs/cli/infer.md`, and keeps the added local/gateway/invalid-input tests. Address the hard-coded level-list review comment before merge.
