## Summary

Keep this PR open. Current main still does not implement `openclaw infer model run --thinking <level>`: the CLI flag is absent, the local model-run path has no reasoning option, and the Gateway request omits `thinking` even though the Gateway agent contract already accepts and forwards it. The PR remains a focused implementation candidate, and the reviewed diff does not show unrelated CI, dependency, lockfile, package, or release-script changes.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Review and land this change, or an equivalent one, after confirming `completeSimple` behavior parity, forwarding the normalized thinking level to both local and Gateway model-run paths, keeping local/gateway/invalid-input coverage, and deriving validation text from shared thinking-level definitions where practical.
