## Summary

Keep this PR open. Current main still does not implement `openclaw infer model run --thinking <level>`: the CLI option is absent, the local model-run path has no thinking/reasoning parameter, and the Gateway model-run request still omits `thinking` even though the Gateway agent contract already accepts and forwards it. The provided PR diff remains scoped to CLI/docs/tests and simple-completion plumbing, with no visible CI, lockfile, dependency-source, package-publishing, install, or release-script changes.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Review and land this change, or an equivalent patch, after confirming `complete` versus `completeSimple` parity, forwarding a normalized thinking level to both local and Gateway model-run paths, keeping local/gateway/invalid-input coverage, documenting the flag, and centralizing valid-level text where practical.
