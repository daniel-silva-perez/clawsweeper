## Summary

Keep this PR open. Current main has exact alias lookup and exact bare-model provider inference, but it does not implement this PR’s intended separator normalization, configured-model-id return shape, normalized retry path, or available-alias error hints. The PR is still a plausible fix candidate for the linked bug, but it needs a rebase onto the current split resolver and resolution of the security/behavior review comments before maintainer action.

## Best Possible Solution

Keep this PR open as a candidate. Rebase it onto the current `src/agents/model-selection-shared.ts` resolver, decide the intended fuzzy-alias policy, preserve exact model-id precedence, return matched configured model IDs for normalized inference, sanitize alias hints/log text, and add targeted tests for `gpt 5.4`, `gpt_5.4`, same-provider normalized collisions, and disallowed-model error formatting. If maintainers decide fuzzy alias matching is not desired, close manually with that product rationale rather than treating it as implemented.
