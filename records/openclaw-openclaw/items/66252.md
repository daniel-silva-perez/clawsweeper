## Summary

Keep open. Current main implements the per-agent TTS half, but the requested per-agent STT/language/provider override is still missing from the agent config contract and the main transcription paths.

## What This Changes

Add per-agent TTS and STT configuration overrides so different agents can use different speech providers, voices, languages, and transcription defaults in one OpenClaw instance.

## Best Possible Solution

Keep this issue open and implement the remaining per-agent STT half: add a public `agents.list[].stt` contract or a clearly documented equivalent, deep-merge it over `tools.media.audio` for the active agent, thread the effective config through media-understanding, file transcription, audio preflight, Discord voice transcription, and Discord preflight where agent routing is known, then update schema/types/generated config metadata/docs/tests while preserving current per-agent TTS behavior.
