## Summary

Keep open. Current main implements the per-agent TTS half, but the requested per-agent STT/language/provider override is still missing. The issue discussion already scoped this boundary, and source/docs still show `agents.list[].tts` without an equivalent `agents.list[].stt` contract or active-agent STT resolution path.

## Best Possible Solution

Keep this issue open and narrow the remaining work to per-agent STT: define an `agents.list[].stt` or equivalent media-audio override contract, decide merge precedence against global `tools.media.audio`, wire the active-agent config through media-understanding, audio preflight, and Discord voice transcription paths, and add schema/type/docs/tests while preserving the already-shipped `agents.list[].tts` behavior.
