## Summary

Keep open. Current main still lacks an explicit one-way dispatch/handoff mode for `sessions_send`: the tool schema exposes only target/message/timeout fields, fire-and-forget still starts the A2A follow-up path, and the A2A helper still performs reply-back turns plus announce delivery. Related open items are adjacent but do not supersede this per-call dispatch semantic.

## Best Possible Solution

Add a backward-compatible explicit delivery intent for `sessions_send`, preferably `mode: "conversation" | "dispatch"` with current behavior as the default. In dispatch mode, enqueue the message into the target session through the existing internal-channel/inter-session provenance path, return a minimal delivery acknowledgement, and skip `runSessionsSendA2AFlow` so there is no requester-side reply reinjection and no announce unless a separate explicit option is later designed. Update focused tests and the public session tool docs.
