## Summary

Current main still does not implement this PR's intended change. The Pi auto-compaction guard only checks `contextEngineInfo.ownsCompaction`, while safeguard mode is a separate compaction mode and the embedded-runner/CLI call sites do not pass that mode into the guard. The PR is also paired with the still-open bug report #73003, so it should remain open for maintainer review rather than be closed as obsolete.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for #73003. The best fix is to thread the resolved compaction mode into the Pi auto-compaction guard for embedded runs and CLI compaction, add targeted regression coverage for safeguard/default/off or equivalent modes, and verify the guard is still applied after settings reload before landing. After a merge, the linked bug can be closed as implemented.
