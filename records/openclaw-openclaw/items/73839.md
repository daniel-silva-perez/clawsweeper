## Summary

Current main still does not implement this PR's intended fix. The Pi auto-compaction guard only keys off `contextEngineInfo.ownsCompaction`, while safeguard compaction is a separate mode and is now the materialized default. The PR is also paired with the still-open bug #73003 via closing syntax, so it should stay open for maintainer review rather than be closed as obsolete.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for #73003. The best fix is to thread a resolved compaction mode into `applyPiAutoCompactionGuard()` for both embedded runs and CLI compaction, reapply the guard after `DefaultResourceLoader.reload()`, add targeted tests for safeguard/default behavior and provider-forced safeguard if intended, then land once the targeted type/test gate is green.
