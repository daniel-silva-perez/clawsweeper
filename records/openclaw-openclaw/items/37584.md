## Summary

Keep open. Current main still lacks per-agent `agents.list[].tools.web.search.*` settings: global `tools.web.search` is schema-backed, but the strict per-agent tools schema/type only covers tool policy/runtime controls, and web search runtime still resolves provider settings from root `tools.web.search` plus plugin-owned provider config rather than an agent-scoped override. The reporter also refreshed the issue on 2026-04-20, and the prior ClawSweeper comment reached the same unresolved conclusion.

## Best Possible Solution

Keep this issue open for maintainer design. The best path is to define and document an agent-scoped web-search settings contract, decide merge precedence with plugin-owned provider config and env fallbacks, add schema/types for `agents.list[].tools.web.search`, thread the effective per-agent search config through `createOpenClawTools()` to `createWebSearchTool()`/`runWebSearch()` and provider contexts, then add regression tests and docs. If maintainers want provider plugins to own this entirely, add a generic SDK/runtime seam for resolving per-agent web-search credentials safely before closing the issue.
