## Summary

Keep open. Current main still lacks per-agent `agents.list[].tools.web.search.*` settings: the root schema supports `tools.web.search`, but the strict per-agent tools schema/type only covers policy/runtime controls, and the web-search runtime still resolves the effective search config from root `tools.web.search` plus plugin-owned provider config rather than an agent-scoped override. The reporter also refreshed the issue on April 20, and the prior ClawSweeper review on April 26 reached the same unresolved conclusion.

## Best Possible Solution

Keep this issue open for maintainer design. The best path is to define a documented agent-scoped web-search settings contract, decide merge precedence with plugin-owned provider config and env fallbacks, add schema/types for `agents.list[].tools.web.search`, thread the effective per-agent search config through `createOpenClawTools()` to `createWebSearchTool()`/`runWebSearch()` and provider contexts, then add regression tests and docs. If maintainers want provider plugins to own this entirely, add a generic SDK/runtime seam for resolving per-agent web-search credentials safely before closing the issue.
