## Summary

Current main still does not implement per-agent web-search settings. Root `tools.web.search` and plugin-owned `plugins.entries.<plugin>.config.webSearch.*` are wired, but `agents.list[].tools.web.search.*` has no schema/type contract, generated help labels, docs, or runtime merge path. The reporter refreshed the issue on 2026-04-20, so this is active rather than stale.

## Best Possible Solution

Keep this open for maintainer design. Define and document an agent-scoped web-search settings contract, decide merge precedence with plugin-owned provider config and env fallbacks, add schema/types/help labels for the accepted per-agent surface, thread the effective per-agent search config through `createOpenClawTools()`, `createWebSearchTool()`, `runWebSearch()`, and provider contexts, then add focused regression tests and docs. If maintainers want provider plugins to own this entirely, add a generic SDK/runtime seam for resolving per-agent web-search credentials safely before closing the issue.
