## Summary

Keep this issue open. Current main still has the generic pre-tool choke point and session/plugin state primitives, but it does not implement the requested framework-owned session PIN gate: no `session.auth.pin` config/schema field, no first-class authenticated session state on `SessionEntry`, and no trusted TypeScript PIN verifier wired before tool execution. The discussion already identified this as a real security/core-hardening gap rather than optional ClawHub plugin work.

## Best Possible Solution

Keep this issue open for maintainer design. The likely path is to add a documented session-auth config using existing SecretRef/redaction conventions, persist a framework-owned authenticated flag on `SessionEntry`, clear it on `/new` and `/reset`, enforce it in the trusted pre-tool path before ordinary plugin hooks, and add a native command/gateway/internal verifier that compares PIN attempts in TypeScript outside model context. Cover it with config/schema tests, before-tool blocking tests, reset lifecycle tests, transcript/log redaction tests, and security docs updates.
