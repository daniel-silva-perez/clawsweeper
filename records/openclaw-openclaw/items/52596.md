## Summary

Keep this issue open. Current main has the generic pre-tool enforcement primitive and plugin/session extension surfaces, but it does not implement the requested framework-owned session PIN gate: there is no `session.auth.pin` config/schema field, no first-class authenticated bit on `SessionEntry`, and no trusted TypeScript verifier path that compares a PIN outside model context before unblocking tools. The issue is also a security/core-hardening request, not a routine optional integration that should be swept to ClawHub.

## Best Possible Solution

Keep the issue open for maintainer design. The likely core path is to add a documented session-auth config using the existing secret/redaction conventions, store a framework-owned authenticated flag on `SessionEntry`, clear it on `/new` and `/reset`, enforce it in the trusted pre-tool path, and add a native command or gateway/internal verifier that compares PIN attempts in TypeScript outside model context. Cover the behavior with config/schema tests, before-tool-call blocking tests, reset lifecycle tests, and security docs updates.
