## Summary

Keep this issue open. Current main still exposes only the existing bootstrap/core-file `agents.files.*` surface, while an open same-author implementation PR (#62417) specifically targets this request and is unmerged. Under the cleanup rules, that PR is an implementation candidate, not a reason to close the issue early.

## Best Possible Solution

Keep the issue open as the product tracker and review #62417 on its merits. The best path is a maintainer-reviewed implementation that adds a protocol-backed workspace file API, uses the existing `fs-safe` boundaries or stronger equivalents, wires method scopes deliberately, adds focused gateway and UI tests, and documents the new remote/container workflow. If maintainers reject that shape, close or redirect the PR first, then update the issue with the chosen product direction.
