## Summary

Keep this issue open. Current main has OpenAI Codex catalog/model-resolution work for gpt-5.4/gpt-5.5, but it does not implement or prove a fix for the ChatGPT/Codex account entitlement rejection reported here. The latest discussion still reports runtime rejection on 2026.4.24, and the related open #72966 is adjacent but not a superseding duplicate because it reports a silent hang rather than the explicit unsupported-ChatGPT-account error.

## Best Possible Solution

Keep this issue open. The best maintainer path is an OpenAI Codex provider-boundary fix: detect the backend unsupported-ChatGPT-account model error, classify it as a model-unavailable or entitlement condition for fallback/status, avoid presenting synthesized rows as definitely runnable when current account probing says otherwise, and update OpenAI provider docs/model-list diagnostics. Use #72966 as related signal, but do not treat it as the canonical replacement unless maintainers decide to merge the explicit rejection and silent-hang tracks.
