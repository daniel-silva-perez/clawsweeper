## Summary

Keep open. Current main still lacks the requested main-session stopReason:"length" recovery before tool execution: the runner has adjacent malformed-tool-call repair and empty-response recovery, but no truncation policy that preserves/acts on provider length stops, blocks parsed partial tool calls, or injects a recovery prompt. The related open PR #66146 only partially mitigates one OpenAI-compatible max-token cause and does not supersede this issue.

## Best Possible Solution

Keep this open and implement a narrow main-session Phase 1 fix: preserve provider stopReason:"length" through the stream/result path, detect any assistant turn that ends in length while tool calls are pending or present, prevent execution of suspect tool calls, and steer the model with a recovery message. Coordinate with #63188 for subagent behavior and #66146 for max-token forwarding, but do not treat either as a replacement for main-session truncation recovery.
