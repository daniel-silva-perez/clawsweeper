## Summary

Keep open. Current main still maps provider length/max_tokens signals, but the main embedded runner has no pre-dispatch output-truncation recovery path for length-terminated tool-call turns. Existing code covers malformed tool names, limited malformed argument repair, incomplete/empty assistant fallbacks, and optional tool-loop detection, but not the requested length-aware tool-call blocking plus model-facing recovery prompt.

## Best Possible Solution

Keep this issue open and implement a focused main-session Phase 1 fix: preserve and consume provider stopReason:"length" through the stream/result path, detect length-terminated turns with tool-call activity before dispatch, block suspect parsed or malformed tool execution, and inject a model-facing recovery/continuation message. Treat Phase 2 retry-budget/circuit-breaker config as separate follow-up unless it falls out naturally from the narrow guard.
