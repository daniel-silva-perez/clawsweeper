## Summary

Keep open. Current main still has provider length/max_tokens signals and adjacent safeguards, but it does not implement the requested main interactive-session recovery path that detects stopReason:"length" before tool dispatch, blocks suspect truncated tool calls, and injects a model-facing continuation/recovery prompt. The confirmed commenter context and related reports make this a valid active bug/feature, not stale cleanup.

## What This Changes

Request a main interactive-agent runtime guard that detects output truncation from stopReason:"length", prevents truncated tool-call execution, and gives the model a recovery prompt, with broader retry-budget/circuit-breaker behavior as follow-up.

## Best Possible Solution

Keep this issue open and implement a focused Phase 1 main-session fix: preserve/consume provider stopReason:"length" across the stream/result path, detect length-terminated turns with tool-call activity before dispatch, refuse suspect truncated tool execution, and inject a model-facing recovery/continuation message. Treat the broader retry-budget and circuit-breaker config as a separate follow-up unless maintainers explicitly expand scope.
