## Summary

Keep open. Current main still does not implement this PR’s intended outbound-message secret redaction, and the PR is not safe to close as implemented or merge as-is. The send and poll paths still forward raw tool params into gateway/plugin handlers, while the proposed diff redacts only separate fallback values and adds security-sensitive dynamic config/env scanning with unresolved review findings.

## Best Possible Solution

Keep this PR open for maintainer rework or explicit closure in favor of a better-designed security guardrail. A viable fix should sanitize send and poll params before both gateway/plugin dispatch and core fallback delivery, cover plugin-handled Telegram/Slack/Discord-style sends and polls with regression tests, preserve existing logging redaction semantics, and avoid broad blind config/env scanning unless it is tied to schema-backed secret metadata or another reviewed secret-source design. Coordinate with #72733 if maintainers want one broader guardrail direction, but neither current main nor that open PR implements this today.
