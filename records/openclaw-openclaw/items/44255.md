## Summary

Keep open. Current main still does not implement outbound-message secret redaction for send or poll delivery, and the PR’s proposed diff has unresolved security/functionality review findings. The main send/poll path still forwards raw action params to channel plugin handlers and raw fallback content to core delivery; Slack, Telegram, and Discord handlers read message and poll text directly from those params.

## Best Possible Solution

Keep this PR open for maintainer/security rework or explicit closure in favor of a better-designed guardrail. A viable implementation should sanitize send and poll params before both plugin dispatch and core fallback delivery, cover Telegram/Slack/Discord-style plugin sends and polls with regression tests, preserve existing logging redaction semantics, and avoid blind config/env scanning unless it is backed by reviewed secret metadata. Coordinate with #72733, but do not treat it as superseding until a guardrail actually lands.
