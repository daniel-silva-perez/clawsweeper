## Summary

Keep open. Current main still records raw `process.argv.slice(0, 8)` into config-audit records on the write helper and observe/recovery paths, and the audit append helper writes that JSON directly without applying the existing redaction utility. The two related fix PRs in the provided context were closed unmerged, so they are useful implementation references but not evidence that current main is fixed.

## Best Possible Solution

Keep this issue open. The best fix is to centralize config-audit process-info sanitization, or omit argv entirely, before any write/observe/recovery audit record is appended. Reuse the existing logging redaction infrastructure where possible, add missing OpenClaw-specific CLI flag patterns, cover both `--flag value` and `--flag=value`, and add focused tests in `src/config/io.audit.test.ts` plus observe/recovery coverage if those paths keep their own record builders.
