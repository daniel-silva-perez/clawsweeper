## Summary

Keep open. Current main still routes Discord voice autoJoin through a READY-only listener, does not export or register a RESUMED listener, and still calls `connection.destroy()` in voice cleanup paths without a destroyed-state guard. The issue is also paired with open PR #40676 using closing syntax, so it should remain open until that PR is reviewed, ported/replaced, merged, or closed.

## What This Changes

Ensure Discord voice autoJoin reliably rejoins configured voice channels after health-monitor/account restarts by handling RESUMED events and stale or already-destroyed `@discordjs/voice` connections.

## Best Possible Solution

Keep this as the canonical Discord voice reliability report. The fix should live in `extensions/discord`: trigger autoJoin on READY and RESUMED, export/register the new runtime listener, clean up or ignore stale destroyed `@discordjs/voice` connections before joining or destroying, preserve same-guild and multi-account behavior, and add focused regression tests for RESUMED autoJoin plus stale destroyed-connection cleanup.
