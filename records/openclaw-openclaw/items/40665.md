## Summary

Keep open. Current main still auto-joins Discord voice only from the READY listener, registers no RESUMED voice listener, and still calls `VoiceConnection.destroy()` directly without a destroyed-state guard or stale-connection lookup. PR #40676 by the same reporter remains open, unmerged, and explicitly targets this issue, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open as the canonical Discord voice reliability bug. Review and land PR #40676, or replace it with an equivalent Discord plugin fix that triggers autoJoin on READY and RESUMED, cleans stale `@discordjs/voice` connections idempotently around already-destroyed state, scopes cleanup safely for same-guild multi-account setups, and adds regression coverage for RESUMED autoJoin plus destroyed stale-connection cleanup.
