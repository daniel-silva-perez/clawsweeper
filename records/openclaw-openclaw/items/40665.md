## Summary

Keep open. Current main still wires Discord voice auto-join through READY only, still has no DiscordVoiceResumedListener, and still calls VoiceConnection.destroy() directly without stale-connection or already-destroyed guards. The same reporter's PR #40676 remains an open implementation candidate with closing syntax, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open as the canonical Discord voice reliability bug. Review and land PR #40676, or replace it with an equivalent Discord plugin fix that registers autoJoin on both READY and RESUMED, makes stale VoiceConnection cleanup idempotent when the dependency reports a destroyed connection, scopes cleanup safely for same-guild multi-account setups, and adds regression coverage for RESUMED autoJoin plus already-destroyed stale cleanup.
