## Summary

Keep open. Current main still has the Discord voice autoJoin path wired only to READY, has no RESUMED voice listener, and still calls `VoiceConnection.destroy()` without checking for an already-destroyed stale connection. The same reporter's open PR #40676 remains an unmerged implementation candidate with closing syntax, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Discord voice should reliably auto-join after health-monitor account restarts by handling RESUMED events and making stale VoiceConnection cleanup idempotent.

## Best Possible Solution

Keep this issue open as the canonical Discord voice reliability report. The desired end state is a Discord plugin fix that runs autoJoin on both READY and RESUMED, safely ignores or removes already-destroyed stale voice connections before joining, preserves same-guild multi-account behavior, and adds regression coverage for RESUMED autoJoin plus stale destroyed-connection cleanup.
