## Summary

Keep open. Current main still lacks a typed, schema-validated Gemini safety-settings config surface, and the direct Google Generative AI payload builder does not emit `safetySettings`; open PR #49529 remains the active implementation candidate.

## What This Changes

Allow users to configure Gemini safety thresholds for Google/Gemini providers or models and map them into Gemini API requests.

## Best Possible Solution

Keep this issue open while maintainers review and land PR #49529 or an equivalent focused implementation that adds a strict `models.providers.*.safetySettings` contract, generated schema/help coverage, Google-only request injection, docs, and focused config/runtime tests.

## Reproduction Assessment

Not applicable as a normal bug reproduction; this is a feature request. The current-main verification path is high-confidence source inspection: a provider-level `safetySettings` key is rejected by strict config schema, and the Google request builder has no payload field for it.

## Solution Assessment

Yes. The narrow maintainable solution is a Google-specific, schema-validated config field wired into the direct Gemini request path while preserving explicit per-request payload values; reviewing PR #49529 is preferable to starting duplicate work.
