## Summary

Keep open. Current main still lacks a typed, schema-validated, documented Gemini safety-settings surface, and the direct Google Generative AI transport still builds requests without `safetySettings`; the linked open PR is an implementation candidate, not a merged fix.

## What This Changes

Request to let OpenClaw configure Gemini safety thresholds per provider or model and pass them into Google Gemini API requests.

## Best Possible Solution

Review and land PR #49529 or an equivalent focused implementation that adds a strict `models.providers.*.safetySettings` contract, generated schema/help/docs coverage, Google-only request injection, payload precedence tests, and targeted config/runtime regression coverage.
