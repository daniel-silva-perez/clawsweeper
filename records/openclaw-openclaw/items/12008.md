## Summary

Keep open. Current main still does not expose a typed, schema-validated, documented Gemini safety-settings config surface, and the direct Google Generative AI payload builder still sends requests without `safetySettings`; the linked open PR remains the active implementation candidate rather than a merged fix.

## What This Changes

Request to add provider/model-level Gemini safety-settings configuration and map those thresholds into Gemini API requests to reduce blocked or empty Gemini responses.

## Best Possible Solution

Review and land PR #49529 or an equivalent focused implementation that adds a strict `models.providers.*.safetySettings` contract, updates generated schema/help/docs, injects validated settings only for direct Gemini requests, preserves explicit payload-level precedence, and covers config plus runtime behavior with targeted tests.
