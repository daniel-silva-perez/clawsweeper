## Summary

Keep open. Current main still lacks a typed or schema-validated Gemini safety-settings config surface, and the direct Google Generative AI request builder still returns payloads without `safetySettings`. Open PR #49529 remains the active implementation candidate rather than a reason to close this issue before merge.

## What This Changes

Add configurable Gemini safety-setting thresholds for Gemini providers or models and map them into Google Generative AI request payloads.

## Best Possible Solution

Keep this issue open while maintainers review and land PR #49529 or an equivalent implementation that adds a strict, documented `models.providers.*.safetySettings` contract, generated schema/help coverage, Google-only request injection, explicit payload precedence, and focused config/runtime regression tests.
