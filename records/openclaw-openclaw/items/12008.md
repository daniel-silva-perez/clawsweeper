## Summary

Keep open. Current main still lacks a typed/schema-validated Gemini safety-settings config surface, and the direct Google Generative AI request path still builds payloads without `safetySettings`. Open PR #49529 remains the active implementation candidate for this capability, so this issue should stay open until that PR or an equivalent maintainer-reviewed implementation lands.

## Best Possible Solution

Keep this issue open until PR #49529 or an equivalent maintainer-reviewed implementation lands. The implementation should add a typed and schema-validated `models.providers.*.safetySettings` config surface, generated schema/help/docs updates, runtime mapping only for `google-generative-ai` requests, explicit per-request precedence, and focused regression tests for config validation and payload injection.
