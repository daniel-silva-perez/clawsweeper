## Summary

Keep open. Current main still lacks a typed, schema-validated, documented Gemini safety-settings surface, and the direct Google Generative AI transport still builds requests without `safetySettings`; open PR #49529 remains the active implementation candidate rather than a merged fix.

## What This Changes

Allow Gemini provider/model configuration to set safety thresholds and pass those values through to Google Gemini API requests.

## Best Possible Solution

Review and land PR #49529 or an equivalent focused implementation that adds a strict `models.providers.*.safetySettings` contract, generated schema/help/docs coverage, Google-only request injection, payload precedence tests, and targeted config/runtime regression coverage.
