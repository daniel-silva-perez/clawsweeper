## Summary

Keep open. Current main still lacks a typed, schema-validated, documented Gemini safety-settings config surface, and the direct Google Generative AI request builder still emits payloads without `safetySettings`; open PR #49529 remains the active implementation path rather than a merged fix.

## What This Changes

The issue asks OpenClaw to allow provider/model-level Gemini safety threshold configuration and map it into Gemini API requests.

## Best Possible Solution

Review and land PR #49529 or an equivalent focused implementation that adds a strict `models.providers.*.safetySettings` contract, generated schema/help/docs coverage, Google-only request injection, explicit payload precedence, and targeted config/runtime regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The current-main verification is high-confidence: schema/type searches and the Google request builder show the requested config and payload mapping are absent.

## Solution Assessment

Yes, a typed provider-level config mapped only into the direct `google-generative-ai` request path is the maintainable direction. The safer next step is maintainer review of PR #49529 or a focused replacement, not closing this issue or spawning duplicate automation.
