## Summary

Current main still lacks a typed, schema-validated, documented Gemini safety-settings configuration surface, and the direct Google Generative AI request builder still emits payloads without `safetySettings`; the existing open PR remains the active implementation path rather than a merged fix.

## What This Changes

The issue requests configurable Gemini safety settings at provider or model level and mapping those thresholds into Gemini API requests.

## Best Possible Solution

Review and land PR #49529 or an equivalent focused implementation that adds a strict `models.providers.*.safetySettings` contract, generated schema/help/docs coverage, Google-only request injection, explicit payload precedence, and targeted config/runtime regression tests.
