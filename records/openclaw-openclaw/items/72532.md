## Summary

Keep this PR open for maintainer handling, but the patch is not safe to merge as written. It addresses a real Feishu multi-bot mention bug, yet making `botStripId` a no-op globally would regress direct-chat slash commands; the same underlying bug is now also tracked by the narrower open replacement #72661.

## What This Changes

The PR changes Feishu mention normalization to preserve the receiving bot's own `<at>` tag in all chats and updates Feishu tests/comments to expect preserved bot self-mentions.

## Best Possible Solution

The desired end state is a narrow Feishu plugin fix that preserves receiving-bot mention context for multi-bot group prompts while keeping leading addressing mentions normalized away for slash-command dispatch, with regression coverage for both group multi-bot content and direct `@Bot /model` commands. The open #72661 approach appears closer to that boundary than this global no-op.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: current main strips the bot self-mention, while this PR would make direct `@_bot_1 /model` normalize to `<at user_id="ou_bot">Bot</at> /model`; direct command parsing does not strip that tag before requiring a leading `/`.

## Solution Assessment

No. Preserving all bot self-mentions globally is broader than the maintainable fix because it solves the group NO_REPLY context problem by removing the direct-chat slash-command invariant.
