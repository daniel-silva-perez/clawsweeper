## Summary

Keep open. Current main has meaningful prompt-injection hardening for external content, including marker wrapping, special-token stripping, suspicious-pattern logging, tool policy, and optional sandbox/read-only guidance. It still does not implement the requested default cron reader/manager split, strict JSON sanitizer handoff, or plugin seam that can sanitize external content and then continue into a privileged manager turn. The ClawMoat plugin noted in discussion and closed unmerged PR #30046 are relevant, but they do not make the core security/product item obsolete.

## Best Possible Solution

Keep this open as the canonical security/product design item. The best path is either a first-class read-only/tool-disabled reader-worker flow for cron and external-content ingestion, with strict structured sanitizer output before any privileged manager turn, or a narrow documented plugin hook that can replace or block wrapped external content and then continue safely. ClawHub plugins such as ClawMoat can provide scanner/sanitizer implementations once the core seam and default cron behavior are decided.
