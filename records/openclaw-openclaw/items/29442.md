## Summary

Keep open. Current main has meaningful prompt-injection hardening, but it still uses wrapping, warnings, special-token sanitization, logging, tool policy, and opt-in sandbox/read-only patterns rather than the requested cron default privilege-separated reader/manager flow with strict sanitizer handoff. The linked ClawMoat plugin is useful defense-in-depth, and closed unmerged PR #30046 explored a relevant hook/plugin direction, but neither makes the core security/product item obsolete.

## Best Possible Solution

Keep this open as the canonical security/product design item. The best path is either a first-class read-only/tool-disabled reader-worker flow for cron and external-content ingestion, with strict structured sanitizer output before any privileged manager turn, or a narrow documented plugin hook that can replace or block wrapped external content and then continue safely. ClawHub plugins such as ClawMoat can provide scanner/sanitizer implementations once the core seam and default cron behavior are decided.
