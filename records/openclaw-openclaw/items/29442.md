## Summary

Keep open. Current main has meaningful prompt-injection hardening for external cron/hook/web content, but it still wraps untrusted content into the normal cron agent prompt path. It does not implement the requested default read-only worker LLM, schema-validated worker-to-manager handoff, or manager continuation boundary for privileged cron execution.

## Best Possible Solution

Keep this open for maintainer security/product design. The next step is to decide whether OpenClaw should add a first-class read-only/tool-disabled reader-worker flow for cron and other external-content ingestion, or a narrower structured sanitizer/taint hook that can replace or block wrapped content before any privileged manager turn continues. The design should account for ClawMoat, #28913, closed unmerged scanner/sanitizer proposals such as #29494, and tests proving untrusted external content cannot silently reach privileged cron execution or memory capture as ordinary prompt text.
