## Summary

Current main already solves the staged bundled-runtime wrapper problem through the newer runtime mirror layout: dist-runtime plugin wrappers stay under the staged dist-runtime root while the matching canonical dist implementation is mirrored into the same install root, so the baked ../../../dist/extensions/... specifier resolves to the staged implementation instead of the wrapper. The PR is also dirty for a narrow plugin fix because it carries unrelated release, changelog, workflow, and test commits, including security-sensitive CI workflow changes.

## Best Possible Solution

Close this PR as implemented on current main and keep the shipped mirror-based solution. If maintainers want extra belt-and-suspenders coverage, add a small follow-up test on main that imports the staged dist-runtime wrapper and asserts the bundled-channel-entry default contract survives, without merging this dirty branch.
