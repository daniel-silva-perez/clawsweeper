## Summary

Close as implemented on current main. The reported v2026.4.26 failure path staged `dist-runtime` wrappers under `installRoot/dist`, where the baked `../../../dist/...` specifier could resolve back to the wrapper. Current main stages `dist-runtime` roots under `installRoot/dist-runtime` and mirrors canonical `dist` alongside them, so the same wrapper specifier resolves to the staged implementation instead. The PR's wrapper-regeneration pass is redundant against that layout.

## Best Possible Solution

Close this PR as obsolete on current main. Keep the current `dist-runtime` staging plus canonical `dist` mirror behavior; if maintainers want extra belt-and-suspenders coverage, add a narrower assertion that resolves the staged wrapper specifier, but the runtime regeneration pass is not needed for the current layout.
