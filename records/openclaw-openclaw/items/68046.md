## Summary

Keep open. Current main still drops non-text MCP tool-result fields on the HTTP tools/call path, and the linked bug remains valid. This PR targets the right gateway surface, but it should be revised before merge because the pass-through guard accepts unknown or malformed typed blocks and the new coverage is helper-level rather than an HTTP tools/call regression.

## What This Changes

This PR changes the MCP HTTP content normalizer to return broader content blocks, passes through typed blocks, and adds helper-level tests for text, image, audio, resource, resource_link, and fallback cases.

## Best Possible Solution

Revise this PR, or a narrow replacement, so the gateway preserves only valid MCP text, image, audio, embedded resource, and resource_link blocks according to the SDK contract; malformed or unknown blocks should fall back to text. Add an HTTP tools/call regression that returns an image block from a loopback tool and verifies data and mimeType survive before maintainer review and merge for #68037.
