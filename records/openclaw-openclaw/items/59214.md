- **[P1] Import AppKit before using NSApp:** `apps/shared/OpenClawKit/Sources/OpenClawKit/ColorPreferencesStore.swift:155`
  - body: `ColorPreferencesStore` is added to the shared OpenClawKit target and uses `NSApp`/`NSAppearance` names under the macOS branch, but the file only imports Foundation, Observation, and SwiftUI. Shared files in this package import AppKit explicitly before using AppKit symbols, so this can fail the macOS SwiftPM build; add a guarded AppKit import in this file.
  - confidence: 0.82
- **[P2] Keep the default bubble color unchanged:** `apps/shared/OpenClawKit/Sources/OpenClawKit/ColorPreferencesStore.swift:118`
  - body: The new store defaults `useAccentColor` to true, so existing users with a non-blue system accent color will see their chat bubbles change before they choose anything. Current main uses a fixed `Color(red: 127/255, green: 184/255, blue: 212/255)`, and the PR says the default preserves existing behavior; seed the no-preference state with the old bubble color or migrate explicitly.
  - confidence: 0.86
- **[P2] Drop the unrelated package pin change:** `apps/macos/Package.resolved:27`
  - body: This UI feature should not alter the resolved `ElevenLabsKit` revision. The lockfile change broadens the diff into dependency resolution/supply-chain territory and is stale against current main, whose manifest and lockfile already pin `ElevenLabsKit` 0.1.1; remove this `Package.resolved` change unless a separate dependency update is intended.
