## Summary

Keep this PR open. Current main does not implement the PR’s intended fix for live memory plugin false negatives: `status --deep` still has no gateway memory-status fallback, the status renderer has no `gateway active` path, and `doctor.memory.status` still exposes embedding health without a separate runtime health signal. The linked bug remains open, and the PR discussion includes useful validation from another memory plugin setup, so the best cleanup path is review/rebase rather than closure.

## Best Possible Solution

Keep the PR open as the active implementation candidate for the memory status false-negative work. The maintainer path is to rebase it onto current main, preserve the live-gateway memory fallback, split `doctor.memory.status` runtime and embedding health, keep active-registry memory capability restoration covered, and run the targeted status/gateway/plugin-loader tests before merge.
