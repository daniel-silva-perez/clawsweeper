## Summary

Keep this issue open. Current main still has memory-core recall/read tooling plus read and prompt-injection budgets, but it does not implement the requested built-in bounded memory write tool with hard per-store character limits, over-limit rejection, add/list/replace/remove semantics, duplicate detection, or credential screening. The related discussion and open draft PR #70540 are useful context, but that PR is add-only and would not close the broader hard-budget CRUD request.

## Best Possible Solution

Keep this open for memory-core product design and implementation. The likely path is a default memory-plugin write surface with config-backed per-store character budgets, atomic add/list/replace/remove behavior, over-limit responses that include usage plus current entries for consolidation, duplicate detection, and credential-pattern screening. If an add-only slice lands first, track the remaining hard-budget CRUD work before closing this issue.
