## Summary

Keep this issue open. Current main still exposes default memory-core as recall/read plus adjacent consolidation and optional LanceDB store/forget paths, but it does not implement the requested built-in bounded memory CRUD tool with hard per-store character limits, over-limit rejection, substring replace/remove, durable duplicate checks, or credential screening. The related open draft PR #70540 is an add-only slice and would not fully close the requested hard-budget CRUD design even if merged.

## Best Possible Solution

Keep this open for memory-core product design and implementation. The best path is a model-facing memory write surface in the default memory plugin with config-backed per-store character budgets, atomic add/list/replace/remove behavior, duplicate detection, credential-pattern screening, and over-limit responses that include current usage plus entries for consolidation. If the add-only PR lands first, extend it or file a narrower canonical follow-up before closing this issue.
