## Summary

Keep this issue open. Current main still has memory recall/read tooling and adjacent memory flush, QMD injection limits, dreaming promotion, and optional LanceDB store/forget tools, but it does not implement the requested built-in bounded memory CRUD tool with hard per-store character budgets, over-limit rejection with usage/entries, substring replace/remove, or durable default-store duplicate and credential screening. Related #70540 is an open draft add-only slice and is not a complete shipped replacement.

## Best Possible Solution

Keep this open for memory-core design and implementation. The best product path is a structured model-facing memory write surface with config-backed per-store character budgets, atomic add/list/replace/remove behavior, duplicate detection, credential-pattern screening, and over-limit responses that include current usage plus entries for consolidation. If #70540 lands first, either extend it or open a canonical follow-up for the remaining hard-budget CRUD work before closing this issue.
