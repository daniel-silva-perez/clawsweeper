## Summary

Keep this issue open. Current main still has memory recall/read tools, bootstrap injection caps, QMD/search injection caps, memory flush, and promotion flows, but it does not implement the requested model-facing bounded memory CRUD tool with hard per-store character budgets, over-limit rejection with current entries/usage, substring replace/remove, duplicate detection, or credential-pattern screening. Related #70540 is still only an open draft first slice for memory_add, not a shipped or complete replacement for this request.

## Best Possible Solution

Keep this open for memory-core design and implementation. The best path is a structured model-facing memory write surface with config-backed per-store character budgets, atomic add/list/replace/remove semantics, duplicate detection, credential-pattern screening, and over-limit responses that include current usage plus entries for consolidation. If #70540 lands as memory_add only, either extend it to the full hard-budget CRUD behavior or split the remaining replace/remove/list/budget enforcement into a canonical follow-up before closing this issue.
