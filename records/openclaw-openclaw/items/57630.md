## Summary

Keep open. Current main partially implements the PR’s `appendContext` idea, but it does not implement the requested `prompt` replacement field in the public hook result contract, merge logic, prompt resolver, embedded runner, CLI runner, docs, or tests. The PR is therefore not obsolete or implemented-on-main.

## Best Possible Solution

Keep this PR open, but require a rebase or successor patch that adds `prompt?: string` to the current hook result type, includes it in `PLUGIN_PROMPT_MUTATION_RESULT_FIELDS`, merges it with higher-priority-wins semantics, propagates it through embedded and CLI prompt assembly, updates current docs, and adds focused tests for replacement, empty-string redaction, legacy hook compatibility, and bootstrap-warning ordering.
