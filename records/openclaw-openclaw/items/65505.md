## Summary

Current main still does not implement the central request. There is adjacent timeout work: terminal dedupe snapshots, model-call diagnostic timing, and a policy that avoids persisting generic transport timeouts to auth-profile health. But `agent.wait` still returns a non-phase-aware timeout snapshot, embedded-run state lacks queue/session-lock/preflight/provider/post-turn timing, lifecycle events remain coarse, and auth-profile cooldown accounting suppresses all timeout reasons instead of counting only provider-started timeouts. The linked implementation PR #65511 is closed unmerged, so the issue should stay open.

## Best Possible Solution

Keep this open. The best path is a focused core/gateway implementation that records structured embedded-run timing phases, carries a phase-aware timeout reason through lifecycle, dedupe, and `agent.wait`, and changes auth-profile failure accounting so only provider-started timeouts affect shared cooldown/rotation while queue/preflight/session-lock/post-turn/gateway-draining timeouts remain excluded. Add regression coverage for queued-never-started runs, provider-started timeout fallback/cooldown behavior, and terminal snapshot preference; update the auth/failover docs to match.
