## Summary

Current main does not implement this issue's central request. There is adjacent timeout/failover work: `agent.wait` dedupe prefers terminal snapshots, and auth-profile failure policy avoids persisting generic transport timeouts. But main still lacks structured queue/session-lock/preflight/provider/post-turn timing, lacks a phase-aware timeout reason in `agent.wait`, and does not distinguish provider-started timeouts from pre-provider timeouts for auth-profile cooldown. The linked implementation PR #65511 was closed unmerged, so it is not a reason to close this issue.

## Best Possible Solution

Keep this issue open. The best path is a focused implementation that adds a structured embedded-run timing/phase record, threads a phase-aware timeout reason through lifecycle/dedupe/`agent.wait`, and changes auth-profile cooldown accounting to mark only provider-started timeout failures while preserving existing wall-clock timeout defaults and terminal snapshot preference.
