## Summary

Keep this issue open. Current main at 230f7122dd5cd8a37a55e671254bd9be8278f6e8 still contains the reported hardcoded `/data/.clawdbot` references in `skills/gh-issues/SKILL.md`, while OpenClaw’s source, tests, and skills docs continue to define standard config/state paths under `~/.openclaw` or `OPENCLAW_STATE_DIR`. The related PR #57146 is still open and uses closing syntax for this issue, so it should be treated as the implementation candidate rather than a reason to close early.

## Best Possible Solution

Keep the issue open and review/land PR #57146 or an equivalent maintainer-approved patch that replaces every `/data/.clawdbot` reference in `skills/gh-issues/SKILL.md` with the active `OPENCLAW_CONFIG_PATH` / `OPENCLAW_STATE_DIR` / `~/.openclaw` pattern. Add a focused regression assertion to `src/agents/skills.env-path-guidance.test.ts` so the bundled gh-issues skill cannot reintroduce `/data/.clawdbot` in token, claims, cursor, or setup guidance.
