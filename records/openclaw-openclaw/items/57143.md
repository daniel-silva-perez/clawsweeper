## Summary

Keep this issue open. Current main still contains the reported hardcoded `/data/.clawdbot` references in the bundled `gh-issues` skill, while the current config/path contract points to `OPENCLAW_STATE_DIR`, `OPENCLAW_CONFIG_PATH`, and default `~/.openclaw` paths. The provided GitHub context also shows open PR #57146 uses closing syntax for this issue and is unmerged, so it should be treated as the implementation candidate rather than a cleanup close reason.

## Best Possible Solution

Keep the issue open and review/land PR #57146 or an equivalent maintainer-approved patch that replaces every `/data/.clawdbot` reference in `skills/gh-issues/SKILL.md` with the active `OPENCLAW_CONFIG_PATH` / `OPENCLAW_STATE_DIR` / default `~/.openclaw` pattern. Add a focused assertion in `src/agents/skills.env-path-guidance.test.ts` so the bundled gh-issues skill cannot reintroduce the legacy path in token, claims, cursor, or setup guidance.
