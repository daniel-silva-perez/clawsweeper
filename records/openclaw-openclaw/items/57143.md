## Summary

Keep this issue open. Current main at f256eeba still contains the nine reported `/data/.clawdbot` references in the bundled `gh-issues` skill, while OpenClaw’s config/state path code and skills docs point standard installs at `~/.openclaw`. The provided GitHub context also shows open PR #57146 uses closing syntax for this issue and is unmerged, so it should be treated as the implementation candidate rather than a reason to close the issue early.

## Best Possible Solution

Keep the issue open and review/land PR #57146 or an equivalent patch that replaces every `/data/.clawdbot` reference in `skills/gh-issues/SKILL.md` with the active OpenClaw config/state path pattern. Add a focused regression assertion so the bundled gh-issues skill cannot reintroduce `/data/.clawdbot` in token, claims, or cursor guidance.
