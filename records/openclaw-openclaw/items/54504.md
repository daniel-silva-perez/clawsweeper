## Summary

Keep open. Current main still lacks a native, label-resolved role-file convention for `sessions_spawn`; the current implementation supports explicit spawn params, broad subagent defaults, inline attachments, workspace/bootstrap inheritance, and system prompt override surfaces, but not workspace/user `subagents/<label>.md` files with frontmatter-driven model/tool controls.

## Best Possible Solution

Keep open for maintainer product/API design. A good implementation would add a native resolver in the `sessions_spawn`/`spawnSubagentDirect` path for workspace-then-user `subagents/<label>.md`, parse constrained frontmatter, let explicit tool-call params override frontmatter, and add docs plus focused tests for precedence, invalid frontmatter, allowed model overrides, tool-policy narrowing versus broadening, run/session modes, and unchanged behavior when no role file exists.
