## Summary

Keep this PR open. Current main does not implement the PR’s stated Paperclip exec-env plumbing, and the PR discussion includes an unresolved correctness concern that the real embedded-run path still would not forward the new env into `createOpenClawCodingTools`. The work may still be useful, but it needs a current-main rework rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or ask the contributor to rebase/rework it against current main. The clean implementation path is to define a typed, generic runtime exec-env surface only if maintainers want this capability, forward it through `runEmbeddedPiAgent` -> `runEmbeddedAttempt` -> `createOpenClawCodingTools` -> `createExecTool`, merge it into the effective per-invocation exec env without mutating `process.env`, and add a regression test on the real embedded-attempt tool wiring path. If maintainers decide the newer provider runtime-auth model intentionally supersedes Paperclip exec env injection, that should be an explicit product decision rather than an automated cleanup close.
