## Summary

Keep open. Current main still drops custom-scheme browser origins to the opaque string "null", so explicit `gateway.controlUi.allowedOrigins` entries like `tauri://localhost` cannot match; this PR remains a focused fix candidate for the linked open bug and the proposed diff is narrow.

## What This Changes

The PR changes Gateway origin normalization to preserve `scheme://host` for non-standard URL schemes, adds origin-check regression cases, and adds a changelog entry for custom Control UI origins.

## Best Possible Solution

Review and land this narrow Gateway normalization change once CI passes and the authorization semantics are confirmed: custom desktop-wrapper schemes should match only when explicitly listed, while wildcard, host-header fallback, local-loopback, literal-null, and malformed-origin behavior should stay unchanged.

## Reproduction Assessment

Yes. The reproduction is high-confidence from current main source plus the Node URL check: `tauri://localhost` parses with `url.origin === "null"`, so `checkBrowserOrigin` compares `"null"` against an explicit `tauri://localhost` allowlist entry and rejects it.

## Solution Assessment

Yes, with maintainer review. Reconstructing `scheme://host` only when `url.origin === "null"` and a host exists is the narrowest maintainable fix because it feeds the existing lowercase allowlist path instead of adding a special Tauri/Electron bypass.
