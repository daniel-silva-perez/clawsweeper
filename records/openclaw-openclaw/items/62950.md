## Summary

Keep this PR open, but do not merge it as-is. Current main still drops mixed-content `/think` and `/verbose` levels before run resolution, so the bug remains valid; however, the PR restores the normal directive flags and would route mixed one-shot levels into the existing directive persistence path.

## What This Changes

This PR changes `resolveReplyDirectives` to capture parsed `thinkLevel` and `verboseLevel` before `clearInlineDirectives` runs, then restore them for authorized mixed-content messages.

## Best Possible Solution

Preserve mixed-content `thinkLevel` and `verboseLevel` for the current run without reusing the session-persistence directive flags, or add explicit per-message state that resolution can consume while persistence ignores it. Add focused tests for authorized and unauthorized mixed `/think` and `/verbose`, plus Gateway `chat.send` and `sessions.send` `thinking` injection coverage; keep #38729 as complementary provider support.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep mixed levels out of directive persistence:** `src/auto-reply/reply/get-reply-directives.ts:300-304`
  - body: Restoring `hasThinkDirective` and `hasVerboseDirective` on a mixed-content message makes the later `hasAnyDirective` path run the fast lane and `persistInlineDirectives`. That persists `sessionEntry.thinkingLevel` and, where allowed, `verboseLevel`, so a one-shot Gateway `thinking` injection or inline `/think high hello` can become the session default even though the docs say inline levels apply only to that message. Preserve the levels for run resolution without reusing the persistence flags, or teach persistence to distinguish per-message preserved levels.
  - confidence: 0.87
