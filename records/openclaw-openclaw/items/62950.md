## Summary

Current main does not implement this PR's intended fix. Mixed-content inline `/think` and `/verbose` directives are still parsed, then cleared before run resolution, so the PR remains a plausible focused bug-fix candidate rather than conservative cleanup material.

## Best Possible Solution

Keep this PR open. The best maintainer path is to land this preservation behavior or an equivalent implementation that carries authorized mixed-content `thinkLevel` and `verboseLevel` into the current run, avoids unintended session-default persistence, and adds focused resolver plus Gateway regression tests.

## Risks / Open Questions

- The PR's current approach restores `hasThinkDirective`/`hasVerboseDirective` after clearing; current directive persistence code treats `hasThinkDirective && thinkLevel` as a session update, so maintainers should verify mixed-content inline levels apply only to the current run unless persistence is intended.
- Regression coverage is still needed for mixed-content `/think` and `/verbose`, unauthorized mixed-content directives, and the `chat.send`/`sessions.send` `thinking` injection path.
- Related PR #38729 is complementary provider support and remains open; it does not supersede this core directive-preservation bug fix.
