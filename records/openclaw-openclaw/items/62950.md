## Summary

Current main still clears mixed-content `/think` and `/verbose` directive levels before run resolution, so this PR is not implemented on main. The WebChat-specific example in the PR body is partly stale because the current UI patches session `thinkingLevel` instead of sending a `thinking` field with normal chat messages, but the Gateway `chat.send` and `sessions.send` protocol paths still expose `thinking` and still synthesize the mixed `/think <level> <message>` command body that current directive clearing drops. The related #38729 Anthropic provider PR is complementary and remains open; it does not replace this core directive-preservation work.

## Best Possible Solution

Keep this PR open as a focused bug-fix candidate. The best maintainer path is to land this preservation behavior or an equivalent implementation that carries authorized mixed-content `thinkLevel` and `verboseLevel` into the current run, avoids unintended session persistence, and adds targeted `resolveReplyDirectives` plus Gateway `chat.send`/`sessions.send` regression tests. Treat #38729 as complementary provider support, not a replacement.

## Risks / Open Questions

- The PR's current approach sets `hasThinkDirective`/`hasVerboseDirective` back to true after clearing. On current main, `hasThinkDirective && thinkLevel` persists `sessionEntry.thinkingLevel`, so the final fix should prove mixed-content inline levels apply to the current run without unintentionally becoming session defaults unless that is intended.
- Regression coverage is still missing in the PR diff for `resolveReplyDirectives` mixed-content `/think` and `/verbose`, unauthorized mixed-content directives, and the `chat.send`/`sessions.send` `thinking` injection path.
- The WebChat example in the PR body is partly stale because current UI send code calls `chat.send` without a `thinking` field and the picker patches session state, but the Gateway protocol path remains live.
