## Summary

Keep open. Current main does not implement the Discord stuck-session circuit breaker this PR proposes. The existing follow-up path can enqueue behind an active run and drain only when the run is no longer active; there is no Discord-specific timer, non-streaming active-run abort, or recovery notice on main. The related earlier PRs #73233 and #73239 are closed unmerged, so this draft is the active implementation candidate rather than obsolete.

## Best Possible Solution

Keep this PR open for normal maintainer review. The likely path is to review the proposed auto-reply queue circuit breaker, require focused tests plus Discord-safe operational proof if needed, and decide whether the threshold/recovery notice belongs as fixed behavior or configurable channel policy. The earlier closed PRs #73233 and #73239 can remain historical context; this PR is the current candidate.
