## Summary

Keep open. Current main still emits `response.completed` for streamed pending-tool-call responses whose response resource is `status: "incomplete"`, and the current schema/docs/tests still reflect the old event vocabulary; the PR is a narrow, plausible fix but needs normal maintainer review and a docs update before merge.

## What This Changes

The branch updates OpenResponses doc-comment URLs, adds `response.incomplete` to the streaming event type union, changes the pending-tool-call SSE terminal event to `response.incomplete`, and adds regression assertions in the gateway OpenResponses SSE test.

## Best Possible Solution

Update the OpenResponses handler, streaming schema, public SSE docs, and gateway regression tests so streamed pending-tool-call responses terminate with `response.incomplete` while ordinary completed streams continue to emit `response.completed`.
