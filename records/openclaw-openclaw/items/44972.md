## Summary

Keep this PR open. Current main still emits `response.completed` for a streamed OpenResponses tool-call response whose response resource is already `status: "incomplete"`; the PR targets that real gateway/API mismatch, but it still needs normal maintainer review, rebase handling, and a docs update for the new public SSE event name.

## What This Changes

The PR updates OpenResponses doc-comment URLs, adds a `response.incomplete` streaming event type, changes the streamed pending-tool-call terminal event to `response.incomplete`, and adds gateway regression assertions.

## Best Possible Solution

Update the OpenResponses handler, streaming schema, public SSE docs, and gateway regression tests so streamed pending-tool-call responses terminate with `response.incomplete`, while ordinary completed streams continue to emit `response.completed`.
