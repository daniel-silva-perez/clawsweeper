## Summary

Keep open. Current main still emits `response.completed` for streamed `/v1/responses` tool-call stops even though the response status is `incomplete`, and the PR targets the right handler/schema/test surface. It needs maintainer rebase/review and a small public-docs update, not closure or a separate repair branch.

## What This Changes

The PR updates OpenResponses doc-comment URLs, adds a `response.incomplete` streaming event schema/type, changes the streamed pending-tool-call terminal SSE event to `response.incomplete`, and adds gateway regression assertions.

## Best Possible Solution

Update the OpenResponses handler, streaming schema, public SSE docs, and gateway regression tests so streamed pending-tool-call responses terminate with `response.incomplete` while ordinary completed streams continue to emit `response.completed`, then rebase the branch onto current main.
