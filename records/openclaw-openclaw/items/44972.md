The PR changes OpenResponses streaming tool-call responses to emit `response.incomplete`, adds a matching streaming-event schema type, corrects OpenResponses documentation URLs in comments, and adds regression coverage for the incomplete SSE path.

## Best Possible Solution

Rebase or refresh the PR so the handler, schema union, current streamed function-call tests, and OpenResponses streaming docs all agree on `response.incomplete` as the terminal event for incomplete tool-call streams, then run the targeted gateway test before broader changed-gate validation.
