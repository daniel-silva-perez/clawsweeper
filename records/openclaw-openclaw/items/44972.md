## Summary

Keep open. Current main still emits `response.completed` while carrying an `incomplete` response for pending tool-call streams, and the schema, tests, and public docs still reflect the old terminal event; this PR targets the right gateway path but needs maintainer review, rebase, and a small docs update before merge.

## What This Changes

The PR updates OpenResponses source comment URLs, adds `response.incomplete` to the streaming event type surface, changes pending-tool-call SSE termination to `response.incomplete`, and adds gateway regression assertions.

## Best Possible Solution

Land a rebased version that keeps the handler, streaming schema/types, public SSE docs, and regression tests aligned so pending-tool-call streams terminate with `response.incomplete` while normal completed streams continue to emit `response.completed`.
