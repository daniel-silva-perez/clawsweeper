## Summary

Keep open. Current main still emits a completed SSE event while carrying an incomplete response for pending tool-call streams, so this PR addresses a real gateway behavior gap; the remaining maintainer action is normal review/rebase plus a small public-docs update for the new event name.

## What This Changes

The PR corrects OpenResponses doc-comment URLs, adds a `response.incomplete` streaming event type, changes the pending-tool-call SSE terminal event to `response.incomplete`, and adds gateway regression assertions.

## Best Possible Solution

Land a rebased version that keeps the handler, streaming schema, public SSE docs, and regression tests aligned so pending-tool-call streams terminate with `response.incomplete` while normal completed streams continue to emit `response.completed`.
