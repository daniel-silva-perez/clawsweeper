## Summary

Keep open. The PR addresses a real current-main mismatch: the streamed pending client-tool path builds a response with `status: "incomplete"` but still emits a `response.completed` SSE terminal event, and upstream OpenAI Responses types include `response.incomplete`; however the branch is stale against current main and needs test/docs updates before merge.

The PR updates OpenResponses source comment URLs, adds a `response.incomplete` streaming schema/type, changes the streamed pending client-tool terminal SSE event to `response.incomplete`, and adds regression coverage for that event.
