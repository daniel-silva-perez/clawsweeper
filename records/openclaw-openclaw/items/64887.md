## Summary

Keep open. Current main still records embedded-run `livenessState` in metadata and lifecycle events, but the user-facing reply payload and outbound delivery shapes do not carry a stable blocked/abandoned signal, so channels or CLI paths that render only payload text can still miss the terminal state.

## What This Changes

The issue requests surfacing embedded agent `livenessState` for blocked or abandoned terminal runs in user-visible reply payloads.

## Best Possible Solution

Add a generic surfacing layer for terminal embedded runs so final visible payload text carries a stable, user-readable blocked/abandoned signal and hint, while deferring any public structured payload field to a separate contract change if maintainers want it.

## Reproduction Assessment

Yes. The current strict-agentic blocked regression already produces a payload with only the blocked text and `isError: true` while `meta.livenessState` is `blocked`; following that result through auto-reply or CLI delivery shows no projection from meta into the visible payload.

## Solution Assessment

Yes, the issue’s Option A is the narrowest maintainable first fix because it repairs observable channel/CLI behavior without expanding the public reply payload contract. Option B should be a separate API design if maintainers want structured channel envelopes.
