## Summary

Keep open. Current main still records terminal embedded-run liveness in `meta`, but the user-facing reply payload shapes, auto-reply payload builder, CLI delivery normalization, and outbound JSON projection do not surface a stable blocked/abandoned signal in `payloads[]`; the existing ClawSweeper comment remains accurate against the newer main snapshot.

## What This Changes

Surface embedded-run `livenessState` for blocked or abandoned terminal exits in user-facing reply payloads so channel and CLI renderers do not drop that state.

## Best Possible Solution

Add a generic terminal-liveness surfacing helper that appends a stable, user-readable blocked/abandoned footer to the final visible payload for embedded runs, then prove the strict-agentic blocked path and an abandoned incomplete-turn path reach at least auto-reply and CLI delivery. A structured `ReplyPayload` field can remain a separate maintainer-approved API change.
