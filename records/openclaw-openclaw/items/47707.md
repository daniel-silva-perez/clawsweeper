## Summary

Keep this issue open. The report is still plausible on current main: OpenRouter owns a replay-policy hook, but that hook only applies passthrough Gemini thought-signature cleanup and does not enable `sanitizeToolCallIds` or `toolCallIdMode: "strict9"` for Mistral-family OpenRouter model IDs. The direct Mistral provider has strict9 replay policy, and Mistral model hints contribute transport compat, but I did not find the equivalent replay-policy path for `provider: "openrouter"` plus `modelId: "mistralai/..."`. Several linked PRs attempted this fix, but the provided GitHub context shows them closed and unmerged.

## Best Possible Solution

Implement the missing provider-owned replay behavior for OpenRouter: recognize Mistral-family routed model IDs such as `mistralai/devstral-2512:free`, return `sanitizeToolCallIds: true` with `toolCallIdMode: "strict9"`, and add focused regression coverage in the OpenRouter/provider replay policy path plus a sanitize-session-history proof that a prior tool call ID is rewritten to a 9-character alphanumeric ID.
