## Summary

Keep open. Current main does not implement `ackReactionTiming`, so the PR is still useful, but the proposed branch is not merge-ready because generated config surfaces remain stale and the deferred simple-ack path bypasses Telegram emoji validation that current main applies before calling the reaction API.

## What This Changes

The PR adds shared and Telegram-scoped `ackReactionTiming` config, threads it through Telegram message context and dispatch, defers Telegram ack/status reactions until agent run start in `run-start` mode, updates docs/help/schema files, and adds Telegram-focused tests.

## Best Possible Solution

Revise the PR on current main, keep the behavior inside the Telegram plugin boundary, preserve supported-emoji validation for both immediate and deferred simple acks, regenerate the public config schema/channel metadata/docs baselines, and coordinate the shared timing contract with the Slack and Discord companion PRs.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. The review findings are source-reproducible from the PR head and current-main comparison: generated outputs lack the new config field, and the deferred path sends the raw configured ack value while current main validates it first.

## Solution Assessment

No, not yet. The runtime direction is narrow and the Greptile `setThinking()` gap appears fixed, but the patch needs generated artifact updates and a validation-preserving deferred ack path before it is the best merge path.
