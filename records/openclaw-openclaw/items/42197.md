## Summary

Keep this PR open. Current main still does not implement the PR’s intended usage-panel behavior: skill, tool, and file labels in the context breakdown still lack full-name `title` attributes, and the consolidated usage stylesheet still has no scoped ellipsis rule for those labels. The PR is useful, but needs to be ported to the current UI stylesheet layout.

## Best Possible Solution

Keep this PR open and port the focused UI fix onto current main: add `title` attributes for skill, tool, and file labels in `ui/src/ui/views/usage-render-details.ts`, add a scoped `.context-breakdown-item .mono` ellipsis rule in `ui/src/styles/usage.css`, avoid any unscoped `.mono` override, and verify the usage panel with long names before landing.
