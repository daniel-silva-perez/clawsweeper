## Summary

Keep this PR open. Current main still does not implement the usage-panel label behavior from the PR: skill, tool, and file names in the context breakdown still render without full-name title attributes, and the consolidated usage stylesheet still lacks a scoped ellipsis rule for those labels. The PR remains useful but needs to be ported to the current file layout.

## Best Possible Solution

Keep this PR open and port the intended focused UI fix onto current main: add `title` attributes for skill, tool, and file labels in `ui/src/ui/views/usage-render-details.ts`, add a scoped `.context-breakdown-item .mono` ellipsis rule in `ui/src/styles/usage.css`, avoid any unscoped `.mono` override, and verify the panel with long names before landing.
