## Summary

Keep this PR open. Current main still lacks the PR's intended usage-panel behavior: skill, tool, and file names in the context breakdown render without full-name title tooltips, and the current consolidated usage stylesheet has no scoped ellipsis rule for those labels. The PR needs to be ported to the current UI stylesheet layout before it can land.

## Best Possible Solution

Keep this PR open and port the focused UI fix onto current main: add full-name `title` attributes for skill, tool, and file labels in `ui/src/ui/views/usage-render-details.ts`, add a scoped `.context-breakdown-item .mono` ellipsis rule in `ui/src/styles/usage.css`, avoid unscoped `.mono` overrides, and verify long names visually in the usage panel.
