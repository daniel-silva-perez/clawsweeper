## Summary

Keep this PR open. Current main does not implement the PR's central session-health lifecycle feature: there are no session-health infra modules, no remediation plan or execution flags in `openclaw sessions cleanup`, no `sessionHealthSurface` in the gateway health summary, and no Mission Control session-health card. The PR also still has security-sensitive cleanup/executor behavior and unresolved review findings that need maintainer review rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to split or tighten the session-health visibility layer from the destructive remediation executor, address the remaining P1 cleanup-safety comments, and decide separately whether the pre-commit gate behavior belongs in this PR or a focused infra PR.
