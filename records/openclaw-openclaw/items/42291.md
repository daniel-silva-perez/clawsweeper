## Summary

Keep this issue open. Current main still resolves `openclaw models auth login` to a local agent auth store, persists OAuth profiles through that local `agentDir`, and has no remote node-host guard or gateway credential mutation path. PR #42381 remains the open implementation candidate and explicitly targets this bug, so the tracker should stay open until that PR is reviewed, merged, or rejected.

## Best Possible Solution

Keep this issue open and review PR #42381 as the conservative fix. The likely short-term path is to block or clearly warn on OAuth login from remote node hosts in `src/commands/models/auth.ts` with regression coverage in `src/commands/models/auth.test.ts`; a later direct-sync design should add a scoped gateway auth-profile mutation API first.
