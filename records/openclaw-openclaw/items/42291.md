## Summary

Keep open. Current main still resolves `openclaw models auth login` to the local agent auth store and persists provider OAuth results through that local `agentDir`, with no remote node-host guard or gateway credential mutation path. Open PR #42381 is still the active implementation candidate and explicitly targets this issue, so this tracker should remain open until that PR is reviewed, merged, or rejected.

## Best Possible Solution

Keep this issue open and review PR #42381 as the current conservative fix. The likely short-term path is to block or clearly warn on OAuth login from remote node hosts in `src/commands/models/auth.ts` with regression coverage in `src/commands/models/auth.test.ts`; if maintainers want direct node-host OAuth login later, design a scoped gateway auth-profile mutation API first.
