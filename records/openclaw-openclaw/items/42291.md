## Summary

Keep open. Current main still writes `openclaw models auth login` results to the locally resolved agent auth store and does not block, warn, or sync credentials when the CLI is running from a remote node-host setup. The provided GitHub context also shows open PR #42381 as an unmerged implementation candidate that explicitly targets this issue, so this tracker should stay open until that PR is reviewed, merged, or rejected.

## Best Possible Solution

Keep this issue open and review PR #42381 as the current conservative fix. The likely short-term path is to block or clearly warn on OAuth login from remote node hosts in `src/commands/models/auth.ts` with regression coverage in `src/commands/models/auth.test.ts`; if maintainers want direct node-host OAuth login later, design a scoped gateway auth-profile mutation API first.
