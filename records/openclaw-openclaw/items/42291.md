## Summary

Keep open. Current main still routes `openclaw models auth login` through the locally resolved agent auth store and does not add a remote node-host guard or gateway-side credential sync path. The issue also has an open implementation candidate, PR #42381, with closing syntax, so cleanup policy says the issue should remain open until that PR is reviewed, merged, or superseded.

## Best Possible Solution

Keep this issue open and review or supersede PR #42381. A conservative fix likely belongs in `src/commands/models/auth.ts` with regression coverage in `src/commands/models/auth.test.ts`: either block OAuth login from remote node hosts with a clear message, or first design a secure gateway-side auth-profile mutation API before supporting token sync.
