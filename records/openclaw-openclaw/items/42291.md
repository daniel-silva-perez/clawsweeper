## Summary

Keep this issue open. Current main still runs `openclaw models auth login` against the local resolved agent directory and persists OAuth profiles there, while the gateway surface only exposes auth status reads. PR #42381 is still the active implementation candidate and uses closing syntax for this bug, so the issue should remain open until that PR is reviewed, merged, or rejected.

## Best Possible Solution

Keep this issue open and review PR #42381. The conservative near-term fix is to block or clearly warn on OAuth login from remote node hosts in `src/commands/models/auth.ts`, with regression coverage in `src/commands/models/auth.test.ts`; a later sync design should add a scoped gateway auth-profile mutation API first.
