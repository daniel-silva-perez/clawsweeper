## Summary

Keep this PR open for maintainer API review. Current main now has a merged `cron_changed` plugin hook for gateway cron lifecycle events, so adding `cron_lifecycle` as a second overlapping public hook is not merge-ready; however, current main still omits the explicit public `agentId` audit field requested here, so the PR should not be closed as fully implemented.

## What This Changes

The PR adds a typed `cron_lifecycle` plugin hook, gateway cron start/finish emissions, hook documentation, and focused gateway/plugin hook tests.

## Best Possible Solution

The desired end state is one clear public cron lifecycle contract: extend `cron_changed` with any accepted missing fields such as `agentId` and update docs/tests, or deliberately replace/alias it with a documented migration if maintainers prefer the `cron_lifecycle` name.

## Reproduction Assessment

Not applicable as a feature/API PR rather than a runtime bug. The high-confidence verification path is static inspection of current main plus the PR diff: current main emits `cron_changed`, while the PR adds a competing `cron_lifecycle` hook.

## Solution Assessment

No. The proposed hook name is not the narrowest maintainable solution against current main because `cron_changed` already owns this lifecycle surface; the safer path is to rework the existing hook contract or make an explicit maintainer-approved migration decision.
