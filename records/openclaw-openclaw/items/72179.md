## Summary

Keep this PR open. Current main now covers much of the requested cron lifecycle plugin surface through the typed `cron_changed` hook, including gateway start/finish emissions, docs, and focused hook tests. However, it is not a high-confidence close: the PR proposes a distinct public `cron_lifecycle` hook and explicitly includes agent identity, while current main exposes `cron_changed` and the public cron job snapshot omits `agentId` even though internal cron jobs carry it.

## Best Possible Solution

Keep this PR open for maintainer review, but rework it against current main's `cron_changed` implementation. The best path is either to close the PR if maintainers accept `cron_changed` as the intended public hook, or land a narrower follow-up that adds the missing accepted fields such as `agentId` to the existing hook, updates docs/tests, and avoids introducing a duplicate hook name.
