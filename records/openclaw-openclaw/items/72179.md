## Best Possible Solution

Rework the PR against current main’s `cron_changed` implementation: either confirm `cron_changed` is the accepted public hook and add any accepted missing fields such as `agentId` there with docs/tests, or deliberately choose a new `cron_lifecycle` contract and retire the overlap explicitly.

## Solution Assessment

No. Adding `cron_lifecycle` as a second public hook is not the best fit against current main’s existing `cron_changed` surface; the narrower maintainable path is to extend or clarify the existing hook unless maintainers intentionally want a new public hook name.
