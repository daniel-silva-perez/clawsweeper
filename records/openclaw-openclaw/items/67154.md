## Summary

Keep this open. Current main still models usage metadata as part of the exec approvals allowlist and still records allowlist use by saving back to `exec-approvals.json`; the related corruption class has an atomic-write mitigation, but the requested `exec-runtime.json` split is not present.

## What This Changes

The issue requests moving mutable exec usage tracking fields such as `lastUsedAt` and `lastUsedCommand` out of `exec-approvals.json` into a backward-compatible `exec-runtime.json` runtime store.

## Best Possible Solution

Add a reviewed core hardening change that introduces a backward-compatible `exec-runtime.json` usage store, stops ordinary gateway, node-host, and macOS allowlist execution from writing runtime metadata to `exec-approvals.json`, preserves existing allowlist reads, and updates CLI/UI/docs/tests to show merged legacy and runtime usage metadata without broadening exec policy.
