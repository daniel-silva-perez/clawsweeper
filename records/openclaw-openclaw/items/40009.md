## Summary

Keep open. Current main still renders the Control UI Raw config surface as a native textarea, with textarea-specific styling/tests and no CodeMirror/Monaco/Ace/jsoneditor integration. The related PR #40732 is still open and appears to improve raw-editor readability with status/breadcrumb behavior, but it is not merged and does not cover the requested folding/highlighting/code-editor replacement.

## Best Possible Solution

Keep the issue open. Implement a first-class Raw config code editor in the Control UI, likely as a focused component used by `ui/src/ui/views/config.ts`, with JSON/JSON5 syntax highlighting, folding, validation, formatting, and editor-state tests. Preserve `rawAvailable`, sensitive-value redaction/reveal, `onRawChange`, Reset to saved, `config.set`, `config.apply`, `baseHash`, and safe raw round-trip behavior; review #40732 as a possible partial foundation rather than treating it as closure.
