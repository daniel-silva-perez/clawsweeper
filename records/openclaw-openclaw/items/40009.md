## Summary

Keep open. Current main still renders the Control UI Raw config surface as a native textarea, with textarea-specific styling and tests, no CodeMirror/Monaco/Ace/jsoneditor dependency, and no raw-editor component with folding/highlighting/validation. Related PR #40732 is still open and is scoped to readability/status/breadcrumb work, while overlapping PR #56401 appears to be a more direct CodeMirror implementation candidate but is also open and not shipped.

## Best Possible Solution

Keep this issue open. Consolidate or review the active Raw config editor work, then land one tested Control UI component that provides JSON/JSON5 syntax highlighting, folding, validation/error display, formatting, and editor-state coverage while preserving `rawAvailable`, sensitive-value redaction/reveal, `onRawChange`, reset-to-saved, raw diff, `config.set`, `config.apply`, and `baseHash` behavior.
