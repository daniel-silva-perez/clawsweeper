## Summary

Keep this PR open. Current main still has the narrow final-tag regex in the streaming delivery scanner and user-facing sanitizer, so the attributed/self-closing Gemini `<final ...>` leak targeted by this PR is not implemented on main. The PR still needs maintainer review for the noted exact tag-name matching issue and possible consolidation with overlapping #66225, while the broader plain-text Gemini reasoning leak remains tracked separately in #41494.

## Best Possible Solution

Keep this PR open or replace it with an equivalent maintainer patch. The best implementation is a shared exact reasoning-tag matcher/parser used by streaming cleanup, final user-facing sanitization, and live-test helpers, supporting attributed and self-closing `<final>`/`<think>` tags without matching hyphenated custom tags, plus focused regression tests for streaming, sanitizer, and enforced-final paths.
