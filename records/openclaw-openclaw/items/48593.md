## Summary

Keep open. Current main still has the narrow delivery-specific final-tag regexes, but this PR is not ready as-is because it is dirty against main and its widened word-boundary regex can misclassify hyphenated custom tags; maintainers should consolidate it with #66225 and #65867.

## What This Changes

The PR widens final/thinking tag cleanup regexes and self-closing handling across embedded streaming, user-facing cleanup, shared reasoning-tag cleanup, live leak checks, and regression tests.

## Best Possible Solution

Land or replace this with a current-main patch that uses one exact reasoning-tag matcher or parser across streaming cleanup, user-facing sanitization, and live-test leak assertions, supporting attributed and self-closing `<final>`/`<think>` tags while rejecting hyphenated custom tag names.

## Reproduction Assessment

Yes. A focused reproduction is to stream or sanitize text containing `<final id="1">Visible</final>` or `<final/>` against current main; code inspection shows the delivery-specific regexes still only match `final\s*>`.

## Solution Assessment

No, not as-is. The PR is directionally right, but the maintainable fix should use an exact tag-name delimiter such as whitespace, `/`, or `>` and be rebased onto the current sanitizer/streaming code before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Require exact reasoning tag delimiters:** `src/agents/pi-embedded-subscribe.ts:25`

In `enforceFinalTag` streaming this can treat non-final custom markup as a real final block and deliver text that should remain suppressed; use a delimiter that only accepts whitespace, `/`, or `>` after the tag name.
