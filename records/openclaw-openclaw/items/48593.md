## Summary

Keep this PR open. Current main still does not implement the PR’s intended delivered-message cleanup for attributed Gemini `<final>` tags, and the PR remains a plausible implementation candidate rather than obsolete work. The discussion also correctly separates this narrow wrapper/tag leak from the broader open Gemini reasoning leak tracked by #41494.

## Best Possible Solution

Keep this PR open or replace it with an equivalent maintainer patch. The best fix is a shared exact reasoning-tag matcher/parser used by streaming cleanup, user-facing sanitization, and live-test helpers, supporting attributes and self-closing tags without matching hyphenated custom tags, plus regression coverage for attributed `<final>`/`<think>` tags and self-closing behavior.
