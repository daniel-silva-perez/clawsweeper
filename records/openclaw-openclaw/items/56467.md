## Summary

Keep open. Current main does not implement the requested N-strike agent/subagent policy: subagent runs store terminal outcome only, the agent defaults schema/docs have no `failureThreshold` or `onFailure` config, and the plugin hook event exposes outcome but not a failure count. The stale label is not enough for cleanup because the report is recent, concrete, and still needs maintainer policy decisions.

## What This Changes

The issue requests configurable consecutive-failure tracking and enforcement for agents/subagents, including failure counts on run records and `subagent_ended` hook context that plugins can act on.

## Best Possible Solution

Keep the issue open for maintainer product/API review. If accepted, the end state should be a typed failure-policy contract in the subagent lifecycle path, aligned config schema/types/docs, explicit outcome classification, hook context for plugins, and focused tests covering counter reset, threshold crossing, and chosen actions.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. There is a high-confidence static verification path: inspect current-main subagent registry/completion code plus config/docs and confirm the requested counters, config keys, and enforcement path are absent.

## Solution Assessment

Unclear. Adding core schema keys and enforcement may be appropriate if OpenClaw core owns this safety policy, but the exact outcomes that count as failures and the meaning of `disable`/`escalate` need maintainer decisions before this is the best fix.
