## Summary

Keep this PR open. Current main only partially implements the intended strict-mode recovery: it recovers empty-result unclosed reasoning tails, but the guard is still trim-based, so the whitespace-prefix safety concern raised in review is not resolved. The PR is also far broader than its stated Telegram text fix and needs to be reduced before it is reviewable.

## Best Possible Solution

Keep this PR open but do not land it as-is. The best path is a narrow shared-text fix for the linked #49104 path: decide the strict-mode policy for whitespace-only prefixes, add explicit regression tests for that case plus the zero-captured recovery cases, verify Telegram split/dispatch behavior, and remove or split the unrelated dist.bak, hook, gateway, UI, config, and command changes into separate PRs.
