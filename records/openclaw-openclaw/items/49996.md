## Summary

Keep this PR open. Current main contains a related partial reasoning-tag recovery, but it does not make this PR safely obsolete: the linked Telegram bug #49104 is still open, current tests do not cover the whitespace-prefix safety case or the main unclosed-second-block path, and this PR is too broad/security-sensitive to land as-is.

## Best Possible Solution

Keep this PR open but do not land it as-is. The best path is a narrow shared-text and Telegram delivery fix for #49104: decide the strict-mode policy for whitespace-only prefixes, add explicit regression tests for that case plus the zero-captured and unclosed-second-block recovery paths, verify Telegram split/dispatch behavior, and move the unrelated dist.bak, hook, gateway, UI, config, command, and exec-related work into separate reviewed PRs or drop it from this branch.
