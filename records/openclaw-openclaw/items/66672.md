## Summary

Keep open. Current main has useful safety primitives, but it still does not provide first-class Ask, Plan, and Modify workflow modes with enforced read-only behavior, plan preview/approval, and explicit transition into execution.

## Best Possible Solution

Keep this open for core product/runtime design. A good implementation would add an enforced workflow/session mode layer: Ask blocks mutating tools at the policy boundary, Plan records intended file/action changes and requires explicit approval before execution, and Modify enables existing write/exec-capable paths under the current sandbox, tool-policy, and exec-approval rules. It should include UI/session controls and tests at the runtime/tool-policy boundary, not just prompt wording.
