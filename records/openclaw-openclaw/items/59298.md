## Summary

Keep this PR open: the cached update-hint CLI helper is concrete and tested in the branch, but the claimed automatic skill notification path is not wired to an executed hook. Current skill delivery advertises or materializes SKILL.md files; it does not execute fenced Markdown bash blocks before a skill runs.

## What This Changes

The branch adds an `openclaw update hint` subcommand with tests and inserts cached update hint bash blocks into seven `.agents/skills/*/SKILL.md` files.

## Best Possible Solution

If maintainers want this feature, keep the small cached-state CLI helper but surface update availability through a shared executed runtime hook or documented generic skill prelude, with tests proving the hint appears for eligible skill invocations without hand-editing individual bundled skill files.
