## Summary

Keep open. Current main still lacks `openclaw update hint`, and the related proposal #59304 was closed as superseded by this PR, but the patch is not ready as-is because the SKILL.md bash blocks are passive Markdown in the current OpenClaw skill paths rather than an executed preamble.

## What This Changes

The PR adds an `openclaw update hint` CLI subcommand that reads cached update-check state and adds update-hint bash blocks to selected `.agents/skills/*/SKILL.md` files.

## Best Possible Solution

If maintainers want this feature, keep the CLI helper but surface it through an executed OpenClaw skill/Claude CLI runtime prelude or documented generic hook, with tests proving the hint runs for eligible skill invocations without hand-editing each skill file.
