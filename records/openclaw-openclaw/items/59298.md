## Summary

Keep this PR open. Current main does not implement the proposed `openclaw update hint` subcommand or the skill preambles, and the PR is a focused implementation candidate with review feedback already addressed in the provided discussion. Existing main behavior only reports update availability through gateway startup/status surfaces, which does not solve the PR’s stated skill-invocation use case.

## Best Possible Solution

Keep the PR open for normal maintainer review. If the feature direction is accepted, land an implementation that adds the cached-state `openclaw update hint` command, JSON-negative output guarantees, schema validation for persisted state, focused unit coverage, and carefully reviewed skill preambles. If maintainers decide skill preambles should not execute commands, close the PR with that product/security rationale rather than as already implemented.
