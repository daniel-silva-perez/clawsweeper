## Summary

Keep open. Current main still lets ANSI-bearing `logs.tail` text reach the dedicated Control UI Logs tab unchanged; only the Overview tail has a local strip helper. The related fix PR #64431 is closed unmerged, and #45001 was already closed as a duplicate pointing back to this remaining bug.

## What This Changes

The issue requests that Control UI gateway log panels stop displaying raw ANSI escape sequences and instead render or strip them for readable browser display.

## Best Possible Solution

Add a shared Control UI ANSI display helper and apply it consistently to every `logs.tail` UI consumer, including structured message, subsystem, and fallback text. Preserve raw log/export behavior deliberately, remove the duplicated Overview-only helper, and add regression tests for ANSI-bearing structured and plain lines.

## Reproduction Assessment

Yes. A source-level reproduction is clear: feed an ANSI-bearing JSONL line into `parseLogLine()`, then render the resulting entry through `renderLogs()`; current main preserves the escape sequences in `message` and displays that value directly.

## Solution Assessment

Yes. A narrow UI display normalization is the maintainable fix boundary; changing the server-side log tail would risk altering raw export and CLI behavior, while a shared UI helper covers both the Logs tab and Overview tail.
