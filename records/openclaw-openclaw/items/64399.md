## Summary

Keep open. Current main only strips ANSI in the Overview Gateway Logs tail; the dedicated Control UI Logs tab still consumes `logs.tail`, preserves ANSI-bearing `message`/`raw` text, and renders it directly. The related PR #64431 was closed unmerged, so it cannot close the remaining bug.

## Best Possible Solution

Keep this issue open and land a focused Control UI logging fix. The likely best path is a shared ANSI display strategy for every `logs.tail` UI consumer: strip or safely render ANSI in the dedicated Logs tab and Overview tail, preserve raw export behavior intentionally, and add regression coverage for ANSI-bearing JSON messages, subsystem/context parsing, and raw fallback lines. PR #64431 is relevant prior art but needs to be revived or replaced because it was closed unmerged.
