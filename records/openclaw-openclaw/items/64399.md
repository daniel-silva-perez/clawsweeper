## Summary

Keep open. Current main still preserves ANSI-bearing `logs.tail` text in the dedicated Control UI Logs tab and renders it directly; the Overview tail has only a local display strip helper. The related fix PR #64431 is closed unmerged, and #45001 was already closed as a duplicate pointing back to this remaining bug.

## What This Changes

The issue requests that Control UI gateway log output stop showing raw ANSI escape sequences and instead render or normalize them for readable browser display.

## Best Possible Solution

Use one shared Control UI ANSI display strategy for every `logs.tail` consumer, normalize displayed message/subsystem/fallback text, keep raw export behavior intentional, and add regression coverage for ANSI-bearing structured JSON and plain log lines.
