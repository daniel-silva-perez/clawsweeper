## Summary

Keep this PR open because it targets a real current-main cron/tool filtering bug, but the submitted patch should not merge as-is. The one-line final `applyEmbeddedAttemptToolsAllow()` pass fixes exec-only leakage while breaking the existing plugin-aware `toolsAllow: ["bundle-mcp"]` path for bundled MCP tools.

## What This Changes

The PR re-applies runtime `toolsAllow` filtering to the combined embedded-runner tool list after bundled MCP/LSP tools are materialized.

## Best Possible Solution

Enforce runtime `toolsAllow` after bundled MCP/LSP materialization with the same plugin-aware semantics used by the final bundled-tool policy, preserving exact bundled tool names and plugin/group keys such as `bundle-mcp`, then add focused regression coverage for exec-only cron filtering and bundled MCP allowlist cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: schedule an `agentTurn` with `toolsAllow: ["exec"]` while bundle MCP tools are available to see the current-main leak, and apply this PR with `toolsAllow: ["bundle-mcp"]` to see all bundled MCP tools filtered out by exact name.

## Solution Assessment

No. The repair direction is right, but this exact patch is not the best fix because it applies an exact-name helper after the plugin-aware bundled-tool policy, losing supported plugin-key allowlist semantics.
