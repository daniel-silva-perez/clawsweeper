## Summary

Keep open. Current main still lacks ACP child-session cancellation during subagent stop, so this PR addresses a real runtime gap, but the current PR head is stale/unmergeable and has blocking correctness issues that need normal maintainer/author handling before merge.

## What This Changes

The PR makes subagent stop propagation async, awaits stop/reset callers, cancels ACP-backed child sessions, persists child aborted state, and adds fast-abort regression coverage.

## Best Possible Solution

Keep the child-stop cascade direction, but bound child ACP cancel like the existing gateway ACP cleanup path, preserve current main's dedupe behavior during rebase, and add regression coverage for successful ACP child cancellation plus hung-cancel fallback before merge.

## Reproduction Assessment

Yes. A child ACP runtime whose cancel promise never resolves reproduces the reset hang once the PR makes reset await stopSubagentsForRequester; the PR-head test failure is also reproducible from the missing latest-run mock on that branch.

## Solution Assessment

No, not as proposed. Propagating stop into ACP-backed child sessions is the right direction, but the mergeable solution needs bounded cancellation and a refreshed test setup/rebase against current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Bound ACP child cancellation during stop cascade:** `src/auto-reply/reply/abort.ts:199-203`
  - body: The new child-stop loop awaits acpManager.cancelSession for each active child. AcpSessionManager.cancelSession can wait on the backend cancel promise without a local timeout, and session reset now awaits this cascade, so one stuck ACP backend can hang stop/reset. Wrap this child cancel in a bounded cleanup path and continue/log on timeout.
  - confidence: 0.9
- **[P2] Mock the latest child run in the ACP cascade test:** `src/auto-reply/reply/abort.test.ts:600-613`
  - body: On the PR head, stopSubagentsForRequester skips a run when getLatestSubagentRunByChildSessionKey(childKey) returns null. This test only mocks the list call, so the child run is discarded before cancelSession is reached and the new assertions fail. Return the same run from the latest-run mock before invoking runStopCommand, or rebase and adjust the test for current main's dedupe behavior.
  - confidence: 0.86
