## Summary

Keep open. Current main still has the underlying ACP-backed child-stop gap, but this PR is not ready to merge because it introduces an unbounded awaited ACP cancel path from reset/stop cleanup and the submitted regression test is incomplete against the PR head.

## What This Changes

The PR makes subagent stop propagation async, awaits stop/reset callers, cancels ACP-backed child sessions, persists child aborted state, and adds fast-abort regression coverage.

## Best Possible Solution

Keep the ACP child-stop cascade, but make child `cancelSession` bounded like gateway ACP cleanup, preserve current main's dedupe and reply-run abort behavior during rebase, and refresh regression coverage for both successful ACP child cancellation and hung-cancel fallback.

## Reproduction Assessment

Yes. The blocking findings are reproducible by static path inspection of the PR head: reset awaits the child-stop cascade, child stop awaits ACP cancel, and ACP cancel awaits backend cancellation without its own timeout; the added test also skips its intended path because the required latest-run mock is absent on the PR head.

## Solution Assessment

No, not as proposed. Propagating stop into ACP-backed child sessions is the right direction, but the mergeable solution needs bounded cancellation plus a rebased test setup that exercises the intended child run.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Bound ACP child cancellation during stop cascade:** `src/auto-reply/reply/abort.ts:199-203`
  - body: The new child-stop loop awaits `acpManager.cancelSession(...)` for each active child. `AcpSessionManager.cancelSession` waits on the backend cancel promise without its own timeout, and session reset now awaits this cascade, so one stuck ACP backend can hang stop/reset. Wrap this child cancel in the same bounded cleanup pattern used by gateway ACP cleanup and continue/log on timeout.
  - confidence: 0.9
- **[P2] Mock the latest child run in the ACP cascade test:** `src/auto-reply/reply/abort.test.ts:600-613`
  - body: On the PR head, `stopSubagentsForRequester` skips a run when `getLatestSubagentRunByChildSessionKey(childKey)` returns null. This test only mocks the list call, so the child run is discarded before `cancelSession` is reached and the new assertions fail. Return the same run from the latest-run mock before invoking `runStopCommand`.
  - confidence: 0.86
