## Summary

Keep this issue open. Current main still launches the built-in `profile="user"` Chrome MCP path through the upstream MCP SDK stdio transport, and the explicit maintainer comment says the Termux stdin parent-death failure is separate from the merged #71560 command/args work.

## Best Possible Solution

Keep this issue open. The best fix is a targeted browser Chrome MCP lifecycle change for Android/Termux, with a regression test that simulates the Chrome MCP subprocess or watchdog closing after stdin-end/parent-death and verifies `profile="user"` reports, recovers, or avoids that failure. The #71560 command/args work should stay related context, not the canonical closure.
