## Summary

Keep this PR open. Current main still lacks the shared proxy-fetch circuit breaker the PR proposes, and the PR branch needs a rebase plus correctness/security fixes before it is mergeable.

## Best Possible Solution

Keep this PR open as useful but not mergeable as-is. The best path is a rebased implementation that adds proxy circuit-breaker behavior at the shared proxy helper/plugin-facing surface and active Discord/Telegram call sites, preserves managed-proxy fail-closed semantics, allows only one half-open probe per cooldown, fixes direct-error attribution, splits unrelated rate-limit matcher changes, and adds focused regression tests for fallback policy, cooldown, single-probe behavior, and recovery.
