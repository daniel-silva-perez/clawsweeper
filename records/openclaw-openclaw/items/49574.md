## Summary

Keep this PR open. Current main still does not implement a shared proxy-fetch circuit breaker, and the PR branch is stale against current plugin paths and current proxy contracts. The proposed direction is useful, but it needs a rebase plus correctness and security fixes before it is mergeable.

## Best Possible Solution

Keep this PR open as useful but not mergeable as-is. The best path is a rebased, policy-aware implementation that adds circuit-breaker behavior at the shared proxy helper/plugin-facing surface and active Discord/Telegram call sites, preserves managed-proxy fail-closed semantics, allows only one half-open probe per cooldown, records failures only for requests that actually used the proxy, splits unrelated rate-limit matcher changes, and adds focused tests for fallback policy, cooldown, single-probe behavior, recovery, and managed-proxy behavior.
