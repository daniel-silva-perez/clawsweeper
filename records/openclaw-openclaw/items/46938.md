## Summary

Keep open. Current main has related pieces, but not the requested timeout-triggered session preservation/recovery behavior. Pre-compaction memory flush is gated before a normal run, optional dreaming ingests session transcripts into a short-term corpus, and experimental session memory search indexes transcripts asynchronously. None of these detect consecutive model timeouts, write a pending session artifact, or recover it into `memory/YYYY-MM-DD.md` after the model comes back.

## Best Possible Solution

Keep this issue open and implement a generic timeout recovery path in the session/model-fallback lifecycle. On exhausted consecutive timeout failures, record a redacted pending transcript artifact with session id, session key, timestamps, and source lines; after model availability returns, process it through a bounded maintenance job or memory-core API into the intended daily memory surface, with tests for timeout detection, dedupe, redaction, and recovery cleanup.
