## Summary

Keep this PR open. Current main still has the sandbox skill prompt gap this PR targets: non-rw sandbox attempt and compaction runs switch to the sandbox workspace, but skill loading still skips live workspace entries when a snapshot has resolved skills, and prompt resolution still returns the host-built snapshot prompt before considering sandbox-local entries. The provided PR context also links this work to open bug #43383 and shows recent maintainer-side repair activity, so it is not obsolete.

## Best Possible Solution

Keep this PR open and review or land the repaired branch, or replace it with an equivalent focused agents fix. The final implementation should force-load sandbox-local, config-eligible skill entries for non-rw attempt and compaction runs, rebuild prompts from prompt-visible sandbox entries instead of host snapshots, avoid empty-entry fallback to host prompts, preserve snapshot filters and remote eligibility, align env overrides with the prompt-visible entries, and keep regression coverage for both attempt and compaction paths.
