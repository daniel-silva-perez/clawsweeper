## Summary

Keep this PR open. The target main SHA still has the bug the PR is meant to fix: non-rw sandbox runs switch to the sandbox workspace, but resolved skill snapshots still skip live sandbox skill loading and the prompt helper still returns the saved snapshot prompt before considering sandbox entries. The PR is also active after a maintainer repair push, so the best cleanup action is review and land this PR or an equivalent focused fix, not close it as obsolete.

## Best Possible Solution

Keep this PR open and review/land the repaired branch, or replace it with an equivalent focused agents fix. The final implementation should force-load config-eligible sandbox skill entries for non-rw sandbox attempt and compaction runs, rebuild the prompt from those entries instead of host snapshots, return no snapshot prompt when the sandbox has no eligible skills, preserve the saved skill filter and remote eligibility context, align env overrides with the advertised skill set, and include regression coverage for both attempt and compaction paths.
