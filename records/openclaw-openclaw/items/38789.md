## Summary

Keep this PR open. Current main still has the sandbox skill prompt behavior the PR targets: non-rw sandbox runs switch to the sandbox workspace, but existing skill snapshots still prevent live sandbox skill entry loading and the prompt resolver still returns the saved host-side snapshot prompt first. The PR is an active implementation candidate for open bug #43383, with a recent maintainer repair push and merge-ready review labels, so the maintainer path is to review or land this branch or an equivalent focused fix.

## Best Possible Solution

Keep this PR open and review or land the repaired branch, or replace it with an equivalent focused agents fix. The final implementation should force-load config-eligible sandbox skill entries for non-rw attempt and compaction runs, rebuild prompts from sandbox entries instead of host snapshots, avoid falling back to host snapshot prompts when sandbox entries are empty, preserve snapshot skill filters and remote eligibility, align env overrides with the prompt-visible skills, and keep the focused regression coverage. After that lands, #43383 can be closed as fixed.
