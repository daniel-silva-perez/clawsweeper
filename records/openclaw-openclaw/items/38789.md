## Summary

Keep this PR open. Current main still has the sandbox skill prompt bug the PR targets: non-rw sandbox runs switch to the sandbox workspace, but resolved skill snapshots still prevent live sandbox skill loading and `resolveSkillsPromptForRun` still returns the host-side snapshot prompt before considering sandbox entries. The latest PR branch is a focused implementation candidate, recently repaired to address the earlier review gaps around empty sandbox skill sets, skill filters, remote eligibility, and env override alignment. Security review did not find supply-chain or CI/dependency changes; the diff is confined to agents skill/runtime code, tests, and changelog, with the main security-sensitive behavior being narrower env override application from filtered live entries in sandboxed runs.

## Best Possible Solution

Keep this PR open and review or land it as the canonical fix for #43383, or replace it with an equivalent agents patch that forces live sandbox skill entry loading for non-rw sandbox attempt and compaction runs, rebuilds prompts from filtered sandbox entries, avoids host snapshot fallback for empty sandbox skill sets, preserves snapshot skill filters and remote eligibility, and keeps skill env overrides aligned with the advertised live skill set.
