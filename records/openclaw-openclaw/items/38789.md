## Summary

Keep this PR open. Current main still has the sandbox skill prompt behavior the PR is trying to fix: non-rw sandbox runs switch to the sandbox workspace, but resolved skill snapshots still suppress live sandbox skill loading and the saved snapshot prompt is returned before sandbox-local entries are considered. The PR remains an active implementation candidate for open bug #43383, with recent maintainer repair comments and merge-ready labels in the provided GitHub context.

## Best Possible Solution

Keep this PR open and review or land the repaired branch, or replace it with an equivalent focused agents fix. The final implementation should force-load config-eligible sandbox skill entries for non-rw attempt and compaction runs, rebuild prompts from sandbox prompt-visible entries instead of host snapshots, avoid falling back to host snapshot prompts when sandbox entries are empty, preserve snapshot skill filters and remote eligibility, align env overrides with the prompt-visible skill set, and keep the focused regression coverage. After that lands, #43383 can close as fixed.
