## Summary

Keep this PR open. Current main at e583db63c602d8d21b4ebdcd119061571a2e40a9 still has the sandbox skill prompt gap this PR targets: non-rw sandbox runs switch to the sandbox workspace, but skill loading still skips live workspace entries when a snapshot has resolved skills, and prompt resolution still returns the host-built snapshot prompt first. The PR branch is an active, focused fix candidate linked to open bug #43383 and has been repaired to address earlier review concerns around empty-entry fallback, skill filters, remote eligibility, and env override alignment.

## Best Possible Solution

Keep this PR open and either land the repaired branch or replace it with an equivalent focused agents fix. The final implementation should force-load sandbox-local, config-eligible skill entries for non-rw attempt and compaction runs; rebuild prompts from prompt-visible sandbox entries instead of host snapshots; avoid empty-entry fallback to host prompts; preserve snapshot filters and remote eligibility; align env overrides with the prompt-visible entries; and keep the regression coverage added by the PR.
