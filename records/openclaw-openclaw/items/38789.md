## Summary

Keep this PR open. Current main at 955b4df0939f90b4b6de5e7e6f3149dd125e1e50 still has the sandbox skill prompt gap this PR targets: non-rw sandbox runs switch to the sandbox workspace, but the run still reuses host-built skill snapshots first, skips live sandbox skill entry loading when resolved snapshot skills exist, and returns the snapshot prompt before entries. The linked bug #43383 is still open, and this PR remains the active focused implementation candidate.

## Best Possible Solution

Keep this PR open and either land the repaired branch or replace it with an equivalent focused agents fix. The final implementation should force-load sandbox-local, config-eligible skill entries for non-rw attempt and compaction runs; rebuild prompts from prompt-visible sandbox entries instead of host snapshots; avoid empty-entry fallback to host prompts; preserve snapshot skill filters and remote eligibility; align env overrides with the prompt-visible entries; and keep the added regression coverage.
