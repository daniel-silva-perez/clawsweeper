## Summary

Keep this PR open. Current main has not implemented the heartbeat/session activity UI or progress-prompt changes, and the PR branch still contains blocking behavior regressions plus a security-sensitive expansion of session abort semantics that needs maintainer design review.

## What This Changes

This PR changes WebChat/Control UI and OpenClawKit chat state to show heartbeat/session stop controls, adds a `sessions.activity` RPC, broadens `sessions.abort` cleanup, forwards assistant deltas, and adds progress/milestone prompt guidance.

## Best Possible Solution

Split and rebase the work: keep ordinary chat generation Stop on `chat.abort`, design any session activity/autonomy stop API against #39127 with protocol docs and authorization coverage, and land progress-prompt guidance separately with focused prompt or behavior tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep regular Stop on chat.abort:** `ui/src/ui/controllers/chat.ts:337`
  - body: The regular WebChat abort path now calls `sessions.abort`. That server handler also clears queues, stops subagents/ACP, and disables session-bound cron jobs, so stopping one in-flight answer can pause unrelated session automation. Keep normal generation abort on `chat.abort` or gate the broad cleanup behind an explicit autonomy-stop action.
  - confidence: 0.9
- **[P2] Replace snapshots instead of appending them:** `ui/src/ui/controllers/chat.ts:208`
  - body: When a non-prefix assistant snapshot arrives without `delta`, this fallback appends it to the existing stream. Gateway snapshots are authoritative in that case, so rewritten or replayed text can be duplicated or corrupted in the UI and in locally committed abort partials.
  - confidence: 0.86
- **[P2] Exclude dropped followups from pending size:** `src/auto-reply/reply/queue/state.ts:43`
  - body: `droppedCount` tracks discarded queue entries, not pending work. Including it makes `followupQueued > 0` and `canStop: true` after the queue is actually empty, causing the stop control to appear for work that cannot be stopped.
  - confidence: 0.91
- **[P2] Return cleanup state after timeout errors:** `src/gateway/server-methods/sessions.ts:1030-1040`
  - body: By this point queues, subagents, ACP sessions, and cron jobs may already have been cleaned up. If the embedded-run wait times out, the handler returns only `UNAVAILABLE`, so clients cannot reconcile the work already stopped and may keep stale UI state or retry unnecessarily.
  - confidence: 0.75
- **[P3] Reset heartbeat visibility from incoming state changes:** `ui/src/ui/app-gateway.ts:413-419`
  - body: The heartbeat event handler only sets `heartbeatsEnabled` to true. If another client disables heartbeats through `set-heartbeats`, this connected UI has no incoming path to set the flag back to false, so the heart button can stay visible until reconnect.
  - confidence: 0.72
