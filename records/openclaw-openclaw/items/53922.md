## Summary

Keep this PR open for maintainer handling. Current main does not implement the proposed heartbeat/session activity stop UI or progress-prompt changes, and the PR branch still mixes UI, Gateway protocol, session-control, Swift client, and prompt-policy changes with blocking correctness and authorization-scope concerns.

## What This Changes

The PR adds WebChat heartbeat/session stop controls, a new `sessions.activity` RPC, broader `sessions.abort` cleanup, stream/progress handling changes, Swift chat progress handling, and system-prompt guidance for progress and milestone updates.

## Best Possible Solution

Split and rebase the work: keep ordinary chat generation Stop on `chat.abort`, design any session/autonomy stop and activity API against #39127 with protocol docs and authorization coverage, and land progress-prompt guidance separately with focused tests or prompt snapshots.

## Reproduction Assessment

Yes for the review findings: the PR diff and current-main code provide a high-confidence static reproduction path for the changed abort routing, stream merge behavior, queue count calculation, and session cleanup response handling. A live UI run was not needed to identify those logic regressions.

## Solution Assessment

No. This PR is not the best way to solve the issue because it routes normal Stop through broader destructive cleanup and combines multiple product surfaces; the safer path is a split design with a distinct autonomy-stop/activity API and separate progress-prompt change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep normal chat Stop on chat.abort:** `ui/src/ui/controllers/chat.ts:337`
  - body: The regular WebChat stop path now calls `sessions.abort`, whose handler also clears queues, stops subagents, cancels ACP sessions, and disables session-bound cron jobs. Stopping one answer can pause unrelated session automation, so ordinary generation Stop should stay on `chat.abort` or the broader cleanup needs a distinct explicit action.
  - confidence: 0.9
- **[P2] Use snapshots as replacements when no delta is present:** `ui/src/ui/controllers/chat.ts:208`
  - body: When a non-prefix assistant snapshot arrives without `delta`, this fallback appends it to the current stream. Gateway snapshots are authoritative in that case, so rewritten or replayed text can be duplicated or corrupted in the UI and in locally committed abort partials.
  - confidence: 0.86
- **[P2] Exclude dropped followups from pending size:** `src/auto-reply/reply/queue/state.ts:43`
  - body: `droppedCount` tracks discarded queue entries, not pending work. Including it makes `followupQueued > 0` and `canStop: true` after the queue is actually empty, causing the stop control to appear for work that cannot be stopped.
  - confidence: 0.91
- **[P2] Return cleanup state after timeout errors:** `src/gateway/server-methods/sessions.ts:1030-1039`
  - body: By this point queues, subagents, ACP sessions, and cron jobs may already have been cleaned up. If the embedded-run wait times out, the handler returns only `UNAVAILABLE`, so clients cannot reconcile the work already stopped and may keep stale UI state or retry unnecessarily.
  - confidence: 0.75
