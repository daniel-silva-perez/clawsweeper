## Summary

Keep this PR open. Current main has the ordinary WebChat Stop button and `chat.abort` path, but it still does not contain the PR's heartbeat-specific stop UI, `sessions.activity` API, heartbeat-disable wiring, or explicit progress/milestone system-prompt guidance. The PR remains potentially useful, though it needs rework because its review discussion identifies concrete correctness and behavior risks.

## Best Possible Solution

Keep the PR open for maintainer review and rework. The clean path is to split the heartbeat/autonomous-task stop work from the progress-prompt work, define a clear session activity/stop contract, keep ordinary `chat.abort` from disabling unrelated cron jobs, and add focused tests for queue size, heartbeat state synchronization, abort cleanup, and progress-update prompt behavior.
