Keep open: current main still has the reported live embedded-session compaction bug. The subscribe handler still zeroes every assistant usage snapshot after a non-retry compaction end, while the replay sanitizer already has boundary-aware logic that preserves fresh post-compaction usage; the linked fix PRs are closed unmerged and the merged provider-usage PR explicitly left this helper out of scope.

The issue requests preserving valid post-compaction assistant usage snapshots so the TUI and status context counter do not remain at zero after compaction.

## Best Possible Solution

Patch the embedded subscribe compaction handler to identify the latest compaction summary and zero only assistant usage snapshots before that boundary, using timestamp comparison plus index fallback for legacy messages. Preserve post-compaction usage, add focused handler/session regression coverage, and avoid changing provider usage normalization, replay sanitization policy, status formatting, or unrelated compaction behavior.
