## Summary

Keep open: current main still has the reported live embedded-session compaction bug. The subscribe handler still zeroes every assistant usage snapshot after a non-retry compaction end, while the replay sanitizer already has boundary-aware logic that preserves fresh post-compaction usage; related fix PRs are closed unmerged, and the merged provider-usage PR explicitly left this helper out of scope.

## What This Changes

The issue asks OpenClaw to preserve assistant usage created after compaction so the TUI/context status counter does not remain at zero.

## Best Possible Solution

Patch the subscribe compaction handler to use latest-compaction boundary semantics, ideally shared with or matched to replay sanitization, then add regression coverage proving stale pre-compaction usage is zeroed while fresh post-compaction usage survives.
