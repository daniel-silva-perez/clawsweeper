## Summary

Keep open: current main still has the reported embedded-session compaction bug. The live subscribe handler still zeroes every assistant usage snapshot after a non-retry compaction end, while the replay sanitizer already has boundary-aware behavior that preserves fresh post-compaction usage; the related fix PR is closed unmerged and the merged provider-usage PR explicitly left this helper unchanged.

## What This Changes

The issue asks OpenClaw to preserve assistant usage created after compaction so the TUI/context status counter does not remain at 0.

## Best Possible Solution

Make the subscribe compaction handler use the same latest-compaction boundary semantics as replay sanitization, then add regression coverage that proves fresh assistant usage after the compaction summary survives while older stale usage is zeroed.
