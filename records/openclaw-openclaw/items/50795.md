## Summary

Keep open: current main still has the reported live embedded compaction bug. The subscribe handler still zeroes every assistant usage snapshot after a non-retry compaction end, while the replay sanitizer already has boundary-aware logic that preserves fresh post-compaction usage; the related fix PR is closed unmerged and the merged provider-usage PR explicitly left this helper out of scope.

## What This Changes

The issue asks the embedded subscribe compaction handler to preserve fresh post-compaction assistant usage so TUI/status context token counts do not stay at zero after compaction.

## Best Possible Solution

Patch the embedded subscribe compaction handler so it identifies the latest compaction summary and zeroes only assistant usage snapshots before that boundary, preserving fresh post-compaction usage. Add focused regression coverage around the handler/subscription path and leave provider usage normalization, UI display logic, replay sanitization, and session-store token persistence unchanged unless a small shared helper can be extracted cleanly.
