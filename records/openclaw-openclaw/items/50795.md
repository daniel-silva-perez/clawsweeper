## Summary

Current main does not implement the requested fix. The embedded subscribe compaction handler still calls the stale-usage cleanup on every non-retry compaction end, and that cleanup still overwrites every assistant message usage snapshot with zeros. A correct timestamp/index-gated pattern exists in the replay-history sanitizer, but it has not been applied to this handler; the linked fix PR #50845 is closed unmerged.

## Best Possible Solution

Keep this issue open and implement the fix in the embedded subscribe compaction handler: either share the replay-history timestamp/index-gated helper or port that behavior into clearStaleAssistantUsageOnSessionMessages(), then add a handler-level regression test with pre-compaction and post-compaction assistant usage around the latest compactionSummary.
