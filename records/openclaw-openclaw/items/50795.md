## Summary

Keep open. Current main still has the reported runtime bug in the embedded subscribe compaction handler: after a non-retry compaction end it zeroes every assistant usage snapshot, while the replay sanitization path already has the narrower compaction-boundary behavior this issue asks for. The linked fix PR #50845 is closed unmerged, and the merged #68711 explicitly left this helper out of scope.

## What This Changes

The issue asks for post-compaction assistant usage snapshots to be preserved so the TUI context token count does not remain at zero after compaction.

## Best Possible Solution

Patch the embedded subscribe compaction handler so it identifies the latest compaction summary and zeroes only assistant usage snapshots before that boundary, preserving fresh post-compaction usage. Add focused regression coverage in the subscribe/handler tests and leave provider usage normalization, UI display logic, replay sanitization, and session-store token persistence unchanged unless a shared helper is extracted cleanly.
