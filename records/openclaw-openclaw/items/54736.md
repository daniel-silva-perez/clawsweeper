## Summary

Current main still has the reported core suppression gap. Bare repeated silent-token payloads such as `NO_REPLY\n\nNO_REPLY` do not match the shared predicate, and the normalization path can strip only one trailing token and leave `NO_REPLY` visible for downstream delivery.

## Best Possible Solution

Keep this issue open. The best fix is to update the shared silent-token predicate or normalization path in `src/auto-reply/tokens.ts` and `src/auto-reply/reply/normalize-reply.ts` so token-only sequences like `NO_REPLY\n\nNO_REPLY` are suppressed while preserving #19537 mixed-content behavior, then add focused regression tests in `src/auto-reply/tokens.test.ts`, `src/auto-reply/reply/reply-utils.test.ts`, and the runtime delivery plan surface.
