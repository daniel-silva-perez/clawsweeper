## Summary

Keep this PR open. Current main still lacks importance-weighted temporal decay for memory search, and the PR remains a plausible implementation candidate, but it needs a real port from the old `src/memory` layout to the active `extensions/memory-core` implementation plus cleanup of the existing review-comment issues.

## Best Possible Solution

Keep this PR open as the implementation candidate. Port the feature into `extensions/memory-core/src/memory/temporal-decay.ts`, `hybrid.ts`, and `manager.ts`; thread a well-specified `importanceBoost` config through `src/agents/memory-search.ts`, runtime schemas, public config types, generated schema/help/label artifacts, and docs; add focused memory-core tests for marker matching, file-pattern matching, deep-merge/default semantics, blank-pattern rejection, and ranking behavior; and split or drop the already-shipped session-memory reset and unwired `periodicExtraction` pieces. Coordinate the product direction with the broader open memory-importance issue #57307 if maintainers want scoring beyond marker-based decay.
