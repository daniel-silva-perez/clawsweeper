## Summary

Keep this PR open. Current main still lacks importance-weighted temporal decay for memory search, and the PR remains a plausible implementation candidate, but it needs to be ported from the old `src/memory` layout to the active `extensions/memory-core` implementation and cleaned up before merge.

## Best Possible Solution

Keep this PR open as an implementation candidate. Port the feature into `extensions/memory-core/src/memory/temporal-decay.ts`, `hybrid.ts`, and `manager.ts`; thread a well-specified `importanceBoost` config through `src/agents/memory-search.ts`, runtime schemas, public config types, generated schema/help/label artifacts, and docs; add focused memory-core tests for marker matching, file-pattern matching, nested merge/default behavior, blank-pattern rejection, and ranking behavior; then split or drop the already-shipped session-memory reset and unwired `periodicExtraction` pieces. Coordinate with #57307 if maintainers want broader memory importance scoring beyond this PR’s marker-based decay boost.
