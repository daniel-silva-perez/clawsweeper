## Summary

Keep this PR open. Current main still lacks the central feature, importance-weighted temporal decay for memory search. The active implementation only supports plain temporal decay with `enabled` and `halfLifeDays`; the PR remains a live implementation candidate, but it needs rework for the current `extensions/memory-core` layout and the unresolved config/schema review issues.

## Best Possible Solution

Keep this PR open as the live implementation candidate. Port the ranking feature to `extensions/memory-core/src/memory/temporal-decay.ts` and the current hybrid/manager wiring, thread a well-specified `importanceBoost` config through `src/agents/memory-search.ts`, runtime schemas, generated config artifacts, and docs, and add focused memory-core tests for markers, path patterns, default/override merging, and blank-pattern rejection. Drop or split the already-shipped session-memory reset changes and the unrelated `periodicExtraction` schema work unless they are fully wired and tested separately.
