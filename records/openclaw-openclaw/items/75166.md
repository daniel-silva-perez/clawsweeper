## Summary

Keep open. Current main still contains the compact provider-inventory emission bypass for image/video list results, and Discord guild-channel turns are still treated as channel surfaces that can deliver tool-result text; the closed #70912 discussion is useful context for the leak class but does not cover this separate provider-inventory path.

## What This Changes

The issue requests suppressing image/video provider inventory from user-visible Discord group-channel tool output while preserving internal tool results and generated media delivery.

## Best Possible Solution

Keep provider inventory available inside the tool result for the agent, but stop the compact `action=list` inventory from being emitted as external chat tool output by default. The fix should be in the shared tool-output visibility path or a generic surface policy, with tests for image/video list results and Discord guild-channel delivery; generated media delivery should remain unchanged.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists: feed a `video_generate` or `image_generate` list result with `details.providers` through `handleToolExecutionEnd` while `shouldEmitToolOutput()` is false, then follow the Discord guild-channel dispatch path where `ChatType: "channel"` remains eligible for visible tool-result delivery.

## Solution Assessment

Yes, the suggested direction is the right class of fix. The narrow maintainable solution is to remove or surface-gate the compact inventory bypass while preserving internal tool context and media attachments; redacting the text alone is less robust unless maintainers intentionally want a minimal visible list mode.
