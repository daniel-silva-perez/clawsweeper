## Summary

Keep open. Current main still contains the compact provider-inventory emission path, the existing media test asserts that it emits even when normal tool output is disabled, and Discord guild messages are treated as channel surfaces where text tool results can be delivered. The related #70912 fix covers Slack verbose progress traces, not this separate image/video inventory output path.

## What This Changes

The issue reports that `image_generate` and `video_generate` provider inventory list output can leak provider/model/configuration/auth-hint details as visible Discord group-channel tool output.

## Best Possible Solution

Keep the full provider inventory in the agent/tool result context, but stop default external chat emission for `action=list` inventory text on shared surfaces. The likely implementation should remove or surface-gate the compact inventory bypass while preserving generated media attachment delivery.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: return an `image_generate` or `video_generate` list result with `details.providers` while `shouldEmitToolOutput()` is false, and current main still calls `emitToolOutput`; Discord guild turns are `ChatType: "channel"`, so the generic dispatcher can deliver the resulting text.

## Solution Assessment

Yes, the requested direction is the right class of fix. The narrow maintainable solution is to suppress or explicitly gate compact provider-inventory output before external delivery; redacting the text is weaker unless maintainers intentionally want a minimal visible inventory mode.
