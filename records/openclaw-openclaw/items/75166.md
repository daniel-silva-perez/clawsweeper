## Summary

Keep open. Current main and the latest release tag still contain the compact provider-inventory bypass for image/video list results, and Discord guild messages are classified as channel surfaces where tool-summary payloads are not suppressed. The related #70912 Slack fix covers a similar visibility class but not this provider-inventory output path.

## What This Changes

Prevent image_generate/video_generate provider inventory from being emitted as visible Discord group/channel tool output by default.

## Best Possible Solution

Keep the full provider list available inside the agent/tool result, but stop sending image/video action=list inventory as external chat tool-output text by default. The fix should preserve generated media attachment delivery and add focused coverage for the embedded tool-output gate plus a non-direct Discord or generic channel delivery assertion.

## Reproduction Assessment

Yes. A high-confidence reproduction path exists from current source and tests: emit an image_generate or video_generate list result with details.providers while shouldEmitToolOutput is false, then follow the Discord guild-channel dispatch path where text tool summaries are still deliverable.

## Solution Assessment

Yes, the suggested fix direction is the right class of solution. The narrowest maintainable fix is to remove or surface-gate the compact inventory bypass while keeping internal tool context intact; redaction alone is weaker unless maintainers intentionally want some public list output.
