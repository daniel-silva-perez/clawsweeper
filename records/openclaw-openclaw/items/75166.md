## Summary

Keep open. Current main still contains the compact provider-inventory bypass, a regression test asserts that video provider inventory emits even when normal tool output is disabled, and Discord guild messages are normalized as channel surfaces that the shared dispatcher treats as tool-summary eligible. The related #70912 fix covers Slack verbose progress traces, not this image/video inventory output path.

## What This Changes

The issue asks OpenClaw to stop `image_generate` and `video_generate` provider inventory list output from being delivered as visible Discord group/channel tool output by default.

## Best Possible Solution

Make provider inventory from media `action=list` an internal tool result by default on shared chat surfaces, while preserving full inventory for the agent context and preserving generated media delivery. Add regression coverage for image and video inventory suppression plus Discord/channel dispatch behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: return an `image_generate` or `video_generate` list result with `details.providers` while `shouldEmitToolOutput()` is false, and current main still calls `emitToolOutput`; Discord guild turns are `ChatType: "channel"`, so that text can be delivered by the shared dispatcher.

## Solution Assessment

Yes, the requested direction is the right class of fix. The narrow maintainable solution is to remove or surface-gate the compact inventory bypass before external chat delivery, rather than changing provider list generation or media attachment delivery.
