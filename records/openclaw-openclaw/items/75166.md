## Summary

Keep open. Current main still contains the compact provider-inventory emission path for image/video list results, and Discord guild channels are still treated as a visible tool-summary surface; the related #70912 discussion is useful context for the same leak class, but it does not cover this separate provider-inventory output path.

## What This Changes

The issue asks OpenClaw to stop showing `image_generate` and `video_generate` provider inventory details as visible Discord group/channel tool output by default.

## Best Possible Solution

Keep the full provider inventory available inside the agent/tool result, but stop emitting `action=list` provider inventory as external chat tool output by default. The fix should be in the shared embedded tool-output gate, with focused coverage for image/video list results and Discord guild-channel or generic non-direct delivery behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: produce an `image_generate` or `video_generate` list result with `details.providers` while `shouldEmitToolOutput()` is false, observe `emitToolOutput`, then follow the Discord guild-channel dispatch path to visible tool-result delivery.

## Solution Assessment

Yes. The issue’s suggested direction is the right class of fix: remove or surface-gate the compact inventory bypass while preserving internal tool results and generated media delivery; redaction alone is less robust unless maintainers intentionally want a minimal visible inventory.
