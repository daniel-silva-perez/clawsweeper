## Summary

Keep this issue open. Current main still has the compact provider-inventory bypass for `image_generate` and `video_generate`, the media handler test still asserts that inventory output emits even when normal tool output is disabled, and Discord guild messages use `ChatType: "channel"`, which remains eligible for generic tool-summary delivery. The related #70912 fix is adjacent but only covers Slack non-direct verbose progress/tool traces, not this media provider inventory path.

## What This Changes

The issue reports that `image_generate` and `video_generate` list/inventory output can be posted as visible Discord channel tool output, exposing provider/model/configuration/auth-hint details.

## Best Possible Solution

Make media provider inventory from `action=list` internal by default on shared chat surfaces, while preserving full inventory for the agent/tool context and preserving generated media attachment delivery. Add focused coverage for image/video inventory suppression and Discord channel dispatch behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to return an `image_generate` or `video_generate` list result with `details.providers` while `shouldEmitToolOutput()` is false; current main still calls `emitToolOutput`, and Discord guild messages are `ChatType: "channel"` so the shared dispatcher can deliver the text.

## Solution Assessment

Yes, the requested fix direction is the right class of solution. The narrow maintainable fix is to remove or surface-gate the compact inventory bypass before external chat delivery, not to change provider list generation or generated-media delivery.
