## Summary

Current main still does not implement the three native capabilities requested here: batched file/search tool calls, AI-callable named context block save/load/unload/list tools, or submitted-plan step enforcement. There is adjacent groundwork in strict-agentic planning retries, `update_plan`, context-engine compaction, `/context`, `/compact`, and provider parallel-tool-call support, but those are not equivalent to the requested product/API surface.

## Best Possible Solution

Keep this open and split it into maintainer-reviewed tracks: one for native or gateway-level batching of file/search tools, one for an explicit context-block API if AI-controlled save/load/unload/list semantics are a product goal, and one for runner-owned opt-in plan state if strict step gating is desired. Treat `parallel_tool_calls`, `update_plan`, strict-agentic retries, context-engine compaction, `/context`, and `/compact` as groundwork rather than closure.
