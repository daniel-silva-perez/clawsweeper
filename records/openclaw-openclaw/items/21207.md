## Summary

Keep open. Current main still lacks the three native capabilities requested here: batched file/search tool inputs, AI-callable named context block save/load/unload/list tools, and submitted-plan step enforcement. Existing `parallel_tool_calls`, `update_plan`, strict-agentic plan-only retries, context-engine compaction, `/context`, and `/compact` are adjacent groundwork, but they do not satisfy the requested product/API surface.

## Best Possible Solution

Keep this open and split it into maintainer-reviewed tracks: one for native/gateway-level batching of file and search tools, one for an explicit context-block API if AI-controlled save/load/unload/list semantics are a product goal, and one for runner-owned opt-in plan state if strict submitted-plan gating is desired. Treat `parallel_tool_calls`, `update_plan`, strict-agentic retries, context-engine compaction, `/context`, and `/compact` as groundwork rather than closure.
