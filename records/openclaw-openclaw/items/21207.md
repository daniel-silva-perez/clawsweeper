## Summary

Keep open. Current main has adjacent groundwork, including provider `parallel_tool_calls`, context-engine lifecycle/compaction hooks, `update_plan`, and strict-agentic plan-only retry handling, but it still does not implement the requested native batched file/search APIs, AI-callable named context block save/load/unload/list tools, or submitted-plan step enforcement API.

## Best Possible Solution

Keep this open and split it into concrete maintainer-reviewed tracks: native or gateway-level batching for file/search tools, an explicit AI-callable named context block API if save/load/unload semantics are a product goal, and runner-owned opt-in plan state with strict step gating if enforceable sequential execution is desired. Existing `parallel_tool_calls`, context-engine compaction/pruning, `/context` inspection, `update_plan`, and strict-agentic retries should be treated as groundwork rather than closure.
