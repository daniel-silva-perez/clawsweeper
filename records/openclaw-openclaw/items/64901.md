## Summary

Keep this PR open. Current main still does not implement the proposed warning for context windows resolved from the generic default, and the PR remains incomplete for the Bedrock fallback path that motivated the discussion; meanwhile #65030 is an open, broader provenance-based follow-up that overlaps the remaining design space.

## What This Changes

The PR adds a `shouldWarnDefault` flag to the context-window guard, logs a warning from runtime setup when the guard source is `default`, and extends the existing context-window guard tests.

## Best Possible Solution

Keep one coherent fix path: either update this PR to use source-aware provenance for hardcoded provider fallbacks, or close it in favor of the broader #65030 approach after maintainer review. The end state should warn only when OpenClaw is using a hardcoded fallback value, while preserving known Bedrock context-window metadata and avoiding noisy warnings for real model or user-configured values.
