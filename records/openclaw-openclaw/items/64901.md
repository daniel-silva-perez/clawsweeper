## Summary

Keep this PR open for maintainer review. Current main still lacks the proposed default-warning signal, but the patch does not cover the Bedrock fallback scenario named in the PR because Bedrock discovery still materializes the hardcoded fallback as ordinary model metadata; the overlapping open #65030 is the more complete provenance path but has not merged.

## What This Changes

The PR adds a `shouldWarnDefault` flag to the context-window guard, logs a runtime warning when the guard source is `default`, and extends the existing guard tests.

## Best Possible Solution

Keep one coherent fix path: warn only when OpenClaw is using a hardcoded fallback value, preserve known Bedrock context-window metadata and user-configured `defaultContextWindow`, and avoid noisy warnings for real model or explicit config values. That can be done by updating this PR with provenance-aware fallback handling or by closing it after a maintainer accepts the broader #65030 approach.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Track Bedrock fallback provenance before warning:** `src/agents/context-window-guard.ts:80`

Carry provenance from discovery or otherwise mark those fallback values as defaults before relying on this guard.
