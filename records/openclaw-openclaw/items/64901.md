## Summary

Keep open for maintainer review. The PR is focused and useful for the generic `source: "default"` path, but it does not warn for the Bedrock hardcoded fallback named as the root cause, and the related open #65030 takes a broader provenance-based approach that overlaps this surface.

## What This Changes

The PR adds a `shouldWarnDefault` flag to the context-window guard, logs a runtime warning when that flag is set, and extends guard tests for default-source behavior.

## Best Possible Solution

Land one coherent provenance-aware path: keep known Bedrock context-window metadata as model-owned, mark only literal hardcoded fallback values as default-derived, and emit the warning from runtime based on that provenance while preserving user-configured overrides.

## Reproduction Assessment

Yes. The current-main check is to discover an unknown Bedrock model or inference profile, observe `contextWindow: 32000` from `extensions/amazon-bedrock/discovery.test.ts`, then pass that value into runtime resolution and see it classify as `source: "model"` rather than `source: "default"`.

## Solution Assessment

No. The proposed `source === "default"` flag is a reasonable generic signal, but it is not the narrowest complete fix for the stated Bedrock hardcoded fallback because that fallback is already represented as model metadata before runtime sees it.

## Review Findings

- **[P2] Detect Bedrock fallback provenance before warning:** `src/agents/context-window-guard.ts:80`
