## Summary

Keep open. Current main has newer cautious stale-session recovery, but it does not implement this PR's configurable abort threshold or `session.aborted` behavior. The PR addresses a real gateway queue blockage, but the patch should not merge as-is because the new abort path does not release the active lane slot and the abort threshold is silently gated by the warning threshold.

## What This Changes

The PR adds a `diagnostics.stuckSessionAbortMs` config/schema/help surface and extends the diagnostics heartbeat to clear a session lane, reset diagnostic state, log, emit `session.aborted`, and cover threshold/reload behavior with tests.

## Best Possible Solution

Implement configurable stuck-session recovery through the existing recovery runtime: resolve the abort threshold independently, use the session-lane resolver, abort or reset active work through the established lane/embedded-run primitives, make queued-message handling intentional, emit stable diagnostics, and add heartbeat integration coverage plus config docs/changelog as needed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Release the active lane before marking the session idle:** `src/logging/diagnostic.ts:678-682`
  - body: A stuck session blocks later messages because its lane has an active task. `clearCommandLane` only rejects queued entries; it leaves the active task slot in `activeTaskIds`, so with `maxConcurrent=1` the lane remains blocked while diagnostics now reports the session as idle. Route this through the existing abort/reset recovery path before resetting state.
  - confidence: 0.91
- **[P2] Check the abort threshold outside the warn gate:** `src/logging/diagnostic.ts:676`
  - body: The abort branch is nested under `ageMs > stuckSessionWarnMs`, so a valid configured abort threshold below the warn threshold cannot fire at the configured time. Either enforce `abort >= warn` during resolution or evaluate the abort condition independently from warning emission.
  - confidence: 0.86
