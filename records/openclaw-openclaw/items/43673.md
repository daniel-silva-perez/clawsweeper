## Summary

Keep open. Current main has strong adjacent primitives for isolated agents, per-agent workspaces/auth stores, per-agent tool policy, gateway operator method scopes, and channel/group `toolsBySender` restrictions, but it still does not implement the central org/team deployment request: no team workspace scaffold command, no declarative `agents.yaml`-style deployment manifest/importer, and no complete session/operator-scope RBAC bridge. The related RBAC items remain narrower building blocks, with #49183 open and #53496/#53530 still open and unmerged.

## What This Changes

The issue requests first-class multi-user org/team deployment support: team workspace scaffolding, session/tool RBAC, declarative deployment manifests, and official org deployment documentation.

## Best Possible Solution

Keep this open as the umbrella product/design tracker unless maintainers split it. The durable end state should define a supported org deployment model, decide whether workspace scaffolding and deployment manifests belong in core CLI/config or a plugin workflow, and land a clear authorization bridge between gateway/operator/session identity and agent tool policy before documenting it as a safe team pattern.
