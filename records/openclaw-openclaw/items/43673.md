## Summary

Keep open. Current main has adjacent multi-agent workspaces, per-agent tool policy, sandbox controls, and group/sender tool overrides, but it still lacks the requested team scaffold command, declarative deployment manifest, and operator/session-scope RBAC. The linked policy work is useful incremental context, not a complete implementation of the broader org/team deployment request.

## What This Changes

Add first-class org/team deployment support with workspace scaffolding, per-session or RBAC-style tool policies, deployment manifests, and documented org deployment patterns.

## Best Possible Solution

Keep this as a maintainer-scoped umbrella or convert it into a design/RFC, then split the agreed plan into narrower implementation issues for operator/session RBAC, workspace/team scaffolding, deployment manifest schema, and org deployment documentation.
