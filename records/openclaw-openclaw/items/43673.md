## Summary

Keep open. Current main has adjacent multi-agent workspaces, per-agent tool/sandbox policy, and group/sender tool overrides, but it still lacks the requested team scaffold command, declarative deployment manifest, and operator/session-scope RBAC. The related RFC and PR context covers narrower policy slices rather than the full org/team deployment request.

## What This Changes

Requests first-class org/team deployment support through a team workspace scaffold CLI, per-session/RBAC tool policies, declarative agent deployment manifests, and official org deployment documentation.

## Best Possible Solution

Keep this as a maintainer-scoped umbrella or convert it into a design/RFC, then split the agreed plan into narrower implementation issues for operator/session RBAC, workspace/team scaffolding, deployment manifest schema, and org deployment documentation. The linked policy work can land or be replaced independently as the RBAC slice matures.
