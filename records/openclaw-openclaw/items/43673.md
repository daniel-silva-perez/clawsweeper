## Summary

Keep open. Current main has useful adjacent primitives for isolated agents, per-agent workspaces/auth/session stores, per-agent tool restrictions, gateway operator scopes, and group/sender-scoped channel tool policies. It still does not implement the central first-class org/team deployment request: no team workspace scaffold CLI, no declarative agents.yaml-style deployment manifest/importer, and no complete per-session/operator-scope RBAC layer. Related #49183, #53496, and #53530 remain narrower RBAC building blocks, not a replacement for this umbrella request.

## Best Possible Solution

Keep this open as the umbrella product/design tracker unless maintainers split it. The concrete path is to review or close the focused RBAC work in #49183, #53496, and #53530, then decide whether team workspace scaffolding and deployment manifests belong in core CLI/config, a documented plugin/ClawHub pattern, or a smaller core API that enables external implementation.
