## Summary

Keep open. Current main has adjacent multi-agent workspaces, per-agent tool policy, group/sender-scoped tool restrictions, and Gateway operator scopes, but it still lacks the requested first-class team scaffold command, declarative deployment manifest, and operator/session-scoped RBAC model. The linked RFC and PRs are useful narrower slices, not a complete replacement for this umbrella.

## What This Changes

Request first-class org/team deployment support covering multi-agent workspace scaffolding, per-session or role-based tool policies, declarative deployment manifests, and org deployment documentation.

## Best Possible Solution

Keep this as a maintainer-scoped umbrella or convert it into an RFC, then split accepted work into narrower issues for RBAC semantics, CLI/team scaffolding, deployment manifest schema, and org deployment documentation. The existing group/sender policy plumbing and open direct-session policy PR can be evaluated as incremental building blocks.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is source and docs inspection of CLI surfaces, config types, tool-policy resolution, group policy helpers, and Gateway security docs, which shows adjacent support but not the requested first-class org/team deployment model.

## Solution Assessment

Unclear that one implementation issue is the best way to solve this. The maintainable path is explicit maintainer product/security design followed by smaller PRs, especially for the RBAC subset that must align with Gateway operator-scope semantics.
