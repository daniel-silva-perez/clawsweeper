## Summary

Keep open. Current main has strong adjacent pieces for isolated agents, per-agent workspaces/auth stores, routing bindings, per-agent sandbox/tool policy, gateway operator scopes, and channel/group `toolsBySender`; it still does not implement the central org/team request: no team workspace scaffold command, no declarative deployment manifest/importer, and no complete session/operator-scope RBAC bridge into agent tool policy. The focused RBAC follow-up #49183 remains open, and the narrower Feishu/direct-session PRs #53496 and #53530 are still open and unmerged.

## What This Changes

Requests first-class org/team deployment support through workspace scaffolding, per-session RBAC/tool policies, declarative deployment manifests, and documented org deployment guidance.

## Best Possible Solution

Keep this as an umbrella product/security tracker unless maintainers split it. The durable end state should define the supported org deployment model, decide which parts belong in core CLI/config versus plugin workflow, and land a clear authorization bridge from gateway/operator/session identity into agent tool policy before documenting mixed-role team deployments as safe.
