## Summary

Keep open. Current main has adjacent multi-agent workspaces, per-agent tool policy, sandbox isolation, and some sender/group policy plumbing, but it does not provide the requested first-class team scaffold command, declarative deployment manifest, or operator/session-scoped RBAC model. The remaining work is broad and security-sensitive enough to need maintainer product/security scoping before implementation.

## What This Changes

The issue requests first-class org/team deployment support through workspace scaffolding, per-session tool policies, declarative deployment manifests, and official org deployment documentation.

## Best Possible Solution

Keep this as a maintainer-scoped umbrella or convert it into a design/RFC, then split the accepted plan into narrower implementation issues for operator/session RBAC, workspace/team scaffolding, deployment manifest schema, and org deployment documentation. The linked policy slices can land or be replaced independently as the RBAC design matures.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is source/docs search plus inspection of the CLI, config types, tool-policy resolver, and security docs, which shows the requested first-class org/team surfaces are still missing.

## Solution Assessment

Unclear that a single implementation issue is the best way to solve it. The maintainable path is a design decision followed by smaller PRs, especially because the RBAC portion has to align with Gateway trust semantics rather than only adding config fields.
