## Summary

Current main still persists auto-enable-derived `enabled: true` flags during gateway startup and doctor repair. The older #67557 hash guard mitigates the self-induced restart loop, but the requested runtime-only, non-persistent auto-enable behavior is not implemented and current tests still assert persistence.

## Best Possible Solution

Keep this issue open for maintainer product judgment on config canonicalization. If accepted, implement runtime-only auto-enable for gateway startup and doctor diagnostics, do not persist inferred `enabled` flags, preserve explicit user enable/disable writes and real repair migrations, keep the existing self-write hash guard, and update the startup/doctor regression tests that currently expect persistence.
