## Summary

Keep open. Current `main` still materializes auto-enable-derived `enabled: true` values and persists them from gateway startup and `doctor --fix`; the older restart-loop guard for startup writes is present, but the requested runtime-only/non-persistent auto-enable behavior is not implemented.

## Best Possible Solution

Keep the issue open for maintainer product judgment on config canonicalization. If accepted, add a runtime-only auto-enable path for gateway startup and doctor diagnostics, avoid persisting inferred `enabled` flags, preserve explicit user enable/disable flows, keep the existing self-write hash guard, and add regression coverage for startup and `doctor --fix` not reintroducing the reported keys.
