## Summary

Keep this PR open. Current main still only enqueues generic Telegram emoji reaction system events; it does not implement `channels.telegram.reactionSemantics`, stable `reaction_key` values, Telegram `custom_emoji` normalization, semantic `wake|queue|ignore` actions, or a heartbeat wake from the reaction handler. The PR remains a focused implementation candidate for the still-open #47677 request, and the review discussion shows the author addressed concrete feedback rather than abandoning the work.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #47677. Maintainers should review the Telegram-owned config/runtime design and the public wake seam, then either land a refined version with schema/docs/tests or redirect the remaining work into a narrower canonical issue if the API shape needs redesign.
