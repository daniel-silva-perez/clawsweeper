The PR changes the shared CLI watchdog defaults in `src/agents/cli-watchdog-defaults.ts`, raising the fresh minimum to 300000 ms and both fresh and resume maximums to 1800000 ms.

## Best Possible Solution

Land this or a successor with maintainer-approved CLI watchdog defaults, focused tests for fresh and resumed timeout resolution plus the global-timeout clamp, and a clear split from the separate Monitor-aware expected-idle follow-up.
