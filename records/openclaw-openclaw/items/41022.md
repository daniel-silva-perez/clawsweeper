## Summary

Keep this PR open. Current main still has the old CLI no-output watchdog defaults and still routes the resolved timeout into the CLI process supervisor, so the PR is not obsolete; however, the branch only changes constants and does not update an existing test that still asserts the old resume max, so it is not merge-ready as-is.

## What This Changes

The PR raises CLI watchdog defaults in `src/agents/cli-watchdog-defaults.ts`, changing the fresh minimum from 180s to 300s and both fresh/resume maximums to 1800s.

## Best Possible Solution

Keep this PR open, update it with focused tests for fresh/resume default resolution and the global-timeout clamp, then have maintainers decide whether the 5-minute fresh floor and 30-minute caps are the right defaults; leave disable-cap and Monitor-aware idle handling to the linked follow-up scope.
