## Summary

Keep this PR open. Current main still has the old CLI no-output watchdog defaults this PR targets, the resolution path still clamps resumed runs through the 180000 ms default max, and the resulting timeout is still enforced by the CLI runner/process supervisor. The PR remains a narrow implementation candidate rather than obsolete.

## Best Possible Solution

Keep the PR open as a plausible implementation candidate. Rebase or reapply the constants change on current main, add a focused unit test for default fresh/resume no-output timeout resolution plus the global-timeout clamp, then review and land it if maintainers accept the longer default wait. Keep the optional null/0 disable-cap behavior as a separate product decision or follow-up issue.
