## Summary

Keep open. Current main already has configured Telegram proxy support and standard HTTP(S) proxy env routing for Telegram, but it still does not implement the PR's Telegram-specific OPENCLAW_TELEGRAM_PROXY override. The PR also targets the removed src/telegram layout and review discussion flagged malformed proxy handling that should be addressed in a refreshed patch.

## Best Possible Solution

Keep this PR open for a narrow refresh against extensions/telegram. If maintainers want the override, implement a Telegram-owned resolver with precedence config/per-account proxy > OPENCLAW_TELEGRAM_PROXY > current standard HTTP(S) env-proxy behavior, then thread it through send, monitor/webhook, probe, audit, and chat lookup paths with focused tests for precedence, blank values, malformed values, NO_PROXY behavior, and redaction. If configured/global proxy support is considered sufficient, close only after that product decision is explicit.
