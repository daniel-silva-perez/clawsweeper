## Summary

Keep this issue open. Current main still routes task registry and Task Flow registry persistence through `node:sqlite` with no availability probe, JSON fallback, or supported replacement dependency; only the scheduled sweep rejection leak appears mitigated, so Homebrew-style runtimes without `node:sqlite` can still fail durable task restore/persistence.

## What This Changes

The issue requests a runtime-safe persistence backend for Homebrew Node.js builds where `node:sqlite` is unavailable, so Gateway task maintenance no longer crashes or loses durable task state.

## Best Possible Solution

Add a maintainer-approved runtime-safe storage selection layer for both registries: probe `node:sqlite` before default use, keep SQLite where available, provide a durable and hardened fallback or supported dependency where unavailable, update the public storage docs, and cover restore, upsert/update, stale-task maintenance, and Task Flow persistence in regression tests.
