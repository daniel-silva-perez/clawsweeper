## Summary

Keep this issue open. Current main still uses `node:sqlite` directly for task registry and Task Flow registry persistence, with no runtime availability probe, JSON fallback, or replacement dependency; the previously linked fallback PR is closed and unmerged, so the reported Homebrew runtime failure remains unresolved.

## What This Changes

The issue requests preventing gateway task-registry crashes on Node.js builds where the built-in `node:sqlite` module is unavailable, such as the reported Homebrew Node.js runtime.

## Best Possible Solution

Add a maintainer-approved runtime-safe storage selection layer for both task registries: probe `node:sqlite` before default use, keep SQLite where available, provide a documented durable fallback or supported dependency where unavailable, and cover restore, upsert/update, stale-task maintenance, and Task Flow persistence in regression tests.
