## Summary

Keep this PR open. Current main still does not preserve Discord card/component text when ordinary button or select fallback events are sent to the agent. The proposed fix addresses the right bug, but it targets the removed `src/discord` layout and includes a core prompt special-case that should now live through the Discord plugin hint path. Security review found no supply-chain-sensitive file changes in the provided diff.

## Best Possible Solution

Keep this PR open and ask for a rebase/port into the current Discord plugin. The best patch would add `cardText` or an equivalent structured context field to `extensions/discord/src/components.types.ts`, collect `spec.text`, fallback text, and visible text/section block content in `extensions/discord/src/components.ts`, persist it through the component registry, include it in ordinary fallback event text from `extensions/discord/src/monitor/agent-components.ts` only where it does not break explicit callbackData semantics, and add focused tests in `extensions/discord/src/components.test.ts` and `extensions/discord/src/monitor/monitor.test.ts`. Prompt guidance should use the existing Discord plugin `messageToolHints` path rather than core Discord special-casing.
