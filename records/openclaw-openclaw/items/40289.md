## Summary

Keep this PR open. The underlying Discord bug is still present on current main: component card text is rendered into the Discord message but not persisted on component entries or included when fallback button/select interactions are dispatched to the agent. The PR is useful but stale because its Discord hunks target the removed `src/discord` layout instead of the current `extensions/discord` plugin. Security review found no supply-chain-sensitive changes in the provided diff.

## Best Possible Solution

Keep this PR open and ask for a rebase/port into the current Discord plugin. The best patch would add `cardText` or an equivalent structured context field to `DiscordComponentEntry`, collect `spec.text`, fallback text, and visible text/section block content in `extensions/discord/src/components.ts`, preserve it through the component registry, include it only in ordinary fallback event text from `extensions/discord/src/monitor/agent-components.ts`, and add regression coverage in `extensions/discord/src/components.test.ts` and `extensions/discord/src/monitor/monitor.test.ts`. Prompt guidance should use the current Discord plugin `messageToolHints` path rather than core Discord special-casing.
