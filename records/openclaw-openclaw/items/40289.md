## Summary

Keep this PR open. Current main still does not carry Discord component/card text into fallback button or select interaction turns, but this PR is not merge-ready because it changes the removed `src/discord` implementation path and adds Discord-specific prompt guidance in core instead of using the current Discord plugin hint surface.

## What This Changes

This PR stores Discord card text on component entries, appends it to synthesized button/select fallback event text, and adds a Discord-specific Components v2 message-tool hint in the agent system prompt.

## Best Possible Solution

Port the useful behavior into the current Discord plugin: collect `spec.text`, fallback text, and visible text/section block content while building component messages; store that context on component entries; append it only to synthesized fallback button/select event text when no plugin handler or explicit button callback payload handles the click; keep Discord-specific prompt guidance in `extensions/discord/src/channel.ts`; and add focused regression coverage for both buttons and selects.
