## Summary

Keep open. The underlying Discord card-context bug still exists on current main, but this PR cannot merge as written because it edits the removed `src/discord` path and puts Discord-specific prompt guidance in core instead of the active Discord plugin.

## What This Changes

The PR adds card text storage to Discord component entries, appends that text to synthesized component interaction fallback text, and adds Discord Components v2 guidance to core message-prompt assembly.

## Best Possible Solution

Port the useful behavior into `extensions/discord`: collect visible card text while building component entries, persist it on button/select entries, append it only to synthesized fallback event text, preserve explicit `callbackData` and handled plugin-dispatch semantics, and keep Discord prompt wording in `messageToolHints`.

## Reproduction Assessment

Yes. Current-main source and tests provide a high-confidence reproduction path: active Discord button/select fallback dispatch still produces context-free `BodyForAgent` strings, and tests assert those exact strings.

## Solution Assessment

No. The intended fix is valid, but this implementation targets removed files and crosses the core/plugin boundary; the narrow maintainable solution is a plugin-local port with focused Discord regression tests.
