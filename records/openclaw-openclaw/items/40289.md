## Summary

Keep this PR open, but the submitted patch is not mergeable as-is. The requested Discord card-context behavior is still missing in the active plugin on current main, while the PR edits the obsolete `src/discord` tree and adds Discord-specific prompt wording in core instead of the plugin hook.

## What This Changes

The PR stores Discord card text on component entries, appends it to synthesized button/select interaction text, and adds Discord Components v2 prompt guidance.

## Best Possible Solution

Move the useful behavior into the active Discord plugin: capture visible component-card text while building entries, store it with button/select entries, append it to synthesized fallback interaction text when no explicit button callback fallback overrides the body, keep Discord prompt wording in `messageToolHints`, and cover button plus select regressions.
