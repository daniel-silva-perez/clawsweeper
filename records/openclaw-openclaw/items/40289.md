## Summary

Keep open. The PR addresses a real Discord button/select context gap, but the branch changes obsolete pre-migration paths and hard-codes Discord prompt wording in core; current main still routes active Discord component interactions through the plugin without preserving surrounding card text.

## What This Changes

The PR stores Discord card text on component entries, appends it to synthesized button/select interaction text, and adds Discord-specific Components v2 message-tool prompt guidance.

## Best Possible Solution

Move the useful behavior into the active Discord plugin: collect visible component-card text while building entries, store it with button/select entries, append it to synthesized fallback interaction text when no explicit button callback payload overrides the body, keep Discord prompt wording in `messageToolHints`, and cover button plus select regressions.
