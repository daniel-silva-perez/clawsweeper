## Summary

Keep this PR open. The requested Discord button/select context bug still appears present on current main, but the proposed patch is not merge-ready because it targets the removed `src/discord` implementation and puts Discord-specific prompt guidance in core instead of the active Discord plugin surface.

## What This Changes

This PR stores Discord card text on component entries, appends it to synthesized Discord button/select fallback event text, and adds a Discord-specific Components v2 message-tool hint to the agent system prompt.

## Best Possible Solution

Port the useful behavior into the current Discord plugin: capture visible card text while building component messages, persist it with component entries, append it only to synthesized fallback button/select event text, keep Discord-specific prompt guidance in `extensions/discord/src/channel.ts`, and cover buttons plus selects with focused regressions.
