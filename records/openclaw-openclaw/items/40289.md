## Summary

Keep open. The reported missing-context behavior still appears valid on current main, but this PR cannot merge as-is because it edits the retired `src/discord` path and adds Discord-specific prompt guidance in core instead of the active Discord plugin surface.

## What This Changes

The PR stores card text on Discord component entries, appends it to synthesized Discord button/select event text, and adds Discord Components v2 prompt guidance to system prompt assembly.

## Best Possible Solution

Port the useful behavior into `extensions/discord`: collect visible component text while building entries, store it on button/select entries, append it only to synthesized fallback event text, preserve explicit `callbackData` and handled plugin-dispatch semantics, keep Discord prompt wording in `messageToolHints`, and add focused regression coverage plus a changelog credit.

## Reproduction Assessment

Yes. Current-main source and tests provide a high-confidence reproduction path: a Discord component click without handled plugin dispatch or explicit button `callbackData` synthesizes `BodyForAgent` from `formatDiscordComponentEventText`, and existing tests assert context-free strings.

## Solution Assessment

No. The intended fix is valid, but this implementation targets removed/stale files and crosses the core/plugin boundary; the maintainable solution is a plugin-local port in the active `extensions/discord` implementation.
