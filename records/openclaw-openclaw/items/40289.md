## Summary

Keep open. Current main still lacks the requested Discord card-context behavior, but this PR is not mergeable as written because it edits removed `src/discord` files while active Discord component handling now lives in `extensions/discord`; the prompt change also crosses a channel/plugin ownership boundary.

## What This Changes

The PR carries Discord card/embed text on component entries, appends it to synthesized component interaction text, and adds Discord-specific Components v2 guidance to core system-prompt assembly.

## Best Possible Solution

Port the useful behavior into `extensions/discord`: collect visible card text when building component entries, persist it on button/select entries, include it in synthesized fallback event text only where it does not override explicit callback/plugin dispatch semantics, and keep Discord prompt guidance in the plugin’s `messageToolHints` with focused regression tests.

## Reproduction Assessment

Yes. Current main has a source-backed reproduction path: active Discord button/select fallback dispatch still produces plain `Clicked "Approve".` and `Selected Alpha from "Pick".` bodies, and tests assert those context-free strings.

## Solution Assessment

No. The intended fix is valid, but this implementation targets removed files and adds channel-specific guidance in core; the maintainable solution is a narrow port to the active Discord plugin plus plugin-owned prompt hints.
