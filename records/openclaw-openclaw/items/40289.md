## Summary

Keep this PR open, but do not merge it as-is. The requested Discord interaction context is still a real gap on current main, while the branch changes removed `src/discord` files and adds Discord-specific prompt behavior in core instead of the active `extensions/discord` plugin hook.

## What This Changes

The PR stores Discord component card text on component entries, appends it to synthesized button/select fallback text, and adds Discord-specific message-tool prompt guidance.

## Best Possible Solution

Port the useful card-context behavior into the active Discord plugin: collect visible component text while building entries, store it on button/select entries, append it only to synthesized fallback event text, preserve explicit button `callbackData` fallback behavior, and keep Discord prompt wording in `messageToolHints`.

## Reproduction Assessment

Yes. Current main has a high-confidence source and test reproduction: component clicks still dispatch plain fallback bodies such as `Clicked "Approve".` and select fallbacks such as `Selected Alpha from "Pick".`, with no card context stored or passed through.

## Solution Assessment

No. The intended behavior is right, but this diff is not the best way to solve it because it targets removed Discord files and adds channel-specific prompt logic in core; the safer fix is a focused port to `extensions/discord` plus plugin-owned prompt hints.
