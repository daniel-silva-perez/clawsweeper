## Summary

Keep this PR open. The reported Discord button context gap still exists on current main, but the branch edits obsolete `src/discord` files and adds Discord-specific prompt text in core while the active implementation now lives in `extensions/discord`. The useful behavior should be ported to the plugin-owned component path before merge.

## What This Changes

The PR stores Discord card text on component entries, appends that text to synthesized button/select fallback text, and adds Discord-specific inline-component guidance to core system-prompt assembly.

## Best Possible Solution

Port the card-context behavior into `extensions/discord`: collect visible card text while building component entries, store it on button/select entries, append it to synthesized fallback event text only when appropriate, and keep Discord prompt guidance in plugin `messageToolHints` with focused button/select tests.

## Solution Assessment

No. The intended behavior is right, but this diff is not the best implementation because it targets removed Discord files and puts channel-specific prompt logic in core; the maintainable fix is a focused port to `extensions/discord` plus plugin-owned prompt hints.
