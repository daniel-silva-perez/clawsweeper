## Summary

Current main still lacks the requested native `announceTarget` option for `sessions_spawn` and `agents.defaults.subagents`. The existing implementation only gives nested requester subagents internal `deliver=false` injection; top-level requester completions still route through channel/direct delivery by default. Related #33478 and #8299 are complementary, not replacements.

## Best Possible Solution

Keep this open as a core subagent announce-routing policy request. The best fix is a backwards-compatible `announceTarget: "channel" | "parent"` policy on both `sessions_spawn` and `agents.defaults.subagents`, defaulting to `"channel"`, with parent-targeted native subagent completion injected into the requester session with `deliver=false` and no direct channel announce. Update config/schema/docs/changelog and add regression tests for channel default, top-level parent routing, nested requester behavior, active/dormant requester paths, and ACP `streamTo` separation.

## Clownfish Work Prompt

Implement native subagent announce routing for this issue. Add a backwards-compatible `announceTarget` policy with allowed values `"channel"` and `"parent"`, defaulting to `"channel"`, supported both as a `sessions_spawn` parameter and as `agents.defaults.subagents.announceTarget`. Keep ACP `streamTo` behavior unchanged and ACP-only.

Expected behavior: for native `runtime="subagent"` completion announce with `announceTarget: "parent"`, inject the completion into the requester session using the existing internal/inter-session path with `deliver=false` so the parent agent wakes and decides what to send. Do not directly deliver the subagent announce to the channel in that mode. Preserve current channel delivery for omitted/default `announceTarget` and for config default `"channel"`. Preserve current nested subagent behavior and fallback handling unless explicitly covered by the new policy.

Likely implementation boundary: update the `sessions_spawn` schema/execution path, spawn/run registry or lifecycle metadata as needed to carry the chosen policy to `runSubagentAnnounceFlow`/`deliverSubagentAnnouncement`, update strict config types/zod/generated schema/help, update `docs/tools/subagents.md`, and add a user-facing changelog entry crediting @good-warning and relevant reporters if their context is used. Do not rename `streamTo`, do not broaden unsupported channel-delivery params, and do not change default delivery behavior.

Regression coverage should prove: default channel delivery remains `deliver=true`; per-spawn `announceTarget: "parent"` for a top-level requester uses `deliver=false` and wakes the requester session; config-level default applies when the spawn omits the parameter; per-spawn override wins over config; nested requester subagent delivery remains internal; ACP `streamTo` remains ACP-only/ignored for native subagents as currently documented; thread/account route metadata is preserved where needed for session context but not delivered externally in parent mode.
