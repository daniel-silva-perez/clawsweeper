## Summary

Current main still lacks the requested native `announceTarget` option. `sessions_spawn` has no native subagent announce routing parameter, `agents.defaults.subagents` is strict and has no config default, ACP `streamTo` remains separate and explicitly ignored for native subagents, and top-level requester completions still use external delivery while only nested requester subagents get `deliver=false` parent injection.

## Best Possible Solution

Keep this open as a core subagent announce-routing feature request. The best fix is a backwards-compatible `announceTarget: "channel" | "parent"` policy on both `sessions_spawn` and `agents.defaults.subagents`, defaulting to `"channel"`. `"parent"` should inject the completion announce into the requester session with `deliver=false` and no direct channel announce, while preserving current nested behavior and keeping ACP `streamTo` separate. Update config schema/types/generated schema, docs, changelog, and targeted regression tests.
