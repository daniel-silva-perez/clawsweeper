## Summary

Current main still does not implement native `announceTarget` routing for `sessions_spawn` or `agents.defaults.subagents`. The tool schema has no native announce routing parameter, the config schema/types have no default, docs still describe top-level requester completion delivery as external `deliver=true`, and tests still preserve the ACP-only `streamTo` behavior. The issue remains valid and is narrow enough for a focused implementation PR.

## Best Possible Solution

Keep this issue open and implement a focused, backward-compatible native subagent announce-routing feature: add `announceTarget: "channel" | "parent"` to `sessions_spawn` and `agents.defaults.subagents`, default to current channel behavior, route `parent` completions into the requester session with `deliver: false` and no direct channel announce, keep ACP `streamTo` separate, and update schema, generated config docs, user docs, changelog, and targeted tests.
