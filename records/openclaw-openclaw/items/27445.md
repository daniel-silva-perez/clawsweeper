## Summary

Current main still does not implement a native `announceTarget` option for native `sessions_spawn` or `agents.defaults.subagents`. The current tool schema only exposes ACP-only `streamTo`, the config schema is strict without `announceTarget`, and completion delivery still routes top-level requesters through external delivery while only nested subagent/cron requesters get `deliver: false` session injection. A recent maintainer comment already reached the same keep-open conclusion, and related #33478/#8299 are adjacent rather than canonical replacements.

## Best Possible Solution

Keep this issue open and implement a focused, backward-compatible native subagent announce-routing feature: add `announceTarget: "channel" | "parent"` to `sessions_spawn` and `agents.defaults.subagents`, default to current channel behavior, route `parent` completions into the requester session with `deliver: false` and no direct channel announce, keep ACP `streamTo` separate, and update schema/docs/changelog/tests.
