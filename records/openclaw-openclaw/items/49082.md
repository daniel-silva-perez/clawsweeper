## Summary

Keep this PR open. Current main still lacks the Feishu reply-path mediaLocalRoots propagation, while the PR head is a focused implementation candidate that wires the dispatcher and both bot construction paths. Related PRs #51375 and #58591 overlap, but the provided discussion does not identify a maintainer-selected canonical replacement.

## Best Possible Solution

Keep this PR open or consolidate it with a maintainer-chosen canonical Feishu mediaLocalRoots PR. The landed fix should pass getAgentScopedMediaLocalRoots(cfg, agentId) from both Feishu bot dispatcher call sites, add mediaLocalRoots to CreateFeishuReplyDispatcherParams, forward it into sendMediaFeishu, and add narrow regression coverage for single-agent and broadcast reply media paths.
