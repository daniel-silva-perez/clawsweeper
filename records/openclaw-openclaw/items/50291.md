## Summary

Keep open. Current main partially addresses the report by adding runId propagation for legacy before_agent_start/agent_end and a diagnostic trace carrier, but the complete observability context requested here is still missing across plugin hooks. Agent hook contexts still do not expose inbound messageId/senderId or direct parent/current span fields, message_sending lacks run/message correlation on its event and default outbound context, and subagent lifecycle hooks still do not expose requesterRunId for runId-based parent-child trace trees.

## Best Possible Solution

Keep this issue open and implement a focused, additive plugin-hook correlation contract: propagate inbound messageId/senderId into agent run contexts, carry runId through message_sending and remaining lifecycle/compaction hooks when known, expose a stable DiagnosticTraceContext or equivalent traceId/spanId/parentSpanId fields consistently, and add subagent childRunId/requesterRunId coverage plus contract tests for concurrent group messages and nested subagent runs.
