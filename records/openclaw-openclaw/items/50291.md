## Summary

Keep open. Current main has partially addressed the report by adding runId backfill for legacy before_agent_start/agent_end and by exposing message-hook context trace/identity fields, but the requested observability contract is still incomplete: agent hook contexts omit inbound messageId/senderId and direct span fields, message_sending still lacks correlation fields on its event/default dispatch, compaction/session events still lack runId on the event payload, and subagent hooks do not expose requesterRunId for runId-based trace trees.

## Best Possible Solution

Keep this issue open and have maintainers decide the additive plugin-hook correlation contract before implementation. A good path is to split a P0 slice that propagates inbound messageId/currentMessageId and senderId into agent hook contexts, adds run/session/message correlation to message_sending, and exposes childRunId/requesterRunId for subagent lifecycle hooks, then follow with a documented trace/span hierarchy slice that reuses DiagnosticTraceContext consistently across hook families.
