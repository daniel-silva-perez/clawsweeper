## Summary

Keep open. Current main partially addressed the report by adding runId backfill for legacy agent hooks and exposing a DiagnosticTraceContext carrier, but the requested plugin-hook observability contract is still incomplete across agent identity, outbound message correlation, compaction/session run correlation, and subagent parent-run hierarchy.

## What This Changes

Request to add complete trace and correlation context to plugin hooks, including messageId/senderId, runId across hook families, and span or parent-run fields for nested trace construction.

## Best Possible Solution

Keep this issue open and have maintainers decide the additive plugin-hook correlation contract. A good path is to split a P0 slice that propagates inbound messageId/currentMessageId and senderId into agent hook contexts, adds explicit run/session/message correlation to message_sending, and exposes childRunId/requesterRunId for subagent lifecycle hooks; then follow with a documented trace/span hierarchy slice that consistently uses DiagnosticTraceContext or explicit span aliases across hook families.
