## Summary

Keep open. Current main partially addresses the original runId boundary by backfilling runId for before_agent_start and agent_end, but the public hook contract still lacks key correlation fields for agent inbound identity, generic outbound message_sending, and subagent requester-run hierarchy.

## What This Changes

Request to expand plugin hook trace context so observability plugins can correlate agent, message, tool, compaction, and subagent hooks by run, message, sender, and span relationships.

## Best Possible Solution

Define one backwards-compatible plugin-hook correlation contract, then land focused implementation slices for agent inbound identity, outbound delivery correlation, subagent child/requester run IDs, and trace/span aliases, with SDK baseline and plugin docs updated together.

## Reproduction Assessment

Yes. Current-main source inspection reproduces the remaining contract gaps: agent hook contexts omit inbound identity/span alias fields, message_sending lacks event correlation on generic outbound delivery, and subagent hooks lack requesterRunId.

## Solution Assessment

Unclear. The requested direction is valid, but the best implementation needs maintainer agreement on the public SDK shape now that current main already has ctx.trace and partial runId backfill.
