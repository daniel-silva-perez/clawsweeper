## Summary

Keep open. Current main has meaningful partial coverage for run correlation and diagnostic trace transport, but the public plugin-hook observability contract is still incomplete for agent hook identity fields, event-level outbound message correlation, and subagent parent-run hierarchy.

## What This Changes

The issue requests a fuller plugin-hook trace context contract, including runId across hook families, message and sender identity, span parent/child fields, call depth, and child/requester run IDs for subagents.

## Best Possible Solution

Define a backwards-compatible plugin-hook correlation contract, then land focused slices for agent hook inbound identity, outbound message correlation, subagent child/requester run IDs, and trace/span aliases, with SDK baseline and docs updates for plugin authors.
