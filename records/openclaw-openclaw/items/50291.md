## Summary

Keep open. Current main partially addresses the original run-correlation gap by backfilling `runId` onto legacy `before_agent_start` and `agent_end` events and by adding message-hook trace/context plumbing, but the requested observability contract is still incomplete for agent inbound identity, outbound `message_sending` correlation, and subagent parent-run hierarchy.

## What This Changes

The issue requests a complete plugin-hook trace context contract so observability plugins can correlate concurrent group messages, agent/model/tool spans, outbound messages, and subagent runs with `runId`, message/sender identity, and parent span/run fields.

## Best Possible Solution

Define one backwards-compatible plugin-hook correlation contract, then land focused slices for agent inbound identity, outbound delivery correlation, subagent child/requester run IDs, and trace/span aliases, with plugin SDK API baseline and public docs updates for plugin authors.
