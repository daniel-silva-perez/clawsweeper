## Summary

Keep open. Current main partially addresses the original run-correlation problem by backfilling runId onto before_agent_start and agent_end and by adding trace-aware message hook context, but the public plugin hook contract is still incomplete for agent inbound identity, first-class span aliases, generic outbound message_sending correlation, and subagent requester-run hierarchy.

## What This Changes

The issue requests expanded plugin hook event/context payloads so observability plugins can correlate concurrent messages, agent runs, model/tool spans, outbound messages, and subagent parent-child runs.

## Best Possible Solution

Define one backwards-compatible plugin-hook correlation contract, then land focused implementation slices for agent inbound identity, outbound delivery correlation, subagent child/requester run IDs, and trace/span aliases, with SDK baseline and public plugin docs updated together.
