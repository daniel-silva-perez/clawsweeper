## Summary

Keep open. Current main partially addresses the report with legacy `runId` backfill and a diagnostic trace carrier, but the requested plugin-hook observability contract is still incomplete for first-class message/sender identity on agent hooks, event-level outbound message correlation, all hook families, and subagent requester-run hierarchy.

## What This Changes

Requests an additive plugin hook observability contract that carries run, message, sender, and span/parent trace identifiers consistently across agent, message, compaction/session, and subagent hooks.

## Best Possible Solution

Define and document a backwards-compatible plugin-hook correlation contract, then land it in focused slices: propagate inbound message identity into agent hook contexts, add outbound message correlation consistently, expose child/requester run IDs for subagent lifecycle hooks, and align span/trace naming across hook families with contract tests and SDK baseline updates.
