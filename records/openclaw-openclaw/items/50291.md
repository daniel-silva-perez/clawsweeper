## Summary

Keep open. Current main partially addresses the original boundary-runId complaint, but the public plugin hook contract still lacks the full correlation surface requested for agent inbound identity, generic outbound message_sending correlation, and subagent requester-run hierarchy.

## What This Changes

The issue requests richer plugin hook trace context so observability plugins can correlate concurrent messages, agent boundaries, message delivery, and subagent parent/child runs.

## Best Possible Solution

Define one backwards-compatible plugin-hook correlation contract, then land focused implementation slices for agent inbound identity, outbound delivery correlation, subagent child/requester run IDs, and trace/span aliases, with SDK baseline and plugin docs updated together.
