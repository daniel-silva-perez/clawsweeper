## Summary

Keep open. Current main partially addresses run correlation by backfilling runId onto before_agent_start and agent_end and by adding trace-aware message hook context, but the requested public hook contract is still incomplete for agent inbound identity, outbound message_sending correlation, and subagent parent-run hierarchy.

## What This Changes

The issue requests trace and correlation fields such as runId, messageId, senderId, parent span IDs, and subagent parent/child run IDs across plugin hook events and contexts for observability plugins.

## Best Possible Solution

Define one backwards-compatible plugin-hook correlation contract, then implement focused slices for agent inbound identity, outbound delivery correlation, subagent child/requester run IDs, and trace/span aliases, with SDK baseline and public plugin docs updated together.
