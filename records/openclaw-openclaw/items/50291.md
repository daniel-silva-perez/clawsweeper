## Summary

Keep open. Current main has closed part of the original P0 gap by documenting and testing runId backfill for before_agent_start and agent_end, and message hook contexts now expose several correlation fields, but the requested public hook contract is still incomplete for agent inbound identity, first-class agent span aliases/call depth, outbound message_sending correlation, and subagent requester-run hierarchy.

## What This Changes

Add complete trace and correlation data to plugin hook events and contexts, including runId, messageId/senderId, span parent/current identifiers, call depth, and subagent child/requester run IDs.

## Best Possible Solution

Define one backwards-compatible plugin-hook correlation contract, then land focused implementation slices for agent inbound identity, outbound delivery correlation, subagent child/requester run IDs, and trace/span aliases, with SDK baseline and plugin docs updated together.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the remaining gaps: the agent hook context omits inbound identity and first-class span/call-depth aliases, message_sending lacks event correlation fields, and subagent hooks lack requesterRunId despite child run IDs being available.

## Solution Assessment

Unclear. The requested direction is valid, but the best implementation needs maintainer agreement on the public SDK shape now that current main already has ctx.trace and partial runId backfill.
