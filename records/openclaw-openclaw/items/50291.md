## Summary

Keep open. Current main has partial correlation support, including runId backfill for before_agent_start and agent_end plus trace-aware message-hook context, but the issue's core plugin SDK contract is still incomplete for agent inbound identity, first-class span aliases, generic message_sending correlation, and subagent requester-run hierarchy.

## What This Changes

Request to add trace and correlation fields such as runId, messageId, senderId, parentSpanId/currentSpanId/callDepth, and subagent child/requester run IDs across plugin hook events and contexts.

## Best Possible Solution

Define one backwards-compatible plugin-hook correlation contract, then land focused implementation slices for agent inbound identity, outbound delivery correlation, subagent child/requester run IDs, and trace/span aliases, with SDK baseline and public plugin docs updated together.
