## Summary

Keep open. Current main only covers the Opus 4.7 part of the problem; the Sonnet 4.7 and broader future-version behavior described by this PR is still missing in the transport/profile paths, and the PR contains useful review-addressed work. ## What This Changes

The PR replaces hardcoded Anthropic Claude adaptive-thinking checks with forward-compatible Opus/Sonnet 4.6+ patterns, adds Sonnet 4.7 model resolution, and adds regression tests for adaptive thinking and date-stamped model IDs.

## Best Possible Solution

Rebase the PR onto current main and implement the model-family decision in the shared Claude thinking-profile helper plus the Anthropic transport/resolver paths, with focused tests for Sonnet 4.7 and any explicitly supported future-minor behavior.

The old-base patch fixes the right surfaces for its base and addresses bot feedback, but current main moved thinking level/default ownership into src/plugin-sdk/provider-model-shared.ts, so the maintainable solution must update that shared helper and its tests during rebase.
