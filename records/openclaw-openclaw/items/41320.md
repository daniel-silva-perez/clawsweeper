## Summary

Keep open. Current main still does not implement Tirith pre-exec scanning, and this PR remains useful but not mergeable because the gateway path can launch the configurable scanner before the existing effective `security=deny` guard runs. Earlier review feedback about node scan ordering and bounded findings appears addressed, but the remaining issue is a real security-boundary blocker.

## What This Changes

The PR adds optional default-on Tirith pre-exec command scanning for gateway, sandbox, and node exec paths, wires `tools.exec.commandSecurity` through config/runtime surfaces, and adds command-security tests.

## Best Possible Solution

A mergeable version should resolve effective host deny before any scanner process can start, add gateway deny regression coverage, document the operator-facing config and failure model, and get explicit maintainer agreement on whether this belongs in core or behind a trusted plugin/policy surface.
