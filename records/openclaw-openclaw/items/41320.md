## Summary

Keep open. Current main still does not implement Tirith command scanning, so this PR is not obsolete, but the patch should not merge as-is because the gateway scanner runs before effective deny policy is resolved and the branch is currently dirty against main.

## What This Changes

The PR adds default-on optional Tirith pre-exec command scanning across gateway, sandbox, and node exec paths, plus `tools.exec.commandSecurity` config/schema labels and command-security approval tests.

## Best Possible Solution

A mergeable version should preserve effective `security=deny` before any scanner process can start, add regression coverage for denied gateway policy, document the new config and operational model, and get explicit maintainer sign-off on whether this belongs in core or behind a trusted plugin policy surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Honor deny policy before launching the scanner:** `src/agents/bash-tools.exec.ts:422-425`
  - body: With `tools.exec.security` or the host approval file resolving to `deny`, gateway exec currently rejects in `resolveExecHostApprovalContext`. This new block calls `checkCommandSecurity` before the gateway path reaches that denial, so denied gateway exec can still spawn the configured/PATH scanner binary. Move the scan after effective deny resolution or resolve deny before the scan, and add a gateway `security=deny` regression.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete security-boundary regression because denied gateway exec can still launch the configured scanner binary before existing deny policy is enforced.

Concerns:

- **[high] Scanner runs before gateway deny policy:** `src/agents/bash-tools.exec.ts:422`
  - body: The PR invokes `checkCommandSecurity` in the gateway path before `processGatewayAllowlist` calls the shared approval resolver that rejects `hostSecurity === "deny"`. Because `tirithPath` is configurable and PATH-resolved, this can execute a local binary even when exec policy says the command must be denied.
