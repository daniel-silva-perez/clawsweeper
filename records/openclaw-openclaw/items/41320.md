## Summary

Keep open. The requested Tirith scanner is not implemented on current main, and the PR still introduces a security-boundary regression by launching the configured scanner before gateway deny policy is resolved.

## What This Changes

The PR adds default-on Tirith pre-exec command scanning, a `tools.exec.commandSecurity` config surface, gateway/sandbox/node exec hook points, and command-security plus exec-approval tests.

## Best Possible Solution

A mergeable version should preserve existing deny semantics before any scanner process can start, add regression coverage for denied gateway policy, document the new config and operational model, and get explicit maintainer sign-off on whether this belongs in core or behind the plugin/trusted-policy surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Honor deny policy before launching the scanner:** `src/agents/bash-tools.exec.ts:422-425`
  - body: With `tools.exec.security` or the host approval file resolving to `deny`, gateway exec currently rejects in `resolveExecHostApprovalContext`. This new block calls `checkCommandSecurity` before the gateway path reaches that denial, so a denied exec can still spawn the configured/PATH `tirith` binary. Move the scan after effective deny resolution or skip it when policy is deny, and add a gateway `security=deny` regression.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete security-boundary regression because denied gateway exec can still launch the configured scanner binary before existing deny policy is enforced.

Concerns:

- **[high] Scanner runs before gateway deny policy:** `src/agents/bash-tools.exec.ts:422`
  - body: The PR invokes `checkCommandSecurity` in the gateway path before `processGatewayAllowlist` calls the shared approval resolver that rejects `hostSecurity === "deny"`. Because `tirithPath` is configurable and PATH-resolved, this can execute a local binary even when exec policy says the command must be denied.
