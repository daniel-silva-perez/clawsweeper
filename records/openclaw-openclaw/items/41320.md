## Summary

Keep open. Current main does not implement Tirith command scanning, and this PR is not mergeable yet because the gateway path can launch the configurable scanner before the existing effective `security=deny` policy is resolved. This is security-boundary work that needs an author fix plus maintainer trust-model review.

## What This Changes

The PR adds optional default-on Tirith pre-exec scanning for gateway, sandbox, and node exec paths, wires `tools.exec.commandSecurity` through config/runtime surfaces, and adds command-security tests.

## Best Possible Solution

A mergeable version should resolve effective host deny before any scanner process can start, add gateway `security=deny` regression coverage, document the operator-facing config and failure model, and get explicit maintainer agreement on whether this belongs in core or behind a trusted plugin/policy surface.
