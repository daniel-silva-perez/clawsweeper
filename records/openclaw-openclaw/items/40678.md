## Summary

Keep this open. Current main still lacks an opt-in TUI observe or mirror mode: the TUI CLI/docs expose only session-scoped options, `/new` still creates isolated `tui-<uuid>` sessions, live TUI chat/BTW events are filtered to the active session key, and startup/history loading still requests one session at a time. Newer Gateway session preview/subscription APIs provide useful building blocks, but they are not wired into the TUI as a same-agent cross-channel activity view. Related #41548 remains open and broader, so it does not clearly supersede this focused TUI monitoring request.

## What This Changes

Request an opt-in TUI mode or config that observes or mirrors same-agent activity from iMessage, Telegram, cron, and other channels while preserving default `/new` session isolation.

## Best Possible Solution

Add an explicit core TUI/Gateway observe path that preserves default `/new` isolation, aggregates recent same-agent activity through documented operator-read Gateway surfaces, labels channel/session/run origin, and makes read-only versus interactive behavior explicit. Update TUI docs and add focused regression coverage for same-agent different-session events, channel-originated final events, aggregate history/preview loading, and read-only behavior.
