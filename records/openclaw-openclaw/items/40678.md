## Summary

Keep open. Current main still has no opt-in TUI observe or mirror mode: the CLI/runtime options are session-scoped, `/new` still isolates TUI clients with `tui-<uuid>` keys, live chat events are filtered to the active session, and history loading reads one selected session. The Gateway has session subscription primitives that could help, but the TUI does not consume them for a same-agent cross-channel activity view.

## What This Changes

Request an opt-in TUI mode or config that mirrors or observes iMessage, Telegram, cron, and other same-agent activity without undoing `/new` session isolation.

## Best Possible Solution

Add an explicit TUI/Gateway observe path that keeps default `/new` isolation, exposes same-agent activity only through authorized operator-read surfaces, clearly labels channel/session/run origin, separates read-only observation from interactive sending, and updates docs plus focused tests for cross-session same-agent activity and aggregate history/preview loading.
