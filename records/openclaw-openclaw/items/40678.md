## Summary

Keep open. Current main still lacks an opt-in TUI observe or mirror mode: the command surface, runtime options, history loader, and live event handler remain scoped to one active session, while the existing Gateway session-message primitives are not consumed by the TUI. The request is valid but security-sensitive enough to need maintainer design before an automated repair PR.

## What This Changes

The issue requests an opt-in TUI observe or mirror mode that can show same-agent iMessage, Telegram, cron, and other channel activity without undoing isolated `/new` TUI sessions.

## Best Possible Solution

Add an explicit authorized TUI/Gateway observe path that keeps default `/new` isolation, labels channel/session/run origin, separates read-only observation from interactive sending, supports live aggregate events plus recent-history previews, and updates focused TUI, Gateway, and docs coverage.
