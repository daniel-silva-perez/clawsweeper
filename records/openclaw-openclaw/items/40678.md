## Summary

Keep open. Current main still lacks an opt-in TUI observe or mirror mode: the CLI/options/docs remain session-scoped, `/new` still creates isolated `tui-<uuid>` sessions, live TUI events are filtered to the active session, and history loading requests only one session. Gateway session subscription primitives exist, but the TUI does not consume them for a same-agent cross-channel activity view, and the broader related cross-channel sync request remains open.

## What This Changes

Add an opt-in TUI mode that can observe or mirror same-agent iMessage, Telegram, cron, subagent, and other channel activity without undoing isolated `/new` TUI sessions.

## Best Possible Solution

Add an explicit authorized TUI/Gateway observe path that keeps default `/new` isolation, clearly labels channel/session/run origin, separates read-only observation from interactive sending, and covers live aggregate events plus recent-history previews with focused TUI and Gateway tests.
