## Summary

Keep this open. Current main still does not provide an opt-in TUI observe or mirror mode: the CLI and docs expose only session-scoped TUI options, `/new` still creates isolated `tui-<uuid>` sessions, live TUI chat events are filtered to the active session key, and history loading still fetches one session at a time. Gateway session message/subscription APIs look like useful building blocks, but they are not wired into the TUI cross-channel view requested here.

## What This Changes

Request an opt-in TUI observe or mirror mode that shows same-agent activity from other channels and sessions while preserving `/new` session isolation.

## Best Possible Solution

Add an explicit core TUI/Gateway observe path that keeps default `/new` isolation, exposes same-agent activity only through authorized operator-read surfaces, labels channel/session/run origin clearly, and separates read-only observation from interactive sending. The implementation should update the TUI docs and add focused coverage for cross-session same-agent activity, channel-originated final events, aggregate history/preview loading, and read-only behavior.
