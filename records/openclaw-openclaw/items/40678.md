## Summary

Keep this issue open. Current main still has no opt-in TUI observe or mirror mode for same-agent cross-channel activity: the CLI/docs expose no observe/mirror option, the TUI runtime has no mirror state, `/new` still creates isolated `tui-<uuid>` sessions, live TUI chat/BTW events are scoped to the current session key, and history loading requests a single session. Related #41548 overlaps but is broader and remains open, so it does not clearly supersede this focused TUI monitoring workflow.

## Best Possible Solution

Keep this open as a core TUI/Gateway UX feature. Implement an explicit `openclaw tui --observe` or `tui.mirrorChannels` mode that preserves default `/new` isolation, aggregates recent same-agent activity through a deliberate Gateway event/history surface, labels channel and session origin, and defines read-only versus interactive behavior. Add focused tests for same-agent different-session events, channel-originated final/side-result events, aggregate history loading, and read-only behavior; coordinate with #41548 only if maintainers choose it as the canonical umbrella.
