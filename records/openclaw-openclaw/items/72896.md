## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks the proposed Slack retry and failed-delivery accounting behavior, so the work is useful, but the retry classifier misses the wrapped `@slack/web-api` request-error shape and the dispatch changes are stale against current main's shared turn-dispatch path.

## What This Changes

The branch adds bounded Slack `chat.postMessage` retries for selected transient delivery failures, changes Slack reply-delivery accounting to subtract failed queued replies, and adds Slack send plus preview-fallback tests.

## Best Possible Solution

Land a rebased Slack plugin fix that keeps write retries bounded, recognizes wrapped Slack `WebAPIRequestError` transport causes such as `err.original.code`, and adapts failed-visible-reply accounting through the current shared channel-turn dispatch helper with focused regression tests.

## Reproduction Assessment

Yes. The branch includes focused unit-level paths for bare transient retry and swallowed failed final delivery, and current-main inspection confirms those behaviors are not implemented. The remaining review reproduction is dependency-contract-based: real Slack WebClient request failures expose transport details under `original`, not as the top-level retry code.

## Solution Assessment

No, not as-is. The direction is narrow and maintainable, but the classifier should inspect the wrapped Slack request error shape and the accounting change should be rebased onto current main's shared turn-dispatch result path.
