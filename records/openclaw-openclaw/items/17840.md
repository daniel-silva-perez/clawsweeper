## Summary

Keep open. Current main still has passive reaction notification plumbing, not an opt-in mode that wakes the agent on reactions. The issue remains the canonical tracker: the overlapping #56708 was closed as a duplicate, while #73681 is a narrower Google Chat inbound-reaction gap rather than a replacement for this cross-channel wake-mode request.

## Best Possible Solution

Keep this issue open as the canonical product request. Implement a documented opt-in channel/plugin wake mode, for example reactionTriggersTurn: "off" | "own" | "all" | "allowlist", that enqueues the reaction event and requests a scoped heartbeat/agent turn only after existing reaction filters and authorization pass. Update config schemas, docs, and focused channel tests while preserving the passive default.
