## Summary

Keep open. Current main still has passive reaction notification plumbing and no documented opt-in mode that wakes agent turns on reactions. The issue remains the canonical cross-channel tracker; #56708 and #11747 are closed overlaps, while #73681 and #38714 cover narrower Google Chat and Hooks gaps.

## Best Possible Solution

Keep this issue open as the canonical cross-channel product/API request. A maintainer should choose the opt-in contract, such as reactionTriggersTurn: "off" | "own" | "all" | "allowlist" or a narrower per-message awaitReaction flag, then implement it behind the current passive default after existing channel authorization and reaction notification filters pass. The change should update config schemas/generated metadata, channel docs, and focused reaction tests for the bundled reaction-capable channels.
