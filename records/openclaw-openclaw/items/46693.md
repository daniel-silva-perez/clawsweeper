## Summary

Keep this PR open. Current main still drops explicit OpenAI/OpenAI-Codex cache retention before stream options, while the downstream OpenAI Responses transport already knows how to emit `prompt_cache_retention: "24h"` when it receives `cacheRetention: "long"` on direct OpenAI hosts. The PR is directionally useful but needs to be reworked onto the current `prompt-cache-retention.ts` resolver instead of the older file layout.

## What This Changes

The PR extracts cache-retention resolution into a provider-neutral helper, forwards explicit `cacheRetention` for OpenAI and OpenAI-Codex providers without adding defaults, keeps Anthropic behavior, and adds focused unit coverage.

## Best Possible Solution

Add explicit OpenAI/OpenAI-Codex passthrough to the current `src/agents/pi-embedded-runner/prompt-cache-retention.ts` resolver without introducing defaults, preserve the existing Anthropic, Bedrock, custom Anthropic, and Google eligibility rules, and add regression tests for config-to-stream-options plus direct OpenAI Responses payload mapping.
