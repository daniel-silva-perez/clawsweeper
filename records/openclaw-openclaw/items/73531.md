## Summary

Keep open. Current main still follows the duplicated status reconciliation path, and PR #73650 is an open, unmerged implementation candidate linked to this issue, so the issue should stay open for maintainer review or landing of that PR.

## What This Changes

The issue requests eliminating redundant `reconcileInspectableTasks()` work during `openclaw status`, preferably by sharing or caching the reconciled task list within one invocation.

## Best Possible Solution

Review and land PR #73650 if its shared-snapshot approach is correct, or apply an equivalent narrow fix that reconciles tasks once inside `getStatusSummary()` and derives both registry and audit summaries from that same snapshot while preserving zero-argument helper behavior for other call sites.

## Reproduction Assessment

Yes. The source-level path is high-confidence: `getStatusSummary()` calls both summary helpers, and each helper currently invokes `reconcileInspectableTasks()` independently; timing tests were not run because this review is read-only.

## Solution Assessment

No for a module-level process cache as the first fix. A per-status-call shared reconciled snapshot is narrower and avoids cache invalidation and shared mutable-reference hazards while removing the duplicated work.
