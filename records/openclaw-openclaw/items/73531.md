## Summary

Keep open. Current main still follows the reported duplicated reconciliation path, and open PR #73650 is an unmerged implementation candidate with closing syntax for this issue, so the issue should remain open until that PR lands or maintainers choose a different fix.

## What This Changes

Request to remove duplicated task reconciliation in `openclaw status` by sharing or caching the reconciled task list used by registry and audit summaries.

## Best Possible Solution

Review and land PR #73650 if its shared-snapshot approach is correct, or apply an equivalent narrow fix that reconciles tasks once inside `getStatusSummary()` and derives both registry and audit summaries from that snapshot, with focused tests proving the call-count contract.

## Reproduction Assessment

Yes. The duplicate path is reproducible from current main by tracing `getStatusSummary()` through the two helpers; I did not run timing tests because this review is read-only, but the source-level double call is deterministic.

## Solution Assessment

No for the issue's module-level cache as the first implementation. A per-status-call shared reconciled snapshot is the safer maintainable fix because it avoids cache invalidation and mutable-reference hazards while removing the duplicated work reported here.
