## Summary

Keep open. Current main still has the duplicated `openclaw status` task reconciliation path, and PR #73650 is an open, unmerged implementation candidate with closing syntax for this issue, so the issue should remain open until that PR lands or maintainers choose a different fix.

## What This Changes

The issue asks to remove duplicated task reconciliation in `openclaw status` so registry and audit summaries do not independently clone and sort the same task records.

## Best Possible Solution

The maintainable end state is for `getStatusSummary()` to reconcile inspectable tasks once and derive both registry and audit summaries from that same snapshot, with focused tests pinning the one-reconcile call contract and preserving zero-argument helper behavior for other callers.

## Reproduction Assessment

Yes. The duplicate path is reproducible from current main by static source tracing: one `getStatusSummary()` call invokes two helpers, and each helper independently calls `reconcileInspectableTasks()`. I did not rerun a 10K-task timing profile because this review is read-only.

## Solution Assessment

No for the issue's suggested module-level cache as the first fix; a per-status-call shared snapshot is narrower and avoids cache invalidation and mutable-reference risks while removing the duplicated work. PR #73650 appears to target that safer shape but is still unmerged.
