## Summary

Keep this PR open. Current main still does not implement the PR’s intended behavior: `openclaw status` fetches `channels.status`, but the visible Channels table is still built only from read-only channel discovery, with the live gateway payload used only for issue overlays. The PR remains a focused implementation candidate for open #73525, and its provided file list does not show security or supply-chain sensitive side changes.

## Best Possible Solution

Review and land this PR, or an equivalent focused fix, so reachable `channels.status.channelAccounts` augments the visible `openclaw status` Channels rows when read-only discovery has no row. Keep the existing read-only table as the fallback/richer source for existing rows, and retain the helper plus scan-overview regression coverage tied to #73525.
