## Summary

Keep this PR open. Current main still does not implement the intended behavior: `openclaw status` fetches `channels.status`, but the visible Channels table is still built from `buildChannelsTable` without merging live `channelAccounts`. The PR remains a focused implementation candidate for the open regression tracked in #73525, and its diff does not show unrelated security or supply-chain sensitive changes.

## Best Possible Solution

Review and land this PR, or an equivalent focused fix, so reachable `channels.status.channelAccounts` augments the visible `openclaw status` Channels rows when read-only discovery has no row. Keep the existing read-only table as the richer source for rows it can discover, preserve the unreachable-gateway fallback, and retain the helper plus scan-overview regression coverage tied to #73525.
