## Summary

Keep this PR open. Current main still does not implement the PR’s intended gateway-snapshot synthesis: `openclaw status` queries `channels.status`, but the visible Channels table is still built from read-only plugin discovery only. The linked bug remains a live implementation candidate, and the provided diff is focused on status display code/tests with no visible CI, dependency, lockfile, secret, or supply-chain sensitive side change.

## Best Possible Solution

Review and land this PR, or an equivalent focused fix, so `openclaw status` augments or populates visible Channels rows from the reachable `channels.status` account snapshot while keeping the current read-only discovery table as the unreachable-gateway fallback. Retain regression coverage proving live Telegram account rows flow into `result.channels.rows`.
