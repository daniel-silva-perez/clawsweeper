## Summary

Keep open. Current main still returns the raw `file-type` MIME from the shared buffer-sniff path, and gateway attachments still prefer a non-generic sniffed MIME over the caller-provided MIME. Since `file-type` 22.0.1 can return `image/apng` for APNG/PNG buffers, the PR remains a plausible narrow fix for open #51881, but it needs rebase/adaptation and changelog cleanup before merge.

## Best Possible Solution

Keep this PR open as an implementation candidate for #51881. The best path is to rebase it onto current main, normalize `image/apng` to `image/png` in the shared buffer-sniff MIME path before gateway attachment MIME selection, keep a focused regression test for APNG sniffing, clean up the changelog attribution/placement, then review and land or close it after an equivalent fix merges.
