## Summary

Keep open. Current main still lacks the PR’s intended normalization: buffer sniffing returns the raw `file-type` MIME, and chat attachments still prefer a non-generic sniffed MIME over the provided one. The PR remains a plausible narrow fix for open issue #51881, but it needs rebase/adaptation and changelog cleanup rather than conservative closure.

## Best Possible Solution

Keep the PR open as an implementation candidate. The best path is to update it against current main, normalize `image/apng` to `image/png` in the shared buffer-sniff MIME path before `detectMime` returns, keep a focused regression test for APNG sniffing, clean up the changelog attribution, and close #51881 after the fix lands.
