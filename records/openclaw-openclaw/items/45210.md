## Summary

Keep open. Current main still has only global session maintenance plus a transient active-key safeguard, so it does not provide durable per-session or per-topic pins that survive normal cleanup. The related retention-rule issue and per-agent override PR overlap the maintenance area, but neither supersedes the explicit protect/pin behavior requested here.

## What This Changes

Add a way to pin or protect selected session keys or channel topics so they are excluded from age and count-based session cleanup.

## Best Possible Solution

Define one documented retention policy that can protect selected session keys, channel topics, or patterns from both age and count cleanup, then align config types, strict schema, cleanup behavior, CLI/docs, and regression tests with that contract while coordinating with the broader retention-rule work.
