## Summary

Keep open. Current main still exposes session maintenance as global retention settings, with only a transient active-session preservation path during cleanup; it does not provide durable per-session or per-topic pins, and preserved keys still count against the configured maxEntries cap. Related #50900 and #50755 overlap with retention policy, but neither fully supersedes the requested explicit session/topic protection.

## What This Changes

The issue requests a per-session or per-topic pin/protect mechanism so selected conversation sessions are excluded from maintenance pruning and max-entry cleanup.

## Best Possible Solution

Add a documented session-retention policy that can protect selected session keys or channel topics from age and count cleanup, with aligned config types, strict schema, cleanup behavior, CLI/docs, and regression tests; coordinate the design with the broader per-pattern and per-agent retention work already being discussed.
