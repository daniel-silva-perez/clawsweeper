## Summary

Keep open. Current main still provides global session maintenance plus a transient active-key safeguard only; it has no durable session/topic pinning config or command-managed pin state, and the related retention-rule issue and per-agent override PR do not implement this issue's explicit protect-from-age/count cleanup behavior.

## What This Changes

Add a way to pin or protect selected session keys or channel topics so they are excluded from `pruneAfter` and `maxEntries` cleanup.

## Best Possible Solution

Define one documented retention-protection contract that can exempt selected sessions, topics, or approved matchers from both stale pruning and count capping, then align config types, strict schema/generated schema, cleanup call sites, CLI/docs, and regression tests with that contract.
