## Summary

Keep open. Current main still exposes session cleanup as global retention settings with only request-scoped active-key preservation, and it does not provide durable per-session or per-topic pins. Related #50900 and #50755 overlap retention policy, but neither supersedes the explicit pin/protect behavior requested here.

## What This Changes

Add a durable mechanism to pin or protect selected session keys or channel topics so they are excluded from age-based pruning and entry-count cleanup.

## Best Possible Solution

Define one documented retention policy that can protect selected session keys or channel topics from both age and count cleanup, then align config types, strict schema, cleanup behavior, CLI/docs, and regression tests with the chosen contract while coordinating with the broader retention-rule work.
