## Summary

Keep open. Current main still has only global session maintenance plus transient active-key preservation; it does not provide durable per-session/topic pins or a documented rule that exempts selected sessions from both age pruning and entry-count capping. The related #50900 is relevant, but it is still open and does not clearly cover explicit pinning or count-cap exemption as the chosen public contract.

## What This Changes

Add a durable per-session/topic pin or protection mechanism so selected session keys can be excluded from session cleanup and max-entry capping.

## Best Possible Solution

Define one maintainer-approved retention-protection contract that can exempt selected session keys, topics, or matchers from stale pruning and entry-count capping, then update config types, strict/generated schemas, cleanup call sites, CLI/docs, and regression tests around that contract.
