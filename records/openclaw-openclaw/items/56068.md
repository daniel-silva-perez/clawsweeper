## Summary

Keep this open. Current main still accepts and exposes `ui.seamColor`, and native app paths consume it, but the browser Control UI bootstrap contract, endpoint payload, bootstrap loader, theme application path, and CSS defaults still do not transport or apply that value. The linked fix PR #56103 is closed unmerged, so the requested web UI behavior remains unresolved.

## What This Changes

This issue asks for `ui.seamColor` in `openclaw.json` to drive browser Control UI accent colors instead of leaving the UI on hardcoded theme defaults.

## Best Possible Solution

Add a narrow Control UI runtime color path: include optional `ui.seamColor` in the authenticated bootstrap config, normalize it on the frontend, derive the related accent CSS variables from it only when configured, and preserve current built-in and imported-theme behavior otherwise. Update docs/changelog if the browser behavior is intentionally shipped.
