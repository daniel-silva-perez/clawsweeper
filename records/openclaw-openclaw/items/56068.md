## Summary

Keep open. Current main still does not wire `ui.seamColor` into the web Control UI: the backend config and `talk.config` path expose it, but the Control UI bootstrap contract/endpoint omit it, the frontend bootstrap consumer does not read it, and the CSS accent variables remain theme-defined/static for the web UI. The related PR #56103 was closed unmerged, so it is not a shipped fix.

## Best Possible Solution

Keep this issue open until maintainers choose the intended contract. If `ui.seamColor` should drive the browser Control UI, add a sanitized color field to the Control UI bootstrap contract/endpoint, apply derived CSS variables for `--accent`, `--accent-hover`, `--accent-subtle`, `--accent-glow`, `--ring`, and `--primary` during frontend bootstrap, and cover the endpoint plus frontend application with focused tests. If the setting is native-only by design, update schema help/labels and docs so the web UI behavior is explicit.
