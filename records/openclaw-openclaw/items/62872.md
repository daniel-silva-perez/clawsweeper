## Summary

Keep open. Current main still runs the normal peer-session `sessions_send` A2A path: the tool schema has no per-call A2A opt-out, the default ping-pong count remains 5, and non-skipped sends still run reply turns followed by the announce step. Merged #69817 is relevant but narrower; it skips parent-owned ACP child loops and explicitly leaves normal persistent/peer sessions on the existing A2A behavior.

## What This Changes

The issue asks OpenClaw to prevent `sessions_send` between persistent sessions from producing duplicate ping-pong and announce-step messages, either through per-call ping-pong control or context-aware A2A defaults.

## Best Possible Solution

Keep this as the canonical tracker for persistent-session duplicate behavior. The desired end state is a context-aware `sessions_send` A2A lifecycle, or explicit per-call A2A control, that prevents unwanted persistent peer follow-up turns while preserving intentional peer announce workflows and the existing parent-owned ACP child skip.
