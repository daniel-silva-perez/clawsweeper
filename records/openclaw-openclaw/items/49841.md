## Summary

Keep this PR open. Current main at 14f140d6f0d31fed802869622c38c7e8f8e19587 still has no Nacos config source, no OPENCLAW_CONFIG_SOURCE/NACOS_* environment surface, and no subscribe-based external gateway reload path. The PR is an unmerged implementation candidate for a real missing capability, with substantive review discussion still attached around Nacos listener correctness, runtime snapshot semantics, CLI behavior, and the security/product boundary for remote gateway config.

## Best Possible Solution

Keep this PR open for maintainer product and security review. If remote config belongs in core, land a generic, tested config-source seam with explicit Nacos support, correct listener/auth/error handling, source/runtime snapshot preservation, clear persistent-versus-in-memory write semantics, CLI and gateway consistency, docs/changelog updates, and targeted or live Nacos validation. If maintainers decide this belongs outside core, first define the missing plugin/config-source API rather than closing this as already possible through today’s plugin surface.
