## Summary

Keep this PR open. Current main still has no Nacos-backed config source, so the PR is not obsolete, but the proposed patch still has blocking correctness issues in gateway startup and Nacos write/reload behavior, plus a remote-config trust boundary that needs maintainer security/product review before merge.

## What This Changes

The PR adds an opt-in Nacos-backed gateway config source with file/Nacos adapters, environment resolution, Nacos HTTP GET/long-poll subscription, gateway reload/startup/CLI/doctor wiring, docs, changelog, and tests.

## Best Possible Solution

If maintainers want remote config in core, land a generic source seam with file as the default and Nacos as an explicit, validated adapter that routes all startup/read/write/reload paths through the active source, preserves source/runtime snapshots, emits correct runtime reload notifications, handles auth/transport/error semantics, documents persistence limits, and has real Nacos validation. If this belongs outside core, define the needed pre-start config-source plugin API first and move the Nacos adapter to plugin space.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use the active source for startup metadata reads:** `src/gateway/server-startup-config.ts:206`
  - body: `loadGatewayStartupConfigSnapshot()` sets the Nacos source, but then calls `readConfigFileSnapshotWithPluginMetadata()`, which still constructs `createConfigIO()` and reads the local file. In Nacos mode the gateway boots from local/default config while later reload wiring subscribes to Nacos, so the configured remote payload is not the startup config.
  - confidence: 0.91
- **[P2] Notify the reloader for Nacos writes:** `src/config/io.ts:2481-2501`
  - body: The Nacos branch updates the runtime snapshot and returns without emitting a config write notification, and subscribe-mode reload does not register `subscribeToWrites`. Successful `config.set`/`config.patch` calls can update `config.get` while hot-reloadable services keep running the old config until a remote Nacos edit or restart.
  - confidence: 0.87

## Security Review
