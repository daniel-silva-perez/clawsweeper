## Summary

Keep this PR open. Current main still does not implement a Nacos-backed config source, remote config subscription, or the proposed env surface, and today’s plugin APIs do not appear to provide a startup config-source seam that could replace core gateway config loading. The PR is also not safe to close as obsolete: it is the active implementation candidate, but it needs maintainer product/security review and more correctness work before merge.

## Best Possible Solution

Keep this PR open for maintainer product review. If remote config belongs in core, the best path is a generic, tested config-source seam with explicit Nacos support, correct Nacos long-poll/auth/error handling, source/runtime snapshot preservation, clear persistent-vs-in-memory write semantics, consistent CLI/gateway behavior, docs/changelog updates, and targeted validation. If maintainers decide this belongs outside core, design the missing plugin/config-source API first rather than closing this as already possible through today’s plugin surface.
