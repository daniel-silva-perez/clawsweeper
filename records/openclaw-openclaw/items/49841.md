## Summary

Keep this PR open. Current main still has no Nacos-backed config source, so the contribution is not obsolete, but the current PR head is not mergeable as-is: startup can still bypass the active source, Nacos in-process writes do not drive the reload path, the docs overstate standalone CLI write behavior, and the new remote config trust boundary needs maintainer security review.

## What This Changes

The PR adds an opt-in Nacos-backed gateway config source with env-based source selection, file/Nacos adapters, long-poll reload subscription, CLI/doctor startup wiring, docs, changelog, and tests.

## Best Possible Solution

If maintainers want remote config in core, land a generic source seam with file as the default and a Nacos adapter that routes every startup, preflight, read, write, and reload path through the active source, preserves source/runtime snapshot semantics, documents standalone CLI persistence limits and trust requirements, and has targeted plus live Nacos validation. If this belongs outside core, define the required pre-start config-source plugin API first and move the Nacos adapter to plugin space.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Read startup config through the active source:** `src/gateway/server-startup-config.ts:202-206`
  - body: `loadGatewayStartupConfigSnapshot()` resolves the Nacos source, but the next read still goes through `readConfigFileSnapshotWithPluginMetadata()`. In this PR that helper still delegates directly to `createConfigIO()`, so Nacos mode can start from local/default config while the CLI/reloader are pointed at Nacos.
  - confidence: 0.92
- **[P2] Notify the reloader after Nacos writes:** `src/config/io.ts:2466-2501`
  - body: The Nacos branch updates the runtime snapshot and returns, while subscribe-mode reloaders never subscribe to in-process write notifications. Config writes can therefore report success and update `config.get` while hot-reloadable gateway services keep running the old config until a remote Nacos edit or restart.
  - confidence: 0.88
- **[P3] Document that standalone CLI writes are unsupported:** `docs/gateway/configuration.md:744`
  - body: The docs say CLI `config set` updates the in-memory snapshot in Nacos mode, but `writeConfigFile()` now rejects Nacos writes from a standalone process with no runtime snapshot or refresh handler. Update the docs/CLI wording or route the CLI through a running gateway session.
  - confidence: 0.86
