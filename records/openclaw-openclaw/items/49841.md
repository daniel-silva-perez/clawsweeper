## Summary

Keep this PR open. Current main still has no Nacos-backed config source, but the branch changes core gateway/config startup, reload, CLI, docs, and an opt-in remote config network path, so it needs maintainer product and security review rather than conservative cleanup.

## What This Changes

The branch adds an optional Nacos-backed gateway config source with file/Nacos abstraction, long-poll hot reload, Nacos env vars, CLI/gateway integration, tests, docs, and a changelog entry.

## Best Possible Solution

If maintainers want remote config in core, land a generic config-source seam with file as the default, explicit Nacos behavior, correct listener/auth/error handling, preserved source/runtime snapshots, consistent CLI and gateway semantics, docs/changelog alignment, and real Nacos validation. If maintainers want this outside core, define the pre-start config-source plugin API first, then move the Nacos adapter to a plugin path.
