Keep this PR open. Current main still does not implement a Nacos-backed config source, the proposed env surface, or subscribe-based external config reload. The PR remains an unmerged implementation candidate for a real missing capability, and the discussion contains substantive correctness and security/product review items rather than evidence that the work is obsolete.

If remote config belongs in core, land a generic, tested config-source seam with file as the default, explicit Nacos support, correct listener/auth/error handling, source/runtime snapshot preservation, CLI and gateway consistency, clear persistent-versus-in-memory write semantics, docs/changelog updates, and targeted or live Nacos validation. If maintainers decide this belongs outside core, define the missing config-source plugin API first and then move a Nacos implementation to ClawHub.
