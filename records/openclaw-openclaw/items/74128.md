## Summary

Keep open. Current main has useful adjacent mitigations, including `models.pricing.enabled: false` and lazy pricing module import, but the default-enabled pricing refresh still starts from initial Gateway runtime services before post-attach channel/sidecar readiness. The open, unmerged #73486 is the direct narrow fix candidate for this startup ordering problem, while #73323 remains the broader runtime degradation tracker.

## Best Possible Solution

Keep this open and route it with the Gateway pricing/startup work. Review and land, revise, or close #73486 as the narrow fix that moves model-pricing refresh until after sidecars/channels are ready. Then verify current main or the next release on macOS and Windows with OpenRouter/LiteLLM fetches forced to hang or timeout; keep #73323 open for any broader long-running Gateway/Telegram/RPC degradation that remains after pricing is no longer on the startup path.
