## Summary

Keep open. Current main still starts the default-enabled model-pricing refresh from initial Gateway runtime services before post-attach channel/sidecar readiness, while the direct narrow fix PR #73486 remains open and unmerged. The `models.pricing.enabled: false` opt-out is implemented and documented, but it is only a mitigation; it does not satisfy this issue's default-path request that pricing fetch failures must not delay normal boot.

## Best Possible Solution

Keep this issue open and route it with the Gateway pricing/startup work. Review and land, revise, or close #73486 as the narrow fix that moves the optional model-pricing refresh until after sidecars/channels are ready, then verify macOS and Windows startup with OpenRouter/LiteLLM fetches forced to hang or timeout. Keep #73323 open for broader long-running Gateway/Telegram/RPC degradation after the pricing startup path is fixed.
