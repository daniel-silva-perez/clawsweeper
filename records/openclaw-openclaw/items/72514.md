## Summary

Keep open. Current main has meaningful primitives for session visibility, cross-session send/history, subagent steering, restart-aborted orphan recovery, identity-linked channel routing, and memory plugins, but it does not fully implement the issue's requested product behavior: explicit subagent retry/resume, general abort recovery/notification, and seamless QClaw/WebChat-WeChat continuity remain unresolved or owner/plugin-dependent.

## Best Possible Solution

Keep this open and narrow it into an owner-reviewed session/subagent UX task: reproduce the QClaw/WebChat-WeChat flow on current main, decide whether parent-to-child send/history needs product docs or stronger tests, design an explicit subagent retry/resume/status surface for aborted runs if approved, and document the supported cross-channel continuity path using dmScope, identityLinks, docking, and memory/plugin options.

## Risks / Open Questions

- Cross-channel continuity has privacy and multi-user safety implications; automatic sharing should stay explicit through identity links, docking, memory, or a reviewed UX path.
- Blind retry/resume for aborted subagents can duplicate side effects unless it is tied to idempotency, task state, and clear parent notification semantics.
- The report names QClaw and WeChat, so maintainers should verify the affected OpenClaw, QClaw, and openclaw-weixin plugin versions before assigning ownership.
