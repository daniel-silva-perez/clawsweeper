## Summary

Keep open. Current main has a doctor-side canonical comparison that mitigates the orphan warning, but the session path resolver still returns realpath-based transcript paths and `resolveAndPersistSessionFile()` persists that value. An open unmerged implementation candidate, #49796, is explicitly marked as closing this issue, so the maintainer path is to review/land/supersede that PR rather than close this issue early.

## Best Possible Solution

Keep this issue open while maintainers review #49796 or an equivalent fix. The likely fix belongs in `src/config/sessions/paths.ts` and `src/config/sessions/session-file.ts`: preserve the caller-visible sessions directory when returning/persisting `sessionFile`, keep realpath-based containment checks for safety, and add regression coverage proving symlinked state directories persist logical paths while doctor handles legacy realpath-backed entries.
