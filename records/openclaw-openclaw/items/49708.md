## Summary

Keep this issue open. Current main still resolves and persists session transcript paths through the real sessions directory, so the portability half of the report remains unresolved. The doctor orphan warning path has a canonical comparison mitigation, but the linked open PR #49796 is still the implementation candidate for preserving logical symlinked paths and is marked as closing this issue.

## Best Possible Solution

Keep this issue open while maintainers review, rebase, or supersede PR #49796. The best fix belongs in src/config/sessions/paths.ts and the session-file persistence path: keep realpath-based containment checks for safety, return/persist the caller-visible logical sessions directory, normalize legacy realpath-backed entries, and add regression coverage for symlinked state directories plus doctor orphan detection.
