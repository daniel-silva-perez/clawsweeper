## Summary

Keep this issue open. Current main has a doctor-side canonical comparison that should reduce false-positive orphan warnings, but session path creation still resolves through the real sessions directory before persisting `sessionFile`. An open implementation candidate, #49796, explicitly uses closing syntax for this issue, so the best maintainer path is to review or supersede that PR rather than close the issue early.

## Best Possible Solution

Keep this issue open while maintainers review #49796 or an equivalent fix. The likely fix belongs in `src/config/sessions/paths.ts` / `src/config/sessions/session-file.ts` with regression coverage that proves symlinked state directories persist logical paths and that doctor handles legacy realpath-backed entries without false orphan warnings.
