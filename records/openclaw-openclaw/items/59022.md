## Summary

Keep open. Current main does have the Gateway-side A2UI route and scoped canvas auth, and direct `curl` returning `Unauthorized` is now expected behavior. The macOS node path named in the report still has a specific gap: `MacNodeRuntime` resolves A2UI from `GatewayConnection.shared` and has no capability-refresh retry, while iOS and Android use the node session's scoped canvas URL plus `node.canvas.capability.refresh`.

## Best Possible Solution

Keep this issue open and fix the macOS node A2UI path: inject or expose the node `GatewayNodeSession` scoped canvas host URL to `MacNodeRuntime`, retry failed A2UI readiness with `refreshNodeCanvasCapability()` as iOS/Android do, and add a macOS regression test for stale or missing scoped A2UI capability. It would also help to clarify in docs or diagnostics that direct `/__openclaw__/a2ui/` returning 401 is expected under canvas auth.
