## Summary

Keep this PR open. Current main still hardcodes the reply typing TTL to 2 minutes, does not expose `typingTtlSeconds` through session or agent defaults config, and the docs only describe typing mode and interval. The PR is a focused implementation of the requested behavior and the author addressed the Codex precedence review feedback; the open #43204 thread is adjacent TTL-expiry feedback, not a replacement for configurable TTL.

## Best Possible Solution

Review and land this PR, or request focused changes if maintainers want a max TTL guard or different precedence. The best implementation path is to keep this as the canonical configurable-TTL PR: add `typingTtlSeconds` to session and agent defaults schemas/types/help/generated schema, wire it into `getReplyFromConfig` with documented precedence, update typing docs, and keep the regression tests for default TTL and config resolution.
