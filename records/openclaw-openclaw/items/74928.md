## Summary

Keep open. Current main still has phase-aware visible-text handling but no explicit public/internal assistant audience contract in the shared signature parser, provider stream/replay paths, OpenResponses schema, or delivery extraction helpers. The reporter’s sanitized package smoke evidence matches the current source-level gap, and the related phase/commentary leak reports do not supersede this broader delivery-boundary contract request.

## What This Changes

The issue requests an explicit public/internal assistant audience contract at visible-text extraction and channel delivery boundaries, including fail-closed unknown audience handling and silent-sentinel suppression.

## Best Possible Solution

Define one assistant audience contract in the shared message/signature layer, carry it through provider stream parsing, replay conversion, transcript projection, and delivery, then enforce it consistently at user-visible extraction boundaries while preserving documented legacy phase-only fallback for older transcripts.

## Reproduction Assessment

Yes. A source-level synthetic assistant message with textSignature JSON containing v: 1, an id, phase: "final_answer", and audience: "internal" reproduces the gap because current parsing ignores audience and visible extraction selects final_answer text.

## Solution Assessment

Unclear as a ready implementation, but the requested direction is sound. The safest solution is a maintainer-reviewed cross-boundary contract rather than a sanitizer-only patch, because audience must be encoded, parsed, persisted, replayed, projected, and enforced consistently.
