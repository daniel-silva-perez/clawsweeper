## Summary

Current main fixes the reported `pnpm check` TS2883 failure path by running normal tsgo checks with declaration emit disabled unless explicitly requested. The maintained fix is the shared tsgo wrapper policy, not the unmerged xAI-only annotation PR, and it shipped in `v2026.4.10`.

## Best Possible Solution

Keep the shared tsgo wrapper policy as the maintained fix for normal `pnpm check` runs. Reopen or file a narrower follow-up only if OpenClaw decides bundled extension declaration emit should become part of the supported check/build contract.
