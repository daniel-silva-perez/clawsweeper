## Summary

Keep this issue open. Current main still lacks the requested shell-facing `openclaw notify <message>` command, a typed admin-scoped Gateway notification RPC/event, and a Control UI handler that renders visible in-app notifications to connected UI sessions. The shipped Web Push work in #44590 and the broader open WebUI toast request in #20237 are adjacent, but neither fully supersedes this CLI/admin broadcast feature.

## What This Changes

Add an `openclaw notify <message>` CLI command, or equivalent, that broadcasts a visible notification bubble to connected UI sessions through an admin-scoped Gateway path without running an agent turn.

## Best Possible Solution

Define an intentional core notification contract: a typed admin-scoped Gateway RPC and allowlisted broadcast event, a CLI entry point such as `openclaw notify <message>`, Control UI rendering for connected clients, and docs/tests covering authorization, delivery semantics, and its relationship to Web Push and existing session/system-event paths.
