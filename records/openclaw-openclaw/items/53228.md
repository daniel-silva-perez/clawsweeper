## Summary

Keep this issue open. Current main still lacks a first-class shell-facing notification command, a typed admin-scoped Gateway notification RPC/event, and Control UI rendering for connected-session notification bubbles; the adjacent Web Push, node-local notify, task notify policy, `chat.inject`, and system-event paths do not cover the requested no-agent-turn UI broadcast.

## What This Changes

Request to add `openclaw notify <message>` or equivalent so shell scripts can broadcast visible notifications to connected Control UI sessions through an admin-scoped Gateway path without starting an agent turn.

## Best Possible Solution

Define an intentional core notification contract: an admin-scoped Gateway RPC plus allowlisted broadcast event, a shell entry point such as `openclaw notify <message>`, Control UI rendering for connected-session notifications, and docs/tests that describe authorization, delivery semantics, and its relationship to the existing chat, system-event, Web Push, and session-broadcast surfaces.
