## Summary

Keep this issue open. Current main still has overlapping notification pieces, but it does not implement the requested shell-facing `openclaw notify <message>` command, admin-scoped all-Control-UI broadcast method, or Control UI notification-bubble event rendering.

## Best Possible Solution

Keep this open as a focused core CLI, Gateway, and Control UI feature request. The likely implementation should add an intentional `openclaw notify <message>` or `openclaw gateway notify` command backed by an admin-scoped Gateway RPC and typed broadcast event that connected Control UI clients render as an in-app notification. Web Push can remain an optional fan-out for subscribed browsers, with docs/tests covering auth scope, rate limits, failure behavior, and differences from `chat.inject`, `system event`, `tasks notify`, `nodes notify`, and `push.web.test`.
