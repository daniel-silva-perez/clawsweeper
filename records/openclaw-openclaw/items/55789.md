## Summary

Keep open. Current main still accepts reserved `x-openclaw-session-key` overrides on the OpenAI-compatible and OpenResponses HTTP paths, and the PR is a narrow security hardening patch for that behavior rather than obsolete work.

## What This Changes

The PR adds reserved session-key prefix validation to gateway HTTP request-context resolution, maps that dedicated validation error to 400 `invalid_request_error` in the OpenAI-compatible and OpenResponses handlers, and adds request-context plus endpoint regression coverage.

## Best Possible Solution

Review and land the narrow guarded validation for these two HTTP surfaces, preserving ordinary custom session keys and keeping broader ingress policy changes in separate focused PRs.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: send an OpenAI-compatible or OpenResponses request with `x-openclaw-session-key: agent:main:subagent:worker`; the explicit key is returned unchanged and passed toward agent execution.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix for the stated endpoints because it centralizes prefix detection in request-context resolution and maps only the dedicated override error to a 400 response.
