## Summary

Keep open. Current main still accepts explicit `x-openclaw-session-key` values unchanged in the OpenAI-compatible and OpenResponses HTTP paths and forwards them into agent execution. The PR's intended reserved-prefix guard is not implemented on main, and the provided PR discussion shows the main review concern was addressed on the PR branch rather than shipped.

## Best Possible Solution

Keep this PR open for maintainer review or landing. The best path is to merge or recreate the narrow guard on current main for `resolveGatewayRequestContext` callers in the OpenAI-compatible and OpenResponses HTTP endpoints, preserve normal custom session keys, test `subagent:`, `acp:`, and `cron:` cases, and keep only the dedicated reserved-session override error mapped to a 400 `invalid_request_error`.
