## Summary

Keep open. Current main still accepts explicit `x-openclaw-session-key` values unchanged in the OpenAI-compatible and OpenResponses HTTP endpoints, so the PR’s intended reserved internal-session override guard has not shipped.

## Best Possible Solution

Keep this PR open for maintainer review or landing. The best path is to apply the narrow guard on current main for `resolveGatewayRequestContext` callers in the OpenAI-compatible and OpenResponses endpoints, preserve ordinary custom session keys, test `subagent:`, `acp:`, and `cron:` variants, and keep only the dedicated reserved-session override error mapped to a 400 `invalid_request_error`.
