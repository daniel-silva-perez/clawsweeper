## Summary

Keep open. Current main has useful session/subagent lineage metadata and chat-side kill/steer commands, but the requested browser Control UI orchestration surface is still missing: there is no dedicated orchestration route, Sessions remains a flat paginated table, and selected-row bulk actions only delete/archive sessions. The related discussion also distinguishes #69364 as an adjacent independent-agent graph request and #54797 as narrower cleanup work, so neither supersedes this broader runtime subagent operations board.

## What This Changes

The issue requests a scalable Control UI orchestration surface showing agent -> session -> subagent hierarchy with active-first filters, per-row runtime/status/model data, virtualization, and bulk kill/steer/retry/cleanup operations.

## Best Possible Solution

Keep this as the umbrella Control UI orchestration request and split implementation into reviewed slices: first render existing session/task lineage and active/failed/completed state in a scalable hierarchy, then add carefully authorized bulk intervention and cleanup actions. Keep #54797 as the narrow completed-session cleanup follow-up and #69364 as the separate durable independent-agent graph discussion.
