## Summary

Keep open. Current main has useful session lineage metadata, task records, and chat-side subagent controls, but it still lacks the requested browser Control UI orchestration surface: no dedicated mission/orchestration route, no agent -> session -> subagent hierarchy rendering, and no bulk kill/steer/retry/cleanup UI. The adjacent #54797 cleanup request and #69364 independent-agent graph request do not supersede this broader runtime subagent operations board.

## What This Changes

Add a scalable Control UI orchestration view for multi-agent and subagent operations with hierarchy, active-first filters, per-row runtime metadata, virtualization, and bulk control actions.

## Best Possible Solution

Keep this as the umbrella Control UI runtime orchestration request and split implementation into reviewed slices: first surface existing session/task lineage and active/failed/completed state in a scalable hierarchy, then add carefully authorized bulk intervention and cleanup actions. Keep #54797 for narrow completed-session cleanup and #69364 for the separate durable independent-agent graph discussion.
