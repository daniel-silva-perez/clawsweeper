## Summary

Keep open. Current main has useful session/subagent metadata and chat-side subagent controls, but it still does not provide the requested browser Control UI orchestration surface: no mission/orchestration route, no hierarchical agent/session/subagent rendering, and no bulk kill/steer/retry/cleanup UI. The related #54797 cleanup request and #69364 independent-agent graph request are adjacent follow-ups, not replacements for this broader runtime orchestration board.

## What This Changes

Add a scalable Control UI orchestration view for agent-to-session-to-subagent hierarchy, active/history filters, per-row runtime metadata, and bulk kill/steer/retry/cleanup operations.

## Best Possible Solution

Keep this as the umbrella Control UI runtime orchestration request and split implementation into reviewed slices: first surface existing session/task lineage and active/failed/completed state in a scalable hierarchy, then add carefully authorized bulk intervention and cleanup actions. Keep #54797 for narrow completed-session cleanup and #69364 for the separate durable independent-agent graph discussion.
