## Summary

Keep open. Current main still lacks the requested browser Control UI orchestration surface: the navigation has no mission/orchestration route, the Sessions view renders a flat paginated table, and selected-row bulk actions still archive/delete sessions rather than kill, steer, retry, or cleanup running subagent work. The related discussion supports keeping this as an umbrella request rather than treating #54797 or #69364 as replacements.

## What This Changes

The issue requests a scalable Control UI orchestration view with agent/session/subagent hierarchy, active-first filters, bulk operations, per-row runtime metadata, and large-session-count rendering.

## Best Possible Solution

Keep this as the umbrella Control UI runtime orchestration request and split it into reviewed slices: first surface existing session/task lineage and active/failed/completed state in a scalable hierarchy, then add carefully authorized bulk intervention and cleanup actions. Keep #54797 as the narrower completed-session cleanup follow-up and #69364 as the separate durable independent-agent graph discussion.
