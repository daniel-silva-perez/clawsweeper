## Summary

Keep open. Current main still lacks the requested browser Control UI orchestration surface: there is no mission/orchestration route, the Sessions view is still a flat filtered and paginated table, and selected-row bulk actions are limited to deleting/archiving sessions rather than bulk kill, steer, retry, cleanup, or scalable parent-child operation. The related discussion also distinguishes #69364 as adjacent durable-agent graph work, while #54797 covers only a narrower cleanup slice.

## What This Changes

Request to improve Control UI for scalable multi-agent and subagent orchestration with a hierarchical mission board, active-first filters, runtime metadata, large-session-count rendering, and bulk kill/steer/retry/cleanup actions.

## Best Possible Solution

Keep this as the umbrella Control UI runtime orchestration request and split implementation into reviewed slices: first render existing session/task lineage and active/failed/completed state in a scalable hierarchy, then add authorized bulk intervention and cleanup actions. Keep #54797 as the narrow completed-session cleanup follow-up and #69364 as the separate durable independent-agent graph discussion.
