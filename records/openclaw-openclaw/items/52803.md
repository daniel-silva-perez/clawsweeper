## Summary

Keep open. Current main has useful session/task primitives and chat-side task visibility, but the requested Control UI orchestration board is still missing: there is no dedicated route, the Sessions view remains a flat paginated table, and the only selected-row bulk operation is delete.

## What This Changes

Improve the Control UI with a scalable orchestration surface for agent -> main session -> subagent/task hierarchy, active-first filtering, status metadata, and bulk kill/steer/retry/cleanup actions.

## Best Possible Solution

Keep this issue open as the umbrella Control UI product request. A good implementation would add a dedicated orchestration surface or substantial Sessions view mode that consumes existing session/task metadata, renders agent -> main session -> subagent/task hierarchy with scalable/virtualized rows, adds active/failed/completed and history controls, and wires carefully scoped bulk kill/steer/retry/cleanup actions. Keep #54797 as a narrow cleanup follow-up and #69364 as an adjacent independent-agent graph discussion.
