## Summary

Keep open. Current main has session lineage metadata, task records, and chat-side subagent controls, but the requested browser Control UI orchestration surface is still missing: there is no orchestration route, the Sessions tab renders a flat paginated table, and selected-row bulk actions only delete/archive sessions. The related #54797 cleanup request and #69364 independent-agent graph request are adjacent, not replacements for this runtime subagent operations board.

## What This Changes

The issue requests a scalable Control UI orchestration view for agent/session/subagent hierarchy, active-first filters, per-row runtime metadata, and bulk kill/steer/retry/cleanup operations.

## Best Possible Solution

Keep this as the umbrella Control UI orchestration request and split it into reviewed slices: first render existing session/task lineage and active/failed/completed state in a scalable hierarchy, then add carefully authorized bulk intervention and cleanup actions. Keep #54797 as the narrow completed-session cleanup follow-up and #69364 as the separate durable independent-agent graph discussion.
