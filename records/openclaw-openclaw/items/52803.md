## Summary

Keep open. Current main has useful subagent/session metadata, task records, and chat-side subagent controls, but it still lacks the requested browser Control UI orchestration board: no dedicated route, no hierarchy/tree rendering, no active-first runtime filters, no virtualization, and no bulk kill/steer/retry/cleanup controls. The related #54797 and #69364 discussions cover adjacent narrower/broader surfaces rather than replacing this runtime subagent operations request.

## What This Changes

The issue asks for a scalable Control UI mission/orchestration panel with agent-to-session-to-subagent hierarchy, active-first filters, per-row runtime status, virtualization, and bulk operational actions.

## Best Possible Solution

Keep this as the umbrella Control UI runtime orchestration request and split it into reviewed slices: first surface existing session/task lineage and active/failed/completed state in a scalable hierarchy, then add carefully authorized bulk intervention and cleanup actions. Keep #54797 as the narrower completed-session cleanup follow-up and #69364 as the separate durable independent-agent graph discussion.
