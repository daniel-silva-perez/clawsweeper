## Summary

Keep open. Current main has useful session lineage/status metadata and chat-side subagent kill/steer controls, but the requested browser Control UI orchestration board is still missing: there is no route or tab for it, the Sessions view remains a flat paginated table, and selected-row bulk actions only delete/archive sessions. Related #54797 and #69364 are adjacent cleanup/independent-agent graph requests, not replacements for this runtime subagent hierarchy and bulk-ops feature.

## What This Changes

The issue requests a scalable Control UI orchestration surface for agent-to-session-to-subagent hierarchy, active-first filters, per-row runtime status, virtualization, and bulk kill/steer/retry/cleanup actions.

## Best Possible Solution

Keep this as the umbrella Control UI runtime orchestration request and split it into reviewed slices: first render existing session/task lineage and active/failed/completed state in a scalable hierarchy, then add carefully scoped bulk intervention and cleanup actions. Keep #54797 as the narrow completed-session cleanup follow-up and #69364 as the separate durable independent-agent graph discussion.

## Reproduction Assessment

Not applicable as a feature request, but the absence is high-confidence from current-main source inspection: the Control UI navigation has no orchestration route and Sessions renders a flat paginated table with delete-only bulk actions.

## Solution Assessment

No for an automated one-shot repair. The requested direction is valid, but the maintainable path is a maintainer-reviewed product design split into smaller UI, gateway-contract, authorization, and validation slices.
