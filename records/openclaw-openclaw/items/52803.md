## Summary

Keep open. Current main has useful session/subagent metadata and chat-side controls, but the requested browser Control UI orchestration surface is still missing: there is no dedicated orchestration route, the Sessions view remains a flat paginated table, and selected-row bulk actions only delete/archive sessions. The follow-up discussion usefully distinguishes #69364 as an adjacent independent-agent graph request rather than a replacement for this subagent/session operations board, and #54797 is narrower cleanup work rather than a full solution.

## What This Changes

Add a scalable Control UI orchestration surface for multi-agent/subagent work with parent-child hierarchy, active-first filters, status/model/runtime columns, and bulk kill/steer/retry/cleanup actions.

## Best Possible Solution

Keep this as the umbrella Control UI orchestration request, then split implementation into reviewed UI/product slices: first render the existing session/task parent-child metadata with active/failed/completed controls and scalable rows, then add carefully authorized bulk kill/steer/retry/cleanup operations. Keep #54797 as the narrow clear-completed/dropdown cleanup follow-up and #69364 as the separate durable independent-agent graph discussion.
