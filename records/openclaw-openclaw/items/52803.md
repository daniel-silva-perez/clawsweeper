## Summary

Keep open. Current main has useful primitives for subagent/session metadata and chat slash-command controls, but it does not implement the requested Control UI orchestration board: there is no dedicated hierarchy/tree view, no active/failed/completed UI filter, no bulk kill/steer/retry controls, and no virtualized large-session rendering surface.

## Best Possible Solution

Keep this issue open and implement a Control UI orchestration surface that consumes existing `sessions.list` hierarchy/status/model/runtime fields plus task status where useful. The likely path is a dedicated tree/list view with active/failed/completed filters, history toggle, scalable rendering for 50+ rows, and multi-select actions wired to existing session/task controls such as abort/steer/delete/cancel, with any retry semantics defined explicitly before adding UI.
