## Summary

Keep this PR open. Current main still lacks the Telegram `humanDelay` wiring that this PR adds, while the shared reply dispatcher and docs already define `humanDelay` as the config path for paced block replies. The PR is a focused implementation candidate for #68945, has maintainer-positive discussion, and the diff does not show a security-sensitive or supply-chain change.

## Best Possible Solution

Review and land this PR, or an equivalent patch, after normal maintainer validation. The change belongs in `extensions/telegram/src/bot-message-dispatch.ts` plus the local runtime barrel and a focused regression test proving route-scoped `humanDelay` reaches the block dispatcher.
