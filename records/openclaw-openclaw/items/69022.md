## Summary

Keep this PR open. Current main still does not forward configured `humanDelay` into Telegram's streamed block dispatcher, while the shared dispatcher, resolver, docs, and peer channels already support the behavior. The PR remains the focused implementation candidate for #68945 and has positive maintainer triage; the remaining path is normal review/landing, not cleanup closure.

## Best Possible Solution

Review and land this PR, or an equivalent scoped patch, after normal maintainer validation. The implementation should export `resolveHumanDelayConfig` through `extensions/telegram/src/bot-message-dispatch.agent.runtime.ts`, pass `resolveHumanDelayConfig(cfg, route.agentId)` as `dispatcherOptions.humanDelay` in `extensions/telegram/src/bot-message-dispatch.ts`, keep the focused regression test, and preserve the scope note that this is not a Telegram polling-stall fix for #69147/#64288.
