## Summary

Keep open. Current main still has a reasoning-enabled Copilot Opus 4.7 path that emits Anthropic adaptive `output_config.effort` from model id alone, without a provider/model compatibility rule. The default bundled Copilot catalog now sets generated/default Copilot Claude models to `reasoning: false`, which mitigates the common default path, but it does not fix configured reasoning-enabled models or custom catalog entries.

## What This Changes

The issue requests provider-aware handling for `github-copilot/claude-opus-4.7` so unsupported Anthropic adaptive `output_config.effort` values do not cause Copilot 400s and auth-profile cooldown cascades.

## Best Possible Solution

Keep this open until maintainers land or explicitly reject a provider/model adaptive-effort compatibility rule for `github-copilot/claude-opus-4.7`. The narrow fix likely belongs in `src/agents/anthropic-transport-stream.ts` or the Copilot Anthropic wrapper: omit or clamp unsupported `output_config.effort` values for that provider/model pair, preserve direct Anthropic Opus 4.7 behavior, add Copilot-specific matrix tests, and reconcile #70322/#70584 into one canonical track.
