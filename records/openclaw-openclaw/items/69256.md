## Summary

Keep open. Current main still lacks this PR's intended active-descendant guard for isolated cron direct-delivery cleanup, and the PR remains the active implementation candidate with maintainer follow-up already incorporated in the branch.

## Best Possible Solution

Keep this PR open, update it against current main, and review or land this branch or an equivalent patch. The fix belongs in `src/cron/isolated-agent/delivery-dispatch.ts`: direct `deleteAfterRun` cleanup should use the run-scoped descendant count, preserve the parent cron session while descendants are active, retry cleanup after the descendant tree drains, delete once, and keep the regression tests plus docs/changelog aligned.
