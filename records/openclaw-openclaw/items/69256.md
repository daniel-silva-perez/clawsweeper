## Summary

Keep open. Current main still lacks the active-descendant guard for isolated cron direct-delivery cleanup, while this PR remains the active implementation path and includes maintainer follow-up plus regression coverage in its branch context.

## Best Possible Solution

Keep this PR open and review or land this branch, or an equivalent patch, after updating it against current main. The implementation should keep direct deleteAfterRun cleanup in src/cron/isolated-agent/delivery-dispatch.ts, use the run-scoped descendant count before deleting, defer cleanup while descendants are active, retry deletion after the descendant tree drains without leaking indefinitely, and keep the regression tests plus cron docs/changelog aligned.
