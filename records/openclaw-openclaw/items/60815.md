## Summary

Keep open: current main overlaps with parts of the runtime hardening, but it does not fully implement this PR's intended branch, and the PR is not merge-ready. The diff is broad, draft, and mixes runtime fixes with launch/legal/docs work while introducing regressions in reset model override semantics, CLI banner config behavior, and readiness probe proxy handling.

## What This Changes

This draft PR changes session model override persistence/reset behavior, Discord watchdog and disconnect handling, outbound and heartbeat delivery fallbacks, gateway readiness/connect timing, agent directive defaults, Apple gateway timeouts, README/package discoverability, and adds VeriClaw legal/privacy/launch documents.

## Best Possible Solution

Split the useful runtime hardening into narrow PRs rebased on current main, preserve the existing user-vs-auto model override and CLI/gateway config contracts, and route the VeriClaw launch/legal/privacy documents through separate owner-reviewed product/legal changes.

## Solution Assessment

No. The current PR shape is not the best way to solve the reported instability; the safer path is smaller patches that keep current documented behavior intact and isolate legal/product launch copy from runtime changes.
