## Summary

Keep this PR open. Current main still returns keyword-search snippets by selecting the FTS row `text` and truncating it from the beginning, so the match-centered SQLite FTS5 `snippet()` behavior has not landed. The PR is directionally useful and its latest head addressed the earlier token-limit and ellipsis review feedback, but it targets the old `src/memory/manager-search.ts` path; the current implementation lives under `extensions/memory-core/src/memory/manager-search.ts` and also has a trigram substring-only fallback that a rebased fix must preserve.

## Best Possible Solution

Keep this PR open as a valid implementation candidate. The best fix is to port the patch to `extensions/memory-core/src/memory/manager-search.ts`, use SQLite FTS5 `snippet()` for MATCH-backed keyword results with explicit 1..64 token clamping and visible ellipsis markers, keep the substring-only trigram fallback on the existing text-truncation path, preserve fallback lexical ranking, and add a regression test where the keyword appears deep in a large chunk. Include a user-facing changelog fix entry when the code lands.
