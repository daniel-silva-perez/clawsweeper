## Summary

Keep this PR open. Current main still builds keyword-search snippets from the raw FTS row text and truncates from the beginning, so the match-centered FTS5 snippet behavior has not landed. The PR is directionally useful and its latest patch addresses the token-limit and ellipsis feedback, but it targets the obsolete `src/memory/manager-search.ts` path; the current code lives in `extensions/memory-core/src/memory/manager-search.ts` and includes trigram substring fallback behavior that a rebased fix must preserve.

## Best Possible Solution

Keep this PR open as a valid implementation candidate. Port the change to `extensions/memory-core/src/memory/manager-search.ts`, use SQLite FTS5 `snippet()` for MATCH-backed keyword results with explicit 1..64 token clamping and visible ellipsis markers, keep the substring-only trigram fallback on the existing text-truncation path, preserve fallback lexical ranking, and add a regression test where the keyword appears deep in a large chunk. Add a user-facing changelog fix entry when the code lands.
