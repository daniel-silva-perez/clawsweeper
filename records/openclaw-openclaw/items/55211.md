## Summary

Keep open. Current main and the latest release still dispatch internal hooks without a same-chain recursion guard, while this PR now proposes a narrow AsyncLocalStorage-based guard and tests; I did not find a discrete blocking defect in the current PR head, so the remaining action is maintainer review of hook semantics and merge priority.

## What This Changes

The PR adds an AsyncLocalStorage-backed re-entrancy guard around `triggerInternalHook` and regression tests for same-event recursion, sequential dispatch, cross-session dispatch, and cross-action dispatch.

## Best Possible Solution

Land a narrow internal-hook recursion guard if maintainers accept the AsyncLocalStorage same-chain semantics, preferably with an explicit concurrent same-key test before merge. Keep reset-specific duplicate dispatch or lifecycle policy changes in the reset/session reset paths rather than relying on this guard as a general dedupe mechanism.
