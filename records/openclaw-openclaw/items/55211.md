Keep open. Current main still dispatches internal hooks without a re-entrant guard, and the PR is not fully superseded by the reset-hook fallback fixes; the latest head resolves earlier review blockers and has successful CI checks. This needs maintainer review/landing rather than cleanup closure.

## What This Changes

Adds an AsyncLocalStorage-backed re-entrant guard to `triggerInternalHook` plus regression tests for blocking same-event recursion while allowing sequential and distinct event/session dispatches.

## Best Possible Solution

Land a narrow internal-hook recursion guard if maintainers accept the AsyncLocalStorage same-chain semantics, ideally with an explicit concurrent same-session test before merge. Keep reset-specific dedupe logic in the reset command/session reset paths rather than using this guard as the only duplicate-dispatch defense.
