## Summary

Keep open. Current main has the shared 10s gateway/client timeout work, but it still does not implement this PR's central semantic fix: clearing the pre-auth WebSocket timer after a valid connect frame and replacing that safety net with a bounded connect-auth timer. The PR is also active, has maintainer repair comments, and is marked for human review/merge readiness, so it remains the implementation candidate rather than cleanup noise.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent fix on main: clear the pre-auth challenge timer once the connect frame is syntactically/protocol valid, bound stalled async connect auth before registration, keep the regression coverage and protocol docs, and preserve the original contributor credit.
