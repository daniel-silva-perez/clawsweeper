## Summary

Keep open. Current main has raised and centralized the preauth/connect-challenge timeout, but it still clears the WebSocket preauth timer only after async connect auth and registration prep. The PR branch remains a focused implementation candidate because it clears the preauth timer after a valid connect frame, adds a bounded connect-auth watchdog, and adds docs/tests for that behavior. Security review found no supply-chain or CI/dependency changes in the PR diff; the remaining risk is functional auth-handshake timing and close semantics.

## Best Possible Solution

Keep this PR open for maintainer review and either land it or land an equivalent main-branch fix: clear the preauth timer once a connect frame is syntactically/protocol valid, bound stalled async connect auth before registration, prevent late successful registration after timeout, align the docs/changelog with the current 15 second configurable timeout policy, and preserve the original contributor credit.
