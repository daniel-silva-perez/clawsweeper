## Summary

Keep open. Current main has already shipped the shared 10s gateway/client handshake timeout, but it has not implemented this PR's remaining semantic fix: clearing the pre-auth WebSocket timer after a valid connect request and replacing it with a bounded async connect-auth timer. The PR also has recent maintainer activity and validation, so it remains an active implementation candidate rather than cleanup noise.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent fix on main: clear the pre-auth timer once a connect frame is syntactically and protocol-valid, add a bounded connect-auth timer for stalled async auth before registration, keep the gateway regression coverage and protocol docs, and preserve the original contributor credit.
