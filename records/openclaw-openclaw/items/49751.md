## Summary

Keep open. Current main already has the shared 10s preauth/client challenge timeout helper, but it still awaits async connect auth before clearing the WebSocket preauth handshake timer, so the central slow-auth race this PR addresses is not implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent fix on main: clear the preauth challenge timer after a syntactically/protocol-valid connect frame, bound stalled async connect auth before registration, keep the regression coverage and protocol docs, and preserve the original contributor credit.
