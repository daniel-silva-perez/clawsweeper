## Summary

Keep open. Current main has the shared preauth/connect-challenge timeout helper raised to 15 seconds, but it still awaits async connect auth before clearing the WebSocket preauth timer. The PR branch context still contains useful, focused work: clear the preauth timer after a valid connect frame, add a bounded connect-auth watchdog, and cover/document the behavior.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent fix on main: clear the preauth timer once a connect frame is syntactically/protocol valid, bound stalled async connect auth before client registration, keep the regression coverage and protocol docs, and preserve the original contributor credit.
