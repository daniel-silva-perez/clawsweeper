Keep open. The report is current, concrete, and expanded by a fourth same-day duplicate-emission reproduction; current main still has source paths that can freeze an incomplete announce payload and format missing child usage as zero, while the related hardening PRs are open rather than merged.

## What This Changes

Investigate and fix intermittent subagent completion announcements that lose or truncate result payloads, report zero token stats, or emit duplicate completion events despite successful child runs.

## Best Possible Solution

Keep this issue open as the canonical ASCval reproduction set. Maintainers should review or replace the active payload/delivery hardening PRs, then ensure completion announce stats recover durable child usage from transcript/session data instead of reporting zero when the session-store counters are stale.

## Reproduction Assessment

Yes for triage, though not yet as a deterministic repo test. The issue now has four concrete live reproductions, and current main still contains code paths that can freeze a partial/raw capture and format missing usage as zero.

## Solution Assessment

No, current main is not the best solution yet. The maintainable fix should make completion delivery idempotent, preserve or recover the final child payload, and compute stats from durable child usage rather than only transient session-store counters.
