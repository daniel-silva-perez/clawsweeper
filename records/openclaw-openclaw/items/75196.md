## Summary

Keep open. The issue is current and concrete, and current main still matches the reported failure family: completion output is frozen from the current captured child-session snapshot, announce stats still format missing session-store token fields as zero, and the later duplicate-emission repro is not covered by a shipped fix.

## What This Changes

The issue requests hardening native subagent completion announces so successful child runs reliably deliver the final result payload, accurate token stats, and no duplicate completion events.

## Best Possible Solution

Keep this issue open as the ASCval reproduction cluster. The desired end state is an announce path that captures final child output only after terminal child state is durable, recovers token usage from transcript/session fallback data before formatting stats, and makes completion delivery idempotent across retry/resume paths.

## Reproduction Assessment

Yes for triage, though not as a deterministic local test yet. The issue gives four detailed live reproductions, and current main has source-backed paths that can freeze a partial result and report zero tokens when session-store usage is missing.

## Solution Assessment

No. Current main is not the best solution yet; the safer path is to land or replace the active payload-hardening PRs where they apply, then add focused token-usage fallback and duplicate-emission regression coverage.
