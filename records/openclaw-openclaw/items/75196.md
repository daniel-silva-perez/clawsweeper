## Summary

Keep open. The report is current and concrete: the payload-loss side overlaps still-open subagent completion hardening work, but the token-stat zeroing has independent current-main evidence because the announce stats path reads only the live session-store fields and falls back to zero instead of using the transcript usage fallback that other session surfaces already have.

## What This Changes

The issue reports intermittent subagent completion events that can lose or truncate the worker result payload and/or show zero token usage even when the child run completed successfully.

## Best Possible Solution

Keep this issue open as the ASCval reproduction and token-accounting follow-up. Maintainers should land or replace the active payload hardening PRs if they cover the missing-result cases, and separately make completion announce stats recover final child usage from durable transcript/session data instead of reporting zero when the session-store row is stale.

## Reproduction Assessment

Yes for triage, though not as a deterministic unit test yet. The issue provides three detailed live reproductions, and current main has source-backed paths that can freeze a latest fragment and can format missing session-store usage as zero.

## Solution Assessment

Partly. The issue's suggested instrumentation would help localize the race, but the durable fix should preserve or recover the final payload and token usage before announcing completion; current main is not that best solution.
