## Summary

Keep open. The report is current, concrete, and still matches current-main risk areas: completion output is frozen from a transient child-session snapshot, announce stats still format missing live session-store counters as zero, and the related payload-hardening PRs are open rather than merged. The new duplicate-emission repro also points at delivery/idempotency sequencing that should be triaged with the adjacent announce hardening work.

## What This Changes

The issue reports intermittent subagent completion announcements that can lose the final result payload, zero token stats, or emit duplicate completion events after successful worker runs.

## Best Possible Solution

Keep this issue open as the canonical ASCval reproduction for the combined completion-announce failure family. Maintainers should merge or replace the active payload-hardening PRs where they cover missing/truncated results, add transcript-backed token recovery for completion announce stats, and add focused regression coverage for duplicate completion emission before treating the issue as fixed.

## Reproduction Assessment

Yes for triage, though not yet as a deterministic local test. The issue gives four detailed live reproductions with run timing, payload shape, token stats, and operational verification, and current main has source-backed paths that can freeze a transient result and format missing usage as zero.

## Solution Assessment

No, current main is not the best solution. The durable fix should preserve or recover the terminal child payload before announcing, use durable transcript/session usage for stats when live counters are stale, and make completion delivery idempotent across retry/cleanup paths.
