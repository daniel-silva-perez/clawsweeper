## Summary

Keep open. Current main still has source-backed paths matching both reported symptoms: completion can freeze the currently selected child output, while compact announce stats read only session-store counters and format missing usage as zero; the related payload-hardening PRs remain open rather than shipped.

## What This Changes

The issue reports intermittent subagent completion events that can lose or truncate the child result payload and/or report zero token usage even when the worker completed successfully.

## Best Possible Solution

Keep this issue open as the ASCval reproduction and token-accounting follow-up. The desired end state is durable final-payload delivery plus announce stats that recover final child usage from transcript/session data instead of reporting zero when the live session-store row is stale.

## Reproduction Assessment

Yes for triage, though not as a deterministic automated repro yet. The issue gives three detailed live reproductions, and current main has static source paths that can freeze an incomplete selected output and format missing session-store usage as zero.

## Solution Assessment

No. Instrumenting the announce path would help localize the race, but the maintainable fix is to preserve or recover the final child payload and usage before announcing completion; current main does not do that yet.
