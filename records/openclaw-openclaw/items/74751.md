## Summary

Keep this PR open. Current main does not contain the Gateway abort/wait fix, and the PR is protected by both MEMBER author context and the maintainer label. The latest diff addresses the earlier agent/chat dedupe namespace collision by choosing the prefix from the pre-abort run kind, but static review found one remaining correctness gap in the session-wide abort path.

## What This Changes

The PR emits Gateway lifecycle cancellation events for aborted chat runs, avoids writing normal chat completion dedupe entries after abort, records cancelled wait snapshots from sessions.abort with a run-kind-aware dedupe prefix, and adds targeted gateway test coverage.

## Best Possible Solution

Have the branch owner or maintainer keep the Gateway-only scope, write prefix-safe terminal snapshots for every run id returned by a session-wide abort, preserve the agent RPC idempotency boundary, and rerun the targeted gateway wait/abort validation. SDK chat-projection behavior should remain with the sibling #74750 work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.83

Full review comments:

- **[P2] Record abort snapshots for every aborted run:** `src/gateway/server-methods/sessions.ts:1350-1351`
  - body: sessions.abort can be called with only a session key, and chat.abort returns every aborted run id for that session. This patch writes the terminal wait snapshot only for runIds[0], so any other aborted run has no dedupe snapshot; an immediate agent.wait on that run can still report a wait-only timeout instead of the cancelled terminal state. Iterate all returned run ids and write the prefix-safe snapshot for each.
  - confidence: 0.82
