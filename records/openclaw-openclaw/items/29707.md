## Summary

Keep this issue open. Current main still excludes direct Amazon Nova Bedrock IDs from cache-retention eligibility and forces them through the Bedrock no-cache wrapper, while AWS now documents Nova-compatible prompt-cache checkpoints on `system`/`messages` and recommends explicit prompt caching for cost savings. ([docs.aws.amazon.com](https://docs.aws.amazon.com/bedrock/latest/userguide/prompt-caching.html)) Recent Bedrock cache work covers Claude application inference profiles, not direct Nova.

## What This Changes

Request native Amazon Nova prompt-caching support in the bundled Amazon Bedrock provider by removing the direct Nova no-cache restriction and using Bedrock-compatible cache checkpoints without invalid Anthropic-only metadata.

## Best Possible Solution

Add provider-owned Nova prompt-cache support in the bundled Amazon Bedrock plugin: recognize supported direct, regional/global, and relevant profile Nova IDs; preserve valid explicit cache settings; add only Bedrock/Nova-compatible `cachePoint` checkpoints in supported `system` and `messages` positions; keep unsupported non-Anthropic Bedrock models on the no-cache path; and update tests, public docs, and changelog together.
