## Summary

Keep this issue open. Current main still drops explicit `cacheRetention` for direct Amazon Nova Bedrock model IDs, wraps direct Nova streams with `cacheRetention: "none"`, and documents/tests that non-Anthropic Bedrock models are forced off prompt caching. AWS current Bedrock docs now list Amazon Nova prompt caching support with `cachePoint` on `system` and `messages`, so this remains a valid bundled `amazon-bedrock` plugin/runtime gap rather than stale cleanup.

## Best Possible Solution

Keep this open and implement a focused bundled `amazon-bedrock` plugin fix: recognize supported Amazon Nova direct and region-prefixed Bedrock model IDs as cache-eligible, preserve explicit `cacheRetention` without allowing unsupported Nova 1-hour TTL, inject Bedrock/Nova-compatible `cachePoint` blocks on `system` and last user `messages`, keep unsupported non-Anthropic Bedrock models forced to no-cache, and update tests, prompt-caching docs, and changelog.
