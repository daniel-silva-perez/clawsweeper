## Summary

Keep open. Current main still routes direct Amazon Nova Bedrock model IDs through the non-Anthropic no-cache path, while current AWS Bedrock docs describe Nova prompt caching and recommend explicit prompt caching for cost savings and more consistent performance. ([docs.aws.amazon.com](https://docs.aws.amazon.com/bedrock/latest/userguide/prompt-caching.html))

## What This Changes

The issue requests enabling native prompt caching for Amazon Nova Bedrock models by recognizing supported Nova IDs, passing valid cache settings through, and avoiding incompatible cache metadata.

## Best Possible Solution

Add provider-owned Nova prompt-cache support in the bundled Amazon Bedrock plugin: recognize supported direct, region-prefixed, global, and profile Nova model IDs; pass through valid explicit cache settings; inject only Bedrock/Nova-compatible cachePoint checkpoints for supported payload fields; avoid applying Claude-only TTL assumptions to Nova; keep unsupported non-Anthropic Bedrock models on the no-cache path; and update tests, docs, and changelog together.

## Repair Work Prompt

Fix native prompt caching for Amazon Nova on the bundled Amazon Bedrock provider. Observable bug: direct Nova IDs such as amazon.nova-micro-v1:0 and global.amazon.nova-2-lite-v1:0 currently resolve no cacheRetention and then get wrapped with cacheRetention: "none", even though AWS Bedrock supports Nova prompt caching. Keep the fix provider-owned in extensions/amazon-bedrock and the existing prompt-cache family helpers; do not broadly enable caching for every non-Anthropic Bedrock model, do not add core bundled provider IDs outside the existing owner boundary, and do not assume Claude-only 1-hour TTL semantics apply to Nova without verifying the AWS contract. Expected shape: add a Nova cache eligibility helper for supported direct/region/global/profile model IDs, allow explicit cacheRetention to survive for those IDs, inject or preserve only Bedrock/Nova-compatible cachePoint checkpoints in system/messages, keep unsupported Bedrock models on createBedrockNoCacheWrapper, update existing disabled-path tests to assert Nova is enabled, add negative coverage for unsupported non-Anthropic models, update docs/reference/prompt-caching.md and docs/providers/bedrock.md, and add a user-facing changelog entry with reporter credit. Related refs: this issue, #20815, #20866, #22303, #69953. Validate with targeted tests and oxfmt first, then pnpm check:changed in Testbox before handoff if code/docs/changelog changed.
