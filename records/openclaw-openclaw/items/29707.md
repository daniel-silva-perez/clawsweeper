## Summary

Keep this issue open. Current main still excludes direct Amazon Nova Bedrock model IDs from cache-retention eligibility and then forces them through the Bedrock no-cache wrapper, while current AWS Bedrock docs list Nova prompt-cache support on `system` and `messages` with a 5-minute TTL and recommend explicit prompt caching for cost savings. 

## What This Changes

Request native prompt-caching support for Amazon Nova models on the bundled Amazon Bedrock provider instead of forcing non-Anthropic Bedrock models to `cacheRetention: "none"`.

## Best Possible Solution

Add provider-owned Nova prompt-cache support in the bundled Amazon Bedrock plugin: recognize supported direct, region-prefixed, global, and profile Nova model IDs; allow explicit cache settings only where AWS supports them; avoid Anthropic-only TTL assumptions; inject or pass through only Bedrock/Nova-compatible cache behavior for `system` and `messages`; keep unsupported non-Anthropic Bedrock models on the no-cache path; and update tests, public docs, and changelog together.
