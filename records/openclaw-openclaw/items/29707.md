## Summary

Keep this issue open. Current main still treats direct Amazon Nova Bedrock model IDs as non-cache-eligible, forces their stream options to `cacheRetention: "none"`, and has tests/docs asserting that behavior, while AWS now documents Nova prompt caching support for supported Nova models in `system` and `messages` fields. 

## What This Changes

Request native prompt caching support for Amazon Nova models on Bedrock by allowing `cacheRetention` and using Nova-compatible Bedrock cache-point handling instead of forcing caching off.

## Best Possible Solution

Implement the change in the bundled amazon-bedrock provider and shared cache-retention gate: classify supported Nova direct/system/region-prefixed model IDs as cache-eligible, preserve explicit cache retention, add Bedrock-compatible cache points only where Nova supports them, avoid `ttl: "1h"` for Nova, keep unsupported non-Anthropic Bedrock models forced off, and update tests, docs, and changelog.

## Repair Work Prompt

Implement native prompt caching support for supported Amazon Nova models in the bundled amazon-bedrock provider. Treat this issue as the canonical tracker and preserve the earlier #20815 safety goal: do not send Anthropic-only cache metadata where Nova rejects it. Update `resolveCacheRetention` / Bedrock family helpers so supported direct, region-prefixed, and system-prefixed Nova IDs preserve explicit `cacheRetention`; update the amazon-bedrock stream wrapper so those IDs do not go through `createBedrockNoCacheWrapper`; inject Bedrock Converse `cachePoint` blocks only in Nova-supported `system` and `messages` positions, not tools; do not add `ttl: "1h"` for Nova because AWS documents 1-hour TTL for selected Claude models, not Nova; keep unsupported non-Anthropic Bedrock models forced off. Add tests for Nova Micro/Lite/Pro/Nova 2 Lite and at least one unsupported non-Anthropic model, update `docs/reference/prompt-caching.md`, and add a user-facing changelog entry with reporter credit. Do not move provider-owned Nova behavior into generic core beyond the minimal shared cache-retention seam.
