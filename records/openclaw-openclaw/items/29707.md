## Summary

Keep this issue open. Current main still treats direct Amazon Nova Bedrock model IDs as non-cache-eligible and forces them through the no-cache Bedrock wrapper, while AWS now documents prompt caching support for Amazon Nova models with `system` and `messages` cache checkpoints. ([docs.aws.amazon.com](https://docs.aws.amazon.com/bedrock/latest/userguide/prompt-caching.html))

## What This Changes

The issue requests native prompt-caching support for supported Amazon Nova models on Bedrock by preserving cache retention and avoiding unsupported Anthropic-specific cache metadata.

## Best Possible Solution

Add provider-owned Nova prompt-cache support in the bundled Amazon Bedrock plugin and shared retention gate: recognize supported Nova direct, region-prefixed, and global model IDs; preserve explicit cache retention; add only Bedrock/Nova-compatible cache checkpoints; keep unsupported non-Anthropic Bedrock models on the no-cache path; and update tests, prompt-caching docs, and changelog together.

## Repair Work Prompt

Implement native prompt-caching support for supported Amazon Nova models in the bundled Amazon Bedrock provider. Current main still forces direct/region/global Nova Bedrock IDs through `createBedrockNoCacheWrapper`, and `resolveCacheRetention` rejects explicit Nova cache retention. Update the provider/runtime boundary so supported Nova IDs are cache-eligible, explicit `cacheRetention` is preserved, Nova payloads receive only Bedrock/Nova-compatible cache checkpoints, and unsupported non-Anthropic Bedrock models remain forced off. Do not apply Anthropic-only TTL behavior to Nova; AWS docs currently show Nova with 5-minute prompt-cache TTL and `system`/`messages` checkpoint support. Update focused tests, `docs/reference/prompt-caching.md`, and add a user-facing changelog entry with proper contributor attribution. Prefer provider-owned helpers and existing SDK seams; avoid putting bundled provider IDs into unrelated core paths unless the existing shared cache-retention seam must be extended for this provider.
