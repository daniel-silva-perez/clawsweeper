## Summary

Keep this issue open. Current main still excludes direct Amazon Nova Bedrock model IDs from cache-retention eligibility and then forces them through the Bedrock no-cache wrapper, while current AWS Bedrock docs show Nova-compatible prompt-cache checkpoints and recommend explicit prompt caching for cost savings. ([docs.aws.amazon.com](https://docs.aws.amazon.com/bedrock/latest/userguide/prompt-caching.html))

## What This Changes

Request native Amazon Nova prompt caching on Bedrock by allowing cache retention and using Nova-compatible cache-point handling instead of Anthropic-only cache metadata.

## Best Possible Solution

Add provider-owned Nova prompt-cache support in the bundled Amazon Bedrock plugin: recognize supported direct, region-prefixed, and global Nova model IDs; preserve explicit cache retention where valid; inject only Bedrock/Nova-compatible `cachePoint` checkpoints on `system` and `messages`; keep unsupported non-Anthropic Bedrock models on the no-cache path; and update tests, public docs, and changelog together.

## Repair Work Prompt

Fix native prompt caching for supported Amazon Nova models on the bundled `amazon-bedrock` provider. Current main drops explicit Nova `cacheRetention` in `resolveCacheRetention` and then wraps direct Nova model IDs with `cacheRetention: "none"`; AWS Bedrock now supports Nova explicit prompt caching via `cachePoint` on `system` and `messages`. Keep this provider-owned and narrow: add a Nova model-family helper that recognizes supported direct, region-prefixed, and global Nova model IDs such as `amazon.nova-micro-v1:0`, `amazon.nova-lite-v1:0`, `amazon.nova-pro-v1:0`, `amazon.nova-premier-v1:0`, and `amazon.nova-2-lite-v1:0`; allow explicit `cacheRetention` to pass for those Nova IDs; inject only Nova-compatible Bedrock Converse `cachePoint` markers into `system` and the last user message when needed; keep unsupported non-Anthropic Bedrock models on the no-cache wrapper. Do not reuse Anthropic `cache_control`; do not add core bundled IDs outside provider/shared cache-family seams unless needed; do not send `ttl: "1h"` for Nova unless AWS docs/live verification prove that specific model supports it. Update the existing no-cache tests to cover supported Nova cache pass-through/injection and keep Mistral/unsupported model no-cache coverage. Update `docs/reference/prompt-caching.md`, likely `docs/providers/bedrock.md`, and add a single-line user-facing changelog fix with proper contributor attribution. Related refs: this issue, #20815, #20866, #22303, #69953.
