## Summary

Keep this issue open. Current main still excludes direct Amazon Nova Bedrock model IDs from cache-retention eligibility and still forces direct non-Anthropic Bedrock models through the no-cache wrapper, while AWS current Bedrock docs describe Nova prompt caching and `cachePoint` usage. This remains a valid bundled `amazon-bedrock` plugin/runtime gap, with a narrow implementation path but some provider-contract details to verify.

## Best Possible Solution

Keep this open for a focused bundled `amazon-bedrock` plugin fix: add explicit support for supported Amazon Nova direct model IDs, preserve or inject Bedrock/Nova-compatible `cachePoint` metadata, map OpenClaw cache-retention values to the current AWS Nova TTL contract without sending unsupported 1-hour TTLs, and keep unsupported non-Anthropic Bedrock models protected by the no-cache wrapper.
