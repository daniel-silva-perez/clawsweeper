## Summary

Keep this issue open. Current main still excludes direct Amazon Nova Bedrock model IDs from cache-retention eligibility and forces them through the Bedrock no-cache wrapper, while AWS now documents Nova prompt caching with explicit Bedrock cache checkpoints for cost savings. The prior ClawSweeper keep-open comments still match current main.

## What This Changes

The issue requests native Amazon Nova prompt-caching support in the bundled Amazon Bedrock provider instead of forcing Nova models to `cacheRetention: "none"`.

## Best Possible Solution

Add provider-owned Nova prompt-cache support in the bundled Amazon Bedrock plugin: recognize supported direct, region-prefixed, global, and relevant profile Nova IDs; preserve valid explicit cache settings; add only Bedrock/Nova-compatible `cachePoint` checkpoints in supported `system` and `messages` positions; keep unsupported non-Anthropic Bedrock models on the no-cache path; and update tests, docs, and changelog together.

## Repair Work Prompt

Fix native Amazon Nova prompt-caching support for the bundled Amazon Bedrock provider. Current main treats direct Nova model IDs such as `amazon.nova-micro-v1:0`, `amazon.nova-lite-v1:0`, `amazon.nova-pro-v1:0`, `amazon.nova-premier-v1:0`, and `amazon.nova-2-lite-v1:0` as non-cache-eligible and forces `cacheRetention: "none"`. Implement provider-owned Nova support without changing unrelated providers: add a dedicated helper for supported Nova family IDs, including direct IDs and region/global-prefixed Bedrock IDs; preserve explicit `cacheRetention` where valid; add Bedrock Converse `cachePoint` checkpoints for Nova only in supported `system` and `messages` positions; do not send Anthropic `cache_control` metadata to Nova; keep unsupported non-Anthropic Bedrock models on the no-cache path. Verify AWS Bedrock Nova TTL semantics before mapping `cacheRetention: "long"`; do not add `ttl: "1h"` for Nova unless current AWS docs or SDK behavior explicitly supports it. Update focused tests in the Amazon Bedrock plugin and cache-retention helpers, update `docs/reference/prompt-caching.md` and `docs/providers/bedrock.md`, and add a user-facing changelog fix with reporter/contributor attribution. Do not broaden core with bundled-provider IDs unless a generic seam is strictly needed.
