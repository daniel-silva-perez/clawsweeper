## Repair Work Prompt

Fix native prompt caching for Amazon Nova models on the bundled Amazon Bedrock provider. Current main still treats direct Nova IDs such as amazon.nova-micro-v1:0 and global.amazon.nova-2-lite-v1:0 as non-Anthropic Bedrock models, so resolveCacheRetention drops explicit settings and the provider wrapper forces cacheRetention: "none". Add provider-owned Nova cache eligibility for supported direct, regional/global, and profile IDs; preserve cacheRetention: "none"; allow valid explicit cache settings; inject only Bedrock/Nova-compatible cachePoint checkpoints into supported Converse payload fields such as system and messages; do not apply Claude-only ttl: "1h" semantics to Nova unless verified from AWS docs/source; and keep unsupported non-Anthropic Bedrock models on the no-cache path. Update focused tests in the Bedrock plugin, cache-retention resolver/defaults, public prompt-caching and Bedrock docs, and add a user-facing changelog entry crediting @tribecar-shurn. Do not move provider-specific Nova policy into core beyond generic helper seams already used by provider plugins, and do not change unrelated Bedrock guardrail, temperature, or application-inference-profile behavior.
