## Summary

Current main has meaningful partial UX work: Quick Settings, schema-backed config editing, gateway wizard RPCs, ClawHub skill search/install in Control UI, and CLI/chat-native plugin install paths. It still does not satisfy the central request for a broader WordPress-like guided dashboard configuration experience with dashboard-native ClawHub plugin discovery, installation, and configuration.

## Best Possible Solution

Keep this open as product/UI roadmap work, or split it into narrower issues for guided Control UI reconfiguration flows, contextual setup help, dashboard-native ClawHub plugin/package discovery-install-update, plugin configuration UX, and any missing gateway/admin API needed to expose those flows safely. Existing Quick Settings, schema-backed config editing, wizard RPCs, and ClawHub skill UI are useful building blocks but do not close the central request.
