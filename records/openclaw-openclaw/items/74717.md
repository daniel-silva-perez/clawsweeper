Keep open. Current main still lacks an empty-text guard in the Feishu inbound handler, while this active contributor PR is a narrow implementation of the linked Feishu/MiniMax failure with focused regression coverage and no definite blocking bug in the patch.

The PR adds an early Feishu text-message guard that drops whitespace-empty text payloads before route resolution or session dispatch, plus a unit test for that behavior.

## Best Possible Solution

Feishu should ignore whitespace-empty text events before route resolution or transcript/session writes, while preserving the existing media, post, share_chat, merge_forward, and audio-transcription paths.
