## Summary

Keep this PR open for maintainer review. Current main still allows Feishu empty text payloads to become an empty inbound body, and the PR’s guard is narrowly placed before route resolution/session dispatch with focused regression coverage; I did not find a blocking correctness or security issue in the proposed diff.

## What This Changes

The PR adds a Feishu text-message guard that returns early for whitespace-empty text payloads and adds a regression test asserting that such events do not resolve an agent route or send a Feishu reply.

## Best Possible Solution

Feishu should ignore whitespace-empty text events before routing or transcript/session writes, while preserving existing handling for media, post, share, merge-forward, and audio transcription paths.
