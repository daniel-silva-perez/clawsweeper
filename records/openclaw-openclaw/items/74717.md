## Summary

Keep this PR open for maintainer review. Current main still lets a Feishu text payload that parses to an empty string continue toward routing and agent dispatch, while the proposed guard is at the Feishu-owned inbound boundary and has focused coverage for skipping routing/sends. I found no blocking functional or security issue in the patch.

## What This Changes

The branch adds a Feishu inbound-message guard that drops whitespace-empty text events before routing and adds a unit test covering that no agent route resolution or Feishu send occurs.

## Best Possible Solution

Feishu blank text events should be ignored before routing or transcript/session writes, while richer Feishu message types keep flowing through the existing media, post, share, and merge-forward handling.
