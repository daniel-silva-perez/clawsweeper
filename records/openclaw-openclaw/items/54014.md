## Summary

Keep open. Current main still only partially honors OPENCLAW_HOME: state/config and agent defaults are covered, but the dev/git checkout default, install.sh bootstrap paths, and user-facing docs still use or document the OS account home for internal OpenClaw defaults.

## What This Changes

The issue asks OpenClaw to respect OPENCLAW_HOME as the effective home for state, config, git checkout, installer bootstrap, and related internal paths unless more specific path environment variables are set.

## Best Possible Solution

Define the supported OPENCLAW_HOME contract for internal OpenClaw defaults, update the update/dev checkout and installer bootstrap paths to derive from it unless explicit path overrides are set, add focused regression tests, and align public docs with any intentional OS-home exceptions.
