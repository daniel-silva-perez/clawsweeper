## Summary

Keep open. Current main partially honors OPENCLAW_HOME for state/config and agent defaults, but the reported broader contract is still false for the dev/git checkout default, installer bootstrap workspace/config checks, and public docs alignment. Related #65536 is adjacent sandbox tilde work, not a fix for these remaining installer/git/docs gaps.

## What This Changes

The issue asks OpenClaw to respect OPENCLAW_HOME as the effective home for state, config, git checkout, installer bootstrap, and related internal paths unless more specific env vars are set.

## Best Possible Solution

Define a clear OPENCLAW_HOME contract for internal OpenClaw defaults, update the dev/git checkout and installer bootstrap defaults to derive from it unless explicit path env vars are set, add focused regression tests, and align user-facing docs with documented OS-home exceptions.
