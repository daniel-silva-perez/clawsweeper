## Summary

Keep open. Current main partially implements OPENCLAW_HOME for state/config and agent defaults, but the broader requested contract is still not true: the dev/git checkout path and installer bootstrap defaults still fall back to the OS account home, and the docs still conflict about whether OPENCLAW_HOME covers all internal path resolution. Related #65536 is useful adjacent sandbox tilde work, but it does not resolve the git checkout, installer bootstrap, or docs gaps.

## What This Changes

The issue asks for OPENCLAW_HOME to seed OpenClaw’s default state, config, git checkout, and related storage paths unless more-specific environment variables are set.

## Best Possible Solution

Define a clear OPENCLAW_HOME contract for internal OpenClaw defaults, update the dev/git checkout and installer/onboarding defaults to derive from it unless explicit path env vars are set, add focused tests, and align the public docs with documented OS-home exceptions.
