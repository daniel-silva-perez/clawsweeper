## Summary

Keep open. Current main partially honors `OPENCLAW_HOME` for default state/config/workspace paths, but the issue’s broader contract is still unresolved: git checkout defaults, installer git-mode defaults/help, and docs still treat OS home as the default unless narrower overrides are set.

## Best Possible Solution

Keep this issue open and narrow it to the `OPENCLAW_HOME` path contract. Maintainers should decide whether git checkout and installer/onboarding defaults are internal OpenClaw paths; if yes, update `src/cli/update-cli/shared.ts`, `scripts/install.sh`, relevant docs, and focused tests so `OPENCLAW_HOME` seeds those defaults unless `OPENCLAW_GIT_DIR`, `OPENCLAW_STATE_DIR`, or `OPENCLAW_CONFIG_PATH` is explicit. If OS-home behavior is intentional for personal skills, host `~` paths, shell files, or Trash, document those as explicit exceptions.
