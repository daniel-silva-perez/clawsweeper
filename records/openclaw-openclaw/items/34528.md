## Summary

Keep this issue open. Current main still constructs Feishu synthetic reaction message IDs with the reported `:reaction:<emoji>:<uuid>` suffix and can pass that suffixed ID into Feishu API `message_id` path parameters. The related #39558 PR is still open and explicitly uses closing syntax, so it is an implementation candidate rather than a reason to close the issue before merge.

## Best Possible Solution

Keep this issue open and review, update, or supersede #39558 with a narrow Feishu plugin fix that strips synthetic reaction suffixes at Feishu API boundaries while preserving internal synthetic IDs, plus regression tests for reply, typing/reaction, and media/card/message API paths as applicable.
