## Summary

Keep open. Current main still has the Mattermost slash-command auth gap this PR targets, and the PR is paired with the same author's open bug via closing syntax, so it should remain open as an implementation candidate. The patch direction is useful, but the diff still needs maintainer/security review because the new trigger binding can silently no-op when omitted and explicit non-HTTPS callback URLs still register.

## What This Changes

The PR updates Mattermost slash command docs, tests, and code to reject non-HTTPS derived callback URLs and bind registered callback tokens to their slash triggers before processing callbacks.

## Best Possible Solution

Keep the PR open while maintainers choose the canonical Mattermost slash-auth fix. The desired end state is one implementation that fails closed or clearly enforces the chosen policy for non-HTTPS callbacks, validates each callback against the registered/current command and trigger, and keeps Mattermost docs plus focused regression tests aligned.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: enable Mattermost native slash commands, omit commands.callbackUrl, and current main derives an http:// callback while accepting any registered account token for the caller-supplied trigger.

## Solution Assessment

No, not as submitted. The patch is in the right area, but the best fix should make trigger binding non-optional or fail closed when missing, and should handle explicit http:// callbackUrl values consistently with the security policy and docs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Reject explicit non-HTTPS callback URLs:** `extensions/mattermost/src/mattermost/monitor-slash.ts:168`
  - body: The new guard only fires when callbackUrl is omitted. If an operator explicitly configures an http:// callback, registration still proceeds and sends Mattermost command tokens over cleartext, despite the new docs saying the callback should be HTTPS. Apply the same fail-closed or warning policy to the resolved URL regardless of whether it was derived.
  - confidence: 0.9
- **[P2] Make trigger binding required:** `extensions/mattermost/src/mattermost/slash-http.ts:55`
  - body: commandTokenTriggers is optional, so a handler constructed without it silently falls back to token-only validation and accepts a valid token for any trigger. Since this PR relies on token-to-trigger binding for the security fix, make the map required or fail closed when it is absent.
  - confidence: 0.84
