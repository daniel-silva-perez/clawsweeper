## Summary

Keep open. Current main still has the reported source-level failure mode for recovered-looking assistant output plus trailing error payloads, and the issue is already paired with open unmerged PR #74840, so it should stay open until that implementation or a narrow equivalent lands.

## What This Changes

The issue asks cron isolated `agentTurn` announcement summaries to prefer the final assistant-visible reply over an intermediate recovered tool error.

## Best Possible Solution

Review and land PR #74840 or a narrow equivalent that distinguishes recovered/presentation-only error payloads from real fatal cron errors, then let the linked issue close after merge.

## Reproduction Assessment

Yes. The current helper test boundary reproduces the core classifier behavior with all-error payloads plus `finalAssistantVisibleText`, and the payload tests show how a mutating tool error can trail assistant success text.

## Solution Assessment

No, not as a blanket rule. Final assistant text should win only when the earlier tool error was actually recovered or presentation-only; fatal structured errors, denial signals, run-level failures, and unresolved mutating-tool failures must remain fatal.
