## Summary

Keep open. Current main still treats all-error delivery payloads as fatal even when `finalAssistantVisibleText` contains a recovered final answer, and the issue is already paired with open unmerged PR #74840, so it should not be closed before that implementation or a narrow equivalent lands.

## What This Changes

Cron `agentTurn` summaries should prefer the agent's final self-corrected assistant reply over an intermediate tool error when building the final announcement.

## Best Possible Solution

Review and land PR #74840 or a narrow equivalent that lets recovered final assistant text drive text-only cron announcements while preserving fatal handling for real run-level errors, denial signals, and unresolved mutating-tool failures.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. The helper boundary currently reproduces the classifier choosing the last error payload over `finalAssistantVisibleText`, and the payload tests show how a successful assistant answer can be followed by a mutating tool error payload.

## Solution Assessment

Is this the best way to solve the issue? No, not as a blanket rule. The maintainable fix should prefer final assistant text only for recovered or presentation-only errors while keeping structured fatal errors and unresolved mutating-tool failures fatal.
