## Summary

Keep open. Current main still lets fatal structured error payloads override `finalAssistantVisibleText` in the cron outcome classifier, and this issue is already paired with open unmerged PR #74840, so it should not be closed before that implementation or a narrow equivalent lands.

## What This Changes

The issue asks isolated cron `agentTurn` summaries and announcements to prefer the agent's final self-corrected assistant reply over an intermediate tool error.

## Best Possible Solution

Review and land PR #74840 or a narrow equivalent that lets recovered assistant output drive the cron summary while preserving fatal handling for real run-level errors, denials, and unresolved mutating-tool failures.

## Reproduction Assessment

Yes. The helper boundary can reproduce the classifier choosing error payloads over `finalAssistantVisibleText`, and the payload tests show how a mutating tool warning can trail a successful-looking assistant reply.

## Solution Assessment

No, not as a blanket rule. The maintainable fix should prefer final assistant text only for recovered or presentation-only errors, while keeping structured fatal errors and unresolved mutating-tool failures fatal.
