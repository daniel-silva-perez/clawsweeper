## Summary

Keep open. Current main has implemented important pieces of the recommendation: the managed `openclaw` profile is the default, `user` is an explicit existing-session profile, and explicit `profile="user"` is kept off the sandbox fallback path. But this issue is also explicitly treated by maintainer discussion as the canonical attach/fallback diagnostics tracker, and current code still leaves material acceptance gaps around target selection, aggregate diagnostics, and Chrome MCP attach failure classification.

## Best Possible Solution

Keep this issue open as the canonical browser existing-session attach/fallback diagnostics tracker. The best path is a focused browser plugin fix that preserves the shipped managed-`openclaw` default and explicit-`user` routing, then adds structured attach reason codes, robust Chrome MCP spawn/packaging handling, default filtering away from devtools/internal tabs, and a status/doctor view that clearly shows managed profile health separately from existing-session health.
