## Summary

Keep open. Current main partially satisfies the request: WebChat/Control UI accepts document uploads and Gateway stages non-image files, and PDF/TXT extraction exists. The remaining Word part is still missing: Office-style DOC/DOCX MIME types are treated as binary and skipped before text extraction, and the bundled document extractor only declares PDF support.

## Best Possible Solution

Keep this open, or manually consolidate it into a narrower WebChat document-extraction tracker. The best product path is to define the Word policy first: likely add DOCX extraction through the existing document-extractor plugin seam, explicitly reject or defer legacy .doc unless a safe extractor is chosen, and align MIME limits, docs, and tests across Control UI, Gateway staging, media-understanding, and document-extract.

## Risks / Open Questions
