## Summary

Keep open. Current main partially satisfies the WebChat document-upload request: Control UI now accepts non-video files including PDF, TXT, DOC, and DOCX, serializes them as file attachments, and Gateway stages them as managed media. The remaining requested Word behavior is still missing: PDF/text extraction exists, but Office-style application/vnd Word/DOCX attachments are treated as binary and skipped, the bundled document extractor only registers PDF, and native provider document metadata is still PDF-only.

## Best Possible Solution

Keep this issue open or manually consolidate it into a canonical WebChat document-extraction tracker. The maintainer path should define the Word policy first: likely DOCX extraction through the document-extractor plugin seam, an explicit unsupported path for legacy .doc/OLE unless a safe extractor is chosen, aligned MIME/size/docs behavior, and focused tests across Control UI, Gateway attachment staging, media-understanding, and document-extractors.
