## Summary

Keep open. Current main has usage/cost visibility, provider quota snapshots, context-size knobs, model fallback after provider errors, opt-in tool-loop detection, and internal retry caps, but it still lacks built-in policy that enforces per-request, per-session, daily token, or request budgets before model dispatch.

## What This Changes

Requested built-in token and request budget controls at request, session, and daily scopes, with warning, interruption, fallback, and loop-safety behavior.

## Best Possible Solution

Define an additive core budget-policy contract at the gateway or agent-runtime model-dispatch boundary, reuse existing usage/cost summaries only where reliable, enforce configured limits before model calls where possible, expose status/doctor visibility, and keep loop detection as a related but separately configurable guardrail.
