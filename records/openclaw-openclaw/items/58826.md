## Summary

Keep open. Current main has usage/cost visibility, provider quota snapshots, context-size knobs, opt-in tool-loop detection, and defensive retry caps, but it does not expose configurable hard enforcement for per-request, per-session, daily token, or request budgets. The related #42475 and #9912 cover important slices, but they do not fully cover the per-provider quota/fallback and token/request budget controls requested here.

## What This Changes

Add built-in configurable token budget, request cap, warning/interruption, fallback, and loop-control policies across request, session, daily, provider, and agent scopes.

## Best Possible Solution

Define a core budget policy contract at the gateway/agent runtime boundary, reuse existing session usage and provider quota snapshots where reliable, enforce before model dispatch where possible, expose status/doctor visibility, and keep loop detection as a configurable guardrail rather than treating usage telemetry alone as enforcement.
