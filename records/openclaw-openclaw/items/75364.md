## Summary

Keep open. Current main still has the linked Feishu topic-thread bug, and this PR is a focused owner-local fix, but it is not ready to land because the media-send branch still cannot carry the new thread semantics and the required user-facing changelog entry is missing.

## What This Changes

The PR adds Feishu topic-session auto-threading for `message(action="send")` text/card sends and adds text-path regression tests in `extensions/feishu/src/channel.test.ts`.

## Best Possible Solution

Update this PR so Feishu topic-scoped `send` actions propagate thread semantics consistently for every supported payload path, add the required changelog entry, then land it with focused Feishu channel tests.

## Reproduction Assessment

Yes. On current main, a mocked Feishu action call with `action: "send"`, a topic-style session key, and `toolContext.currentMessageId` reaches the `send` branch with `replyInThread: false`; the PR’s added text tests reproduce that path, and the remaining media gap is visible by tracing the media branch into `feishuOutbound.sendMedia`.

## Solution Assessment

Mostly yes, but incomplete. Keeping this Feishu-specific behavior in the Feishu plugin is the right boundary; the safer implementation is to thread the same auto-thread decision through text, card, and media sends and cover each with regression tests.
