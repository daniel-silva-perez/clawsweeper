## Summary

Keep open. Current main still uses the same dev-update preflight structure and still calls `pnpm lint` for candidate commits; the local lint wrapper now applies low-memory thread limits, but `pnpm lint` still runs three oxlint shards concurrently and there is no current source/test evidence that the updater-specific SIGKILL path is solved. The linked #63627 context is relevant to OOM pressure, but it is not a complete duplicate because this report is about `openclaw update --channel dev`, not the pre-commit hook.

## What This Changes

The issue reports that switching an npm install to the dev channel fails because dev update preflight runs lint on candidate commits and the lint process is SIGKILLed on Ubuntu.

## Best Possible Solution

Keep the issue open until the dev update preflight is made memory-safe for small Linux hosts or maintainers explicitly change the preflight contract. A good end state would preserve CI-grade validation while avoiding a user-facing update failure from local lint OOM, with tests covering the chosen updater behavior.

## Reproduction Assessment

Yes, at the behavior level: the issue provides the exact command and current main still runs the same `preflight lint` step on Linux dev updates. I did not run `openclaw update --channel dev` because this review is read-only and that command mutates the checkout/install state.

## Solution Assessment

Unclear. Current main has local lint throttling, but it still runs full sharded lint during dev update preflight; a safer fix likely needs maintainer agreement on whether to serialize shards, use a lower-memory preflight lane, or change which checks block user updates.
