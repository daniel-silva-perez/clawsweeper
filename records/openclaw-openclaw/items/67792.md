## Summary

Keep open. Current main still renders the Telegram model button fallback from the provider-local model id, so an OpenRouter ref like `openrouter/openai/gpt-5.4-mini` still displays without the `openrouter/` prefix. There is also an open, unmerged fix PR (#72752) that explicitly closes this issue, so the right cleanup path is to review and land or close that PR rather than close this issue early.

## What This Changes

The issue requests that Telegram model selection buttons display the full provider/model ref for nested OpenRouter model ids instead of only the provider-local id.

## Best Possible Solution

Update the bundled Telegram plugin so fallback button labels for nested provider-local ids render as canonical provider/model refs, while leaving selection callback data, resolution, pagination, truncation, and catalog display-name overrides intact. Add focused regression tests for OpenRouter nested ids, including the current-model marker.
