## Summary

Keep open. Current main does not implement the preserve-Funnel behavior: serve mode still unconditionally reapplies Tailscale Serve, there is no `gateway.tailscale.preserveFunnel` config surface, and linked regression #57241 is still open. This PR is a focused implementation candidate rather than something already handled on main.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #57241. Review the Tailscale status parser and security posture, run the targeted gateway/config tests plus generated schema checks, and land or request changes if the opt-in `gateway.tailscale.preserveFunnel` behavior is the desired product direction.
