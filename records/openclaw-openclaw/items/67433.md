## Summary

Keep open. Current main still implements `POST /hooks/agent` as a fire-and-forget hook that returns only `{ ok: true, runId }`; it does not parse `blocking`, await the isolated agent result, return response text, or surface blocking-run failures. This PR remains a live implementation candidate for #49019.

## Best Possible Solution

Keep this PR open as the concrete implementation path for #49019. Maintainers should rebase/review it against current main’s split hook request handler, validate blocking success/error/idempotency/trust semantics with targeted Gateway tests and docs, then either land it if the API shape is accepted or keep #49019 as the canonical product/API tracker if the contract needs redesign.
