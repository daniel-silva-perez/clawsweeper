## Summary

Keep open. Current main still has only fire-and-forget `/hooks/agent` behavior, so this PR remains a live implementation candidate for the open result-return-mode request; however, the PR head is stale against the current split hook handler and would regress current hook event routing and announcement policy without a careful rebase.

## What This Changes

This PR adds a `blocking` flag to `/hooks/agent`, changes Gateway hook dispatch to optionally await isolated agent runs and return text/error responses, and adds Gateway handler tests for blocking, error, and idempotency behavior.

## Best Possible Solution

Rebase this implementation onto current main’s `src/gateway/server/hooks-request-handler.ts` split, preserve the current `shouldAnnounceHookRunResult`, target-agent event session, sanitization, and `trusted:false` behavior in both blocking and non-blocking paths, add public docs for the accepted API contract, and land only after targeted Gateway hook tests cover success, failure, idempotency, timeout, and trust/session routing semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep hook events in the target agent session:** `src/gateway/server/hooks.ts:97`
  - body: Current main resolves completion events through the target agent session before enqueueing. This helper always enqueues with `mainSessionKey`, so failures or fallback summaries from a hook targeting another agent would surface in the default main session instead of that agent’s session, regressing the current hook event routing behavior.
  - confidence: 0.86
- **[P2] Honor the current hook announcement policy:** `src/gateway/server/hooks.ts:95`
  - body: Current main only announces ok results when delivery was requested and not already attempted or delivered; `deliver:false` successful hooks are intentionally quiet. This new helper announces every `!result.delivered` result, so a quiet machine-callback run would add unwanted system events and fail the existing deliver:false regression behavior.
  - confidence: 0.87
