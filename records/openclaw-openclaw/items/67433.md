## Summary

Keep open. Current main still has only fire-and-forget `POST /hooks/agent` behavior and returns `{ ok: true, runId }`; it does not parse a blocking/result flag, await the agent run, return response text, or surface blocking run failures. This PR remains a live implementation candidate for the related open feature request #49019.

## Best Possible Solution

Keep this PR open as the concrete implementation path for #49019. Maintainers should rebase/review it against current main's split hook request handler, settle the public API shape, add or retain targeted Gateway tests for blocking success, blocking failure, idempotency/retry behavior, timeout handling, and trust sanitization, then land it if the contract is accepted or keep #49019 as the canonical product/API tracker if redesign is needed.
