## Summary

Keep open. Current main still normalizes hosted custom-scheme origins through `URL.origin`, which is the opaque string `null` for `tauri://localhost` and `electron://app`, so exact `gateway.controlUi.allowedOrigins` entries cannot match. The issue is paired with open implementation PR #73511, while earlier related PRs #46523 and #46555 were closed unmerged, so the issue should remain open until the linked fix lands or is explicitly abandoned.

## What This Changes

The issue requests exact `gateway.controlUi.allowedOrigins` matching for custom desktop-app URL schemes such as `tauri://localhost` and `electron://app` without requiring wildcard origins.

## Best Possible Solution

Land #73511 or an equivalent narrow gateway fix that reconstructs `scheme://host` only for hosted custom schemes with opaque `URL.origin`, keeps literal `Origin: null` invalid, and adds accepted and rejected `tauri://` / `electron://` regression tests in `src/gateway/origin-check.test.ts`.
