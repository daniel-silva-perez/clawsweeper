## Summary

Keep open. Current main still returns `URL.origin` directly for browser origins, and Node serializes hosted custom schemes such as `tauri://localhost` and `electron://app` as the opaque origin `null`, so exact `gateway.controlUi.allowedOrigins` entries cannot match. The GitHub context also shows open PR #73511 uses closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests exact `gateway.controlUi.allowedOrigins` matching for browser clients that send hosted custom URL schemes such as `tauri://localhost` or `electron://...`.

## Best Possible Solution

Update the gateway origin parser to preserve exact `scheme://host` matching only for hosted custom schemes with opaque `URL.origin`, keep literal `Origin: null` invalid, and add accepted and rejected `tauri://` / `electron://` regression tests for the existing origin-check helper.
