## Summary

Keep open. Current main and the latest release still do not provide native `.gitignore`-style exclusions for `openclaw backup create`; the only archive filter is the narrow installed-plugin `extensions/*/node_modules` skip. The Keep My Claw plugin linked in the discussion is useful prior art and a workaround, but it does not close the first-party backup CLI gap, and open PR #44288 still targets this issue with closing syntax.

## What This Changes

Add user-controlled `.gitignore`-style exclude patterns and exclude-file support to `openclaw backup create`, with defaults that avoid large dependency/cache trees and sensitive files.

## Best Possible Solution

Keep this as the user-facing tracker for native backup exclusions. The desired end state is a core backup implementation with repeatable gitignore-style patterns, exclude-file parsing, clear default and protected-path semantics, manifest behavior that avoids leaking sensitive filenames, focused CLI/archive regression tests, and updated public docs at https://docs.openclaw.ai/cli/backup.

## Reproduction Assessment

Yes. This is a high-confidence missing-feature check: current source, tests, and docs show no user-supplied exclude-pattern path, while existing coverage confirms only installed-plugin dependency trees are skipped and root `state/node_modules` remains included.

## Solution Assessment

Unclear for the exact implementation, but keeping this issue open is the right tracking outcome. PR #44288 is the active candidate and should be reviewed, narrowed, landed, replaced, or closed by maintainers before creating another fix path.
