## Summary

Keep this open. Current main still lacks native `.gitignore`-style `--exclude` or `--exclude-file` support for `openclaw backup create`; the only new filtering I found is a narrow installed-plugin `node_modules` skip under the state `extensions/` tree. The Keep My Claw plugin linked in the discussion is a useful workaround, but it does not close the native backup CLI feature gap, and open PR #44288 explicitly targets this issue with closing syntax.

## What This Changes

The issue requests `.gitignore`-style exclude patterns for `openclaw backup create`, including repeatable exclude flags, an exclude-file option, and sensible default exclusions for large or sensitive files.

## Best Possible Solution

Keep this as the user-facing tracker for native backup exclude support. The desired end state is a core backup implementation with repeatable exclude patterns, exclude-file parsing, restore-safety handling for protected paths, manifest behavior that avoids leaking sensitive filenames, regression coverage, and public docs at `https://docs.openclaw.ai/cli/backup`.
