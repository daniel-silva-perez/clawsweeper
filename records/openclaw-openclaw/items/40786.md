## Summary

Keep open. Current main still lacks native `.gitignore`-style `--exclude` or `--exclude-file` support for `openclaw backup create`; it only has coarse backup scope flags and a narrow installed-plugin `node_modules` filter. The Keep My Claw plugin linked in the discussion is a useful workaround, but it does not close the native CLI feature gap, and open PR #44288 explicitly targets this issue with closing syntax.

## What This Changes

Add native `.gitignore`-style exclude patterns and exclude-file support to `openclaw backup create` so users can omit dependency trees, temporary files, logs, and sensitive files from backup archives.

## Best Possible Solution

Keep this as the user-facing tracker for native backup exclude support. The desired end state is a core backup implementation with repeatable exclude patterns, exclude-file parsing, clear default/protected-path semantics, manifest behavior that avoids leaking sensitive filenames, regression coverage, and public docs at `https://docs.openclaw.ai/cli/backup`.
