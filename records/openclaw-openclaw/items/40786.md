## Summary

Keep open. Current main and the latest release still lack native `.gitignore`-style `--exclude` or `--exclude-file` support for `openclaw backup create`; only the narrower installed-plugin `extensions/*/node_modules` filter exists. The Keep My Claw plugin linked in the discussion is useful prior art/workaround, but it does not close the native CLI feature gap, and open PR #44288 still targets the issue with closing syntax.

## What This Changes

The issue requests native `.gitignore`-style backup exclusions, including repeatable exclude patterns, an exclude-file option, and sensible defaults for dependency, cache, log, and secret-like files.

## Best Possible Solution

Keep this issue as the user-facing tracker for native backup exclude support. The desired end state is a core backup implementation with repeatable patterns, exclude-file parsing, clear default and protected-path semantics, manifest behavior that avoids leaking sensitive filenames, regression coverage, and public docs at https://docs.openclaw.ai/cli/backup.
