## Summary

Keep this issue open. Current main still lacks native `.gitignore`-style backup exclude support; `openclaw backup create` only has coarse scope flags plus a narrow installed-plugin `extensions/*/node_modules` filter. The provided discussion also shows open PR #44288 explicitly targets this request, so the issue should remain open until that candidate is reviewed, replaced, or merged.

## Best Possible Solution

Keep this issue open as the user-facing tracker for native backup exclude support. Review and either land #44288 after maintainer/security scrutiny or replace it with a smaller core implementation that adds repeatable `backup create --exclude` flags, exclude-file parsing, typed options, archive filtering, restore-safety rules for protected paths, manifest behavior that avoids leaking sensitive filenames, regression tests, and public docs at `https://docs.openclaw.ai/cli/backup`.
