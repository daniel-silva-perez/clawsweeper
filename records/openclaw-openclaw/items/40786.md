## Summary

Keep this issue open. Current main only added a targeted archive filter for installed plugin `extensions/*/node_modules`; it still does not expose `.gitignore`-style `--exclude` or `--exclude-file` controls for `openclaw backup create`, and the provided GitHub context shows open PR #44288 explicitly targets this issue with closing syntax.

## Best Possible Solution

Keep this issue open as the user-facing tracker for native backup exclude support. Retain the shipped narrow plugin-dependency pruning from `e11eb031`, then review #44288 or replace it with a smaller core implementation that adds `backup create` CLI flags, typed backup options, archive filtering, restore-safety/protected-path behavior, manifest handling that avoids leaking sensitive filenames, regression tests, and docs updates at `docs.openclaw.ai/cli/backup`.
