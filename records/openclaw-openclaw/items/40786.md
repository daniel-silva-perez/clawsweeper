## Summary

Keep open. Current main and the latest release still do not provide native `.gitignore`-style `--exclude` or `--exclude-file` support for `openclaw backup create`; the only shipped archive filter is the narrower installed-plugin `extensions/*/node_modules` skip. The Keep My Claw plugin linked in the discussion is useful prior art and a workaround, but it does not close the native CLI feature gap, and open PR #44288 still targets this issue with closing syntax.

## What This Changes

Add `.gitignore`-like exclude patterns and exclude-file support to `openclaw backup create` so users can omit large, noisy, or sensitive files from local backup archives.

## Best Possible Solution

Keep this as the user-facing tracker for native backup exclude support. The desired end state is a core backup implementation with repeatable patterns, exclude-file parsing, clear default and protected-path semantics, manifest behavior that avoids leaking sensitive filenames, regression coverage for archive contents and CLI forwarding, and public docs at https://docs.openclaw.ai/cli/backup.
