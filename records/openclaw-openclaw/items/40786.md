## Summary

Keep open: current `main` still lacks native `.gitignore`-style `--exclude` or `--exclude-file` support for `openclaw backup create`. The current implementation only skips installed-plugin `node_modules` under the state `extensions/` tree; the Keep My Claw plugin linked in the discussion is a useful workaround, but it does not close the native CLI feature gap, and open PR #44288 explicitly targets this issue with closing syntax.

## What This Changes

Add `.gitignore`-like exclude pattern support to `openclaw backup create`, including repeatable `--exclude`, an `--exclude-file`, and sensible exclusions for noisy or sensitive paths.

## Best Possible Solution

Keep this as the user-facing tracker for native backup exclude support. The desired end state is a core backup implementation with repeatable exclude patterns, exclude-file parsing, restore-safety handling for protected paths, manifest behavior that avoids leaking sensitive filenames, regression coverage, and public docs at `https://docs.openclaw.ai/cli/backup`.
