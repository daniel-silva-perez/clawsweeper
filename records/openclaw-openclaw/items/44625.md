## Summary

Keep open. Current main still has the reported Slack capabilities failure path: Slack scope discovery only tries `auth.scopes` and `apps.permissions.info`, does not extract Slack SDK response-metadata scopes, and the capabilities diagnostic still renders that helper's joined errors. The closed fallback PR remains useful prior art, but it was not merged and there is no open replacement PR found for this issue.

## What This Changes

The issue reports that `openclaw channels capabilities` fails to query Slack Bot scopes because Slack returns `unknown_method` for the two scope-discovery methods OpenClaw currently uses.

## Best Possible Solution

Keep the fix inside the Slack plugin: teach `fetchSlackScopes` to support modern granular bot tokens by reading Slack SDK response metadata from `auth.test`, while preserving the legacy methods and the plugin-owned capabilities diagnostics contract.

## Repair Work Prompt

Fix the Slack capabilities scope lookup bug reported in https://github.com/openclaw/openclaw/issues/44625. Current `extensions/slack/src/scopes.ts` only tries `auth.scopes` and `apps.permissions.info`, which can both return `unknown_method` for modern manifest-created `xoxb-*` Slack bot tokens. Keep the change inside the Slack plugin. Update `fetchSlackScopes` so modern granular bot tokens and legacy workspace-app tokens both produce granted scopes, likely by adding `auth.test` as a fallback and extracting `response_metadata.scopes` and `response_metadata.acceptedScopes` from the pinned `@slack/web-api` result. Preserve existing error aggregation and the lazy-loading boundary in `extensions/slack/src/channel.ts`; do not move Slack-specific behavior into core or change Slack dependency versions. Add focused regression coverage, preferably a colocated `extensions/slack/src/scopes.test.ts`, and update existing channel capability tests only if diagnostics/source labels intentionally change. Add a user-facing `CHANGELOG.md` fix entry with appropriate reporter/prior-art credit unless maintainers decide the active changelog has moved.
