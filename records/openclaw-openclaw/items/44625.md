## Summary

Current main still routes Slack scope discovery through only `auth.scopes` and `apps.permissions.info`, so the reported `unknown_method` capabilities output remains plausible. The prior auth.test fallback PR was closed unmerged, and there is no focused regression test for the missing fallback.

## Best Possible Solution

Keep this issue open. The Slack plugin should add a modern-token scope discovery path in `extensions/slack/src/scopes.ts`, likely using `auth.test` as the primary path or as a fallback after the legacy workspace-app methods, preserve existing legacy behavior, and add focused tests for `unknown_method` fallback plus diagnostics output.
