## Summary

Keep open. The PR is a concrete Feishu UX improvement attempt, but the proposed patch introduces visible Typing reactions before current-main reply eligibility gates and bypasses the existing Feishu typing lifecycle/config contract, so it is not safe to merge as-is.

## What This Changes

The PR adds a direct Feishu `im.messageReaction.create` call in `handleFeishuMessage()` to add a `Typing` emoji reaction immediately after an inbound message is logged.

## Best Possible Solution

If immediate Feishu feedback is desired, integrate it with the existing typing lifecycle after reply eligibility is known, respect `typingIndicator`, clean up the reaction after dispatch, and add focused tests for disabled groups, group/sender allowlists, mention-required groups, and disabled typing indicators while resolving the native-typing question in #69572.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Move ack reaction after reply eligibility checks:** `extensions/feishu/src/bot.ts:327-336`
  - body: The new `Typing` reaction runs before disabled-group, group allowlist, sender allowlist, and mention-required early returns. Messages the bot is configured to ignore would still get a visible processing reaction with no reply, especially in mention-required group chats.
  - confidence: 0.93
- **[P2] Clean up the immediate typing reaction:** `extensions/feishu/src/bot.ts:330-334`
  - body: This direct create call does not keep the returned reaction id or pair it with a delete call. Current Feishu typing reactions are tracked and removed through the existing lifecycle, so eligible messages would keep a stale Typing reaction after the bot replies.
  - confidence: 0.88
- **[P2] Honor the typingIndicator setting:** `extensions/feishu/src/bot.ts:330-334`
  - body: The new path unconditionally sends a Typing reaction, while the existing Feishu typing path skips reaction calls when `account.config.typingIndicator` is false. Users who disabled typing indicators would still get this API call and visible reaction.
  - confidence: 0.9
