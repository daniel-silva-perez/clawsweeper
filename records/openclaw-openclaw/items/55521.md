## Summary

Keep this PR open. Current main has Feishu typing reactions only through the reply dispatcher, not the PR's intended immediate receipt acknowledgement, and the submitted diff still needs changes before it is safe to merge because it fires before policy gates and ignores the documented typing toggle.

## Best Possible Solution

Keep the PR open and ask for a revision: place the acknowledgement only after all silent-drop gates and before actual agent dispatch, honor `typingIndicator`, preferably reuse the existing typing lifecycle/backoff helpers, and add focused Feishu tests proving no reaction is sent for ignored messages while eligible messages get the intended early feedback.
