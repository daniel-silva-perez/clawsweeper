## Summary

Keep open. Current main already buffers Telegram albums by `media_group_id`, but the LINE side of the request is still unresolved: LINE media webhook events are downloaded, contextualized, and dispatched one event at a time with no same-sender media buffer or LINE batching config/docs surface.

## What This Changes

Request optional media batching so rapid Telegram album media or LINE same-sender media bursts reach the agent as one turn instead of separate image turns.

## Best Possible Solution

Land a LINE-plugin scoped implementation: add an optional LINE/account media batching delay, buffer downloadable LINE media by account plus conversation plus sender, flush on timer and before same-key text/non-media events, and aggregate media through the existing `allMedia` context fields without moving channel-specific heuristics into core.

## Reproduction Assessment

Yes. A high-confidence reproduction path is visible from source: send two LINE image webhook message events from the same sender/chat and `handleLineWebhookEvents` will call `handleMessageEvent` twice, each with a fresh single-event `allMedia` array and immediate `processMessage`.

## Solution Assessment

No for the proposed generic gateway-layer placement. The narrower maintainable fix is plugin-owned LINE batching using existing context/media seams, while leaving the already-implemented Telegram album buffering unchanged.
