## Summary

Keep open. Current main implements Telegram album buffering, but the LINE half of the request is still unresolved: LINE image/media events are still handled one webhook message at a time, with no same-sender media buffer or LINE config/docs surface. The related #55216 report is a separate Telegram media-group download-error issue, not a canonical replacement for the remaining LINE batching work.

## What This Changes

The issue requests optional media batching so rapid image bursts, especially LINE media messages and Telegram albums, are delivered to the agent as one media-rich turn.

## Best Possible Solution

Implement the remaining behavior inside the LINE plugin boundary: add an optional media batching delay, buffer downloadable LINE media by account plus conversation plus sender, flush on timer and before same-key text or non-media events, aggregate media into one LineInboundContext, and keep Telegram album buffering and shared text debounce semantics unchanged.
