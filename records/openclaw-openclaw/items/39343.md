## Summary

Keep open. Current main still implements the Telegram album side, but the LINE plugin still handles each downloadable media webhook event independently and has no LINE media batching config or docs. The 2026-04-26 ClawSweeper comment identified the same gap, and the relevant files are unchanged between that review SHA and current main.

## Best Possible Solution

Implement the missing bundled LINE behavior in the LINE plugin: add an optional per-account/top-level `mediaBatchDelayMs`-style config, buffer downloadable media events by account + conversation + sender, flush on timer and before same-key text/non-media events, aggregate all media into one `LineInboundContext`, preserve access control/mention/replay/reply-token behavior, document it, and add focused LINE regression tests. Keep the shipped Telegram `media_group_id` behavior unchanged.
