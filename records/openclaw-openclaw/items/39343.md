## Summary

Keep open. Current main implements the Telegram album side, but the LINE side of the request is still missing: LINE webhook media is downloaded and sent to the agent one event at a time, with no same-sender media buffer or LINE config/docs surface.

## What This Changes

Add optional inbound media batching so rapid image/media messages, especially Telegram albums and LINE image bursts, can become one agent turn instead of one turn per media event.

## Best Possible Solution

Implement the remaining behavior inside the LINE plugin: add an optional media batching delay, buffer downloadable media briefly by account plus conversation plus sender, flush on timer and before same-key text/non-media events, and aggregate the media into one LineInboundContext while preserving existing Telegram album handling and shared text-only debounce semantics.
