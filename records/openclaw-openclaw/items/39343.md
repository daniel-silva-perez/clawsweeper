## Summary

Keep open. Current main still solves the Telegram album side, but not the LINE side: LINE downloadable media events are handled one event at a time with a fresh `allMedia` array and immediate `processMessage`, and the LINE config/docs expose no media batching delay. The shared inbound debounce path is explicitly text-only, so it does not cover LINE image batches.

## Best Possible Solution

Keep this issue open and implement the missing LINE plugin behavior: add an optional LINE media batching delay, buffer downloadable LINE media by account + conversation + sender, flush on timer and before same-key text/non-media events, aggregate all media into one `LineInboundContext`, preserve access control/mention/replay/reply-token behavior, document the option, and leave existing Telegram `media_group_id` buffering unchanged.
