## Summary

Keep open. Current main solves the Telegram album part of the request, but LINE still processes rapid image/media webhook events independently, so the central LINE image-burst problem remains actionable and narrow.

## What This Changes

The issue requests optional gateway or channel-level media batching so rapid LINE images and Telegram albums are delivered to the agent as one turn instead of separate media turns.

## Best Possible Solution

Implement the remaining behavior inside the LINE plugin boundary: add an optional media batching delay, buffer downloadable LINE media by account plus conversation plus sender, flush on timer and before same-key text or non-media events, aggregate media into one `LineInboundContext`, and leave existing Telegram album buffering and shared text debounce semantics unchanged.
