## Summary

Keep open. Current main already handles Telegram albums by buffering `media_group_id`, but the LINE part of the request is still missing: LINE webhook image/media events are still downloaded and forwarded one event at a time, with no same-sender media buffer or LINE config/docs option.

## What This Changes

Request to batch rapid inbound image/media updates so one agent turn receives the related media together, using Telegram album metadata when available and LINE sender/chat/time proximity otherwise.

## Best Possible Solution

Implement the remaining behavior inside the LINE plugin boundary: add an optional LINE media batching delay, buffer downloadable LINE media by account plus conversation plus sender, flush on timer and before same-key text/non-media events, aggregate the media into one `LineInboundContext`, and leave the existing Telegram album path unchanged.
