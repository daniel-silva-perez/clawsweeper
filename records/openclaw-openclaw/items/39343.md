## Summary

Keep open. Current main still addresses the Telegram album half but not the LINE image-burst half: Telegram buffers `media_group_id` updates into one aggregated agent turn, while LINE still handles each downloadable media event immediately and independently. The prior ClawSweeper comment remains accurate against the newer main SHA because the relevant LINE, Telegram, shared debounce, and docs files are unchanged since that review.

## What This Changes

The issue requests optional gateway or channel-level media batching so rapid LINE images and Telegram albums are delivered to the agent as one turn instead of separate image turns.

## Best Possible Solution

Implement the remaining behavior inside the LINE plugin boundary: add an optional media batching delay, buffer downloadable LINE media by account + conversation + sender, flush on timer and before same-key text/non-media events, aggregate all media into one `LineInboundContext`, and leave existing Telegram `media_group_id` buffering and shared text debounce semantics unchanged.
