## Summary

Keep open. Current main still leaves the LINE half of the request unresolved: Telegram album messages are already buffered by media group, but LINE image/media webhook events are still handled one event at a time with no same-sender media buffer or LINE config/docs surface.

## What This Changes

The issue requests optional media batching so rapid LINE images and Telegram albums can be delivered to the agent as one turn instead of separate media turns.

## Best Possible Solution

Implement the remaining behavior inside the LINE plugin boundary: add an optional media batching delay, buffer downloadable LINE media by account plus conversation plus sender, flush on timer and before same-key text or non-media events, aggregate media into one `LineInboundContext`, and leave existing Telegram album buffering plus shared text debounce semantics unchanged.
