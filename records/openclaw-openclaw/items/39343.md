## Summary

Keep open. Current main covers the Telegram album side with media_group_id buffering, but the LINE side of the requested media batching remains unresolved: LINE media events still download and dispatch one webhook event per agent turn, with no LINE batching config or docs surface.

## What This Changes

Request optional media batching so rapid Telegram albums or LINE image bursts are delivered to the agent as one media-rich turn instead of one turn per image.

## Best Possible Solution

Add LINE-owned media batching that buffers downloadable media briefly by account plus conversation plus sender, flushes on timer or before same-key text/non-media work, and builds one LineInboundContext containing all media while preserving existing Telegram album handling and shared text-only debounce behavior.
