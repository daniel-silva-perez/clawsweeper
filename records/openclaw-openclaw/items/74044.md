## Summary

Keep this issue open. The report is fresh, concrete, and current main still lacks direct source, test, release, or live-environment proof that the Raspberry Pi arm64 Discord startup CPU wedge is fixed under the default Discord configuration.

## What This Changes

The issue reports that enabling the Discord channel on OpenClaw 2026.4.26 can wedge the gateway at about 100% CPU on Raspberry Pi arm64, while disabling Discord makes gateway probes healthy again.

## Best Possible Solution

Validate current main or the next build on the reported Raspberry Pi arm64 setup with a real Discord bot token. If it still wedges, keep the fix inside the Discord plugin startup/lifecycle/voice surface, add targeted regression coverage for the narrowed failure mode, and document `channels.discord.voice.enabled=false` only as a confirmed workaround if live testing proves it avoids the spin.

## Reproduction Assessment

Yes for the reported release: the issue provides a concrete Pi arm64, Node 22.22.2, Discord-token startup path plus a Discord-disabled recovery check. No for current main in this environment, because reproducing it requires Raspberry Pi arm64 hardware and live Discord credentials.

## Solution Assessment

Unclear. The current-main Discord mitigations are in the right area, but they are not enough evidence to close; the safer path is live validation first, then a narrow Discord plugin fix if the wedge remains.
