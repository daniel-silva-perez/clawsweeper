## Summary

Keep this issue open. The report is fresh, concrete, and still lacks current-main live proof on Raspberry Pi arm64; current main contains adjacent Discord 429/app-id and REST retry fixes, but those do not prove the reported post-startup 100% CPU gateway wedge is resolved.

## What This Changes

The issue reports that enabling the Discord channel on Raspberry Pi arm64 can wedge the OpenClaw gateway near 100% CPU after startup, while disabling Discord restores gateway responsiveness.

## Best Possible Solution

Validate current main or the next build on the reported Raspberry Pi arm64 setup with a real Discord bot token. If the wedge still reproduces, keep the fix inside the Discord plugin startup, lifecycle, gateway, or voice surface that live testing narrows, then add focused regression coverage and document only confirmed workarounds.
