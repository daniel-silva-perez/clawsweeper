## What This Changes

The issue reports a Discord channel startup regression on Raspberry Pi arm64 where enabling Discord causes the gateway to become unresponsive and spin near 100% CPU, while disabling Discord restores gateway health.

## Best Possible Solution

Validate current main or the next build on the reported Raspberry Pi arm64 setup with a real Discord bot token. If the wedge still reproduces, keep the fix inside the Discord plugin startup/lifecycle/voice surface, add targeted regression coverage for the narrowed failure mode, and document any confirmed workaround such as disabling Discord voice only after live proof shows it avoids the spin.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the reported 2026.4.26 environment because the issue gives concrete platform, config, startup, probe, CPU, and disabled-Discord recovery steps; no for current main because the same live Raspberry Pi arm64 and Discord-token path was not exercised here.

## Solution Assessment
