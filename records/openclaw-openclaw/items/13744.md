## Summary

Keep open. Current main has partial mitigations for session-lock contention and failover masking, but it still lacks a public timeout/stale configuration surface and the embedded attempt still holds the session write lock through prompt execution and teardown; the latest discussion also includes a fresh v2026.4.26 operator reproduction.

## What This Changes

Request configurable session write-lock timeout/stale settings, narrower session lock scope, and clearer handling for lock-timeout failures.

## Best Possible Solution

Add a supported session write-lock configuration contract with schema, exported types, generated help/metadata, and docs; wire it consistently into embedded attempts and other transcript/session-store mutation paths; keep the typed lock-timeout failover suppression; and stage any deeper single-writer or prompt-lifecycle refactor as an explicitly scoped follow-up if needed.
