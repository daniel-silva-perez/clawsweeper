## Summary

Keep this issue open. Current main has useful partial mitigations: per-session run queueing and typed session write-lock timeout classification that keeps lock waits out of model failover. The requested public `openclaw.json`/env timeout and stale-lock configuration is still missing, embedded attempts still hold the session write lock through prompt/model execution, and several transcript/session mutation helpers still acquire the lock with default acquisition settings.

## What This Changes

The issue requests configurable session write-lock timeout/stale settings, narrower lock lifetime around session writes, and clearer lock-timeout error handling instead of model failover.

## Best Possible Solution

Keep this issue open until OpenClaw adds a supported session write-lock timeout/stale configuration surface, threads it through embedded runs, compaction, transcript rewrite/truncation, and session-store mutation paths as appropriate, and either narrows the embedded lifecycle lock to transcript/session critical sections or introduces a documented per-session single-writer design. Preserve the existing `SessionWriteLockTimeoutError` failover suppression and add focused concurrency/config regression coverage.
