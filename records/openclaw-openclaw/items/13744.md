## Summary

Keep open. Current main still has only partial mitigations: the lock primitive keeps local timeout/stale defaults when no caller passes values, no public session config/docs surface exists, and the embedded attempt still holds the session lock through prompt execution and end-of-attempt cleanup. Recent discussion includes a v2026.4.26 reproduction and an overlapping open PR, so this needs maintainer staging rather than cleanup closure.

## What This Changes

Request to make session write-lock timeout/stale settings configurable, narrow embedded lock scope, and surface lock contention separately from model/provider failures.

## Best Possible Solution

Add a supported session write-lock configuration contract, wire it through embedded attempts plus session-store and transcript mutation paths, preserve the typed lock-timeout failover suppression, and split any deeper single-writer or prompt-lifecycle lock redesign into an explicit follow-up if maintainers want to ship the operator-unblocking config first.
