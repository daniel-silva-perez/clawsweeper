## Summary

Keep open. Current main has partial mitigations for the reported failure mode, including per-session run lane queueing and typed lock-timeout failover suppression, but the requested public `openclaw.json`/env timeout and stale-lock configuration surface is still missing, embedded attempts still hold the session write lock through model execution and cleanup, and the latest issue comment reports the same operator-visible timeout on `v2026.4.26`.

## What This Changes

The issue requests configurable session write-lock timeout/stale settings, narrower session lock scope or single-writer serialization, and lock-timeout handling that does not surface as model/provider failover.

## Best Possible Solution

Add a supported session write-lock configuration surface with schema/help/types/docs, thread it through embedded runs, compaction, transcript rewrite/truncation, and session-store mutation paths, preserve the existing typed lock-timeout failover suppression, and then make an explicit maintainer decision on narrowing the embedded lifecycle lock or replacing it with a documented single-writer design.
