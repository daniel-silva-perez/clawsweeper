## Summary

Keep this issue open. Current main has partial lock-timeout/error-classification improvements, but it still lacks a public `openclaw.json` or env surface for session write-lock `timeoutMs`/`staleMs`, embedded runs still acquire the session write lock before the prompt path and release it only during attempt cleanup, and several transcript/session mutation paths still call the lock helper without configurable lock acquisition settings.

## Best Possible Solution

Keep this issue open until OpenClaw adds a supported session write-lock timeout/stale configuration surface, threads those settings through embedded run, compaction, transcript rewrite/truncation, and session-store mutation paths as appropriate, and either narrows the lifecycle lock to actual transcript/session critical sections or introduces a documented per-session single-writer queue. Preserve the existing `SessionWriteLockTimeoutError` failover suppression and add focused concurrency/config regression coverage.
