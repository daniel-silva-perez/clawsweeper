## Summary

Current main still has a plausible SwiftUI layout coupling path: the message list and composer are vertically coupled, and the iOS composer still uses a variable-height `TextEditor`. I did not find a merged fix, direct duplicate, or source change that isolates composer-height changes from chat scroll-view relayout.

## What This Changes

The issue reports brief iOS/iPad app freezes when the chat input bar grows vertically during multiline typing.

## Best Possible Solution

Stabilize the shared SwiftUI chat layout so composer growth changes only the bottom composer area, then verify on iPhone/iPad with a long transcript and profiling or before/after responsiveness evidence.

## Reproduction Assessment

Unclear. The best reproduction path is to run the iOS app with a long chat transcript, focus the composer, type enough text to make the `TextEditor` grow, and profile SwiftUI layout invalidation.

## Solution Assessment
