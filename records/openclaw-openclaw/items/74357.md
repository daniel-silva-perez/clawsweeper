## Summary

Keep open. Current main still applies a private-network-allowing SSRF policy through the shared trusted web-tools endpoint used by bundled web-search helpers and SDK exports, and the linked implementation PR #74360 remains open rather than merged. Because this is SSRF-sensitive and affects public plugin-facing helpers, it needs maintainer/security review before closure.

## What This Changes

Request to restrict private-network SSRF bypass in shared web-tools and web-search helpers to self-hosted providers such as SearXNG instead of applying it to all bundled search providers and SDK consumers.

## Best Possible Solution

Review, land, or supersede PR #74360 with a strict-by-default shared web-search/web-tools policy, an explicit self-hosted/private-network opt-in for SearXNG-style providers, additive SDK exports, and tests proving public providers block private redirects while self-hosted SearXNG remains usable.
