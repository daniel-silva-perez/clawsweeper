## Summary

Keep open. Current main still applies a private-network-allowing SSRF policy through the shared trusted web-tools endpoint used by bundled web-search helpers and SDK exports, and PR #74360 is still the open paired implementation path rather than a merged fix. Because this is SSRF-sensitive and affects plugin-facing helpers, it needs maintainer/security review before closure.

## What This Changes

Split the shared web-tools SSRF policy so only self-hosted providers such as SearXNG can opt into private-network access while public web-search providers remain strict by default.

## Best Possible Solution

Review, land, or supersede PR #74360 with a strict-by-default trusted web-tools/web-search policy, an explicit self-hosted/private-network opt-in for SearXNG-style providers, additive SDK exports, and tests proving public providers block private redirects while self-hosted SearXNG remains usable.
