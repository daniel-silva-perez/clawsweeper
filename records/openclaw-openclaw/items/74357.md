## Summary

Keep open. Current main still routes the shared web-search helper through `withTrustedWebToolsEndpoint`, whose policy sets `dangerouslyAllowPrivateNetwork: true`; the same helper is used by the named bundled web-search providers and exported through the plugin SDK. The latest release tag `v2026.4.26` contains the same policy, so this is not already fixed or shipped.

## What This Changes

Request to split or tighten the web-tools SSRF policy so private-network access is only opt-in for self-hosted/configured endpoints such as SearXNG, while fixed public web-search provider endpoints stay strict.

## Best Possible Solution

Keep this open for security/maintainer triage. The likely code direction is to make the shared web-search helper strict by default, add an explicit self-hosted/private-network helper or option for SearXNG and similar configured endpoints, update the SDK additively, and add tests proving fixed public providers block private redirects while self-hosted SearXNG still works.
