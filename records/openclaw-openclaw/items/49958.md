## Summary

Keep this PR open. Current main still does not implement the intended Feishu group slash-command fix for admitted groups with no sender `allowFrom`, so the PR is not obsolete. The PR remains a plausible candidate for #49915, but the current patch should not merge as-is because it widens a shared command-authorization helper beyond the Feishu group case and leaves the root SDK CJS mirror out of sync. The earlier Greptile summary correctly identified the bug shape; the later review comments about DM fallback and SDK drift are supported by current main.

## Best Possible Solution

Keep this PR open for contributor or maintainer revision. The best path is a narrower Feishu group command-auth fix that treats an already admitted Feishu group with no sender `allowFrom` as unrestricted for that group command flow, while preserving open-DM non-authorization, keeping the root SDK CJS mirror aligned or generated, and adding direct Feishu group regression coverage plus DM negative coverage.

## Risks / Open Questions

- Merging the PR as currently described could authorize control commands in open DMs with no explicit allowlists, contradicting existing shared and Signal DM tests.
- Merging only the TypeScript helper would leave the root SDK CJS shim with different command authorization behavior for plugins importing `openclaw/plugin-sdk`.
- Closing this PR now would drop an active implementation candidate for still-open #49915; current main still reproduces the central no-`allowFrom` Feishu group command denial.
