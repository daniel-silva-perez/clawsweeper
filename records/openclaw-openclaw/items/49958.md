## Summary

Keep this PR open. Current main still has the Feishu group slash-command denial targeted by the PR, so the work is not obsolete or implemented on main. The PR is still a plausible implementation candidate for #49915, but the current patch needs revision because it broadens shared command authorization semantics beyond the Feishu group flow and leaves the root SDK CommonJS mirror inconsistent.

## Best Possible Solution

Keep this PR open for revision. The best path is a narrower Feishu group command-auth fix: after a Feishu group has already passed group admission, treat missing sender `allowFrom` as unrestricted for that group command flow, while preserving open-DM non-authorization, keeping the root SDK CJS mirror or generated equivalent aligned, and adding direct Feishu group regression coverage plus DM negative coverage.
