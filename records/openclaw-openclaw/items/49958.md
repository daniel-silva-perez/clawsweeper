## Summary

Keep this PR open. Current main still has the Feishu group slash-command denial that the PR targets, so the work is not obsolete. The PR is still a plausible implementation candidate for #49915, but the current patch should not merge as-is because it changes a shared command authorization helper for DM paths too and leaves the root SDK CommonJS mirror out of sync.

## Best Possible Solution

Keep this PR open for revision. The best path is a narrower Feishu group command-auth fix that treats an already admitted Feishu group with no sender `allowFrom` as unrestricted for that group command flow, while preserving open-DM non-authorization, keeping the root SDK CJS mirror or generated equivalent aligned, and adding direct Feishu group regression coverage plus DM negative coverage.
