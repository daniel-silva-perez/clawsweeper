## Summary

Keep this issue open. Current main still defaults both plugin discovery and manifest snapshot cache windows to 1000 ms, and the capability/web provider snapshot caches use the minimum of those values for expiration. I did not find a gateway-specific longer default or launcher env override in current source; the related ClawSweeper reports in the provided context are timeout/cache-adjacent but do not track this plugin cache default problem.

## Best Possible Solution

Keep the issue open for a maintainer/product decision. A focused fix should either raise the shared plugin cache defaults in `src/plugins/cache-controls.ts` and align `src/plugins/discovery.ts` / `src/plugins/manifest-registry.ts`, or introduce an explicit gateway-mode default/launcher env path. Add tests covering the chosen default and update plugin architecture/internal docs so operators know the intended cache window.
