## Summary

Current main still uses 1000 ms defaults for plugin discovery and manifest snapshot caches, applies the shortest TTL to capability/web provider snapshot memoization, and has no gateway launcher override. The related open PRs #73847 and #73853 address adjacent reload/cache-key problems but do not implement this default-policy request.

## Best Possible Solution

Keep this open for a focused maintainer cache-policy decision. The best implementation is to either raise the shared plugin discovery/manifest snapshot defaults or add a gateway-mode default/launcher env path, then align src/plugins/cache-controls.ts, src/plugins/discovery.ts, src/plugins/manifest-registry.ts, capability/web snapshot tests, and the public plugin internals docs with the chosen invalidation model. Coordinate with #73847 and #73853 because they address adjacent reload surfaces but do not replace this request.
