## Summary

Current main still defaults the shared plugin discovery and manifest snapshot cache controls to 1000 ms, still applies that shortest TTL to capability and web-provider snapshot caches, and has no gateway launcher/default override. The related open PRs #73847 and #73853 address adjacent cache-miss/reload paths, but neither is merged nor directly implements this default-policy request.

## Best Possible Solution

Keep this open for a focused maintainer decision on cache policy. The best implementation is either to raise the shared plugin discovery/manifest snapshot defaults or add an explicit gateway-mode default/launcher env path, then align src/plugins/cache-controls.ts, src/plugins/discovery.ts, src/plugins/manifest-registry.ts, capability/web snapshot tests, and the public plugin internals docs with the chosen invalidation model. Coordinate with #73847 and #73853 because they address adjacent cache-miss/reload surfaces but do not replace this default-policy work.
