## Summary

Keep this issue open. Current main still defaults both plugin discovery and manifest snapshot cache controls to 1000 ms, the capability and web provider snapshot paths still use the shorter of those two values for expiration, and I did not find a gateway-specific longer default or launcher env override. The related open reports #73729 and #73730 are adjacent cache-miss problems, not canonical replacements for this default-policy question.

## Best Possible Solution

Keep the issue open for a maintainer decision. A focused fix should either raise the shared plugin discovery/manifest snapshot defaults or add an explicit gateway-mode default/launcher env path, then cover `resolvePluginSnapshotCacheTtlMs` and the capability/web provider snapshot caches with targeted tests and update plugin internals docs to document the intended gateway cache window.
