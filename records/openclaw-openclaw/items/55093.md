## Summary

Keep this PR open. Current main does not already implement the requested dirty local version marker or automatic runtime facade entry discovery, and the PR is useful but incomplete because it writes a `displayVersionMarker` into `build-info.json` without threading that marker into version resolution or `openclaw --version` output.

## What This Changes

The PR adds recursive tsdown entries for `src/**/*.runtime.ts`, refactors `scripts/write-build-info.ts` into testable exports with dirty-worktree marker inference, and adds focused build-info and tsdown config tests.

## Best Possible Solution

Finish the PR by defining the dirty marker display contract, reading the marker from `build-info.json`, formatting it in the root version output, and keeping the recursive runtime facade entry test so future runtime files are covered automatically.
