## Summary

Keep open: current main still has a scripts/ui Windows shell spawn path that can trigger the reported deprecation, while the proposed fix would break safe forwarded arguments containing spaces. The linked gateway issue #62881 was closed separately for Codex app-server startup, but that does not resolve this scripts/ui surface.

## What This Changes

The PR refactors scripts/ui.js to fold Windows shell-runner args into the command string before spawn/spawnSync and adds a prepareSpawnCommand unit assertion.

## Best Possible Solution

Revise scripts/ui.js to reuse the shared cmd.exe command-line escaping path, preferably buildCmdExeCommandLine() or the pnpm-runner seam, so Windows .cmd/.bat/.com launches avoid shell:true plus a separate args array while preserving each forwarded argument. Add focused tests for Program Files launcher paths, no DEP0190-triggering spawn shape, unsafe metacharacter rejection, and a safe forwarded argument containing spaces.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve forwarded argument boundaries:** `scripts/ui.js:91`
  - body: The shell path now builds the command with `${quotedCmd} ${args.join(" ")}`. That removes the DEP0190 args array, but a safe forwarded arg containing spaces, such as `node scripts/ui.js test -- -t "path with spaces"`, is split by cmd.exe before it reaches pnpm. Reuse `buildCmdExeCommandLine()` or the pnpm runner wrapper so each argument is escaped before spawning.
  - confidence: 0.86
