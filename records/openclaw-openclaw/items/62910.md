## Summary

Keep this PR open. Current main still has the Node v24 DEP0190-triggering `spawn`/`spawnSync` shape for Windows `.cmd`/`.bat`/`.com` UI runner launches, so the PR is not obsolete. The submitted patch is not merge-ready because it changes a security-sensitive Windows shell construction path by joining an unquoted command path and `args.join(" ")`, while adding no regression coverage for Program Files paths, arguments with spaces, or the final spawn invocation shape.

## Best Possible Solution

Keep this PR open and request a revision that reuses the existing Windows command-line helper or shared pnpm runner shape: `cmd.exe /d /s /c buildCmdExeCommandLine(...)`, `shell: false`, and `windowsVerbatimArguments: true` for Windows `.cmd` launchers. The final patch should preserve Program Files launcher paths, preserve safe forwarded args with spaces, keep unsafe metacharacter rejection, and add tests for the final `spawn`/`spawnSync` invocation shape.

## Risks / Open Questions

- Current main can still emit DEP0190 on Node v24 for Windows UI runner launches through `scripts/ui.js`.
- Merging the PR as written could break `.cmd` launcher paths containing spaces, such as under `C:\Program Files`, because the raw command path is folded into a shell command string without quoting.
- Merging the PR as written could split otherwise safe forwarded arguments containing spaces, because `args.join(" ")` does not preserve argument boundaries.
- The touched path constructs shell commands, so unsafe metacharacter rejection is necessary but not enough; quoting and final spawn/spawnSync invocation shape need regression tests.
