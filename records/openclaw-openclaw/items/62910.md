## Summary

Keep this PR open. Current main still has the Node v24 DEP0190-triggering Windows UI runner spawn shape, but the submitted patch is not merge-ready because it replaces Node's implicit space join with an explicit args.join(" ") command string instead of using the repo's existing cmd.exe quoting helper that preserves argument boundaries.

## What This Changes

The PR refactors scripts/ui.js so Windows shell-runner launches fold args into a single command string before spawn/spawnSync, and adds a prepareSpawnCommand unit test.

## Best Possible Solution

Revise scripts/ui.js to route Windows .cmd/.bat/.com UI runner calls through the existing cmd.exe wrapper/escaping seam, preferably scripts/pnpm-runner.mjs or buildCmdExeCommandLine(), with shell:false and windowsVerbatimArguments:true where appropriate. Add targeted tests that assert the final spawn/spawnSync shape for Program Files launcher paths, no DEP0190-triggering args array, unsafe metacharacter rejection, and a forwarded argument containing spaces.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve forwarded arg boundaries:** `scripts/ui.js:91`
  - body: The new shell path builds the command with `${quotedCmd} ${args.join(" ")}`. A safe forwarded arg containing spaces, such as `node scripts/ui.js test -- -t "path with spaces"`, is split by cmd.exe before it reaches pnpm. Reuse `buildCmdExeCommandLine()` or the pnpm runner wrapper so each argument is escaped before spawning.
  - confidence: 0.84
