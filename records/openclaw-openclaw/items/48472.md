## Summary

Keep this PR open. Current main still does not implement the PR's central changes: daemon read probes still call the heavier status RPC, probeGateway has no health-only detail mode, restart reachability probes still use a fixed 3 second timeout without a shrinking retry budget, and daemon status URL overrides are not wired through the strict explicit-auth path proposed here. The merged #72405 work only covers local-vs-remote probe credential separation, so it does not fully supersede this PR.

## Best Possible Solution

Keep this PR open for maintainer judgment or ask the contributor to rebase and narrow it. The best path is a focused update that adds a health-only probe/detail mode, uses it only for daemon status/read probes where appropriate, adds deadline-bounded restart probe timeouts with tests for both restart and listener waits, and explicitly resolves the override URL, insecure ws://, and device identity/token fallback security questions while preserving #72405's local probe auth separation.
