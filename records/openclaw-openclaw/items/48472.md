## Summary

Keep this PR open. Current main still does not implement the PR’s central behavior: daemon read probes still use `callGateway(... method: "status")`, `probeGateway` has no `detailLevel: "health"`, restart reachability still uses a fixed 3 second probe timeout, and daemon status URL overrides still pass daemon-resolved credentials without the strict explicit-auth boundary proposed here. The merged #72405 work is related but only covers local-vs-remote probe credential separation.

## Best Possible Solution

Keep this PR open for maintainer judgment or ask the contributor to rebase and narrow it. The best path is a focused update that adds a health-only probe/detail mode, uses it only for daemon status/read probes where appropriate, adds deadline-bounded restart probe timeouts with tests for both restart and listener waits, and explicitly resolves the override URL, insecure ws://, and device identity/token fallback security questions while preserving #72405’s local probe auth separation.
