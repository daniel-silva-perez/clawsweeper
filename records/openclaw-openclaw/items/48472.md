## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.8

Full review comments:

- **[P1] Disable device identity for unauthenticated remote probes:** `src/gateway/probe.ts:50-55`
  - body: `disableDeviceIdentity` only becomes true for unauthenticated loopback URLs, so a non-loopback probe with no token/password leaves `deviceIdentity` enabled. `GatewayClient` can then load a cached device identity/device-token fallback and send it to the probe URL; make probes opt out by default and opt in only for trusted loopback or explicitly pinned targets.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The PR changes gateway auth and probe transport paths and still has concrete device-token and plaintext ws:// policy concerns to resolve before merge.

Concerns:

- **[high] Remote probes can use cached device identity:** `src/gateway/probe.ts:50`
  - body: The proposed probe logic disables identity only for unauthenticated loopback probes. Non-loopback probes without explicit shared auth can still use the stored device identity/device-token fallback, which is unsafe for untrusted or misdirected URLs.
  - confidence: 0.84
- **[high] Private ws break-glass accepts arbitrary hostnames:** `src/gateway/net.ts:500`
  - body: isSecureWebSocketUrl returns true for any hostname when OPENCLAW_ALLOW_INSECURE_PRIVATE_WS is enabled, so the new daemon override guard inherits a plaintext remote-host exception broader than the comment describes.
  - confidence: 0.78
