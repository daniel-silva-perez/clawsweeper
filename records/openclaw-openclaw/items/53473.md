## Summary

Keep open. Current main still lacks the improved local-embedding doctor and FAQ guidance, and this PR is useful narrow work, but the proposed auto-mode hint conflicts with the runtime and reference-doc contract for local auto-selection, so the patch should not merge as-is.

## What This Changes

The branch changes `openclaw doctor` memory-search messages and FAQ text for local GGUF/node-llama-cpp embeddings, adds regression assertions, and reformats one subagent workspace test block.

## Best Possible Solution

Land a narrow version that aligns doctor guidance, helper behavior, tests, and FAQ text with the memory embedding contract: explicit `provider = "local"` may use the default downloadable GGUF or `hf:` refs, `provider = "auto"` should only present local as ready for an existing local file, Ollama remains explicit, and pnpm build-approval guidance stays scoped to node-llama-cpp.
