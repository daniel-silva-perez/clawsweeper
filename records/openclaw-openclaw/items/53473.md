## Summary

Keep open. Current main still lacks the proposed guidance, so this PR is not obsolete, but the branch remains incorrect because its auto-mode remediation tells users an `hf:` model path works with `provider = "auto"` even though runtime auto-selection and reference docs require an existing local file.

## What This Changes

The PR updates `openclaw doctor` memory-search guidance and FAQ text for local GGUF/node-llama-cpp embeddings, adds regression assertions, and includes formatting churn in one subagent workspace test.

## Best Possible Solution

Land a narrow version that keeps the useful GGUF/node-llama-cpp and pnpm-approval guidance, removes `hf:` from the auto-mode local-provider hint and test expectation, keeps Ollama explicit, and avoids unrelated formatting churn outside the doctor/docs/test surface.
