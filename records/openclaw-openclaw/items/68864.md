## Summary

Keep open. Current main still answers the Control UI checkpoint list, detail, branch, restore, and session-row count from stored `entry.compactionCheckpoints`; it does not reconstruct historical checkpoint metadata from existing `.checkpoint.*.jsonl` files. The directly related fix PR #68883 was closed unmerged, so the reported behavior remains actionable.

## What This Changes

The issue asks for the Web UI session checkpoint list to show and use historical compaction checkpoint files that exist on disk even when session-store checkpoint metadata is missing.

## Best Possible Solution

Add a narrow gateway-side reconciliation path that discovers valid checkpoint transcript files for the requested session when stored checkpoint metadata is empty or stale, reuses the existing filename parser and transcript validation, and makes list/get plus branch/restore work consistently for historical sessions.
