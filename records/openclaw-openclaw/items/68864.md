## Summary

Keep open. Current main still derives the Control UI checkpoint list, count, and lookup paths from `entry.compactionCheckpoints` in the session store; there is no gateway fallback that reconstructs checkpoint metadata from `.checkpoint.*.jsonl` files on disk. The directly related fix PR #68883 was closed unmerged, and current tests only cover checkpoints already present in store metadata.

## What This Changes

The issue asks for existing historical pre-compaction checkpoint files on disk to appear in the Control UI session checkpoint list and remain inspectable/restorable.

## Best Possible Solution

Add a narrow gateway-side reconciliation path that discovers valid checkpoint transcript files for the requested session when store metadata is empty or stale, reuses the existing checkpoint filename parser, and makes list/get plus branch/restore behave consistently for historical sessions.
