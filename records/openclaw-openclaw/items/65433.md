## Summary

Keep open. Current main still lacks two of the PR's three intended behavior changes, and the proposed Sonnet 4.6 fallback has a concrete correctness issue in model-only context lookup paths.

## What This Changes

The PR strips NUL bytes from generic untrusted JSON strings, changes Google Gemini CLI forward-compatible template IDs to prefer 3.1 preview IDs with fallbacks, and adds default 1M context handling/tests for anthropic/claude-sonnet-4-6.

## Best Possible Solution

Keep the useful hardening idea, but rework the branch before merge: land the remaining generic NUL sanitizer gap separately, coordinate or supersede the Gemini template change with #67758 after current catalog validation, and add Sonnet 4.6 1M behavior only behind an unambiguous provider source while preserving configured overrides.

## Reproduction Assessment

Yes. The review path is static and high-confidence: current main lacks the Gemini and Sonnet fallback behavior, and the PR diff would make model-only Sonnet 4.6 calls take a provider-specific 1M fallback despite the existing model-only guard pattern.

## Solution Assessment

No. The NUL sanitizer line is narrow, but the combined PR is not the best landing shape because the Sonnet fallback needs an explicit-provider guard and the Gemini portion needs current provider/catalog validation.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Gate the Sonnet fallback on explicit provider:** `src/agents/context.ts:432-434`
