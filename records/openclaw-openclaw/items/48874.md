## Summary

Current main has adjacent session primitives, but it does not implement the RFC’s central architecture: default direct-message routing still collapses to main unless dmScope is changed, session entries/API do not expose active/archive/pin/tag/channel-binding lifecycle, delete archive is transcript-renaming rather than restorable archived sessions, Control UI New Session still resets via /new on main, and memory/RAG remains per-agent or opt-in rather than a new public knowledge layer.

## Best Possible Solution

Keep this issue open as an umbrella/design thread, or split it into smaller maintainer-owned issues: canonical session identity/routing policy, first-class active/archive/pin/tag lifecycle and archive/restore APIs, Control UI create/archive/restore/pin workflows, and a scoped shared memory/RAG contract. Review PR #73725 as a narrow WebChat session-create slice, and treat the linked global-architecture fork discussion as prior art rather than a completed implementation.
