## Summary

This issue should stay open. Current main has useful adjacent session routing, dashboard session APIs, Control UI session listing, and memory search, but it does not implement the RFC’s central architecture: direct-message routing still collapses to main by default, session entries/API do not expose active/archive/pin/tag/channel-binding lifecycle, Control UI New Session still sends /new rather than creating a preserved session on main, archive/delete is not a restorable archived-session layer, and memory search remains agent-workspace/plugin scoped rather than a new public knowledge base contract. The discussion and Artyomkun’s linked global-architecture fork are useful prior art, not a completed OpenClaw implementation.

## Best Possible Solution

Keep this issue open as an umbrella/design thread, or split it into smaller maintainer-owned issues: canonical session identity and dmScope/mainKey routing policy; first-class active/archive/pin/tag lifecycle and archive/restore APIs; Control UI create/archive/restore/pin workflows; and a scoped shared memory/RAG contract. Review and either land or close PR #73725 as the narrow WebChat create/switch slice, while treating the linked global-architecture fork discussion as prior art rather than a completed implementation.
