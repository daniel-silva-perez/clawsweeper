## Summary

Keep this PR open. Current main still hardcodes `disableBlockStreaming: true` on the Mattermost draft-preview reply path, while the sibling Mattermost paths derive it from `account.blockStreaming`. The PR is a focused one-file implementation candidate for the Mattermost streaming portion of open bug #70253, not work already implemented on main.

## Best Possible Solution

Keep this PR open for normal maintainer review. The strongest path is to add a focused regression assertion in `extensions/mattermost/src/mattermost/monitor.test.ts` or a small extracted helper test that pins `account.blockStreaming: true` to `disableBlockStreaming: false` for the draft-preview path, confirm the intended Mattermost preview-vs-block-streaming behavior, then land this one-line fix if that behavior is desired. Keep #70253 open until its broader agent-to-agent routing report is separately resolved.
