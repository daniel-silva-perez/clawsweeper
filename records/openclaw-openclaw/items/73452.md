## Summary

Keep this PR open. Current main still hardcodes `disableBlockStreaming: true` in the Mattermost draft-preview reply path, while the sibling Mattermost reply paths and slash-command path already derive the value from `account.blockStreaming`. The PR is a focused implementation candidate for the Mattermost streaming-config portion of open bug #70253, not work already implemented on main.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is to add a focused Mattermost regression assertion, either in `extensions/mattermost/src/mattermost/monitor.test.ts` or through a small extracted helper, that pins `account.blockStreaming: true` to `disableBlockStreaming: false` for the draft-preview dispatch path; then land this one-line fix if maintainers agree config should override the preview-path suppression. Keep #70253 open until its broader agent-to-agent routing report is separately resolved.
