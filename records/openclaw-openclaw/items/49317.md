## Summary

Keep this issue open. Current main at e583db63c602d8d21b4ebdcd119061571a2e40a9 still has the reported WhatsApp mention path: `isBotMentionedFromTargets()` derives self-chat mode from `allowFrom`, checks normalized mention identity overlap only when self-chat is false, and leaves the `hasMentions && isSelfChat` branch as a non-detecting path. Group gating passes the configured WhatsApp allowlist into that helper and uses the resulting `wasMentioned` value for activation. The provided GitHub context also shows open, unmerged PR #73453 references this issue with closing syntax, so policy says the issue should remain open until that implementation candidate lands or is rejected.

## Best Possible Solution

Keep this issue open and review PR #73453 as the current implementation candidate. The fix should live in `extensions/whatsapp/src/auto-reply/mentions.ts` or its caller contract: group conversations should still compare normalized mention identities against the bot identity even when the bot's own E.164 appears in `allowFrom`, while preserving intentional 1:1 self-chat suppression. Add a regression test covering the reporter's LID/group/self-in-allowFrom scenario, then close this issue only after that PR or an equivalent fix lands.
