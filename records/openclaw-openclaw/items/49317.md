## Summary

Keep this issue open. Current main still derives WhatsApp mention self-chat mode from `allowFrom`, skips normalized mention identity matching when that flag is true, and routes group activation through that result. Provided GitHub context also shows open, unmerged PR #73453 uses closing syntax for this issue, so the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review PR #73453. The fix should preserve true 1:1 self-chat suppression while ensuring group conversations still compare normalized mention identities against the bot identity when the bot's own E.164 appears in `allowFrom`. Add a regression test for a group LID/@mention matching self with self present in `allowFrom`, then close this issue after the PR or an equivalent fix lands.
