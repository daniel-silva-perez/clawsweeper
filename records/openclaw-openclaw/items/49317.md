## Summary

Keep this issue open. Current main still derives self-chat mode from `allowFrom` for the mention-detection helper, and the identity-overlap check for WhatsApp mentions only runs when that self-chat flag is false. The exact group/self-allowlist case is being handled by open, unmerged PR #73453, which uses closing syntax for this issue, so the issue should remain open until that implementation is merged or rejected.

## Best Possible Solution

Keep the issue open and review PR #73453 as the implementation candidate. The desired fix belongs in `extensions/whatsapp/src/auto-reply/mentions.ts`: group conversations should still compare normalized mention identities against the bot identity even when the bot's own E.164 is present in `allowFrom`, while preserving true 1:1 self-chat suppression. Close this issue only after that PR, or an equivalent fix plus regression test, lands.
