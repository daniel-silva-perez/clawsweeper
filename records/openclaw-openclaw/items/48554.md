## Summary

Keep open. Current main still lacks FormData normalization in the proxy fetch wrapper, and an open contributor PR already targets the reported proxy-specific multipart failure with closing syntax, so the issue should remain open until that PR or a replacement lands.

## What This Changes

The issue requests that proxy-backed fetch preserve multipart FormData bodies for audio transcription and batch uploads when HTTP_PROXY or HTTPS_PROXY is configured.

## Best Possible Solution

Review, rebase, land, or replace #48556 with a narrow proxy-fetch fix that normalizes global/FormData-like bodies before undici dispatch, avoids double-converting undici FormData, clears stale multipart body headers when conversion happens, and adds regression coverage for both proxy fetch wrappers.
