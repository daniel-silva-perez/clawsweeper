## Summary

Keep open. Current main still has the reported proxy-specific multipart gap, and an open contributor PR already targets this issue with closing syntax, so the issue should remain open until that PR or a replacement lands.

## What This Changes

Fix proxy-backed fetch wrappers so audio transcription and batch upload FormData bodies serialize as multipart/form-data instead of the literal string [object FormData] when HTTP_PROXY or HTTPS_PROXY is configured.

## Best Possible Solution

Land or replace #48556 with a narrow proxy-fetch fix that normalizes global FormData before undici dispatch, keeps existing proxy selection semantics intact, and adds regression coverage for both makeProxyFetch and resolveProxyFetchFromEnv with global and undici FormData bodies.
