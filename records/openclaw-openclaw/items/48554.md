## Summary

Keep open. Current main still has the proxy-specific multipart gap: the proxy fetch wrappers pass caller bodies directly into undici, while the audio transcription path still builds global FormData and can receive that proxy fetch. An open contributor PR already targets this issue with closing syntax, so the issue should stay open until that PR or a replacement lands.

## What This Changes

Fix proxy-aware fetch wrappers so global FormData bodies remain multipart/form-data under HTTP or HTTPS proxy for audio transcription and batch uploads.

## Best Possible Solution

Land or replace #48556 with a narrow proxy-fetch fix that normalizes global FormData before undici dispatch, avoids double-converting undici FormData, removes stale multipart body headers when needed, and adds regression coverage for both makeProxyFetch and resolveProxyFetchFromEnv.
