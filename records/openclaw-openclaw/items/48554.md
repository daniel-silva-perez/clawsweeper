## Summary

Keep open. Current main still has the proxy-specific FormData gap, and this issue is paired with an open contributor PR that uses closing syntax, so it should remain open until that fix is reviewed, landed, or replaced.

## What This Changes

Fix proxy-aware fetch so multipart FormData bodies used by audio transcription and uploads are not serialized as `[object FormData]` when proxy env vars route requests through undici.

## Best Possible Solution

Review, rebase, land, or replace #48556 with a narrow proxy-fetch fix that normalizes global/FormData-like multipart bodies before undici dispatch in both proxy wrapper entry points, preserves existing undici FormData, clears stale multipart content headers when rebuilding, and adds proxy-fetch regression coverage.

## Reproduction Assessment

Yes. The issue includes a focused FormData/undici reproduction script, and current main still follows the same source path from global FormData through resolveProxyFetchFromEnv() into undiciFetch without normalization.

## Solution Assessment

Yes. Fixing the proxy-fetch wrapper is the narrowest maintainable boundary because callers are already correct to use standard FormData, and the wrapper is what swaps in undici under proxy configuration.
