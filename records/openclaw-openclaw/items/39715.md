## Summary

Keep this PR open. Current main still resolves child wait and snapshots exec/background output without a stdout/stderr drain turn, and the current tests do not cover the block-buffered, flush-at-exit background-output case this PR targets. The related #43876 fixed a sibling stdin/write-payload path, not this stdout/stderr capture path.

## Best Possible Solution

Keep this PR open. The best maintainer path is to land or adapt a minimal stdout/stderr drain before RunExit and finished background-session aggregate snapshots, preserve the settled/cancel/finalize safeguards already discussed on the PR, and add focused regression coverage for backgrounded stdout/stderr flushed at process exit.
