## Summary

Keep open. Current main still does not implement the PR's central stdout/stderr drain before exec completion and finished background-session snapshots, and current tests do not cover the block-buffered, flush-at-exit background-output case. The PR remains a focused candidate for the stdout half of #30711; #43876 addressed a sibling stdin/write-payload issue, not this output-drain path.

## Best Possible Solution

Keep this PR open. The best maintainer path is to land or adapt a minimal stdout/stderr drain before RunExit and finished-session aggregate snapshots, preserve the settled/cancel/finalize safeguards already discussed on the PR, and include focused regression coverage for backgrounded stdout/stderr flushed at process exit.

## Evidence

There is no setImmediate or equivalent drain turn before the output snapshot.
