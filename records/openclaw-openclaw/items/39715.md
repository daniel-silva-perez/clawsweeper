## Summary

Keep open. Current main still does not implement this PR's central stdout/stderr drain before background exec completion is snapshotted, and the PR remains a plausible focused fix rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open. The best maintainer path is to land or adapt a minimal stdout/stderr drain before completion and finished-session aggregation are snapshotted, retain the later cancel/finalize race safeguards from the PR discussion, and keep focused regression coverage for backgrounded stdout/stderr flushed at process exit.
