## Summary

Current main still does not fix the direct `acpx codex sessions new` stale-record case. OpenClaw has adjacent cleanup for OpenClaw-owned ACP task sessions, but the bundled ACPX plugin remains pinned to `acpx@0.6.1`, the wrapper delegates lifecycle to `acpx/runtime`, and the pinned direct CLI still creates/displays stored `closed` and `pid` metadata without making `sessions show` reconcile against the process table.

## Best Possible Solution

Keep this open. Fix the direct session lifecycle reconciliation in `openclaw/acpx` so `sessions new`, `sessions ensure`, `sessions show`, and `status` reconcile `closed`, `pid`, and last-exit fields against the actual agent/queue-owner lifecycle; then publish acpx, bump `extensions/acpx` to that release, and add focused regression coverage in this repo for the consumed behavior.
