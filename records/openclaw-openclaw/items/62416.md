## Summary

Keep open. This is a recent, actionable core Control UI accessibility report, and current main only has partial accessibility treatment in the Settings surfaces. The related closed aria-live work covers chat transcript announcements, not the Settings/Config panel’s labels, grouping, focus, and generated form controls.

## Best Possible Solution

Keep this open as core Control UI accessibility work. Audit Settings, Quick Settings, and Advanced Config with keyboard navigation plus at least one screen reader; add semantic grouping, associated labels/ids/descriptions, aria-pressed or native radio semantics for segmented choices, live/status treatment for settings state changes, and focused DOM tests across `ui/src/ui/views/config-quick.ts`, `ui/src/ui/views/config.ts`, and `ui/src/ui/views/config-form.node.ts`. Treat #46504 as related prior work, not as a duplicate.
