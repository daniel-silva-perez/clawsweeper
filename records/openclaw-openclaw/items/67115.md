## Summary

Keep this PR open. Current main still short-circuits Darwin sessions with lingering SSH variables before reaching the macOS `open` branch, and the linked bug remains concrete and open. The PR is narrow and does not introduce a supply-chain-sensitive change, but it should be landed or replaced with focused regression coverage and current-main browser-open hardening preserved.

## What This Changes

This PR changes `src/infra/browser-open.ts` so the SSH/no-display guard no longer applies on Darwin and adds a comment explaining the macOS/Tailscale SSH environment case.

## Best Possible Solution
