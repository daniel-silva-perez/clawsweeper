## Summary

Keep open. Current main has useful compaction-retry, liveness, and empty-response safeguards, but the reported long-context gateway/TUI delivery loss is not proven fixed; source inspection still shows partial retry-state handling and related open reports cover overlapping reply-obligation and delivery-barrier gaps.

## What This Changes

The issue requests that automatic compaction during an in-progress agent turn preserve/resume the turn and deliver the final assistant output instead of silently dropping it.

## Best Possible Solution

Land a regression-backed compaction-resume path that tracks the active turn/reply obligation across local auto-compaction and proves post-compaction tool events plus final assistant output traverse the normal gateway/WebSocket/TUI delivery path. Coordinate with #66126 for turn identity and #47335 for the retry delivery barrier rather than closing this as already implemented.

## Reproduction Assessment

Unclear for current main. The issue and comment provide credible logs plus a deterministic high-context vLLM trigger, but this read-only review did not run that live scenario and no in-repo test reproduces the active-client output loss after compaction.

## Solution Assessment

No. The current retry wait, liveness, and empty-response retry plumbing are useful mitigations, but the best fix is a turn/reply-obligation and delivery-barrier change with an end-to-end regression for the reported TUI/gateway disappearance.
