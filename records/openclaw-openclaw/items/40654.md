## Summary

Current main still does not implement this PR's intended behavior. `/acp close` closes the ACP session and unbinds session bindings, but it does not identify ACP-created Discord child threads or request a post-reply archive. The PR remains useful, though it needs to be ported from the old Discord/core outbound layout to the current `extensions/discord` plugin layout and reviewed for a guarded archive side effect.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. The best maintainer path is to port it onto current main by recording ACP-created Discord child-thread ownership metadata during ACP thread binding, having `/acp close` request archive-after-reply only after close/unbind succeeds for the current owned Discord thread, implementing the post-reply archive in `extensions/discord`, and adding focused tests for ACP-created threads, manual/reused threads, non-Discord bindings, archive failures, degraded close notices, and outbound send overrides.
