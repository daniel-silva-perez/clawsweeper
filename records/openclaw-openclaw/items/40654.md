## Summary

Keep this PR open. The requested Discord ACP-thread cleanup is still missing on current main, but the branch targets the pre-plugin Discord layout and puts Discord-specific close behavior in ACP core, so it should be ported or replaced rather than merged as-is.

## What This Changes

The PR records ACP-created Discord thread ownership metadata and archives only those owned threads after `/acp close` sends its close reply.

## Best Possible Solution

Port the useful behavior onto current main with an extension-safe close hook or payload contract, store ACP-created child-thread ownership in binding metadata, let `extensions/discord` archive the owned thread after the close reply, preserve runtime notices and partial-failure text, and add focused ACP close plus Discord outbound/thread-binding tests and a credited changelog entry.
