## Summary

Keep this PR open. Current main still has the non-delivery exec-approval followup path that can omit a channel hint when the local helper cannot classify the originating channel, while elevated exec availability depends on the followup run inheriting `defaults.messageProvider`; the PR is a narrow implementation with no blocking review finding, but it still needs maintainer review and CI rather than cleanup closure.

## What This Changes

The PR threads `turnSourceChannel` into exec-approval followup argument construction, uses it as the non-delivery channel fallback, adds a regression test, and adds a changelog fix entry for the linked elevated exec approval bug.

## Best Possible Solution

Have a maintainer review and land the focused fallback if CI is green, ideally with test coverage that proves the process-boundary case where the followup helper lacks local channel recognition while the gateway still accepts and applies the channel as the run message provider.
