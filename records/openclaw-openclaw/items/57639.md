## Summary

Keep this issue open. Current main still has only concurrency, depth, timeout, auto-archive, and active-child controls for subagents; I did not find a durable lifetime per-session spawn cap equivalent to the requested maxPerSession/maxTotal. The prior ClawSweeper comment is still consistent with current main at 230f7122dd5cd8a37a55e671254bd9be8278f6e8.

## Best Possible Solution

Keep the issue open and implement a real lifetime/session budget if maintainers want this behavior. The likely path is a documented agents.defaults.subagents.maxPerSession or maxTotal field, schema/types/generated-docs support, a durable count by requester session key across native and ACP spawn paths, a clear sessions_spawn rejection once the cap is reached, and focused tests proving sequential completed children count differently from active children.
