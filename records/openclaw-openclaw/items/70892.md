## Summary

Keep open. Current main still only partially covers the request: existing agent skill allowlists can narrow the injected `<available_skills>` list, but the requested per-section prompt gates and the systemPromptOverride cookbook are still absent from source, schema, tests, and docs.

## What This Changes

The issue requests default-on prompt config gates for the CLI quick reference, reply-tag/directive guidance, messaging component guidance, plus a filter for injected available skills and docs for safe full system prompt overrides.

## Best Possible Solution

Maintainers should decide the supported prompt customization boundary, then either document the existing skill allowlists plus a safe override cookbook, or add narrowly scoped default-true prompt gates with schema, generated docs, user docs, and focused prompt-builder coverage.

## Reproduction Assessment

Not applicable as a feature/config request. The current-main verification path is source, schema, docs, and prompt-test inspection showing the requested keys are absent while the full prompt still includes the relevant sections.

## Solution Assessment

Unclear. The requested gates are plausible, but one part overlaps existing skill allowlists and the rest would create a new public prompt-section API, so the narrowest maintainable solution needs maintainer product judgment first.
