The issue requests default-on prompt config gates for the CLI quick reference, reply-tag/directive guidance, messaging component guidance, plus an include/exclude filter for injected available skills and docs for safe system prompt overrides.

## Best Possible Solution

Decide the supported product boundary first: either document the existing skill allowlists plus a safe systemPromptOverride cookbook, or add narrowly scoped default-true prompt gates with schema, generated docs, user docs, and focused prompt-builder coverage.
