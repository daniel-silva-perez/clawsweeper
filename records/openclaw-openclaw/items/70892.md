## Summary

Keep open. Current main already covers the include-style `<available_skills>` filtering case through `agents.defaults.skills` and per-agent `skills`, but the requested operator gates for the default prompt sections and the `systemPromptOverride` required-vs-optional cookbook are still missing from source, schema, tests, and docs.

## What This Changes

Request default-preserving agent prompt configuration gates for CLI quick reference, reply-tag/output-directive guidance, messaging component guidance, plus skills filtering and docs for safer full prompt overrides.

## Best Possible Solution

Add a default-preserving prompt-section config surface for the remaining full-prompt blocks, wire it through both embedded and CLI prompt builders, document the existing skills allowlist as the supported `<available_skills>` include filter, and add a concise override cookbook that separates required structural prompt pieces from optional sections.

## Reproduction Assessment

Not applicable as a runtime bug. Static current-main inspection is enough to verify the issue: the prompt renderer still emits the named blocks without operator section gates, while config/docs lack the requested prompt-gate and cookbook surfaces.

## Solution Assessment

Partly yes. Per-section gates are the narrowest maintainable fix for the prompt-cost problem, but the skills portion should reuse `agents.defaults.skills` / `agents.list[].skills` rather than adding a second overlapping include/exclude schema unless maintainers explicitly request subtractive filters.
