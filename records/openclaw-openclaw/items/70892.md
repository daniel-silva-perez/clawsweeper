## Summary

Keep open. Current main already covers include-style skill filtering through the existing agent skill allowlists, but it still does not expose operator-configurable gates for the requested full-prompt sections or the requested system-prompt override cookbook.

## What This Changes

The issue requests default-preserving prompt controls to omit CLI quick reference, reply-tag, and messaging-component guidance, plus a documented safe path for trimming `<available_skills>` and using `systemPromptOverride`.

## Best Possible Solution

Add a default-true prompt-section config surface for the remaining full-prompt blocks, wire it through the embedded and CLI prompt builders, keep skills filtering on the existing allowlist contract, and document the safe structural versus optional pieces for override users.

## Reproduction Assessment

Not applicable as a runtime bug. Static current-main inspection is enough to verify the missing surface: the named prompt blocks are still emitted without operator gates, while schema and docs lack the requested prompt-gate and cookbook fields.

## Solution Assessment

Partly yes. Per-section gates are the narrowest maintainable fix for the prompt-cost request, but the skills portion should reuse the existing allowlist instead of introducing a second include/exclude schema without maintainer approval.
