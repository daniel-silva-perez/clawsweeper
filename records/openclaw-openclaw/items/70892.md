## Summary

Current main still only partially covers the request. Existing `agents.defaults.skills` and `agents.list[].skills` can narrow the rendered `<available_skills>` list, but the requested per-section prompt gates for the CLI quick reference, reply/directive guidance, and Messaging component guidance are not present, and there is no `agents.defaults.prompt.availableSkills` include/exclude API or override cookbook that distinguishes structural from safely droppable prompt blocks.

## Best Possible Solution

Keep this open for maintainer API/product judgment. The best path is to decide whether the supported boundary should be the existing `agents.defaults.skills`/`agents.list[].skills` allowlists plus a documented `systemPromptOverride` cookbook, or a new default-true `agents.defaults.prompt.*` config surface. If new gates are accepted, update the prompt builder, config schema/types/help/generated docs, public docs, and focused prompt-rendering tests together.
