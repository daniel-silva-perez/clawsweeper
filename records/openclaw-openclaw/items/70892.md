## Summary

Keep open. Current main already has an include-style skill allowlist that covers the locked-down `<available_skills>` case, but it still lacks operator-configurable prompt-section gates for the CLI quick reference, reply/output directives, and messaging component guidance, and the docs still do not provide the requested safe `systemPromptOverride` cookbook.

## What This Changes

The issue requests default-preserving prompt-section gates for selected full-prompt blocks, a way to trim `<available_skills>`, and documentation that separates structural system-prompt pieces from optional ones.

## Best Possible Solution

Add a default-true prompt-section config surface for the remaining full-prompt blocks, wire it through both embedded and CLI prompt construction, keep skills trimming on the existing `agents.defaults.skills` / `agents.list[].skills` allowlist contract, and document structural versus optional prompt pieces for override users.

## Reproduction Assessment

Not applicable as a runtime bug. Static current-main inspection is enough to reproduce the gap: the named prompt blocks still render without operator gates, while schema/docs/tests only cover existing default behavior and the skills allowlist.

## Solution Assessment

Partly yes. Per-section gates are a narrow maintainable solution for the prompt-cost request, but a new `availableSkills.include/exclude` schema would duplicate the existing skill allowlist and should be avoided unless maintainers explicitly want a second skills filter.
