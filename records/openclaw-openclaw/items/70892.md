## Summary

Keep open. Current main partly covers the `<available_skills>` concern through existing agent skill allowlists, but it does not provide per-section config gates for the CLI quick reference, reply-tag/output-directive guidance, or Messaging interactive hints, and the `systemPromptOverride` docs still lack the requested structural-vs-optional cookbook.

## What This Changes

Request to add opt-in prompt-size controls for several hardcoded system-prompt blocks, filter available skills, and document safe `systemPromptOverride` building blocks.

## Best Possible Solution

Land a focused config/docs change that adds default-true prompt-section gates for the remaining hardcoded blocks, keeps structural prompt pieces and the cache boundary safe by default, documents existing skill allowlists as the supported include filter, and adds a concise `systemPromptOverride` cookbook for advanced deployments.

## Reproduction Assessment

Yes. Source inspection on current main shows the requested prompt gates are absent while full prompts still include the relevant blocks; existing tests assert unconditional inclusion of the CLI quick reference and skills behavior.

## Solution Assessment

Yes for the overall direction, with one adjustment: use the existing `agents.defaults.skills` / `agents.list[].skills` allowlist for include-style `<available_skills>` filtering, and only add subtractive exclude semantics if maintainers want that separate public contract.
