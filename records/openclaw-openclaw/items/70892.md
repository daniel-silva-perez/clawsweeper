## Summary

Current main partially overlaps with the request but does not implement the requested per-section prompt gates. The skills portion already has an include-style allowlist through agents.defaults.skills and agents.list[].skills, but CLI quick reference and Messaging still render directly in the full system prompt, and the docs only describe systemPromptOverride plus the existing heartbeat gate. This needs maintainer product/API judgment rather than cleanup closure.

## Best Possible Solution

Keep this open for maintainer API judgment. The likely path is either document the existing agents.defaults.skills allowlist as the supported available_skills filter and add narrowly scoped default-true prompt-section gates for the remaining optional blocks, or decide that systemPromptOverride plus existing skills allowlists is the intended boundary and document a safe override cookbook. Either way, tests should pin emitted/omitted sections and docs/schema/help should move together.
