## Summary

Keep open. ## What This Changes

The PR adds `GOOGLE_CLOUD_API_KEY` recognition for Google Vertex auth, secret and shell-env discovery, targeted tests, and model-provider documentation.

## Best Possible Solution

Rebase this PR onto current main, remove the obsolete core/static env-list edits, keep or refresh the Google Vertex docs so `GOOGLE_CLOUD_API_KEY` is documented alongside ADC, and retain only non-duplicative regression coverage that maintainers still want.

Yes. The PR-specific defect is visible from the diff because the added core candidate would shadow the manifest-owned provider metadata after rebase.

## Solution Assessment

No. The PR is no longer the best-shaped fix because current main already solved credential recognition through plugin metadata; the maintainable path is to keep Google Vertex auth metadata in `extensions/google/openclaw.plugin.json` and narrow this branch to docs and any still-useful tests.
