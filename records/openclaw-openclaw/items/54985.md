## Summary

Keep open. Current main does not implement the requested OpenClaw request-context headers, but the PR is not ready to merge because it forwards stable session/run identifiers to provider transports without a trust or opt-in boundary and still has incorrect fallback/turn metadata semantics; related #69463 is the closer opt-in observability direction.

## What This Changes

The PR adds OpenClaw session, attempt, fallback, and per-turn context headers to embedded LLM requests, propagates fallback state through model-fallback callers, and adds `isFallback` to plugin LLM output events.

## Best Possible Solution

Revise the design around an explicit, trust-scoped diagnostics/session-tracing configuration, derive fallback status from `ModelFallbackRunOptions` in every caller, keep a run-level monotonic model-request counter that survives embedded retries, and coordinate the public header contract with #69463 and #22021.

## Reproduction Assessment

Yes. Static review of the latest patch plus current main shows the issue paths: the PR writes headers into model/provider headers unconditionally, `runWithModelFallback` can skip candidates before the callback, and embedded retry `continue` paths can restart the same run before assistant output is saved.

## Solution Assessment

No. The proposed branch is directionally useful, but unscoped headers and local fallback/turn counters are not the narrowest maintainable solution; an opt-in trusted tracing surface with tested runner semantics is safer.

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Gate context headers to trusted gateway targets:** `src/agents/pi-embedded-runner/run/attempt.ts:2179-2193`
  - body: The new `gatewayContextHeaders` are merged into `params.model.headers`, so any provider transport that forwards model headers can receive stable OpenClaw session/run IDs during normal provider calls, not only through an operator-configured AI gateway. Make this opt-in or restrict it to trusted gateway/proxy targets before merge.
  - confidence: 0.86
- **[P2] Use fallback runner options for CLI fallback status:** `src/agents/agent-command.ts:520`
  - body: `runWithModelFallback` now computes `options.isFallback` from the candidate index, but this path still forwards `params.isFallbackRetry`, which counts only executed callbacks. If the primary candidate is skipped before invocation, the first executed fallback is labeled primary in the new header and hook metadata.
  - confidence: 0.88
- **[P2] Keep turn IDs unique across embedded retries:** `src/agents/pi-embedded-runner/run/attempt.ts:2454-2475`
  - body: `llmTurnCount` is local to one `runEmbeddedAttempt` and is seeded from persisted assistant messages. Inner retry paths can restart the attempt before an assistant message is written, so the next HTTP request can reuse the same `{runId}-turn-{index}` value even though the PR promises a unique turn ID per LLM HTTP request.
  - confidence: 0.82
