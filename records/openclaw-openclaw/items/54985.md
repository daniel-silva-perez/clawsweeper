## Summary

Keep this PR open. Current main does not implement the requested OpenClaw session/attempt/fallback/turn headers or `llm_output.isFallback`, so the contribution is not obsolete; however, the latest patch still needs maintainer review because it unconditionally forwards stable OpenClaw identifiers to outbound provider traffic and still has fallback-labeling and turn-id correctness issues called out by bot review discussion.

## What This Changes

The PR adds OpenClaw session, attempt, fallback, and per-turn context headers to embedded LLM requests, propagates fallback state through model-fallback callers, and adds `isFallback` to plugin LLM output events.

## Best Possible Solution

Keep the work open but require a revised design that makes request-context header propagation explicit and trust-scoped, derives fallback state from `ModelFallbackRunOptions` in every call path, uses a per-run monotonic call counter that survives embedded retries, and adds focused tests plus docs/changelog coverage coordinated with #69463 and #22021.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Gate context headers to trusted gateway targets:** `src/agents/pi-embedded-runner/run/attempt.ts:2179-2193`
  - body: The new `gatewayContextHeaders` are merged into `params.model.headers`, so any provider transport that forwards model headers can receive stable OpenClaw session/run IDs during normal provider calls, not only through an operator-configured AI gateway. That broadens outbound metadata and can leak cross-request correlation identifiers; make this opt-in or restrict it to trusted gateway/proxy targets before merge.
  - confidence: 0.86
- **[P2] Use fallback runner options for CLI fallback status:** `src/agents/agent-command.ts:520`
  - body: `runWithModelFallback` now computes `options.isFallback` from the candidate index, but this path still forwards `params.isFallbackRetry`, which counts only executed callbacks. If the primary candidate is skipped before invocation due to cooldown, the first executed fallback is labeled primary in the new header and hook metadata. Pass `runOptions?.isFallback` through this call path instead.
  - confidence: 0.88
- **[P2] Keep turn IDs unique across embedded retries:** `src/agents/pi-embedded-runner/run/attempt.ts:2458-2475`
  - body: `llmTurnCount` is local to one `runEmbeddedAttempt` and is seeded from persisted assistant messages. Inner retry paths can restart the attempt before an assistant message is written, so the next HTTP request can reuse the same `{runId}-turn-{index}` value even though the PR promises a unique turn ID per LLM HTTP request. Store the counter at the enclosing run level or otherwise carry it across retries.
  - confidence: 0.82

## Security Review
