## Summary

Keep this PR open. Current main does not implement the proposed loop-aware compaction instruction hook: there is no `agents.defaults.compaction.guard` schema/type surface, no `guardEnabled` runtime field, no transcript-tail detector/scorer modules, and the safeguard compaction path still resolves static/custom instructions without guard augmentation. The PR is also a draft in an active same-author stack tied to #48238, with prerequisite and follow-up PRs still open.

## Best Possible Solution

Keep this PR open. If maintainers still want the #48238 loop-aware saturated-session compaction guard, review and land or close the stack in order: #48278, #48293, #48312, then this PR, followed by #48350 and #48368 as appropriate. This PR should be rebased onto the current `src/agents/pi-hooks` compaction pipeline and reviewed specifically for bounded, sanitized handling of transcript-derived prompt text.
