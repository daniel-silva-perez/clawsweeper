## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Bound guard-derived failure signatures:** `src/agents/pi-extensions/compaction-instructions.ts:96`
  - body: The guard block appends `failure.signature` after the base instruction resolver has already applied its 800-character cap. Since signatures are derived from recent tool-result text and are not length-limited here, repeated long stderr/stdout patterns can massively expand `customInstructions` and give transcript text instruction-like weight during compaction. Truncate and wrap these diagnostics as untrusted data before appending them.
  - confidence: 0.86
- **[P2] Update guard help text after wiring runtime behavior:** `src/config/schema.help.ts:1053`
  - body: This branch wires `compaction.guard.enabled` into the safeguard runtime and conditionally augments summarization instructions, but the config help still says setting the guard has no runtime effect. If the stack ships like this, operators and generated help will describe the guard as inert even though it changes compaction prompts. Update the help/docs to match the runtime behavior.
  - confidence: 0.9
