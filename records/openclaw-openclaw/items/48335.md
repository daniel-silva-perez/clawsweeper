## Summary

Keep this PR open. Current main still does not implement the proposed loop-aware compaction guard hook: there is no `agents.defaults.compaction.guard` config contract, no `guardEnabled` safeguard runtime field, no transcript-tail detector/scorer module on main, and the current `session_before_compact` path only composes static/custom compaction instructions plus the existing quality guard. The provided GitHub context also shows this is a draft PR in an open same-author stack tied to #48238, #48278, #48293, #48312, #48350, and #48368, so it should be reviewed or closed as part of that stack rather than independently swept.

## Best Possible Solution

Keep this PR open. If maintainers still want the loop-aware saturated-session compaction guard from #48238, review the stack in order (#48278, #48293, #48312, then this PR) and rebase this PR onto the current `src/agents/pi-hooks` compaction pipeline. If product direction changed, close or redirect the whole stack with an explicit maintainer decision.
