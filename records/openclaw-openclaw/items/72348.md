## Summary

Keep this PR open: current main still has the Nextcloud Talk reaction dispatch gap, and the PR uses the right plugin-owned adapter seam, but the proposed handler has one blocking behavior bug because it ignores the shared `remove` reaction flag and can add a reaction when removal was requested.

## What This Changes

The branch adds and registers a Nextcloud Talk message action adapter, adds adapter and reaction sender unit coverage, and adds a changelog entry for the reaction dispatch bug.

## Best Possible Solution

Land this PR or an equivalent Nextcloud Talk-owned fix after the adapter preserves the shared reaction contract by handling or clearly rejecting removal requests, keeps the existing SSRF-guarded sender path, and reconciles the duplicate PR path so #70110 closes from one canonical merge.

## Reproduction Assessment

Yes. Current main can be reproduced by dispatching a Nextcloud Talk `message` tool `react` action: the channel advertises reactions but has no `actions.handleAction`, so dispatch returns unsupported instead of calling `sendReactionNextcloudTalk`; the PR-specific bug is visible by calling the new handler with `remove: true`.

## Solution Assessment

No, not as currently proposed. Registering a plugin-owned `ChannelMessageActionAdapter` is the right narrow seam, but the handler must not turn a removal request into an add request; rejecting unsupported removal is the safer minimal fix if DELETE support is intentionally out of scope.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Handle remove reactions before sending:** `extensions/nextcloud-talk/src/message-actions.ts:60-66`
  - body: `remove` is part of the shared `react` tool contract, but this handler never reads it. A call such as `{ remove: true, emoji: "👍" }` still calls `sendReactionNextcloudTalk` and returns `added`, performing the opposite of what the caller requested. If Nextcloud Talk removal is out of scope, reject `remove` explicitly before calling the sender.
  - confidence: 0.88
