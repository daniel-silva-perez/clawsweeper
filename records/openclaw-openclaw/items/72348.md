## Summary

Keep this PR open. Current main still advertises Nextcloud Talk reactions and contains the HTTP reaction sender, but the Nextcloud Talk channel plugin has no message `actions` adapter, so `message action=react` has no registered execution path. The PR remains a focused implementation candidate for linked bug #70110, and its diff is consistent with that scope.

## Best Possible Solution

Keep this PR open and review/land it, or land an equivalent Nextcloud Talk-owned fix that adds a `ChannelMessageActionAdapter`, registers it on the plugin, dispatches `react` to `sendReactionNextcloudTalk`, preserves the existing SSRF-guarded sender behavior, and keeps focused adapter plus sender tests. The linked bug #70110 should remain open until this PR or an equivalent fix merges.
