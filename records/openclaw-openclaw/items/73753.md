## Summary

Keep open. Current main contains the related #73709 fix for text-only Discord configs, but it still leaves the reported default voice-enabled path active: voice remains enabled unless explicitly disabled, `GuildVoiceStates` is requested in that default path, Carbon `VoicePlugin` is installed, and `DiscordVoiceManager`/`DiscordVoiceReadyListener` are loaded when voice is enabled. No current source, test, docs, or changelog evidence shows that the default voice-enabled idle CPU spin has been fixed.

## Best Possible Solution

Keep this issue open and investigate in `extensions/discord`: reproduce/profile gateway idle CPU with `channels.discord.voice.enabled=true`, isolate whether the loop is in Carbon `VoicePlugin`, `@discordjs/voice`, `DiscordVoiceManager` lifecycle/listeners, auto-join/session recovery, or gateway voice-state handling, then make enabled voice idle safely or make the runtime/default more explicit. Add focused regression or lifecycle/perf coverage around the identified loop and update Discord docs/changelog if defaults or behavior change.
