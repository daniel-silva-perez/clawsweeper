## Summary

Keep open. Current main has the related #73709 text-only intent fix, but it does not resolve the reported default voice-enabled CPU path: Discord voice remains enabled by default, `GuildVoiceStates` is still requested for that default path, and the Carbon voice plugin plus `DiscordVoiceManager` are still initialized when voice is enabled.

## Best Possible Solution

Keep the issue open and investigate in `extensions/discord`: reproduce/profile gateway idle CPU with `channels.discord.voice.enabled=true`, isolate whether the loop is in Carbon `VoicePlugin`, `@discordjs/voice`, voice-state handling, auto-join/session lifecycle, or OpenClaw listener wiring, then make enabled voice idle safely or make the runtime more explicitly opt-in. Add focused regression or lifecycle/perf coverage around the identified loop and update docs/changelog if defaults or behavior change.
