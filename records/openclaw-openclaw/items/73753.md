## Summary

Keep open. Current main includes the related #73709 text-only intent fix, but the reported default voice-enabled path remains active: Discord voice is still enabled by default, `GuildVoiceStates` is still requested in that mode, Carbon `VoicePlugin` is still registered, and `DiscordVoiceManager` is still loaded and wired on startup.

## Best Possible Solution

Keep this open and investigate inside `extensions/discord`: reproduce/profile idle CPU with `channels.discord.voice.enabled=true`, isolate whether the loop is in Carbon `VoicePlugin`, `@discordjs/voice`, voice-state events, auto-join/session lifecycle, or OpenClaw listener wiring, then make enabled voice idle safely or make the runtime more explicitly opt-in. Add focused lifecycle/perf regression coverage and update the Discord docs/changelog if defaults or behavior change.
