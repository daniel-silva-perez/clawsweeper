## Summary

Keep open. Current main still has the reported Discord empty-modal failure, but this PR needs revision before merge: its patch targets the pre-split parser body and introduces regressions by silently accepting malformed `modal.fields` values and by dropping container-only component styling when top-level message text supplies the body.

## What This Changes

The PR changes the Discord component parser to treat empty LLM-generated component defaults as absent and adds regression tests for empty blocks, empty text, empty modal fields, and a valid modal field case.

## Best Possible Solution

Apply the empty-default tolerance in the current Discord parser while preserving strict errors for malformed modal field types and preserving component specs that carry container or reusable metadata. Add targeted regression coverage for empty LLM defaults, malformed `modal.fields`, and container metadata with top-level text fallback, then coordinate with the broader #46455 sanitizer if both remain active.

## Reproduction Assessment

Yes. A current-main `sendMessage` call with non-empty object-shaped `components` containing `modal: { fields: [] }` reaches `readDiscordComponentSpec` and throws `components.modal.fields must be a non-empty array`; this follows directly from the send guard and parser source.

## Solution Assessment

No. The intended behavior is reasonable, but this implementation is not the best form because it drops validation and container metadata and needs to be rebased onto the current `components.parse.ts` split.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Reject malformed modal fields:** `extensions/discord/src/components.ts:600`
  - body: The new guard only parses modal fields when `fieldsRaw` is a non-empty array, so a present but malformed value like `fields: "name"` or `fields: {}` is silently treated as no modal. That regresses the current validation contract and can fall through to a plain message while discarding an intended form; keep empty arrays as absent, but still throw when `fields` is present and not an array.
  - confidence: 0.9
- **[P2] Preserve container-only component specs:** `extensions/discord/src/components.ts:623-624`
  - body: This null guard ignores `components.container`, but callers add top-level `content` as the component text fallback after parsing. A payload such as `components: { container: { accentColor: "#5865F2" } }` with message content will now return `null`, skip `sendDiscordComponentMessage`, and drop the intended container styling.
  - confidence: 0.87
