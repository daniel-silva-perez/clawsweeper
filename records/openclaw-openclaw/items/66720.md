## Summary

Keep open. Current main still does not implement the PR’s intended provider/gateway-specific `sessions_spawn` schema simplification: it only hides ACP-only fields when ACP is unavailable, while `lightContext`, `attachments`, and `attachAs` remain advertised in the schema. The branch remains a plausible implementation candidate, but it needs maintainer review around activation and tests.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is a provider/deployment-aware `sessions_spawn` schema property filter wired into the actual OpenAI-compatible gateway/model compatibility path, while preserving the default full schema and existing ACP availability hiding. Add tests for default, ACP-unavailable, and strict-gateway simplified schema cases before merge.

## Risks / Open Questions

- Merging an option-only implementation without activation in the actual provider/gateway compatibility path would leave affected OpenAI-compatible gateway deployments unfixed.
- Applying the simplified schema too broadly could hide useful native subagent fields such as `attachments`, `attachAs`, and `lightContext` from providers that can handle the full schema.
- The main remaining risk is functional, not supply-chain: the PR needs focused regression coverage for default full schema behavior, ACP-unavailable behavior, and strict-gateway simplified schema behavior.
