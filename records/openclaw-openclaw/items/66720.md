## Summary

Keep this PR open. Current main still does not implement provider/gateway-specific stripping of the `sessions_spawn` schema fields requested by the PR; it only hides ACP-only fields when ACP is unavailable, and the live tool assembly/provider compatibility paths do not remove `lightContext`, `attachments`, or `attachAs`. The PR remains a plausible implementation candidate for the related closed report, but it needs maintainer review/rework around activation and scope.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is a provider/deployment-aware `sessions_spawn` schema property filter wired into the actual OpenAI-compatible gateway/model compatibility path, while preserving existing ACP availability hiding and adding schema-shape tests for default, ACP-unavailable, and strict-gateway cases.
