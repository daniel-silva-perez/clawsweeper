## Summary

Keep this PR open. Current main still does not implement the PR's CaMeL engine, taint-propagating planner/quarantined extractor, agent config/schema surface, before-tool-call CaMeL policy enforcement, or the proposed exec-approval `source: "camel"` routing field. Existing main has prompt-injection hardening through external-content wrappers, provenance annotations, tool policy, sandboxing, and approvals, but that is not equivalent to this PR's opt-in CaMeL architecture. The PR is also XL and security-sensitive, with review discussion documenting unresolved design risks, so it needs explicit maintainer/security product review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for explicit maintainer/security product judgment. If OpenClaw wants CaMeL in core, the best path is a reviewed design that preserves isolated planner/extractor sessions, durable taint provenance through tool results and rewritten params, replay-safe side-effect handling, clear config semantics, gateway approval protocol/UX compatibility, docs, and focused regression tests. If maintainers prefer the current structural-wrapper direction, close only after linking a canonical replacement that covers the same opt-in CaMeL/taint-tracking security goal.
