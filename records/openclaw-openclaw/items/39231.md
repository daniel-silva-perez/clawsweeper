## Summary

Keep this PR open. Current main still does not implement the PR's CaMeL engine, taint-propagating planner/quarantined extractor, agent config/schema surface, before-tool-call CaMeL policy enforcement, or proposed exec-approval `source: "camel"` routing. Existing main has prompt-injection hardening through external-content wrappers, tool policy, sandboxing, approvals, and docs, but that is not equivalent to this opt-in CaMeL architecture. The PR is XL and security-sensitive, with review discussion documenting design risks, so it needs explicit maintainer/security product review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for explicit maintainer/security product judgment. If OpenClaw wants CaMeL in core, the best path is a reviewed design that preserves isolated planner/extractor sessions, durable taint provenance through tool results and rewritten params, replay-safe side-effect handling, clear config semantics, gateway approval protocol/UX compatibility, docs, and focused regression tests. If maintainers prefer the current structural-wrapper/tool-policy direction, close only after linking a canonical replacement that covers the same opt-in CaMeL/taint-tracking security goal.
