## Summary

Keep open. Current main still applies the old Fireworks Kimi wrapper behavior that deletes all `reasoning*` payload fields, so this PR is not obsolete or implemented on main. The PR is focused, but it still needs normal maintainer review because the linked report may be a different `kimi/k2p6` route, the new test duplicates production model detection, and the provider request-shape/security tradeoff needs confirmation before merge.

## What This Changes

The PR changes the Fireworks provider stream wrapper so Kimi K2.6 keeps `reasoning`, `reasoning_effort`, and `reasoningEffort` while still setting `thinking: { type: "disabled" }`, with added Fireworks stream unit coverage.

## Best Possible Solution
