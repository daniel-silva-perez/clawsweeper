## Summary

Keep open. Current main still lets the subagent announce flow deliver `(no output)` when no child completion text, child findings, or fallback reply exists, and the focused regression test still asserts that delivery. The open hardening PR #68464 remains the active implementation candidate rather than shipped behavior.

## What This Changes

Prevent async/subagent completion announcements from reaching parent sessions before a usable result or fallback payload is available.

## Best Possible Solution

Land PR #68464 or a narrower successor that persists a user-safe completion payload or retryable delivery state, defers completion delivery until real child output or a sanitized fallback exists, and replaces the current `(no output)` announce expectation with a fail-closed regression test.

## Reproduction Assessment

Yes. A focused current-main reproduction exists in `src/agents/subagent-announce.format.e2e.test.ts`: child history contains only user text, no assistant/tool output is available, `expectsCompletionMessage` is true, and the announce still succeeds with `(no output)`. The issue discussion also includes production reproduction details for Telegram/UI with OpenRouter on OpenClaw 2026.4.15.

## Solution Assessment

No. The current implementation is not the best solution because it promotes a placeholder as a completion result; the narrow maintainable fix is to defer/retry or persist a user-safe payload before delivery, as proposed by PR #68464 or a smaller successor.
