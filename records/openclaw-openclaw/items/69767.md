## Summary

Keep open. Current main has a related fallback for parent announce turns that return no visible payload, but the child-result side still can proceed with `(no output)` when no usable subagent result or fallback payload exists. The existing tests still assert that behavior, and open PR #68464 remains the active implementation candidate for the broader fail-closed/persisted-payload fix.

## Best Possible Solution

Keep this issue open and route it through review of PR #68464 or a narrower successor. The fix should make completion-message announce fail closed when no usable child result, structured delivery payload, or sanitized fallback exists: preserve retryable cleanup state, avoid marking delivery complete, and add a regression test that currently fails because `(no output)` is delivered.
