## Summary

Current main still permits the subagent completion announce path to continue with `(no output)` when no child completion text, child findings, or fallback reply exists. A newer sessions_yield pause fix and the shipped parent no-visible-payload direct fallback are related, but they do not remove the missing child-payload announce path. Open PR #68464 remains the active implementation candidate for persisted user-safe payloads and fail-closed completion delivery.

## Best Possible Solution

Keep this issue open and route it through review of PR #68464 or a narrower successor. The code should fail closed when completion delivery is expected but no usable payload exists, persist a user-safe delivery payload or retryable claim state, avoid delivering `(no output)` as a completion result, and add a regression test that currently fails because the announce path still sends `(no output)`.
