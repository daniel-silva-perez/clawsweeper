## Summary

Keep open. Current main still allows the subagent completion announce path to deliver a requester-facing `(no output)` completion result when no child output, child findings, or fallback reply exists; the focused regression test still asserts that behavior, and PR #68464 remains open rather than shipped.

## What This Changes

The issue asks OpenClaw to defer async/subagent completion announcements until a usable result payload, structured payload, or sanitized fallback exists.

## Best Possible Solution

Land PR #68464 or a narrower successor that persists a user-safe completion payload or retryable delivery state, defers completion delivery until real child output or a sanitized fallback exists, and replaces the current `(no output)` announce expectation with a fail-closed regression test.

## Reproduction Assessment

Yes. Current main has a focused test path where child history contains only user text, `expectsCompletionMessage` is true, and the announce still succeeds with `(no output)`; the issue discussion also includes production symptoms from Telegram and Control UI.

## Solution Assessment

No. The current implementation is not the best way to solve the issue because it promotes a placeholder as the completion result; the narrower maintainable fix is to defer/retry or persist a user-safe payload before delivery.
