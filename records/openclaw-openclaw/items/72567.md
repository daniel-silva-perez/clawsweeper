## Summary

Keep this PR open, but do not merge it as-is. The diff changes fallback/default thinking lists and docs toward an all-level dropdown contract, while current main returns non-empty provider-owned profiles before that fallback is used; main providers such as OpenAI would still not expose `adaptive` or `max` through Gateway/session rows, so the user-visible behavior would not match the PR claim.

## What This Changes

The PR expands shared and UI fallback thinking-level lists with `xhigh`, `adaptive`, and `max`, changes thinking ranks and labels, and updates tests/docs to describe all eight levels for non-binary providers.

## Best Possible Solution

Maintainers should first confirm the desired thinking-level contract. If all-level dropdown behavior is wanted, implement it in the provider-profile path or a documented central post-processing seam, then cover at least one real profiled provider plus Gateway row and Control UI selector behavior; otherwise keep the current provider-accurate docs/tests and reject the fallback-only expansion.

## Reproduction Assessment

Yes. A high-confidence static reproduction is available: with any non-empty provider profile, `resolveThinkingProfile` returns before the base profile is used, and Gateway/UI rows render that returned `thinkingLevels` list.

## Solution Assessment

No. Expanding fallback lists and UI defaults is not the narrowest maintainable solution because it misses explicit provider profiles and changes docs toward behavior the runtime does not actually enforce for main providers.
