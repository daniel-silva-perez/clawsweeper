## Summary

Keep open. Current main still treats `--resources` values as case-sensitive, the mixed-case examples still exit with unknown-resource errors, and the related implementation PRs in the supplied context are closed and unmerged rather than a canonical fix.

## What This Changes

Make `skills/skill-creator/scripts/init_skill.py --resources` parse resource names case-insensitively while preserving validation and order-preserving deduplication.

## Best Possible Solution

Normalize each non-empty resource token by stripping and lowercasing before validation and first-seen deduplication, return canonical lowercase resource names, and add focused `parse_resources()` regression tests for mixed-case valid input, case-insensitive duplicates, empty-token handling, and invalid-resource rejection.
