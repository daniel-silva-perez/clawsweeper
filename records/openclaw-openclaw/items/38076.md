## Summary

Keep open. Current main still rejects valid mixed-case `--resources` values in the bundled skill-creator parser, the latest release tag has the same strip-only implementation, and the related implementation PRs are closed and unmerged.

## What This Changes

The issue requests case-insensitive parsing for `skill-creator` `init_skill.py --resources` values while preserving validation and order-preserving deduplication.

## Best Possible Solution

Normalize each non-empty resource token by stripping and lowercasing before validation and first-seen deduplication, return canonical lowercase resource names, and add focused `parse_resources()` regression tests for mixed-case valid input, case-insensitive duplicates, empty-token handling, and invalid-resource rejection.

## Reproduction Assessment

Yes. Importing `parse_resources()` with `PYTHONDONTWRITEBYTECODE=1 python3 -B` reproduces current-main behavior: lowercase values succeed, while `Scripts,References,ASSETS` and `scripts,Scripts` exit with unknown-resource errors.

## Solution Assessment

Yes. A narrow parser normalization plus focused Python unittest coverage is the smallest maintainable fix; no core architecture, plugin API, or product decision is needed.
