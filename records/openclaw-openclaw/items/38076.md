## Summary

Keep open. Current main still rejects valid mixed-case `--resources` values because `parse_resources()` validates the stripped tokens before lowercasing them, and the related implementation PRs are closed and unmerged.

## What This Changes

Make `skills/skill-creator/scripts/init_skill.py --resources` accept valid resource names case-insensitively while preserving validation and order-preserving deduplication.

## Best Possible Solution

Normalize each non-empty resource token by stripping and lowercasing before validation and first-seen deduplication, return canonical lowercase resource names, and add focused `parse_resources()` regression tests for mixed-case valid input, case-insensitive duplicates, empty-token handling, and invalid-resource rejection.
