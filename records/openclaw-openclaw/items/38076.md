## Summary

Keep open. Current main still validates stripped `--resources` tokens against lowercase allowed names without lowercasing first, so valid mixed-case values still fail; the related implementation PRs are closed and unmerged, leaving a narrow reproducible parser bug.

## What This Changes

Make `skills/skill-creator/scripts/init_skill.py --resources` accept resource names case-insensitively while preserving validation and order-preserving deduplication.

## Best Possible Solution

Normalize each non-empty resource token by stripping and lowercasing before validation and first-seen deduplication, return canonical lowercase resource names, and add focused `parse_resources()` regression coverage for mixed-case valid input, case-insensitive duplicates, empty-token handling, and invalid-resource rejection.
