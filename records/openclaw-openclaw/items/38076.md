## Summary

Keep open. Current main still rejects mixed-case `--resources` values because `parse_resources()` validates and deduplicates the original stripped tokens without lowercasing; a read-only behavior check reproduces the failure, no focused regression test exists, and the related fix PRs are closed/unmerged.

## What This Changes

The issue requests case-insensitive parsing for `skill-creator` `init_skill.py --resources` values while preserving validation and order-preserving deduplication.

## Best Possible Solution

Normalize each non-empty `--resources` token by stripping and lowercasing before validation and deduplication, return canonical lowercase resource names in first-seen order, and add focused regression coverage for mixed-case valid input, case-insensitive duplicates, and unknown-resource rejection.
