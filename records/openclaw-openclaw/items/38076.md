## Summary

Keep open. Current main still strips `--resources` tokens without lowercasing them before validation or deduplication, so mixed-case valid values reproduce the reported failure; the related implementation PRs are closed and unmerged.

## What This Changes

Make `skill-creator` `init_skill.py --resources` parsing case-insensitive while preserving validation, canonical resource names, and order-preserving deduplication.
