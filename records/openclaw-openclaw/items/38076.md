## Summary

Current main still rejects mixed-case `init_skill.py --resources` values because `parse_resources()` strips tokens but validates and deduplicates them with their original casing. The linked fix attempts are closed and unmerged, so the issue remains a narrow, valid repair candidate.

## What This Changes

Make `skills/skill-creator/scripts/init_skill.py --resources` accept resource tokens case-insensitively while preserving canonical lowercase output, validation, and order-preserving deduplication.

## Best Possible Solution

Normalize each `--resources` token by stripping and lowercasing before validation and deduplication, return canonical lowercase resource names in first-seen order, and add focused unit coverage for mixed-case valid input, case-insensitive duplicate handling, and unknown-resource rejection.

Add focused regression tests in `skills/skill-creator/scripts/test_init_skill.py` for mixed-case valid resources, case-insensitive duplicate handling such as `scripts,Scripts`, and unknown-resource rejection. No changelog entry is expected for this internal skill tooling fix unless a maintainer requests one.
