## Summary

Keep this issue open. Current main still parses `init_skill.py --resources` case-sensitively: tokens are stripped but not lowercased before validation, and a read-only import check rejects `Scripts,References,ASSETS`. The linked implementation PRs in the supplied context are all closed and unmerged, and current tests do not include an `init_skill` regression test.

## Best Possible Solution

Implement a narrow skill-creator fix: normalize each nonempty `--resources` token to lowercase before validation and deduplication, keep the resulting directory names as `scripts`, `references`, and `assets`, and add focused unit tests for mixed-case input, case-insensitive dedupe, and unknown-resource rejection.
