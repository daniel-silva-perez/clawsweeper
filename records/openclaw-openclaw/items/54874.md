## Summary

Keep open. Current main still sends every WebChat textarea input through reactive Lit state (`chatMessage`), and `renderChat` still rebuilds the chat item/thread structure during host renders. The related closed PRs were not merged, and the provided related context points to an active follow-up PR (#62790), so this remains actionable rather than cleanup-closeable.

## Best Possible Solution

Keep this issue open and use the active WebChat keystroke-delay PR as the implementation path. The fix should remove normal typing from the Lit host render loop or otherwise cache/guard the expensive chat thread work, then add a regression test proving draft-only input changes do not rebuild the message thread.
