## Summary

Keep this issue open. Current main still lacks the requested persistent active-agent plus workspace indicator in the Control UI chat header. The code only provides adjacent session selection and agent metadata plumbing: session options can be grouped by agent and the backend exposes workspace metadata through agents.list, but there is no always-visible workspace label/tooltip and no switch confirmation feedback. PR #31602 directly attempted this, but it is closed unmerged and does not supersede the issue.

## Best Possible Solution

Keep this issue open as the Control UI tracker. Implement a minimal persistent chat-header context indicator derived from the active sessionKey plus agents.list, showing agent id/name and workspace path with a full-path tooltip, updating through switchChatSession and session-selection paths, surviving reload through existing URL/session state, adding clear switch feedback, and staying quiet for single-agent setups. Closed PR #31602 is useful prior art but is not shipped or canonical.
