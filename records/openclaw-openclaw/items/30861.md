## Summary

Keep this issue open. Current main still lacks the requested persistent active-agent plus workspace indicator in the Control UI chat header. The chat header renders the grouped session selector plus model/thinking/chat controls, and the selector partially disambiguates duplicate agent sessions, but it does not show workspace metadata or provide switch-confirmation feedback. Existing Gateway/UI types already expose agent workspace metadata, so this is a focused UI implementation candidate rather than a missing backend API.

## Best Possible Solution

Keep this issue open as the canonical minimal Control UI tracker. Implement a persistent chat context indicator using `state.sessionKey`, parsed agent id, and `state.agentsList.agents[].workspace`, with truncated workspace text plus full-path tooltip, immediate updates through all session/agent switch paths, switch confirmation feedback, reload/reconnect correctness, and minimal/no clutter for single-agent setups. Keep #69668 separate for broader team/project hierarchy.
