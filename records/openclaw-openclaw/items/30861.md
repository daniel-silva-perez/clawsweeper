## Summary

Keep this issue open. Current main at 9bb1e59447a5fdf2c0164672df6c748188b26442 still lacks the requested persistent active-agent plus workspace indicator in the Control UI chat header. The header renders the grouped session selector plus model/thinking/chat controls, and the selector can partially disambiguate duplicate agent sessions, but it does not show agent workspace metadata or provide explicit switch-confirmation feedback. Existing Gateway/UI types already expose agent workspace metadata, so this remains a focused UI implementation candidate rather than a backend API gap.

## Best Possible Solution

Keep this issue open as the canonical minimal Control UI tracker. Implement a persistent chat context indicator using `state.sessionKey`, parsed agent id, and `state.agentsList.agents[].workspace`, with truncated workspace text plus full-path tooltip, immediate updates through all session/agent switch paths, explicit switch confirmation feedback, reload/reconnect correctness, and minimal/no clutter for single-agent setups. Keep #69668 separate for broader team/project hierarchy.
