## Summary

Keep this issue open. Current main still has only adjacent session-selection and agent metadata plumbing: the Control UI chat header renders a session/model/thinking control row, and session options can be grouped by agent, but there is no persistent active-agent plus workspace indicator, no full workspace tooltip, and no switch confirmation feedback. The backend already exposes workspace metadata through agents.list, so this is a focused Control UI feature gap. Related PR #31602 was closed unmerged and does not supersede the issue.

## Best Possible Solution

Keep this issue open as the Control UI tracker. Implement a minimal persistent chat-header context indicator derived from the active sessionKey plus agents.list, showing agent id/name and workspace path with a full-path tooltip, updating through switchChatSession and any existing session-selection/command path, surviving reload through existing URL/settings state, adding clear switch feedback, and staying quiet for single-agent setups. Closed PR #31602 is useful prior art but is not shipped or canonical.
