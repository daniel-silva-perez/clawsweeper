## Summary

Keep open. The PR targets a real current-main gap: `update.run` still rejects and drops `continuationMessage`, while restart-continuation startup handling already exists. The implementation is narrow and the security-sensitive surface remains admin-scoped/rate-limited, but the changelog currently overclaims the broader related issue as fixed even though the PR body says that follow-up scope remains open.

## What This Changes

This PR adds `continuationMessage` to the `update.run` protocol/client models, forwards it from the gateway agent tool, writes an `agentTurn` continuation into successful update restart sentinels, and updates tests, docs, generated Swift models, and changelog.

## Best Possible Solution

Revise the changelog to describe this as partial coverage or a reference to #71178, then land a narrow additive Gateway protocol/server/agent-tool change that queues one follow-up `agentTurn` only after successful `update.run` restarts while keeping failed/skipped updates without continuations and leaving #71178 open for remaining drain/reset/message-loss work.
