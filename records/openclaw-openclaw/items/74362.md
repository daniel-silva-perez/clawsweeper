## Summary

Current main still rejects or drops `continuationMessage` on `update.run` and tests the absence of a restart continuation, while this PR is an active, scoped implementation against a broader linked incident. Keep it open for maintainer review and CI rather than closing or replacing it.

## What This Changes

The PR adds `continuationMessage` to the `update.run` Gateway protocol and generated Swift models, forwards it from the agent gateway tool, writes an agent-turn continuation into successful update restart sentinels, and updates tests, docs, and changelog.

## Best Possible Solution

Land or revise a narrow additive Gateway protocol/server/agent-tool change that queues one follow-up `agentTurn` only after successful `update.run` restarts, leaves failed and skipped updates without continuations, keeps generated protocol clients/docs/tests in sync, and preserves separate tracking for any broader message-loss behavior not fixed here.
