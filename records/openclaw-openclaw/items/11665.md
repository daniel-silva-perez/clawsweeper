## Summary

Keep open. Current main still routes direct and mapped hook-agent requests through an isolated cron-style run, so a stable hook `sessionKey` reaches `resolveCronSession` with `forceNew: true` and cannot reuse the stored transcript session. The prior implementation PRs are useful source material, especially #27193, but they are closed unmerged; the related #42621 cleanup was closed as a duplicate of this remaining work rather than as an implementation.

## What This Changes

The issue requests multi-turn webhook hook support by reusing an existing transcript session when a stable hook `sessionKey` is supplied.

## Best Possible Solution

Add a documented, backward-compatible hook option such as `sessionMode: "isolated" | "persistent"` for direct `/hooks/agent` payloads, hook mappings, and transform results. Preserve isolated behavior by default, propagate the mode through dispatch, and only pass `forceNew: false` for explicit persistent hook-agent runs with stable allowed session keys.
