## Summary

Current main still does not implement persistent multi-turn webhook hook sessions. Direct and mapped agent hooks can carry a stable sessionKey, but dispatch still wraps them as isolated cron-style runs and the runner still passes forceNew: true, so resolveCronSession mints a fresh sessionId. No sessionMode/reuseSession/targetExistingSession field exists in hook payloads, mappings, transform results, or the strict hook mapping schema.

## Best Possible Solution

Keep this issue open and implement an explicit, backward-compatible persistent hook-session option in core: add sessionMode: "isolated" | "persistent" for direct /hooks/agent payloads, hook mappings, and transform results; default to isolated; preserve existing hook security gates; and only pass forceNew: false for the explicit persistent hook-agent path so stable sessionKey values reuse fresh existing session ids without changing cron isolation.

## Clownfish Work Prompt

Implement explicit persistent multi-turn webhook hook sessions for this issue. Add `sessionMode: "isolated" | "persistent"` to direct `/hooks/agent` payloads, hook mapping config, and transform return results. Default to `isolated` for full backward compatibility. For `persistent`, keep using the resolved/normalized hook `sessionKey`, but make the isolated-agent runner call `resolveCronSession` with `forceNew: false` for that explicit hook-agent path so a fresh existing `sessionId` is reused. Do not change cron job isolation semantics, do not make stable `sessionKey` automatically persistent, and do not bypass `hooks.allowedAgentIds`, `hooks.allowRequestSessionKey`, `hooks.allowedSessionKeyPrefixes`, idempotency replay keys, external-content safety wrapping, or agent session-key normalization. Introduce a narrow runner option if needed rather than overloading unrelated cron `sessionTarget` behavior. Update types, strict zod schema, mapping merge/threading, direct request normalization, dispatch plumbing, docs/config reference, generated config docs if required, and focused regression tests.
