## Summary

Keep open. Current main still dispatches direct and mapped agent webhooks as isolated cron-style turns, and that path still passes `forceNew: true`, so a stable hook `sessionKey` cannot reuse the existing transcript/session id. The hook payload, mapping transform/config, and strict config schema still expose `sessionKey` but no `sessionMode`, `reuseSession`, or equivalent persistent-session control. Prior implementation PRs #15084, #27123, and #27193 are closed unmerged, and #42621 was closed as superseded by this issue rather than as implemented.

## Best Possible Solution

Implement a backward-compatible persistent hook-session option in core, likely `sessionMode: "isolated" | "persistent"` on direct `/hooks/agent` payloads, hook mappings, and transform results. Preserve isolated as the default, thread the option through config/schema/help/docs and hook dispatch, and pass `forceNew: false` only for the persistent hook path so stable `sessionKey` values can reuse fresh existing session ids without changing cron isolation.
