## Summary

Keep open. Current main has explicit ACP runtime model/thinking controls, but it still lacks the requested ACPX config-level per-harness defaults: the plugin schema and manifest are strict, the `agents` override only accepts `command`, and the resolver only carries command overrides into the runtime.

## What This Changes

Request ACPX plugin configuration for per-harness default `model` and `reasoning_effort` or other runtime config options when spawning ACP sessions.

## Best Possible Solution

Keep the issue open for a maintainer design decision, then add one schema-backed ACP default surface with adapter-aware validation/filtering, tests for Codex and non-Codex behavior, and docs showing how defaults interact with explicit `/acp model`, `/acp set`, and `sessions_spawn` overrides.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main rejects the proposed `agentDefaults` shape through strict ACPX config schema/manifest validation and has no resolver path that would feed such defaults into ACP runtime options.

## Solution Assessment

Unclear. The requested capability is valid, but a plugin-only `agentDefaults.configOptions` implementation may not be the safest design until maintainers choose how it should relate to core `agents.list[].runtime.acp` defaults and adapter capability checks.
