## Summary

Keep open. Current main has partial explicit ACP runtime-option support for `sessions_spawn` model/thinking values and persisted runtime options, but it still does not implement the requested per-agent ACPX config defaults for model or reasoning/config options. The ACPX plugin config and manifest remain command-only for per-agent entries, `agents.list[].runtime.acp` only covers agent/backend/mode/cwd, and `/acp spawn` has no model/thinking spawn flags.

## Best Possible Solution

Keep the issue open and implement a typed per-agent ACP default surface. The clean product path is to extend either `agents.list[].runtime.acp` or `plugins.entries.acpx.config.agents.<id>` with `model`, `thinking`/`reasoning_effort`, and a generic backend config-options bag, then merge those defaults before ACP session initialization for both `sessions_spawn(runtime="acp")` and `/acp spawn`, with explicit call/session options taking precedence. Update the strict schema, plugin manifest/UI hints, ACP docs, and tests covering Codex normalization plus generic config-option behavior.
