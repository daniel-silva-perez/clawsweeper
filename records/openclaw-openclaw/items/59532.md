## Summary

Keep open. Current main has explicit per-spawn ACP model/thinking plumbing and ACP runtime config controls, but it still lacks the requested ACPX plugin config surface that would apply defaults per harness when `/acp spawn` or `sessions_spawn(runtime="acp")` omits those options.

## What This Changes

Add ACPX plugin configuration for per-agent default model and runtime config options such as `reasoning_effort`.

## Best Possible Solution

Add a plugin-owned ACPX config surface such as `plugins.entries.acpx.config.agentDefaults`, validate it in the ACPX schema/manifest, normalize harness ids, and apply defaults only when a spawn omits explicit runtime options. Reuse the existing ACP runtime option and `setConfigOption` machinery, document adapter capability limits, and keep core free of bundled harness-specific defaults.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: adding the proposed `agentDefaults` object would not pass the current strict ACPX plugin schema, and a spawn without explicit `model` or `thinking` reaches `initializeAcpSpawnRuntime()` with no runtime options to apply.

## Solution Assessment

Yes, a narrow ACPX plugin fix is the best direction. This should stay in `extensions/acpx` plus existing ACP runtime-control seams, rather than adding core hardcoded harness defaults or asking operators to script slash commands after every spawn.

## Repair Work Prompt

Implement per-agent ACPX runtime defaults for https://github.com/openclaw/openclaw/issues/59532. Add a validated `plugins.entries.acpx.config.agentDefaults` map under the bundled `acpx` plugin, keyed by ACP harness id. Each entry should allow an optional `model` string and optional `configOptions` record of string keys/values, with safe trimming/validation and normalized agent ids. Apply defaults only when spawning/ensuring an ACPX session for that harness and only when the caller did not provide an explicit runtime option; explicit `sessions_spawn` `model`/`thinking`, `/acp model`, and `/acp set` must continue to win. Reuse existing ACP runtime option/setConfigOption paths and preserve adapter unsupported-control failures. Keep the change in `extensions/acpx` and existing ACP runtime-control seams; do not add hardcoded core harness defaults or broad migrations. Update `extensions/acpx/openclaw.plugin.json`, ACPX config tests/runtime tests/service tests as needed, and document the config in `docs/tools/acp-agents-setup.md`. No changelog entry is required unless maintainers want one for the user-facing feature.
