## Summary

Keep open. Current main still lacks the requested `api.runtime.gateway.dispatchMethod` surface, and the motivating sanitized-history/cross-plugin call use case is real, but this PR is not merge-ready because it turns an internal Gateway dispatcher into a broad public plugin runtime API without explicit authorization, capability, or plugin identity semantics.

## What This Changes

The PR exports the internal Gateway dispatcher and adds a `PluginRuntime.gateway.dispatchMethod<T>(method, params)` namespace wired through `createPluginRuntime` and Gateway plugin runtime options.

## Best Possible Solution

Land a reviewed plugin runtime contract that preserves the useful sanitized-history and cross-plugin request/response use cases while making caller identity, allowed methods/scopes, fallback behavior, docs, and contract tests explicit, likely coordinated with the broader open #39000 API discussion.
