## Summary

Keep open. Current main still has the reported Telegram document failure mode: a `getFile` failure such as Telegram's public Bot API size limit returns no media, the body path can render only `<media:document>`, and the finalized agent context has no `MediaPath`/`MediaPaths`. The related PR #55970 is still the implementation candidate, but it is open and its proposed user-visible unavailable reason is not present on main.

## Best Possible Solution

Keep this issue open. Review and land PR #55970 if its scoped behavior is acceptable, or implement an equivalent Telegram-owned fix that preserves `<media:document>` while adding an agent-visible unavailable reason. The fix should also document the public Bot API download limit versus `channels.telegram.mediaMaxMb`, and point larger-file users toward the existing self-hosted `apiRoot` plus `trustedLocalFileRoots` path where appropriate.
