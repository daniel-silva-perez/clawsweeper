## Summary

Current main still has the reported Telegram document failure mode: a Telegram getFile file-too-big failure returns no resolved media, the inbound handler forwards an empty media list, the body can remain only the document placeholder, and the finalized agent context has no MediaPath/MediaPaths. The related PR #55970 is still open and unmerged, so this issue should stay open for maintainer follow-up.

## Best Possible Solution

Keep this issue open. Review and land PR #55970 if the scoped behavior is acceptable, or implement an equivalent Telegram plugin fix that preserves <media:document> while adding an agent-visible unavailable reason. The fix should also clarify the public Bot API download limit versus channels.telegram.mediaMaxMb and point larger-file users toward the existing self-hosted apiRoot plus trustedLocalFileRoots path where appropriate.
