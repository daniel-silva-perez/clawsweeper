## Summary

Keep this PR open. Current main still has the stale-token display problem in TUI state and the Control UI session-table formatter, but the proposed TUI guard can preserve an old displayed total and blocks fresh null totals from clearing state.

## What This Changes

The PR carries `totalTokensFresh` through TUI and Control UI session row types and changes display/state paths so totals marked stale are not shown.

## Best Possible Solution

Port a narrow display fix onto current main: TUI should receive `totalTokensFresh`, clear the displayed total whenever a row or patch explicitly marks totals stale, still allow fresh `totalTokens: null` to clear state, and Control UI session-table formatting should return `n/a` for stale rows.

## Reproduction Assessment

Yes: a focused check can seed TUI `sessionInfo.totalTokens`, apply a session row with `totalTokensFresh: false` and no fresh total, and observe the old footer value remains. The Control UI formatter path is also directly reproducible by passing a row with `totalTokensFresh: false` and a numeric `totalTokens` to `formatSessionTokens`.

## Solution Assessment

No: the PR direction is right, but the current TUI guard is not the best fix because it handles freshness only after requiring a non-null total. The safer fix is to handle `totalTokensFresh === false` first by clearing the stored total, then apply fresh numeric or null totals normally.
