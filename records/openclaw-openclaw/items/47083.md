## Summary

Keep this PR open. Current main still has the TUI stale-token display gap, while the proposed TUI guard would not clear cached totals for stale rows without a raw total and would block fresh null resets.

## What This Changes

The PR adds `totalTokensFresh` handling to TUI session row/state types and Control UI session-token formatting so stale token totals are hidden.

## Best Possible Solution

Land a narrow current-main repair that carries `totalTokensFresh` through the active TUI session-list/session-info path, clears or suppresses cached TUI footer totals when freshness is false, allows fresh `totalTokens: null` to clear state, keeps Control UI formatting freshness-aware, and adds focused regressions for both display paths.

## Reproduction Assessment

Yes. Seed TUI state with a numeric `sessionInfo.totalTokens`, then apply a same-session row or patch with `totalTokensFresh: false` and no fresh total; current main leaves the old footer value visible. The PR-specific regression is also reproducible with a fresh `totalTokens: null` patch, which the proposed non-null guard would ignore.

## Solution Assessment

No. The PR direction is right, but the TUI guard must handle freshness before value/null checks and must clear stale cached state; the current patch still preserves misleading totals in edge cases.
