## Summary

Keep open. Current main still lacks the TUI freshness propagation and the Control UI token formatter guard, while this PR's TUI change would still leave stale or fresh-null token-state cases visible after merge.

## What This Changes

The PR adds `totalTokensFresh` handling to TUI session rows and Control UI session-token formatting so stale token totals are hidden.

## Best Possible Solution

Land a narrow current-main repair: carry `totalTokensFresh` through the active TUI session-list/session-info types, make TUI state clear or suppress cached footer totals when freshness is false while still allowing fresh `null` to clear, keep Control UI formatting freshness-aware, and add focused regressions for both display paths.

## Reproduction Assessment

Yes. Seed TUI state with a numeric `sessionInfo.totalTokens`, then apply a same-session row or patch with `totalTokensFresh: false` and no fresh total; current main leaves the old footer value visible. The PR-specific regression is also reproducible with a fresh `totalTokens: null` patch, which the proposed non-null guard would ignore.

## Solution Assessment

No. The PR direction is correct, but the TUI guard must handle freshness before value/null checks and the stale branch needs a current-main port; otherwise it still preserves misleading cached totals in edge cases.
