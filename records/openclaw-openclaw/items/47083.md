## Summary

Keep this PR open. Current main still exposes `totalTokensFresh` from the Gateway but does not fully consume it in the affected TUI footer/session state or the Control UI sessions table formatter, so the intended stale-token display fix is still useful. The submitted patch needs a rebase to the current TUI type shape and a small correctness fix for fresh-null clearing before it should merge.

## What This Changes

The PR adds `totalTokensFresh` to TUI and Control UI session-row types and changes TUI/UI token display paths to avoid showing stale `totalTokens` values.

## Best Possible Solution

Update this work on top of current main: carry `totalTokensFresh` through the TUI session list and `SessionInfo`, ignore stale totals while preserving fresh-null clearing semantics, make the Control UI sessions-table formatter return `n/a` when `totalTokensFresh === false`, and cover those cases with focused TUI and UI tests without changing Gateway freshness derivation.
