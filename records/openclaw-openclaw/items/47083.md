## Summary

Keep this PR open. The stale-token display problem is still real on current main for TUI state and the Control UI formatter, but the proposed TUI update would still leave old totals visible when a session becomes stale and would also block fresh null totals from clearing the display.

## What This Changes

The PR carries `totalTokensFresh` through TUI and Control UI session row types and changes token display/state paths so totals marked stale are not shown.

## Best Possible Solution

Port a narrow display fix onto current main: TUI should clear the displayed total when a row or patch explicitly marks totals stale, still allow fresh null totals to clear state, and Control UI session-table formatting should return `n/a` for rows marked stale.
