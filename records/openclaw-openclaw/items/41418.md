## Summary

Keep this issue open. Current main still has only scoped dry-run support; it does not expose `openclaw agent --dry-run`, does not thread dry-run state through the gateway or embedded agent run options, and the shared Pi tool adapter still calls managed tools for real after `before_tool_call` policy handling. The related dry-run items are narrower command/workflow requests, not a canonical replacement for global agent tool execution preview.

## Best Possible Solution

Keep this open as core agent execution work. The best path is to add `openclaw agent --dry-run`, thread request-scoped dry-run state through CLI, gateway protocol, embedded agent options, and the Pi tool-definition adapter, intercept managed and client tool definitions after policy hooks, return structured preview payloads to the model, log redacted previews to stderr, and add focused CLI, protocol, adapter, docs, and regression coverage. The existing plugin hook surface is useful context, but it is not a complete substitute for a first-class core preview mode.
