## Summary

Current main still lacks the requested global `openclaw agent --dry-run` mode. The agent CLI does not expose a dry-run flag, the CLI/Gateway/embedded agent option types do not carry `dryRun`, the Gateway agent schema rejects unknown fields, and the Pi tool adapters still execute managed tools or delegate client tools instead of returning structured `dry_run` previews. The linked dry-run issues are narrower command/workflow requests, not duplicates or shipped implementations of this core agent execution feature.

## Best Possible Solution

Keep this issue open for core agent execution work. Add an `openclaw agent --dry-run` user surface, thread request-scoped dry-run state through CLI, Gateway protocol, embedded `AgentCommandOpts`, and the Pi tool-definition adapters, intercept managed and client tool calls after policy hooks, return structured `dry_run` payloads to the model, log redacted previews to stderr, and add focused adapter, CLI/protocol, docs, and regression coverage. Keep narrower command-specific dry-run issues separate unless a maintainer creates a canonical umbrella.
