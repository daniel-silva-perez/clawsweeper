## Summary

Keep this issue open. Current main still does not implement a global `openclaw agent --dry-run` mode: the agent CLI has no flag, gateway agent params/schema do not carry dry-run state, embedded agent options do not include it, and the universal tool adapter still executes managed tools for real after `before_tool_call` policy handling. The provided related issues are scoped dry-run requests, not a canonical replacement for this agent execution pipeline work.

## Best Possible Solution

Keep this open as core agent execution work. The best path is to add `openclaw agent --dry-run`, thread request-scoped dry-run state through CLI, gateway protocol, embedded agent params, and the tool-definition adapter, intercept both managed and client tool definitions after policy hooks, return structured preview payloads to the model, log redacted previews to stderr, and add focused CLI, protocol, adapter, docs, and regression coverage. This belongs in OpenClaw core because the missing behavior is the shared agent tool-execution pipeline, not an optional ClawHub plugin capability.

## Risks / Open Questions

- A correct implementation should use request-scoped dry-run state for gateway and concurrent embedded runs, not a process-global flag that could leak across runs.
- Preview logging needs to use existing redaction and console-sanitization paths before printing tool params to stderr.
- Dry-run interception should remain after trusted policy and `before_tool_call` hooks so preview mode does not bypass plugin safety policy.
