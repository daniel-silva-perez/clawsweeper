## Summary

Keep this issue open. Current main still does not provide a global `openclaw agent --dry-run` mode: the CLI has no flag, the agent gateway protocol rejects unknown dry-run fields, embedded agent options do not carry dry-run state, and the universal tool adapter still executes managed tools for real after before-tool-call hooks. The linked contributor branch exists and may be useful, but the behavior has not landed on main.

## Best Possible Solution

Keep this open as core agent execution work. The best implementation should add `openclaw agent --dry-run`, thread request-scoped dry-run state through CLI, gateway protocol, embedded agent params, and the tool definition adapter, intercept both managed and client tool definitions after policy hooks, return structured preview payloads to the model, log redacted previews to stderr, and update focused adapter, CLI, protocol, and docs coverage. This belongs in OpenClaw core because the missing behavior is the shared agent tool-execution pipeline, not an optional plugin capability.

## Risks / Open Questions

- A correct gateway-capable design should use request-scoped dry-run state, not a process-global flag, to avoid leakage between concurrent agent runs.
- Preview logging must reuse existing redaction and console-sanitization paths before printing tool params to stderr.
- Dry-run interception should run after `before_tool_call` policy blocks so preview mode does not bypass plugin safety policy.
