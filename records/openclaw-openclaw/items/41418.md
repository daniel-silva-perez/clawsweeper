## Summary

Keep this issue open. Current main at 6d323ee73640afdf02ed9fa3aded13da23359b98 still has only scoped dry-run behavior; it does not expose `openclaw agent --dry-run`, does not carry a dry-run value through gateway or embedded agent options, and the Pi tool adapter still executes managed tools after policy hooks. The related dry-run issues in the provided context are narrower command/workflow requests, not a canonical replacement for global agent tool execution preview.

## Best Possible Solution

Keep this open as core agent execution work. Add `openclaw agent --dry-run`, thread request-scoped dry-run state through CLI, gateway protocol, embedded `AgentCommandOpts`, and the Pi tool-definition adapters, intercept managed and client tool calls after policy hooks, return structured `dry_run` payloads to the model, log redacted previews to stderr, and add focused adapter, CLI/protocol, docs, and regression coverage.

## Risks / Open Questions

- A correct implementation should use request-scoped dry-run state rather than process-global state, especially for gateway and concurrent embedded runs.
- Preview logging needs to reuse existing redaction and console sanitization before printing tool parameters to stderr.
- Dry-run interception should remain after trusted policy and `before_tool_call` hooks so preview mode does not bypass safety policy.
