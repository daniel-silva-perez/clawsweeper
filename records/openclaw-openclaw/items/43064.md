## Summary

Current main still appears to have the reported heartbeat `responsePrefix` template interpolation bug. The heartbeat path reads the raw prefix and passes it directly to heartbeat normalization, while the normal reply path has the model-selection context needed to resolve `{model}`, `{provider}`, and related placeholders. The issue also has an open same-author implementation candidate, #43065, with closing syntax, so it should not be closed before that PR or another fix lands.

## Best Possible Solution

Keep this issue open and review #43065 or a successor fix. The implementation should route heartbeat runs through the shared reply-prefix context, pass `onModelSelected` into `getReplyFromConfig`, resolve the prefix after model selection before `normalizeHeartbeatReply`, and add focused regression coverage for templated heartbeat alert prefixes plus prefixed `HEARTBEAT_OK` suppression.
