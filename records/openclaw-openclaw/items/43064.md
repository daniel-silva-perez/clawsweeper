## Summary

Keep this issue open. Current main still appears to have the heartbeat responsePrefix interpolation bug: the heartbeat runner reads the raw configured prefix, does not pass the model-selection hook into getReplyFromConfig, and then prepends the raw prefix during heartbeat reply normalization. The supplied GitHub context also shows open same-author PR #43065 with closing syntax, so the issue should remain open until that PR or a successor fix is merged.

## Best Possible Solution

Keep this issue open and review #43065 or a successor patch. The fix should route heartbeat runs through the shared reply-prefix context, pass onModelSelected into getReplyFromConfig, resolve responsePrefix after model selection and before normalizeHeartbeatReply, and add focused regression tests for templated heartbeat alert prefixes plus prefixed HEARTBEAT_OK suppression.
