## Summary

Keep this PR open. Current main still stages backup archives under `os.tmpdir()`, which matches the linked macOS LaunchAgent duplicate-manifest bug, while the latest PR head now addresses the earlier fallback-containment review finding by checking both preferred and fallback temp roots before archiving.

## What This Changes

The PR changes backup archive staging to choose a temp root outside all included backup sources and adds regression coverage for duplicate `manifest.json` entries and unsafe fallback roots.

## Best Possible Solution

Land this PR after normal maintainer and CI review so backup staging always stays outside selected backup sources and fails before archiving when no safe temp root is available.

## Reproduction Assessment

Yes. The high-confidence reproduction is the linked macOS LaunchAgent path: `TMPDIR` resolves to `<stateDir>/tmp`, current main stages `manifest.json` there, and tar includes it both explicitly and while walking the state source.

## Solution Assessment

Yes. The latest PR head is the narrowest maintainable fix: keep the behavior in backup archive creation, validate both preferred and fallback temp roots against the planned sources, and avoid changing service TMPDIR or tar semantics.
