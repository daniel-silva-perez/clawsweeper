## Summary

Keep this PR open. It moves backup staging away from direct `os.tmpdir()`, which is the right direction for the linked macOS `TMPDIR` bug, but it is not merge-ready because the chosen helper can still fall back to a `TMPDIR`-derived root and the patch does not guard that resolved staging root against included backup sources.

## What This Changes

The PR replaces backup archive staging under `os.tmpdir()` with `resolvePreferredOpenClawTmpDir()` and adds a regression test for `TMPDIR` pointing inside the OpenClaw state directory.

## Best Possible Solution

Land a focused backup fix that stages temporary manifest files outside every included backup source, keeps the existing archive publication flow, and adds regression coverage for both the LaunchAgent `TMPDIR` case and the resolver fallback/containment case.

## Reproduction Assessment

Yes. The linked report gives a concrete path: macOS LaunchAgent sets `TMPDIR` under the state directory, backup create writes `manifest.json` under that temp root, and tar also walks the state source tree. Source inspection confirms the same path on current main.

## Solution Assessment

No. Using `resolvePreferredOpenClawTmpDir()` is a useful starting point, but the best fix also needs backup-local containment validation or an alternate root when the resolved temp root is inside any included asset.
