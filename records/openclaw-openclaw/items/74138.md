## Summary

Keep open: current main still lacks the legacy `openclaw/plugin-sdk/feishu` facade needed by older npm-installed Feishu packages, and two open PRs already attempt to fix the same compatibility gap. Per policy, this issue should remain open until one canonical fix merges.

## What This Changes

The issue reports that upgrading to OpenClaw 2026.4.26 breaks Feishu/Lark multi-account dispatch because a legacy Feishu package calls missing `createScopedPairingAccess` from the old Feishu SDK facade.

## Best Possible Solution

Land one canonical compatibility fix that restores a narrow, documented legacy `openclaw/plugin-sdk/feishu` facade for published npm Feishu packages while keeping new bundled code on generic SDK subpaths, with SDK export/API baseline updates and a regression contract test.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to run a legacy npm-installed Feishu package that imports `openclaw/plugin-sdk/feishu` against current host SDK behavior; current main lacks that subpath and keeps `createScopedPairingAccess` internal, matching the reported dispatch TypeError.

## Solution Assessment

Unclear until maintainer review chooses between the open fixes. The narrowest maintainable direction appears to be an additive legacy compatibility facade plus contract coverage, rather than changing Feishu runtime behavior or closing this before a PR lands.
