## Summary

Keep open. The report still matches a real Feishu compatibility gap: v2026.4.26 exposed `openclaw/plugin-sdk/feishu` but did not export `createScopedPairingAccess`, while current main has removed the Feishu SDK subpath entirely and still preserves npm-installed bundled plugin records, so an older external `@openclaw/feishu` install can remain broken after host update.

## Best Possible Solution

Keep this open and choose a compatibility path: either restore a narrow `openclaw/plugin-sdk/feishu` compatibility facade that exports `createScopedPairingAccess` for older external Feishu packages, publish/update `@openclaw/feishu` so npm-installed Feishu matches the current generic SDK, or migrate affected npm-installed Feishu configs back to the bundled plugin during update. Add a regression fixture for an old installed Feishu package importing the old facade against the current host SDK.
