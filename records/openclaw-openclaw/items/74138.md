Keep open. Current main still lacks the legacy Feishu SDK facade needed by older npm-installed Feishu packages, and two open unmerged PRs already reference this issue with closing syntax, so the issue should remain open until a canonical compatibility fix lands.

The issue reports a Feishu/Lark regression after upgrading where multi-account message dispatch fails because a legacy npm-installed Feishu plugin expects `createScopedPairingAccess` from `openclaw/plugin-sdk/feishu`.

## Best Possible Solution

Review and land one narrow compatibility repair that restores the legacy `openclaw/plugin-sdk/feishu` facade needed by published Feishu packages, adds focused SDK contract coverage, and then closes this issue after merge or release proof.

## Solution Assessment
