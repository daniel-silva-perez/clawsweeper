## What This Changes

The issue reports that after upgrading to OpenClaw 2026.4.26, a multi-account Feishu/Lark setup fails dispatching messages because the published Feishu package calls `createScopedPairingAccess` from the removed `openclaw/plugin-sdk/feishu` facade.

## Best Possible Solution

Land one narrow, backward-compatible Feishu SDK compatibility facade for the legacy published package imports, wire it into the generated SDK exports, and add focused subpath contract coverage for `openclaw/plugin-sdk/feishu` before closing this issue.

## Solution Assessment

Yes. The narrowest maintainable fix is a compatibility SDK facade plus export and contract-test coverage; rewriting Feishu runtime behavior would be broader than the observed regression.
