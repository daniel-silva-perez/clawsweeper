The PR remains a narrow active implementation candidate with source and regression-test changes limited to the iMessage monitor surface plus changelog.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or explicitly supersede it with an equivalent canonical iMessage fix. The best product path is a narrow owner-plugin change in `extensions/imessage/src/monitor/echo-cache.ts` that strips only leading imsg corruption bytes, preserves interior text/control characters, adds regression coverage, and coordinates the NUL-prefix overlap with #63581.
