## Summary

Keep open. Current main does not implement the PR's intended dynamic TUI startup conversation summary; it only loads history on connect and already has the lower-level session title/preview API that this PR could use. The PR is still the active implementation candidate, with remaining review concerns around reconnect duplication, tests, and the product/security implications of surfacing prior-session snippets automatically.

## Best Possible Solution

Keep this PR open for maintainer review. If the feature direction is accepted, rebase it onto current main, keep the implementation in the TUI using the existing `sessions.list` derived-title/preview API, add a first-connect-only guard, and add focused tests for success, empty result, error swallowing, local-mode skip, and reconnect behavior before merge.
