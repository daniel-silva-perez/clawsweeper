## Summary

Keep open. Current main still does not implement the PR's intended dynamic TUI startup conversation summary, while the existing Gateway session API already exposes derived titles and last-message previews that make the approach feasible. The PR should receive normal maintainer review rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review or revision. A strong version would rebase against current main, add the dynamic summary after initial non-local history load only, avoid duplicate rendering on reconnect, scope the session query to the current agent/session semantics, handle failures without hiding real test failures, and add focused TUI unit coverage for initial connect, reconnect, no prior session, and failed `listSessions` paths. If the behavior ships, update the TUI docs to describe the new startup summary.
