## Summary

Keep open. Current main still does not implement a dynamic TUI startup conversation summary, and the PR is the active implementation candidate for the closed companion feature request. The latest patch restores the missing formatter and CI checks are green, but the remaining decision is maintainer review of default prior-session preview behavior, agent scoping, docs, and focused tests rather than conservative cleanup closure.

## What This Changes

The PR adds a formatter and a Gateway-backed `client.listSessions({ limit: 10, includeDerivedTitles: true, includeLastMessage: true })` call in the TUI connection path to display a previous non-current session title or last-message preview after startup history loads.

## Best Possible Solution

Keep this PR open for maintainer review. If the feature direction is accepted, land a rebased version that uses the existing sessions preview API, makes an explicit default/privacy and agent-scope choice, adds focused TUI tests around the startup summary path, and updates the public TUI docs if this becomes shipped behavior.

## Reproduction Assessment

Yes for the proposed behavior: a TUI connection test or manual `openclaw tui` session with at least one previous Gateway session can exercise `client.onConnected`, `loadHistory`, and the mocked or live `listSessions` response. I did not run the live TUI because this was a read-only review, but the code path is deterministic from source inspection.

## Solution Assessment

Unclear pending maintainer product review. The Gateway sessions preview API is a reasonable implementation source, but the best shipped solution should settle default visibility, current-agent scoping, reconnect semantics, docs, and regression tests before merge.
