## Summary

Keep open. Current main still has the reported default behavior and several requested follow-ups remain unresolved: missing `agents.defaults.contextInjection` resolves to per-turn bootstrap injection, `MEMORY.md` is still loaded as normal bootstrap context when present, and dynamic bootstrap-file ordering is still hard-coded.

## What This Changes

The issue requests reducing session context bloat by making `continuation-skip` the default, avoiding full `MEMORY.md` bootstrap injection, and adding configurable dynamic bootstrap-file/cache-boundary controls.

## Best Possible Solution

Keep this issue open until maintainers decide the default bootstrap reinjection policy, then land a scoped runtime/config/docs/UI/test change for that policy. Treat the `MEMORY.md` lazy-stub behavior and dynamic bootstrap-file ordering as explicit follow-ups or link them to #65438 if that becomes the canonical tracker.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: missing config still resolves to `always`, tests assert that behavior, and `MEMORY.md` is still included by the bootstrap loader when present; the newer gateway-restart report also has a plausible code path because assistant turns without the full-bootstrap marker do not count as completed bootstrap evidence.

## Solution Assessment

Unclear. Switching the default may be the narrowest fix for the repeated bootstrap default, but it is a product-wide behavior change; the restart-marker portion should be reviewed through #74813, and the memory/dynamic-file requests should be split rather than bundled into one automatic cleanup.
