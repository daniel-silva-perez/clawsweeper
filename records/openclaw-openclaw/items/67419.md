## Summary

Keep open. Current main still defaults missing `agents.defaults.contextInjection` to per-turn bootstrap injection, keeps `continuation-skip` opt-in, still injects `MEMORY.md` as normal bootstrap context, and has no `bootstrapDynamicFiles` or `bootstrapFileOrder` config surface. The related fix PR is closed unmerged, and the maintainer umbrella explicitly keeps concrete reports with unresolved evidence open.

## What This Changes

The issue asks OpenClaw to reduce session context bloat by making bootstrap reinjection less aggressive by default, making `MEMORY.md` cheaper to load, and adding configurable dynamic bootstrap-file handling.

## Best Possible Solution

Keep this issue open until maintainers decide the default bootstrap reinjection policy. If the desired direction is lower default token use, implement the runtime/config/docs/test/UI default switch together, then handle `MEMORY.md` lazy-load/stub behavior and dynamic bootstrap-file ordering as scoped follow-ups or under #65438.
