## Summary

Keep open. Current main does not implement `sessions_broadcast`, so the PR is not obsolete, but the branch is not merge-ready: it adds a sensitive cross-session injection tool with an undeclared TypeBox import, an input provenance value rejected by the current gateway schema, and missing policy coverage where comparable `sessions_send` access is denied.

## What This Changes

This PR adds a core `sessions_broadcast` tool, registers it in the tool catalog and OpenClaw tool factory, and adds tests for broadcasting a message to active sessions for the current agent.

## Best Possible Solution

If maintainers want this capability in core, land it as a policy-aware sibling of `sessions_send`: use the declared `typebox` dependency, use the existing `inter_session` provenance contract or formally extend the schema with tests/docs, annotate received content as non-user inter-session data, add matching deny coverage for subagents, Gateway HTTP, realtime direct tools, sandbox/default policy, and mutating-tool tracking, then update session-tool docs and changelog expectations.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: the PR's `agent` calls contain `inputProvenance.kind: "broadcast"`, while current gateway validation only accepts `external_user`, `inter_session`, and `internal_system`; the policy gap is reproduced by comparing the new default tool registration with the existing `sessions_send` deny lists.

## Solution Assessment

No. The current PR is a useful implementation sketch, but it is not the best mergeable solution because it bypasses existing dependency, provenance, and restricted-surface contracts for cross-session injection.

  - body: Current main blocks `sessions_send` for subagents and direct Gateway HTTP/realtime tool surfaces, but this PR registers `sessions_broadcast` without equivalent deny-list coverage.
