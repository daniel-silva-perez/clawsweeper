## Summary

Keep open. Current main still does not implement `sessions_broadcast`, and the PR remains blocked by concrete merge issues: the new tool imports an undeclared package, sends a provenance kind rejected by the current Gateway schema, and exposes a new cross-session injection primitive without matching the restricted-tool policy coverage used for `sessions_send`.

## What This Changes

This PR adds a core `sessions_broadcast` tool, registers it in the tool catalog and OpenClaw tool factory, and adds tests for broadcasting messages to active sessions.

## Best Possible Solution

If maintainers want this as a core primitive, land it as a policy-aware sibling to `sessions_send`: use the existing `inter_session` provenance contract or intentionally extend the schema with docs/tests, annotate received content as non-user inter-session data, add deny coverage for subagents, Gateway HTTP, realtime direct tools, and mutation tracking, then update session-tool docs and profile expectations.

  - body: Current package metadata declares `typebox`, and existing tools import `Type` from that package. Importing `@sinclair/typebox` here relies on an undeclared package name, so installs using pnpm's strict layout can fail to resolve the new tool.
  - confidence: 0.9

  - body: Every broadcast `agent` call sends `inputProvenance.kind: "broadcast"`, but current Gateway validation only accepts `external_user`, `inter_session`, and `internal_system`. These requests can be rejected as invalid before reaching the target sessions.
  - confidence: 0.94
- **[P1] Apply broadcast to restricted-tool policy:** `src/agents/openclaw-tools.ts:189-194`
  - body: Registering this tool in the default tool set exposes a new cross-session injection path, but the PR does not add `sessions_broadcast` to the subagent, Gateway HTTP, or realtime direct-tool deny surfaces that already block `sessions_send`. Add matching policy coverage and tests before enabling it broadly.
  - confidence: 0.88
