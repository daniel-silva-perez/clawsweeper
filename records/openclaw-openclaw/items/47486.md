## Summary

Keep open, but this PR is not merge-ready. Current main still has no `sessions_broadcast` tool, so the contribution is not obsolete; however the branch violates the current TypeBox dependency contract, sends an invalid Gateway input-provenance kind, and exposes a new cross-session injection path without matching the existing restricted-tool policy surfaces for `sessions_send`.

## What This Changes

The PR adds a core `sessions_broadcast` agent tool, registers it in the tool catalog and OpenClaw tool factory, and adds unit tests for broadcasting messages to active sessions with optional channel scope filtering.

## Best Possible Solution

If maintainers want this capability in core, land it as a policy-aware sibling of `sessions_send`: use the declared `typebox` dependency, use the existing `inter_session` provenance contract or formally extend the protocol with tests/docs, annotate received content as non-user inter-session data, add deny/mutation coverage across subagents, Gateway HTTP, realtime direct tools, sandbox/default policy, and mutating-tool tracking, then update the session-tool docs and changelog.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: the PR's `agent` calls send `inputProvenance.kind: "broadcast"`, while current main's Gateway schema only accepts `external_user`, `inter_session`, and `internal_system`; the policy gap is reproduced by comparing the new default registration with the existing `sessions_send` deny and mutation surfaces.

## Solution Assessment

No. The PR is a useful implementation sketch, but it is not the best mergeable solution because it bypasses current dependency, Gateway provenance, and restricted-tool contracts for cross-session injection.
