## Summary

Keep this PR open. Current main still does not implement a `sessions_broadcast` tool, and the contribution is not merge-ready because the PR head uses an undeclared TypeBox import, sends an input provenance kind rejected by the current gateway schema, and adds a cross-session injection tool without updating the policy surfaces that already restrict `sessions_send`.

## What This Changes

This PR adds a built-in `sessions_broadcast` session tool, registers it in the core tool catalog and OpenClaw tool factory, and adds tests for broadcasting messages to active sessions.

## Best Possible Solution

If maintainers want this core primitive, implement it as a policy-aware sibling to `sessions_send`: use the existing provenance contract or intentionally extend it with schema/docs/tests, add deny coverage for subagents, Gateway HTTP, realtime direct tools, and mutation tracking, then update session-tool docs and profile/group expectations.

  - body: Current package metadata declares `typebox`, and the existing tools import `Type` from that package. Importing `@sinclair/typebox` here adds an undeclared dependency name, so the new tool can fail to resolve under pnpm’s strict package layout.

  - confidence: 0.93
- **[P1] Deny broadcast on restricted tool surfaces:** `src/agents/openclaw-tools.ts:189-194`
  - body: Registering this tool in the default tool set exposes a new cross-session injection path, but the PR does not add `sessions_broadcast` to the subagent deny list or the Gateway HTTP/realtime direct-tool deny surfaces that already block `sessions_send`. Add matching policy coverage and tests before enabling the tool broadly.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The PR introduces a security-sensitive cross-session injection surface without updating the existing deny-list policy boundaries for comparable session-send tools.

Concerns:

- **[high] Broadcast bypasses sensitive tool deny surfaces:** `src/agents/openclaw-tools.ts:189`
  - body: Current main blocks `sessions_send` for subagents and direct Gateway HTTP by default, but the new `sessions_broadcast` tool is registered without equivalent deny-list coverage. That can expose cross-session message injection in contexts where direct session sends are intentionally unavailable.
  - confidence: 0.88
- **[medium] Undeclared TypeBox import changes package resolution:** `src/agents/tools/sessions-broadcast-tool.ts:2`
  - body: The PR imports `@sinclair/typebox` even though the repo declares and uses `typebox`. Besides breaking builds, relying on an undeclared package name can make dependency resolution depend on transitive layout rather than the package contract.
  - confidence: 0.9
