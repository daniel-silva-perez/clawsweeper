## Summary

Keep this PR open. Current main still does not implement the PR’s intended cleanup: TLS chmod failures are still swallowed, fs-safe close failures are still suppressed, malformed allow-from JSON still falls back without logging in the current helper module, and the internal hook non-null assertion remains. The submitted PR is also not merge-ready as-is because it logs TLS permission failures at info level, patches a stale pairing-store path for the allow-from behavior, and passes `{ error }` where `logWarn` expects a `RuntimeEnv`, which can make cleanup/fallback paths throw.

## Best Possible Solution

Keep this PR open for author or maintainer revision. A safe version should rebase onto current main, log TLS chmod failures at warn severity, add a close helper that reports errors without throwing, move malformed allow-from JSON logging into `src/pairing/allow-from-store-file.ts` while preserving the `exists: true` empty fallback, keep the internal-hook type cleanup, and add focused regression tests for those error paths.
