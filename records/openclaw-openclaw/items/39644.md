## Summary

Keep this PR open. Current main still advertises PowerShell completion and generates a `.ps1` cache, but the installer path still rejects PowerShell, still writes POSIX `source` syntax, and Windows time-format detection still hard-codes `powershell` instead of the existing resolver that prefers PS7. The PR remains useful but needs a manual port because current main now keeps install behavior in `src/cli/completion-runtime.ts`.

## Best Possible Solution

Keep this PR open, or supersede it only with a landed port. The fix should update `src/cli/completion-runtime.ts` to install PowerShell completion using `. "..."`, emit a PowerShell-safe reload hint, update `src/agents/date-time.ts` to use `resolvePowerShellPath()` with non-interactive flags, and add focused regression tests for PowerShell profile install/detection plus Windows date-time executable selection.
