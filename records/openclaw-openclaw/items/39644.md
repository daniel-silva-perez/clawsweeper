## Summary

Current main still has the Windows/PowerShell bugs this PR targets. PowerShell is advertised as a completion shell and completion scripts can be generated, but `installCompletion()` still falls through to the unsupported branch for PowerShell, still formats profile source lines with POSIX `source`, and still prints a POSIX reload hint. Windows time-format detection also still hard-codes `powershell` instead of using the existing resolver that prefers PS7. The PR remains useful but needs a manual port because main now uses `src/cli/completion-runtime.ts` for the install path.

## Best Possible Solution

Keep this PR open, or supersede it only with a landed port. The fix should update `src/cli/completion-runtime.ts` to install PowerShell completion using `. "..."`, emit a PowerShell-safe reload hint, update `src/agents/date-time.ts` to call `resolvePowerShellPath()` with non-interactive flags, and add focused regression tests for PowerShell install detection/profile content and Windows date-time executable selection.
