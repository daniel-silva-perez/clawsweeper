## Summary

Keep this PR open. Current main still only supports Slack `thread.requireExplicitMention` at the account/global Slack config level; the per-channel override proposed here is not implemented in the current strict schema, exported Slack config types, channel resolver, mention gate, docs, or tests. The PR is a focused implementation candidate, but it needs a rebase and a small contract cleanup before it is mergeable.

## Best Possible Solution

Keep this PR open and request a focused revision: add a narrow channel-level thread config type/schema that only supports `requireExplicitMention`, resolve exact channel then wildcard then account default in `resolveSlackChannelConfig`, use the resolved value in Slack mention gating, add resolver and message-preparation tests for both override directions and wildcard fallback, and update docs, generated config metadata, and changelog for the new public config key. After that, run the targeted Slack tests and the normal changed gate before merge.
