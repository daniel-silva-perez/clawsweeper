## Summary

Keep open. Current main still logs config warnings directly in the config load and write paths, so the PR is not implemented-on-main; the proposed patch is useful but has a P2 edge-case bug and overlaps with the broader open maintainer PR #38295.

## What This Changes

The PR adds an in-memory per-config-path warning fingerprint cache around `loadConfig()` warning logs and adds regression tests for repeated loads of unchanged and changed warning-producing configs.

## Best Possible Solution

A single config-warning dedupe implementation should live in `src/config/io.ts`, cover both load and write warning emitters, clear per-path state on no-warning and early-exit transitions, re-log when source content or formatted warning details change, and include focused regression tests for load/write repeats plus warning-to-non-warning-to-warning transitions.
