## Summary

Keep open. The stale Feishu binding cleanup is a valid problem, but the submitted patch is not mergeable because an agent-id mismatch is not a safe stale-binding predicate. Current main and docs support intentional cross-agent ACP/subagent conversation bindings, so the PR needs a narrower cleanup rule that does not delete explicit bindings.

## What This Changes

The PR changes Feishu inbound routing to unbind runtime conversation bindings when the bound session agent differs from the resolved route agent, adds regression coverage, and updates Feishu lifecycle fixtures to use main-agent session keys.

## Best Possible Solution

Implement Feishu stale cleanup with a narrow route-shaped rule: remove only legacy session bindings whose parsed Feishu route matches the current conversation route except for agent id, while preserving explicit ACP, subagent, plugin-owned, and metadata-backed conversation bindings.
