## Summary

Keep this PR open, but the current patch should not merge as-is. The stale Feishu binding cleanup goal is valid, yet the proposed agent-mismatch unbind would also delete intentional cross-agent runtime bindings used by `/focus` and ACP bound conversations.

## What This Changes

The PR changes Feishu inbound routing to unbind a runtime conversation binding when the bound session agent differs from the resolved route agent, and adds/adjusts Feishu tests around that cleanup path.

## Best Possible Solution

Implement stale cleanup with a route-shaped predicate: remove only legacy Feishu session bindings that structurally match the current conversation route except for agent id, while preserving explicit ACP, subagent, plugin-owned, and metadata-backed runtime conversation bindings.

## Reproduction Assessment

Yes. A high-confidence review reproduction is to create a Feishu runtime binding via `/focus` or `/acp spawn --bind/--thread` to a target session whose agent differs from the resolved route agent, then process the next inbound message through the PR branch; the new mismatch path unbinds that valid binding.

## Solution Assessment

No. Blanket cleanup on agent mismatch is not the best fix because agent mismatch is also the expected shape of intentional cross-agent bindings; the safer fix is a narrow stale-route detector with regression coverage for both stale legacy leftovers and valid explicit bindings.
