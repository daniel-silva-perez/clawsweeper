## Summary

Keep this PR open. Current main does not implement the PR's intended active cleanup of stale Feishu cross-agent conversation bindings, so it cannot be closed as implemented. The submitted approach also is not safe to merge as-is: it treats any bound-session agent mismatch as stale, while current routing code, tests, and ACP docs allow bound conversations to intentionally route from a default/main route to another agent's ACP or focused session.

## Best Possible Solution

Revise the PR instead of closing it: keep valid cross-agent runtime bindings, add regression coverage that preserves /focus and ACP cross-agent Feishu bindings, and implement cleanup only when a binding is provably stale or invalid. The likely home is the shared conversation-binding/session-binding path or a Feishu adapter helper, with Feishu-specific tests for topic-scoped stale cleanup plus valid cross-agent persistence.

## Risks / Open Questions

- Merging the PR as written can remove valid /focus or ACP cross-agent bindings after the next inbound Feishu message, causing follow-ups to fall back to the route agent instead of the explicitly bound session.
