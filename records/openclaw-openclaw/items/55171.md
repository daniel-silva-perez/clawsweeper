## Summary

Keep this PR open for revision. The stale Feishu binding cleanup goal is valid, but the patch's agent-mismatch rule is too broad and would remove intentional cross-agent conversation bindings; current main also moved this path into shared runtime binding routing, so the fix should be rebased around the current helper rather than the older inline Feishu block.

## What This Changes

The branch modifies Feishu message routing to unbind a bound conversation when the bound session's agent differs from the resolved route agent, adds a bot regression test, and adjusts Feishu lifecycle fixtures to use main-agent session keys.

## Best Possible Solution

Keep the stale-binding cleanup objective, but implement a narrower Feishu rule modeled on Discord: clean up only route-shaped legacy bindings whose parsed session route matches the current route except for agent id, while preserving explicit ACP, subagent, plugin-owned, and metadata-backed conversation bindings.

Those bindings carry explicit target kind or metadata and should keep routing follow-up messages to the bound session; unbinding them here makes the next message fall back to the route agent instead.
