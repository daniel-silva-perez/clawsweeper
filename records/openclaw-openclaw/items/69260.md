## Summary

Keep this issue open. Current main still has no ACP auth-contract metadata or Gemini-specific ACPX launch hardening: ACP runtime/session/config types only carry existing model/thinking/cwd/permission/timeout options, the manager forwards only those options into runtime.ensureSession, and the acpx wrapper delegates Gemini as an ordinary non-Codex harness without auth-env scrubbing or fail-closed preflight.

## What This Changes

The issue requests ACP auth-contract metadata, ambient provider auth-env scrubbing, fail-closed preflight hooks, and mixed-credential-host documentation for Gemini and other ACP-backed agents.

## Best Possible Solution

Keep this open for maintainer product/security review. The best path is an additive ACP auth-contract seam shared by global ACP config, per-agent runtime defaults, persistent bindings, session runtime options, and runtime.ensureSession; then the acpx plugin can apply Gemini-specific Gemini/GenAI auth-env scrubbing and fail-closed preflight using that seam, with docs for mixed-credential hosts and disposable-state regression coverage.
