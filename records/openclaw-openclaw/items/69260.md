## Summary

Keep this issue open. Current main still does not implement the requested ACP auth-contract surface or Gemini-specific ACP launch hardening: runtime/config/session types have no expected-auth, fallback, env-scrub, or preflight contract; the ACP manager does not pass such policy into runtime.ensureSession; and the acpx wrapper still delegates Gemini as a normal non-Codex harness without Gemini/GenAI auth-env scrubbing or fail-closed preflight.

## Best Possible Solution

Keep this open for maintainer product/security review. The best path is an additive ACP auth-contract seam shared by global ACP config, per-agent runtime defaults, persistent bindings, session runtime options, and runtime.ensureSession; then acpx can use that seam for Gemini-specific Gemini/GenAI env scrubbing and fail-closed auth preflight, with mixed-credential docs and disposable-state regression coverage.
