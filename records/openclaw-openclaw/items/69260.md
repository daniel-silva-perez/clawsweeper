## Summary

Keep open. Current main still lacks ACP auth-contract metadata, Gemini/GenAI auth-env scrubbing, fail-closed auth preflight hooks, and mixed-credential ACP docs. The existing ACP config/runtime surfaces only carry routing, session, model/thinking, cwd, and backend options, while the acpx plugin delegates Gemini as a normal harness path.

## Best Possible Solution

Keep this open for maintainer product/security review. The likely direction is an additive ACP auth-contract seam usable by global config, agent runtime defaults, and bindings; acpx should then implement Gemini-specific env scrubbing and fail-closed preflight through that generic seam, with docs for mixed OAuth/API-key hosts and staged regression coverage.
