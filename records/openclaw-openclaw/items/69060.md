## Summary

Keep this PR open. Current main still does not implement the proposed `x-openclaw-session` fallback: the OpenAI-compatible HTTP session resolver only honors `x-openclaw-session-key`, then the OpenAI `user` field, then a random UUID. The PR remains a plausible narrow implementation candidate for the session-warm/QMD part of open #68920, but it needs maintainer review, focused tests, docs, and a small security hardening pass around the new caller-controlled header value before landing.

## Best Possible Solution

Keep this PR open as the narrow implementation candidate. The best path is to revise or review it to add `x-openclaw-session` after `user` and before random UUID fallback, keep `x-openclaw-session-key` as the full override, validate or bound the hint, add regression tests for priority/order and agent scoping, and document the new header on the OpenAI-compatible HTTP API page while leaving the broader lightContext work tracked in #68920.
