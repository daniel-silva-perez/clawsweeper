## Summary

Keep this PR open. Current main still does not implement the proposed `x-openclaw-session` fallback for OpenAI-compatible HTTP sessions; the resolver only honors `x-openclaw-session-key`, then the OpenAI `user` field, then a random UUID. The PR remains a plausible narrow implementation candidate for the session-warm/QMD part of #68920, but needs maintainer review, focused tests/docs, and input validation for the new caller-controlled header before landing.

## Best Possible Solution

Keep this PR open as the narrow implementation candidate. The best path is to revise or review it to add `x-openclaw-session` after `user` and before the random UUID fallback, preserve `x-openclaw-session-key` as the full override, validate or bound the hint, add focused Gateway HTTP regression tests for priority and agent scoping, and document the new header on the public OpenAI-compatible HTTP API page while leaving the broader lightContext work tracked in #68920.
