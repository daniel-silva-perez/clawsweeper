## Summary

Keep open. Current main still routes Mattermost draft-preview phase changes through one editable preview post, and the requested `streaming.mode="block"` boundary split is not implemented in Mattermost schema/types/runtime. The issue is already paired with open closing PR #75256, so the right cleanup path is to review and land or revise that PR rather than close this issue early.

## What This Changes

Mattermost users request a draft-preview streaming mode that preserves earlier phase content by splitting preview posts at reasoning, tool, and assistant-message boundaries instead of repeatedly editing one post.

## Best Possible Solution

Keep the fix Mattermost-local: add a validated account/top-level preview streaming mode, preserve current edit-in-place behavior by default, and in block mode split the draft preview at assistant, reasoning, tool-start, and tool-end boundaries with focused Mattermost draft-stream, account, schema, and monitor tests.

## Reproduction Assessment

Yes. The issue has a live Mattermost DM/tool-call reproduction path, and the current source gives a high-confidence code reproduction because every phase transition updates the same draft stream post until `forceNewMessage()` or cleanup clears the post id.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix: match the existing Discord boundary-splitting semantics inside the Mattermost plugin rather than changing shared core dispatch. The implementation should be reviewed alongside #75256 and #73252 because both touch the same streaming-mode contract.
