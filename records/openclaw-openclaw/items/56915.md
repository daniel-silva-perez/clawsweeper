## Summary

Current main still does not implement the reported default discovery path for `~/.openclaw/skills/<group>/<skill>/SKILL.md`. The loader scans immediate child directories only, while the issue has two open implementation PRs with closing syntax, so this issue should remain open until a canonical fix lands or those PRs are closed.

## Best Possible Solution

Keep this issue open. Review and land one canonical bounded grouped-discovery fix in `src/agents/skills/workspace.ts`, add regression coverage for `~/.openclaw/skills/<group>/<skill>/SKILL.md`, then let the merged PR close this issue and close or deduplicate the overlapping PR.
