## Summary

Current main still skips the reported `~/.openclaw/skills/<group>/<skill>/SKILL.md` layout. The scanner enumerates only immediate children of the managed skills root and checks each child for `SKILL.md`, while two open PRs (#69663 and #72534) are active implementation candidates for the same issue, so this should remain open until one canonical fix lands or the PRs are closed/deduplicated.

## Best Possible Solution

Keep this issue open. Review and land one canonical bounded grouped-discovery fix in `src/agents/skills/workspace.ts`, add regression coverage for `~/.openclaw/skills/<group>/<skill>/SKILL.md` in `src/agents/skills.loadworkspaceskillentries.test.ts`, then let the merged PR close the issue and close or deduplicate the overlapping PR.
