## Summary

Keep this PR open. The underlying payload leak still exists on current main, but the branch targets the old chat-history sanitizer layer; current display paths use the shared projector, so the useful redaction work needs to be ported there before merge.

## What This Changes

The PR branch adds inline image payload redaction and accumulated omitted-byte accounting to gateway chat.history sanitization, with regression tests for source-data and multi-payload image blocks.

## Best Possible Solution

Port the redaction into src/gateway/chat-display-projection.ts, preserve accumulated omitted-byte accounting, cover image source.data, image data-URL url, OpenAI-style image_url.url, and malformed multi-payload blocks across projector/history/live-display tests, and leave durable transcript storage changes to #1210.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to project a transcript message containing content[].source.data on a type:image block; current main's projector does not delete that nested image data, and chat.history plus session.message use that projector.

## Solution Assessment

No. The PR's intent and byte accounting are good, but the implementation should move to the shared display projector so every current display surface uses one sanitizer.
