## Summary

Keep this PR open. Current main still does not implement the PR's intended chat-history sanitizer for inline image payload shapes such as image `source.data`, image data-URL `url`, or OpenAI-style `image_url.url`. The PR remains a plausible focused fix, but it needs to be ported from the older server-methods helper into the current shared display projection path.

## Best Possible Solution

Keep this PR open for maintainer review or contributor follow-up. The best path is to port the inline-image sanitizer into `src/gateway/chat-display-projection.ts`, preserve accumulated omitted-byte accounting, add projection/history tests for image `source.data`, image data-URL `url`, OpenAI-style `image_url.url`, and malformed multi-payload blocks, while leaving the broader durable transcript-media storage work tracked by #1210.
