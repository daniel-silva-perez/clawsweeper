## Summary

Keep this PR open. Current main still does not implement the proposed one-shot continuity hint after daily/idle session rollover, and the PR remains a plausible implementation path if its review issues are addressed.

## Best Possible Solution

Keep this PR open for a rebased implementation or a canonical replacement. The best fix is to capture a bounded assistant-only transcript tail during scheduled daily/idle rollover before archive, delimit or sanitize it as untrusted prior output, inject it through both immediate and queued prompt paths, clear it only after actual delivery, preserve explicit `/new` and `/reset` as blank slates, and add focused regression tests.

## Risks / Open Questions

- The PR as proposed interpolates previous assistant text into a prompt wrapper without strong delimiting or sanitization, leaving a small indirect prompt-injection surface for multiline or instruction-shaped prior output.
- The current prompt construction means a hint added only through `applySessionHints` can be omitted from queued/deferred follow-up prompts while still being cleared.
- The PR lacks focused regression coverage for daily rollover, idle rollover, explicit `/new` and `/reset` exclusion, one-shot clearing, multiline assistant text, and queue modes.
