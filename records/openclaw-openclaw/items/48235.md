## Summary

Keep this PR open. Current main at 891c7d9f1c1f5e0f52935fadfeacf695b0ef1eb8 still does not implement the proposed one-shot continuity hint after daily/idle session rollover, and the PR remains a plausible fix path if its review issues are addressed.

## Best Possible Solution

Keep this PR open for a rebased implementation or a clearly canonical replacement. The best fix is to capture a bounded assistant-only transcript tail during scheduled daily/idle rollover before archive, delimit or sanitize it as untrusted prior output, inject it through both immediate and queued prompt paths, clear it only after actual delivery, preserve explicit `/new` and `/reset` as blank slates, and add focused regression tests.

## Risks / Open Questions

- The proposed diff interpolates previous assistant text into a prompt wrapper without strong delimiting or sanitization, which creates a small indirect prompt-injection surface for multiline or instruction-shaped content.
- The proposed clear-after-apply behavior can clear the hint before the model actually receives it in queued/deferred run paths.
- The PR currently lacks focused regression coverage for daily and idle scheduled rollovers, explicit `/new` and `/reset` exclusion, one-shot delivery, multiline assistant text, and queue modes.
