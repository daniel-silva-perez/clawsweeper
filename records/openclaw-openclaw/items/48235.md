## Summary

Keep this PR open. Current main has a related, newer transcript-tail replay for silent in-reply session rotations, but it does not implement this PR's scheduled daily/idle reset continuity hint. The PR also still needs design fixes around prompt delimiting, one-shot clearing, queued delivery, and regression coverage before it is merge-ready.

## Best Possible Solution

Keep this PR open for a rebased implementation or a canonical replacement. The best path is to handle scheduled daily/idle rollover in `initSessionState` before transcript archive, capture only bounded assistant-derived continuity context or reuse a safer transcript-tail strategy, delimit prior output as untrusted, ensure both immediate and queued prompt paths receive it before clearing, preserve explicit `/new` and `/reset` as blank slates, and add targeted regression tests.

## Risks / Open Questions

- The PR's current prompt wrapper interpolates previous assistant text without strong delimiting or sanitization, leaving a small indirect prompt-injection surface for multiline or instruction-shaped prior output.
- A hint applied and cleared through the existing `applySessionHints` shape can be omitted from queued/deferred prompt bodies while still being consumed.
- The current main branch now has #70898's transcript-tail replay helper for a neighboring continuity problem; the best implementation for scheduled daily/idle resets may want to reuse or deliberately avoid that approach rather than merge this PR unchanged.
- The PR does not include focused regression coverage for daily rollover, idle rollover, explicit `/new` and `/reset` exclusion, one-shot clearing, multiline prior output, and queue modes.
