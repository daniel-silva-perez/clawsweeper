## Summary

Current main still binds `TaskRecord.task` directly into the SQLite `task_runs.task` parameter, and a live in-memory `node:sqlite` check shows object values fail binding. The PR’s intended fix remains relevant, narrow, and not implemented on current main; Greptile’s existing review also found the proposed change minimal and safe.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent narrow fix that serializes non-string task values before SQLite binding, preserves string values unchanged, and adds the structured-payload regression test in `src/tasks/task-registry.store.test.ts`.
