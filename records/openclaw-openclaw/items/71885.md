Current main still binds `record.task` directly into the SQLite task row, and an in-memory `node:sqlite` check still rejects object values with the same binding error this PR targets. ## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase and land this PR or an equivalent narrow fix that serializes non-string task values before SQLite binding, preserves existing string values and the read path, and keeps the structured-payload regression in `src/tasks/task-registry.store.test.ts`.
