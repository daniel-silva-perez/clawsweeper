## Summary

Keep this PR open. Current main still emits zsh leaf-command completion specs from options only, while the reported `wiki ingest` command still has a filesystem positional argument. The PR remains the focused implementation candidate, with related duplicate PRs already closed and a maintainer repair pushed to keep this contributor path canonical.

## Best Possible Solution

Keep this PR open for normal maintainer review and landing. The best path is to land the contributor-preserving zsh positional completion implementation with regression tests in `src/cli/completion-cli.test.ts`, keep the heuristic narrow for non-filesystem `<path>` arguments, and coordinate final zsh escaping behavior with #64490 before merge.
