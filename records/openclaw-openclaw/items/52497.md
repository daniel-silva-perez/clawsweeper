## Summary

Keep open. Current main still can collapse an available target-run lifecycle/model error into a bare `status: "timeout"`: `sessions_send` passes through an error only when `agent.wait` returns one, while the gateway wait path still returns a bare timeout when lifecycle/dedupe snapshots are unavailable before the wait timer expires.

## Best Possible Solution

Keep this issue open and implement a narrow gateway-side improvement: carry recent lifecycle/model failure context through `waitForAgentJob` or the `agent.wait` timeout response, then rely on the existing `sessions_send` pass-through to expose that error to the sender. Add targeted tests for a target run that emits a model/runtime error before the caller wait budget expires, plus a true no-error timeout case.
