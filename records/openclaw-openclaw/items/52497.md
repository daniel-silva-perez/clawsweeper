## Summary

Current main has not fully implemented the requested diagnostic. The `sessions_send` tool will include `error` only if `agent.wait` returns one, but the gateway wait path can still return a bare `{ status: "timeout" }` when no terminal snapshot is available before the wait timer expires. There is also an existing gateway test where a lifecycle `error: "boom"` arrives, yet `agent.wait` still resolves as a timeout because the retry-grace window outlasts the requested wait.

## Best Possible Solution

Keep this issue open and implement a narrow gateway-side improvement: carry recent lifecycle/model failure context through `waitForAgentJob` or `agent.wait` timeout responses, then let the existing `sessions_send` pass-through surface that error to the sender. Add targeted tests for a target run that emits a model/runtime error before the caller's wait budget expires, plus a no-error true-timeout case.
