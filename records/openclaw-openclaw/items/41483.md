## Summary

Keep this issue open. Current main still resolves `--agent` requests to the agent main session before `--channel`/`--to` can produce a per-recipient key, and open PR #73403 explicitly tracks the implementation candidate. The earlier focused PR #41557 was closed unmerged, so there is no shipped fix to close against yet.

## Best Possible Solution

Keep this issue open and handle PR #73403 as the active implementation candidate. The fix should define precedence for `--session-key`, `--session-id`, `--agent`, `--channel`, and `--to`; derive per-recipient agent-scoped keys for explicit channel targets; preserve existing main-session behavior where intended; add focused Gateway and embedded regression coverage for distinct bridge targets; and update the CLI/session-routing docs when the behavior changes.
