## Summary

Keep this issue open. Current main still resolves an explicit `--agent` path to that agent's main session before `--to` can create per-recipient routing, and the supplied GitHub context shows PR #73403 as an open, unmerged implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review PR #73403 as the active implementation candidate. The fix should define precedence for `--session-key`, `--session-id`, `--agent`, `--channel`, and `--to`; derive agent-scoped per-recipient keys for explicit deliverable-channel targets; preserve intentional main-session behavior; add resolver, Gateway CLI, and Gateway server regression coverage; and update `openclaw agent` docs if session-routing behavior changes.
