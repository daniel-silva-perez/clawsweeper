## Summary

Keep this issue open. Current main still lets an explicit `--agent` resolve to that agent's main session before `--channel`/`--to` can derive a per-recipient key, and PR #73403 is still open as the active implementation candidate. The public PR page shows it is open and says it fixes this issue by deriving per-recipient agent session keys for `openclaw agent --agent ... --channel ... --to ...`. citeturn1view3

## Best Possible Solution

Keep this issue open and review PR #73403 as the active implementation candidate. The fix should define precedence for `--session-key`, `--session-id`, `--agent`, `--channel`, and `--to`; derive agent-scoped per-recipient keys for explicit channel targets using the existing session-key helpers; preserve intentional main-session behavior; add resolver, Gateway CLI, and Gateway server regression coverage for distinct bridge targets; and update `openclaw agent` docs when behavior changes.

## Risks / Open Questions
