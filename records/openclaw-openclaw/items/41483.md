## Summary

Keep this issue open. Current main still lets an explicit `--agent` resolve to that agent's main session before `--to` can create per-recipient routing, and the provided GitHub context shows PR #73403 as an open, unmerged implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review PR #73403 as the active implementation candidate. The fix should define precedence for `--session-key`, `--session-id`, `--agent`, `--channel`, and `--to`; derive agent-scoped per-recipient keys for explicit deliverable-channel targets; preserve intentional main-session behavior where documented; and add resolver, Gateway CLI, Gateway server, and docs coverage for the changed session-routing contract.
