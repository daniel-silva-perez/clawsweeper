## Summary

Keep this issue open. Current main still resolves `openclaw agent --agent <id> --channel <channel> --to <target>` through an agent-main session key before channel-aware recipient routing can affect the session, and PR #73403 is an open, unmerged implementation candidate that explicitly closes this bug.

## Best Possible Solution

Keep this issue open and handle PR #73403 as the active implementation candidate. The fix should define precedence for `--agent` with explicit `--channel`/`--to`, preserve explicit `--session-key` and existing `--session-id` behavior, derive agent-scoped per-recipient keys through the existing routing/session helpers, add Gateway and embedded regression coverage for distinct `cw_111`/`cw_222` targets, and update CLI docs once behavior changes.
