## Summary

Keep this issue open. Current main still routes `openclaw agent --agent <id> --channel <channel> --to <target>` through an agent-main session key before channel-aware per-recipient routing can participate, and PR #73403 is an open, unmerged implementation candidate with closing syntax for this bug.

## Best Possible Solution

Keep this issue open and treat PR #73403 as the active implementation candidate. The fix should define precedence for `--agent` plus explicit `--channel`/`--to`, preserve explicit `--session-key` and `--session-id` behavior, derive agent-scoped per-recipient keys through the existing routing/session helpers when isolated direct-message routing is intended, add Gateway and embedded regression coverage for distinct `cw_111`/`cw_222` targets, and update the CLI docs once behavior changes.
