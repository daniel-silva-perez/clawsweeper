## Summary

Keep this PR open. Current main already has the Google Chat threaded-send portion, but it still does not expose `channels.googlechat.sessionThread` or derive per-thread Google Chat session keys. The PR remains a plausible implementation candidate, especially after its review discussion addressed space-based routing, cached route mutation, mixed-case thread names, envelope timestamp leakage, and explicit `replyToId` precedence.

## Best Possible Solution

Keep this PR open and review or adapt it as the implementation candidate for `channels.googlechat.sessionThread`: add the config/schema/generated/docs surface, derive a thread-scoped Google Chat session key without changing space-based route matching, pass that key through envelope/session metadata, preserve current threaded-send behavior and explicit reply targets, and keep focused regression coverage for distinct threads in the same space.
