## Summary

Keep this PR open. Current main still redirects a configured skills root to its nested `skills` subdirectory before checking whether the configured root already contains loadable skills, so the user-visible bug described by the PR remains present. The PR is a focused skills-loader change with a targeted regression test and no visible unrelated security-sensitive surface in the provided diff.

## What This Changes

The PR updates skills discovery to avoid redirecting to a nested `skills` directory when the configured root already contains a valid root-level skill, passes the skill file size cap into the probe, and adds a regression test for the root-plus-nested layout.

## Best Possible Solution

Land a reviewed version of this PR that checks whether the configured directory is already a loadable skills root before applying the repo-style nested-root heuristic, while preserving containment checks, `SKILL.md` size caps, nested repo compatibility, and a regression test for the root-plus-nested layout.
