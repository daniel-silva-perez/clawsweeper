## Summary

Keep open. Current main still does not extract inline reasoning from assistant string payloads in the TUI formatter paths, and the PR is useful but not safe to merge as written because it can rewrite non-assistant transcript text and does not mirror the shared nested-tag contract.

## What This Changes

The PR adds shared inline `<think>...</think>` extraction, routes TUI string content through that split, and adds formatter regression tests for tagged assistant strings.

## Best Possible Solution

Land a current-main TUI hardening patch that applies inline reasoning parsing only to assistant string content, preserves user and command strings literally, and shares the existing reasoning-tag semantics for nested tags, malformed tags, final tags, and code-region exclusions across history reload and live stream formatting.

## Reproduction Assessment

Yes. A high-confidence unit reproduction is an assistant message with string content like `<think>hidden</think>Visible`; source inspection shows current main returns no thinking text for string content and live TUI assembly uses the unsplit string content path.

## Solution Assessment

No, not as written. The direction is right, but the maintainable fix should be role-aware and should reuse or extend the shared reasoning-tag contract instead of adding a divergent boolean parser and routing every generic string through it.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

  - confidence: 0.82
