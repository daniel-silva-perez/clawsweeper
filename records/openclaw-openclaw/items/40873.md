## Summary

Keep open. Current main still has the TUI tagged-string leak path, so the PR is not obsolete; however, the proposed patch is not merge-ready because it applies reasoning-tag parsing to non-assistant string transcripts and its new extractor does not preserve the nested-depth contract of the existing stripper.

## What This Changes

The PR adds inline reasoning-tag extraction for TUI string content, routes tagged string content through thinking/content splitting, and adds formatter regression coverage for assistant string payloads.

## Best Possible Solution

Revise this into a current-main, assistant-only TUI hardening patch: split assistant string content into reasoning and visible text, preserve literal tags in user/command transcript text, keep nested and malformed tag behavior aligned with the shared stripper, and cover formatter, live stream, and history reload paths.

## Reproduction Assessment

Yes. The source path is explicit: an assistant message with string content like `<think>hidden</think>Visible` reaches TUI stream/history formatting through helpers that currently preserve the string content and return no separate thinking text.

## Solution Assessment

No, not as written. The direction is right, but the safer solution is a role-aware helper or assistant-only branch that reuses the existing reasoning-tag contract instead of changing the generic string block path for all transcript roles.
