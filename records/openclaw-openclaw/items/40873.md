## Summary

Keep open. The PR is useful source work for the linked TUI reasoning-tag leak, and the Greptile comment correctly identified the missing string-content path, but current main still does not fully implement the intended behavior and the proposed patch has a blocking role-boundary regression.

## What This Changes

The PR adds inline `<think>...</think>` extraction for TUI string content, splits reasoning from visible assistant text, and adds formatter regression tests for tagged assistant string payloads.

## Best Possible Solution

Revise this into a current-main, assistant-only TUI hardening patch: split inline reasoning from assistant string content, keep user and command strings literal, keep nested and malformed tag behavior aligned with the shared stripper, and cover history reload plus live stream paths.

## Reproduction Assessment

Yes. A high-confidence unit reproduction is an assistant message with string content like `<think>hidden</think>Visible`; current main’s formatter paths either return empty thinking or render the raw sanitized string content.

## Solution Assessment

No, not as written. The direction is right, but the safer solution is a role-aware assistant-string helper that reuses the existing reasoning-tag contract and leaves generic string rendering alone.
