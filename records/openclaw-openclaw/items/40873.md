## Summary

Keep this PR open. Current main still does not implement the PR’s intended TUI behavior for assistant messages whose `content` is a plain string containing inline `<think>...</think>` reasoning. The PR targets a real remaining gap, but the proposed diff still needs an assistant-only role gate before merge so user and command transcript text preserve literal tag examples.

## Best Possible Solution

Rebase and revise this PR into a narrow assistant-only TUI fix: split assistant string content into reasoning and visible text, hide tagged reasoning when thinking display is off, surface it separately when thinking display is on, preserve literal tags in user and command messages, and add formatter plus stream-assembler regression coverage.
