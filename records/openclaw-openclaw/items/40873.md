## Summary

Keep this PR open. Current main has shared reasoning-tag sanitizers, but the TUI formatter and stream assembler still do not strip or separately surface inline `<think>...</think>` reasoning when assistant payloads arrive as plain string content. The PR remains relevant, but its proposed diff still needs a role-gated adjustment before it is safe to merge.

## Best Possible Solution

Keep this PR open for rebase and integration. The best fix is a narrow, assistant-only TUI formatter/stream-assembler change that strips inline tagged reasoning from visible assistant content when thinking display is off, surfaces that reasoning separately when thinking display is on, preserves literal tags in user and command messages, and reuses the existing shared reasoning-tag utilities where possible.
