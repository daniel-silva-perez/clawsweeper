## Summary

Keep this PR open, but not mergeable as written. Current main still lacks TUI-side extraction for inline tagged reasoning in assistant string content, while the PR’s current diff applies tagged parsing too broadly to generic string transcript content and diverges from the existing nested-tag stripping contract.

## What This Changes

The PR adds shared extraction for inline `<think>...</think>` text, routes TUI string content through that split, and adds formatter regression tests for assistant string payloads.

## Best Possible Solution

Revise this into a current-main TUI hardening patch that splits inline reasoning only for assistant string content, preserves user and command strings literally, shares the existing nested/malformed/code-region reasoning-tag contract, and covers both history reload and live stream formatter paths.

## Reproduction Assessment

Yes. A high-confidence unit reproduction is an assistant message with string content like `<think>hidden</think>Visible`; source inspection shows current main returns no thinking text and renders the raw sanitized string through the TUI formatter paths.

## Solution Assessment

No, not as written. The direction is right, but the maintainable fix is a role-aware assistant-string helper that reuses the shared reasoning-tag contract and leaves generic string rendering alone.
