## Summary

Keep this PR open. Current main still misses the TUI string-content path, so the proposed work is not obsolete; however, the branch should not merge as-is because its tagged-string parsing can rewrite non-assistant transcript text.

## What This Changes

The PR adds shared tagged-reasoning extraction, wires TUI string content through reasoning-tag splitting, and adds formatter regressions for assistant string payloads.

## Best Possible Solution

Revise this PR into an assistant-only TUI formatter and stream-assembler fix: split assistant string content into reasoning and visible text, hide tagged reasoning when thinking is off, show it separately when thinking is on, preserve non-assistant literal tags, and add formatter plus streaming/history regressions.
