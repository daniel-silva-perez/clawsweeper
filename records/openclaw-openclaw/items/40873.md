## Summary

Keep open. Current main still does not implement this PR's TUI string-content reasoning split, but the proposed patch should not merge as-is because it parses all string transcript content as reasoning-tagged assistant output and its new extractor is inconsistent with the existing nested-tag stripper.

## What This Changes

The PR adds shared extraction of inline reasoning tags from string content, wires TUI string formatting through reasoning/content splitting, and adds formatter regression tests for assistant string payloads.

## Best Possible Solution

Revise this PR into an assistant-only TUI formatter and stream-assembler fix: split assistant string content into reasoning and visible text, hide tagged reasoning when thinking is off, show it separately when thinking is on, preserve non-assistant literal tags, handle nested reasoning tags consistently with the shared stripper, and add formatter plus streaming/history regressions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate tagged reasoning parsing to assistant messages:** `src/tui/tui-formatters.ts:307-311`
  - body: `extractTextBlocks` is used by `extractTextFromMessage` for user and command records, so routing every string through tagged reasoning parsing will strip literal `<think>` examples from non-assistant transcript text. Restrict this parsing to assistant content and leave user/command strings on the existing render-sanitization path.
  - confidence: 0.94
- **[P3] Track nested thinking depth when extracting reasoning:** `src/shared/text/reasoning-tags.ts:109-112`
  - body: The new extractor flips a boolean while `stripReasoningTagsFromText` tracks nested depth. For nested `<think>` blocks it stops collecting at the inner close tag and drops the rest of the outer reasoning when thinking display is enabled.
  - confidence: 0.82
