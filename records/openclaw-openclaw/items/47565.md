## Summary

Keep open. Current main still has fixed Control UI CSP and fixed Permissions-Policy behavior: same-origin microphone access is now allowed, but there is still no configurable `gateway.controlUi` surface for `connect-src` additions or Permissions-Policy values, so the custom Control UI plus local Whisper endpoint use case remains unresolved.

## What This Changes

Request to add configurable Control UI Permissions-Policy and CSP header options under `gateway.controlUi` for custom Control UI roots and local inference endpoints.

## Best Possible Solution

Keep this open for maintainer product/security review. The best path is an explicit, validated `gateway.controlUi` header policy surface, likely narrow allowlists for safe `connect-src` additions and Permissions-Policy feature values, with config types, strict schema, help/labels/tags, docs, and focused gateway tests updated together. Avoid a raw arbitrary CSP string unless maintainers intentionally accept that security tradeoff.
