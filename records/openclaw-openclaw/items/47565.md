## Summary

Keep open. Current main partially addresses the microphone example by allowing same-origin microphone access, but the Control UI CSP and Permissions-Policy remain fixed and there is still no supported config surface for adding local `connect-src` origins or policy overrides. This is valid remaining work, but it changes browser security policy and needs maintainer API-shape review.

## What This Changes

The issue asks for configurable Control UI security headers so custom static UI roots can allow selected browser permissions and network origins through `openclaw.json`.

## Best Possible Solution

Add a maintainer-approved, narrow, typed Control UI header policy config in the gateway owner area, with strict validation, generated schema/help/labels/tags, public docs, and focused tests for CSP serialization, Permissions-Policy serialization, config rejection, and HTTP header behavior.
