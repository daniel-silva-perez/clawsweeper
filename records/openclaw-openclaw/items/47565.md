## Summary

Keep open. Current main still has fixed Control UI CSP and fixed default Permissions-Policy behavior, and the config/schema/docs surface does not expose a supported way to add custom local connect-src origins or policy overrides. The same-origin microphone change partly addresses one example, and #71886 is a narrower open CSP allowlist PR, but neither resolves the requested configurable header policy.

## What This Changes

The issue requests openclaw.json options under gateway.controlUi to customize Control UI Permissions-Policy and CSP directives for custom Control UI roots.

## Best Possible Solution

Add a maintainer-approved, narrow, typed Control UI header-policy config in the gateway owner area, with strict validation, generated schema/help/labels/tags, public docs, and focused tests for CSP serialization, Permissions-Policy serialization, config rejection, and HTTP header behavior.
