## Summary

Keep open. Current main still has fixed Control UI CSP and fixed default Permissions-Policy behavior, and neither the config schema nor public docs expose a supported way to add custom local connect-src origins or policy directives. The merged same-origin microphone change only covers part of the example, while the related open CSP PR is a narrow fixed-origin allowlist change rather than the requested configurable header policy.

## What This Changes

The issue requests typed `gateway.controlUi` configuration for Control UI Permissions-Policy directives and CSP overrides, especially local `connect-src` origins for custom Control UI roots.

## Best Possible Solution

Add a maintainer-approved, narrow, typed Control UI header-policy config in the gateway owner area, with strict directive/origin validation, generated schema/help/labels/tags, public docs, and focused tests for CSP serialization, Permissions-Policy serialization, config rejection, and HTTP header behavior.
