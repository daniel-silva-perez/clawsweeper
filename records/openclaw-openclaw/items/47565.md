## Summary

Keep open. Current main partially covers the microphone example by allowing same-origin microphone access, but the requested configurable Control UI CSP and Permissions-Policy surface is still absent; adding it is security-sensitive and needs maintainer API-shape review before implementation.

## What This Changes

Request configurable `gateway.controlUi` CSP and Permissions-Policy options so custom Control UI roots can allow required browser permissions and local network origins without patching installed dist files.

## Best Possible Solution

Add a maintainer-approved, typed Control UI header-policy config in the gateway owner area, with allowlisted CSP directives, strict origin validation, generated config metadata/help/docs, and focused tests for serialization, config rejection, and HTTP header behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: the proposed `gateway.controlUi` keys are absent from the type and strict schema, while current header helpers still emit fixed CSP and Permissions-Policy values.

## Solution Assessment

Unclear as proposed. The capability is reasonable for custom Control UI roots, but raw `cspOverrides` or arbitrary policy strings would not be the safest implementation; a constrained typed schema with validation is the narrower maintainable path.
