## Summary

Keep open. Current main still has fixed Control UI CSP and Permissions-Policy behavior, with no supported config field for adding local `connect-src` origins or policy overrides. The same-origin microphone change partially addresses the example, and the related open PR #71886 is narrower, but the requested configurable header policy remains valid and security-sensitive enough for maintainer API review.

## What This Changes

The issue requests configurable Control UI CSP and Permissions-Policy header settings for custom Control UI roots.

## Best Possible Solution

Add a narrow, typed Control UI header-policy config in the gateway owner area, with strict validation, generated schema/help/labels/tags, public docs, and focused tests for CSP serialization, Permissions-Policy serialization, config rejection, and HTTP header behavior. Prefer explicit allowlisted directives and origins unless maintainers approve a broader escape hatch.
