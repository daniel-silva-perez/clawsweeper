## Summary

Keep open. Current main still has fixed Control UI CSP and fixed default Permissions-Policy behavior, while the config schema/docs do not expose a supported way to add custom local `connect-src` origins or policy directives. The shipped same-origin microphone change covers part of the example, but the configurable header-policy request remains and needs maintainer security/API review.

## What This Changes

Request configurable `gateway.controlUi` support for Control UI Permissions-Policy and CSP overrides so custom Control UI roots can enable browser permissions and additional local connection origins.

## Best Possible Solution

Add a maintainer-approved, narrow, typed Control UI header-policy config in the gateway owner area, with strict directive/origin validation, generated config metadata, public docs, and focused tests for CSP serialization, Permissions-Policy serialization, config rejection, and HTTP header behavior.
