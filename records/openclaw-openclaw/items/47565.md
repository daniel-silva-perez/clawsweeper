## Summary

Keep open: current main partially addresses the microphone use case by allowing same-origin microphone access, but it still has fixed Control UI CSP and Permissions-Policy headers with no supported config surface for local `connect-src` additions or policy overrides. The remaining work is valid and security-sensitive.

## What This Changes

Request adds `gateway.controlUi` configuration for Permissions-Policy directives and CSP overrides so custom Control UI roots can use browser permissions and local inference servers without patching distributed files.

## Best Possible Solution

Add a narrow, typed Control UI header policy config in the gateway owner area, with strict validation, generated schema/help/labels/tags, public docs, and focused tests. Prefer explicit allowlisted CSP directives and Permissions-Policy feature values unless maintainers intentionally approve a broader raw-header override.
