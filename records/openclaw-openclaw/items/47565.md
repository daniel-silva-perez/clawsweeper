## Summary

Keep open. Current main still has fixed Control UI CSP and Permissions-Policy behavior, and the strict gateway config contract has no supported field for adding local connect-src origins or policy overrides. The same-origin microphone change covers part of the example, but the remaining configurable-header request is valid and security-sensitive enough for maintainer API review.

## What This Changes

The issue requests configurable Control UI CSP and Permissions-Policy headers under gateway.controlUi for custom Control UI roots that need browser permissions and local fetch origins.

## Best Possible Solution

Add a narrow, typed Control UI header-policy config in the gateway owner area, with strict validation, schema/help/label/tag metadata, public docs, and focused tests for CSP serialization, Permissions-Policy serialization, config rejection, and HTTP header behavior. Prefer explicit allowlisted directives and origins unless maintainers approve a broader escape hatch.
