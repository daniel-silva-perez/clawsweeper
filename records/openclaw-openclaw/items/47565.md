## Summary

Keep open. Current main still has hardcoded Control UI CSP and fixed Permissions-Policy behavior: same-origin microphone access is now allowed, but there is still no configurable `gateway.controlUi` surface for Permissions-Policy or `connect-src` additions, so the custom Control UI plus local Whisper endpoint use case remains unsolved.

## Best Possible Solution

Keep this open for maintainer product/security review. The best path is an explicit, validated `gateway.controlUi` header policy surface, probably narrow allowlists for safe `connect-src` additions and Permissions-Policy feature values, with config type/schema/help/labels/tags/docs/tests updated together. Avoid a raw arbitrary CSP string unless maintainers intentionally accept that security tradeoff; review #71886 separately as a narrow tweakcn allowlist fix.
