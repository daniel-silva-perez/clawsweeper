## Summary

Keep open. Current main partially covers the microphone example by allowing same-origin microphone access, but the Control UI CSP and Permissions-Policy are still fixed and there is no supported config/schema/docs path for adding custom local `connect-src` origins or policy directives. The request is valid remaining work, but it changes browser security policy and needs maintainer API and validation review before implementation.

## What This Changes

Request to add `gateway.controlUi` configuration for Control UI Permissions-Policy and CSP overrides so custom Control UI roots can use browser permissions and local fetch targets without patching built files.

## Best Possible Solution

Add a maintainer-approved, narrow, typed Control UI header-policy config in the gateway owner area, with strict directive/origin validation, generated config metadata, public docs, and focused tests for CSP serialization, Permissions-Policy serialization, config rejection, and HTTP header behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: proposed `gateway.controlUi` keys are absent from the type and strict schema, while the current CSP and Permissions-Policy helpers still emit fixed header values.

## Solution Assessment

Unclear as proposed. The requested capability is reasonable for custom Control UI roots, but raw `cspOverrides` or arbitrary policy strings would not be the safest implementation; a constrained typed schema with allowlisted directives and origin validation is the safer path.
