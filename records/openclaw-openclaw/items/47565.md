## Summary

Keep open. Current main has already fixed the narrower same-origin microphone case, but this issue asks for configurable Control UI Permissions-Policy and CSP behavior, especially adding local HTTP inference endpoints to CSP connect-src. That configuration surface still does not exist.

## Best Possible Solution

Keep the issue open for a focused core gateway/config implementation. Add a narrow, validated `gateway.controlUi` header policy surface, wire it through GatewayControlUiConfig, zod and generated schemas, docs/help, and the Control UI header builders, with tests proving defaults remain hardened while explicit operators can add same-origin/browser permissions and trusted local `connect-src` endpoints for custom Control UI roots.

## Risks / Open Questions

- A broad arbitrary-header escape hatch would be security-sensitive because the Control UI is an admin surface; a fix should validate narrow CSP/Permissions-Policy additions and preserve locked-down defaults.
- Closing this as implemented would lose the remaining local-inference `connect-src` use case even though the related same-origin microphone case is already solved.
