## Summary

Keep open. Current main partially addresses the microphone part by allowing same-origin microphone access, but it still has fixed Control UI CSP and Permissions-Policy headers with no accepted `gateway.controlUi` config surface for local `connect-src` additions or policy overrides.

## What This Changes

The issue requests `gateway.controlUi` configuration for Control UI Permissions-Policy values and CSP directive/connect-src overrides for custom Control UI roots.

## Best Possible Solution

Add a deliberately scoped, validated Control UI header policy config in the gateway owner area, with typed config, strict schema/help/labels/tags, user-facing docs, and focused gateway tests. Prefer explicit allowlisted directives and Permissions-Policy feature values unless maintainers intentionally approve a broader raw-header override.
