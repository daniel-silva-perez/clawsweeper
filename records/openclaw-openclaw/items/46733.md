## Summary

Current main still does not resolve the reported Anthropic OAuth/Claude Max 1M path: the Anthropic stream wrapper removes the `context-1m-2025-08-07` beta for `sk-ant-oat` credentials, tests still assert that fallback, and docs still tell users OAuth/subscription tokens fall back to the standard context window. The open #66766/#66862 path covers a related context-display problem, but it does not fully cover the request-header/auth contract reported here.

## What This Changes

The issue requests Anthropic Opus 4.6 1M context support for Claude Max/OAuth credentials, including preserving the long-context beta/header behavior and reporting a 1M context window when configured.

## Best Possible Solution

Treat this as an Anthropic provider/auth contract update: align request beta/header injection, Opus/Sonnet 4.6 context budgeting, `models list`/status display, OAuth-required betas, docs, and tests around the current Claude Max 1M behavior, with an explicit maintainer decision for setup-token/OAuth eligibility.
