## Summary

Keep this issue open. Current main still intentionally strips the Anthropic `context-1m-2025-08-07` beta when the credential looks like an OAuth/subscription token, and tests/docs lock in that fallback. The configured `context1m` context budget path appears partially present, but the reported OAuth long-context behavior is not implemented on main.

## Best Possible Solution

Keep the issue open and fix it in the Anthropic plugin if current Anthropic OAuth credentials really support 1M context: stop removing `context-1m-2025-08-07` for eligible OAuth tokens, preserve the required OAuth betas, update the stream-wrapper and embedded-runner tests, and revise the Anthropic/token-use docs. If live verification shows only some OAuth plans are eligible, gate the behavior or document/configure that distinction explicitly.
