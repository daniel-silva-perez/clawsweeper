## Summary

Keep open. Current main still encodes the Anthropic OAuth/setup-token fallback: `context1m` adds the 1M beta for eligible models, but the OAuth path removes that beta before the request, tests assert the fallback, and docs still tell users OAuth/subscription tokens are rejected for 1M context. The related open work overlaps the context-window/display side but does not fully resolve this OAuth request-header contract.

## What This Changes

The issue requests that Anthropic Claude Opus 4.6 with `context1m: true` and Claude Max/OAuth credentials use and display the 1M context path instead of falling back around 200k.

## Best Possible Solution

Confirm the current Anthropic OAuth/setup-token long-context contract, then update the Anthropic provider wrapper, context budgeting, model/status display, docs, and tests so eligible OAuth users get a real 1M request path and ineligible credentials get an accurate, explicit fallback.
