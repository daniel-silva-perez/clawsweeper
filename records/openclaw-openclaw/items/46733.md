## Summary

Keep open. Current main still strips the Anthropic 1M beta from OAuth/setup-token requests, still documents OAuth fallback for long context, and still has 4.6 context behavior gated in ways that do not fully satisfy the reported Claude Max OAuth path. The related open context-display and OAuth reports overlap, but they do not fully supersede this request-header/auth contract issue.

## What This Changes

Allow Claude Opus/Sonnet 4.6 1M context for Anthropic OAuth/Claude Max users by preserving the required long-context request behavior and reporting the correct context window when configured.

## Best Possible Solution

Update the Anthropic provider/auth contract so OAuth/setup-token long-context eligibility is represented consistently across beta/header merging, context budgeting, `models list` and status display, docs, and tests, with related work coordinated against #41444, #66766, and #66862.
