## Summary

Keep open. Current main still removes the Anthropic 1M beta from OAuth/setup-token requests, still has tests and docs that enforce the OAuth fallback, and the related open context-display work does not fully cover this request-header/auth contract problem.

## What This Changes

The issue asks OpenClaw to support Claude Opus 4.6 1M context for Anthropic OAuth/Claude Max users, including preserving the needed request headers and reporting the correct context window.

## Best Possible Solution

Confirm the current Anthropic OAuth/setup-token long-context contract, then update the Anthropic provider wrapper, context budgeting, models/status display, docs, and tests so configured eligible OAuth users either get a real 1M request path or a clearly correct fallback.
