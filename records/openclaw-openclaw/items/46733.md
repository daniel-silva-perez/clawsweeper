## Summary

Keep open. Current main still intentionally removes the Anthropic 1M beta from OAuth/setup-token requests, still documents that fallback, and still has tests encoding the behavior; related open context-window work does not resolve the OAuth request-header contract reported here.

## What This Changes

The issue asks OpenClaw to support Anthropic Claude Opus 4.6 1M context for OAuth/Claude Max users instead of falling back to a 200k-scale context window and omitting the long-context beta header.

## Best Possible Solution

Confirm the current Anthropic OAuth/setup-token long-context contract, then update the Anthropic provider wrapper, context budgeting, model/status display, docs, and tests so eligible OAuth users get a real 1M request path and ineligible credentials get an accurate explicit fallback.
