## Summary

Keep open. Current main and the latest release still encode the reported fallback: Anthropic OAuth/setup-token credentials get the 1M beta removed, Claude 4.6 only resolves to 1M when `context1m` is explicitly set, and the models-list row builder still reports raw row metadata rather than applying configured `context1m` params. Official Anthropic release notes now say Opus 4.6 and Sonnet 4.6 support requests over 200k automatically with no beta header, so the old OpenClaw docs/source contract needs maintainer/provider-contract review rather than cleanup closure. citeturn1view0

## What This Changes

The issue requests that OpenClaw support and display Claude Opus 4.6 1M context for OAuth/Claude Max users instead of stripping the long-context path or showing a 200k-class limit.

## Best Possible Solution

Update the Anthropic provider-owned contract so Claude Opus/Sonnet 4.6 use the current no-beta 1M behavior where eligible, keep explicit fallback/error handling for ineligible credentials and older beta-only models, and align the context resolver, models-list/status display, docs, and tests in one coordinated change.

## Reproduction Assessment

Yes for a source-level reproduction: current main has tests and code paths that strip the 1M beta for OAuth credentials and keep no-flag Opus 4.6 at 200k. Live Anthropic reproduction was not available because no Anthropic API/setup-token credentials were present in the environment.

## Solution Assessment

Unclear. The current hardcoded OAuth fallback no longer looks like the best long-term solution given Anthropic's 4.6 release notes, but a safe fix should be provider-contract-driven and coordinated with the open context-window PR rather than a narrow display-only patch.
