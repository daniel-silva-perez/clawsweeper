## Summary

Keep open. Current main and the latest release still encode the reported old Anthropic 1M-context contract: OAuth/setup-token traffic strips the `context-1m` beta, Claude 4.6 only resolves to 1M when `context1m` is explicitly set, and the docs still describe the 1M path as beta-gated even though Anthropic now says Opus 4.6 and Sonnet 4.6 support requests over 200K without a beta header. ([claude.com](https://claude.com/blog/1m-context-ga))

## What This Changes

The issue asks OpenClaw to support and display Claude Opus 4.6 1M context for Anthropic OAuth/Claude Max users now that the provider no longer requires the long-context beta header.

## Best Possible Solution

Update the Anthropic provider-owned long-context contract so eligible Opus/Sonnet 4.6 requests use the current no-beta 1M behavior, retain correct handling for older beta-only models or ineligible credentials, and align runtime headers, context budgeting, models/status display, docs, and tests in one coordinated change.

## Reproduction Assessment

Yes for a source-level reproduction: current main has code and tests that strip 1M beta headers for OAuth credentials and keep no-flag Opus 4.6 at 200k. Live Anthropic reproduction was not run because no Anthropic or Claude credential names were present in the environment/profile check.

## Solution Assessment

Unclear. The current hardcoded fallback no longer matches Anthropic's published 4.6 behavior, but the best fix is a provider-contract update coordinated with #66862 and #41444 rather than a narrow display-only patch.
