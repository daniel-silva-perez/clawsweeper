## Summary

Keep open. Current main and the latest stable tag still encode the older Anthropic long-context behavior: OAuth/setup-token requests strip the 1M beta header, Opus/Sonnet 4.6 context resolution remains `context1m`-flag gated except for separate Opus 4.7 handling, and docs still describe 1M as beta-gated/OAuth-rejected despite Anthropic's March 13, 2026 GA announcement.

## What This Changes

The issue requests that OpenClaw support Anthropic Claude Opus 4.6's 1M context window with OAuth/Claude Max auth instead of stripping the long-context path or reporting a 200k-style context limit.

## Best Possible Solution

Update the Anthropic plugin-owned long-context contract for 4.6 GA in one coordinated change: runtime context budgeting and models/status display should expose 1M for eligible Opus/Sonnet 4.6, request headers should match the current no-beta-required GA behavior while preserving correct handling for older beta-only models or ineligible credentials, and the Anthropic/token-use docs plus tests should be aligned.

## Reproduction Assessment

Yes for a source-level reproduction: current main has code and tests that strip the 1M beta for OAuth credentials and keep no-flag Opus 4.6 at 200k. A live provider reproduction was not run because no Anthropic/Claude live credential variable names were present.

## Solution Assessment

Unclear. The requested direction matches Anthropic's published 4.6 GA contract, but the best repository fix needs maintainer/provider-contract review so context resolution, OAuth/header behavior, display, docs, and overlapping #66862/#41444 work do not drift.
