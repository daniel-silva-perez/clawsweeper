## Summary

Keep open. Current main still encodes the reported fallback: OAuth/setup-token Anthropic credentials get the 1M context beta removed, the no-flag Claude 4.6 context path remains capped at 200k, and the models-list row path still does not apply configured `context1m` params. Related open work covers only part of the context-window behavior, not the OAuth request-header/provider-contract path.

## What This Changes

The issue asks OpenClaw to support and accurately display Claude Opus 4.6 1M context for OAuth/Claude Max credentials instead of falling back to the standard context window.

## Best Possible Solution

Confirm the current Anthropic OAuth/setup-token long-context contract, then update the Anthropic provider wrapper, context budgeting, model-list/status display, docs, and tests so eligible Claude 4.6 OAuth users get a real 1M request path and ineligible credentials get an explicit accurate fallback.
