## Summary

Keep open. Current main still encodes the older Anthropic 1M-context behavior for this report: OAuth/setup-token requests remove the 1M beta, 4.6 context resolution is still `context1m`-gated, model list rows do not resolve agent model params, and docs still describe 1M as beta/OAuth-rejected while Anthropic's March 13, 2026 announcement says Opus 4.6 and Sonnet 4.6 are GA at 1M with no beta header required. ([claude.com](https://claude.com/blog/1m-context-ga))

## What This Changes

The issue asks OpenClaw to support Anthropic Claude Opus 4.6 1M context for OAuth/Claude Max users instead of stripping the long-context header or displaying/enforcing the older 200k limit.

## Best Possible Solution

Update the Anthropic plugin-owned long-context contract in one coordinated change: treat eligible Opus/Sonnet 4.6 as GA 1M where the provider contract now allows it, align OAuth/setup-token header behavior with the no-beta-required contract while preserving correct older/ineligible fallbacks, and keep runtime budgeting, models/status display, docs, and tests consistent.

## Reproduction Assessment

Yes for a source-level reproduction: current main has code and tests that strip the 1M beta for OAuth credentials and keep Opus 4.6 at 200k unless `context1m` is set. I did not run a live Anthropic request in this read-only review.

## Solution Assessment

Unclear. The requested direction matches Anthropic's published 4.6 GA contract, but the best repository fix needs maintainer/provider-contract review so request headers, OAuth credentials, runtime context, display, docs, and overlapping #41444 work do not drift.
