## Summary

Keep open. Current main still intentionally removes the Anthropic 1M beta for OAuth/setup-token credentials, tests and docs still encode that fallback, and the related open context-window PR only covers part of the display/budgeting behavior rather than the OAuth request-header contract reported here.

## What This Changes

The issue asks OpenClaw to support Claude Opus 4.6 1M context for OAuth/Claude Max users by allowing configured `context1m` to affect both Anthropic request headers and model/context display.

## Best Possible Solution

Confirm the current Anthropic OAuth/setup-token long-context contract, then update the Anthropic provider wrapper, context budgeting, model-list/status display, docs, and tests so eligible OAuth users get a real 1M request path and ineligible credentials get an accurate explicit fallback.
