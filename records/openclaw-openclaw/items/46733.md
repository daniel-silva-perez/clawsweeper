## Summary

Keep open. Current main still strips the Anthropic 1M beta header from `sk-ant-oat` OAuth/setup-token requests, tests and docs still encode that fallback, and the related open context-window work does not fully address the OAuth request-header contract reported here.

## What This Changes

The issue requests that Claude Opus 4.6 1M context work for Anthropic OAuth/Claude Max users, including correct 1M context reporting and long-context request handling when `context1m` is configured.

## Best Possible Solution

Confirm the current Anthropic OAuth/setup-token long-context contract, then update the Anthropic provider wrapper, context budgeting, model/status display, docs, and tests so eligible OAuth users get a real 1M request path and ineligible credentials get an accurate explicit fallback.
