## Summary

Keep this PR open. Current main still does not implement immediate live-tail rendering of inbound user messages in TUI --session mode, and the PR remains a plausible implementation candidate for the linked #45388 workflow. The branch still needs maintainer review/revision because its current broadcast hook is on chat.send while the primary Telegram/external-channel path runs through shared inbound dispatch, a gap that was also raised in PR review and acknowledged by the author.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The strongest implementation path is to make TUI --session subscribe to sessions.messages.subscribe for the active session and render session.message user turns with session filtering and local echo/dedupe protection, or to emit a versioned user-turn event from the shared external-channel ingress/transcript pipeline used by Telegram and other channels. Update the gateway protocol/schema/docs and TUI regression coverage, and split the unrelated model-selection test change.
