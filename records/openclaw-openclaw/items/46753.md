## Summary

Keep open. The memory-growth problem is still present on current main, but this PR should not merge as-is because the branch bundles the useful cache caps with unrelated web_search provider/credential rewrites and tool-schema changes that introduce blocking regressions.

## What This Changes

The PR adds oldest-entry caps to several agent runtime Map caches, broadens many agent tool schema constraints/descriptions, and rewrites web_search into inline provider and credential handling.

## Best Possible Solution

Land a narrow cache-only replacement on current main: cap the intended Maps with deterministic oldest-entry eviction only when inserting new keys, preserve the existing web_search provider and SecretRef contracts, leave unrelated tool schemas unchanged, and add focused regression tests for cap and refresh semantics.

## Reproduction Assessment

Yes. Current main has module-level Maps in the reported paths with distinct-key write sites and no maximum size; repeated distinct session keys, workspace paths, trace file paths, or model IDs reproduce unbounded growth at the source level.

## Solution Assessment

No. The cache-cap direction is appropriate, but this PR is not the best way to solve it because it mixes the fix with unrelated provider, credential, and schema behavior changes; a cache-only replacement is the safer maintainable path.

Full review comments:

- **[P1] Route web_search through the provider registry:** `src/agents/tools/web-search.ts:25`
  - body: The PR replaces the registry-backed resolveWebSearchDefinition()/runWebSearch() path with hard-coded inline provider handling. Runtime or bundled providers outside that list can stop working or fall back incorrectly, which breaks the current plugin-provider contract.
  - confidence: 0.93
- **[P1] Use plugin-owned SecretRefs for web_search keys:** `src/agents/tools/web-search.ts:553-563`
  - body: The new credential resolvers read legacy tool config or env vars instead of plugin-owned plugins.entries.<plugin>.config.webSearch.* paths and resolved SecretRefs. Existing users with valid plugin config can see missing-key failures.
  - confidence: 0.9
- **[P2] Evict workspace cache only for new keys:** `src/agents/workspace.ts:92`
  - body: The eviction call runs before every workspaceFileCache.set, including refreshes for a path already in the map. At capacity, refreshing an existing file evicts an unrelated oldest entry and reduces effective capacity.
  - confidence: 0.88
- **[P2] Preserve sub-second node clip durations:** `src/agents/tools/nodes-tool.ts:165-170`
  - body: Adding minimum: 1000 to durationMs rejects 0-999 ms values that current main accepts up to the existing 300000 ms maximum. That is an unrelated behavior change for camera clips.
  - confidence: 0.82
- **[P3] Apply the model cache cap consistently:** `src/agents/context.ts:52-58`
  - body: Only the discovered-model path enforces the new model-cache cap, while configured context windows write to the same cache without a guard. A large configured model list can exceed the cap and later have configured entries evicted.
  - confidence: 0.76
