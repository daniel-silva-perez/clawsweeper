## Summary

Current main still does not implement the central request. `chat.send` and `sessions.send` remain closed protocol schemas without reply-target fields, the Gateway builds agent `MsgContext` without mapping any parent/reply target into the existing `ReplyTo*` fields, and the Control UI send state/request has no outbound selected reply target. The no-output side has current `NO_REPLY` suppression, but that does not resolve the missing structured reply metadata path.

## Best Possible Solution

Keep this open and implement a focused additive change: add validated reply metadata to `chat.send` and likely `sessions.send`, capture a selected parent/reply target in the Control UI composer, map the metadata in Gateway `chat.send` into the existing `MsgContext.ReplyTo*` fields with bounded sanitized quoted content, update protocol/docs/tests, and preserve current silent-output filtering.
