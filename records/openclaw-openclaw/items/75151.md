## Summary

Keep open. Current main still has the reported data-integrity gap: the embedded/direct reset path can persist a new `sessionFile` before the successor transcript exists, the replay helper can return without creating that file, and history lookup returns an empty list when no resolved transcript candidate exists. The existing doctor and cleanup paths detect or prune missing transcript entries later, but they do not prevent the bad active mapping or recover the prior valid transcript.

## What This Changes

The issue asks OpenClaw to prevent context-overflow session reset from updating a direct-session mapping to a nonexistent transcript file and orphaning the real prior history.

## Best Possible Solution

Enforce a generic reset invariant in the embedded/direct session path: create or prewarm the successor JSONL transcript before `sessions.json` is updated, preserve the previous valid mapping if creation fails, refresh queued follow-up state only after the successor is durable, and add regression coverage for missing, empty, and non-replayable prior transcripts.

## Reproduction Assessment

Yes. A focused unit reproduction can call `resetReplyRunSession` with a previous entry whose `sessionFile` is missing or has no replayable user-starting tail, then assert the persisted successor `sessionFile` exists and `readSessionMessages` cannot silently return an empty history because the mapped file is absent.

## Solution Assessment

Yes, the issue’s requested durability invariant is the narrowest maintainable fix. Cleanup and doctor diagnostics remain useful, but relying on later pruning after a broken active mapping is not sufficient.
