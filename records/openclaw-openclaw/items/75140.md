## Summary

Keep this PR open. The task-event seam and memory-core recorder are useful, but the current patch is not merge-ready because the memory-core registration installs a process-wide task-registry listener and discards its unsubscribe, so plugin reload or disable/re-enable can leave stale listeners that duplicate outcome writes.

## What This Changes

This PR adds a plugin SDK task-registry event subpath, fans task registry observer events to additional listeners, and wires memory-core to append terminal task outcome JSONL records for dreaming feedback.

## Best Possible Solution

Keep the additive SDK seam only if the recorder is lifecycle-managed: register the unsubscribe with plugin runtime cleanup, cover duplicate-listener behavior in a focused test, then verify the memory-core recorder test, SDK export/baseline checks, and the changed gate.

## Reproduction Assessment

Yes. A focused reproduction would load memory-core, rebuild or replace the plugin registry without process restart, then terminalize one task; because the old listener is not unsubscribed, multiple listeners can append duplicate outcome rows.

## Solution Assessment

No for merge as-is. The narrow maintainable fix is to have `registerTaskOutcomeRecorder` or the memory-core entry register `api.registerRuntimeLifecycle({ cleanup: unsubscribe })`, with a regression test for plugin cleanup or double registration.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Unsubscribe the recorder on plugin cleanup:** `extensions/memory-core/index.ts:34`
  - body: `registerTaskOutcomeRecorder` installs a process-wide task-registry listener, but the plugin registration discards the returned unsubscribe instead of registering it with the plugin runtime lifecycle. When the plugin registry is rebuilt or memory-core is disabled and enabled again, the old listener remains active, so each terminal task can be appended multiple times and the stale listener keeps using the old config/logger. Register a lifecycle cleanup that calls the unsubscribe.
  - confidence: 0.91
