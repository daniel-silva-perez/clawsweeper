## Summary

Keep this issue open as the canonical tracker. ## What This Changes

The issue requests a stable configurable compatibility layer for generic `api: "openai-responses"` proxy endpoints, including non-empty `instructions`, array input, stream behavior, and optional omission of unsupported fields such as `max_output_tokens`.

## Best Possible Solution

Keep this issue open as the canonical tracker and leave #10140 closed as the duplicate.
