## Summary

Keep open. Current main has adjacent lock-based restart recovery and diagnostic lane release, but it does not implement the requested startup scan/replay of chat sessions by transcript tail: ordinary running sessions without `abortedLastRun` are skipped, assistant empty-tail cases are not recoverable, and the restart resume path calls `agent` with `deliver:false` rather than replaying through `sessions.send`.

## What This Changes

The issue requests gateway startup recovery that detects unanswered or invisible-tail chat sessions after restart and replays them through the existing `sessions.send` pipeline.

## Best Possible Solution

Add a bounded gateway startup recovery pass that runs after request context and channel sidecars are ready, scans persisted chat session stores newest-first, selects only safe stuck transcript-tail candidates, and replays them through the same `sessions.send`/chat delivery path users rely on while leaving diagnostic lane release and lock-based main-session recovery as separate safeguards.
