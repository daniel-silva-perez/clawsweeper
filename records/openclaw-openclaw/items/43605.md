## Summary

Keep this issue open. Current main still lacks native cross-gateway `sessions_send` / `sessions_spawn` routing, and the provided GitHub context shows open same-author PR #43656 explicitly tracks this work with closing syntax. The related `xihe-jianmu-ipc` MCP bridge is useful workaround/context, but it does not supersede native OpenClaw session history, visibility, and gateway-auth semantics.

## What This Changes

Request native cross-gateway session messaging so agents on different OpenClaw Gateway instances can use `sessions_send` and related session tooling without SSH workarounds.

## Best Possible Solution

Keep this as the canonical native cross-gateway session messaging tracker. Maintainers should review, rework, land, or close #43656; a final solution should be additive, documented, and cover config schema, session tool schemas, gateway auth/token resolution, remote transcript/history ownership, visibility policy, tests, and user-facing docs.
