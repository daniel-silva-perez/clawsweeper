## Summary

Keep this issue open. Current main still exposes `sessions_send` and `sessions_spawn` as local session tools, the gateway config has only a single `gateway.remote` target rather than a peer registry, and open PR #43656 already uses closing syntax for this issue, so the next step is maintainer review of that implementation candidate rather than cleanup closure.

## What This Changes

Add native cross-gateway session messaging/delegation so `sessions_send` and `sessions_spawn` can target agents on other OpenClaw gateway instances instead of only local sessions.

## Best Possible Solution

Keep this as the canonical native cross-gateway session messaging tracker until maintainers land, rework, or reject the paired PR with an additive design that covers config schema/help, tool schemas, gateway auth, visibility rules, transcript ownership, tests, and docs.
