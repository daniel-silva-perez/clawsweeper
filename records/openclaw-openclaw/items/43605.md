## Summary

Keep this issue open. Current `main` still implements `sessions_send` and `sessions_spawn` as local-gateway session tools, the config/docs model multiple gateways as isolated instances plus one remote client target rather than peer routing, and the provided discussion shows open same-author PR #43656 uses closing syntax for the native implementation path.

## What This Changes

The issue requests native cross-gateway session messaging and delegation so agents on separate OpenClaw Gateway instances can communicate without SSH scripts.

## Best Possible Solution

Keep this as the canonical native cross-gateway session messaging tracker. Maintainers should review, rework, land, or deliberately replace #43656 with an additive design covering config schema/help, session-tool schemas, gateway auth, remote transcript/history ownership, visibility policy, tests, and public docs.
