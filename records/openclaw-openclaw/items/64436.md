## Summary

Keep open. Current main still does not expose `api.runtime.usage` or an equivalent public plugin SDK path for model pricing/cost estimation, while the internal pricing helpers already exist and are used by core surfaces. The PR remains an active, narrow public runtime API proposal, not cleanup-close material.

## Best Possible Solution

Keep this PR open for maintainer API review. If the API shape is accepted, rebase onto current main, wire the additive `api.runtime.usage` facade, add SDK runtime docs and a focused runtime-shape/assembly test, keep the regenerated plugin SDK API baseline, and run the relevant API/changed-gate proof before merge.
