## Summary

Current main still leaves the TUI final and fallback streamed assistant output path unsanitized for GLM-5/DeepSeek model-special tokens. The shared sanitizer exists and is tested, but it is not wired into the TUI finalization path; the related fix PR was closed unmerged.

## Best Possible Solution

Keep this open and land a narrow TUI fix that applies the shared model-special-token stripping behavior, preferably via the shared assistant-visible sanitizer if it preserves TUI semantics, to final and fallback streamed assistant text before `chatLog.finalizeAssistant`. Add focused regression tests for ASCII tokens, full-width tokens, streamed fallback finalization, and inline/fenced code preservation, plus a user-facing changelog fix entry.
