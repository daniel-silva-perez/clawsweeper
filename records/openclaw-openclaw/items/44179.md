## Summary

Current main still leaves the TUI final and fallback streamed assistant output path unsanitized for GLM-5/DeepSeek model-special tokens. The shared sanitizer exists and is tested, but it is not wired into the TUI finalization path; the related fix PR #46182 was closed unmerged.

## Best Possible Solution

Keep this open and land a narrow TUI fix that applies the shared model-special-token stripping behavior, preferably via `sanitizeAssistantVisibleText` or the narrow shared helper if that better preserves TUI semantics, to final and fallback streamed assistant text before visible chat finalization. Add focused regression tests for ASCII tokens, full-width tokens, streamed fallback finalization, and inline/fenced code preservation, plus a user-facing changelog fix entry.
