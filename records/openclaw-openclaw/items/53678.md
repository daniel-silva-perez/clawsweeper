## Summary

Keep this issue open. Current main still lacks the requested cross-channel Gateway receipt timestamp: the shared inbound context, agent-visible conversation metadata, SDK direct-DM ingress helper, internal/plugin hook payloads, and Telegram buffered ingress path still expose only source-channel timestamps or Telegram-local debug/buffer timing.

## What This Changes

Add a new inbound metadata timestamp recording when the Gateway receives a message so agents can separate channel delivery latency from processing latency.

## Best Possible Solution

Add a backward-compatible shared receipt timestamp stamped at inbound Gateway ingress, preserve source-channel `Timestamp`, expose the new value to agent-visible metadata and hook/SDK payloads, and cover buffered or long-poll-style paths so the field reflects Gateway receipt rather than source-channel time.
