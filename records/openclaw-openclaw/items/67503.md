## Summary

Keep this issue open. Current main still duplicates a 160-character inbound-body preview into model-visible system-event text for Slack and Microsoft Teams while separately delivering the full message body; Mattermost has been fixed, but the central Slack/MS Teams behavior remains and PR #67761 is still open with closing intent.

## What This Changes

The issue asks to remove duplicated 160-character inbound message previews from Slack, Microsoft Teams, and Mattermost system-event headers while keeping the full inbound body in the normal agent envelope.

## Best Possible Solution

Review, rebase, land, close, or replace PR #67761. The intended end state is label-only notification system events for Slack and Microsoft Teams, full inbound content only in the normal agent body fields, the current Mattermost behavior preserved, and focused regression tests for long inbound messages.

## Reproduction Assessment

Yes. The current-main source gives a high-confidence reproduction path: send or simulate a Slack or Microsoft Teams inbound body longer than 160 characters and the handler will enqueue a truncated preview while separately placing the full text in `BodyForAgent`/`RawBody`.

## Solution Assessment

Yes. Removing message-body previews from notification-only system events is the narrowest maintainable fix; the safer completion path is to apply it only to the remaining Slack/MS Teams handlers and cover the behavior with channel tests.
