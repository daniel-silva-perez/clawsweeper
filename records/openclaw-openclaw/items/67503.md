## Summary

Keep this issue open. Current main still enqueues 160-character Slack and MS Teams inbound-body previews into model-visible trusted system events while separately delivering the full body through the normal inbound context. Mattermost has already been fixed on main, but the central Slack/MS Teams behavior remains and open PR #67761 still targets this issue with closing intent.

## What This Changes

Remove message-body previews from inbound Slack, MS Teams, and Mattermost system-event notifications so full inbound content is delivered only through the normal message envelope.

## Best Possible Solution

Review and update the existing fix path so Slack and MS Teams enqueue only notification labels, keep full inbound content in `BodyForAgent`/`RawBody`, preserve the current Mattermost behavior, and add focused channel regression tests before closing after merge.
