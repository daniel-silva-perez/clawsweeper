## Summary

Keep this PR open. Current main does not implement the PR’s central claude-sdk runtime dispatch, config schema, native-tool bridge, or plugin SDK `runAgent` surface. Main has adjacent ACP/Claude CLI support, including a transitive Claude Agent SDK dependency through the ACP adapter, but that is not the same product path as this PR.

## Best Possible Solution

Keep the PR open for maintainer review or ask the contributor to split/rebase it into smaller pieces: schema/API surface, runtime dispatch, Claude SDK adapter, native-tool bridge, generated sub-agent tooling, and default-flip policy should be reviewed separately. Before landing, it should address the existing review comments, prove live claude-sdk turns and channel/tool parity, and include a clear security review of credential and tool-policy behavior.
