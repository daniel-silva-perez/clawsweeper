## Summary

Keep open for normal PR review. Current main still has the reported display mismatch: the runner skips on dirty porcelain output, while the progress UI can label the same zero-exit clean-check step as clean. The PR is narrow, covered by focused tests in the diff, and I did not find a blocking correctness or security issue.

## What This Changes

This PR forwards `stdoutTail` to update step completion, changes update progress/result rendering to treat non-empty clean-check stdout as dirty, surfaces dirty entries with `git stash -u` guidance, and adds CLI/progress tests plus a changelog entry.

## Best Possible Solution

Keep the runner’s existing dirty-tree detection and land a small display/reporting fix that passes the clean-check stdout through the progress/result path, with targeted tests for the contradictory output case and the skipped-result guidance.

## Reproduction Assessment

Yes. A code-level reproduction is high-confidence: make `git status --porcelain` emit dirty entries while exiting 0, and current main’s runner returns `reason: dirty` while the progress renderer only sees the zero exit code and can print a clean working tree message.

## Solution Assessment

Yes. The PR’s direction is the narrowest maintainable fix because it reuses the runner’s existing stdout signal, centralizes display resolution, and avoids changing git update semantics.
