## Summary

Keep open. Current main still resolves an unset memory sync watcher to enabled, and both the builtin and QMD full manager paths can create long-lived chokidar watchers after gateway memory search opens the non-transient manager. The related watcher freshness report is adjacent but does not implement the requested gateway/headless default policy.

## What This Changes

Request `memorySearch.sync.watch` to default to `false` in headless gateway mode while preserving explicit `sync.watch: true` overrides and existing CLI/desktop behavior.

## Best Possible Solution

Add a runtime-aware default so unset memory sync watching resolves to disabled only for the intended headless gateway manager context, while explicit true/false config remains authoritative and interactive CLI/desktop freshness keeps its existing default.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to leave `sync.watch` unset, resolve memory sync config for an enabled agent, then follow a gateway memory_search call into the default/full manager path where builtin and QMD managers can create chokidar watchers because the resolved value is true.

## Solution Assessment

Unclear as originally phrased. The safer maintainable fix is not a blanket `gateway.mode` check, but a narrow headless/default-policy context threaded into sync resolution or manager creation so explicit overrides and interactive local behavior remain stable.
