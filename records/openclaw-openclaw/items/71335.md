## Summary

Keep this open. Current main still resolves an unset memory sync watcher to true, and the normal gateway memory_search path opens full/default memory managers that can start builtin or QMD chokidar watchers; the related closed #64549 is adjacent watcher behavior, not this gateway default policy change.

## What This Changes

Request to default unset `memorySearch.sync.watch` to false for headless gateway operation while preserving explicit overrides and CLI/desktop watch defaults.

## Best Possible Solution

Add a context-aware memory sync default so unset `sync.watch` resolves false only for the intended headless gateway manager context, while explicit true/false overrides remain authoritative and CLI/status/desktop freshness stays covered by tests and schema help.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to leave `sync.watch` unset, resolve memory sync config, then follow a normal gateway memory_search call into a default builtin or QMD manager; current main resolves watch true and starts chokidar on the full manager path.

## Solution Assessment

No, not if implemented as a broad raw `gateway.mode` config default. The safer fix is to thread an explicit headless/default-policy context into memory sync resolution or manager creation so only the gateway full-manager default changes.
