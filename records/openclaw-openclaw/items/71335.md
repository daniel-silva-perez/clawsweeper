## Summary

Keep this issue open. Current main still resolves unset memory sync watching to true, and the normal memory_search manager path can create long-lived builtin or QMD chokidar watchers; the related closed #64549 is adjacent watch behavior, not this gateway default policy change.

## What This Changes

Request to make unset `memorySearch.sync.watch` default to false for headless gateway use while preserving explicit overrides and CLI/desktop watch behavior.

## Best Possible Solution

Add a runtime-aware memory sync default so unset `sync.watch` resolves false only for the intended headless gateway manager context, while explicit true/false overrides remain authoritative and CLI/status/desktop freshness stays covered by tests and schema help.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: leave `sync.watch` unset, resolve memory sync config, then follow a normal memory_search call into a default builtin or QMD manager that starts chokidar when the resolved value is true.

## Solution Assessment

Unclear as originally phrased if the implementation keys only on `gateway.mode`. The safer solution is to thread an explicit headless/default-policy context into memory sync resolution or manager creation rather than broadly changing all local/remote gateway-mode behavior.
