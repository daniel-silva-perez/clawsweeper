## Summary

Keep this open. Current main still resolves an unset memory sync watcher to true, and the normal gateway memory_search path can still create long-lived builtin or QMD chokidar watchers; the related #64549 watcher-freshness item is adjacent, not the requested gateway default-policy change.

## What This Changes

Request defaulting `memorySearch.sync.watch` to false for headless gateway mode while preserving explicit config overrides and CLI/desktop defaults.

## Best Possible Solution

Add a runtime-aware memory sync default so unset `sync.watch` resolves to false only for the intended headless gateway manager context, while explicit true/false config remains authoritative and CLI/desktop freshness behavior stays covered by tests and docs.

## Reproduction Assessment

Yes. Leave `sync.watch` unset, resolve memory sync config for an enabled agent, then follow a gateway memory_search call into the full manager path where builtin and QMD managers can start chokidar watchers because the resolved value is true.

## Solution Assessment

Unclear as originally phrased. The requested policy is reasonable, but the safer implementation is to pass an explicit headless/default-policy context into sync resolution or manager creation instead of keying only on raw `gateway.mode`.
