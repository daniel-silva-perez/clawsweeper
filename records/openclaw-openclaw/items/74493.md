## Summary

Keep this PR open. Current main still has the reported Claude CLI token-shaped auth-epoch instability, while the PR is a focused implementation with no blocking code-review finding; it still needs normal maintainer/CODEOWNER review and completion of the remaining CI before merge.

## What This Changes

The PR changes CLI auth-epoch hashing to ignore rotating token material for Claude CLI token-shaped credentials and token auth profiles, adds a null-read Claude fallback, bumps the auth-epoch version, adds regression tests, and updates the changelog.

## Best Possible Solution

Land this PR after maintainer/CODEOWNER review and green relevant checks if the identity-only token contract is accepted; any parser hardening for misleading token-shaped Claude OAuth reads can be handled as a separate follow-up.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: current main can simulate OAuth, token-shaped, and null Claude credential reads through `setCliAuthEpochTestDeps`, and the current token branches plus session reuse logic explain the reported `auth-epoch` reset path without needing to reproduce the live macOS race.

## Solution Assessment

Yes, with one product-contract caveat. Moving token-shaped Claude credentials to the same identity-only epoch used by OAuth/Gemini is the narrowest fix for rotation-induced resets, and the version bump handles stored-hash migration; maintainers still need to accept that static token material itself is no longer an identity discriminator.
