## Summary

Keep this PR open for normal maintainer review. Current main still has the reported auth-epoch reset path, and the PR is a focused fix with targeted coverage and no discrete code-review finding from this pass; it still needs CODEOWNER acceptance of the identity-only token contract and the currently failing CI gate resolved before merge.

## What This Changes

This PR changes Claude CLI auth-epoch hashing to ignore rotating token material for token-shaped credentials and token auth profiles, adds a null-read Claude fallback, bumps the auth-epoch version, adds regression tests, and updates the changelog.

## Best Possible Solution

Land this PR or an equivalent narrow patch after CODEOWNER review and green required checks, keeping parser hardening as a separate defense-in-depth follow-up unless maintainers want to expand this PR's scope.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main can produce a token-shaped Claude credential when `refreshToken` is missing, token changes alter the current epoch hash, and same-version epoch mismatches reset sessions with `reason=auth-epoch`.

## Solution Assessment

Yes, with a maintainer contract decision. Identity-only epoch hashing plus a version bump and null-read fallback is the narrowest fix for token-rotation resets; changing the parser cache behavior would be a broader follow-up rather than necessary for this symptom.
