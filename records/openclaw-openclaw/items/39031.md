## Summary

Keep open. Current main at 9bb1e59447a5fdf2c0164672df6c748188b26442 still does not forward legacy imsg tapback reactions to the agent: the watcher handles only ordinary message notifications, subscribes only with attachment options, and the parser, payload type, inbound decision, context, and config surfaces are text-message oriented. The issue is also paired with same-author open PR #39322 and overlapping tapback filtering PR #60677, so maintainer review should consolidate those rather than close this issue.

## Best Possible Solution

Keep this issue open as the canonical legacy imsg tapback-to-agent request. Review, rebase, or supersede #39322 with a coordinated implementation that subscribes to reaction notifications, parses reaction target/action metadata, routes tapbacks through either a deliberate typed event or an accepted summary format, defines access-control and dedupe behavior, and updates config, docs, schema, and focused iMessage tests. Coordinate #60677 and #60274 so filtering and opt-in forwarding do not conflict.
