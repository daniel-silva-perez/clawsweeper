## Summary

Keep this issue open. Current main still does not forward legacy iMessage `imsg rpc` tapback/reaction notifications to the agent: the watcher only handles ordinary `message` notifications, the subscription sends only attachment options, the payload/parser model ordinary message fields, and the inbound decision/context path is text-message oriented. The issue is also paired with same-author open PR #39322, so the best maintainer path is to review, land, or supersede that PR rather than close the issue prematurely.

## Best Possible Solution

Keep this issue open as the canonical legacy iMessage `imsg rpc` tapback-to-agent request. Review, rebase, or supersede #39322 with an implementation that subscribes to reaction notifications, parses target/reaction metadata, routes tapbacks as typed reaction/system events or an accepted summary form, defines access-control and dedupe behavior, and updates config, docs, and tests. Coordinate with #60677 so tapback filtering remains compatible with opt-in forwarding.
