## Summary

Keep this issue open. Current main still does not forward legacy iMessage `imsg rpc` tapback reactions to the agent: the watcher only handles ordinary `message` notifications, subscribes without a reaction flag, parses only `raw.message`, and the inbound dispatch path is text-message oriented. The provided discussion also shows this issue is paired with the same-author open implementation PR #39322, so it should remain open until that PR is reviewed, landed, or superseded.

## Best Possible Solution

Keep this issue open as the canonical legacy iMessage `imsg rpc` tapback-to-agent feature request. The best path is to review, rebase, or supersede #39322 with a typed implementation that subscribes to reaction notifications, parses target/reaction metadata, routes tapbacks as reaction/system events or an explicitly accepted summary form, defines access-control and dedupe behavior, and updates config, docs, and tests. Coordinate with #60677 so filtering tapbacks as noise does not conflict with forwarding them when enabled.
