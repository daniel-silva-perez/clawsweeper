## Summary

Current main still has the reported empty-input construction path for `openai-codex-responses`: Codex moves the system prompt into top-level `instructions`, calls `convertResponsesMessages` with `includeSystemPrompt: false`, and assigns the resulting message array directly to `input`. With `context.messages: []`, that array remains empty. I did not find a merged fix, regression assertion, or canonical superseding item in the supplied related context, so this should stay open.

## Best Possible Solution

Fix the OpenAI Codex Responses transport in `src/agents/openai-transport-stream.ts` so a Codex request with instructions but no converted messages still sends a backend-accepted input item or otherwise fails locally with a clear error. Add a focused regression test for `buildOpenAIResponsesParams` with provider `openai-codex`, api `openai-codex-responses`, `systemPrompt` set, and `messages: []`, and consider a runtime-only prompt test if that is the trigger path.
