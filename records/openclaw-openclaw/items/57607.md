## Summary

Keep open. Current main still has the reported Mattermost path: `GROUP_POLICY_NOT_ALLOWLISTED` returns before pending history is recorded, while mention/onchar drops do record pending history. There is also an open same-author implementation candidate, #57609, with closing syntax for this issue, so the maintainer path is to review/land or close that PR rather than closing the issue early.

## Best Possible Solution

Review and either land #57609 or implement the equivalent directly in `extensions/mattermost/src/mattermost/monitor.ts`: record bounded pending history for visible group messages from non-allowlisted senders, then return without generating a response. Add a focused Mattermost monitor regression test for `groupPolicy: "allowlist"` with a non-allowlisted sender.

## Risks / Open Questions

- The fix must preserve access control: non-allowlisted Mattermost users should not trigger responses or commands, only populate bounded pending context when the gateway already received their group messages.

- A regression test should exercise the monitor message-handling path, not only the auth helper layer.
