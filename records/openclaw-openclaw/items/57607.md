## Summary

Keep open. Current main still returns immediately for Mattermost `GROUP_POLICY_NOT_ALLOWLISTED` before the pending-history helper is defined or called, so the reported behavior remains current. The provided GitHub context also shows an open same-author implementation PR, #57609, with closing syntax for this issue; the maintainer path is to review/land or close that PR, not close the issue early.

## Best Possible Solution

Review and either land #57609 or implement the equivalent directly in `extensions/mattermost/src/mattermost/monitor.ts`: route visible non-allowlisted group messages through bounded pending-history recording, then return without generating a response. Add a focused Mattermost monitor regression test for `groupPolicy: "allowlist"` with a non-allowlisted sender, and update `docs/channels/groups.md` / `docs/channels/mattermost.md` if maintainers confirm context visibility should include those messages.
