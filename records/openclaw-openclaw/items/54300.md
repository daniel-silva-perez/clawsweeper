## Summary

Keep open. Current main still leaves the two narrow hardening gaps from the prior review: Docker label names are interpolated into a Go template without validation, and the thread-ownership plugin still builds a forwarder URL from raw path segments. The TLS/private-WS override and Control UI dangerous flags are documented, audited, and tested break-glass mechanisms, so changing those should be a maintainer security/product decision rather than a cleanup close.

## Best Possible Solution

Keep this issue open for maintainer security triage. Land narrow hardening for Docker label validation and thread-ownership path-segment encoding or strict validation, with focused regression tests. Treat the private-WS override and Control UI dangerous flags as a separate policy decision because current main documents, audits, and tests them as explicit break-glass mechanisms.
