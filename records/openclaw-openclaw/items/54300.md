## Summary

Keep open. Current main still leaves the two narrow hardening gaps identified in the prior review: `readDockerContainerLabel` interpolates a string label into a Docker Go template without validation, and the thread-ownership plugin still builds the forwarder ownership URL from raw path segments. The TLS/private-WS and Control UI dangerous flags are documented and tested break-glass behavior, so changing those needs maintainer security/product judgment rather than cleanup closure.

## Best Possible Solution

Keep this open for maintainer security triage. Land narrow hardening for Docker label validation and thread-ownership path-segment encoding or strict validation, with focused regression tests. Treat the private-WS override and Control UI dangerous flags as a separate policy decision because current main documents, audits, and tests them as operator break-glass mechanisms.
