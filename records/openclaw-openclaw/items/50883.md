## Summary

Keep this PR open. Current main still does not implement a `openclaw sessions --kind` filter, and the PR remains a focused CLI improvement, but the submitted change is not complete because the default route-first sessions path would not parse or forward the new option.

## Best Possible Solution

Revise this PR against current main: add the shared session-kind parser/helper, wire repeatable and comma-separated `--kind` through Commander, update `parseSessionsRouteArgs` and route tests so route-first invocations preserve the filter, apply the filter in `sessionsCommand` for both text and JSON output, document the option on `docs/cli/sessions.md`, and keep the session-kind type shared to avoid drift.
