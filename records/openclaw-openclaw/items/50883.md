## Summary

Keep this PR open. Current main still does not implement `openclaw sessions --kind`, and the PR remains a focused CLI feature, but the submitted patch is incomplete because it does not update the default route-first sessions parser or docs.

## Best Possible Solution

Revise this PR against current main: add the shared session-kind parser, wire repeatable and comma-separated `--kind` through Commander and `parseSessionsRouteArgs`, apply the filter in `sessionsCommand` for text and JSON output, add route/command tests, update `docs/cli/sessions.md`, and keep the session-kind type shared to avoid drift.
