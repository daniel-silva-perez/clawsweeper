## Summary

Keep this PR open. Current main still does not implement `openclaw sessions --kind`, and the PR is a focused CLI improvement, but the submitted diff remains incomplete because the default route-first sessions path would drop the new filter before it reaches `sessionsCommand`.

## Best Possible Solution

Revise this PR against current main: add the shared session-kind parser/helper, wire `--kind` through Commander with repeatable and comma-separated values, update `parseSessionsRouteArgs` so route-first invocations preserve the filter, apply the filter in `sessionsCommand` for both text and JSON output, document the option on the sessions CLI page, and add focused tests for helper parsing, Commander forwarding, route-first forwarding, and filtered output.
