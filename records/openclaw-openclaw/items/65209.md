## Summary

Keep open. Current main still has the reported UX gap: the CLI help and human `mcp list` output remain generic, the implementation still reads only OpenClaw config `mcp.servers`, and the tests do not lock in a registry-scope label. The issue is also paired with open PR #73503, which explicitly targets this scope and should be reviewed or replaced before this issue is closed. ([github.com](https://github.com/openclaw/openclaw/pull/73503))

## What This Changes

The issue asks for CLI/help/docs wording that makes `openclaw mcp list` clearly distinguish OpenClaw-managed `mcp.servers` from mcporter-managed `config/mcporter.json` servers.

## Best Possible Solution

Land PR #73503 or an equivalent focused patch that labels the saved registry as OpenClaw-managed `mcp.servers`, states that mcporter `config/mcporter.json` is handled separately by mcporter, adds a CLI regression test for the human output, and leaves machine-readable JSON unchanged.
