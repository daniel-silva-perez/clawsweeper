## Summary

Keep this PR open. Current main still does not implement the requested whole-tool hard timeout for `memory_search`, and the PR remains a relevant implementation candidate for linked liveness bug #49524. It needs to be ported from the old `src/agents/tools` location to the current plugin-owned implementation under `extensions/memory-core`.

## Best Possible Solution

Keep this PR open as a relevant implementation candidate, but rework it against current main: add a bounded timeout around `extensions/memory-core` `memory_search` execution covering manager acquisition plus search, add timeout-specific unavailable metadata in `extensions/memory-core/src/tools.shared.ts`, and add a fake-timer regression test in `extensions/memory-core/src/tools.test.ts`.

## Risks / Open Questions

- Landing the PR as-is would miss the active code path because current main's `memory_search` is implemented in `extensions/memory-core`, while the PR edits the old `src/agents/tools` location.
- A fixed 30-second outer timeout needs maintainer review against existing longer local embedding defaults and QMD behavior.
- A fail-open `Promise.race` timeout can allow the turn to continue while the underlying manager/search work keeps running, so the port should account for pending-manager cache state and side effects.
