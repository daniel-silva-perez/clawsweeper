## Summary

Keep open. The liveness bug remains in current main, and this PR is relevant to the linked report, but the branch targets the legacy `src/agents/tools` memory tool path while current `memory_search` is owned by `extensions/memory-core`.

## What This Changes

The PR adds a 30-second `Promise.race` hard timeout, timeout-specific unavailable metadata, and a fake-timer regression test for the legacy `src/agents/tools` `memory_search` implementation.

## Best Possible Solution

Port the timeout to the active memory-core plugin boundary so manager acquisition plus search always returns a bounded unavailable tool result, while preserving corpus selection, citations, session visibility filtering, QMD debug metadata, and existing subsystem timeout behavior.

## Reproduction Assessment

Yes. A focused code-level reproduction is clear: stub `memory.manager.search` or manager acquisition to return a never-settling promise and call the active `extensions/memory-core` `memory_search` tool; current main has no outer timeout, so the tool result remains pending.

## Solution Assessment

No. The intended timeout is a reasonable narrow fix, but this PR is not the best current implementation because it patches the removed legacy path; the safer path is a memory-core port with timeout-specific metadata and tests on the active files.
