## Summary

Keep open. Current main still has only the CDP screenshot path and does not accept `browser.hyprlandCapture`, while this PR is an active implementation for the still-open Hyprland/Wayland screenshot bug. The branch is useful, but the latest diff still has a setup-failure cleanup leak and appears to leave the Plugin SDK API baseline hash stale after adding a new public SDK subpath, so it needs in-branch revision before merge.

## What This Changes

The PR adds an opt-in `browser.hyprlandCapture` headed Hyprland/Wayland viewport screenshot fallback using `hyprctl` plus `grim`, updates config/schema/docs and a Plugin SDK subpath, and adds focused browser capture and route tests.

## Best Possible Solution

Revise this PR in place so the opt-in browser-plugin fallback remains limited to headed plain viewport screenshots, removes every created Hyprland virtual output on setup, capture, and shutdown failure paths, commits the required generated SDK/API hash updates, and then lands with targeted browser tests plus the changed gate for #64317.

## Reproduction Assessment

Yes. The linked bug gives a concrete Omarchy/Hyprland/Wayland headed Chromium path using GitHub Trending or Yahoo Finance, and current main still routes normal viewport screenshots through CDP `Page.captureScreenshot` with no Hyprland fallback.

## Solution Assessment

No, not as currently proposed. The PR direction is the narrow maintainable fix, but the branch should not merge until post-create setup failures remove the virtual output and the public SDK subpath baseline is aligned.
