## Summary

Keep this PR open. Current main still lacks the Hyprland/grim screenshot fallback and the linked bug has concrete reproduction evidence, but the proposed patch should not merge as-is because it introduces a bundled-plugin import that the repository's boundary guard rejects and it can still leak a virtual Hyprland output when setup fails after output creation.

## What This Changes

The PR adds an opt-in `browser.hyprlandCapture` path that uses Hyprland `hyprctl` plus `grim` for headed viewport screenshots, wires the flag through browser config/schema/docs, exports `resolveSystemBin` through the plugin SDK infra runtime barrel, and adds capture/route tests.

## Best Possible Solution

Land a focused browser-plugin fix for #64317 that keeps the grim fallback opt-in and limited to plain headed viewport screenshots, uses a focused plugin SDK seam for trusted system-binary resolution instead of the deprecated broad barrel, tears down virtual outputs on every setup/capture failure path, preserves CDP/Playwright behavior elsewhere, and updates config schema/docs/tests together.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use a focused SDK subpath for system binaries:** `extensions/browser/src/browser/hyprland-capture.ts:3`
  - body: This bundled browser plugin imports `openclaw/plugin-sdk/infra-runtime`, but that broad barrel is a deprecated compatibility surface and `scripts/check-no-monolithic-plugin-sdk-entry-imports.ts` explicitly rejects bundled plugin imports of it. Add or use a focused plugin SDK runtime subpath for `resolveSystemBin` instead, and keep the SDK docs/API surface aligned.
  - confidence: 0.94
- **[P2] Clean up the output when setup fails:** `extensions/browser/src/browser/hyprland-capture.ts:106-112`
  - body: After `createHeadlessOutput()` returns, `getActiveWorkspaceId()` or the `movetoworkspacesilent` dispatch can still throw. In that path the setup catch only clears `_state` and `_setupInFlight`, and cleanup is not registered yet, so the newly created Hyprland output is left behind. Wrap the post-create setup steps so the output is removed before rethrowing.
  - confidence: 0.88
