## Summary

Keep open. Current main still has only the CDP screenshot path and no `browser.hyprlandCapture` config/docs support, while the linked Hyprland screenshot bug remains open and concrete. The PR is a plausible fix direction, but the latest patch still has blocking review findings around the plugin SDK boundary and virtual-output cleanup before merge.

## What This Changes

The PR adds an opt-in `browser.hyprlandCapture` path for headed Linux/Hyprland viewport screenshots using `hyprctl` virtual outputs and `grim`, updates config/schema/docs, exports `resolveSystemBin` through `plugin-sdk/infra-runtime`, and adds capture and route tests.

## Best Possible Solution

Revise this PR in place so the opt-in browser-plugin fallback stays limited to headed plain viewport screenshots, uses a narrow documented plugin SDK runtime subpath for trusted system-binary resolution, removes any created virtual output on every setup and capture failure path, and keeps schema/docs/tests/API checks aligned before merge.

Overall correctness: patch is incorrect

  - body: This bundled browser plugin imports `openclaw/plugin-sdk/infra-runtime`, but that broad runtime barrel is a deprecated compatibility surface and the bundled-plugin import guard rejects it. Add or use a focused SDK runtime subpath for `resolveSystemBin` and update the export/docs/API checks and tests to use that path.
  - confidence: 0.94

  - confidence: 0.9
