## Summary

Keep open. Current main still has the CDP-only headed viewport screenshot path and no `browser.hyprlandCapture` config/docs surface, while the linked Hyprland bug remains open with concrete reproduction data. The PR is useful but not merge-ready because the latest diff still imports a deprecated broad plugin SDK runtime barrel from bundled browser plugin code and can leak a newly-created Hyprland virtual output if setup fails after `create-output`.

## What This Changes

The PR adds an opt-in `browser.hyprlandCapture` browser screenshot path using Hyprland `hyprctl` plus `grim`, wires config/schema/docs, exports `resolveSystemBin` through `infra-runtime`, and adds browser capture and route tests.

## Best Possible Solution

Revise this PR in place so the opt-in browser-plugin Hyprland/grim fallback stays limited to headed plain viewport screenshots, uses a narrow documented plugin SDK subpath for trusted system-binary resolution, removes any created virtual output on every setup and capture failure path, and keeps schema/docs/tests/API checks aligned before merge.
