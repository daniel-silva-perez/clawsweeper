Keep open. Current main still routes normal browser viewport screenshots through CDP Page.captureScreenshot and has no Hyprland/grim fallback, no opt-in BrowserConfig key, no strict schema support, no docs entry, and no Hyprland route/cleanup tests. The linked bug #64317 has concrete reproduction evidence, and this PR is an active implementation candidate, but the PR still needs maintainer review and changes before it should merge.

## Best Possible Solution

Keep this PR and #64317 open. The best path is a focused browser-plugin fix that adds an opt-in Hyprland/Wayland headed viewport screenshot fallback only for ordinary viewport screenshots, preserves CDP/Playwright behavior for non-Hyprland, full-page, element, ref, and label cases, wires the flag through BrowserConfig plus strict schema/help/generated docs, documents grim/hyprctl setup, tests route selection and virtual-output cleanup, uses per-profile headless state, removes unrelated or misleading branch changes, and resolves the cleanup/lifecycle review comments before merge.
