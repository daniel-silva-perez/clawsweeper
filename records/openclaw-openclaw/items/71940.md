## Summary

Keep open. Current main still routes ordinary browser screenshots through CDP only, while this PR adds the missing Hyprland/grim fallback for the concrete #64317 report; however, the latest patch still has merge-blocking cleanup and contract issues, plus a stale body that points at the unrelated Windows Gmail setup issue #54470.

## What This Changes

The PR adds an opt-in Hyprland/Wayland `grim` viewport screenshot fallback for headed browser sessions, with a browser config flag, docs, route tests, a Hyprland capture helper, and a new `openclaw/plugin-sdk/resolve-system-bin` subpath.

## Best Possible Solution

Revise this PR in place: preserve normal process shutdown while cleaning the virtual output, remove the virtual output on every setup failure after creation, regenerate the Plugin SDK API baseline or avoid the new public subpath, update the PR body to target #64317, and then run focused browser/config/SDK checks plus the changed gate before merge.

## Reproduction Assessment

Yes. #64317 gives a concrete Omarchy/Hyprland/Wayland headed Chromium screenshot timeout reproduction, and the current patch defects are reproducible by source tracing the setup and signal paths at PR head.

## Solution Assessment

No. The feature direction is plausible because current main lacks the fallback, but this patch is not yet the best mergeable form while it can swallow stop signals, leak virtual outputs on setup failure, and leave the public Plugin SDK API baseline stale.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve normal shutdown after signal cleanup:** `extensions/browser/src/browser/hyprland-capture.ts:84-93`
  - body: After Hyprland capture is used, these `SIGINT`/`SIGTERM` listeners run cleanup but do not let the signal terminate the process. Node removes the default signal behavior once a listener is installed, so a normal wrapper/systemd stop can leave the gateway child alive until a later force kill. Clean up the output and then preserve normal shutdown behavior.
  - confidence: 0.86
- **[P2] Remove the virtual output if setup fails:** `extensions/browser/src/browser/hyprland-capture.ts:106-112`
  - body: `createHeadlessOutput()` can succeed before `getActiveWorkspaceId()` or the workspace move throws, but cleanup is not registered until afterward. That leaves the new `HEADLESS-*` output behind even though the route falls back to CDP. Wrap the post-create setup and remove `outputName` before rethrowing.
  - confidence: 0.93
- **[P2] Regenerate the Plugin SDK API baseline:** `scripts/lib/plugin-sdk-entrypoints.json:67`
  - body: Adding `resolve-system-bin` to the public SDK entrypoint list changes the generated Plugin SDK API surface, but the PR does not update the generated baseline artifacts. `pnpm plugin-sdk:api:check` will report drift until the baseline is regenerated and committed, or the new public subpath is removed.
  - confidence: 0.88
