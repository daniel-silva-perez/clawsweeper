## Summary

Keep this PR open. Current main still implements the exact-fingerprint recovery behavior, and it still has a regression test asserting that a same-tool success on a different target keeps `lastToolError`, which is the behavior this PR proposes to change. The linked issue was closed after a Codex review of the same-action retry path, but that does not make this PR redundant because the different-target retry semantics remain absent on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide the intended different-target retry semantics, then either land the narrowed same tool/action-type implementation with its regression tests or close it with an explicit product rationale that exact target fingerprints are the desired warning contract.
