## Summary

Keep this issue open. Current main still has task lifecycle statuses, Task Flow waiting/blocked state, operational audit codes, delivery status, and free-form error/summary text, but it does not yet provide the shared machine-readable blocker/failure taxonomy requested here; related #64413 is narrower delivery-failure work and does not supersede the task-wide design.

## What This Changes

The issue requests a centralized task blocker/failure taxonomy, with shared helpers and mappings reused by audits, summaries, status output, and future retry logic.

## Best Possible Solution

Define an additive shared task blocker/failure classification contract, map existing task, Task Flow, cron, and agent-run conditions into it, expose it through persisted records and DTOs where appropriate, and document how consumers should handle unknown or unclassified cases while treating #64413 as delivery-specific input.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source/docs inspection showing current main has statuses, audit codes, delivery state, and summaries but no shared task blocker/failure taxonomy.

## Solution Assessment

Yes, a centralized additive taxonomy is the maintainable direction. The exact names, persistence shape, DTO compatibility, and rollout order need maintainer design before a safe implementation PR is queued.
