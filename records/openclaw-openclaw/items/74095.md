## Summary

Keep this open. Current main still has coarse task and Task Flow statuses, operational audit codes, delivery status, terminal summaries, and raw error text, but no shared machine-readable blocker/failure taxonomy. The related #64413 is useful input, but it is narrower delivery-failure work and does not supersede this task-wide taxonomy request.

## What This Changes

Add shared task blocker/failure classification types and helpers so blocked, failed, waiting, and transient/recoverable task conditions can be represented consistently across audits, summaries, status output, and future retry logic.

## Best Possible Solution

Design an additive taxonomy in the task subsystem, with shared exported types and mapping helpers, optional persisted/DTO fields, and explicit unknown/unclassified handling. Then map known task, Task Flow, ACP, cron, delivery, cancellation, timeout, and approval-blocked paths, and surface the normalized class consistently in audits, summaries, CLI/status output, plugin runtime DTOs, and docs.

## Reproduction Assessment

Not applicable as a feature/design request. The current-main verification path is static inspection of the task model, Task Flow model, audit codes, runtime DTOs, CLI output, and docs, which shows no shared blocker/failure taxonomy today.

## Solution Assessment

Unclear pending maintainer design. The requested direction is likely the right maintainable shape, but exact class names, persistence, DTO exposure, and interaction with #64413 need an additive compatibility plan before a narrow fix PR is safe.
