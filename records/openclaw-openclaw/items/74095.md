## Summary

Keep this issue open. Current main still has coarse task statuses, Task Flow waiting/blocked state, operational audit codes, and raw error/summary text, but no shared machine-readable blocker/failure taxonomy; related #64413 is narrower delivery-failure work and does not supersede the broader task taxonomy request.

## What This Changes

Define and adopt shared blocker/failure classification types and helpers for task waits, blockers, failures, audits, summaries, status output, and future retry logic.

## Best Possible Solution

Keep this open for a maintainer-owned additive design: define shared task blocker/failure classification types and mapping helpers, add optional persisted/DTO fields where appropriate, map known task/flow/cron/agent-run conditions, reuse the taxonomy in audits, task summaries, CLI/status output, plugin runtime DTOs, and docs, and keep #64413 linked as narrower delivery-failure input.
