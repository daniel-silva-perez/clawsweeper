## Summary

Keep this issue open. Current main still has coarse task, delivery, Task Flow, audit, and plugin runtime DTO statuses, but no shared blocker/failure taxonomy that can distinguish approval waits, external waits, provider/transport/config errors, timeouts, interruptions, unknown failures, or recoverable transient conditions. Related #64413 is narrower delivery-failure work and complements rather than supersedes this broader task taxonomy request.

## What This Changes

The issue requests a shared machine-readable blocker/failure taxonomy for task blocked/failure conditions across audits, summaries, status output, and future retry logic.

## Best Possible Solution

Keep this open for a maintainer-owned additive design: define shared task blocker/failure classification types and mapping helpers, add optional persisted/DTO fields where appropriate, map known task/flow/cron/agent-run conditions, and reuse the taxonomy in audits, task summaries, CLI/status output, plugin runtime DTOs, and docs. Keep #64413 linked as narrower delivery-failure input rather than the canonical replacement.
