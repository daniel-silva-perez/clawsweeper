## Summary

Keep this open. Current main still exposes coarse task and Task Flow statuses, delivery status, operational audit codes, and prose summaries/errors, but not a shared machine-readable blocker/failure taxonomy; the related delivery-failure item is narrower and does not supersede the broader task taxonomy request.

## What This Changes

Request to define and adopt shared machine-readable blocker/failure classes for tasks, audits, summaries, user-facing status output, and future retry logic.

## Best Possible Solution

Define an additive shared task blocker/failure classification contract, map known task, Task Flow, cron, and agent-run conditions onto it, expose optional persisted/DTO fields where compatibility allows, and reuse the normalized class in audits, summaries, CLI/status output, plugin runtime views, and docs while keeping #64413 as narrower delivery-failure input.
