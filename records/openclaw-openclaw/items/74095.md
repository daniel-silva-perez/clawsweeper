## Summary

Keep this issue open. Current main has coarse task statuses, TaskFlow waiting/blocked states, audit codes, and raw error/summary text, but it does not have the requested shared blocker/failure taxonomy that can distinguish approval waits, external waits, provider errors, transport errors, config errors, timeouts, interruptions, and unknown failures in a reusable machine-readable way. Related #64413 is still open and narrower to delivery failure reasons, so it complements rather than supersedes this broader task taxonomy request.

## Best Possible Solution

Keep this issue open for a maintainer-owned additive design. The likely path is a shared task blocker/failure classification type in the task domain, optional fields on TaskRecord and TaskFlow metadata, mapping helpers for known task/flow/cron/agent-run conditions, and reuse in task audit JSON, task summaries, CLI/status output, plugin runtime DTOs, and docs. Related #64413 should stay linked as a narrower delivery-reason follow-up or input, not as the canonical replacement for this broader task taxonomy.
