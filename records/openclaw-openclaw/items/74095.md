## Summary

Keep this open. Current main still has coarse task statuses, delivery statuses, Task Flow waiting/blocked state, operational audit codes, and prose error/summary text, but not the requested shared machine-readable blocker/failure taxonomy; related #64413 is narrower delivery-failure work and does not supersede the broader task-system request.

## What This Changes

Request a shared blocker/failure taxonomy for tasks so blocked, waiting, failed, and recoverable or transient failure states have reusable machine-readable classes across audits, summaries, and status output.

## Best Possible Solution

Define an additive shared task blocker/failure classification contract, map existing task, Task Flow, audit, delivery, and agent-run conditions into it, expose it through persisted records and DTOs where compatible, and document unknown or unclassified handling while using #64413 as delivery-specific input.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source/docs inspection plus targeted search showing current main has statuses and audit codes but no shared task blocker/failure taxonomy.

## Solution Assessment

Yes, the requested centralized additive taxonomy is the maintainable direction, but the exact contract and rollout boundaries should be chosen before implementation so consumers do not branch on another ad hoc field.
