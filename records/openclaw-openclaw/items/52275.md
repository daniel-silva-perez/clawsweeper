## Summary

Keep open. Current main still does not implement this PR's central legacy queue canonicalization: queue files are parsed and cast as canonical `QueuedDelivery` records, and the only legacy migration backfills `lastAttemptAt`. The PR branch contains useful outbound storage work, but it also carries unrelated gateway readiness changes with structurally invalid gateway code and unresolved review blockers, so it is not safe to close as implemented or land as-is.

## Best Possible Solution

Keep this PR open for contributor or maintainer cleanup: split out the unrelated gateway readiness changes, rebase onto current main, implement focused legacy queue canonicalization in `src/infra/outbound/delivery-queue-storage.ts`, preserve `forceDocument`, `mirror.idempotencyKey`, mirror group context, `session`, and `gatewayClientScopes`, and add storage/recovery regression tests for legacy queue files before review.
