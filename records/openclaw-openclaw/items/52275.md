## Summary

Keep open. Current main still lacks the PR's central legacy delivery-queue canonicalization, and the live PR branch is not safe to close as implemented or land as-is because it also carries unrelated gateway readiness changes with unresolved functional blockers.

## Best Possible Solution

Keep this PR open for cleanup or replacement. The focused fix should live in `src/infra/outbound/delivery-queue-storage.ts`: normalize legacy queue records on load and before failure bookkeeping, preserve `forceDocument`, `mirror.idempotencyKey`, mirror group context, `session`, and `gatewayClientScopes`, and add storage/recovery regression tests for legacy queue files. The unrelated gateway readiness changes should be split out or dropped before review.
