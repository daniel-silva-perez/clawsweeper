## Summary

Keep this PR open. Current main still lacks the central legacy queue-entry canonicalization, so the outbound part is not obsolete; however, the PR branch is not mergeable as-is because it includes unrelated gateway readiness/startup changes with parse/type blockers and the queue normalizer still drops several replay metadata fields.

## What This Changes

The PR branch adds legacy outbound delivery-queue normalization and regression tests, changes queued mirror typing/test helpers, and adds gateway agent-create runtime readiness refresh code and tests.

## Best Possible Solution

Land the queue fix as a narrow storage/recovery change: normalize legacy queue records on load and before failure bookkeeping, preserve every QueuedDeliveryPayload field used by recovery, and add legacy-file regression tests. Drop the gateway readiness/startup changes from this PR or move them to a separate gateway-owned review.

## Reproduction Assessment

Yes. A legacy queue JSON with payload/target/attempt/createdAt can be inspected through loadPendingDeliveries or failDelivery on current main; the source shows those paths do not canonicalize those fields before replay or retry bookkeeping.

## Solution Assessment

No. The storage normalizer is the right implementation boundary, but this PR is not the best mergeable form because it mixes unrelated gateway runtime changes and the proposed normalizer still loses queued replay metadata.

  - confidence: 0.93
