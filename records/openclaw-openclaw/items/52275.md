## Summary

Keep this PR open. Current main preserves metadata for newly enqueued canonical delivery records, but it still does not implement the PR's central legacy queue-entry canonicalization before recovery or failure bookkeeping. The PR branch contains useful outbound storage work, but as provided it also carries unrelated gateway readiness/startup changes with serious unresolved review blockers, so it should be cleaned up or replaced rather than closed as implemented.

## Best Possible Solution

Keep this PR open for maintainer cleanup or replacement. The focused fix should live in src/infra/outbound/delivery-queue-storage.ts: normalize legacy queue records on load and before failure bookkeeping, preserve forceDocument, mirror.idempotencyKey, group mirror context, session, and gatewayClientScopes, and add storage/recovery regression tests for legacy queue files. Split out or drop the unrelated gateway readiness changes before review.
