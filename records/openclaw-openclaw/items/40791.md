## What This Changes

The PR adds an opt-in Bash rescue watchdog, a Claude Code incident runner, systemd user timer templates, rescue-watchdog documentation, and small test typing updates.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Reset suppression state after healthy checks:** `scripts/rescue-watchdog.sh:175-177`
  - body: When the watchdog sees a healthy cycle, it exits without updating `state.env`. A fail -> recover -> fail sequence inside the cooldown window can reuse the old fingerprint and suppress the fresh outage, so no incident, runner, or notification fires.
  - confidence: 0.86
- **[P2] Use collision-proof incident directories:** `scripts/rescue-watchdog.sh:196-198`
  - body: `incident_id` has only second-level granularity and `mkdir -p` reuses an existing directory. Two overlapping timer/manual runs in the same second can write into the same bundle, producing mixed or partial incident artifacts.
  - confidence: 0.88
- **[P2] Persist suppression state before runner execution:** `scripts/rescue-watchdog.sh:250`
  - body: `state.env` is saved only after `RUNNER_CMD` returns. During a long rescue run, another watchdog invocation sees no updated fingerprint and can launch a duplicate runner or notification cycle for the same outage.
  - confidence: 0.89

## Security Review

Status: needs_attention

Summary: Security review needs attention because the PR adds opt-in command execution and diagnostic capture without enough hardening for repo-maintained automation.

Concerns:

- **[medium] Harden sourced watchdog state:** `scripts/rescue-watchdog.sh:51`
  - body: `load_state` sources `$STATE_FILE` from an environment-configurable state directory without regular-file, ownership, mode, or symlink checks. Replacing `state.env` would turn the next watchdog run into shell execution.
  - confidence: 0.84
- **[medium] Warn about sensitive incident bundles:** `docs/automation/rescue-watchdog.md:38`
  - body: The docs list health snapshots, status output, gateway log tails, git state, and runner output in the incident bundle, and the runner adds that directory to model context. Operators need explicit permissions, retention, and sensitive-data guidance before enabling this flow.
  - confidence: 0.82
