## Summary

Keep this PR open. Current main still lacks Feishu reply-message orphan-placeholder mention detection, and the paired open bug report #39568 is still the canonical user problem this PR attempts to fix. Main has newer, more nuanced reply-target handling for normal groups, but the mention gate still drops the reported event and the PR needs rebase/rework against the current `bot-content.ts` split plus safer topic-routing behavior.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #39568. Rebase it onto the current `bot-content.ts` split, add regression tests for Feishu reply events where the bot mention is dropped from `mentions`, make the fallback as bot-specific as the Feishu payload allows, and preserve topic/session routing instead of clearing `rootId` broadly.

## Risks / Open Questions

- The PR's orphan-placeholder heuristic treats any unmatched placeholder in a reply as a bot mention, which can create false positives and bypass Feishu group `requireMention`.
- Clearing `rootId` broadly can collapse topic-scoped sessions or reroute replies in configurations that intentionally use `group_topic` or `group_topic_sender`.
- The PR targets an older code shape: current main has `checkBotMentioned()` in `extensions/feishu/src/bot-content.ts`, so the patch must be rebased before it can be evaluated or landed.
