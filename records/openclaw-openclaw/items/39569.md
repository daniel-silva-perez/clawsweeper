## Summary

Keep this PR open. Current main still does not detect Feishu reply-message orphan placeholders as bot mentions, and the paired bug report #39568 remains open with this PR using closing syntax. The PR is still relevant, but it needs rebase/rework because main moved mention detection into `bot-content.ts` and current topic routing makes the proposed blanket `rootId` clearing risky.

## Best Possible Solution

Keep this PR open as a still-relevant implementation candidate for #39568. Rebase it onto the current `bot-content.ts` split, add regression tests for Feishu reply events where the bot mention is dropped from `mentions`, make the fallback as bot-specific as the payload allows, and preserve intentional topic/session routing when handling `rootId`.

## Risks / Open Questions

- A broad orphan-placeholder fallback can mark the bot as mentioned when Feishu drops a non-bot mention or user text contains placeholder-like tokens, bypassing `requireMention`.
- Clearing `rootId` for every mentioned reply can collapse or reroute Feishu topic-scoped sessions that intentionally depend on `root_id`/`thread_id`.
- The PR targets older code shape: current main has `checkBotMentioned()` in `extensions/feishu/src/bot-content.ts`, so the patch needs rebase before it can be evaluated or landed.
