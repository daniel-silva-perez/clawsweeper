## Summary

Keep this PR open. Current main still lacks Feishu reply-message orphan-placeholder mention detection, so the paired bug #39568 remains unresolved. Main now has more nuanced reply-target handling for normal groups, but the PR still needs rebase/rework against the current `bot-content.ts` split and safer topic-routing behavior before it can land.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #39568. Rebase it onto the current `bot-content.ts` split, add regression tests for Feishu reply events where the bot mention is dropped from `mentions`, make the fallback as bot-specific as the Feishu payload allows, and preserve topic/session routing instead of clearing `rootId` broadly.

## Risks / Open Questions

- The PR targets an older code shape: current main has `checkBotMentioned()` in `extensions/feishu/src/bot-content.ts`, so the patch must be rebased before maintainer review can be meaningful.
- The PR's orphan-placeholder heuristic treats any unmatched placeholder in a reply as a bot mention, which can create false positives and bypass Feishu group `requireMention`.
- Clearing `rootId` broadly can break `group_topic` and `group_topic_sender` session isolation, or reroute replies in configurations that intentionally use topic-thread context.
