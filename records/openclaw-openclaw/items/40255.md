## Summary

Keep this issue open. Current main still resolves configured heartbeat prompt text and then unconditionally routes the resulting prompt through the workspace-path hint appender whenever it mentions `HEARTBEAT.md`; the docs still promise custom heartbeat prompts are sent verbatim, and PR #40756 remains the open unmerged fix candidate with closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to stop appending the workspace `HEARTBEAT.md` path hint to user-configured heartbeat prompts and to honor custom prompt text verbatim.

## Best Possible Solution

Land PR #40756 or an equivalent narrow fix that preserves user-authored heartbeat prompts verbatim, keeps the workspace-path hint for built-in/default and internally generated heartbeat prompts, and adds regression coverage for a configured prompt that mentions `HEARTBEAT.md`.
