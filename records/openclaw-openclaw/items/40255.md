## Summary

Keep open. Current main still resolves user-configured heartbeat prompts and then appends the workspace `HEARTBEAT.md` hint based only on prompt text, which conflicts with the documented verbatim custom-prompt contract. The provided GitHub context also shows open PR #40756 uses closing syntax for this issue, so the issue should remain open until that PR or an equivalent replacement lands.

## What This Changes

The issue asks OpenClaw to stop appending the workspace `HEARTBEAT.md` path hint to user-configured heartbeat prompts.

## Best Possible Solution

Land a focused heartbeat-runner change that tracks whether the base heartbeat prompt came from user config, skips the workspace-path hint for non-empty user-authored default and per-agent prompts, preserves the hint for built-in/internal prompts, and adds regression coverage for both default-level and per-agent custom prompts.

## Reproduction Assessment

Yes. A config with `agents.defaults.heartbeat.prompt` or `agents.list[].heartbeat.prompt` containing `HEARTBEAT.md` reaches `resolveHeartbeatPrompt`, then `resolveHeartbeatRunPrompt` appends the workspace hint because the helper only inspects prompt text.

## Solution Assessment

Yes. The narrow maintainable fix is a prompt-origin guard around hint injection while preserving default/internal prompt behavior; PR #40756 targets that direction, but this issue should remain open until it merges or is replaced.
