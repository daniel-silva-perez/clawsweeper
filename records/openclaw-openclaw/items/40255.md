## Summary

Keep open. Current `main` still resolves configured heartbeat prompts and then appends the workspace `HEARTBEAT.md` hint based only on prompt text, which conflicts with the documented verbatim custom-prompt contract; open PR #40756 is still the active unmerged implementation candidate using closing syntax.

## What This Changes

The issue asks for user-configured heartbeat prompts to be sent unchanged, without appending OpenClaw's workspace `HEARTBEAT.md` path hint.

## Best Possible Solution

Carry prompt-origin information through heartbeat run prompt resolution, skip the workspace-path hint for non-empty user-authored heartbeat prompts, preserve the hint for built-in/internal prompts, and land that behavior with runner-level regression coverage.

## Reproduction Assessment

Yes. A config with `agents.defaults.heartbeat.prompt` or per-agent `heartbeat.prompt` containing `HEARTBEAT.md` reaches `resolveHeartbeatPrompt`, then `resolveHeartbeatRunPrompt` appends the workspace hint because the helper only checks prompt text.

## Solution Assessment

Yes. The narrow maintainable fix is a custom-prompt guard around the hint append; PR #40756 targets that direction, but the issue should remain open until that PR or an equivalent replacement merges.
