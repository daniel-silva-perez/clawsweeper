## Summary

Keep open. Current main still resolves configured heartbeat prompts and then appends the workspace HEARTBEAT.md hint based only on prompt text, which conflicts with the documented verbatim custom-prompt behavior; the issue is also paired with open, unmerged PR #40756 using closing syntax.

## What This Changes

User-configured heartbeat prompts should be honored without appending OpenClaw’s workspace HEARTBEAT.md path hint.

## Best Possible Solution

Land PR #40756 or an equivalent narrow fix that tracks whether the base heartbeat prompt came from user configuration, skips the workspace-path hint for user-authored prompts, preserves the hint for built-in/internal prompts, and adds regression coverage for configured prompts that mention HEARTBEAT.md.

## Reproduction Assessment

Yes. A config with `agents.defaults.heartbeat.prompt` or per-agent `heartbeat.prompt` containing HEARTBEAT.md flows through `resolveHeartbeatPrompt`, then `resolveHeartbeatRunPrompt` appends the workspace hint because the helper only checks prompt text.

## Solution Assessment

Yes, the narrow maintainable direction is to keep the helper for default/internal prompts but bypass it for non-empty user-configured prompts. The open PR targets that direction, but the issue should remain open until that PR or a replacement lands.
