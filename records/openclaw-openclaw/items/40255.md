## Summary

Keep this issue open. Current main still routes user-configured heartbeat prompt text through the workspace-path hint appender whenever it mentions HEARTBEAT.md, and the issue is already paired with an open unmerged fix candidate that uses closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to stop appending the HEARTBEAT.md workspace-path hint to user-configured heartbeat prompts.

## Best Possible Solution

Land PR #40756 or an equivalent narrow fix that preserves user-authored heartbeat prompts verbatim, keeps the workspace-path hint for built-in/default and internally generated heartbeat prompts, and adds regression coverage for a configured prompt that mentions HEARTBEAT.md.
