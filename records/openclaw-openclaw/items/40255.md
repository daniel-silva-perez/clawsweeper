## Summary

Keep this issue open. Current main still matches the reported regression: configured heartbeat prompt text is resolved first, then `resolveHeartbeatRunPrompt` unconditionally sends that text through the workspace-path hint appender whenever it mentions `HEARTBEAT.md`. The docs still promise configured heartbeat prompts are sent verbatim, and open PR #40756 uses closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that OpenClaw stop appending the workspace `HEARTBEAT.md` path hint to user-configured heartbeat prompts.

## Best Possible Solution

Land PR #40756 or an equivalent narrow patch that tracks whether the plain heartbeat prompt came from user config, skips the workspace-path hint only for user-authored configured prompts, preserves the hint for built-in/default and internally generated event prompts, and adds a regression test for a configured prompt that mentions `HEARTBEAT.md`.
