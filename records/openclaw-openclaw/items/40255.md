## Summary

Keep this issue open. Current main still matches the reported regression: user-configured heartbeat prompt text is resolved, then the workspace-path hint appender is called unconditionally whenever the prompt mentions `HEARTBEAT.md`. The docs still promise configured heartbeat prompts are sent verbatim, and PR #40756 remains an open unmerged fix candidate with closing syntax for this issue.

## What This Changes

The issue reports that a custom heartbeat prompt mentioning `HEARTBEAT.md` gets an extra workspace-path instruction appended, contradicting the configured prompt.

## Best Possible Solution

Land PR #40756 or an equivalent narrow fix that preserves user-authored heartbeat prompts verbatim, keeps the workspace-path hint for built-in/default and internally generated prompts, and adds regression coverage for a configured prompt that mentions `HEARTBEAT.md`.
