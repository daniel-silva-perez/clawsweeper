## Summary

Current main still appears to reproduce the reported regression: configured heartbeat prompt text is resolved first, then any selected prompt mentioning HEARTBEAT.md is passed through the workspace-path hint appender with no guard for user-authored config. The provided GitHub context also shows open PR #40756 using closing syntax for this issue, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and resolve it by reviewing and landing PR #40756 or an equivalent maintainer patch. The patch should track whether the resolved plain heartbeat prompt came from user config, skip appendHeartbeatWorkspacePathHint for those user-authored prompts, preserve the hint for built-in/default and internally generated event prompts, and add a focused regression test for a configured prompt that mentions HEARTBEAT.md.
