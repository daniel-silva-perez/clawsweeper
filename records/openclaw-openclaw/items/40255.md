## Summary

Keep open. Current main still resolves user-configured heartbeat prompts and then appends the workspace HEARTBEAT.md path hint to any prompt mentioning HEARTBEAT.md, which conflicts with the docs' verbatim-prompt contract. The issue is also paired with open, unmerged PR #40756 using closing syntax, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to honor user-configured heartbeat prompts by not appending the workspace HEARTBEAT.md path hint to custom prompt text.

## Best Possible Solution

Land PR #40756 or an equivalent narrow fix that tracks whether the base heartbeat prompt came from user configuration, skips the workspace-path hint for user-authored prompts, preserves the hint for built-in/internal prompts, and adds regression coverage for a configured prompt that mentions HEARTBEAT.md.
