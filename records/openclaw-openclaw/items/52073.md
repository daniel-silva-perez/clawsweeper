## Summary

Keep open. Current main has Telegram typing/status/progress/final-delivery scaffolding, but the native ClawHub skill install path still has a real stalled-body timeout gap: it clears the request timeout after headers, then consumes the archive body with `response.arrayBuffer()` without a live abort signal. That can still make an agent-invoked `openclaw skills install` appear hung/unresponsive. The linked PR #52140 was closed unmerged and does not resolve the current issue.

## Best Possible Solution

Keep this issue open. The best fix is to keep an abort timeout alive through ClawHub archive body consumption, either by streaming the response body with a signal-aware pipeline or by passing a live signal through the body read, then add a stalled-mid-stream regression test for `downloadClawHubSkillArchive`/`openclaw skills install`. After that, verify Telegram still emits typing/status/tool progress and final/error delivery while same-chat follow-ups queue behind the active run.
