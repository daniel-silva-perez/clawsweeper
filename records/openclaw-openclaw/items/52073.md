## Summary

Keep open. Current main still leaves the native ClawHub skill archive body read without an active timeout after response headers, so an agent-invoked skill install can still appear hung and leave Telegram waiting for completion. The linked PR #52140 was closed unmerged and does not put a fix on main.

## Best Possible Solution

Keep this issue open. The best fix is to keep an abort timeout alive through ClawHub archive body consumption in `downloadClawHubSkillArchive` or stream the body through a signal-aware path, add a stalled-mid-stream regression test for the native ClawHub skill install path, and then verify Telegram still emits typing/progress plus final or error delivery while same-chat follow-ups queue behind the active run.
