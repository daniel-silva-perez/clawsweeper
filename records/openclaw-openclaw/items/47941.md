## Summary

Keep open. Current main still builds context-report bootstrap file labels directly from `file.name`, and the user-visible `/context` renderer prints that stat name directly. An open unmerged PR, #48021, explicitly targets this issue, so the best maintainer path is to review and land or close that PR before resolving this issue.

## Best Possible Solution

Review #48021 against current main, ensure it adds a runtime-safe label fallback in `buildBootstrapInjectionStats` plus regression coverage for hook-injected path-only virtual bootstrap files, then land it or close it with a clear replacement. After the fix merges, close this issue with the merge commit and release provenance; keep the broader session-memory behavior from #39013 tracked separately.
