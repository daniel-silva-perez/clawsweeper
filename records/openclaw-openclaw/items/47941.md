## Summary

Keep open. Current main still renders bootstrap injection stats from `f.name`, and the stat builder still assigns `name: file.name` without a runtime fallback for hook-injected files that have `path` but no `name`. PR #48021 remains open and unmerged as an explicit implementation candidate for this issue, so the issue should stay open until that PR is reviewed/landed or closed.

## Best Possible Solution

Review PR #48021 against current main, ensure it adds a runtime-safe label fallback plus regression coverage for path-only virtual bootstrap files, then land it or close it with a clear replacement. After the fix is merged, close this issue as implemented with the merge commit and release provenance.
