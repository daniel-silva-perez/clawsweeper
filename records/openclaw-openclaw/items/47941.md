## Summary

Keep this issue open. Current main still lets a hook-provided bootstrap file with a valid path but missing name flow into `buildBootstrapInjectionStats`, where the report name is taken directly from `file.name`; `/context` then renders that name directly. The provided GitHub context also shows an open, unmerged implementation PR that uses closing syntax for this issue, so the right maintainer path is to review and land or close that PR before resolving the issue.

## Best Possible Solution

Review #48021 against current main, ensure it normalizes bootstrap stat labels from non-empty name, path basename, then a stable placeholder, and add a regression test for hook-injected path-only virtual bootstrap files. After that PR or an equivalent fix merges, close this issue with the merge commit and release provenance.
