## Summary

Keep this issue open. Current main and the latest shipped tag still support only a height hint for Control UI `[embed ...]` previews: the parser/type shape has `preferredHeight` but no `preferredWidth`, the renderer passes only height to the iframe, and CSS keeps the preview frame at `width: 100%`. The request is a narrow Control UI enhancement and is suitable for one focused fix PR.

## Best Possible Solution

Implement a bounded `width`/`preferredWidth` presentation hint for Control UI hosted embeds: parse `width="..."` from `[embed ...]` and structured canvas preview payloads where appropriate, carry it through preview/sidebar types, render it with responsive constraints, and update tests, docs, prompt guidance, and changelog with reporter attribution. Keep sandboxing, external URL policy, and non-web channel behavior unchanged.
