## Summary

Keep this issue open. Current main still supports only a height hint for Control UI `[embed ...]` previews: the parser/type shape has `preferredHeight` but no width field, `renderPreviewFrame` only receives/applies height, and the iframe CSS remains `width: 100%`. The request is clear and narrow enough for a focused Control UI fix PR with parser, type, render, docs, prompt, and test updates.

## Best Possible Solution

Implement a bounded `width`/`preferredWidth` presentation hint for Control UI hosted embeds: parse it from `[embed ...]` and structured canvas preview payloads, carry it through the shared preview types, render it in the preview container/frame with responsive constraints, and update tests, docs, prompt guidance, and changelog with attribution. Keep sandbox and external URL behavior unchanged.
