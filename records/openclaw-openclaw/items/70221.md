## Summary

Keep open. Current main still implements hosted Control UI embeds as a height-only path: width is not parsed into the canvas preview contract, not preserved through UI/sidebar normalization, and the iframe CSS still forces previews to the container width.

## What This Changes

Request to add a `width` parameter to the Control UI `[embed]` directive so hosted iframe previews can render at useful desktop mockup widths instead of only the chat column width.

## Best Possible Solution

Add a bounded `preferredWidth` presentation hint end to end for hosted Control UI embeds, including parser/contract/types/sidebar/rendering/docs/prompt/tests, while preserving iframe sandboxing, external URL blocking, and non-web channel behavior.

## Reproduction Assessment

Yes. A static reproduction is `[embed ref="cv_demo" title="Demo" height="600" width="900" /]`: current main parses and applies height, but ignores width before render and then forces the iframe to `width: 100%`.

## Solution Assessment

No, not as the renderer-only sketch in the issue body. The maintainable fix needs a contract-level width field propagated through parser, stored preview shape, UI normalization, sidebar rendering, CSS constraints, prompt/docs, and tests rather than a local inline-style change only.
