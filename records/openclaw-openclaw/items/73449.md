## Summary

Keep open. Current main still lacks the Codex app-server normalized tool stream bridge, so the PR is not obsolete, but the current branch still publishes unsanitized tool args and full-output/update payloads on the live agent-event path. The author addressed the earlier file-diff and collab-agent bulk-output comments, but sanitizer parity with the established Pi tool stream remains a blocking security/correctness issue.

## What This Changes

The PR adds normalized Codex app-server `stream: "tool"` start/update/result projection plus tests for command events, result deduplication, and limited bulky payload gating.

## Best Possible Solution

Keep the Codex tool-stream bridge, but route start args, update chunks, and final result payloads through an SDK-safe sanitizer equivalent to the Pi path before emitting them. Add focused projector tests with secret-bearing command text, MCP/dynamic arguments, and full-output result text while preserving the existing output gate and result deduplication behavior.
