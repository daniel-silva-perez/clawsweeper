## Summary

Keep this PR open. Current main still lacks the Codex app-server normalized tool-stream bridge, and the PR is focused on the right projector/test surface, but the proposed patch is not ready because the new tool events publish raw argument/result payloads instead of matching the Pi tool-stream sanitizer contract.

## What This Changes

The PR adds normalized Codex app-server `stream: "tool"` start/update/result projections plus regression coverage in the Codex app-server event projector tests.

## Best Possible Solution

Land the Codex projector bridge after it reuses or exposes the same sanitizer policy as the Pi tool stream, preserves the existing full-output gate, keeps the deduplication behavior, and adds regression tests for redacted command args, dynamic/MCP args, streamed chunks, and final results.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Redact tool stream args before emitting them:** `extensions/codex/src/app-server/event-projector.ts:592`
  - body: The new Codex `stream: "tool"` start event forwards `buildToolStreamArgs(item)` directly to agent-event consumers. That includes command strings and MCP/dynamic tool arguments; Control UI renders `data.args` into tool cards, while the Pi path runs args through `sanitizeToolArgs` first. A command or tool arg containing an API key would now be exposed even when full output is disabled, so sanitize or reduce this payload before emission.
  - confidence: 0.91
- **[P1] Redact streamed tool results before publishing:** `extensions/codex/src/app-server/event-projector.ts:611-634`
  - body: The new update/result events emit raw `partialResult` chunks and, when full output is enabled, raw command output/MCP result/content items. Existing verbose full output passes through `formatToolProgressOutput`, and the Pi tool stream uses `sanitizeToolResult`; this path bypasses that redaction before it reaches the global event bus and Control UI. Apply the same sanitizer to partial and final tool payloads.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The patch introduces a concrete data-exposure regression by publishing unsanitized Codex tool arguments and results to live agent-event consumers.

Concerns:

- **[high] Unsanitized tool event payloads:** `extensions/codex/src/app-server/event-projector.ts:592`
  - body: The PR does not add dependencies or execution paths, but it adds new live event payloads that can contain command strings, tool arguments, streamed output, MCP results, and image prompts without the Pi tool-stream redaction layer. These events are consumed by Control UI and the global event bus, so secrets or private context can be exposed beyond the existing redacted verbose path.
