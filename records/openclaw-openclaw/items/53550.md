## Summary

Keep open. Current main still leaves the reported gateway-to-DM recall path unresolved because builtin session memory, QMD transcript export, and session-hit visibility are independent controls; no merged fix or narrower closed item supersedes this issue.

## What This Changes

The issue asks for `experimental.sessionMemory` with `sources: ["memory", "sessions"]` to surface gateway-dispatched session transcript content in `memory_search` across sessions and backends.

## Best Possible Solution

Define one explicit contract for session transcript recall: either make `experimental.sessionMemory` plus `sources: ["sessions"]` enable builtin and QMD transcript indexing with clear logs, or keep QMD opt-in separate while surfacing diagnostics/docs that require `memory.qmd.sessions.enabled: true` and a visibility scope such as `tools.sessions.visibility: "agent"` for same-agent gateway-to-DM recall. Add a regression test with a gateway-style session key queried from an unrelated DM key.
