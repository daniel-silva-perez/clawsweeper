## Summary

Keep open. Current main still leaves the reported gateway-to-DM recall path unresolved: `experimental.sessionMemory` only admits the builtin `sessions` source, QMD transcript export is still a separate `memory.qmd.sessions.enabled` switch, and session-sourced `memory_search` hits are filtered through session-tool visibility whose default `tree` scope hides unrelated same-agent gateway sessions from a DM requester.

## What This Changes

The issue requests that `memory_search` surface gateway-dispatched session transcript content when `agents.defaults.memorySearch.experimental.sessionMemory` is enabled with `sources: ["memory", "sessions"]`, including QMD/Gemini-backed setups.

## Best Possible Solution

Define one explicit transcript recall contract: either make `experimental.sessionMemory` plus `sources: ["sessions"]` enable builtin and QMD session indexing with clear diagnostics, or keep QMD opt-in separate while surfacing actionable status/docs that require `memory.qmd.sessions.enabled: true` and an intentional visibility setting such as `tools.sessions.visibility: "agent"`. Add regression coverage for a gateway-style same-agent session queried from an unrelated DM session.
