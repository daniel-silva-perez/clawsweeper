## Summary

Keep open. Current main still leaves the reported gateway-to-DM recall path unresolved: agent-level `experimental.sessionMemory` enables the builtin `sessions` source, but QMD transcript export is still controlled by a separate `memory.qmd.sessions.enabled` switch, and any session-sourced `memory_search` hits are then filtered through session-tool visibility whose default `tree` scope hides unrelated same-agent gateway sessions from a DM requester. The related #51386 umbrella is relevant, but this issue remains a concrete, narrower reproduction rather than something already implemented or safely superseded.

## What This Changes

The issue requests that `memory_search` surface gateway-dispatched session transcripts across sessions when `agents.defaults.memorySearch.experimental.sessionMemory` and `sources: ["memory", "sessions"]` are enabled, including QMD/Gemini/builtin behavior, logs, and docs.

## Best Possible Solution

Define one explicit session transcript recall contract: either make `experimental.sessionMemory` plus `sources: ["sessions"]` enable both builtin and QMD transcript indexing with observable diagnostics, or keep QMD opt-in separate while surfacing clear docs/status guidance requiring `memory.qmd.sessions.enabled: true` and an intentional visibility setting such as `tools.sessions.visibility: "agent"`. Add regression coverage for a gateway-style same-agent session queried from an unrelated DM session.
