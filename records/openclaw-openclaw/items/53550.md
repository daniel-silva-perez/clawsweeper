## Summary

Keep open. Current main still leaves the reported gateway-to-DM recall path unresolved: `experimental.sessionMemory` only gates the builtin `sessions` source, QMD transcript export remains a separate `memory.qmd.sessions.enabled` switch, and session-sourced `memory_search` hits are filtered through session history visibility whose default `tree` setting hides unrelated same-agent gateway sessions from a DM requester.

## What This Changes

The issue requests that `experimental.sessionMemory` make gateway-dispatched session transcripts discoverable through `memory_search` across sessions, including QMD/Gemini and builtin fallback paths.

## Best Possible Solution

Define one explicit contract for session transcript recall: either make `experimental.sessionMemory` plus `sources: ["sessions"]` enable both builtin and QMD transcript indexing with observable logs, or keep QMD opt-in separate while surfacing clear diagnostics and docs that require `memory.qmd.sessions.enabled: true` plus an intentional visibility setting such as `tools.sessions.visibility: "agent"` for same-agent gateway-to-DM recall. Add regression coverage with a gateway-style same-agent session key queried from an unrelated DM key.
