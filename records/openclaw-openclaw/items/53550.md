## Summary

Keep open. Current main still leaves the reported gateway-to-DM recall path unresolved: the agent-level `experimental.sessionMemory` flag enables the builtin sessions source, but QMD transcript export is still controlled by a separate `memory.qmd.sessions.enabled` switch, and any session hits that are indexed are filtered through session-tool visibility whose default `tree` scope hides unrelated same-agent gateway sessions from a DM requester.

## What This Changes

The issue requests that `agents.defaults.memorySearch.experimental.sessionMemory` with `sources: ["memory", "sessions"]` make `memory_search` surface gateway-dispatched session transcript content across same-agent sessions, including QMD/Gemini-backed setups.

## Best Possible Solution

Define one explicit transcript recall contract: either make `experimental.sessionMemory` plus `sources: ["sessions"]` enable and diagnose both builtin and QMD session indexing, or keep QMD opt-in separate while surfacing clear docs/status guidance that requires `memory.qmd.sessions.enabled: true` and an intentional visibility setting such as `tools.sessions.visibility: "agent"` for same-agent gateway-to-DM recall. Add regression coverage for a gateway-style same-agent session queried from an unrelated DM session.
