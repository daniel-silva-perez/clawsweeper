## Summary

Keep open. Current main still does not make `agents.defaults.memorySearch.experimental.sessionMemory` surface unrelated gateway-dispatched same-agent sessions through QMD-backed `memory_search` by itself. The previous Codex review reached the same conclusion, and current `68561a8c` still has the same split: builtin session sources are controlled by `memorySearch.experimental.sessionMemory`, QMD transcript export is separately controlled by `memory.qmd.sessions.enabled`, and session hits are filtered through session-tool visibility, whose default `tree` policy hides unrelated same-agent gateway sessions from a DM. Related #51386 and #59933 remain open for the broader session-memory and same-agent visibility defaults, but neither supersedes this concrete QMD/gateway recall gap.

## Best Possible Solution

Keep this issue open. The maintainer path is to either bridge `agents.defaults.memorySearch.experimental.sessionMemory` plus `sources: ["sessions"]` into QMD transcript export when `memory.backend` is `"qmd"`, or explicitly diagnose and document that QMD transcript recall requires `memory.qmd.sessions.enabled: true` plus broader session visibility such as `tools.sessions.visibility: "agent"` or `"all"`. Add a regression test with a gateway-style same-agent session key queried from an unrelated DM key so the intended gateway-to-DM recall contract is explicit.

## Risks / Open Questions

- Automatically surfacing all session transcripts in `memory_search` could expose unrelated channel, user, or cross-agent conversations unless the current conservative visibility defaults remain explicit.
- Bridging `experimental.sessionMemory` into QMD export could surprise users by writing sanitized transcript Markdown for QMD indexing when they only expected the builtin experimental source; docs, diagnostics, and config migration need to be explicit.
- This was a read-only source/docs/history review, not a live QMD/Gemini/gateway reproduction, but the unresolved behavior follows directly from current implementation and tests.
