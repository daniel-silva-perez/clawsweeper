## Summary

Keep open. Current main still leaves the reported gateway-to-DM recall path unresolved: `experimental.sessionMemory` enables the builtin session source, but QMD transcript export remains a separate `memory.qmd.sessions.enabled` switch, and session-sourced `memory_search` hits are still filtered through session-tool visibility, whose default `tree` setting hides unrelated same-agent gateway sessions from a DM session.

## What This Changes

The issue asks for `memory_search` to index and return gateway-dispatched session transcripts when session memory is enabled, including QMD/Gemini and builtin memory paths.

## Best Possible Solution

Define the intended contract explicitly: either make QMD honor `experimental.sessionMemory` plus `sources: ["sessions"]` by enabling/exporting session transcript collections with clear logs, or keep QMD opt-in separate but surface an actionable diagnostic/docs path requiring `memory.qmd.sessions.enabled: true` and a visibility setting such as `tools.sessions.visibility: "agent"` for same-agent gateway-to-DM recall. Add a regression test with a gateway-style same-agent session key queried from an unrelated DM key.
