## Summary

Keep open. Current main has adjacent malformed streamed tool-call argument repair for Kimi/openai-completions, but the reported path is Amazon Bedrock, whose documented API is `bedrock-converse-stream`. The Bedrock path still depends on pi-ai parsing `toolUse.input`; in the pinned pi-ai 0.70.5 source that parser initializes tool calls with `{}` and returns `{}` for empty or unparseable partial JSON, so the reported large-content empty-argument failure is still plausible and not directly covered by current tests.

## Best Possible Solution

Keep this issue open and add a Bedrock Converse-specific fix: detect empty or malformed `toolUse.input` for parameterized tool calls before dispatch, either repair the streamed argument buffer for `bedrock-converse-stream` or fail closed with a clear provider-stream/truncation error and bounded retry behavior. Add fake Bedrock stream coverage for large `write` content and `exec` heredoc-style arguments, plus a guard that parameterless tools are not blocked.
