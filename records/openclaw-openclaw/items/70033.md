## Summary

Keep open. Current main still enables malformed streamed tool-call argument repair only for Kimi over anthropic-messages and openai-completions, while the reported Amazon Bedrock path uses bedrock-converse-stream. The pinned pi-ai Bedrock parser still initializes streamed tool-call arguments to {} and re-parses empty or malformed toolUse.input as {}, so large write/exec failures can still reach tool validation as empty arguments.

## Best Possible Solution

Keep this issue open and implement a focused Bedrock Converse fix in the embedded runner/provider stream path: repair malformed streamed toolUse.input where possible, and fail closed with a clear bounded retry/error when a parameterized tool reaches dispatch with empty or malformed arguments. Add fake Bedrock stream coverage for large write content and exec command/heredoc-style arguments, while preserving parameterless tools and the existing openai-completions repair behavior.
