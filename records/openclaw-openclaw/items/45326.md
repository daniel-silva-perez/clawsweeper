## Summary

Current main still leaves the reported TUI normal-submit path unguarded during an active run. The submit handler clears the editor before routing, the production TUI passes no busy predicate into that handler, and `sendMessage` still sends normal chat input without rejecting `state.activeChatRunId`. Adjacent pending-send/history fixes are present, but they do not implement the requested active-generation submit block or draft preservation.

## Best Possible Solution

Keep this issue open and implement a TUI-side normal-message submit guard before `editor.setText("")`. When `state.activeChatRunId` or an optimistic normal send is pending, preserve the draft, show a short status/system hint such as pressing Esc or `/abort` to cancel, and avoid calling `sendMessage`. Keep explicit slash-command and bang-line behavior intentionally defined and covered by tests.
