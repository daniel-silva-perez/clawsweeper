## Summary

Keep open. Current main still has the core TUI behavior reported here: normal Enter submits clear the editor before acceptance, the submit path is wired without any active-run guard, and `sendMessage` does not reject sends while `state.activeChatRunId` is already set. The related fix PR #45586 was closed unmerged, and the current tests do not cover the proposed active-run draft-preservation regression.

## Best Possible Solution

Keep the issue open and land a focused TUI fix: block normal chat submits while `state.activeChatRunId` or pending send state is active, preserve the editor draft when blocked or rejected, show a concise busy/abort status, keep intentional slash-command and local-shell behavior explicit, and add regression coverage for active-run submit, draft preservation, and abort-then-send behavior. The closed unmerged #45586 can be used as context, but the current main still needs an accepted implementation.
