## Summary

Keep this issue open. Current main still does not expose `moonshotai/kimi-k2.5` as a bundled OpenRouter onboarding/model-picker catalog row: the OpenRouter static catalog, docs, and focused tests only include `openrouter/auto` and `moonshotai/kimi-k2.6`. Dynamic OpenRouter model resolution can still run arbitrary model IDs manually, but that does not satisfy the selector/catalog request. OpenRouter's public models API currently still lists `moonshotai/kimi-k2.5`, so this remains a narrow valid catalog/docs/test follow-up.

## Best Possible Solution

Keep this issue open and make a narrow OpenRouter plugin/catalog update: add `moonshotai/kimi-k2.5` as a bundled selector row with current OpenRouter metadata, keep `openrouter/auto` as the onboarding default, update focused OpenRouter catalog tests and `docs/providers/openrouter.md`, and add a user-facing changelog entry if the catalog change is accepted. If maintainers decide K2.6 supersedes K2.5 for curated OpenRouter picker rows, document that explicitly instead.
