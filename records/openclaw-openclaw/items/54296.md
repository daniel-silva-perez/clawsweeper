## Summary

Keep this open. Current main has the exec-approvals atomic replacement hardening, but the transcript half of the report is still current across gateway/session paths that read whole JSONL files before limiting, scanning, compaction, or usage extraction. The prior ClawSweeper comment’s split assessment still matches current main.

## Best Possible Solution

Keep this issue open. Preserve the shipped exec-approvals atomic write behavior, then land a focused core fix that introduces shared streaming or bounded JSONL transcript scan/tail helpers and applies them to idempotency checks, latest-assistant/equivalent-message scans, history/sequence reads, usage extraction, maxLines compaction, and related external transcript imports, with regression coverage for large transcripts and malformed lines.
