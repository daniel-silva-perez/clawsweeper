## Summary

Keep open. Current main has fixed the exec-approvals atomic-write half, but the transcript memory-pressure half remains: several gateway/session helpers still read complete transcript files before idempotency scans, latest-assistant/equivalence checks, message history responses, sequence calculation, and usage extraction.

## Best Possible Solution

Keep this issue open. Preserve the shipped exec-approvals atomic write, then land a focused core fix that introduces shared streaming or bounded JSONL transcript scan/tail helpers and applies them to idempotency checks, latest-assistant/equivalent-message scans, history/sequence reads, and usage extraction, with large-transcript and malformed-line regression coverage.
