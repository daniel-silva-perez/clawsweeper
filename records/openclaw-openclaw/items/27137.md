## Summary

Current main still lacks the requested unified `authorizeIngress()` / `IngressContext` gate. The repo has useful shared DM/group primitives and several hardened per-channel helpers, but authorization remains distributed across channel and event surfaces, and the maintainer comment already kept this security/design RFC open.

## What This Changes

Request to add a unified ingress authorization gate that centralizes channel/event authorization for messages, reactions, pins, interactions, callbacks, slash commands, and voice ingress.

## Best Possible Solution

Keep this open for maintainer security design. The best path is an additive, versioned core/security or SDK-backed `authorizeIngress`-style contract that accepts normalized channel, account, event kind, sender identity, chat context, configured/store allowlists, command surface, and standardized denial reasons, then migrates representative message/reaction/pin/interaction/callback/slash/voice paths with contract tests proving new ingress surfaces cannot bypass the shared gate.
