## Summary

Current main still does not contain the requested unified `authorizeIngress()` contract. The repository has useful shared policy primitives and several channel-local hardening paths, but authorization is still assembled per channel and per event type, so this remains a valid security architecture RFC rather than a cleanup-close candidate.

## What This Changes

Request to add a unified ingress authorization gate that centralizes DM/group allowlist and policy decisions while letting channel adapters provide sender and context normalization.

## Best Possible Solution

Keep this RFC open until maintainers approve an additive security/SDK contract for ingress authorization. The target end state should centralize policy decisions while preserving channel-specific identity adapters, then migrate representative message, reaction, pin, interaction, callback, slash, voice, and media ingress paths with contract tests.
