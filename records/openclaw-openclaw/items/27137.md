## Summary

Current main still does not implement the requested unified `authorizeIngress()` / `IngressContext` gate. The existing shared helpers centralize DM/group allowlist policy, but channel/event authorization remains distributed across Slack, Discord, WhatsApp, LINE, direct-DM, and other ingress paths. The prior maintainer comment also kept this open for design follow-up; #58245 is a narrow Discord voice fix and #49782 is unrelated ACP work, so neither supersedes the RFC.

## Best Possible Solution

Keep this open for maintainer security design. The best path is an additive, versioned core/security or SDK-backed `authorizeIngress`-style contract that accepts normalized adapter inputs for channel, account, event kind, sender identity, chat context, configured/store allowlists, command surface, and standardized reason codes, then migrates representative message, reaction, pin, interaction, callback, slash, and voice paths with contract tests proving new ingress surfaces cannot bypass the shared gate.
