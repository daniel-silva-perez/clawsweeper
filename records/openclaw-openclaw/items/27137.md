## Summary

Keep open. Current main still has shared lower-level DM/group policy primitives and several per-channel hardening helpers, but it does not contain the requested unified cross-channel ingress authorization contract. The existing maintainer comment also explicitly keeps the RFC open for security/design follow-up, and the later current-main helpers are config/DM access consolidation rather than the proposed ingress gate.

## What This Changes

Request to add a unified cross-channel ingress authorization gate that centralizes DM/group allowlist and policy evaluation while channel adapters provide identity, account, event-kind, and context inputs.

## Best Possible Solution

Keep the RFC open until maintainers approve an additive ingress authorization contract, then centralize policy evaluation behind that contract while preserving channel-specific identity/context adapters and migrating representative message, reaction, pin, interaction, callback, slash, voice, and media paths with contract tests.
