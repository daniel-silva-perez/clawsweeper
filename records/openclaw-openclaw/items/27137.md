## Summary

Keep open. Current main still lacks the requested unified ingress authorization gate, and representative Slack, Discord, LINE, WhatsApp, Telegram, Signal, and iMessage paths continue to make lower-level or channel-local authorization decisions. The existing maintainer discussion also treats this as a security/design RFC, so this is not a conservative cleanup-close candidate.

## What This Changes

The issue requests a unified cross-channel `authorizeIngress()` authorization contract to replace repeated per-channel and per-event ingress authorization logic.

## Best Possible Solution

Keep the RFC open until maintainers approve an additive ingress authorization contract. The target end state should centralize policy evaluation behind a shared gate, keep channel-specific identity/context adapters local, and migrate representative message, reaction, pin, interaction, callback, slash, voice, and media ingress paths with contract and bundled-plugin tests.
