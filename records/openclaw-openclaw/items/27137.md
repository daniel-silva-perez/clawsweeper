## Summary

Keep open. Current main has shared DM/group policy, direct-DM, and command-gating helpers, but it still does not implement the requested unified `authorizeIngress()`/`IngressContext` gate across channel, account, event kind, sender identity, chat context, allowlist/store reads, command authorization, and standardized reason codes. The existing maintainer-posted Codex review also explicitly kept this issue open, and the only related item in the provided context, #49782, is an unrelated ACP completion-relay RFC.

## Best Possible Solution

Keep this open for maintainer design. The best path is a narrow core/security or SDK-backed ingress authorization contract that accepts normalized adapter inputs for channel, account, event kind, sender identity, chat context, configured/store allowlists, and command surface; centralizes DM/group policy, allowlist merging, pairing-store reads, command authorization, and reason codes; then migrates representative message, reaction, pin, interaction, callback, slash, and voice paths with contract tests proving new ingress surfaces cannot bypass the shared gate.
