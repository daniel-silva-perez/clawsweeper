## Summary

Keep open. Current main still lacks the requested cross-channel `authorizeIngress()` contract, and representative ingress paths continue to authorize through lower-level shared helpers or channel-local gates. The existing maintainer discussion also treats this as a security/design RFC, so cleanup closure would be premature.

## What This Changes

The issue requests an RFC-style shared `authorizeIngress()` gate that centralizes authorization decisions across channel and event ingress paths while leaving identity and context adaptation to channels.

## Best Possible Solution

Keep the RFC open until maintainers and secops approve an additive shared ingress authorization contract. The target end state should centralize policy decisions in a shared security or SDK seam, preserve channel-specific identity/context adapters, and migrate representative message, reaction, pin, interaction, callback, slash, voice, and media paths with contract tests.
