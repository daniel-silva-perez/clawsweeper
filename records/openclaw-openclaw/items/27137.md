## Summary

Keep open. Current main has shared DM/group policy, direct-DM, and command-gating helpers, but it still does not implement the requested unified authorizeIngress()/IngressContext gate; channel and event authorization remains distributed, and the maintainer-posted Codex review already kept this issue open for design follow-up.

## Best Possible Solution

Keep this open for maintainer security design. The best path is an additive core/security or SDK-backed authorizeIngress-style contract that accepts normalized adapter inputs for channel, account, event kind, sender identity, chat context, configured/store allowlists, command surface, and standardized reason codes; then migrate representative message, reaction, pin, interaction, callback, slash, and voice paths with contract tests proving new ingress surfaces cannot bypass the shared gate.
