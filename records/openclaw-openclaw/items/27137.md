## Summary

Keep open. Current main still lacks the requested unified `authorizeIngress()` contract, and representative Slack, Discord, LINE, WhatsApp, Signal, and iMessage paths continue to assemble authorization through lower-level helpers or channel-local code. The existing maintainer comment also treats this as a security/design RFC rather than a cleanup-close candidate.

## What This Changes

The issue requests a unified cross-channel ingress authorization gate that centralizes policy decisions currently assembled per channel and event type.

## Best Possible Solution

Keep the RFC open until maintainers approve an additive ingress authorization contract, then centralize policy evaluation behind that contract while preserving channel-specific identity/context adapters and migrating representative message, reaction, pin, interaction, callback, slash, voice, and media paths with contract tests.
