## Summary

Current main still does not implement the requested unified `authorizeIngress()` / `IngressContext` gate. The repository has shared DM/group policy primitives and several channel-local authorization helpers, but ingress authorization remains distributed across channel and event surfaces. The provided maintainer discussion also explicitly kept this security/design RFC open, and the related Discord voice fix is narrow rather than superseding the RFC.

## Best Possible Solution

Keep this open for maintainer security design. The best path is an additive, versioned core/security or SDK-backed `authorizeIngress`-style contract that accepts normalized adapter inputs for channel, account, event kind, sender identity, chat context, configured/store allowlists, command surface, and standardized denial reasons, then migrates representative message, reaction, pin, interaction, callback, slash, and voice paths with contract tests proving new ingress surfaces cannot bypass the shared gate.
