## Summary

Keep open. Current main still supports explicit WhatsApp outbound targets, but it does not provide a trusted contact-name or alias resolution path for a prompt target like “Jack” from another approved WhatsApp DM; the prior ClawSweeper comment remains consistent with current source.

## What This Changes

The issue asks for a trusted WhatsApp direct contact to relay a new outbound message by natural-language contact name to another approved WhatsApp contact.

## Best Possible Solution

Keep this open until the WhatsApp plugin has a trustworthy configured alias/contact metadata path and regression coverage showing an authorized direct sender can relay by name to another approved direct WhatsApp target without changing the generic core cross-context policy unless a missing plugin API is proven.
