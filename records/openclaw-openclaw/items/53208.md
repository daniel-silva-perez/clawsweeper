## Summary

Keep open. Current main still handles explicit WhatsApp targets and allows same-provider cross-context sends, but it still has no configured WhatsApp contact-name or alias resolver for a bare prompt target like “Jack” from another approved DM. The report remains distinct from the related closed explicit-target WhatsApp fixes and needs maintainer product/security direction before changing outbound name-based delivery.

## What This Changes

Allow an approved WhatsApp direct-message contact to ask OpenClaw to relay a message by another contact’s name to an approved WhatsApp recipient instead of being told that recipient is unavailable.

## Best Possible Solution

Add a WhatsApp-owned, configured contact alias or metadata path that resolves approved names to approved direct WhatsApp targets, rejects ambiguous or unauthorized names, and has regression coverage for an authorized DM sender relaying to another approved target without weakening the generic core cross-context policy.
