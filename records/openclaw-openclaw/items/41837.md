## Summary

Keep this issue open. Current main still lacks Mattermost WebSocket ping/pong liveness or provider-proven transport activity reporting, so the generic health monitor cannot detect the connected-but-deaf Mattermost socket described here. The related open PR #57621 is still only an implementation candidate, not shipped behavior on main.

## Best Possible Solution

Keep this open until PR #57621 or an equivalent Mattermost-owned fix lands. The fix should add WebSocket ping/pong keepalive with missed-pong termination, report lastTransportActivityAt only from real transport evidence if the generic health monitor should participate, and add focused regression tests for connected-but-deaf WebSocket recovery. After merge, maintainers can close this as implemented and consolidate duplicate Mattermost liveness reports.
