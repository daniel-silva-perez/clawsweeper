## Summary

Keep open. Current main still does not implement Mattermost transport liveness for a connected-but-deaf WebSocket. Mattermost is a bundled plugin, the WebSocket wrapper has only passive open/message/close/error handling plus bot update_at polling, and the generic gateway health monitor only performs stale-socket recovery when a channel reports lastTransportActivityAt. The supplied related context shows #57621 is still open/unmerged, so it is an implementation candidate rather than shipped behavior.

## Best Possible Solution

Keep this issue open and review/land #57621 or an equivalent Mattermost-owned fix. The likely implementation belongs in extensions/mattermost/src/mattermost/monitor-websocket.ts: add WebSocket ping/pong keepalive or another active liveness check that terminates deaf sockets so the existing reconnect loop runs, and add focused regression tests for connected-but-not-delivering recovery.
