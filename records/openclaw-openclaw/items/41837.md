## Summary

Keep this issue open. Current main still does not implement Mattermost WebSocket ping/pong liveness or provider-proven transport activity reporting, so the generic health monitor cannot detect a connected-but-deaf Mattermost socket. The supplied context also shows open PR #57621 is still only an implementation candidate, not shipped behavior.

## Best Possible Solution

Keep this open until PR #57621 or an equivalent Mattermost-owned fix lands. The fix should add provider-proven WebSocket liveness, preferably ping/pong with missed-pong termination, publish lastTransportActivityAt only from real transport evidence if the generic health monitor should participate, and add focused regression tests for connected-but-deaf WebSocket recovery. After merge, maintainers can close this as implemented or consolidate the duplicate Mattermost liveness reports.
