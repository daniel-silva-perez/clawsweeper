## Summary

Keep this issue open. Current main still lacks Mattermost WebSocket ping/pong liveness or another provider-proven transport activity signal for the connected-but-deaf socket described here. The existing Mattermost watchdog only detects bot `update_at` changes, and the related PR #57621 is still an open implementation candidate rather than shipped behavior.

## Best Possible Solution

Keep this open until PR #57621 or an equivalent Mattermost-owned fix lands. The fix should add WebSocket ping/pong keepalive with missed-pong termination in `extensions/mattermost/src/mattermost/monitor-websocket.ts`, publish real `lastTransportActivityAt` only from transport evidence if the generic health monitor should participate, and add focused regression tests for connected-but-deaf WebSocket recovery. After merge, maintainers can close this as implemented and consolidate the related Mattermost liveness reports.
