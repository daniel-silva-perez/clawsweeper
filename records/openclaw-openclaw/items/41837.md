## Summary

Current main still does not implement Mattermost WebSocket ping/pong liveness or publish provider-proven transport activity for the connected-but-deaf socket described here. The shipped Mattermost watchdog only detects bot account `update_at` changes, and the related PR #57621 is still an open implementation candidate rather than merged behavior.

## Best Possible Solution

Keep this open until PR #57621 or an equivalent Mattermost-owned fix lands. The fix should add WebSocket ping/pong keepalive with missed-pong termination in `extensions/mattermost/src/mattermost/monitor-websocket.ts`, publish `lastTransportActivityAt` only from real transport evidence if the generic health monitor participates, and add focused regression tests for connected-but-deaf WebSocket recovery.
