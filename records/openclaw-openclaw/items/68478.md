## Summary

Keep this issue open. Current main has partial noise controls, especially exec lifecycle suppression and task notification policy, but it still does not implement the requested broad default-silent or configurable suppression policy for routine chat confirmations across skill/tool commands, cron delivery, install acknowledgements, and generic done replies.

## Best Possible Solution

Keep this issue open and design a coherent notification/confirmation policy, likely under `messages` or `commands`, that can suppress or group routine action confirmations across slash-command skill/tool dispatch, cron/system-event delivery, install/update acknowledgements, and generic done replies. Reuse existing narrow controls such as `tools.exec.notifyOnExit`, cron `delivery.mode`, task notify policy, heartbeat visibility, and `NO_REPLY` delivery suppression, then add targeted Telegram/Discord plus command-dispatch regression coverage.
