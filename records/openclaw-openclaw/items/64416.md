## Summary

Keep open. Current main still does not implement this PR's ACP completion-normalization behavior, and overlapping open PR #65242 by the same author is still an active implementation candidate for the same duplicate ACP completion-delivery surface. The PR also needs maintainer review for security and durability concerns before it could be merged as-is.

## Best Possible Solution

Keep this PR open for maintainer review or consolidation with #65242. The best path is to rebase on current main, choose one durable CompletionDeliveryGate or silent-wake design, persist any task metadata it depends on, keep untrusted completion text inside escaped structured internal events rather than the steer body, and add targeted tests for ACP prompt normalization, live-output preference, restart-safe delivery, and duplicate suppression.
