## Summary

Keep open. Current `main` still exposes and forwards the DAVE-disabled Discord voice join path, waits a fixed 15 seconds for `@discordjs/voice` to reach Ready, and returns the same abort-shaped failure if the voice WebSocket closes before Ready; I did not find a shipped or main-only fix for the specific after-Hello/before-Ready failure described here.

## What This Changes

The issue reports that Discord `/vc join` aborts after the voice WebSocket Hello and before Ready/UDP handshaking when `channels.discord.voice.daveEncryption` is disabled.

## Best Possible Solution

Keep this open until the Discord voice owner decides the canonical behavior for DAVE-disabled joins, then either implement a targeted join-handshake fix with tests and docs or explicitly document that current Discord voice requires DAVE enabled and deduplicate the overlapping open reports against that decision.
