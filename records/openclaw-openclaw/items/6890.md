## Summary

Keep open. Current main still lacks a Ralph Loop feature and still has no per-agent `loop.max_ralph_iterations` or equivalent iteration-limit config surface. Existing loop-related code is tool-loop safety detection or strict-agentic plan-only retry behavior, not a STOP-driven repeated same-prompt loop with the requested CLI > agent > global precedence.

## Best Possible Solution

Keep this issue open for maintainer product judgment. If accepted, add an OpenClaw-native loop/default config surface to `AgentConfig`, strict zod schemas, generated config/help docs, and `resolveAgentConfig`, then wire the embedded runner with explicit precedence against higher-priority runtime or CLI overrides. Add focused tests for global default, per-agent override, override precedence, disabled, finite, and unlimited behavior. If maintainers decide this belongs outside core, first define the plugin/runtime API needed to express the loop safely.
