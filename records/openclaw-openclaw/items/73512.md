## Summary

Keep this PR open. Current main does not implement the proposed behavior, and the linked bug remains a plausible implementation target. The PR needs maintainer review/revision before landing because the provided diff conflicts with an existing regression test and current docs that define QMD `searchMode: "search"` as lexical-only with embedding maintenance skipped.

## Best Possible Solution

Keep this PR open for normal review. If maintainers want explicit `memory.qmd.update.embedInterval` to opt lexical QMD setups into embedding, the implementation should preserve whether the field was explicitly configured, update both `shouldRunEmbed()` and `shouldScheduleEmbedTimer()`, replace or split the existing lexical-mode skip test, and update the QMD memory docs. If maintainers instead want `searchMode: "search"` to remain strictly BM25-only, the linked bug and this PR should be closed with that documented behavior explained.
