## Summary

Keep this PR open. Current main still does not implement the proposed lexical-mode embedding behavior, so the PR is not obsolete, but the patch is not merge-ready because it treats a resolved default `embedIntervalMs` as an explicit user signal and conflicts with current tests and docs that define `searchMode: "search"` as BM25-only.

## What This Changes

The PR changes QMD embed execution and timer scheduling guards so a positive `embedIntervalMs` bypasses the vector-search-mode check, and adds a regression test for `searchMode: "search"` with `embedInterval: "5m"`.

## Best Possible Solution

If maintainers want explicit `memory.qmd.update.embedInterval` to opt lexical QMD setups into embedding, the implementation should carry an explicit-config flag through QMD config resolution, update both runtime guards around that flag, replace or split the lexical-mode tests, and update the memory config docs. If lexical `searchMode: "search"` should remain strictly BM25-only, close the linked bug and this PR with that documented behavior called out.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Track explicit embedInterval before bypassing search mode:** `extensions/memory-core/src/memory/qmd-manager.ts:1620`
  - body: The new guard uses resolved `embedIntervalMs` as if it proves the user configured `embedInterval`, but `resolveEmbedIntervalMs(undefined)` returns the default 60 minute value. That makes default lexical QMD configs eligible to run `qmd embed` on forced or boot sync, and can schedule an embed timer when only `interval` is disabled. Preserve an explicit flag or raw-config check before bypassing `qmdUsesVectors`.
  - confidence: 0.92
- **[P2] Reconcile the opposite lexical-mode test:** `extensions/memory-core/src/memory/qmd-manager.test.ts:3609`
  - body: This added test expects `update` and `embed` for `searchMode: "search"` with `embedInterval: "5m"`, but the existing nearby test still uses the same effective setup and expects no calls. The targeted suite will have contradictory expectations until the old lexical-mode contract test is replaced, split, or updated with an explicit/default distinction.
  - confidence: 0.9
