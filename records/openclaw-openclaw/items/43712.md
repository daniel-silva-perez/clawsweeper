## Summary

Keep this issue open. A fresh read-only pass against current main confirms the prior ClawSweeper assessment still applies: explicit git-to-package channel switching has improved, but stored-channel and direct git update paths can still build in the same checkout while the A2UI bundler still neither cleans nor rejects ignored stale renderer output before compiling.

## What This Changes

The issue requests that in-place git updates from an existing Linux source checkout under a global node_modules path either succeed or detect and clean stale generated A2UI build state instead of failing during `pnpm build`.

## Best Possible Solution

Add a small source-build guard around the A2UI renderer compile: clean or fail with an actionable message when stale `vendor/a2ui/renderers/lit/dist` exists before bundling, cover that behavior with a focused script regression test, and document the recovery path for source checkout updates.
