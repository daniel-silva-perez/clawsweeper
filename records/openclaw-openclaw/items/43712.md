## Summary

Keep this issue open. Current main still permits the reported in-place source-update failure class: ignored A2UI renderer output can survive the git dirty check, then `pnpm build` invokes the A2UI renderer compile in the same checkout without cleaning or rejecting stale `dist` output first.

## What This Changes

The issue reports that updating an existing Linux git/source install to `v2026.3.11` failed during A2UI build in the live checkout while the same tag built successfully from a clean checkout on the same host.

## Best Possible Solution

Add a focused A2UI source-build guard that removes stale ignored renderer output before compilation when safe, or fails with a precise recovery message when cleanup cannot proceed, and document the source-checkout recovery path for in-place updates.

## Reproduction Assessment

Yes. A high-confidence synthetic reproduction path is an existing git checkout with ignored `vendor/a2ui/renderers/lit/dist` output: it can pass the updater dirty check and still reach the A2UI renderer compile during `pnpm build`.

## Solution Assessment

Yes. The narrowest maintainable solution is in the A2UI source-build path, with docs for recovery; changing the updater's whole install-mode strategy would be broader than the observed failure requires.
