## Summary

Keep this issue open. Current main mitigates one explicit CLI path by switching a requested git install to package mode for `openclaw update --channel stable`, but the underlying in-place git update/build path still exists and still can carry ignored stale A2UI renderer output into `pnpm build`. The original report includes concrete logs and a clean-checkout comparison, so this remains actionable rather than stale or unreproducible.

## Best Possible Solution

Keep this issue open and fix the source-update/A2UI edge case directly. The most targeted path is to clean or explicitly reject stale `vendor/a2ui/renderers/lit/dist` before `canvas:a2ui:bundle` runs in in-place git builds, add regression coverage for ignored renderer output causing TS5055-style collisions, and document a recovery step for source checkouts in the public updating docs.
