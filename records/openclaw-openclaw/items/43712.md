## Summary

Keep this issue open. Current main still allows the reported in-place source-update failure class: ignored A2UI renderer output can survive in an existing checkout, the updater builds in that same git root after checkout, and the A2UI bundle script runs the renderer TypeScript compile without first cleaning or rejecting stale `dist` output.

## What This Changes

The issue asks OpenClaw to make git-checkout updates succeed, or clearly detect and recover from stale/generated build state, when updating an existing source install under `/usr/lib/node_modules/openclaw`.

## Best Possible Solution

Add a focused A2UI source-build guard that removes stale ignored renderer output before compilation when safe, or fails with a precise recovery message when cleanup cannot proceed, then document the source-checkout recovery path for in-place updates.

## Reproduction Assessment

Yes. A high-confidence synthetic path is an existing git checkout with ignored `vendor/a2ui/renderers/lit/dist` output: it can pass the updater dirty check and still reach the A2UI renderer compile during `pnpm build`.

## Solution Assessment

Yes. The narrowest maintainable fix is in the A2UI source-build path, with update docs for recovery; changing the whole updater install-mode strategy would be broader than the observed failure requires.
