## Summary

Keep this open. Current main still allows the reported in-place source-update failure class: ignored A2UI renderer output can survive the updater clean check, then the updater runs the full build in the same checkout and the A2UI bundler invokes the renderer TypeScript build without cleaning that stale output first.

## What This Changes

The issue reports source-update/build failures when updating an in-place Linux git install under `/usr/lib/node_modules/openclaw` from `2026.3.8` to `v2026.3.11`, while a clean checkout of the same tag builds successfully.

## Best Possible Solution

Add a focused A2UI source-build guard that removes stale ignored renderer output before compilation when safe, or fails with a precise recovery message when cleanup cannot proceed, and document the source-checkout recovery path for in-place updates.

## Reproduction Assessment

Yes. A high-confidence synthetic reproduction path is an existing git checkout with ignored `vendor/a2ui/renderers/lit/dist` output: it can pass the updater clean check and still reach the A2UI renderer compile during `pnpm build`.

## Solution Assessment

Yes. The narrowest maintainable fix is in the A2UI source-build path with recovery docs; changing the updater's broader install-mode strategy would be larger than the observed failure requires.
