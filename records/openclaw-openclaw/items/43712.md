## Summary

Keep this open. Current main still permits the reported in-place source-update failure class: ignored A2UI renderer output can survive the updater dirty check, then `pnpm build` invokes the A2UI renderer compile in the same checkout without cleaning or rejecting stale `dist` output first.

## What This Changes

The issue asks OpenClaw to make in-place git/source updates under an existing `/usr/lib/node_modules/openclaw`-style checkout succeed, or detect and recover from stale generated A2UI build state before `pnpm build` fails.

## Best Possible Solution

Add a focused A2UI source-build guard that removes stale ignored renderer output before compilation when safe, or fails with a precise recovery message when cleanup cannot proceed, and document the source-checkout recovery path for in-place updates.

## Reproduction Assessment

Yes. A high-confidence synthetic reproduction path is an existing git checkout with ignored `vendor/a2ui/renderers/lit/dist` output: it can pass the updater clean check and still reach the A2UI renderer compile during `pnpm build`.

## Solution Assessment

Yes. The narrowest maintainable fix is in the A2UI source-build path, with docs for recovery; changing the updater's broader install-mode strategy would be larger than the observed failure requires.

## Repair Work Prompt

Fix the in-place git/source update failure class reported at https://github.com/openclaw/openclaw/issues/43712. Current main lets ignored `vendor/a2ui/renderers/lit/dist` output survive the source-update dirty check, then `pnpm build` runs `scripts/bundle-a2ui.mjs` and can fail during the renderer `tsgo` compile. Add a focused guard in the A2UI source-build path: safely remove stale renderer `dist` output before compilation when it is clearly generated/owned, or fail early with a precise recovery message when cleanup cannot proceed. Add regression coverage in `test/scripts/bundle-a2ui.test.ts`; update `src/infra/update-runner.test.ts` only if the updater messaging/flow is changed. Update `docs/install/updating.md` and/or `docs/cli/update.md` with the source-checkout recovery path. Add a user-facing changelog entry crediting `Thanks @turrizt` if behavior or docs change. Do not rewrite the whole updater install-mode strategy, do not change package-manager selection, and do not touch unrelated A2UI runtime behavior.
