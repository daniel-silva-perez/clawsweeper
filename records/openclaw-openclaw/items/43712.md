## Summary

Keep this issue open. Current main has improved explicit CLI channel switching, and the secondary Telegram SDK-helper mismatch is fixed, but the central reported failure mode remains plausible for in-place git update paths: ignored/stale A2UI renderer output can survive the git clean check, and the normal build still runs the A2UI bundler without cleaning or rejecting that renderer dist directory first.

## Best Possible Solution

Keep this issue open and fix the source-update/A2UI edge case directly. The safest product/code path is to clean or explicitly reject stale `vendor/a2ui/renderers/lit/dist` before `canvas:a2ui:bundle` during in-place git builds, add regression coverage for ignored renderer dist causing TS5055-style collisions, and document a recovery step for source checkouts in the public updating docs.
