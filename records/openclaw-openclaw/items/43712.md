## Summary

Keep this open. Current main still has the in-place source-update failure class: ignored A2UI renderer output can survive the updater clean check, and the full build still invokes the A2UI renderer TypeScript build without cleaning or rejecting that stale output first.

## What This Changes

The issue reports that upgrading a mutable git install in a global node_modules-style location failed during the build, while the same release tag built successfully in a clean checkout on the same host.

## Best Possible Solution

Add a focused source-build guard that safely removes stale A2UI renderer output before renderer compilation, or fails with a precise recovery message when cleanup cannot proceed; document the source-checkout recovery path without broadening the package-update staging design.

## Reproduction Assessment

Yes. A high-confidence synthetic reproduction path is a git checkout with ignored `vendor/a2ui/renderers/lit/dist` output from an earlier build: current main can pass the updater clean check and then reach `canvas:a2ui:bundle` during `pnpm build`.

## Solution Assessment

Yes. The narrowest maintainable fix is in the A2UI source-build path, with updater/docs changes only for clear recovery messaging; treating all ignored files as dirty would be broader and riskier than this failure requires.
