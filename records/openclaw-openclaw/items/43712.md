## Summary

Keep this issue open. Current main has improved explicit channel switching, but stored/non-explicit git update paths can still build inside the existing checkout, and the A2UI bundle script still neither cleans nor rejects ignored stale renderer output before compiling.

## What This Changes

The issue reports that updating an existing git/source install from 2026.3.8 to v2026.3.11 failed in-place with A2UI TypeScript build errors while the same tag built in a clean checkout.

## Best Possible Solution

Add a source-build guard around A2UI renderer compilation that removes stale ignored renderer output, or fails with a precise recovery message if cleanup cannot proceed, then document the recovery path for source checkout updates.

## Repair Work Prompt

Fix openclaw/openclaw#43712. The observable bug is that in-place git/source updates can fail during `canvas:a2ui:bundle` when ignored stale `vendor/a2ui/renderers/lit/dist` output is present from an older checkout/build. Add a small guard before `scripts/bundle-a2ui.mjs` invokes `tsgo -p vendor/a2ui/renderers/lit/tsconfig.json`: either remove stale ignored renderer dist safely before compiling, or fail with an actionable message if cleanup cannot proceed. Preserve deterministic hash input behavior and keep `vendor/a2ui/renderers/lit/dist` excluded from bundle hashes. Add focused regression coverage in `test/scripts/bundle-a2ui.test.ts` for the stale-dist guard. Update `docs/install/updating.md` and/or `docs/cli/update.md` with the source-checkout recovery behavior if user-visible. Add a single-line changelog fix entry with `Thanks @turrizt` if docs/changelog policy considers this user-facing. Do not change package-update staging, bundled plugin runtime dependency staging, or unrelated update-channel behavior.
