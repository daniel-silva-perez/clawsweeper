## Summary

Keep this issue open. Current main has hardened explicit git-to-package channel switching, but the remaining reported failure mode is still credible: stored-channel/direct git update paths can still build in place, while the A2UI bundler still compiles the lit renderer without cleaning or rejecting ignored stale renderer output first.

## What This Changes

The issue requests that in-place git updates from an existing Linux global node_modules checkout either succeed or detect/clean stale generated A2UI state instead of failing during `pnpm build`.

## Best Possible Solution

Add a small source-build guard around the A2UI renderer compile: clean or fail with an actionable message when stale `vendor/a2ui/renderers/lit/dist` exists before bundling, cover that behavior with a focused script test, and document the recovery path for source checkout updates.
