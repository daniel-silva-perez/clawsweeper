## Summary

Keep this PR open. Current main still has the reported closed-stdin gap in `openclaw plugins uninstall <id>`: the shared yes/no prompt awaits `readline.question()` directly, and the uninstall command awaits that prompt without handling readline close/EOF. The PR remains a focused implementation candidate for the paired open bug #73562, and the latest supplied PR diff appears to have addressed the earlier bot review concern by using `instanceof` plus a real `PromptInputClosedError` in tests.

## Best Possible Solution

Keep this PR open for normal maintainer review. If accepted, land the focused EOF-aware prompt handling and regression tests, letting it close #73562 after merge. Keep #73551 separate unless a broader plugin enable/disable validation change explicitly resolves that stale-entry seed path too.
