## Summary

Keep open. Current main and the latest tagged release still directly await `readline.question()` in `promptYesNo()`, and `plugins uninstall` reaches that helper before config or install-record mutation. The PR’s latest diff addresses the linked crash with typed close detection, command-level `--force` guidance, and focused regression tests; I did not find a blocking correctness or security issue, so the remaining action is maintainer review and merge.

## What This Changes

The PR exports a `PromptInputClosedError`, races `readline.question()` against readline `close`, catches that typed error in `plugins uninstall`, and adds prompt/uninstall regression tests.

## Best Possible Solution

Land this focused EOF-aware prompt handling after maintainer review, preserving piped `y`/`n` answers and `--force`; let the linked crash report close through the merge while keeping the separate stale enable/disable entry report tracked independently.
