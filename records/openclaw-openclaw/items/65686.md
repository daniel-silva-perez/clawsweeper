## Summary

Keep open. Current main still lacks a configurable isolated-cron outbound payload policy: the config/schema/docs surface has no `cron.deliveryPayloads` or equivalent, the resolver still returns all successful deliverable payloads by default, and direct delivery still falls back to synthesized text when no payloads are selected. The linked implementation PR #63650 was closed unmerged, and no open PR appears to reference this issue with closing syntax.

## Best Possible Solution

Keep this issue open and implement a scoped isolated-cron outbound payload policy. The conservative path is a typed `cron.deliveryPayloads` setting with default `full`, `last` selection after existing payload resolution, and `none` that suppresses outbound runner delivery without falling back to synthesized text; update schema/help/docs/changelog and targeted cron resolver/dispatch tests.
