## Summary

Current main still lacks a user-configurable isolated-cron outbound payload policy. There is no `cron.deliveryPayloads` or equivalent config/schema/docs surface, payload resolution still preserves all successful deliverable payloads by default except for channel-owned final-text preference, and direct dispatch still falls back to synthesized text when payloads are empty. The linked implementation PR #63650 is closed unmerged, and open-PR search found no active implementation referencing this issue.

## Best Possible Solution

Keep this issue open and implement a scoped isolated-cron outbound payload policy. The conservative path is a typed `cron.deliveryPayloads` setting with default `full`, `last` selection after existing payload resolution, and `none` that suppresses outbound runner delivery without falling back to synthesized text; update schema/help/docs/changelog and targeted resolver/dispatch tests.
