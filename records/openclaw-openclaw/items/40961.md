## Summary

Keep this PR open, but not merge-ready as-is. Current main still has unresolved A2A timeout behavior, while this branch removes one 60-second guard without replacing it with an explicit bound and also diverges from the current 120-second subagent announce default.

## What This Changes

The PR raises the subagent announce default and matching test expectation from 60 seconds to 180 seconds, and changes the `sessions_send` A2A wait phase to use `announceTimeoutMs` directly instead of capping it at 60 seconds.

## Best Possible Solution

Rework the A2A timeout handling on current main with an explicit bounded policy for `sessions_send.timeoutSeconds`, keep or deliberately change the current 120-second subagent announce default, handle the remaining 60-second caps consistently, and add focused regression tests for waits above 60 seconds and timeout-triggered background announce behavior.
