## Summary

Keep open and do not merge as-is. The PR targets a real A2A timeout problem that current main still only partially addresses, but the branch is stale against the current 120-second default and removes a runtime wait cap without adding a product-level maximum for caller-controlled waits.

## What This Changes

The PR raises the subagent announce timeout default and matching test expectation from 60 seconds to 180 seconds, and changes the first `sessions_send` A2A wait to use the configured announce timeout instead of a 60-second cap.

## Best Possible Solution

Define one A2A timeout policy on current main: preserve or deliberately change the 120-second default, add an explicit maximum or shared clamp for `sessions_send.timeoutSeconds`, and make the initial wait, nested reply/announce waits, and timeout-triggered background announce behavior consistent with focused regression coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Bound the A2A wait timeout before removing the cap:** `src/agents/tools/sessions-send-tool.a2a.ts:34`
