## Summary

Keep open, but do not merge as-is. Current main partially moved the subagent announce default to 120 seconds, but the proposed cap removal is still incomplete and unsafe because `sessions_send.timeoutSeconds` has no maximum, nested A2A steps remain capped at 60 seconds, and the related timeout-delivery behavior in #45926 still needs a coordinated policy decision.

## What This Changes

The PR raises the subagent announce default and matching timeout test to 180 seconds, and changes the first `sessions_send` A2A wait to use the configured announce timeout instead of a 60-second cap.

## Best Possible Solution

Define one A2A timeout policy on current main: choose the intended default and maximum, clamp `sessions_send.timeoutSeconds` through a shared helper or schema rule, align the initial wait, nested reply/announce waits, and timeout-triggered background announce behavior, then cover those cases with focused regression tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Bound the A2A wait timeout before removing the cap:** `src/agents/tools/sessions-send-tool.a2a.ts:34`
  - body: The changed line lets `sessions_send.timeoutSeconds` drive the initial `agent.wait` with no upper bound. Add an explicit maximum or shared clamp before honoring waits above 60 seconds.
