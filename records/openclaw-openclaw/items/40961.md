## Summary

Keep this PR open. Current main only partially overlaps with the contribution: subagent announce delivery now defaults to 120s and is configurable, but the `sessions_send` A2A wait path still caps the target-run wait at 60s, which is one of the PR's central intended fixes. The old patch is stale and needs a bounded timeout policy before it is mergeable, but the intended behavior is not implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review, but rework it against current main: preserve or explicitly revisit the shipped 120s subagent announce default, add a bounded/configurable policy for `sessions_send.timeoutSeconds`, remove inappropriate 60s A2A caps consistently, and add focused regression coverage for waits above 60 seconds plus the fire-and-forget A2A path.
