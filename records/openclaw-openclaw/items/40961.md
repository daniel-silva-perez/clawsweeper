## Summary

Keep open, but do not merge as-is. The PR addresses a real A2A timeout gap that still exists on current main, but the branch is stale against the current 120-second subagent announce default and removes a 60-second wait cap without adding a product-level maximum for caller-controlled timeouts.

## What This Changes

The PR raises the subagent announce timeout default and matching test expectation from 60 seconds to 180 seconds, and changes the `sessions_send` A2A wait path to use the configured announce timeout instead of a 60-second cap.

## Best Possible Solution

Rework A2A timeout handling on current main with an explicit maximum or shared clamp for `sessions_send.timeoutSeconds`, preserve or deliberately revise the current 120-second default, and handle the first wait, nested reply/announce waits, and timeout-triggered background announce behavior consistently with focused regression coverage.
