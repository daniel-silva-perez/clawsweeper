## Summary

Keep open. Current main still has the reported heartbeat mid-text leak, and this PR is useful source work, but the proposed patch is incomplete and has drifted from current main’s transcript/provider architecture.

## What This Changes

The PR adds heartbeat-mode fallback stripping for mid-text `HEARTBEAT_OK` tokens, heartbeat unit tests, and older transcript/provider capability plumbing for MiniMax thinking-block replay.

## Best Possible Solution

Land a current-main heartbeat fix that removes all `HEARTBEAT_OK` occurrences in heartbeat mode after markup normalization, preserves message-mode behavior, adds mid-only, edge+middle, multiple-token, markup-wrapped, newline, short-ack, and long-reply tests, and leaves MiniMax replay policy on the existing plugin-hook path.

## Reproduction Assessment

Yes. On current main, a heartbeat reply like `Some preamble HEARTBEAT_OK actual reply` reaches the `!picked.didStrip` branch and returns the original token-containing text; the existing mid-token test only asserts message-mode preservation.

## Solution Assessment

No. The PR’s approach is not the best current fix because it only removes mid-text tokens when no edge token was stripped and it operates on raw markup-bearing text; a normalized remove-all heartbeat path before the length gate is safer and narrower.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Remove all heartbeat tokens after edge stripping:** `src/auto-reply/heartbeat.ts:156-168`
  - body: The fallback only runs when `picked.didStrip` is false. For heartbeat output with an edge token plus another token in the middle, edge stripping succeeds and the new fallback is skipped, so a long reply can still return a remaining `HEARTBEAT_OK`. Run the heartbeat remove-all pass after edge normalization and add an edge+middle regression.
  - confidence: 0.88
- **[P2] Strip fallback text through markup normalization:** `src/auto-reply/heartbeat.ts:162-166`
  - body: The new fallback splits raw `trimmed`, so a mid-text wrapped token such as `preamble <b>HEARTBEAT_OK</b> long content` leaves wrapper fragments like `<b> </b>` in delivered text and in the ack length gate. Use the same normalized content path for heartbeat fallback removal.
  - confidence: 0.85
