Keep this PR open, but do not merge it as-is. Current main still has the reported edge-only heartbeat behavior, while this PR only partially fixes the token leak and its non-heartbeat MiniMax/transcript-policy commits are stale against the current provider-hook replay architecture.

## What This Changes

## Best Possible Solution

Land a narrow current-main heartbeat fix that removes all `HEARTBEAT_OK` occurrences in heartbeat mode after markup normalization, preserves message-mode behavior, covers mid-token, edge-plus-middle, repeated-token, and HTML/Markdown-wrapped cases, and updates the heartbeat response-contract docs. ## Reproduction Assessment

Yes. Current main can be reproduced with a focused unit case such as `stripHeartbeatToken("Some preamble HEARTBEAT_OK actual reply", { mode: "heartbeat" })`, because the helper only strips when the token is at an edge and heartbeat delivery calls that helper directly.

## Solution Assessment

No. The PR’s direction is useful, but this is not the best final patch because it skips remaining middle tokens after an edge strip, processes raw markup-wrapped text in the fallback, and carries stale non-heartbeat replay-policy changes that current main has already redesigned.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Strip remaining heartbeat tokens after edge stripping:** `src/auto-reply/heartbeat.ts:157-158`
  - body: The new fallback only runs when `picked.didStrip` is false. If a heartbeat reply has an edge token and another token later in long content, the edge strip sets `didStrip`, the fallback is skipped, and the later `HEARTBEAT_OK` can still be delivered. Run the heartbeat-mode removal after edge stripping and add an edge-plus-middle regression case.
  - confidence: 0.9
- **[P2] Strip mid-token markup before the length gate:** `src/auto-reply/heartbeat.ts:162-166`
  - body: The fallback splits raw `trimmed` text, so `<b>HEARTBEAT_OK</b>` or markdown-wrapped middle tokens leave wrapper artifacts and count those bytes toward `maxAckChars`. That can turn a short ack into a delivered message and leak markup; use normalized content or strip wrappers before measuring.
  - confidence: 0.86
