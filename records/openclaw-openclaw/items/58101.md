## Summary

Keep this PR open. The central heartbeat bug is still present on current main: `stripHeartbeatToken` still returns the original text when `HEARTBEAT_OK` is only in the middle, and the docs still state that mid-reply tokens are not treated specially. The MiniMax/context side of the PR appears partly superseded by the newer provider replay-policy hook architecture, so the best path is a rebased, focused heartbeat fix plus separate handling for any remaining MiniMax work.

## Best Possible Solution

Keep this PR open, but rework it into a focused current-main heartbeat fix: in heartbeat mode, remove all `HEARTBEAT_OK` occurrences from normalized text before the skip/length decision, preserve message-mode behavior, add regression tests for middle-only, edge-plus-middle, multiple-token, and markup-wrapped cases, and update `docs/gateway/heartbeat.md`. Treat any remaining MiniMax replay-policy work as a separate current-architecture PR if a gap still exists after the provider hook changes.
