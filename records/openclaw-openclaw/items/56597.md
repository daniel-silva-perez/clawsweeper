## Summary

Keep open. Current main still persists `/exec` session defaults into durable session state and resolves those stored values before agent/global exec defaults, while docs and tests currently treat that stickiness as intended session behavior. Because the report affects exec security semantics, the fix needs maintainer policy input rather than a cleanup close.

## What This Changes

The issue asks for Discord sessions with prior per-session exec directives to respect later global `tools.exec` policy changes instead of indefinitely reusing persisted session exec overrides.

## Best Possible Solution

Define the intended admin-policy precedence for exec settings, then update the generic session exec persistence/resolution path, reset/patch behavior if needed, docs, and focused regression tests so tightened global exec policy takes effect for existing Discord sessions without a Discord-only workaround.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by resolving a persisted session entry with `execSecurity: "full"` after global config is changed to a stricter value; current main reads the stored session value before configured defaults.

## Solution Assessment

Unclear. The report identifies a real policy conflict, but the best fix depends on maintainer semantics for durable session overrides versus operator-enforced global exec policy.
