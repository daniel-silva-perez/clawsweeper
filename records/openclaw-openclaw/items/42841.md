## Summary

Current main still does not implement the PR’s central behavior: `openclaw node run/install` cannot accept or persist custom WebSocket upgrade headers, and `GatewayClient` has no header option. The PR remains a useful implementation candidate for the Cloudflare Zero Trust workflow, with credential-handling details that need maintainer review before merge.

## Best Possible Solution

Keep the PR open as the active implementation candidate for #42792. The best path is to review or land this PR, or an equivalent core change, that adds repeatable `--header` support for `openclaw node run` and `openclaw node install`, validates and resolves headers with clear config/env/CLI precedence, forwards them through `GatewayClient` as WebSocket upgrade headers, avoids leaking credential-bearing values in service args/status/logs, updates `docs/cli/node.md`, fixes changelog attribution, and runs targeted node CLI, node daemon install, node-host, GatewayClient, service-env, and docs checks.
