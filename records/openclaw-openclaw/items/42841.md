## Summary

Current main still does not implement the PR's central behavior: there is no `openclaw node run` or `openclaw node install` `--header` option, no node-host config field for WebSocket upgrade headers, no `GatewayClientOptions.headers`, and no `OPENCLAW_NODE_HEADERS` or Cloudflare Access env propagation. The PR remains the active implementation candidate for the Cloudflare Zero Trust 302 failure described in #42792, and it needs maintainer review rather than cleanup closure because it handles credential-bearing headers through config/service environment/status surfaces.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #42792. Maintainers should review the credential-handling model, verify explicit `--header` versus saved config and env precedence, ensure status output cannot expose secrets, fix the changelog attribution, rebase if needed, and run targeted node CLI, daemon install, node-host, and gateway client validation before merge.
