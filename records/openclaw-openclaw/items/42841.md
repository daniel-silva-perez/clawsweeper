## Summary

Current main still does not implement the PR’s central behavior. `openclaw node run` and `openclaw node install` have no repeatable `--header` option, node host config/runtime has no header field or resolver, `GatewayClient` has no custom header option, and the node docs still document only `OPENCLAW_GATEWAY_*` auth env vars. The PR remains a plausible implementation candidate for the Cloudflare Zero Trust WebSocket-upgrade workflow, but it needs maintainer review for credential handling before merge.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #42792. The best path is to review or land this PR, or an equivalent core change, that adds repeatable `--header` support to `openclaw node run` and `openclaw node install`, validates header syntax, defines CLI/config/env precedence, forwards headers through `GatewayClient`, avoids leaking credential values in service args/status/logs, updates `docs/cli/node.md`, fixes changelog attribution, and runs targeted node CLI, node-host, GatewayClient, service-env, daemon install, and docs checks.
