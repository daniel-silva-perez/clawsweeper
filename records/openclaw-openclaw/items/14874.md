## Summary

Keep open. Current main still lacks a TypeScript skill-loader/config SIE envelope or signature verification path, and the related implementation PR #14893 was closed unmerged. The adjacent SHA-256 fingerprint work discussed in #56256 is not a substitute for issuer/signature verification, so the requested trust-boundary design remains open.

## What This Changes

Add a default-off optional SIE verification gate to the skill loader/config path, with warn/strict unsigned behavior and invalid-signature rejection.

## Best Possible Solution

Define a default-off TypeScript SIE verification contract for all skill roots, with deterministic reason codes, warn/strict unsigned handling, invalid-signature fail-closed behavior, documented trust-root and revocation policy, config/schema support, and docs explaining its relationship to ClawHub scans and skill install/update flows.
