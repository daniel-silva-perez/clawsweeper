## Summary

Keep open. Current main still lacks optional SIE envelope/signature verification in the TypeScript skill loader, workspace skill merge path, config schema, tests, and public skills docs; the directly related implementation PR #14893 is closed unmerged, and the open SHA-256 fingerprint proposal #56256 does not provide issuer/signature verification semantics.

## What This Changes

The issue requests an optional, feature-flagged SIE verification gate in the skill loader with warn/strict handling for unsigned skills and rejection for invalid signatures.

## Best Possible Solution

Define a default-off TypeScript SIE verification contract across all skill roots, with config/schema/docs support, deterministic reason codes, documented trust-root and revocation policy, warn/strict unsigned handling, invalid-signature fail-closed behavior, and clear composition with ClawHub install/update and audit flows.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence source verification path. Targeted search plus inspection of the loader, workspace root loading, config schema, docs, tests, related PR state, and adjacent hashing surface show the requested SIE behavior is absent on current main.

## Solution Assessment

Unclear until maintainers choose the trust model. The requested direction is plausible, but the best fix should be a maintainer-approved TypeScript loader/config contract rather than the closed standalone Python/reference PR or SHA-256 hashing alone.
