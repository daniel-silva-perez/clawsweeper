## Summary

Keep open. Current main still lacks the requested optional SIE warn/strict verification gate in the TypeScript skill loader and config surface. The loader reads `SKILL.md`, parses frontmatter, and exposes `SkillEntry` records to prompts, slash commands, snapshots, and sandbox sync without signature sidecar lookup, verification state, deterministic SIE reason codes, warnings, or strict-mode rejection. The directly related PR #14893 was closed unmerged, so it remains prior art rather than a shipped fix.

## Best Possible Solution

Keep this issue open. The best path is a disabled-by-default, core-owned skill-loader verification policy or a first-class loader verification hook/API that produces deterministic decisions for unsigned, valid-signed, and invalid-signed skills before loaded entries are exposed to prompts, slash commands, skill snapshots, sandbox sync, or runtime use. The closed PR #14893 is useful prior art, but any new implementation should align with the current TypeScript loader, exported skill entry/config contracts, docs, and maintainer security design.
