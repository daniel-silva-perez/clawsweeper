## Summary

Keep open. Current main still lacks an optional SIE envelope/signature verification contract in the TypeScript skill loader/config path, and the existing maintainer-facing review comment correctly distinguishes this from adjacent SHA-256 integrity work. This is security-sensitive trust-boundary design, not a safe auto-close or auto-repair item.

## What This Changes

Request an optional, feature-flagged SIE verification gate for skill loading with warn/strict behavior for unsigned skills and rejection for invalid signatures.

## Best Possible Solution

Design a default-off TypeScript skill loader/config contract for SIE verification, with deterministic reason codes, warn/strict unsigned handling, invalid-signature rejection, coverage for all skill roots, and docs that explain how SIE relates to ClawHub scans, archive hashes, lockfiles, install/update flows, trust roots, and revocation.
