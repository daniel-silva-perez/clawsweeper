## Summary

Keep open. Current main still lacks loader-time SIE envelope/signature verification, warn/strict unsigned handling, trust-root configuration, and docs/tests for that behavior. The related implementation PR #14893 is closed unmerged, and the open SHA-256 integrity proposal #56256 is a content-hash primitive rather than issuer/signature verification.

## What This Changes

The issue requests a default-off SIE verification gate in the TypeScript skill loader with warn/strict unsigned handling, invalid-signature rejection, deterministic reason codes, and config/test coverage.

## Best Possible Solution

Define a default-off TypeScript SIE verification contract for all skill roots, with config/schema/docs support, deterministic reason codes, documented trust-root and revocation policy, warn/strict unsigned behavior, and invalid-signature fail-closed behavior that composes clearly with ClawHub install/update flows.
