## Summary

Keep open. Current main still lacks loader-time SIE envelope/signature verification, warn/strict config, and docs/tests for that behavior; the linked implementation PR is closed unmerged, and the adjacent SHA-256 fingerprint proposal does not provide issuer/signature verification.

## What This Changes

The issue requests an optional, feature-flagged Signed Instruction Envelope verification gate in the skill loader with warn and strict modes.

## Best Possible Solution

Define a default-off TypeScript SIE verification contract for all skill roots, with deterministic reason codes, config/schema/docs support, documented trust-root and revocation policy, warn/strict unsigned behavior, and invalid-signature fail-closed behavior that composes clearly with ClawHub install/update flows.
