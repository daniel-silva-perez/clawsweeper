## Summary

Keep open. Current main still has safe skill file loading, frontmatter parsing, root precedence, and ClawHub install/download flows, but no TypeScript SIE envelope lookup, signature verifier, issuer policy, warn/strict unsigned handling, or invalid-signature rejection. The related implementation PR #14893 is closed unmerged, and the open #56256 SHA-256 lockfile proposal is adjacent integrity work rather than signed issuer verification.

## What This Changes

Add an optional, feature-flagged SIE verification gate in the skill loader with warn/strict unsigned handling and invalid-signature rejection.

## Best Possible Solution

Define a default-off TypeScript SIE verification contract for all skill roots, with deterministic reason codes, config/schema/docs support, documented trust-root and revocation policy, warn/strict unsigned behavior, and invalid-signature fail-closed behavior that composes clearly with ClawHub install/update flows.
