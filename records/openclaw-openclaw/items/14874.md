## Summary

Keep open. Current main still lacks the requested SIE envelope/signature verification path in the TypeScript skill loader, root merge flow, config schema, tests, and public docs; the related implementation PR #14893 is closed unmerged, and the adjacent SHA-256 integrity proposal #56256 is not equivalent to issuer/signature verification.

## What This Changes

Requests an optional feature-flagged Signed Instruction Envelope verification gate in the skill loader, with warn/strict handling for unsigned skills and rejection of invalid signed envelopes.

## Best Possible Solution

Define a default-off TypeScript SIE verification contract across all skill roots, with config/schema/docs support, deterministic reason codes, documented trust-root and revocation policy, warn/strict unsigned behavior, invalid-signature fail-closed behavior, and clear composition with ClawHub install/update and audit flows.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence source verification path. Targeted search plus inspection of the loader, workspace root loading, config schema, docs, related PR state, and adjacent ClawHub hashing surface show the requested SIE behavior is absent on current main.

## Solution Assessment

Unclear until maintainers choose the trust model. The requested direction is plausible, but the best fix should be a maintainer-approved TypeScript loader/config contract rather than the closed standalone Python/reference PR or SHA-256 hashing alone.
