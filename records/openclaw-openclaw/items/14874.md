## Summary

Keep open. Current main still loads skills from SKILL.md files with path, size, frontmatter, source precedence, and visibility checks, but it has no optional SIE envelope verification, trusted issuer policy, warn/strict unsigned behavior, invalid-signature rejection, or deterministic SIE reason-code contract. The related implementation PR #14893 is closed unmerged, and open PR #56256 is an adjacent SHA-256 integrity proposal rather than the same signature-verification design.

## Best Possible Solution

Keep this open for maintainer security design. The best path is a default-off TypeScript loader/config contract for SIE verification, deterministic machine-readable reason codes, warn/strict behavior for unsigned skills, reject behavior for invalid signatures, coverage for bundled/managed/workspace/extra/plugin skill roots, and docs explaining how SIE relates to ClawHub scans, archive hashes, lockfiles, and skill install/update flows.
