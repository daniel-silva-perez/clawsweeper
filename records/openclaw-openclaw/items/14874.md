## Summary

Keep open. Current main still lacks loader-time SIE envelope or signature verification, a warn/strict config contract, and docs/tests for that behavior; the related implementation PR is closed unmerged, and adjacent SHA-256 fingerprint work is not issuer/signature verification. Because this defines a security trust boundary, maintainers should choose the trust model before any automated fix is queued.

## What This Changes

The issue requests an optional, feature-flagged SIE verification gate for skill loading with unsigned warn/strict handling and invalid-signature rejection.

## Best Possible Solution

Define a default-off TypeScript SIE verification contract for all skill roots, with deterministic reason codes, config/schema/docs support, documented trust-root and revocation policy, warn/strict unsigned behavior, and invalid-signature fail-closed behavior that composes with ClawHub install/update flows.
