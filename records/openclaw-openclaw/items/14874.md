## Summary

Keep open. Current main still lacks the requested SIE envelope/signature verification path in the TypeScript skill loader, config schema, and public docs; the related implementation PR is closed unmerged, and the open SHA-256 integrity PR is adjacent but does not provide issuer/signature verification.

## What This Changes

The issue requests optional, feature-flagged Signed Instruction Envelope verification in the skill loader, with warn/strict behavior for unsigned skills and rejection for invalid signatures.

## Best Possible Solution

Define a default-off TypeScript SIE verification contract for all skill roots, with config/schema/docs support, deterministic reason codes, documented trust-root and revocation policy, warn/strict unsigned behavior, and invalid-signature fail-closed behavior that composes clearly with ClawHub install/update flows.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence source verification path: targeted searches plus inspection of the loader, workspace root loading, config schema, and docs show the requested SIE behavior is absent on current main.

## Solution Assessment

Unclear until maintainers choose the trust model. A narrow implementation should follow a maintainer-approved TypeScript contract; the closed Python/reference PR and adjacent SHA-256 proposal are not the best final shape by themselves.
