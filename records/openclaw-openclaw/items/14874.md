## Summary

Keep open. Current main still loads skills through path, size, and frontmatter checks only, with no SIE envelope/signature verification, config knob, or docs/tests for warn/strict behavior; the related implementation PR is closed unmerged, and the adjacent SHA-256 ClawHub fingerprint work is not issuer/signature verification.

## What This Changes

The issue asks OpenClaw to add an optional, feature-flagged Signed Instruction Envelope verification gate to the skill loader, with warn/strict handling for unsigned skills and rejection for invalid signatures.

## Best Possible Solution

Define a default-off TypeScript SIE verification contract for all skill roots, with deterministic reason codes, config/schema/docs support, documented trust-root and revocation policy, warn/strict unsigned behavior, and invalid-signature fail-closed behavior that composes clearly with ClawHub install/update flows.
