## Summary

Keep open. Current main still lacks the requested SIE envelope/signature verification path for skill loading, and the related implementation PR was closed unmerged. The existing ClawHub scan/download metadata and SHA-256 fingerprint helper are adjacent integrity work, not a substitute for issuer/signature verification with warn/strict loader policy.

## What This Changes

The issue requests an optional, default-off SIE verification gate in the skill loader with warn mode for unsigned skills, strict rejection for unsigned skills, and fail-closed handling for invalid signatures.

## Best Possible Solution

Define a default-off TypeScript SIE verification contract for all skill roots, with config/schema/docs support, deterministic reason codes, documented trust-root and revocation policy, warn/strict unsigned behavior, and invalid-signature fail-closed behavior that composes clearly with ClawHub install/update flows.
