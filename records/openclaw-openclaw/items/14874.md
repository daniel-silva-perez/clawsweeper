## Summary

Keep open. Current main still lacks the requested SIE envelope/signature verification path in the TypeScript skill loader, skill root merge flow, config schema, tests, and public docs; the related implementation PR is closed unmerged, and the adjacent SHA-256 integrity proposal is not a substitute for issuer/signature verification.

## What This Changes

The issue requests an optional, feature-flagged Signed Instruction Envelope verification gate for skill loading with warn/strict handling for unsigned skills and rejection of invalid signatures.

## Best Possible Solution

Define a default-off TypeScript SIE verification contract for all skill roots, with config/schema/docs support, deterministic reason codes, documented trust-root and revocation policy, warn/strict unsigned behavior, invalid-signature fail-closed behavior, and clear composition with ClawHub install/update flows.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence source verification path. Targeted searches plus inspection of the loader, workspace root loading, config schema, docs, and related tests show the requested SIE behavior is absent on current main.

## Solution Assessment

Unclear until maintainers choose the trust model. The requested direction is plausible, but the best implementation should be a maintainer-approved TypeScript loader/config contract rather than the closed standalone Python/reference PR or the adjacent SHA-256 proposal alone.
