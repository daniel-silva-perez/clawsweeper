## Summary

Keep this PR open. Current main still has the reported legacy-key mismatch: `isSubagentSessionKey()` accepts raw `subagent:<id>` keys, but `getSubagentDepth()` still counts only `:subagent:` and returns 0 for a leading legacy `subagent:` segment. The PR remains a focused candidate fix with targeted helper tests and no visible security-sensitive diff surface.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land the focused `getSubagentDepth()` fix with the legacy helper tests, ideally adding one metadata-missing `getSubagentDepthFromSessionStore("subagent:flat")` regression case, then run the targeted test and normal changed gate.
