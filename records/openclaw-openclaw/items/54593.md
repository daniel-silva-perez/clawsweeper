## Summary

Keep this PR open. Current main still has the reported mismatch: raw legacy keys like `subagent:worker` are accepted as subagent session keys, while `getSubagentDepth()` still counts only `:subagent:` substrings and returns 0 for a leading legacy `subagent:` segment. The PR is still a focused candidate fix with targeted tests and no visible supply-chain or secrets-handling changes.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land the targeted helper fix with the legacy `getSubagentDepth()` unit cases, ideally adding one store-fallback regression case for metadata-missing raw legacy keys, then run the targeted test and normal changed gate.
