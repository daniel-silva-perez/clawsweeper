## Summary

Keep this issue open. Current main has added adjacent ACP task tracking, `session_status` task visibility, `sessions_send`, and hook `deliver: false`, but the central channel-less persistent ACP workflow remains blocked: ACP `mode: "session"` still requires `thread: true`, accepted spawn results still do not expose ACP backend/agent resume identity, and `/hooks/agent` still routes through isolated cron execution rather than first dispatching ACP session keys through `acpManager.runTurn`.

## What This Changes

The issue requests reliable tracking, resume identity, headless persistent ACP sessions, hook-targeted ACP follow-ups, and agent-controlled hook delivery for ACP sessions spawned by webhook-triggered agents.

## Best Possible Solution

Define an explicit headless/unbound persistent ACP contract, expose stable ACP resume identity through spawn/status surfaces, document whether follow-up turns should use `sessions_send` or direct hook-side ACP dispatch, and add focused regression coverage for hook-triggered channel-less ACP workflows while keeping `deliver: false` as the hook auto-delivery suppression mechanism.
