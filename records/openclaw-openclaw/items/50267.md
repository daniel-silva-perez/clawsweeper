## Summary

Keep this PR open. Current main still does not pass provider request-policy headers into the image-understanding direct `complete()` calls, so the OpenRouter image-tool attribution problem remains real. The submitted patch is stale against the current file layout and should be reworked to use endpoint-aware request-policy helpers rather than provider-only attribution.

## Best Possible Solution

Rework this PR against `src/media-understanding/image.ts`: resolve headers with `resolveProviderRequestHeaders` or `resolveProviderRequestPolicyConfig` using the resolved model provider, api, baseUrl, `capability: "image"`, and `transport: "media-understanding"`; pass those headers into both direct `complete()` attempts; add regression tests for OpenRouter default/native-route attribution and custom-base-url non-injection.
