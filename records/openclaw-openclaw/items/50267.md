## Summary

Keep this PR open. Current main still does not pass endpoint-aware OpenRouter attribution headers into the image-understanding direct `complete()` call, and this PR remains aimed at a real gap, but its patch is stale and uses a provider-only helper that bypasses the current endpoint-gated request policy.

## Best Possible Solution

Rework this PR against current main in `src/media-understanding/image.ts`: resolve direct-completion headers with `resolveProviderRequestHeaders` or `resolveProviderRequestPolicyConfig` using `provider`, `api`, `baseUrl`, `capability: "image"`, and `transport: "media-understanding"`, pass those headers into `complete()`, and add focused tests for OpenRouter/default-route attribution plus custom-base-url non-injection. If maintainers want the broader direct-completion fix, revive that scope separately from #57561 rather than merging this stale provider-only helper unchanged.
