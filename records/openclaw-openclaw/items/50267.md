## Summary

Keep this PR open. Current main still does not pass endpoint-aware attribution headers into the image-understanding direct `complete()` call, so the reported OpenRouter image-tool gap remains. The PR is still aimed at a real problem, but its patch is stale: it targets a deleted path and uses provider-only attribution headers instead of the current request-policy helpers that gate attribution by endpoint/API/capability/transport.

## Best Possible Solution

Rework this PR against current main in `src/media-understanding/image.ts`: resolve direct-completion headers with `resolveProviderRequestHeaders` or `resolveProviderRequestPolicyConfig` using `provider`, `api`, `baseUrl`, `capability: "image"`, and `transport: "media-understanding"`, pass those headers into `complete()`, and add regression tests for OpenRouter default/native-route attribution plus custom-base-url non-injection. If maintainers want the broader direct-completion fix, revive that separately from the closed unmerged #57561 scope.
