## Summary

Keep this open. Current main still rejects the reported config shapes and restores last-known-good with generic startup/reload logs and a generic main-agent recovery notice, while open PR #75081 uses closing syntax and is the active diagnostics-only implementation candidate with a protected maintainer label.

## What This Changes

The issue asks OpenClaw to avoid silently restoring last-known-good config when invalid direct edits such as `agents.defaults.execution` or malformed `gateway.auth.password` are rejected under `gateway.bind: "lan"`, either by accepting the config or surfacing clear validation errors.

## Best Possible Solution

Land or supersede #75081 with a diagnostics-only change that preserves last-known-good recovery, strict agent-defaults schema, and SecretRef validation while surfacing concise rejected validation paths/messages in recovery logs and the main-agent recovery notice.

## Reproduction Assessment

Yes. The reproduction path is a direct invalid config edit followed by startup or reload recovery; static inspection and existing tests show the reported shapes are rejected and current recovery messaging remains generic. I did not run tests because this review is read-only.

## Solution Assessment

Yes. A diagnostics-only fix is the narrow maintainable path; accepting `agents.defaults.execution` or the malformed SecretRef shape would be a broader product/security decision.
