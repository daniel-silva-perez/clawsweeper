## Summary

Keep open. Current main still restores invalid root config from last-known-good with generic recovery logging and a generic main-agent notice, and the issue is already paired with open PR #75081 using closing syntax, so this should wait for that PR to be reviewed, landed, or superseded.

## What This Changes

The issue asks OpenClaw to stop making rejected direct config edits look silent when `agents.defaults.execution` or a legacy-shaped `gateway.auth.password` object is restored under a LAN-bound gateway.

## Best Possible Solution

Ship a narrow diagnostics-only change that carries the original invalid config validation paths/messages into startup and reload recovery logs plus the main-agent recovery notice and troubleshooting docs, while preserving last-known-good recovery and strict schema behavior.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence reproduction path: make the reported invalid direct config edits, restart or reload the gateway with a last-known-good backup, and current main will restore the backup while emitting only generic recovery messages.

## Solution Assessment

Yes, the linked diagnostics-only direction is the narrowest maintainable fix. Accepting the unsupported config shapes or changing LAN/direct execution policy would be a broader product and security decision.
