## Summary

Keep open. Current main still builds the published gateway runtime from the slim Node image and the default runtime apt package list omits `python3`; the existing build-arg escape hatch only helps users who rebuild locally. The related open PRs address adjacent fs-safe fallback and `python` symlink behavior, not the reported published-image contract for user workspace Python scripts.

## What This Changes

Restore or explicitly document and diagnose `python3` availability for user-authored workspace Python scripts in published gateway Docker images after the slim-base regression.

## Best Possible Solution

Restore baseline `python3` in the published gateway runtime image, add a Dockerfile regression assertion and image smoke proof, and update Docker docs plus the changelog. If maintainers intentionally keep the image interpreter-free, the docs and `.py` execution failure should make that breaking change explicit and actionable.
