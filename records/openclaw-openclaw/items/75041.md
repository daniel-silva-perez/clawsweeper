## Summary

Keep open. Current main still builds the published gateway runtime from the slim Node base and the default runtime package list still omits `python3`; the documented apt hook only applies to local image builds. ## What This Changes

The issue asks OpenClaw to restore or document actionable support for Python-based workspace scripts in the published gateway Docker image after v2026.4.x slim images stopped shipping `python3`.

## Best Possible Solution

Restore baseline `python3` in the published gateway runtime image, add a Dockerfile regression assertion and image smoke proof, and update Docker docs plus the changelog. If maintainers intentionally keep the gateway image interpreter-free, the docs and missing-`python3` failure path should make that breaking change explicit and actionable.
