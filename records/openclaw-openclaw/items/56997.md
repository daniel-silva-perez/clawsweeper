## Summary

Keep this issue open. Current main has partial separated-session navigation in the Control UI: it can list and switch to individual session keys, including channel-derived sessions. It does not implement the requested channel-based view-mode toggle, and the chat history contract still loads one session at a time, so there is no merged all-channel timeline surface to close this as implemented.

## Best Possible Solution

Keep this issue open for a core Control UI and Gateway protocol design. The likely path is to preserve the existing per-session selector as the separated-session foundation, add an explicit separated/merged view mode in the Chat surface, and add a permission-aware gateway method or extension to aggregate selected channel session transcripts into a timestamped timeline with channel/account/source metadata. This belongs in OpenClaw core UI/protocol rather than ClawHub because it changes the built-in Control UI experience and cross-session transcript access.

## Risks / Open Questions

- A merged timeline has privacy and permission implications because it would intentionally cross currently isolated channel/group/room sessions; it should respect operator scopes and make channel provenance visible.
- There is already a session picker for single-session navigation, so the fix should extend that UX rather than replacing it with a duplicate session list.
