## Summary

Keep this issue open. Current main has the per-session building blocks: channel/group sessions are discoverable in `sessions.list`, the chat header can switch to one selected session key, and `chat.history` reads that one active session. It still does not provide the requested separated/merged Control UI mode toggle, channel tabs, or a Gateway history contract for an aggregated all-channel timeline.

## Best Possible Solution

Keep this open for a core Control UI and Gateway protocol design. Build on the existing session selector for separated mode, add an explicit separated/merged view toggle in the chat surface, and add an additive, permission-aware Gateway method or extension that can aggregate selected channel session transcripts into a timestamped timeline with channel/account/source metadata.
