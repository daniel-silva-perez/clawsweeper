## Summary

Keep this issue open. Current main supports listing and switching to individual session keys, including channel-derived sessions, but the Control UI still renders one active chat session at a time. There is no separated/merged view toggle and no Gateway chat-history contract for a unified all-channel timeline.

## Best Possible Solution

Keep this open for a core Control UI and Gateway protocol design. The best path is to preserve the existing per-session selector as the separated-session foundation, add an explicit separated/merged mode in the chat surface, and introduce a permission-aware Gateway method or extension that aggregates selected channel session transcripts into a timestamped timeline with channel/account/source metadata. This belongs in OpenClaw core UI/protocol rather than ClawHub because it changes the built-in Control UI and cross-session transcript access.

## Risks / Open Questions

- A merged timeline would intentionally cross currently isolated channel/group/room sessions, so the implementation should preserve operator-read authorization and expose channel/account/source provenance clearly.
- The existing single-session selector is useful separated-session groundwork; replacing it with a parallel list could regress current session navigation unless the new mode builds on it.
