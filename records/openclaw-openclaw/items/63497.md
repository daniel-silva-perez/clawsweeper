## Summary

Current main has shared media-reference helpers and a direct image-tool test for `media://inbound`, but the normal agent tool path still passes `workspaceDir` and `createImageTool` still rewrites `media://inbound/<id>` as a workspace-relative filesystem path before `loadWebMedia` can resolve the managed media URI. This PR or a rebased replacement remains relevant.

## Best Possible Solution

Keep this PR or a replacement fix open. Rebase it onto the current shared media-reference helpers and update `src/agents/tools/image-tool.ts` so managed media-store URLs are preserved or resolved before workspace-relative path handling, ideally by excluding `refInfo.isMediaStoreUrl` from the `workspaceDir` branch or by using `resolveMediaReferenceLocalPath`. Add regression coverage through `createImageTool` or `createOpenClawTools` with `workspaceDir` and `fsPolicy: { workspaceOnly: true }`, covering both `media://inbound/<id>` and `MEDIA:media://inbound/<id>`.
