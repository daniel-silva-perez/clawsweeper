## Summary

Keep open. Current main partially addresses the modality part for custom providers by inferring or asking about image input, but it still does not prompt for context window, max output tokens, or arbitrary modalities during the interactive OpenAI-compatible custom-provider onboarding flow. The related open issue #64213 covers runtime use of configured modality metadata, not this onboarding UX request.

## Best Possible Solution

Keep this issue open and implement it in the custom provider onboarding path: add optional validated interactive inputs for contextWindow and maxTokens, extend modality collection beyond the current image-only boolean if product wants audio/video metadata, pass those values through `ApplyCustomApiConfigParams`, mirror the behavior in non-interactive flags if appropriate, and update the onboarding docs/tests. Coordinate with #64213 so configured input metadata is honored at runtime.
