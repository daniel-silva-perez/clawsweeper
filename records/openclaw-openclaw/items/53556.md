## Summary

Keep open. Current `main` still only partially addresses the reported Feishu compatibility break: `groupPolicy: "allowall"` and legacy `topicSessionMode` have compatibility coverage, but legacy `botId` is not accepted or normalized through the Feishu schema, credential resolution, setup/status detection, doctor migration seam, manifest metadata, or docs. The linked #52885 is a closed Weixin/plugin-SDK export mismatch, not a canonical tracker for this Feishu config migration.

## What This Changes

The issue asks for backward-compatible channel configuration migration, especially Feishu `botId` to `appId`, `topicSessionMode` to `groupSessionScope`, and `groupPolicy: "allowall"` to `"open"`.

## Best Possible Solution

Feishu should accept legacy top-level and per-account `botId`, normalize it to canonical `appId` in a Feishu-owned compatibility path, expose a doctor migration that rewrites stored configs, keep the existing `allowall` and `topicSessionMode` behavior, and document the upgrade path on the Feishu channel docs.
