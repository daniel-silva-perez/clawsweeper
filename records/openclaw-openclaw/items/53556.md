## Summary

Keep open. Current main partially addresses the reported Feishu compatibility break by accepting `groupPolicy: "allowall"` and honoring legacy `topicSessionMode`, but legacy `botId` is still not accepted by the strict Feishu schema, credential resolution, setup/status detection, doctor migration, or docs. The linked #52885 is closed and covers a Weixin/plugin-SDK export mismatch, so it is not a canonical tracker for this Feishu config migration.

## What This Changes

Request backward-compatible channel config handling for Feishu legacy `botId`, `topicSessionMode`, and `allowall`, plus broader migration, versioning, validation, and documentation guidance for channel config changes.

## Best Possible Solution

Feishu should accept legacy top-level and per-account `botId`, normalize it to canonical `appId` through a Feishu-owned compatibility path, expose a doctor migration that rewrites stored configs, preserve the existing `allowall` and `topicSessionMode` behavior, and document the upgrade path on the Feishu channel docs.
