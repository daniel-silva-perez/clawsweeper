## Summary

Current main only partially implements the requested Feishu compatibility. Legacy `groupPolicy: "allowall"` and `topicSessionMode` handling exist, but `botId` to `appId` compatibility is still missing from Feishu config validation, credential/status resolution, metadata/docs, and doctor migration hooks. The linked #52885 is a closed Weixin plugin-SDK issue, not a canonical Feishu config migration tracker.

## Best Possible Solution

Keep this open and implement the remaining Feishu-owned compatibility: accept and normalize top-level and per-account `botId` to `appId`, preserve `appId` precedence when both are present, wire `openclaw doctor --fix` through Feishu `legacyConfigRules` / `normalizeCompatibilityConfig`, update setup/status and credential-resolution tests, and document the upgrade path on the public Feishu channel docs.
