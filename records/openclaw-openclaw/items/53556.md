## Summary

Keep open. Current main partially addresses the reported Feishu compatibility break by accepting `groupPolicy: "allowall"` and routing legacy `topicSessionMode`, but it still does not accept or normalize legacy `botId` through Feishu validation, credential resolution, setup/status detection, doctor migration, or docs. The linked #52885 is closed and covers a Weixin/plugin-SDK export problem, so it is not a canonical tracker for this Feishu config migration.

## What This Changes

The issue requests backward-compatible channel config migration for renamed Feishu fields such as `botId` to `appId`, `topicSessionMode` to `groupSessionScope`, and `groupPolicy: "allowall"` to `"open"`.

## Best Possible Solution

Feishu should accept legacy top-level and per-account `botId`, normalize it to `appId` through a Feishu-owned compatibility path, expose a doctor migration that rewrites stored configs, keep existing `allowall` and `topicSessionMode` behavior intact, and document the upgrade path on the Feishu channel docs.

## Repair Work Prompt

Fix Feishu legacy config compatibility for `botId` from this issue. Current main already handles `groupPolicy: "allowall"` and legacy `topicSessionMode`, but legacy top-level and per-account `channels.feishu.botId` are still rejected or ignored because the strict Feishu schema and runtime/setup credential paths only know `appId`.

Expected boundary:
- Add Feishu-owned compatibility for `botId` -> `appId` at top level and under `channels.feishu.accounts.<id>`.
- Preserve `appId` precedence when both `appId` and `botId` are present.
- Make runtime validation/credential resolution/setup status work for legacy configs, and add a doctor compatibility migration that rewrites stored `botId` keys to canonical `appId` with clear change messages.
- Prefer the existing bundled channel doctor contract seam (`doctor-contract-api.ts` / `contract-api.ts`) rather than adding Feishu-specific startup-time core migration behavior.
- Keep the existing `allowall` and `topicSessionMode` behavior intact.
- Update Feishu docs at `docs/channels/feishu.md` with the upgrade note and add a user-facing changelog entry crediting `@webzol`; do not treat linked #52885 as the canonical Feishu fix because it is a closed Weixin/plugin-SDK issue.

Likely files: `extensions/feishu/src/config-schema.ts`, `extensions/feishu/src/accounts.ts`, `extensions/feishu/src/setup-surface.ts`, new or updated Feishu doctor contract files such as `extensions/feishu/src/doctor-contract.ts`, `extensions/feishu/doctor-contract-api.ts`, `extensions/feishu/contract-api.ts`, Feishu tests, `docs/channels/feishu.md`, and `CHANGELOG.md`.
