## Summary

Keep open. Current main still only partially covers the reported Feishu compatibility break: `groupPolicy: "allowall"` and legacy `topicSessionMode` are handled, but legacy `botId` is not accepted or normalized in the Feishu schema, credential resolver, setup/status detection, manifest metadata, doctor repair path, or docs. The linked #52885 is a closed Weixin/plugin-SDK compatibility issue, not a canonical tracker for this Feishu config migration.

## What This Changes

The issue requests backward-compatible channel configuration migration, especially Feishu legacy `botId`, `topicSessionMode`, and `groupPolicy: "allowall"` configs after channel/plugin changes.

## Best Possible Solution

Add a Feishu-owned compatibility path that accepts legacy top-level and per-account `botId`, normalizes it to canonical `appId`, exposes an `openclaw doctor --fix` migration for stored configs, keeps the existing `allowall` and `topicSessionMode` behavior intact, and documents the upgrade path without adding startup-time core migrations.
