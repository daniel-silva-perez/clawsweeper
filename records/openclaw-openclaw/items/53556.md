## Summary

Keep this open. Current main partially covers the reported Feishu compatibility break: `groupPolicy: "allowall"` and legacy `topicSessionMode` are handled, but legacy `botId` is still not accepted or normalized through Feishu validation, credential resolution, setup/status, doctor migration, or docs. The linked #52885 is closed and tracks a Weixin plugin-SDK export problem, so it is not a canonical tracker for this Feishu config migration.

## What This Changes

The issue requests backward-compatible migration and normalization for legacy channel config fields, especially Feishu `botId`, `topicSessionMode`, and `groupPolicy` values after channel/plugin changes.

## Best Possible Solution

Feishu should accept legacy top-level and per-account `botId`, normalize it to `appId` through the existing doctor/channel compatibility seam, keep direct `appId` authoritative when both names are present, and document the upgrade path on the public Feishu channel docs.
