## Summary

Keep open. Current main still matches the reported routing leak, and the provided GitHub context shows an open closing PR (#73968) that specifically targets this bug, so this issue should remain open until that PR is reviewed and merged or replaced.

## What This Changes

A main-session cron `systemEvent` heartbeat wake should route replies to the bound Telegram topic session instead of inheriting a global `heartbeat.to` DM target.

## Best Possible Solution

Review and land or replace #73968 with a narrow fix that prevents cron-forced `target: "last"` wakes from inheriting explicit destination fields such as `to`, while preserving normal explicit heartbeat targeting and adding a regression for a Telegram forum-topic session plus global `heartbeat.to`.

## Reproduction Assessment

Yes. Code inspection gives a high-confidence static reproduction path: configure global `agents.defaults.heartbeat.to`, run a `sessionTarget: "main"` systemEvent cron job bound to a topic session, and the cron heartbeat override is shallow-merged before outbound delivery treats the inherited `to` as explicit.

## Solution Assessment

Yes. Clearing or non-inheriting explicit destination fields for cron-supplied `target: "last"` is the narrowest maintainable direction; changing all heartbeat `last` routing semantics would be riskier.
