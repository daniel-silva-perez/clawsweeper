## Summary

Keep open. Current `main` still has the shallow heartbeat-default merge that can preserve an inherited `heartbeat.to` while cron forces `target: "last"`, and the issue is paired with open PR #73968, which should be reviewed, landed, replaced, or closed before this issue is closed.

## What This Changes

The issue requests fixing main-session `systemEvent` cron wakes so a Telegram forum-topic reminder routes to the bound session instead of inheriting a global DM `heartbeat.to` destination.

## Best Possible Solution

Keep this issue open and land #73968 or an equivalent narrow fix that clears inherited explicit destination fields for cron-forced `target: "last"` main-session `systemEvent` wakes, with a regression test proving a topic-bound session stays on its group/topic route when global heartbeat defaults include a DM `to`.

## Reproduction Assessment

Yes. The issue provides concrete config, cron payload, session key, and observed logs; current `main` also statically reproduces the merge and delivery-resolution path without needing a live Telegram run.

## Solution Assessment

Yes. Clearing inherited explicit destination fields only for cron-forced `target: "last"` wakes is the narrowest maintainable fix; changing general heartbeat `target: "last"` semantics would be broader and riskier.
