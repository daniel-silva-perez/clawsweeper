## Summary

Keep open. Current main still shallow-merges cron-forced `heartbeat.target = "last"` over resolved heartbeat defaults, so an inherited global `heartbeat.to` can remain explicit and win over the bound session route; the issue is also paired with open PR #73968, which should be reviewed or replaced before closing this issue.

## What This Changes

A `sessionTarget: "main"` systemEvent cron wake should route reminder replies to the bound Telegram forum topic instead of inheriting a global DM `heartbeat.to` destination.

## Best Possible Solution

Keep this issue open and land #73968 or an equivalent narrow fix that clears inherited explicit destination fields for cron-forced `target: "last"` main/systemEvent wakes, with a regression test proving a Telegram forum-topic session stays on the bound group/topic route when global heartbeat defaults include a DM `to`.

## Reproduction Assessment

Yes. The issue provides a concrete config, cron payload, session key, and observed route leak, and the current source statically reproduces the merge and delivery-resolution path without needing a live Telegram run.

## Solution Assessment

Yes. Clearing inherited explicit destination fields only for cron-forced `target: "last"` wakes is the narrow maintainable fix; changing general heartbeat `target: "last"` semantics globally would be broader and riskier.
