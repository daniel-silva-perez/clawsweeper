## Summary

Keep this open for maintainer follow-up. Current main and the v2026.4.26 tag have a Telegram transport path that should honor HTTP_PROXY/HTTPS_PROXY via Undici EnvHttpProxyAgent, and docs already expose channels.telegram.proxy. However, the report also names ALL_PROXY, and current Telegram tests explicitly treat ALL_PROXY-only environments as direct transport, so the reported proxy regression is not fully resolved or confidently dismissible from static review alone.

## Best Possible Solution

Keep the issue open and have a maintainer reproduce with a local proxy against current main and v2026.4.26 while confirming the gateway process actually sees HTTP_PROXY/HTTPS_PROXY/ALL_PROXY and NO_PROXY. If ALL_PROXY is intended to work for Telegram, add a narrow Telegram transport change to pass explicit EnvHttpProxyAgent options, update SDK exports if needed, add uppercase HTTP_PROXY and ALL_PROXY regression tests, and document the supported env vars alongside channels.telegram.proxy.
