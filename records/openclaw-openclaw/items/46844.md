## Summary

Keep open. Current main still lacks the requested between-turn Talk Mode idle timeout: Voice Wake can activate Talk Mode directly, but Talk only uses the short transcript-finalization silence window and then resumes listening after replies or failures. The related heartbeat/session-cleanup reports are broader lifecycle requests, not a replacement for this Talk-specific wake path.

## What This Changes

Add a configurable Talk Mode idle timeout that deactivates idle Voice Wake-triggered Talk sessions after a period with no user or assistant interaction.

## Best Possible Solution

Add an additive Talk lifecycle setting that is validated in config, included in talk.config, documented on the public Talk page, and honored by macOS Talk so idle sessions return to wake listening while silenceTimeoutMs remains scoped to transcript finalization.
