## Summary

Keep open. Current main still only has the short intra-turn Talk silence window and does not expose a public config/protocol field or macOS lifecycle timer that deactivates Talk Mode after between-turn idle time. The related heartbeat/session-cleanup issues do not cover this Voice Wake-triggered Talk lifecycle gap, and no open PR appears to be paired with this issue.

## What This Changes

Add a configurable Talk Mode idle timeout that automatically deactivates Voice Wake-started Talk sessions after a period without user speech or assistant replies.

## Best Possible Solution

Add an additive Talk idle-timeout config and gateway protocol field, document it on the public Talk docs, and implement a macOS lifecycle timer that resets on user speech and assistant reply while keeping `silenceTimeoutMs` strictly for transcript-finalization pauses.
