## Summary

Keep open. Current `main` still lacks a configurable between-turn Talk Mode idle timeout: Voice Wake can activate Talk Mode, but the Talk runtime only uses the short silence window to finalize a non-empty transcript and then resumes listening after replies or failures. The related idle-timeout reports are broader session/heartbeat lifecycle work, not a replacement for this Talk-specific wake path.

## What This Changes

The issue requests a configurable Talk Mode idle timeout so Voice Wake-triggered Talk sessions auto-deactivate after no user interaction instead of staying active until manual exit or the hard stop.

## Best Possible Solution

Add an additive Talk lifecycle setting that is validated in config, exposed through `talk.config`, documented in the public Talk docs, and honored by macOS Talk so idle Voice Wake-triggered sessions return to wake listening while `silenceTimeoutMs` remains scoped to transcript finalization.
