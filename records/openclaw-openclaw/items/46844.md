## Summary

Keep open. Current main still lacks a between-turn Talk idle timeout: Voice Wake can activate Talk Mode directly, but Talk Mode only uses the short transcript-finalization silence window and then resumes listening after replies or failures. Related idle-timeout reports are either broader lifecycle work or older closed triage, not a shipped fix for this Voice Wake-triggered Talk gap.

## What This Changes

The issue requests a configurable Talk Mode idle timeout that automatically deactivates Voice Wake-triggered Talk sessions after a period with no user interaction.

## Best Possible Solution

Add an additive Talk lifecycle setting that is validated in config, returned through talk.config, documented on https://docs.openclaw.ai/nodes/talk, and honored by macOS Talk so idle sessions return to wake listening while silenceTimeoutMs remains scoped to transcript-finalization pauses.
