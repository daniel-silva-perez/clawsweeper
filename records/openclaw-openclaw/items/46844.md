## Summary

Keep open. Current `main` still lacks a configurable between-turn idle timeout for Voice Wake-triggered Talk Mode; Talk only finalizes non-empty transcripts after the short `silenceTimeoutMs` window and then resumes listening after replies. The related heartbeat/session issues do not supersede this macOS Talk lifecycle request.

## What This Changes

Add a configurable Talk Mode idle timeout that auto-deactivates Voice Wake-triggered Talk sessions after a period with no user interaction.

## Best Possible Solution

Add an additive Talk idle-timeout config/protocol field, document it on the public Talk docs, and implement a macOS Talk lifecycle timer that resets on user speech and assistant reply while preserving `silenceTimeoutMs` as the intra-turn transcript-finalization pause.
