## Summary

Keep open. Current main still lacks a configurable between-turn idle timeout for Voice Wake-triggered macOS Talk Mode; the implementation only uses `silenceTimeoutMs` to finalize a non-empty transcript within a turn and then resumes listening after replies.

## What This Changes

Add a configurable idle timeout for Voice Wake-triggered Talk Mode that deactivates after no between-turn user interaction, separate from `silenceTimeoutMs` and the Voice Wake hard stop.

## Best Possible Solution

Keep this issue open. A maintainer should decide the field name, default, units, disabled behavior, and whether the timer applies only to Voice Wake-triggered Talk Mode or all Talk Mode sessions; then implement an additive config/protocol/docs/test update plus a macOS Talk lifecycle timer that deactivates Talk Mode after between-turn inactivity without changing `silenceTimeoutMs` semantics.
