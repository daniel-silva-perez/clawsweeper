## Summary

Current main still lacks the requested Talk Mode idle auto-deactivation. Voice Wake can activate Talk Mode and pause the wake listener, Talk Mode resumes listening after each turn while enabled, and the only Talk timing config is the intra-turn `silenceTimeoutMs`, not a between-turn idle timeout.

## Best Possible Solution

Keep this issue open and implement a first-class Talk idle timeout: add a validated `talk.idleTimeoutSeconds` or similar config field, parse it through shared Talk config, implement a between-turn idle timer in macOS/iOS/Android Talk runtimes that resets on user speech and assistant playback completion, disables Talk Mode on expiry, resumes Voice Wake where appropriate, and document it on the public Talk Mode docs page.

## Evidence
