## Summary

Keep open. Current main still lacks the requested between-turn Talk Mode idle timeout: Voice Wake can activate Talk Mode directly, but Talk only uses the short transcript-finalization silence window and then resumes listening after replies or failures. The related heartbeat/session-cleanup reports are broader lifecycle requests and do not replace this Talk-specific wake path.

## What This Changes

Add a configurable idle timeout that automatically deactivates Talk Mode after Voice Wake-triggered sessions sit idle between user interactions.

## Best Possible Solution

Add an additive Talk lifecycle setting that is validated in config, included in the gateway talk.config payload, documented on https://docs.openclaw.ai/nodes/talk, and honored by macOS Talk so idle sessions return to wake listening while silenceTimeoutMs remains scoped to transcript finalization.
