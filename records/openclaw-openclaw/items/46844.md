## Summary

Keep open. Current main still lacks a between-turn Talk Mode idle timeout for the Voice Wake-triggered Talk path: the runtime only finalizes non-empty transcripts after the short silence window, then resumes listening after replies or errors, and the public Talk config/protocol/docs expose no idle-timeout setting. The related idle-timeout reports are broader heartbeat/session lifecycle items, and the merged Voice Wake-to-Talk PR introduced the activation path without adding idle auto-deactivation.

## What This Changes

Request to add a configurable idle timeout that automatically deactivates Voice Wake-triggered Talk Mode after no user interaction between turns.

## Best Possible Solution

Add an additive Talk lifecycle setting with maintainer-approved naming, units, default, bounds, disabled behavior, and reset events; validate and normalize it in config, expose it through talk.config, document it on the public Talk docs, and honor it in macOS Talk so idle Voice Wake-triggered sessions return to wake listening while silenceTimeoutMs remains scoped to transcript finalization.
