## Summary

Keep open. Current main still lacks a between-turn Talk idle timeout: Voice Wake can activate Talk Mode, but the macOS Talk runtime only has the short transcript-finalization silence window and then resumes listening after replies or errors. The related heartbeat/session-cleanup reports are broader agent/session lifecycle requests, not a replacement for this Voice Wake-triggered Talk lifecycle gap, and a current search found no open PR referencing this issue.

## What This Changes

Request a configurable Talk Mode idle timeout that deactivates Talk after a period of no user interaction following Voice Wake activation.

## Best Possible Solution

Add an additive Talk lifecycle setting that is validated in config, exposed through talk.config, documented on the public Talk docs, and honored by macOS Talk so idle sessions return to wake listening while keeping silenceTimeoutMs scoped to transcript-finalization pauses.
