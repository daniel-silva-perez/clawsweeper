## Summary

Keep open. Current main still lacks a configurable between-turn idle timeout for macOS Voice Wake-triggered Talk Mode; the implemented behavior only finalizes a non-empty transcript after the intra-turn silence window, resumes listening after replies, and exposes only `talk.silenceTimeoutMs` in config/schema/docs.

## Best Possible Solution

Keep this issue open. A maintainer should decide the field name, default, and whether the timeout applies only to Voice Wake-triggered Talk sessions or all Talk Mode sessions, then implement an additive Talk idle-timeout config and macOS lifecycle timer that preserves `silenceTimeoutMs` as the intra-turn pause window and keeps the Voice Wake 120s capture hard stop separate.
