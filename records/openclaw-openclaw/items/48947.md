## Summary

Keep this issue open. Current main contains useful partial fixes for live Control UI tool-card updates, enabled loop detection for repeated exec outcomes, unknown-tool loops, and null argument normalization, but it still does not implement a default-safe runaway tool-call budget or forced progress/failure summary for the reported default web UI/OpenRouter path. The earlier ClawSweeper keep-open review remains accurate on the current main SHA; the relevant files have not materially changed since that review.

## Best Possible Solution

Keep the issue open and narrow it into an actionable runtime/UI reliability fix: add a default-safe runaway guard or documented per-agent tool-call budget, force a user-visible progress or failure summary after repeated tool cycles without assistant-visible progress, and verify the Control UI refresh path with a current web UI/OpenRouter reproduction. Treat #34574, #72587, and #67401 as partial mitigations, not closure for this broader report.
