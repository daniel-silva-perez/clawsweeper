## Summary

Keep this issue open. Current main still evaluates cron expressions with the supplied timezone at runtime, but the reported bug is model-facing argument selection: the cron tool schema and tool description still do not tell the model that a cron expression paired with `tz` must use that timezone's local wall-clock time, not a UTC-converted hour.

## Best Possible Solution

Make a narrow model-facing fix in `src/agents/tools/cron-tool.ts`: for `schedule.kind="cron"`, say `expr` is written in the supplied `tz` timezone's local wall-clock time and must not be converted to UTC first. Include the reported example, such as 6pm Shanghai -> `expr: "0 18 * * *"`, `tz: "Asia/Shanghai"`, and add focused schema/tool coverage in `src/agents/tools/cron-tool.schema.test.ts` or the existing cron tool tests.
