## Summary

Keep this PR open. Current main still has the exact outer Promise.race around the Tailscale binary probe, so the PR’s intended change is not implemented on main. The patch is narrow and still plausibly useful: runExec already forwards timeoutMs into Node execFile, whose timeout path kills the child process, while the current outer timeout can resolve first and leave the execFile promise running in the background.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land the scoped removal of the outer Promise.race, preferably with a small regression test or helper-level proof that the binary probe relies on runExec/execFile timeout cleanup and still returns false on timeout.
