## Summary

Keep this PR open. Current main still has the redundant outer timeout in the Tailscale binary probe, while the PR is a narrow one-file cleanup that appears correct and has green remote checks; the remaining action is normal maintainer review rather than closure or an automated replacement.

## What This Changes

This PR removes the separate `Promise.race` timeout around the `tailscale --version` binary probe and relies on `runExec(path, ["--version"], { timeoutMs: 3000 })` instead.

## Best Possible Solution

Land the narrow cleanup after maintainer review and green CI, with an optional focused regression test around `findTailscaleBinary` timeout ownership if maintainers want executable coverage for this lifecycle edge case.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: `checkBinary` races an independent timer against `runExec`, so the helper can return from the outer timer while the child process promise continues in the background. A deterministic process-table reproduction would need a controlled hanging `tailscale --version` candidate and tight timing.

## Solution Assessment

Yes. For the reported lifecycle race, removing the outer `Promise.race` is the narrowest maintainable fix because it preserves the timeout value and catch behavior while letting `runExec`/`execFile` own child termination. If maintainers want protection against SIGTERM-resistant binaries, that should be handled as a separate `runExec` kill-escalation improvement.
