## Summary

Keep this PR open. Current main still has the older suffix-revealing `maskApiKey` behavior, so the intended change is not implemented on main. The PR is a plausible active fix for the credential-display problem, but it needs revision before merge because its proposed `> 4` branch increases exposure for 5- and 6-character secrets.

## Best Possible Solution

Keep this PR open and request a small revision before merge. Maintainers should choose a credential display policy that never exposes suffixes and does not increase exposure for 5- or 6-character secrets, update `maskApiKey`, move the misplaced `" short "` assertion, add explicit 1-6 character and suffix-nondisclosure regression coverage, then run the targeted mask-api-key and models auth overview tests.
