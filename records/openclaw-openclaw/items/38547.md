## Summary

Keep this open. Current main still does not implement native remote compute delegation for `sessions_spawn` or `sessions_send`: the session tool schemas omit `gatewayUrl`/`gatewayToken`, native and ACP spawn contexts do not carry a remote target, dispatch calls the current gateway, and the documented sub-agent model still shares the same gateway process resources. The existing gateway override helper is present for other gateway-backed tools, but it is not wired into these session tools.

## Best Possible Solution

Keep this issue open for maintainer design. The best path is to define an explicit remote target contract for `sessions_spawn` and `sessions_send`, reuse the existing safe gateway override and credential-resolution seam, decide remote credential/workspace/transcript ownership, and add targeted tests proving child work executes on the remote gateway instead of only routing metadata locally.
