## Summary

Keep this issue open. Current main still does not expose or thread `gatewayUrl`/`gatewayToken` through `sessions_spawn` or `sessions_send`, and the native subagent/ACP dispatch paths still call the current gateway without URL/token override options. The request is concrete and not superseded by the title-overlap related items in the provided context.

## Best Possible Solution

Keep this issue open until maintainers either design and implement native remote session delegation or explicitly reject it. A good implementation would add documented `gatewayUrl`/`gatewayToken` support to `sessions_spawn` and `sessions_send`, reuse the existing safe gateway override validation and credential resolution, thread resolved remote call options through native subagent, ACP spawn, send, wait/history, cleanup, and registry paths, and add tests proving model calls, tool execution, transcripts, filesystem effects, and credential/account use occur on the remote gateway.
