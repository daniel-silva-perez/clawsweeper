## Summary

Keep this issue open. Current main at ab5c8025c9d0a31873d311ae3e73f3664b0a1185 still does not expose or thread `gatewayUrl`/`gatewayToken` through `sessions_spawn` or `sessions_send`; native subagent and ACP dispatch still call the current gateway. The prior ClawSweeper review remains accurate because the relevant files have not changed since that review SHA.

## Best Possible Solution

Keep this open for maintainer design. The best path is to explicitly design native remote session delegation for `sessions_spawn` and `sessions_send`, reuse the existing safe gateway override and credential-resolution seam, define how remote model/account/filesystem/transcript ownership is selected, and add tests that prove work executes on the remote gateway rather than only routing metadata locally.
