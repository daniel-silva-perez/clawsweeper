## Summary

Keep this PR open. The reported auth-health pollution is real, but the proposed policy narrows too far: it also suppresses OpenAI Codex WHAM cooldown probes for ambiguous failures that current main intentionally routes through auth-profile failure handling.

## What This Changes

The PR changes `resolveAuthProfileFailureReason` to cooldown profiles only for `auth`, `auth_permanent`, `billing`, `rate_limit`, and `session_expired`, with focused helper tests for non-auth and positive reasons.

## Best Possible Solution

Keep the narrower auth-health policy direction, but preserve provider-owned health probes such as OpenAI Codex WHAM for ambiguous failures. Add regression coverage that proves non-auth Copilot format failures do not cooldown shared profiles while Codex ambiguous failures still reach WHAM-based cooldown handling.

## Reproduction Assessment

Yes. The source path is high-confidence: a non-null policy reason reaches `markAuthProfileFailure`, while a null reason returns early; existing Codex tests show `unknown` is an intentional cooldown-probe input.

## Solution Assessment

No. The generic narrowing is maintainable, but this patch drops ambiguous reasons before provider-specific cooldown logic can classify real Codex profile exhaustion; a safer fix needs provider context or a separate probe path.
