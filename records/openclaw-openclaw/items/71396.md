## Summary

Keep this PR open. Current main still has the Feishu automatic mention cascade this PR targets, but the latest branch still needs another pass because it removes inherited mention dispatch without preserving a reliable explicit-mention path for card, streaming-card, and post/rich-text flows; the PR discussion also has an active maintainer request for Feishu-owner follow-up.

## What This Changes

This PR changes the Feishu plugin to stop passing inbound `mentionTargets` into outbound reply dispatch, rewrites the agent prompt to treat mentioned users as context only with display-name hardening, adds Feishu regression tests, and adds a changelog entry.

## Best Possible Solution

Land a Feishu-owned fix that disables unconditional inbound `mentionTargets` propagation for fallback, error, ack, ordinary, card, and streaming replies while preserving intentional explicit mentions through stable `{openId, name}` context and a safe allowlisted conversion path for text/card delivery, backed by focused regression tests and the existing changelog entry.
