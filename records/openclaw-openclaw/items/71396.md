## Summary

Keep this PR open. Current main still has the Feishu mention cascade: inbound mentionTargets are captured, passed into reply dispatch, and converted into outbound Feishu @mention markup. The PR is still an open, unmerged implementation candidate for linked bug #70065, with recent author follow-up and a small security/behavior review surface that needs maintainer judgment.

## Best Possible Solution

Keep this PR open for Feishu plugin maintainer review. The best path is to land or revise a focused Feishu plugin fix that stops unconditional inbound mention propagation for fallback, error, ack, plain-text, card, and streaming-card replies; keeps mention context sanitized and context-only for the agent; and either preserves intentional explicit @mention conversion where needed or documents/tests the new behavior.
