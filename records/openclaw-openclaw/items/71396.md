## Summary

Keep this PR open. Current main still captures inbound Feishu `mentionTargets`, tells the agent the reply will automatically mention those users, forwards those targets into reply dispatch, and converts them into outbound Feishu mention markup. The PR remains a focused implementation candidate for linked bug #70065 and has recent author follow-up plus a maintainer ping rather than abandonment.

## Best Possible Solution

Keep this PR open for Feishu plugin maintainer review. The best path is to land or revise a focused Feishu plugin fix that stops unconditional `mentionTargets` propagation for fallback, error, ack, ordinary plain-text, card, and streaming-card replies; keeps mentioned users as sanitized context only for the agent; makes an intentional decision on explicit mention conversion in card/streaming replies; and either fixes or explicitly tracks the remaining mention display-name logging hardening.
