## Summary

Keep open. Current main still has the Feishu mention cascade the PR targets, so the PR is not obsolete; however, the latest branch still appears to remove the remaining stable-ID conversion path for intentional card/post mentions, and maintainer discussion explicitly asks for one more Feishu-owner pass before preparing it.

## What This Changes

The PR removes Feishu inbound `mentionTargets` from outbound reply dispatch, changes the agent prompt to present mentions as sanitized context only, adds text/card/streaming-card regression tests, and adds a changelog entry for #70065.

## Best Possible Solution

Land a Feishu-owned revision that disables unconditional inbound `mentionTargets` propagation for fallback, error, ack, ordinary, card, and streaming replies while preserving intentional mentions through allowlisted stable `{openId, name}` context and a safe explicit-only text/card conversion path, with focused regression coverage and the changelog entry retained.

## Reproduction Assessment

Yes. A high-confidence reproduction path exists by sending a Feishu group message that mentions multiple users and then triggers a fallback/ack/reply: current main statically passes `ctx.mentionTargets` into the dispatcher, and the dispatcher forwards them into Feishu send helpers that emit real mention markup.

## Solution Assessment

No. The PR is aimed at the right Feishu-owned boundary, but the current patch removes the automatic cascade by dropping mention metadata wholesale; the safer solution is to stop inherited propagation while preserving explicit mentions through stable IDs and allowlisted conversion.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve stable mention IDs in context:** `extensions/feishu/src/bot.ts:312-314`
  - body: After dispatcher-side mention injection is removed, explicit mentions have to be reconstructed from agent-visible context, but this hint serializes only display names. In post/rich-text inputs the model can see only `@name` while the stable open ID stays internal, so intentional mention-forward replies can degrade to plain text or target the wrong duplicate name.
  - confidence: 0.86
- **[P2] Keep card mention conversion for explicit replies:** `extensions/feishu/src/reply-dispatcher.ts:528-535`
  - body: Card dispatch now sends chunks without mention metadata, and streaming-card close sends raw combined text. Feishu card mentions need the `<at id=...></at>` form produced from stable open IDs, so explicit mention replies in card mode can stop notifying users unless an explicit-only rewrite path replaces the removed automatic prefixing.
  - confidence: 0.84
