## Summary

Keep open. Current main still has the Feishu mention-cascade behavior, and the latest PR branch remains an incomplete fix because it removes the existing explicit mention conversion path for card, streaming-card, and post/rich-text flows. The PR discussion now has an explicit maintainer handoff and the author agreed to pause before preparation, so this needs Feishu maintainer follow-up rather than cleanup closure.

## What This Changes

The branch removes Feishu `mentionTargets` from outbound reply dispatch, changes the agent mention hint to context-only sanitized names, adds plain/card/streaming-card regression tests, and adds a changelog entry.

## Best Possible Solution

Land a Feishu-owned fix that disables unconditional inherited `mentionTargets` propagation for all reply paths while preserving intentional explicit mentions through stable allowlisted `{openId, name}` context and a safe explicit-only conversion path for text/card delivery, with the focused regression tests and changelog entry retained.

## Reproduction Assessment

Yes. The source-level reproduction path is high confidence: current main captures inbound `mentionTargets`, passes them into the Feishu reply dispatcher, and the dispatcher converts them into real Feishu mention markup on first text/card chunks and streaming-card close.

## Solution Assessment

No. Removing automatic inherited mention propagation is the right direction, but the latest patch is not the best complete fix because it drops the remaining stable-ID card/streaming mention conversion path instead of replacing it with explicit-only conversion.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve stable IDs in mention context:** `extensions/feishu/src/bot.ts:316`
  - body: After dispatcher-side mention injection is removed, explicit Feishu mentions depend on agent-visible context. This hint only provides display names, while post/rich-text mention parsing keeps `openId` internal and renders `@name`, so duplicate or changed names cannot be converted into a real notifying mention. Include an allowlisted stable mapping before relying on explicit mentions.
  - confidence: 0.86
- **[P2] Keep explicit card mention conversion:** `extensions/feishu/src/reply-dispatcher.ts:528-535`
  - body: Card replies now call `sendStructuredCardFeishu` without mention metadata, and streaming-card close sends raw combined text. That removes the path that turns `MentionTarget.openId` into card-compatible `<at id=...></at>` markup, so explicit mention-forward replies in card or streaming mode can render as plain text. Add an explicit-only rewrite path instead of deleting all card mention conversion.
  - confidence: 0.84
