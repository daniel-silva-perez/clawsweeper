## Summary

Keep open. Current main still has the Feishu mention-cascade path, while the latest PR head is an active fix candidate with a real but incomplete behavior change: it stops inherited outbound mentions, but still drops the stable-ID conversion path needed for intentional Feishu mentions in card, streaming-card, and post/rich-text flows. The PR discussion already has maintainer feedback asking for one more Feishu-owner pass, and the author agreed to pause before preparation.

## What This Changes

The PR removes inherited Feishu `mentionTargets` from reply dispatch, changes the agent prompt to sanitized context-only mention names, adds regression tests for no automatic mentions, and adds a changelog entry for the Feishu cascade bug.

## Best Possible Solution

Land a Feishu-owned fix that disables unconditional inherited mention propagation for fallback, error, ack, ordinary, card, and streaming replies while preserving intentional explicit mentions through allowlisted stable target context and a safe explicit-only conversion path for text/card delivery, with focused regression coverage and the changelog entry retained.

## Reproduction Assessment

Yes. The source path on current main is high-confidence: incoming Feishu mentions become `ctx.mentionTargets`, are passed into reply dispatch, and are converted into Feishu mention markup on first outbound text/card chunks and streaming-card close.

## Solution Assessment

No. The PR is directionally right for the cascade, but removing dispatcher mention metadata without replacing explicit-only stable-ID conversion is not the narrowest complete fix for the Feishu mention contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve stable mention IDs in context:** `extensions/feishu/src/bot.ts:316`
  - body: After this removes dispatcher-side mention injection, explicit mentions have to be reconstructed from agent-visible context. The new hint only exposes sanitized display names, while post/rich-text inputs can render mentions as `@name` with the stable open ID kept internal, so intentional mention replies can fail or target duplicate names. Include allowlisted `{openId, name}` context or an equivalent internal mapping before relying on model-authored explicit mentions.
  - confidence: 0.86
- **[P2] Keep card mention conversion for explicit replies:** `extensions/feishu/src/reply-dispatcher.ts:528-535`
  - body: Card sends now call `sendStructuredCardFeishu` without mention metadata, and streaming-card close sends raw combined text. Feishu card mentions require the card form produced from stable open IDs, so intentional mention replies in card/streaming mode can render as plain text and stop notifying users unless a safe explicit-only rewrite path replaces the removed automatic prefixing.
  - confidence: 0.84
