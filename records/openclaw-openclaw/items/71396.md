## Summary

Keep this PR open. Current main still has the Feishu mention-cascade behavior the PR targets, but the latest branch remains an active implementation with unresolved card/post explicit-mention semantics and a Feishu maintainer handoff in the discussion, so it should not be closed as obsolete or merged without another pass.

## What This Changes

The PR removes inbound Feishu `mentionTargets` from reply-dispatcher delivery, rewrites the agent mention hint as context-only with display-name hardening, adds Feishu regression tests, and adds a changelog entry for the mention-cascade bug.

## Best Possible Solution

Land a Feishu-owned fix that disables unconditional inbound mention propagation for fallback, error, ack, ordinary, card, and streaming replies while preserving intentional explicit mentions through stable allowlisted `{openId, name}` context and a safe explicit-only text/card conversion path, backed by focused regression coverage and the changelog entry.

## Reproduction Assessment

Yes. The source path on current main is high-confidence: incoming Feishu mentions become `ctx.mentionTargets`, are passed into reply dispatch, and are converted into Feishu `<at>` markup on first outbound text/card chunks and streaming-card close.

## Solution Assessment

No. The PR is directionally right for the cascade, but removing dispatcher mention metadata without replacing explicit-only stable-ID/card conversion is not the best complete fix for the Feishu mention contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve stable Feishu mention IDs:** `extensions/feishu/src/bot.ts:316`
  - body: After dispatcher-side mention injection is removed, explicit mentions have to come from agent-visible context. The new hint only exposes sanitized display names, while post/rich-text inputs can render mentions as `@name` with the stable `open_id` kept internal, so intentional mention-forward replies can fail or target duplicate names. Include allowlisted `{openId, name}` context before relying on model-authored explicit mentions.
  - confidence: 0.86
- **[P2] Keep explicit card mention conversion:** `extensions/feishu/src/reply-dispatcher.ts:528-535`
  - body: Card sends no longer pass mention metadata, and streaming-card close no longer rewrites mention targets before delivery. Feishu cards require the card form produced from stable open IDs (`<at id=...></at>`), so intentional mention replies in card/streaming mode can render as plain text unless a safe explicit-only rewrite path replaces the removed automatic prefixing.
  - confidence: 0.84
