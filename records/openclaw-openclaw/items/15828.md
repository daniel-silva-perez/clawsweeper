## Summary

Keep open. Current main still keeps bundled session-memory on the explicit command path only; idle/daily rollover and gateway shutdown infrastructure exists, but it does not invoke the bundled hook, and the requested message-count and summary-output config is still absent. The active related daily/idle PRs are partial, open implementation candidates rather than a merged fix for this broader feature request.

## What This Changes

The issue requests automatic session-memory checkpoints on idle, shutdown, periodic message counts, and optional summarized output instead of only command-triggered saves.

## Best Possible Solution

Keep this as the broad session-memory automation tracker. Maintainers should choose one canonical daily/idle implementation, then scope shutdown checkpointing, message-count checkpoints, trigger config, and summary-output semantics with docs and focused tests before closing the issue.
