## Summary

Keep open. Current main still has the broader session-memory automation gap: the bundled hook is command-only, while idle/daily rollover and gateway shutdown infrastructure exist on separate paths that do not make session-memory checkpoint automatically. The active related PRs cover only part of the idle/daily path and are still open, so they are not a merged fix for shutdown, periodic message checkpoints, or summary-output configuration.

## What This Changes

Request to extend bundled session-memory so it can checkpoint automatically on idle, shutdown, and periodic message-count triggers, with optional summarized output instead of raw transcript excerpts.

## Best Possible Solution

Keep this as the broad session-memory automation tracker. The end state should be one consolidated implementation that wires session-memory into agreed lifecycle triggers, documents the config contract, and covers idle/daily, shutdown, periodic checkpoint, and summary-output behavior with focused tests.
