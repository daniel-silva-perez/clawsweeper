Keep open. Current main still implements bundled session-memory persistence only through explicit `/new` and `/reset` command events; adjacent idle/daily rollover and gateway shutdown hooks exist, but they do not invoke this bundled hook, and the active related PRs remain partial and unmerged.

## What This Changes

Requests automatic session-memory checkpoints on idle, shutdown, periodic message counts, and optional summarized output instead of only command-triggered saves.

## Best Possible Solution

Keep this as the broad session-memory automation tracker until maintainers choose one lifecycle/config design, land or supersede the daily/idle PRs, wire the bundled hook into the agreed triggers, and document/test idle, shutdown, periodic checkpoint, and summary-output behavior together.
