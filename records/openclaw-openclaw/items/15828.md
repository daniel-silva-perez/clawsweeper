## Summary

Current main still does not implement the requested bundled session-memory automation. The bundled hook is still registered for `/new` and `/reset` only, the handler returns for non-command/non-reset events, daily/idle stale rollovers only fire plugin session lifecycle hooks, and the requested shutdown/message-count/summarize config surface is absent. Open PRs #61675 and #71792 cover the daily/idle subset, but they do not cover the full feature request, so this issue should remain open for maintainer scoping.

## Best Possible Solution

Keep this issue open as the broader session-memory automation tracker. Review and choose the daily/idle implementation path in #61675 versus #71792, then split or scope the remaining shutdown checkpoint, message-count checkpoint, and summary-output config work with tests and docs before implementation.
