## Summary

Keep open. Current main still keeps bundled session-memory persistence tied to explicit command events, while the requested idle, shutdown, periodic message-count, trigger-config, and summary-output behavior remains absent; the related daily/idle PRs are open partial implementations rather than merged fixes for the broader request.

## What This Changes

The issue requests automatic session-memory checkpoints on idle, gateway shutdown, periodic message counts, and optional LLM summary output instead of relying only on `/new`.

## Best Possible Solution

Keep this as the broad session-memory automation tracker. The end state should extend the bundled hook or a documented replacement path with explicit trigger config, shutdown-safe checkpointing, message-count checkpoints, summary-output semantics, docs, and focused regression tests while preserving existing `/new` and `/reset` behavior.
