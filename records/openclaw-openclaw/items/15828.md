## Summary

Keep this issue open. Current main still implements bundled session-memory as a command-only /new and /reset hook; idle/shutdown/periodic triggers, configurable trigger lists, message-count checkpoints, and LLM summary output remain unimplemented. Related PRs #61675 and #71792 are active narrower idle/daily candidates, not complete replacements for the broader feature request.

## Best Possible Solution

Keep this issue open unless maintainers intentionally split it. The best path is to choose or land one canonical idle/daily rollover PR, then separately design the broader session-memory trigger contract: configurable command/idle/shutdown/message-count triggers, dedupe rules, summary-generation behavior, schema/docs updates, and focused tests for session-memory plus lifecycle dispatch.
