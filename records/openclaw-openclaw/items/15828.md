## Summary

Keep open. Current main still persists bundled session-memory only through explicit `/new` and `/reset` command hook events; idle/daily rollover, gateway shutdown, periodic message-count checkpoints, trigger configuration, and summary-output behavior remain absent, while the related daily/idle PRs are still open partial implementations.

## What This Changes

The issue requests automatic session-memory checkpoints on idle, shutdown, and periodic message-count triggers, plus configurable trigger selection and optional LLM summary output.

## Best Possible Solution

Keep this as the broader session-memory automation tracker. The end state should extend the bundled hook or an explicitly documented replacement path with trigger configuration, shutdown-safe checkpointing, periodic message-count checkpoints, summary-output semantics, docs, and focused regression tests while preserving existing `/new` and `/reset` behavior.
