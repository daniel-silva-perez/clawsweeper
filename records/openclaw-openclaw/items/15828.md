## Summary

Keep open. Current main still keeps the bundled session-memory hook on explicit `/new` and `/reset` command events only; adjacent idle/daily rollover, gateway shutdown, and pre-compaction memory-flush infrastructure exists, but it does not satisfy the requested idle, shutdown, periodic message-count, trigger-config, or summary-output behavior.

## What This Changes

Add session-memory automation so the bundled hook can save on idle, shutdown, periodic message-count checkpoints, and optionally write LLM summaries instead of raw transcript excerpts.

## Best Possible Solution

Keep this as the broad session-memory automation tracker. The end state should extend the bundled hook or a documented replacement path with explicit trigger config, shutdown-safe checkpointing, message-count checkpoints, summary-output semantics, docs, and focused regression tests while preserving the existing `/new` and `/reset` behavior.
