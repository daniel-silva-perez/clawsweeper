## Summary

Keep this issue open. Current main still leaves bundled session-memory wired to command:new and command:reset only; idle/daily stale rollover exposes plugin session_end hooks but does not fire the internal session-memory hook, gateway:shutdown exists but session-memory is not registered for it, and message-count checkpoints, configurable trigger lists, and LLM-generated summary output remain unimplemented. Open PRs #61675 and #71792 are narrower idle/daily candidates, not full replacements for this broader feature request.

## Best Possible Solution

Keep this issue open unless maintainers intentionally split it. The best path is to choose or land one canonical idle/daily rollover PR, then separately design the broader session-memory trigger contract: configurable command/idle/shutdown/message-count triggers, dedupe semantics, summary-generation behavior, schema/docs updates, and focused tests for session-memory plus lifecycle dispatch.
