## Summary

Keep this PR open. Current main still does not ship Vietnamese Control UI localization: `vi` is absent from the active locale type, lazy registry, navigator fallback, generated i18n locale list, tests, generated locale artifacts, and Control UI docs. The PR remains a coherent implementation candidate, but it needs a refresh against the current generated Control UI i18n workflow. Related #65190 is docs/scripts localization work and does not supersede this runtime UI locale change.

## Best Possible Solution

Keep this PR open and refresh it against current main: add Vietnamese to the Locale union, lazy registry, navigator fallback, `scripts/control-ui-i18n.ts`, generated `ui/src/i18n/locales/vi.ts`, matching `.i18n` glossary/meta/translation-memory files, language labels, registry tests, and `docs/web/control-ui.md`. Keep #65190 separate unless maintainers want to coordinate Vietnamese glossary terminology across docs and the Control UI.
