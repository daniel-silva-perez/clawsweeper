## Summary

Keep open. Current main already has substantial guided setup through classic onboarding, `openclaw configure`, Gateway wizard RPC, plugin setup metadata, and Crestodian typed setup/repair, but it does not implement the requested agent-guided configuration assistant that interviews the user, probes host resources such as CPU/GPU/RAM, recommends best-practice settings, and revisits recommendations as new features ship. The remaining work is product and security scoping, not conservative cleanup closure.

## What This Changes

The issue requests a built-in agent-guided configuration wizard that probes the user’s machine and goals, recommends best-practice settings, and can revisit configuration as new OpenClaw features are released.

## Best Possible Solution

Define a next-generation setup UX around the existing onboarding/Crestodian boundary: keep configuration writes typed, approved, audited, and reversible; add goal and environment discovery only where recommendations are explainable; and decide which recommendations belong in core versus plugin-owned setup hooks.
