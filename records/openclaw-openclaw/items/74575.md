## Summary

Keep open. Current main has guided onboarding, section-based reconfiguration, plugin setup prompts, and Crestodian typed setup/repair, but it still lacks the requested goal-driven assistant that interviews the user, probes host capability such as CPU/GPU/RAM, recommends best-practice settings, and refreshes recommendations as OpenClaw features ship. This is also broad and config-write/security-sensitive, so it needs maintainer product scoping rather than a conservative cleanup close.

## What This Changes

The issue requests a built-in agent-guided configuration wizard that asks the user goal-oriented questions, tests host capabilities, recommends best-practice settings, and updates recommendations as OpenClaw changes.

## Best Possible Solution

Define a scoped next-generation setup UX on top of existing onboarding and Crestodian boundaries: collect user goals, expose explainable host/resource discovery, route plugin-specific recommendations through plugin-owned setup metadata, and keep all config writes typed, approved, audited, reversible, and documented.
