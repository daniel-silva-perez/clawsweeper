## Summary

Keep open. Current main has fixed-step onboarding, section-based reconfiguration, plugin config prompts, and Crestodian typed setup/repair, but it still does not implement the requested goal-driven assistant that interviews the user, probes host resources, recommends best-practice settings, and revisits recommendations as features ship. The existing ClawSweeper comment reached the same product-boundary conclusion, and the later maintainer mention/subscription is a reason to leave this for maintainer scoping rather than cleanup-close it.

## What This Changes

Add a built-in configuration wizard or assistant that asks users about goals, probes CPU/memory/GPU-style host capability, recommends best-practice OpenClaw settings, and refreshes configuration guidance for new features.

## Best Possible Solution

Define a scoped next-generation setup UX that builds on existing onboarding and Crestodian boundaries: collect user goals, expose explainable and consented host/resource discovery, let plugins contribute typed recommendation metadata where appropriate, and keep every persistent config write approved, audited, reversible, schema-backed, and documented.

## Reproduction Assessment

Not applicable as a feature request, but current-main inspection gives a high-confidence gap check: onboarding, configure, plugin prompts, and Crestodian do not provide the requested goal-driven host-capability recommender.

## Solution Assessment

No current implementation is sufficient, and this should not be solved by a blind automated config writer. The maintainable path is a maintainer-scoped design that extends typed setup/config surfaces and plugin metadata before adding recommendation logic.
