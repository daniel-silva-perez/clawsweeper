## Summary

Keep open. Current main has guided onboarding, section-based reconfiguration, plugin config prompts, and Crestodian typed setup/repair, but it still does not implement the requested assistant that interviews users, probes host resources, recommends best-practice settings, and revisits recommendations as features change. The later maintainer mention/subscription is also a signal to leave this for product/security scoping rather than cleanup-close it.

## What This Changes

The issue requests a built-in assistant-style configuration wizard that interviews users, probes host capabilities, recommends best-practice settings, and can revisit recommendations as OpenClaw features change.

## Best Possible Solution

Define a scoped next-generation setup UX that builds on existing onboarding, configure, plugin uiHints, and Crestodian typed operations: collect user goals, expose consented host/resource discovery, let plugins contribute schema-backed recommendation metadata where appropriate, and keep every persistent config write explicit, audited, reversible, and documented.

## Reproduction Assessment

Not applicable as a feature request. The current-main gap is high confidence from docs and source inspection: onboarding, configure, plugin prompts, and Crestodian do not provide the requested goal-driven host-capability recommender.

## Solution Assessment

No current implementation solves the issue, and a blind built-in config writer would not be the best approach. The maintainable path is maintainer-scoped design that extends typed setup/config surfaces and plugin metadata before adding recommendation logic.
