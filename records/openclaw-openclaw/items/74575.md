## Summary

Keep open. Current main already has guided CLI onboarding, section-based reconfiguration, plugin setup prompts, and Crestodian typed setup/repair, but it does not yet provide the requested goal-driven assistant that interviews the user, probes CPU/GPU/RAM-style host capability, recommends best-practice settings, and refreshes recommendations as OpenClaw features change. The remaining work is broad, product-scoped, and config-write/security-sensitive, so it is not a conservative cleanup close.

## What This Changes

The issue requests a built-in, agent-guided configuration wizard that interviews users, probes host capabilities, recommends best-practice settings, and refreshes configuration guidance as features change.

## Best Possible Solution

Define a scoped next-generation setup experience on top of existing onboarding and Crestodian boundaries: collect user goals, expose explainable host/resource discovery, route plugin-specific recommendations through plugin-owned metadata, and keep all config writes typed, approved, audited, reversible, and documented.
