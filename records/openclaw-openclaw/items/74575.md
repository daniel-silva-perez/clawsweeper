## Summary

Keep open. Current main already has substantial guided setup through classic onboarding, `openclaw configure`, Gateway wizard RPC, and the Crestodian conversational setup/repair preview, but the requested product is broader: an agent-guided configuration assistant that asks goal-oriented questions, probes host capabilities such as CPU/GPU/RAM, and revisits best-practice settings as new features ship. That remaining scope is not fully implemented and needs maintainer product direction rather than conservative cleanup closure.

## What This Changes

The issue requests a built-in, agent-guided configuration wizard that probes the user's environment, asks about intended OpenClaw usage, and tunes settings over time.

## Best Possible Solution

Define the next-generation setup UX around the existing onboarding/Crestodian boundary: keep writes typed and approval/audit-backed, add goal and environment discovery only where recommendations are explainable and reversible, and document which recommendations belong in core versus plugin-owned setup hooks.
