## Summary

Keep open. Current main has guided onboarding, section-based reconfiguration, plugin config prompts, doctor repairs, and Crestodian typed setup/repair, but it still does not provide the requested assistant that interviews users, probes host resources, recommends best-practice settings, and revisits recommendations when features change. The existing ClawSweeper comment and later maintainer mention also point toward product/security scoping rather than cleanup-close.

## What This Changes

The issue requests a built-in, agent-guided configuration wizard that asks users about their goals, probes the host, recommends best-practice settings, and updates recommendations as OpenClaw features change.

## Best Possible Solution

Define a scoped next-generation setup UX that builds on onboarding, configure, plugin uiHints, and Crestodian typed operations: collect user goals, allow consented host/resource discovery, let plugins contribute schema-backed recommendation metadata where appropriate, and keep persistent config writes explicit, auditable, reversible, and documented.

## Reproduction Assessment

Not applicable as a feature request. The current-main gap is high confidence from docs and source inspection: onboarding, configure, plugin prompts, doctor, and Crestodian do not provide the requested goal-driven host-capability recommender.

## Solution Assessment

No current implementation solves the request, and a broad blind config-writing assistant would not be the best approach. The maintainable solution is a maintainer-scoped design that extends typed setup/config surfaces and plugin metadata before adding recommendation logic.
