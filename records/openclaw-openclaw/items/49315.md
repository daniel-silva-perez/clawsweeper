## Summary

Keep this PR open. Current main still does not support or document `--profile-id` for `openclaw models auth login`, the related tracker #40402 remains open, and this PR is still a plausible implementation candidate. It needs maintainer revision because the proposed post-login profile rewrite can conflict with provider-owned auth flows that persist credentials before returning their auth result.

## Best Possible Solution

Keep this PR open for maintainer revision or replacement. The best fix is to add the login option, pass the requested profile target through provider auth context before credentials are persisted, validate provider ownership and collision behavior, have OpenAI Codex honor the target in its OAuth and device-code paths, avoid generic post-login rewrites, and update public docs plus CLI/provider regression tests linked to #40402.
