## Summary

Keep this PR open. Current main still lacks `--profile-id` on `openclaw models auth login`, the related tracker #40402 is still open, and this PR remains a plausible implementation candidate, but the proposed generic post-login profile rewrite needs revision because provider-owned auth flows can persist credentials before returning their auth result.

## Best Possible Solution

Keep this PR open for maintainer revision or replacement. The best fix is to add the login option, carry the requested profile target through a typed provider auth context before credentials are persisted, validate provider ownership and collisions, have OpenAI Codex OAuth and device-code login honor that target in the provider, avoid generic post-login rewrites, and update public docs plus CLI/provider regression tests linked to #40402.
