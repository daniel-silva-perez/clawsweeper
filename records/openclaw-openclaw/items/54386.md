## Summary

The PR addresses a real data-loss problem, but it is not ready to merge because the new omitted-path fallback diverges from OpenClaw's active workspace contract; current main also does not already implement omitted-path skill creation, so this remains useful source work rather than cleanup-closeable.

## What This Changes

The PR makes the bundled skill-creator initializer accept an omitted `--path`, adds a default skills-directory resolver, and updates the skill-creator instructions to discourage global or relative output paths.

## Best Possible Solution

Land a narrow fix that keeps explicit `--path` backward-compatible, defaults omitted output to the same active workspace `skills/` directory used by OpenClaw runtime and `openclaw skills`, removes relative output examples, and adds focused regression coverage for default-path resolution.
