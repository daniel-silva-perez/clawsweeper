## Summary

Keep this PR open. Current main still requires `--path` and still shows relative skill-creator examples, so the proposed user problem is not already fixed; however, the PR’s default resolver does not match OpenClaw’s active workspace contract and can still create skills where the active agent will not load them.

## What This Changes

The PR makes the skill-creator initializer accept an omitted `--path`, adds a Python default skills-directory resolver, and updates bundled skill-creator instructions to warn against global or relative output paths.

## Best Possible Solution

Land a narrow fix that preserves explicit `--path` behavior, defaults omitted paths to the same active workspace `skills/` directory used by runtime and native `openclaw skills` commands, removes hard-coded default-workspace examples, and adds focused regression coverage around default path resolution.
