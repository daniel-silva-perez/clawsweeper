## Summary

Keep open. Current main still has the unsafe skill-creator workflow, but this PR is not ready to merge because its new fallback can still create skills outside the active OpenClaw workspace in configured, profiled, or multi-agent setups.

## What This Changes

The PR makes the skill-creator initializer accept an omitted `--path`, adds a default workspace skills-directory resolver, and updates bundled skill-creator instructions to discourage global or relative output paths.

## Best Possible Solution

Land a narrow fix that keeps explicit `--path` backward-compatible, defaults omitted paths to the same active workspace `skills/` directory that OpenClaw runtime and `openclaw skills` commands use, removes relative output examples from bundled instructions and the script docstring, and adds focused regression tests around path resolution.
