## Summary

Keep this PR open. Current main still has the reported skill-creator behavior, and the PR is directionally useful, but the proposed default resolver should not merge as-is because it does not match OpenClaw's active workspace semantics for profile, home, and configured agent workspace selection, and it lacks regression coverage.

## What This Changes

The PR makes the skill-creator initializer accept an omitted `--path`, adds a default workspace skills-directory resolver, and updates the bundled skill-creator instructions to discourage global or relative output paths.

## Best Possible Solution

Land a narrow fix that keeps explicit `--path` backward-compatible, defaults omitted paths to the same active workspace `skills/` location OpenClaw uses elsewhere, removes relative output examples from the bundled skill instructions and script docstring, and adds focused Python regression tests before the changed gate.
