## Summary

Keep this PR open. Current main still requires `--path` for skill initialization and still documents relative `skills/public` output paths, so the data-loss concern remains valid. The PR is useful but should not merge as-is because the proposed default must follow OpenClaw’s active workspace semantics, including `OPENCLAW_HOME` and non-default `OPENCLAW_PROFILE`, and needs targeted regression coverage.

## What This Changes

The PR makes `skills/skill-creator/scripts/init_skill.py --path` optional, adds a default workspace skills-directory resolver, and updates the skill-creator instructions to discourage global install or relative output paths.
