## Summary

Keep open. Current main still prints only the bare dashboard URL in the combined clipboard/browser delivery failure path, while the linked closing PR remains open and the earlier alternative PR was closed unmerged.

## What This Changes

The issue asks for `openclaw dashboard` to provide usable token-auth guidance when the printed dashboard URL is bare in WSL/headless environments.

## Best Possible Solution

Review and land #72802, or an equivalent narrow dashboard command change, so failed automatic delivery prints safe manual recovery guidance naming `OPENCLAW_GATEWAY_TOKEN` and `gateway.auth.token` without logging the token value or fragment marker.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a configured plaintext `gateway.auth.token`, clipboard copy returning false, and either `--no-open` or browser open unavailable; current main logs the bare URL/fallback without manual token recovery guidance.

## Solution Assessment

Yes. The best fix direction is narrow and owner-local: keep tokenized URLs limited to clipboard/browser delivery, preserve log redaction, and add guidance only when automatic delivery fails.
