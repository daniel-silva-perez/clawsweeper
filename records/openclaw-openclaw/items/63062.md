## Summary

Current main still lacks the OpenRouter trailing-user cache marker that this PR adds, and the PR is paired with the still-open bug it closes. The change is focused and appears functionally aligned with Anthropic/OpenRouter cache-control semantics, but it should stay open for maintainer review because it changes a shared exported helper used beyond OpenRouter and needs docs alignment before merge.

## What This Changes

The PR adds a call to mark the trailing user message with ephemeral Anthropic cache_control in the OpenRouter cache-marker helper and updates cache-control regression tests to expect the new user-message marker.

## Best Possible Solution

Keep this PR open for maintainer review. If the OpenRouter behavior is intended, land or revise the focused helper change with regression tests, explicitly account for shared SDK/GitHub Copilot impact, and update the prompt-caching/OpenRouter docs before or alongside merge; after merge, the paired bug can close as implemented.
