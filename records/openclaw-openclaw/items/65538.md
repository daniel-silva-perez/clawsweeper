## Summary

Keep open. Current main still renders the Control UI chat transcript as a polite live log while streamed assistant text is appended into that transcript, so the reported screen-reader behavior remains plausible. There is also an open linked implementation PR, #65575, with closing syntax and focused tests for the same `aria-live` behavior, so this issue should remain open until that PR is reviewed and merged or rejected.

## Best Possible Solution

Keep this issue open and use #65575 as the current implementation candidate. The fix should stay scoped to Control UI chat rendering: suppress transcript live-region announcements while `sending` or streaming, restore polite announcements when idle, and add focused tests in `ui/src/ui/views/chat.test.ts`. If maintainers reject #65575, an equivalent main-branch fix should address the same live-region behavior before this issue is closed.
