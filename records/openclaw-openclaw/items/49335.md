## Summary

Keep this PR open. Current main still full-reads and synchronously parses session transcripts before `chat.history` applies response limits, so the freeze fix is not implemented; however, the PR branch is stale/dirty against current main and the proposed reader still has blocking correctness issues around stable history sequence metadata and current active-branch transcript semantics.

## What This Changes

The PR changes `src/gateway/session-utils.fs.ts` so `readSessionMessages()` tail-reads large transcript files, caps oversized JSONL records, emits placeholder messages for oversized lines, and logs truncation or oversize conditions.

## Best Possible Solution

Land a bounded Gateway transcript reader that limits read/parse cost while preserving current active-branch behavior, compaction dividers, sanitized media-heavy history, and stable absolute session-history sequence numbers. The fix should include focused regression tests for large transcripts, oversized UTF-8 lines, active branches, compaction records, media-heavy records, and pagination cursors before merge.

## Reproduction Assessment

Yes. The PR body gives a concrete 447 KB single-line JSONL reproduction, and current main still full-reads and synchronously parses every transcript line before `chat.history` response caps run; this review did not run live reproduction because the task is read-only.

## Solution Assessment

No, not as currently proposed. The direction is right, but the current PR branch is stale and the patch needs to preserve active-branch reads plus stable absolute `__openclaw.seq` metadata rather than adding tail truncation and placeholders inside the legacy line parser only.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve absolute sequence numbers when tailing:** `src/gateway/session-utils.fs.ts:189`
  - body: When the tail-read path skips the file prefix, `messageSeq` still starts at zero for the retained tail. Session-history pagination uses `__openclaw.seq` as its cursor, so once a transcript crosses the tail window the same message can receive different seq values across reads and clients can skip or duplicate history. Seed sequence numbers from the skipped prefix or another stable transcript position before returning tail messages.
  - confidence: 0.91
- **[P1] Include oversized placeholders in sequence metadata:** `src/gateway/session-utils.fs.ts:260-270`
  - body: The oversized-line branch pushes a synthetic message without incrementing `messageSeq` or setting `__openclaw.seq`. If that placeholder lands near a page boundary, pagination can report `hasMore` without a usable cursor and later messages are numbered as if the record did not exist. Assign placeholders the next transcript seq before continuing.
  - confidence: 0.9
- **[P2] Retain active-branch transcript reads:** `src/gateway/session-utils.fs.ts:187-189`
  - body: The PR head parses the split content directly and no longer uses `SessionManager.open(filePath).getBranch()` for parentId-based transcripts. Current main relies on that path to hide abandoned rewrite branches; rebasing this reader without carrying it forward will show superseded prompts in WebChat history again.
  - confidence: 0.87
- **[P2] Classify oversized records before emitting chat placeholders:** `src/gateway/session-utils.fs.ts:196-201`
  - body: The size guard emits a synthetic chat message before knowing whether the JSONL record is actually a message. Oversized compaction or other non-message records will be rendered as omitted chat turns instead of preserving existing compaction-divider semantics. Add bounded type detection or a non-message skip path before creating a chat placeholder.
  - confidence: 0.84
