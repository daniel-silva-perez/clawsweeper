## Summary

Keep open. Current main still matches the reported subagent lifecycle gaps, and the later related orphan-recovery work only covers restart-aborted sessions rather than clean gateway-close classification, pending wait rearming, or still-running backing-session reattachment.

## What This Changes

The issue asks for subagent wait/lifecycle recovery to treat clean gateway close, pending waits, and still-running backing sessions as recoverable rather than terminal failures.

## Best Possible Solution

Implement a focused `src/agents` fix: classify clean gateway-close wait interruptions as recoverable for subagent supervision, rearm `waitForSubagentCompletion` after `pending`, and re-wait or reattach when the child session store still says `running`, while preserving existing terminal, missing-session, stale, and restart-aborted failure semantics.

## Reproduction Assessment

Yes. Current-main source inspection reproduces the three reported branches directly: the 1000 close string is not recoverable, pending returns without a new waiter, and a still-running session can still fall through to lost-context completion.

## Solution Assessment

Yes. The narrow source-level fix proposed by the report is the best maintainable direction; it should be implemented with regression tests rather than folded into the broader #74131 readiness/finalize retry work unless that PR is deliberately expanded.
