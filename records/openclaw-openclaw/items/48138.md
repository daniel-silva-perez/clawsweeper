## Summary

Keep this PR open. Current main still does not implement Feishu document ownership transfer: `feishu_doc` has no `transfer` schema or dispatch case, `feishu_perm` only lists/adds/removes collaborators, and the current Feishu docs/tests do not cover transfer-owner behavior or the required scope. The PR remains a focused implementation candidate, but it changes a sensitive document-ownership path and should get maintainer security review before landing.

## Best Possible Solution

Keep this PR open for maintainer review and likely landing after deciding whether ownership transfer belongs in `feishu_doc` or the default-disabled permission tool, confirming Feishu `transferOwner` authorization semantics, constraining or documenting caller-supplied principal selection, updating Feishu skill/docs scope guidance for `docs:permission.member:transfer`, and preserving the `old_owner_perm` default regression test.
