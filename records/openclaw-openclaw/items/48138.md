## Summary

Keep this PR open. Current main still does not implement Feishu document ownership transfer: the `feishu_doc` schema and dispatch omit `transfer`, the adjacent `feishu_perm` tool only supports list/add/remove permission members, and current tests/docs have no transfer-owner coverage or scope guidance. The PR remains a focused implementation candidate, with a recent maintainer repair preserving the contributor path, but the ownership-transfer path should get maintainer security review before landing.

## Best Possible Solution

Keep this PR open for maintainer review and likely landing after rebasing against current main, confirming Feishu `transferOwner` authorization semantics, deciding whether the new owner must be constrained to trusted requester context or explicitly documented as an admin-level operation, updating Feishu skill/docs scope guidance, and preserving the `old_owner_perm` default regression test.
