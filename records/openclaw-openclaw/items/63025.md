## Summary

Keep this PR open, but the current patch should not merge as-is. Current main still has the zero-output embedded dispatch path the PR is trying to address, yet the PR fallback can bypass existing delivery-suppression policy for direct sessions.

## What This Changes

The PR adds a static error final reply when direct/private embedded dispatch produces no final, tool, or block output, plus regression tests for the direct fallback and channel no-op path.

## Best Possible Solution

Land a narrow dispatch-level empty-output fallback only after final queued counts are known, guarded by delivery policy, direct-style conversation policy, no routed-origin delivery, no queued final/tool/block output, no block-stream/TTS output, and no intentional handled/no-text branch. Use `normalizeChatType` unless maintainers intentionally add `private` as a supported alias.

## Reproduction Assessment

Yes. The current-main gap is reproducible with `dispatchReplyFromConfig` in a direct/private context, `replyResolver: async () => undefined`, and no block/tool output; the PR regression is reproducible by adding a `sendPolicy: "deny"` session entry to that same path.

## Solution Assessment

No. The requested dispatch-level fallback is the right general shape, but this patch is missing the existing `suppressDelivery` guard and should align with the canonical chat-type contract before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Honor delivery suppression for fallback replies:** `src/auto-reply/reply/dispatch-from-config.ts:1050-1057`
  - body: The fallback is evaluated after the normal `if (!suppressDelivery)` delivery block, but the condition itself does not check `suppressDelivery`. In a direct session with `sendPolicy: "deny"` or message-tool-only delivery, an empty resolver result would now send this error final even though current tests require final/tool/block delivery to stay suppressed. Gate the fallback with `!suppressDelivery` and add a deny-policy regression test.
  - confidence: 0.91
