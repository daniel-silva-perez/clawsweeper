## Summary

Keep this issue open. Current main still has the reported failure shape: final replies go through the generic TTS hook, but a hidden `[[tts:text]]hello[[/tts:text]]` block is stripped from visible text, then skipped by the sub-10-character TTS guard, leaving an `audioAsVoice` payload with no text or media for Telegram to skip.

## Best Possible Solution

Keep the issue open and fix the final-reply TTS fallback semantics. Add a regression test for a claude-cli-style final payload containing `[[audio_as_voice]] [[tts:text]]hello[[/tts:text]]`, then either synthesize explicitly tagged short hidden text or clear `audioAsVoice` and preserve a visible fallback so Telegram never receives an empty voice-only payload.
