## Summary

Keep open. Current main still matches the reported failure shape: final replies do reach the generic TTS hook, but a hidden `[[tts:text]]hello[[/tts:text]]` block is stripped from visible text and then skipped by the sub-10-character TTS guard. That leaves an `audioAsVoice` payload with no text or media for Telegram to skip.

## Best Possible Solution

Keep this open and fix the final-reply TTS fallback semantics. Add a regression test for a claude-cli-style final payload such as `[[audio_as_voice]] [[tts:text]]hello[[/tts:text]]`, then either synthesize explicitly tagged short hidden text or clear `audioAsVoice` and preserve a visible fallback so Telegram never receives an empty voice-only payload.
