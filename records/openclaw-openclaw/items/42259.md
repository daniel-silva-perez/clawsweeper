## Summary

Keep this PR open. Current main still lacks configurable image-tool JPEG quality or max-side controls, and the linked feature issue #41771 was closed as tracked by this PR. The PR should not be closed as obsolete, but it should not merge as-is because its head is dirty/outdated, targets the old `src/web/media.ts` path, has unresolved constraint-fallback review bugs, and bundles unrelated docs/gateway/provider-link changes.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate, or supersede it with a focused rebased PR. The best patch should add a typed and documented `agents.defaults` image-tool compression contract, thread options through current `src/agents/tools/image-tool.ts` and `src/media/web-media.ts`, preserve SSRF and local media access guards, add regression tests for out-of-ladder values such as quality 85/95 and maxSide 600, and split unrelated docs/gateway/provider-link changes into separate work.
