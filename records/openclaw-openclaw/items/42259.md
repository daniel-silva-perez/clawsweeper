## Summary

Keep this PR open. Current main still does not implement configurable image-tool JPEG quality or max-side controls, so the PR is not obsolete. It should not merge as-is because the PR head targets an older media path, has review-identified constraint fallback bugs, and mixes unrelated docs/gateway/provider-link changes into the image compression feature.

## Best Possible Solution

Keep this PR open as an implementation candidate, or supersede it with a focused rebased PR. The best patch should add a typed and documented `agents.defaults` image-tool compression contract, thread options through current `src/agents/tools/image-tool.ts` and `src/media/web-media.ts`, preserve SSRF and local media access guards, add regression tests for out-of-ladder values such as quality 85/95 and maxSide 600, and split unrelated docs/gateway/provider-link changes into separate work.
