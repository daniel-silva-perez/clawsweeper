## Summary

Keep this PR open. Current main still does not implement configurable image-tool JPEG quality or max-side controls, and the linked feature request #41771 was closed as tracked by this PR while related PR #42058 is closed unmerged. This PR remains a useful implementation candidate, but it needs a focused rebase and fixes before merge.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate, or replace it with a focused rebased PR. The best patch should add a typed and documented agents.defaults image-tool compression contract, thread it through src/agents/tools/image-tool.ts and src/media/web-media.ts, preserve existing media access guards, add regression tests for out-of-ladder values such as quality 85/95 and maxSide 600, and split unrelated docs/gateway/provider-link changes into separate work.
