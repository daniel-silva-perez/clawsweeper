## Summary

Keep this PR open. Current main still has only fixed image-tool JPEG compression ladders and no typed, validated, or documented `agents.defaults.imageCompressionQuality` / `imageCompressionMaxSide` contract. The linked feature request is closed as tracked by this PR, the broader related implementation PR #42058 is closed unmerged, and this PR remains a plausible implementation candidate, but it needs rebase/scope cleanup and the reviewed constraint-fallback bug fixed before maintainer review.

## Best Possible Solution

Keep this PR open as a plausible implementation candidate, or replace it with a focused rebased PR. The best patch should choose the public config contract deliberately, add typed/schema/help/docs coverage under `agents.defaults`, thread compression options through current `src/agents/tools/image-tool.ts` and `src/media/web-media.ts`, preserve media access and SSRF guards, add regression tests for out-of-ladder values such as quality 85/95 and maxSide 600, and split unrelated README/provider-link/doctor/gateway wording changes into separate work.
