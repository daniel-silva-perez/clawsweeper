## Summary

Keep open. Current main still implements the Control UI dropdown as an optimistic session patch, while the send path and send controls do not observe or await that patch; `chat.send` also has no model override to compensate. The prior ClawSweeper review remains applicable against `acae48b`.

## What This Changes

The issue requests that Control UI model dropdown changes be persisted, or sending be blocked, before the next `chat.send` so the next turn uses the selected model.

## Best Possible Solution

Control UI should track a per-session pending model-patch operation, surface that state to the picker and composer controls, await it before dispatching a send, roll back the displayed selection on patch failure, and keep the `chat.send` protocol unchanged unless maintainers choose a broader API change.

## Repair Work Prompt

Fix the Control UI model dropdown/send ordering race described in https://github.com/openclaw/openclaw/issues/54240. Implement a per-session pending model-switch barrier in the Control UI: when the chat model select starts `sessions.patch`, store the pending operation in shared app/chat state, disable the picker and send/queue controls while it is pending, have `handleSendChat` await the pending operation before dispatching `chat.send`, and if the patch fails, roll back the picker state, surface the error, and do not send the just-submitted message. Keep the fix scoped to the Control UI; do not add a `model` field to `chat.send` or make core protocol changes unless the existing UI-only approach proves impossible. Preserve existing duplicate-submit, queue, stop/abort, and slash-command behavior. Add regression coverage that leaves the `sessions.patch` promise unresolved, attempts an immediate send, verifies `chat.send` has not been called until the patch resolves, and verifies patch failure suppresses send and restores the draft/selection. Add a user-facing changelog fix entry with a real contributor attribution; do not touch unrelated model dropdown bugs or broad provider catalog behavior.
