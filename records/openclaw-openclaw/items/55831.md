## Summary

Keep this PR open. Current main still has the stale-cache bug this PR targets, and the linked bug #55705 remains open, but the current PR diff is broader than the stated fix because it also clears the sessions cache on every exception. The best path is a narrower UI controller fix with regression tests.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #55705, but revise it narrowly: on successful `sessions.list`, assign `state.sessionsResult = res ?? null`; run checkpoint invalidation/refetch only when `res` exists; preserve the current generic-error last-known-good cache behavior; keep the missing-scope clear path; and add focused sessions controller tests for both undefined-success clearing and transient-error preservation.
