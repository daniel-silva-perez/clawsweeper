## Summary

Keep this PR open. Current main still has the stale-cache behavior this PR targets, and the linked bug remains open, but the current PR diff also clears the cached sessions result on every exception, matching the unresolved review concern about transient errors wiping the last-known-good UI state.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #55705, but revise it narrowly: on successful `sessions.list`, set `sessionsResult = res ?? null`; run the existing checkpoint invalidation/refetch logic only when `res` exists; preserve the current generic-error last-known-good cache behavior; keep the missing-scope clear path; and add focused tests for undefined-success clearing and transient-error preservation.
