## Summary

Keep this PR open. Current main still does not implement effective iMessage inbound untrusted-content wrapping on the model-bound path, and this PR is a focused security-hardening attempt. It is not merge-ready as submitted because it wraps only the legacy `formatInboundEnvelope(...)` body while current prompt/session assembly prefers raw `BodyForAgent` and appends `UntrustedContext` separately.

## Best Possible Solution

Revise this PR or replace it with a maintainer-reviewed iMessage hardening patch. The implementation should first settle the intended trust boundary for iMessage DMs, then protect the actual model-bound path through `UntrustedContext` or an equivalent prompt-bound field, keep `RawBody` and `CommandBody` raw/deterministic for command parsing, preserve reply/history behavior intentionally, and add focused iMessage tests for prompt assembly, commands, reply context, history, and echo behavior.

## Risks / Open Questions

- Merging this PR as-is could create a false sense that iMessage inbound prompt-injection hardening is active while raw `BodyForAgent` still drives the normal prompt path.
- The patch adds randomized external-content markers to legacy envelope/history text, which may affect deterministic history or rendering without protecting the model-bound body.
- No iMessage regression coverage is included for prompt assembly, command parsing, reply context, group history, or echo behavior.
