## Summary

Keep open. Current main still has the reported source-level failure mode: isolated `agentTurn` cron jobs route through the embedded OpenAI-compatible stream path, tool-call argument chunks are blindly appended into `partialArgs`, completed tool-call blocks still retain that scratch field, and the existing malformed-argument repair rejects a repeated complete JSON object as unsafe trailing content rather than normalizing it.

## What This Changes

The issue requests fixing malformed duplicated `partialArgs` JSON for isolated `agentTurn` cron jobs when OpenAI-compatible tool-call streams repeat a complete arguments object.

## Best Possible Solution

Normalize repeated or cumulative complete `function.arguments` payloads at the OpenAI-compatible stream boundary, or safely collapse that exact duplicate shape in the repair wrapper, then remove internal `partialArgs` scratch state from completed tool-call blocks and add focused regressions for the isolated cron/OpenRouter-Kimi-shaped stream.
