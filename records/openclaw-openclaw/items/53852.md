## Summary

Keep this issue open. Current main still routes isolated `agentTurn` cron jobs through the embedded OpenAI-compatible stream path, that stream path still appends every `function.arguments` chunk into `partialArgs`, and the existing repair wrapper still rejects a second complete JSON object as unsafe trailing content rather than safely collapsing duplicate cumulative arguments. Recent Kimi/OpenRouter tool-call repair work covers small trailing junk, preambles, and fragmented arguments, but not the reported duplicated complete JSON payload shape.

## Best Possible Solution

Keep this issue open. The best fix belongs at the OpenAI-compatible streaming/tool-call boundary: detect cumulative or repeated complete `function.arguments` payloads before appending, safely collapse identical duplicated balanced JSON objects, strip scratch `partialArgs` from completed tool-call blocks, and add a focused regression test for isolated `agentTurn` cron using an OpenRouter/Kimi-shaped stream that emits the same complete arguments twice.
