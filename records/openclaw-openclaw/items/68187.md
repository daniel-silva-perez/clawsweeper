## Summary

Keep open. Current main still caches session-scoped MCP clients and forwards bundle MCP tool calls directly; it does not detect stale SSE/HTTP MCP sessions or recreate/retry after `Session not found`. The cross-referenced #71545 is open, unmerged, and explicitly scoped to streamable-http while leaving SSE untouched, so it is a related implementation candidate rather than a close reason.

## Best Possible Solution

Keep the issue open. Maintainers should either broaden and validate the MCP session runtime recovery path for SSE-backed sessions, or explicitly split this into a narrower streamable-http follow-up once #71545 lands. The likely code path is `src/agents/pi-bundle-mcp-runtime.ts`, with regression coverage in `src/agents/pi-bundle-mcp-runtime.test.ts` and a restart/reset-capable MCP test harness.
