## Summary

Keep open. Current main still caches session-scoped bundle MCP clients and calls the cached SDK client directly; there is no stale SSE/HTTP session detection, reconnect, catalog refresh, or one-shot retry for `Session not found`. The related #71545 is still only an open, unmerged streamable-http implementation candidate in the provided GitHub context and explicitly leaves SSE unaffected.

## Best Possible Solution

Keep this issue open. The best fix is to add a generic bundle-MCP stale-session recovery path in `src/agents/pi-bundle-mcp-runtime.ts`: classify lost-session errors from SSE and streamable-http transports, dispose the dead client/transport, recreate via `resolveMcpTransport`, reconnect with `connectWithTimeout`, and retry the original `callTool` once while preserving non-stale errors. Add restart/reset-capable coverage in `src/agents/pi-bundle-mcp-runtime.test.ts` using `src/agents/pi-bundle-mcp-test-harness.ts`, and decide whether #71545 should be broadened or remain a narrower streamable-http follow-up.
