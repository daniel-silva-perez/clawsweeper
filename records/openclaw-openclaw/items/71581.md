## Summary

Keep this issue open. Current main still preserves explicit same-agent exec/cron heartbeat session keys instead of applying the single-owner `session.dmScope="main"` routing rule, and open PR #72648 is already linked with closing syntax as the implementation candidate.

## Best Possible Solution

Keep this issue open and review/land #72648 or replace it with a focused core fix. The implementation should normalize same-agent direct-message system-event routing to `resolveAgentMainSessionKey` only when `session.dmScope="main"` and the same single-owner non-wildcard allowlist rule used by inbound DM routing applies, then apply that consistently across exec enqueue/wake, cron enqueue/wake, heartbeat preflight, and event consumption. Add regression coverage for exec notify-on-exit, explicit exec system events, cron next-heartbeat events, and negative cases for wildcard/multi-owner allowlists, non-main dmScope, subagents, isolated heartbeat, and non-DM explicit sessions.
