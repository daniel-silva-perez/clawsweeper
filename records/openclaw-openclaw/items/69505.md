## Summary

Current main still does not implement the requested `assistantPrefix` contract. The field is absent from cron payload schema/types, agent RPC params, agent-default validation/types, CLI flags, docs, and the cron execution path. Existing `messages.responsePrefix` is only a delivery-time outbound prefix and does not make the provider continue from a fixed assistant prefill, so the reported automation reliability gap remains real and open.

## Best Possible Solution

Keep this open for maintainer API/design review. If accepted, implement an additive `assistantPrefix` or equivalent across cron `agentTurn` payloads, direct agent RPC/CLI runs, and agent defaults; validate and document it; carry it through cron and normal agent execution into a typed provider hook; support at least Anthropic and one OpenAI-compatible route; define explicit no-op/debug behavior for unsupported providers; and add focused tests for streaming and fallback delivery.
