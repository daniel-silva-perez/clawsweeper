## Summary

Keep this PR open. Current main still does not implement the proposed top-level `safetyPosture` config, its `development`/`balanced`/`strict` presets, or the `full`/`limited`/`public` agent profile abstraction. Existing main has explicit sandbox, session, and tool-policy knobs plus docs for hardening, but that is not the same as this PR’s preset API, and the PR head is still separate from main.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is to rebase it onto current main, integrate preset expansion into `materializeRuntimeConfig`, translate `full`/`limited`/`public` into real existing tool policy or add a typed profile contract, make `secureDmMode` and memory posture effective or remove those promises, update docs/generated config help, and add tests for config load plus the resulting sandbox, tool, session, and memory behavior.
