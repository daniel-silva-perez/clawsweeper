## Summary

Keep this PR open. Current main and v2026.4.27 still reject `message` tool `action: "send"` calls when non-empty poll creation params are present, and no stripping helper has landed; the PR remains useful but needs maintainer review, a current-main rebase, and coordination with overlapping #46455.

## What This Changes

The PR strips poll-creation fields from `message` tool `action: "send"` params instead of throwing, adds a poll-param stripping helper, and updates focused tests to expect the send path to proceed.

## Best Possible Solution

Land a current-main implementation that strips stray poll-creation params only for `action: "send"`, preserves all `action: "poll"` validation and plugin poll dispatch behavior, remains symmetric with current poll-param detection including snake_case and channel-specific `poll*` keys, and coordinates the overlap with #46455.
