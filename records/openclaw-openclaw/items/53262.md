## Summary

Keep open: the PR is useful but not mergeable as-is because it makes a shared `sessions_spawn` field globally required before the ACP path can use the existing default-agent fallback; the native subagent strict-agent use case already has a narrower config path on current `main`.

## What This Changes

The PR makes `sessions_spawn.agentId` required in the shared tool schema/runtime reader and updates spawn-related tests and mocks to pass explicit agent IDs.

## Best Possible Solution

Preserve ACP `acp.defaultAgent` compatibility while improving explicit target reliability for native subagents through the existing `requireAgentId` configuration, clearer runtime-specific prompt/schema guidance, or an additive runtime-specific schema path that requires `agentId` only where omission is not a supported contract.
