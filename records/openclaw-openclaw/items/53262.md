## Summary

Keep open. Current main still keeps `sessions_spawn.agentId` optional in the tool schema and runtime read, while also preserving ACP `acp.defaultAgent` fallback and a config-level `requireAgentId` gate. The PR addresses a real model-reliability concern, but landing it as-is is a maintainer compatibility decision because it would make the shared tool contract globally require `agentId` and conflict with the documented ACP omission path.

## What This Changes

The PR makes `sessions_spawn.agentId` required in the TypeBox schema and runtime parameter read, then updates affected agent spawn tests and config mocks to pass explicit agent IDs and assert the schema requirement.

## Best Possible Solution

Keep this PR open for maintainer product and compatibility review, but do not land it unchanged. The best path is to improve native subagent explicit-target reliability while preserving ACP `acp.defaultAgent` compatibility, likely through the shipped `requireAgentId` config path, clearer tool-description/prompt guidance, or a runtime-specific schema/tool split.
