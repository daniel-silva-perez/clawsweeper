## Summary

Keep open for maintainer product and compatibility review. Current main has not implemented the PR's schema-required behavior, but the proposed global requirement conflicts with the documented ACP `acp.defaultAgent` fallback and overlaps with an existing config-backed strict-agent policy path.

## What This Changes

This PR makes `sessions_spawn.agentId` required in the TypeBox schema and runtime parameter read, updates spawn tests to pass explicit agent IDs, and adjusts config mocks in affected agent tests.

## Best Possible Solution

Preserve ACP `acp.defaultAgent` compatibility while improving native subagent explicit-target reliability, either by leaning on the existing `agents.defaults.subagents.requireAgentId` and per-agent override path, strengthening schema/prompt guidance, or splitting runtime-specific tool schemas so native subagents can require `agentId` without making ACP fallback unreachable.
