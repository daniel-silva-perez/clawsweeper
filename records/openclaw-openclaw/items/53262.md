## Summary

Keep open: the motivation is plausible, but the proposed global requirement would regress the documented ACP default-agent path. Current main still treats `agentId` as optional and already has a configurable native strict mode, so this needs maintainer direction rather than merge-as-is cleanup.

## What This Changes

The PR branch makes `sessions_spawn.agentId` required in the shared tool schema and runtime reader, adds schema coverage, and updates spawn-related tests and mocks to pass explicit agent IDs.

## Best Possible Solution

Preserve ACP `acp.defaultAgent` compatibility while tightening native subagent target reliability through the existing `requireAgentId` configuration, stronger prompt/schema guidance, or runtime-specific schema behavior that only requires `agentId` where omission is not a supported contract.
