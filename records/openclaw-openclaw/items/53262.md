## Summary

Keep this PR open. Current main does not make `sessions_spawn.agentId` schema-required; it remains optional, and the implementation still passes omitted values through to native subagent or ACP resolution. The PR also still carries a real functional concern: making `agentId` unconditionally required would bypass the documented ACP `acp.defaultAgent` fallback before `spawnAcpDirect` can resolve it. Current main does have a safer runtime/config path for the linked native subagent concern via `subagents.requireAgentId`, but that is not the same as this PR’s schema-level change.

## Best Possible Solution

Do not close this as implemented yet. The best path is maintainer review of a narrower design: keep current `subagents.requireAgentId` runtime guard for native subagents, and only add schema-level enforcement if it can be conditional/config-aware without breaking ACP `acp.defaultAgent` fallback. If schema enforcement is still desired, update `sessions_spawn` schema generation, ACP docs, prompts, and tests together.
