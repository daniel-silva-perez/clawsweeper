## Summary

Keep open. The PR targets a real reliability problem, but the proposed global schema/runtime requirement regresses the current shared ACP/native `sessions_spawn` contract by making ACP default-agent fallback unreachable before `spawnAcpDirect` can resolve it.

## What This Changes

The PR makes `sessions_spawn.agentId` required in the TypeBox schema and runtime reader, then updates spawn-related tests and mocks to pass explicit agent IDs.

## Best Possible Solution

Preserve ACP `acp.defaultAgent` compatibility while improving native subagent target reliability through the existing `requireAgentId` config path, stronger prompt/schema guidance, or a runtime-specific schema that only requires `agentId` where omission is not a supported contract.
