## Summary

Keep open. Current main does not implement the PR's required-`agentId` schema contract, but landing the PR unchanged would change the shared ACP/native `sessions_spawn` API in a way that conflicts with current ACP default-agent behavior and docs. This needs maintainer API judgment rather than cleanup closure or an automated replacement patch.

## What This Changes

This PR makes `sessions_spawn.agentId` required in the tool schema and runtime parameter reader, then updates spawn-related tests and config mocks to pass explicit agent IDs.

## Best Possible Solution

Preserve ACP `acp.defaultAgent` compatibility while improving explicit target reliability for native subagents, either by leaning on the existing `requireAgentId` config path with stronger prompt/schema guidance or by splitting runtime-specific schema behavior so native subagents can require `agentId` without making ACP omission invalid.
