## Summary

Keep open. Current main and the latest release still keep `sessions_spawn.agentId` optional in both schema and execution, while current code already has a config-level `requireAgentId` gate for strict hierarchies. The PR is still a real product/compatibility question because the proposed runtime-level required read would fail before ACP `acp.defaultAgent` fallback can run.

## Best Possible Solution

Keep this PR open for maintainer product/compatibility review, but do not land it unchanged. The best path is to preserve ACP `acp.defaultAgent` compatibility while improving native subagent explicit-target reliability, likely by building on the shipped `requireAgentId` config path, prompt/tool-description guidance, or a runtime-specific schema/tool split.
