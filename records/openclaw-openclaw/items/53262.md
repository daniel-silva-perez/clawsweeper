## Summary

Keep open for maintainer review. The reliability goal is real, but the current PR makes `agentId` globally required and would cut off the documented ACP `acp.defaultAgent` fallback; current main instead keeps `agentId` optional while supporting config-level strict mode for native subagent spawning.

## What This Changes

The PR makes `sessions_spawn.agentId` required in the tool schema/runtime reader and updates spawn-related tests and mocks to pass explicit agent IDs.

## Best Possible Solution

Preserve ACP `acp.defaultAgent` compatibility while improving explicit target reliability for native subagents, either by strengthening the existing `requireAgentId` config/prompt path or by adding runtime-specific schema behavior so only paths where omission is unsafe require `agentId`.
