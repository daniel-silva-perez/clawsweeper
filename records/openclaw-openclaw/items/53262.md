## Summary

Keep open for maintainer review. The reliability goal is valid, but the patch makes the shared ACP/native `sessions_spawn` contract stricter in a way that regresses the documented ACP `acp.defaultAgent` fallback, and current main still preserves that fallback while offering native subagent strict mode through config.

## What This Changes

The PR makes `sessions_spawn.agentId` required in the tool schema/runtime reader and updates spawn-related tests and mocks to pass explicit agent IDs.

## Best Possible Solution

Preserve ACP `acp.defaultAgent` compatibility while improving explicit target reliability for native subagents, either by strengthening the existing `requireAgentId` config path and prompt/schema guidance or by introducing runtime-specific schema behavior that requires `agentId` only where omission is not a supported contract.

## Review Findings

- **[P2] Keep ACP default-agent fallback reachable:** `src/agents/tools/sessions-spawn-tool.ts:101`
