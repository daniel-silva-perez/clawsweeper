## Summary

Keep open. Current main does not implement the PR's required-schema behavior, and the proposed global requirement still conflicts with the documented ACP default-agent path; this needs maintainer direction on the shared ACP/native `sessions_spawn` contract before merge.

## What This Changes

The PR makes `sessions_spawn.agentId` required in the tool schema and runtime reader, then updates spawn-related tests and mocks to pass explicit agent IDs.

## Best Possible Solution

Preserve ACP `acp.defaultAgent` compatibility while improving explicit target reliability for native subagents, either by strengthening the existing `requireAgentId` config/prompt path or by introducing runtime-specific schema behavior that requires `agentId` only where omission is not a supported contract.
