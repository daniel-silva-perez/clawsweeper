## Summary

Keep open. The reported native subagent targeting problem is real, but this PR is not safe to merge as written because the global shared-tool requirement would regress the documented ACP default-agent path; maintainers need to choose the ACP/native contract before changing schema enforcement.

## What This Changes

The PR makes `sessions_spawn.agentId` mandatory in the shared tool schema/runtime reader and updates spawn-related tests and mocks to pass explicit agent IDs.

## Best Possible Solution

Keep the shared `sessions_spawn.agentId` contract compatible with ACP defaults, and improve native subagent targeting reliability through the existing `requireAgentId` strict mode, clearer prompt guidance, or an additive runtime-specific schema path that does not remove the ACP fallback.

## Reproduction Assessment

Yes. A high-confidence static reproduction is `sessions_spawn({ runtime: "acp", task: "..." })` with `acp.defaultAgent` configured: the PR's required reader throws before `spawnAcpDirect` can call `resolveTargetAcpAgentId`.

## Solution Assessment

No. A global shared-schema requirement is not the narrowest maintainable fix because ACP intentionally supports omitted `agentId`; a runtime-specific or config-backed strict path is safer.
