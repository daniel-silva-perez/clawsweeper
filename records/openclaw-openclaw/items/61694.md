## Summary

Keep open. Current main still lacks the requested global opt-out: built-in external CLI auth discovery is wired unconditionally into runtime auth-profile overlays, the provider list still reads Codex CLI, Claude CLI, and MiniMax CLI sources, and the public auth config/env surface has no `auth.externalCliSync.enabled` or `OPENCLAW_AUTH_EXTERNAL_CLI_SYNC` equivalent.

## Best Possible Solution

Keep this issue open and implement a small core auth-profile opt-out. Add a public `auth.externalCliSync.enabled` config field plus `OPENCLAW_AUTH_EXTERNAL_CLI_SYNC` env precedence, thread the resolved policy through `overlayExternalAuthProfiles` and the OAuth bootstrap/readManagedExternalCliCredential paths, either remove or correctly wire the existing unused `syncExternalCli` option, and add focused tests for default-enabled behavior, env override, config disablement, readonly mode, and Codex/Claude/MiniMax coverage. Update config help/schema docs when the key lands.
