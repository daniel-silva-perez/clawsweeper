## Summary

Keep this PR open. Current main at ef58307f843eed1a7832e9c23c519591f1b7cdbd still does not implement the proposed legacy iMessage/imsg tapback reaction forwarding: there is no includeReactions config field, no reaction payload/parser, no reaction notification handler, and watch.subscribe still sends only the attachments option. The same-author feature issue #39031 is still open, so the paired contributor issue/PR should remain open for maintainer review.

## Best Possible Solution

Keep this PR open alongside #39031. The best path is to refresh it against current main, verify the imsg reaction RPC payload/subscribe flag, add includeReactions across config types/schema/generated metadata/docs, keep parser and monitor tests, and explicitly decide how tapback events interact with group mention gating. If maintainers decide legacy imsg should not gain this feature, close or supersede both the PR and #39031 with that product decision.
