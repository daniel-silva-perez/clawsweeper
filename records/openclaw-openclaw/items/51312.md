## Summary

Keep this issue open. Current main still only supports the global `agents.defaults.userTimezone`; it does not accept `agents.list[].userTimezone`, and the prompt/time/heartbeat call sites still resolve timezone directly from the global default. The requested per-agent override is clear, plausible, and not covered by the related per-agent override items in the supplied context.

## Best Possible Solution

Implement a shared per-agent timezone resolver, add `userTimezone?: string` to `AgentConfig` and `AgentEntrySchema`, and update system prompt, cron-style current-time, heartbeat active-hours, timestamp/envelope-related gateway paths, tests, and config docs so precedence is `agents.list[].userTimezone` -> `agents.defaults.userTimezone` -> host timezone.
