## Summary

Keep this PR open. The linked Windows Gmail setup bug is valid on current main and the PR targets the right helper-command area, but the latest head is not merge-ready because it still carries unrelated Hyprland/browser SDK changes, references a missing Hyprland capture module, and leaves the long-running `gog` watcher on a direct resolved `.cmd` spawn path.

## What This Changes

The PR adds a Windows executable resolver for Gmail `gcloud`, `gog`, and `tailscale` calls, while also carrying Hyprland browser screenshot config/docs/tests and an SDK `infra-runtime` export.

## Best Possible Solution

Narrow the branch to the Windows Gmail helper-command fix: resolve PATHEXT shims for `gcloud`, `gog`, and `tailscale`, ensure long-running `.cmd` invocations go through the shared Windows-safe wrapper or an equivalent generic spawn-safe seam, remove the unrelated Hyprland/browser SDK changes, and cover the setup/run/watcher paths with focused Windows tests.

## Reproduction Assessment

Yes. The linked bug has concrete native PowerShell reproduction, current main still shows the bare helper-command paths, and the PR-specific failures are statically reproducible from the latest diff.

## Solution Assessment

No. The resolver direction is appropriate, but this implementation is not the best fix because it keeps unrelated browser/SDK work in the branch and still bypasses the shared Windows wrapper for the watcher process.
