## Summary

Keep this PR open, but it should not merge as-is. Current main still has the Windows Gmail helper spawn gap reported in #54470, but this branch mixes that narrow fix with unrelated Hyprland browser work and introduces blocking issues in the proposed patch: a bundled plugin imports the deprecated broad SDK runtime barrel, and the gateway Gmail watcher can still directly spawn a resolved `.cmd` shim.

## What This Changes

The PR adds a Windows helper-command resolver for `gcloud`, `gog`, and `tailscale`, wires it into Gmail setup/watcher paths, and also adds an unrelated opt-in Hyprland browser screenshot implementation with docs, config, tests, and an SDK export.

## Best Possible Solution

Rework this as a narrow Gmail helper-command fix: resolve Windows PATHEXT shims into an invocation that is compatible with both `runCommandWithTimeout` and the long-running watcher, cover `gcloud`, `gog`, and `tailscale` with Windows-focused tests, and split the Hyprland browser work into a separate PR with a focused SDK/runtime seam.

## Reproduction Assessment

Yes. The linked bug has concrete native PowerShell reproduction, and current main still shows the matching bare `gcloud`, `gog`, and `tailscale` spawn paths; the PR-specific boundary failure is also statically reproducible from the scanner source and new import.

## Solution Assessment

No. The intended Windows helper resolution is the right direction, but this is not the best implementation because it mixes unrelated browser behavior, violates the bundled-plugin SDK import guard, and leaves a direct `.cmd` watcher spawn path outside the existing Windows wrapper.
