## Summary

Keep this PR open. The linked Windows Gmail setup bug is real and still present on current main, but this branch has blocking correctness/boundary findings and mixes unrelated Hyprland browser/SDK work into a Gmail command-spawn fix.

## What This Changes

The PR adds a Windows executable resolver for Gmail `gcloud`, `gog`, and `tailscale` calls, and also adds opt-in Hyprland browser screenshot capture with config, docs, tests, and an SDK export.

## Best Possible Solution

Rework this as a narrow Windows Gmail helper-command fix: resolve PATHEXT shims into spawn-safe invocations for `gcloud`, `gog`, and `tailscale`, cover setup/run/watcher paths with Windows-focused tests, and split the Hyprland browser capture work into a separate PR with a focused SDK/runtime seam.

## Reproduction Assessment

Yes. The linked bug has concrete native PowerShell reproduction, and current main still shows the matching bare helper-command paths; the PR-specific failures are statically reproducible from the diff and boundary scanner source.

## Solution Assessment

No. The Windows resolver direction is appropriate, but this PR is not the best implementation because it leaves the watcher on a direct resolved `.cmd` spawn path and folds unrelated browser/SDK work into a targeted Gmail fix.
