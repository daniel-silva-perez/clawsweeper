## Summary

Keep this PR open. Current main still lacks the gateway reset-archive fallback that the PR adds: `readSessionMessages` and title-field derivation only read existing primary transcript candidates, while reset flows still archive transcripts as `.reset.<timestamp>` files. Related items cover broader or different surfaces rather than superseding this focused read-path fix.

## Best Possible Solution

Keep this PR open as the focused implementation candidate for reading the newest valid reset archive when gateway history or title derivation is asked for a session whose primary transcript is missing. The best path is maintainer review of the stale-candidate privacy boundary, then landing this patch or folding equivalent behavior into the gateway session read helpers while leaving archive UI, compaction-chain recovery, memory-hook behavior, and session-log discovery to their related items.
