## Summary

Keep this PR open. Current main still returns an empty history when the primary transcript is missing, while reset flows still archive transcripts as `.reset.<timestamp>` files. The related open items cover broader or disjoint surfaces rather than superseding this focused read-path fix.

## Best Possible Solution

Keep this PR open as the focused implementation candidate for reading the newest valid reset archive when `readSessionMessages` or session title derivation is asked for a session whose primary transcript is missing. The best path is maintainer review of the stale-candidate privacy boundary, gateway changed checks, then either landing this patch or folding equivalent behavior into a broader reset-archive recovery change while leaving compaction chains, memory hooks, session-log discovery, and archive UX to their related items.
