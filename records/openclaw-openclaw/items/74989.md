## Summary

Keep open. Current main still has the undefined-result crash path, and this PR is a plausible narrow fix with focused regression coverage, but it needs maintainer review and duplicate-fix selection because #74838 and #74856 are open for the same linked regression.

## What This Changes

The PR normalizes nullish or non-object plugin command results to empty reply payloads in the shared executor, generic command dispatch, and Telegram native command handling, with regression tests for those paths.

## Best Possible Solution

Land one canonical narrow fix that normalizes plugin command results at the shared executor boundary, keeps direct or mocked dispatch paths resilient, and then close or supersede the overlapping PRs. If no-payload command handlers are supported behavior, align the plugin SDK contract afterward.

## Reproduction Assessment

Yes. On current main, a plugin command handler that resolves undefined can pass through executePluginCommand and then crash when handlePluginCommand reads result.continueAgent; #74800 provides matching v2026.4.27 Telegram logs.

## Solution Assessment

Yes, mostly. Normalizing at the executor boundary is the narrowest maintainable fix, and the caller-side guards are reasonable defense for direct or mocked runtime paths; the remaining decision is duplicate PR selection and contract wording.
