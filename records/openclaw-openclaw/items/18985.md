## Summary

Keep open. Current main still rejects Windows 11 MSYS2/MINGW-style `install.sh` execution as an unknown OS, and the installer PATH guidance/persistence remains bash/zsh-only rather than fish-aware. The related Microsoft tracker #49126 is closed and unmerged, so it does not supersede the remaining work.

## Best Possible Solution

Keep this issue open until maintainers either explicitly support MSYS2/MINGW in `scripts/install.sh` with environment detection, Windows/MSYS path and package-manager handling, fish PATH persistence or guidance, docs, and installer smoke coverage; or make a documented product decision that Windows Unix-like environments should use WSL2 or the PowerShell installer instead.

## Risks / Open Questions

- Supporting MSYS2/MINGW by only removing the unsupported-OS guard could run Linux package-manager, service, and path assumptions in a Windows/MSYS environment with different behavior.
- The installer is a curl-piped code execution path, so platform detection and PATH persistence changes should get maintainer review plus real Windows 11 MSYS2/MINGW smoke coverage.
- Fish PATH persistence needs fish syntax and safe handling for `~/.config/fish/config.fish`; bash-style `export PATH=...` guidance is not enough for fresh fish terminals.
