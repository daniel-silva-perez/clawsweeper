## Summary

Keep open. Current main still rejects Windows 11 MSYS2/MINGW-style `install.sh` execution as an unknown OS, and the installer PATH repair/warning paths remain bash/zsh-oriented rather than fish-aware. The refreshed Microsoft tracker #74163 is open tracking work, not a merged implementation, and the older tracker #49126 is closed unmerged.

## Best Possible Solution

Keep this issue open until maintainers either explicitly support MSYS2/MINGW in `scripts/install.sh` with environment detection, Windows/MSYS-safe install decisions, fish PATH persistence or fish-specific guidance, docs, and installer tests/smoke coverage; or make a documented product decision that Windows Unix-like environments should use WSL2 or the PowerShell installer instead.

## Risks / Open Questions

- Supporting MSYS2/MINGW by only removing the unsupported-OS guard could run Linux package-manager, service, and path assumptions in a Windows/MSYS environment with different filesystem and process behavior.
- The installer is a curl-piped code execution path, so platform detection and PATH persistence changes should get maintainer review plus real Windows 11 MSYS2/MINGW smoke coverage before release.
- Fish PATH persistence needs fish syntax and safe handling for `~/.config/fish/config.fish`; bash-style `export PATH=...` guidance is not enough for new fish terminals.
