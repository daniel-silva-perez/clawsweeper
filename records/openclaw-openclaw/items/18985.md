## Summary

Keep open. Current main still rejects the reported MSYS2/MINGW `install.sh` path, still gives bash/zsh-only PATH persistence and warning guidance, and the docs continue to route Windows users to WSL2 or the PowerShell installer. The issue is valid, but enabling MSYS2/MINGW is a Windows support-boundary decision rather than a conservative cleanup close.

## What This Changes

Request support for installing and running OpenClaw from Windows 11 MSYS2/MINGW Unix-like shells, with fish shell PATH guidance.

## Best Possible Solution

Maintain this as an open platform-support decision: either add tested MSYS2/MINGW support to `scripts/install.sh` with Windows/MSYS-safe install behavior, fish-aware PATH handling, docs, and smoke coverage, or explicitly document that Windows Unix-like environments should use WSL2 or the PowerShell installer.
