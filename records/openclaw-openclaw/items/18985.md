## Summary

Keep open. Current main still rejects the reported Windows MSYS2/MINGW `install.sh` path, still persists and recommends PATH fixes only for bash/zsh, and the related Microsoft tracker #74163 is an open tracking PR rather than an implementation.

## What This Changes

The issue requests support for running the Unix installer under Windows 11 MSYS2/MINGW shells and for fish-aware PATH guidance or persistence.

## Best Possible Solution

Maintain the current boundary until maintainers either explicitly support MSYS2/MINGW in `scripts/install.sh` with tested Windows/MSYS-safe install behavior, fish-aware PATH handling, docs, and smoke coverage, or document that Windows Unix-like environments should use WSL2 or the PowerShell installer.

## Reproduction Assessment

Yes. A read-only shell check against current main reproduces the installer rejection by sourcing `scripts/install.sh` with `OPENCLAW_INSTALL_SH_NO_RUN=1`, setting `OSTYPE=msys` and `MSYSTEM=MINGW64`, then calling `detect_os_or_die`.

## Solution Assessment

No. The proposed workaround of commenting out the unsupported-OS guard demonstrates that the npm path can proceed, but it bypasses the platform contract and does not address Windows/MSYS safety, fish PATH persistence, docs, or smoke coverage.
