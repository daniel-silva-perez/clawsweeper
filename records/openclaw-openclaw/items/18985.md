## Summary

Keep open. Current main still does not implement the requested Windows 11 MSYS2/MINGW `install.sh` support, and installer PATH persistence guidance remains bash/zsh-only rather than fish-aware. The related Microsoft tracker #49126 is closed and unmerged, so it does not supersede this issue.

## Best Possible Solution

Keep this issue open until maintainers either support MSYS2/MINGW in `scripts/install.sh` with explicit environment detection, Windows/MSYS path and package-manager handling, fish `config.fish` PATH persistence or guidance, docs, and installer smoke coverage; or make a documented product decision that Windows Unix-like environments should use WSL2 or the PowerShell installer instead.
