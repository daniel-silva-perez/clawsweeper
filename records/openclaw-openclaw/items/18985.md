## Summary

Keep open. Current main still rejects MSYS2/MINGW-style `install.sh` execution as an unknown OS, still documents `install.sh` as macOS/Linux/WSL only, and still gives bash/zsh-only PATH persistence and warning guidance instead of fish-aware guidance. The refreshed Microsoft tracker #74163 is open tracking work, not an implementation.

## What This Changes

Request support for installing and running OpenClaw from Windows 11 MSYS2/Unix-like shells, including fish shell PATH guidance.

## Best Possible Solution

Keep this open until maintainers either explicitly support MSYS2/MINGW in `scripts/install.sh` with tested OS detection, Windows/MSYS-safe install decisions, fish PATH persistence or fish-specific guidance, docs, and smoke coverage; or document that Windows Unix-like environments should use WSL2 or `install.ps1` instead.
