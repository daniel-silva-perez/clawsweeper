## Summary

Keep open. Current main still rejects the reported Windows MSYS2/MINGW install.sh path and still gives bash/zsh-only PATH persistence guidance, while the related Microsoft tracker is open tracking work rather than an implementation.

## What This Changes

Request support for running OpenClaw install.sh in Windows 11 MSYS2/MINGW Unix-like environments and for fish shell PATH configuration guidance.

## Best Possible Solution

Maintain the current boundary until maintainers either explicitly support MSYS2/MINGW in scripts/install.sh with Windows/MSYS-safe install behavior, fish-aware PATH handling, docs, and smoke coverage, or document that Windows Unix-like environments should use WSL2 or the PowerShell installer.

## Reproduction Assessment

Yes. A read-only shell check against current main reproduces the installer rejection by sourcing scripts/install.sh with OPENCLAW_INSTALL_SH_NO_RUN=1 and calling detect_os_or_die under OSTYPE=msys and MSYSTEM=MINGW64.

## Solution Assessment

No for the proposed workaround. Commenting out the unsupported-OS guard demonstrates that the npm path can advance, but it bypasses the platform contract and does not address Windows/MSYS safety, fish PATH persistence, docs, or smoke coverage.
