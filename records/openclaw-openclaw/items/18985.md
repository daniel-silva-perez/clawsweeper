## Summary

Keep open. Current main still rejects the reported MSYS2/MINGW `install.sh` path, still gives bash/zsh-only PATH persistence and warning guidance for the fish-shell part of the request, and the related Microsoft tracker is open coordination rather than an implementation. A maintainer has also already reviewed this as an open Windows platform support decision.

## What This Changes

Request support for installing and running OpenClaw from Windows 11 MSYS2/MINGW Unix-like shells, including fish-aware installer PATH guidance.

## Best Possible Solution

Maintain this as an open platform-support decision: either add tested MSYS2/MINGW support to `scripts/install.sh` with Windows/MSYS-safe install behavior, fish-aware PATH handling, docs, and smoke coverage, or explicitly document that Windows Unix-like environments should use WSL2 or the PowerShell installer.
