## Summary

Keep open. Current main still does not implement the requested Windows 11 MSYS2/MINGW `install.sh` support, and the installer still gives bash/zsh-only PATH persistence guidance instead of fish-aware `config.fish` handling. Docs and release validation continue to route native Windows to `install.ps1` and WSL2 to `install.sh`; existing fish completion support is adjacent, not a fix for installer PATH persistence.

## Best Possible Solution

Keep this issue open until maintainers either explicitly support MSYS2/MINGW in `scripts/install.sh` with environment detection, Windows/MSYS path and package-manager handling, fish `config.fish` PATH persistence or guidance, docs, and installer smoke coverage; or make a documented product decision that Windows Unix-like environments should use WSL2 or the PowerShell installer instead.
