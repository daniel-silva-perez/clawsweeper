## What This Changes

The issue reports that `memoryFlush` is skipped on alternating eligible auto-compaction cycles because flush metadata is persisted with the post-increment compaction count.

## Best Possible Solution

Land #51421 or an equivalent focused patch that preserves the pre-increment flush marker after flush-triggered compaction, updates the synchronized-counter test expectation, adds consecutive eligible auto-compaction coverage, and keeps the documented once-per-cycle behavior.
