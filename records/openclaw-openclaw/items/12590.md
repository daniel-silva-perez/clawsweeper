## Summary

Current main still appears to preserve the reported bug: a memory-flush-triggered compaction writes the post-increment compaction count into `memoryFlushCompactionCount`, while the flush gate skips when that marker equals `compactionCount`. The latest shipped release still has the same path, and open PR #51421 is already a closing fix candidate.

## Best Possible Solution

Keep this issue open and review/land PR #51421, or an equivalent focused patch, so a flush-triggered compaction preserves the pre-increment flush marker instead of writing the post-increment `compactionCount` into `memoryFlushCompactionCount`. Update the synchronized-counter test expectation and add regression coverage for consecutive eligible auto-compaction cycles; coordinate with #47143 if the initial-counter edge overlaps.
