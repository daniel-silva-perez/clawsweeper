## Summary

Keep open. Current main and the latest shipped tag still persist the post-increment compaction count as the memory-flush marker while the gate skips equal counters, and open PR #51421 already targets this issue with closing syntax.

## What This Changes

Make pre-compaction memoryFlush run before every eligible auto-compaction cycle instead of alternating because the flush marker is synchronized with the compaction counter.

## Best Possible Solution

Land #51421 or an equivalent focused patch that preserves the pre-increment flush marker after flush-triggered compaction, updates the synchronized-counter test expectation, adds consecutive eligible auto-compaction coverage, and preserves the documented once-per-cycle behavior.
