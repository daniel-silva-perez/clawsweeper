## Summary

Current main already has Control UI localization support for zh-CN, including browser-locale detection, a manual Language picker, and many translated navigation/button labels. The specific remaining request is not fully implemented: the event/audit log still renders raw event identifiers, chat tool cards still use English shared tool display labels for exec/read/cron/subagents/browser, and the Debug view still has hardcoded English copy.

## Best Possible Solution

Keep this issue open and narrow the implementation: add i18n-backed display labels for audit/event/tool operation names such as exec, read, cron run, subagents list, and browser; replace remaining hardcoded Debug/Event Log copy with translation keys; preserve raw identifiers in payload/details where useful; and add focused i18n tests for zh-CN rendering and English fallback.
