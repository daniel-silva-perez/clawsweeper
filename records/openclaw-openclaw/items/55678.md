## Summary

Keep this open. Current main still cannot show Qwen/DashScope provider quota windows because qwen, qwencloud, modelstudio, and dashscope do not resolve to a supported usage-provider id, and the bundled Qwen plugin still has no usage auth or usage snapshot hook.

## What This Changes

The issue requests DashScope/Qwen usage-display support because UI usage remains 0 even while Qwen/DashScope calls consume tokens.

## Best Possible Solution

Add Qwen-owned usage-window support through the existing provider plugin hooks, with minimal core type/alias additions so qwen, qwencloud, modelstudio, and dashscope resolve to the bundled Qwen usage provider. Keep provider-specific endpoint/auth parsing in extensions/qwen, add focused usage/status tests, and update Qwen plus token-use docs.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure provider qwen, qwencloud, modelstudio, or dashscope and resolveUsageProviderId returns undefined, so status, models.authStatus, and model-list usage paths never request Qwen/DashScope quota windows.

## Solution Assessment

No current solution exists on main. The narrow maintainable fix is not only adding dashscope to usageProviders; it also needs Qwen-owned resolveUsageAuth and fetchUsageSnapshot behavior backed by the official DashScope/Qwen usage API contract.
