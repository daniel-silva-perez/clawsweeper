## Summary

Keep this issue open. Current main still has Qwen/DashScope model and streaming-token support, but it does not implement DashScope/Qwen provider-usage quota snapshots, so the reported UI usage-window gap remains real.

## Best Possible Solution

Implement this in the bundled Qwen provider boundary: add Qwen/DashScope usage provider ids or alias resolution, have extensions/qwen own resolveUsageAuth and fetchUsageSnapshot for qwen/qwencloud/modelstudio/dashscope using QWEN_API_KEY, MODELSTUDIO_API_KEY, and DASHSCOPE_API_KEY, cover Standard and Coding Plan endpoints, then update usage-tracking/Qwen docs and add focused provider-usage/status tests.
