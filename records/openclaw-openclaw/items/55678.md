## Summary

Keep this issue open. Current main still does not implement DashScope/Qwen provider-usage quota snapshots, so the reported UI/status usage-window gap remains real.

## Best Possible Solution

Implement the remaining work in the bundled Qwen provider boundary: add Qwen/DashScope usage-provider id or alias resolution, implement Qwen-owned usage auth and snapshot fetching for qwen/qwencloud/modelstudio/dashscope using the accepted env/config/profile credentials, cover both Standard and Coding Plan usage endpoints as applicable, then add focused provider-usage/status tests and update Qwen/usage-tracking docs.
