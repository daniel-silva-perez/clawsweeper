## Summary

Keep open. Current main still implements boundary-based steering with streaming predicates and followup fallback, and the issue is paired with open implementation PRs, including a protected maintainer-labeled replacement path, so cleanup closure would be premature.

## What This Changes

The issue requests a soft-steer mode that can interrupt an active generation, inject new user input into context, and resume with that input visible immediately.

## Best Possible Solution

Keep this issue open as the runtime/product tracker until #73365, #64758, or a narrower maintainer-approved successor lands with documented lifecycle semantics and regression coverage for embedded runner, reply-run registry, backend failure, fallback behavior, and abort/resume handling.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path is an active session with `messages.queue.mode: "steer"` where `isActive` is true but `isStreaming` is false; current main skips immediate injection and falls through to followup.

## Solution Assessment

Unclear until maintainer review completes. The requested direction is reasonable, but the maintainable fix should be a narrow lifecycle-aware steer path like #73365 or a successor rather than a new autonomous repair branch.
