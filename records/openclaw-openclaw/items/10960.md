## Summary

Keep this issue open. Current main still implements steer as a streaming/model-boundary path with follow-up fallback, not the requested pause-or-abort, inject, and resume lifecycle; the issue is also paired with open implementation PRs, including maintainer-labeled #73365 and closing-syntax PR #64758, so it should not be closed before that work lands.

## What This Changes

The issue requests soft-steer behavior that lets a user's mid-generation follow-up interrupt or pause an active run, enter context immediately, and resume with the new input instead of waiting for a tool or model boundary.

## Best Possible Solution

Keep this as the product/runtime tracker until #73365 or a narrower successor lands, with documented and tested lifecycle semantics for streaming, active non-streaming/tool execution, abort or resume behavior, backend failure, and follow-up fallback.
