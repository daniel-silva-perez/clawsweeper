## Summary

Keep open. Current main improves batched steering, but it still implements boundary-based queueing rather than the requested pause-or-abort, inject, and resume lifecycle; the issue is also paired with open implementation PRs, including #64758 and maintainer-labeled #73365.

## What This Changes

The issue requests a soft-steer queue/runtime mode that can interrupt or pause an active agent generation, inject a new user message immediately, and resume with the updated context.

## Best Possible Solution

Keep this issue open as the product/runtime tracker until #73365, #64758, or a narrower maintainer-approved successor lands with documented lifecycle semantics and regression coverage for embedded runner, reply-run registry, backend failure, and fallback behavior.
