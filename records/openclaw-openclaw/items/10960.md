## Summary

Keep this issue open. Current main still implements steer as streaming/model-boundary injection with followup fallback, not the requested pause-or-abort, inject, and resume lifecycle. The issue is also paired with open implementation PRs using closing syntax, and there is an open maintainer-labeled replacement PR for the same active non-streaming steer gap.

## What This Changes

Add soft-steer behavior that can pause or abort active generation, inject the user's new message into context, and resume with awareness of that input instead of waiting for the next tool or model boundary.

## Best Possible Solution

Keep this issue open as the product/runtime tracker. A complete fix should define soft-steer lifecycle semantics across streaming generation, active non-streaming/tool execution, backend failure, fallback, and abort/resume boundaries; keep hard interrupt behavior distinct; update queue and agent docs; and land focused regression coverage through the canonical maintainer-reviewed PR path.
