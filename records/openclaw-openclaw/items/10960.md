## Summary

Keep open. Current main still gates steering through streaming-only predicates and documented model-boundary/followup behavior, and the issue is paired with unmerged implementation work, including closing PR #64758 and maintainer-labeled PR #73365 referenced in the discussion.

## What This Changes

The issue requests true mid-stream soft steering: pause or abort an active generation, inject the user’s new message into context immediately, and resume generation with that input instead of waiting for the next tool or model boundary.

## Best Possible Solution

Keep this issue as the product/runtime tracker until #73365, or a narrower maintainer-approved successor, lands with documented lifecycle semantics and regression coverage for embedded runner and reply-run registry steering paths.
