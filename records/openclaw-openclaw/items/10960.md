## Summary

Keep this issue open. Current main still implements steer as a streaming/model-boundary path with follow-up fallback, not the requested pause-or-abort, inject, and resume lifecycle, and the issue is paired with active implementation work including closing-syntax PR #64758 and maintainer-labeled PR #73365.

## What This Changes

Request true mid-stream soft steering so a user message can interrupt active generation, enter the run context immediately, and let the agent continue with the new input.

## Best Possible Solution

Keep this as the product/runtime tracker until #73365 or a narrower successor lands with documented and tested lifecycle semantics for streaming, active non-streaming/tool execution, abort or resume behavior, backend failure, and fallback follow-up behavior.
