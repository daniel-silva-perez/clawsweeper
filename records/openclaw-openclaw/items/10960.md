## Summary

Keep this issue open. Current main still implements steer as a streaming/model-boundary path with followup fallback, not the requested pause-or-abort, inject, and resume soft-steer lifecycle. There is also an open PR with closing syntax for this issue, plus an open maintainer replacement PR, so the right cleanup path is to review/land or close those PRs first.

## Best Possible Solution

Keep this issue open as the product/runtime tracker. Review and either land or close #64758 and the maintainer replacement #73365. A complete fix should define soft-steer lifecycle semantics across streaming generation, active non-streaming/tool execution, backend failure, and fallback; replace streaming-only gates with a safe lifecycle predicate; keep gateway hard-interrupt behavior distinct; update queue/agent docs; and add regression coverage for streaming steer, active non-streaming steering, fallback behavior, and abort/resume boundaries.
