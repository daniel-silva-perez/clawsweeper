## Summary

Keep this issue open: current main still gates steer injection on streaming/model-boundary behavior and falls back to followup rather than implementing the requested pause-or-abort, inject, and resume lifecycle. It is also actively paired with open implementation PRs, including a maintainer-labeled replacement PR, so closing before that work lands would bypass the normal issue/PR flow.

## What This Changes

Add soft-steer behavior so a user's mid-generation follow-up can pause or abort the active run, enter context immediately, and resume with that new input instead of waiting for the next boundary.

## Best Possible Solution

Keep this issue open as the product/runtime tracker. Land #73365 or a narrower successor after maintainer review, with lifecycle semantics documented for streaming, active non-streaming/tool execution, abort/resume, fallback, backend failure, and regression coverage in the embedded runner and reply-run registry paths.
