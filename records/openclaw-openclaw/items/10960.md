## Summary

Keep this issue open. Current main still only attempts steer injection while a run is streaming, rejects active-but-not-streaming queue injection, and documents/test-covers followup fallback rather than the requested pause-or-abort, inject, and resume soft-steer lifecycle. A linked open implementation PR targets this issue, so the right path is review/land or close that PR first.

## Best Possible Solution

Keep this issue open as the product/runtime tracker. The best path is to review and either land or close the linked implementation PR #64758, and also account for the maintainer replacement work in #73365. A complete fix should define soft-steer lifecycle semantics across streaming generation, tool execution, backend failure, and fallback; replace streaming-only acceptance gates with a safe active/stopped predicate; keep gateway `sessions.steer` hard-interrupt semantics distinct; update queue/agent docs; and add regression coverage for streaming steer, active non-streaming/tool-wait steering, fallback behavior, and abort/resume boundaries.
