## Summary

Keep open. Current main still ships `skills/canvas/SKILL.md` without YAML frontmatter, and the active bundled skill loader requires frontmatter-derived `description` metadata before a skill is loaded. The provided GitHub context also shows open, unmerged PR #54361 as the implementation candidate, so this issue should remain open until that or an equivalent fix lands.

## Best Possible Solution

Keep this issue open. Review and land PR #54361, or apply an equivalent fix on main: add valid `name` and trigger-rich `description` YAML frontmatter to `skills/canvas/SKILL.md`, then add narrow bundled-skill metadata coverage so canvas and future bundled skills cannot regress.
