## Summary

Keep this PR open. Current main still does not implement the PR's intended leading `to=... {json}` user-facing strip pass or matching regression coverage. Adjacent sanitizer, diagnostic, and structured tool-call repair code covers other tool-text shapes, but not the raw final assistant text shape from #41844.

## Best Possible Solution

Keep this PR open for maintainer or contributor follow-up. Rebase it onto the split sanitizer module, constrain matching to a conservative registered-tool transcript shape, and add regressions for the Discord `to=cron ... {json}` leak plus leading-prose false positives. Treat the cron non-execution behavior from #41844 as a separate fix path.
