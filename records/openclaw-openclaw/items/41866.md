## Summary

Keep this PR open. Current main still does not implement the PR’s intended leading `to=... {json}` user-facing strip pass or matching regression tests. The existing code strips adjacent XML/bracketed/downgraded tool scaffolding and logs diagnostics for tool-shaped assistant text, but it still treats this leak shape as text rather than sanitizing it.

## Best Possible Solution

Keep this PR open for maintainer or contributor follow-up. Rebase it onto current main, port the logic into `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts` or the shared assistant-visible sanitizer, constrain matching to a conservative registered-tool transcript shape, and add regressions for the Discord `to=cron ... {json}` leak plus leading-prose false positives. Treat the structured cron execution failure from #41844 as a separate root-cause path from display sanitization.
