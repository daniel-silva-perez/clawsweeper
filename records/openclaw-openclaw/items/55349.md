## Summary

Keep this issue open. Current main has some assistant-side `[[reply_to_*]]` display metadata, but it does not implement the requested Control UI workflow: selecting a prior chat message, showing a quoted draft above the composer, sending structured/visible quote context, and rendering that quote on the resulting user reply. The related open project-tree/thread-entry issue is broader and not an exact canonical replacement for this quote-reply request.

## Best Possible Solution

Implement this as a focused Control UI chat feature: add selected-reply state in the chat view/app state, expose a per-message Reply affordance for mouse and touch, render a cancellable quote preview above the composer, and persist/send enough quote metadata or normalized quoted text through `chat.send`/transcript history so the sent user reply and later history reloads keep the context. Add focused UI tests for selecting, cancelling, sending, and reloading quoted replies. Keep #7817 as the broader project/thread organization discussion rather than treating it as a replacement for this narrower quote-reply request.
