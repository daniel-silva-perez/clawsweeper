## Summary

Keep open. Current main does not implement the requested Control UI quote-reply workflow: the UI can render an existing assistant reply marker, but there is no message-level Reply action, no selected quote state above the composer, and no `chat.send` reply/quote metadata. The related #7817 is relevant but broader, so it is not a precise duplicate for this specific UX request.

## What This Changes

Add a Control UI chat affordance to reply to an earlier message, show the selected quote above the composer, and render sent replies with quoted context.

## Best Possible Solution

Decide whether Control UI quote replies are a lightweight per-message quote or part of the broader project/thread model, then implement the chosen path with durable transcript/protocol support and message-bubble plus composer UX that works on desktop hover and mobile long-press.
