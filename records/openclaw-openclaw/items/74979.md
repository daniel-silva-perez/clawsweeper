## Summary

Keep open. The PR fixes the already-aborted-before-prompt case, but it still leaves the abort-during-`AgentSession.prompt()` preflight window where Pi has not created `activeRun`, so the new `agent.abort()` backstop is a no-op and the prompt promise can later start an uncancelled Agent loop.

## What This Changes

The PR adds an embedded-runner pre-prompt abort guard, abort/queue cleanup in `abortable()` and `finally`, test harness support, and a new abort-before-prompt regression test.

## Best Possible Solution

Make the actual prompt-start boundary cancellation-aware. The maintainable end state is either upstream `PromptOptions.signal` support in Pi or an OpenClaw wrapper that checks the run abort signal through preflight and prevents `Agent.prompt()` from starting after cancellation, with regression coverage for pre-aborted, abort-during-preflight, and abort-after-activeRun cases.

## Reproduction Assessment

Yes. The linked bug gives a deterministic pre-aborted-signal reproduction, and the remaining PR gap can be reproduced by making `AgentSession.prompt()` await during preflight, aborting after the new guard passes, and verifying that `agent.abort()` has no `activeRun` to cancel before `Agent.prompt()` starts.

## Solution Assessment

No. The patch is a useful partial fix, but it is not the best complete solution because the new backstops only help before `prompt()` is called or after Pi has created `activeRun`; they do not cancel the preflight interval inside `AgentSession.prompt()`.
