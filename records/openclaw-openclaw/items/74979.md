## Summary

Keep open. The PR fixes the already-aborted pre-prompt case, but the latest head still leaves the abort-after-guard/before-`Agent.prompt()` window that the prior review called out; current `@mariozechner/pi-coding-agent` preflight can await before creating an active Pi run, while `Agent.abort()` is a no-op until that active run exists.

## What This Changes

The PR adds an embedded-runner pre-prompt abort guard, Agent abort/queue cleanup in abort handling and cleanup, and a new abort-before-prompt regression test with test harness support.

## Best Possible Solution

Make the prompt-start boundary abort-aware so no `AgentSession.prompt()` promise can advance into a new Pi `Agent` run after the OpenClaw run signal aborts. The narrowest maintainable fix is either a lazy abortable prompt helper that checks immediately before the actual prompt start or a session/upstream cancellation seam that lets `AgentSession.prompt()` preflight observe the run `AbortSignal`, with regression coverage for the abort-during-preflight window.

## Reproduction Assessment

Yes. The linked issue gives a deterministic pre-aborted-signal reproduction, and the remaining PR-specific gap can be reproduced by aborting after this PR's new guard but before `AgentSession.prompt()` reaches `Agent.prompt()` after an awaited preflight step.

## Solution Assessment

No. The pre-prompt guard is a good part of the fix for already-aborted signals, but the proposed cleanup is not sufficient because `Agent.abort()` cannot cancel a prompt that is still in `AgentSession.prompt()` preflight before an active Pi run exists.

- **[P2] Close the preflight abort window:** `src/agents/pi-embedded-runner/run/attempt.ts:2084-2085`
  - body: `AgentSession.prompt()` can still await extension/input/auth/compaction/before-start preflight after the new guard and before it calls `Agent.prompt()`. If the run aborts in that interval, these `agent.abort()` calls are no-ops because `activeRun` has not been created yet, then the floating prompt promise can later start an uncancelled Agent run. Please make prompt creation abort-aware at the actual start boundary, or pass a cancellation signal into the session preflight, and cover that race.
