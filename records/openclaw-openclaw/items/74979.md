## Summary

Keep open. The linked bug is valid and current main still starts `activeSession.prompt()` before `abortable()` can reject, but this PR remains a partial fix: it handles the already-aborted path while leaving the acknowledged abort-during-`AgentSession.prompt()` preflight window where Pi has not created `activeRun` yet.

## What This Changes

The PR adds an embedded-runner pre-prompt abort guard, abort/queue cleanup during abort handling and cleanup, test-harness support for agent abort methods, and a new abort-before-prompt regression test.

## Best Possible Solution

Revise this branch so the actual prompt-start boundary is cancellation-aware across preflight and Agent startup, while keeping the narrow already-aborted and active-run backstop coverage. The expected end state is regression coverage for already-aborted, abort-during-preflight-before-`activeRun`, and abort-after-`activeRun` cases without bundling the separate per-run LLM hard-cap idea into this PR.

## Reproduction Assessment

Yes. The linked bug report provides deterministic pre-aborted-signal coverage and production log signatures; the remaining PR gap can be reproduced by aborting after the new guard while `AgentSession.prompt()` is awaiting preflight before `Agent.activeRun` exists.

## Solution Assessment

No. The PR is a useful partial fix, but it is not the best complete solution because `agent.abort()` cannot cancel Pi session preflight before `Agent.prompt()` creates `activeRun`; cancellation needs to reach that preflight or the prompt-start boundary must be made lazy and abort-aware.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Make prompt preflight cancellation-aware:** `src/agents/pi-embedded-runner/run/attempt.ts:2718-2726`
  - body: The new guard only checks the signal before calling `AgentSession.prompt()`. In the pinned Pi version, `AgentSession.prompt()` can still await input handlers, auth/model checks, compaction, and `emitBeforeAgentStart()` before it calls `agent.prompt()`. If the run aborts in that window, the added `agent.abort()` calls are still no-ops because `activeRun` does not exist yet, and the prompt promise can later start an uncancelled Agent loop after `abortable()` rejects. Please pass cancellation through session preflight or move the abort-aware boundary to the actual `Agent.prompt()` start.
  - confidence: 0.84
