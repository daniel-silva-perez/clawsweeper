## Summary

Keep open. Current main still has the zombie-loop-prone `abortable(activeSession.prompt(...))` call shape, and this PR is a relevant active fix candidate, but the proposed patch still leaves an abort window while `AgentSession.prompt()` is in preflight before a Pi `Agent.activeRun` exists.

## What This Changes

The PR adds an embedded-runner pre-prompt abort guard, abort/queue cleanup in abort handling and cleanup, and a new abort-before-prompt regression test with small test-harness support.

## Best Possible Solution

Make the prompt-start boundary abort-aware so no `AgentSession.prompt()` promise can advance into a new Pi `Agent` run after the OpenClaw run signal aborts. The maintainable end state is a preflight-aware cancellation path or lazy prompt helper with regression coverage for already-aborted, abort-during-preflight-before-activeRun, and abort-after-activeRun cases.

## Reproduction Assessment

Yes. The linked issue gives a deterministic pre-aborted-signal reproduction, and the remaining PR gap can be reproduced by aborting after this PR's guard while `AgentSession.prompt()` is awaiting preflight before it reaches `Agent.prompt()`.

## Solution Assessment

No. The PR is a useful partial fix for the already-aborted path, but it is not the best complete solution because `agent.abort()` cannot cancel a prompt that is still in `AgentSession.prompt()` preflight before `activeRun` exists.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Make prompt start abort-aware:** `src/agents/pi-embedded-runner/run/attempt.ts:2724-2726`
  - body: `AgentSession.prompt()` can still await extension/input/auth/compaction/before-start preflight after this guard and before it calls `Agent.prompt()`. If the run aborts in that interval, the new `agent.abort()` calls are no-ops because Pi has not created `activeRun` yet, and the already-created prompt promise can later start an uncancelled Agent loop. Please make the actual prompt-start boundary observe the run abort signal, or pass cancellation into the session preflight, and cover that race.
  - confidence: 0.84
