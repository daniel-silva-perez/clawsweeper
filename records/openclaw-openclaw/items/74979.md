## Summary

Keep open. The linked bug is valid and current main still has the prompt-before-abortable call-order hazard, but this active contributor PR leaves a real mid-preflight abort window because its cleanup can call `Agent.abort()` before the Pi `Agent` has created an active run to abort.

## What This Changes

The PR adds a pre-prompt abort guard and finally-block Pi Agent cleanup in the embedded runner, plus a focused abort-before-prompt regression test and small test-harness support methods.

## Best Possible Solution

Make the prompt-start boundary abort-aware so no `AgentSession.prompt()` promise can continue into `Agent.prompt()` after the run signal is aborted. A narrow fix can recheck immediately before each actual prompt start or use a lazy abortable helper/session-level cancellation path, with regression coverage for the abort-after-guard preflight window.

## Reproduction Assessment

Yes. The linked issue provides a deterministic pre-aborted `AbortSignal` reproduction, and the remaining PR-specific bug can be reproduced by aborting after the new guard but before `AgentSession.prompt()` reaches `Agent.prompt()` and creates `Agent.activeRun`.

## Solution Assessment

No. The pre-prompt guard is the right minimal direction for the already-aborted case, but the cleanup backstop is not sufficient because `Agent.abort()` is a no-op until the Pi `Agent` has an active run controller.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Close the prompt-start abort window:** `src/agents/pi-embedded-runner/run/attempt.ts:3115-3116`
  - body: `AgentSession.prompt()` can await extension/input/compaction preflight after this PR's guard but before it calls `Agent.prompt()` and creates `activeRun`. If the run aborts in that interval, `abortable(...)` rejects and this finally block can call `Agent.abort()` while it is still a no-op; the still-running prompt promise can then start an uncancelled Agent run. Please recheck/cancel at the actual prompt-start boundary or make prompt creation lazy/abort-aware, and cover that race.
  - confidence: 0.84
