## Summary

Keep this PR open for maintainer review. Current main still lacks the proposed lazy-prune/map-size changes, but the PR should not be treated as obsolete or safe to land automatically because review comments identify unresolved saturated-cache performance and invalid-limit behavior concerns on shared plugin-facing helpers.

## What This Changes

The PR changes `src/infra/dedupe.ts` to prune missed keys only when the cache exceeds `maxSize`, and rewrites `src/infra/map-size.ts` to delete oldest entries with one forward pass.

## Best Possible Solution

Keep this PR open for maintainer review, but do not land it automatically. The best path is a shared dedupe/map-size fix that removes the per-message full scan even under saturated caches, preserves or intentionally validates invalid numeric limit behavior, adds focused helper and Discord replay-path regressions, and then runs the changed shared-helper/plugin validation before merge.
