## Summary

Keep open. Current main still only makes fenced and indented code blocks selectable in the Android chat renderer, while normal paragraphs, headings, lists, and tables render as plain Text without a message-level copy/select/share/reply affordance. There is also an open PR with closing syntax for this issue, so the best maintainer path is to review and land or replace that implementation before closing the issue.

## Best Possible Solution

Review PR #59603 or a successor implementation against current main. The likely product path is to add a message-level Android chat action surface in apps/android/app/src/main/java/ai/openclaw/app/ui/chat, wire whole-message copy, selectable-copy dialog, Android share intent, and composer quote/reply behavior, then close this issue after the implementation merges.
