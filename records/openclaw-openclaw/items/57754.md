## Summary

Keep open. Current main still only makes Android fenced and indented code blocks selectable; normal chat message text renders without a message-level long-press action surface for copy, selective copy, share, or reply. PR #59603 is an open, unmerged implementation candidate with closing syntax for this issue, so the issue should remain open until that PR or a successor lands.

## Best Possible Solution

Review PR #59603 or a successor implementation against current main. The likely fix belongs in apps/android/app/src/main/java/ai/openclaw/app/ui/chat: add a message-level Android chat action surface, wire whole-message copy, selectable-copy dialog, Android share intent, and composer quote/reply behavior, then close the issue after the implementation merges and is manually validated on Android.
