## Summary

Keep this issue open. Current main still applies `sanitizeTransportPayloadText(block.thinking)` when converting signed Anthropic thinking blocks into the Anthropic Messages payload, and the sanitizer still strips unpaired UTF-16 surrogates. Existing replay policy and docs preserve signatures earlier in history sanitization, but the final transport conversion can still mutate the signed text before send.

## Best Possible Solution

Patch the Anthropic Messages transport and the Anthropic Vertex/pi-ai path as needed so signed `thinking` and `redacted_thinking` blocks are replayed unchanged, while unsigned fallback text continues through `sanitizeTransportPayloadText`. Add a focused regression test that builds an Anthropic payload from a signed thinking block containing an unpaired surrogate and asserts the emitted `thinking` value is byte-for-byte the original with the signature preserved.
