## Summary

Keep open. Current main still routes inline skill `command-dispatch: tool` through raw OpenClaw tools plus owner-only filtering, so the PR addresses a real remaining policy gap. The proposed branch is not ready to merge because it does not yet mirror the full command-session context used by normal coding-tool policy resolution and it reintroduces an eager tool-stack import on the inline-actions path.

## What This Changes

The PR changes inline skill `command-dispatch: tool` handling to select tools via `createOpenClawCodingTools` instead of raw OpenClaw tools plus owner-only filtering, and adds regression coverage for non-owner messaging-profile dispatch.

## Best Possible Solution

Inline skill `command-dispatch: tool` should resolve callable tools through the same policy-resolving factory used by normal command sessions, behind a lazy runtime wrapper, while forwarding target-session, route, sender, model, group, and plugin-binding context and keeping focused regression tests for profile-disallowed exec-style tools.
