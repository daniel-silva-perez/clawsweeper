## Summary

Keep this PR open. Current main still dispatches inline skill `command-dispatch: tool` through raw OpenClaw tools plus owner-only filtering, while the normal command path uses `createOpenClawCodingTools` and its broader profile/provider/group/sandbox/subagent policy pipeline. The PR remains a focused security-policy fix, with review comments already addressed in the PR discussion, but it should be rebased/adapted to preserve the current lazy runtime boundary.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase or adapt it on current main so inline skill `command-dispatch: tool` resolves through `createOpenClawCodingTools`, removes the standalone owner-only fallback, preserves the lazy runtime boundary, forwards route/session/group/sender/model/config context, and keeps focused regression tests for profile-disallowed tools plus policy-critical argument forwarding.
