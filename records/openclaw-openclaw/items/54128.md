## Summary

Keep this issue open. Current main still does not expose or forward a `memorySearch.local.maxThreads` setting for the built-in local node-llama-cpp embedding path. The current local tuning surface is `contextSize`; the runtime calls `getLlama` with only `logLevel`, the strict config schema/generated docs omit `maxThreads`, and upstream node-llama-cpp does have a separate `getLlama` `maxThreads` option, so this remains a valid unimplemented feature request.

## Best Possible Solution

Add an optional validated positive-integer `agents.defaults.memorySearch.local.maxThreads` setting, merge it through default/per-agent memory config resolution, carry it through the memory embedding provider option types and local adapter path, update the node-llama-cpp shim to accept it, and pass it to `getLlama` only when configured. Add focused tests for schema acceptance/rejection, default and override merging, runtime forwarding/omission, plus generated config help/schema and public memory docs updates.
