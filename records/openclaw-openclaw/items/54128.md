## Summary

Keep this open. Current main still exposes only `modelPath`, `modelCacheDir`, and `contextSize` for local memory embeddings; there is no `memorySearch.local.maxThreads` config carrier and the local provider still calls `getLlama` without a thread limit.

## What This Changes

Request to add an optional `memorySearch.local.maxThreads` setting for local `node-llama-cpp` embeddings and pass it to `getLlama`.

## Best Possible Solution

Add a narrow, validated `memorySearch.local.maxThreads` field for the built-in local embedding provider, preserve it through default/per-agent config resolution and provider option types, pass it to `getLlama` only when configured, and update focused tests plus generated/public config docs.
