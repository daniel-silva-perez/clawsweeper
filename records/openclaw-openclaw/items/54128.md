## Summary

Current main still does not implement the requested local embedding thread limit. The config schema, resolved memory config, provider create options, node-llama shim, and runtime call all omit `maxThreads`, while the upstream node-llama-cpp API supports it at the `getLlama` options boundary.

## What This Changes

The issue requests an optional `agents.defaults.memorySearch.local.maxThreads` setting that is validated, resolved, and passed to node-llama-cpp for local embedding CPU thread control.

## Best Possible Solution

Add a narrow, validated `memorySearch.local.maxThreads` setting for the built-in local embedding provider, carry it through default/per-agent resolution and provider option types, pass it to `getLlama` only when configured, and update tests plus public/generated config documentation.
