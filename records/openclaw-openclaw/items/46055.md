## Summary

Keep open. Current main still lacks a queued waiting status and the PR contains useful implementation work, but it is not ready to merge because the new waiting emoji is added to the status reaction API without updating the strict config/schema/help/generated-doc contract that users rely on for `messages.statusReactions.emojis`.

## What This Changes

The PR adds a Telegram waiting reaction state, threads a per-message `onQueued` callback through queue, embedded-runner, auto-reply, and Telegram dispatch paths, adds status-reaction tests, and adds Telegram reaction 429 backoff.

## Best Possible Solution

Land a rebased version that keeps the callback-based per-message design, fires queued notification from both session and global lanes with a once guard, preserves Telegram waiting-to-thinking transitions and per-account+chat 429 backoff, and updates the full config type, zod schema, generated schema/help, and user docs for the new waiting status key.
