## Summary

Keep this issue open. Current main has a cleanup/TTL mitigation for completed subagent records, but the hot read path still synchronously reloads and parses `subagents/runs.json` for non-test `getSubagentRunsSnapshotForRead()` calls. The related PR #57863 proposed an mtime cache and had closing syntax for this issue, but it was closed unmerged in the provided GitHub context, so the central O(n) read concern is not implemented on main.

## Best Possible Solution

Keep this issue open and implement the missing hot-path fix in `src/agents/subagent-registry-state.ts` or `src/agents/subagent-registry.store.ts`: cache the parsed registry by mtime or maintain a process-local snapshot invalidated by `saveSubagentRegistryToDisk()`, while preserving cross-process freshness. Add a focused test proving repeated subagent read snapshots or `sessions.list` calls do not repeatedly parse an unchanged `runs.json`.
