## Summary

Current main still does not support externally managed CDP/noVNC endpoints for the sandbox browser. The sandbox browser config and runtime path remain Docker/container-shaped, non-Docker sandbox backends are documented as not supporting browser sandboxes, and the existing remote CDP profile support is for the normal browser service rather than `agents.defaults.sandbox.browser`.

## Best Possible Solution

Keep this open and implement a core browser-sandbox seam: either the proposed minimal `sandbox.browser.externalCdpEndpoint`/`externalNoVncEndpoint` path that skips Docker lifecycle, or a fuller browser backend API tied to `SandboxBackendHandle`. The change should update config schema/types/docs, add tests for non-Docker backend behavior, and preserve current Docker sandbox browser defaults.

## Risks / Open Questions

- Closing as implemented would conflate global remote CDP profiles with sandbox browser support; they solve different runtime and noVNC/lifecycle paths.
- Any implementation should get a security review for CDP/noVNC exposure, SSRF policy, endpoint reachability checks, and auth/token handling across sandbox boundaries.
