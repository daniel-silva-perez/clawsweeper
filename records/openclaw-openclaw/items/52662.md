## Summary

Current main still does not implement the requested externally managed CDP/noVNC path for sandbox browsers. The sandbox browser runtime remains Docker-lifecycle based, the config/schema/docs expose only Docker-shaped sandbox browser fields, and the existing remote CDP support belongs to the global browser profile service rather than `agents.defaults.sandbox.browser`.

## Best Possible Solution

Keep this open and implement a core browser-sandbox seam. A conservative first version would add explicit validated `agents.defaults.sandbox.browser` external CDP/noVNC endpoint settings that skip Docker lifecycle only when configured, update config types/schema/help/docs, and add regression tests proving non-Docker sandbox backends can use the browser without requiring Docker. A fuller solution could make browser lifecycle a backend capability, but the important behavior is external lifecycle support while preserving current Docker defaults.
