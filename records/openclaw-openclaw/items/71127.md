## Summary

Keep open. Current main has conservative stale-lane recovery, but it still does not implement the requested configurable active-run abort path, and the issue is paired with open closing PR #73243 for maintainer review.

## What This Changes

The issue requests gateway self-recovery for sessions that remain in `processing` after diagnostics detects them as stuck, including a configurable abort threshold and queue release.

## Best Possible Solution

Review and either land or revise #73243 so diagnostics keeps the current warning threshold, adds a separate conservative abort threshold, uses existing reply/embedded-run cleanup primitives, emits structured recovery diagnostics, and preserves long-running healthy work until the abort threshold is crossed.

## Reproduction Assessment

Yes. The report has production logs plus a deterministic v2026.4.25 Telegram restart-window reproduction in the discussion, and current-main source confirms active stuck work is still not aborted by diagnostics.

## Solution Assessment

No for current main as-is. The existing conservative stale-lane recovery is useful, but the maintainable fix for this report is the narrower linked PR path with explicit threshold/config review rather than closing the issue now.
