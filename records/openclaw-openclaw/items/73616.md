## Summary

Keep this PR open. Current main still shows the QQBot slash-command auth bypass, pre-dispatch registration gap, clear-storage path mismatch, and slash-command file-send option gap that the PR targets. The PR is a plausible focused implementation candidate, but it should not be closed or treated as ready because the provided review comments identify a likely `/bot-version` group-chat regression and a `dm` private-chat semantics question.

## Best Possible Solution

Keep the PR open as an implementation candidate. The best path is to update the PR to remove the unintended `bot-version` private-chat restriction unless maintainers explicitly want that behavior, decide and test `dm` semantics for private-only commands, add focused unit coverage for the new auth and gating behavior plus the storage/file-delivery seams, then review and land the corrected QQBot fix.

## Risks / Open Questions

- The PR changes authorization for sensitive QQBot admin commands (`/bot-logs`, `/bot-clear-storage`, `/bot-approve`), so a regression could either preserve the bypass or lock out valid operators without clear coverage.

- The provided review comment flags `dm` handling for the new `c2cOnly` guard; current `SlashCommandContext.type` includes `dm`, so maintainers should decide whether guild DMs count as private chat and test that path.
- The PR body says no new tests were added yet; the auth resolver, pre-dispatch matching, c2c/private gating, clear-storage directory, and file-send option are all small enough to cover with focused tests.
