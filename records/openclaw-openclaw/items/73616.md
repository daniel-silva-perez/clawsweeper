## Summary

Keep this PR open. Current main still has the QQBot slash-command auth, pre-dispatch registration, clear-storage path, and log-file delivery gaps this PR targets, so the intended change is not obsolete or implemented on main. The PR is a plausible fix candidate, but its current head still needs maintainer follow-up for the reviewed /bot-version group regression, dm/private-chat semantics, and focused regression coverage.

## Best Possible Solution

Keep this PR open as the broader QQBot fix candidate. Update it to remove the unintended /bot-version private-chat restriction unless maintainers explicitly want it, decide whether dm counts as private chat for c2cOnly, add focused tests for auth resolution, registry matching, c2c/group/dm gating, clear-storage directory resolution, and file-delivery options, then review it as the canonical fix and coordinate or supersede #71158 after the broader path is ready.
