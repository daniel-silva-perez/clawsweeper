## Summary

Keep this PR open. Current main still does not implement code-file memory indexing or declaration-boundary code chunking: memory discovery accepts Markdown/multimodal files only, indexing still routes text through fixed-window `chunkMarkdown()`, and the public docs still describe Markdown-only builtin memory roots. The PR remains potentially useful, but it now needs a rebase/adaptation from the old `src/memory/*` layout to the current memory-host SDK plus `extensions/memory-core` implementation and maintainer coordination with the overlapping open #68019 chunking-strategy PR.

## Best Possible Solution

Keep this PR open and ask for a current-main adaptation. The best landing path is a focused implementation in `packages/memory-host-sdk/src/host/internal.ts` plus `extensions/memory-core/src/memory/manager-embedding-ops.ts` and `manager-sync-ops.ts`, with code-file discovery, declaration-aware chunking, watcher/ignore parity including case behavior, reindex metadata if chunking output changes, docs updates, and focused tests. Coordinate scope with #68019 before merge.
