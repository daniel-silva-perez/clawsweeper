## Summary

Keep this PR open. Current main still does not implement code-file memory indexing or declaration-aware chunking, and the branch now targets an old memory layout, so it needs maintainer review and a current-main adaptation rather than cleanup closure.

## What This Changes

The PR branch adds source-code file discovery and regex-based declaration-boundary chunking for memory indexing, with code-specific tests and watcher updates under the old `src/memory` layout.

## Best Possible Solution

Land a current-main implementation that adds code-file discovery and declaration-aware chunking in the memory host SDK and memory-core plugin, with watcher/filter parity, reindex metadata for chunk-output changes, user-facing docs for the expanded indexed data scope, and coordination with the broader chunking strategy design in #68019.
