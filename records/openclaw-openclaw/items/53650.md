## Summary

Keep this PR open for maintainer review. Current main still has Markdown/multimodal-only memory file discovery and fixed `chunkMarkdown()` indexing, so the intended code-file chunking feature is not implemented; however, the branch targets the old `src/memory` layout and overlaps the still-open chunking-strategy work in #68019, so it needs a current-main design decision rather than conservative cleanup.

## What This Changes

The PR adds code-file detection, declaration-boundary chunking, indexing dispatch, watcher updates for code extensions, and code-chunking tests under the old `src/memory` layout.

## Best Possible Solution

Land a current-main implementation that treats source-code memory files as an explicit supported input type in the memory host SDK and memory-core plugin, with discovery, watcher/filter parity, chunk-output reindex metadata, focused tests, and user-facing docs aligned with the broader chunking strategy direction in #68019.
