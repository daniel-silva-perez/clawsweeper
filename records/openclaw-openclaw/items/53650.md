## Summary

Keep this PR open. Current main still does not implement code-file memory indexing or declaration-boundary chunking: memory discovery accepts Markdown or enabled multimodal files, indexing still routes text entries through fixed-window `chunkMarkdown()`, and docs still describe Markdown-only builtin memory roots. The branch remains useful but targets the obsolete `src/memory/*` layout and needs maintainer coordination with the open #68019 chunking-strategy PR before it can be adapted safely.

## What This Changes

The PR adds source-code file discovery and declaration-boundary chunking for memory indexing, with tests and watcher updates for code files under memory roots.

## Best Possible Solution

Keep this PR open and ask for a current-main adaptation. The best landing path is a focused implementation in `packages/memory-host-sdk/src/host/internal.ts` plus `extensions/memory-core/src/memory/manager-embedding-ops.ts`, `manager-sync-ops.ts`, and reindex/config/docs surfaces as needed: add code-file discovery and declaration-aware chunking, preserve watcher/ignore parity including case behavior, add reindex metadata if chunk output changes, document the data-scope change, and coordinate chunking strategy shape with #68019 before merge.
