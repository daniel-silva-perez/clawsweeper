Keep this PR open. Current main already has the runtime-deps lock-owner starttime implementation, but this PR adds narrow writer-level regression coverage that is not present on main, and I found no blocking correctness issue in the proposed test patch.

## What This Changes

Adds a bundled runtime-deps test that acquires the filesystem lock, reads `owner.json`, and asserts the lock owner persists the current process raw starttime and is not reclaimed as stale.

## Best Possible Solution

Keep the existing raw-starttime lock-owner implementation and land this writer-level regression test, or an equivalent test, so future runtime-deps lock refactors cannot drop persisted starttime coverage while the mocked predicate tests remain in place.
