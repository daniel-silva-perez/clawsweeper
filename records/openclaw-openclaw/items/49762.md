## Summary

Keep this PR open. Current main still does not add response duration to the per-reply usage footer: the footer formatter has no duration parameter, and the runner only emits duration in diagnostics, not in the user-visible footer. The PR is a plausible small feature, but it still needs rework for the existing review comments before merge.

## Best Possible Solution

Keep this PR open for author or maintainer rework: add duration to the intended usage-footer mode, preserve existing formatting when duration is absent, decide whether `/usage tokens` should remain token-only, choose a timing point or label that matches the measured span, and add targeted formatter/runner tests plus docs if the footer behavior changes.

## Risks / Open Questions

- The submitted formatter refactor changes the cost separator from ` · est` to ` / est` even when `durationMs` is omitted, contradicting the backward-compatibility claim in the PR description.
- Passing `durationMs` unconditionally would also change `/usage tokens`, even though current docs describe that mode as token-only.
- The proposed sampling point is after block-reply flushing and pending tool delivery drains, so the label should either be sampled earlier or clearly described as end-to-end wall time.
- The PR lacks focused tests for formatter output, mode gating, cost separator preservation, and duration display.
