## Summary

Keep this PR open. Current main still does not add response duration to the per-reply usage footer, and the submitted change remains a plausible implementation candidate. It needs rework before merge because the existing review feedback identifies real behavior questions around footer formatting, `/usage tokens`, and the timing span.

## Best Possible Solution

Keep this PR open for author or maintainer rework. The best path is to port the change onto the current `agent-runner-usage-line.ts` split, preserve existing formatting when duration is absent, gate the stopwatch to the intended usage mode, define the timing span clearly, and add focused formatter/runner tests. The related open #57442 can help guide whether response time belongs only in the footer, in session status, or both.

## Risks / Open Questions

- Merging the submitted formatter refactor as-is could regress the existing cost separator from ` · est` to ` / est` when no duration is provided.
