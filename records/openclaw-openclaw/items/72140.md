## Summary

Keep open. Current main suppresses stale pre-compaction totals, but it still does not use the latest compaction checkpoint `tokensAfter` value for the shared status formatter; open PR #72285 remains the targeted fix path.

## What This Changes

The issue asks `/status` and shared post-message status cards to report the current post-compaction context size instead of stale pre-compaction totals.

## Best Possible Solution

Review and land PR #72285, or an equivalent narrow shared formatter change that prefers fresh totals and falls back to the latest valid checkpoint `tokensAfter` only when displayed totals are stale or missing, with regression coverage for post-compaction status display.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildStatusMessage` with stale `totalTokens`, `totalTokensFresh: false`, and a latest compaction checkpoint containing `tokensAfter`; current main ignores that checkpoint and renders unknown context.

## Solution Assessment

Yes. The narrow maintainable solution is in the shared status formatter using the existing freshness marker and checkpoint data, not a WhatsApp-specific workaround or a broader session migration.
