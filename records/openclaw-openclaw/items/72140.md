## Summary

Keep open. Current main no longer displays the stale pre-compaction total when `totalTokensFresh` is false, but it still does not display the latest post-compaction `tokensAfter` value; the linked open PR #72285 remains the targeted fix path.

## What This Changes

The issue asks for `/status` and shared status cards to show the current post-compaction context size instead of stale pre-compaction totals.

## Best Possible Solution

Review and land PR #72285, or an equivalent narrow shared formatter change that keeps fresh totals preferred and falls back to the latest valid compaction checkpoint `tokensAfter` only when displayed totals are stale or missing, with regression coverage for post-compaction status display.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildStatusMessage` with stale `totalTokens`, `totalTokensFresh: false`, and a latest compaction checkpoint containing `tokensAfter`; current main ignores that checkpoint and renders unknown context instead of the post-compaction size.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix because the bug is in the shared status formatter, not in a channel-specific WhatsApp surface or a broader session migration.
