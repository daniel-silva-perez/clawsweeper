## Summary

Keep open. Current main suppresses the stale pre-compaction `totalTokens` value, but the shared status formatter still falls back to unknown context instead of using the latest compaction checkpoint `tokensAfter`; the related open PR #72285 remains the targeted fix path.

## What This Changes

The issue requests that `/status` and shared post-message status cards show current post-compaction context size rather than stale pre-compaction totals.

## Best Possible Solution

Review and land PR #72285, or an equivalent narrow shared formatter change that prefers fresh totals, falls back to the latest valid checkpoint `tokensAfter` only when totals are stale or missing, and keeps regression coverage for post-compaction status display.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildStatusMessage` with stale `totalTokens`, `totalTokensFresh: false`, and a latest compaction checkpoint containing `tokensAfter`; current main ignores that checkpoint and renders unknown context.

## Solution Assessment

Yes. The narrow maintainable solution is a shared status formatter fallback using the existing freshness marker and checkpoint data, not a WhatsApp-specific workaround or a broader session migration.
