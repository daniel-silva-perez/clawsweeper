## Summary

Keep open. Current main still has the risky status-formatting path: the status card formats `Context:` from `sessionEntry.totalTokens` or `inputTokens + outputTokens` without checking `totalTokensFresh` or falling back to the latest compaction checkpoint `tokensAfter`. There is already an open implementation PR, #72285, that appears to target exactly this issue, so the best cleanup path is to review and land or close that PR rather than closing this issue now.

## Best Possible Solution

Keep this issue open and review #72285. The narrow fix should make shared status formatting prefer fresh context totals and fall back to the latest compaction checkpoint `tokensAfter` when `totalTokensFresh` is false, with a regression test in the status formatter tests.
