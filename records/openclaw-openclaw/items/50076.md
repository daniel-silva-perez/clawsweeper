## Summary

Keep this PR open. Current main still lacks write-time validation for `gateway.nodes.denyCommands` in `openclaw config set` and `config unset`, and the linked bug #50011 is still open. The PR is unmerged and currently dirty, but it remains a focused implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open and review it as an implementation candidate for #50011. The best path is to rebase it, resolve exact-command semantics and catalog sharing, then land this or an equivalent smaller patch that validates relevant `gateway.nodes.allowCommands`/`denyCommands` set and unset writes, preserves structured dry-run JSON, and shares a runtime-accurate command vocabulary with the security audit.
