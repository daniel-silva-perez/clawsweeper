## Summary

Keep open. Current main still does not implement this PR's write-time `gateway.nodes.denyCommands` validation for `openclaw config set`/`unset`; it only schema-validates the field as `string[]` and relies on security audit warnings after the config is already written. The linked bug #50011 remains real on current main, and this PR is still a plausible implementation candidate, but it needs maintainer review around exact runtime command matching before landing.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for #50011. The best path is to land this PR after resolving the remaining exact-command semantics questions, or replace it with a smaller equivalent patch that validates `gateway.nodes.allowCommands`/`denyCommands` writes and unsets, preserves structured dry-run JSON, and shares a runtime-accurate known-command catalog with the security audit path.
