## Summary

Keep this PR open. Current main still accepts `gateway.nodes.denyCommands` writes through `openclaw config set`/`unset` without PR-style command-name validation; the only current checks are schema shape validation and security-audit findings after the config has already been accepted. The linked bug #50011 is still open, and this PR remains a plausible implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open and review it as an implementation candidate for #50011. The best path is to land this PR after resolving exact-command semantics and known-command catalog questions, or replace it with a smaller equivalent patch that validates `gateway.nodes.allowCommands`/`denyCommands` writes and unsets, preserves structured dry-run JSON, and shares a runtime-accurate command catalog with the audit path.
