## Summary

Keep this PR open. Current main still lacks write-time validation for `gateway.nodes.denyCommands` in `openclaw config set`/`unset`, while the linked bug #50011 remains open and this PR is still a plausible implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open and review it as an implementation candidate for #50011. The best path is to land this PR after resolving exact-command semantics and catalog sharing, or replace it with a smaller equivalent patch that validates relevant `gateway.nodes.allowCommands`/`denyCommands` set and unset writes, preserves structured dry-run JSON, and reuses a runtime-accurate known-command catalog with the security audit path.

## Risks / Open Questions

- The PR touches a security-sensitive config path. Maintainers should review exact command ID semantics carefully, especially custom node commands containing punctuation such as `*`, `|`, parentheses, `group:`, or other pattern-looking text that the runtime may still treat as exact strings.
- The audit helper and the proposed validator should share a runtime-accurate command catalog; related open issue #65745 shows the current audit catalog can mislabel valid dangerous commands when it is derived from effective allowlists.
