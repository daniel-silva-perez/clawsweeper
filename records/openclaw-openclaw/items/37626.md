## Summary

Current main still does not paginate the Feishu `feishu_wiki` `nodes` action. The implementation calls `client.wiki.spaceNode.list` once, the schema exposes only `space_id` and optional `parent_node_token`, the latest release tag has the same behavior, and the related fix PR #37641 is closed unmerged. This is a narrow valid plugin bug and a good fix-PR candidate.

## Best Possible Solution

Keep this issue open and fix it in the bundled Feishu plugin. Update `extensions/feishu/src/wiki.ts` so `listNodes` requests a documented bounded `page_size`, follows `has_more` and `page_token`, accumulates all node items, and returns the existing `nodes` array plus lightweight metadata such as `total` and `pages_fetched`. Add a focused `extensions/feishu/src/wiki.test.ts` regression test that mocks multiple `spaceNode.list` pages, including an empty intermediate page with `has_more=true`. Manual pagination input fields are a separate product choice; automatic full pagination addresses the reported data truncation without changing the normal tool call.
