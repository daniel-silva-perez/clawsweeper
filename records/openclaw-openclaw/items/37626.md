## Summary

Current main still does not paginate the bundled Feishu `feishu_wiki` `nodes` action. `listNodes` makes one `client.wiki.spaceNode.list` call with only `parent_node_token`, the `nodes` dispatcher uses that helper directly, the schema and Feishu wiki skill docs expose no pagination or fetch-all behavior, and release `v2026.4.26` has the same single-page implementation. The Feishu SDK/API contract supports `page_size`, `page_token`, `has_more`, and `listWithIterator`, and the related fix attempts #37641 and #37681 are both closed unmerged, so this remains a valid narrow plugin bug and a good fix-PR candidate.

## Best Possible Solution

Keep this issue open and land a focused Feishu plugin fix. Update `extensions/feishu/src/wiki.ts` so the `nodes` action requests a bounded page size, follows `has_more` and `page_token`, accumulates all node items, handles empty pages that still have `has_more=true`, caps pages, and returns the existing `nodes` array plus lightweight metadata such as `total`, `pages_fetched`, and possibly a truncation indicator. Add focused Feishu wiki tests and update `extensions/feishu/skills/feishu-wiki/SKILL.md` if result metadata or optional pagination inputs become part of the tool contract.
