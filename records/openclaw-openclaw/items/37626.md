## Summary

Current main still does not paginate the bundled Feishu `feishu_wiki` `nodes` action. `listNodes` calls `client.wiki.spaceNode.list` once with only `parent_node_token`, the `nodes` dispatcher calls that helper directly, and neither the schema nor the skill docs expose or describe pagination. The locked Feishu SDK version supports `page_size`, `page_token`, `has_more`, and an iterator for this endpoint, and the latest release tag still has the same single-page implementation. The related fix PR #37641 is closed unmerged, so this remains a valid narrow plugin bug and a good fix-PR candidate.

## Best Possible Solution

Keep this issue open and fix it in the bundled Feishu plugin. Update `extensions/feishu/src/wiki.ts` so the `nodes` action requests a documented bounded page size, follows `has_more` and `page_token`, accumulates all node items, and returns the existing `nodes` array plus lightweight metadata such as `total` and `pages_fetched`. Add a focused `extensions/feishu/src/wiki.test.ts` regression test for multiple pages, including an empty intermediate page with `has_more=true`; manual pagination inputs can remain a separate product choice.
