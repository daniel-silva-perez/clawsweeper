## Summary

Keep this PR open. Current main still does not include a Helm chart or `charts/openclaw/**`; the shipped Kubernetes path is Kustomize/raw manifests. The PR is a plausible deployment contribution, but it creates a new supported packaging surface and has unresolved chart correctness and security-adjacent deployment review comments, so conservative cleanup should not close it.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is either to revise the chart so it preserves the raw-manifest object/data contract, follows current image/version policy, adds Helm render-time guards and checksum rollout annotations, documents loopback exposure clearly, and adds chart validation, or to have a maintainer explicitly decide that Helm packaging should live outside the core repository despite the Kubernetes docs saying a chart can be layered on top of the manifests.
