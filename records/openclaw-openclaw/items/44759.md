## Summary

Current main still auto-promotes browser screenshot file paths into outbound channel media. The implementation has no screenshot-level private/suppress-delivery option, `imageResult` still records `details.media.mediaUrl`, `browser` is still a trusted local media tool, and the tool-result pipeline still queues that media for the next visible reply or media-only lifecycle flush. The Pinchy comment is useful production corroboration, and the nearby broader tool-error visibility issues do not cover this screenshot media leak.

## Best Possible Solution

Keep this issue open and implement a maintainer-reviewed tool-result media visibility policy. The narrow fix is a browser screenshot option such as private: true that keeps the image content block available to the model while preventing details.media.mediaUrl from entering pending channel delivery. A broader per-tool default can layer on the same pipeline, with tests proving suppressed screenshots remain model-visible and intentional generated media still auto-delivers.
