## Summary

Keep this PR open. Current main still does not implement the intended sandbox-specific `media://inbound` fix: sandboxed image/media paths are bridge-resolved before any media-store URI handling, so the malformed `media:/inbound` path problem remains. The PR is a focused implementation path for #74123, but it touches sandbox path resolution and should get maintainer/security review before merge.

## Best Possible Solution

Keep this PR open as the focused implementation path for #74123. The best fix is a narrow early branch in `resolveSandboxedBridgeMediaPath` that validates `media://inbound/<id>` before bridge resolution, maps it to `media/inbound/<id>` only when the staged sandbox file exists, enforces the existing workspace boundary on resolved host paths, handles malformed URI parsing without an uncaught exception, and adds targeted sandbox-helper plus image-tool coverage without changing broader media or sandbox contracts.
