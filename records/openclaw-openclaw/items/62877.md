## Summary

Current main still emits and tests the raw `Usage: /btw <side question>` reply, and the shared outbound sanitizer still strips tag-like angle-bracket text while preserving autolinks. The issue remains a concrete producer-side markdown/channel rendering bug.

## Best Possible Solution

Keep this issue open. Update `src/auto-reply/reply/commands-btw.ts` so the missing-question usage renders safely, preferably inline code like `Usage: `/btw <side question>``, update `src/auto-reply/reply/commands-btw.test.ts`, and audit nearby auto-reply usage strings with raw angle-bracket placeholders for the same rendering bug.
