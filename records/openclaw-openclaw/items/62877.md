## Summary

Current main still emits the raw `Usage: /btw <side question>` text for a missing side question, and the colocated test still locks in that raw placeholder. The issue remains a concrete, recent producer-side bug because outbound sanitizers strip unknown HTML-like tags while preserving angle-bracket autolinks, so a broad channel escape would be the wrong fix point.

## Best Possible Solution

Keep this issue open. Update `src/auto-reply/reply/commands-btw.ts` so the missing-question usage renders safely, preferably as inline code such as `Usage: `/btw <side question>``, update `src/auto-reply/reply/commands-btw.test.ts`, and audit nearby auto-reply usage strings with raw angle-bracket placeholders for the same markdown/HTML rendering problem.
