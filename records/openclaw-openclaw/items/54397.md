## Summary

Keep this issue open. Current main has meaningful partial session management, including Gateway create/list/search/patch/delete RPCs, a Control UI Sessions table with label editing and bulk delete, a chat session selector, TUI session switching, and CLI session listing/cleanup. The requested first-class Topic/Session Management UI and lifecycle surface is still not implemented end to end: no Control UI topic sidebar/create workflow, no CLI create/rename/show/export/delete commands, no description/tags/message-count session contract, and no deleted-session archive restore or import/export API. The latest discussion also treats this as active work rather than resolved, with a focused CLI implementation offer and adjacent open items #57210 and #58028 called out as related consumers.

## Best Possible Solution

Keep this issue open as the umbrella tracker, but narrow implementation into focused PRs: first-class Control UI topic/session creation and switching around `sessions.create`, `sessions.patch`, and `sessions.delete`; a maintainer decision on whether description/tags/createdAt/messageCount belong in `GatewaySessionRow` or a separate topic API; CLI lifecycle commands such as list/show/export/delete if accepted; and a separate design for deleted-session archive restore plus import/export.
