## Summary

Keep this issue open. Current main has meaningful partial session management: Gateway session list/search/create/patch/delete RPCs, a Control UI Sessions table with label editing and bulk delete, chat/TUI session selectors, and CLI session listing/cleanup. The requested first-class Topic/Session Management UI is still not implemented end to end: no Control UI topic sidebar/create workflow, no CLI create/rename/delete commands, no description/tags/createdAt/message-count session-row contract, and no deleted-topic archive restore or session import/export API. The prior ClawSweeper review comment on this issue kept it open for the same reason, and current main still matches that assessment.

## Best Possible Solution

Keep this issue open and narrow it into concrete work: add a first-class Control UI topic/session workflow around sessions.create, sessions.patch, and sessions.delete; decide whether description, tags, createdAt, and messageCount belong in GatewaySessionRow or a separate usage/topic API; add CLI/TUI create/rename/delete/archive surfaces only if maintainers want those entry points; and design deleted-session archive restore plus import/export with protocol docs and focused tests.
