## Summary

Keep this issue open. Current main has meaningful partial session support: Gateway session create/list/patch/delete RPCs, label editing and bulk delete in the Control UI Sessions table, a chat session selector, TUI session picker, and CLI session listing/cleanup. The requested first-class Topic/Session Management UI is still not implemented end to end: there is no Control UI topic sidebar/create workflow, no CLI create/rename/delete commands, no description/tags/message-count session-row contract, and no deleted-topic archive restore or session import/export API. The prior ClawSweeper comment on the issue reached the same keep-open conclusion, and current main still matches that assessment.

## Best Possible Solution

Keep this issue open and narrow it into maintainer-owned work: add a first-class Control UI topic/session workflow around sessions.create, sessions.patch, and sessions.delete; decide whether description, tags, createdAt, and messageCount belong in GatewaySessionRow or a separate usage/topic API; add CLI/TUI create/rename/delete/archive only if maintainers want those surfaces; and design deleted-session archive restore plus import/export with protocol docs and tests.

## Risks / Open Questions

- Deleted-session archive restore plus import/export need explicit storage, conflict-resolution, permission, and UI semantics before implementation.
