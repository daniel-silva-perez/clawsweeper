## Summary

Keep open. Current main still has a credible Homebrew/package auto-update failure path, and open PR #74870 already references this issue with closing syntax, so the issue should stay open until that PR or a replacement lands.

## What This Changes

The issue asks the gateway auto-updater to install stable updates after the configured delay for Homebrew installs instead of only detecting them.

## Best Possible Solution

Resolve the Homebrew auto-update contract explicitly: either implement a safe Homebrew-aware apply flow outside the live gateway process, or detect Homebrew-managed installs and skip with a clear `brew upgrade openclaw` hint plus aligned docs and tests.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a package-managed gateway service with auto-update due: the auto-update child inherits gateway service markers, then `openclaw update` refuses package replacement inside that service process.

## Solution Assessment

Unclear. The linked PR is a plausible narrow fix if maintainers choose skip-and-hint for Homebrew, but it does not implement the originally expected auto-install behavior.
