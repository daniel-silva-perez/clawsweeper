## Summary

Keep this issue open. Current main still has a credible gateway auto-update failure path: the gateway schedules and attempts package auto-updates, but the child update command inherits the gateway service environment and the package updater refuses to run inside a gateway service process. The Homebrew-specific part also needs maintainer judgment because current package-update support is modeled around npm, pnpm, and bun rather than a documented `brew upgrade openclaw` path.

## What This Changes

The issue asks for the gateway auto-updater to install a detected stable update after the configured delay for a Homebrew-installed OpenClaw gateway.

## Best Possible Solution

Keep the issue open and either add a safe out-of-process package auto-apply path that can stop and restart a managed gateway without inheriting service-mode blockers, or explicitly persist/log that auto-apply is unsupported for Homebrew formula installs. The docs and tests should be aligned with whichever install surfaces are supported.
