## Repair Work Prompt

Fix the auto-update path reported in https://github.com/openclaw/openclaw/issues/74754. Current main schedules gateway package auto-update from `src/infra/update-startup.ts`, but the spawned `openclaw update` child inherits `OPENCLAW_SERVICE_MARKER`/`OPENCLAW_SERVICE_KIND` and is then rejected by the package-update guard in `src/cli/update-cli/update-command.ts`. Preserve the guard for real in-service package replacement, but make the auto-update child/trampoline use a safe environment or equivalent path so it can stop/restart the managed gateway and verify the updated version. Add regression coverage in `src/infra/update-startup.test.ts` and, if needed, `src/cli/update-cli.test.ts`. Also handle the Homebrew install surface explicitly: either implement a supported Homebrew update path with restart verification, or persist/log and document that Homebrew auto-apply is unsupported rather than silently appearing configured. Keep the change scoped to updater code/docs/tests; do not touch installer release metadata or unrelated package-manager behavior. Add a user-facing changelog fix entry crediting the reporter, and only update docs if behavior or support boundaries change.
