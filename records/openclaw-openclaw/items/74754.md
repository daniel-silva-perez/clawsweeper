## Summary

Keep this open. Current main still has a credible Homebrew/package-managed gateway auto-update failure path, and open PR #74870 already references this issue with closing syntax, so this should be resolved by reviewing or replacing that implementation rather than auto-closing or queueing duplicate repair work.

## What This Changes

The issue asks for a Homebrew-installed gateway to automatically install a detected stable update after the configured rollout delay instead of requiring manual `brew upgrade openclaw`.

## Best Possible Solution

Define the Homebrew package auto-update contract explicitly and implement it in the updater/docs: either perform a safe Homebrew-aware apply outside the live gateway service, or skip Homebrew-managed installs with a clear `brew upgrade openclaw` hint and aligned documentation.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a package-managed gateway service with auto-update due: the spawned update child inherits gateway service markers, and `openclaw update` refuses package replacement before invoking the package manager.

## Solution Assessment

Unclear. PR #74870 is a narrow plausible fix if maintainers choose manual Homebrew upgrades as the supported contract, but it changes the expected behavior from auto-install to skip-and-hint, so maintainer product review is still needed.
