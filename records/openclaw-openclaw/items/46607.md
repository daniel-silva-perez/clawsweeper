## Summary

Keep open. Current main still leaves Google/Gemini string prompts containing local audio refs as text, so the requested behavior is not implemented, but this PR is not merge-ready because it patches a stale/fallback wrapper path and introduces prompt-controlled file reads bounded to process cwd instead of the active workspace or sandbox boundary.

## What This Changes

The PR rewrites Google/Gemini headless payload text parts so resolvable local audio `@file` references become Gemini `inlineData` parts, with tests for inlined, missing, absolute, and oversized refs.

## Best Possible Solution

Implement the audio-ref conversion in the active Google provider wrapper or transport seam, thread the session workspace/sandbox boundary into the resolver with realpath checks, reuse shared media MIME detection where possible, and cover valid audio, cwd-versus-workspace, sandbox escape, oversized, unresolved, and common M4A brand cases.

## Reproduction Assessment

Yes. A focused static reproduction is available: on current main, a Google/Gemini string user prompt containing `@sample.ogg` becomes a `{ text: "...@sample.ogg..." }` part in `convertGoogleMessages`, not an `inlineData` audio part.

## Solution Assessment

No. The requested behavior is valid, but this PR is not the best solution yet because it patches a stale/fallback wrapper, uses `process.cwd()` as the file trust boundary, and duplicates narrow audio sniffing instead of using the active provider and media seams.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Use the active workspace for audio refs:** `src/agents/pi-embedded-runner/extra-params.ts:251-256`
  - body: `resolveGoogleInlineAudioPath` bounds prompt-controlled file reads to `process.cwd()`, but embedded/headless runs pass the effective session workspace into `applyExtraParamsToAgent`. When the gateway cwd differs from the workspace or sandbox copy, valid refs are missed or files can be read from the wrong boundary; thread the active workspace or sandbox media root into this resolver and realpath-check against it.
  - confidence: 0.88
- **[P2] Move the hook to the active Google wrapper:** `src/agents/pi-embedded-runner/extra-params.ts:368`
  - body: The patch wires the rewrite into a local `createGoogleThinkingPayloadWrapper`, but current main routes Google/Gemini CLI provider wrapping through the plugin hook and shared provider stream helper. Rebase and place the payload hook in the active Google wrapper or transport seam so it actually runs for `google-gemini-cli`.
  - confidence: 0.77
- **[P2] Accept common M4A MP4 brands:** `src/agents/pi-embedded-runner/extra-params.ts:204`
  - body: The `.m4a` sniff only accepts a small audio-specific major-brand set, so valid M4A files with common MP4 brands such as `isom` or `mp42` remain plain text refs. Reuse shared MIME detection or accept compatible `.m4a` brands before rejecting.
  - confidence: 0.8
