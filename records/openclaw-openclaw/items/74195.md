## Summary

Keep this open. Current main has prior fixes for the JSON variant of `agents list` and several related CLI hang classes, but the reported text path is still materially different: `agents list` in human mode still opts into plugin loading, `openclaw agents` falls through to the same text list action, and `status` text still runs the richer channel/status scan while `status --json` skips channel table work. The issue is detailed, recent, unprotected, not paired with an open PR, and not fully covered by the closed related reports.

## Best Possible Solution

Keep the issue open and fix the text command paths directly: make `openclaw agents` and `openclaw agents list` avoid eager full plugin/runtime bootstrap while preserving useful text summaries from read-only metadata, then audit `openclaw status` text for unbounded plugin/metadata/channel work so `--timeout` is honored and the text path exits when JSON does.
