## Summary

Keep open. Current main still lacks the requested `memory/recent` continuity layer and the related feature issue was consolidated here, but this PR is not merge-ready because the advertised compaction path is not registered, snapshot parsing drops the final conversation summary section, and the session-memory archive path regresses local-time naming behavior.

## What This Changes

The PR adds memory-host-sdk continuity helpers and tests, an LLM continuity extractor, session-memory recent snapshot writes, and post-compaction/compaction-summary reads of recent continuity state.

## Best Possible Solution

Keep this PR open as the active implementation candidate only if maintainers want this core continuity layer. Before merge, preserve existing session-memory contracts, register and document the compaction hook path, fix snapshot parsing with round-trip coverage, update memory/hooks docs for the new surface, and run the changed gate on a rebased branch.

## Solution Assessment

No as-is. The additive `memory/recent` direction may be maintainable, but this implementation is not the best current fix until it preserves existing reset/new behavior, wires the advertised event through hook metadata, and fixes parser read-back correctness.
