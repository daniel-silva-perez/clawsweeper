## Summary

Keep this PR open. Current main still computes memory-flush and preflight thresholds with raw `contextWindow - reserveTokensFloor - softThresholdTokens` arithmetic, so `reserveTokensFloor >= contextWindowTokens` still collapses the token gate and leaves the linked bug unfixed. The PR is still open, merge-ready, and a maintainer comment says a narrow repair was pushed to keep the contributor path canonical.

## Best Possible Solution

Keep this PR open and review/land this branch, or an equivalent fix, that centralizes the memory-flush threshold calculation, preserves a positive context-aware gate for oversized reserve floors and small context windows, uses the same helper in preflight/runtime diagnostics, and keeps regression coverage for `reserveTokensFloor >= contextWindowTokens`. After merge, close the linked #50611 as fixed.
