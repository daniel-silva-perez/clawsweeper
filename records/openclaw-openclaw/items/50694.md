## Summary

Keep this PR open. Current main still uses raw `contextWindow - reserveTokensFloor - softThresholdTokens` threshold math in the memory-flush gate, preflight diagnostics, and runtime flush path, so the linked `reserveTokensFloor >= contextWindowTokens` bug is not implemented on main. The PR remains a focused implementation candidate with maintainer follow-up context and no obvious supply-chain/security-sensitive file changes in the provided diff.

## Best Possible Solution

Keep this PR open and review or land this branch, or an equivalent fix, that centralizes the memory-flush threshold calculation, preserves a positive context-aware gate for oversized reserve floors and small context windows, uses the same helper for preflight/runtime diagnostics, and keeps regression coverage for `reserveTokensFloor >= contextWindowTokens`. After merge, close #50611 as fixed.
