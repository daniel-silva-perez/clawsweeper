## Summary

Keep this PR open. Current main still uses raw `contextWindow - reserveTokensFloor - softThresholdTokens` threshold math in the memory-flush and preflight paths, so the linked `reserveTokensFloor >= contextWindowTokens` bug remains unfixed on main. The PR is still the active contributor-preserving implementation candidate, and its diff is limited to source, tests, and changelog.

## Best Possible Solution

Keep this PR open and review/land this branch, or an equivalent fix, that centralizes the memory-flush threshold calculation, preserves a positive context-aware gate for oversized reserve floors and small context windows, uses the same helper for preflight/runtime diagnostics, and keeps regression coverage for `reserveTokensFloor >= contextWindowTokens`. After merge, close #50611 as fixed.
