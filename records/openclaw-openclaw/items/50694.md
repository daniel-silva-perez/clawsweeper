## Summary

Keep this PR open. Current main still computes memory flush and preflight thresholds directly from `contextWindowTokens - reserveTokensFloor - softThresholdTokens`, so the reported `reserveTokensFloor >= contextWindow` case still collapses the token gate and is not implemented on main. The PR remains an active implementation candidate, with maintainer context noting a narrow repair was pushed to keep this contributor PR canonical.

## Best Possible Solution

Keep this PR open as the active/canonical implementation candidate. Maintainers should review the repaired threshold helper and tests, resolve any remaining review comments if needed, and land this PR or an equivalent fix that makes both preflight compaction and memory flush use the same positive, context-aware threshold calculation.
