## Summary

Keep open. Current main still has the reported abort race, and the issue is now paired with open PR #74979 using closing syntax, so the issue should remain open until that PR is corrected, merged, or replaced.

## What This Changes

The issue asks OpenClaw to prevent pre-aborted embedded agent prompt attempts from starting floating Pi Agent loops after the attempt lifecycle exits.

## Best Possible Solution

Revise and review #74979, or an equivalent narrow PR, so the actual Pi prompt-start boundary observes the run abort signal, pre-aborted runs never call `activeSession.prompt(...)`, escaped active runs are cancelled and queues cleared during abort cleanup, and focused regression coverage locks the pre-aborted and abort-during-preflight paths.

## Reproduction Assessment

Yes. The issue provides a deterministic pre-aborted `AbortSignal` test shape, and current main still has the same call-order hazard; this read-only review did not execute the test.

## Solution Assessment

Yes, the narrow repair direction is the best fit: make prompt startup abort-aware and add regression tests, while deferring the configurable max-LLM-call cap as defense-in-depth unless maintainers explicitly want broader policy work now.
