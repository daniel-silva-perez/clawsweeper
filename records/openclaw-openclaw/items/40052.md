## Summary

Keep this PR open. Current main still routes default-agent model and fallback edits through `agents.list[index].model`, while the documented default model contract is `agents.defaults.model`. The PR addresses a real Control UI bug, but should be updated to handle the stale default-agent override cleanup and missing `defaultId === null` regression coverage raised in review.

## Best Possible Solution

Keep this open or replace it with a maintainer-owned patch that routes default-agent primary and fallback edits to `agents.defaults.model`, keeps non-default edits on `agents.list[].model`, clears or migrates stale default-agent `agents.list[].model` overrides during the save path, and adds focused regression coverage for default-agent, non-default-agent, and `defaultId === null` behavior.
