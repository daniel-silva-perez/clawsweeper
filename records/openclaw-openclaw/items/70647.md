## Summary

Keep open. The PR adds useful Nemotron empty-turn regression coverage that current main still lacks, but it is not merge-ready because its single-attempt/no-retry assertion does not match current main's OpenAI-compatible retry policy for the bundled NVIDIA provider.

## What This Changes

The PR adds two tests in `src/agents/pi-embedded-runner/run.incomplete-turn.test.ts` for NVIDIA/Nemotron empty terminal turns with unsigned thinking-only content and fully empty content.

## Best Possible Solution

Keep the coverage test-only, but make it faithful to the current provider contract: mock NVIDIA/Nemotron as `api: "openai-completions"` and assert the retry/exhaustion behavior, unless maintainers intentionally add a narrow production exception for NVIDIA/Nemotron and cover that exception explicitly.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a mocked empty NVIDIA/Nemotron stop turn with `effectiveModel.api === "openai-completions"` will enter the empty-response retry path, while the PR currently relies on the harness default `api: "messages"`.

## Solution Assessment

No. Test-only coverage is the right boundary, but this implementation pins a no-retry behavior that does not represent the bundled NVIDIA provider metadata on current main.

## Review Findings
