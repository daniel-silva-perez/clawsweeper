## Summary

Keep open. Current main still has transcript-backed usage/cost summaries, provider quota snapshots, and Control UI aggregation, but it does not implement the requested native append-only per-call usage ledger with durable event IDs, source/job/run attribution, idempotent writes, backfill, or ledger-first readers. Recent maintainer and bot discussion also treats the open CLI/gateway usage PRs as adjacent work, not closure for this ledger request.

## What This Changes

Request native per-model usage logging through a dedicated usage log or ledger so users can track model costs, source attribution, cron/background usage, and budget trends without parsing all session transcripts.

## Best Possible Solution

Add a core typed append-only UsageLedgerEntry written at the normalized model-call usage/cost seam, storing provider/model, agent/session/run/source/job attribution, token/cache/cost/status/latency metadata, durable event IDs, and no prompt or completion content. Then make usage.cost, CLI/UI summaries, and an explicit backfill path prefer that ledger while preserving transcript scanning for older data and keeping provider quota snapshots separate.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The gap is verifiable with high confidence by source/docs inspection: current main has transcript-backed summaries and provider quota surfaces, but no ledger type, usage.jsonl storage path, ledger writer, backfill command, or ledger-first reader.

## Solution Assessment

Yes, keeping this open for maintainer review is the best current handling. A narrow automated fix would be premature because the remaining work is a core storage/API/privacy/idempotency/backfill design, not a small missing parser or UI hook.
