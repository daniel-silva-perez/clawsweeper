## Summary

Keep this issue open. Current main still has transcript-backed usage summaries, `/usage cost`, gateway `usage.cost`, provider quota snapshots, and Control UI usage views, but it does not implement the requested native append-only per-call usage ledger with durable event IDs, source/job/run attribution, idempotency/backfill semantics, and ledger-first readers. The latest maintainer comment and related open PRs also treat #49064 and #70205 as adjacent rather than closure.

## Best Possible Solution

Keep this open until core adds a typed append-only UsageLedgerEntry written at the normalized model-call usage/cost seam. The ledger should store provider/model, agent/session/run/source/job attribution, token/cache/cost fields, cost status/breakdown, status, latency, and durable idempotency metadata without prompt or completion content. `usage.cost`, CLI/UI summaries, and an explicit backfill command should prefer the ledger while preserving transcript scanning as a compatibility fallback; #49064 should be reviewed or retargeted because it is a narrower transcript-scanning CLI summary.
