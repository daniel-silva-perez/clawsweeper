## Summary

Keep open. Current main still provides transcript-backed usage and cost summaries plus provider quota/status surfaces, but it does not implement the requested append-only per-call usage ledger with durable event IDs, source/job/run attribution, idempotent writes, backfill, or ledger-first readers. The issue discussion also treats the adjacent CLI and gateway PRs as non-closure for the ledger request.

## What This Changes

The issue requests native per-model usage logging for cost tracking, model-mix analysis, budget monitoring, and source/session attribution without manually parsing all session transcripts.

## Best Possible Solution

Add a core typed append-only UsageLedgerEntry written at the normalized model-call usage/cost seam, storing provider/model, agent/session/run/source/job attribution, token/cache/cost/status/latency metadata, durable event IDs, and no prompt or completion content. Then make usage.cost, CLI/UI summaries, and an explicit backfill path prefer that ledger while preserving transcript scanning for older data and keeping provider quota snapshots separate.
