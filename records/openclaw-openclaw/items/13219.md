## Summary

Keep open. Current main still has transcript-backed usage and cost summaries plus provider quota snapshots, but it does not implement the requested native append-only per-call usage ledger; related open PRs are adjacent transcript-summary or aggregation work and should not close this feature before maintainer design review.

## What This Changes

Request a native per-model usage ledger/log for per-call token and cost tracking with session, run, source, and job attribution.

## Best Possible Solution

Add a core typed append-only UsageLedgerEntry written at the normalized model-call usage/cost seam, storing provider/model, agent/session/run/source/job attribution, token/cache/cost/status/latency metadata, durable event IDs, and no prompt or completion content. Then make usage.cost, CLI/UI summaries, and an explicit backfill path prefer that ledger while preserving transcript scanning for older data and keeping provider quota snapshots separate.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The gap is verifiable by source/docs inspection: current main has transcript-backed summaries and provider quota surfaces, but no ledger type, usage.jsonl storage path, ledger writer, backfill command, or ledger-first reader.

## Solution Assessment

Yes for triage: keeping this open for maintainer review is the best current handling. A transcript-only CLI command or aggregation fix does not solve the durable per-call ledger request, and a narrow automated patch would be premature without storage, API, privacy, and idempotency decisions.
