## Summary

Keep open. Current main still provides transcript-backed usage/cost summaries, provider quota snapshots, and UI aggregation, but it does not implement the requested native append-only per-call usage ledger with durable event IDs, source/job/run attribution, idempotent writes, backfill, or ledger-first readers. The related open CLI and gateway PRs are adjacent transcript-summary work and should not close this ledger request before maintainers decide the storage/API boundary.

## What This Changes

Request a native per-model usage log or ledger that records each LLM call with token, cache, cost, model, session, and source attribution for cost tracking.

## Best Possible Solution

Add a core typed append-only UsageLedgerEntry written at the normalized model-call usage/cost seam, storing provider/model, agent/session/run/source/job attribution, token/cache/cost/status/latency metadata, durable event IDs, and no prompt or completion content. Then make usage.cost, CLI/UI summaries, and an explicit backfill path prefer that ledger while preserving transcript scanning for older data and keeping provider quota snapshots separate.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The gap is verifiable with high confidence by source/docs inspection: current main has transcript-backed summaries and provider quota surfaces, but no ledger type, usage.jsonl storage path, ledger writer, backfill, or ledger-first reader.

## Solution Assessment

Yes, keeping this open for maintainer review is the best current handling. A narrow automated fix would be premature because the remaining work is a core storage/API/privacy/idempotency design, not a small missing parser or UI hook.
