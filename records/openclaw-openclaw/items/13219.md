## Summary

Keep this issue open. Current main has useful transcript-backed usage summaries, a usage dashboard, provider quota snapshots, cron run-log usage fields, and an open transcript-scanning CLI PR, but it still lacks the requested native append-only per-call usage ledger with durable event IDs, source/job/run attribution, backfill/idempotency semantics, and ledger-first readers. The issue discussion also contains explicit maintainer context to keep this open and treat #49064/#70205 as adjacent rather than closure.

## Best Possible Solution

Keep this open until core adds a typed append-only `UsageLedgerEntry` written at the normalized model-call usage/cost seam. It should store provider/model, agent/session/run/source/job attribution, token/cache/cost fields, cost status/breakdown, status, latency, and durable idempotency metadata without prompt or completion content. `usage.cost`, CLI/UI summaries, and an explicit backfill command should prefer the ledger while preserving transcript scanning as compatibility fallback; provider quota snapshots and real-time third-party integrations should remain adjacent surfaces. Review or retarget #49064 because it currently uses closing syntax for this issue but only adds a transcript-scanning CLI summary.
