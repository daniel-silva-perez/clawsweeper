## Summary

Keep open. Current main still provides transcript-backed usage and cost summaries plus provider quota/status surfaces, but it does not implement the requested native append-only per-call usage ledger. The issue discussion also explicitly treats the open CLI and gateway work as adjacent rather than closure for this request.

## What This Changes

Request native per-model and per-call usage logging for cost tracking so users can aggregate model, session, source, token, cache, and cost data without parsing session transcripts.

## Best Possible Solution

Add a core typed append-only usage ledger written at the normalized model-call usage/cost seam, storing provider/model, agent/session/run/source/job attribution, token/cache/cost/status/latency metadata, durable event IDs, and no prompt or completion content. Then make usage.cost, CLI/UI summaries, and an explicit backfill path prefer the ledger while preserving transcript scanning for older data.
