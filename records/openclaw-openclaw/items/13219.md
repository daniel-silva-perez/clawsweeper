## Summary

Keep open. Current main still has transcript-backed token/cost summaries, provider quota snapshots, and UI aggregation, but not the requested native append-only per-call usage ledger with durable event IDs, source/run/job attribution, idempotent writes, backfill, or ledger-first readers. The related open PRs are adjacent work and the issue discussion explicitly says they should not close the ledger request.

## What This Changes

Request a native per-model/per-call usage log for cost tracking, model-mix optimization, source attribution, and future budget or quota reporting.

## Best Possible Solution

Add a core typed append-only UsageLedgerEntry written at the normalized model-call usage/cost seam, storing provider/model, agent/session/run/source/job attribution, token/cache/cost/status/latency metadata, durable event IDs, and no prompt or completion content. Then make usage.cost, CLI/UI summaries, and an explicit backfill path prefer that ledger while preserving transcript scanning for older data and keeping provider quota snapshots separate.
