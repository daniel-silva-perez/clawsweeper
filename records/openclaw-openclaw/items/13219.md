## Summary

Keep open. Current main still has transcript-backed usage/cost summaries, provider quota snapshots, and UI/CLI-adjacent usage surfaces, but it does not implement the requested native append-only per-call usage ledger; the linked open usage CLI and aggregation PRs are adjacent transcript-summary work rather than closure for this ledger request.

## What This Changes

The issue requests native per-model usage logging with a dedicated append-only usage log for cost tracking, source attribution, budget monitoring, and model-mix optimization.

## Best Possible Solution

Add a core typed append-only UsageLedgerEntry written at the normalized model-call usage/cost seam, storing provider/model, agent/session/run/source/job attribution, token/cache/cost/status/latency metadata, durable event IDs, and no prompt/completion content. Then make usage.cost, CLI/UI summaries, and an explicit backfill path prefer that ledger while preserving transcript scanning for older data and keeping provider quota snapshots separate.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The gap is verifiable with high confidence by source/docs inspection: current main has transcript-backed summaries and provider quota surfaces, but no ledger type, usage.jsonl storage path, ledger writer, backfill command, or ledger-first reader.

## Solution Assessment

Yes for triage: keeping this open for maintainer review is the right handling. A transcript-only CLI or aggregation fix does not solve the durable per-call ledger request, and a small automated patch would be premature without storage/API/privacy/idempotency decisions.
