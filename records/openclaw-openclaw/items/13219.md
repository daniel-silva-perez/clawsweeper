## Summary

Keep open. Current main still provides transcript-backed token/cost summaries, provider quota snapshots, and Control UI aggregation, but it does not implement the requested native append-only per-call usage ledger with durable event IDs, source/run/job attribution, idempotent writes, backfill, or ledger-first readers. The open CLI and gateway PRs are adjacent and the discussion explicitly treats them as non-closure for this request.

## What This Changes

Request native per-model usage logging through a dedicated usage log for cost tracking, budget monitoring, cron/session attribution, and model-mix optimization.

## Best Possible Solution

Add a core typed append-only `UsageLedgerEntry` written at the normalized model-call usage/cost seam, storing provider/model, agent/session/run/source/job attribution, token/cache/cost/status/latency metadata, durable event IDs, and no prompt or completion content. Then make `usage.cost`, CLI/UI summaries, and an explicit backfill path prefer that ledger while preserving transcript scanning for older data.
