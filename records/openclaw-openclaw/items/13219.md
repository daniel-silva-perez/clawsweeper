## Summary

Keep open. Current main has transcript-backed token/cost summaries, provider quota snapshots, per-session/per-model dashboard aggregation, and a gateway usage-cost command, but it does not implement the requested native append-only per-call usage ledger with durable event IDs, source/run/job attribution, idempotent writes, backfill, or ledger-first readers. The linked open CLI PR is transcript-scan based, and maintainer discussion explicitly says adjacent usage summaries should not close the ledger request.

## What This Changes

The issue requests a native per-model usage log or ledger for LLM calls so users can track cost by model, session, source, and automated job without parsing all session transcripts.

## Best Possible Solution

Add a core typed append-only UsageLedgerEntry written at the normalized model-call usage/cost seam, storing provider/model, agent/session/run/source/job attribution, token/cache/cost/status/latency metadata, durable event IDs, and no prompt or completion content. Then make usage.cost, CLI/UI summaries, and an explicit backfill path prefer that ledger while preserving transcript scanning for older data and keeping provider quota snapshots separate.
