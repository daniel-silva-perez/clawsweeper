## Summary

Keep this issue open. Current main has transcript-backed usage summaries, `/usage cost`, gateway `usage.cost`, provider quota snapshots, and adjacent open PRs, but it still lacks the requested native append-only per-call usage ledger with durable event IDs, source/job/run attribution, idempotency/backfill semantics, and ledger-first readers. The latest issue discussion also explicitly distinguishes #49064 and #70205 as adjacent rather than closing this work.

## Best Possible Solution

Keep this open until core adds a typed append-only UsageLedgerEntry written at the normalized model-call usage/cost seam. The ledger should store provider/model, agent/session/run/source/job attribution, token/cache/cost fields, cost status/breakdown, status, latency, and durable idempotency metadata without prompt or completion content. usage.cost, CLI/UI summaries, and an explicit backfill command should prefer the ledger while preserving transcript scanning as compatibility fallback; #49064 should be reviewed or retargeted because it is a narrower transcript-scanning CLI summary.
