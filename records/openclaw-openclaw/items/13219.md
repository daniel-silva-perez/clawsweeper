## Summary

Keep open. Current main still provides transcript-backed usage and cost summaries plus provider quota/status surfaces, but it does not implement the requested append-only per-call usage ledger with durable event IDs, source/job/run attribution, idempotent writes, backfill, or ledger-first readers. The issue discussion also treats the adjacent open CLI and gateway PRs as non-closure for this ledger request.

## What This Changes

Add native per-model usage logging through a dedicated per-call usage ledger for cost tracking, source attribution, and model-mix analysis.

## Best Possible Solution

Add a core typed append-only UsageLedgerEntry written at the normalized model-call usage/cost seam, storing provider/model, agent/session/run/source/job attribution, token/cache/cost/status/latency metadata, durable event IDs, and no prompt or completion content. Then make usage.cost, CLI/UI summaries, and an explicit backfill path prefer that ledger while preserving transcript scanning for older data and keeping provider quota snapshots separate.
