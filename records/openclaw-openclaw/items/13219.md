## Summary

Keep open. Current main has transcript-backed usage and cost summaries, provider quota/status surfaces, and Control UI aggregation, but it still does not implement the requested native append-only per-call usage ledger. The issue discussion and adjacent open PRs explicitly treat transcript-summary work as separate from the ledger request.

## What This Changes

Request native per-model usage logging that records each LLM call with model, session/source attribution, tokens/cache, and cost for cost tracking and model-mix optimization without manually scanning all session transcripts.

## Best Possible Solution

Add a core typed append-only `UsageLedgerEntry` written at the normalized model-call usage/cost seam, storing provider/model, agent/session/run/source/job attribution, token/cache/cost/status/latency metadata, durable event IDs, and no prompt or completion content. Then make `usage.cost`, CLI/UI summaries, and an explicit backfill path prefer that ledger while preserving transcript scanning for older data.
