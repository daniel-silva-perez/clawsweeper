## Summary

Keep open. Current main still has adjacent high-context UI, manual `/new` and `/reset`, and compaction/overflow recovery, but it does not implement topic-change detection, a smart new-session reminder at high context usage, or automatic session switching at 95%. The prior ClawSweeper review reached the same conclusion, and the relevant files are unchanged from that reviewed SHA to current main.

## Best Possible Solution

Keep this open for maintainer product review. A good implementation should likely be opt-in, start with a non-destructive reminder rather than silent auto-switching, reuse fresh token snapshots and existing reset/session APIs, and define consistent behavior for Control UI and message channels before coding.
