## Summary

Keep open. Current main has manual session reset, daily/idle session rollover, and high-context compaction UI, but it still lacks topic-change detection, a high-context new-session reminder, or automatic session switching at the requested 95% threshold. The nearby topic/session-management and token-cap reports are related but do not fully supersede this request.

## What This Changes

Add an intelligent session-switch prompt when context usage is high and the user changes topics, with automatic new-session behavior at very high context usage.

## Best Possible Solution

Define an opt-in session hygiene feature that uses fresh token snapshots, starts with a non-destructive reminder instead of silent rollover, and shares a documented policy across Control UI and message-channel surfaces before considering any automatic 95% reset behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. The absence of the requested feature is high-confidence from source inspection: high-context UI only offers compaction, and reset confirmation only runs for explicit /new or /reset commands.

## Solution Assessment

Unclear that the proposed 95% auto-switch is the best default. A safer maintainable direction is maintainer-reviewed, opt-in reminder behavior first, with automatic switching gated by explicit policy and durable-state safeguards.
