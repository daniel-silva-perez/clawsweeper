## Summary

Keep open. Current main has adjacent context-pressure UI, manual `/new` and `/reset`, and automatic/manual compaction, but it still does not implement topic-change detection, a smart new-session reminder at high context usage, or automatic session switching at 95%.

## Best Possible Solution

Keep this open for maintainer product review. A good path would likely be opt-in, start with a non-destructive reminder rather than silent auto-switching, reuse fresh token snapshots and existing reset/session APIs, and define consistent behavior for Control UI plus message channels before implementation.
