## Summary

Keep open. Current main has useful adjacent pieces: fresh context usage display, a high-context compact action, auto-compaction, and manual/daily/idle session rollover. It does not implement the requested behavior of detecting topic changes and prompting or auto-starting a new session based on context usage. The related #51485 context is Telegram topic labeling, not this session lifecycle behavior.

## Best Possible Solution

Keep this open for a focused session-lifecycle design. The best path is either a core opt-in policy that combines fresh context usage with explicit or reliable topic-change signals and surfaces a user prompt before rollover, or a new plugin/runtime seam that can request a session rollover prompt with access to incoming message text, current token usage, and the active session key. Tests should cover high-context same-topic continuation, explicit topic-change wording, manual decline, and no auto-switch for related follow-up work.
