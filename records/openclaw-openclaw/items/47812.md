## Summary

Keep open. Current main still has manual session reset, daily/idle rollover, and high-context compaction/notice behavior, but it does not implement topic-change detection, a high-context new-session reminder, or automatic session switching at the requested threshold. Related topic/session-management and context-usage reports are adjacent rather than a full replacement for this request.

## What This Changes

Request adding a context-usage and topic-change-aware session reminder that prompts for a new session around 80% usage and automatically starts one around 95% usage.

## Best Possible Solution

Define an opt-in session hygiene feature shared across Control UI and message-channel surfaces: use fresh token snapshots, detect explicit or high-confidence topic changes, show a non-destructive reminder first, and gate any automatic rollover behind explicit config and durable-state safeguards.

## Reproduction Assessment

Not applicable as a bug reproduction. The absence of the requested feature is high-confidence from current source inspection: high-context UI only offers compaction, and reset paths are explicit command/button or daily/idle policy.

## Solution Assessment

Unclear. The reminder portion may be a useful direction, but the proposed 95% automatic switch is not clearly the safest default; a maintainer-reviewed opt-in prompt-first policy is narrower and less surprising.
