## Summary

Keep open. Current main has localized fixes for image-model routing, Gemini usage mapping, and MS Teams Graph diagnostics, but this issue’s remaining Moonshot/WhatsApp duplicate tool-call example is still represented by open #51593, and current source does not show a canonical cross-channel/content-verification contract that resolves the broader blind-spot concern.

## Best Possible Solution

Keep this open but narrow it. Mark the fixed examples as progress, leave #51593 as the concrete Moonshot/WhatsApp tracker, and turn this issue into a specific contract for content verification: fetched media payloads must be non-empty and inspectable, image/PDF routing should emit structured fallback diagnostics, provider usage mappings should distinguish missing from zero usage, and each channel/provider path should have targeted regression coverage.
