## Summary

Keep open. Current main has point fixes for several examples in the report: text-only image attachments are preserved/offloaded for image-model inspection, Gemini usageMetadata is mapped into OpenClaw usage fields, and MS Teams Graph media failures now surface diagnostics. But the umbrella is not fully resolved: the provided context still has the Moonshot/WhatsApp duplicate tool-call report open, and current source/docs show localized validation and attempt tracking rather than a canonical cross-channel/content-verification contract that answers the broader blind-spot concern. The issue is recent, coherent, and not eligible for stale cleanup.

## Best Possible Solution

Keep this issue open for now, but narrow it. The best maintainer path is to treat the fixed examples as evidence of progress, leave #51593 to track the remaining Moonshot/WhatsApp group-chat failure, and convert this issue into a specific cross-cutting contract: verify fetched media payloads and provider usage/routing outputs, surface missing or empty content as structured diagnostics, and add targeted regression tests for each channel/provider path before closing.
