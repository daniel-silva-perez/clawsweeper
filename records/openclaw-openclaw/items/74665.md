## Summary

Keep open. Current main still appears to leave the reported OpenAI Responses replay path vulnerable to overlong or malformed same-model tool ids, and open PR #74850 now references this issue with closing syntax, so the right cleanup state is to keep the issue open until that PR lands or is closed.

## What This Changes

The issue asks OpenClaw to normalize OpenAI Responses replay tool identifiers so stored `call_id|item_id` pairs are not replayed as invalid `function_call.id` or `function_call_output.call_id` values.

## Best Possible Solution

Review and land PR #74850, or an equivalent targeted patch, that adds Responses-specific deterministic id normalization before pi-ai converts replay messages and includes regression coverage for overlong `call_id` values and malformed function-call item ids.

## Reproduction Assessment

Yes. The issue includes a no-credential payload-capture reproduction, and current source inspection shows main plus `@mariozechner/pi-ai@0.70.6` can still pass same-model stored ids through to the Responses payload without the requested normalization.

## Solution Assessment

Yes. A Responses-specific normalizer before pi-ai payload conversion is the narrow maintainable direction; broad generic tool-id sanitization is riskier because OpenAI Responses reasoning replay depends on preserving valid `call_id|fc_id` pairs.
