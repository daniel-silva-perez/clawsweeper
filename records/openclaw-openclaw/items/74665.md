## Summary

Keep open. Current main still has a plausible same-provider OpenAI Responses replay gap: OpenAI-family replay policy disables generic tool-call-id sanitization for Responses APIs, while the pinned pi-ai converter only normalizes tool-call IDs for cross-model/cross-provider replay and then splits same-model IDs directly into `function_call.id` and `call_id`. Existing OpenClaw tests cover canonical `call_123|fc_123` preservation and reasoning downgrade cases, but not overlong or non-`fc_` same-provider IDs like the report describes.

## What This Changes

The issue asks OpenClaw to normalize OpenAI Responses tool-call and item identifiers before replay so follow-up requests after tool use are accepted by Responses-compatible providers.

## Best Possible Solution

Add an OpenAI Responses-specific replay identifier normalization step that deterministically caps and sanitizes the call id, ensures the item id is replay-safe with an `fc_` shape when present, and rewrites the matching tool result to the same normalized call id without breaking reasoning-paired `call_id|fc_id` replay.

## Repair Work Prompt

Fix the OpenAI Responses replay identifier bug reported in https://github.com/openclaw/openclaw/issues/74665. Current main disables generic OpenAI Responses tool-call-id sanitization, and pi-ai only normalizes tool ids for cross-model replay, so same-provider stored ids like `{long_call_id}|{long_non_fc_item_id}` can be replayed as invalid `function_call.call_id`, `function_call.id`, and `function_call_output.call_id`. Implement a Responses-specific normalizer in the OpenClaw replay path or OpenAI provider-owned policy/helpers, not a broad core migration. Do not blindly enable the existing strict alphanumeric sanitizer if it collapses `call_id|fc_id` and loses the Responses item-id shape. Preserve current reasoning replay behavior for canonical `call_123|fc_123` with replayable `rs_*` metadata and the existing downgrade behavior when reasoning is absent. Add focused tests for overlong same-provider OpenAI Responses and OpenAI Codex Responses replay ids, including matching `function_call_output.call_id`. Update `docs/reference/transcript-hygiene.md` if the documented OpenAI provider matrix changes. Add a changelog fix entry crediting `@flashosophy`; do not touch unrelated provider policies or dependency versions.
