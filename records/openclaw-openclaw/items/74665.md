## Summary

Keep open. Current main still leaves the reported same-model OpenAI Responses replay path able to pass raw overlong or malformed tool ids into the outbound Responses payload, and open PR #74850 already references this issue with closing syntax.

## What This Changes

Normalize OpenAI Responses replay tool identifiers so overlong or malformed `call_id` and function-call item ids are not resent after tool use.

## Best Possible Solution

Land PR #74850, or an equivalent focused patch, that normalizes Responses `call_id` and `fc_*` item ids at the replay conversion boundary while preserving valid canonical `call_id|fc_id` pairs, with regression coverage for same-model overlong ids and malformed item ids.

## Reproduction Assessment

Yes. The issue includes a no-credential payload-capture reproduction, and current source inspection shows the same-model path preserves raw tool ids before directly splitting them into Responses `function_call` and `function_call_output` payload fields.

## Solution Assessment

Yes. A Responses-specific normalizer at the replay/transport boundary is the narrow maintainable fix; a broad generic tool-id sanitizer would risk breaking valid OpenAI reasoning replay pairs.
