## Summary

Keep open: current main still preserves raw same-model OpenAI Responses tool IDs through the replay sanitizer and converter, and #74850 is an open, unmerged PR with closing syntax for this issue.

## What This Changes

The issue requests normalizing stored OpenAI Responses `call_id|response_item_id` tool identifiers before replay emits `function_call` and `function_call_output` payload fields.

## Best Possible Solution

Review and land #74850, or an equivalent focused patch, that normalizes Responses `call_id` and `fc_*` item IDs at the replay conversion boundary while preserving valid canonical `call_id|fc_id` reasoning pairs.

## Reproduction Assessment

Yes. The issue includes a no-credential payload-capture reproduction, and source inspection on current main shows the same-model replay path can preserve and directly split raw stored tool IDs into outbound Responses fields.

## Solution Assessment

Yes. A Responses-specific normalizer at the replay/transport boundary is the narrow maintainable fix; a broad generic sanitizer would risk breaking valid OpenAI reasoning replay pairs.
