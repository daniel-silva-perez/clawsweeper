## Summary

Current main still has the reported foot-gun: a non-empty `agents.list` with no explicit `default: true` resolves the default agent to the first listed entry, unmatched routes use that default, and `openclaw config set agents.list ...` writes the list with only the generic update message. This should stay open for a narrow UX guard rather than closing as implemented or stale.

## Best Possible Solution

Keep this open and implement a conservative UX guard: warn when `agents.list` becomes non-empty without an explicit `default: true` and without `id: "main"`, especially when the previous effective default was implicit `main` or existing `main` state is present. Preserve the documented first-entry fallback unless maintainers explicitly approve a breaking routing-contract change.

## Repair Work Prompt

Implement a conservative UX guard for this issue. Current behavior should remain: `resolveDefaultAgentId` uses `agents.list[].default`, otherwise the first list entry, otherwise `main`. Do not change that fallback contract unless a maintainer explicitly asks. Add a warning when `openclaw config set` or batch set makes `agents.list` non-empty while the resulting list has no `default: true` entry and no `id: "main"`, especially from a previously absent/empty `agents.list` where the effective default was implicit `main`. The warning should tell the user that unbound traffic will route to the first listed agent and that adding `{ "id": "main", "default": true }` preserves main continuity. Keep machine-readable `--json` output clean; follow existing runtime logging conventions. Add focused unit/integration coverage for config set and any helper you introduce. Update docs only if needed to mention the warning while preserving the documented first-entry fallback. Add a changelog fix entry with appropriate thanks, likely `Thanks @barronlroth`; consider also crediting related prior context from #24554 if the final PR uses it. Do not migrate sessions, auto-inject agents, or rewrite existing user config beyond the requested set operation.
