## Summary

Current main still has the reported behavior: once `agents.list` is non-empty, unresolved routing and bare session keys use `agents.list[].default`, otherwise the first list entry, and only fall back to implicit `main` when the list is absent or empty. The `openclaw config set` path still writes `agents.list` without a targeted warning that implicit `main` continuity may be displaced.

## Best Possible Solution

Keep this open and implement a conservative UX guard: warn when `agents.list` becomes non-empty without an explicit `default: true` and without `id: "main"`, especially when the previous effective default was implicit `main` or existing `main` state is present. Preserve the documented first-entry fallback unless maintainers intentionally approve a breaking routing-contract change.
