## Summary

Keep open. Current main still creates reasoning payloads without preserving `isReasoning` into the returned embedded-runner payloads, and the cron announce selection helpers still filter only `isError`; an open closing PR already exists, so cleanup should wait for the canonical fix to land or be replaced.

## What This Changes

The issue asks OpenClaw to prevent thinking/reasoning payloads from leaking into Matrix and Feishu isolated cron announce delivery and to address the related duplicate-delivery behavior.

## Best Possible Solution

Land one canonical, rebased fix that preserves `isReasoning` through embedded-runner payloads and filters reasoning payloads in the shared cron outcome/picker path before summary, output text, deliverable payloads, awareness queuing, and fallback delivery can reach Matrix or Feishu. The final patch should include focused regression tests and explicitly settle whether reasoning-only output should ever be delivered.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: current main creates a reasoning reply item, drops its marker in the returned payload, then cron announce helpers select payload text while filtering only `isError`.

## Solution Assessment

No for current main; it leaves the reported leak path open. The likely best fix is the narrow shared payload-marker propagation plus cron picker filtering, but maintainer review should reconcile PR #73363's fallback behavior against the stricter #40534 approach.
