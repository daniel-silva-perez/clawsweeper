## Summary

Keep open. Current `main` still has the cron announce reasoning-leak path, and an open PR with closing syntax is already paired to this issue, so cleanup should wait for maintainer review and canonical fix selection.

## What This Changes

Prevent Matrix and Feishu isolated cron announce delivery from exposing model reasoning/thinking payloads or duplicate visible output.

## Best Possible Solution

Land one canonical fix that preserves `isReasoning` through embedded-runner payloads and filters reasoning payloads in the shared cron outcome path before summary, output text, deliverable payload selection, awareness events, and direct Matrix/Feishu announce delivery, with focused regression tests for the chosen reasoning-only fallback policy.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current `main`: a reasoning reply item is created, the returned payload drops `isReasoning`, and the cron announce helpers select non-error text or deliverable payloads without a reasoning guard.

## Solution Assessment

Unclear pending maintainer choice. The right boundary is the shared embedded-payload and cron outcome path; a Matrix- or Feishu-only filter would leave shared cron selectors and other direct announce paths exposed.
