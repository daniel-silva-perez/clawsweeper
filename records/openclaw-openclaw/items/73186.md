Keep open. Current main still has the reported cron announce leak path, and an open PR with closing syntax is already paired to this issue, so it should wait for maintainer review rather than be closed or sent to a competing repair lane.

## What This Changes

Prevent Matrix and Feishu isolated cron announce delivery from exposing model reasoning/thinking payloads or duplicate visible output.

## Best Possible Solution

Land one canonical fix on main that preserves `isReasoning` through embedded-runner payloads and filters reasoning payloads in the shared cron outcome path before summary, output text, deliverable payload selection, awareness events, and direct Matrix/Feishu announce delivery, with focused regression tests for the selected reasoning-only fallback policy.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: a reasoning reply item is created, the returned payload drops `isReasoning`, and the cron announce helpers select non-error text or deliverable payloads without a reasoning guard.

## Solution Assessment

Unclear on the exact patch until maintainer review chooses the fallback policy. The best boundary is the shared embedded-payload and cron outcome path; per-channel Matrix or Feishu filtering alone would leave other cron announce selectors exposed.
