## Summary

Keep open. Current main still has the payload propagation and cron picker gaps, and an open closing PR already exists, so this issue should wait for maintainer review/merge of the canonical fix rather than be closed by cleanup.

## What This Changes

The issue reports leaked reasoning/thinking text and occasional duplicate delivery in cron announce output for Matrix and Feishu.

## Best Possible Solution

Land one canonical, rebased fix that preserves `isReasoning` through embedded-runner payloads and filters reasoning payloads in the shared cron outcome/picker path before summary, output text, deliverable payloads, and any awareness/fallback delivery can reach Matrix or Feishu. The final patch should include focused regression tests for the embedded payload marker and cron announce picker behavior, and it should explicitly settle the reasoning-only fallback policy.

## Reproduction Assessment

Yes. The source path is enough to reproduce at high confidence: a reasoning payload is created with `isReasoning: true`, the returned payload drops that marker, and the cron announce helpers currently only skip `isError` before selecting text or deliverable payloads.

## Solution Assessment

Unclear pending maintainer choice. PR #73363 is a focused implementation candidate and now covers Feishu deliverable selection, but its reasoning-only fallback behavior differs from the stricter no-reasoning fallback requested in the issue and described by #40534.
