## Summary

Keep open. Current main still has the reported cron announce leak path: the embedded runner creates reasoning payload items but does not copy `isReasoning` into the returned payload object, and the shared cron selectors still skip only `isError` before selecting summary/output/deliverable payloads. An open closing PR already exists, so this should wait for maintainer review rather than be closed or routed to a competing repair job. https://github.com/openclaw/openclaw/issues/73186

## What This Changes

The issue requests preventing reasoning/thinking payloads from appearing in Matrix and Feishu cron announce delivery, including duplicate delivery caused by mixed visible/reasoning payloads.

## Best Possible Solution

Land one canonical, rebased fix that preserves `isReasoning` through embedded-runner payloads and filters reasoning payloads in the shared cron outcome path before summary, output text, deliverable payloads, awareness events, and direct Matrix/Feishu announce delivery. The final patch should include focused regression tests for marker propagation, picker selection, deliverable payload selection, and the chosen reasoning-only fallback policy.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: a reasoning reply item is created, the returned payload drops `isReasoning`, and the cron announce helpers select non-error text or deliverable payloads without a reasoning guard.

## Solution Assessment

Unclear pending maintainer choice. The right boundary is the shared embedded-payload and cron outcome path, but the open implementation candidates differ on whether reasoning-only payloads should ever fall back to delivery.
