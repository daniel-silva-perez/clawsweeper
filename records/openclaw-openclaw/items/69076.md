## Summary

Current main does not implement this PR’s intended behavior. Manual-cancel exec exits still classify as `failed`, background manual cancels with output still enqueue notify-on-exit wakeups, and process `kill`/`remove` still return failed tool details after successful termination requests. The PR is a focused implementation candidate, not obsolete.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to validate and either land a version of the PR that classifies explicit manual cancellations as `killed`, suppresses notify-on-exit for killed sessions, and keeps failed/timeout/spawn-error paths distinct, or request targeted changes if maintainers want a narrower exec-only fix before the Telegram diagnostics addition.
