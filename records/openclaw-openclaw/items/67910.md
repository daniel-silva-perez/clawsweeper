## Summary

Keep open. Current main still renders `openclaw memory status` dreaming through the short-term/deep promotion config, while the linked user-visible bug is specifically about light-only dreaming being hidden as `Dreaming: off`. This PR remains an active implementation thread for #67868, but the provided head also has review concerns around deep-disabled/`limit=0` semantics and adjacent `--force`/timezone cleanup, so it needs maintainer review rather than cleanup closure.

## What This Changes

This PR changes memory-core CLI status output to report dreaming phase state from the full dreaming config, adds phase-oriented CLI tests, adds `memory status --force` handling, and trims blank timezone labels in `/dreaming status`.

## Best Possible Solution

Keep the PR open for maintainer review or replace it with a narrower memory-core change that makes `openclaw memory status` accurately show configured light/deep/REM phase state while preserving the effective managed deep-promotion signal. Cover light-only, REM-only, deep-disabled, global-disabled, and deep `limit=0` cases, and split unrelated `--force` or timezone cleanup unless maintainers explicitly want them bundled.
