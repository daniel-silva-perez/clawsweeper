## Summary

Keep open. Current main still does not implement the install-time warning this PR proposes: `fix_npm_permissions()` still rewrites npm's prefix to `~/.npm-global` with the old info/success messages and no explicit follow-up warning about future `npm i -g` updates without sudo. The linked bug was closed as superseded by this open PR, so this remains the canonical implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this small installer-message change, or replace it with an equivalent maintainer-owned warning in `fix_npm_permissions()` that says the installer is saving `prefix=~/.npm-global` to the user's npm config and that future global npm updates for that install should use `npm i -g` without sudo. A focused test around the non-writable-prefix branch would make the cleanup stronger.
