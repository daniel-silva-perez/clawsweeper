## Summary

Keep open. Current main still does not implement this PR’s intended installer warning: `fix_npm_permissions()` silently switches npm to `~/.npm-global` with the old info/success messages, while the newer duplicate-install warning and update docs only help after or outside that install-time prefix rewrite. The linked bug #44365 was closed as superseded by this open PR, so this PR remains the canonical target-repo implementation candidate rather than obsolete work. Security pass found only installer message/log-level changes in the provided PR patch, with no dependency, workflow, secret, permission, or command-execution expansion.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this small installer-message change, or replace it with an equivalent maintainer-authored warning in `fix_npm_permissions()` that clearly says the installer is saving `prefix=~/.npm-global` to the user's npm config and that future global npm updates for that install should use `npm i -g` without sudo. A focused test around the non-writable-prefix branch would make the cleanup stronger.
