## Summary

Keep open. Current main and the latest stable tag still route short cron retention through a cron-run-only reaper, while default keep-mode subagent sessions and ACP sessions do not have an equivalent durable, short-retention cleanup path.

## What This Changes

The issue requests that completed subagent and ACP sessions be covered by `cron.sessionRetention` or an equivalent retention policy so `sessions.json` and transcripts do not accumulate indefinitely.

## Best Possible Solution

Keep this as the canonical tracker and design a restart-safe retention path for completed native subagent and ACP sessions at the session-store or lifecycle boundary. The finished behavior should preserve active and intentionally persistent sessions, use existing transcript archival/deletion helpers, cover startup catch-up, and document whether the controlling knob is `cron.sessionRetention`, a new subagent/ACP retention setting, or `session.maintenance`.
