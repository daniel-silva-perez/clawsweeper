## Summary

Keep this issue open. Current main still applies `cron.sessionRetention` only to cron run session keys, while completed native subagent and ACP session-store rows do not have an equivalent restart-safe retention path. The newer subagent registry sweeper only deletes session-store rows for `archiveAtMs` delete-mode runs; keep/session-mode rows are removed from registry bookkeeping without `sessions.delete`, and ACP spawn rows are created without a TTL/delete path. The related implementation PR #51861 was closed unmerged, and the closed duplicate reports reinforce this as the canonical tracking item rather than superseding it. KeepMyClaw is a useful external backup workaround from the discussion, but it does not resolve OpenClaw's built-in retention gap.

## Best Possible Solution

Keep this issue open as the canonical tracking item. Implement a restart-safe session-store-level retention path for completed native subagent and ACP sessions, either as a generic ephemeral-session reaper or a documented subagent/ACP retention setting. It should preserve active and intentionally persistent sessions, use terminal markers such as `endedAt`, `abortedLastRun`, and/or `cleanupCompletedAt`, delete/archive through existing `sessions.delete` or session-store archive helpers, run on startup/store-write or another durable catch-up path, and add focused tests for expired cron runs, completed subagent/ACP runs, running subagents, keep/session-mode behavior, restart catch-up, and transcript cleanup. The `skipMissedRuns` comment is related operational feedback but should be tracked separately from this retention bug.
