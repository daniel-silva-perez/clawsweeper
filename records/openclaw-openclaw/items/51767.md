## Summary

Keep open. Current main and the latest release still apply `cron.sessionRetention` only to isolated cron run keys, while completed native subagent and ACP session-store rows remain outside a restart-safe retention path. The related implementation PR was closed unmerged, and the duplicate reports add scope rather than superseding this canonical issue.

## What This Changes

Apply retention cleanup to completed subagent and ACP session-store rows instead of only pruning isolated cron run sessions.

## Best Possible Solution

Keep this as the canonical tracking item and design a restart-safe session-store-level retention path for completed native subagent and ACP run sessions. The end state should preserve active and intentionally persistent sessions, use durable terminal markers such as `endedAt`, `abortedLastRun`, or `cleanupCompletedAt`, delete/archive through existing session deletion or transcript-archive helpers, run on startup/store-write or another durable catch-up path, and document whichever retention config owns subagent/ACP cleanup.
