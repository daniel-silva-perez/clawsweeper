## Summary

Keep open. Current main still implements `cron.sessionRetention` as a cron-run-only session-store sweep. The subagent registry can delete explicit `cleanup: "delete"` archive runs, but default keep-mode/session-mode subagent rows and ACP rows still do not have a durable retention path. The related implementation PR #51861 is closed unmerged, and the closed duplicate/adjacent reports reinforce this as the canonical tracker.

## What This Changes

Apply retention cleanup to completed subagent and ACP session-store rows instead of only pruning isolated cron run sessions.

## Best Possible Solution

Keep this as the canonical tracker and design a restart-safe session-store-level retention path for completed native subagent and ACP run sessions. The end state should preserve active and intentionally persistent sessions, use durable terminal markers, delete/archive through existing session deletion or transcript archive helpers, run on startup or another durable catch-up path, and document the owning retention config.
