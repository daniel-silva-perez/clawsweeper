## Summary

Current main still appears to route low-level `gateway call agent` requests by `sessionKey`/`sessionId`, while treating `to` as a delivery target. A WeChat-shaped `agent:main:openclaw-weixin:direct:...` value in `to` is therefore not used to select that session and can fall back to the configured main session, matching the reported bug. Related #69576 fixed account-scoped delivery mirror matching, but it does not cover this low-level RPC `to`-as-session-key path.

## Best Possible Solution

Keep this issue open and fix the core gateway `agent` RPC boundary: when no explicit `sessionKey` or `sessionId` is provided and `to` is a canonical `agent:<agentId>:...` session key, route the run to that session or return a clear validation error. The fix should stay plugin-agnostic, add regression coverage for `gateway call agent` with a channel session key in `to`, and preserve existing delivery-target semantics for normal channel destinations.
