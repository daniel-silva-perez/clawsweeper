## Summary

Keep this PR open. Current main at bdfb408ce6eeb724a638c106104dbbcae50b612c still lacks Discord ack reaction timing: Discord queues the initial ack before dispatch, the queue helper has no timing mode, config schemas/types expose only ackReaction and ackReactionScope, and docs/generated metadata do not document a run-start option. The PR remains a plausible implementation candidate, but it needs to be ported from the older src/discord paths to the current extensions/discord plugin surface.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to port the feature onto extensions/discord, expose a Discord-scoped channels.discord.ackReactionTiming config type/schema/help/docs/generated metadata surface, keep the default at received, and retain regression coverage for immediate default behavior, run-start deferral, scope gating, and no status reaction before run start. Review it alongside the open Slack #45421 and Telegram #46258 companion PRs for consistent channel semantics.
