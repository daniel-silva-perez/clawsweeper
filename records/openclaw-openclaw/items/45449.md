## Summary

Keep this PR open. Current main at e476523082a6fff3a0c3048077d2f0e52413fbf2 still does not implement optional Discord ack reaction timing: Discord resolves the ack reaction and immediately queues the initial status/ack reaction before dispatch, while config types, schemas, generated metadata, docs, and tests do not expose or cover `ackReactionTiming`. The PR remains a plausible implementation candidate, but it needs to be ported from the older `src/discord` paths to the current `extensions/discord` plugin surface and reviewed alongside the open Slack and Telegram companion PRs.

## Best Possible Solution

Keep this PR open as an implementation candidate. Port the feature to `extensions/discord`, expose a Discord-scoped `channels.discord.ackReactionTiming` / account override with default `received`, and add current-surface regression coverage for immediate default behavior, run-start deferral, scope gating, and no status reaction before run start. Review it together with the Slack #45421 and Telegram #46258 companion PRs for consistent channel semantics.
