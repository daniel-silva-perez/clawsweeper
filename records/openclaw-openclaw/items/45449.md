## Summary

Keep this PR open. Current main still lacks Discord ack reaction timing: the Discord plugin queues the initial ack immediately before dispatch, the queue helper has no timing mode, Discord/shared config schemas expose only ackReaction and ackReactionScope, and the public Discord docs do not document a run-start option. The PR remains a plausible implementation candidate, but it needs to be ported from the older src/discord layout to the current extensions/discord plugin surface and rechecked against the review feedback already addressed in the PR branch.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to port the feature onto extensions/discord, expose a channel-scoped channels.discord.ackReactionTiming config type/schema/help/docs/generated metadata surface, keep the default at received, and retain focused regression coverage for immediate default behavior, run-start deferral, scope gating, and no status reaction before run start. Review it alongside the open Slack #45421 and Telegram #46258 companion PRs for consistent channel semantics.
