## Summary

Keep this PR open. Current main at 16906780fd2c481308ec4b80f96eb3b93d0add46 still lacks Discord `ackReactionTiming`: Discord queues the ack before dispatch, the queue helper has no timing mode, config/schema/docs expose only `ackReaction` and `ackReactionScope`, and targeted searches found no current implementation. The PR remains a plausible implementation candidate, but it needs a rebase/port to the current `extensions/discord` plugin layout.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to port it onto `extensions/discord`, add a channel-scoped `channels.discord.ackReactionTiming` type/schema/help/docs surface, preserve the guarded deferred lifecycle from the later PR commits, and keep focused regression tests for default immediate behavior, run-start deferral, scope gating, and no status reaction before run start.
