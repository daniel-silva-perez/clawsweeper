## Summary

Keep this PR open. Current main still has the Anthropic provider beta-header wrapper gated on resolved extra-param betas, while the reported OAuth failure needs the OAuth/default beta merge even when no `context1m` or `anthropicBeta` extra param resolves. The PR is paired with open #41444, but its diff targets the older embedded-runner inline path rather than the current provider-owned runtime path.

## Best Possible Solution

Keep this PR open as an implementation candidate for #41444, but port the useful logic into `extensions/anthropic/stream-wrappers.ts`: apply the OAuth/default beta merge for direct Anthropic requests even when `resolveAnthropicBetas` returns `undefined`, preserve custom-endpoint suppression, and add focused regression tests for the model/provider-header-only OAuth case.
