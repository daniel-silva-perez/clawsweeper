## Summary

Keep this PR open. Current main still has the same remaining Anthropic OAuth/header issue in the active provider-owned path: the beta wrapper is only composed when extra-param betas resolve, while the reported failing shape needs OAuth/default betas re-merged even with no `context1m` or `anthropicBeta` extra param. This PR is a useful implementation candidate for #41444, but its code change targets the older embedded-runner inline path instead of the current Anthropic plugin hook.

## Best Possible Solution

Keep this PR open as an implementation candidate for #41444, but port the useful logic into `extensions/anthropic/stream-wrappers.ts`: compose the Anthropic beta header wrapper for direct public Anthropic requests even when `resolveAnthropicBetas` returns no explicit betas, pass an empty beta list so OAuth/default beta merging still runs, preserve custom-endpoint suppression, and add a focused regression test for an OAuth token plus model/provider-level `anthropic-beta` with no `context1m` or `anthropicBeta` extra param.
