## Summary

Keep this PR open. Current main still gates the active Anthropic provider beta wrapper on explicit extra-param betas, so OAuth/default beta merging is skipped when the user only has model/provider-level `anthropic-beta` headers. The PR is also paired with the same-author open bug #41444, so it remains an implementation candidate, but it needs to be ported from the older embedded-runner path to the current Anthropic plugin hook.

## Best Possible Solution

Keep this PR open and update it as the implementation candidate for #41444: move the unconditional Anthropic beta wrapper composition into extensions/anthropic/stream-wrappers.ts, pass an empty beta list when resolveAnthropicBetas returns no explicit betas so OAuth/default beta merging still runs, preserve custom-endpoint suppression, and add a focused regression test for an OAuth token plus model/provider-level anthropic-beta with no context1m or anthropicBeta extra param.
