## Summary

Keep this PR open. Current main still gates the bundled Anthropic provider beta-header wrapper on resolved extra-param betas, so the OAuth/default beta merge path is skipped for the reported model-header-only shape. The PR is also paired with same-author open #41444 via closing syntax, but its patch targets the older embedded-runner shim rather than the current provider-owned runtime path.

## Best Possible Solution

Keep this PR open as an implementation candidate for #41444, but port the useful logic into the provider-owned Anthropic path. The best fix is to apply the OAuth/default beta merge for direct Anthropic requests even when resolveAnthropicBetas returns undefined, while preserving custom endpoint suppression, service-tier, fast-mode, thinking-prefill, and API-key behavior. Add focused regression coverage for sk-ant-oat plus model/provider anthropic-beta headers with no context1m or anthropicBeta extra param.
