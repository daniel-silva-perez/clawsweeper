## Summary

Keep this PR open. Current main still skips the Anthropic beta-header wrapper in the active provider plugin path when `resolveAnthropicBetas()` has no explicit extra-param betas, so the OAuth/default beta merge is not guaranteed for model/provider-level `anthropic-beta` headers. The PR is also paired with the same-author open bug #41444, but it needs to be ported from the older embedded-runner location to the current `extensions/anthropic/stream-wrappers.ts` path.

## Best Possible Solution

Keep this PR open as the implementation candidate for #41444, but update it for current main: move the unconditional beta-wrapper composition into `extensions/anthropic/stream-wrappers.ts`, pass an empty beta list when `resolveAnthropicBetas` returns no explicit betas, preserve direct-Anthropic/custom-endpoint suppression, and add a focused regression test for an OAuth token plus model/provider-level `anthropic-beta` with no `context1m` or `anthropicBeta` extra param.
