## Summary

Keep this PR open. Current main still does not implement `agents.defaults.systemPrompt.identityLine` or `disabledSections`; the only shipped config surface is full `systemPromptOverride`, which is not equivalent to preserving the assembled OpenClaw prompt while changing the identity preamble. The PR remains a plausible implementation candidate for the open identity-preamble request in #43562, but it needs a refresh before merge because the proposed config is not in the strict schema, is not wired through runtime prompt call-sites, and raises a maintainer/security decision around disabling safety sections.

## Best Possible Solution

Keep this PR open as an implementation candidate. Align it with #43562, decide whether the public scope is only a safe identity-line override or also section disabling, then add the typed config, strict schema, generated help/docs, runtime wiring through every prompt assembly path, focused validation tests, and end-to-end prompt composition coverage before merge.
