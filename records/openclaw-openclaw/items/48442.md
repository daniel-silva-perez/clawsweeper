## Summary

Keep this PR open. Current main still does not implement `agents.defaults.systemPrompt.identityLine` or `disabledSections`; it only offers full `systemPromptOverride`, which is not equivalent to changing the identity preamble while preserving the rest of the assembled prompt. The PR remains a plausible implementation candidate for the still-open identity-preamble request in #43562, but the provided PR discussion correctly identifies merge blockers around schema validation, runtime wiring, and the safety implications of disabling prompt sections.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a refresh before merge. Align the public surface with #43562, decide whether the scope is only a safe identity-line override or also section disabling, then add typed config, strict Zod schema, generated schema/help/docs, runtime wiring through all prompt assembly paths, validation tests, and end-to-end prompt composition tests.
