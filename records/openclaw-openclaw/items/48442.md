## Summary

Keep this PR open. Current main still does not implement the proposed `agents.defaults.systemPrompt.identityLine` or `disabledSections` config, and the PR remains an incomplete implementation candidate because review comments identified missing strict-schema support and missing runtime wiring. The related identity-preamble issue #43562 is still open, so the underlying product problem is not superseded or shipped.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a refresh before merge. The best path is to align it with open issue #43562, decide whether section disabling is acceptable or whether the feature should be limited to a safer identity-line override, then add the typed config, strict Zod schema, generated schema/help/docs, and runtime wiring through embedded, compact, CLI, and auto-reply prompt assembly with tests covering validation and actual runtime prompt composition.
