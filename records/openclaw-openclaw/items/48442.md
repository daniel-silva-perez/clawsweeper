## Summary

Keep this PR open. Current main still lacks the proposed granular `agents.defaults.systemPrompt.identityLine` and `disabledSections` surface; the existing `systemPromptOverride` only replaces the whole assembled prompt. The PR is a plausible implementation candidate for open #43562, but it needs schema/docs/generated-config updates, production prompt-path wiring, a rebase, and an explicit security/product decision on disabling safety sections before merge.

## Best Possible Solution

Keep this PR open as an implementation candidate for #43562. Maintainers should decide whether OpenClaw wants only a narrow identity-line override, a broader section-disabling API, or a different identity model; then implement typed config plus strict schema/generated help/docs, wire the values through all prompt assembly paths, add config-validation and end-to-end prompt composition tests, and explicitly review the safety-section disabling behavior before merge.
