## Summary

Keep this PR open. Current main still does not implement the proposed `agents.defaults.systemPrompt.identityLine` or `disabledSections` config surface; it only supports full prompt replacement through `systemPromptOverride`. The PR remains a plausible implementation candidate for open #43562, but it needs schema/generated-config/docs/runtime wiring and explicit security/product review before it could merge.

## Best Possible Solution

Keep this PR open as an implementation candidate for #43562. Maintainers should decide whether OpenClaw wants a narrow identity-line override, a broader section-disabling API, or a different identity model; then implement the chosen typed config through strict schemas, generated config docs/help, embedded/compact/CLI/auto-reply prompt paths, and end-to-end prompt composition tests, with explicit review for disabling safety guidance.
