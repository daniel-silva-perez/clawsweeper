## Summary

Keep this PR open. Current main still does not implement `agents.defaults.systemPrompt.identityLine` or `disabledSections`; it only supports full `systemPromptOverride`. The PR remains a plausible implementation candidate for the still-open identity-preamble problem in #43562, but it needs a refresh because the proposed config is not wired through validation or production prompt assembly and the section-disabling surface needs security review.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a refresh before merge. Align it with #43562, decide whether the public surface should be limited to a safe identity-line override or also include section disabling, then add typed config, strict Zod schema, generated schema/help/docs, runtime wiring through embedded/compact/CLI/auto-reply prompt assembly, and tests that cover both validation and actual runtime prompt composition.
