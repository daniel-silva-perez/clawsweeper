## Summary

Keep this PR open. Current main still does not implement `agents.defaults.systemPrompt.identityLine` or `disabledSections`; it only has full `systemPromptOverride`. The PR remains a plausible implementation candidate for open #43562, but the proposed surface needs schema, docs/help generation, runtime wiring, and an explicit security/product decision before merge.

## Best Possible Solution

Keep this PR open as an implementation candidate for #43562. Maintainers should decide whether the supported surface is a narrow identity-line override or also section disabling; then add typed config plus strict schema/generated help/docs, wire it through all prompt assembly paths, add config validation and end-to-end prompt composition tests, and explicitly review the safety-section disabling behavior before merge.
