## Summary

Keep open. Current main still has the reported local TUI slash-command mismatch: `/status` and `/compact` are advertised through the shared command list, but the TUI command switch has no local handlers for them and falls through to sending the raw slash text into the local embedded chat/model path.

## What This Changes

The issue asks for TUI local embedded mode to stop sending advertised slash commands like `/status` and `/compact` to the model as ordinary user prompts.

## Best Possible Solution

Add a local embedded TUI command boundary for advertised shared commands. `/status` and `/compact` should either execute through reusable shared command logic or render explicit local-mode responses, with regression coverage proving they do not enter `sendChat()` or `agentCommandFromIngress()` as ordinary prompts.

## Reproduction Assessment

Yes. The current-main static path is high confidence: slash input enters `handleCommand()`, autocomplete advertises `/status` and `/compact`, no local cases handle them, and the default branch forwards the raw text into the embedded chat path.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix: handle advertised commands locally or through a shared adapter, rather than continuing to send known slash commands to the model or removing shared commands globally.
