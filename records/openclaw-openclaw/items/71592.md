## Summary

Current main still matches the reported local TUI failure path: shared slash-command metadata advertises `/status` and `/compact`, but the TUI local command switch has no local branch for either command and the default path sends the raw text as a chat message. This is a valid narrow bug, not a cleanup close.

## What This Changes

The issue asks for TUI local embedded mode to stop sending advertised slash commands such as `/status` and `/compact` to the embedded model as ordinary user prompts.

## Best Possible Solution

Add a local embedded TUI command boundary for shared advertised commands. `/status` and `/compact` should either execute through reusable shared command logic or render explicit local-mode responses, with regression coverage proving they do not enter `sendChat` or `agentCommandFromIngress` as model prompts.

## Reproduction Assessment

Yes. The current-main static path is high confidence: slash input routes to `handleCommand()`, shared metadata advertises `/status` and `/compact`, no local cases handle them, and the default branch sends the raw text into the embedded chat path.

## Solution Assessment

No patch is attached to this issue. The maintainable fix is narrow local-mode handling or a shared command adapter for advertised commands, not removing shared registry commands globally or allowing known slash commands to fall through to the model.
