## Summary

Keep open. Current main still matches the reported failure path: TUI slash input is parsed locally, `/status` and `/compact` are advertised through shared command metadata, but the local TUI command switch has no cases for them and falls through to the embedded send path.

## What This Changes

Ensure advertised TUI local-mode slash commands such as `/status` and `/compact` are handled locally or explicitly rejected instead of being sent to the model as user text.

## Best Possible Solution

Add a local embedded TUI command boundary for shared advertised commands. `/status` and `/compact` should either execute through reusable shared command logic or render explicit local responses, with regression coverage proving they do not enter the embedded agent run path.

## Reproduction Assessment

Yes. The current-main static path is high confidence: slash input routes to `handleCommand()`, the advertised commands lack local cases, the default branch calls `sendMessage(raw)`, and local mode turns that into `agentCommandFromIngress()`.

## Solution Assessment

No patch is attached to this issue. The maintainable fix is narrow local-mode command handling or a shared command adapter, not removing the shared registry commands globally or blocking all unknown slash text.
