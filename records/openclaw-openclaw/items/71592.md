## Summary

Keep this issue open. Current `main` still has the reported local TUI mismatch: `/status` and `/compact` are advertised through shared slash-command autocomplete, but the TUI command switch does not handle them locally and falls through to the embedded chat/model path.

## What This Changes

The issue requests that advertised TUI local-mode slash commands such as `/status` and `/compact` be handled locally, routed through shared command execution, or explicitly reported as unsupported instead of being sent as model prompts.

## Best Possible Solution

Add a local embedded TUI command boundary for advertised shared commands. `/status` and `/compact` should either execute through reusable shared command logic or render explicit local-mode responses, with regression coverage proving they do not enter `sendChat()` or `agentCommandFromIngress()` as ordinary prompts.

## Reproduction Assessment

Yes. The current-main static path is high confidence: slash input enters `handleCommand()`, autocomplete advertises `/status` and `/compact`, no local cases handle them, and the default branch forwards the raw text into the embedded chat path.

## Solution Assessment

Yes, the requested direction is the right narrow fix. The maintainable solution is local-mode handling or a shared command adapter for advertised commands, not removing shared commands globally or continuing to send known slash commands to the model.
