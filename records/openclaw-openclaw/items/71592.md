## Summary

Keep this issue open. Current main still has the reported local TUI command-routing gap: `/status` and `/compact` are advertised from the shared command registry, but the TUI-local command switch does not handle them and its fallback path sends the raw slash text into the embedded agent run path.

## What This Changes

The issue requests that advertised TUI local slash commands such as `/status` and `/compact` execute locally, route through shared command handling, or show an explicit unsupported response instead of being sent as ordinary model prompts.

## Best Possible Solution

Add a TUI local command boundary for advertised shared commands so `/status` and `/compact` produce operator-visible local/shared command output, or a deliberate unsupported-command response, with regression coverage proving they do not enter the embedded model path as ordinary prompts.

## Reproduction Assessment

Yes. The current-main static path is high confidence: slash input enters `handleCommand()`, autocomplete advertises shared `/status` and `/compact`, the local switch lacks those cases, and the fallback sends the raw slash text through `sendChat()` into `EmbeddedTuiBackend.runTurn()`.

## Solution Assessment

Yes. The narrow maintainable fix is to handle or explicitly reject these known shared commands at the TUI local boundary; removing them from the shared registry or allowing known slash commands to reach the model would leave command parity and UX unclear.
