## Summary

Keep this issue open. Current main still advertises shared `/status` and `/compact` commands in TUI autocomplete, but the local TUI command switch has no local handler for either command and falls through to the embedded chat path.

## What This Changes

The issue requests that advertised TUI local-mode slash commands such as `/status` and `/compact` stop being sent to the embedded model as ordinary user text.

## Best Possible Solution

Add a local embedded TUI command boundary for advertised shared commands. `/status` and `/compact` should either execute through reusable shared command logic or render explicit local-mode responses, with regression coverage proving they do not enter `sendChat` or `agentCommandFromIngress` as model prompts.

## Reproduction Assessment

Yes. The current-main static path is high confidence: slash input routes to `handleCommand()`, shared metadata advertises `/status` and `/compact`, no local cases handle them, and the default branch sends the raw text into the embedded chat path.

## Solution Assessment

No patch is attached to this issue. The narrow maintainable fix is local-mode handling or a shared command adapter for advertised commands, not removing shared registry commands globally or allowing known slash commands to fall through to the model.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/71592. Current main advertises `/status` and `/compact` in TUI autocomplete via the shared command registry, but `src/tui/tui-command-handlers.ts` has no local `status` or `compact` cases, so `handleCommand()` falls through to `sendMessage(raw)`. In `openclaw tui --local`, that reaches `EmbeddedTuiBackend.sendChat()` and then `agentCommandFromIngress()`, starting a model run with the slash command as user text.

Expected fix boundary: update the TUI local command boundary so known advertised commands do not become model prompts. Prefer a reusable shared command adapter if one already fits local embedded mode; otherwise add explicit local handling/unsupported-command responses for `/status` and `/compact`. Preserve gateway-mode behavior and keep `/gateway-status` as the gateway diagnostics command. Do not remove `/status` or `/compact` from the shared registry globally, and do not broaden unrelated TUI command behavior.

Add regression coverage in `src/tui/tui-command-handlers.test.ts` proving local `/status` and `/compact` do not call `sendChat` / do not enter the embedded model path. Consider `src/tui/commands.test.ts` coverage if autocomplete/local support rules change. Add a `CHANGELOG.md` fix entry if the change is user-facing, credited to `@goslingmanagment` unless maintainers choose a different attribution.

Related context: #51270 is adjacent `/model list` / `/model status` work but does not fix this issue; #61966 is autocomplete-only; #7202 is older cross-interface context, not the current local embedded TUI fallthrough.
