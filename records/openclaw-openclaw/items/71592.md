## Summary

Keep this issue open. Current main still advertises `/status` and `/compact` to the TUI, but local embedded mode does not execute the shared command pipeline before `agentCommandFromIngress()`, so the reported slash text can still become an ordinary local model prompt.

## What This Changes

The issue requests that local embedded TUI slash commands advertised by autocomplete, especially `/status` and `/compact`, stop falling through as ordinary model prompts.

## Best Possible Solution

Add a local embedded TUI command boundary for advertised shared commands. `/status` and `/compact` should either execute through reusable shared command logic or render explicit local-mode responses, with regression coverage proving they do not enter `sendChat()` or `agentCommandFromIngress()` as model prompts.

## Reproduction Assessment

Yes. The current-main static reproduction is high confidence: slash input reaches `handleCommand()`, `/status` and `/compact` are advertised through shared metadata, no local switch cases handle them, and local embedded `sendChat()` calls `agentCommandFromIngress()` directly.

## Solution Assessment

No patch is attached to this issue. The best fix is a narrow local-mode command handling path or shared adapter, not removing `/status` or `/compact` from the shared registry or letting known slash commands continue into the model.
