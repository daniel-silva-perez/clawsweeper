## Summary

Current main still has the reported request-visible failure path. Successful embedded runs await auth-profile bookkeeping, lock-protected save failures fall back to direct saves, and those direct `auth-profiles.json` saves still throw through `saveAuthProfileStore`/`saveJsonFile`. The provided discussion also shows open, unmerged PR #67077 with closing syntax for this issue, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review or land #67077, or an equivalent focused patch. The fix should make `markAuthProfileGood`, `markAuthProfileUsed`, `markAuthProfileFailure`, and cooldown bookkeeping tolerate `auth-profiles.json` persistence failures with warning logs, preserve fatal behavior for credential/OAuth writes, and add regression coverage for EPERM from both lock-guarded and direct fallback save paths.
