## Summary

Keep open. Current main still has the reported cascade path: post-success auth-profile bookkeeping is awaited on the request path, lock-path persistence failures return null, and the direct fallback saves can still throw EPERM. The issue is also paired with open, unmerged PR #67077, so it should not be closed before that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to prevent Windows EPERM failures while writing `auth-profiles.json` from turning post-success auth-profile bookkeeping into a full gateway and fallback-chain failure.

## Best Possible Solution

Review and land #67077, or an equivalent focused patch, so post-success `lastGood`, usage, failure, and cooldown bookkeeping warn-log auth-profile persistence failures while credential and OAuth token writes continue to fail loudly. Add EPERM regression coverage for both lock-guarded and direct fallback save paths.
