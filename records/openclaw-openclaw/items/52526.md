## Summary

Keep open. The issue comments identify a real result-surface mismatch, and current main still has the same shape: CLI-runner results are built from the raw/final CLI output buffer, transcript persistence happens later on a separate path, and JSON output is emitted from the unchanged result payloads rather than a post-`before_message_write` finalized assistant message.

## Best Possible Solution

Keep this issue open and fix the CLI result path so `agent --json` uses the same finalized assistant text that is persisted after `before_message_write`. The likely implementation is to thread the finalized message from the write-hook/persistence path back into the run result, or re-read the last persisted assistant after the guarded write, then add a focused regression test for a `before_message_write` hook that prepends known text and assert both session JSONL and JSON `payloads[].text` match.
