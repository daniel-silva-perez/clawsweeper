## Summary

Keep this PR open. Current main still does not implement the Slack-owned CDN redirect-auth behavior this PR targets: it preserves Slack bearer auth only for same-origin redirects and still strips auth for `files.slack.com` to `slack-edge.com` / `downloads.slack-edge.com` hops. The PR is a plausible implementation candidate, but it needs a current-main rebase/rework before landing because the provided diff is stale around Slack media tests and runtime dispatcher boundaries.

## Best Possible Solution

Keep this PR open as a focused implementation candidate, but require a current-main rebase or replacement Slack plugin patch. The best fix belongs in `extensions/slack/src/monitor/media.ts`: retain `Authorization` only across HTTPS Slack-owned redirect hosts, strip credentials for every non-Slack redirect, preserve `fetchRemoteMedia` / SSRF protections and the local `media.runtime.ts` dispatcher boundary, keep the HTML-login-page guard, and update tests for Slack CDN auth retention, non-Slack stripping, HTTP downgrade rejection, and SSRF-blocked redirect targets.
