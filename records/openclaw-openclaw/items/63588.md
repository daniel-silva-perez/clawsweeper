## Summary

Keep this PR open. Current main still preserves Slack bearer auth only for same-origin redirects and still strips it on Slack CDN redirects, so the user-visible bug this PR targets is not implemented on main. The PR is relevant, but the branch is stale and currently has blocking review issues: its Slack media test import does not resolve, and the final rework must preserve current-main hostname allowlist/SSRF constraints while extending auth only to Slack-owned HTTPS hosts.

## What This Changes

The PR changes Slack media download auth handling to keep bearer auth across Slack-owned HTTPS redirect hosts, strip it outside Slack hosts, and updates Slack media regression tests.

## Best Possible Solution

Rebase or replace this as a focused Slack plugin patch: keep the current `media.runtime.ts` dispatcher boundary, retain `hostnameAllowlist` plus IP-range SSRF checks, preserve Authorization only for HTTPS Slack-owned redirect hosts, avoid sending credentials outside Slack hosts, and update Slack media tests for CDN retention and unsafe redirect rejection.
