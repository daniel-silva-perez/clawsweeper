## Summary

Keep this PR open. Current main still lacks the PR’s two central behaviors: HTTP Responses/Codex Responses continuations do not send `previous_response_id`, and OpenAI-compatible completions do not capture or replay `encrypted_content`; the branch is relevant but needs maintainer review because the discussion flags replay coalescing, memory-budgeting, and route-capability concerns.

## What This Changes

The PR adds OpenAI-compatible encrypted reasoning replay for Volcengine/Doubao completions, HTTP `previous_response_id` continuations for Responses/Codex Responses tool turns, and payload-shape tests for those outbound requests.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to preserve the useful encrypted-content replay and HTTP continuation work, but ensure `previous_response_id` is gated to native Responses-capable routes, replay metadata is coalesced and byte-budgeted correctly, encrypted-only reasoning keeps the full replay envelope, and targeted payload-shape tests plus core typecheck proof land with the change.
