## Summary

Keep open. The requested feature is still absent from current main, and this active contributor PR is a close match for the validated request in #64612, but the supplied patch needs fixup before merge because main now owns the memory resolver in the package host SDK and the generated/config schema surfaces are not fully aligned.

## What This Changes

The PR adds optional `agents.list[].memory` config for per-agent memory backend and QMD overrides, updates memory docs/config schema/types, and adds schema/runtime tests while keeping citations global.

## Best Possible Solution

Land this PR or an equivalent patch after rebasing onto current main, implementing the override in the package host SDK resolver/config mirror, source config types/schema/generated config surfaces, public memory docs, and focused schema/runtime regression tests for backend selection, QMD overlay behavior, isolation, and citations rejection.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Update the package memory resolver:** `src/memory-host-sdk/host/backend-config.ts:392-393`
  - body: Current main made `src/memory-host-sdk/host/backend-config.ts` a compatibility re-export, while the real resolver and host config mirror live under `packages/memory-host-sdk/src/host`. As written, this changes the old source path and leaves the package resolver/config types reading only root `params.cfg.memory`, so per-agent overrides will not work after rebasing unless the package files are updated too.
  - confidence: 0.88
- **[P2] Mirror root QMD startup fields:** `src/config/zod-schema.agent-runtime.ts:777-788`
  - body: The new per-agent QMD update schema omits `startup` and `startupDelayMs`, even though root `memory.qmd.update` accepts them and the resolver consumes them. A canary that needs gateway-start refresh settings would be rejected despite the PR promising `memory.qmd.*` overrides.
  - confidence: 0.89
- **[P2] Regenerate the public config schema:** `src/config/zod-schema.agent-runtime.ts:879`
  - body: Adding `memory` to `AgentEntrySchema` changes the generated base config schema, but this PR does not update `src/config/schema.base.generated.ts`. That leaves the published/config-editor schema without `agents.list[].memory` and should fail the generated-schema regression outside the narrow test-name filter.
  - confidence: 0.87
