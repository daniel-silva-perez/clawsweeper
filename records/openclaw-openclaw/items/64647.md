## Summary

Keep open: current main still lacks per-agent memory backend overrides, and the PR matches the requester-backed rollout need from #64612, but the patch needs maintainer review and a branch refresh before merge because it updates the old resolver location, leaves generated config schema surfaces stale, and does not fully mirror root QMD update fields.

## What This Changes

The PR adds `agents.list[].memory` for per-agent backend and QMD overrides, updates config types/schema/docs, and adds schema/runtime tests while keeping `memory.citations` global.

## Best Possible Solution

Land this PR or an equivalent patch after rebasing onto current main, implementing the override in the package memory-host SDK resolver and config mirror, reusing or fully mirroring the root QMD schema except for global-only citations, updating generated schema/help/docs, and keeping focused schema/runtime regression coverage for backend selection, QMD overlay behavior, isolation, and citations rejection.

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Update the package memory resolver:** `src/memory-host-sdk/host/backend-config.ts:392-393`
  - body: Current main made `src/memory-host-sdk/host/backend-config.ts` a compatibility re-export, while the real resolver and config mirror live under `packages/memory-host-sdk/src/host`.   - confidence: 0.86
- **[P2] Regenerate the public config schema:** `src/config/zod-schema.agent-runtime.ts:879`
  - body: Adding `memory` to `AgentEntrySchema` changes the computed base config schema, but the PR does not update `src/config/schema.base.generated.ts`. That leaves the published/config-editor schema without `agents.list[].memory` and will fail the generated-schema regression when run without the narrow test-name filter.
  - confidence: 0.88
- **[P2] Mirror root QMD startup fields:** `src/config/zod-schema.agent-runtime.ts:777-788`
  - body: The new per-agent QMD update schema omits `startup` and `startupDelayMs`, even though root `memory.qmd.update` accepts them and the resolver consumes them. A canary that needs to tune gateway-start refresh settings would be rejected despite the PR promising `memory.qmd.*` overrides.
  - confidence: 0.89
