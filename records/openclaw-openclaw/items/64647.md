## Summary

Keep open: current main still lacks the per-agent memory backend override this PR targets. The current config type/schema accept per-agent `memorySearch` but not `agents.list[].memory`, and backend resolution still reads `memory.backend` and `memory.qmd` from the root config only.

## What This Changes

The PR adds `agents.list[].memory` for per-agent backend and QMD overrides, updates config schema/types/docs, and adds regression/runtime coverage while keeping `memory.citations` global.

## Best Possible Solution

Land this PR or an equivalent patch after rebasing onto current main, implementing the override in the package host SDK resolver/config mirror, source config types/schema/generated config surfaces, public memory docs, and focused schema/runtime regression tests.
