## Summary

Keep open. Current main still hard-disables Feishu's generic block-streaming pipeline and only accepts `info.kind === "block"` text when it can be folded into the Card Kit streaming-card path, so the requested true progressive Feishu block delivery remains missing. The related closed Feishu reports cover silent-drop prevention and duplicate-card suppression, but they do not supersede this narrower behavior gap.

## What This Changes

Request Feishu to deliver runtime `block` chunks immediately as normal Feishu messages/cards when block streaming is enabled, instead of only replaying them through streaming cards or dropping them when card streaming is off.

## Best Possible Solution

Implement the fix inside the Feishu plugin boundary: resolve explicit top-level and per-account Feishu block-streaming config using the existing channel streaming helpers, let the core block pipeline run only when that setting is enabled, deliver `info.kind === "block"` through normal Feishu text/card sends, and keep Card Kit preview streaming as a separate mode with the existing silent-drop and duplicate-final protections covered by regression tests.
