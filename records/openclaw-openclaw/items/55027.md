## Summary

Keep open. Current main still does not implement true progressive Feishu block delivery: Feishu hard-disables the generic block-streaming pipeline, drops non-card block chunks, and only consumes accepted card-mode blocks through the Feishu Card Kit streaming-card path with playback animation. The related closed reports do not supersede this remaining behavior.

## Best Possible Solution

Keep this open for a Feishu plugin-owned fix. The best path is to separate Feishu preview-card streaming from true block-message streaming, add and honor an explicit channels.feishu.blockStreaming control in schema/manifest/docs, set disableBlockStreaming from that resolved config instead of hardcoding true, deliver info.kind === "block" progressively through normal Feishu message/card sends when enabled, and add regression coverage for disabled blocks, enabled progressive block delivery, and existing streaming-card duplicate suppression.
