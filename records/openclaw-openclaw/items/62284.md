## Summary

Keep this issue open. Current main still does not implement time-bounded reusable exec approvals: the only expiry is the pending approval request timeout, approval decisions remain limited to allow-once, allow-always, and deny, and durable no-prompt behavior is still permanent allow-always allowlist state. The related open PR #39126 is an implementation candidate, not shipped behavior or a clearly canonical replacement.

## Best Possible Solution

Keep the issue open and treat it as core exec approval work. The likely path is to design one bounded trust-window model, coordinated with #39126, then update the decision/protocol schema, session or in-memory approval store, enforcement checks before prompting, /approve parsing, Control UI buttons, approvals CLI visibility, docs, and regression tests across gateway/node/chat paths.
