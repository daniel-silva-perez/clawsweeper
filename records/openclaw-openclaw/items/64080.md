## Summary

Keep this PR open. Current main does not implement the PR’s intended change: Slack routing still preserves ordinary top-level channel/group messages on the shared room session, while `MessageThreadId` can still be set separately for `replyToMode=all`. The paired bug issue remains open and this PR is the linked implementation candidate, so cleanup closure would be premature.

## Best Possible Solution

Keep the PR open for maintainer review. If the duplicate-processing report is accepted, rebase the fix into `extensions/slack/src/monitor/message-handler/prepare-routing.ts`, preserve the newer seeded mention/runtime-binding behavior, and update the tests that currently assert shared top-level channel sessions under `replyToMode=all`. If maintainers intentionally want the current shared-session behavior, close this PR and the paired issue with that product rationale.
