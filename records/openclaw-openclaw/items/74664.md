## Summary

Keep open: the report is still valid in the checked-out current main. The config contract and embedded runner already recognize `compat.supportsTools: false`, but the active OpenAI Completions payload builder still emits `tools`, `tool_choice`, and the tool-history `tools: []` fallback without checking that flag. The linked fix PR #74738 is open and uses closing syntax for this issue, so the issue should remain open until that PR is reviewed and merged or replaced.

## What This Changes

Honor `compat.supportsTools: false` for OpenAI-compatible chat-completions payloads by omitting tool schemas, `tool_choice`, and the tool-history `tools: []` fallback when tools are disabled.

## Best Possible Solution

Carry the existing `supportsModelTools()` decision into `buildOpenAICompletionsParams()` itself, so direct transport callers and embedded runs both suppress tool payload fields when a model opts out, with focused regression tests for non-empty tools and replayed tool history while preserving default tool behavior.
