## Summary

Keep open. Current main still lacks an equivalent practical orchestration guide, and #43034 was closed in favor of this open PR, but the branch is not merge-ready because its peer agent-to-agent examples conflict with the current session-access contract.

## What This Changes

The PR adds `docs/concepts/multi-agent-patterns.md`, a standalone guide covering hub-and-spoke, cost-tiered routing, channel-bound specialists, heartbeat monitoring, pipeline processing, and anti-patterns.

## Best Possible Solution

Keep this as the canonical docs follow-up for #43034, but revise the guide so every copyable example matches current session-access semantics, then have docs maintainers decide the final page placement and navigation entry.

## Reproduction Assessment

Yes. The docs correctness issues reproduce by comparing the PR's copyable examples against `createAgentToAgentPolicy` and `createSessionVisibilityChecker`: the omitted `engineer` target fails the allow policy, and cross-agent `sessions_*` calls require `tools.sessions.visibility: "all"`.

## Solution Assessment

No. The proposed guide is useful in scope, but the current diff is not the best merge-ready solution until it documents both the A2A allow policy and the session visibility requirement, and either adds a nav entry or intentionally folds the material into existing multi-agent docs.
