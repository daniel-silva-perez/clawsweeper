## Summary

Keep this PR open. Current main does not contain an equivalent multi-agent orchestration patterns guide, and this remains the canonical docs follow-up for the closed related request, but the proposed page is not merge-ready because two copyable examples contradict current agent-to-agent session access behavior.

## What This Changes

This PR adds `docs/concepts/multi-agent-patterns.md`, a standalone guide covering hub-and-spoke routing, cost-tiered model selection, channel-bound specialists, heartbeat monitoring, pipeline processing, and multi-agent anti-patterns.

## Best Possible Solution

Keep the guide as the canonical docs follow-up, but revise its copyable examples so they match current `tools.agentToAgent.allow` and `tools.sessions.visibility` semantics, then make an explicit docs placement decision by adding the page to navigation or integrating the material into the existing multi-agent page.

## Reproduction Assessment

Yes. The docs correctness issues reproduce by comparing the PR's hub-and-spoke example with `createAgentToAgentPolicy` and `createSessionVisibilityChecker`: the advertised `engineer` target is not allowlisted, and cross-agent `sessions_*` calls require `tools.sessions.visibility: "all"`.

## Solution Assessment

No. The proposed guide is useful, but the current patch is not the best merge-ready solution until its A2A examples include every routed target and document the session visibility requirement for cross-agent collaboration.

## Review Findings
