## Summary

Keep this issue open. Current main has meaningful adjacent controls, including tool allow/deny policy, per-agent/group/sender tool filtering, trusted before_tool_call policy, plugin approvals, Gateway HTTP high-risk tool denies, exec approvals, sandbox/workspace guidance, and security audit warnings. It still does not implement the requested capability-manifest, explicit grant/revoke, pre-model default-deny filtering, central execution-boundary capability authorization, or durable high-risk capability audit layer for skills/tools. The linked @vainplex/openclaw-governance plugin is useful prior art for deny-at-call-time governance, but it does not replace a core/plugin-API grant model.

## Best Possible Solution

Keep this open as a core/plugin-API security hardening item. A reviewable path is to add required/optional sensitive capability metadata for plugins and skills, persist explicit operator grants and revocations, expose diagnostics in CLI/UI/doctor, filter ungranted high-risk tools before model exposure, enforce the same capability decision at each execution boundary, and write durable audit records for high-risk capability decisions. Treat @vainplex/openclaw-governance as complementary runtime policy layered after static grants, not as the canonical grant system.

## Risks / Open Questions

- Closing now would conflate adjacent name-based tool policy, HTTP deny lists, owner-only filtering, exec approvals, plugin approvals, and before_tool_call governance with the requested static capability/grant/audit system.
- A future implementation needs one authorization decision reused by model-exposed tools, direct HTTP tools, MCP/loopback paths, native harnesses, and plugin-owned command paths, otherwise pre-exposure filtering could be bypassed by another execution surface.
- The @vainplex/openclaw-governance plugin remains useful dynamic policy prior art, but it runs after tool availability has been determined and does not provide core manifest declarations, persisted grants/revocations, or durable audit records.
