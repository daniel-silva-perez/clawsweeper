## Summary

Keep this issue open. Current main has adjacent controls, but still lacks the requested core capability-manifest, explicit grant/revoke, pre-model default-deny filtering, execution-boundary authorization, and durable high-risk audit model for skills/tools.

## Best Possible Solution

Keep this open as a core/plugin-API security hardening item. A reviewable path is to add required/optional sensitive capability metadata for plugins and skills, persist explicit operator grants and revocations, expose diagnostics in CLI/UI/doctor, filter ungranted high-risk tools before model exposure, enforce the same decision at execution boundaries, and write durable audit records. Coordinate with #12219 for skill manifest shape and #63881 for MCP deny-list parity, while treating @vainplex/openclaw-governance as complementary dynamic policy.
