## Summary

Keep this issue open. Current main still renders the Control UI cron surface as a summary strip, filters, a flat job list, run history, and an edit form, while the requested timeline/day/week/calendar/heatmap/drag-to-reschedule experience is not present. PR #41892 is also still open and references this issue with closing intent, so this issue should remain open until that implementation candidate is merged or closed.

## What This Changes

Requests a Control UI cron calendar/timeline experience with timeline, day and week views, quick stats, heatmap, job-detail popups, drag-to-reschedule, and cron expression validation.

## Best Possible Solution

Keep this issue as the canonical Control UI cron calendar tracker while maintainers review, rebase, shrink, land, or close PR #41892; if only the timeline slice lands, split the remaining calendar modes and rescheduling features into narrower follow-ups.
