## Summary

Keep this issue open. Current main at ab5c8025c9d0a31873d311ae3e73f3664b0a1185 still lacks a complete Anthropic native server-tools implementation: no accepted Anthropic serverTools/PTC config surface, no request-payload injection for web_fetch/code_execution or allowed_callers, and no stream handling for Anthropic server-tool result blocks. Open PR #50692 covers a narrower Anthropic native web_search provider for #49949, but it does not cover the broader web_fetch/code_execution/PTC request here, and #23469/#23573 remain closed unmerged reference work.

## Best Possible Solution

Keep this issue open as the broad Anthropic native server-tools tracker. ## Risks / Open Questions

- Anthropic code_execution/PTC would allow Anthropic-hosted code to invoke selected OpenClaw tools via allowed_callers, so the design needs explicit opt-in, allowlists, auditability, and conservative defaults.
- Server-side web_fetch/web_search/code_execution result blocks need transcript preservation and rendering; dropping or misclassifying them could break replay, citations, or future turns.
- Open PR #50692 is useful reference material for web_search but is narrower than this issue and has already been flagged for functional/security-sensitive gaps, so it should not be treated as a complete superseding implementation.
