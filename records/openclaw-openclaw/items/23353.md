## Summary

Keep this issue open. Current main still does not implement the broad Anthropic native server-tools request: the typed and strict runtime config only expose managed web search/fetch plus OpenAI/Codex-native search; the Anthropic transport still converts OpenClaw tools into client-side Anthropic tool definitions; the stream parser only models text, thinking, redacted_thinking, and client tool_use blocks; and the bundled Anthropic plugin manifest has provider/media contracts but no web search/fetch or server-tool contract. Related prior PRs #23469 and #23573 were closed unmerged, while open #50692 is scoped to #49949's narrower native web_search provider request.

## Best Possible Solution

Keep this issue open as the broad Anthropic native server-tools tracker. Let #49949/#50692 cover the narrower native web_search provider if that PR passes review, then separately design and implement web_fetch, code_execution/PTC, allowed_callers, server content-block preservation/rendering, docs, tests, and security defaults in the Anthropic plugin/transport boundary.
