## Summary

Keep open. Current main still lacks a documented native `sessions_spawn` option for silent, parent-only, or inherited-channel completion delivery; the existing path defaults native sub-agent completion announcements back through the requester channel, and the related #13911 tracks a broader channel-level suppression design rather than this per-call parent-delivery request.

## What This Changes

The issue requests a per-call `sessions_spawn` option to suppress or reroute sub-agent completion announcements so channel-bound parent sessions can review sub-agent output before replying.

## Best Possible Solution

Add a documented native sub-agent completion delivery policy that preserves channel delivery as the default while supporting parent-only internal injection and silent completion handling; coordinate the naming and semantics with #13911 so per-call and per-channel suppression remain consistent.
