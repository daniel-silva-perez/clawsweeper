## Summary

Keep open. Current main still lacks a documented native `sessions_spawn` completion-delivery policy for silent, parent-only, or channel delivery; the existing hidden boolean only suppresses completion tracking and the ACP-only `streamTo: "parent"` does not cover native sub-agent spawns. The related #13911 is relevant but not a complete duplicate because it focuses on channel-level suppression/config, while this issue asks for per-spawn control and parent-only delivery.

## What This Changes

The issue requests a public `sessions_spawn` option to suppress or reroute sub-agent completion announcements away from the inherited channel.

## Best Possible Solution

Add a documented native sub-agent completion delivery policy that preserves channel delivery as the default while supporting parent-only internal injection and silent completion handling; coordinate naming and semantics with #13911 so per-call and per-channel suppression do not diverge.
