## Summary

Keep open. Current main still lacks a documented native `sessions_spawn` caller policy for `none | parent | channel` completion delivery, and the related older #13911 is similar but not an exact duplicate because it focuses on channel-level suppression/config rather than per-spawn parent-only delivery.

## What This Changes

The issue requests a `sessions_spawn` parameter to suppress sub-agent completion announcements or route them internally to the parent instead of auto-delivering every completion to the inherited chat channel.

## Best Possible Solution

Add a documented native sub-agent completion delivery policy that preserves current channel delivery as the default while supporting parent-only internal injection and silent completion handling; coordinate the design with #13911 so per-call and per-channel suppression do not diverge.
