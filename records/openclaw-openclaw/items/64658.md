## Summary

Keep open. Current main still lacks a documented native `sessions_spawn` policy for silent or parent-only completion delivery from a top-level channel-bound session; the existing behavior still externally delivers top-level completion announces, while parent-only injection is limited to nested sub-agent requesters. Related #13911 is relevant but not a terminal duplicate because it tracks channel-level suppression/config, while this issue asks for per-spawn caller control.

## What This Changes

Add a `sessions_spawn` option to control sub-agent completion announcements with silent, parent-only, or channel-delivered behavior.

## Best Possible Solution

Add a documented native sub-agent completion delivery policy that preserves current channel delivery as the default while supporting silent completion handling and parent-only internal injection for top-level channel-bound requesters; align naming and precedence with #13911 so per-call and per-channel controls do not diverge.

## Reproduction Assessment

Yes. The current source and regression tests provide a high-confidence reproduction path: a top-level channel requester origin such as `bluebubbles` still produces an external `agent` delivery with `deliver: true`, while native `streamTo: "parent"` is explicitly ignored.

## Solution Assessment

Unclear. A per-spawn enum is a plausible narrow solution for this issue, but the exact public names, interaction with the hidden boolean, and precedence relative to #13911's channel-level suppression need maintainer API design before implementation.
