## Summary

Keep open. Current main still lacks the Google Meet API space access controls and end-active-conference surface, and the PR is a narrow Google Meet plugin implementation with regression coverage, but it still has one user-facing docs command error before merge.

## What This Changes

The PR adds Google Meet SpaceConfig access and entry-point options to CLI/tool create flows, adds the meetings.space.settings OAuth scope, adds end-active-conference CLI/tool support, and updates Google Meet docs and tests.

## Best Possible Solution

Land the Google Meet plugin changes after correcting the docs command and completing normal CI, keeping the behavior plugin-owned and preserving the API-only access-policy handling and alias resolution before ending an active conference.

Yes. The source comparison is high-confidence: current main posts an empty spaces.create body and lacks the end_active_conference action, while the PR head adds those paths; the remaining docs issue is reproduced by comparing the new docs line to the CLI's registered `auth login` command.

## Solution Assessment

Yes, with a minor docs correction. The implementation stays in the Google Meet plugin, applies SpaceConfig only on the OAuth/API creation path, rejects unsupported browser fallback, and resolves user-friendly inputs before calling the API resource method.
