## Summary

Keep this issue open. Current main still applies `maybeApplyTtsToPayload` in the normal reply/ACP dispatch paths, but isolated cron announce delivery still turns final text into plain outbound payloads and sends them through `deliverOutboundPayloads` without a TTS directive transform. The related closed media/tool-result issue #33082 is separate: it covered `MEDIA:` voice payload delivery, not model-emitted `[[tts]]` / `[[tts:text]]` tags in cron announce text.

## Best Possible Solution

Implement a shared pre-delivery TTS transform for isolated cron announce payloads before `deliverOutboundPayloads`, using `cfgWithAgentDefaults`, the resolved delivery channel/account, agent id, final reply kind, and preserved session `ttsAuto` value. Add a focused regression test for isolated cron `delivery.mode: "announce"` where `[[tts]]` or `[[tts:text]]` output is stripped from visible text and delivered as synthesized voice/media. Keep #33082 treated as related but separate because it covered tool-produced `MEDIA:` voice payloads, not model-emitted TTS tags.
