## Summary

Keep this issue open. Current main still runs TTS directive processing in normal reply dispatch, but isolated cron announce delivery still normalizes silent tokens and sends text payloads directly through outbound delivery. The related #33082 media/tool-result fix is present, but it does not cover model-emitted `[[tts]]` / `[[tts:text]]` tags in cron announce text.

## Best Possible Solution

Implement a shared pre-delivery TTS transform for isolated cron announce payloads before `deliverOutboundPayloads`, using `cfgWithAgentDefaults`, the resolved delivery channel/account, agent id, final reply kind, and the preserved session `ttsAuto` value. Add a regression test for isolated cron `delivery.mode: "announce"` where `[[tts]]` or `[[tts:text]]` output becomes synthesized media/voice and the raw directive is removed from the visible text. Keep #33082 treated as related but already separate because it covered tool-produced `MEDIA:` voice payloads, not model-emitted TTS tags.
