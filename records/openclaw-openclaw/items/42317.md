## Summary

Keep this PR open. Current main still does not implement the intended custom `session.mainKey` routing behavior: runtime config materialization still overwrites configured main keys, central route builders still hard-code `DEFAULT_MAIN_KEY`, and current outbound/Telegram/WhatsApp paths still do not thread the configured key through. The linked issue #41790 describes a real remaining product gap even though that issue is now closed.

## Best Possible Solution

Keep this PR open as an implementation candidate, but rebase or port it onto current main. The complete fix should preserve configured `session.mainKey` through runtime materialization, thread the normalized key through central route/session-key builders and plugin SDK callers, update current outbound/Telegram/WhatsApp paths, make delivery guards recognize configured custom main keys and legacy aliases, align docs/generated config help, and add focused regression coverage for config preservation, route resolution, outbound/plugin routing, and delivery-continuity behavior.
