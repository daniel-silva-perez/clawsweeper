## Summary

Keep this PR open. Current main still erases configured `session.mainKey` during runtime config materialization and still hard-codes `DEFAULT_MAIN_KEY` in central route/session-key builders, so the PR’s intended behavior is not implemented on main. The PR remains a relevant implementation candidate, but it needs a rebase/port onto current plugin paths plus delivery-guard and docs/schema alignment before maintainer review or merge.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, preserve configured `session.mainKey` through runtime defaults and every route/session-key builder, port the Telegram/WhatsApp/outbound changes to the current plugin paths, update delivery guards to recognize configured custom main keys and legacy aliases, align docs/schema help, and add focused regression coverage for config preservation, route resolution, outbound/plugin routing, and delivery-continuity behavior.
