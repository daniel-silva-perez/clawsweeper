## Summary

Keep this PR open. Current main still does not implement the PR’s intended behavior: runtime config materialization overwrites configured `session.mainKey`, central route builders still hard-code `DEFAULT_MAIN_KEY`, and current outbound/Telegram/WhatsApp paths still lack custom-main-key propagation. The PR remains a relevant implementation candidate, but it needs a current-main rebase/port plus delivery-guard and docs/schema cleanup before it is mergeable.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase or port it onto current main, preserve configured `session.mainKey` through runtime materialization, thread the normalized key through central route/session-key builders and the plugin SDK surface, update current outbound/Telegram/WhatsApp call sites, make delivery guards recognize configured custom main keys and legacy aliases, align docs/generated config help, and add focused regression coverage for config preservation, route resolution, outbound/plugin routing, and delivery-continuity behavior.
