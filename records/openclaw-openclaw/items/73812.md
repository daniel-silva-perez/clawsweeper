## Summary

Keep open. Current main still lacks the restart-continuation reply-delivery override and explicit per-request restart deferral timeout, so the branch is not obsolete. The PR also is not ready to close or merge because the proposed timeout can be lost in the coalesced scheduled-restart path and the explicit timeout constant is duplicated across callers.

## What This Changes

The branch adds automatic source reply delivery for restart-sentinel continuation turns, adds a 30s per-request deferral timeout for explicit `gateway.restart` and `/restart` callers, and extends regression coverage for Telegram-style group topics and restart deferral.

## Best Possible Solution

Revise this PR, or open a narrow replacement, so restart-sentinel continuations explicitly request automatic source delivery while explicit restart deferral timeout state is centralized and stored with pending restart state rather than captured only in the first scheduled closure. The finished change should cover group-topic continuation visibility, direct per-request timeout, and coalesced explicit-restart ordering before merge.
