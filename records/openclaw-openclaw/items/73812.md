## Summary

Keep open. Current main still lacks the two central intended changes: restart-sentinel continuation turns do not force automatic source reply delivery in group/topic contexts, and explicit restart callers still cannot pass a per-request deferral timeout. The PR remains a focused implementation candidate, but the live review context flags a coalesced-restart timeout gap and duplicate timeout constants that should be fixed before merge.

## Best Possible Solution

Keep this PR open and request a small revision before landing: preserve the automatic source reply delivery override for restart-sentinel continuations, make per-request deferral timeout state survive coalescing/rescheduling, share the explicit restart timeout constant from one scheduler-owned location, and add a regression test for an explicit restart coalescing onto an already-pending restart with no timeout.
