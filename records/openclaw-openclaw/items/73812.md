## Summary

Keep this PR open. Current main still lacks both intended changes: restart-sentinel continuation dispatch does not force automatic source delivery for Telegram-style group/topic continuations, and explicit restart callers cannot provide a per-request deferral timeout. The PR is a focused implementation candidate with matching regression coverage, not obsolete or already implemented on main.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The best maintainer path is to land a narrow fix that forces restart-sentinel continuation turns to use automatic source delivery, adds group/topic regression coverage, and bounds explicit `gateway.restart` plus `/restart` deferral with a reviewed per-request timeout.
