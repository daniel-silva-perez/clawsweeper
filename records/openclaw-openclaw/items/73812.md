## Summary

Keep open. Current main still lacks the PR's intended restart-continuation visibility fix: the restart-sentinel dispatch path carries Telegram group/topic context, but it does not pass an explicit automatic source reply mode, so the current group/channel default can still suppress normal final replies. The PR is a narrow draft bug-fix candidate with regression coverage and no observed security-sensitive diff surface.

## Best Possible Solution

Keep this PR open and review it as the focused implementation candidate. The likely right path is the narrow gateway change proposed here: restart-sentinel continuation turns should explicitly request automatic source delivery, with the added Telegram group-topic regression test guarding the default message_tool_only group behavior.
