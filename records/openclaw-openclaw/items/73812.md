## Summary

Keep this PR open. Current main does not implement the PR’s intended restart-continuation visibility fix: the restart-sentinel dispatch path records Telegram group/topic context, but it does not pass an explicit `sourceReplyDeliveryMode: "automatic"`, so the existing group/channel default can still suppress normal final replies. The PR is a focused draft bug-fix candidate with regression coverage, not obsolete or implemented on main.

## Best Possible Solution

Keep the PR open and review it as the implementation candidate. The likely right fix is the narrow gateway change proposed here: restart-sentinel continuation turns should explicitly request automatic source delivery, with the added Telegram group-topic regression test guarding the default `message_tool_only` group behavior.
