## Summary

Keep this issue open. Current main has collapsible tool-output rows, raw tool details, and pure-JSON details blocks, but normal assistant chat messages still render their full markdown text inline with only copy/open-in-side-panel actions. The requested long assistant-message collapse/show-more workflow is not implemented, and the provided related context does not identify a canonical duplicate or merged PR for this specific UX request.

## Best Possible Solution

Implement the feature in the Control UI chat renderer: add per-message collapse state keyed by stable message/group ids, collapse long assistant responses by line or measured height while keeping active/latest streaming output expanded, provide individual show-more/show-less controls and optionally collapse-all/expand-all controls, then cover it with focused `ui/src/ui/chat/grouped-render.test.ts` and `ui/src/ui/views/chat.test.ts` cases plus English i18n strings and docs if the controls become user-facing.
