## Summary

Current main still has the reported subagent exec-event heartbeat wake path, and the same-author implementation PR #66749 remains open and unmerged with closing syntax for this issue. Keep this issue open until that PR is reviewed/merged or explicitly superseded by the broader #69492 design.

## Best Possible Solution

Keep this issue open and review #66749 as the narrow fix path, or explicitly retarget the work to #69492 if maintainers want the broader event-audience design first. The concrete fix should skip requestHeartbeatNow for isSubagentSessionKey(sessionKey) in both maybeNotifyOnExit and emitExecSystemEvent while preserving enqueueSystemEvent, with regression coverage for both paths.
