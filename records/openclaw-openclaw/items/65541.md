## Summary

Keep this PR open. Current main and the latest release still implement the documented final-only behavior for `blockStreamingDefault: "off"`, so the PR is not obsolete; however, the branch intentionally changes product semantics and intersects with the same-channel queued delivery/final-fallback gap tracked in #32868, so it needs maintainer review rather than cleanup closure.

## What This Changes

This PR changes block reply delivery so disabled block streaming direct-sends text and captioned media block payloads immediately, with tests updated to expect progressive text delivery.

## Best Possible Solution

Keep this PR open for maintainer judgment. If disabled block streaming should support progressive naturally bounded text, land a coordinated change covering `reply-delivery.ts`, disabled-streaming tests, `docs/concepts/streaming.md`, final-payload dedupe semantics, and the same-channel dispatcher await/fallback contract tracked in #32868. If `blockStreamingDefault: "off"` should remain final-only, close this PR with that product decision.
