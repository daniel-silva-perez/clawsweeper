## Summary

Keep this PR open. Current main still awaits the BlueBubbles server-info fetch before registering the webhook target, so the PR’s title change is not implemented. The PR also is not merge-ready as-is because it bundles unrelated Slack/json-file/crash-loop changes, includes a log artifact, and the Slack diff has a concrete request-body limit concern from review comments.

## Best Possible Solution

Keep this PR open for a focused rebase or replacement: move registerBlueBubblesWebhookTarget before fetchBlueBubblesServerInfo in extensions/bluebubbles/src/monitor.ts, preserve server-info caching/logging, and add a regression test proving the route is registered while server-info is still pending. Split Slack, json-file retry, and crash-loop accounting into separate reviewed PRs, and drop the committed log artifact.
