## Summary

Keep this PR open. Current main still awaits the BlueBubbles server-info fetch before registering the webhook target, so the PR’s central route-first fix is not implemented. The PR is also not merge-ready as-is: it bundles unrelated Slack/json-file/crash-loop changes, includes a committed log artifact, and the Slack change has a concrete request-body limit security concern from review comments.

## Best Possible Solution

Keep this PR open for a focused rebase or replacement: move registerBlueBubblesWebhookTarget before fetchBlueBubblesServerInfo in extensions/bluebubbles/src/monitor.ts, preserve server-info caching/logging, and add a regression test proving the route is registered while server info is still pending. Split Slack, json-file retry, and crash-loop accounting into separate reviewed PRs, and drop the committed log artifact.
