## Summary

Keep this PR open. Current main still does not implement the PR's intended startup behavior: channel startup enumerates only loaded channel plugins, while direct channel lookup can resolve bundled channel plugins that are absent from that loaded registry snapshot. The PR remains a focused implementation candidate, with a hot-path review needed around bundled plugin materialization.

## Best Possible Solution

Keep this PR open for maintainer review. The likely best path is to land or revise the focused startup fix so `startChannels()` considers configured channel ids resolved through `getChannelPlugin()`, dedupes by plugin id, preserves the gateway startup hot-path constraints, keeps the diagnostic `context.assembled` output stable, and retains QA-channel thread roundtrip coverage.
