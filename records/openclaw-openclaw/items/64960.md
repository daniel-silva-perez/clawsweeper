## Summary

Keep open. Current main still resolves compaction to one provider/model target and retries summarization with that same model; #74470 is an open closing PR for this bug, so the issue should stay open until that PR is updated, reviewed, and merged.

## What This Changes

Add compaction model fallback handling for Azure content-filter summarization failures without persisting fallback choices, with a hard escape after all compaction attempts fail.

## Best Possible Solution

Land #74470 or an equivalent focused fix that applies ephemeral compaction fallback candidates around the direct compaction orchestration path, preserves explicit compaction model semantics, updates config/schema/docs, and adds regression coverage for Azure-style content-filter failure; keep broader hard-escape behavior coordinated with #64962.
