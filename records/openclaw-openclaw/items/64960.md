## Summary

Keep open. Current main still has only one resolved compaction model target and no `agents.defaults.compaction.fallbacks` contract, so an Azure content-filter rejection during summarization can fail compaction instead of trying an operator-configured alternate compaction model.

## What This Changes

Request a compaction-specific model fallback chain that is tried ephemerally when the primary compaction model rejects or fails summarization, without persisting session model changes.

## Best Possible Solution

Add an explicit compaction fallback candidate list to config/schema/docs, resolve candidates without mutating persisted session model fields, try alternate compaction models around the direct compaction orchestration path, and add regression coverage for an Azure-style content-filter failure on the primary compaction model.
