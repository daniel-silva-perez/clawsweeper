## Summary

Keep open: current main still has the empty-message compaction skip and fallback-boundary behavior, while the discussion has fresh multi-channel repros showing repeated safeguard triggers after the #71325 mitigation. The related #69270 PR is a plausible implementation path, but it is still open and not present on current main.

## What This Changes

The issue reports that empty or unpopulated compaction message arrays cause real-content guards to skip compaction or repeatedly write fallback boundaries instead of compacting long sessions.

## Best Possible Solution

Preserve the empty-preparation safety boundary, but reconcile trigger and accounting state so fallback or zero-message compactions reset stale usage and do not keep scheduling compaction. The fix should include focused regression coverage for manual, automatic, context-engine, and preflight compaction paths.
