## Summary

Keep open. Current main still performs `devices approve` through the Gateway before local fallback and still lacks the proposed explicit `--token`/`--password` no-fallback guard, so this PR is not obsolete; it needs maintainer review because the change touches device-pairing approval semantics and must be adapted to current parser and broadcast behavior.

## What This Changes

The PR changes the devices CLI to try local loopback pairing approval before Gateway approval, skip local fallback for explicit token/password auth, and update tests for direct local approval plus unknown local request fallback.

## Best Possible Solution

Keep the PR open and route it through maintainer review: adapt the implementation to current main, preserve structured pairing-required parsing, keep local-first approval limited to implicit local loopback, fall back to Gateway approval when the local request is unknown, document accepted token/password behavior, and either mirror or deliberately waive Gateway approval side effects.
