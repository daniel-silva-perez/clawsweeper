## Summary

Keep open. Current main still lacks the proposed durable `worker-state.json` snapshot, so this PR is not obsolete, but the current branch still carries blocking doctor regressions and unrelated `openclaw doctor --json` work that should be split or revised before merge.

## What This Changes

The PR adds a task-registry observer that persists `worker-state.json` snapshots and also adds structured `openclaw doctor --json` output.

## Best Possible Solution

Keep the worker-state snapshot work as a focused PR after rebasing on current main, split or drop the doctor JSON changes, restore doctor’s live Gateway RPC health check, and document the snapshot if maintainers want `worker-state.json` to be a supported operator surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restore the live Gateway health check:** `src/commands/doctor-gateway-health.ts:27`
  - body: `checkGatewayHealth` is supposed to prove the configured Gateway is reachable and accepts the current auth. `getHealthSnapshot()` builds a local config/plugin snapshot in-process, so this can set `healthOk` true even when the Gateway is down or the token is wrong, skipping the existing failure path.
  - confidence: 0.92
- **[P2] Always emit JSON before returning in --json mode:** `src/flows/doctor-health.ts:81-82`
  - body: When the pre-doctor update flow handles the run, this early return exits before the JSON write block. Interactive `openclaw doctor --json` consumers can therefore receive no JSON payload at all.
  - confidence: 0.84
