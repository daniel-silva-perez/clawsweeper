## Summary

Keep this PR open: current main still lacks /v1/responses reasoning output, so the contribution addresses a real gap, but the proposed patch is not merge-ready because it exposes raw thinking outside the existing reasoning-visibility boundary.

## What This Changes

The PR adds raw thinking capture from Embedded Pi agent events, returns reasoning items and reasoning SSE events from /v1/responses, filters the new raw event from server chat broadcasts, and adds gateway/agent tests.

## Best Possible Solution

Ship a revised /v1/responses reasoning implementation that is explicitly gated by documented reasoning visibility or request opt-in, keeps raw thinking scoped away from generic plugin/session event consumers, preserves existing output/tool ordering, and adds regression tests for reasoning off/default, authorized opt-in, streaming, tool-call, and plugin-event exposure cases.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the current-main gap and the PR regression: current main has no response.reasoning events, while the PR emits thinking-raw before the streamReasoning visibility check and then the HTTP handler serializes it.

## Solution Assessment

No. Adding reasoning output to the OpenResponses endpoint is the right direction, but this patch bypasses the existing visibility/privacy contract; the safer solution is to gate capture and dispatch before raw thinking reaches HTTP output or generic agent-event subscribers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Gate raw reasoning on the visibility setting:** `src/agents/pi-embedded-subscribe.ts:592-596`
  - body: This emits `thinking-raw` before the `state.streamReasoning`/reasoning visibility checks, and the /v1/responses handler subscribes to that raw stream unconditionally. For agents where reasoning visibility falls back to `off`, model thinking that is currently hidden can be returned over HTTP and delivered to plugin agent-event subscriptions, so the raw capture needs an explicit authorized opt-in or a scoped non-public channel.
