## Summary

This PR is still relevant because current main does not return OpenResponses reasoning output or reasoning SSE events, but the proposed implementation should not merge as-is because it emits raw thinking before the existing reasoning visibility gate and can expose hidden reasoning through HTTP capture and plugin agent-event subscriptions.

## What This Changes

The PR adds raw thinking capture from embedded agent runs, returns `type: "reasoning"` items and `response.reasoning.*` SSE events from `/v1/responses`, filters the new raw stream from server chat broadcasts, and adds gateway/agent tests.

## Best Possible Solution

Ship a revised `/v1/responses` reasoning implementation that is explicitly gated by documented reasoning visibility or a new request opt-in, keeps raw thinking out of broad plugin/session event streams by default, preserves tool/output ordering, and adds regression coverage for reasoning off/default, authorized opt-in, streaming, tool-call, and plugin subscription exposure cases.

## Reproduction Assessment

Yes. Source inspection is enough: current main has no `response.reasoning.*` handling, while the PR diff emits `thinking-raw` before the `streamReasoning` visibility check and then serializes that raw stream on the HTTP path.

## Solution Assessment

No. Exposing reasoning on OpenResponses is the right direction, but this implementation bypasses the existing visibility/privacy contract; the safer solution gates capture before raw thinking enters HTTP output or generic agent-event subscribers.
