## Summary

Keep open. The endpoint work addresses a real OpenResponses gap and the author appears to have addressed the earlier reset, delta-shape, and ordering comments, but the latest head still has a blocking privacy/API contract issue: raw reasoning is emitted on the global agent event path before the existing reasoning visibility gate, so `/v1/responses` and plugin event subscribers can observe thinking that current main would keep hidden when reasoning visibility resolves to off.

## What This Changes

The PR adds raw thinking capture for the Gateway OpenResponses `/v1/responses` API, returning reasoning output items for non-streaming responses and `response.reasoning.*` SSE events for streaming responses.

## Best Possible Solution

Ship a revised OpenResponses reasoning implementation that explicitly gates raw reasoning capture through documented reasoning visibility or a new authorized request opt-in, keeps hidden thinking off broad agent-event/plugin subscription paths by default, and adds regression coverage for reasoning off/default, authorized opt-in, streaming, tool-call finalization, and plugin subscription exposure.

## Reproduction Assessment

Yes. Source inspection is enough: current main guards reasoning streaming/extraction on `streamReasoning`/`includeReasoning`, while the PR head emits `thinking-raw` before that gate and the HTTP/plugin event paths can consume it.

## Solution Assessment

No. Exposing reasoning on `/v1/responses` is the right direction, but this patch is not the best implementation until raw capture is gated or moved to a scoped internal channel that cannot bypass the existing visibility/privacy contract.
