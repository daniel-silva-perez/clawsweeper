## Summary

Keep open. Current main has adjacent session lifecycle persistence and gateway shutdown hooks, but I did not find the requested shutdown/restart path that enumerates active sessions and fires plugin `session_end` for each one. The related closed report about `sessions.json` status finalization is adjacent, not a canonical replacement for this plugin hook gap.

## Best Possible Solution

Implement a bounded gateway shutdown/restart finalization path in the gateway owner code. It should discover the active sessions/runs that are still current at close time, fire typed `session_end` with a documented shutdown/restart reason, update hook types/docs/tests, and preserve the existing `gateway_stop` service hook as the separate plugin-service lifecycle surface.
