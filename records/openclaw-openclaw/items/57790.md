## Summary

Keep open. Current main still drains active work and runs gateway-level shutdown hooks, but I did not find a shutdown/restart path that enumerates active sessions and fires the typed plugin `session_end` hook for each active session; the related closed `sessions.json` lifecycle-status report is adjacent, not a replacement for this plugin hook gap.

## Best Possible Solution

Implement a bounded gateway-owned shutdown/restart finalization path that identifies active sessions at close time, emits typed plugin `session_end` with a documented shutdown/restart reason, updates hook types/docs/tests, and keeps `gateway_stop` as the separate plugin-service lifecycle hook.

## Risks / Open Questions

- A shutdown implementation needs a short bounded timeout so slow plugin hooks do not block SIGTERM/SIGINT indefinitely.
- The fix should avoid double-firing `session_end` for sessions already finalized by normal lifecycle events, reset/delete, compaction, or a completed restart drain.
