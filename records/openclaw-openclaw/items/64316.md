## Summary

Keep this PR open. Current main has adjacent MCP cleanup work, including shipped idle eviction and session-rollover cleanup, but it still does not implement this PR’s exact mid-run `resetReplyRunSession` behavior. The reset helper rotates the session id and updates queued follow-up state without calling `disposeSessionMcpRuntime` or `retireSessionMcpRuntime`, and its regression tests do not cover MCP runtime disposal or disposal-failure logging. Security review found no supply-chain-sensitive file changes in the provided PR context.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is either to update/land the narrow reset-helper cleanup against current main using the existing `retireSessionMcpRuntime` seam and tests, or explicitly supersede it with a merged broader MCP lifecycle PR that covers mid-run session reset with equivalent regression coverage.
