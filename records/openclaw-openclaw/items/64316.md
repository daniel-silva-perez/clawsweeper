## Summary

Keep this PR open. Current main has several related MCP lifecycle mitigations, including idle TTL cleanup, config reload cleanup, gateway shutdown cleanup, and normal reply-session rollover cleanup, but `resetReplyRunSession` still rotates the active `sessionId` without retiring the old bundle MCP runtime. The PR’s two-file diff targets that remaining reset-path gap and does not introduce security-sensitive supply-chain or CI changes.

## Best Possible Solution

Keep this PR open for maintainer review and either land it or rework it slightly to use the current shared MCP retirement helper. The desired fix belongs in `resetReplyRunSession`: after the replacement session is established, retire the previous session id’s bundle MCP runtime without blocking the retry path, log failures, and keep tests for disposal success, disposal failure, and transcript cleanup continuity. A changelog entry is optional but reasonable if maintainers want the reset-path leak called out separately from the already-shipped broader MCP cleanup notes.
