## Summary

Keep open. Current main still hardcodes the OpenAI API base URL in the OpenAI dynamic GPT fallback registry, and the shared resolver then uses that discovered model base URL whenever no provider config override exists. The PR is a focused implementation candidate with no obvious unrelated security-sensitive or supply-chain changes, but it still needs normal maintainer review and validation before merge.

## What This Changes

The PR adds an OpenAI default base URL helper that reads `OPENAI_BASE_URL`, replaces the OpenAI dynamic GPT fallback registry's hardcoded base URL literals with that helper, and adds unit coverage for the helper behavior.

## Best Possible Solution

Land a focused OpenAI-provider fix that makes dynamic GPT defaults honor `OPENAI_BASE_URL` only as the no-config fallback, keeps explicit `models.providers.openai.baseUrl` precedence unchanged, and includes resolver-level regression coverage for that precedence.
