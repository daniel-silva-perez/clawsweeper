## Summary

Keep open. Current main still hardcodes the OpenAI API base URL in the dynamic GPT fallback registry, and the shared resolver uses that discovered base URL whenever no provider config override exists. The PR is a focused implementation for a real gap and does not appear to introduce unrelated security-sensitive or supply-chain changes.

## What This Changes

The PR adds an OpenAI default base URL resolver that reads `OPENAI_BASE_URL`, uses it in the OpenAI GPT forward-compat dynamic registry, and adds unit coverage for default/env handling.

## Best Possible Solution

Land a focused OpenAI-provider fix that makes dynamic GPT defaults honor `OPENAI_BASE_URL` only as the no-config fallback, preserves explicit `models.providers.openai.baseUrl` precedence, and ideally includes resolver-level coverage for both env fallback and config override.
