Keep this issue open. Current main still has the fixed-width coarse UTF-16 split in `chunkMarkdown`, still passes embedding text and structured embedding inputs to providers without an unpaired-surrogate sanitizer, and the issue is paired with open PR #65783 using closing syntax. After that PR or replacement lands, this issue can close as implemented.
