## Summary

Keep open. Current main still has the reported coarse chunking gap and no embed-boundary sanitizer, and an open linked PR (#65783) explicitly uses closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports a permanent memory indexing stall when `chunkMarkdown` splits an emoji or supplementary-plane UTF-16 surrogate pair and the resulting lone surrogate reaches the embeddings provider.

## Best Possible Solution

Review and land #65783, or a narrow equivalent replacement, with both the outer `chunkMarkdown` surrogate-boundary fix and embed-boundary lone-surrogate sanitization plus focused regression tests in the memory host SDK and memory-core embedding path.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: with `tokens=10`, `maxChars=40`, a mostly-Latin line containing an emoji at the 40th UTF-16 code unit is split between high and low surrogates by the current outer loop.

## Solution Assessment

Yes. The paired PR direction is the narrowest maintainable fix because it addresses the producer bug in the coarse chunker and adds defense in depth at the embeddings boundary; the issue should not close until that fix or an equivalent lands.
