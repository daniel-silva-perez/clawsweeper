## Summary

Keep open. Current main still has the reported fixed-width coarse UTF-16 chunk split and still forwards embedding inputs without a lone-surrogate sanitizer. The issue is also paired with open PR #65783, which is the active implementation candidate linked back to this issue with GitHub closing syntax.

## Best Possible Solution

Keep this issue open while maintainers review PR #65783. The best fix is a focused memory change that prevents the coarse `chunkMarkdown` split from bisecting UTF-16 surrogate pairs, sanitizes lone surrogates at the embedding boundary, and adds regression coverage for the mostly-ASCII coarse-boundary emoji case plus sanitizer behavior.
