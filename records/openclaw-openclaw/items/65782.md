## Summary

Keep this issue open. Current main still has the unsafe fixed-width coarse UTF-16 split in `chunkMarkdown`, still sends embedding text/structured inputs to providers without an unpaired-surrogate sanitizer, and the issue is paired with open PR #65783 using closing syntax, so the right cleanup path is review/land or close that PR rather than closing this issue first.

## What This Changes

Request fixing memory indexing stalls by preserving surrogate pairs in `chunkMarkdown`'s outer split and sanitizing unpaired surrogates before embedding provider calls.

## Best Possible Solution

Keep this issue open while maintainers review, rebase, and land PR #65783 or an equivalent focused fix. The fix should update the outer `chunkMarkdown` coarse split so it never bisects UTF-16 surrogate pairs, sanitize unpaired surrogates at the embedding boundary for both text and structured inputs, and add regression tests for the mostly-ASCII coarse-boundary emoji case plus sanitizer behavior.
