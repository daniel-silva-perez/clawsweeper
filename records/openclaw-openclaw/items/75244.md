## Summary

Keep open. Current main still rejects plugin tools whose returned object lacks a non-array object at `parameters`, and the repository docs/tests intentionally treat `inputSchema`-only tools as malformed. The reported external plugin failure is not proved fixed, but the TypeBox-specific theory remains unconfirmed because the inspected TypeBox package source returns ordinary object schemas that should pass the current validator.

## What This Changes

The issue reports that an external memory plugin's `memory_search` and `memory_get` tools are dropped every turn because their TypeBox schemas are reported as missing `parameters`.

## Best Possible Solution

Keep the report open until maintainers can capture the failing plugin's actual returned tool object or decide on a targeted contract change. The desired end state is either a plugin-side correction to return the documented `parameters` object, or a deliberate OpenClaw change that improves diagnostics and documents any new schema alias behavior with focused contract tests.

## Reproduction Assessment

Unclear. The issue provides concrete external steps, but the external plugin value is not present here, and both @sinclair/typebox source and bundled memory-core patterns suggest a real `Type.Object()` schema should pass the current `isRecord` check.

## Solution Assessment

Unclear. Better diagnostic detail would be a narrow safe first step, but accepting `inputSchema` as a native `registerTool` alias is a public plugin-contract change rather than an obvious bug fix.
