## Summary

Keep this PR open. Current main still lacks the requested backup exclude-pattern surface: no `--exclude`, `--exclude-file`, `--smart-exclude`, `.backupignore`, protected-path guard, or `excludedStats` manifest support. A later main change only skips `node_modules` under installed plugin directories, which is useful but does not supersede the broader feature or the still-open backup exclude and large-backup reports.

## Best Possible Solution

Keep this PR open as the active core implementation candidate. The best path is maintainer review of the current branch, explicit resolution or tracking for external-asset filtering semantics, validation of protected-path guards and `excludedStats` compatibility, and then either landing a coherent backup exclude implementation or splitting a smaller PR that still solves the central user problem.
