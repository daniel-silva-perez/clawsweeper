## Summary

Keep this PR open. Current main still does not implement the requested backup exclude-pattern surface, and the PR remains a plausible active implementation candidate tied to open backup exclude/large-backup reports. Main only skips installed plugin `node_modules` under the state `extensions/` tree, which is narrower than `--exclude`, `--exclude-file`, `--smart-exclude`, `.backupignore`, protected-path guards, and `excludedStats` manifest support.

## Best Possible Solution

Keep this PR open as the active core implementation candidate. The best path is maintainer review of the branch, explicit resolution or tracking for external-asset filtering semantics, validation of protected-path guards and `excludedStats` compatibility, and then either landing a coherent backup exclude implementation or asking for a smaller split that still solves the central user problem.
