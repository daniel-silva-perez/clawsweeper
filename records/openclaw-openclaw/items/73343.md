## Summary

Keep this issue open. Current main still has no sender/channel identity path into `before_tool_call` or shell execution environments; the only adjacent implementation is native plugin-tool context, which does not cover exec, skill scripts, MCP tools, or hook-provided env injection.

## What This Changes

Expose channel sender identity, channel/account context, or plugin-provided identity env vars to skill/tool shell execution so downstream APIs can attribute the triggering user.

## Best Possible Solution

Add an additive, opt-in plugin-SDK/core seam that carries normalized sender, channel, account, and conversation identity into tool-hook context, then provides a typed exec-env contribution path that flows through the existing gateway/node/sandbox env validation. Document the security model and add focused tests for `before_tool_call`, exec env merging, native plugin tools, and skill/script execution.
