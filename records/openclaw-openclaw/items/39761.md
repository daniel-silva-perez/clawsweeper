## Summary

Keep this PR open. Current main still exposes `message_sending` as a delivery-level hook only: no `agentContext`, `agentMetadata`, tool-call count/list, token usage, context-fill, provider/model, or agent identity is passed to the hook event. The embedded runner now has relevant metadata such as `agentMeta`, `toolSummary`, and `completion`, but that data is not threaded into outbound delivery hooks. The PR remains a plausible backward-compatible plugin API addition, with prior review feedback addressed in discussion; it needs maintainer review and adaptation to the current `hook-message.types.ts` surface rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or adaptation. The best path is a backward-compatible typed `agentContext` or equivalent field on `PluginHookMessageSendingEvent` in `src/plugins/hook-message.types.ts`, plumbed through `deliverOutboundPayloads()` where agent-run metadata is available, aligned with current `EmbeddedPiRunMeta` shapes (`agentMeta`, `toolSummary`, `completion`), covered by focused hook contract tests, and documented at `docs/plugins/hooks.md` / https://docs.openclaw.ai/plugins/hooks with explicit retry and non-CLI delivery semantics. Coordinate model identity with #69110 and stop-reason metadata with #66235 instead of closing this PR as obsolete.
