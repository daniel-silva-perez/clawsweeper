## Summary

Keep open. Current main still has the reported Telegram regression: plain inbound messages under non-off `replyToMode` populate the native quote candidate map, and the dispatcher still treats any populated map as a reason to disable answer draft preview streaming. Closed PR #73534 remains useful source work, while open PR #39905 is broader conversation-scoped configuration work rather than a landed fix for this specific bug.

## What This Changes

The issue requests that Telegram `streaming.mode: "partial"` continue to show/edit a preview when `replyToMode` is non-off for plain replies, disabling preview streaming only for genuine Telegram selected-quote replies.

## Best Possible Solution

Land a narrow Telegram dispatcher fix that scopes the preview-stream disable guard to real selected quotes, preserves `replyQuoteByMessageId` for final delivery fallback, updates the current-message candidate test expectation, and adds direct regression coverage for `replyToMode: "first"` plus `streaming.mode: "partial"` on a plain Telegram message. Keep broader per-conversation streaming settings with #39905.

## Reproduction Assessment

Yes. The current source path reproduces with high confidence: `replyToMode: "first"`, `streamMode: "partial"`, and a plain message with `MessageSid` or `message_id` populate `replyQuoteByMessageId`, make `hasNativeQuoteReply` true, and prevent `createTelegramDraftStream` from being created.

## Solution Assessment

Yes. The narrowest maintainable fix is to disable preview streaming only when a genuine selected Telegram quote exists, such as `replyQuoteText != null && replyQuoteMessageId != null`, while leaving the native quote candidate map available for final delivery.
