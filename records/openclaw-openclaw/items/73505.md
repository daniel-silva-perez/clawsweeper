## Summary

Keep open. Current main still has the reported Telegram delivery regression: the dispatcher still disables the answer draft stream whenever any native quote candidate exists, and plain inbound messages still create such a candidate under non-off reply threading. The closed focused PR #73534 remains useful source work but was not merged; open #39905 is adjacent broader configuration work, not a landed fix for this bug. https://github.com/openclaw/openclaw/issues/73505

## What This Changes

The issue requests that Telegram `streaming.mode: "partial"` keep working for plain replies when `channels.telegram.replyToMode` is non-off, while still avoiding draft streaming for real selected Telegram quotes.

## Best Possible Solution

Land a narrow Telegram dispatcher fix that scopes the draft-stream disable guard to real selected quotes, preserves native quote candidates for final delivery, updates the current-message candidate expectation, and adds direct regression coverage for `replyToMode: "first"` plus `streaming.mode: "partial"` on a plain Telegram message.

## Repair Work Prompt

Fix the Telegram regression from https://github.com/openclaw/openclaw/issues/73505. Current `extensions/telegram/src/bot-message-dispatch.ts` disables answer draft preview streaming whenever `replyQuoteByMessageId` is non-empty; plain inbound messages populate that map when `replyToMode` is non-off, so `streaming.mode: "partial"` produces no preview. Recreate the narrow source intent from closed PR #73534: only disable draft streaming when there is a real selected Telegram quote (`replyQuoteText != null` and `replyQuoteMessageId != null`), while preserving `replyQuoteByMessageId` for final delivery fallback. Update `extensions/telegram/src/bot-message-dispatch.test.ts` so the current-message native quote candidate test still checks the candidate map but expects the draft stream to be created, and add an explicit regression test for `replyToMode: "first"` plus `streamMode`/`streaming.mode: "partial"` on a plain Telegram message. Do not implement the broader per-conversation streaming/reply configuration from #39905. Add a single-line user-facing changelog fix entry under the active Fixes section, crediting @choury for the report and @hclsys if reusing the closed PR source work; do not credit bots.
