## Summary

Keep open. Current main still disables Telegram answer draft streaming whenever replyToMode is non-off and a native quote candidate exists; normal inbound text creates such a candidate even when the user did not select a Telegram quote. The same guard is present in the v2026.4.26 release snapshot named in the report, and existing tests do not cover the plain replyToMode + partial streaming case.

## Best Possible Solution

Fix `extensions/telegram/src/bot-message-dispatch.ts` so `canStreamAnswerDraft` is blocked only for genuine selected/native quote replies, or extend `createTelegramDraftStream` to carry native quote parameters on the first preview send. Add a focused regression test for `replyToMode: "first"` plus `streaming.mode: "partial"` on an ordinary text message, and keep the selected-quote fallback behavior covered.
