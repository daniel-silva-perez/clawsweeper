## Summary

Keep open. Current main still has the reported Telegram regression: plain inbound messages under non-off `replyToMode` can populate the native quote candidate map, and the answer draft preview guard still treats any populated candidate map as a reason to disable streaming. The closed focused PR #73534 remains useful source work but was not merged; #39905 is adjacent broader Telegram work, not a merged fix for this bug.

## What This Changes

The issue asks for Telegram `streaming.mode: "partial"` to keep preview streaming for plain replies when `replyToMode` is non-off, while still disabling previews for genuine selected native quote replies.

## Best Possible Solution

Land a narrow Telegram dispatcher fix that scopes the streaming-disable guard to real selected quotes, updates the current-message candidate test, adds a direct regression test for `replyToMode: "first"` plus `streaming.mode: "partial"`, and leaves broader per-conversation streaming configuration to #39905.
