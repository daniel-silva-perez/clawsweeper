## Summary

Keep open. Current main still has the reported Telegram regression: ordinary plain replies under non-off `replyToMode` can populate the native quote candidate map, and the dispatcher still treats any populated candidate map as a reason to disable answer draft preview streaming. Closed PR #73534 remains useful source work but was not merged, while open PR #39905 is broader scoped-configuration work rather than a landed fix for this bug.

## What This Changes

Telegram `streaming.mode: "partial"` should keep showing live preview edits when `channels.telegram.replyToMode` is `first` or another non-`off` value, except for genuine selected quote replies.

## Best Possible Solution

Land a narrow Telegram dispatcher fix that scopes the preview-stream disable guard to real selected quotes, preserves `replyQuoteByMessageId` for final delivery fallback, updates the current-message candidate expectation, and adds direct regression coverage for `replyToMode: "first"` plus `streaming.mode: "partial"` on a plain Telegram message. Broader per-conversation streaming settings should stay with #39905.

## Reproduction Assessment

Yes. The current source path is enough to reproduce with high confidence: `replyToMode: "first"`, `streamMode: "partial"`, and a plain text message with `MessageSid`/`message_id` populate `replyQuoteByMessageId`, make `hasNativeQuoteReply` true, and prevent `createTelegramDraftStream` from being created.

## Solution Assessment

Yes. The narrowest maintainable fix is to disable preview streaming only when a genuine selected Telegram quote exists, such as `replyQuoteText != null && replyQuoteMessageId != null`, while leaving the native quote candidate map available for final delivery.
