## Summary

Keep open. Current main still has the reported Telegram regression: with `replyToMode` non-off, ordinary inbound messages can populate `replyQuoteByMessageId`, and the dispatcher still treats any populated native-quote candidate map as a reason to disable partial answer draft streaming. The latest release tag also contains the same guard, while closed PR #73534 is useful unmerged source work and open PR #39905 is broader scoped-configuration work rather than a landed fix for this bug.

## What This Changes

The request is to keep Telegram `streaming.mode: "partial"` active for plain threaded replies when `channels.telegram.replyToMode` is non-off, disabling preview streaming only for real selected Telegram quote replies.

## Best Possible Solution

Land a narrow Telegram dispatcher fix that scopes the preview-stream disable guard to actual selected quotes, keeps `replyQuoteByMessageId` available for final native-quote delivery fallback, updates the current-message candidate test expectation, adds direct regression coverage for `replyToMode: "first"` plus `streaming.mode: "partial"` on a plain Telegram message, and records the user-visible fix in the changelog.

## Reproduction Assessment

Yes. The source path is enough to reproduce with high confidence: `replyToMode: "first"`, `streamMode: "partial"`, and a plain message with text and `MessageSid` populate `replyQuoteByMessageId`, making `hasNativeQuoteReply` true and preventing answer draft stream creation.

## Solution Assessment

Yes. The narrowest maintainable fix is to disable preview streaming only when a genuine selected Telegram quote exists, such as `replyQuoteText != null && replyQuoteMessageId != null`, while leaving native quote candidates intact for final sends.
