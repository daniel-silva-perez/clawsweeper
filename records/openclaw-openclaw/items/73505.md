## Summary

Keep open. Current main still has the reported Telegram regression: plain text messages with non-off `replyToMode` populate the native quote candidate map, and the dispatcher still treats any populated map as a reason to disable answer draft preview streaming. Closed PR #73534 remains useful source work; open PR #39905 is broader per-conversation configuration work, not a landed fix for this specific bug.

## What This Changes

Telegram partial preview streaming should continue to work with non-off `replyToMode` for plain replies, disabling preview streaming only for genuine selected Telegram quote replies.

## Best Possible Solution

Land a narrow Telegram dispatcher fix that scopes the answer-preview disable guard to actual selected quote metadata, preserves final-delivery quote candidates, flips the current broken test expectation, and adds a direct regression test for `replyToMode: "first"` plus `streaming.mode: "partial"` on a plain message.

## Reproduction Assessment

Yes. The source-level path is high confidence: `replyToMode: "first"`, `streaming.mode: "partial"`, and a plain non-empty Telegram message populate `replyQuoteByMessageId`, make `hasNativeQuoteReply` true, and prevent `createTelegramDraftStream` from being created.

## Solution Assessment

Yes. The narrowest maintainable solution is to base the preview-stream skip on genuine selected quote state, such as `replyQuoteText != null && replyQuoteMessageId != null`, rather than on the final-delivery candidate map; broader per-conversation settings in #39905 are separate work.
