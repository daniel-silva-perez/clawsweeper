## Summary

Keep open. Current main still disables Telegram answer draft streaming whenever the native quote candidate map is populated, and ordinary plain replies populate that map when `replyToMode` is non-`off`. Closed PR #73534 remains useful source work, but it was not merged; open PR #39905 is adjacent broader configuration work rather than a shipped fix for this regression.

## What This Changes

Fix Telegram preview streaming so `streaming.mode: "partial"` still works for plain replies when `channels.telegram.replyToMode` is non-`off`, disabling it only for genuine selected native quote replies.

## Best Possible Solution

Land a narrow Telegram plugin fix that scopes the preview-stream disable guard to actual selected quote replies, preserves `replyQuoteByMessageId` for final delivery fallback, and adds direct regression coverage for `replyToMode: "first"` plus `streaming.mode: "partial"` on a plain Telegram message.

## Reproduction Assessment

Yes. The current source path is enough to reproduce with high confidence: `replyToMode: "first"`, `streamMode: "partial"`, and a plain inbound message with `MessageSid` or `message_id` populate `replyQuoteByMessageId`, make `hasNativeQuoteReply` true, and prevent `createTelegramDraftStream` from being created.

## Solution Assessment

Yes. The narrowest maintainable fix is to disable preview streaming only when a genuine selected Telegram quote exists, such as `replyQuoteText != null && replyQuoteMessageId != null`, while leaving the native quote candidate map available for final sends.
