## Summary

Keep this PR open. Current main still drops Discord upload MIME types in standard sends, component sends, and native interaction replies, and WebP still flows through the shared image optimizer. The PR is a plausible implementation candidate for the linked bug, but it should not be auto-closed or merged as-is because its shared WebP bypass changes WhatsApp/core media behavior and has unresolved validation/privacy review concerns.

## Best Possible Solution

Keep this PR open for maintainer review or supersede it only after a canonical fix is chosen. The best implementation is to preserve `media.contentType` in all Discord Blob upload paths, including standard sends, component sends, and native interaction replies, then handle WebP preservation deliberately: make it Discord-specific or add a shared media option with verified WebP detection, pixel-limit enforcement, metadata policy, and regression tests. If shared behavior changes intentionally, update WhatsApp tests/docs; otherwise keep WhatsApp recompression intact and consolidate the overlapping work with #41265 and #41419.

## Risks / Open Questions
