## Summary

Keep open: current main has useful mitigations, including docs/source consultation guidance, `openclaw status` guidance, and a status line that exposes the OpenClaw version, but it does not implement the requested version-aware self-knowledge system, top-level `openclaw explain <topic>`, agent-readable change notes, or known-behavior registry.

## What This Changes

The issue requests version-aware agent diagnostics so OpenClaw can distinguish intentional post-upgrade behavior changes from real bugs.

## Best Possible Solution

Define a small core-owned version-awareness path for diagnostics: expose the current OpenClaw version and concise current-release behavior notes to the agent, make diagnostics consult that source before declaring post-upgrade behavior a bug, and keep any prompt injection cache-conscious with tests around system-prompt content and command availability.
