## Summary

Keep open. Current main gives the agent docs/source/status guidance and config-schema lookup, and the user-facing docs point humans to the changelog, but I did not find a shipped default mechanism for version-aware self-knowledge: no agent-readable “what changed in this version” prompt block, no `openclaw explain <topic>` command, and no known-behavior registry for upgrade diagnostics.

## Best Possible Solution

Keep this open for a core product/design decision. A strong implementation would add a small authoritative version-aware path: expose the current OpenClaw version to agent diagnostics, provide an `openclaw explain <topic>` or docs-backed known-behavior lookup for changed defaults/schema/session/UI behavior, and update the prompt/tool guidance so upgrade-related bug diagnosis checks that source before recommending rollbacks or repairs.
