## Summary

Keep open. Current main contains several partial defenses for this empty-input cluster, but it still lacks the requested last-chance guard for blank visible user prompts when replay history exists, and an open PR already references this issue as its intended fix.

## What This Changes

Prevent Telegram or related channel blank user prompts from being persisted and submitted to LLM providers so raw provider empty-input errors are not emitted back into chat.

## Best Possible Solution

Land PR #74168 or an equivalent narrow embedded-runner guard that drops blank non-runtime visible user prompts before provider submission, while keeping media-only and runtime-only flows valid and leaving session repair as a fallback safety net.
