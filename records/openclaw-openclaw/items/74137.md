## Summary

Keep open. Current main has several defenses for this empty-input cluster, but the shared embedded-runner guard requested by the report is still missing: blank visible prompts are still considered submit-worthy when replay history exists. An open PR already references this issue with closing syntax, so maintainers should review, land, or close that PR before resolving the issue.

## What This Changes

The issue asks OpenClaw to prevent blank Telegram-originated user turns from being persisted or submitted to provider APIs, especially around new-session/reset flows.

## Best Possible Solution

Ship a narrow pre-provider embedded-runner guard, likely by landing #74168 or an equivalent patch, that skips blank non-runtime user-visible prompts before provider submission while preserving media-only turns and runtime-only internal context events. Keep replay/session repair as a safety net and cover Telegram reset plus helper behavior in tests.
