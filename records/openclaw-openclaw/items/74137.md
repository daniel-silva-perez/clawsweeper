## Summary

Keep open. Current main has useful defenses for this empty-input cluster, including Telegram empty-body filtering, a bare reset transcript marker, replay repair, and provider-side OpenAI Codex validation, but it still lacks the shared last-chance guard requested by the report; an open PR already references this issue with closing syntax and should be reviewed or closed before this issue is resolved.

## What This Changes

Telegram and related channel runs can submit blank user turns during session reset or new-session paths, surfacing provider empty-input errors before repair removes the persisted blank message.

## Best Possible Solution

Ship a narrow pre-provider embedded-runner guard, likely by landing #74168 or an equivalent patch, that drops or skips blank non-runtime user-visible prompts before persistence/provider submission while preserving media-only turns and internal runtime-only events. Keep session-file repair as a safety net and cover Telegram DM/group reset plus embedded-runner helper behavior in tests.
