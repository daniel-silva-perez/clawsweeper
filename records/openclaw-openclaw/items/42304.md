## Summary

Keep this PR open. Current main still has no Android Adaptive Card parser or Compose renderer; Android chat renders text through markdown and non-text content as base64 media only. The PR is also paired with the same author's still-open iOS/macOS companion #42350, and the proposed renderer still needs product/security review for model/plugin-supplied card JSON before it can be merged or closed as out-of-scope.

## Best Possible Solution

Keep this PR open for maintainer product and security review. If native Android rich cards are desired, rework it around the current MessagePresentation direction or a documented Android client-renderer extension path, preserve ordered mixed text/card content, support multiple cards and template data, add defensive render limits plus URL/action policy, and add focused Android parser/render tests. If maintainers decide this belongs outside core, document the missing Android client-renderer API needed by the external openclaw-adaptive-cards plugin and close this together with the companion client PRs.
