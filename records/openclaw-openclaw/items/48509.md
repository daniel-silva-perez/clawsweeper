## Summary

Keep this issue open. Current main still does not expose `before_persistence_write` or an equivalent durable-write classification/veto point; plugins can only approximate it through generic tool-call hooks, transcript/message persistence hooks, and the observer-only cron hook.

## Best Possible Solution

Keep the issue open for focused core/plugin-SDK design. The best path is an additive durable persistence-write hook or explicit durable-write context on `before_tool_call`, with stable target/session/path/provenance metadata, blocking semantics at minimum, and focused docs/tests across memory, file, cron, config, and preference writes. Coordinate with the related open security/provenance work in #48503, #48517, and #48519.

## Risks / Open Questions
