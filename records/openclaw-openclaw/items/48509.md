## Summary

Current main still does not implement `before_persistence_write` or an equivalent durable-write context/veto. The plugin hook surface supports generic tool-call blocking, transcript/message persistence hooks, and an observer-only `cron_changed` hook, but durable memory, file, cron, and config writes still lack a stable pre-write target/provenance event.

## Best Possible Solution

Keep this issue open for focused core/plugin-SDK design. The best path is an additive durable persistence-write hook or explicit durable-write context on `before_tool_call`, with stable target/session/path/provenance metadata, blocking semantics at minimum, and targeted SDK/docs/tests covering memory, file, cron, config, and preference writes. Coordinate the design with #48503, #48517, and #48519.
