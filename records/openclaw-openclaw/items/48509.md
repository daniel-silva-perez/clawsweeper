## Summary

Keep open. Current main still lacks the requested `before_persistence_write` hook or an equivalent durable-write context; existing plugin hooks cover tool calls, transcript result persistence, and message writes, while cron/config/file/memory writes still commit through their normal paths without a dedicated durable-state veto/audit event.

## Best Possible Solution

Keep this issue open for focused core/plugin-SDK design. The best path is an additive durable-write hook or explicit `before_tool_call` persistence context that fires before memory, file, cron, config, and preference commits, supports blocking at minimum, carries stable target/session/provenance metadata, and lands with SDK types, docs, and targeted tests coordinated with #48503 and #48519.
