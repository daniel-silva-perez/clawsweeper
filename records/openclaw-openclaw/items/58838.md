## Summary

Keep this issue open. Current main still retries built-in compaction summarization per chunk and then continues staged compaction after fallback summaries, so persistent non-timeout model failures can still spend repeated retry budgets across split chunks and the merge step.

## What This Changes

Add a bounded failure guard for built-in compaction summarization so an unavailable summarizer model stops later chunk attempts and falls back instead of repeatedly retrying across staged compaction.

## Best Possible Solution

Implement a built-in compaction failure budget that stops later split and merge LLM attempts after persistent non-timeout summarizer failures, returns the previous summary or existing structured fallback, and covers multi-split staged compaction with focused unit tests.

## Reproduction Assessment

Yes. A deterministic repro can mock generateSummary() to reject with a non-timeout error and call summarizeInStages() with enough messages and parts to split; current code will spend the retry budget for each split and then for the merge instead of opening a guard.

## Solution Assessment

Yes, a scoped failure guard in the built-in compaction path is the narrow maintainable fix. A broad global circuit breaker would be riskier unless it is explicitly keyed and cooled down per compaction target.
