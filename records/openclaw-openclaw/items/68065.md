## Summary

Keep this issue open. Current main still returns a hard `status: "timeout"` from `sessions_send` as soon as the nonzero-timeout `agent.wait` call times out, without post-timeout reply or accepted-delivery compensation. The same reporter has open implementation PR #68196 with explicit closing syntax, so the issue should remain open until that PR is reviewed, landed, or superseded.

## What This Changes

The issue requests that `sessions_send` distinguish wait-window expiry from delivery failure by returning a late reply or accepted delivery when the target agent run was accepted.

## Best Possible Solution

Implement the fix in the `sessions_send` and `run-wait` boundary: after a successful `agent` RPC start, handle wait-window expiry by checking for a new assistant reply since the pre-wait baseline, returning `ok` for a late reply, returning `accepted` only with a real acceptance signal, and preserving hard timeout when delivery cannot be established.
