## Summary

Keep open. Current main still lacks the explicit cross-file instruction precedence rule, and the PR is already the narrow implementation, but the proposed hierarchy is a maintainer policy decision because it makes an active SKILL.md outrank workspace instruction files while the docs still treat third-party skills as untrusted. The Greptile test-coverage note was addressed by the PR's second commit, so the remaining blocker is policy and trust-boundary review rather than a cleanup close.

## What This Changes

This PR adds a one-line skills instruction precedence directive to `buildSkillsSection` and asserts the full directive in the system prompt test.

## Best Possible Solution

Maintain a clear, maintainer-approved conflict-resolution rule for skills in the system prompt and regression tests; if active skills should not outrank workspace policy, adjust the rule before landing so the prompt matches the intended trust model.
