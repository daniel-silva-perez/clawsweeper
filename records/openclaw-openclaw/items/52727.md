## Summary

Keep open. Current main still lacks the requested instruction-file precedence directive, and the PR is the narrow implementation, but the proposed ordering is a trust-boundary change because it ranks active SKILL.md instructions above workspace policy files while skills docs treat third-party skills as untrusted.

## What This Changes

The PR adds a one-line SKILL.md > AGENTS.md > TOOLS.md > SOUL.md precedence directive to the Skills system-prompt section and asserts it in the system-prompt test.

## Best Possible Solution

Land a maintainer-approved conflict-resolution rule in the Skills prompt section, with the test and docs aligned; if workspace policy should remain above untrusted skills, revise the hierarchy before merge so the prompt matches OpenClaw's intended trust model.

## Review Findings
