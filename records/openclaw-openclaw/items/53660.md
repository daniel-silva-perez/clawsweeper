## Summary

Keep open. Current main still does not implement parsed `SessionFileEntry` reuse for append-only session transcript sync: the active parser full-reads and reparses selected JSONL files, and the active sync path calls it without a previous-entry cache. The PR remains potentially useful, but it targets removed `src/memory` paths and has unresolved correctness and transcript-retention review risks.

## What This Changes

The PR adds a previous-entry option to `buildSessionEntry`, caches parsed session entries during memory sync, and adds an append-only reuse regression test in the old `src/memory` implementation.

## Best Possible Solution

Keep this PR open for maintainer or contributor follow-up. The best path is a rebase or narrow replacement against `packages/memory-host-sdk/src/host/session-files.ts` and `extensions/memory-core/src/memory/manager-sync-ops.ts`, using byte-safe and prefix-validated append detection, bounded cache behavior with reset/prune cleanup, and regression coverage for ASCII append, non-ASCII append, prefix rewrite fallback, cache cleanup, hash-equal behavior, and preservation of current parser metadata fields.

## Risks / Open Questions

- The PR branch targets removed `src/memory` files, so it needs a rebase or replacement against the current `packages/memory-host-sdk` and `extensions/memory-core` layout before it can be evaluated for merge.
- The proposed append-only fast path must validate the unchanged byte prefix before reusing cached content; otherwise rewritten or compacted transcripts can leave stale indexed messages.
- The proposed byte offset must not be used as a JavaScript string index; non-ASCII transcripts can otherwise drop or duplicate parsed records.
- A cache of full `SessionFileEntry` objects retains transcript content in a long-lived process and needs bounds, reset/prune cleanup, and careful privacy/memory treatment.
- Any rebased implementation must preserve current-main fields and filtering behavior, including `messageTimestampsMs`, `generatedByDreamingNarrative`, `generatedByCronRun`, line wrapping, internal-runtime filtering, and session-store classification.
