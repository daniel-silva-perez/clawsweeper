## Summary

Keep open. Current main still full-parses session transcripts and has no parsed-entry cache, so the intended performance improvement is not implemented. The PR is not mergeable as-is because it targets removed `src/memory/*` paths while active code now lives under `packages/memory-host-sdk` and `extensions/memory-core`, and existing review comments identify correctness and memory-retention blockers.

## What This Changes

The PR branch adds a previous-entry append-only fast path to `buildSessionEntry`, caches `SessionFileEntry` objects during memory sync, and adds an append-only reuse regression test under the old `src/memory` implementation.

## Best Possible Solution

Replace or rebase the optimization against the active memory-host and memory-core files, with byte-level prefix validation, safe fallback to the existing full parser, bounded cache lifecycle, and regression coverage for ASCII append, non-ASCII append, rewritten-prefix fallback, line maps, timestamps, redaction, and transcript classification.
