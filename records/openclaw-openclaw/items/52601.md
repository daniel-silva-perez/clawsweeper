## Summary

Keep open for maintainer review. Current main still has the FreeBSD pinned-helper identity mismatch path, and this PR is a small but security-sensitive workaround that extends legacy write semantics to FreeBSD rather than proving a normalized pinned-helper fix.

## What This Changes

The PR changes `writeFileWithinRoot` and `copyFileWithinRoot` so FreeBSD uses the existing legacy filesystem fallback path instead of the Python pinned-write helper.

## Best Possible Solution

Land a tested FreeBSD-compatible filesystem write/copy fix that avoids the Python-vs-Node device identity mismatch while preserving the safety guarantees maintainers want for new destination creation, hardlink rejection, symlink boundaries, and post-write verification.
