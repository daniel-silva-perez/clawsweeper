## Summary

Keep this PR open. Current main still routes FreeBSD write/copy operations through the Python pinned-write helper, and current file identity comparison only has a Windows zero-device exception, so the PR's FreeBSD/ZFS compatibility fix is not implemented on main.

## What This Changes

The PR changes `writeFileWithinRoot` and `copyFileWithinRoot` in `src/infra/fs-safe.ts` to send FreeBSD through the existing legacy fallback path currently used for Windows.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is either to land an amended narrow FreeBSD fallback with explicit route-selection tests and an accepted legacy-write tradeoff, or supersede it with pinned-write identity normalization that fixes FreeBSD/ZFS while preserving the pinned helper's all-or-nothing target creation behavior.
