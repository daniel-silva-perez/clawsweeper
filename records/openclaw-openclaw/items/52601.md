## Summary

Keep open: current main still has the reported FreeBSD failure mechanism and no merged alternative, while the proposed workaround touches hardened filesystem write semantics that need maintainer review before merge.

## What This Changes

The PR changes the FreeBSD route for `writeFileWithinRoot` and `copyFileWithinRoot` in `src/infra/fs-safe.ts` to use the existing legacy fallback path instead of the pinned Python-helper path.

## Best Possible Solution

The desired end state is a tested FreeBSD-compatible write/copy implementation that avoids the Python-vs-Node device identity mismatch while preserving the filesystem safety guarantees maintainers want for new destination creation and post-write verification.
