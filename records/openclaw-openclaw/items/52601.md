## Summary

Keep this PR open. Current main still routes FreeBSD write/copy operations through the Python pinned-write helper and still has strict non-Windows file identity comparison, so the PR's intended FreeBSD/ZFS compatibility fix is not implemented on main. The PR is narrow and plausibly valid, but it touches hardened filesystem write semantics and already has a review thread about extending the legacy zero-byte target window to FreeBSD, so it needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is either to land an amended narrow FreeBSD fallback with explicit route-selection tests and an accepted legacy-write tradeoff, or supersede it with a pinned-write identity normalization that fixes FreeBSD/ZFS while preserving the pinned helper's all-or-nothing target creation behavior.
