## Summary

Keep this PR open. Current main still has the Unix/macOS stale-gateway PID detection bug this PR targets, and the linked bug remains open, but the PR head should not merge as-is because both new polling-path argv verifications use the full default spawn timeout instead of the existing short poll timeout.

## What This Changes

The PR changes Unix stale-gateway cleanup to parse raw lsof listener PIDs, verify candidate gateways with ps argv inspection, and update stale-PID/restart tests for macOS lsof p_comm behavior.

## Best Possible Solution

Land this PR or an equivalent narrow fix after the poll-path verifier calls are bounded by `POLL_SPAWN_TIMEOUT_MS`, while preserving raw lsof PID parsing, argv-based gateway verification, self/ancestor exclusions, Windows behavior, and focused regression coverage for macOS `cnode` gateway listeners and unrelated listeners.

## Reproduction Assessment

Yes. The linked issue gives a concrete macOS launchd reproduction, and current main can be source-reproduced by feeding lsof output with `p<PID>\ncnode` for a gateway process into the current parser; the PR-specific timeout defect is directly visible in the diff.

## Solution Assessment

No, not yet. The argv-verifier direction is the right narrow fix, but both poll branches need to pass the short poll timeout before this is the safest maintainable solution.
