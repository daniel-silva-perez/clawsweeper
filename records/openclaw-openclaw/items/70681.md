## Summary

Current main still has the Unix `lsof` command-name filter that the PR is trying to replace, and there is no Unix `ps` argv verifier on main. This PR remains a relevant implementation path for the linked macOS launchd/stale-gateway bug, but it should stay open for normal review because the existing review comments identify timeout fixes needed before merge.

## Best Possible Solution

Keep this PR open as the active implementation candidate for the linked macOS stale-gateway cleanup bug. The best next step is to update the PR so Unix discovery parses listening PIDs, verifies gateway identity via argv with bounded `ps` timeouts in both initial scan and polling paths, removes the now-unused `lsof` command field if desired, and keeps focused regression tests for macOS `cnode` output, non-gateway listeners, ancestor exclusion, and poll timeout behavior.
