## Risks / Open Questions

- PR head still calls `verifyGatewayPidByArgvSync(pid)` without `POLL_SPAWN_TIMEOUT_MS` in both `pollPortOnce` branches, so a slow `ps` can consume the full 2000 ms port-free polling budget.
- The PR still requests `lsof -Fpc` even after removing use of the `c` field; switching to `-Fp` would avoid carrying dead command-field output.
- Because this path can terminate processes, the verifier must keep false positives very hard: `ps` errors should stay false and tests should cover unrelated listeners and macOS `cnode` output.
