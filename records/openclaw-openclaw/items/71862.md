## Summary

Keep this PR open. Current main at 7ddd815e469e5f5a9407d67cd8dc306eff771b84 still exposes `before_model_resolve` as provider/model-only, the hook merger drops any auth-profile preference, and the embedded runner still derives runtime auth profile selection only from request/session state. The older related #68621 is an overlapping predecessor, not a merged or superseding fix.

## Best Possible Solution

Keep this PR open as the active candidate implementation. The best path is to refresh it onto current main, add `authProfileOverride` to the typed `before_model_resolve` and legacy `before_agent_start` result contract, merge it with the same priority rule as provider/model overrides, consume it in runner auth-profile selection only when no explicit user pin is present, preserve legacy automatic compaction markers, and keep docs, SDK API baseline, and focused hook/runner/live-switch tests aligned. If this PR lands, the older #68621 should be closed as superseded.
