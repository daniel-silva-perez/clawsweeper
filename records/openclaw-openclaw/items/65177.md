## Summary

Keep open. Current main still matches the reported Feishu upgrade gap: Feishu account config is strict and supports `name`, runtime account resolution reads only `name`, and the doctor compatibility path has no Feishu `botName` to `name` normalizer. The focused implementation candidate is still #65384; earlier #65202 was closed unmerged.

## What This Changes

The issue asks for `openclaw doctor --fix` to migrate legacy Feishu account `botName` fields to `name` after upgrade so strict config validation no longer blocks Gateway startup.

## Best Possible Solution

Review and land #65384, or an equivalent Feishu-owned doctor compatibility migration, so `doctor --fix` runs before strict validation, moves legacy `channels.feishu.accounts.*.botName` into `name`, preserves an existing nonblank `name`, removes the stale key, and adds regression coverage for the migration behavior.
