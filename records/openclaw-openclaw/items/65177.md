## Summary

Keep open. Current main still has the reported Feishu upgrade gap: the bundled doctor compatibility path exists, but Feishu does not expose a doctor normalizer/rule for `channels.feishu.accounts.*.botName`, while the account schema is strict and runtime reads only `name`. The focused implementation candidate is still open in #65384; the earlier #65202 was closed unmerged.

## What This Changes

Requests that `openclaw doctor --fix` migrate legacy Feishu `channels.feishu.accounts.*.botName` fields to `name` so upgraded configs validate and the gateway can start.

## Best Possible Solution

Review and land #65384, or an equivalent Feishu-owned doctor compatibility migration, so `doctor --fix` runs before strict validation, moves legacy account `botName` into `name`, preserves an existing nonblank `name`, removes the stale key, and covers the behavior with regression tests.
