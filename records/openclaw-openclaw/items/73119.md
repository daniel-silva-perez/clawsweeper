## Summary

Keep open. Current main still has the branch-and-reappend transcript rewrite path that can leave duplicate raw JSONL suffix entries after large tool-result truncation, and the issue already has overlapping open repair PRs that should be reviewed rather than bypassed.

## What This Changes

The issue asks for `sessions_history` and related startup paths to stop causing duplicate raw JSONL transcript entries while preserving correct provider billing and model context.

## Best Possible Solution

Keep this open until #69486, #72511, or an equivalent maintained fix lands and is verified against the `sessions_history` raw JSONL case. The end state should keep `sessions_history` as a bounded read while making transcript rewrite/truncation avoid low-value duplicate suffix replay or safely reconcile abandoned entries without breaking append-only branch semantics.

Yes. The issue includes concrete raw JSONL evidence, a follow-up comment reproduced the mechanism, and current-main source still contains the truncation-to-rewrite suffix replay path; this review did not create live transcript artifacts because it was read-only.
