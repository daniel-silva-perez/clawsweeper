## Summary

Keep open. Current main still has the raw transcript replay mechanism described in the report: `sessions_history` is read-only, but its large tool result can later trigger persisted tool-result truncation, whose rewrite path branches at the first replacement and appends the rest of the active-branch suffix, including unchanged messages.

## What This Changes

The issue asks OpenClaw to stop raw JSONL transcript duplication caused when `sessions_history` output later triggers tool-result truncation and transcript rewrite.

## Best Possible Solution

Fix the maintenance rewrite path narrowly: keep `sessions_history` as a pure read and keep active-branch rendering, while changing tool-result truncation/transcript rewrite so it avoids low-value deep suffix replays or rewrites only the necessary contiguous tail ranges without breaking labels, compaction markers, branch summaries, or provider overflow recovery.

## Reproduction Assessment

Yes. The issue includes concrete transcript evidence, a follow-up comment reproduced the mechanism on upstream main, and current main still contains tests and source that branch at the first replacement and append the suffix.

## Solution Assessment

Unclear. The current implementation is not the complete fix, but the safest repair strategy should be chosen deliberately because tail-only rewrites, pre-append truncation, and skip-threshold approaches have different effects on the append-only transcript contract.
