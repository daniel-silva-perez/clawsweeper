## Summary

Keep open. The PR is useful source work, but it is not merge-ready against current main because the polling fix is applied to the old core GitHub Copilot auth file while the active implementation now lives in the bundled GitHub Copilot plugin, and the new `--wait` flag is not carried through the provider-auth path.

## What This Changes

The PR adds a GitHub Copilot `--wait` login flag and changes GitHub device-flow polling timing and `slow_down` handling in the old core auth module.

## Best Possible Solution

GitHub Copilot device-flow login should honor GitHub's polling interval contract in the plugin-owned login implementation, expose `--wait` through the active CLI/provider-auth boundary, and document the new remote-terminal flow with focused tests.
