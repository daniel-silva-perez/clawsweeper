## Summary

Keep open. Current main still has the GitHub Copilot device-flow behavior this PR tries to fix, but the PR is stale against the plugin-owned implementation and is not merge-ready as-is.

## What This Changes

This PR adds a GitHub Copilot `--wait` login flag, delays the first device-flow token poll, and changes `slow_down` handling in the old core GitHub Copilot auth module.

## Best Possible Solution

Port the useful behavior into the current plugin-owned GitHub Copilot auth path: extend the login option contract, wire `--wait` through the active CLI/provider-auth route, update the polling loop in `extensions/github-copilot/login.ts`, and add focused timing/flag-forwarding tests plus docs and changelog coverage.
