Likely implementation path: update the device token response type to include optional `interval`; schedule the first poll no earlier than the GitHub-provided interval after the device-code response; on `authorization_pending`, wait the current interval before retrying; on `slow_down`, use the returned interval when present and otherwise add GitHub's documented five seconds; add a `wait`/remote confirmation option that reaches the GitHub Copilot plugin-owned flow without forcing unrelated providers to understand GitHub-specific behavior.
