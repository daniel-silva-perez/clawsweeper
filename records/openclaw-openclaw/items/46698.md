## Summary

Keep open. The reported GitHub Copilot device-flow problem is still real on current main, but this PR now patches a removed legacy core provider file, so it would not fix the active plugin-owned login path without being ported.

## What This Changes

The PR adds a `--wait` flag to the GitHub Copilot login shortcut and changes device-flow polling to delay before polling and honor `slow_down` interval responses.

## Best Possible Solution

Port the polling contract and remote-terminal wait UX into `extensions/github-copilot/login.ts` and the current provider-auth command path, with focused tests, docs, and changelog credit for the contributor while leaving token storage and unrelated providers unchanged.
