## Summary

Keep this PR open. Current main still has the GitHub Copilot device-flow behavior the PR is trying to fix: the token loop polls before any initial delay, `slow_down` still waits `intervalMs + 2000` instead of applying GitHub's returned interval, and `login-github-copilot` has no `--wait` flag or pass-through path. The PR is stale against the current plugin-owned auth implementation, but the underlying WSL/remote-terminal improvement remains useful.

## Best Possible Solution

Keep this PR open, but require it to be ported to the current implementation. The best fix belongs in `extensions/github-copilot/login.ts` for polling behavior, with `DeviceTokenResponse` carrying `interval`, first-poll scheduling that honors the initial GitHub interval, and `slow_down` applying the returned interval. Wire `--wait` through `src/cli/models-cli.ts`, `src/commands/models/auth.ts` or an explicit provider-auth option, the GitHub Copilot plugin auth call, and `src/plugin-sdk/github-copilot-login.ts`; then update the GitHub Copilot docs and add targeted tests for timing, slow_down, and CLI forwarding.
