## Summary

Keep open. Current main still emits only visible assistant text on both HTTP compatibility surfaces, the OpenResponses stream schema has no reasoning delta/done events, and the Claude CLI stream-json parser currently ignores `thinking_delta` blocks. The related open PR #56674 is a plausible implementation candidate to review or replace, not evidence that this issue is already resolved.

## What This Changes

Expose Claude CLI thinking blocks as opt-in reasoning output on the Gateway HTTP `/v1/responses` and `/v1/chat/completions` surfaces.

## Best Possible Solution

Keep this issue open while maintainers review #56674 or replace it with a refreshed implementation that wires Claude/provider thinking into agent events, applies `reasoningDefault` or an explicit HTTP opt-in, emits structured `/v1/responses` reasoning output for stream and non-stream modes, defines the chat-completions compatibility shape, and updates docs plus focused gateway/parser tests.
