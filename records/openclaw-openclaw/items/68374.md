## Summary

Keep open. Current main still discards reasoning on the HTTP surfaces: Claude CLI stream-json handling only turns text deltas into assistant events, `/v1/responses` and `/v1/chat/completions` consume assistant text only, and the public docs still describe Responses `reasoning` as ignored. Related PR #56674 is an open implementation candidate to review or replace, not evidence that the issue is resolved.

## What This Changes

Expose Claude CLI thinking/reasoning blocks through the Gateway HTTP OpenResponses and chat-completions surfaces, gated by existing reasoning controls or an explicit HTTP opt-in.

## Best Possible Solution

Keep this issue open while maintainers review #56674 or replace it with a refreshed implementation that preserves Claude `thinking_delta` as agent reasoning, gates HTTP exposure through `reasoningDefault` or an explicit per-request opt-in, emits structured reasoning for streaming and non-streaming `/v1/responses`, defines the `/v1/chat/completions` compatibility shape, and updates gateway/parser docs plus focused tests.
