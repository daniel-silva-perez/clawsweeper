## Summary

Keep open. Current main still contains the named channel-dashboard render hot paths and has not shipped the end-user dashboard redesign; I also did not find an open PR that closes this issue.

## What This Changes

The issue requests a performance and UX overhaul of the Control UI channel dashboard, including deferred debug JSON rendering, shared timestamp/indexing work, lazy channel config forms, search/filtering, clearer status cards, and first-run guidance.

## Best Possible Solution

Keep the issue open and split it into focused work: first remove the always-visible/debug-heavy render costs, then make channel config sections lazy and memoized, then handle the status-card/search/filter/empty-state UX redesign with product review.

## Reproduction Assessment
