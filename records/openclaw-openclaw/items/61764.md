## Summary

Keep this issue open. Current main still contains the concrete Channel dashboard performance and UX problems described in the report: the live Channels tab renders all channel cards, eagerly renders per-channel config forms, still pretty-prints the full status snapshot inline, and lacks the proposed developer toggle/search/filter/progressive-disclosure redesign.

## Best Possible Solution

Keep the issue open, but split the implementation into reviewable pieces: first remove the eager raw snapshot/config render hot paths in `ui/src/ui/views/channels.ts`, `ui/src/ui/views/channels.config.ts`, and `ui/src/ui/views/config-form.node.ts`; then add measured dashboard perf coverage for multi-account snapshots; finally design the end-user Channels experience with search/filter/status grouping, progressive config disclosure, docs links, and developer-only debug surfaces.
