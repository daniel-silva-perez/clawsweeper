## Summary

Keep this PR open rather than auto-closing it. Current main has already solved the reported Discord attachment failure through the narrower merged Discord fix, but it has not implemented this PR's broader `src/infra/net/proxy-fetch.ts` behavior: generic FormData conversion and Request/input normalization for `makeProxyFetch` and `resolveProxyFetchFromEnv`. The PR also needs a rebase/security cleanup before maintainer action.

## Best Possible Solution

Keep the PR open for maintainer review, but require a rebase onto current main. The best path is to decide whether generic proxy-fetch normalization is still wanted after the narrower Discord fix, then update `src/infra/net/proxy-fetch.ts` to preserve current env proxy option semantics, address the security review items, and keep the FormData/Request regression tests. If maintainers prefer narrower fixes, consolidate the remaining generic FormData work with the active proxy-fetch PR and leave Request normalization as a separate explicit design/review item.
