## Summary

Keep open. Current main has useful partial controls for scheduled automation, including isolated cron sessions, light context, tool allow-lists, timeouts, fallback overrides, hook session/agent restrictions, run history, and post-run usage/cost telemetry. It does not implement the requested first-class scoped/script-limited mode with an admin script registry, dry-run cost previews, per-job token/spend ceilings, daily budget aborts, or planned-vs-actual run logs. Related #38248 remains an open narrower cost-guard tracker, not a replacement for this broader mode.

## Best Possible Solution

Keep this open as the broader scoped automation product/API tracker. The best path is a designed native mode spanning cron payload/schema, /hooks/agent mapping, CLI/UI/docs, isolated runner accounting, and tests: admin-defined script-only execution, per-job token/spend caps, daily ceilings, graceful budget aborts, dry-run cost previews, and structured planned-vs-actual run logs. Treat #38248 as an overlapping cost-guard component, not a replacement.
