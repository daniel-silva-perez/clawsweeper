## Summary

Keep open. Current main has useful partial controls for cron and hook-driven automation, but it still does not implement the requested first-class scoped/script-limited mode with an admin script registry, dry-run cost previews, per-job token or spend ceilings, daily budget aborts, and planned-vs-actual run logs. The recent issue discussion also explicitly kept this open and treated #38248 as only a narrower cost-guard component.

## Best Possible Solution

Keep this open as the broader scoped automation product/API tracker. The best path is a designed native mode across cron payload/schema, hook mappings, CLI/UI/docs, isolated-run accounting, and tests: admin-defined script-only execution, optional dry-run/cost preview, per-job token/spend caps, daily ceilings, graceful budget aborts, and structured planned-vs-actual run logs. Coordinate with #38248 so the cost guard is shared rather than duplicated.
