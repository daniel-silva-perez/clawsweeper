## Summary

Keep open. Current main still has partial scoped automation controls, but not the requested first-class scoped/script-limited mode with per-job/daily budget enforcement, budget aborts, dry-run cost previews, script registry, or planned-vs-actual run logs. The existing April 26 maintainer/Codex review also kept this open, and #38248 remains an open narrower cost-guard tracker rather than a shipped fix for this broader mode.

## Best Possible Solution

Keep this open as the broader product/API tracker. The best path is a native scoped automation mode spanning cron, /hooks/agent, gateway protocol/schema, CLI, embedded runner accounting, docs, and tests: admin-defined script-only execution, per-job token caps, daily spend ceilings, graceful budget aborts, dry-run cost previews, and structured planned-vs-actual run logs. Treat #38248 as a narrower cost-guard component rather than the canonical replacement.
