## What This Changes

The issue reports that `messages.queue.mode: "interrupt"` can abort an active run but still resend the previous assistant reply before the new reply is delivered.

## Best Possible Solution

Review and land PR #50147, or apply an equivalent narrow patch, so aborted interrupt runs with no new assistant output cannot produce or deliver payloads; add focused regression coverage at the payload-builder and reply-runner boundary.
