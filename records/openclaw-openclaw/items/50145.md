Keep open. Current main still has the reported stale-delivery path, and the open paired PR #50147 still has the closing-syntax implementation path unmerged, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Prevent interrupt-mode aborted runs with no new assistant text from replaying the previous assistant reply, so only the replacement turn delivers a response.

## Best Possible Solution

Land PR #50147 or an equivalent narrow patch that suppresses payload delivery from aborted interrupt runs with no new assistant output, and add focused regression coverage around the payload-builder and reply-runner boundary.
