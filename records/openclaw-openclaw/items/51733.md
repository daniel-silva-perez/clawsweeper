## Summary

Keep open. Current main supersedes the watcher-glob portion by watching skill root directories, but it still does not implement the PR's loader goal for repo-style bundles placed under first-class skill roots like `<workspace>/skills/<repo>/skills/<skill>/SKILL.md` or `~/.openclaw/skills/<repo>/skills/<skill>/SKILL.md`. The PR remains a useful implementation candidate after rebasing and narrowing.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. Rebase it onto current main, drop the obsolete watcher-glob changes, and narrow the patch to loading repo-style nested bundles under workspace, project-agent, personal-agent, and managed skill roots with targeted regression tests and explicit safety checks for containment, symlink cycles, candidate limits, size caps, and hybrid bundle precedence.

## Risks / Open Questions

- The PR needs a rebase because current main uses root-directory watching rather than explicit `SKILL.md` glob targets.
- The loader-side change broadens filesystem discovery; final review should re-check realpath containment, symlink-cycle handling, `maxCandidatesPerRoot`, `maxSkillsLoadedPerSource`, and `maxSkillFileBytes`.
- Hybrid bundle precedence remains unresolved when a repo contains both a top-level `SKILL.md` and nested `skills/*/SKILL.md` entries.
- Open PR #72534 overlaps nested skill discovery but targets grouped layouts, not the same repo-style bundle shape, so maintainers should coordinate the two rather than close this PR as implemented.
