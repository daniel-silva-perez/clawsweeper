## Summary

Keep this PR open. Current main supersedes the PR's old watcher-glob approach by watching skill root directories, but it still does not implement the loader-side goal for repo-style nested bundles under managed, workspace, project-agent, or personal-agent skill roots such as `skills/<repo>/skills/<skill>/SKILL.md`.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, but narrow it to the still-missing loader behavior: discover repo-style nested skill bundles under workspace, project-agent, personal-agent, and managed skill roots, with targeted regression tests. Adapt it to current main's root-watch model and coordinate with #72534 only if maintainers want one generic nested-discovery implementation.
