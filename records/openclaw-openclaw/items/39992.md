## Summary

Current main still does not implement the requested `openclaw doctor` warning for runtime/provider-owned model catalog entries that are available but absent from a non-empty `agents.defaults.models` allowlist. The related implementation PR #46235 is closed unmerged, and the current doctor registry only has the hooks-specific model check.

## Best Possible Solution

Keep this issue open. Add a read-only doctor health contribution that loads the same runtime/provider-augmented model catalog used by model list and gateway surfaces, identifies provider-owned supplemental or forward-compatible rows through a stable contract, compares them with a non-empty `agents.defaults.models` allowlist, and emits a warning with concrete model IDs plus `openclaw models allowlist add <model-id>` guidance. It should skip when no allowlist is configured, never auto-modify config, and include focused tests for no allowlist, empty allowlist, no gaps, detected gaps, marker/contract behavior, and warning wording. Treat API-vs-subscription cost warnings as follow-up scope.
