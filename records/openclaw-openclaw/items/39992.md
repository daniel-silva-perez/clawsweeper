## Summary

Current main still lacks the requested `openclaw doctor` warning for provider-augmented or forward-compatible catalog models that are available but missing from a non-empty `agents.defaults.models` allowlist. The only current doctor model/allowlist check is the hooks-specific validation, provider supplemental catalog rows still exist, and the prior implementation PR #46235 is closed unmerged.

## What This Changes

Add a warn-only `openclaw doctor` diagnostic for runtime/provider-supplied synthetic or supplemental model catalog rows that are absent from an explicit model allowlist.

## Best Possible Solution

Keep this open and implement a read-only doctor health contribution. The end state should expose provider-augmented or forward-compatible catalog rows through a stable marker/contract, compare those rows against a non-empty `agents.defaults.models` allowlist, and warn with concrete model refs plus `openclaw models allowlist add <model-id>` guidance while skipping absent or empty allowlists.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-empty `agents.defaults.models` allowlist while provider `augmentModelCatalog` rows exist, then current `openclaw doctor` has only the hooks model check and no general provider-catalog allowlist-gap warning path.

## Solution Assessment

Yes, the requested warn-only doctor diagnostic is the narrow maintainable direction, provided it uses a stable catalog-source marker rather than provider-id heuristics. Auto-adding models or changing runtime allowlist enforcement would be less appropriate because the allowlist is an explicit operator control boundary.
