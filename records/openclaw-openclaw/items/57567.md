## Summary

Keep open. Current main still does not fully repair the reported upgrade path: legacy Feishu account config with `botName` is still rejected by the strict account schema, the Feishu plugin does not implement a doctor compatibility migration for that field or the old `accounts.default` policy shape, and `doctor --fix` still only has narrow malformed-JSON repair rather than general rollback from numbered backups. Startup last-known-good recovery reduces some clobbered-config impact, but it is not the same as the requested doctor repair path. Related #65177 remains open for the narrower Feishu `botName` gap and does not cover the broader corrupted JSON rollback request.

## Best Possible Solution

Keep this open or split it into two explicit maintainer tracks. The Feishu plugin should add a doctor/plugin compatibility migration that rewrites `channels.feishu.accounts.*.botName` to `name`, handles legacy `accounts.default` policy fields, and adds regression coverage. Separately, core doctor should restore from last-known-good or the newest valid config backup for unparseable JSON before applying migrations, or maintainers should open and link a narrower canonical issue for that recovery work.
