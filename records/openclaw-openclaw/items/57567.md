## Summary

Keep open. Current main has newer last-known-good config recovery for some clobbered or invalid startup states, but the specific Feishu legacy schema path from this report is still not fully implemented: `botName` is not migrated to `name`, and `doctor --fix` only repairs the non-JSON prefix case rather than restoring an arbitrary unparseable config from backups. The related #65177 tracks the narrower Feishu `botName` gap, but it is still open and does not cover the whole corrupted-JSON recovery request here.

## Best Possible Solution

Keep this open or explicitly split it. The Feishu-owned path should add a doctor/plugin compatibility migration that rewrites `channels.feishu.accounts.*.botName` to `name`, removes or moves legacy `accounts.default` policy fields into the current top-level/account shape, and adds regression coverage. Separately, core doctor should either restore from last-known-good or the newest valid config backup for unparseable JSON, or maintainers should open a narrower follow-up and link it before closing this broader upgrade report.
