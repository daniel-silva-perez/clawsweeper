## Summary

Keep open. Current main still has the two divergent paths described in the report: `api.runtime.subagent.run()` dispatches the Gateway `agent` method, while the tracked `sessions_spawn` paths register `SubagentRunRecord`s. The `subagent_ended` hook is emitted from the registry lifecycle, so plugin runtime runs that never enter the registry still lack the completion hook path.

## Best Possible Solution

Keep this issue open and fix the core Gateway/plugin-runtime lifecycle contract. The likely path is to make plugin SDK subagent runs create a registry-backed `SubagentRunRecord` or an equivalent lifecycle bridge before dispatch, then complete it through the existing registry lifecycle so `subagent_ended` fires exactly once. Add targeted coverage around `api.runtime.subagent.run()` plus `api.on("subagent_ended")`, and update plugin docs if maintainers choose a narrower contract instead.
