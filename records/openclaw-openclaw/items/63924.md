## Summary

Keep this PR open. Current main already contains the Feishu schema fix, but the PR is not fully obsolete or safe to close: the health hunk is not present in getHealthSnapshot, and the provider-normalization and skill-symlink hunks conflict with current provider identity/auth-alias and realpath containment contracts. The PR should be rebased and split rather than closed as implemented.

## Best Possible Solution

Keep the PR open but ask the author to rebase and split it. Drop the already-shipped Feishu schema change; make any health fix focused on getHealthSnapshot with a regression test around pinned runtime snapshots or external config reload; preserve plan provider ids and use manifest providerAuthAliases for shared auth; handle skill symlink behavior only in a separate security-focused path-canonicalization PR with escape and allowed-alias tests.
