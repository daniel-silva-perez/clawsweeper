## Summary

Keep this PR open. Current main still does not implement the PR’s intended behavior: Feishu card-mode Markdown tables are still emitted as markdown text inside Card JSON 2.0 cards, and current structured outbound, reply, streaming, and quoted-card paths would need coverage before this can safely land or be considered obsolete.

## Best Possible Solution

Keep this PR open and request a current-main rework. The best path is a shared Feishu Card JSON 2.0 Markdown-table conversion helper used by buildMarkdownCard, buildStructuredCard, outbound card-mode sends, and reply delivery, with explicit streaming/fallback semantics and quoted-card table extraction. Coordinate table-limit behavior with #43690 and the closed unmerged #70651 context, and add focused tests for send, outbound, reply, streaming, fenced-code/backtick and tilde fences, table limits, and quote extraction.
