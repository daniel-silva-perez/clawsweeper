## Summary

Keep this PR open. Current main still does not implement native Feishu Card JSON 2.0 table conversion: Markdown-table-shaped text is routed into card paths, but the builders still emit markdown elements rather than native table components. The PR remains a relevant implementation candidate, but it needs a current-main rework because it only covers buildMarkdownCard and does not address structured cards, reply dispatch, streaming cards, quoted-card extraction, or the related table-limit fallback work.

## Best Possible Solution

Keep this PR open and request a current-main rework. The best path is a shared Feishu Card JSON 2.0 markdown-table conversion helper used by buildMarkdownCard and buildStructuredCard/static outbound/reply paths where appropriate, with table-aware quoted-card extraction, explicit streaming behavior or fallback, backtick and tilde fenced-code protection, current `width_mode: "fill"` card config, and table-count/page-size fallback semantics aligned with #43690 and #70651.
