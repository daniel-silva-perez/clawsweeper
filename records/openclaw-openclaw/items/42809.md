## Summary

Keep this PR open. The intended Feishu card-table fix is still not implemented on current main, but the PR branch is stale and incomplete against today’s Feishu card paths: it only hooks the older markdown-card builder, regresses current card width config, and still misses table-count, quoted-card extraction, tilde-fence, structured outbound, reply, and streaming behavior.

## What This Changes

The PR adds a Feishu markdown-card helper that parses Markdown pipe tables into Card JSON 2.0 table elements, delegates buildMarkdownCard to it, and adds unit/integration tests.

## Best Possible Solution

Build a current-main Feishu card table normalization layer that preserves Card JSON 2.0 width_mode, is shared by markdown and structured static cards, has deliberate fallback/table-count behavior, protects both backtick and tilde fenced code blocks, preserves quote extraction for emitted tables, and defines streaming behavior instead of silently leaving streamed table output as raw markdown.

## Reproduction Assessment

Yes. Static current-main inspection shows table-like text in card mode reaches markdown-only card elements, and the PR head can be checked directly with inputs containing multiple tables or tilde-fenced tables to see native table elements emitted where the current extraction/fallback paths are not ready.

## Solution Assessment

No. The PR direction is useful, but the current branch is not the best complete fix because it is based on an older buildMarkdownCard-only surface and does not preserve or extend the structured, reply, streaming, quoted-card, and table-limit behavior now present on main.

  - body: Current main sends renderMode=card and auto table-like outbound text through sendStructuredCardFeishu/buildStructuredCard, not only through buildMarkdownCard.   - confidence: 0.88
- **[P2] Protect tilde-fenced code blocks:** `extensions/feishu/src/markdown-card.ts:25`
  - body: The protected-span regex only recognizes backtick fences. Markdown tables inside valid ~~~ fenced code blocks will still be converted into live Feishu table components, corrupting code examples that should remain markdown text.
  - confidence: 0.86
- **[P2] Bound table conversion before Feishu rejects the card:** `extensions/feishu/src/markdown-card.ts:184-186`
  - body: The splitter emits a native table component for every matched Markdown table. Related Feishu reports show cards can fail with the platform table-count limit, so messages containing several tables need a cap or fallback instead of blindly pushing every table element.
  - confidence: 0.84
- **[P2] Keep quoted replies readable for emitted tables:** `extensions/feishu/src/send.ts:407`
  - body: After this change, outgoing cards can contain native table elements, but current interactive-card extraction only reads div, markdown/lark_md, and plain_text elements. Replies quoting a table-only card can therefore lose the table content and fall back to the generic interactive-card text.
  - confidence: 0.82
