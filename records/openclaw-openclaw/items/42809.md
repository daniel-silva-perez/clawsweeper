## Summary

Keep this PR open. Current main still does not implement the PR’s intended behavior: Feishu card-mode Markdown tables are still emitted as markdown text inside Card JSON 2.0 cards, and the newer structured outbound, reply, streaming, and quoted-card paths would not be covered by the PR’s original narrow `buildMarkdownCard` hook.

## Best Possible Solution

Keep this PR open and request a current-main rework. The best path is a shared Feishu Card JSON 2.0 Markdown-table conversion helper used by `buildMarkdownCard`, `buildStructuredCard`, relevant outbound presentation paths, and reply delivery where card mode is selected, with explicit streaming/fallback semantics, quoted-card table extraction, table-limit handling coordinated with #43690 and #70651, and focused tests for send, outbound, reply, streaming, fenced-code, and quote extraction behavior.
