## Summary

Keep this PR open. Current main still does not implement the PR’s central behavior: Feishu card-mode Markdown tables are still emitted as markdown text inside interactive cards rather than native Card JSON 2.0 table elements. The PR remains a useful implementation candidate, but it needs current-main rework across the structured outbound, reply, streaming, and quoted-card paths before it is ready to close or land.

## Best Possible Solution

Keep this PR open and request a current-main rework. The best path is a shared Feishu Card JSON 2.0 Markdown-table conversion helper used by both `buildMarkdownCard` and `buildStructuredCard` where appropriate, with explicit behavior for streaming output, backtick and tilde fenced-code protection, table-count/page-size fallback semantics coordinated with #43690 and #70651, quoted-card table extraction, and focused tests for outbound, reply, and card parsing paths.
