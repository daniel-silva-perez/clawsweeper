## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: Feishu card-mode Markdown tables are still emitted as markdown text elements rather than native Card JSON 2.0 table components. The PR is a relevant implementation candidate, but it needs a current-main rework across the newer structured-card, outbound, reply, streaming, and quoted-card paths.

## Best Possible Solution

Keep this PR open and request a current-main rework. The best path is a shared Feishu Card JSON 2.0 Markdown-table conversion helper used by buildMarkdownCard and the structured outbound/reply paths where appropriate, with explicit streaming behavior or fallback, backtick and tilde fenced-code protection, quoted-card table extraction, and table-count/page-size fallback semantics coordinated with #43690 and #70651.
