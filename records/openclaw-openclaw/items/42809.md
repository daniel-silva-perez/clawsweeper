## Summary

Keep this PR open. Current main still sends Feishu card-mode Markdown tables as markdown-only Card JSON elements, so the intended native table rendering is not implemented on main. The PR remains relevant, but it is stale/incomplete because current primary card paths use buildStructuredCard/sendStructuredCardFeishu and streaming cards, while the submitted diff only delegates buildMarkdownCard.

## Best Possible Solution

Keep this PR open as a useful Feishu native-table rendering candidate, but request a current-main rework before merge. The best path is a shared Card JSON 2.0 table conversion helper that preserves width_mode="fill", integrates into buildMarkdownCard, buildStructuredCard/sendStructuredCardFeishu, reply-dispatcher, and streaming-card behavior as appropriate, protects backtick and tilde fenced code blocks, preserves quoted-card text extraction for table elements, and coordinates table-count fallback semantics with #43690, #50429, and #70651.
