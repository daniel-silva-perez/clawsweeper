## Summary

Current main still matches the reported failure mode: the guarded provider response wrapper releases its dispatcher only on no body, EOF, read error, or explicit cancel, while direct Anthropic streaming can exit the parser without canceling or draining the response. The issue has credible macOS and Linux reproduction data and remains actionable.

## What This Changes

The issue requests reliable cleanup of gateway provider streamed responses so abandoned Anthropic streams do not leak undici dispatcher sockets and file descriptors.

## Best Possible Solution

Add an idempotent safety net in the shared provider response wrapper so release runs when a streamed response is abandoned before EOF, then make direct Anthropic consumers cancel or drain explicitly where feasible and keep the shared wrapper as the backstop for provider SDK paths.
