## Summary

Current main still does not implement the safety net requested by this issue: buildManagedResponse releases the guarded fetch dispatcher only for no-body responses, natural EOF, read errors, or explicit body cancellation. The current Anthropic Messages and Anthropic Vertex paths appear better than the originally reported SDK path because they drain streams in the normal background-consumer flow, but the abandoned-stream-before-EOF case remains untested and still has no FinalizationRegistry, watchdog, or equivalent cleanup path in OpenClaw.

## Best Possible Solution

Keep this issue open. Add a focused regression test in src/agents/provider-transport-fetch.test.ts for abandoning a managed streamed response before EOF without body.cancel(), then patch src/agents/provider-transport-fetch.ts so release() is invoked exactly once for that path. Also keep the current Anthropic Messages and Anthropic Vertex drain/cancel behavior, with the shared wrapper acting as the generic safety net.
