## Summary

Current main still leaves the reported abandoned-stream path unresolved. The shared guarded provider response wrapper only calls the guarded-fetch release hook for no body, natural EOF, read error, or explicit body cancellation; Anthropic Messages still streams through that wrapper; Anthropic Vertex still delegates to the dependency stream path without a local cleanup wrapper; and the focused provider transport tests do not cover abandoning a streamed response before EOF without canceling it. The issue discussion also includes current macOS and Linux reproduction evidence, plus a linked external Nessie FinalizationRegistry fix as useful prior art, but equivalent OpenClaw code is not present.

## Best Possible Solution

Keep this issue open. Patch src/agents/provider-transport-fetch.ts so managed streamed provider responses reliably invoke release() when the consumer abandons the stream before EOF without explicit cancel, then add focused regression coverage in src/agents/provider-transport-fetch.test.ts. Also audit Anthropic Messages and Anthropic Vertex/pi-ai streaming consumers so they drain or cancel explicitly where feasible, with the shared wrapper acting as the safety net.
