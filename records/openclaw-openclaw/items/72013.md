## Summary

Keep open. Current main still lets pending ACP one-shot records with stable ACPX or agent session IDs enter startup identity reconciliation and trigger the gateway warning; the exact focused fix PR is closed unmerged, and the adjacent open identity PR does not cover historical terminal metadata.

## What This Changes

The issue requests that ACP startup identity reconciliation skip or downgrade terminal one-shot or closed ACP session records whose persisted identity remains pending.

## Best Possible Solution

Add a narrow guard inside AcpSessionManager.reconcilePendingSessionIdentities before checked increments and runtime work so terminal one-shot or legacy closed ACP records are skipped, while preserving normal persistent pending-identity reconciliation and adding focused regression tests for pending one-shot records with stable ACPX or agent IDs.

## Reproduction Assessment

Yes. A manager unit fixture can return a persisted ACP record with mode:"oneshot", identity.state:"pending", and identity.acpxSessionId or identity.agentSessionId; current main will pass the pending-plus-stable predicate, increment checked, and let gateway startup log the warning when reconciliation does not resolve it.

## Solution Assessment

Yes. The startup skip is the narrowest maintainable fix for historical terminal one-shot metadata; the existing post-turn one-shot identity resolution helps future successful runs but does not prevent already-persisted pending one-shot records from being retried on every restart.
