## Summary

Keep open. Current main still routes pending ACP one-shot records with stable ACPX or agent session IDs through startup identity reconciliation, and the related focused fix PR is closed unmerged.

## What This Changes

Request that ACP startup reconciliation skip or downgrade noisy handling for terminal one-shot or closed ACP session metadata whose persisted identity remains pending.

## Best Possible Solution

Add a narrow guard inside AcpSessionManager.reconcilePendingSessionIdentities before checked increments and runtime handle work so one-shot ACP records, and any clearly terminal persisted ACP metadata, are skipped while preserving persistent pending-identity reconciliation.

## Reproduction Assessment

Yes. A focused unit fixture can return a persisted ACP entry with mode:"oneshot", identity.state:"pending", and acpxSessionId or agentSessionId; current main passes that entry into startup reconciliation and can produce the warning.

## Solution Assessment

Yes. The startup skip is the narrowest maintainable fix for historical persisted metadata; marking future one-shot identities resolved would help new sessions but would not stop old pending records from being retried on every restart.
