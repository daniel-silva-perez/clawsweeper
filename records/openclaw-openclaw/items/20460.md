## Summary

Keep this issue open. Current main still stores config backups beside the config file as `.bak` plus numbered `.bak.N` slots, with tests and recovery/docs still built around that adjacent numbered ring. The directly related directory-plus-datetime PR #20877 is closed unmerged, while #39982 and #40392 remain open implementation candidates for the narrower datetime-suffix behavior rather than shipped current behavior.

## Best Possible Solution

Keep this issue open as the core config-backup ergonomics request unless maintainers explicitly consolidate it into a narrower canonical item. A complete fix should define the dedicated backup directory, timestamp format, bounded retention, collision handling, legacy adjacent `.bak.N` compatibility or migration, recovery/doctor behavior, docs, and tests across `src/config/backup-rotation.ts`, config write/recovery paths, and the config backup tests, coordinating with #20877, #39923, #39982, #40392, and #33724.
