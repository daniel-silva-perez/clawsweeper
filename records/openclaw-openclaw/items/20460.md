## Summary

Keep this issue open. Current main still stores config backups beside the active config as `.bak` plus numbered `.bak.N` slots, and the tests, recovery paths, audit text, and docs still assume that adjacent numbered layout. The directly related directory-plus-datetime PR #20877 is closed unmerged, while #39923, #39982, and #40392 cover narrower datetime-suffix behavior rather than the dedicated backup directory requested here.

## Best Possible Solution

Keep this issue open as the core config-backup ergonomics request unless maintainers explicitly consolidate it into a narrower canonical item. A complete fix should define the dedicated backup directory, timestamp format, bounded retention, collision handling, legacy adjacent `.bak.N` compatibility or migration, recovery/doctor/audit behavior, docs, and tests across the config backup and recovery paths, coordinating with #20877, #39923, #39982, #40392, and #33724.
