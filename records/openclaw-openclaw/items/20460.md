## Summary

Current main still stores config backups beside the active config as `.bak` plus numbered `.bak.N` files. The requested dedicated `config-backup/` directory with datetime-based names is not implemented. The direct implementation PR is closed unmerged, and the currently open related PRs cover narrower timestamp-suffix work rather than the full directory-plus-datetime request.

## Best Possible Solution

Keep this open as the core config-backup ergonomics request unless maintainers explicitly consolidate it into a broader backup-policy issue. A complete fix should define the dedicated backup directory, timestamp format, bounded retention, collision handling, legacy adjacent `.bak.N` compatibility or migration, recovery/doctor/audit behavior, docs, and tests across config backup and recovery paths.
