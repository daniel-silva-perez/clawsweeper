## Summary

Keep open. Current main still has the unsafe OpenAI-completions stream path, and this PR targets the right parser layer, but the patch drops a real nested Mistral thinking-block shape and lacks the required user-facing changelog entry before it can merge.

## What This Changes

The PR adds structured `delta.content` normalization to the OpenAI completions stream, reuses it for reasoning-field fallbacks, exports the helper for tests, and adds unit coverage for string, object, and array content shapes.

## Best Possible Solution

Land one parser-side OpenAI-completions fix that normalizes structured `delta.content` arrays, including nested Mistral thinking sub-block arrays, preserves source order, falls through for unsupported/empty content so same-chunk reasoning and tool calls still process, and includes focused regression tests plus the required changelog credit.

Yes. A mocked OpenAI-completions stream chunk with `delta.content` set to Mistral-style typed blocks exercises the current main path that appends content as text and can produce `[object Object]`; the linked issue and Mistral docs provide the real provider shape.

## Solution Assessment

Mostly yes, but not as submitted. The OpenAI-completions stream parser is the right shared layer, but the implementation should flatten nested thinking arrays and add release-note coverage before merge.
