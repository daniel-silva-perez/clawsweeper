## Summary

Keep open. Current main still has the reported ACP runtime-control behavior: generic runtime options are converted to ACP config IDs and forwarded when the backend does not advertise a strict key list. An open linked PR already uses closing syntax for this issue, so cleanup should wait until that PR is reviewed and landed or replaced.

## What This Changes

The issue asks OpenClaw-managed ACP sessions to stop sending unsupported generic config options such as timeout, thinking, and approval_policy to Claude ACP unless the adapter advertises support.

## Best Possible Solution

Update the ACP runtime-control layer so it derives supported config IDs from backend/session capabilities, skips or degrades unsupported generic keys for strict adapters, keeps timeout as an OpenClaw execution timeout unless explicitly advertised by the adapter, and adds regression coverage for both strict-key and no-key backends.

## Reproduction Assessment

Yes. The issue gives a concrete live reproduction path, and current main has the static precondition: `sessions_spawn` can persist model/thinking/timeout options, those become generic ACP config IDs, and the manager forwards them when no strict key list is advertised.

## Solution Assessment

Unclear until PR #74048 is reviewed. Its skip-when-unadvertised direction addresses the reported failure mode, but the maintainable fix should align the capability contract and include regression tests before this issue is closed.
