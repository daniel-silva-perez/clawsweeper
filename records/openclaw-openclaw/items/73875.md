## Summary

Keep open. Current main still has the reported ACP runtime-control path, and PR #74048 is an open, unmerged implementation candidate with closing syntax, so this issue should remain open until that PR lands or is replaced.

## What This Changes

OpenClaw-managed Claude ACP sessions should avoid sending unsupported generic runtime config options such as timeout, thinking, and approval_policy to strict ACP adapters.

## Best Possible Solution

The ACP runtime-control layer should derive supported config IDs from backend or session capabilities, skip or gracefully degrade unsupported generic keys for strict adapters, keep timeout as an OpenClaw execution timeout unless explicitly advertised by the adapter, and add regression coverage for strict-key and no-key backends.

## Reproduction Assessment

Yes. The issue gives a concrete live reproduction path, and current main shows the static precondition: `sessions_spawn` stores generic ACP runtime options, those options become ACP config IDs, and the manager forwards every pair when no strict key list is present.

## Solution Assessment

Unclear for the linked PR until it is reviewed. The direction of skipping unsupported config options is correct, but the maintainable fix should align the capability contract and include regression tests before this issue is considered solved.
