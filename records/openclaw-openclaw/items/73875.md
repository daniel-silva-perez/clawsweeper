## Summary

Current main still matches the reported Claude ACP failure mode. `sessions_spawn(runtime:"acp")` stores generic OpenClaw runtime options, the ACP control plane maps them to `model`, `thinking`, `approval_policy`, and `timeout`, and the acpx wrapper only special-cases unsupported timeout/thinking controls for Codex. The pinned Claude ACP adapter exposes `mode`, `model`, and optional `effort`, and rejects unknown config IDs, while current acpx capabilities do not advertise a strict `configOptionKeys` list that OpenClaw can filter against.

## Best Possible Solution

Keep this issue open and fix the ACP runtime-control path generically: derive supported config option IDs from the target ACP session/adapter, map OpenClaw `thinking` to adapter-specific IDs such as Claude `effort` when advertised, keep run timeout as OpenClaw/acpx execution timeout rather than a session config option unless supported, skip unsupported generic keys without failing the turn, and add tests with a Claude-like non-Codex ACP adapter that advertises only `mode`, `model`, and `effort`.
