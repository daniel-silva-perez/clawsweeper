## Summary

Keep this issue open. Current main still has Copy-only markdown command blocks and an approval UI for already-pending exec/plugin approval events, but it does not provide an assistant-suggested command card with Approve and Run, cwd/host binding, risk labels, or output streaming back into chat.

## Best Possible Solution

Implement a narrow Control UI command-card path for assistant shell snippets: preserve Copy, add Approve and Run only when an exact command, cwd, host/node/session, and safe execution target are available, route execution through the existing exec approval/node invoke pipeline, stream output back into chat, and add focused UI, gateway, and docs coverage.

## Risks / Open Questions

- This feature touches a high-risk local execution path; implementation should reuse the existing exec approval and `systemRunPlan` safeguards rather than adding a direct chat-to-shell path.
- Current markdown rendering has no command risk classifier or target metadata, so read-only/destructive/external/config-changing labels would need explicit design and tests.
