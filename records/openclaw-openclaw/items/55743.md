## Summary

Current main still does not implement the requested Control UI command-card workflow. Assistant markdown code blocks have Copy only, while the existing approval UI resolves already-pending exec/plugin approval events rather than launching assistant-suggested terminal commands from chat.

## Best Possible Solution

Keep this issue open. Implement a narrow Control UI command-card layer for assistant markdown shell snippets that preserves Copy, adds Approve and Run only when a safe execution target is available, binds exact command/cwd/host/session, routes execution through the existing gateway/node exec approval pipeline, streams output back into chat, and adds focused UI/gateway tests plus docs.

## Risks / Open Questions

- This feature touches a high-risk execution path. A future implementation should route through the existing exec approval and `systemRunPlan` safeguards instead of adding a parallel direct-run path from chat markdown.
- Current markdown rendering has no risk classifier for shell snippets, so destructive, external/network, config-changing, and read-only labels would need explicit design and tests.
