## Summary

Current main still matches the reported Microsoft Teams DM file attachment bug. The `file.download.info` branch still returns the raw `content.downloadUrl`, while the existing Graph shares rewrite only applies to the separate `contentUrl` path. The latest release tag has the same behavior, and tests still cover Graph shares only for `contentUrl` reference attachments, not SharePoint/OneDrive `downloadUrl` values inside `file.download.info`.

## Best Possible Solution

Keep this issue open. Patch `extensions/msteams/src/attachments/download.ts` so the `file.download.info` branch applies `tryBuildGraphSharesUrlForSharedLink(downloadUrl)` and uses the Graph shares URL when available. Add focused regression coverage where `content.downloadUrl` is a SharePoint/OneDrive URL and only the Graph shares endpoint succeeds with Graph-scoped auth retry. Preserve the current warning logger behavior for failed downloads.
