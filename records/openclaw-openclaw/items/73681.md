## Summary

Keep open. Current main still has the Google Chat ingress gap described in the issue: the webhook parser accepts non-MESSAGE event payloads, but the registered processor returns before routing anything unless the event type is exactly MESSAGE. Google Chat has outbound reaction actions, but no inbound reaction event shape, reaction notification config, system-event routing, or docs for reaction ingress. Related #9764 and #17840 are adjacent but do not supersede this issue.

## Best Possible Solution

Keep this issue open and implement the Google Chat-owned ingress path: extend the Google Chat event types/parser for reaction created/deleted payloads, route them through the same access controls as messages, enqueue a normalized passive reaction system event or typed hook, add focused tests for allowed/blocked reaction events, and document the Google Chat inbound reaction configuration. Keep #9764 scoped to OAuth/outbound reaction creation and #17840 scoped to cross-channel reaction-triggered turns.
