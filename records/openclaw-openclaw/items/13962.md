## Summary

Keep open. Current main still only buffers skipped mention-gated WhatsApp group messages for later context; it does not route unmentioned messages through a background model, choose a different model based on mention state, or expose a per-group mention context window setting. The existing channel model override work covers group/channel routing, but not the per-mention behavior requested here.

## Best Possible Solution

Keep this issue open until the WhatsApp/group pipeline supports mention-state-aware routing plus configurable per-group mention context sizing, or until OpenClaw adds a deliberate trusted pre-gating plugin/API hook that can process unmentioned group messages with a background model without bypassing group safety policy. The implementation should update WhatsApp config types/schema/help/docs and add focused tests for skipped-message buffering, mentioned-message routing, and per-group context limits.
