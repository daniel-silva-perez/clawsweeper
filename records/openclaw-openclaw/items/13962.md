## Summary

Keep this issue open. Current main at 955b4df0939f90b4b6de5e7e6f3149dd125e1e50 still buffers skipped mention-gated WhatsApp group messages as pending context only; it does not route unmentioned messages through a background model, choose a model based on mention state, or expose a per-group mention-context window. The related per-chat/channel model override work covers channel/group-id routing, but not the per-mention behavior requested here.

## Best Possible Solution

Keep this issue open until the WhatsApp/group pipeline supports mention-state-aware model routing plus configurable per-group mention context sizing, or until OpenClaw adds a deliberate trusted pre-gating plugin/API hook that can process unmentioned group messages with a background model without bypassing group safety policy. The implementation should update WhatsApp config types/schema/help/docs and add focused tests for skipped-message buffering, mentioned-message routing, and per-group context limits.
