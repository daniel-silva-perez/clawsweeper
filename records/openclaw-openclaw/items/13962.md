## Summary

Current main still has only pending-history buffering for mention-gated WhatsApp group messages plus channel/group model overrides. It does not implement mention-state-aware model routing, background-model processing for unmentioned messages, or a per-group mention context-window setting.

## Best Possible Solution

Keep this issue open until OpenClaw implements mention-state-aware routing plus configurable per-group mention context sizing in the WhatsApp/group pipeline, or adds a deliberate trusted pre-gating plugin/API hook that can process unmentioned group messages with a background model without bypassing group safety policy.
