## Summary

Keep this PR open. Current main still does not implement a per-request `tool_deny` override for `POST /v1/responses`: the strict OpenResponses request schema omits the field, the HTTP handler never parses or forwards a deny override, and the agent tool-policy pipeline has no per-request deny step. The PR remains a focused additive implementation candidate, and its review thread shows the one docs mismatch was addressed in the PR head commit.

## Best Possible Solution

Keep the PR open for maintainer review. If the feature is accepted, land the additive `tool_deny` schema/docs/plumbing/policy step with focused tests; close only after an equivalent per-request OpenResponses built-in tool denial implementation is merged to main.
