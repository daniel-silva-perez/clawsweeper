## Summary

Keep this issue open. Current main still downloads media from `reply_to_message` after only checking that the reply target has inbound media; there is no guard for a reply target authored by the Telegram bot, and open PR #57298 is still the explicit implementation candidate for this bug.

## Best Possible Solution

Review and either land or close PR #57298. The product fix should add a guard in `extensions/telegram/src/bot-handlers.runtime.ts` so `resolveReplyMediaForMessage` returns `[]` when the reply target was authored by `ctx.me`, plus a regression test proving a user text reply to a bot-sent photo does not call `getFile` or attach that media.
