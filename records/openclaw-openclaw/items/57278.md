## Summary

Keep this issue open. Current main still lacks the self-authored reply-target guard in Telegram reply media resolution, and open PR #57298 explicitly closes this issue but has not merged yet.

## Best Possible Solution

Review and either land or close PR #57298. The intended fix belongs in `extensions/telegram/src/bot-handlers.runtime.ts`: return `[]` from `resolveReplyMediaForMessage` when the reply target was authored by `ctx.me`, with a regression test proving a user text reply to a bot-sent photo does not call `getFile` or attach media.
