## Summary

Keep this issue open. Current main has active assistant name/avatar display and user-vs-assistant bubble styling, but it still lacks the central requested per-agent bubble color contract and per-message assistant attribution needed for mixed-agent chat history.

## Best Possible Solution

Keep this open until maintainers either implement or explicitly reject a Control UI/chat-session contract for assistant-message attribution and optional per-agent bubble styling. A complete implementation likely needs message or group-level assistant identity metadata, a validated color/style field or render hook, CSS variables/classes for assistant-specific bubbles, and focused UI tests for mixed-agent and historical sessions.
