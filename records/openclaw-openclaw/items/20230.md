## Summary

Keep open. Current main partially covers the name/avatar side: `agent.identity.get` resolves the active assistant identity and the chat renderer uses the active assistant name, while CSS already distinguishes user bubbles from generic assistant bubbles. The central request remains unimplemented because chat history groups do not carry assistant agent/style metadata, assistant messages are still rendered with one active assistant identity, and there is no validated per-agent bubble color/style config or CSS render hook.

## Best Possible Solution

Keep this open until maintainers either implement or explicitly reject a Control UI chat identity/style contract. A complete implementation should add assistant attribution at message or group level, define a validated optional per-agent bubble color/style setting or render hook, render assistant-specific bubbles safely in CSS, update config/docs if a public key is added, and add focused tests for active-session, mixed-agent, and historical chat rendering.
