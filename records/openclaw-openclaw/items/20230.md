## Summary

Keep open. Current main covers the active assistant name/avatar path, and user bubbles are already visually distinct from generic assistant bubbles, but the core request is still not implemented: chat history groups do not carry assistant agent/style metadata, assistant messages render with one active assistant identity, and there is no validated per-agent chat bubble color/style setting or render hook.

## Best Possible Solution

Keep this open until maintainers either implement or explicitly reject a Control UI chat identity/style contract. A complete implementation should add assistant attribution at message or group level, define a validated optional per-agent bubble color/style setting or safe render hook, render assistant-specific bubbles in CSS, update docs/config if a public key is added, and add focused tests for active-session, mixed-agent, and historical chat rendering.
