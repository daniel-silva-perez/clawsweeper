## Summary

Keep this issue open. Current main still has reply threading through assistant output tags, payload metadata, channel replyToMode settings, and channel-specific thread configs, but it does not implement a global inbound user directive that strips trailing phrases like "reply in thread" / "rit" and forces the outbound reply to target the current inbound message. Related PR #47297 attempted the feature but was closed unmerged, with review feedback pointing to fixable implementation details rather than obsolescence.

## Best Possible Solution

Implement a narrow core auto-reply inbound directive: detect deliberate trailing user phrases, strip them from Body, RawBody, CommandBody, BodyForAgent, and BodyForCommands after finalization, carry a per-turn flag through MsgContext/run preparation, and force replyToCurrent/replyToId via the existing reply-threading payload pipeline. Default behavior should avoid the bare "thread" trigger unless maintainer product review explicitly accepts that false-positive risk.
