## Summary

Current main still does not implement the requested global inbound user directive. Reply threading is implemented through assistant output tags, payload metadata, `replyToMode`, and channel-specific thread settings, but there is no inbound preprocessing that strips trailing phrases such as "reply in thread" or "rit" and forces the outbound response to reply to the current inbound message. Related PR #47297 attempted this but was closed unmerged, so the issue remains valid and narrow enough for a focused fix PR.

## Best Possible Solution

Implement a narrow core auto-reply inbound directive. Detect deliberate trailing user phrases, strip them from all finalized inbound body fields, carry a per-turn flag through `MsgContext` and agent-runner payload construction, and force `replyToCurrent` or `replyToId` through the existing reply-threading pipeline. Avoid enabling bare `thread` by default without maintainer product approval.
