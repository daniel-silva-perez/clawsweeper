## Summary

Keep open. Current main still delivers the announce-step text rather than the full target reply, so the reported behavior gap remains; however, this PR is not merge-ready because the new full-reply send bypasses ANNOUNCE_SKIP external-delivery suppression and the branch is stale against current main's A2A helper seams.

## What This Changes

The PR changes `src/agents/tools/sessions-send-tool.a2a.ts` to send the post-ping-pong `latestReply` directly to the announce target, suppress duplicate announce sends, run the announce step in the requester session, and include `threadId` on the send params.

## Best Possible Solution

Rework the A2A delivery on current main so the visible thread receives the intended post-ping-pong final content only under an explicit, documented suppression policy; keep current dependency injection and per-session lane helpers, preserve `accountId`/`threadId`, and add targeted tests for full-reply delivery, ping-pong final reply delivery, ANNOUNCE_SKIP suppression, fallback behavior, and thread/account passthrough.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor ANNOUNCE_SKIP before external delivery:** `src/agents/tools/sessions-send-tool.a2a.ts:106-117`
  - body: The new full-reply send posts `latestReply` to the announce target before the announce step can return `ANNOUNCE_SKIP`. That changes the existing suppression contract from “do not post” into “post the full reply anyway,” which can leak content into a linked channel or thread even when the announce step intentionally asks to stay silent.
  - confidence: 0.9
- **[P2] Use the target channel for announce provenance:** `src/agents/tools/sessions-send-tool.a2a.ts:148-150`
  - body: This announce step marks `sourceSessionKey` as the target session but keeps `sourceChannel` as the requester channel. When requester and target channels differ, downstream provenance will say the message came from the wrong channel, which can affect channel-specific formatting or routing in the announce response.
  - confidence: 0.78
