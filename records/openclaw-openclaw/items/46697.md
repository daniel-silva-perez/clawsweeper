## Summary

Keep this PR open. Current main still does not post the full target reply to the visible announce target, so the contribution is not obsolete, but the patch is not merge-ready because it performs an external full-reply send before honoring announce suppression and it diverges from current main's lane/provenance behavior.

## What This Changes

The PR changes the A2A `sessions_send` flow to send the target session's full final reply directly to the announce target, run the announce step in the requester session, suppress fallback announce delivery after that send, and pass `threadId` on fallback sends.

## Best Possible Solution

Define an explicit A2A delivery policy on current main: a visible announce target should receive exactly one allowed final post, with `ANNOUNCE_SKIP`, target channel/account/thread provenance, per-session lanes, and transcript hygiene preserved by focused tests.

## Reproduction Assessment

Yes. Against PR head, use an announce target, a non-empty target reply, and an announce step that returns `ANNOUNCE_SKIP`; the PR sends the full reply before the skip check, while current main has coverage asserting no external send in that case.

## Solution Assessment

No. The current PR is not the best fix because it makes full-reply delivery an unconditional external side effect before the announce policy decision; the safer approach is to decide suppression/provenance first and then perform exactly one external send with the intended final content.

  - confidence: 0.78
