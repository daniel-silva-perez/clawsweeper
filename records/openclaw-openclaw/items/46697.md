## Summary

Keep this PR open, but do not merge it as-is. Current main has not implemented the PR's central full-target-reply delivery behavior, but the proposed patch introduces a blocking delivery-policy regression by sending the full reply to the external announce target before the announce step can return `ANNOUNCE_SKIP`; it also needs rebase-level review against current main's session-scoped lane and provenance changes.

## What This Changes

The PR changes `src/agents/tools/sessions-send-tool.a2a.ts` so A2A `sessions_send` posts the latest target reply directly to the resolved announce target after ping-pong, runs the announce step in the requester session, suppresses fallback announce sends after direct delivery, and passes `threadId` on fallback sends.

## Best Possible Solution

Define an explicit current-main A2A delivery policy: a visible announce target should receive exactly one allowed final post, after ping-pong completes, with `ANNOUNCE_SKIP`, target-channel provenance, per-session lanes, and transcript hygiene preserved by focused tests.

## Reproduction Assessment

Yes. Against PR head, use an announce target, a non-empty target reply, and an announce step that returns `ANNOUNCE_SKIP`; the PR sends the full reply at line 106 before the skip check, while current main has coverage asserting no gateway `send` occurs in that case.

## Solution Assessment

No. The PR is not the best fix as written because it performs an external full-reply send before the announce policy decision; the safer path is to decide suppression and provenance first, then perform exactly one external send with the intended final content.

  - body: The new full-reply send publishes `latestReply` to the announce target before the announce step can return `ANNOUNCE_SKIP`.   - body: This announce step marks the source session as the target session but keeps `sourceChannel` as the requester channel. When requester and target channels differ, downstream provenance labels target content as coming from the wrong channel, which can affect formatting and routing decisions.
  - confidence: 0.82
- **[P2] Scope the requester announce lane by session:** `src/agents/tools/sessions-send-tool.a2a.ts:147`
  - body: Current main scopes nested agent lanes per session to avoid cross-session blocking, but the new requester-side announce run uses the shared `AGENT_LANE_NESTED` lane. Concurrent A2A announces for different sessions can block each other again unless this uses the session-scoped lane helper.
  - confidence: 0.78
- **[P3] Skip dead announce runs after full-reply sends:** `src/agents/tools/sessions-send-tool.a2a.ts:141-158`
  - body: When full-reply delivery succeeds, this function still runs the announce model call even though the later send branch is guarded by `!fullReplyPosted`. That adds latency/cost and writes a synthetic announce turn without a deliverable result.
