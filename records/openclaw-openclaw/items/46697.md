## Summary

Keep this PR open, but do not merge it as written. The PR addresses a real A2A delivery gap, but the current patch posts full target replies before the announce policy can suppress delivery, mixes requester/target provenance, and regresses current-main per-session nested lane behavior.

## What This Changes

The PR changes `src/agents/tools/sessions-send-tool.a2a.ts` to post the target session's latest reply directly to the resolved announce target, run the announce step in the requester session, suppress fallback announce delivery after direct send, and pass `threadId` on fallback sends.

## Best Possible Solution

Define an explicit current-main A2A delivery policy: a visible announce target should receive exactly one allowed final post after ping-pong completes, with `ANNOUNCE_SKIP`, target-channel provenance, per-session lanes, and transcript hygiene preserved by focused tests.

## Reproduction Assessment

Yes. Against PR head, use an announce target, a non-empty target reply, and an announce step that returns `ANNOUNCE_SKIP`; the PR sends the full reply at line 106 before the skip check at line 157 can suppress delivery.

## Solution Assessment

No. Posting the full reply directly is a plausible product direction, but this implementation bypasses the existing announce policy; the safer fix is to decide suppression/provenance first and then perform exactly one external send with the intended final content.
