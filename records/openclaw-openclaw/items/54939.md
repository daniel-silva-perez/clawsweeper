## Summary

Keep this PR open. Current main still does not implement the requested Exa `webSearch.baseUrl` support: the Exa manifest schema only allows `apiKey`, the runtime uses the hard-coded Exa search endpoint, and the docs still document only `apiKey`. The PR remains a plausible implementation candidate for the linked Exa baseUrl issue, but it needs review/rebase against current main because the Exa runtime and generated metadata layout have changed since the PR diff.

## Best Possible Solution

Keep the PR open and ask for a rebase/port onto current main: add `webSearch.baseUrl` to `extensions/exa/openclaw.plugin.json`, implement endpoint resolution in `extensions/exa/src/exa-web-search-provider.runtime.ts`, include the resolved endpoint in the Exa cache key, add endpoint-resolution/cache tests in `extensions/exa/src/exa-web-search-provider.test.ts`, update `docs/tools/exa-search.md` and current generated/config artifacts as required, then review/land it to close #54928 via merge.
