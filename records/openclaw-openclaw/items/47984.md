## Summary

Keep this PR open. Current main at c0ec58f4b6906ad06fd4c6f265d43be053b2d193 still has Discord `appliedTags` support only on thread creation, not on `channelEdit`; the edit payload type, shared parser, guild/guild-admin dispatch path, REST PATCH body, and tests still omit `applied_tags`. The PR targets a real Discord Modify Channel thread field, but it is stale against the current Discord plugin runtime layout and still needs explicit empty-array preservation through dispatch.

## Best Possible Solution

Keep this PR open and request a current-main rebase/refit. The best implementation is to add `appliedTags?: string[]` to `DiscordChannelEdit`, extend `readDiscordChannelEditParams` so supported camelCase and snake_case inputs reach both guild and guild-admin channelEdit paths, preserve explicit empty arrays for tag clearing, map `appliedTags` to `applied_tags` in `editChannelDiscord`, and add focused tests for non-empty tag updates plus empty-array tag clearing. No security-sensitive changes are needed for this feature.
