## Summary

Keep this PR open. Current main still does not implement the requested Discord channelEdit appliedTags path: the edit payload type lacks appliedTags, the REST edit body never sends applied_tags, and both channel-edit action paths route through a shared parser that only reads availableTags. The PR is still a relevant implementation candidate, but it needs to be rebased/refit onto the current Discord plugin runtime layout and must preserve explicit empty arrays for tag clearing.

## Best Possible Solution

Keep this PR open and ask for a rebase onto the current Discord plugin action runtime. The best implementation is to add appliedTags?: string[] to DiscordChannelEdit, parse camelCase and preferably snake_case tag arrays in readDiscordChannelEditParams while preserving explicit empty arrays, forward that shared parser through both guild and guild-admin paths, map appliedTags to applied_tags in editChannelDiscord, and add focused tests for non-empty tag updates plus appliedTags: [] tag clearing.
