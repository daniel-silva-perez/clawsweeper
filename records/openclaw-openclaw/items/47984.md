## Summary

Keep this PR open. Current main at 230f7122dd5cd8a37a55e671254bd9be8278f6e8 still supports `appliedTags` only for Discord forum/media thread creation, not for `channelEdit`: the edit payload type, shared parser, guild/guild-admin dispatch path, and REST PATCH body all omit `applied_tags`. The PR targets a real missing Discord API field, but it is stale against the current Discord plugin runtime layout and still needs explicit empty-array preservation through dispatch.

## Best Possible Solution

Keep this PR open and request a current-main rebase/refit. The best implementation is to add `appliedTags?: string[]` to `DiscordChannelEdit`, extend `readDiscordChannelEditParams` so camelCase and snake_case inputs reach both guild and guild-admin channelEdit paths, preserve explicit empty arrays for clearing tags, map `appliedTags` to `applied_tags` in `editChannelDiscord`, and add focused tests for non-empty tag updates plus empty-array tag clearing.
