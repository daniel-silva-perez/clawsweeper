## Summary

Keep this PR open. Current main at e476523082a6fff3a0c3048077d2f0e52413fbf2 has shipped Discord `appliedTags` only for forum/media thread creation, not for `channelEdit`; the edit type, parser, REST PATCH body, and current tests still omit `applied_tags`.

## Best Possible Solution

Keep this PR open and request a current-main rebase/refit. The best implementation is to add `appliedTags?: string[]` to `DiscordChannelEdit`, parse camelCase and probably snake_case applied tag inputs in `readDiscordChannelEditParams`, preserve explicit empty arrays for clearing all tags, map `appliedTags` to `applied_tags` in `editChannelDiscord`, expose the parameter in the current channel action/tool schema where needed, and add focused tests for both non-empty tag updates and empty-array clearing. No security-sensitive changes are needed for this feature.
