## Summary

Keep this issue open. The report is recent, concrete, and specific to macOS `openclaw tui` with Korean/English input switching. Current `main` has moved from `@mariozechner/pi-tui` 0.70.2 to 0.70.5, but I did not find a targeted OpenClaw fix, upstream fix reference, release note, regression test, or live reproduction proof showing the reported stray glyph/separator artifacts are fixed.

## Best Possible Solution

Keep the issue open as a TUI/macOS IME rendering bug. Maintainers should reproduce on current `main` or the latest release with Korean/English input switching, capture raw input/render examples, then either fix the OpenClaw wrapper or route a focused fix upstream to pi-tui. Add a regression test or documented upstream fix before closing.
