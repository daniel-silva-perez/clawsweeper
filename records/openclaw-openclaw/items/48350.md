## Summary

Keep open. Current main has adjacent safeguard-mode summary quality audits and newer pre-compaction checkpoint snapshots, but it still does not implement this PR's intended pure post-compaction validator, guard-signal input contract, validator reason codes, or `shouldRecommendReset` output. This remains a draft stacked implementation candidate, not work that has been merged or superseded.

## Best Possible Solution

Keep this PR open as a candidate in the loop-aware compaction stack. The best path is to resolve or supersede the earlier stack PRs, rebase onto current main, and review a focused validator implementation that either integrates cleanly with the existing compaction checkpoint snapshots or documents why validation remains diagnostic-only. Follow-up reset delivery should stay separate and require explicit maintainer product/security review.
