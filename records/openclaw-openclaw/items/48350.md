## Summary

Keep open. Current main does not implement this PR's intended pure post-compaction validator, guard-signal contract, validator reason codes, or `shouldRecommendReset` result. Main has adjacent compaction summary quality retries and pre-compaction checkpoint snapshots, but those are not the same post-compaction validation surface. This PR is also a draft stacked on same-author open compaction-guard work, so the right maintainer path is review/rebase or explicit supersession, not cleanup closure.

## Best Possible Solution

Keep this PR open as an implementation candidate in the loop-aware compaction stack. The best path is to resolve or supersede the earlier stack PRs, rebase onto current main, and review a focused validator that either integrates with existing compaction checkpoint snapshots or clearly documents why validation remains diagnostic-only. Reset delivery should stay separate and require explicit maintainer product/security review.
