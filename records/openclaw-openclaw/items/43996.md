## Summary

Keep open. Current main still unconditionally applies Docker `no-new-privileges` for sandbox containers, does not expose a typed opt-out or fallback retry, and PR #70229 remains the open linked implementation candidate rather than a merged fix.

## What This Changes

The issue requests a way for Docker sandbox sessions to work on hosts where Docker `--security-opt no-new-privileges` makes even `/usr/bin/sleep` fail, via an opt-out, fallback retry, or clearer diagnostics.

## Best Possible Solution

Land or replace PR #70229 with a secure default-on `sandbox.docker.noNewPrivileges` setting that is wired through config types, schema, resolution, Docker args, dangerous-config reporting, docs, and regression tests, then close this issue after the fix merges and ships.
