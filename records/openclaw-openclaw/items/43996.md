## Summary

Keep open. Current main still applies Docker `no-new-privileges` unconditionally for sandbox containers, and PR #70229 is an open, unmerged implementation candidate that specifically links this issue as its fix.

## What This Changes

The issue requests a Docker sandbox compatibility path for hosts where `--security-opt no-new-privileges` makes even `/usr/bin/sleep` fail with EPERM, via an opt-out, fallback retry, or clearer diagnostic.

## Best Possible Solution

Land or replace PR #70229 with a default-on Docker sandbox `noNewPrivileges` setting threaded through config types, strict schema, config resolution, Docker create args, dangerous-config/security audit surfaces, docs, and regression tests; close this after the fix merges and ships.
