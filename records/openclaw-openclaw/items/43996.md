## Summary

Keep open. Current main still applies Docker `no-new-privileges` unconditionally for sandbox containers and has no opt-out, retry fallback, or targeted diagnostic for the reported EPERM startup failure; PR #70229 is the open, unmerged implementation candidate that links this issue as its fix.

## What This Changes

The issue asks for a Docker sandbox escape hatch, fallback, or diagnostic when `--security-opt no-new-privileges` makes sandbox containers exit with `exec /usr/bin/sleep: operation not permitted`.

## Best Possible Solution

Land or replace PR #70229 with a default-on Docker sandbox `noNewPrivileges` setting threaded through config types, strict schema, config resolution, Docker create args, dangerous-config/security audit surfaces, docs, and regression tests, then close this issue after the fix merges and ships.
