## Summary

Keep open. Current main still hard-codes Docker `--security-opt no-new-privileges` for sandbox containers, has no accepted `sandbox.docker.noNewPrivileges` opt-out or fallback retry path, and the provided thread context shows open PR #70229 explicitly targets this issue with closing syntax but has not merged.

## Best Possible Solution

Keep this issue open while PR #70229 remains open. Review that PR as the implementation candidate: it should add a default-on `sandbox.docker.noNewPrivileges` or equivalent explicit override, update schema/types/docs/security reporting, and add regression coverage for both default hardening and the disabled override. After a vetted fix lands on main and ships, this issue can close as implemented.

## Risks / Open Questions

- Disabling Docker `no-new-privileges` weakens sandbox hardening, so the fix should keep the secure default and make any opt-out explicit in config, docs, and security/audit reporting.
- A silent automatic fallback would reduce sandbox isolation without clear operator consent; an explicit break-glass config path is safer.
