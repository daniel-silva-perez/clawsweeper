## Summary

Keep open. Current main still unconditionally adds Docker `no-new-privileges` to sandbox containers, has no typed opt-out or fallback retry, and the linked implementation PR remains open rather than merged.

## What This Changes

The issue requests a compatibility path for Docker sandbox hosts where `--security-opt no-new-privileges` makes sandbox containers exit immediately with `exec /usr/bin/sleep: operation not permitted`.

## Best Possible Solution

Land or replace PR #70229 with a secure default-on `sandbox.docker.noNewPrivileges` setting wired through config types, schema, resolution, Docker args, dangerous-config reporting, docs, and regression tests; close this issue only after that fix merges and ships.

## Reproduction Assessment

Yes. The reporter provided a minimal Docker command that isolates the EPERM to `--security-opt no-new-privileges`, and current source inspection shows OpenClaw still always applies that flag to sandbox containers.

## Solution Assessment

Yes, the narrow maintainable direction is an explicit default-on break-glass config knob with audit/docs coverage. A silent fallback retry would be riskier because it could weaken sandbox hardening without operator intent.
