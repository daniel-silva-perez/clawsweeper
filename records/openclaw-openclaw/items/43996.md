## Summary

Keep open. Current main still applies Docker `no-new-privileges` unconditionally to sandbox containers, the accepted config/schema surface has no opt-out or fallback path, and open PR #70229 is linked with closing syntax as the active implementation candidate rather than evidence that this issue can close.

## What This Changes

The issue requests a safe compatibility path for hosts where Docker `--security-opt no-new-privileges` causes sandbox containers to fail immediately with `exec /usr/bin/sleep: operation not permitted`.

## Best Possible Solution

Review, refresh, and land or replace PR #70229 with a secure default-on Docker sandbox setting wired through config types, strict schema, config resolution, Docker create args, dangerous-config reporting, docs, and regression tests; close this issue only after that fix merges and ships.

## Reproduction Assessment

Yes. The reporter provided a minimal `docker run --security-opt no-new-privileges openclaw-sandbox:bookworm-slim /usr/bin/sleep 1` reproduction, and source inspection confirms OpenClaw still applies that flag to Docker sandbox containers on current main.

## Solution Assessment

Yes. An explicit default-on break-glass configuration knob with audit/docs coverage is the narrowest maintainable fix; a silent fallback retry would reduce sandbox hardening without clear operator intent.
