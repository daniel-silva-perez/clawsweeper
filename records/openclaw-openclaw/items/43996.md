## Summary

Keep open. Current main and the latest release still unconditionally pass Docker `no-new-privileges` to sandbox containers, there is no accepted config/schema/audit opt-out, and the linked implementation PR remains open and unmerged.

## What This Changes

The issue requests a compatibility path for Docker hosts where sandbox containers fail to exec when OpenClaw applies `--security-opt no-new-privileges`.

## Best Possible Solution

Land or replace PR #70229 with a secure default-on Docker sandbox setting wired through config types, strict schema, config resolution, Docker create args, dangerous-config reporting, docs, and regression tests; close this issue only after that fix merges and ships.

## Reproduction Assessment

Yes. The reporter provided a minimal `docker run --security-opt no-new-privileges ... /usr/bin/sleep 1` reproduction, and current source inspection shows OpenClaw still applies that flag to every Docker sandbox create.

## Solution Assessment

Yes. The safest maintainable direction is an explicit default-on break-glass config knob with audit/docs coverage; a silent fallback retry would make sandbox hardening weaker without clear operator intent.
