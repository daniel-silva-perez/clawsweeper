## Summary

Keep open. Current main still has the reported UX gap: the bundled GitHub skill requires `gh`, skill eligibility is snapshot-based, and the dependency install success paths still surface generic `Installed` messages without clear new-session or `openclaw gateway restart` guidance. The prior ClawSweeper comment remains accurate after checking the current source, docs, UI forwarding path, onboarding path, and tests.

## Best Possible Solution

Keep this issue open and make a narrow skills dependency UX fix: add explicit GitHub CLI setup/restart guidance, make successful CLI/bin dependency installs tell users to start a new session and use `openclaw gateway restart` when the binary is still missing, update the public skills docs, and add targeted tests for the shared installer, Gateway forwarding, UI message handling, and onboarding success path. Leave automatic restart as a separate maintainer product decision.
