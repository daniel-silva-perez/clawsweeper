## Summary

Keep open. Current main still has the same UX gap: the bundled GitHub skill declares and can install `gh`, but neither the GitHub skill setup text nor the gateway-backed install success path clearly tells users to start a new session or restart the gateway when a newly installed/authenticated CLI is not visible yet.

## Best Possible Solution

Keep this issue open and fix the skills dependency UX path: add explicit GitHub CLI setup guidance, make gateway-backed skill dependency install completion mention starting a new session and using `openclaw gateway restart` when the CLI is still missing, and cover the message/docs behavior with targeted tests. Auto-restart can remain a separate product decision.
