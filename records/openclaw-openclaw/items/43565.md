Keep open. Current main still does not implement `tools.fs.allowedRoots` or an equivalent multi-root filesystem policy for `read`/`write`/`edit`/`apply_patch`. The linked feature issue #33635 is closed and the earlier attempt #33708 was closed unmerged, so this PR remains the active implementation candidate rather than cleanup-closable stale work.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate. Maintainers should either review and land this implementation after any needed cleanup, or replace it with an equivalent core implementation covering config types/schema/help, read/write/edit, sandbox mode, media and prompt-image loading, and `apply_patch`, with regression tests for tilde expansion, relative-root rejection, symlink/hardlink escapes, root-target writes/deletes, workspaceOnly gating, and sandbox bind-mount behavior.
