## Summary

Keep open. Current main still lacks `tools.fs.allowedRoots`, and this contributor branch is the active implementation, but the proposed patch still has blocking filesystem-boundary regressions around workspace-only `apply_patch`, sandbox mount path resolution, and default apply_patch availability.

## What This Changes

This branch adds a `tools.fs.allowedRoots` config surface and threads extra filesystem roots through config schema/help/types, sandbox path checks, host read/write/edit tools, apply_patch, media loading, and regression tests.

## Best Possible Solution

Land a corrected, rebased implementation that routes all filesystem-tool root decisions through one canonical policy path, keeps pinned root-contained helpers for workspace-only mutations, resolves sandbox mount paths before allowed-root matching, preserves existing apply_patch defaults, and aligns config help/docs/tests with the final contract.

## Reproduction Assessment

Yes. Source comparison against current main and the PR head gives a high-confidence reproduction path: main lacks the config, while PR head shows the unpinned delete, pre-bridge allowed-root check, and changed apply_patch default at concrete files/lines.

## Solution Assessment

No. The feature direction is useful, but this branch should not merge until it restores pinned deletion, performs allowed-root checks after sandbox bridge/mount resolution, and preserves default apply_patch availability.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Restore pinned deletion for apply_patch deletes:** `src/agents/apply-patch.ts:311`
  - body: The workspace-only delete path validates with `assertSandboxPath()` and then calls `fs.rm()` on `sandboxResult.resolved`. Current main uses `removePathWithinRoot()`, so this reintroduces a check-then-use mutation path that can be redirected outside the matched root if a parent is swapped between validation and deletion.
  - confidence: 0.9
- **[P2] Resolve mounted sandbox paths before allowed-root checks:** `src/agents/pi-tools.read.ts:601-609`
  - body: In sandbox runs this wrapper compares the container path against host-configured `additionalRoots` before `SandboxFsBridge.resolvePath` maps bind mounts. A bind such as `/host/shared:/shared` with `allowedRoots: ["/host/shared"]` still rejects `/shared/file`, so the new setting fails for the Docker bind-mount case.
  - confidence: 0.87
- **[P2] Preserve default apply_patch enablement:** `src/agents/pi-tools.ts:360-361`
  - body: This branch requires `tools.exec.applyPatch.enabled` to be explicitly true before creating `apply_patch`. Current main keeps `apply_patch` available for OpenAI models unless it is explicitly disabled, so merging this would silently remove the tool from normal coding profiles with no `applyPatch` config.
  - confidence: 0.86
