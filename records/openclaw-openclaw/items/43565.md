## Summary

Keep open. Current main still lacks `tools.fs.allowedRoots`, so the PR is not obsolete, but the PR head is not mergeable because it regresses a pinned filesystem mutation path, breaks sandbox bind-mount allowlist behavior, and changes default `apply_patch` availability.

## What This Changes

This PR adds `tools.fs.allowedRoots` and threads additional filesystem roots through config schema/help, sandbox path guards, read/write/edit tools, media loading, `apply_patch`, and regression tests.

## Best Possible Solution

Land a corrected, rebased version that keeps a single filesystem policy path, uses pinned root-contained helpers for every workspace-only mutation, compares allowed roots after sandbox mount resolution, preserves current `apply_patch` defaults, and keeps config, help text, docs, and tests aligned.

## Reproduction Assessment

Yes. Source inspection against current main and PR head gives a high-confidence reproduction path: main lacks the config, and the PR regressions are visible in the changed lines for delete, sandbox wrapper validation, and apply_patch enablement.

## Solution Assessment

No. The feature direction is maintainable, but this branch should not merge until it restores pinned deletion, resolves sandbox paths through the bridge or mount table before matching allowed roots, and preserves current apply_patch default enablement.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Restore pinned removal for apply_patch deletes:** `src/agents/apply-patch.ts:311`
  - body: The workspace-only delete path validates with `assertSandboxPath()` and then calls `fs.rm()` on the resolved string. Current main uses `removePathWithinRoot()`, so this reintroduces a check-then-use mutation path that can be redirected outside the matched root if a parent is swapped between validation and deletion. Use the pinned removal helper with `sandboxResult.matchedRoot` and `sandboxResult.relative`.
  - confidence: 0.9
- **[P2] Resolve sandbox paths before comparing allowed roots:** `src/agents/pi-tools.read.ts:601`
  - body: In sandbox runs, this wrapper checks the container path against host-configured `additionalRoots` before `SandboxFsBridge.resolvePath` maps bind mounts. A bind such as `/host/shared:/shared` with `allowedRoots: ["/host/shared"]` still rejects `/shared/file`, so the new setting fails for the Docker bind-mount case.
  - confidence: 0.87
- **[P2] Preserve default apply_patch enablement:** `src/agents/pi-tools.ts:360-361`
  - body: This branch requires `tools.exec.applyPatch.enabled` to be explicitly true before creating `apply_patch`. Current main keeps `apply_patch` available for OpenAI models unless it is explicitly disabled, so merging this would silently remove the tool from normal coding profiles with no `applyPatch` config.
  - confidence: 0.86
