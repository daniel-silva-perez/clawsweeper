## Summary

Keep open. Current main still does not implement `tools.fs.allowedRoots`, and this PR is the active implementation candidate, but the proposed filesystem-boundary expansion has a concrete sandbox bind-mount correctness gap that should be fixed and reviewed before merge.

## What This Changes

This PR adds a `tools.fs.allowedRoots` config field and threads it through filesystem, sandbox, media, prompt-image, and `apply_patch` path checks with regression tests.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate, fix the sandbox bind-mount policy gap by resolving through the sandbox bridge or mount table before host-root checks, rebase onto current main, and validate all touched filesystem/media/apply_patch surfaces before maintainer review.

Full review comments:

- **[P2] Validate allowed roots after sandbox mount resolution:** `src/agents/pi-tools.read.ts:601`
  - body: In sandbox runs, this wrapper checks the raw/container path against host-configured `additionalRoots` before `SandboxFsBridge.resolvePath` can map Docker bind mounts. Resolve through the bridge/mount table first, or compare against the corresponding container roots.
