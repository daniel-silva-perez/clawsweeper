## Summary

Holding this open: the issue has an explicit maintainer signal to keep it open after a partial fix, and current `main` still has install-path code/tests proving only the durable `.env` / config-service env projection class is covered, while the broader macOS LaunchAgent SecretRef/plaintext vectors remain unproven.

## Best Possible Solution

Keep this issue open as the umbrella for macOS LaunchAgent install-path hardening. The best path is to finish the remaining implementation by preventing all supported SecretRef-backed and secret-bearing provider/channel/plugin/auth-profile values from being serialized into LaunchAgent metadata, loading them through a launchd-safe runtime source, repairing legacy inline copies, and adding regression coverage for `gateway install --force`, update/npm reinstall, ENV/FILE/structured refs, provider/channel/plugin secrets, and gateway-token behavior. Coordinate narrower related work such as #67595 and the service-identity discussion in #69066 rather than closing this prematurely.
