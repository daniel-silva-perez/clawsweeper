## Summary

Keep open. Current main still lacks native Anthropic advisor support, and this PR is useful source work, but it is not merge-ready: the advertised shorthand uses an advisor model that Anthropic's current compatibility table rejects for the documented Sonnet executor case, and the implementation does not preserve advisor result blocks for multi-turn replay despite the API contract requiring verbatim round-tripping.

## What This Changes

The PR adds Anthropic advisor-tool beta header/tool injection, server-side tool-result display handling, stop-reason hardening, docs, live validation scripts, and focused tests for the Anthropic transport and wrapper paths.

## Best Possible Solution

Rebase the PR onto current main, align the default and examples with Anthropic's supported executor/advisor pairs, preserve server-side advisor result blocks through transcript storage/replay and pruning, then validate with targeted Anthropic wrapper/transport tests plus the changed gate before using it to close the remaining advisor tracker.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Use a valid default advisor model:** `extensions/anthropic/stream-wrappers.ts:225`
  - body: `params.advisor: true` currently injects `claude-sonnet-4-6` as the advisor model. Anthropic's advisor compatibility table requires a Sonnet 4.6 executor to use Opus 4.7 as the advisor, and invalid pairs return `400 invalid_request_error`, so the advertised shorthand config will fail for the documented common case. Default to a compatible advisor or require users to choose one explicitly.
  - confidence: 0.91
- **[P2] Round-trip advisor result blocks:** `src/agents/anthropic-transport-stream.ts:283-290`
  - body: The replay path drops `serverToolDisplay` blocks or turns them into plain text fallback, so `advisor_tool_result` and `advisor_redacted_result.encrypted_content` are never sent back verbatim on later turns. Anthropic documents these blocks as part of the multi-turn message history, and the encrypted variant must be round-tripped for the server to reconstruct the advisor advice, so display-only storage loses required advisor state.
  - confidence: 0.88
