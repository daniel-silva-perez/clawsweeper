## Summary

Keep open. Current main still lacks native Anthropic advisor support, and this PR is useful source work, but it is not merge-ready because the advertised shorthand uses an invalid executor/advisor pairing and the implementation drops required advisor result state on multi-turn replay.

## What This Changes

The PR adds Anthropic advisor-tool beta header/tool injection, server-side tool-result display handling, stop-reason hardening, docs, live validation scripts, and focused Anthropic transport/wrapper tests.

## Best Possible Solution

Rebase the PR onto current main, align the default and examples with Anthropic's supported executor/advisor pairs, preserve raw server-side advisor result blocks through transcript storage/replay and pruning, then validate the Anthropic wrapper/transport/chat-content tests plus the changed gate before using it to close the advisor tracker.

## Reproduction Assessment

Yes. The main-branch absence is reproducible with targeted `rg` searches and source inspection, and both PR defects are reproducible by reading the PR head: `params.advisor: true` resolves to `claude-sonnet-4-6`, and replay converts or skips `serverToolDisplay` text instead of preserving `advisor_tool_result` blocks.

## Solution Assessment

No. The PR is aimed at the right product boundary, but the current diff is not the best fix because it ships an invalid default pairing and a display-only server-tool representation where Anthropic requires raw advisor blocks to be replayed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Use a valid default advisor model:** `extensions/anthropic/stream-wrappers.ts:225`
  - body: `params.advisor: true` injects `claude-sonnet-4-6` as the advisor model, but Anthropic's compatibility table requires a Sonnet 4.6 executor to use Opus 4.7 as the advisor. The documented shorthand will therefore fail with `400 invalid_request_error`; default to a compatible advisor or require users to choose one explicitly.
  - confidence: 0.91
- **[P2] Round-trip advisor result blocks:** `src/agents/anthropic-transport-stream.ts:283-290`
  - body: The replay path drops `serverToolDisplay` blocks or falls back to plain text, so `advisor_tool_result` and `advisor_redacted_result.encrypted_content` are never sent back verbatim on later turns. Anthropic documents these blocks as part of multi-turn history, so display-only storage loses required advisor state.
  - confidence: 0.88
