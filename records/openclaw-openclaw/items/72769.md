## Summary

Keep open. The PR adds a useful exec-approval audit trail, but the current diff still exposes historical approval data through a broad read scope, uses narrower custom argv redaction than the existing tool redaction path, and reads the append-only audit file synchronously from a gateway RPC.

## What This Changes

The PR adds an append-only exec approval decision JSONL audit log, an `exec.approval.auditLog` gateway method with filters, protocol registration, and custom persisted-argument redaction.

## Best Possible Solution

Keep the feature direction, but revise it so approval audit reads require `operator.approvals` or stronger, persisted argv uses the shared tool redaction coverage, log reads are bounded and nonblocking, and focused tests cover approve, deny, restart persistence, filters, scope denial, malformed lines, and argv redaction cases.

## Reproduction Assessment

Yes. The access-control issue is reproducible by checking that the PR authorizes `exec.approval.auditLog` with only `operator.read`; the redaction issue is reproducible by passing `--token secret`, `--password=secret`, lower-case `token=secret`, or bearer-header argv entries through the PR redactor.

## Solution Assessment

No. Persisting approval decisions is a reasonable additive gateway feature, but this implementation is not the narrowest safe version until access control, redaction reuse, and bounded log reads are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Gate audit-log reads with approval scope:** `src/gateway/method-scopes.ts:109`
  - body: Approval request/get/list/resolve methods and approval broadcasts require `operator.approvals`, but this adds the historical audit query to `READ_SCOPE`. That lets a read-scoped token retrieve past command text, args, session keys, and resolver IDs; move this method to `APPROVALS_SCOPE` or stronger.
  - confidence: 0.9
- **[P1] Reuse existing redaction for argv secrets:** `src/infra/exec-approval-audit.ts:35-40`
  - body: The array redactor only masks uppercase `KEY=value` tokens. Common `commandArgv` forms such as `--token secret`, `--password=secret`, lower-case `token=secret`, and bearer-header arguments will be persisted in plaintext; reuse the existing tool redaction path or equivalent tested coverage before writing argv.
  - confidence: 0.88
- **[P2] Avoid synchronous full-file reads in the gateway RPC:** `src/infra/exec-approval-audit.ts:90`
  - body: `readFileSync` loads and parses the complete append-only audit log before applying `limit`. A long-lived gateway can accumulate a large file, and one `exec.approval.auditLog` call would block the event loop; use an async bounded or streaming read path instead.
  - confidence: 0.82
