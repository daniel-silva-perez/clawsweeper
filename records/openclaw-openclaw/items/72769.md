## Summary

Keep open. Current main does not have persisted exec-approval decision auditing, but this PR is not merge-ready because it weakens approval-data access control, leaves common argv secret forms unredacted, and adds an unbounded synchronous log read path.

## What This Changes

The PR adds an append-only exec approval decision JSONL audit log, a new `exec.approval.auditLog` gateway RPC with filters, protocol params, method registration, and custom persisted-argument redaction.

## Best Possible Solution

Keep the feature direction, but revise it so persisted approval decisions reuse the existing tool secret-redaction path, historical audit reads require `operator.approvals` or stronger, the log query is bounded and nonblocking, protocol/docs are updated, and focused tests cover approve, deny, restart persistence, filters, scope denial, malformed lines, and argv redaction cases.

## Reproduction Assessment

Yes. A focused unit test can prove the access-control issue by checking `authorizeOperatorScopesForMethod("exec.approval.auditLog", ["operator.read"])`, and another can call the redactor with `--token=...`, `--password value`, and lower-case `token=...` argv entries to show they persist unchanged.

## Solution Assessment

No. Persisting approval decisions is a reasonable additive gateway feature, but this implementation is not the narrowest safe version until access control, redaction reuse, and bounded log reads are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Gate audit-log reads with approval scope:** `src/gateway/method-scopes.ts:110`
  - body: Approval requests, resolves, and approval lifecycle events are currently gated by `operator.approvals`, but this adds the historical approval audit query under `operator.read`. A read-only token would be able to retrieve past command text, argv, session keys, and resolver IDs, so this should require `operator.approvals` or a stronger scope.
  - confidence: 0.9
- **[P1] Reuse existing redaction for argv secrets:** `src/infra/exec-approval-audit.ts:35-40`
  - body: The array redactor only handles uppercase `KEY=value` entries. Common argv forms such as `--token=secret`, `--password secret`, lower-case `token=secret`, and bearer-header arguments remain in plaintext even though the audit file persists `commandArgv`; reuse the existing tool redaction coverage or equivalent tests before writing argv to disk.
  - confidence: 0.88
- **[P2] Avoid synchronous full-file reads in the gateway RPC:** `src/infra/exec-approval-audit.ts:85`
  - body: `readFileSync` loads and parses the full append-only audit log before applying `limit`. A long-lived gateway can accumulate a large file, and one `exec.approval.auditLog` call would block the event loop; use an async bounded or streaming read path instead.
  - confidence: 0.82
