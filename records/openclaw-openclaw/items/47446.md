## Summary

Keep open. The PR is aimed at a real Discord/proxy gap, but current main does not clearly implement the intended Discord WebSocket env-proxy fallback and the branch still has merge-blocking correctness and security issues around plugin package boundaries, NO_PROXY handling, and undici proxy option placement.

## What This Changes

The PR adds env-proxy fallback for Discord gateway and REST traffic, installs an EnvHttpProxyAgent during gateway startup, changes proxy-agent keep-alive options, updates focused proxy tests, and includes small gateway node-command refactors.

## Best Possible Solution

Revise the branch into a narrow Discord proxy fix that uses public plugin SDK exports, chooses env proxies with target-aware NO_PROXY checks for Discord API and Gateway hosts, configures undici proxy tunnel policy through supported proxyTls/requestTls option surfaces, rebases onto current main, and leaves unrelated gateway node-command refactors out.

## Reproduction Assessment

Yes for the merge-blocking patch defects: they reproduce by diff and source inspection against the extension boundary rules, the proxy-env helper contract, and undici v8.1.0 source. The original live Discord proxy failure is still unclear because the PR's manual local-proxy verification remains unchecked.

## Solution Assessment

No. The direction is useful, but this is not the best implementation because it bypasses the public plugin boundary, applies env proxy fallback without target-aware NO_PROXY handling, and places the proxy tunnel keep-alive policy on unsupported undici option paths.
