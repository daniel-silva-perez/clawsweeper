## Summary

Keep open. Current main has already implemented the gateway-wide undici env-proxy bootstrap, but it has not implemented this PR's Discord-specific ambient proxy fallback or its claimed local-proxy ECONNRESET mitigation. The branch is useful source work, but it should not merge as-is because the diff crosses the plugin/core package boundary and configures undici proxy tunnel behavior through unsupported option shapes.

## What This Changes

The PR adds gateway and Discord proxy-env handling, attempts to disable proxy keep-alive for undici proxy agents, updates focused proxy tests, and includes small gateway command-path refactors.

## Best Possible Solution

Keep the PR open for maintainer handling, but require a narrow revision or replacement that implements Discord ambient proxy support through public plugin SDK exports, gates each Discord target with NO_PROXY-aware selection, configures undici proxy tunnel policy through supported proxy transport options, and drops unrelated gateway command-path refactors.

## Reproduction Assessment

Yes for the merge-blocking review findings: the package-boundary violation is visible directly in the PR diff, and the proxy option issue follows from undici 8.1.0 docs/source plus the changed constructor calls. A live Discord proxy reproduction for the original user problem is still missing.

## Solution Assessment

No, this is not the best way to solve the issue as written. The direction is useful, but the maintainable fix should use openclaw/plugin-sdk/fetch-runtime, shouldUseEnvHttpProxyForUrl-style NO_PROXY checks for Discord REST and gateway hosts, and undici's supported proxyTls/requestTls option surfaces.
