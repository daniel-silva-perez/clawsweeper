## Summary

Keep this PR open. Current main still does not auto-switch the final agent reply run from a text-only selected/default model to `agents.defaults.imageModel` when image input is present. The PR is aimed at a real gap linked from #38796, but it still needs maintainer-quality follow-up for the review concerns around detection, sequencing, capability certainty, URL-only media, and fallback-only image model config.

## Best Possible Solution

Keep this PR open for maintainer review or author follow-up. The production fix should live in the reply/model-selection path, detect normalized inbound image inputs, resolve imageModel primary and fallbacks through the real alias/catalog data, switch the final `runPreparedReply` model only when the active model is positively known to lack image support, respect explicit heartbeat/session/channel choices, avoid redundant media-understanding work, and add regression coverage proving the final agent run receives the configured vision model.
