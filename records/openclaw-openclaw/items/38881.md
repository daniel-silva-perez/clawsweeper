## Summary

Keep this PR open. Current main still does not auto-switch the final agent reply run from a text-only selected/default model to `agents.defaults.imageModel` when image input is present. Main has separate media-understanding routing through `agents.defaults.imageModel`, but the final `runPreparedReply` path still receives the active provider/model and then uses normal model fallback, not image-model fallback. The PR addresses a real gap linked from #38796, but it still needs maintainer-quality follow-up for the outstanding functional review concerns before merge.

## Best Possible Solution

Keep this PR open for maintainer review or author follow-up. The production fix should live in the reply/model-selection path, detect normalized image attachments, resolve `agents.defaults.imageModel` primary and fallbacks through the real alias/catalog helpers, switch the final `runPreparedReply` model only when the active model is positively known not to accept images, respect explicit heartbeat/session/channel model choices, avoid duplicate media-understanding work, and add focused regression coverage proving the final agent run receives the configured vision model.
