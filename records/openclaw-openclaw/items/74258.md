## Summary

Keep open. Current main still masks the OpenAI Codex device-pairing code in remote mode, and this PR is a narrow, valid fix that surfaces the code through the interactive prompter while preserving the persistent runtime-log boundary. ## What This Changes

The PR changes the OpenAI Codex provider to always include the OAuth device user code in the interactive prompter note, and refactors provider tests to assert the prompter shows the code while `runtime.log` omits it in remote mode.

## Best Possible Solution

Once merged, #74212 can close and #74221 can remain superseded.

Yes. Current main has a direct source-level reproduction: invoke the device-code auth method with `isRemote: true` and the prompter note receives the placeholder instead of `Code: <userCode>`; the existing unit test asserts that same broken behavior.

## Solution Assessment

Yes, with one process fix. Showing the device code only on the interactive prompter is the narrowest maintainable fix because it makes SSH/headless auth usable without persisting the code in `runtime.log`; the PR should add the required changelog entry before merge.
