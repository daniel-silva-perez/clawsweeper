## Summary

Keep open. Current main still lacks the cron-run enqueue/wake key remapping this PR proposes, and the linked discussion narrowed the remaining value to the #52305 session-key targeting path rather than the already-shipped delivery-target fixes.

## What This Changes

The PR adds cron-run session-key remapping via `resolveEventSessionKey`, extends `scopedHeartbeatWakeOptions`, threads the remap through async exec, CLI watchdog, ACP stream, and gateway node exec-event paths, and adds session-key/classification tests.

## Best Possible Solution

Land a slim fix that keeps enqueue-side and wake-side session keys aligned for cron-run async completion events, preserves the intentional `heartbeat.target: "none"` behavior, and tracks transport/global-scope concerns separately if maintainers want those solved.

## Reproduction Assessment

Yes for the session-key mismatch: current code shows system events are queued by exact session key while heartbeat preflight peeks one resolved queue, and the exec/ACP/CLI/gateway paths currently enqueue/wake without cron-run remapping. Live Telegram/websocket reproduction was not repeated in this read-only review.

## Solution Assessment

Yes for the narrowed #52305 session-targeting fix: centralizing cron-run remapping and threading the configured `mainKey` through the affected enqueue/wake call sites is the narrow maintainable direction. It should not reintroduce the dropped target-none delivery override in this PR.
