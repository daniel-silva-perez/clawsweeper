## Summary

Keep open. Current main does not implement the PR's central cron-run session remap, and the PR remains a plausible #52305 fix candidate, but the latest diff still leaves enqueue and wake keys mismatched for custom `session.mainKey` setups in the gateway node-event and ACP stream paths. The discussion shows useful tightening after prior review, including cron-session isolation feedback, but this branch still needs author or maintainer revision before merge.

## What This Changes

The branch adds `resolveEventSessionKey`, extends `scopedHeartbeatWakeOptions` for cron-run session remapping, applies the remap across bash exec, CLI watchdog, ACP stream, and gateway node-event paths, and adds session-key plus exec-event classification tests.

## Best Possible Solution

Keep this PR as the narrow #52305 implementation candidate, but compute or load the relevant `mainKey` once in each affected path and pass the same value to both event enqueue remapping and heartbeat wake scoping, with custom-main-key cron-run regression coverage before the changed gate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Pass the configured main key to node-event wakes:** `src/gateway/server-node-events.ts:686`
  - body: This enqueue now remaps cron-run events with `cfg.session?.mainKey`, but the matching wake below still calls `scopedHeartbeatWakeOptions(sessionKey, ...)` without that value. With a custom `session.mainKey`, the event can be queued under `agent:<id>:<custom>` while the wake targets `agent:<id>:main`, leaving heartbeat on the wrong queue.
  - confidence: 0.91
- **[P2] Keep ACP enqueue and wake remaps aligned:** `src/agents/acp-spawn-parent-stream.ts:198`
  - body: The ACP stream enqueue path loads `session.mainKey`, but `wake()` still remaps `parentSessionKey` without the same value. Custom-main-key cron-run sessions can enqueue ACP progress under the configured main key and wake the default `:main` queue, so the event remains invisible to heartbeat.
  - confidence: 0.89
