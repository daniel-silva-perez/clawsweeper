## Summary

Keep open. Current main still lacks the cron-run session-key remap this PR is trying to add for #52305, but the slimmed diff has two concrete custom-main-key enqueue/wake mismatches that can still drop events before merge.

## What This Changes

This PR adds cron-run-to-agent-main session-key remapping for async exec, ACP stream, CLI watchdog, and gateway node-event paths, plus session-key and exec-completion classification tests.

## Best Possible Solution

Keep this PR as the narrow #52305 implementation candidate, align enqueue and wake remapping by threading the same mainKey through the gateway node-event and ACP stream wake paths, refresh the stale description, and land only after targeted tests plus changed-gate validation are green.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Pass mainKey to the node exec wake remap:** `src/gateway/server-node-events.ts:686`
  - body: This enqueue now uses `cfg.session?.mainKey`, but the matching `requestHeartbeatNow(scopedHeartbeatWakeOptions(...))` call below still omits that main key. For a cron-run session with `session.mainKey: "captain"`, the event is queued under `agent:<id>:captain` while the wake targets `agent:<id>:main`, so heartbeat drains the wrong queue and the exec completion can still be lost.
  - confidence: 0.9
- **[P2] Keep ACP enqueue and wake keys aligned:** `src/agents/acp-spawn-parent-stream.ts:198`
  - body: The ACP stream enqueue path now remaps with `loadConfig().session?.mainKey`, but `wake()` still calls `scopedHeartbeatWakeOptions(parentSessionKey, ...)` without that same value. Custom-main-key cron runs will enqueue ACP progress under the configured main key and wake the default `:main` key, leaving the queued event invisible to heartbeat.
  - confidence: 0.88
