## Summary

Keep this PR open. Current main still has the older memory-core fallback that relaxes to the configured text weight, while the PR proposes deriving fallback retention from already-merged keyword-backed scores after temporal decay/MMR and adds regression coverage that is not present on main.

## Best Possible Solution

Keep this PR open. Port the intended fallback into `extensions/memory-core/src/memory/manager.ts`, derive relaxed retention from already-merged keyword-backed scores after temporal decay/MMR, add focused memory-core tests for BM25 `textScore < 1` and temporal-decay fallback behavior, and resolve the remaining max-score recall concern before landing.
