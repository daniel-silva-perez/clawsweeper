## Summary

Keep this PR open. Current main still has the older memory-core hybrid fallback that relaxes to the configured text weight, while this PR’s useful behavior is to derive fallback retention from already-merged keyword-backed entries after temporal decay/MMR. The work is not implemented on main and needs a current-layout port plus focused regression coverage.

## Best Possible Solution

Keep this PR open. Port the fallback into `extensions/memory-core/src/memory/manager.ts`, derive the relaxed fallback threshold from already-merged keyword-backed entries after temporal decay/MMR, add active memory-core regression tests for BM25 textScore below 1.0 and temporal-decay fallback behavior, and resolve the remaining recall-vs-noise choice for multiple keyword-backed fallback candidates.

## Risks / Open Questions

- The PR is stale against the current `extensions/memory-core` layout and cannot land as-is without a rebase or manual port.
