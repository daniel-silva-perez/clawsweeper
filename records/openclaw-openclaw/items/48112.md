## Summary

Keep this PR open. Current main still has the older memory-core hybrid fallback that relaxes to the configured text weight, not the PR's post-merge keyword-backed threshold, and current regression coverage for this specific fallback is still absent or skipped. The PR is useful but needs to be ported from the old `src/memory` layout into `extensions/memory-core` and should resolve the remaining recall-threshold review concern before landing.

## Best Possible Solution

Keep this PR open. Port the fallback into `extensions/memory-core/src/memory/manager.ts`, derive retention from already-merged keyword-backed entries after temporal decay/MMR, add focused memory-core regression tests for BM25 sub-1.0 and temporal-decay fallback behavior, and resolve whether fallback should return only the strongest score group or all keyword-backed candidates up to `maxResults`.

## Risks / Open Questions

- The PR is stale against the current `extensions/memory-core` layout and will need a rebase or manual port before it can land.
- The last review concern is still substantive: deriving the relaxed floor from the maximum keyword-backed merged score may keep only the top-score tie group, so maintainers should decide the intended fallback recall behavior.
- Changing fallback scoring can affect memory recall noise, so the port should include focused tests for BM25 `textScore < 1`, temporal decay, and multiple keyword-backed fallback candidates.
