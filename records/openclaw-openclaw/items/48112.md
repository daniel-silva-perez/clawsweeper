## Summary

Keep this PR open. Current main still uses the older text-weight fallback in memory hybrid search and does not implement this PR’s post-merge keyword-backed threshold behavior or its regression coverage. The PR discussion also shows a real review loop: the temporal-decay concern was addressed on the PR branch, but a later recall concern still needs maintainer review before landing.

## Best Possible Solution

Keep this PR open, rebase/port the intended behavior into `extensions/memory-core/src/memory/manager.ts`, derive fallback retention from already-merged keyword-backed scores after temporal decay/MMR, add focused regression tests for BM25 `textScore < 1` and temporal decay in the current memory-core test layout, then review the remaining max-score recall concern before landing.
