## Summary

Keep open: current main still mitigates replay/display bloat rather than fixing transcript storage. Inbound Discord media is saved to disk and surfaced as media context, but the session persistence path still allows user image content blocks with raw base64 to reach `SessionManager.appendMessage`.

## What This Changes

The issue requests storing Discord inbound image references in session JSONL transcripts instead of embedding base64 image payloads, then hydrating bounded references when building model context.

## Best Possible Solution

Add a session-transcript claim-check at the shared persistence seam: persist compact media reference metadata for inbound image content, hydrate only bounded recent references during model-context construction, keep legacy inline-base64 transcripts readable, and align Gateway history display with the same representation.
