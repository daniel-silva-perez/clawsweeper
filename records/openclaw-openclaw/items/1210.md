## Summary

Keep open: current main has replay-time pruning, display redaction, and Gateway attachment offload paths, but it still lacks the requested session-transcript claim-check for inbound image content. The central persistence guard does not rewrite user `type: "image"` blocks before `SessionManager.appendMessage`, and upstream pi-coding-agent still builds user messages by appending image blocks to the text content before prompting.

## What This Changes

Store Discord and other inbound images in session transcripts as compact media references instead of inline base64, then hydrate bounded image data from disk when building model context.

## Best Possible Solution

Add a backward-compatible transcript claim-check for inbound image content: persist compact media reference and metadata instead of raw `image.data`, hydrate only bounded recent references during model-context construction, keep legacy inline-base64 transcripts readable, and align the display redaction path with #43982.
