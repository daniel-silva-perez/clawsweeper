## Summary

Current main still has only mitigation/replay cleanup for persisted image blocks, not the requested transcript-level media-reference storage for Discord images. The Discord/media path still saves inbound files, renders text media notes, rehydrates those notes into base64 image blocks for the prompt, and prunes older persisted image blocks only in replay views.

## Best Possible Solution

Keep this issue open until OpenClaw has a backward-compatible session transcript claim-check for inbound image blocks: persist compact media references/metadata instead of base64, hydrate bounded recent image refs when building model context, keep old base64 transcripts readable through existing sanitizers, and coordinate with the narrower #42941/#43982 work so gateway history redaction and agent replay do not diverge.

## Repair Work Prompt

Fix the remaining session-transcript base64 image persistence bug for inbound Discord/media images. Current main saves media files and includes `[media attached: ...]` prompt notes, then rehydrates those paths into inline `{ type: "image", data: base64, mimeType }` blocks for `activeSession.prompt`; the raw session transcript can still persist those image blocks. Implement a backward-compatible claim-check/persistence path so user image content written to PI session JSONL does not store raw base64. Preserve current-turn model vision behavior, preserve enough media reference metadata or text for later bounded hydration, and keep old transcripts with inline base64 readable through existing sanitization/pruning. Start in `src/agents/session-tool-result-guard-wrapper.ts`, `src/agents/session-tool-result-guard.ts`, `src/agents/pi-embedded-runner/run/images.ts`, `src/agents/pi-embedded-runner/replay-history.ts`, and `src/agents/pi-embedded-runner/run/history-image-prune.ts`; only touch Discord plugin code if needed for metadata propagation. Add focused regression coverage proving a user image sent through a guarded session does not persist the original base64, while model prompt submission still receives the image for the active turn. Check related #42941 for the previous unmerged persistence-redaction approach and #43982 for adjacent gateway history redaction, but do not make this a gateway-only fix. Update `docs/concepts/session-pruning.md` and `docs/reference/transcript-hygiene.md` if the raw-transcript behavior changes.
