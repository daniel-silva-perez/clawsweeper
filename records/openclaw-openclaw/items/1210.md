## Summary

Keep open: current main has mitigations for replay pruning, media-path notes, and WebChat transcript updates, but the core request is still unresolved because session JSONL user image blocks are not rewritten into durable media references before persistence.

## What This Changes

The issue requests replacing inline base64 Discord image content in session transcripts with compact media reference metadata and bounded hydration from disk.

## Best Possible Solution

Add a shared, backward-compatible transcript claim-check for inbound image content: persist compact media reference and metadata instead of raw `image.data`, hydrate only bounded recent references during model-context construction, keep legacy inline-base64 transcripts readable, and align Gateway history/export display with the compact representation.
