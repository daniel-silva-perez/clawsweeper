## Summary

Keep this PR open. Current main still does not rewrite Volta `tools/image/node/<version>/bin/node` paths to the stable Volta shim, so the linked daemon LaunchAgent fragility remains unresolved; however, the submitted implementation still has unresolved review concerns around custom `VOLTA_HOME` and Windows Volta layouts.

## Best Possible Solution

Keep this PR open and update it, or land a replacement, with Volta path resolution that handles default `~/.volta`, custom `VOLTA_HOME` roots, Windows Volta layouts, `.exe` shims, and inaccessible-shim fallback, with focused regression coverage in the stable-node/runtime path tests. Then use that implementation to resolve #52184.
