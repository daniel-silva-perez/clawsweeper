## Summary

Keep this PR open. Current main still leaves Volta image Node paths unchanged, but the proposed patch only covers the default Unix `~/.volta` shape and remains incomplete for supported custom `VOLTA_HOME` roots and Windows Volta layouts.

## What This Changes

The PR adds Volta `tools/image/node/<version>` handling to `resolveStableNodePath` plus tests for resolving to an accessible shim and falling back when the shim is missing.

## Best Possible Solution

Update this PR, or land a narrow replacement, so Volta image-path rewriting is layout-aware across default Unix, custom `VOLTA_HOME`, Windows image locations, and missing-shim fallback while keeping the existing daemon runtime selection order intact.

## Reproduction Assessment

Yes. A focused unit check can call `resolveStableNodePath` with a Volta image path and an accessible shim; current main returns the input unchanged, while this PR still reproduces the bug for `/opt/volta/tools/image/node/<version>/bin/node` and Windows Volta layouts.

## Solution Assessment

No, not as submitted. The direction is right, but the implementation should match Volta home layouts generically and handle Windows explicitly instead of hard-coding `~/.volta` plus `bin/node[.exe]`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Match Volta home layouts instead of only ~/.volta:** `src/infra/stable-node-path.ts:19`
  - body: The new regex only accepts paths containing `/.volta/tools/image/.../bin/node`, so an install with `VOLTA_HOME=/opt/volta` still returns `/opt/volta/tools/image/node/<ver>/bin/node`. OpenClaw already preserves `VOLTA_HOME/bin` in service PATH, so this supported setup remains pinned to a stale image path after Volta changes versions.
  - confidence: 0.94
- **[P2] Handle Windows Volta image and shim layout explicitly:** `src/infra/stable-node-path.ts:19-25`
  - body: The added Windows branch looks for `.../bin/node.exe` and returns `bin/node.exe`, but Volta's Windows layout uses a local-data `Volta` home, stores Node images directly under the version directory, and creates command shims under `bin`. As written, Windows Volta installs will not be rewritten to the stable shim.
  - confidence: 0.89
