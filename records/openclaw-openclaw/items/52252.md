## Summary

Keep this PR open. Current main still does not rewrite Volta tools/image Node paths to the stable Volta shim, so the PR’s intended daemon fix is not implemented on main. The submitted change remains a plausible implementation candidate for the linked Volta LaunchAgent issue, but the open review concern about custom VOLTA_HOME and Windows Volta layouts is material and should be addressed before landing.

## Best Possible Solution

Keep this PR open and update it, or land a replacement, to implement Volta tools/image path rewriting in src/infra/stable-node-path.ts with regression tests for default ~/.volta, custom VOLTA_HOME, POSIX and Windows separators, and inaccessible-shim fallback. Then review/land it as the implementation candidate for #52184.
