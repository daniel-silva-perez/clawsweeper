## Summary

Keep open. Current main still has partial cron error observability and generic backoff, but not the requested reason-aware recurring-cron guard layer: recurring jobs do not circuit-break billing/auth/format failures, do not consume Retry-After for recurring scheduling, and have no job-level cooldown/disabled guard state.

## Best Possible Solution

Keep this issue open until recurring cron has an explicit reason-aware guard layer. The likely implementation should extend `src/cron/service/timer.ts`, cron state/config types, Gateway schema, Control UI/macOS surfaces, docs, and regression tests so cron classifies terminal run errors once, persists/display guard state, respects Retry-After where available, auto-pauses or circuit-breaks recurring jobs for billing/auth/format with safe configurable defaults and one alert, and keeps timeout/network/rate-limit jobs enabled with bounded backoff coordinated with auth-profile/provider cooldowns.
