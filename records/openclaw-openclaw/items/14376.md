## Summary

Keep open. Current main has partial cron error observability and one-shot transient retry behavior, but recurring cron still lacks the requested reason-aware guard layer: no billing/auth/format circuit breaker, no Retry-After-aware recurring scheduling, and no job-level cooldown/disabled guard state.

## Best Possible Solution

Keep this issue open until recurring cron has an explicit reason-aware guard layer. The implementation should extend src/cron/service/timer.ts, cron state/config types, Gateway schema, Control UI/macOS/plugin-visible surfaces, docs, and regression tests so cron classifies terminal run errors once, persists/display guard state, respects Retry-After where available, auto-pauses or circuit-breaks recurring billing/auth/format failures with safe configurable defaults and one alert, and keeps timeout/network/rate-limit jobs enabled with bounded backoff coordinated with auth-profile/provider cooldowns.
