## Summary

Keep open. Current main has partial observability and generic retry/backoff, but it still lacks the requested recurring-cron guard policy: no reason-specific circuit breaker for billing/auth/format, no job-level cooldown/disabled guard state, no Retry-After-aware recurring scheduling, and no provider/global coordination for rate-limit cascades. The existing issue discussion also includes concrete retry-storm reports and a maintainer review that recently kept this open.

## Best Possible Solution

Keep this issue open until recurring cron has an explicit reason-aware guard layer. The likely path is to extend `src/cron/service/timer.ts`, cron state/config types, Gateway protocol schema, Control UI/macOS surfaces, docs, and regression tests so cron classifies terminal run errors once, persists/display guard state, respects Retry-After where available, auto-pauses or circuit-breaks recurring jobs for billing/auth/format with safe configurable defaults and one alert, and keeps timeout/network/rate-limit jobs enabled with bounded backoff coordinated with auth-profile/provider cooldowns.
