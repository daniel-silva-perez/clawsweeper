## Summary

Keep this PR open. Current main still rejects `timeoutSeconds: 0` in the Control UI and drops zero from the UI-built cron payload, while the backend cron contract already accepts zero as no-timeout. The PR is a focused implementation candidate for the still-open paired issue, and its current head also fixes the earlier malformed-input review concern by rejecting non-finite values.

## What This Changes

The PR changes the Control UI cron controller to accept finite non-negative `timeoutSeconds` values and preserve `0` in the built agent-turn payload while throwing on malformed timeout input.

## Best Possible Solution

Land a focused Control UI fix that accepts and preserves `timeoutSeconds: 0`, continues rejecting malformed and negative input, updates the controller regression tests, and aligns the visible timeout copy with the backend no-timeout contract.
