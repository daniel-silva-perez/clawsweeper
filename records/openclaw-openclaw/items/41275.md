## Summary

Current main still rejects and drops `timeoutSeconds: 0` in the Control UI while the protocol and cron runtime already support zero as no-timeout; this contributor PR is a plausible implementation paired with open issue #41272, but it still needs normal maintainer review for test and copy follow-up.

The PR changes the Control UI cron controller to accept finite non-negative `timeoutSeconds` values and preserve `0` in built agent-turn payloads.

## Best Possible Solution

Land a focused Control UI fix that accepts and preserves `timeoutSeconds: 0`, continues rejecting malformed and negative input, updates the controller regression tests, and aligns the visible timeout copy with the backend no-timeout contract.
