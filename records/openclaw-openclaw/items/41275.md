## Summary

Keep this PR open. Current main still rejects `timeoutSeconds: 0` in the Control UI and drops zero from the UI-built cron payload, while the backend/protocol already treats zero as a valid no-timeout value. The PR is also paired with same-author open issue #41272 via closing syntax, so it should remain available for maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open as the implementation candidate for #41272. The best path is to finish the focused Control UI fix in `ui/src/ui/controllers/cron.ts`, update controller tests so `timeoutSeconds: "0"` is accepted and preserved in the payload, keep malformed/non-finite input rejected, and update the English Control UI timeout help/error copy before maintainer review.
