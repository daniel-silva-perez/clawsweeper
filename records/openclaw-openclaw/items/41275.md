## Summary

Keep open. The PR addresses a real Control UI cron mismatch: current main still rejects and drops explicit `timeoutSeconds: 0` even though the protocol and runtime allow zero as no-timeout. The branch is not merge-ready because it changes the validation semantics without updating the existing controller test that still expects zero to be invalid, and it leaves the user-facing timeout error copy contradicting the new behavior.

## What This Changes

The PR changes `ui/src/ui/controllers/cron.ts` validation and payload building to reject non-finite or negative timeout input while preserving an explicitly entered `timeoutSeconds: 0` for agent-turn cron payloads.

## Best Possible Solution

Amend this branch so the Control UI accepts and preserves explicit `timeoutSeconds: 0`, continues rejecting malformed and negative timeout input, updates controller tests for validation and payload construction, and refreshes the English timeout copy plus generated locale outputs if the i18n workflow requires it.
