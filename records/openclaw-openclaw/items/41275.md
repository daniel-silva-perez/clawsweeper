## Summary

Keep open. Current main at 4a6e10ec still rejects and drops `timeoutSeconds: 0` in the Control UI even though the cron protocol and runtime already treat zero as the no-timeout value; this PR is the active implementation candidate for the paired open issue and should get normal maintainer review rather than cleanup closure.

## What This Changes

The PR changes the Control UI cron controller to accept finite non-negative `timeoutSeconds` values during validation and to preserve an explicitly entered `0` in the built agent-turn payload.

## Best Possible Solution

Land a focused Control UI fix that accepts and preserves an explicit `timeoutSeconds: 0`, continues rejecting negative and malformed timeout input, updates the cron controller tests, and updates the user-facing timeout strings to document zero as no-timeout.
