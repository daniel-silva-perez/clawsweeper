## Summary

Keep open. Current main still rejects `timeoutSeconds: 0` in the Control UI and drops an explicit zero from the submitted payload, while the cron protocol and runtime already support zero as no-timeout. This PR addresses the core behavior and has no security-sensitive changes, but it should get a small test and English copy update before merge.

## What This Changes

The PR changes `ui/src/ui/controllers/cron.ts` to reject non-finite or negative timeout input and preserve an explicitly entered `timeoutSeconds: 0` in the built agent-turn payload.

## Best Possible Solution

Land a focused Control UI fix that accepts and preserves explicit `timeoutSeconds: 0`, continues rejecting malformed and negative timeout input, updates the cron controller tests, and updates the English timeout help/error source string to document zero as no-timeout.
