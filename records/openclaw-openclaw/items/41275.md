Keep open. Current main still rejects `timeoutSeconds: 0` in the Control UI and drops an explicit zero from the built payload, while the protocol and cron runtime already support zero as no-timeout. The PR changes the Control UI cron controller to accept finite non-negative `timeoutSeconds` values during validation and preserve an explicitly entered `0` in the built agent-turn payload.

## Best Possible Solution

Land a focused Control UI fix that accepts and preserves explicit `timeoutSeconds: 0`, continues rejecting malformed and negative timeout input, updates the cron controller tests, and updates the English timeout help/error source string to document zero as no-timeout.
