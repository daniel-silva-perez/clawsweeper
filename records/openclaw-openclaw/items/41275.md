## Summary

Keep this PR open. Current main still rejects `timeoutSeconds: 0` in the Control UI and drops zero from UI-built cron payloads, while the backend schema and cron runtime already support zero as the no-timeout value. The PR is also paired with the same author's still-open issue #41272, so it should remain available as the implementation candidate rather than being closed by cleanup.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for #41272. The best path is to finish the focused Control UI fix in `ui/src/ui/controllers/cron.ts`, preserve `timeoutSeconds: 0` in `buildCronPayload`, continue rejecting negative and malformed/non-finite timeout input, update controller tests and English/i18n validation copy, and make an explicit maintainer call on CLI parity.
