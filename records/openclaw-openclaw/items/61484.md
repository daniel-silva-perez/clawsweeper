## Summary

Keep open. Current main still lacks media support on the agent-facing `sessions_send` path: the tool accepts and forwards text only, while existing `message`/Gateway send media support does not cover this inter-session bridge. Related media work such as #57254 and #63497 covers other surfaces, not this requested `sessions_send` behavior.

## What This Changes

The issue requests `sessions_send` support for local image/media paths and native media delivery between agent sessions, comparable to `openclaw message send --media`.

## Best Possible Solution

Define a generic `sessions_send` media contract that reuses existing attachment normalization and outbound media delivery plumbing, preserves session visibility and sandbox boundaries, and avoids per-channel special cases except through existing channel capabilities.

## Reproduction Assessment

Yes for the missing capability. Source inspection shows the `sessions_send` tool advertises and forwards only text, while media-capable paths live on `message`, Gateway `send`, and lower-level chat attachment handling; no runtime test was run because this review is read-only.

## Solution Assessment

Unclear. The request points at the right bridge, but the maintainable fix should first decide whether media becomes target-session model input, visible native delivery to the target channel, or both, then implement that through shared media helpers rather than channel-specific code in the session tool.
