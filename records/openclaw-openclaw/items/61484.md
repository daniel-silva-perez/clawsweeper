## Summary

Keep this issue open. Current main has media support in adjacent message/chat surfaces, and related #57254 was correctly closed for the shared `message` tool, but the `sessions_send` agent tool itself is still text-only and the A2A announce path only performs text delivery.

## Best Possible Solution

Keep the issue open and implement this as an additive core session-tool feature: extend `sessions_send` with media/mediaUrls/attachments or local-path media params, reuse existing message-action and gateway attachment normalization, pass media into the target `agent` run, and update A2A reply/announce delivery so native channel media send paths are used where available. Add focused tests for the tool schema, target-run attachment forwarding, A2A announce media delivery, and docs under `docs/concepts/session-tool.md`.
