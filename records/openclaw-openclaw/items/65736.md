## Summary

Keep this PR open. Current main still resolves exec approvals JSON and socket defaults through home-relative `~/.openclaw` paths, while `OPENCLAW_STATE_DIR` is the documented state-root override elsewhere. The PR remains an active implementation candidate, especially after the maintainer repair comment and merge-ready labeling; overlapping #62938 is also open, not a shipped replacement.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or consciously supersede it with #62938 if maintainers choose that branch. The final shipped fix should make exec approvals JSON and socket defaults honor an explicit `OPENCLAW_STATE_DIR`, preserve the legacy `~/.openclaw` fallback when unset, update effective-policy reporting and docs, keep path safety checks intact, and include focused regression tests for resolver and store writes.
