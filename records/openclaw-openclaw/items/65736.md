## Summary

Keep this PR open. Current main still resolves exec approval JSON and socket defaults under the home-relative `~/.openclaw` paths, while `OPENCLAW_STATE_DIR` remains the documented mutable state-root override for hosted deployments. The PR is a focused implementation candidate, and the provided maintainer comment says a narrow repair was pushed to this branch so the original contributor path can stay canonical rather than closing it as a duplicate of overlapping #62938.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or explicitly supersede it with one canonical implementation. The shipped fix should make exec approvals JSON and socket defaults honor an explicit `OPENCLAW_STATE_DIR`, preserve the legacy `~/.openclaw` fallback when unset, update effective-policy reporting and docs/changelog, and keep focused regression coverage for both file and socket path resolution.
