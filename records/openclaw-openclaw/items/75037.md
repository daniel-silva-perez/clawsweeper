## Summary

Keep open. Current main still has compaction and memory-flush metadata paths that can refresh `updatedAt`, while daily/idle freshness still falls back to `updatedAt` for legacy rows; the linked closing PR is open and unmerged, so the issue should remain the tracker.

## What This Changes

The issue asks for compaction and memory-flush metadata writes to preserve activity timestamps so legacy sessions without explicit lifecycle fields still reset under daily or idle policy.

## Best Possible Solution

Land PR #75066 or an equivalent narrow patch that makes compaction and memory-flush bookkeeping preserve activity timestamps, with regression coverage for legacy rows missing `sessionStartedAt` and `lastInteractionAt`.

## Reproduction Assessment

Yes. A high-confidence reproduction is a legacy session-store row missing lifecycle timestamps, followed by current compaction or memory-flush metadata persistence and then `initSessionState`/`evaluateSessionFreshness` on the next real user message.

## Solution Assessment

Yes. The preserve-activity policy already used for route metadata is the narrowest maintainable fix; changing the global freshness fallback would be broader and could alter unrelated legacy-session behavior.
