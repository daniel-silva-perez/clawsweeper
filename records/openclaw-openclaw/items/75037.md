## Summary

Keep open. Current main still has the reported compaction and memory-flush metadata paths that can refresh `updatedAt`, and the linked closing PR is still open/unmerged in the provided GitHub context, so this issue should remain the tracker until that PR or an equivalent fix lands.

## What This Changes

Prevent compaction and memory-flush bookkeeping writes from refreshing session activity timestamps so legacy daily and idle reset checks still expire stale sessions.

## Best Possible Solution

Review and land PR #75066, or an equivalent narrow patch, so compaction and memory-flush bookkeeping preserve activity timestamps while real user or agent turns remain responsible for extending daily/idle freshness. The fix should include regression coverage for legacy rows missing `sessionStartedAt` and `lastInteractionAt` without rewriting the broader reset policy.

## Reproduction Assessment

Yes. A high-confidence reproduction is a headerless legacy session-store row missing `sessionStartedAt` and, for idle mode, `lastInteractionAt`, followed by current compaction or memory-flush metadata persistence and then `initSessionState`/`evaluateSessionFreshness` on the next real user message.

## Solution Assessment

Yes. The narrow preserve-activity approach used for route metadata is the best fit for this bug; changing the global freshness fallback would be broader and could affect other legacy-session behavior.
