## Summary

Keep open. Current main still has both reported metadata paths that can refresh `updatedAt`, and the linked fix PR is open and unmerged, so this issue remains a valid tracking item until that fix or an equivalent one lands.

## What This Changes

The issue requests that compaction count and memory-flush metadata writes preserve session activity timestamps so legacy daily and idle reset freshness is not based on bookkeeping writes.

## Best Possible Solution

Land PR #75066 or an equivalent narrow patch that makes compaction and memory-flush bookkeeping preserve activity timestamps, while keeping real user or agent turns responsible for activity updates and adding regression coverage for legacy rows missing lifecycle fields.

## Reproduction Assessment

Yes. A high-confidence reproduction is a legacy session-store row missing `sessionStartedAt` and, for idle mode, `lastInteractionAt`, followed by current compaction or flush metadata persistence and then `evaluateSessionFreshness` during the next real session initialization.

## Solution Assessment

Yes. The targeted preserve-activity approach for compaction and flush metadata is the narrowest maintainable fix; rewriting reset fallback semantics globally would be broader and risk changing unrelated legacy-session behavior.
