## Summary

Keep open. Current main still contains the reported timestamp-refresh path, and the issue is paired with open PR #75066 using closing syntax, so the maintainer path is to review that implementation rather than close this issue before merge.

## What This Changes

The issue reports that compaction and memory-flush bookkeeping writes advance `updatedAt`, causing legacy session rows without lifecycle timestamps to skip daily or idle reset.

## Best Possible Solution

Land or revise a narrow preserve-activity fix for compaction and memory-flush bookkeeping, keeping real user or agent turns responsible for activity timestamps, then close this issue after the linked PR merges with its final SHA and release provenance.

## Reproduction Assessment

Yes. A high-confidence reproduction is a legacy session row missing `sessionStartedAt` and, for idle reset, `lastInteractionAt`, followed by a compaction or flush metadata write that moves `updatedAt` past the relevant boundary before `initSessionState` evaluates freshness.

## Solution Assessment

Yes. Reusing the existing preserve-activity merge pattern for these metadata writes is the narrowest maintainable direction; a broad reset-policy rewrite or store migration is not necessary for this bug.
