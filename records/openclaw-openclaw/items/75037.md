## Summary

Keep open. Current main still has the reported activity timestamp refresh path, and an open PR specifically targets this issue, so the issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

The issue asks OpenClaw to stop compaction and memory-flush bookkeeping writes from advancing `updatedAt` in ways that prevent legacy daily and idle session reset checks from expiring stale sessions.

## Best Possible Solution

Review and land PR #75066 or an equivalent narrow fix that makes compaction and memory-flush bookkeeping preserve activity timestamps, while keeping real user/agent turns responsible for activity updates and adding regression coverage for legacy rows missing lifecycle fields.

## Reproduction Assessment

Yes. The high-confidence reproduction is a legacy session-store row missing `sessionStartedAt` and, for idle mode, `lastInteractionAt`, followed by compaction or flush metadata persistence that advances `updatedAt` before the next `initSessionState` freshness check. I did not run the proposed tests because this review was read-only, but the source path is concrete.

## Solution Assessment

Yes. A targeted preserve-activity path for compaction and flush metadata is the narrowest maintainable fix; changing reset fallback semantics globally would be broader and could disturb other legacy-session behavior.
