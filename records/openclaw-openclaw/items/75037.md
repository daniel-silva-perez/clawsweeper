## Summary

Keep open. Current main still has both reported metadata write paths that can refresh `updatedAt`, while the linked closing PR remains open and unmerged in the provided GitHub context, so this issue should remain the tracker until that fix or an equivalent patch lands.

## What This Changes

The issue requests that compaction and memory-flush bookkeeping preserve session activity timestamps so legacy daily/idle reset decisions are not masked by refreshed `updatedAt` values.

## Best Possible Solution

Review and land PR #75066, or an equivalent narrow patch, so compaction and memory-flush bookkeeping preserve activity timestamps while real user/channel turns remain responsible for extending daily and idle freshness. The fix should keep the default merge behavior for true activity writes and include regression coverage for legacy rows missing lifecycle timestamps.

## Reproduction Assessment

Yes. The issue gives a high-confidence reproduction path: create a legacy session row missing `sessionStartedAt` and, for idle mode, `lastInteractionAt`; run current compaction or memory-flush metadata persistence; then evaluate freshness on the next real user message.

## Solution Assessment

Yes. The narrow preserve-activity approach is the best fit because the repo already uses that policy for route metadata in #49515, while changing the global freshness fallback would be broader and riskier for other legacy-session behavior.
