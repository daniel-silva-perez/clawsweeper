## Summary

Keep open. Current main still has the reported metadata-write paths that can refresh `updatedAt`, and an open closing PR already targets the issue, so the issue should remain the tracker until that PR or an equivalent fix lands.

## What This Changes

Compaction and memory-flush bookkeeping should preserve activity timestamps so legacy daily and idle session reset decisions are not made fresh by metadata-only writes.

## Best Possible Solution

Land PR #75066 or an equivalent narrow change that opts compaction and memory-flush bookkeeping into preserve-activity semantics, while keeping real user or agent turns responsible for activity timestamps and adding regression coverage for legacy rows missing lifecycle fields.

## Reproduction Assessment

Yes. A high-confidence reproduction is a legacy session-store row missing `sessionStartedAt` and, for idle mode, `lastInteractionAt`, followed by current compaction or flush metadata persistence and then `evaluateSessionFreshness` during the next real session initialization.

## Solution Assessment

Yes. A targeted preserve-activity path for compaction and flush metadata is the narrowest maintainable fix; changing reset fallback semantics globally would be broader and risk disturbing other legacy-session behavior.
