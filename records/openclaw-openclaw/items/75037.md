## Summary

Keep open. Current main still has the reported compaction and memory-flush metadata paths that can refresh `updatedAt`, and PR #75066 is open, unmerged, and explicitly targets this bug, so this issue should remain the tracker until that PR or an equivalent fix lands.

## What This Changes

Compaction and memory-flush bookkeeping should preserve session activity timestamps so legacy daily and idle reset decisions are not made fresh by metadata-only writes.

## Best Possible Solution

Review and land PR #75066, or an equivalent narrow change, so compaction and memory-flush bookkeeping opt into preserve-activity semantics while real user or agent turns remain responsible for activity timestamp updates and regression coverage protects legacy rows missing lifecycle fields.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a legacy session-store row missing `sessionStartedAt`, and for idle mode also missing `lastInteractionAt`, followed by current compaction or flush metadata persistence and then `evaluateSessionFreshness` on the next real session initialization.

## Solution Assessment

Yes. The narrow preserve-activity path is the best maintainable direction; changing reset fallback semantics globally would be broader and could disturb other legacy-session behavior.
