## Summary

Keep open. Current main has generic Gateway and node primitives that make an external Raspberry Pi voice bridge feasible, and the discussion includes afalk42's working external implementation, but OpenClaw still does not ship or document a first-class Brabble/Raspberry Pi distributed voice-wake node. The remaining gap is a product and security-boundary decision, not an already-implemented core feature.

## What This Changes

Request to run Brabble as a Raspberry Pi or other OpenClaw node that performs local wake-word capture, sends speech or text to the Gateway, and returns agent responses through TTS.

## Best Possible Solution

Keep the issue open until maintainers choose the product boundary: either document a minimal external voice-node contract and promote/link the Raspberry Pi companion through ClawHub or docs, or define a small generic node/plugin API only if the current Gateway surfaces are insufficient.
