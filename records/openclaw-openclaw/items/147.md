## Summary

Keep open. Current main has generic Gateway/node voice primitives and an OpenAI-compatible HTTP endpoint that can support an external Raspberry Pi voice bridge, and the discussion includes a working external implementation, but OpenClaw still does not ship or document a first-class Brabble/Raspberry Pi distributed voice-wake node. The remaining decision is product and security boundary work, not conservative cleanup.

## What This Changes

The issue requests running Brabble on a Raspberry Pi or other device as a distributed OpenClaw voice-wake node that sends wake-word audio or text through the Gateway and receives TTS responses.

## Best Possible Solution

Maintainers should choose the official boundary first: document a minimal external voice-node contract and promote or link the Raspberry Pi companion through ClawHub/docs, or define a small generic node/plugin API only if current Gateway surfaces are insufficient. That path should include clear security guidance for operator credentials and always-on microphone use.
