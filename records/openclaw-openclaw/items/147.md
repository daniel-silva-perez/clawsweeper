## Summary

Keep open. Current main has enough generic gateway primitives for an external Raspberry Pi voice bridge, and the discussion includes a working external implementation, but OpenClaw still does not ship or document a first-class Brabble/Raspberry Pi distributed voice-wake node. The remaining choice is product and architecture scope, not a stale or already-implemented request.

## What This Changes

The issue requests a Brabble or Raspberry Pi-style distributed voice-wake node that connects to the OpenClaw gateway, routes wake-triggered voice commands to an agent, and returns responses via TTS.

## Best Possible Solution

Keep the issue open for maintainer product review, with the likely end state being a documented minimal external voice-node contract and a link or promotion path for the working Raspberry Pi companion, adding a new generic plugin/API seam only if maintainers identify a real gap.
