## Summary

Keep open. Current main has enough primitives for an external Raspberry Pi/Brabble-style voice bridge, and the discussion includes a working external implementation, but OpenClaw still does not ship or document a first-class Brabble/Raspberry Pi distributed voice-wake node. The thread also leaves a product decision open: external ClawHub/plugin companion, documented minimal node contract, or native node integration.

## Best Possible Solution

Keep this open for maintainer product review. The likely path is to document the minimal external voice-node contract and promote an afalk42-style implementation through ClawHub or a companion repo, adding core/plugin APIs only if maintainers identify a missing generic node or voice extension seam.
