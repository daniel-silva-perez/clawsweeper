## Summary

Keep open. Current main has generic Gateway and node primitives that make an external Raspberry Pi voice bridge feasible, and the discussion includes a working external implementation, but OpenClaw still does not ship or document a first-class Brabble/Raspberry Pi distributed voice-wake node. The remaining decision is a product and security boundary choice, not a completed cleanup.

## What This Changes

Request support for running Brabble as a distributed OpenClaw/Clawdis node that performs local voice wake on Raspberry Pi-like hardware, sends speech to the Gateway, and returns responses through TTS.

## Best Possible Solution

Maintainers should choose the boundary first: either document a minimal external voice-node contract and promote or link the Raspberry Pi companion through ClawHub/docs, or define a small generic node/plugin API only if current Gateway surfaces are insufficient. That path should include clear guidance for credentials, pairing, and always-on microphone privacy.
