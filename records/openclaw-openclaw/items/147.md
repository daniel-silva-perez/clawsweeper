## Summary

Keep open. Current main has generic Gateway/node voice primitives, a VoiceClaw realtime endpoint, and an OpenAI-compatible HTTP path that make an external Raspberry Pi voice bridge feasible, and the discussion includes a working external reference. It still does not ship or document a first-class Brabble/Raspberry Pi distributed voice-wake node, so the remaining work is a maintainer product and security boundary decision rather than conservative cleanup.

## What This Changes

Request to support a Brabble/Raspberry Pi-style distributed voice-wake node that connects to the Gateway, sends wake-triggered audio or transcripts to an agent, and returns spoken responses.

## Best Possible Solution

Maintainers should choose the official boundary first: document a minimal external voice-node contract and promote or link the Raspberry Pi companion through docs/ClawHub, or define a small generic node/plugin API only if the current Gateway surfaces are insufficient. Brabble-specific core code should wait for a clear product and security decision.
