## Summary

Keep open. Current main has generic Gateway/node primitives that make an external Raspberry Pi voice bridge feasible, and the discussion includes a working external implementation, but OpenClaw still does not ship or document a first-class Brabble/Raspberry Pi distributed voice-wake node. The remaining choice is product and security-boundary work, not a completed core cleanup.

## What This Changes

Request to support Brabble or an equivalent Raspberry Pi voice node as a distributed wake-word entry point into OpenClaw.

## Best Possible Solution

Maintainers should choose the product boundary first: either document a minimal external voice-node contract and promote or link the Raspberry Pi companion through ClawHub/docs, or define a small generic node/plugin API only if the current Gateway surfaces are insufficient. That path should include clear guidance for credentials, pairing, and always-on microphone privacy.
