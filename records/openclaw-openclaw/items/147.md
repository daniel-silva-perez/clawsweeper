## Summary

Keep open. Current main has generic Gateway/node primitives that make an external Raspberry Pi voice bridge feasible, and the discussion includes a working companion implementation, but OpenClaw still does not ship or document a first-class Brabble/Raspberry Pi distributed voice-wake node. The remaining decision is an official product and security boundary, not conservative cleanup.

## What This Changes

The issue requests support for running Brabble on a Raspberry Pi or similar device as a distributed voice-wake node that forwards speech to the OpenClaw Gateway and receives TTS responses.

## Best Possible Solution

Maintainers should establish the official boundary: document the existing external voice-node contract and promote the companion through ClawHub/docs, or add a small generic node/plugin API only after identifying a concrete API gap. That path should include credential, pairing, revocation, and always-on microphone privacy guidance.
