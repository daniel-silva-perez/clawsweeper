## Summary

Keep open. Current main has enough generic Gateway and node primitives to make an external Raspberry Pi voice bridge feasible, and the discussion includes a working external implementation, but OpenClaw still does not ship or document a first-class Brabble/Raspberry Pi distributed voice-wake node. The remaining decision is product and security boundary work, not a completed core cleanup.

## What This Changes

Run Brabble or a similar Raspberry Pi voice pipeline as a distributed OpenClaw node that wakes locally, sends speech or text to the Gateway, and returns spoken replies.

## Best Possible Solution

Maintainers should choose the product boundary first: either document a minimal external voice-node contract and promote or link the Raspberry Pi companion through ClawHub/docs, or define a small generic node/plugin API only if the current Gateway surfaces are insufficient. That path should include clear security guidance for operator credentials and always-on microphone use.
