## Summary

Keep open. Current main has generic Gateway and node primitives that make an external Raspberry Pi voice bridge feasible, and the discussion includes a working external implementation, but OpenClaw still does not ship or document a first-class Brabble/Raspberry Pi distributed voice-wake node. The remaining decision is a product and security boundary choice, not completed cleanup.

## What This Changes

Request to support Brabble as a Raspberry Pi or similar node that performs local wake-word capture and sends speech or text to the Gateway for agent routing and TTS replies.

## Best Possible Solution

Maintainers should choose the boundary first: document a minimal external voice-node contract and promote or link the Raspberry Pi companion through ClawHub/docs, or define a small generic node/plugin API only if current Gateway surfaces are insufficient. That path should include clear guidance for credentials, pairing, revocation, and always-on microphone privacy.
