## Summary

Keep this issue open. Current main still has no final-reply policy, formatter, or config switch that expands `~/...` in user-visible assistant replies. It still intentionally shortens home paths in shared display helpers and skill prompt paths, expands tilde only for host file-tool execution, and can surface the original tilde path in edit-tool success text. The related #38495 PR proposed `OPENCLAW_EXPAND_PATHS`, but it was closed unmerged and that env var is absent from current main.

## Best Possible Solution

Keep the issue open. Maintainers should decide the product boundary, then implement either explicit final-reply prompt guidance, a delivery-time formatter, or a documented config switch with regression coverage for final replies, while preserving intentional skill-prompt compaction and file-tool tilde input expansion. #38495 is useful prior art, but its broad `src/utils.ts` toggle should be reviewed against CLI/display blast radius before reuse.
