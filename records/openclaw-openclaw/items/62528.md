## Summary

Keep this PR open. Current main does not implement the proposed `POST /hooks/message` ingress: the gateway hook handler only has built-in `wake` and `agent` branches, then falls through to user-configured `hooks.mappings`, and the public docs likewise list only `/hooks/wake`, `/hooks/agent`, and mapped hooks. This PR is still a live implementation candidate, not obsolete on main.

## Best Possible Solution

Keep the PR open for maintainer review. If OpenClaw wants a core generic `/hooks/message` ingress, rebase the PR onto current `src/gateway/server/hooks-request-handler.ts`, preserve or explicitly migrate existing mapped `/hooks/message` configurations, resolve the remaining review concerns, and keep the auth/safety/idempotency/session-policy/chat-visibility tests and docs aligned before landing. If maintainers decide this should not be core, close it manually with that product rationale rather than as implemented-on-main.
