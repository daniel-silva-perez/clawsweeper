## Summary

Current main still does not implement the requested two-phase macOS LaunchAgent restart. The normal `restartLaunchAgent` path still enables the service and calls `launchctl kickstart -k`; tests still assert no normal-path `bootout` or `bootstrap`; and the detached self-restart handoff still schedules `mode: "kickstart"`. The merged #72616 work is adjacent installed-but-unloaded supervision recovery, not a superseding fix for this restart-ordering request.

## Best Possible Solution

Keep this issue open for maintainer judgment on the macOS LaunchAgent restart contract. The best path is to decide whether normal `openclaw gateway restart` should replace the current kickstart-based contract with an explicit stop/wait/start sequence, then update `src/daemon/launchd.ts`, `src/daemon/launchd-restart-handoff.ts`, and their tests to prove old-service exit-before-start ordering without regressing detached self-restart or installed-but-unloaded recovery. Live macOS launchd proof should be part of landing.
