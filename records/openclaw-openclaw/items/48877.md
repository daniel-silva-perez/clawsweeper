## Summary

Keep this PR open. Current main still does not implement declarative Telegram `customCommands` menus/routes, and the PR remains a plausible bundled Telegram plugin implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to finish the feature in the Telegram plugin/config surface with typed `menus` and `routes`, auth parity with native commands, callback routing scoped to the originating validated custom command, callback payload validation using the existing Telegram limit/sanitizer contract, and focused tests/docs for menu display, route edits, fallback-to-agent behavior, duplicates/conflicts, and malformed buttons.
