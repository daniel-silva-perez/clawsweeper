## Summary

Keep this PR open. Current main still only supports Telegram `customCommands` as native BotFather menu entries with `command` and `description`; it does not expose declarative `menus`/`routes` or edit callback messages for custom menu navigation. The PR targets a real missing Telegram plugin/config capability, and the discussion shows the author addressed several auth and collision review points, but it still needs normal maintainer review before merge.

## Best Possible Solution

Keep the PR open as an implementation candidate. The best path is to finish the feature in the Telegram plugin/config surface with typed `menus` and `routes`, authorization parity with existing native commands, callback routing scoped to the validated originating custom command, callback payload validation using the existing Telegram limit/sanitizer contract, and focused tests/docs for menu display, route edits, callback fallback, duplicate/conflict handling, and malformed buttons.
