## Summary

Keep this PR open. Current main still lacks Telegram customCommand menu/routes support, so the contribution targets a real missing feature, but the proposed branch is not merge-ready against the current command/config surfaces and has concrete correctness blockers.

## What This Changes

The PR adds Telegram customCommand `menus`/`routes` config fields, registers menu-backed slash-command handlers, routes matching callback queries to menu edits, and adds a validated-custom-command-index helper.

## Best Possible Solution

Land a refreshed Telegram plugin implementation that adds typed `menus` and `routes` on the current config surfaces, validates menu button text and callback payloads against Telegram constraints, routes only authenticated originating menu callbacks, updates generated config metadata and public Telegram docs, and includes focused Telegram schema/native-command/callback tests.

## Reproduction Assessment

Yes. Current main can be checked by config parsing and source inspection: `menus`/`routes` are rejected by the strict schema, and callback payloads still fall through as synthetic messages; the PR’s import/export mismatch is a build-time failure against the current SDK surface.

## Solution Assessment

No, not as currently proposed. The feature direction belongs in the Telegram plugin/config surface, but the branch should be refreshed onto `src/plugin-sdk/telegram-command-config.ts` and the current generated config/docs flow, with callback-data validation before merge.

  - confidence: 0.86
