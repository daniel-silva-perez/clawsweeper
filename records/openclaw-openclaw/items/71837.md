## Summary

Keep open. Current main still lacks the central REM-to-grounded promotion behavior, and the PR head still has correctness issues around synthetic query diversity, grounded signal weighting, and REM meta tag filtering that should be fixed before maintainer consideration.

## What This Changes

The PR updates memory-core Dreaming and session transcript extraction to strip more scaffold text, denoise REM themes/truths, feed REM candidate truths into grounded short-term promotion signals, adjust deep ranking, and add focused tests.

## Best Possible Solution

Land a rebased implementation that preserves the useful denoise and REM-to-grounded promotion direction while using stable synthetic REM query identity, confidence-scaled grounded signal strength, complete meta stopword coverage for runtime/system tags, and the current package-owned memory-host SDK surface.

## Reproduction Assessment

Yes. The PR diff gives a high-confidence reproduction path: the same REM candidate selected on multiple day buckets gets distinct synthetic query hashes, marginal REM truths receive signalCount 3, and system is boosted as a positive REM tag despite current main treating it as reflection metadata.

## Solution Assessment

No. The direction is plausible, but the current PR branch is not the safest merge shape until the query identity, signal weighting, meta tag coverage, and stale-base issues are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use a stable REM query id:** `extensions/memory-core/src/dreaming-phases.ts:1856-1857`
  - body: The REM grounding path writes a day-suffixed query, and recordGroundedShortTermCandidates hashes that query into queryHashes. If the same candidate is selected on multiple days, it gains distinct synthetic query hashes and can satisfy diversity gates without distinct recall contexts; keep the synthetic query stable and rely on dayBucket for day-scoped dedupe.
  - confidence: 0.9
- **[P2] Scale REM grounded signal strength:** `extensions/memory-core/src/dreaming-phases.ts:1855`
  - body: Every REM truth that passes the confidence threshold is written with signalCount: 3, so a borderline candidate and a high-confidence candidate get the same grounded promotion weight. Derive the count from calculateCandidateTruthConfidence(entry) or use a lower floor for marginal candidates.
  - confidence: 0.84
- **[P2] Keep system out of REM themes:** `extensions/memory-core/src/dreaming-phases.ts:1467`
  - body: The PR adds system to REM_POSITIVE_TAGS without adding it to REM_META_TAGS, while current main explicitly blacklists system reflection tags. That lets transcript/runtime meta language become a boosted REM theme, which works against the denoise goal.
  - confidence: 0.86
