## Summary

Keep open. Current main has substantial partial inline media support for Control UI chat, including MEDIA directive parsing, inline image/audio/video attachment renderers, and managed generated-image persistence. The umbrella request is still not complete: remote markdown images are still flattened and blocked by CSP policy, automatic web/link previews are absent, related remote-image/tool-result image work remains open, and raw/generated audio is not consistently normalized into durable UI-renderable media attachments.

## Best Possible Solution

Keep this open as the umbrella unless maintainers explicitly split it. Preserve the shipped MEDIA directive, inline attachment player, and managed generated-image paths; then finish the remaining slices through narrower work: decide and implement the remote-image policy for #40945/#41025, add server-side web/link previews with explicit SSRF/CSP/privacy constraints, finish direct tool-result image rendering for #50779-style payloads, and normalize generated audio/video into durable managed-media attachment references that replay after chat history reloads.
