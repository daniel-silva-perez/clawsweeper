## Summary

Keep open. Current main has meaningful partial inline media support for Control UI chat, but the umbrella request is still not complete: web/link previews are absent, remote markdown images remain intentionally flattened/CSP-blocked, direct tool-result media work still has open follow-ups, and raw audio blocks produced by the Gateway are not rendered by the UI as playable chat media.

## Best Possible Solution

Keep this issue open as the umbrella unless maintainers explicitly split it. Preserve the shipped MEDIA directive, inline attachment player, and managed generated-image paths; then finish the remaining slices through narrower work: decide the remote-image policy for #40945/#41025, add server-side link previews with explicit SSRF/CSP/privacy constraints, complete direct tool-result image/media rendering for #50779-style payloads, and normalize generated audio/video into durable UI-renderable media attachments that replay after chat history reloads.
