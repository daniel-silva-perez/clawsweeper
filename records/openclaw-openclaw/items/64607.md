## Summary

Keep open. Current main has meaningful partial inline media support in Control UI chat, but the umbrella request is not complete: remote markdown images are still deliberately flattened/blocked, automatic web/link previews are absent, and generated/raw audio is still not normalized into durable UI-renderable media attachments the same way generated images are.

## What This Changes

Request inline chat previews and players for images, audio, video, generated media, and web page links instead of requiring external open commands.

## Best Possible Solution

Keep this as the umbrella unless maintainers intentionally split it. Preserve the shipped MEDIA directive, inline attachment/player, and managed generated-image paths; finish narrower slices for remote-image policy (#40945/#41025), server-side link previews with explicit SSRF/CSP/privacy constraints, direct tool-result image rendering (#50779/#52025), and durable generated audio/video attachment references that replay after history reloads.
