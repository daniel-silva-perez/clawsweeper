## Summary

Current main still does not implement Telegram pinned-message reads. The shared message-action surface already has `list-pins` and the CLI maps `openclaw message pins` to it, but the bundled Telegram plugin only discovers and dispatches send/poll/react/delete/edit/sticker/topic actions. Telegram `getChat` is present only for chat-id lookup and parses `result.id`, not `pinned_message`. Closed PRs #38613 and #38718 are useful prior art, but neither merged.

## Best Possible Solution

Keep this issue open and land a focused bundled Telegram plugin change that exposes Telegram `list-pins`/`pins` through the existing shared message action surface, backed by Bot API `getChat.pinned_message`. Add a `channels.telegram.actions.pins` config/schema/docs gate, normalize the single Telegram pinned message into the shared `pins` payload shape, document the chat-level limitation, and use #38613/#38718 as prior art.
