## Summary

Current main still does not implement Telegram pinned-message reads. The shared message action contract and CLI already have `list-pins`/`pins`, but the bundled Telegram plugin does not advertise, map, gate, or execute that action; its existing `getChat` helper only extracts `result.id`. Closed PRs #38613 and #38718 are useful prior art but neither merged, so the feature request remains valid and narrow enough for a focused fix PR.

## Best Possible Solution

Keep this issue open and land a focused bundled Telegram plugin change that exposes `list-pins`/`pins` through the existing shared message action surface, backed by Bot API `getChat.pinned_message`, with a Telegram `actions.pins` config/schema/docs gate, normalization into the shared pins payload, tests, and docs for Telegram's chat-level limitation.

## Repair Work Prompt

Implement Telegram pinned-message reads for the bundled Telegram plugin. The observable gap: the shared `list-pins` action and CLI `openclaw message pins` exist, but Telegram does not advertise or execute pin reads. Scope the fix to Telegram plugin/config/docs/tests only: add a `pins` boolean gate to `TelegramActionConfig` and the Telegram zod schema; expose `list-pins`/`pins` from `extensions/telegram/src/channel-actions.ts` when enabled; implement runtime support in `extensions/telegram/src/action-runtime.ts` backed by Bot API `getChat.pinned_message` using existing Telegram token/account/fetch/proxy/api-root patterns; normalize Telegram's zero-or-one pinned message into the shared `pins` payload shape with message id, chat id, text/caption, sender, and date where available; support explicit target fields using existing Telegram target parsing conventions. Use #38613 and #38718 as prior art, but do not import unrelated system-prompt, cron, ACP, or slash-command changes. Do not add automatic session-start pin fetching in this PR. Document Telegram's chat-level limitation and the new `channels.telegram.actions.pins` gate. Add a user-facing changelog entry crediting the reporter per repo changelog rules.
