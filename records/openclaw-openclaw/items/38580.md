## Summary

Current main still does not implement Telegram pinned-message reads. The Telegram plugin advertises and dispatches send/poll/react/delete/edit/sticker/topic actions only, `getChat` usage only resolves chat ids, Telegram config has no `actions.pins` gate, and docs/tests still list pinned-message reads for other channels rather than Telegram. Closed PRs #38613 and #38718 remain useful prior art, but neither was merged.

## Best Possible Solution

Keep this issue open and land a focused bundled Telegram plugin change that exposes Telegram `list-pins`/`pins` through the existing message action surface, backed by Bot API `getChat.pinned_message`. Keep it opt-in/controllable through `channels.telegram.actions.pins`, normalize the result into the shared pins payload shape, document the chat-level most-recent-pin limitation, and treat #38613 and #38718 as prior art rather than canonical fixes.
