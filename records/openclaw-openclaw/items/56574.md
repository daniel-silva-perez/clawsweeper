## Summary

Keep open. Current main still lacks the central plugin tool dedupe and tool-factory `pluginConfig` threading that this PR targets, and the gateway-bindable/default registry reuse remains only partially covered. The PR is not merge-ready as-is because the live-test staging change snapshots env before loading `~/.profile`, and the branch needs a current-main rebase across the now-expanded plugin loader/cache surfaces.

## What This Changes

The PR changes plugin registry/loading/tool resolution to dedupe repeated per-plugin tool registrations, pass per-plugin config into tool factories, relax gateway-bindable registry reuse for default requests, and update live-test HOME/profile staging docs and tests.

## Best Possible Solution

Rebase the plugin fixes onto current main, keep the dedupe/config-threading behavior in the plugin registry/tool contract with focused regression tests, preserve all current cache-key compatibility dimensions, and either fix or split the live-test HOME/profile staging changes before merge.

## Reproduction Assessment

Yes for the source-level defects: on current main, repeated `api.registerTool(...)` calls append multiple `registry.tools` entries, and tool factories are called with `params.context` without `pluginConfig`. The live-test staging issue is also reproducible by defining `OPENCLAW_CONFIG_PATH` only in `~/.profile` and running a live test that stages config/auth state.

## Solution Assessment

Unclear as currently proposed. The registry-level dedupe and tool-context threading are the right product boundary, but the branch should be rebased and the live-test env snapshot bug fixed before it is a maintainable merge path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Snapshot env after loading the profile:** `test/test-env.ts:426`
  - body: `liveEnvSnapshot` is captured before `loadProfileEnv(realHome)`, but `stageLiveTestState` later uses that snapshot to resolve `OPENCLAW_CONFIG_PATH` and `OPENCLAW_STATE_DIR`. When those are only exported from `~/.profile`, live tests stage the default `~/.openclaw` instead of the operator-selected config/auth state, so the documented live setup silently runs against the wrong profile. Move the snapshot after profile loading or pass post-load env to staging.
