## Summary

Keep open. Current main does not implement `agents.defaults.bootstrapTier`, and this active contributor PR targets a real prompt-pressure problem linked from #75189, but the proposed patch is not merge-ready because it changes delegated/background bootstrap semantics and does not keep the extra-bootstrap hook path tier-aware.

## What This Changes

The PR adds an opt-in `agents.defaults.bootstrapTier` config value, minimal bootstrap filtering, a loopback local-model doctor warning, config metadata, docs, changelog text, and focused tests.

## Best Possible Solution

Land the feature only after the tier semantics are explicit, session-aware, and hook-aware: keep `standard` as the default, reduce ordinary bootstrap predictably, preserve or deliberately document delegated/background session behavior, and add regression coverage for the bundled extra-bootstrap hook.

## Reproduction Assessment

Yes. Static inspection is enough: on PR head, `filterBootstrapFilesForSession(files, "agent:default:subagent:task-1", "minimal")` takes the new minimal branch before the subagent/cron branch, and the bundled hook still calls `filterBootstrapFilesForSession` without a tier.

## Solution Assessment

No. The config and doctor-warning direction is plausible, but the current patch is not the best maintainable fix until filtering is applied consistently across hook mutation paths and delegated/background behavior is preserved or intentionally redesigned.
