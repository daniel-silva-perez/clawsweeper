## Summary

Current main still has the reported Memory Wiki contract mismatch: `sources/*.md` files are compiled and linted as structured wiki source pages requiring frontmatter, while the public plugin docs still describe `sources/` as imported raw material. This should stay open for a narrow docs-contract clarification rather than closing as implemented.

## What This Changes

The issue asks OpenClaw to resolve a Memory Wiki lint/docs mismatch where files under `sources/` are described as raw imported material but `wiki lint` requires `id`, `pageType`, and freshness frontmatter.

## Best Possible Solution

Keep this open and align the Memory Wiki source-page contract in docs. Clarify that `sources/*.md` are structured wiki source pages requiring frontmatter, while raw originals should be imported through `openclaw wiki ingest`, bridge, or unsafe-local and preserved inside source-page content/provenance. If maintainers want direct raw files inside the vault, track that separately as a raw mode or ignored subdirectory rather than weakening lint globally.
