## Summary

Current main still has the reported memory-wiki contract mismatch. The implementation treats `sources/*.md` as structured wiki source pages that compile, lint, and carry frontmatter, while the public Memory Wiki docs still describe `sources/` as imported raw material. The safest narrow fix remains a docs-contract clarification, not a lint exemption, unless maintainers explicitly want a new raw-file mode.

## Best Possible Solution

Keep this open and align the Memory Wiki source-page contract. The focused fix should clarify docs that `sources/*.md` are structured wiki source pages requiring frontmatter, while raw originals are imported through `openclaw wiki ingest`, bridge, or unsafe-local and preserved in source-page content/provenance. If maintainers want direct raw files inside the vault, track that separately as an explicit raw mode or ignored subdirectory rather than weakening source-page lint.
