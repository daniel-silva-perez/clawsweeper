## Summary

Keep this issue open. Current main has partial adjacent controls (`collect`, `steer`, `interrupt`, `/stop`, and `/btw`), but it still does not implement the requested owner-priority or priority-flagged soft in-turn delivery that preserves the active run and lets the agent incorporate new owner context before final response.

## Best Possible Solution

Keep the issue as a core agent-turn/message-queue feature request. The best path is a small, explicit core design for owner/priority followups: add a supported priority or owner-priority queue mode/config/API, inject priority messages at safe tool/model boundaries without aborting the active run, preserve authorization/routing metadata, and cover default `collect`, Telegram/group routing, streaming, non-streaming fallback, and `/stop` interactions with focused tests and docs.

## Evidence
