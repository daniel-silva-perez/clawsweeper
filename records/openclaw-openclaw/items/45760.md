Keep this PR open. Current main still does not implement the proposed change: full bootstrap may include `MEMORY.md`, but minimal subagent and cron bootstrap filters it out. The current tests and workspace-template docs also encode that memory should not be injected into these delegated/shared contexts, so this needs maintainer product/security judgment rather than conservative cleanup closure.

## Best Possible Solution

Do not close this PR as implemented. Maintainers should either close/reject it after confirming the minimal-session privacy contract is intended, or redesign the feature as an explicit opt-in memory/context policy for subagent and cron runs with updated source, tests, docs, and security review.
