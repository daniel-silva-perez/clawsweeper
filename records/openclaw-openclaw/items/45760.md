## Summary

Keep this PR open. Current main still does not implement the proposed behavior: normal bootstrap can load canonical MEMORY.md, but subagent and cron session keys are filtered through a minimal allowlist that excludes it. The PR would change a privacy-sensitive bootstrap boundary that current tests and workspace docs still encode, and its proposed lowercase-memory symbol is no longer present on main.

## Best Possible Solution

Keep the PR open for maintainer judgment. The best product path is either to reject the default allowlist change as conflicting with the current main-session memory boundary, or redesign it as an explicit opt-in memory/context-sharing policy for subagent and cron runs, with source, tests, docs, and security review updated together.
