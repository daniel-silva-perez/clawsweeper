## Summary

Keep this PR open. Current main still does not implement the proposed behavior: `MEMORY.md` is a recognized bootstrap file for ordinary sessions, but subagent and cron session keys are filtered through a minimal allowlist that excludes memory. The PR is therefore not obsolete, but it also changes a privacy-sensitive context boundary that current tests and workspace guidance intentionally encode, and the proposed lowercase-memory constant is stale against current main.

## Best Possible Solution

Keep the PR open for maintainer follow-up. The best path is either to explicitly reject this default allowlist expansion as conflicting with the current main-session memory boundary, or redesign it as an opt-in memory/context-sharing policy for subagent and cron runs with source, tests, docs, and security review updated together.
