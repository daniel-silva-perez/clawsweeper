## Summary

Keep this PR open. Current main still does not implement the proposed behavior: `MEMORY.md` is loaded for full/main bootstrap when present, but the minimal subagent/cron filter excludes it. The PR would change a documented and tested privacy-sensitive contract, and the existing review discussion already flags that risk rather than showing the PR is obsolete or implemented.

## Best Possible Solution

Keep the PR open for maintainer judgment. Maintainers should either reject it as conflicting with the current main-session-only memory contract, or redesign it as an explicit opt-in memory/context-sharing policy for subagent and cron runs with updated source, tests, docs, and security review.
