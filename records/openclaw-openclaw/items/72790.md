## Summary

Keep open. Current main still models `agents.defaults.compaction.reserveTokensFloor`, `keepRecentTokens`, and `reserveTokens` as absolute numeric token counts in the public type, strict config schema, generated schema, docs, and runtime resolver. The request is a narrow, valid enhancement and is not implemented or superseded by the related compaction items in the provided context.

## What This Changes

Add percentage-string support for compaction token-budget settings so values such as `reserveTokensFloor` and `keepRecentTokens` can scale with the active model context window.

## Best Possible Solution

Add a small typed token-budget value contract that preserves current numeric behavior, accepts documented percentage strings for the selected compaction settings, resolves them against the active model or effective context window at runtime, and updates generated schema, docs, changelog, and focused tests together.

## Reproduction Assessment

Not applicable as a feature request. There is a high-confidence static verification path: current types, Zod schema, generated schema, docs, and runtime resolver all accept numbers only and contain no percentage parser for these settings.

## Solution Assessment

Yes. Explicit percentage-string support is the narrowest maintainable direction because it keeps existing numeric configs backward-compatible; auto-scaling existing absolute numbers would silently change user behavior.
