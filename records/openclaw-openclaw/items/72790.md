## Summary

Keep open. Current main still has numeric-only compaction token settings across the public type, Zod schema, generated schema, docs, and runtime resolver, so the requested percentage-based config remains unimplemented and is a narrow enhancement candidate.

## What This Changes

Request support for percentage-based compaction token-budget settings so values such as reserve headroom and recent-history retention scale with the active model context window.

## Best Possible Solution

Add a backward-compatible token-budget value contract that preserves existing numeric configs, accepts documented percentage strings for compaction token settings, resolves them against the effective model context budget at runtime, and updates schema generation, docs, changelog, and focused tests together.

## Reproduction Assessment

Not applicable as a feature request, but yes for static verification. Current main consistently exposes and consumes these compaction settings as numbers only across type, schema, docs, generated schema, runtime resolver, and tests.

## Solution Assessment

Yes. Additive percentage-string support is the narrowest maintainable path because it keeps existing absolute numeric configs stable; silently auto-scaling existing numbers would change established behavior.
