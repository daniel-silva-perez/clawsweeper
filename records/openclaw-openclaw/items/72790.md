## Summary

Keep open. Current main still exposes the requested compaction token settings as absolute numeric values in the public type, Zod schema, generated schema, docs, runtime resolver, and focused tests; percentage strings are not accepted or resolved against the active context window.

## What This Changes

Request support for percentage-based compaction token settings such as `reserveTokensFloor: "10%"` and `keepRecentTokens: "15%"` so configs scale across model context windows.

## Best Possible Solution

Add a typed token-budget value contract that preserves existing numeric configs, accepts documented percentage strings for the selected compaction settings, resolves them against the effective model context budget at runtime, and updates generated schema, docs, changelog, and focused tests together.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence static verification path: current schema, TypeScript types, generated schema, docs, and runtime code all accept numbers only and contain no percentage parser for these settings.

## Solution Assessment

Yes. Explicit percentage-string support is the narrowest maintainable direction because it keeps existing numeric configs backward-compatible; auto-scaling existing absolute numbers would silently change established behavior.
