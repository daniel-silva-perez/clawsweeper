## Summary

Keep open. Current main still treats the relevant compaction token settings as absolute numeric values across public types, validation, generated schema, docs, runtime resolution, and focused tests; percentage strings are not implemented. The item is recent, unprotected, and fits a narrow core config/runtime enhancement rather than a cleanup close reason.

## What This Changes

Request percentage-based compaction token settings so values like `reserveTokensFloor` and `keepRecentTokens` scale with the active model context window.

## Best Possible Solution

Add a typed token-budget value contract that preserves existing numeric configs, accepts documented percentage strings for the selected compaction token settings, resolves them against the effective model context budget at runtime, and updates generated schema, docs, changelog, and focused tests together.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence static verification path: current type definitions, Zod schema, generated schema, docs, and runtime resolver all accept numbers only and have no percentage parser for these settings.

## Solution Assessment

Yes. Explicit percentage-string support is the narrowest maintainable direction because it keeps existing numeric configs backward-compatible; auto-scaling existing absolute numbers would silently change established behavior.
