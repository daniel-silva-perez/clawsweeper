## Summary

Keep open. Current main still treats the requested compaction token settings as absolute numeric values in the public type, strict Zod schema, generated config schema, docs, and runtime resolver; percentage strings such as `"10%"` are not accepted or resolved against the model context window.

## What This Changes

Add support for percentage-based values in context-aware compaction token settings such as `reserveTokensFloor` and `keepRecentTokens`.

## Best Possible Solution

Add a small typed token-budget value contract that preserves existing numeric behavior, accepts documented percentage strings for the selected compaction settings, resolves them against the active model or effective context window at runtime, and updates generated schema, docs, changelog, and focused tests together.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence static verification path: current schema, TypeScript types, generated schema, docs, and runtime code all accept numbers only and contain no percentage parser for these settings.

## Solution Assessment

Yes. Explicit percentage-string support is the narrowest maintainable direction because it keeps existing numeric configs backward-compatible; auto-scaling existing absolute numbers would silently change established behavior.
