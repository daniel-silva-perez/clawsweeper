## Summary

Keep open. Current main still has the voice-call Twilio auth-token SecretRef mismatch this PR targets: the manifest, docs, and bundled SecretRef collector advertise `plugins.entries.voice-call.config.twilio.authToken` as SecretRef-capable, but the voice-call runtime zod schema still accepts only strings and the runtime still passes that value directly into the Twilio provider.

## Best Possible Solution

Review and land this focused PR, or apply the same fix on main: make `twilio.authToken` a SecretInput in the voice-call schema, treat configured SecretRefs as present during setup validation, strictly resolve to a string immediately before constructing `TwilioProvider`, keep provider config typed on strings, and retain regression coverage for config parsing, plugin registration, runtime lifecycle, and provider construction.
