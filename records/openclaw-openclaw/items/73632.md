## Summary

Keep open. Current main still has the exact voice-call SecretRef schema mismatch this PR targets: the manifest/docs/SecretRef collector advertise `plugins.entries.voice-call.config.twilio.authToken` as a SecretRef surface, but the voice-call runtime zod schema still accepts only strings before runtime resolution can happen. This PR remains a focused implementation candidate rather than obsolete work.

## Best Possible Solution

Review and land this focused PR, or apply the same fix on main: make `twilio.authToken` a `SecretInput`, use configured-secret detection for setup validation, require strict string resolution before constructing `TwilioProvider`, and keep regression coverage for schema parsing, plugin registration, runtime lifecycle, and Twilio provider construction.
