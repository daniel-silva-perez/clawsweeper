## Summary

Keep this PR open. Current main still has the voice-call SecretRef mismatch the PR targets: `twilio.authToken` is advertised as SecretRef-capable in the manifest/docs but the runtime zod schema still accepts only strings before plugin registration completes. The Google realtime voice API-key SecretRef path is also still missing from bundled plugin SecretInput discovery, so the PR remains a live implementation candidate rather than obsolete.

## Best Possible Solution

Review and land this PR, or apply the same focused fix on main: widen `twilio.authToken` to the shared SecretInput schema, use `hasConfiguredSecretInput` for setup validation, strictly resolve to a string before constructing `TwilioProvider`, add `realtime.providers.*.apiKey` to the bundled voice-call SecretInput manifest/docs/tests, and keep the regression coverage for registration, config validation, runtime lifecycle, and bundled assignment collection.
