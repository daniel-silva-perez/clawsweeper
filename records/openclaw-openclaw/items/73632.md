## Summary

Keep open. Current main still has the voice-call SecretRef mismatch this PR targets: the manifest/docs advertise `twilio.authToken` as SecretRef-capable, but the runtime zod schema still only accepts strings before plugin registration completes. The realtime Google API-key path also is not yet declared in the bundled plugin SecretInput contract, so the PR is still a useful active fix candidate rather than obsolete.

## Best Possible Solution

Review and land this PR, or apply the same focused fix on main: widen the voice-call Twilio auth-token schema to the shared SecretInput contract, treat configured SecretRefs as present during setup validation, require a resolved string before constructing `TwilioProvider`, add `realtime.providers.*.apiKey` to the bundled voice-call SecretInput manifest/docs/tests, and keep the regression coverage described in the PR.
