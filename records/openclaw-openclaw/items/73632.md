## Summary

Keep open. Current main still has the exact voice-call SecretRef mismatch this PR targets: the manifest/docs/SecretRef collector advertise `plugins.entries.voice-call.config.twilio.authToken` as SecretRef-capable, but the voice-call runtime schema still accepts only strings before the runtime snapshot can resolve refs.

## Best Possible Solution

Review and land this focused PR, or apply the same fix on main: make `twilio.authToken` a `SecretInput`, treat configured SecretRefs as present during setup validation, require strict resolved-string normalization before `TwilioProvider` construction, and keep the proposed regression coverage for schema parsing, plugin registration, runtime lifecycle, and provider construction.
