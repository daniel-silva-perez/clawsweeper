## Summary

Keep this PR open. Current main still has the voice-call SecretRef mismatch: the manifest/docs advertise SecretRef-capable voice-call credentials, but the Twilio runtime zod schema still accepts only strings before plugin registration completes, and the Google realtime API-key SecretInput path is still absent from bundled voice-call SecretInput collection.

## Best Possible Solution

Keep this PR open and review/land it, or apply the same focused fix on main: widen the voice-call Twilio auth-token schema to the shared SecretInput contract, treat configured SecretRefs as present during setup validation, strictly resolve to a string immediately before constructing `TwilioProvider`, add `realtime.providers.*.apiKey` to the bundled voice-call SecretInput contract/docs/tests, and keep the targeted regression coverage.
