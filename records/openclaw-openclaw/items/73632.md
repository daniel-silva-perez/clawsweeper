## Summary

Keep open. Current main still has the voice-call manifest and public docs declaring SecretRef support for `plugins.entries.voice-call.config.twilio.authToken`, but the runtime zod schema still accepts only a non-empty string and plugin registration parses that schema before SecretRef runtime resolution can occur. This PR remains a focused implementation candidate, not obsolete work.

## Best Possible Solution

Review and land this focused PR, or apply the same fix on main: make `twilio.authToken` a `SecretInput`, treat configured SecretRefs as present in setup validation, resolve strictly to a string before constructing `TwilioProvider`, and keep regression coverage for schema parsing plus plugin registration with a SecretRef-backed Twilio auth token.
