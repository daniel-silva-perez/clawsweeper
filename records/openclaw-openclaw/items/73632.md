## Summary

Keep open. Current main still has the voice-call SecretRef mismatch this PR targets: Twilio auth tokens are advertised as SecretRef-capable, but the runtime zod schema still accepts only strings before plugin registration finishes, and the Google realtime API-key SecretInput path is still absent from bundled voice-call metadata. The PR is not obsolete and appears narrowly scoped, with no unrelated security-sensitive surface in the provided diff.

## What This Changes

This PR widens voice-call Twilio auth-token config to SecretInput, resolves it strictly before constructing TwilioProvider, adds realtime provider API-key SecretInput metadata/docs/tests, and adds focused regression coverage.

## Best Possible Solution

Land this PR or an equivalent focused patch that makes voice-call config parsing accept SecretInput for Twilio auth tokens, treats configured SecretRefs as present during setup validation, resolves strictly to a string only at Twilio provider construction, and adds realtime provider API-key SecretInput metadata/docs/collector coverage.
