## Summary

Keep open. The PR addresses a real current-main SecretRef gap in voice-call, but it is not ready to merge because adding the new realtime provider SecretInput manifest path leaves the generated SecretRef credential matrix/reference docs out of sync and CI is already failing on that drift.

## What This Changes

The PR widens voice-call Twilio auth-token config to SecretInput, resolves it before Twilio provider construction, declares realtime provider API-key SecretInputs in the voice-call manifest/docs, and adds focused config/runtime/collector tests.

## Best Possible Solution

Land this PR or an equivalent focused patch after syncing the generated SecretRef credential matrix and reference surface docs while preserving strict provider-time secret resolution.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: voice-call registration parses plugin config before runtime materialization while `twilio.authToken` is string-only, so a structured SecretRef object fails before the resolver can replace it; CI also reproduces the PR's docs-matrix regression.

## Solution Assessment

No, not as submitted. The functional approach is the narrow maintainable fix, but the manifest SecretInput expansion must also update the generated SecretRef matrix/reference docs before merge.
