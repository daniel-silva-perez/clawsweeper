## Summary

Keep open. Current main still has the voice-call SecretRef mismatch this PR targets: the manifest, docs, and bundled SecretRef collector advertise `plugins.entries.voice-call.config.twilio.authToken` as SecretRef-capable, but the voice-call runtime schema still accepts only strings before runtime resolution can happen. This PR remains a focused implementation candidate, not obsolete cleanup.

## Best Possible Solution

Review and land this focused PR, or apply the same fix on main: make `twilio.authToken` a `SecretInput`, treat configured SecretRefs as present during setup validation, require strict string resolution immediately before constructing `TwilioProvider`, and keep regression coverage for schema parsing, plugin registration, runtime lifecycle, and provider construction.
