## Summary

Keep open. Current main still has the voice-call SecretRef gap this PR targets: `twilio.authToken` is advertised as SecretRef-capable but the runtime zod schema still only accepts strings before resolution, and the newly referenced realtime Google API-key path is not in the bundled plugin SecretInput contract, so raw SecretRefs can still reach strict provider initialization.

## Best Possible Solution

Review and land this PR, or apply the same focused fix on main: widen the voice-call Twilio auth-token schema to the shared SecretInput contract, treat configured SecretRefs as present during setup validation, require a resolved string before constructing `TwilioProvider`, add `realtime.providers.*.apiKey` to the bundled plugin SecretInput manifest/docs/tests, and keep the regression coverage described in the PR.
