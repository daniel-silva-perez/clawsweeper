## Summary

Keep this issue open. Current main still has only a cron-specific Control UI chat-header filter and no general session-kind filter, while PR #73311 remains open and explicitly references this issue as its implementation candidate.

## What This Changes

Add a Control UI session dropdown filter that lets users show or hide sessions by kind, such as main, subagent, dreaming, cron, and isolated sessions.

## Best Possible Solution

Review and either land or close PR #73311. The desired end state is a UI-local desktop and mobile session-kind filter that preserves the selected session, avoids a gateway protocol change unless server-side filtering is deliberately added, and includes focused render/grouping coverage plus required user-facing release notes.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: current main renders only the existing session select and cron visibility toggle, and the Control UI sessions loader calls sessions.list without a kinds filter.

## Solution Assessment

Unclear pending PR review. A UI-local filter appears to be the narrowest maintainable direction because the gateway sessions.list schema currently does not accept kinds; passing kinds through the API would require an additive protocol/schema/test change.
