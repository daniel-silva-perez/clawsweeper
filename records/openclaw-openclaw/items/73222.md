## Summary

Keep this issue open. Current main still lacks a Control UI session-kind filter in the chat header dropdown, and an open linked PR already references this issue with closing syntax, so the right cleanup path is to review, land, revise, or close that PR before closing the issue.

## What This Changes

The issue requests a Control UI chat-header session dropdown filter for showing or hiding sessions by kind, such as main/direct, group, subagent, dreaming, cron, isolated, and other sessions.

## Best Possible Solution

Keep the issue open until PR #73311 is reviewed and either landed or closed. The maintainable end state should add a clear chat-header filter with tests for the visible session categories and docs for the Control UI behavior; if maintainers want server-side filtering instead of UI-local filtering, that should be an additive gateway protocol change with matching schema and contract tests.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: current main renders the chat session dropdown without any kind-filter control, and the Control UI session loader requests `sessions.list` without a kind filter, so all returned eligible rows can appear in the dropdown aside from the existing global/unknown and cron special cases.

## Solution Assessment

Unclear pending PR review. A UI-local filter appears to be a narrow path because the gateway API does not currently expose `kinds`, but maintainers should verify PR #73311’s taxonomy, selected-session preservation, desktop/mobile behavior, and docs before treating it as the final fix.
