## Summary

Keep this issue open. Current main still only has a cron-session hide toggle in the chat header controls, not a general session-kind filter for subagent, dreaming, isolated, or other noisy session categories, and open PR #73311 already references this issue as the active implementation candidate.

## What This Changes

Add a Control UI chat-header session dropdown filter for session kinds such as main, subagent, dreaming, cron, and isolated sessions.

## Best Possible Solution

Keep this issue open until PR #73311 is reviewed and either landed or closed. The desired end state is a tested desktop/mobile chat-header filter that preserves the selected session while hiding noisy categories, with docs updated for the Control UI behavior and a protocol change only if maintainers decide server-side filtering is required.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: current main renders the chat session dropdown without a general kind-filter control, and the sessions loader does not request or apply kind filters beyond the existing cron-key hide path.

## Solution Assessment

Unclear pending PR review. A UI-local filter is likely the narrowest maintainable path because the gateway protocol currently lacks a kinds parameter and the sessions_list tool performs its kind filtering locally.
