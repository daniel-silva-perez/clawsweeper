Keep this issue open. Current main still lacks a general Control UI chat-header session-kind filter, and PR #73311 remains open and unmerged with closing syntax for this issue, so the maintainer path is to review, update, land, or close that PR before resolving this issue.

## What This Changes

Add a Control UI session dropdown filter that lets users hide or show sessions by kind, such as main, subagent, dreaming, cron, and isolated sessions.

## Best Possible Solution

Keep this issue open until PR #73311 is reviewed and either landed or closed. The desired end state is a clear desktop and mobile chat-header kind filter that preserves the selected session, avoids unnecessary gateway protocol changes unless server-side filtering is intentionally added, and includes focused render and grouping tests.

## Reproduction Assessment

## Solution Assessment

Unclear pending PR review. A UI-local filter appears to be the narrowest current path because the gateway sessions.list schema does not accept kinds; a server-side filter would need an additive protocol/schema/test path.
