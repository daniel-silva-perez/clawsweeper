## Summary

Keep this issue open. Current main still lacks a general Control UI chat-header session-kind filter, and open PR #73311 explicitly targets this issue, so the maintainable path is to review, land, revise, or close that PR before closing the issue.

## What This Changes

Add a Control UI session dropdown filter so users can show or hide sessions by kind, reducing noise from subagent, dreaming, cron, isolated, and related non-main sessions.

## Best Possible Solution

Keep the issue open until PR #73311 is reviewed and either merged or closed. The desired end state is a clear desktop and mobile chat-header kind filter that preserves the selected session when hidden, avoids protocol churn unless server filtering is deliberately added, and has focused Control UI tests and docs.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main renders the chat session selector without a general kind-filter control, and the Control UI sessions loader calls sessions.list without a kinds parameter.

## Solution Assessment

Unclear pending PR review. A UI-local filter appears to be the narrowest current path because the gateway protocol does not currently accept kinds, but maintainers should verify the linked PR's kind mapping, responsive behavior, selected-session handling, docs, and tests before treating it as final.
