## Summary

Keep this PR open. Current main still does not implement the central chat-streaming changes from the branch, so it is not obsolete; however, the branch still needs maintainer review because it makes `load()` permanently no-op after the first successful bootstrap and includes a security-sensitive Windows installer script change outside the stated chat UI scope.

## What This Changes

This PR updates shared OpenClawKit chat parsing, agent-event payload decoding, chat view scrolling, ChatViewModel session/stream handling, related Swift tests, and a Windows installer failure-handling path.

## Best Possible Solution

Keep the chat stabilization work as an implementation candidate, but require a clean rebase on current main, preserve automatic refresh behavior outside the active-stream/bootstrap churn case, split or separately approve the installer script change, and cover decoded `sessionKey`, pending run IDs, session switching, and concurrent same-session runs with focused OpenClawKit tests plus an iOS build.
