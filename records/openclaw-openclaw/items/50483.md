## Summary

Keep this PR open. Current main still lacks the full shared chat streaming/layout fix, and the PR contains useful source work, but the latest head still blocks merge because `load()` becomes one-shot after first bootstrap and an unrelated Windows installer execution-path change is mixed into the chat patch.

## What This Changes

This PR updates shared OpenClawKit chat parsing, agent-event payload decoding, chat view scrolling and typing state, ChatViewModel session/stream handling, Swift regression tests, and Windows installer failure handling.

## Best Possible Solution

Revise the branch into a focused shared chat fix: keep the useful session-key decoding, parser stability, stream scoping, scroll/typing stability, and regression tests; preserve a later refresh path for chat appearances; and remove or split the Windows installer behavior into its own installer-reviewed PR.
