## Summary

Keep this PR open. Current main still has the underlying shared chat-streaming gaps this PR is trying to address, including transient parser segment IDs and no Swift decode field for `agent` event `sessionKey`; however, the PR is not merge-ready because it makes `load()` permanently skip later refreshes after a successful bootstrap and it mixes an unrelated PowerShell installer execution-path change into a chat UI fix.

## What This Changes

This PR updates shared OpenClawKit chat parsing, agent-event payload decoding, chat view scrolling, ChatViewModel session/stream handling, related Swift tests, and `scripts/install.ps1` failure handling.

## Best Possible Solution

Revise this PR into a focused shared chat fix: preserve a refresh path for later chat appearances while still avoiding bootstrap churn during active streams, keep the session-key/parser regression coverage, and remove or split the Windows installer change into its own reviewed installer PR.
