## Summary

Keep this PR open. Current main has overlapping chat terminal-event reconciliation, but it still does not implement the PR's central changes: stable parser segment IDs, debounced streaming scroll, Swift decoding of agent-event session keys, or the PR's broader live agent stream routing model. The PR also still carries an unrelated Windows installer change that should be split or explicitly reviewed.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to rebase onto current main, split or justify the scripts/install.ps1 change, preserve the still-missing stable parser IDs and streaming scroll debounce where valid, and settle the ChatViewModel live-stream routing contract around decoded agent-event sessionKey, pending run IDs, session switching, and concurrent same-session runs with focused OpenClawKit tests plus an iOS build.
