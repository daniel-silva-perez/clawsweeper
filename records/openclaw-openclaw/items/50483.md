## Summary

Keep this PR open. Current main still lacks the PR's central shared chat changes, including stable assistant-parser segment IDs and decoded agent-event session keys, so it is not obsolete; however, the branch still needs maintainer follow-up because it makes `load()` permanently no-op after the first successful bootstrap and includes an unrelated Windows installer execution-path change.

## What This Changes

This PR updates shared OpenClawKit chat parsing, agent-event payload decoding, chat view scrolling, ChatViewModel session/stream handling, related Swift tests, and `scripts/install.ps1` failure handling.

## Best Possible Solution

Keep the shared chat stabilization work as an implementation candidate, but require a focused update that preserves automatic refresh behavior outside the active bootstrap/stream churn case, removes or separately reviews the installer-script change, and keeps regression coverage for decoded `sessionKey`, session switching, pending run IDs, and concurrent same-session runs.
