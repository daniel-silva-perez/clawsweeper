## Summary

Keep this PR open. Current main still lacks the session-key agent-event decoding and stable parser IDs that make the chat fix useful, but the current branch still has merge-blocking review issues: `load()` stops refreshing after the first successful bootstrap, and the chat-focused PR includes an unrelated Windows installer execution-path change.

## What This Changes

This PR updates shared OpenClawKit chat parsing, agent-event payload decoding, chat view scrolling, ChatViewModel session/stream handling, Swift regression tests, and Windows installer failure handling.

## Best Possible Solution

Revise this into a focused shared chat fix: keep the session-key decoding, parser stability, stream-scoping, and regression coverage; preserve a refresh path for later chat appearances while still avoiding bootstrap churn during active streams; and remove or split the Windows installer change into its own reviewed installer PR.
