## Summary

Keep open. Current main still uses strict `minuteField === "0"` checks for default top-of-hour cron staggering, so the reported step/list/range cases are not implemented yet. There is also an open implementation candidate, PR #51594, that uses GitHub closing syntax for this issue, so the issue should remain open until that PR is reviewed/merged or rejected.

## Best Possible Solution

Review PR #51594 against the exact cron semantics OpenClaw wants, add/keep regression coverage for step, list, range, and 6-field minute-zero expressions, then close this issue only after the accepted fix lands on main.
