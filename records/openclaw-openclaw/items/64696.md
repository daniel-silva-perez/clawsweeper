## Summary

Keep open. Current main still lacks the requested memory-wiki bridge and lint durability fixes: link extraction still scans raw markdown, broken-link lint still compares raw targets without resolving page-relative Markdown paths, stale-pages still emits vault-root-relative native links from a report page, bridge render fingerprints do not include a renderer contract token, raw reply tags remain in fenced bridge content, and imported-source skip logic still trusts file existence rather than validating source-page frontmatter. The earlier fix PR was closed unmerged, and the newer open lint PR appears scoped to only part of the resolver problem, leaving the bridge render/cache repair criteria unhandled.

## What This Changes

Requests memory-wiki bridge import and lint fixes for relative Markdown link resolution, stale-pages report links, reply-tag sanitization, bridge render invalidation, and malformed cached source-page repair.

## Best Possible Solution

Implement the remaining fixes in the memory-wiki plugin source with regression coverage: make link extraction/resolution handle Markdown structure and page-relative native links, emit report-local native links for generated reports, prevent bridged reply directives from becoming lint targets, include a bridge renderer-contract token in source render fingerprints, and require valid imported source-page frontmatter before sync-state skips. Coordinate overlap with #70986 and #73722, but keep the bridge cache/render invalidation and malformed cached-page repair criteria covered in the final landed change.
