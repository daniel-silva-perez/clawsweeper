## Summary

Keep open. The PR targets a valid cron debugging gap, but the proposed implementation introduces a process-wide cwd mutation inside the Gateway and therefore is not safe to merge as-is; current main already routes cron execution through per-run `workspaceDir` plumbing rather than changing `process.cwd()`.

## What This Changes

The PR adds an `openclaw cron run --as-scheduled` flag and propagates an `asScheduled` boolean through the CLI, Gateway schema/handler, cron service, and isolated-agent runner, where it changes the process cwd around an isolated cron run.

## Best Possible Solution

Revise the PR so manual preview behavior is modeled through cron's existing per-run workspace, service-env, and session-context plumbing, without `process.chdir()`, then add focused CLI/Gateway/service regression tests plus docs and changelog coverage for the new flag.

## Reproduction Assessment

Yes for the introduced regression: during an `--as-scheduled` isolated cron run, any concurrent Gateway code that reads `process.cwd()` will observe the cron workspace until the awaited run completes. No high-confidence live reproduction was available for every part of the linked environment/context report in this read-only review.

## Solution Assessment

No. The patch changes a process-global cwd in an async Gateway process; the safer solution is to keep scheduled/manual parity inside the per-run cron execution context and runner APIs that already accept `workspaceDir`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep cwd changes out of the Gateway process:** `src/cron/isolated-agent/run.ts:1050`
  - body: When `--as-scheduled` is used, this calls `process.chdir()` before awaiting the full isolated agent run. `process.cwd()` is process-global, so concurrent Gateway RPCs, plugin hooks, or agent turns can resolve relative paths against this cron workspace until the run restores it. The cron runner already receives `workspaceDir`; use that per-run boundary instead of mutating global process state.
  - confidence: 0.9
