## Summary

Keep open. Current main still lacks the requested session-registry sweep in `openclaw tasks maintenance`, and the open linked PR is an implementation candidate that should be reviewed before the issue is closed.

## What This Changes

Request that `openclaw tasks maintenance --apply` also prune stale cron session registry entries while preserving jobs still marked running in durable cron job state.

## Best Possible Solution

Review and land PR #74023, or an equivalent narrow PR, that adds a `tasks maintenance` session-registry sweep with preview/apply JSON and text output, durable running-job protection, focused tests, and matching docs if the output changes.

## Reproduction Assessment

Yes. A source-level reproduction path is clear: on current main, inspect or run `openclaw tasks maintenance --json` with stale cron run session rows and the command has no code path or output field that can report or prune session-registry candidates.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix: extend the existing maintenance command with a conservative cron run session sweep guarded by durable cron job state, rather than changing general session cleanup or cron scheduling behavior.
