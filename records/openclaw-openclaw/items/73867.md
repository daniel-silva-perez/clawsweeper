## Summary

Keep open. Current main still leaves `openclaw tasks maintenance` scoped to task and Task Flow maintenance, while the requested session-registry sweep is not wired into that command and an open linked PR is already the implementation candidate.

## What This Changes

The request is to extend `openclaw tasks maintenance --apply` so it prunes stale cron session registry entries older than seven days unless they are protected by the running set in `jobs-state.json`.

## Best Possible Solution

Land PR #74023 or an equivalent focused change that adds preview/apply session-registry maintenance to `tasksMaintenanceCommand`, preserves active cron jobs using durable job state, updates JSON/text output, and covers the behavior in targeted command tests and docs.

## Reproduction Assessment

Yes. A stale `agent:<id>:cron:<jobId>:run:<runId>` row in `sessions.json` will not be touched by current `tasksMaintenanceCommand` because that command only invokes task and Task Flow maintenance; I verified this by source inspection rather than running a destructive apply path.

## Solution Assessment

Yes. The requested command-level sweep with `jobs-state.json` protection is the narrowest maintainable shape for operator cleanup, and reviewing the already-open focused PR is better than creating a duplicate repair branch.
