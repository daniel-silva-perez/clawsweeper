## Summary

Keep open. Current main still limits `openclaw tasks maintenance` to task and Task Flow maintenance, and the requested conservative session-registry sweep is already paired with open PR #74023, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that `openclaw tasks maintenance --apply` prune stale cron run session registry entries older than seven days while protecting jobs still marked running in `jobs-state.json`.

## Best Possible Solution

Keep this issue open until PR #74023, or an equivalent focused PR, lands a `tasks maintenance` session-registry sweep with preview/apply reporting, conservative cron-run filtering, `jobs-state.json` running-job protection, and targeted command coverage.

## Reproduction Assessment

Yes. Seed an old cron run session entry in a session store and run the current `tasks maintenance` code path in preview/apply mode; source inspection shows that path only invokes task and Task Flow maintenance, so it will not touch the session registry. I did not run the destructive `--apply` command in this read-only review.

## Solution Assessment

Yes. Reviewing and landing the linked focused PR is the best current route because it is already tied to this issue with closing syntax and targets the narrow command surface; a parallel ClawSweeper repair PR would duplicate that candidate.
