## Summary

Keep open. Current main still limits `openclaw tasks maintenance` to task and Task Flow registry maintenance, while the requested session-registry sweep remains unmerged and is already paired with open PR #74023.

## What This Changes

The issue requests that `openclaw tasks maintenance --apply` also prune stale session registry entries using conservative age and running-job guards.

## Best Possible Solution

Keep this issue open until PR #74023, or an equivalent focused PR, lands a `tasks maintenance` session-registry sweep with preview/apply output, durable running-job protection, and targeted command coverage.

## Reproduction Assessment

Yes. Seed an old cron run session entry in `sessions.json`, run `openclaw tasks maintenance --apply`, and the current command path will not touch it because it only invokes task and Task Flow registry maintenance; I did not execute that destructive path in this read-only review.

## Solution Assessment

Yes, reviewing and landing the linked focused PR is the best current route. A parallel ClawSweeper repair PR would duplicate the open implementation candidate unless maintainers close or reject that PR.
