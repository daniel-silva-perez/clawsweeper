## Summary

Keep open. Current main still limits `openclaw tasks maintenance` to task and Task Flow registry maintenance, while the requested stale cron session-registry sweep is not implemented and is already paired with open PR #74023 using closing syntax.

## What This Changes

Request that `openclaw tasks maintenance --apply` also prune stale cron run entries from `sessions.json` when they are older than seven days and not protected by the `jobs-state.json` running set.

## Best Possible Solution

Land PR #74023 or an equivalent focused change that adds preview/apply session-registry maintenance to `tasksMaintenanceCommand`, includes JSON/text output, protects currently running cron jobs using durable job state, and updates targeted command tests and docs without changing unrelated session lifecycle behavior.

## Reproduction Assessment

Yes. Seed an old `agent:<id>:cron:<jobId>:run:<runId>` entry in `sessions.json`, run `openclaw tasks maintenance --apply`, and current main will not touch it because the command only invokes task and Task Flow maintenance; I did not execute that destructive path in this read-only review.

## Solution Assessment

Yes. Reviewing and landing the linked focused PR is the best current route because it matches the requested boundary; a parallel ClawSweeper repair branch would duplicate an open implementation candidate.
