## Summary

Keep open. Current main still leaves `openclaw tasks maintenance` limited to task and Task Flow maintenance, and PR #74023 is an open, unmerged implementation candidate that references this issue for closure.

## What This Changes

Request `openclaw tasks maintenance --apply` to also prune stale cron session registry entries while preserving entries for cron jobs still marked running in `jobs-state.json`.

## Best Possible Solution

Land PR #74023 or an equivalent focused change that adds preview/apply session-registry maintenance to `tasksMaintenanceCommand`, protects currently running cron jobs using durable job state, updates text/JSON output and docs, and covers the behavior in targeted command tests.

## Reproduction Assessment

Yes. Source inspection shows a stale cron run session row in `sessions.json` is outside the current `tasksMaintenanceCommand` path, because that command only runs task-registry and Task Flow maintenance; I did not run a destructive apply path.

## Solution Assessment

Yes. A command-level sweep with jobs-state protection is the narrow maintainable fix, and the already-open linked PR is the right implementation path to review rather than creating duplicate automation.
