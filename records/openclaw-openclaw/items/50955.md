## Summary

Keep open. Current main still only partially overlaps this PR: `sessions_send` now has plugin-owned session-key parsing and thread propagation, but native ACP `sessions_spawn(runtime:"acp", thread:true, mode:"session")` still does not share the `/acp spawn` current-context placement and ownership path, and current main still lacks the PR's binding-aware reverse-delivery hook. The PR also still has an unresolved Feishu regression risk from its diff.

## Best Possible Solution

Keep this PR open for maintainer or contributor follow-up. Best path is a fresh rebase or narrower replacement that shares one origin-aware placement resolver between `/acp spawn` and native `sessions_spawn`, preserves Feishu action handlers, enforces the current-binding `boundBy` ownership gate, adds binding-aware reverse delivery for bootstrap/completion routing, and keeps focused regression tests for Discord DM/thread/parent-channel, Telegram DM/topic, Feishu topic routing, bound completion delivery, and threaded `sessions_send` announce routing. Open PR #64322 is complementary for Discord per-thread session keys, not a superseding fix for this broader placement and delivery-routing scope.

## Risks / Open Questions

- Landing this PR as-is can regress Feishu message actions because the provided diff removes handlers that current main still advertises and implements.
- The native current-binding path needs the same `boundBy` ownership check as `/acp spawn`; otherwise current-surface rebinding can bypass the ownership gate.
- The PR's broad routing changes touch channel messaging contracts and high-traffic ACP/subagent delivery paths, so it needs a fresh rebase and focused regression coverage before maintainer merge consideration.
- Security/supply-chain pass found no new workflow/action refs, dependency sources, lockfiles, release scripts, package lifecycle hooks, or secret-scope changes in the provided file list; the review blockers are routing correctness and Feishu action handling.
