## Summary

Keep this PR open. Current main only partially overlaps the intended fix: `sessions_send` announce routing has moved forward, but native ACP `sessions_spawn(runtime:"acp", thread:true, mode:"session")` still does not share the `/acp spawn` current-vs-child placement path, and the PR still has unresolved functional/security-sensitive review concerns.

## Best Possible Solution

Keep this PR open for maintainer or contributor follow-up. The best path is a fresh rebase or narrower replacement that shares one origin-aware placement resolver between `/acp spawn` and native `sessions_spawn`, preserves Feishu action handlers, enforces the current-binding `boundBy` ownership gate, fixes Telegram direct-binding topic parsing, and adds focused regression tests for Discord DM/thread/parent-channel, Telegram DM/topic, Feishu topic routing, bootstrap delivery, bound completion delivery, and threaded `sessions_send` announce routing.

## Risks / Open Questions

- Landing this PR as-is can regress Feishu advertised message actions by dropping handlers that current main still provides.
- The PR's current-binding direction must include the same `boundBy` ownership gate as `/acp spawn`; otherwise another participant could rebind an existing conversation surface.
- Closing this PR now would lose unresolved native ACP placement, bootstrap delivery, bound completion routing, and Telegram DM/topic edge cases that current main has not definitely implemented.
- Related open PR #64322 is complementary for Discord per-thread session keys, but it does not supersede this PR's broader native placement and delivery-routing scope.
