## Summary

Keep this PR open. Current main has later task-maintenance hardening for stale chat-backed CLI rows, but it does not implement this PR's intended behavior: background exec/process runs still do not create linked `runtime:"cli"` task records, and `process remove` still hides a live cancelled session immediately. This is not safe to close as implemented-on-main, and the PR also has unresolved review findings that need maintainer/rework attention before merge.

## Best Possible Solution

Keep the PR open for maintainer review or rework. A viable version should rebase onto current main, reuse the current detached-task/runtime-aware maintenance model, create/finalize task records only for actual background exec runs, keep task `lastEventAt` fresh on output, avoid overriding session status from unrelated tasks, and preserve process visibility during pending termination with focused regression tests.
