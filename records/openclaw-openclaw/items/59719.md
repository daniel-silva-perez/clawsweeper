## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks the PR's central behavior, because background exec/process runs only enter the process registry and `process remove` still hides a live cancelled session immediately; however the PR branch has unresolved correctness findings and is stale against later task-maintenance work on main.

## What This Changes

The PR wires background `exec`/`process` runs into linked `runtime:"cli"` task records and updates process removal, task maintenance, and session-row status derivation around those linked CLI tasks.

## Best Possible Solution

Rework the PR on current main: create and finalize task records only for real background exec runs, keep process visibility while termination is pending, update task heartbeat on every meaningful output/liveness event, and scope session-row overrides to the linked background exec CLI task without regressing other CLI task producers.

## Reproduction Assessment

Yes. The current-main gap is reproducible by source inspection and existing tests: background exec yield only calls `markBackgrounded()`, and `process remove` has a regression test asserting immediate removal; the PR regressions are reproducible from the diff paths that route all CLI tasks through a bash-process-only liveness hook.

## Solution Assessment

No. The product direction is useful, but the submitted implementation is not the best fix because it treats all CLI tasks as bash-process-backed, lets unrelated owner tasks drive session rows, and does not keep task heartbeat fresh on ordinary output.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restrict CLI orphan checks to exec-backed run IDs:** `src/tasks/task-registry.maintenance.ts:68-74`
  - body: This change routes every `runtime:"cli"` task through the bash process registry observer. CLI tasks created by other producers, such as gateway agent or media runs, use non-process run IDs and can be marked `lost` after the short CLI grace even while their own runtime is active.
  - confidence: 0.91
- **[P2] Only let linked CLI tasks override session rows:** `src/gateway/session-utils.ts:1214-1229`
  - body: `buildGatewaySessionRow()` now derives row status and timing from `listTasksForOwnerKey(key)` across all task runtimes. A recent non-CLI or unrelated owner task can make a completed session appear running or failed, so this override needs to be scoped to the background exec task this PR adds.
  - confidence: 0.86
- **[P2] Refresh CLI task heartbeat on every output chunk:** `src/agents/bash-tools.exec-runtime.ts:673-692`
  - body: `reportCliTaskProgress()` only writes `lastEventAt` when a summary string is present. After the first output, normal ongoing output passes no summary, leaving the task heartbeat and session-row `updatedAt` stale for a live command.
  - confidence: 0.82
