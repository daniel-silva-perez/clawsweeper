## Summary

Keep open. Current main still lacks a Feishu thread-specific mention policy, but this PR is not safe to merge as-is because the latest predicate can treat ordinary quote replies as thread override candidates and the new public config key is not reflected in docs/plugin metadata. The overlapping #48945 implementation also needs maintainer reconciliation before choosing the final API shape.

## What This Changes

The PR adds a Feishu `requireMentionInThread` global/per-group config option, wires thread-awareness into `resolveFeishuReplyPolicy`, and adds policy tests for relaxing or tightening @mention requirements in thread replies.

## Best Possible Solution

Converge on one Feishu-owned thread follow-up model that distinguishes topic-scoped bot follow-ups from ordinary group quote replies, preserves the top-level @mention gate, updates schema/plugin metadata/docs/generated config together, and covers topic mode, explicit `replyInThread`, quote replies, broadcast routing, and bidirectional override cases.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep normal quote replies behind the mention gate:** `extensions/feishu/src/bot.ts:421-425`
  - body: The new predicate treats `groupSession.replyInThread` as proof that a message is a thread reply. On current main, `resolveFeishuGroupSession` sets `replyInThread` true for any inbound `root_id` or `thread_id`, including ordinary group quote replies, so `requireMentionInThread: false` can let unmentioned quote replies pass the group @mention gate.
  - confidence: 0.93

That leaves accepted config out of sync with user-facing docs and config editors.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff changes Feishu group mention-gating and introduces a concrete path for ordinary unmentioned quote replies to reach the agent when the new thread override is disabled.

Concerns:

- **[medium] Mention-gate bypass for quote replies:** `extensions/feishu/src/bot.ts:421`
  - body: Because the PR includes `groupSession.replyInThread` in the thread override predicate, and current main sets that value for any rooted/threaded inbound reply, `requireMentionInThread: false` can cause normal group quote replies to bypass the @mention gate. That broadens which group messages can trigger processing under a noise/access-control setting.
  - confidence: 0.91
