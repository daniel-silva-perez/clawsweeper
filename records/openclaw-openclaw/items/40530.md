## Summary

Keep this PR open. Current main still lacks the Feishu thread-specific mention override, but this PR should not land as-is because the latest predicate can weaken ordinary group quote-reply mention gating and the user-facing config surface is incomplete; the overlapping #48945 needs Feishu maintainer reconciliation.

## What This Changes

The PR adds a Feishu `requireMentionInThread` boolean at global and per-group config levels, wires it into reply mention policy through an `isThreadReply` input, and adds policy tests for relaxing and tightening thread mention requirements.

## Best Possible Solution

Land one Feishu-owned implementation that treats topic-scoped follow-ups as intentional only when the session really belongs to a bot thread, preserves the ordinary group mention gate, updates schema/docs/generated config metadata together, and includes regression coverage for topic sessions, reply-in-thread groups, normal quote replies, top-level messages, and bidirectional override behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Keep quote replies behind the mention gate:** `extensions/feishu/src/bot.ts:421`
  - body: The added predicate includes `groupSession?.replyInThread`, but current main sets that flag whenever an inbound message has `root_id` or `thread_id`. With `requireMentionInThread: false`, an ordinary quote reply in a normal group can be treated as a thread reply and skip `requireMention`, so the bot may answer unmentioned main-chat replies.
  - confidence: 0.9

That leaves the accepted option undiscoverable from the documented/config-editor surfaces that are expected to stay aligned with channel schema changes.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The PR changes a mention-gating decision and currently introduces a concrete unmentioned-message processing risk for ordinary Feishu quote replies.

Concerns:

- **[medium] Mention-gate bypass for quote replies:** `extensions/feishu/src/bot.ts:421`
  - body: Because the PR treats `groupSession.replyInThread` as a thread override signal, `requireMentionInThread: false` can cause normal group quote replies with `root_id` to bypass the `@mention` gate. That broadens which group messages can reach the agent under a security-relevant access/noise control.
  - confidence: 0.88
