## Summary

Keep this PR open. Current main still lacks a Feishu thread-specific mention override, and this PR remains a plausible implementation candidate, but the latest PR predicate appears unsafe against current main's `replyInThread` semantics and overlaps with open #48945.

## Best Possible Solution

Keep this PR open for Feishu maintainer review. The best path is to reconcile it with #48945, then land one rebased implementation that adds schema, generated config metadata, docs, and regression tests for topic-scoped sessions, `replyInThread: "enabled"` groups, bidirectional overrides, ordinary quote replies, and top-level mention gating.
