## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks the requested Feishu thread mention override, so the contribution is not obsolete; however, the latest PR predicate can still let ordinary rooted quote replies bypass the @mention gate, and the new public config key is not carried through docs or plugin metadata. The overlapping #48945 thread-follow-up API also needs maintainer reconciliation before this lands.

## What This Changes

The PR adds a Feishu global and per-group `requireMentionInThread` config option, passes thread context into Feishu reply-policy resolution, and adds policy tests for thread mention overrides.

## Best Possible Solution

Converge on one Feishu-owned thread follow-up design that only relaxes @mention requirements for bot-owned or topic-scoped follow-ups, keeps ordinary group quote replies gated, and updates schema, plugin metadata, docs, generated config checks, and targeted bot/policy tests together.

## Security Review

Status: needs_attention
