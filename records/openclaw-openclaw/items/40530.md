## Summary

Keep this PR open, but do not land it as-is. Current main still does not implement a Feishu thread-specific mention override, and the contribution is aimed at a real Feishu friction point, but the latest patch still lets ordinary Feishu quote replies bypass the mention gate through `groupSession.replyInThread`. It also leaves the new user-facing config key out of the docs/generated config metadata, and it overlaps with the more conservative active-topic design in #48945.

## What This Changes

The PR adds a Feishu `requireMentionInThread` boolean at global and per-group config levels, passes a thread-reply flag into reply mention policy, and adds policy tests for relaxing and tightening mention requirements in threads.

## Best Possible Solution

Land one Feishu-owned thread follow-up policy that only treats intentional bot topic follow-ups as implicit mentions, preserves ordinary group quote replies behind the mention gate, aligns schema/docs/generated config metadata, and includes bot-level regression coverage for topic scopes, configured `replyInThread`, normal quote replies, top-level messages, and bidirectional override behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep quote replies behind the mention gate:** `extensions/feishu/src/bot.ts:421`
  - body: The added predicate includes `groupSession?.replyInThread`, but current main sets `replyInThread` true whenever an inbound Feishu group message has `root_id` or `thread_id`. With `requireMentionInThread: false`, ordinary quote replies in normal groups can therefore bypass `requireMention` and dispatch to the agent without an @mention.
  - confidence: 0.92

  - body: Adding `requireMentionInThread` to the strict Feishu schema makes it a public config key, but the PR does not update the Feishu docs or generated bundled channel config metadata. That leaves documented/config-editor surfaces out of sync and should fail the config docs generation check.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff changes Feishu mention-gating behavior and currently allows a concrete unmentioned-message processing path for ordinary quote replies.

Concerns:

- **[medium] Mention-gate bypass for quote replies:** `extensions/feishu/src/bot.ts:421`
  - body: Because the PR treats `groupSession.replyInThread` as a thread override signal, `requireMentionInThread: false` can cause normal group quote replies with `root_id` to bypass the @mention gate and reach the agent. That broadens which group messages can trigger processing under an access/noise control.
  - confidence: 0.9
