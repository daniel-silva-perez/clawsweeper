## Summary

Keep this PR open. Current main still lacks any Feishu thread-specific mention override, so the intended behavior is not implemented. The PR remains a plausible contribution, but its latest predicate is risky against current main because `groupSession.replyInThread` is already true for any inbound `root_id` or `thread_id`, which can make ordinary quote replies bypass the group mention gate. There is also overlapping open work in #48945, so maintainers should reconcile the product shape rather than auto-close this PR.

## Best Possible Solution

Keep this PR open for Feishu maintainer review. The best path is to reconcile it with #48945, choose one Feishu thread-follow-up configuration model, then land a rebased implementation that updates schema, runtime policy, docs, plugin metadata, and regression tests for topic-scoped sessions, replyInThread-enabled groups, ordinary quote replies, and bidirectional mention overrides without weakening normal group mention gating.
