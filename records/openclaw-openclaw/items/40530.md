## Summary

Keep open. Current main does not implement a Feishu thread mention override, and this PR’s latest predicate still misclassifies ordinary rooted quote replies as thread replies because current session resolution makes `replyInThread` true for any `root_id` or `thread_id`. The overlapping #48945 proposes a safer Feishu-owned thread follow-up model, so this needs maintainer API reconciliation and branch rework before merge.

## What This Changes

The PR adds a Feishu global and per-group `requireMentionInThread` config option, passes thread context into Feishu reply-policy resolution, and adds policy tests for thread mention overrides.

## Best Possible Solution

Converge on one Feishu thread follow-up design that distinguishes bot-owned or topic-scoped follow-ups from ordinary group quote replies, then update the schema, plugin metadata, docs, generated config checks, and bot/policy tests together.

## Reproduction Assessment

Yes for the blocking PR bug: current main shows `replyInThread` becomes true for any `root_id` or `thread_id`, and the PR uses that value in the mention-bypass predicate. A focused bot test with a normal group quote reply, `requireMention: true`, and `requireMentionInThread: false` should reproduce the unwanted dispatch.

## Solution Assessment

No. The current PR is not the best fix because it adds a broad boolean override without safely distinguishing topic follow-ups from ordinary quote replies; #48945’s active/topic/off direction is a safer API candidate for maintainer reconciliation.

## Review Findings

- **[P1] Keep quote replies behind the mention gate:** `extensions/feishu/src/bot.ts:421-425`
  - body: The new `isThreadReply` predicate includes `groupSession?.replyInThread`, but current `resolveFeishuGroupSession` sets that value to true whenever an inbound message has `root_id` or `thread_id`. In a normal group, a plain quote reply with `requireMentionInThread: false` would therefore skip `requireMention` even though current main treats that case as a normal quote reply, not a topic follow-up.
  - confidence: 0.93

  - confidence: 0.84
