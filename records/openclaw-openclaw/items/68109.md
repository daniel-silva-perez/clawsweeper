## Summary

Keep open. Current main still does not show an exit-safe Linux systemd restart handoff for the direct `gateway restart` path; it awaits `systemctl --user restart` synchronously, while adjacent launchd and update-restart code already handle the same process-tree termination class with detached handoff logic. The related fix PR is closed unmerged and was not a safe canonical fix.

## What This Changes

The issue asks for successful `openclaw gateway restart` runs on systemd-managed Linux gateways to report exit code 0 instead of a null child-process exit result.

## Best Possible Solution

Add a narrow Linux/systemd restart handoff for direct gateway service restarts when the caller is at risk of being killed by the managed service restart, using the existing scheduled-restart result shape where appropriate. Keep normal synchronous failure reporting for external invocations, and add a regression that observes the spawned CLI child exit result rather than only mocked systemctl arguments.
