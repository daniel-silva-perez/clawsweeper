## Summary

Keep open. Current main still shows the reported transport gap: `--force-document`/`forceDocument` is parsed and accepted, but the gateway send schema and gateway request payload do not carry it, and the WhatsApp outbound adapter/send API do not consume it. The docs and agent tool schema describe the flag as Telegram-only, but the CLI still accepts it globally, so the issue remains actionable rather than closeable.

## Best Possible Solution

Keep the issue open. The best fix is either to reject `--force-document`/`forceDocument` for WhatsApp before dry-run/dispatch, or to add a supported WhatsApp document-forcing contract end-to-end: CLI/action params, gateway `SendParamsSchema`, `sendMessage` gateway payload, gateway server handler, delivery context, WhatsApp outbound adapter, `sendMessageWhatsApp`, `ActiveWebSendOptions`, and the Baileys payload builder, with regression tests for dry-run payloads and real adapter payload selection.
