## Summary

Keep this issue open. Current main still lacks a typed global or per-model total generation timeout for active model generation: `agents.defaults.models` entries only allow alias/params/streaming, the strict/generated schemas reject timeout keys there, provider `timeoutSeconds` is request/idle-timeout plumbing, and the embedded runner still schedules the total abort from the run-level `params.timeoutMs`. The follow-up first-token discussion and closed #22368 are related but do not implement this requested total generation budget.

## Best Possible Solution

Implement a typed global and per-model text-generation timeout contract, resolve the effective total generation budget per fallback candidate, enforce it with a timer that does not reset when chunks arrive, classify/log the abort as a timeout for fallback handling, and update config types, zod/generated schema, help/docs, changelog, and focused regression tests. Keep first-token timeout as a separate follow-up unless maintainers explicitly choose to include it.
