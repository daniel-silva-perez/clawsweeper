## Summary

Keep open because current main still lacks the requested typed global/per-model non-resetting generation timeout. The code has adjacent run-level, provider request, and reset-on-chunk idle timeout controls, but no config contract or runtime lookup that applies a generation budget per selected model candidate.

## What This Changes

Add configurable global and per-model generation timeout budgets that abort long-running model generation, log timeout events, and optionally enable fallback handling.

## Best Possible Solution

Add a backwards-compatible, documented generationTimeout contract at agent defaults and model catalog entries, resolve the effective non-resetting budget for each selected fallback candidate, enforce it during model generation with timeout classification/logging, and leave first-token/provider-skip behavior to separate work.
