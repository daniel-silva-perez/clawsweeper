## Summary

Keep open because current main still does not provide the requested global or per-model non-resetting generation timeout. The adjacent provider request timeout and stream idle watchdog help with unresponsive or idle providers, but they do not enforce a total generation budget for a model that keeps streaming reasoning/planning chunks.

## What This Changes

The issue requests configurable global and per-model generation timeouts that abort long-running model generation, log the timeout, and optionally allow fallback.

## Best Possible Solution

Add a backwards-compatible, documented generation timeout contract at agent defaults and model catalog entries, resolve the effective non-resetting budget for the selected fallback candidate, enforce it during active generation with clear timeout classification/logging, and leave first-token/provider-skip behavior to separate follow-up work.
