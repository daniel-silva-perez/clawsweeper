## Summary

Keep this open. Current main still lacks the requested typed global/per-model total-generation timeout contract under `agents.defaults.models`; the existing knobs cover the whole run timeout, provider HTTP/request timeout, and an idle watchdog that restarts after stream chunks. The April 26 review in the discussion reached the same conclusion, with #44310 closed as a duplicate of this remaining work and first-token timeout requests kept separate.

## What This Changes

Request a global and per-model generation timeout so runaway model output/planning can be aborted, logged, and handled by fallback logic.

## Best Possible Solution

Add a backwards-compatible typed total-generation timeout surface at the agent default and per-model catalog levels, resolve the effective budget for each selected fallback candidate, enforce it with a non-resetting timer during active generation, classify/log the abort as a timeout for fallback handling, and update schema/help/docs/changelog/tests while preserving the existing run-level and provider-request timeout meanings.

## Repair Work Prompt

Implement per-model total generation timeout support for this issue. Add a typed, backwards-compatible config surface for a global `agents.defaults.generationTimeout` duration and per-model `agents.defaults.models["provider/model"].generationTimeout` override, using existing duration parsing conventions and preserving current meanings of `agents.defaults.timeoutSeconds` (whole run timeout) and `models.providers.*.timeoutSeconds` (provider HTTP/request timeout). Resolve the effective generation budget for the selected fallback candidate before each embedded model attempt, enforce it with a non-resetting active-generation timer, abort/classify/log it as a timeout so model fallback can continue, and ensure the timer is cleaned up on success, error, external abort, compaction, and stream close. Update config types, zod schema, generated schema/help/labels if needed, docs, changelog with reporter credit, and focused tests. Do not implement first-token timeout or provider-level skip in this PR unless a maintainer explicitly expands scope; #2222, #8802, #22364, and PR #22368 are adjacent context only.
