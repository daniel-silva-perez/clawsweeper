## Summary

Keep this issue open. Current main still has adjacent run-level, provider-request, and reset-on-chunk idle timeout controls, but it does not expose or enforce the requested typed global/per-model non-resetting generation timeout for active model output loops; the maintainer and prior ClawSweeper reviews in the discussion reached the same conclusion.

## What This Changes

Add a global and per-model generation timeout configuration that aborts long-running model generation, returns/logs a timeout, and can participate in fallback handling.

## Best Possible Solution

Add a documented generation-timeout contract at agent defaults and per-model catalog entries, resolve the effective budget per fallback candidate, enforce it with a non-resetting timer during active generation, and keep first-token/provider-skip behavior as separate work.

## Reproduction Assessment

Yes for the missing feature: the proposed per-model key is outside the strict schema, and current runner enforcement uses params.timeoutMs plus a reset-on-chunk idle watchdog. A live Gemini infinite-planning loop is not needed to prove the config/runtime gap.

## Solution Assessment

Yes. A narrow core config/schema plus embedded-runner enforcement change is the maintainable fix; provider-specific request timeouts or first-token timeout changes solve related but different failure modes.
