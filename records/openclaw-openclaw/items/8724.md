## Summary

Keep this issue open. Current main still lacks a typed global or per-model total generation timeout for active streaming/model loops. Existing timeout surfaces are a whole-agent run timeout, provider-scoped HTTP/request timeout metadata, and an idle/no-chunk watchdog that resets as chunks arrive. Related #22368 was closed unmerged, and related #44310 is closed but did not ship this contract.

## Best Possible Solution

Implement a typed global and per-model text-generation timeout contract. The runtime should resolve the effective total generation budget per fallback candidate, enforce it with a timer that does not reset on streamed chunks, abort/log/classify it as a timeout for fallback handling, and update config types, zod/generated schema, help/docs, changelog, and focused regression tests. First-token timeout can remain a separate follow-up unless maintainers intentionally include it in the same contract.
