## Summary

Keep this issue open. Current main still lacks the requested typed global/per-model non-resetting generation timeout: the per-model catalog schema remains strict with only alias/params/streaming, the run path still forwards one run-level timeout into every attempt, and the model idle watchdog resets around each streamed chunk rather than enforcing a total generation budget.

## What This Changes

The issue requests configurable global and per-model generation timeouts so long-running streaming model generations can be aborted and handled by fallback/routing without relying only on provider request or idle timeouts.

## Best Possible Solution

Add a backwards-compatible, documented generation-timeout contract at agent defaults and per-model catalog entries, resolve the effective non-resetting budget for each selected fallback candidate, enforce that budget during active generation with clear timeout classification/logging, and leave first-token/provider-skip work to separate issues or PRs.
