## Summary

Keep this issue open. Current main still lacks a typed global or per-model total generation timeout for active streaming loops; the implemented timeout surfaces are strict per-model catalog metadata without timeout keys, whole-run timeout, provider-scoped HTTP/request timeout, and idle/no-chunk watchdog behavior.

## Best Possible Solution

Keep this issue open until OpenClaw implements a typed global and per-model text-generation timeout contract. The fix should resolve the effective timeout per fallback candidate, enforce a total generation-budget timer that does not reset on streamed chunks, abort/log/classify that timeout for fallback handling, and update config schema/help/docs plus focused regression tests.

## Clownfish Work Prompt

Implement the still-missing per-model/global text-generation timeout for this issue. Work in OpenClaw core, not a plugin. Read `pnpm docs:list` output first and read only relevant docs such as `docs/concepts/agent-loop.md`, `docs/concepts/model-providers.md`, and `docs/cli/config.md`.

Observable problem: `agents.defaults.models.*` has no typed timeout key, and the current runtime only has whole-run timeout plus provider HTTP/request and idle/no-chunk timeouts. A model that keeps streaming reasoning/planning chunks can run until the whole agent timeout. Add a typed generation timeout that expires based on total model generation elapsed time and does not reset when stream chunks arrive.

Expected fix boundary:
- Add a repo-consistent global and per-model config contract under `agents.defaults` / `agents.defaults.models[provider/model]` for text generation timeout. Prefer existing timeout naming/unit conventions unless nearby config docs/schema point to a better established pattern; avoid unreviewed broad alias proliferation.
- Update TypeScript config types, zod schema, generated schema/help/labels/baseline, and user docs.
- Resolve the effective generation timeout per model/fallback candidate, with per-model override winning over global default and unset preserving current behavior.
- Enforce the timeout in the embedded model generation path as a total generation budget separate from `streamWithIdleTimeout` and separate from the whole-run `params.timeoutMs` timer.
- Abort, log, and classify generation-budget expiry so fallback can try the next candidate when configured, without changing user abort, compaction timeout, context-overflow, or provider HTTP timeout semantics.
- Add regression coverage for a stream that keeps emitting chunks past the configured generation budget, plus config schema acceptance/rejection tests. Do not change provider-owned request timeout semantics or bundled plugin behavior beyond consuming the generic core seam.

Changelog: add a user-facing Fixes/Changes entry only if the repo’s current changelog policy requires it for this config/runtime behavior, with proper `Thanks @...` attribution to the reporter.

Do not edit unrelated provider catalogs, dependency pins, lockfiles, or baseline/snapshot files except generated config/docs artifacts required by the schema change.
