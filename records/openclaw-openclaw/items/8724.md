## Summary

Keep this issue open because current main still lacks a typed global or per-model non-resetting generation timeout. The existing controls are adjacent run-level, provider request, and reset-on-chunk idle timeouts, and the proposed per-model catalog key is still rejected by the strict agent defaults schema.

## What This Changes

The issue requests configurable global and per-model generation timeouts so long-running model output can be aborted and optionally routed through fallback models.

## Best Possible Solution

Add a backwards-compatible generation-timeout contract at agent defaults and model catalog entries, resolve the effective non-resetting budget for each selected fallback candidate, enforce it during active generation with timeout classification/logging, and keep first-token timeout/provider-skip behavior as separate work.
