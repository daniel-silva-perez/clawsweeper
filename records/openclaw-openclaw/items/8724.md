## Summary

Keep this issue open. Current main still lacks a typed global/per-model total text-generation timeout contract: `agents.defaults.models.*` accepts only `alias`, `params`, and `streaming`; the strict and generated schemas reject timeout keys there; provider `models.providers.*.timeoutSeconds` is request/HTTP timeout plumbing; and the embedded runner still schedules the total abort from the run-level `params.timeoutMs`. Related #44310 was closed as a duplicate of this issue, and closed #22368 covered first-token/provider-skip behavior rather than this total generation budget.

## Best Possible Solution

Implement a typed global and per-model total text-generation timeout contract, resolve the effective budget per selected fallback candidate, enforce it with a non-resetting timer for active generation, classify/log the abort as timeout for fallback handling, and update config types, zod/generated schema, help/docs, changelog, and focused regression tests. Keep first-token timeout as a separate follow-up unless maintainers intentionally include it.
