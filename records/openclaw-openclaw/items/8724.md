## Summary

Keep this issue open. Current main still has adjacent run/provider/idle timeout controls, but it does not implement the requested typed global and per-model non-resetting generation timeout, and the existing idle watchdog still resets after each streamed chunk.

## What This Changes

The issue requests configurable global and per-model generation timeouts so runaway model generations can be aborted and optionally routed into fallback handling.

## Best Possible Solution

Add a backwards-compatible generation-timeout contract at agent defaults and per-model catalog entries, resolve the effective non-resetting budget for each selected fallback candidate, enforce it during active generation, and leave first-token/provider-skip behavior to the already separate related work.
