## Summary

Keep this issue open. Current main still lacks the requested typed global/per-model non-resetting generation timeout: per-model agent defaults are still strict and do not accept a timeout key, the embedded runner still forwards a single run timeout into attempts, and the adjacent idle watchdog resets after each streamed chunk rather than enforcing total generation duration.

## What This Changes

Request a global and per-model generation timeout configuration that aborts long-running LLM generation and can participate in fallback handling.

## Best Possible Solution

Add a backwards-compatible, documented generation-timeout contract at agent defaults and per-model catalog entries, resolve the effective non-resetting budget for each selected fallback candidate, enforce it during active generation with clear timeout logging/classification, and keep first-token/provider-skip behavior as separate work.

## Reproduction Assessment

Yes for the missing feature: the proposed per-model timeout key can be traced to a strict schema that rejects it, and current streaming timeout code is reset-on-chunk rather than total-duration based. A live Gemini infinite-planning loop was not run and is not needed to prove the config/runtime gap.

## Solution Assessment

Yes, the best fix is a narrow core config and runner change rather than a provider-specific workaround. First-token timeout and provider-level skip are related but separate failure modes and should remain out of scope for this issue.
