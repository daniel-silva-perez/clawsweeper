## Summary

Keep this issue open. Current main has adjacent timeout controls, but not the requested typed global/per-model non-resetting total-generation timeout: the proposed per-model key is still outside the strict `agents.defaults.models` schema, and the runner still enforces the existing run-level timeout plus a reset-on-chunk idle watchdog.

## What This Changes

Add global and per-model generation timeout configuration so long-running LLM generations can be aborted, logged, and optionally routed through fallback without relying on provider request timeouts.

## Best Possible Solution

Add a backwards-compatible `generationTimeout` contract at the agent default and per-model catalog levels, resolve the effective budget for each selected fallback candidate, enforce it with a non-resetting timer during active generation, classify/log timeout aborts for fallback handling, and keep first-token/provider-skip behavior as separate work.
