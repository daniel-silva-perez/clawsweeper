## Summary

Keep this issue open. Current main still lacks a typed global/per-model total generation timeout contract; the existing knobs cover run-level agent timeout, provider HTTP/request timeout, and an idle watchdog that resets on stream chunks. A maintainer review also already identified #44310 as a duplicate of this remaining work and kept first-token timeout work separate.

## What This Changes

Add global and per-model total text-generation timeout configuration that aborts active LLM generation after the configured budget, logs/classifies the abort as a timeout, and participates correctly in fallback routing.

## Best Possible Solution

Implement a typed global and per-model total text-generation timeout contract, resolve the effective budget per selected fallback candidate, enforce it with a non-resetting active-generation timer, classify/log the abort as a timeout for fallback handling, and update config types, zod/generated schema, docs/help, changelog, and focused regression tests. Keep first-token timeout separate unless maintainers intentionally expand scope.

## Risks / Open Questions

- The implementation must keep provider HTTP/request timeout, idle timeout, first-token timeout, and total generation timeout semantically distinct.

- The abort timer is near compaction and streaming cleanup logic, so tests should cover timer cleanup, compaction grace, and no orphaned runs.
- Config schema, generated docs metadata, public docs, and changelog can drift if the new key is only added to runtime code.
