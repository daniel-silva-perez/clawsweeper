## Summary

Keep this issue open. Current main still has adjacent timeout controls, but not the requested typed global/per-model non-resetting generation timeout: the per-model catalog schema remains strict, the embedded runner forwards one run-level timeout into each attempt, and the model idle watchdog resets after streamed chunks rather than enforcing a total generation budget.

## What This Changes

Request a global and per-model generation timeout that aborts long-running model output loops, logs the timeout, and can participate in fallback routing.

## Best Possible Solution

Add a backwards-compatible, documented generation-timeout contract at agent defaults and per-model catalog entries, resolve the effective non-resetting budget for each selected fallback candidate, enforce that budget during active generation with clear timeout logging/classification, and leave first-token/provider-skip behavior to separate work.
