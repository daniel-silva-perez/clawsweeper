## Summary

Keep this open because current main still lacks the requested typed global/per-model non-resetting generation timeout. The current implementation has adjacent run-level, provider request, and reset-on-chunk idle timeout behavior, but the proposed per-model catalog key is still outside the accepted config contract and no runtime code resolves an effective generation budget per selected model.

## What This Changes

The issue requests global and per-model generation timeout configuration that aborts runaway model generation and can participate in fallback/logging behavior.

## Best Possible Solution

Add a backwards-compatible generation-timeout contract at agent defaults and model catalog entries, resolve the effective non-resetting budget for each selected fallback candidate, enforce it during active generation with timeout classification/logging, and keep first-token/provider-skip behavior as separate work.
