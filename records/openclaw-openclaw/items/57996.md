## Summary

Keep this issue open. Current main still gives each agent its own QMD XDG cache and `qmd/index.sqlite`; existing mitigations reduce some startup/search pressure but do not provide a shared QMD cache, QMD WAL/checkpoint/vacuum controls, or QMD cache size/LRU limits.

## Best Possible Solution

Keep this open as a memory-core/QMD backend design issue. Maintainers should choose between a shared, agent-namespaced QMD home/index and bounded per-agent maintenance controls such as checkpoint/cleanup/vacuum and cache-size policy, then document the config and cover multi-agent startup/update behavior with focused tests.

## Risks / Open Questions

- The exact macOS 34.36GB write-volume report was not live-reproduced in this read-only review, so final thresholds and acceptance criteria still need production-style measurement.
- QMD owns the SQLite index behind its CLI; a robust OpenClaw fix may need upstream QMD support, use of QMD's SDK/maintenance APIs, or a carefully versioned OpenClaw-QMD compatibility contract.
