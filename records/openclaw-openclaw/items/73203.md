## Summary

Keep open. Current main still lacks the Feishu interactive-card text-tag and nested-array extraction this PR targets, but the proposed recursive array traversal should be bounded before merge because it walks externally supplied card JSON.

## What This Changes

The PR adds Feishu interactive-card extraction for direct `tag: "text"` elements and nested array layouts, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Land a bounded version of the narrow Feishu parser fix, or fold its text-tag and nested-array regression cases into the broader Feishu interactive-card parser if maintainers choose #56795. The final implementation should stay in the Feishu plugin and include regression coverage for quoted/replied card messages.

## Reproduction Assessment

Yes. A current-main fetched Feishu interactive message whose `body.content` contains `elements: [{ "tag": "text", "text": "..." }]` or row arrays under `elements` has a clear source-level reproduction path: the parser has no `text` branch and does not traverse nested arrays.

## Solution Assessment

Not yet. Direct `text` support is a narrow maintainable fix, but the recursive nested-array walk should be changed to a one-level flatten matching the stated scope or guarded with depth/node/output limits before merge.

## Review Findings
