## Summary

Keep this PR open. Current main still lacks the PR’s central Feishu interactive-card parsing behavior: direct `tag: "text"` elements are ignored, and nested array row/cell layouts are not traversed. The PR remains a focused implementation candidate with tests; related #41609 is still open and #45936 was closed unmerged, so there is no shipped replacement for these exact cases.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or adjust the focused Feishu parser fix so current main extracts direct `text` tag content and nested row/cell card layouts, with regression tests and bounded traversal/output handling appropriate for inbound Feishu card JSON.
