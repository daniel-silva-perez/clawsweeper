## Summary

Current main does not yet implement the PR’s central Feishu interactive-card parsing change. The parser still handles div, markdown/lark_md, and plain_text elements plus the newer post-format fallback, but it does not extract interactive-card elements shaped as { tag: "text", text: "..." } or nested array row layouts. The PR is active, narrow, and linked to the still-open Feishu card parsing report, so it should stay open for maintainer review/merge rather than be swept closed.

## Best Possible Solution

Keep this PR open. The best path is maintainer review of the focused Feishu parser/test changes, with a quick robustness check on bounded nested-array traversal, then merge if that review is satisfied so the still-open Feishu interactive-card parsing gap is fixed on main.
