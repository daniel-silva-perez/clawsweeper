## Summary

Keep this PR open. Current main still does not implement the PR’s central Feishu interactive-card parsing change: `tag: "text"` elements with string `text` are ignored, and nested array row layouts are skipped before the post-format fallback returns `[Interactive Card]`. The supplied PR context is active, narrow, and includes tests for the missing cases, so it should go through maintainer review rather than be closed as obsolete.

## Best Possible Solution

Keep this PR open. The best path is maintainer review of the focused Feishu parser and regression-test changes, with a bounded nested-array traversal check, then merge or adjust the PR so current main actually extracts text-tag and nested-row interactive-card content.
