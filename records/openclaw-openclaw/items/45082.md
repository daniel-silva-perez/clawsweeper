## Summary

Keep open. Current main still leaves the reported Mattermost thread-context paths unresolved: the message-tool send action only uses explicit reply targets, and Mattermost route persistence still records the last route only for direct sessions. The issue is also paired with open closing PRs #52120 and #52236, so it should remain open until those PRs land or are superseded.

## What This Changes

This issue asks for Mattermost proactive message-tool sends and cron/heartbeat wake replies from thread-scoped sessions to inherit the active Mattermost thread context instead of posting at the channel root or DM.

## Best Possible Solution

Land or supersede the paired Mattermost fixes so the plugin supplies thread-aware tool context for proactive sends and persists routable thread delivery context for thread-scoped group/channel sessions, with regression tests for both message-tool sends and cron/heartbeat wake delivery.
