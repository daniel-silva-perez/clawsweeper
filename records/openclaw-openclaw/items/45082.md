## Summary

Keep open. Current `main` still lacks the Mattermost-owned message-tool thread context and send-action fallback needed for implicit same-thread proactive sends, and the cron/heartbeat delivery metadata path still only persists last-route data for direct sessions. Open closing PRs already target both reported paths, so this should stay open for normal maintainer PR review rather than be closed or queued as a duplicate repair.

## What This Changes

Mattermost proactive `message` tool sends and cron/heartbeat wake replies should inherit the active thread context when `replyTo` is omitted.

## Best Possible Solution

Review, rebase, land, or replace the linked Mattermost PRs with a Mattermost-owned tool-context builder or guarded send-action inheritance for same-target thread sends, plus non-direct delivery-context persistence where needed and focused regressions for explicit `replyTo`, implicit same-thread sends, cross-channel suppression, `replyToMode`, and cron/heartbeat wake delivery.
