## Summary

Keep this PR open. Current main still scans fenced code blocks in `extractWikiLinks`, so the reported false-positive broken wikilink warning is not implemented on main. The PR is narrow and relevant, but the latest PR discussion identifies a remaining strip-order regression that should be resolved before merge.

## Best Possible Solution

Keep the PR open as a focused memory-wiki bug fix candidate. Maintainers should ask for or make a small revision in `extensions/memory-wiki/src/markdown.ts` that ignores fenced code without letting literal comment delimiters inside code remove later prose links, then keep the added `extractWikiLinks` tests and add a regression for that strip-order case before merge.
