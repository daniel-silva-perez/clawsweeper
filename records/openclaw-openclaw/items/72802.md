## Summary

Keep open. Current main still lacks a recovery hint for the combined clipboard/browser delivery failure, and this PR is a narrow, CVE-safe fix with matching regression coverage; the overlapping #73227 is not a better superseding path because its open review notes a failing test assertion.

## What This Changes

The PR adds a dashboard manual-auth fallback log message for failed token URL delivery, updates dashboard link tests for redaction and no-open behavior, and adds a changelog entry.

## Best Possible Solution

Land one narrow dashboard fix that preserves token redaction, gives headless users enough information to recover manually, and closes #72081; this PR is the cleaner candidate unless CI or maintainer review finds a new issue.

## Reproduction Assessment

Yes. Configure a literal gateway token, make clipboard copy return false, and take a headless/SSH/WSL browser-open failure path; current main logs the bare URL plus hint without telling the user how to recover token auth, and the PR adds a focused unit test for that path.

## Solution Assessment

Yes. The patch is the narrowest maintainable approach: it changes only the dashboard command/test/changelog surface, gates the hint on actual delivery failure, keeps the token value and `#token=` out of logs, and avoids broader auth or plugin changes.
