## Summary

Keep open. Current main still lacks the manual-auth fallback for failed dashboard token delivery, and this PR is a narrow candidate fix, but it should go through maintainer review because the latest public check-runs include failures.

## What This Changes

The PR adds a CVE-safe `openclaw dashboard` manual-auth hint when clipboard and browser delivery both fail, extends dashboard link regression tests, and adds a changelog entry.

## Best Possible Solution

Land one CVE-safe dashboard implementation that preserves token and `#token=` log redaction while giving WSL/headless users enough token-source and fragment-key guidance to authenticate manually when automatic delivery fails.

## Reproduction Assessment

Yes. Configure a literal gateway token, force clipboard copy to fail, and run a headless/SSH/WSL path where browser open is unavailable; current main logs only the bare URL plus SSH/no-open guidance, while the PR adds a unit test for that combined-failure path.

## Solution Assessment

Yes, the proposed direction is the narrowest maintainable fix: it stays in `src/commands/dashboard.ts`, fires only when a tokenized URL was intended and neither delivery channel succeeded, and preserves the existing redaction contract. The remaining work is validation/CI follow-up, not a different design.
