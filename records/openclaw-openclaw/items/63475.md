## Summary

Keep open: current main still allows concurrent Feishu per-account startup probes through the gateway path, and the paired serialization PRs are open/unmerged rather than shipped.

## What This Changes

The issue requests serializing or staggering Feishu startup bot identity probes across multiple configured accounts to avoid concurrent Feishu API/token requests and rotating startup timeouts.

## Best Possible Solution

Keep this issue open until PR #63481, fork PR BingqingLyu/openclaw#2190, or an equivalent Feishu-owned fix is reviewed and merged. The fix should throttle or serialize bot identity probing across concurrent gateway `startAccount` calls, add regression coverage for the actual gateway per-account path, and reconcile the approach with #67466.
