## Summary

Keep open. Current main does not implement the requested shared memory store or `--shared` memory CLI flow, but the PR branch is stale against the current memory-plugin architecture and still has blocking correctness/security issues in the proposed implementation.

## What This Changes

The PR adds a `_shared` memory index, `memorySearch.sharedPaths`, shared-result merging, reserved `_shared` agent validation, and `--shared` flags for memory status/index/search.

## Best Possible Solution

Implement the feature in the active `memory-core` plugin and host SDK surfaces, with explicit opt-in config, strict non-empty canonical shared path validation, deterministic default/agent override precedence, custom store-path support, docs, and focused tests for builtin and QMD interactions.

## Reproduction Assessment

Yes for review purposes: the current-main checks show the requested `sharedPaths`/`_shared`/`--shared` behavior is absent, and the PR diff has deterministic config cases for the review findings. I did not run live indexing because this review was read-only.

## Solution Assessment

No. The direction may be useful, but this PR is not currently the best way to solve it because it patches old core memory files instead of the active plugin-owned memory runtime and leaves concrete path/security bugs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject empty shared paths before allowing reads:** `src/config/zod-schema.agent-runtime.ts:601`
  - body: `sharedPaths` accepts `""`, and `mergeConfig` can preserve that as `sp.path === ""`; the later shared read check treats `absPath.startsWith("/")` as allowed on POSIX, so a blank config entry can expand `memory_get` access far beyond the intended shared directory. Trim and reject blank shared paths before resolving or matching them.
  - confidence: 0.91
- **[P2] Resolve agent extraPaths relative to the workspace:** `src/agents/memory-search.ts:262`
  - body: This maps per-agent `extraPaths` through `resolveUserPath`, so relative entries like `../team-docs` are resolved from the process cwd instead of the agent workspace. Current indexing resolves those paths with `normalizeExtraMemoryPaths(workspaceDir, ...)`; changing that silently drops or re-points existing memory files.
  - confidence: 0.88
- **[P2] Apply override weights when shared paths repeat:** `src/memory/manager.ts:428`
  - body: Merged `sharedPaths` keeps defaults before agent overrides, but the score weighting loop stops at the first matching path. If an agent overrides the weight for a default shared path, the default entry wins and the per-agent weighting silently has no effect.
  - confidence: 0.86
- **[P2] Report status from the actual shared store path:** `src/memory/manager.ts:881`
  - body: Shared status reads and displays `this.settings.sharedStorePath`, but the `_shared` manager persists to its resolved `settings.store.path`, which can be customized through `memorySearch.store.path`. With a custom store path, status can show the wrong DB and stale zero counts even after indexing succeeds.
  - confidence: 0.84
- **[P2] Avoid mkdir on file shared paths:** `src/memory/manager.ts:252`
  - body: The constructor calls `ensureDir(sp.path)` for every shared path, but shared paths are otherwise treated like `extraPaths` and may be individual markdown files. A not-yet-created `~/shared/policy.md` entry becomes a directory named `policy.md`, blocking the intended file from being created and indexed.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete path-validation issue in a new cross-agent read surface.

Concerns:

- **[medium] Blank shared path can broaden memory reads:** `src/config/zod-schema.agent-runtime.ts:601`
  - body: Because the schema accepts empty shared paths and the shared read check matches `absPath.startsWith(`${sp.path}${path.sep}`)`, an empty path becomes `/` on POSIX and can allow reads of arbitrary absolute `.md` paths rather than only explicitly configured shared directories.
  - confidence: 0.9
