## Summary

Keep open. Current main has heartbeat-specific `HEARTBEAT_OK` ack suppression and generic `NO_REPLY` handling, but it does not implement the broader job-defined no-op sentinel suppression requested here, and the explicitly paired PR #73785 is still open and unmerged in the provided GitHub context.

## What This Changes

The issue requests first-class cron/heartbeat/system-event no-op delivery semantics so successful silent runs do not leak sentinel strings or short no-op summaries to chat channels.

## Best Possible Solution

Review and land, revise, or replace PR #73785 with a narrow delivery-boundary fix that centralizes exact no-op sentinel handling for heartbeat/system-event outputs while preserving meaningful cron and exec summaries. A structured `noReply` or `suppressDelivery` result flag can remain a follow-up if exact sentinel matching is not enough for future jobs.

## Reproduction Assessment

Yes. The issue discussion gives a concrete heartbeat channel-delivery trace, and current-main inspection confirms the requested custom no-op sentinels are not recognized in the shared delivery helpers.

## Solution Assessment
