## Summary

Keep open. Current main still has no cron postcondition contract or scheduler-owned verifier for successful-but-incomplete isolated agent runs, and the existing maintainer-side review comment already routes this issue to follow-up rather than closure. Recent cron timeout cleanup improves stuck-run handling, but it does not add deterministic outcome verification, retry context injection, or postcondition state.

## What This Changes

Add optional cron postcondition checks that run deterministic verification after agent-turn cron executions and trigger retry or escalation when the check fails.

## Best Possible Solution

Add a reviewed, additive cron post-run verification design: typed config/state/protocol support, bounded deterministic verifier execution after isolated agent completion and before state application, durable run-log/UI/CLI/docs projection, retry context injection, and integration with existing retry/failure-alert delivery. If maintainers prefer plugin ownership, first add a supported fail-capable cron post-run hook that can request scheduler-owned retry or escalation.
