## Summary

Keep this issue open. Current main still has no cron postcondition contract, no deterministic post-run checker, no durable postcondition state, no retry prompt injection, and no plugin/API hook that can turn a clean isolated agent run into scheduler-owned failure, retry, or escalation. The existing maintainer-side Codex comment already kept this open, and the delta from that review to f2405c83 does not add the missing surface.

## Best Possible Solution

Keep this open for maintainer/product design. The best path is an additive cron postcondition contract with protocol/schema, CLI/UI, docs, deterministic bounded verifier execution after isolated agent completion and before applyJobResult(), durable verifier result fields, run-log projection, retry prompt injection, and integration with existing retry/failure-alert delivery. If this should be plugin-owned, first add a supported cron post-run hook/API that can fail the run and request scheduler-owned retry/escalation.
