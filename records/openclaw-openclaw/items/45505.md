## Summary

Keep open. Current main contains the narrow count-only reconciliation from the merged late-compaction fix, but it does not yet provide the broader authoritative reconciliation layer requested here: token/status snapshots, session lifecycle fields, and other compaction-derived state still converge through separate finalization paths.

## What This Changes

Generalize post-timeout compaction completion reconciliation so durable session state, status token snapshots, and downstream compaction-derived consumers converge after timeout-late-success outcomes.

## Best Possible Solution

Define one idempotent compaction-success reconciliation boundary that owns durable compaction-derived session updates, then deliberately route count, token freshness, rotated transcript identity, and dependent memory/auth/status state through that boundary with late-success and duplicate-signal regression coverage.

## Reproduction Assessment

Yes, as a focused code/test reproduction: simulate a successful compaction_end after run finalization and assert that current main only repairs compactionCount while token/status-derived fields do not flow through the same reconciliation path. I did not run that test because this review was read-only.

## Solution Assessment

No, the current narrow count-only repair is not the full solution requested here. The maintainable direction is to promote the existing hook into an authoritative, monotonic reconciliation helper/event rather than spread more one-off writes across finalizers.
