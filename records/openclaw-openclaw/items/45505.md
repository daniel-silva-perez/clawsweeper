## Summary

Keep open. Current main still implements only a narrow late-success compaction count reconciliation plus adjacent finalizer-time token/session updates; it does not yet provide the generalized authoritative reconciliation boundary across session, status, memory, and auth-derived state requested here.

## What This Changes

The issue asks for a generalized, idempotent post-timeout compaction completion reconciliation model across durable session state and user-visible session readers.

## Best Possible Solution

Define one monotonic compaction-success reconciliation boundary that owns durable compaction-derived session updates, then route count, post-compaction token freshness, rotated transcript identity, and dependent status/memory/auth state through it with late-success and duplicate-signal coverage.

## Reproduction Assessment

Yes. A focused test can simulate a successful compaction_end after finalization has already moved on and assert that current main repairs only compactionCount while richer token/session/status-derived state does not converge through the late path.

## Solution Assessment

No. The current count-only repair was the right narrow predecessor, but the maintainable full solution is a single idempotent reconciliation helper or event rather than additional scattered finalizer writes.
