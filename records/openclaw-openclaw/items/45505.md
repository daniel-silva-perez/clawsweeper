## Summary

Keep this issue open. Current main has the narrow #45493 late-success repair plus later partial token-snapshot persistence, but the authoritative post-timeout completion reconciliation model requested here is still not implemented across session state, status/UI snapshots, memory flush gating, auth-profile override state, and lifecycle consumers.

## Best Possible Solution

Keep the issue open as a design/implementation follow-up. The best path is to promote successful compaction completion into an authoritative, idempotent reconciliation boundary that deliberately converges compactionCount, post-compaction token totals/freshness, session rotation metadata, and dependent session fields, with regression coverage for timeout-late success and duplicate completion signals.
