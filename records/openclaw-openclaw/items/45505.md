## Summary

Keep this issue open. Current main has the narrow late-success repair from the related merged PR, but it is still a compaction-count-only reconciliation path plus separate run-finalization accounting; the broader requested state convergence across token snapshots, UI/status/trace metadata, memory flush/auth-profile dependent fields, and timeout-late completion semantics is not implemented as a single authoritative boundary.

## Best Possible Solution

Keep the issue open and treat it as a design/implementation follow-up. The best fix is to promote the existing successful-compaction hook into an authoritative, idempotent reconciliation boundary that can converge compactionCount, post-compaction token snapshots/freshness, rotated session identity, status/UI/trace metadata, and dependent session fields deliberately, with regression tests for timeout-late success and duplicate completion signals.
