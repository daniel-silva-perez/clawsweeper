## Summary

Keep this PR open. Current main still does not surface SQLite FTS5 availability through `openclaw doctor`: the memory manager records FTS state and `openclaw memory status` can print it, but `doctor.memory.status`, `probeGatewayMemoryStatus`, and `noteMemorySearchHealth` still only carry embedding readiness. The PR remains a plausible implementation candidate for the open #62328 doctor-warning portion, with no security-sensitive diff concerns from the provided file list.

## Best Possible Solution

Keep the PR open as the implementation candidate for #62328. Rebase or apply the same narrow change on current main: add `fts` to the doctor memory payload, carry it through `GatewayMemoryProbe`/`probeGatewayMemoryStatus`, emit a targeted `noteMemorySearchHealth` warning when FTS is enabled but unavailable, and keep the targeted gateway and doctor tests.
