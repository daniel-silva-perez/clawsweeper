## Summary

Keep this PR open. Current main still does not implement catalog lookup, catalog-miss warning, or rejection in `openclaw models set`, so the PR is not obsolete or implemented. The latest PR revision is a plausible implementation candidate, but it now warns and persists instead of rejecting invalid IDs, while the PR body and linked bug still describe hard rejection.

## Best Possible Solution

Keep this PR open and ask for a focused update: choose the intended product behavior for catalog misses (reject vs warn), update the PR body to match it, preserve configured/custom/stale-catalog allowlist semantics, keep the established config mutation path unless direct writes are justified, and add regression tests for catalog hit, catalog miss, catalog-unavailable fallback, configured custom-model acceptance, and existing routing-config preservation.
