## Summary

Keep open. Current main still archives reset transcripts and has better daily-reset freshness semantics, but it does not provide the requested bulk restore/re-index utility, archived-session selector for sessions_history/chat.history, or first-class UI/logical-history recovery path. The supplied discussion and related open items show this is an active recovery gap, not obsolete cleanup.

## Best Possible Solution

Keep this open until maintainers choose the product path: add a conservative first-party restore/re-index command or doctor repair for .jsonl.reset.* archives, or explicitly supersede it once archived reset sessions are first-class in chat.history, sessions_history, and the UI. Preserve stable session identity plus original/restored path and reset timestamp metadata for downstream history viewers.
