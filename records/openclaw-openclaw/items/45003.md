## Summary

Keep this issue open. Current main has partial session-reset hardening, but it does not provide the requested first-party restore/re-index utility, and the main `sessions_history`/`chat.history` path still reads the active indexed transcript only. The provided discussion also points to still-open archive/history follow-up work and an open implementation PR candidate, so this should not be swept closed yet.

## Best Possible Solution

Keep this issue open and either implement a supported `openclaw sessions` or `openclaw doctor` recovery path with dry-run, backup, rename/re-index, and retention warnings, or explicitly supersede it only after the archive-history PRs land and a maintainer decides that restore tooling is no longer needed.
