## Summary

Keep open. Current main still treats restart-aborted subagent sessions as recoverable solely from active run membership plus `abortedLastRun`, has no recovery max-age config, and still has docs/tests that preserve recovery for stale aborted runs. The discussion now includes two concrete incident reports and a distinct related follow-up, while read-only GitHub API search found no open PR already paired with this issue.

## What This Changes

The issue asks OpenClaw to bound gateway restart recovery for interrupted CLI/subagent tasks so stale prompts are marked terminal instead of automatically re-executed after a long outage.

## Best Possible Solution

Add a bounded restart-aborted recovery policy in the core subagent/task recovery path: keep fresh interrupted work recoverable, skip and mark stale interrupted run/task records terminal without synthetic redispatch, expose a documented config default, and update docs plus regression coverage for both stale-skip and fresh-resume behavior.

## Reproduction Assessment

Yes. A high-confidence reproduction exists by persisting an active subagent run with an old `startedAt` plus a child session marked `abortedLastRun: true`, then restarting the registry/gateway and observing orphan recovery schedule a synthetic resume instead of marking it lost; the current persistence test already asserts this stale-recovery behavior for a three-hour-old run.

## Solution Assessment

Yes. An age gate at the orphan-recovery decision point, with task/run terminalization and config/docs/tests, is the narrowest maintainable fix; filtering only task restore or only documentation would leave the redispatch path intact.

## Repair Work Prompt

Treat #69778 as the canonical age-bound recovery bug. Implement bounded recovery for gateway-restart-aborted CLI/subagent work so old prompts are not automatically redispatched after a long outage. Add a configurable max age under the agent/task config surface, default it conservatively around 120 minutes unless existing config conventions point to a better name/value, and apply it where restart-aborted subagent orphan recovery decides whether to synthesize a resume. Fresh interrupted runs should still recover; stale interrupted runs should become terminal/lost without calling `callGateway` or sending the synthetic resume prompt. Keep task registry, subagent run registry, cancellation, announcements, and maintenance coherent, and log a concise skipped-stale recovery summary if the local logging surface supports it. Update `docs/tools/subagents.md`, `docs/automation/tasks.md`, and `docs/gateway/config-tools.md`; update generated config docs/schema only through the normal repo generators if required. Add regression coverage for stale-skip and fresh-resume behavior. Add a user-facing changelog Fixes entry crediting `@FelixRen-FY` if runtime bug fixes currently require a changelog entry. Do not implement durable partial generation resume, do not change unrelated task retention cleanup, do not weaken recent restart recovery, and do not try to solve #74864 in this PR beyond avoiding regressions.
