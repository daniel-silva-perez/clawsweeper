## Summary

Keep open. Current main still intentionally recovers restart-aborted subagent sessions without an age bound, and a regression test explicitly preserves a 3-hour-old aborted restored run for recovery. The linked #66909 was a documentation clarification, not a fix for stale auto-resume.

## Best Possible Solution

Implement a bounded recovery policy for restart-aborted subagent/task recovery: keep fresh interrupted work recoverable, but mark or finalize stale interrupted work as lost/failed without redispatching it. The age check should live where recovery is scheduled/executed, be configurable through the agent/task config schema with docs, and have regression coverage proving old aborted runs are skipped while recent interrupted runs still resume.
