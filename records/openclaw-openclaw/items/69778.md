## Summary

Keep open. Current main still resumes restart-aborted subagent sessions without an age bound: the recovery loop only checks active run membership and `abortedLastRun`, the task registry restores all persisted tasks before maintenance, the config schema has no task-recovery max-age knob, and regression/docs still preserve stale aborted recovery behavior. The related #66909 is a docs clarification, not a bounded recovery implementation.

## Best Possible Solution

Implement a bounded recovery policy for restart-aborted subagent/task recovery. Add a configurable max-age under the agent/task config surface, default it conservatively, keep fresh interrupted work recoverable, and mark stale interrupted work terminal without synthetic redispatch. Update docs and regression tests for stale-skip and fresh-resume behavior.

## Clownfish Work Prompt

Fix the stale restart-aborted subagent/task recovery bug described by this issue. Current main resumes any active restored subagent run whose child session has `abortedLastRun: true`, regardless of age. Add a bounded recovery policy so recent interrupted work remains recoverable, but stale interrupted work is finalized/marked lost without redispatching the synthetic resume prompt.

Expected boundary:
- Add a configurable max age for restart-aborted recovery under the agent/task config surface, using the requested shape if it fits existing config conventions: `agents.defaults.taskRecovery.maxAgeMinutes` or a clearly documented equivalent. Default should be conservative, around 120 minutes.
- Apply the age check before `resumeOrphanedSession` sends the synthetic prompt in `src/agents/subagent-orphan-recovery.ts`.
- Ensure stale restored subagent runs/tasks become terminal (`lost` or equivalent failed/lost finalization) and are not left active or repeatedly retried.
- Keep fresh restart-aborted recovery working.
- Update docs for `docs/tools/subagents.md` and `docs/automation/tasks.md`; mention the config in the relevant config docs if a new public key is added.
- Add/update regression tests for stale-skip and fresh-resume behavior. Do not change unrelated recovery semantics, dependency pins, release metadata, or broad task registry policy.

Related refs: this issue and #66909 for the prior docs-only clarification.
