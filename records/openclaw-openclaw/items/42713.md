## Summary

Keep open. Current main still lacks request-origin metadata in the failover wrapper path, and PR #73506 is an open, unmerged implementation candidate with closing syntax for this issue. Per-attempt provider/model data is present, but sessionId/sessionKey/lane cannot be carried into the final wrapper error or model_fallback_decision logs yet.

## Best Possible Solution

Keep this issue open. Review PR #73506 and either land, revise, or close it; the eventual fix should thread normalized request-origin metadata such as sessionId/sessionKey or lane through FailoverError construction/description/coercion, runWithModelFallback/FallbackSummaryError or equivalent wrapper logging, and model_fallback_decision logs, with focused regression tests for exhausted multi-provider fallback attribution and JSONL session_id promotion.
