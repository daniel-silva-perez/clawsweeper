## Summary

Keep open. Current main still does not fully solve the reported failover wrapper observability gap, and PR #73506 is an open, unmerged implementation candidate with closing syntax for this issue. The current code records provider/model on individual fallback attempts, but the exhausted wrapper error and model fallback decision logging still have no request-origin field such as sessionId, sessionKey, session_id, or lane to promote into structured gateway error logs.

## Best Possible Solution

Keep this issue open. Review PR #73506 and either land, revise, or close it; the eventual fix should thread normalized request-origin metadata such as sessionId/sessionKey or lane through runWithModelFallback, FailoverError description/coercion, FallbackSummaryError or equivalent wrapper logging, and model_fallback_decision logs, with focused tests for exhausted multi-provider fallback attribution and JSONL session_id promotion.
