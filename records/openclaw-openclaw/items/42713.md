## Summary

Keep open. Current main still lacks request-origin session/lane metadata in the failover wrapper and model fallback decision path, and PR #73506 is an open, unmerged implementation candidate that is explicitly paired with this issue.

## What This Changes

The issue requests that exhausted failover wrapper errors include attempted provider attribution plus normalized session or lane metadata in structured error logs.

## Best Possible Solution

Review PR #73506 and either land a complete version or request revisions. The eventual fix should thread populated request-origin metadata through FailoverError construction/description/coercion, runWithModelFallback or its caller context, FallbackSummaryError and model_fallback_decision logging, while preserving existing structured logger redaction and adding focused regression tests for exhausted multi-provider fallback attribution and session_id normalization.

Yes.
