## Summary

Keep open. Current main still does not fully solve the reported structured logging gap for exhausted failover wrapper errors, and PR #73506 is an open, unmerged implementation candidate that explicitly closes this issue. The existing code carries provider/model details for individual attempts, but FailoverError, FallbackSummaryError, runWithModelFallback, and model_fallback_decision logs still lack normalized request-origin metadata such as sessionId, sessionKey, session_id, or lane.

## Best Possible Solution

Keep this issue open. Review PR #73506 and either land, revise, or close it; the eventual fix should thread normalized request-origin metadata such as sessionId/sessionKey or lane through runWithModelFallback, FailoverError description/coercion, FallbackSummaryError or equivalent wrapper logging, and model_fallback_decision logs, with focused tests for exhausted multi-provider fallback attribution and JSONL session_id promotion.
