## What This Changes

The PR adds a default-off `gateway.http.endpoints.responses.exposeBuiltInToolCalls` flag, captures non-client tool-start events for `/v1/responses`, exposes them as JSON/SSE `function_call` output items, and updates config schema, docs, tests, generated baseline hash, and changelog.

## Best Possible Solution

Land a maintainer-approved version that keeps the feature opt-in, preserves existing client-tool delegation semantics, documents the audit-only contract, and has CI proof for JSON/SSE ordering plus config-schema drift.

## Reproduction Assessment

Yes for the reported feature gap: current-main code inspection shows `/v1/responses` only emits assistant items and caller-provided client `function_call` items, while built-in tool-start events already exist on the agent event bus. This is a feature/contract gap rather than a crash reproduction.

## Solution Assessment

Mostly yes: using a default-off Gateway config flag over existing sanitized `stream: "tool"` start events is a narrow, backward-compatible way to provide audit visibility. The remaining decision is endpoint contract/product approval and mergeability, not a different technical shape identified by this review.
