## Summary

Keep open. The Feishu topic-group bug appears real, but this branch is not merge-ready because it classifies every `chat_type: "p2p"` plus `oc_` chat ID as a group even though current main still preserves `p2p` `oc_` direct-chat behavior in routing and card-action paths.

## What This Changes

The PR adds a Feishu chat classification helper, uses it in inbound routing and mention-forward detection, adds helper tests, and rewrites Feishu DM fixtures from `oc_` IDs to `p2p_` IDs.

## Best Possible Solution

Land a Feishu-owned classifier that distinguishes real OpenChat/topic-group webhook events from true DMs using verified event fields or chat metadata, then apply it consistently to inbound routing and mention-forward detection while leaving outbound/session-key consistency to the #58537 follow-up.
