## Summary

Keep this PR open because the Feishu topic-group routing bug remains real on current main, but the current patch is not merge-ready: its `p2p` + `oc_` prefix rule is broader than the current DM/security contract and can route real direct chats through group policy paths.

## What This Changes

The PR adds a Feishu chat classification helper, uses it in inbound routing and mention-forward detection, adds helper tests, and changes Feishu DM fixtures from `oc_` IDs to `p2p_` IDs.

## Best Possible Solution

Land a Feishu-owned classification path that distinguishes actual OpenChat/topic-group webhook events from true DMs using a verified discriminator, then apply it consistently to inbound routing and mention-forward detection while keeping outbound session-key behavior aligned with #58537.
