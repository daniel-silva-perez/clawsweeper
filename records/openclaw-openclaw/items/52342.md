## Summary

Keep open. The linked Feishu topic-group routing bug is real and still needs a fix, but this PR is not merge-ready against current main because its prefix-only classifier would regress existing `topic_group` handling and `p2p` `oc_` direct/card-action paths.

## What This Changes

The PR adds a Feishu chat classifier used by inbound bot routing and mention-forward detection, adds classifier tests, and changes several Feishu DM fixtures from `oc_` IDs to `p2p_` IDs.

## Best Possible Solution

Revise the Feishu-owned classifier so it preserves existing `group` and `topic_group` behavior, distinguishes actual `p2p` topic-group webhook events using a verified signal such as thread/root fields or Feishu chat metadata, applies the final decision consistently in inbound routing and mention-forward detection, and coordinates session-key consistency with #58537.
