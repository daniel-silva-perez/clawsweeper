## Summary

Keep open. The linked Feishu topic-group routing bug is still valid on current main, but this patch is not merge-ready because it replaces the current chat-type contract with a helper that both over-classifies `p2p` `oc_` events as groups and under-classifies existing `topic_group` events as direct chats.

## What This Changes

The PR adds a Feishu group-chat classifier based on `chat_type` or `p2p` plus an `oc_` chat ID, uses it in inbound bot routing and mention-forward detection, adds classifier tests, and rewrites Feishu DM fixtures from `oc_` IDs to `p2p_` IDs.

## Best Possible Solution

Keep this PR open for revision: preserve the existing `group`/`topic_group` group contract, distinguish `p2p` topic-group events with a verified Feishu signal such as thread/root fields or chat metadata, keep `p2p` `oc_` DM regression coverage, apply the final helper consistently in bot routing and mention-forward detection, and coordinate outbound session consistency with #58537.
