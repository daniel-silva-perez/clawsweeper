## Summary

Keep open. Current main still classifies Feishu inbound group/direct routing from `chat_type` only, so the PR's `chat_type: "p2p"` plus `oc_...` OpenChat/topic-group case is not implemented. The PR remains relevant to #52238, but the prefix-only promotion needs maintainer review because existing Feishu code and prior fixes treat bare `oc_` as ambiguous for access control and session routing.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is to adapt it to current main's `isFeishuGroupChatType`/topic-session code and prove that `chat_type: "p2p"` plus `oc_...` is a safe inbound-only OpenChat/topic-group signal, or use stronger evidence such as `thread_id`, `root_id`, or verified Feishu chat metadata. The fix should cover both bot routing and mention-forward classification while preserving true p2p DM/card-action behavior; outbound session mismatch can remain tracked by #58537 if not solved here.
