## Summary

Keep open for maintainer review. Current main still does not route Feishu `chat_type: "p2p"` plus `oc_` topic-group events as groups, but this PR’s proposed discriminator is overbroad against existing shipped Feishu behavior and could regress `oc_` direct-chat handling.

## What This Changes

The PR adds a Feishu chat classification helper, uses it in inbound routing and mention-forward detection, adds helper tests, and changes Feishu DM fixtures from `oc_` IDs to `p2p_` IDs.

## Best Possible Solution

Land a Feishu-owned classification path that distinguishes actual OpenChat/topic-group webhook events from true DMs using a verified discriminator, then apply it consistently to inbound routing and mention-forward detection while keeping outbound session-key consistency aligned with the #58537 follow-up.

- **[P1] Preserve p2p oc_ direct chats:** `extensions/feishu/src/chat-type.ts:15`
  - body: The new helper treats every `chat_type === "p2p"` event whose `chat_id` starts with `oc_` as a group. Please use a verified topic-group discriminator instead of classifying all p2p `oc_` IDs as groups.
