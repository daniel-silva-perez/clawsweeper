## Summary

Keep this PR open. The linked Feishu topic-group bug is real and current main still routes inbound `p2p`/`oc_` topic-group events as direct messages, but this branch is not merge-ready because it classifies every `p2p` `oc_` chat as group traffic and drops existing direct-message fixture coverage instead of preserving the current DM policy boundary.

## What This Changes

The PR adds a Feishu chat classifier, applies it to inbound bot routing and mention-forward detection, adds classifier tests, and changes several Feishu DM fixtures from `oc_`-prefixed IDs to `p2p_` IDs.

## Best Possible Solution

Revise the Feishu-owned classification path so actual topic-group webhook events are routed as group traffic using a verified signal such as Feishu chat metadata or thread/root fields, while preserving current `p2p` direct-message behavior and coordinating outbound session-key consistency with #58537.

## Reproduction Assessment

Yes. The linked bug has concrete steps in #52238: add the bot to a Feishu topic group, send a message, and observe an `oc_...` chat ID reported as `chat_type: "p2p"` being dispatched to the main/direct session; current main source confirms that event remains direct.

## Solution Assessment

No. The PR changes the right inbound boundary, but the `p2p` plus `oc_` prefix rule is too broad for current shipped DM and card-action policy, so a verified topic-group discriminator is the safer maintainable fix.
