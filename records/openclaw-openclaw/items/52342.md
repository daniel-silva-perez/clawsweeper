## Summary

Keep open. The reported Feishu topic-group routing bug still appears real on current main, but this PR is not merge-ready because its bare `p2p` + `oc_` prefix rule conflicts with current direct-chat and card-action paths that intentionally preserve `p2p` `oc_` conversations as DMs.

## What This Changes

The branch adds a Feishu group-chat classifier, applies it to inbound bot routing and mention-forward detection, adds classifier tests, and rewrites Feishu DM fixtures from `oc_` IDs to `p2p_` IDs.

## Best Possible Solution

Land a Feishu-owned classification path that distinguishes actual OpenChat/topic-group webhook events from true DMs using verified event fields or chat metadata, apply it consistently to inbound routing and mention-forward detection, keep `p2p` `oc_` DM regression coverage, and coordinate outbound session-key consistency with #58537.
