## Summary

Keep open. Current main still does not implement the PR's `chat_type: "p2p"` plus `oc_...` inbound OpenChat/topic-group classification, so the underlying #52238 behavior remains plausible. The branch also needs maintainer review before merge because current main now has `topic_group`/thread-session handling and prior Feishu fixes deliberately avoid treating bare `oc_` as enough to prove group-vs-DM routing.

## What This Changes

The PR adds a Feishu chat-type helper that treats `chat_type: "p2p"` with `oc_` chat IDs as group chats, wires it into bot routing and mention-forward detection, and updates Feishu DM test fixtures plus a new helper test.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is to adapt the fix to current main's `isFeishuGroupChatType` and topic-session code, prove whether `chat_type: "p2p"` plus `oc_...` is a safe inbound-only OpenChat/topic-group signal, and cover both bot routing and mention-forward classification while preserving true p2p DM, pairing, and card-action behavior. The broader outbound/session mismatch can remain tracked by #58537 unless the maintainer chooses to handle it here.
