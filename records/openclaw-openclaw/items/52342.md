## Summary

Keep open. Current main still does not implement the PR's intended `chat_type: "p2p"` plus `oc_` OpenChat/topic-group classification, and the linked Feishu bug remains open. The PR is a plausible contributor fix, but it needs maintainer review because current tests still use `oc_`-prefixed `p2p` fixtures as DMs and a related open report tracks outbound `oc_` session mismatch that this inbound-only diff does not cover.

## What This Changes

The PR branch adds a Feishu chat-type helper, uses it in inbound message routing and mention-forward detection, and updates Feishu tests so DM fixtures use `p2p_` IDs instead of `oc_` IDs.

## Best Possible Solution

Implement one Feishu-owned group-classification path that handles actual OpenChat/topic-group webhook events without regressing true DMs, use it consistently in inbound routing and mention-forward logic, and keep the outbound session-key consistency either in the same reviewed plan or in the linked narrower follow-up.
