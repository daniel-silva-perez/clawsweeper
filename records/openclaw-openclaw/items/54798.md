## Summary

Keep open. Current main only has per-cron-job overlap prevention and configurable cron parallelism; it still lacks the requested shared atomic counter, `openclaw sessions kv incr` command/RPC, named `sessions_spawn` mutex, or agent-facing file-lock primitive for concurrent cron sub-agents sharing daily counters. The related skip-if-running work helps one overlap class but does not solve cross-agent shared-state coordination.

## Best Possible Solution

Keep this open for a focused coordination primitive decision. The strongest core path is a small generic atomic state API, such as a workspace/session-scoped `kv incr` tool/RPC/CLI backed by existing lock-safe store helpers, with concurrency tests covering two cron-spawned sub-agents incrementing/resetting the same daily counter. If maintainers prefer keeping this outside core, document an explicit ClawHub plugin path using `api.registerTool` plus `openclaw/plugin-sdk/file-lock` or session-store helpers, but that should be a deliberate product call.
