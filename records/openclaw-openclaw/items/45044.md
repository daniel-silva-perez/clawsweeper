## Summary

Keep open: current main still has the reported mid-turn transcript-loss path, and the PR addresses the core guard correctly, but the branch should not merge as-is because the archival path can recreate the transcript with non-private permissions and the new regression test hardcodes `/tmp`.

## What This Changes

The PR narrows session initialization resets to header-only session files, archives before clearing, adds regression coverage for user-message-only transcripts, and adds a shell wrapper for the targeted test.

## Best Possible Solution

Amend the PR so header-only reset archives safely while preserving private transcript permissions, keep the `hasAnyMessage` behavior for mid-turn restarts, and make the regression fixture portable with `os.tmpdir()` plus `path.join()` before running the targeted test and changed gate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve transcript file mode after archiving:** `src/agents/pi-embedded-runner/session-manager-init.ts:56`
  - body: The reset path now renames the existing transcript and then creates a replacement with `fs.writeFile(..., "utf-8")`. Header-only session files are created with private `0o600` permissions elsewhere, but this replacement can be created with the process default mode, and the next assistant flush will append transcript content to that less-private file. Recreate the file with `mode: 0o600` or preserve the original mode after archiving.
  - confidence: 0.87
- **[P2] Build temp fixtures from os.tmpdir():** `src/agents/pi-embedded-runner/session-manager-init.test.ts:10-11`
  - body: The new test calls `fs.mkdtemp("/tmp/session-init-test-")` and builds `sessionFile` with string concatenation. That parent directory is not portable, so the regression suite can fail before testing the session-preservation behavior on hosts without `/tmp`. Use `os.tmpdir()` with `path.join()` for the prefix and session path.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: Security review found a concrete transcript-permission regression in the new archive-before-clear path.
