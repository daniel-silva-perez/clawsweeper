## Summary

Keep open. Current main still has the broad reset condition that can clear a message-bearing incomplete transcript, so the PR addresses a real data-loss path, but the proposed patch should not merge until the new regression test stops hard-coding `/tmp` for its fixture directory.

## What This Changes

The PR narrows session initialization resets to header-only session files, archives before clearing, adds session-state regression tests, and adds a shell wrapper for the targeted test.

## Best Possible Solution

Fix the regression test to create its fixture under `os.tmpdir()` with `path.join()`, then land this PR or a maintainer-owned successor after targeted test proof and the changed gate. The desired end state is that only true header-only pre-created files reset, while any transcript that already contains messages survives a gateway restart.
