## Summary

Keep open. Current main still has the mid-turn transcript reset problem, and the PR fixes the central predicate, but the proposed archive/reset path introduces a transcript file-mode regression and the new test/helper surfaces need small portability fixes before merge.

## What This Changes

The PR narrows embedded session initialization resets to header-only transcripts, adds best-effort archiving before reset, adds regression tests for mid-turn gateway restart transcripts, and adds a shell verification wrapper.

## Best Possible Solution

Amend the PR so the hasAnyMessage guard remains, the header-only archive/reset path recreates or preserves the transcript at 0o600, fixtures use os.tmpdir() with path.join(), and any kept verification helper uses the repository test entrypoint.

## Reproduction Assessment

Yes. A transcript state of header plus user message and no assistant maps directly to current main's reset branch, and the pinned SessionManager behavior confirms why this state can exist during a restart before assistant flush. I did not run tests because this was a read-only review.

## Solution Assessment

No as submitted. The predicate change is the right narrow fix, but adding archiveFileOnDisk before fs.writeFile changes the file creation path and needs an explicit private mode before this is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve transcript file mode after archiving:** `src/agents/pi-embedded-runner/session-manager-init.ts:56`
  - body: The new archive step renames the existing private transcript before this write runs. Because the original path no longer exists, fs.writeFile(..., "utf-8") creates a replacement using default process permissions instead of the 0o600 mode used by transcript creation paths, so later transcript content can land in a less-private file. Recreate the file with mode 0o600 or restore the archived file's mode.
  - confidence: 0.87
- **[P2] Use the OS temp directory for fixtures:** `src/agents/pi-embedded-runner/session-manager-init.test.ts:10-11`
  - body: The new test hardcodes /tmp as the mkdtemp parent and then concatenates paths. Windows CI may not have that parent, so the suite can fail before it exercises the session-preservation behavior. Use os.tmpdir() and path.join() for the prefix and session file path.
  - confidence: 0.91
- **[P3] Use the repo test wrapper in the verification script:** `scripts/verify-session-init-fix.sh:13`
  - body: The added helper invokes pnpm vitest run directly, bypassing the repo's pnpm test wrapper that applies the repository's Vitest scheduling and process handling. Use pnpm test with the target file so the script matches the documented validation path.
  - confidence: 0.76
