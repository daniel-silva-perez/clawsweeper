## Summary

Keep this PR open. Current main still has the mid-turn transcript reset behavior, and the PR fixes the central predicate, but the branch should not merge as-is because it introduces a transcript file-mode regression in the new archive-before-clear path and adds a nonportable test fixture.

## What This Changes

The PR narrows session initialization resets to header-only transcripts, archives before clearing, adds regression coverage for user-message-only mid-turn transcripts, and adds a shell wrapper for the targeted test.

## Best Possible Solution

Amend the existing PR so the `hasAnyMessage` behavior stays, the archive-before-clear path recreates or preserves transcript files with private `0o600` permissions, and the new regression fixture uses `os.tmpdir()` with `path.join()` before running the targeted test and changed gate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve transcript file mode after archiving:** `src/agents/pi-embedded-runner/session-manager-init.ts:56`
  - body: The archive step renames the existing private transcript and then creates a replacement with `fs.writeFile(..., "utf-8")`. Existing session-header creation paths request `0o600`, but this replacement can be created with the process default permissions, and later transcript appends can land in that less-private file. Recreate it with `mode: 0o600` or restore the archived file's mode.
  - confidence: 0.87
- **[P2] Build temp fixtures from the OS temp directory:** `src/agents/pi-embedded-runner/session-manager-init.test.ts:10-11`
  - body: The new test hardcodes `/tmp` as the `mkdtemp` parent and concatenates paths. On Windows CI that parent may not exist, so the regression suite can fail before exercising the session-preservation behavior. Use `os.tmpdir()` with `path.join()` for the prefix and session file.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete transcript-permission regression in the new archive-before-clear path.

Concerns:

- **[medium] Transcript reset recreates file without private mode:** `src/agents/pi-embedded-runner/session-manager-init.ts:56`
  - body: After `archiveFileOnDisk()` renames the original session file, `fs.writeFile(..., "utf-8")` creates a new file using default permissions. Future transcript content can then be appended to a less-private file than the existing `0o600` session-header creation paths intend.
  - confidence: 0.87
