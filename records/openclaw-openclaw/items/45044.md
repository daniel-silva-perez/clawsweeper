## Summary

Keep open: current main still has the reported mid-turn data-loss path, and the PR fixes the central predicate, but the submitted patch introduces a transcript privacy regression in the reset/archive path and still has test/script portability issues that should be amended before merge.

## What This Changes

The branch narrows embedded session initialization resets to header-only transcripts, archives before header-only reset, adds session-manager-init unit tests, and adds a shell verification wrapper.

## Best Possible Solution

Amend this PR so the hasAnyMessage guard remains, the header-only archive/reset path recreates or preserves the transcript at 0o600, fixtures use os.tmpdir() with path.join(), and any kept verification helper runs through the repository test entrypoint or is removed.

## Reproduction Assessment

Yes. A session file containing a header plus a user message and no assistant message maps directly to current main's reset branch after SessionManager.open loads the file; the pinned dependency source explains why flushed state makes that state plausible during restart.

## Solution Assessment

No as submitted. The predicate change is the right narrow fix for the data-loss bug, but adding archiveFileOnDisk before fs.writeFile needs explicit private file mode or mode restoration before this is the safest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve transcript file mode after archiving:** `src/agents/pi-embedded-runner/session-manager-init.ts:56`
  - body: The new archive step renames the existing private transcript before this write runs. Because the original path no longer exists, fs.writeFile(..., "utf-8") creates a replacement using default process permissions instead of the 0o600 mode used by transcript creation paths, so later transcript content can land in a less-private file. Recreate the file with mode 0o600 or restore the archived file's mode.
  - confidence: 0.88
- **[P2] Use the OS temp directory for fixtures:** `src/agents/pi-embedded-runner/session-manager-init.test.ts:10-11`
  - body: The new test hardcodes /tmp as the mkdtemp parent and then concatenates paths. Windows-style environments may not have that parent, so the suite can fail before it exercises the session-preservation behavior. Use os.tmpdir() and path.join() for the prefix and session file path.
  - confidence: 0.86
- **[P3] Use the repo test wrapper in the verification script:** `scripts/verify-session-init-fix.sh:13`
  - body: The added helper invokes pnpm vitest run directly, bypassing the repo's pnpm test wrapper that applies OpenClaw's Vitest routing and process handling. Use pnpm test with the target file so the script matches the documented validation path.
  - confidence: 0.78
