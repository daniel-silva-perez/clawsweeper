## Summary

Keep open: current main still has the reset path, so this PR is still useful, but the new regression test hardcodes `/tmp` and can fail before it exercises the bug. The PR needs a small portability cleanup before merge.

## What This Changes

The PR narrows session initialization resets to header-only session files, archives before clearing, adds regression coverage for key session states, and adds a shell wrapper for the new test.

## Best Possible Solution

Update the regression test to create its fixture under `os.tmpdir()` with `path.join()`, then land this fix or a maintainer-owned successor after targeted test proof and the changed gate. The intended end state is to reset only true header-only pre-created files while preserving any transcript that already contains messages.
