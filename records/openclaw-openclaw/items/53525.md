## Summary

Keep open. Current main still routes Qianfan through the generic OpenAI-compatible completions path, does not add a Qianfan runtime error normalizer, and the shared error formatter still misses Baidu legacy error fields while the stream parser skips top-level chunks without choices.

## What This Changes

Qianfan/Baidu API errors should be parsed and shown to users instead of producing an empty assistant response.

## Best Possible Solution

Add a narrow provider-error normalization path for OpenAI-compatible transports that recognizes Baidu/Qianfan non-2xx and streamed error payloads, preserves valid usage-only chunks, and surfaces a sanitized errorMessage with focused regression tests.
