## Summary

Keep this issue open. Current main still has Qianfan on the generic OpenAI-compatible path, and the code does not normalize Baidu/Qianfan-specific error payload fields into the user-visible assistant error.

## Best Possible Solution

Keep this issue open. Add a narrow Qianfan provider adapter or shared OpenAI-compatible error-normalization helper that promotes Baidu/Qianfan non-2xx and streamed error payloads into errorMessage, including nested error.message/code/type, top-level code/message/requestId, and legacy error_code/error_msg/error_message shapes. Add focused regression coverage in src/agents/provider-http-errors.test.ts and src/agents/openai-transport-stream.test.ts.
