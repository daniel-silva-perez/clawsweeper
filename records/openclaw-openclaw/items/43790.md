## Summary

Keep open. The branch targets a real routing gap that current main still lacks, but the submitted diff is not ready to merge because it can still make an inferred fallback agent authoritative for outbound message routing when config is unavailable at tool construction, and the branch is currently dirty against main.

## What This Changes

The PR threads an explicit agent id through OpenClaw coding-tool and message-tool construction, changes tool-policy session fallback resolution, adds focused routing tests, and adds a changelog bullet.

## Best Possible Solution

Land a rebased routing fix that distinguishes trusted runtime-supplied active agent ids from inferred policy fallback ids, preserves config-aware fallback routing when config is loaded lazily, covers explicit precedence and no-config opaque-session fallback in focused tests, and credits the changelog entry correctly.
