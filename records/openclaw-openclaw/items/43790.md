## Summary

Keep open. Current main still does not thread a trusted active agent id into the message tool, so the PR is not obsolete; however, the patch should not merge as-is because it can promote a fallback-derived policy agent id into an authoritative outbound route and the added changelog entry misses required credit.

## What This Changes

The PR threads an agentId option through OpenClaw tool construction into the message tool, changes tool-policy session fallback resolution, adds focused routing/policy tests, and adds a changelog bullet.

## Best Possible Solution

Land a rebased routing fix that separates trusted active-agent state from inferred policy/session fallback: pass an explicit message-routing agent id only from runtime call sites that truly know the active embedded agent, keep config-aware fallback inside the message action path, cover the opaque-session/no-config default-agent case, and credit the changelog entry.

## Reproduction Assessment

Yes. A high-confidence static/unit reproduction is to build coding/message tools without config for an opaque session key while persisted runtime config has a non-main default agent; the PR can precompute a fallback id and make it authoritative before the message route is resolved from config.

## Solution Assessment

No. The direction is valid, but this implementation conflates trusted explicit routing with fallback inference; the safer solution is a distinct active/message-routing agent input, passed only from trusted runtime call sites, plus regression coverage for the no-config fallback boundary.
