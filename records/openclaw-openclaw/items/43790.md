## Summary

Keep open. Current main still does not give the message tool a trusted explicit active agent id, but this PR is not merge-ready because the diff can still turn an inferred fallback agent into an authoritative outbound route and the changelog entry is missing required contributor credit.

## What This Changes

The PR threads an explicit active agent id through OpenClaw coding-tool construction into the message tool, adds focused routing/policy tests, and adds a changelog entry.

## Best Possible Solution

Land a rebased, guarded version of the routing fix: only pass a trusted runtime-supplied active agent id as explicit, preserve config-aware fallback behavior for inferred or opaque session keys, cover both precedence and fallback cases in focused tests, and add the required changelog attribution.
