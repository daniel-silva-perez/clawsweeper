## Summary

Keep open. Current main still does not implement the PR's explicit message-tool agent routing, but the PR branch has blocking review issues: it can turn an inferred fallback agent into an authoritative outbound route, and its changelog entry is missing the required contributor credit.

## What This Changes

The PR threads an explicit agent id through OpenClaw tool construction into the message tool, adjusts tool-policy session fallback behavior, adds focused routing tests, and adds a changelog entry.

## Best Possible Solution

Land a rebased routing fix that only treats a trusted runtime-supplied active agent id as explicit, preserves config-aware fallback resolution for inferred or opaque session keys, covers explicit precedence and fallback cases in focused tests, and credits the changelog entry correctly.
