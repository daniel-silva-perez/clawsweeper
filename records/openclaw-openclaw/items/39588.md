## Summary

Keep this issue open. Current main still derives the macOS desktop app “Launch at login” toggle from live launchd state for `ai.openclaw.mac`, without a separate persisted user intent or Sparkle/post-update reconciliation path. The related #40213 change is present, but it only removed `KeepAlive` from the desktop app LaunchAgent to prevent relaunch-after-quit behavior; it does not solve update-time persistence or UI/launchd drift.

## Best Possible Solution

Keep the issue open and fix the desktop macOS Launch at login contract directly: persist the user’s intended launch-at-login preference separately from live launchd state, distinguish disabled from stale/unloaded/repair-needed LaunchAgent states, reconcile or re-bootstrap `ai.openclaw.mac` on app startup and after Sparkle updates, and add focused tests for stale plist, unloaded job, and UI reconciliation behavior. Treat #40213 as the separate relaunch-after-quit fix, not the canonical resolution for this issue.
