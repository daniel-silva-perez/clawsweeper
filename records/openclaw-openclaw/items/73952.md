## Summary

Keep open. Current main still has the unscoped external CLI overlay path this PR targets: models.authStatus loads the runtime auth store before computing configured refreshable providers, and the built-in external CLI resolver still reads Codex, Claude CLI, and MiniMax credential sources without a provider scope. The PR is a focused implementation candidate for #73908, not obsolete on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a narrow provider-scoped external CLI overlay fix if targeted auth tests and the changed gate pass, while preserving unrestricted behavior for callers that do not pass a provider scope. Do not close this PR as implemented until current main threads provider scope and Keychain prompt policy through the external CLI overlay path or a different merged PR covers the same behavior.
