## Summary

Keep open. Current main still lacks the provider-scoped external CLI overlay path this PR proposes: `models.authStatus` loads the runtime auth store before computing configured refreshable providers, and the runtime overlay still evaluates every built-in external CLI reader without provider eligibility or keychain prompt policy.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this provider-scoped overlay fix, or an equivalent canonical fix, after targeted auth tests pass and after reconciling it with #74033. The implementation should keep the boundary narrow: scope built-in external CLI overlay reads for `models.authStatus`, propagate keychain prompt policy through the runtime auth-store overlay, and avoid changing supported credential formats or persistence rules.
