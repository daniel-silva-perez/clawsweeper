## Summary

Keep open. Current main has a replay-history stripping mechanism, but the policy remains scoped to Gemma 4 OpenAI-compatible replay and selected provider-owned paths. The Qwen/oMLX-style OpenAI-compatible path described here can still preserve provider reasoning thinking blocks across turns, and the related fix PRs are closed and unmerged.

## What This Changes

The issue asks OpenClaw to stop replaying Qwen/oMLX `reasoning_content` into later OpenAI-compatible conversation history where it causes strict request parsing failures.

## Best Possible Solution

Add a targeted replay-history policy for affected local/self-hosted Qwen-style OpenAI-compatible backends that removes or downgrades historical provider reasoning thinking blocks before follow-up requests, while preserving contracts that require replayed reasoning and current same-turn tool-call continuations.
