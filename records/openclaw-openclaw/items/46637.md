## Summary

Keep open. Current main still has OpenAI-compatible replay-history stripping only for Gemma 4 and selected provider-owned paths, while the reported Qwen/oMLX-style path can still preserve provider reasoning thinking blocks across follow-up turns. The related attempted fixes are closed and unmerged, so there is no strong implemented-on-main close path.

## What This Changes

Prevent Qwen/oMLX `reasoning_content` from being replayed into later OpenAI-compatible conversation-history requests where it can trigger provider-side JSON parse failures.

## Best Possible Solution

Add a targeted replay-history policy for affected Qwen/local self-hosted OpenAI-compatible backends that removes or downgrades historical provider reasoning thinking blocks before follow-up requests, while preserving same-turn tool-call continuations and provider contracts that explicitly require replayed reasoning.
