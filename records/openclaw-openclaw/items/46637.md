## Summary

Keep open. Current main has a replay-history fix for Gemma 4 on OpenAI-compatible transports, but I did not find an equivalent oMLX/Qwen or generic `reasoning_content` fix. The issue discussion also points to a real transcript-replay compatibility gap, and the linked fix PRs were closed unmerged.

## Best Possible Solution

Add a targeted replay policy or provider/compat flag for strict OpenAI-compatible Qwen/oMLX-style backends that strips or safely sanitizes prior-turn `reasoning_content` thinking blocks before `openai-completions` request construction, and cover it with a regression test using a Qwen/oMLX-shaped model id plus `thinkingSignature: "reasoning_content"`. Keep the existing Gemma and provider-owned reasoning behavior intact.
