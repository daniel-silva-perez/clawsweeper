## Summary

Keep open. Current main has a reasoning replay stripping mechanism, but the implemented policy is scoped to Gemma 4 OpenAI-compatible replay and selected provider-owned paths; the Qwen/oMLX-style local OpenAI-compatible replay described here can still preserve `reasoning_content` thinking blocks across turns. The issue discussion, reporter follow-up, and closed unmerged fix PRs point to a real remaining replay-policy bug rather than stale or already-shipped behavior.

## What This Changes

Qwen/oMLX OpenAI-compatible follow-up turns should stop replaying prior assistant `reasoning_content` thinking blocks in conversation history when that replay causes strict backend request rejection.

## Best Possible Solution

Add a targeted replay-history policy for affected local/self-hosted Qwen-style OpenAI-compatible backends that removes or downgrades historical provider reasoning thinking blocks before follow-up requests, while preserving contracts that require replayed reasoning such as DeepSeek/Z.AI preserved-thinking paths and same-turn tool-call continuations.
