Keep open. Current main has the newer reasoning-history stripping machinery, but the policy is still Gemma 4-scoped and does not cover the reported Qwen/oMLX-style OpenAI-compatible replay path. ## What This Changes

The issue requests that OpenClaw stop replaying Qwen/oMLX `reasoning_content` in later OpenAI-compatible conversation history when it causes strict backend request parsing failures.

## Best Possible Solution

Add a targeted replay-history policy for affected Qwen/local self-hosted OpenAI-compatible backends that removes or downgrades historical provider reasoning thinking blocks before follow-up requests, while preserving providers that intentionally require replayed reasoning and current same-turn tool-call continuations.
