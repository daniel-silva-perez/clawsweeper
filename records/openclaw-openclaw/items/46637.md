## Summary

Keep open. Current main has a replay-stripping mechanism, but it is only enabled for Gemma 4 OpenAI-compatible histories and selected provider-owned cases; Qwen/oMLX-style local OpenAI-compatible histories can still replay `reasoning_content`/`reasoning`/`reasoning_text` thinking blocks. The issue discussion and closed unmerged fix PRs support a real remaining bug rather than a stale report.

## What This Changes

The issue requests that OpenClaw stop replaying or safely sanitize Qwen/oMLX `reasoning_content` in subsequent OpenAI-compatible conversation-history requests so follow-up turns do not fail with JSON parse errors.

## Best Possible Solution

Add a targeted replay-history policy for affected local/self-hosted Qwen-style OpenAI-compatible backends that removes or downgrades historical provider reasoning thinking blocks before follow-up requests, while preserving provider-owned contracts that explicitly require replayed reasoning such as DeepSeek/Z.AI preserved-thinking paths and same-turn tool-call continuations.
