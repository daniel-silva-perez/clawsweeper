## Summary

Keep open. Current main has a replay-history stripping mechanism, but it is still scoped to Gemma 4 OpenAI-compatible replay and provider-owned hooks, not the reported Qwen/oMLX-style path. The code still captures `reasoning_content` as replayable thinking, and linked fix attempts are closed unmerged while the reporter confirmed the regression after the original report.

## What This Changes

The issue requests that OpenClaw stop replaying Qwen/oMLX `reasoning_content` in subsequent OpenAI-compatible conversation history when it causes strict JSON parse failures.

## Best Possible Solution

Add a targeted replay-history policy for affected local/self-hosted Qwen-style OpenAI-compatible backends that removes or downgrades historical provider reasoning thinking blocks before follow-up requests, while preserving providers and same-turn tool continuations that intentionally require replayed reasoning.
