## Summary

Keep open. Current main still has a replay-history cleanup mechanism, but the policy remains scoped to Gemma 4 and selected provider-owned paths; the reported Qwen/oMLX-style OpenAI-compatible path can still preserve `reasoning_content` thinking blocks across turns. The issue discussion provides a clear two-turn failure pattern, and the related fix attempts in the provided context are closed unmerged.

## What This Changes

The issue reports that Qwen/oMLX `reasoning_content` is replayed into later OpenAI-compatible conversation history, causing follow-up requests to fail with JSON parse errors.

## Best Possible Solution

Add targeted replay-history hygiene for affected Qwen-style local or self-hosted OpenAI-compatible backends so historical provider reasoning blocks are removed or downgraded before follow-up requests, while preserving same-turn tool-call continuations and explicit provider contracts that require replayed reasoning.

## Solution Assessment

Yes for the repair direction. The narrow maintainable fix is replay policy/hygiene for affected Qwen-style OpenAI-compatible history, not a generic JSON escaping patch or unconditional stripping across all providers.
