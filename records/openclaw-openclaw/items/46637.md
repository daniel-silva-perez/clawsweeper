Keep open. Current main has a reasoning-history stripping mechanism, but the active OpenAI-compatible policy still only opts Gemma 4 into `dropReasoningFromHistory`; the reported Qwen/oMLX-style replay path can still carry `reasoning_content` thinking blocks into later turns. ## What This Changes

The issue requests stripping or sanitizing Qwen/oMLX `reasoning_content` from replayed OpenAI-compatible conversation history so follow-up turns no longer send backend-rejected payloads.

## Best Possible Solution

Add a targeted replay-history policy for affected Qwen/local self-hosted OpenAI-compatible backends that removes or downgrades historical provider reasoning thinking blocks before follow-up requests, while preserving providers and same-turn tool continuations that explicitly require replayed reasoning.
