## Summary

Current main still reproduces the reported tilde-expansion bug for the default builtin memory backend. The QMD backend path resolver expands `~`, but `normalizeExtraMemoryPaths()` itself still trims and then feeds `~/...` through `path.isAbsolute()`, so it resolves under the workspace. The related fixing PR was closed unmerged, so there is no shipped implementation to close against.

## Best Possible Solution

Keep this issue open and fix the builtin memory extra path normalizer in both `packages/memory-host-sdk/src/host/internal.ts` and `src/memory-host-sdk/host/internal.ts` by expanding `~` with the existing home-path helper before the absolute/workspace-relative branch. Add focused regression tests for `~/...` in the normalizer and, if practical, a `listMemoryFiles()` or `readMemoryFile()` path test to prove configured home-relative extra paths are indexed/read correctly without changing QMD behavior.
