## Summary

Current main still reproduces the reported builtin memory extra path behavior: `~/...` is trimmed and passed to `path.isAbsolute()`, which treats it as relative, so it resolves under the agent workspace. The QMD resolver has separate tilde handling, but the default backend remains `builtin`, and the related fixing PR was closed unmerged.

## Best Possible Solution

Keep this issue open. The best fix is to expand a leading `~` in `normalizeExtraMemoryPaths()` before the absolute/workspace-relative branch, in both `packages/memory-host-sdk/src/host/internal.ts` and `src/memory-host-sdk/host/internal.ts`, using the existing path helper semantics. Add focused regression coverage for `~/...` in the normalizer and, ideally, one list/read path test proving a home-relative configured extra path is indexed or authorized correctly without changing QMD behavior.
