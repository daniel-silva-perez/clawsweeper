## Summary

Keep open. Current main still lacks the WhatsApp pre-built linkPreview pass-through, lacks a WhatsApp config/socket opt-in for automatic high-quality previews, and exposes only the single-message Gateway send RPC. The prior related work is useful context, but #18414 targeted older src/web paths and the field is absent from the current plugin path, #43397 is closed unmerged, and #6722 is closed as superseded by this broader tracker.

## What This Changes

Request support for caller-supplied WhatsApp link preview metadata, an opt-in automatic high-quality preview setting, and a batch send Gateway RPC for bulk outbound messages.

## Best Possible Solution

Split the request into staged work: first add a typed WhatsApp-owned pre-built linkPreview pass-through through the outbound/Gateway send path with tests and docs, then separately design automatic preview fetching and batch Gateway sending with explicit security and protocol semantics.
