## Summary

Keep this issue open. Current main still lacks the requested WhatsApp pre-built `linkPreview` pass-through, lacks a `channels.whatsapp.generateHighQualityLinkPreview` config option, and has no batch send RPC method. The related link-preview PR was closed unmerged in the provided context, and the current implementation/tests still send plain WhatsApp text payloads as `{ text }` only.

## Best Possible Solution

Keep this open and implement it as an additive contract change: add a typed WhatsApp-owned `linkPreview` pass-through from gateway/message payloads into `ActiveWebSendOptions` and Baileys text content, add an explicit opt-in config for automatic high-quality preview fetching if maintainers accept the SSRF tradeoff, and design batch sending either as a generic gateway `send.batch` method or a message action with per-message results, tests, docs, and idempotency/rate-limit behavior.

## Risks / Open Questions

- The issue bundles two separable changes: WhatsApp link-preview payload/config support and a broader gateway batch-send RPC. Maintainers may want to split implementation or tracking before landing code.
- Automatic high-quality link-preview fetching needs an explicit SSRF/trust-boundary design; pre-built preview pass-through is lower risk but still needs schema validation for untrusted metadata.
