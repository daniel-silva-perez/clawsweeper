## Summary

Keep open. Current main still lacks the WhatsApp pre-built link preview pass-through, lacks a WhatsApp config/socket opt-in for automatic high-quality previews, and exposes only the single-message Gateway send RPC; the related prior items are useful context but do not resolve current behavior.

## What This Changes

The issue requests WhatsApp pre-built link preview pass-through, a configurable automatic high-quality link preview option, and a batch outbound send RPC.

## Best Possible Solution

Split the work into staged changes: first add a typed WhatsApp-owned pre-built linkPreview pass-through through the outbound/Gateway send path with tests and docs, then handle automatic preview fetching and batch sending as separate security and protocol designs.
