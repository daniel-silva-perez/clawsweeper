## Summary

Keep this issue open. Current main still lacks the requested WhatsApp pre-built link preview pass-through, has no WhatsApp socket/config opt-in for automatic high-quality previews, and only exposes the single-message Gateway `send` RPC. The related prior work is useful context, but it does not resolve current behavior: #18414 was reported as merged but the field is absent on current main, #43397 is closed unmerged, and #6722 was closed as superseded by this broader tracker.

## What This Changes

The issue requests WhatsApp support for pre-built link preview payloads in sendMessage, an opt-in automatic link-preview config, and a batch Gateway send RPC method.

## Best Possible Solution

Split the request into staged work: first add a typed WhatsApp-owned pre-built `linkPreview` pass-through through the outbound/Gateway send path with tests and docs, then handle automatic preview fetching and batch Gateway sending as separate design decisions with explicit security and protocol semantics.
