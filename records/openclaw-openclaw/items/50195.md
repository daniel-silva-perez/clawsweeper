## Summary

Keep open. Current main still lacks the WhatsApp pre-built link preview pass-through, lacks a WhatsApp config/socket opt-in for automatic high-quality preview fetching, and exposes only the existing single-message Gateway send RPC. The prior related items are useful context, but #43397 is closed unmerged, #6722 was closed as superseded by this broader tracker, and the prior #18414 link-preview plumbing is not present in current main.

## What This Changes

Request support for caller-supplied WhatsApp link preview metadata, an opt-in Baileys high-quality preview setting, and a batch Gateway send method for multi-message workflows.

## Best Possible Solution

Split the request into staged work: first add a typed WhatsApp-owned pre-built linkPreview pass-through through the outbound/Gateway send path with tests and docs, then handle automatic preview fetching and batch Gateway sending as separate designs with explicit security and protocol semantics.
