## Summary

Keep open. Current main still lacks the requested WhatsApp pre-built link preview pass-through, has no WhatsApp config/socket opt-in for Baileys high-quality preview fetching, and exposes only the existing single-message Gateway send RPC. The related prior work is useful context, but #18414 is not present in the current plugin paths, #43397 was closed unmerged, and #6722 was closed as superseded by this broader tracker.

## What This Changes

Request support for WhatsApp pre-built link preview payloads, an opt-in automatic high-quality preview setting, and a batch Gateway send RPC for multiple outbound messages.

## Best Possible Solution

Split the work into staged, reviewable changes: first add a typed WhatsApp-owned pre-built linkPreview pass-through through the outbound/Gateway send path with focused tests and docs, then handle automatic URL-preview fetching and batch Gateway sending as separate protocol/security designs.
