## Summary

Keep this issue open. Current main still lacks all three requested surfaces: WhatsApp does not accept caller-supplied `linkPreview` data, the WhatsApp socket/config path does not expose `generateHighQualityLinkPreview`, and Gateway only exposes the existing single-message `send` RPC. The request is valid, but it combines a narrow WhatsApp pass-through with SSRF-sensitive automatic fetching and a new protocol method, so it should be split before an automated repair is queued.

## What This Changes

The issue requests WhatsApp support for caller-supplied pre-built link preview metadata, an opt-in for automatic high-quality link previews, and a batch send RPC for multiple outbound messages.

## Best Possible Solution

Split the work into staged maintainable changes: first add a typed WhatsApp-owned pre-built `linkPreview` pass-through through the outbound/Gateway send path with focused tests and docs, then separately design automatic preview fetching with SSRF controls and batch Gateway sending with explicit protocol semantics.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists: current main rejects extra Gateway `send` fields through `SendParamsSchema`, has no `send.batch` method, and the WhatsApp send path builds plain text Baileys payloads as `{ text }` with no `linkPreview` bridge.

## Solution Assessment

No, not as one combined automated fix. The pre-built preview pass-through is narrow, but automatic URL fetching and a new batch RPC are separate security/protocol decisions that should be split and reviewed by maintainers.

## Risks / Open Questions

- Automatic high-quality preview fetching would make the Gateway/server fetch URLs found in outbound messages, so it needs explicit SSRF and privacy controls before any opt-in ships.
- The batch RPC needs product and protocol semantics for per-message idempotency keys, rate limiting, ordering, partial failures, transcript mirroring, and per-message result shape.
- The narrow pre-built `linkPreview` pass-through is likely implementable, but its public shape should be typed and documented rather than copied from the older type-unsafe prior attempt.
