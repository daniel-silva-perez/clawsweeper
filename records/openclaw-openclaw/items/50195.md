## Summary

Keep open. Current main still lacks the requested WhatsApp pre-built link preview pass-through, does not expose a WhatsApp `generateHighQualityLinkPreview` opt-in, and only has the existing single-message Gateway `send` RPC. The report is valid, but it bundles a narrow WhatsApp pass-through with SSRF-sensitive automatic fetching and a new Gateway protocol surface, so it should be split before an automated fix PR is queued.

## What This Changes

The issue requests WhatsApp support for caller-supplied link preview metadata, an opt-in high-quality link preview setting, and a batch send Gateway RPC for bulk sends.

## Best Possible Solution

Split the work into staged maintainer-owned pieces: first add a typed WhatsApp-owned pre-built `linkPreview` pass-through through the outbound/Gateway send path with tests and docs, then handle automatic preview fetching and batch Gateway sending as separate security and protocol designs.
