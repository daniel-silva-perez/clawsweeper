## Summary

Keep open: current main does not implement the new verification parser module, and the PR branch is not ready to merge because its parser targets and tests have drifted from the current tool catalog and owner-only policy shape. The remaining work is security-verification tooling with fail-closed parser expectations and CI/scope questions, so it needs maintainer review rather than conservative cleanup closure.

## What This Changes

This PR adds a new `src/verify` parser module, README, Vitest test, and standalone test runner for extracting tool catalog, aliases, owner-only policy, subagent deny lists, and policy-pipeline metadata for future SMT-LIB2 verification.

## Best Possible Solution

Rebase the PR onto current main, update the parsers and fixtures for the current tool catalog and owner-approval-class policy shape, make unsupported AST forms fail closed instead of truncating extracted data, and align the README with the actual parser-only scope or a maintainer-approved verification lane.
