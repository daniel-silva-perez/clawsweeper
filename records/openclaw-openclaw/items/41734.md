## Summary

Keep open: current main does not implement this `src/verify` work, and the PR is a plausible security-verification feature, but the branch is stale against current tool-policy code and has blocking correctness issues. The parser still targets retired policy names, the tests/docs encode the old 25-tool catalog, and the README documents generator files that the parser-only PR does not add.

## What This Changes

The branch adds a new `src/verify` parser module with README, AST parsers, shared types, a Vitest suite, and a standalone runner for extracting tool catalog, alias, owner-only, subagent-deny, and policy-pipeline metadata for future SMT-LIB2 verification.

## Best Possible Solution

If maintainers want this verification lane in core, rebase the branch, update extraction for the current 32-tool catalog, approval-class owner-only map, and current subagent policy, make unsupported AST forms fail closed, and align the README with parser-only scope unless the generator and Z3 lane are added in the same reviewed change.
