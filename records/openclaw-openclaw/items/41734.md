## Summary

Keep this PR open. Current main still does not contain the proposed `src/verify` AST parser layer, and the work is core security/tool-policy verification infrastructure rather than a ClawHub plugin. It is not merge-ready as-is because current main’s tool catalog and policy symbols have drifted from the PR assumptions, and the parser needs a fresh security/soundness review before it can support verification claims.

## Best Possible Solution

Keep this PR open as an implementation candidate, but request a rebase and update before substantive review: parse the current 32-tool catalog, target `OWNER_ONLY_TOOL_APPROVAL_CLASS_FALLBACKS`, align with current subagent deny lists and pipeline labels, fail closed on unsupported/non-literal AST shapes, and decide whether the verification parser belongs in formal-verification docs plus a CI/check lane before merging.
