## Summary

Keep open: current main does not implement this verification parser module, and the PR still contains merge-blocking drift against the current tool-policy surface. The branch is useful source work, but it needs maintainer scope review plus a rebase to the current 32-tool catalog, approval-class owner-only map, and current subagent deny lists before it can be considered for merge.

## What This Changes

The PR adds a new `src/verify` parser module with README, AST parsers, shared types, a Vitest suite, and a standalone runner for extracting tool catalog, alias, owner-only, subagent-deny, and policy-pipeline metadata for future SMT-LIB2 verification.

## Best Possible Solution

If maintainers want this lane in core, rebase the branch, update extraction for the current catalog and policy shapes, make unsupported AST forms fail closed, and narrow the README to parser-only scope unless the generator and Z3 lane are added in the same reviewed change.
