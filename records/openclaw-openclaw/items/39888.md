## Summary

Keep this PR open. Current main still does not emit a `model.usage` diagnostic event from the `openclaw agent` CLI path, while the reply-runner path does. The PR addresses a real gap, but it needs to be refreshed because it modifies the now-obsolete `src/commands/agent.ts` wrapper and should use the trusted diagnostic emitter used by the existing parity path.

## Best Possible Solution

Refresh this PR or replace it with a small current-main change in `src/agents/agent-command.ts` or a shared usage-diagnostics helper. Emit a trusted `model.usage` event after `updateSessionStoreAfterAgentRun` for CLI/local `openclaw agent` runs, reusing the session-store model/provider/context/cost resolution so telemetry matches persisted session metadata. Add a focused CLI regression test while keeping existing reply-runner behavior unchanged.
