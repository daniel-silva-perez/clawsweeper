## Summary

Keep open. Current main still exposes skills as model-followed instructions with load, eligibility, and prompt-visibility controls, but not the requested trusted/untrusted instruction boundary, role locking, per-skill memory/tool capability enforcement, or runtime monitoring. The issue also has a recent maintainer-side review explicitly keeping it open, and the central prompt/skills files have not changed since that review.

## What This Changes

Request to isolate SKILL.md instructions from trusted agent context by adding role locking, behavioral detection, and per-skill role, memory, and tool capability controls.

## Best Possible Solution

Define and document a trusted/untrusted skill precedence model, extend skill metadata and runtime policy with per-skill role, memory, and tool capabilities, enforce that boundary through prompt assembly and runtime tool/memory policy, and add malicious SKILL.md regression coverage.
