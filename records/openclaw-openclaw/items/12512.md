## Summary

Keep open. Current main still exposes eligible skills through a model-facing prompt catalog and tells the agent to read and follow selected SKILL.md files. Existing controls cover skill visibility, allowlists, size/path limits, installer scanning, sandbox guidance, and plugin prompt-hook policy, but I did not find the requested SKILL.md instruction isolation, role locking, role/memory/tool capability metadata, or behavioral monitoring.

## Best Possible Solution

Keep this open as core security hardening. Define a SKILL.md instruction-isolation and precedence model, add enforceable per-skill capability metadata for role, memory, and tool influence, apply it through prompt assembly/read handling/tool policy/memory/sandbox behavior, and add malicious-skill regression tests plus docs. ClawHub scanning and moderation can supplement that boundary but should not replace it.
