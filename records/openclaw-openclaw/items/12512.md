## Summary

Keep open. Current main still presents SKILL.md as model-followed instructions through the skills prompt/catalog and has load, visibility, env, sandbox, and install-scanning controls, but not the requested trusted/untrusted instruction boundary, role locking, per-skill memory/tool capability enforcement, or runtime monitoring.

## What This Changes

The issue requests instruction isolation and capability boundaries for SKILL.md content so untrusted skill instructions cannot override agent role, memory, tool, or prompt behavior.

## Best Possible Solution

Define and document a maintainer-approved trusted/untrusted skill precedence model, extend skill metadata and runtime policy with per-skill role, memory, and tool-influence controls, enforce that boundary through prompt assembly plus runtime policy, and add malicious SKILL.md regression coverage and docs.
