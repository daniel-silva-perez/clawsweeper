## Summary

Keep open. Current `main` still exposes eligible skills through a model-facing prompt catalog and instructs the agent to read and follow selected `SKILL.md` files. I did not find the requested instruction isolation, role locking, role/memory/tool capability metadata, or behavioral monitoring; the April 26, 2026 maintainer/Codex comment remains accurate against current main.

## Best Possible Solution

Keep this open as core security hardening. Define the trusted/untrusted precedence model for `SKILL.md`, add enforceable per-skill capability metadata for role, memory, and tool influence, apply it through prompt assembly/read handling/tool policy/memory/sandbox behavior, and add malicious-skill regression tests plus docs. ClawHub moderation and scanning can supplement this boundary, but should not replace an explicit core boundary for installed skills.
