## Summary

Keep open. Current main still exposes skills through a model-facing available-skills catalog and tells the model to read and follow a selected SKILL.md; the current metadata/frontmatter policy covers visibility and install gating, not the requested instruction isolation, role locking, per-skill memory/tool capability enforcement, or behavioral monitoring. The recent maintainer-side Codex review already kept this open, and the current main inspection still matches that assessment.

## What This Changes

Request to add instruction isolation and security boundaries so SKILL.md content cannot override agent role, memory, or tool policy.

## Best Possible Solution

Keep this open as core security hardening. Define a trusted/untrusted precedence model for SKILL.md, add enforceable per-skill capability metadata for role, memory, and tool influence, wire it through prompt assembly/read handling/tool policy/memory/sandbox behavior, and then document it with malicious-skill regression coverage.
