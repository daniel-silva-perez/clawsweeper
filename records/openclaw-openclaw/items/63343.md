## Clownfish Work Prompt

Fix the browser plugin `ensureTabAvailable()` race reported in openclaw/openclaw#63343. Current main still discards the `openTab("about:blank")` return value, then immediately re-lists tabs and filters local-managed candidates by `wsUrl`, so a newly created tab can be filtered out while CDP `/json/list` lags. Implement a narrow fix in `extensions/browser/src/browser/server-context.selection.ts` and any needed local constants/tests: capture the `openTab()` result; when `supportsPerTabWs` filtering leaves no candidates but tabs exist, poll briefly for `wsUrl` discovery; tolerate transient `listTabs()` failures while polling; fall back to the latest non-empty tab snapshot or the opened tab when safe. Preserve explicit invalid/stale `targetId` behavior and do not change unrelated browser architecture, dependencies, lockfiles, public plugin APIs, or old branch history from PR #63355. Add focused tests in `extensions/browser/src/browser/server-context.tab-selection-state.test.ts` covering: newly opened tab initially missing `wsUrl` then gaining one, polling exhaustion falling back to a tab without `wsUrl`, `listTabs()` staying empty after `openTab()` so the opened result is used, and transient polling `listTabs()` failures. Add a single user-facing changelog fix entry with proper attribution if repo convention requires it for browser tool bug fixes.
