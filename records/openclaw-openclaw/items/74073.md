## Summary

Keep open. Current main still has the stale-output path: raw tool/toolResult text can be selected before a delayed post-compaction assistant reply, and the retry window is only reached when the first read is empty. The issue is also paired with open implementation PRs, including an open closing PR, so it should not be closed before maintainers choose and land a fix.

## What This Changes

Prevent subagent announce and completion capture from returning stale raw tool JSON when auto-compaction delays the final assistant reply.

## Best Possible Solution

Land one focused agents fix, or combine the overlapping PRs, so auto-compaction is treated as a continuation window across output selection, announce retry, and frozen-result capture, with regression coverage for raw tool output appearing before a delayed post-compaction assistant reply.
