## Summary

Keep open. Current main still has the stale raw-tool-output path after compaction, and the issue is tied to open overlapping fix PRs, including #74113 with closing syntax, so it should not be closed before a coordinated fix lands.

## What This Changes

The issue reports that subagent announce/completion output can select stale raw tool JSON after auto-compaction instead of waiting for the post-compaction assistant reply.

## Best Possible Solution

Land one consolidated fix that treats compaction markers as continuation boundaries for both announce output selection and frozen completion capture, with regression coverage for toolResult-then-compaction limbo, delayed assistant recovery, and timeout partial-progress behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: a child history with a toolResult followed by a synthetic compaction system marker and no later assistant text leaves latestRawText selectable as the subagent output.

## Solution Assessment

No for the issue’s narrow retry-only suggestion as a complete fix. A unified retry may help the announce path, but the maintainable solution also needs to suppress stale raw output at the compaction boundary and cover captureSubagentCompletionReplyUsing.
