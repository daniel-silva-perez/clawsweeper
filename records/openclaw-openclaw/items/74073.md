Keep this issue open. Current main still has the reported stale-output shape in the subagent announce/output path, and open PRs already target this exact bug family, so the maintainer action is to review and choose a fix rather than close the report early.

## What This Changes

The issue requests that subagent completion/announce output wait for post-compaction assistant output instead of selecting stale raw tool or toolResult JSON as the completion reply.

## Best Possible Solution

Land a focused agents fix that treats auto-compaction as a continuation window across both announce and completion-capture paths, prefers delayed assistant or silent-token output over stale raw tool text, and includes regression coverage for raw tool output visible before a post-compaction assistant reply.
