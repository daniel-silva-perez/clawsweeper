## Summary

Keep open. Current main still has the stale raw-tool-output path across compaction markers, and the issue is already tied to open overlapping fix PRs, including one with closing syntax, so it should not be closed before a coordinated fix lands.

## What This Changes

The issue requests that subagent announce/completion output wait for or prefer the post-compaction assistant reply instead of treating stale tool-result JSON as the completed child reply.

## Best Possible Solution

Keep this issue open until maintainers consolidate the open fixes into one path that treats compaction markers as continuation boundaries for both announce output selection and frozen completion capture, with regression tests for toolResult-then-compaction limbo, delayed assistant recovery, and timeout partial progress.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: a child history with a toolResult followed by the synthetic compaction system marker and no later assistant text leaves latestRawText selected as the subagent output.

## Solution Assessment

Unclear. The suggested unified retry helps the announce branch, but the maintainable fix should also cover the snapshot marker boundary and captureSubagentCompletionReplyUsing path that freezes completion output.
