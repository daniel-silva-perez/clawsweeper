## Summary

Keep this issue open. Current main still has the reported race shape in the subagent announce/output path, and this issue is already paired with active open implementation PRs, so the right cleanup action is maintainer PR review rather than closing the report.

## What This Changes

The issue asks for subagent announce output selection to wait or refresh after auto-compaction so delayed assistant replies or silent tokens beat stale raw tool JSON before the parent completion announcement is built.

## Best Possible Solution

Land or consolidate a bounded fix in the subagent announce/output layer that treats auto-compaction as a continuation window, refreshes or polls for delayed assistant or silent output before finalizing, covers both immediate announce and frozen-result capture paths, and adds regression tests for raw tool JSON followed by delayed post-compaction output.
