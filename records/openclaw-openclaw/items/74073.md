## Summary

Keep open. Current main still has a source-level path where subagent announce output can resolve to latest raw tool text instead of waiting for a post-compaction assistant reply, and an open PR uses closing syntax for this report while related PRs cover overlapping parts of the same leak.

## What This Changes

Prevent auto-compaction races in subagent announce and completion capture from forwarding stale raw tool JSON, preferring post-compaction assistant output or silence handling instead.

## Best Possible Solution

Land a single fix, or coordinated set of fixes, that treats synthetic compaction markers as a continuation boundary and prevents successful subagent completion capture from using raw tool output while a post-compaction assistant reply may still arrive. The landed change should cover both `runSubagentAnnounceFlow` and frozen completion capture in `subagent-registry-lifecycle.ts`, with regression tests for the compaction marker window and delayed assistant recovery.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: `chat.history` can include a compaction system marker after tool output, and current `readSubagentOutput` ignores that marker and can return the latest raw tool result before a later assistant reply is available.
