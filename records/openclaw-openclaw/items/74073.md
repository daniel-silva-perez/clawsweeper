## Summary

Keep open. Current main still has a source-level path where subagent completion capture can return raw tool output across a compaction marker, and the issue is already paired with open fix PRs, including one with closing syntax.

## What This Changes

The issue reports a subagent auto-compaction race where completion capture can return stale raw tool output instead of waiting for the post-compaction assistant reply.

## Best Possible Solution

Land one coordinated fix that treats synthetic compaction system markers as continuation boundaries and prevents successful subagent completion capture from using raw tool output while a post-compaction assistant reply may still arrive. The fix should cover both announce output selection and frozen completion capture, with regression tests for the compaction limbo window and delayed assistant recovery.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a child history containing a raw `toolResult` followed by a synthetic compaction system marker and no later assistant text leaves `latestRawText` selected on current main.

## Solution Assessment

Unclear for the current open PR set. The narrow retry approach addresses part of the suggested fix, but the safer maintainable solution needs to cover the snapshot/marker boundary and the frozen-result capture path as well.
