## Summary

Keep this issue open. Current main still lets a non-empty raw tool/toolResult payload win before a delayed post-compaction assistant reply is checked, and the announce flow only retries when the selected reply is empty. An open unmerged PR, #74113, explicitly targets this issue, so the best path is to review and land or revise that PR rather than close the issue now.

## Best Possible Solution

Keep this issue open and review/land or revise PR #74113. The implementation should add a bounded post-wait refresh that prefers a delayed latest assistant reply or silent token over raw tool output before building the completion event, and should include regression coverage for raw tool JSON followed by delayed post-compaction assistant text or silence.
