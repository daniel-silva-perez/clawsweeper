## Summary

Keep this PR open. Current main still has the reported direct-session gap: tool policy resolution is group/channel-only, Feishu DM config still lacks `tools` and `toolsBySender`, and empty sender-scoped policies can still shadow fallback policies. The PR is a valid implementation candidate with substantial regression coverage and prior bot-review fixes, but it is large, stale against current main, and touches security-sensitive tool access behavior, so it needs maintainer review and a rebase rather than cleanup closure.

## What This Changes

The PR adds direct/DM tool-policy resolution for direct session keys, Feishu DM `tools`/`toolsBySender` schema support, generated config baseline updates, and regression tests for DM/group fallback edge cases.

## Best Possible Solution
