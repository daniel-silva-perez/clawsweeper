## Summary

Keep this PR open. Current main still lacks the PR's intended direct-session behavior: the core tool-policy resolver is group/channel-only, Feishu DM config does not accept tools/toolsBySender, and Feishu only wires group tool policy resolution. The PR is not obsolete; it needs rebase plus maintainer and security review because it changes authorization-sensitive tool policy precedence.

## Best Possible Solution

Keep the PR open as useful implementation work. Best path is to rebase or split it into narrow slices: core direct/dm session policy resolution with regression tests, Feishu DM schema/docs/baseline support, and the empty sender-override fallthrough fix. Land only after maintainer review confirms the direct/group/account precedence is fail-closed and compatible with related open work such as #44886 and #49183.
