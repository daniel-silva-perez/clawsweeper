## Summary

Keep open. Current main still advertises `github-copilot/claude-opus-4.6` in the bundled Copilot catalog, and the failure path still turns unclassified 400 responses into `format` failures that flow into shared auth-profile cooldown state. ## What This Changes

The issue asks OpenClaw to remove the stale GitHub Copilot `claude-opus-4.6` default and prevent `model_not_supported` for one Copilot model from cooling down the entire provider profile.

## Best Possible Solution

Implement the two-part fix: make the bundled Copilot catalog conservative or runtime-backed, and treat `model_not_supported`/unsupported-model responses as model-scoped unavailability so fallback can continue on other Copilot models without poisoning the provider profile.

Yes. The issue provides a concrete v2026.4.29 command and gateway log, and current main still contains the advertised model plus the source path that classifies an unrecognized 400 as `format` and records it as profile-wide cooldown.

## Solution Assessment

Yes, if the fix includes both catalog cleanup and model-scoped unsupported-model handling. Only dropping `claude-opus-4.6` would reduce the immediate blast radius but would leave the same failure mode for the next Copilot catalog rotation.
