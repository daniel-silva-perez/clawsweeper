## Summary

Current main still does not implement a config-backed default for `openclaw tui --deliver`. The CLI still defines `--deliver` with a hard false default and passes `Boolean(opts.deliver)`, `runTui` still falls back to `opts.deliver ?? false`, the send path passes that fixed value to `sendChat`, and the public config type/schema/help surfaces still have no `tui.deliver` key. The related implementation PRs were closed unmerged; the later stuck-spinner work addressed a related symptom, not this config request.

## Best Possible Solution

Keep this issue open and implement a narrow public config contract for the TUI: add optional `tui.deliver?: boolean`, resolve omitted CLI input as `config.tui?.deliver ?? false`, preserve `--deliver` as an explicit true override, add or verify `--no-deliver` as an explicit false override, and add focused tests/docs without flipping the absent-config default.
