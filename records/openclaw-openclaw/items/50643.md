Keep this PR open. Current main still has the macOS node first-use TLS path that saves and accepts the presented certificate before standard trust evaluation, and the PR remains an active security fix candidate paired with the open report #50642.

## Best Possible Solution

Keep this PR open for maintainer security review. The best path is to land or revise a hardened macOS GatewayTLSPinningSession fix, add focused regression coverage where feasible, and make the supported private-TLS enrollment path explicit through gateway.remote.tlsFingerprint wiring or a user-confirmed prompt before resolving #50642.
