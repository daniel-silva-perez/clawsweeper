Keep this PR open. Current main still stores and accepts the first presented macOS node TLS certificate before normal trust evaluation, and the same contributor has an open paired security issue linked by closing syntax. This PR is an active security fix candidate, not a cleanup-close candidate.

## Best Possible Solution

Keep this PR open for maintainer security review. The best path is to land or revise a hardened `GatewayTLSPinningSession` fix, add focused regression coverage where feasible, and explicitly support secure private-TLS enrollment through `gateway.remote.tlsFingerprint` or a prompt flow before closing #50642.
