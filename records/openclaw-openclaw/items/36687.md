## Summary

Keep this issue open. Current main has strong shipped guardrails for the WebChat/TUI stale external-route leakage symptom, but the remaining claim is narrower: a 2026.3.2 update/reset path allegedly changed, dropped, or ignored an existing isolated `session.dmScope`. Current main still treats omitted `session.dmScope` as `main`, and the maintainer discussion explicitly separated the fixed routing symptom from the still-unverified update-path regression.

## Best Possible Solution

Keep this open for maintainer verification of the 2026.3.2 update/reset path. If an update can drop or ignore an existing isolated `session.dmScope`, add a targeted migration or doctor repair plus regression coverage around config preservation. If omitted `dmScope` intentionally remains `main` and no reset-path reproduction appears, close only after explicit maintainer product judgment while keeping the shipped WebChat/TUI stale-route guardrails.
