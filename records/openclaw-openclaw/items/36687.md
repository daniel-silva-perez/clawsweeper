## Summary

Keep this issue open. Current main has strong guardrails for the WebChat/TUI stale external-route leakage symptom, but the issue discussion deliberately separates that fixed symptom from the still-unverified claim that the 2026.3.2 update path reset, dropped, or ignored an existing isolated `session.dmScope`. Current main still documents and implements omitted `session.dmScope` as `main`, so closing as implemented would require evidence that the update/reset path itself is now fixed or impossible.

## Best Possible Solution

Keep the issue open for maintainer verification of the 2026.3.2 update/reset path. If an update can drop or ignore an existing isolated `session.dmScope`, add a targeted migration or doctor repair plus regression coverage around config preservation. If omitted `dmScope` intentionally remains `main` and no reset-path reproduction appears, close only after explicit maintainer product judgment while keeping the shipped WebChat/TUI stale-route guardrails.
