## Summary

Keep open. Current main has a generic auto-expand plumbing helper, but the Control UI still passes `autoExpandToolCalls: false` into chat rendering, and the UI does not thread the gateway/session `verboseLevel` into that prop. The linked PR #50034 was closed unmerged, so it is useful context but not shipped implementation evidence.

## Best Possible Solution

Keep this issue open and finish the Control UI wiring: store the resolved `verboseLevel` from `chat.history` or expose the effective default through `sessions.list`, pass `verboseLevel === "full"` as `autoExpandToolCalls`, and add a focused Control UI regression test that a session using `agents.defaults.verboseDefault = "full"` opens tool message/card bodies by default.
