## Summary

Current main still leaves the reported Control UI behavior unfixed: the gateway returns the effective verbose level, but the UI controller drops it and the top-level chat render still hard-codes tool-call auto-expansion off. The closed related PR #50034 is useful prior work, but it was not merged and no equivalent implementation is present in the inspected source.

## What This Changes

The issue requests that Control UI tool-call and tool-result cards open by default when the effective verbose level from session/default config is `full`.

## Best Possible Solution

Add a narrow Control UI fix that stores the effective `chat.history.verboseLevel`, clears stale values on session/error paths alongside `chatThinkingLevel`, and passes `autoExpandToolCalls: chatVerboseLevel === "full"` into `renderChat` while preserving manual per-card toggles. Cover the controller/render path with focused tests and add a user-facing changelog fix entry with reporter/contributor attribution.
