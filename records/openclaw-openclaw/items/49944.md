## Summary

Keep open. Current main still has the reported Control UI gap: the gateway includes the effective `verboseLevel` in `chat.history`, but the Control UI loader ignores it and the top-level chat render still passes `autoExpandToolCalls: false`. The issue also has open PR #74398 with closing syntax for the same bug, so it should remain open until that PR lands or is replaced.

## What This Changes

Control UI should expand tool cards by default when the effective verbose setting is `full`.

## Best Possible Solution

Review and land #74398 if it remains correct, or replace it with an equivalent narrow Control UI fix that stores the effective `chat.history.verboseLevel`, derives tool auto-expansion from `verboseLevel === "full"`, clears stale verbose state on session/history resets, preserves manual toggles, and adds focused regression coverage plus a user-facing changelog entry.
