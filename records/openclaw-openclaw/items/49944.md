## Summary

Current main still matches the reported Control UI bug: the gateway returns the effective verbose level, but the Control UI history loader does not store it and the top-level chat render still hardcodes tool-card auto-expansion off. The issue should stay open because #74398 is an open, unmerged PR with closing syntax for the same bug.

## What This Changes

Control UI should expand tool cards by default when the effective chat verbose level from `agents.defaults.verboseDefault` or session state is `full`.

## Best Possible Solution

Carry the effective `chat.history.verboseLevel` through Control UI state, derive `autoExpandToolCalls` from `verboseLevel === "full"`, preserve manual expansion toggles, clear stale verbose state on the same paths as thinking state, and cover the controller/render path with focused regression tests plus the appropriate user-facing changelog entry.
