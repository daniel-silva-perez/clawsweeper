## Summary

Keep open. Current main still has the reported Control UI data-flow gap: the gateway returns the effective verbose level, but the Control UI history loader does not preserve it and the top-level chat render still hardcodes tool-card auto-expansion off. The issue is also paired with open PR #74398, so it should remain open until that PR lands, is replaced, or is closed by a maintainer.

## What This Changes

The issue asks for Control UI tool cards to default expanded when the effective verbose setting is `full`.

## Best Possible Solution

Land or replace the focused Control UI plumbing so the effective `chat.history.verboseLevel` is stored in UI state, stale verbose state is cleared with the same lifecycle as thinking state, and `verboseLevel === "full"` drives the existing `autoExpandToolCalls` render path without changing gateway contracts or manual disclosure behavior.
