## Summary

Keep open. The checked-out main still has the Control UI data-flow gap: `chat.history` returns an effective `verboseLevel`, but the UI loader stores only `thinkingLevel` and the top-level chat render still passes `autoExpandToolCalls: false`. The live GitHub thread also remains open and has an open closing PR, #74398, so the issue should stay open until that PR lands or is replaced. ([github.com](https://github.com/openclaw/openclaw/issues/49944))

## What This Changes

Tool cards in Control UI should expand by default when the effective verbose level is `full`.

## Best Possible Solution

Review and land #74398 if it remains correct, or replace it with an equivalent narrow Control UI fix that stores the effective `chat.history.verboseLevel`, derives tool auto-expansion from `verboseLevel === "full"`, preserves manual toggles, clears stale verbose state with session/history resets, and adds focused regression coverage plus the appropriate user-facing changelog entry.
