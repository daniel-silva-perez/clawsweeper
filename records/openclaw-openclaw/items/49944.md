## Summary

Keep open. Current main still drops the effective verbose level before Control UI rendering and still hardcodes tool-card auto-expansion off, while open PR #74398 is already paired to this bug with closing syntax and has not merged.

## What This Changes

Control UI should expand tool cards by default when the effective verbose level from `agents.defaults.verboseDefault` or session state is `full`.

## Best Possible Solution

Carry the effective `chat.history.verboseLevel` through Control UI state, derive `autoExpandToolCalls` from `verboseLevel === "full"`, clear stale verbose state on the same lifecycle paths as thinking state, and cover the controller/render path with focused regression tests plus a user-facing changelog entry.

## Reproduction Assessment

Yes. The source path is enough for a high-confidence static reproduction: current main returns `verboseLevel` from `chat.history`, then the Control UI drops it and renders chat with `autoExpandToolCalls: false`.

## Solution Assessment

Yes. The narrow maintainable fix is Control UI state/render plumbing only; #74398 appears to target that exact boundary, so the next step is review, landing, or replacement of that PR rather than closing this issue now.
