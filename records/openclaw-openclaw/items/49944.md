## Summary

Current main still has the reported Control UI gap: the gateway returns the effective verbose level, but the UI controller does not store it and the app render still passes `autoExpandToolCalls: false`. The issue should stay open because #74398 is an open PR with closing syntax for this exact bug; it should close only after that PR, or an equivalent fix, merges.

## What This Changes

The issue requests that Control UI tool cards expand by default when the effective verbose setting is `full`.

## Best Possible Solution

Review and land #74398, or an equivalent narrow Control UI change, so the UI stores the effective `chat.history.verboseLevel`, clears stale verbose state on the same paths as thinking state, passes `autoExpandToolCalls: chatVerboseLevel === "full"`, preserves manual tool-card toggles, and includes focused regression coverage plus a user-facing changelog fix entry.
