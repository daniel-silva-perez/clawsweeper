## Summary

Current main still does not wire the effective verbose level into Control UI tool-card expansion. The gateway returns `chat.history.verboseLevel`, and the chat renderer/tool-expansion state already supports `autoExpandToolCalls`, but the UI controller only stores `thinkingLevel` and `app-render` still passes `autoExpandToolCalls: false`. The focused fix PR #50034 is closed unmerged, and targeted search found no open replacement PR.

## What This Changes

Tool cards should honor `agents.defaults.verboseDefault = "full"` in Control UI by opening tool-card details by default.

## Best Possible Solution

Keep this open and land a narrow Control UI fix that stores the effective `chat.history.verboseLevel`, resets it on session switches/errors like `chatThinkingLevel`, and passes `autoExpandToolCalls: chatVerboseLevel === "full"` into `renderChat`. Add focused controller/rendering regression tests for global `agents.defaults.verboseDefault = "full"`, explicit non-full behavior, stale history responses, and manual tool-card toggles, plus a user-facing changelog fix entry with reporter/contributor attribution.
