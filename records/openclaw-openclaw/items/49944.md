## Summary

Keep this open. Current main still returns the effective verbose level from `chat.history`, but the Control UI drops it before render and still passes `autoExpandToolCalls: false`; the issue is also paired with open, unmerged PR #74398 using closing syntax for the same bug.

## What This Changes

Control UI should expand tool cards by default when the effective session verbose level is `full`.

## Best Possible Solution

Review and land #74398 if it remains correct, or replace it with an equivalent small Control UI fix that persists `chat.history.verboseLevel`, derives tool auto-expansion from `verboseLevel === "full"`, preserves manual toggles, and adds focused regression coverage plus a user-facing changelog entry.
