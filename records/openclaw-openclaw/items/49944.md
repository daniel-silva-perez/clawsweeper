## Summary

Keep open. Current main still matches the reported Control UI bug: the gateway returns the effective verbose level, but the Control UI history loader drops it and the top-level chat render still hardcodes tool-card auto-expansion off. Provided issue context also shows #74398 is an open, unmerged PR with closing syntax for this bug, so this issue should remain open until that PR is reviewed, landed, replaced, or closed.

## What This Changes

The issue requests that Control UI tool cards expand by default when `agents.defaults.verboseDefault` resolves to `full`.

## Best Possible Solution

Carry the effective `chat.history.verboseLevel` through Control UI state, derive `autoExpandToolCalls` from `verboseLevel === "full"`, clear stale verbose state on the same lifecycle paths as thinking state, preserve manual expansion behavior, and cover the controller/render path with focused regression tests plus the appropriate changelog entry.

## Reproduction Assessment

Yes. The source path is enough for a high-confidence static reproduction: current main returns `verboseLevel` from `chat.history`, then the Control UI drops it and renders chat with `autoExpandToolCalls: false`.

## Solution Assessment

Yes. The narrow maintainable fix is Control UI state/render plumbing only; #74398 appears to target that exact boundary, so the next step is review, landing, replacement, or closure of that PR rather than closing this issue now.
