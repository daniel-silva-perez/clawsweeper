## Summary

Keep this PR open. Current main still lacks the remote node-host guard for `openclaw models auth login`, the linked bug remains open, and this PR is a focused implementation candidate rather than obsolete work. The security pass did not find supply-chain or CI risk in the provided diff; it only touches the models auth command and its colocated test.

## Best Possible Solution

Keep this PR open, rebase it onto current main, and review or land a focused remote node-host auth-write guard in `src/commands/models/auth.ts` with regression coverage in `src/commands/models/auth.test.ts`. Prefer shared loopback classification where possible, and explicitly test the intended scope across OAuth, device-code, API-key, token, and custom provider auth methods. After merge, the linked bug can close as implemented.
