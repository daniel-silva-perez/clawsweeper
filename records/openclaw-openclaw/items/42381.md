## Summary

Keep this PR open. Current main still lacks an equivalent remote node-host guard for `openclaw models auth login`, the linked bug #42291 is still open, and this PR remains a focused implementation candidate rather than obsolete work. The security review did not find supply-chain or CI risk in the provided diff; it only changes the models auth command and its colocated test.

## Best Possible Solution

Keep this PR open, rebase it if needed, and review or land a focused remote node-host auth-write guard in `src/commands/models/auth.ts` with regression coverage in `src/commands/models/auth.test.ts`. Prefer shared loopback classification if available, explicitly decide the scope across OAuth, token, API-key, and custom auth methods, then close #42291 after the fix merges.
