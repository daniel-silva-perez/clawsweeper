Keep this PR open. I also did a PR security pass: the provided diff only touches the model auth command and its test, with no workflow, dependency, lockfile, install, build, or release-script changes.

## Best Possible Solution

Keep this PR open, rebase it onto current main, and review or land a focused remote node-host auth-write guard in `src/commands/models/auth.ts` with regression coverage in `src/commands/models/auth.test.ts`. Prefer the shared loopback host classification from `src/gateway/net.ts`, and document the intended scope across OAuth, device-code, token, API-key, and custom provider auth methods. After merge, #42291 can close as implemented.
