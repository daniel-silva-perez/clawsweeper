## Summary

Keep open. Current main still lacks an equivalent remote node-host guard for `openclaw models auth login`; the linked bug #42291 is still open, and this PR remains a focused implementation candidate rather than obsolete work. Security pass found no supply-chain or CI-risk changes in the provided diff.

## Best Possible Solution

Keep this PR open, rebase it if needed, and review or land a focused remote node-host auth-write guard in `src/commands/models/auth.ts` with regression coverage in `src/commands/models/auth.test.ts`. Prefer reusing the existing shared loopback classification, decide the intended scope across OAuth/API-key/token auth methods, then close #42291 after the fix merges.
