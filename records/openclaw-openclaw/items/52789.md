## Summary

Keep open. Current main still has only `none`, `announce`, and `webhook` cron delivery modes. The existing `sessionTarget: "session:<id>"` support runs the cron job in a persistent session, but it does not deliver the completed output to a separate session through `sessions_send`, which is the central capability requested here. Related prior ClawSweeper context covers adjacent cron delivery bugs, not this missing delivery mode.

## Best Possible Solution

Keep this issue open and implement it as a core cron delivery feature if maintainers want the product behavior: extend cron delivery types, protocol schema, normalizer, CLI/UI/docs, and tests; add a framework-owned dispatch path that sends the final cron summary to a target session with `sessions_send`-equivalent provenance and access-policy checks; record delivery diagnostics without posting to a visible chat channel.
