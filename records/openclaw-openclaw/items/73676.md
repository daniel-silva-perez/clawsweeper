## Summary

Keep this issue open. Current main still has unguarded deleted-cwd-sensitive `process.cwd()` calls in CLI startup, and open PR #73700 uses closing syntax for this issue but has not merged, so it is the implementation candidate rather than a close reason.

## Best Possible Solution

Keep this issue open and use PR #73700 as the active implementation candidate. The best fix is a small shared safe-cwd path for CLI bootstrap, covering `shouldLoadCliDotEnv`, `loadCliDotEnv`, and `isMainModule` entrypoint detection, plus focused regression tests that mock `process.cwd()` throwing `ENOENT: uv_cwd`. After the PR merges and current main contains the fix, the issue can close as implemented.
