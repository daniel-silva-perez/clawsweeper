## Summary

Keep open. Current main still has unguarded `process.cwd()` calls on the startup, dotenv, PATH bootstrap, and TUI paths exercised by the report, and the issue is paired with open closing PR #74994, so it should remain open until maintainers merge or close that candidate.

## What This Changes

The issue asks OpenClaw CLI startup to tolerate a deleted current working directory instead of crashing with Node's `ENOENT: no such file or directory, uv_cwd`.

## Best Possible Solution

Land one consolidated deleted-cwd startup fix that carries a safe cwd/default through main-module detection, CLI/runtime dotenv loading, home-dir fallback, PATH bootstrap, and TUI startup, with focused regressions for the original `openclaw tui` repro while preserving normal cwd-local `.env` precedence.
