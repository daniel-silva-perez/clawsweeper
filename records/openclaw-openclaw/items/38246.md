## Summary

Keep this issue open. Current `main` still sends visible WhatsApp auto-replies through the inbound message's `reply`/`sendMedia` closures, which are bound to the originating chat. The existing main-DM pinning only protects stored `lastRoute` metadata and does not block, reroute, or force explicit targets for plain replies from allowlisted non-owner DMs.

## Best Possible Solution

Keep this issue open and choose a WhatsApp plugin routing policy for non-owner DMs. The likely implementation belongs in the WhatsApp visible auto-reply delivery path before `deliverWebReply` sends payloads, with typed config/schema/docs and focused regression coverage for owner DMs, allowlisted non-owner DMs, and explicit `message` tool sends that should remain intentional.
