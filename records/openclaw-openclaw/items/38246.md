## Summary

Current main still lacks a WhatsApp safeguard that blocks or forces explicit routing for plain auto-replies from allowlisted non-owner DMs. The inbound WhatsApp message object still binds `reply`/`sendMedia` to the originating chat, and the auto-reply dispatcher/delivery path still sends visible replies through those closures without checking owner-vs-third-party routing policy.

## Best Possible Solution

Keep this open and choose an explicit WhatsApp plugin routing policy for non-owner DMs. The likely implementation belongs in the WhatsApp visible auto-reply path before `deliverWebReply` sends payloads, with typed config/schema/docs and regression tests for owner DMs, allowlisted non-owner DMs, and explicit `message` tool sends that should remain intentional.
