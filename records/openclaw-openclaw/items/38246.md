## Summary

Current main still lacks the requested WhatsApp DM safeguard. Visible auto-replies are still delivered through the inbound message's `reply`/`sendMedia` closures, which are bound to the originating chat, while current docs/config only cover DM allowlists, explicit outbound targets, and main-DM `lastRoute` pinning. No open PR referencing this issue was found, so this should remain open for maintainer product/security review rather than close as implemented or duplicate.

## Best Possible Solution

Keep this open for maintainer product/security review. The likely implementation belongs in the WhatsApp plugin visible auto-reply path before `deliverWebReply` sends payloads, with an opt-in typed root/account config policy, schema/help/docs updates, and regression tests covering owner DMs, allowlisted non-owner DMs, and explicit `message` tool sends that should remain intentional.
