## Summary

Keep open. Current main still does not implement Vapi as a bundled Voice Call provider, and recent maintainer discussion already kept the item open for product and ownership judgment rather than treating it as stale, duplicate, or implemented. The related Clawdia Vapi bridge is useful context, but it does not satisfy this issue's requested `provider: "vapi"` support in `@openclaw/voice-call`.

## What This Changes

The issue requests adding Vapi as a bundled Voice Call plugin provider with provider code, config/env schema, runtime registration, tests, and documentation.

## Best Possible Solution

Maintainers should choose the ownership path first. If Vapi belongs in the bundled Voice Call plugin, implement it end-to-end across provider code, config/env/schema validation, runtime selection, manifest metadata/secret inputs, tests, docs, and changelog. If it should remain external, document the Clawdia bridge path or add a public telephony-provider seam before tracking Vapi as ClawHub/plugin work.
