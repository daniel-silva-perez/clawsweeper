## Summary

Keep open. Current main still does not implement Vapi as a bundled Voice Call provider, and the latest maintainer discussion already kept the request open for product and ownership judgment. The external Clawdia phone bridge is useful related context, but it is not `provider: "vapi"` support in `@openclaw/voice-call`, and the broader Vapi-channel issue does not supersede this specific bundled-provider request.

## What This Changes

Add Vapi as a Voice Call plugin provider with provider code, config and environment schema, runtime registration, tests, and docs.

## Best Possible Solution

Maintainers should choose the ownership path first. If Vapi belongs in the bundled Voice Call plugin, implement it end-to-end across provider code, config/env/schema validation, runtime selection, manifest metadata/secret inputs, tests, docs, and changelog; if it should remain external, document the Clawdia bridge path or add a public telephony-provider seam before tracking Vapi as ClawHub/plugin work.
