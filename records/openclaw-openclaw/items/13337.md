## Summary

Keep open. Current main still lacks Vapi support in the bundled Voice Call provider surface, and recent maintainer discussion kept the item open for product and ownership judgment. The external Clawdia Vapi bridge is useful context, but it does not satisfy the requested `provider: "vapi"` path in `@openclaw/voice-call`.

## What This Changes

The issue requests adding Vapi as a Voice Call plugin provider, including provider implementation, config schema/env support, runtime registration, tests, and docs.

## Best Possible Solution

Maintainers should choose the ownership path first. If Vapi belongs in the bundled Voice Call plugin, implement it end-to-end across provider code, config/env/schema validation, runtime selection, manifest metadata/secret inputs, tests, docs, and changelog. If it should stay external, document the Clawdia bridge or add a public telephony-provider seam before tracking Vapi as ClawHub/plugin work.
