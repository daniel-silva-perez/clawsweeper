## Summary

Keep open. Current main still does not implement Vapi as a bundled Voice Call provider, and the latest issue discussion already kept the item open for maintainer product and ownership judgment. The external Clawdia Vapi bridge is useful related context, but it is not `provider: "vapi"` support in `@openclaw/voice-call`, and #6838 does not supersede this more specific bundled-provider request.

## What This Changes

Add a Vapi provider to the bundled Voice Call plugin, including provider implementation, config/schema/env wiring, runtime selection, exports, tests, and docs.

## Best Possible Solution

Maintainers should decide the ownership path first. If Vapi belongs in the bundled Voice Call plugin, implement it end-to-end across provider code, config/env/schema validation, runtime selection, manifest metadata/secret inputs, tests, docs, and changelog; if it should remain external, document the Clawdia bridge path or add a public telephony-provider seam before tracking Vapi as ClawHub/plugin work.
