## Summary

Keep open. Current main still does not implement Vapi as a bundled Voice Call telephony provider: provider ids, config/env handling, runtime provider selection, provider exports/files, manifest schema, tests, and public docs all remain limited to Telnyx, Twilio, Plivo, and Mock. The existing Clawdia Vapi bridge is useful adjacent work, but it is documented as an external HTTP bridge, not `provider: "vapi"` support inside `@openclaw/voice-call`, and the current plugin capability docs do not expose a public telephony-provider registration seam that makes this request clearly external-only.

## Best Possible Solution

Keep this issue open for maintainer/product judgment. If Vapi should be first-party in the bundled Voice Call plugin, implement a Vapi VoiceCallProvider, Vapi config/env validation, runtime provider selection, manifest metadata, provider/runtime tests, and docs at https://docs.openclaw.ai/plugins/voice-call. If maintainers prefer external ownership, document the Clawdia bridge path or add a public Voice Call telephony-provider registration seam, then track Vapi as a ClawHub/plugin item.
