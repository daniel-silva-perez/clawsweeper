## Summary

Keep open. Current main still does not implement Vapi as a bundled Voice Call telephony provider: no Vapi provider file, provider id, config/env/schema support, runtime provider branch, manifest metadata, tests, changelog entry, or public Voice Call docs entry exists. The documented Clawdia bridge is useful adjacent external work, but it is an HTTP bridge, not `provider: "vapi"` support in `@openclaw/voice-call`.

## Best Possible Solution

Keep this open for maintainer/product judgment. If Vapi should be first-party in the bundled Voice Call plugin, add a `VapiProvider`, Vapi config/env/schema validation, runtime provider selection, manifest metadata, provider/config/runtime tests, and docs at https://docs.openclaw.ai/plugins/voice-call. If maintainers prefer external ownership, document the Clawdia bridge path or first add a public Voice Call telephony-provider registration seam, then track Vapi as a ClawHub/plugin item.
