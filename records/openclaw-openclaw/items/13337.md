## Summary

Keep open. Current main still does not implement Vapi as a bundled Voice Call provider, and the latest maintainer discussion explicitly kept this item open for product and ownership judgment. The external Clawdia Vapi bridge is useful related context, but it is not `provider: "vapi"` support in `@openclaw/voice-call`, and the broader Vapi-channel request does not supersede this more specific bundled-provider request.

## What This Changes

Add a Vapi provider to the bundled Voice Call plugin, including provider implementation, config/schema/env wiring, runtime selection, exports, tests, and docs.

## Best Possible Solution

Maintainers should decide the ownership path first. If Vapi belongs in the bundled Voice Call plugin, add it end-to-end across provider code, config/env/schema validation, runtime selection, manifest metadata/secret inputs, tests, docs, and changelog. If it should remain external, document the Clawdia bridge path or add a public telephony-provider seam before tracking Vapi as a ClawHub/plugin item.
