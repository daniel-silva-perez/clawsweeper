## Summary

Keep open. Current main still does not implement Vapi as a bundled Voice Call provider, and recent maintainer discussion kept this as a product and ownership decision rather than treating it as stale, duplicate, or implemented. The related closed Vapi-channel and config issues do not satisfy this specific `provider: "vapi"` request for `@openclaw/voice-call`.

## What This Changes

Add a Vapi telephony provider to the bundled Voice Call plugin, including provider code, config/schema/env support, registration, tests, and docs.

## Best Possible Solution

Maintainers should choose the ownership path first. If Vapi belongs in the bundled Voice Call plugin, implement it end-to-end across provider code, config/env/schema validation, runtime selection, manifest metadata/secret inputs, tests, docs, and changelog. If it should remain external, document the external bridge path or add a public telephony-provider seam before tracking Vapi as ClawHub/plugin work.
