## Summary

Keep this issue open. Current main at 43da08979035503401a21a06a55e66920411ad67 still does not implement a Vapi telephony provider for the bundled Voice Call plugin: the provider enum, config/env handling, runtime switch, provider files, manifest, and public docs all list Telnyx, Twilio, Plivo, and Mock only. The existing Clawdia Vapi bridge is useful adjacent external work, but it is an HTTP bridge, not provider: "vapi" support in @openclaw/voice-call, and there is still no public telephony-provider registration seam that makes this exact request external-only.

## Best Possible Solution

Keep this issue open for maintainer/product judgment. If Vapi should be first-party in the bundled Voice Call plugin, implement a Vapi VoiceCallProvider, Vapi config/env validation, runtime provider selection, manifest metadata, provider/runtime tests, and docs at https://docs.openclaw.ai/plugins/voice-call. If maintainers prefer external ownership, document the Clawdia bridge path or add a public Voice Call telephony-provider registration seam, then track Vapi as a ClawHub/plugin item.
