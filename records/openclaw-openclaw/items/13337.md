## Summary

Keep open. Current main still has no bundled Vapi Voice Call provider, and the recent maintainer discussion explicitly kept this request open for product and ownership judgment. The Clawdia Vapi bridge is useful external context, but it does not provide `provider: "vapi"` support inside `@openclaw/voice-call`, and the related closed Vapi issues do not supersede this more specific bundled-provider request.

## What This Changes

The issue requests adding Vapi as a new provider in the bundled Voice Call plugin, including provider code, config/env/schema support, registration/runtime selection, tests, and docs.

## Best Possible Solution

Maintainers should choose the ownership path first. If Vapi belongs in the bundled Voice Call plugin, implement it end-to-end across provider code, config/env/schema validation, runtime selection, manifest metadata and secret inputs, tests, docs, and changelog. If Vapi should stay external, document the Clawdia bridge path or add a public telephony-provider capability before tracking Vapi as ClawHub/plugin work.
