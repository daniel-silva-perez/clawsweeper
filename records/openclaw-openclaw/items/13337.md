## Summary

Keep open. Current main still does not implement Vapi as a bundled Voice Call telephony provider, and the recent maintainer discussion explicitly kept this issue open for product and ownership judgment. The Clawdia Vapi bridge is useful related context, but it does not satisfy the requested `provider: "vapi"` path in `@openclaw/voice-call`.

## What This Changes

The issue requests adding Vapi as a Voice Call plugin provider, including provider implementation, config/env/schema support, runtime registration, tests, and docs.

## Best Possible Solution

Maintainers should choose the ownership path first. If Vapi belongs in the bundled Voice Call plugin, implement it end-to-end across provider code, config/env/schema validation, runtime selection, manifest metadata/secret inputs, tests, docs, and changelog. If it should stay external, document the Clawdia bridge path or add a public telephony-provider seam before tracking Vapi as ClawHub/plugin work.
