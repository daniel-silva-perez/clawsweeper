## Summary

Keep this PR open. Current main still serializes full session entries, including `skillsSnapshot`, and disk-budget sizing still counts those bytes. The PR is a focused implementation candidate for the sessions.json bloat reports, especially open #45718, but its current global key-name replacer needs revision before merge because it can drop nested/plugin-owned data named `skillsSnapshot`.

## Best Possible Solution

Keep the PR open and revise it into a session-entry-aware serialization helper that strips only top-level `SessionEntry.skillsSnapshot`, preserves `systemPromptReport`, makes disk-budget sizing share the exact save serialization contract, and adds regression coverage proving nested/plugin-owned `skillsSnapshot` keys survive save/load. Then review and land it against the linked sessions.json bloat reports, especially #45718.
