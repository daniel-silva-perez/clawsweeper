## Summary

Keep this PR open. Current main still has the reported sessions.json bloat path, so the change is not obsolete, but the PR should not merge as-is because its global key-name replacer can drop persisted non-transient data named skillsSnapshot elsewhere in a session entry.

## What This Changes

The PR adds a shared JSON.stringify replacer to omit skillsSnapshot from session-store saves and disk-budget sizing, while preserving systemPromptReport and adding serialization tests.

## Best Possible Solution

Revise the PR into a session-entry-aware serialization helper that omits only top-level SessionEntry.skillsSnapshot, preserves systemPromptReport and nested/plugin-owned skillsSnapshot keys, and makes disk-budget sizing share the exact same serialization contract as saveSessionStoreUnlocked.

## Reproduction Assessment

Yes. A high-confidence reproduction path exists by saving a store entry with skillsSnapshot on current main and observing plain JSON.stringify persistence; the PR-specific regression can be reproduced by stringifying a session entry that has a nested acp.runtimeOptions.backendExtras.skillsSnapshot key with the proposed replacer.

## Solution Assessment

No. Excluding transient skillsSnapshot from disk is the right direction, but the current PR implements it with a global key-name replacer; a path-aware or entry-copy serialization helper is the narrower maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Scope the replacer to session entries:** `src/config/sessions/store-serialization.ts:10`
  - body: This replacer runs for every property visited by JSON.stringify, so it drops any nested key named skillsSnapshot, not only the transient SessionEntry.skillsSnapshot field. ACP runtime backendExtras accepts arbitrary safe keys and persists them under session ACP metadata, so a saved backendExtras.skillsSnapshot value would disappear after save/restart. Please make the serialization helper path-aware or strip only the top-level field from each session entry.
  - confidence: 0.91
