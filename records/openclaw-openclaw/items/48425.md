Keep this PR open. Current main still writes full session entries to sessions.json and still measures disk budget with unfiltered JSON.stringify output, so the PR is not obsolete or implemented. The PR is a focused candidate for the linked session-store bloat reports, but it needs a small revision so serialization removes only the intended top-level SessionEntry.skillsSnapshot field and preserves nested/plugin-owned keys.
