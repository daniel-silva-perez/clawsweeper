## Summary

Keep this PR open. Current main still serializes and disk-budgets full session entries, so the intended sessions.json bloat fix is not implemented. The PR is a focused candidate for the linked skillsSnapshot bloat reports, but it should be revised before merge because the proposed replacer removes any nested key named skillsSnapshot, not only the top-level SessionEntry field.

## Best Possible Solution

Keep the PR open and revise it into a session-entry-aware serialization helper that strips only top-level SessionEntry.skillsSnapshot, preserves systemPromptReport, makes disk-budget sizing share the exact save serialization path, and adds regression coverage proving nested/plugin-owned skillsSnapshot keys survive save/load. Then review and land it against the linked sessions.json bloat reports, especially open #45718.
