## Summary

Keep open. Current main has moved replay hygiene into provider replay-policy hooks, but it still does not implement this PR's central behavior: there is no openai-completions boundary sanitizer for incompatible thinking signatures, policy selection is not history-aware, and OpenRouter-wrapped Mistral model IDs still do not get strict9 replay tool-call IDs.

## Best Possible Solution

Keep this PR open for maintainer follow-up, or replace it with a rebased current-main patch in `src/agents/pi-embedded-runner/replay-history.ts`, `src/agents/pi-embedded-helpers/openai.ts`, and provider replay-policy helpers. The fix should preserve compatible provider-native thinking signatures until the replay target is known, strip unsupported signatures only for `openai-completions` including current fields such as `reasoning_details`, and handle OpenRouter-wrapped Mistral strict9 replay without a broad global hint table.
