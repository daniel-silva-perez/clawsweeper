## Summary

Keep open. Current main still has the reported outbound hook/session-key gap, and this PR remains a focused implementation candidate, but the new public hook-context docs overstate fields that the patch does not actually deliver.

## What This Changes

The PR threads a canonical session key into shared outbound `message_sending`, aligns native redirect routed replies with the agent runtime session key, and adds tests plus hook/session context documentation.

## Best Possible Solution

Land the narrow session-key runtime fix with the public contract kept exact: either thread the same canonical fields into every hook the docs promise, or narrow the JSDoc so it only promises implemented `message_sending` session-key behavior and treats outbound `runId` as future work.

## Reproduction Assessment

Yes. Current-main source inspection gives a high-confidence reproduction path: invoke `deliverOutboundPayloads` with a session and an enabled `message_sending` hook, and the constructed hook context omits `sessionKey`; native redirect routing similarly diverges from the agent runner's `targetSessionKey || ctx.SessionKey` resolution.

## Solution Assessment

Not as written. The runtime changes are the right narrow direction, but the documentation part should not merge while it advertises `message_sent` session keys and `message_sending` run ids that the patch does not provide.
