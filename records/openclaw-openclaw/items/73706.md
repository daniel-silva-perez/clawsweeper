## Summary

Keep this PR open. Current main still omits `sessionKey` from the shared outbound `message_sending` hook context even though that value is already resolved in `deliverOutboundPayloadsCore`, the plugin hook context type supports it, and the docs describe `ctx.sessionKey` as an available correlation field. The PR remains a small additive implementation candidate with focused tests and no observed security-sensitive file changes.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land the focused shared-outbound `sessionKey` threading with its regression tests after normal validation, then either include the same context threading for Telegram direct delivery or file a narrow follow-up if maintainers want that handled separately.
