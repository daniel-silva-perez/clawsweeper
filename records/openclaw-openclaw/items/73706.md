## Summary

Keep this PR open. Current main still omits the canonical session key from the shared outbound `message_sending` hook context, and native-command redirects still pass `ctx.SessionKey` to outbound delivery even though the agent runtime runs with `CommandTargetSessionKey || ctx.SessionKey`. The PR remains a focused implementation candidate with tests and no security-sensitive file changes in the reviewed diff.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land the focused shared-outbound `sessionKey` threading, native redirect session-key alignment, and regression tests after normal validation, then decide whether Telegram direct delivery and outbound `runId` propagation are in scope or should be tracked separately.
