## Summary

Keep this PR open. Current main still omits the canonical session key from the shared outbound `message_sending` hook context and still passes `ctx.SessionKey` into routed replies for native-command redirects, while the docs/type surface already advertise `ctx.sessionKey` as a hook correlation field. The PR remains a focused implementation candidate with no security-sensitive file changes visible in the reviewed diff.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land the focused shared-outbound `sessionKey` threading, native redirect session-key alignment, and regression tests after normal validation, then decide whether Telegram direct delivery belongs in the same PR or a follow-up.
