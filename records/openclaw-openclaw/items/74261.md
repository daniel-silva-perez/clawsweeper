## Summary

Keep open. Current main does not expose a WhatsApp group-creation agent tool, message action, or active-listener operation. Baileys appears to support `groupCreate(subject, participants)`, so the request is technically plausible, but turning that into an AI-callable WhatsApp group/invite tool is security- and product-sensitive and should get maintainer review first. The issue body also mixes this request with `responsePrefix` examples; that prefix surface already exists separately, so the remaining scope should be clarified as WhatsApp group creation.

## Best Possible Solution

Keep this open for maintainer product/security review. If accepted, implement a tightly gated WhatsApp-owned group-creation capability in the bundled WhatsApp plugin, or first design a narrow plugin API that lets an external plugin safely use the linked WhatsApp session. The implementation should validate/normalize participants, call Baileys `groupCreate`, redact identifiers in logs, record outbound activity, document the owner-only/security boundary, and keep the existing `responsePrefix` config discussion separate.
