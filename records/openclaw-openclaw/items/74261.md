## Summary

Keep open. Current main still does not expose a WhatsApp group-creation agent tool, message action, active-listener method, runtime helper, or config gate. Baileys does expose `groupCreate(subject, participants)`, so the request is technically plausible, but making it AI-callable would create groups and invite phone-number participants through the linked WhatsApp identity. That is product- and security-sensitive, so this needs maintainer review rather than an automated close or repair PR.

## Best Possible Solution

Keep this open for maintainer product/security review. If accepted, implement the capability in the WhatsApp-owned plugin, or first add a narrow documented WhatsApp runtime/API seam that lets a trusted plugin use the linked session safely. The implementation should normalize participants to WhatsApp JIDs, call Baileys `groupCreate`, be owner-only or otherwise explicitly gated, redact identifiers in logs, record outbound activity, document the security boundary, and keep the unrelated `responsePrefix` discussion separate.
