## Summary

Keep open. Current main still lacks a WhatsApp group-creation agent tool or runtime wrapper, and the requested behavior would let an agent create groups and invite phone-number participants through a linked WhatsApp identity, so it needs maintainer product/security review rather than auto-close or an automated repair PR.

## What This Changes

Add a `whatsapp_create_group` agent tool that creates a WhatsApp group with a name and participant phone numbers for the reporter's property-listing workflow.

## Best Possible Solution

If maintainers accept the product/security tradeoff, implement the capability in the WhatsApp-owned plugin or first add a narrow documented WhatsApp runtime/API seam. The accepted design should owner-gate or approval-gate the tool, normalize phone numbers to WhatsApp JIDs, call group creation through the active linked listener, redact identifiers in logs, record outbound activity, add focused WhatsApp tests, and update WhatsApp docs.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is high-confidence: inspect WhatsApp agent-tool registration, action discovery, active-listener types, and send API, none of which exposes group creation.

## Solution Assessment

Unclear until maintainer product/security review. A direct ungated agent tool is not the best solution; the safer path is a WhatsApp-owned implementation with explicit gating, participant normalization, redacted logging, and tests, or a documented runtime seam if this must be externalized.
