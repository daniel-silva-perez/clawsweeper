## Summary

Keep open. Current main still lacks a WhatsApp group-creation agent tool, message action, runtime helper, or active-listener method. The requested capability is plausible but agent-callable group creation and participant invites are privacy/security-sensitive, so this needs maintainer product/security review rather than an automated close or repair PR.

## What This Changes

Add a WhatsApp agent tool named `whatsapp_create_group` that accepts a group name and participant phone numbers and creates a WhatsApp group for listing workflows.

## Best Possible Solution

If maintainers accept the product/security tradeoff, implement the capability in the WhatsApp-owned plugin or add a narrow documented WhatsApp runtime/API seam first. The accepted design should gate the tool, normalize participants to WhatsApp JIDs, call the linked session through the active listener, redact identifiers in logs, record outbound activity, add focused WhatsApp tests, update docs, and keep the unrelated `responsePrefix` text from the report out of scope.
