## Summary

Keep open. Current main still does not expose a WhatsApp group-creation agent tool, message action, runtime listener method, or send API wrapper; the requested workflow is plausible, but inviting phone-number participants through a linked WhatsApp identity needs maintainer product/security approval before implementation.

## What This Changes

Request to add a `whatsapp_create_group` agent tool to the WhatsApp plugin that accepts a group name and participant phone numbers and creates a WhatsApp group through the linked session.

## Best Possible Solution

If maintainers accept the product/security tradeoff, implement this in the WhatsApp-owned plugin or first add a narrow documented WhatsApp runtime/API seam. The design should gate the tool to the owner or an equivalent trusted policy, normalize phone numbers to WhatsApp JIDs, call the active linked session, redact identifiers in logs, record outbound activity, add focused WhatsApp tests, and update user-facing WhatsApp docs.

## Reproduction Assessment

Not applicable as a feature request. The current-main check is high-confidence: exact searches for `whatsapp_create_group`/`groupCreate` are empty, and the inspected WhatsApp tool/action/listener/send surfaces expose login, send, poll, reaction, and composing only.

## Solution Assessment

Unclear until maintainer review. A WhatsApp-owned, gated tool or narrowly documented runtime seam is the maintainable direction; adding a broad core helper or ungated generic tool would not be the best way to solve this.
