## Summary

Keep open. Current main still has no WhatsApp group-creation agent tool, message action, active-listener method, or send/runtime API, and this should not be closed as ClawHub-only because external plugin surfaces do not expose a way to create groups through the existing active WhatsApp session. The requested workflow is plausible, but inviting phone-number participants through the operator's linked WhatsApp identity needs maintainer product/security approval first.

## What This Changes

The issue requests a WhatsApp-owned `whatsapp_create_group` agent tool that creates a group from a group name and participant phone numbers via the linked WhatsApp session.

## Best Possible Solution

If maintainers approve the product/security tradeoff, implement this in the WhatsApp plugin as a gated owner/trusted-requester capability: normalize participant numbers to WhatsApp JIDs, add an active-listener/send API method that calls the active linked session, redact identifiers in logs, record outbound activity, add focused WhatsApp tests, and update WhatsApp docs. If maintainers reject agent-created group invites, document that limitation explicitly.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification is high-confidence: exact searches found no group-create symbol, and inspected WhatsApp tool, action, listener, send, runtime, and CLI docs expose login/send/poll/reaction/typing only.

## Solution Assessment

Unclear until maintainer product/security review. A WhatsApp-owned, gated tool or narrow additive runtime seam would be the maintainable path; a broad core helper or ClawHub-only close would not address access to the existing linked WhatsApp session.
