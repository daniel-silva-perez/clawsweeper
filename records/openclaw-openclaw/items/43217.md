## Summary

Keep open. Current main still does not implement this PR's intended resolved-model `model.reasoning` gate for the shared Google thinking payload sanitizer. The useful fix needs rebasing onto the current Plugin SDK shared helper and should align Gemini 2.5 fixed-level behavior with the native Google transport's budget mapping.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Rebase or rework it onto `src/plugin-sdk/provider-stream-shared.ts`, thread `ctx.model?.reasoning` or an equivalent resolved capability into the Google thinking sanitizer, preserve current Gemini 3, Gemma 4, and adaptive Gemini 2.5 behavior, align fixed Gemini 2.5 handling with the native Google transport's budget mapping, and add focused tests for `gemini-2.5-flash` with `reasoning: true` plus a non-reasoning negative case.
