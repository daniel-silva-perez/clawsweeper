## Summary

Keep open. Current main still does not implement the PR's intended resolved-model reasoning gate for the shared Google thinking sanitizer, although the old patch needs adaptation because the helper moved and native Google transport now uses a different Gemini 2.5 budget mapping.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Rebase/adapt the fix onto `src/plugin-sdk/provider-stream-shared.ts`, thread the resolved model capability from the existing provider wrapper context where appropriate, preserve current Gemini 3/Gemma/adaptive handling, align Gemini 2.5 fixed-level behavior with the native Google transport, and add focused tests for fixed-level `gemini-2.5-flash` with `reasoning: true` plus a non-reasoning negative case. Coordinate with #73416 only for the adjacent Google API guard widening; it does not replace this reasoning-capability gate.
