## Summary

Keep open. Current main still has an unbounded provider-auth wait before `status --usage --json` can write JSON, and the latest tagged release has the same path. The linked fix PR #74185 is open and unmerged, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for `openclaw status --usage --json` to reliably return machine-readable JSON from non-TTY subprocesses instead of hanging or exiting without output.

## Best Possible Solution

Land a narrow fix that bounds provider usage auth resolution before JSON status output waits on it, returns parseable degraded JSON when auth resolution cannot complete, and adds a regression test where auth resolution never settles.

## Reproduction Assessment

Yes. The issue includes a concrete Node `spawnSync('openclaw', ['status','--usage','--json'])` reproduction, and current main still has a source path where JSON output waits on unbounded provider auth resolution.

## Solution Assessment

Yes for triage: keeping this issue open while #74185 or an equivalent narrow fix is reviewed is the right path. The implementation should put the timeout around auth resolution itself, not only around later provider usage fetches.
