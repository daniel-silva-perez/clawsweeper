## Summary

Keep open. Current main still waits for provider usage auth resolution before `status --usage --json` can emit JSON, and that auth path reaches OpenAI Codex OAuth hooks without a timeout boundary. This issue is also paired with open fix PR #74185, so it should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that `openclaw status --usage --json` hangs or exits without output from a non-TTY subprocess after the 2026.4.26 release.

## Best Possible Solution

Land PR #74185 or an equivalent narrow fix that bounds provider usage auth resolution before status JSON waits on it, returns parseable degraded JSON when auth resolution times out, and adds a regression test where auth resolution never settles.

## Reproduction Assessment

Yes. The issue includes a concrete non-TTY `spawnSync('openclaw', ['status','--usage','--json'])` reproduction, and current main still has a source path where JSON output waits on unbounded provider auth resolution before writing stdout.

## Solution Assessment

Yes for triage. Keeping this issue open while PR #74185 or an equivalent bounded-auth fix is reviewed is the narrowest maintainable path; closing before the referenced PR merges would lose the canonical bug report.
