## Summary

Keep open. Current main still awaits provider usage auth resolution before writing `status --usage --json` output, and the linked open PR #74185 explicitly tracks this fix, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that `openclaw status --usage --json` hangs or exits without stdout/stderr from non-TTY subprocess automation after upgrading to OpenClaw 2026.4.26.

## Best Possible Solution

Keep this issue open and review/land PR #74185 or an equivalent narrow fix. The fix should bound provider usage auth resolution before JSON output waits on it, return parseable degraded JSON instead of empty stdout or a hang, and add a regression test where usage auth resolution never settles.
