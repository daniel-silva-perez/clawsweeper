## Summary

Keep open. Current main still awaits provider usage auth resolution before writing JSON output, and the linked open PR #74185 is the implementation candidate that should be reviewed or replaced before this issue closes.

## What This Changes

Ensure `openclaw status --usage --json` returns machine-readable JSON from non-TTY subprocesses instead of hanging or failing when provider usage auth resolution stalls.

## Best Possible Solution

Keep the issue open until PR #74185, or an equivalent narrow fix, lands with bounded provider auth resolution and a regression test for a never-settling auth resolver.
