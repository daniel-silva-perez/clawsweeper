## Summary

Keep this issue open. Current main still preserves OpenAI-compatible `image_url` inputs as `existingImages` and returns them unsanitized when the prompt text has no image references, and the streaming path can still finalize on lifecycle error before the async catch writes error content. There is also an open same-author implementation candidate, #62070, whose body uses closing syntax for this issue, so the right maintainer path is to review, rebase, land, or close that PR rather than close this issue now.

## Best Possible Solution

Keep this issue open. Review, rebase, and either land #62070 or replace it with an equivalent fix that sanitizes `existingImages` on the no-prompt-ref path and surfaces streaming lifecycle errors before `[DONE]`, with regression tests for both paths. After that lands on main, close this issue as implemented.
