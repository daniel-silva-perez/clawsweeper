## Summary

Keep open. Current main still has the reported MCP bridge retention pattern, and the same-author closing PR #71648 is open rather than merged, so this issue should remain the canonical tracker until that PR or an equivalent fix lands.

## What This Changes

The issue requests bounded lifetime and shutdown cleanup for `OpenClawChannelBridge`'s pending Claude permission and approval maps.

## Best Possible Solution

Land #71648 or an equivalent maintainer-approved MCP bridge patch that clears both pending maps on close, prevents post-close writes, expires stale Claude permission and approval entries, preserves the existing MCP protocol surface, and adds focused regression tests. If burst caps are intentionally out of scope, track that as a separate follow-up before treating the entire bounded-memory concern as complete.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: repeated unique Claude permission requests without matching `yes/no <id>` replies grow `pendingClaudePermissions`, approval requested events remain in `pendingApprovals` unless a matching resolved event arrives, and `close()` does not clear either Map.

## Solution Assessment

Yes for keeping this open. The narrow maintainable fix belongs in `src/mcp/channel-bridge.ts` with focused MCP tests, and the already-linked closing PR #71648 is the right implementation path to review, land, or replace rather than queueing a second repair job.
