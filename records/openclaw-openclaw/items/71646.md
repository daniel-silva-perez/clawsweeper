## Summary

Keep open. Current main still has the reported MCP channel bridge retention pattern, and the same-author closing PR #71648 is still open and unmerged, so this should wait for maintainer review or an equivalent landed fix.

## What This Changes

The issue asks to bound `OpenClawChannelBridge`'s `pendingClaudePermissions` and `pendingApprovals` maps with expiry, close-time cleanup, and a cap to avoid unbounded growth in long-running `openclaw mcp serve` processes.

## Best Possible Solution

Land #71648 or an equivalent maintainer-approved MCP bridge patch that clears both pending maps on close, prevents post-close writes, expires stale Claude permission and approval entries, preserves the MCP protocol surface, and adds focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: unique Claude permission requests without matching replies grow `pendingClaudePermissions`, approval requested events remain in `pendingApprovals` unless a matching resolved event arrives, and `close()` does not clear either map.

## Solution Assessment

Yes for keeping this issue open. The narrow maintainable path is to review, land, or replace #71648 rather than closing this issue before the linked implementation merges.
