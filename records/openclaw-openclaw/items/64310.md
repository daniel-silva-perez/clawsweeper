## Summary

Keep this PR open. Current main still has the mutable module-global workspacePackagePaths array, collectWorkspacePackagePaths still appends to it on each in-process main() call, and buildMissingPackages still reads the accumulated module state. The provided one-file PR remains relevant and has not been implemented or superseded on main.

## Best Possible Solution

Review and land a focused version of this PR instead of closing it. The maintainer path is to make collectWorkspacePackagePaths side-effect-free, pass the fresh deterministic package-path list into buildMissingPackages, address the redundant-sort review note, and add a small regression test in test/scripts/audit-seams.test.ts that calls main twice in-process or otherwise asserts missingPackages does not duplicate entries.
