Full review comments:

- **[P1] Remove the duplicate modelAllowlist key:** `src/plugins/provider-validation.test.ts:106`
  - body: This fixture declares `loadCatalog` twice in the same `modelAllowlist` object, and the expected fixture repeats the same duplicate a few lines later. TypeScript treats duplicate object-literal properties as an error/noisy invalid fixture, so drop the repeated keys before relying on this test to prove the normalization behavior.
  - confidence: 0.94
- **[P2] Record completion before tail-dispatch returns:** `src/auto-reply/reply/dispatch-from-config.ts:1097`
  - body: The new telemetry starts dispatch tracking before `replyResolver`, but the `AcpDispatchTailAfterReset` path can return when the tail hook handles the request without calling `recordAgentDispatchCompleted`. Successful reset-tail dispatches will then accumulate `started > completed` and make `status` report a false delivery gap, so complete or skip this telemetry before that early return.
  - confidence: 0.84
