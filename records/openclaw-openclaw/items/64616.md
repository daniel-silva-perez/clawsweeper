## Summary

Keep this PR open. Current main still keeps ACP tasks alive whenever ACP session metadata has any entry, so the PR’s intended zombie-session behavior is not implemented on main; the PR is a focused candidate but should address the remaining transcript-existence concern before landing.

## Best Possible Solution

Keep the PR open as the focused implementation candidate. Rebase onto current main, decide whether ACP zombie detection must validate transcript file existence/readability instead of only missing `sessionFile`, add or adjust regression coverage for that case, then run the targeted task-registry tests and the appropriate changed gate before landing.
