## Summary

Keep open. Current main still has the reported QMD scope failure: spawned subagent keys are created as `agent:<id>:subagent:<uuid>`, QMD scope normalization strips the `agent:<id>:` prefix, then returns `undefined` for `subagent:*`, so default-deny rules with `chatType` matches do not apply. The related default-scope PR did not resolve subagent classification.

## What This Changes

The issue requests that QMD memory search classify spawned subagent sessions correctly instead of resolving their channel/chatType as unknown and denying them under default-deny scope rules.

## Best Possible Solution

Keep this open and fix subagent QMD scope semantics so spawned sessions inherit an intentional requester/session classification, fail closed when that classification is unavailable, and have regression tests for default scope behavior and explicit rawKeyPrefix overrides.
