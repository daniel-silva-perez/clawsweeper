## Summary

Keep open. Current main still converts iMessage Markdown tables and strips internal directive tags, but it does not strip general Markdown before legacy iMessage text delivery; the broader fix remains represented by open PR #56283 rather than merged code.

## Best Possible Solution

Keep this issue open until #56283 or an equivalent main-branch implementation lands. The best fix is shared outbound Markdown stripping for non-Markdown-capable/plain-text delivery, applied after existing HTML/plain-text sanitization, table handling, and internal directive cleanup, with tests for direct iMessage sends, monitor replies, media captions, config override behavior, and empty-after-strip edge cases.
