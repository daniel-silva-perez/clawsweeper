## Summary

Keep open. Current main still handles iMessage Markdown tables and internal directive/scaffolding cleanup, but it does not strip general Markdown before legacy iMessage text delivery, and the broader implementation remains represented by open PR #56283 rather than merged code.

## Best Possible Solution

Keep this issue open until #56283 or an equivalent main-branch implementation lands. The best fix is a shared outbound Markdown stripping stage for non-Markdown-capable/plain-text delivery, applied after existing HTML/plain-text sanitization and internal directive cleanup, with tests for direct iMessage sends, monitor replies, media captions, config override behavior, Markdown-capable channel preservation, and empty-after-strip edge cases.
