## Summary

Keep open. Current main and the latest release still do not expose an agent-facing Feishu Bitable delete-record tool, and this PR is a coherent narrow implementation, but it needs maintainer choice because it adds irreversible data deletion and overlaps with the broader open #41368 and tracker #55654.

## What This Changes

The PR adds a Feishu Bitable `feishu_bitable_delete_record` helper, schema, and tool registration in `extensions/feishu/src/bitable.ts` that calls the Lark record-delete API and returns deletion metadata.

## Best Possible Solution

Land one Feishu-owned implementation that exposes single-record deletion through the existing Bitable registration and account-routing helper, keeps explicit irreversible-action wording, adds focused tool coverage if maintainers want proof beyond CI, and resolves the overlap with #41368 and #55654 cleanly.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. The high-confidence verification path is static: current main and `v2026.4.27` lack the tool registration, while the PR patch adds it through the existing Feishu Bitable registration flow.

## Solution Assessment

Unclear whether this exact PR is the best final merge target because #41368 is broader and overlaps. The implementation itself is narrow and matches existing patterns, so the maintainable path is a maintainer decision between landing this PR or consolidating it into the broader Feishu deletion-tool PR.
