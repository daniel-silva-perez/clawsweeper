## Summary

Keep open. Current main still does not expose an agent-facing Feishu Bitable delete-record tool, and this PR is a plausible narrow implementation, but it needs maintainer review because it adds an irreversible deletion capability and overlaps with the broader open #41368 / tracker #55654 work.

The branch adds a Feishu Bitable `feishu_bitable_delete_record` tool in `extensions/feishu/src/bitable.ts` that calls `client.bitable.appTableRecord.delete`, validates the Lark response, and returns deletion metadata.

Land one Feishu-owned implementation that exposes the single-record delete tool through the existing Bitable account-routing helper, keeps the destructive-action warning, adds focused tool coverage if maintainers want proof beyond CI, and either closes or supersedes the overlapping broader PR cleanly.
