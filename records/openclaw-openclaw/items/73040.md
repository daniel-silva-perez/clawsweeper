## Summary

Keep open. The PR addresses a real adjacent system-event defect, but the proposed exact `(text, contextKey)` queue scan is still incomplete for the linked heartbeat-failover cascade because the current exec approval and node exec paths embed fresh IDs in the same fields the PR compares.

## What This Changes

The branch changes `src/infra/system-events.ts` to queue-scan duplicate `(text, contextKey)` system events, preserve `lastContextKey` on skipped or contextless enqueues, remove `lastText`, and add four regression tests.

## Best Possible Solution

Carry or derive a stable exec intent dedupe key from the approval or heartbeat-failover layer into the system-event path, while preserving context-specific events such as distinct channel reactions. The final patch should include a regression where the same command/session intent retries under different approval or run IDs.

## Reproduction Assessment

Yes. A focused code-level reproduction is to enqueue or forward the same exec command twice with different approval/run IDs; current main shows those IDs are embedded in both text and context fields, and the PR compares those exact fields.

## Solution Assessment

No. Queue-wide exact `(text, contextKey)` dedupe is useful for non-consecutive exact duplicates, but the reported cascade needs a stable intent-level key or normalization of ID-bearing fields before this is the narrowest complete fix.

## Review Findings
