## Summary

Keep open. Current main still lacks the `</arg_value>` suffix cleanup for exec/read arguments, and the linked bug remains open; the PR is useful source work but is unmergeable against current main and has blocking sanitizer-scope issues.

## What This Changes

The PR adds a helper to strip trailing `</arg_value>` artifacts from normalized tool parameters, applies it to exec commands, and adds unit tests for suffix stripping.

## Best Possible Solution

Rework the fix on current main by adding a narrowly scoped sanitizer at the active tool-argument boundary for command/path-like fields, intentionally covering read path and all relevant exec string params while preserving write/edit payload text, with regression and negative tests.

## Reproduction Assessment

Unclear for a live provider reproduction because this read-only review did not run Windows with DashScope/Qwen. Source inspection gives a high-confidence code path: a string containing `</arg_value>>` would currently pass unchanged through exec/read arguments, and the proposed helper would truncate legitimate write/edit text fields.

## Solution Assessment

No, this PR is not the best fix as-is. The safer path is a current-main rework scoped to path/command-like fields, plus deliberate exec coverage and negative tests proving content and edit text are not modified.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restrict suffix stripping to path-like fields:** `src/agents/pi-tools.params.ts:109`
  - body: `stripXmlArgValueSuffixFromRecord()` mutates every string parameter, so write/edit payloads such as `content`, `oldText`, or `newText` that legitimately end with `</arg_value>` would be silently truncated before execution. Scope this cleanup to the fields that represent command or path-like arguments.
  - confidence: 0.93
- **[P2] Sanitize the other exec string parameters:** `src/agents/bash-tools.exec.ts:230-232`
  - body: The exec handler strips only `params.command`, but the same provider artifact can corrupt `workdir`, `host`, `security`, `ask`, or `node`; those fields are consumed later for target selection, policy, cwd resolution, and node dispatch. Apply the sanitizer consistently or document why each field is excluded.
  - confidence: 0.86
