## Summary

Keep open. Current main now satisfies most of the request: WebChat advertises audio uploads, serializes audio as file attachments, persists non-image uploads, routes offloaded non-image media through MediaPaths/MediaTypes for the active turn, and renders assistant audio attachments with HTML audio controls. The remaining concrete gap is audio/webm: the picker advertises audio/*, but the selected-file/drop helper still rejects .webm filenames, and the UI/shared MIME maps do not clearly classify .webm as audio.

## Best Possible Solution

Keep this issue open but narrow it. Preserve the shipped WebChat audio upload/playback and MediaPaths routing work, then finish the explicit audio/webm part with a small, tested MIME/classification change: allow audio/webm through selection/drop without accepting video/webm, classify audio/webm as audio for WebChat rendering/history, and add focused regression tests for picker support plus Gateway media-field routing.
