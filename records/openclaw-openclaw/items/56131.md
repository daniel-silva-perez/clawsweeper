## Summary

Keep this issue open. Current main still archives reset transcripts as `.jsonl.reset.*`, but the supported history surfaces continue to read only the current resolved transcript candidates and do not expose reset/archive chain discovery, `includeArchived`/`followResets`, archive metadata, or post-reset provenance. The related PR #60409 is an open partial candidate, not a merged implementation.

## Best Possible Solution

Keep this issue open. Implement a gateway-owned reset-archive discovery helper used by `chat.history`, `sessions_history`, sessions list/preview hydration, and the HTTP history surface where appropriate. The fix should expose archive/reset metadata, cover active primary plus archived prior fragments, multiple archives, missing primary fallback, truncation metadata, and post-reset system-event provenance. Review or expand #60409 if it remains the active candidate, but close only after a merged implementation covers the logical session-chain recovery described here.
