## Summary

Keep this issue open. Current main still creates `.jsonl.reset.*` archives on reset, but the supported `chat.history`, `sessions_history`, session preview, and session list paths read only resolved current transcript candidates and do not expose archive discovery, archive-chain metadata, `includeArchived`/`followResets`, or cross-reset event provenance. The related PR #60409 is still an open partial candidate per the provided GitHub context, not a merged implementation.

## Best Possible Solution

Keep this issue open. Implement a gateway-owned reset archive discovery and logical session-chain recovery path used consistently by `chat.history`, `sessions_history`, `sessions.list`/preview hydration, and the HTTP history surface. It should expose archive/reset metadata, support active primary plus archived prior fragments and multiple archives, handle missing-primary fallback, preserve truncation metadata, and add enough event provenance for post-reset exec completion messages to be tied back to the originating work context. Review #60409 as a partial candidate, but close this only after a merged implementation covers the broader recovery contract.
