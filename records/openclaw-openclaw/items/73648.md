## Summary

Current main still has the stop-timeout zombie-task failure mode that this PR targets, and the linked channel restart issue remains open. This PR is an active implementation candidate, not obsolete or implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review as the active gateway fix for #71412. The best path is to land this PR or an equivalent core fix that releases stale task/abort entries on stop timeout, prevents stale task callbacks from overwriting fresh runtime state, preserves startup instrumentation, and keeps regression coverage for restart-after-timeout and stale settlement behavior.
