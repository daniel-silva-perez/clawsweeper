## Summary

Current main still has the channel stop-timeout zombie-task behavior this PR targets, and the PR remains an active implementation candidate for open issue #71412 rather than obsolete or already implemented. The security review pass does not show supply-chain or secret-handling changes, but the low-severity concurrent old/new task concern remains a maintainer decision before merge.

## Best Possible Solution

Keep this PR open for maintainer review as the active core gateway fix for #71412. The likely path is to land this PR or an equivalent gateway change that clears stale task/abort entries on stop timeout, marks the runtime stopped, prevents stale task callbacks from overwriting fresh runtime state, preserves startup instrumentation, and keeps regression coverage for both restart-after-timeout and stale-settlement behavior.
