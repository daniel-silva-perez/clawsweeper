## Summary

Keep this PR open. Current main still has the stop-timeout zombie-task behavior it targets, #71412 remains open, and this PR is the active focused gateway implementation candidate rather than obsolete or already implemented.

## Best Possible Solution

Keep this PR open for maintainer review as the active core gateway fix for #71412. The best path is to land this PR or an equivalent gateway change that releases stale task/abort entries on stop timeout, marks the runtime stopped, ignores stale lifecycle/status writes from replaced tasks, preserves startup instrumentation, and keeps the regression coverage; track stronger plugin-level I/O cancellation separately if maintainers want to resolve Aisle's residual concurrency concern.
