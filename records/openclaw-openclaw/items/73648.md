## Summary

Current main still contains the reported stop-timeout failure mode, and this PR is an active implementation candidate for the still-open channel restart bug tracked in #71412. It should not be closed as obsolete or implemented-on-main.

## Best Possible Solution

Keep this PR open and review it as the active gateway fix for #71412. The likely maintainer path is to land this PR or an equivalent fix that releases stale task/abort entries on timeout, prevents stale task callbacks from overwriting the fresh runtime, preserves the startup instrumentation path, and keeps the added regression coverage for both fresh restart and stale-settlement behavior.
