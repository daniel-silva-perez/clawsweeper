## Summary

Keep this issue open. Current main still implements gateway LAN/BYOH binding as IPv4-only: the bind mode enum/schema have no IPv6 or dual-stack mode, `lan` resolves to `0.0.0.0`, `custom` rejects non-IPv4 hosts, and listener host expansion only adds `::1` for loopback. The provided comment asking for `::` or a specific IPv6 address remains actionable core gateway work, not a duplicate of the adjacent Docker/LAN auth reports.

## Best Possible Solution

Implement first-class core IPv6 or dual-stack gateway binding. The likely path is to decide the public contract, either allowing IPv6 `customBindHost` or adding a clear dual-stack/LAN mode, then update `resolveGatewayBindHost`, `resolveGatewayListenHosts`, runtime validation, Control UI link/status/probe handling, tests for `::` and specific IPv6 addresses, and gateway security/config docs. A docs-only clarification such as related PR #71594 can help in the interim, but it does not close the requested capability.
