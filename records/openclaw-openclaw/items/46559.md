## Summary

Keep open and do not merge as-is. The current main branch still has the verbose-off session/channel leak, but this PR is incomplete: it guards only part of the assistant text path, leaves final chat payloads exposed, makes verbose decisions stale, and carries unrelated channel/SDK churn that should not be bundled into the privacy fix.

## What This Changes

The PR gates some assistant chat delta and buffered flush session sends by verbose level, adds gateway regression tests, and includes unrelated plugin SDK, channel, and docs changes from branch history.

## Best Possible Solution

Land a focused gateway fix that suppresses assistant text to session/channel subscribers whenever the effective verbose level is off across delta, buffered flush, final done, and final error paths, while preserving WebSocket/control UI broadcasts and leaving plugin SDK/channel changes for separate review.

## Reproduction Assessment

Yes. The gateway event handler harness can register a verbose-off run, emit assistant text plus tool/lifecycle events, and observe nodeSendToSession receiving chat payloads on current main; the existing verbose-off flush test already asserts those session sends.

## Solution Assessment

No. The PR is useful direction, but it is not the best mergeable fix because it misses final payloads, caches verbose decisions across in-flight setting updates, uses the wrong fallback for unresolved verbose, and bundles unrelated channel/API churn.

  - body: The patch suppresses some delta and flush session sends, but emitChatFinal still sends final done/error chat payloads to the session unconditionally. With verbose off, subscribers can still receive the complete buffered assistant text or error payload at run end, so the leak remains.
  - confidence: 0.96

  - confidence: 0.9

  - body: The new cache returns the first resolved verbose value for the whole run. Runtime /verbose changes are documented to affect in-flight output, so later assistant/tool events can keep using a stale mode and leak or suppress text incorrectly.
  - confidence: 0.87
