## Summary

Keep open. Current main still sends assistant chat deltas, flushed deltas, final payloads, and error payloads to session/channel subscribers without a verbose-off guard, so the PR is not obsolete. The branch is also not merge-ready because it only partially gates the leak, caches verbose decisions in a way that can go stale, and mixes unrelated SDK/channel changes into a gateway privacy fix.

## What This Changes

The PR gates some assistant delta and buffered-delta session sends by verbose level, adds gateway regression tests, introduces a per-run verbose cache, and includes unrelated bundled channel, SDK barrel, and docs changes.

## Best Possible Solution

Land a focused gateway fix that resolves verbose level at each assistant/session delivery decision, suppresses assistant text delivery to session/channel subscribers for delta, flushed delta, final done, and error chat paths when verbose is off, preserves websocket/control UI broadcasts, and leaves SDK/channel refactors to separate owner-reviewed work. Keep #25592 as the broader tracking issue until that narrow fix ships.
