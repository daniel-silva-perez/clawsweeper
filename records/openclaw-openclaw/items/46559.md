## Summary

Keep open. The gateway privacy bug is still present on current main, and this PR is useful direction but is not mergeable as-is because it misses final/default verbose paths, makes verbose decisions stale for an in-flight run, and carries unrelated stale plugin SDK/channel churn.

## What This Changes

This PR gates some assistant chat delta and buffered flush session sends by verbose level, adds gateway regression tests, and also changes plugin SDK, channel plugin, and docs files from branch history.

## Best Possible Solution

Land a focused gateway fix that suppresses assistant text to session/channel subscribers whenever the effective verbose level is off across delta, buffered flush, final done, and final error paths, while preserving WebSocket/control UI broadcasts and leaving plugin SDK/channel changes for separate review.

## Reproduction Assessment

Yes. The gateway event-handler harness can register a verbose-off run, emit assistant text plus tool/lifecycle events, and observe nodeSendToSession receiving chat payloads on current main; the existing verbose-off flush test already asserts those session sends.

## Solution Assessment

No. The PR is directionally useful, but it is not the best mergeable fix because it misses final payloads, treats unresolved verbose as streamable despite the off default, caches verbose decisions across in-flight updates, and bundles unrelated stale channel/API churn.
