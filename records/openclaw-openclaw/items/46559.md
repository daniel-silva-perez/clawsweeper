## Summary

Keep open. Current main still has the verbose-off session/channel leak, and this PR is useful source work but not mergeable as-is because it misses final payloads/default verbose behavior, caches verbose state across in-flight changes, and carries unrelated stale plugin/channel changes.

## What This Changes

The PR gates some gateway assistant delta and buffered-flush session sends by verbose level, adds gateway regression tests, and also changes plugin SDK, channel plugin, and docs files from branch history.

## Best Possible Solution

Land a focused gateway fix that suppresses assistant text to session/channel subscribers whenever effective verbose is off across delta, buffered flush, final done, and final error paths, while preserving WebSocket/control UI broadcasts and leaving plugin SDK/channel changes for separate review.

## Reproduction Assessment

Yes. The gateway event-handler harness can register a verbose-off run, emit assistant text plus tool/lifecycle events, and observe `nodeSendToSession` receiving chat payloads on current main; the existing verbose-off flush test already asserts those session sends.

## Solution Assessment

No. The PR is directionally useful, but it is not the best mergeable fix because it misses final payloads, treats unresolved verbose as streamable despite the off default, caches verbose decisions across in-flight updates, and bundles unrelated stale channel/API churn.

  - confidence: 0.88
