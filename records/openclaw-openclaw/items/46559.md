## Summary

Keep open. Current main still has the verbose-off chat leak, but this PR is not mergeable as-is because it only guards assistant delta/flush sends and leaves final done/error chat payloads plus default/in-flight verbose semantics unsafe.

## What This Changes

The PR gates assistant chat delta and buffered-delta session sends by verbose level, adds gateway regression tests, and also carries unrelated plugin SDK/channel/doc changes from branch history.

## Best Possible Solution

Land a focused gateway fix that suppresses assistant text to session/channel subscribers whenever the effective verbose level is off across delta, buffered flush, final done, and final error paths, while preserving WebSocket/control UI broadcasts. The fix should resolve missing verbose as off, avoid stale per-run verbose decisions when session settings change, and keep unrelated plugin SDK/channel churn out of the privacy patch.

## Reproduction Assessment

Yes. The gateway event harness can register a verbose-off run, emit assistant text and lifecycle/tool flush events, and observe nodeSendToSession receiving chat payloads; current main's existing verbose-off flush test still expects those session sends.

## Solution Assessment

No. The PR is useful direction, but it is not the best mergeable fix because it guards only delta/flush paths, defaults unresolved verbose to streamable, caches verbose decisions across runtime updates, and carries unrelated SDK/channel changes.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Gate final chat payloads by verbose level:** `src/gateway/server-chat.ts:495-507`
  - body: The patch suppresses delta and flush session sends, but emitChatFinal still sends final done/error payloads to the session unconditionally. With verbose off, subscribers can still receive the complete buffered assistant text or error payload at run end, so the leak remains.
  - confidence: 0.96
- **[P1] Default unresolved verbose to off:** `src/gateway/server-chat.ts:543-546`
  - body: shouldStreamAssistantTextToSession returns true when resolveVerboseLevel is undefined. Since verbose off is the documented default, sessions that inherit the default mode still stream assistant text unless they have an explicit off entry.
  - confidence: 0.9
- **[P1] Re-read verbose after session updates:** `src/gateway/server-chat.ts:510-513`
  - body: The new cache returns the first resolved verbose value for the whole run. Runtime /verbose changes are documented to affect in-flight output, so later assistant/tool events can keep using a stale mode and leak or suppress text incorrectly.
  - confidence: 0.87
