## Summary

Keep open. The PR targets a real current-main gateway leak, but the proposed patch is still incomplete: it suppresses some assistant deltas while leaving final chat payloads, inherited/default verbose behavior, and runtime verbose changes able to leak or misroute session/channel text.

## What This Changes

The PR gates gateway assistant delta and buffered-delta session sends by verbose level, adds gateway regression tests, and carries additional plugin SDK/channel/docs changes from the branch history.

## Best Possible Solution

Land a focused gateway fix that suppresses assistant text to session/channel subscribers whenever the effective verbose level is off across delta, buffered flush, final done, and final error paths, while preserving WebSocket/control UI broadcasts. Resolve missing verbose as off by default, avoid stale per-run verbose decisions or invalidate them when session/run settings change, add regression coverage for final payloads and inherited defaults, and split out unrelated plugin/channel/docs churn.

## Reproduction Assessment

Yes. The current-main handler path is reproducible with the existing gateway event harness: register a verbose-off run, emit assistant text and lifecycle end/tool flush, and observe nodeSendToSession receiving chat payloads; current tests still assert verbose-off flush delivery.

## Solution Assessment

No. The PR is useful direction, but it is not the best mergeable fix because it only gates delta/flush sends, defaults unresolved verbose to streamable, and caches verbose decisions for the whole run despite documented live /verbose changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Gate final chat payloads by verbose level:** `src/gateway/server-chat.ts:496`
  - body: The patch suppresses delta and flush session sends, but emitChatFinal still sends final done/error payloads to the session unconditionally. With verbose off, subscribers can still receive the complete buffered assistant text or error payload at run end, so the leak remains.
  - confidence: 0.96
- **[P1] Default missing verbose to off:** `src/gateway/server-chat.ts:543-548`
  - body: shouldStreamAssistantTextToSession returns true when resolveVerboseLevel is undefined. Since verbose off is the documented default, sessions that inherit the default mode still stream assistant text unless they have an explicit off entry.
  - confidence: 0.9
- **[P1] Re-read verbose after session updates:** `src/gateway/server-chat.ts:510-513`
  - body: The new cache returns the first resolved verbose value for the whole run. Runtime /verbose changes are documented to affect in-flight output, so later assistant/tool events can keep using a stale mode and leak or suppress text incorrectly.
  - confidence: 0.87
