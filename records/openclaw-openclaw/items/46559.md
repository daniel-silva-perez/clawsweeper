## Summary

Keep open. Current main still sends assistant chat delta, buffered flush, final done, and final error payloads to session/channel subscribers without a verbose gate, and the PR head only fixes the delta/flush part while leaving final payloads and default/in-flight verbose behavior wrong.

## What This Changes

The PR gates gateway assistant delta and buffered-delta session sends by verbose level, adds gateway regression tests, and also carries unrelated plugin SDK/channel/docs adjustments.

## Best Possible Solution

Land a focused gateway fix that suppresses session/channel assistant text when the effective verbose level is off across delta, buffered flush, final done, and final error chat paths, while preserving WebSocket/control UI broadcasts. Resolve missing verbose as off by default, avoid stale per-run cache behavior or invalidate it when session settings change, add regression coverage for final payloads and inherited defaults, split out unrelated SDK/channel refactors, and leave #25592 open for the broader channel-delivery policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Gate final chat payloads by verbose level:** `src/gateway/server-chat.ts:496`
  - body: The patch suppresses delta and flush session sends, but emitChatFinal still sends final done/error payloads to the session unconditionally. With verbose off, subscribers can still receive the complete buffered assistant text or error payload at run end, so the leak remains.
  - confidence: 0.96
- **[P1] Treat missing verbose config as off for session streaming:** `src/gateway/server-chat.ts:543-548`
  - body: shouldStreamAssistantTextToSession returns true when resolveVerboseLevel is undefined. Since verbose off is the default, sessions that inherit the default mode still stream assistant text unless they have an explicit off entry.
  - confidence: 0.9
- **[P1] Re-read verbose after session changes:** `src/gateway/server-chat.ts:510-513`
  - body: The new cache returns the first resolved verbose value for the whole run. Runtime /verbose changes are documented to affect in-flight tool bubbles, so later assistant/tool events can keep using a stale mode and leak or suppress text incorrectly.
  - confidence: 0.87
