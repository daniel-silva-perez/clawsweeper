## Summary

Keep open. Current main still lacks the requested managed child-task TaskFlow UX, and this PR contains useful work, including a follow-up that addressed the earlier ACP registration failure review. The latest head still has blocking lifecycle bugs in retry, cancellation, and child matching, plus smaller UI/docs issues, so it should not merge or be closed as implemented.

## What This Changes

The PR adds managed child TaskFlow creation from `sessions_spawn`, child-completion synchronization, progress/guidance delivery, retry operations, CLI/gateway/runtime/UI controls, and related docs/tests.

## Best Possible Solution

Revise the branch into a mergeable TaskFlow v1 control-surface slice: keep the useful `sessions_spawn(taskFlow=...)` and retry concepts, but reopen flows before child linking, give cancellation precedence, require precise wait identifiers, forward requester origin for subagent retries, derive guidance from retry eligibility, make UI flow loading session-token safe, and remove localized docs edits from this repo.

## Reproduction Assessment

Yes. A focused static reproduction path is clear: create a failed or lost managed child flow with stored launch metadata and call retry; the PR passes `parentFlowId` into spawn before reopening, while the linking guard rejects terminal parent flows. The cancellation, identifier-matching, and stale UI cases also have small deterministic unit-test paths.

## Solution Assessment

No. The direction is plausible, but the current patch is not the safest implementation because several lifecycle operations happen in the wrong order or use weaker matching than the existing flow contracts require. The safer path is to repair those invariants first, then run targeted task/runtime/CLI/UI tests plus the normal changed gate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reopen terminal flows before retry child spawns:** `src/tasks/task-executor.ts:569`
  - body: `resolveRetryableManagedChildTaskFlow` allows retries from `failed` and `lost`, but this path passes `parentFlowId` into the child spawn while the parent flow is still terminal. Task creation rejects terminal parent flows, so subagent retries fail before relaunch and ACP retries can start detached from the flow. Move the flow back to a nonterminal waiting/retrying state before spawning/linking the child.
  - confidence: 0.92
- **[P1] Give cancellation precedence over completion sync:** `src/tasks/task-registry.ts:1121`
  - body: `updateTask` calls `syncManagedFlowCompletionFromTask` before `syncManagedFlowCancellationFromTask`, and completion sync does not skip flows with `cancelRequestedAt`. If the user cancels a managed flow and the child later exits, the flow can be finalized as succeeded/failed/lost before cancellation gets a chance to apply.
  - confidence: 0.9
- **[P2] Require wait identifiers when matching child completions:** `src/tasks/task-registry.ts:815-825`
  - body: When `waitJson` names a specific `runId` or `childSessionKey`, a linked terminal task that omits that field still matches because mismatches are only rejected when both sides provide the identifier. Require any identifier present in the wait state to be present and equal on the task before closing the flow.
  - confidence: 0.88
- **[P2] Pass requester origin into subagent retry spawns:** `src/tasks/task-executor.ts:572-574`
  - body: Subagent retry calls `spawnSubagentDirect` with only `agentSessionKey`, so retries for stored launches with `thread: true` lose the original channel/thread target used to bind the child session. Forward `flow.requesterOrigin` here as the ACP retry path already does.
  - confidence: 0.84
- **[P2] Base retry guidance on actual retry eligibility:** `src/tasks/task-flow-guidance.ts:41-47`
  - body: `getTaskFlowGuidance` reports `retry_available` for every failed or lost managed flow, but the retry resolver can reject missing launch metadata, non-retryable launches, or active child tasks. Derive guidance from the same eligibility check so CLI/UI text does not advertise an unavailable retry.
  - confidence: 0.9
- **[P2] Discard stale flow loads after session switches:** `ui/src/ui/controllers/task-flows.ts:23-33`
  - body: `loadLatestTaskFlow` returns early while a load is in flight and writes the result after awaiting using mutable state. If the user switches sessions during the request, an old response can populate the new chat with the previous session's flow card until another refresh happens.
  - confidence: 0.87
- **[P3] Remove localized docs edits from this repo:** `docs/zh-CN/tools/subagents.md:65-71`
  - body: `docs/AGENTS.md` says foreign-language docs under `docs/<locale>/**` are not maintained here. This branch edits `docs/zh-CN`, which can leave generated translation output out of sync; keep the English docs as source and let the docs publish pipeline regenerate localized content.
  - confidence: 0.86
