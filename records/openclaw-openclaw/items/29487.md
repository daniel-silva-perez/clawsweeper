## Summary

Keep this issue open. Current main carries Feishu parent-id quoted context and has the earlier fetched-message parsing improvements, but it still does not model, parse, or prefer the UI quote identifier called out in the discussion (`quote.message_id` or equivalent), so quote-only Feishu events can still reach the agent without the quoted content.

## Best Possible Solution

Implement the missing quote-id path inside the bundled Feishu plugin: verify the Feishu UI quote payload shape, extend the event/context types with a separate quoted-message id, parse the verified `quote.message_id` equivalent, prefer that id when fetching quoted content while retaining `parent_id` fallback, and add regression tests for quote-only UI replies plus existing parent-id replies and group visibility behavior.
