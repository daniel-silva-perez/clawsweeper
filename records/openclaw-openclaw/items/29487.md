## Summary

Keep this issue open. Current main still fetches Feishu quoted/replied content only through `parent_id`; it does not model or prefer the separate UI quote id discussed in the issue comments, so quote-only Feishu events can still miss the quoted content. The earlier related work improved fetched-message parsing, but not inbound quote-id selection.

## Best Possible Solution

Implement the missing Feishu UI quote-id path inside the bundled Feishu plugin: verify the incoming quote payload shape, add a typed optional quote message id to the event/context model, parse it, prefer it when fetching quoted content while retaining `parent_id` fallback, and add regression tests for quote-only UI replies plus existing parent-id and group visibility behavior.
