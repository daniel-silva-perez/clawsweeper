## Summary

Keep this issue open. Current main still fetches Feishu quoted/replied content only through `parent_id`; it does not model or prefer the separate UI quote id discussed in the issue comments, so quote-only Feishu events can still miss quoted content. The related parser work from #29501 improved fetched-message parsing, but not inbound quote-id selection.

## Best Possible Solution

Implement the missing Feishu UI quote-id path inside the bundled Feishu plugin: verify the incoming quote payload shape, add a typed optional quote message id to the event/context model, parse it, prefer it when fetching quoted content while retaining `parent_id` fallback, and add regression tests for quote-only UI replies plus existing parent-id and group visibility behavior.

The issue discussion reports Feishu UI quote replies can provide a separate quote message id, likely `quote.message_id`, so quote-only events do not fetch/prepend the quoted content.

Expected boundary: stay inside `extensions/feishu` plus one user-facing changelog entry if behavior changes. Tests: add Feishu bot regression coverage for a quote-only UI reply id fetching and prepending quoted content; keep or extend coverage for `parent_id` fallback and group quote allowlist filtering.
