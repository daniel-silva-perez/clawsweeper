## Summary

Keep this issue open. Current main still fetches Feishu quoted/replied content only from `parent_id`; it does not model or parse the separate UI quote id described in the discussion, so quote-only Feishu events can still reach the agent without the quoted message content.

## Best Possible Solution

Implement the missing quote-id path inside the bundled Feishu plugin: verify the Feishu UI quote payload shape, add a separate quoted-message id to event/context typing, parse it, prefer that id when fetching quoted content while retaining `parent_id` fallback, and add regression tests for quote-only UI replies plus the existing parent-id and group visibility cases.
