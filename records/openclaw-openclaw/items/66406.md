## Summary

Keep open. Current main and the latest release still pass the user-provided Feishu reaction emoji directly into the Feishu reaction API, and the linked normalization PR remains open/unmerged with closing syntax for this issue.

## What This Changes

Feishu `message action="react"` should accept common Unicode emoji such as `👍` and convert them to Feishu reaction type strings before calling the Feishu reactions API.

## Best Possible Solution

Keep this issue open and review/land PR #66478, or an equivalent Feishu-owned fix, that normalizes Unicode emoji and accepted Feishu type strings before all Feishu add/remove/list reaction API calls. The merged fix should include a focused regression test proving `emoji="👍"` sends a Feishu type such as `THUMBSUP`.

## Reproduction Assessment

Yes. The issue gives concrete steps, and current-main source inspection shows the reported Unicode emoji is trimmed and sent unchanged into Feishu `reaction_type.emoji_type`; no live Feishu mutation was run during this read-only review.

## Solution Assessment

Yes, the normalization direction is the narrowest maintainable fix because the bug is Feishu-specific and the current plugin already has a single reaction helper boundary. The issue should close only after that implementation or an equivalent Feishu plugin fix merges.
