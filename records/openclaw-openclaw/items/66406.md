## Summary

Keep open. Current main still supports Feishu `react`/`reactions`, but the `react` path trims `ctx.params.emoji` and passes it directly as `emojiType` to the Feishu reactions API. The local source and docs agree Feishu expects an `emoji_type` such as `THUMBSUP`, and there is no normalization helper on current main. The provided GitHub context plus a read-only remote ref check show open PR #66478 is still the active implementation candidate with closing syntax, so this issue should not be closed before that PR is merged or rejected.

## Best Possible Solution

Review PR #66478 as the canonical implementation candidate, require a focused regression test showing `message action=react emoji="đź‘Ť"` is normalized to a Feishu `emoji_type` such as `THUMBSUP`, and close this issue only after the fix lands or after maintainers decide the PR should be rejected in favor of another concrete implementation.
