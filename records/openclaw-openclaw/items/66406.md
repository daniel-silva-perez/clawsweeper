## Summary

Keep open. Current main still trims the Feishu reaction emoji and forwards it unchanged into the Feishu reaction API, and open PR #66478 still has closing syntax for this issue but is open and unmerged, so this should not be closed before that PR or an equivalent Feishu plugin fix lands.

## What This Changes

The issue reports that Feishu `message(action="react", emoji="👍")` returns Feishu error code 231001 because OpenClaw sends an invalid reaction type when reactions are enabled.

## Best Possible Solution

Keep this issue open while maintainers review, rebase, and land PR #66478 or an equivalent Feishu plugin fix that normalizes Unicode emoji to Feishu-supported `emoji_type` values at the Feishu reaction boundary and includes focused regression coverage for `emoji="👍"` sending a valid Feishu type such as `THUMBSUP`.

## Reproduction Assessment

Yes. The issue gives concrete steps, and current-main source inspection shows `emoji="👍"` is trimmed and passed unchanged into Feishu reaction create/list API parameters; no live Feishu write was attempted in this read-only review.

## Solution Assessment

Yes, the best direction is a narrow Feishu-owned normalization fix with tests, not a core channel change. The current open PR is the paired implementation candidate, so the issue should remain open until that PR or an equivalent fix merges.
