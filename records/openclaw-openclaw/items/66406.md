## Summary

Keep open. Current main still advertises Feishu reaction actions and passes the raw `message action=react` emoji string directly into the Feishu reactions API as `emoji_type`; no landed normalizer or regression test is present. Open PR #66478 is still the active implementation candidate with closing syntax, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Review and land PR #66478 or an equivalent Feishu-owned fix that normalizes unicode emoji and known Feishu type strings before add/remove/list reaction API calls, with focused tests proving `message action=react emoji="👍"` sends a Feishu type such as `THUMBSUP`. Close this issue only after that implementation merges, or after maintainers explicitly reject the PR and choose another concrete fix path.
