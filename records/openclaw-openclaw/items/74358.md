## Summary

Keep open. Current main still leaves the reporter's Slack object-form config without a guaranteed in-flight answer preview when `streaming.preview.toolProgress` is `false`, and the discussion identifies open PR #56544 as an adjacent flat-DM fix rather than a complete fix for this multi-tool silent-preview case.

## What This Changes

The issue asks Slack verbose/live preview streaming to keep showing in-flight answer text while hiding tool-progress lines for object-form `streaming.preview.toolProgress: false` config.

## Best Possible Solution

Slack should make `streaming.preview.toolProgress: false` consistently mean answer-preview-on and tool-lines-off across object-form config, native streaming, and draft-preview fallback, with regression coverage for the confirmed object-only config and docs/config-help text aligned to that contract.

## Reproduction Assessment

Yes, at source level: the reporter supplied a stable object-form Slack config, and current main still routes that config through partial/native defaults while omitting the draft partial-text callback on the native path and leaving the disabled-progress path untested. A live Slack workspace run was not performed.

## Solution Assessment

Unclear. The issue body's suggested line-level coupling fix is not sufficient by itself; the maintainable solution needs to separate answer-preview delivery from tool-progress visibility and reconcile that with Slack native streaming and the open flat-DM PR.
