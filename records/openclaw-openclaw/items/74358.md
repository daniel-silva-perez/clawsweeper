## Summary

Keep open. Current main still has Slack streaming paths where `streaming.preview.toolProgress: false` only disables tool-progress preview lines and does not guarantee a draft answer preview for the reporter's object-form config, while the related flat-DM PR is explicitly described in discussion as a sibling rather than a complete fix.

## What This Changes

The issue asks Slack to keep live answer-preview updates when `streaming.preview.toolProgress: false` hides tool-progress lines, including object-form Slack streaming config without explicit `mode` or `nativeTransport`.

## Best Possible Solution

Slack should make `streaming.preview.toolProgress: false` mean answer-preview-on and tool-lines-off across the object-form config path, with regression tests for native and draft preview behavior and Slack docs/config help updated to match the shared streaming contract. PR #56544 should remain scoped to the flat-DM gate unless maintainers deliberately broaden it with this regression coverage.

## Reproduction Assessment

Yes, for a high-confidence source-level reproduction path: the reporter supplied a stable object-form config, and current main still defaults that config into Slack partial/native behavior while omitting the draft partial-text callback on the native path. A live Slack run was not performed.

## Solution Assessment

No, the issue body's initial line-coupling fix is not sufficient by itself. The maintainable solution needs to separate answer-preview delivery from tool-progress visibility and account for Slack native streaming versus draft preview fallback.
