## Summary

Keep open. Current main still has a source-backed Slack silent-preview path for the reporter's object-only config: it defaults to partial/native streaming, flat DM preview eligibility remains false without a thread, and partial-text draft updates are only wired when preview streaming is enabled and native streaming is not active. Open PR #56544 overlaps the flat-DM gate, so this needs maintainer reconciliation rather than cleanup closure.

## What This Changes

The issue asks Slack to keep answer-preview streaming active while hiding tool-progress preview lines when `streaming.preview.toolProgress` is `false`.

## Best Possible Solution

Slack should make `streaming.preview.toolProgress: false` mean answer-preview-on and tool-lines-off for the reporter's DM/object-config path, with regression tests for object-form config, flat DMs, native-transport fallback, and updated Slack docs/config help to match the shared streaming contract.

## Reproduction Assessment

Yes. The reporter confirmed a stable object-only Slack config repro, and current source tracing reproduces the wiring path without a live Slack run: defaults select partial/native streaming, flat DMs lack a thread target, preview streaming is disabled, and `onPartialReply` is not installed.

## Solution Assessment

Unclear as currently proposed. PR #56544 likely addresses the flat-DM preview gate, but this issue also needs explicit `toolProgress: false` regression coverage and docs/help alignment before maintainers can know the silent-preview regression is fully fixed.
