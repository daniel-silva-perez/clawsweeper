## Summary

Keep open. Current main still has a credible source-level path for the reported Slack silent-preview behavior, and the related flat-DM preview PR is still open rather than merged. The item should be reconciled with that PR and the Slack/shared streaming contract, not closed as fixed or stale.

## What This Changes

The issue asks Slack to keep in-flight answer preview streaming active while hiding tool-progress lines when `streaming.preview.toolProgress` is `false`.

## Best Possible Solution

Slack should treat object-form `streaming.preview.toolProgress: false` as answer-preview-on and tool-lines-hidden wherever Slack preview delivery is intended to work, including the flat-DM path if maintainers accept #56544's direction. The final change should align Slack docs/config help with the shared streaming contract and add regression coverage for object-only config, top-level DMs, nativeTransport defaults, and disabled in-preview tool progress.

## Reproduction Assessment

Yes for source-level reproduction. The reporter supplied the exact object-only config, and current main resolves it through defaults that leave top-level Slack DMs without preview eligibility or an `onPartialReply` draft-update callback; I did not run a live Slack socket-mode session.

## Solution Assessment

No for the issue body's narrow coupling-only suggested fix. The safer solution needs to address the flat-DM preview gate, native/draft callback split, and default tool-progress suppression semantics together, likely by updating or replacing the overlapping #56544 approach.
