## Summary

Keep open. Current main still leaves Slack's disabled tool-progress preview path coupled to default progress suppression, while the shared streaming contract expects answer previews to continue when only tool-progress lines are disabled. There is also no Slack regression coverage for this exact `toolProgress: false` path.

## What This Changes

The issue asks Slack preview streaming to keep showing in-flight answer draft text when `streaming.preview.toolProgress` is false, while suppressing tool-progress lines and generic progress chatter.

## Best Possible Solution

Slack should treat `streaming.preview.toolProgress: false` as answer-preview-on and in-preview-tool-lines-off whenever Slack preview streaming is active, while also suppressing generic visible progress chatter and aligning Slack docs/config help with the shared streaming contract.
