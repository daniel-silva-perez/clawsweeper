## Summary

Keep open. Current main still has a plausible Slack preview-silence path for the reporter's object-only `streaming.preview.toolProgress: false` config: Slack defaults that config to partial/native streaming, top-level DMs without a thread still disable draft preview eligibility, and Slack's callback wiring only sends draft partial-text updates when preview streaming is enabled and native streaming is not in use. The issue also overlaps open PR #56544, which targets the flat-DM preview gate, so this should be reconciled rather than closed.

## What This Changes

The issue requests that Slack keep in-flight answer preview text visible while hiding tool-progress preview lines when `streaming.preview.toolProgress` is false.

## Best Possible Solution

Resolve the Slack DM/object-config path with tests that prove `streaming.preview.toolProgress: false` means answer-preview-on and tool-lines-off, then align Slack docs/config help with the shared streaming contract. If #56544 is the right vehicle, land or update it with this regression coverage; otherwise use it as source context for a focused replacement.
