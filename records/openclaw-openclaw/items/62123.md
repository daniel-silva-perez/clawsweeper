## Summary

Keep this issue open. Current main still has the gap the reporter describes: `after_tool_call` is async but observational/void, `tool_result_persist` is intentionally synchronous and ignores Promise results, and the only async pre-model tool-result middleware is documented and enforced as bundled-plugin-only. Related open PR #64907 and issue #29132 are useful context, but they target a synchronous/text-only pre-model hook and do not cover this issue's requested async validation/mutation/suppression path for external plugins.

## Best Possible Solution

Keep the issue open for maintainer/API design. The best path is an additive documented plugin hook or a carefully promoted external version of the existing tool-result middleware that awaits handlers before model/transcript visibility, defines priority and failure behavior, respects `allowPromptInjection=false`, supports mutation and suppression semantics, and adds contract/runtime tests around ordering, persistence, and model-visible content.
