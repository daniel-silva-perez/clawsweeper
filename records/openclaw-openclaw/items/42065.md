## Summary

Keep this PR open. Current main still does not implement the central user-facing request: there is no global Control UI locale switcher in the topbar, and the current locale selector remains in Overview -> Gateway Access. The PR also needs rework against the current generated Control UI i18n workflow and has an unresolved async locale persistence review finding.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up. Best path: rebase or rewrite against current main, decide whether the global topbar locale picker is the desired product placement, use `scripts/control-ui-i18n.ts` as the single sync/check workflow, route remaining shell/theme/topbar/update copy through the current i18n source of truth, regenerate locale outputs and metadata, fix async locale persistence ordering, and add focused UI/i18n coverage for the picker.
