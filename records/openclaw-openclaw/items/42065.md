## Summary

Keep this PR open. Current main has a generated Control UI i18n pipeline and an Overview-only language picker, but it still does not implement the PR's central user-facing change: a global topbar locale switcher that works across tabs. The PR also needs rework for the current generated-locale workflow and the async locale persistence issue already raised in review.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up. Best path: rebase or rewrite against current main, use scripts/control-ui-i18n.ts as the single sync/check workflow, get maintainer judgment on global topbar placement, implement the topbar language picker if desired, route remaining shell/topbar/theme/update copy through the current i18n source of truth, regenerate locale output, and fix async setLocale/settings persistence ordering before review.
