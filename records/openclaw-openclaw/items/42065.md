## Summary

Keep this PR open. Current main still does not implement the PR's central user-facing change: a global topbar locale switcher that works across tabs. Main only exposes the language picker in Overview -> Gateway Access, and the PR still needs rework for the current generated Control UI i18n workflow plus the async locale persistence issue already raised in review.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up. Best path: rebase or rewrite against current main, get maintainer judgment on global topbar placement, use scripts/control-ui-i18n.ts as the single sync/check workflow, route remaining shell/theme/update copy through the current i18n source of truth, regenerate locale outputs and metadata, fix async setLocale/settings persistence ordering, and add focused UI/i18n coverage for the topbar picker.
