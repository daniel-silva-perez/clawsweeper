## Summary

Keep this PR open. Current main has a broader generated Control UI i18n pipeline and an Overview-only language selector, but it still does not implement the PR's central user-facing change: a global topbar locale switcher that works across tabs. Some hardcoded shell/topbar/update copy also remains, and the PR needs rework for the current generated-locale workflow plus the async locale persistence concern raised in review.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up. The best path is to rebase or rewrite it against current main, use scripts/control-ui-i18n.ts as the single sync/check workflow, get maintainer product judgment on global topbar placement, implement the topbar language picker if desired, localize remaining topbar/theme/update/docs copy through ui/src/i18n/locales/en.ts plus generated locale output, and fix the async setLocale/settings persistence ordering before review.
