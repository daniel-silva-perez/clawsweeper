## Summary

Keep this PR open. Current main still does not implement the PR's central user-facing change, a global Control UI locale switcher in the topbar. Main supports locale selection only from Overview -> Gateway Access, and the PR also needs reconciliation with the current generated Control UI i18n workflow plus the unresolved async locale persistence review finding.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up. Best path: rebase or rewrite against current main, get maintainer judgment on global topbar placement, use scripts/control-ui-i18n.ts as the single sync/check workflow, route remaining shell/theme/update/topbar copy through the current i18n source of truth, regenerate locale outputs and metadata, fix async locale persistence ordering, and add focused UI/i18n coverage for the topbar picker.
