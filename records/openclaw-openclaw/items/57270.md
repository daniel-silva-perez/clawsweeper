## Summary

Keep this issue open. Current main still has the reported docs/schema mismatch: the docs, config help, TypeScript config type, and runtime merge path all expose `tools.exec.approvalRunningNoticeMs`, but the strict global `tools.exec` Zod schema still omits it. The provided GitHub context also shows open unmerged PR #57293 with closing syntax for this issue, so maintainers should review, land, or supersede that PR rather than close the issue early.

## Best Possible Solution

Keep this issue open and review #57293 or an equivalent fix that moves `approvalRunningNoticeMs` into `ToolExecBaseShape`, regenerates the base schema, and adds regression coverage that global `ToolsSchema` accepts `exec.approvalRunningNoticeMs` while unknown `tools.exec` keys remain rejected.
