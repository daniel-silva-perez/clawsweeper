## Summary

Keep this issue open. Current main still has the documented/global-schema mismatch: `tools.exec.approvalRunningNoticeMs` is documented and wired into runtime config types, but the strict global `tools.exec` Zod shape still omits it. There is also an open unmerged PR, #57293, whose body uses closing syntax for this issue, so the best maintainer action is to review/land or supersede that PR rather than close the issue early.

## Best Possible Solution

Keep this issue open and review #57293 or an equivalent fix that moves `approvalRunningNoticeMs` into `ToolExecBaseShape`, regenerates the base schema, and adds regression coverage that `ToolsSchema.safeParse({ exec: { approvalRunningNoticeMs: 3000 } })` succeeds while unknown keys remain rejected.
