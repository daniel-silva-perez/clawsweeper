## Summary

Keep open. Current main still has the cleanup-specific stale-path bug: `--fix-missing` resolves one persisted `sessionFile` path and prunes when that path is absent, while the normal transcript read path already checks the canonical `<sessionId>.jsonl` candidate first. The issue is also paired with open PR #69954, so it should remain open until that PR or a replacement fix lands.

## What This Changes

Prevent `openclaw sessions cleanup --fix-missing` from pruning valid fresh cron or heartbeat sessions when persisted `sessionFile` metadata is stale but the canonical `<sessionId>.jsonl` transcript exists.

## Best Possible Solution

Keep this issue open and review PR #69954. If that PR is not suitable, replace it with a focused cleanup fix that checks the canonical `<sessionId>.jsonl` transcript before pruning, safely repairs stale `sessionFile` metadata when that canonical file exists, preserves true-missing pruning behavior, and adds regression coverage in `src/commands/sessions-cleanup.test.ts`. Keep broader doctor/orphan archival behavior in separate follow-up issues.
