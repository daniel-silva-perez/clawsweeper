## Summary

Keep this issue open. Current main has writer-side hardening for new cron/session rollover, but the reported cleanup safety hole is still present: `sessions cleanup --fix-missing` resolves one transcript path, prefers a persisted `sessionFile` when it is structurally valid, and prunes the entry if that path is missing without checking the canonical `<sessionId>.jsonl` fallback or a live `.jsonl.lock`. An open implementation candidate, #69954, explicitly closes this issue, so the issue should remain open until that PR is reviewed and merged or replaced.

## Best Possible Solution

Keep this issue open and review #69954 or an equivalent patch that makes `sessions cleanup --fix-missing` check the canonical `<sessionId>.jsonl` path, avoid pruning live locked/recent sessions, and add focused regression coverage. If broader transcript archival semantics remain after that patch, track them separately, likely alongside #73471.
