## Summary

Keep open. Current main still prunes `--fix-missing` entries from a single resolved `sessionFile` path with no canonical `<sessionId>.jsonl` fallback or repair path. Source-side rollover mitigations exist, but the cleanup-specific data-loss risk remains, and open PR #69954 is explicitly linked as the implementation candidate for this issue.

## Best Possible Solution

Keep this issue open and review PR #69954. If that PR is not suitable, close or replace it with a focused cleanup fix that checks the canonical `<sessionId>.jsonl` transcript before pruning, repairs stale `sessionFile` metadata when safe, preserves true-missing pruning behavior, and adds regression coverage in `src/commands/sessions-cleanup.test.ts`. Keep broader doctor/orphan transcript archival concerns tracked separately, such as #73471.
