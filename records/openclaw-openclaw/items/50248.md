## Summary

Keep open. Current main still has the cleanup-specific stale-path bug: `--fix-missing` resolves one persisted `sessionFile` path and prunes when that path is absent, while the normal transcript read path already checks the canonical `<sessionId>.jsonl` candidate before a stale generated `sessionFile`. PR #69954 is open and explicitly closes this issue, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Prevent `openclaw sessions cleanup --fix-missing` from pruning fresh cron or heartbeat sessions when stored `sessionFile` metadata is stale but a valid canonical `<sessionId>.jsonl` transcript exists.

## Best Possible Solution

Review and land PR #69954 if its fallback and repair behavior is acceptable; otherwise replace it with a focused cleanup fix that checks the canonical `<sessionId>.jsonl` transcript before pruning, repairs stale `sessionFile` metadata when that canonical file exists, preserves true-missing pruning, and keeps broader doctor/orphan archival work in separate follow-ups.
