## Summary

Keep this issue open. Current main still has configurable transient retry only for one-shot cron jobs, while recurring cron failures schedule the later of the natural next run and a generic backoff floor. There is no per-job retry contract, Gateway schema field, or CLI add/edit flag for configurable delayed re-execution before the next daily/weekly slot. Related PR #69252 was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open and implement an additive recurring cron execution retry policy: job-level retry config in cron types and Gateway schemas, CLI add/edit flags, durable retry attempt state, transient-error classification, scheduling at `endedAt + configured delay` before the next natural slot while attempts remain, and docs/tests that distinguish one-shot retry, recurring backoff floor, and delayed recurring re-execution.
