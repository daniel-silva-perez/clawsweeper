## Summary

Keep open. Current main partially overlaps the PR by adding a robust persisted-details byte cap, but it still counts hidden `toolResult.details` in the tool-result character estimator used by the preemptive context guard. That means the false-overflow part of the PR is still a real current-main gap, while the persistence portion needs rebasing to preserve the newer implementation.

## Best Possible Solution

Keep this PR open, rebase it on current main, and narrow the remaining code change to stop the estimator from counting model-invisible `toolResult.details` while preserving the existing persisted-details byte cap and tests. Add regression coverage for the aggregate false-overflow path, not only the single-result truncation path.
