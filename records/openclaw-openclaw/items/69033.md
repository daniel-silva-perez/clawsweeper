## Summary

Keep this PR open. Current main still has the false-overflow bug the PR targets: the preemptive context estimator counts hidden toolResult.details even though docs and compaction treat details as runtime metadata, not model-visible content. Main has already implemented a stronger persisted-details byte cap, so the PR should be rebased and narrowed to preserve that newer persistence behavior while landing the estimator fix and regression coverage.

## Best Possible Solution

Keep this PR open, rebase it on current main, drop or rewrite the stale persistence portion to preserve the newer byte-bounded persisted-details implementation, and land the narrow estimator change that stops counting model-invisible toolResult.details. Add the aggregate false-overflow regression test from the PR against the current guard behavior.
