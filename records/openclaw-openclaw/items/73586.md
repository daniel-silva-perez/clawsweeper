## Summary

Keep open. The report is fresh and current main still does not prove the sandbox-off Docker probe path is fixed: the isolated cron embedded-runner handoff still passes only the run-scoped session key, the embedded runner still falls back from `sandboxSessionKey` to `sessionKey`, and the Docker image inspection helper still throws on daemon-unavailable errors if the sandbox backend path is reached. The provided GitHub context also shows open PR #73627 with closing syntax for this issue, plus alternate PR #73671, so the issue should remain open until maintainers review and land or reject those fixes.

## Best Possible Solution

Keep this issue open until maintainers review and either land PR #73627, choose PR #73671, or apply an equivalent focused fix. The preferred fix should ensure automated isolated runs pass the correct sandbox policy key, guarantee `sandbox.mode: "off"` cannot initialize Docker/backend code, and add regression coverage for Docker-unavailable cron, heartbeat, and sub-agent startup paths.
