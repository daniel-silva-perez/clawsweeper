## Summary

Keep open. The report is fresh, detailed, and current main still has an open implementation candidate, PR #73627, that explicitly targets this issue and is unmerged. Current main also still passes the run-scoped cron session key into the embedded runner without an explicit sandbox policy key, while Docker image inspection still throws if the sandbox backend is reached unexpectedly.

## Best Possible Solution

Keep this issue open until maintainers review and either land PR #73627, choose PR #73671, or apply an equivalent fix. The preferred fix should make automated runs pass the correct sandbox policy key, guarantee sandbox.mode off cannot initialize Docker/backend code, and add focused regression coverage for cron, heartbeat, and sub-agent startup with Docker unavailable.
