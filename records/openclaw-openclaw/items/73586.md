## Summary

Keep open. The report is fresh, detailed, and still paired with open implementation candidates. Current main still has the cron embedded runner passing only the run-scoped session key, while the embedded runner resolves sandbox policy from sandboxSessionKey first and then sessionKey, so the central automated-run policy handoff is not proven fixed on main.

## Best Possible Solution

Keep this issue open and have maintainers review PR #73627, PR #73671, or an equivalent patch. The preferred fix should route automated runs through the correct sandbox policy key, ensure sandbox.mode off cannot initialize Docker/backend code, and add focused regression coverage for cron, heartbeat, and sub-agent startup with Docker unavailable.
