## Summary

Keep open. This is a fresh, detailed regression report with live implementation candidates, including open PR #73627 explicitly tied to this issue with closing syntax. Current main still has the isolated cron embedded-run call without an explicit sandboxSessionKey, while the embedded runner falls back to sessionKey/sessionId for sandbox resolution and the Docker image inspect path still throws the reported error shape if reached.

## Best Possible Solution

Keep this issue open and review #73627, #73671, or an equivalent maintainer patch. The fix should make automated cron, heartbeat, and sub-agent sandbox policy resolution use the correct session/runtime-policy key, ensure sandbox.mode off cannot initialize Docker/backend code, and add focused regression coverage proving Docker is not touched when sandboxing is off.
