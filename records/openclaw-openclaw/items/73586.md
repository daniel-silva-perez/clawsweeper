## Summary

Keep open. This is a fresh, detailed regression report with an open implementation candidate, #73627, that uses closing syntax and is not merged. Current main still lacks an explicit `sandboxSessionKey` on the isolated cron embedded-run call, and the existing code/test surface does not prove that sandbox mode `off` automated sessions can never reach Docker backend setup.

## Best Possible Solution

Keep this issue open and review #73627, #73671, or an equivalent maintainer patch. The fix should make cron, heartbeat, and sub-agent sandbox policy resolution use the correct runtime-policy session key, ensure `sandbox.mode: "off"` cannot initialize Docker/backend code, and add focused regression coverage proving Docker is not touched for automated sessions with sandboxing disabled.
