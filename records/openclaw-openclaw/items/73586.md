## Summary

Keep open. The report is recent, detailed, and an open implementation candidate, #73627, uses closing syntax for this issue. Current main still lacks the PR's proposed explicit `sandboxSessionKey` handoff for isolated cron embedded runs, and there is no focused regression test proving `sandbox.mode: "off"` avoids Docker image inspection for cron/heartbeat/sub-agent startup.

## Best Possible Solution

Keep this issue open and use #73627 as the implementation candidate. Review whether it passes the correct policy session key for isolated cron, heartbeat, and native sub-agent paths, keeps the `mode: "off"` resolver guard ahead of all Docker/backend probes, and adds a focused regression test that fails if `docker image inspect` is reachable when sandboxing is disabled.
