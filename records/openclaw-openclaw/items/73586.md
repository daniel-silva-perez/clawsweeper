## Summary

Keep open. This is a recent, detailed regression report with an open implementation candidate, #73627, explicitly tied to this issue. Current main still has the cron embedded-run call without an explicit sandboxSessionKey, so the best maintainer path is to review/land or replace that PR rather than close the issue early.

## Best Possible Solution

Keep this issue open and review #73627, #73671, or an equivalent maintainer patch. The fix should make cron, heartbeat, and sub-agent sandbox policy resolution use the correct session/runtime-policy key, ensure sandbox.mode off cannot initialize Docker/backend code, and add focused regression coverage proving Docker is not touched for sandbox.mode off automated sessions.
