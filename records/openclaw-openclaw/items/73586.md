## Summary

Keep open. This is a fresh, detailed bug report and the provided GitHub context shows an open, unmerged PR with closing syntax for this issue. Current main still lacks the explicit cron sandboxSessionKey handoff that the implementation candidate proposes, and there is no focused regression proof that sandbox.mode off automated runs avoid Docker/backend initialization.

## Best Possible Solution

Keep this issue open and have maintainers review PR #73627, PR #73671, or an equivalent patch. The fix should route automated run sandbox policy through the correct agent/runtime-policy key, ensure sandbox.mode off cannot initialize Docker/backend code, and add focused regression coverage for cron, heartbeat, and sub-agent startup with Docker unavailable.
