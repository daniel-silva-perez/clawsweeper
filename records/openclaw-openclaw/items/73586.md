## Summary

Keep open. Current main still has the reported cron embedded-run path without an explicit sandbox policy key, and the embedded runner still falls back to the run-scoped session key before resolving the sandbox context. There is also an open implementation candidate, #73627, whose body uses closing syntax for this issue, so policy says maintainers should review or close that PR before this issue is closed.

## Best Possible Solution

Keep this issue open and have maintainers review #73627, #73671, or an equivalent patch. The fix should ensure automated cron, heartbeat, and sub-agent runs resolve sandbox policy with the intended agent/session key, guarantee `sandbox.mode: "off"` cannot reach Docker/backend initialization, and add focused regression coverage for Docker-daemon-unavailable behavior.
