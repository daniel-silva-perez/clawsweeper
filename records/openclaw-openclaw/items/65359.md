## Summary

Keep this PR open. Current main still rejects `messages.groupChat.historyLimit: 0` in both the Zod schema and generated public config schema, while docs say `0` disables group history; the PR is a focused fix paired with still-open #65305.

## What This Changes

This PR changes `GroupChatSchema.historyLimit` from `.positive()` to `.min(0)`, updates the generated base config schema, and adds boundary tests for zero, positive, and negative values.

## Best Possible Solution

Land this PR or an equivalent narrow update that keeps the Zod schema, generated base schema, and regression coverage in sync; then close #65305 as implemented.

## Reproduction Assessment

Yes. The source-level reproduction is deterministic: current main uses `.positive()` for `GroupChatSchema.historyLimit`, so parsing `{ historyLimit: 0 }` fails, and the generated schema also uses `exclusiveMinimum: 0`.

## Solution Assessment

Yes. The proposed `.min(0)` validator change plus regenerated schema and boundary tests is the narrowest maintainable fix and aligns group chat with the documented DM/channel history semantics.
