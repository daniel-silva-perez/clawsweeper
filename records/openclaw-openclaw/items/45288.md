## Summary

Keep this PR open. The reported gateway-status destination-hardening gap still exists on current main, but this patch is incomplete because IPv4-mapped IPv6 metadata addresses can bypass the new string checks while targeting the same link-local range.

## What This Changes

The PR adds gateway-status WebSocket hostname filtering in `src/commands/gateway-status/helpers.ts` and a new security test file covering metadata, link-local, wildcard, internal/local, loopback, LAN, public-hostname, and non-WebSocket URL cases.

## Best Possible Solution

Land a current-main gateway-status hardening that parses explicit and configured target URLs once, uses the shared SSRF/IP classifier or equivalent canonical IP-literal handling for mapped IPv6 and legacy forms, preserves intended loopback and LAN diagnostics, and adds regression coverage for mapped and hex IPv6 metadata URLs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Reject IPv4-mapped metadata addresses:** `src/commands/gateway-status/helpers.ts:103-106`
  - body: The new guard only rejects exact `0.0.0.0`, dotted `169.254.*` hosts, and blocked hostname suffixes. Node normalizes `ws://[::ffff:169.254.169.254]:80` to `[::ffff:a9fe:a9fe]`, which bypasses those checks while still targeting the link-local metadata range. Use shared IP classification or equivalent canonical IP-literal handling and add mapped/hex IPv6 cases.
  - confidence: 0.94

## Security Review

Status: needs_attention

Summary: The diff is a security fix and does not add dependency or supply-chain changes, but it leaves a concrete SSRF bypass for IPv4-mapped IPv6 metadata endpoints.

Concerns:

- **[high] IPv4-mapped metadata URLs still bypass:** `src/commands/gateway-status/helpers.ts:103`
  - body: The patch blocks plain dotted `169.254.*` strings but not IPv4-mapped IPv6 literals such as `ws://[::ffff:169.254.169.254]:80` or `ws://[::ffff:a9fe:a9fe]:80`; those forms target the same metadata range while bypassing the new string checks.
