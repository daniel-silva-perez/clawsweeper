## Summary

Keep open. Current main still has only static `extra_body` pass-through plus provider/header observability pieces; it does not expose session identity to the provider extra-param or OpenRouter stream-wrapper path, and it does not implement dynamic `user` or `trace` request-body metadata for OpenRouter/OpenAI-compatible calls. The related open PRs #54725 and #64813 cover different parts of the design and should be reviewed before this issue is closed.

## What This Changes

The issue requests session-aware provider request metadata, such as OpenRouter/OpenAI-compatible `trace` or `user` fields, so API spend and observability can be attributed to graphs, tasks, agents, or sessions.

## Best Possible Solution

Keep the issue open until maintainers choose a privacy-safe attribution design: likely an additive provider SDK/runtime seam for sanitized session attribution, plus OpenRouter/LiteLLM-compatible `user` and `trace` handling, explicit precedence rules, tests, and docs for opt-in third-party metadata forwarding.
