## Summary

Keep this PR open. Current main already covers much of the stale-plugin cleanup and bundled-plugin load-path cleanup, including the dangling-channel follow-up from #65293, but it still does not implement the PR’s other stated behavior: ordinary `gateway status` and doctor do not report CLI-vs-service state-dir mismatch, `~/.openclaw-tests/*` service usage, or service-vs-CLI runtime version mismatch.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, drop or adapt the stale-plugin and bundled-load-path repair pieces already covered by shared doctor helpers, and focus the patch on tested `gateway status`/doctor runtime-drift reporting for CLI-vs-service state-dir mismatch, test-state service usage, and service-vs-runtime version mismatch while preserving service-environment redaction.
