## Summary

Keep this PR open. Current main already covers the stale-plugin and redundant bundled-plugin load-path cleanup, but it still does not implement the PR's runtime drift surface for ordinary gateway status or doctor: no CLI-vs-service state-dir fields, no test-state classifier, and no service-vs-CLI version mismatch reporting in those paths.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, drop or adapt the stale-plugin and bundled-load-path repair pieces already handled by shared doctor helpers, and focus the patch on tested gateway status/doctor runtime drift reporting for CLI-vs-service state-dir mismatch, test-state usage, and service-vs-runtime version mismatch.
