## Summary

Keep open. Current main already covers the stale-plugin cleanup and bundled-plugin load-path cleanup pieces, including the dangling-channel follow-up from #65293, but it still does not implement the PR’s runtime-drift behavior for `gateway status` or doctor: no CLI/service state-dir comparison, no `~/.openclaw-tests/*` service classification, and no service-vs-CLI version mismatch reporting.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, drop or adapt the stale-plugin and bundled-load-path cleanup pieces already covered by shared doctor helpers, and focus the patch on tested runtime-drift reporting in `src/cli/daemon-cli/status.gather.ts`, `src/cli/daemon-cli/status.print.ts`, and `src/commands/doctor-gateway-services.ts`: CLI-vs-service state-dir mismatch, service use of `~/.openclaw-tests/*`, and service-vs-runtime version mismatch, while preserving redacted service metadata output.
