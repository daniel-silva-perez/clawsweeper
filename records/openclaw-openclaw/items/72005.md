## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks the server-rendered outbound footer feature, while the proposed patch has concrete correctness issues in gateway delivery, dry-run state mutation, and one-shot threshold persistence.

## What This Changes

This PR adds server-side outbound footer rendering and fabricated-footer stripping, context-warning config and session state, sendMessage wiring, unit tests, generated schema updates, and a new docs page.

## Best Possible Solution

Keep the PR as the active implementation thread for #71984, but revise it so the same processed text reaches direct, gateway, mirror, and dry-run surfaces correctly; dry runs do not mutate session state; crossed thresholds are recorded atomically; and the docs/tests cover those contracts.

## Reproduction Assessment

Yes. Static inspection of the PR branch reproduces the blocking cases: gateway sends params.content after processedContent is computed, dry-run runs the stateful hook before returning, and a threshold jump records only one crossed threshold.

## Solution Assessment

No. The product direction is sound, but the current patch is not the narrowest maintainable solution until it applies processed content across all delivery modes and makes warning state updates non-spurious and atomic.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Send processed content through gateway mode:** `src/infra/outbound/message.ts:337`
  - body: `processedContent` is only used to build the outbound payload plan, but the gateway request still sends `params.content`. Any channel using `deliveryMode: "gateway"` will still deliver fabricated model footers and miss server-authored warnings.
  - confidence: 0.95
- **[P2] Avoid consuming warnings during dry runs:** `src/infra/outbound/message.ts:244-249`
  - body: The stateful footer hook runs before the `dryRun` early return and can persist `contextWarningThresholdsTriggered`. A dry-run send can therefore mark a warning as delivered without sending anything, causing the next real send to omit it.
  - confidence: 0.91
- **[P2] Record skipped thresholds when warning fires:** `src/infra/outbound/footer-hook.ts:195-198`
  - body: When usage jumps across multiple thresholds, the hook persists only the chosen highest threshold. Lower crossed thresholds remain unwarned and can fire later out of order, breaking the once-per-threshold contract.
  - confidence: 0.9
- **[P2] Make threshold check and record atomic:** `src/infra/outbound/footer-hook.ts:155-157`
  - body: The hook decides whether to warn from `alreadyWarned` before the session-store update. Two concurrent sends for the same session can both read the old state and both emit the same one-shot warning.
  - confidence: 0.84
