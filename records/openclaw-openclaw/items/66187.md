## Summary

Keep open. Current main still classifies ended subagent orchestrators with pending descendants as active and uses that same count for per-session spawn quota rejection; the related stale-unended orphan work covers a different liveness case.

## What This Changes

The issue requests that subagent runs with `endedAt` set stop appearing active, stop consuming spawn quota, and be archived or pruned after a configurable duration.

## Best Possible Solution

Define a separate liveness policy for user-active/quota accounting versus cleanup and announce settlement. If maintainers accept the product behavior, ended orchestrators with pending descendants should remain cleanup-aware but stop appearing in the active list and stop consuming the requester’s spawn slots.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an ended parent subagent run with a live or cleanup-pending descendant; current tests construct that shape and assert it remains active and quota-counted. I did not run tests because this review is read-only.

## Solution Assessment

Unclear until maintainers confirm the intended policy. The best implementation is not just hiding rows from `/subagents list`; it should split active/quota counting from pending descendant cleanup and announce gating.
