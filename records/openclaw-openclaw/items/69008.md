## Summary

Keep this issue open. Current main still gates gateway agent reply delivery on `request.deliver === true`, while Telegram group and topic sessions with the reported key shapes are current behavior. The linked open implementation candidate from the reporter should be reviewed or ported before this issue is closed.

## Best Possible Solution

Review the linked implementation candidate, or port the scoped gateway delivery-flag helper and focused tests into `src/gateway/server-methods/agent.ts` and `src/infra/outbound/agent-delivery.test.ts`: auto-deliver only `agent:*:telegram:group:*` and `agent:*:telegram:group:*:topic:*` sessions when `deliver` is omitted, preserve `deliver:false`, and keep internal-message-channel suppression.
