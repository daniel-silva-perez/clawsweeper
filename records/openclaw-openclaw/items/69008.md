## Summary

Keep open. Current main still requires an explicit `deliver: true` before gateway agent replies can leave the session, and the reported Telegram group/topic session key shapes are still live. The reporter also has an open implementation candidate in https://github.com/glfruit/openclaw/pull/1, so the right cleanup path is review, land, or port that fix rather than closing this issue now.

## What This Changes

Default Telegram group-bound gateway agent sessions to external delivery when `deliver` is omitted, while preserving `deliver: false`, internal-channel suppression, and no auto-delivery for unrelated session shapes.

## Best Possible Solution

Add or land a small gateway delivery-flag helper plus focused tests that auto-enable delivery only for `agent:*:telegram:group:*` and `agent:*:telegram:group:*:topic:*` when `deliver` is omitted, while keeping `deliver:false` authoritative and suppressing external sends for the internal message channel.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: send a gateway `agent` request with a Telegram group or topic session key and no `deliver`; current main computes both `wantsDelivery` and final `deliver` as false, so the run can update transcript state without external Telegram delivery.

## Solution Assessment

Yes. The suggested gateway-local helper is the narrowest maintainable fix because the decision is made before downstream outbound delivery, and tests can cover Telegram group/topic keys without changing Telegram DM, webchat, main, or other channel semantics.
