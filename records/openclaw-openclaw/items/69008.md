## Summary

Keep this issue open. Current main still gates gateway agent reply delivery on explicit `deliver: true`, while Telegram group/topic session keys matching the report are still current behavior. The reporter also has an open implementation candidate, so the issue should remain open until that PR is reviewed, ported, merged, or closed.

## Best Possible Solution

Review the reporter's open implementation candidate, or port the same scoped gateway delivery-flag helper and focused tests into `src/gateway/server-methods/agent.ts` and `src/infra/outbound/agent-delivery.test.ts`: auto-deliver only `agent:*:telegram:group:*` and `agent:*:telegram:group:*:topic:*` sessions when `deliver` is omitted, preserve explicit `deliver:false`, and keep internal-message-channel suppression.
