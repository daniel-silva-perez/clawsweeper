## Summary

Keep open: current main still rejects workspace bootstrap symlink targets that resolve outside the workspace root and then renders the failed reads as missing. Open PRs already cover the same behavior, including same-author #64473 with closing syntax and maintainer-labeled #73401, so this needs maintainer review of the trust boundary before closure.

## What This Changes

Allow workspace bootstrap files such as AGENTS.md, SOUL.md, USER.md, and MEMORY.md to be symlinks into sibling ~/.openclaw directories without appearing as missing.

## Best Possible Solution

Land a narrowly scoped, security-reviewed bootstrap-reader fix that loads explicit workspace bootstrap symlinks only when the resolved target is a readable regular file and preserves rejection of arbitrary outside paths, dangling symlinks, non-regular files, hardlinks, and oversized files. If maintainers reject external bootstrap symlinks, document that boundary clearly and close the linked PRs with the rationale.
