## Summary

Keep open. Current main still routes workspace bootstrap reads through the workspace-root boundary, still rejects symlink/canonical targets outside that root, and still converts failed reads into missing bootstrap entries. The same-author PR #64473 remains an open implementation candidate with closing syntax, and maintainer PR #73401 is also open as a more targeted alternative.

## Best Possible Solution

Keep this issue open while maintainers review the open implementation candidates, especially the maintainer-labeled #73401 and same-author #64473. The best fix likely belongs narrowly in the workspace bootstrap reader, with tests for readable fixed bootstrap symlinks plus rejection of hardlinks, non-regular files, oversized files, dangling links, extra/glob bootstrap escapes, and arbitrary outside targets. After the chosen PR lands, close this issue as implemented and close the superseded PRs with a clear rationale.
