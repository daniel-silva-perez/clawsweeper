## Summary

Keep this issue open. Current main still routes workspace bootstrap reads through the workspace-root boundary, still rejects canonical symlink targets outside that boundary, and still turns failed guarded reads into missing bootstrap entries. The same-author implementation candidate #64473 remains open/unmerged with closing syntax, so this issue should not be closed before that PR is reviewed, landed, or rejected.

## Best Possible Solution

Keep this issue open and review #64473 as the same-author implementation candidate, coordinated with #40210/#40230. If maintainers accept ~/.openclaw sibling directories as trusted for bootstrap reads, implement the relaxation narrowly with tests for an AGENTS.md symlink to a sibling ~/.openclaw/kb target plus rejection of arbitrary outside targets. If that trust-boundary change is rejected, close or revise the PR with a clear security rationale and consolidate the remaining user-facing problem under the canonical issue.
