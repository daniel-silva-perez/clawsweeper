## Summary

Keep this issue open. Current main still forces text-only isolated cron deliveries with a resolved thread/topic id through direct outbound delivery, and the cron delivery contract still has no supported option for topic/forum targets to opt into the main-session announce/agent-turn route. The related group/topic announce work is real context, but it does not implement this cron completion-route feature request.

## Best Possible Solution

Keep this open for maintainer product/API design. A good fix would preserve direct delivery as the default for thread/topic targets and structured payloads, add a backward-compatible opt-in route for text-only isolated cron completions to use the main-session announce/agent-turn flow, define explicit fail-closed versus direct-fallback behavior, and update cron types/normalization, Gateway protocol schema, agent cron tool schema, CLI/UI surfaces if applicable, docs, and regression tests.
