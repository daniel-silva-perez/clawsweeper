## Summary

Current main still forces resolved thread/topic isolated cron deliveries through direct outbound delivery, and the cron delivery contract still has no supported option for text-only topic/forum completions to opt into a main-session announce/agent-turn route. The related #34160 subagent announce work is useful context, but it covers subagent completion handoff rather than cron completion routing.

## Best Possible Solution

Keep this open for maintainer product/API work. Add a backward-compatible cron completion-route option that preserves current direct delivery by default for thread/topic targets, lets text-only isolated cron completions opt into main-session announce/agent-turn handling, keeps structured payloads on direct delivery, defines fail-closed versus direct-fallback behavior, and updates cron types/storage/normalization, gateway protocol, agent tool schema, CLI/UI, docs, and regression tests.
