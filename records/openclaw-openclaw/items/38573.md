## Summary

Current main still forces isolated cron deliveries with a resolved thread/topic id down the direct outbound-delivery path, and the public cron delivery contract still has no supported selector for text-only topic/forum completions to opt into a main-session announce/agent-turn route. The related group/topic subagent announce work exists, but it is separate and does not implement this cron delivery-route feature.

## Best Possible Solution

Keep this open for maintainer product/API design. A good solution would preserve direct delivery as the default for thread/topic targets and structured payloads, add a backward-compatible opt-in route for text-only isolated cron completions to use the main-session announce/agent-turn flow, define explicit fail-closed versus direct-fallback behavior, and update cron types/normalization, Gateway protocol schema, agent cron tool schema, CLI/UI surfaces if applicable, docs, and regression tests.
