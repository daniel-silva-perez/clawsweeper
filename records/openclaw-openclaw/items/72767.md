## Summary

Keep open. Current main does not contain the new `agent.capabilities` Gateway RPC, and the PR is directionally useful, but the branch misreports inherited agent skills and is missing the required user-facing changelog entry before it can merge.

## What This Changes

The PR adds an `agent.capabilities` Gateway RPC with READ scope, protocol schemas/generated Swift models, method registration, and a handler aggregating model, tool, MCP, skill, channel-binding, and session-count data.

## Best Possible Solution

Keep the additive Gateway RPC direction, but compute `skills` through the existing effective skill-filter contract, add focused handler/protocol coverage for inherited defaults and explicit replacement, and record the user-facing method in the changelog before maintainer review resumes.

## Reproduction Assessment

Yes. With `agents.defaults.skills: ["github"]` and `agents.list: [{ id: "writer" }]`, this PR's static code path returns `[]` for `writer` because `resolveAgentConfig()` owns `skills` as `undefined`; the expected effective result is `github`.

## Solution Assessment

No. The new RPC is a reasonable maintainable surface, but this implementation should reuse `resolveAgentSkillsFilter` or `resolveEffectiveAgentSkillFilter` instead of detecting raw skills overrides on a resolved config object.
