## Summary

Current main does not implement the requested guardrail. The repo already loads trusted state-dir `.env` values into the CLI process and has existing macOS `launchctl` and service-token drift diagnostics, but I found no doctor/status/gateway-status warning specifically for a non-empty `OPENCLAW_GATEWAY_TOKEN` in `~/.openclaw/.env` coexisting with `gateway.auth.token` in local gateway mode. This is a valid, narrow diagnostic enhancement and is suitable for a focused ClawSweeper fix PR.

## Best Possible Solution

Keep this issue open and add a shared read-only diagnostic helper that detects local gateway mode with configured `gateway.auth.token` plus non-empty `OPENCLAW_GATEWAY_TOKEN` in the state-dir `.env`, redacts secrets, and surfaces the same warning in `openclaw doctor`, `openclaw status`/`status --all`, and `openclaw gateway status`. Install/update flow coverage can reuse the same helper where gateway service repair/status already has config and state-dir context.
