## Summary

Keep open. Current main still has the reported precedence behavior and adjacent diagnostics, but no guardrail that reads the state-dir `.env` and warns when a local `gateway.auth.token` is shadowed by `OPENCLAW_GATEWAY_TOKEN`. The issue is concrete enough for a focused diagnostic PR.

## What This Changes

Request a redacted doctor/status/gateway-status warning when local gateway mode has `gateway.auth.token` configured while `~/.openclaw/.env` contains a non-empty `OPENCLAW_GATEWAY_TOKEN` that can override it.

## Best Possible Solution

Add a shared diagnostic that reads the trusted state-dir `.env`, checks only local gateway mode with configured `gateway.auth.token` and non-empty `OPENCLAW_GATEWAY_TOKEN`, redacts all values, and surfaces consistent remediation in doctor, status, and gateway status/probe output plus user-facing docs and changelog.
