## Summary

Keep open. The underlying bug is still present on current main: manifest registry records retain `skills`, but loaded `PluginRecord`s and loader record construction still omit that metadata. The PR targets the right behavior, but its current head is syntactically broken and edits stale registry/test surfaces, so it should not merge as-is.

## What This Changes

The PR attempts to add `skills` to loaded plugin records, thread manifest-declared skills through loader-created records, default non-manifest records to an empty array, and add regression coverage.

## Best Possible Solution

Reapply the fix against current main by updating the canonical `PluginRecord` contract in `src/plugins/registry-types.ts`, forwarding `manifestRecord.skills` through the manifest-backed loader and CLI registry record creation paths, defaulting non-manifest records to `[]`, and placing focused regression coverage in the current loader test layout.
