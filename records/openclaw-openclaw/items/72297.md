## Summary

Keep this PR open. Current main still lacks the PR's latest stderr-derived manual repair hint for QMD path/pattern conflicts, and the PR is an active implementation candidate for open #71783. The main maintainer question is scope: the PR body still describes a full rebind fix, while the latest patch is warning-only and intentionally refuses automatic removal when QMD list metadata is name-only.

## Best Possible Solution

Keep this PR open for maintainer review. Either narrow the PR title/body and remove the closing relationship to #71783 so it lands as a focused warning/manual-repair improvement, or implement a verified repair path that safely reconciles stale QMD path/pattern conflicts without relying on unverifiable name-only list metadata.
