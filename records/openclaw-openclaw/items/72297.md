## Summary

Keep this PR open. Current main still does not implement the PR head's stderr-name diagnostic, and it still cannot safely auto-rebind the legacy QMD case where `collection list` exposes only a name without verifiable path metadata. The latest PR head is warning-only rather than a full repair for #71783, but the author already narrowed it after review and the diff is small enough for maintainer review instead of cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. Either narrow the title/body and remove the closing relationship to #71783 so it can land as a warning-only diagnostics improvement, or implement a verified repair path that uses the parsed stale name only to fetch/confirm structured path and pattern metadata before any remove+add operation, with regression coverage for both safe rebind and name-only refusal cases.
