## Summary

Keep this PR open for maintainer review. Current main still lacks the clearer QMD stale-name warning, while the latest PR scope is focused, non-destructive, and no blocking correctness issue was found in this pass.

## What This Changes

This PR adds QMD add-error parsing, logs a manual repair hint for unverifiable path/pattern conflicts, and adds regression coverage that the stderr-only case does not remove collections automatically.

## Best Possible Solution

Land this focused memory-core warning if maintainers accept the parser and log wording, while keeping #71783 open for any future verified automatic reconciliation path that can safely repair different-name collections from structured metadata.

## Reproduction Assessment

Yes. The high-confidence path is the PR's regression scenario: `qmd collection list --json` returns only `["workspace-legacy"]`, then `qmd collection add` fails with QMD stderr naming `workspace-legacy`; current main only emits the generic skipped-add warning.

## Solution Assessment

Yes for the narrowed scope. A warning and manual repair hint is the safer maintainable fix when list output lacks verifiable path/pattern metadata; automatic removal should remain reserved for structured matches.
