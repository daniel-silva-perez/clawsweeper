## Summary

Keep this PR open. Current main still lacks `--agent-id` and `--session-key` on `openclaw system event`, and the wake path still passes only `mode` and `text` through the gateway and cron service. The PR remains a plausible implementation candidate, but it needs refresh before merge because the session-key-only flow can enqueue the system event under the default agent while waking a session-derived agent.

## Best Possible Solution

Refresh this PR or land an equivalent focused implementation: add the CLI flags and docs, type and normalize wake params consistently, thread `agentId` and `sessionKey` through `CronService.wake`/`wakeNow`/`timer.wake`, and make `buildGatewayCronService` derive the enqueue target from `sessionKey` when `agentId` is omitted so event enqueue and heartbeat wake always target the same agent/session. Add regression coverage for `--agent-id`, `--session-key` alone, both together, and blank/invalid target values.
