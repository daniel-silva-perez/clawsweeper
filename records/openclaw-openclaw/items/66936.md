## Summary

Keep open. Current main has fixed the unresolved SecretRef crash and the `agents list`/bare `agents` plugin-preload slice, but the reported `agents add --non-interactive` hang vector remains exposed because `agents add` still inherits the broad parent plugin-load policy.

## What This Changes

The issue asks for `openclaw agents list` to tolerate unresolved Telegram SecretRefs and for `openclaw agents` subcommands to exit cleanly after completing.

## Best Possible Solution

Keep the shipped SecretRef and list/bare-agents fixes, then add a narrow follow-up for the remaining `agents add` startup path so non-interactive config-only adds do not preload plugin runtimes before completing and exiting.

## Reproduction Assessment

Yes for the remaining source-level exposure path: current main has no exact `agents add` startup-policy override, so that command inherits parent plugin preload. The full indefinite hang still needs the reporter’s plugin/runtime setup for live reproduction.

## Solution Assessment

No. The current partial solution is not the complete fix because it leaves `agents add` on the broad plugin-preload path; a safer follow-up is a focused startup-policy/test change for the non-interactive add path while preserving interactive setup behavior.

Fix the remaining `openclaw agents add ... --non-interactive` CLI hang exposure from plugin preload. Current `src/cli/command-catalog.ts` has a broad parent `agents` policy with `loadPlugins: "always"` and exact no-plugin overrides for bare `agents`, `agents list`, bind/bindings/unbind/set-identity/delete, but no exact or option-aware `agents add` policy. Add the narrowest startup-policy change so non-interactive/config-only `agents add` invocations do not preload plugin runtimes before running, while preserving interactive wizard behavior if setup plugins are still required there. Add regression coverage in the CLI startup/path policy tests for the add path, and update only focused command tests/docs/changelog as needed. Do not change global plugin loading, SecretRef runtime strictness, or the already-shipped read-only provider metadata behavior from #74220.
