## Summary

Keep open. Current main fixes the unresolved SecretRef crash and the plain JSON `agents list` fast path, but the reported post-completion hang class is still exposed by plugin preload on human `agents list`, bare `openclaw agents`, and `agents add`; related PR #74220 is still the active implementation path for part of that surface.

## What This Changes

The issue asks for `openclaw agents list` to tolerate unresolved Telegram SecretRefs and for `openclaw agents` subcommands to exit cleanly after completing.

## Best Possible Solution

End state should make the affected agents commands use a lightweight config/read-only path or a generic CLI plugin lifecycle teardown so list/add/bind/delete exit after output, while preserving useful provider/routing summaries from manifest/read-only APIs. Closure should wait for built CLI smoke proof with the previously problematic plugin installed.
