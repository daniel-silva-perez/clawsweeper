## Summary

Keep open. Current main resolves the unresolved SecretRef crash and the plain JSON `agents list` plugin-load path, but it still exposes the remaining CLI hang class through plugin preload on human `agents list`, bare `openclaw agents`, and `agents add`; related PR #74220 is open for the text list/bare agents slice, so this issue should not be closed before that lands and the residual add path is handled or explicitly split.

## What This Changes

The issue asks for `openclaw agents list` to tolerate SecretRef-backed Telegram bot tokens and for `openclaw agents` subcommands to exit cleanly after completing their work.

## Best Possible Solution

Land or adapt #74220 for bare `openclaw agents` and text `agents list`, then either add a narrow `agents add` startup-policy/teardown fix if that path still reproduces or split it into a focused follow-up; after that, verify the built CLI exits for list/add/bind/delete with a plugin that previously kept handles alive.
