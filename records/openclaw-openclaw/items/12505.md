## Summary

Keep open. Current main has useful optional tool sandbox backends and an SDK seam for registering sandbox backends, but the central request is still not implemented: secure-by-default tiered presets, process-isolated plugin execution, IPC-proxied plugin SDK calls, per-plugin env/fs/net/resource manifest policy, and default credential scoping. The maintainer and ClawSweeper comments in the issue already treat this as the canonical sandbox/isolation architecture tracker rather than resolved work.

## What This Changes

The issue requests a secure-by-default unified sandbox architecture with multi-platform isolation backends, tiered presets, IPC/proxy plugin execution, and manifest-declared per-plugin env/fs/net/resource policies.

## Best Possible Solution

Keep this as the canonical owner-reviewed sandbox/isolation tracker unless maintainers split it into narrower milestones. The desired end state is a staged design and implementation for secure-by-default presets, plugin or skill execution isolation, IPC/proxy SDK enforcement, manifest-declared env/fs/net/resource policies, credential scoping, resource controls, and migration guidance while preserving the current Docker/SSH/OpenShell backend groundwork.
