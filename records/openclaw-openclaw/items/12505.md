## Summary

Keep open. Current main has opt-in agent tool sandbox backends and a plugin SDK seam for sandbox backends, but it still does not implement the central request: secure-by-default tiered sandbox presets, native plugin/process isolation, IPC-proxied plugin SDK execution, or per-plugin manifest policies for env/fs/net/resources. The prior maintainer/Codex comment and PR #29974 context correctly frame the Docker work as partial groundwork, not a full resolution.

## Best Possible Solution

Keep this issue open as the canonical sandbox/isolation architecture tracker unless maintainers split it into narrower milestones. The best path is a core design for secure-by-default tiered presets, plugin/skill isolation or an equivalent stronger execution boundary, IPC/proxy SDK enforcement, manifest-declared env/fs/net/resource policies, credential scoping, resource controls, and migration guidance that treats current Docker/SSH/OpenShell tool sandbox backends as partial groundwork.
