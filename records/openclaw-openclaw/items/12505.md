## Summary

Keep open. Current main has useful opt-in tool sandbox backends and a plugin SDK seam for sandbox backends, but it still does not implement the central request: secure-by-default tiered presets, plugin/process isolation, IPC-proxied plugin SDK execution, or per-plugin manifest policies for env/fs/net/resources. The recent maintainer comment also explicitly keeps this as the canonical sandbox/isolation architecture tracker.

## What This Changes

Requests a secure-by-default, multi-platform sandbox architecture with tiered presets, plugin/process isolation, IPC-proxied SDK execution, manifest-declared capabilities, credential scoping, and resource controls.

## Best Possible Solution

Keep this as the canonical sandbox/isolation tracker unless maintainers split it into narrower milestones. The end state should be an owner-reviewed design and staged implementation for secure-by-default presets, plugin or skill execution isolation, IPC/proxy SDK enforcement, manifest-declared env/fs/net/resource policies, credential scoping, resource controls, and migration guidance while preserving current opt-in Docker/SSH/OpenShell backend groundwork.
