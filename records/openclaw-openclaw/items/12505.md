## Summary

Keep this issue open. Current main still has opt-in agent tool sandboxing via Docker/SSH/OpenShell and a plugin SDK seam for sandbox backends, but it does not implement the central request: secure-by-default tiered sandbox presets, native plugin process isolation, IPC-proxied plugin SDK calls, or per-plugin manifest policies for env/fs/net/resources. The existing discussion correctly treats PR #29974 as partial Docker sandbox progress, not a full resolution.

## Best Possible Solution

Keep this issue open as the canonical sandbox/isolation architecture tracker unless maintainers split it into narrower issues. The best path is a core design for secure-by-default tiered presets, plugin/skill isolation or an equivalent stronger execution boundary, IPC/proxy SDK enforcement, manifest-declared env/fs/net/resource policies, credential scoping, resource controls, and migration guidance that treats current Docker/SSH/OpenShell tool sandbox backends as partial groundwork.
