## Summary

Keep open. Current main has optional tool sandbox backends and a plugin SDK sandbox-backend seam, but it still does not implement the central request: secure-by-default tiered presets, plugin/process isolation, IPC-proxied plugin execution, per-plugin env/fs/net/resource manifest policies, or default credential scoping. Recent maintainer and ClawSweeper discussion also treats this as the canonical sandbox/isolation architecture tracker, not resolved work.

## What This Changes

Request a secure-by-default unified sandbox architecture with tiered presets, multi-platform process/filesystem/network/resource isolation, IPC-proxied plugin execution, and manifest-declared plugin capability policies.

## Best Possible Solution

Keep this as the canonical owner-reviewed sandbox/isolation tracker unless maintainers split it into narrower milestones. The desired end state is a staged design and implementation for secure-by-default presets, plugin or skill execution isolation, IPC/proxy SDK enforcement, manifest-declared env/fs/net/resource policies, credential scoping, resource controls, and migration guidance while preserving the current Docker/SSH/OpenShell backend groundwork.
