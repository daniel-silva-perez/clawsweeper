## Summary

Keep open. Current main has useful opt-in tool sandbox backends and a plugin SDK seam for sandbox backends, but it still does not implement the central request: secure-by-default tiered presets, plugin/process isolation, IPC-proxied plugin execution, per-plugin env/fs/net/resource manifest policies, or default credential scoping. The maintainer discussion also already treats this as the canonical sandbox/isolation architecture tracker rather than resolved work.

## What This Changes

Add a secure-by-default unified sandbox architecture with multi-platform plugin/process isolation, tiered presets, per-plugin capability manifests, IPC SDK proxying, credential scoping, and resource/network/filesystem controls.

## Best Possible Solution

Keep this as the canonical owner-reviewed sandbox/isolation tracker unless maintainers split it into narrower milestones. The desired end state is a staged design and implementation for secure-by-default presets, plugin or skill execution isolation, IPC/proxy SDK enforcement, manifest-declared env/fs/net/resource policies, credential scoping, resource controls, and migration guidance while preserving the current Docker/SSH/OpenShell backend groundwork.
