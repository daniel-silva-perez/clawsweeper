## Summary

Keep open. Current main has useful opt-in tool sandbox backends and Docker/SSH/OpenShell groundwork, but it still does not implement the central architecture requested here: secure-by-default tiered presets, plugin/process isolation, IPC-proxied plugin execution, per-plugin manifest policies, or default credential scoping. The provided maintainer discussion also explicitly keeps this as the sandbox/isolation architecture tracker rather than treating the partial sandbox work as a resolution.

## What This Changes

The issue requests a unified, secure-by-default sandbox architecture with multi-platform process isolation, tiered presets, plugin capability manifests, credential scoping, IPC-proxied execution, and resource controls.

## Best Possible Solution

Keep this as the canonical owner-reviewed sandbox/isolation tracker unless maintainers split it into narrower milestones. The desired end state is a staged design and implementation for secure-by-default presets, plugin or skill execution isolation, IPC/proxy SDK enforcement, manifest-declared env/fs/net/resource policies, credential scoping, resource controls, and migration guidance while preserving current Docker/SSH/OpenShell backend compatibility.
