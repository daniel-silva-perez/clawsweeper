## Summary

Keep open. Current main has opt-in Docker/SSH/OpenShell sandbox backends for agent tool execution and a plugin SDK seam for sandbox backends, but the central request remains unresolved: secure-by-default tiered presets, native plugin/process isolation, IPC-proxied plugin execution, manifest-declared per-plugin env/fs/net/resource policy, and default credential scoping are not implemented. The existing discussion already treats this as a canonical security architecture tracker rather than completed work.

## What This Changes

The issue requests a unified, secure-by-default sandbox architecture with tiered presets, multi-platform process isolation, IPC-proxied plugin execution, per-plugin capability manifests, credential scoping, and resource controls.

## Best Possible Solution

Keep this as the canonical owner-reviewed sandbox/isolation tracker unless maintainers split it into narrower milestones. The desired end state is a staged design and implementation for secure-by-default presets, plugin or skill execution isolation, IPC/proxy SDK enforcement, manifest-declared env/fs/net/resource policies, credential scoping, resource controls, and migration guidance while preserving the current Docker/SSH/OpenShell backend groundwork.
