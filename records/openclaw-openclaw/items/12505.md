## Summary

Keep open. Current main has opt-in Docker/SSH/OpenShell sandbox backends for agent tool execution and a plugin SDK seam for sandbox backends, but it still does not implement the central request: secure-by-default tiered presets, plugin/process isolation, IPC-proxied plugin execution, per-plugin manifest env/fs/net/resource policies, or default credential scoping. The issue discussion also already treats this as the canonical sandbox/isolation architecture tracker rather than resolved work.

## What This Changes

Request a unified built-in sandbox architecture with multi-platform process, filesystem, network, credential, and resource isolation plus secure-by-default tiered presets and per-plugin manifest policy.

## Best Possible Solution

Keep this as the canonical owner-reviewed sandbox/isolation tracker unless maintainers split it into narrower milestones. The desired end state is a staged design and implementation for secure-by-default presets, plugin or skill execution isolation, IPC/proxy SDK enforcement, manifest-declared env/fs/net/resource policies, credential scoping, resource controls, and migration guidance while preserving the current Docker/SSH/OpenShell backend groundwork.
