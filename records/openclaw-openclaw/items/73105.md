## Summary

Keep open. The PR addresses a real embedded-run lifecycle gap that current main still has, but the current patch can synthesize the fallback terminal event even when this subscription has already received agent_end and is only waiting for its queued async handlers to finish, which can finalize a clean run with fallback abandoned/replayInvalid metadata too early.

## What This Changes

The PR moves terminal lifecycle idempotency into embedded subscription state, emits a synthetic end/error lifecycle event from unsubscribe when agent_end was missed, and adds unsubscribe fallback regression tests.

## Best Possible Solution

Keep the backend subscription-level guarantee, but update it so unsubscribe only synthesizes a terminal lifecycle when agent_end was not delivered to this subscription at all. If agent_end has been observed but is queued behind async message/block flushing, let the normal handler finish and emit the accurate terminal metadata.

## Reproduction Assessment

Yes. The original bug has a high-confidence code-level reproduction: a timeout/abort path calls activeSession.abort() without awaiting it, cleanup calls unsubscribe(), and the pi-coding-agent session can deliver agent_end after this subscription listener has been removed.

## Solution Assessment

No, not as currently written. The subscription seam is the right place to fix this, but the fallback needs an observed-agent_end guard or pending-handler coordination so it does not preempt the normal terminal path when agent_end is already queued.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Avoid fallback when agent_end is already queued:** `src/agents/pi-embedded-subscribe.ts:836`
  - body: `unsubscribe()` now synthesizes a terminal lifecycle whenever `handleAgentEnd` has not finished yet. If this subscription already received `agent_end` but it is queued behind an async `message_end`/block flush, this fallback sets `lifecycleTerminalEmitted` first and emits `livenessState: "abandoned"` plus `replayInvalid: true`, so the normal handler later skips the accurate terminal event. Track whether `agent_end` was observed, or drain/coordinate the pending event chain, before synthesizing.
  - confidence: 0.86
