## Summary

Keep open. Current main still has the embedded-run lifecycle gap described by the linked bug, and this PR remains a targeted implementation candidate rather than obsolete cleanup. I found no blocking correctness or security issue in the proposed patch, and the earlier async-hook review concern appears addressed by the latest PR commit.

## What This Changes

The PR shares terminal lifecycle idempotency through subscription state, emits a fallback lifecycle end/error event during embedded subscription unsubscribe when agent_end was missed, and adds unsubscribe fallback regression tests.

## Best Possible Solution

Review and land this narrow subscription-level fallback after CI, keeping the linked bug open until the PR merges and GitHub closes it. The backend lifecycle guarantee is the right primary fix; UI watchdog work can remain a separate mitigation if maintainers want extra recovery behavior.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is an embedded timeout/abort where unsubscribe removes the listener before pi-agent delivers agent_end; current main then has no lifecycle end/error event, and the PR test simulates unsubscribe without prior agent_end.

## Solution Assessment

Yes. Solving this at subscribeEmbeddedPiSession teardown is the narrowest maintainable fix because it guarantees the terminal lifecycle contract for every embedded-run caller without adding UI timers, runner-specific timeout special cases, or dependency changes.
