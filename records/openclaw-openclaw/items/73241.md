## Summary

Keep open. Current main still lacks the PR's BlueBubbles quoted-message API fallback and still logs attachment download failures only through verbose logging. The PR is recent, focused, and not superseded by the related BlueBubbles client consolidation, but it still needs normal review for the new webhook-driven HTTP lookup and the bot-raised sender/visibility concerns.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a BlueBubbles-owned fix that uses the existing SSRF-aware client for single-message reply fallback, preserves the current context-visibility/allowlist model, logs attachment download failures with sanitized fields, and adds targeted regression tests for the new fallback and logging behavior.
