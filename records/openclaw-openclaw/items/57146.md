## Summary

Keep open. Current main still contains the legacy path bug this PR targets, and the linked bug remains open, but the PR head is stale and has concrete docs correctness defects: Phase 2 repeats the same token read, and the replacement snippets do not preserve active config-path semantics.

## What This Changes

The PR edits `skills/gh-issues/SKILL.md` to replace `/data/.clawdbot` examples for token lookup, claims tracking, cursor tracking, and sub-agent prompts with `~/.openclaw` or `$HOME/.openclaw` paths.

## Best Possible Solution

Update this PR branch or create a narrow replacement that removes every `/data/.clawdbot` reference from `skills/gh-issues/SKILL.md`, uses the active OpenClaw config/state path pattern for token and state-file examples, and adds a focused guard in `src/agents/skills.env-path-guidance.test.ts` so the legacy path cannot return.

## Reproduction Assessment

Yes. Static inspection reproduces both sides: current main still has the nine legacy `/data/.clawdbot` references, and the PR head shows the duplicate Phase 2 fallback reading the same file twice.

## Solution Assessment

No. The direction is correct, but this patch is not the best mergeable form because it replaces the legacy fallback with duplicate/default-only reads rather than a single active `OPENCLAW_CONFIG_PATH` / `OPENCLAW_STATE_DIR` aware pattern.

  - confidence: 0.92

Agents using a non-default config path will still fail to load `GH_TOKEN`; reuse the `CONFIG_PATH=${OPENCLAW_CONFIG_PATH:-...}` pattern here.
  - confidence: 0.84
