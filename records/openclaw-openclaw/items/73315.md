## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Parse OpenClaw config with JSON5:** `apps/desktop/src-tauri/src/main.rs:297`
  - body: `openclaw.json` is a JSON/JSON5 config and current main parses it through the JSON5 loader. This path uses `serde_json::from_str`, so a valid config with comments or trailing commas is treated as invalid and the desktop app falls back to loopback defaults, losing configured port, auth, and bind settings.
  - confidence: 0.91
- **[P2] Permit configured gateway origins in CSP:** `apps/desktop/src-tauri/tauri.conf.json:14`
  - body: The backend honors `gateway.bind: "custom"` and builds dashboard/health URLs from `gateway.customBindHost`, but the Tauri CSP only permits `127.0.0.1` and `localhost` for `connect-src` and `frame-src`. A valid custom host such as `192.168.1.8` will be generated by the Rust code and then blocked by the webview.
  - confidence: 0.88
- **[P2] Update platform docs before landing:** `apps/desktop/README.md:3-4`
  - body: This adds a Linux/Windows desktop companion, but the public platform docs still say those companion apps are planned or unavailable. If this merges as-is, docs.openclaw.ai platform guidance will be stale for the new app surface.
  - confidence: 0.82
- **[P2] Add labeler coverage for the desktop app:** `pnpm-workspace.yaml:3`
  - body: The PR registers `apps/desktop` as a new workspace, but `.github/labeler.yml` still routes only Android, iOS, macOS, and web UI app changes. Repo policy requires new app surfaces to update label routing, otherwise future desktop changes will miss the expected app label.
  - confidence: 0.8
- **[P2] Add a changelog entry for the desktop feature:** `package.json:1302-1303`
  - body: The PR exposes a new user-facing desktop app and root desktop scripts, but `CHANGELOG.md` has no Unreleased entry for the feature. Repo policy requires user-facing `feat` changes to carry a changelog entry with contributor credit before landing.
  - confidence: 0.84
