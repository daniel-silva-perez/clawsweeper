## Summary

Keep this PR open. Current main still has no Linux/Windows desktop companion, and the project vision treats those companion apps as first-party platform scope, but the branch is not merge-ready because its CSP conflicts with its own custom/TLS gateway URL handling and it misses required platform-docs and labeler updates. The latest head does address the Greptile P2 feedback, so the remaining concerns are narrower than the original review.

## What This Changes

The branch adds a private `apps/desktop` Tauri v2 workspace with Rust tray, gateway health/restart integration, a React/Tailwind shell embedding the Gateway Control UI, root desktop scripts, workspace registration, and Cargo/pnpm lockfile updates.

## Best Possible Solution

Keep one canonical desktop companion PR if maintainers want the Tauri direction, fix the CSP/docs/labeler gaps, document the expected desktop validation path, prove Linux and Windows behavior, and leave installer packaging as a separately tracked follow-up.
