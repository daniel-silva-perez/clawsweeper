## Summary

Keep this PR open, but it is not merge-ready. Current main still has no Linux/Windows Tauri desktop companion, and the platform vision/docs treat that as planned first-party app work rather than a ClawHub-only idea. The PR is an active successor to older desktop work, but it still needs maintainer product/security review plus concrete fixes for custom-bind CSP support, public docs, and labeler routing before merge consideration.

## What This Changes

The PR adds a private Tauri v2 desktop companion workspace under `apps/desktop`, with Rust tray/health/restart code, a React/Tailwind shell embedding the Gateway Control UI, root desktop scripts, workspace registration, and pnpm/Cargo lockfile updates.

## Best Possible Solution

Keep this as an active desktop-app implementation candidate only if maintainers want the Tauri direction: rebase it onto current main, address the CSP/docs/labeler findings, add CI/build ownership for the new app surface, provide Linux and Windows validation, and consolidate the related desktop PR/issue threads intentionally.

  - body: The backend honors `gateway.bind: "custom"` and builds dashboard URLs from `gateway.customBindHost`, including IPv6/custom hosts, but the Tauri CSP only allows frames/connects to `127.0.0.1` and `localhost`.
