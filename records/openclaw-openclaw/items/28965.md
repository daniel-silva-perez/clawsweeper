## Summary

Keep this issue open. Current main documents and implements several model/status/session-selection layers, but it still does not provide the requested first-class `openclaw models reconcile` command or scoped `openclaw sessions model-reset` operator command. The existing gateway reset path intentionally preserves user-selected model/auth overrides, so it is not equivalent to a bulk reset for default-model migrations.

## Best Possible Solution

Keep this issue open for maintainer design. The best path is to define the canonical model/config reconciliation contract first, then implement documented operator commands for `openclaw models reconcile` and scoped `openclaw sessions model-reset` with dry-run/JSON output, explicit target selection, and tests proving user/session metadata is preserved unless intentionally reset.
