## Summary

Keep this issue open. Current main has model status/list/set flows, session listing/cleanup, and gateway-level single-session reset behavior, but it still does not provide the proposed first-class `openclaw models reconcile` command or a scoped bulk `openclaw sessions model-reset` CLI. The proposal is valid but broad enough to need maintainer design before an autonomous fix PR.

## Best Possible Solution

Keep this issue open until maintainers either define the canonical model/config reconciliation design or implement documented operator commands for `openclaw models reconcile` and scoped `openclaw sessions model-reset`. The implementation should reuse the existing models and sessions CLI surfaces, session-store targeting helpers, dry-run/JSON patterns, and add coverage for clearing model/provider/auth-profile overrides without disturbing unrelated session metadata.
