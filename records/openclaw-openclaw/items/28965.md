## Summary

Keep this issue open. Current main still has targeted model setters/status/listing, per-session model patch/reset behavior, and session cleanup, but it does not provide the proposed first-class `openclaw models reconcile` workflow or scoped bulk `openclaw sessions model-reset` command. The remaining work is valid but broad enough that maintainers should settle the command/product boundary before queueing an autonomous fix PR.

## Best Possible Solution

Keep this issue open until maintainers either define a canonical model/config reconciliation design or implement documented operator commands for `openclaw models reconcile` and scoped `openclaw sessions model-reset`. The implementation should live in the existing `models` and `sessions` CLI surfaces, reuse current session-store targeting helpers, support dry-run/JSON and agent/store filters, and add regression coverage for clearing model/provider/auth-profile overrides without disturbing unrelated session metadata.
