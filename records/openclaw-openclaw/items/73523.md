## Summary

Keep this PR open. Current main does not implement the cache-only Chrome MCP health probe, and the passive browser status/listing paths still reach session-allocation code that can spawn `chrome-devtools-mcp` when the in-memory cache is empty. The PR is a focused implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open and review it as the active implementation candidate. The likely best fix is the PR's cache-only Chrome MCP health probe wired into `GET /`, `GET /profiles`, and `GET /tabs`, with focused browser plugin regression tests proving those passive paths do not call the session factory or spawn Chrome MCP when the cache is empty.
