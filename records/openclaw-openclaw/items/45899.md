## Summary

Keep open. Current main still lacks the Feishu-specific blocked-local-MEDIA hint and one-time final-reply fallback this PR proposes; newer docs partly cover the MEDIA local-path guidance, and related #51375 overlaps only with `mediaLocalRoots` plumbing, not the user-facing recovery behavior.

## What This Changes

The PR adds Feishu reply-dispatcher detection for disallowed local `MEDIA:` path errors, sends a one-time plain-text final-reply fallback, adds targeted dispatcher tests, and adds MEDIA local-path documentation notes.

## Best Possible Solution

Rebase this PR onto current main, preserve the newer generic MEDIA documentation, and land or adapt the Feishu-specific `LocalMediaAccessError`/`path-not-allowed` hint plus one-time final fallback if maintainers want blocked local media paths explained directly in Feishu.

## Reproduction Assessment

Yes. The focused reproduction path is a Feishu final reply containing a local `MEDIA:` path outside allowed roots: current media loading throws `LocalMediaAccessError` with `path-not-allowed`, and current Feishu `onError` only logs/closes without a user-facing fallback.

## Solution Assessment

Partly yes. The runtime fallback is a narrow maintainable Feishu-channel recovery path, but the submitted docs hunks should be adapted to current `workspaceOnly` docs and coordinated with #51375’s `mediaLocalRoots` plumbing before merge.
