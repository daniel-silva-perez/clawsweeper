## Summary

Keep open. Current main still lacks the Feishu-specific blocked-local-MEDIA detection and final fallback; the generic docs have moved on and #51375 is complementary media-root plumbing, so this PR needs maintainer review and rebase rather than cleanup closure.

## What This Changes

The PR adds Feishu blocked local MEDIA-path detection with a runtime hint and final-reply fallback, dispatcher tests, and setup/FAQ notes about local MEDIA path restrictions.

## Best Possible Solution

Adapt the runtime portion onto current main, preserving current streaming error-state handling and newer docs; coordinate with #51375 so valid agent-scoped media sends pass roots while disallowed paths get a clear Feishu fallback.

## Reproduction Assessment

Yes. A focused reproduction is a Feishu final reply containing a local `MEDIA:` path outside allowed roots; current media loading throws `LocalMediaAccessError` with `path-not-allowed`, and current Feishu `onError` only logs and closes out.

## Solution Assessment

Partly yes. The runtime fallback is a narrow Feishu-channel recovery path that preserves the allowlist, but the docs portion should be adapted to current main and the final shape should account for the complementary media-root plumbing in #51375.
