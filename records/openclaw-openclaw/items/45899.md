## Summary

Keep open. Current main has largely superseded the docs additions, but it still lacks the Feishu-specific blocked local `MEDIA:` hint and one-time final-reply fallback; related #51375 is complementary media-root plumbing, not a replacement for this user-facing recovery path.

## What This Changes

The branch changes Feishu reply dispatching to detect blocked local `MEDIA:` path errors, log a hint, send one final-reply plain-text fallback, add dispatcher tests, and add local media path notes to setup and FAQ docs.

## Best Possible Solution

Rebase this PR onto current main, keep the newer generic `MEDIA:` documentation, and land or adapt the Feishu-specific `LocalMediaAccessError`/`path-not-allowed` hint plus one-time final fallback if maintainers want blocked local media paths explained directly in Feishu.

## Reproduction Assessment

Yes. A focused reproduction is a Feishu final reply containing a local `MEDIA:` path outside allowed roots; current media loading throws `LocalMediaAccessError` with `path-not-allowed`, and current Feishu `onError` logs/closes without a user-facing fallback.

## Solution Assessment

Partly yes. The runtime fallback is a narrow Feishu-channel recovery path that preserves the allowlist, but the docs portion should be adapted to current main and coordinated with #51375 rather than merged as-is.
