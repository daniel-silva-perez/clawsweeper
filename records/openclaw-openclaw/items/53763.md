## Summary

Keep this issue open. Current `main` has a first-class bundled browser plugin, managed/headless local launch, remote CDP, Chrome MCP existing-session attach, optional Docker Chromium installation, and an optional sandbox browser image, but it still does not bundle or install a Chromium runtime for normal installs. Local managed launch remains dependent on a separately installed/discoverable Chromium-family browser or an explicit remote/CDP setup, which is the central gap reported here.

## Best Possible Solution

Keep the issue open and narrow the remaining work to browser runtime packaging: either ship a pinned Chromium runtime by default, publish an official browser-enabled Docker image, or add installer/doctor repair that installs and persists Playwright Chromium and teaches executable discovery to use that path. Reuse the existing browser plugin, headless mode, extraArgs, remote CDP, Chrome MCP, sandbox browser, and optional Docker build paths rather than adding a parallel browser stack.

## Risks / Open Questions
