## Summary

Keep open. Current main has a bundled browser plugin, headless local launch, remote CDP support, Chrome MCP existing-session attach, and optional Docker browser installation, but normal installs still do not bundle or resolve a Chromium runtime independent of host-installed browser binaries.

## What This Changes

Request to bundle a headless Chromium runtime as a first-class OpenClaw browser tool so agents can access JS-rendered and login-required pages without depending on user Chrome, third-party extract APIs, or external browser setup.

## Best Possible Solution

Narrow the issue to a maintainer-approved packaging direction: ship a pinned Chromium runtime by default, publish an official browser-enabled image, or add installer/doctor repair that installs Playwright Chromium into a persistent path and wires that path into executable discovery with a clear update policy.
