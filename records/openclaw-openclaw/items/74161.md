## Summary

Keep open. Current main still lacks the exact-`<location>` skill-path guard, and the PR is useful hardening but incomplete because the multi-skill path can still instruct the model to read a chosen skill without the same never-guess constraint.

## What This Changes

This PR adds an exact-`<location>` / never-guess constraint to the single-skill branch of the agent skills prompt in `src/agents/system-prompt.ts`.

## Best Possible Solution

Land a narrow prompt hardening that applies the exact-`<location>` / never-guess rule to every skills selection path, covers that behavior in focused prompt tests, and leaves the broader sandbox/discovery bugs on their existing issue tracks.

## Reproduction Assessment

Yes. The static reproduction is `buildSkillsSection()` on current main: the single-skill and multi-skill prompt lines can be inspected directly, and the PR diff shows only the single-skill line is changed.

## Solution Assessment

No. The direction is right, but the submitted patch is not the best complete fix because it leaves the multi-skill branch unguarded; the safer narrow fix is to apply the same constraint to both branches and update prompt coverage.
