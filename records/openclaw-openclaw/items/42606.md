## Summary

Keep this PR open. The hardening remains useful, but current main still does not implement it on the live noVNC bootstrap route, and the PR branch targets the stale `src/browser` path instead of the active browser plugin files.

## Best Possible Solution

Keep this PR open, but require a narrow port onto `extensions/browser/src/browser/bridge-server.ts` and `extensions/browser/src/browser/bridge-server.auth.test.ts`. Preserve the one-time observer token flow and fragment-only password behavior, add the nonce-based CSP plus `nosniff` and frame-deny assertions, then run the targeted browser bridge auth test and the appropriate changed gate before merge.
