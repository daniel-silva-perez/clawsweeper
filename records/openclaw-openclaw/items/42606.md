## Summary

Keep this PR open. The hardening is still useful and current main still lacks it on the live noVNC bootstrap route, but the PR targets the stale `src/browser` path rather than the active browser plugin files under `extensions/browser`.

## Best Possible Solution

Keep this PR open, but do not merge the current branch as-is. Port the nonce-based bootstrap response and header assertions to `extensions/browser/src/browser/bridge-server.ts` and `extensions/browser/src/browser/bridge-server.auth.test.ts`, preserving the one-time observer-token flow and fragment-only noVNC password behavior.
