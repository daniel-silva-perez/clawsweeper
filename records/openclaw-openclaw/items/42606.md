## Summary

Keep this PR open. The hardening is still useful, but current main has not implemented it on the active browser plugin route, and the PR branch targets the removed `src/browser` path rather than `extensions/browser`. Maintainer comments already called out the same required port, and the current tree still matches that assessment.

## Best Possible Solution

Keep this PR open and require a narrow current-path port onto `extensions/browser/src/browser/bridge-server.ts` and `extensions/browser/src/browser/bridge-server.auth.test.ts`. Preserve the one-time observer token flow and fragment-only password behavior, add nonce-based CSP plus `nosniff` and frame-deny assertions, then run the targeted browser bridge auth test and the appropriate changed gate before merge.
