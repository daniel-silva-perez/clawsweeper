## Summary

Keep this PR open. The hardening request is still valid, but current main has not implemented it on the active browser plugin path, and the PR body/diff still targets the removed `src/browser` path instead of `extensions/browser`.

## Best Possible Solution

Port the same narrow hardening to `extensions/browser/src/browser/bridge-server.ts` and `extensions/browser/src/browser/bridge-server.auth.test.ts`: generate a per-response script nonce, return CSP plus HTML together, set `Content-Security-Policy`, `X-Content-Type-Options: nosniff`, and `X-Frame-Options: DENY`, and preserve the current observer-token redirect semantics with the noVNC password only in the URL fragment.
