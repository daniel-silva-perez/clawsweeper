## Summary

Keep this PR open. The hardening is still useful, but current main has not implemented it on the active browser plugin route, and this PR still targets the removed `src/browser` path rather than `extensions/browser`. Maintainer comments in the thread already reached the same conclusion and asked for a narrow port.

## Best Possible Solution

Keep the PR open and ask for a narrow current-path rewrite: port the nonce-based bootstrap response and header assertions to `extensions/browser/src/browser/bridge-server.ts` and `extensions/browser/src/browser/bridge-server.auth.test.ts`, preserving the current one-time observer-token behavior and keeping the noVNC password in the URL fragment only.
