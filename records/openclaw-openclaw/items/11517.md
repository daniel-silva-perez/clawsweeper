## Summary

Keep open. Current main still has the import-heavy embedded attempt runner and broad Discord monitor boundaries described in the report, and the merged #70743/#70772 seam work did not complete a prepared RunContext/AgentContext split or Discord monitor decomposition. ## What This Changes

The issue requests a dependency-injection style refactor that reduces import coupling in the embedded attempt runner and Discord monitor by moving resolved config and services into typed run/context objects.

## Best Possible Solution

Keep this as the canonical staged architecture refactor: first introduce behavior-preserving prepared context or service bundles around runEmbeddedAttempt, then split Discord preflight, process, and provider responsibilities behind typed local helpers with focused regression coverage.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The static architecture concern is high-confidence: current-main import counts, file sizes, function signatures, and context types still match the reported coupling pattern.

## Solution Assessment

Yes directionally, but only as a staged refactor. A prepared context/service bundle is the maintainable direction, while Discord should be decomposed through local typed helpers rather than a broad rewrite.
