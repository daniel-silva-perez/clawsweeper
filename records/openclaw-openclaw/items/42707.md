## Summary

Keep this PR open. Current main still has the Telegram native thread-bound subagent session gap: Telegram has session-binding infrastructure, but it does not register the subagent lifecycle hooks that `sessions_spawn({thread:true, mode:"session"})` needs, and CLI session resume still does not hydrate requester routing from persisted session delivery context. The PR is stale and not merge-ready as submitted, but it remains a relevant implementation candidate alongside open tracker #42986.

## Best Possible Solution

Keep this PR open as a stale but still relevant implementation candidate. The best path is to rebase or replace it with a boundary-compliant implementation that adds a Telegram `registerFull` hook bridge, registers `subagent_spawning`, `subagent_delivery_target`, and `subagent_ended`, reuses the existing Telegram thread-binding manager and target parser, hydrates `resolveAgentRunContext` from persisted `SessionEntry` delivery context on session resume, updates native subagent docs/tests, and leaves #42986 as the user-facing tracker until the corrected implementation lands.
