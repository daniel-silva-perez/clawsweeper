## Summary

Keep this PR open. Current main still does not implement the PR's central Telegram native subagent session-thread wiring: Telegram has thread-binding manager/config support, but the bundled Telegram entry does not register subagent lifecycle hooks, and resumed agent runs still do not hydrate requester routing from the resolved session entry. The related tracker #42986 remains open, so this PR is still a relevant implementation candidate rather than cleanup that can close.

## Best Possible Solution

Keep this PR open as a stale but still relevant implementation candidate. Rebase or replace it with a boundary-compliant implementation that registers Telegram subagent_spawning, subagent_delivery_target, and subagent_ended hooks from registerFull via a Telegram hook API/barrel; uses the existing Telegram thread-binding manager and target parser; hydrates resolveAgentRunContext from the resolved SessionEntry deliveryContext/last delivery fields on session resume; passes run-context fallback to messageTo/messageThreadId; and keeps #42986 as the user-facing tracker until the corrected implementation lands.
