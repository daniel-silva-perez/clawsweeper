## Summary

Keep this PR open. Current main still does not implement Telegram topic-bound native subagent session spawns: Telegram has thread-binding infrastructure, but no bundled-entry `registerFull` hook bridge for `subagent_spawning`, `subagent_delivery_target`, or `subagent_ended`, and CLI session resume still does not hydrate requester routing from persisted session delivery context. The PR remains a relevant implementation candidate, but it is not merge-ready as-is because the submitted hook file reaches across the extension boundary into core internals.

## Best Possible Solution

Keep this PR open as a stale but still relevant implementation candidate. The best path is to rebase or replace it with a boundary-compliant implementation that adds a Telegram `registerFull` hook bridge, registers `subagent_spawning`, `subagent_delivery_target`, and `subagent_ended`, reuses the existing Telegram thread-binding manager and target parser, hydrates `resolveAgentRunContext` from persisted `SessionEntry` delivery context on `--session-id` resume, updates native subagent docs/tests, and leaves #42986 as the user-facing tracker until the corrected implementation lands.
