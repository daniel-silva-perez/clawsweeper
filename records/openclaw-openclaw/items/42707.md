## Summary

Keep this PR open. Current main still has the central Telegram native thread-bound subagent session gap: Telegram has session-binding infrastructure, but it does not register the subagent lifecycle hooks that `sessions_spawn({thread:true, mode:"session"})` requires, and the CLI session-resume run context still does not hydrate routing from the resolved persisted session entry. The PR remains relevant, with #42986 as the user-facing tracker, but it needs boundary-compliant rework before it is merge-ready.

## Best Possible Solution

Keep this PR open as a stale but still relevant implementation candidate. Rebase or replace it with a boundary-compliant implementation that registers Telegram subagent_spawning, subagent_delivery_target, and subagent_ended hooks from the bundled entry through a Telegram hook API/barrel; uses openclaw/plugin-sdk/conversation-runtime plus the existing Telegram thread-binding manager and target parser; hydrates resolveAgentRunContext from the resolved SessionEntry deliveryContext/last route fields on session resume; and keeps #42986 as the user-facing tracker until the corrected implementation lands.
