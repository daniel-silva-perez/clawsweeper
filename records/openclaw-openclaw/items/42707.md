## Summary

Keep this PR open. Current main has partial Telegram thread-binding infrastructure, but it still does not implement the PR's central behavior: Telegram does not register subagent lifecycle hooks, and resumed agent runs still do not hydrate requester routing from the resolved session entry. The related open issue #42986 remains a useful product tracker, and this PR remains a relevant implementation candidate after rebase and cleanup.

## Best Possible Solution

Keep this PR open as a stale but relevant implementation candidate. Rebase it onto current main; add Telegram `registerFull` lifecycle hook wiring like Discord, Matrix, and Feishu; keep Telegram hook code inside the plugin boundary through SDK/local API facades; reuse the existing Telegram thread-binding manager and session-binding adapter; hydrate `resolveAgentRunContext` from the resolved `SessionEntry` on `--session-id` resumes; normalize legacy Telegram target prefixes; and add focused tests for Telegram `sessions_spawn(thread=true, mode="session")`, resume routing hydration, delivery-target resolution, and unbind cleanup. The related open issue #42986 can continue tracking the feature until a corrected implementation lands.

- The submitted patch is not merge-ready as-is because its new Telegram production hook crosses the extension boundary with relative imports into core internals; a rebased fix should use `openclaw/plugin-sdk/conversation-runtime` or add a documented SDK seam.
