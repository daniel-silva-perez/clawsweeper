## Summary

Keep open. Current main still lacks Telegram subagent lifecycle hook registration and CLI session-resume routing hydration, so the reported Telegram topic-bound subagent session gap remains; the PR is relevant but not merge-ready because the submitted patch is stale against the current plugin layout and has concrete correctness issues.

## What This Changes

The PR adds Telegram subagent lifecycle hook registration, Telegram hook handlers, session-binding singleton changes, session-resume routing hydration, and tests for topic-bound Telegram subagent session spawning.

## Best Possible Solution

Land a refreshed Telegram-local implementation that mirrors the current Discord registerFull hook bridge, uses plugin SDK conversation-runtime exports and Telegram-local parsing/barrels, hydrates resumed agent runs from persisted SessionEntry delivery context, and updates native subagent docs/tests once Telegram support is actually enabled.

## Reproduction Assessment

Yes. A static reproduction path is a Telegram requester calling sessions_spawn with thread=true and mode="session": current main has no Telegram subagent_spawning hook, so src/agents/subagent-spawn.ts returns the hook-unavailable path before any Telegram binding can be prepared.

## Solution Assessment

No. The intended direction is right, but this patch is not the best current fix because it imports core internals from an extension, is stale against the current bundled entry layout, and mishandles supported Telegram target prefixes; a boundary-compliant replacement should reuse openclaw/plugin-sdk/conversation-runtime and Telegram-local target parsing.

Overall confidence: 0.88

Full review comments:

- **[P2] Use the plugin SDK seam for Telegram hooks:** `extensions/telegram/src/subagent-hooks.ts:6-8`
  - body: The new Telegram production hook module imports core internals with ../../../src paths, and one of those paths no longer exists on current main. Extensions are expected to cross into core through plugin SDK exports or local barrels, so this needs to be rewired through the current registerFull/API pattern before it can compile and pass boundary checks.
  - confidence: 0.9
- **[P2] Normalize Telegram target prefixes before binding:** `extensions/telegram/src/subagent-hooks.ts:51-56`
  - body: resolveTelegramConversationId only strips channel: and telegram:, so valid targets such as telegram:group:-100123:topic:55 or tg:-100123:topic:55 become malformed conversation IDs. Those malformed IDs later build delivery targets like channel:group:-100123, which the Telegram send path cannot resolve.
  - confidence: 0.88
- **[P3] Initialize the global service before asserting it:** `src/infra/outbound/session-binding-service.test.ts:216-224`
  - body: The added test reads globalThis.__openclawSessionBindingService before getSessionBindingService() is called in the test, but the PR initializes that global lazily. When this test runs in isolation, globalService is undefined and the final assertion fails.
  - confidence: 0.86
