## Summary

Keep open. Current main has adjacent improvements for execution bias, GPT-5 strict-agentic plan-only retries, exec approval/tool-event surfacing, trajectory export, and provider refusal stop-reason metadata. It still does not implement the issue’s central ask: a first-class distinction between real SSH/tool failure, permission or approval denial, and model-level soft refusal for explicitly authorized operational diagnostics.

## Best Possible Solution

Keep this issue open for a core agent-runtime design pass. The best path is to add first-class run observability that classifies tool execution failure, authorization or approval denial, and model/tool-routing soft refusal; refine prompt/runtime policy for explicitly authorized read-only infrastructure diagnostics; and add regression coverage with a simulated provider response that refuses SSH/log inspection instead of calling the available exec path.
