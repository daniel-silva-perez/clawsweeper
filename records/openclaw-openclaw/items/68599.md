## Summary

Keep open. The requested operator hint is useful and current main does not already provide it, but the PR branch still has a correctness issue: the hint is gated by a run-scoped overflow counter, so it can fire on a later independent overflow rather than only on an immediate post-compaction retry.

## What This Changes

The PR adds a `[context-overflow-hint]` warn log for repeated overflow after compaction, documents the troubleshooting symptom, adds two regression tests, and updates the changelog.

## Best Possible Solution

Keep the additive hint, but make it depend on explicit immediate post-compaction retry state and add a negative regression for an unrelated later overflow after a non-overflow continuation.

## Reproduction Assessment

Yes for the PR-introduced bug. The existing harness can mock overflow, successful compaction, a nonterminal continuation such as planning-only or empty response, then a later overflow; with the current PR guard that later overflow would still satisfy `overflowCompactionAttempts > 0`.

## Solution Assessment

No. The log/docs/tests direction is narrow and maintainable, but the implementation should key the hint to an immediate compact-then-retry overflow or reset the marker before unrelated retry paths.
