## Summary

Keep open. Current main still contains the reported divergence path: the gateway seeds Dashboard update availability from persisted `update-check.json` state and returns during the throttle window, while the CLI status command performs a registry-backed check. The fresh-empty path also still refreshes `lastCheckedAt` and writes state when npm resolution has no version, which can suppress a later banner for the normal interval.

## What This Changes

The issue asks OpenClaw to keep the Dashboard/Overview update banner in sync with `openclaw update status` when `~/.openclaw/update-check.json` is stale or missing an available version.

## Best Possible Solution

Fix the gateway update-check path so Dashboard/Overview availability is based on a valid registry result, stale cache cannot suppress a newer known npm version for the full interval, empty npm resolution preserves the previous good cache, and empty lookups retry on a short backoff with focused regression coverage.
