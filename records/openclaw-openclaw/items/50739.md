## Summary

Keep open. Current main still routes `openclaw system event --mode now` through the normal queued system-event plus heartbeat path, and the heartbeat runner intentionally skips delivery while the target session lane is busy. The KeepMyClaw discussion is useful recovery context, but snapshots do not provide the requested immediate in-session alert surface.

## What This Changes

The issue requests a high-priority or queue-bypass mode for `openclaw system event --mode now` so outage alerts reach the active session immediately instead of waiting behind congested model work.

## Best Possible Solution

Define a first-class high-priority system alert contract that delivers immediate operator-visible notifications, and only writes to transcript or prompt state through a separately ordered, trust-reviewed path with aligned CLI, Gateway protocol, heartbeat/session handling, TUI behavior, docs, and regression coverage.

## Risks / Open Questions
