## Summary

Current main still routes `openclaw system event --mode now` through the normal wake/heartbeat path. The CLI and protocol have no priority/bypass option, the wake path enqueues a system event and requests a heartbeat, and the heartbeat runner intentionally skips when the main or resolved session lane is busy. The KeepMyClaw comments are useful recovery context, but they do not replace the requested immediate in-session alert path.

## Best Possible Solution

Keep this issue open and design a core-owned high-priority system alert path. The likely implementation should add an additive Gateway/CLI contract field, preserve existing `wake` behavior by default, and use a narrowly scoped transcript/UI injection path with tests for busy main lane, busy session lane, trust labeling, and TUI/WebChat delivery.
