## Summary

Keep open. Current main still has no priority or queue-bypass contract for `openclaw system event --mode now`; it enqueues a prompt-visible system event and wakes heartbeat delivery, which intentionally defers while the target session lane is busy. The KeepMyClaw comments are useful recovery context, but snapshots do not satisfy the requested immediate in-session alert path.

## What This Changes

The issue requests a high-priority or bypass-queue mode for system events so urgent alerts can appear during congested in-session LLM work.

## Best Possible Solution

Define an explicit high-priority alert contract that delivers immediate operator-visible notifications, and only mutates transcript or prompt state through a separately ordered, trust-reviewed path with aligned CLI, Gateway protocol, heartbeat/session handling, docs, and regression coverage.

## Risks / Open Questions

- A prompt-visible bypass could race an active streaming turn or corrupt transcript ordering; current heartbeat code explicitly avoids busy session lanes.

- The product shape is unresolved: immediate operator/TUI notification may need to be separate from prompt-visible transcript injection.
