## Summary

Keep open. Current main still routes `openclaw system event --mode now` through queued system-event plus heartbeat wake delivery, with no CLI flag, wake contract field, gateway handling, or session-stream bypass for urgent alerts. The KeepMyClaw comments are useful recovery context, but external snapshots do not satisfy the requested immediate in-session alert path.

## What This Changes

Add a high-priority or bypass mode for `openclaw system event --mode now` so urgent system alerts can reach the active operator without waiting behind congested LLM/session-lane work.

## Best Possible Solution

Ship an explicit high-priority system alert contract that delivers immediate operator-visible notifications, and only mutates the transcript or prompt through a separately ordered, trust-reviewed path.

## Risks / Open Questions

- A prompt-visible bypass could race an active streaming turn or corrupt transcript ordering; current heartbeat code explicitly avoids busy session lanes.
- Because wake params allow unknown metadata, callers can send a `priority` field today and have it validate while current main silently ignores it.
- System events become trusted `System:` prompt-visible lines by default, so any bypass path needs spoofing and prompt-injection review.
- The product shape is unresolved: immediate operator/TUI notification may need to be separate from prompt-visible transcript injection.
