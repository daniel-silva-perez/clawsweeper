## Summary

Keep open. Current main still has no priority or bypass contract for `system event`, and `--mode now` still enqueues a system event then asks heartbeat to run; heartbeat deliberately skips busy session lanes. The KeepMyClaw and quota-prealert comments are useful adjacent mitigation context, but they do not provide the requested immediate operator-visible in-session alert path.

## What This Changes

Add a high-priority or bypass-queue mode for `openclaw system event --mode now` so urgent system alerts can appear promptly during provider outages instead of waiting behind congested session lanes.

## Best Possible Solution

Define a first-class urgent system alert contract that delivers immediate operator-visible notifications while handling transcript or prompt mutation through a separately ordered, trust-reviewed path. The final design should align the CLI, Gateway wake protocol, cron/heartbeat handling, TUI or operator event surface, docs, and regression tests.

## Risks / Open Questions

- A prompt-visible bypass can race an active streaming turn or corrupt transcript ordering; current heartbeat code explicitly skips busy session lanes to avoid interrupting active work.
- Wake params currently accept unknown metadata, so callers can send a `priority` field that validates but is silently ignored.
- System events default to trusted `System:` prompt lines, so bypass semantics need spoofing, prompt-injection, and trust-boundary review.
- The product shape is unresolved: immediate operator/TUI notification may need to be separate from transcript or prompt mutation, and quota pre-failure alerting may belong in a provider or circuit-breaker layer.
