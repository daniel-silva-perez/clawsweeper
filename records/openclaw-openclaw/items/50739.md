## Summary

Keep open. Current main still implements `openclaw system event --mode now` as queued system-event plus heartbeat delivery, and the heartbeat runner explicitly skips delivery when the target session lane is busy. The KeepMyClaw discussion is useful recovery context, but snapshots do not provide the requested immediate in-session alert surface.

## What This Changes

Request a high-priority or bypass-queue mode for `openclaw system event --mode now` so urgent in-session alerts appear immediately even when the session lane is congested.

## Best Possible Solution

Define a first-class urgent system alert contract that delivers immediate operator-visible notifications, with any transcript or prompt mutation handled through a separately ordered and trust-reviewed path. The implementation should align the CLI, Gateway protocol, cron wake path, heartbeat/session handling, TUI behavior, docs, and regression tests.

## Risks / Open Questions

- A prompt-visible bypass could race an active streaming turn or corrupt transcript ordering; current heartbeat code explicitly avoids busy session lanes.
- Wake params currently allow unknown metadata, so callers can send a `priority` field that validates but is silently ignored by the gateway wake handler.
- System events become trusted `System:` prompt lines by default, so any bypass path needs spoofing and prompt-injection review.
- The product shape is unresolved: immediate operator/TUI notification may need to be separate from prompt-visible transcript injection.
