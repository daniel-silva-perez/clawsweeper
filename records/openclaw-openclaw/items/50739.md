## Summary

Keep open. Current main still routes system events through the queued wake/heartbeat path and deliberately skips busy session lanes, so the requested reliable urgent in-session alert path is not implemented. The KeepMyClaw snapshot comments and the quota-prealert comment are useful mitigation context, but neither provides immediate TUI/session delivery.

## What This Changes

The issue requests a high-priority or bypass-queue mode for `openclaw system event --mode now` so urgent in-session alerts are not delayed behind congested session-lane LLM work.

## Best Possible Solution

Define a first-class urgent system alert contract that delivers immediate operator-visible notifications, while handling transcript or prompt mutation through a separately ordered and trust-reviewed path. The end state should align the CLI, Gateway wake protocol, cron/heartbeat handling, TUI or operator event surface, docs, and regression tests.

## Reproduction Assessment

Yes. The source-level reproduction path is high-confidence: `openclaw system event --mode now` sends `wake`, cron enqueues the event and requests heartbeat, and heartbeat skips with `requests-in-flight` when the resolved session lane is busy; the existing busy-lane test covers that path.

## Solution Assessment

Unclear/no for the proposed raw bypass flag. Immediate alert delivery is valid, but a prompt-visible queue bypass is not obviously the safest maintainable shape; a separate operator-alert surface plus ordered transcript handling is the safer design direction.

## Risks / Open Questions

- A prompt-visible bypass could race an active streaming turn or corrupt transcript ordering; current heartbeat code explicitly avoids busy session lanes for that reason.
- Wake params currently allow unknown metadata, so callers can send a `priority` field that validates but is silently ignored by current main.
- System events render as trusted `System:` prompt lines by default, so bypass semantics need spoofing and prompt-injection review before implementation.
- Adjacent queue/steer issues such as #48003 and #50880 overlap on active-turn injection mechanics but do not currently track the system-event CLI/protocol alert contract as a replacement.
