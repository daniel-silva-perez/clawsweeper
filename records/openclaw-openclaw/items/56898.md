## Summary

Keep this issue open. Current main and the latest release still ship the bundled `wacli` skill with only generic recipient/message confirmation text, so the requested recipient allowlist convention, redacted audit trail, and bulk-send protection remain unimplemented; this belongs in the bundled skill because `wacli` is already shipped from this repository.

## What This Changes

The issue requests stronger bundled `wacli` skill instructions for recipient allowlisting, explicit confirmation before every send, redacted audit logging, and bulk-send protection.

## Best Possible Solution

Update `skills/wacli/SKILL.md` with explicit outbound-send rules: consult a per-user recipient allowlist, always show recipient name/number or JID plus full message and attachments, require explicit approval for every send, write only redacted audit entries, and require explicit full-list plus per-recipient confirmation for bulk or similar sends. Keep runtime enforcement as a separate design if maintainers want guarantees beyond skill instructions.
