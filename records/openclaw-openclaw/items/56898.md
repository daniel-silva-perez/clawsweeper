## Summary

Keep this issue open. Current main contains a bundled `wacli` skill, but its safety section only requires an explicit recipient/message and a confirmation; it does not include the requested recipient allowlist convention, redacted audit trail, or bulk-send protections. This is a narrow bundled-skill hardening request, not a ClawHub close, because the affected skill is already shipped from this repository.

## What This Changes

The issue asks OpenClaw to strengthen the bundled `wacli` SKILL.md instructions with recipient allowlist handling, mandatory send confirmation, redacted audit logging, and bulk-send protection.

## Best Possible Solution

Update the existing bundled `skills/wacli/SKILL.md` with clear outbound-send rules: always show recipient name/number or JID, full message content, and attachments before sending; define how to consult a per-user recipient allowlist; require explicit approval for non-allowlisted recipients and all sends; log redacted send attempts/results; and require explicit full-list plus per-recipient confirmation for bulk or similar sends. Leave hard runtime enforcement to a separate design if maintainers want guarantees beyond skill instructions.

Not applicable as a feature/safety hardening request. The current-main verification path is high confidence: read `skills/wacli/SKILL.md` and search the repo for the requested allowlist, audit, and bulk-send terms; they are missing.
