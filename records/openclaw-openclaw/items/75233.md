## Summary

Keep open. Current main still implements the reported macOS LaunchAgent PATH behavior, and an open unmerged PR already references this issue as its closing fix, so this should stay open until that PR lands or is closed.

## What This Changes

The issue requests that macOS gateway LaunchAgent installation write a deterministic canonical PATH instead of persisting user Node-version-manager and package-manager PATH entries.

## Best Possible Solution

Review and either land or close PR #75246 as the canonical implementation path, while reconciling the macOS canonical PATH policy with related version-manager support requests such as #52184 and the Linux-focused #75220.

## Reproduction Assessment

Yes for the core source-level behavior: current main constructs a macOS service PATH containing version-manager/package-manager dirs ahead of system dirs, and tests assert that policy. The broader update timeout/version-mismatch symptoms are plausible but were not live-reproduced here.

## Solution Assessment

Unclear until PR #75246 is reviewed. The requested canonical macOS PATH is a plausible narrow fix, but the final solution should intentionally preserve or document any compatibility boundary for users relying on version-manager-provided child tools.
