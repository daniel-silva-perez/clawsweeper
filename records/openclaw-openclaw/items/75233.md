## Summary

Keep open. Current main still builds the macOS gateway service PATH from helper logic that includes Volta/asdf/fnm/pnpm-style user directories and then persists that effective service environment during install/repair, so the requested deterministic canonical PATH is not implemented.

## What This Changes

Request the macOS gateway service installer and repair path to write a deterministic system-only gateway service PATH instead of preserving or generating Node version-manager and package-manager directories.

## Best Possible Solution

Define a maintainer-approved macOS gateway service PATH policy, then update the installer, doctor repair/audit logic, docs, and focused service-env/install tests together. A maintainable end state should separate the absolute Node executable used to launch the gateway from the subprocess search PATH used by gateway children.

## Reproduction Assessment

Yes for the core source-level behavior: with macOS HOME and existing version-manager directories, current helpers include those directories before system dirs and the install plan persists that PATH. No for the broader timeout/version-mismatch symptoms because this review did not run a live macOS LaunchAgent.

## Solution Assessment

Unclear. The requested canonical PATH is plausible for deterministic update behavior, but it conflicts with current tests/docs and related version-manager support requests, so the best fix needs maintainer policy judgment before implementation.
