## Summary

Keep this issue open. The linked implementation candidate remains open and unmerged, and current main still only attempts raw JSON.parse plus ASCII-quote balanced JSON extraction for malformed tool-call arguments, so the reported Unicode smart-quote failure is not implemented on main.

## Best Possible Solution

Keep this issue open and review or replace PR #67551 as the active implementation path. The fix should narrowly normalize only structural Unicode smart quotes before JSON.parse, preserve literal smart quotes inside string values, make balanced JSON scanning consistent with that behavior, and add a focused regression test for malformed edit-tool arguments in isolated cron delivery.
