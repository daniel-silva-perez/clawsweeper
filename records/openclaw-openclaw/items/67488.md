## Summary

Keep open. Current main still lacks Unicode structural smart-quote handling in the malformed tool-call argument repair path, and the directly linked implementation candidate remains open and unmerged. The reported delivered-but-error cron state is consistent with the cron state code, so this is still a valid active bug rather than cleanup-ready.

## What This Changes

Cron isolated agent-turn jobs should not be marked as failed when malformed edit-tool argument JSON contains Unicode smart quotes or mixed CJK/Markdown content but the agent run and delivery completed successfully.

## Best Possible Solution

Review and land PR #67551 or an equivalent narrow fix that normalizes only structural Unicode smart quotes before JSON.parse, keeps literal smart quotes inside string values intact, aligns balanced JSON scanning with that behavior, and adds focused regression coverage for isolated cron edit-tool arguments.
