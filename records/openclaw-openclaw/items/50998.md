## Summary

Keep this PR open. Current main still has the iOS Contacts/Calendar first-use permission gap: Contacts and Calendar both fast-fail from `.notDetermined`, and the iOS plist/project config still lacks the required Contacts/Calendar usage-description keys. The PR is an active implementation candidate, not implemented or superseded, but it still needs maintainer review for Contacts prompt cancellation/headless behavior and an unrelated Windows installer change.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a scoped iOS fix that adds the missing usage-description keys, restores first-use Contacts and Calendar permission behavior with explicit timeout/cancellation semantics, preserves least-privilege Calendar write handling, adds focused tests plus simulator/device invoke proof, and splits or justifies the unrelated Windows installer change. Coordinate #51011 for the in-app Privacy & Access UX rather than treating it as a replacement for this bug fix.
