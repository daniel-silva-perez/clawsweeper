## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Make Contacts permission waits cancellation-aware:** `apps/ios/Sources/Contacts/ContactsService.swift:106-110`
  - body: This branch now awaits `CNContactStore.requestAccess` with a bare continuation. `invokeWithTimeout` cancels the invoke task after timeout, but this continuation does not observe cancellation, so locked or headless invokes can stay suspended until the system prompt is answered. Use a cancellation-aware permission helper or keep a fail-fast path for non-interactive invokes.
  - confidence: 0.9
- **[P2] Gate Calendar prompt start after cancellation wins:** `apps/ios/Sources/Calendar/CalendarService.swift:183-184`
  - body: The `isResolved` guard and `start` call are separate operations. If cancellation fires after the guard but before `start`, the continuation has already been resumed with `false` and this code can still launch an EventKit permission prompt after the invoke timed out. Make the install-and-start decision atomic so cancelled invokes cannot start late OS prompts.
  - confidence: 0.84
