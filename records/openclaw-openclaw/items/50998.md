## Summary

Keep this PR open. Current main still has the iOS Contacts/Calendar first-use permission gap, and the latest PR remains an active implementation candidate, but it introduces cancellation-sensitive permission waits and mixes an installer-script change into an iOS data-access fix.

## What This Changes

The PR adds iOS Contacts/Calendar usage-description keys, changes Contacts and Calendar authorization paths to request permissions on first use, adds Calendar permission-box tests, and changes Windows installer failure handling.

## Best Possible Solution

Land a scoped iOS-owner fix that adds the missing usage-description keys, makes Contacts and Calendar first-use permission requests cancellation-safe, preserves least-privilege Calendar write-only handling, and validates the node invoke flows on simulator or device. Keep #51011 as the companion settings UX and split the installer change into its own reviewed PR.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Make the Contacts permission wait cancellation-aware:** `apps/ios/Sources/Contacts/ContactsService.swift:106-110`
  - body: This branch now awaits `CNContactStore.requestAccess` with a bare continuation. When `invokeWithTimeout` times out it cancels the invoke task, but this continuation does not observe cancellation, so headless or locked-device invokes can remain suspended until the system prompt is answered. Use the same cancellation-aware permission helper intended for Calendar before starting the Contacts prompt.
  - confidence: 0.9
- **[P2] Gate Calendar prompt start after cancellation wins:** `apps/ios/Sources/Calendar/CalendarService.swift:183-184`
  - body: The `isResolved` check and `start` call are separate operations. If cancellation fires after the guard but before `start`, the continuation has already been resumed with `false` and the code can still launch the EventKit permission prompt after the invoke timed out. Make install-and-start an atomic decision so cancelled invokes cannot start a late OS prompt.
  - confidence: 0.84
