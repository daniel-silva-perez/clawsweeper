## Summary

Keep open. Current main still treats session transcript indexing as an opt-in experimental feature rather than a graduated/default-on stable memory behavior. The implementation pieces exist, including transcript parsing, session-source syncing, QMD session export, visibility filtering, and post-compaction hooks, but the default source list remains memory-only, the experimental flag defaults false, docs still label the feature experimental/opt-in, and there is no defaulting pipeline such as the proposed applyMemorySearchDefaults. The KeepMyClaw comment is useful external recovery context, but it does not replace the requested OpenClaw core product/default/privacy/docs decision. Related PR #51341 is closed unmerged, #53550 remains open for a gateway-dispatched session recall gap, and #59577 remains open for incremental session sync performance work.

## Best Possible Solution

Keep this issue open for memory/core maintainers to make a deliberate graduation decision. A good implementation path would first resolve or explicitly scope #53550, decide DM-only versus direct+channel privacy defaults, align builtin and QMD behavior, add any default-on behavior through the config materialization/default pipeline with focused tests, and update the public memory docs with opt-out, scope, stale-results, and embedding-cost guidance.
