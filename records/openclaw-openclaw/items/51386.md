## Summary

Keep open. Current main still treats session transcript indexing as an opt-in experimental memory feature, not a graduated/default-on stable behavior. The related KeepMyClaw comment is useful external context for continuity/recovery, but it does not replace the requested OpenClaw core policy/default/docs work. The closed related PR #51341 was not merged, and open #53550 still tracks a concrete sessionMemory visibility gap for gateway-dispatched sessions.

## Best Possible Solution

Keep this issue open for the memory/core maintainers to decide and implement graduation deliberately: resolve or explicitly scope the gateway-dispatched session gap, choose privacy defaults such as DM-only vs channel inclusion, wire any default-on behavior through the config/default pipeline with tests, and update the public memory docs with opt-out and embedding-cost guidance.
