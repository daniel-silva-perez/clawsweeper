## Summary

Keep open. Current main fixes much of the display consistency path: Gateway session rows/defaults expose profile-derived thinking options/defaults, `/think` reads those rows and rejects unsupported levels, `sessions.patch` validates explicit thinking levels, and commit a722da3e aligns `chat.history` with session/default resolution. But the issue also asks for GitHub Copilot xhigh support on models such as `gpt-5.4-mini` / `gpt-5-mini`; current main still advertises xhigh only for `gpt-5.4`, `gpt-5.3-codex`, `gpt-5.2`, and `gpt-5.2-codex`, while tests still treat `gpt-5.4-mini` as a non-reasoning synthetic Copilot model. A meaningful requested capability remains, so this should not be closed as implemented.

## Best Possible Solution

Keep this issue open but narrow the remaining work: treat the display consistency portion as mostly handled by current profile-driven session/default plumbing and #63418, then either add verified GitHub Copilot xhigh support for the requested model ids in `extensions/github-copilot/index.ts` with provider/profile tests, or document/close a narrower follow-up if those Copilot models do not actually support xhigh.
