The report is recent, concrete, and current main still contains the same gateway/session/model/control-plane surfaces implicated by the logs; related open trackers cover important slices, but no single canonical item fully preserves this post-2026.4.23 release-to-release rollback evidence.

## What This Changes

The issue reports a regression where OpenClaw releases after 2026.4.23 can saturate the Gateway event loop and make sessions.list, models.list, node.list, and Control UI surfaces take 10-60 seconds, while rollback to 2026.4.23 restores stability.

## Best Possible Solution

Keep this issue as the broad operator evidence thread, then coordinate narrow fixes through the existing focused trackers: #75287 for provider/plugin reload and Control UI polling, #74328 for the broader gateway event-loop regression, #74345 for ACP cleanup/event-loop monopolization, and #57715/#75236 for sessions.list scaling and pagination.

Yes for the reported environment: the issue gives concrete version-switch evidence, logs, RPC timings, diagnostic liveness metrics, and state-size data. A standalone current-main reproduction remains unproven without a comparable large session store and live gateway run.

## Solution Assessment

Unclear. The best path is not to collapse this into one speculative patch, but to preserve the cross-version evidence while maintainers land or reject focused repairs in the implicated gateway, plugin catalog, ACP maintenance, sessions.list, and UI polling paths.
