## Summary

Keep open. The report is fresh, detailed, and current main at 3c485103 only shows partial mitigations for runtime-deps and model-catalog/session-list pressure; it does not prove the post-2026.4.23 gateway event-loop saturation is fixed or fully covered by one canonical tracker.

## What This Changes

The issue reports a post-2026.4.23 Gateway regression where large session/model/control UI surfaces saturate the event loop and make `sessions.list` and `models.list` very slow, while rollback restores stability.

## Best Possible Solution

Keep this open until maintainers pick a canonical tracker or bisect the 2026.4.23-to-current range with a large multi-agent session store, then land focused fixes for the Gateway session/model/control-plane hot paths and verify the same state no longer saturates the event loop.
