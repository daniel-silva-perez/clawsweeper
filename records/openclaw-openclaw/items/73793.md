## Summary

Keep this issue open. Current main still has the reported `plugins.entries` cache-bypass gate in `resolvePluginCapabilityProviders`, and the same author has an open, unmerged PR (#73794) with closing syntax for this issue. Policy keeps the issue open until that PR, or an equivalent fix, lands.

## Best Possible Solution

Keep this issue open while PR #73794 is reviewed and either landed or replaced with an equivalent core fix. The likely fix belongs in `src/plugins/capability-provider-runtime.ts`: reuse active non-memory/non-speech capability providers even when `cfg.plugins.entries` is non-empty, while preserving memory and speech config reconciliation and adding a regression test for active `mediaUnderstandingProviders` with explicit plugin entries.
