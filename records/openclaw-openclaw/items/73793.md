## Summary

Keep this issue open. Current main still has the reported `hasExplicitPluginConfig(params.cfg?.plugins)` guard in the non-memory/non-speech active-provider fast path, and non-empty `plugins.entries` still makes that guard false. The same author also has open PR #73794 with closing syntax for this issue, so policy says the issue should stay open until that PR, or an equivalent fix, lands.

## Best Possible Solution

Keep this issue open while PR #73794 is reviewed and either landed or replaced with an equivalent core fix. The likely code path is `src/plugins/capability-provider-runtime.ts`: reuse active non-memory/non-speech capability providers even when `cfg.plugins.entries` is non-empty, preserve memory and speech config reconciliation, and add a regression test covering an active non-speech provider with explicit plugin entries.
