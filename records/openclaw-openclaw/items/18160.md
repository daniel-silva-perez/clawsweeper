## Summary

Keep open. Current main still has no zero-LLM cron exec payload: the cron type union, Gateway schema, job validation, runtime dispatch, CLI, UI, and docs all expose only `systemEvent` or `agentTurn`. The related implementation PR #51276 is closed unmerged, and the recent comments add production evidence plus an external workaround rather than resolving the core request.

## What This Changes

Request an additive cron mode that runs a configured shell command directly on schedule, captures stdout/stderr, updates cron run state, and optionally delivers output without starting an LLM agent turn.

## Best Possible Solution

Keep this as the canonical core tracker and land an additive, owner-reviewed cron payload kind for deterministic command execution, with Gateway schema support, runtime dispatch, run-log/error accounting, delivery behavior, CLI/UI/docs coverage, focused regression tests, and explicit secops policy for scheduled host commands.
