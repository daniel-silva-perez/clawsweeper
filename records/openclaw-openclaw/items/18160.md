## Summary

Keep open. Current main still lacks the requested zero-LLM cron exec payload, and the latest release tag has the same limitation; the issue remains a valid core cron feature with unresolved host-command security policy rather than stale cleanup.

## What This Changes

Add a direct cron exec payload/CLI mode that runs scheduled shell commands without an agent turn, captures output, records run state, and optionally delivers output to channels.

## Best Possible Solution

Keep this as the canonical core tracker and land an additive, owner-reviewed cron command payload with Gateway protocol/schema updates, scheduler dispatch, run history/accounting, delivery behavior, CLI and Control UI support, docs, focused tests, and a documented host-command security model.

## Reproduction Assessment

Yes. A high-confidence current-main check is to validate or create a cron job with payload.kind="exec"; the source shows the schema has no such variant, validation rejects non-agentTurn isolated jobs, and detached runtime dispatch has no direct execution branch.

## Solution Assessment

Yes, the requested additive core cron payload is the right direction for deterministic scheduled shell work because external cron or the linked ollama-exec-shim workaround cannot satisfy OpenClaw's cron store, run history, retry accounting, and channel delivery contract. The exact implementation should wait for maintainer and secops policy review.
