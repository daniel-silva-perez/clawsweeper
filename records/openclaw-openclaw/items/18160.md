## Summary

Keep this issue open. Current main still does not implement direct zero-LLM cron execution: cron payloads, Gateway validation, job validation, runtime dispatch, CLI add/edit, Control UI, and docs remain built around `systemEvent` and `agentTurn`. The related implementation PR #51276 is closed unmerged in the provided GitHub context, and the maintainer/Codex review comment from April 26, 2026 explicitly kept this thread as the canonical tracker after duplicates were consolidated here.

## Best Possible Solution

Keep this issue open as the canonical core tracker. The best path is an additive core cron exec payload that updates cron types, Gateway schema/validators, normalization, runtime dispatch, CLI add/edit, Control UI, docs, and focused tests, with a clear host-command security policy and integration with existing run history, retries, failure alerts, timeout/cancellation, and delivery behavior.
