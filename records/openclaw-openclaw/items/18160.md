## Summary

Keep open. Current main still has only `systemEvent` and `agentTurn` cron payloads across the core type contract, Gateway schema, validation, runtime dispatch, CLI, Control UI, and docs; the later discussion adds production datapoints and an interim external shim, but no merged or shipped OpenClaw direct-exec cron implementation.

## What This Changes

The issue requests an additive cron payload mode that runs scheduled shell commands directly, captures stdout/stderr, and optionally delivers output to channels without starting an agent or LLM turn.

## Best Possible Solution

Keep this as the canonical core tracker and land an additive, owner-reviewed cron command payload that updates the protocol contract, scheduler runtime, run history/accounting, delivery path, CLI, Control UI, docs, and tests with an explicit host-command security policy.
