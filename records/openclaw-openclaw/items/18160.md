## Summary

Keep open. Current main still does not provide a zero-LLM cron exec payload: cron types, Gateway schemas, job validation, runtime dispatch, CLI, UI, and docs all expose only systemEvent or agentTurn paths. The discussion has also consolidated multiple duplicate shell/exec cron requests here, and the related implementation PR is closed unmerged.

## What This Changes

Add a direct cron execution mode that runs a shell command on schedule, captures stdout/stderr, and optionally delivers the result through OpenClaw without creating an LLM agent turn.

## Best Possible Solution

Keep this as the canonical core tracker and land an additive, owner-reviewed cron payload kind for deterministic command execution with explicit host-command policy, Gateway schema support, runtime dispatch, run-log/error accounting, delivery behavior, CLI/UI/docs coverage, and focused regression tests.
