## Summary

Keep this issue open. Current main still has no direct zero-LLM cron exec payload: the core cron type/schema only allows systemEvent and agentTurn, validation rejects non-agentTurn isolated/current/session jobs, detached cron execution still calls the isolated agent runner, and the CLI/UI/docs expose only the existing agent-mediated surfaces. The related implementation PR #51276 is closed unmerged, while several duplicate direct-exec cron requests were already consolidated into this thread.

## Best Possible Solution

Keep this issue open as the canonical core tracker. The best implementation path is an additive cron payload kind for direct command execution, designed in core because it touches cron persistence, Gateway protocol validation, job validation, runtime dispatch, CLI add/edit, Control UI, docs, run history, retries, failure alerts, and delivery. It should include focused tests plus an explicit host-command security policy reviewed by the cron and secops owners.
