## Summary

Keep open. Current main and the latest shipped release still only support cron `systemEvent` and `agentTurn` payloads through the type contract, Gateway schema, validation, runtime dispatch, CLI, Control UI, and docs. The issue discussion adds multiple production datapoints, a closed unmerged implementation PR, and an external shim workaround, but none of those provide a merged zero-LLM cron exec path in OpenClaw core.

## What This Changes

Add a direct cron command payload that runs scheduled shell/script commands without an LLM turn, captures output/status, and optionally delivers results through OpenClaw cron delivery.

## Best Possible Solution

Keep this as the canonical core tracker and land an additive, owner-reviewed cron command payload with a documented host-command security model, protocol/schema updates, scheduler dispatch, run history/accounting, delivery behavior, CLI and Control UI support, docs, and focused regression tests.
