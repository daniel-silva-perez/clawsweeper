## Summary

Keep open. Current main still retries only Feishu WebSocket startup failures; after `WSClient.start()` succeeds it parks until abort, so SDK retry exhaustion or a later silent stall is not supervised. The open maintainer PR #73739 still has closing syntax for this issue, so the best path is to review and land or replace that implementation candidate before closing.

## Best Possible Solution

Keep this issue open and use #73739 as the current maintainer-owned implementation candidate. The fix should live in the Feishu plugin transport, add a supervisor/backoff path that observes SDK retry exhaustion or silent stalls after startup, preserve abort cleanup, and include focused regression coverage. Close this issue only after that PR or an equivalent replacement lands on main.
