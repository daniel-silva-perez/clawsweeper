## Summary

Keep this PR open. Current main still forwards `openclaw nodes` gateway transport timeouts through raw `Number(opts.timeout ?? 10_000)` in the active runtime path, so invalid values are not yet validated with the shared helper. The PR remains a useful implementation candidate, but it needs rebasing onto the current `rpc.runtime.ts` split and should address the existing review concern about `parseInt` changing inputs such as `1e3`. Security review found no supply-chain, CI, dependency, secret, or permission changes in the proposed diff.

## Best Possible Solution

Rebase this PR onto current main, move the timeout parsing change into `src/cli/nodes-cli/rpc.runtime.ts`, and add the focused invalid-timeout regression test. While doing that, decide explicitly whether non-decimal numeric forms like `1e3` should be preserved via `Number(...)`-style parsing or rejected with a clear CLI validation error.
