## Summary

Keep open: current main still only implements partial spawned-session lineage. The persistent session model, gateway schema/projections, agent-facing sessions_list rows, and native/ACP spawn writes still lack the requested rootSessionKey and session-chain traceId contract; related PR #11043 is closed and unmerged, and the issue discussion already points toward a focused replacement.

## What This Changes

The issue requests optional first-class session-chain metadata for spawned sessions, including parent/root session keys, trace IDs, and spawn depth in persistence, spawn writes, and session-list outputs.

## Best Possible Solution

Land a focused additive core contract that persists optional rootSessionKey and session-chain traceId, derives them consistently for native and ACP child sessions, exposes them through gateway and agent session-list projections, documents the fields, and proves multi-hop reconstruction while preserving compatibility for existing entries.

## Repair Work Prompt

Implement the Phase 1 session-chain metadata requested by openclaw/openclaw#11040. Add optional rootSessionKey and session-chain traceId to the persisted SessionEntry model and the gateway/tool projections while preserving backward compatibility for existing sessions. In native subagent spawn and ACP spawn, derive parentSessionKey from the requester, rootSessionKey from the parent root or parent key, traceId from the parent trace or a new stable UUID, and spawnDepth from parent depth + 1. Keep the PR focused on this metadata contract only; do not mix lifecycle hooks, provider behavior, or unrelated UI work from closed PR #11043. Update docs for subagents/session tools if public output changes, and add regression tests covering multi-hop reconstruction in both native and ACP paths. Changelog is likely needed because this adds user-visible API/tool fields; credit the original reporter appropriately, not Codex/OpenClaw.
