## Summary

Keep open because current main still only provides partial spawned-session lineage. The code exposes spawnedBy, parentSessionKey, spawnDepth, and childSessions in some paths, but it still lacks the requested first-class rootSessionKey and session-chain traceId contract in the persistent model, gateway schema/projections, agent-facing sessions_list output, and native/ACP spawn writes. The linked implementation PR is closed and unmerged, and the maintainer discussion already points the same way.

## What This Changes

Request to add optional parent/root/trace/depth session-chain metadata to persisted sessions, spawn write paths, and sessions.list/sessions_list rows so spawned-session DAGs can be reconstructed without external logging.

## Best Possible Solution

Land a focused additive core session-chain contract: persist optional rootSessionKey and traceId, derive them consistently for native and ACP child sessions, expose them through gateway and agent session-list projections, document the fields, and cover multi-hop reconstruction while preserving compatibility for older entries.
