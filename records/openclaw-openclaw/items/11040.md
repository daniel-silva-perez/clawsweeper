## Summary

Keep open: current main still implements only partial spawned-session lineage. The stored session entry, gateway protocol/list rows, agent-facing sessions_list output, and native/ACP spawn write paths expose spawnedBy/parent/depth in places, but they do not implement the requested durable rootSessionKey plus session-chain traceId contract; related PR #11043 is closed and unmerged.

## What This Changes

Add first-class session/task chain metadata so spawned sessions persist and expose parent, root, trace, and depth information for deterministic chain reconstruction.

## Best Possible Solution

Land a focused additive core contract: persist optional rootSessionKey and traceId, derive them consistently for native and ACP spawned sessions, expose parent/root/trace/depth through gateway and agent session-list projections, document the fields, and prove multi-hop reconstruction while preserving compatibility for existing sessions.
