## Summary

Keep open. Current main still only has partial spawned-session lineage: parent/depth metadata exists in several session surfaces, but the requested persisted root session key and stable chain trace identifier are absent from the session model, Gateway protocol/list rows, agent-facing sessions_list output, and native/ACP spawn writes. Related PR #11043 is closed and unmerged, so it does not resolve the remaining work.

## What This Changes

Add first-class optional parent/root/trace/depth metadata for spawned sessions so OpenClaw can reconstruct multi-hop session/task chains from session APIs.

## Best Possible Solution

Land a focused additive core contract that persists optional rootSessionKey and traceId, derives them consistently for native and ACP child sessions, exposes them through Gateway and agent session-list projections, documents the fields, and proves multi-hop reconstruction while preserving compatibility for older session entries.
