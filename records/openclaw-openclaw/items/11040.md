## Summary

Keep open: current main still has only partial spawned-session lineage. The model, gateway schema/projections, agent-facing list rows, and native/ACP spawn writes expose parent/depth fields in places, but not the requested durable root-session and session-chain trace contract; the linked implementation PR is closed and unmerged.

## What This Changes

Add optional first-class session-chain metadata so spawned sessions persist and expose parent, root, trace, and depth information for deterministic chain reconstruction.

## Best Possible Solution

Land a focused additive core session-chain contract: persist optional rootSessionKey and traceId, derive them consistently for native and ACP child sessions, expose parent/root/trace/depth through gateway and agent session-list projections, document the fields, and add multi-hop regression coverage while preserving compatibility for older entries.

## Reproduction Assessment

Not applicable as a runtime bug report, but there is a high-confidence source-level reproduction: inspect current main's session model, list projections, and spawn write paths, and the requested rootSessionKey plus session-chain traceId fields are absent.

## Solution Assessment

Yes: the best fix is a narrow core contract change because this metadata must be persisted and projected by the session APIs that reconstruct chains. Dedicated traversal APIs and UI visualization should remain follow-up work.
