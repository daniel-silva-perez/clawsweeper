## Summary

Keep open: current main still has only partial spawned-session lineage. The persistent session entry, gateway protocol/list rows, agent-facing sessions_list output, and native/ACP spawn writes expose spawnedBy, parentSessionKey, spawnDepth, or childSessions in places, but they do not implement the requested durable rootSessionKey plus session-chain traceId contract; related PR #11043 is closed and unmerged.

## What This Changes

Add first-class optional parent/root/trace/depth metadata for spawned sessions so session APIs can reconstruct multi-hop execution chains.

## Best Possible Solution

Land a focused additive core contract: persist optional rootSessionKey and traceId, derive them consistently for native and ACP child sessions from the parent chain, expose parent/root/trace/depth through gateway and agent session-list projections, document the fields, and cover multi-hop reconstruction while preserving older session entries.

## Reproduction Assessment

Not applicable as a runtime bug report, but there is a high-confidence source-level reproduction of the missing feature: inspect sessions.list/session_list projections and spawn write paths on current main and the requested rootSessionKey/traceId fields are absent.

## Solution Assessment

Yes: the maintainable path is a narrow additive core session contract because the missing data belongs in persisted session entries, gateway protocol surfaces, tool projections, and native/ACP spawn writes. Dedicated traversal APIs and UI graphing should remain Phase 2 follow-ups.
