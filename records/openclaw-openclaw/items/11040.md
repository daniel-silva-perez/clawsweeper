## Summary

Keep open: current main still has only partial spawned-session lineage. The persistent session model, gateway schema/projections, agent-facing sessions_list output, and native/ACP spawn writes expose spawnedBy, parentSessionKey, and spawnDepth in places, but they do not implement the requested durable rootSessionKey plus session-chain traceId contract; the related implementation PR is closed and unmerged, and the maintainer discussion already reached the same keep-open conclusion.

## What This Changes

The issue requests first-class optional parent/root/trace/depth metadata for spawned sessions so OpenClaw can reconstruct multi-hop session/task chains through session APIs.

## Best Possible Solution

Land a focused additive core session-chain contract: persist optional rootSessionKey and traceId on session entries, derive them consistently for native and ACP child sessions, expose parent/root/trace/depth through gateway and agent session-list projections, document the fields, and add multi-hop reconstruction coverage while preserving compatibility for existing sessions.

## Reproduction Assessment

Not applicable as a runtime bug report, but there is a high-confidence source-level reproduction: inspect current main's session model, list projections, and spawn write paths, and the requested rootSessionKey plus session-chain traceId fields are absent.

## Solution Assessment

Yes: a narrow core contract change is the best path because this metadata must be persisted and projected by the session APIs that reconstruct chains. Dedicated chain traversal APIs and UI visualization should stay follow-up work.
