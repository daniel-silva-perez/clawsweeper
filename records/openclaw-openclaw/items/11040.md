## Summary

Keep open. Current main still has only partial spawned-session lineage: `spawnedBy`, `parentSessionKey`, `spawnDepth`, and child-session relationships appear in some paths, but the requested persisted `rootSessionKey` and session-chain `traceId` contract is absent from the session model, Gateway protocol/projections, agent-facing `sessions_list` rows, and native/ACP spawn writes. The linked implementation PR is closed and unmerged, and the existing maintainer discussion already points to keeping this work open.

## What This Changes

Request adds first-class optional session/task chain metadata for spawned sessions, including parent, root, trace, and depth fields exposed through spawn and listing APIs.

## Best Possible Solution

Land a focused additive core contract that persists optional `rootSessionKey` and session-chain `traceId`, derives them consistently for native and ACP child sessions, exposes them through Gateway and agent session-list projections, documents the fields, and proves multi-hop reconstruction while preserving compatibility for older entries.
