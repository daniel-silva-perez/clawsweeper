## Summary

Current main still does not implement the requested Phase 1 session-chain contract. It has direct lineage pieces through `spawnedBy`, `parentSessionKey`, `spawnDepth`, and `childSessions`, but the persistent session model, gateway protocol/list rows, agent-facing `sessions_list`, and native/ACP spawn paths still lack `rootSessionKey` and a session-chain `traceId`. The directly related implementation PR #11043 is closed and unmerged, so this remains actionable core follow-up.

## Best Possible Solution

Keep this open and implement the minimal backward-compatible Phase 1 contract in core: persist optional `parentSessionKey`, `rootSessionKey`, `traceId`, and `spawnDepth` on spawned sessions; derive them consistently in native and ACP spawn paths; expose them through gateway schema/list rows/events and `sessions_list`; update docs; and add multi-hop regression tests. If maintainers decide root/trace lineage is not a core contract, close only with that explicit product decision.
