## Summary

Keep open: current main still implements only partial spawned-session lineage. The session model, Gateway schema/projections, agent-facing sessions_list output, and native/ACP spawn write paths expose spawnedBy, parentSessionKey, and spawnDepth in places, but they still lack the requested durable rootSessionKey and session-chain traceId contract; the only linked implementation PR is closed and unmerged.

## What This Changes

Add first-class session/task chain metadata so spawned sessions persist and expose parent, root, trace, and depth information for deterministic DAG reconstruction.

## Best Possible Solution

Land an additive core session-chain contract: persist optional rootSessionKey and traceId on session entries, derive them consistently for native and ACP child sessions, expose parent/root/trace/depth through Gateway and agent list projections, document the fields, and cover multi-hop reconstruction while preserving compatibility for existing sessions.

## Solution Assessment

Yes: the issue's Phase 1 additive metadata contract is the narrowest maintainable fix because chain reconstruction depends on persisted session metadata and existing session APIs. Dedicated chain traversal APIs and UI graphing should remain follow-up work.
