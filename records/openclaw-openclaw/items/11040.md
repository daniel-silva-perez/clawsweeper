## Summary

Keep open: current main still has only partial spawned-session lineage and lacks the requested persisted `rootSessionKey` and session-chain `traceId` contract in the session model, gateway schema/projections, agent-facing `sessions_list` rows, and native/ACP spawn writes. ## What This Changes

Add first-class spawned-session chain metadata so OpenClaw can expose parent, root, trace, and depth lineage for deterministic task DAG reconstruction.

## Best Possible Solution

Land a focused additive core contract that persists optional `rootSessionKey` and session-chain `traceId`, derives them consistently for native and ACP child sessions, exposes them through Gateway and agent session-list projections, documents the fields, and proves multi-hop reconstruction while preserving compatibility for existing entries.
