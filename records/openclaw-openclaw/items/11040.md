## Summary

Keep open. Current main still has only partial spawned-session lineage: parent/depth metadata exists in some session surfaces, but the requested root session and stable chain trace contract is absent from persistence, Gateway protocol/list projections, agent-facing `sessions_list`, and native/ACP spawn writes. The linked implementation PR is closed and unmerged, and maintainer discussion already kept this open for follow-up.

## What This Changes

Request first-class session/task chain metadata so spawned sessions expose parent, root, trace, and depth fields for deterministic multi-hop reconstruction.

## Best Possible Solution

Land a focused additive core contract that persists optional `rootSessionKey` and `traceId`, derives them consistently for native and ACP child sessions, exposes them through Gateway and agent session-list rows, documents the fields, and covers multi-hop reconstruction without breaking older session entries.
