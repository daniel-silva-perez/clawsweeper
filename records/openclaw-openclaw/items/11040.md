## Summary

Current main still does not implement the requested Phase 1 session-chain contract. It has partial direct lineage (`spawnedBy`, `parentSessionKey`, `spawnDepth`, `childSessions`), but no persisted `rootSessionKey`, no session-chain `traceId`, no gateway/session-list contract for those fields, and no agent-facing `sessions_list` output for the full parent/root/trace/depth set. The directly related implementation PR #11043 is closed and unmerged, so this remains actionable core follow-up rather than cleanup-close material.

## Best Possible Solution

Keep this open and implement the minimal backward-compatible Phase 1 contract in core: persist optional `parentSessionKey`, `rootSessionKey`, `traceId`, and `spawnDepth` on spawned sessions; derive them in native and ACP spawn paths; expose them through gateway schema/list rows/events and `sessions_list`; update docs; and add multi-hop regression tests. If maintainers decide root/trace lineage is not a core contract, close explicitly with that product decision rather than as cleanup.
