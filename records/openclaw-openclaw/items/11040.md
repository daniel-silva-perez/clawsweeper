## Summary

Keep open because current main still implements only partial spawned-session lineage. The persistent session model, gateway protocol/list projection, agent-facing sessions_list rows, and native/ACP spawn write paths expose spawnedBy/parentSessionKey/spawnDepth in places, but they still do not provide the requested first-class rootSessionKey and traceId contract; the related implementation PR is closed and unmerged, and the existing maintainer review comment reached the same result.

## What This Changes

Request first-class session/task chain metadata for spawned sessions, including parent/root/trace/depth fields in persisted sessions and session listing APIs so execution DAGs can be reconstructed without external loggers.

## Best Possible Solution

Add optional persisted rootSessionKey and traceId alongside the existing parent/depth metadata, derive them consistently for native and ACP spawns, project them through gateway and agent session-list outputs, document the contract, and cover multi-hop reconstruction while preserving compatibility for older sessions.
