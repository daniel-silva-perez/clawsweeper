## Summary

Keep open. The existing ClawSweeper P2 finding is supported by current main source and tests: the PR’s early materialization proof is narrower than the current config-dependent runtime-deps plan, so it can skip a required manifest-declared dependency. The protected `maintainer` label also means this should not be cleanup-closed.

## What This Changes

The PR adds a pre-scan fast path in `ensureBundledPluginRuntimeDeps` that skips all bundled manifest scanning when requested package-level deps plus mirrored root deps appear materialized, with a regression test asserting an unrelated bundled plugin manifest is not read.

## Best Possible Solution

Keep the scan-avoidance optimization, but make the early proof cover the full dependency set required for the requested plugin under the current config, including manifest-declared runtime dependencies, or fall through to the existing full package-level plan when that cannot be proven cheaply.

## Reproduction Assessment

Yes. A high-confidence reproduction is a packaged `memory-core` fixture with local memory search configured, generated manifest and installed packages satisfying `chokidar`/`typebox`, and `node-llama-cpp` absent; the PR can return before the current manifest scan that would add and install `node-llama-cpp`.

## Solution Assessment

No. The fast-path shape is reasonable for package.json-only plugins, but as proposed it is not the safest maintainable solution because it proves fewer dependencies than current main can require for the requested plugin.
