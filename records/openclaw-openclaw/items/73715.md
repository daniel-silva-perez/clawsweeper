## What This Changes

The issue asks `infer model run --model <provider>/<model>` to either canonicalize case-only model-id mismatches or reject them before provider dispatch with a clear unknown-model error.

## Best Possible Solution

Review and consolidate #73717 and #74945, then land a catalog-aware fix that canonicalizes unique known model matches before both local and gateway dispatch while preserving or clearly rejecting unmatched custom refs, with regression coverage in `src/cli/capability-cli.test.ts`.

## Reproduction Assessment

Yes. The issue provides a deterministic CLI matrix, and current source inspection reproduces the mechanism without a live provider: local dispatch forwards raw `params.model`, gateway parsing preserves sliced model casing, and Anthropic runtime resolution can keep the mixed-case model identity.

## Solution Assessment

Unclear until a PR is chosen. A unique catalog-match canonicalization path is the safer maintainable solution; broad lowercasing is riskier for custom mixed-case ids.
