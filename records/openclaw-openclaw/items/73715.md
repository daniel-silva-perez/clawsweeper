## What This Changes

The issue requests that `infer model run --model <provider>/<model>` either canonicalize case-only model-id mismatches before dispatch or reject them upfront with a clear unknown-model error.

## Best Possible Solution

Review and consolidate #73717 and #74945, then land a narrow catalog-aware fix that canonicalizes unique known model matches before local and gateway dispatch while preserving or explicitly rejecting unmatched custom refs, with regression coverage in `src/cli/capability-cli.test.ts`.

## Reproduction Assessment

Yes. The issue provides a deterministic CLI matrix, and source inspection reproduces the mechanism without a live provider: local dispatch forwards raw `params.model`, gateway dispatch preserves sliced casing, and Anthropic runtime resolution can keep the mixed-case model identity.

## Solution Assessment

Unclear until a PR lands. A unique catalog-match canonicalization path looks like the safest maintainable solution; broad lowercasing is riskier for custom mixed-case provider model ids.
