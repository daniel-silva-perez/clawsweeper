## Summary

Keep open. Current main still forwards explicit `infer model run --model` overrides without a catalog-aware case canonicalization step on either local or gateway dispatch, and the linked PR #73717 is open/unmerged with closing syntax for this issue. That PR is the implementation candidate, not a reason to close before merge.

## Best Possible Solution

Keep this issue open and review #73717 or an equivalent patch. The fix should canonicalize unique case-insensitive catalog matches for explicit `infer model run --model` refs before both local and gateway dispatch, preserve custom mixed-case refs when there is no unique catalog match, and add focused regression tests in `src/cli/capability-cli.test.ts`.
