## Summary

Current main still appears to preserve mixed-case model IDs through the `infer model run` dispatch path, and the provided GitHub context shows an open, unmerged PR (#73717) with closing syntax for this issue. Per policy, this should stay open until that PR is reviewed/merged or closed.

## Best Possible Solution

Keep this issue open and review #73717. The best fix is to canonicalize unique case-insensitive catalog matches for explicit `infer model run --model` refs before local and gateway dispatch, while preserving custom mixed-case model IDs when there is no unique catalog match, with regression tests for Anthropic mixed-case refs.
