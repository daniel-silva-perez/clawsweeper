## Summary

Keep open. Current main still forwards mixed-case `infer model run --model` values through the local and gateway dispatch paths, and two open unmerged PRs already reference this issue with closing syntax, so the issue should remain open until maintainers review and land or close those implementation candidates.

## What This Changes

The issue requests that `infer model run --model <provider>/<model>` either canonicalize case-only known model matches before dispatch or reject case-mismatched model ids with a clear unknown-model error.

## Best Possible Solution

Land a narrow catalog-aware resolver for explicit `infer model run` overrides that dispatches canonical provider/model ids only when a unique known case-insensitive match exists, preserves unmatched custom refs, and adds regression coverage for both local and gateway dispatch.

## Reproduction Assessment

Yes. The issue provides a deterministic CLI matrix, and source inspection confirms the mechanism without a live provider: local dispatch forwards raw `params.model`, gateway dispatch preserves sliced casing, and the Anthropic runtime can retain the mixed-case model identity.

## Solution Assessment

Unclear for the open issue because no fix is merged yet and the open PRs propose different scopes. The catalog-aware unique-match approach is the narrower maintainable solution than unconditional lowercasing because third-party or custom providers may expose mixed-case ids.
