## Summary

Keep this issue open. Current main still preserves the user-supplied model casing through `infer model run`, and open PR #73717 explicitly tracks the fix with closing syntax, so this should remain open until that PR is reviewed and merged or closed.

## Best Possible Solution

Keep this issue open and review PR #73717. The best fix is to canonicalize explicit `infer model run --model` overrides only when the loaded catalog has a unique case-insensitive provider/model match, dispatching the canonical ref for both local and gateway transports while preserving custom mixed-case refs with no unique catalog match. Add regression tests for `Anthropic/CLAUDE-OPUS-4-7`, `anthropic/Claude-Opus-4-7`, and a custom mixed-case ref that must remain unchanged.
