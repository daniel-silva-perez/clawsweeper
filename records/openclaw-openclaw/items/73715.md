## Summary

Keep open. Current main still preserves explicit mixed-case `infer model run --model` values on both local and gateway paths, and the provided GitHub context shows two open, unmerged PRs with closing syntax for this issue, so this should stay open until maintainers review and land or close the implementation candidates.

## What This Changes

The report asks OpenClaw to stop forwarding case-mismatched `infer model run --model` refs to providers and either canonicalize known model ids or reject them before dispatch.

## Best Possible Solution

Review and consolidate the linked implementation PRs, then land a catalog-aware fix that canonicalizes unique known model matches before both local and gateway dispatch while preserving or explicitly rejecting unmatched custom refs, with regression coverage in `src/cli/capability-cli.test.ts`.

## Reproduction Assessment

Yes. The issue provides a deterministic CLI matrix, and source inspection reproduces the mechanism without a live provider: local dispatch forwards raw `params.model`, gateway dispatch preserves sliced casing, and Anthropic runtime resolution can keep the mixed-case model identity.

## Solution Assessment

Unclear for the open issue because there is no merged fix yet and two PRs propose different behavior. A unique catalog-match canonicalization path is the narrowest maintainable solution; broad lowercasing is riskier for custom mixed-case ids.
