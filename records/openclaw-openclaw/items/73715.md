## Summary

Keep this issue open. Current main still forwards explicit mixed-case `infer model run --model` refs without a catalog-aware canonicalization step, and the provided thread context shows an open, unmerged PR (#73717) with closing syntax for this issue. Per policy, that PR is an implementation candidate, not a reason to close the issue before merge.

## Best Possible Solution

Keep this issue open and review #73717. The best fix is to canonicalize unique case-insensitive catalog matches for explicit `infer model run --model` refs before both local and gateway dispatch, while preserving custom mixed-case model IDs when there is no unique catalog match, with regression coverage in `src/cli/capability-cli.test.ts`.
