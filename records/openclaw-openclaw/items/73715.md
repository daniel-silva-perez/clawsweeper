## Summary

Keep this issue open. Current main still lacks the catalog-aware canonicalization needed for explicit mixed-case `infer model run --model` refs, and the supplied GitHub context shows open PR #73717 with closing syntax for this issue, so the issue should stay open until that PR is reviewed and either merged or closed.

## Best Possible Solution

Keep this issue open and review #73717. The best fix is to canonicalize unique case-insensitive catalog matches for explicit `infer model run --model` refs before both local and gateway dispatch, preserve custom mixed-case refs when no unique catalog match exists, and add regression coverage in `src/cli/capability-cli.test.ts`.
