## Summary

Keep this PR open. Current main has the shared chat session/model/thinking controls, and it now groups session options by agent, but it still does not render the PR's dedicated agent selector in the chat controls. The PR remains a plausible implementation candidate, but it needs a rebase onto the current `ui/src/ui/chat/session-controls.ts` split and should preserve the current gateway-backed thinking option/default behavior.

## Best Possible Solution

Keep this PR open as a useful implementation candidate. Rebase it onto current main by adding the dedicated agent selector to `ui/src/ui/chat/session-controls.ts` and the mobile chat-control path, keep the existing session/model/thinking selector behavior, build agent main-session keys from a stripped main key rather than an already-prefixed session key, preserve the empty-agent disabled guard, and add focused desktop/mobile UI tests for rendering and agent switching.
