## Summary

Keep this PR open. Current main already has the chat header model and thinking selectors, but it still does not implement the PR's central agent selector in the desktop/mobile chat controls. The PR also appears stale against current main's session-control split and the latest review feedback: its thinking selector path would drop session-advertised thinking levels/defaults that current main now preserves.

## Best Possible Solution

Keep this PR open as a plausible implementation candidate. Rebase it onto current main by adding the agent selector to `ui/src/ui/chat/session-controls.ts` and the mobile dropdown path, preserve the existing gateway-backed thinking level/default logic, fix agent session-key synthesis for legacy/non-agent sessions, keep the empty-agent disabled guard, and add focused UI tests for desktop/mobile rendering and agent switching.
