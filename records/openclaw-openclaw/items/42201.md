## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: unknown `openclaw.json` keys still pass into strict Zod validation and become fatal config errors on the read/startup path. The PR is still a plausible implementation candidate, but it needs a current-main rebase or narrower rewrite plus config-safety and baseline review.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Rebase or rewrite it narrowly against current main, implementing sanitized warning-based tolerance for unknown read/startup config keys while preserving fail-closed behavior for invalid known fields, legacy keys, plugin schemas, and union/discriminator mismatches. Review `.secrets.baseline` separately, then run focused config validation/write/legacy coverage plus the required changed gate for the final touched files.
