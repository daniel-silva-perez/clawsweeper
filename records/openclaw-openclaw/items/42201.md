## Summary

Keep this PR open. Current main still does not implement the PR’s central behavior: unknown config keys still flow into strict schema validation and become fatal config errors before plugin-aware validation or startup can continue. The PR remains a plausible implementation candidate, but it needs a current-main rebase or narrower rewrite plus focused config-safety and security review.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Rebase or rewrite it narrowly against current main to add sanitized warning-based tolerance for unknown read/startup config keys while preserving fail-closed validation for invalid known fields, legacy keys, plugin schemas, unsupported secret surfaces, and union/discriminator mismatches. Review `.secrets.baseline` separately, then run focused config validation/write/legacy coverage plus the required changed gate for the final touched files.
