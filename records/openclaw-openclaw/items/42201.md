## Summary

Keep this PR open. Current main still rejects generic unknown config keys through the strict root schema and validation path, while this PR is a focused implementation candidate for tolerating unknown read/startup keys with warnings and fail-closed handling for invalid known fields, legacy keys, and union/discriminator edge cases.

## Best Possible Solution

Keep this PR open for maintainer review.
