## Summary

Keep this PR open. Current main still treats unknown OpenClaw config keys as strict schema failures on the base validation path, and load/startup still converts that invalid snapshot into INVALID_CONFIG rather than stripping or warning through. The PR remains a plausible focused implementation candidate, with useful prior review history from #40198, #40268, and #47654, but it needs a current-main rebase or narrower rewrite plus security/config-safety review before merge.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Rebase or rewrite it narrowly against current main, preserve the useful fixes already discussed in review, and land one config-validation change that strips or ignores unknown read/startup keys with sanitized warnings while keeping invalid known fields, legacy keys, plugin-owned config, and union/discriminator mismatches fail-closed. Review the .secrets.baseline delta separately and run targeted config validation/write/legacy tests plus the required changed gate for the final touched files.
