## Summary

Keep open. Current main has adjacent newer-config protections, but it still does not implement this PR's central API change: a structured version-skew result plus `versionWarning` in successful `config.set`, `config.patch`, and `config.apply` gateway responses. The provided PR context also shows review feedback about missing `config.apply` was addressed by the contributor's second commit, so this remains plausible reviewable work rather than an obsolete PR.

## Best Possible Solution

Keep this PR open for maintainer review or an updated patch against current main. The best path is to add a structured skew helper, return `versionWarning` consistently from successful `config.set`, `config.patch`, and `config.apply` responses, add focused tests for skewed and non-skewed write paths, and document the RPC response behavior while aligning guidance with the existing newer-config guard.
