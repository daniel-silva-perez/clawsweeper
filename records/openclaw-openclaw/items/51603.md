## Summary

Keep this PR open. Current main covers the chat model picker part with provider-qualified option values, but it still does not implement the PR's central top-level `llm` shorthand config key. The existing Greptile and Codex review comments are directionally consistent: the shorthand needs product/config-owner review and the proposed implementation still has correctness and credential-surface gaps before it could safely merge.

## Best Possible Solution

Keep this PR open for maintainer/product judgment or request a focused revision. If OpenClaw wants the shorthand, implement it in the config owner area with explicit type/schema/generated-schema/help support, docs, SecretInput and redaction coverage, provider/API/ref validation, non-destructive provider/model merging, fallback-preserving primary assignment, and regression tests. Retain the current-main chat model picker behavior rather than reintroducing that piece from the PR.
