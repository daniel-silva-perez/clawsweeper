## Summary

Keep this PR open. Current main already covers the chat model picker half by building provider-qualified option values and normalizing stale/bare selections, but it still does not implement the PR's central top-level `llm` shorthand config key. Because this is a PR, it should only close if current main definitely implements the intended change; it does not.

## Best Possible Solution

Keep this PR open for maintainer/product judgment or request a focused revision. If OpenClaw wants this shorthand, implement it in the config owner area with explicit type/schema/generated-schema/help support, public docs, SecretInput/sensitive metadata coverage, provider/API validation, ref-safe provider ids, non-destructive provider/model merging, fallback-preserving primary assignment, and tests for existing-provider, invalid-input, redaction, and config-load cases. Keep the current-main chat model picker behavior instead of reintroducing that part from the PR.
