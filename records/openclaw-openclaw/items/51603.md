## Summary

Keep this PR open. Current main already solves the chat model picker part by normalizing catalog options to provider-qualified refs, but it still does not implement the PR's central top-level `llm` shorthand config key. Because this is a PR, it should only be closed if main definitely implements the intended change; it does not.

## Best Possible Solution

Keep the PR open for maintainer/product judgment or request a focused revision. If OpenClaw wants the shorthand, implement it in the config owner area with explicit schema/type support, `SecretInput` handling, model-ref-safe provider ids, supported provider/API validation, non-destructive provider/model merging, regression tests for existing-provider and fallback cases, and docs for the public config surface. Retain the current-main chat model picker behavior rather than reintroducing it from this PR.
