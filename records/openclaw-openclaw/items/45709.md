## Summary

Keep open: current main still renders both the shared Usage page header and the Usage view’s local header, and the related fix PR is still open rather than merged.

## Best Possible Solution

Keep this issue open until a focused UI fix lands. The likely fix is to choose one Usage header source, either by suppressing the shared shell header for `usage` or by removing the local `usage-page-header`, and add a regression assertion that the Usage tab renders only one visible header/subtitle source.
