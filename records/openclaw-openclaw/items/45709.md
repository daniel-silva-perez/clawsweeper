## Summary

Keep open because current main still renders both the shared shell Usage title/subtitle and the local Usage page header, and the latest release v2026.4.27 contains the same duplicate path. Open PR #67290 directly targets this bug but is still unmerged, so the issue should remain available for normal maintainer review.

## What This Changes

The issue requests that the Control UI Usage tab render a single Usage header instead of duplicated shared and local header blocks in the v2 UI.

## Best Possible Solution

Land or revise #67290, or an equivalent narrow fix, so Usage has exactly one title/subtitle source. The final fix should include a focused regression check around app shell plus Usage tab composition and keep generated locale state consistent.

## Reproduction Assessment

Yes. Opening the Control UI Usage tab on current main follows a source-visible path that renders both the shared shell page title/subtitle and the local usage-page-header block.

## Solution Assessment

Unclear until maintainer review of #67290. The narrow maintainable solution is to remove one of the two header sources, with the canonical source chosen consistently with the current app shell layout convention.
