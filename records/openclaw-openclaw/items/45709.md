## Summary

Current main still has the duplicate Usage header composition: the app shell renders the shared tab header for Usage, and the Usage view renders its own local header inside the tab body. The issue should stay open while open PR #67290, or an equivalent narrow fix, is reviewed and landed.

## Best Possible Solution

Keep this issue open and review or land #67290, or replace it with an equivalent narrow UI patch. The fix should remove exactly one Usage header source, preserve the intended mobile/desktop heading behavior, keep English locale and generated locale outputs aligned, and add a focused render regression check proving the Usage tab shows a single header/subtitle source.
