## Summary

Keep this issue open. Current main still has no explicit Telegram channel_post policy for sender-scoped groups[chatId].allowFrom, the related same-author PR #49063 remains open and explicitly leaves that policy out of scope, and the current code/docs still model allowFrom as sender identity based while Telegram channel posts may not expose a human sender.

## Best Possible Solution

Keep this issue open until maintainers choose and document the Telegram channel_post policy for sender-scoped groups[chatId].allowFrom. The best fix is either explicit-chat-only authorization that ignores sender-scoped overrides for channel posts, or a fail-closed unsupported-configuration path with a clear user message and docs, plus targeted tests for native channel_post commands and group access behavior. Review or resolve PR #49063 separately because it is related implementation work but does not settle this policy.
