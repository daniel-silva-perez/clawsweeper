## Summary

Keep this issue open. Current main still does not define or document Telegram `channel_post` behavior when sender-scoped `groups[chatId].allowFrom` is configured, and the related same-author PR is still open and explicitly leaves that semantics decision out of scope.

## Best Possible Solution

Define the Telegram plugin policy explicitly: either ignore sender-scoped `groups[chatId].allowFrom` for `channel_post` and authorize only by explicit chat allow, or reject that configuration path with a clear message. Then update `extensions/telegram/src/bot-native-commands.ts` / shared group access as needed, add channel_post-focused tests, and document the rule in the public Telegram and slash-command docs. Review or close #49063 separately for the adjacent native channel command support.
