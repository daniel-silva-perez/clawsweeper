Keep open. Current main has materially better Telegram token diagnostics, channel reload behavior, polling stall recovery, status warnings, and Windows/network mitigations than the reported v2026.3.13 build, but the report still combines Telegram 401/auth, config visibility, cross-surface delivery, and sub-agent loop symptoms without current Windows logs proving the whole user problem is fixed.
