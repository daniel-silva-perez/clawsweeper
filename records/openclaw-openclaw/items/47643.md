## Summary

Keep open. Current main has materially improved Telegram token diagnostics, config reload planning, polling stall recovery, status warnings, and Windows/proxy transport handling since the reported v2026.3.13 setup, but the report still combines Telegram 401/config visibility, cross-surface delivery, Windows gateway degradation, and sub-agent/session loop symptoms without a current Windows retest proving the whole user problem is fixed. The open #73323 and closed #74086 cover important subsets, but they do not fully supersede this mixed report.

## What This Changes

## Best Possible Solution
