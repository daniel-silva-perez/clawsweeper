## Summary

Keep this PR open. Current main still preserves an explicit `--history-limit 0`, forwards it through TUI history loading to `chat.history`, and the Gateway protocol still rejects `limit: 0`; the PR is narrow, relevant, and not implemented on main.

## Best Possible Solution

Keep this PR open and review/rebase/land the narrow TUI-side guard with its regression test, or recreate the same behavior on main. The Gateway protocol should continue rejecting `chat.history.limit: 0`; the TUI should treat explicit zero as local no-history mode and avoid sending `chat.history` at all.
