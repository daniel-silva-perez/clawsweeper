## Summary

Keep open. The runtime fix is narrow and matches the current-main bug, but this user-facing TUI behavior still needs the missing public documentation and changelog entry before merge.

## What This Changes

The PR treats `openclaw tui --history-limit 0` as no-history mode by clearing local TUI history state, skipping the gateway history request, and adding regression coverage.

## Best Possible Solution

Merge the TUI-side zero-limit guard with its regression test after adding user-facing docs and an Unreleased changelog entry; the gateway protocol should keep its positive-limit contract.

## Reproduction Assessment

Yes. Current main parses `--history-limit 0`, forwards it to `chat.history`, and the gateway schema rejects limits below 1; the PR test covers the expected no-gateway-call path.

## Solution Assessment

Yes for the runtime fix. Handling zero in the TUI before calling `chat.history` is narrower than relaxing the gateway protocol for every client, but the public docs and changelog need to be updated before merge.
