## Summary

Keep this PR open. Current main still preserves `--history-limit 0`, forwards it through the TUI history loader into `chat.history`, and the Gateway protocol still rejects zero as an invalid history limit. The PR is narrow, relevant, and not implemented on main.

## Best Possible Solution

Keep this PR open and review/rebase/land the narrow TUI-side guard with its regression test, or recreate the same behavior on main. The Gateway protocol should keep rejecting `chat.history.limit: 0`; the TUI should interpret explicit zero as local no-history mode and avoid sending `chat.history` at all.
