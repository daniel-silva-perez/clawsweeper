## Summary

Keep open. Current main still forwards a zero TUI history limit into `chat.history`, while the gateway contract rejects `limit: 0`; the PR is a focused fix for that mismatch and does not appear obsolete.

## What This Changes

The PR adds a TUI `historyLimit === 0` branch that clears the local chat log, shows the session marker, refreshes session info, skips backend history loading, and covers that path with a unit test.

## Best Possible Solution

Land this branch, or an equivalent narrow TUI-side no-history handling path, while keeping the gateway `chat.history` minimum intact for real history requests and preserving the regression coverage.

## Reproduction Assessment

Yes. Current main parses `--history-limit 0`, passes it through TUI session actions, and then violates the gateway `chat.history` schema that requires `limit >= 1`.

## Solution Assessment

Yes. Treating zero as a TUI-local no-history mode before the backend call is the narrowest maintainable fix because zero is not a valid gateway page size.
