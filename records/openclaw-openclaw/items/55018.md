## What This Changes

The PR changes `src/config/schema.hints.ts` so `mapSensitivePaths` delegates recursion to a private in-place helper and documents the mutation contract instead of shallow-copying `hints` at every traversal step.

## Best Possible Solution

Land the narrow config-schema traversal performance fix after maintainer review, keeping the documented mutation contract and rerunning the targeted schema-hints test plus the normal changed gate on a refreshed branch.

## Reproduction Assessment

Yes for the code-level cause: current main still shallow-copies the growing `hints` object on every recursive `mapSensitivePaths` call. The exact 30s wall-clock report was not reproduced because this was a read-only review and no tests or benchmarks were run.

## Solution Assessment

Yes, the proposed direction is the narrowest maintainable fix: mutate a single accumulator during traversal while returning the same object for call-site convenience. The documented mutation contract addresses the only semantic difference I found.
