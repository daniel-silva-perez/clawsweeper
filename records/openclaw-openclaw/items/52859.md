## Summary

Keep open. Current main still sends Telegram inbound text through `expandTextLinks(...)` only; that helper filters for `text_link` and existing tests still expect `bold` entities to leave text unchanged, so the reported agent-prompt formatting regression remains actionable.

## Best Possible Solution

Implement this in the Telegram plugin inbound body path: convert Telegram formatting entities such as bold, italic, code, pre, strikethrough, underline, and spoiler into agent-visible Markdown before `rawBody` and `bodyText` are finalized, while preserving current `text_link` behavior. Add focused tests for plain formatted text, captions, links plus formatting, adjacent entities, and offset interactions.
