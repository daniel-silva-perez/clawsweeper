## Summary

Keep open. Current main still routes Telegram inbound text through `expandTextLinks(...)` only, and that helper deliberately ignores non-`text_link` entities such as bold and italic, so the reported agent-prompt formatting regression remains actionable in the Telegram plugin.

## Best Possible Solution

Implement the fix in the Telegram plugin inbound text path: convert Telegram formatting entities such as bold, italic, code, pre, strikethrough, underline, and spoiler into agent-visible Markdown before `rawBody`/`bodyText` are finalized, while preserving existing `text_link` behavior. Add focused tests in `extensions/telegram/src/bot-message-context.body.test.ts` or helper tests for formatted text, text links, captions, and offset interactions.
