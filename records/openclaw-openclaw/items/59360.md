## Summary

Current main still matches the reported Feishu card footer behavior: `resolveCardNote` hard-codes colon-form `Agent:`, `Model:`, and `Provider:` fields, and both streaming and structured card paths render that note as Feishu markdown. No Feishu config/docs/schema surface currently disables or reformats the footer. Related footer/customization items remain open, but they are not a shipped fix or a clearly canonical replacement for this specific Markdown parsing bug.

## Best Possible Solution

Keep this open as a concrete Feishu bundled-plugin bug. The fix should live in `extensions/feishu`: change `resolveCardNote` to a Feishu-safe non-definition-list format and/or add a documented per-channel/per-account option to hide the card note. Cover streaming and structured card paths in `reply-dispatcher.test.ts`, update `config-schema.ts`, generated config metadata, and `docs/channels/feishu.md`, and coordinate with the broader open footer customization work rather than closing this specific report yet.
