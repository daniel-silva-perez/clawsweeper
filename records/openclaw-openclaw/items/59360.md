## Summary

Keep open. Current main still constructs the Feishu card note as colon-delimited `Agent:`, `Model:`, and `Provider:` Markdown and passes it into both streaming and non-streaming card paths, with no Feishu config or docs surface to disable the footer. The broader footer customization threads do not remove this narrow bug.

## What This Changes

The issue requests a Feishu card footer option or formatting change so the hardcoded Agent/Model/Provider note no longer triggers Feishu Markdown definition-list parsing.

## Best Possible Solution

Make the Feishu plugin render card note labels in a Markdown-safe format across streaming and non-streaming cards, with focused tests covering agent/model/provider notes. Keep broader footer hiding, custom fields, token usage, and card customization work in the existing feature threads unless maintainers intentionally expand this fix.

## Reproduction Assessment

Yes for the OpenClaw side: set Feishu to card rendering and inspect or exercise the dispatcher path; current tests already assert `note: "Agent: agent"` on streaming cards. The external Feishu visual artifact was not live-verified because no Feishu/Lark credential environment names were present.

## Solution Assessment

Yes, the narrowest maintainable fix for this report is to stop using `Key: value` labels in the Feishu card note or escape them before Markdown rendering. Adding a full footer customization API would solve more requests but is broader than this parsing bug.
