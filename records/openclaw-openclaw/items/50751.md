## Summary

Keep this PR open. Current main still does not implement the PR’s intended behavior: local gateway connection details still target loopback regardless of tailnet/custom bind, the server still only listens on the resolved non-loopback bind host, and reasonless 1000/1008 gateway closes still do not get an auth-mode-aware hint. The PR remains useful, but it is not merge-ready as written because its client-side non-loopback `ws://` path conflicts with the current plaintext WebSocket security policy and its auth hint is too token-specific.

## Best Possible Solution

Keep this PR open for maintainer follow-up. If maintainers prefer the #40247 server-side fix for tailnet/custom bind reachability, this PR should narrow to a tested, mode-aware auth diagnostic. If client-side host resolution is still desired, it should be rebased into the current `src/gateway/connection-details.ts` split, preserve the plaintext WebSocket security policy, update diagnostics such as `urlSource`, and add tests for tailnet/custom binds plus token/password/trusted-proxy close-hint behavior.
