## Summary

Current main still does not implement the PR’s intended behavior, and the PR is not safe to close as obsolete. The bind mismatch remains in current code, the silent-close auth hint is still missing, and the proposed client-side non-loopback URL approach still needs security-policy handling before it would be merge-ready.

## Best Possible Solution

Keep this PR open for maintainer follow-up. #40247 appears to be the narrower server-side fix for the tailnet/custom bind mismatch; if maintainers choose that path, this PR should narrow to a tested, mode-aware auth diagnostic. If client-side host resolution is still desired, it should be implemented in the current connection-details split, preserve the existing plaintext WebSocket security policy, update diagnostics such as urlSource, and add tests for tailnet/custom binds plus token/password/trusted-proxy auth behavior.
