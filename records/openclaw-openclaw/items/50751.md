## Summary

Keep this PR open. Current main still does not implement the PR’s full intended change: local gateway connection details still target loopback, server listen-host resolution still does not add loopback alongside non-loopback bind hosts, and `formatGatewayCloseError()` still has no mode-aware hint for reasonless 1000/1008 closes. The bind portion overlaps with open #40247, but the auth diagnostic is not fully covered there, and the PR as written still needs security/test follow-up before merge.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is to let #40247 or an equivalent server-side change handle tailnet/custom loopback reachability, then either narrow this PR to a tested, mode-aware silent-close diagnostic or rebase it onto `src/gateway/connection-details.ts` with the plaintext WebSocket security policy preserved. Add tests covering tailnet/custom bind behavior and token/password/trusted-proxy close-hint behavior before merge.
