## Summary

Keep open. Current main still emits loop-detection warning logs and warning diagnostic events for generic repeated tool calls, and the before-tool-call hook still has no isolated-cron session target, elapsed lifetime, or schedule interval metadata needed to demote only bounded isolated cron runs safely.

## What This Changes

Downgrade loop-detection warning logs to info for bounded isolated cron sessions whose run lifetime is within the cron schedule interval.

## Best Possible Solution

Land a focused core/cron fix that passes explicit bounded isolated-cron context into loop-warning emission, demotes only warning-level loop logs/diagnostics for isolated cron runs whose elapsed lifetime is within their schedule interval, and preserves current critical blocking, thresholds, run-id scoping, non-cron warnings, and auditability.
