## Summary

Keep this PR open. Current main has several Slack media fixes, but it does not implement this PR's central remaining behavior: inbound Slack media hydration from partial file metadata, a media-read token fallback path, or cross-turn-safe staged media filenames. The open #41657 is related and broader, but it does not supersede this concrete PR work; the closed #50129 does not make this PR obsolete on current main.

## Best Possible Solution

Keep this PR open for maintainer review or replacement with a narrowed patch. The best path is to land a Slack-owned inbound media fix that hydrates id-only/partial Slack file metadata via `files.info`, threads a safe bot/user media-read token fallback, and prevents staged inbound basename collisions with tests, while preserving SSRF guardrails and removing unrelated test churn.
