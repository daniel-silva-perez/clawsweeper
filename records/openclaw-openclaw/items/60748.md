## Summary

Keep open. Current main only partially addresses the reported Feishu card UX by omitting the generic `main` header when no identity name is configured; the broader request remains unimplemented: no configurable header/footer schema, no elapsed/status footer rendering, no footer hiding or field toggles, and no reply-to context header mode. Related #71079 was already closed as a duplicate of this canonical tracker, while open PR #54979 covers only a narrower header-hide subset.

## Best Possible Solution

Keep this issue open as the canonical Feishu card customization tracker. The best implementation belongs in the Feishu plugin: add top-level and per-account config for header mode and footer fields, wire it through both streaming and static card reply paths, preserve backward-compatible defaults, update the plugin manifest/generated metadata and `docs/channels/feishu.md`, and add focused schema/dispatcher tests for hidden/custom/reply-to headers plus elapsed/status/footer toggles. PR #54979 can be reviewed as a narrower header subset or superseded by a broader PR against this issue.
