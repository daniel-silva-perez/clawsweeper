## Summary

Keep open. Current main still lacks the requested channel-chat workflow to list configured agents and temporarily switch the active conversation to another configured agent without editing bindings or restarting the Gateway. The existing `/agents` chat command is for subagent/session binding status, the TUI `/agent` command is local to the TUI session picker, and routing still falls through static bindings plus the default agent.

## What This Changes

The issue requests `/agents` and `/agent <name|main>` chat commands that temporarily switch the current channel conversation to another configured agent without changing global bindings or restarting the Gateway.

## Best Possible Solution

Add a Gateway/channel routing feature that authorizes configured-agent selection, lists configured agents in chat, stores a temporary per-conversation route override ahead of static binding fallback, cleanly resets or expires that override, and preserves each selected agent's workspace, auth, and memory boundaries. If this should be plugin-first, expose a documented route-override API for configured agent ids first.
