## Summary

Current main still does not implement the requested chat/Gateway workflow to list configured agents and switch the current conversation with `/agent <name>` without changing global bindings or restarting. Existing `/agents` is for thread-bound subagent/session bindings, and route selection remains binding/default driven.

## Best Possible Solution

Keep this issue open for maintainer-designed Gateway/channel routing work. A complete fix should add authorized `/agents` and `/agent <id|main>` handling that lists configured agents, stores a temporary per-conversation configured-agent route override before normal binding fallback, creates or targets the selected agent's main session as needed, and defines reset/expiry plus workspace/auth/memory isolation. If maintainers want this plugin-first, core should first expose a documented route-override API for configured agent ids.
