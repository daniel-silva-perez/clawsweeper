## Summary

Current main still does not fully resolve the reported Anthropic OAuth/model-level `anthropic-beta` regression. The Anthropic plugin beta wrapper is still gated on extra params, model-level headers still reach runtime request construction, and the direct transport still merges configured headers after the OAuth-required beta header. There is also an open same-author PR with closing intent for this issue, so the issue should remain open until that PR is reviewed, landed, replaced, or closed.

## Best Possible Solution

Keep this issue open and review, land, replace, or close PR #41461. The eventual fix should preserve `oauth-2025-04-20` independently of `params.context1m`, merge or protect OAuth-required Anthropic betas when provider/model headers include `anthropic-beta`, align with the current policy for OAuth plus `context-1m-*`, and add regression coverage for OAuth plus model-level `anthropic-beta` without `params.context1m`.
