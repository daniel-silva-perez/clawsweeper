## Summary

Keep this issue open. Current `main` still leaves the reported model-header path actionable: the Anthropic plugin wrapper that injects OAuth betas is still only installed when extra params produce Anthropic betas, and the direct Anthropic OAuth transport can still let later model/request headers replace the OAuth-required `anthropic-beta` value. The issue is also paired with open same-author PR #41461 using closing syntax, so cleanup policy says the issue should remain open until that implementation candidate is landed, replaced, or closed.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #41461. The best fix should preserve `oauth-2025-04-20` for `sk-ant-oat` requests across both the Anthropic plugin wrapper and direct transport paths, while explicitly stripping or ignoring model/provider `context-1m-*` headers for OAuth/subscription tokens if that remains the documented contract. Add regression coverage for an OAuth token with model-level `anthropic-beta: context-1m-2025-08-07` and no `params.context1m`.
