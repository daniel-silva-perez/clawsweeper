## Summary

Keep open. Current main still leaves the reported model-header path unresolved: the Anthropic beta wrapper only runs when extra params produce betas, and the direct Anthropic OAuth transport still lets later `model.headers` or `options.headers` replace the OAuth-required `anthropic-beta` value. There is also an open same-author implementation PR with closing syntax for this issue, so cleanup policy says this issue should remain open until that PR is landed, replaced, or closed.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #41461. The fix should merge Anthropic beta lists at the Anthropic wrapper and/or direct transport header boundary so OAuth-required betas survive model/provider `anthropic-beta` headers, and it should add regression coverage for `sk-ant-oat` auth with model-level `anthropic-beta: context-1m-2025-08-07` and no `params.context1m`.
