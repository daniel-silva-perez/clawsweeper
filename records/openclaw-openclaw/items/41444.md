## Summary

Keep open. Current main still leaves the reported model-header path actionable: the Anthropic plugin wrapper that injects OAuth betas is only installed when extra params produce Anthropic betas, and the direct Anthropic OAuth transport still lets later model or call headers replace the OAuth-required `anthropic-beta` value. Current docs now say OAuth/subscription tokens should skip `context-1m-*`, so the best fix should preserve `oauth-2025-04-20` while stripping or ignoring model-level `context-1m` for OAuth. There is also an open same-author implementation PR with closing syntax for this issue, so cleanup policy says this issue should remain open until that PR is landed, replaced, or closed.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #41461. The fix should merge Anthropic beta lists at the plugin wrapper and direct transport header boundaries so `oauth-2025-04-20` survives model/provider `anthropic-beta` headers, and it should strip or ignore `context-1m-2025-08-07` for `sk-ant-oat` OAuth/subscription tokens to match current docs. Add regression coverage for `sk-ant-oat` auth with model-level `anthropic-beta: context-1m-2025-08-07` and no `params.context1m`.
