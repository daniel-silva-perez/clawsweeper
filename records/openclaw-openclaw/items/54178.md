## Summary

Keep this issue open. Current main still renders the Agents page selector as a native `<select>` with only a change handler, and the recent main diff only remounts overview model controls; it does not add pointer/click behavior, replace the dropdown with an agent list, or document simulated-click automation as unsupported.

## Best Possible Solution

Keep this issue open until current main is verified with the supported browser automation path. If pointer-based automation still cannot operate the Agents selector, replace the native dropdown with an accessible listbox/sidebar/list view and add focused browser/UI coverage. If maintainers decide synthetic click-to-open on native selects is unsupported, document the supported automation path and coordinate with the broader #57067 Agents-page dropdown UX work.
