## Summary

Keep open. Greptile's documentation ambiguity was addressed in the second PR commit, but `/m` still has a blocking runtime gap: the alias is only registered for command normalization, while model switching is parsed earlier as a directive and there is no `/model` command handler fallback.

## What This Changes

The PR adds nine short built-in text aliases for existing slash commands and updates the slash-command docs, including a follow-up docs split for `/model` and `/models`.

## Best Possible Solution

Keep the alias set only after the model alias follows the directive-owned `/model` path end to end, with a focused regression test for `/m <provider/model>`; otherwise remove `/m` from the PR and leave the handler-backed aliases plus corrected docs.

## Reproduction Assessment

Yes. A high-confidence source reproduction is `/m <provider/model>` on the PR branch: directive parsing will not mark it as a model directive, and normalized `/model ...` has no command handler fallback.

## Solution Assessment

No. Registry aliases are the right mechanism for handler-backed commands, but `/model` is a directive-owned surface, so `/m` must be wired into directive parsing or omitted.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Wire `/m` through model directive parsing:** `src/auto-reply/commands-registry.shared.ts:831`
  - body: Registering `/m` only in the command registry does not make it switch models. Model selection is parsed earlier by `extractModelDirective()`, which recognizes `/model` plus configured aliases, and there is no `/model` handler after normalization. As a result, `/m gpt-4.1` falls through as ordinary text instead of applying the advertised model switch.
