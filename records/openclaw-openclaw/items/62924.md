## Summary

Keep open. Current main still preserves the actual media-understanding provider/model internally, but the assistant-visible inbound body is still formatted from only the media title, optional user text, and transcript/description text, so the reporter's guessed-model problem remains unresolved.

## What This Changes

Expose the actual media-understanding provider/model used for the current inbound media turn in the assistant-visible body, or otherwise prevent the assistant from reporting a guessed analysis model.

## Best Possible Solution

Add a narrow media-understanding formatter/apply update that renders `Analysis model:` from real successful `MediaUnderstandingOutput.provider` and `model` values, handles provider-only and synthetic placeholder outputs explicitly, and covers the assistant-visible body with focused formatter/apply tests plus a user-facing changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is available on current main: pass an output with `provider` and `model` into `formatMediaUnderstandingBody`, and the returned body contains only the transcript/description, which `applyMediaUnderstanding` then promotes into `BodyForAgent`.

## Solution Assessment

Yes. The best fix is in the media-understanding formatter/apply path, using the existing typed output telemetry; changing conversation-model fallback behavior would not expose the actual media analyzer for the current turn.
