## Summary

Keep open. Current main still carries media-understanding provider/model telemetry internally, but the assistant-visible inbound body is formatted from only the media title, optional user text, and transcript/description text. The reporter’s local Discord validation and the earlier ClawSweeper review still match the current source; this is a narrow unresolved core media-understanding formatter/apply-path gap.

## What This Changes

Expose the actual current-turn media-understanding provider/model in the inbound body so the assistant can report the media analysis model without guessing.

## Best Possible Solution

Add a narrow media-understanding formatter/apply behavior that renders `Analysis model:` from `MediaUnderstandingOutput.provider` and `model` for real successful outputs, handles missing model and synthetic placeholders explicitly, and updates focused formatter/apply tests plus a user-facing changelog entry.

Fix openclaw/openclaw#62924. Current main stores `MediaUnderstandingOutput.provider` and optional `.model`, and `runner.entries.ts` populates them, but `formatMediaUnderstandingBody` omits them before `applyMediaUnderstanding` makes the formatted body assistant-visible as `BodyForAgent`.
