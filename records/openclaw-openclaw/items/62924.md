## Summary

Keep this issue open. Current main still carries media-understanding provider/model telemetry internally, and status output can display chosen provider/model, but the assistant-visible inbound body formatter still renders only the media header, optional user text, and transcript/description text. The reporter’s local `Analysis model:` workaround and the previous Codex review remain applicable to current main.

## Best Possible Solution

Implement a scoped media-understanding body formatter/apply-path change that renders an `Analysis model:` line from successful, non-synthetic `MediaUnderstandingOutput.provider` and `.model` when available. Add focused tests in `src/media-understanding/format.test.ts` and `src/media-understanding/apply.test.ts` proving `BodyForAgent` receives the actual current-turn telemetry and that missing or synthetic telemetry is not misrepresented.

## Risks / Open Questions

- A narrow fix should expose only successful current-turn media-understanding telemetry, not the conversation model, stale fallback state, or provider/model from skipped attempts.
- Synthetic placeholders such as `provider: "openclaw", model: "synthetic-empty-audio"` are currently real outputs for too-small audio; a formatter change should avoid presenting those as a real analysis model.
