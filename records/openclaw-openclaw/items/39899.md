## Summary

Keep this PR open. Current main has partial surrogate mitigation for text payloads and failed assistant-turn replay, but it does not implement the PR's surrogate-specific detector, friendly error formatting, or runner-level session reset. A plausible gap remains for persisted non-text payloads, especially OpenAI Responses function-call arguments that are replayed without surrogate sanitization.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, address the Greptile-noted fallback-copy gap, and add focused tests for surrogate-bearing provider error formatting, runner reset behavior, OpenAI Responses function-call arguments, and Anthropic tool_use input replay. The best implementation may be either the PR's detector/reset/friendly-message path or a narrower recursive sanitizer if tests prove reset recovery is unnecessary.
