## Summary

Keep open. Current main still lacks this PR's proposed `sessions_send` field-level schema descriptions and the matching docs guidance for choosing `sessionKey` versus `label`; the existing runtime rejection only proves the behavior being documented, not that the schema/docs clarification is already implemented. ## What This Changes

This PR adds TypeBox `description` metadata to the `sessions_send` `sessionKey`, `label`, and `agentId` schema fields and adds session-tool docs guidance to use either `sessionKey` or `label`, not both.

## Best Possible Solution

Keep this PR open for maintainer review. No runtime behavior change is required for this PR's stated scope.
