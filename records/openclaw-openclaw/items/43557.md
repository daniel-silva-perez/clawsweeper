## Summary

Keep open. Current main still has Gmail/gog integration and scheduled-run docs, but it does not yet document or diagnose the reported intermittent non-interactive macOS `No auth for gmail` failure path. The report is recent, concrete, and the prior Codex review plus the related administrative issue both point toward keeping this as actionable follow-up rather than closing it.

## Best Possible Solution

Keep this issue open. The best next maintainer path is a focused docs and diagnostics update for non-interactive gog Gmail auth on macOS: update `skills/gog/SKILL.md`, `docs/automation/cron-jobs.md`, and `docs/cli/webhooks.md` with stable `HOME`/`PATH`, explicit account/client selection (`--account`, `--client`, `GOG_ACCOUNT`, `GOG_CLIENT`), Keychain vs encrypted file backend guidance (`gog auth keyring`, `GOG_KEYRING_BACKEND=file`, `GOG_KEYRING_PASSWORD`), and debug capture commands (`gog auth status`, `gog auth list --check`, `gog auth doctor --check`, relevant OpenClaw logs). If those steps still reproduce the intermittent failure, add a small watcher/doctor diagnostic or escalate to upstream gogcli with redacted logs.
