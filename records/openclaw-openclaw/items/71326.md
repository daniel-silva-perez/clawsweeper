## Summary

Keep open. The report is recent, detailed, and still matches current main: gateway-host YOLO exec skips the script preflight layer, host exec still builds a per-run shell command through the process supervisor, and there is no visible stale-read mitigation, F_NOCACHE/fcntl handling, retry, or regression test for atomic rename followed by a separate exec read. The prior ClawSweeper keep-open review remains consistent with current main and I found no open PR or canonical duplicate that supersedes this issue.

The issue reports stale file contents across separate macOS/APFS gateway-host exec calls after v2026.4.20 and asks OpenClaw to investigate or fix the host exec path.

## Best Possible Solution

Keep the issue open for focused maintainer investigation on an affected macOS/APFS gateway-host setup. The useful end state is a confirmed repro across v2026.4.19, v2026.4.20, v2026.4.27, and current main, followed by a core host-exec/runtime fix or narrowly scoped platform workaround with regression coverage for atomic rename followed by reads from separate exec calls; if the race cannot be reproduced, document the diagnostic and workaround scope.
