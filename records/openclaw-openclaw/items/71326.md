## Summary

Keep open. Current main still matches the reported gateway-host YOLO exec behavior: script preflight is skipped for `host=gateway`, `security=full`, `ask=off`, and each local non-PTY exec builds a fresh shell child process. I found no source, docs, changelog, test, or related-item evidence that current main fixes or supersedes the reported macOS/APFS stale-read window.

## What This Changes

Investigate and fix reported macOS/APFS stale file reads when separate gateway-host YOLO exec calls read files after atomic rename writes.

## Best Possible Solution

Keep this open for focused maintainer investigation on an affected macOS/APFS gateway-host setup, compare v2026.4.19, v2026.4.20, v2026.4.29, and current main, then land the narrowest runtime fix or documented platform workaround for atomic rename writes followed by reads from separate exec calls.
