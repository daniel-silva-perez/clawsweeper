## Summary

Keep open. Current main still has the gateway-host YOLO exec path described in the report, still launches non-PTY host exec calls through a fresh shell child process, and I did not find source, docs, tests, changelog, or related-item evidence for a stale-read mitigation or superseding tracker.

## What This Changes

The issue reports stale file reads across separate gateway-host YOLO exec calls on macOS/APFS after atomic rename writes and asks OpenClaw to investigate or mitigate the cross-process cache race.

## Best Possible Solution

Keep the issue open for focused maintainer investigation on an affected macOS/APFS gateway-host setup, compare 2026.4.19, 2026.4.20, v2026.4.27, and current main, then land the narrowest runtime fix or documented platform workaround with regression coverage for atomic rename followed by reads from separate exec calls.

Unclear. The proposed long-lived interpreter and F_NOCACHE directions are plausible but broad; the maintainable path is to first prove the regression and then choose a narrow host-exec/runtime boundary with targeted tests.
