## Summary

Keep open. Current main has adjacent improvements for DeepSeek availability, default-model admission, configured model listing, and unmanaged gateway restart, but the issue’s broader model-switch UX is still not resolved: the agent-facing gateway config tool still blocks provider config writes, explicit allowlists still constrain normal model visibility, and current-session model changes remain explicit session overrides.

## What This Changes

Improve the model-switch experience when moving from `bailian/qwen3.6-plus` to `deepseek/deepseek-v4-pro` across provider setup, dual model config files, allowlists, gateway restart, and active-session model selection.

## Best Possible Solution

Split this into maintainer-approved UX changes: keep provider setup in first-class `openclaw models`/onboarding flows, add focused doctor or model-status warnings for stale per-agent model config and allowlist-hidden defaults, and provide an explicit current-session reset/switch path instead of silently remapping active sessions after config reload.

## Reproduction Assessment

Partially. The report gives concrete model refs, product version, OS, and observed symptoms, and current source/tests confirm the remaining allowlist and explicit session-selection behavior; the exact patch payload and local config state are missing for a full five-step replay.

## Solution Assessment

Unclear. Automatic allowlist expansion and silent active-session remapping may violate intentional scoping, while warnings plus a first-class setup/switch command look like the narrower maintainable direction.
