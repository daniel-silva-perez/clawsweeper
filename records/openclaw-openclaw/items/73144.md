## Summary

Keep this issue open. Current main already has adjacent improvements for DeepSeek onboarding, Gateway `config.patch`, and unmanaged gateway restarts, but the reported model-switch UX is not fully implemented: the configured model view still intentionally prefers `agents.defaults.models` when present, and changing the default model does not imply automatic mutation of an existing session override.

## What This Changes

The issue requests lower-friction model-provider switching from Qwen to DeepSeek, including provider addition through config patching, clearer global-vs-agent model config, allowlist warnings, gateway restart behavior, and active-session model updates after reload.

## Best Possible Solution

Split the issue into explicit maintainer decisions: keep the bundled DeepSeek onboarding path, add high-signal doctor/model-status warnings when the default model or configured provider is hidden by `agents.defaults.models`, and consider a first-class `openclaw models add/set` flow rather than making active sessions silently inherit new defaults.

## Reproduction Assessment

Partially. The report gives concrete symptoms and an environment, and current tests/source reproduce the allowlist precedence behavior, but the exact patch payload and full local config needed to replay all five steps are missing.

## Solution Assessment

Unclear. Some requested directions, especially automatic allowlist expansion and active-session re-evaluation after reload, are not obviously the safest fix because they can override intentional user scoping; warnings plus a first-class model setup command look safer.
