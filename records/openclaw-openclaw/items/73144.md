## Summary

Keep open. Current main has adjacent fixes for DeepSeek catalog/onboarding, default-model admission, safer non-destructive config writes, and unmanaged gateway restart, but the central model-switch UX request is still not fully resolved because the agent-facing gateway config tool still blocks provider writes, configured model views still honor explicit allowlists, and active-session model changes remain explicit per-session state.

## What This Changes

Improve the UX for switching from `qwen3.6-plus` to `deepseek-v4-pro`, covering provider setup, dual model config files, allowlist/default-model visibility, gateway restart behavior, and current-session model updates.

## Best Possible Solution

Split this into maintainer-approved UX work: keep the DeepSeek bundled onboarding path, add targeted doctor/status warnings for provider entries hidden by agent model config or allowlists, design a first-class `openclaw models add/set` flow for custom providers, and keep active-session switching explicit through a clear reset/switch command instead of silently remapping running sessions.

## Reproduction Assessment

Partially. The report includes concrete model refs, version, OS, setup, and symptoms, and current source/tests confirm several remaining behaviors by inspection; the exact config patch payload and local config files are missing for a full five-step replay.

## Solution Assessment

Unclear. The requested automatic allowlist expansion and active-session remapping may conflict with intentional scoping, while explicit CLI/status/doctor flows look like the narrower maintainable direction.
