## Summary

Keep open. Current main has fixed or improved several adjacent pieces, but the broader model-switch UX is still not fully resolved: allowlists still intentionally constrain configured/visible models, per-agent `models.json` remains part of the provider registry story, and current-session model changes are still explicit user/session overrides rather than automatic remaps after config reload.

## What This Changes

The issue requests a smoother switch from `bailian/qwen3.6-plus` to `deepseek/deepseek-v4-pro`, covering provider addition via config patch, dual config/model registry visibility, allowlist mismatch warnings, unmanaged gateway restart, and active-session model refresh behavior.

## Best Possible Solution

Split this into maintainer-approved UX work: keep the bundled DeepSeek onboarding path, add high-signal model status/doctor warnings for providers or defaults hidden by allowlists or stale per-agent model registry state, and design an explicit `openclaw models add/set` plus current-session reset/switch flow that preserves intentional per-session overrides.

## Reproduction Assessment

Partially. The report gives model refs, version, OS, setup, and concrete symptoms, and current source/tests confirm the remaining allowlist and explicit session-selection semantics; the missing exact patch payload and config snapshots prevent a high-confidence end-to-end replay of all five steps.

## Solution Assessment

Unclear. Automatic allowlist expansion and automatic active-session remapping may violate OpenClaw's explicit scoping model; warnings plus first-class model setup and session-switch commands look like the narrower maintainable direction.
