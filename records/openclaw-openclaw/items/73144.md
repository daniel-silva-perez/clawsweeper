## Summary

Keep open. Current main has adjacent fixes and documentation for DeepSeek onboarding, default-model admission, safer additive config edits, and unmanaged gateway restart, but the central model-switch UX is still unresolved because agent-facing config writes still block provider paths, configured model views still honor explicit allowlists, and current-session model changes remain explicit per-session state.

## What This Changes

Improve the UX for switching from `bailian/qwen3.6-plus` to `deepseek/deepseek-v4-pro` across provider setup, dual model config files, allowlist/default-model visibility, gateway restart behavior, and current-session model updates.

## Best Possible Solution

Split this into maintainer-approved UX work: keep the bundled DeepSeek onboarding path, add targeted doctor/status warnings for provider entries hidden by per-agent model config or allowlists, design a first-class `openclaw models add/set` flow for custom providers, and keep active-session switching explicit through a clear reset/switch command unless maintainers choose a different session policy.

## Reproduction Assessment

Partially. The report includes concrete model refs, version, OS, setup, and symptoms, and current source/tests confirm several remaining behaviors by inspection; the exact config payload and local config files are missing for a full five-step replay.

## Solution Assessment

Unclear. The requested automatic allowlist expansion and active-session remapping may conflict with intentional scoping and trust boundaries, while explicit CLI/status/doctor flows look like the narrower maintainable direction.
