## Summary

Keep this issue open. Current main still supports memorySearch configuration, docs, doctor diagnostics, and openclaw memory status --deep, but the classic setup/onboarding and configure flows still do not offer a dedicated memory embedding setup step for provider, model, API key/SecretRef, or readiness verification. The April 26 maintainer/Codex keep-open comment remains accurate against ab5c8025.

## Best Possible Solution

Keep this open until a maintainer decides the desired product policy, then add a dedicated setup/configure memory path that prompts for agents.defaults.memorySearch provider, model, and API key or SecretRef, verifies readiness via an embedding probe or openclaw memory status --deep, and updates the public onboarding/configure/memory docs for API-key, Codex OAuth, and daemon environment behavior.
