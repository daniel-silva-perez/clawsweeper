## Summary

Keep open. Current main has memorySearch documentation plus doctor and `openclaw memory status --deep` diagnostics, but `openclaw setup`, CLI onboarding, and `openclaw configure` still do not offer a dedicated memory embedding provider/model/API-key or SecretRef setup step. The requested mandatory onboarding behavior remains a product and credential-UX decision.

## What This Changes

Request to make memory embedding provider/model/API-key setup a prominent or mandatory onboarding step so semantic memory works across sessions.

## Best Possible Solution

Add a maintainer-approved memory setup path in CLI onboarding and `openclaw configure` that configures `agents.defaults.memorySearch` provider/model plus API-key or SecretRef input, verifies readiness with a controlled embedding probe or `openclaw memory status --deep`, and updates setup/configure/onboarding/memory docs for API-key, Codex OAuth, and daemon-environment behavior.
