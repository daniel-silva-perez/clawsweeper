## Summary

Keep open. Current main still documents memorySearch setup and exposes doctor plus `openclaw memory status --deep`, but `openclaw setup`, the interactive onboarding wizard, and `openclaw configure` still lack a dedicated memory embedding provider/model/API-key or SecretRef setup step. The issue discussion and prior maintainer review comments support the gap, and the requested mandatory behavior needs maintainer UX and credential-policy judgment before implementation.

## What This Changes

The issue requests a mandatory onboarding/setup wizard step that configures memory embedding provider, model, API key or SecretRef, and verifies memory search readiness.

## Best Possible Solution

Add a maintainer-approved memory setup path in CLI onboarding and `openclaw configure` that writes `agents.defaults.memorySearch` provider/model plus API-key or SecretRef configuration, verifies readiness through a controlled embedding probe or `openclaw memory status --deep`, and updates setup/configure/onboarding/memory docs for API-key, Codex OAuth, and daemon-environment behavior.
