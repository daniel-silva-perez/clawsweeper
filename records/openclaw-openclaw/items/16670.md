## Summary

Keep open. Current main documents memorySearch setup and provides doctor plus `openclaw memory status --deep` diagnostics, but the setup/onboarding/configure paths still do not prompt for a memory embedding provider, model, API key, SecretRef, or readiness probe; the mandatory-versus-prominent-optional behavior also needs maintainer UX and credential-policy judgment.

## What This Changes

The issue requests a mandatory onboarding wizard step that configures `agents.defaults.memorySearch`, collects the required embedding credentials, and verifies the embedding endpoint before users finish setup.

## Best Possible Solution

Add a maintainer-approved memory setup path in CLI onboarding and `openclaw configure` that configures `agents.defaults.memorySearch` provider/model plus API-key or SecretRef input, verifies readiness through a controlled embedding probe or `openclaw memory status --deep`, and updates the setup/configure/onboarding/memory docs for API-key, Codex OAuth, and daemon-environment behavior.
