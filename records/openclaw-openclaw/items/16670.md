## Summary

Keep open. Current main has memorySearch configuration docs, doctor diagnostics, and `openclaw memory status --deep`, but the setup/onboarding/configure flows still lack a dedicated memory embedding provider/model/API-key or SecretRef setup step; the requested mandatory behavior needs maintainer product and credential-UX judgment.

## What This Changes

The issue requests a mandatory onboarding/configure step that sets up `agents.defaults.memorySearch` provider, model, API-key or SecretRef credentials, and verifies embedding readiness.

## Best Possible Solution

Add a maintainer-approved memory setup path in CLI onboarding and `openclaw configure` that configures `agents.defaults.memorySearch` provider/model plus API-key or SecretRef input, verifies readiness with a controlled embedding probe or `openclaw memory status --deep`, and updates setup/configure/onboarding/memory docs for API-key, Codex OAuth, and daemon-environment behavior.
