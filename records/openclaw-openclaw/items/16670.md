## Summary

Keep open. Current main has memorySearch configuration docs, doctor guidance, and `openclaw memory status --deep`, but the setup, onboarding, and configure flows still do not provide a dedicated memory embedding provider/model/API-key or SecretRef setup step. The request also asks whether that step should be mandatory, which needs maintainer product judgment before a safe implementation path is chosen.

## What This Changes

The issue requests a prominent mandatory onboarding step that configures `agents.defaults.memorySearch`, captures embedding provider credentials, and verifies embedding readiness during setup.

## Best Possible Solution

Add a maintainer-approved memory setup path in CLI onboarding and `openclaw configure` that configures `agents.defaults.memorySearch` provider/model and API-key or SecretRef input, verifies readiness with an embedding probe or `openclaw memory status --deep`, and updates setup/configure/onboarding/memory docs for API-key, Codex OAuth, and daemon environment behavior.
