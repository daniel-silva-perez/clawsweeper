## Summary

Keep this issue open. Current main still documents and diagnoses memorySearch, including doctor checks and `openclaw memory status --deep`, but the setup/onboarding and configure flows still do not provide a dedicated memory embedding provider/model/API-key or SecretRef setup step or readiness probe. The April 26 maintainer/Codex keep-open comment remains accurate against current main.

## Best Possible Solution

Keep this issue open until maintainers decide whether memory setup should be mandatory, prominent optional, or diagnostic-only. If approved, add a dedicated setup/configure memory path that prompts for agents.defaults.memorySearch provider, model, and API key or SecretRef, verifies readiness through an embedding probe or `openclaw memory status --deep`, and updates the public onboarding/configure/memory docs for API-key, Codex OAuth, and daemon environment behavior.
