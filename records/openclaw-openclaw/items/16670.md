## Summary

Keep this issue open. Current main still has memory configuration, diagnostics, and `openclaw memory status --deep`, but the classic setup/onboarding and configure flows still do not provide a dedicated memory embedding setup step for provider, model, API key or SecretRef, or readiness verification. The April 26 maintainer/Codex comment already kept it open for the same gap, and current main still matches that assessment.

## Best Possible Solution

Keep this open until OpenClaw adds a dedicated onboarding/configure memory setup path or a maintainer explicitly decides memory embedding setup should remain optional. The likely product/code path is a memory/embeddings section that prompts for provider, model, and API key or SecretRef under `agents.defaults.memorySearch`, verifies readiness with an embedding probe or `openclaw memory status --deep`, and updates public docs for API-key, Codex OAuth, and daemon environment behavior.
