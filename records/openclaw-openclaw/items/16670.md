## Summary

Keep open. Current main still documents and diagnoses memory embedding readiness, but the setup/onboarding/configure paths do not provide a dedicated memory embedding provider, model, API-key, or SecretRef setup step. The requested mandatory behavior also needs maintainer product and credential-UX direction before an autonomous fix PR is safe.

## What This Changes

The issue requests making memory embedding configuration a prominent mandatory step in the onboarding setup flow.

## Best Possible Solution

Add a maintainer-approved memory setup path in CLI onboarding and openclaw configure that configures agents.defaults.memorySearch provider/model plus API-key or SecretRef input, verifies readiness with a controlled embedding probe or openclaw memory status --deep, and updates setup/configure/onboarding/memory docs for API-key and daemon-environment behavior.
