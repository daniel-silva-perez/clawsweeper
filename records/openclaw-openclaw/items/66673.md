## Summary

Keep open. Current main at 68561a8c948b49e85ae012646de550a49d6c63a3 still does not implement the proposed `doctor.memory.remHarness` gateway RPC. The existing REM preview surface is CLI-only, while the PR remains a focused implementation candidate with a real maintainer decision to make around exposing REM-derived memory diagnostics through `operator.read`.

## Best Possible Solution

Keep this PR open for maintainer review. If remote read-only REM diagnostics are desired, land a bounded gateway implementation that registers `doctor.memory.remHarness`, scopes it under `operator.read`, routes through memory-core public/facade exports, preserves REM/CLI lookback parity, and includes targeted gateway, facade, and memory-core tests plus changed-gate/build proof. If maintainers want REM previewing to remain CLI-only, close this PR only with that explicit product decision.
