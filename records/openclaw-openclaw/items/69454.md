## Summary

Keep open. Current main has not absorbed this PR’s intended Control UI churn/cache changes: chat/session refresh still requests an unbounded session universe, `sessions.changed` still reloads immediately, node polling/preloads still run outside the Nodes flow, and Control UI `index.html` still uses `Cache-Control: no-cache`. The PR also has an unresolved review finding about deferred session reloads after tab switches.

## Best Possible Solution

Keep this PR open, update it against current main, address the deferred session reload tab-switch review comment, and re-run the targeted Control UI/Gateway tests listed in the PR body. If maintainers want smaller changes, split the `index.html` `no-store` hardening from the Control UI churn reductions and coordinate the polling policy with #45930.
