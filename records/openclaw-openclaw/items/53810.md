## Summary

Keep open. Current main still lacks the provider/model-specific normalization described here: Qwen/ModelStudio only patches request thinking flags, OpenAI-compatible streaming only promotes structured `delta.tool_calls`, and the Kimi marker parser is limited to standalone text blocks rather than tagged markers embedded in thinking/reasoning content.

## What This Changes

Request to prevent subagents from announcing completion before affected Qwen/ModelStudio/Kimi/MiniMax models' tagged tool calls in thinking or reasoning text are normalized and executed.

## Best Possible Solution

Add provider-owned Qwen/ModelStudio response normalization, or a narrowly gated shared OpenAI-compatible helper reused by Qwen and Kimi, that extracts only validated standalone tagged tool-call sections from response text/thinking/reasoning content, emits structured `toolCall` blocks, and changes `stopReason` to `toolUse` only after successful extraction.

## Repair Work Prompt

Fix the provider/model-specific subagent premature announce bug from https://github.com/openclaw/openclaw/issues/53810. Current main misses tagged tool-call markers emitted inside Qwen/ModelStudio/Kimi thinking or reasoning text, so OpenAI-compatible transport can end with `stopReason: "stop"` and no structured `toolCall`, causing subagent announce to report completion before intended tools run. Implement the smallest conservative fix: add provider-owned Qwen/ModelStudio response normalization, or a narrowly gated shared helper reused by Qwen and Kimi, that extracts only validated standalone `<|tool_calls_section_begin|>...<|tool_call_begin|>...<|tool_call_argument_begin|>{...}<|tool_call_end|>...<|tool_calls_section_end|>` sections from response text/thinking/reasoning content, emits structured `toolCall` blocks, and changes `stopReason` to `toolUse` only after successful extraction. Do not parse arbitrary inline prose that merely contains marker-looking text, do not broaden unrelated providers, and do not change core subagent announce semantics unless a focused regression proves it is necessary. Add/extend tests for Qwen/ModelStudio marker promotion, Kimi thinking/reasoning marker coverage, and the OpenAI-compatible stop-reason path. Add a user-facing changelog fix entry crediting the reporter as `Thanks @FiredMosquito831` if code behavior changes.
