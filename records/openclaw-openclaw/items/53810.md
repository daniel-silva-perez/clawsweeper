## Summary

Keep open. Current main still lacks Qwen/ModelStudio normalization for tagged tool calls embedded in reasoning/thinking output, and the existing Kimi marker parser only handles standalone text blocks. The affected bundled models remain user-facing, so the reported subagent completion path can still be reached when a provider returns `stop` without structured tool-call blocks.

## What This Changes

The issue reports provider/model-specific tagged tool-call text in thinking/reasoning blocks that is not converted into structured tool calls before subagent completion is announced.

## Best Possible Solution

Add provider-owned Qwen/ModelStudio stream normalization, or a narrowly gated shared OpenAI-compatible helper reused by Qwen and Kimi, that extracts only validated standalone tagged tool-call sections from text/thinking/reasoning content, emits structured `toolCall` blocks, and changes `stopReason` to `toolUse` only after successful extraction.
