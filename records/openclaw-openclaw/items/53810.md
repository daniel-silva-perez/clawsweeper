## Summary

Keep open. Current main still lacks the specific provider/model normalization described here: Qwen/ModelStudio response handling only patches request thinking fields, the generic OpenAI-compatible stream path only promotes structured `delta.tool_calls`, and the existing Kimi marker parser does not cover marker sections carried in thinking/reasoning blocks.

## What This Changes

The issue requests handling for provider/model responses that emit tagged tool-call sections inside thinking text so subagents do not announce successful completion before those tool calls execute.

## Best Possible Solution

Add provider-owned Qwen/ModelStudio response normalization, or a narrowly scoped shared OpenAI-compatible helper reused by Qwen and Kimi, that extracts only validated standalone tagged tool-call sections from response text/thinking/reasoning content, emits structured `toolCall` blocks, and changes `stopReason` to `toolUse` only after successful extraction.

## Repair Work Prompt

Fix the provider/model-specific tagged tool-call handling reported in https://github.com/openclaw/openclaw/issues/53810. Current main leaves `<|tool_calls_section_begin|>...<|tool_call_begin|>...` sections unparsed when they arrive in Qwen/ModelStudio OpenAI-compatible reasoning/thinking content, and the existing Kimi marker wrapper only rewrites `text` blocks. Implement a narrow normalization path in the Qwen/ModelStudio provider wrapper or a small shared OpenAI-compatible helper reused by Qwen and Kimi. Only promote complete standalone marker sections with valid JSON object arguments, preserve unrelated content, cap payload size, strip only the known trailing counter from tool ids/names, and set `stopReason` to `toolUse` only after emitting at least one structured `toolCall`. Do not add broad global parsing for arbitrary JSON/XML/ReAct text, do not weaken tool approval or before_tool_call hooks, and do not move provider-owned behavior into core unless the helper remains generic and provider-gated. Add regression tests for marker sections in thinking/reasoning blocks, plain text blocks, malformed markers, mixed normal text, and no false positives. Add a user-facing changelog fix entry crediting `Thanks @FiredMosquito831` unless maintainers decide the change is internal-only.
