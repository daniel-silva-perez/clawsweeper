## Summary

Keep open. Current main still does not normalize the reported tagged tool-call markup for Qwen/ModelStudio reasoning or thinking output, so affected subagent runs can still look complete after the provider returns `stop` without any structured tool-call blocks.

## What This Changes

The issue reports provider/model-specific tagged tool-call markup inside thinking/reasoning blocks causing subagents to announce completion before the intended tool calls execute.

## Best Possible Solution

Add provider-owned Qwen/ModelStudio stream normalization, or a narrowly gated shared OpenAI-compatible helper, that extracts validated standalone tagged tool-call sections from reasoning/thinking content, emits structured tool-call blocks, and sets `stopReason` to `toolUse` only after successful extraction.
