## Summary

Current main still lacks the reported provider/model-specific tagged tool-call normalization. Qwen/ModelStudio now has a stream wrapper, but it only patches thinking request fields; tagged tool calls embedded in reasoning/thinking text are still not promoted to structured tool calls, and the affected bundled models remain in the Qwen catalog and docs.

## What This Changes

The issue requests provider/model-specific handling for tagged tool-call output in thinking or reasoning blocks so subagents do not announce completion before their intended tool calls execute.

## Best Possible Solution

Add provider-owned Qwen/ModelStudio stream normalization, or a narrowly gated shared OpenAI-compatible helper, that extracts only validated standalone tagged tool-call sections from reasoning/thinking content, emits structured `toolCall` blocks, and changes `stopReason` to `toolUse` only after successful extraction.
