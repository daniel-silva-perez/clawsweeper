## Summary

Current main still does not implement the reported provider/model-specific tagged tool-call normalization. The Qwen/ModelStudio plugin now has a stream wrapper, but it only patches Qwen thinking payload parameters; tagged Kimi-style tool calls embedded in reasoning/thinking text are still not converted into structured toolCall blocks on the Qwen/OpenAI-compatible path. The affected Qwen catalog models remain bundled and documented, and the existing tagged-marker parser remains provider-local to the Kimi plugin and text-block-only.

## Best Possible Solution

Keep this open. The best fix is provider-owned stream normalization in the Qwen/ModelStudio plugin, or a narrowly gated shared OpenAI-compatible compat seam, that detects validated Kimi-style tagged tool-call sections inside reasoning/thinking text, converts them to structured toolCall blocks, and sets stopReason to toolUse only after successful extraction. Add regression coverage for qwen/kimi-k2.5, qwen/MiniMax-M2.5, and the existing Kimi wrapper so text-block, thinking-block, and structured delta.tool_calls behavior are all explicit.
