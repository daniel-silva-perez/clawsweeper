## Summary

Current main and the v2026.4.26 release already fix the reported macOS launchd failure. Gateway service install planning now supplies a state-dir WorkingDirectory for darwin when the lower-level program-args resolver does not provide one, and the launchd plist writer forwards and emits that value. Regression coverage and the v2026.4.26 changelog explicitly cover the WorkingDirectory fix for this issue.

## Best Possible Solution

Close this issue as implemented by the shipped macOS launchd WorkingDirectory fix in v2026.4.26. Any broader defense-in-depth change to re-anchor arbitrary relative `agentDir` overrides can be tracked separately if maintainers want that behavior beyond the launchd/default-config failure reported here.
