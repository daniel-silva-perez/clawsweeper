## Summary

Keep this PR open. Current main still does not implement the intended rw-mode non-workspace skill sync, and the PR head is not safe to merge as-is because its sandbox sync path remains unreachable for normal rw agent-scoped workspaces and it imports the bundled-skill resolver from the wrong module. The UI locale change in the PR is already covered on main and should be split out of this sandbox fix.

## Best Possible Solution

Revise or replace this PR with a focused sandbox fix in `src/agents/sandbox/context.ts` and the skill prompt/runtime path: prepare a sandbox-readable skill mirror for rw sessions, ensure prompt skill locations point to readable sandbox-workspace copies, preserve existing skill eligibility/filtering and filesystem guards, and add a targeted rw-mode regression test. Leave the already-shipped locale dropdown change out of the sandbox PR.
