## Summary

Keep this PR open. Current main still does not implement the intended `workspaceAccess: "rw"` non-workspace skill mirroring, and the PR head is not safe to merge as submitted because the sync remains in a branch normal rw agent workspaces do not reach and the proposed resolver import is wrong. The unrelated Control UI locale change is already covered on main.

## Best Possible Solution

Revise or replace this PR with a focused sandbox fix in `src/agents/sandbox/context.ts` and the embedded skill prompt/runtime path: prepare a sandbox-readable mirror for rw sessions, ensure prompt skill locations point to readable workspace copies, preserve skill eligibility/filtering and filesystem guards, and add a targeted rw-mode regression test. Leave the already-covered locale dropdown change out of this sandbox PR.
