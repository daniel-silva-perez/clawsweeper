## Summary

Keep open. The report is low-detail, but current main has several plausible "search" surfaces and does not prove a generic, cross-surface summary behavior. Some web search providers already support summaries or synthesized answers, so this needs maintainer/reporter clarification rather than closure.

## Best Possible Solution

Leave this issue open for clarification. Ask which search surface is meant and what summary output is expected. If the target is web_search, maintainers should decide whether to document/use existing provider support such as Exa, add provider-specific summary fields where supported, or design a generic summarized-result contract or middleware for all web-search providers.
