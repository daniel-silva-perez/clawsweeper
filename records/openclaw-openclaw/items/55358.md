## Summary

Keep this issue open. Current main still does not recover full Slack DM body text from top-level Slack blocks or rich-text payloads when `message.text` is already a shortened preview, and the 160-character slice in the current code is only a compact system-event preview, not the agent input body.

## Best Possible Solution

Fix this in the Slack plugin inbound content path: preserve top-level Slack `blocks` on message events, add a block/rich-text-to-body resolver for inbound messages, use it when `message.text` is missing or appears to be only a preview, and add a regression test with a >160-character DM payload proving `BodyForAgent` and `RawBody` include the full tail while the system-event preview remains compact.
