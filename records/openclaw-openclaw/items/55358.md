## Summary

Keep this issue open. Current main still routes Slack DM inbound content through the text/attachment/media resolver and does not preserve or parse top-level Slack blocks/rich-text payloads for the agent body when the event text is only a preview. The existing 160-character slice is only the system-event preview, so the reported symptom still needs a Slack plugin fix rather than cleanup closure.

## Best Possible Solution

Fix this in the Slack plugin inbound content path: preserve top-level Slack blocks on message events, add a Block Kit/rich-text-to-body resolver for inbound messages, use it when message.text is missing or appears to be only a preview, and add a regression test with a >160-character DM payload proving BodyForAgent and RawBody include the full tail while the system-event preview remains compact.
