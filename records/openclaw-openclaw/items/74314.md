## Summary

Keep open. Current main still routes google-antigravity through the pinned @mariozechner/pi-ai Cloud Code Assist transport whose Antigravity User-Agent default is 1.21.9, and the auth runtime snapshot path still prefers an active in-memory snapshot until same-process save or explicit secrets reload replaces it. The earlier ClawSweeper comment matches the current code after rechecking the newer main SHA.

## What This Changes

The issue requests fixing google-antigravity by updating or overriding the deprecated Antigravity client version and invalidating refreshed auth runtime snapshots so token recovery works without a gateway restart.

## Best Possible Solution

Implement a maintainer-reviewed Google provider/auth fix: choose a supported Antigravity version strategy for the pinned Cloud Code Assist transport, then add a targeted auth snapshot refresh or invalidation path so a successful credential refresh or explicit reload makes the active gateway observe fresh credentials without restart.
