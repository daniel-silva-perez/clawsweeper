## Summary

Keep open. Current main still has an active google-antigravity route through the pinned pi-ai Cloud Code Assist transport, and that upstream transport still defaults Antigravity to user-agent version 1.21.9. The auth snapshot side is only partially covered: same-process saves refresh active snapshots, but external credential refresh still requires an explicit runtime snapshot replacement path such as secrets.reload.

## What This Changes

The issue reports that google-antigravity uses a deprecated Antigravity API version and that runtime auth snapshots can keep stale credentials after refresh.

## Best Possible Solution

Keep this issue open for a maintainer-reviewed provider/auth fix: choose an Antigravity version strategy for the pinned Cloud Code Assist transport, then add a targeted runtime auth snapshot refresh or invalidation path so successful credential refresh or explicit reload makes the active gateway observe fresh credentials without a restart.
