Keep open. Current main still has the reported MEDIA parser shape, and open PR #75202 specifically references this issue with closing syntax, so the issue should stay open until that PR or an equivalent fix merges.

## What This Changes

The issue asks OpenClaw to harden shared MEDIA directive parsing so a local media path followed by serialized JSON is trimmed or rejected before outbound channel delivery.

## Best Possible Solution

Review and land PR #75202, or an equivalent shared parser-level fix, with focused regression coverage in src/media/parse.test.ts. The fix should happen in the shared MEDIA parsing path rather than as a primary per-channel workaround.

## Reproduction Assessment

Yes. A one-line payload like MEDIA:/path/to/image.png"}],"details":... follows a high-confidence source path: captured as one payload, split only on whitespace, accepted as a slash-prefixed local path, merged into outbound payload media, and later opened as that full local path.

## Solution Assessment

Yes. The narrowest maintainable solution is shared parser boundary handling for local MEDIA sources followed by serialized JSON, with channel-level guards only as optional defense-in-depth.
