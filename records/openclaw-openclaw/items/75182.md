## Summary

Keep open. Current main still has the shared parser path that can treat a slash-prefixed polluted `MEDIA:` payload as a valid local media source, and the issue is now paired with open closing PR #75202, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that a local `MEDIA:` directive can absorb trailing serialized JSON into the parsed file path, causing outbound media delivery to open a polluted non-existent path.

## Best Possible Solution

Review and land #75202 or an equivalent shared parser fix that trims or rejects serialized JSON after a valid local media extension, with focused coverage in `src/media/parse.test.ts`; close this issue only after that fix reaches main.

## Reproduction Assessment

Yes. The source trace on current main is enough to reproduce the bug path: full-line capture, whitespace-only splitting, broad local-path validation, outbound payload propagation, and local file read all line up with the reported polluted path.

## Solution Assessment

Yes for the direction, but not yet complete on main. The narrow maintainable fix belongs in shared `splitMediaFromOutput` parsing with regression coverage, while the open linked PR should be reviewed and landed or replaced before closing this issue.
