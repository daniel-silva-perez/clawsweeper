## Summary

Keep open. Current main still lacks a core assistant-output repetition circuit breaker, and the closest implementation is open and unmerged, so there is no implemented-on-main or duplicate close path.

## What This Changes

The issue requests a core stream-level safeguard that detects repeated assistant output, halts the LLM stream, alerts the user, and requires manual confirmation before continuing.

## Best Possible Solution

Review PR #39961 or a replacement as the canonical implementation path, then land a conservative core guard at the assistant stream subscription boundary with documented config, provider-safe abort semantics, tests for streaming and final-flush leakage, and a clear user-facing stop/cancel policy.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature request. Source inspection gives a high-confidence absence check: repeated assistant deltas would continue through the current stream/chunk path without a repetition threshold or halt-and-confirm flow.

## Solution Assessment

Unclear. A core stream guard is the right area, but the exact halt-and-confirm design needs maintainer product judgment because aborting a provider stream is supported while resuming the same stream after approval is not established in current code.
