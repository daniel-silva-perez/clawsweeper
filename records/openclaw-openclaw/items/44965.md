## Summary

Keep open. Current main has chunking, coalescing, duplicate-delivery suppression, and a separate tool-call loop detector, but it does not implement the requested stream-level repeated-text circuit breaker that aborts generation, alerts the user, and waits for approve/cancel.

## Best Possible Solution

Implement this as core stream-safety work in the agent event pipeline: maintain a configurable recent-text window, detect repeated visible assistant sequences before delivery, abort the provider/runtime stream through the existing cancellation path, send a channel-safe alert, and reuse or extend existing approve/cancel UX to resume or abort. Add regression coverage for PI/OpenAI/ACP or CLI stream paths plus block/preview delivery behavior.

## Risks / Open Questions

- A future implementation needs careful placement before channel delivery so it covers block streaming, preview streaming, WebChat deltas, and non-channel agent streams consistently.
- The existing tool-loop detector is easy to confuse with this request, but it does not protect against pure assistant text repetition.
