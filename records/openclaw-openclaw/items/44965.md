## Summary

Keep this open. Current main still has streaming chunking, duplicate-provider-event suppression, ACP status/tool repeat suppression, tool-call loop detection, and abort plumbing, but no core assistant-output repetition guard that halts a repeating model stream, alerts the user, and waits for confirmation.

## What This Changes

Request a core stream-level circuit breaker that detects repeated assistant output, stops the stream, alerts the user, and waits for approve/cancel confirmation.

## Best Possible Solution

Add an opt-in core assistant-output loop guard at the stream subscription or stream-wrapper layer, with conservative phrase detection, provider-safe abort/cancel behavior, one clear user-facing alert, config/docs/tests, and explicit semantics for what confirmation does after a stream is stopped.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature request. Source inspection gives a high-confidence absence check: repeated assistant deltas would flow through the existing streaming/chunking path without any repetition threshold or halt-and-confirm flow.

## Solution Assessment

Unclear until maintainers define the product semantics. The requested core stream guard is the right area, but pause/resume confirmation needs design first because provider streams are typically abortable rather than resumable.
