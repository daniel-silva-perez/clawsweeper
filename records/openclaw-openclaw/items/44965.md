## Summary

Current main still does not implement the requested assistant-output repetition circuit breaker. The repo has adjacent protections for block duplicate delivery, ACP status/tool repeat suppression, tool-call loops, and aborts, but nothing that detects repeated assistant prose chunks, halts the upstream stream, sends a single alert, and waits for user confirmation.

## Best Possible Solution

Keep this issue open for maintainer product/design review. The best path is likely an opt-in core assistant-output loop guard near the embedded stream subscription or stream wrapper layer, with conservative detection, configurable thresholds, provider-safe abort behavior, one user-facing alert, and an explicit decision on whether confirmation resumes a queued follow-up, retries, or only cancels the turn.
