## Summary

Keep this PR open. Current main does not implement the intended `openclaw agent --cwd` feature: the CLI command has no `--cwd` option, `AgentCliOpts` has no `cwd` field, gateway agent params do not expose `workspaceDir`, and the server still only forwards workspace overrides for stored spawned-session inheritance. The PR is still a plausible implementation candidate, but it needs normal review/landing rather than cleanup closure.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to rebase it onto current main, resolve the remaining trusted-workspace normalization review, keep generated protocol artifacts and docs aligned, and run the targeted CLI/gateway regression tests before landing.

## Risks / Open Questions

- The PR touches a security-sensitive workspace-selection surface on the gateway agent method; review should confirm only trusted owner-authorized callers can set `workspaceDir` and that paths are normalized at request ingress before agent dispatch.

- The implementation should preserve the existing internal spawned-session workspace inheritance behavior while adding the new CLI-originated override.
