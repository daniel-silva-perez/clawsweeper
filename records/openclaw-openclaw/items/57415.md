## Summary

Keep this PR open. Current main still does not expose `openclaw agent --cwd`, and the PR is a plausible implementation path, but the proposed gateway-side owner-authorized `workspaceDir` override still forwards raw trimmed paths for non-CLI callers, which can select the wrong workspace for relative or `~` inputs.

## What This Changes

The PR adds an `openclaw agent --cwd <dir>` option, forwards it into local and gateway agent runs, extends the gateway agent protocol and generated Swift models with `workspaceDir`, and adds CLI/gateway regression tests.

## Best Possible Solution

Land a rebased version that preserves existing spawned-session workspace inheritance, normalizes or rejects gateway `workspaceDir` overrides at request ingress, keeps generated protocol clients and CLI docs in sync, and verifies the focused CLI/gateway regression tests before merge.

## Reproduction Assessment

Yes. Current main absence is reproducible by inspecting the CLI registration, forwarding code, and agent RPC schema; the remaining PR bug is reproducible from the diff by sending an owner-authorized gateway `agent` request with `workspaceDir: "."` or `"~/repo"` and observing that the server forwards the raw string.

## Solution Assessment

No, not yet. The CLI-side normalization is the right direction, but the gateway-side accepted override should also be normalized with the same path resolver or limited to absolute paths before dispatch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Normalize trusted workspace overrides before dispatch:** `src/gateway/server-methods/agent.ts:102`
  - body: The server accepts `workspaceDir` from owner-authorized clients but returns the trimmed raw value. A request with `workspaceDir: "."` or `"~/repo"` then reaches `agentCommand` and is resolved later by `ensureAgentWorkspace` in the gateway process, so non-CLI admin clients can run against a different workspace than intended. Normalize at gateway ingress, or require absolute paths, before assigning `trustedWorkspaceOverride`.
  - confidence: 0.9
