## Summary

Keep this PR open, but do not merge it as-is. Current main does not already implement the PR's mirrored sendMessage fallback, input_length overflow matching, Argus lock protection, or LongCat/plain-text sanitizer changes, so the branch still contains useful work; however the proposed diff introduces concrete outbound delivery and routing regressions that need maintainer-guided splitting or replacement.

## What This Changes

The PR changes outbound delivery fallback and recovery, mirrored session channel selection, context-overflow matching, plain-text outbound sanitization, Argus session metadata/protocol handling, Telegram polling logs, and related tests.

## Best Possible Solution

Land focused, owner-reviewed patches instead of this broad branch: keep narrow sanitizer and overflow-classifier fixes where still needed, make explicit channel selection fail closed before mirrored fallback, add a real retry/drain path for deferred silent deliveries, and only add Argus metadata after configured-store lookup and protocol semantics are settled.

## Reproduction Assessment

Yes. The blockers have high-confidence static reproduction paths: enqueue a silent delivery while user-visible queue work exists, call sendMessage with an invalid explicit channel plus mirrored session context, and configure session.store away from the transcript directory before Argus lock inspection.

## Solution Assessment

No. The branch addresses real reliability gaps, but the combined patch is not the best maintainable solution because it mixes unrelated surfaces and introduces correctness and data-routing regressions; narrower patches with fail-closed routing and configured-store handling are safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Avoid stranding deferred silent deliveries:** `src/infra/outbound/deliver.ts:493-501`
  - body: This returns after enqueueing the silent delivery, before any send, ack, fail, or live retry path. Since recovery drains pending snapshots on startup/reconnect rather than continuously watching this queue, normal runtime overlap with a user-visible delivery can leave the follow-up pending until an unrelated later drain.
  - confidence: 0.9
- **[P2] Reject explicit channel failures before fallback:** `src/infra/outbound/message.ts:113-120`
  - body: Passing the mirrored fallback into resolveMessageChannelSelection also applies it when params.channel is explicit. A typo or unavailable explicit channel can therefore deliver through the mirrored session channel instead of failing fast, which can route content to an unintended channel.
  - confidence: 0.86
- **[P2] Use the configured session store for Argus checks:** `src/agents/session-write-lock.ts:313-317`
  - body: resolveSessionStorePathForLock always reads sessions.json beside the session file, but session.store supports configured paths and templates. In those setups Argus metadata is missed, so stale-lock cleanup and the watchdog can still reclaim locks the new protection is meant to preserve.
  - confidence: 0.84

## Security Review
