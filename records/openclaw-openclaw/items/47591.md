## Summary

Keep this issue open. Current main has useful mitigations for long-turn visibility and outbound recovery, but it still does not implement the requested gateway-level silence watchdog or response-delivery confirmation state machine.

## Best Possible Solution

Keep this open as a core/runtime design item. Align the active-run visibility portion with #65919, then add a generic delivery-confirmation contract around the shared reply/outbound pipeline: delivered/failed/no-response states, turn/response IDs for retry dedupe, per-channel capability/config, and regression coverage for final replies, message-tool sends, chunked delivery, preview/final transitions, and final-only channels.
