## Summary

Keep this issue open. Current main has partial mitigations: preview tool-progress updates, typing keepalive, task fallback delivery, and a write-ahead outbound delivery queue. It does not implement the requested gateway-level silence watchdog keyed to operator-visible delivery, nor the end-of-turn response-delivery state machine with no-response detection, immediate retry/alerting, correlation IDs, and per-channel configuration. The related #65919 is still open and narrower; it is not a merged implementation covering the full request.

## Best Possible Solution

Keep this open as a core/runtime design item. The best path is to align the active-run visibility portion with #65919, then add a generic delivery-confirmation contract around the shared reply/outbound pipeline: explicit delivered/failed/no-response states, response IDs for retries/dedupe, per-channel capability/config, and tests covering inline replies, message-tool sends, chunked delivery, and final-only channels.
