## Summary

Keep this issue open. Current main still has the reported override-first behavior in the shared context-token resolver and in session/status persistence paths, and the same-author implementation PR #39859 is still open/unmerged in the provided context.

## Best Possible Solution

Keep this issue open until PR #39859 or an equivalent narrow maintainer-approved fix lands. The fix should make configured and persisted positive contextTokens values act as ceilings when a smaller selected-model window is known, preserve override behavior when the model window is unknown, align the /status and session persistence paths with the shared resolver, and add focused regression coverage for resolver, reply setup, /status, follow-up, and session metadata persistence paths.
