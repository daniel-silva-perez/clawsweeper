## Summary

Keep open. Current main still lets configured context token overrides win before known model windows are considered in the shared resolver, reply setup, status, and session persistence paths. The same-author implementation candidate #39859 is still open/unmerged in the provided context and uses closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open until #39859 or an equivalent narrow maintainer-approved fix lands. The fix should make positive configured or persisted contextTokens act as ceilings when a smaller selected-model window is known, preserve current override behavior when no model window is known, and add focused regression coverage for resolveContextTokensForModel, reply model selection, /status, follow-up/session usage, and session metadata persistence.
