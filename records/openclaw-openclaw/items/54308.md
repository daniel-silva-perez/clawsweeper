## Summary

Keep open. The intended Discord teardown drain is still a plausible fix area, but the current PR is not safe to merge because it reintroduces fail-open Discord startup when bot identity lookup fails, targets stale queue architecture, and mixes unrelated Telegram/plugin type changes into the Discord teardown work.

The PR branch adds Discord managed bot identity tracking and idle-drain teardown plumbing, adds SenderManagedAccountId context, extends queue/provider tests, and includes small Telegram/plugin type-surface changes.

## Best Possible Solution

Rebase and narrow the work to a Discord-owned teardown fix that preserves fetchDiscordBotIdentity fail-closed behavior, adds a minimal idle-drain lifecycle primitive to the current createChannelRunQueue/message-run-queue path, and splits Telegram or plugin contract edits into separate PRs if they are still needed.
