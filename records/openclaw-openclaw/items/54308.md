## Summary

Keep open. Current main still does not contain the Discord idle-drain/managed-identity teardown behavior, and the proposed patch has a blocking correctness issue because it logs bot identity lookup failures and continues startup without the bot user id that Discord message filtering depends on.

## What This Changes

This PR adds Discord managed bot identity tracking, threads an idle-drain lifecycle through inbound handling and provider teardown, adds SenderManagedAccountId context, updates queue/provider tests, and includes smaller Telegram/plugin type changes.

## Best Possible Solution

Rebase and narrow the work to a Discord-owned teardown fix that fits the current createChannelRunQueue/provider.cleanup architecture, preserves fatal bot identity startup failures, adds a small idle-drain primitive only where needed, and splits unrelated Telegram/plugin contract edits into separate changes if they are still wanted.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve bot identity fetch failures:** `extensions/discord/src/monitor/provider.ts:1040-1051`
  - body: The patch catches fetchUser('@me') failures and continues startup with botUserId left undefined. Current Discord startup treats a missing bot identity as fatal, and the handler uses botUserId to suppress the bot's own messages before debounce/preflight. Starting without it can let the monitor process its own messages instead of surfacing the startup failure, so this path should rethrow or keep using fetchDiscordBotIdentity.
  - confidence: 0.86
