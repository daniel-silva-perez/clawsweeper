## Summary

Keep this PR open. Current main now implements the concurrency part through the typed `agents.*.memorySearch.remote.nonBatchConcurrency` surface, but the retry attempt/base-delay/max-delay behavior that motivated #68812 remains hardcoded. The PR is still a plausible implementation candidate, but it is not merge-ready because it reads `cfg.memorySearch.embedding.*` instead of the resolved agent-scoped config, does not extend the strict schema/types/docs/tests, and includes an unrelated root log file. Security/supply-chain review found no workflow, dependency, install, release, or package-publishing changes; the concrete security hygiene issue is the committed runtime log artifact.

## Best Possible Solution

Keep this PR open for revision or maintainer review. A mergeable version should remove the log file, add retry/backoff fields to the typed and validated agent-scoped `agents.*.memorySearch` config contract, document the public keys, add config/runtime tests, and reconcile the concurrency knob with existing `remote.nonBatchConcurrency` rather than adding a parallel `memorySearch.embedding.indexConcurrency` surface.
