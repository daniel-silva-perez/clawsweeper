## Summary

Current main still drops explicit Feishu `replyTo` targets on the ordinary message-tool `send` path: core resolves the reply target and the lower Feishu adapters can use it, but the Feishu plugin action handler only forwards a reply target for `thread-reply`. This remains a valid, narrow plugin bug.

## What This Changes

The issue requests stable Feishu quoted replies when the message tool sends with `replyTo` on OpenClaw 2026.3.23-2, especially for WebSocket message/card replies.

## Best Possible Solution

Fix this in the Feishu plugin action layer: ordinary `send` should map explicit `replyTo`/`reply_to` and direct `replyToId` parameters to `replyToMessageId` for text/card sends and `replyToId` for media sends, while preserving `thread-reply` as the explicit `reply_in_thread` path that requires a message id.
