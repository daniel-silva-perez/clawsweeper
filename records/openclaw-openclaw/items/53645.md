## Summary

Keep this issue open. Current main still drops explicit Feishu `replyTo` / `reply_to` values on the ordinary message-tool `send` action path, even though the generic message tool and lower Feishu outbound adapter can carry a reply target. The prior bot comment remains consistent with current source at 68561a8c948b49e85ae012646de550a49d6c63a3.

## Best Possible Solution

Keep this issue open and fix it in the Feishu plugin owner module. `extensions/feishu/src/channel.ts` should resolve `replyTo` / `reply_to` for ordinary `action: "send"`, pass it as `replyToMessageId` or media `replyToId` for text, presentation/card, and media sends, and preserve `thread-reply` as the explicit `reply_in_thread` path requiring `messageId`. Add focused Feishu channel action tests for explicit `replyTo` on ordinary text, presentation/card, and media sends.
