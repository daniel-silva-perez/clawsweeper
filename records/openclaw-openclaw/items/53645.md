## Summary

Keep open. Current main still drops explicit Feishu `replyTo` targets on the plugin message-action `send` path: core resolves and keeps `replyTo`, and lower Feishu send helpers can issue `im.message.reply`, but the Feishu action handler only maps a reply target for `thread-reply`. This remains a narrow, source-backed bundled Feishu plugin bug.

## What This Changes

The issue requests stable Feishu quoted replies when the message tool sends with an explicit `replyTo` target after OpenClaw 2026.3.23-2.

## Best Possible Solution

Fix this in the bundled Feishu plugin action layer by mapping ordinary `send` parameters `replyTo`/`reply_to` and any direct `replyToId`-style alias into the existing Feishu `replyToMessageId`/`replyToId` send helper inputs, while preserving `thread-reply` as the explicit `reply_in_thread` path that requires a message id.
