## Summary

Current main still appears to drop explicit Feishu `replyTo` targets on the ordinary plugin `send` path. Core resolves the generic message-tool reply target and the lower Feishu outbound adapter can forward it, but `extensions/feishu/src/channel.ts` only maps a reply target for `thread-reply`, and the current Feishu action tests still assert `undefined` reply fields for ordinary text, card, and media sends.

## What This Changes

The issue asks for Feishu/Lark message-tool replies with `replyTo` to consistently show as quoted replies after the 2026.3.23-2 upgrade.

## Best Possible Solution

Fix this in the Feishu plugin action layer: ordinary `send` should map explicit send-level reply targets such as `replyTo`/`reply_to` and direct reply id aliases into `replyToMessageId` for text/card sends and `replyToId` for media sends, while preserving `thread-reply` as the explicit `reply_in_thread` path that requires a message id.
