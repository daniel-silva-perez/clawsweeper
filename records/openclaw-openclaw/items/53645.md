## Summary

Current main still appears to have the Feishu plugin bug: the generic message tool accepts and resolves `replyTo`, and the Feishu lower outbound/send helpers can send replies, but the Feishu plugin action handler only maps a reply target for `thread-reply`, not ordinary `send`. The issue should stay open and is a good narrow repair candidate.

## What This Changes

The issue requests stable Feishu quoted replies when the `message` tool uses the `replyTo` parameter on ordinary sends.

## Best Possible Solution

Fix this in the Feishu plugin action layer so ordinary `send` maps explicit `replyTo`/`reply_to` and direct reply-id aliases into the existing Feishu reply helpers, while preserving `thread-reply` as the explicit `reply_in_thread` path. Add focused text, card, and media regression tests plus a user-facing changelog entry.
