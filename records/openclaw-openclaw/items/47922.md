## Summary

Keep this PR open. Current main still does not implement the proposed `messages.outbound.text.emDash` setting, resolver, outbound text rewrite helper, direct/gateway send rewrite, mirror rewrite, or post-`message_sending` hook rewrite. The PR remains a plausible implementation candidate, but the existing review comments identify real punctuation and hook-ordering issues that need maintainer resolution before merge.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is to fix the rewrite helper so it only changes punctuation caused by em-dash replacement, add tests for leading em dashes and existing trailing commas, explicitly document or adjust the `message_sending` hook ordering, rebase away stale generated-doc drift, and then run the normal changed-gate review.

## Risks / Open Questions

- The PR's rewrite helper can produce a leading comma for text that starts with an em dash and can remove a pre-existing trailing comma even when no em dash was rewritten.
- The PR changes what `message_sending` hooks observe and may need explicit maintainer agreement plus docs or comments before merge.
- The PR appears to include generated config baseline files that are not present in the current main tree, so it likely needs a rebase/regeneration pass before normal review.
- Because this is an opt-in global outbound text mutation surface, edge-case tests should cover leading em dashes, trailing punctuation, direct sends, gateway sends, mirrors, and hook-modified content.
