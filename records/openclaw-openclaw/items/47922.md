## Summary

Keep this PR open. Current main still does not implement the proposed `messages.outbound.text.emDash` config, resolver, shared outbound text transform, gateway-mode rewrite, mirror rewrite, or post-`message_sending` hook rewrite. The PR remains a plausible implementation candidate, but the existing review comments identify real text-mutation edge cases and hook-ordering semantics that need maintainer resolution before merge.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is to fix the rewrite helper so it only changes punctuation caused by em-dash replacement, add tests for leading em dashes and existing trailing commas, explicitly document or adjust the `message_sending` hook ordering, rebase away stale generated-doc drift, and then run the normal changed-gate review.
