## Summary

Keep open. Current main does not implement granular per-tool/per-operation `tools.fs.workspaceOnly` policy. It has a single `tools.fs.workspaceOnly` boolean for read/write/edit/apply_patch, plus a separate `tools.exec.applyPatch.workspaceOnly` knob that protects only `apply_patch`; write and edit still become non-workspace writes when `tools.fs.workspaceOnly=false`.

## Best Possible Solution

Keep the issue open and implement a backwards-compatible fs policy extension, likely by preserving the existing boolean as the coarse default while adding per-operation policy such as read/search/write/edit/apply_patch or explicit read-only external roots. Update `src/config` schema/types/help, `src/agents/tool-fs-policy.ts`, `src/agents/pi-tools.ts`, security/docs config pages, and tests proving the requested read-outside plus write/edit/apply_patch-inside behavior.
