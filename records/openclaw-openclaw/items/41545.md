## Summary

Keep open. Current main still contains the reported Overview state transition: editing the WebSocket URL sets `gatewayUrl` and clears `token` when the typed value differs from the prior URL. The issue also has same-author PR #41546 and other open PRs with closing syntax for this exact bug, so the right cleanup path is maintainer review/landing of one canonical fix rather than closing the issue now.

## Best Possible Solution

Keep the issue open and review/land one focused PR that preserves the Gateway Token during the manual WebSocket URL edit workflow while retaining selected-gateway token scoping. The chosen fix should update the browser test that currently asserts clearing and add regression coverage for typed-token and existing-token flows.
