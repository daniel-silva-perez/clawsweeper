## Summary

Keep open. The checked-out main and a live remote-main spot check still clear the Overview Gateway Token whenever the WebSocket URL edit differs from the previous trimmed string, current browser coverage still asserts that clearing behavior, and open PR #73923 uses closing syntax for this issue as the likely canonical fix path.

## Best Possible Solution

Keep this issue open and review or land one canonical fix PR, preferably #73923 if it passes maintainer review. The implementation should preserve the in-memory Gateway Token during same-normalized-endpoint WebSocket URL edits, clear or reload the token when the normalized gateway endpoint actually changes, and update browser/node regression coverage so the token-scoping boundary is explicit.
