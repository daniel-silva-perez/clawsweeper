## Summary

Keep open. Current main still has separate agent event, trajectory, ACP stream-log, diagnostics timeline, and task ledger shapes rather than a canonical run trace schema adopted across agent, ACP, subagent, and task-flow paths; the related run-trace PR in the provided context remains open and unmerged.

## What This Changes

Define and adopt a canonical run trace schema covering main agent runs, ACP parent-child relay, subagents, task execution progress, and task-flow terminal events.

## Best Possible Solution

Keep this as an architecture/observability tracker until a small versioned run-trace contract is merged on main, with adapters from agent events, trajectory/runtime events, ACP relay, subagent lifecycle, task records, and Task Flow state changes plus focused contract tests and docs.

## Reproduction Assessment

Not applicable as a feature/architecture request. The verification path is static inspection of current main: searches for the requested API/schema names and inspection of existing event/task/trajectory types show the unified schema is still missing.

## Solution Assessment

Unclear because no merged solution exists on main. The maintainable direction is a versioned contract that maps existing producers and clarifies its relationship to trajectory bundles rather than adding another unowned parallel trace surface; #43145 should be reviewed as prior art before implementation.
