## Summary

Keep open for maintainer review. Current main and the latest release still normalize only Google Workspace Add-on `messagePayload` events, while this PR is a narrow, mergeable contributor patch for the remaining lifecycle payload gap; it should be finished with explicit branch selection, regression tests, and a changelog entry rather than closed or replaced automatically.

## What This Changes

The PR updates the Google Chat webhook parser to unwrap Google Workspace Add-on `messagePayload`, `addedToSpacePayload`, and `removedFromSpacePayload` variants into flat Google Chat event types.

## Best Possible Solution

Land a narrow Google Chat parser fix that explicitly normalizes each add-on payload variant, keeps add-on token verification unchanged, adds focused webhook tests for added-to-space and removed-from-space envelopes, and leaves the broader #65007 and #65326 follow-ups tracked separately.

## Reproduction Assessment

Yes. A current-main Google Workspace Add-on envelope with `commonEventObject.hostApp: "CHAT"`, a body `authorizationEventObject.systemIdToken`, and only `chat.addedToSpacePayload` or `chat.removedFromSpacePayload` skips normalization and reaches the `invalid payload` path by source inspection.

## Solution Assessment

Yes, the requested direction is the narrowest maintainable fix for this parser gap. The safest version is to keep payload selection and event labeling in explicit branches, then add regression coverage and changelog credit before merge.
