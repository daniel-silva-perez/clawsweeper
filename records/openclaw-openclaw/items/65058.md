## Summary

Keep open. Current main still only normalizes Google Chat add-on `messagePayload` envelopes, while the PR addresses a real lifecycle-payload gap; it needs maintainer review, focused tests, and a small nonblocking cleanup around payload selection before merge.

## What This Changes

The PR updates the Google Chat webhook parser to accept `chat.addedToSpacePayload` and `chat.removedFromSpacePayload` add-on envelopes alongside `chat.messagePayload`, mapping them to lifecycle event types while preserving add-on token extraction.

## Best Possible Solution

Land a narrow Google Chat parser change that explicitly normalizes the three supported add-on payload variants, preserves add-on token verification for each accepted variant, and adds focused webhook tests for added-to-space and removed-from-space envelopes while leaving the remaining #65007 work to separate review.
