## Summary

Keep this PR open. Current main still declares the `skills-python-tests` hook dependency as `[pytest>=8, <9]`, which YAML loads as two entries and leaves pip with a standalone invalid `<9` requirement. The PR remains a narrow one-line fix, and the existing discussion correctly identifies the same remaining root cause.

## Best Possible Solution

Keep this PR open for maintainer review and merge it, or apply the same one-line change directly on main so the hook declares `additional_dependencies: ["pytest>=8,<9"]`. No broader pre-commit, plugin, or runtime change is needed.
