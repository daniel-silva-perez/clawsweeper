## Summary

Keep open. Current main still has advisory prompt guidance, adaptive read paging, web_fetch truncation metadata, web_search provider fallback, and a generic before-finalize hook, but I found no mandatory completeness scan guardrail that broadly covers read/search operations or blocks finalization on incomplete evidence; the issue is also paired with open unmerged PR #73618 using closing syntax.

## What This Changes

Request a mandatory completeness scan guardrail for read/search operations so agents must retry, continue scanning, or clearly block/mark partial evidence before finalizing.

## Best Possible Solution

Keep this open while maintainers define the enforcement scope and review PR #73618. A durable implementation should name the covered read/search surfaces, preserve partial/truncated/no-result state where needed, and add tests showing the agent either performs a follow-up scan or clearly reports blocked/partial evidence before finalizing.

## Reproduction Assessment

Not applicable as a user-facing bug reproduction. The current-main gap is reproducible by source inspection: the relevant prompt, read, web_fetch, web_search, and before-finalize paths do not contain a broad mandatory completeness guardrail.

## Solution Assessment

Unclear. A prompt-only contribution may be enough if maintainers accept instruction-level enforcement, but mandatory semantics may be better served by the existing before_agent_finalize revision seam plus tool-result completeness metadata.
