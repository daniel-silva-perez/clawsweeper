## Summary

Keep open. Current main still shows advisory prompt guidance and bounded read/fetch handling, but no mandatory completeness scan guardrail that broadly covers read/search operations or blocks finalization on partial evidence. The issue is also paired with open unmerged PR #73618, which uses closing syntax for this issue, so it should stay open until maintainers review, land, or close that PR.

## What This Changes

The issue requests a mandatory completeness scan guardrail for agent read/search operations before final answers.

## Best Possible Solution

Keep this open while maintainers define the enforcement scope and review PR #73618. A durable implementation should name the covered read/search surfaces, preserve partial/truncated/no-result state where needed, and add tests showing the agent either performs a follow-up scan or clearly reports blocked/partial evidence before finalizing.
