## Summary

Keep open. Current main still has advisory prompt guidance, adaptive read paging, web_fetch truncation metadata, web_search provider fallback, and an optional before-finalize hook, but no broad mandatory completeness scan guardrail for read/search operations. The issue is also paired with open unmerged PR #73618 using closing syntax, so it should remain open until that PR is reviewed, landed, or closed.

## What This Changes

Request a mandatory completeness guardrail that covers read/search operations before an agent finalizes an answer.

## Best Possible Solution

Keep this issue open while maintainers review PR #73618. A durable fix should define the covered read/search surfaces, preserve incomplete or truncated result state where needed, and include tests showing the agent either performs a follow-up scan or clearly reports blocked/partial evidence before finalizing.

Not applicable as a user-facing bug reproduction. The current-main gap is reproducible by source inspection: the prompt, read, web_fetch, web_search, and before-finalize paths do not contain a broad mandatory completeness guardrail.

## Solution Assessment

Unclear. A prompt-only implementation may be acceptable if maintainers want instruction-level behavior, but stronger mandatory semantics likely belongs in the existing before_agent_finalize seam plus tool-result completeness metadata.
