## Summary

Keep open. Current main still has advisory prompt guidance and bounded read auto-paging, but I did not find a mandatory completeness scan guardrail that broadly covers read/search operations or blocks finalization when evidence is partial. The open implementation PR #73618 is explicitly paired with this issue and is not merged, so this issue should remain open until that PR is reviewed, landed, or closed.

## What This Changes

Request a mandatory agent guardrail that verifies completeness for read/search operations before the agent finalizes an answer.

## Best Possible Solution

Keep this issue open while maintainers decide the intended scope and review PR #73618. The durable end state should define which read/search tools are covered, add runtime-visible partial/truncated/no-result metadata where needed, and include tests proving the agent either performs a follow-up scan or clearly reports a partial/blocked result before finalizing.
