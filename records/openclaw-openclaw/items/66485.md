## Summary

Keep this PR open. Current main already covers the context-percentage alignment for the local `/usage` surface, but I could not verify that main implements the PR's other intended behavior: refreshing active chat history after agent compaction completes. The PR should be rebased and narrowed rather than closed as already implemented.

## Best Possible Solution

Keep the PR open as a possible implementation candidate, but require a rebase onto current main. The useful remaining work is a tested, session-scoped refresh of active chat history after successful compaction completion. It should preserve current `/usage` total-token logic, avoid sessionless/unrelated compaction reloads, and include a focused `app-gateway` test proving only the active session refreshes.
