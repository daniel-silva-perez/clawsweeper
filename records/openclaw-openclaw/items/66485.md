## Summary

Keep this PR open. Current main already has the `/usage` context-percentage alignment, but it still does not implement the PR's other intended behavior: refreshing the active chat history after compaction completes.

## Best Possible Solution

Keep the PR open as a possible implementation candidate, but require a rebase onto current main. Preserve the shipped `/usage` total-token logic, then add a narrow active-session-only refresh path after successful compaction completion, with an app-gateway test proving unrelated or sessionless compaction events do not reload the active chat.
