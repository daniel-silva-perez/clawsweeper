## Summary

Keep open. Current main contains meaningful hardening since the reported 2026.4.8 incident, including hot model-config reload support, bounded timeout-compaction retries, explicit restart/drain handling, and subagent announce fallback paths. It does not fully resolve the umbrella failure chain: gateway drain still rejects new enqueues, reply handling still surfaces a restart message for GatewayDrainingError instead of retrying the original turn, subagent announce cleanup still has retry-limit/expiry give-up paths, timeout compaction still falls through after bounded attempts, and runtime fallback/session-override semantics remain subtle enough to need maintainer splitting or product direction. Merged PR #70743 is useful partial hardening, not a complete superseding fix for this issue.

## What This Changes

The issue requests investigation and fixes for a 2026.4.8 large Telegram session failure chain involving compaction overflow/timeouts, gateway draining task rejection, subagent completion announce loss, and stale runtime fallback routing after config changes.

## Best Possible Solution

Keep this as the umbrella until maintainers either split it into narrower canonical follow-ups or land a coordinated reliability fix covering restart/drain retry semantics, durable or explicitly bounded subagent completion delivery, compaction timeout circuit-breaker or reset policy, and documented live fallback/session-override behavior after config changes.
