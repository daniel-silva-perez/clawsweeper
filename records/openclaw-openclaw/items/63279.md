## Summary

Keep open. Current main has meaningful partial fixes for compaction timeout handling, subagent announce fallback/queueing, and fallback classification, but it still does not implement the full requested behavior. Gateway drain still rejects new work and is surfaced as final restart text, subagent cleanup can still terminally give up on delivery, and fallback/session override semantics remain an explicit maintainer product/code decision rather than a completed fix.

## Best Possible Solution

Keep this issue open as the umbrella until maintainers either split the remaining work into narrower canonical issues or land a focused follow-up. The likely path is to route GatewayDrainingError retry behavior through #55412 or a successor, decide whether repeated compaction failures need a configured timeoutAction/circuit breaker, and clarify or change how hot-applied fallback config interacts with active auto-fallback session overrides during drain windows.
