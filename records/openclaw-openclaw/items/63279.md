## Summary

Keep open. Current main has meaningful partial fixes for this failure chain, especially compaction timeout handling and subagent announce queueing, but it does not fully implement the requested behavior. New work is still visible around GatewayDrainingError retry semantics and the interaction between hot-reloaded fallback config, auto fallback overrides, and active drain windows. The merged GPT-5.4 hardening PR explicitly treated this report as only partially addressed, so closing as implemented would overstate the current state.

## Best Possible Solution

Keep this issue open as the umbrella until maintainers either split the remaining work into narrower canonical items or land a focused fix. The best path is to route GatewayDrainingError retry/user-visible behavior through #55412 or a successor, verify whether repeated compaction failures need a circuit breaker or timeoutAction, and clarify or adjust fallback-chain reload semantics for active sessions during drain windows.
