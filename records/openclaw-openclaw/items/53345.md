## Summary

Keep open. Current main already covers the Korean Control UI locale, and that part shipped in v2026.4.27, but the requested normal agent response-language persistence and automatic Korean-aware search behavior remain unimplemented; the paired implementation PR is closed unmerged.

## What This Changes

Request to add Korean Control UI localization plus AI-agent Korean language persistence and localized search behavior.

## Best Possible Solution

Keep the shipped Korean Control UI locale, then define and implement a generic user-language persistence policy and search-language hinting contract that works across locales/providers without Korean-specific hardcoding.

## Reproduction Assessment

Not applicable as a pure feature request. Source inspection gives a high-confidence current-state check: Korean UI locale support is present, while normal agent reply-language persistence and automatic Korean search-language selection are absent.

## Solution Assessment

Unclear for the remaining work. The Control UI implementation is already in the right owner surface, but adding Korean-specific system-prompt and Brave-only defaults would not be the best long-term fix; the safer path is a generic language policy and provider-aware search contract.
