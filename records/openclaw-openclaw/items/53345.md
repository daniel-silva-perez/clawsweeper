## Summary

Keep this issue open. Current main implements the Korean Control UI locale, registry wiring, docs, and structural tests, but the same request also asks for default agent primary-language persistence and automatic Korean search localization; those behaviors are still not implemented as platform defaults, and the paired same-author PR is closed unmerged.

## What This Changes

Add Korean locale support to the Control UI and add Korean/primary-language consistency behavior for agent responses and localized search.

## Best Possible Solution

Keep the shipped Korean Control UI locale, then either split or implement the remaining work as a generic agent primary-language policy plus provider-aware search localization. The agent behavior should be tested around user overrides and literal-output preservation, and search defaults should use each provider's supported language parameters rather than assuming one universal `search_lang` field.
