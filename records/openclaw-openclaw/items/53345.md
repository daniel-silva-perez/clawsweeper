## Summary

Keep this issue open. Current main implements Korean Control UI localization, registration, docs, and tests, but the issue also asks for default AI-agent language persistence and automatic Korean search localization. Those remaining behaviors are not implemented on main, and the paired PR #53346 is closed unmerged.

## Best Possible Solution

Keep this issue open but narrow the remaining maintainer scope. Preserve the shipped Korean Control UI locale, then design and test a default primary-language response policy in the agent prompt or conversation layer with override and literal-output preservation rules. Add Korean search localization only through provider-aware web-search defaults or configuration, not by assuming a universal `search_lang` field.

## Risks / Open Questions

- Search localization should be provider-aware because `search_lang` is Brave-specific while other search providers expose different language or region controls.
