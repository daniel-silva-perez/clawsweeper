## Summary

Keep this issue open. Current main has useful task/subagent records, `openclaw tasks` list/show/cancel, `/subagents` inspection/control, transient state-change notifications, and internal orphan resume recovery, but it still does not implement the requested durable per-subagent lifecycle event ledger, persisted tool/warning/error timeline, artifact-pointer completion payloads, first-failure/recommended-action diagnostics, or idempotent audit-logged cancel/steer/resume supervision surface. Related #62478 is closed background-spawn work and remains complementary rather than a canonical replacement.

## Best Possible Solution

Keep the issue open unless a maintainer intentionally narrows or supersedes it. The best path is to design a durable per-run event ledger for subagent/task spawn, queued/running, tool start/end, warning/error, completion, and supervision actions; persist it beside task/subagent records; expose it through `/subagents info/log`, `openclaw tasks show/list --json`, gateway/control UI surfaces, and docs; include artifact pointers plus terminal first-failure/recommended-action diagnostics; and make cancel, steer, and operator resume idempotent with auditable action records. Treat #62478 as shipped complementary background-spawn work, not the canonical replacement for this observability scope.
