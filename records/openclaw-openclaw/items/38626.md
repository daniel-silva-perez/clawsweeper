## Summary

Keep this issue open. Current main has useful task/subagent records, `openclaw tasks` inspection/cancel support, `/subagents` list/info/log/kill/send/steer/spawn controls, transient state-change notifications, and orphan recovery, but it still lacks the requested durable per-subagent lifecycle event ledger, persisted tool/warning/error timeline, artifact-pointer completion payloads, first-failure/recommended-action diagnostics, and idempotent audit-logged cancel/steer/resume supervision surface.

## Best Possible Solution

Keep this issue open unless a maintainer intentionally narrows or supersedes it. The best path is to design a durable per-run event ledger for subagent/task spawn, queued/running, tool start/end, warning/error, completion, and supervision actions; persist it beside task/subagent records; expose it through `/subagents info/log`, `openclaw tasks show/list --json`, gateway/control UI surfaces, and docs; include artifact pointers plus terminal first-failure/recommended-action diagnostics; and make cancel, steer, and operator resume idempotent with auditable action records. Treat #62478 as shipped complementary background-spawn work, not the canonical replacement for this observability scope.
