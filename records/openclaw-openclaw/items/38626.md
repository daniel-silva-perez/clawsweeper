## Summary

Keep this issue open. Current main has scalar task/subagent run records, `/subagents` info/log/control commands, task list/show/cancel/notify, and push-style state-change notifications, but it still does not provide the requested durable per-subagent event ledger, persisted tool/warning/error timeline, artifact-pointer completion payloads, first-failure/recommended-action diagnostics, or idempotent audit-logged cancel/steer/resume supervision surface. The closed #62478 background-spawn work is complementary, not a replacement for this observability/control scope.

## Best Possible Solution

Keep this open for maintainer design and likely split it into narrower implementation issues: add a durable per-run event ledger for subagent/task spawn, queued/running, tool start/end, warning/error, completion, and supervision actions; persist artifact pointers and terminal first-failure/recommended-action diagnostics; expose the data through `/subagents info/log`, `openclaw tasks list/show --json`, gateway/control UI surfaces, and docs; and make cancel, steer, and resume deterministic, idempotent, and audit-logged. Treat #62478 as shipped complementary background-spawn work, not the canonical replacement.
