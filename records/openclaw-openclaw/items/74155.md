## Summary

Keep this PR open. Current main still has the Slack `download-file` schema gap: Slack advertises `download-file` and the dispatcher requires `fileId`, but `describeSlackMessageTool` contributes no model-visible `fileId` schema. The PR is directionally correct and plugin-owned, but it needs changes before merge because its PR-head copy drifts from Slack’s actual action surface and CI is failing.

## What This Changes

The PR adds Slack-owned `message` tool schema fragments for `fileId`, `messageId`, and `message_id`, adds a targeted `download-file` error when a Slack message timestamp is passed instead of `fileId`, and expands Slack tests for that behavior.

## Best Possible Solution

Keep this PR open, fix the lint failures, and align the schema descriptions plus friendly error text with the actual Slack action surface on main. The best implementation remains Slack-owned `describeMessageTool` schema contributions plus a focused dispatcher hint; after landing, use it as the canonical fix for #45574/#45626 while keeping broader structured file-id propagation work separate.
