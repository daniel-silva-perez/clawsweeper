## Summary

Keep this PR open. Current main still has the Slack `download-file` schema gap: Slack advertises `download-file` and the dispatcher/runtime require `fileId`, but `describeSlackMessageTool` returns only actions/capabilities and contributes no model-visible `fileId` or `messageId` schema. The PR is a plausible Slack-plugin-owned fix, but the supplied PR context shows lint failures and model-visible action wording drift that should be corrected before merge. Security review found no workflow, dependency, lockfile, secret, install script, publishing, or artifact-download changes in the PR file list.

## What This Changes

This PR adds Slack-owned `message` tool schema descriptions for `fileId`, `messageId`, and `message_id` around `download-file`, adds a targeted Slack dispatcher error when `messageId` is supplied without `fileId`, and updates focused Slack tests.

## Best Possible Solution

Keep the PR open, fix the lint/style failures, then align the schema descriptions and friendly error text with the actual Slack messageId actions before review. Prefer the plugin-owned `describeMessageTool` schema contribution approach here over a generic global `fileId` field, then close or supersede the older related #45574/#45626 pair once one canonical fix lands.
