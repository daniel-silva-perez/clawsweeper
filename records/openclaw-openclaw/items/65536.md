## Summary

Keep open. Current main still has the direct `os.homedir()` sandbox tilde expansion and sandbox error-shortening behavior this PR changes, and the latest release tag has the same sandbox code. The PR is still relevant, but it should be revised before merge to reuse the central OS-home helper contract and add focused sandbox regression coverage; the host write issue it references was already closed separately as implemented for non-sandbox host tools.

## Best Possible Solution

Keep this PR open, but revise before merge: replace the local `resolveHome()` helper with the shared `resolveOsHomeDir()`/`expandHomePrefix()` contract in `src/agents/sandbox-paths.ts`, cover both `expandPath()` consumers and `shortPath()`, add sandbox resolver or sandbox bridge tests for HOME/USERPROFILE/fallback behavior plus `OPENCLAW_HOME` separation, and keep the broader OPENCLAW_HOME policy work in #54014 separate.
