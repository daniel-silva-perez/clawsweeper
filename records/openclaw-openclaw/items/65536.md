## Summary

Keep open. Current main still has the sandbox `~` expansion and error-shortening behavior this PR changes, and the latest release still contains the same direct `os.homedir()` calls. The PR is relevant, but it should be revised before merge to use the shared OS-home helper contract and add focused sandbox regression coverage.

## What This Changes

This PR adds a local home-resolution helper in `src/agents/sandbox-paths.ts` and uses it for sandbox `~` expansion and sandbox error-path shortening.

## Best Possible Solution

Keep this PR open, but revise it before merge: replace the local `resolveHome()` helper with the shared `resolveOsHomeDir()` plus `expandHomePrefix(..., { home })` pattern in `src/agents/sandbox-paths.ts`, cover both `expandPath()` consumers and `shortPath()`, add focused sandbox regression tests for `HOME`/`USERPROFILE`/fallback behavior and `OPENCLAW_HOME` separation, and keep the broader `OPENCLAW_HOME` policy discussion in #54014 separate.
