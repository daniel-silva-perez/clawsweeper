## Summary

Keep this PR open. Current main still does not implement the requested optional `spawnedBy` field on live `chat` delta/final/error broadcasts or non-tool `agent` broadcasts, and the Gateway protocol schemas/generated Swift event models still omit that field. Existing session snapshot/tool/session paths already expose lineage, so this is a real remaining core Gateway protocol gap rather than a ClawHub item or duplicate.

## Best Possible Solution

Review and land a polished version of this PR or an equivalent patch that adds optional `spawnedBy` to subagent/ACP `chat` and non-tool `agent` event payloads, updates `AgentEventSchema` and `ChatEventSchema`, regenerates Swift protocol models, and keeps targeted tests for all broadcast paths plus non-subagent omission. Resolve the hot-path lookup/caching tradeoff during review rather than closing the PR as obsolete.
