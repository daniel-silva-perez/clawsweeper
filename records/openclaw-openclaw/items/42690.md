## Summary

Current main still does not implement the requested `session_status` split between the selected/session-default model and a non-fallback routed active model. The shared status renderer computes selected and active refs, and transcript fallback can recover an active model, but rendering only emits `🧠 Model:` plus `↪️ Fallback:` for persisted fallback state. The related #55425 change shipped the narrower behavior of using the persisted runtime model as the selected card model when there is no explicit override; it does not add the separate `↪️ Active:` line requested here.

## Best Possible Solution

Keep this issue open and implement the split in the shared status renderer: keep `🧠 Model:` tied to the configured or explicit session selection, add `↪️ Active: provider/model` when the resolved active runtime model differs and fallback state is not active, preserve the existing `↪️ Fallback:` behavior for model failover, and add focused coverage in `src/auto-reply/status.test.ts` plus the `session_status` tool tests for the no-explicit-override/runtime-routed case.
