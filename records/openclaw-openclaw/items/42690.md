## Summary

Current main still does not implement the requested `session_status` selected/default model plus routed active model split. The shared status renderer computes selected and active model identities, but it only renders the selected `🧠 Model:` line and a fallback-specific `↪️ Fallback:` line. The session_status tool also collapses no-override runtime identity into the primary model label, which addresses the adjacent persisted-runtime bug from #55425 but not this issue’s requested non-fallback `↪️ Active:` line.

## Best Possible Solution

Keep this issue open. Implement the split in the shared status rendering path: keep `🧠 Model:` tied to the configured or explicit session selection, add `↪️ Active: provider/model` when the resolved active runtime model differs and fallback state is not active, preserve existing `↪️ Fallback:` behavior for failover, and add focused coverage in `src/auto-reply/status.test.ts` plus `src/agents/openclaw-tools.session-status.test.ts`. If the session entry lacks active identity, add a bounded last-`model-snapshot` or assistant metadata lookup without mutating the session default.
