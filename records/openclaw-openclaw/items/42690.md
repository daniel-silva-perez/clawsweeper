## Summary

Keep open. Current main still does not implement the requested `session_status` split between the selected/session-default model and a non-fallback routed active model. The adjacent #55425 behavior is present and shipped in the changelog, but it promotes the runtime model onto the main model line instead of preserving the configured default plus adding a separate active runtime line.

## Best Possible Solution

Keep this open and implement the split in the shared status renderer: keep `🧠 Model:` tied to the configured or explicit session selection, add `↪️ Active: provider/model` when the resolved active runtime model differs and fallback state is not active, preserve existing `↪️ Fallback:` output for failover, and add focused coverage in `src/auto-reply/status.test.ts` plus `src/agents/openclaw-tools.session-status.test.ts` for no-explicit-override runtime-routed sessions.
