## Summary

Current main still does not implement the requested selected-vs-active split for `session_status`. The renderer computes selected and active model identities, but it only emits the selected `🧠 Model:` line and a fallback-only `↪️ Fallback:` line, with no non-fallback `↪️ Active:` line for routed runtime drift.

## Best Possible Solution

Keep this issue open. Implement the split in the shared status rendering path: keep `🧠 Model:` tied to the configured or explicit session selection, add `↪️ Active: provider/model` when the resolved active runtime model differs and fallback state is not active, preserve existing `↪️ Fallback:` behavior for failover, and add focused coverage in `src/auto-reply/status.test.ts` plus the `session_status` tool tests. If the session entry lacks active identity, add a bounded last-`model-snapshot` or assistant metadata lookup without mutating the session default.
