## Summary

Current main still does not implement the requested `session_status` split between the selected/session-default model and the routed active model. The renderer can compute selected and active identities, but it only emits the selected `Model:` line plus a fallback-only line; non-fallback runtime routing still has no `Active:` line. The adjacent #55425 fix shipped the persisted-runtime fallback behavior, but that is narrower than this issue’s requested status-card truthfulness for routed active models.

## Best Possible Solution

Keep this issue open. Implement the split in the shared status rendering path: keep `🧠 Model:` tied to the configured or explicit session selection, add `↪️ Active: provider/model` when the resolved active runtime model differs and fallback state is not active, preserve existing `↪️ Fallback:` behavior for failover, and add focused coverage in `src/auto-reply/status.test.ts` plus `src/agents/openclaw-tools.session-status.test.ts`. If the live session entry lacks active identity, add a bounded last-`model-snapshot` or assistant metadata lookup without mutating the session default.
