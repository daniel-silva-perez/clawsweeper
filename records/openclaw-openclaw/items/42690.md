## Summary

Current main still does not implement the requested `session_status` split between the selected/session-default model and a non-fallback routed active model. The status renderer resolves both selected and active refs, and transcript fallback can recover an active model, but rendering only emits the `🧠 Model:` line plus `↪️ Fallback:` for persisted fallback state. The related merged #55425 fix is adjacent but narrower: it uses the persisted runtime model as the selected card model when no explicit override exists, rather than preserving the configured default and adding a separate active runtime line.

## Best Possible Solution

Keep this issue open. Implement the split in the shared status renderer: keep `🧠 Model:` tied to configured or explicit session selection, add `↪️ Active: provider/model` when the resolved active runtime model differs and fallback state is not active, preserve existing `↪️ Fallback:` behavior for model failover, and add focused coverage in `src/auto-reply/status.test.ts` plus `src/agents/openclaw-tools.session-status.test.ts` for no-explicit-override runtime-routed sessions.
