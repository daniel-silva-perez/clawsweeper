## Summary

Keep this PR open. Current main still does not implement the PR’s intended chat-channel session switching: there is no `/sessions` chat handler or registry entry, `/session` still only handles `idle` and `max-age`, and the session config/store contract has no `session.historyLimit` or `sessionHistory` retention model. The PR remains a plausible implementation candidate for the linked chat-session switching request, with security-sensitive session-state review still needed before merge.

## Best Possible Solution

Keep this PR open for normal maintainer review or replacement by a successor implementation. The best path is a focused chat command implementation that registers `/sessions`, supports `/session <number|sessionId|back>`, persists bounded per-SessionKey history, deliberately snapshots/restores or clears all session-scoped runtime and security fields, documents `session.historyLimit`, and includes regression coverage for direct-chat switching, disabled history, transcript paths, LRU eviction, in-flight run handling, and snapshot restore.
