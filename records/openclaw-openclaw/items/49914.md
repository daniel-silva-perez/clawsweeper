## Summary

Keep this PR open. Current main does not implement the PR’s intended chat-channel session switching: `/sessions` is not registered or loaded, `/session` still only manages focused conversation idle/max-age settings, and `session.historyLimit`/`sessionHistory` are absent from the current session config and store contract. The PR remains a plausible implementation candidate rather than obsolete maintainer-cleanup material.

## Best Possible Solution

Keep this PR open for normal maintainer review or replacement by a successor implementation. The best path is to land a focused chat-command session switcher that registers `/sessions`, supports `/session <number|sessionId|back>`, persists bounded per-SessionKey history, restores or clears all session-scoped runtime/security fields deliberately, documents `session.historyLimit`, and includes regression tests for direct-chat switching, history disabled, transcript paths, eviction, and snapshot restore.
