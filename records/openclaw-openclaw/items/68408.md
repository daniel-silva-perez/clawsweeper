## Summary

Keep open. Current main still does not implement the PR’s intended `memory_get` read-path filtering: daily files are read and returned raw for both partial and full reads. Main has a related sanitizer for dreaming’s own daily-ingestion path, but that does not protect callers that directly read `memory/YYYY-MM-DD.md`, so this PR remains useful but needs review fixes before merge.

## What This Changes

This PR adds a Plugin SDK markdown helper and wires `memory-core` daily-memory reads to strip managed Light Sleep and REM dreaming blocks from full and partial `QmdMemoryManager.readFile()` results, with focused tests.

## Best Possible Solution

Land a corrected memory-core read-path filter that applies after safe path resolution to both full and paginated daily-memory reads, strips only managed Light/REM blocks, preserves raw files for dreaming internals, and keeps ordinary user-authored formatting unchanged. If the helper remains public SDK API, update the baseline and relevant SDK docs/checks in the same change.
