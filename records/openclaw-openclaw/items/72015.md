## Summary

Keep this issue open. Current main contains partial mitigations for Active Memory timeouts, circuit breaking, and QMD startup laziness, but the central reliability/defaults request is still unresolved: Active Memory remains a blocking pre-reply hook, still inherits the current/session or primary model when no plugin model is pinned, and a draft unmerged PR (#73667) is already paired with this issue via closing syntax.

## What This Changes

The issue requests safer multi-agent reliability defaults for Active Memory and QMD startup by bounding or avoiding blocking pre-reply recall, avoiding slow inherited models, enforcing strict budgets/concurrency, and reducing QMD boot bursts.

## Best Possible Solution

Keep the issue open until #73667 or a replacement lands a maintainable default profile: pre-reply recall should have a strict user-visible budget or nonblocking/fail-open path, model selection should avoid surprising slow inherited models or clearly warn, stacked recall work should be bounded, and QMD startup should remain lazy or explicitly concurrency-limited for multi-agent gateways.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists by enabling Active Memory for an eligible chat session without `config.model` and using a slow inherited session or primary model: the `before_prompt_build` hook awaits the embedded recall before prompt construction can finish. The QMD side is also source-reproducible with multi-agent QMD `onBoot` plus default memory search enabled, where current tests show all QMD agents are initialized.

## Solution Assessment

No. Current main is only a partial mitigation: circuit breaking and QMD deferral help, but the blocking hook, inherited-model default, broad default timeout, and lack of nonblocking/deadline/concurrency config remain. The active narrow path is #73667, but maintainers should decide whether its bounded blocking approach is sufficient or whether true nonblocking recall and a safer model default are required.
