## Summary

Keep this issue open. Current main has partial mitigations for hung Active Memory recall and repeated timeouts, but the central reliability/defaults request is still unresolved: Active Memory remains a blocking pre-reply hook, still inherits the current/session or agent model when no plugin model is pinned, and open draft PR #73667 is already paired with this issue via closing syntax.

## What This Changes

The issue requests safer Active Memory and QMD defaults for multi-agent gateways, including nonblocking or strict-deadline recall, safer recall model selection, recall concurrency limits, QMD startup throttling or laziness, and operational diagnostics.

## Best Possible Solution

Keep this issue open until #73667 or a replacement lands a maintainable default profile: pre-reply recall should have a strict user-visible budget or nonblocking fail-open path, inherited slow models should be avoided or loudly surfaced, stacked recall work should be bounded, and QMD startup should be lazy, jittered, or concurrency-limited for multi-agent gateways.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists by enabling Active Memory for an eligible interactive session without config.model: the before_prompt_build hook awaits the embedded recall before prompt construction can finish. The QMD side is source-reproducible with multi-agent QMD startup and agents.defaults.memorySearch.enabled, where current tests assert all QMD agents initialize.

## Solution Assessment

No, current main is only a partial solution. The existing hard-return and circuit-breaker coverage help with hung or repeated timeouts, but the blocking hook, inherited-model default, broad default timeout plus setup grace, and missing nonblocking/deadline/concurrency settings remain.
