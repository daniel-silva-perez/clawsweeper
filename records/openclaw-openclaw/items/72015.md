## Summary

Keep open. Current main has added useful mitigations around Active Memory timeouts, a timeout circuit breaker, and lazy QMD gateway startup, but the central report is still valid: Active Memory remains a blocking pre-reply embedded run that inherits the active or primary model by default, and the issue is already paired with an open draft fix PR that should be reviewed rather than closing this issue early.

## What This Changes

The issue requests safer reliability defaults and controls for Active Memory recall and QMD startup on multi-agent gateways.

## Best Possible Solution

Keep the issue open until a focused reliability patch lands: make Active Memory fail-open with a short deadline or nonblocking mode, remove setup grace from the pre-reply critical path, add per-agent/session recall concurrency or dedupe/drop behavior, avoid silently inheriting heavy chat models without warning or a safer recall default, and keep QMD startup lazy/opt-in with bounded startup work and operator-visible diagnostics.

## Reproduction Assessment

Yes. The source path is clear: an eligible Active Memory before_prompt_build hook awaits an embedded recall subagent before the main prompt is built, and tests cover inherited session model selection plus timeout behavior. A live multi-agent CPU/latency reproduction was not run because this review is read-only.

## Solution Assessment

No, current main is not the complete fix. It narrows several failure modes, but the best path is to review and update the linked draft PR or a replacement so the blocking/defaults and startup-load concerns are addressed together.
