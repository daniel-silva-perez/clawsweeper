## Summary

Keep open. Current main has partial mitigations for the reliability path, but the central behavior remains a blocking Active Memory pre-reply hook that can inherit the session model, and an open draft PR with closing syntax is already the implementation candidate.

## What This Changes

The issue requests safer reliability defaults for Active Memory and QMD startup, including fail-open bounded recall, safer recall model selection, concurrency control, hard deadlines, QMD startup throttling, and operational visibility.

## Best Possible Solution

Keep the issue open while maintainers review, land, or supersede PR #73667 with a focused fix that bounds pre-reply recall latency, avoids unsafe heavy-model defaults, adds recall drop/dedupe or concurrency behavior, and preserves the newer opt-in QMD startup controls for multi-agent gateways.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: enable Active Memory for an eligible persistent chat session and make recall slow or timeout; the hook awaits recall before the main reply can proceed, and existing tests simulate timeout and hard-deadline behavior.

## Solution Assessment

Unclear. Current main is not the full fix because Active Memory is still a blocking pre-reply hook with model inheritance; PR #73667 is the right active candidate to review first, but it is still open and draft in the provided context.
