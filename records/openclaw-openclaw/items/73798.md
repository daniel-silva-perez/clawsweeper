## Summary

Keep this PR open. Current main still emits browser screenshot and AI snapshot `MEDIA:` lines through `shortenHomePath(...)`, while the shared media parser still rejects `~`-prefixed media candidates. The PR remains a focused implementation candidate for the linked browser media delivery bug and has recent author follow-up addressing the review gap.

## Best Possible Solution

Keep this PR open for maintainer review and CI, then land the producer-side browser CLI fix if checks pass. The right product direction is to emit absolute paths in machine-readable browser `MEDIA:` lines while preserving the shared parser's rejection of `~` and traversal-style media sources.
