## Summary

Keep this PR open. Current main still has the CLI/docs mismatch the PR targets: the interactive `openclaw agents add <id>` wizard seeds the `Workspace directory` prompt from the runtime resolver, while docs describe peer-level `workspace-<agentId>` paths. The linked bug #71889 remains open, and related implementation candidates #71899 and #71941 are also open, so this needs maintainer review and canonicalization rather than cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to pick one canonical implementation among this PR, #71899, and #71941, then land a narrow regression-tested wizard-prefill fix that suggests the documented peer-level `workspace-<id>` for new agents in the default setup while preserving existing runtime resolution and intentional shared-base configurations. After that merges, close the duplicate implementation PRs and let #71889 close through the merged fix.
