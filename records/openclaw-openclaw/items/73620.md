## Summary

Keep this PR open. Current main still does not implement the intended `agents add` wizard-prefill fix: the wizard still seeds `Workspace directory` from `resolveAgentWorkspaceDir`, while docs still describe peer-level `workspace-<agentId>` defaults. The PR remains an implementation candidate, but it overlaps with open candidates #71899 and #71941, so maintainers should choose a canonical fix rather than close this automatically.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose one canonical implementation among this PR, #71899, and #71941, then land a narrow regression-tested wizard-prefill fix that suggests peer-level `workspace-<id>` for new agents in the documented default setup while preserving existing runtime resolution and intentional shared-base configurations. After that merges, close the duplicate PRs and the linked bug through the merged fix.
