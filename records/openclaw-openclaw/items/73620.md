## Summary

Keep this PR open. Current main still does not implement the intended `agents add` wizard prefill change: the interactive `Workspace directory` prompt still uses `resolveAgentWorkspaceDir`, which nests a new non-default agent under `agents.defaults.workspace` when setup has written the main workspace there. This PR remains an active implementation candidate for open bug #71889, alongside open overlapping candidates #71899 and #71941.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to select one canonical implementation among this PR, #71899, and #71941, then land a narrow regression-tested wizard-prefill fix that suggests the documented peer-level workspace for new agents in the default setup while preserving existing runtime resolution and intentional shared-base configurations. After one implementation merges, close the duplicate implementation PRs and let #71889 close through the merged fix.
