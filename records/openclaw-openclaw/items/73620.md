## Summary

Keep open. Current main still has the reported wizard prefill path, and this PR is a relevant implementation, but the patch is not merge-ready because it changes every custom default workspace into a sibling suggestion and can bypass the shared-base behavior preserved for #59789.

## What This Changes

The PR adds and exports a peer-workspace suggestion helper, uses it for the interactive `agents add` new-agent workspace prompt, and adds helper-level tests.

## Best Possible Solution

Land one canonical, regression-tested wizard-prefill fix that suggests the documented peer-level workspace only for the setup/default workspace case, preserves existing-agent updates and intentional custom shared-base configurations, and then closes the linked bug through the merged PR.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: setup writes `agents.defaults.workspace`, `agents add` uses `resolveAgentWorkspaceDir` as the prompt default, and the resolver appends non-default ids under that fallback.

## Solution Assessment

No. The wizard-only direction is sound, but this implementation is too broad; a safer fix should gate the peer-level suggestion to the default setup workspace case or defer to `resolveAgentWorkspaceDir` for custom shared roots.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve custom shared workspace defaults:** `src/agents/agent-scope-config.ts:204`
  - body: When `agents.defaults.workspace` is an intentional shared base, current main resolves non-default agents under that base and has regression coverage for it. This helper instead builds a sibling of the resolved default agent workspace for every new agent, and the wizard then persists that value, so accepting the prompt can silently bypass the configured shared root. Gate the peer-level suggestion to the setup/default workspace case or defer to the resolver for custom shared roots.
  - confidence: 0.84
