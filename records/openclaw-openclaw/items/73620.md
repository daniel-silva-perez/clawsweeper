## Summary

Keep this PR open. Current main still does not implement the intended wizard-prefill change: `openclaw agents add <id>` continues to seed the interactive `Workspace directory` prompt from the runtime workspace resolver, which nests non-default agents under `agents.defaults.workspace` when that fallback is set. The PR remains an active implementation candidate for open bug #71889, with overlapping open candidates #71899 and #71941 that need maintainer selection rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose a canonical implementation among this PR, #71899, and #71941, then land a narrow regression-tested wizard-prefill fix that suggests the documented peer-level `workspace-<id>` for new agents in the default setup while preserving existing runtime resolution and intentional shared-base configurations. After one implementation merges, close the duplicate implementation PRs and let the linked bug close through the merged fix.
