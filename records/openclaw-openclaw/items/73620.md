## Summary

Keep open. Current main and the latest release still have the reported wizard prefill behavior, so the PR is not obsolete, but the branch has a blocking semantic regression around custom shared workspace defaults and competes with two other open implementations for the same bug.

## What This Changes

This PR adds a peer-workspace suggestion helper, exports it through the agent scope barrel, uses it for new interactive `agents add` workspace defaults, and adds resolver tests for default and configured workspace cases.

## Best Possible Solution

Land one narrow regression-tested fix for #71889 that suggests the documented peer-level workspace for brand-new agents in the setup/default case, preserves existing-agent updates and intentional shared-base configurations, adds direct wizard prompt coverage, and then closes the duplicate implementation PRs after the selected fix merges.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: setup persists `agents.defaults.workspace`, the interactive wizard passes `resolveAgentWorkspaceDir(cfg, agentId)` into the workspace prompt, and the non-default resolver branch appends the new id under that fallback.

## Solution Assessment

No. The wizard-only direction is appropriate, but this branch applies the sibling-path suggestion to custom configured defaults too; a safer version should gate the peer-level suggestion to the documented default/setup case or otherwise preserve intentional shared roots.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve custom shared workspace defaults:** `src/agents/agent-scope-config.ts:204`
  - body: When `agents.defaults.workspace` is a custom shared base, current main intentionally resolves non-default agents under that base, and the #59789 regression test covers that behavior. This helper always derives a sibling of the default agent workspace, so accepting the wizard prompt can turn a configured shared root such as `/shared-ws/tutor` into `/shared-ws-tutor`. Gate the peer-level suggestion to the setup/default workspace case, or defer to `resolveAgentWorkspaceDir` for custom shared roots.
  - confidence: 0.84
