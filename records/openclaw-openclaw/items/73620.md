## Summary

Keep open. The PR targets a real current-main bug in the `agents add` wizard, but the branch is not merge-ready because its new helper converts custom shared workspace defaults into sibling paths, conflicting with the existing #59789 shared-base contract and regression test.

## What This Changes

The PR adds a peer-workspace suggestion helper, exports it, uses it for new `openclaw agents add` wizard defaults, and adds agent-scope tests for default and configured workspace cases.

## Best Possible Solution

Land one narrow regression-tested fix for the wizard prefill that suggests the documented peer-level workspace only for the default setup workspace case, preserves existing-agent updates and intentional shared-base configurations, and then close the overlapping implementation PRs once the canonical fix merges.

## Reproduction Assessment

Yes. Source inspection is enough for the original bug, and the PR regression is reproducible from a config with `agents.defaults.workspace` as a custom shared base plus a separate default-agent workspace.

## Solution Assessment

No. The wizard-only direction is sound, but this patch is broader than the maintainable fix because it changes suggestions for custom shared roots; the safer path is to gate peer-level suggestions to the setup/default workspace case or defer to `resolveAgentWorkspaceDir` otherwise.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve custom shared workspace defaults:** `src/agents/agent-scope-config.ts:204`
  - body: When `agents.defaults.workspace` is a custom shared base, current main intentionally resolves non-default agents under that base, and the #59789 regression test covers that contract. This helper always derives a sibling of the default agent workspace, so accepting the `agents add` prompt can persist a path outside the configured shared base. Gate the peer-level suggestion to the default setup workspace case, or defer to `resolveAgentWorkspaceDir` for custom shared roots.
  - confidence: 0.84
