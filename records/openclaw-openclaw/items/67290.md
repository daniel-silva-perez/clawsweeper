## Summary

Keep open. Current main still has the duplicated Usage header behavior, so the PR is not obsolete; however, the proposed patch still has a blocking i18n parity problem because it removes the English key while leaving several shipped locale bundles and translation-memory entries untouched.

## What This Changes

The PR removes the local Usage page title/subtitle block, deletes `usage.page.subtitle` from English plus a subset of locale and translation-memory files, and adds small Usage CSS helper rules without changing usage data fetching or routing.

## Best Possible Solution

Land a focused Usage-header fix that removes the desktop duplicate, preserves the Filters card context, regenerates every shipped locale and `.i18n` metadata from the English source, and either documents or tests the intended mobile heading behavior.

## Reproduction Assessment

Yes. Current main statically reproduces the duplicate composition: the app shell renders the shared Usage title/subtitle and `renderUsage` also renders a local Usage title/subtitle block; the related bug report includes screenshot evidence for the same path.

## Solution Assessment

No, not as currently authored. Removing the local Usage header is the narrow right direction, but the patch needs complete i18n regeneration and an explicit maintainer call on mobile heading behavior before it is the safest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Regenerate every locale before dropping the English key:** `ui/src/i18n/locales/en.ts:657-659`
  - body: Deleting `usage.page.subtitle` from English changes the expected flattened key set, but this PR leaves shipped locales and translation-memory files for `ar`, `it`, `th`, `vi`, `nl`, and `fa` untouched. `translate.test.ts` compares every shipped locale against English, so those locales will now have an extra key and the i18n parity gate should fail. Please remove the key through the normal i18n sync flow so all locale bundles and `.i18n` metadata update together.
  - confidence: 0.91
