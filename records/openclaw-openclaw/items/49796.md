## Summary

Keep this PR open. Current main still returns and persists session transcript paths anchored to the real symlink target, while this PR's stated purpose is to preserve caller-visible logical session paths and remap legacy real paths back to that logical namespace. Main has a doctor-side realpath comparison mitigation, but that only addresses the orphan-warning symptom, not the stored-path portability issue from the linked bug.

## Best Possible Solution

Keep this PR open and review or adapt it against current main. The likely product fix still belongs in src/config/sessions/paths.ts and resolveAndPersistSessionFile: keep realpath-based containment checks, return and persist logical paths under the caller-visible sessions directory, preserve sibling-agent fallback safety, and keep regression coverage for symlinked sessions, legacy real-path remapping, sibling fallback, and doctor orphan detection.
