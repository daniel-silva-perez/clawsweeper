## Summary

Keep this PR open. Current main still resolves and persists session transcript paths through the real symlink target in the core session path resolver, while the PR’s central change is to preserve caller-visible logical paths for symlinked state/session directories. Main has a doctor-side realpath comparison mitigation, but that only addresses the orphan-warning symptom and not the stored-path portability problem described in the linked bug #49708.

## Best Possible Solution

Keep this PR open and review or adapt it against current main. The likely fix still belongs in src/config/sessions/paths.ts and resolveAndPersistSessionFile: use realpaths for containment checks, return and persist logical paths under the caller-visible sessions directory, preserve sibling fallback safety, and keep regression coverage for symlinked sessions, legacy real-path remapping, sibling fallback, and doctor orphan detection.
