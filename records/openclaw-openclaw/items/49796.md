## Summary

Keep this PR open. Current main partially mitigates the doctor orphan warning by comparing transcript realpaths, but it still does not implement the PR's core behavior: preserving and persisting caller-visible logical session paths when the sessions directory is reached through a symlink. The linked bug #49708 is still open, and this PR remains a plausible implementation candidate.

## Best Possible Solution

Keep the PR open and review or adapt it against current main. The likely best implementation remains in src/config/sessions/paths.ts and the session persistence path: use realpaths only for containment checks, return logical paths under the caller-visible sessions directory, persist those logical paths through resolveAndPersistSessionFile, and keep focused regression tests for symlinked session directories, legacy real-path remapping, sibling fallback, and doctor orphan detection.
