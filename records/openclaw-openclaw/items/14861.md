## Summary

Keep open. Current main still lacks the requested structured Gateway boot outcome summary, and the concrete Gmail watcher missing-account path remains silent during normal boot because startup only invokes the Gmail lifecycle wrapper when an account is already configured. Existing startup trace timing, generic startup logs, and plugin initialization warnings are adjacent observability, but they do not report started/skipped/failed sidecar or hook outcomes.

## What This Changes

The issue requests observable Gateway boot startup outcomes for subsystems and hooks, especially skipped or failed Gmail watcher startup reasons.

## Best Possible Solution

Add one concise, structured, redacted Gateway boot outcome collector after sidecars settle, covering the startup outcomes the gateway already knows how to observe and preserving current readiness and failure-swallowing semantics.
