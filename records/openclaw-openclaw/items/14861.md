## Summary

Keep open. Current main still lacks the requested Gateway boot outcome summary: startup has timing traces, readiness gating, and some targeted error logs, but no structured started/skipped/failed collector, and the Gmail missing-account case remains silent during normal Gateway startup. The existing review discussion correctly identified the same remaining gap, and the current source still matches that assessment.

## What This Changes

The issue requests a Gateway boot summary that reports which startup subsystems, sidecars, hooks, and watchers started, skipped, or failed.

## Best Possible Solution

Add a structured, redacted Gateway startup outcome collector that emits one concise normal boot summary after startup sidecars settle, covering Gmail watcher, internal hooks, channels, plugin services, memory scheduling, recovery sidecars, and gateway_start hooks without changing existing readiness or failure semantics.

## Repair Work Prompt

Implement a structured, redacted Gateway startup outcome collector for this issue. Current main's startGatewaySidecars returns only pluginServices, and Gmail missing-account startup is silent because Gateway startup bypasses startGmailWatcherWithLogs unless hooks.gmail.account is set. Add a small internal collector around startGatewaySidecars and post-attach startup that records safe started/skipped/failed outcomes for Gmail watcher, internal hooks, channel startup, plugin services, qmd memory startup scheduling, restart/session recovery sidecars, and gateway_start hooks. Emit one concise normal [gateway] boot summary after sidecars settle, near the existing gateway ready log. For Gmail, ensure hooks.enabled=true with missing hooks.gmail.account is represented as skipped/not started with reason "no gmail account configured" without warning-level noise. Keep messages redacted: do not include account IDs, hostnames, credentials, local paths, webhook payloads, or provider secrets. Preserve readiness gating, deferSidecars behavior, and best-effort sidecar semantics; do not convert existing non-fatal startup failures into fatal errors. Add focused regression coverage for hooks.enabled=true with missing hooks.gmail.account and for representative started/skipped/failed summary output. Add a user-facing changelog entry only if the active-version changelog policy requires one for this Gateway logging enhancement, crediting the reporter appropriately.
