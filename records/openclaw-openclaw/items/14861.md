## Summary

Keep open. Current main still has timing traces and targeted startup logs, but it does not implement the requested aggregate Gateway boot summary with per-subsystem started/skipped/failed outcomes. The Gmail missing-account case remains silent in normal startup because the gateway only calls the Gmail watcher wrapper when an account is configured, and the wrapper suppresses that reason if invoked directly.

## Best Possible Solution

Implement a structured, redacted Gateway startup outcome collector around startGatewaySidecars and post-attach startup. It should record safe started/skipped/failed outcomes for Gmail watcher, internal hooks, channels, plugin services, memory scheduling, and recovery sidecars, then emit one concise normal boot summary after sidecars settle, with regression coverage for hooks.enabled=true and missing hooks.gmail.account.
