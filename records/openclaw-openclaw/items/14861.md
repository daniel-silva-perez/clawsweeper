## Summary

Keep open. Current main still lacks the requested structured Gateway boot outcome summary: it has timing traces, the generic startup log, and a plugin-init failure warning, but not a started/skipped/failed collector for sidecars/hooks. The concrete Gmail watcher gap remains: missing account is still silent during normal startup, and I found no open PR with closing syntax for this issue.

## What This Changes

Request to add Gateway boot observability that reports which startup subsystems and hooks initialized, skipped, or failed, with Gmail watcher missing-account and missing-gog cases surfaced after restarts.

## Best Possible Solution

Add a structured, redacted Gateway startup outcome collector that emits one concise normal boot summary after sidecars settle, covering Gmail watcher, internal hooks, channels, plugin services, memory scheduling, recovery sidecars, and gateway_start hooks while preserving existing readiness and failure behavior.
