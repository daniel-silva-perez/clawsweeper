Keep open. Current main still does not emit the requested structured Gateway boot outcome summary, and the concrete Gmail watcher missing-account path remains silent because startup skips the lifecycle wrapper unless an account is already configured. The issue requests Gateway startup observability that reports which subsystems, hooks, and sidecars started, skipped, or failed.

## Best Possible Solution

Add one concise, structured, redacted Gateway boot outcome summary after sidecars settle, with regression coverage proving Gmail watcher skipped/failed reasons are visible without changing startup behavior.
