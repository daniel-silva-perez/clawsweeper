## Summary

Keep open. Current main still does not provide the requested Gateway boot outcome summary: startup has timing traces, generic readiness logs, plugin-init failure warnings, and narrow startup-task logging, but there is no single started/skipped/failed collector for Gateway sidecars and hooks. The Gmail watcher example remains valid because missing account is still silent on normal startup.

## What This Changes

Add Gateway boot observability that reports which subsystems, hooks, and sidecars initialized, skipped, or failed, including Gmail watcher non-start reasons such as missing account or missing `gog`.

## Best Possible Solution

Gateway should emit one concise, structured, redacted boot outcome summary after sidecars settle, with tests proving Gmail watcher skipped/failed reasons are visible and existing startup readiness behavior is preserved.
