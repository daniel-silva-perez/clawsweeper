## Summary

Keep open. Current main still lacks the requested structured Gateway boot outcome summary, and the concrete Gmail watcher missing-account path remains silent because startup only calls the Gmail lifecycle wrapper when an account is already configured. Existing startup logs, startup trace timing, and plugin initialization warnings improve adjacent observability but do not cover sidecar/hook outcomes as requested.

## What This Changes

The issue requests observable Gateway boot startup outcomes for subsystems and hooks, especially skipped or failed Gmail watcher startup reasons.

## Best Possible Solution

Add one concise, structured, redacted Gateway boot outcome summary after sidecars settle, with regression coverage proving Gmail watcher skipped and failed reasons are visible without changing startup readiness or failure semantics.
