## Summary

Keep open. Current main has partial version-coupled docs support through packaged docs, local docs path discovery, prompt guidance, and config schema lookup, but the central request remains missing: there is no native local-first docs search/get/API retrieval surface for agents, and the CLI docs search still calls the live docs MCP endpoint.

## What This Changes

Request to ship version-matched local documentation with OpenClaw releases and expose local-first docs retrieval tools for agent-guided onboarding.

## Best Possible Solution

Define a version-coupled docs manifest/search index as part of packaging or release, expose local-first docs search/get lookup to agents through a maintained runtime surface, keep live Mintlify search as explicit fallback, and document/tests the installed-package behavior so prompt guidance is not mistaken for the native retrieval layer.

## Reproduction Assessment

Not applicable: this is a feature/product request rather than a runtime bug report. The missing capability is high-confidence by source inspection of the gateway tool actions, gateway method list, docs path resolver, system prompt, and CLI docs command.

## Solution Assessment

Unclear: the requested product direction is plausible and aligned with onboarding reliability, but the best concrete interface still needs maintainer choice. A local-first docs manifest/index with an agent-facing retrieval API is safer than relying only on prompt guidance, generic file reads, or the live docs MCP endpoint.
