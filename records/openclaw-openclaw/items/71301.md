## Summary

Keep open. Current main has useful groundwork, including packaged docs, local-doc prompt guidance, and config schema lookup, but it still lacks the requested native local-first docs search/get/API/changed-since retrieval surface for agents. The open live-doc drift report #48920 remains complementary evidence that installed-version versus latest-doc semantics still need a product/API decision.

## What This Changes

Request to ship version-matched bundled docs with releases and expose native local-first documentation retrieval for agent-guided onboarding.

## Best Possible Solution

Define a release-packaged docs manifest/search index, expose maintained local-first docs search/get/schema/API lookup to agents, keep live Mintlify as an explicit latest-doc fallback, and document/test installed-version versus latest-version behavior.

## Reproduction Assessment

Not applicable as a runtime reproduction because this is a feature request. Source inspection gives a high-confidence verification path across package allowlisting, docs path resolution, prompt guidance, Gateway tool actions, Gateway RPC methods, and live-backed CLI docs search.

## Solution Assessment

Unclear until maintainers choose the package/index contract and agent-facing API. The local-first retrieval direction is appropriate, but the durable solution should be a deliberate core contract rather than prompt-only guidance or live-doc search.
