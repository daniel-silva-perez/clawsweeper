## Summary

Keep open. Current main has packaged docs discovery and prompt guidance, but it still lacks the requested native local-first docs search/get/API retrieval layer for agents; the CLI docs command remains live-Mintlify backed and the Gateway/agent tool surfaces only expose config schema lookup, not general docs retrieval.

## What This Changes

The issue requests release-packaged, version-matched OpenClaw docs plus first-class local docs retrieval tools for agent-guided onboarding.

## Best Possible Solution

Define a release-packaged docs manifest/search index, expose local-first docs search/get/schema/API retrieval through a maintained agent-facing surface, keep live Mintlify as an explicit latest-docs fallback, and add tests/docs that distinguish packaged docs discovery from native retrieval.

## Reproduction Assessment

Not applicable as a runtime reproduction because this is a feature request. Source inspection gives high-confidence proof that current main has local docs path discovery and config schema lookup but no native local docs retrieval surface.

## Solution Assessment

Unclear until maintainers choose the package/index contract and agent-facing API. The requested local-first retrieval direction is sound, but prompt guidance and live `openclaw docs` search are not enough to solve the version-matched onboarding problem.
