## Summary

Keep open. Current main has partial support through packaged docs, local docs path discovery, prompt guidance, and `config.schema.lookup`, but the core request remains open: agents still do not have a native local-first docs search/get/API retrieval surface backed by a version-matched docs index.

## What This Changes

The issue requests release-packaged, version-matched local OpenClaw documentation plus native local-first docs retrieval tools for agents during onboarding.

## Best Possible Solution

Define and ship a release-packaged docs manifest/search index, expose local-first docs search/get/schema/API lookup through a maintained agent-facing surface, keep live Mintlify as an explicit fallback, and add tests/docs that distinguish packaged docs discovery from native retrieval.

## Reproduction Assessment

Not applicable as a runtime reproduction because this is a feature request. Source inspection gives high-confidence proof of the missing capability across package allowlist, docs path resolution, system prompt, Gateway tool actions, Gateway RPC inventory, and CLI docs search.

## Solution Assessment

Unclear until maintainers choose the package/index contract and agent-facing API. The requested local-first retrieval direction is narrower and more maintainable than relying on prompt guidance, ad hoc file reads, or the live docs MCP endpoint alone.
