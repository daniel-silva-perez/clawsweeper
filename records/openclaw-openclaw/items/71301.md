## Summary

Keep open. Current main has partial support through packaged docs, local docs path discovery, prompt guidance, and `config.schema.lookup`, but the requested native, version-matched, local-first docs retrieval layer for agents is still missing.

## What This Changes

Request to ship a version-matched OpenClaw docs bundle with releases and expose local-first docs search/get/schema/API retrieval to agents for onboarding and configuration guidance.

## Best Possible Solution

Define and ship a release-packaged docs manifest/search index, expose local-first docs search/get/schema/API lookup through a maintained agent-facing surface, keep live Mintlify as an explicit fallback, and add docs/tests that distinguish packaged docs discovery from native retrieval.

## Reproduction Assessment

Not applicable as a runtime reproduction because this is a feature request. Source inspection gives high-confidence proof of the missing capability across package allowlist, docs path resolution, system prompt, Gateway tool actions, Gateway RPC inventory, and CLI docs search.

## Solution Assessment

Unclear until maintainers choose the package/index contract and agent-facing API. The local-first retrieval direction is the right shape, but it needs a deliberate core contract rather than ad hoc prompt guidance or live-doc search.
