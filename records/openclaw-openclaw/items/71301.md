## Summary

Keep open. Current main has partial groundwork: packaged docs are included, agents are pointed at local docs, and config schema lookup exists, but there is still no native local-first docs search/get/API/changed-since retrieval surface for agents; the related live-doc drift report #48920 remains complementary open evidence.

## What This Changes

Request to ship version-matched bundled docs with releases and expose native local-first documentation retrieval for agent-guided onboarding.

## Best Possible Solution

Define a release-packaged docs manifest/search index, expose maintained local-first docs search/get/schema/API lookup to agents, keep live Mintlify as an explicit latest-docs fallback, and document/test the installed-version versus latest-version behavior.

## Reproduction Assessment

Not applicable as a runtime reproduction because this is a feature request. Source inspection gives a high-confidence way to verify the gap across package allowlisting, docs path resolution, prompt guidance, Gateway tool actions, Gateway RPC methods, and live-backed CLI docs search.

## Solution Assessment

Unclear until maintainers choose the package/index contract and agent-facing API. The local-first retrieval direction is appropriate, but the best implementation should be a deliberate core contract rather than prompt-only guidance or live-doc search.
