## Summary

Keep this issue open. Current main still does not implement Mattermost WebSocket protocol ping/pong keepalive, and the active implementation candidate, PR #57621, is still open and unmerged. The existing bot `update_at` polling and configurable stale-event threshold are useful fallback work, but they do not prove transport liveness for a half-dead Mattermost socket.

## Best Possible Solution

Keep this issue open and review or land PR #57621, or an equivalent Mattermost-owned change in `extensions/mattermost/src/mattermost/monitor-websocket.ts` that adds `ws.ping()`, handles `pong`, terminates after missed pongs so `runWithReconnect` reconnects, cleans up timers on close/error/abort, and adds focused tests using the fake WebSocket. After that merges, close this issue and related Mattermost half-dead WebSocket reports such as #41837 as implemented or duplicates of the landed fix.
