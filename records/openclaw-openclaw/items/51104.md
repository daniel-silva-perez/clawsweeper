## Summary

Keep this issue open. Current main still lacks the primary Mattermost WebSocket ping/pong keepalive requested here; it only has close/error handling plus a REST `update_at` poll for bot-account changes. The secondary stale-threshold config exists, but it does not provide transport-level liveness for a half-dead Mattermost socket. PR #57621 is the active open implementation candidate, and the KeepMyClaw snapshot comment is complementary operational context rather than a core fix.

## Best Possible Solution

Keep this issue open and review or land PR #57621, or an equivalent Mattermost-owned change in `extensions/mattermost/src/mattermost/monitor-websocket.ts` that adds `ws.ping()`, handles `pong`, terminates on missed pongs so `runWithReconnect` reconnects, and adds focused tests for the timeout behavior. After that merges, close this issue and related Mattermost half-dead WebSocket reports as implemented or duplicates of the landed fix.
