## Summary

Keep this PR open. Current main still does not implement Feishu text keyword mention triggers, and this PR is a focused implementation candidate, but it is not merge-ready because the existing review comments identify real functional gaps in broadcast dispatch, effective WasMentioned propagation, empty trigger validation, and per-group/config precedence.

## Best Possible Solution

Keep this PR open and request a Feishu plugin revision. A mergeable version should choose the final config contract, preferably reusing the shared mentionPatterns helpers unless maintainers explicitly want a Feishu-specific field; normalize or reject empty triggers; resolve top-level/account/group precedence alongside requireMention; treat keyword matches as effective mentions for single-agent and broadcast dispatch; update activeAgentId and WasMentioned; and add focused schema, policy, bot, broadcast, and Feishu docs coverage.

## Risks / Open Questions

- Merging the PR as-is would likely let keyword-only messages through the early gate in single-agent paths but still fail or behave inconsistently in Feishu broadcast groups because activeAgentId and WasMentioned remain tied to ctx.mentionedBot.
- The proposed z.array(z.string()) plus String.includes check accepts empty strings; an empty trigger would match every message and effectively bypass requireMention.
- The config contract is unsettled: a Feishu-specific mentionTriggers field overlaps with the existing shared messages.groupChat.mentionPatterns and agents.list[].groupChat.mentionPatterns model used by other channels.
- No current tests or docs cover Feishu keyword triggers, per-group/account precedence, empty-trigger rejection, or broadcast effective-mention propagation.
