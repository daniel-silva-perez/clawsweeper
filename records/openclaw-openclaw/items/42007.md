## Risks / Open Questions

- Merging the PR as-is could accept keyword-only Feishu messages but fail to create a Feishu reply dispatcher in broadcast groups because activeAgentId and WasMentioned are not updated for keyword triggers.
- The proposed schema accepts empty strings, and ctx.content.includes("") would make every group message satisfy the trigger list, effectively bypassing requireMention.
- The user-facing contract is incomplete without normalized non-empty triggers, top-level/account/wildcard-group/exact-group precedence, focused Feishu tests, and Feishu docs/config reference updates.
