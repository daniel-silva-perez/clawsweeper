## Summary

Keep this PR open. Current main still does not implement Feishu text-based mention triggers, and this PR remains a focused Feishu plugin implementation candidate, but it needs revision before merge for the existing broadcast reply path, empty-trigger validation, config precedence, docs, and tests.

## Best Possible Solution

Keep this PR open and request a Feishu plugin revision. A mergeable version should choose the final config contract, preferably reusing the existing shared mentionPatterns helpers unless maintainers explicitly want a Feishu-specific field; normalize and reject empty triggers; resolve top-level/account/group precedence alongside requireMention; treat keyword matches as effective mentions for single-agent and broadcast dispatch; update activeAgentId and WasMentioned; and add focused schema, policy, bot, broadcast, and Feishu docs coverage.

## Risks / Open Questions

- Merging the PR as-is could make keyword-only Feishu messages pass the early gate but still fail to create a real reply dispatcher in broadcast groups because activeAgentId and WasMentioned remain tied to ctx.mentionedBot.
- The proposed schema accepts empty strings, and String.includes("") would make every group message satisfy the trigger list, effectively bypassing requireMention.
- The config contract is unsettled: Feishu-specific mentionTriggers overlaps with the existing shared messages.groupChat.mentionPatterns / agents.list[].groupChat.mentionPatterns model used by other channels.
- No current tests or docs cover Feishu keyword triggers, group/account precedence, empty-trigger rejection, or broadcast effective-mention propagation.
