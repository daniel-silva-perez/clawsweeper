## Summary

Keep this PR open. Current main still does not implement Feishu text keyword mention triggers: the Feishu schemas and docs omit `mentionTriggers`, group handling still drops non-@mentioned messages when `requireMention` is true, and broadcast dispatch still derives active reply behavior from `ctx.mentionedBot`. The PR is a focused implementation candidate, but the provided review comments identify real merge blockers around broadcast dispatch, `WasMentioned`, empty trigger validation, and config precedence.

## Best Possible Solution

Keep this PR open and request a Feishu plugin revision. A mergeable version should choose the final config contract, preferably reusing or intentionally aligning with shared `mentionPatterns`; validate or normalize empty trigger entries; define top-level/account/group/agent precedence; treat keyword matches as effective mentions for both single-agent and broadcast dispatch; update `activeAgentId` and `WasMentioned`; and add focused schema, policy, bot, broadcast, and Feishu docs coverage.
