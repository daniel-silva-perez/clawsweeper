## Summary

Keep open. Current main still lacks Feishu `mentionTriggers` in schema, policy, inbound gating, tests, and docs, so this PR is not obsolete or already implemented. It remains a focused Feishu plugin implementation candidate, but it needs revision before merge because the current diff does not fully cover broadcast dispatch semantics, empty-trigger validation, per-group precedence, tests, or docs.

## Best Possible Solution

Keep this PR open and request a Feishu plugin revision. A mergeable version should add normalized non-empty `mentionTriggers` with clear top-level, account, wildcard-group, and exact-group precedence; treat keyword matches as effective mentions for single-agent and broadcast dispatch; update `activeAgentId` and `WasMentioned`; and add focused config/policy/bot tests plus Feishu docs at https://docs.openclaw.ai/channels/feishu.

## Risks / Open Questions

- Merging the current PR as-is could silently fail in Feishu broadcast groups because keyword-only messages would pass the new early gate while no active Feishu reply dispatcher is selected.
- The proposed empty-string trigger handling would effectively bypass `requireMention` for every group message if an empty trigger is configured.
- The user-facing contract remains incomplete without normalized non-empty triggers, account/group precedence, Feishu tests, and docs/config-reference updates.
