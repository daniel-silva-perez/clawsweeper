## Summary

Keep this PR open. Current main still does not publish Feishu connection lifecycle or transport-activity status into the gateway health snapshot, so the intended fix is not implemented on main. The PR direction is useful, and the provided review discussion already resolved some early callback/timing concerns, but the latest PR context still shows an incomplete status-threading path against the current code.

## Best Possible Solution

Keep this PR open for maintainer review or contributor revision. The best path is to consolidate it with #39487/#39489 into one Feishu-owned implementation that passes ctx.setStatus end to end, publishes connected/mode/lastConnectedAt on websocket/webhook start, connected:false plus disconnect/error details on abort/error/shutdown, and updates lastTransportActivityAt from real transport heartbeat/socket/webhook activity. Add focused Feishu tests proving the runtime snapshot drives the gateway health policy.

## Risks / Open Questions
