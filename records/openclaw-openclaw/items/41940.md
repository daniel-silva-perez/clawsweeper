## Summary

Keep open. Current main still does not implement Feishu lifecycle status reporting into the gateway health snapshot, and the PR head described in the provided context remains incomplete against the current code path. The direction is useful, but it is not merge-ready and current main has not made it obsolete.

## Best Possible Solution

Keep this PR open for maintainer review or contributor revision, but do not merge it as-is. The best path is to choose one canonical Feishu health-signal implementation with #39487/#39489, then wire ctx.setStatus end to end from extensions/feishu/src/channel.ts through monitorFeishuProvider and monitorSingleAccount into monitor.transport.ts. The completed fix should publish connected/mode/lastConnectedAt on websocket/webhook start, connected:false and error/disconnect fields on abort/error/shutdown, lastTransportActivityAt from real transport activity, optional lastEventAt for app events, and focused tests proving Feishu snapshots drive the gateway health policy.
