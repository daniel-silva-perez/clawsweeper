## Summary

Keep open. Current main still does not implement Feishu lifecycle/status reporting to the gateway health monitor, and this PR head is not complete enough to merge as the canonical fix because the new callback does not reach the real Feishu account monitor or channel startup path and it does not satisfy the current lastTransportActivityAt-based stale-socket contract.

## Best Possible Solution

Keep this PR open for maintainer review or contributor revision unless maintainers explicitly choose #39489 as the canonical replacement. The best fix is Feishu-owned: pass ctx.setStatus from channel.ts through monitorFeishuProvider and monitorSingleAccount into both transport modes and event handlers; report connected, mode, lastConnectedAt, lastEventAt, and validated lastTransportActivityAt where appropriate; publish connected:false plus disconnect/error state on abort/error/shutdown; and add focused Feishu tests proving the runtime snapshot drives the current gateway health policy.
