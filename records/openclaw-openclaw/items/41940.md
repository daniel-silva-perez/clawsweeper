## Summary

Keep open. Current main still lacks Feishu lifecycle/status reporting into the gateway health monitor, and this PR head is not a complete implementation because its provided diff does not update the real Feishu channel startup or account monitor path. The overlapping open Feishu health-status PRs should be consolidated or one should be chosen as the canonical fix.

## Best Possible Solution

Keep this PR open for maintainer review or contributor revision unless maintainers explicitly consolidate the work into #39489 or another canonical PR. The best fix is Feishu-owned: pass ctx.setStatus/statusSink through channel.ts, monitor.ts, monitor.account.ts, and monitor.transport.ts; publish connected, mode, lastConnectedAt, lastEventAt, and validated lastTransportActivityAt; clear connected on abort/error/shutdown; and add focused tests proving the runtime snapshot drives the current gateway health policy.
