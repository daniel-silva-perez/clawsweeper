## Summary

Keep this PR open. Current main still lacks the shared normalization for leading `functions ` tool-call IDs and `functions.` / `functions ` prefixed tool names in transcript repair and ID sanitization. The prior related PR #39091 is closed unmerged, and this PR remains the active implementation candidate, though its bundled host=node exec approval-path change needs maintainer security review or a split before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a narrow agent-side fix that normalizes only the leading `functions ` versus `functions.` ID equivalence where matching needs it, strips `functions.` / `functions ` prefixes before transcript-repair allowed-name checks and `sessions_spawn` redaction, preserves raw IDs for pending-state and replay-safe signed-thinking behavior, and keeps the regression tests. Review or split the host=node exec workdir/approval change before merge.
