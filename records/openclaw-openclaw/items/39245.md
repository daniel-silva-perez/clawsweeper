## Summary

Keep this PR open. Current main has related Kimi support for native dot-form IDs and live/replay tool-name cleanup, but it still lacks this PR's central leading `functions ` to `functions.` ID-equivalence behavior and persisted transcript prefix stripping. The predecessor #39091 is closed unmerged, and the contributor has kept this PR active while addressing prior bot review feedback, so this remains the live implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or adapt the narrow agent-side fix that treats only leading `functions ` and `functions.` tool-call IDs as equivalent where sanitizer pairing needs it, strips `functions.` / `functions ` prefixes before persisted transcript allowlist checks, `sessions_spawn` redaction, and tool-result name normalization, preserves raw IDs for pending-state and replay-safe thinking behavior, keeps the regression tests, and separately reviews or splits the node-host exec explicit-workdir approval-path change.
