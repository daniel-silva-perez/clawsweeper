## Summary

Keep this PR open. Current main has related Kimi native `functions.<name>:<index>` ID preservation and live/replay tool-name normalization, but it still does not implement this PR's main `functions ` versus `functions.` ID-equivalence path or persisted transcript repair for `functions.` / `functions ` prefixed tool names. The earlier #39091 attempt is closed unmerged, so this PR remains the active implementation candidate.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or adapt a narrow agent-side fix that treats only leading `functions ` and `functions.` tool-call IDs as equivalent where sanitizer pairing needs it, strips `functions.` / `functions ` prefixes before persisted transcript repair and tool-result name normalization, preserves raw IDs for pending-state and replay-safe thinking behavior, and keeps the regression coverage. Review or split the bundled node exec explicit-workdir approval-path change before merging.
