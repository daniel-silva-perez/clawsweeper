## Summary

Keep open. The PR fixes part of the reported best-effort action-discovery path, but it does not cover the Telegram embedded prompt-prep path that the linked report names: runtime capability and reaction-guidance collection still call Telegram account helpers that resolve the raw SecretRef-backed token before model output.

## What This Changes

The PR adds unresolved-SecretRef handling to Discord and Telegram channel-action discovery, adds regression tests for `describeMessageTool`, and adds a changelog entry for #75433.

## Best Possible Solution

Keep runtime send/probe token resolution strict, but make every best-effort Telegram prompt/discovery reader that can run on raw source config degrade when channel credentials are unresolved, ideally by reading account capability/reaction config without resolving the token or by using a shared unresolved-SecretRef-tolerant helper with focused regression coverage.

## Reproduction Assessment

Yes. A config with `channels.telegram.botToken` as a file/exec SecretRef still follows `collectRuntimeChannelCapabilities` to Telegram `messageToolCapabilities`, then to `resolveTelegramInlineButtonsScope`, `resolveTelegramAccount`, and strict token resolution before the new `describeMessageTool` catch is involved.

## Solution Assessment

No. The PR's direction is right for Discord and Telegram action discovery, but it is not the narrow complete fix for #75433 until the Telegram prompt-capability and reaction-guidance paths also stop treating unresolved source-config SecretRefs as fatal discovery errors.
