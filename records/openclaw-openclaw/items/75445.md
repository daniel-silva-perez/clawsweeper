## Summary

Keep open. The PR fixes part of the SecretRef discovery failure, but it does not cover the Telegram prompt-capability path that embedded prompt prep runs before message-action discovery, so the linked embedded Telegram crash can still happen with a raw non-env SecretRef.

## What This Changes

The PR adds Discord and Telegram channel-action try/catch handling for unresolved SecretRef errors, regression tests for direct `describeMessageTool` discovery, and a changelog entry for the linked channel SecretRef crash.

## Best Possible Solution

Keep the action-discovery tolerance, but also harden the Telegram prompt-capability and reaction-guidance discovery boundary so prompt prep treats unresolved discovery credentials as unavailable without weakening strict send/probe SecretRef behavior.

## Reproduction Assessment

Yes. A source-level reproduction is to pass a config with `channels.telegram.botToken` or a scoped account token as an exec/file SecretRef into `collectRuntimeChannelCapabilities({ channel: "telegram" })`; current main reaches `resolveTelegramAccount` through Telegram prompt hooks and can throw before model output.

## Solution Assessment

No. The PR chooses a reasonable layer for `describeMessageTool`, but it is not sufficient for the linked embedded crash because earlier Telegram prompt helpers still resolve raw credentials; the safer fix covers those helpers or the shared prompt-capability wrapper as well.

Specifically, make Telegram `agentPrompt.messageToolCapabilities` and `reactionGuidance` (or a shared helper they use) return empty/undefined discovery when account resolution fails only because of an unresolved SecretRef. Do not weaken strict token resolution for runtime send/probe/status paths and do not add core channel IDs or global SecretRef suppression. Add regression coverage that `collectRuntimeChannelCapabilities` / `resolveChannelPromptCapabilities` and `resolveChannelReactionGuidance` for Telegram do not throw when `channels.telegram.botToken` or `channels.telegram.accounts.ops.botToken` is an exec/file SecretRef.
