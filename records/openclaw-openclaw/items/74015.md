## Summary

Keep this issue open. Current main still exposes `llm_input` and `llm_output` as observation-only Plugin SDK hooks, while the existing prompt hooks can only add context or system-prompt text. The requested writable prompt/history/output mutation remains a real public SDK gap and needs maintainer API/security design.

## Best Possible Solution

Keep this issue open for Plugin SDK design. The best path is an additive, versioned mutating pre-provider and post-output contract, or a deliberately designed result extension to the existing hooks, with explicit permission gates, deterministic ordering/merge behavior, and coverage across the embedded runner, Codex app-server/harness helpers, CLI harness, docs, and SDK contract tests.
