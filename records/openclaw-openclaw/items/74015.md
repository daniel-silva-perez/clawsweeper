## Summary

Keep this issue open. Current main still treats `llm_input` and `llm_output` as observation-only, fire-and-forget hooks whose return values are ignored, while the existing prompt hooks can only prepend/append context or replace system prompt text. The requested writable prompt/history/output mutation is a real public Plugin SDK gap and needs maintainer API/security design rather than conservative cleanup closure.

## Best Possible Solution

Keep the issue open for Plugin SDK design. The best path is an additive, versioned mutating pre-provider and post-output contract, or a clearly designed result extension to the existing hooks, with explicit permission gates, deterministic ordering/merge behavior, and coverage across the embedded runner, Codex app-server, CLI harness, docs, and SDK contract tests.
