## Summary

Keep open. The report is fresh and credible, current main still relies on model-facing identifier preservation plus bounded suffixes rather than deterministic exact path-literal carry-forward, and PR #75093 is already open with closing syntax as the direct fix candidate.

## What This Changes

The issue asks for context compaction to preserve slash-separated file path literals exactly so later tool calls do not replace path separators with spaces.

## Best Possible Solution

Review and land PR #75093, or an equivalent narrow compaction safeguard change, so path-like literals needed for future tool calls are collected from model-facing compaction input and appended deterministically with regression coverage proving slash-separated paths survive unchanged.

## Reproduction Assessment

No, there is not a high-confidence end-to-end reproduction path from the checkout alone. The raw tool logs are credible and source review shows a plausible current-main gap, but reproducing the corruption requires the original long transcript or a deterministic MiniMax compaction replay.

## Solution Assessment

Yes, the linked direction is the best maintainable fix: add deterministic path-literal preservation in the compaction/safeguard pipeline rather than relying on prompt wording alone. Because PR #75093 already targets that path with closing syntax, this issue should remain open until that PR is reviewed and landed or rejected.
