## Summary

Keep open for maintainer review. The PR points at a real Android SMS/MMS performance pain, but current main has since split sensitive Android features by flavor: Play stays Play-safe while thirdParty carries SMS and call-log support. The branch conflicts with that boundary and its MMS batching drops candidates beyond the hydration cap before filtering and pagination.

## What This Changes

The PR enables SMS and call-log capability in the Android Play flavor, removes Play manifest permission stripping, updates SMS UI copy to SMS/MMS, and refactors MMS by-phone hydration into capped batch queries with a timeout.

## Best Possible Solution

Keep the Play flavor restricted, and port a corrected MMS hydration performance fix into the current thirdParty SmsManager with tests that prove large MMS result sets preserve filtering, ordering, and offset pagination beyond any batch size.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partly: the PR-introduced bug is reproducible with more than 50 hydratable MMS rows plus a later matching row or offset, and the original timeout risk is code-backed by current per-row MMS subqueries; live Android reproduction was not run here.

## Solution Assessment

Is this the best way to solve the issue? No. The batching direction is reasonable, but this PR should not enable restricted permissions in Play and must avoid discarding MMS candidates before filters and pagination.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep restricted permissions out of Play:** `apps/android/app/build.gradle.kts:81-82`
  - body: This flips the Google Play flavor to advertise SMS and call-log capabilities. With the manifest removals also deleted, Play builds can carry `READ_SMS`, `SEND_SMS`, and `READ_CALL_LOG`, while current policy keeps those in the `thirdParty` flavor. The self-hosted fix should target the unrestricted flavor or get an explicit Play policy decision.
  - confidence: 0.92
- **[P1] Preserve MMS candidates beyond the hydration cap:** `apps/android/app/src/main/java/ai/openclaw/app/node/SmsManager.kt:1061`
  - body: Only the first 50 MMS candidates are hydrated and every later MMS row is discarded before keyword/type/read filters and pagination run. A query with more than 50 MMS rows, a selective filter, or `offset > 50` can return empty or incomplete pages even when matching messages exist.
  - confidence: 0.94
- **[P3] Drop the unused tools namespace:** `apps/android/app/src/play/AndroidManifest.xml:3`
  - body: After removing all `tools:node` attributes, the `xmlns:tools` declaration is no longer referenced and can trigger Android lint's unused namespace warning.
  - confidence: 0.82
