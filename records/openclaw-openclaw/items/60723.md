## Summary

Keep open. Current main does not implement this PR’s intended change: the Play flavor intentionally disables SMS and Call Log, while the third-party flavor keeps those capabilities. The MMS by-phone path in the current third-party SmsManager still hydrates MMS rows with per-row address/body/meta queries and has no batch hydration timeout, so the performance work is not shipped. The PR also broadens Play SMS/Call Log access against current documented Play-policy split, and the existing bot review flags a correctness risk in the proposed hydration cap, so this should be reworked rather than closed as implemented.

## Best Possible Solution

Keep this PR open only as a candidate, but ask for a rework against current main: preserve the current Play-safe permission split unless a maintainer explicitly approves a Play-policy change, port the MMS hydration performance fix into the thirdParty SmsManager, and add regression coverage for large MMS histories, filters, pagination, and the cap behavior called out in review.
