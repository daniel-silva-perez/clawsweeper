## Summary

Keep open. Current main still compares canonical two-space JSON output against the raw previous config byte size, so a PowerShell/BOM or otherwise verbose `openclaw.json` can still trigger the blocking `size-drop` guard during `models auth login`; related #70941 was closed in favor of this canonical tracker, and #74163 still lists it as unresolved Windows work.

## What This Changes

The issue asks for `models auth login` to persist OAuth/config changes when `openclaw.json` was created with PowerShell-style BOM or verbose formatting instead of being rejected by the size-drop safety guard.

## Best Possible Solution

Fix the shared config writer so formatting-only shrink is judged against a canonicalized previous config or semantic diff, while preserving fail-closed protection for real clobbers such as `gateway.mode` removal; add focused writer and auth-login regression coverage for the BOM/verbose-format case.

## Reproduction Assessment

Yes. A valid config whose raw BOM/verbose formatting is more than twice the canonical two-space JSON size still reaches `size-drop` through `models auth login` -> `updateConfig` -> `replaceConfigFile` on current main.

## Solution Assessment

Yes. A shared writer fix is the narrowest maintainable solution; an auth-only `allowDestructiveWrite` bypass would leave the false positive in other config mutations and weaken clobber protection unevenly.

## Repair Work Prompt

Fix the config writer regression tracked by https://github.com/openclaw/openclaw/issues/71865. Current behavior compares canonical two-space JSON output against the raw previous `openclaw.json` byte size, so a PowerShell-created UTF-8 BOM / verbose-indented config can shrink by more than 50% on a formatting-only rewrite and trip `CONFIG_WRITE_REJECTED` during `openclaw models auth login --provider openai-codex`. Implement the fix in the shared writer, likely `src/config/io.ts`, by judging formatting-only shrink against a canonicalized previous config or semantic diff while preserving fail-closed rejection for real clobbers such as actual required-shape loss. Do not add an auth-only `allowDestructiveWrite` bypass and do not weaken `gateway-mode-removed` protection. Add focused regression coverage in `src/config/io.write-config.test.ts` for a BOM/verbose formatted config that persists a small semantic auth/profile change without rejection, and add or adjust `src/commands/models/auth.test.ts` / `src/commands/models/shared.test.ts` only as needed to prove `models auth login` still uses the safe shared path. Add a single-line user-facing `CHANGELOG.md` fix entry with `Thanks @balric-seo`. Related refs: #71865 and closed duplicate #70941. Keep changes scoped to config writer semantics, tests, and changelog.
