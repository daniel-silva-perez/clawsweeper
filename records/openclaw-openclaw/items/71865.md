## Summary

Keep open. Current main still compares the canonical two-space rewrite against the raw previous config byte size, so a BOM/verbose PowerShell-shaped config can still trip the blocking size-drop guard; `models auth login` still reaches that shared writer through `updateConfig`. The latest tagged release contains the same guard, and the related #70941 context makes this the canonical tracker.

## What This Changes

Fix `models auth login` config persistence so a PowerShell/BOM-formatted `openclaw.json` is not rejected when the only large shrink is formatting normalization.

## Best Possible Solution

Fix the shared config writer to compare against a canonicalized previous config or semantic diff for formatting-only rewrites while preserving fail-closed protection for real clobbers, then cover the BOM/verbose-format case through both the writer and auth-login persistence path.

## Reproduction Assessment

Yes. A valid config whose raw BOM/verbose formatting is more than twice the canonical two-space JSON size will still reach `size-drop` through `models auth login` -> `updateConfig` -> `replaceConfigFile` on current main.

## Solution Assessment

Yes. A shared writer fix is the narrowest maintainable direction; an auth-only `allowDestructiveWrite` bypass would leave the false positive in other config mutations and weaken clobber protection unevenly.

## Repair Work Prompt

Fix the config writer regression tracked by https://github.com/openclaw/openclaw/issues/71865. Current behavior compares canonical two-space JSON output against the raw previous `openclaw.json` byte size, so a PowerShell-created UTF-8 BOM / verbose-indented config can shrink by more than 50% on a formatting-only rewrite and trip `CONFIG_WRITE_REJECTED` during `openclaw models auth login --provider openai-codex`. Implement the fix in the shared writer, likely `src/config/io.ts`, by judging formatting-only shrink against a canonicalized previous config or semantic diff while preserving fail-closed rejection for real clobbers such as actual required-shape loss. Do not add an auth-only `allowDestructiveWrite` bypass and do not weaken `gateway-mode-removed` protection. Add focused regression coverage in `src/config/io.write-config.test.ts` for a BOM/verbose formatted config that persists a small semantic auth/profile change without rejection, and add or adjust `src/commands/models/auth.test.ts` / `src/commands/models/shared.test.ts` only as needed to prove `models auth login` still uses the safe shared path. Add a single-line user-facing `CHANGELOG.md` fix entry with `Thanks @balric-seo`. Related refs: #71865 and closed duplicate #70941. Keep changes scoped to config writer semantics, tests, and changelog.
