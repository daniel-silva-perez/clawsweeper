## Summary

Keep open. Current main still lacks the proposed archived-history selector and read path, and the PR is a focused implementation candidate, but it needs maintainer follow-up before merge because the patch introduces ambiguous transcript sequence metadata and exposes reset-archived content through an agent tool surface that needs explicit privacy/semantics sign-off.

## What This Changes

The PR adds an optional `includeArchived` parameter to `chat.history` and `sessions_history`, adds reset-archive discovery and history aggregation helpers with tests, and regenerates Swift gateway protocol models.

## Best Possible Solution

Keep the PR open for maintainer review. If the direction is accepted, land the additive protocol/tool feature with continuous transcript metadata, a changelog entry, public docs for the opt-in archive behavior, targeted gateway and agent tests, regenerated clients, and coordination with the broader #45003 archive-recovery work.

## Reproduction Assessment

Yes. Current main can be verified statically: the schema rejects `includeArchived`, `chat.history` only calls `readSessionMessages`, and `sessions_history` cannot pass the flag. For the PR-specific sequence issue, two reset archives reproduce duplicate `__openclaw.seq` values because the added helper resets `messageSeq` for each archive segment.

## Solution Assessment

Unclear. A read-only opt-in is a plausible direction for archived-session recovery, but this PR is not yet the safest merge form until sequence metadata is continuous, release/docs coverage is added, and maintainers approve reset-archive visibility semantics for agent tools.
