## Summary

Keep this PR open for maintainer review. Current main still has no supported configurable exec approval timeout, but this branch is not landable as-is because it leaves timeout wait budgets/default documentation inconsistent and introduces an unbounded host-exec approval window; the still-open #51287 discussion should drive the final public config surface.

## What This Changes

The PR adds a proposed `tools.exec.approvalTimeoutMs` config key to types/schema/help/labels and threads it through exec approval registration, gateway/node host execution, CLI defaults, and exec tool defaults.

## Best Possible Solution

Choose one stable, bounded timeout surface, document it against the current 30-minute default, thread the resolved value through agent, gateway, node, approval record expiration, and transport wait budgets, then cover propagation and expiry behavior with focused tests. If maintainers prefer the #51287 design over this branch’s key name, explicitly supersede this PR under that issue.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep approval waits in sync with the configured timeout:** `src/agents/bash-tools.exec-approval-request.ts:51`
  - body: This introduces `approvalTimeoutMs` for the registration payload, but `waitForExecApprovalDecision` still waits with `DEFAULT_APPROVAL_REQUEST_TIMEOUT_MS`. If a user configures a timeout longer than the default, the wait RPC can time out before the approval window expires, so the configured value is not honored end-to-end.
  - confidence: 0.86
- **[P2] Bound the approval timeout value:** `src/config/zod-schema.agent-runtime.ts:393`
  - body: The schema accepts any positive integer for a host-exec approval window. Very large values can leave approvals actionable far longer than intended and can also outgrow transport assumptions, so this needs a documented maximum shared with the runtime resolver before exposing it as config.
  - confidence: 0.82
- **[P3] Update the documented default timeout:** `src/config/schema.help.ts:447-448`
  - body: The added help text says the default is `120000ms = 2 minutes`, but current main defines `DEFAULT_EXEC_APPROVAL_TIMEOUT_MS = 1_800_000` and the docs say pending exec approvals expire after 30 minutes by default. This would publish misleading config help.
  - confidence: 0.9
