## Summary

Current main still does not implement a user-facing configurable exec approval timeout. It has a fixed 30-minute default and accepts per-request timeoutMs internally, but no tools.exec.approvalTimeoutMs or approvals.timeoutMs config surface is exposed or wired through; #51287 remains open, and the recent maintainer comment explicitly kept this PR open for follow-up.

## Best Possible Solution

Keep this PR open for maintainer review, or explicitly supersede it under #51287 if maintainers choose a different public key such as approvals.timeoutMs. A landable fix should choose one supported config surface, use bounded validation, align docs/help with the current 30-minute default, thread the resolved timeout through agent/gateway/node approval waits and transport timeouts, and add focused propagation and expiry tests.
