## Summary

Keep open. Current main does not implement a configurable exec approval timeout, and this PR is not landable as-is: the head branch improves the original missing plumbing but still leaves approval wait budgets fixed, accepts an unbounded host-exec approval window, and documents the obsolete two-minute default. Related #51287 remains open and #51292 was closed unmerged, so this needs maintainer API/security review or explicit supersession rather than cleanup closure.

## What This Changes

The PR adds a proposed `tools.exec.approvalTimeoutMs` config key to config types/schema/help/labels and threads it through exec approval registration, gateway/node host execution, CLI defaults, and exec tool defaults.

## Best Possible Solution

Expose one stable, bounded approval-timeout surface only if maintainers want this product contract, keep the current 30-minute default in help/docs, propagate the same resolved value through registration, expiry, gateway/node wait budgets, and CLI transport floors, then cover propagation and expiry with focused tests. If maintainers prefer the #51287 direction, explicitly supersede this branch under that issue.

Full review comments:

- **[P2] Keep approval waits in sync with the configured timeout:** `src/agents/bash-tools.exec-approval-request.ts:51`
  - body: This introduces `approvalTimeoutMs` for the registration payload, but `waitForExecApprovalDecision` still waits with `DEFAULT_APPROVAL_REQUEST_TIMEOUT_MS`. If a user configures a timeout longer than the default, the wait RPC can time out before the approval window expires, so the configured value is not honored end-to-end.
  - confidence: 0.86
- **[P2] Bound the approval timeout value:** `src/config/zod-schema.agent-runtime.ts:393`
  - body: The schema accepts any positive integer for a host-exec approval window. Very large values can leave approvals actionable far longer than intended and can outgrow transport assumptions, so this needs a documented maximum shared with the runtime resolver before exposing it as config.
  - confidence: 0.82
- **[P3] Update the documented default timeout:** `src/config/schema.help.ts:447-448`
  - body: The added help text says the default is `120000ms = 2 minutes`, but current main defines `DEFAULT_EXEC_APPROVAL_TIMEOUT_MS = 1_800_000` and the docs say pending exec approvals expire after 30 minutes by default. This would publish misleading config help.
  - confidence: 0.9
