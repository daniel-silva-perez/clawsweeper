## Summary

Keep this PR open. Current main still leaves tool-loop detection disabled by default, the default-off behavior is locked in by tests/docs/schema metadata, and the read-offset patch is not present as a read-specific fix. The PR is still potentially useful, but it needs maintainer follow-up because it mixes a safety default change with read-tool error handling and overlaps with open loop-detection PRs such as #70877, #56403, and #62775.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up, but ask for a rebase and likely split. The loop-detection-default change should preserve current main's run-id scoping, update runtime/tests/docs/schema together, keep explicit opt-out behavior, and be reviewed alongside #70877, #56403, and #62775. The read-offset portion should either be dropped as covered by the generic diagnostic adapter for #52680 or reworked as a focused read-tool enhancement that preserves partial adaptive output and has dedicated tests.
