## Summary

Keep open. ## What This Changes

This PR flips tool-loop detection to enabled by default, updates related tests/docs/config metadata and generated baselines, and adds read-tool handling for out-of-range offsets.

## Best Possible Solution

Land two focused, rebased changes if maintainers want both behaviors: one canonical loop-detection default decision that preserves run-id scoping and aligns tests, schema help, generated checksums, and current docs; and one read-offset recovery fix inside adaptive paging with regression coverage for preserving already-read partial output.
