## Summary

Keep this PR open. Current main still leaves tool-loop detection disabled by default in runtime, tests, schema help, and docs, and the PR's read-offset behavior is not present as a read-specific fix. The branch remains potentially useful but needs maintainer follow-up because it mixes the loop-detection default change with read-tool error handling and overlaps with open loop-detection PRs.

## What This Changes

The PR flips `tools.loopDetection.enabled` to true by default, updates related tests/docs/schema metadata, regenerates config baselines, and wraps the OpenClaw read tool to return an offset-out-of-range diagnostic instead of throwing.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up, but ask for a rebase and likely split. The loop-detection-default change should be reviewed with #70877, #56403, and #62775, preserve current run-id scoping, and update runtime, tests, docs, schema help, and generated artifacts together. The read-offset portion should either be dropped if #52680 is considered covered by the generic diagnostic adapter, or reworked as a focused read-tool enhancement inside the adaptive paging path that preserves partial output and has dedicated tests.
