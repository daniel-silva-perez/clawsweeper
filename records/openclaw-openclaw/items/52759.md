## Summary

Keep open. The underlying gaps are still visible on current main, but this branch is not merge-ready: it is behind current run-scoped loop-history behavior, mixes the loop-default decision with read-offset recovery, and introduces concrete patch defects that need maintainer reconciliation.

## What This Changes

The PR enables tool-call loop detection by default, updates related tests/docs/config metadata and generated baselines, and wraps read-tool offset failures with a diagnostic result.

## Best Possible Solution

Land two focused, rebased changes if maintainers want both behaviors: one canonical loop-detection default decision that preserves run-id scoping and aligns tests, schema help, generated checksums, and docs; and one read-offset recovery fix inside adaptive paging with regression coverage for preserving already-read partial output.

## Reproduction Assessment

Yes. The loop-default behavior is reproducible with repeated detectToolCallLoop calls without config, and the read-offset behavior follows from #52680's offset-beyond-EOF path plus current main's direct adaptive read call; I did not run tests because this review is read-only.

## Solution Assessment

No. The direction may be valid, but this PR is not the best form because it mixes two fixes, carries local-only generated artifacts, and handles EOF recovery outside the adaptive pager where partial output can be lost.
