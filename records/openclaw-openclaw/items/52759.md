## Summary

Keep open. Current main still leaves tool-loop detection disabled by default, tests and docs/schema still assert that default, and the read wrapper still has no graceful out-of-range offset handling. The newer open related PR overlaps the loop-detection-default part only, so this PR is not fully implemented or clearly superseded.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up, but require a rebase and likely split. The loop-detection-default change should preserve current main's run-id scoped history, update docs/schema/tests together, and be reviewed alongside the overlapping open loop-detection PR. The read-offset behavior should be handled as a focused fix with tests that preserve partial adaptive read output when EOF is reached mid-pagination.
