## Summary

Keep open. Current main still leaves loop detection opt-in, and the PR is useful but not ready: it overlaps several open loop-detection default PRs, mixes in a read-offset behavior change, and still has concrete correctness/maintenance issues that should block merge until rebased and split or reconciled.

## What This Changes

This PR enables tool-call loop detection by default, updates related tests/docs/config metadata, and adds read-tool handling for out-of-range offsets.

## Best Possible Solution

Land two focused, rebased changes if maintainers want both behaviors: one canonical loop-detection default decision that preserves run-id scoping and updates tests, schema help, generated checksums, and current docs; and one read-offset recovery fix inside adaptive paging with regression coverage for preserving already-read partial output.
