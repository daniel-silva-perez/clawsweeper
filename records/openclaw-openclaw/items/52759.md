## Summary

Keep open. Current main still ships the loop detector disabled by default, so the core loop-detection change is not obsolete, but this PR should not merge as-is: it mixes the loop-default decision with read-offset behavior, carries local-only generated docs artifacts, leaves contradictory docs guidance, and the read-offset wrapper can discard already-read adaptive pages. The linked read-offset issue has also been closed as implemented through the generic tool adapter, so that part needs maintainer judgment rather than automatic landing.

## What This Changes

The PR enables tool-call loop detection by default, updates related tests/docs/config schema metadata and generated baselines, and adds read-tool handling for out-of-range offsets.

## Best Possible Solution

Land a focused, rebased loop-detection default decision that keeps run-id scoping intact and aligns tests, schema help, generated checksums, and docs. Treat read-offset clamping or partial-output recovery as a separate focused enhancement only if maintainers want behavior beyond the current generic diagnostic result.
