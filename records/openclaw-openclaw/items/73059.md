## Summary

Keep open. Current main still has the direct-execution guard described in the report, the gateway auto-build path can still surface the missing `dist/control-ui/index.html` symptom after a no-op script exit, and related PR #52291 was adjacent Windows `pnpm.cmd` work that closed unmerged rather than a fix for direct junction-launched `node scripts/ui.js`.

## What This Changes

The issue asks OpenClaw to make `scripts/ui.js build` execute correctly when launched through a Windows global `pnpm link` junction so first-start Control UI auto-build recreates `dist/control-ui/index.html`.

## Best Possible Solution

Normalize the `scripts/ui.js` direct-execution guard through realpath on both `process.argv[1]` and `import.meta.url`, with a resolved-path fallback for missing paths, then add a focused regression test for linked/junction-style launch behavior without changing Control UI app code or package metadata.

## Reproduction Assessment

Yes. The issue provides exact Windows `pnpm link --global` junction steps, and current main still contains the raw path comparison that can skip `main()` when launch and module paths resolve differently.

## Solution Assessment

Yes. A narrow guard fix in `scripts/ui.js` plus focused coverage is the maintainable solution; a gateway-only workaround would leave the script's direct CLI behavior broken.
