## Summary

Keep open. Current main still has the raw `scripts/ui.js` direct-execution guard described in the report, the gateway auto-build path still treats a no-op script exit as a missing Control UI bundle, and the focused tests still do not cover a junction/symlink-launched entry path.

## What This Changes

The issue asks for Windows linked source installs launched through the global npm junction to run the same Control UI build path as repo-local `scripts/ui.js build`.

## Best Possible Solution

Normalize the direct-execution guard in `scripts/ui.js` through realpath on both the launch path and module path, with a resolved-path fallback, then lock the behavior with a focused regression test while leaving Control UI app code and package metadata unchanged.

## Reproduction Assessment

Yes. The issue provides concrete Windows `pnpm link --global` junction steps, and current main still has the exact raw path comparison that can make `node ...\scripts\ui.js build` skip `main()` when the module path resolves differently.

## Solution Assessment

Yes. A narrow guard fix plus `test/scripts/ui.test.ts` coverage is the maintainable path; a gateway-level workaround would leave the script's direct CLI behavior broken.

## Repair Work Prompt

Fix the Windows linked-source Control UI auto-build no-op reported in https://github.com/openclaw/openclaw/issues/73059. In `scripts/ui.js`, make the direct-execution guard recognize that `process.argv[1]` and `import.meta.url` may refer to the same file through a Windows global npm junction or symlink: normalize both paths through realpath, falling back to `path.resolve` if realpath fails, before deciding whether to call `main()`. Add focused coverage in `test/scripts/ui.test.ts` for a launch path that realpaths to the module path so the script would execute `main()` under a junction-launched `node .../scripts/ui.js build`. Keep the change scoped to the wrapper/test; do not change Control UI app code, package metadata, gateway auto-build semantics, or the Windows pnpm shell quoting behavior from adjacent PR #52291. Because this is a user-visible Windows bug fix, add a single-line `CHANGELOG.md` fix entry with `Thanks @LouisGameDev` if the active changelog section accepts it. Validate with `pnpm test test/scripts/ui.test.ts`, targeted oxfmt for touched code files, and `pnpm check:changed` in Testbox by default before handoff.
