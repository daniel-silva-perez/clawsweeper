## What This Changes

The issue requests a fix for Windows linked source installs where launching `scripts/ui.js` through the global npm junction can skip the Control UI build because the script does not recognize itself as directly executed.

## Best Possible Solution

Add a small, script-owned direct-execution helper that compares the launch path and module path after realpath normalization, keeps the module importable for tests, and adds focused regression coverage for the junction/symlink mismatch without changing UI app code or package metadata.

## Reproduction Assessment

Yes. The issue gives a concrete Windows linked-source path using `node %APPDATA%\npm\node_modules\openclaw\scripts\ui.js build`, and current source still has the raw path comparison that explains the no-op.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize both sides through `fs.realpathSync.native` or an equivalent helper before deciding whether to call `main()`, with a fallback to resolved paths if realpath fails.

## Repair Work Prompt

Fix the Windows linked-source Control UI build wrapper bug reported at https://github.com/openclaw/openclaw/issues/73059. Current `scripts/ui.js` only calls `main()` when `path.resolve(process.argv[1]) === fileURLToPath(import.meta.url)`, which can be false when the script is launched through the global npm junction path on Windows. Add a small helper that compares the entry path and module path after realpath normalization, preferably using `fs.realpathSync.native` when available with a safe fallback to `path.resolve`, and keep the script importable for tests. Add or extend `test/scripts/ui.test.ts` to cover a junction/symlink launch mismatch, ideally by testing a helper rather than requiring a real Windows junction. Preserve the existing Windows shell launcher quoting and unsafe-argument behavior from the PR #52291 context. Do not change UI app code, package publishing metadata, dependency versions, or broad gateway behavior. Changelog is not expected for this narrow script/test fix unless maintainer policy asks for one.
