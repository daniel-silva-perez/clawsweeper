## Summary

Keep this open. Current main has adjacent fallback observability in attempt arrays, subsystem logs, trajectory sidecars, lifecycle events, and session-store fallback notice fields, but it still does not persist the requested ordered session JSONL audit event before the fallback `model-snapshot`.

## What This Changes

The issue requests an ordered, sanitized session JSONL audit record explaining the failed primary model attempt before a successful fallback model switch.

## Best Possible Solution

Add a small transcript audit event at the fallback/reply-runner boundary that records the failed selected provider/model, chosen fallback provider/model, reason/class, status/code when available, chain position, and bounded sanitized or hashed error detail before the fallback model snapshot/retry. Preserve existing fallback routing, logs, lifecycle events, status state, and user-visible notices.
