## Summary

Keep this open. Current main records failed fallback candidates in runtime attempt arrays, subsystem logs, lifecycle events, session-store fallback notice fields, and optional user-visible notices, but it still does not append the requested ordered session JSONL custom audit event before the fallback `model-snapshot`. The prior ClawSweeper keep-open review remains accurate on current main at `2d53b49b20e19790afbba4f4cbb2b78042bb3530`.

## What This Changes

The issue requests a structured, sanitized session JSONL audit event before `model-snapshot` whenever a primary model failure triggers a successful fallback.

## Best Possible Solution

Add a small transcript audit event at the fallback/reply-runner boundary that records the failed selected provider/model, chosen fallback provider/model, reason/class, status/code when available, duration if available, and bounded sanitized or hashed error detail. It should be appended before the fallback model snapshot/retry and covered by focused ordering and redaction tests while preserving existing logs, lifecycle events, status state, and user-visible notices.
