## Summary

Keep this PR open. Current main still lacks the intended `/bash` inheritance of effective `tools.exec` defaults, and the PR is an active focused implementation candidate carrying forward #50392 and #41598. It should not be cleanup-closed, but it needs maintainer/security review before merge because the patch makes `/bash` consume session-stored exec policy fields that current directive persistence can set for non-internal authorized chat sessions.

## Best Possible Solution

Keep this PR open as the active implementation candidate, but require maintainer/security review before merge. The best fix is to make `/bash` inherit the same effective exec defaults as the normal exec tool while preserving bash/elevated gates and bang routing, and to explicitly resolve the trust model for session-level exec overrides before applying them to `/bash`.
