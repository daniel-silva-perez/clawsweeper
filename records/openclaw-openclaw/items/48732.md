## Summary

Keep this PR open. Current main still does not implement the PR's intended healthcheck-specific instruction: `skills/healthcheck/SKILL.md` has no rule telling the scheduled healthcheck audit to render alerts, reports, or logs in the user's local timezone. The generic cron runner does inject a mixed local/UTC `Current time:` line, but that does not replace the skill-level output-format instruction proposed by this PR.

## Best Possible Solution

Keep the PR open and ask for a rebase/update against current `skills/healthcheck/SKILL.md`. The best implementation is a small additive core rule for healthcheck that tells scheduled alerts, reports, logs, and memory summaries to use the user's local timezone from the cron `Current time:` line or `agents.defaults.userTimezone`, with UTC permitted only as a secondary comparison value.
