## Summary

Keep open. The PR targets a real plugin lifecycle gap linked to open session-end work, but the current patch is not merge-ready because failed-spawn cleanup would emit duplicate or mis-keyed `session_end` events and cron failures would still miss the hook.

## What This Changes

The PR adds `session_end` hook emissions for failed subagent spawn cleanup and cron isolated-agent completion, updates hook comments, and adds an Unreleased changelog entry.

## Best Possible Solution

Implement `session_end` as an exactly-once lifecycle signal from the lifecycle owners. Failed subagent cleanup should rely on or adjust the existing `sessions.delete` emission using the stored child session id, while cron isolated runs should emit once from a terminal/finally-style path after the run session exists and is persisted, with focused regression tests for success, abort, and error paths.

## Reproduction Assessment

Yes. A high-confidence static reproduction is enough here: the failed-spawn path still calls `sessions.delete`, which already emits `session_end`, and the cron function has visible early/error returns that bypass the PR's post-finalize hook block.

## Solution Assessment

No. The feature direction is useful, but this patch is not the narrowest maintainable solution because it adds ad hoc emissions that can duplicate or mis-key failed-spawn events and does not cover every cron terminal path.
