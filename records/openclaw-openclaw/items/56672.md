## Summary

Keep open. The PR addresses a real plugin lifecycle gap, but the current patch is not merge-ready: the failed-spawn path would duplicate or mis-key `session_end`, and the cron emission is not placed on every terminal path that the PR says it covers.

## What This Changes

The PR adds `session_end` hook emissions for failed subagent spawn cleanup and cron isolated-agent completion, updates hook comments, and adds a changelog entry.

## Best Possible Solution

Implement `session_end` as an exactly-once lifecycle signal from the owners of each session lifecycle. For failed subagent cleanup, rely on or adjust the existing `sessions.delete` emission with the stored session id; for cron isolated runs, emit from a finally-style terminal path after the run session exists, using `runSessionId` and `runSessionKey`, with focused regression tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction is enough here: the failed-spawn path still calls `sessions.delete` after the proposed emission, and `sessions.delete` already emits `session_end`; cron error and early-return paths are visible in `runCronIsolatedAgentTurn` and bypass the proposed post-finalize hook block.

## Solution Assessment

No. The direction is useful, but adding ad hoc fire-and-forget calls at individual call sites is not the best fix because it creates duplicate/mis-keyed failed-spawn events and leaves cron failure paths uncovered. A shared exactly-once emission helper or finally-style lifecycle wrapper is safer.
