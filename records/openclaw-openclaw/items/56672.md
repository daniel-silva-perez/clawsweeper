## Summary

Keep open. The requested lifecycle-hook direction is useful and matches related reports, but this PR is not ready to merge: it can emit a failed-spawn `session_end` with the wrong ID, can duplicate the existing delete-owned `session_end`, and only covers part of the cron terminal lifecycle.

## What This Changes

The PR adds `session_end` hook emissions for failed subagent spawn cleanup and cron isolated-agent completion, updates hook type comments, and adds an Unreleased changelog entry.

## Best Possible Solution

Implement `session_end` as an exactly-once signal from lifecycle-owning paths. Failed subagent cleanup should rely on or adjust the existing `sessions.delete` emission using the stored child session id; cron isolated runs should emit once from a terminal/finally-style path after a run session exists, with tests for success, abort, skipped/preflight failure, and executor error paths.

## Reproduction Assessment

Yes. A high-confidence static reproduction is available from the PR diff plus current main: the failed-spawn path still deletes the provisional session, and the cron runner has visible terminal returns that bypass the proposed hook block.

## Solution Assessment

No. The feature direction is sound, but this patch is not the best implementation because it adds ad hoc emissions instead of centralizing exactly-once lifecycle ownership and it misses several cron terminal outcomes.
