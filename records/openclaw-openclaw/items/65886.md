## Summary

Keep this PR open. Current main still does not implement configurable TUI status-line command rendering, and the existing plugin surface does not expose a terminal TUI status/render insertion point. The linked petsonality workaround is useful demand evidence, but it remains an external patch rather than a supported OpenClaw API.

## Best Possible Solution

Keep the PR open for maintainer/product review. If accepted, land it as a core TUI/config feature with updated types, strict Zod schema, generated schema/docs metadata, lifecycle tests, and explicit trusted-local-command documentation. If maintainers prefer externalization, first add a public TUI status/render plugin seam, then move command rendering into a plugin.
