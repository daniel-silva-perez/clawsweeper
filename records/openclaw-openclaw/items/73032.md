## Summary

Keep open. Current main still only wires `/voiceclaw/realtime` to Gemini, and this draft PR is useful but not merge-ready because its xAI adapter misses documented Realtime protocol events and ordering constraints.

## What This Changes

The PR adds an xAI `/voiceclaw/realtime` provider by extending provider selection/types, adding an xAI realtime adapter and mocked tests, and updating gateway and xAI provider docs.

## Best Possible Solution

Land a revised implementation after maintainer design agreement that keeps the existing Gemini path stable, maps xAI's voice transcript and setup events correctly, preserves the server-side key boundary, handles parallel tool-call resume semantics, and adds focused mocked coverage plus opt-in live xAI validation.

## Reproduction Assessment

Yes for the review findings: a focused mocked adapter test can inject `response.output_audio_transcript.delta` and observe no assistant transcript, can send `session.created` before `session.updated` and observe early setup completion, and can simulate multiple tool calls to observe premature `response.create`. No live xAI session was run.

## Solution Assessment

No, not yet. Provider-aware `/voiceclaw/realtime` dispatch is a plausible direction, but the current patch should not merge until the documented xAI event mapping and ordering gaps are fixed or maintainers choose a different shared adapter design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Map audio transcript events:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:397-398`
  - body: The xAI Realtime API emits assistant voice captions as `response.output_audio_transcript.delta` and `.done`, but this switch only handles text-mode `response.text.delta` / `response.output_text.delta`. In a normal voice session the adapter will forward audio while never emitting assistant transcript events, so clients lose captions and `getTranscript()` misses assistant turns. Route the audio transcript events through the assistant transcript path.
  - confidence: 0.91
- **[P2] Wait for session.updated before setup completion:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:277-282`
  - body: xAI sends `session.created` automatically when the socket opens and `session.updated` after applying this adapter's `session.update`. Resolving setup on `session.created` can make the gateway emit `session.ready` and flush queued audio/control messages before the configured voice, instructions, audio settings, and tools are acknowledged. Treat `session.created` as informational and complete setup on `session.updated`.
  - confidence: 0.87
- **[P2] Defer response.create until all tool results are sent:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:165-169`
  - body: xAI documents that parallel tool calls may arrive together and that clients must send every `function_call_output` before a single `response.create`. This method sends `response.create` after each individual tool result, so the first completed tool can resume generation without the remaining tool outputs. Track pending calls and resume only once all outputs for the batch have been submitted.
  - confidence: 0.82
