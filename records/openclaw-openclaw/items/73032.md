## Summary

Keep open. Current main still only wires Gemini for `/voiceclaw/realtime`, and this draft PR is the active implementation path for #73019, but the patch is not landable yet because the new xAI adapter misses documented voice-transcript events and can mark setup ready before xAI acknowledges `session.update`.

## What This Changes

The draft PR adds an xAI `/voiceclaw/realtime` provider by extending provider selection, types, docs, and adding a new `VoiceClawXaiRealtimeAdapter` with mocked adapter tests.

## Best Possible Solution

Land a revised implementation after maintainer design agreement that keeps the existing Gemini path stable, maps xAI's documented voice transcript and setup events, preserves the server-side key boundary, and adds focused mocked coverage plus opt-in live xAI validation for the Realtime wire contract.

## Reproduction Assessment

Yes for the review findings: compare the PR adapter's event switch and setup completion logic against the official xAI Realtime event flow. No live xAI session was run in this read-only review.

## Solution Assessment

No, not yet. Provider-aware `/voiceclaw/realtime` dispatch is a plausible direction, but the current draft needs protocol fixes before it is the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Handle xAI audio transcript events:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:397-398`
  - body: The xAI Realtime API emits assistant voice transcripts as `response.output_audio_transcript.delta` / `.done`, but this adapter only handles `response.text.delta` and `response.output_text.delta`. In a voice session it will forward audio while never emitting assistant `transcript.delta`/`done`, so clients lose captions and `getTranscript()` misses assistant turns. Map the audio transcript events through the same assistant transcript path.
  - confidence: 0.9
- **[P2] Wait for session.updated before marking setup ready:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:277-282`
  - body: xAI sends `session.created` automatically when the socket opens, and `session.updated` after applying this adapter's `session.update`. Resolving setup on either event can make the gateway send `session.ready` and flush queued audio before the selected voice, instructions, audio settings, and tool catalog are acknowledged. Treat `session.created` as informational and complete setup on `session.updated`.
  - confidence: 0.86
