## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Handle xAI audio transcript events:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:397-398`
  - body: The xAI Realtime API emits assistant voice transcripts as `response.output_audio_transcript.delta` / `.done`, but this adapter only maps text-output events. Voice sessions will stream audio without assistant `transcript.delta`/`done`, so clients lose captions and `getTranscript()` misses assistant turns. Map the audio transcript events through the assistant transcript path.
  - confidence: 0.9
- **[P2] Wait for session.updated before marking setup ready:** `src/gateway/voiceclaw-realtime/xai-realtime.ts:277-282`
  - body: xAI sends `session.created` automatically when the socket opens and `session.updated` after applying this adapter's `session.update`. Resolving setup on either event can make the gateway send `session.ready` and flush queued audio before the selected voice, instructions, audio settings, and tools are acknowledged. Treat `session.created` as informational and complete setup on `session.updated`.
  - confidence: 0.86
