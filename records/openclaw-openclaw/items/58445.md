## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks the requested `before_block_reply` hook, so the feature is not implemented elsewhere; however the branch is stale against the current hook/type layout, the gateway session-key part has already landed separately, and the proposed block-reply hook still has correctness gaps around coalescing and canceled direct sends.

## What This Changes

This PR adds a `before_block_reply` plugin hook for inspecting, rewriting, or canceling block reply payloads, threads block-flush trigger context through the embedded reply pipeline, and includes a gateway hook session-key rebinding change.

## Best Possible Solution

Rebase and narrow the PR to a current-main `before_block_reply` API: add the hook to `hook-types.ts` and `hooks.ts`, preserve trigger context through block chunking and coalescing, make cancellation consistently mean not delivered across pipeline and direct paths, and document/test the new plugin contract before merge.

## Reproduction Assessment

Yes. A high-confidence focused check is to enable block streaming with coalescing, register a `before_block_reply` hook that records `event.trigger`, and verify that coalesced text blocks and tool-start flushes preserve the expected trigger; the current PR diff would deliver `undefined` on the coalescer flush path.

## Solution Assessment

No. The hook direction is plausible, but this PR is not the best current solution until it is rebased onto the current hook contract layout and fixes trigger preservation plus cancellation semantics without carrying the already-landed gateway change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve trigger context through coalescing:** `src/auto-reply/reply/block-reply-pipeline.ts:181`
  - body: When coalescing is enabled, `enqueue` receives the new context but the coalescer stores only the payload, and `onFlush` forwards `undefined` to `sendPayload`. That means `before_block_reply` receives no `trigger` for coalesced text blocks, which breaks the advertised ability to distinguish `tool_start` narration from normal text chunks. Store context with coalesced payloads or flush trigger-sensitive blocks without losing it.
  - confidence: 0.9
- **[P2] Move before_block_reply into hook-types:** `src/plugins/types.ts:1808`
  - body: Current main defines `PluginHookName`, `PLUGIN_HOOK_NAMES`, hook events, and `PluginHookHandlerMap` in `src/plugins/hook-types.ts`; `src/plugins/types.ts` only imports those types. Adding the new hook contract here leaves the runner and registry typed against the old hook list after rebase, so the hook will not register cleanly on current main.
  - confidence: 0.86
- **[P2] Record direct block sends only after success:** `src/auto-reply/reply/reply-delivery.ts:127`
  - body: The direct block path records the payload in `directlySentBlockKeys` before awaiting `onBlockReply`. If `before_block_reply` cancels by throwing `BlockReplyCancelledError`, final-reply filtering can still treat that canceled block as already sent and suppress the final content. Move sent-key tracking after a successful send or undo it when cancellation is caught.
  - confidence: 0.82
