## Summary

Keep this PR open. Current main still does not implement the requested `before_block_reply` plugin hook, the trigger-bearing block reply context, or a hook runner/call site for pre-delivery block reply interception. Adjacent `message_sending` and `reply_dispatch` hooks exist, but they do not expose the block flush reason (`tool_start`, `text_end`, `message_end`, `run_end`) that this PR is trying to add. The provided review discussion also shows meaningful implementation concerns around coalescing and cancellation semantics that still need maintainer review before landing.

## Best Possible Solution

Keep the PR open but ask for a narrow rebase onto current main. The best path is to drop the already-shipped gateway session-routing changes, implement the block-reply hook or an equivalent trigger-aware core seam, preserve coalescing/default delivery behavior, document the new plugin API, and add regression tests for trigger propagation, coalescing, cancellation, and final-answer fallback.
