## Summary

Keep open. Current main covers the first part of the request with per-agent `thinkingDefault` and flat `agents.list[].params`, but the broader requested public API is still missing: there is no per-agent `agents.list[].models` map and no `thinkingStrategy` rules, escalation, or AGENTS.md-sourced policy surface. Related #47852 and PR #44126 are useful overlap, but they are still open and do not fully supersede this issue.

## What This Changes

Request to add per-agent thinking defaults, per-agent generation/model parameter overrides, and adaptive thinking strategy controls.

## Best Possible Solution

Keep this open as the canonical maintainer design item or split it into narrower issues: retain the shipped per-agent `thinkingDefault` and flat `params` behavior, decide whether a per-agent per-model map belongs in core, and separately define any versioned adaptive thinking strategy contract for rules, failure escalation, or AGENTS.md policy.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/API request. The remaining gap is statically reproducible by inspecting the strict agent schema, exported config type, runtime params merge, docs, and tests: `agents.list[].models` and `thinkingStrategy` are not current-main config surfaces.

## Solution Assessment

Unclear. The current flat `agents.list[].params` implementation is the narrowest maintainable answer for per-agent temperature-style overrides, but the requested per-model map and adaptive strategies need maintainer product/API direction before an implementation can be judged best.
