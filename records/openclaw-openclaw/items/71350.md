## Summary

Keep open. Current main has already fixed the narrow `agents.list[].thinkingDefault` gap and supports flat per-agent `agents.list[].params`, but the broader requested API is still not implemented: there is no per-agent `agents.list[].models`/`generation` map and no configurable `thinkingStrategy` surface for rules, failure escalation, or AGENTS.md-sourced policy. Related #47852 and PR #44126 overlap with parts of the request, while #64792 was closed as superseded by this issue, so this remains the best canonical design item for the remaining work.

## What This Changes

Request per-agent thinking defaults, per-agent model parameter overrides, and adaptive thinking strategies including rule matching, failure escalation, and AGENTS.md-driven policy.

## Best Possible Solution

Keep this as a maintainer design item and likely split it after direction is chosen: preserve the shipped per-agent `thinkingDefault` and flat `params` behavior, route generic generation-parameter forwarding with #47852, and define a separate versioned adaptive thinking policy for rules, tool/test failure escalation, and any AGENTS.md-sourced behavior.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/API request. The remaining gap is statically reproducible from current main because strict schema, exported types, runtime merge code, docs, and search all show no `agents.list[].models` or `thinkingStrategy` surface.

## Solution Assessment

Unclear. The current flat `agents.list[].params` implementation is the narrowest shipped answer for per-agent temperature-style overrides on the selected model, but the requested per-agent model map and adaptive strategies need maintainer API/product direction before a best implementation can be chosen.
