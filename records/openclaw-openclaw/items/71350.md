## Summary

Keep open. Current main has implemented the per-agent `thinkingDefault` schema/docs/runtime path and flat `agents.list[].params` overrides, but the requested per-agent `agents.list[].models` map and `thinkingStrategy` rules/escalation/AGENTS.md policy surface are still absent. Related #47852 and #44126 cover parts of the remaining space, but neither fully supersedes this broader config/API request.

## What This Changes

Request to add per-agent thinking defaults, per-agent generation parameter/model-map overrides, and adaptive thinking policies driven by rules, escalation, or AGENTS.md.

## Best Possible Solution

Keep this open as a product/API design item, or split it into narrower canonical issues: one for whether to add `agents.list[].models` despite the existing flat params override, and one for a versioned adaptive `thinkingStrategy` contract. Preserve the current `thinkingDefault` and `agents.list[].params` behavior unless maintainers choose a migration path.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/API request. The remaining gap is statically verifiable from the strict schema, exported types, runtime merge function, docs, and searches showing no `agents.list[].models` or `thinkingStrategy` implementation.

## Solution Assessment

Unclear. The shipped flat `agents.list[].params` is the narrowest current solution for per-agent temperature-style overrides, but the requested per-model map and adaptive strategy modes need maintainer API/product selection before a best implementation can be judged.
