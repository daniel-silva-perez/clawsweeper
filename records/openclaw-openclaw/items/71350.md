## Summary

Keep open. Current main satisfies the per-agent `thinkingDefault` part and supports flat per-agent `agents.list[].params`, but the requested per-agent per-model `agents.list[].models` map and `thinkingStrategy` rules/escalation/AGENTS.md policy surface are still absent, and the related open items do not fully supersede this broader product/API request.

## What This Changes

Request per-agent thinking defaults, per-agent model parameter overrides, and adaptive thinking strategies driven by message rules, failures, or AGENTS.md policy.

## Best Possible Solution

Keep this open as a product/API design item or split it into narrower tracked issues. Preserve the shipped `agents.list[].thinkingDefault` and flat `agents.list[].params` behavior, then decide whether OpenClaw should add a per-agent per-model params catalog and a `thinkingStrategy` contract for rules, escalation, and AGENTS.md-sourced policy.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/API request. The missing surface is statically verifiable because the strict agent schema and public `AgentConfig` type do not include `agents.list[].models` or `thinkingStrategy`.

## Solution Assessment

Unclear. The implemented flat `agents.list[].params` is a narrow maintainable partial solution, but adding all requested strategy modes in one PR would be a broader product/config contract change that should be split or explicitly approved by maintainers.
