## Summary

Keep open. Current main covers part of the request through per-agent `thinkingDefault` and flat `agents.list[].params`, but it still does not expose the requested per-agent `agents.list[].models` map or any `thinkingStrategy` rules, escalation, or AGENTS.md-sourced policy surface. Related #47852 and PR #44126 are useful context, but they do not fully supersede this broader product/API design item.

## What This Changes

Request per-agent thinking defaults, per-agent model parameter overrides, and adaptive thinking strategies for rules, failures, and AGENTS.md policy.

## Best Possible Solution

Keep this open as the canonical product/API design item, or split it into narrower maintainer-owned issues: preserve the shipped per-agent `thinkingDefault` and flat `params` behavior, decide whether per-agent per-model params belong in core, and separately define a versioned adaptive thinking strategy contract if OpenClaw should support rules, failure escalation, or AGENTS.md policy.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/API request. The remaining missing surface is statically verifiable because the strict schema, exported agent config type, runtime parameter merge, and docs omit `agents.list[].models` and `thinkingStrategy`.

## Solution Assessment

Unclear. The current flat `agents.list[].params` design is the narrowest shipped answer for per-agent temperature-style overrides, but the requested per-model map and adaptive strategy modes need maintainer scope selection before a durable implementation can be judged best.
