## Summary

Keep open. Current main implements per-agent `thinkingDefault` and flat `agents.list[].params`, but the broader requested API is still not present: there is no `agents.list[].models` per-agent model map and no configurable `thinkingStrategy` surface for rules, escalation, or AGENTS.md-driven policy. Related #47852 and PR #44126 overlap with parts of the request but do not fully supersede this design item.

## What This Changes

Request per-agent thinking defaults, per-agent per-model generation parameters, and adaptive thinking strategy config for rules, failure escalation, and AGENTS.md-driven policies.

## Best Possible Solution

Keep this as a maintainer design item or split it into narrower issues: preserve the shipped per-agent `thinkingDefault` and flat `params` behavior, decide whether a per-agent per-model map belongs in the core config contract, and separately define a versioned adaptive thinking policy for rules, failure escalation, and AGENTS.md-sourced behavior.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/API request. The remaining gap is high-confidence statically verifiable from the strict schema, exported config type, runtime merge path, docs, and tests: `agents.list[].models` and `thinkingStrategy` are not current-main config surfaces.

## Solution Assessment

Unclear. The current flat `agents.list[].params` implementation is the narrowest maintainable answer for per-agent temperature-style overrides on the selected model, but the requested per-agent model map and adaptive strategies need maintainer API/product direction before an implementation can be judged best.
