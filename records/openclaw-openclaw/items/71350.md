## Summary

Keep open. Current main implements the narrow per-agent `thinkingDefault` and flat `agents.list[].params` pieces, but the broader requested public API is still missing: no `agents.list[].models` map and no `thinkingStrategy` rules, escalation, or AGENTS.md policy surface. Related #47852 and PR #44126 overlap but remain open and do not fully supersede this broader design request.

## What This Changes

The issue requests per-agent thinking defaults, per-agent model parameter overrides, and adaptive thinking strategies driven by rules, escalation signals, or AGENTS.md policy.

## Best Possible Solution

Keep this as a maintainer design item or split it into narrower issues: keep the shipped per-agent `thinkingDefault` and flat `params` behavior, decide whether per-agent per-model maps belong in the core config contract, and separately define a versioned adaptive thinking policy for rules, failure escalation, and any AGENTS.md-sourced behavior.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/API request. The remaining gap is high-confidence statically verifiable by inspecting the strict schema, exported config type, runtime params merge, docs, and tests: `agents.list[].models` and `thinkingStrategy` are not current-main config surfaces.

## Solution Assessment

Unclear. The current flat `agents.list[].params` implementation is the narrowest maintainable answer for per-agent temperature-style overrides, but the requested per-model map and adaptive strategies need maintainer API/product direction before an implementation can be judged best.
