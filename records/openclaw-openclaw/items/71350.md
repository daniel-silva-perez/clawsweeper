## Summary

Keep open. Current main implements the per-agent `thinkingDefault` portion and supports flat per-agent `agents.list[].params`, but it still does not expose the requested per-agent `agents.list[].models` map or any `thinkingStrategy` rules, escalation, or AGENTS.md-sourced policy surface. Related #47852 and open PR #44126 cover parts of the request, but they do not fully supersede this broader config/API design item.

## What This Changes

The issue requests per-agent thinking defaults, per-agent model parameter overrides, and adaptive thinking strategy configuration for rules, failure escalation, and AGENTS.md-driven policy.

## Best Possible Solution

Keep this open as a product/API design item or split it into narrower canonical issues. Preserve the shipped per-agent `thinkingDefault` and flat `params` behavior, then decide whether OpenClaw should add per-agent per-model params and a versioned `thinkingStrategy` contract for rules, escalation, and AGENTS.md policy.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/API request. The missing surface is statically verifiable because the strict schema, exported `AgentConfig` type, runtime parameter merge, and docs do not include `agents.list[].models` or `thinkingStrategy`.

## Solution Assessment

Unclear. The current flat `agents.list[].params` design is the narrowest shipped answer for per-agent temperature-style overrides, but the requested per-model map and adaptive strategy modes need maintainer scope selection before a durable fix can be judged best.
