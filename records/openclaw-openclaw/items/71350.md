## Summary

Keep open. Current main partially satisfies the request with per-agent `thinkingDefault` and flat `agents.list[].params`, but the broader public API remains unresolved: there is still no per-agent `models` map, no `thinkingStrategy` schema, and no rules/escalation/AGENTS.md policy path. Related #47852 and PR #44126 are useful adjacent work, but they do not fully supersede this canonical design item.

## What This Changes

Request per-agent thinking defaults, per-agent model parameter overrides, and adaptive thinking strategies for rules, failures, and AGENTS.md policy.

## Best Possible Solution

Keep this as the canonical product/API design discussion or split it into maintainer-owned narrower issues: preserve the shipped per-agent `thinkingDefault` and flat `params` behavior, decide whether per-agent per-model params belong in core, and separately define a versioned adaptive thinking strategy contract if rules, failure escalation, or AGENTS.md policy should be supported.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/API request. The remaining gap is statically reproducible by checking the strict schema, exported agent config type, runtime merge code, docs, and searches for the requested strategy fields.

## Solution Assessment

Unclear. The current flat `agents.list[].params` design is the narrowest shipped answer for per-agent temperature-style overrides, but the requested per-model map and adaptive strategy modes need maintainer scope selection before a durable implementation can be judged best.
