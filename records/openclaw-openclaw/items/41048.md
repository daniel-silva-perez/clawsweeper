## Summary

Keep this PR open, but do not merge the current diff as-is. The underlying Telegram bug is still present on current main, while the PR is stale against the Telegram plugin move and introduces blocking functional and security issues around media groups and unthrottled warning replies.

## What This Changes

The PR adds a Telegram warning when media resolution returns null and suppresses media placeholders when no attachment was resolved.

## Best Possible Solution

Update or replace the change under extensions/telegram so null media-resolution failures produce an accurate, rate-limited user-visible failure signal for both single messages and media groups, while preserving useful caption/file metadata without implying that an attachment was downloaded.

## Reproduction Assessment

Yes. A focused test can make ctx.getFile() throw Telegram's file-too-big error or exhaust retries, causing resolveMedia() to return null; current main then enqueues allMedia=[] and the body helper still emits placeholder context for captioned messages without a null-path warning.

## Solution Assessment

No. The single-message intent is good, but this PR is stale and incomplete: it misses media groups, sends unthrottled automatic replies, and uses wording that over-points users at the 20 MB limit even though the same null path covers transient retry exhaustion.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Handle all-failed media groups before suppressing placeholders:** `src/telegram/bot-message-context.body.ts:141`
  - body: Conditioning the placeholder on allMedia.length > 0 makes media-only albums with all null media resolutions return no body. Media-group updates return before the new warning branch, so an all-failed album can be silently ignored with no user-visible failure signal; add media-group warning or fallback handling before suppressing the marker.
  - confidence: 0.88
- **[P2] Throttle repeated media-failure replies:** `src/telegram/bot-handlers.ts:1031`
  - body: This branch sends a Telegram warning for every null media resolution in an allowed chat. A sender can repeatedly submit media that fails getFile() and make the bot spam the chat or burn send capacity; add per-chat, per-sender, or media-group cooldown before sending the automatic warning.
  - confidence: 0.78
- **[P3] Use reason-neutral fallback warning text:** `src/telegram/bot-handlers.ts:1031`
  - body: resolveTelegramFileWithRetry() returns null for both Bot API size-limit failures and exhausted transient retries, but this message points mainly at the 20 MB limit. Use wording that also covers temporary download failures unless the failure reason is carried through explicitly.
  - confidence: 0.79

## Security Review

Status: needs_attention

Summary: The diff has no supply-chain changes, but it adds an unthrottled automatic Telegram reply path for attacker-controlled media failures.

Concerns:

- **[medium] Unthrottled media-failure warnings can spam chats:** `src/telegram/bot-handlers.ts:1031`
  - body: Any permitted sender can repeatedly trigger null media resolution and cause the bot to send one warning reply per message, which can spam group chats or consume Telegram send capacity. Add cooldown, dedupe, or scope limits before merge.
  - confidence: 0.78
