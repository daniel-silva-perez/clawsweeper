## Summary

Keep open. Current main still has a Telegram getFile null-return path with no user-visible warning, but this PR is stale against the pre-plugin file layout and conflicts with the later shipped placeholder/file_id preservation behavior, so it needs rework rather than cleanup closure.

## What This Changes

The PR adds a Telegram warning when media resolution returns null and suppresses inbound media placeholders when no attachment was resolved.

## Best Possible Solution

Rework the fix against extensions/telegram so getFile null failures produce an explicit unavailable-media signal: keep useful media metadata such as type and file_id where appropriate, add a user-visible warning for single media and all-failed media groups, and cover captioned, media-only, and album cases with regression tests.
