## Summary

Keep this PR open. The linked bug is still not fully implemented on current main, but this branch is not mergeable as-is because it edits the old `src/telegram` layout while the live Telegram runtime now lives under `extensions/telegram/src`; it also conflicts with later shipped behavior that preserves unavailable-media context instead of suppressing it.

## What This Changes

The PR adds a Telegram warning reply when media resolution returns null and suppresses inbound media placeholders when no attachment was resolved.

## Best Possible Solution

Rework the fix against `extensions/telegram` so null `getFile()` failures have an explicit unavailable-media state: keep useful placeholder and `file_id` context, add a throttled user-visible warning where maintainers want one, cover single media plus all-failed albums, and preserve the later #59948 caption behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Port the fix to the live Telegram plugin path:** `src/telegram/bot-handlers.ts:53`
  - body: Current main no longer runs the `src/telegram` files changed here; the live Telegram handler and body builder are under `extensions/telegram/src`. As written, this branch will not affect the shipped Telegram runtime after rebase.
  - confidence: 0.91
- **[P2] Preserve a failure signal for all-failed media groups:** `src/telegram/bot-message-context.body.ts:141`
  - body: Conditioning the placeholder on `allMedia.length > 0` can drop media-only bodies when every album item resolves to null. Media-group updates return before the new warning branch, so an all-failed album can still disappear without user-visible or agent-visible context.
  - confidence: 0.86
- **[P3] Broaden the fallback warning text:** `src/telegram/bot-handlers.ts:1031`
  - body: The null media path covers both Telegram size-limit failures and exhausted transient retries. This reply mostly points users at the 20 MB limit, which can be misleading when a small file hit a temporary network failure.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The patch adds an automatic Telegram warning reply on a user-triggered media-failure path without cooldown or deduplication; no dependency, workflow, or package-resolution changes were found.

Concerns:

- **[medium] Throttle media-failure warning replies:** `src/telegram/bot-handlers.ts:1026`
  - body: Any authorized sender in a configured chat can repeatedly send media that makes `getFile()` return null and trigger one bot warning reply per input. Add per-chat, per-sender, or per-media-group suppression, or constrain repeated warnings in groups to avoid chat/API spam.
  - confidence: 0.72
