## Summary

Keep this PR open, but do not merge it as-is. Current main has moved Telegram runtime code under `extensions/telegram` and already partially superseded the placeholder-suppression approach through #59948, while the PR still leaves unbounded warning replies and can silently drop all-failed media groups.

## What This Changes

The PR adds a Telegram warning reply when inbound media resolution returns `null` and suppresses media placeholders when no attachment was resolved.

## Best Possible Solution

Rework the fix on current `extensions/telegram` code so media resolution preserves accurate unavailable-media metadata, sends an accurate bounded user warning for single messages and media groups, and covers the null `getFile()` paths with focused Telegram regression tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve a failure signal for all-failed albums:** `src/telegram/bot-message-context.body.ts:141`
  - body: Conditioning the placeholder on `allMedia.length > 0` makes media-only updates with no resolved attachments return `null`. Media-group updates return before the new single-message warning branch, so an album where every `getFile()` call returns `null` is silently dropped instead of notifying the user or preserving unavailable-media context.
  - confidence: 0.91
- **[P2] Throttle failed-media warning replies:** `src/telegram/bot-handlers.ts:1019-1036`
  - body: This branch sends a Telegram reply for every media message whose resolution returns `null`. Any authorized sender can repeatedly trigger `getFile()` failures, causing chat spam and extra Bot API calls; add per-chat/per-sender or media-group suppression before emitting the warning.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff does not add dependencies or workflow changes, but it introduces an attacker-triggerable Telegram warning reply without suppression.

Concerns:

- **[medium] Failed-media warning lacks abuse suppression:** `src/telegram/bot-handlers.ts:1019`
  - body: The new `sendMessage` path runs for every null media-resolution result. In an allowed chat, repeated failing media can spam the conversation and consume Telegram Bot API send capacity unless the warning is rate-limited, deduplicated, or restricted.
  - confidence: 0.86
