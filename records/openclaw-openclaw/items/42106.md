## Summary

Keep this issue open. Current main still loses paragraph-boundary semantics in the shared block-streaming path: the chunker can emit text before a paragraph separator and advance past the separator, downstream block reply payloads carry only plain text/media fields, and coalescing can only reinsert a joiner while chunks remain in the same in-memory buffer before a flush.

## Best Possible Solution

Keep this open and fix the shared block-streaming pipeline. Preserve boundary semantics from EmbeddedBlockChunker through block reply coalescing and delivery, either by attaching consumed separators to emitted chunks while normalizing only dedupe comparisons, or by carrying explicit boundary metadata such as paragraph/newline/hard split. Add a regression test that streams Markdown like "# Title\n\nFirst paragraph." across at least two flushed text_end block deliveries and asserts reconstructed visible text remains structurally equivalent.

## Risks / Open Questions

- A fix needs to avoid double-inserting paragraph separators when multiple chunks are coalesced into the same outbound payload, while preserving the separator when those chunks cross flush boundaries.
- Final-payload suppression currently normalizes whitespace across streamed fragments, so regression coverage should include both visible reconstruction and duplicate-suppression behavior.
