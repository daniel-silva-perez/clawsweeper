## Summary

Keep this PR open. Current main still does not load a selected agent workspace `.env` into the exec tool environment at request time; the PR remains a focused implementation candidate for that missing behavior, but it needs maintainer and security review before landing.

## Best Possible Solution

Keep this PR open as the implementation candidate. The best path is to review and land a core exec change that loads the resolved gateway workspace `.env` at request time, preserves non-override semantics including Windows casing, skips node and sandbox hosts, reuses or mirrors the existing workspace dotenv blocklist, filters dangerous host env overrides, uses isolated regression tests, and updates environment docs once shipped.

- Docs should be aligned when this lands: current environment docs describe current-CWD/global dotenv loading, while security docs already discuss workspace-local `.env` policy.
