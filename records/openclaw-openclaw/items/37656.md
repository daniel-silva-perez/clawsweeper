## Summary

Keep this PR open. Current main still does not load a resolved per-agent workspace `.env` into the `exec` tool environment at request time; the PR remains a focused implementation candidate for that missing behavior, with useful review history around security and reliability edge cases.

## Best Possible Solution

Keep this PR open as the canonical implementation path for the remaining per-agent workspace `.env` exec behavior. The best maintainer path is to review and land a core `exec` change that loads the resolved gateway workspace `.env` at request time, preserves non-override semantics including Windows casing, filters dangerous host env overrides, skips node/sandbox hosts, adds isolated regression tests, and updates environment docs once shipped.
