## Summary

Current main still does not load the selected agent workspace `.env` into exec at request time. This PR remains a focused implementation candidate for the missing behavior, but it changes host command environment sourcing and should stay open for maintainer/security review before landing.

## Best Possible Solution

Keep this PR open as the implementation candidate. The best path is to review and land a core exec change that loads the resolved gateway workspace `.env` at request time, preserves non-override semantics including Windows casing, skips node and sandbox hosts, filters through workspace dotenv and host override policies, uses isolated regression tests, and updates the environment docs once shipped.
