## Summary

Keep this open. Current main still renders the `Dreaming` line through the deep/short-term promotion resolver, so a valid light-enabled/deep-disabled setup can still appear as `Dreaming: off`. The issue is also paired with open, unmerged PR #67910, which explicitly proposes the phase-aware status fix and uses closing syntax, so the issue should wait for that PR or a replacement fix to land.

## What This Changes

The issue requests that `openclaw memory status` report active dreaming phases instead of showing `Dreaming: off` when global dreaming and the light phase are enabled but the deep phase is disabled.

## Best Possible Solution

Review and either land or replace #67910 with a focused phase-aware status fix: derive the `memory status` summary from the full dreaming config or the light/deep/REM phase resolvers, preserve accurate `off` output only when no phase is active, and add regression coverage for global dreaming enabled with light enabled and deep disabled.
