## Summary

Keep this open because current main still renders the `memory status` dreaming summary from the deep-derived short-term promotion resolver, so the reported light-enabled/deep-disabled false-negative remains possible. The issue is also paired with open, unmerged PR #67910, which explicitly targets this bug with closing syntax, so the issue should wait for that PR or a replacement fix to land.

## What This Changes

The issue requests phase-aware `openclaw memory status` output so light-only dreaming configurations are not reported as fully off.

## Best Possible Solution

Review and either land or replace #67910 with a focused memory-core fix that derives the status summary from the full dreaming config or the light/deep/REM phase resolvers, reports `off` only when no phase is active, and adds regression coverage for global dreaming enabled with light enabled and deep disabled.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: with global dreaming enabled, light enabled, and deep disabled, the current status formatter reads the deep-derived short-term promotion config and can return `off` without consulting the light phase.

## Solution Assessment

Yes, the requested phase-aware status summary is the narrow maintainable fix. The best path is normal review of #67910, or a replacement with the same boundary if that branch cannot be landed cleanly.
