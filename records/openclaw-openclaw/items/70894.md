## Summary

Current main still appears to reproduce the webhook session-continuity regression: `/hooks/agent` accepts and forwards caller-approved session keys, but the isolated-agent runner always passes `forceNew: true` for `sessionTarget: "isolated"`, so `resolveCronSession` bypasses the fresh-session reuse branch. The issue is also paired with open PR #70895 using closing syntax, so it should remain open until that PR, or a narrow equivalent, is reviewed and landed.

## Best Possible Solution

Keep this issue open and review/land PR #70895 or a narrow equivalent. The fix should compute `forceNew` so scheduled isolated cron jobs still get a fresh transcript, while hook/webhook-sourced isolated runs with approved stable session keys can reuse fresh existing sessions, with regression coverage for both paths.
