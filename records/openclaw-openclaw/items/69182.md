## Summary

This issue should stay open. Current main still implements Active Memory as a bounded but blocking `before_prompt_build` recall pass, so eligible chat turns can wait for the configured recall budget before prompt construction and first model output. Recent merged work improves timeout enforcement, setup-budgeting, partial timeout output, and recall-tool wiring, but it does not make Active Memory non-blocking or lower/auto-tune the default latency budget.

## Best Possible Solution

Keep this open as the focused Active Memory first-output latency follow-up. Maintainers should decide whether the plugin should add an explicit non-blocking/background mode, lower or auto-tune its default recall budget, or keep blocking semantics while adding a clearer low-latency preset and docs that set expectations for reply delay. The implementation should stay in `extensions/active-memory` unless a generic plugin hook capability is genuinely needed.
