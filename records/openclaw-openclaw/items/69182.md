## Summary

Current main fixes the more severe unbounded-timeout class, but this issue’s core complaint is still valid: active-memory remains a bounded blocking `before_prompt_build` pass that can delay first model output up to the configured recall budget. The docs and tests describe tuning/fail-open behavior, not a non-blocking default or reduced latency budget.

## Best Possible Solution

Keep this issue open as the focused active-memory latency/design follow-up. Maintainers should decide whether Active Memory should gain an explicit non-blocking or background mode, reduce/auto-tune the default timeout, or keep blocking semantics but expose a safer low-latency preset and clearer defaults.
