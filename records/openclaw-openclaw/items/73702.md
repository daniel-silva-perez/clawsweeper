## Summary

Current main fixes the reported sessions_send cross-session injection path by marking same-turn inter-session prompts and A2A reply prompts with a model-visible isUser=false envelope plus provenance before they reach the model. The fix is covered by current tests and docs, and was added after the latest release, so it is main-only for now.

## Best Possible Solution

Close this as implemented on current main. Keep the same-turn inter-session envelope and transcript provenance behavior; any broader cryptographic sender-envelope or capability-intersection design should be tracked separately from this already-fixed sessions_send bare-prompt path.
