Keep open. Current main still only partially mitigates bootstrap-file truncation through diagnostics and truncation-only warnings; it does not implement the requested Control UI near-limit warning, save refusal, or broader MEMORY.md archive workflow, and the issue is already paired with an open draft implementation PR with explicit closing linkage.

## What This Changes

The issue requests visible MEMORY.md/bootstrap-file size warnings, startup warnings, save-time enforcement near the bootstrap truncation limit, and optional auto-archive support.

## Best Possible Solution

Keep this issue open and use #64874, or a maintainer-owned successor, as the implementation vehicle. The end state should expose generic bootstrap-file health in Control UI, warn before truncation, make save diagnostics explicit, and treat auto-archive as a separate product decision unless maintainers want it in core.
