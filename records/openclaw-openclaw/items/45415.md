## Summary

Keep open. Current main has partial bootstrap-file visibility through doctor, /context, and prompt truncation warnings, but it does not fully implement the requested MEMORY.md/Control UI warning and limit-enforcement behavior. Also, open draft PR #64874 explicitly uses closing syntax for this issue, so the issue should stay open until that PR is reviewed and merged or closed.

## Best Possible Solution

Keep this issue open and use #64874 as the current implementation candidate. Maintainers should review whether that PR, possibly combined with the existing /context and doctor behavior, is enough to close the user-facing data-loss risk; any remaining Control UI save enforcement or auto-archive behavior should be either added there or split into a narrower follow-up.
