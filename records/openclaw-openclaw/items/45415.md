## Summary

Keep open. Current main has useful partial mitigation for bootstrap-file truncation through `/context`, doctor, prompt metadata, and truncation-only prompt warnings, but it still does not implement the requested MEMORY.md/Control UI size indicator, save-limit enforcement, near-limit session startup warning, or auto-archive flow. Open draft PR #64874 remains an unmerged implementation candidate that explicitly targets this issue, so this issue should stay open until that PR or a successor is reviewed and merged or closed.

## Best Possible Solution

Keep this issue open and use #64874, or a maintainer-owned successor, as the current implementation candidate. The best fix should derive thresholds from the resolved bootstrap limits, add concise `/status` or Control UI workspace-file health visibility, preserve `/context` as the detailed breakdown, improve edit-failure diagnostics, and split auto-archive/save-refusal into a narrower follow-up if it is not included in the first PR.
