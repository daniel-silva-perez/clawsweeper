## Summary

Keep open. Current main has partial bootstrap-file visibility through doctor, `/context`, and truncation-only prompt warnings, but it still lacks the requested Control UI MEMORY.md near-limit warning/save enforcement and there is an open draft implementation candidate, #64874, that explicitly targets this issue.

## What This Changes

The issue requests size indicators, warning thresholds, startup warnings, and optional archive/save enforcement for MEMORY.md as it approaches or exceeds the bootstrap file limit.

## Best Possible Solution

Keep this issue open and use #64874, or a maintainer-owned successor, as the implementation candidate. The fix should reuse the generic bootstrap budget analysis, expose per-file health to `/status` and/or the Control UI, warn before truncation, improve edit/save diagnostics, and split auto-archive into a narrower follow-up if needed.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to save a `MEMORY.md` larger than `agents.defaults.bootstrapMaxChars`, start or inspect a session, and compare `/context list` truncation data with the Control UI file editor/save path; code inspection shows the save path accepts the content while prompt injection later truncates it.

## Solution Assessment

Unclear as implemented today. Current main is a partial mitigation, and #64874 looks like a narrow candidate for visibility and diagnostics, but Control UI save enforcement and auto-archive should be reviewed as separate product decisions or added deliberately to the same generic bootstrap-file health path.
