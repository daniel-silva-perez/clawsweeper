## Summary

Keep open. Current main has useful bootstrap budget diagnostics and truncation-only warnings, but it does not implement the requested Control UI near-limit indicator, save refusal, or archive workflow; the issue is also paired with an open draft implementation PR that should be reviewed, landed, or replaced before closing.

## What This Changes

The issue requests MEMORY.md size visibility, warning thresholds, save-time enforcement at the bootstrap limit, a startup warning, and optional archiving for old memory sections.

## Best Possible Solution

Keep the issue open and use #64874, or a maintainer-owned successor, as the implementation vehicle. The end state should expose generic bootstrap-file budget metadata through the gateway, show warning/error thresholds in the Control UI file editor, make over-limit saves explicit or blocked according to maintainer policy, and split auto-archive into a deliberate follow-up if needed.

## Reproduction Assessment

Yes. A source-level reproduction is to save a MEMORY.md larger than agents.defaults.bootstrapMaxChars through agents.files.set or the Control UI file editor; current main accepts the save, and later bootstrap injection trims the file before prompt inclusion.

## Solution Assessment

No, current main is not a complete solution. The narrow maintainable path is to reuse the existing bootstrap budget analyzer for RPC/UI save-time diagnostics and review #64874 for the partial warning/status work before closing this issue.
