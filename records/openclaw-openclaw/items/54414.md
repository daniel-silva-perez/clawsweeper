## Summary

Keep this PR open. Current main already shipped the Slack thread explicit-mention control under the finalized key `channels.slack.thread.requireExplicitMention`, but this PR still contains a useful account-merge fix that current main does not have: top-level Slack `thread.*` settings are shallow-replaced by named account `thread` overrides. The PR should not merge as-is with the old `autoReplyOnParticipation` key, but the account deep-merge part should be ported or tracked before closing it as obsolete.

## Best Possible Solution

Port the useful part of this PR onto current main by passing `nestedObjectKeys: ["thread"]` in `mergeSlackAccountConfig` and adding a Slack account test that proves global `thread.requireExplicitMention: true` survives account-level `thread.historyScope` overrides. Keep the shipped public key and docs; after that lands or a canonical follow-up exists, this PR can be closed as superseded/implemented.
