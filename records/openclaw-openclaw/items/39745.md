## Summary

Keep this PR open. Current main and the latest release already include the Twitch timeout cleanup, but Synology Chat and Zalouser still have setTimeout-backed Promise.race timeouts without clearing the losing timer, so the PR remains partially useful rather than obsolete.

## Best Possible Solution

Keep this PR open. Ask the author to rebase on current main, drop or resolve the obsolete Twitch hunk, keep the Synology Chat and Zalouser timer cleanup, and add focused success-path fake-timer coverage where feasible. Maintainers should also make the product call on whether Synology Chat should retain the local 120s timeout or remove it in favor of the core agent timeout.
