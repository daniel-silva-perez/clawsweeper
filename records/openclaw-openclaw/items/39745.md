## Summary

Keep this PR open. Current main already has the Twitch timeout cleanup, but Synology Chat and Zalouser still create setTimeout-backed Promise.race timeouts without storing and clearing the timer when the primary operation wins, so the PR remains partially useful rather than obsolete.

## Best Possible Solution

Keep this PR open. Ask the author to rebase on current main, drop or resolve the obsolete Twitch hunk, retain the Synology Chat and Zalouser timer cleanup, and add focused success-path fake-timer coverage where feasible. Maintainers should still choose whether Synology Chat should keep the plugin-local 120s timeout with cleanup or follow the closed #38360 direction and rely on the core agent timeout instead.
