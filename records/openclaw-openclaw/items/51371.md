## Summary

Keep this PR open. Current main and the latest release still resume the WebSocket ping CheckedContinuation directly, while the PR is a narrow one-file guard for the same active keepalive crash path and appears to be the remaining canonical implementation candidate after overlapping reports were closed.

## What This Changes

The PR adds a private NSLock-backed OneShotContinuation wrapper in OpenClawKit and routes WebSocketTaskBox.sendPing() through it so duplicate pong callbacks only resume the CheckedContinuation once.

## Best Possible Solution

Land one canonical at-most-once guard in apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayChannel.swift, ideally with a focused OpenClawKit regression test for duplicate ping callbacks; keep the older overlapping PRs closed and use this PR as the current review target unless a maintainer chooses a replacement implementation.

## Reproduction Assessment

Yes. A focused fake WebSocketTasking that invokes pongReceiveHandler twice would deterministically exercise the current-main double-resume path, and the PR body plus #55878 provide matching CheckedContinuation crash signatures from the active keepalive code.

## Solution Assessment

Yes. The PR's one-shot wrapper is a narrow owner-module fix for the implicated callback contract; the main improvement before landing would be adding regression coverage, not broadening the reconnect or keepalive design.
