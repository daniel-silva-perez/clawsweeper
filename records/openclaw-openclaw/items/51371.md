## Summary

Current main still has the unguarded `CheckedContinuation` resume in `WebSocketTaskBox.sendPing()`, and the gateway keepalive loop still calls that method. This PR remains a plausible narrow fix for an active crash path, with overlapping open PRs that should be consolidated by maintainers rather than auto-closing this candidate.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land one narrow at-most-once guard around `WebSocketTaskBox.sendPing()` in `apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayChannel.swift`, ideally with a duplicate-callback regression test in OpenClawKit tests, then close the overlapping PRs against the selected merged implementation.
