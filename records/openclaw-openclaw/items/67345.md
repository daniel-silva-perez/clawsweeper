## Summary

Keep this PR open. Current main still does not implement the proposed claw-code parity aliases, unified MCP/LSP bridge tools, or compat utility-tool surface, and the PR branch has concrete security and correctness issues that should block merge until maintainers decide whether to split, harden, or move parts of it out of core.

## What This Changes

The PR adds claw-code tool-call alias normalization, MCP/LSP/Hermes/task/team/compat tools, sandbox and file-tool hardening, a parity harness, catalog/tool-registry wiring, and related tests.

## Best Possible Solution

Split the work into reviewable pieces: one narrow runner normalization PR, one MCP resources/auth bridge PR, one LSP dispatch extension PR, one sandbox/parity-harness path coordinated with #67333, and separate product/security decisions for Hermes/task/team/compat utility tools. Any compat utility that remains in core should use existing owner-only gating, SSRF-protected networking, redacted or allowlisted config reads, bounded waits, and repo-compliant flat enum schemas.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Route remote_trigger through guarded fetch:** `src/agents/tools/claw-compat-tools.ts:635-640`
  - body: `remote_trigger` performs a direct `fetch()` against a model-supplied URL with caller-controlled method, headers, and body. This bypasses the existing SSRF guard path, so private IPs, localhost services, and metadata endpoints become reachable from the broadly registered coding-profile tool.
  - confidence: 0.94
- **[P1] Redact config_compat results before returning config:** `src/agents/tools/claw-compat-tools.ts:577-582`
  - body: When neither `setting` nor `path` is supplied, this returns the entire `loadConfig()` object to the model context; with a path, callers can still target known secret-bearing fields. Config results need an allowlist or sensitive-field redaction before this tool is exposed.
  - confidence: 0.9
- **[P1] Gate config_compat writes behind owner authorization:** `src/agents/tools/claw-compat-tools.ts:551-555`
  - body: Passing `setting` plus `value` writes persisted config with `writeConfigFile`, but the tool is not marked owner-only. Non-owner sessions that can call the coding-profile tool could mutate runtime/provider/gateway configuration.
  - confidence: 0.91
- **[P1] Restrict plan-mode tools before persisting config:** `src/agents/tools/claw-compat-tools.ts:238-239`
  - body: `enter_plan_mode` and `exit_plan_mode` persist `permissions.defaultMode` changes, but these compat tools are registered as normal coding tools rather than owner-only control-plane tools. That lets unprivileged callers change persisted permission mode.
  - confidence: 0.88
- **[P2] Preserve LSP path so it becomes a file URI:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:413-417`
  - body: The alias shim copies `path` into `uri`, but `createLspTool` treats any `uri` as already canonical and only converts `path` to a `file://` URI when `uri` is absent. Common calls like `LSP { path: "src/main.ts" }` will send a non-URI document identifier to the language server.
  - confidence: 0.86
- **[P2] Resolve safe MCP server names before calling tools:** `src/agents/tools/mcp-tool.ts:151-153`
  - body: Qualified MCP names parse the prefix directly into `serverName`, but the runtime stores sessions under the original config key while catalog entries expose sanitized `safeServerName` values. Catalog-driven calls for sanitized server keys will fail with `server ... is not connected`.
  - confidence: 0.83
- **[P2] Do not report message delivery without sending:** `src/agents/tools/claw-compat-tools.ts:189-192`
  - body: `send_user_message` returns `delivered: true` but only echoes JSON and never calls the message/channel delivery path. Models using this tool can silently drop user-facing output while receiving a success signal.
  - confidence: 0.84
- **[P2] Replace Type.Union action schemas with flat enums:** `src/agents/tools/mcp-tool.ts:9-12`
  - body: The new tool schemas use `Type.Union([...Type.Literal])`, which emits `anyOf`-style schemas that this repo avoids for provider tool compatibility. Use the existing flat string enum helpers for action/runtime/status fields before exposing these tools broadly.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: The PR introduces security-sensitive network and control-plane tools that are broadly exposed without the repo's existing guards.

Concerns:

- **[high] SSRF-capable remote_trigger:** `src/agents/tools/claw-compat-tools.ts:635`
  - body: The new `remote_trigger` tool performs direct outbound fetches to model-supplied URLs rather than using the SSRF-protected fetch helper, exposing localhost, link-local, cloud metadata, and internal network targets.
  - confidence: 0.94
- **[high] Runtime config disclosure and mutation:** `src/agents/tools/claw-compat-tools.ts:535`
  - body: `config_compat` can return full or targeted runtime config values and can persist config writes without owner-only authorization; inline provider keys, gateway tokens, plugin secrets, and permission settings may be exposed or changed.
  - confidence: 0.91
- **[high] Unprivileged persisted permission-mode changes:** `src/agents/tools/claw-compat-tools.ts:238`
  - body: The plan-mode compatibility tools write `permissions.defaultMode` through `writeConfigFile` without owner-only gating, creating a control-plane mutation path for normal coding-profile callers.
  - confidence: 0.88
