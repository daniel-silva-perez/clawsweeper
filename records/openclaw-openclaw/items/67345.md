## Summary

Keep this PR open. Current main still lacks the broad claw-code parity bridge, unified MCP/LSP compatibility tools, and claw-compat utility-tool surfaces described by the PR, while the PR is paired with open same-author work in #67333 and has unresolved security-sensitive review findings.

## Best Possible Solution

Keep this PR open for maintainer product and security review, but ask for a split or restack. A practical path is separate PRs for runner/tool-call normalization, MCP resources/auth support, LSP dispatch extensions, sandbox/parity-harness work with #67333, and any Hermes/task/team surfaces maintainers actually want in core. Compatibility utility tools should either move to plugin/ClawHub surfaces or land only with SSRF protection, config redaction or strict allowlists, owner-only persisted mutations, bounded waits, and schema patterns that match repo guardrails.
