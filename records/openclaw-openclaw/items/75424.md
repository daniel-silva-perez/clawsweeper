## Summary

Keep open. This is a targeted, technically sound fix for the Discord SecretRef failure described by #75324, and I found no blocking code or security regression in the proposed patch. The only concrete merge blocker I found is repo policy: this user-visible Discord fix needs an `Unreleased` changelog entry before landing.

## What This Changes

The branch changes Discord message-action discovery to inspect configured accounts without resolving token SecretRefs, updates the agent message tool to resolve scoped channel SecretRefs even when constructed with an injected config snapshot, and adds regression tests for both paths.

## Best Possible Solution

Land this targeted fix after adding a single-line `Unreleased > Fixes` changelog entry with proper contributor/reporter attribution, keeping the implementation in the Discord plugin and the shared scoped command-secret resolver. #75324 should close when this merges.

## Reproduction Assessment

Yes. Current main has a clear reproduction path from #75324: a Discord token stored as a SecretRef reaches discovery through `resolveDiscordAccount` and strict token normalization, while the PR adds focused regression coverage for both discovery and injected-config execution.

## Solution Assessment

Yes for the technical fix. Using `inspectDiscordAccount` for discovery avoids reading secrets, and using the existing scoped command-secret resolver for execution is the narrowest maintainable path; the remaining issue is release-note hygiene, not the code approach.
