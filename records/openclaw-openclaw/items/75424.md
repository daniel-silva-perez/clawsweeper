The PR updates Discord message-action discovery to inspect configured accounts without resolving token SecretRefs, resolves scoped channel SecretRefs during agent message-tool execution even with injected config snapshots, and adds focused regression tests for both paths.

## Best Possible Solution

Land the targeted Discord/plugin-owned implementation after adding a single-line `CHANGELOG.md` entry under `Unreleased > Fixes` with proper non-bot attribution, then let the linked bug close when this PR merges.

## Reproduction Assessment

Yes. The linked bug gives concrete SecretRef config and failure steps, and current main shows the same source-level path: discovery enters strict Discord token normalization before action schema generation, while injected-config execution skips scoped command-time resolution.

## Solution Assessment

Yes for the code approach. Using config-only Discord account inspection for discovery avoids reading secrets, and reusing the existing scoped command-secret resolver for execution is the narrowest maintainable fix; the remaining issue is release-note hygiene.
