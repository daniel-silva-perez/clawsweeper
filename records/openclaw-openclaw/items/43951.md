## Summary

Keep this PR open. Current main still drops `accountId` before `/hooks/agent` reaches isolated cron delivery, and the same-author tracker #43866 is still open, so this is not conservative cleanup-close material. The PR remains useful but needs maintainer review before landing because it is merge-conflicted against current main and introduces a concrete hook-name sanitization regression in the proposed dispatch rewrite.

## What This Changes

The PR adds `accountId` to hook payload normalization and hook delivery, routes hook dispatch through cron normalization, adjusts shared cron message-tool policy, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Keep the PR and its linked issue open, then either rebase this branch or replace it with a focused hook-owned fix that preserves sanitized hook metadata, normalizes/trims `accountId`, propagates it into cron delivery, includes it in replay scope, documents the field publicly, and adds regression coverage for direct hook dispatch plus any intended mapped-hook account routing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Sanitize the hook job name before normalization:** `src/gateway/server/hooks.ts:60`
  - body: The current hook path sanitizes `value.name` before storing it on the synthetic cron job. This rewrite passes raw `value.name` into `normalizeCronJobCreate()`, and cron external-content handling later emits `job.name` in the prompt's `Task:` context outside the untrusted-content wrapper. A hook caller can therefore inject `System:`/`[System]` style markers through the `name` field. Pass `safeName` into the normalizer instead.
  - confidence: 0.9
- **[P2] Add the required changelog attribution:** `CHANGELOG.md:1263`
  - body: OpenClaw changelog bullets require a `Thanks @...` attribution. This new entry has no attribution, so it needs the credited GitHub username before the PR can land cleanly.
  - confidence: 0.98

## Security Review

Status: needs_attention

Summary: The diff has a concrete prompt-safety regression from raw hook names entering cron prompt context, and the account-routing capability needs maintainer authorization review.

Concerns:

- **[high] Raw hook name reaches prompt context:** `src/gateway/server/hooks.ts:60`
  - body: The PR passes caller-controlled `value.name` into `CronJob.name`; cron external-content prompts include that job name in a `Task:` context line, bypassing the existing `sanitizeInboundSystemTags()` protection used on main.

  - confidence: 0.72
