## Summary

Keep this issue open. Current main has partial runtime handling for OpenAI-compatible reasoning deltas, Nemotron/vLLM visible-content request shaping, and retry recovery for reasoning-only turns, but it does not implement the requested reasoning-aware benchmark/report schema, separate visible-content versus reasoning-aware scores, extraction-source counters, or generic model/profile policy fields. The reporter also clarified that this broader product/runtime/reporting work should remain separate from #74017.

## Best Possible Solution

Keep this open for maintainer design review. The clean path is a reporting-first change that records raw visible content/reasoning presence, extraction source, compatibility counters, and separate reasoning-aware versus visible-content scores, followed by a reviewed runtime/provider-profile contract for opt-in reasoning-field use and visible-content enforcement. Keep #74017 as the separate scorer-bug history.
