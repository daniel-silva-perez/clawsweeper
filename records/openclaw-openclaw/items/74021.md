## Summary

Keep this issue open. Current main has useful partial runtime support for reasoning fields and non-visible/reasoning-only turns, including OpenAI-compatible reasoning-delta ingestion and vLLM Nemotron `force_nonempty_content` when thinking is off. It does not implement the requested benchmark/reporting surface, extraction-source result schema, scoring modes, or model-profile knobs, and the related scorer bug remains a separate open issue.

## Best Possible Solution

Keep this open for maintainer design review, then split the work into focused changes: one runtime/provider-profile contract for reasoning-field and visible-content policy, and one benchmark/reporting change that records raw content/reasoning, extraction source, compatibility counters, and separate reasoning-aware versus visible-content-only scores. The narrower scorer bug in #74017 should stay tracked separately.
