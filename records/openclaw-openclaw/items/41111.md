## Summary

Keep this open. Current main still has first-party backup create and verify only; there is no native `openclaw backup restore <archive>` command in the CLI registration, command implementation files, docs, generated CLI index, or parser tests. The keepmyclaw ClawHub plugin linked in discussion is useful prior art and a workaround, and open PR #59280 is a plausible implementation path, but neither means the requested core restore workflow has shipped.

## Best Possible Solution

Keep this issue open until OpenClaw ships a safe native `openclaw backup restore <archive>` workflow or maintainers explicitly consolidate it into a canonical backup/restore tracker. The practical next step is to review and either land or close PR #59280, with focused implementation in the backup CLI surface, docs at `docs/cli/backup.md`, generated CLI index updates, and tests for parsing, validation, dry-run, conflict, force/staging, and rollback behavior.
