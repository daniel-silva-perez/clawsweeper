## Summary

Keep this issue open. Current main still implements first-party `openclaw backup create` and `openclaw backup verify` only; there is no native `openclaw backup restore <archive>` command. The linked keepmyclaw ClawHub plugin is useful prior art and a workaround, and PR #59280 remains an open implementation candidate, but neither makes the requested core CLI workflow shipped on main.

## Best Possible Solution

Keep this issue open until OpenClaw ships a safe native `openclaw backup restore <archive>` workflow or maintainers explicitly consolidate the request into a canonical backup/restore tracker. The practical next step is to review and either land or close PR #59280, with implementation in the backup CLI surface plus docs and focused tests for parsing, validation, dry-run, conflict handling, force/staging, and rollback behavior.
