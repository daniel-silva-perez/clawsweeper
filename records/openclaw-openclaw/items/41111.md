## Summary

Keep this issue open. Current main at 14f140d6f0d31fed802869622c38c7e8f8e19587 still exposes and documents only `openclaw backup create` and `openclaw backup verify`; there is no native `openclaw backup restore <archive>` command. The keepmyclaw ClawHub plugin is useful prior art and a workaround, and PR #59280 is still an open implementation candidate, but the first-party core workflow requested here has not landed.

## Best Possible Solution

Keep this issue open until OpenClaw ships a safe native `openclaw backup restore <archive>` workflow or maintainers explicitly consolidate the request into a canonical backup/restore tracker. The next concrete path is to review, land, or close PR #59280, with implementation in the backup CLI surface plus docs and focused tests for validation, dry-run, conflict handling, force/staging, and rollback behavior.
