## Summary

Keep this issue open. Current main still exposes and documents only `openclaw backup create` and `openclaw backup verify`; there is no native `openclaw backup restore <archive>` command. The keepmyclaw ClawHub plugin is useful prior art and workaround, and open PR #59280 remains the direct implementation candidate, but the first-party restore workflow requested here has not shipped.

## Best Possible Solution

Keep this issue open until OpenClaw ships a safe native `openclaw backup restore <archive>` workflow or maintainers explicitly consolidate it into a canonical backup/restore tracker. The concrete path is to review, land, or close PR #59280, with implementation in the backup CLI surface, focused restore infra/command tests, docs at `docs/cli/backup.md`, and safety behavior for validation, dry-run, conflicts, force/staging, and rollback.
