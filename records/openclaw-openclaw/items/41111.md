## Summary

Keep this issue open. Current main still exposes first-party `openclaw backup create` and `openclaw backup verify` only; a native `openclaw backup restore <archive>` command has not shipped. The keepmyclaw ClawHub plugin linked in discussion is useful prior art and a workaround, and open PR #59280 remains the direct implementation candidate, but neither resolves the requested core CLI workflow on main.

## Best Possible Solution

Keep this issue open until OpenClaw ships a safe native `openclaw backup restore <archive>` workflow or maintainers explicitly consolidate the work into a canonical backup/restore tracker. The practical next step is to review and either land or close PR #59280, with implementation in the backup CLI surface, docs at `docs/cli/backup.md`, CLI index updates, and focused tests for parsing, validation, dry-run, conflicts, force/staging, and rollback behavior.
