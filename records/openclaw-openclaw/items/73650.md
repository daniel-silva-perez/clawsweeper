## Summary

Current main still has the exact duplicated status-task reconciliation this PR targets: `getStatusSummary()` calls the registry and audit helpers separately, and each helper independently calls `reconcileInspectableTasks()`, which in turn calls `listTaskRecords()` and maps the result. The PR is an unmerged, focused implementation candidate for the still-open performance issue it closes, so it should stay open for normal review rather than be swept as obsolete.

## Best Possible Solution

Keep this PR open and review or land this focused snapshot-sharing implementation, or an equivalent main-branch change, with the proposed regression tests. The linked issue should remain open until the PR or equivalent fix merges.
