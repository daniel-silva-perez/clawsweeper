## Summary

Keep open. Current `main` still performs two independent task reconciliations in the `openclaw status` summary path, and this PR is a focused implementation candidate for open performance issue #73531. The touched diff is limited to CLI/status task summary code, regression tests, and changelog; I did not find unrelated security-sensitive or supply-chain changes in the provided PR file set.

## Best Possible Solution

Keep this PR open for normal maintainer review. Land this shared-snapshot implementation with its proposed regression coverage, or replace it with an equivalent main-branch fix that makes `getStatusSummary()` reconcile inspectable tasks once while preserving the zero-argument helper behavior used by other call sites.
