## Summary

Keep this PR open. Current main still calls the registry and audit summary helpers separately, and those helpers still reconcile task records independently, so the performance issue this PR targets is not implemented on main. The PR is a focused, recent implementation candidate for the still-open #73531, with no visible security-sensitive or supply-chain side changes in the provided diff.

## Best Possible Solution

Keep this PR open for normal maintainer review. Land this shared-snapshot implementation with its regression tests, or replace it with an equivalent main-branch fix that makes getStatusSummary() reconcile inspectable tasks once and passes the same snapshot into the registry and audit summaries. Keep #73531 open until a fix merges.
