## Summary

Keep open. Current main still renders TUI media through text/Markdown paths and explicitly converts tool-result image blocks into bracketed placeholders. I found no Kitty, iTerm2, or Sixel rendering path, no `tui.inlineImages`-style config, and no public TUI docs for inline terminal images. Related PR #49022 remains an open implementation candidate, not a shipped replacement.

## Best Possible Solution

Keep this issue open. The best path is a core TUI implementation that preserves assistant/tool image blocks through the render model, detects or configures iTerm2/Kitty/Sixel support, emits safe terminal-specific image output with placeholder fallback, documents the setting/behavior, and adds focused regression coverage. Related PR #49022 is the likely starting point or canonical candidate to review/land/supersede.
