## Summary

Keep open. Current main still converts TUI tool-result image blocks into bracketed text placeholders and the assistant/chat rendering path remains string/Markdown-only. I found no iTerm2, Kitty, or Sixel graphics protocol path, no `tui.inlineImages`-style config, and the related draft PR #49022 is still an implementation candidate rather than shipped behavior.

## Best Possible Solution

Keep this issue open. The best path is to review and either land or supersede #49022 with a core TUI implementation that preserves image media through the render model, safely detects or configures iTerm2/Kitty/Sixel support, renders images inline with graceful placeholder fallback, and documents/tests the behavior.
