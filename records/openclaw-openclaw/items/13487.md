## Summary

Keep open. Current main still lacks deterministic Discord multi-bot target selection: Discord preflight resolves the normal route before target signals, only checks mentions/replies against the running handler bot ID, and has no suppression state to stop a default owner when another configured bot is targeted.

## What This Changes

The issue requests deterministic Discord multi-bot routing precedence: explicit bot mention first, replied-to bot second, and channel default owner only when no targeted signal exists.

## Best Possible Solution

Implement target selection inside the Discord plugin before shared default routing: map configured Discord bot user IDs to OpenClaw accounts/agents, select explicit mentioned bot accounts first, select the referenced-message bot author second, and fall back to the channel default owner only when no targeted signal exists. Add focused coverage proving targeted cases suppress the default owner.

## Reproduction Assessment

Yes. GTZhou provided a live guild-channel repro, and current main has a static reproduction path: route selection happens before mention/reply target evaluation, and target checks only compare against the running botUserId.

## Solution Assessment

Yes. The narrow maintainable path is a Discord-owned target-selection step plus a small generic suppression seam only if shared mention-gating needs to express default-owner suppression; hardcoding Discord identities in core would cross the plugin boundary.
