## Summary

Keep this issue open. Current main still resolves Discord routing from account/conversation bindings before evaluating Discord mention or reply-target metadata, and the mention/reply checks only apply to the current monitor's bot user id. That does not yet provide the requested Discord-wide precedence of explicit bot mention, then replied-to bot, then default owner with default-owner suppression. GTZhou's reproduction note and the maintainer comment already point in the same direction; related PRs #47393 and #67460 are useful partial work but are not merged and do not fully cover reply-target precedence plus default suppression.

## Best Possible Solution

Keep this issue open until the Discord plugin makes one deterministic target decision before default routing: map configured Discord bot user IDs to OpenClaw accounts/agents, select explicit bot mentions first, select the referenced-message author second, and use the per-channel default owner only for untagged non-reply messages. Add focused Discord preflight tests for explicit mention, reply-target, untagged default fallback, and default-owner suppression. Review #47393 as a possible base and #67460 as a possible shared suppression seam, but neither is sufficient alone on current main.
