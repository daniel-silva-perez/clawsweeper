## Summary

Keep open. Current main still does not implement the requested Discord multi-bot precedence: routing is selected before Discord mention/reply target signals are interpreted, those signals are checked only against the running handler's bot user ID, and the shared mention-gating/default-route path has no suppression result for a default owner when another configured bot is targeted.

## What This Changes

Request deterministic Discord routing where explicit bot mentions win over reply-target bots, reply-target bots win over the per-channel default owner, and the default owner is suppressed whenever a targeted signal exists.

## Best Possible Solution

Implement target selection in the Discord plugin before shared default routing: derive a configured Discord bot user ID to OpenClaw account/agent map, choose explicit mentioned bot accounts first, choose the referenced-message bot author second, and fall back to the per-channel default owner only when no targeted signal exists. Add focused Discord preflight, shared mention-gating, and route resolver coverage proving the default owner is suppressed in targeted cases.
