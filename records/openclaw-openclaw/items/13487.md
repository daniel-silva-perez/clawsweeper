## Summary

Keep this issue open. Current main still routes Discord messages to a per-account/channel route before evaluating mention or reply-target metadata, and the mention logic only gates the current bot handler. That does not yet implement the requested cross-account precedence of explicit bot mention, then replied-to bot, then default owner with default-owner suppression.

## Best Possible Solution

Keep this issue open until the Discord plugin makes one deterministic target decision before default routing: map configured Discord bot user IDs to OpenClaw accounts/agents, select explicit bot mentions first, then referenced-message author, and use the per-channel default owner only for untagged non-reply messages. A reviewed version of #47393 could be the base if it is expanded to cover reply-target precedence and default-owner suppression; #67460 may cover a generic suppression seam but is not enough by itself for Discord reply-target routing.
