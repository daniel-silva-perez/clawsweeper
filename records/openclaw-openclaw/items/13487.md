## Summary

Keep open. Current `origin/main` still does not implement the requested Discord multi-bot precedence: routing is resolved before Discord target signals are interpreted, explicit mentions and reply-target checks compare only against the running handler's bot ID, and shared mention gating has no default-owner suppression result. The related open PRs are partial overlap, not a merged fix for this issue.

## What This Changes

The issue requests deterministic Discord multi-bot routing where explicit bot mentions outrank reply targets, reply targets outrank the channel default owner, and targeted messages suppress the default owner.

## Best Possible Solution

Implement target selection inside the Discord plugin before shared default routing: map configured Discord bot user IDs to OpenClaw accounts/agents, select explicit mentioned bot accounts first, select the referenced-message bot author second, and fall back to the per-channel default owner only when no targeted signal exists. Add focused Discord preflight, shared mention-gating, and route resolver coverage proving targeted cases suppress the default owner.
