## Summary

Keep open. Current main still does not implement the requested Discord multi-bot precedence: Discord preflight resolves the normal route before target signals, mention and reply-target detection only compare against the currently running handler bot ID, and shared mention gating has no default-owner suppression result. The related open PRs are partial overlaps, not merged fixes.

## What This Changes

Request strict Discord multi-bot routing precedence where explicit bot mentions win over reply targets, reply targets win over the per-channel default owner, and targeted cases suppress the default owner.

## Best Possible Solution

Implement target selection inside the Discord plugin before shared default routing: map configured Discord bot user IDs to OpenClaw accounts/agents, select explicit mentioned bot accounts first, select the referenced-message bot author second, and fall back to the per-channel default owner only when no targeted signal exists. Add focused Discord preflight, shared mention-gating, and route resolver coverage proving targeted cases suppress the default owner.
