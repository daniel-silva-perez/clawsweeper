## Summary

Keep open. Current main still does not implement deterministic Discord multi-bot routing with mention > reply-target > default-owner precedence: Discord preflight only knows the running handler's bot user ID, resolves the effective route before interpreting target signals, treats reply metadata as an implicit mention only for that same bot ID, and has no default-owner suppression decision for messages targeting another configured bot.

## What This Changes

The issue requests deterministic single-target routing for Discord multi-bot channels: explicit bot mentions win first, replies to a bot message win second, and the channel default owner responds only when no targeted signal is present.

## Best Possible Solution

Implement the precedence in the Discord plugin before shared default routing: build a configured Discord bot user ID to OpenClaw account/agent map, choose explicitly mentioned bot accounts first, choose the referenced-message bot author second, and use the per-channel default owner only when no targeted signal exists. Add focused Discord preflight, shared mention-gating, and route resolver coverage proving default-owner suppression in targeted cases.
