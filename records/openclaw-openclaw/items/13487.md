## Summary

Keep this issue open. Current main at 9bb1e59447a5fdf2c0164672df6c748188b26442 still resolves the Discord route before mention/reply-target selection, only compares mention/reply metadata to the current handler bot id, and lacks a default-owner suppression result or acceptance tests for mention > reply-target > default-owner precedence.

## Best Possible Solution

Keep this issue open until the Discord plugin makes one deterministic target decision before default routing: map configured Discord bot user IDs to OpenClaw accounts/agents, select explicit bot mentions first, select the referenced-message author second, and use the per-channel default owner only for untagged non-reply messages. Review #47393 and #67460 as ingredients, then add focused Discord preflight and routing tests for the requested precedence and suppression behavior.
