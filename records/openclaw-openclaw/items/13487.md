## Summary

Keep open. Current main still routes Discord messages before making a cross-bot target decision, only carries the running handler's bot user ID into preflight, and has no suppression result that would stop the default owner when another configured bot is explicitly mentioned or replied to. The related discussion and open PRs are useful source material, but no merged implementation covers the requested precedence yet.

## What This Changes

Request deterministic Discord multi-bot routing where explicit bot mentions win first, reply-to-bot metadata wins second, and the per-channel default owner handles only untagged non-reply messages.

## Best Possible Solution

Implement the precedence inside the Discord plugin before shared default routing: build a configured Discord bot user ID to OpenClaw account/agent map, select explicit mentioned bot accounts first, select the referenced-message bot author second, and use the per-channel default owner only when no targeted signal exists. Add focused preflight/routing tests proving the default owner is suppressed in targeted cases.
