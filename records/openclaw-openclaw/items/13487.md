## Summary

Keep this issue open. Current main still lacks Discord single-target precedence for explicit bot mentions, reply-target bot selection, and default-owner suppression. The issue discussion includes a fresh maintainer review saying the same gap remains, and the current code still routes before mention/reply targeting and only compares mention/reply metadata to the current handler's bot id.

## Best Possible Solution

Keep this open until the Discord plugin makes one deterministic target decision before default routing: map configured Discord bot user IDs to OpenClaw accounts/agents, select explicit bot mentions first, select the referenced-message author second, and use the per-channel default owner only for untagged non-reply messages. Add focused Discord preflight tests for explicit mention, reply-target, untagged default fallback, and default-owner suppression. Maintainers should review #47393 and #67460 as possible ingredients, but neither is sufficient alone on current main.
