## Summary

Keep open. Current main still lacks deterministic Discord multi-bot target selection: the Discord preflight route is resolved before explicit mention/reply-target signals are interpreted, mention/reply checks only compare against the running handler's bot ID, and shared mention gating has no default-owner suppression result for messages targeting another configured bot.

## What This Changes

Request deterministic Discord routing precedence where explicit bot mentions win, then Discord reply-target bot authors, then the per-channel default owner, with default-owner suppression for targeted messages.

## Best Possible Solution

Implement the precedence inside the Discord plugin before shared default routing: build a configured Discord bot user ID to OpenClaw account/agent map, choose explicitly mentioned bot accounts first, choose the referenced-message bot author second, and use the per-channel default owner only when no targeted signal exists. Add focused Discord preflight, shared mention-gating, and route resolver coverage for default-owner suppression in targeted cases.
