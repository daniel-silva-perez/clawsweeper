## Summary

This issue should stay open. Current main still uses a route-relative `favicon.svg` fallback for an empty Control UI base path, the sidebar header renders that helper directly, and tests still assert the old behavior. The provided GitHub context also shows PR #56357 as an open, unmerged implementation candidate for this exact header logo/favicon regression, so this issue should not be closed before that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review PR #56357 against current main, or land an equivalent fix that makes the empty-basePath logo/favicon URLs absolute and updates regression coverage for the header/logo surfaces. Close only after that change merges.
