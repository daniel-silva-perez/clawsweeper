## Summary

Keep this PR open. Current main still uses exact-only `name`/`skillKey` matching for `openclaw skills info <name>`, while the PR is a focused contributor fix for the still-present CLI mismatch from #38546. The PR should not be closed as implemented or superseded; it needs review/land work, with one remaining ambiguity issue in the PR resolver to address before merge.

## Best Possible Solution

Keep this PR open. The best path is to update `src/cli/skills-cli.format.ts` so exact matches still win, both case-insensitive and separator-normalized fallbacks require a unique match, and not-found output stays trimmed/sanitized; then keep the focused regression coverage in `src/cli/skills-cli.test.ts` and land after review. Track any separate agent-runtime activation bug independently if it still reproduces after the CLI lookup fix.
