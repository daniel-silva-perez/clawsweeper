## Summary

Keep open. Current main has process-level proxy routing and provider-request proxy overrides, but it still has no auth-profile-scoped transport field: auth profiles and resolved auth results cannot carry proxy metadata, strict config rejects profile-level proxy metadata, and runtime auth request overrides reject proxy/TLS settings.

## What This Changes

The issue requests an optional auth-profile-scoped `proxyUrl` field so different provider profiles can route model-provider requests through distinct proxies.

## Best Possible Solution

Define a first-class profile-scoped transport contract, document its security model, make proxy credentials secret-aware and redacted, then thread the selected profile's transport policy into model request construction while preserving existing provider-level request proxy behavior.
