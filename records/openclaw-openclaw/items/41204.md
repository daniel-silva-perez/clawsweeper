Keep open: the source branch is not safe to merge, and current main does not fully implement the stated hardening. ## What This Changes

The PR adds a session-store lock queue cap, subagent orphan cleanup during sweeping, skills installer package/formula validation, and three related Vitest files.

## Best Possible Solution

Do not merge this branch as-is. Preserve main's current subagent and installer behavior, then decide separately whether a narrow session-store queue/backpressure patch with deterministic tests should be requested or opened as a replacement.

## Reproduction Assessment

Yes. The regressions are reproducible by static diff inspection against current main: the PR removes main's archive guards/retry ordering, applies one shared installer validator across incompatible package syntaxes, and adds a long serial queue stress test.

## Solution Assessment

No. The current PR mixes three security-sensitive surfaces and reimplements two areas that main now handles more carefully; the maintainable path is a narrow, policy-approved queue-cap change if maintainers still want it.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Restore the archive guard before sweep cleanup:** `src/agents/subagent-registry.ts:784-791`
  - body: After the orphan branch, non-orphan entries without `archiveAtMs` or with a future `archiveAtMs` fall through into the standard cleanup path. That can delete active or not-yet-due subagent records and call `sessions.delete` prematurely.
  - confidence: 0.93
- **[P2] Keep archive records until session deletion succeeds:** `src/agents/subagent-registry.ts:728-745`
  - body: The due-archive branch deletes the run record and attachments before `sessions.delete`, then ignores delete failures. If transcript deletion fails, the registry loses the state needed to retry cleanup.
  - confidence: 0.9
- **[P2] Use installer-specific validators for package specs:** `src/agents/skills-install.ts:197-217`
  - body: The shared validator rejects valid node, Go, and uv installer syntax that current metadata accepts, especially uv extras/version pins such as `uvicorn[standard]==0.31.0`. Legitimate skills can fail installation at runtime.
  - confidence: 0.9
- **[P2] Bound the queue-limit stress test:** `src/config/sessions/store.lock-queue.test.ts:18-37`
  - body: This test queues 1100 lock tasks that each sleep for 100 ms behind a serial lock, so `Promise.allSettled` can take roughly 100 seconds. That is likely to time out or make the targeted suite impractically slow.
  - confidence: 0.86
