## Summary

Keep open. Current main already has stronger skills-install validation and subagent cleanup than parts of this PR, but it still does not implement the requested session-store lock queue cap, so the PR is not fully obsolete. The branch is also not safe to merge as-is because the subagent sweep changes introduce availability/data-retention regressions and the package validator would reject currently valid install specs.

## What This Changes

The PR adds a session-store lock queue cap, subagent orphan cleanup in the sweeper, skills-install package/formula validators, and three new test files.

## Best Possible Solution

Keep the current main behavior for skills installation and subagent cleanup. If maintainers still want resource-exhaustion hardening for the session-store lock queue, implement that as a narrow backpressure PR with explicit cap semantics and small deterministic lock-queue tests, without replacing the current per-installer validators or subagent archive retry logic.
