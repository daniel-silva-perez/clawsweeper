## Summary

Keep open. The PR is security-sensitive and not mergeable as-is: the proposed subagent sweep change can delete runs that are still active or not due for archive cleanup, and the shared installer validation regresses valid safe package specs that current main handles with installer-specific contracts. Current main already has stronger H2/H3 coverage than the PR, while the lock queue cap remains a narrow unimplemented hardening idea.

## What This Changes

The PR adds a per-session-store lock queue cap, extends subagent sweep cleanup for orphaned runs, adds shared package/formula validation in skill installation, and adds three related test files.

## Best Possible Solution

Keep the current main implementations for skill installer validation and subagent orphan/stale cleanup, and consider a separate focused session-store lock queue cap if maintainers agree on the concurrency limit and error behavior.

## Reproduction Assessment

Yes for the PR regressions: diff inspection shows the missing archive guard in `sweepSubagentRuns`, and the shared package-name validator rejects uv, Go, and npm specs that current main explicitly supports. The resource-exhaustion concern has a clear code-level path because the session-store lock queue is unbounded, but this read-only review did not run a stress reproduction.

## Solution Assessment

No. The PR mixes one still-plausible queue-limit hardening with two areas that current main already solves through more specific contracts, and the proposed diff introduces runtime regressions in those areas.

Full review comments:

- **[P1] Restore the archive guard before sweep cleanup:** `src/agents/subagent-registry.ts:784-785`
  - body: After the new orphan branch, any entry that is not recognized as an orphan falls through into the standard cleanup block even when it has no `archiveAtMs` or a future archive time. That lets the sweeper delete active or recently ended runs and call `sessions.delete` before the run is actually eligible for archive cleanup.
  - confidence: 0.94
- **[P2] Preserve the context-engine cleanup notification:** `src/agents/subagent-registry.ts:728-746`
  - body: The new early archive branch handles due `archiveAtMs` entries and then continues before the old cleanup path, but it omits `notifyContextEngineSubagentEnded`. Archived subagent runs can therefore be purged without telling the context engine that the child ended.
  - confidence: 0.82
- **[P2] Use installer-specific safe specs:** `src/agents/skills-install.ts:128`
  - body: The shared package-name validator rejects valid metadata that current main accepts through installer-specific validators, including uv requirement extras/version pins and Go or npm version syntax with characters outside this whitelist. This would make legitimate skill installers fail at runtime, so the hardening needs to preserve each installer contract and pass trimmed validated values to command builders.
  - confidence: 0.89
