## Summary

Keep open. Current main has not implemented the PR's session-store lock queue cap, so the branch is not fully obsolete, but the proposed patch is not safe to merge because it regresses subagent cleanup and skill installer compatibility on security-sensitive paths already handled more carefully on main.

## What This Changes

The PR adds a per-store session lock queue limit, subagent orphan cleanup during sweeping, shared package/formula validation for skill installers, and three related Vitest files.

## Best Possible Solution

Do not merge this branch as-is. Preserve current main's subagent cleanup and installer validation behavior, then decide separately whether to add a narrow, policy-approved session-store queue/backpressure cap with bounded tests.

## Reproduction Assessment

Yes. The regressions are reproducible by static diff inspection against current main: the PR removes main's archive guards/retry ordering, applies one shared validator across incompatible package syntaxes, and adds a long serial queue stress test.

## Solution Assessment

No. This branch mixes three security-sensitive surfaces and reimplements two areas that main now handles more carefully; the maintainable path is a narrow queue-cap change only if maintainers still want that policy.

## Review Findings

Overall confidence: 0.91

Full review comments:

- **[P1] Restore the archive guard before sweep cleanup:** `src/agents/subagent-registry.ts:784-791`
  - body: After the orphan branch, non-orphan entries without `archiveAtMs` or with a future `archiveAtMs` fall through into the standard cleanup path. That can delete active or not-yet-due subagent records and call `sessions.delete` prematurely.
  - confidence: 0.93

  - body: The due-archive branch deletes the run record and attachments before `sessions.delete`, then ignores delete failures. If transcript deletion fails, the registry loses the state needed to retry cleanup.
  - confidence: 0.9
- **[P2] Use installer-specific validators for package specs:** `src/agents/skills-install.ts:197-217`
  - body: The shared validator rejects valid node, Go, and uv installer syntax that current metadata accepts, especially uv extras/version pins such as `uvicorn[standard]==0.31.0`. Legitimate skills can fail installation at runtime.
  - confidence: 0.9
- **[P2] Bound the queue-limit stress test:** `src/config/sessions/store.lock-queue.test.ts:18-37`
  - body: This test queues 1100 lock tasks that each sleep for 100 ms behind a serial lock, so `Promise.allSettled` can take roughly 100 seconds. That is likely to time out or make the targeted suite impractically slow.
  - confidence: 0.86
