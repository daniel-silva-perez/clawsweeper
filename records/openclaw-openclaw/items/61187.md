## Summary

Keep this PR open. Current main only partially covers the reported behavior: Moonshot now has manifest/static catalog coverage, but Kimi still has only an auth-gated catalog path and no `augmentModelCatalog`, `buildStaticProvider`, or manifest `modelCatalog` rows for unauthenticated picker display. The PR discussion is useful: Greptile identified the same missing hook pattern, and the author addressed the spread-copy and test-coverage review comments on the PR branch.

## Best Possible Solution

Keep the PR open, but update it to current main’s catalog architecture. The best path is to expose Kimi `kimi-code` and `k2p5` through a provider-owned offline catalog path, preferably manifest/static catalog or a narrowly scoped `augmentModelCatalog` if that remains the intended picker surface. Leave Moonshot on the current manifest/buildStaticProvider path unless a focused picker regression still reproduces there, and add regression coverage for provider-filtered model picking without an API key.
