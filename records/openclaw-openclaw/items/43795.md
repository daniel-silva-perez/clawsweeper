## Summary

Keep open. Current main still has unguarded OpenAI Responses assistant-content handling that can call array methods or iterate provider `content` before proving it is an array, and the linked repair PRs were closed unmerged.

## What This Changes

The issue asks OpenClaw to stop the `500 v.content is not iterable` retry loop when OpenAI-compatible provider responses or replayed assistant turns carry non-array `content`.

## Best Possible Solution

Add a defensive normalization boundary for malformed OpenAI Responses assistant `message.content` before array methods or iteration, and align replay/session sanitization so string, object, null, and invalid content shapes cannot re-enter provider payloads or retry loops while preserving well-formed array behavior.
