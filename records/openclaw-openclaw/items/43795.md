## Summary

Keep open. Current main still has unguarded OpenAI Responses assistant-content paths that can call array methods or iterate `content` before proving it is an array, and the two linked repair PRs were closed unmerged.

## What This Changes

The issue reports a regression where malformed assistant `content` from a third-party OpenAI-compatible GLM provider causes 500 `v.content is not iterable` errors and retry loops.

## Best Possible Solution

Add one defensive normalization boundary for malformed OpenAI Responses assistant `message.content` before conversion, replay, and gateway iteration, preserving well-formed array behavior while safely handling string, object, null, and invalid shapes with focused regression tests.
