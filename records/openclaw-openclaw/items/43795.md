## Summary

Keep open. Current main still has the malformed assistant-content crash path for OpenAI Responses/WebSocket provider output, and the related fix PRs in the discussion were closed unmerged.

## Best Possible Solution

Keep this issue open and land a focused core fix in the OpenAI Responses/WebSocket ingress and replay boundaries: guard `buildAssistantMessageFromResponse` with `Array.isArray`, normalize malformed assistant replay content for string/object/null shapes, review the OpenResponses HTTP guard in light of schema validation, and add regression tests for malformed provider payloads. #43836 and #43864 are useful prior analysis but should not close this issue because they were not merged.
