## Summary

Keep this PR open. Current main still does not implement the PR's intended narrow suppression for Discord thread-bound ACP parent relays: progress, stall, and resumed relay events are still enqueued whenever `surfaceUpdates` is enabled, and the spawn path does not compute or pass a Discord/thread-specific suppression option. The linked bug #55807 remains the right product problem to fix, and this PR is still an implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Review, rebase, and either land this PR or reimplement the same narrow behavior on current main. The implementation should add an explicit ACP parent relay suppression seam for `:progress`, `:stall`, and `:resumed`, compute it only for Discord thread-bound parent sessions using both active thread context and thread-bound parent session keys, pass it through both relay construction sites, preserve relay logging plus `:start`, `:done`, `:error`, and `:timeout`, and add regression tests for both Discord thread detection paths.
