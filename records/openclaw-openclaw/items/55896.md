## Summary

Keep this PR open. Current main still does not implement the PR's intended Discord thread-bound ACP relay suppression: parent-stream relays enqueue `:progress`, `:stall`, and `:resumed` system events whenever surfaced updates are enabled, and the ACP spawn path does not pass any Discord/thread-specific suppression option. The linked bug and PR discussion describe a focused, plausible fix, but that fix has not landed on main.

## Best Possible Solution

Review, rebase, and either land this PR or reimplement its narrow suppression on current main. The implementation should add an explicit Discord thread-bound suppression seam in ACP parent relay wiring, cover both explicit `agentThreadId` and thread-bound parent-session-key cases, and leave non-Discord, non-thread, logging, and terminal lifecycle behavior unchanged.
