## Summary

Keep open. Current main still derives plain-HTTP Mattermost native slash callback URLs when `commands.callbackUrl` is omitted, still proceeds after only warning on suspicious callback URLs, and still authorizes callbacks with an account-level token set before trusting the caller-supplied slash trigger. The issue is also paired with open, unmerged closing-linked PR #65655, so the maintainer path is review/land that PR or an equivalent approved security fix.

## What This Changes

Request to harden Mattermost native slash-command callbacks by rejecting non-HTTPS derived callback URLs and binding each Mattermost command token to its registered slash trigger.

## Best Possible Solution

Keep this issue open and review/land #65655, or land an equivalent maintainer-approved Mattermost plugin fix. The fix should reject non-HTTPS resolved native slash callback URLs before registration, preserve token-to-trigger bindings through slash activation and HTTP dispatch, update Mattermost docs for the HTTPS callback requirement, and add regression tests for HTTPS enforcement plus cross-trigger token replay.
