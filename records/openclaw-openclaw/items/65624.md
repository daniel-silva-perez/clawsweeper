## Summary

Keep open. Current main still derives Mattermost native slash callback URLs as plain HTTP when `commands.callbackUrl` is omitted, registration only warns on suspicious loopback callbacks and continues, and callback authorization still stores a token set without binding tokens to the registered trigger. The issue also has open, unmerged closing-linked PR #65655, so the right path is maintainer review of that PR or an equivalent approved fix.

## Best Possible Solution

Keep this issue open and review/land #65655, or land an equivalent maintainer-approved Mattermost plugin fix. The fix should reject non-HTTPS resolved native slash callback URLs before registration, preserve token-to-trigger bindings through slash activation and HTTP dispatch, update Mattermost docs for the HTTPS callback requirement, and add regression tests for HTTPS enforcement plus cross-trigger token replay.
