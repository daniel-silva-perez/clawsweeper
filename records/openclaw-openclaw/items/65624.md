## Summary

Keep this issue open. Current main still derives plain-HTTP Mattermost slash callback URLs when `commands.callbackUrl` is omitted, still registers those callbacks without an HTTPS fail-closed gate, and still validates slash callbacks by token-set membership rather than token-to-trigger binding. The provided GitHub context also shows #65655 is open and unmerged with closing linkage for this issue, so that PR is the active implementation candidate rather than a reason to close early.

## Best Possible Solution

Keep this issue open and review #65655 or an equivalent maintainer-authored patch. The fix should live in the Mattermost plugin: reject non-HTTPS resolved native slash callback URLs before registration, preserve token-to-trigger bindings through slash activation and HTTP dispatch, and update Mattermost docs plus focused regression tests for both behaviors. After the PR merges and ships, close this issue as implemented with the merge SHA and release provenance.
