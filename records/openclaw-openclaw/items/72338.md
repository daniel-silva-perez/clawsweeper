## What This Changes

The issue asks OpenClaw to prevent Gateway CPU/event-loop starvation that stalls Telegram replies and causes loopback status or channel probes to time out.

## Best Possible Solution

Keep this as the broad Gateway CPU/event-loop starvation tracker until maintainers either tie the resolution to a shipped release with affected-host verification or split any remaining root causes into narrower canonical issues such as #73532, #73581, or #74630. The target state is a packaged Gateway that idles normally, answers loopback status/probe calls reliably, and completes Telegram/WebChat turns without plugin-runtime, subagent-registry, or stuck-session work starving the event loop.
