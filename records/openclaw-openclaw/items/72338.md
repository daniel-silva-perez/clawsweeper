## Summary

Keep open. Current main contains substantial Gateway CPU/starvation mitigations, including runtime-mirror hardening, subagent registry read caching, Gateway readiness/probe changes, and stuck-session recovery, but the report is recent, multi-host, and broader than any single proven shipped fix. ## What This Changes

The issue reports a Gateway high-CPU/event-loop starvation regression where Telegram replies stall and local loopback status probes time out on packaged OpenClaw releases around 2026.4.24-2026.4.27.

## Best Possible Solution

Keep this as the broad Gateway CPU/event-loop starvation tracker until maintainers verify current main/dev on affected hosts, decide which remaining cases belong in #73532, #73581, #74630, or #68944, and then close this issue only when a specific release or canonical follow-up covers the central symptoms: low idle CPU, responsive loopback probes, and timely Telegram/WebChat replies.
