## Summary

Keep this issue open. Current main still has the reported failure shape: clean-install bootstrapping seeds BOOTSTRAP.md and docs expect the first run to complete it, but the embedded runner strips raw BOOTSTRAP.md out of injected Project Context and relies on model/tool behavior to read it. The prior ClawSweeper comment identified the same path, and no current-main evidence shows a shipped fix or canonical replacement issue.

## Best Possible Solution

Keep this issue open. Reproduce the clean-install Control UI path on current main with qwen3.5 or another affected local model, then fix the first-run embedded bootstrap path by adding a safe fallback or clear diagnostic when model tool-calling/read access is unavailable or unreliable. Add regression coverage around bootstrap-pending routing, BOOTSTRAP.md stripping, and the fallback behavior.
