## Summary

Keep this issue open. Current main still has the reported failure shape: the embedded agent path strips `BOOTSTRAP.md` from Project Context, then the full bootstrap prompt depends on the model successfully using the `read` tool to fetch it. The report is fresh, concrete, on the latest release in the provided context, and there is no linked merged fix or canonical successor.

## Best Possible Solution

Keep this issue open. Reproduce the clean-install Control UI path with `ollama/qwen3.5:9b`, then patch the first-run embedded bootstrap path so a bootstrap-pending primary run can recover when `read` is unavailable or unreliable. A good fix would preserve the context-injection safety goal while adding fallback content, a targeted diagnostic, or a guarded bootstrap-only read path, plus regression coverage for context stripping and local-model/tool-disabled bootstrap behavior.
