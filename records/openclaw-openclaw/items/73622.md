## Summary

Keep this issue open. Current main still matches the reported failure shape: clean workspaces seed BOOTSTRAP.md, but the embedded runner removes BOOTSTRAP.md from injected Project Context and tells the model to read it through tools, with no deterministic fallback when the local model/tool path cannot complete that read.

## Best Possible Solution

Keep this issue open. Reproduce the clean-install Control UI path on current main with qwen3.5 or another affected local model, then fix the first-run embedded bootstrap path by adding a scoped fallback or diagnostic when BOOTSTRAP.md cannot be read through tools. Add regression coverage around bootstrap-pending routing, BOOTSTRAP.md stripping, and the fallback behavior.
