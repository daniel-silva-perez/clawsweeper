## Summary

Current main now implements the missing first-run bootstrap fallback. Clean workspaces still seed BOOTSTRAP.md, and embedded primary bootstrap turns now append BOOTSTRAP.md contents into the user prompt before stripping it from privileged system context. Runs without bootstrap file access now get limited bootstrap wording instead of silently losing the bootstrap prompt. The fix is commit 0dcab4e34760cac5d9cd02649d4b3a5c2f7e5712 on main; no release tag contains it yet.

## Best Possible Solution

Close this issue as implemented on current main, keep the 0dcab4e3 bootstrap fallback behavior, and ship it in the next OpenClaw release. If a live qwen/Ollama clean install still fails after that release, it should be tracked as a new, narrower provider/tool execution bug with fresh logs.
