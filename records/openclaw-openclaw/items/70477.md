## Summary

Keep open: current main still has the authenticated bootstrap-config behavior and lacks the PR's loopback bootstrap-token and chat-activity plumbing, while the proposed patch still contains blocking auth-boundary and stale terminal-event issues.

## What This Changes

This PR adds loopback Control UI bootstrap-token auth, UI token plumbing, chat activity/run-state indicators, related dashboard/channel auth updates, and targeted gateway/UI tests.

## Best Possible Solution

Land only after the bootstrap endpoint either remains normally authenticated or proves direct local provenance without trusting spoofable request headers, the pending-abort terminal path is changed to swallow stale terminal events, and the branch is rebased or split so bootstrap auth and chat activity can be reviewed independently from unrelated drift.

## Reproduction Assessment

Yes. The auth issue is reproducible by code inspection with a same-host reverse proxy that omits forwarded headers and lets an external caller spoof Host/Origin/Sec-Fetch headers; the pending-abort issue is reproducible through the unit-level state path already described in the review thread.

## Solution Assessment

No. The current PR direction is useful but not yet the narrowest maintainable solution because it mixes a security-sensitive token exposure with broad UI run-state changes and still leaves concrete review findings unresolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Bind bootstrap token bypass to server-trusted provenance:** `src/gateway/control-ui.ts:687-692`
  - body: This branch treats a request as local when the socket peer is loopback and no forwarding headers are present, then uses client-controlled Host/Origin/Sec-Fetch headers to skip normal auth and return `gatewayToken`. A same-host reverse proxy that omits forwarded headers makes external traffic satisfy that socket check, so unauthenticated callers can spoof the browser headers and receive the gateway token. Keep bootstrap config behind normal auth unless the server can prove the request came directly from the local Control UI.
  - confidence: 0.9
- **[P2] Swallow terminal events for pending-abort runs:** `ui/src/ui/controllers/chat.ts:443-444`
  - body: The pending-abort branch clears the marker but returns `payload.state`; `handleChatGatewayEvent` then treats the stale terminal as a real terminal event and can reset tool streams or reload history while a replacement run is active. Return `null` after clearing the marker so the old run does not disturb the new run.
  - confidence: 0.94

## Security Review

Status: needs_attention

Summary: The diff introduces a token-bearing unauthenticated bootstrap response whose loopback gate can be spoofed through a same-host reverse proxy that omits forwarded headers.

Concerns:

  - body: `allowUnauthenticatedLoopbackBootstrap` relies on loopback socket metadata plus spoofable browser headers before including `gatewayToken`; reverse-proxied external traffic can appear as a loopback socket when the proxy does not add forwarding headers.
