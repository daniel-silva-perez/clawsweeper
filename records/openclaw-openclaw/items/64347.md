## Summary

Keep open. Current main and the latest release still create native subagent sessions with fresh UUID-backed keys, keep resume behavior ACP-only, and expose no native reuse key, warm pool, or stable child-session selector in the tool, config, docs, or tests. Related merged PRs #64468 and #72331 fixed adjacent bootstrap-marker and ACP-field behavior, not this native warm-reuse contract.

## What This Changes

Add native warm-reuse, pooling, or stable session-key support for `sessions_spawn` so repeated delegations can avoid a fresh cold subagent session each time.

## Best Possible Solution

Define an explicit native subagent reuse contract, then implement it through the tool schema, config/types, child-session key selection, docs, and focused tests. The desired end state should keep isolated fresh sessions as the safe default while allowing deliberate warm reuse with clear history-sharing semantics.

## Reproduction Assessment

Yes. The source-level reproduction is the current native `runtime="subagent"` path: the tool ignores `resumeSessionId` for subagent runtime, passes no reuse argument, and `spawnSubagentDirect` constructs a new UUID-backed child session key on each call.

## Solution Assessment

Unclear. The issue's proposed options are plausible, but the best fix needs maintainer design around isolation, history sharing, concurrency locking, cleanup, and thread-bound session behavior before choosing opt-in reuse, automatic pooling, or main-session routing.
