## Summary

Keep this issue open. Current main still has no native warm-reuse path for `sessions_spawn`: native subagent spawning always creates a UUID-backed child session key, the tool path only passes `resumeSessionId` to ACP spawns and deliberately strips ACP-only fields for native spawns, and the config/docs/tests still describe isolated UUID sessions rather than a reuse key, pool, or native resume contract. Related merged PRs #64468 and #72331 addressed adjacent cache-marker and ACP-field behavior, not native subagent warm reuse.

## Best Possible Solution

Keep this issue open. The maintainer path is to design an explicit native subagent reuse contract around `src/agents/subagent-spawn.ts`, `src/agents/tools/sessions-spawn-tool.ts`, `src/config/zod-schema.agent-defaults.ts`, and `docs/tools/subagents.md`, choosing opt-in reuse versus automatic pooling, documenting isolation/history tradeoffs, handling concurrency and archive cleanup, and adding focused native `runtime="subagent"` regression tests. Treat #64468 and #72331 as related cache/runtime cleanup, not as the canonical resolution.

## Risks / Open Questions

- A native warm-reuse design could mix unrelated child histories unless reuse is explicit, scoped, and documented.
- Automatic pooling would need allocation and locking semantics so concurrent spawns do not reuse an active child session unsafely.
- Archive cleanup, thread-bound sessions, follow-up routing, and context preparation need a defined contract for reused native child sessions.
