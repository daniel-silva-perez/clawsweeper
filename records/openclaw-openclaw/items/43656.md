## Summary

Keep open. Current main still lacks cross-gateway session targeting, and this PR is the active implementation candidate for open #43605, but the patch still has blocking remote RPC/auth defects and a security-sensitive raw URL bypass that need maintainer review before merge.

## What This Changes

The branch adds a `gateway.peers` config map, cross-gateway `gateway`/`gatewayUrl`/`gatewayToken` parameters for `sessions_send` and `sessions_spawn`, a peer resolver, config schema/help/label/type coverage, and peer resolver tests.

## Best Possible Solution

Keep this PR open as source work, but require a rebase onto current main and a maintainer-approved gateway federation design that preserves explicit URL guardrails, defines peer credential/TLS policy, sends valid Gateway protocol payloads, and adds end-to-end coverage plus user-facing docs before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep raw gatewayUrl on the guarded path:** `src/agents/tools/gateway-peer.ts:66-67`
  - body: The helper accepts any explicit `gatewayUrl` and the new session tool paths pass it directly to `callGateway`, bypassing the current agent-tool allowlist that restricts overrides to loopback or `gateway.remote.url`. Keep raw URL overrides behind the existing guard or remove them from these tools while a peer-specific config path is reviewed.
  - confidence: 0.88
- **[P1] Include idempotencyKey in remote spawn requests:** `src/agents/tools/sessions-spawn-tool.ts:167-179`
  - body: The remote `sessions_spawn` branch calls the Gateway `agent` method without `idempotencyKey`, but `AgentParamsSchema` requires it. Cross-gateway spawn requests will be rejected as invalid params before the remote run starts.
  - confidence: 0.93
- **[P2] Require peer credentials before returning URL overrides:** `src/agents/tools/gateway-peer.ts:59-62`
  - body: Named peers can return `gatewayToken: undefined`, and the tests document tokenless peers as supported. `callGateway` requires explicit credentials for any explicit URL override, so those peers fail before the remote request is attempted unless the implementation uses a different trusted config-backed path or makes credentials required.
  - confidence: 0.87
- **[P2] Reject sessionKey and label together on remote sends:** `src/agents/tools/sessions-send-tool.ts:71-80`
  - body: The local `sessions_send` path rejects callers that provide both `sessionKey` and `label`, but the remote branch only checks that at least one exists and then silently prefers `sessionKey`. That can skip the requested remote label resolution and route to the wrong session.
  - confidence: 0.84
