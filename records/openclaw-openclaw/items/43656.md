  - confidence: 0.92

  - body: The helper returns a URL override with `gatewayToken: undefined` for peers without a token, and the tests advertise that as supported. `callGateway` treats any explicit URL as an override and requires explicit credentials, so tokenless peers fail before dialing the remote gateway.
  - confidence: 0.88

  - body: The local `sessions_send` path rejects callers that provide both `sessionKey` and `label`, but the remote branch only rejects the neither-provided case and then silently uses `sessionKey`. That makes cross-gateway behavior inconsistent and can skip the requested remote label resolution.
  - confidence: 0.84

  - body: Current main validates agent-tool `gatewayUrl` overrides through `validateGatewayUrlOverrideForAgentTools`, limiting them to loopback or the configured remote gateway.
