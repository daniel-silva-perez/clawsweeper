## Summary

Keep open. Current main has Anthropic-family transcript hygiene, but it only coalesces consecutive user turns; the current test suite explicitly preserves consecutive assistant turns for Anthropic, and the native Anthropic Messages transport returns converted payload messages without a final same-role coalescing pass. That leaves the reported custom proxy 400 path for assistant duplicates unresolved.

## What This Changes

The issue requests Anthropic-compatible provider normalization that coalesces consecutive same-role messages and avoids invalid trailing assistant-prefill payloads before sending custom `anthropic-messages` requests.

## Best Possible Solution

Add a focused Anthropic-compatible normalization step at the replay or transport boundary that coalesces adjacent user and assistant payloads while preserving tool-result adjacency, signed thinking blocks, and existing prefill behavior. Cover custom `anthropic-messages` providers with regression tests, update transcript hygiene docs, and add a user-facing changelog fix entry if the patch changes shipped provider behavior.

## Reproduction Assessment

Yes. A focused reproduction is to pass an Anthropic history with two consecutive assistant messages through `validateAnthropicTurns` or through the native Anthropic transport conversion and observe that current main preserves adjacent assistant payload roles.

## Solution Assessment

No. The current user-only Anthropic merge is not the best complete fix for strict Anthropic-compatible proxies; the safer direction is same-role coalescing for Anthropic-family outbound payloads, with any all-trailing-assistant stripping handled as a separate capability/prefill decision.
