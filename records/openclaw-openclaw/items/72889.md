## Summary

Keep open. Current main still preserves adjacent assistant turns in Anthropic replay validation and can return uncoalesced Anthropic Messages payload entries from the outbound transport, so the reported strict custom-proxy 400 path is still plausible and source-proven. The related MiniMax and prefill issues do not supersede this custom `anthropic-messages` same-role payload bug.

## What This Changes

The issue requests Anthropic-compatible message normalization so consecutive same-role entries are merged before API calls to strict custom proxies.

## Best Possible Solution

Add a focused Anthropic Messages normalization step at the replay or transport boundary that coalesces adjacent user and assistant payload entries without changing unrelated provider policies. Cover assistant/assistant and transport-generated user/user adjacency with regression tests, then update transcript hygiene docs and the changelog if shipped behavior changes.

## Reproduction Assessment

Yes. A deterministic source-level reproduction is to feed `validateAnthropicTurns` a `user, assistant, assistant` history and observe the current test expects the assistant turns to remain separate; the transport converter also has no final adjacent-role coalescing pass before returning payload messages.

## Solution Assessment

Yes. A narrow Anthropic-compatible replay or outbound-payload normalization is the best direction; a global all-provider merge or a direct bundled dependency patch would be broader and riskier than enforcing the strict Anthropic Messages wire contract at OpenClaw's boundary.
