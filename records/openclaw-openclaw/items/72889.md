## Summary

Keep open. Current main still normalizes only consecutive user turns for Anthropic-family replay and explicitly preserves adjacent assistant turns; the Anthropic Messages transport also returns converted payload messages without a final same-role coalescing pass. The related closed MiniMax, prefill, and older Anthropic reports do not provide a shipped current fix for the custom `anthropic-messages` assistant-duplicate path.

## What This Changes

The issue requests Anthropic-compatible message normalization so consecutive same-role messages are merged before OpenClaw sends payloads to strict custom `anthropic-messages` proxies.

## Best Possible Solution

Add a focused Anthropic-compatible normalization step at the replay or transport boundary that coalesces adjacent user and assistant payloads while preserving tool-result grouping, signed thinking blocks, and existing trailing-prefill policy. Cover custom `anthropic-messages` providers with regression tests and update the transcript hygiene docs plus changelog if shipped behavior changes.

## Reproduction Assessment

Yes. A focused reproduction is to pass a user, assistant, assistant history through `validateAnthropicTurns` or the Anthropic transport conversion; current main preserves adjacent assistant roles, and the test suite explicitly asserts that behavior.

## Solution Assessment

Yes, a narrow Anthropic-family replay or outbound-payload normalization is the best direction. A broad all-provider merge or a prefill policy change would be riskier than fixing the strict Anthropic-compatible payload contract directly.
