## Summary

Keep open. Current main still has an Anthropic-compatible payload path where adjacent same-role turns can survive: replay validation only coalesces consecutive user turns, the focused test still preserves adjacent assistant turns, and the Anthropic transport returns converted request messages without a final coalescing pass.

## What This Changes

The issue requests Anthropic-compatible `anthropic-messages` normalization that merges consecutive same-role messages and avoids invalid trailing assistant payloads before provider requests.

## Best Possible Solution

Add narrow Anthropic-compatible normalization at the replay or outbound transport boundary that coalesces adjacent user and assistant payload entries and strips invalid trailing assistant payloads without changing unrelated provider policies or patching `node_modules`. Preserve signed thinking/tool-use semantics with focused tests and update transcript hygiene docs plus the changelog if behavior changes.

## Reproduction Assessment

Yes. Feed `validateAnthropicTurns` a `user, assistant, assistant` history and current main preserves all three turns; separately, `convertAnthropicMessages` can emit adjacent user entries when a tool-result payload follows a user entry because it returns `params` without final coalescing.

## Solution Assessment

Yes. A focused Anthropic Messages replay/transport normalization is the narrow maintainable fix; a bundled dependency patch or global all-provider same-role merge would be broader and risk changing providers with different replay contracts.
