## Summary

Keep open. Current main still has an Anthropic-compatible payload-shape gap: OpenClaw normalizes consecutive user turns and strips some trailing assistant prefill, but it preserves adjacent assistant turns and the native Anthropic Messages transport returns payload messages without a final same-role coalescing pass.

## What This Changes

The issue requests Anthropic-compatible message normalization for custom `anthropic-messages` providers so adjacent same-role turns do not produce strict-alternation 400 responses from proxies.

## Best Possible Solution

Add a focused Anthropic-compatible outbound normalization step that coalesces adjacent same-role payloads for custom and native `anthropic-messages` paths while preserving tool-result grouping, signed thinking blocks, and existing prefill policy. Update the validator/transport tests and transcript hygiene docs, with a changelog fix entry if provider behavior changes.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: pass a history with two adjacent assistant messages through `validateAnthropicTurns` or inspect `convertAnthropicMessages`; the current test suite already asserts that Anthropic assistant duplicates remain unmerged.

## Solution Assessment

No for current main. The current user-only merge is not enough for strict Anthropic-compatible proxies; the safer repair is a narrow same-role coalescing pass at the Anthropic-family replay or transport boundary, with trailing-assistant stripping treated as a separate prefill policy concern.
