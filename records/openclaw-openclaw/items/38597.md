## Summary

Keep this issue open. Current main still uses whitespace-only Discord long-line splitting with a hard UTF-16 slice fallback, so CJK text can still split away from punctuation and even across supplementary code points. There is an open repair PR, #73960, but it is not merged yet.

## Best Possible Solution

Keep this issue open and use #73960, or a successor PR, as the active fix path. The implementation should stay narrowly scoped to the Discord plugin chunker: preserve whitespace priority, add guarded CJK terminal/closing punctuation fallback, add code-point-safe fallback splitting, and cover length mode, newline mode, mixed script, long-token fallback, preserveWhitespace/fence behavior, and surrogate-pair safety in focused Discord chunk tests.
