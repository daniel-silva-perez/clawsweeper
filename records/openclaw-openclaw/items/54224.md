## Summary

Keep open because current main covers Slack channel history and thread history through the existing `read` path, but the requested Slack message search action is still absent from main. The related open PR #65691 is the active implementation candidate for the remaining `search.messages` work, so this issue should wait on that review rather than be closed or duplicated.

## What This Changes

Request read-only Slack message retrieval actions for history, search, and thread reads that work with user-token read-only workflows without enabling Slack writes.

## Best Possible Solution

Review and either land or close #65691. The desired end state is a Slack-owned generic `search` message action backed by Slack `search.messages`, advertised only for usable user-token read credentials, wired through the existing message action contract and CLI, documented on the public Slack channel page, and covered by focused discovery, dispatch, runtime, formatting, and least-privilege tests. The existing `read` action can remain the channel history and thread history solution.
