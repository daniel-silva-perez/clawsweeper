## Summary

Keep this issue open. Current main already covers Slack channel history and thread reads through the generic `read` path, with read operations preferring a configured user token, but the requested Slack message search action is still absent. Open PR #65691 is the active implementation candidate for the remaining search work.

## Best Possible Solution

Keep this issue open and narrow the remaining work to Slack message search. Review and either land or close #65691; if accepted, the implementation should stay Slack-owned, wire the generic `search` message action to Slack `search.messages` with `search:read` user-token gating, advertise it only when usable credentials exist, update CLI and Slack docs, and add focused discovery, dispatch, runtime, and formatting tests. The existing `read`/`readMessages` path can remain the solution for channel history and thread history.
