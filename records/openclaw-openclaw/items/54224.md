## Summary

Keep open. Current main already covers Slack channel history and thread reads through the existing `read`/`readMessages` path with user-token-preferred read behavior, but it still does not implement or advertise Slack message search; the related open PR #65691 is the active implementation candidate for that remaining work.

## What This Changes

The issue requests read-only Slack message retrieval actions for channel history, message search, and thread history when Slack user-token read-only workflows are enabled.

## Best Possible Solution

Review and either land or close #65691. The desired end state is a Slack-owned generic `search` message action backed by Slack `search.messages`, advertised only for usable user-token read credentials, wired through the existing message action contract and CLI, documented in the Slack channel docs, and covered by focused discovery, dispatch, runtime, formatting, and least-privilege tests; the existing `read` action can remain the channel history and thread history solution.
