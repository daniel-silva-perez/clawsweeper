## Summary

Keep this issue open. Current main includes Telegram Business update names in allowed-update coverage, but it still lacks runtime `business_message` / `business_connection` handlers, business update dedupe/sequencing fields, outbound `business_connection_id` reply support, tests, and docs. Open PR #53629 uses closing syntax for this issue, so the right path is to review/land or replace that PR rather than close this tracker now.

## Best Possible Solution

Keep this issue open as the canonical Telegram Business support tracker. Review and land PR #53629, or an equivalent Telegram plugin fix, that registers `business_connection` and `business_message` handlers before the normal message handler, routes `business_message` through the existing inbound pipeline, extends dedupe/sequential key handling, tracks and persists `business_connection_id` for outbound replies where Telegram supports it, and adds focused tests plus Telegram docs.
