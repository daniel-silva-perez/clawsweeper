## Summary

Keep open. Current main at 43da08979035503401a21a06a55e66920411ad67 still has only Telegram Business update-name coverage, not the feature itself: there are no business_message/business_connection handlers, no business fields in Telegram update dedupe, and no business_connection_id handling on outbound Telegram sends. Open PR #53629 explicitly closes this issue and remains open/unmerged, so the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open as the canonical Telegram Business support tracker.
