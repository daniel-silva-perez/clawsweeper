Keep this issue open. Current main includes Telegram Business update names in allowed-update coverage, but the Telegram plugin still has no runtime handlers for `business_message` or `business_connection`, no business-aware dedupe/sequential key fields, no outbound `business_connection_id` mapping/injection, and no Telegram Business docs or focused tests. Open PR #53629 uses closing syntax for this issue, so the right path is to review, land, replace, or close that PR rather than closing this tracker now.

## Best Possible Solution

Keep this issue open as the canonical Telegram Business support tracker.
