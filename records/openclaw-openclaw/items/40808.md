## Summary

Keep this PR open. Current main still does not implement or document a `sessions_send_concurrent` tool, and this PR remains an unmerged implementation candidate for a plausible core agents/gateway feature. It needs maintainer review rather than cleanup closure because the intended batch API is absent on main and the PR carries nontrivial behavior, concurrency, and security-review surface.

## Best Possible Solution

Keep this PR open for maintainer review or ask for a smaller rebase onto current main. The best implementation path is to build the batch API on top of the existing `sessions_send` validation, visibility, A2A, and per-session nested-lane helpers; keep enqueue and reply timeouts separate; reject ambiguous target references; preserve sandbox and cross-agent policy checks per target; use the repo's `typebox` import style; and add focused tests for real multi-target concurrency, fire-and-forget A2A, per-target timeout/error isolation, and session-visibility enforcement.
