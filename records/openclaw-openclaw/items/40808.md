## Summary

Keep this PR open because current main still does not provide a `sessions_send_concurrent` tool, but the submitted implementation is not merge-ready against current sessions/gateway contracts. The useful earlier A2A review was partly addressed by later commits, but the branch still conflicts with current `sessions_send` validation, per-session nested-lane routing, reply-baseline handling, ACP child skip behavior, and the repo's current TypeBox dependency surface.

## What This Changes

The PR adds a core `sessions_send_concurrent` agent tool, A2A helper, docs, tests, tool registration/catalog entries, and gateway nested-lane concurrency changes for concurrent sends to multiple sessions.

## Best Possible Solution

Keep the feature candidate open, but require a rebase and a narrower implementation that reuses current `sessions_send` validation, visibility checks, inter-session provenance, per-session nested lanes, baseline-aware reply reading, ACP child skip semantics, and A2A helper behavior before adding the batch/progress API and docs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Import the repo's typebox package:** `src/agents/tools/sessions-send-concurrent-tool.ts:3`
  - body: Current main depends on `typebox` and imports schemas from that package. This new file imports `@sinclair/typebox`, which is not a current direct dependency, so the branch can fail after rebasing unless it uses the repo package or updates dependencies deliberately.
  - confidence: 0.88
- **[P2] Keep the shared nested lane serialized:** `src/gateway/server-lanes.ts:10`
  - body: Current main split cron nested concurrency away from the shared `CommandLane.Nested` lane and has a test that the shared nested lane remains at default concurrency. Widening `CommandLane.Nested` here can interleave unrelated nested flows and conflicts with the per-session nested lane model this PR should use instead.
  - confidence: 0.85
- **[P2] Reject ambiguous target references:** `src/agents/tools/sessions-send-concurrent-tool.ts:132-134`
  - body: A target can include both `sessionKey` and `label`; this path keeps `sessionKey` and silently ignores `label`/`agentId`. `sessions_send` rejects that combination because generated payloads can otherwise route to a different session than the caller intended.
  - confidence: 0.9
- **[P2] Route each target through its session lane:** `src/agents/tools/sessions-send-concurrent-tool.ts:364`
  - body: The batch send uses the shared `AGENT_LANE_NESTED` lane for every target. Current main scopes nested work with `resolveNestedAgentLaneForSession(...)` so different sessions can proceed independently while repeated sends to the same session remain ordered; this branch loses that behavior.
  - confidence: 0.89
- **[P2] Read the reply produced by this run:** `src/agents/tools/sessions-send-concurrent-tool.ts:520-528`
  - body: After `agent.wait`, the tool reads the latest assistant message from session history without a pre-run baseline or run correlation. If the session already had a reply, or another concurrent send finishes nearby, this can return and announce stale or wrong text instead of this target run's reply.
  - confidence: 0.87
- **[P2] Mirror the ACP child A2A skip:** `src/agents/tools/sessions-send-concurrent-tool.ts:404-416`
  - body: Current `sessions_send` skips ping-pong/announce when the requester is the parent of its own background ACP child, avoiding a bounded but user-visible parent/child loop. The new tool starts the concurrent A2A flow unconditionally, so it needs the same guard before launching follow-up delivery.
  - confidence: 0.82
