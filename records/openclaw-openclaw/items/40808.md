## Summary

Keep open: current main does not provide the proposed batch-send tool, but this PR should not merge as-is because the latest diff still diverges from current `sessions_send` lane, selector, reply-correlation, and ACP child-session contracts.

## What This Changes

The PR adds a `sessions_send_concurrent` agent tool with docs, tests, tool registration, concurrent batch sends to 1-20 sessions, A2A follow-up handling, and gateway nested-lane concurrency changes.

## Best Possible Solution

Build the batch surface as a thin orchestrator over shared `sessions_send` helpers, preserving target validation, session-scoped lanes, reply correlation, ACP child-session guards, and A2A semantics before adding focused docs and regression tests.

## Reproduction Assessment

Yes for the review findings: the latest PR diff can be compared directly against current main’s `sessions_send`, A2A, lane, dependency, and regression-test contracts. Live runtime reproduction was not attempted because the task is read-only.

## Solution Assessment

No. The branch adds a useful capability, but forking the sensitive `sessions_send` path and changing the shared nested lane is not the narrowest maintainable solution; extracting or reusing current helpers is safer.

## Review Findings

Overall confidence: 0.9

Full review comments:

- **[P2] Use the repo's typebox package:** `src/agents/tools/sessions-send-concurrent-tool.ts:3`
  - body: This new file imports `Type` from `@sinclair/typebox`, but current main declares and imports the direct `typebox` package. A clean install/build can fail to resolve this module unless the import follows the repo dependency.
  - confidence: 0.9
- **[P2] Reject ambiguous target selectors:** `src/agents/tools/sessions-send-concurrent-tool.ts:132-133`
  - body: A target can provide both `sessionKey` and `label`; this parser keeps both and later silently prefers `sessionKey`. `sessions_send` rejects that combination so generated payloads do not route to the wrong session without an error.
  - confidence: 0.9
- **[P2] Keep the shared nested lane serialized:** `src/gateway/server-lanes.ts:10`
  - body: Raising `CommandLane.Nested` concurrency conflicts with current main's per-session nested lane design and its regression test that the shared bare lane stays at default concurrency. The batch path should use session-scoped lanes instead of widening the shared lane.
  - confidence: 0.88
- **[P2] Use session-scoped lanes for target sends:** `src/agents/tools/sessions-send-concurrent-tool.ts:365`
  - body: The primary target run is enqueued on the bare nested lane. Current `sessions_send` uses `resolveNestedAgentLaneForSession(resolvedKey)` so different target sessions can proceed independently while repeated sends to the same session remain ordered.
  - confidence: 0.88
- **[P2] Use session-scoped lanes for A2A steps:** `src/agents/tools/sessions-send-concurrent-a2a.ts:177`
  - body: The concurrent A2A helper passes `AGENT_LANE_NESTED` to `runAgentStep`, overriding that helper's current session-scoped default. Ping-pong and announce work should run on the active session's nested lane.
  - confidence: 0.86
- **[P2] Correlate replies to the waited run:** `src/agents/tools/sessions-send-concurrent-tool.ts:521-529`
  - body: After `agent.wait`, the code reads the latest assistant message from session history without a baseline or run correlation. If the session already had a reply or another concurrent turn finishes nearby, the tool and A2A announce can use unrelated text.
  - confidence: 0.84
- **[P2] Preserve the ACP child A2A guard:** `src/agents/tools/sessions-send-concurrent-tool.ts:405-417`
  - body: Current `sessions_send` skips A2A follow-up when the requester owns a background ACP child session. This tool starts follow-up unconditionally, which can re-enter the parent/child completion path and produce a visible loop.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: Security review needs attention because the PR forks session-boundary behavior and omits current guards that protect inter-session and parent-owned ACP flows.

Concerns:

- **[medium] Preserve the owned ACP child boundary:** `src/agents/tools/sessions-send-concurrent-tool.ts:405`
  - body: The concurrent tool does not mirror current `sessions_send` logic that skips A2A when the requester owns a background ACP child. That can wake the parent and feed parent output back into the child across a boundary current main intentionally avoids.
  - confidence: 0.8
- **[medium] Avoid announcing an unrelated session reply:** `src/agents/tools/sessions-send-concurrent-tool.ts:521`
  - body: The wait path reads the latest assistant history item without a baseline or run correlation before starting A2A. A stale or nearby concurrent reply from the same target session can be treated as this run's output and announced externally.
  - confidence: 0.76
