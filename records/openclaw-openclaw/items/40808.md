## Summary

Keep this PR open, but the proposed patch is not merge-ready. Current main still does not provide a `sessions_send_concurrent` tool, so the PR is not obsolete, but the branch conflicts with the current `sessions_send` scheduling, target-validation, reply-correlation, and parent-owned ACP child A2A guard contracts.

## What This Changes

This PR adds a `sessions_send_concurrent` agent tool with docs/tests/catalog registration, concurrent batch sends, A2A follow-up handling, and gateway nested-lane concurrency changes.

## Best Possible Solution

Rework the feature as a batch wrapper around the current `sessions_send` validation, run submission, run-wait/reply correlation, A2A flow, and per-session lane helpers; keep the shared nested lane serialized, preserve the owned ACP child skip, and add focused tests for ambiguous targets, reply correlation, ACP child behavior, and concurrent per-session scheduling.

## Reproduction Assessment

Yes for the review findings: the PR diff can be checked against current main's `sessions_send` contracts for ambiguous target rejection, per-session lane routing, reply correlation, and the ACP child A2A skip. I did not run tests because this was a read-only review.

## Solution Assessment

No. The branch adds useful batch-send surface area, but the implementation forks core `sessions_send` behavior instead of composing the existing validated path, which creates correctness and security regressions.

Full review comments:

- **[P2] Import the repo's typebox package:** `src/agents/tools/sessions-send-concurrent-tool.ts:3`
  - body: Current main imports schemas from the direct `typebox` dependency. This new tool imports `@sinclair/typebox` without adding that package as a direct dependency, so a clean install can fail to resolve the module.
  - confidence: 0.9
- **[P2] Keep the shared nested lane serialized:** `src/gateway/server-lanes.ts:10`
  - body: Current main has a regression test that the shared `CommandLane.Nested` lane stays at default concurrency while per-session nested lanes provide cross-session parallelism. Raising the shared lane here conflicts with that scheduler contract.
  - confidence: 0.88
- **[P2] Reject ambiguous target references:** `src/agents/tools/sessions-send-concurrent-tool.ts:121-122`
  - body: A target can provide both `sessionKey` and `label`; this implementation resolves the `sessionKey` path and ignores the label. `sessions_send` rejects that combination to avoid silently routing generated payloads to a different session than intended.
  - confidence: 0.9
- **[P2] Route target sends through session lanes:** `src/agents/tools/sessions-send-concurrent-tool.ts:338`
  - body: The primary target run is enqueued on the shared nested lane. Current main scopes nested agent work with `resolveNestedAgentLaneForSession(...)`, which lets different sessions run independently while preserving order for repeated sends to the same session.
  - confidence: 0.87
- **[P2] Route A2A steps through session lanes:** `src/agents/tools/sessions-send-concurrent-a2a.ts:158`
  - body: The concurrent A2A helper also runs ping-pong and announce steps on the shared nested lane. These follow-up turns should use the current session-scoped lane helper for the active session.
  - confidence: 0.84
- **[P2] Read the reply produced by this run:** `src/agents/tools/sessions-send-concurrent-tool.ts:483-490`
  - body: After `agent.wait`, the tool reads the latest assistant message from session history without a baseline or run correlation. If the session already had a reply or another send completes nearby, the result and A2A flow can use stale or unrelated text.
  - confidence: 0.84
- **[P2] Mirror the owned ACP child A2A skip:** `src/agents/tools/sessions-send-concurrent-tool.ts:373-384`
  - body: Current `sessions_send` skips ping-pong/announce when the requester owns its background ACP child because that child already reports completion through the internal completion path. The concurrent tool starts A2A follow-up unconditionally, which can re-enter that parent/child path.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: Security review needs attention because the new batch send can bypass the current owned ACP child A2A skip and re-enter a guarded parent/child session boundary.

Concerns:

- **[medium] Preserve the owned ACP child A2A guard:** `src/agents/tools/sessions-send-concurrent-tool.ts:373`
  - body: The PR starts A2A follow-up for accepted targets without mirroring the current `sessions_send` guard for parent-owned background ACP child sessions. That path can wake the parent and feed a parent response back into the child, crossing the session-control boundary current main intentionally avoids.
