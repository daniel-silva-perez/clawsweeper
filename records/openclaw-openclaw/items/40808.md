## Summary

Keep open. Current main still does not provide the new batch-send tool, but this PR is not merge-ready: the gateway lane portion is superseded by the merged per-session nested-lane work in #67785, and the new tool still forks current `sessions_send` behavior in ways that break routing, reply correlation, dependency resolution, and the parent-owned ACP child guard.

## What This Changes

The PR adds a `sessions_send_concurrent` agent tool with docs/tests/catalog registration, concurrent batch sends, A2A follow-up handling, and gateway nested-lane concurrency changes.

## Best Possible Solution

Rework the feature as a batch wrapper around current `sessions_send` validation, run submission, run-wait/reply correlation, A2A flow, and per-session lane helpers. Keep the shared nested lane serialized, preserve the owned ACP child skip, use the repo's `typebox` dependency, and add focused coverage for ambiguous targets, reply correlation, ACP child behavior, and concurrent per-session scheduling.

## Reproduction Assessment

Yes for the review findings: the latest PR diff can be checked directly against current main's `sessions_send` contracts, lane tests, dependency list, and ACP guard. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No. The proposed branch adds useful batch-send surface area, but it forks current `sessions_send` behavior instead of composing the maintained path, which creates correctness and security regressions.

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Import the repo's typebox package:** `src/agents/tools/sessions-send-concurrent-tool.ts:3`
  - body: Current main imports `Type` from the direct `typebox` dependency. This new tool imports `@sinclair/typebox` without adding that package to package metadata, so a clean install can fail to resolve the module.
  - confidence: 0.9
- **[P2] Keep the shared nested lane serialized:** `src/gateway/server-lanes.ts:10`
  - body: Current main scopes nested work with per-session lanes and has a regression test that the shared `CommandLane.Nested` lane stays at default concurrency. Raising the shared lane here weakens that scheduler invariant.
  - confidence: 0.88
- **[P2] Reject targets that provide both sessionKey and label:** `src/agents/tools/sessions-send-concurrent-tool.ts:132-133`
  - body: A target can provide both `sessionKey` and `label`; this code keeps both and later ignores the label when `sessionKey` is present. `sessions_send` rejects that ambiguous combination to avoid silently routing generated payloads to the wrong session.
  - confidence: 0.9
- **[P2] Route target sends through session lanes:** `src/agents/tools/sessions-send-concurrent-tool.ts:364`
  - body: The primary target run is enqueued on the shared nested lane. Current main uses `resolveNestedAgentLaneForSession(...)` so different target sessions run independently while repeated sends to the same session remain ordered.
  - confidence: 0.87
- **[P2] Route A2A steps through session lanes:** `src/agents/tools/sessions-send-concurrent-a2a.ts:176`
  - body: The concurrent A2A helper also forces ping-pong and announce work onto the shared nested lane. Those follow-up turns should use the current session-scoped lane helper for the active session.
  - confidence: 0.84
- **[P2] Read the reply produced by this run:** `src/agents/tools/sessions-send-concurrent-tool.ts:520-528`
  - body: After `agent.wait`, the tool reads the latest assistant message from session history without a baseline or run correlation. If the session already had a reply or another send completes nearby, the result and A2A flow can use stale or unrelated text.
  - confidence: 0.84
- **[P2] Preserve the owned ACP child A2A guard:** `src/agents/tools/sessions-send-concurrent-tool.ts:404`
  - body: Current `sessions_send` skips ping-pong/announce when the requester owns its background ACP child because that child already reports completion through the internal completion path. The concurrent tool starts A2A follow-up unconditionally, which can re-enter that parent/child path.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: Security review needs attention because the new batch send can bypass the current owned ACP child A2A skip and re-enter a guarded parent/child session boundary.
