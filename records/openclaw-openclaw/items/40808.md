## Summary

Keep this PR open: current main does not have a `sessions_send_concurrent` tool, but this branch is not ready to merge because it diverges from the current `sessions_send` contracts for schema imports, nested lane scheduling, ambiguous target validation, reply correlation, and owned ACP child A2A handling.

## What This Changes

This PR adds a `sessions_send_concurrent` agent tool with docs/tests/catalog registration, concurrent batch sending, A2A follow-up support, and gateway nested-lane concurrency changes.

## Best Possible Solution

Rework the feature as a batch wrapper around the current `sessions_send` validation, run submission, run-wait/reply correlation, A2A flow, and per-session lane helpers; keep the shared nested lane serialized, preserve the owned ACP child skip, then update docs and focused tests for ambiguous targets, reply correlation, ACP child behavior, and concurrent per-session scheduling.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Import the repo's typebox package:** `src/agents/tools/sessions-send-concurrent-tool.ts:3`
  - body: Current main imports schemas from the direct `typebox` dependency. This new tool imports `@sinclair/typebox` without adding that package as a direct dependency, so a clean install can fail to resolve the module. Use the repo's existing `typebox` import contract or make an explicit dependency change with maintainer approval.
  - confidence: 0.9
- **[P2] Keep the shared nested lane serialized:** `src/gateway/server-lanes.ts:10`
  - body: Current main has a regression test that the shared `CommandLane.Nested` lane stays at default concurrency, while per-session nested lanes provide cross-session parallelism. Raising the shared lane here can interleave unrelated nested flows and conflicts with the current scheduler contract.
  - confidence: 0.88
- **[P2] Reject ambiguous target references:** `src/agents/tools/sessions-send-concurrent-tool.ts:121-122`
  - body: A target can provide both `sessionKey` and `label`, and this implementation records both but resolves the `sessionKey` path while ignoring the label. `sessions_send` rejects that combination to avoid generated payloads silently routing to a different session than intended, so the batch tool should enforce the same validation before sending.
  - confidence: 0.9
- **[P2] Route target sends through session lanes:** `src/agents/tools/sessions-send-concurrent-tool.ts:338`
  - body: The primary target run is enqueued on the shared `AGENT_LANE_NESTED` lane. Current main scopes nested agent work with `resolveNestedAgentLaneForSession(...)`, which lets different sessions run independently while preserving order for repeated sends to the same session. This branch loses that ordering and concurrency contract.
  - confidence: 0.87
- **[P2] Route A2A steps through session lanes:** `src/agents/tools/sessions-send-concurrent-a2a.ts:158`
  - body: The concurrent A2A helper also runs ping-pong and announce steps on the shared nested lane. These follow-up turns should use the current session-scoped lane helper for the active session, otherwise concurrent batches can interleave turns within the same target/requester sessions.
  - confidence: 0.84
- **[P2] Read the reply produced by this run:** `src/agents/tools/sessions-send-concurrent-tool.ts:483-490`
  - body: After `agent.wait`, the tool reads the latest assistant message from session history without a baseline or run correlation. If the session already had a reply or another send completes nearby, the returned result and A2A flow can use stale or unrelated text instead of the reply from this target run.
  - confidence: 0.84
- **[P2] Mirror the owned ACP child A2A skip:** `src/agents/tools/sessions-send-concurrent-tool.ts:373-384`
  - body: Current `sessions_send` skips ping-pong/announce when the requester owns its background ACP child, because that child already reports completion through the internal completion path. The concurrent tool starts A2A follow-up unconditionally, which can re-enter the parent/child path that current main intentionally avoids.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: Security review needs attention because the new batch send can bypass the current owned ACP child A2A skip and re-enter a guarded parent/child session boundary.

Concerns:

- **[medium] Preserve the owned ACP child A2A guard:** `src/agents/tools/sessions-send-concurrent-tool.ts:373`
  - body: The PR starts A2A follow-up for accepted targets without mirroring the current `sessions_send` guard for parent-owned background ACP child sessions. That path can wake the parent and feed a parent response back into the child, crossing the session-control boundary that current main intentionally avoids.
  - confidence: 0.78
