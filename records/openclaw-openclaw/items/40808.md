## Summary

Keep this PR open: current main still lacks the requested batch/concurrent session-send tool, but the branch is not merge-ready against current main because it diverges from current session-send validation, reply tracking, lane isolation, dependency style, and A2A edge-case handling.

## What This Changes

The PR adds a `sessions_send_concurrent` core agent tool with docs, tests, tool registration/catalog entries, an A2A helper, and gateway nested-lane concurrency changes for concurrent cross-session sends.

## Best Possible Solution

Keep the feature candidate open, but require a rebase and a narrower implementation that composes the current `sessions_send` validation, visibility checks, per-session nested lanes, baseline-aware reply reading, and A2A helper semantics before adding a batch/progress surface and docs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Import the repo's typebox package:** `src/agents/tools/sessions-send-concurrent-tool.ts:3`
  - body: Current main depends on `typebox` and the codebase imports schemas from that package. This new file imports `@sinclair/typebox`, which is not a current main dependency, so the branch will fail after rebasing unless it uses the repo package or updates dependencies deliberately.
  - confidence: 0.86
- **[P2] Reject ambiguous target references:** `src/agents/tools/sessions-send-concurrent-tool.ts:132-134`
  - body: A target can include both `sessionKey` and `label`; this path keeps `sessionKey` and silently ignores `label`/`agentId`. `sessions_send` rejects that combination because generated payloads can otherwise route to a different session than the caller intended.
  - confidence: 0.9
- **[P2] Route each target through its session lane:** `src/agents/tools/sessions-send-concurrent-tool.ts:364`
  - body: The batch send uses the shared `AGENT_LANE_NESTED` lane for every target. Current main scopes nested work with `resolveNestedAgentLaneForSession(...)` so different sessions can proceed independently while repeated sends to the same session remain ordered; this branch's shared-lane widening loses that behavior.
  - confidence: 0.88
- **[P2] Read the reply produced by this run:** `src/agents/tools/sessions-send-concurrent-tool.ts:520-528`
  - body: After `agent.wait`, the tool reads the latest assistant message from session history without a pre-run baseline or run correlation. If the session already had a reply, or another concurrent send finishes nearby, this can return and announce stale or wrong text instead of this target run's reply.
  - confidence: 0.87
- **[P2] Mirror the ACP child A2A skip:** `src/agents/tools/sessions-send-concurrent-tool.ts:404-416`
  - body: Current `sessions_send` skips the A2A ping-pong/announce flow when the requester is the parent of its own background ACP child, avoiding a bounded but user-visible parent/child loop. The new tool starts the concurrent A2A flow unconditionally, so it needs the same guard before launching follow-up delivery.
  - confidence: 0.8
