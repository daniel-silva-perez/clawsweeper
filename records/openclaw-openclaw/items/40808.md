## Summary

Keep open: current main still does not implement a `sessions_send_concurrent` tool, so the PR is not obsolete. The branch is not mergeable as-is because it is stale against current `sessions_send` and gateway lane behavior and introduces concrete functional regressions in dependency imports, nested-lane routing, ambiguous target validation, reply correlation, and ACP child A2A handling.

## What This Changes

This PR adds a core `sessions_send_concurrent` agent tool, A2A helper, docs, tests, tool registration/catalog entries, and gateway nested-lane concurrency changes for concurrent sends to multiple sessions.

## Best Possible Solution

Keep the feature candidate open, but require a rebase and a narrower implementation that reuses current `sessions_send` validation, dependency imports, per-session nested lanes, baseline-aware reply reading, ACP child skip semantics, and A2A helper behavior before adding the batch/progress API and docs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Import the repo's typebox package:** `src/agents/tools/sessions-send-concurrent-tool.ts:3`
  - body: Current main depends on `typebox` and imports schemas from that package. This new file imports `@sinclair/typebox`, which is not a current direct dependency, so the branch can fail to resolve unless it uses the repo package or deliberately updates dependencies.
  - confidence: 0.9
- **[P2] Keep the shared nested lane serialized:** `src/gateway/server-lanes.ts:10`
  - body: Current main split cron nested concurrency onto `CommandLane.CronNested` and has a test that the shared `CommandLane.Nested` lane stays at default concurrency. Widening `Nested` here can interleave unrelated nested flows and conflicts with the current per-session nested-lane model.
  - confidence: 0.88
- **[P2] Reject ambiguous target references:** `src/agents/tools/sessions-send-concurrent-tool.ts:131-134`
  - body: A target can include both `sessionKey` and `label`; this path keeps `sessionKey` and silently ignores `label`/`agentId`. `sessions_send` rejects that combination because generated payloads can otherwise route to a different session than the caller intended.
  - confidence: 0.9
- **[P2] Route each target through its session lane:** `src/agents/tools/sessions-send-concurrent-tool.ts:364`
  - body: The batch send uses the shared `AGENT_LANE_NESTED` lane for every target. Current main scopes nested work with `resolveNestedAgentLaneForSession(...)` so different sessions can proceed independently while repeated sends to the same session remain ordered; this branch loses that behavior.
  - confidence: 0.88
- **[P2] Read the reply produced by this run:** `src/agents/tools/sessions-send-concurrent-tool.ts:520-528`
  - body: After `agent.wait`, the tool reads the latest assistant message from session history without a pre-run baseline or run correlation. If the session already had a reply, or another send finishes nearby, this can return and announce stale or wrong text instead of this target run's reply.
  - confidence: 0.86
- **[P2] Mirror the ACP child A2A skip:** `src/agents/tools/sessions-send-concurrent-tool.ts:404-416`
  - body: Current `sessions_send` skips ping-pong/announce when the requester is the parent of its own background ACP child. The new tool starts the concurrent A2A flow unconditionally, so parent-owned ACP child sends can re-enter the user-visible parent/child loop that current main avoids.
  - confidence: 0.82
