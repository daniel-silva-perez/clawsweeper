## Summary

Keep this PR open. Current main still does not implement the PR’s intended structured Slack file ID propagation from `SlackMediaResult` into agent message context; Slack file IDs are only present in placeholder/raw text, while Slack `download-file` still requires a caller-supplied `fileId`. The PR remains a plausible implementation candidate, but it needs rebase onto the current file layout, focused tests, and maintainer review of the shared context API shape.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, adapt the type change to `extensions/slack/src/monitor/media-types.ts`, populate `fileId` from Slack `file.id` in `resolveSlackMedia()`, wire accepted fields through `prepareSlackMessage()` and `MsgContext`, add focused Slack media/context tests, and coordinate the shared context/API shape with #45626 before merging.
