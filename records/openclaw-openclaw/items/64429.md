## Summary

Keep this PR open. Current main still does not implement the PR's intended structured Slack file ID propagation from `SlackMediaResult` into agent message context; Slack file IDs remain available only inside placeholder text and the Slack `download-file` action still expects a caller-supplied `fileId`. The PR is a plausible implementation candidate, but it changes the shared context contract and needs maintainer review plus focused tests on the current file layout.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, add the `SlackMediaResult.fileId` field in `extensions/slack/src/monitor/media-types.ts`, wire it through `resolveSlackMedia()` and `prepareSlackMessage()`, add focused Slack media/context tests, and coordinate the context contract with the related message-tool schema work in #45626 before merging.
