## Summary

Keep this PR open. Current main still does not implement the PR's intended behavior of retaining self-declared operator scopes for non-local, device-less token/password WebSocket clients, and the related open issue remains unresolved. The PR is still a plausible implementation candidate, but it is stale against the newer scope-clearing path and needs maintainer review because this is an authorization boundary.

## Best Possible Solution

Keep this PR open for maintainer review alongside #51396. The best path is to decide the supported policy for non-local shared-secret WebSocket operator clients, then adapt the PR to current main by updating both the missing-device decision and the scope-clearing path, with token and password regression coverage plus a remote backend `chat.send` or write-RPC proof. If maintainers decide remote device-less shared-secret clients must not retain scopes, close this PR later with that explicit rationale and document the supported device identity, device token, trusted-proxy, or HTTP trusted-operator alternative.
