## Summary

Keep open. Current main still does not implement the intended behavior of retaining self-declared operator scopes for non-local, device-less token/password WebSocket clients. The PR remains a plausible implementation candidate for the linked gateway regression, but it is stale against the current scope-clearing path and changes an authorization boundary that needs maintainer review.

## Best Possible Solution

Keep this PR open for maintainer review alongside #51396. The best path is to decide the supported policy for non-local shared-secret WebSocket operator clients, then adapt the PR to current main by updating both the missing-device decision and the scope-clearing path, with token and password regression coverage plus a remote backend `chat.send` or write-RPC proof. If maintainers decide remote device-less shared-secret clients must not retain scopes, close the PR later with that explicit rationale and document the supported device identity, device token, trusted-proxy, or HTTP trusted-operator alternative.
