## Summary

Keep open. Current main still does not implement this PR's intended token/password path for non-local device-less WebSocket operator clients: the missing-device decision is still keyed to sharedAuthOk, and a separate helper explicitly clears token/password scopes outside narrow preservation cases. The linked #51396 regression remains a gateway auth policy question, so this PR needs maintainer review or adaptation rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review alongside #51396 and the related open #57889 attempt. The best path is to decide the supported policy for non-local shared-secret WebSocket operator clients, then either rebase/adapt this PR against current main with token and password regression coverage plus a remote backend chat.send/write-RPC proof, or explicitly close it later with a maintainer rationale if remote clients must use device identity, device tokens, trusted-proxy auth, or the documented HTTP trusted-operator surfaces instead.
