## Summary

Keep open. Current main still does not implement this PR's intended token/password path for non-local device-less WebSocket operator clients: the missing-device decision still depends on sharedAuthOk, and the current scope-clearing helper still clears token/password scopes outside narrow explicit preservation cases. The PR remains tied to the open #51396 regression and touches a security-sensitive gateway auth boundary, so it needs maintainer review or adaptation rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review alongside #51396. The best path is to decide the supported policy for non-local shared-secret WebSocket operator clients, then either rebase/adapt this PR against current main with token and password regression coverage plus a remote backend chat.send/write-RPC proof, or explicitly close it later with a maintainer rationale if remote clients must use device identity/device tokens instead.
