## Summary

Keep open. Current main still does not implement this PR's intended non-local token/password device-less operator scope retention, and current tests encode scope clearing for generic shared-token/password operator connects. The PR remains an implementation candidate for #51396, but it needs maintainer review and adaptation against the newer scope-clearing path.

## Best Possible Solution

Keep this PR open for maintainer review alongside #51396 and the related #57889 attempt. The best path is to decide the supported policy for non-local shared-secret WebSocket operator clients, then either adapt this PR to current main by updating both the missing-device decision and scope-clearing path with token and password regression coverage plus a remote backend chat.send/write-RPC proof, or close it later with an explicit maintainer rationale if remote clients must use device identity, device tokens, trusted-proxy auth, or documented HTTP trusted-operator surfaces.
