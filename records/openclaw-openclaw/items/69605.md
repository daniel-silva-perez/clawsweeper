## Summary

Keep this issue open. Current main at f2405c830bd925bd24f91b6bac5d417c6d36e5ba still has the reported Control UI behavior: code-block copy buttons store payloads in `data-code`, the delegated chat handler copies `dataset.code`, assistant bubbles still render message-level copy/expand actions without adding the existing `has-copy` spacing class, and there is no rendered-code resolver or focused regression test. The issue is also paired with open, unmerged fix PRs #69634 and #73633 that use closing syntax, so it should remain open until maintainers review and land or close the canonical fix.

## Best Possible Solution

Keep this issue open and review the linked fix PRs. The best product fix is one canonical Control UI patch that prevents message-level action overlap and makes the code-block Copy handler copy only the rendered block content, with focused regression coverage for assistant bubbles with actions, multiple fenced blocks, special characters, and code-block click targets.
