## Summary

Keep this issue open. Current main at 39f810911c382f92ea80b2fb406fdf7dbd405721 still renders code-block Copy buttons with a `data-code` payload, the delegated Control UI chat handler copies `dataset.code` rather than rendered `pre > code` text, and assistant bubbles still render message-level copy/expand actions without applying the existing `has-copy` spacing class. The issue is also paired with open, unmerged fix PRs #69634 and #73633 that use closing syntax, so it should remain open until maintainers review and land or close the canonical fix.

## What This Changes

Fix Control UI/WebChat code-block Copy so it copies only the selected fenced or indented code block instead of the surrounding assistant message.

## Best Possible Solution

Keep this issue open and review the linked fix PRs. The best product fix is one canonical Control UI patch that prevents message-level action overlap and makes the code-block Copy handler copy only the rendered block content, with focused regression coverage for assistant bubbles with actions, multiple fenced blocks, special characters, and code-block click targets.
