## Summary

Keep this issue open. Current main still has the relevant Control UI copy path: code-block buttons are rendered with `data-code`, the delegated click handler copies only that dataset value, and the existing `.chat-bubble.has-copy` spacing rule is still not applied to assistant bubbles that render message-level copy/expand actions. The issue is also paired with open, unmerged fix PRs #69634 and #73633 that use closing syntax, so it should remain open until maintainers review and land or close the canonical fix PR.

## Best Possible Solution

Keep this issue open and review the linked fix PRs, preferably landing one canonical Control UI patch that prevents message-level action overlap and makes the code-block Copy handler copy only the rendered block content. Add focused regression coverage around assistant bubbles with message-level actions, multiple fenced blocks, special characters, and code-block click targets.
