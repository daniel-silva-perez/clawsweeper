## Summary

Keep this open. Current main still exposes the reported LiteLLM/OpenAI MCP schema path: bundled MCP tools retain raw MCP input schemas, they are appended after the runtime tool normalization pass, and LiteLLM has no provider hook that applies the OpenAI-family schema compatibility rewrite.

## What This Changes

The issue reports that LiteLLM-routed OpenAI models reject Microsoft 365 MCP tool schemas because OpenClaw may forward JSON Schema shapes that OpenAI's tool validator does not accept.

## Best Possible Solution

Keep the issue open and land a targeted fix that normalizes post-materialization bundled MCP schemas for OpenAI-family routes, or rewrites the OpenAI transport payload at serialization time, with diagnostics that name the offending tool/schema path and regression coverage for nested array-valued type fields while preserving non-OpenAI proxy behavior.
