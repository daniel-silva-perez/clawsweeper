## Summary

Keep this issue open. Current main still lacks an enforced DM-level `toolsBySender` policy path: Telegram accepts `direct.*.toolsBySender` in config types/schema, but the enforced tool-policy pipeline remains group/channel scoped, and Signal still documents/exposes `toolsBySender` only for groups. The prior ClawSweeper comment matches the current code shape at main SHA 17811480dae20f65b526454d75e971ae8424ec10.

## Best Possible Solution

Keep this open and implement it as a generic channel-policy feature. Add a direct-message sender tool-policy contract, wire Signal and Telegram direct sender identity resolution into a direct-policy resolver, apply it in both `createOpenClawCodingTools` and `applyFinalEffectiveToolPolicy`, document the supported DM config surface, and add regression tests proving one allowlisted DM sender can retain tools while another is denied.
