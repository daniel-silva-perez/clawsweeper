## Summary

Current main still does not implement automatic rendering of a bare image URL typed into the Control UI chat. Image rendering is present for explicit attachments, structured image blocks, transcript media, and assistant MEDIA/directive paths, but plain chat text remains markdown/link text and remote image hotlinking is intentionally blocked by CSP.

## Best Possible Solution

Keep this issue open for a deliberate Control UI/Gateway feature: detect eligible bare image URLs in user chat text, ingest or proxy them through same-origin managed media instead of hotlinking, preserve normal link/text fallback, and add focused coverage for composer/send, chat.send history projection, grouped rendering, Markdown sanitization, managed media, and CSP behavior.

## Risks / Open Questions

- A direct regex-to-img implementation would conflict with the current Control UI CSP and privacy model by encouraging arbitrary browser image fetches from chat text.
- A safe implementation likely needs Gateway-managed fetch or media ingestion with SSRF/public-network guards, MIME and size validation, caching/error states, and preserved text/link fallback.
