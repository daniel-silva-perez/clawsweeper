## Summary

Keep open. Current main still treats a user-typed image URL as normal chat text; image rendering exists for uploads, structured image blocks, transcript media, and managed outgoing image blocks, but there is no promotion path for a bare URL in Control UI chat input. The existing CSP also blocks remote image hotlinking, so this needs a deliberate Gateway-managed media design rather than a regex-to-img renderer.

## Best Possible Solution

Keep this issue open for a deliberate Control UI/Gateway feature: detect eligible bare image URLs in user chat text, ingest or proxy them through same-origin managed media, preserve normal text/link fallback, and add focused coverage for composer/send, chat.send history projection, grouped rendering, markdown sanitization, managed media, and CSP behavior.

## Risks / Open Questions

- A direct client-side regex-to-img change would conflict with the current Control UI CSP and privacy model because arbitrary remote image fetches are intentionally blocked.
- A Gateway-managed implementation needs SSRF/public-network guardrails, MIME and size validation, caching/error states, and preserved text/link fallback before this should ship.
