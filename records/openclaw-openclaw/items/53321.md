## Summary

Keep open. Current main still has auto-compaction, pre-compaction memory flush, successor transcript rotation, session lifecycle hooks, and queued-followup retargeting, but it does not implement the requested configurable auto session split lifecycle with token/window thresholds, active-run boundaries, routing rollover, or continuity payloads for task and confirmation state. The same-author related issue #53330 is already closed as a duplicate, so it is useful context but not an active replacement tracker.

## What This Changes

The issue requests configurable automatic session rollover before context-window exhaustion, preserving continuity so users do not need to run `/new` manually.

## Best Possible Solution

Keep this as the canonical core product/design tracker and implement a supported `session.autoSplit` or equivalent policy that reuses existing compaction, memory flush, successor transcript rotation, queued-followup retargeting, and session hooks while adding explicit threshold configuration, routing updates, continuity payload handling, docs, and regression coverage.
