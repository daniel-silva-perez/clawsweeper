## Summary

Keep open. Current main has auto-compaction, pre-compaction memory flush, successor transcript rotation, and queued-followup retargeting, but it still does not implement the requested configurable auto session split lifecycle with token/window thresholds, routing rollover, active-run/tool-call boundaries, and continuity payloads for task or confirmation state. Related #53330 is closed/cross-referenced rather than a replacement tracker.

## What This Changes

The issue requests automatically starting a continuity-preserving new session when an agent session approaches the model context window limit.

## Best Possible Solution

Keep this as the canonical core product/design tracker and implement a supported session.autoSplit policy that reuses current compaction, memory flush, successor transcript rotation, queued-followup retargeting, and session hooks while adding explicit threshold configuration, active-run boundaries, routing updates, continuity payload handling, docs, and regression coverage.
