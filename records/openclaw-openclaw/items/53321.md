## Summary

Keep open. Current main has auto-compaction, memory flush, compaction notices, successor transcript rotation, and queued-followup retargeting, but it does not implement the requested configurable auto session split lifecycle with channel binding rollover and continuity payloads. Same-author #53330 is already closed as the duplicate, so this remains the canonical tracker and needs maintainer design rather than an automated repair lane.

## What This Changes

Request to add a configurable auto session split when a session nears its context window, including pre-split memory flush, continuity context injection, pending state carryover, and invisible routing to the new session.

## Best Possible Solution

Keep this as the canonical core session-lifecycle design item and define an explicit auto-split policy that reuses existing compaction, memory flush, transcript rotation, queue retargeting, and session hooks while adding threshold configuration, channel binding rollover, continuity payload handling, docs, and regression coverage.
