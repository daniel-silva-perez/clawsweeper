## Summary

Keep open. Current main has auto-compaction, pre-compaction memory flush, successor transcript rotation, and queued-followup retargeting, but it still does not implement the requested configurable auto session split lifecycle with token/window thresholds, routing rollover, active-run/tool-call boundaries, or continuity payloads for task and confirmation state. The same-author #53330 is already closed and cross-referenced, so it is not an open replacement tracker.

## What This Changes

Requested configurable automatic session rollover before context-window exhaustion, preserving continuity across the new session.

## Best Possible Solution

Keep this as the canonical core product/design tracker and implement a supported session.autoSplit policy that reuses existing compaction, memory flush, successor transcript rotation, queued-followup retargeting, and session hooks while adding explicit threshold configuration, active-run boundaries, routing updates, continuity payload handling, docs, and regression coverage.
