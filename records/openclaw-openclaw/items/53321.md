## Summary

Keep open. Current main still provides auto-compaction, pre-compaction memory flush, successor transcript rotation, failure recovery resets, and queued-followup retargeting, but it does not implement the requested configurable token-threshold session split lifecycle with channel routing rollover and continuity payloads. The same-author #53330 was already closed as a duplicate, so this issue remains the canonical tracker.

## What This Changes

The issue requests automatic session rollover near a model context-window limit, preserving summary, task, confirmation, and channel continuity so users do not need to manually run `/new`.

## Best Possible Solution

Keep this as the canonical core session-lifecycle design tracker and add a supported `session.autoSplit` or equivalent policy that reuses existing compaction, memory flush, successor transcript rotation, queued-followup retargeting, and session hooks while adding explicit threshold configuration, routing updates, continuity payload handling, docs, and regression coverage.
