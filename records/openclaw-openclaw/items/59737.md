## Summary

Keep open. Current main still has only the comprehensive root PR template and no `.github/PULL_REQUEST_TEMPLATE/` directory or CONTRIBUTING guidance for quick/full template selection. Related implementation attempts exist, but the visible PRs were closed unmerged and prior review context raised real maintainer-policy questions around GitHub's query-parameter behavior and dropping required security/human-verification sections.

## What This Changes

The issue requests separate GitHub pull request templates for quick fixes and substantial changes, plus contributor guidance for selecting the right template.

## Best Possible Solution

Decide the contributor-policy shape first, then implement a GitHub-compatible template setup: keep or replace the root default deliberately, document exact `?template=` links or another supported selection path, and make any quick template short while retaining abbreviated security, human-verification, and review-conversation checks.

## Reproduction Assessment

Not applicable as a runtime bug. The repository-state check is high confidence: current main lacks `.github/PULL_REQUEST_TEMPLATE/quick_fix.md`, `.github/PULL_REQUEST_TEMPLATE/full_change.md`, and CONTRIBUTING guidance for choosing them.

## Solution Assessment

Unclear. Splitting templates is a reasonable DX direction, but the issue's exact proposal is not yet clearly the best fix because GitHub's documented multiple-template flow uses query parameters and the quick template needs maintainer-approved handling of required security and verification sections.
