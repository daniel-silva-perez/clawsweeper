## Summary

Keep open. Current main still does not implement the PR’s exact fail-fast overflow behavior for streamed media bodies: the shared response limiter still awaits `reader.cancel()` before surfacing the overflow error, and the focused non-returning-cancel regression test is still absent. The branch is useful but stale against current main because Matrix now inherits this path through the shared SDK wrapper and the agent import cleanup no longer applies.

## What This Changes

The PR changes the shared bounded-response reader to fire-and-forget cancellation for oversized streamed media, adds a regression test for a never-resolving `cancel()`, applies the same pattern to the then-current Matrix helper copy, and removes two unused agent imports.

## Best Possible Solution

Land or replace this with a narrow current-main patch in `src/media/read-response-with-limit.ts` plus the focused never-resolving-cancel unit test in `src/media/read-response-with-limit.test.ts`; let Matrix inherit the shared behavior through its wrapper and leave unrelated agent cleanup out of the patch.
