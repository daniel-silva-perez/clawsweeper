## Summary

Keep open. The core shared-reader fix is still relevant on current main and in the latest release, but this PR should not be merged as-is because its Matrix edit is stale against the current SDK-wrapper design.

## What This Changes

The PR makes overflow cancellation fire-and-forget in the shared bounded-response reader and the then-current Matrix copy, adds a unit test for never-resolving `cancel()`, and removes two unused Google API base URL imports from agent files.

## Best Possible Solution

Carry forward the useful shared-reader behavior on current main: make overflow cancellation non-blocking in `src/media/read-response-with-limit.ts`, add the focused never-resolving-cancel unit regression, let Matrix inherit the fix through the SDK wrapper, and keep unrelated agent cleanup out of the patch.

## Reproduction Assessment

Yes. A mocked streamed response whose first chunk exceeds `maxBytes` and whose `reader.cancel()` never resolves reproduces the shared-helper hang on current main; the live Discord channel stall is now partly bounded by the 120s total attachment timeout.

## Solution Assessment

No, not as currently shaped. The shared reader change and regression test are the right narrow fix, but the Matrix edit targets an obsolete copied implementation and the agent import cleanup is unrelated.
