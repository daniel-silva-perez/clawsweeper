## Summary

Keep this PR open. Current main still waits for `reader.cancel()` on the max-bytes overflow path in the shared bounded response reader, and the current tests still do not cover the non-returning cancel case this PR adds. Discord now has a 120s total attachment timeout, which bounds the hang in that path, but it does not implement the PR's intended fail-fast overflow behavior. The PR remains useful, but it is stale against current main because Matrix now delegates to the shared reader instead of carrying the copied helper patched by the PR.

## Best Possible Solution

Keep the PR open and review or rebase it into a current-main patch that changes only the shared `src/media/read-response-with-limit.ts` overflow cleanup to cancel without awaiting, adds the non-returning `reader.cancel()` regression test in `src/media/read-response-with-limit.test.ts`, leaves Matrix on the shared wrapper, and drops unrelated unused-import cleanup if it is no longer needed.
