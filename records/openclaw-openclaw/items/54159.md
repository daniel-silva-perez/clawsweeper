## Summary

Keep this PR open. Current main has later Discord mitigations for stuck media downloads, including a per-attachment total timeout and fallback behavior, but it does not implement this PR's central shared-reader fix: the overflow path in `readResponseWithLimit` still awaits `reader.cancel()`, and the non-returning cancel regression test is still absent.

## Best Possible Solution

Review or rework this PR as a narrow current-main patch: make the shared `src/media/read-response-with-limit.ts` overflow cleanup fire-and-forget like the idle-timeout path, add the never-resolving `cancel()` regression test in `src/media/read-response-with-limit.test.ts`, and drop stale Matrix-local and unrelated import-cleanup hunks. If maintainers decide the later Discord total-timeout behavior is sufficient, close with that product decision rather than as already implemented.
