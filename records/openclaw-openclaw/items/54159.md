## Summary

Keep this PR open. Current main still awaits `reader.cancel()` in the shared overflow path, so the branch's fail-fast cancellation behavior is not implemented; Discord now has total/idle timeout mitigation, but that does not provide immediate overflow rejection or the proposed shared-reader regression coverage.

## What This Changes

The branch changes the shared bounded response reader and the old Matrix helper copy to cancel overflowed streams without awaiting `reader.cancel()`, adds a never-resolving-cancel regression test, and removes two unused agent imports.

## Best Possible Solution

Land a narrow current-main fix in the shared response reader: fire-and-forget `reader.cancel()` on max-byte overflow, add the never-resolving-cancel unit regression, let Matrix inherit the behavior through the SDK wrapper, and leave Discord's existing total/idle timeouts as backup protection.
