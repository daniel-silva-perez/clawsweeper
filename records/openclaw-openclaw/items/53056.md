## Summary

Keep this issue open. Current main still routes Slack outbound text through helpers that escape `&` to `&amp;`, the linked fix PRs are closed unmerged, and current tests do not cover the reported visible `&#38;` regression.

## Best Possible Solution

Keep this issue open. Add focused Slack regression tests for raw `R&D`/`B&A`, already-escaped `&amp;`, and numeric `&#38;` inputs across normal send, slash replies, preview final edits/readback, draft/tool-progress previews, and native-stream fallback paths; then fix the duplicate-encoding boundary inside `extensions/slack` while preserving Slack's required single escaping pass.
