## Summary

Keep open: current main and the latest release still route Slack outbound text through helpers that encode `&` as `&amp;`, while the two linked fix PRs are closed unmerged and the discussion identifies a real safety question around Slack's exactly-one escaping contract.

## What This Changes

The issue asks for Slack outbound messages containing literal ampersands, such as `R&D`, to display `&` instead of visible `&#38;`.

## Best Possible Solution

Keep the report open until Slack rendering is verified directly, then update the Slack-owned formatter/send paths to provide exactly one safe escaping pass for visible text while preserving intentional Slack angle tokens and literal entity safety.
