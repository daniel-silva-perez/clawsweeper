## Summary

Keep this PR open. Current main still renders Overview recent sessions with `displayName` before `label`, so the intended UI fix is not implemented on main. The related #45130 work fixed the chat/session dropdown helper, but this overview-card surface still has its own fallback expression. The PR is narrow and not superseded; it needs maintainer review and preferably a focused regression test before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to make Overview recent sessions use the same label-first, key-aware display semantics as the chat session dropdown, preferably by reusing `resolveSessionDisplayName` or extracting a shared UI helper, and add a focused render/unit test for `label > displayName > key` plus the `label === key` case.
