## Summary

Keep this PR open. Current main still renders the Overview Recent Sessions card with `displayName` before `label`, so the PR’s intended UI fix is not implemented on main. The related #45130 work fixed the chat session display helper, but this overview-card surface still has its own fallback expression.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to make Overview Recent Sessions use the same label-first, key-aware display semantics as the chat session dropdown, preferably with a focused rendering/unit test for `label > displayName > key` and the `label === key` case.
