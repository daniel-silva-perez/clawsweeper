## Summary

Keep this PR open. Current main still renders the overview Recent Sessions card with `displayName` before `label`, so the PR’s intended UI fix is not implemented on main. The related shipped work for #45130 covers the chat session dropdown helper, not this overview-card surface. The PR remains narrow and security-sensitive files are not touched, but it should get maintainer review and ideally a small overview rendering test.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a focused overview-card fix that makes Recent Sessions use the same label-first, key-aware display semantics as the chat session dropdown, plus a small rendering/unit test for `label > displayName > key` and the `label === key` case.
