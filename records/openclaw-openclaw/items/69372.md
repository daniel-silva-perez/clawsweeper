## Summary

Keep this PR open. Current main still emits raw protected Control UI canvas iframe URLs when no scoped canvas host is available and the browser client still has no Control UI-side capability refresh. The PR addresses the right surface, but its timer-driven WebSocket reconnect can reject pending RPCs and reset active chat run state; current main already has a non-teardown `node.canvas.capability.refresh` seam that looks like the safer direction.

## Best Possible Solution

Keep the PR open for rework. Preserve the fail-closed protected iframe URL behavior, but replace the scheduled WebSocket reconnect with an idle-safe or non-teardown refresh that uses the existing `node.canvas.capability.refresh` capability where appropriate. Add tests proving protected canvas/a2ui URLs do not render raw without a scoped host, scoped canvas/a2ui URLs still rewrite correctly, capability refresh does not reject pending browser RPCs, and active chat run state is not reset by refresh.
