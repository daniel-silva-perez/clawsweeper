## Summary

Keep this issue open. Current main still routes Telegram inbound updates through one receiving account and one resolved agent route; there is no `forwardToAccounts`, `broadcastToAccounts`, or Telegram broadcast/fan-out surface. The related open PR #63380 is adjacent account-level `agentId` schema work for #62985, not an implementation of bot-to-bot group forwarding.

## What This Changes

Request support for routing Telegram group messages across multiple bot accounts so separate agents can see and respond to bot-to-bot mentions without sharing one bot identity.

## Best Possible Solution

Keep this open for maintainer design. The best path is to decide whether Telegram should adopt the existing generic broadcast-group model, a Telegram-specific cross-account forwarding rule, or documented alternatives; any implementation should be additive, preserve account identity and sender allowlists, prevent loops, handle bot-authored messages deliberately, and add Telegram multi-account group tests plus docs.
