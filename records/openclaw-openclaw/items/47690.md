## Summary

Keep this issue open. Current main still resolves each Telegram inbound update to one route for the receiving account, has no cross-account forwarding or broadcast config surface, and documents Telegram multi-account support as one-account-to-one-agent bindings rather than group fan-out across bot identities. The related open PR #63380 is adjacent account-level `agentId` schema work for #62985, not an implementation of this requested group bot-to-bot routing.

## What This Changes

Add Telegram multi-account group routing so a message received by one bot account can be forwarded or fanned out to other configured accounts/agents while preserving distinct bot identities.

## Best Possible Solution

Design an additive Telegram cross-account or broadcast routing model, or explicitly document the supported alternative, with clear loop prevention, per-account identity preservation, allowlist semantics, bot-authored message handling, and focused Telegram multi-account group coverage.
