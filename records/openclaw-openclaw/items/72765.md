## Summary

Keep this PR open. The underlying `sessions.compact` `maxLines` data-loss risk is still present on current main, and the production fix follows the existing active-run guard pattern, but the branch is currently dirty because its regression test targets a session test file that main has since removed.

## What This Changes

The PR adds an active-run interrupt guard before the `sessions.compact` `maxLines` transcript archive/rewrite path, makes `hasTrackedActiveSessionRun` tolerate a missing `chatAbortControllers` map, and adds a regression test for a stuck embedded run.

## Best Possible Solution

Land a rebased narrow gateway fix that calls `interruptSessionRunIfActive` before the `maxLines` transcript read/archive/rewrite and keeps the regression in `src/gateway/server.sessions.compaction.test.ts`, preserving unchanged behavior when no run is active.

## Reproduction Assessment

Yes. The current-main code path is enough to reproduce the risk: create an active embedded session run, call `sessions.compact` with `maxLines`, and the handler reaches `archiveFileOnDisk` and `writeFileSync` without first interrupting the run.

## Solution Assessment

Yes in concept, but not as currently rebased. Reusing `interruptSessionRunIfActive` is the narrowest maintainable fix because the AI compaction and restore paths already use it; the regression test needs to move to the current compaction suite.
