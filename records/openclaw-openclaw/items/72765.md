## Summary

Keep this PR open. Current main still lacks the active-run guard on the `sessions.compact` `maxLines` archive/rewrite path, and the PR’s production direction matches the existing AI compaction and restore guards. ## What This Changes

The PR adds an active-run interrupt guard before `sessions.compact` performs `maxLines` transcript archive/rewrite, adds a null-safe fallback for missing `chatAbortControllers`, and adds a regression test for a stuck embedded run.

## Best Possible Solution

Land a rebased narrow gateway fix that calls `interruptSessionRunIfActive` before the `maxLines` transcript read/archive/rewrite, keeps the active-run regression in `src/gateway/server.sessions.compaction.test.ts`, and records the fix in the Unreleased changelog.

Yes. The current-main code path is enough to reproduce the risk: start or simulate an active embedded session run, call `sessions.compact` with `maxLines`, and the handler can reach `archiveFileOnDisk` and `writeFileSync` without first interrupting the active run.

## Solution Assessment

Yes in concept, but not as currently rebased. Reusing `interruptSessionRunIfActive` is the narrowest maintainable fix because the AI compaction and restore paths already use it; the test and changelog need to match current main policy.
