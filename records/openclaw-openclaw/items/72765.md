## Summary

Keep this PR open. Current main still has the unsafe `sessions.compact` `maxLines` transcript archive/rewrite path, and the production fix matches the existing active-run guard used by AI compaction and restore. The branch is not ready as-is because current main removed the monolithic session test file that this PR modifies, so the regression needs to be rebased into the current compaction test file.

## What This Changes

The PR adds an active-run interrupt guard before `sessions.compact` performs `maxLines` transcript archive/rewrite, adds a null-safe `chatAbortControllers` fallback in `hasTrackedActiveSessionRun`, and adds a regression test for a stuck embedded run.

## Best Possible Solution

Land a rebased narrow gateway fix that calls `interruptSessionRunIfActive` before the `maxLines` transcript read/archive/rewrite and keeps the regression in the current `src/gateway/server.sessions.compaction.test.ts` suite, preserving unchanged behavior when no run is active.

## Reproduction Assessment

Yes. The current-main code path is enough to reproduce the risk: create an active embedded session run, call `sessions.compact` with `maxLines`, and the handler reaches `archiveFileOnDisk`/`writeFileSync` without the active-run interrupt guard.

## Solution Assessment

Yes in concept, but not as currently rebased. Reusing `interruptSessionRunIfActive` is the narrowest maintainable fix because the AI compaction and restore paths already use it; the regression just needs to move to the maintained compaction test file.
