## Summary

Keep this PR open. Current main still builds default backup archive roots and basenames through the shared session archive timestamp formatter, which uses UTC `toISOString()` output and keeps the trailing `Z` filename shape. The docs and CLI help still show UTC-style backup filenames, and there is no backup-specific local-time/UTC-offset formatter on main. The PR remains a focused implementation candidate rather than obsolete or already implemented.

## Best Possible Solution

Keep this PR open, rebase it onto current main, and review or land a scoped backup-specific timestamp formatter that uses host-local time with an explicit UTC offset. Keep the shared session archive formatter unchanged unless maintainers intentionally want broader archive behavior changes, and align backup docs/help plus regression tests with the final filename contract.
