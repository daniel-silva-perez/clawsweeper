## Summary

Keep this PR open. Current main still builds backup archive names through the shared session archive timestamp formatter, which uses UTC `toISOString()` and produces the trailing `Z` basename form. The PR remains a focused implementation candidate for backup-specific local-time filenames with explicit UTC offsets.

## Best Possible Solution

Keep this PR open, rebase it onto current main, and review or land a scoped backup-specific timestamp formatter that uses host-local time with an explicit UTC offset. Keep the shared session archive formatter unchanged unless maintainers intentionally want that broader behavior change, and align backup docs/help plus regression tests with the final filename contract.
