## Summary

Keep this PR open. Current main still builds backup archive roots and default archive basenames from the shared session archive timestamp formatter, which is UTC `toISOString()` output with a trailing `Z`. The backup docs and CLI help also still show trailing-`Z` examples, and a targeted search found no backup-specific local-time/UTC-offset formatter on main. The provided PR remains a focused implementation candidate, and its diff does not show unrelated CI, dependency, release, permissions, or supply-chain changes.

## Best Possible Solution

Keep this PR open, rebase it onto current main, and review or land a scoped backup-specific timestamp formatter that uses host-local time with an explicit UTC offset. Keep the shared session archive formatter unchanged unless maintainers intentionally want broader archive behavior changes, and align backup docs/help plus regression tests with the final filename contract.
