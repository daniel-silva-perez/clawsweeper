## Summary

Keep this PR open. Current main still builds default backup archive names from the shared session archive formatter, which emits UTC `toISOString()` timestamps with a trailing `Z`; no backup-specific local-time formatter or offset-bearing backup filename example is present on main. The PR remains a focused implementation candidate, and the provided diff does not introduce unrelated security-sensitive CI, dependency, release, or script changes.

## Best Possible Solution

Keep this PR open, rebase it onto current main, and review or land a scoped backup-specific timestamp formatter that uses host-local time with an explicit UTC offset. Keep the shared session archive formatter unchanged unless maintainers intentionally want a broader session archive behavior change, and align backup docs/help plus regression tests with the final filename contract.
