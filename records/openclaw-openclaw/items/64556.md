## Summary

Current main still has the reported wake-hook routing mismatch: `hooks.mappings[]` accepts `agentId` and `sessionKey`, but wake mappings drop those fields before dispatch and the wake dispatcher enqueues into the default main session. PR #70268 is still open and explicitly targets this issue, so this should remain open until that PR is reviewed and merged, or closed as superseded by an equivalent fix.

## Best Possible Solution

Keep this issue open and treat PR #70268 as the active implementation candidate. The fix should thread `agentId` and `sessionKey` through wake mapping resolution and dispatch, preserve the main-session fallback when routing is omitted, reuse the existing hook agent/session policy guardrails, add regression coverage for mapped wake routing, and separately track the config reload follow-up if maintainers agree it is in scope.
