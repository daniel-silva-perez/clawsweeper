## Summary

Keep this issue open. Current main still drops `agentId` and `sessionKey` for `action: "wake"` mappings, then dispatches wake events to the default main session. PR #70268 remains open and targets the same bug, so the issue should stay open until that PR is reviewed and merged or superseded.

## Best Possible Solution

Keep this issue open and treat PR #70268 as the active implementation candidate. The fix should thread `agentId` and `sessionKey` through wake mapping resolution, merge, policy checks, and dispatch; preserve the main-session fallback when routing is omitted; add regression coverage for mapped wake routing and config guardrails; and handle any direct `/hooks/wake` session targeting as an explicit API/security decision.
