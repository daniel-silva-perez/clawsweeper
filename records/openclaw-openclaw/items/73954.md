## Summary

Keep this issue open. The report is fresh against v2026.4.26, and current main still does not show a clear fix for the Discord exec-approval failure. The central approval-pending tool result still exposes both the full shell command and the `/approve <slug> ...` line to the model, while Discord native approval delivery is a separate path whose success/fallback is not proven by a post-report code change or regression test.

## Best Possible Solution

Keep this open and fix the Discord native exec-approval path so enabled native approval sessions do not depend on a model copying mixed free text correctly. The best fix should make native delivery success/failure explicit, avoid exposing ambiguous `/approve` plus full command text to weak models when native cards are the intended UI, and add targeted regression coverage for Discord native delivery, local-prompt suppression, unavailable/fallback behavior, and final assistant-output suppression.
