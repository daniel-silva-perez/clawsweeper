## Summary

Keep this PR open. Current main does not contain the PR's intended OpenRouter onboarding behavior: the OpenRouter onboarding helper still forces `openrouter/auto`, the static catalog still has no `free` routing entry, and the current stream wrapper still only skips reasoning injection for `auto` or known unsupported proxy models. The linked bug is closed, but the PR's proposed code is not implemented on main and the remaining question is whether maintainers want `openrouter/free` as a supported pseudo-model or prefer the newer concrete `:free` scan / `openrouter:free` path.

## Best Possible Solution

Keep the PR open for maintainer review rather than auto-closing it. The best path is to rebase/adapt it to current main, decide whether `openrouter/free` should remain a supported pseudo-model versus steering users to concrete `:free` models or `openrouter:free`, and add focused regression coverage in `extensions/openrouter/onboard.test.ts` plus the current OpenRouter stream-wrapper tests if `free` remains supported.
