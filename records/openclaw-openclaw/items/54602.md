## Summary

Keep open. The PR is useful but stale against main and needs focused regression tests.

## Best Possible Solution

Keep this PR open and refresh it, or replace it with an equivalent plugin-local fix. The fix should preserve an existing `openrouter/*` primary when re-applying OpenRouter auth/config, add `openrouter/free` to the OpenRouter static catalog if it is intended to be selectable, update the current `extensions/openrouter/stream.ts` wrapper to skip reasoning injection for the free route, and add focused OpenRouter onboarding/catalog/stream tests. No core/provider architecture change appears necessary.
