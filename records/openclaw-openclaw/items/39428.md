## Summary

Current main still does not implement implicit Discord accountId-to-agentId routing. Discord inbound messages pass the accountId into the generic resolver, and that resolver still evaluates explicit binding tiers before falling through to the configured default agent. The linked implementation candidate, PR #39485, remains open and unmerged with closing syntax for this issue, so this issue should remain open until that PR or an equivalent guarded fix lands.

## Best Possible Solution

Keep this issue open and review or land PR #39485, or an equivalent maintainer-approved fix, that adds a narrow Discord-only fallback after explicit binding tiers and before default-agent fallback. The fix should skip accountId "default", require the candidate agent to exist in agents.list, use a distinct route match reason, and add regression tests for positive matching, default-account behavior, and the empty-agent-list guard.
