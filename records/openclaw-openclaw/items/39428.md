## Summary

Keep this issue open. Current main still routes Discord inbound messages through the generic route resolver, which only evaluates explicit binding tiers before default-agent fallback; it does not implement implicit Discord accountId-to-agentId matching. The provided GitHub context also shows an open, unmerged implementation candidate, PR #39485, with closing syntax for this issue, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review or land PR #39485, or an equivalent maintainer-approved fix, that adds a Discord-only fallback after explicit binding tiers and before default-agent fallback. The fix should preserve the literal default account behavior and add regression coverage for both accountId-to-agentId matching and the default-account guard.
