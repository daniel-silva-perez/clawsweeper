## Summary

Keep open. Current main still lacks an outbound OpenAI-compatible Chat Completions message-key serialization policy for strict providers, and the existing string-content, replay-hygiene, request-policy, and proxy-shaping paths do not strip tool-call or tool-result keys from converted message objects.

## What This Changes

The issue requests sanitizing outgoing `openai-completions` message objects so strict OpenAI-compatible providers such as Infomaniak do not reject extra keys beyond the keys they accept.

## Best Possible Solution

Add a provider/model-level OpenAI-compatible Chat Completions serialization policy that runs after message conversion and before dispatch in both direct and embedded payload paths, preserves stored transcripts, and covers prior assistant tool calls plus tool-result history in tests and docs.
