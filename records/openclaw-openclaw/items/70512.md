## Summary

Keep open. Current main still parses Feishu/Lark DM root/thread metadata but routes direct messages by sender Open ID and flat chat sequential keys, with no DM topic session-scope config or docs. The same remaining work is already covered by open PR #63148, so maintainers should review or land that PR rather than close this issue now.

## What This Changes

Request Feishu direct-message topic/thread session isolation so different DM topics from the same sender can route to separate gateway sessions.

## Best Possible Solution

Review and land #63148 or an equivalent Feishu-plugin patch that scopes only topic-bearing DMs to stable topic-specific conversation IDs, preserves existing main-surface DM behavior, keeps replies and sequential/debounce keys thread-aware, and updates config/docs if maintainers choose an opt-in setting rather than default topic isolation.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: construct Feishu p2p events from the same sender with different root_id/thread_id values; current main still resolves both through peer kind direct with id ctx.senderOpenId and uses a chatId-only sequential key.

## Solution Assessment

Unclear until the open PR is reviewed. The narrow maintainable direction is inside the bundled Feishu plugin, but maintainers should decide whether #63148's default-on topic behavior is acceptable or whether the issue's explicit opt-in config is required for compatibility.
