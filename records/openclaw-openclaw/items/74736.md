## Summary

Keep open. The PR addresses a real narrow observability gap, but the proposed lock-timeout branch drops the detailed human-readable timeout/remediation text from normal console output while moving only part of that data into structured metadata; because subsystem console output renders metadata only in JSON style, this needs adjustment before merge.

## What This Changes

The PR adds a lock-timeout detector/parser to the bundled channel entry loader and new Vitest coverage for structured warning metadata on lock timeouts while preserving flat warnings for other import errors.

## Best Possible Solution

Land a narrow logging change that keeps the existing timeout detail visible to non-JSON console users while also adding structured fields such as `failureReason`, `bundledChannelId`, and parsed lock metadata, with tests asserting both the structured metadata and message-detail preservation.
