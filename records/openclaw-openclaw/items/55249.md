## Summary

Keep open. Current main has the backend/session-store and Control UI pieces for labels, but it does not implement the issue's requested CLI surface: there is no `openclaw sessions label` subcommand, the `openclaw sessions` table has no Label column, and the CLI JSON rows are built from a display row type that omits stored `label`/`displayName`. Related ClawSweeper closures covered dashboard/TUI/subagent label display, not the remaining CLI labeling workflow described here.

## Best Possible Solution

Implement the missing CLI workflow in `src/cli/program/register.status-health-sessions.ts` and `src/commands/sessions*.ts`: add a `openclaw sessions label <session-key> <label>` command that updates the resolved session store using the same validation as Gateway `sessions.patch`, include `label`/`displayName` in CLI JSON output, add a Label column or otherwise show labels in text output, and update `docs/cli/sessions.md` with focused CLI tests.
