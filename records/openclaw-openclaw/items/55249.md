## Summary

Keep open. Current main has substantial backend/session-store support for labels, including `sessions.patch`, agent-turn label persistence, `sessions_spawn` labels, and gateway `sessions.list` label/displayName output, but this issue's CLI workflow remains incomplete: `openclaw sessions` has no label subcommand, the CLI row/table omits labels, and the CLI docs do not document label management.

## What This Changes

Request adding human-readable session labels or nicknames that persist and appear in `openclaw sessions` and JSON listings, with optional programmatic and auto-labeling support.

## Best Possible Solution

Implement the remaining CLI layer on top of the existing session label storage: add label management for stored sessions, include labels in text and JSON listings, preserve existing validation/uniqueness semantics, and update CLI docs and changelog with focused tests.
