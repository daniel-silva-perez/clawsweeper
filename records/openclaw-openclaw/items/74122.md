## Summary

Current main only partially satisfies the report. The gateway config watcher has a configurable debounce and path-based hot-reload planner, but the default debounce is still 300ms, watcher callbacks do not carry the triggering file path into reload logging, plugin/config-prefix changes can still require gateway restart, and the related open issues cover only adjacent pieces rather than the full debounce plus visibility request.

## Best Possible Solution

Keep this issue open and scope it into concrete follow-up work: preserve the existing config reload planner, decide whether the default debounce should change or only docs/config guidance should improve, add trigger-source aggregation/logging if desired, and route plugin source/dist hot reload through the canonical plugin hot-reload work in #14438.
