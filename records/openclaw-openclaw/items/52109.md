## Summary

Keep this PR open. Current main still does not apply the successful-run refire floor to `every` schedules, so the PR is not obsolete or implemented-on-main. The PR does need a current-main rebase/update because newer scheduler code clears unresolved cron next-runs instead of synthesizing a 2s retry.

## Best Possible Solution

Keep this PR open, but update it against current main. The best fix is a narrow scheduler change that applies the 2s refire floor to successful `every` completions only when the computed next run is at or before the just-finished `endedAt`, preserves valid future ticks and forced-run anchor semantics, preserves current unresolved-next-run clearing behavior, and adds focused regression tests for overrun, exact-boundary, malformed `everyMs`, and `preserveSchedule` cases.
