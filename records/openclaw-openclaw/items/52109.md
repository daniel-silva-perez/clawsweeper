## Summary

Keep this PR open. Current main still does not implement the PR’s intended successful `every`-schedule refire-gap behavior: cron-expression completions get a lower bound, but non-cron recurring completions persist `naturalNext` directly. The PR remains a focused scheduler fix with useful regression coverage and prior review narrowing the risky cases.

## Best Possible Solution

Keep this PR open. The best path is to rebase or replace it with a narrow cron scheduler fix that applies the refire floor to `every` schedules only when the computed next run would be at or before the just-finished run, preserves valid future ticks and malformed-schedule behavior, and lands focused regression tests for overrun, exact-boundary, malformed `everyMs`, and forced-run `preserveSchedule` cases.
