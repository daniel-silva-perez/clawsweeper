## Summary

Keep this PR open. Current main still has the schema/implementation mismatch the PR targets: `node` is optional in the `nodes` tool schema, but the `describe` branch still hard-requires it and calls `resolveNodeId` without the default-node fallback. The PR is a focused implementation candidate for open issue #51903, but it still needs a targeted regression test before it is cleanly landable.

## Best Possible Solution

Keep the PR open, add a focused regression test for `{ action: "describe" }` with no `node` that asserts `resolveNodeId` is called with `allowDefault=true` and `node.describe` receives the resolved node id, then review and land this PR to close the paired open issue #51903.
