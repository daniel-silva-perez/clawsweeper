## Summary

Keep this PR open for maintainer review. Current main is not fully redundant with the proposed branch: it covers the narrow Google Vertex path with a hard-coded preload, but `models list` still does not ask `resolveEnvApiKey` on demand for resolver-only provider auth paths, and current resolver/setup code shows those paths exist.

## What This Changes

The PR replaces the one-off Google Vertex auth-index preload with a cached on-demand `resolveEnvApiKey` fallback in `createModelListAuthIndex` and adds a focused regression test for resolver-only env auth.

## Best Possible Solution

Land the generic cached resolver fallback if maintainers want `models list` availability to honor provider-owned resolver auth, or replace it with an equally generic declarative setup contract that removes the remaining provider-specific branch without losing resolver-backed availability.
