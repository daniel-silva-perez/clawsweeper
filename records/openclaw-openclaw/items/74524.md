## Summary

Keep this PR open for maintainer review. Current main already covers the specific Google Vertex availability path with setup env metadata and a Google Vertex ADC fallback, but it still lacks the broader cached on-demand resolver fallback that this PR proposes for resolver-only auth paths, so the branch is not fully redundant.

## What This Changes

The PR replaces a one-off Google Vertex model-list auth preload with a cached on-demand `resolveEnvApiKey` fallback in `createModelListAuthIndex` and adds a regression test that forces an empty env-candidate map.

## Best Possible Solution

Keep the fix narrow: either land the cached on-demand resolver fallback with focused regression coverage, or replace it with an equally generic manifest/setup contract that lets `models list` reflect resolver-only auth without reintroducing provider-specific branches.
