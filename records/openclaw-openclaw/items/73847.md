## Summary

Keep open. Current main still uses an identity-keyed `WeakMap<OpenClawConfig, ...>` for web-provider snapshot memoization, so this PR is not obsolete or implemented on main. The linked #73730 diagnosis matches the current code path, and the PR is a focused implementation candidate with source/tests/changelog changes only. It still needs maintainer review, especially for the bot-raised stable-fingerprint concern and bounded cache behavior after changing the cache from WeakMap-backed storage to a flat Map.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a web-provider snapshot cache fix for #73730, either this content-keyed approach or an equivalent config-canonicalization design, after addressing stable fingerprinting and bounded cache eviction. The linked issue should close only after the fix merges and passes the targeted plugin/web-provider tests plus the changed-lane gate.

## Risks / Open Questions

- Closing this PR would drop a focused implementation candidate while current main still has the reported identity-keyed cache miss behavior.
- The proposed fingerprint uses `hashJson`/`JSON.stringify`, which is insertion-order-sensitive; maintainers should decide whether to use a stable sorted serializer before landing.
- The proposed flat `Map` no longer benefits from per-config WeakMap garbage collection, so expired-entry pruning or a size bound may need review if config content churns.
- Tests were not run during this read-only review; the PR body reports targeted Vitest passes, but maintainer validation is still needed.
