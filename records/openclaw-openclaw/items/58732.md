## Summary

Keep this PR open. Current main still does not implement AbortSignal support in runTasksWithConcurrency, so the work is not obsolete or implemented. The requested API is plausible because the helper is exported through plugin-facing runtime facades, but the submitted patch needs a contract fix before merge: cancellation would produce partial results while reporting hasError as false.

## Best Possible Solution

Keep the PR open and ask for a small contract revision: define cancellation as a non-success outcome, preferably by throwing or surfacing an explicit abort state, then add deterministic tests for pre-aborted and mid-run cancellation plus the unchanged no-signal behavior. After that, maintainers can review whether the public helper should accept AbortSignal and whether any plugin-facing docs or API notes are needed.
