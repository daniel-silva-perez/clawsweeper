## Summary

Keep this PR open. Current main still does not emit the requested gateway startup warning for an explicit `agents.defaults.sandbox.mode="off"`; the existing sandbox-off audit path is narrower and only reports inactive Docker sandbox settings. The PR patch is plausibly useful after cleanup, but it also adds gateway auth startup warnings outside the stated sandbox scope and lacks regression tests.

## Best Possible Solution

Keep the PR open and ask for a focused revision: add the startup dangerous-config flag for the explicit raw value `cfg.agents?.defaults?.sandbox?.mode === "off"`, prove absent sandbox config does not warn, add targeted tests in `src/security/dangerous-config-flags.test.ts`, and either split the gateway auth warning additions or align them with existing audit/runtime semantics before retaining them.
