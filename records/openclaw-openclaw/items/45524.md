## Summary

Keep open. Current main still lacks the requested startup warning for explicit `agents.defaults.sandbox.mode="off"`, and this PR is not merge-ready because it is stale against the current collector split and its extra trusted-proxy warning misses the omitted-allowlist case.

## What This Changes

The PR adds startup dangerous-config warnings for explicit `agents.defaults.sandbox.mode="off"`, `gateway.auth.mode="none"`, and trusted-proxy auth with an explicit empty `allowUsers` list in `src/security/dangerous-config-flags.ts`.

## Best Possible Solution

Carry forward the narrow sandbox startup warning in the current collector structure, warning only for an explicit raw `agents.defaults.sandbox.mode === "off"`, with tests proving absent sandbox config stays quiet. Split the gateway-auth startup warnings into a separate secops-reviewed change, or make them match the existing runtime and audit semantics before including them.
