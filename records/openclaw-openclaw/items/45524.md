## Summary

Keep this PR open. Current main still lacks the requested startup dangerous-config warning for an explicit `agents.defaults.sandbox.mode="off"`, and the PR remains useful, but it is dirty against current main's refactored dangerous-flag collector and its extra gateway-auth warnings need semantic review before merge.

## What This Changes

The PR adds startup dangerous-config warnings for explicit `agents.defaults.sandbox.mode="off"`, `gateway.auth.mode="none"`, and trusted-proxy auth with an explicit empty `allowUsers` list in `src/security/dangerous-config-flags.ts`.

## Best Possible Solution

Carry forward the narrow sandbox warning in the current collector structure, warning only for an explicit raw `agents.defaults.sandbox.mode === "off"`, with tests proving absent sandbox config stays quiet. Split the auth warnings into a separate secops-reviewed change, or make them match the existing runtime and audit semantics before including them.
