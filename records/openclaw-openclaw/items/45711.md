## Summary

Current main still contains the shortened Sessions UI labels reported here: `Store`, `Active`, `Global`, and `Unknown`. The latest release tag available in the checkout contains the same text, and the colocated Sessions view tests do not cover this readability regression. This should remain open as a narrow Control UI Sessions bug, with #38420 treated as broader related UX context rather than a replacement.

## Best Possible Solution

Keep this issue open and make a narrow Control UI fix in `ui/src/ui/views/sessions.ts`: replace ambiguous copy such as `Store`, `Active`, `Global`, and `Unknown` with human-readable labels like `Session store`, `Active within (minutes)`, `Include global sessions`, and `Include unknown sessions`. Add a focused Sessions view test for the rendered labels. Avoid changing the `sessions.list` API, controller contract, or broader dashboard layout.
