Keep this PR open. Current main still has the block-reply final-flush gap described by the ClawSweeper report: assistant stream events preserve a trailing literal final-tag prefix, but the text_end onBlockReply path still flushes without telling the tag stripper this is the final chunk. Keep this PR open as the implementation candidate. Review the patch, confirm the final-flush behavior for both unchunked and chunked text_end block replies, and run the expected changed gate before merge; no separate cleanup close is appropriate while current main lacks the change.
