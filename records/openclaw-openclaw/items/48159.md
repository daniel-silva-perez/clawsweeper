## Summary

Keep this issue open. Current main does not implement a hard per-session ephemeral/no-persistence mode: session config has no `ephemeral` flag, session entries do not carry an ephemeral state, transcript/session-store writes still persist files and metadata, memory flush still runs from the normal compaction pipeline, and plugin hook context does not expose an `isEphemeral()`-style contract. The related #62436 is useful context for stateless HTTP passthrough, but it does not supersede the broader secure chat/session privacy request.

## Best Possible Solution

Keep the issue open for a core design/implementation pass. The likely path is an additive `ephemeral` session contract in config/session state, hard gates in transcript/session-store/compaction/memory-flush paths, a documented plugin SDK/session context signal for persistence-aware plugins, and UI/command/status surfaces to verify the mode is active.
