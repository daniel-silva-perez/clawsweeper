## Summary

Keep open. Current main still lacks a hard per-session ephemeral/no-persistence mode: session config, session state, Gateway session RPC schemas, transcript creation/append paths, memory flush, and plugin/context-engine contracts do not carry a durable privacy flag or no-write gate. The related #62436 is narrower HTTP passthrough work, not a replacement for secure chat across all channels and session/plugin layers.

## What This Changes

Request a per-session secure/ephemeral chat mode that prevents transcripts, session metadata, compaction summaries, memory flushes, and plugin indexes from being persisted.

## Best Possible Solution

Keep this as a core design item for an additive ephemeral session contract across config, Gateway protocol, runtime SessionEntry, session creation/patch flows, transcript/session-store persistence, memory flush, compaction/context-engine hooks, plugin SDK visibility, command/UI/status verification, and tests proving no durable transcript, index, memory file, compaction summary, or surviving metadata is created for ephemeral sessions.

## Reproduction Assessment

Not applicable as a feature request, but current-main verification is high-confidence. The focused source checks show the requested flag cannot be configured or passed through the session RPC surface, and the normal creation/append/store paths still write durable state.

## Solution Assessment

Unclear for implementation details. The requested direction is the right product boundary, but the best exact fix needs maintainer design because it spans core session contracts, plugin APIs, memory/compaction behavior, channel defaults, and user-visible verification.
