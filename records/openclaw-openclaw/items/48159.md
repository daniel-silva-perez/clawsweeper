## Summary

Keep this issue open. Current main still does not implement a hard per-session ephemeral/no-persistence mode: there is no session config/state/protocol field, normal session creation and transcript paths still write durable `sessions.json` and JSONL files, memory flush still uses the normal persistence pipeline, and plugin hook/session context has no `isEphemeral()` or persistence-policy signal. The related #62436 remains narrower and open; it tracks stateless HTTP passthrough, not secure chat across all channels/session/plugin layers.

## Best Possible Solution

Keep this issue open for a core design and implementation pass. The best path is an additive `ephemeral` session contract in config, Gateway protocol, runtime `SessionEntry`, and session creation/patch flows; hard no-write gates in transcript/session-store/compaction/memory-flush paths; a documented plugin SDK/session-context persistence signal; command/UI/status verification; and focused tests proving that ephemeral sessions do not create durable transcripts, searchable memory/index entries, memory files, compaction summaries, or surviving session metadata. Treat #62436 as a narrower HTTP passthrough companion, not the canonical replacement.
