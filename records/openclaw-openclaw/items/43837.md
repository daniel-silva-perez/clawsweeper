## Summary

Keep this PR open. Current main still does not implement the requested one-shot `/think <level> <body>` behavior: there is no one-shot directive field/parser, `/think` with arguments is still classified as a control command, mixed directive+body messages clear the parsed think directive, and thinking resolution has no non-persistent one-shot fallback. The linked feature issue was already treated as superseded by this open PR, so the best cleanup path is maintainer review and landing or replacing this implementation, not closing it.

## Best Possible Solution

Keep the PR open as the current implementation candidate for one-shot `/think <level> <body>`. The best path is to review and land this PR, or replace it with a maintainer-owned patch that adds an explicit non-persistent one-shot think field, routes one-shot `/think` bodies through the reply path, preserves Telegram multiline bodies, keeps directive-only command chains on the command path, and adds the proposed regression coverage.
