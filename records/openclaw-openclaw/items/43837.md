## Summary

Keep this PR open. Current main still does not implement the PR's central one-shot `/think <level> <body>` behavior: there is no non-persistent one-shot think field, leading `/think` messages with arguments still match the control-command path, mixed directive+body parsing clears the parsed think directive, and Telegram command parsing still uses the newline-truncating command body.

## Best Possible Solution

Keep the PR open as the current implementation candidate. The best path is to review and land this PR, or replace it with a maintainer-owned patch that adds an explicit non-persistent one-shot think field, routes leading `/think <level> <body>` through the reply path, preserves Telegram multiline bodies, keeps directive-only chains on the command path, and adds focused regression coverage for session non-pollution and channel command gating.
