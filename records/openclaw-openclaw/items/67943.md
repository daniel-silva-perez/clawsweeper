## Summary

Keep open. The linked same-author implementation PR is still open and unmerged, and current main still contains the source path that can reuse persisted subagent lineage for an ordinary run before `sessions_spawn` computes depth from stored session metadata.

## What This Changes

The issue asks OpenClaw to prevent normal top-level sessions from inheriting stale subagent lineage metadata that can incorrectly trigger `sessions_spawn` depth-limit failures.

## Best Possible Solution

Maintain the current open issue until #67946 or an equivalent maintainer-approved fix lands, with code that clears subagent lineage fields on ordinary top-level session turns, preserves lineage only for real spawned/ACP sessions, stops normal runs from using stale stored `spawnedBy`, and adds regression coverage for the stale-lineage scenario.
