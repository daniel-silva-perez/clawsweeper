## Summary

Keep open. Current main still resolves directory matches with the initially inferred kind instead of the matched directory entry kind, so this PR’s intended fix is not implemented on main. The PR is focused and plausibly addresses a real routing bug, but the latest `channel` to `group` normalization should get maintainer review before merge because current contracts and routing code distinguish those kinds.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land the focused resolver/test fix after deciding whether directory entries with `kind: "channel"` should be preserved as `channel` or intentionally normalized to `group`; current main does not already implement the central change.
