## Summary

Keep this issue open. Current main has fixed several bullets from the audit, including breadcrumbs, masked gateway token entry, invalid-config warning, usage states, and session delete confirmation, but the full dashboard UX audit is still not implemented. Meaningful requested work remains in Instances, Sessions, Config action semantics, keyboard/help shortcuts, and dashboard-wide action feedback. The related #39780 remains a narrower Config hardening tracker, not a replacement for this full Control UI audit.

## Best Possible Solution

Keep this issue open unless a maintainer intentionally splits it. The best path is to create or land focused follow-ups for Instances per-instance actions and clearer presence affordances, Sessions key copy/truncation and less repetitive advanced controls, clearer Config action labels/help, dashboard shortcut/help UX, and save/delete/apply feedback or activity history. Treat #39780 as only the narrower Config hardening follow-up.
