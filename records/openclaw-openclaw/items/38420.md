## Summary

Keep open. Current main addresses several individual UX audit bullets, including breadcrumbs, token masking, invalid-config bannering, config diffs, usage loading/empty/error states, and session-delete confirmation. The full Control UI UX audit is still not implemented: Instances remains list/refresh-only, Sessions still lacks a copy affordance and keeps dense per-row advanced selects, Config still exposes ambiguous Save/Apply/Update actions without explanatory labels/tooltips, global shortcuts are limited to the command palette, and there is no general toast or action audit trail for config/session/update operations. The related #39780 remains a narrower Config hardening request, not a replacement for this dashboard-wide audit.

## Best Possible Solution

Keep this open unless a maintainer intentionally splits it. The best path is focused follow-up work for Instances per-instance actions and clearer presence/reason affordances, Sessions key copy/truncation plus less repetitive advanced controls, clearer Config action labels/help, dashboard shortcut/help UX, and durable action feedback or activity history. Treat #39780 as a narrower Config hardening tracker only.
