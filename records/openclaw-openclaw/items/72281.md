## What This Changes

This PR changes `scripts/install.ps1` so the custom PowerShell `Write-Host` wrapper discards the inner `Microsoft.PowerShell.Utility\Write-Host` command result with `[void]`.

## Best Possible Solution

Land the one-line installer wrapper guard if the focused install-ps1 test, the `core-support-boundary` shard, build, and Windows/PowerShell validation pass on the rebased branch. Keep any follow-up inside `scripts/install.ps1` unless validation proves a broader installer output-handling issue.

## Reproduction Assessment

Yes. The linked report names the failing `core-support-boundary` CI surface, and current main has a focused fixture that checks `Main` does not return an array or non-true value when npm chatter is present; this read-only review did not run artifact-writing tests.

## Solution Assessment

Yes, with the Windows validation caveat. Making the custom wrapper side-effect-only is the narrowest maintainable fix for the reported wrapper leak and does not alter installer control flow; if the direct npm output writes are the proven source, guarding those output sites is the safer follow-up.
