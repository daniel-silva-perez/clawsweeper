## Summary

Current main still has the Signal daemon shutdown race this PR targets. The Signal daemon handle still exposes synchronous `stop`, `spawnSignalDaemon().stop()` only sends SIGTERM, monitor teardown does not await the child process exit, and the linked bug #22676 remains open with this PR as the direct implementation candidate.

## Best Possible Solution

Keep this PR open for maintainer review and CI. The best path is the Signal-owned fix proposed here, or an equivalent focused patch: make daemon `stop()` idempotent and async, send SIGTERM, wait for child exit, escalate to SIGKILL after a bounded timeout, await cleanup from `monitorSignalProvider`, and keep regression coverage for stop wait/escalation plus aborted monitor shutdown. No core/plugin API change appears necessary.
