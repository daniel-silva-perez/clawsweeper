## Summary

Current main still does not implement this PR's intended Signal daemon shutdown fix. `SignalDaemonHandle.stop` remains synchronous, `spawnSignalDaemon().stop()` only sends SIGTERM and returns, and `monitorSignalProvider` still calls daemon cleanup without awaiting child-process exit. The PR remains directly relevant to the linked open bug #22676, and its diff is limited to the Signal plugin daemon/monitor paths plus regression tests.

## Best Possible Solution

Keep this PR open for maintainer review and CI. The best implementation path is the Signal-owned fix proposed here: make `SignalDaemonHandle.stop` idempotent and async, wait for signal-cli child exit after SIGTERM, escalate to SIGKILL after a bounded timeout, await that cleanup from `monitorSignalProvider`, and keep focused regression coverage for daemon stop wait/escalation plus aborted monitor shutdown. No core/plugin API change appears necessary for this specific bug.
