## Summary

Keep this PR open. Current main still does not implement the PR's intended media-runner behavior: the runner creates an env-proxy fetch before it knows the configured audio/video backend baseUrl, then passes that fetch into OpenAI-compatible multipart STT. Main has shared target-aware NO_PROXY helpers, but the media runner is not wired to them and the current runner tests still assert env-only fetch selection.

## Best Possible Solution

Keep this PR open and adapt it to current main: resolve the media provider execution context before choosing the fetch function, reuse or deliberately extend `src/infra/net/proxy-env.ts` for target-aware proxy decisions, decide CIDR NO_PROXY semantics explicitly, and add focused runner/proxy tests for a NO_PROXY-covered self-hosted audio backend plus private/internal backends that must remain proxied when not excluded.
