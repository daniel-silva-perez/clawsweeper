## Summary

Keep open. Current main still returns `doctor.memory.status` for the active default agent only, and the current docs describe that default-agent scope. The PR’s intended `perAgent` payload is not implemented on main, so this PR is not obsolete; it needs maintainer review/rebase, especially around the existing review finding that a secondary-agent manager acquisition error could reject the whole status call.

## Best Possible Solution

Keep this PR open and have maintainers review a rebased revision that adds `perAgent` to `doctor.memory.status`, documents the additive response field, and handles each non-default agent’s manager acquisition/status errors independently. If maintainers decide the related closed feature request should stay closed, this PR should still be explicitly landed or closed on its own merits because main does not yet contain the proposed RPC field.
