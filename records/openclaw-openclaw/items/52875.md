## Summary

Keep open. Current main still requires an existing session row for `sessions.send`, does not register a `sessions.start` or `sessions_start` surface, and does not return the discussed delivery metadata on `sessions_send` timeouts. Open PR #52921 is still the implementation candidate linked with closing syntax, so this report should stay open until maintainers review, land, replace, or close that PR.

## What This Changes

The issue asks for agent-to-agent `sessions_send` to recover when a configured target agent has no active session, with an explicit/idempotent session start path and clearer delivery or timeout status.

## Best Possible Solution

Review PR #52921 as the current implementation candidate. If it is correct and safe, land it and let the linked report close through GitHub; otherwise keep this as the canonical report and implement a focused replacement covering idempotent session start, `sessions_send` auto-bootstrap for known agent configs, structured not-found errors, accepted/timeout delivery metadata, regression tests, and docs updates.
