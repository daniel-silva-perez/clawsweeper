## Summary

Keep this open. Current main still does not implement the reported recovery path: `sessions.send` still requires an existing session entry, no `sessions.start`/`sessions_start` surface is registered, and the agent `sessions_send` timeout path still lacks the delivery metadata discussed in the thread. The provided GitHub context shows open PR #52921 explicitly closing this issue, so the issue should remain open until that PR is reviewed, landed, replaced, or closed.

## What This Changes

The issue asks for `sessions_send` to recover when a configured target agent has no active session, ideally through an idempotent session-start/prewarm path plus clearer delivery metadata for accepted and timeout results.

## Best Possible Solution

Review PR #52921 as the current implementation candidate. If it is correct and safe, land it and let the issue close through that linked PR; otherwise keep this issue as the canonical report and implement a focused replacement covering idempotent session start, `sessions_send` auto-bootstrap for known agent configs, structured not-found errors, delivery metadata on accepted/timeout paths, regression tests, and docs updates.
