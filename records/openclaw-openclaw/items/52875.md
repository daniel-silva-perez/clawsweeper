## Summary

Current main still does not fully implement the reported missing-session recovery path. `sessions.send` still requires an existing session row, the gateway/protocol surfaces still have no `sessions.start`, the agent tool registry still has no `sessions_start`, and agent-facing `sessions_send` still errors when target label resolution fails. The provided discussion also shows open PR #52921, plus a fork copy, explicitly using closing syntax for this issue, so the issue should remain open until maintainers review, land, or close that implementation candidate.

## Best Possible Solution

Keep this issue open while maintainers review PR #52921. If the PR is correct and safe, land a version that adds an idempotent `sessions.start`/`sessions_start`, `sessions_send` auto-bootstrap for known agent configs, structured not-found errors, accepted/timeout delivery metadata, focused regression tests, and matching docs/tool-surface updates. If that PR is rejected, keep this issue as the canonical report for implementing that recovery path.
