## Summary

Keep open. Current main still has the reported Windows ACL classification gap, and open PR #74383 is already the paired implementation candidate, so this issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

The issue requests treating Anonymous Logon, Guests, Interactive, Local, and Network Windows ACL principals/SIDs as world-equivalent so security audits report affected paths as world-writable rather than group-writable.

## Best Possible Solution

Review and land a narrow security patch, likely through #74383, that expands the Windows world-equivalent SID/principal policy after maintainer confirmation and locks it with classifier plus state-dir audit regression tests.

## Risks / Open Questions

- The proposed list mixes unauthenticated principals with broad contextual logon groups such as Interactive, Local, and Network; maintainers should confirm the exact world-equivalent policy before merging.

- Some audit consumers already treat group and world together, so the fix should include severity-sensitive filesystem regression coverage, not only classifier unit tests.
