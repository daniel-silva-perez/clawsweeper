## Summary

Keep open. Current main still has the reported Windows ACL classification gap, and open PR #74383 is the paired closing implementation candidate, so this security issue should remain open until that PR or an equivalent patch is reviewed and merged.

## What This Changes

Treat Anonymous Logon, Guests, Interactive, Local, and Network Windows SID/principal entries as world-equivalent in ACL security audit classification.

## Best Possible Solution

Review PR #74383's SID/principal policy and land a narrow Windows ACL fix with classifier and filesystem-severity regression coverage; close this issue only after that patch or an equivalent fix is on main.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed summarizeWindowsAcl or inspectPathPermissions an ACL entry like `*S-1-5-7:(F)`; current main classifies that SID as group, which maps to groupWritable rather than worldWritable.

## Solution Assessment

Unclear until maintainer security policy review. PR #74383 is the best current vehicle, but maintainers should confirm the exact broad-principal set before landing it.
