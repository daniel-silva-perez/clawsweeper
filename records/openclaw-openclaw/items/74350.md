## Summary

Keep open. Current main still has the reported Windows ACL classification gap, and open PR #74383 is already the closing implementation candidate, so this issue should remain open until that security patch is reviewed and merged or closed.

## What This Changes

The issue asks OpenClaw to classify broad Windows ACL principals such as Anonymous Logon, Guests, Interactive, Local, and Network as world-equivalent for security audit severity.

## Best Possible Solution

Review and land #74383, or an equivalent narrow security patch, after secops confirms the world-equivalent SID/principal policy; keep the fix limited to Windows ACL classification plus focused classifier and filesystem-audit regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: an ACL entry such as `*S-1-5-7:(F)` is not in WORLD_SIDS, so classifyPrincipal falls through to group and inspectPathPermissions reports groupWritable rather than worldWritable.

## Solution Assessment

Yes, the paired PR direction is the narrowest maintainable fix: expand the Windows world-equivalent SID/principal policy in src/security/windows-acl.ts and lock the severity path with targeted tests. The only unresolved part is secops confirmation of the exact broad-principal list.
