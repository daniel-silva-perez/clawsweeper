## Summary

Keep open. Current main still has the reported Windows ACL classification gap, and open PR #74383 is the paired closing implementation candidate, so this issue should remain open until that security patch is reviewed and merged or closed.

## What This Changes

The issue requests classifying Anonymous Logon, Guests, Interactive, Local, and Network Windows SIDs/principal names as world-equivalent in ACL security auditing so write access is not downgraded from worldWritable to groupWritable.

## Best Possible Solution

Review PR #74383's SID/principal policy, land a narrow Windows ACL fix if approved, and keep regression coverage at both classifier and filesystem-severity layers; close this issue only after that PR or an equivalent merged fix is on main.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed summarizeWindowsAcl or inspectPathPermissions an ACL entry like *S-1-5-7:(F); because S-1-5-7 is absent from WORLD_SIDS, current main classifies it as group and downstream audit sets groupWritable rather than worldWritable.

## Solution Assessment

Unclear until maintainer security policy review. PR #74383 is the best current vehicle, but maintainers should decide whether all proposed broad logon groups belong in the world-equivalent policy and require classifier plus state-dir regression tests.
