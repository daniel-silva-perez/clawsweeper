## Summary

Keep open. Current main still leaves the embedded Pi SDK retry path enabled, and this issue is paired with open closing PRs, so it should stay open until maintainers land one canonical fix or close the candidate PRs.

## What This Changes

The issue reports a Qwen/Feishu regression where a failed tool call is replayed by the runtime instead of letting the agent reason and choose a new action, causing repeated tool calls and runaway context growth.

## Best Possible Solution

Land one canonical, narrow runtime fix that prevents the embedded Pi auto-retry layer from hiding the failed assistant turn before OpenClaw's outer retry/failover layer runs, with regression coverage for a failed `exec` result followed by a changed or no-op next assistant action. Treat broader default loop-detection or thinking-repetition policy as a separate maintainer decision.

## Reproduction Assessment

Unclear. The report gives a concrete failed-`exec` replay pattern and the dependency code makes the root cause plausible, but this read-only pass did not run the live Qwen/Feishu path and current tests do not cover that exact replay sequence.

## Solution Assessment

Unclear. Disabling the embedded Pi SDK auto-retry looks like the narrowest likely root-cause fix because OpenClaw already owns outer retry/failover, while a broader thinking-repetition guard may be useful but should not substitute for preserving failure context.
