## Summary

Keep open. Current main does not already implement observe-first bundled MCP runtime guardrails, and the PR is useful directionally, but it is not merge-ready because it adds a supported `mcp.runtimeGuardrails` config key without the matching config help, labels, public configuration reference, and config-doc baseline alignment required by the repo contract.

## What This Changes

The PR adds an observe-only bundled MCP runtime guardrails module and tests, wraps bundled MCP tool calls with circuit-breaker and budget observation, exposes runtime snapshots, and adds `mcp.runtimeGuardrails` config types/schema/generated schema output.

## Best Possible Solution

Land the observe-only runtime seam only after the public config contract is complete: add field help/labels, document `mcp.runtimeGuardrails` in the configuration reference, regenerate the config baseline hash, and keep fail-closed enforcement as a separate reviewed change.
