## Summary

Keep open. Current main and the latest release still allow audio auto-detect to fall through to Gemini CLI, and the public audio docs still advertise Gemini CLI for audio. The PR is a narrow relevant defensive fix, but it needs maintainer review and a docs refresh against the current audio auto-detect wording before merge.

## What This Changes

The PR adds an audio-only guard to Gemini CLI auto-detection, adds regression coverage for audio falling through to provider transcription when a `gemini` binary is present, and updates audio docs to explain that Gemini CLI remains auto-detected only for image/video.

## Best Possible Solution

Land a rebased equivalent of the audio-only guard so auto-detect never selects Gemini CLI for audio unless explicitly configured, while preserving Gemini CLI auto-detect for image/video and keeping audio docs aligned with the current provider/local fallback order.
