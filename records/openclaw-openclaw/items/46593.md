## Summary

Keep open. Current main and the latest release still auto-resolve Gemini CLI for audio after provider/local audio backends fail, so this PR’s defensive skip is not implemented. The PR is useful but currently dirty against main and needs maintainer review or a rebase before it can land.

## What This Changes

The PR skips Gemini CLI auto-detect for audio, adds regression coverage around a `gemini` binary on `PATH`, and updates audio docs to say Gemini CLI remains auto-detected only for image/video.

## Best Possible Solution

Land a rebased equivalent of the audio-only guard so auto-detect never selects Gemini CLI for audio unless explicitly configured, while preserving Gemini CLI auto-detect for image/video and keeping docs aligned with the current provider/local fallback order.
