## Summary

Keep open. Current main and v2026.4.27 still let audio auto-detection fall through to Gemini CLI, so this PR is not obsolete; the branch is a narrow valid defensive fix, but its docs hunk needs refresh against the current audio auto-detect table before merge.

## What This Changes

The PR adds an audio-only guard that skips Gemini CLI auto-detection for media-understanding audio flows, adds regression coverage for provider transcription when a `gemini` binary is present, and updates audio docs to explain the audio exception.

## Best Possible Solution

Land a rebased equivalent of the audio-only guard so auto-detect never selects Gemini CLI for audio unless explicitly configured, while preserving Gemini CLI auto-detect for image/video and keeping the public audio docs aligned with the current provider/local fallback order.
