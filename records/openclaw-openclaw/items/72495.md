## Summary

Keep this PR open. Current main still omits `ctimeMs` from the guarded workspace bootstrap file cache identity, and the merged session-refresh path still relies on that lower-level cache for unchanged file reads. The PR remains a focused, useful fix, but it is currently dirty against main and needs rebase or replacement before normal merge review.

## What This Changes

The PR adds `stat.ctimeMs` to the guarded workspace bootstrap file cache identity, updates the identity comment, adds same-inode/same-size/restored-mtime regression coverage, and adds a changelog entry.

## Best Possible Solution

Land this PR or an equivalent rebased maintainer patch that adds `ctimeMs` to the guarded workspace file identity, keeps focused same-inode/same-size/restored-mtime coverage, and preserves the changelog credit without broadening the bootstrap cache design.

## Reproduction Assessment

Yes. The source-level reproduction is clear: on current main, load a workspace `AGENTS.md`, rewrite it in place with equal-size content, restore the original `mtime`, and the cache identity can still match because it lacks `ctimeMs`; the PR adds a focused regression for that path.

## Solution Assessment

Yes. Adding `ctimeMs` to the existing identity is the narrowest maintainable fix for this cache gap; hashing every read or removing the lower cache would be heavier and would reopen the performance tradeoff called out around #72406.
