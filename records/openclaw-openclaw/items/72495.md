## Summary

Keep this PR open. Current main still keys the guarded workspace bootstrap file cache by canonical path, dev, inode, size, and mtime only, so the ctime hardening proposed here remains a focused, unsuperseded follow-up to the merged #72406/#64871 session-refresh work. The PR’s security-sensitive surface is narrow and the earlier Greptile test-flake concern appears addressed by the latest PR patch in the provided context.

## Best Possible Solution

Review and land this PR, or an equivalent maintainer patch, that adds `stat.ctimeMs` to `workspaceFileIdentity`, updates the identity comment, and keeps focused regression coverage for same-inode, same-size edits with restored mtime. After merge, this PR can be closed as implemented by the landed commit.
