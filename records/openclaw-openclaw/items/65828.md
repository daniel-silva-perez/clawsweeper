## Summary

Keep this PR open. Current main still has the MiniMax web-search metadata/onboarding mismatch: provider metadata omits MINIMAX_API_KEY and onboardingScopes, while the runtime resolver already accepts MINIMAX_API_KEY. The PR is a focused contributor fix, but it still needs maintainer judgment on whether generic MiniMax API keys should trigger search auto-detection when search requires Token Plan access.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. Decide the intended MiniMax search credential policy first: accept MINIMAX_API_KEY auto-detection with clear Token Plan docs, require explicit MiniMax Search configuration, lower or constrain the auto-detect path, or add a search-capability check if MiniMax exposes one. Then land the metadata/docs alignment with focused tests for setup visibility and the selected auto-detection behavior.
