## Summary

Keep this issue open. Current main does implement `agents.defaults.compaction.model` for explicit `/compact` and context-engine compaction runtime contexts, and that shipped in the changelog, but the safeguard hook itself still prefers `ctx.model`, while the embedded extension runtime is still populated from the active session model. The linked PR was closed unmerged, and the remaining Pi auto-compaction path is close enough to the reported bug that this should not be swept closed as implemented.

## Best Possible Solution

Keep the issue open and fix the remaining safeguard auto-compaction path explicitly. The likely maintainer path is to resolve the effective compaction target before registering the safeguard runtime, make the hook prefer the configured compaction model when present, and add a regression where `ctx.model` is the session model while the configured `agents.defaults.compaction.model` is different.

## Risks / Open Questions

- A narrow one-line precedence swap may not be sufficient on current main if the safeguard runtime still receives the active session model; the fix should verify both the model precedence and the value placed in runtime state.
