## Summary

Keep this issue open. Current main still has the safeguard hook choosing `ctx.model` before `runtime.model`, and the active embedded runner still seeds the safeguard runtime from the active session model. Current main does implement `agents.defaults.compaction.model` for manual/context-engine runtime contexts, but that is only partial coverage and does not close the reported safeguard auto-compaction bug.

## Best Possible Solution

Fix the remaining safeguard auto-compaction path explicitly: resolve the effective compaction target before seeding the safeguard runtime, make the safeguard hook prefer the configured compaction model when present, and add a regression where `ctx.model` is the session model while `agents.defaults.compaction.model` points at a different provider/model.
