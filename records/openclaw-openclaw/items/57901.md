## Summary

Keep this issue open. Current main still matches the reported safeguard compaction bug, and the provided GitHub context shows an open same-author PR (#73704) with closing syntax that is the active implementation candidate.

## Best Possible Solution

Keep this issue open and review/land #73704, or an equivalent maintainer patch, that resolves the configured compaction provider/model/auth before registering the safeguard runtime, makes the hook prefer the resolved runtime model when present, and adds regression coverage where the session model differs from `agents.defaults.compaction.model`.
