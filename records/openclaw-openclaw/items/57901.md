## Summary

Keep this issue open. Current main still has the safeguard-specific model-selection path that the report describes, and an open same-author PR (#73704) explicitly tracks the intended fix with closing syntax, so the right maintainer action is to review or land that PR rather than close the issue early.

## Best Possible Solution

Keep the issue open and review/land #73704, or an equivalent maintainer patch, that resolves the effective compaction target before registering the safeguard runtime, makes the hook prefer the configured runtime model when present, and adds regression coverage where the session model and configured compaction model differ.
