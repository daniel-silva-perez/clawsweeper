## Summary

Keep open. Current main has a partial dependency-side mitigation for duplicated unmodified Kitty printable input, but the central AltGr failure is still present in the TUI input path: OpenClaw delegates editor input to pi-tui, and pi-tui 0.70.5 still rejects printable CSI-u events carrying Alt/Ctrl modifier bits before the editor fallback can insert them.

## Best Possible Solution

Keep this open and fix it in the TUI input layer, preferably upstream in pi-tui or via a narrow OpenClaw adapter while waiting for upstream. The fix should treat AltGr-produced printable CSI-u sequences as text without breaking real Alt/Ctrl shortcuts, then add focused regression coverage for German-layout @/€ and duplicated printable Kitty events before bumping or patching the dependency.

## Risks / Open Questions
