## Summary

Keep open. Current main still pins @mariozechner/pi-tui to 0.70.5, OpenClaw still delegates TUI editor input to pi-tui with only a local backspace deduper, and the pi-tui 0.70.5 keyboard decoder still rejects printable Kitty/CSI-u events carrying Alt or Ctrl modifier bits. That leaves the reported AltGr @/€ path unresolved, even if duplicate unmodified printable input may have been partly mitigated upstream.

## Best Possible Solution

Keep this open and fix the TUI input layer, ideally upstream in pi-tui or through a narrow OpenClaw adapter while waiting for upstream. The implementation should decode AltGr-produced printable CSI-u sequences as text without breaking real Alt/Ctrl shortcuts, then add focused regression coverage for German-layout @/€ and duplicated printable Kitty events before bumping or patching the dependency.
