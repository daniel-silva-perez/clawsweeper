## Summary

Keep this PR open. Current main still has the simpler Android chat implementation: sessions are filtered to a 24-hour recent window, the chat UI uses horizontal session chips, the Android view model and chat models lack the timeline/status/defaults/catalog state proposed by the PR, and the controller has no Android-side create/delete chat flow. The PR is recent, open, and the author responded to review findings, so the right maintainer path is Android/gateway review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for Android/gateway maintainer review. The concrete path is to review the current PR head, verify the operator.admin elevated delete flow and session-title sanitization, run the focused Android regression pack plus the appropriate changed gate, and either land a coherent parity slice or ask the author to split the broad UI/session/auth changes into smaller reviewable PRs.
