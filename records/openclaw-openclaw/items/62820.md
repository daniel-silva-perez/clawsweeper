## Summary

Keep open. Current main has adjacent assistant-visible stripping and diagnostics for plain-text tool-call output, but the specific sanitizer path named in this report is not fully fixed: `sanitizeUserFacingText` still strips only `<final>` tags, delivery callers still depend on it, and streaming block replies appear to bypass the XML stripper before channel delivery.

## Best Possible Solution

Keep this issue open and land a focused fix that applies the shared assistant-visible/tool-call XML stripping at every user-facing delivery boundary, especially `sanitizeUserFacingText` and streaming block replies. Add regression tests for the reported MiniMax `<minimax:tool_call><invoke ...>` leak, dangling tool XML, and the channel/block-reply path, while preserving the current policy that plain text should not be converted into executable tool calls.
