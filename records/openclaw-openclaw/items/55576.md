## Summary

Keep open. Current main still does not add `google-vertex` to the config `MODEL_APIS` or the generated JSON schema enums, so this PR's stated config-validation change is not implemented on main. Main does have partial Google Vertex support through the bundled Google plugin and pi-ai already registers a `google-vertex` stream, but the PR also needs the reviewer-noted runtime policy follow-through before it should be merged.

## Best Possible Solution

Keep this PR open, but do not merge the three-line enum/schema change by itself. The maintainable path is to add `google-vertex` to `MODEL_APIS` and regenerate the schema, then update the Google runtime policy seam so `google-vertex` receives the same transcript sanitization, turn-order validation, and payload handling as the other Google transports. Add targeted config-schema and transcript-policy coverage, and update provider docs for the API-key/ADC behavior.
