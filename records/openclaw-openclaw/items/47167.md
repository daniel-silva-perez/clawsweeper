## Summary

Current main still does not provide the requested cross-channel Plugin SDK hook for passively observing group messages before `requireMention` drops them. `message_received` is still emitted after channel ingress reaches reply dispatch, and affected channels still have local no-mention return paths. Telegram has a partial internal ingest hook, and several channels store pending history, but those are not a supported full-fidelity plugin archive API.

## Best Possible Solution

Keep this open and design a documented Plugin SDK seam for read-only early inbound observation. It should fire before mention gating drops, carry channel/account/conversation/thread/sender/message metadata, avoid creating sessions or replies by default, define privacy/opt-in behavior clearly, and add regression coverage for the affected bundled plugins.

## Risks / Open Questions

- Adding a full-fidelity early archive hook can expose group messages that intentionally did not trigger the bot, so the design should be opt-in, privacy-explicit, and read-only by default.
- The implementation is cross-channel: fixing only one bundled plugin would leave archive plugins with inconsistent behavior across the affected channels.
