## Summary

Current main still lacks the requested generic, supported plugin API for observing no-mention group messages before `requireMention` skips them. `message_received` remains emitted from reply dispatch after channel ingress succeeds, while Discord, Slack, LINE, iMessage, Telegram, and BlueBubbles still have channel-local early exits for skipped group messages. Telegram has a partial internal `message:received` ingest path, and several channels now record pending history, but those do not provide the reporter's full-fidelity archive plugin with a cross-channel Plugin SDK hook.

## Best Possible Solution

Keep this issue open and design a documented Plugin SDK seam for read-only early inbound observation. It should fire before mention-gating drops, carry channel/account/conversation/thread/sender/message metadata, be explicit about privacy and opt-in behavior, avoid sessions or replies by default, and add regression coverage across the affected bundled plugins.

## Risks / Open Questions

- An early inbound hook needs a privacy-aware, read-only contract so opted-in archive plugins can observe skipped group chatter without creating sessions, triggering replies, or growing model context by default.
- Pending-history storage is short-window context, not a durable full-fidelity archive API; treating it as the fix would still leave third-party archive plugins patching channel ingress paths.
- The hook should be generic across bundled and third-party channel plugins rather than adding more channel-specific internal hooks like Telegram's current ingest path.
