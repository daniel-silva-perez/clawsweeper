## Summary

Keep open. Current main has improved some mention-skipped group handling by storing pending history for later context, and Telegram has a channel-specific internal-hook ingest path, but the reported plugin API gap remains: skipped no-mention group messages still return from channel ingress before the normal plugin `message_received` hook in the reply dispatch pipeline. The reporter’s public `@dashhuang/conversation-archive` plugin is useful context: standard mode can archive messages that reach official hooks, but full-fidelity archive mode still needs a core/plugin API seam for early-dropped inbound messages.

## Best Possible Solution

Keep the issue open and design a generic, documented, read-only early inbound observation hook or equivalent Plugin SDK seam. It should fire before mention-gating drops, carry enough channel/account/conversation/sender/message metadata for archive plugins, avoid creating sessions or replies by default, and include tests across at least the affected bundled channels so future mention-gating changes preserve plugin visibility.

## Risks / Open Questions

- A new early inbound hook needs a careful read-only/privacy-aware contract so skipped group chatter becomes observable to opted-in plugins without accidentally triggering replies, sessions, or model context growth.
- Current pending-history behavior is channel-local and not a durable full-fidelity archive; treating it as the requested plugin API would leave third-party archive plugins patching channel ingress paths.
