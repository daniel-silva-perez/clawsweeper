## Summary

Keep open. Current main still lacks a supported cross-channel Plugin SDK hook for passively observing group messages before `requireMention` drops them; Telegram now has a partial opt-in internal hook, but Discord, Slack, LINE, iMessage, and BlueBubbles still return before Plugin SDK `message_received`, and the reporter's archive plugin still needs a patch-free public seam for full-fidelity mode.

## What This Changes

The issue requests a read-only early inbound plugin hook so archive/history plugins can observe group messages skipped by `requireMention` without creating sessions or replies.

## Best Possible Solution

Design and document a Plugin SDK early inbound observation seam that fires before mention gating for supported channels, is read-only, does not create sessions or replies by default, includes channel/account/conversation/thread/sender/message metadata, has explicit privacy opt-in semantics, and adds regression coverage for the affected bundled plugins.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: configure a group channel with `requireMention: true`, send a non-mention message, and the affected channel ingress code returns before `dispatchReplyFromConfig` can emit Plugin SDK `message_received`.

## Solution Assessment

Unclear pending maintainer design. The reporter's requested read-only early hook is the right direction, but the best implementation needs a deliberate public SDK contract and privacy model rather than ad hoc channel-local patches.
