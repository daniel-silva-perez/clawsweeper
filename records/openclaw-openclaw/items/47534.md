## Summary

Keep open. Current main still lacks a doctor diagnostic or repair path for persisted `skillsSnapshot` metadata that points at stale bundled-skill runtime roots. The existing doctor state-integrity path checks session-store/transcript drift, while the agent and gateway reply paths still reuse persisted skill snapshots based on missing/version/filter checks rather than validating cached path contents. The report includes concrete failure details and a safe field-level remediation, so this is actionable rather than stale.

## What This Changes

Add an `openclaw doctor` diagnostic and safe remediation for persisted session skill snapshots that still reference old, missing, inactive, or temporary runtime roots after OpenClaw moves or upgrades.

## Best Possible Solution

Add a focused doctor state diagnostic that resolves the active bundled skills root, scans persisted session `skillsSnapshot.prompt` and `skillsSnapshot.resolvedSkills` path fields for inactive, missing, or temporary bundled runtime roots, reports the problem as stale cached session metadata while saying the live runtime root is healthy, and in repair mode backs up the session store then removes only affected `skillsSnapshot` fields so the next run rebuilds them. Update doctor docs and the active changelog with reporter/context attribution.

Expected behavior:
- Resolve the active bundled skills root using existing skills/runtime helpers; do not hard-code install roots or bundled ids.
- Scan persisted session entries for `skillsSnapshot.prompt` and `skillsSnapshot.resolvedSkills` path-bearing fields such as `filePath`, `baseDir`, and source-info path data.
