## Summary

Keep open. Current main still has the reported gap: `openclaw doctor` checks session-store/transcript integrity, but it does not inspect persisted `skillsSnapshot` prompt or `resolvedSkills` paths for stale bundled runtime roots, temp-backed install roots, or missing roots. The issue discussion includes concrete stale-path examples and a verified narrow remediation, and the adjacent draft PR #71497 is about snapshot refresh versioning rather than the requested doctor diagnostic.

## What This Changes

The issue requests an `openclaw doctor` diagnostic and safe remediation for persisted session `skillsSnapshot` metadata that still references stale runtime roots after installs or runtime paths change.

## Best Possible Solution

Add a focused doctor state diagnostic that resolves the active bundled skills directory, scans persisted session `skillsSnapshot.prompt` and `skillsSnapshot.resolvedSkills[].filePath` values for missing, inactive, or temp-backed bundled runtime roots, reports cached-session metadata drift separately from live runtime health, and in repair mode backs up the session store before deleting only affected `skillsSnapshot` fields.
