## Summary

Keep open. Current main still reuses persisted skill snapshots based on missing/version/filter checks and `openclaw doctor` state-integrity checks sessions/transcripts, but there is no doctor diagnostic or repair path for persisted `skillsSnapshot` metadata that points at stale bundled-skill runtime roots. The report includes concrete failure context and a safe field-level remediation, so it is actionable rather than stale.

## What This Changes

`openclaw doctor` should detect and optionally repair persisted session skill snapshots that reference stale bundled-skill runtime paths after install or runtime-root changes.

## Best Possible Solution

Add a focused `openclaw doctor` diagnostic that resolves the active bundled skills root, scans persisted session `skillsSnapshot.prompt` and `skillsSnapshot.resolvedSkills` for inactive, missing, or temporary bundled runtime roots, reports the problem as stale cached session metadata while distinguishing live runtime health, and in repair mode backs up the session store then removes only affected `skillsSnapshot` fields so the next turn rebuilds them. Update doctor docs, targeted tests, and the active changelog with reporter/context attribution.
