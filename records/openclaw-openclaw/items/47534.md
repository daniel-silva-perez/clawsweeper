## Summary

Keep open. Current main still reuses persisted skill snapshots based on snapshot version and skill-filter state, and `openclaw doctor` state integrity does not inspect cached `skillsSnapshot.prompt` or `resolvedSkills[].filePath` for inactive, missing, or temp-backed bundled runtime roots. The issue has concrete reproduction context and a verified narrow remediation, so it is actionable rather than stale. The open draft PR #71497 is adjacent restart-version work, but it does not add the requested doctor diagnostic or runtime-root drift repair.

## What This Changes

Request `openclaw doctor` to detect and safely remediate cached session skill snapshots that still reference stale bundled runtime roots after install or runtime path changes.

## Best Possible Solution

Add a focused doctor state diagnostic that resolves the active bundled skills directory, scans persisted session `skillsSnapshot.prompt` and `skillsSnapshot.resolvedSkills[].filePath` values for missing, inactive, or temp-backed bundled runtime roots, reports cached-session metadata drift separately from live runtime health, and in repair mode backs up the session store before deleting only the affected `skillsSnapshot` fields.

## Repair Work Prompt

Implement the stale cached skill snapshot doctor diagnostic requested by this issue. Add a focused `openclaw doctor` state check that resolves the active bundled skills directory, scans persisted session entries for `skillsSnapshot.prompt` `<location>` paths and `skillsSnapshot.resolvedSkills[].filePath` values, and warns when bundled-skill paths point at missing roots, inactive install roots, `/tmp`, `/private/tmp`, or another non-current OpenClaw runtime root. The output must distinguish live runtime health from stale cached session metadata. In repair mode, back up the affected session store first and delete only the affected `skillsSnapshot` fields so the existing rebuild paths refresh them on the next run; do not delete whole sessions or transcripts. Keep the check scoped to bundled runtime roots and avoid false positives for workspace, `.agents`, personal, managed, extra-dir, or plugin skills. Add tests for warning-only and repair behavior, update doctor docs, and add a user-facing changelog entry crediting the reporter/commenters per repo policy. Do not change unrelated runtime refresh semantics or dependency pins.
