## Summary

Keep open. Current main still lacks a doctor diagnostic or repair path for stale persisted skill snapshots; the existing doctor state-integrity path checks session-store/transcript health, while CLI and gateway skill snapshots are still reused on version/filter state without validating cached bundled-skill paths against the active runtime root. The reporter supplied concrete remediation evidence, so this is actionable rather than stale.

## What This Changes

The issue requests an `openclaw doctor` diagnostic and safe remediation for cached session skill snapshots that still reference inactive runtime roots after installs, moves, upgrades, or temp-backed runs.

## Best Possible Solution

Add a focused doctor state-integrity diagnostic that resolves the active bundled skills directory, scans persisted `skillsSnapshot.prompt` and `skillsSnapshot.resolvedSkills[].filePath` values for missing, inactive, or temp-backed bundled runtime roots, reports cached-session metadata drift separately from live runtime health, and in repair mode backs up the session store before deleting only affected snapshot fields. Update the doctor docs and changelog because this is user-visible repair behavior.
