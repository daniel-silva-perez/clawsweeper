## Summary

Keep open. Current main still has session-store and skills-snapshot machinery, but `openclaw doctor` does not inspect persisted `skillsSnapshot` prompt or `resolvedSkills` paths for stale bundled runtime roots, and the runtime refresh paths still reuse cached snapshots based on version/filter state rather than path validity. The issue has concrete reproduction context and a safe remediation approach, so it is actionable rather than stale.

## What This Changes

Add an `openclaw doctor` diagnostic and safe repair path for stale persisted session skill snapshots that reference inactive, missing, or temporary OpenClaw runtime roots.

## Best Possible Solution

Add a focused doctor state diagnostic that resolves the active bundled skills directory, scans persisted session `skillsSnapshot.prompt` and `skillsSnapshot.resolvedSkills[].filePath` values for missing, inactive, or temporary bundled runtime roots, reports cached-session metadata drift separately from live runtime health, and in repair mode backs up the session store before deleting only the affected `skillsSnapshot` fields so normal snapshot rebuild logic refreshes them.
