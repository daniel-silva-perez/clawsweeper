## Summary

Keep open. Current main has adjacent session-store, transcript, lock, and skills-snapshot refresh machinery, but `openclaw doctor` still does not detect stale cached `skillsSnapshot` metadata that points at inactive runtime roots or temp paths. The reporter’s follow-up provides a concrete safe remediation path, and the remaining work is a focused doctor diagnostic/fix rather than an obsolete or duplicate request.

## Best Possible Solution

Add a focused doctor contribution that resolves the active OpenClaw runtime/bundled-skill root, scans each agent’s `sessions.json` for cached `skillsSnapshot.prompt` and `resolvedSkills` paths under missing, inactive, `/tmp`, or `/private/tmp` roots, reports affected session keys as stale cached session metadata, and in repair mode backs up the store then deletes only those stale `skillsSnapshot` fields so normal snapshot rebuilding repopulates them. Cover it with unit tests using synthetic session stores and update `docs/cli/doctor.md`.
