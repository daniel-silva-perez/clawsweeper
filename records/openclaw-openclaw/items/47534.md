## Summary

Keep open. Current main still lacks the requested doctor diagnostic or repair for stale persisted skill snapshots after runtime-root changes. The existing doctor state-integrity path checks session-store and transcript health, while CLI and gateway skill snapshot reuse still depend on version/filter state rather than validating cached bundled-skill paths against the active runtime root. The reporter supplied concrete remediation evidence, so this is actionable rather than stale.

## What This Changes

Request `openclaw doctor` detection and safe remediation guidance for persisted session skill snapshots that reference inactive, missing, or temp-backed OpenClaw runtime roots after an install/runtime path change.

## Best Possible Solution

Add a focused doctor state-integrity diagnostic that resolves the active bundled skills directory, scans persisted `skillsSnapshot.prompt` and `skillsSnapshot.resolvedSkills[].filePath` values for missing, inactive, or temp-backed bundled runtime roots, reports cached-session metadata drift separately from live runtime health, and in repair mode backs up the session store before deleting only affected snapshot fields. Update the doctor docs and changelog because this is user-visible repair behavior.

## Repair Work Prompt

Implement a focused fix for openclaw/openclaw#47534. Add an `openclaw doctor` state-integrity diagnostic for stale cached session skill snapshots after runtime-root changes. The observable bug is that persisted session `skillsSnapshot` metadata can keep old bundled-skill file paths after OpenClaw moves or is run from a temporary/wrong install root while the live runtime is healthy. Resolve the active bundled skills directory, scan persisted session store entries for `skillsSnapshot.prompt` and `skillsSnapshot.resolvedSkills[].filePath` values that reference missing, inactive, or temp-backed bundled runtime roots, and report this as cached-session metadata drift distinct from live runtime health. In repair mode, back up the session store and remove only affected `skillsSnapshot` fields so snapshots rebuild on the next run. Do not wipe sessions, transcripts, or unrelated metadata. Do not fold in oversized toolResult cleanup or general session-store pruning. Coordinate with #71497 conceptually, but do not depend on that PR or duplicate its restart-version refresh semantics.
