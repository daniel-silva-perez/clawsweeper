## Summary

Current main still lacks the requested doctor diagnostic and repair path. The existing state-integrity check covers session-store/transcript health, while persisted skill snapshots are still reused from version/filter state without checking cached bundled-skill paths against the active runtime root; the reporter also supplied a concrete safe remediation, so this is actionable rather than stale.

## What This Changes

The issue requests an `openclaw doctor` diagnostic and safe remediation for stale persisted session skill snapshots that reference old runtime roots after OpenClaw moves or upgrades.

## Best Possible Solution

Add a focused doctor state-integrity diagnostic that resolves the active bundled skills directory, scans persisted `skillsSnapshot.prompt` and `skillsSnapshot.resolvedSkills[].filePath` values for missing, inactive, or temp-backed bundled runtime roots, reports cached-session metadata drift separately from live runtime health, and in repair mode backs up the session store before deleting only the affected snapshot fields.
