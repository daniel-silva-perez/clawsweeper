## Summary

Keep this issue open. Current main still does not expose Slack global shortcut or message-action shortcut events through `channels.slack`: the Slack monitor registers block actions and modal lifecycle handlers, but there is no `app.shortcut` registration, shortcut payload summary, system-event enqueue path, docs entry, or shortcut-focused test coverage. The prior Codex review comment in the issue remains accurate against the reviewed main SHA.

## Best Possible Solution

Implement this in the bundled Slack plugin, or first add a typed plugin-facing Slack shortcut/raw-interaction API. The bundled path should add a shortcut event module that registers Bolt `app.shortcut` handlers for global and message-action shortcuts, acknowledges and mismatch-checks payloads, applies existing Slack sender/session authorization, enqueues sanitized stable system events with context keys, and updates Slack interaction tests plus `docs/channels/slack.md`.
