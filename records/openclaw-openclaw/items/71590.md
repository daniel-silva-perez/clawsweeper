## Summary

Keep this PR open. Current main still has only the existing exponential temporal-decay path, so the contribution is not obsolete, but the PR is not correct as-is: the new config keys are accepted by schema/type surfaces without being threaded through the resolved memory-search runtime config, and the added docs/schema semantics still disagree with the runtime math.

## What This Changes

The PR adds an opt-in Weibull temporal decay model for memory scoring, config/schema/type plumbing, generated schema updates, a PR design document, and targeted Weibull decay tests.

## Best Possible Solution

Land a revised memory-core change that keeps exponential as the default, carries `model` and `weibullShape` through `resolveMemorySearchConfig` into `MemoryIndexManager`, aligns schema validation with the documented clamping behavior, fixes the β-direction docs, and leaves broader importance/access-boost design to #57307 unless maintainers explicitly expand this PR’s scope.

## Reproduction Assessment

Yes for the blocking PR behavior: PR-head source inspection shows a config setting for `memorySearch.query.hybrid.temporalDecay.model: "weibull"` is accepted by schema but dropped by `resolveMemorySearchConfig` before memory-core receives it. Current-main verification also shows no existing Weibull support.

## Solution Assessment

No as-is. The core math direction is plausible and narrow, but the implementation must include the resolved config path and user-facing docs/schema consistency before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Thread the model through resolved memory config:** `src/config/zod-schema.agent-runtime.ts:766-767`
  - body: The new schema accepts `temporalDecay.model` and `weibullShape`, but the PR does not update `ResolvedMemorySearchConfig` or `resolveMemorySearchConfig`; on the PR head that resolver still returns only `enabled` and `halfLifeDays`, and `MemoryIndexManager.search` passes that object into `mergeHybridResults`. A user setting `model: "weibull"` in OpenClaw config will still run the default exponential path.
  - confidence: 0.93
- **[P2] Correct the beta less-than-one decay text:** `docs/pr-weibull-decay.md:40`
  - body: For `0 < β < 1`, `(t / halfLife)^β` is larger than the exponential ratio before half-life and smaller after half-life, so the multiplier decays more early and less later. The added table says the opposite and later labels β=0.5 as slow-start, which would teach users to tune the inverse curve.
  - confidence: 0.95
- **[P2] Align shape validation with clamp semantics:** `src/config/zod-schema.agent-runtime.ts:767`
  - body: The runtime helper clamps out-of-range shapes, but `z.number().min(0.1).max(5)` rejects those same user config values before runtime. Since the PR docs describe values like `1000` and `0.001` self-healing to the bounds, either accept and clamp through the runtime path or document strict config validation.
  - confidence: 0.9
