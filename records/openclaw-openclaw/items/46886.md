## Summary

Keep this issue open. Current main still drops `agentId` and `sessionKey` for the agent-facing `cron` tool wake action: the tool and Gateway wake handler pass only `mode` and `text`, and `CronService.wake` enqueues/wakes without target options. The lower Gateway cron adapter already knows how to route `agentId`/`sessionKey` when it receives them, so the remaining work is focused plumbing plus regression coverage. Related #64556 and open PR #70268 cover the hook wake side, but they do not make the cron tool wake surface implemented on main yet.

## Best Possible Solution

Keep this issue open. The best path is to expand or coordinate with #70268, or land a focused PR that threads optional `agentId` and `sessionKey` through the cron tool schema, Gateway wake protocol/handler, `CronServiceContract`/`CronService.wake`, and the existing Gateway cron routing adapter, with regression tests for the agent-facing cron wake action and the direct Gateway wake method.
