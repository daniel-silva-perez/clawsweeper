Keep open. Current main still drops `agentId`/`sessionKey` on the cron wake path before it reaches `CronService`, while the lower Gateway cron adapter already has target-resolution support once those fields are supplied. Add optional `agentId` and `sessionKey` targeting to cron wake actions so wake events can route to a specified agent/session instead of always falling back to the default agent.
