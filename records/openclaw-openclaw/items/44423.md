## Summary

Keep this PR open. Current main still documents Telegram polling-stall recovery and retry/network knobs, but it does not include the PR's restart/update recovery failure signature, host-level recovery watchdog guidance, or systemd path note. The PR remains docs-only, with one useful remaining security-doc concern around warning operators not to log or send full health JSON.

## Best Possible Solution

Keep the PR open for maintainer review. Preserve the focused Telegram recovery-hardening docs, keep the corrected `%h`/absolute-path systemd wording, rebase onto the current table, and add a concise warning or sanitized `jq` extraction example so watchdog scripts only act on minimal health booleans and never transmit full health JSON.
