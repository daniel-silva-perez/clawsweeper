## Summary

Keep this issue open. Current main still has separate WhatsApp reaction, direct text send, and auto-reply/quoted-reply delivery paths, and the WhatsApp auto-reply path still does not surface a delivery-state reconciliation signal when model/session text exists but provider delivery is not confirmed.

## Best Possible Solution

Keep this issue open. The best fix is a WhatsApp-owned delivery-state record around `dispatchWhatsAppBufferedReply`, `deliverWebReply`, direct `sendMessageWhatsApp`, and `sendReactionWhatsApp`: record model-produced text, queued delivery kind, provider dispatch invocation, provider message id or error per mode, and whether transcript/session text was recorded. Surface a clear operator/debug warning when text is produced but no WhatsApp provider delivery is invoked or accepted, and add regression coverage where ack reaction succeeds while plain text or quoted reply delivery fails.
