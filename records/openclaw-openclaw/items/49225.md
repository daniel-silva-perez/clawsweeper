## Summary

Keep this issue open. Current main still has the split the report describes: WhatsApp ack reactions, plain sends, and quoted replies are separate paths, and the auto-reply dispatcher does not yet surface a delivery-state reconciliation warning when assistant text is produced but provider delivery fails or is not invoked.

## Best Possible Solution

Keep this issue open. The best fix is a WhatsApp-owned delivery-state record around dispatchWhatsAppBufferedReply/deliverWebReply and the reaction/send APIs: capture model-produced text, queued delivery kind, provider dispatch invocation, provider message id/error per mode, and surface a clear 'text produced but not delivered' warning in logs/status/debug UI. Add regression coverage where ack reaction succeeds while plain text or quoted reply delivery fails.
