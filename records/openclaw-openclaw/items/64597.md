## Summary

Keep this PR open. Current main does not implement the PR’s central overflow-recovery change: the post-compaction fallback still attempts tool-result truncation whenever oversized tool results are detected, without checking whether the reducible tool output can plausibly clear the overflow. The current attempt result type also lacks the PR’s attempt-local `estimatedContextTokens` field, and the auto-reply path still auto-resets on embedded context-overflow messages rather than only embedded compaction failures.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best implementation path is in the embedded runner overflow recovery path: compare estimated reducible tool-result characters against an attempt-local overflow estimate after compaction fails, avoid stale prompt-usage provenance, keep the safer auto-reply reset behavior for true compaction failures only, and land focused regression coverage for the one-shot truncation and attempt-estimate paths.
