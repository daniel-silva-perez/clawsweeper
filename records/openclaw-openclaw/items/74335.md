## Summary

Keep this PR open. Current main still uses the per-turn `params.timeoutMs` as the Codex app-server startup ceiling and does not expose a separate `appServer.startupTimeoutMs` config/docs/manifest field, while the PR is a narrow implementation with targeted tests and no evident supply-chain-sensitive diff. This should go through maintainer review and CI rather than conservative cleanup closure.

## What This Changes

This PR adds optional Codex app-server `startupTimeoutMs` config/runtime support, uses it for startup instead of the per-turn timeout, updates manifest/docs/changelog, and adds resolver plus regression tests.

## Best Possible Solution

Land this PR or an equivalent narrow change so Codex app-server startup has its own documented, configurable timeout while preserving existing per-turn `timeoutMs` and control-plane `requestTimeoutMs` semantics.
