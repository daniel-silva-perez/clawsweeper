## Summary

Keep open. Current main still lacks a first-class ephemeral or tmpfs-backed sandbox workspace lifecycle, and open PR #73738 already references this issue with closing syntax; the next step is maintainer/secops review of that PR rather than cleanup closure.

## What This Changes

Request an opt-in ephemeral or tmpfs-backed workspace mode for isolated sessions so sandbox file changes are discarded between runs.

## Best Possible Solution

Land a reviewed opt-in sandbox workspace lifecycle with persistent behavior as the backward-compatible default, an ephemeral mode that uses run-owned workspace/runtime paths, guarded cleanup, schema metadata, docs, and targeted tests, then explicitly document whether tmpfs-backed storage is included or deferred.
