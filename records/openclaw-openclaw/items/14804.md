Keep open. ## What This Changes

Request an opt-in ephemeral or tmpfs-backed workspace mode for isolated sandbox sessions so each run starts from a clean disposable file state.

## Best Possible Solution

Review and either land or close PR #73738 after confirming an opt-in sandbox workspaceLifecycle contract with persistent as the backwards-compatible default, safe run-scoped cleanup on success and setup failure, aligned config/schema/generated metadata/docs/tests, and a clear decision on tmpfs-backed workspace support.

## Reproduction Assessment

Not applicable as a runtime bug, but there is a high-confidence current-main feature-gap check: workspaceLifecycle is absent from the config type and strict schema, and sandbox workspace resolution still derives stable per-scope directories.

## Solution Assessment

Unclear until PR #73738 is reviewed. The requested direction is maintainable if it stays opt-in, preserves persistent defaults, and constrains deletion to run-owned paths; the safest path is review/land or explicitly close that linked PR rather than starting a separate repair.
