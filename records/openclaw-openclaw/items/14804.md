## Summary

Keep open. Current main still lacks a first-class ephemeral or tmpfs-backed sandbox workspace lifecycle, and the linked open implementation PR is unmerged; the remaining action is maintainer/secops review rather than cleanup closure.

## What This Changes

Request to add an opt-in ephemeral or tmpfs-backed workspace mode for isolated sandbox sessions so file changes do not persist across runs.

## Best Possible Solution

Land a reviewed, opt-in sandbox workspace lifecycle with persistent behavior as the backward-compatible default and an ephemeral mode that uses run-owned workspace/runtime paths, guarded cleanup, schema metadata, docs, and targeted tests; decide explicitly whether disposable directories satisfy the tmpfs part of the request or whether tmpfs needs a separate follow-up.
