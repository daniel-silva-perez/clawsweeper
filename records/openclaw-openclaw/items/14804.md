## Summary

Keep open. Current main still has persistent per-scope sandbox workspaces with no first-class ephemeral workspace lifecycle in config, schema, runtime resolution, or docs, and PR #73738 is open, unmerged, and directly proposes the missing behavior for this issue.

## What This Changes

Request an opt-in ephemeral or tmpfs-backed sandbox workspace mode so isolated sessions start from clean disposable filesystem state without manual cleanup.

## Best Possible Solution

Land or explicitly close PR #73738 after secops review confirms an opt-in sandbox workspaceLifecycle contract with persistent as the default, safe run-scoped ephemeral workspace/runtime cleanup, aligned config/schema/generated metadata/docs/tests, and clear documentation of whether tmpfs-backed workspaces are supported or deferred.
