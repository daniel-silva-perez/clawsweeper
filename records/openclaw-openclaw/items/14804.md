## Summary

Keep open. Current main still lacks a first-class ephemeral/tmpfs-backed sandbox workspace lifecycle: sandbox config and schema have no lifecycle key, runtime workspace resolution reuses stable per-scope directories, and docs describe manual recreate/prune rather than disposable per-run workspaces. The provided timeline also shows open, unmerged PR #73738 referencing this issue for closure, so the correct next step is maintainer/security review of that PR rather than closing the issue now.

## What This Changes

Add an option for isolated sandbox sessions to use disposable or tmpfs-backed workspaces so file changes do not persist across runs.

## Best Possible Solution

Add a first-class sandbox workspace lifecycle setting with persistent as the default and ephemeral as an opt-in, wire it through config types/schema/generated metadata/docs, derive safe per-run workspace/runtime paths, and clean them up after success and failure with targeted sandbox/config tests. If #73738 satisfies those requirements after secops review, landing it should resolve this issue.
