## Summary

Current main still does not provide first-class ephemeral or tmpfs-backed sandbox workspace lifecycle support. The sandbox config/schema expose persistent per-scope workspace settings only, workspace setup preserves existing contents, and Docker tmpfs is separate from the OpenClaw workspace bind mount. This issue should stay open because open PR #73738 is the paired implementation candidate and references this issue with closing syntax.

## Best Possible Solution

Keep this issue open and route maintainer attention to PR #73738. The best outcome is to review that PR against the core sandbox lifecycle boundary, land it if the config/schema/runtime cleanup/docs/tests are correct, or close/request changes there while keeping this issue as the canonical feature request.
