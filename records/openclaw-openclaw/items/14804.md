## Summary

Keep open. Current main still lacks a first-class ephemeral or tmpfs-backed sandbox workspace lifecycle, and PR #73738 is an open, unmerged implementation candidate that uses closing syntax for this issue. Because that PR directly covers the requested config, runtime, cleanup, docs, and tests, the issue should stay open for maintainer/secops review rather than be closed or routed to a separate repair lane.

## What This Changes

Add an option for isolated sessions to use ephemeral or tmpfs-backed workspaces so file changes do not persist across runs.

## Best Possible Solution

Land a reviewed, opt-in sandbox workspace lifecycle with persistent behavior as the default and an ephemeral mode that uses run-owned workspace/runtime paths, guarded cleanup, config/schema metadata, docs, and targeted coverage for sandbox context resolution, config validation, and failure cleanup.
