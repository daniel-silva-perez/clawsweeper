## Summary

Keep open. Current main still lacks a first-class ephemeral or tmpfs-backed sandbox workspace lifecycle: config types and strict schema have no lifecycle key, sandbox workspace resolution still reuses stable per-scope directories, and docs describe recreate/prune flows rather than disposable per-run workspaces. Open PR #73738 directly references this issue with closing syntax and remains open, so the issue should stay open for maintainer/secops review instead of being closed or routed to a separate repair PR.

## What This Changes

Request adds an opt-in ephemeral or tmpfs-backed workspace mode for isolated sessions so sandbox file changes are discarded between runs.

## Best Possible Solution

Add a reviewed opt-in sandbox workspace lifecycle setting with `persistent` as the backward-compatible default and an `ephemeral` mode that uses run-scoped workspaces, safe deletion guards, config/schema metadata, docs, and targeted coverage across sandbox context resolution, embedded-run cleanup, and config validation.
