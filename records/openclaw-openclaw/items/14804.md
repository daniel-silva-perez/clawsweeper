## Summary

Keep open. Current main still lacks a first-class ephemeral or tmpfs-backed sandbox workspace lifecycle: the sandbox config types and strict schema have no `workspaceLifecycle` key, runtime workspace resolution still derives stable per-scope directories, and docs describe manual recreate/prune flows rather than disposable per-run workspaces. Open PR #73738 directly references this issue with closing syntax and proposes the missing lifecycle/cleanup behavior, so the right path is maintainer and secops review of that PR rather than closing this issue now.

## What This Changes

The issue requests an option for isolated sessions to use a disposable, preferably tmpfs-backed workspace so file changes do not persist across runs.

## Best Possible Solution

Add an opt-in sandbox workspace lifecycle setting with `persistent` as the backwards-compatible default and `ephemeral` as a per-run disposable mode, wire it through config types, validation, generated metadata, docs, sandbox context resolution, and embedded-run cleanup, and land #73738 only if secops review confirms the deletion boundaries and failure cleanup are safe.
