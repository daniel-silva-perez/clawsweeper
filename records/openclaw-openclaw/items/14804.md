## Summary

Keep open. Current main still lacks a first-class ephemeral or tmpfs-backed sandbox workspace lifecycle, and the linked implementation PR is open and unmerged with closing syntax, so the issue should remain tracked for maintainer/secops review rather than be closed now.

## What This Changes

Request an opt-in ephemeral or tmpfs-backed sandbox workspace mode so isolated sessions start from a clean disposable filesystem and do not preserve file changes across runs.

## Best Possible Solution

Land a secops-reviewed opt-in sandbox workspace lifecycle with persistent behavior as the default, an ephemeral mode using run-owned workspace and runtime paths, guarded cleanup on success and setup failure, aligned config schema/generated metadata/docs/tests, and explicit documentation for whether tmpfs-backed workspaces are supported or deferred.
