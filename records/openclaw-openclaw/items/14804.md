## Summary

Keep open. Current main still lacks first-class ephemeral or tmpfs-backed sandbox workspace lifecycle support, and open PR #73738 is an unmerged implementation candidate with closing syntax for this issue. The remaining action is maintainer/secops review, not cleanup closure.

## What This Changes

Request an opt-in ephemeral or tmpfs-backed workspace mode for isolated sessions so sandbox filesystem changes are discarded between runs.

## Best Possible Solution

Land a reviewed, opt-in sandbox workspace lifecycle with persistent behavior as the default and an ephemeral mode that uses run-owned workspace/runtime paths, guarded cleanup, config/schema metadata, docs, and targeted coverage for sandbox context resolution, config validation, and failure cleanup.
