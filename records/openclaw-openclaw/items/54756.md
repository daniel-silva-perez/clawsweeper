## Summary

Keep open: current `main` still has the auto-update root fallback selecting legacy `entry` files before canonical `index` files, so the PR is not fully implemented on `main`. The update-command half is already superseded by shared resolver work, so the useful path is a narrowed rebase or replacement focused on `src/infra/update-startup.ts`.

## What This Changes

The PR branch reorders gateway update entrypoint fallback candidates to prefer `dist/index.js` before `dist/entry.js` in update refresh and auto-update paths, with focused Vitest coverage.

## Best Possible Solution

Use the shared gateway entrypoint resolver for the auto-update root fallback, preserve the existing `process.argv[1]` reuse and non-node/bun executable branches, and add coverage proving `dist/index.js` wins when both built entrypoints exist.

## Reproduction Assessment

Yes: set `process.argv` to only `process.execPath`, provide a package root containing both `dist/index.js` and `dist/entry.js`, and the current root fallback will choose `entry.js` first by source order. The PR’s proposed `src/infra/update-startup.test.ts` case is the right focused reproduction shape.

## Solution Assessment

No: the index-first direction is correct, but the best current-main fix is narrower than this PR branch. Update-command already uses the shared resolver, so the remaining solution should reuse that resolver in `src/infra/update-startup.ts` rather than carrying the obsolete update-command change.
