## Summary

Current main only partially covers this PR. The service-refresh/update-command side now uses a shared index-first gateway entrypoint resolver, but the auto-update root fallback that this PR also changes still checks `dist/entry.js` before `dist/index.js`. The PR still contains useful remaining work, ideally rebased to reuse the shared resolver and add focused coverage for that fallback.

## Best Possible Solution

Keep this PR open or ask for a small rebase: drop the now-obsolete update-command candidate-list change, update `src/infra/update-startup.ts` to use the shared index-first gateway entrypoint resolver for the root fallback, and keep/add the focused auto-update fallback test.
