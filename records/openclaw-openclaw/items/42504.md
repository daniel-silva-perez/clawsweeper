## Summary

Keep this issue open. Current main still lacks the requested Agents-tab avatar upload path: it has string avatar metadata, read-only authenticated avatar serving, and browser-local assistant avatar overrides, but no upload RPC or POST route that writes image bytes into an agent workspace and refreshes identity metadata. The closed, unmerged PR #11603 and the linked gist remain useful design context, not shipped behavior.

## What This Changes

The issue requests an Agents-tab Control UI flow for uploading agent avatar image files, storing them in the selected agent workspace, and updating agent identity metadata.

## Best Possible Solution

Add an additive admin-scoped gateway RPC or authenticated HTTP endpoint with typed protocol/schema coverage, strict image type and size validation, symlink-safe bounded writes under the selected agent workspace, identity updates through existing helpers, Agents-tab picker and refresh behavior, docs, and focused gateway/UI regression tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The negative check is high-confidence from current-main source inspection: there is no upload RPC, no POST avatar handler, and no Agents-tab file picker.

## Solution Assessment

Yes, keeping this open for maintainer review is the best current path. The feature is scoped, but the endpoint contract and workspace write behavior are security-sensitive enough that maintainers should approve the design before an automated fix lane attempts it.
