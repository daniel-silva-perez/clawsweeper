## Summary

Keep open. Current main can display configured agent avatars through the authenticated avatar route and has a browser-local Quick Settings assistant-avatar upload, but it still lacks the requested Agents-tab upload flow that persists image bytes under the agent workspace and updates agent identity metadata. The linked gist and closed unmerged PR #11603 provide useful implementation context, but there is no merged implementation or open paired fix PR to close against.

## What This Changes

Add a Control UI Agents-tab avatar picker plus an admin upload RPC that stores the selected image in the agent workspace and updates `IDENTITY.md`.

## Best Possible Solution

Add an additive admin-scoped gateway method or authenticated endpoint with typed protocol schema, strict image validation, bounded storage under the selected agent workspace, identity updates through existing helpers, Agents-tab picker and refresh wiring, public Control UI docs, and focused gateway/UI regression coverage.
