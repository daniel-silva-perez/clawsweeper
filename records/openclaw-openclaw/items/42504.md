## Summary

Keep this issue open. Current main can store string avatar metadata, serve already configured local avatars through the authenticated read-only avatar route, and provide a browser-local Quick Settings assistant-avatar override, but it still lacks the requested Agents-tab upload flow that persists image bytes under the selected agent workspace and updates identity metadata. The linked gist and closed, unmerged PR #11603 remain useful implementation context rather than a shipped fix.

## What This Changes

The issue requests an Agents-tab avatar picker plus an admin upload path that saves a selected image under the agent workspace and updates `IDENTITY.md`.

## Best Possible Solution

Add an additive, admin-scoped gateway RPC or authenticated endpoint with typed protocol schema, strict image validation, bounded symlink-safe writes under the selected agent workspace, identity updates through existing helpers, Agents-tab picker and refresh wiring, docs, and focused gateway/UI regression coverage.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The negative check is high-confidence from current-main source inspection: there is no registered upload RPC, no POST avatar handler, and no Agents-tab file picker.

## Solution Assessment

Yes, keeping this open for maintainer review is the best current path. The feature is scoped, but the upload endpoint and workspace write semantics are security-sensitive enough that product/security shape should be approved before an automated fix lane attempts it.
