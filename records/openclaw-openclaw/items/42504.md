## Summary

Keep open. Current main can display configured agent avatars through the authenticated `/avatar/<agentId>` route and has a browser-local Quick Settings assistant-avatar override, but it still lacks the requested Agents-tab upload flow that persists image bytes under the agent workspace and updates `IDENTITY.md`.

## What This Changes

The issue requests a Control UI Agents-tab avatar picker/upload flow that stores the chosen image under the selected agent workspace, updates `IDENTITY.md`, and renders it through `/avatar/<agentId>`.

## Best Possible Solution

Add an additive admin-scoped gateway method or authenticated endpoint with typed protocol schema, strict image validation and storage under the selected agent workspace, `IDENTITY.md` updates through existing identity helpers, Agents-tab picker/refresh wiring, docs for the persistent workflow, and focused gateway/UI regression coverage.
