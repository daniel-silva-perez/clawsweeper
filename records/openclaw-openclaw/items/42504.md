## Summary

Keep open. Current main still supports configured avatar display through the authenticated avatar route and a browser-local Quick Settings assistant-avatar override, but it does not implement the requested Agents-tab upload flow that persists image bytes under the agent workspace and updates IDENTITY.md.

## What This Changes

The issue requests an Agents-tab avatar picker that uploads a local image, stores it in the selected agent workspace, updates IDENTITY.md, and renders it through `/avatar/<agentId>`.

## Best Possible Solution

Add an additive admin-scoped gateway method or authenticated endpoint with typed protocol schema, safe image validation/storage under the selected agent workspace, IDENTITY.md updates through existing identity helpers, Agents-tab picker and refresh wiring, docs for the persistent workflow, and focused gateway/UI regression coverage.
