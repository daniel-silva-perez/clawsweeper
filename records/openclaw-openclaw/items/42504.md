## Summary

Keep this issue open. Current main still supports read-only agent avatar resolution/serving and browser-local assistant avatar overrides, but it still lacks the requested Agents-tab upload flow, upload RPC or POST route, and binary write path under an agent workspace. The linked gist and closed, unmerged #11603 remain useful design context rather than shipped behavior.

## What This Changes

Add an Agents-tab avatar picker/upload flow that stores a selected image under the agent workspace and updates the agent identity so `/avatar/<agentId>` renders it.

## Best Possible Solution

Add a maintainer-approved, additive admin-scoped avatar upload path with typed protocol/schema coverage, strict image size and type validation, safe filename handling, symlink-safe bounded writes under the selected agent workspace, identity updates through existing helpers, Agents-tab picker and refresh behavior, docs, and focused gateway/UI regression tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The negative check is high-confidence from current-main source inspection: there is no upload RPC, no POST avatar route, and no Agents-tab file picker for agent workspace avatars.

## Solution Assessment

Unclear whether the exact proposed RPC shape is the best final API. The user-facing direction is valid, but maintainers should choose the endpoint contract, auth scope, and identity persistence model before implementation.
