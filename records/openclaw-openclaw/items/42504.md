## Summary

Keep open. Current main can serve and display already configured agent avatars and supports a browser-local Quick Settings assistant-avatar override, but it still does not provide the requested Agents-tab upload flow that persists image bytes into an agent workspace and updates identity metadata. The linked gist and closed, unmerged PR #11603 are useful implementation context, but there is no merged implementation or open paired fix PR to close against.

## What This Changes

Add an Agents-tab avatar picker plus an admin upload path that saves the chosen image under the agent workspace and updates the agent identity metadata.

## Best Possible Solution

Add an additive, admin-scoped gateway RPC or authenticated endpoint with typed protocol schema, strict image validation, bounded writes under the selected agent workspace, identity updates through existing helpers, Agents-tab picker and refresh wiring, public Control UI docs, and focused gateway/UI regression coverage.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The current-main negative check is high-confidence from source inspection: there is no registered upload RPC, no POST avatar handler, and no Agents-tab file picker.

## Solution Assessment

Yes, keeping this open for maintainer review is the best current path. The feature is narrow, but the upload endpoint and workspace write semantics are security-sensitive enough that the product/security shape should be approved before an automated fix lane attempts it.
