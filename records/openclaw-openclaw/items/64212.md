## Summary

Keep open. Current main has generic custom-provider image routing and configurable image-tool timeouts, but it still does not prove the exact NVIDIA NIM Kimi K2.5 image path; the bundled NVIDIA catalog remains text-only for Kimi K2.5, and the default image timeout can still produce the reported abort shape around the reporter's observed latency.

## What This Changes

The issue reports that a custom NVIDIA NIM Kimi K2.5 provider marked as image-capable fails in the image tool with "Request was aborted" after roughly 70 seconds, despite direct API calls working.

## Best Possible Solution

Keep this open until maintainers verify the exact configured-provider image path against NVIDIA NIM Kimi K2.5. If a longer media timeout fixes it, document the required tools.media.image timeout or adjust defaults; if not, fix the OpenAI-compatible multimodal payload/stream handling or surface a clear unsupported-model error, and update NVIDIA catalog/docs only if image support is confirmed.
