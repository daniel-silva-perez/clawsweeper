## Summary

Keep open. Current main has generic custom-provider image routing, configurable image-tool timeouts, and newer diagnostics, but it still does not prove the exact NVIDIA NIM Kimi K2.5 image path. The bundled NVIDIA catalog remains text-only for Kimi K2.5, and the default 60s image timeout plus abort propagation can still match the reported abort after roughly 70 seconds.

## What This Changes

The issue requests that a custom NVIDIA NIM Kimi K2.5 image model parse inbound Feishu or Telegram images through the image tool instead of failing with "Request was aborted."

## Best Possible Solution

Maintain this as an open provider/image triage item until maintainers verify the exact NVIDIA NIM Kimi K2.5 custom-provider image flow. If a longer tools.media.image timeout fixes it, document the required configuration or revisit defaults; if not, fix the OpenAI-compatible multimodal request/stream path or surface a clear unsupported-model error.
