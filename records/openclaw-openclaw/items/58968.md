## Summary

Keep open. Current main still leaves the Playwright-backed remote-CDP tab enumeration path unbounded, so the reported hang remains actionable even though newer client/gateway request timeouts may mask some user-facing failures.

## What This Changes

The issue requests timeout protection and graceful failure handling for `listPagesViaPlaywright` when a remote CDP connection stalls during browser tab listing.

## Best Possible Solution

Add a bounded, browser-plugin-owned timeout around Playwright page enumeration, evict or close stale cached CDP connections on timeout, and return a controlled browser error or documented empty-tab fallback with a regression test for a never-resolving CDP target-info/title read.

## Reproduction Assessment

Yes. A high-confidence unit repro can mock a connected browser whose page `newCDPSession().send('Target.getTargetInfo')` or `page.title()` never resolves, then call `listPagesViaPlaywright`; current main has no deadline around that await chain.

## Solution Assessment

Yes. The narrowest maintainable fix is at the Playwright enumeration boundary; outer gateway/client request timeouts are useful defense-in-depth but leave the hung CDP operation and cached connection state unresolved.
