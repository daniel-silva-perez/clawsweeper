## Summary

Keep open. Current main still routes remote/persistent Playwright tab listing through `listPagesViaPlaywright` with no timeout around page enumeration, `Target.getTargetInfo`, or `page.title()`. The linked fix PR #59043 was closed unmerged, and current tests cover reconnect/error propagation but not a never-settling CDP command timeout.

## Best Possible Solution

Keep this issue open and implement the fix in the browser plugin: add a bounded timeout around `listPagesViaPlaywright` or its per-page CDP/title reads, evict or disconnect stale Playwright state on timeout when appropriate, return a graceful tab-list result or actionable browser error instead of wedging the request path, and add a regression test with a never-resolving CDP session. PR #59043 is a useful starting point, but it would need to be rebased and reviewed against current `extensions/browser` code.
