## Summary

Keep this PR open. Current main partially fixes the linked gateway-agent freshness bug, but it has not implemented the PR author's follow-up `server-node-events` change: `voice.transcript` and `agent.request` still pass a stored `sessionId` together with a canonical `sessionKey`, and downstream session resolution still treats that explicit `sessionId` as authoritative.

## Best Possible Solution

Rebase this PR on current main, preserve the remaining `server-node-events` fix so `voice.transcript` and `agent.request` dispatch by canonical `sessionKey` without forwarding stale stored `sessionId`, and add or keep focused regression tests for both node-event paths. Then review it as an implementation candidate for #54164, reconciling the gateway-agent handler with the freshness guard already on main.
