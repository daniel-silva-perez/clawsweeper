## Summary

Keep this PR open. Current main partially addresses the linked gateway `agent` freshness bug, but it does not implement the PR author's follow-up node-event fix: `voice.transcript` and `agent.request` still pass a stored `sessionId` together with a canonical `sessionKey`, and downstream session resolution still treats that explicit `sessionId` as authoritative.

## Best Possible Solution

Rebase this PR on current main, preserve the remaining `server-node-events` fix so `voice.transcript` and `agent.request` dispatch by canonical `sessionKey` without forwarding stale stored `sessionId`, and reconcile the gateway-agent handler with the freshness guard already on main. Keep or add focused regression tests for both node-event paths, then review it as the implementation candidate for the linked stale-session rollover bug.
