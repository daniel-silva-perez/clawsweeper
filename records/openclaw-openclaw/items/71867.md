## Summary

Current main still has the reported doctor-auth labeling gap. `noteAuthProfileHealth` checks and loads the implicit/default auth store, emits generic auth note titles, and receives no configured-agent context from the doctor flow. The open PR #71923 is a direct implementation candidate and its changelog entry links itself to this issue with GitHub closing syntax, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/rebase/land #71923 or an equivalent focused fix. The implementation should enumerate configured agents, label doctor auth diagnostics with the agent id when more than one direct store is checked, preserve existing single-store output, inspect per-agent stores read-only, avoid refreshing labeled per-agent diagnostic passes unless deliberately scoped, and keep regression coverage for divergent `main`/`coder` stores plus inherited-only no-duplication behavior.
