## Summary

Keep open. Current main still runs the doctor auth health check through an implicit/default auth store and emits generic auth note titles, while open PR #71923 already targets the bug and should be reviewed, landed, or closed by maintainers.

## What This Changes

The issue requests that `openclaw doctor` identify which agent store each model-auth status comes from, or at least warn when auth state differs across multiple agent stores.

## Best Possible Solution

Review and land #71923, or an equivalent narrow fix, so doctor enumerates configured agent stores, labels multi-agent auth diagnostics, keeps single-agent output stable, scopes refresh attempts to the inspected store, and adds divergent-store regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: configure multiple agents with divergent `auth-profiles.json` stores, then current main runs doctor auth health through `ensureAuthProfileStore(undefined, ...)` and reports generic auth note titles.

## Solution Assessment

Yes. The best direction is a focused doctor/auth reporting fix that reuses existing agent-scope helpers and read-only auth-store loading rather than changing auth inheritance, credential selection, or provider runtime semantics.
