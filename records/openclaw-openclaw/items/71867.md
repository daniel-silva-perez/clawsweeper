## Summary

Keep open. Current main still checks one implicit/default auth profile store in the doctor auth path, emits generic auth note titles, and does not pass configured-agent context into that flow; the linked open PR #71923 is the active implementation candidate rather than a reason to close before merge.

## What This Changes

The issue requests that `openclaw doctor` identify which agent auth store it is reporting when multiple per-agent auth stores exist.

## Best Possible Solution

Review and land #71923 or an equivalent narrow fix that enumerates configured agents, resolves each agent directory, labels multi-agent doctor auth diagnostics, preserves the existing single-agent output, and adds regression coverage for divergent auth stores.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: configure multiple agents with divergent `auth-profiles.json` stores, then `openclaw doctor` reaches `noteAuthProfileHealth` through the doctor flow without any agent context and emits generic auth titles.

## Solution Assessment

Yes, the focused doctor/auth change is the right shape: use existing agent-scope helpers and read-only auth store loading rather than changing core auth semantics or per-agent credential inheritance.
