## Summary

Keep open. Current main still checks doctor auth health through an implicit/default auth store and emits generic auth note titles, while the related implementation PR remains open and unmerged.

## What This Changes

The issue requests that `openclaw doctor` identify which agent auth store it is reporting when multiple agent stores exist.

## Best Possible Solution

Review and land #71923, or land an equivalent narrow fix that enumerates configured agent stores, labels multi-agent auth diagnostics, preserves single-agent output, keeps refresh scoped to the inspected store, and adds divergent-store regression coverage.

Yes. A high-confidence static reproduction path exists: configure multiple agents with divergent `auth-profiles.json` stores, then current main runs doctor auth health through one unscoped store and emits generic note titles.

## Solution Assessment

Yes. The best direction is a focused doctor/auth reporting fix that reuses existing agent-scope helpers and read-only auth-store loading rather than changing auth inheritance, credential selection, or provider runtime semantics.
