## Summary

Keep this issue open. Current main still chooses the Discord route from static inputs before mention detection, so a thread inheriting a parentPeer binding can still route to the parent-channel agent even when the message text or mentionPatterns point at a different agent. Related #35821 attempted this but was auto-closed unmerged; related #13304 is broader content-pattern routing and is not a canonical replacement for this Discord mention-override request.

## Best Possible Solution

Keep this issue open. A maintainer should define the precedence contract for Discord explicit bot mentions and per-agent mentionPatterns, including how they interact with /focus runtime bindings and configured ACP/channel bindings. Then implement a Discord-owned mention-route selection path before mention gating, with route-cache safety, regression coverage for a thread inheriting parentPeer while mentioning another configured agent, and docs updates for channel routing, Discord, and multi-agent behavior.
