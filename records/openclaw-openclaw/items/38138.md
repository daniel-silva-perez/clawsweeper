## Summary

Keep this issue open. Current main still resolves Discord agent routing from static route inputs before mention detection, and mention handling still gates the already selected effective route instead of rerouting to a different mentioned agent. The broader content-based routing issue is related, but it is not a canonical replacement for this Discord mention override request.

## Best Possible Solution

Keep this issue open. Define the intended precedence for explicit Discord bot mentions and per-agent mentionPatterns versus runtime focus/configured bindings, then implement mention-based route selection before mention gating in the Discord preflight/route-resolution path. Add a regression for a Discord thread inheriting a parentPeer binding while mentioning another configured agent, and update the Discord plus multi-agent docs with the final routing order.
