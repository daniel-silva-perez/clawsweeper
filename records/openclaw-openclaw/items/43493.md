## Summary

Keep this PR open. Current main still supports custom provider setup and model metadata config, but it does not implement the PR's interactive custom-provider metadata prompt for contextWindow/contextTokens/maxTokens/reasoning/input/cost. The related custom-provider context UX issue #48338 is still open, so this PR remains a plausible implementation candidate rather than conservative cleanup.

## Best Possible Solution

Keep this PR open as a plausible implementation path. The best maintainer path is to rebase it onto current main, align the wizard with current ModelDefinitionConfig and ModelProviderConfig fields, keep it limited to the custom provider setup path, and land it with focused configure/model-metadata tests. If maintainers prefer a different UX, supersede it with an equivalent fix for #48338 rather than closing the remaining user problem.
