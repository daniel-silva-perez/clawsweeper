## Summary

Keep open: current main still lacks an interactive metadata step for custom-provider setup, so the feature remains relevant, but the PR is not merge-ready for the `openclaw configure --section model` custom provider surface because it has concrete model-ref, API override, and input-modality regressions.

## What This Changes

The PR adds a custom-provider model metadata prompt to `openclaw configure --section model`, helper functions for editing configured model definitions, and unit tests for the prompt/update flow.

## Best Possible Solution

Rebase the PR onto current main, integrate the metadata prompt into the current custom-provider configure flow, reuse the shared model-ref/modelKey helpers, preserve the full `ModelDefinitionConfig` input contract, define API inheritance so Auto can actually clear model-level overrides or relabel it accurately, and add focused tests for those cases.

## Reproduction Assessment

Yes. The current-main gap is reproducible by source inspection of the custom provider configure path and default metadata writes, and the PR defects are reproducible with small unit scenarios for provider-prefixed IDs, an existing per-model API override, and existing audio/video inputs.

## Solution Assessment

No. The feature direction is useful, but this patch is not the best mergeable solution yet because it reimplements model-ref parsing, narrows input modalities, and exposes provider inheritance without clearing existing model-level API overrides.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Handle provider-prefixed primary model IDs:** `src/commands/model-metadata.ts:77-80`
  - body: `findPrimaryModelEntry` splits the primary model key and searches only for the suffix. Current `modelKey` preserves native IDs that already start with the provider prefix, so a custom entry like provider `openrouter` with model id `openrouter/foo` stores the primary as `openrouter/foo`, but this code searches for `foo` and silently skips the metadata prompt.
  - confidence: 0.87
- **[P2] Clear API overrides when Auto is selected:** `src/commands/model-metadata.ts:131-132`
  - body: Selecting `Auto (inherit from provider)` leaves `result.api` unset, and the later merge spreads the existing model before the update, so any existing model-level `api` value survives. The wizard offers provider inheritance but cannot actually remove a previous per-model adapter override.
  - confidence: 0.86
- **[P2] Preserve audio and video input modalities:** `src/commands/model-metadata.ts:140-149`
  - body: `ModelDefinitionConfig.input` accepts `video` and `audio`, but the prompt only offers `text` and `image` and then overwrites `entry.model.input` with the result. Editing a media-capable custom model can drop valid existing modalities when the user only meant to change token metadata.
  - confidence: 0.78
