## Summary

Keep this PR open. Current main still lacks an interactive custom-provider metadata prompt, so the contribution is not obsolete, but the patch is not mergeable as-is because it misses provider-prefixed model IDs, cannot clear an existing model-level API override, and can drop supported audio/video input modalities.

## What This Changes

The PR adds a custom-provider model metadata prompt, a custom-api-key configure hook, and unit tests for editing configured model display/API/reasoning/input/context/max-token/cost metadata.

## Best Possible Solution

Rebase the PR onto current main and revise it to integrate with the current custom-provider configure flow, use shared model-ref/modelKey helpers, preserve the full ModelDefinitionConfig input contract, make API inheritance semantics actually clear existing overrides or label the option accurately, and add focused tests for provider-prefixed IDs, API override clearing, and audio/video input preservation.

## Reproduction Assessment

Yes. The current-main gap is reproducible by source inspection of configure.gateway-auth.ts and onboard-custom-config.ts, and the PR defects are reproducible with small unit scenarios for openrouter/openrouter-style IDs, an existing model-level api override, and input arrays containing audio or video.

## Solution Assessment

No. The feature direction is useful, but this patch is not the best mergeable solution yet because it reimplements model-ref parsing, narrows the input modality contract, and presents provider inheritance without clearing the existing model-level override.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Handle provider-prefixed primary model IDs:** `src/commands/model-metadata.ts:77-80`
  - body: findPrimaryModelEntry splits the primary key and searches only for the suffix. Current modelKey preserves native IDs that already start with the provider prefix, so a custom entry such as provider openrouter with model id openrouter/foo stores the primary as openrouter/foo, but this code searches for foo and silently skips the metadata prompt.
  - confidence: 0.87
- **[P2] Clear API overrides when Auto is selected:** `src/commands/model-metadata.ts:131-132`
  - body: Selecting Auto leaves result.api unset, and the final merge spreads entry.model before updated, so any existing model-level api value survives. The wizard offers provider inheritance but cannot actually clear a previous per-model adapter override.
  - confidence: 0.86
- **[P2] Preserve existing audio and video inputs:** `src/commands/model-metadata.ts:140-149`
  - body: ModelDefinitionConfig.input accepts video and audio, but the prompt only offers text and image and then overwrites entry.model.input with that result. Editing a media-capable custom model can drop valid existing modalities even when the user only intended to change token metadata.
  - confidence: 0.78
