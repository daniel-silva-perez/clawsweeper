## Summary

Keep open: current main still lacks the guided metadata UX for custom providers, but the PR is not merge-ready because verified source-level bugs remain and the branch is stale against the current configure flow.

## What This Changes

The PR adds a custom-provider model metadata prompt with unit tests and invokes it after custom API-key provider setup in the configure wizard.

## Best Possible Solution

Rebase onto current main and integrate the metadata step with the current custom-provider configure path, using shared model-ref/modelKey helpers, preserving the full ModelDefinitionConfig input contract, making API inheritance semantics explicit and functional, and adding focused tests for provider-prefixed model IDs, API override clearing, and media-capable inputs.

## Reproduction Assessment

Yes. The current-main gap is reproducible by source inspection of configure.gateway-auth.ts plus onboard-custom-config.ts, and the PR defects are reproducible with small unit scenarios: provider openrouter with model id openrouter/foo, an existing model-level api override, and an input array containing audio or video.

## Solution Assessment

No. The direction is useful, but this patch is not the best mergeable solution yet because it reimplements model-ref parsing, uses a text/image-only modality prompt against a broader config contract, and presents provider inheritance without actually clearing existing model-level API overrides.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Resolve provider-prefixed primary model IDs:** `src/commands/model-metadata.ts:77-80`
  - body: findPrimaryModelEntry splits the primary key and searches for only the suffix. Current modelKey preserves native IDs that already start with the provider prefix, so a custom entry such as provider openrouter with model id openrouter/foo persists the primary as openrouter/foo, but this code searches for foo and silently skips the metadata prompt.
  - confidence: 0.87
- **[P2] Make Auto clear model-level API overrides:** `src/commands/model-metadata.ts:131-132`
  - body: Selecting Auto (inherit from provider) leaves result.api unset, and the final merge spreads entry.model before updated, so any existing model-level api value survives. The wizard offers provider inheritance but cannot actually clear a previous override.
  - confidence: 0.86
- **[P2] Preserve all supported input modalities:** `src/commands/model-metadata.ts:140-149`
  - body: ModelDefinitionConfig.input accepts video and audio, but the prompt only offers text and image and then overwrites entry.model.input with that result. Editing a media-capable custom model can drop valid existing modalities even when the user only wanted to change token metadata.
  - confidence: 0.78
