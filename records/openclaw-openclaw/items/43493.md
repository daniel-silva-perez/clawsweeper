## Summary

Keep open. Current main still lacks an interactive custom-provider metadata prompt, so the PR is aimed at a real remaining UX gap, but the latest PR head is not merge-ready because it still mishandles provider-prefixed model IDs and API inheritance, and it is stale against newer model/configure semantics on main.

## What This Changes

The PR adds a new model metadata prompt module and tests, then invokes it after custom API-key provider setup so `openclaw configure --section model` can edit configured model metadata fields.

## Best Possible Solution

Rebase on current main and revise the prompt to use shared model-ref/modelKey helpers, implement true provider inheritance clearing or relabel the API choice, preserve all current `ModelDefinitionConfig.input` modalities, and add focused tests for those cases before maintainer product review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Resolve provider-prefixed primary model IDs:** `src/commands/model-metadata.ts:77-80`
  - body: `findPrimaryModelEntry` splits the primary key and searches for only the suffix. Current `modelKey` preserves native IDs that already start with the provider prefix, so a custom entry such as provider `openrouter` with model ID `openrouter/foo` persists the primary as `openrouter/foo`; this code searches for `foo` and silently skips the metadata prompt.
  - confidence: 0.87
- **[P2] Make Auto clear model-level API overrides:** `src/commands/model-metadata.ts:131-132`
  - body: Selecting `Auto (inherit from provider)` leaves `result.api` unset, and the final merge spreads `entry.model` before `updated`, so any existing model-level `api` survives. The wizard offers provider inheritance but cannot actually clear a previous override.
  - confidence: 0.86
- **[P2] Preserve all supported input modalities:** `src/commands/model-metadata.ts:140-149`
  - body: Current `ModelDefinitionConfig.input` accepts `video` and `audio`, but the prompt only offers `text` and `image` and then overwrites `entry.model.input` with that multiselect result. Editing a media-capable custom model can drop valid existing modalities even when the user is only changing token metadata.
  - confidence: 0.78
