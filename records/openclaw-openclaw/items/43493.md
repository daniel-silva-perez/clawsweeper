## Summary

Keep this PR open because the custom-provider metadata prompt is still a real current-main UX gap, but the branch is not merge-ready: the proposed helper misses provider-prefixed model IDs, cannot clear an existing model-level API override when the UI says Auto/inherit, and narrows valid model input modalities.

## What This Changes

The PR adds a custom-provider-only model metadata prompt to the configure flow, plus a new `src/commands/model-metadata.ts` helper module and unit tests.

## Best Possible Solution

Rebase the branch onto current main, integrate the metadata prompt after the current custom-provider setup path, reuse shared model-ref/modelKey helpers, preserve the full `ModelDefinitionConfig` metadata contract, define Auto API inheritance so it can actually clear overrides or relabel the option, and add focused tests for provider-prefixed IDs, media modalities, and API override clearing.

## Reproduction Assessment

Yes. The current-main gap is reproducible by source inspection of `promptAuthConfig` plus the fixed custom-provider defaults, and the PR defects are reproducible with small unit scenarios for provider-prefixed native model IDs, existing model-level API overrides, and audio/video input metadata.

## Solution Assessment

No. The feature direction is useful, but this patch is not the best mergeable solution yet because it reimplements model-ref parsing, narrows valid input modalities, and exposes provider API inheritance without clearing existing overrides.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Handle provider-prefixed primary model IDs:** `src/commands/model-metadata.ts:77-80`
  - body: `findPrimaryModelEntry` splits the primary key and searches only for the suffix. Current model refs can preserve native IDs that already start with the provider prefix, so a custom entry like provider `custom` with model id `custom/vision-model` stores the primary as `custom/vision-model`, but this code searches for `vision-model` and silently skips the metadata prompt.
  - confidence: 0.87
- **[P2] Clear API overrides when Auto is selected:** `src/commands/model-metadata.ts:131-132`
  - body: Selecting `Auto (inherit from provider)` leaves `result.api` unset, and the later merge spreads the existing model before the update, so any previous model-level `api` value survives. The wizard offers provider inheritance but cannot actually remove a previous per-model adapter override.
  - confidence: 0.86
- **[P2] Preserve audio and video input modalities:** `src/commands/model-metadata.ts:140-149`
  - body: `ModelDefinitionConfig.input` accepts `video` and `audio`, but the prompt only offers `text` and `image` and then overwrites `entry.model.input` with the result. Editing a media-capable custom model can drop valid existing modalities when the user only meant to change token metadata.
  - confidence: 0.78
