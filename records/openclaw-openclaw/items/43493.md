## Summary

Keep open. Current main still does not implement the interactive custom-provider metadata prompt, and the related custom-provider context-window UX issue remains open; however, this PR needs maintainer review and rebase work because the patch is now inconsistent with current model-ref and API-inheritance behavior.

## What This Changes

The PR adds a custom-provider-only configure wizard step plus helper tests for editing configured model metadata such as API adapter, reasoning, input modality, context limits, max output tokens, and token costs.

## Best Possible Solution

Rebase the PR onto current main, keep the prompt limited to custom-provider setup, use the shared model-ref parsing/modelKey helpers, include current contextWindow/contextTokens expectations, allow users to clear model-level API overrides back to provider inheritance, and add focused wizard/helper tests for those cases. If maintainers prefer a broader UX, supersede this through the related #48338 path rather than dropping the user problem.
