## Summary

Keep open. Current main still has adjacent agent-runtime selection, harness, sub-agent, node, and multi-agent routing machinery, but it does not implement a Gateway control plane with separately registered per-agent compute runtimes; the narrower Phase 0 request for per-agent runtime start/stop/restart is also still missing.

## What This Changes

The issue requests splitting the monolithic Gateway into a lightweight control plane and independently managed per-agent compute runtimes while preserving embedded mode for compatibility.

## Best Possible Solution

Keep this as an architecture tracker and sequence it additively: preserve embedded mode, define a stable Gateway-to-runtime protocol with registration, health, capability, session, and delivery semantics, add local per-agent process lifecycle first, then layer container/Kubernetes backends and per-agent secret/resource isolation.
