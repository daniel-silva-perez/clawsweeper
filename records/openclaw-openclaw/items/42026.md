## Summary

Keep open. Current main has adjacent embedded agent-runtime selection, harness contracts, node hosts, multi-agent routing, and Gateway control-plane machinery, but it still does not implement a Gateway control plane that registers and routes to separately managed per-agent compute runtimes. The narrower Phase 0 request from the discussion, per-agent runtime start/stop/restart without a Gateway restart, is also still absent.

## What This Changes

The issue proposes splitting the monolithic Gateway into a lightweight control plane and separately managed per-agent runtimes for model loops, tools, MCP servers, memory, workspace state, and isolated compute.

## Best Possible Solution

Keep this as an architecture tracker and sequence it additively: preserve embedded mode, define a stable Gateway-to-runtime protocol with registration, health, capability, session, and delivery semantics, add local per-agent process lifecycle first, then layer container/Kubernetes backends and per-agent secret/resource isolation.
