## Summary

Keep this open. Current main has useful adjacent pieces, including multi-agent routing, per-agent auth/state, agent runtime selection, ACP/CLI harness paths, node capability hosts, and gateway lifecycle controls, but it still does not implement the requested split between a Gateway control plane and separately registered per-agent compute runtimes. The narrower Phase 0 from the discussion, per-agent start/stop/restart without restarting the Gateway, is also still missing.

## What This Changes

Request to split the monolithic Gateway into a lightweight control plane plus separately managed per-agent Agent Runtime processes with local-split and distributed deployment modes.

## Best Possible Solution

Keep this as an architecture tracker and turn it into an additive design that preserves embedded mode, defines a stable Gateway-to-runtime protocol and runtime role/capability handshake, adds local per-agent process lifecycle and health first, then layers container/Kubernetes backends plus per-agent secret and resource isolation.
