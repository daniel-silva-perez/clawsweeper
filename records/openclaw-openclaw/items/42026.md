## Summary

Keep open. Current main has adjacent multi-agent routing, per-agent state/auth, embedded harness selection, node capability hosts, and gateway/node protocol roles, but it still does not implement the requested control-plane plus separately registered per-agent compute-runtime split. The narrower Phase 0 discussed in comments, per-agent start/stop/restart without a Gateway restart, is also not present.

## What This Changes

The issue requests splitting the current gateway into a control plane plus separately managed per-agent compute runtimes with local-split and distributed deployment modes.

## Best Possible Solution

Keep this as an architecture tracker and turn it into an additive design: preserve embedded mode, define a stable Gateway-to-runtime protocol with registration/health/capability semantics, add local per-agent process lifecycle first, then layer container/Kubernetes backends plus per-agent secret and resource isolation.
