## Summary

Current main still does not solve the reported Volcengine Plan imageModel path: the explicit imageModel route remains active, but the bundled `volcengine-plan` catalog is text-only, the reported model is absent from the catalog, and there is no Volcengine-owned media-understanding adapter or regression coverage for this Ark coding image endpoint. The related #55688 remains an open partial catalog-capability candidate, not a merged fix for this exact model/endpoint.

## What This Changes

The issue asks OpenClaw to support or clearly reject `agents.defaults.imageModel.primary = volcengine-plan/doubao-seed-2.0-pro` for inbound image understanding, where direct Ark API calls succeed but OpenClaw returns 404 or schema errors.

## Best Possible Solution

Keep the issue open until maintainers verify the exact Volcengine Ark coding image endpoint and model ID. Then either add Volcengine-owned image-capable catalog/media-understanding support with regression coverage, or explicitly document `volcengine-plan` imageModel as unsupported and guide users to a supported image-capable `volcengine/*` model or custom provider config with `input: ["text", "image"]`.
