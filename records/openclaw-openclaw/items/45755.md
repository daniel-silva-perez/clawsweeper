## Summary

Keep open. Current main still resolves assistant identity emoji in the gateway, but the Control UI bootstrap/state/chat rendering path does not carry or render that emoji in assistant chat group footers, so the reported v2 UI regression remains valid.

## What This Changes

The issue requests restoring the agent identity emoji inline with the assistant/agent name in the Control UI chat transcript after the v2 UI regression.

## Best Possible Solution

Carry the resolved assistant identity emoji through the bootstrap contract, RPC normalization, app state, and chat rendering, then display it inline with the assistant sender name for both streaming and persisted assistant groups while preserving current avatar override behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to mock bootstrap or agent.identity.get with an emoji, render assistant chat groups, and observe that current app state and render props discard that value before .chat-sender-name is produced.

## Solution Assessment

Yes. A narrow Control UI propagation and rendering fix is the best path because the gateway already resolves emoji; changing selector dropdowns or avatar fallback behavior alone does not restore the reported transcript sender label.
