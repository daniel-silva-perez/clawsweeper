## Summary

Keep open. Current main still preserves the configured agent identity emoji at the gateway/RPC boundary, but the Control UI chat transcript path drops it before rendering assistant message footers/headers. The Sessions table now uses identity emoji, and #45898 tried a selector-dropdown fix, but neither addresses the chat message header/footer regression described in this issue discussion.

## Best Possible Solution

Keep this issue open and land a focused Control UI fix that carries assistant identity emoji through bootstrap/RPC normalization and UI state, then renders it inline with the assistant name in chat transcript group footers/headers, including streaming groups. Preserve the existing Sessions-table emoji behavior and add targeted UI tests for the chat transcript path rather than treating the selector/session-list work as sufficient.
