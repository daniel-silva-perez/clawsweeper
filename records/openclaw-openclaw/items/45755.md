Current main still preserves agent identity emoji in the gateway/RPC identity result, but the Control UI chat rendering path drops it before assistant message footers and streaming labels are rendered. The issue remains valid; the related selector-only PR #45898 is closed unmerged and would not have covered the chat transcript path by itself.

Keep this issue open and land a focused Control UI fix that carries assistant identity emoji through bootstrap/RPC normalization and UI state, then renders it inline with the assistant name in chat transcript group footers/headers, including streaming groups. Preserve the existing Sessions-table emoji behavior and do not treat a selector-only fix as sufficient.

## Repair Work Prompt

Fix the Control UI regression reported in openclaw/openclaw#45755: agent identity emoji from IDENTITY.md/config should render inline with the assistant name in chat transcript group footers/headers, including streaming groups. Current backend `agent.identity.get` already returns `emoji`; the UI drops it in `ui/src/ui/assistant-identity.ts` / `ui/src/ui/controllers/assistant-identity.ts`, app state has no assistantEmoji, and `ui/src/ui/views/chat.ts` / `ui/src/ui/chat/grouped-render.ts` pass/render only name and avatar. Add an `assistantEmoji` field through bootstrap/RPC normalization and app state, pass it through `renderChat`, `renderStreamingGroup`, and `renderMessageGroup`, and render it as escaped Lit text next to the assistant name. Do not use `unsafeHTML`. Do not change gateway identity resolution unless a type/contract update is needed to expose an already-resolved value. Keep existing Sessions-table emoji behavior intact and do not revive the selector-only approach from #45898 as the sole fix. Add targeted tests proving an emoji returned by identity state appears in the assistant chat sender label and streaming sender label, while user/tool labels remain unchanged. Add a user-facing changelog fix entry with attribution, e.g. Thanks @HenryLoenwind for the report and @dsantoreis for the context.
