## Summary

Keep this issue open. Current main still stores OpenRouter chat-completion stream usage through the shared OpenAI-compatible parser and local pricing calculation, with no provider-specific post-stream `/generation` lookup to replace the transcript cost with OpenRouter's authoritative billed `total_cost`. The existing tiered-pricing support only recomputes aggregate transcript summaries from configured pricing tables and does not correct the assistant JSONL record or use OpenRouter generation metadata.

## What This Changes

The issue requests that OpenRouter streamed response costs be reconciled against OpenRouter's authoritative generation billing record for tier-priced models, instead of persisting the lower streamed/local estimated cost as the final total.

## Best Possible Solution

Add OpenRouter-owned billed-cost reconciliation for completed streaming chat completions: when the response id and a real OpenRouter credential are available, fetch `/generation`, validate a numeric `total_cost`, persist it before the assistant message is finalized, and leave the existing estimate untouched on lookup failure, aborts, missing ids, or custom non-OpenRouter endpoints.

## Repair Work Prompt

Implement authoritative OpenRouter billed-cost reconciliation for this issue. In current main, OpenRouter chat completions use the shared OpenAI-compatible stream path, which records `output.responseId` and locally computes `message.usage.cost.total`; for tier-priced OpenRouter models this can undercount billed cost. Add a focused fix in the bundled OpenRouter/provider stream boundary or the smallest generic transport seam needed so that, after a verified OpenRouter streaming completion finishes and before the assistant message is finalized, OpenClaw fetches OpenRouter generation metadata using the resolved OpenRouter API key and the normalized native OpenRouter base URL. When `GET /generation?id=<responseId>` returns a finite non-negative `total_cost`, overwrite the persisted usage cost total with that authoritative billed value. Preserve the current local/streamed estimate on missing response id, missing API key, aborted/error runs, non-OpenRouter custom base URLs, lookup timeout/failure, or malformed metadata. Do not log credentials or full billing responses, do not broaden private-network access, and do not change unrelated provider pricing/catalog behavior. Add regression coverage for successful billed-cost overwrite, no lookup without response id, lookup failure fallback, and no lookup for custom non-OpenRouter proxy base URLs.
