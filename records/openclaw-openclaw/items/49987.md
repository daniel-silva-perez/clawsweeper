## Summary

Keep this PR open. Current main still does not implement the PR‚Äôs intended direct `openclaw gateway restart` sentinel behavior, and the PR‚Äôs current patch needs lifecycle rework before it can safely land.

## Best Possible Solution

Keep this PR open or replace it with a maintainer patch that writes the CLI restart sentinel after restart preflight/no-op decisions but before the actual service restart or SIGUSR1 emission, removes it if the restart is rejected or fails, resolves the sentinel path using the daemon service environment when available, and adds focused lifecycle tests for managed and unmanaged restart paths.
