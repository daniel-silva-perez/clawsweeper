## Summary

Keep this PR open. Current main still does not implement direct `openclaw gateway restart` restart-sentinel notification, and the PR’s current v2 shape writes the sentinel too late for the restarted gateway’s startup sentinel check.

## Best Possible Solution

Keep this PR open or replace it with a maintainer patch that writes the CLI restart sentinel after preflight/no-op decisions but before the actual service restart or SIGUSR1 emission, removes it if restart is rejected or fails, resolves the sentinel path with the daemon service environment when available, and adds focused tests for managed, unmanaged, no-op, failure, scheduled, and custom-state restart paths.

## Risks / Open Questions

- The PR’s v2 post-restart sentinel write can miss the restarted gateway’s startup check and leave a stale notification for a later, unrelated startup.
- A sentinel written with the CLI shell environment can land in the wrong state directory for profile/custom-state service installs.
- The PR adds no focused lifecycle tests for managed restart, unmanaged SIGUSR1 restart, no-op restart, restart failure cleanup, scheduled restart, or daemon environment resolution.
