## Summary

Keep open. Current main still has credible Gmail hook failure paths: the Gmail preset still uses a templated `hook:gmail:<id>` session key, current hook policy rejects template-derived mapping session keys unless `hooks.allowRequestSessionKey` and prefixes are configured, and `openclaw webhooks gmail setup` still enables the preset without writing those policy fields. The related #60574 Tailscale token fix is closed unmerged, and main still contains that query-token mismatch too.

## Best Possible Solution

Keep this issue open and fix the Gmail hook owner path: make `openclaw webhooks gmail setup` write a policy compatible with the built-in preset or change the preset/setup to a safe static/server-derived session key, ensure mapped `deliver/channel/to` still reaches Signal, and re-apply the #60574 token-in-header/Tailscale endpoint fix with focused regression tests for `/hooks/gmail` and setup output.
