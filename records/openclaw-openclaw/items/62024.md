## Summary

Current main and the shipped v2026.4.26 release already implement the reported Discord explicit-target fix. The issue comments correctly identified the one-field `target.id` to `target.normalized` change, and current source plus regression tests now preserve `user:<id>` through Discord cron/heartbeat routing instead of downgrading it to a channel target.

## What This Changes

The issue reports that Discord cron and heartbeat delivery strip explicit `user:` DM targets to bare numeric IDs, which are later normalized as `channel:` targets and fail with Discord `Unknown Channel`.

## Best Possible Solution

Keep the shipped explicit-target preservation behavior, close this issue as fixed, and handle open PR #72580 separately as redundant unless maintainers find a narrower remaining Discord routing edge case in that branch.
