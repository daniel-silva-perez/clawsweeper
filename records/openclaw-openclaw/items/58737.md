## Summary

Keep open. Current main still supports custom Slack identity on normal sends but does not forward that identity through the Slack edit/update paths named in the report, and open PR #58800 already claims to close this issue with a narrow fix. The issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Slack edited messages should keep the configured agent username and avatar after tool calls or streaming edits instead of reverting to the Slack bot default identity.

## Best Possible Solution

Preserve the resolved Slack agent identity for every OpenClaw-authored Slack message edit/update path, with regression tests for draft streaming, preview finalization, and status-final updates; land #58800 if it still applies cleanly or replace it with an equivalent narrow fix.

## Reproduction Assessment

Yes. The issue's path is reproducible at source level: configure a Slack agent identity, then trigger a response path that reaches `editSlackMessage`, preview finalization, or `status_final` completion update; current main sends identity on normal posts but omits it on those update payloads.

## Solution Assessment

Yes. The narrow maintainable fix is to extend the Slack edit helper to accept the same identity shape used by sends, then thread that through draft stream, preview finalization, and status-final dispatch rather than adding a core workaround.
