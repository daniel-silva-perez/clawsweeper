## Summary

Keep open. Current main still has the reported Slack identity gap: the send path resolves and posts custom identity, but the edit and preview-update paths do not carry that identity through. The supplied context also shows an open closing PR for this issue, so the issue should remain open until that PR is reviewed and merged or replaced.

## What This Changes

Requests that Slack edited messages preserve the active agent username/avatar identity instead of reverting to the Slack app default after OpenClaw updates them.

## Best Possible Solution

Keep this open while maintainers review PR #58800 or a replacement. The end state should preserve Slack agent identity across preview, finalization, and status-final update flows, with targeted regression tests, and should include proof that the chosen Slack API strategy actually preserves custom identity on edited messages.

## Reproduction Assessment

Yes. The issue has a high-confidence reproduction path: route Slack to an agent with custom identity, trigger a streamed/tool-call reply, and compare the initial `chat.postMessage` identity path against later `chat.update` paths that current main calls without identity fields.

## Solution Assessment

Unclear. Passing the existing `slackIdentity` through the edit helpers is the narrowest OpenClaw code change, but the PR review should require Slack API proof or choose an alternative such as avoiding identity-losing edits when custom identity is active.
