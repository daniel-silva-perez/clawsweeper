## Summary

Keep open. Current main has registry-side ClawHub scan summaries and a built-in scanner for gateway-backed installer metadata, but native ClawHub skill install/update still downloads and copies skill folders without a local scan or policy gate. The discussion also raises scan-on-update, provenance, and per-agent policy questions that remain security-sensitive product decisions.

## What This Changes

Request built-in security scanning for skill installation, loading, updates, and ClawHub publish flows, with AgentShield suggested as one possible scanner backend.

## Best Possible Solution

Define a first-class OpenClaw skill supply-chain policy for ClawHub install and update, including local scan execution, registry scan-state handling, critical-finding blocking, operator overrides, provenance/version evidence, UI/CLI/Gateway result surfaces, and docs/threat-model alignment. External scanners can remain optional integrations unless maintainers choose one as the default backend.
