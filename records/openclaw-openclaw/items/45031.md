## Summary

Keep open. Current main has registry-side ClawHub scan summaries and a built-in scanner for gateway-backed skill dependency installs, but the native ClawHub skill install and update paths still download, extract, and copy skill archives without a local scan or policy gate. The remaining work is security-sensitive product design, with external scanners and an open protected security PR in the related context, so no conservative close reason applies.

## What This Changes

The issue requests built-in security scanning and policy enforcement for skill installation, update/loading, and ClawHub publishing, with AgentShield suggested as one possible scanner backend.

## Best Possible Solution

Define a first-class OpenClaw skill supply-chain policy for ClawHub install and update: local scan execution, registry scan-state handling, critical-finding blocking, operator overrides, provenance/version evidence, UI/CLI/Gateway result surfaces, and docs/threat-model alignment. External scanners can remain optional integrations unless maintainers choose one as the default backend.
