## Summary

Keep open. Current main has partial hardening and ClawHub registry-side scan summaries, and the discussion usefully points to AgentShield and ClawGuard as existing external scanners, but native OpenClaw ClawHub skill install/update still downloads skill folders without invoking the built-in scanner or a local policy gate. Docs and the threat model continue to describe this as a remaining supply-chain risk.

## What This Changes

The issue requests built-in security scanning and policy enforcement for ClawHub skill install, update, load, and publish flows.

## Best Possible Solution

Define a first-class ClawHub skill security policy that covers install and update at minimum, with clear handling for registry scan state, local scanner results, critical-finding blocking, operator overrides, and docs/threat-model alignment. Keep external scanners as optional ClawHub/plugin implementations unless maintainers explicitly select one as a default backend.
