## Summary

Keep open. Current main has registry-side ClawHub scan summaries and a built-in dangerous-code scanner for gateway-backed skill dependency installs, but native ClawHub skill install/update still downloads and copies skill archives without a local scan or policy gate. The report is concrete and security-sensitive, with useful external scanner options already posted and overlapping protected security work, so it should stay with maintainers rather than be stale-closed or auto-fixed.

## What This Changes

The issue requests built-in security scanning and policy enforcement for ClawHub skill install, update, loading, and publish flows, with AgentShield-style local scanning, provenance, and configurable policy behavior.

## Best Possible Solution

Define a first-class OpenClaw skill supply-chain policy for ClawHub install and update: local scan execution, registry scan-state handling, critical-finding blocking, operator overrides, provenance/version evidence, UI/CLI/Gateway result surfaces, and docs/threat-model alignment. External scanners such as AgentShield or ClawGuard can remain optional integrations unless maintainers choose one as the default backend.
