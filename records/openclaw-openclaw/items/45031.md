## Summary

Keep open. Current main still has partial hardening only: registry-side ClawHub scan summaries and gateway-backed installer-metadata scanning exist, but native ClawHub skill install/update still downloads and installs skill folders without a local scan or policy gate.

## What This Changes

The issue requests built-in security scanning and policy enforcement for ClawHub skill install, update, load, and publish flows, with AgentShield proposed as one possible scanner backend.

## Best Possible Solution

Define a first-class ClawHub skill security policy that covers install and update at minimum, with explicit treatment for registry scan state, local scanner results, critical-finding blocking, operator overrides, UI/CLI/Gateway behavior, and docs/threat-model alignment. External scanners can stay optional unless maintainers select one as the default backend.
