## Summary

Keep open. Current main has registry-side ClawHub scan summaries and a built-in scanner for Gateway-backed skill dependency installs, but native ClawHub skill install and update still download, extract, and copy skill folders without a local scan or policy gate. The AgentShield and ClawGuard discussion is useful, but the remaining work is security-sensitive and overlaps protected security work, so it needs maintainer policy rather than stale cleanup or an autonomous repair PR.

## What This Changes

This issue requests built-in security scanning and policy enforcement for skill installation, skill loading, ClawHub updates, and ClawHub publishing, with AgentShield proposed as one possible scanner backend.

## Best Possible Solution

Define a first-class OpenClaw skill supply-chain policy for native ClawHub install and update: local scan before copy/activation, scan-on-update, critical-finding blocking, explicit operator overrides, provenance/version evidence, CLI/Gateway/UI result surfacing, and docs/threat-model alignment. AgentShield and ClawGuard can remain candidate scanner backends or plugin integrations unless maintainers choose one as the default.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists by tracing `openclaw skills install <slug>` or Gateway `skills.install` with `source: "clawhub"` into `src/agents/skills-clawhub.ts`; current main downloads, extracts, and installs the archive without invoking `scanSkillInstallSource`, and `openclaw skills update` reuses that forced install path.

## Solution Assessment

Unclear for an AgentShield-only integration. The narrow maintainable direction is a backend-agnostic core scan/policy seam for native ClawHub install/update, with third-party scanners and ClawHub registry scan data plugged in only after maintainers settle the default policy model.
