## Summary

Keep open. Current main still has partial hardening only: gateway-backed installer metadata runs the built-in dangerous-code scanner and ClawHub exposes registry-side scan summaries, but native ClawHub skill install/update still downloads and installs skill folders without a local scan or policy gate.

## What This Changes

The issue requests built-in security scanning and policy enforcement for ClawHub skill install, update, load, and publish flows, with AgentShield proposed as one possible scanner backend.

## Best Possible Solution

Define a first-class ClawHub skill security policy that covers install and update at minimum, with explicit handling for registry scan state, local scanner results, critical-finding blocking, operator overrides, and docs/threat-model alignment. External scanners can remain optional plugin or ClawHub implementations unless maintainers select one as the default backend.
