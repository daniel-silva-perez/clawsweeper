## Summary

Keep open. Current main still has a local enforcement gap for native ClawHub skill install/update, and the issue is security-sensitive enough that it should stay with maintainers rather than be stale-closed or converted into an automated repair without a policy decision.

## What This Changes

Request to add built-in security scanning and policy gates for skill installation, update, first load, and ClawHub publishing, with AgentShield suggested as one possible scanner backend.

## Best Possible Solution

Define a first-class OpenClaw skill supply-chain policy for native ClawHub install and update: local scan execution, critical-finding blocking, operator overrides, provenance/version evidence, result surfacing across CLI/Gateway/UI, and docs/threat-model alignment. AgentShield and ClawGuard can remain candidate scanner backends or plugin integrations unless maintainers choose one as the default.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: trace `openclaw skills install <slug>` or `skills.update` through `src/agents/skills-clawhub.ts`; the archive is downloaded and installed without invoking `scanSkillInstallSource`, while the separate gateway-backed install path does invoke it.

## Solution Assessment

Unclear for the specific AgentShield-only integration. A backend-agnostic core scan/policy seam for ClawHub skill install/update would be the narrower maintainable direction, with external scanners plugged in only after maintainers choose the default policy model.
