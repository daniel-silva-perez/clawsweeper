## Summary

Keep open. Current main still has the reported post-open write-failure leak in both lock helpers, so this PR is not obsolete or implemented. The useful path is a current rebase/port with focused regression tests, plus dropping or reworking the stale docs-i18n pipe cleanup change.

## Best Possible Solution

Keep the PR open as a valid bug-fix candidate. Rebase or port it onto current main, patch src/infra/gateway-lock.ts and src/plugin-sdk/file-lock.ts so every post-open write failure closes the handle and removes the partial lock file, add narrow regression tests for both helpers, and drop the obsolete scripts/docs-i18n/pi_rpc_client.go change unless a current equivalent still needs complete pipe cleanup.
