## Summary

Keep open. Current main still has the reported post-open write failure paths in both lock helpers, so the PR is not obsolete or implemented. The patch needs a current rebase/port, added regression tests, and cleanup of the obsolete docs-i18n RPC client change/review concern before it is merge-ready.

## Best Possible Solution

Keep the PR open as a valid bug-fix candidate, but require a rebase/port against current main that fixes src/infra/gateway-lock.ts and src/plugin-sdk/file-lock.ts with cleanup after successful fs.open plus failing writeFile. Add narrow regression tests that mock the write failure and assert handle.close plus partial lock removal. Drop the obsolete scripts/docs-i18n/pi_rpc_client.go change unless a current equivalent still needs the pipe cleanup, and if it does, close every acquired pipe on each partial-failure path.
