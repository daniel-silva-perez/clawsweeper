## Summary

Keep this PR open. The underlying Discord retry gap still exists on current main, but this branch is not ready to merge because it is stale against the current Discord client internals and broadens transient retries through a shared runner that also wraps non-idempotent send and thread-creation POSTs.

## What This Changes

The PR adds a Discord transient-error retry predicate to `createDiscordRetryRunner` and a new retry unit test file covering rate limits, transient errors, permanent errors, and exhausted attempts.

## Best Possible Solution

Update this PR or replace it with a focused Discord retry policy that imports the current local Discord `RateLimitError`, preserves `retry_after` handling, retries safe transient failures for idempotent REST/lookup/edit/delete operations, and uses a separate send-safe predicate for message, component, poll, sticker, voice, reply, and thread-creation POSTs. Cover both retry and no-retry cases in Discord tests and update retry docs if the public behavior changes.
