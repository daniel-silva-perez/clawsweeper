## Summary

Keep this PR open. Current main still does not implement the intended Discord transient retry behavior, and the PR is paired with open bug #52396 via closing syntax. The live PR remains useful but needs maintainer review because it broadens the shared Discord retry runner that current send/thread POST paths also use, preserving the duplicate-message/thread risk called out in earlier review comments.

## What This Changes

The PR adds a Discord transient-error regex and tests so the Discord retry runner retries 502/503/network-style failures in addition to Carbon RateLimitError.

## Best Possible Solution

Keep this PR open and review it together with #52396. The best fix is a Discord retry policy that retries rate limits plus safe transient failures, preserves retry_after behavior, and separates idempotent request retries from non-idempotent send/thread POST retries, with tests for both retry and no-retry cases.
