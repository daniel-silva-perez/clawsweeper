## Summary

Keep this PR open. Current main still assigns the 32K fallback to ordinary Mantle `/v1/models` discoveries, so the open-weight context-window bug is not implemented on main; however, the PR as written should not merge unchanged because its final 200K fallback overstates omitted 128K models and current main has since added a separate Claude Opus 4.7 Anthropic-route path that needs rebase consideration.

## What This Changes

The branch adds a static Mantle model context-window lookup and changes the Mantle discovery fallback from 32K to 200K in `extensions/amazon-bedrock-mantle/discovery.ts`.

## Best Possible Solution

Land a rebased Mantle-owned discovery change that mirrors the Bedrock provider pattern: known context windows for verified larger models, a conservative fallback for unknowns, targeted unit coverage for 1M/256K/128K/default behavior, and Claude kept on the Anthropic Messages route without duplicating OpenAI-compatible metadata.

## Reproduction Assessment

Yes. A static check or mock Mantle `/v1/models` response with `minimax.minimax-m2` or `qwen.qwen3-coder-next` shows current main returning the 32K fallback; the PR finding is also reproducible from the patch because unlisted 128K models fall through to 200K.

## Solution Assessment

No. The static lookup is the right narrow direction, but the final 200K default is not the safest fix; the fallback should remain conservative, with only verified larger models overridden and tests added.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

- **[P2] Keep Mantle's unknown-model fallback at 128K:** `extensions/amazon-bedrock-mantle/discovery.ts:18`
