## Summary

Keep this PR open. Current main still gives every Mantle `/v1/models` discovery result the hardcoded 32K context window, while the PR’s intended change adds Mantle-specific known context windows and raises the unknown-model fallback. Related #65952 landed the same pattern for the standard Amazon Bedrock provider, but that implementation is separate and does not cover `amazon-bedrock-mantle`.

## Best Possible Solution

Keep the PR open and either land a reviewed version of its Mantle-local context-window metadata, or ask for a small revision that adds focused tests for known Mantle model IDs, default fallback behavior, and any Claude duplicate/API-routing edge case.
