## Summary

Keep this issue open. Current main still lets a sniffed `image/apng` win MIME resolution in `parseMessageWithAttachments`, then passes that MIME into the inline image payload and onward to Anthropic-style provider blocks. The provided GitHub context also shows open PR #51964 with closing syntax for this exact bug, so cleanup should wait until that PR is reviewed and merged or explicitly rejected.

## Best Possible Solution

Keep this issue open and review #51964, or replace it with a maintainer patch that normalizes sniffed `image/apng` to `image/png` before `finalMime` is used for inline/offloaded image payloads. Add regression coverage in `src/gateway/chat-attachments.test.ts` or the shared media MIME tests, then close after the fix lands with release provenance.
