## Summary

Keep this issue open. Current main still routes the sniffed MIME through `parseMessageWithAttachments` without normalizing `image/apng` to `image/png`, and the provided GitHub context shows an open PR with closing syntax for this exact bug, so the issue should remain open until that implementation is reviewed/landed or deliberately closed.

## Best Possible Solution

Keep this issue open and review #51964, or replace it with a maintainer-owned patch that normalizes `image/apng` to `image/png` in the attachment MIME resolution path and adds regression coverage in `src/gateway/chat-attachments.test.ts` or `src/media/mime.test.ts`. After the fixing PR lands, close this as implemented with the landed commit and release provenance.
