## Summary

Keep this PR open. Current main still lacks the persisted tail-drop behavior the PR targets: session-file repair rewrites empty errored assistant entries to the sentinel and has no `droppedTrailingErrorEntries` field or backward walk to remove trailing repaired assistant turns. The PR is focused and plausibly addresses the Anthropic heartbeat replay loop, but it changes the persisted JSONL deletion contract and the non-message-tail boundary still deserves maintainer review before merge.

## What This Changes

This PR adds tail-aware session-file repair that drops trailing errored or sentinel-repaired assistant entries, reports the drop count, expands session repair regression tests, and updates transcript-hygiene docs.

## Best Possible Solution

Land a focused persisted session-file repair once maintainers accept the tail deletion contract, with docs kept in sync and regression coverage proving empty trailing errors, already-healed sentinel tails, mixed mid-transcript rewrites, and non-message-tail behavior.
