## Summary

Keep open. Current main has absorbed much of the Control UI shell localization, but it still hardcodes the command-palette shortcut as `⌘K` in both the visible topbar kbd and the localized title, so the platform-aware shortcut behavior in this PR is not implemented. The branch is still useful, but it overlaps newer main-branch i18n structure and locale generation work, so it needs maintainer-directed rebase/review rather than cleanup closure.

## What This Changes

The PR localizes Control UI shell/topbar/update/status/theme labels, refreshes generated locale metadata, and adds a command-palette shortcut helper/tests to show `⌘K` on Apple user agents and `Ctrl+K` elsewhere.

## Best Possible Solution

Rebase the branch onto current main, preserve the current `chat.*`/`common.*` localization structure unless maintainers intentionally rename it, parameterize the command-palette title so the same platform-aware shortcut drives both the title and visible kbd, keep current translated locale values, regenerate all current locale outputs, and rerun the focused Control UI/i18n checks.
