## Summary

Keep open. Current main still forwards bundle MCP tool inputs unchanged, and this PR addresses a valid failure reported in #70872, but the patch should not merge as-is because it misses JSON Schema array-form types and adds caller-controlled object copying plus unbounded synchronous parsing in the tool-call path.

## What This Changes

The PR adds a bundle MCP materialization helper that parses top-level stringified object/array tool arguments according to the MCP tool input schema, plus regression tests for coerced and already-correct params.

## Best Possible Solution

Land a hardened materialization-boundary fix that preserves current main lifecycle handling, coerces only schema-declared object/array properties, supports nullable JSON Schema `type` arrays, uses safe own-property copying, applies a conservative parse budget, and keeps focused regression coverage for the Synology-style failure.

## Reproduction Assessment

Yes. A materialized bundle MCP tool with `inputSchema.properties.params.type === "object"` and input `{ params: "{}" }` is forwarded unchanged on current main, then `SessionMcpRuntime.callTool` passes that record as MCP arguments, matching the strict-server validation failure reported in #70872.

## Solution Assessment

No, not yet. The materialization boundary is the right place to solve the issue, but this implementation needs type-array support plus safe copying and bounded parsing to be the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Accept JSON Schema array type forms:** `src/agents/pi-bundle-mcp-materialize.ts:50-51`
  - body: Schemas may declare nullable object params as `type: ["object", "null"]`. This guard only accepts a string `type`, so those params still skip coercion and strict MCP servers can continue rejecting stringified object values.
  - confidence: 0.86
- **[P2] Use a null-prototype result object:** `src/agents/pi-bundle-mcp-materialize.ts:43`
  - body: The helper copies model-controlled argument keys into a plain `{}` and returns it after any coercion. A `__proto__` object key can mutate the returned object's prototype before it is sent to the MCP runtime; use a null-prototype object or define own data properties.
  - confidence: 0.84
- **[P2] Bound JSON.parse for coerced arguments:** `src/agents/pi-bundle-mcp-materialize.ts:54`
  - body: Every string under an object/array schema is synchronously parsed without a size budget. Oversized tool arguments can block the event loop or spike memory during tool execution, so this path should reject or skip values above a conservative limit.
  - confidence: 0.78

## Security Review
