## Summary

Keep open. Current main does not implement this PR's intended MCP argument coercion: bundle MCP tools still pass raw tool input through to the session runtime, and the runtime forwards record inputs as MCP `arguments` unchanged. The PR is a plausible focused fix for the concrete #70872 failure, but it still needs maintainer review and likely patch updates for the bot-raised security and schema edge cases before landing.

## Best Possible Solution

Keep this PR open and review it as the active implementation candidate for #70872. The best path is to update the patch against current main, preserve the narrow bundle MCP materialization boundary, add the coercion regression tests, and address the security review items before landing. A broader tool-call normalization layer should only be considered if the same stringified nested-argument behavior is proven outside bundle MCP.
