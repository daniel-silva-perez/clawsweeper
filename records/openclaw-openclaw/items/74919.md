## Summary

Keep open. Current main does not already implement the abort cleanup fast path, and the proposed patch is useful but incomplete: the final cleanup path keys off only the external abort signal, so OpenClaw's own timeout-triggered aborts still take the existing idle wait while holding the session lock.

## What This Changes

The PR adds an immediate-timeout path to `flushPendingToolResultsAfterIdle`, passes abort-signal state into embedded-run cleanup, and extends the focused guard helper test.

## Best Possible Solution

Keep the helper-level immediate timeout behavior, but pass an explicit cleanup fast-path flag or computed internal abort state from `runEmbeddedAttempt` so external aborts, run timeouts, and idle timeouts all skip the idle wait while normal successful cleanup still waits for idle.

## Reproduction Assessment

Yes. The review path is source-level: `scheduleAbortTimer` calls `abortRun(true)`, which sets `timedOut` and aborts `runAbortController`, while the PR's final cleanup checks only `params.abortSignal`; a focused test with an internal timeout and never-idle agent should reproduce the missed fast path.

## Solution Assessment

No. The helper change is a reasonable direction, but the call site should not depend only on the external abort signal; the safer fix is to pass a boolean derived from the runner's internal `aborted`, `timedOut`, and `idleTimedOut` state and add coverage for those cases.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Use the internal abort state for timeout cleanup:** `src/agents/pi-embedded-runner/run/attempt.ts:3340`
  - body: This passes only `params.abortSignal` into final cleanup, but the built-in run timeout path calls `abortRun(true)`, which sets `timedOut` and aborts `runAbortController.signal`; it does not necessarily abort `params.abortSignal`. Ordinary `params.timeoutMs` or idle-timeout aborts therefore still call `flushPendingToolResultsAfterIdle` without `timeoutMs: 0` and can hold the session lock for the existing 30s wait. Pass the existing internal abort/timeout state, or an explicit skip flag, into cleanup and cover that path.
