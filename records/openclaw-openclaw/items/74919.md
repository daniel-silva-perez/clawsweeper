## Summary

Keep open for maintainer review. Current main still waits for the agent idle path before releasing the embedded session lock, and the PR is a narrow relevant fix, but the latest patch has a concrete whitespace issue that should be cleaned up before merge validation.

## What This Changes

The PR adds an immediate-timeout path for pending tool-result cleanup, threads abort and timeout state into embedded attempt cleanup, and adds focused coverage for zero or negative cleanup timeouts.

## Best Possible Solution

Land a narrow version after removing the whitespace errors and getting CI: successful cleanup should still wait for idle to preserve real tool results, while aborted or timed-out cleanup should clear pending tool results immediately so the session lock releases promptly.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: use an agent whose `waitForIdle` never resolves, then current main keeps `flushPendingToolResultsAfterIdle` pending until the 30s default and `cleanupEmbeddedAttemptResources` does not release `sessionLock` until that await completes.

## Solution Assessment

Yes, modulo the whitespace cleanup. The helper-level `timeoutMs <= 0` fast path plus abort-state threading is the narrowest maintainable fix because it preserves normal successful idle waits and only skips the wait on abort or timeout cleanup paths.
