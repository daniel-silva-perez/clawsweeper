## Summary

Keep open. Current main still holds the session lock while pending tool-result cleanup waits for agent idle, and the PR is a narrow viable fix rather than obsolete; it needs maintainer review plus whitespace/CI cleanup before merge.

## What This Changes

The PR adds a `timeoutMs <= 0` fast path to pending tool-result cleanup, threads abort/timeout state into embedded attempt cleanup, and adds focused test coverage for immediate cleanup.

## Best Possible Solution

Land a narrow version after removing whitespace errors and getting CI/Testbox proof: successful cleanup should still wait for idle to preserve real tool results, while aborted or timed-out cleanup should clear pending tool results immediately so the session lock is released promptly.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: use an agent whose `waitForIdle` never resolves, then current main keeps `cleanupEmbeddedAttemptResources` awaiting the 30s default before `sessionLock.release()` runs.

## Solution Assessment

Yes, modulo whitespace cleanup. The helper-level `timeoutMs <= 0` fast path plus abort/timeout-state threading is the narrowest maintainable approach because it preserves the normal idle wait and only skips it on interrupted cleanup paths.
