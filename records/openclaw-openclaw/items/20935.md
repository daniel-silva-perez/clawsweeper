## Summary

Keep open. Current main has a memory host event journal and exposes it as a public artifact, but it is still a lifecycle journal for recall, promotion, and dreaming events, not an append-only mutation audit log for every MEMORY.md and memory/*.md write with what/when/why/session/user/integrity context. The April 26, 2026 maintainer comment also explicitly kept this open for first-class memory/security design, and the Pinchy comment is useful prior art rather than a core implementation.

## Best Possible Solution

Keep this open for maintainer-led design of a first-class memory mutation audit layer. The likely direction is an append-only JSONL audit stream around shared memory-file write surfaces, covering MEMORY.md and memory/*.md mutations with timestamp, operation, relative path, changed-content summary or hash, trigger/why, session/run/user context when available, and an explicit integrity/privacy strategy. Once designed, document the contract in the public memory docs.
