## Summary

Keep open. Current main still excludes Markdown from the built-in skill scanner, so this PR's core security-hardening goal is not implemented. The PR remains the active candidate after superseding the stale closed predecessor, but its current download-and-exec regex still misses practical wrapped interpreter variants, so it needs revision before merge rather than closure.

## Best Possible Solution

Keep this PR open as the active Markdown skill-scanning hardening candidate. Revise markdown-download-exec to cover sudo/env/wrapper prefixes, absolute interpreter paths, versioned interpreters, and line-continuation pipelines; add regression tests for those cases; then run the targeted scanner tests and the appropriate changed gate before merge. Separately decide whether direct ClawHub skill folder installs should invoke the built-in scanner before writing into the workspace.
