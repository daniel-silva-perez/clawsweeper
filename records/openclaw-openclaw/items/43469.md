## Summary

Keep open. Current main still excludes Markdown from the built-in skill scanner and the current tests still expect `.md` files to be non-scannable, while this PR remains the focused implementation candidate for SKILL.md scanning. The PR is not implemented on main or superseded; it should get maintainer review and hardening before merge.

## Best Possible Solution

Keep this PR open as the active Markdown skill-scanning hardening candidate. Rebase it onto current main, keep current scanner options such as test-file exclusion, harden `markdown-download-exec` for sudo/env wrappers, absolute and versioned interpreters, and line-continuation pipelines, add regression tests for those cases, then run the targeted scanner tests and the changed gate before merge. Separately, maintainers should decide whether direct ClawHub skill installs should run the built-in scanner before writing into the workspace.
