## Summary

Keep open. Current main still does not implement the PR’s intended markdown skill scanning: the built-in scanner only allows JS/TS-family files, tests still assert `.md` is non-scannable, and active install/audit paths rely on that scanner. This is a focused core security-hardening PR, but the proposed download-and-execute markdown rule still needs revision before merge.

## Best Possible Solution

Keep this PR open as the active core security-hardening candidate for markdown skill scanning. Ask for a revision that broadens `markdown-download-exec` coverage for wrapped interpreters, absolute paths, versioned interpreters, embedded pipes, and line-continuation pipelines, with regression tests, then run the relevant changed gate before merge.
