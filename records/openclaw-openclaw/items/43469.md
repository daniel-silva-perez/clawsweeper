## Summary

Keep open. Current main still excludes Markdown from the built-in skill scanner, while this PR is the active implementation candidate for scanning SKILL.md content. It is not implemented on main or superseded, but the PR needs maintainer follow-up before merge because the download-and-exec rule still misses common wrapped interpreter forms and the branch is stale against current scanner changes.

## Best Possible Solution

Keep this PR open as the active Markdown skill-scanning hardening candidate. Rebase it onto current main, preserve current scanner options such as test-file exclusion, harden `markdown-download-exec` for sudo/env wrappers, absolute and versioned interpreters, and line-continuation pipelines, add regression tests for those cases, then run the targeted scanner tests and changed gate before merge. Separately, maintainers should decide whether direct ClawHub skill folder installs should invoke the built-in scanner before writing into the workspace.
