## Summary

Keep this PR open. Current main still does not implement the Memory v2 sidecar, `memoryV2` config, ingest registration, rerank hook, or sidecar CLI, so the PR is not obsolete; however the proposed patch is a large bundled memory-core change with concrete review findings around sidecar CLI targeting, read-only behavior, rejected-write exits, rerank shadow touches, deterministic ranking, and a privacy-sensitive persistent ingest surface.

## What This Changes

The PR adds a default-off SQLite-backed Memory v2 foundation to `memory-core`, including sidecar schema/repository code, ingest and rerank pipelines, dreaming shadow-touch wiring, sidecar CLI commands, manifest config, docs, and tests.

## Best Possible Solution

Keep the Memory v2 work under maintainer review, preferably split into sidecar schema/store, ingest, rerank, dreaming shadow-touch, and CLI slices. Merge only after the bundled-memory product direction is accepted, the persistence/privacy contract is explicit, and the concrete CLI/rerank findings are fixed with focused tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the review findings by source inspection: `resolveAgentIds` expands omitted `--agent`, stats/list call an initializer-backed opener, rejected write paths only log, and shadow touch runs before final result slicing. I did not run runtime tests because this review was required to remain read-only.

## Solution Assessment

Is this the best way to solve the issue? Not yet. A default-off foundation is a plausible direction, but the current PR is too broad and contains blocking correctness and privacy-review concerns; the safer path is narrower reviewed slices with fixed CLI semantics and explicit sidecar data-retention behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Scope sidecar writes to one default agent:** `extensions/memory-core/src/cli.runtime.ts:2255`
  - body: The sidecar write commands reuse `resolveAgentIds`, which expands an omitted `--agent` to every configured agent. The docs and option text say these commands default to the default agent, so a plain `pin`, `status`, `salience`, or `supersede` can mutate multiple sidecar databases unintentionally. Resolve a single default agent unless `--agent` is explicit.
  - confidence: 0.91
- **[P2] Keep stats and list read-only:** `extensions/memory-core/src/cli.runtime.ts:2000`
  - body: The read-only `stats`/`list` paths open existing DBs with `openSidecarDatabase`, which always runs schema initialization and upserts metadata/ALTERs columns. That means inspection can write to disk and fail in read-only contexts; use a no-migration read opener for these commands.
  - confidence: 0.88
- **[P2] Fail rejected ref resolutions:** `extensions/memory-core/src/cli.runtime.ts:2229-2231`
  - body: When a ref prefix is missing or ambiguous, the renderer only logs the rejection and the command still exits successfully. Scripts cannot distinguish a skipped mutation from a successful one, so set a non-zero exit code whenever any requested write is rejected.
  - confidence: 0.9
- **[P2] Fail invalid sidecar arguments:** `extensions/memory-core/src/cli.runtime.ts:2283-2289`
  - body: Invalid sidecar arguments such as an unknown status or salience value only print a warning and return without setting a failing exit code. Automation will treat the command as successful even though no mutation happened; these validation failures should set `process.exitCode` before returning.
  - confidence: 0.87
- **[P2] Touch only surfaced search hits:** `extensions/memory-core/src/memory-v2/rerank/wrapper.ts:90-100`
  - body: `shadowOnRecall` records touches for every candidate passed into the wrapper, but `memory_search` sorts and slices after this call. Rows that are dropped by `maxResults` can still get `last_accessed_at` refreshed, which skews later recency scoring toward content the user never saw. Move the touch step to the final surfaced list or pass that list into the wrapper.
  - confidence: 0.84
- **[P2] Add a deterministic duplicate-location tie-breaker:** `extensions/memory-core/src/memory-v2/rerank/lookup.ts:31-35`
  - body: Duplicate `location_id` rows with equal salience and `last_accessed_at` are selected according to SQLite's unspecified row order because the query has no `ORDER BY` and `preferCandidate` has no final tie-break. Add a stable SQL ordering or compare a stable field such as `ref_id` so rerank output cannot drift between runs.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The PR introduces an opt-in but persistent conversation-derived sidecar, so privacy and redaction behavior need explicit maintainer review before merge.

Concerns:

- **[medium] Define sidecar retention and redaction guarantees:** `extensions/memory-core/src/memory-v2/ingest/handler.ts:149`
  - body: When ingest is enabled, the patch extracts the last user message and stores normalized candidate text in `memory_v2_ingest_text`; the secret filter is explicitly best-effort and notes that a stronger redactor is future work. That is a concrete new local data-retention surface and should be reviewed before bundling.
  - confidence: 0.82
- **[medium] Prevent unintended cross-agent sidecar mutations:** `extensions/memory-core/src/cli.runtime.ts:2255`
  - body: The new mutating sidecar CLI commands can target all configured agents when `--agent` is omitted, contrary to their default-agent contract. Besides correctness, this broadens the data-integrity scope of a single operator command.
  - confidence: 0.86
