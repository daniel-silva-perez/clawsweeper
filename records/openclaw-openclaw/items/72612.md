## Summary

Keep this PR open. Current main still uses stale replacement-style chat attachment updates, reads selected/dropped/pasted files with FileReader before a browser-side size guard, and does not expose the gateway-effective attachment limit through Control UI bootstrap. The PR remains a focused implementation candidate; #33404 is closed unmerged broader prior art, and #67916 is a separate local assistant-media bootstrap issue.

## Best Possible Solution

Keep this PR open for maintainer review and landing if CI remains clean. The best fix is this focused change or an equivalent one: append completed file reads through live attachment state, reject files over the gateway-effective agents.defaults.mediaMaxMb limit before FileReader.readAsDataURL(), surface attachment errors in the chat callout, pass the effective limit through Control UI bootstrap, and keep the targeted chat/bootstrap/gateway regression tests.
