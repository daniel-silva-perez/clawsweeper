## Summary

Keep this PR open. Current main still has the stale async attachment append path, no browser-side pre-read size guard, no attachment error callback, no Control UI bootstrap exposure for the effective attachment limit, and no regression tests for the race or oversize handling. The PR is a focused implementation candidate; #33404 is closed unmerged prior art, and #67916 is a separate local assistant-media bootstrap fix rather than a superseding change.

## Best Possible Solution

Keep this PR open for maintainer review and landing if CI/review stay clean. The best fix is this focused change or an equivalent one: append completed file reads through live attachment state, reject files over the gateway-effective `agents.defaults.mediaMaxMb` limit before `FileReader.readAsDataURL()`, surface attachment errors in the chat callout, pass the effective limit through Control UI bootstrap, and keep the targeted chat/bootstrap regression tests.
