## Summary

Keep this PR open. Current main still has the stale async attachment append path, lacks a browser-side pre-read attachment size guard and attachment error callback, and does not expose the gateway-effective attachment limit to the Control UI bootstrap config. The PR is a focused implementation candidate, and the related items do not supersede it: #33404 is closed unmerged prior work, while #67916 tracks a different local assistant-media bootstrap bug.

## Best Possible Solution

Keep this PR open for maintainer review and landing if CI/review stay clean. The best fix is this focused change or an equivalent one: append completed file reads through live attachment state, reject files over the gateway-effective `agents.defaults.mediaMaxMb` limit before `FileReader.readAsDataURL()`, surface attachment errors in the chat callout, pass the effective limit through Control UI bootstrap, and keep the targeted chat/bootstrap/gateway regression tests.
