## Summary

Keep open. Current main still has the stale async attachment update behavior and lacks the browser pre-read limit, while the PR is a focused implementation with no blocking review findings from this pass.

## What This Changes

The PR branch changes Control UI chat attachment handling and bootstrap config so uploads append through live state, reject oversized files before reading, surface upload errors, and cover the behavior with UI/gateway tests.

## Best Possible Solution

Land this PR or an equivalent focused change after maintainer review and the normal changed-gate, keeping the fix at the Control UI compose/bootstrap boundary and leaving the gateway as the authoritative attachment validator.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: trigger overlapping paste/select/drop file reads and each callback can write from an old `props.attachments` snapshot; selecting a file above the effective gateway limit reaches `FileReader.readAsDataURL()` before any browser-side size guard.

## Solution Assessment

Yes. The proposed fix is the narrow maintainable direction: append completed reads through live UI state, expose the existing gateway-effective limit through bootstrap, reject oversized files before reading, and add targeted regression coverage without changing transport or model execution semantics.
