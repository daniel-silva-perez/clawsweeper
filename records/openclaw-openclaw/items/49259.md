## Summary

Keep open. Current main still provides manual selected-session deletion in the Dashboard and generic CLI/store cleanup, but it does not implement a Dashboard prune workflow with a 48-hour default, candidate preview for stale orphaned sessions, or Telegram/Discord channel/topic existence checks. The related #50310 is narrower Discord cleanup work and remains open, so it does not supersede this broader request.

## Best Possible Solution

Keep this issue open. The best path is a gateway-backed Dashboard prune workflow: add an additive preview RPC that returns stale orphan candidates with reasons, expose plugin-owned Telegram/Discord reachability checks through a safe generic seam, revalidate candidates during deletion, and wire the Sessions screen to show a 48-hour default age input, candidate preview, and explicit prune action. Keep #50310 as the narrower Discord thread-session cleanup item rather than treating it as a duplicate.
