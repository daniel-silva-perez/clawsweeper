## Summary

Keep this issue open. Current main still provides manual selected-session deletion and generic session-store maintenance, but it does not implement the requested Dashboard Sessions prune workflow with a 48-hour default age field, orphan-candidate preview, or Telegram/Discord channel/topic existence checks. Related #50310 remains open and narrower Discord thread-session cleanup work, not a replacement for this broader Dashboard feature.

## Best Possible Solution

Keep this issue open. The best path is an additive gateway-backed prune workflow: add a preview RPC that returns stale orphan candidates with reasons, expose safe plugin-owned Telegram/Discord reachability checks through a generic seam, revalidate candidates during apply, and wire Dashboard Sessions to show a 48-hour default age field, candidate preview, and explicit Prune action. Keep #50310 as the narrower Discord cleanup thread rather than treating it as a duplicate.

## Risks / Open Questions

- A prune implementation must distinguish confirmed deletion from transient Telegram/Discord API failures, auth failures, permission changes, and rate limits before deleting session records.
- Reusing the existing age-only maintenance path would not satisfy the request and could remove reachable sessions without channel/topic verification.
- The cleanest fix likely needs a generic gateway/plugin seam for channel reachability so core does not hard-code Telegram or Discord behavior.
