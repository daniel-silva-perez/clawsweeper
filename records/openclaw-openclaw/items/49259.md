## Summary

Keep open. Current main still lacks the requested Dashboard Sessions prune workflow: it has manual selected-session deletion and generic local session-store maintenance, but no 48-hour default age field, orphan-candidate preview, or Telegram/Discord channel/topic reachability checks. Related #50310 remains open and narrower Discord cleanup work rather than a replacement for this broader Dashboard feature.

## What This Changes

Request a Dashboard Sessions workflow that previews and prunes stale orphaned Telegram or Discord-backed sessions older than a configurable age, defaulting to 48 hours.

## Best Possible Solution

Add an additive, gateway-backed prune preview/apply workflow. It should return stale orphan candidates with explicit reasons, use plugin-owned Telegram/Discord reachability checks through a generic seam, revalidate during apply, and wire Dashboard Sessions to show the 48-hour default age field, candidate preview, and explicit Prune action while keeping #50310 as narrower Discord cleanup context.

## Reproduction Assessment

Not applicable as a feature request; the requested workflow is absent by source inspection. Current main can verify the gap through the Sessions UI, controller, protocol schema, and local-store list/cleanup code without needing live Telegram or Discord credentials.

## Solution Assessment

Yes, keeping this open is the best outcome. The maintainable fix is an additive preview/apply contract with safe plugin reachability semantics, not reusing manual selected delete or the generic age-based cleanup path.
