## Summary

Keep open. Current main has a generic descriptor foundation, but not the requested Control UI contribution-slot model or client rendering behavior, and the latest discussion explicitly keeps the broader RFC active while related PRs continue underneath it.

## What This Changes

Add data-driven Control UI plugin contribution slots for plugin chat modes, approval cards, event classifiers, input guards, and status surfaces.

## Best Possible Solution

Keep this as the Control UI contribution-slot design thread. Maintainers should decide the slot taxonomy and projection model, then land focused additive follow-ups that include host-owned descriptor schemas, Gateway projection, Control UI rendering, disablement semantics, sanitization, and fixture coverage without arbitrary plugin browser JavaScript.

## Reproduction Assessment

Not applicable; this is an RFC/feature request, not a bug report. The verification path is source/docs inspection against current main, which shows generic descriptor plumbing but no broader slot vocabulary or Control UI consumer.

## Solution Assessment

Unclear; the requested direction is plausible, but the best maintainable shape needs maintainer API judgment. A manifest-first, host-owned component slice for panels/status may be a safer first increment than a broad mutable runtime registry.
