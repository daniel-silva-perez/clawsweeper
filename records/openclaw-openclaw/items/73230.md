## Summary

Keep open. Current main still treats cron `delivery.accountId` as a preserved explicit value, and delivery target resolution gives that value priority over session- or binding-derived accounts. I found no implemented `auto`/`active` sentinel, no stale-account fallback, and no startup warning/repair path for rotated channel-plugin account IDs; the WeChat plugin is external, but the failure is in a generic cron delivery contract.

## What This Changes

The issue requests dynamic cron delivery account resolution, or warning/repair for stale stored account IDs, for channel plugins such as `openclaw-weixin` whose account IDs can rotate after login or Gateway restart.

## Best Possible Solution

Implement a channel-agnostic cron account-resolution path that preserves explicit account IDs when they still resolve, but can recover or fail clearly when a stored account ID is stale. The maintainable end state is a documented generic rule, backed by tests and SDK/plugin seams if active account state is needed, with no WeChat-specific branch in core.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: create an isolated cron delivery with a stale `delivery.accountId`, then `resolveDeliveryTarget` will return that stale value because current code gives it highest precedence over session and binding fallbacks. I did not run a live WeChat plugin test because this review is read-only and the static path already demonstrates the reported contract gap.

## Solution Assessment

Yes for the requested direction, if implemented generically. The best fix is not a startup migration or WeChat special case; it is delivery-time account resolution with an unambiguous single-account fallback or typed active-account plugin capability, plus clear errors when resolution is unsafe.

## Repair Work Prompt

Implement a conservative generic fix for https://github.com/openclaw/openclaw/issues/73230. Current main preserves stale `delivery.accountId` and gives it highest precedence during cron delivery target resolution. Add delivery-time account resolution without hardcoding `openclaw-weixin`: preserve an explicit accountId when it still resolves for the selected channel, but when it is absent or stale, recover only if the selected channel has exactly one safe configured/enabled/active account available through existing channel plugin config/status surfaces.
