## Summary

Keep open and do not merge as-is: current main still defaults `openclaw logs` to UTC unless `--local-time` or a valid `TZ` is present, so the user problem is real, but this PR does not implement the promised UTC opt-out and also mixes in an unrelated security-sensitive downgrade of escaped skill path warnings.

## What This Changes

The branch changes the logs CLI `--local-time` option default from `false` to `true` and downgrades escaped skill path logging from `warn` to `debug`.

## Best Possible Solution

Land a focused logs CLI/docs/tests change, either by updating this PR or choosing a cleaner canonical PR, that defaults human-readable log output to local time, provides an explicit UTC opt-out independent of `TZ`, and leaves escaped skill path logging unchanged unless that security trade-off is reviewed separately.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Add a real UTC opt-out:** `src/cli/logs-cli.ts:209`
  - body: This flips the default to local time but still only declares the positive `--local-time` option. Commander does not accept an implicit `--no-local-time`, and the existing `localTime` expression would also let a valid `TZ` force local output even after an explicit false value. That breaks the PR's promised way to restore UTC output.
  - confidence: 0.93
- **[P2] Keep escaped-path warnings visible:** `src/agents/skills/workspace.ts:193`
  - body: This unrelated downgrade moves escaped skill path notices from warn to debug. The helper is used for workspace and bundled path escapes, and current tests assert these messages are warn-level console warnings, so normal operators would lose visibility into skipped path-escape cases.
  - confidence: 0.94
