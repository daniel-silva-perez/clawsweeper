## Summary

Keep open: the requested logs behavior is still valid, but this PR is not safe to merge as-is because its advertised UTC opt-out is not actually registered and it bundles an unrelated downgrade of escaped skill-path warnings. Current main still treats local log timestamps as opt-in, so this is useful work that needs maintainer/author revision rather than closure.

## What This Changes

The PR changes `openclaw logs` so `--local-time` defaults to `true` and downgrades escaped skill-path diagnostics from `warn` to `debug`.

## Best Possible Solution

Land a focused logs-only change that makes human-readable `openclaw logs` output local time by default, provides a documented UTC mode that cannot be overridden by `TZ`, updates `docs/cli/logs.md`, adds regression tests for default local output and UTC opt-out behavior, and leaves escaped skill-path warning visibility unchanged unless reviewed separately.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: current main has UTC-by-default tests and opt-in `--local-time`, while this PR only flips the positive option default and does not add a registered negated option or equivalent UTC flag.

## Solution Assessment

No. Defaulting log display to local time is a reasonable fix direction, but this patch is not the best implementation because it lacks the promised opt-out, omits docs/tests for the behavior change, and mixes in an unrelated security-observability change.
