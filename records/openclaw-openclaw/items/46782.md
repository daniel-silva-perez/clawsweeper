## Summary

Keep this PR open for maintainer follow-up. The intended default-local log display is still not implemented on current main, and the submitted patch is not merge-ready because the advertised UTC opt-out is not actually registered and the branch also hides unrelated security-relevant skill path warnings.

## What This Changes

The PR changes the `openclaw logs` `--local-time` option default from `false` to `true` and downgrades escaped skill-path logging from `warn` to `debug`.

## Best Possible Solution

Land or replace with a focused logs-only fix for #46748: default human-readable `openclaw logs` and `openclaw logs --follow` output to local timestamps, provide a tested UTC opt-out such as `--no-local-time` or `--utc`, preserve JSON/raw semantics, update `docs/cli/logs.md` and `docs/logging.md`, and keep escaped skill-path warnings visible unless a separate owner-reviewed change narrows the noisy symlink case.
