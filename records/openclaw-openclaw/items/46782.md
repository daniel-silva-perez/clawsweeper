## Summary

Keep open: current main still keeps `openclaw logs` local timestamps opt-in, and this PR has blocking issues before it can be merged. The logs direction is useful, but the branch advertises a UTC escape hatch that is not actually registered and mixes in an unrelated downgrade of escaped skill-path warnings.

## What This Changes

The PR changes `openclaw logs --local-time` to default to `true` and changes escaped skill-path diagnostics in `src/agents/skills/workspace.ts` from `warn` to `debug`.

## Best Possible Solution

Land a focused logs-only change that makes human-readable `openclaw logs` output local time by default, keeps existing `--local-time` compatibility, provides a documented UTC opt-out, updates `docs/cli/logs.md`, adds regression tests, and leaves escaped skill-path warning visibility unchanged unless reviewed separately.

## Reproduction Assessment

Yes. On current main, `openclaw logs --plain` without `--local-time` and without a valid `TZ` follows the UTC `toISOString()` path; this PR can also be checked source-side because it does not add a registered `--no-local-time` option.

## Solution Assessment

No. Defaulting human log output to local time is a reasonable direction, but this patch is not the best implementation because it lacks the promised opt-out, omits docs/tests for the changed contract, and includes an unrelated security-observability change.
