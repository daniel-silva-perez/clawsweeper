## Summary

Keep open: current main still has the UTC-by-default logs behavior, but this PR is not safe to merge as-is because it lacks the promised UTC escape hatch and includes an unrelated security-sensitive logging downgrade. The same logs problem is also tracked by open issue #46748 and another open implementation PR, #47365, so a maintainer should choose or revise the canonical path.

## What This Changes

The PR changes the logs CLI `--local-time` option default to `true` and downgrades escaped skill-path logging from `warn` to `debug`.

## Best Possible Solution

Land a focused logs CLI, docs, and tests change that defaults human-readable log output to local time, provides an explicit UTC opt-out independent of `TZ`, and leaves escaped skill-path warnings unchanged unless that security trade-off is handled in a separate review.

## Reproduction Assessment

Yes. Current source and tests show UTC is still the default unless `--local-time` or a valid `TZ` path enables local formatting, and the PR diff does not register a real `--no-local-time` or equivalent UTC opt-out.

## Solution Assessment

No. Defaulting logs to local time is a reasonable direction, but this diff is not the best fix because it mixes an unrelated warning downgrade with the timezone change and does not implement the advertised UTC escape hatch with tests/docs.
