## Summary

Keep open. Current main still leaves the logs timezone change unimplemented, but this PR is not mergeable as-is because it only flips the positive `--local-time` default, does not implement the advertised UTC opt-out or update docs/tests, and also includes an unrelated security-observability regression in skill path warning logs.

## What This Changes

The PR changes `openclaw logs` so `--local-time` defaults to `true` and downgrades escaped skill-path reporting from `warn` to `debug`.

## Best Possible Solution

Land a focused logs-only change that makes human-readable `openclaw logs` output local time by default, provides an explicit UTC mode that cannot be overridden by `TZ`, updates `docs/cli/logs.md`, adds regression tests for default local output and UTC opt-out behavior, and leaves escaped skill-path warning visibility unchanged unless reviewed separately.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: current main has UTC-by-default tests and an opt-in `--local-time`, while this PR only flips the positive option default and does not register the promised `--no-local-time` or an equivalent UTC flag.

## Solution Assessment

No. Defaulting logs to local time is a reasonable direction, but this patch is not the best implementation because it lacks the advertised opt-out, omits docs/tests, and bundles an unrelated security-sensitive logging downgrade.
