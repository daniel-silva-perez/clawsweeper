Keep open. Current main contains the merged #72287 host-hook foundation, but not this PR's workflow action, outbound attachment, scheduled wakeup, priority injection, or bounded finalize-retry layer. Exact source searches find no `plugins.sessionAction`, `registerSessionAction`, `scheduleSessionTurn`, `sendSessionAttachment`, `PluginSessionAction`, or `host-hook-workflow` implementation on main. The related follow-up PRs are active stack layers, not replacements, and the PR still has review findings that need maintainer resolution before merge.

## Best Possible Solution

Keep this PR open as the workflow-seam implementation layer above #72287. Maintainers should review it after the foundation merge state, resolve the outstanding scheduler/protocol and security-sensitive review threads, then either land this layer with exact-head SDK/protocol/test validation or explicitly reject the workflow APIs and update the follow-up fixture/docs PRs accordingly.

## Risks / Open Questions

- The PR is security-sensitive if merged: it adds public SDK and Gateway protocol surfaces for session actions, outbound attachments, and scheduled wakeups, including admin-scoped scheduler behavior. Maintainers should re-check trusted/bundled gating, session ownership, quotas, cleanup, and attachment path boundaries on the exact head.
- The latest provided review context includes a P1 finding that `schedulePluginSessionTurn` adds unsupported fields to a `cron.add` payload; current main's cron agent-turn payload schema has `additionalProperties: false`, so that review thread needs resolution before this workflow seam can be considered ready.
- This is a large public contract change spanning SDK types, Gateway protocol, generated Swift models, scheduler/runtime cleanup, outbound delivery, and agent lifecycle behavior. It needs exact-head CI and generated baseline proof; current main only proves the lower #72287 foundation layer.
- Because #72384 and #72333 are stacked fixture/docs follow-ups, landing or closing this PR out of order would leave the review stack inconsistent.
