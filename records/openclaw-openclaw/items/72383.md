## Summary

Keep this PR open. Current main contains the merged #72287 host-hook foundation, but it still does not implement this PR’s workflow action dispatch, host-mediated outbound attachments, scheduled session wakeups, priority next-turn ordering, plugin-emitted workflow events through OpenClawPluginApi, or bounded finalize-retry contract. The PR is an active public SDK/Gateway contract change with security-sensitive review threads, not conservative cleanup.

## Best Possible Solution

Keep this PR open as the workflow-seam implementation layer above #72287. Maintainers should review or reject the proposed SDK/Gateway contracts explicitly, resolve the remaining security-sensitive review threads, and require protocol/API generation plus focused host-hook contract tests before landing. If the APIs are accepted, this PR or an equivalent implementation should add typed session actions, host-mediated attachments, scheduled session turns, priority next-turn injections, trusted session reads, and bounded finalize retry semantics on top of the already-merged foundation.
