## Summary

Keep this PR open. Current main still does not implement outbound view-once media for WhatsApp or Signal, and the PR remains an implementation candidate rather than obsolete. It does need a rebase and more review: the branch is currently unmergeable against main, it touches an old WhatsApp shared path that no longer exists, and the latest review comments raise unresolved privacy/semantic issues around preflight media fetching and WhatsApp auto-reply text leakage.

## Best Possible Solution

Keep the PR open as an implementation candidate, but require a rebase onto current main, move the WhatsApp threading into the current extensions/whatsapp/src/outbound-base.ts and outbound type surfaces, add focused tests for agent tool schema, gateway validation, queue recovery, WhatsApp/Signal media-kind validation, text-only rejection, unsupported-channel rejection, and multi-media failure behavior, then resolve the two remaining review comments before maintainer merge consideration.
