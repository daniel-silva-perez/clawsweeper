## Summary

Keep this PR open. Current main still lacks outbound view-once support for WhatsApp and Signal, so the contribution is not obsolete, but the proposed branch still has privacy-sensitive correctness issues that should block merge until maintainer review and patch revisions happen.

## What This Changes

This PR adds a `viewOnce` send option across the CLI, agent tool schema, gateway send protocol, outbound delivery and queue recovery, WhatsApp Baileys media sends, Signal JSON-RPC sends, and auto-reply forwarding paths.

## Best Possible Solution

Land a rebased, maintainer-approved implementation that keeps the `viewOnce` contract additive, documents the CLI/tool/gateway behavior, supports only media kinds actually honored by each channel, validates unsupported cases before any delivery starts, preserves queue/recovery semantics, and avoids fetching remote media twice for validation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Reject Signal view-once videos before RPC send:** `extensions/signal/src/send.ts:179-180`
  - body: `signal-cli` documents view-once as supported only for messages with an image attachment, but this branch treats `attachmentKind === "video"` as supported and sends `"view-once": true`. A Signal video send can therefore reach the daemon as an unsupported view-once request instead of failing predictably.
  - confidence: 0.91
- **[P2] Avoid preflight media downloads for view-once validation:** `src/infra/outbound/deliver.ts:698-701`
  - body: This validation loads every media URL, but the WhatsApp and Signal senders load the same URL again for the actual send. One-shot or short-lived URLs can be consumed by validation, so an otherwise valid view-once send fails before delivery.
  - confidence: 0.86
- **[P2] Reject view-once replies that still send permanent text:** `extensions/whatsapp/src/auto-reply/deliver-reply.ts:151`
  - body: Forwarding `replyResult.viewOnce` only on media payloads leaves the existing text-only and remaining-text branches untouched. A view-once auto-reply with no media, or a long caption split into later chunks, will still send permanent WhatsApp text despite the disappearing-media flag.
  - confidence: 0.79
