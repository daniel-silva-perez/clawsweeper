## Summary

Keep open. Current main still does not implement outbound WhatsApp or Signal view-once media, and the PR remains relevant but needs maintainer review because it expands public send/tool/gateway contracts and privacy-sensitive channel behavior.

## What This Changes

This PR branch adds a `viewOnce` send option across the agent tool schema, CLI send command, gateway send protocol, outbound delivery and queue recovery, WhatsApp Baileys media sending, Signal JSON-RPC sending, and auto-reply forwarding paths.

## Best Possible Solution

Land a rebased, maintainer-approved implementation after the public contract is accepted: the flag should be additive, fail closed before any send, avoid consuming single-use media during validation, preserve queue/recovery semantics, document CLI/tool behavior, and cover direct plus auto-reply paths for WhatsApp and Signal.
