## Summary

Keep this PR open. Current main still lacks outbound WhatsApp/Signal view-once support, and the PR remains useful implementation work, but it needs maintainer review because it changes public send/tool/gateway contracts and has privacy-sensitive fail-closed behavior still under discussion.

## What This Changes

This PR branch adds a `viewOnce` send option across agent tool, CLI, gateway, outbound delivery, queue recovery, WhatsApp Baileys sending, Signal JSON-RPC sending, and auto-reply forwarding paths.

## Best Possible Solution

Land a rebased, maintainer-approved implementation only after the public contract is accepted: the flag should be additive, fail closed before any send, avoid consuming single-use media during validation, preserve queue/recovery semantics, document CLI/tool behavior, and cover direct plus auto-reply paths for WhatsApp and Signal.
