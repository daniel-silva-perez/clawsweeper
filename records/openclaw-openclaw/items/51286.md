## Summary

Keep this PR open. Current main still does not normalize non-agent-scoped colon session keys in `openclaw sandbox explain`, so the PR remains a focused implementation candidate for the linked Telegram slash-session diagnostic bug.

## Best Possible Solution

Keep this PR as a focused implementation candidate. If `openclaw sandbox explain --session telegram:slash:<id>` remains a supported diagnostic path, rebase it, keep the downstream `channel` and `allowedByConfig` assertions, and land it with targeted sandbox-explain test proof. If maintainers want agent-scoped keys only, update the slash-command/sandbox docs and resolve linked #51245 accordingly instead of closing this as already implemented.
