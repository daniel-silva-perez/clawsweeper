## Summary

Keep this open. Current main still has a finite subagent announce retry-limit give-up path, best-effort completion dispatch can still return an undelivered result, timeout handling finalizes a timed-out run rather than defining an auto-restart or immediate user-notification policy, and the same-day discussion reports the warning still reproducing on v2026.4.26. Open PR #45200 is relevant but only a partial Pattern 1 mitigation, not a complete fix for the broader durable-delivery, timeout-policy, dedupe, and result-verification request.

## What This Changes

The issue requests reliable subagent completion handling so announce failures, timeouts, and unverifiable file-result claims do not silently lose user-visible results.

## Best Possible Solution

Keep this as the canonical tracking issue for reliable async subagent completion semantics. The desired end state is an explicit completion and delivery state model that separates child completion, parent-session recording, external channel delivery, timeout policy, deduplication, and result/file verification, with regression coverage for fast responders and Telegram-style async delivery.
