## Summary

Keep this open. Current main has useful subagent delivery hardening, but the central bug is still present as an explicit finite give-up path: announce retries can exhaust, delivery is marked failed, completion-mode dispatch can still return undelivered, and timeout handling records a timeout instead of restarting or guaranteeing user-visible notification. The same retry-limit warning was also reported as still reproducing on April 29, 2026.

## What This Changes

The issue requests reliable subagent completion handling: retry and surface failed announces, notify or restart on timeouts, and verify claimed file outputs before reporting success.

## Best Possible Solution

Keep this as the canonical tracking issue for reliable async subagent completion semantics. The desired end state is an explicit delivery state model that separates child completion, parent-session recording, external channel delivery, retry durability, timeout or restart policy, duplicate prevention, and result/file verification, with regression coverage for fast responders and Telegram-style async delivery.
