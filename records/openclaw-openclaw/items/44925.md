## Summary

Keep open. Current main and the latest release still contain terminal subagent announce give-up paths, completion dispatch can still end as undelivered when direct and queue fallback both fail, timeout handling records a timeout rather than defining restart semantics, and the same-day issue discussion reports the retry-limit warning still reproducing with fast responders.

## What This Changes

The issue asks OpenClaw to prevent async subagent results from being silently lost by retrying or surfacing announce failures, defining timeout recovery or notification behavior, and verifying file/result claims before reporting completion.

## Best Possible Solution

Keep this as the canonical tracking issue for reliable async subagent completion semantics. The desired end state is an explicit delivery state model that separates child completion, parent-session recording, user-visible channel delivery, timeout policy, duplicate prevention, and result/file verification, with regression coverage for fast responders and Telegram-style async delivery.
