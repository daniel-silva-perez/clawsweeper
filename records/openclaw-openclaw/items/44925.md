## Summary

Keep this open. Current main still has documented and implemented terminal announce give-up paths, completion dispatch can still return an undelivered direct failure when queue fallback also fails, timeout handling finalizes a timed-out run rather than defining auto-restart semantics, and same-day issue discussion reports the retry-limit warning still reproducing on v2026.4.26.

## What This Changes

The issue requests reliable subagent completion delivery, timeout notification or restart behavior, and verification of claimed file outputs so subagent results are not silently lost.

## Best Possible Solution

Keep this as the canonical tracking issue for reliable async subagent completion semantics. The desired end state is an explicit delivery state model that separates child completion, parent-session recording, external channel delivery, timeout policy, deduplication, and result verification, with regression coverage for fast responders and Telegram-style async delivery.
