## Summary

Current main still has the reported behavior: `readLogSlice` sets `reset = true` when an existing cursor is valid but the file grew by more than `maxBytes`, and `openclaw logs` renders any `payload.reset` as `Log cursor reset (file rotated).` The PR is a narrow, relevant fix with regression tests, and it is not implemented on main or superseded by the related logging items in the provided context. Security review found no workflow, dependency, package, install, secret-handling, network, or code-execution surface changes in the provided PR file list.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best fix is the PR's narrow semantic split: leave `reset` true only when the previous cursor is past EOF, keep `truncated` true for byte-budget skips, and land focused regression tests for both fast growth and real shrink/rotation. Because this changes a user-visible CLI notice, maintainers should decide whether to add a one-line changelog Fixes entry before merge.
