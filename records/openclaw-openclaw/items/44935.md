## Summary

Keep open. Current main still only probes cooldowned primaries inside active fallback attempts; it has sticky auto fallback overrides and per-run observability, but it does not provide the requested configurable background recovery loop or recovered-primary routing signal for new sessions, subagents, and cron work.

## What This Changes

Request health-probe-based automatic recovery from fallback models back to the primary model, with sticky existing sessions, recovered-primary routing for new work, configurable probe cadence, and recovery notification support.

## Best Possible Solution

Design a core fallback-recovery state machine that reuses existing model/auth probe contracts, records recovered-primary availability for future work, keeps active sessions pinned to their selected fallback, exposes documented configuration, and coordinates recovery notifications with #44941.
