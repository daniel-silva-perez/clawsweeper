## Summary

Keep open. The earlier ClawSweeper review is still accurate on current main: OpenClaw now has sticky fallback state, per-run cooldown probe decisions, and fallback lifecycle notices, but it does not yet have the requested configurable background health-probe recovery loop that marks the primary healthy for future work while keeping existing sessions pinned.

## What This Changes

The issue requests health-probe-based auto-recovery from a fallback model back to the primary model for new sessions, subagents, and scheduled work, with sticky active sessions and recovery notifications.

## Best Possible Solution

Design a core fallback-recovery state machine that reuses existing model/auth probe contracts, keeps active sessions pinned to their selected fallback, exposes documented configuration, records recovered primary availability for new sessions/subagents/cron work, and coordinates recovery notifications with #44941.
