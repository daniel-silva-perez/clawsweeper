## Summary

Keep open. Current main has per-run cooldown probing and sticky fallback session state, but it still does not implement the requested configurable background health-probe recovery loop that marks a recovered primary available for new sessions/subagents/cron work while keeping existing sessions pinned.

## What This Changes

Request configurable health-probe-based auto-recovery from fallback to the primary model for new work, with sticky existing sessions and recovery notifications.

## Best Possible Solution

Design a core fallback-recovery state machine that reuses existing model/auth probe contracts, persists or otherwise scopes recovered-primary availability for new work, keeps active sessions pinned to their selected fallback, exposes documented configuration, and coordinates recovery notifications with #44941.
