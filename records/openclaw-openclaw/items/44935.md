## Summary

Keep open. Current main has per-run cooldown probes and sticky auto fallback overrides, but it still lacks the requested background recovery loop, configurable probe interval, recovered-primary routing signal, and new-work recovery behavior for sessions, subagents, and cron runs.

## What This Changes

The issue requests health-probe-based model fallback recovery that returns new work to the primary model once healthy while keeping existing sessions pinned to their fallback model.

## Best Possible Solution

Design a core fallback-recovery state machine that reuses existing model/auth probe contracts, records recovered-primary availability for future work, keeps active sessions pinned to their selected fallback, exposes documented configuration, and coordinates recovery notifications with #44941.
