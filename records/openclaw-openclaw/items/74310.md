## Summary

Keep this open. Current main still contains the reported provider-namespace hazard, and the issue is already paired with open closing PR #74315, so the correct cleanup path is to review, adjust, and land that PR rather than closing the issue early.

## What This Changes

The issue asks OpenClaw to stop collapsing a configured `kimi-coding/k2p5` provider namespace into `kimi/k2p5` during model resolution and status display.

## Best Possible Solution

Review and land or revise PR #74315 so exact configured provider IDs such as `kimi-coding` survive default resolution, runtime execution, allowlist checks, stored overrides, and status display while preserving intentional `kimi`/`kimi-code` compatibility for the bundled Kimi provider.

## Reproduction Assessment

Yes. A source-level reproduction path is clear from current main: the configured ref can be parsed as `kimi-coding/k2p5`, but runtime selection and stored overrides pass it back through `normalizeModelRef`, which still normalizes the provider to `kimi`.

## Solution Assessment

Unclear. The requested behavior is valid, but the best implementation should be decided in the linked PR because removing `kimi-coding` from the global alias chain may be broader than a config-aware exact-provider preservation fix.
