## Summary

Keep this issue open. Current main still normalizes `kimi-coding` to `kimi`, and the runtime paths for configured defaults and stored overrides still call that normalizer after the exact configured-provider parse path. The issue is also paired with open PR #74315 using closing syntax, so the correct cleanup path is to review and land or close that PR rather than close the issue early.

The issue requests a model-selection fix that preserves `kimi-coding/k2p5` as a distinct configured provider namespace instead of rewriting it to `kimi/k2p5`.
