## Summary

Keep this PR open. Current main still lacks `fileId` in the shared message tool schema, and Slack still advertises `download-file` while requiring `fileId` at dispatch/runtime. The related #64429 work is complementary, not a replacement: it concerns propagating Slack file IDs in media results, while this PR addresses the tool schema gap for #45574.

## Best Possible Solution

Keep this PR open as a plausible fix for #45574. Land a scoped schema fix for Slack `download-file`, ideally with a regression test; if maintainers prefer provider-owned parameters, move the `fileId` schema addition into Slack discovery via the existing plugin schema contribution seam.
