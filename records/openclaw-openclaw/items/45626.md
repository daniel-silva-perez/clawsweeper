## Summary

Keep this PR open. Current main still lacks `fileId` in the shared `message` tool schema, while Slack still advertises `download-file` and requires `fileId` at dispatch/runtime. The related #64429 work is complementary because it concerns carrying Slack file IDs in media results, not exposing the tool parameter needed by #45574.

## Best Possible Solution

Keep this PR open as a plausible fix for #45574. Land a scoped schema fix, ideally with a regression test; if maintainers prefer provider-owned parameters, move the `fileId` schema addition into Slack discovery through the existing plugin schema contribution seam. Treat #64429 as complementary rather than superseding.
