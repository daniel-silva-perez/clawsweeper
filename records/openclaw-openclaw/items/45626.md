## Summary

Keep this PR open. Current main still omits `fileId` from the shared `message` tool schema while Slack still advertises `download-file` and requires `fileId` at dispatch/runtime. The related #64429 work is complementary because it propagates Slack file IDs through media results, not the tool schema parameter this PR adds.

## Best Possible Solution

Keep this PR open as a plausible fix for the linked Slack bug. Land a scoped schema fix with a regression test; if maintainers prefer provider-owned parameters, move the `fileId` schema addition into Slack discovery through the existing plugin schema contribution seam. Treat #64429 as complementary rather than superseding.
