## What This Changes

The PR adds non-null, non-array object guards to Control UI chat message normalization and text/thinking extraction, skips malformed content-array entries, and adds regression tests for null, undefined, and array-valued blocks.

## Best Possible Solution

Land a rebased defensive guard in the Control UI chat helpers that skips invalid content-array entries while preserving current normalized content variants and newer rendering behavior. Backend transcript persistence and Telegram failed-media delivery should stay out of scope unless a separate bug proves those layers still need changes.

## Reproduction Assessment

Yes. A high-confidence helper-level reproduction is a message whose `content` array contains `null`, `undefined`, or an array beside a valid text/thinking block; current main still reads `.type` on those malformed entries in the normalizer and extract helpers.

## Solution Assessment

Yes, directionally. A narrow non-null, non-array object guard at the UI normalization/extraction boundary is the maintainable fix boundary, but the final patch should be applied against current main rather than the stale branch snapshot.
