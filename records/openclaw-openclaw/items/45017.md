## Summary

Keep open. Current main still has the malformed-content dereference in the Control UI normalizer and thinking/raw-text extractors, so this PR is not implemented on main. The linked failed-media issue is already closed for the broader media-rendering crash, but this narrower null/undefined content-block hardening remains useful; the branch is currently dirty and needs a careful rebase before merge.

## What This Changes

The PR adds non-null, non-array object guards to Control UI chat message normalization and text/thinking extraction, skips malformed content entries, and adds regression tests for null, undefined, and array-valued blocks.

## Best Possible Solution

Land a rebased defensive guard in the current Control UI chat helpers that skips invalid content-array entries while preserving all current normalized variants: attachments, canvas previews, assistant text expansion, tool-call/tool-result blocks, sender labels, and existing visible-text filtering. Backend transcript persistence and Telegram media delivery should stay out of scope unless a separate reproduction shows those layers still fail.

## Reproduction Assessment

Yes. A high-confidence helper-level reproduction is a message whose `content` array contains `null`, `undefined`, or an array beside a valid text/thinking block; current main still reaches `x.type` or `item.type` on those malformed entries in the normalizer and extract helpers.

## Solution Assessment

Yes, directionally, but the PR is not merge-ready as-is. The narrow UI-boundary guard is the right fix shape; it needs to be rebased or manually ported into the current normalizer implementation instead of replaying the older array-content mapper.
