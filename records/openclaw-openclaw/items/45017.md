## Summary

Keep open. Current main still exposes the malformed content-block crash path, so the PR is not obsolete; however the branch is stale against the current Control UI normalizer and should not merge until the guard is rebased onto the newer attachment, canvas, tool-block, and assistant directive handling.

## What This Changes

The PR adds non-null, non-array object guards to Control UI chat message normalization and text/thinking extraction, plus regression tests for null, undefined, and nested-array content blocks.

## Best Possible Solution

Land a rebased defensive guard in the current Control UI chat helpers that skips invalid content-array entries while preserving all current normalized variants. Backend transcript persistence and Telegram media delivery should stay out of scope unless a separate reproduction shows those layers still fail.

## Reproduction Assessment

Yes. A helper-level reproduction is a message whose `content` array contains `null`, `undefined`, or an array beside a valid text/thinking block; current main still reaches `x.type` or `item.type` on those malformed entries in the normalizer and extract helpers.

## Solution Assessment

No, not as currently diffed. The narrow UI-boundary guard is the right solution shape, but it needs to be rebased or manually ported into the current normalizer implementation instead of replaying the older array-content mapper.
