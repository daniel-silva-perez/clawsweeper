## Summary

Keep open. Current main and the latest release still dereference malformed `content` array entries in the Control UI chat helpers, while this PR adds the right narrow guard and regression coverage. The branch is currently dirty/unmergeable against newer chat-helper changes, so the remaining action is maintainer review and rebase, not cleanup closure.

## What This Changes

The PR adds non-null, non-array object guards in Control UI chat message normalization and text/thinking extraction, plus regression tests for malformed content blocks.

## Best Possible Solution

Land a rebased defensive guard for malformed content-array entries in the Control UI chat helpers, preserving current normalized content variants and covering null, undefined, and array-valued blocks with focused tests.

## Reproduction Assessment

Yes. A focused helper-level reproduction is a message whose `content` array contains `null` or `undefined` alongside a valid block; current `normalizeMessage`, `extractThinking`, and `extractRawText` still read `.type` on those entries without a guard.

## Solution Assessment

Yes directionally. The narrow UI helper guard is the right fix boundary, but the final merge should adapt it onto current main instead of applying the stale branch over newer chat rendering behavior.
