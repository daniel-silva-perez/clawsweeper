## Summary

Keep this PR open. Current main still does not implement the PR’s intended cron-store repair: legacy jobs missing `createdAtMs` can still be loaded into `cron.list`, even though the protocol and macOS models require both timestamps. The PR’s cron change remains useful, but its unrelated secret-scanning/Fastfile churn should be removed or split before merge.

## Best Possible Solution

Keep this PR open or replace it with an equivalent scoped patch. The best path is to backfill finite non-negative `createdAtMs` and `updatedAtMs` during cron store/service load, persist only when legacy storage is actually mutated, and extend regression coverage so both `cron.list` and the persisted store expose numeric timestamps for legacy jobs. Drop or split the unrelated `.secrets.baseline` and `apps/ios/fastlane/Fastfile` changes before merge.
