## Summary

Keep this PR open. Current main still does not implement the intended legacy cron timestamp backfill: store load can synthesize `updatedAtMs` but not missing `createdAtMs`, `cron.list`/`listPage` return stored jobs directly, and both the gateway protocol and macOS model require numeric timestamps. The PR remains a useful implementation candidate, but it needs rebase/scope cleanup for the current split cron store and should drop unrelated secret-baseline/iOS Fastfile changes.

## Best Possible Solution

Keep this PR open or replace it with an equivalent scoped patch. Rebase onto current main, implement finite non-negative `createdAtMs` and `updatedAtMs` backfill during cron store/service load, persist only when legacy storage is actually mutated, preserve legacy job visibility through `cron.list` and `listPage`, and update tests for both the in-memory response and the current split persisted files. Drop or split the unrelated secret-baseline and iOS Fastfile changes before merge.
