## Summary

Keep this PR open. Current main still does not implement the intended cron timestamp backfill: legacy jobs can be loaded with missing createdAtMs, cron.list/listPage returns stored jobs directly, and both the gateway protocol and macOS model still require createdAtMs and updatedAtMs. The branch remains useful, but it needs rebase/scope cleanup because it also carries unrelated secret-baseline and iOS Fastfile churn.

## Best Possible Solution

Keep this PR open or replace it with an equivalent scoped patch. The fix should backfill finite non-negative createdAtMs and updatedAtMs during cron store/service load, persist only when legacy storage is actually mutated, preserve legacy job visibility through cron.list/listPage, and add regression coverage for both the in-memory response and the current split persisted files. Drop or split the unrelated .secrets.baseline and apps/ios/fastlane/Fastfile changes before merge.
