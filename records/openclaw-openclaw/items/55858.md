## Summary

Keep open. Current main covers the already-noted per-channel mode support and now exposes `messages.queue.debounceMsByChannel`, but it still does not expose or resolve per-channel queue `cap` or `drop` overrides. The issue remains actionable as a narrow core config/runtime follow-up.

## Best Possible Solution

Keep this issue open and narrow it to the remaining core work: add backward-compatible per-channel queue cap/drop overrides, define the public config shape, update schema/help/docs, preserve existing mode and debounce behavior, and add focused resolver/overflow tests for channel-specific fallback order.
