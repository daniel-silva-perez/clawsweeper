## Summary

Keep open. Current main supports per-channel queue mode via `messages.queue.byChannel` and per-channel queue debounce via `messages.queue.debounceMsByChannel`, but queue `cap` and `drop` remain global-only in the public config schema and runtime resolver. The prior ClawSweeper keep-open comment remains accurate against current main.

## Best Possible Solution

Keep this issue open and narrow it to the remaining core work: add backward-compatible per-channel queue cap/drop overrides, choose and document the public config shape, update schema/help/docs, and add focused resolver and overflow tests for channel-specific fallback order.
