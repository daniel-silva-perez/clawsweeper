## Summary

Keep open. Current main still does not expose a browser `act` kind for CDP/IME-native text insertion, and the existing text paths still route through Playwright `type`/`fill` or Chrome MCP `fill`. The issue is recent, concrete, unprotected, and the prior Codex review comment also found the same missing capability.

## Best Possible Solution

Implement an additive browser plugin `act` kind such as `insertTextIme` or `cdpInsertText`. The change likely belongs in `extensions/browser`: update the tool schema, `BrowserActRequest`, `ACT_KINDS`, normalization, CLI/docs, `/act` dispatch, and regression tests. For managed/raw CDP profiles, focus the requested target and send CDP `Input.insertText`; for existing-session Chrome MCP profiles, either provide equivalent support or return a specific unsupported error.
