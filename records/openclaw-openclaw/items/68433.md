## Summary

Keep open. Current main still lacks outbound mirrored-session prewarm, and this PR remains the active implementation candidate for #68429, but the latest diff still skips prewarm when the stored snapshot filter matches without checking snapshot-version freshness. That means a workspace skill change can leave the outbound session stale until the first callback, preserving the cold-start path the PR is intended to avoid.

## What This Changes

This PR adds a mirrored-session prewarm helper, calls it after outbound session metadata is recorded, exports it through the lazy runtime boundary, and adds targeted regression tests alongside incidental formatting/test-expectation churn.

## Best Possible Solution

Keep this PR as the implementation path for #68429, but update prewarm so its skip condition uses the same snapshot freshness criteria as normal reply execution, including snapshot-version checks. Preserve the fix that leaves `systemSent` untouched, add a stale-snapshot regression test, and trim unrelated churn before merge if practical.
