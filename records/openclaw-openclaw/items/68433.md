## Summary

Keep this PR open. Current main does not already implement outbound session prewarming, and the PR is a useful implementation candidate for #68429, but the proposed helper can skip stale skill snapshot refreshes and leave the first callback paying the cold-start cost the PR is meant to avoid.

## What This Changes

The PR adds `prewarmMirroredSession`, calls it from outbound session metadata creation, extends session/outbound regression tests, and changes the ClawSweeper dispatch workflow's rate-limit handling.

## Best Possible Solution

Keep the PR open, update prewarm so it either lets `ensureSkillSnapshot` perform the full freshness decision or mirrors both version and filter checks before returning, add regression coverage for an existing stale snapshot, then run the targeted outbound/session tests and split or justify the workflow edit.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Refresh stale snapshots during prewarm:** `src/auto-reply/reply/session-updates.ts:244-248`
  - body: The new prewarm path returns as soon as an existing snapshot has the same skill filter. Normal reply execution also checks the cached snapshot version, so after a skills refresh this path can skip warmup and leave the first callback to rebuild the snapshot, reintroducing the cold-start this PR is meant to avoid. Let `ensureSkillSnapshot` make the freshness decision, or include the version check before returning.
  - confidence: 0.85
