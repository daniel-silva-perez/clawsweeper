## Summary

Keep open. Current main has `pnpm docs:list`, a human docs hub, `openclaw docs` search, and public `llms.txt`, but it does not provide the requested generated `docs_map.md`/`docs-map.md` with H2/H3/H4 heading hierarchy for LLM navigation.

## Best Possible Solution

Add a generated public `docs_map.md` sibling to `llms.txt`, built from the source docs with deterministic ordering and H2/H3/H4 extraction. Prefer a small Node stdlib generator wired into the existing docs sync/check path, with no new third-party actions, no broadened permissions, and tests for heading extraction and excluded docs directories.
