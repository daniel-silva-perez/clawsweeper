## Summary

Current main still contains the reported failure path: explicit `gateway.bind="loopback"` can resolve to `0.0.0.0` after a failed loopback preflight, and runtime config then rejects it with the exact error reported. Fix the macOS Gateway loopback bind regression so explicit `gateway.bind="loopback"` stays on `127.0.0.1` instead of degrading to `0.0.0.0` and failing startup/health after a loopback preflight failure.

Review and either land or deliberately reject #65635.
