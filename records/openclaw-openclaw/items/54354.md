## Summary

Keep this PR open. Current main has related compatibility mechanisms, but it does not implement the PRâ€™s central behavior: npm plugin install/update does not read `engines.openclaw`, does not query all npm versions for engine metadata, and does not select the newest compatible npm version. The PR is still a plausible implementation candidate, though it needs review fixes before merge.

## Best Possible Solution

Keep the PR open as useful but not merge-ready. The best path is to rebase onto current main, preserve npm tag/exact-version semantics, validate external registry metadata defensively, align the compatibility contract with existing `openclaw.install.minHostVersion` / ClawHub compatibility decisions, and add docs plus focused install/update tests for compatible fallback, no-compatible-version, exact-version, and dist-tag cases.
