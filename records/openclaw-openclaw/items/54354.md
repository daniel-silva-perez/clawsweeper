## Summary

Keep open. Current main has related compatibility protection through `openclaw.install.minHostVersion` and ClawHub metadata, but it does not implement this PR's npm `engines.openclaw` packument filtering, compatible-version selection, or `--ignore-engine` surface. The branch also has unresolved review findings in security-sensitive install/update behavior, so this needs maintainer product and security review rather than conservative cleanup closure.

## What This Changes

The PR adds `engines.openclaw`-aware npm version resolution for plugin install/update, custom semver and registry helpers, an `--ignore-engine` CLI flag, load-time compatibility warnings, and related tests.

## Best Possible Solution
