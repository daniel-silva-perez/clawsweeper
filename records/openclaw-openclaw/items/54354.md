## Summary

Keep open. Current main does not implement the PR's npm packument-based `engines.openclaw` version selection, and the existing shipped compatibility model is the different `openclaw.install.minHostVersion`/ClawHub contract. The PR is useful source work, but it changes public plugin compatibility and supply-chain resolution semantics and still has blocking review findings around dist-tags, malformed ranges, and untrusted registry metadata.

## What This Changes

The PR adds `engines.openclaw`-aware npm version resolution for plugin install/update, custom semver and npm registry helpers, an `--ignore-engine` CLI flag, load-time compatibility warnings, and related tests.

## Best Possible Solution

Decide the public plugin compatibility contract first. If `engines.openclaw` is accepted, rework the branch so npm tags and prereleases preserve user intent, registry metadata is parsed defensively, invalid ranges fail closed, non-release host versions do not produce noisy warnings, and install/update/load behavior is covered by focused tests; otherwise keep the shipped `openclaw.install.minHostVersion` and ClawHub compatibility model as the supported path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Honor explicit dist-tags during engine resolution:** `src/plugins/install.ts:743-747`
  - body: For specs like `pkg@beta`, `parseRegistryNpmSpec` marks the selector as a tag and `selectorIsPrerelease` stays false. This resolver then scans all package versions, excludes prereleases, and rewrites the request to an exact stable version, so a user asking for the beta channel can silently get a different channel. Resolve the requested tag first or constrain candidates to that tag's version semantics.
  - confidence: 0.9
- **[P1] Reject malformed engine ranges:** `src/infra/semver-range.ts:237-238`
  - body: When a non-empty branch has no tokenized comparators, `allComparators.every(...)` returns true. Unsupported ranges such as `>=2026.3` or `foo` therefore match every host version, bypassing the install/update gate and suppressing load warnings for bad metadata.
  - confidence: 0.92
- **[P2] Validate registry engine values before resolving:** `src/infra/npm-registry-versions.ts:82-87`
  - body: The registry `engines` object is cast to `Record<string, string>` without checking `engines.openclaw`. A package that publishes a non-string value can pass a number into `satisfiesRange`, which calls `range.trim()` and throws during install/update instead of treating the metadata defensively.
  - confidence: 0.88
- **[P2] Preserve prerelease lower bounds for caret and tilde ranges:** `src/infra/semver-range.ts:158-164`
  - body: Caret and tilde parsing replaces the lower-bound prerelease with `null`, so `^2026.3.14-beta.1` becomes `>=2026.3.14` instead of including the beta lower bound. That diverges from npm semver behavior and can reject or select the wrong versions for prerelease engine constraints.
  - confidence: 0.82
- **[P3] Suppress load warnings for unparsable host versions:** `src/plugins/loader.ts:987-995`
  - body: The loader only checks that `VERSION` is truthy before calling `satisfiesRange`. Local or development builds with a non-semver version string will fail every constrained range and warn for all constrained plugins even when compatibility is unknown rather than incompatible.
  - confidence: 0.8
