## Summary

Keep open for maintainer review. Current main does not implement the PR's intended npm-version selection from `engines.openclaw`; it only has the separate `openclaw.install.minHostVersion` and ClawHub compatibility checks, while the PR adds a new public metadata surface and security-sensitive install/update resolution behavior that needs explicit product and robustness review.

## What This Changes

The PR adds `engines.openclaw`-aware npm version resolution for plugin install/update, custom semver and npm packument helpers, an `--ignore-engine` flag, load-time compatibility warnings, and related tests.

## Best Possible Solution

Decide the public plugin compatibility contract first, then either rework this around `openclaw.install.minHostVersion` and ClawHub compatibility metadata or explicitly bless `engines.openclaw` as a supported npm-facing surface with safe registry parsing, npm-compatible semver/dist-tag behavior, and focused install/update/load tests.
