## Summary

Current main still appears to have the reported ACP bridge failure path: the debug client creates metadata-free bridge sessions, the bridge defaults them to an ACP-shaped `acp:<uuid>` Gateway key, the Gateway dispatch path passes that ACP-shaped key into the ACP reply-dispatch hook, and ACP session resolution treats ACP-shaped keys without persisted `SessionAcpMeta` as stale `ACP_SESSION_INIT_FAILED` instead of falling through to normal Gateway-backed generation.

## Best Possible Solution

Fix the ACP bridge/runtime boundary before closing: either make default Gateway-backed `openclaw acp` bridge sessions use a key shape that ACP runtime dispatch does not claim, or persist valid metadata before the first prompt if maintainers decide these bridge sessions should route through ACP runtime. Add focused regression coverage for default debug-client/new-session first prompt proving assistant `agent_message_chunk` output instead of `ACP_SESSION_INIT_FAILED`, and update `docs/cli/acp.md` plus the changelog if the default session-key wording changes.

## Clownfish Work Prompt

Fix the default ACP bridge session-init failure reported in openclaw/openclaw#38907. Current main behavior: `openclaw acp client` calls `client.newSession({ cwd, mcpServers: [] })`; `AcpGatewayAgent.newSession` falls back to `acp:${sessionId}`; Gateway canonicalizes that to an agent-scoped ACP-shaped key; ACP `reply_dispatch` resolves it as an ACP runtime session; because no `SessionAcpMeta` exists, `AcpSessionManager.resolveSession` returns stale `ACP_SESSION_INIT_FAILED`, so the client sees echoed input plus `end_turn`/error instead of assistant `agent_message_chunk` output. Implement the smallest fix at the bridge/runtime boundary. Prefer preventing default Gateway-backed ACP bridge sessions from using a key shape that `isAcpSessionKey`/ACP runtime dispatch claims, unless after reading the code you determine valid metadata persistence is the safer intended contract. Preserve `/acp spawn` and `sessions_spawn({ runtime: "acp" })` semantics for real ACP runtime sessions. Add focused regression coverage for the default `createAcpClient`/`AcpGatewayAgent.newSession` first-prompt path proving a Gateway-backed assistant chunk is emitted and ACP runtime stale handling is not invoked. Update `docs/cli/acp.md` if the default bridge session key wording changes. Add a user-facing changelog fix entry under the active version with `Thanks @ninaopenclaw`; do not credit Codex/OpenClaw. Do not change unrelated ACPX runtime behavior or provider auth/env stripping.
