## Summary

Keep open. The PR targets a real current-main gap: `stopReason: "aborted"` is still not treated as an error surface, so aborted assistant text or reasoning can become payload output and prevent the timeout fallback. The branch is useful but very stale against current main, and its payload hunk predates current final-answer, media-directive, and thinking-level guards, so it needs adaptation before merge.

## What This Changes

The PR updates embedded run payload assembly to surface formatted errors for aborted assistant turns, suppress aborted answer/reasoning text, add regression coverage, and adjust one follow-up runner test cast.

## Best Possible Solution

Land a small rebased patch on current `payloads.ts` that treats aborted assistant stops as error surfaces, suppresses aborted answer and reasoning artifacts, preserves current phase-aware visible-text/media/no-reply/thinking behavior, and adds focused regression tests for aborted answer and reasoning leakage.
