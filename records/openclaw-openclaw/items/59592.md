## Summary

Keep this PR open for maintainer handling. Current main has shipped related leak hardening for commentary-phase output and empty-timeout fallback, but it does not directly implement this PR's narrower payload-builder policy of treating `stopReason: "aborted"` as an error surface and suppressing aborted answer/reasoning text; the branch is also stale against later payload-builder and failover changes, so it should not be auto-closed as implemented.

The PR changes embedded run payload assembly to suppress aborted assistant answer and reasoning text, surface a formatted timeout/error payload instead, adds aborted-leak regression tests, and adjusts one follow-up-runner test cast.
