## Summary

Keep open. Current main still has the aborted embedded payload leak path when partial assistant payloads exist, but the PR needs adaptation before merge because its blanket handling of all aborted assistant turns conflicts with current main’s explicit external-abort/user-stop behavior.

## What This Changes

The PR changes embedded run payload assembly to treat aborted assistant turns as error surfaces, suppress partial answer and reasoning payloads, and adds regression tests for timeout/abort leakage plus a test fixture cast.

## Best Possible Solution

Land a current-main fix that suppresses partial answer and reasoning artifacts only for timeout/internal aborted embedded runs, surfaces a clean timeout/error payload, preserves external-abort partial-output behavior, and keeps the newer phase-aware final-answer/media extraction intact.
