## Repair Work Prompt

Implement a focused replacement for the embedded PI runner context-overflow recovery described by this PR, without taking the unrelated extension peer dependency or pnpm-lock.yaml changes from the submitted branch.

Observable problem: when context overflow recurs after compaction and tool-result truncation fail, prompt/memory hook injections may be the remaining cause. Current main returns terminal context_overflow immediately instead of trying one bounded retry with hook-injected prompt context suppressed.

Expected fix boundary:
- Work primarily in src/agents/pi-embedded-runner/run.ts, src/agents/pi-embedded-runner/run/attempt.ts, src/agents/pi-embedded-runner/run/types.ts, and focused tests.
- Add a single-attempt recovery path after normal overflow recovery fails: suppress prompt-hook injected context for one retry, then restore normal hook behavior for later attempts/turns.
- Preserve compaction_failure behavior: do not run the suppress-hook retry for compaction_failure.
- Gate the retry on remaining run-loop budget so exhausted overflow still reports context_overflow, not retry_limit.
- Gate the retry on meaningful hook context availability or prior hook mutation so hook-free runs do not pay for an extra deterministic failing model call.
- Be explicit about suppression scope. Prefer suppressing injected context fields only, such as prepend/append prompt context and system-context prepend/append, while preserving unrelated hook behavior such as systemPrompt overrides unless there is a strong reason and tests cover it.
- Keep raw stop-reason normalization coverage for model_context_window_exceeded/context_window_exceeded where it is still missing on current main. Note that formatRawAssistantErrorForUi now lives in src/shared/assistant-error-format.ts, while context-overflow classification lives in src/agents/pi-embedded-helpers/errors.ts.
- Respect the current first-class context_overflow failover contract; do not force these stop reasons into timeout unless current tests/contracts require it.
- Do not change extension package.json files, dependency versions, pnpm-lock.yaml, release metadata, or unrelated provider/channel behavior.
- Changelog: if production behavior changes, add a single-line Fixes entry in the active version with appropriate human attribution, likely Thanks @everest-an. Do not credit Codex/OpenClaw.
