## Summary

Keep this PR open. The in-process orphan-reconciliation direction is useful, but the proposed predicate is not ownership-aware and would cancel legitimate active runs whose supervisor owner is a non-empty sessionKey rather than the generated process-session slug.

## What This Changes

The PR adds a real process-supervisor orphan reconciliation contract, wires reconciliation into exec/process-tool paths with a registry hasSession predicate, exports the new types, and adds focused supervisor/process-tool tests.

## Best Possible Solution

Keep the PR open for changes: add an ownership-aware reconciliation predicate that preserves runs tracked by either generated process-session id or matching sessionKey, narrow orphan-reconciliation reasons to forced-cancellation values, add regression coverage for sessionKey-owned runs, and leave #65983 open unless durable restart/cold-start reaping is implemented separately.

## Reproduction Assessment

Yes. A high-confidence code-path reproduction is to start a background exec with a non-empty sessionKey, then trigger the PR's reconciliation; the supervisor record is owned by sessionKey while the registry is keyed by generated session id, so hasSession(sessionKey) returns false and the run is cancelled.

## Solution Assessment

No. The smaller in-process reconciliation approach is reasonable, but the current PR is not the best fix because it compares supervisor ownership against the wrong registry key and exposes an overbroad reason contract; an ownership-aware registry helper is the safer narrow path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use session-key-aware orphan ownership checks:** `src/agents/bash-tools.exec-runtime.ts:607`
  - body: The new reconciliation predicate only checks the bash-process registry by generated session slug. runExecProcess stores supervisor ownership as opts.sessionKey?.trim() || sessionId, so any active run launched with a non-empty sessionKey is treated as untracked and cancelled when another exec starts or the process tool runs. Use an ownership-aware check that also preserves registry sessions with a matching sessionKey.
  - confidence: 0.93
- **[P3] Restrict orphan reconciliation to forced reasons:** `src/process/supervisor/types.ts:13`
  - body: ReconcileOrphansParams.reason allows "exit", "signal", and "spawn-error", but reconciliation force-cancels live orphan runs through cancel(...). A caller could record a killed orphan as a natural exit or spawn failure, so the contract should only accept forced-cancellation reasons.
  - confidence: 0.78
