## Summary

Keep this PR open. Current main still has the deliberate no-op process-supervisor orphan reconciliation contract, the process tool does not reconcile before serving requests, the linked issue remains open, and the PR has live review feedback showing a false-cancellation bug for runs owned by `sessionKey` rather than the process session slug.

## Best Possible Solution

Keep this PR open and request changes: make reconciliation ownership-aware across both process session slug and `sessionKey`, narrow the reconciliation cancellation reason type to forced-cancel reasons, add regression coverage for sessionKey-owned runs, and then decide whether this partial in-process fix should land while the linked issue continues tracking durable restart orphan reaping.
