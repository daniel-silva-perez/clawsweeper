## Summary

Keep this PR open. Current main still has the active Discord `allowBots=false` bot-drop branch in `extensions/discord`, and it still only emits the terse verbose message. The PR’s intended behavior is useful, but its patch targets the removed `src/discord` layout rather than the current bundled Discord plugin files.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. Port the same diagnostic behavior to `extensions/discord/src/monitor/message-handler.preflight.ts` and `extensions/discord/src/monitor/message-handler.preflight.test.ts`: add a `logDebug` file-log diagnostic and clearer `logVerbose` config hint on the existing `allowBotsMode === "off"` bot-drop branch, preserve the current default filtering behavior, and add a regression test for that exact log path.
