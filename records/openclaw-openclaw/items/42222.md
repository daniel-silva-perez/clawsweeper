## Summary

Keep this PR open. Current main still has the active Discord allowBots=false bot-drop path in the bundled Discord plugin, and that path still emits only the terse verbose message. The PR is still a focused implementation candidate, but its patch targets removed src/discord files and needs to be ported to extensions/discord before it can fix runtime behavior.

## Best Possible Solution

Port the PR's narrow diagnostic change to extensions/discord/src/monitor/message-handler.preflight.ts and extensions/discord/src/monitor/message-handler.preflight.test.ts: keep the existing default filtering behavior, add a logDebug file-log diagnostic plus a clearer logVerbose hint on the allowBotsMode === "off" bot-drop branch, and add a regression test for that exact path.
