Keep this PR open. The PR is a useful focused fix, but its patch targets the removed `src/discord/...` layout rather than the active bundled Discord plugin files.

## Best Possible Solution

Port the same behavior to `extensions/discord/src/monitor/message-handler.preflight.ts` and `extensions/discord/src/monitor/message-handler.preflight.test.ts`: add a `logDebug` diagnostic and clearer `logVerbose` config hint on the existing `allowBotsMode === "off"` bot-drop branch, preserve the current default filtering behavior, and add a regression test for that exact log path.
