Keep this PR open. Current main still drops default-blocked Discord bot messages with only `logVerbose("discord: drop bot message (allowBots=false)")`; it has not gained the proposed file-log `logDebug` diagnostic or the actionable `channels.discord.allowBots=true` / `"mentions"` hint. The PR is useful, but it targets the removed `src/discord/...` layout instead of the active `extensions/discord/...` plugin files.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. Port the same behavior to `extensions/discord/src/monitor/message-handler.preflight.ts` and `extensions/discord/src/monitor/message-handler.preflight.test.ts`: add the `logDebug` file-log diagnostic and clearer `logVerbose` config hint on the existing `allowBotsMode === "off"` bot-drop branch, preserve the current default filtering behavior, and add a regression test for that exact log path.
