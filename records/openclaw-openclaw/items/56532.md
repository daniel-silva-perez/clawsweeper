## Summary

Keep this PR open. Current main still does not implement configurable `memory-lancedb` embedding timeout or retry settings, and the open PR remains a focused implementation candidate rather than obsolete cleanup. The current PR revision also addresses earlier parser/manifest validation feedback, but it still needs maintainer review for the baseUrl/retry security concern, runtime option coverage, and stale PR-body scope text.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to refresh it against current main, keep the scoped memory-lancedb config/schema/runtime implementation, add focused runtime tests for OpenAI timeout/maxRetries constructor wiring, resolve or explicitly accept the baseUrl/retry security concern, and update the PR body so launchd changes are removed from scope or handled separately. The closed predecessor #56517 can remain historical context; this PR is the active implementation candidate.
