## Risks / Open Questions

- The PR is still an implementation candidate for behavior current main lacks; closing it would drop an unshipped memory-lancedb configuration feature.
- Security review remains open: configurable `maxRetries` can increase attempts to an already configurable `embedding.baseUrl`, and current main expands environment variables in that URL before it reaches the OpenAI client.
- Runtime verification should cover constructor/request behavior for configured timeout and retry values, not only config parsing and manifest acceptance.
- The PR body still mentions a LaunchAgent ThrottleInterval bump that is absent from the supplied current diff, so scope text should be corrected before merge.
