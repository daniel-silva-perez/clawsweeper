## Summary

Keep this PR open. Current main still does not implement the PR’s intended TypeScript Gateway behavior: the logger and temp-dir resolver do not read `OPENCLAW_LOG_DIR` or `OPENCLAW_TMP_DIR`, and docs/tests still describe the existing `/tmp/openclaw` plus `logging.file` path model.

## Best Possible Solution

Keep this PR open as a plausible core infra fix. The best path is maintainer review of the latest head, focused tests for both env vars, alignment of all default log path helpers, preservation of tmp-root trust checks, and docs updates before landing or requesting changes.

## Risks / Open Questions

- The PR introduces env-controlled runtime path roots; landing safely needs regression tests for existing `/tmp/openclaw`, missing directories, symlink targets, wrong owner, broad permissions, and creation failures.
- If `OPENCLAW_LOG_DIR` is accepted, `src/logging/log-file-path.ts` and status/local-log fallback surfaces should align with the logger, otherwise different surfaces may report or tail different default paths.
- Docs currently do not describe `OPENCLAW_LOG_DIR` or `OPENCLAW_TMP_DIR`, so users would get an undocumented operator surface if the PR landed as-is.
- No local tests were run because this review was explicitly read-only and tests in this repo create temp/cache artifacts.
