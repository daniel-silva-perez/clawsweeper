## Summary

Keep open. Current main still lacks a structural per-channel or per-spawn policy for suppressing sub-agent completion announcements, and the recent maintainer context ties the remaining work to #69492 rather than treating it as implemented, stale, or a duplicate that can close.

## What This Changes

The issue requests configurable suppression or rerouting of sub-agent completion announcements for channel-bound sessions such as Telegram and iMessage.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior but can mark sub-agent completions as suppressed or internal-only for selected channels and/or spawn calls. Apply it before both announce-agent delivery and direct fallback sends, then update config schema/types, generated config docs, public Sub-agents docs, and focused delivery tests in coordination with #69492.
