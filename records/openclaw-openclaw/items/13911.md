## Summary

Current main still lacks first-class per-channel or per-spawn sub-agent announce suppression. The public config/schema/docs expose announce timeout and target allowlists, but no `subagentAnnounce`, `announcePolicy`, `denyChannels`, `completionDelivery`, or audience policy. The delivery code still resolves an external channel target for top-level requester sessions and uses that target for both primary announce-agent delivery and fallback sends. Existing tests and docs preserve this external-delivery behavior, and related open issues show the product/API shape still needs maintainer coordination.

## Best Possible Solution

Keep this open and coordinate it with related suppress/reroute/audience work. The best implementation is a documented, backwards-compatible completion delivery policy that preserves current channel delivery by default while allowing per-channel suppression or parent/internal-only delivery before both primary announce-agent delivery and fallback sends. It should update config types, zod/generated schema, public subagent docs, and focused delivery tests.
