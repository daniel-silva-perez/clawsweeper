## Summary

Keep open. Current main still has no first-class per-channel or per-subagent announce suppression policy, while the subagent announce delivery path, tests, and docs still preserve external delivery for top-level requesters when a usable route exists. The recent history improves routing, active-session steering, and fallback reliability, but it does not add the requested structural suppression/reroute control.

## Best Possible Solution

Keep this issue open and implement a core-supported subagent completion delivery policy, or first extend the plugin API so plugins can explicitly suppress subagent completion announcements by provenance and channel. The fix should wire through config types, zod schema, generated config docs, public subagent docs, and regression tests, and it should apply before both primary external delivery and fallback sends.
