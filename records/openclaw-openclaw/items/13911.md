## Summary

Keep open. Current main still lacks a public per-channel or per-spawn policy to suppress or reroute sub-agent completion announces, while the delivery path still resolves external chat targets and uses direct fallback sends; the discussion also has recent user confirmations and maintainer direction tying the API shape to #69492.

## What This Changes

The issue requests a structural config or tool-level control to suppress or reroute sub-agent completion announcements for selected channels such as Telegram or iMessage.

## Best Possible Solution

Ship a backward-compatible completion delivery policy that defaults to current behavior but can mark sub-agent completions as suppressed, internal-only, or rerouted for selected channels and/or spawn calls, then apply it before all external delivery paths and document the supported config/tool contract.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the schema, public type, docs, and `sessions_spawn` schema expose no suppression/reroute setting, while the delivery code and tests still route top-level completions to external chat targets and fallback sends.

## Solution Assessment

Unclear. The requested direction is valid, but the exact primitive should be coordinated with #69492 so OpenClaw gets a durable audience/delivery policy rather than a one-off channel flag that conflicts with existing delivery semantics.
