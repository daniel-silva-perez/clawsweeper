## Summary

Keep this issue open. Current main still has no config, tool parameter, or delivery-policy surface for suppressing or rerouting sub-agent completion announces per channel, and the active discussion includes Telegram and iMessage confirmations plus a recent maintainer/Codex comment explicitly keeping the item open for coordinated follow-up.

## Best Possible Solution

Keep this issue open and coordinate it with the broader announce/audience work, especially #69492. The best implementation is a backward-compatible completion delivery policy that keeps current external delivery by default while allowing documented per-channel and/or per-spawn internal-only or suppressed delivery before both the primary announce-agent path and direct fallback sends. It should update config types, zod/generated schema, public sub-agent docs, and focused delivery tests.
