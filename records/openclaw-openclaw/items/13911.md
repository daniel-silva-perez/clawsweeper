## Summary

Keep open. Current main still lacks a structural per-channel or per-spawn policy for suppressing sub-agent completion announces, and the existing delivery path intentionally keeps top-level completion delivery external with direct channel fallback behavior. The recent maintainer comment also points this toward #69492-level API/product alignment rather than treating it as implemented, stale, or a simple duplicate.

## What This Changes

The issue requests a structural configuration or spawn-time policy to suppress or reroute sub-agent completion announcements per channel.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior but can mark sub-agent completions as suppressed or internal-only for selected channels and/or spawn calls. Apply that policy before both primary announce-agent delivery and direct fallback sends, then update config types, generated schema/help, public docs, and focused delivery tests in line with #69492's audience-classification direction.
