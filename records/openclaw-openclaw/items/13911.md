## Summary

Keep open. Current main still lacks a structural per-channel or per-spawn suppression/reroute policy for sub-agent completion announces; the existing completion path remains direct-first for top-level requesters and preserves direct channel fallbacks. The recent maintainer discussion also points this toward broader #69492 API alignment rather than treating it as implemented, stale, or superseded.

## What This Changes

Request to add a per-channel or general sub-agent announce policy so channels such as Telegram and iMessage can suppress or reroute completion announcements while keeping normal interactive replies enabled.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior but can mark sub-agent completions as suppressed or internal-only for selected channels and/or spawn calls. Apply it before both primary direct announce delivery and direct fallback sends, then update config types, generated schema/help, public docs, and focused delivery tests in line with the audience-classification direction.
