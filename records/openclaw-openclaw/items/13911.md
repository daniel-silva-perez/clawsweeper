## Summary

Keep open. Current main still has no public per-channel or per-spawn suppression/reroute policy for sub-agent completion announces, while the delivery path and docs still preserve external requester delivery plus direct fallback sends. The discussion also has recent Telegram and iMessage confirmations and an explicit maintainer comment keeping the item active pending API direction with #69492.

## What This Changes

The issue requests a structural config or spawn-time control to suppress or reroute sub-agent completion announcements for selected channels so raw completion output does not clutter user chats.

## Best Possible Solution

Ship a backward-compatible completion delivery policy that defaults to current behavior but can classify sub-agent completion announces as external, internal-only, suppressed, or rerouted for selected channels and/or spawn calls. Apply that policy before both announce-agent delivery and direct fallback sends, then update config types, generated schema/help, public docs, and focused delivery tests in coordination with #69492.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: schema/type/docs/tool surfaces expose no suppression or reroute option, while delivery code and tests still route completion announces to external chat targets and direct fallback sends.

## Solution Assessment

Unclear. The requested direction is valid, but the narrowest maintainable solution should align with #69492's broader audience/delivery classification instead of adding a one-off Telegram or channel-only flag first.
