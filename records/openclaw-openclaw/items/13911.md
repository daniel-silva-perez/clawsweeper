## Summary

Keep open. Current main still lacks a structural per-channel or per-spawn policy to suppress or reroute sub-agent completion announcements, and recent maintainer discussion keeps the item open pending alignment with broader audience-classification work.

## What This Changes

The issue requests a config or spawn-level control that can suppress or reroute sub-agent completion announcements for selected channels while preserving normal interactive replies.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior but can mark sub-agent completions as suppressed or internal-only for selected channels and/or spawn calls. Apply it before both direct announce delivery and direct fallback sends, then update config types, generated schema/help, public docs, and focused delivery tests in line with the #69492 audience-classification direction.
