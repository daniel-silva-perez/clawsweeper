## Summary

Keep open. Current main still lacks a structural per-channel or per-spawn suppression policy, and the delivery code, tests, and docs continue to preserve external sub-agent completion delivery for top-level requester sessions. The remaining work is valid but should be coordinated with the broader audience-classification design in #69492 before implementation.

## What This Changes

The issue requests a configurable way to suppress or reroute sub-agent completion announcements per channel so chat channels do not receive raw sub-agent completion output.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior but can mark sub-agent completions as suppressed or internal-only for selected channels and/or spawn calls. Apply it before both primary announce-agent delivery and direct fallback sends, then update config schema/types, generated config docs, public Sub-agents docs, and focused delivery tests in coordination with #69492.
