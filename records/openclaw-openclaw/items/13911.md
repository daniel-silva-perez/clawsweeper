## Summary

Keep open. Current main still has no public per-channel or per-spawn suppression/reroute policy for sub-agent completion announces, and the active discussion includes Telegram and iMessage confirmations plus a maintainer comment explicitly keeping the work open pending API direction with #69492.

## What This Changes

Request a structural config or tool policy to suppress or reroute sub-agent completion announcements for selected channels while keeping normal interactive replies working.

## Best Possible Solution

Ship a backward-compatible completion delivery policy that defaults to current behavior but can classify or route sub-agent completion announces as external, internal-only, suppressed, or rerouted for selected channels and/or spawn calls. Apply it before both announce-agent delivery and direct fallback sends, then update config types, generated schema/help, public docs, and focused delivery tests in coordination with #69492.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: public config/tool/docs surfaces expose no suppression or reroute option, while delivery code and tests still preserve external requester delivery and direct fallback sends.

## Solution Assessment

Unclear. The requested direction is valid, but the best fix should likely use #69492's broader audience/delivery classification rather than committing to a narrow `channels.telegram.subagentAnnounce` flag first.
