## Summary

Keep open. Current main still has no structural per-channel or per-spawn setting to suppress or reroute sub-agent completion announcements, and recent maintainer discussion explicitly keeps the work open while aligning it with the broader audience-classification design in #69492.

## What This Changes

The issue requests a config or spawn-time policy to suppress or reroute sub-agent completion announcements per channel so Telegram, iMessage, and similar chats do not receive raw completion output.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior but can mark sub-agent completions as suppressed, internal-only, or rerouted for selected channels and/or spawn calls. Apply it before direct announce delivery and direct fallback sends, then update config types, generated schema/help, public docs, and focused delivery tests in line with #69492.
