## Summary

Keep open. Current main still lacks first-class per-channel, per-spawn, or global policy to suppress/reroute sub-agent completion announces; the source, docs, generated schema, and tests continue to preserve external delivery for top-level requester sessions and direct fallback sends.

## Best Possible Solution

Keep this open and coordinate it with related suppression/reroute/audience requests. The best implementation is a documented, backward-compatible completion delivery policy that preserves current channel delivery by default while allowing per-channel and/or per-spawn internal-only or suppressed delivery before both primary announce-agent delivery and fallback sends. It should update config types, zod/generated schema, public sub-agent docs, and focused delivery tests.
