## Summary

Keep open. Current main still has no structural per-channel or per-spawn policy to suppress or reroute sub-agent completion announcements, and the existing delivery path still intentionally resolves external chat targets plus direct fallback sends.

## What This Changes

Add a configurable way to suppress or reroute sub-agent completion announcements for selected channels or spawn contexts while keeping normal interactive replies available.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior but can mark sub-agent completions as suppressed, internal-only, or rerouted for selected channels and/or spawn calls. Apply it before both announce-agent delivery and direct fallback sends, then update config types, generated schema/help, public docs, and focused delivery tests in line with #69492.

## Reproduction Assessment

Yes. Current-main source, schema, docs, and tests provide a high-confidence static reproduction: there is no public suppression/reroute setting, while top-level completion delivery still resolves external targets and direct fallback sends.

## Solution Assessment

Unclear. The requested direction is valid, but the exact public primitive should be coordinated with #69492 rather than picked by an automated narrow fix in isolation.
