## Summary

Keep open. Current main still lacks a public per-channel or per-spawn suppression/reroute policy for sub-agent completion announces, and the current delivery path still targets external chat routes plus direct fallback sends. Recent user confirmations and maintainer context keep this active and tie the final API shape to #69492 rather than making it a stale or implemented item.

## What This Changes

The issue requests a configurable way to suppress or reroute sub-agent completion announcements per channel or per spawn so Telegram, iMessage, and similar chats are not cluttered by raw sub-agent results.

## Best Possible Solution

Ship a backward-compatible completion delivery policy that defaults to current behavior but can classify completion announces as external, internal-only, suppressed, or rerouted for selected channels and/or spawn calls, then apply it before both announce-agent delivery and direct fallback sends with config/tool/docs/test coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the strict config schema, public type, generated schema, and `sessions_spawn` schema expose no suppression/reroute setting, while delivery code and tests still route top-level completion announces to external chat targets and direct fallback sends.

## Solution Assessment

Unclear. The requested direction is valid, but the exact primitive should be coordinated with #69492 so OpenClaw gets a durable audience/delivery policy rather than a one-off Telegram or channel-only flag.
