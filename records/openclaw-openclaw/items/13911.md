## Summary

Keep open. Current main still lacks a public per-channel or per-spawn suppression/reroute policy for sub-agent completion announces, and the current completion-delivery path still uses external requester routes plus direct fallback sends. Recent Telegram and iMessage confirmations, the maintainer comment, and the related open #69492/#64658 context all point to active remaining work rather than an implemented, stale, or superseded item.

## What This Changes

The issue requests a config or tool policy to suppress or reroute sub-agent completion announcements per channel so channel-bound sessions can keep normal replies without raw sub-agent results being posted to chat.

## Best Possible Solution

Ship a backward-compatible completion delivery policy that defaults to current behavior but can classify sub-agent completion announces as external, internal-only, suppressed, or rerouted for selected channels and/or spawn calls. Apply it before both announce-agent delivery and direct fallback sends, then update config types, generated schema/help, public docs, and focused delivery tests while coordinating with #69492.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: schema/type/docs/tool surfaces expose no suppression or reroute policy, while delivery code and tests still route completion announces to external chat targets and direct fallback sends.

## Solution Assessment

Unclear. The requested direction is valid, but the narrowest maintainable solution should align with #69492's broader audience/delivery classification instead of adding a one-off Telegram or channel-only flag first.
