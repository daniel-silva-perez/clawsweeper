## Summary

Keep open. Current main has partial friendlier labeling through Feishu sender/group labels and Control UI agent identity display, but it still does not propagate `bindings[].comment` into routed session metadata or make it the preferred Web Control UI session label.

## What This Changes

The issue asks the Web Control UI to show recognizable session or binding names by preferring binding comments, then agent names, then technical identifiers.

## Best Possible Solution

Add a single Gateway-owned session display-label contract that carries the matched route binding comment into session rows, preserves manual session labels, falls back to agent identity/name, and then uses existing technical fallbacks. The Control UI chat selector and sessions table should consume that same row field instead of inventing separate UI-only naming rules.

## Reproduction Assessment

Yes. A static reproduction is high-confidence: configure a Feishu direct route binding with `comment: "税务师"`, receive a DM from `ou_...`, and current main has no path from the matched binding comment into `sessions.list`, the chat session selector, or the sessions table.

## Solution Assessment

Yes. The narrow maintainable fix is to add or derive the label at the Gateway session-row/route-metadata boundary; a UI-only formatter cannot reliably know which binding matched and would leave other `sessions.list` consumers inconsistent.
