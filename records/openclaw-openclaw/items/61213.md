## Summary

Keep this issue open. Current main has partial friendlier session labeling, but it does not implement the requested priority of `bindings[].comment` -> agent name -> technical identifier for Web Control UI session/binding display. The relevant code still derives Control UI session labels from session row `label`/`displayName`, agent identity in some views, or raw session-key fragments; the matched route binding's `comment` is accepted in config but not propagated into session rows or UI labels. The related ClawSweeper items are about different session-key or docs behavior and do not supersede this request.

## Best Possible Solution

Keep this issue open. The best fix is to add a single session display-label contract for Control UI: resolve route-binding comment first, then agent identity/name, then existing row label/displayName or technical session key, and expose it consistently in `sessions.list`, the chat session selector, and the sessions table with tests for a Feishu `ou_xxx` route binding.
