## Summary

Current main still uses Baileys useMultiFileAuthState for the bundled WhatsApp plugin, so the requested production SQL/consolidated signal-key auth store has not landed. The related implementation PR was closed unmerged, and the shipped hardening currently covers creds.json atomic writes and backup recovery while retaining Baileys multi-file key persistence.

## Best Possible Solution

Keep this issue open until the bundled WhatsApp plugin replaces Baileys useMultiFileAuthState with a production auth store, migrates existing multi-file credentials and signal keys safely, preserves logout/status/config behavior, updates public docs, and adds focused tests for fresh install, migration, corruption recovery, and key persistence. PR #66863 is useful prior art but needs a corrected implementation before this can close.
