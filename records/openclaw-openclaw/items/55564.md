## Summary

Keep open. Current main still persists outbound deliveries before send and recovers pending delivery-queue entries on gateway startup without any stale-age, TTL, or restart-generation filter. The related discussion points to this issue as the canonical tracker, and the currently visible implementation candidate (#54900) is still open rather than merged.

## Best Possible Solution

Keep this issue open and implement a conservative core freshness policy in src/infra/outbound/delivery-queue-recovery.ts, with config/schema/docs coverage for a gateway delivery max-age or equivalent restart-generation policy. Preserve brief crash recovery, but drop/quarantine/fail old first-attempt entries on startup, and add regression tests for stale Telegram/forum-style payloads plus fresh crash-replay behavior. Review or replace the open #54900 implementation candidate rather than closing this issue before a fix lands.
