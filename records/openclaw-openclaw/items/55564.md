## Summary

Keep this issue open. Current main still runs outbound delivery recovery on gateway startup and replays pending queue entries using retry, backoff, and max-retry checks only; it does not apply a stale-age TTL or restart-generation cutoff. PR #54900 remains the focused open implementation candidate, but it is not merged, so the reported behavior remains actionable.

## What This Changes

The issue requests a gateway outbound delivery freshness policy so queued messages from before a restart are dropped or aged out instead of replayed in a stale batch.

## Best Possible Solution

Review, rebase, and either land PR #54900 or replace it with a narrow core outbound recovery freshness fix that skips old never-attempted queued replies while preserving intentional retry recovery for real transient send failures; add regression coverage around startup recovery and only update config/docs/changelog if an operator knob is exposed.
