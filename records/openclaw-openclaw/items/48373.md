## Summary

Keep open. Current main still does not implement one-call Feishu document creation with initial content, and an open same-author PR with closing syntax remains the active implementation candidate. The create schema was narrowed, but the runtime handler still casts raw params and the agent-facing normalized tool schema flattens union properties, so this is not a clearly resolved rejection path either.

## Best Possible Solution

Review PR #48377 as the active implementation candidate. Maintainers should either land create-plus-write with regression coverage and clear partial-failure behavior, or explicitly reject `content` on `create` and update the Feishu skill/docs to direct agents through create then write.
