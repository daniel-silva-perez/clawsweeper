## Summary

Current main still appears to reject workspace bootstrap file symlinks whose targets resolve outside the workspace, and an open unmerged PR explicitly tracks this issue as its implementation candidate. This issue should stay open until that PR is landed, rejected, or superseded.

## Best Possible Solution

Keep this issue open. Maintainers should review PR #40230 as the concrete implementation candidate, decide whether trusted workspace bootstrap files may follow symlinks outside the workspace, and if accepted land a narrow `src/agents/workspace.ts` fallback with regression coverage. After a fix ships, consolidate the overlapping open reports #38622 and #64472.
