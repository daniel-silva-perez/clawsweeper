## Summary

Keep this issue open. Current main still routes recognized workspace bootstrap files through the workspace-bound boundary reader, so an out-of-workspace symlink target is still rejected and then rendered as a missing bootstrap entry. There is also an open PR with closing syntax (#40230) and a newer maintainer-labeled implementation candidate (#73401), so the right cleanup path is to review, land, or supersede those PRs before closing this issue.

## Best Possible Solution

Keep this issue open. Maintainers should review #73401 as the current maintainer-labeled implementation candidate, or explicitly supersede it with #40230 or another narrow fix. The code change should live in the workspace bootstrap reader layer, load trusted fixed bootstrap-file symlinks to readable regular files, preserve existing hardlink and boundary protections for other paths, and include regression coverage for external symlinks plus rejected unsafe targets. After a fix lands, close this issue as implemented and consolidate #38622 and #64472 as appropriate.
