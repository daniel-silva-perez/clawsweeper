## Summary

Keep open. Current main still does not implement configurable per-candidate model retry or a fallback context safe/strict mode; model fallback runs each candidate once, the public model config schema has no retry/context policy fields, and escaped context-overflow errors are still deliberately kept out of the fallback chain.

## What This Changes

The issue requests configurable per-candidate model fallback retry plus a safe/strict context mode that compacts oversized history before trying a smaller fallback model.

## Best Possible Solution

Keep this issue open and design the fallback policy on the existing agent model config surface: bounded retries for classified transient failures, retry observations/tests, and explicit safe/strict context semantics that reuse the existing compaction engine only when the retry is replay-safe and the target candidate budget is known.

## Reproduction Assessment

Yes. A focused unit reproduction can mock runWithModelFallback with a transient primary failure and show the primary is called once before the fallback; source checks also show context overflow is rethrown and the schema rejects the proposed policy fields.

## Solution Assessment

Unclear. The requested direction is plausible, but the exact proposed config and retry behavior need maintainer design around replay safety, candidate-specific context windows, and how compaction should affect persistent session history.
