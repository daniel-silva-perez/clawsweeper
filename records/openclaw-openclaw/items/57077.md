## Summary

Keep open. ## What This Changes

The PR updates Control UI chat session select rendering to mark active desktop/mobile options with Lit live property selection, gives the selects targetable data attributes and a distinct mobile class, adjusts mobile CSS, and adds fallback-session rerender tests.

## Best Possible Solution

Port the PR's narrow selector-rendering fix onto current main: apply live property-driven session selection to the split desktop selector and mobile picker, keep the fix out of gateway/history/navigation state, and land focused desktop/mobile regression coverage that exercises stale DOM selection during fallback-option rerenders.

## Reproduction Assessment

Yes. The component-level reproduction is to render the chat session picker, let the live DOM-selected option drift from `state.sessionKey`, then rerender while the current session is injected as a fallback option; the PR describes this path and adds partial desktop/mobile coverage for fallback rerender selection.

## Solution Assessment

Yes, with a current-main port. Lit's `live` directive is designed for values changed outside Lit, so applying live property selection at the picker/options layer is narrower than changing chat history or navigation state; the safer final patch should update `ui/src/ui/chat/session-controls.ts` and strengthen the regression test around live DOM drift.
