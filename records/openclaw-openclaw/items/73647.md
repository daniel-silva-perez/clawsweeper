## Summary

Keep open. This is a fresh, concrete regression report with multiple same-class reproductions across Telegram, Mattermost, Docker/Synology, macOS, and acpx-only setups. Current main has adjacent startup/runtime-deps diagnostics and mitigations, but I did not find merged source, test, release, or history evidence that the reported v2026.4.26 channel-runtime/versioned-stage-root CPU spin is fixed. PR #73678 is an open draft partial fix slice, not a reason to close this issue before review and merge.

## Best Possible Solution

Keep this issue open as the canonical follow-up for the v2026.4.26 gateway CPU spin after the openclaw-unknown runtime-deps fix. Review and land or close PR #73678 on its merits, then reproduce with a packaged npm/global or Docker gateway using Telegram-only plus the reported Mattermost/acpx variants. Capture OPENCLAW_GATEWAY_STARTUP_TRACE output and an event-loop/CPU profile around repeated versioned stage-root reads, then fix the responsible path in manifest loading, plugin registry snapshot reuse, bundled runtime mirror/deps staging, channel startup, or the affected channel plugins. Add a regression that distinguishes the already-covered openclaw-unknown case from repeated openclaw-<version>-* runtime-deps inspection after gateway readiness.
