## Summary

Current main is ahead of the reported v2026.4.26 release and already contains the targeted gateway/runtime-deps fixes for this Telegram-only CPU-spin path: bundled channel startup uses a lightweight runtime surface, runtime-deps staging is scoped to the selected startup plugin IDs, and repeated runtime config schema requests no longer rebuild plugin/channel runtime metadata.

## Best Possible Solution

Close this release-specific regression as implemented on current main. Keep the main-line scoped runtime-deps, lightweight bundled channel startup runtime, and config-schema metadata reuse behavior; reopen or file a fresh report only if the CPU spin reproduces on a build from 891c7d9f1c1f5e0f52935fadfeacf695b0ef1eb8 or later.
