## Summary

Keep this PR open. Current main still has the exact WhatsApp mention-gating bug targeted by the PR: implicit self-chat mode is derived from `allowFrom` without checking whether the conversation is a group, so group native/LID mentions can bypass the identity-overlap check. The linked bug #49317 is open, and this PR is a focused implementation candidate rather than something conservative cleanup should close.

## Best Possible Solution

Keep this PR open for maintainer review and CI. The best path is to land a focused WhatsApp plugin fix that confines implicitly derived self-chat suppression to non-group conversations, preserves explicit self-chat overrides and direct self-DM behavior, adds the group LID mention regression test, and keeps the changelog credit aligned before running the targeted WhatsApp test and normal changed gate.
