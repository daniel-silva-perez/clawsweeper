## Summary

Keep this PR open. Current main still has the WhatsApp group mention bug the PR targets: implicit self-chat mode is derived from `allowFrom` without checking whether the conversation is a group, so LID/native group mentions can still bypass the identity-overlap check. The linked bug report #49317 is still the canonical user problem, and this PR is a focused implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a focused WhatsApp plugin fix that confines implicitly derived self-chat suppression to non-group conversations, preserves explicit `mentionCfg.isSelfChat` overrides and true 1:1 self-DM suppression, adds the LID group mention regression test for #49317, and then runs the targeted WhatsApp test plus the normal changed gate.
