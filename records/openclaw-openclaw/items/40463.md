## Summary

Keep this PR open. Current main has absorbed some Microsoft Teams media work, including hostedContents `/$value` fallback, Bot Framework DM attachment routing, and a narrow `smba.trafficmanager.net` auth allowlist, but it still does not fully implement the PR's channel fallback, tenant-scoped token-provider contract, or bounded hosted-content body read.

## Best Possible Solution

Keep the PR open for Microsoft Teams maintainer cleanup. Rebase it onto current main, preserve the shipped Bot Framework DM routing, exact `smba.trafficmanager.net` auth allowlist, and `<attachment id>` fallback gate, then either implement and test the channel `/chats/{threadId}/messages` fallback or reject it with Microsoft Graph evidence. If cross-tenant media is supported, add an explicit tenant-token provider contract, and use `readResponseWithLimit` or equivalent bounded streaming for hosted-content `/$value` reads.
