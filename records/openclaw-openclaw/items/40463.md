## Summary

Keep this PR open. Current main has absorbed several Microsoft Teams media fixes, including the hostedContents `/$value` fallback and Bot Framework DM attachment routing, but it still does not implement the PR's channel `/chats/{threadId}/messages` fallback, tenant-scoped Graph token path, or bounded hosted-content `$value` read. The PR remains useful but needs a careful rebase and security review before merge or closure.

## Best Possible Solution

Keep the PR open for Microsoft Teams maintainer cleanup. Rebase it onto current main, preserve shipped Bot Framework DM routing and `<attachment id>` gating, then either implement and test the channel `/chats/{threadId}/messages` fallback or reject it with Microsoft Graph evidence. Add the explicit tenant-token provider contract if the cross-tenant scenario is supported, switch hosted-content `$value` reads to `readResponseWithLimit`, and keep media auth host allowlists narrowly scoped.
