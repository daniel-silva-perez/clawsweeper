## Summary

Keep this PR open. Current main has absorbed parts of the Microsoft Teams media work, including hostedContents /$value fallback, Bot Framework DM routing, and a narrow smba.trafficmanager.net auth allowlist, but it still does not fully implement the PR's channel /chats fallback, DM chatId preference, or bounded hosted-content body read.

## Best Possible Solution

Keep the PR open for Microsoft Teams maintainer review. Rebase it onto current main, preserve the shipped Bot Framework DM routing, exact smba.trafficmanager.net auth allowlist, and <attachment id> fallback gate, then land or explicitly reject the channel /chats/{threadId}/messages fallback with Microsoft Graph evidence. If the cross-tenant token behavior is still intended, add it through a typed token-provider contract rather than an undeclared overload, and keep the hosted-content /$value read bounded with readResponseWithLimit plus focused tests.
