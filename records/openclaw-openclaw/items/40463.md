## Summary

Keep this PR open. Current main has absorbed several Microsoft Teams media fixes, but it still does not implement the PR's channel `/chats/{threadId}/messages` fallback, tenant-scoped Graph token path, or bounded hosted-content `$value` read. The PR is useful, but it needs a careful rebase over current Teams media behavior rather than closure or direct merge.

## Best Possible Solution

Keep the PR open for Microsoft Teams maintainer cleanup. Rebase it onto current main, preserve the shipped Bot Framework DM routing, diagnostics, and `<attachment id>` gating, then either implement and test the channel `/chats/{threadId}/messages` fallback or reject it with Microsoft Graph evidence. Adopt the bounded `readResponseWithLimit` hosted-content read and decide the explicit tenant-token provider contract during review.
