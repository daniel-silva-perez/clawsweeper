## Summary

Keep this PR open. Current main has absorbed several Microsoft Teams media fixes, including hosted-content `/$value` fetching, Bot Framework DM routing, attachment-tag gating, and exact `smba.trafficmanager.net` auth forwarding, but it still does not implement the PR’s channel `/chats/{threadId}/messages` fallback and still leaves useful hardening work around bounded hosted-content reads. The PR should not be merged as-is because its auth-sensitive tenant-token path uses an undeclared token-provider overload and its older HTML fallback shape would need to preserve current attachment-tag gating.

## Best Possible Solution

Keep the PR open for Microsoft Teams maintainer cleanup. Rebase it onto current main, preserve the shipped hosted-content, Bot Framework DM, diagnostics, attachment-tag gating, and exact auth-allowlist behavior, then either implement and test the channel `/chats/{threadId}/messages` fallback or reject it with Microsoft Graph evidence. Move hosted-content `$value` reads to `readResponseWithLimit`, and replace the tenant-token overload with a declared provider capability or remove that cross-tenant path from this PR.
