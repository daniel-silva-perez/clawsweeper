## Summary

Keep this PR open: current main still has the session-store/transcript gap the PR targets, but the proposed helper creates the new transcript with non-atomic overwrite semantics, so it can clobber transcript content under a concurrent rollover race.

## What This Changes

The PR adds an optional immediate transcript-header creation path to session-file resolution, enables it during `initSessionState()`, and adds rollover/reset regression tests.

## Best Possible Solution

Keep the PR open and amend the transcript bootstrap so it creates the header with exclusive create semantics, treats `EEXIST` as success, and proves existing transcript content is never overwritten during rollover.

## Reproduction Assessment

Yes. A focused regression can run two header-bootstrap callers for the same missing rollover transcript, append content after the first create, then verify the slower caller cannot truncate it back to header-only; the PR's current stat-then-default-write sequence would fail that check.

## Solution Assessment

No. Creating the header before the store points at the new session is the right direction, but the implementation should use an atomic create such as `flag: "wx"` and handle `EEXIST` as success before it is safe to merge.
