## Summary

Keep open: current main still lacks immediate transcript-header creation during `initSessionState()` rollover, so the PR addresses a real consistency gap, but the proposed helper should not merge while it can overwrite an already-created transcript under concurrent callers.

## What This Changes

The PR adds an `ensureTranscriptHeader` option to session-file resolution, enables it from `initSessionState()`, and adds rollover/reset regression tests that expect a header-only transcript to exist immediately.

## Best Possible Solution

Keep the PR open, change the bootstrap write to an atomic create that treats `EEXIST` as success, add a focused regression test for the non-clobbering behavior, and then validate the session tests before maintainer review/merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Create the transcript header without overwriting:** `src/config/sessions/session-file.ts:30-33`
  - body: This helper stats the path and later calls `writeFile()` with default overwrite behavior. If two callers see the same missing primary transcript, one can create and append to it while the other later reaches this write and truncates it back to a header-only file. Use exclusive create (`flag: "wx"` or equivalent) and treat `EEXIST` as success so existing transcript content is never overwritten.
  - confidence: 0.87
