## Summary

Keep this PR open. The linked blank tool-name loop is still not fixed on current main and this PR is the active implementation candidate, but the current patch is not merge-ready because the sentinel behaves like a mutable, dispatchable tool name.

## What This Changes

The PR changes agent tool-call normalization to replace blank or whitespace-only tool names with `_blank`, treats that sentinel as incomplete for unknown-tool loop classification, and updates focused tests to expect the sentinel.

## Best Possible Solution

Fix the blank-name loop without storing a dispatchable sentinel in the message object. The safer path is a reserved, non-dispatchable failure representation or a dispatch-only error path that preserves later ID-based recovery, rejects sentinel collisions, and adds focused tests for same-object streaming deltas, replay sanitization, and a real `_blank` tool name.

## Reproduction Assessment

Yes. The linked report provides concrete blank-name loop evidence, and current main has focused unit coverage showing blank no-id calls remain blank. The PR regressions are reproducible with unit cases for a reused streaming block whose id arrives after a blank name and for an allowlisted `_blank` tool.

## Solution Assessment

No. Replacing blanks with the literal `_blank` inside the mutable tool-call object is not the best fix because it changes later normalization behavior and leaves `_blank` dispatchable as a real tool name. A non-dispatchable or reserved error path would be narrower and safer.
