## Summary

Keep this PR open, but the current patch should not merge as-is. The blank-tool-name failure still appears plausible on current main, but the literal `_blank` fallback introduces concrete streaming recovery, replay sanitization, and unintended tool-dispatch regressions.

## What This Changes

The PR changes embedded runner tool-call normalization so blank names fall back to `_blank`, treats that sentinel as incomplete for the unknown-tool loop guard, and updates related tests.

## Best Possible Solution

Revise the fix so blank in-flight tool calls remain recoverable until the final name/id is known, and any synthetic failure path cannot collide with registered tools or pass replay sanitization as a real tool call. Add focused coverage for blank-first/id-later streaming, a real `_blank` allowlisted tool, and no-allowlist replay pruning.

## Reproduction Assessment

Yes. The source-level path is reproducible with `wrapStreamFnTrimToolCallNames` on blank or whitespace-only tool-call names, and #34129 provides provider logs for the user-visible loop; the PR regressions are reproducible from the OpenAI-compatible stream path where `currentBlock` is reused and `id` can arrive after an initial blank name.

## Solution Assessment

No. A literal `_blank` dispatch name is not the safest fix because it mutates streaming state and remains a valid tool name under the current allowlist; a safer path should generate a non-dispatching failure or reserve an internal sentinel excluded from allowlists, replay, and later id-based recovery.

  - body: The fallback is a normal literal tool name, while `collectAllowedToolNames` accepts any trimmed tool or client-tool name. If a plugin/client tool named `_blank` is registered, exact-name dispatch can execute that real tool for malformed blank model output instead of failing closed.
  - confidence: 0.88

  - body: `resolveReplayToolCallName` accepts any non-empty normalized name when no allowlist is available. With this fallback, persisted assistant tool calls with blank names can be rewritten to `_blank` and replayed instead of being dropped, reintroducing malformed history that the sanitizer currently prunes.
