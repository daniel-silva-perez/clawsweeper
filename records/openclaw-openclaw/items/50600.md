## Summary

Keep this issue open. Current main partially covers adjacent queue behavior and already skips empty outbound replies, but it does not implement configurable busy acknowledgments, does not send an interim channel message when a queued inbound message waits past a threshold, and does not expose a `queue:wait-exceeded` hook. Related ClawSweeper context for #53023, #54622, #63864, and #14747 is adjacent but not a canonical duplicate that resolves this requested busy-ack/hook surface.

## Best Possible Solution

Keep this issue open and implement the remaining surface as an additive queue feature: add `messages.queue.busyAckMs` and `messages.queue.busyAckText` to types, schema, generated config metadata, docs, and UI/config help; schedule a per-origin interim acknowledgment when an active-run followup remains queued past the threshold; expose a typed `queue:wait-exceeded` hook with enough context for custom handling; and retain the existing empty-reply no-op guard.
