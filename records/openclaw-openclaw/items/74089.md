## Summary

Keep this PR open. Current main still drops OpenAI `[[tts:speed=N]]` directives at the provider parser, while the normal OpenAI synthesis path and request builder already know how to carry a speed value; the PR is a focused provider-owned fix with matching tests and no obvious security-sensitive surface changes. Maintainer review is still needed for validation and for the telephony providerOverride gap flagged in review.
