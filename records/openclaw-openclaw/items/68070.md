## Summary

Keep this PR open. Current main at acae48b790fac67c3f46afae310bbba837f5e678 still throws plain parameter-validation errors and serializes tool failures without a `retryable: false` marker, so the PR is not obsolete or implemented. Existing loop-detection and retry-limit paths bound some runaway behavior, but they do not implement the requested immediate non-retryable handling for malformed tool parameters.

## What This Changes

The PR marks Pi tool parameter validation errors as non-retryable and propagates retryability metadata through the tool-definition adapter error-result payload.

## Best Possible Solution

Keep the PR active as a focused agents fix. The desired end state is a documented, tested validation-error contract that marks missing or malformed tool parameters non-retryable in the serialized tool result, while preserving ordinary tool-error behavior and existing loop guards.
