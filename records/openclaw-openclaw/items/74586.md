## Summary

Keep open. The issue is fresh, has a detailed reporter reproduction plus an independent confirmation on v2026.4.27, and current main still contains the Active Memory hard-deadline abort path and timeout classification behavior that match the reported symptoms. ## What This Changes

The issue asks OpenClaw to fix Active Memory embedded recall so `memory_search` tool calls are not prematurely aborted, completed `NONE` recall is not classified as a timeout, and stale prompt-error metadata does not leak into later embedded runs.

## Best Possible Solution

Keep the Active Memory hard deadline, but decouple that deadline from ordinary memory tool execution until the budget actually expires; a completed `NONE` recall should be stored as a clean empty result, and fresh Active Memory recall prompts should exclude prior `openclaw:prompt-error` metadata.

Yes. The issue gives a high-confidence reproduction path with Active Memory enabled, persisted transcripts, direct `memory_search` as a control, and a follow-up confirmation on v2026.4.27; source inspection on current main matches the reported abort and timeout-classification path.

## Repair Work Prompt

Fix OpenClaw issue https://github.com/openclaw/openclaw/issues/74586. Active Memory embedded recall currently creates one hard-deadline AbortController in `maybeResolveActiveRecall`, passes that signal through `runRecallSubagent` into `runEmbeddedPiAgent`, and the embedded runner/tool wrapper can surface `memory_search` tool results as `Aborted`; completed `NONE`/empty output on the timeout path is also classified as `status=timeout`, and prior `openclaw:prompt-error` metadata can leak into later Active Memory recall context. Implement a narrow fix that preserves the Active Memory hard deadline but prevents ordinary memory tool calls from receiving an already-aborted run signal before the deadline actually expires; treat a completed `NONE` recall as a clean empty result rather than a timeout; and filter prior prompt-error custom entries from fresh Active Memory recall prompts or transcripts as appropriate. Do not change `modelFallback` semantics or rework the broader embedded-runner architecture. Add focused regression coverage in Active Memory and embedded-runner tests, update docs only if behavior or troubleshooting text changes, and add a single-line user-facing changelog fix entry crediting @islandpreneur007 and @jorgeszabo if runtime behavior changes.
