## Summary

Keep open. Current main still has the same cancellation boundary that can turn Active Memory’s watchdog abort into embedded tool aborts and timeout classification, and the newer cleanup-timeout commit only bounds stuck cleanup; it does not prove the reported `memory_search` aborts, clean `NONE` completion, or stale prompt-error replay are fixed.

## What This Changes

Fix Active Memory embedded recall so `memory_search` tool calls are not aborted incorrectly, a clean `NONE` completion before the budget is not reported as a timeout, and prior prompt-error events do not leak into the next recall run.

## Best Possible Solution

Preserve Active Memory’s hard deadline while separating it from normal memory tool execution until the budget actually expires, treat a completed `NONE` or empty embedded reply as a non-timeout result, and filter or retire stale prompt-error custom entries so a previous failed recall cannot contaminate the next recall prompt.
