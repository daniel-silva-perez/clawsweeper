## Summary

Keep open. Current main still preserves the reported failure shape: Active Memory passes its watchdog signal into the embedded run and tool layer, a late abort can override a completed embedded result, and the timeout recovery path still classifies normalized `NONE` or empty output as `status=timeout`. The stale `prompt-error` context leak also remains plausible because those custom entries are still persisted under `openclaw:prompt-error` while the generic runtime-context stripper only removes a different custom type.

## What This Changes

Active Memory embedded recall should allow embedded `memory_search` calls to complete normally, classify completed `NONE` output as empty instead of timeout, and avoid carrying prior `prompt-error` metadata into later recall prompts.

## Best Possible Solution

Keep Active Memory's hard deadline, but make the embedded recall result win when it completes within budget, treat `NONE` as a clean empty recall result, and ensure prior prompt-error custom entries cannot enter later recall prompts.

## Repair Work Prompt

Fix the Active Memory embedded recall abort/timeout bug reported in https://github.com/openclaw/openclaw/issues/74586. Current main passes the Active Memory watchdog abort signal through `runEmbeddedPiAgent` and the OpenClaw tool wrapper, so embedded `memory_search` can return `Aborted`, and a completed embedded response such as `NONE` can still be classified as `timeout`. Keep the hard Active Memory deadline, but adjust the AM and/or embedded-runner abort boundary so ordinary memory tool calls are not pre-aborted before the budget actually expires, completed embedded results win over watchdog/cleanup races, and completed `NONE` is classified as `empty`. Also prevent stale `openclaw:prompt-error` custom entries from a prior AM embedded run from entering the next recall prompt. Do not change `modelFallback` semantics; #74587 is a related docs/copy issue and should stay separate unless the touched code makes a wording update unavoidable. Add focused regression coverage.
