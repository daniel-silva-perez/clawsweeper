## Summary

Current main still routes successful ACP child-session task completion through the visible background-task delivery path, while this PR adds the missing silent parent heartbeat wake behavior; the PR is useful but currently dirty against main and has minor review feedback to resolve before maintainer merge.

## What This Changes

The PR adds a task-policy helper and task-registry branch to silently heartbeat-wake parent sessions for successful non-blocked ACP child-session completions, with focused policy and registry regression tests.

## Best Possible Solution

Keep the PR open, rebase it onto current main, remove or simplify the redundant terminal-status guard, preserve duplicate-suppression coverage with non-child ACP fixtures, and then land the silent-wake behavior if maintainers agree that successful non-blocked ACP child-session completions should wake the parent without user-visible banners.

## Solution Assessment

Yes, with cleanup. A small task-policy helper plus a pre-notify branch in `maybeDeliverTaskTerminalUpdate` is a narrow maintainable fix for this behavior, but the branch needs rebase and the existing duplicate-suppression test should keep covering its intended path.
