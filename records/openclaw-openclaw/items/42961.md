## Summary

Keep this PR open. Current main still does not respect `cacheRetention` for OpenRouter Anthropic models: the post-plugin OpenRouter cache wrapper has no retention input, always applies the shared short-lived Anthropic marker helper, and the cache-retention resolver does not classify OpenRouter's `openai-completions` Anthropic route as cache-retention eligible. The PR is still a plausible implementation candidate, though it needs a refresh against the current helper split and SDK/public helper call sites.

## Best Possible Solution

Refresh and review this PR rather than close it. The best fix is to route the effective OpenRouter Anthropic `cacheRetention` into cache-marker application, preserve default short behavior, suppress markers for `"none"`, emit `{ type: "ephemeral", ttl: "1h" }` for `"long"`, keep verified-OpenRouter route checks and custom proxy exclusions intact, preserve or deliberately evolve the public SDK helper contract, and add regression coverage for default/short/none/long plus non-Anthropic and bundled-provider consumer paths.
