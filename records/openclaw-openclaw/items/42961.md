## Summary

Keep this PR open. Current main still does not implement the PR’s intended behavior: OpenRouter Anthropic cache markers are still applied by a post-plugin wrapper that has no `cacheRetention` input, the shared marker helper always emits the short `{ type: "ephemeral" }` form, and the cache-retention resolver still does not make OpenRouter `openai-completions` Anthropic routes eligible for `none`/`long` handling. The supplied discussion remains useful: Greptile identified this as a real focused bug fix, and the prior ClawSweeper review reached the same keep-open conclusion; current main still matches that gap.

## Best Possible Solution

Refresh and review this PR. The best implementation is to route the effective OpenRouter Anthropic `cacheRetention` value into cache-marker application, keep default short behavior, suppress markers for `"none"`, emit `{ type: "ephemeral", ttl: "1h" }` for `"long"`, preserve verified OpenRouter route gating and custom proxy exclusions, and add regression tests for those paths.
