## Summary

Keep this PR open. Current main still injects short OpenRouter Anthropic cache markers unconditionally on verified OpenRouter routes and does not route `cacheRetention` into that wrapper, so the PR's central bug fix remains useful. The contributor's live OpenRouter observations, Greptile's review, and the previous ClawSweeper review all point at the same still-present gap.

## Best Possible Solution

Refresh and review this PR instead of closing it. The best implementation is to resolve `cacheRetention` for OpenRouter Anthropic `openai-completions` routes, pass that value into the OpenRouter system/developer cache-marker wrapper or helper, keep default short behavior, suppress markers for `"none"`, emit `{ type: "ephemeral", ttl: "1h" }` for `"long"`, preserve verified OpenRouter endpoint gating and thinking-block cleanup, and add regression tests for default, `none`, `short`, `long`, non-Anthropic OpenRouter, and custom proxy exclusions.
