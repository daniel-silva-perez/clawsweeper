## Summary

Keep this PR open. Current main still does not respect `cacheRetention` for OpenRouter Anthropic models: the OpenRouter cache wrapper has no retention input, the shared marker helper always writes the short `{ type: "ephemeral" }` form, and the retention resolver does not make OpenRouter `openai-completions` Anthropic routes eligible for `none` or `long`. The PR discussion is useful evidence: Greptile identified the same focused bug and the prior ClawSweeper review reached the same keep-open conclusion; current main at e583db63 still matches that gap.

## Best Possible Solution

Refresh and review this PR rather than closing it. The best implementation is to route the effective OpenRouter Anthropic `cacheRetention` value into cache-marker application, keep default short behavior, suppress markers for `"none"`, emit `{ type: "ephemeral", ttl: "1h" }` for `"long"`, preserve verified OpenRouter route gating and custom proxy exclusions, and add focused regression tests for those paths.
