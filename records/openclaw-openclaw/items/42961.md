## Summary

Keep this PR open. Current main still injects OpenRouter Anthropic `cache_control` markers through a post-plugin wrapper that has no `cacheRetention` input, so `cacheRetention: "none"` cannot suppress markers and `cacheRetention: "long"` cannot emit the OpenRouter-supported 1-hour TTL.

## Best Possible Solution

Refresh and review this PR rather than close it. The best fix is to route the effective OpenRouter Anthropic `cacheRetention` into cache-marker application, preserve default short behavior, suppress markers for `"none"`, emit `{ type: "ephemeral", ttl: "1h" }` for `"long"`, keep custom OpenAI-compatible proxy exclusions intact, preserve public SDK helper compatibility, and add regression coverage for default/short/none/long plus non-Anthropic and bundled-proxy consumer cases.
