## Summary

Keep open. Current main still rejects mixed DNS answers when any resolved address is blocked, before the IPv4-first pinned lookup can use an allowed public IPv4 address. The related implementation PRs in the issue context are closed and unmerged, and the shipped RFC2544 opt-in is a separate fake-IP/proxy compatibility path rather than a fix for special-use IPv6 mixed with public IPv4.

## Best Possible Solution

Keep this issue open and implement a narrow fix in `src/infra/net/ssrf.ts`: classify resolved DNS answers, filter out blocked addresses, throw only when no allowed answer remains, and build pinned lookup/dispatcher candidates from the filtered allowed set. Add focused regression coverage in `src/infra/net/ssrf.pinning.test.ts` and, if useful, `src/agents/tools/web-fetch.ssrf.test.ts` for public IPv4 plus `::`, public IPv4 plus Teredo/reserved IPv6, all-blocked DNS still failing, IPv4-first ordering after filtering, and pinned lookup results excluding blocked addresses.
