## Summary

Current main still has the reported strict mixed-DNS behavior: the SSRF resolver rejects a hostname when any resolved address is blocked, before IPv4-first pinning can use an allowed public IPv4 answer. The related fix PRs in the provided GitHub context are closed and unmerged, so this issue should remain open as the active tracker.

## Best Possible Solution

Keep this issue open. Implement a focused change in `src/infra/net/ssrf.ts` that classifies DNS answers, filters out blocked addresses, throws only when no allowed address remains, and builds pinned lookup/dispatcher candidates from the filtered allowed set. Add regression coverage in `src/infra/net/ssrf.pinning.test.ts` and, if useful, `src/agents/tools/web-fetch.ssrf.test.ts` for public IPv4 plus `::`, public IPv4 plus blocked Teredo/reserved IPv6, all-blocked DNS still failing, IPv4-first ordering after filtering, and blocked addresses excluded from pinned lookup results.
