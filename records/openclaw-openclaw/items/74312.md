## Summary

Keep open. Current main still hashes Claude token-shaped credentials and token auth-profile material into the CLI auth epoch, so token rotation or a transient token-shaped Claude credential can still invalidate reusable CLI sessions. The open, unmerged PR #74493 is already linked as the implementation path and should be reviewed rather than closing this issue early.

## What This Changes

Claude CLI auth epochs should remain stable during OAuth token rotation or transient token-shaped reads so active sessions are not reset mid-conversation.

## Best Possible Solution

Review and land #74493 or an equivalent narrow fix that makes Claude token-shaped epochs identity-based, removes raw token material from token auth-profile epoch hashing where appropriate, bumps the epoch version, and adds regression coverage for both token rotation and real identity changes.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to return a Claude CLI credential with type "token", change only credential.token, and compare resolveCliAuthEpoch; current main changes the epoch, and same-version epoch mismatches reset sessions with reason "auth-epoch".

## Solution Assessment

Yes. Identity-only hashing at the auth-epoch layer is the narrowest maintainable solution for the observed reset loop, while parser hardening can remain a defense-in-depth follow-up.
