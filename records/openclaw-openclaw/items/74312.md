## Summary

Keep open. Current main still contains the reported failure mode for token-shaped Claude CLI credentials: a parsed access-only Claude credential hashes the mutable access token, auth-profile token credentials also include token material, and reusable CLI sessions reset on auth-epoch mismatch. The earlier ClawSweeper comment is still aligned with current source; this has not been implemented on main.

## What This Changes

The issue asks OpenClaw to keep Claude CLI session auth epochs stable across OAuth token rotation or transient access-only keychain reads so active conversations are not reset mid-conversation.

## Best Possible Solution

Update the Claude CLI auth-epoch path so OAuth token rotation and transient access-only Claude keychain reads do not change the reusable session epoch, while genuine account/profile identity changes still invalidate sessions. Cover the parser/epoch behavior with targeted tests and add a user-facing changelog fix credited to the reporter when it ships.
