## Summary

Keep open. Current main does not implement Chinese/CJK trigger support for memory-lancedb: the source and tests still cover English/Czech/contact patterns only. The PR is relevant, but the proposed one-line regex is incomplete because `shouldCapture` rejects messages shorter than 10 characters before testing triggers, so common short Chinese memory statements would still not auto-capture. The related PRs in the provided context are useful prior attempts, but they were closed unmerged and do not supersede this PR on main.

## Best Possible Solution

Keep this PR open and ask for an amended implementation: add focused CJK trigger coverage, account for short high-signal Chinese inputs before or within the length gate, add Chinese category-detection tests if category quality is in scope, and reuse the prior closed PRs #16392, #16672, and #20241 as context without treating them as merged fixes.
