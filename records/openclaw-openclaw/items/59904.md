## Summary

Keep open. Current main still lacks the Discord bot-to-bot allowlist clarification this PR adds, and the runtime code supports the final direction: guild user and role allowlists are evaluated together, while allowBots separately controls bot-authored message admission. The PR remains useful but may need one more docs revision because its final diff no longer appears to cover the linked allowFrom bot: prefix clarification.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to merge or lightly revise the Discord docs change so the Bot to bot loops section explains guild users/roles member access, allowBots="mentions", and, if the linked issue is meant to be fully resolved, the allowFrom bot: prefix behavior in the inbound allowlist context.
