## Summary

Keep this PR open for maintainer review. Current main still ships fixed-size memory chunking, while the live PR branch is dirty against main and the proposed patch currently has a blocking SDK facade/export mismatch that would fail before the feature can be evaluated for merge.

## What This Changes

This PR adds selectable memory-search chunking strategies, strategy-specific config/schema/help, per-strategy reindex metadata, and chunking/reindex tests for memory-core.

## Best Possible Solution

Keep this as an implementation candidate only after it is rebased onto current main, moved through the package-owned memory-host-sdk and plugin SDK facades, and narrowed to a maintainer-approved chunking API with focused compatibility, reindex, and model-call tests.

## Reproduction Assessment

Not applicable as a feature PR, but there is a high-confidence current-main verification path. The current schema accepts only `chunking.tokens` and `chunking.overlap`, and indexing still routes through `chunkMarkdown`, so selectable strategies are not currently supported.

## Solution Assessment

No, not in its current branch shape. The product direction may be useful, but the patch needs rebase, package/facade alignment, and security/product review for the model-assisted strategies before it is a maintainable solution.
