## Summary

Keep this PR open. Current main still uses the shipped fixed-size memory chunker with only `tokens` and `overlap`, and there is no merged multi-strategy resolver, strategy config surface, or per-strategy reindex metadata. The overlapping memory chunking work is either still open or closed unmerged, so this PR is not implemented or clearly superseded; it needs maintainer product/API/security review rather than cleanup closure.

## What This Changes

This PR adds selectable memory-search chunking strategies, extends chunking config/schema/help, routes memory-core indexing through a chunking strategy resolver, persists per-strategy reindex metadata, and adds chunking plus reindex-state tests.

## Best Possible Solution

Keep this PR as an implementation candidate while maintainers choose the memory chunking API shape, reconcile the related open memory chunking work, re-review the latest branch after bot feedback, and require proof for default fixed-size compatibility, strategy-specific behavior, schema/help alignment, model-call failure handling, and reindex metadata migration.
