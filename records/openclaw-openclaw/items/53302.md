## Summary

Keep this PR open. Current `main` still lacks the requested Discord `useComponentsV2` config surface, still advertises v2 containers in Discord prompt hints, and still routes cross-context markers through Discord semantic presentation with no per-channel or per-account opt-out. The PR remains useful, but its implementation is stale against the current presentation-based outbound path and needs maintainer review or a rebase before merge.

## What This Changes

The PR adds a Discord `useComponentsV2` config toggle intended to disable v2 cross-context containers for cross-context markers while preserving regular Discord interactive components.

## Best Possible Solution

Keep this PR open and ask for a current-main adaptation: add the Discord `useComponentsV2` base/per-account config surface to types, schema, generated config docs, and public Discord docs; make Discord cross-context presentation and prompt hints honor it; preserve buttons, selects, forms, and other interactive components; and cover the behavior with focused outbound and Discord config tests.
