## Summary

Keep this PR open. Current main does not implement the advertised Discord `useComponentsV2` opt-out: the config key is absent, Discord still always advertises v2 containers in message-tool hints, and cross-context sends still attach Discord presentation whenever the plugin exposes `buildCrossContextPresentation`. The PR is useful but stale against the current message-presentation refactor, so it needs rework rather than cleanup closure. Security pass: the provided PR file list is functional/config/docs-test scoped and does not show workflows, lockfiles, package resolution, install/build/release scripts, or secret-handling changes.

## Best Possible Solution

Keep the PR open for maintainer review or contributor rework. The best path is to rebase the idea onto current `buildCrossContextPresentation`: add a Discord-owned or generic config gate that disables only cross-context presentation for Discord, leaves interactive components/forms intact, updates config schema/help/docs and the current generated baseline hashes, and adds focused outbound-policy plus Discord prompt-hint tests.
