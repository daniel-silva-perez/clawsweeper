## Summary

Keep open. Current main still builds Feishu card notes from fixed Agent/Model/Provider fields, has no Feishu footer configuration or card-footer directive, and existing usage/status surfaces do not populate the Feishu card footer requested here.

## What This Changes

Request Feishu message cards support configurable custom footer metadata such as context usage, compaction count, or session age.

## Best Possible Solution

Implement this in the bundled Feishu plugin as a typed `channels.feishu.cardFooter` option for a small set of built-in fields, then compose the same footer for streaming and non-streaming card paths with focused tests and Feishu docs. Treat assistant-supplied footer directives as a later opt-in extension only if maintainers accept the added sanitization surface.

## Reproduction Assessment

Not applicable as a runtime bug, but there is a high-confidence verification path: inspect current main with `renderMode: "card"` paths and search the Feishu config/directive surfaces. Those checks show no custom footer field support today.

## Solution Assessment

No, current main is not yet a solution for the requested behavior. The narrowest maintainable direction appears to be a typed Feishu config for built-in footer fields first, rather than a free-form assistant directive as the initial implementation.
