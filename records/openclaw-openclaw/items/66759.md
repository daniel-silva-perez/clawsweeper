## Summary

Keep open. Current main still builds the Feishu card footer from a fixed agent/model/provider note, has no `channels.feishu.cardFooter` or equivalent config surface, and has no `[[card_footer: ...]]` style directive parser. The related items in the provided context cover other Feishu/message features and do not supersede this request.

## Best Possible Solution

Keep this open and implement it in the bundled Feishu plugin: add a typed footer configuration or reply-metadata path, compose footer fields in `extensions/feishu/src/reply-dispatcher.ts`, sanitize any assistant-supplied directive text before delivery, and cover both streaming and non-streaming card paths with tests plus Feishu docs.
