## Summary

Keep open. Current main still builds the Feishu card footer from fixed Agent/Model/Provider fields, has no `channels.feishu.cardFooter` or equivalent config, and has no `[[card_footer: ...]]` directive. Existing `/status` and `/usage` surfaces expose some usage data, but `/usage` appends text to the reply body rather than populating the Feishu card footer, so it does not solve the requested card metadata surface. The related Feishu items in the provided context cover quoted messages, audio, broadcast, and recall/delete work; none supersede this request.

## Best Possible Solution

Keep this open and implement the feature in the bundled Feishu plugin. A conservative path is to add a typed `channels.feishu.cardFooter` option for built-in fields such as context usage and compactions, compose those fields in `extensions/feishu/src/reply-dispatcher.ts`, reuse existing session/status metadata where possible, and cover streaming plus non-streaming card paths with tests and Feishu docs. If maintainers also want an assistant directive, add it through the shared directive parser with strict sanitization.
