## Summary

Keep open. Current main still builds Feishu card notes from fixed Agent/Model/Provider fields and has no Feishu footer config, custom footer directive, or built-in context/compaction footer support. A related open PR, #64805, is an implementation candidate for token/context usage, but it is unmerged and does not cover the full custom-field/config scope requested here.

## What This Changes

Request support for configurable or dynamically injected custom Feishu message-card footer fields, especially context usage and compaction metadata.

## Best Possible Solution

Implement this in the bundled Feishu plugin as a typed `channels.feishu.cardFooter` option for a small set of built-in fields, share the footer composition across streaming and non-streaming card paths, and add Feishu docs plus focused dispatcher/config tests. Treat assistant-supplied footer directives as a later opt-in extension only if maintainers accept the added sanitization surface.

## Reproduction Assessment

Not applicable as a runtime bug, but there is a high-confidence verification path: inspect current `renderMode: "card"` code paths, config schema, directive parser, docs, and tests. Those checks show no custom Feishu footer support on current main.

## Solution Assessment

No, current main is not the best solution because the requested footer customization is still missing. The related open PR #64805 may be useful source work for a token/context subset, but maintainers still need to decide the public config/API shape and whether dynamic footer directives are acceptable.
