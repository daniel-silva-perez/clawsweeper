## Summary

Keep open. Current main has absorbed part of the internal gateway scope cleanup, especially `/exec` persistence blocking and command-level `requiredScopes`, but it still does not implement the PR's central `chat.send` `CommandAuthorized` change or a request-scoped plugin runtime config sink guard. The PR also has an unresolved review concern about breaking read-only slash commands for `operator.write` internal clients, so this needs maintainer security/product direction rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer direction. The best path is to preserve main's command-specific least-privilege model, decide whether command metadata/`requiredScopes` or a broader capability model is the canonical command policy layer, and separately decide whether plugin runtime config writes need a generic request-scoped sink guard across all current config mutation APIs.
