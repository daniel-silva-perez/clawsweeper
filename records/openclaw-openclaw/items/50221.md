## Summary

Keep this PR open. Current main has absorbed part of the internal-scope cleanup, especially `/exec` persistence and command-specific admin gates, but it has not implemented the PR's central `chat.send` authorization change or the proposed plugin runtime config sink guard. The PR also has unresolved review feedback because a blanket `CommandAuthorized=false` for `operator.write` clients can block least-privilege read-only commands before command-specific policy runs.

## Best Possible Solution

Keep the PR open for maintainer direction. The best path is to preserve main's command-specific least-privilege model, use command capability or `requiredScopes` metadata where commands need stricter scopes, decide explicitly whether runtime config writes need a generic sink-level guard, and revise the PR plus tests around that chosen policy.
