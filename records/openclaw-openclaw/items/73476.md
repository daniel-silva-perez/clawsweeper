## Summary

Keep open. Current main does not implement the proposed direct-reply tool-result contract, so this PR is not obsolete. The branch changes core agent-loop/tool-delivery semantics and needs maintainer product/security review, especially around trust rules and treating a `direct_reply` abort as an intentional successful lifecycle rather than a generic prompt abort.

## What This Changes

This PR adds a `directReply` tool-result flag and helper constructors, forwards an abort callback into embedded Pi subscription handling, emits direct block replies for successful marked results, adds handler tests and a changelog entry, and adjusts several auth, gateway, and script tests.

## Best Possible Solution

Keep this PR open for maintainer product/security review. If OpenClaw wants this capability, land it as a typed generic tool-result direct-delivery contract with explicit trust rules, preserved sanitized text/media filtering, a lifecycle reason that is recorded as intentional success rather than a prompt error, integration proof that no follow-up model pass runs, docs/changelog coverage, and split or clearly justified unrelated test changes.
