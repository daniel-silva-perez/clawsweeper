## Summary

Keep open. Current main does not implement the proposed direct tool-result delivery contract, so this PR is not obsolete; the remaining question is whether maintainers want this core agent-loop behavior and its trust/lifecycle semantics.

## What This Changes

The branch adds a `directReply` tool-result flag and helper constructors, wires an abort callback into embedded Pi subscription handling, emits direct block replies for marked successful tool results, adds handler coverage, updates the changelog, and adjusts several existing auth, gateway, and script tests.

## Best Possible Solution

If maintainers want this capability, land it as a typed generic tool-result direct-delivery contract with documented trust rules, preserved sanitized text/media filtering, an intentional successful lifecycle reason, integration proof that no follow-up model pass runs, docs/changelog coverage, and separated or clearly justified unrelated test changes.
