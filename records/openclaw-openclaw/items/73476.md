## Summary

Current main still does not implement this PR's intended direct tool-result reply behavior. The PR remains a live implementation candidate, and the author's follow-up clarifies a real external-skill service use case, but the change lets successful tool output bypass final model synthesis and overlaps with upstream AgentToolResult.terminate semantics, so it needs explicit maintainer product/security review rather than automated cleanup or closure.

## Best Possible Solution

Keep this PR open for explicit maintainer product/security review. If OpenClaw wants this capability, land it as a generic typed tool-result contract with a clear trust policy, a documented relationship to upstream AgentToolResult.terminate, existing media filtering, a typed abort reason, focused tests proving direct delivery without a follow-up model pass, and docs/changelog coverage. Split the unrelated CI assertion-test updates unless they are required for this branch to validate.
