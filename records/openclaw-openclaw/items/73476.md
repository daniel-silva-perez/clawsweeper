## Summary

Keep this PR open. Current main still does not implement the proposed `directReply` tool-result contract, does not pass an `abortRun` callback into embedded tool handlers, and has no helper API for tools to mark a result as a direct reply. The PR remains a live implementation candidate, with maintainer product/security review needed because it would let successful tool output bypass final model synthesis and send text/media directly to users.

## Best Possible Solution

Keep this PR open for maintainer review. If OpenClaw wants this capability, land it as a generic documented tool-result contract with a typed direct-reply marker, explicit trust policy, typed abort reason, media filtering through the existing block-reply pipeline, and an embedded-run integration test proving the direct reply is delivered once while the follow-up LLM inference is skipped. If maintainers prefer ACP/subagent/external-harness delivery instead, close only after that product decision is explicit.

## Risks / Open Questions

- Security/trust boundary: the proposed marker lets a successful tool result bypass final model synthesis and deliver text/media directly to the user. Maintainers should decide whether this is allowed for all tools, trusted tools only, or an explicit capability.
- Media delivery should be reviewed carefully. The PR appears to reuse current media filtering, which is the right direction, but direct block delivery still needs proof that local-media and external/MCP restrictions remain intact.
- Contract durability: the marker should be typed and documented as an OpenClaw tool-result contract, not treated as an incidental extra property on the upstream Pi result shape.
- PR hygiene: the supplied changelog patch includes a forbidden `Thanks @codex` attribution and appears to duplicate/incompletely place the new entry, so it needs cleanup before merge.
- Security review pass: the supplied diff does not add workflows, third-party dependencies, lockfile changes, install/build/release scripts, or package publishing metadata; the notable security-sensitive change is the new runtime direct-delivery behavior.
