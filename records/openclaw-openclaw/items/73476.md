## Summary

Keep this PR open. Current main still lacks a direct-reply tool-result contract, so the contribution is not obsolete, but the latest diff has a definite test compile bug, incomplete intentional-abort lifecycle handling, and a security/product trust boundary that needs maintainer review before merge.

## What This Changes

The PR adds an optional `directReply` tool-result flag, helper constructors, embedded subscription and runner abort wiring, handler logic that emits tool text/media directly, related tests, a changelog entry, and several test-adjustment patches.

## Best Possible Solution

Land this only as a typed, documented direct-delivery contract if maintainers accept the trust model: restrict or declare which tools may use it, preserve sanitized text and media filtering, handle `direct_reply` as intentional success, add integration proof that no follow-up model pass or prompt-error telemetry occurs, and split or justify unrelated CI/test adjustments.
