## Summary

Keep this PR open. Current main still does not implement the proposed tool-result direct reply path, and this PR remains the live implementation candidate. It is not ready to close as implemented or merge as-is because it changes the tool trust boundary, needs a stable result contract and end-to-end run proof, and the current PR head contains conflict markers.

## Best Possible Solution

Keep this PR open for maintainer review. If OpenClaw wants the capability, implement it as a generic documented tool-result contract with a stable typed marker, media filtering through the existing block-reply pipeline, a typed abort reason, clear trust policy, and at least one embedded-run integration test showing the direct reply is delivered once and the follow-up LLM inference is skipped. If maintainers prefer external harness delivery through ACP/subagent flows instead, close only after that product decision is explicit.
