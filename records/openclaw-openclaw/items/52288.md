## Summary

Keep open. Current main still has the reported single-slot client tool call path: `attempt.ts` overwrites `clientToolCallDetected`, `run.ts` converts that one value into a one-element `pendingToolCalls` array, and `/v1/responses` reads only `pendingToolCalls[0]` for both non-stream and stream responses. The linked contributor PR #52287 was a relevant implementation candidate, but it is closed and unmerged, and I did not find a later main-branch replacement.

## Best Possible Solution

Keep the issue open and either revive/rework the closed PR #52287 or land an equivalent fix on main: accumulate client tool calls as an ordered array in the embedded attempt result, preserve all calls in run metadata, emit every pending call in both non-stream and streamed OpenResponses output, and add regression tests for multiple pending client tool calls.
