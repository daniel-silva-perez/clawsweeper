## Summary

Keep this issue open. Current main has provider-specific `reasoning_content` replay fixes for DeepSeek/Venice-style DeepSeek V4 paths and generic stream capture, but I did not find a Qianfan or `baiduqianfancodingplan/qianfan-code-latest` replay/backfill implementation.

## Best Possible Solution

Keep this open and add provider-owned replay support for Qianfan-like OpenAI-compatible thinking models. The likely fix is either a Qianfan-specific stream wrapper/replay hook that backfills `reasoning_content` on replayed assistant messages when thinking is enabled, or a documented custom-provider compatibility option for providers with that contract. Add focused regression coverage for Qianfan/custom OpenAI-compatible tool-call replay and update the Qianfan docs if the old Coding Plan route is retired or renamed.
