## Summary

Keep open. Current main still rejects a generic OpenAI-compatible request-body `model` before chat completions dispatch, even when `x-openclaw-agent-id` has selected the target agent. The active comments describe a concrete production compatibility need, and current source, tests, and docs still encode the stricter agent-target model contract.

## Best Possible Solution

Implement a narrow header-aware compatibility path in the OpenAI-compatible model validation flow: when `x-openclaw-agent-id` selects an agent, ignore the request-body `model` for agent-target validation and let the selected agent's configured model run, while still validating any explicit `x-openclaw-model` override. Add regression coverage for `/v1/chat/completions` with `model: "gpt-4"` plus `x-openclaw-agent-id`, decide whether `/v1/responses` should share the behavior, and update `https://docs.openclaw.ai/gateway/openai-http-api` if this becomes the supported contract.
