## Summary

Keep this PR open. Current main still has the iOS node emitting `location.update`, but the gateway does not handle that event, heartbeat has no location-specific wake or prompt path, and the user-facing docs still describe only `location.get`. The PR remains a focused implementation candidate, not obsolete cleanup.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for the iOS-to-gateway gap. The best path is to land a sanitized, tested gateway and heartbeat handler for `location.update`, or explicitly remove/disable the iOS passive location event path and docs expectations if maintainers decide passive updates should not be a supported product surface.
