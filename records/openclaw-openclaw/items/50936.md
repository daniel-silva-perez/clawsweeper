## Summary

Keep this PR open. Current main still has the iOS node producing `location.update`, but the gateway does not handle that event, heartbeat has no location-specific wake/prompt path, and docs still describe only `location.get`. The PR remains a focused implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for the current iOS-to-gateway gap. The best path is to land a sanitized, tested gateway/heartbeat handler for `location.update`, or explicitly remove/disable the iOS passive location event path and docs expectations if maintainers decide passive location updates should not be supported.
