## Summary

Keep open. Current main still lacks graceful ENOSPC/inotify watcher handling, and this PR is a relevant implementation candidate, but the latest diff should not merge as-is because its global transient classifier still suppresses message-only disk-full/ENOSPC failures without requiring watcher context.

## What This Changes

The PR adds builtin memory watcher error logging plus transient file-watch error classification and unit coverage for ENOSPC/inotify watcher failures.

## Best Possible Solution

Land a narrowed version of this PR that handles the builtin watcher locally, keeps global transient suppression limited to errors with watcher/inotify context, adds no-code disk-full negative tests, and either handles QMD watcher parity in the same patch or records it as an explicit follow-up.

## Reproduction Assessment

Yes. The source path is high-confidence: `memory_search` can open the non-transient builtin manager, current main calls `ensureWatcher()` with `sync.watch` defaulting to true, and current main has neither a watcher `error` listener nor file-watch transient classification; live sysctl exhaustion was not run in this read-only review.

## Solution Assessment

No, not as-is. Adding a source-level chokidar `error` handler is the narrow maintainable seam, but the global message-only classifier must require watcher/inotify context before suppressing `ENOSPC` or disk-full text, and QMD parity needs a maintainer decision.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Tighten message-only disk-full matches:** `src/infra/unhandled-rejections.ts:387-388`
  - body: This branch says ENOSPC must carry watch context, but the no-code branch returns true for any message containing `no space left on device` or `enosp`. A wrapped disk-full write error without `.code` would now be treated as a transient watcher failure by the global unhandled-rejection handler, so require watcher/inotify context for these message-only storage strings and add a no-code disk-full negative test.
  - confidence: 0.92
