## Summary

Keep open. Current main still does not implement graceful ENOSPC/inotify watcher handling, and this active PR is the implementation candidate. The latest PR head is focused and CI checks are green, but it still needs maintainer review for global error-suppression scope and QMD watcher parity.

## Best Possible Solution

Keep this PR open as the implementation candidate. The best path is to tighten `isTransientFileWatchError` so message-only disk-full errors are not globally swallowed, keep regression tests for ENOSPC watcher positives and disk-full negatives, add direct watcher error-handler coverage if feasible, and decide whether `extensions/memory-core/src/memory/qmd-manager.ts` needs the same degraded-mode handling before merge.
