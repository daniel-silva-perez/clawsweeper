## Summary

Keep this PR open. Current main still locally intercepts native shared Chat UI reset and compact slash commands before the generic `chat.send` path, so the PR's central behavior is not implemented; however, the branch has grown into broader iOS/macOS chat UI and exec-approval work with blocking correctness and security-sensitive review findings that need maintainer review or splitting.

## What This Changes

The PR removes shared native Chat UI slash-command fast paths so `/new`, `/reset`, and `/compact` are sent through `chat.send`, adds Swift regression tests, and expands iOS/macOS chat UI, model selection, tool-result, and exec-approval handling.

## Best Possible Solution

Keep the useful slash-command routing direction, but split or re-scope the branch so the shared native Chat UI forwarding change lands with focused Swift regressions while model-picker/UI polish, tool-result behavior, APNs expectations, and exec-approval scope changes receive separate review.

- **[P1] Respect approval-scope gating:** `apps/ios/Sources/Model/NodeAppModel.swift:2452`
  - body: This branch ignores `includeApprovalScope` and always requests `operator.approvals`. Current main preserves compatibility for older paired operator tokens by adding that scope only when a fresh token/password is used or the stored token already has it; forcing a scope upgrade can fail reconnects or require re-pairing.
  - confidence: 0.95
- **[P1] Refresh external final snapshots before returning:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:1255-1258`
  - body: For runs that are not in `runStore`, a final event with a `snapshotMessage` appends only the assistant snapshot and returns before `refreshHistoryAfterRun()`. In a multi-client session this client never loads the corresponding user turn from history, so the transcript remains incomplete until a manual refresh.
  - confidence: 0.9
- **[P1] Accept valid shorter histories after compaction:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:1377-1381`
  - body: `incomingLooksIncomplete` treats any refresh with fewer messages than the previous list as transient while streaming text exists. Legitimate history-shrinking results such as `/compact` or pruning can then retry every 300ms and keep stale messages instead of accepting the completed state.
  - confidence: 0.9
- **[P1] Fallback to loaded tool output before remote fetch:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMessageViews.swift:484-488`
  - body: Expanding a tool-result disclosure depends entirely on `requestDetail(toolCallId)`. If the history fetch fails, the UI shows “Tool output unavailable” even when the loaded message already contains the tool-result text that previous rendering displayed directly.
  - confidence: 0.86
- **[P2] Remove the hard transcript cap:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatView.swift:248`
  - body: Capping `visibleMessages` to the last 50 entries silently drops older loaded chat history with no pagination or load-older path. Long sessions lose scrollback that current main keeps visible.
  - confidence: 0.92
- **[P2] Preserve aborted and error snapshots:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:1306-1307`
  - body: Tracked `aborted` and `error` chat events clear streaming state without consuming `chat.message` or refreshing history. Gateway abort/error events can carry a partial assistant snapshot, so stopping a run can make already generated text disappear.
  - confidence: 0.84
- **[P2] Keep tool-call trace messages visible:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatView.swift:442`
  - body: After the new filtering branches, trace mode still returns `false` for assistant messages that contain only `tool_call` blocks. With `showsAssistantTrace` enabled, those historical tool-call steps should remain visible as they do on current main.
  - confidence: 0.86

  - body: The composer suggests `/clear` as “Clear chat history,” but the branch only marks `/new` and `/reset` as reset-style commands and the current command registry has no `/clear` command. Selecting that suggestion no longer clears history reliably.
