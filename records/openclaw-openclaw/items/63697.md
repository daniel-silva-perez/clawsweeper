## Summary

Keep this PR open. The checked-out current main still implements the opposite of the PR's central behavior: the shared native Chat UI intercepts `/new`, `/reset`, `/clear`, and `/compact` locally before the normal `chat.send` transport path. The PR is an active implementation candidate with related same-author APNs follow-up work and broader iOS/macOS Chat UI/security-sensitive review concerns that need maintainer review or splitting, not cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to split or re-scope it into a focused shared native Chat UI change that forwards `/new`, `/reset`, `/clear`, and `/compact` through `chat.send`, preserves reset/compact transcript semantics with Swift regression tests, and reviews the broader model-picker, message rendering, and exec-approval changes separately. Coordinate the offline APNs client expectations with #67493 rather than treating this PR as obsolete.
