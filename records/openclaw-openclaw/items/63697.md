## Summary

Keep this PR open. Current main still intercepts native iOS/shared Chat UI `/new`, `/reset`, `/clear`, and `/compact` before the generic `chat.send` path, so the PR's central behavior is not implemented on main. The PR is also paired with same-author APNs follow-up work and has broad UI/security-sensitive review concerns that need maintainer review or splitting, not conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to split or re-scope it into a focused shared native Chat UI change that forwards reset/compact slash commands through `chat.send`, preserves transcript/reset semantics with Swift regression tests, and reviews the broader UI/model-picker/exec-approval changes separately. Coordinate offline APNs client expectations with #67493 before shipping that full experience.
