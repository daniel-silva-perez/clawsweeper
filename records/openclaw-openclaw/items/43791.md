## Summary

Keep open. Current main still does not preserve full embedded tool-result `ReplyPayload` values in the final embedded Pi run payload list, so this PR is not obsolete. The branch should not merge as-is because it can suppress the explicit timeout error when a timed-out run has only buffered tool-result payloads, and its changelog entry lacks the required attribution.

## What This Changes

The PR buffers embedded Pi runner tool-result `ReplyPayload` values, clears that buffer on compaction retry, prepends buffered payloads for non-streaming final consumers, updates subscribe types/tests, and adds a changelog entry.

## Best Possible Solution

Rebase the useful payload-preservation work onto current main, preserve full `ReplyPayload` metadata only for final consumers that did not already receive live tool results, clear buffered data across compaction retries, make timeout synthesis depend on assistant/final reply production rather than buffered tool callbacks, and credit the changelog entry per repo policy.
