## Summary

Keep this PR open: current main still lacks the full embedded tool-result payload preservation the PR is trying to add, so the work is not obsolete. The patch should not merge as-is because buffered tool-result payloads can still mask timeout failures, and the changelog entry is missing the required contributor attribution.

## What This Changes

The PR buffers embedded Pi runner tool-result `ReplyPayload` values in attempt results, replays them into final run payloads for non-streaming callers, resets the buffer on compaction retry, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Rebase the useful payload-preservation work onto current main, preserve full `ReplyPayload` metadata only for final consumers that did not already receive live tool results, make timeout synthesis depend on assistant/final reply production rather than buffered tool callbacks, keep compaction-retry buffer reset coverage, and credit the changelog entry per repo policy.
