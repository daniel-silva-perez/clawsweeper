## Summary

Keep open. Current main still does not preserve full embedded tool-result `ReplyPayload` values in final embedded run results, so this PR is not obsolete; however, the patch should not merge yet because buffered tool-result payloads can suppress the explicit timeout error when no assistant reply was produced.

## What This Changes

The PR buffers embedded Pi runner tool-result `ReplyPayload` values, replays them into final run payloads only when no live tool-result callback is present, clears the buffer on compaction retry, and adds regression tests plus a changelog entry.

## Best Possible Solution

Rebase the useful payload-preservation work onto current main, preserve full `ReplyPayload` metadata for non-live final-payload consumers, keep live-streamed tool results from being replayed, clear buffers across compaction retries, and make timeout synthesis depend on assistant/final reply production rather than any buffered tool callback, with targeted regression coverage for each case.
