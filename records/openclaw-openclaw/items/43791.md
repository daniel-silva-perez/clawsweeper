## Summary

Keep this PR open. Current main still lacks a buffered full-ReplyPayload return path for embedded tool results; it only has a narrower orphaned-media fallback that carries media URLs/audio flags into final payload assembly. The PR remains a plausible implementation candidate, but it needs a rebase onto the split embedded-runner tests/current reset path, explicit timeout-contract review, and changelog attribution before merge.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up. The best path is to rebase onto current main, preserve full ReplyPayloads only for callers that are not already streaming onToolResult, reconcile the existing orphaned tool-media fallback so media/audio are not duplicated or reordered, settle timeout behavior with a regression test, keep the compaction-retry buffer reset in the current reset path, and fix changelog attribution before merge.
