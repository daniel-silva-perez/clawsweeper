## Summary

Keep open. Current main has meaningful prompt-cost mitigations for skills, workspace bootstrap caps/optional continuation skipping, context reporting, and compaction, but it still does not implement the central request: automatic per-turn relevance routing for tool schemas. The runner still builds a static effective tool set and converts every effective tool into model-facing custom tool definitions with descriptions and parameter schemas.

## Best Possible Solution

Keep this issue open as the broad prompt-overhead tracker. The best implementation is a policy-preserving per-turn tool relevance/schema-routing layer near the `effectiveTools` to `customTools` boundary, while retaining static allow/deny/profile enforcement and fail-closed semantics. Preserve the existing mitigations for compact/on-demand skills, bootstrap caps and `contextInjection`, context reporting, prompt-cache stability, and compaction.
