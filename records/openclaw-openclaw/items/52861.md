## Summary

Keep open. Current main has meaningful prompt-cost mitigations, but it still does not implement the central request: automatic per-turn conditional relevance routing for tool schemas. The runner builds the full effective tool set after static policy/profile filtering, converts every effective tool into a model-facing custom tool definition with description and parameters, and passes the whole list into the embedded agent session. Skills are partly addressed through compact/on-demand loading, workspace bootstrap has caps and configurable skipping, and compaction exists, but those do not fully solve the broad per-turn overhead request.

## Best Possible Solution

Keep this issue open as the broad prompt-overhead tracker. The best implementation would add a policy-preserving per-turn tool relevance/schema-routing layer near the `effectiveTools` to `customTools` boundary, while retaining static allow/deny/profile enforcement and fail-closed semantics. Separately, add or refine default/opt-in workspace bootstrap summary/index/on-demand behavior. Preserve the already-shipped mitigations: compact/on-demand skills, tool profiles and allow/deny lists, bootstrap caps and `contextInjection`, prompt-cache ordering, and compaction.
