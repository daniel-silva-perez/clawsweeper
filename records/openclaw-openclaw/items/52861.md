## Summary

Keep open. Current main has meaningful prompt-cost mitigations for skills, workspace bootstrap, and compaction, but it still does not implement the central request for automatic per-turn relevance routing of tool schemas; the embedded runner still converts the full effective tool set into model-facing custom tools for each run.

## What This Changes

The issue requests lower repeated token overhead through conditional per-turn tool schema injection, lazy skill/workspace context loading, and conversation compaction.

## Best Possible Solution

Keep this as the broad prompt-overhead tracker and split implementation into policy-preserving pieces: a per-turn tool schema relevance layer near the effectiveTools-to-customTools boundary, plus narrower follow-ups for skill conditional activation and workspace bootstrap indexing while preserving existing compact skills, tool profiles, bootstrap caps, and compaction behavior.
