## Summary

Keep this PR open. Current main still misses FastAPI-style top-level string `detail` payloads in the shared assistant error formatter, while the PR is a narrow code-and-test fix for that remaining raw JSON leak and has no blocking review finding from this sweep.

## What This Changes

This PR adds top-level string `detail` recognition and message extraction in shared assistant error formatting, with exact-output regression tests for assistant-message and raw-error formatting paths.

## Best Possible Solution

Land a rebased or ported version of the focused string-`detail` parser support with the exact-output regression tests, while keeping provider credential-gating and unconfigured-provider suppression under the broader linked bug.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: on current main, `{"detail":"..."}` is rejected by `parseApiErrorPayload`, so the formatter path cannot extract the message and can leak the raw JSON text.

## Solution Assessment

Yes. For the raw JSON leak covered by this PR, recognizing top-level string `detail` and using it as the fallback top-level message is the narrowest maintainable fix; broader provider-probe suppression should remain separate.
