## Summary

Current main still contains the duplicated Control UI bootstrap config path behavior, so this remains a valid narrow gateway/UI bug rather than a cleanup-close candidate. The shared constant is still an absolute OpenClaw-prefixed path, while the frontend loader and gateway handler both prepend the configured or inferred base path.

## What This Changes

The issue reports that opening Control UI under `/__openclaw__/` causes the frontend to request `/__openclaw__/__openclaw/control-ui-config.json`, receive a 404, and leave chat unusable.

## Best Possible Solution

Land a focused Control UI/gateway fix that makes the bootstrap config endpoint consistently base-path-relative across the shared contract, frontend loader, gateway route, Vite dev stub, tests, and public docs, with explicit coverage for `gateway.controlUi.basePath: "/__openclaw__"`.
