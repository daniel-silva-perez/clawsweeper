## Summary

Current main still contains the duplicated Control UI bootstrap config path behavior, so this remains a valid narrow gateway/UI bug rather than a cleanup-close candidate. The shared constant is still an OpenClaw-prefixed path, while both the frontend loader and gateway handler prepend the configured or inferred base path.

## What This Changes

The issue reports that opening the Control UI under `/__openclaw__/` makes the frontend request `/__openclaw__/__openclaw/control-ui-config.json`, receive a 404, and leave chat unusable.

## Best Possible Solution

Land a focused Control UI/gateway fix that makes the bootstrap config endpoint consistently base-path-relative across the shared contract, frontend loader, gateway route, Vite dev stub, tests, and public docs, with explicit coverage for `gateway.controlUi.basePath: "/__openclaw__"` or the equivalent mounted route.

## Reproduction Assessment

Yes. The current source gives a high-confidence static reproduction path: `/__openclaw__/` can infer `/__openclaw__` as `basePath`, the frontend then requests `/__openclaw__/__openclaw/control-ui-config.json`, and the default gateway bootstrap route remains `/__openclaw/control-ui-config.json`.

## Solution Assessment

Yes. The narrowest maintainable fix is to make the bootstrap config path base-path-relative and update the gateway route, frontend loader, dev stub, tests, and docs together, without broad changes to Control UI routing, auth policy, plugin routes, or gateway probe behavior.
