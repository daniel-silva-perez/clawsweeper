## Summary

Current main still contains the duplicated Control UI bootstrap path behavior, so this remains a valid recent behavior bug rather than a cleanup close candidate. The old constant, frontend concatenation, gateway route, tests, and docs all still encode the endpoint shape that can produce the reported 404 when the UI is opened under `/__openclaw__/`.

## What This Changes

The issue reports that opening Control UI under `/__openclaw__/` makes the frontend request `/__openclaw__/__openclaw/control-ui-config.json`, returning 404 and preventing chat from working.

## Best Possible Solution

Land a focused Control UI/gateway fix that makes the bootstrap config endpoint consistently base-path-relative, keeps the gateway route, frontend loader, dev stub, tests, and docs in agreement, and includes explicit coverage for `gateway.controlUi.basePath: "/__openclaw__"`.
