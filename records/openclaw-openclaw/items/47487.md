## Summary

Keep open. Current main still grants runtime tools from `tools.exec` before restricted profile filtering, and the focused fix PR was closed unmerged while the broader related reports do not supersede this exact authorization-boundary bug.

## What This Changes

The issue asks OpenClaw to enforce restricted agent tool profiles so configured `tools.exec` safety settings do not implicitly expose `exec` when an agent's `alsoAllow` excludes it.

## Best Possible Solution

Revise the tool-policy contract so global `tools.exec` configuration cannot widen restricted agent profiles; document the intended global-versus-agent opt-in semantics, add regression coverage for `messaging` plus explicit `alsoAllow`, and evaluate the analogous `tools.fs` behavior in the same policy pass.
