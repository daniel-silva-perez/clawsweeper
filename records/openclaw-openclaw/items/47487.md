## Summary

Keep open. Current main still grants `exec` and `process` when a global or agent `tools.exec` section exists, then unions that implicit grant with explicit `alsoAllow`, so the reported restricted-profile bypass remains present. The focused fix PR #47991 is closed and unmerged, and the broader #69190 report does not supersede this specific policy bug.

## What This Changes

Agent-specific restricted tool profiles should not expose `exec` merely because `tools.exec` safety settings are configured when `exec` is absent from the allowed tool list.

## Best Possible Solution

Revise the tool-policy contract so global `tools.exec` safety configuration does not widen restricted agent profiles. Add focused regression coverage for `messaging` plus explicit `alsoAllow` with global `tools.exec`, preserve or deliberately revise per-agent opt-in semantics, and update the public tool-policy docs if the access model changes.
