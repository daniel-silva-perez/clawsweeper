## Summary

Keep open. Current main still has the reported restricted-profile bypass: configured global or agent `tools.exec` injects `exec` and `process` into `profileAlsoAllow`, and that implicit grant is unioned with explicit `alsoAllow` before gateway and embedded-runner policy filtering.

## What This Changes

The issue asks for agent tool profiles with explicit `alsoAllow` lists to block `exec` unless it is explicitly allowed, even when global `tools.exec` safety settings are configured.

## Best Possible Solution

Define the tool-policy contract so global exec safety configuration cannot widen a restricted agent profile, preserve or intentionally revise per-agent opt-in behavior, add focused regression tests for explicit `alsoAllow`, and update the public tool-policy docs if the access model changes.
