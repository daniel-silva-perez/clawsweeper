## Summary

Keep open. Current main still grants `exec`/`process` from any configured global or agent `tools.exec` section and merges that implicit grant with explicit `alsoAllow`, so the reported restricted-profile bypass remains present. The related fix PR #47991 is closed and unmerged; #69190 may overlap but is broader and does not supersede this focused bug.

## What This Changes

Agents configured with restricted `tools.profile`/`alsoAllow` should not regain `exec` access merely because `tools.exec` safety settings are configured.

## Best Possible Solution

Keep this issue open and fix the policy contract: configuring global `tools.exec` safety settings should not grant `exec`/`process` to agents whose restricted profile and explicit `alsoAllow` omit runtime tools. Add regression coverage for agent-level `messaging` plus explicit `alsoAllow` with global `tools.exec`, decide whether agent-level `tools.exec` should remain an explicit opt-in, and review the matching global `tools.fs` implicit exposure path in the same policy pass.
