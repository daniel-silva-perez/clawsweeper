## Summary

Keep open. Current main still grants `exec` and `process` whenever `tools.exec` is configured globally or per-agent, then unions that implicit grant with explicit `alsoAllow`; the related focused fix PR was closed unmerged, and the broader bundled-tool policy fix does not cover this profile-stage grant.

## What This Changes

The issue asks for agent tool profiles to block `exec` when a restricted profile such as `messaging` has an explicit `alsoAllow` list that excludes `exec`, even if global exec safety settings are configured.

## Best Possible Solution

Separate execution safety configuration from access grants: global `tools.exec` should not widen restricted agent profiles, any per-agent opt-in behavior should be intentional and documented, and regression coverage should lock the gateway and embedded-runner policy paths together.
