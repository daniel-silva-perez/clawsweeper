## Summary

Keep open. Current main still re-adds `exec` and `process` at the profile stage whenever global `tools.exec` is configured, so an agent using `tools.profile: "messaging"` with `tools.alsoAllow` that excludes `exec` can still get `exec`. The related PR #47991 was closed unmerged, and current tests still assert the implicit re-exposure behavior.

## Best Possible Solution

Keep this issue open and fix `resolveImplicitProfileAlsoAllow()` so global `tools.exec`/`tools.fs` safety sections do not grant profile-stage tool access for restricted agents with explicit profiles or `alsoAllow` exceptions. Preserve intentional agent-level tool-section opt-ins if maintainers still want that behavior, and add regression tests for a `messaging` agent with explicit `alsoAllow` excluding `exec`, plus release-note or docs clarification for the safety-config-vs-access-config distinction.
