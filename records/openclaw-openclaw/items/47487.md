## Summary

Keep open. Current main still widens restricted agent profiles when `tools.exec` is configured, and the focused fix PR was closed unmerged while related reports do not supersede this exact exec authorization-boundary bug.

## What This Changes

The issue requests that restricted agent tool profiles, especially `messaging` with an explicit `alsoAllow` list excluding `exec`, prevent `exec` and `process` from being exposed when global `tools.exec` is configured.

## Best Possible Solution

Revise the tool-policy contract so global `tools.exec` configuration cannot widen restricted agent profiles, preserve any intended per-agent explicit opt-in semantics, add focused regression coverage for `messaging` plus explicit `alsoAllow`, document the global-versus-agent behavior, and review the matching `tools.fs` path in the same policy pass.
