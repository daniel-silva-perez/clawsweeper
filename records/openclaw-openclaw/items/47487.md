## Summary

Keep open. Current main still grants `exec`/`process` from any configured global or agent `tools.exec` section and merges that implicit grant with explicit `alsoAllow`, so an agent using a restricted `messaging` profile can still get runtime tools contrary to the documented profile boundary.

## Best Possible Solution

Keep this issue open. The best fix is to make the contract explicit and enforce it: configuring global `tools.exec` safety settings must not grant `exec`/`process` access to agents with restricted profiles and explicit `alsoAllow` that excludes runtime tools. Add regression coverage for agent-level `messaging` plus explicit `alsoAllow` with global `tools.exec`, preserve intentional agent-level `tools.exec` opt-in if that remains the desired contract, and review the adjacent global `tools.fs` implicit exposure path in the same policy pass.
