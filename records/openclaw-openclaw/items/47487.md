## Summary

Keep open. Current main still widens restricted tool profiles when `tools.exec` is configured, and the linked focused fix PR was closed unmerged; the issue is valid but needs maintainer policy judgment because it changes an exec authorization boundary.

## What This Changes

The issue asks for agent `tools.profile: "messaging"` plus an explicit `alsoAllow` list that excludes `exec` to actually block the `exec` tool, even when global `tools.exec` safety settings are configured.

## Best Possible Solution

Revise the tool-policy contract so global `tools.exec` configuration cannot widen restricted agent profiles, preserve any intended per-agent explicit opt-in semantics, add regression coverage for `messaging` plus explicit `alsoAllow`, document the global-versus-agent behavior, and review the analogous `tools.fs` path in the same policy pass.
