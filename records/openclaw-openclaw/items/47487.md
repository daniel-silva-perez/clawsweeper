## Summary

Keep open. Current main still grants `exec` and `process` from a configured global or agent `tools.exec` section, then unions that implicit grant with explicit `alsoAllow`, so the reported restricted-profile bypass remains present. The focused fix PR #47991 is closed and unmerged, and the broader #69190 report does not supersede this specific policy bug.

## What This Changes

The issue asks OpenClaw to enforce agent tool-profile restrictions so a `messaging` agent with an explicit `alsoAllow` list that omits `exec` cannot use `exec` merely because global `tools.exec` safety settings are configured.

## Best Possible Solution

Keep the issue open and change the policy contract so global `tools.exec` safety settings do not widen restricted agent profiles. Add focused coverage for `messaging` plus explicit `alsoAllow` with global `tools.exec`, preserve or deliberately revise per-agent opt-in semantics, and align docs if the access model changes.
