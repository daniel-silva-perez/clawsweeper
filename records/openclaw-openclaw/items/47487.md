## Summary

Keep open. Current main still grants `exec` and `process` when a global or agent `tools.exec` section exists, then unions that implicit grant with explicit `alsoAllow`, so the restricted-profile bypass remains present. The closed, unmerged #47991 was a focused fix attempt, while the broader #69190 does not supersede this specific policy bug.

## What This Changes

The issue asks for agent tool profiles and explicit `alsoAllow` lists to block `exec` unless that agent explicitly allows it, even when `tools.exec` is configured globally.

## Best Possible Solution

Revise the tool-policy contract so global `tools.exec` configuration does not widen restricted agent profiles. Add focused regression coverage for `messaging` plus explicit `alsoAllow` with global `tools.exec`, decide and document the per-agent opt-in semantics, and review the analogous `tools.fs` path at the same time.
