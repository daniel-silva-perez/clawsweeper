## Summary

This PR should stay open. Current `main` still treats `llm_input` and `llm_output` as observation-only hooks: the public hook handler types return `void`, the hook runner dispatches both through `runVoidHook`, and the embedded runner invokes both without awaiting or consuming a result. The author’s manual gateway/security-plugin testing and follow-up review work make this a plausible implementation candidate, but the intended behavior is not implemented on `main`. Related #20802 is also open and broader, so it is not a shipped superseding fix.

## Best Possible Solution

Keep this PR open for maintainer review or consolidation with #20802. The best path is to decide the product contract for modifying `llm_input` and `llm_output`, land one reviewed implementation with focused tests and docs updates, and close overlapping PRs only after the chosen implementation merges or maintainers explicitly pick a canonical thread.
