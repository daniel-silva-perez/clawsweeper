## Summary

Keep open. Current main still shows the gateway completion-wake path forwarding only the caller-provided extra system prompt while normal chat builds richer inbound/group prompt context, and the open #63096 implementation candidate still uses closing syntax for this issue. The image-history cache churn raised in comments also remains a separate unresolved concern rather than a proven fixed behavior.

## What This Changes

The issue asks OpenClaw to keep same-session prompt/cache prefixes stable across normal chat, heartbeat, and ACP announce paths, and to avoid image-history pruning that churns Anthropic prompt caches.

## Best Possible Solution

Review #63096 on its merits, require a regression or live cache-trace proof that same-session normal chat, heartbeat, and completion-wake prompts remain stable enough for Anthropic cache reuse, and split the image-history cache churn into a focused follow-up if it is outside that PR's scope.
