## Summary

Keep this open. Current main still has real cron run queuing, no cron test/validate command, no input-driven skill test command, and no global tool dry-run interception; the maintainer-side issue discussion also already kept this open and tied the tool-interception part to the open global dry-run design in #41418.

## What This Changes

The issue requests dry-run and validation support for cron jobs and skills, including CLI commands, mockable tool interception, CI-friendly output, and side-effect-free execution previews.

## Best Possible Solution

Design an additive, documented contract that adds cron test/validate RPC plus CLI, a skill test runner with explicit input and JSON/exit-code semantics, and a coordinated dry-run or mock/read-only tool interception layer shared with the broader #41418 work.
