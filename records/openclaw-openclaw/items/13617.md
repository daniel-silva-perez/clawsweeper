## Summary

Keep open. Current main still lacks the requested side-effect-free cron test/validate commands, input-driven skills test command, and shared mock/read-only tool interception layer; the recent maintainer and ClawSweeper comments also treated this as valid remaining design work.

## What This Changes

The issue requests side-effect-free test and validation support for cron jobs and skills, including dry-run execution previews, API/tool mocking, JSON/CI output, and related CLI commands.

## Best Possible Solution

Keep this open and design an additive implementation: cron test/validate RPC plus CLI, an input-driven skills test runner with JSON/exit-code semantics, and a shared dry-run/mock/read-only interception contract coordinated with #41418, followed by docs and focused CLI/protocol/runtime tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is static inspection of current CLI registration, Gateway protocol/handlers, docs, and the shared tool adapter showing the requested surfaces are still absent.

## Solution Assessment

Unclear until maintainer design is settled. The requested direction is useful, but the maintainable path is a coordinated protocol/runtime/tool-interception design rather than ad hoc cron-only or skill-only stubs.
