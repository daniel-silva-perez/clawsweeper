## Summary

Keep open. Current main still lacks the requested cron test/validate commands, input-driven skill test runner, and shared side-effect-free tool interception layer; the recent maintainer and ClawSweeper comments reached the same keep-open conclusion and the central code/docs still match that finding.

## What This Changes

Add a dry-run/test framework for cron jobs and skills, including cron validation/test CLI commands, skill testing with input, API/tool mocking, JSON/exit-code CI support, and configuration for dry-run/mock/read-only behavior.

## Best Possible Solution

Design an additive implementation with cron test/validate RPC plus CLI, a skill test runner with explicit input and JSON/exit-code semantics, and a shared dry-run/mock/read-only interception contract aligned with #41418, then document and test the new CLI/protocol/runtime behavior.
