Keep open. Current main still lacks the requested side-effect-free cron test/validate commands, input-driven skill test command, and global/mockable tool interception layer; the issue discussion already kept this open and linked the shared dry-run layer to open #41418.

Design an additive implementation: cron test/validate RPC plus CLI, a skill test runner with explicit input and JSON/exit-code semantics, and a coordinated dry-run/mock/read-only tool interception contract aligned with #41418, with docs and focused coverage for each affected surface.
