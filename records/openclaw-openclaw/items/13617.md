## Summary

Keep this open. Current main still lacks side-effect-free cron test/validate commands, an input-driven skill test runner, and a global/mockable tool interception layer; the relevant cron, skills, protocol, docs, and tool-adapter files have not changed since the prior maintainer-side ClawSweeper review that also kept this open.

## What This Changes

The issue requests a dry-run testing framework for cron jobs and skills, including cron test/validate commands, skill input testing, mocked or read-only tool/API behavior, JSON output, and CI-friendly exit codes.

## Best Possible Solution

Keep this issue open and design an additive implementation: cron test/validate RPC plus CLI, a skills test runner with explicit input and CI output semantics, and a coordinated tool-interception contract with #41418 for any global dry-run or mock/read-only layer.
