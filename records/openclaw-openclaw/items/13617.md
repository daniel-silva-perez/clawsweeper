## Summary

Keep open. Current main still lacks the requested side-effect-free cron test/validate commands, input-driven skill test runner, and shared dry-run/mock tool interception contract; the issue has recent maintainer-side review comments reaching the same conclusion, and the related global dry-run proposal remains open rather than shipped.

## What This Changes

Add side-effect-free dry-run/test/validate support for cron jobs and skills, including CLI commands, mock/read-only tool behavior, CI-friendly output, and configuration.

## Best Possible Solution

Keep this issue open and design an additive implementation: cron test/validate RPC plus CLI, a skill test runner with explicit input and JSON/exit-code semantics, and a coordinated dry-run/mock/read-only tool interception contract aligned with #41418, with docs and focused coverage for each affected surface.
