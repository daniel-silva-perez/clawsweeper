## Summary

Keep this open. Current main still lacks the requested side-effect-free cron test/validate commands, input-driven skill test runner, and global tool/API dry-run interception; the existing cron and skills surfaces only cover real queued cron runs, run history/status, and skill discovery/readiness checks.

## What This Changes

The issue requests a dry-run testing framework for cron jobs and skills, including cron test/validate commands, skill input testing, mocked or read-only tool/API behavior, and CI-friendly JSON/exit-code output.

## Best Possible Solution

Keep the issue open and design an additive split: cron test/validate RPC plus CLI, an explicit skills test runner with input and CI output semantics, and a coordinated decision with #41418 for any global mock/read-only tool interception layer.
