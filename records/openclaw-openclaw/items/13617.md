## Summary

Keep this issue open. Current main still has cron run/history/inspection and skills search/install/update/list/info/check surfaces, but no side-effect-free cron test/validate command, no input-driven skill test runner, and no global tool dry-run interception layer. The related global dry-run design remains open in #41418, so this needs maintainer design or splitting rather than cleanup closure.

## Best Possible Solution

Keep this issue open until maintainers either split it into narrower canonical issues or design an additive testing surface: cron test/validate Gateway protocol plus CLI, an explicit skills test-runner model, structured JSON/exit-code behavior for CI, and alignment with #41418 for global tool interception.
