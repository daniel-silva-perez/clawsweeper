## Summary

Keep this issue open. Current main still lacks the requested side-effect-free cron/skills testing framework: cron has real run/history/inspection commands but no test or validate command, skills has search/install/update/list/info/check but no test runner, and the shared agent tool path still calls tool.execute without a global dry-run interception layer. The related global dry-run design remains tracked separately in open #41418, so this needs maintainer design/splitting before implementation.

## Best Possible Solution

Keep this issue open until maintainers either ship an additive side-effect-free testing surface or split the work into canonical narrower issues. The likely path is cron test/validate CLI and gateway protocol support, an explicit skill test-runner decision, structured JSON output for CI, and alignment with the global dry-run/tool-interception design tracked by #41418.
