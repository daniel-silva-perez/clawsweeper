## Summary

Keep open. Current main has a failure-class-aware fallback foundation, but it still does not implement the requested hard-auth provider quarantine, invalid-request fail-fast policy, or provider quarantine/health in the normal status surface; related #61328 closed only the broader foundation work.

## What This Changes

Request provider fallback to classify failures so auth-broken providers are quarantined/skipped, rate limits respect Retry-After, network errors get short cooldowns, invalid requests fail fast, and status exposes quarantine health.

## Best Possible Solution

Adopt a documented provider/profile health model that distinguishes hard auth, transient auth refresh, rate limits, network failures, and invalid requests; apply that model consistently in fallback selection; and expose a redacted provider health snapshot in `openclaw status` while keeping detailed diagnostics in `openclaw models status`.
