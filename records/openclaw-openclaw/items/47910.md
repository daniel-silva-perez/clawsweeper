## Summary

Keep open. Current main has the shipped failure-class-aware fallback foundation, but the requested hard-auth quarantine policy, invalid-request fail-fast behavior, and provider quarantine/health in the normal status surface are still not implemented; related #61328 covered the broader foundation rather than this remaining operator-facing behavior.

## What This Changes

Request provider fallback to classify failures so auth-broken providers are quarantined and skipped, rate limits and network failures get class-specific handling, invalid requests fail fast, and status exposes provider quarantine health.

## Best Possible Solution

Define a documented provider/profile health model that separates hard auth failures from transient refresh, rate-limit, and network failures; apply that state consistently in fallback selection; decide the invalid-request fail-fast policy; and expose a redacted provider health snapshot in status while leaving deeper diagnostics in models status.
