## Summary

Keep open. Current main has useful failure-class-aware fallback groundwork, including auth/auth_permanent cooldown skipping, but it still does not implement the requested hard-auth quarantine policy, invalid-request fail-fast behavior, or provider quarantine/health in the normal status surface.

## What This Changes

Request provider fallback to classify failures so auth-broken providers are quarantined and skipped, rate limits and network failures get class-specific handling, invalid requests fail fast, and status exposes provider quarantine health.

## Best Possible Solution

Define a documented provider/profile health model that separates hard auth failures from transient refresh, rate-limit, network, billing, quota, and schema failures; apply it consistently in fallback selection; decide invalid-request fail-fast semantics; and expose a redacted provider health snapshot in normal status while keeping detailed diagnostics under models status.
