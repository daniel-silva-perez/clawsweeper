## Summary

Keep open. Current main still only shows the Homebrew installer command during skill onboarding, and gateway-backed skill dependency installs still fail with a missing-brew message instead of bootstrapping Homebrew. The remaining decision is not whether the gap exists, but whether OpenClaw should run Homebrew's external installer from onboarding/gateway flows; that needs maintainer product and security policy before implementation.

## What This Changes

The issue requests that skill onboarding install Homebrew automatically when it is missing instead of only showing the installation command.

## Best Possible Solution

Keep the issue open until maintainers decide the Homebrew bootstrapping policy. If allowed, implement an explicit opt-in macOS flow shared by skill onboarding and gateway-backed installs, re-detect `brew` after installation, preserve security scanning and admin/operator consent, update UI/CLI errors and public docs, and add focused tests. If not allowed, make the manual Homebrew requirement clearer in the prompt, Skills UI, and docs.
