## Summary

Keep open. Current main still matches the reported behavior: the skill onboarding path only offers to show the Homebrew install command, while gateway-backed skill installs return a missing-brew failure when a brew-backed skill is selected without `brew`. This remains a valid feature request, but the actual fix needs maintainer product/security policy because it would execute Homebrew's external installer and may require administrator privileges.

## What This Changes

The issue requests that OpenClaw automatically install Homebrew when a brew-backed skill install or onboarding path detects that `brew` is missing.

## Best Possible Solution

Keep this open until maintainers decide the policy for Homebrew bootstrapping in skill dependency onboarding. If allowed, add an explicit macOS confirmation flow in the gateway-backed install path, reuse the standalone installer's admin/failure expectations, re-detect `brew`, and cover CLI, gateway, and Skills UI behavior; if not allowed, make the prompt and docs clearly present Homebrew as a manual prerequisite.
