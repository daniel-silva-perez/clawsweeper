## Summary

Keep open. Current main still only shows a Homebrew install command during skill onboarding and still returns a missing-brew failure from gateway-backed skill installs. The standalone macOS installer has a separate Homebrew bootstrap path, but that path is not wired into skill dependency onboarding or `skills.install`; implementing this would execute Homebrew's external installer and needs maintainer product/security policy first.

## What This Changes

The issue requests that OpenClaw automatically install Homebrew when a selected skill dependency installer needs `brew` and it is missing.

## Best Possible Solution

Keep the issue open until maintainers choose the Homebrew bootstrapping policy. If allowed, add an explicit opt-in flow for macOS skill dependency onboarding and gateway-backed installs, use a reviewed execution path, re-detect `brew`, and update UI/CLI errors plus docs; if not allowed, make Homebrew clearly manual in the prompt, Skills UI, and docs.
