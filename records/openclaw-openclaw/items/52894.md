## Summary

Keep open. Current main still matches the reported behavior: skill onboarding shows the Homebrew install command instead of running it, and gateway-backed skill installs fail with a missing-brew error when no brew executable is available. The remaining decision is product/security policy for whether OpenClaw should execute Homebrew's external installer from onboarding or gateway-controlled flows.

## What This Changes

Request automatic Homebrew bootstrapping when a selected skill dependency needs brew and brew is not installed.

## Best Possible Solution

Keep the issue open for maintainer product/security direction. If approved, implement an explicit opt-in shared Homebrew bootstrap path for skill onboarding and gateway-backed skill installs, re-check brew after installation, preserve skill install security scanning and user consent, update CLI/UI/docs, and add focused tests; if not approved, improve the manual Homebrew requirement and failure messaging instead.
