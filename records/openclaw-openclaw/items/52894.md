## Summary

Keep open. Current main still does not perform the requested Homebrew bootstrap: onboarding only shows the command and the gateway-backed skill installer returns a missing-brew failure. Because the requested change would execute Homebrew's external installer and can involve administrator privileges, it needs maintainer product/security policy before implementation.

## What This Changes

Request automatic Homebrew installation when a user selects brew-backed skill dependencies and `brew` is not installed.

## Best Possible Solution

Keep this open until maintainers choose the Homebrew bootstrapping policy. If allowed, add an explicit opt-in flow for macOS skill dependency onboarding and gateway-backed installs, run the Homebrew installer through a reviewed execution path, re-detect `brew`, and surface clear UI/CLI errors; if not allowed, tighten the prompt, Skills UI, and docs so Homebrew is clearly a manual prerequisite.
