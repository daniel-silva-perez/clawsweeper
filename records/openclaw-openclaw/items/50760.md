## Summary

Keep this PR open. Current main still does not implement the intended browser Control UI Task Board tab/page; it ships task tracking through the CLI and chat `/tasks`, plus separate Sessions and Cron panels, but no unified `/tasks` Control UI route, state, renderer, or task-board module. The PR remains net-new UI work rather than obsolete or already implemented.

## Best Possible Solution

Keep this PR open if maintainers still want a browser Control UI Task Board. Rebase it onto current main, align the page with the shipped task ledger or an explicit Gateway/UI task contract, wire the `tasks` tab through navigation/state/i18n/rendering, and resolve the existing adapter/page review findings with focused tests.
