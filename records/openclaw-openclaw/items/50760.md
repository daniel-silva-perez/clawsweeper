## Summary

Keep open. Current main has a background-task ledger, CLI inspection, and chat-native `/tasks`, but it still does not implement this PR’s intended browser Control UI Task Board tab, route, state, service, or components. The PR remains net-new UI work, with review findings still to address before merge.

## Best Possible Solution

Keep this PR open if maintainers still want a browser Control UI Task Board. The best path is to rebase it onto current main, decide whether the page should read from the shipped task ledger through an appropriate Gateway/UI contract rather than merging sessions and cron data directly, wire the tab through navigation/state/i18n/rendering, address the existing review findings, and keep focused Control UI tests. Do not close it as implemented by the chat-native `/tasks` command or CLI task ledger alone.
