## Summary

Current main still has the misleading config.patch restart notice wording, so this issue should stay open. The formatter, config.patch handler, sentinel writer, startup delivery path, and existing tests all still line up with the reported `Gateway restart config-patch ok (config.patch)` summary shape rather than a restart-required notice.

## What This Changes

The issue requests clearer config.patch restart notice wording that says a gateway restart is required, while preserving the note text below it.

## Best Possible Solution

Implement a focused restart sentinel UX fix so config.patch restart-required notices use wording like `Gateway restart required (config.patch)`, preserve the submitted note beneath the summary, and keep unrelated update/restart/auto-recovery summaries unchanged.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a config.patch write that creates a `kind: "config-patch"`, `stats.mode: "config.patch"` sentinel will be formatted through `summarizeRestartSentinel` into the reported old summary shape.

## Solution Assessment

Yes. The narrowest maintainable fix is in the restart sentinel formatter or config-write sentinel payload path, backed by targeted formatter coverage, rather than changing restart scheduling or broader config reload behavior.
