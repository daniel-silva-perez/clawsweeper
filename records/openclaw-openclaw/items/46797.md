## Summary

Keep open. Current main still routes `config.patch` restart notices through the generic `Gateway restart ... ok` sentinel summary, the latest release has the same formatter, and the existing tests still assert the old completed-looking wording rather than the requested restart-required wording.

## What This Changes

The issue asks for `config.patch` restart notices to say that a gateway restart is required instead of implying that a restart already completed.

## Best Possible Solution

Change the restart-sentinel summary for successful `config.patch` restart-required notices to explicit required wording such as `Gateway restart required (config.patch)`, while preserving the note, distinct reason, doctor hint, and existing summaries for update, explicit restart, config.apply, and auto-recovery cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `config.patch` writes a `config-patch` sentinel with `stats.mode: "config.patch"`, startup formats it through `formatRestartSentinelMessage`, and the summary currently renders as `Gateway restart config-patch ok (config.patch)`.

## Solution Assessment

Yes. A targeted formatter or sentinel-payload wording change is the narrowest maintainable fix; changing reload planning, Windows restart mechanics, or restart scheduling would be broader than the reported UX bug.
