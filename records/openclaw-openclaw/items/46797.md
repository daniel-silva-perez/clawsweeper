## Summary

Current main has not implemented the requested wording change. `config.patch` still creates a `config-patch` restart sentinel with `status: "ok"`, and the user-facing formatter still renders that as `Gateway restart config-patch ok (config.patch)` before appending the note text. Related config.patch cleanup appears to address restart scheduling/hot-reload behavior, not this notice copy.

## Best Possible Solution

Keep this issue open. The best fix is in the gateway restart-sentinel/config-write path: encode whether a config.patch restart is required, scheduled, completed, or skipped, then format config.patch notices with clear copy such as `Gateway restart required (config.patch)` while preserving the note underneath. Add focused coverage in `src/infra/restart-sentinel.test.ts` and config.patch restart-write tests.
