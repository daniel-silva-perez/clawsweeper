## Summary

Keep open. Current main still routes config.patch restart sentinels through the generic summary formatter, which produces the reported completed-looking `Gateway restart config-patch ok (config.patch)` shape instead of a restart-required notice.

## What This Changes

The issue asks for config.patch restart notices to clearly say a gateway restart is required while keeping the submitted note text below the summary.

## Best Possible Solution

Implement a focused restart-sentinel UX fix so config.patch restart notices use wording like `Gateway restart required (config.patch)`, preserve the submitted note beneath it, and keep unrelated restart/update/auto-recovery summaries unchanged.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a config.patch write creates a `kind: "config-patch"`, `stats.mode: "config.patch"` sentinel, which current startup delivery formats through the old generic restart summary.

## Solution Assessment

Yes. The narrowest maintainable solution is a targeted formatter or config-write sentinel payload change backed by formatter coverage, rather than changing restart scheduling or broader config reload behavior.
