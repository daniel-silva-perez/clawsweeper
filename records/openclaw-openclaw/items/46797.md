## Summary

Keep open. Current main still formats `config.patch` restart sentinels through the generic completed-looking `Gateway restart config-patch ok (config.patch)` summary, and no current source or test implements the requested restart-required wording.

## What This Changes

Request that `config.patch` restart notices say a gateway restart is required while preserving the note text underneath.

## Best Possible Solution

Change the restart-sentinel formatter or `config.patch` sentinel payload so `config.patch` restart-required notices use explicit required wording, with tests preserving the note/reason/doctor lines and keeping unrelated update, explicit restart, and auto-recovery summaries stable.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `config.patch` writes a `config-patch`/`config.patch` sentinel, startup delivers `formatRestartSentinelMessage(payload)`, and the formatter builds the reported generic summary.

## Solution Assessment

Yes. A targeted formatter or sentinel-payload adjustment is the narrowest maintainable fix; changing reload planning, Windows restart mechanics, or restart scheduling would be broader than the reported wording bug.
