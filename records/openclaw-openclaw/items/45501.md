## Summary

Keep open. Current main still builds the bare `/new` and `/reset` startup text from a hardcoded `BARE_SESSION_RESET_PROMPT_BASE`, and the config/types/schema/docs do not expose `session.resetPrompt` or a per-agent equivalent. The related discussion is useful and correctly scopes this as prompt configurability, not a reset archive feature.

## Best Possible Solution

Keep this issue open and implement it narrowly in core if maintainers accept the product shape: add a typed `session.resetPrompt?: string` global config, decide whether per-agent support belongs under `agents.list[].session` or an agent defaults surface, resolve precedence in `buildBareSessionResetPrompt`/`resolveBareSessionResetPromptState`, update the zod/generated schema and public config docs, and add regression tests for fallback, global override, per-agent override, timestamp handling, startup-context interaction, and bootstrap-pending behavior.
