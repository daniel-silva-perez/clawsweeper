## Summary

Current main still has no `session.resetPrompt` or per-agent reset-prompt override. Bare chat/channel `/new` and `/reset` now acknowledge without invoking the model in some paths, but model-facing reset prompts remain hardcoded for agent RPC and reply-producing reset flows, and tests still assert the stock wording.

## Best Possible Solution

Keep this issue open as the canonical narrow feature request. Implement `session.resetPrompt` as a full replacement for the normal hardcoded reset prompt with fallback to current behavior, add `agents.list[].session.resetPrompt` precedence for per-agent overrides, preserve bootstrap-specific reset prompts, and keep reset archival/backup behavior out of scope. Related #51586 should be linked or closed as the duplicate after maintainers confirm the final config shape.
