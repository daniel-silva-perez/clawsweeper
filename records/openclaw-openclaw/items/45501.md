## Summary

Keep open. Current main and the latest release still use the stock bare reset startup prompt and expose no root or per-agent config key for replacing it; the issue discussion includes concrete Ollama/gemma4 impact tied to the exact prompt wording, and overlapping reset-greeting or multi-session issues do not supersede this narrower request.

## What This Changes

Add a configurable `session.resetPrompt` override, globally and per agent, for the startup message used after bare `/new` or `/reset` while preserving the current default when unset.

## Best Possible Solution

Keep this as the canonical narrow prompt-override feature: implement a full replacement string with fallback to the current default, support the requested root and per-agent precedence, wire both gateway and reply-producing reset paths through one helper, and document that reset backups, greeting-model selection, suppression, and multi-session architecture are separate work.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: send a bare `/new` or `/reset` through a reply-producing or gateway agent path and current main reaches `buildBareSessionResetPrompt()`, while strict root and per-agent schemas reject the requested override shape.

## Solution Assessment

Yes, if maintainers accept this public key. A full replacement string with current behavior as fallback is the narrowest maintainable fix for the reported wording problem; bootstrap safety semantics should be explicit rather than folded into unrelated reset-backup or greeting-model work.
