## Summary

Keep open. Current main still has no `session.resetPrompt` or equivalent config surface, and the model-backed bare `/new`/`/reset` startup paths still build their prompt from the hardcoded reset prompt text. The discussion also contains a concrete Ollama/gemma4 tool-call-loop report tied to the current wording, while the existing reset archive and hook mechanisms mean the broader backup idea should stay separate.

## What This Changes

Add a root and per-agent `session.resetPrompt` config override for model-facing bare `/new` and `/reset` startup prompts, falling back to the current prompt when unset.

## Best Possible Solution

Keep this as the canonical narrow config feature. Add a full replacement prompt string with fallback to the existing default, support root and per-agent precedence, wire it into the model-facing reset prompt builder used by gateway and auto-reply reset paths, and update schema/help/generated config docs/tests/changelog without expanding into reset backup or greeting-model policy.

## Reproduction Assessment

Yes. Current-main source and tests provide a high-confidence reproduction path: model-backed reset startup turns still use the hardcoded prompt, and strict config schemas currently reject the requested root and per-agent `resetPrompt` shape.

## Solution Assessment

Yes, if maintainers accept the proposed public config shape. A full replacement string with fallback is the narrowest maintainable fix for the reported wording problem; pre-reset backups and configurable greeting model/suppression belong in separate work such as #12931.

## Repair Work Prompt

Implement the narrow configurable reset startup prompt requested by https://github.com/openclaw/openclaw/issues/45501. Add a public `session.resetPrompt?: string` root config key and a per-agent override matching the issue's `agents.list[].session.resetPrompt` shape, with per-agent taking precedence over root and both falling back to the existing `BARE_SESSION_RESET_PROMPT_BASE` behavior when unset. Wire the selected full replacement string into the normal non-bootstrap model-facing bare `/new` and `/reset` prompt builder used by `src/auto-reply/reply/session-reset-prompt.ts`, `src/gateway/server-methods/agent.ts`, and `src/auto-reply/reply/get-reply-run.ts`; keep timestamp appending behavior. Preserve existing bootstrap-pending and limited-bootstrap safety wording unless maintainers explicitly decide `resetPrompt` should override those modes. Update config schemas/types/help/labels/generated config docs as required, add focused tests for root fallback, per-agent precedence, gateway path, and auto-reply path, and add a user-facing changelog entry crediting the issue reporter/commenters.
