## Summary

Keep open. Current main still hardcodes the bare `/new` and `/reset` startup prompt and has no root or per-agent `session.resetPrompt` config surface; the issue discussion adds concrete Ollama/gemma4 impact tied to the exact wording, while the related greeting-model and multi-session RFCs are adjacent rather than superseding this narrower request.

## What This Changes

Add a global and per-agent `session.resetPrompt` config key that can replace the default bare `/new` and `/reset` startup message while preserving the current default when unset.

## Best Possible Solution

Keep this as the canonical narrow config feature: add a full replacement prompt string with fallback to the existing default, support root plus per-agent precedence, wire it into both gateway and auto-reply bare reset/new prompt construction, and update schema/help/generated config docs/tests/changelog without expanding into reset backup or greeting-model policy.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: send bare `/new` or `/reset` without a trailing message and current main reaches `buildBareSessionResetPrompt()`, while strict root and per-agent config schemas reject the requested override shape.

## Solution Assessment

Yes. If maintainers accept the public config key, a full replacement string with current behavior as fallback is the narrowest maintainable fix for the reported wording problem; reset backups and configurable greeting model/suppression should remain separate work.
