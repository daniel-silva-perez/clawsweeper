## Summary

Keep this PR open. Current main still has the Discord security-audit false positive: the audit ignores top-level `channels.discord.allowFrom` when computing `ownerAllowFromConfigured`, while runtime authorization, docs, and config types all treat that top-level path as valid. This PR is the focused implementation candidate for #39341, which remains open; #67222 was closed as a duplicate, not as implemented.

## Best Possible Solution

Keep this PR open and review or land it as the focused fix for #39341. The best code path is to update `extensions/discord/src/security-audit.ts` so `ownerAllowFromConfigured` normalizes top-level `discordCfg.allowFrom` together with legacy `dm.allowFrom` and the pairing-store allowlist, keep the added regression test for top-level `allowFrom`, and retain the changelog attribution already present in the PR diff.
