## Summary

Keep this issue open. Current main implements the GPU half through `sandbox.docker.gpus`, including schema, config resolution, docs, and Docker `--gpus` create args, but it still has no `sandbox.docker.ipc`, `--ipc=host`, or reviewed raw Docker argument passthrough. The remaining request is a sandbox namespace/isolation policy decision, so it needs maintainer/security review rather than cleanup closure.

## Best Possible Solution

Keep this open for maintainer/security review. Treat `sandbox.docker.gpus` as implemented on current main, then decide whether the remaining need should become a narrow typed and guarded `sandbox.docker.ipc` setting or be rejected as out of scope. Avoid broad raw Docker args unless maintainers explicitly accept the isolation, validation, docs, audit, and support tradeoffs.
