## Summary

Keep this issue open. Current main has added the GPU half through `sandbox.docker.gpus`, but it still does not expose `--ipc=host`, a typed `sandbox.docker.ipc` setting, or a reviewed raw Docker argument passthrough. Because host IPC and arbitrary Docker flags materially weaken sandbox isolation, the remaining request needs maintainer/security product judgment rather than conservative cleanup closure.

## Best Possible Solution

Keep this issue open for maintainer/security review. Treat `sandbox.docker.gpus` as the already-implemented GPU portion, then decide whether the remaining request should become a narrow, typed, guarded `sandbox.docker.ipc` policy or be rejected as out of scope. Avoid broad raw Docker args unless maintainers explicitly accept the validation, config-hash, docs, auditability, and sandbox-isolation tradeoffs.
