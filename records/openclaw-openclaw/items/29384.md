## Summary

Keep open. Current main now implements the Docker GPU half through `sandbox.docker.gpus`, but it still does not implement host IPC (`--ipc=host`), raw Docker create-arg passthrough, `devices`, `privileged`, or `capAdd`. The remaining request is real but security-sensitive, so it needs maintainer product/security review rather than conservative cleanup closure.

## Best Possible Solution

Keep this issue open and have maintainers decide the supported Docker sandbox contract. The safest product path is to treat GPU support as already handled on main, then either split/retitle the remaining work to an explicit, guarded `sandbox.docker.ipc` policy or close that part by policy if host IPC is out of scope. Avoid broad raw Docker args unless maintainers explicitly accept the validation, audit, config-hash, and sandbox-isolation tradeoffs; any accepted change should update public types, strict Zod schema, generated schema/help/docs, resolver/config-hash behavior, runtime validation, Docker create args, and focused tests.
