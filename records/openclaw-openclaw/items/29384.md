## Summary

Keep this issue open. Current main implements the GPU portion with `sandbox.docker.gpus`, but it still does not expose `--ipc=host`, a typed `sandbox.docker.ipc`, or a reviewed raw Docker argument passthrough. The remaining request affects Docker sandbox isolation policy, so it needs maintainer/security review rather than conservative cleanup closure.

## Best Possible Solution

Keep this issue open for maintainer/security review. Treat `sandbox.docker.gpus` as already implemented, then decide whether the remaining request should become a narrow typed and guarded `sandbox.docker.ipc` setting or be rejected as out of scope. Avoid broad raw Docker args unless maintainers explicitly accept the isolation, validation, docs, and auditability tradeoffs.
