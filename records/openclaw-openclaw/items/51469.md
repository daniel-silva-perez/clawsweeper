## Summary

Current main still does not implement the remaining request. The pre-auth WebSocket handshake default is 10 seconds, the server handshake timer still resolves from environment only, `gateway.handshakeTimeoutMs` is not part of the gateway config type or strict config schema, and the related implementation PR #51503 is closed unmerged. Recent startup-sidecar work reduces adjacent timeout symptoms, but it does not prove this cold-start handshake case is fixed or make the requested configurable timeout obsolete.

## Best Possible Solution

Keep this issue open. Maintainers should either land a scoped fix along the #51503 direction, with bounded validation and docs for `gateway.handshakeTimeoutMs`, or explicitly decide from reproducible current-main timing evidence that the existing 10s default plus `OPENCLAW_HANDSHAKE_TIMEOUT_MS` and startup-import slimming are sufficient.
