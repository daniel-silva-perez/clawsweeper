## Summary

Keep open. Current main still lacks an `agent_error` or equivalent lifecycle-error replacement hook, so the PR is not obsolete, but the proposed core plugin API has blocking review issues around conversation-access policy and timeout semantics before it should merge.

## What This Changes

The PR adds an `agent_error` plugin hook, invokes it from embedded Pi agent error lifecycle handling with a 2s outer timeout, adds lifecycle tests, and adds a changelog entry.

## Best Possible Solution

Keep the PR as an active core plugin API proposal, but require the new hook to follow the existing plugin access policy and timeout contract, document the public raw/processed/redacted semantics, add loader/runner contract coverage, fix changelog attribution, and rebase onto current hook-contract main before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate agent_error behind conversation access:** `src/plugins/hook-types.ts:68`
  - body: Adding `agent_error` to the public hook list without adding it to `CONVERSATION_HOOK_NAMES` lets non-bundled plugins register it without `plugins.entries.<id>.hooks.allowConversationAccess=true`, because the registry only applies that opt-in to names in that guarded list. The lifecycle call passes provider error text before observation redaction, so this creates a new unguarded error-content surface. Please add the hook to the guarded list and extend the loader policy tests.
  - confidence: 0.91
- **[P2] Honor per-handler timeoutMs in agent_error:** `src/plugins/hooks.ts:690-695`
  - body: This custom loop awaits each hook handler directly, so `api.on("agent_error", handler, { timeoutMs })` is ignored. A hung high-priority handler can prevent later handlers from producing a replacement until the outer lifecycle timeout discards all replacements, which contradicts the documented per-hook timeout contract. Use the existing timeout helper per registration or adapt `runModifyingHook` while preserving accumulated-error semantics.
  - confidence: 0.87
