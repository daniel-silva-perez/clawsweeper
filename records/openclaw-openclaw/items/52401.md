## Summary

Keep this PR open. Current main still has no `agent_error` hook or equivalent lifecycle-error replacement seam, but the proposed core plugin API change still needs maintainer review because it introduces an unguarded error-content hook and bypasses the documented per-handler timeout contract.

## What This Changes

The PR adds an `agent_error` plugin hook, invokes it from embedded Pi agent error lifecycle handling with a 2s outer timeout, adds lifecycle tests, and adds a changelog entry.

## Best Possible Solution

Keep the PR active as a core plugin API proposal, but require the new hook to follow the existing plugin access policy and timeout contract, document the processed/redacted error semantics, add loader/runner contract coverage, and fix changelog attribution before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Gate agent_error behind conversation access:** `src/plugins/hook-types.ts:68`
  - body: Adding `agent_error` to the public hook list without adding it to `CONVERSATION_HOOK_NAMES` lets non-bundled plugins register it without `plugins.entries.<id>.hooks.allowConversationAccess=true`, because the registry only applies that opt-in to guarded hook names. The lifecycle caller hands provider error text to the hook before observation redaction, so this creates a new unguarded error-content surface.
  - confidence: 0.91
- **[P2] Apply per-handler timeoutMs in agent_error:** `src/plugins/hooks.ts:687-692`
  - body: This custom loop awaits each `agent_error` handler directly, so `api.on("agent_error", handler, { timeoutMs })` is ignored. A hung high-priority handler can prevent later handlers from producing a replacement until the outer 2s lifecycle timeout discards all replacements, which violates the documented per-hook timeout contract.
  - confidence: 0.87
- **[P3] Add required changelog attribution:** `CHANGELOG.md:3136`
  - body: The added changelog bullet has no `Thanks @...` credit, but repo changelog rules require every added entry to include at least one credited GitHub username. Please add contributor attribution before merge.
  - confidence: 0.94
