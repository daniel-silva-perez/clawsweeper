## Summary

Keep this PR open. Current main still does not implement `agent_error` or an equivalent pre-broadcast lifecycle-error replacement hook, and the PR is an active, focused core plugin API proposal rather than obsolete or implemented.

## Best Possible Solution

Keep this PR open for maintainer review as a core plugin API proposal. If accepted, land it with an explicit processed/redacted error contract, conversation-access gating, lifecycle timeout behavior, focused lifecycle and plugin-contract tests, public `docs.openclaw.ai` hook documentation, and changelog attribution. ## Risks / Open Questions

- The PR adds `agent_error` to the hook list but the diff shown does not add it to `CONVERSATION_HOOK_NAMES`; maintainers should decide whether non-bundled plugins need `allowConversationAccess=true` before seeing lifecycle provider error text.
- The hook receives processed error text before `buildTextObservationFields` redaction in the proposed patch, so the contract should explicitly define whether handlers see raw, formatted, or redacted material.
- Even with the proposed 2s timeout, the hook remains on the terminal lifecycle error path; maintainer review should confirm the timing is acceptable for clients and compaction retry bookkeeping.
- The PR currently changes the changelog but does not update public plugin hook docs, and the changelog entry needs the repo-required contributor attribution before merge.
