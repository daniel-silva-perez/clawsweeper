## Summary

Keep this PR open. Current main still has a single preauth WebSocket handshake timer that is cleared only after async connect auth and registration prep, so main has not implemented the PR's intended valid-connect/auth-phase split. The PR is relevant and carries forward @sashakhar1's #49751 work, but the branch needs the posted security review finding fixed before merge: repeated preauth connect frames can re-arm the new connect-auth watchdog and defeat the bounded-auth guarantee.

## Best Possible Solution

Keep this PR open and fix the branch in place: make the connect-auth watchdog one-shot or add an explicit connecting state before the first auth await, reject or close duplicate preauth connect frames deterministically, clear both timers on all close/register paths, and add regression tests for stalled auth plus repeated valid connect frames. Preserve @sashakhar1's #49751 contribution credit and keep the changelog/docs aligned with the final behavior.
