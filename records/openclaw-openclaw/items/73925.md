## Summary

Keep this PR open. Current main still uses a single preauth WebSocket handshake timer and does not implement the PR's valid-connect/auth-phase split. The PR remains relevant as the replacement for #49751, but the branch has an unresolved security review finding: repeated valid preauth connect frames can reset the new connect-auth watchdog and defeat the bounded-auth guarantee.

## Best Possible Solution

Keep this PR open and repair the branch in place: make the connect-auth watchdog one-shot or set an explicit connecting state before the first auth await, reject or close duplicate preauth connect frames deterministically, keep both timers cleared on close/register paths, and add regression coverage for stalled auth plus repeated valid connect frames. Preserve @sashakhar1's #49751 contribution credit and update the changelog/docs to match the final behavior.
