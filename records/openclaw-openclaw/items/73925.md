## Summary

Keep open: the PR addresses a real Gateway WebSocket auth-timing gap, but the current branch still has a blocking resettable-watchdog flaw visible in the diff and independently flagged in review comments. It should not merge until duplicate preauth connect frames cannot extend the auth deadline.

## What This Changes

The PR branch adds a Gateway WebSocket connect-auth watchdog after valid connect parsing, clears it on registration/close, updates gateway protocol docs and the changelog, and adds gateway timeout/auth tests.

## Best Possible Solution

Repair the existing PR branch with a one-shot connect-auth watchdog or explicit connecting state set before the first auth await, reject or close duplicate preauth connect frames deterministically, keep timer cleanup on close and successful registration, and add regression coverage for repeated valid connect frames during stalled auth.

## Reproduction Assessment

Yes: send a valid connect frame that stalls in resolveConnectAuthState or resolveConnectAuthDecision, then send another valid preauth connect before the watchdog fires. Because no in-flight state is set and getClient remains null, the second frame can re-enter the connect path and rearm the watchdog.

## Solution Assessment

No: the two-phase timer direction is the right narrow design, but this implementation is not the best fix while the auth-phase deadline is resettable. The safer solution is a one-shot watchdog or explicit connecting state before any auth await, plus duplicate-connect regression coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Make the auth watchdog one-shot:** `src/gateway/server/ws-connection/message-handler.ts:426`
  - body: `armConnectAuthTimer()` clears and recreates the connect-auth timer while the socket still has no client during the auth awaits. A second valid preauth `connect` frame can re-enter this block and extend the timeout indefinitely, so the new bounded-auth guarantee is bypassed; reject duplicate connects or make the watchdog one-shot before merging.
