## Summary

Keep this PR open. Current main at 43da089 still has a single WebSocket preauth timer that is cleared only near client registration after async auth work, so main has not implemented the PR's intended valid-connect/auth-phase split. The PR is a relevant repair of #49751 and preserves @sashakhar1's contribution path, but the branch currently has a security-sensitive watchdog bypass: repeated valid connect frames can re-arm the new connect-auth timer before auth completes.

## Best Possible Solution

Keep this PR open and require a branch fix before merge: make connect-auth timing one-shot/non-resettable, prevent preauth handshake re-entry once a valid connect enters auth, close or reject duplicate connect frames deterministically, clear both timers on all close/success paths, retain isClosed guards after awaits, and add regression coverage proving duplicate connect frames cannot extend the auth deadline. Preserve @sashakhar1's #49751 contribution credit and keep the protocol docs/changelog aligned with the current 15 second configurable timeout policy.
