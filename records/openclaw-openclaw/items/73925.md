## Summary

Keep this PR open. Current main still uses a single preauth WebSocket timer that is only cleared near client registration after async auth, so main has not implemented the PR's intended valid-connect/auth-phase split. The replacement PR is relevant and carries forward @sashakhar1's #49751 work, but the posted Aisle review identifies a security-sensitive preauth DoS in the PR branch: repeated connect frames can reset the new connect-auth watchdog before auth completes.

## Best Possible Solution

Keep this PR open, but require a follow-up on the branch before merge: prevent preauth handshake re-entry once a connect frame has entered auth, make the connect-auth watchdog one-shot/non-resettable, close or reject duplicate connect frames deterministically, clear both timers on all close paths, and add regression tests for stalled auth plus repeated connect frames. Preserve @sashakhar1's #49751 contribution credit and keep the docs/changelog aligned with current main's 15 second configurable timeout policy.
