## Summary

Keep open. Current main still starts the classic `openclaw onboard` wizard without an upfront duration or long-running setup expectation, and the user-facing onboarding docs list the setup scope without telling beginners that provider, channel, daemon, skills, plugin, or network steps can take a while. The related open items are adjacent but do not supersede this narrower ETA/expectation request.

## What This Changes

Request to show an upfront ETA or long-running setup warning when `openclaw onboard` starts.

## Best Possible Solution

Add a concise early note to the classic CLI onboarding flow before the main prompts begin, using conservative wording such as setup can take several minutes or longer depending on selected providers, channels, daemon install, skills, plugins, and network conditions; mirror that expectation in the CLI onboarding docs without changing provider or runtime behavior.

## Reproduction Assessment

Yes. Source inspection gives a deterministic reproduction path: interactive `openclaw onboard` calls `setupWizardCommand` -> `runInteractiveSetup` -> `runSetupWizard`, and the first wizard output has no ETA or long-running setup note.

## Solution Assessment

Yes. A small UX/docs change is the narrowest maintainable solution; a fixed 40-minute promise would be less safe than a conservative duration expectation unless maintainers have timing data.
