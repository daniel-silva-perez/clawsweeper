## Summary

Keep open. Current main still starts the default classic `openclaw onboard` wizard without an upfront duration or long-running setup expectation, and the public onboarding docs still list the setup phases without timing guidance. The existing ClawSweeper comments already identified this as a narrow current-main UX/docs gap, and the current checkout does not show a post-report fix.

## What This Changes

Request to show an upfront ETA or long-running setup warning when `openclaw onboard` starts.

## Best Possible Solution

Ship a conservative early expectation note in the classic CLI wizard and mirror it in the onboarding docs, saying setup can take several minutes or longer depending on selected providers, channels, daemon install, skills/plugins, and network conditions, without changing runtime behavior or promising a fixed duration.

## Reproduction Assessment

Yes. Source inspection gives a deterministic path: default interactive `openclaw onboard` enters `runSetupWizard`, whose first user-visible outputs have no duration note before setup choices begin.

## Solution Assessment

Yes, with adjusted wording. A concise UX/docs expectation note is the narrowest maintainable fix; hard-coding a 40-minute ETA is not the best wording without timing data.
