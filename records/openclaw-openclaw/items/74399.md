## Summary

Keep open. Current main still starts the classic `openclaw onboard` flow without an upfront duration or long-running setup expectation, and the onboarding docs describe the setup scope without an ETA note. The request is narrow, current, and not superseded by the adjacent heads-up or slowdown reports.

## What This Changes

Request adding an upfront ETA or time-expectation notice to `openclaw onboard` so beginners know setup can take a long time before starting.

## Best Possible Solution

Add a concise early note to the classic CLI onboarding wizard before the main prompts, using conservative wording that setup can take several minutes or longer depending on selected providers, channels, daemon install, skills/plugins, and network conditions; mirror the same expectation in the public CLI onboarding docs without changing runtime behavior.

## Reproduction Assessment

Yes. Source inspection gives a deterministic reproduction path: non-modern `openclaw onboard` enters `runSetupWizard`, whose first user-visible outputs have no duration note, while the docs list the setup phases without time guidance.

## Solution Assessment

Yes. A small UX/docs change is the narrowest maintainable solution; a fixed 40-minute promise should be avoided unless maintainers have timing data.
