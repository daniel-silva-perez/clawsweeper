## Summary

Keep open. Current main still lacks an upfront duration expectation in the classic `openclaw onboard` path, and the onboarding docs list the setup steps without warning that provider auth, channels, daemon install, search, skills, and optional dependencies can take a while. The prior discussion already identified the same gap, and the current checkout still matches that finding.

## What This Changes

Request to add an upfront duration or ETA notice to `openclaw onboard` so new users know onboarding may take a long time.

## Best Possible Solution

Add a concise early note in the classic CLI wizard before the main prompts, explaining that full setup can take several minutes and longer when channel auth, daemon install, skills, or optional dependencies are selected; mirror the same expectation in the public onboarding docs.

## Reproduction Assessment

Not applicable as a bug reproduction. We do have a high-confidence current-main verification path: inspect the wizard entry and onboarding docs, which still show no upfront ETA or long-running setup copy.

## Solution Assessment

Yes, a narrow UX/docs update is the best fit. The safer wording is a conservative duration expectation rather than a literal 40-minute promise unless maintainers have timing data.
