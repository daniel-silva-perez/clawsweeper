## Summary

Keep open. Current main still does not show an upfront duration or long-running setup expectation when the classic `openclaw onboard` flow starts, and the public onboarding docs do not cover that expectation in the main CLI onboarding pages. The request is narrow, current, and appropriate for a focused UX/docs repair rather than closure.

## What This Changes

Request to add an upfront ETA or long-running setup notice to `openclaw onboard` so new users know onboarding can take a while.

## Best Possible Solution

Add a concise early notice to the classic CLI onboarding flow before the main prompts begin, saying setup can take several minutes or longer depending on provider auth, channel setup, daemon install, skills, plugins, and network conditions; mirror that expectation in the CLI onboarding docs and avoid changing provider/runtime behavior.

## Reproduction Assessment

Yes. The source path deterministically starts `openclaw onboard` through `setupWizardCommand` -> `runInteractiveSetup` -> `runSetupWizard`, where the first visible wizard actions contain no ETA or long-running setup note before prompts begin.

## Solution Assessment

Yes, a small UX/docs change is the narrowest maintainable direction. The safer wording is a conservative duration expectation rather than a fixed 40-minute promise unless maintainers have timing data.
