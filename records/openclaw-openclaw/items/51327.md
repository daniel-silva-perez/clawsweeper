## Summary

Keep open. Current main still does not implement this PR's central behavior: after `agents.delete` with `deleteFiles=false`, preserved transcripts can remain on disk, but `sessions.usage` only walks configured gateway agents and does not scan disk-backed deleted-agent transcript directories.

## Best Possible Solution

Keep this PR open, or replace it with a maintainer-owned successor that adds a validated read-only usage discovery target for preserved deleted-agent transcript directories. The fix should preserve the real sessions directory, use async/non-fatal filesystem discovery, keep deleted-agent chat/session mutation guards intact, and add regression coverage for `agents.delete` with `deleteFiles=false` across `sessions.usage`, specific-key lookup, timeseries, and logs.
