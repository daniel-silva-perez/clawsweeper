## Summary

Keep open. Current main still scopes `sessions.usage` discovery to gateway-visible configured agents, so preserved transcript directories for deleted agents remain missed after `agents.delete` with `deleteFiles=false`. This PR is the active implementation candidate for the linked gateway usage bug, and the current head addresses the earlier review concerns by using async disk discovery, carrying the real `sessionsDir`, adding keyed lookup fallback, and covering non-round-tripping deleted-agent directories.

## What This Changes

The PR adds usage-only discovery of disk-backed agent session directories, threads explicit `sessionsDir` values into transcript discovery and keyed usage lookups, and expands `sessions.usage` regression coverage for deleted-agent history, non-round-tripping directories, logs, timeseries, and scan-error handling.

## Best Possible Solution

Land this PR or an equivalent maintainer-owned successor after accepting the usage-only disk target enumeration design, keeping normal agent visibility unchanged while preserved transcript paths flow through list, keyed detail, timeseries, and logs.
