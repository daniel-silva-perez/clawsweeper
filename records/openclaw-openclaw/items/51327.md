## Summary

Keep open: current main still drops preserved disk-only agent transcripts from gateway usage after an agent is deleted with deleteFiles=false, and this PR targets that valid bug. The patch looks directionally correct, but the open PR should receive maintainer review and CI resolution rather than be closed or replaced.

## What This Changes

The PR adds usage-only discovery of disk-backed agent session directories, passes explicit sessionsDir values into transcript discovery, and expands sessions.usage regression tests for deleted-agent history and keyed lookups.

## Best Possible Solution

Land this PR or an equivalent maintainer-owned successor after accepting the usage-only disk target enumeration design, keeping normal agent-list visibility configured-agent scoped while preserved transcript paths flow through list, keyed detail, timeseries, and log usage surfaces.
