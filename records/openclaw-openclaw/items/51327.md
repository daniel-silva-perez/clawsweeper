## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: preserved transcript files for an agent deleted with `deleteFiles=false` are not discovered by `sessions.usage` once that agent is removed from gateway-visible configuration and its session-store rows are purged.

## Best Possible Solution

Keep this PR open, or replace it with a maintainer-owned successor that adds a validated read-only usage discovery target for preserved deleted-agent transcript directories. The implementation should keep `listAgentsForGateway()` config-scoped, pass the real `sessionsDir` through list/specific-key/timeseries/log usage paths, tolerate expected filesystem errors, and add regression coverage for `agents.delete` with `deleteFiles=false`.
