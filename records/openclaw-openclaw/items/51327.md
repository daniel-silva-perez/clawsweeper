## Summary

Keep open. Current main still scopes `sessions.usage` discovery to gateway-visible agents, so preserved transcript directories for deleted agents remain missed when `agents.list` is explicit. The PR is the active implementation candidate and its latest head addresses the earlier automated review concerns by using async disk discovery, carrying the real `sessionsDir`, adding keyed lookup fallback, and expanding regression coverage; it still needs maintainer review and CI triage before merge.

## What This Changes

The PR adds usage-only discovery of disk-backed agent session directories, passes explicit `sessionsDir` values into transcript discovery and keyed usage lookups, and extends `sessions.usage` regression tests for deleted-agent history, non-round-tripping directories, logs, timeseries, and scan-error handling.

## Best Possible Solution

Land this PR or an equivalent maintainer-owned successor after accepting the usage-only disk target enumeration design, preserving normal agent visibility while retained transcript paths flow through list, keyed detail, timeseries, and log usage surfaces, and clearing or explaining the exact-head CI failures.
