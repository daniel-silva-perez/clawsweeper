## Summary

Keep this PR open. Current main still does not implement the PR’s intended usage-only discovery path for preserved deleted-agent transcript directories: `sessions.usage` only walks `listAgentsForGateway()` agents, while `agents.delete` with `deleteFiles=false` leaves transcript files on disk but removes the deleted agent from config and purges its session-store rows.

## Best Possible Solution

Keep this PR open, or replace it with a maintainer-owned successor that adds a validated read-only usage discovery target for preserved deleted-agent transcript directories. The fix should keep `listAgentsForGateway()` config-scoped, pass the real `sessionsDir` through list, specific-key, timeseries, and logs usage paths, tolerate expected filesystem errors, and add regression coverage for `agents.delete` with `deleteFiles=false`.
