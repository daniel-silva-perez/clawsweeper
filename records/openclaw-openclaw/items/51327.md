## Summary

Keep open. Current main still misses the deleted-agent transcript usage path: usage discovery is configured-agent scoped, detail/log lookups rebuild paths from agent IDs, and agent deletion can remove config/store visibility while leaving transcript files behind when files are preserved.

## What This Changes

This PR adds usage-only disk discovery for preserved `agents/<id>/sessions` directories, lets transcript discovery accept an explicit sessions directory, and adds regression tests for deleted-agent usage history.

## Best Possible Solution

Land this PR or a maintainer-owned successor once the disk-backed usage target enumeration is accepted, the real transcript path is preserved through list/key/timeseries/log paths, normal agent-list visibility remains unchanged, and filesystem edge cases are covered.
