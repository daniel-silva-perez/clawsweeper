## Summary

Keep open. Current main still lacks the requested llama.cpp runtime context discovery, but this PR is not merge-ready because it reads `data.n_ctx` while current llama.cpp exposes the runtime cap as `default_generation_settings.n_ctx`, so the intended fix would not work against the upstream response shape.

## What This Changes

The PR adds opportunistic llama.cpp `/props` discovery to self-hosted OpenAI-compatible model discovery, preserves explicit configured context windows, maps `/v1/models` `meta.n_ctx_train` to `contextWindow`, and adds focused provider setup tests.

## Best Possible Solution

Revise this PR so the provider setup parser reads the current llama.cpp `/props` shape, keeps the `contextWindow` and `contextTokens` split, preserves explicit configured caps, updates tests to match upstream, and adds the required changelog entry before maintainer review/merge.

## Reproduction Assessment

Yes. A static reproduction is enough: compare PR head line 114 with current llama.cpp source/docs, where `n_ctx` is nested under `default_generation_settings`; a real current server would leave `runtimeContextTokens` undefined.

## Solution Assessment

No. The direction is narrow and appropriate, but the implementation should parse `default_generation_settings.n_ctx` and cover that response shape in tests; optionally keeping a top-level fallback is fine if older llama.cpp builds need it.
