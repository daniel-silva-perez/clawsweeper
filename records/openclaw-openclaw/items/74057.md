## Summary

Current main still does not implement this PR's intended llama.cpp runtime context discovery. Self-hosted OpenAI-compatible discovery only calls `<baseUrl>/models`, ignores `meta.n_ctx_train`, never calls `/props`, and session rows still resolve context from stored/transcript/config/discovery-cache data without live `/props` lookup. This PR remains the active implementation path for linked #73664 and needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to validate or revise the guarded llama.cpp `/props.n_ctx` discovery so runtime context becomes `contextTokens`, `n_ctx_train` remains native `contextWindow` metadata when useful, explicit configured caps stay authoritative, and the Sessions denominator reflects the actual loaded runtime cap. If accepted, land it with the focused provider setup tests and core type checks; if rejected, close it only after another merged fix covers #73664.
