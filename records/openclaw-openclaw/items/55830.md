## Summary

Keep open for maintainer judgment: current main overlaps by logging loaded plugin IDs in the gateway ready line and warning on plugin initialization failures, but it does not emit this PR's per-plugin `[plugins] load:` lines, disabled-plugin status, or capability counts. The Greptile-reported defects were addressed in the PR's second commit, so this is now a product/log-verbosity decision rather than a cleanup-close candidate.

## What This Changes

The PR adds a `logPluginLoadSummary` helper in `src/gateway/server-plugin-bootstrap.ts` and calls it after plugin diagnostics to emit per-plugin load status, capability counts, and a loaded/disabled/error summary.

## Best Possible Solution

Decide whether the extra per-plugin startup/reload lines are desirable now that current main has a compact ready-line summary and failure warnings. If yes, rebase the PR onto current main and add focused coverage for loaded, disabled, errored, and `logDiagnostics: false` cases; if no, close it as superseded by the current startup observability path.
