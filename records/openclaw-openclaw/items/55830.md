## Summary

Keep open. Current main has a compact gateway startup line that lists loaded plugin IDs, plus CLI/doctor plugin status surfaces, but it does not implement this PR's requested per-plugin gateway log lines or consolidated loaded/disabled/errored summary on startup and config reload. The provided review context shows the PR's first-round defects were addressed in a follow-up commit, so the useful next step is maintainer review or a rebase against current startup logging rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review or contributor refresh. The best path is a rebased implementation in the gateway plugin bootstrap/startup logging surface that adds per-plugin loaded/disabled/error summary logs, respects `logDiagnostics: false`, avoids overlap with the newer compact startup line, and adds focused tests for startup, config reload, and deferred-reload suppression. If maintainers decide the compact startup line plus CLI/doctor surfaces are enough, close with that explicit product decision rather than as already implemented.
