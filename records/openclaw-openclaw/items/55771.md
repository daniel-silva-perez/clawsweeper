## Summary

Keep this issue open. Current main still treats `before_model_resolve` as an override-only hook: the public result type has no `block`/`blockReason`, the hook merger has no terminal block semantics, and the model-selection path cannot return a blocked outcome before provider dispatch. The reported budget-guard use case remains a real core plugin API gap rather than ClawHub-only plugin work.

## Best Possible Solution

Add an additive core plugin API change: extend `PluginHookBeforeModelResolveResult` with `block?: boolean` and `blockReason?: string`, merge it with terminal sticky-block semantics matching `before_tool_call`, propagate a blocked model-resolution outcome through `resolveHookModelSelection` and the embedded/CLI runner paths before `resolveModelAsync`, surface the reason to the user, and update hook merger tests, model-selection tests, public SDK types, and plugin docs.
