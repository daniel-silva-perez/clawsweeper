## Summary

Keep open. Current main has Telegram group/supergroup ingress, but it still does not implement live Telegram group discovery and only migrates explicitly configured old group IDs. The reporter's reopened migration log matches the current `migration logged only` path, so this is not safe to close as implemented.

## What This Changes

The issue reports that Telegram group chats are not discovered or handled after group-to-supergroup migration even though direct Telegram messages work.

## Best Possible Solution

Define Telegram's intended post-migration behavior explicitly: keep directory groups config-backed and improve status/docs/doctor guidance for the new `-100...` ID, or add a read-only observed-groups surface that never grants access by itself. Runtime changes should preserve group allowlist and sender-authorization boundaries and add targeted migration/directory coverage.
