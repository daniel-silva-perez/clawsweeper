## Summary

Keep open. Current main has Telegram group/supergroup ingress and exact configured-ID migration, but the reported migration branch for an unconfigured old group ID still logs a no-op, while group directory output remains config-backed rather than live-discovered. The recent reporter and second-user comments make this too concrete to close as stale, fixed, or setup-only.

## What This Changes

Telegram groups should be discovered and should respond to normal group messages after bot setup, including group-to-supergroup ID migration, while direct Telegram messages already work.

## Best Possible Solution

Define the intended Telegram behavior for newly observed and migrated groups. Either keep directories config-backed and improve diagnostics/docs/doctor guidance for the new `-100...` ID, or add a read-only observed-groups surface that does not grant access by itself. Any runtime change should preserve group allowlist and sender authorization boundaries while adding focused coverage for migration, wildcard config, directory output, and group ingress.
