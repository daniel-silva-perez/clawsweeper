## Summary

Keep open. Current main has model failover plus scoped controls for defaults, sessions, cron jobs, and subagents, but it does not provide one safe operation that retargets all persisted and active model selections. The requested capability is valid, but broad enough that maintainers should define the product and safety rules before implementation.

## What This Changes

Request a single command to move the default/current Telegram session, subagents, cron jobs, and other persisted model selections to another configured provider/model when rate limited.

## Best Possible Solution

Design an owner-only bulk model migration flow that previews every proposed change, supports explicit scopes, preserves intentional overrides by default, and can update configured defaults plus selected active/persisted session and job records only when the operator confirms those scopes.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection: current main exposes scoped model controls and failover behavior, but no single bulk model migration command or API.

## Solution Assessment

Unclear as phrased. A dry-run-capable owner command with explicit scopes would be the maintainable direction; an unqualified `/moveall` that silently rewrites every model reference would be too risky.
