## Summary

Keep open. Current main has fallback handling and scoped model controls for defaults, individual chat sessions, queued followups, cron jobs, and subagent runs, but it still lacks one safe owner operation that retargets all persisted and active model selections across those surfaces.

## What This Changes

Request one command, such as `/moveall`, to switch the main agent, subagents, cron jobs, other tasks, and the current Telegram context to a selected configured provider/model when a provider is rate-limited.

## Best Possible Solution

Design an owner-only bulk model migration flow that validates the target provider/model, previews every affected config/session/cron/subagent change, supports explicit scopes, and preserves deliberate per-agent, per-session, cron, fallback, media-model, ACP, and auth-profile overrides unless the operator opts into changing them.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection: current main exposes scoped model controls and fallback behavior, but no single bulk model migration command or API.

## Solution Assessment

Unclear as phrased. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unqualified `/moveall` that silently rewrites every model reference would be too risky.
