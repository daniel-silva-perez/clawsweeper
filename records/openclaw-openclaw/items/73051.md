## Summary

Keep open. Current main has scoped model controls for defaults, the active chat session, queued followups, cron jobs, and subagents, but it still does not provide one safe bulk retarget operation across all persisted and active model selections.

## What This Changes

Request a command like `/moveall` that changes every relevant OpenClaw model selection to a different configured provider/model during provider rate limits.

## Best Possible Solution

Design an owner-only bulk model migration flow under the model-management surface with target validation, dry-run preview, explicit scopes for defaults/sessions/cron/subagents/fallbacks/auth profiles, and override-preservation defaults.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection: current main exposes scoped model controls but no single bulk retarget command or API.

## Solution Assessment

Unclear as proposed. A dry-run-capable, scoped owner command is the maintainable direction; an unqualified `/moveall` that rewrites every model reference would be too risky.
