## Summary

Keep open. Current main still has scoped model controls for defaults, per-session chat, gateway session patches, cron jobs, and subagent spawn paths, but no single owner-safe operation that retargets all persisted and active model selections; the prior ClawSweeper comment reached the same conclusion and no implementation is linked in the issue context.

## What This Changes

The issue requests one command or flow to move OpenClaw’s default, current chat, subagent, cron, task, and other persisted model selections to another configured provider/model during provider rate limits.

## Best Possible Solution

Design an owner-only bulk model migration flow under the model command surface that validates the target provider/model, shows a dry-run preview, supports explicit scopes for defaults/sessions/cron/subagents/fallbacks/media/auth profiles, and preserves deliberate overrides unless the operator opts into changing them.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection: current main exposes scoped model controls and fallback behavior, but no single bulk model migration command or API.

## Solution Assessment

Unclear as phrased. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unqualified `/moveall` that silently rewrites every model reference would be too risky.
