## Summary

Keep open. Current main has model failover plus scoped model controls for defaults, sessions, cron jobs, and subagents, but it does not expose one safe operation that retargets all persisted and active model selections; the requested behavior needs maintainer product design before implementation.

## What This Changes

The issue requests a single command, such as `/moveall`, that switches all agents, subagents, cron jobs, tasks, and the active channel context to another configured provider/model during provider rate limits.

## Best Possible Solution

Design a core owner-only bulk model migration flow that enumerates proposed changes before applying them, supports explicit scopes, and preserves deliberate per-agent, subagent, session, cron, fallback, and media-model overrides unless the operator opts in to changing them.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection: current main exposes scoped model controls, but no single bulk model migration command or API.

## Solution Assessment

Unclear as phrased. A dry-run-capable owner command with explicit scopes is a maintainable direction; an unqualified `/moveall` that silently rewrites every model reference would be too risky.
