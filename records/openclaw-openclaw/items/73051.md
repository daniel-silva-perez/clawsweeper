## Summary

Keep open. Current main has fallback handling and scoped model controls for defaults, individual sessions, queued followups, cron jobs, and subagents, but it still lacks one safe operation that retargets all persisted and active model selections across those surfaces; the existing ClawSweeper discussion reached the same keep-open conclusion and did not link an implementation PR or workaround.

## What This Changes

The issue requests one command to retarget all configured and persisted OpenClaw model selections to another provider/model when a provider is rate limited.

## Best Possible Solution

Design an owner-only bulk model migration flow that validates the target provider/model, previews every affected config/session/cron/subagent change, supports explicit scopes, and preserves deliberate per-agent, per-session, cron, fallback, media-model, and auth-profile overrides unless the operator opts in to changing them.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection: current main exposes scoped model controls and fallback behavior, but no single bulk model migration command or API.

## Solution Assessment

Unclear as phrased. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unqualified /moveall that silently rewrites every model reference would be too risky.

## Risks / Open Questions

- The intended scope of “everything” is still a product decision: defaults, per-agent overrides, session overrides, active runs, cron payload models, fallback lists, image/media models, auth-profile choices, and channel-bound contexts should not all necessarily be rewritten by default.
- A bulk mutation surface could surprise users or break security expectations unless it is owner-only, validates the target model/provider, previews changes, and preserves deliberate overrides unless explicitly scoped in.
- Active-run behavior needs design: live switches, queued followups, cron retries, and subagent sessions have different persistence and retry semantics today.
