## Summary

Keep open. Current main has model failover and several separate model-switch controls, but it does not have one operation that retargets every persisted and active model selection. Because the request spans model config, per-session overrides, subagent defaults, cron payloads, active chats, and owner-only command behavior, it needs maintainer product design before a repair lane can safely implement it.

## What This Changes

Request one command, callable from chat, to move the current chat plus agents, subagents, cron jobs, and other model-bound tasks to a different configured provider/model when a provider is rate limited.

## Best Possible Solution

Design a core, owner-only bulk model migration flow with a dry run and explicit scopes. It should clearly show and optionally update default model/fallbacks, configured model allowlists, per-agent and subagent defaults, selected session overrides, and cron payload model/fallback settings while preserving intentional overrides unless the operator opts in.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection: current main exposes separate default, session, subagent, and cron model controls, but no single bulk retarget command.

## Solution Assessment

Unclear as phrased. A scoped, dry-run-capable owner command is a maintainable direction; an unqualified `/moveall` that silently rewrites every model reference would be too risky for existing per-agent and scheduled-work configuration.
