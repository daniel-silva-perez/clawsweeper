## Summary

Keep open. Current main has scoped model controls for defaults, sessions, queued followups, cron, and subagents, but it still lacks a single safe operation that retargets all persisted and active model selections; the existing ClawSweeper discussion reached the same keep-open conclusion and no linked implementation or workaround has appeared.

## What This Changes

Add one command or flow to move OpenClaw’s default, session, subagent, cron, task, and current chat model selections to another configured provider/model during rate limits.

## Best Possible Solution

Design an owner-only bulk model migration flow, likely under the model command surface, that validates the target provider/model, supports dry-run preview, offers explicit scopes for defaults/sessions/cron/subagents/fallbacks/media/auth profiles, and preserves deliberate overrides unless the operator opts into changing them.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection: current main exposes scoped model controls and fallback behavior, but no single bulk model migration command or API.

## Solution Assessment

Unclear as phrased. A dry-run-capable owner command with explicit scopes is the maintainable direction; an unqualified /moveall that silently rewrites every model reference would be too risky.
