## Summary

Keep open. Current main still lacks the requested configurable media-store/staging limit, and the submitted branch is not merge-ready against current main because key runtime paths remain unwired or broken.

## What This Changes

The branch adds `media.maxBytes` config/schema/help/types plumbing and changes media storage, serving, and sandbox staging to read a mutable media byte limit.

## Best Possible Solution

Ship a refreshed, bounded media limit contract that is generated into config metadata, initialized during gateway startup, applied consistently to store defaults, sandbox staging, and chat sandbox preflight, while preserving intentional per-channel or per-caller caps where they are meant to be independent.
