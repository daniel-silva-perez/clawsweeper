## Risks / Open Questions

- The current PR head has unresolved review blockers in the provided context: missing `getMediaMaxBytes` imports in media serving and sandbox staging, plus missing startup setter wiring, would make the knob ineffective or break affected paths if merged as described.
- The global `media.maxBytes` design still needs explicit precedence against caller-provided or channel-specific `maxBytes`; otherwise lower global caps may not consistently constrain inbound persistence.
- Raising accepted media sizes is resource-sensitive and needs bounded schema validation, cleanup/TTL behavior, clear errors, and regression tests for default, configured, reset-to-default, oversized, and channel-limit cases.
- A separate open PR #40894 is related prior art but is broader and mixed with unrelated changes, so it does not clearly supersede this narrower PR.
