## Summary

Keep this PR open. Current main still does not implement `media.maxBytes`: the shared media store default, `/media/:id` serving path, and sandbox staging path remain tied to the hardcoded 5 MB `MEDIA_MAX_BYTES`, and the public config schema/types/help surface still lacks the requested field. The PR is still a plausible implementation candidate for #40880, but it needs a refresh against current main and must address the existing review-blocking runtime concerns before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate for #40880, but require an update against current main. The best fix is to add `media.maxBytes` consistently to schema, exported types, labels/help, generated config metadata, and user docs; initialize the effective value on every gateway boot with a default reset; expose the getter through the media runtime boundary; apply it to store, media serving, and sandbox staging; define how it composes with channel-specific limits; and add focused regression tests for default, configured, reset-to-default, oversized, and caller-limit cases.
