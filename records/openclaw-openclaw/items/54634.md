## Summary

Keep open. Current main has adjacent config/state diagnostics and service-env hardening, but the reported cross-HOME update safeguard is still missing: update discovery starts from the updater process environment, Linux systemd unit lookup is HOME-keyed, and forced reinstall intentionally drops prior HOME and OPENCLAW_* control roots instead of preserving, migrating, warning, or blocking when the installed service root differs.

## What This Changes

The issue requests update-time protection for Linux systemd installs so changing HOME from a dedicated OpenClaw user to root does not silently abandon the existing config and state directory.

## Best Possible Solution

Add a Linux update/service-refresh guard that discovers the previously installed gateway service root before rewriting it, compares HOME, OPENCLAW_HOME, OPENCLAW_STATE_DIR, and OPENCLAW_CONFIG_PATH against the updater environment, and then either preserves the prior root when safe or emits a clear migration/blocking warning. Cover the dedicated-user service refreshed from root case with update, daemon install, systemd, and doctor regression tests; keep GHCR version-tag rollback/pinning as separate work.
