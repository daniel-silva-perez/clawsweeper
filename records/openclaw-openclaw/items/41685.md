## Summary

Current main still leaves the reported Control UI listing boundary unresolved. The merged deleted-agent work rejects several runtime resolution/send paths, but the Sessions tab still consumes broad Gateway session discovery and renders returned agent-scoped rows without checking whether the agent is configured; URL/browser state also still preserves explicit non-alias session keys.

## What This Changes

The issue asks for the Control UI to stop showing or restoring sessions for unregistered or deleted agents so it matches configured-agent expectations from the CLI.

## Best Possible Solution

Keep broad Gateway/ACP discovery available for non-UI consumers, but make the Control UI default to configured-agent sessions or request an additive configured-only mode. Also validate explicit restored `agent:<id>:*` URL/storage selections against configured agents and fall back to the current main session key, with regression tests for both the Sessions tab and restore path.
