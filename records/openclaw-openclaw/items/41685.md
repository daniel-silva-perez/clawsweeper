## Summary

Keep open. Current main only partially mitigates the reported problem: merged runtime guards reject deleted-agent session resolution/send paths, but Control UI still consumes broad Gateway `sessions.list` results from the combined store, and that combined store intentionally includes disk-discovered agent stores. URL/browser state also still preserves explicit non-alias `agent:<id>:*` selections, so the user-facing listing/rehydration boundary remains unresolved.

## What This Changes

Request that Control UI only display and restore sessions for agents registered in `openclaw.json`, matching CLI session behavior after an agent is deleted.

## Best Possible Solution

Keep broad Gateway/ACP discovery available for non-UI consumers, but make Control UI default to configured-agent sessions or request an additive configured-only mode. Validate restored `agent:<id>:*` URL/storage selections against configured agents and fall back to the current main session key, with regression coverage for both the Sessions tab and restore path.
