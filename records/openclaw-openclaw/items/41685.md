## Summary

Keep this open. Current main partially mitigates the fresh agent-delete path through the later orphaned-session invalidation work, but the Control UI Sessions tab still consumes broad Gateway session discovery and renders returned rows without a configured-agent boundary. The URL/browser restore path also still accepts explicit `agent:<id>:*` session keys before any configured-agent validation, matching the broader variant described in the discussion.

## What This Changes

Make the Control UI show and restore sessions only for agents registered in `openclaw.json`, matching the configured-agent behavior users see from the CLI.

## Best Possible Solution

Preserve broad Gateway/ACP `sessions.list` discovery for non-UI consumers, but add a Control UI default boundary that filters visible rows to configured agents or uses an additive configured-only option. Also validate restored `agent:<id>:*` URL/browser-storage session keys against configured agents and fall back to `sessionDefaults.mainSessionKey`, with regression coverage for both the Sessions tab and restore path.
