## Summary

Keep this issue open. Current main has adjacent deleted-agent guards, but the Control UI Sessions tab still consumes the broad Gateway `sessions.list` result, and that Gateway path still intentionally merges disk-discovered agent stores. The follow-up restore concern also remains: explicit `agent:<id>:*` URL/browser session keys are preserved unless they are aliases of the current default agent.

## Best Possible Solution

Keep this issue open. Add a Control UI-specific configured-agent boundary: either add an additive `sessions.list` parameter/default suitable for configured-agent-only UI rows, or filter the Sessions tab against `agents.list`, while preserving broad Gateway/ACP discovery for disk-only callers. Also reject or normalize restored `agent:<id>:*` URL/browser session keys when `<id>` is no longer configured, falling back to `sessionDefaults.mainSessionKey`, with regression coverage for both the Sessions tab and restore path.
