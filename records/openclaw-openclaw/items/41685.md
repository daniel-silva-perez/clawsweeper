## Summary

Keep open. Current main still only partially mitigates the report: deleted-agent send/resolve paths now reject orphaned sessions, but the Control UI still lists sessions from the broad Gateway combined store and still preserves explicit agent-scoped URL/storage selections for agents that are no longer configured.

## What This Changes

The issue requests that Control UI show only sessions for agents registered in OpenClaw config, matching the CLI behavior after an agent is deleted.

## Best Possible Solution

Keep broad Gateway/ACP discovery available for non-UI consumers, but make Control UI default to configured-agent sessions or use an additive configured-only list mode. Validate restored `agent:<id>:*` URL/storage selections against configured agents and fall back to the current main session key when the agent is unknown.
