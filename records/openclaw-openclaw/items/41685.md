## Summary

Keep open. Current main still leaves the user-facing Control UI listing and restore behavior unresolved: the Sessions tab requests and renders broad Gateway session discovery, while the Gateway combined store intentionally includes disk-discovered agent stores. The merged deleted-agent guards reduce send/resolve targeting of orphaned sessions, but they do not make the Control UI default to configured agents only.

## What This Changes

The issue requests that Control UI session listing and restored session selection ignore sessions for agents no longer registered in OpenClaw config.

## Best Possible Solution

Keep broad Gateway/ACP disk discovery available for recovery and non-UI consumers, but make Control UI default to configured-agent sessions and validate restored agent-scoped URL/storage selections against known agents before falling back to the current main session key.
