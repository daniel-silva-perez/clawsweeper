## Summary

Keep open. Current main still leaves the reported user-facing boundary unresolved: Control UI consumes broad Gateway session discovery from the combined store, and URL/browser restore paths still preserve explicit stale agent-scoped session keys. The merged deleted-agent guards reduce send/resolve targeting of orphaned sessions, but they do not make the Sessions tab or restore behavior default to configured agents only.

## What This Changes

The issue asks the Control UI Sessions tab and restored session selection to ignore deleted or unregistered agent namespaces and match the CLI's configured-agent scoping.

## Best Possible Solution

Preserve broad Gateway/ACP discovery for recovery and non-UI consumers, but make Control UI default to configured-agent sessions and validate restored agent-scoped URL/storage selections against configured agents before falling back to the current main session key.
