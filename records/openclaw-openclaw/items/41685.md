## Summary

Keep open. Current main partially mitigates deleted-agent targeting by purging/rejecting orphaned session keys, but the reported Control UI listing and browser/URL restore behavior is still unresolved because the UI consumes broad Gateway session discovery and preserves explicit non-alias agent-scoped keys.

## What This Changes

The issue asks for Control UI sessions to exclude or safely fall back from sessions whose agent id is no longer registered in openclaw.json, matching the CLI’s configured-agent behavior.

## Best Possible Solution

Keep broad Gateway/ACP session discovery available for recovery and non-UI consumers, but make Control UI default to configured-agent sessions or request an additive configured-only mode. Validate restored agent-scoped URL/storage selections against configured agents and fall back to the current main session key, with regression coverage for both the Sessions tab and restore path.
