## Summary

Keep open. Current main still leaves the reported Control UI listing and restore behavior unresolved: the UI consumes broad Gateway session discovery, the Gateway combined store still includes disk-discovered agent stores, and explicit agent-scoped URL/storage selections are still preserved. The related merged deletion guards mitigate targeting orphaned sessions, but they do not make the Control UI default to configured agents only.

## What This Changes

Request that the Control UI stop displaying or restoring sessions for deleted or unregistered agents so it matches configured-agent CLI session listing.

## Best Possible Solution

Keep broad Gateway/ACP discovery available for non-UI consumers, but make Control UI default to configured-agent sessions through a UI filter or additive configured-only list mode. Validate restored agent-scoped URL/storage selections against configured agents and fall back to the current main session key when the agent namespace is unknown, with regression coverage for both the Sessions tab and restore path.
