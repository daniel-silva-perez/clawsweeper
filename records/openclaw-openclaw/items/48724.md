## Summary

Keep this PR open. Current main still has the Docker token-output, default Docker port-publishing, and mutable CI action-ref concerns that this PR is trying to address, so it is not obsolete. However, the PR is stale against the active Docker setup path and carries unresolved security/portability review findings in its added secure Compose file.

## Best Possible Solution

Keep this PR open for maintainer follow-up or replace it with a focused rebased hardening PR: mask tokens in scripts/docker/setup.sh with a safe copyable health-check flow, make an explicit Docker exposure decision and update docs, audit and pin the current CI action refs, and drop or repair docker-compose.secure.yml before merge.
