## Summary

Keep this PR open. Current main still has the core gaps the PR is trying to harden, and the PR head is not safe to merge as-is because its added secure Compose file has unresolved portability/security-review concerns.

## Best Possible Solution

Keep the PR open for maintainer follow-up or a focused replacement: rebase onto current main, move token masking into scripts/docker/setup.sh with a copyable safe health-check flow, decide and document whether Docker defaults should remain LAN-friendly or become loopback-only, update CI action pinning against the current workflow versions, and drop or repair docker-compose.secure.yml before merge.

## Risks / Open Questions

- Security review: the PR touches GitHub Actions execution and Docker network exposure. The action SHA pins should be updated to current action versions or replaced by a documented policy, and Docker exposure should match the intended LAN/Tailscale/loopback model.
- Merging the PR head as-is would add a non-portable docker-compose.secure.yml with user-specific mount paths and a host-specific VPN IP, likely broken outside the author's machine.
- The token masking direction is still useful, but the health-check command needs a safe, self-contained path such as reading the generated .env file or documenting an explicit export without printing the token.
