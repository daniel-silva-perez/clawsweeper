## Summary

Keep this PR open. Current main still has the Docker token-output, default port-publishing, and mutable CI action-ref gaps the PR targets, but the PR is stale against the active Docker setup path and its added secure Compose file has unresolved security/portability review findings.

## Best Possible Solution

Keep this PR open for maintainer follow-up or replace it with a focused rebased hardening PR: mask tokens in scripts/docker/setup.sh with a safe copyable health-check flow, make an explicit Docker exposure decision and update docs, audit/pin the current CI action refs, and drop or repair docker-compose.secure.yml before merge.
