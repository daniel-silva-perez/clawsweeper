## Summary

Keep this issue open. Current main still depends on sharp-backed media image ops for resizing, sharp 0.34.5 still has no android-arm64 prebuilt/runtime package, and the OpenClaw update path still only retries npm installs with --omit=optional rather than detecting Termux or rebuilding sharp with a Termux-native toolchain. The prior ClawSweeper keep-open review is still accurate against current main.

## Best Possible Solution

Keep this open and implement a focused image-path fallback first: when sharp-backed resizing cannot load on unsupported platforms, preserve the original canonical image block only if it is valid, supported, and within the existing safety envelope; otherwise keep omitting unsafe images with a clearer reason. Leave a Termux-native rebuild utility or broader update repair policy as a separate maintainer decision.
