## Summary

Keep open. Current main still has a reachable Android/Termux sharp failure path: the media-understanding plugin still depends on sharp, the lockfile has no android-arm64 sharp/libvips optional package, and normal image/file loading can still enter sharp-backed resize or optimization. Prior ClawSweeper comments correctly narrowed this to a graceful-fallback repair while leaving Termux-native rebuild automation as separate packaging work.

## What This Changes

Address the Android/Termux sharp/libvips native-module failure after updates, with possible native-module repair tooling or graceful image fallback behavior.

## Best Possible Solution

Add a bounded backend-unavailable fallback in shared image loading and sanitization: preserve original image inputs only when MIME, byte size, and header-derived dimensions prove they are already safe; otherwise return a clear backend-unavailable omission/error. Keep Termux-specific native rebuild automation as a separate packaging decision.

## Repair Work Prompt

Implement a focused fix for this issue's graceful image fallback only. Current main still depends on sharp through `extensions/media-understanding-core`, and `loadWebMedia` image optimization can fail when sharp cannot load on Android/Termux. Add a bounded fallback in shared image loading/sanitization: when the sharp-backed backend is unavailable, preserve the original only when the input is an image, byte size is within the applicable cap, and header-derived dimensions for common formats prove it is within limits; otherwise return a clear backend-unavailable omission/error. Cover tool image blocks and web-media file/URL paths, including safe-under-cap PNG/JPEG/WebP/GIF passthrough and resize-required or unknown-dimension rejection. Do not add Termux-specific native rebuild automation, `openclaw fix-native-modules`, dependency overrides, sharp version changes, package-manager policy changes, or lockfile churn. Changelog is not required for a purely defensive fallback unless the final user-facing behavior warrants a single-line Fixes entry crediting the reporter. Validate with the targeted tests and formatter listed in this work item, then run `pnpm check:changed` via Blacksmith Testbox before handoff if code/tests changed.
