## Summary

Keep open. Current main still implements Discord guild/channel access as `requireMention` plus `users`/`roles` allowlists. It has role allowlist support, but not the requested `bypass`/`mention`/`deny` access lists, permission shortcuts, deny-first precedence, sender-specific mention behavior, additive channel inheritance with `inherit: false`, or a migration/deprecation path. The updated proposal usefully aligns channel inheritance with open #73387, but that related issue is a design dependency rather than a complete superseding fix.

## Best Possible Solution

Keep this open for a Discord plugin access-control design pass. The best path is a backward-compatible schema and evaluator in the bundled Discord plugin, aligned with #73387: support deny-first `bypass`/`mention`/`deny` lists, role and permission entries, groupPolicy-aware channel inheritance, explicit `enabled: false`/`inherit: false` behavior, old-field compatibility warnings or doctor migration, public docs, schema updates, and regression tests for mixed senders and channel overrides.
