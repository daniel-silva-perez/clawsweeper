## Summary

Keep open. Current main still uses Discord `requireMention` plus `users`/`roles` allowlists, so it cannot express the requested “owner bypasses mention, everyone else may mention” policy, deny precedence, permission shortcuts, or group-policy-aware channel inheritance. The updated discussion usefully ties this to #73387 while preserving broader access-list work that remains unimplemented.

## What This Changes

Request replacing Discord guild/channel `requireMention` and `users` with bypass, mention, and deny access lists that support users, roles, permissions, and per-channel overrides.

## Best Possible Solution

Keep this open as a Discord plugin access-control design item. The eventual fix should update the Discord config schema, runtime evaluation, doctor/deprecation path, docs, and regression tests together, while coordinating the channel inheritance piece with #73387.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: configure a guild with `requireMention: false` and `users: [owner]`; current code drops every other sender before mention gating, so it cannot allow everyone else to trigger only with an @mention.

## Solution Assessment

Unclear. The bypass/mention/deny model is a plausible direction, but because it changes authorization semantics and migration behavior, the best implementation should be approved by a maintainer before an automated fix PR starts.
