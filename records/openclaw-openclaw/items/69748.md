## Summary

Keep open. Current main still uses the existing Discord guild/channel config model: `requireMention` plus `users`/`roles` allowlists. It has partial role allowlist support, but it does not implement the requested `bypass`/`mention`/`deny` access lists, permission shortcuts, explicit deny precedence, backward-compatible migration, or additive channel inheritance. The open #73387 overlap is a real prerequisite/design concern rather than a reason to close this broader request.

## Best Possible Solution

Keep this open for a Discord plugin access-control design pass. The best path is a backward-compatible schema/evaluator in the bundled Discord plugin, aligned with #73387, with docs, strict config schema updates, doctor/migration behavior for `requireMention` and `users`, and regression tests for deny precedence, role/permission matching, mention-only users, bypass users, and channel inheritance.
