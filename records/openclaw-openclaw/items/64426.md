## Summary

Keep open. Current main still has only adjacent generic tool-loop detection: it is disabled by default, hashes exact tool arguments, and warning results are diagnostic-only rather than a model-visible read-specific reread nudge. The related exec proposal remains separate and open, so it does not supersede this first-class read-tool request.

## What This Changes

The issue requests a warning-only same-turn guard that detects repeated first-class `read` calls over the same file or small file set and nudges the agent before token burn stalls a turn.

## Best Possible Solution

Add a core read-aware guard that normalizes read file identities per run, detects repeated same-file and small-file-set rereads, suppresses or resets after real progress, emits diagnostics, and returns a model-visible soft warning while leaving the existing exec policy unchanged by default.
