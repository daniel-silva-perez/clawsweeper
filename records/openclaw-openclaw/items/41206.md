## Summary

Keep this PR open. Current main still does not implement per-job cron storage, and the open #20950 feature request still tracks the same user problem. This PR is a relevant implementation candidate, but it is not ready to close or merge: the diff only adds store helpers/tests, does not wire the runtime/config/CLI/docs/doctor surface, and the PR discussion identifies real filename-keying data-integrity bugs.

## Best Possible Solution

Keep this PR open as a useful but incomplete implementation candidate for #20950. Before maintainer merge review, revise it around a stable id-based filename contract or load/remove-by-persisted-id behavior, add tests for id/name mismatch, rename, delete, collisions, migration, and symlink/GitOps usage, then wire the runtime service, config/schema/help, CLI migration command, doctor overlap warnings, and public cron docs so per-job storage is a supported surface.
