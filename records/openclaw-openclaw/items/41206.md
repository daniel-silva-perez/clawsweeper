## Summary

Keep this PR open, but it is not merge-ready. Current main still uses the aggregate cron `jobs.json` store, while this patch adds disconnected directory-store helpers with correctness and path-safety defects: files are saved by mutable job names, removed by filename-derived ids, not wired into service load/persist operations, and the slug fallback can preserve unsafe path separators.

## What This Changes

The PR adds `src/cron/store.ts` helpers and `src/cron/store.dir.test.ts` coverage for an optional `cron/jobs/` directory where each JSON file represents one cron job while preserving `jobs.json` support.

## Best Possible Solution

Rework this on current main as an additive cron directory-store design: key files by immutable ids, keep `jobs.json` compatibility, preserve `jobs-state.json` runtime state semantics, wire Gateway/service load and add/edit/remove paths, add config/docs/doctor support, and cover migration, path confinement, duplicate ids, rename/delete, permissions, and state-sidecar behavior in focused cron tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Key job files by immutable ids:** `src/cron/store.ts:246-247`
  - body: `saveCronJobFile` builds the path from `job.name`, but names are mutable and not unique while current cron jobs use UUID ids. Two jobs with the same name can overwrite each other, and renaming a job writes a second file with the same persisted id, causing duplicate jobs on reload.
  - confidence: 0.96
- **[P1] Delete files by persisted job ids:** `src/cron/store.ts:271-272`
  - body: `removeCronJobFile` compares the requested id with the id inferred from the filename, but files saved by this patch are named from the job name. A normal UUID-backed job named `Daily Brew` saves as `daily-brew.json`, so removal by id returns false and leaves the job file behind.
  - confidence: 0.97
- **[P2] Wire the directory store into service persistence:** `src/cron/store.ts:187-236`
  - body: The new directory helpers are never selected by `ensureLoaded`, `persist`, Gateway startup, or cron add/edit/remove operations. As written, the feature is mostly dead code: current service paths still read and write only the aggregate `jobs.json` store.
  - confidence: 0.88
- **[P2] Keep raw fallback strings out of paths:** `src/cron/store.ts:151`
  - body: When a name contains no alphanumeric characters, `slugifyCronJobName` returns the raw input and `saveCronJobFile` joins it into a filesystem path. A punctuation-only name containing separators can therefore write outside `cron/jobs/`; the fallback must be a safe generated filename or sanitized id.
  - confidence: 0.82

## Security Review

Status: needs_attention
