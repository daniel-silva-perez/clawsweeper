## Summary

Current main does not implement the PR's intended same-agent stale-root remap. The resolver still preserves canonical cross-root absolute `.../agents/<agent>/sessions/<file>` paths through the structural fallback, and the linked same-author bug #61321 is still open. The PR is focused and plausibly useful, but it should stay open because the supplied review discussion identifies a real custom `session.store` / `sessionsDir` consistency concern in the proposed patch.

## Best Possible Solution

Keep this PR open and revise/review it rather than closing it. The bug still appears present on current main, but the implementation should remap stale same-agent paths relative to the active sessions directory, preserve the existing cross-agent fallback behavior, and add regression coverage for both default-state and custom `session.store` cases before merge.
