## Summary

Keep this PR open. Current main still appears to preserve stale same-agent cross-root sessionFile paths, and the linked bug remains open, but the proposed patch has a blocking custom session.store consistency bug: the new remap ignores the active sessionsDir and can redirect transcript writes into the default state tree.

## What This Changes

The PR adds a same-agent stale absolute sessionFile remap in session path resolution plus a unit regression test for remapping into the current state dir.

## Best Possible Solution

Revise the PR so same-agent stale-root remaps are built under the resolved active sessionsDir, preserve the existing cross-agent structural fallback behavior, and add regression coverage for both default state-root and custom session.store paths.

## Reproduction Assessment

Yes. A session entry whose sessionFile is an old-root absolute path like /home/rai/.openclaw/agents/shikamaru/sessions/sess-1.jsonl while the current state root is different reaches the structural fallback on current main and remains pointed at the stale path. The PR-specific blocker is also reproducible by passing a custom sessionsDir/store path and observing that the proposed helper remaps to resolveAgentSessionsDir() instead of that active directory.

## Solution Assessment

No. The fix direction is right, but this implementation is not the best way to solve it because it remaps through the default state-derived agent sessions directory rather than the active sessionsDir. The safer narrow solution is to remap only the validated filename under the already-resolved sessionsDir/realBase used by resolvePathWithinSessionsDir().

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor the active sessionsDir when remapping stale paths:** `src/config/sessions/paths.ts:188`
  - body: The new remap returns resolveAgentSessionsDir(...), which ignores opts.sessionsDir from a custom session.store. In that configuration, a stale same-agent sessionFile is redirected into OPENCLAW_STATE_DIR instead of the active store directory, splitting transcript writes away from the configured store. Build the remapped file under the resolved active sessionsDir instead.
