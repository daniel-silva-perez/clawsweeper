## Summary

Keep open. Current main still preserves and forwards the SSRF guard dispatcher on the Slack inbound media path, while this PR strips it; that is not implemented on main and it changes a security-sensitive pinned-DNS transport contract that needs maintainer review before merge or replacement.

## What This Changes

The PR strips the `dispatcher` field from Slack media fetch init before delegating, updates the Slack media test to expect no forwarded dispatcher, and adjusts raw-fetch allowlist line numbers.

## Best Possible Solution

A narrow fix should preserve the SSRF guard's intended network policy while resolving the Slack media failure: either keep the pinned dispatcher and fix the actual Undici mismatch, or introduce an explicit Slack-safe `pinDns: false`/preflight mode with tests, comments, and security rationale covering redirects and hostname validation.
