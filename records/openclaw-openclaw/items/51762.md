Current main still does not implement configurable fallback default agent IDs: bootstrap paths, default-agent resolution, config schema/types, session-key helpers, docs, and existing tests still fall back to `main`. The linked feature issue #17477 is closed stale, so this PR remains the active implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for configurable fallback default agent IDs. The best path is to route bootstrap, config/env fallback resolution, main-session/session-key synthesis, provider auth/model state paths, docs, and tests through one normalized default-agent contract; validate the new env/config ID before filesystem use; then land after focused changed-gate proof and remaining review concerns are addressed.
