## Summary

Keep open. Current main still does not implement the PR's configurable fallback default agent ID behavior: bootstrap directory resolution, default-agent resolution, config schema/types, session-key helpers, docs, and tests still use the hardcoded `main` fallback unless `agents.list` supplies an agent. The linked feature issue #17477 is closed stale, so this PR remains the active implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for configurable fallback default agent IDs. The best path is to route bootstrap, config/env fallback resolution, main-session/session-key synthesis, store path selection, docs, and tests through one normalized default-agent contract; validate the new env/config ID before filesystem use; then land after focused changed-gate and schema-generation proof.
