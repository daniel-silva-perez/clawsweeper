## Summary

Keep this PR open. Current main still has the Nostr relay URL validation gap, and the PR remains directionally useful after the author addressed the initial review concern by restricting plaintext `ws://` to loopback hosts. It needs a refresh because main now has separate schema, CLI setup, and Control UI setup relay parsers, while the PR only updates the schema and adds tests under a test file that no longer exists on main.

## Best Possible Solution

Refresh this PR against current main with a shared Nostr relay URL validator used by `NostrConfigSchema`, CLI setup, Control UI setup, generated config metadata, and focused current-main tests. The validator should allow remote `wss://`, allow `ws://` only for localhost/loopback development hosts, and reject other schemes. Drop unrelated `.secrets.baseline` churn.
