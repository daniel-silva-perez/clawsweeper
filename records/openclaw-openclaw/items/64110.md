## Summary

Keep open. Current main does not already implement the PR’s intended deletion-aware cancellation for timed cron executions: deleting a cron job removes the stored definition, but due jobs already collected by the timer still execute from the in-memory job object before the post-run store reload can discard the result. The PR is not obsolete, but it needs maintainer review because the provided review discussion still identifies correctness risks in the proposed guard/error handling.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The best implementation is a deletion-aware pre-execution guard for timed and queued/manual cron runs, with existence-check failures handled as per-job outcomes inside `runDueJob`, minimal reload overhead, and regression tests proving that deleting a scheduled task prevents queued-but-not-started delivery. If a separate PR lands the same behavior first, this PR can then be closed as superseded or implemented.
