## Summary

Keep this PR open. Current main has the shared markdown-table metadata seam and general Slack Block Kit message support, but it still does not implement this PR’s Slack markdown-table send path: table metadata is not consumed by the Slack sender, table block attachments are absent, and block table mode is still coerced back to code.

## Best Possible Solution

Keep the PR open as a plausible implementation candidate. Rebase it onto current main, adapt it to the plugin-registry-based table-mode defaults, wire Slack markdown table metadata into the Slack-owned send path, preserve safe code fallback for non-Slack and unsupported Slack reply paths, and retain focused tests for single-table attachments, multi-table fallback, media+table delivery, truncation limits, fallback text, and slash replies.
