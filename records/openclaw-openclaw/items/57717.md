## Summary

Keep this PR open. Current main has the shared markdown/config seam for table metadata, and it has general Slack Block Kit support, but it does not implement this PR’s Slack markdown-table send path. Slack text delivery still renders only mrkdwn chunks, `block` table mode is still coerced back to `code`, and the table attachment helpers from the PR are absent. The PR also has useful follow-up discussion on #57591 and the broader open Slack Block Kit request #12602, so the best path is review/rebase rather than cleanup closure.

## Best Possible Solution

Keep the PR open as a plausible implementation candidate. Rebase it onto current main, adapt the Slack default/gating to the current plugin messaging-default architecture or scope block mode to the Slack send path, keep non-Slack and slash-reply paths from dropping table content, and retain focused tests for single-table attachments, multi-table code fallback, media+table delivery, truncation limits, and fallback text.
