## Summary

Keep this PR open, but it is not merge-ready. Current main still lacks the Slack markdown-table send path this PR targets, while the PR branch has a remaining data-loss bug from making Slack block table mode the global default for Slack flows that do not send table attachments.

## What This Changes

The PR wires Slack markdown-table extraction into outbound Slack sends so a single markdown table is delivered as a Block Kit table attachment and multiple tables fall back to text/code rendering.

## Best Possible Solution

Land a rebased Slack-owned implementation that keeps shared IR/config seams generic, enables block table mode only for Slack flows that consume table metadata, preserves code/text fallback for slash and other non-table-aware paths, and adds regression coverage for normal sends, media sends, multi-table fallback, truncation, and slash replies.

## Reproduction Assessment

Yes. On the PR head, a Slack slash-command response containing a markdown table can be traced through resolveMarkdownTableMode returning block and deliverSlackSlashReplies rendering with markdownToSlackMrkdwnChunks only; block mode removes table text into metadata, but the slash response path never sends that metadata as an attachment.

## Solution Assessment

No. The PR is aimed at the right Slack plugin boundary, but the global Slack default is not the best fix because it affects non-table-aware Slack delivery paths; the safer solution is send-path-scoped block mode or explicit code fallback for those paths, rebased onto the current plugin metadata architecture.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Scope block table mode to table-aware sends:** `src/config/markdown-tables.ts:18`
  - body: Setting Slack's default table mode to `block` makes every Slack caller of `resolveMarkdownTableMode` use block mode. Slash-command replies still render through `markdownToSlackMrkdwnChunks` and only send response_url text, so markdown tables are collected out of the text but never sent as table attachments. Keep block mode scoped to `sendMessageSlack` or coerce slash replies back to `code` until that path consumes table metadata.
