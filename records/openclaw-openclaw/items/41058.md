## Summary

Keep this issue open. Current main still hardcodes the pairing reply text and CLI approval hint, the operator-facing `channels.*.pairingMessage` / `showCliHint` config surface is absent from types and strict schemas, and PR #42617 remains the open same-author implementation candidate with closing syntax for this issue.

## Best Possible Solution
