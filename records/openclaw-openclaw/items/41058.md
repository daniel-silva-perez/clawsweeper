## Summary

Keep this issue open. Current main still hardcodes the DM pairing reply and has no supported `channels.*.pairingMessage` or `showCliHint` config surface; same-author PR #42617 remains open as the active implementation candidate.

## Best Possible Solution

Keep this issue open and review PR #42617 as the current implementation candidate. The likely maintainer path is a typed, documented `pairingMessage` config surface with schema validation and tests, plus a scope decision on per-channel configuration versus shared defaults across all pairing-capable channels.
