## What This Changes

The issue reports that the `edit` tool rejects both stringified and native `edits` array payloads in a DeepSeek/OpenAI-compatible environment, making edits unusable while write and sed still work.

## Best Possible Solution

Add a focused regression that exercises the DeepSeek/OpenAI-compatible tool-call path with a native `edits` array, capture the raw parameters that reach OpenClaw, then fix the narrow boundary that creates or receives the wrapper while keeping `{ oldText, newText }` as the public edit contract.
