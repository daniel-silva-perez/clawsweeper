## Summary

Keep open. Current main has a partial `sessions_send` A2A follow-up path, but it does not implement this PR’s proposed Phase 1 contract: no `handoff_id`, no delivered/accepted/queued/rejected acknowledgement envelope, and no append-only handoff ledger. The PR is also an active draft proposal with an explicit maintainer comment saying the lower-level primitive is still useful and should stay separate from the meeting lifecycle work in #65403.

## Best Possible Solution

Keep this draft PR open as the focused transport-contract discussion. The best next step is a narrow implementation PR that adds a stable handoff id, structured acknowledgement/result states, visibility handling for `NO_REPLY`/`REPLY_SKIP`/`ANNOUNCE_SKIP`, an append-only audit record, and regression tests around the current `sessions_send` A2A flow, while keeping the meeting lifecycle layer in #65403.
