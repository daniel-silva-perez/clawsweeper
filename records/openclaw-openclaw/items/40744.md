## Summary

Keep this PR open. Current main still lacks the PR's central behavior: shared tool-schema normalization preserves top-level non-array `required` values for non-Gemini/OpenAI-compatible providers, and the Gemini cleaner can still carry `required: null` through recursive cleaning. The PR remains a focused bug fix, but it needs a rebase onto the current `pi-tools-parameter-schema.ts` split and maintainer judgment on the related Gemini key-copy hardening note.

## Best Possible Solution

Keep the PR open and ask for a rebase onto current main. The best implementation is a narrow shared schema-normalization fix in the current `normalizeToolParameterSchema`/`normalizeToolParameters` path that strips own non-array `required` values while preserving valid arrays, plus regression tests for top-level OpenAI-compatible schemas and nested Gemini schemas. Maintainers should separately decide whether the Gemini key-copy hardening belongs in the same patch or a follow-up.
