## Summary

Keep this PR open. Current main still lacks the PR's central behavior: shared tool-schema normalization does not strip top-level non-array `required` values for OpenAI-compatible providers, and the Gemini recursive cleaner can still copy nested `required: null`. The PR is a focused source/test fix with useful review follow-up already addressed, but it needs a rebase/adaptation to the current `pi-tools-parameter-schema.ts` split and maintainer judgment on the related Gemini key-copy hardening note.

## Best Possible Solution

Keep this PR open and ask for a rebase onto current main. The best implementation is a narrowly tested schema-normalization fix in the current shared path that strips malformed non-array `required` values while preserving valid arrays, plus a nested Gemini regression test. Maintainers should decide whether the Gemini prototype-pollution hardening belongs in this PR or a separate follow-up.
