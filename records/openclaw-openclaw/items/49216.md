## Summary

Keep this PR open. Current main has default cleanup for deleted archives when reset archive retention is enabled, but it has not definitely implemented this PR's intended behavior: the cleanup block is still gated when `resetArchiveRetention` is disabled and no live session was archived, and the archive timestamp parser still rejects the legacy `+HH-MM` and compact timestamp forms the PR adds.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. The best path is to refresh it against current main, make the regression test cover `resetArchiveRetention: false`, keep the legacy timestamp parser narrowly scoped with accepted and rejected cases, then land it if maintainers want deleted archive cleanup to run during enforced maintenance regardless of reset-archive retention.
