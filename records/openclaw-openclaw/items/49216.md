## Summary

Current main still does not implement this PR’s intended behavior. Enforced maintenance can still skip expired `.deleted.*` archive cleanup when no live session was pruned and reset archive cleanup is disabled, and the archive timestamp parser still rejects the legacy `+HH-MM` and compact timestamp forms added by the PR. The PR is a focused implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open. Refresh it against current main, keep the focused source changes in `src/config/sessions/store.ts` and `src/config/sessions/artifacts.ts`, add regression coverage for no live-session pruning with `resetArchiveRetention: false`, add parser accepted/rejected legacy timestamp tests, then land if maintainers agree deleted archive cleanup should run during enforced maintenance independently of reset archive retention.
