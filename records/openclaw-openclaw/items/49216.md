## Summary

Current main still does not implement the PR's intended behavior for deleted archive cleanup when no live session is archived and reset archive cleanup is disabled, and it still rejects the legacy timestamp forms the PR adds. This PR should remain open as a focused implementation candidate rather than being closed as obsolete.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. Refresh it against current main, add explicit `resetArchiveRetention: false` regression coverage plus accepted and rejected legacy timestamp parser cases, then land it if maintainers agree deleted archive cleanup should run during enforced maintenance independently of reset archive retention.
