## Summary

Current main still does not implement this PR's intended session-maintenance behavior. Enforced maintenance still skips deleted archive cleanup when no live session was pruned and reset archive cleanup is disabled, and the archive timestamp parser still rejects the legacy offset and compact timestamp forms this PR adds. The PR remains a focused implementation candidate, not obsolete cleanup.

## Best Possible Solution

Keep this PR open. Refresh it against current main, keep the focused changes in `src/config/sessions/store.ts` and `src/config/sessions/artifacts.ts`, add a regression test for enforced maintenance with no pruned live session and `resetArchiveRetention: false`, add accepted and rejected legacy timestamp parser tests, then land if maintainers agree deleted archive cleanup should run during enforced maintenance independently of reset archive retention.
