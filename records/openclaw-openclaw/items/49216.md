## Summary

Keep this PR open. Current main still does not fully implement the PR‚Äôs intended behavior: enforced save maintenance still gates archive cleanup when no live session was archived and reset archive retention is disabled, and archive timestamp parsing still rejects the legacy `+HH-MM` and compact timestamp forms added by the PR.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. Refresh it against current main, strengthen the regression test with `resetArchiveRetention: false`, keep the legacy timestamp parser narrowly scoped with accepted and rejected cases, then land it if maintainers want deleted archive cleanup to run during enforced maintenance independently of reset-archive retention.
