## Summary

Keep open. Current main only partially addresses this class of bug: it hashes `extraSystemPromptStatic` instead of all inbound metadata, but the static set still includes first-turn-only `groupIntro`, so a normal group turn-1 to turn-2 transition can still change `extraSystemPromptHash` and trigger `reason=system-prompt`.

## Best Possible Solution

Keep this open and land a narrow CLI-session reuse fix: keep injecting `groupIntro` into the actual first-turn group system prompt, but remove first-turn/reintro-only `groupIntro` from the stable prompt text used for `extraSystemPromptHash`. Add a regression test for a group session turn 1 to turn 2 path that proves the stored CLI binding is reused and no `system-prompt` invalidation occurs. Then decide separately whether Claude live-session fingerprinting should ignore this same turn-scoped prompt delta to preserve warm stdio reuse.
