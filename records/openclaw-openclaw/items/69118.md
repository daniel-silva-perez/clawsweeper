## Summary

Keep open. Current main at f2405c830bd925bd24f91b6bac5d417c6d36e5ba still injects groupIntro only on the first group turn or re-intro, but includes that same turn-scoped text in extraSystemPromptStaticParts. prepareCliRunContext hashes that static text for CLI reuse, and resolveCliSessionReuse still invalidates on a prompt-hash mismatch, so the reported group turn-1 to turn-2 reset remains plausible on current main.

## Best Possible Solution

Keep this issue open and land a narrow CLI-session reuse fix: keep groupIntro in the actual first-turn/reintro group system prompt, but exclude first-turn-only groupIntro from the stable prompt text used for extraSystemPromptHash. Add a regression test for a group session turn 1 to turn 2 path proving the stored CLI binding is reused and no system-prompt invalidation occurs. Also inspect whether Claude live-session fingerprinting should align with the same stable prompt identity.
