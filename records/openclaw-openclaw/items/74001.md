## Summary

Keep this PR open. Current main still preserves Feishu `thread_id` from inbound events but routes P2P direct-message replies through the non-thread path: `replyInThread` and `threadReply` are forced false for non-groups, and both dispatcher call sites suppress reply metadata for DMs. The PR is a narrow replacement for closed source PR #38808 and targets open bug #38806, but it still needs maintainer review, a rebase because GitHub reports it dirty, and follow-up on the low Aisle type-safety finding before merge.
