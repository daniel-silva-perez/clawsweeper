## Summary

Keep open. Current main still has the Active Memory lightweight embedded recall path that allowlists Memory Core tool names without threading that runtime allowlist into plugin-tool materialization, and open PR #74592 is already linked with closing syntax as the implementation candidate.

## What This Changes

The issue requests that Active Memory lightweight embedded recall runs compose with Memory Core memory tools instead of resolving the memory allowlist to no callable tools and persisting a placeholder assistant turn.

## Best Possible Solution

Review and land or reject #74592. The desired end state is a generic, extension-agnostic fix where Active Memory embedded recalls can materialize only explicitly allowlisted memory plugin tools, retain the final allowlist filter, and avoid persisting a stream-error sentinel when the memory tool set is unavailable.
