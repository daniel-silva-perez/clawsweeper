## Summary

Keep open. Current main still matches the reported CLI/media-understanding bug, and three open PRs already reference this issue with closing syntax, so the issue should remain open until maintainers land or reject a canonical fix.

## What This Changes

The issue requests clearer missing-provider configuration errors for `infer audio transcribe` and `infer image describe` when no audio or image-understanding provider is configured.

## Best Possible Solution

Review and land one canonical fix that preserves the existing generic errors for provider-ran-but-empty-output, while surfacing a distinct missing-provider/configuration error for skipped/no-attempt media-understanding decisions. The fix should include focused CLI/runtime tests for `image describe`, `image describe-many`, and `audio transcribe` under all-unconfigured-provider conditions.

## Reproduction Assessment

Yes. The current source path is enough to reproduce with high confidence: no resolved media provider yields a skipped decision with no outputs, `runMediaUnderstandingFile` returns undefined text, and the CLI converts that into the reported path-based error.

## Solution Assessment

Yes. A narrow CLI/media-runtime distinction is the maintainable fix; the safest implementation should carry enough decision metadata to distinguish no-provider from empty provider output rather than guessing from the input path.
