## Summary

Keep open. Current main still conflates the no-media-provider path with provider-ran-but-empty output, and three open PRs currently reference this issue with closing syntax, so closing before a fix merges would be premature.

## What This Changes

The issue asks for `infer audio transcribe` and `infer image describe` to report missing media provider configuration instead of generic no-transcript/no-description file-path errors when no provider is configured.

## Best Possible Solution

Land one canonical fix that preserves the existing provider-empty-output errors while carrying skipped/no-provider state far enough for the CLI to emit actionable missing audio or image-understanding provider configuration errors, with focused tests for image describe, describe-many, and audio transcribe.

## Reproduction Assessment

Yes. The source path on current main is enough to reproduce with high confidence: no resolved media provider becomes a skipped decision with no outputs, then the CLI converts `text: undefined` into the generic no-description/no-transcript file-path error.

## Solution Assessment

Yes. The narrow maintainable solution is to separate missing-provider/no-attempts from provider-empty-output at the media runtime or CLI boundary; the issue should stay open until one linked fix PR merges.
