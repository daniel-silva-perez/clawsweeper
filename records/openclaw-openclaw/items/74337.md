## Summary

Keep this PR open for maintainer review. Current main still lacks per-session token quarter-hour buckets, and the Usage Mosaic plus hour filter still use session-span allocation; this PR is the active implementation candidate for the follow-up to the already-merged #49396 message-bucket work.

## What This Changes

This PR adds optional UTC quarter-hour token usage buckets to session cost summaries, aggregates them from transcript usage entries, updates Control UI Usage Mosaic/hour filtering to prefer those buckets with legacy fallback, and adds backend/UI tests plus a changelog entry.

## Best Possible Solution

Land a reviewed version that adds the optional token-bucket contract, derives sparse buckets from transcript usage, uses them in Usage Mosaic and hour selection with validated legacy fallback, and keeps focused backend/UI coverage for bucket aggregation, invalid data, timezone mapping, and fallback behavior.
