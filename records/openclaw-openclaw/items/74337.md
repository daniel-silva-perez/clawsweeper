## Summary

Keep this PR open. Current main still lacks token usage quarter-hour buckets and Usage Mosaic/hour filtering still use the legacy session-span behavior, while the PR is a live implementation branch that needs normal maintainer review rather than cleanup closure.

## What This Changes

This PR adds optional `utcQuarterHourTokenUsage` buckets to session cost summaries, derives them from transcript usage timestamps, updates Usage Mosaic/hour filtering to prefer those buckets with fallback, and adds backend/UI tests plus a changelog entry.

## Best Possible Solution

Land a reviewed version that adds the optional sparse token-bucket contract, derives buckets from transcript usage timestamps, uses validated bucket data in Usage Mosaic and hour selection, preserves legacy fallback for older summaries, and keeps focused backend/UI coverage for aggregation, invalid data, timezone mapping, and fallback behavior.
