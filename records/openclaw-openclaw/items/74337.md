## Summary

Keep this PR open for maintainer review. Current main still has the related quarter-hour message-count path from #49396, but it does not have the PR’s token-bucket contract or Usage Mosaic/hour-filter behavior; the existing bot discussion also treats this as useful unmerged implementation work rather than obsolete work.

This PR adds optional per-session UTC quarter-hour token usage buckets to session cost summaries and updates Control UI Usage Mosaic/hour filtering to prefer those buckets with legacy session-span fallback, plus tests and a changelog entry.

Review and iterate the PR on its merits, then land it if maintainers accept the optional token-bucket contract, Usage Mosaic aggregation, fallback behavior, tests, changelog entry, and changed-lane validation; otherwise keep the request open until equivalent token-bucket behavior is implemented.
