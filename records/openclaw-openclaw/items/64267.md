## Summary

Keep open. Current main has partial fixes for bare reasoning tags and commentary-phase suppression, but the streaming/block parser still does not share the canonical reasoning-tag grammar and can miss attributed thinking tags before user-visible delivery.

## Best Possible Solution

Keep this issue open and align the core embedded streaming utilities with the canonical reasoning-tag grammar. The fix should cover attributes and optional `antml:` prefixes in `stripBlockTags`, `splitThinkingTaggedText`, and tagged-reasoning extraction while preserving code-span and chunk-boundary behavior, with focused regression tests for attributed `<think>`, `<thinking>`, and `<antml:thinking>` blocks.

## Evidence

- **Issue discussion identifies a concrete core path:** A commenter traced the leak to the streaming/block scanner missing namespaced and attributed thinking tags, and the reporter later said the problem persisted after 2026.4.12. That makes this a concrete core sanitizer gap rather than a stale WeChat-plugin-only report.
- **Streaming scanner still rejects attributed thinking tags:** `THINKING_TAG_SCAN_RE` is built as `<\s*(\/?)\s*${THINKING_TAG_NAME_PATTERN}\s*>`, so it requires the tag name to be followed only by whitespace and `>`. It does not accept `<think reason="...">`, `<thinking hidden="1">`, or `<antml:thinking hidden="1">`.
