## Summary

Keep this PR open. Current main still has only partial Telegram Business groundwork: update names are available via grammY defaults/fallback constants, but main does not register business_connection/business_message handlers, model Business updates in dedupe or sequencing, inject business_connection_id for outbound replies, or include Business-specific tests/docs. The PR is relevant source work for #20786, but it needs a maintainer-guided refresh because existing review comments identify correctness issues in the proposed connection mapping.

## What This Changes

The PR adds Telegram Business update names to allowed updates, extends Telegram update key context, registers business_connection and business_message handlers with an in-memory chat-to-connection map plus API transformer, and adds a focused business handler test file.

## Best Possible Solution

Keep this PR open and ask for a current-main refresh or maintainer replacement that adds account-scoped Telegram Business connection tracking with disconnect cleanup, registers business_connection/business_message before the normal message handler, routes business_message through the existing DM authorization and inbound pipeline without double-dedupe skipping, injects business_connection_id only for supported outbound methods, and adds focused tests plus Telegram docs/changelog coverage.
