## Summary

Current main still does not support `payload.model: "default"` or `--model default` as a cron sentinel. Cron model selection trims any non-empty payload model and resolves it as an explicit model ref, while fallback policy also treats any non-empty payload model as an override. The related implementation PRs were closed unmerged, and docs/tests still describe `--model` as a concrete allowed job primary rather than a runtime-default sentinel.

## Best Possible Solution

Keep this open and implement a small shared cron payload-model helper that trims `payload.model` and treats exact `default` as unset. Use it in both model selection and fallback override handling, add focused tests for default/empty/explicit/invalid/fallback cases, and update cron CLI help/docs if `--model default` is intended as public behavior.
