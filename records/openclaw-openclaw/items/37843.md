## Summary

Keep this issue open. Current main still trims cron `payload.model` and treats every non-empty value, including `default`, as an explicit model override; fallback policy also treats it as an override. The related fix PRs in the provided GitHub context were closed unmerged, and current docs/tests do not document or verify `--model default` as shipped behavior.

## Best Possible Solution

Keep this issue open and implement a small shared cron payload-model helper that trims `payload.model` and treats exact `default` as unset. Use it in both `resolveCronModelSelection` and `resolveCronFallbacksOverride`, add focused tests for default/empty/explicit/invalid/fallback behavior, and update CLI help/docs if `--model default` is intended as a user-facing value.
