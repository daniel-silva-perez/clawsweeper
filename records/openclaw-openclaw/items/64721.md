## Summary

Keep open. Current main still only partially addresses the report: `model` and `timeoutSeconds` are present in the cron tool/protocol path, but `contextTokens` and `maxSpendUsd` are still absent from the advertised tool schema, strict gateway protocol schema, cron payload types, normalization hints, docs, and isolated cron runtime selection.

## What This Changes

The issue requests cron add/update tool support for `payload.model`, `payload.timeoutSeconds`, `payload.contextTokens`, and `maxSpendUsd` so agents can create and update fully configured cron jobs without editing `jobs.json` by hand.

## Best Possible Solution

Add a deliberate `contextTokens` cron `agentTurn` contract end to end: tool schema/descriptions, flat recovery, gateway protocol schema, exported cron types, normalizer, service patch behavior, isolated runtime precedence, docs, and regression tests. For `maxSpendUsd`, either implement enforceable per-run spend-cap behavior with tests or split it into a focused product/API issue.
