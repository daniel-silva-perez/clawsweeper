## Summary

Keep open. Current main still only covers the `model` and `timeoutSeconds` parts of the report; `contextTokens` and `maxSpendUsd` are absent from the cron tool, strict Gateway protocol schema, stored cron payload type, normalizer, docs, and payload-level isolated runtime selection.

## What This Changes

The issue requests that cron add/update accept and preserve `payload.model`, `payload.timeoutSeconds`, `payload.contextTokens`, and `maxSpendUsd` for agent-turn cron jobs.

## Best Possible Solution

Add a deliberate `contextTokens` contract for agent-turn cron jobs end to end, including tool schema/descriptions, flat argument recovery, Gateway protocol schema, exported cron types, normalization, service patch behavior, isolated runtime precedence, docs, and regression tests. Handle `maxSpendUsd` separately unless maintainers are ready to implement real per-run spend enforcement with tests.
