## Summary

Keep open. Current main deliberately removed the legacy `openclaw nodes run` wrapper and blocks `system.run` through `openclaw nodes invoke`; the only shipped node shell path is the agent exec tool with `host=node`. That means the reporter’s requested direct CLI remote-command/batch workflow is still not implemented, and it is a core node/exec CLI ergonomics and security-design question rather than a ClawHub/plugin cleanup item.

## Best Possible Solution

Keep this issue open for maintainer product judgment. A good implementation would add an explicit, security-reviewed CLI path for direct paired-node shell execution, likely under `openclaw nodes` or a clearly named exec CLI surface, while reusing the existing `exec host=node` approval policy, node resolver, timeout handling, and systemRunPlan protections instead of bypassing them.
