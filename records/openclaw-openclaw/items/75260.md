## What This Changes

The PR adds a Vesicle native iMessage channel plugin, signed webhook ingress, REST client wiring, docs, generated channel config metadata, secret-surface entries, and an install smoke script.

## Best Possible Solution

Keep OpenClaw core unchanged for this optional channel and publish Vesicle as an external ClawHub/npm plugin using the existing channel, setup, secret, HTTP route, webhook, SSRF, and runtime seams. Reopen core work only if external publication exposes a missing generic plugin API or maintainers explicitly decide Vesicle should become bundled.

## Reproduction Assessment

Not applicable for the additive channel request; the scope decision is based on current main policy/docs/source plus the PR diff. For the webhook finding, yes: configure two enabled Vesicle accounts with the same webhook path and secret, then send a valid signed POST and observe that the handler accepts the first matching account instead of rejecting ambiguity.

## Solution Assessment

No. Bundling Vesicle into OpenClaw core is not the narrowest maintainable path under the current plugin direction; the safer solution is an external plugin and a small webhook fix using the existing ambiguous-target helper before publication.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Reject ambiguous Vesicle webhook matches:** `extensions/vesicle/src/webhook.ts:206-212`
  - body: The handler uses `targets.find(...)` to pick the first target whose HMAC matches. If two Vesicle accounts reuse the same path and secret, a valid webhook is accepted for whichever account was registered first, which can misroute inbound messages and replies; resolve exactly one target and reject ambiguous matches instead.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff adds no new third-party dependency or CI execution path, but the new signed webhook ingress has an account-confusion risk on ambiguous same-path/same-secret matches.

Concerns:

- **[medium] Ambiguous webhook target match:** `extensions/vesicle/src/webhook.ts:206`
  - body: The Vesicle webhook handler authenticates by selecting the first target whose HMAC matches, so duplicate path/secret configuration can route a valid inbound event to the wrong account instead of failing closed.
  - confidence: 0.86
