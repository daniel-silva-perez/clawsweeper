## Summary

Keep this PR open. Current main still does not implement the intended explicit WhatsApp outbound bypass: explicit direct targets are still checked against `allowFrom`, the PR regression test is absent, and the related `allowTo` proposal is open but unmerged. The PR also changes a documented security boundary, so it needs maintainer product/security judgment rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer judgment, but do not merge it as-is. The best path is to decide the WhatsApp outbound authorization model alongside #49549, #25513, and #25039: either port an explicitly approved `mode: "explicit"` bypass into `extensions/whatsapp/src` with regression tests and docs, or land an opt-in `allowTo`/outbound policy and then close this PR as superseded after that replacement ships.
