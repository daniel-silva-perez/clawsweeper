## Summary

This is a valid optional context-policy idea, but current main already exposes the relevant context assembly and compaction lifecycle as a plugin-owned context-engine slot. OpenClaw policy keeps optional capabilities out of core when the plugin surface exists, so the conservative cleanup outcome is to close the core feature request and route a rolling/prioritized FIFO engine to ClawHub unless a plugin author identifies a specific missing API.

## What This Changes

Request an optional rolling/prioritized FIFO session context policy as an alternative to the built-in compaction behavior for long, tool-heavy sessions.

## Best Possible Solution

Keep OpenClaw core focused on the context-engine extension point, and implement a rolling/prioritized FIFO policy as a ClawHub-distributed context-engine plugin. A new core issue should be opened only if that plugin work finds a concrete missing lifecycle hook, token-budget field, or runtime helper.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. The high-confidence check is source and docs inspection, which shows current main already has a plugin-owned context assembly and compaction surface for custom policies.

## Solution Assessment

Yes: routing this to a context-engine plugin is the narrowest maintainable solution because it allows experimentation with rolling context without adding another built-in core policy. Core API work would be appropriate only for a specific extension gap discovered during plugin implementation.
