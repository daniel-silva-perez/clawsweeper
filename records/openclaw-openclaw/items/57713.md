## Summary

Close as implemented on main. The default Docker sandbox image path now fails fast instead of tagging plain Debian as `openclaw-sandbox:bookworm-slim`, the intended sandbox Dockerfile includes `python3`, regression coverage exists, docs describe the contract, and the fix shipped in `v2026.4.27`.

## What This Changes

The issue asks OpenClaw to ensure the default Docker sandbox image includes the Python runtime required by file mutation tools, or to fail earlier with clear build/repair guidance.

## Best Possible Solution

Keep the shipped `v2026.4.27` default-image guard and documented rebuild path. Operators with already-misbuilt local images should rebuild with `scripts/sandbox-setup.sh` and recreate stale sandbox containers; any extra missing-Python diagnostic work for already-created incompatible runtimes belongs in the narrower diagnostic follow-up rather than keeping this root-cause report open.

## Reproduction Assessment

Yes. The issue describes a deterministic missing-Python Docker sandbox path, and current main has a focused mocked Docker regression test for the missing default-image case; no live Docker mutation was run because this review was read-only.

## Solution Assessment

Yes. Fixing `ensureDockerImage()` to preserve the default image contract is the narrowest maintainable root-cause fix; changing the mutation helper or auto-installing Python inside read-only runtimes would be broader and less reliable.
