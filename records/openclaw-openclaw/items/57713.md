## Summary

Close as implemented. Current main no longer satisfies a missing default Docker sandbox image by tagging plain Debian, the default sandbox Dockerfile includes python3, regression coverage asserts the fail-fast behavior, and the fix shipped in v2026.4.27 via merged PR #73362.

## What This Changes

The issue reports that the default Docker sandbox image could lack Python, causing file edit/write mutation helpers to fail while read operations still worked.

## Best Possible Solution

Keep the shipped fail-fast default-image contract and documented rebuild path; operators with a stale locally tagged bad image should rebuild the default image with scripts/sandbox-setup.sh and recreate affected sandbox containers.

## Reproduction Assessment

Yes. The report gives concrete missing-python Docker sandbox symptoms, and current main has a focused unit reproduction for the missing default image path in src/agents/sandbox/docker.test.ts.

## Solution Assessment

Yes. Failing early with explicit build guidance is the narrow maintainable fix because it preserves the Python-bearing default image contract without silently pulling or retagging an unprepared base image at runtime.
