## Summary

Keep open: current main and latest v2026.4.26 still swallow Lobster workflow-file resolution errors and fall back to inline pipeline execution, while #68106 remains open as the explicit fix candidate. Closing before that PR or an equivalent focused fix lands would be premature.

## What This Changes

The report asks Lobster to treat relative workflow-file-like `pipeline` values as workflow paths and surface resolution/path errors instead of falling back to inline pipeline command parsing.

## Best Possible Solution

Review and land #68106, or an equivalent focused change in `extensions/lobster/src/lobster-runner.ts`, that rethrows workflow resolution errors for workflow-file-like inputs and adds regression coverage for missing relative workflow files while preserving normal inline pipelines.
