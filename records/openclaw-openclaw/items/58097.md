## Summary

Keep this PR open. Current main still has the Nextcloud Talk webhook signature verifier behavior the PR is meant to harden: an early return on signature-length mismatch followed by a manual XOR comparison. The PR is narrow, security-relevant, and the provided discussion reviewed the same implementation positively, but it has not been implemented on main.

## Best Possible Solution

Keep this PR open until maintainers either review and land this focused Nextcloud Talk hardening, or choose a canonical successor such as #50516. The desired fix belongs in the Nextcloud Talk plugin verifier, with a regression test that proves malformed short signatures do not take a length-mismatch early-return path.
