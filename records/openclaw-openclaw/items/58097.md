## What This Changes

The PR replaces Nextcloud Talk HMAC signature verification with padded `crypto.timingSafeEqual` and adds a regression test that asserts the compare path still runs for short signatures.

## Best Possible Solution

Land a rebased version of this narrow plugin-owned hardening on current main: keep the Nextcloud Talk signature logic local to the plugin, preserve current header-normalization changes, add the focused no-early-return regression test, and rerun the Nextcloud Talk extension lane before merge.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists on current main: a short non-empty signature reaches `signature.length !== expected.length` and returns `false` before any comparison, while same-length invalid signatures enter the manual XOR loop.

## Solution Assessment

No as a landing path until rebased; yes for the core runtime approach. The safest path is to apply the same padded `timingSafeEqual` comparison on current main while resolving the stale test/import conflict and preserving current Nextcloud Talk refactors.
