## Summary

Keep this PR open. Current `main` still has the Nextcloud Talk signature verifier's length-mismatch early return and manual XOR comparison, so the hardening remains relevant; the proposed patch is functionally sound from static review, but it needs maintainer/security review and a rebase/helper decision before landing.

## What This Changes

The PR replaces Nextcloud Talk webhook signature verification with padded `crypto.timingSafeEqual` and adds a regression test proving the compare path still runs for truncated signatures.

## Best Possible Solution

Land a narrow, rebased Nextcloud Talk signature hardening that removes the length-mismatch early return, uses the repository-approved constant-time string comparison path, and keeps the regression test for truncated signatures.

## Reproduction Assessment

Yes. Current `main` reproduces the issue by inspection: a short supplied signature exits at the length check before the compare path, and the PR's test provides a concrete truncated-signature guard.

## Solution Assessment

Unclear as the exact best implementation. The PR's padded `timingSafeEqual` approach is sound and matches LINE, but current `main` also has the plugin SDK `safeEqualSecret` helper already used by Feishu, so maintainers should choose the consistent path while rebasing.
