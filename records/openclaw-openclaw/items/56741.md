## Summary

Keep open. Current main still bypasses Telegram topic/group `groupPolicy` overrides for native slash commands, and this PR is a narrow implementation candidate rather than something already implemented or obsolete.

## What This Changes

This PR changes Telegram native command authorization so slash commands evaluate topic and group `groupPolicy` overrides before executing.

## Best Possible Solution

Land this patch or an equivalent change in the Telegram plugin with focused regression coverage for forum topics and groups where an override sets `groupPolicy: "disabled"` while the account-level policy would otherwise allow the command. Maintainers should also decide whether override-disabled commands should be silent in multi-bot forum topics or keep the existing visible disabled-command response.

## Reproduction Assessment

Yes. Configure a Telegram forum topic with `topicConfig.groupPolicy: "disabled"` while the account-level policy allows groups, then send `/reset`; current main skips the topic policy because the command path passes `useTopicAndGroupOverrides: false`.

## Solution Assessment

Yes for the authorization bypass: reusing the existing policy resolver with topic/group overrides enabled is the narrowest maintainable fix and matches the regular-message path. The remaining product choice is silent versus visible handling for disabled-topic native commands.
