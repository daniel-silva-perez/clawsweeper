## Summary

Current main still has the reported behavior: `resolveBuildRequirement()` returns `shouldBuild: true` for a dirty watched source tree before it reaches the source mtime freshness check. `pnpm gateway:watch` still starts through `scripts/watch-node.mjs` into `scripts/run-node.mjs`, and the existing test suite explicitly expects the dirty watched source tree to rebuild. The GitHub context also shows an open same-author implementation PR with closing syntax, so this issue should remain open until that PR or an equivalent fix is reviewed and merged or rejected.

## Best Possible Solution

Keep this issue open and review #57236 or an equivalent patch. The likely fix belongs in `scripts/run-node.mjs`: a dirty watched source tree should fall through to `hasSourceMtimeChanged()` instead of forcing a rebuild, with focused coverage in `src/infra/run-node.test.ts` for dirty-but-not-newer and dirty-and-newer source files.
