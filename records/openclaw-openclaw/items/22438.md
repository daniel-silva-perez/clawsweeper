## Summary

Keep open. Current main still lacks the requested config-backed bootstrap tier contract: there is no `bootstrapTier` schema/API/docs symbol, bootstrap filtering remains a fixed session allowlist plus per-run lightweight mode, and #22439 is still the open closing PR that should be reviewed rather than bypassed.

## What This Changes

The issue requests a configurable `agents.defaults.bootstrapTier` setting with `minimal`, `standard`, and `full` tiers for controlling which bootstrap files main, subagent, and cron sessions load.

## Best Possible Solution

Add an additive, documented bootstrap-tier contract that preserves current defaults, resolves the effective tier from session type plus config override, wires that tier through bootstrap filtering and hook-loaded extras, updates schema/types/help/docs, and adds focused tests for main, subagent, cron, lightweight, and extra-file behavior.
