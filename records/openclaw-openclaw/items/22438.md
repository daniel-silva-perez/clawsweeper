## Summary

Keep this issue open. Current main still lacks the requested schema-backed `agents.defaults.bootstrapTier` contract and tier resolver, and open PR #22439 is the paired implementation candidate with closing syntax for this issue. The original binary-loading description is partly stale because current main now gives subagent/cron sessions a broader allowlist and has per-run lightweight context, but the configurable `minimal | standard | full` bootstrap policy remains unimplemented.

## What This Changes

Add configurable tiered workspace bootstrap loading through `agents.defaults.bootstrapTier` with `minimal`, `standard`, and `full` behavior for main, subagent, cron, and extra bootstrap files.

## Best Possible Solution

Keep this issue open while maintainers review, land, or close PR #22439. A complete solution should add an additive `agents.defaults.bootstrapTier` contract, wire tier resolution through `src/agents/workspace.ts`, `src/agents/bootstrap-files.ts`, and the `bootstrap-extra-files` hook, update schema/types/generated metadata/help/docs, preserve current default behavior, define `full` and extra-file semantics, and add focused tests for main, subagent, cron, lightweight, and hook-extra interactions.
