Keep open. Current main still lacks the requested config-backed `bootstrapTier` schema and resolver, and the issue is already paired with an open closing implementation PR, so cleanup should wait for maintainer review, landing, replacement, or closure of that PR.

## What This Changes

The issue requests configurable tiered bootstrap loading via `agents.defaults.bootstrapTier` with `minimal`, `standard`, and `full` tiers for main, subagent, and cron session context control.

## Best Possible Solution

Add an additive, documented bootstrap-tier contract that resolves the effective tier from session type plus config override, preserves current defaults, wires the result through bootstrap filtering and hook-loaded extras, updates schema/types/help/docs, and covers main, subagent, cron, lightweight, and extra-file behavior with focused tests.
