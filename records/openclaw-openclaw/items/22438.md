Keep open. Current main still lacks the requested `agents.defaults.bootstrapTier` schema, resolver, and tier-aware filtering, and the issue is paired with open closing PR #22439, so maintainers should review, land, replace, or close that PR before resolving this issue.

## What This Changes

The issue requests configurable tiered bootstrap loading so main, subagent, and cron sessions can choose minimal, standard, or full bootstrap file sets.

## Best Possible Solution

Review and either land or deliberately replace #22439 with an additive, documented bootstrap-tier contract that preserves current defaults, resolves the effective tier from session type plus config override, wires tier selection through bootstrap filtering and hook-loaded extras, updates schema/types/help/docs, and adds focused coverage for main, subagent, cron, lightweight, and extra-file behavior.
