## Summary

Keep open. The requested config-backed bootstrap tier surface is still absent from current main: there is no `bootstrapTier` schema/docs/API symbol, current filtering is still driven by a fixed session allowlist, and #22439 remains the open closing PR that should be reviewed rather than bypassed. The follow-up discussion narrowed the still-missing behavior after `lightContext` and broader subagent defaults landed, so this is an active product/API follow-up, not stale cleanup.

## What This Changes

The issue proposes an `agents.defaults.bootstrapTier` option with `minimal`, `standard`, and `full` tiers to control which workspace bootstrap files load for main, subagent, cron, and extra-file contexts.

## Best Possible Solution

Add an additive, documented bootstrap-tier contract that preserves current defaults, resolves the effective tier from session type plus config override, wires that tier through bootstrap filtering and hook-loaded extras, updates schema/types/help/docs, and adds focused tests for main, subagent, cron, lightweight, and extra-file behavior.
