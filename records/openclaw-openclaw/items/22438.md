## Summary

Keep open. Current main still lacks the requested `agents.defaults.bootstrapTier` config/schema/docs surface and tier-aware bootstrap resolver, and open PR #22439 is already the direct implementation candidate with closing syntax for this issue.

## What This Changes

The issue requests configurable tiered workspace bootstrap loading so main, subagent, and cron sessions can choose `minimal`, `standard`, or `full` bootstrap context instead of relying only on hard-coded session defaults.

## Best Possible Solution

Review and either land or deliberately replace open PR #22439 with an additive, documented bootstrap-tier contract that preserves current defaults, resolves effective tier from session type plus config override, wires tier selection through bootstrap filtering and hook-loaded extras, updates schema/types/help/docs, and adds focused coverage for main, subagent, cron, lightweight, and extra-file behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: current main has no tier symbols, no schema key, and no resolver path that reads a bootstrap-tier config value.

## Solution Assessment

Unclear until maintainer review of open PR #22439 and the adjacent extra-file work. The requested config-driven tier surface is plausible, but the safest solution is an additive contract coordinated with existing lightweight context and hook behavior.
