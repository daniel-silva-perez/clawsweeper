## Summary

Keep open. Current main still only seeds boundary-checked bootstrap files, while this PR expands the sandbox filesystem boundary and has unresolved P1 security findings around trusted-root validation and hardlink aliases. Because `src/agents/sandbox/**` is CODEOWNERS security-sensitive, this needs maintainer/secops review rather than conservative cleanup closure.

## What This Changes

The PR adds top-level symlink authority hydration in `ensureSandboxWorkspace` for allowlisted names such as `README.md`, `CLAUDE.md`, `docs`, and `voro-docs`, plus tests for materialization, allowlisting, nested symlink skipping, and non-symlink exclusion.

## Best Possible Solution

Define a documented authority-link model before merging: explicit supported names, resolved-target validation against the intended trusted roots, symlink and hardlink rejection throughout copied trees, deliberate refresh behavior for reused sandboxes, Windows-safe tests, and docs that explain exactly which authority files can be hydrated.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Validate authority links before copying:** `src/agents/sandbox/workspace.ts:86-91`
  - body: `fs.realpath(src)` resolves an allowlisted workspace name and then copies that resolved target without checking it is inside the seed workspace or another documented trusted root. In non-rw sandbox modes, `README.md` or `docs` can point at an arbitrary host path and make that content readable under `/workspace`.
  - confidence: 0.93
- **[P1] Reject hardlinks while copying authority trees:** `src/agents/sandbox/workspace.ts:56-57`
  - body: `lstat()` plus `copyFile()` treats hardlinked files as ordinary files. The current bootstrap path rejects hardlink aliases through `openBoundaryFile`, but this new recursive copy path would copy a same-filesystem hardlink inside an allowed authority tree into the sandbox.
  - confidence: 0.9
- **[P2] Refresh existing authority copies:** `src/agents/sandbox/workspace.ts:40-42`
  - body: `copyAuthorityTreeIfMissing` returns as soon as the destination exists. Agent-scoped sandboxes are reused, so later source changes or sandbox-side edits can leave future sessions reading stale authority content instead of the current workspace docs.
  - confidence: 0.86
- **[P2] Guard symlink tests on Windows:** `src/agents/sandbox.workspace.test.ts:25-34`
  - body: The new tests create file and directory symlinks unconditionally. Nearby sandbox alias tests are skipped on `win32` because Windows runners may lack symlink privileges, so these cases can fail the Windows lane before reaching the intended assertions.
  - confidence: 0.88
