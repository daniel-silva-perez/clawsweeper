## Summary

Keep this PR open for maintainer review. Current main does not implement authority-link hydration, and the proposed branch changes the sandbox filesystem boundary with unresolved trust-model and security questions around resolved symlink targets, hardlinks, refresh behavior, Windows coverage, and documentation.

## What This Changes

The PR adds sandbox workspace hydration for allowlisted top-level authority symlinks such as `README.md`, `CLAUDE.md`, `docs`, and `voro-docs`, plus unit tests for materialization, allowlisting, and nested symlink skipping.

## Best Possible Solution

Define a documented authority-link model for sandbox seed workspaces before merging: explicit supported names, resolved-target validation against the intended root policy, symlink and hardlink rejection throughout copied trees, deliberate refresh behavior for reused sandboxes, Windows-safe tests, and docs that explain exactly which authority files can be hydrated.
