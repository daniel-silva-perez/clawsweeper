## Summary

Current main still does not implement sandbox authority-link materialization, and the PR remains potentially useful but security-sensitive. Its latest diff copies resolved authority symlink targets into the sandbox without the same root/alias guarantees used by current bootstrap seeding, so this should stay open for maintainer/security review rather than be auto-closed or queued as an automated repair.

## What This Changes

This PR adds sandbox workspace hydration for allowlisted top-level authority symlinks such as README.md, CLAUDE.md, docs, and voro-docs, plus tests for materialization, allowlisting, and nested symlink skipping.

## Best Possible Solution

Keep this PR open for maintainer/security review. A mergeable version should define the authority-link trust model, reuse or match openBoundaryFile-style root and alias checks where practical, validate resolved targets against an explicit root policy, reject hardlinks, handle reused-sandbox refresh behavior deliberately, gate symlink tests on Windows, and document the supported authority names.
