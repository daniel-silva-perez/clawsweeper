## Summary

Keep this PR open. Current main still has final-tag filtering machinery and some phase-aware commentary suppression, but it does not implement the PR's intended user-facing `agents.defaults.enforceFinalTag` option or propagate an explicit config opt-in through interactive and cron agent runs. The linked leakage report #25592 remains a real security-adjacent delivery concern, and the PR discussion includes useful independent validation from a local backport.

## Best Possible Solution

Review and land this PR, or an equivalent patch, that adds `agents.defaults.enforceFinalTag` to the exported config type and strict schema, documents it on the public agents configuration page, propagates it through interactive reply and cron runs, enables the system prompt tag hint from the explicit flag, and keeps focused regression coverage for provider-gated defaults plus config opt-in. Keep #25592 open until an implementation merges.

## Risks / Open Questions

- The remaining behavior is security-adjacent: external messaging channels can expose unintended assistant text when non-tag providers emit plain text between tool calls.
- The opt-in can suppress visible replies if a model fails to follow the `<final>` contract, so the final patch should keep the default off and document the failure mode clearly.
- Current main includes phase-aware commentary suppression, but that only helps when provider output carries `commentary`/`final_answer` phase metadata; it is not equivalent to this PR's explicit final-tag opt-in for unphased plain text.
