## Summary

Keep this PR open. Current main has the lower-level final-tag filtering path and run-level `enforceFinalTag` plumbing, but it still does not implement the PR's user-facing `agents.defaults.enforceFinalTag` config option or propagate a config opt-in through interactive and cron runs. The linked leakage report #25592 is still open, and the PR discussion includes useful independent backport validation.

## Best Possible Solution

Review and land this PR, or an equivalent patch, that adds `agents.defaults.enforceFinalTag` to the exported config type and strict schema, documents it on `docs.openclaw.ai/gateway/config-agents`, propagates it through interactive reply and cron runs, enables the system prompt tag hint from the explicit flag, and keeps focused regression coverage for both provider-gated defaults and config opt-in. Keep #25592 open until an implementation merges.
