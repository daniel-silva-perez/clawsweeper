## Summary

Keep this PR open. Current main still has the lower-level `enforceFinalTag` filtering and run-param plumbing, but it does not implement the PR's user-facing `agents.defaults.enforceFinalTag` config option or propagate a config opt-in through interactive and cron runs. The PR discussion also contains useful production/backport validation, and the linked leakage issue remains an active product bug rather than something already shipped on main.

## Best Possible Solution

Review and land this PR, or an equivalent patch, after rebasing it onto current main. The implementation should add `agents.defaults.enforceFinalTag` to the exported config type, strict Zod schema, generated schema/help surfaces, and public docs at `docs.openclaw.ai/gateway/config-agents`; propagate the flag through interactive reply and current cron executor paths; make `reasoningTagHint` honor the explicit run flag; and keep focused tests for provider-gated defaults plus config opt-in. Keep the linked leakage issue open until that implementation merges.
