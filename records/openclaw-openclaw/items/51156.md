## Summary

Keep this PR open. Current main has the lower-level `enforceFinalTag` run parameter and subscriber filtering, but it still does not implement the PR's user-facing `agents.defaults.enforceFinalTag` config option or propagate a config opt-in through interactive and cron runs. The PR also has useful discussion evidence: a reviewer noted the shape was correct after docs were added, and Mdx2025 reported a local 2026.3.24 backport solving the Anthropic plain-text reasoning leak described in #25592.

## Best Possible Solution

Review and land this PR, or an equivalent rebased patch, rather than closing it. The implementation should add `agents.defaults.enforceFinalTag` to the exported config type, strict Zod schema, generated/config help surfaces, and public docs; propagate the flag through interactive reply and the current cron executor path; make `reasoningTagHint` honor explicit enforcement; and keep focused tests for provider-gated defaults plus config opt-in. Keep #25592 open until the implementation merges.
