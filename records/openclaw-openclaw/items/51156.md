## Summary

Keep this PR open. Current main has the final-tag filtering machinery and a run-level `enforceFinalTag` helper, but it still does not implement the PR's user-facing `agents.defaults.enforceFinalTag` config option or propagate that config opt-in through interactive and cron runs. The linked leakage report #25592 remains open, and the PR discussion includes useful independent local backport validation.

## Best Possible Solution

Review and land this PR, or an equivalent patch, that adds `agents.defaults.enforceFinalTag` to the exported config type and strict schema, documents it on the public agents configuration page, propagates it through interactive reply and cron runs, enables the system prompt tag hint from the explicit flag, and keeps focused regression coverage for both provider-gated defaults and config opt-in. Keep #25592 open until an implementation merges.
