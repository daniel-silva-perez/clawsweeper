Keep open. Current main still lacks the DBus/XDG environment recovery described by the report, and the provided context shows an open, unmerged implementation candidate with closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that `openclaw gateway status` and `openclaw gateway install` correctly detect an active user systemd service when direct `systemctl --user` works but OpenClaw reports DBus permission-denied unavailability.

## Best Possible Solution

Review and land #73888 or an equivalent narrow fix in `src/daemon/systemd.ts` that derives missing Linux user-scope DBus/XDG environment for `systemctl --user` spawns, preserves the intended user scope, and adds regression coverage for gateway status and install.
