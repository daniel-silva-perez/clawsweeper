## Summary

Keep open. Current main still does not implement the env recovery described by the exact matching fix attempt, and the current tests/source still treat `systemctl --user` bus `Permission denied` as unavailable rather than proving the enabled/active service state the reporter observed.

## Best Possible Solution

Implement the missing user-systemd spawn recovery in `src/daemon/systemd.ts`: preserve existing env, derive `XDG_RUNTIME_DIR=/run/user/<uid>` and `DBUS_SESSION_BUS_ADDRESS=unix:path=/run/user/<uid>/bus` when absent, wire that env into all `systemctl --user` spawns, and add focused regression coverage for `gateway status` and `gateway install` with a healthy user service but broken inherited bus env. Separately, consider landing the narrower `not found` classifier behavior from #63625 if unit-level not-found false positives are still in scope.
