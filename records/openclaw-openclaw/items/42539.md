## Summary

Keep this issue open. Current main still lacks a configurable Telegram TTS delivery mode; normal auto-TTS final replies keep visible text on the payload while attaching audio, and Telegram still turns short text into a voice/audio caption, splitting only overflow or special video-note paths.

## Best Possible Solution

Keep this issue open. Implement a Telegram-owned delivery policy or shared messages.tts.delivery: "bundled" | "separate" contract, then update strict config types/schema, Telegram bot and direct send paths, tests, generated/public config docs, https://docs.openclaw.ai/tools/tts, and https://docs.openclaw.ai/channels/telegram. In separate mode, send visible text as a normal Telegram message first, then send the voice note without a caption.
