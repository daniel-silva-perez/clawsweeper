## Summary

Keep this issue open. Current main has useful context, skill, tool-policy, pruning, compaction, local-model, plugin, and workflow controls, but it still does not answer the maintainer guidance being requested: tested high-count tool/skill ceilings, SKILL.md trigger-collision practices, workload-specific tuning defaults, low-end local-model reference profiles, and an explicit recommendation on hybrid aggregation tools.

## Best Possible Solution

Keep open and convert the useful maintainer answer into a focused docs task. The best path is a practical guide under Context, Skills, Tools/plugins, or Local models that states whether tested tool/skill-count ceilings exist; gives SKILL.md trigger-collision practices; recommends concrete use of /context detail, skills.limits, agents.defaults.skills, tools.profile, tools.allow/deny, loopDetection, pruning, compaction, tool-result caps, and localModelLean for high-count sessions; and explicitly describes when a hybrid domain aggregation plugin or workflow-backed tool is acceptable, including traceability, latency, output-size, and composability tradeoffs.
