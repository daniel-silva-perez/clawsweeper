## Summary

Keep open. Current main does not implement the requested completeness-scan prompt section, and this PR remains the active implementation candidate for the linked feature request, but the branch is not merge-ready because it adds a new provider prompt-section id without aligning the runtime-plan/harness contract and it leaves the new user-facing prompt surface undocumented.

## What This Changes

The PR adds a mandatory Completeness Scan system-prompt section for agent read/search operations, makes it provider-overridable with a new `completeness_scan` section id, and adds guardrail tests for full, minimal, none, and provider override modes.

## Best Possible Solution

Keep this PR as the implementation candidate for #73470, then make it mergeable by adding `completeness_scan` to the runtime-plan prompt-section contract with focused contract tests, updating the system-prompt/provider docs and changelog, and resolving the prompt overlap with `execution_bias`.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature PR. The blocking contract issue is reproducible by comparing the PR-added provider section id with current `AgentRuntimeSystemPromptSectionId`, which still omits `completeness_scan`.

## Solution Assessment

No, not yet. A dedicated prompt section is a plausible narrow implementation, but the branch should keep provider and runtime-plan prompt-section contracts aligned and document the new public prompt surface before it is the best mergeable solution.
