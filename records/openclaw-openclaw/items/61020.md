## Summary

Keep open. Current main has not implemented this PR's central payload-path fix: it still derives `lastAssistant` from the whole session snapshot and passes that session-level assistant into `buildEmbeddedRunPayloads`. Main has a newer `currentAttemptAssistant` helper for some failover/error-context paths, but terminal payload construction still can use stale visible text when `assistantTexts` is empty. The PR also needs revision because the direct `slice(prePromptMessageCount)` approach has a real compaction risk already called out in review.

## Best Possible Solution

Keep the PR open, but revise it rather than merging as-is. The robust fix should route a compaction-aware current-turn assistant into the user-visible payload path, preserve a deliberate session-level fallback only where compaction/error handling requires it, and add regression coverage for both zero-token idle timeout with prior visible assistant history and compaction cases where the current assistant must not be lost.
