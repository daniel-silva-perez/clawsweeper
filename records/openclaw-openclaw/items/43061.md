## Summary

Keep this PR open. Current main already carries the directive-tag indentation protection and the model-scan optional-chaining hunk, but it still does not implement the minimal-profile tool policy fix: explicit tools.fs or tools.exec sections are still converted into profileAlsoAllow without checking whether the resolved profile is minimal. The PR branch remains useful after a rebase and narrowing pass for #42165.

## Best Possible Solution

Keep this PR open, rebase it on current main, and narrow it to the remaining #42165 policy bug. The best patch should pass the resolved profile into implicit tool-section expansion, suppress implicit fs and exec/process additions when the profile is minimal, audit resolveEffectiveToolFsRootExpansionAllowed, and add regression tests for global and agent-scoped tools.fs, tools.exec, and both together under minimal. Drop the directive-tag and model-scan hunks that current main already covers.
