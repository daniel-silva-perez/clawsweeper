## Summary

Keep this PR open. Current main already carries the directive-tag whitespace preservation and model input hardening paths, but the tools.profile "minimal" bug remains: explicit tools.fs or tools.exec config still becomes profileAlsoAllow without checking whether the selected profile is minimal.

## Best Possible Solution

Keep this PR open, rebase it on current main, and narrow it to the remaining #42165 policy bug. The best patch should pass the resolved profile into implicit tool-section expansion, suppress implicit fs and exec/process additions when the profile is minimal, audit the analogous fs root-expansion helper, and add regression tests for tools.fs, tools.exec, and both together under minimal. Drop the already-shipped directive-tag and model-scan hunks from the refreshed PR.
