## Summary

Keep open. The PR targets a real current-main queue bug, but the proposed restore helper appends older deferred events after any uninspected tail queued during the heartbeat run, so it can violate FIFO ordering and needs a small correction before merge.

## What This Changes

The branch adds a system-event restore helper, changes exec-heartbeat cleanup to restore non-exec inspected events after consuming the inspected prefix, and adds regression coverage for mixed exec/non-exec queues and duplicate deferred reminders.

## Best Possible Solution

Keep the narrow deferred-event preservation fix, but implement restoration so the deferred inspected entries remain ahead of any uninspected tail, only restore after `consumeSystemEventEntries` successfully removes the inspected prefix, and keep duplicate reminders plus `MAX_EVENTS` metadata behavior covered by focused tests.

## Reproduction Assessment

Yes. A deterministic unit path is to snapshot `[Reminder A, exec E]`, enqueue `Reminder B` before cleanup, then consume the inspected prefix and restore deferred entries; the PR's append-based helper yields `[Reminder B, Reminder A]` instead of preserving FIFO order.

## Solution Assessment

No. Restoring deferred entries is the right direction, but appending them after the current queue is not the best implementation; a safer helper should rebuild or prepend the deferred inspected entries ahead of the uninspected tail and guard on successful prefix consumption.

## Review Findings
