## Summary

Keep open. Current main still has the mixed exec/non-exec queue-consumption bug this PR is trying to fix, but the proposed restore helper would append deferred entries after any later queued tail, so the patch can reorder older reminders behind newer events.

## What This Changes

The PR adds a system-event restore helper, changes exec-heartbeat cleanup to restore non-exec inspected events after consuming the inspected prefix, and adds regression tests for mixed queues plus duplicate deferred reminders.

## Best Possible Solution

Keep the narrow deferred-event preservation direction, but restore deferred inspected entries ahead of any uninspected tail, restore only after the inspected prefix was actually consumed, and keep focused tests for duplicate reminders, ordering, and queue metadata behavior.

## Reproduction Assessment

Yes. A deterministic unit path is to snapshot `[Reminder A, exec E]`, enqueue `Reminder B` before cleanup, then consume the inspected prefix and restore deferred entries; the PR's append-based helper yields `[Reminder B, Reminder A]` instead of preserving FIFO order.

## Solution Assessment

No as written. Restoring non-exec entries is the right direction, but appending them after the current queue is not the narrowest correct implementation; a safer helper should rebuild/prepend the deferred inspected entries ahead of any uninspected tail and guard on successful prefix consumption.
