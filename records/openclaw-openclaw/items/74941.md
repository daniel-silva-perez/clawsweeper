## Summary

Keep open. The PR addresses a real current-main bug where exec-heartbeat cleanup can consume mixed exec and non-exec system events, but the proposed restore helper appends deferred entries after any later queued tail, which can reorder older reminders behind newer events.

## What This Changes

The branch adds a system-event restore helper, changes exec-heartbeat cleanup to restore non-exec inspected events after consuming the inspected prefix, and adds focused regression tests for mixed exec/non-exec queues and duplicate deferred reminders.

## Best Possible Solution

Keep the narrow deferred-event preservation direction, but restore deferred inspected entries ahead of the uninspected tail, gate restoration on successful prefix consumption, and cover duplicate reminders, tail ordering, and `MAX_EVENTS` metadata behavior with focused tests.

## Reproduction Assessment

Yes. A deterministic unit path is to snapshot `[Reminder A, exec E]`, enqueue `Reminder B` before cleanup, then consume the inspected prefix and restore deferred entries; the PR's append-based helper yields `[Reminder B, Reminder A]` instead of preserving FIFO order.

## Solution Assessment

No as written. Restoring non-exec entries is the right direction, but appending them after the current queue is not the narrowest correct implementation; the safer helper should rebuild/prepend the deferred inspected entries ahead of any uninspected tail and restore only after consume succeeds.
