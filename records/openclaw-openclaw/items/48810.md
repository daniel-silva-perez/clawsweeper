## Summary

Keep this open. Current main documents transcripts as a tree and core history readers follow the active SessionManager branch, so OpenClaw itself does not rely on naive same-parent child selection. However, the compaction retry path still does not supersede, deterministically replace, or reconcile duplicate retry compaction entries, and current tests tolerate multiple retry-end events plus preserved sibling branches.

## What This Changes

The issue requests a compaction retry integrity guard so retry compaction cannot create duplicate same-parent compaction entries that break downstream parentId-chain reconstruction.

## Best Possible Solution

Maintainers should decide the transcript contract explicitly: either add a compaction-retry reconciliation mechanism with regression coverage for duplicate same-parent compaction entries, or document and expose the active-branch reader contract clearly enough that downstream tools do not treat raw parentId as a globally linear list.

## Reproduction Assessment

Unclear. The issue includes concrete raw JSONL showing the duplicate same-parent compaction shape, but the reporter says they cannot reproduce it and this read-only review did not run a live compaction retry scenario.

## Solution Assessment

Unclear. The current active-branch behavior is appropriate for OpenClaw's documented tree model, but it is not a complete solution if maintainers want raw JSONL parentId chains to remain linear across compaction retries.
