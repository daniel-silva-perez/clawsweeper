## Summary

Keep this issue open. Current main documents and implements Pi transcripts as a tree/DAG and OpenClaw’s own history surfaces follow the active branch, so a naive linked-list walker is not the core reader model. However, the requested compaction-retry integrity guard is still not implemented: retry compaction entries are not marked superseded or made deterministic, successor transcript rotation remains opt-in, and tests intentionally preserve sibling branches rather than rejecting or reconciling duplicate compaction siblings.

## Best Possible Solution

Keep this open for maintainer product/API judgment. The best fix is either a compaction-specific reconciliation contract, such as marking retry-superseded compaction entries or making retry compaction IDs deterministic, plus a regression test for duplicate same-parent compaction entries; or an explicit documented contract that downstream readers must use SessionManager/active-branch traversal and must not interpret parentId as a globally linear list.

## Risks / Open Questions

- Raw archived JSONL can still contain same-parent sibling branches, so downstream systems that assume a linear linked list remain exposed unless they adopt active-branch traversal or OpenClaw adds explicit supersession metadata.
- There is no targeted regression test for duplicate sibling compaction entries with identical firstKeptEntryId/tokensBefore, so the exact reported shape could regress or remain ambiguous.
