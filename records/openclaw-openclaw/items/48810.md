## Summary

Keep this issue open. Current main does not show the simple in-repo replay failure described by a naive child-walker: OpenClaw/Pi treats session JSONL as a tree/DAG and core history surfaces follow the active leaf branch. But the exact requested integrity guard is not implemented: retry-created sibling compaction entries are not marked superseded or made deterministic, successor transcript rotation is opt-in/default false, and tests explicitly preserve sibling branches. The report has concrete JSONL and follow-up analysis, so this needs maintainer product judgment rather than conservative auto-close.

## Best Possible Solution

Keep open for maintainer decision. The likely best path is either a compaction-specific reconciliation contract, such as marking retry-superseded compaction entries or making retry compaction ids deterministic, plus a regression test; or an explicit docs/API decision that downstream parsers must use leaf-aware tree traversal instead of assuming a linear parentId list.
