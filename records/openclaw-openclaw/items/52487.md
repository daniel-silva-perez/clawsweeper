## Summary

Keep this PR open. Current main still has the Windows scheduled-task self-restart retry path without an already-running guard, so the branch addresses a real remaining race, but the proposed PowerShell probe is not ready to merge because it embeds the task name in a batch command without cmd-safe quoting and runs PowerShell profiles before the status check.

## What This Changes

The PR adds a PowerShell `Get-ScheduledTask` running-state probe to `src/infra/windows-task-restart.ts` before the Windows self-restart helper retries `schtasks /Run`.

## Best Possible Solution

Keep the narrow Windows restart fix, but revise the helper to run a profile-free, noninteractive PowerShell state check with the task name passed through existing cmd-safe assignment/quoting or the same validation used by the update restart helper. Add focused tests for normal task names, custom task names with cmd metacharacters, and the generated guard ordering before rerunning the targeted Windows restart tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Quote the task name before the PowerShell probe:** `src/infra/windows-task-restart.ts:47`
  - body: The new probe embeds `taskName` directly in a `.cmd` line and only doubles PowerShell single quotes. Values from `OPENCLAW_WINDOWS_TASK_NAME` or profile-derived task names containing `%`, `!`, or `"` are expanded or can break the outer `-Command` string before PowerShell sees them, so the guard can miss the already-running task or execute unintended batch text. Pass the task name through a cmd-safe variable/assignment or validate it like the update restart helper.
  - confidence: 0.88
- **[P2] Run the probe without PowerShell profiles:** `src/infra/windows-task-restart.ts:47`
  - body: `powershell -Command` loads user/system profiles before `Get-ScheduledTask`. Profile output containing `Running` can make `findstr` take the cleanup path even when the task is not running, and slow or interactive profile code can block the self-restart helper before it reaches `/Run`. Add `-NoProfile` and preferably `-NonInteractive` for this generated restart probe.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff adds a PowerShell-in-batch execution path whose task-name interpolation is not cmd-safe under crafted local task/profile names.

Concerns:

- **[low] Unescaped task name in generated batch command:** `src/infra/windows-task-restart.ts:47`
  - body: The new PowerShell probe interpolates the task name inside a `.cmd` command without escaping cmd metacharacters or validating the Windows task name. A crafted local `OPENCLAW_WINDOWS_TASK_NAME` or profile-derived task name could alter the generated batch command when the restart helper runs, so the probe should pass the value through a safe environment assignment or reject unsafe names before embedding it.
  - confidence: 0.78
