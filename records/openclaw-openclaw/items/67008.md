## Summary

Keep open. Current main does not include `zai-org/GLM-5.1-TEE` in the bundled Chutes static catalog, manifest catalog, or provider docs, and the static fallback path is still the relevant behavior when Chutes model discovery fails. The PR is focused provider-catalog work and is not obsolete; it likely needs matching manifest/docs updates and a small fallback-catalog assertion before merge.

## Best Possible Solution

Keep this PR open for normal review. The best path is to add `zai-org/GLM-5.1-TEE` to the bundled Chutes static catalog while keeping `CHUTES_DEFAULT_MODEL_ID` unchanged, mirror the entry into `extensions/chutes/openclaw.plugin.json` and `docs/providers/chutes.md`, and add a focused test assertion that the fallback/static catalog includes the new model id.
