## Summary

Keep this issue open. Current main still provides memory search, promotion/dreaming review output, Active Memory recall injection, memory-wiki companion metadata workflows, and LanceDB recall/store/forget tools, but it does not provide the requested first-class operator review/list, edit, non-destructive forget/tombstone or supersede, and conflict-resolution flows for canonical durable memory records. The same-author Memory MVP schema/write/surface items remain open, so this is active product work rather than conservative cleanup.

## Best Possible Solution

Keep this issue open and implement it with the memory owner modules alongside #42646, #42648, and #42651. The concrete path is a canonical durable memory record/status/revision model plus operator-facing review/list, edit, non-destructive forget/tombstone or superseded transitions, and conflict-resolution flows exposed through CLI/RPC or UI, with docs and focused tests.
