## Summary

Keep this issue open. Current main still has memory search, short-term promotion/dreaming review output, Active Memory recall injection, a companion memory-wiki layer, and LanceDB recall/store/forget tools, but it still lacks the requested first-class operator review/list, edit, non-destructive forget/tombstone or supersede, and conflict-resolution flows for canonical durable memory records. The fresh KinthAI production-context comment supports the requested design direction rather than making the OpenClaw work obsolete.

## Best Possible Solution

Keep this issue open and implement it with the memory owner modules alongside #42646, #42648, and #42651. The concrete path is a canonical durable memory record/status/revision model plus operator-facing review/list, edit, non-destructive forget/tombstone or superseded transitions, and conflict-resolution flows exposed through CLI/RPC or UI, with docs and focused tests.
