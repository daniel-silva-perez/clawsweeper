## Summary

Keep this issue open. Current main still has memory search, promotion/dreaming review output, a companion memory-wiki layer, and LanceDB recall/store/forget tooling, but it does not provide the requested first-class operator review/list, edit, non-destructive forget/tombstone or supersede, and conflict-resolution flows for canonical durable memory records. The same-author Memory MVP schema/write/surface issues remain open, so this is active product work rather than conservative cleanup.

## Best Possible Solution

Keep this issue open and implement it in the memory owner modules alongside #42646, #42648, and #42651. The concrete path is a canonical durable memory record/status/revision model plus operator-facing review/list, edit, non-destructive forget/tombstone or superseded transitions, and conflict-resolution flows exposed through CLI/RPC or UI, with docs and focused tests.
