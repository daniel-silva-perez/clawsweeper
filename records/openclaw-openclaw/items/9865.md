## Summary

Keep this issue open. Current main still lacks first-class Batch API routing for delayed/background LLM agent turns. Existing batch-related code is limited to memory embeddings and media-style jobs, while cron agent turns, sessions_send, sessions_spawn/subagents, heartbeat, and provider plugin contracts still route through immediate stream-style execution. Related #49326, #56126, and #70606 were closed as narrower duplicates/context for this broader tracker, not as shipped fixes.

## Best Possible Solution

Keep this as the canonical product/API tracker. The best path is an additive provider/runtime batch-turn contract, followed by explicit opt-in fields for cron agentTurn jobs, heartbeat/background runs, sessions_send, and sessions_spawn/subagents, plus provider plugin implementations, docs, config/schema updates, and focused regression coverage.
