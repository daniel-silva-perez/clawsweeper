## Summary

Keep open. Current main still lacks first-class provider Batch API routing for delayed/background LLM turns; the latest maintainer/Codex comment treats this as the canonical tracker, and the closed related issues are context rather than replacements. Existing batch code is limited to memory embeddings/media-style batching, while cron, sessions_send, subagent runtime, and provider hooks have no batch-turn submit/poll/cancel/result contract.

## Best Possible Solution

Keep this issue open as the canonical product/API tracker. The best path is an additive provider/runtime batch-turn contract, then explicit opt-in fields for cron agentTurn jobs, heartbeat/background runs, sessions_send, and sessions_spawn/subagents, with OpenAI/Anthropic provider plugin implementations, docs, and focused regression coverage.
