## Summary

Keep open. Current main still lacks first-class Batch API routing for delayed/background agent turns: cron, sessions_send, sessions_spawn/subagents, heartbeat, and provider runtime surfaces remain immediate or stream-oriented, while existing batch support is limited to memory embeddings and media-style provider paths. The prior discussion also treats this as the canonical tracker, with #49326, #56126, and #70606 as narrower closed context rather than shipped fixes.

## What This Changes

Request support for opting background OpenClaw agent work, such as cron jobs, heartbeats, scheduled maintenance, and subagents, into provider Batch APIs for lower-cost delayed execution.

## Best Possible Solution

Keep this as the canonical product/API tracker for an additive delayed/batch agent-turn contract, with public SDK/provider runtime support, protocol/config opt-ins for cron, sessions_send, sessions_spawn/subagents, and heartbeat/background runs, plus OpenAI and Anthropic provider implementations and docs for fallback, cancellation, accounting, and delayed delivery.
