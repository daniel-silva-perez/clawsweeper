## Summary

Keep open. Current main still lacks first-class Batch API routing for delayed/background LLM agent turns, and the issue discussion already treats this as the canonical tracker while the related closed reports are narrower duplicate context rather than shipped fixes.

## What This Changes

Add opt-in provider Batch API routing for non-real-time background agent work such as cron jobs, subagents, sessions_send, and heartbeat tasks.

## Best Possible Solution

Keep this as the canonical product/API tracker for an additive delayed/batch agent-turn contract: public SDK/provider runtime support, protocol/config opt-ins for cron, sessions.send, sessions_spawn/subagents, and heartbeat/background runs, plus provider implementations and docs for fallback, cancellation, accounting, provider eligibility, and delayed delivery.
