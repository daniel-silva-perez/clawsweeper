## Summary

Keep open. Current main still lacks first-class Batch API routing for delayed/background agent turns, and the issue discussion already treats this as the canonical tracker rather than something fixed by the related closed reports.

## What This Changes

Add support for opting non-real-time background work such as cron jobs, spawned subagents, and heartbeat checks into provider Batch APIs for lower-cost delayed execution.

## Best Possible Solution

Keep this as the canonical product/API tracker for an additive delayed/batch agent-turn contract: public SDK/provider runtime support, protocol/config opt-ins for cron, sessions_send, sessions_spawn/subagents, and heartbeat/background runs, plus OpenAI and Anthropic provider implementations and docs covering fallback, cancellation, accounting, and delayed result delivery.
