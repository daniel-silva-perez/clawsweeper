## Summary

Keep open. Current main still lacks first-class Batch API routing for delayed/background agent turns: existing batch support is limited to memory/media-style paths, while cron, sessions_send, sessions_spawn/subagents, heartbeat, and provider runtime contracts remain immediate/stream-oriented. The issue discussion already treats this as the canonical tracker and the related closed reports as narrower context, so this needs maintainer product/API design rather than cleanup closure.

## What This Changes

Request adds opt-in provider Batch API routing for non-real-time background agent work such as cron jobs, spawned subagents, and heartbeat checks to reduce provider costs.

## Best Possible Solution

Keep this as the canonical product/API tracker for an additive delayed/batch agent-turn contract, with public SDK/provider runtime support, protocol/config opt-ins for cron, sessions_send, sessions_spawn/subagents, and heartbeat/background runs, plus OpenAI and Anthropic implementations and docs for fallback, cancellation, accounting, and delayed delivery.
