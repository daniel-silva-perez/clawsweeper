## Summary

Keep this issue open. Current main still lacks first-class provider Batch API routing for delayed/background LLM agent turns; existing batch code is limited to memory embedding indexing/media-style provider jobs, and the cron, sessions_send, subagent runtime, heartbeat, and provider plugin contracts do not expose a submit/poll/cancel/result lifecycle for delayed model turns.

## Best Possible Solution

Keep this as the canonical product/API tracker. The best path is an additive provider/runtime batch-turn contract, followed by explicit opt-in fields for cron agentTurn jobs, heartbeat/background runs, sessions_send, and sessions_spawn/subagents, plus provider plugin implementations, docs, config/schema updates, and focused regression coverage.
