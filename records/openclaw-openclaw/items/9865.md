## Summary

Keep open. Current main still lacks first-class provider Batch API routing for delayed/background agent turns, and the issue discussion already treats this as the canonical tracker rather than an already-shipped feature or ClawHub-only request.

## What This Changes

The issue requests provider Batch API support for non-real-time OpenClaw background work such as cron jobs, subagents, scheduled maintenance, and heartbeat checks.

## Best Possible Solution

Keep this as the canonical product/API tracker for an additive delayed/batch agent-turn contract: public SDK/provider runtime support, protocol/config opt-ins for cron, sessions.send, sessions_spawn/subagents, and heartbeat/background runs, plus provider implementations and docs for fallback, cancellation, accounting, provider eligibility, and delayed delivery.
