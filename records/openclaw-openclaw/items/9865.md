## Summary

Keep this issue open. Current main still lacks first-class Batch API routing for delayed/background LLM agent turns: provider plugin contracts are stream-oriented, cron/sessions/subagent schemas do not accept a batch opt-in, heartbeat/subagent config has no batch policy, and the only provider batch support found is scoped to memory embeddings rather than agent turns. The closed related issues #49326, #56126, and #70606 are narrower context for this canonical tracker, not shipped fixes.

## Best Possible Solution

Keep this as the canonical product/API tracker. The best path is an additive provider/runtime batch-turn contract first, then opt-in protocol/config fields for cron agentTurn jobs, sessions_send, sessions_spawn/subagents, and heartbeat/background runs, followed by OpenAI/Anthropic provider plugin implementations, usage/accounting/fallback semantics, docs, and focused protocol/provider/runtime tests.
