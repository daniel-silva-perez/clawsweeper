## Summary

Keep open. Current main still lacks first-class Batch API routing for delayed/background agent turns, and the issue discussion already treats this as the canonical tracker while the related closed reports are narrower context rather than shipped fixes.

## What This Changes

Add opt-in provider Batch API support for background or non-real-time agent work such as cron jobs, subagents, and heartbeats to reduce model costs.

## Best Possible Solution

Keep this as the canonical product/API tracker for an additive delayed/batch agent-turn contract, with public SDK/provider runtime support, protocol/config opt-ins for cron, sessions_send, sessions_spawn/subagents, and heartbeat/background runs, plus OpenAI and Anthropic implementations and docs for fallback, cancellation, accounting, and delayed result delivery.
