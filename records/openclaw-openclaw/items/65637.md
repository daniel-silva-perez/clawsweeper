## Summary

Keep this PR open. Current main and the latest release still serialize the raw parsed config for both full and path-specific `/config show` replies, while the existing gateway config API already uses the redaction pipeline. The PR is the paired implementation candidate for open bug #65623 and the production diff is targeted; the remaining review issue is test quality, because the mocked hints should include `sensitive: true` to exercise schema-driven redaction.

## Best Possible Solution

Keep this PR open, address the review comment by making the mocked schema hints explicitly sensitive or by using a non-heuristic sensitive field, then review and land this PR or apply the same narrow fix directly. The fix should stay in `src/auto-reply/reply/commands-config.ts`, redacting both full and path-specific `/config show` replies through the existing schema-aware config redaction pipeline, with regression coverage in `src/auto-reply/reply/commands-gating.test.ts` and maintainer/security review for release handling.
