## Summary

Keep open. Current main only has a narrow generic warning when every normalized reply media item fails; it does not implement this PR's per-item dropped-media contract, sanitized notice formatting, JSON/outbound/transcript propagation, or routing/followup preservation. The related #70109 PR is closed and unmerged, so it does not supersede this active PR.

## Best Possible Solution

Keep this PR open. The best path is a maintainer-reviewed implementation that preserves sanitized per-item media-drop details through reply media normalization, catastrophic fallback, block/routing/followup filters, outbound JSON/delivery, and transcript mirroring, while retaining the privacy and delivery-safety fixes already discussed in the review thread.
