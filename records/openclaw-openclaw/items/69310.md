## Summary

Keep this PR open. Current main still does not implement the proposed dropped-media contract: per-item reply media failures are logged at verbose level, partial drops remain invisible, catastrophic normalization still clears media without a user-visible diagnostic, and the payload/outbound surfaces have no `droppedMedia` field. The PR is an active implementation candidate, but it needs maintainer review and cleanup because its latest head has failing checks and it touches privacy-sensitive notice/log/transcript behavior.

## Best Possible Solution

Keep this PR open for maintainer review. The best outcome is to land this PR, or an equivalent maintainer-owned implementation, after fixing the current CI/type issues and validating that sanitized dropped-media diagnostics survive per-item normalization, catastrophic fallback, block/followup routing, outbound JSON/delivery, hook/audit behavior, and transcript mirroring without exposing local paths, URL secrets, or mention-triggering filenames.
