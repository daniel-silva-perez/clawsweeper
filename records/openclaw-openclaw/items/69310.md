## Summary

Keep this PR open. Current main still does not implement the PR’s intended dropped-media pipeline: it only emits a generic warning when all reply media fails, silently ignores partial drops, has no `DroppedMediaItem`/`droppedMedia` contract, and cannot carry per-item diagnostics through routing, outbound JSON/delivery, followups, or transcript mirroring. The related #70109 candidate is closed and unmerged, so it does not supersede this active PR.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with an equivalent implementation that preserves sanitized per-item dropped-media details through media normalization, catastrophic fallback, block/routing/followup filters, outbound JSON and delivery, hook/audit behavior, and transcript mirroring. Before merge, resolve the current type/test readiness issues and explicitly accept or adjust the notice/hook moderation tradeoff.
