## Summary

Keep this PR open. Current main has only a coarse all-media-dropped warning in the normal per-item normalization path, but it still has no `droppedMedia` payload contract, no per-item/partial-drop notice pipeline, and the catastrophic media-normalization catch still clears media after a verbose log. The PR is therefore not obsolete, but it needs maintainer review rather than cleanup closure because it changes shared outbound delivery, transcript mirroring, and sanitized user-visible diagnostics.

## What This Changes

The PR adds a `DroppedMediaItem` pipeline through reply media normalization, routing, outbound payload projection, delivery, transcript mirroring, and tests so dropped `MEDIA:` attachments produce user-visible sanitized warnings.

## Best Possible Solution

Review and land a main-compatible implementation that preserves sanitized dropped-media diagnostics across normal, partial, routed, follow-up, JSON, and catastrophic normalization paths without leaking paths or credentials, and with targeted tests for delivery fallbacks, hooks/mirror behavior, and message-id preservation.
