## Summary

Keep this PR open. Current main still lacks the specific dropped-media visibility pipeline for mixed or catastrophic media-normalization failures, and the PR remains a relevant fix, but the latest head still has one blocking correctness issue: all-media-failed payloads keep the old generic warning and also carry the new dropped-media notice, producing duplicate user-visible warnings.

## What This Changes

The PR adds DroppedMedia metadata, propagates it through reply normalization, routing, outbound projections, delivery, transcript mirroring, warning logs, and focused media-drop tests.

## Best Possible Solution

Keep the PR active, remove the legacy generic all-media-failed warning when droppedMedia is populated, add coverage for the final delivered text in that path, then run the focused reply-media/outbound tests and changed gate before merge.
