## Summary

Keep open: current main still has a real mixed-media visibility gap, and the PR head is useful but has one blocking regression where all-failed media payloads can show both the legacy generic warning and the new detailed dropped-media notice.

## What This Changes

The PR adds a dropped-media payload field, threads media-normalization failures through reply routing/outbound delivery/transcript projection, formats user-visible attachment-failure notices, upgrades warning logs, and adds focused tests.

## Best Possible Solution

Keep the PR active, remove the legacy generic warning from the all-media-failed droppedMedia path or otherwise ensure only one user-facing notice is rendered, add a regression test for the delivered all-media-failed text, then run the focused reply/outbound tests plus the changed gate before merge.
