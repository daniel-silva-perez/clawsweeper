## Summary

Keep this PR open. Current main still has the reported dropped-media visibility gap, and the latest PR head addresses much of it, but it still double-reports all-failed MEDIA directives by keeping the legacy generic warning and also carrying the new dropped-media notice into outbound delivery.

## What This Changes

The PR adds dropped-media tracking for failed MEDIA normalization, preserves it through reply routing/outbound projections, formats user and transcript notices, updates warning logs, and adds focused tests.

## Best Possible Solution

Keep the droppedMedia pipeline, but remove the old generic all-media-failed text mutation or otherwise ensure each failed attachment produces only the new sanitized notice in user-visible delivery and transcript mirrors.
