## Summary

Keep this PR open. Current main still does not implement the proposed dropped-media contract: per-item failures are only logged at verbose level, partial drops are not user-visible, catastrophic normalization still clears media without a warning, and there is no `droppedMedia` field through reply payloads, routing, outbound JSON/delivery, or transcript mirroring. The closed #70109 PR is unmerged and does not supersede this active implementation candidate.

## Best Possible Solution

Keep this PR open for maintainer review. The best outcome is to land this PR, or an equivalent maintainer-owned implementation, after final cleanup and validation that sanitized dropped-media diagnostics survive per-item normalization, catastrophic fallback, block/followup routing, outbound JSON/delivery, hook/audit behavior, and transcript mirroring without exposing local paths or URL secrets.
