## Summary

Keep open. Current main still has the mixed-media gap where a failed MEDIA item is logged and skipped while surviving media sends without a user-visible per-item notice, so the PR remains useful; however the current head can render duplicate warnings when every media item fails and needs author/maintainer follow-up before merge.

## What This Changes

The PR adds droppedMedia tracking for failed MEDIA normalization, formats user/transcript notices, preserves dropped-media-only payloads through reply/outbound routing, upgrades warning logs, and adds focused tests.

## Best Possible Solution

Keep the contributor PR active, remove the legacy all-failed generic warning when droppedMedia will render the structured notice, add an all-media-failed test that asserts a single warning, then run the focused reply/outbound tests and changed gate before merge.
