## Summary

Keep open. The PR addresses a real dropped-media visibility gap on current main, but the current branch still has blocking CI and behavior regressions: silent `NO_REPLY` media failures become visible payloads, the copied runtime reply-payload type is out of sync, and lint fails on unused imports.

The PR adds a `droppedMedia` pipeline for failed `MEDIA:` normalization, formats user-visible attachment-failure notices, preserves those payloads through routing/outbound projections, updates warning logs, and adds focused tests.

## Best Possible Solution

Keep the PR active, fix the branch so dropped-media visibility does not override explicit silent-turn suppression, update the copied runtime reply-payload contract, remove lint-only leftovers, then rerun the focused reply/outbound tests plus changed checks before normal maintainer review.
