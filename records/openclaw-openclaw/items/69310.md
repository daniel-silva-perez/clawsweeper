## Summary

Keep this PR open. Current main still does not implement the PR's dropped-media pipeline: it only appends a generic `⚠️ Media failed.` warning when all normalized media are dropped, while partial drops, catastrophic normalization failures, outbound JSON/delivery, routing/followup, block dedupe, hooks, and transcript mirroring still have no structured dropped-media carrier. The related #70109 candidate is closed and unmerged in the provided context, so it does not supersede this active PR.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR after resolving current head issues and validating the touched lanes, or replace it with an equivalent maintainer-approved implementation that preserves sanitized per-item dropped-media details through normalization, catastrophic fallback, routing/followup filters, outbound JSON/delivery, hooks/audit behavior, and transcript mirroring.
