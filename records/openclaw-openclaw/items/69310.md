## Summary

Keep this PR open. Current main still does not implement the PR's intended dropped-media pipeline: it has only a generic all-media-dropped warning in one normalizer path, no structured droppedMedia contract, no per-item partial-drop user notice, and no outbound JSON/delivery/transcript mirror handling for dropped-media diagnostics. The related #70109 candidate is closed and unmerged, so it does not supersede this active PR.

## Best Possible Solution

Keep this PR open for maintainer review. The best outcome is to land this PR after resolving current validation issues and security-review tradeoffs, or replace it with an equivalent maintainer-approved implementation that preserves sanitized per-item dropped-media details through per-item normalization, catastrophic fallback, routing/followup filters, outbound JSON/delivery, hooks/audit behavior, and transcript mirroring.
