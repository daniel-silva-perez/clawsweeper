## Summary

Keep open. Current main still treats `tools.sessions.visibility` as one four-value scope and feeds that same scope into send, history, list, and status checks; the requested per-capability object form is not implemented. The related #57483 and #55420 proposals are adjacent open work, not a merged or superseding implementation.

## What This Changes

The issue requests a backward-compatible structured `tools.sessions.visibility` object with separate send, history, list, and metadata permissions so cross-agent sends can be allowed without granting transcript read/list access.

## Best Possible Solution

Define one coherent, backward-compatible session-tool capability model: preserve the existing string shorthands, add a validated object form for separate send/history/list/metadata permissions, keep sandbox and A2A restrictions fail-closed, update schema/help/docs, and add focused tests proving cross-agent send can be allowed without exposing transcript history or broad listing.
