## Summary

Keep open. Current main still models `tools.sessions.visibility` as a single string enum and applies one shared visibility guard across session send/history/list/status; the requested backward-compatible per-capability object is not implemented, and related open work is narrower rather than a full superseder.

## What This Changes

Request to replace the single `tools.sessions.visibility` string shorthand with structured per-capability `send`, `history`, `list`, and `metadata` visibility controls while preserving existing shorthand behavior.

## Best Possible Solution

Keep this issue open for maintainer design follow-up. The maintainers should decide whether the narrower A2A-gated send behavior is the intended MVP or whether to implement the full backward-compatible structured `tools.sessions.visibility` object across the SDK visibility helper, session tool call sites, config schema/types/generated help, docs, and focused tests.
