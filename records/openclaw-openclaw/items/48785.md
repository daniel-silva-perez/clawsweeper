## Summary

Keep open. Current main still exposes `tools.sessions.visibility` as one string-valued scope and applies that single visibility decision across send, history, list, and status; the requested backward-compatible per-capability object is not implemented. The related open items cover narrower A2A-send, allowlist, or per-agent override work rather than a full superseder.

## What This Changes

Request to add an opt-in structured `tools.sessions.visibility` object with separate `send`, `history`, `list`, and `metadata` controls while preserving the existing string shorthands.

## Best Possible Solution

Define one backward-compatible capability model for session tools: keep the existing string shorthands, add a validated object form for separate send/read/list/metadata capabilities, preserve sandbox and A2A fail-closed behavior, update schema/types/generated help/docs, and add focused tests proving send can be allowed without exposing transcript history.
