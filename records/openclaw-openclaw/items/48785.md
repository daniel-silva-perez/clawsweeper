## Summary

Keep open. Current main still models `tools.sessions.visibility` as one string-valued scope and applies that single scope across send, history, list, and status access; the requested per-capability object form is not implemented. The related open allowlist issue and partial A2A-send PR are adjacent, but neither supersedes this permission-model request.

## What This Changes

Request a backward-compatible structured `tools.sessions.visibility` object with separate `send`, `history`, `list`, and `metadata` capability knobs while preserving existing string shorthands.

## Best Possible Solution

Define one backward-compatible session-tool capability model: keep the existing string shorthands, add a validated object form for separate send/history/list/metadata permissions, preserve sandbox and A2A restrictions, update schema/types/generated help/docs, and add tests proving cross-agent send can be allowed without exposing transcript history or broad listing.
