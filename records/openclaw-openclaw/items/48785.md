## Summary

Keep open. Current main still models `tools.sessions.visibility` as one string enum and applies the same shared visibility guard across `sessions_send`, `sessions_history`, `sessions_list`, and `session_status`; the requested object-shaped per-capability knobs are not implemented. Related #57447/#57483 and #59170 are narrower overlapping work, not full superseders for this permission-model request.

## Best Possible Solution

Keep this issue open for maintainer decision. The likely path is to choose whether #57483's narrower A2A-gated send behavior is the intended MVP, or to implement the full backward-compatible structured `tools.sessions.visibility` object with explicit `send`, `history`, `list`, and `metadata` semantics across the shared visibility helper, config schema/types/generated help, docs, and focused session-tool tests.
