## Summary

Keep open. Current main still exposes `tools.sessions.visibility` as a single string scope and applies that one scope across send, history, list, and status access, so the requested per-capability object form is not implemented or superseded by the adjacent allowlist/A2A work.

## What This Changes

Request to replace the single `tools.sessions.visibility` string enum with a backward-compatible structured object that separates send, history, list, and metadata permissions.

## Best Possible Solution

Define one backward-compatible session-tool capability model: keep the existing string shorthands, add a validated object form for separate send/history/list/metadata permissions, preserve sandbox and A2A restrictions, update schema/types/generated help/docs, and add tests proving cross-agent send can be allowed without exposing transcript history or broad listing.
