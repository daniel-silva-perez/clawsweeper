## Summary

Keep open. Current main has partial mitigations for commentary/final-answer separation, message_sending rewrite/cancel hooks, outbound scaffolding stripping, and cron-specific stale-ack retry/suppression, but it still lacks the requested cross-channel runtime pre-send policy tied to durable execution state, typed ack/progress classification, repeat-ack suppression, progress-debt enforcement, and policy metrics/tests.

## What This Changes

The issue requests a runtime-enforced pre-send policy layer that classifies outbound assistant messages and blocks fake-progress acknowledgements unless backed by durable execution/progress state.

## Best Possible Solution

Keep this open as a core architecture/design tracking issue. The best path is to align it with the broader outbound-boundary work in #56349/#72539, define a first-class runtime policy boundary that classifies outbound assistant text, binds acknowledgements to durable execution/task state, tracks repeat-ack and progress-debt state per session/surface, emits structured suppression metrics, and adds shared outbound plus at least one channel integration test and docs.
