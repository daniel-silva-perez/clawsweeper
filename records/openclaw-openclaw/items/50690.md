## Summary

Keep open. Current main has useful partial mitigations for commentary/final-answer separation, plugin `message_sending` rewrite/cancel hooks, and isolated-cron stale-ack retry, but it still lacks the requested cross-channel runtime policy that classifies outbound assistant text and gates acknowledgement delivery on durable execution/progress state.

## What This Changes

Request a runtime-enforced pre-send policy layer that blocks fake-progress acknowledgements, commentary leakage, repeated ack spam, and post-start silence across user-facing delivery channels.

## Best Possible Solution

Define a first-class outbound policy boundary aligned with #56349 and #72539, then layer this fake-progress policy on top with durable execution state, typed ack/progress/blocker/completion classification, per-session repeat and progress-debt tracking, structured suppression metrics, and shared outbound plus channel integration coverage.
