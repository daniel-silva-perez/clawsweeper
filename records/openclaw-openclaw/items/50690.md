## Summary

Keep open. Current main has useful pieces: a generic `message_sending` hook, phase-aware commentary suppression, and isolated-cron stale-ack suppression. It does not implement the requested runtime policy layer: there is no typed ack/progress/blocker/completion classifier, no execution-state-required ack gate, no per-session repeat-ack or progress-debt state, and no requested policy metrics. The existing hook surface also does not expose enough execution/history/tool state to make this purely a ClawHub/plugin close.

## Best Possible Solution

Keep this open as a core runtime design item. The best path is a generic outbound policy layer or a stronger plugin-facing policy seam backed by durable run/session/task state: classify outbound text, gate ack-like messages on real execution state, track per-session ack/progress/blocker/completion timestamps, enforce progress-debt timeouts, emit structured policy metrics, and add cross-surface tests for direct chat, channel topics, cron, subagents, and ACP delivery. Existing phase-aware and cron-specific fixes should remain as supporting pieces, not the whole solution.
