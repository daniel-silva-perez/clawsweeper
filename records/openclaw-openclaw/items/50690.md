## Summary

Keep open. Current main has useful partial mitigations for outbound rewrite/cancel hooks, phase-aware commentary suppression, and cron interim-ack retry, but it still lacks the requested execution-state-backed runtime policy that classifies and gates fake-progress acknowledgements across user-facing outbound delivery paths.

## What This Changes

The issue requests a runtime pre-send policy layer that blocks low-value acknowledgement or commentary-style outbound messages unless they are backed by real execution/progress state.

## Best Possible Solution

Choose one first-class outbound policy boundary and durable execution-state model, then implement core fake-progress enforcement with typed ack/progress/blocker/completion classification, repeat suppression, progress-debt faulting, structured suppression metrics, and shared outbound plus channel/agent coverage.
