## Summary

Keep open. Current main has partial mitigation in the shared outbound hook, commentary-aware text extraction, and isolated-cron acknowledgement retry, but it still lacks the requested runtime-owned fake-progress classifier and execution-state-backed pre-send gate across user-facing delivery paths.

## What This Changes

The issue requests a runtime pre-send policy layer that classifies outbound assistant messages and blocks fake-progress acknowledgements unless backed by real execution/progress state.

## Best Possible Solution

Maintainers should first choose the shared outbound enforcement boundary and durable execution-state contract, then implement core fake-progress enforcement with typed ack/progress/blocker/completion classification, repeated-ack suppression, progress-debt faulting, structured suppression metrics, and coverage across shared outbound, channel adapters, agents, subagents, ACP, and cron.

## Reproduction Assessment

Unclear for a live end-to-end repro in this read-only review. The issue body gives direct chat and Telegram topic repro reports, and current source inspection shows a plausible gap because final user-visible ack text can still pass through delivery unless an existing hook/plugin cancels it.

## Solution Assessment

Yes directionally: a runtime-enforced pre-send policy backed by durable execution state is the right class of fix for this failure mode. The exact implementation is not yet a safe narrow change because it depends on a maintainer decision about the shared outbound boundary and execution-state model.
