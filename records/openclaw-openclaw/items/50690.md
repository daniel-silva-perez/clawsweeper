## Summary

Keep open. Current main has several partial mitigations, but it still does not implement the requested execution-state-backed pre-send fake-progress policy across user-facing delivery paths. The related open outbound-boundary and pre-send-hook issues overlap but do not supersede this issue’s ack/progress classification, durable state, repeat-suppression, and progress-debt requirements.

## What This Changes

The issue requests a runtime pre-send policy layer that classifies outbound assistant messages and blocks fake-progress acknowledgements or commentary unless backed by durable execution/progress state.

## Best Possible Solution

Define one first-class outbound policy boundary, then implement fake-progress enforcement on top with durable execution state, typed ack/progress/blocker/completion classification, per-session repeat and progress-debt tracking, structured suppression metrics, and integration coverage through shared delivery plus channel/tool-specific sends.
