## Summary

Keep open. Current main has useful partial mitigations, including a generic outbound rewrite/cancel hook, phase-aware commentary suppression, and isolated-cron interim-ack retry, but it still does not implement the requested execution-state-backed fake-progress policy across user-facing delivery paths.

## What This Changes

The issue requests a runtime pre-send policy layer that classifies outbound assistant messages and blocks fake-progress acknowledgements unless backed by execution/progress state.

## Best Possible Solution

Define one first-class outbound policy boundary, then implement fake-progress enforcement on top with durable execution state, typed ack/progress/blocker/completion classification, repeat suppression, progress-debt timeout/faulting, structured suppression metrics, and coverage through shared outbound plus channel/tool-specific sends.
