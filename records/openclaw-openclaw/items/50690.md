## Summary

Keep open. Current main has useful partial mitigations for commentary/final-answer separation, message_sending rewrite/cancel hooks, outbound scaffolding stripping, and isolated-cron stale-ack retry, but it still does not implement the requested cross-channel runtime policy that classifies outbound assistant text and gates acknowledgements on durable execution/progress state.

## What This Changes

Request to add a runtime pre-send policy layer that classifies outbound assistant messages and blocks fake-progress acknowledgements unless they are backed by durable execution or meaningful progress state.

## Best Possible Solution

Define a first-class outbound policy boundary aligned with #56349 and #72539, then add a fake-progress policy on top: durable execution state for user-visible acknowledgements, typed ack/progress/blocker/completion classification, per-session repeat and progress-debt tracking, structured suppression metrics, and shared outbound plus channel integration coverage. Ryce's keepmyclaw restore-point comment is useful design input for the durable-state part.
