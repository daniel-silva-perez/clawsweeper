## Summary

Keep open. Current main has partial commentary/final-answer separation, generic message_sending hooks, outbound scaffolding stripping, and cron-specific stale-ack retry handling, but it still lacks the requested core runtime pre-send policy tied to durable execution state, ack/progress classification, repeated-ack suppression, progress-debt enforcement, and cross-channel tests.

## Best Possible Solution

Keep this open as a core architecture/design tracking issue. The best path is a maintainer-reviewed outbound policy design that defines typed assistant-message classification, binds allowed acknowledgements to durable execution/task state, records per-session repeat-ack and progress-debt state, emits structured suppression metrics, and adds focused shared outbound plus at least one channel integration test. Current message_sending hooks and phase-aware extraction can be inputs to that design, but they are not the complete policy layer.
