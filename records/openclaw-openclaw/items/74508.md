## Summary

Keep open. Current main still has the Telegram preview overflow stop behavior, so the PR is addressing a real gap, but the revised diff is not merge-ready: a single throttled reasoning update whose remaining slice is still over Telegram's limit will still stop the stream instead of draining into additional continuation messages. The second commit appears to address the earlier shared-renderer scoping concern, but not the full overflow case.

## What This Changes

The PR branch updates Telegram draft preview rendering, draft overflow handling, reasoning split logic, and colocated tests so long reasoning streams can continue across Telegram messages and partial think tags do not leak into answer previews.

## Best Possible Solution

Revise the PR so Telegram draft overflow drains the pending remainder across as many max-sized continuation messages as needed, passes explicit reasoning-lane continuation state instead of inferring from the slice prefix, and keeps the partial closing-tag suppression with targeted regression tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Split oversized overflow remainders before stopping:** `extensions/telegram/src/draft-stream.ts:334`
  - body: After the first overflow rotation, a throttled update whose remaining slice is still over `maxChars` recurses with delivered/sent state reset. The next call takes the stop path instead of emitting another max-sized continuation, so long reasoning can still freeze when one pending update spans multiple Telegram messages. Keep chunking the remainder until each sent slice fits, and add a regression for a single update spanning at least two continuations.
  - confidence: 0.88
- **[P3] Label continuations without depending on the slice prefix:** `extensions/telegram/src/bot-message-dispatch.ts:340-341`
  - body: The reasoning renderer only adds `Reasoning (cont.):` when the sliced text starts with `_`, but overflow slicing happens at the previous raw offset inside the already-formatted reasoning string. Most splits land mid-line and start with a normal character, so the continuation is sent unlabeled. Pass an explicit continuation signal or prefix reasoning-lane slices after the first message.
  - confidence: 0.78
