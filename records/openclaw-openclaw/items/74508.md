## Summary

Keep open. The PR targets a real Telegram reasoning-preview gap that current main still has, but the proposed patch should not merge as-is because the shared draft renderer would mislabel italic answer previews as reasoning continuations, and the overflow path can rotate to a new message without actually sending the still-oversized remainder.

## What This Changes

The PR changes Telegram draft preview rendering, draft overflow handling, and reasoning split logic so long reasoning streams can continue across Telegram messages and partial thinking-tag content does not flash in the answer lane.

## Best Possible Solution

Revise the PR so continuation labeling is scoped to the reasoning draft lane, overflow chaining keeps draining and sending repeated max-sized slices instead of acknowledging unsent text, and regression tests cover italic answer previews, two-or-more chained overflow messages, and partial closing reasoning tags.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Scope continuation labels to reasoning drafts:** `extensions/telegram/src/bot-message-dispatch.ts:330-331`
  - body: `renderDraftPreview` is shared by both the answer and reasoning lanes, so this `^_` check also runs for normal answer previews. A Telegram answer that legitimately starts with italic Markdown, such as `_note_`, would be sent with `Reasoning (cont.):` prepended. Pass the lane into the renderer or apply this prefix only for reasoning drafts.
  - confidence: 0.92
- **[P2] Drain oversized overflow slices instead of dropping them:** `extensions/telegram/src/draft-stream.ts:325-332`
  - body: When a single update jumps far enough that the remainder is still over `maxChars`, this branch rotates to a new message and returns `true` without sending anything. The draft loop treats `true` as success and clears the pending text; the next update has the delivered/sent text reset, so the stream can lose that slice or hit the old stop path. Keep splitting/sending chunks instead of returning success without delivery.
  - confidence: 0.86
