## Summary

Keep open. Current main still stops Telegram draft previews when the rendered preview exceeds the Telegram 4096-character cap, so the PR is addressing a real Telegram reasoning-stream gap. The current draft branch still introduces offset-reset and preview-cleanup regressions that should be revised before merge.

## What This Changes

The PR changes Telegram draft streaming to split oversized previews into chained messages, label reasoning continuations, suppress partial reasoning-tag leakage, scale draft throttles when answer and reasoning lanes both stream, and add Telegram tests.

## Best Possible Solution

Revise the branch so overflow-internal rotations preserve the active split offset while public fresh-message rotations reset it for a new logical preview, and either surface overflow-created preview IDs to lane cleanup or scope chaining to retained reasoning previews with explicit cleanup semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reset overflow offsets for new logical previews:** `extensions/telegram/src/draft-stream.ts:345`
  - body: After an overflow chain, `textBaseOffset` remains nonzero. Existing callers use `forceNewMessage()` to start unrelated fresh previews, so the next update can be sliced from the old offset and lose the beginning of the new message, or send nothing if it is shorter than that offset. Split the internal overflow rotation from the public fresh-message reset, and cover overflow followed by `forceNewMessage()` plus a short update.
  - confidence: 0.9
- **[P2] Track chained answer previews before final cleanup:** `extensions/telegram/src/draft-stream.ts:345`
  - body: This generic overflow path can also run for the answer lane. When it calls `forceNewMessage()` internally, the previous preview message ID is dropped from the stream but is not added to `archivedAnswerPreviews`; final cleanup only clears the active stream and known archives, so long answer drafts can leave earlier preview chunks in chat after the final answer is sent. Either make overflow rotations observable to lane cleanup or restrict chaining to reasoning messages that are intentionally retained.
  - confidence: 0.82
