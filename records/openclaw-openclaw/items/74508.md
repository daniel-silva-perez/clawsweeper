## Summary

Current main still has the Telegram reasoning overflow problem, and this draft PR is the active implementation candidate rather than obsolete. The proposed branch usefully targets the right Telegram draft-stream surface, but the current diff has a blocking cleanup-lifecycle bug that can delete intentionally chained reasoning chunks, so it should stay open for revision instead of being closed or replaced by an automated repair lane.

## What This Changes

The PR changes Telegram draft streaming to split overlong reasoning previews into chained continuation messages, adds reasoning-continuation rendering and 429/rate-limit handling, suppresses partial reasoning-tag leakage into the answer lane, and expands Telegram streaming tests.

## Best Possible Solution

Revise the PR so intentionally completed overflow chunks are retained while stale or superseded previews remain cleanup targets, using an explicit lifecycle signal or separate retention path. Keep the fix scoped to the Telegram plugin draft-stream and dispatch surfaces, with a regression that drives a long streamed reasoning preview through final cleanup and verifies all intended chained messages remain visible.
