## Summary

Keep open. Current main still has the Telegram draft-overflow stop behavior, but this draft PR has a blocking cleanup regression: overflow-created reasoning chunks are routed through the existing superseded-preview cleanup path and would be deleted at turn cleanup, leaving only the last chained slice visible.

## What This Changes

The PR modifies Telegram draft streaming/rendering to split overlong reasoning previews into continuation messages, tune shared edit/typing rate limiting with 429 retry behavior, suppress partial thinking-tag leakage, and add Telegram tests.

## Best Possible Solution

Revise the PR so intentional reasoning overflow chunks are retained while true stale or superseded previews are still cleaned up, then add a dispatch-level regression that drives a long reasoning stream through cleanup and verifies all chained messages remain visible.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve chained reasoning previews during cleanup:** `extensions/telegram/src/draft-stream.ts:379-386`
  - body: The overflow path reports an intentionally completed reasoning chunk through `onSupersededPreview`, but dispatch records reasoning previews from that callback in `archivedReasoningPreviewIds` and deletes every archived reasoning preview during final cleanup. A long reasoning stream would lose earlier chained chunks and leave only the last slice visible; use a distinct retention path or flag for overflow chunks and cover it with a dispatch cleanup regression.
  - confidence: 0.9
