## Summary

Keep open. The underlying ACP compatibility bug is still present on current main, and this PR is useful source work, but the branch is dirty against current main and the submitted guard does not match the stated SDK validation contract.

## What This Changes

The PR adds a `coerceAcpStream()` wrapper in `src/acp/server.ts` to rewrite non-number `initialize.params.protocolVersion` values to the ACP SDK `PROTOCOL_VERSION` before constructing `AgentSideConnection`.

## Best Possible Solution

Land a rebased current-main ACP server compatibility fix that normalizes incoming `initialize.params.protocolVersion` before SDK validation, preserves valid ACP numeric versions, adds focused regression coverage for VS Code/Cursor-style date-string input, and credits the original bug reporter plus this PR's contributor in the changelog.

## Reproduction Assessment

Yes. The linked bug logs provide concrete VS Code/Cursor initialize messages with a string date `protocolVersion`, and source plus dependency inspection shows current main feeds that value into SDK validation that requires an integer from 0 through 65535.

## Solution Assessment

No, not as submitted. A stream-boundary transform is the right maintainable shape, but this PR is stale against the current server bootstrap and its guard is narrower than the stated non-integer normalization contract.
