## Summary

Keep open. The linked ACP interop bug still exists on current main, and this PR is a relevant narrow fix, but it is not merge-ready because the branch is dirty/unrebaseable against current main and the shim does not fully match the SDK's uint16 protocol-version contract.

## What This Changes

This PR wraps the ACP server's `ndJsonStream` with a transform that rewrites non-number `initialize.params.protocolVersion` values to the ACP SDK `PROTOCOL_VERSION` before constructing `AgentSideConnection`.

## Best Possible Solution

Land a rebased ACP server compatibility fix on current main that normalizes only incoming `initialize` protocol-version values before the ACP SDK validates them, covers the date-string case in a focused test, and closes the linked bug after merge.

## Reproduction Assessment

Yes. The linked report includes concrete VS Code/Cursor initialize logs with `protocolVersion: '2025-11-25'`, and current main still passes raw `ndJsonStream` messages into the ACP SDK path that validates protocolVersion as a uint16 number.

## Solution Assessment

No, not as submitted. The transform approach is the right narrow place to solve this SDK-boundary incompatibility, but the PR needs to be rebased onto the current server bootstrap and its guard should match the SDK integer/range contract before it is the best fix.
