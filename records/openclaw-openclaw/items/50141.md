## Summary

Keep this PR open. Current main already implements the chat dropdown/provider-qualified picker side of the fix, but it still does not implement the PR’s remaining `/model` slash-command display behavior: bare `/model` still prints current and available model IDs without provider qualifiers, while the docs tell users to type `provider/model` refs for unambiguous model switching.

## Best Possible Solution

Keep this PR open for maintainer review. Land or cherry-pick the remaining `/model` slash-command behavior: qualify the current display only from the active session’s `modelProvider`, qualify available models from catalog `provider` metadata with `buildQualifiedChatModelValue()`, preserve the no-provider fallback, and keep the regression tests. Close only after that lands or another canonical PR fully covers the same slash-command gap.
