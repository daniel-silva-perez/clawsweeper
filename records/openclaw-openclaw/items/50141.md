## Summary

Keep this PR open. Current main already preserves provider-qualified values through the Control UI model picker, but the PR still contains unmerged `/model` slash-command behavior: current main prints current and available models as bare IDs, while the PR qualifies them from session/catalog provider metadata.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or cherry-pick the remaining `/model` slash-command behavior: qualify the current display only from the active session's `modelProvider`, qualify available models from catalog `provider` metadata, preserve the legacy no-provider fallback, and keep the regression tests. Close only after that lands or another canonical PR fully covers the same slash-command gap.
