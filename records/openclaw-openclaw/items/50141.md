## Summary

Keep this PR open. Current main already preserves provider-qualified values through the Control UI model picker, but it still has the bare `/model` display behavior that this PR fixes and tests. The PR is not obsolete on current main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to reduce, land, or cherry-pick the remaining `/model` slash-command change so current and available model displays use provider-qualified values from session/catalog data, while preserving the PR's guard against incorrectly borrowing the defaults provider for legacy session rows.
