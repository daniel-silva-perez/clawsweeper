## Summary

Keep this PR open. Current main already preserves provider-qualified values through the Control UI model picker and sends those values to `sessions.patch`, but bare `/model` still prints the current model and available models as unqualified IDs. The PR’s slash-command display fix and regression tests remain useful, and the author recently rebased and addressed review feedback.

## Best Possible Solution

Keep this PR open for maintainer review. The likely path is to reduce, land, or cherry-pick the remaining `/model` slash-command behavior so both the current model and available model list use provider-qualified values via the existing model-ref helper, with regression tests for provider-backed and missing-provider cases. The Control UI picker portion can be treated as already covered on main.
