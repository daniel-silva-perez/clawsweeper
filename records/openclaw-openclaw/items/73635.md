## Summary

Keep this PR open. Current main has direct leading `403 <html>...` auth handling, but it does not implement the PR's central embedded-HTML case such as `Error: fetch failed: 403 <html>...`; that path can still fall through to generic transport/error rewriting. The PR is a plausible focused bug fix, but the reviewer-noted mojibake truncation regression should be fixed before merge.
