## Summary

Keep this PR open. Current main still handles direct leading `403 <html>...` auth pages, but it does not implement the PR's central embedded-wrapper case such as `Error: fetch failed: 403 <html>...`; that path can still fall through to transport-style rewriting. The PR remains a plausible focused bug fix, but the reviewed PR head still needs the mojibake ellipsis regression fixed before merge.

## Best Possible Solution

Keep the PR open as a focused implementation candidate. The best path is to fix the mojibake ellipsis, consider sharing the HTML auth-copy predicate, then rerun the targeted formatter, sanitizer, and runtime-classification tests before maintainer review or merge.
