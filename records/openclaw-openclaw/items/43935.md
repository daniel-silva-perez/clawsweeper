## Summary

Keep this issue open. Current `main` still plans channel config hot reloads as channel-wide restarts, Telegram still advertises only a broad `channels.telegram` reload prefix, and the hot reload handler still calls channel lifecycle methods without an account id. The same-author implementation PR #43938 remains open with closing syntax for this issue, so the maintainer path is to review and land that PR or an equivalent fix.

## Best Possible Solution

Keep this issue open while #43938 remains open. The best maintainer path is to review and land #43938 or an equivalent gateway change that detects strictly account-scoped paths under `channels.<channel>.accounts.<accountId>.*`, records account-specific restart targets, calls `stopChannel(channelId, accountId)` and `startChannel(channelId, accountId)`, keeps full-channel restarts for channel-global or mixed changes, and adds focused planner/handler regression tests.
