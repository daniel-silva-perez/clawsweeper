## Summary

Keep open. Current main still represents channel reloads only as whole-channel targets and the hot-reload handler still calls channel start/stop without an account id, so account-only Telegram config edits remain capable of restarting the full channel. The open implementation PR that closes this issue is still unmerged, so closing before that PR lands would be premature.

## What This Changes

The issue asks for account-scoped gateway channel reloads for changes under `channels.<channel>.accounts.<accountId>.*` instead of restarting the entire channel.

## Best Possible Solution

Represent account-scoped channel restart targets in the reload plan, detect strict `channels.<channel>.accounts.<accountId>.*` changes, make channel-global or mixed changes take precedence as full-channel restarts, route account-only changes through the existing account-aware lifecycle, and add focused planner and handler regression tests.
