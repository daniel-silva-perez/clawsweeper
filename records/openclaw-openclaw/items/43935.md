## Summary

Keep open: current main still collapses channel-account config paths into whole-channel reload targets, and the hot reload handler still calls channel-wide stop/start. The linked same-author implementation PR #43938 remains open and unmerged with closing syntax, so this should stay tracked until that PR is reviewed, landed, replaced, or closed.

## What This Changes

The issue requests account-scoped gateway hot reloads for `channels.<channel>.accounts.<accountId>.*` changes so account-only edits do not restart the entire channel.

## Best Possible Solution

Land or replace PR #43938 with a rebased implementation that represents account-scoped restart targets in the reload plan, treats mixed/global channel changes as full-channel restarts, routes strict account-only changes through `stopChannel(channel, accountId)` and `startChannel(channel, accountId)`, preserves active-work deferral, and adds focused planner/handler regression tests.
