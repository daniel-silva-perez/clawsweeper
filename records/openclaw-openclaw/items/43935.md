## Summary

Keep open: current main still collapses account-scoped channel config changes into whole-channel reload targets, and the same-author implementation PR #43938 remains open with closing syntax, so the issue should stay tracked until that PR is reviewed, landed, replaced, or closed.

## What This Changes

The issue requests gateway hot reload planning that restarts only affected channel accounts for strict `channels.<channel>.accounts.<accountId>.*` config changes while preserving full-channel restarts for global or mixed channel changes.

## Best Possible Solution

Land or replace PR #43938 with a rebased implementation that represents account-scoped restart targets in the reload plan, promotes mixed/global channel changes to full-channel restarts, routes strict account-only changes through account-aware channel lifecycle calls, preserves active-work deferral, and adds focused planner and handler regression tests.
