## Summary

Keep open. Current main still collapses account-scoped channel config changes into whole-channel restart targets, and an open linked implementation PR is already the right review path, so this issue should remain tracked until that PR lands, is replaced, or is closed.

## What This Changes

Account-scoped channel configuration changes should restart only the affected channel account instead of restarting the whole channel, while preserving full-channel restarts for global or mixed changes.

## Best Possible Solution

Land or replace PR #43938 with a rebased implementation that models account-scoped restart targets in the reload plan, promotes mixed or channel-global changes to full-channel restarts, preserves active-work deferral, updates all reload-plan consumers, and adds focused planner and handler regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is clear: channels.telegram.accounts.engineer.dmPolicy matches the Telegram channels.telegram reload prefix, the planner adds telegram to restartChannels, and the handler calls stopChannel/startChannel without an account id; the discussion also includes timestamped Telegram logs showing the same path.

## Solution Assessment

Yes. Account-scoped reload planning is the narrow maintainable fix for this issue, provided the implementation keeps full-channel precedence and active-work deferral intact; any remaining Telegram delivery retry/drain hardening should be handled separately if it still reproduces.
