## Summary

Keep open: current main still collapses account-scoped channel config changes into channel-wide hot reloads, and the linked implementation PR #43938 remains open and unmerged, so closing this issue would be premature.

## What This Changes

The issue asks the gateway to restart only affected channel accounts for strict `channels.<channel>.accounts.<accountId>.*` config changes while preserving full channel restarts for channel-global or mixed changes.

## Best Possible Solution

Add account-scoped restart targets to the gateway reload plan, detect strict account paths under `channels.<channel>.accounts.<accountId>.*`, make full-channel restarts take precedence for global or mixed changes, route account-only reloads through the existing account-aware channel lifecycle, and cover planner plus handler behavior with focused regression tests.
