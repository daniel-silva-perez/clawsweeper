## Summary

Current main still has the followup cross-target dedupe bug described by this PR. The normal reply path has the same-target guard, but `resolveFollowupDeliveryPayloads` still filters text/media before checking the messaging-tool target. The PR is useful, but it should stay open for maintainer/contributor narrowing because its current diff bundles unrelated model retry, delivery serialization, and Mattermost dependency changes.

## Best Possible Solution

Keep this PR open, but ask for a narrowed update or replacement that only ports the same-target dedupe guard into `resolveFollowupDeliveryPayloads`, preserves legacy no-target and mixed target/no-target dedupe behavior, and adds followup text/media cross-target regression tests. Split model retry, delivery serialization, and Mattermost dependency work into separate PRs with their own review and validation.
