## Summary

Keep this PR open. Current main still mutates plugin policy config from the raw `plugins enable` / `plugins disable` argument before checking whether the plugin exists, and this PR is the active focused implementation candidate for the linked open regression.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. Maintainers should review the registry-gated enable/disable behavior, decide whether name fallback belongs here or the command should be id-only, address any remaining test symmetry concern, run the normal changed validation, and land this PR or an equivalent maintainer patch. After merge, the linked regression can close as implemented.
