Keep this PR open. Current main still writes plugin policy config from the raw `plugins enable/disable <id>` argument before any registry existence check, while the PR is an active, focused implementation candidate for open bug #73551 and is not superseded by the two closed alternative PRs.

## What This Changes

This PR adds an exact-id plugin registry lookup before `openclaw plugins enable/disable <id>` mutates config, reports missing ids with exit 1, and extends CLI policy tests/test helpers for no-write rejection cases.

## Best Possible Solution
