## Summary

Keep this PR open. Current main still mutates plugin policy config from the raw `plugins enable` / `plugins disable` argument before verifying that the plugin exists, and the current policy tests do not cover the missing-id no-write behavior. The provided PR context shows this is the active focused implementation candidate for open regression #73551, with a narrow CLI/test diff and follow-up manual verification.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. Maintainers should review and land this patch or an equivalent fix that resolves the requested plugin against the discovered registry before enable/disable config writes, keeps missing-id no-write tests, settles id-only versus name fallback behavior, and then close #73551 after the fix reaches main.
