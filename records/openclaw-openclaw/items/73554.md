## Summary

Keep this PR open. Current main still mutates plugin policy config from the raw `plugins enable` / `plugins disable` argument before any registry validation, and the linked bug #73551 is still the active user-facing problem this PR addresses. The provided PR file list is limited to CLI behavior and tests, with no apparent security-sensitive or supply-chain changes.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. Maintainers should review and land the registry-gated enable/disable behavior, decide the id-vs-name fallback policy, keep the missing-id no-write tests, and then close the linked bug as implemented after merge.
