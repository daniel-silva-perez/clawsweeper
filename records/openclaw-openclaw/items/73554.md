## Summary

Current main still has the reported bug: `plugins enable` and `plugins disable` write config from the raw CLI argument before any discovered-plugin registry check. This PR remains a relevant implementation candidate for #73551; the overlapping PRs #73932 and #74016 are closed and unmerged, so they do not supersede it.

## What This Changes

This PR adds an id-only registry lookup before `openclaw plugins enable/disable <id>` writes config, reports missing plugin ids with exit 1, and extends plugin CLI policy tests/test helpers for no-write rejection paths.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #73551. Maintainers should review the exact-id registry gate, resolve or classify current CI failures, then land one focused fix that rejects unknown ids before config writes; cleanup of previously written stale entries should stay separate unless deliberately expanded.
