## Summary

Keep this PR open. Current main still has the reported CLI behavior: `plugins enable` and `plugins disable` write policy config from the raw argument before proving that the plugin id exists. The PR is a focused implementation candidate for the linked bug #73551, and it should be reviewed or landed rather than closed as obsolete.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #73551. The best path is maintainer review plus the normal changed validation, then land it or apply the same registry-gated enable/disable behavior on a maintainer branch. After it merges, the linked issue can close as implemented.
