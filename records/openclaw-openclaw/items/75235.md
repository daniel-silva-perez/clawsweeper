## Summary

Keep open. Current main still matches the reported failure class: assistant-only error turns with the sentinel text are treated as already repaired, Anthropic replay can still leave no replayable user message, and the auto-reset detector does not match the Anthropic `messages: at least one message is required` schema error, so the path falls through to the generic external failure. The related empty-payload issues and PRs are adjacent work, but they do not fully cover the requested leading-assistant/no-user transcript reset or the channel reset acknowledgement gap.

## What This Changes

Detect and recover from sessions whose transcript contains only leading or orphan assistant error turns with no user message, then provide clearer post-reset confirmation instead of repeating Anthropic 400 failures.

## Best Possible Solution

Add a focused corruption-recovery path for Anthropic-compatible schema failures caused by zero replayable user messages or leading assistant-only error transcripts. The durable end state should archive the bad transcript, rotate or drop the active session, avoid replaying assistant-only history into the replacement session, return the existing friendly corruption-reset message, and document the new transcript hygiene behavior; any broader reset ping should match the existing gateway fresh-session behavior rather than inventing a separate channel convention.

## Reproduction Assessment

Yes. The issue provides a concrete transcript shape, and current main has a clear source-level reproduction path: a transcript containing only assistant `stopReason: "error"` sentinel messages is not repaired, those turns are filtered before Anthropic transport, and the reset detector does not catch the resulting schema error.

## Solution Assessment

Yes for a narrow auto-reset or read-time repair guard that treats assistant-only/no-user transcripts as corrupt. A generic empty Anthropic-message fallback alone would be incomplete because it can hide the 400 while leaving the poisoned session and unclear reset UX in place.
