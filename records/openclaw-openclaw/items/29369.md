## Summary

Keep this issue open. Current main still does not expose or honor a spawn-time `tools` allowlist on `sessions_spawn`; sub-agent tool policy remains configured through global/per-agent/provider/group/sandbox/subagent policy layers rather than per call.

## Best Possible Solution

Keep this open until maintainers design and implement spawn-time tool policy for `sessions_spawn`. The likely path is a narrow `tools` policy object in the schema/types, propagation through both native and ACP spawn paths, application as a final narrowing step in the effective tool-policy pipeline, and docs/tests that pin precedence and deny-wins behavior.

- Docs and tests need to define precedence against `tools.allow`, `agents.list[].tools`, provider/group policy, sandbox policy, and `tools.subagents.tools`.
