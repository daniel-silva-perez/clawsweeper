## Summary

Keep this issue open. Current main still does not expose or honor a spawn-time `tools` allowlist on `sessions_spawn`; tool policy remains configured through profile/global/provider/agent/group/sandbox/subagent policy layers, not per call.

## Best Possible Solution

Keep this open until maintainers design and implement spawn-time tool policy for `sessions_spawn`. The likely path is to add schema/type support for a narrow `tools` policy object, propagate it through native and ACP spawn paths, apply it as a final narrowing step in the effective tool-policy pipeline, document precedence and deny-wins behavior, and add tests for both runtimes.
