## Summary

Keep this PR open. Current main still lacks the proposed durable `session-state.json` sidecar, so the change is not obsolete, but the current diff has blocking correctness issues in the secondary write path, shared-store attribution, and custom same-directory store layouts.

## What This Changes

The PR adds a session observable-state module and tests, then wires session store writes to emit a `session-state.json` sidecar next to each store.

## Best Possible Solution

Keep this PR open for maintainer review and contributor revision. If OpenClaw wants a core filesystem snapshot, make the sidecar write best-effort, derive per-entry agent identity from canonical session keys, use collision-free snapshot paths for every supported store path, and document the new artifact's privacy and retention behavior. If maintainers prefer existing Gateway, CLI, or Control UI inspection surfaces, close the PR with that product rationale.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Isolate snapshot write failures from store saves:** `src/config/sessions/store.ts:537-540`
  - body: `saveObservableSessionState` runs after `sessions.json` has already been written and caches updated, but it writes through `saveJsonFile`, which can throw on filesystem errors. A failed sidecar write would make callers treat an already-committed primary session-store save as failed, so this auxiliary write needs to be best-effort or otherwise isolated.
  - confidence: 0.92
- **[P2] Scope sidecar names to the source store file:** `src/config/sessions/observable-state.ts:41-42`
  - body: `resolveObservableSessionStatePathForStore` always writes `session-state.json` in the parent directory. Supported templates such as `/tmp/sessions-{agentId}.json` put multiple store files in the same directory, so different agents would overwrite one another's observable snapshots.
  - confidence: 0.88
- **[P2] Derive each entry agent from its session key:** `src/config/sessions/observable-state.ts:77`
  - body: The snapshot passes one `agentId` from the store path into every entry. Shared stores are supported when `session.store` omits `{agentId}`, so entries like `agent:ops:*` in a shared file would be emitted as `agentId: "main"`, breaking filters and attribution.
  - confidence: 0.9
