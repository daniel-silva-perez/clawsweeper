## Summary

Keep open. Current main restores conversation history through `chat.history`, but the queued-message and unsent-draft parts of the report still live in Control UI component state. The supplied GitHub context also shows open PR #73154 declares it fixes this issue, so policy says this issue should stay open until that PR is reviewed and either merged or closed.

## Best Possible Solution

Keep the issue open while maintainers review PR #73154. The best path is a tested implementation that persists and restores queued messages across refresh, reconnect, and session switch, keeps history reload on the existing `chat.history` contract, and either implements draft persistence or creates a clear follow-up before closing this issue.
