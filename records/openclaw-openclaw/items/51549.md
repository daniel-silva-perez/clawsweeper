## Summary

Keep open. Current main partially addresses history reload through `chat.history`, but queued messages and drafts are still component-memory state, and open PR #73154 is an unmerged implementation candidate that declares it closes this issue. Policy says the issue should stay open until that PR is reviewed and landed or closed.

## Best Possible Solution

Keep the issue open while maintainers review PR #73154. The best path is to land a tested implementation that persists/restores queued messages across refresh, reconnect, and session switch, keeps history reload through the existing gateway `chat.history` path, and either implements draft persistence in the same change or tracks that remaining requirement in a narrower follow-up before closing.
