## Summary

Keep this issue open. Current main still has only global MCP server credentials under `mcp.servers`, while per-agent config, runtime merge, and session MCP cache paths have no agent-qualified env/header overlay for same-server identity attribution.

## What This Changes

Add per-agent MCP env/header override support so multiple agents can use the same external MCP server with distinct credentials for correct identity attribution.

## Best Possible Solution

Add a schema-backed, redacted per-agent MCP identity overlay for env and headers, merge it into the effective MCP config before embedded Pi and CLI backend MCP creation, include the effective overlay in runtime/cache fingerprints, and add regression coverage proving two agents can use the same MCP server id with different credentials.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: the proposed `agents.list[].mcp.overrides` shape is rejected by the strict agent schema, and the runtime merge path reads only `cfg.mcp.servers` with no agent input.

## Solution Assessment

Unclear because there is no patch yet. The requested direction is valid, but the best maintainable implementation needs maintainer design for config shape, SecretRef/redaction behavior, and MCP runtime lifecycle isolation.
