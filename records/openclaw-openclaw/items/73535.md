## Summary

Keep open. Current main has a related scope-limited status fix, but it does not conclusively cover this report's default local probe timeout / `Capability: unknown` split where `--require-rpc` and direct `status` RPC still succeed. The default and `--require-rpc` paths remain materially different, and related status/probe work is still open.

## Best Possible Solution

Keep the issue open and either reproduce the default local probe versus `--require-rpc` split on a token-authenticated local/systemd loopback setup or add a focused test that simulates `probeGateway` timeout/unknown capability while direct `status` RPC succeeds. If #64400, #48360, #68280, or #73171 becomes the chosen canonical fix path, preserve this issue's regression-window evidence there before closing it as duplicate/superseded.
