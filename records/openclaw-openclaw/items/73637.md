## Summary

Keep this PR open. Current main still drops the subagent `extraSystemPrompt` whenever `systemPromptOverride` resolves, so the PR is not obsolete or implemented. The PR is a focused implementation candidate for #73624, but it still needs maintainer review because the proposed concatenation changes the documented override contract and has an unresolved prompt-injection concern from the security review.

## Best Possible Solution

Keep the PR open for maintainer review. The likely path is to fix all three prompt-assembly call sites so subagent `extraSystemPrompt` survives `systemPromptOverride`, but maintainers should decide whether the PR's combined-system-prompt approach is acceptable or whether the task block should be delivered through a lower-priority/separate message contract. Add or keep regression tests for no override, override-only, and override-plus-extra behavior, and update the config docs if the override semantics change.

## Risks / Open Questions

- The PR is not implemented on current main, so closing it would leave #73624's reported subagent task-loss path unfixed.

- If the PR lands, `docs/gateway/config-agents.md` likely needs an update because `systemPromptOverride` would no longer strictly replace the entire assembled prompt for runs with `extraSystemPrompt`.
