## Summary

Keep this PR open. Current main still has the exact override short-circuit described in #73624 across the embedded attempt, compaction retry, and CLI harness paths, so this PR is not obsolete or implemented. It remains a focused implementation candidate, but maintainers still need to decide the prompt-authority/security model and update the documented `systemPromptOverride` contract if the concatenation approach lands.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to preserve subagent task instructions in all three prompt assembly paths while keeping operator-defined `systemPromptOverride` authoritative, add regression coverage for no-override, override-only, and override-plus-extra behavior, and update the config docs if the override semantics intentionally change.
