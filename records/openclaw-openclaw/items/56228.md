## Summary

Keep this issue open. Current main still routes Discord reply context through `resolveDiscordMessageText()`, and that resolver does not inspect Discord Components v2 `components` trees for TextDisplay content. The linked fix PR was closed unmerged, and I found no current main implementation or regression test covering Components v2 text extraction from `referenced_message`.

## Best Possible Solution

Implement this in the Discord plugin, not core: add a small recursive Components v2 text extractor in `extensions/discord/src/monitor/message-utils.ts` that reads TextDisplay/content fields from `message.components` and snapshot/referenced-message component trees when normal content/media/embed text is empty, then add focused regression tests in `extensions/discord/src/monitor/message-utils.test.ts` for a Components v2 `referencedMessage` and forwarded snapshot.
