## Summary

Keep this issue open. Current main still builds Discord reply context from `message.referencedMessage`, but `resolveDiscordMessageText()` only considers `content`, media placeholders, first embed text, and fallback text before forwarded-message handling. It does not inspect Components v2 `components` trees or TextDisplay content, and the related fix PR was closed unmerged.

## Best Possible Solution

Implement this in the Discord plugin: add a small recursive Components v2 text extractor in `extensions/discord/src/monitor/message-utils.ts` that reads TextDisplay/content fields from `message.components` and snapshot component trees when normal content/media/embed text is empty, then add focused regression tests in `extensions/discord/src/monitor/message-utils.test.ts` for a Components v2 `referencedMessage` and forwarded snapshot. The closed PR #56248 is useful prior art but should be reviewed or reimplemented against current main.
