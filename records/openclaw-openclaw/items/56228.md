## Summary

Keep this issue open. Current main at 6d323ee still builds Discord reply context from the referenced message, but the shared Discord text resolver only falls back through content, media placeholders, first embed text, and explicit fallback text. It does not inspect Components v2 component trees or TextDisplay nodes, and the related fix PR #56248 was closed unmerged.

## Best Possible Solution

Implement the fix in the Discord plugin: add a small recursive Components v2 text extractor in extensions/discord/src/monitor/message-utils.ts that reads TextDisplay/content fields from message.components and snapshot component trees when normal content/media/embed text is empty, then add focused regression tests in extensions/discord/src/monitor/message-utils.test.ts for a component-only referencedMessage and forwarded message_snapshots. Use closed PR #56248 as prior art, but rework it against current main.
