## Summary

Keep this PR open. Current main still does not implement the intended heartbeat routing change: `forcedSessionKey` is resolved before configured `heartbeat.session`, and preflight only peeks the resolved session queue without migrating wake-triggered events. The PR remains a plausible implementation candidate for #41165, but the discussion still includes a source-session canonicalization/data-isolation concern that should be resolved before landing.

## Best Possible Solution

Keep this PR open as the implementation candidate for #41165. The best path is to finish the source-session canonicalization guard, preserve the already-discussed dedup/identity/skip-restore/trusted protections, and land focused regression tests for configured heartbeat-session precedence plus cross-session event migration. After an equivalent patch merges, the linked bug can close through normal GitHub linkage.

- The PR branch migrates prompt-visible system events across session queues; without canonicalizing and agent-scoping the source session key before draining, it could leak, strand, or remove events across session boundaries.
