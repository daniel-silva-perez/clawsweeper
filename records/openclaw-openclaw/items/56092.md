## Summary

Current main still lacks the core behavior this PR is trying to add: wake-triggered heartbeats can resolve through the originating forced session before configured `heartbeat.session`, and main has no cross-session system-event migration/restore infrastructure. The PR remains a plausible implementation candidate for #41165, but it should not be closed or merged until the unresolved source-session canonicalization/data-isolation review concern is addressed.

## Best Possible Solution

Keep this PR open as an implementation candidate for #41165. The best maintainer path is to finish the source-session canonicalization guard, preserve the trusted/dedup/skip-restore protections already discussed in the PR, and land focused regression tests for configured heartbeat-session precedence plus cross-session event migration. After an equivalent patch merges, the linked bug can close through normal GitHub linkage.
