## Summary

Keep this PR open. Current main still has no Wear OS app, Wear module, Wear proxy, or Wear docs, and the no-stale maintainer signal means this should not be auto-closed; however, the branch is not merge-ready because the new Wear manifest enables app backup while the shared device identity store writes a private signing key under the app files domain.

## What This Changes

The PR adds a Wear OS companion app, shared Android gateway helpers, phone-side Wear proxying, direct watch gateway access, Wear chat/settings UI, tile and complication services, background reply notifications, tests, and Android/Wear docs.

## Best Possible Solution

Keep the PR open while maintainers decide whether Wear OS belongs in-tree. If it does, first fix the Wear backup/private-key exposure, then review or split the work into staged pieces for shared gateway helpers, phone proxying, direct watch auth, UI, notifications, tile/complication services, docs, tests, and fresh paired-device proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Disable backup for Wear identity files:** `apps/android/wear/src/main/AndroidManifest.xml:10-12`
  - body: The new Wear app enables Android backup while its backup and data-extraction rules include the entire file domain.
