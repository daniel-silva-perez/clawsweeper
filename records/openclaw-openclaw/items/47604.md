## Summary

Keep this PR open. Current main still lacks the proposed Wear OS companion app, shared Android gateway module, phone-side Wear proxy, Wear Data Layer dependency, tile/complication services, and Wear documentation. The PR is large and security-sensitive, but it remains the active implementation candidate rather than conservative cleanup noise.

## Best Possible Solution

Keep this PR open unless maintainers explicitly decide not to ship Wear OS support in core. The practical path is maintainer-owned staging: shared Android gateway helpers first, then the phone-side Wear proxy, then the standalone Wear app module, then tile/complication/notification polish with focused tests, Android/Wear docs, fresh device proof, and security review.
