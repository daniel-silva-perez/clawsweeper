## Summary

Keep open. Current main still returns immediately for Mattermost `GROUP_POLICY_NOT_ALLOWLISTED` before pending history is recorded, so this PR is not obsolete or implemented on main. The linked bug report #57607 is still open by the same author and this PR is its implementation candidate.

## Best Possible Solution

Keep this PR open as the implementation candidate for #57607. The best path is to review or revise the one-file Mattermost monitor change, add a focused regression test proving non-allowlisted group messages are recorded to pending history but cannot trigger responses, and clarify docs if maintainers accept that trigger authorization is separate from context retention for Mattermost.
