## Summary

Keep open. Current main still serializes each session lane at one active run, exposes only global and subagent concurrency defaults, keeps the default lane-wait warning at 2 seconds, and has only explicit `sessions_yield` rather than an automatic yield policy. The related open PR #63864 covers only the session-concurrency slice, #14747 separately tracks the warning-threshold slice, and auto-yield still needs maintainer design.

## What This Changes

The issue requests configurable per-session lane concurrency, automatic yielding after tool execution time, and a configurable lane-wait warning threshold for interactive agent sessions.

## Best Possible Solution

Keep this as the umbrella tracker: review or rework #63864 for the per-session concurrency slice, decide whether #14747 remains the canonical lane-wait-threshold tracker, and design auto-yield in the embedded runner/tool loop with explicit transcript, cancellation, and user-message ordering semantics before implementation.
