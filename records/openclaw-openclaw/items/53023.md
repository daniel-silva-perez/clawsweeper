## Summary

Keep open. Current main still has only global and subagent concurrency defaults, dynamically created session lanes still default to one active task, the default lane wait warning remains 2 seconds, and yielding remains explicit through `sessions_yield` rather than an automatic time-based policy. The linked open PR #63864 covers only the session-concurrency slice, while #14747 separately tracks the warning-threshold slice, so this umbrella still needs maintainer staging.

## What This Changes

The issue requests configurable per-session lane concurrency, automatic yielding after tool execution time, and a configurable lane-wait warning threshold for session lanes.

## Best Possible Solution

Keep this as the umbrella tracker: review or rework #63864 for per-session concurrency, keep or consolidate #14747 as the warning-threshold tracker, and design auto-yield in the embedded runner/tool loop with explicit cancellation, transcript, ordering, and session-safety semantics before implementing it.
