## Summary

Keep open. Current main still serializes each session lane at one active run, exposes only global and subagent concurrency defaults, keeps the lane-wait warning fallback at 2 seconds, and only yields when `sessions_yield` is explicitly called. The linked open PR covers only the session-concurrency slice, the linked warning-threshold issue remains separate, and auto-yield needs maintainer design before implementation.

## What This Changes

Request to make per-session lane concurrency, automatic yielding after tool execution time, and lane-wait warning thresholds configurable through agent defaults.

## Best Possible Solution

Keep this as the umbrella tracker: review or rework #63864 for per-session concurrency, decide whether #14747 remains the canonical warning-threshold tracker, and design auto-yield in the embedded runner/tool loop with explicit cancellation, transcript, and ordering semantics before implementing it.
