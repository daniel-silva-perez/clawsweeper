## Summary

Keep open: current main still defers the isolated cron `agentTurn` timeout until the execution-start callback, so the hard wall-time deadline requested here is not implemented. The request conflicts with the recently shipped #41783 behavior that intentionally excludes runner-lane wait from the execution timeout, so this needs maintainer contract judgment rather than an automatic cleanup close or narrow bot repair.

## What This Changes

The issue asks for cron isolated `agentTurn` `timeoutSeconds` to be enforced from a fixed wall-clock deadline set at job dispatch/start, independent of delayed callbacks or async work.

## Best Possible Solution

Define cron timeout semantics explicitly, then update implementation, docs, and regression tests together. A robust endpoint would likely split queue/preparation and execution budgets, or make `timeoutSeconds` a total deadline while adding a separate queue-wait policy so #41783 does not regress silently.
