## Summary

Keep this PR open. Current main covers the timeout/failover classification side for the provider phrase, but it still does not implement the PR's central user-visible behavior: rewriting plain-text provider internal errors into friendly copy with request-id handling. The PR remains a plausible core error-normalization fix, though it needs a rebase and maintainer judgment on exposing raw provider request IDs in chat.

## Best Possible Solution

Keep this PR open and ask for a narrow rebase against current main: keep the existing timeout classification, add the generic plain-text provider internal-error rewrite in the current shared formatter/helper structure, cover `formatAssistantErrorText`, `formatRawAssistantErrorForUi`, and sanitizer behavior as needed, and resolve the raw request-id display policy before merge.
