## Summary

Keep this PR open. Current main now covers the failover classification part for the provider phrase, but it still does not implement the PR's main user-visible behavior: rewriting plain-text provider internal errors into friendly copy with request-id handling. The PR needs a rebase into the current shared formatter layout and a maintainer decision on raw provider request IDs in chat.

## Best Possible Solution

Keep the PR open and ask for a narrow rebase against current main: keep the already-shipped timeout classification, add the generic plain-text provider internal-error rewrite in the current shared formatter/sanitizer structure, cover `formatAssistantErrorText`, `formatRawAssistantErrorForUi`, and sanitizer behavior as needed, and resolve whether raw provider request IDs should be displayed, hashed, or omitted before merge.
