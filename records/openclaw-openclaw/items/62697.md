## Summary

Keep open. Current main still has no WhatsApp Newsletter/Channel inbound support: there is no `newsletterPolicy`, no `@newsletter` handling, and the inbound monitor only separates WhatsApp groups from everything else before applying direct-message resolution/access-control behavior. The related open PR #73393 is outbound message-tool work and explicitly says it is related, not a fix for this inbound newsletter-processing request.

## Best Possible Solution

Implement this in the WhatsApp plugin as a first-class inbound newsletter path: classify `@newsletter` JIDs separately from DMs/groups, add a narrowly scoped `newsletterPolicy` or equivalent allowlist/activation model, preserve safe defaults, and cover it with focused inbound monitor/access-control tests plus WhatsApp docs. Keep #73393 review separate because it addresses outbound newsletter targets, not this inbound processing path.
