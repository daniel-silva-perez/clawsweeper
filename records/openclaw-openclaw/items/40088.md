## Summary

Keep open. Current main has improved the memory watcher target shape compared with the old `memory/**/*.md` glob, but the core resilience reported here is still not implemented: the builtin memory path still defaults `sync.intervalMinutes` to `0`, the interval callback does not mark memory dirty or force a lightweight reconciliation, and watcher setup has no error/ready/health handling. The directly linked fix PR #42042 was closed unmerged, so there is no shipped canonical fix to close against.

## Best Possible Solution

Keep this issue open and implement the resilience in `extensions/memory-core`: add watcher error/liveness logging, decide a maintainer-approved fallback interval default or doctor guidance, and make interval sync actually reconcile memory files by marking dirty or running the existing lightweight stat/hash path without forcing a full reindex. Add regression coverage in `extensions/memory-core/src/memory/manager.watcher-config.test.ts` and `src/agents/memory-search.test.ts`, including explicit override behavior for `intervalMinutes: 0`.
