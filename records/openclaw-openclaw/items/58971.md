## Summary

Keep this PR open. Current main still lacks the proposed cron private-network webhook opt-in and continues to call the SSRF-guarded cron webhook path without a policy. The PR is focused and its supplied diff does not add supply-chain-sensitive files, but it intentionally widens an SSRF-guarded network path when enabled, so the remaining action is maintainer/security review rather than cleanup closure.

## What This Changes

The PR adds an optional `cron.webhookAllowPrivateNetwork` config key, wires it into cron webhook delivery/failure webhook paths as `SsrFPolicy.allowPrivateNetwork`, and updates config schema metadata, docs, and targeted cron tests.

## Best Possible Solution

Keep this PR open for maintainer/security review. If OpenClaw wants an operator opt-in, land this PR or a maintainer-authored equivalent that updates `CronConfig`, strict Zod schema, generated config metadata, help/labels, public docs, and targeted tests; preserves the default blocked behavior; and passes the policy through primary delivery, failure destination, failure alert, and legacy webhook fallback paths. If maintainers prefer the stricter #49120 direction, close this PR as superseded by that hardening path.
