## Summary

Current main still resolves Telegram `requireMention` with persisted activation state before explicit topic config, so this PR's intended fix is not implemented on main. The PR remains a focused implementation candidate for the linked Telegram topic override bug, with one useful inverse regression-test gap still called out in review.

## Best Possible Solution

Keep this PR open as the implementation candidate for #49864. The best path is to land a focused Telegram plugin change that gives explicit topic `requireMention` precedence over activation state, with regression tests for both false and true topic overrides against activation state.
