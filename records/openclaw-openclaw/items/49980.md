## Summary

Keep this PR open. Current main still gives persisted Telegram activation state precedence over explicit topic-level `requireMention`, so the PR’s intended fix is not implemented on main. The PR remains a focused implementation candidate for the linked open bug, with one useful test coverage gap still noted by review.

## Best Possible Solution

Keep this PR open as the implementation candidate for #49864. The best path is to land a focused Telegram plugin change that gives explicit topic `requireMention` precedence over activation state, with regression tests for both `false` and `true` topic overrides against activation state.
