## Summary

Keep this PR open. Current main still resolves Telegram inbound `requireMention` with persisted activation state before explicit topic config, so the PR’s intended behavior is not implemented on main. The linked bug report remains open, and this PR is still a focused implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open as the implementation candidate for the linked Telegram topic `requireMention` bug. The best path is to update or land a focused change in `extensions/telegram/src/bot-message-context.ts` so explicit topic `requireMention` wins over activation state, with regression tests for both `false` and `true` topic overrides against activation state.
