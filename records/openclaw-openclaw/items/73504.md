## Summary

Keep open. Current main only partially addresses the reported hang: the Discord adapter now has the right read/search-only gateway resolver, but the registered Discord channel wrapper still drops that resolver, so the PR contains a useful missing piece. The branch is stale against current main because its unconditional resolver would widen gateway routing beyond the tested read/search scope.

## What This Changes

The PR branch adds an unconditional gateway execution-mode resolver to Discord message actions and forwards that resolver through the registered Discord channel wrapper.

## Best Possible Solution

Update the PR so it keeps current main's read/search-only Discord resolver, adds the missing wrapper forwarding, and adds focused coverage for the registered Discord plugin action surface or the shared message-action runner path for Discord read/search.
