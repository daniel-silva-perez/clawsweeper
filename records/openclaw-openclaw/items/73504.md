## Summary

Keep open. Current main has the Discord adapter-side read/search gateway resolver, but the registered Discord channel wrapper still does not expose `resolveExecutionMode`, so the PR's core routing fix remains relevant. The branch is now stale against current main because its unconditional gateway resolver would conflict with the current read/search-only contract.

## What This Changes

The branch adds an unconditional gateway execution-mode resolver to Discord message actions and forwards that resolver through the registered Discord channel wrapper.

## Best Possible Solution

Update this PR so current main keeps the read/search-only gateway resolver, add the missing wrapper forwarding, and add focused coverage around the registered Discord plugin action surface or message-action runner path for Discord read/search.
