## Summary

Keep this PR open. Current main has partial overlap, but it does not implement the PR’s central macOS behavior: status-item left-click still opens the dismissing panel path, and the shared chat composer is still a fixed VStack with a 64pt text-area cap rather than a resizable macOS composer. The Talk Mode crash portion was fixed separately, and file drag/drop already exists on main, but this PR still contains useful unmerged intent plus unresolved functional and security-sensitive review risks.

## Best Possible Solution

Keep this PR open as salvage material, but ask for a rebased, narrower macOS PR: wire status-item left-click to the persistent NSWindow path, add a macOS-only resizable composer while preserving iOS layout, keep the existing drag/drop behavior, rely on the already-shipped Talk overlay fix, and split or redesign the Gemini/image-generation work through the proper provider/plugin/gateway path before merge consideration.
