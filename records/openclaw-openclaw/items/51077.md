## Summary

Keep this PR open. Current main implements some overlapping pieces, but not the PR’s central behavior: status-item left-click still opens the dismissing macOS chat panel, and the shared chat composer is still capped rather than resizable. The PR also has unresolved functional review findings and security-sensitive unrelated direct Gemini image-generation changes, so it is not obsolete or safely closeable as implemented on main.

## Best Possible Solution

Keep this PR open only as salvage material. The best maintainer path is to ask for a rebased, narrower macOS PR that wires the persistent `NSWindow` chat path to the intended status-item action, adds a macOS-only resizable composer while preserving iOS layout, keeps drag/drop behavior platform-safe, removes or splits the direct Gemini/image-generation work, and addresses the existing P1/P2 review comments before merge consideration.
