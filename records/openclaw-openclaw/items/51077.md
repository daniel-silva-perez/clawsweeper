## Summary

Keep this PR open. Current main overlaps with parts of the PR, but it still does not implement the PR's central macOS status-item behavior: left-click still opens the dismissing panel path, not the persistent resizable NSWindow. The resizable composer and chat-window toolbar/settings work are also still absent. The PR remains useful as salvage material, but the current branch is not merge-ready because review comments identify shared iOS layout regressions and double visibility callbacks, and the later commits mix in unrelated direct Gemini image-generation and secret-handling changes.

## Best Possible Solution

Keep this PR open as salvage material, but ask for a rebased and narrowed macOS PR: wire status-item left-click to the existing persistent NSWindow path, add a macOS-only resizable composer without changing iOS layout, keep or drop already-shipped drag/drop and Talk crash fixes as appropriate, and split the Gemini/image-generation work into the existing provider/plugin/gateway image_generate path for separate product and security review.
