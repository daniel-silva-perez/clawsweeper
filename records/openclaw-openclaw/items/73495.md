## Summary

Keep this PR open. Current main still has the Telegram replyToMode/tool-progress preview mismatch, does not document the mutual exclusion, and has no doctor warning for it. The PR is also the open same-author implementation candidate for #73487, so closing it before review/merge would be premature.

## Best Possible Solution

Keep this PR open for normal maintainer review. If the docs plus `openclaw doctor` warning approach is accepted, land it after confirming the PR head handles Telegram account inheritance and sanitizes config-derived warning text; then #73487 can close through the merged PR.
