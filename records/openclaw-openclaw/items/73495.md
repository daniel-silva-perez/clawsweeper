## Summary

Keep this PR open. Current main still has the Telegram replyToMode/tool-progress preview silence and still lacks a doctor warning or docs note; the PR is a focused implementation candidate, with only small doctor-scanner parity edges to fix before normal maintainer landing.

## What This Changes

The PR adds Telegram doctor warnings, docs and changelog notes, regression tests for replyToMode/tool-progress conflicts, and removes two stale awaits in active-memory tests.

## Best Possible Solution

Land the PR after addressing the small scanner parity issues and normal maintainer/CI review, keeping the runtime behavior unchanged while making the documented trade-off visible through docs and `openclaw doctor`. A separate product follow-up can decide whether Telegram should gain a non-preview fallback for tool-progress when quote-reply is enabled.

## Reproduction Assessment

Yes. The source-level reproduction is a Telegram config with `replyToMode` not `off`, preview streaming active, and default `streaming.preview.toolProgress`; current main shows `answerLane.stream` is not created for native quote replies, and the PR body includes live trace evidence for the same path.

## Solution Assessment

Yes, the docs plus doctor warning approach is the narrowest maintainable fix for the silent doc/runtime mismatch. It should mirror Telegram's existing stream-mode and account-resolution helpers for edge cases before merge; a runtime fallback is a larger product change.
