## Summary

Keep this PR open for maintainer review. Current main has not implemented the proposed fallback change, and the PR still has unresolved correctness/security concerns because the fallback can interact with resolved runtime snapshots while the reported bloat in #65020 points at the explicit empty-provider plugin-discovery path.

## What This Changes

The PR changes `resolveModelsConfigInput(config?)` so an omitted config uses the loaded config for `models.json` planning while keeping `runtimeSource` for secret markers, and adds a regression test for `ensureOpenClawModelsJson(undefined)`.

## Best Possible Solution

Maintainers should either revise this into a tested fallback-path fix that uses a true lean source config while preserving source markers for every SecretInput-backed provider field, including nested `request.*`, or close it manually in favor of a separate provider-discovery decision for the empty-provider bloat behavior described in #65020.
