## Summary

Keep open. Current main has not landed the fallback change, and the proposed patch is not merge-ready: the one-line source switch can make `models.json` planning consume a resolved runtime config while only top-level provider secrets are remapped back to markers, and the added regression test appears broken. The related bloat report also says the main startup bloat path comes from provider discovery with an explicit config, so this PR is at best a partial fix.

## What This Changes

The PR changes the omitted-config path in `resolveModelsConfigInput(config?)` to plan `models.json` from the loaded config while keeping `runtimeSource` for secret markers, and adds a regression test for `ensureOpenClawModelsJson(undefined)`.

## Best Possible Solution

Revise this into a tested fallback-path fix that uses a true lean source config or extends source-managed secret preservation across every provider SecretInput-backed field, including nested `request.*`, then handle the explicit-config provider-discovery bloat as a separate maintainer decision.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve source markers for request secrets:** `src/agents/models-config.ts:121`
  - body: Changing the omitted-config branch to pass `loaded` makes this path plan from the resolved runtime snapshot when secrets runtime is active. Provider `request.*` SecretRefs are resolved to plaintext there, but the models writer only restores source markers for `apiKey` and top-level `headers`, so this can persist request tokens, TLS keys, or proxy TLS material into `models.json`. Keep the source snapshot for planning or add marker restoration for the full provider request surface before switching this value.
  - confidence: 0.88
- **[P2] Fix the new regression test harness call:** `src/agents/models-config.runtime-source-snapshot.test.ts:375-383`
  - body: The added test calls `withTempHome()` even though this test file does not import or define it, and it calls `expectGeneratedProviderApiKey()` with two arguments although the helper requires `agentDir`, `providerId`, and `expected`. As written, the test cannot reliably validate the new branch and should fail before proving the regression path.
  - confidence: 0.91
