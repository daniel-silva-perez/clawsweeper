## Summary

Keep open. The report is current, plausible, and already has open linked PR #74425 with closing syntax; current main still routes `status` through the short-lived CLI path and does not contain a merged lifecycle fix for the Windows lingering-process symptom.

## What This Changes

The issue reports that Windows npm CLI invocations such as `openclaw.mjs status` and `openclaw.mjs --version` leave short-lived `node.exe` processes alive after completing.

## Best Possible Solution

A focused fix should make supported short-lived CLI commands release process-wide resources or otherwise terminate cleanly on Windows npm installs, with smoke proof for `status` and `--version`, while preserving long-running gateway/provider behavior.
