Keep open. The report is current, the supported `status` reproduction still maps to a routed short-lived CLI path on the checked-out main, and the issue is already paired with open unmerged PR #74425 using closing syntax.

## What This Changes

The issue asks OpenClaw to stop short-lived Windows npm CLI commands, especially `status`, from leaving `node.exe ... openclaw.mjs` processes alive after completion.

## Best Possible Solution

Review PR #74425 against the current CLI lifecycle and land either a scoped teardown/exit fix or a narrower resource cleanup that makes Windows npm `status` exit promptly, with a smoke check for `status` and `--version` while preserving long-running gateway/provider behavior.
