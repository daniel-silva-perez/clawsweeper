## Summary

Keep open. Current main still has a plausible Windows short-lived CLI lifecycle regression and no merged implementation; the linked open PR #74425 references this issue with closing syntax, so the issue should stay open until that PR or another fix lands.

## What This Changes

The issue asks for Windows npm CLI invocations such as `node .../openclaw.mjs status` and `--version` to exit cleanly without lingering `node.exe` processes.

## Best Possible Solution

Land a focused CLI lifecycle fix that cleans up short-lived routed-command resources on Windows, likely around env-installed Undici/global proxy/debug-capture teardown, with Windows npm smoke proof for `status` and `--version` and without changing long-running gateway/provider proxy behavior.
