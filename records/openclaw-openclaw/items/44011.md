## Summary

Keep open. Current main already exposes some message-hook correlation fields, including `sessionKey`, but it still does not expose `agentId` on the public typed message hook context or thread resolved agent identity through the inbound and outbound plugin hook paths. The PR addresses a real remaining plugin-contract gap, but it needs maintainer review and rebase/adaptation to current hook type locations before merge.

## What This Changes

The PR branch adds optional `sessionKey` and `agentId` to typed plugin message hook contexts, propagates them through inbound `message_received` and outbound `message_sending`/`message_sent` plumbing, and extends related mapper and delivery tests.

## Best Possible Solution

Keep the PR open and adapt it to current main: add optional `agentId` to the public message hook context and canonical mapper types, pass the resolved inbound dispatch session/agent identity into `message_received`, resolve outbound hook context once with mirror identity preferred and use it consistently for `message_sending` and `message_sent`, document `ctx.agentId` on the plugin hooks page, and cover mirror, session-only, and unset fallback paths in targeted tests.
