## Summary

Keep open. Current main still lacks `agentId` on typed plugin message hook context and still does not pass session/agent identity through the outbound `message_sending` and `message_sent` typed contexts. The PR addresses a real public plugin-contract gap, but it needs maintainer review and rebase onto the current hook type layout before merge.

## What This Changes

The PR adds optional `sessionKey` and `agentId` to typed plugin message hook contexts, threads them through inbound and outbound hook delivery, and expands mapper and delivery tests.

## Best Possible Solution

Rebase the change onto current main by adding optional `agentId` to `src/plugins/hook-message.types.ts` and the canonical mapper types, passing the active inbound dispatch session/agent identity into `message_received`, resolving outbound hook context once with mirror identity preferred and session fallback, documenting `ctx.agentId` on the plugin hooks page, and covering mirror, session-only, and unset fallback paths in targeted tests.
