## Summary

Keep open. Current main still does not expose `agentId` on the public typed message-hook context, and the inbound/outbound hook call sites do not pass agent identity into `message_received`, `message_sending`, or `message_sent`. The PR is a relevant additive plugin-contract change, but it needs maintainer review and a careful rebase onto the current split hook type files before merge.

## What This Changes

The PR adds optional `sessionKey` and `agentId` to typed plugin message hook contexts, threads them through inbound and outbound message hook delivery, and expands mapper and delivery tests.

## Best Possible Solution

Land a rebased additive contract update that adds optional `agentId` to the current message-hook type and canonical mapper, passes active inbound session identity into `message_received`, resolves outbound hook context once with mirror identity preferred and session fallback, documents `ctx.agentId`, and covers mirror, session-only, and unset paths in targeted tests.
