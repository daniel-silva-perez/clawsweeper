## Summary

Keep this PR open. Current main still does not expose `agentId` on the public typed message-hook context or thread session/agent identity through the inbound and outbound typed message hook call sites; the PR remains a focused implementation candidate, but it needs rebase/adaptation to the current hook type split and public-contract docs/coverage before maintainer landing.

## What This Changes

The PR adds optional `sessionKey` and `agentId` to typed plugin message hook contexts and threads them through inbound `message_received` plus outbound `message_sending` and `message_sent` delivery paths.

## Best Possible Solution

Keep the PR open and adapt it to the current hook type locations: add optional `agentId` to the public message hook context and canonical mapper types, thread resolved inbound session/agent identity into `message_received`, resolve outbound session context once with mirror identity preferred, pass that same context to `message_sending` and `message_sent`, document `ctx.agentId` on `https://docs.openclaw.ai/plugins/hooks`, and cover the key fallback/precedence paths in targeted tests.
