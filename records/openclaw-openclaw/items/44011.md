## Summary

Keep open. Current main still does not expose `ctx.agentId` on typed message hook contexts, so the underlying plugin contract gap remains, but this PR is not mergeable as-is because its outbound helper can combine a mirror session key with the sender session's agent id and it targets an older hook type location that has since been split.

## What This Changes

The PR adds optional `sessionKey` and `agentId` fields to typed plugin message hook contexts, threads them through inbound and outbound message hook delivery, and adds mapper, dispatch, and delivery tests.

## Best Possible Solution

Land a rebased additive hook contract update in the current split hook type files that exposes optional `ctx.agentId`, derives it from the same selected inbound or outbound session identity as `ctx.sessionKey`, documents the field for plugin authors, and covers unset, session-only, mirror-with-agent, and mirror-without-agent cases.

## Reproduction Assessment

Yes. A source-level reproduction is clear: current main's message hook context and mappers have no `agentId`, and the PR diff reproduces the proposed-patch bug by passing a sender `session.agentId` plus a `mirror.sessionKey` without `mirror.agentId`.

## Solution Assessment

No. The additive contract direction is reasonable, but this PR is not the best mergeable solution until it rebases to `src/plugins/hook-message.types.ts`, keeps outbound session identity internally consistent, and updates the public docs.
