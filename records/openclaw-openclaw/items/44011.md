## Summary

Keep this PR open. Current main still does not implement the PRâ€™s intended public typed message-hook context: `sessionKey` is present in some current message-hook types and mappers, but `agentId` is absent from the public message context, inbound `message_received` does not pass resolved agent identity into the typed context, and outbound `message_sending` / `message_sent` typed contexts still omit session or mirror identity. The PR remains a focused implementation candidate, with one known coverage gap from review discussion.

## Best Possible Solution

Keep this PR open. The best path is to rebase it onto current main, add optional `agentId` to the public typed message hook context and canonical mapper types, thread resolved inbound session/agent identity into `message_received`, resolve outbound hook context once with mirror identity preferred over direct session identity, pass that same context to `message_sending` and `message_sent`, add no-session/session-only/mirror-wins tests for success and failure paths, and document `ctx.agentId` on https://docs.openclaw.ai/plugins/hooks.
