## Summary

Keep open. Current main still lacks `ctx.agentId` on typed message hook contexts, so the underlying plugin-contract request remains valid; the PR is useful but not merge-ready because it targets an older type layout, leaves docs unsynced, and can mix mirror and sender session identity.

## What This Changes

The PR adds optional `sessionKey` and `agentId` to typed plugin message hook context plumbing for inbound and outbound message hooks, with mapper, dispatch, and delivery tests.

## Best Possible Solution

Land a rebased additive hook contract update in the current split type files that exposes optional `ctx.agentId` alongside `ctx.sessionKey` for `message_received`, `message_sending`, and `message_sent`, derives both fields from the same selected session identity, and documents the new field for plugin authors.

## Reproduction Assessment

Yes. A source-level reproduction is clear: current main's typed message context and mapper have no `agentId`, and outbound typed hook calls do not pass session identity. The PR diff also reproduces the proposed-patch bug with a mirror session key falling back to a sender session agent id.

## Solution Assessment

No. The additive API direction is reasonable, but this branch is not the best mergeable solution until it rebases to the current hook type files, keeps mirror/session identity internally consistent, and updates the public hooks docs.
