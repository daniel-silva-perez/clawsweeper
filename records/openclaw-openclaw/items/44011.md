## Summary

Keep open. Current main still lacks `ctx.agentId` in typed message hook contexts, so the PR is not obsolete; however the branch has a concrete mirror/session identity bug and a public-docs gap before it is merge-ready.

## What This Changes

The PR adds optional `sessionKey` and `agentId` to typed plugin message hook contexts, threads them through inbound and outbound hook delivery, and expands mapper, dispatch, and delivery tests.

## Best Possible Solution

Land a rebased additive contract update in the current hook type files that exposes optional `ctx.agentId`, maps it from the active inbound session and resolved outbound delivery identity, derives `agentId` from the same selected session key, documents the new field, and covers unset, session-only, mirror-with-agent, and mirror-without-agent paths in targeted tests.

## Reproduction Assessment

Yes. A source-level reproduction is clear from the PR diff: pass a sender `session` with `agentId` plus a `mirror` with only `sessionKey`, and `resolveMessageHookContext` produces a mirror session key paired with the sender agent id.

## Solution Assessment

No. The additive contract direction is reasonable, but the current patch is not the best mergeable solution until it keeps session identity internally consistent, rebases to the current hook-message type split, and updates the public plugin hooks docs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Keep mirror session identity together:** `src/infra/outbound/deliver.ts:414`
  - body: When `mirror.sessionKey` is present but `mirror.agentId` is absent, this falls back to `session.agentId`, so hooks can receive a mirror session key paired with the sender session's agent. That breaks the correlation contract for plugins; derive the agent from the selected mirror session key or omit it until that identity is available.
  - confidence: 0.74
- **[P3] Document the new plugin context field:** `src/plugins/types.ts:564-565`
  - body: This adds a public typed hook context field but leaves the plugin hooks docs' message context field list without `ctx.agentId`. Update `docs/plugins/hooks.md` alongside the API change so plugin authors can discover the new contract.
  - confidence: 0.67
