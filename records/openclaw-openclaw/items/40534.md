## Summary

Keep open for maintainer handling. The PR targets a real cron announce reasoning leak, but the branch is stale against current main's shared cron outcome path and misses the current multi-payload delivery selector; newer open #73363 overlaps but differs on the reasoning-only fallback policy.

## What This Changes

The PR preserves `isReasoning` on embedded-runner payloads, filters reasoning/thinking payloads from isolated cron announce selection and error recovery, and adds focused helper tests.

## Best Possible Solution

Land one canonical rebased fix in the shared cron outcome path: preserve `isReasoning` from `buildEmbeddedRunPayloads`, skip reasoning in summary/output/fallback, single- and multi-payload delivery, and post-error recovery, with explicit policy for reasoning-only payloads and regression coverage for mixed reasoning plus final text, reasoning-only, and error-then-reasoning-only announce cases.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists with isolated cron payloads containing reasoning text: current main creates but drops the marker, and the shared cron outcome helpers do not consult `isReasoning`; #73186 also reports the same leak live for Matrix/Feishu on v2026.4.24.

## Solution Assessment

No. The PR fixes important older call sites, but current main routes final and interim cron output through `resolveCronPayloadOutcome`, so the maintainable fix needs to move filtering there and cover the multi-payload selector too.
