## Summary

Keep this PR open for maintainer review. The PR addresses a real cron announce reasoning leak, but current main has moved final and interim delivery through `resolveCronPayloadOutcome()` and added multi-payload delivery selection, so the stale branch no longer fixes the full live path after rebase.

## What This Changes

The PR preserves `isReasoning` on embedded-runner payload results, adds it to the embedded result type, filters reasoning payloads from cron announce helper selection plus caller fallback/recovery logic, and adds helper regression tests.

## Best Possible Solution

Land one canonical rebased fix in the shared cron outcome path: propagate `isReasoning` from `buildEmbeddedRunPayloads`, carry it through the cron payload type, skip it in summary/output, single- and multi-payload delivery, and post-error recovery, then add regression coverage for mixed reasoning plus final text, reasoning-only, and error-then-reasoning-only announce cases.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: `buildEmbeddedRunPayloads()` creates a reasoning item, drops the marker in the return map, and `resolveCronPayloadOutcome()` then selects payloads using helpers that only understand `isError`.

## Solution Assessment

No. The source PR fixes important older call sites, but current main centralizes final and interim cron delivery through `resolveCronPayloadOutcome()` and has a multi-payload selector that the PR does not update.
