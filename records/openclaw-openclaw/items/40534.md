## Summary

Keep this PR open: current main still has the cron announce reasoning-leak gap, and the PR contains useful source work, but it needs rebase/adaptation because cron payload resolution has moved into shared helpers and newer overlapping work exists.

## What This Changes

The PR propagates `isReasoning` through embedded run payloads and filters reasoning payloads from cron isolated-agent summary, delivery, fallback, and error-recovery selection paths.

## Best Possible Solution

Rebase this work onto current main and implement the filtering in the shared cron outcome path: preserve `isReasoning` from `buildEmbeddedRunPayloads`, include it in the cron payload type, skip reasoning in summary/output/deliverable and multi-payload delivery selection, skip it in the first-text fallback and post-error recovery checks, and add regression coverage for mixed reasoning plus final text, reasoning-only, and error-then-reasoning-only cron announce cases.
