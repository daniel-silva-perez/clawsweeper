## Summary

Keep this PR open for maintainer handling. Current main still has the cron reasoning-leak gap, but this branch is not mergeable as-is because it targets the older caller flow and misses the current multi-payload delivery selector; open #73363 overlaps with the same bug but uses different reasoning-only fallback semantics.

## What This Changes

The PR propagates `isReasoning` through embedded run payloads, adds the field to the embedded run result type, filters cron helper selection plus the old run fallback/error-recovery paths, and adds focused helper tests.

## Best Possible Solution

Land one canonical rebased fix in the shared cron outcome path: preserve `isReasoning` from `buildEmbeddedRunPayloads`, include it in cron delivery payload typing, skip reasoning in summary/output/first-text fallback, deliverable and multi-payload selection, and post-error recovery checks, with regression coverage for mixed reasoning plus final text, reasoning-only, and error-then-reasoning-only cron announce cases.
