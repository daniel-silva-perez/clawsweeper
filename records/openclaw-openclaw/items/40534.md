## Summary

Keep open. The PR targets a real cron announce reasoning leak, but it is stale against current main: current cron delivery now flows through the shared outcome helper, and the proposed patch does not cover that helper's multi-payload delivery path. It should not merge as-is; maintainers should choose a canonical rebased fix alongside the overlapping #73363 work.

## What This Changes

The PR propagates `isReasoning` through embedded runner payload typing and serialization, filters isolated-cron announce helper selection plus caller fallback/error-recovery logic, and adds focused helper tests.

## Best Possible Solution

Land one canonical rebased fix in the shared cron outcome path: preserve `isReasoning` from `buildEmbeddedRunPayloads`, skip reasoning in summary/output/fallback, single- and multi-payload delivery, and post-error recovery, with regression coverage for mixed reasoning plus final text, reasoning-only, and error-then-reasoning-only announce cases.
