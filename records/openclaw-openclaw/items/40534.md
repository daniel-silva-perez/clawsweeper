## Summary

Keep this PR open for maintainer handling. The source work targets a real cron announce leak, but the branch is stale against current main’s shared `resolveCronPayloadOutcome` path and misses the current multi-payload delivery selector, so it is not merge-correct as-is. The newer open #73363 overlaps the same bug but differs on the reasoning-only fallback policy, so a maintainer should choose the canonical outcome before landing or replacing either PR.

## What This Changes

The PR propagates `isReasoning` through embedded runner payload typing and serialization, filters isolated-cron announce helper selection plus caller fallback/error-recovery logic, and adds focused helper tests.

## Best Possible Solution

Land one canonical rebased fix in the shared cron outcome path: preserve `isReasoning` from `buildEmbeddedRunPayloads`, skip reasoning in summary/output/fallback, single- and multi-payload delivery, and post-error recovery, with explicit policy for reasoning-only payloads and regression coverage for mixed reasoning plus final text, reasoning-only, and error-then-reasoning-only announce cases.
