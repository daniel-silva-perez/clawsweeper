## Summary

Keep this PR open. Current main still does not implement the PR's intended end-to-end cron announce reasoning filter: reasoning payloads are created and the embedded result type exposes `isReasoning`, but the payload builder drops the flag, cron outcome selection does not filter reasoning, and cron tests do not cover the regression. The newer related #73363 and open #73186 confirm the bug is still active, but #73363 does not fully supersede this PR's stricter behavior around deliverability and post-error recovery.

## Best Possible Solution

Keep this PR open for maintainer review or consolidation with #73363. The best implementation path is to propagate `isReasoning` from `buildEmbeddedRunPayloads`, include it in the cron delivery payload surface, make `resolveCronPayloadOutcome` skip reasoning in summary/output/deliverable selection and first-text fallback, exclude reasoning from post-error recovery success checks, and add regression tests for mixed reasoning plus final text, reasoning-only output, and error-then-reasoning-only announce cases.
