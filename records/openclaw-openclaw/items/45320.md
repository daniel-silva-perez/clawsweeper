## Summary

Keep this PR open. Current main still does not implement an independent Discord partial-mode preview boundary rotation option: preview splitting is still hard-gated to `block`, the strict nested streaming config has no boundary-rotation field, and the docs/tests only cover the existing block-mode behavior. The PR remains a plausible implementation candidate, but it needs a current-layout port before maintainer review.

## Best Possible Solution

Keep this PR open as an implementation candidate. Port the feature into the current `extensions/discord` plugin and nested streaming config model, add a resolver that preserves block-mode default rotation while allowing partial-mode opt-in, replace the hardcoded block-only split gate in `message-handler.process.ts`, update schema/types/UI hints/docs/generated metadata, add partial-mode assistant-boundary and reasoning-boundary regression tests, and address the in-flight send failure while preserving mention suppression.
