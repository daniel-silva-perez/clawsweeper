## Summary

Keep this PR open. Current main still does not implement an independent Discord partial-mode preview boundary rotation option: Discord preview splitting is still hard-gated to `block` mode, the strict nested streaming config has no boundary-rotation field, and current docs/tests only describe or exercise the existing block-mode behavior. The PR is still a plausible implementation candidate, but it needs to be ported to the current `extensions/discord` plugin layout and nested `streaming.preview` config shape before maintainer review/merge.

## Best Possible Solution

Keep this PR open as an implementation candidate. Port the feature into the current Discord plugin surface under the nested streaming config model, add a resolver that preserves block-mode default rotation while allowing partial-mode opt-in, wire `extensions/discord/src/monitor/message-handler.process.ts` away from the hardcoded block-only split gate, update schema/types/config UI hints/docs/generated metadata, add partial-mode assistant-boundary and reasoning-boundary regression tests, and address the stale in-flight send failure while preserving mention suppression.
