## Summary

Keep this PR open. Current main still does not expose an independent Discord partial-mode preview rotation option: the runtime split gate is still hardcoded to block mode, the current strict nested streaming config only exposes preview chunk/tool-progress fields, and the docs/tests only describe or assert block-mode boundary rotation. The PR remains a useful implementation candidate, but it needs a port to the current `extensions/discord` plugin layout and nested streaming config shape before it can be merged.

## Best Possible Solution

Keep this PR open as an implementation candidate. Port the feature into the current Discord plugin surface, preferably under the existing nested streaming preview config shape; add a resolver that preserves block-mode default rotation while allowing partial-mode opt-in; wire `extensions/discord/src/monitor/message-handler.process.ts` away from the hardcoded block-only split gate; update strict schema/types/config UI hints/docs/generated metadata; add partial-mode assistant-boundary and reasoning-boundary regression tests; and finish the stale in-flight send failure handling while preserving Discord mention-suppression behavior.
