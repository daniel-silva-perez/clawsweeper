## Summary

Keep open. The Control UI LAN display URL problem still exists on current main, but this branch is not merge-ready because the status --all hunk is stale against the current formatter and the branch now mixes the URL fix with unrelated agent error-copy changes and a sensitive bundled Drop Box intake skill.

## What This Changes

The PR coerces several Control UI display URL callers from `lan` to `loopback`, changes embedded-agent error message formatting/tests, and adds a bundled `dropbox-intake` skill plus PDF rendering helper.

## Best Possible Solution

Land a narrow current-main fix that centralizes display-only LAN-to-loopback normalization for browser Control UI URLs and applies it to dashboard, configure completion, status overview/status-all, daemon status, and setup finalization, while preserving runtime bind, probe, custom-bind, tailnet, and TLS semantics. Split the embedded-agent error-copy changes and Drop Box skill into separate reviews.
