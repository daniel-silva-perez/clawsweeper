## Summary

Keep open. The underlying LAN display URL problem is still valid on current main, but this PR is not mergeable as submitted because part of the URL fix targets stale status code and the branch now contains unrelated security/privacy-sensitive changes.

## What This Changes

The PR coerces several Control UI display URL callers from `lan` to `loopback`, changes embedded-agent error message formatting/tests, and adds a bundled `dropbox-intake` skill with a PDF rendering helper.

## Best Possible Solution

Land a narrow current-main fix that centralizes display-only LAN-to-loopback normalization for browser Control UI URLs and applies it to dashboard, configure completion, status overview/status-all, daemon status, and setup finalization, while preserving runtime bind, probe, custom-bind, tailnet, basePath, and TLS semantics. Split the embedded-agent error-copy work and Drop Box document-intake skill into separate reviews.
