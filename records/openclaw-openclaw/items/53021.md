## Summary

Keep open. The underlying LAN display URL bug still exists on current main, but this branch is not safe to merge as-is because it mixes the valid URL fix with stale status-command hunks and unrelated security/privacy-sensitive agent-error and bundled-skill changes.

## What This Changes

The branch normalizes some Control UI display URL call sites from `lan` to `loopback`, changes embedded-agent error formatting and payload tests, and adds a Drop Box document-intake skill with a PDF renderer.

## Best Possible Solution

Land a narrow current-main fix that centralizes display-only LAN-to-loopback normalization for browser Control UI URLs and applies it to dashboard, configure completion, status overview/status-all, daemon status, and setup finalization while preserving probe/runtime bind, custom bind, tailnet, basePath, and TLS semantics. Split the embedded-agent error-copy work and Drop Box skill into separate reviews.
