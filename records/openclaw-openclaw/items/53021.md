## Summary

Keep this PR open for maintainer action, not cleanup-close. Current main still has the underlying display-URL gap outside `openclaw dashboard`: `resolveControlUiLinks` returns a LAN IP for `bind="lan"`, while configure/status/status-all/onboarding/daemon-status display paths still pass the raw bind through. The PR’s central Control UI fix is useful, but the branch has grown dirty with unrelated agent error-message changes and a new Dropbox skill, including security-sensitive review comments about leaking internal context in user-facing error text.

## Best Possible Solution

Keep the PR open for maintainer cleanup or replacement. The best path is a narrow Control UI display fix that centralizes or consistently applies `lan -> loopback` for user-facing dashboard/status/configure/onboarding URLs, adds focused regression coverage for those display paths, and removes the unrelated Dropbox skill plus agent error-message changes into separate reviewed PRs if they are still desired.
