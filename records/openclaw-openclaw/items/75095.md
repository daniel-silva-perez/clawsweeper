## Summary

Keep open. The latest PR head fixes the earlier allowlist and argv-cap review comments, but the new secret-flag classifier still misses supported key-bearing secret flags, leaving a concrete credential-at-rest leak in the config audit log.

## What This Changes

The PR centralizes config-audit process snapshotting, redacts argv and execArgv, updates write/observe/recovery audit call sites, adds focused tests, and records the fix in the changelog.

## Best Possible Solution

Keep the centralized config-audit snapshot boundary, but make secret-flag detection cover supported credential-bearing CLI names such as private/recovery keys, preferably from credential/setup metadata or a broader well-tested sensitive-name classifier, then add regression tests for those flags across the write and observe audit paths.

## Reproduction Assessment

Yes. Current main has a source-level reproduction through any config write or observe audit with a secret in the first eight argv entries; the PR finding reproduces by passing a supported arbitrary key after an omitted flag such as `--private-key` or `--recovery-key`.

## Solution Assessment

No, not yet. Centralizing audit process snapshot redaction is the right shape, but the current classifier is still too narrow for existing OpenClaw secret-bearing CLI surfaces, so it is not the safest complete fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Redact key-valued secret flags:** `src/config/io.audit.ts:47-48`
  - body: The classifier only treats explicit names and suffixes such as token/secret/password/api-key as secret-bearing. Supported command surfaces still accept arbitrary secrets via `--private-key` for Nostr and `--recovery-key` for Matrix, and the per-element fallback does not reliably recognize those standalone values, so config writes can still persist them to `config-audit.jsonl`. Cover these key-valued secret flags, or source the list from credential metadata, and add tests for them.
  - confidence: 0.9
