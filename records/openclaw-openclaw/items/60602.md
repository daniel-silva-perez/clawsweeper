- A naive automatic injection could expose mutable or sensitive agent display names in AWS CloudTrail or invocation logs.
- AWS Bedrock request metadata has key/value/count and reserved-prefix constraints, so the final implementation should validate or sanitize before forwarding.
- The current provider hook contexts do not expose agent identity, so a provider-owned fix may need a small generic seam rather than hardcoding Bedrock behavior in unrelated core paths.
