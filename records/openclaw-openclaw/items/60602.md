## Risks / Open Questions

- A naive automatic injection could expose mutable or sensitive agent display names in AWS CloudTrail or invocation logs; a fix should use conservative values and document the exposure.
- AWS Bedrock request metadata has key/value/count and reserved-prefix constraints, so the implementation should validate or sanitize before forwarding.
- If implemented through provider hooks, the hook context likely needs a generic documented agent identity field rather than hardcoding Bedrock-specific behavior in unrelated core paths.
