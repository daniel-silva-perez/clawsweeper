## Summary

Keep open. Current main still has the reported gap in the native `/think` menu path: Ollama exposes non-off levels only when catalog `reasoning` metadata reaches its provider thinking profile, but the command-choice/menu resolver still carries only provider/model context. There is also an open implementation PR linked with closing syntax, so this issue should stay open until that work is reviewed and merged or replaced.

## Best Possible Solution

Keep this issue open and use PR #73568, or a replacement, as the implementation vehicle. The robust fix is to add catalog-aware command-choice/menu plumbing, pass the runtime model catalog from native menu surfaces, keep Ollama-specific capability policy inside the Ollama provider/profile, and add regression coverage for an Ollama catalog entry with `reasoning: true` showing `off, low, medium, high, max`. Before closing, also verify the reported inline `/think` and `thinkingDefault` paths against current main so any remaining non-menu bug is either fixed or tracked separately.
