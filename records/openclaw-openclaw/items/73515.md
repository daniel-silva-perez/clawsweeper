## Summary

Keep open. Current main still matches the reported Telegram native `/think` menu gap: Ollama's provider profile exposes `low`, `medium`, `high`, and `max` only when `reasoning === true`, but the native command choice path only passes provider/model and never passes the model catalog, so catalog-only Ollama reasoning metadata is lost for the menu.

## Best Possible Solution

Keep this issue open and fix the generic command-choice pipeline: extend the command argument choice context or native menu resolver so it can pass the runtime model catalog/thinking catalog into `listThinkingLevels`, then add a focused regression test for a Telegram `/think` menu on an Ollama catalog entry with `reasoning: true`. Avoid hardcoding Ollama behavior in core; the provider profile should remain plugin-owned.
