## Summary

Keep open. Current main still matches the reported Telegram native `/think` menu gap: Ollama only advertises non-off thinking levels when catalog `reasoning === true` reaches its provider profile, but the native command menu path still resolves choices with only provider/model and has no catalog field to pass through.

## Best Possible Solution

Keep this issue open and fix the generic native command-choice pipeline: add a model/thinking catalog field to the command argument choice context or menu resolver, pass the runtime catalog from Telegram and any other native menu surfaces, then add a focused regression test for a Telegram `/think` menu on an Ollama catalog entry with `reasoning: true`. Avoid hardcoding Ollama-specific behavior in core; the provider profile should stay plugin-owned.
