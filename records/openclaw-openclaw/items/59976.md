## Summary

Keep this PR open. Current main partially supersedes the original remote-avatar symptom by stripping remote http(s) avatar URLs before rendering, but it does not implement the PR's full intended change: the no-basePath logo helper is still route-relative, and the current split chat avatar components still render accepted avatar image URLs without an error fallback.

## Best Possible Solution

Do not close this PR as implemented. The best path is to rebase or port the useful parts onto current main: make no-basePath bundled logo/avatar fallbacks root-absolute if that is still desired, add scoped image-error fallback handling in `chat-welcome.ts` and `chat-avatar.ts` for accepted local/blob/data avatar failures, keep the remote-avatar stripping policy intact, and update the current colocated tests (`agents-utils.test.ts`, `chat.test.ts`, and `chat-avatar.test.ts`) before maintainer review.
