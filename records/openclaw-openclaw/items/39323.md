## Summary

Keep this PR open. Current main still lacks the requested DM pairing owner notification flow, the proposed `pairing:request` event, and the `pairing.notify` config surface. The adjacent device-pair notifier only covers node/device pairing through Telegram, and related #38998 was closed as superseded by this PR rather than implemented on main.

## Best Possible Solution

Keep this PR open as the active implementation candidate, or replace it with a maintainer-owned version. The best path is a rebased, documented, versioned `pairing:request` internal/plugin hook emitted only for fresh DM pairing requests, plus owner notification with sanitized fixed-format content, channel target normalization, PII-safe logs, schema/docs alignment, and focused tests around the shared pairing path.
