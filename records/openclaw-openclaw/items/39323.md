## Summary

Keep this PR open. Current main still does not implement owner notifications for fresh DM pairing requests, a `pairing:request` hook, or a top-level `pairing.notify` config surface. The related closed #38998 does not supersede this PR because the implementation work is still absent from main.

## Best Possible Solution

Keep this PR open as the active implementation candidate, or replace it with a maintainer-owned version. The best path is a documented, versioned `pairing:request` internal/plugin hook emitted only for fresh DM pairing requests, plus owner notification with sanitized fixed-format content, PII-safe logging, channel target normalization, config/schema/docs alignment, and focused tests around the shared pairing path.
