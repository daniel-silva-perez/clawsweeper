## Summary

Keep this PR open. Current main still lacks owner notifications for fresh DM pairing requests, has no internal or typed plugin `pairing:request` hook, and has no top-level `pairing.notify` config surface. The related closed issue #38998 does not supersede the PR because the implementation work is still absent from main.

## Best Possible Solution

Keep the PR open as the active implementation candidate, or replace it with a narrower maintainer-owned design. The best path is to add a documented, versioned `pairing:request` event or plugin hook emitted for freshly created DM pairing requests, then implement owner notification with fixed-format sanitized text, PII-safe logs, target normalization, and focused tests across the shared pairing path.
