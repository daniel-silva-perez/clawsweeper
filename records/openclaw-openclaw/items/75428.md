## Summary

Close as implemented on main: the reported 4.29 slowdown maps to the shared embedded-run preparation path, and current main has unreleased agent/tool changes that move optional generation provider discovery and related catalog work out of fresh reply setup.

## What This Changes

The issue reports that after updating to 4.29, Weixin and Feishu replies take about 59 seconds in embedded-run prep, especially `system-prompt`, before answering.

## Best Possible Solution

Keep the shared embedded-run/tool-registration optimization on main and ship it in the next release; if the reporter still sees slow prep after a 2026.4.30-or-newer build, ask for a fresh prep-stage trace from the same channel setup.
