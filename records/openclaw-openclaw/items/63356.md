## Summary

Keep open. Current main still has the Signal shared allowlist and DM-policy normalizers on phone-only E.164 paths, while the documented and tested Signal pairing behavior supports `uuid:<id>` senders; the PR remains a focused Signal-owned fix that has not been absorbed into main.

## What This Changes

The PR changes the Signal plugin shared adapter to preserve `uuid:` sender IDs in `allowFrom` and DM-policy normalization while keeping E.164 phone normalization, with regression tests for both paths.

## Best Possible Solution
