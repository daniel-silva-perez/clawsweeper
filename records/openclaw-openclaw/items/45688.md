## Summary

Current main still has the reported Control UI config usability regression. Advanced Settings is routed through the accordion layout, but the search control is only rendered in the tab layout, and the form filtering still applies active-section filtering before search, so search remains section-scoped rather than global.

## Best Possible Solution

Keep this open and make a focused Control UI fix: expose search/filter controls in Advanced Settings accordion mode, make search operate globally across the included config sections rather than only the selected section, and preserve Quick Settings plus the newer scoped top-level tabs unless maintainers choose a broader v2 redesign.
