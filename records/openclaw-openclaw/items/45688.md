## Summary

Keep open: current main still has the reported Control UI config usability regression. Advanced Settings is routed through the accordion layout, that branch still omits the visible search control, and the form renderer still scopes entries to the active section before applying search; the latest release snapshot shows the same behavior.

## What This Changes

The issue asks to restore at least v1-level config usability in the v2 Control UI by making settings easy to browse, search, and filter instead of trapping users in grouped top-level sections.

## Best Possible Solution

Keep the issue open and implement a focused Control UI repair: Advanced Settings should expose search/filter controls in accordion mode and make nonempty searches operate across the included visible config sections, while preserving the existing scoped settings tabs and Quick Settings flow.

## Reproduction Assessment

Yes. Static inspection against current main is enough to reproduce the reported gap: Advanced Settings selects accordion layout, the accordion branch lacks the search input, and form filtering applies activeSection before search matching.

## Solution Assessment

Yes for a narrow repair: adding accordion-mode search/filter and making search global across included sections is the smallest maintainable fix. A full return to the v1 flat list should remain a separate maintainer product decision.
