## Summary

Keep open. Current main has useful follow-up work around Quick Settings and scoped settings tabs, but it still does not restore the central v1 behavior reported here: a single, globally searchable/filterable config surface. Advanced Config is still rendered as an accordion, search is only present in the non-accordion tabs layout, and several config areas are deliberately split into separate Settings sidebar tabs.

## Best Possible Solution

Keep this open for a focused Control UI settings reconciliation: preserve Quick Settings for common tasks, but add a full advanced config path with global search/filter across all settings slices, or add equivalent search/filter affordances to the accordion and separate Settings tabs. Add browser tests that assert the advanced path can find fields across sections.
