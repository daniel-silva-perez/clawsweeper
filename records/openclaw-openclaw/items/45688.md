## Summary

Current main still has the reported Control UI config usability gap: Advanced Settings uses the accordion layout, that layout omits the visible search control, and the form filter still narrows to the active section before applying search. The issue should stay open and is narrow enough for a focused repair PR, while the larger flat-list-versus-grouped-settings design question remains a maintainer/product choice.

## What This Changes

The report asks to restore v1-level config usability in the v2 Control UI, especially a flat searchable/filterable settings experience instead of partially grouped accordion sections.

## Best Possible Solution

Keep the current Quick Settings and scoped top-level tabs, but make Advanced Settings usable by exposing search/filter controls in accordion mode and ensuring searches operate globally across visible config sections instead of being trapped in the selected group.

## Reproduction Assessment

Yes. Static inspection against current main reproduces the user-visible gap: the Advanced Settings path selects accordion layout, that branch has no search input, and the underlying filter is active-section scoped.

## Solution Assessment

Yes for a narrow repair: adding accordion-mode search/filter and changing search to global visible-section behavior is the smallest maintainable fix. A full return to the v1 flat list should remain a separate product decision if maintainers want it.
