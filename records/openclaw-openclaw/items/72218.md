## Summary

Keep this PR open. Current main still bypasses heartbeat-token sanitation in `buildReplyPayloads` whenever `isHeartbeat` is true, so the PR's central fix is not implemented on main. The PR is narrow, source/test-only, and still matches the documented heartbeat contract that `HEARTBEAT_OK` acknowledgments are stripped and dropped while alert content remains deliverable.

## Best Possible Solution

Keep this PR open for maintainer review, or land an equivalent narrow change in `src/auto-reply/reply/agent-runner-payloads.ts`: apply `HEARTBEAT_OK` stripping to heartbeat final payloads, preserve media-bearing payloads after stripping, keep Bun socket error formatting scoped to non-heartbeat runs, and add focused `isHeartbeat: true` regression tests in `src/auto-reply/reply/agent-runner-payloads.test.ts`. Treat #72253 and #69492 as separate broader exec/system-event leakage work.
