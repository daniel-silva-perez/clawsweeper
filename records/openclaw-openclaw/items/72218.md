## Summary

Keep open. Current main still has the heartbeat payload bypass this PR targets, but the branch is not merge-ready because it strips heartbeat payloads in message mode before heartbeat-mode `ackMaxChars` suppression can classify short acknowledgments.

## What This Changes

This PR removes the heartbeat-only bypass in `buildReplyPayloads`, keeps Bun socket error expansion scoped out of heartbeat runs, and adds focused payload-builder regression tests for heartbeat token stripping.

## Best Possible Solution

Revise the branch so exact OK-only heartbeat final payloads are suppressed without erasing the token before heartbeat-mode ack classification; add focused coverage for short ack text, response prefixes, media-bearing payloads, and the Bun socket-error scope.

## Reproduction Assessment

Yes. Current main has a direct code-path reproduction through `buildReplyPayloads({ isHeartbeat: true, payloads: [{ text: "HEARTBEAT_OK" }] })`, and the PR regression follows from `HEARTBEAT_OK all good` being stripped in message mode before heartbeat `ackMaxChars` handling runs.

## Solution Assessment

No, not as written. Removing the bypass is the right narrow direction, but heartbeat payloads need heartbeat-mode semantics or an exact-OK-only suppression path so short acknowledgments do not become deliverable text.
