## Summary

Keep open. Current main still handles Telegram reactions as notification/context events rather than first-class wake or execution triggers, and the issue is already paired with open PR #47687 plus the narrower open wake-trigger bug #64752.

## What This Changes

Request Telegram reactions to become a first-class control/input surface that can wake agents, normalize standard and custom emoji, expose per-emoji semantics, and improve observability.

## Best Possible Solution

Review #47687 against #64752 and either land it or replace it with a narrower Telegram-plugin implementation that normalizes standard and custom emoji, maps configured reaction actions, wakes only authorized mapped reactions, logs ignored/rejected/executed decisions, and updates focused tests plus the public Telegram docs.
