## Summary

Keep open. Current main still treats Telegram reactions as queued context notifications rather than first-class wake/semantic triggers, and the issue is already paired with an open implementation PR plus a narrower open bug report for the wake gap.

## What This Changes

Request first-class Telegram reaction handling that can wake agents, normalize standard and custom emoji reactions, map operator semantics, and expose useful debugging signals.

## Best Possible Solution

Keep this open until maintainers review, land, split, or close #47687. The durable end state should be a Telegram-owned reaction semantics path that normalizes standard and custom emoji, preserves existing notification behavior, wakes the correct session for configured action reactions, and documents/debugs the routing decisions; #64752 can serve as the minimal wake-regression acceptance case.
