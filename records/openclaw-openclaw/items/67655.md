## Summary

Keep this PR open. Current main still gates Windows shell-wrapper allowlist denial on `cmdInvocation`, while PowerShell/pwsh are recognized shell-wrapper transports and `isCmdExeInvocation` only recognizes cmd/cmd.exe. The PR is still a plausible security hardening candidate, but it needs revision for stale tests, docs, and the now-unused `cmdInvocation` parameter before maintainer review.

## Best Possible Solution

Keep this PR open and request a focused revision: make Windows allowlist mode block when `security === "allowlist"`, `isWindows`, and `shellWrapperInvocation` are true; add the PowerShell/pwsh regression for `cmdInvocation: false`; update stale denial-message assertions and node-host docs; handle the now-unused `cmdInvocation` parameter deliberately; then run the targeted node-host exec policy and invoke-system-run tests before maintainer review.
