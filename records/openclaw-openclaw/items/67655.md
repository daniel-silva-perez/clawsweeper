## Summary

Keep open. Current main still has the Windows allowlist gap, and this PR is a focused hardening direction, but the submitted patch is not merge-ready because its denial-message change leaves an existing assertion stale and the public node-host docs would remain cmd-only after the behavior broadens.

## What This Changes

This PR changes node-host exec policy to fail closed for all Windows shell-wrapper invocations in allowlist mode, expands denial guidance to mention PowerShell, and adds a PowerShell-wrapper regression test.

## Best Possible Solution

Revise and land the fail-closed Windows shell-wrapper policy with aligned regression tests and public node-host documentation, preserving explicit approval and durable trust semantics for all recognized Windows inline shell transports.
