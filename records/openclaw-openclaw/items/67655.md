## Summary

Keep this PR open. Current main still only fails closed for Windows shell-wrapper allowlist runs when the wrapper is detected as cmd.exe, while PowerShell and pwsh are recognized shell-wrapper transports that can reach policy with shellWrapperInvocation true and cmdInvocation false. The PR is a focused security hardening candidate, but it still needs the stale test assertions and public docs updated before maintainer review.

## Best Possible Solution

Keep this PR open and request a focused revision: change the Windows allowlist guard to block when security is allowlist, isWindows is true, and shellWrapperInvocation is true; add the PowerShell/pwsh regression for cmdInvocation false; update the stale denial-message assertions and node-host docs; then run the targeted node-host policy and invoke-system-run tests before maintainer review.
