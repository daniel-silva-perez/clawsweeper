## Summary

Current main still does not implement an enforceable per-session Private/Secret/Incognito mode. The repo has partial mitigations such as global bootstrap context suppression and `/btw` side results, but the requested product guarantee still needs protocol/session state, no-persist transcript/media enforcement, memory read/write suppression, and Mission Control/channel controls.

## Best Possible Solution

Keep this issue open as the canonical Private/Secret/Incognito mode request. A complete path should define an explicit private/ephemeral mode in the Gateway protocol and `SessionEntry`, enforce no transcript/media/session-history writes, suppress workspace bootstrap and memory reads/writes, expose Mission Control and per-channel controls, and add regression tests proving private sessions neither read private workspace/memory context nor write persistent history.
