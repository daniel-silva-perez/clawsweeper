## Summary

Keep open. Current main still lacks an enforceable per-session Private/Secret/Incognito mode: existing controls can suppress some bootstrap context globally, but there is no protocol, store, transcript, history, memory, channel, or Control UI contract that prevents reads and durable writes for a private session.

## What This Changes

Request a Private Mode toggle for demos and content creation that suppresses workspace/persona/memory context, avoids touching real workspace files, and uses temporary session memory/history.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito mode tracker and design an additive private-session contract across Gateway protocol, SessionEntry/session store, transcripts/history APIs, bootstrap and memory loading/writing, tool/media persistence, channel triggers, and Control UI indicators, with regression tests proving private sessions do not read protected context or write durable history.
