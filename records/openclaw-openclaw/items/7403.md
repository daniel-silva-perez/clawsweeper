## Summary

Current main still does not implement an enforceable per-session Private/Secret/Incognito mode. There is a global `agents.defaults.contextInjection: "never"` mitigation for workspace prompt injection, plus narrow one-shot non-persistence surfaces such as `/btw`, but the requested product guarantee also needs protocol/session state, transcript/media persistence controls, memory read/write suppression, and Control UI/channel controls. The issue should remain open as the canonical request; related #12242 is closed as a duplicate, and #52478 was closed unmerged after explicitly deferring runtime filesystem/session-history enforcement.

## Best Possible Solution

Keep this issue open as the canonical product/privacy request. A complete implementation should add an explicit private/ephemeral mode through Gateway protocol and `SessionEntry`, enforce no transcript/media/session-history writes, suppress workspace bootstrap and memory reads/writes, expose Mission Control and per-channel controls, and add regression tests proving private sessions neither read private workspace/memory context nor write persistent history.
