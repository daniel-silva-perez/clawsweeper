## Summary

Keep open. Current main still lacks an enforceable per-session Private/Secret/Incognito mode: the Gateway protocol, session config, persisted session state, and Control UI row model have no private/no-persist field, while session creation still writes durable store metadata and a JSONL transcript header. Existing global bootstrap-context controls can reduce injected workspace context, but they do not provide the requested program-level no-read/no-write/no-history guarantee.

## What This Changes

The issue requests an enforceable Private Mode that suppresses workspace context, avoids persistent session history, and exposes a Mission Control toggle/indicator.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito mode tracker and design an additive private/ephemeral session contract across Gateway protocol, SessionEntry, session store/transcripts, agent bootstrap/memory, tool/media persistence, channel triggers, and Control UI, with regression tests proving private sessions neither read private workspace/memory context nor write durable history.
