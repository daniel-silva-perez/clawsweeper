## Summary

Keep open. Current main still does not provide an enforceable per-session Private/Secret/Incognito mode: there is no private/no-persist field in the Gateway create/patch protocol, session config, persisted session entry, session list row, transcript creation path, or Control UI session controls. The discussion correctly expands the request from a demo toggle into a program-level privacy guarantee, and the related validation-scaffolding PR was closed unmerged.

## What This Changes

The issue requests a Mission Control Private Mode that suppresses workspace and memory context, prevents durable session/history writes, uses temporary scratchpad memory, and shows an active UI indicator.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito mode tracker and design an additive private/ephemeral session contract across Gateway protocol, SessionEntry/session store, transcript/history APIs, bootstrap and memory loading/writing, tool/media persistence, channel triggers, and Control UI indicators, with regression tests proving private sessions neither read protected workspace/memory context nor write durable history.
