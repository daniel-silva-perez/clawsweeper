## Summary

Keep open. Current main still lacks an enforceable per-session Private/Secret/Incognito mode: the Gateway session RPC schema, session config, SessionEntry state, and Control UI row model have no private/no-persist field, while session creation still creates durable session metadata and a JSONL transcript. Existing global bootstrap controls can reduce context injection, but they do not provide the program-level no-read/no-write/no-history guarantee requested in the issue discussion.

## What This Changes

Add a Private Mode for demos and content creation that suppresses workspace and memory context, prevents persistent session history or workspace writes, and exposes Mission Control plus channel-level controls.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito mode tracker. The durable product path is an additive private/ephemeral session contract across Gateway protocol, SessionEntry, session store/transcripts, agent bootstrap/memory, media/tool persistence, channel triggers, and Control UI, with regression tests proving private sessions neither read private workspace/memory context nor write durable history.
