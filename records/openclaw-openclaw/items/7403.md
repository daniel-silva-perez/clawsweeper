## Summary

Keep open. Current main still lacks an enforceable per-session Private/Secret/Incognito mode: the Gateway protocol, session config, session entry shape, and Control UI session row model have no private/no-persist field, while session creation still writes a persistent session store entry and JSONL transcript. The discussion also supports keeping this as the canonical tracker because #12242 was closed as a duplicate and #52478 was closed unmerged after covering only partial validation scaffolding.

## What This Changes

Add a Private Mode for demos and content creation that suppresses workspace and memory context, prevents persistent session history or workspace writes, and exposes a Mission Control toggle/indicator plus channel-level controls.

## Best Possible Solution

Keep this issue open as the canonical Private/Secret/Incognito mode tracker. A complete solution should add an explicit private/ephemeral session mode to the Gateway protocol and SessionEntry, enforce no transcript/media/session-history persistence, suppress workspace bootstrap and memory reads/writes, expose Mission Control and per-channel controls, and add regression tests proving private sessions neither read private workspace/memory context nor write persistent history.
