## Summary

Keep open. Current main still lacks an enforceable per-session Private/Secret/Incognito mode: session create/patch protocol fields, persisted session entries, transcript creation, history reads, bootstrap context injection, and Control UI session rows have no private/no-persist contract. The related duplicate request reinforces the no-history/no-memory requirement, and the related validation-scaffolding PR was closed unmerged.

## What This Changes

The issue requests a Mission Control Private Mode that suppresses workspace context and memory, prevents workspace reads/writes and durable session history, uses temporary scratch memory, and shows an active UI indicator.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito mode tracker and design an additive private/ephemeral session contract across Gateway protocol, SessionEntry/session store, transcript/history APIs, bootstrap and memory loading/writing, tool/media persistence, channel triggers, and Control UI indicators, with regression tests proving private sessions neither read protected workspace/memory context nor write durable history.
