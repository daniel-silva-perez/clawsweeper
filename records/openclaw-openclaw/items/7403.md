## Summary

Keep open. Current main still lacks an enforceable Private/Secret/Incognito session mode: the Gateway protocol, session config, persisted session entry, Control UI session row, and transcript/session-store write paths do not expose a per-session no-read/no-write/no-history contract, and the existing contextInjection controls only suppress bootstrap context rather than durable persistence.

## What This Changes

Add a Private Mode toggle and per-session privacy contract that suppresses workspace/memory context and prevents durable session history or memory writes for demos, shared instances, and public content creation.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito mode tracker and design an additive private/ephemeral session contract across Gateway protocol, SessionEntry/store, transcript/history APIs, bootstrap and memory, tool/media persistence, channel triggers, and Control UI, with regression tests proving private sessions neither read private workspace/memory context nor write durable history.
