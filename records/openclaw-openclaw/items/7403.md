## Summary

Current main still does not implement an enforceable per-session Private/Secret/Incognito mode. The Gateway session protocol and session entry types have no private/no-persist bit, session creation still persists a session-store entry and JSONL transcript, history is transcript-backed, and the existing context-injection controls are global bootstrap suppression rather than a per-session privacy guarantee. Prior discussion also supports keeping this as the canonical tracker: #12242 was closed as a duplicate and #52478 was closed unmerged as a partial/superseded attempt.

## Best Possible Solution

Keep this issue open as the canonical Private/Secret/Incognito mode tracker. A complete solution should add an explicit private/ephemeral mode to the Gateway protocol and SessionEntry, enforce no transcript/media/session-history persistence, suppress workspace bootstrap and memory reads/writes, expose Mission Control and per-channel controls, and add regression tests proving private sessions neither read private workspace/memory context nor write persistent history.

## Risks / Open Questions

- This is a privacy/security-sensitive guarantee; a prompt-only or UI-only toggle could create false confidence while transcripts, media, memory, or workspace files still persist.
- The implementation spans Gateway protocol, session store/history, transcript/media persistence, memory/bootstrap context, channel triggers, and Control UI, so it needs maintainer design before a narrow autonomous PR is safe.
- Existing agents.defaults.contextInjection:"never" can reduce bootstrap leakage for some runs, but it does not prevent transcript writes, history reads, tool-driven workspace reads/writes, or channel/session persistence.
