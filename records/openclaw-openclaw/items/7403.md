## Summary

Keep open. Current main still lacks an enforceable Private/Secret/Incognito session contract: the Gateway session protocol, session config, persisted session entry, transcript write path, bootstrap/startup-memory flow, and Control UI session model do not carry or enforce a private/no-persist bit. The discussion and related duplicate make this a program-level privacy guarantee, not a UI-only enhancement.

## What This Changes

The issue requests a Private Mode toggle that suppresses workspace context, avoids durable memory/session writes, uses temporary scratch state, and visibly marks private sessions in Mission Control.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito mode tracker and design an additive private/ephemeral session contract across Gateway protocol, SessionEntry/store, transcript/history APIs, bootstrap/startup memory, tool/media persistence, channel triggers, and Control UI, with regression tests proving private sessions neither read private workspace/memory context nor write durable history.

## Reproduction Assessment

Not applicable as a feature/privacy request rather than a runtime bug. The absence check is high-confidence: current schemas, persisted session types, transcript creation, bootstrap/startup-memory code, docs, and UI session rows all lack the requested private/no-persist contract.

## Solution Assessment

No current patch is the best complete solution. The requested direction is valid, but the maintainable fix needs a maintainer-approved cross-surface privacy contract rather than a UI-only toggle or the closed validation-only scaffolding.
