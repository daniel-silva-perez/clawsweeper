## Summary

Keep open. Current main still lacks the requested enforceable per-session Private/Secret/Incognito mode: the Gateway protocol, session config, persisted session entry, transcript creation path, and Control UI session row have no private/no-persist contract. Existing global bootstrap-context controls can suppress injected workspace files, but they do not provide the program-level no-read/no-write/no-history guarantee requested in the issue discussion.

## What This Changes

Request adds a Mission Control Private Mode/session privacy toggle that suppresses workspace context, uses temporary memory, prevents durable session history, and leaves real workspace files untouched.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito mode tracker and design an additive private/ephemeral session contract across Gateway protocol, SessionEntry/session store, JSONL transcript/history APIs, bootstrap and memory loading/writing, tool/media persistence, channel triggers, and Control UI indicators, with regression tests proving private sessions neither read private workspace/memory context nor write durable history.
