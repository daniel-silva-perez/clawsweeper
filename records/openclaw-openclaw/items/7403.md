## Summary

Current main still lacks an enforceable per-session Private/Secret/Incognito mode. There is no protocol/config/session-entry bit for private or no-persist behavior, session creation still creates JSONL transcript files, history remains transcript-backed, and the existing `agents.defaults.contextInjection: "never"` only suppresses bootstrap/context-file injection globally rather than providing session-level persistence or workspace-access enforcement.

## Best Possible Solution

Keep this issue open as the canonical Private/Secret/Incognito mode tracker. A complete implementation should add an explicit private/ephemeral mode to the Gateway protocol and `SessionEntry`, enforce no transcript/media/session-history persistence, suppress workspace bootstrap and memory reads/writes, expose Mission Control and per-channel controls, and add regression tests proving private sessions neither read private workspace/memory context nor write persistent history.
