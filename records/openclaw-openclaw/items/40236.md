## Summary

Keep open. Current main still lacks a persisted job-level cron lifecycle contract for timeout, retry/cleanup policy, and per-job start/complete/failure hooks; the existing implementation covers adjacent timeout, one-shot deletion, failure alerts, retention, and plugin observation only. The related open PR #45465 overlaps the hook portion but is unmerged and does not cover the full timeout plus cleanup request.

## What This Changes

Request adds cron job lifecycle configuration for job-level timeout, automatic cleanup policy, retries, and start/complete/failure hooks.

## Best Possible Solution

Design one additive cron lifecycle contract across Gateway protocol schemas, stored job normalization, scheduler timeout semantics, cleanup policy, hook execution boundaries, CLI/UI/docs, and focused regression tests while preserving current payload timeout, one-shot deletion, failure alerts, retention, and plugin `cron_changed` observation behavior.

## Reproduction Assessment

Not applicable as a feature/API request. The absence is directly verifiable on current main by inspecting the cron protocol schema, stored job types, scheduler timeout/cleanup logic, CLI flags, docs, and related open PR context.

## Solution Assessment

No. The API sketch is directionally useful, but the hook portion needs maintainer API/security design, and #45465 only covers part of the requested lifecycle surface while timeout and cleanup remain separate scheduler concerns.
