## Summary

Keep open. Current main has adjacent cron safeguards, including payload-level agent-turn timeouts, retry/backoff, one-shot delete-after-success, failure alerts, run-log/session pruning, and an observe-only plugin cron event hook, but it still does not implement the requested persisted job-level lifecycle contract for timeout, cleanup-by-policy, or per-job start/complete/failure hooks. The related open PR #45465 may cover part of the hook story, but it is unmerged and does not cover the timeout and cleanup portions.

## What This Changes

The issue requests cron job-level lifecycle controls: per-job timeout, retry/cleanup policy, and start/complete/failure hooks.

## Best Possible Solution

Keep this as a cron product/API design item. A complete solution should define an additive cron lifecycle contract across Gateway protocol schema, stored job normalization/types, scheduler timeout policy, cleanup semantics, hook delivery, CLI/UI/docs, and regression tests while preserving existing payload.timeoutSeconds, deleteAfterRun, failureAlert, run-log, and session-retention behavior.
