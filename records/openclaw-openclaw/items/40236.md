## Summary

Keep open. Current main has adjacent cron safeguards, payload-level timeout support, run/session cleanup, failure alerts, and plugin-level cron_changed observability, but it still lacks the requested persisted job-level lifecycle contract for timeoutMs, automatic cleanup by run/time/status, and per-job start/complete/failure hooks. The related open PR #45465 may cover part of the hook story, but it is unmerged and does not address the timeout and cleanup portions.

## What This Changes

Add cron job-level lifecycle configuration for timeouts, retry/cleanup policy, and start/complete/failure hooks.

## Best Possible Solution

Keep this as a cron product/API design item. A complete solution should add an explicit additive lifecycle contract across Gateway protocol schemas, stored job normalization/types, scheduler timeout and cleanup behavior, CLI/UI/docs, tests, and security boundaries while preserving existing payload.timeoutSeconds, deleteAfterRun, retry/backoff, failureAlert, session retention, run-log pruning, and plugin cron_changed behavior.
