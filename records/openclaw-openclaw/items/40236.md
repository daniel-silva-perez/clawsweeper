## Summary

Keep open. Current main has adjacent cron safeguards, including payload-level agent-turn timeouts, one-shot delete-after-success, retry/backoff, failure alerts, run logs, session pruning, and observe-only plugin cron events, but it still does not provide the requested persisted job-level lifecycle contract for timeout, automatic cleanup policy, or per-job start/complete/failure hooks. The related open PR #45465 may cover part of the hook story, but it is unmerged and does not cover the timeout and cleanup portions.

## What This Changes

The issue requests cron job-level lifecycle configuration for per-job timeout, retries, automatic cleanup, and start/complete/failure hooks.

## Best Possible Solution

Keep this as a cron product/API design item. A complete solution should define an additive lifecycle contract across Gateway protocol schema, stored job normalization/types, scheduler timeout policy, cleanup semantics, hook delivery, CLI/UI/docs, and regression tests while preserving existing payload.timeoutSeconds, deleteAfterRun, retry/backoff, failureAlert, run-log, and session-retention behavior.

Status: needs_attention

Summary: The issue is security-sensitive because persisted per-job lifecycle hooks could introduce scheduler-time script execution.
