## Summary

Keep open. Current main still lacks the requested persisted cron job-level lifecycle contract: it has payload-level agent-turn timeouts, one-shot delete-after-success, retry/backoff, failure alerts, run-log/session pruning, and an observe-only plugin cron_changed hook, but not lifecycle.timeoutMs, maxRetries, autoCleanup by run/time/status, or per-job onStart/onComplete/onFailure hooks. The open related PR #45465 may cover part of the hook story, but it is unmerged and does not cover the timeout and cleanup portions.

## What This Changes

The issue requests cron job-level lifecycle configuration for timeouts, automatic cleanup policies, retries, and start/complete/failure hooks.

## Best Possible Solution

Keep this as a cron product/API design item. A complete solution should add an additive lifecycle contract across Gateway protocol schema, cron job normalization/store types, scheduler timeout policy, cleanup semantics, hook delivery, CLI/UI/docs, and regression tests, while preserving existing payload.timeoutSeconds, deleteAfterRun, failureAlert, run-log, and session-retention behavior.
