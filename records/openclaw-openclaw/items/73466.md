## Summary

Keep this PR open. Current main still does not implement the Google Live browser-audio interruption fix: it resets the playback cursor on `serverContent.interrupted`, but it does not retain or stop scheduled `AudioBufferSourceNode`s. The same-author linked bug #73460 is still open, so the best path is maintainer review and landing this PR or an equivalent patch.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for #73460. The best product/code path is to add browser-direct Google Live output-source tracking, stop scheduled sources on `serverContent.interrupted` and transport shutdown, keep behavior aligned with `GatewayRelayRealtimeTalkTransport.stopOutput()`, and land the focused regression test in `ui/src/ui/realtime-talk-google-live.test.ts`.
