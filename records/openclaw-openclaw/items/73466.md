## Summary

Keep this PR open. Current main at c9ead1b928b890b7bd7c1f1476a10dd277f8f8b9 still does not implement the intended Google Live browser-audio interruption fix: the browser-direct transport only resets its playback cursor and does not retain or stop scheduled AudioBufferSourceNodes. The PR is also paired with open same-author bug #73460, so the best maintainer path is to review and land this implementation or an equivalent patch rather than close it during cleanup.

## Best Possible Solution

Keep this PR open as the implementation candidate for #73460. The best path is to review the small browser audio-source lifecycle change, keep it aligned with GatewayRelayRealtimeTalkTransport's source tracking and stopOutput behavior, include the focused regression test in ui/src/ui/realtime-talk-google-live.test.ts, and close the paired bug only after the PR or an equivalent fix lands.
