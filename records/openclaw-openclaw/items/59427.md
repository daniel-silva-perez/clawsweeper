## Summary

Keep this issue open. Current main has useful partial mitigations for prompt pressure, including `agents.defaults.contextInjection` with `continuation-skip`/`never`, lightweight bootstrap context for cron/subagent runs, on-demand skill loading, and an experimental lean local-model tool profile. But ordinary chat runs still default to rebuilding and sending the OpenClaw system prompt plus tool schemas and conversation context on each model call, so the reported baseline-token problem is not fully implemented as a lazy or lightweight default mode for minimal replies.

## Best Possible Solution

Keep this open and turn it into a focused core prompt-budget issue: define a supported lightweight ordinary-chat profile that can reduce workspace bootstrap, tool schemas, and other high-volume sections without breaking agent safety or tool availability; document the tradeoffs; and add run-path tests proving minimal turns use the lighter profile while tool-heavy turns still get the needed context.
