## Summary

Keep open. Current main has useful adjacent mitigations for repeated bootstrap context, prompt-cache stability, heartbeat/cron light context, and a full opt-out, but it does not implement the requested default lightweight behavior for ordinary short replies.

## What This Changes

The issue asks OpenClaw to reduce the baseline per-message context load for minimal replies by lazy-loading context/tool/memory material and offering a lightweight-vs-full mode.

## Best Possible Solution

Define a maintainer-approved context-cost policy that keeps full context for first or unsafe turns, reduces repeated workspace/memory/tool-heavy context on safe continuation turns, exposes an explicit full mode, and documents verification through /context and usage reporting.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the structural condition: /context detail plus current source show the rebuilt system prompt, bootstrap files, and tool schemas that drive baseline context cost. The exact 38K/43K values vary by setup and were not live-run during this read-only review.

## Solution Assessment

Is this the best way to solve the issue? Unclear. The existing continuation-skip option is a good narrow mitigation for repeated bootstrap context, but it is not the complete requested small-reply/default-lightweight behavior because normal runs still use full prompt/tool context unless product policy changes.
