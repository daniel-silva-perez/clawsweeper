## Summary

Keep open. Current main has useful opt-in mitigations for repeated bootstrap context and full bootstrap opt-out, but the default short-reply path still builds the normal OpenClaw prompt/tool context rather than lazy-loading a lightweight baseline automatically.

## What This Changes

The issue asks OpenClaw to reduce the baseline per-message context cost for trivial replies by lazy-loading context/tool/memory material and offering lightweight versus full context behavior.

## Best Possible Solution

Define a maintainer-approved context-cost policy that keeps full context for first or unsafe turns, reduces repeated workspace and tool-heavy context on safe continuation turns, preserves an explicit full mode, and documents verification through /context and usage reporting.

## Reproduction Assessment

Yes for the structural condition: current docs and source show the rebuilt system prompt, default full bootstrap injection, and tool schemas that drive baseline context cost. The exact token counts vary by installation and were not live-run during this read-only review.

## Solution Assessment

Unclear. The existing continuation-skip and never modes are good narrow mitigations, but they are not the complete requested default lightweight behavior for ordinary short replies because normal runs still use full prompt and tool context unless configured otherwise.
