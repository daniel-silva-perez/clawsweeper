## Summary

Keep this issue open. Current main has several relevant mitigations, including exact per-message stream dedupe, commentary-phase suppression, WebChat final-event reload guards, chat.send idempotency, and tool/final payload duplicate suppression, but those are narrower than the reported WebChat/Omlx-Qwen repeated visible assistant responses across tool and approval steps. I did not find source, tests, changelog, or discussion evidence proving this specific regression is fixed.

## Best Possible Solution

Keep this issue open and ask the reporter to retest current main or the next release that includes the WebChat final-reconciliation fix. If it still reproduces, collect a trajectory export, raw transcript, gateway `chat` events, provider/model config, and whether approvals or tool calls occurred. Then fix the confirmed owner surface: UI/history rendering for duplicated final bubbles, gateway/subscriber idempotency for duplicate event delivery, or provider/prompt handling for Omlx/Qwen semantic repetition, with a focused regression test around the failing path.
