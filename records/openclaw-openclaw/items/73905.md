## Summary

Keep this issue open. Current main still lacks the requested configurable `agents.defaults.subagents.notifyPolicy` surface: cron task records are explicitly `silent`, subagent task records still omit `notifyPolicy` and fall through to the task registry default of `done_only`, and the config schemas/docs expose no `silent` or `inherit_from_parent` default for subagent tasks. Current main does include safeguards that avoid some duplicate cron/subagent delivery, but that is narrower than the requested configurable policy.

## Best Possible Solution

Add a backwards-compatible subagent task notify default under `agents.defaults.subagents.notifyPolicy`, preserving current `done_only` behavior when unset. Support `silent`, `done_only`, and `inherit_from_parent`; for inheritance, map cron requester sessions to `silent` and fall back to the existing default for ordinary sessions unless a real parent task policy is available. Update config types/schema/generated docs, public docs, and targeted tests for cron-spawned subagents.
