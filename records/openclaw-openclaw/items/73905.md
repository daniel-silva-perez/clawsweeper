## Summary

Keep this open. Current main still lacks a configurable cron-aware subagent notification policy, and the related open #64081 overlaps but does not fully settle the requested parent-inheritance behavior or the subagent announce path.

## What This Changes

The issue requests a configurable default notification policy for subagents spawned from cron jobs, including an option to inherit the parent cron job's silent policy.

## Best Possible Solution

Define one backwards-compatible notification policy resolver that preserves current defaults when unset, supports explicit silent behavior for cron-spawned subagents, and documents whether parent inheritance applies to task records, subagent completion announcements, or both.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: a cron run creates a silent cron task, `sessions_spawn` defaults to expecting a completion message, the subagent task is registered without `notifyPolicy`, and the registry resolves that pending task to `done_only`.

## Solution Assessment

Unclear. A subagent-level default would address this cron use case, but a broader task notification policy from #64081 may be the cleaner public API if it explicitly covers cron parent inheritance and subagent announce delivery.
