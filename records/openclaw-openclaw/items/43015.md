## Summary

Keep open. Current main still exposes shared poll creation fields in the unified message tool schema and still rejects ordinary `action: "send"` calls when non-empty, true, or non-zero poll-looking params are present. The Discord `components` schema exposure has been reduced for the generic tool path, and zero-valued poll defaults are mitigated, but the central auto-populated poll/component-noise problem is not fully resolved. Open PR #46455 also directly references this issue with closing syntax, so the issue should stay open until that implementation candidate is reviewed, landed, or explicitly superseded.

## Best Possible Solution

Keep this issue open. Maintainers should review, rebase, land, or explicitly supersede PR #46455 or an equivalent patch that makes poll creation fields action-aware, strips or ignores auto-populated poll/component noise from ordinary `action: "send"` calls, preserves strict validation for explicit polls and real Discord components/modals, and adds focused regression coverage for Telegram/Feishu/Discord send-with-file plus Discord empty-modal skeleton behavior.
