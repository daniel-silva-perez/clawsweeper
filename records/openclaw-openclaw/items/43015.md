## Summary

Keep open. Current main still leaves the core poll/send coupling reported here: the shared message tool schema includes poll creation fields, and `action: "send"` still rejects when non-empty, true, or non-zero poll-looking params are present. The zero-default mitigation is present and Discord component schema exposure is partly reduced, but open PR #46455 still directly targets the remaining behavior with closing syntax, so the right path is maintainer review, landing, or explicit supersession of that PR.

## Best Possible Solution

Keep this issue open. Maintainers should review, rebase, land, or explicitly supersede PR #46455 or an equivalent patch that makes poll creation fields action-aware, strips or ignores auto-populated poll/component noise from ordinary `action: "send"` calls, preserves explicit poll and Discord component validation, and adds focused regression coverage for Telegram/Feishu/Discord send-with-file plus Discord empty-modal skeleton behavior.
