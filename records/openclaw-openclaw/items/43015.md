## Summary

Keep open. Current main still has the reported core coupling: the unified message tool schema exposes poll creation fields alongside `action: "send"`, and the send path still rejects when those fields contain non-empty, true, or non-zero values. The zero-default mitigation and generic Discord component schema reduction are present, but they do not fully solve the auto-populated poll/component-noise problem, and open PR #46455 still directly targets this issue with closing syntax.

## Best Possible Solution

Keep this issue open. Maintainers should review, rebase, land, or explicitly supersede PR #46455 or an equivalent patch that makes poll creation fields action-aware, strips or ignores auto-populated poll/component noise from ordinary `action: "send"` calls, preserves explicit poll and Discord component validation, and adds focused regression coverage for Telegram/Feishu/Discord send-with-file plus Discord empty-modal skeleton behavior.
