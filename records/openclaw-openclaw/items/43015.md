## Summary

Keep open. Current main has mitigated zero-valued poll defaults and no longer exposes Discord-native `components` through Discord message-tool discovery, but the shared message tool still includes poll creation fields and the send path still rejects `action: "send"` when non-empty, true, or non-zero poll-looking params are present. Open PR #46455 is an active implementation candidate and uses closing syntax for this issue, so the issue should stay open until that PR or an equivalent fix is reviewed, landed, or explicitly superseded.

## Best Possible Solution

Keep this issue open. Maintainers should review, rebase, land, or explicitly supersede PR #46455 or an equivalent patch that makes poll creation fields action-aware, strips or ignores auto-populated poll/component noise from ordinary `action: "send"` calls, preserves strict validation for explicit polls and real Discord components/modals, and adds focused regression coverage for Telegram/Feishu/Discord send-with-file plus Discord empty-modal skeleton behavior.
