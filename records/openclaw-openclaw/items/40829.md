## Summary

Keep this PR open. Current main still converts non-alpha under-cap PNGs through the JPEG optimizer and only renames JPEG output filenames for HEIC sources, so the PR's intended behavior is not implemented. The patch is also stale against the shared media loader location and needs the PNG validation concern addressed before merge.

## Best Possible Solution

Rebase the useful behavior onto `src/media/web-media.ts` and current tests. Preserve under-cap PNG bytes only after validating they are real decodable PNG images, normalize `fileName` whenever the actual optimized output is JPEG, and add regression coverage in the shared loader plus the WhatsApp/plugin-facing media path.
