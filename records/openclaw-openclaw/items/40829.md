## Summary

Keep this PR open. Current main still does not preserve under-cap non-alpha PNGs in the shared web media path, and JPEG filename normalization still only applies to HEIC sources. The PR is stale against the moved `src/media/web-media.ts` layout and should address the existing PNG validation review concern before merge, but the intended behavior is still useful and unresolved.

## Best Possible Solution

Keep this PR open and ask for a rebase onto `src/media/web-media.ts` plus current media tests. The best fix is to preserve under-cap PNGs only after validating the PNG payload, normalize every actual JPEG output filename to `.jpg`, and add focused regression coverage for under-cap PNG preservation and PNG-to-JPEG filename normalization.
