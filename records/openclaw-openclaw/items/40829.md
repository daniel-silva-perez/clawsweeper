## Summary

Keep this PR open. Current main still sends under-cap non-alpha PNGs through JPEG optimization, still only renames JPEG output filenames for HEIC sources, and the PR now needs a rebase because it targets deleted `src/web/*` media files. The intended fix is still useful, but it should be applied to the current shared media loader with a validation-safe PNG preservation path.

## Best Possible Solution

Keep this PR open and ask for a rebase onto `src/media/web-media.ts` plus current media tests. The best fix is to validate/decode PNG bytes first, preserve already-under-cap verified PNGs without JPEG re-encoding, normalize filenames for every actual JPEG output, and add regression coverage for under-cap PNG preservation plus PNG-to-JPEG filename normalization.
