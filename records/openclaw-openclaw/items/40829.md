## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: under-cap non-alpha PNGs still flow through JPEG optimization, and JPEG filename normalization is still limited to HEIC sources. The PR is also stale against the current shared media path and needs a rebase plus payload validation before it can land.

## Best Possible Solution

Rebase the useful parts of this PR onto `src/media/web-media.ts` and current media tests. The best fix is to validate/decode PNG bytes first, preserve verified under-cap PNGs without JPEG re-encoding, normalize filenames whenever the actual optimized output is JPEG, and add regression coverage for under-cap PNG preservation plus PNG-to-JPEG filename normalization through the shared loader and WhatsApp-facing path.
