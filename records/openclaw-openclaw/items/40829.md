## Summary

Keep this PR open. Current main still converts non-alpha under-cap PNGs through the JPEG optimizer, still only renames JPEG output filenames for HEIC inputs, and the PR patch targets the old `src/web/media.ts` location rather than the active shared media loader.

## Best Possible Solution

Rebase the useful behavior onto `src/media/web-media.ts`: validate that under-cap PNG bytes decode as PNG before preserving them, normalize `fileName` whenever the actual optimized output is JPEG, and add focused regression coverage in the shared loader plus the WhatsApp/plugin-facing media wrapper tests.
