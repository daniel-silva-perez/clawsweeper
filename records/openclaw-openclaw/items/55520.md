## Summary

Keep open. Current main has several partial mitigations for reasoning-only turns and provider-specific reasoning controls, but it still does not implement the reported fallback/reclassification behavior for OpenAI-compatible streams that put visible answer text in bare `reasoning_content`/`reasoning`/`reasoning_text` fields, nor can it safely reclassify internal-looking `delta.content` back into thinking without provider evidence.

## Best Possible Solution

Keep open and implement a provider-aware stream-classification fix with regression coverage for the two mirrored samples: empty `content` plus answer-like `reasoning_content`, and provider output where visible `content` is known to be internal reasoning. The safest direction is opt-in provider metadata or typed upstream fields, not a blanket fallback that exposes all reasoning fields.

## Evidence

There is no fallback that converts a reasoning-only final answer into text when `content` is empty.

## Risks / Open Questions

- Closing now could hide a still-current loss path for OpenAI-compatible providers that return user-visible answer text only through bare reasoning fields.
- A generic fallback that always exposes `reasoning_content` as text could leak private chain-of-thought for providers where that field is truly internal reasoning.
