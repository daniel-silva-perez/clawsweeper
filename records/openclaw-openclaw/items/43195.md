## Summary

Keep open. Current main already globalizes the unhandled-rejection handler registry, but it still does not make installUnhandledRejectionHandler() idempotent: every call attaches another process.on("unhandledRejection", ...) listener. The legacy direct entrypoint can call the installer before delegating to runCli(), which calls it again, so the PR’s central fix remains useful.

## Best Possible Solution

Keep the PR open, or replace it with an equivalent maintainer patch, by adding shared installer state and an early return to installUnhandledRejectionHandler() in src/infra/unhandled-rejections.ts. Preserve the current global handler Sets and uncaughtException registry, then add a regression test for listener-count stability before landing.
