## Summary

Keep this PR open. Current main already shares the unhandled-rejection handler registry through globalThis, but it still does not make installUnhandledRejectionHandler() idempotent, so the PR’s central fix remains useful and needs a rebase plus focused regression coverage.

## Best Possible Solution

Keep the PR open, or replace it with an equivalent maintainer patch, by adding shared installer state to installUnhandledRejectionHandler() in src/infra/unhandled-rejections.ts. Preserve the current global handler Sets and uncaughtException registry, make repeated installer calls a no-op, and add regression coverage for listener-count stability.
