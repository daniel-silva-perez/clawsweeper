## Summary

Keep this PR open. Current main already shares the unhandled-rejection and uncaught-exception handler registries through globalThis, but `installUnhandledRejectionHandler()` still unconditionally attaches a new `process.on("unhandledRejection", ...)` listener on every call. The PR's central idempotence fix remains useful after rebase, with focused regression coverage needed before landing.

## Best Possible Solution

Keep this PR open, or replace it with an equivalent maintainer patch, by adding shared installer state and an early return to `installUnhandledRejectionHandler()` in `src/infra/unhandled-rejections.ts`. Preserve the current global handler Sets and uncaught-exception registry, set the installed flag before registering the listener, and add listener-count regression coverage before landing.
