## Summary

Keep this PR open. Current main still does not handle Discord `MessageType.ChannelNameChange`, `ChannelIconChange`, or `ChannelFollowAdd` in `resolveDiscordSystemEvent`, and the Discord preflight path only drops these messages when that resolver returns system text. The PR is a focused fix for the remaining #24108 behavior, not implemented on main.

## Best Possible Solution

Keep the PR open and refresh it against current main. The best path is to land either this targeted three-case Discord mapper fix or the broader same-author #49286 after adding focused regression coverage and fixing the changelog attribution.
