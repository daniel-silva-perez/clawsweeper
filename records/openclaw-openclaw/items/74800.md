## Summary

Keep open: current main still matches the reported failure path. The shared plugin command handler reads `result.continueAgent` immediately after executing a plugin command, while the executor returns the handler value unchanged, so a local/JS plugin that returns `undefined` can still throw before Telegram falls back to the generic error reply.

## What This Changes

Guard Telegram/plugin command dispatch so a plugin command handler that returns `undefined` does not crash and surface Telegram’s generic failure reply.

## Best Possible Solution

Normalize plugin command handler outputs at the shared command executor boundary so nullish or non-object runtime returns become an empty no-op reply, preserve existing `continueAgent` behavior, and add regression coverage for both text command dispatch and Telegram native plugin command dispatch.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/74800. Current main still dereferences plugin command handler results without a runtime guard: `src/auto-reply/reply/commands-plugin.ts` reads `result.continueAgent` after `executePluginCommand()`, and `src/plugins/commands.ts` returns `command.handler(ctx)` unchanged. Implement a shared normalization so plugin handlers that return `undefined`, `null`, or another non-object runtime value do not throw. For a matched no-op plugin command, the text command path should return `shouldContinue: false` with no reply. Preserve existing `continueAgent: true` behavior and do not change command authorization, command conflict logging, plugin discovery, or unrelated Telegram registration behavior. Also audit `extensions/telegram/src/bot-native-commands.ts`, which reads `result.text` from native plugin command execution, and cover it either by the shared normalization or a local guard. Add focused regression tests, especially in `src/auto-reply/reply/commands-plugin.test.ts`, for an undefined plugin command result; add Telegram native command coverage if the native path is not fully covered by the shared executor test. Changelog: add a user-facing fix entry only if repository policy expects one for latest-release Telegram/plugin command regressions, crediting the reporter as `@githubdulong`.
