## Summary

Keep open: current main still has the reported undefined-result failure path, and an open implementation PR already explicitly targets this issue. The issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

Telegram plugin command handlers that return undefined should be treated as no-op replies instead of crashing command dispatch and surfacing the generic Telegram error.

## Best Possible Solution

Review and land #74989 or an equivalent narrow fix that normalizes nullish or non-object plugin command returns at `executePluginCommand`, preserves `continueAgent`, defensively handles existing call sites, and adds regression tests for generic text commands plus Telegram native plugin commands.

## Reproduction Assessment

Yes. A synthetic registered plugin command whose handler resolves to undefined follows the current source path from `executePluginCommand()` to `result.continueAgent`, which reproduces the TypeError without needing the reporter's local plugin.

## Solution Assessment

Yes. Central normalization at the plugin command executor boundary is the narrowest maintainable fix because both generic command handling and Telegram native command handling consume that result; caller guards are useful defense-in-depth.
