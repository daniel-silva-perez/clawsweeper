## Summary

Keep open: current main still has the unguarded undefined plugin-command result path, and the linked open PR #74989 explicitly targets this issue. The issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

Telegram/plugin command handlers that return undefined should not crash Telegram dispatch or surface the generic failure message.

## Best Possible Solution

Review and land #74989 or an equivalent narrow patch that normalizes nullish or non-object plugin command returns at `executePluginCommand`, keeps defensive handling in the generic and Telegram native call sites, preserves `continueAgent`, and adds regression coverage for undefined plugin command returns.

## Reproduction Assessment

Yes. A synthetic plugin command handler that returns undefined follows the current source path from `executePluginCommand()` to `result.continueAgent`, reproducing the TypeError without needing the reporter's local plugin.

## Solution Assessment

Yes. Central normalization at the plugin command executor boundary, plus defensive guards at the two current call sites, is the narrowest maintainable fix because both generic text commands and Telegram native plugin commands consume that result.
