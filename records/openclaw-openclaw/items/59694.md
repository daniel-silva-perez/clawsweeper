## Summary

Keep this PR open. Current main still has `resolveConfigPath` traversing raw dot-separated path segments without `isBlockedObjectKey`, while nearby config path helpers already reject `__proto__`, `prototype`, and `constructor`. The PR remains a focused security hardening change, not implemented or superseded on main.

## Best Possible Solution

Keep this PR open and review it as a small security hardening patch. The best path is to add or request regression tests in `src/shared/config-eval.test.ts` for `__proto__`, `prototype`, and `constructor` path segments returning `undefined`, then merge the guard if CI passes.
