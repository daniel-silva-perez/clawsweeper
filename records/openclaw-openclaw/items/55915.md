## Summary

Keep open. Current main still does not implement the requested cloud relay path: mobile apps connect directly to a reachable Gateway WebSocket, QR remote mode still requires an existing reachable remote URL or Tailscale Serve/Funnel, Gateway config/schema have no `cloud` OAuth/API/heartbeat/relay block, and the root CLI has no `login`, `logout`, `cloud status`, or `invite` command for this flow. The same author’s related invite-code issue #55914 is also still open, so this should stay available for maintainer design review rather than being auto-closed.

## Best Possible Solution

Keep this open for maintainer design review. Decide whether cloud mobile reachability is first-party Gateway/mobile behavior or needs a new explicit plugin/API seam, then implement the chosen path with config/schema/docs, OAuth credential handling, gateway registration/heartbeat, relay lifecycle, mobile list/connect UX, and security tests that preserve device approval. Coordinate with #55914 so invite-code discovery and cloud relay config stay coherent.
