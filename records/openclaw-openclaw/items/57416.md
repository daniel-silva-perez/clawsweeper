## Summary

Keep this PR open. Current main does not implement the PR’s intended behavior: workspace `.env` still blocks `OPENCLAW_CONFIG_PATH`, tests assert that it remains unset, and docs/changelog describe a later fail-closed policy for all `OPENCLAW_*` keys in untrusted workspace `.env` files. The PR is a focused attempt to fix #57408, but it now conflicts with current main’s security policy rather than being obsolete because the behavior shipped elsewhere.

## Best Possible Solution

Do not auto-close this PR as implemented. Maintainers should decide whether project-local config selection via workspace `.env` is still desired after the stricter workspace dotenv policy. If yes, implement a narrow, documented safe mechanism with tests proving no global `.env` or state-dir redirect; if no, close this PR and the linked bug with the security rationale and recommended alternatives such as process env or trusted `~/.openclaw/.env`.
