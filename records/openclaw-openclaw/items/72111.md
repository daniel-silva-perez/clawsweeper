## Summary

Keep open. Current main still resolves mixed `command` + `url` MCP server definitions as stdio before checking `transport: "streamable-http"`, so the reported timeout path remains plausible. There is already an open linked fix PR, #72515, that uses closing syntax for this issue, so the best maintainer action is to review/land or close that PR rather than auto-close this issue now.

## Best Possible Solution

Keep this issue open and review #72515. If maintainers accept early rejection as the policy, land that PR or an equivalent fix in `src/agents/mcp-transport-config.ts` with regression tests. If managed child-process HTTP servers are desired instead, implement that explicitly with readiness probing, HTTP connection setup, lifecycle cleanup, and docs/tests; do not leave the silent stdio fallback timeout as the user-visible behavior.
