## Summary

Current main still promotes valid deep-phase candidates to `MEMORY.md` by carrying and rehydrating snippet text, then formatting that snippet directly into the promotion block. Recent guardrails reject contaminated dreaming/transport artifacts and Dream Diary narrative generation can summarize into `DREAMS.md`, but neither path distills otherwise valid daily-memory snippets before durable promotion. The related open review-queue issue is adjacent, not a shipped replacement for this broader distillation/product-intent question.

## Best Possible Solution

Keep this issue open for memory-core maintainer triage. Decide whether direct raw-snippet promotion to `MEMORY.md` is intended; if yes, document that behavior clearly. If not, implement a distilled or review-first promotion path in the memory-core ranking/apply flow, preserve source provenance, and add focused tests for valid raw daily snippets versus the final `MEMORY.md` output. Coordinate with #72887 if maintainers prefer review queue semantics over immediate distilled writes.
