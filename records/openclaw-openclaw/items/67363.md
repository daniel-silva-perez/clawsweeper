## Summary

Keep this open. Current main now filters several known dreaming-artifact and metadata contamination shapes, but it still carries ordinary selected daily-memory snippets through ranking, rehydration, and append as raw text. The docs/config describe direct `MEMORY.md` promotion and do not expose a distilled-promotion mode; related #67580/#68774/#72887/#74541 cover narrower leakage, review-queue, or Dream Diary timeout work rather than settling this valid-snippet distillation contract.

## What This Changes

The issue asks memory-core dreaming deep promotion to stop writing raw daily-log snippets verbatim into `MEMORY.md`, or to clarify that raw promotion is intentional and provide a distilled-promotion option.

## Best Possible Solution

Memory-core should get an explicit durable-promotion contract: either document raw snippet append as intentional with provenance/review expectations, or add a reviewed/distilled promotion path that preserves source provenance, has deterministic fallback behavior, and updates schema, docs, and focused tests around raw versus distilled `MEMORY.md` output.
