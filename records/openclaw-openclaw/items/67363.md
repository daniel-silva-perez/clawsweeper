## Summary

Current main still promotes eligible short-term memory by carrying the original snippet through ranking, rehydrating the live source text, and writing that snippet directly into `MEMORY.md`. The newer Dream Diary fallback/retry work and metadata-contamination guards are adjacent, but they do not distill valid daily-log snippets or add a distilled-promotion/write-mode config. The KinthAI comment usefully describes an external distillation pipeline, but there is no linked OpenClaw implementation or plugin that resolves the repo issue.

## Best Possible Solution

Keep this issue open for memory-core maintainer triage. Decide whether direct raw-snippet promotion is intended; if it is, document that contract clearly. If distilled promotion is desired, implement it in the memory-core ranking/apply flow before the `MEMORY.md` write, preserve source provenance, add focused tests for raw and distilled outputs, and coordinate with #72887 if maintainers prefer review-queue-first behavior.
