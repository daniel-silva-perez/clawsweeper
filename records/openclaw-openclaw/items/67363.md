## Summary

Keep this open. Current main still carries selected short-term candidates through ranking and rehydration as `candidate.snippet` and appends that text directly to `MEMORY.md`; the current docs/config explain thresholds and direct promotion but do not define or expose a distilled-promotion mode. Related #67580/#68774 handle narrower dreaming-artifact leakage, #72887 tracks review-queue behavior, and #74541 is about Dream Diary timeouts, so none supersedes this broader raw-valid-snippet distillation question.

## What This Changes

The issue asks maintainers to decide or implement whether memory-core deep dreaming should distill daily-memory snippets before writing promoted entries to `MEMORY.md`.

## Best Possible Solution

Keep this open for memory-core maintainer triage. The end state should be an explicit durable-promotion contract: either document raw provenance-preserving appends as intentional, or add a reviewed/distilled promotion path in memory-core that preserves source provenance, has deterministic fallback behavior, and updates schema, docs, and focused tests around raw versus distilled `MEMORY.md` output.
