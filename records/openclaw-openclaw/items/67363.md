## Summary

Keep this open. Current main still ranks and rehydrates eligible daily-memory snippets, then formats the resulting snippet directly into `MEMORY.md`; the recent contamination filters reduce artifact leakage but do not add a distilled-promotion or review-first contract. The KinthAI distillation comment and production audit data add useful implementation context, and related #67580, #68774, and #72887 track adjacent failure modes rather than superseding this broader durable-promotion behavior question.

## What This Changes

The issue asks whether deep dreaming promotion should stop appending raw daily-memory snippets directly to `MEMORY.md` and instead distill or otherwise curate them before long-term storage.

## Best Possible Solution

Define an explicit durable-promotion contract for memory-core: either document raw provenance-preserving appends as intentional, or add a review/distillation path that preserves source provenance, has deterministic fallback behavior, exposes any needed config/schema/docs, and includes focused tests for raw versus curated `MEMORY.md` output and orphan-header rejection.

## Reproduction Assessment

Yes. A high-confidence reproduction path exists by creating or recording an eligible short-term memory candidate from `memory/YYYY-MM-DD.md`, ranking it, and applying promotion; current source shows the selected snippet is rehydrated and appended verbatim to `MEMORY.md`, and the recent production audit provides concrete observed examples.

## Solution Assessment

Unclear. There is no current or proposed merged solution for the broad issue; deterministic chunker/header filters would reduce obvious noise, but the maintainable fix needs maintainer agreement on whether durable promotion should be direct, review-first, distilled, configurable, or some combination of those modes.
