## Summary

Current main still promotes rehydrated source snippets directly into MEMORY.md. Recent fixes and related threads filter some dreaming artifacts and add review surfaces for DREAMS.md/grounded backfill, but they do not distill valid daily-memory snippets or provide a distilled-promotion mode. Related #67580/#68774 and #72887 are narrower or adjacent, not a superseding canonical fix for this broader distillation question.

## Best Possible Solution

Keep this issue open for memory-core maintainer triage. Decide whether direct raw-snippet promotion is intended, and either document it clearly or implement a distilled-promotion path in the memory-core ranking/apply flow. If distilled output is desired, preserve source provenance, add focused tests for raw versus distilled MEMORY.md output, update config/schema only if a user-facing mode is added, and coordinate with #72887 if maintainers prefer a review-queue-first workflow.
