## Summary

Keep open. Current main includes some related announce fixes, including threadId preservation, but it does not implement the PR's central zero-timeout fail-closed/no-I/O announce planning. Zero-timeout sends still return pending announce metadata and start the background A2A flow, which can wait for agent completion/history and then resolve an announce target through the async lookup path.

## Best Possible Solution

Keep the PR open for maintainer review. If the direction is still desired, rebase it onto current main, preserve the already-shipped threadId/deliveryContext and ACP-owned-child behavior, add a minimal no-I/O announce plan for timeoutSeconds=0, address the runSessionsSendA2AFlow compatibility review, and keep the focused regression tests around zero-timeout, lookup latency, and routing precedence.
