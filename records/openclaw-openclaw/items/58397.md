## Summary

Keep this PR open. Current main still has the zero-timeout announce behavior the PR is trying to tighten, so it is not obsolete, but the PR branch is not merge-ready because it makes `announcePlan` mandatory while leaving existing direct A2A flow tests/callers on the old shape.

The PR changes `sessions_send` announce-target planning to classify external, internal, and unknown targets, makes `timeoutSeconds=0` use synchronous fail-closed routing decisions, passes an announce plan into the A2A flow, and splits/expands related sessions/subagents tests.

## Best Possible Solution

Carry the fail-closed zero-timeout planning forward on top of current main, but first update `runSessionsSendA2AFlow` compatibility or all remaining direct callers/tests, then rerun the targeted sessions/A2A tests plus the changed-lane gate.

## Solution Assessment
