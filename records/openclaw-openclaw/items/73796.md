## Summary

Keep this issue open. Current main still rejects inline `MEDIA:~/...` directives and the browser CLI still emits `MEDIA:` lines through `shortenHomePath`, so the reported Telegram screenshot delivery failure remains on main. There is also an open unmerged implementation candidate, #73798, with closing syntax for this issue; policy says to keep the issue open until that PR lands or is closed.

## Best Possible Solution

Keep this issue open while maintainers review #73798. The conservative path is to land or revise that PR so browser machine-readable `MEDIA:` lines use absolute paths, with regression tests for screenshot and AI snapshot output; if maintainers prefer parser normalization instead, add focused parser and outbound-media tests proving only intended `~`/`~/` paths are expanded and traversal or credential paths stay blocked.
