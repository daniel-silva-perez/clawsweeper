## Summary

Keep open. Current main still has the reported producer/validator mismatch: the browser CLI emits tilde-shortened `MEDIA:` lines for screenshots and AI snapshot images, while the shared media parser still rejects `~`-prefixed media sources before outbound delivery can attach them. The related implementation candidate #73798 is closed and unmerged in the provided GitHub context, so there is no landed fix to close against.

## What This Changes

The issue requests fixing browser screenshot delivery when agent replies contain `MEDIA:~/...` paths that are rejected by the shared media-source validator.

## Best Possible Solution

Land a narrow browser plugin fix that emits absolute paths only for machine-readable `MEDIA:` lines from screenshot and AI snapshot image output, keeps human-readable display paths shortened, preserves parser rejection of `~` and traversal paths, and adds focused regression tests plus a user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists: browser screenshot output under the user's home directory is shortened to `MEDIA:~/...`, and the current parser rejects `~` media candidates before outbound delivery can attach them.

## Solution Assessment

No for current main. The best maintainable fix is producer-side absolute `MEDIA:` emission for trusted browser output; parser-side tilde expansion is broader and would need separate security review because current tests intentionally reject home-dir media paths.
