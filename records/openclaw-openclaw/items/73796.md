## Summary

Keep this issue open. The checked-out main branch and current public main still reject inline `MEDIA:~/...` directives, while the browser CLI still emits screenshot and AI snapshot `MEDIA:` lines through `shortenHomePath(...)`. PR #73798 is open, unmerged, and is the active implementation candidate with closing syntax for this issue, so the issue should remain open until that PR lands or is closed.

## Best Possible Solution

Keep this issue open while maintainers review PR #73798. The conservative fix is to land or revise that PR so machine-readable browser `MEDIA:` lines use absolute paths, with regression coverage for screenshot and AI snapshot output, while preserving the shared parser's existing rejection of home-dir and traversal-style media sources.
