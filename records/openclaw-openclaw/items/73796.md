## Summary

Keep open. Current main still has the reported producer/validator mismatch: browser inspect commands can emit tilde-shortened machine-readable media paths, while the shared reply media parser still rejects those paths before channel delivery; the prior focused fix PR was closed unmerged.

## What This Changes

The issue requests reliable chat delivery for browser screenshots and AI snapshot images when browser media output is emitted as a tilde-prefixed `MEDIA:` directive.

## Best Possible Solution

Use a producer-side fix in the browser plugin: emit absolute paths returned by the browser route for machine-readable screenshot and AI snapshot `MEDIA:` lines, keep shortened paths only for human-readable logs, and preserve the shared parser's traversal and home-dir rejection policy.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: browser inspect output can call `shortenHomePath(...)`, and `splitMediaFromOutput` rejects candidates starting with `~`; the current parser tests encode that rejection.

## Solution Assessment

Yes, the producer-side fix is the narrowest maintainable direction. It uses the browser route's already-absolute saved paths and avoids weakening the shared media parser's security boundary.
