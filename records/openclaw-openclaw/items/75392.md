## Summary

Keep open. The PR addresses a real current-main config footgun: both runtime schema sites still accept only the string enum, while the paired issue reports a concrete v2026.4.29 boolean-config failure. The proposed direction is reasonable, but the patch is not ready because it changes the config schema without updating the checked-in generated config schema surface, and its changelog entry is missing the required contributor credit.

## What This Changes

This PR adds a shared Zod `visibleReplies` schema that coerces boolean `true`/`false` to canonical visible-reply modes, wires it into global and group message config, adds regression tests, and documents the fix in the changelog.

## Best Possible Solution

Keep the schema-layer compatibility fix, but land it with the generated config schema and changelog policy aligned. If Zod transform output weakens the public JSON Schema, add a small schema helper or targeted test so the runtime accepts booleans while the generated config schema still represents only the intended string and boolean inputs.

## Solution Assessment
