## Summary

Keep this issue open. Current main partially addresses the report: restart sentinel now uses the async atomic JSON writer, and `saveJsonFile` has a sync temp-file, fsync, rename, chmod, and directory-sync implementation with tests. The requested shell profile path still writes the user profile directly, and exec approvals still has a bespoke temp+rename writer that lacks the same fsync and parent-directory sync guarantees as the durable JSON helpers. The related PR #57895/comment context is useful, but it only covered the restart sentinel portion and is closed unmerged; the broader issue still has remaining work.

## Best Possible Solution

Keep the issue open and narrow the remaining implementation to the unfinished paths: add or reuse a sync durable text/JSON writer for exec approvals with temp write, file fsync, rename, chmod, and parent-directory fsync, and use the async `writeTextAtomic` path for shell profile creation/update with `0o644` mode. A follow-up can decide whether the completion cache write should also move to `writeTextAtomic`, but the user-facing shell profile and exec approvals paths are the important blockers.
