## Summary

Keep this issue open. Current main partially resolves the report: restart sentinel writes and the shared sync JSON helper now use durable atomic paths, but shell completion install still writes shell profiles directly and exec approvals still uses a temp-and-rename helper without the temp-file fsync and parent-directory sync guarantees requested here.

## Best Possible Solution

Keep the issue open and narrow the remaining work to two paths: route shell profile create/update through the shared async atomic text writer with `0o644` mode, and replace `writeExecApprovalsRaw` with a sync durable writer that preserves the current destination guards and mode behavior while adding temp-file fsync, rename, chmod, cleanup, and parent-directory sync. Add focused tests for both paths.
