## Summary

Keep this issue open. Current main partially implements the report: restart sentinel writes and `saveJsonFile` now use durable atomic paths, but shell completion install still writes shell profiles directly and exec approvals still uses a bespoke temp-and-rename writer without the fsync and parent-directory sync guarantees described in the issue. The related PR #57895 context is useful, but it only covered the restart-sentinel slice and is closed unmerged.

## Best Possible Solution

Keep this issue open and narrow the follow-up to the two unfinished paths: route shell profile creation/update through `writeTextAtomic` with `0o644` mode, and replace `writeExecApprovalsRaw` with a sync durable writer that preserves the current symlink guards and mode behavior while adding temp-file fsync, rename, chmod, and parent-directory fsync. Add focused tests for both paths.
