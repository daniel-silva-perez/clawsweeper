## Summary

Keep open. The feature is still useful, and current main has not implemented WhatsApp forwarding metadata, but the PR branch is stale against the current inbound pipeline and would need a rebase onto the newer context builder before merge.

## What This Changes

The PR adds WhatsApp Baileys forwarding metadata extraction, carries optional forwarding fields through inbound message handling, renders a forwarding score in the forwarded-message prompt block, and adds forwarded/non-forwarded tests.

## Best Possible Solution

Rebase the feature onto the current WhatsApp inbound pipeline by extracting forwarding info in the inbound extractor, carrying typed optional fields through WebInboundMessage and monitor enqueue, adding ForwardedFrom and ForwardingScore in buildWhatsAppInboundContext, rendering forwarding_score only inside the fenced untrusted forwarded metadata block, and explicitly deciding the related plugin-hook exposure scope.

## Reproduction Assessment

Yes for a code-level reproduction: a forwarded Baileys proto.IMessage can carry contextInfo.isForwarded and contextInfo.forwardingScore, and current main has no path that extracts those fields into the WhatsApp prompt context. Live WhatsApp account verification remains missing.

## Solution Assessment

No. The requested direction is sound, but this PR is not the best current patch because it targets the old process-message finalization surface instead of the current extractor, monitor, WebInboundMessage, and buildWhatsAppInboundContext path.
