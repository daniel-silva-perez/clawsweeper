## Summary

Keep open. Current main still drops WhatsApp forwarding metadata before it reaches either the agent prompt or plugin hook events, but this PR is stale against the current inbound path and needs a maintainer/contributor rebase before it can merge.

## What This Changes

The PR adds WhatsApp `contextInfo.isForwarded` and `contextInfo.forwardingScore` extraction, carries optional forwarding fields through inbound message handling, renders a forwarding score in the forwarded-context prompt block, and adds forwarded/non-forwarded tests.

## Best Possible Solution

Rebase the feature onto current main by extracting forwarding info in `extensions/whatsapp/src/inbound/extract.ts`, carrying typed optional fields through `WebInboundMessage` and `monitor.ts`, adding `ForwardedFrom` and `ForwardingScore` in `buildWhatsAppInboundContext()`, rendering `forwarding_score` only inside the fenced untrusted forwarded metadata block, and either include or explicitly defer the #69374 hook contract work.

## Reproduction Assessment

Yes for the current-main code path: a forwarded WhatsApp `proto.IMessage` can carry `contextInfo.isForwarded` and `contextInfo.forwardingScore`, but current main has no extraction, inbound type, enqueue, context-builder, or prompt-rendering path for those fields. Live account verification is still missing.

## Solution Assessment

No. The requested direction is sound, but this PR is not the best current patch because it targets the old `process-message.ts` finalization surface instead of the current `buildWhatsAppInboundContext()` and current inbound/hook tests.
