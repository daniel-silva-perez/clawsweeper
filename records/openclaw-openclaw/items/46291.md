## Summary

Keep open. Current main still drops WhatsApp forwarding metadata before it can reach the agent prompt, so the PR’s requested behavior remains useful; however, the branch is stale against the current WhatsApp inbound-dispatch flow and needs maintainer-guided rebase/adaptation before it can land.

## What This Changes

The PR extracts WhatsApp `contextInfo.isForwarded` and `contextInfo.forwardingScore`, threads optional forwarding fields through the WhatsApp inbound message and prompt context, renders `forwarding_score`, and adds forwarded/non-forwarded regression tests.

## Best Possible Solution

Rebase the contribution onto current main by adding a WhatsApp forwarding extractor, carrying the signal through `WebInboundMessage` and `monitor.ts`, populating forwarded metadata in `buildWhatsAppInboundContext()`, rendering any score only inside untrusted prompt metadata, and adding current-path regression tests. Maintainers should also decide whether #69374 should be handled in the same contract pass or left as a separate plugin-hook follow-up.
