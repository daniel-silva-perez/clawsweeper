## Summary

Keep this open. Current main still exposes session reset mode as only `daily | idle` in the public config types, zod validation, generated schema, and docs. There is an internal freshness test for `idleMinutes=0`, but the user-requested public `mode: "never"` is not accepted or documented for `session.reset`, `resetByType`, or `resetByChannel`.

## Best Possible Solution

Implement a first-class `mode: "never"` session reset policy in core config and runtime: extend `SessionResetMode` and the zod/generated schema, make `resolveSessionResetPolicy` and `evaluateSessionFreshness` skip both daily and idle expiry for that mode, keep explicit `/new` and `/reset` behavior unchanged, and add focused tests for base `session.reset`, `resetByType`, and `resetByChannel` precedence plus docs examples on docs.openclaw.ai.
