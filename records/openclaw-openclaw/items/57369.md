## Summary

Keep this issue open. Current main still exposes session reset as `daily | idle` across the public config type, zod validation, runtime policy type, generated schema/help, tests, and docs. The internal `idleMinutes=0` freshness behavior is not a first-class public `mode: "never"` option because structured config validation requires positive idle minutes and rejects any mode outside daily/idle.

## Best Possible Solution

Implement a first-class `mode: "never"` session reset policy in core config and runtime. Extend `SessionResetMode`, zod validation, generated schema/help, docs, and focused tests; make resolved policies skip both daily and idle automatic expiry for that mode while preserving explicit `/new` and `/reset`.
