## Summary

Keep this issue open. Current main at 230f7122 still exposes session reset policy as only `daily | idle` in public types, runtime validation, generated config schema, runtime freshness evaluation, tests, and docs. The existing internal `idleMinutes=0` freshness behavior is not a public solution because structured config validation and generated schema require positive `idleMinutes` and still reject/document no `mode: "never"`.

## Best Possible Solution

Implement a first-class `mode: "never"` session reset policy in core config and runtime. Extend `SessionResetMode`, zod validation, generated schema/help, and docs; make resolved policies skip both daily and idle automatic expiry for that mode while preserving explicit `/new` and `/reset`; add focused tests for base policy, type override, channel override, and precedence.
