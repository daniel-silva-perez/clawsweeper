## Summary

Keep open. Current main partially reduced the false positives by only adding some hard-coded version-manager fallback dirs when they exist, but the macOS PATH audit still compares the service PATH against unconditionally expected user-bin dirs such as `~/.npm-global/bin`, `~/bin`, and `~/.nix-profile/bin`, then reports them as `Gateway service PATH missing required dirs`. That leaves the central cosmetic warning and wording problem from this issue unresolved.

## Best Possible Solution

Implement a narrow macOS service PATH audit fix: stop warning about nonexistent, unconfigured user-bin fallback dirs, and change the human message away from `required dirs` toward recommended/preferred entries. Preserve signal for existing or explicitly configured dirs that are missing from the LaunchAgent PATH.

## Clownfish Work Prompt

Fix the macOS gateway service PATH audit false positives for nonexistent user-bin fallback dirs. Current main already gates several hard-coded version-manager fallback dirs with existence checks, but Darwin still unconditionally expects `~/.npm-global/bin`, `~/bin`, and fallback `~/.nix-profile/bin`, and `auditGatewayServicePath` reports missing entries as `required dirs`. Implement a narrow fix so `openclaw gateway status`/service audit does not warn for nonexistent, unconfigured fallback dirs on macOS, while preserving warnings for existing or explicitly configured PATH roots such as `PNPM_HOME`, `NPM_CONFIG_PREFIX`, `VOLTA_HOME`, `ASDF_DATA_DIR`, `FNM_DIR`, and existing version-manager dirs. Reword the warning away from `required dirs` if it can still include preference-style entries. Likely files: `src/daemon/service-audit.ts`, `src/daemon/service-audit.test.ts`, `src/daemon/service-env.ts`, `src/daemon/service-env.test.ts`, and possibly `docs/gateway/doctor.md` plus `CHANGELOG.md`. Add focused tests for a Darwin service PATH like `/opt/homebrew/opt/node/bin:/Users/test/.local/bin:/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin` with nonexistent `~/.npm-global/bin`, `~/bin`, version-manager dirs, and Nix fallback, and tests that existing or env-configured dirs still warn when missing. Add a user-facing changelog fix entry crediting @kesjam. Do not change Node runtime migration policy, gateway install command shape, Windows Scheduled Task behavior, or unrelated service environment handling.
