## Summary

Keep open. Current main still does not expose Gemini Flex service-tier configuration for the bundled Google provider. The direct Gemini transport builds and serializes the REST payload itself, and the existing params path only forwards Google-specific cachedContent plus shared options such as temperature, max tokens, transport, and thinking. This is a narrow valid Google plugin feature request and a reasonable fix-PR candidate.

## Best Possible Solution

Implement this in the bundled `google` provider: accept `agents.defaults.models["google/<model>"].params.serviceTier` and `service_tier` for direct `google-generative-ai` requests, validate at least `flex`, emit top-level REST `service_tier: "flex"`, preserve caller-supplied payload values from `onPayload`, add focused Google transport and extra-param tests, update Google provider docs, and add a user-facing changelog entry crediting @adriandubiel.
