## Summary

Keep open. Current main does not provide a Gemini `serviceTier`/`service_tier` model-param path: the bundled Google plugin builds the Gemini REST payload directly and only forwards existing options such as temperature, max output tokens, thinking config, and `cachedContent`. The requested Flex field is a top-level Gemini REST request body field, so this is a valid narrow Google plugin feature request rather than a duplicate, stale report, or ClawHub/core-boundary close.

## Best Possible Solution

Implement this in the bundled `google` provider: accept `agents.defaults.models["google/<model>"].params.serviceTier` and `service_tier` for direct `google-generative-ai` requests, validate at least `flex`, emit top-level `service_tier: "flex"` in the Gemini REST payload, preserve any caller-supplied payload value, add focused Google transport/extra-param tests, update the Google provider docs, and add a user-facing changelog entry crediting the reporter.

## Clownfish Work Prompt

Implement Gemini Flex service tier support for the bundled Google provider. The observable user problem is that `agents.defaults.models["google/<model>"].params` cannot currently request Gemini Flex, even though Google requires top-level REST `service_tier: "flex"` (JS SDK config spelling is `serviceTier`). Keep the fix in the Google/provider request path; do not add Google-specific behavior to core defaults, do not change OpenAI/Anthropic/Codex service-tier behavior, and do not add automatic fallback/retry policy for Flex capacity failures.

Expected boundary:
- Accept both `serviceTier` and `service_tier` model params for direct `google-generative-ai` runs, with the higher-precedence model/agent override behavior matching existing extra-param handling.
- Validate conservatively, at minimum supporting `"flex"`; log and ignore invalid values if following existing param patterns.
- Emit top-level `service_tier: "flex"` in the Gemini REST payload built by `extensions/google/transport-stream.ts`, not inside `generationConfig`, and preserve an already-present caller/onPayload `service_tier` value.
- Add focused tests for extra-param propagation and final payload shape, including snake/camel aliases.
- Update Google model-provider docs with the config example and add a user-facing changelog entry under the active version with `Thanks @adriandubiel`.
