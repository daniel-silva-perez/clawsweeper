## Summary

Current main still lacks Gemini Flex service-tier support in the bundled Google provider. The direct Gemini transport has no `serviceTier`/`service_tier` option, the extra-param path only forwards existing generic params plus `cachedContent`, and the Google provider docs still omit Flex configuration.

## What This Changes

Request support for configuring Gemini Flex service tier on Google Gemini model requests so OpenClaw can send `service_tier: "flex"`.

## Best Possible Solution

Implement a narrow bundled Google provider change that accepts documented Gemini service-tier params, normalizes `serviceTier` and `service_tier`, emits top-level `service_tier: "flex"` for direct Gemini REST calls, preserves explicit payload overrides, and documents the configuration with focused tests and a credited changelog entry.
