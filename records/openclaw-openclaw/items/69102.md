## Summary

Current main still lacks Gemini Flex service-tier support for the bundled Google provider: the direct Gemini transport has no `serviceTier`/`service_tier` option, the extra-param wrapper only forwards existing generic params plus `cachedContent`, and the Google provider docs still only document `cachedContent` for native Gemini extras.

## What This Changes

Request support for configuring Gemini Flex service tier on Google Gemini model requests, allowing `service_tier: "flex"` / `serviceTier` to be sent to the Gemini API.

## Best Possible Solution

Implement a narrow bundled Google provider change that accepts documented Gemini service-tier params for direct Gemini REST requests, normalizes `serviceTier` and `service_tier`, emits top-level `service_tier: "flex"`, preserves explicit payload overrides, adds focused transport and extra-param coverage, updates the Google provider docs, and adds a user-facing changelog entry crediting the reporter.
