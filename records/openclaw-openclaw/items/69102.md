## Summary

Current main still does not expose Gemini Flex service-tier configuration for the bundled Google provider. Google documents Flex as `service_tier: "flex"` in the REST request body and `config.serviceTier` in the JavaScript SDK; OpenClaw's direct Gemini transport builds REST JSON itself, but only forwards cachedContent, thinking/tool/temperature/max-token options, not serviceTier/service_tier.

## Best Possible Solution

Keep this issue open and implement a narrow bundled Google provider fix: accept `agents.defaults.models["google/<model>"].params.serviceTier` and `service_tier` for direct Gemini REST requests, validate at least `flex`, emit top-level `service_tier: "flex"`, preserve caller/onPayload payload values, add focused transport and extra-param tests, update the Google provider docs, and add a user-facing changelog entry crediting the reporter.
