## Summary

Keep open. Current main has a strong fix for the `chat.send`/web-upload transcript path, including `offloadedRefs` conversion into `MediaPath`/`MediaPaths`, but the iOS share path still enters through `node.event` / `agent.request` and `server-node-events.ts` does not carry `parsed.offloadedRefs` into transcript media fields. The reporter's link to merged PR #55513 remains useful root-cause context for the claim-check offload behavior, but it does not make this node-path transcript persistence gap closeable.

## Best Possible Solution

Keep the issue open and implement the same media persistence contract for `server-node-events.ts` / `agent.request` that `chat.send` now uses: carry `parsed.offloadedRefs` through the node ingress path, persist or rewrite the user transcript entry with ordered `MediaPath`/`MediaPaths` and `MediaType`/`MediaTypes`, and add a focused node-events/iOS-share regression test for a large image offload. If the intended product direction is to treat node-event image offloads as prompt-only `media://` markers instead, document that explicitly and adjust this issue with maintainer judgment rather than auto-closing it.
