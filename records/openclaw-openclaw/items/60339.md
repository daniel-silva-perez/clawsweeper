## Summary

Keep open. Current main still drops the node/iOS `parsed.offloadedRefs` metadata after attachment parsing, and the linked #74625 is an open, unmerged closing PR for the exact persistence gap.

## What This Changes

Persist offloaded media refs from iOS share and node `agent.request` attachments into transcript `MediaPath`/`MediaPaths` so history replay can resolve images.

## Best Possible Solution

Land #74625 or an equivalent narrow gateway change that reuses the existing `chat.send` media persistence/rewrite contract for `server-node-events.ts`, then keep the node-events regression test that proves offloaded image refs become ordered transcript media fields.

## Reproduction Assessment

Yes. The static reproduction path is high-confidence: iOS share sends `node.event`/`agent.request`, `parseMessageWithAttachments` returns `offloadedRefs` for large images, and current `server-node-events.ts` drops those refs before dispatch or transcript rewrite.

## Solution Assessment

Yes. Reusing the existing `chat.send` persistence and transcript rewrite helpers is the narrowest maintainable direction; duplicating a separate node-only transcript writer would be riskier.
