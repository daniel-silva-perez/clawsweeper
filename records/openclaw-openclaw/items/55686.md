## Summary

Keep this issue open. Current main has the lower-level Feishu interactive-card button and callback infrastructure, but the reported `/reasoning` option flow still returns plain `Options: on, off, stream` text and the direct Feishu reply dispatcher currently sends only text/media replies, so the requested command-option button UX remains unimplemented. The related Feishu inline-button work is useful context, and the open Feishu callback-field work is adjacent, but neither closes this specific command-menu gap.

## What This Changes

The issue requests rendering command-option choices such as `/reasoning` in Feishu as clickable card buttons instead of requiring users to type the option value.

## Best Possible Solution

Add a focused command-option presentation path for known directive/menu replies and make Feishu's normal reply pipeline render those presentation buttons as native cards using the existing `ocf1` quick-action callback path, while retaining the current text fallback for channels or contexts that cannot render buttons.
