## Summary

Keep this PR open. Current main still lacks the requested Slack per-channel `replyToMode` override, and the PR is a focused implementation candidate rather than obsolete cleanup. However, the provided review discussion identifies a real P1 blocker in the PR head: `channelConfig` is referenced out of scope in the routing path, so it needs changes before merge.

## Best Possible Solution

Keep this PR open and ask for a small fix/rebase: add `replyToMode` to the Slack per-channel type and strict schema, resolve exact/wildcard channel overrides in `resolveSlackChannelConfig`, pass the resolved channel config into Slack routing, propagate the resolved mode through message/threading context, and add focused tests plus Slack docs/config metadata updates. After those changes land, this PR or its landed replacement can close the original per-channel threading request.
