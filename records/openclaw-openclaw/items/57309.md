## Summary

Keep this PR open. Current main still does not implement Slack per-channel `replyToMode`, and the PR remains a focused implementation candidate, but the PR head has a real blocker from the review discussion: `channelConfig` is referenced out of scope in the routing path and needs to be plumbed through before merge.

## Best Possible Solution

Keep this PR open and request a small fix/rebase: add `replyToMode` to Slack per-channel config types and strict schema, resolve exact/wildcard channel overrides in `resolveSlackChannelConfig`, pass the resolved channel config into Slack routing, propagate the resolved mode through `MsgContext` and `ChannelThreadingContext`, update Slack docs and generated config metadata, and add focused resolver/routing/threading-tool tests. After that lands, this PR or its landed replacement can close the original per-channel Slack threading request.
