## Summary

Keep this PR open. Current main still has no automatic macOS idle-sleep prevention in the gateway loop, and the existing docs/wrapper support are manual workarounds rather than the behavior proposed here. The PR is a plausible focused fix, but it should be revised before merge because it adds gateway-startup process execution and the submitted patch has an acknowledged async spawn-error gap with no focused tests.

## Best Possible Solution

Keep this PR open for revision or replacement. The best implementation is a small macOS gateway lifecycle helper that starts a trusted `caffeinate -i -w <gateway pid>` assertion only while the gateway is alive, attaches `error` and relevant exit handling, cleans up in the existing run-loop finally path, and adds focused tests that mock platform and child_process behavior. If maintainers decide default idle-sleep prevention is too broad, document the manual wrapper/caffeinate path instead of treating the current code as implemented.

## Risks / Open Questions

- The submitted patch should not merge as-is: a missing or inaccessible caffeinate executable can emit an unhandled child_process error unless the child has an `error` listener.
- The patch adds a new gateway-startup process execution path and currently resolves `caffeinate` through PATH; a revision should prefer a trusted macOS binary path or deliberate resolution strategy.
- Preventing idle sleep by default is user-visible power-management behavior and may need maintainer product judgment, docs, and possibly an opt-out.
- No focused tests are present on current main for platform gating, child error handling, or cleanup of a caffeinate helper.
