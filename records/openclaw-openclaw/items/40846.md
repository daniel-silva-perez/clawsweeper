## Summary

Keep this PR open. Current main still does not implement automatic macOS idle-sleep prevention for the Gateway, and the PR remains a plausible focused implementation candidate. It should not be closed as implemented or obsolete; it needs revision for the command-execution/security review concerns, async spawn error handling, lifecycle tests, and maintainer product judgment on default power-management behavior.

## Best Possible Solution

Keep this PR open for revision or replacement. The best path is a small macOS Gateway lifecycle helper that starts a trusted `caffeinate -i -w <gateway pid>` assertion only while the Gateway is alive, attaches error and exit handling, cleans up in the existing `finally` path, and adds focused tests for Darwin, non-Darwin, spawn failure, and shutdown cleanup. Maintainers should also decide whether this behavior should be default, configurable, or documented as a manual wrapper/workaround.

## Risks / Open Questions

- The PR adds a Gateway startup command-execution path while its security section says the command/tool execution surface did not change.
- The submitted patch resolves `caffeinate` through PATH, which can be fragile or less trustworthy in launchd/app contexts than an explicit resolution strategy.
- Without a child `error` listener, spawn failure can crash the gateway instead of degrading gracefully.
- No focused tests cover Darwin gating, non-Darwin no-op behavior, spawn failure, or shutdown cleanup.
- Preventing idle sleep by default is user-visible power-management behavior and may need maintainer product judgment, docs, and possibly an opt-out.
