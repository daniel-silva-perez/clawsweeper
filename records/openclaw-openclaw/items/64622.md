## Summary

Keep open. Current main at 17811480dae20f65b526454d75e971ae8424ec10 still lacks the PR's central runtime changes: no A2A timeout floor, A2A announce provenance is still injected, webchat-backed main sessions still merge stale group/channel metadata, and outbound mirroring still preserves the resolver-selected route. The PR also is not merge-ready because the proposed timeout and outbound rewrites have source-backed routing risks, and the branch mixes unrelated docs-template policy into the runtime fix. Security pass found no workflow, dependency, lockfile, install, release, or secret-handling changes in the PR file list, but the outbound route rewrite is routing/privacy-sensitive.

## Best Possible Solution

Keep this PR open for revision or replacement. A mergeable path should split the docs-template policy into a separate PR, scope the timeout floor to true cross-agent sends with zero/short/long regression tests, add explicit webchat-backed main-session metadata cleanup in gateway persistence, and preserve outbound session continuity only when the resolver or route helper confirms the current session belongs to the same outbound route/base session.
