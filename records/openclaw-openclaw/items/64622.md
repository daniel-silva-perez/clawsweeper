## Summary

Keep open. Current main at 5828dcdb05aad587bff831467e7fa51f73312ff8 still lacks the PR's central A2A timeout/provenance, webchat main-session metadata cleanup, and outbound continuity changes. The submitted diff is also not merge-ready: two source-backed review blockers remain around timeout scope and outbound route rewriting, and the branch mixes unrelated docs-template policy into the runtime fix. Security pass found no workflow, dependency, lockfile, install, release, or secret-handling changes, but the outbound rewrite is routing/privacy-sensitive because it can persist external channel metadata onto the current session key.

## Best Possible Solution

Keep this PR open for revision or replacement. A mergeable path should split out the docs-template policy commit, scope the timeout floor to true cross-agent sends with focused zero/short/long tests, add explicit webchat-backed main-session metadata cleanup in gateway persistence with regression coverage, and preserve outbound session continuity only when the resolver or route helper confirms the current session belongs to the same outbound route/base session.
