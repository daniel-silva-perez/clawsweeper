## Summary

Keep this PR open. Current main still drops `HookContext` on the unwrapped `toToolDefinitions()` fallback path, while the client-tool path already forwards it. The PR remains a targeted implementation candidate with a regression test, not an obsolete cleanup item.

## Best Possible Solution

Review or refresh this PR rather than close it. The best implementation is to add optional `HookContext` threading through `toToolDefinitions()` and `splitSdkTools()`, pass session metadata from both embedded attempt and compaction paths, keep the regression test proving `ctx.agentId` and session fields reach `before_tool_call`, and decide whether to include `loopDetection` for parity with client tools. Keep broader plugin execute-context work such as #55742 separate unless maintainers explicitly coordinate the patches.
