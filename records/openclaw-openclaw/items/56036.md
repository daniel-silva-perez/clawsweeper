## Summary

Keep open. Current main still lacks the PR's hook-context threading for the unwrapped `toToolDefinitions()` fallback path, while the client-tool path already forwards `hookContext`. The provided PR remains a plausible targeted fix with a focused regression test, and it should be reviewed/refreshed rather than cleaned up as obsolete.

## Best Possible Solution

Review or refresh this PR rather than close it. The best implementation is to add optional `HookContext` threading through `toToolDefinitions()` and `splitSdkTools()`, pass session metadata from both embedded attempt and compaction paths, retain the focused regression test proving `ctx.agentId` and session fields reach `before_tool_call`, and decide whether to include `loopDetection` for parity with client tools. Keep related plugin execute-context work such as #55742 separate unless maintainers choose to coordinate the two patches.
