## Summary

Keep open. Current main does not implement this PR's final early-allow behavior for device-less operator WebSocket connects authenticated through Tailscale/trusted-proxy transport, and the current code, tests, and docs still enforce a narrower device-identity policy. The PR remains a plausible implementation candidate for the linked gateway scope reports, but it changes a security-sensitive auth/scope boundary and should get maintainer review rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or replacement with a policy-aligned fix. If maintainers want device-less Tailscale/trusted-proxy operator sessions to retain scopes, implement that in the gateway WebSocket auth path with an integration test covering connect plus an operator.write-scoped RPC. If the current device-identity policy should stand, close this PR together with a clear resolution on the linked gateway scope reports and the overlapping #51413 PR.
