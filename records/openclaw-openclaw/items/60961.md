## Summary

Keep open. Current main still has the reported OpenShell mirror-mode single-file upload path: `writeFile` syncs the written host file through `syncLocalPathToRemote`, and that method still uploads the original `localPath` into the remote parent directory instead of staging a basename-only file snapshot.

## What This Changes

The issue requests that OpenShell mirror-mode writes into existing remote subdirectories succeed instead of failing when `openshell sandbox upload` preserves a source directory component.

## Best Possible Solution

Fix this inside the OpenShell plugin by staging regular-file mirror writes into a temporary flat directory using `path.basename(localPath)`, uploading that staging directory to `path.posix.dirname(remotePath)`, and cleaning it up, with regression coverage for nested writes into an already existing remote directory.

## Reproduction Assessment

Yes. The issue includes concrete steps and logs, and current main still follows the same static path: nested mirror writes call `syncLocalPathToRemote`, which uploads the original host path into the remote parent directory.

## Solution Assessment

Yes. The proposed basename-only staging approach matches the existing full-workspace upload safety pattern and is the narrowest maintainable fix, provided it keeps symlink handling and root-boundary checks intact.
