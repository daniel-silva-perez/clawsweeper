## Summary

Keep this issue open. Current main still has the reported OpenShell mirror-mode single-file upload path: writes go through `syncLocalPathToRemote`, which uploads the original host file path directly into the remote parent directory. The fork commit linked from the timeline contains the proposed staging fix, but it is not merged into current main and no open PR was found referencing this issue.

## Best Possible Solution

Keep this issue open and implement the forked staging approach in `extensions/openshell/src/backend.ts`: after creating the remote parent directory, copy the regular file into a temporary flat staging directory using `path.basename(localPath)`, upload that staging directory to `path.posix.dirname(remotePath)`, clean it up, and add mirror-mode regression coverage for nested writes into existing directories.
