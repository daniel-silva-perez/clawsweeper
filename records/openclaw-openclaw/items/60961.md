## Summary

Keep this issue open. Current main still routes OpenShell mirror-mode write-tool updates through `syncLocalPathToRemote`, and that method still uploads the original host file path directly into the remote parent directory instead of staging a basename-only file. The reporter's diagnosis and workaround remain consistent with the source, and the later ClawSweeper comment's keep-open finding still applies to current main.

## Best Possible Solution

Keep this issue open and fix it in `extensions/openshell/src/backend.ts`: for regular-file `syncLocalPathToRemote`, create the remote parent, copy the local file into a temporary flat staging directory using `path.basename(localPath)`, upload that staging directory to `path.posix.dirname(remotePath)`, clean up the temp directory, and add regression coverage for mirror-mode nested writes into an already-existing remote subdirectory.
