## Risks / Open Questions

- Benchmark workflow downloads a hyperfine .deb and installs it with sudo without integrity verification, creating a supply-chain risk on PR/push CI runs.
- SBOM workflow scans the full checkout after a full dependency install, which can include dev/test tooling and may not describe the shipped npm tarball or production dependency set.
- Release-triggered SBOM job grants contents:write and uses a third-party release upload action, so permissions, trigger scope, and action choice need explicit maintainer review.
- Benchmark workflow duplicates the existing startup benchmark harness instead of using package.json scripts and checked fixture/budget behavior.
- Devcontainer and workflows use setup choices that diverge from the repository's maintained Node/pnpm setup action and current Node 24 CI defaults.
- The PR body uses closing syntax for a related issue that prior review context says is unrelated, so linkage should be corrected before merge.
