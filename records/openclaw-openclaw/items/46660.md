## Summary

Keep this PR open. The underlying A2A visibility gap is real and the DingTalk channel-plugin use case is credible, but the branch is stale against the current hook-contract split and it introduces a raw A2A transcript hook without the conversation-access gate used for comparable plugin hooks.

## What This Changes

The branch adds a public `agent_to_agent_turn` plugin hook, exports a hook-runner method, and emits A2A turn metadata, reply text, flow identity, and delivery target data from the `sessions_send` A2A flow.

## Best Possible Solution

Ship a first-class A2A turn visibility hook only after moving the contract to `src/plugins/hook-types.ts`, adding it to the conversation-content access policy, documenting the public contract, and covering both loader policy and A2A turn emission with focused tests.

## Reproduction Assessment

Yes. A static current-main check is enough: intermediate A2A turns run through `runAgentStep`, while only the final announce path calls gateway `send`, so channel plugins cannot observe the ping-pong transcript through existing delivery hooks.

## Solution Assessment

No. The feature direction is plausible, but this branch is not the best merge shape because it adds the contract in a stale file and does not apply the current raw-conversation-access policy; the safer path is a rebased `hook-types`/registry/docs/tests change.
