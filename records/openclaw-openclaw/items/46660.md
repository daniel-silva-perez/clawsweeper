## Summary

Keep open. Current main still lacks an A2A turn-level plugin hook or equivalent observable seam, and the PR targets a real gap for channel-visible multi-agent exchanges. The proposed branch still needs maintainer design/security review because it is stale against the current hook-contract split and exposes internal A2A reply text without the current conversation-access gate.

## What This Changes

The branch adds `agent_to_agent_turn` hook types and hook-runner plumbing, then emits turn events from `runSessionsSendA2AFlow` with flow metadata, reply text, and delivery target data before the final announce.

## Best Possible Solution

Ship a first-class A2A turn hook only after rebasing onto the current hook contract surface, deciding and enforcing the conversation-access policy, documenting the public hook at the plugin docs site, and adding focused tests around registration, metadata, turn ordering, timeout behavior, and final announce delivery.
