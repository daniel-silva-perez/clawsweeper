## Summary

Keep this PR open. Current main still lacks an A2A turn visibility hook, and the linked DingTalk multi-agent use case is concrete, but this branch should not merge as-is because it is stale against the current hook contract split and bypasses the raw conversation-content access policy.

## What This Changes

The PR adds an `agent_to_agent_turn` plugin hook, a hook-runner method, and A2A flow emissions carrying turn metadata, reply text, flow identity, and delivery-target data from `sessions_send` ping-pong exchanges.

## Best Possible Solution

Ship a first-class A2A turn visibility hook after rebasing onto current main, placing the public contract in `src/plugins/hook-types.ts`, adding the hook to the conversation-content access policy, documenting the operator opt-in, and covering loader policy plus A2A turn emission with focused tests.

## Reproduction Assessment

Yes. A static current-main check is enough: intermediate A2A ping-pong turns run through `runAgentStep`, only the final announce calls gateway `send`, and no existing hook emits those intermediate replies.

## Solution Assessment

No. The feature direction is plausible, but this branch is not the best merge shape because it adds the contract in a stale file and does not apply the current raw-conversation-access policy.
