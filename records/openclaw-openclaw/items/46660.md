## Summary

Keep this PR open. Current main still lacks an A2A turn visibility hook, and the linked DingTalk channel work shows a real integration need, but the proposed patch is not mergeable as-is because it targets a stale hook contract location and exposes raw internal A2A reply text without the current conversation-access gate.

## What This Changes

The PR adds a public `agent_to_agent_turn` plugin hook, a hook-runner wrapper, and A2A `sessions_send` emissions with turn metadata, reply text, flow identity, and delivery target data.

## Best Possible Solution

Ship a first-class A2A turn visibility hook only after rebasing it onto `src/plugins/hook-types.ts`, adding it to the conversation-content access policy, documenting the public contract, and covering both loader policy and A2A turn emission with focused tests.

## Reproduction Assessment

Yes. A static current-main check is enough: the A2A ping-pong loop runs internal `agent` calls with `deliver: false` and no hook invocation, so channel plugins cannot observe intermediate turns through existing outbound hooks.

## Solution Assessment

No. The feature direction is plausible, but this PR is not the best merge shape because it adds a core plugin API in the obsolete contract location and does not apply the current raw-conversation-access policy.
