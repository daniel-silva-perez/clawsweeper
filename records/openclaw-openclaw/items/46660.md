## Summary

Keep this PR open. Current main still lacks a turn-level A2A plugin hook, and the linked DingTalk/channel-plugin use case plus author analysis show existing hooks do not compose into the requested visibility. The branch is not merge-ready because it targets a stale hook contract surface and exposes internal A2A reply text without current conversation-access gating.

## What This Changes

The PR adds an `agent_to_agent_turn` plugin hook contract and emits A2A turn events from the `sessions_send` ping-pong flow with reply text, flow metadata, and delivery target fields.

## Best Possible Solution

Ship a first-class A2A turn hook only after rebasing it onto `src/plugins/hook-types.ts`, treating it as conversation-content access, documenting the public hook contract, and covering the sessions flow plus loader policy with focused tests.
