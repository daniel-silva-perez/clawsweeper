## Summary

Keep this PR open. Current main still lacks an `agent_to_agent_turn` hook, and the PR discussion plus linked DingTalk work show a real channel-plugin visibility gap, but the branch is stale against the current hook contract split and exposes internal A2A reply text without conversation-access gating.

## What This Changes

The PR adds an `agent_to_agent_turn` plugin hook contract, a hook-runner wrapper, and A2A `sessions_send` emissions with turn metadata, reply text, flow identity, and delivery target data.

## Best Possible Solution

Ship a first-class A2A turn visibility hook by adding the contract in `src/plugins/hook-types.ts`, wiring a hook-runner method and A2A emissions, treating the hook as conversation-content access, documenting the public contract, and covering both loader policy and sessions-flow behavior with focused tests.

## Reproduction Assessment

Yes. Current main can be checked statically: the A2A ping-pong loop runs through `runAgentStep` with `deliver: false` and no hook invocation, so channel plugins cannot observe intermediate turns through existing outbound hooks.

## Solution Assessment

No. The feature direction is plausible, but this PR is not yet the best merge shape because it targets obsolete hook contract locations and does not apply the current conversation-content access policy.
