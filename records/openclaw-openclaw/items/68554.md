## Summary

Keep this PR open, but it needs maintainer review and revision before merge. Current main still performs the locked `skipCache` session-store rereads, so the reported performance problem is not implemented on main; however, the proposed patch removes the fresh-read behavior that currently protects lock-honoring external writers and it assumes cached reads are cheap even though the cache path still synchronously clones large stores.

## What This Changes

The PR removes `{ skipCache: true }` from the locked `updateSessionStore` and `updateSessionStoreEntry` reads in `src/config/sessions/store.ts` and updates the comment to treat the session-store cache as current inside the lock.

## Best Possible Solution

Land a lock-aware fast path that avoids repeated large synchronous disk parses for ordinary queued gateway writes while preserving the accepted cross-process writer contract, or explicitly narrow that contract with tests and comments before removing the forced reread.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve external writes between lock holders:** `src/config/sessions/store.ts:429`
  - body: `withSessionStoreLock` releases the file lock after each queued task, so another OpenClaw process can update `sessions.json` before the next task runs. Switching this read to the in-process cache can miss a same-size/same-mtime external write and then overwrite it on save; keep a fresh read or add lock-aware cache invalidation before removing `skipCache` here and in `updateSessionStoreEntry`.
  - confidence: 0.84
- **[P2] Avoid cloning the full store on the hot path:** `src/config/sessions/store.ts:429`
  - body: The cached path on this PR head still returns `structuredClone(cached.store)`, so every queued write synchronously copies the whole store object. For the reported 33 MB `sessions.json` case, this keeps a large event-loop-blocking operation in the locked write path; the fast path needs a non-cloning cached read or equivalent bounded work.
  - confidence: 0.76
