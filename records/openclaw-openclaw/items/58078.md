## Summary

Keep this issue open. Current main still does not pre-install `iproute2` in the official Docker image; it only supports adding apt packages when a user rebuilds a local image. The official Docker release workflow does not pass that build arg, so the prebuilt `ghcr.io/openclaw/openclaw` image still would not satisfy the reported macvlan/routing use case.

## Best Possible Solution

If maintainers want to support this use case, add `iproute2` to the Docker runtime package list, cover it with the existing Dockerfile tests, and document any required macvlan or NET_ADMIN/runtime caveats. If maintainers decline the default image change, document `OPENCLAW_DOCKER_APT_PACKAGES="iproute2"` plus the required Compose capability/user settings as the supported custom-image path.
