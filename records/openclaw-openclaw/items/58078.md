## Summary

Keep open. Current main still does not preinstall `iproute2` in the official Docker runtime image: the runtime apt list omits it, and the Docker release workflow builds GHCR images without passing any extra apt package override. Existing proxy support and custom-image build args are useful alternatives, but they do not satisfy the requested official-image macvlan/default-route use case.

## What This Changes

The issue requests adding `iproute2` to the official Docker image so Synology/macvlan users can run `ip route replace` inside the container without rebuilding a custom image.

## Best Possible Solution

Keep this open until a maintainer decides the official runtime image policy. If accepted, add `iproute2` to the runtime apt list, update the Dockerfile package-list test, document the macvlan/`NET_ADMIN` caveat and proxy alternative, and validate the Docker image build path; if rejected, document the supported custom-image and `proxy.enabled` approaches instead.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: run the current official image and check `command -v ip` or `ip route replace ...`; the release workflow builds from a Dockerfile package list that omits `iproute2` and passes no extra apt packages.

## Solution Assessment

Unclear. Preinstalling `iproute2` is the direct user-requested fix, but the repository should first choose whether this belongs in the default official image or should remain a documented custom-image/proxy configuration because it changes the released package surface.
