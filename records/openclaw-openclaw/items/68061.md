## Summary

Keep this PR open as the active Mermaid implementation candidate, but it should not merge as-is. Current main still lacks Mermaid rendering in the shared WebChat/Control UI markdown path, while the PR introduces build/runtime blockers, removes existing sanitizer hardening, regresses task-list rendering, and mixes the feature with unrelated Control UI CSS churn.

## What This Changes

The PR adds Mermaid rendering to the WebChat/Control UI markdown path by adding a mermaid dependency, rewriting markdown rendering from markdown-it to marked, adding Mermaid copy/fullscreen/render lifecycle code, updating the lockfile, and changing broad Control UI CSS.

## Best Possible Solution

Keep the Mermaid feature alive, but require a narrower Control UI/WebChat patch: preserve the current markdown-it contracts and tests, add Mermaid as a browser-safe lazy rendering path, keep the DOMPurify URL-scheme allowlist and task-list behavior, avoid unrelated CSS churn, use English/i18n-ready UI copy, and add targeted tests for Mermaid fences plus existing sanitizer, task-list, code-block, and non-browser import behavior.

## Reproduction Assessment

Yes. Static reproduction is high confidence: the PR imports a non-exported attachThemeListener, touches window at module load before a browser guard, and removes allowlist entries that current tests assert for task-list checkboxes; current main also clearly renders Mermaid fences as ordinary code blocks.

## Solution Assessment

No. Adding Mermaid support to the shared WebChat/Control UI renderer is the right product surface, but this branch is not the best implementation because it rewrites the parser, removes hardening, adds eager browser globals, and changes unrelated CSS instead of adding a focused lazy Mermaid rendering seam.

## Security Review

Status: needs_attention

Summary: The PR introduces a new Mermaid SVG rendering dependency/path and removes existing markdown sanitizer hardening, so it needs security review before merge.

Concerns:

- **[high] URL scheme allowlist removed:** `ui/src/ui/markdown.ts:183`
  - body: Current main explicitly removes anchor hrefs unless their protocol is http:, https:, or mailto:. The PR removes that hook logic from a user-controlled markdown surface, reducing defense-in-depth around dangerous URL schemes.
  - confidence: 0.9
- **[medium] Mermaid SVG bypasses DOMPurify output sanitization:** `ui/src/ui/markdown.ts:361`
  - body: `renderMermaidInContainer` assigns Mermaid's returned SVG to `innerHTML` after the normal markdown HTML has already been sanitized. Mermaid strict mode helps, but this is a new trusted-SVG boundary that should be reviewed and ideally isolated behind a lazy browser runtime.
  - confidence: 0.82
- **[low] Large new transitive dependency surface:** `pnpm-lock.yaml:1329`
  - body: Adding mermaid brings a large graph of rendering/parser dependencies into the Control UI bundle and lockfile. That may be acceptable for the feature, but it should be intentionally reviewed and kept out of non-browser/runtime import paths.
  - confidence: 0.75
