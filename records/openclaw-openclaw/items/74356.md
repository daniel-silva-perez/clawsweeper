## Summary

Keep open. Current main still has the reported Docker supply-chain gap, and the issue is paired with open PR #74359 using closing syntax, so the right path is maintainer security review of that implementation rather than cleanup closure or a separate repair lane.

## What This Changes

The issue requests replacing the root Dockerfile Bun bootstrap so Docker builds no longer execute an unverified, unpinned `https://bun.sh/install` script while Node base images are digest-pinned.

## Best Possible Solution

Adopt a maintainer-approved Docker Bun install path that avoids executing an unverified remote installer, pins or verifies the Bun source, updates Docker docs and update policy together, and validates the Docker build/smoke path before merge.

## Reproduction Assessment

Yes. The observable condition reproduces by source inspection on current main: `Dockerfile:47` still pipes `https://bun.sh/install` to `bash`; the exploit scenario itself is a supply-chain risk rather than something to trigger in a live test.

## Solution Assessment

Unclear for the exact proposed implementation until maintainer security review. Replacing `curl | bash` is the right direction, but the best fix should settle digest/checksum/signature verification, Dependabot update handling, docs alignment, and multi-arch Docker compatibility.
