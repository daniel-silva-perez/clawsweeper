## Summary

Keep open. Current `main` still has the reported Docker supply-chain gap: the root Dockerfile pipes the unpinned `https://bun.sh/install` response into `bash`, the same Dockerfile pins Node base images by digest, and the Docker docs repeat the unsafe Bun install pattern. PR #74359 is already an open implementation candidate with closing syntax for this issue, so the issue should stay open for maintainer security review rather than being closed or routed to a separate repair lane.

## What This Changes

The issue asks OpenClaw to stop installing Bun in the root Dockerfile via an unverified `curl https://bun.sh/install | bash` pipeline and instead use a pinned or verified Bun source aligned with CI.

## Best Possible Solution

Adopt a maintainer-approved Docker Bun install path that avoids executing an unverified remote installer, pins or verifies the Bun source, updates Docker docs and image update policy in the same change, and validates the root Dockerfile smoke path before merge.

## Reproduction Assessment

Yes. This is reproducible by source inspection on current `main`: `Dockerfile:47` still pipes `https://bun.sh/install` into `bash`, and `docs/install/docker.md:364` repeats the same pattern.

## Solution Assessment

Unclear until maintainer security review. The narrow direction in PR #74359 matches the issue, but maintainers still need to approve the exact Bun source, digest/update policy, and docs alignment before treating it as the best final fix.
