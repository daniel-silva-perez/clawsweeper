## Summary

Keep open. Current main still installs Bun in the root Dockerfile by piping the unpinned `https://bun.sh/install` script to `bash`, while the same build path pins Node images by digest and CI pins Bun to a fixed version. PR #74359 is an open, unmerged implementation candidate with GitHub closing syntax for this issue, so the issue should remain open until maintainers review and land or reject that PR.

## What This Changes

Harden the root Dockerfile by replacing the unverified Bun `curl | bash` install with a pinned or verified Bun source aligned with CI and Docker image pinning.

## Best Possible Solution

Adopt a maintainer-approved Docker Bun install path that avoids executing an unverified remote installer, pins or verifies the Bun source, aligns CI, Docker docs, and any Dependabot/update policy, and validates the root Dockerfile smoke path before merge.
