## Summary

Keep open. Current main still executes the Bun installer from `https://bun.sh/install` through `curl | bash` in the root Dockerfile, while the same Dockerfile pins Node base images by digest and CI pins Bun to a fixed version. The report is valid and overlaps with broader security-scan issue #66684, but this issue is the narrower, more actionable Docker/Bun supply-chain hardening item.

## What This Changes

Request to replace the Dockerfile's unpinned Bun `curl | bash` installer with a pinned and verified Bun source aligned with CI and the Docker image pinning strategy.

## Best Possible Solution

Harden the Docker build path by acquiring Bun from a versioned and verified source, pin any Docker image or artifact digest/checksum according to the repository's supply-chain policy, keep CI and Docker Bun versions aligned, and update Docker docs plus smoke coverage for the changed build flow.
