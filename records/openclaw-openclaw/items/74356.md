Keep open. Current main still contains the reported Docker supply-chain gap, and the issue is paired with open PR #74359 via closing syntax, so it should remain open for maintainer security review rather than being closed or routed to a separate repair lane.

## What This Changes

The issue requests replacing Docker's unverified Bun `curl | bash` installer with a pinned or verified Bun source aligned with CI and the Dockerfile's digest-pinned Node base images.

## Best Possible Solution

Adopt a maintainer-approved Docker Bun install path that avoids executing an unverified remote installer, pins or verifies the Bun source, updates Docker docs and any update policy in the same change, and validates the root Dockerfile smoke path before merge.

## Reproduction Assessment

Yes. The issue is reproducible by static inspection of current main: `Dockerfile` line 47 still executes the unverified remote Bun installer, and the docs example repeats the same pattern.

## Solution Assessment

Unclear. Replacing `curl | bash` is the right direction, but maintainers still need to choose the exact verification mechanism because a tag-only `oven/bun` copy improves the build path but does not fully match the existing digest-pinning policy.
