## Summary

Current main does not implement this PRâ€™s intended usage aggregation fix. The gateway still applies `limit` before loading session usage and building totals/aggregates, channel and agent rankings still sort by cost rather than token-aware zero-cost behavior, and the UI still recomputes overview aggregates from the returned session subset while displaying Top Agents/Top Channels cost-first. This PR should remain open for maintainer review or rework rather than be closed as obsolete.

## Best Possible Solution

Keep this PR open. The best path is to review or rework the PR into a bounded implementation that makes `sessions.usage` aggregates and UI insight denominators use a consistent scope, keeps `limit` semantics intact for returned rows, preserves mixed-version UI fallback behavior, ranks zero-cost channel/agent usage by tokens while preserving cost-first behavior when costs exist, and lands the regression tests for zero-cost ranking and limit-scoped aggregate correctness.
