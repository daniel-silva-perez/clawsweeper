## Summary

This PR should remain open. Current main still does not implement the intended usage aggregation changes: `sessions.usage` aggregates are built from `limitedEntries`, channel/agent rankings remain cost-first, the result type has no overview scope metadata, and the UI still recomputes overview insights from returned sessions while displaying Top Agents/Top Channels cost first. The PR is still useful, but the provided review threads identify unresolved scope, limit, fallback, and I/O concerns that need maintainer review before merge.

## Best Possible Solution

Keep this PR open and rework it into a bounded fix. The best implementation should make `sessions.usage` totals, aggregates, and UI insight denominators use a consistent documented scope; keep `limit` restricted to returned rows; preserve fallback behavior for responses without `overview`; rank zero-cost channel/agent results by tokens while keeping cost-first semantics when cost exists; bound or cache full matched-session summary I/O; and land focused regression tests for zero-cost ranking and limit-scoped aggregate correctness.
