## Summary

Keep this PR open: the current main branch does not preserve OpenAI-compatible summary payload params, and the PR is aimed at the right compaction surface, but the patch has correctness gaps before it should merge. In particular, it stores raw OpenClaw thinking levels for Pi summary calls and resolves summary extra params without the active agent id, so some configured cases would still diverge from normal assistant turns.

## What This Changes

The PR patches `@mariozechner/pi-coding-agent` summary generation to accept an optional payload hook, threads resolved extra params and thinking state into safeguard compaction summaries, refactors OpenAI-compatible extra payload param patching, and updates tests, lockfile metadata, and the changelog.

## Best Possible Solution

Keep the PR open and revise it so safeguard compaction summaries reuse the same effective transport context as normal assistant turns: mapped Pi-compatible thinking levels, active `agentId`, workspace and agent directories, provider hooks, and focused regression tests for global, model, and per-agent `chat_template_kwargs`/`extra_body` cases.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is available: set safeguard compaction with an OpenAI-compatible model and `/think max` or `adaptive`, then trigger built-in summary fallback; the PR passes that raw value into `SimpleStreamOptions.reasoning`, which the dependency type does not accept. A second source-level repro is `agents.list[].params.chat_template_kwargs`, where normal streams resolve with `agentId` but the proposed summary resolver does not.

## Solution Assessment

No as written. The dependency patch and shared payload helper are a reasonable direction, but the summary runtime should first match the normal assistant path's mapped thinking and full params resolution contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Map thinking before passing it to Pi summaries:** `src/agents/pi-embedded-runner/extensions.ts:168`
  - body: `params.thinkingLevel` is an OpenClaw `ThinkLevel`, which can be `max` or `adaptive`. The patched Pi summary path forwards this as `SimpleStreamOptions.reasoning`, whose dependency type only accepts `minimal`, `low`, `medium`, `high`, or `xhigh`; normal session creation already maps `max -> xhigh` and `adaptive -> medium`. With `/think max` or `adaptive`, safeguard compaction can now send an invalid reasoning value and fail the summary request. Map before storing or before calling `summarizeInStages()`.
  - confidence: 0.9
- **[P2] Pass the active agent id into summary extra params:** `src/agents/pi-embedded-runner/extensions.ts:169-177`
  - body: This call resolves summary extra params without an `agentId`, so it only sees defaults/model params and skips `agents.list[].params`. Normal assistant streams pass `sessionAgentId` into the same resolver, and the documented merge order makes matching agent params the final override. A user who puts `chat_template_kwargs` or `extra_body` on a specific agent will still get different compaction summary payloads from normal turns. Thread `sessionAgentId` through the factory and include it here.
  - confidence: 0.84
