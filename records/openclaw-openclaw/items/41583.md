## Summary

Keep this issue open. Current main still injects inbound channel metadata into both system prompt context and user-role prompt text, and the shared, Telegram, and Discord config surfaces still do not expose a disable/minimize toggle for trusted DM conversations. The related open PR #41299 is a separator/stripping proposal and explicitly leaves the requested config/API surface out of scope.

## Best Possible Solution

Keep this open for maintainer product/security judgment. The best implementation would be a documented shared reply-runtime option that defaults metadata injection on, supports narrow per-channel/account or trusted-DM scoping, applies consistently to Telegram and Discord, and adds tests for both existing metadata-on behavior and the opt-in minimized/disabled path. Review #41299 separately as a separator/stripping improvement, not as the implementation for this configuration request.
