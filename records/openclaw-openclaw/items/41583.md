## Summary

Keep this issue open. Current main still injects inbound channel metadata into agent runs and user-role prompt text without a shared, Telegram, or Discord config toggle to disable or minimize it for trusted DM conversations. The related open PR #41299 is a separator/stripping proposal and explicitly does not add the requested configuration API.

## Best Possible Solution

Keep this open for maintainer product/security judgment. The best path is a documented shared reply-runtime option that defaults metadata injection on, supports narrow per-channel/account or trusted-DM scoping, covers Telegram and Discord, and adds tests for both existing metadata-on behavior and the opt-in minimized/disabled path. Review #41299 separately as a separator/stripping improvement, not as the implementation for this configuration request.
