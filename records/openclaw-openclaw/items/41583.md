## Summary

Keep open. Current main still injects channel inbound metadata into both the trusted system prompt and user-role metadata prefix without a config option to disable or minimize it for trusted DMs. Existing access controls and contextVisibility settings do not cover this core metadata envelope, and related PR #41299 is a separator/stripper proposal rather than a toggle.

## Best Possible Solution

Keep this open for maintainer product/security judgment. The likely best implementation is a documented shared reply-runtime option, defaulting metadata injection to enabled, with narrow per-channel/account or trusted-DM scoping, Telegram and Discord coverage, and docs explaining the provenance tradeoff.
