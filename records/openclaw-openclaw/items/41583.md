## Summary

Keep this issue open. Current main still injects inbound channel metadata into the system prompt and user-role prompt text, and the shared, Telegram, and Discord config surfaces still do not expose a disable or minimize mode for trusted DM conversations. The related open PR #41299 is a separator/stripping proposal and explicitly leaves API/config changes out of scope, so it does not resolve the requested toggle.

## Best Possible Solution

Keep this open for maintainer product/security design. The best implementation is a documented shared reply-runtime option that defaults metadata injection on, supports narrow per-channel/account or trusted-DM scoping, applies consistently to both the system metadata prompt and user-role metadata prefix, and adds regression tests for metadata-on behavior plus the opt-in minimized/disabled path. Review #41299 separately as a separator/stripping improvement, not as this feature.
