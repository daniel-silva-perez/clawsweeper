## Summary

Keep this issue open. Current main still unconditionally builds inbound channel metadata for the agent run, including the trusted system metadata block and user-role untrusted metadata prefix, and the shared, Telegram, and Discord config surfaces do not expose the requested disable/minimize toggle for trusted DM conversations. Related PR #41299 is an open separator/stripping proposal and does not implement the requested config API.

## Best Possible Solution

Keep this open for maintainer product/security judgment. The best path is a documented shared reply-runtime option that defaults metadata injection on, supports narrow per-channel/account or trusted-DM scoping, covers Telegram and Discord, and adds tests for both existing metadata-on behavior and the opt-in minimized/disabled path.
