## Summary

Keep this issue open. Current main has a sandbox skill-copy mechanism, which matches the related #46257 cleanup, but the actual run path still prefers a prebuilt skills snapshot prompt. For sandboxed cron/non-main runs, that snapshot is built from the agent workspace before sandbox resolution and can still contain managed/global skill locations instead of the copied sandbox-local `skills/.../SKILL.md` path.

## Best Possible Solution

Keep open and fix the core skills/sandbox integration: for sandboxed non-`rw` runs, rebuild or remap `skillsSnapshot.prompt` after `resolveSandboxContext`/`syncSkillsToWorkspace` so `<available_skills>` locations point at sandbox-readable `skills/<name>/SKILL.md`. Add regression coverage for a managed `~/.openclaw/skills` skill in a non-main or isolated cron sandbox session, including the snapshot path case.
