## Summary

Keep open. Current main copies eligible skills into the sandbox workspace for non-rw sandbox access, but embedded runs can still reuse a host-built skills snapshot whose prompt advertises the original managed/global skill path, so the reported sandbox read failure is still plausible. The open prompt-guard PR is adjacent but does not fix the runtime remap contract.

## What This Changes

Ensure sandboxed non-main skill resolution advertises sandbox-local SKILL.md copies instead of host/global managed skill paths.

## Best Possible Solution

Sandboxed runs with workspaceAccess none or ro should build the prompt from sandbox-readable skill copies, or remap snapshot prompt locations to those copies, while keeping canonical resolvedSkills available for host-side config and env handling. Add regression coverage for a managed ~/.openclaw/skills/<name>/SKILL.md skill under a non-main sandboxed session.
