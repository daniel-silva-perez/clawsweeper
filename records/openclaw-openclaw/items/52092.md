## Summary

Keep this issue open. Current main has meaningful cron and session-store mitigations that reduce the original catch-up/backlog pressure, but it still does not implement the reported Gateway service heap-headroom/config fix. The related open PR #42853 remains the closest implementation candidate, while #55334 is the broader session-store bloat tracker rather than a complete replacement for this cron-triggered OOM report.

## Best Possible Solution

Keep this issue open as the cron-triggered Gateway OOM tracker. The best maintainer path is to review and land a narrow heap-headroom/config solution such as a redesigned #42853 or equivalent, preserve the current cron concurrency/catch-up/session-maintenance safeguards, and coordinate any broader session-store payload deduplication with #55334.
