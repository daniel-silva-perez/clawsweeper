## Summary

Keep this issue open. Current main has meaningful cron safeguards for concurrency, startup catch-up, and cron-run cleanup, but it still does not implement the reported heap-headroom/config fix for installed Gateway services. Related PR #42853 remains a plausible implementation candidate, while #55334 is the broader session-store bloat tracker rather than a full replacement for this cron-triggered OOM report.

## Best Possible Solution

Keep this issue open as the cron-triggered Gateway OOM tracker. The best maintainer path is to land a reviewed heap-headroom/config solution such as a redesigned #42853 or equivalent, preserve the current cron concurrency/catch-up/reaper safeguards, and coordinate with #55334 for structural session-store payload deduplication so retained cron session rows do not carry large repeated skill/system-prompt snapshots.
