## Summary

Keep this issue open. Current main still resolves Telegram native approval prompts by clearing inline actions rather than deleting the approval message, and open PR #56319 remains the active unmerged implementation path with closing syntax for this bug.

## What This Changes

The issue asks for Telegram native exec approval prompt messages to disappear from chat after an approver clicks approve.

## Best Possible Solution

Review and either land PR #56319 or replace it with a narrow current-main Telegram plugin change that removes inline actions immediately and then deletes the pending native approval message through the existing final-action/delete-message seam, with focused regression coverage in the Telegram approval runtime tests.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists on current main: resolving a Telegram native approval returns `clear-actions`, and that path only edits reply markup, leaving the message in chat.

## Solution Assessment

Yes for keeping this issue open. The narrow maintainable fix belongs in the Telegram plugin approval runtime using the existing shared final-action seam and Telegram delete helper, not in a core migration or unrelated channel behavior.
