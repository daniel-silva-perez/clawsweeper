## Summary

Keep this open. Current main still finalizes Telegram native approval prompts by clearing inline actions rather than deleting the prompt message, and open PR #56319 is still the active unmerged implementation path with closing syntax for this bug.

## What This Changes

The issue asks for Telegram native exec approval prompt messages to disappear after an approver clicks approve.

## Best Possible Solution

Land or replace PR #56319 with a narrow current-main Telegram plugin fix that removes inline actions immediately and then deletes the pending native approval message through the existing delete-message/final-action seam, with focused regression coverage in the Telegram approval runtime tests.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists: a resolved Telegram native approval currently returns `clear-actions`, which calls `editMessageReplyMarkupTelegram` and leaves the message in chat.

## Solution Assessment

Yes for keeping this issue open until a fix merges. The narrow maintainable fix belongs in `extensions/telegram/src/approval-handler.runtime.ts` using existing Telegram deletion support, not in a core migration or unrelated channel behavior.
