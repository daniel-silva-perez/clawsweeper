## Summary

Keep open. Current main fixes the silent no-selector agent path in both gateway and embedded/local execution, but the report still has unresolved surface-area questions: several docs still show no-selector `openclaw agent --message` examples, and Lobster plus llm-task are still registered as optional agent tools rather than root CLI commands. The headless discovery-delay symptom also was not reproduced or ruled out by current source inspection.

## What This Changes

The issue requests regression cleanup for `openclaw agent` silently exiting with no output, missing `openclaw lobster` and `openclaw llm-task` CLI command handling, and slow gateway discovery on headless Ubuntu.

## Best Possible Solution

Split or resolve the item along the actual boundaries: keep the current explicit session-selector runtime guard, update stale no-selector docs, and make an explicit maintainer decision on whether Lobster and llm-task should gain plugin-owned root CLI commands or be documented only as optional agent tools invoked through `openclaw.invoke`. If the discovery delay still matters, capture a focused headless Linux reproduction with command timing, config, and gateway URL mode.

## Reproduction Assessment

No, not end-to-end. Source inspection gives high-confidence static reproduction for the docs/CLI-registration mismatch, but the exact `openclaw agent` silent-exit behavior is fixed in current source and the 30s headless discovery path still needs a live environment trace.

## Solution Assessment

Unclear for the whole bundled report. The selector guard is the right narrow runtime fix for the no-output path, but the Lobster/llm-task root-command expectation needs product/docs ownership before a code change is obviously correct.
