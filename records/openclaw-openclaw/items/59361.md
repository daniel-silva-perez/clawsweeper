## Summary

Keep open. Current main addresses part of the original symptom by requiring a session selector in both gateway and embedded `openclaw agent` paths, but the issue also reports bundled Lobster/llm-task CLI commands as missing. Current main still registers both as optional agent tools, not root CLI commands, and one public doc still shows `openclaw agent --message` without the selector that the source requires.

## Best Possible Solution

Keep this issue open, or split it into narrower follow-ups: one docs/runtime cleanup to make `openclaw agent` no-selector behavior and docs agree, and one maintainer decision on whether bundled Lobster/llm-task should expose plugin-owned root CLI commands or whether docs should explicitly direct users to the agent tool / `openclaw.invoke` flow instead.
