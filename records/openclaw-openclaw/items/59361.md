## Summary

Keep open. Current main appears to fix the no-selector `openclaw agent` silent path by throwing in both gateway and embedded/local execution, but the issue also reports missing `openclaw lobster` and `openclaw llm-task` root CLI commands. Current main still registers both Lobster and llm-task as optional agent tools, not CLI commands, and one agent-send doc still shows a no-selector quickstart that conflicts with the runtime guard.

## Best Possible Solution

Keep this issue open, or split it into focused follow-ups: update the agent-send docs so every example includes a selector, decide whether Lobster and llm-task should expose plugin-owned root CLI commands via `api.registerCli` descriptors or whether docs should explicitly route users to the optional tool and `openclaw.invoke` flow, and separately repro the headless discovery delay if it is still observed on current main.
