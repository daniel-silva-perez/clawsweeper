## Summary

Keep open. Current main and the latest release still do not register or implement `skills.uninstall`, so this PR is not obsolete. The proposed PR is useful, but it needs rework before merge because the backend can delete an existing workspace skill directory without confirming ClawHub provenance, and the handler should be rebased onto the current Gateway runtime-config pattern.

## Best Possible Solution

Keep this PR open for contributor or maintainer rework. The best path is a narrow `skills.uninstall` Gateway RPC that only removes ClawHub-proven skill folders or lockfile-tracked entries, treats invalid slugs as `INVALID_REQUEST`, uses current `context.getRuntimeConfig()`/config mutation patterns, and keeps protocol schemas, docs, generated clients, method scopes, and focused backend/Gateway tests aligned. #63272 can continue tracking the separate CLI uninstall design.
