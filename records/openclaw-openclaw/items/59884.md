## Summary

Keep open. Current main still hard-requires `modelRegistry.getApiKeyAndHeaders` in the compaction safeguard LLM path, and there is no provider-level or legacy `getApiKey` fallback on main. The PR's central compatibility fix is therefore not implemented or obsolete.

## Best Possible Solution

Keep this PR open and review a rebased, current version of the compaction auth resolver. The likely best path is a small helper in `src/agents/pi-hooks/compaction-safeguard.ts` that keeps `getApiKeyAndHeaders` as the preferred path, falls through to provider-level and legacy model-level key lookup only when appropriate, and keeps the strengthened regression tests that prove successful compaction rather than silent cancellation. Drop the already-present Swift formatting diff during rebase, then run the targeted compaction safeguard test and the appropriate changed gate.
