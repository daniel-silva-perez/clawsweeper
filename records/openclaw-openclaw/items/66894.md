## Summary

Keep this PR open. The linked `/compact` cancellation bug still exists on current main, and this PR contains useful direction, but it is not merge-ready: the branch is stale against the current compaction code, leaves harness-owned compaction outside the new active-run handle path, and bundles an unrelated launchd/profile behavior change that can break explicit custom service labels.

## What This Changes

The PR exports embedded active-run helpers, registers queued embedded compaction as an abortable active run with cascaded reply abort, adds a drain guard before manual `/compact`, and changes CLI profile handling/tests for `OPENCLAW_LAUNCHD_LABEL`.

## Best Possible Solution

Land a focused compact cancellation fix that covers both queued Pi/context-engine compaction and harness-owned compaction, registers the active-run handle at the correct lifecycle point, forwards cancellation into the running operation, cascades any same-session reply abort as needed, and includes targeted cancellation regression coverage. Split the launchd/profile label semantics into a separate maintainer-reviewed change for #65643.
