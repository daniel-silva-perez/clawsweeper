## Summary

Keep open. Current main has moved node registration away from the static macOS client ID when a signed device identity is present, but the central reported WebChat behavior is not implemented as a per-Mac/session-isolated default. The macOS chat UI still opens the gateway main session, the picker lists gateway sessions rather than device-scoped sessions, and the docs describe the macOS WebChat default as the main session for the selected agent.

## Best Possible Solution

Keep this issue open for maintainer product judgment and implementation. The likely fix belongs in the macOS WebChat/Gateway session contract: either make the macOS app select a device-specific default session keyed by signed device/node identity, and filter/label the picker accordingly, or explicitly decide/document that WebChat is gateway-main by design and adjust the issue with that decision.
