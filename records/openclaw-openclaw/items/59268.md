## Summary

Keep open: current main still routes the macOS native chat default through the gateway-wide main session, so two paired Macs on the same gateway can still land on the same chat history. The static `openclaw-macos` client id is not the only current identity source because node registration already prefers the signed device id, but the chat picker/default session path is not device-scoped.

## What This Changes

The issue requests that each paired macOS app default to and isolate its own node chat session instead of showing another Mac's label or sharing history.

## Best Possible Solution

Introduce a device-scoped native chat default for the macOS app, likely based on the signed device identity/node id and the existing `node-<nodeId>` session convention, while keeping the global main-session alias behavior intact for other clients and explicit user-selected sessions.

## Reproduction Assessment

Yes: the code-level reproduction is high confidence. Two macOS app instances connected to the same gateway will both resolve the native chat default from the same gateway `sessionDefaults.mainSessionKey`, then `chat.history` and `chat.send` canonicalize to that shared key.

## Solution Assessment

No: current main is not the best solution for the reported multi-Mac node-chat behavior. The safer fix is to scope the macOS app's default node chat session to the current device identity/node id rather than changing the static product `clientId` or globally redefining the gateway main session.

## Repair Work Prompt

Fix the macOS native chat default-session collision for multiple Macs paired to the same gateway. Current main opens chat via `WebChatManager.preferredSessionKey()` -> `GatewayConnection.shared.mainSessionKey()`, and the picker prioritizes the gateway-wide main session. Make each macOS app default to a device/node-scoped chat session for its own signed device identity, likely following the existing `node-<nodeId>` convention used by node-originated events. Preserve explicit session switching, preserve global main-session aliases for non-macOS/non-node defaults, and do not treat the static `openclaw-macos` client id as unique. Add focused Swift tests for the preferred session and picker/default behavior; update user-facing docs/changelog if behavior changes.
