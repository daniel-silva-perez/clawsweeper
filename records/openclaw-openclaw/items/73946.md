## Summary

Keep this PR open. Current main still has the reported cron normalization gap: implicit `payload.text` and legacy top-level `text` are classified as `systemEvent` before agent-turn-only fields such as `model`, `thinking`, `fallbacks`, or `toolsAllow` can make the payload an `agentTurn`, and the `systemEvent` cleanup then prunes those fields. The proposed PR is a narrow repair of #64060/#28905, with no visible workflow, dependency, lockfile, package publishing, or secret-handling changes in the provided diff.

## Best Possible Solution

Keep this PR open and review it as the focused fix: promote implicit text payloads with agent-turn-only hints to `agentTurn` in both `coercePayload()` and `inferTopLevelPayload()`, preserve explicit `systemEvent` pruning, keep #66543 out of scope, retain the #64060/#28905 credit trail in the changelog, and land after focused cron tests plus the changed gate are green.
