## Summary

Keep this PR open. Current main still classifies implicit cron text payloads as systemEvent before checking agent-turn-only fields, so text payloads with model/fallback/thinking/tool overrides can still have those fields pruned. The PR is a narrow repair of the #64060/#28905 behavior and the visible diff scope is limited to cron normalization, focused tests, and changelog credit.

## Best Possible Solution

Review and land this focused PR, or an equivalent patch, that promotes implicit text payloads with agent-turn-only hints to agentTurn in both coercePayload() and inferTopLevelPayload(), keeps explicit systemEvent pruning unchanged, credits the #64060/#28905 trail in the changelog, and leaves the separate #66543 whitespace-alias decision out of scope.
