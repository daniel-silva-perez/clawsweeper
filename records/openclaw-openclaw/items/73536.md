## Summary

Keep this PR open. Current main still invokes bundled MCP `callTool` without a per-request timeout, so the intended fix for #60967 is not implemented on main. The PR remains a plausible implementation candidate, but it still needs maintainer review for timeout default semantics and a focused regression test.

## Best Possible Solution

Keep this PR open as the implementation candidate for #60967. The best path is a narrow change in `src/agents/pi-bundle-mcp-runtime.ts` that passes a per-call timeout via the third `Client.callTool` argument while preserving the SDK’s 60-second default for unconfigured servers, or introduces a separate documented tool/request timeout config. Add a focused test that captures the third SDK argument, update the docs/changelog semantics, and fix the JSDoc config-key typo before landing.
