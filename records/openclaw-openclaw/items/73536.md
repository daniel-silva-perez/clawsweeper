## Summary

Keep this PR open. Current main still calls bundled MCP tools without a per-request timeout, so the intended fix is not implemented on main. The PR is a plausible implementation candidate for #60967, but it still needs maintainer review for timeout-default semantics and a focused regression test.

## Best Possible Solution

Keep this PR open as the implementation candidate for #60967. The best path is a narrow change in `src/agents/pi-bundle-mcp-runtime.ts` that passes a request timeout through the third `Client.callTool` argument while preserving the SDK's 60-second default for unconfigured servers, or adds a separate documented request/tool timeout field. Add a focused test that captures the third SDK argument, then land this PR or a maintainer-authored equivalent.
