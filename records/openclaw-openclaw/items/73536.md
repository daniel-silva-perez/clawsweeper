## Summary

Keep this PR open. Current main still does not pass a per-request timeout to bundled MCP Client.callTool calls, so the #60967 bug is not implemented on main. The PR remains a plausible focused fix, but the current patch can regress unconfigured servers from the MCP SDK's 60s request default to OpenClaw's 30s connection default, has a wrong config-key comment, and lacks a focused test for the third callTool argument.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #60967, but update it before merge: preserve the SDK's 60s request default for unconfigured servers or add a dedicated documented request/tool timeout field, fix the JSDoc config key to connectionTimeoutMs, add a focused test that proves Client.callTool receives the expected RequestOptions.timeout, and include the user-facing changelog/docs update if the config semantics change.
