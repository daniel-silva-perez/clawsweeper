## Summary

Keep open. Current main still does not implement per-agent session maintenance overrides, and this PR remains a plausible implementation candidate rather than obsolete cleanup. The closed tracking issue #52775 is not enough to close this PR because the requested behavior is absent from main.

## Best Possible Solution

Keep this PR open as the implementation candidate. The best path is to finish the product surface by adding `agents.list[].maintenance` schema/type support, resolving overrides from one loaded config snapshot with normalized agent IDs, defining safe behavior for shared stores and explicit `--store` cleanup, threading the resolved policy through normal write/load and cleanup paths, and adding regression tests plus docs/schema updates. If maintainers decide per-agent session maintenance should not be a core feature, close the PR manually with that product decision rather than as automated cleanup.

## Risks / Open Questions

- The PR changes session cleanup and retention behavior that can delete stored sessions or transcripts, so shared stores, explicit `--store` targets, warn/enforce mode, and legacy `pruneDays` merges need focused review before merge.
