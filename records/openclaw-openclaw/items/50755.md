## Summary

Keep open. Current main still does not implement this PR's intended per-agent session maintenance overrides, and this PR remains a plausible implementation candidate rather than obsolete cleanup. The feature is absent from the agent config schema/type, maintenance resolver, regular session write/load paths, and cleanup command policy selection on current main. The closed tracking issue #52775 does not supersede this PR because the requested behavior is still missing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either finish this PR or supersede it with a maintainer-approved implementation that adds agents.list[].maintenance schema/type/generated-schema/docs support, resolves per-agent overrides from one normalized config snapshot, threads the chosen policy through normal session write/load and cleanup paths, and explicitly handles shared stores, explicit --store targets, legacy pruneDays precedence, and agent-id normalization before merge.
