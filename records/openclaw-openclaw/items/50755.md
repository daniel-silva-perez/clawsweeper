## Summary

Keep open. Current main still does not implement per-agent session maintenance overrides: `agents.list[]` does not accept a `maintenance` field, the public agent type has no maintenance override, maintenance resolution remains global-only, and cleanup/store write paths still apply global `session.maintenance`. The closed tracking issue #52775 does not supersede this PR because the requested behavior is still absent on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either finish this PR or supersede it with a maintainer-approved implementation that adds `agents.list[].maintenance` schema/type/generated-schema/docs support, resolves overrides from one normalized config snapshot, handles shared stores and explicit `--store` cleanup safely, threads the selected policy through normal write/load and cleanup paths, and adds regression tests for destructive cleanup behavior.
