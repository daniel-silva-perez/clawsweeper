## Summary

Keep open. Current main still does not implement this PR's Feishu channel/account `mediaLocalRoots` config surface or account-config fallback, and the PR head needs rework for the current local-media safety contract before it would be merge-ready.

## Best Possible Solution

Keep this PR open for maintainer follow-up or a superseding Feishu-owned patch. The right fix should define whether configured roots merge with, override, or are ignored when runtime-scoped roots exist; route any supported `"any"` mode through the current `mediaAccess`/`mediaReadFile` safety path; update schema, generated metadata, and the public Feishu docs; and add regression tests for configured array roots, `"any"`, and ordinary runtime-scoped outbound media sends.

## Risks / Open Questions

- Merging the PR as-is could leave `channels.feishu.mediaLocalRoots` ineffective in normal outbound sends because runtime-injected roots take precedence over the account config fallback.
- The proposed `"any"` forwarding conflicts with the current `loadWebMedia` host-read guard unless Feishu also threads an explicit `mediaReadFile` capability or chooses a narrower config contract.
- The PR adds no regression tests for configured array roots, configured `"any"`, or precedence between configured roots and runtime-scoped roots.
- Docs and generated bundled-channel metadata would remain out of sync if the new config field is accepted without updating those surfaces.
