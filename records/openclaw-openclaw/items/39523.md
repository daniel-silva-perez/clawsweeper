## Summary

Keep open. Current main still does not implement the PR's requested Feishu `mediaLocalRoots` channel/account config surface or account-config fallback. Main does pass runtime-scoped roots into Feishu media sends, but that is separate from the config behavior proposed here, and the submitted patch is not merge-ready against the current `localRoots: "any"` and `readFile` safety contract.

## Best Possible Solution

Keep this PR open, but require a revised Feishu-owned patch or superseding PR. The fix should add the Feishu top-level/account schema, generated metadata, and public docs only if this remains the desired config contract; define how configured roots interact with injected runtime roots; route any supported `"any"` mode through the current `mediaAccess`/`mediaReadFile` safety path; and add regression tests for configured arrays, `"any"`, and normal runtime-scoped outbound media sends.
