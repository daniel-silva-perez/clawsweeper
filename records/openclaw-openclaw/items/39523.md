## Summary

Keep open. Current main still does not implement this PR's Feishu channel/account mediaLocalRoots config surface or config fallback, and the proposed direct "any" forwarding is not compatible with the current local-media safety contract without a readFile/host-read path.

## Best Possible Solution

Keep this PR open for maintainer follow-up or replace it with a Feishu-owned patch that adds mediaLocalRoots to the Feishu top-level and account schemas, updates generated config metadata and docs, defines precedence against runtime-scoped mediaAccess roots, routes any supported "any" mode through the current readFile/host-read safety path, and adds regression tests for array config fallback, account config fallback, "any", and ordinary runtime-scoped local media sends.
