## Summary

Keep open. Current main does not implement this PR's one-line `emitter.off("error", onGatewayError)` change, and the current Discord supervisor tests now assert the opposite behavior: late gateway `error` events after `dispose()` must still be consumed and logged instead of becoming uncaught EventEmitter errors. The Greptile comment treats the patch as safe, but the existing review concern is supported by current code, tests, and Node EventEmitter behavior.

## Best Possible Solution

Keep this PR open for maintainer review, but do not land the current one-line patch as-is. The best fix is to reduce retained supervisor state after dispose while preserving a minimal late-error sink for the gateway emitter, with a regression test covering post-dispose `error` events and listener cleanup behavior.
