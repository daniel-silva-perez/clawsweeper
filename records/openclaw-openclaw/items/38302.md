## Summary

Keep open. Current main still lacks a validated per-account Slack native command prefix or native-name mapping: Slack slash-command config exposes only enabled/name/sessionPrefix/ephemeral, the strict schema and generated metadata reject extra account-level fields, and Slack native registration/dispatch still call provider-scoped native command helpers with only provider: "slack".

## Best Possible Solution

Keep this open and implement the feature in the bundled Slack plugin plus the shared native-command contract. A maintainer-grade fix should add a validated account-level Slack native naming option, thread it through native spec generation, Slack registration, argument-menu values, and native dispatch lookup, preserve current unprefixed behavior and /status to /agentstatus, update Slack/slash-command docs and generated config metadata, and add focused multi-account tests proving prefixed or mapped native commands route to the intended built-ins.
