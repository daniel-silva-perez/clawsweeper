## Summary

Current main still does not implement per-account Slack native command prefixes or native-name mappings. The Slack slash-command config, strict schema, generated metadata, native command registry, Slack registration, and Slack dispatch paths all remain provider-scoped rather than account-scoped. The two linked implementation attempts, #38326 and #38391, are closed and unmerged, and another user confirmed the same multi-app Slack collision in the discussion.

## Best Possible Solution

Keep this issue open and implement the feature in the bundled Slack plugin plus the shared native-command contract. A maintainer-grade fix should add a validated account-level Slack native naming option, thread it through native spec generation, Slack registration, argument-menu values, and native dispatch lookup, preserve current unprefixed behavior and /status -> /agentstatus, update Slack/slash-command docs, and add focused multi-account tests proving prefixed or mapped native commands route to the intended built-in commands.
