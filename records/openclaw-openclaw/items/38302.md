## Summary

Current main still lacks the requested per-account Slack native command prefix or native-name mapping. The Slack account slashCommand config, schema, and generated metadata expose only enabled/name/sessionPrefix/ephemeral; native command generation and lookup remain provider-scoped; and Slack registration/dispatch does not pass account-specific naming data. The linked implementation attempts #38326 and #38391 are closed and unmerged, while the issue discussion includes another user confirming the same multi-app Slack collision.

## Best Possible Solution

Keep this issue open and implement the feature in the bundled Slack plugin plus the native-command contract. A maintainer-grade fix should add a validated account-level Slack native naming option, thread the resolved account naming through native spec generation, Slack registration, argument-menu values, and native dispatch lookup, preserve current unprefixed behavior and the status -> agentstatus override, update Slack setup docs, and add focused multi-account tests proving prefixed or mapped native commands route to the intended built-in commands.
