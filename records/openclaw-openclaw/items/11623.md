## Summary

Keep this issue open. Current main still has adjacent macOS status, HoverHUD, WebChat, Canvas, Talk, and session-menu surfaces, but it does not implement configurable floating per-agent bubbles with persisted positions, drag/anchor or mouse-follow behavior, and per-agent chat launch/routing. The prior issue discussion already reached the same keep-open conclusion, and the current checkout at ef58307f843eed1a7832e9c23c519591f1b7cdbd still matches it.

## Best Possible Solution

Keep this issue open. The best path is either a native macOS implementation under apps/macos/Sources/OpenClaw that derives bubble state from agent/session activity, persists positions, supports drag/anchor or mouse-follow behavior, and routes chat to the selected agent, or a maintainer-designed native macOS UI/window plugin API that would make an external ClawHub implementation feasible.
