## Summary

Current main still has the memoryFlush compaction-counter bug this PR targets: after a flush-triggered compaction, the runner assigns the post-increment count back to `memoryFlushCompactionCount`, while the dedup helper skips flushes when that value equals `compactionCount`. The PR should stay open for maintainer review/landing rather than be closed as obsolete.

## Best Possible Solution

Keep this PR open and land this narrow fix, or an equivalent maintainer patch, after required CI is green or explicitly accepted. The right boundary is to preserve the pre-increment `memoryFlushCompactionCount`, update the persisted-counter expectation from `2` to `1`, keep the consecutive-cycle regression test, and let #12590 close through the normal merge path.
