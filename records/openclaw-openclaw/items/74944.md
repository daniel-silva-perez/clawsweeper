## Summary

Keep open. Current main still has the reported updater reliability gap: global package updates can invoke an absolute npm shim while passing through a launchd-minimal PATH that does not include the active Node or Homebrew bin directories.

## What This Changes

The issue requests hardening the macOS launchd self-update path so Homebrew-installed npm shims can resolve Node under launchd’s minimal environment.

## Best Possible Solution

Centralize update subprocess PATH construction so global package-manager updates and any launchd handoff include the active runtime directory, the owning package-manager prefix bin, Homebrew Node/bin locations, and safe system directories without trusting project-local bins or weakening service PATH hardening.

Yes. The report gives a concrete Homebrew global npm install, launchd default PATH, log signature, and npm exit code; current code inspection shows that minimal PATH can still reach the global npm update step unchanged. I did not execute launchd locally in this read-only Linux review.

## Solution Assessment

Unclear for the report’s one-line script export as the final fix. The maintainable direction is a shared update-environment helper used by both update.run/global package updates and any launchd handoff script, with regression coverage for npm shims resolving Homebrew Node.

## Repair Work Prompt

Fix the macOS Homebrew launchd self-update failure reported at https://github.com/openclaw/openclaw/issues/74944. Current main can invoke the owning npm binary by absolute path, but createGlobalInstallEnv does not add the active Node directory or Homebrew Node/bin paths, so npm shims using `#!/usr/bin/env node` fail under launchd's `/usr/bin:/bin:/usr/sbin:/sbin` PATH. Implement a shared update subprocess PATH helper for global package-manager updates and any launchd update handoff script if one still exists. Keep service PATH hardening intact: do not trust project-local `node_modules/.bin`, and keep user/package-manager directories ordered so they do not shadow immutable system dirs unless the active runtime/owning package-manager binary requires it. Add focused tests modeling a Homebrew global npm install with minimal launchd PATH, including the update.run/global path; update docs/changelog only if repo policy expects a user-facing updater fix entry, with proper reporter attribution. Do not change release/version metadata or unrelated daemon behavior.
