## Summary

Keep open. Current main still has only partial mitigations for this report: `update.run` avoids restarting after failed updates and the gateway/service startup paths have PATH bootstrapping, but the global update environment still does not add the active Node/Homebrew Node directory for a launchd-submitted npm shim running under a minimal PATH, and no regression test covers that exact failure mode.

## What This Changes

The issue asks OpenClaw to make the macOS launchd self-update path resolve Homebrew-installed Node when npm is launched from launchd's minimal PATH.

## Best Possible Solution

Add a centralized update-command environment helper used by global package updates and any macOS launchd handoff so npm shims can resolve the active Node/Homebrew Node path from a minimal launchd environment, with tests for Homebrew npm under minimal PATH and failed-update restart suppression.

## Reproduction Assessment

Yes. The report gives a high-confidence macOS Homebrew LaunchAgent reproduction path and exact artifacts to inspect; I could not live-run launchd from this Linux checkout, but current source still matches the described PATH failure mode.

## Solution Assessment

Unclear. The proposed `export PATH=...` would likely fix the observed job, but the maintainable solution should centralize update PATH construction across `update.run`, CLI update, and any launchd handoff script without weakening existing PATH hardening.
