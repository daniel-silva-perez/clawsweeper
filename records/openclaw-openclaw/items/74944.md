## Summary

Keep open. The report is fresh, concrete, unprotected, and still points at a narrow updater reliability gap: current main has PATH bootstrap pieces, but the global package update path does not have a dedicated, tested update-subprocess environment contract that guarantees Homebrew or active Node paths for npm shims under launchd-style minimal PATH.

## What This Changes

Fix the macOS self-update path so launchd/global npm updates on Homebrew installs can resolve Node when npm's `env node` shebang runs under launchd's minimal PATH.

## Best Possible Solution

Add a shared update-subprocess PATH helper used by global package updates from both Gateway `update.run` and CLI `openclaw update`, including the active runtime dir, safe system dirs, owning npm prefix bin, and Homebrew Node/bin locations without trusting project-local bins.

## Reproduction Assessment

Yes. A Homebrew global npm install whose update subprocess runs with launchd PATH `/usr/bin:/bin:/usr/sbin:/sbin` and invokes `/opt/homebrew/bin/npm` can reproduce the npm shim `env node` failure described in the report.

## Solution Assessment

No, the suggested hard-coded shell `export PATH=...` is only a workaround. The maintainable fix is a shared, tested update env helper wired into the global install subprocess path.

## Repair Work Prompt

Fix the macOS Homebrew launchd update failure reported in https://github.com/openclaw/openclaw/issues/74944. The observable bug is that global npm updates can invoke an absolute Homebrew npm shim while the subprocess PATH is launchd-minimal, causing `env node` to fail with exit 127. Implement a shared update-subprocess PATH/env helper rather than a one-off shell export. Wire it into global package updates used by Gateway `update.run` and CLI `openclaw update`; include the active runtime directory, safe system dirs, owning npm prefix bin when known, and Homebrew Node/bin locations. Do not trust project-local `node_modules/.bin` by default, do not weaken service PATH hardening, and do not resurrect the old launchctl-submit helper. Add focused regression coverage for a Homebrew npm prefix with PATH `/usr/bin:/bin:/usr/sbin:/sbin`. Add a user-facing CHANGELOG.md fix entry crediting `Thanks @jrsdhr`.
