## Summary

Keep this PR open. Current main has absorbed several Telegram fixes, but the intended batch is not fully implemented in the current `extensions/telegram` plugin layout, and direct merge would be unsafe because the PR diff targets older `src/telegram/*` paths plus approval and retry paths that need targeted review.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but do not merge it as one batch. Split and port the still-relevant fixes into `extensions/telegram`: command-menu config hash invalidation, getFile failure surfacing if maintainers want that behavior, mention-gating verbose logging, broader media-group per-item skip handling, and explicit-network fallback behavior. Preserve current main’s newer handling for reply parameters, ACP placement, proxy-aware Telegram fetch, word-boundary chunking, conservative non-idempotent send retries, and direct approval callbacks.
