## Summary

Keep this PR open. Current main has absorbed some Telegram fixes, but the batch is not fully implemented in the current plugin-owned Telegram layout, and the remaining work should be split and ported rather than merged as one stale pre-plugin diff.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but split it. Port the still-relevant Telegram fixes into extensions/telegram in focused changes, especially command-menu config hash invalidation, getFile failure surfacing if maintainers want that behavior, broader media-group per-item skip handling, mention-gating verbose logging, and explicit-network fallback behavior. Preserve current main's newer handling for reply parameters, ACP placement, proxy-aware Telegram fetch, conservative send retries, and direct approval callbacks.
