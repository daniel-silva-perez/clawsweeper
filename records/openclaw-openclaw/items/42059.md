## Summary

Keep this PR open. Current main has absorbed several Telegram fixes, but the PR's intended batch is still not fully implemented in the current `extensions/telegram` plugin layout, and direct merge would be unsafe because the diff targets older `src/telegram/*` paths plus approval/security-sensitive flows.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but do not merge it as one batch. Split and port the still-relevant fixes into `extensions/telegram`: getFile failure surfacing if maintainers want that behavior, command-menu config hash invalidation, mention-gating verbose logging, broader media-group per-item skip handling, and explicit-network fallback behavior. Preserve current main's newer handling for reply parameters, ACP placement, proxy-aware Telegram fetch, word-boundary markdown chunking, conservative non-idempotent send retries, and direct approval callbacks.
