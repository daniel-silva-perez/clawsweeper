## Summary

Keep this PR open. Current main has absorbed several Telegram fixes, but the batch is still only partially ported to the current `extensions/telegram` plugin layout, and multiple intended behaviors remain missing. A direct merge would also be risky because the PR targets old `src/telegram/*` paths and touches approval/security-sensitive code.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but do not merge it as one stale batch. Split and port the still-relevant fixes into `extensions/telegram`, especially getFile failure surfacing, command-menu config hash invalidation, mention-gating verbose logging, broader media-group item skipping, explicit-network fallback behavior, and any approval auth edge cases. Preserve the newer main behavior for reply parameters, approval callback authorization, proxy transport, word-boundary chunking, and conservative non-idempotent send retries.
