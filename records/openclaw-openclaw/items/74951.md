## Summary

Keep this PR open. Current main still has the reported blocking behavior, and the updated PR branch fixed the stale import, but the patch changes a process-policy/code-execution contract without updating the process-hidden running guidance or public docs; maintainer review should decide whether denying `process` only hides follow-up controls or should still prevent detached exec sessions.

## What This Changes

This PR removes the process-policy-derived exec `allowBackground` gate, always enables exec backgrounding, and adds regression coverage for long exec commands when the `process` tool is denied.

## Best Possible Solution

Land a corrected version only if maintainers accept the new policy semantics: keep the narrow exec wiring and regression test, but pass process-tool availability separately so running-result text and tool/docs guidance describe lifecycle timeout or completion wake behavior when `process` is hidden, and document the changed contract in the public exec docs.

## Reproduction Assessment

Yes. The focused reproduction is `createOpenClawCodingTools` with `tools.deny: ["process"]`, a long exec command, and a small `yieldMs`; current main’s source shows `allowBackground: false` for that setup, while the PR adds a regression expecting `status: "running"`.

## Solution Assessment

No, not yet as submitted. The wiring is a narrow way to avoid turn pinning, but a merge-ready solution should not return process follow-up guidance when `process` is denied and should update the documented process-denied exec contract.
