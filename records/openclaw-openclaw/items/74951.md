## Summary

Keep open. The PR addresses a real current-main limitation around process-denied long exec calls, but the submitted patch should not merge as-is because it can leave a backgrounded exec session with no `process` tool and no exec timeout when `timeout: 0` is used; the existing ClawSweeper comment raised the same P2/security concern and this review confirms it.

## What This Changes

This PR keeps exec auto-backgrounding enabled when the separate `process` follow-up tool is denied, updates the hidden-process follow-up text/docs, and adds regression coverage for a long process-denied exec call.

## Best Possible Solution

Land a corrected version only if maintainers accept the new process-denied contract: keep the clearer hidden-process docs/messages, but preserve a finite timeout or reject timeoutless backgrounding whenever the `process` tool is unavailable.

## Reproduction Assessment

Yes. The focused path is `createOpenClawCodingTools` with `tools.deny: ["process"]`, then a long `exec` call with a small `yieldMs`; current main runs that path synchronously, while the PR expects `status: "running"`. The unsafe variant adds `timeout: 0`, which current runtime code proves disables the supervisor timeout.

## Solution Assessment

No, not as submitted. Decoupling backgrounding from process visibility may be acceptable, but the narrow maintainable fix must also prevent timeoutless process-denied background sessions or keep a finite automatic stop path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Guard timeoutless backgrounding without process:** `src/agents/pi-tools.ts:531`
  - body: When `process` is denied, this change still allows `exec` to return a backgrounded session, but `timeout: 0` disables the only remaining automatic stop and the agent has no `process` tool to poll or kill it. Preserve a finite timeout, or reject timeoutless backgrounding, whenever the process tool is unavailable.
