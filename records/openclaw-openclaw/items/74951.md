## Summary

Keep this PR open. Current main still has the process-denied long-exec blocking behavior, and the PR is a relevant implementation attempt, but it changes code-execution policy semantics without updating the process-hidden running guidance or public exec docs.

## What This Changes

This PR removes the process-policy-derived exec backgrounding gate, always passes `allowBackground: true`, and adds a regression test for long exec commands when `process` is denied.

## Best Possible Solution

Land a corrected version only if maintainers accept the new policy semantics: keep the narrow exec wiring and regression test, pass process-tool availability separately into running-result text, and update the public exec docs to describe lifecycle timeout or completion-wake behavior when `process` is hidden.

## Reproduction Assessment

Yes. The focused reproduction is `createOpenClawCodingTools` with `tools.deny: ["process"]`, a long exec command, and a small `yieldMs`; current main source shows the gate disables backgrounding, while the PR adds a test expecting `status: "running"`.

## Solution Assessment

No, not as submitted. Always enabling exec backgrounding is narrow, but a merge-ready fix must not instruct the agent to use an absent `process` tool and must document the changed process-denied contract.
