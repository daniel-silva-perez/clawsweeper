## Summary

Keep open. The PR addresses a real current-main limitation, but the submitted patch should not merge as-is because it can create a backgrounded exec session with no `process` tool and no exec timeout when `timeout: 0` is used; the existing ClawSweeper comment raised the same P2/security concern and current inspection confirms it.

## What This Changes

The branch decouples exec auto-backgrounding from `process` tool availability, updates hidden-process follow-up messaging/docs, and adds regression coverage for a long process-denied exec call.

## Best Possible Solution

Land a corrected version only if maintainers accept the new process-denied contract: keep the clearer hidden-process text, but preserve a finite timeout or reject timeoutless backgrounding whenever `process` is unavailable, and update all exec/background-process docs plus regression coverage for that guard.

## Reproduction Assessment

Yes. The focused path is `createOpenClawCodingTools` with `tools.deny: ["process"]`, then a long `exec` call with a small `yieldMs`; current main source proves that path is synchronous, while the PR adds coverage expecting `status: "running"`. The unsafe variant adds `timeout: 0`, which current runtime code and tests prove disables the supervisor timeout.

## Solution Assessment

No, not as submitted. Decoupling backgrounding from process visibility may be acceptable, but the mergeable solution should not let process-denied background sessions opt out of the only remaining lifecycle timeout.
