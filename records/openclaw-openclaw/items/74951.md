## Summary

Keep open. Current main does not implement this PR's intended process-denied exec backgrounding behavior, and the latest PR revision fixes the earlier stale follow-up-text/docs problem, but the branch still needs maintainer review because it changes code-execution policy semantics and can background a process-denied exec with no remaining in-agent management path when `timeout: 0` is used.

## What This Changes

The PR always enables exec auto-backgrounding even when `process` is denied, passes process-tool availability into exec running-result text, updates exec docs, and adds a regression test for process-denied long commands.

## Best Possible Solution

Land a corrected version only if maintainers accept the new process-denied contract: keep the improved follow-up text and docs, but enforce a finite timeout or reject timeoutless backgrounding whenever the process tool is unavailable.

## Reproduction Assessment

Yes. The focused path is `createOpenClawCodingTools` with `tools.deny: ["process"]`, then a long `exec` call with a small `yieldMs`; current main source proves that path runs synchronously, while the PR adds coverage expecting `status: "running"`.

## Solution Assessment

No, not as submitted. The narrow wiring is plausible and the stale guidance was fixed, but a merge-ready fix should not allow process-denied background sessions to opt out of the only remaining lifecycle timeout.
