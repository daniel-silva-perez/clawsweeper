## Summary

Keep open. The PR addresses a real agent-turn blocking problem, but it is not merge-ready because it enables background exec sessions when `process` is hidden while the running-result guidance and public docs still assume the `process` tool is available.

## What This Changes

The PR removes the process-policy-derived exec backgrounding gate, always passes `allowBackground: true`, and adds regression coverage for a long exec command when `process` is denied.

## Best Possible Solution

Land a corrected version only if maintainers accept the new process-denied semantics: keep the narrow exec wiring, pass process-tool availability into the running-result text, preserve timeout safeguards, and update the public exec docs and regression coverage together.

## Reproduction Assessment

Yes. The focused reproduction is `createOpenClawCodingTools` with `tools.deny: ["process"]`, a long exec command, and a small `yieldMs`; current main source proves that policy disables backgrounding, while the PR adds a test expecting `status: "running"`.

## Solution Assessment

No, not as submitted. Always enabling exec backgrounding is a narrow code change, but the merge-ready fix must avoid pointing agents at an absent `process` tool and must document the changed process-denied contract.

## Security Review

Status: needs_attention

Concerns:
