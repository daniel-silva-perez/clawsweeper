## Summary

Keep open. Current main has not implemented the requested exec behavior, and the PR is a plausible narrow fix, but the submitted patch currently fails lint because it removes the only use of `isToolAllowedByPolicies` while leaving the import in place.

## What This Changes

This PR changes `createOpenClawCodingTools` to always pass `allowBackground: true` to the exec tool and adds regression coverage for a long exec command when the `process` tool is denied.

## Best Possible Solution

Land a corrected version only if maintainers accept the new policy semantics: remove the stale import, keep the regression test, and align `docs/tools/exec.md` and any tool-facing guidance so users understand that exec may still background and rely on lifecycle timeouts when the separate `process` tool is hidden.

## Reproduction Assessment

Yes. The current-main reproduction path is a `createOpenClawCodingTools` config with `tools.deny: ["process"]`, a long exec command, and a small `yieldMs`; source shows current main passes `allowBackground: false`, while the PR's added test captures the desired `running` result.

## Solution Assessment

No, not as submitted. The code direction is narrow, but the patch is not merge-ready while `check-lint` fails on the unused import and the documented process-denied behavior remains stale.
