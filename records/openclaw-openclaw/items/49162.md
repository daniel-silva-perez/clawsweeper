## Summary

Keep this issue open. Current main has adjacent memory-search reliability behavior, including hard-coded query embedding timeouts, batch-index retry/timeout controls, embedding cache controls, QMD search timeout config, and graceful QMD-to-builtin fallback. It does not expose the requested schema-backed `agents.defaults.memorySearch.query.timeoutMs`, query-result `cacheTtlMs`, or query retry policy, and the related #49233/#49056 discussion is adjacent durability/provider work rather than a canonical replacement for this specific config surface.

## Best Possible Solution

Implement this in the memory-core/config boundary: add typed and zod-validated `memorySearch.query.timeoutMs`, `memorySearch.query.cacheTtlMs`, and `memorySearch.query.retry`; resolve them through `src/agents/memory-search.ts`; wire them into `extensions/memory-core` search execution with graceful timeout/retry/fallback behavior; add focused schema/runtime tests; and document the new controls in the public memory configuration reference. Keep #49233/#49056 as adjacent durability/provider context, not as a replacement for this query reliability surface.
