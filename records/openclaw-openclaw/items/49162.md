## Summary

Keep this issue open. Current main still has adjacent memory-search reliability behavior, but it does not expose schema-backed `agents.defaults.memorySearch.query.timeoutMs`, query-result `cacheTtlMs`, or a query retry policy through the normal per-agent memory search config path. Active Memory has its own plugin timeout/cache knobs and QMD has a separate timeout, but neither closes the requested `memorySearch.query` config surface.

## Best Possible Solution

Implement the requested controls in the memory-core/config boundary: add typed and zod-validated `memorySearch.query.timeoutMs`, `memorySearch.query.cacheTtlMs`, and `memorySearch.query.retry`; resolve them in `src/agents/memory-search.ts`; wire them into the memory search execution path with graceful timeout, retry, cache invalidation, and fallback behavior; add focused schema/runtime tests; and document the public options in the memory configuration reference. Keep #49056 and #49233 as adjacent durability/provider context rather than replacements for this query reliability surface.
