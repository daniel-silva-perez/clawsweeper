## Summary

Keep this PR open. Current main still verifies Nextcloud Talk webhook signatures with an early return on length mismatch followed by a manual XOR loop, so the security hardening the PR targets is not implemented on main. The PR is narrowly scoped and useful, but it still needs fixup for the mismatch-branch dummy comparison called out in review.

## Best Possible Solution

Keep this PR open. Ask the contributor to update the mismatch branch to follow the existing LINE/shared timing-safe comparison pattern, retain the focused Nextcloud Talk signature tests, then run the targeted Nextcloud Talk test lane and the normal changed gate before landing.
