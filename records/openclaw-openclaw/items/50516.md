Keep this PR open. Current main still has the Nextcloud Talk verifier behavior the PR targets: it returns early on signature length mismatch and then uses a manual XOR loop. The PR is narrow and security-relevant, but its current branch still has an open review concern around using `timingSafeEqual(expBuf, expBuf)` as the mismatch dummy comparison, so the best path is contributor follow-up rather than cleanup closure.
