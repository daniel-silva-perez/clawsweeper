## Summary

Keep this PR open. Current main still has the Nextcloud Talk webhook signature verifier behavior this PR targets: an early return on length mismatch followed by a manual XOR loop. The PR is narrow and security-relevant, but the open review concern about the dummy mismatch comparison should be fixed before merge.

## Best Possible Solution

Keep this PR open. Ask the contributor to revise the length-mismatch branch to avoid self-comparison, keep the focused signature tests, then run the targeted Nextcloud Talk test lane and normal changed gate before landing.
