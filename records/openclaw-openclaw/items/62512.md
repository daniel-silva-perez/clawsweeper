## Summary

Keep this issue open. Current main does not implement the proposed AI in-app UI guide, and the existing plugin/Control UI surfaces are not sufficient to move the request out to ClawHub without first adding a real UI-extension/action API.

## Best Possible Solution

Keep open for maintainer product judgment. The best path is either a core Control UI feature or a deliberately designed plugin/UI API that supports stable `data-guide` IDs, a tested catalog, live UI snapshots, a typed validated action protocol, permission boundaries, and docs/tests for any guide-agent session behavior.

## Risks / Open Questions

- A partial implementation could plausibly live as a plugin later, but current main only exposes static Control UI descriptors; closing as ClawHub now would skip the missing core/UI extension contract discussion.
- The proposal includes safety-sensitive UI automation, so the right path needs maintainer product/security design around allowed actions, validation, permissions, and tests before implementation.
