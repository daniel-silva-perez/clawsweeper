Kept open: current main does not implement the requested in-app guide machinery, and this is too broad and security-sensitive for a conservative stale, ClawHub, or implemented-on-main close.

## What This Changes

The issue requests an AI-powered in-app guide that answers help questions, snapshots the live UI, navigates/highlights/clicks/reveals stable cataloged elements, and runs a dedicated guide agent loop.

## Best Possible Solution

Use this as a product/design item: define a minimal guide contract first, including stable UI element IDs, drift tests, read-only snapshot shape, a validated action schema, and explicit consent boundaries before adding any auto-click behavior.
