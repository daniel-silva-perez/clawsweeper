## Summary

Keep this issue open. Current main still drops prompt-facing skills by count and character-budget prefix truncation after alphabetical sorting, and the only warning is still a count-only prompt note for the model. The docs now mention the skills prompt budget in a few places, but omitted-skill recovery, user-visible warning behavior, and priority/pinning or required-skill controls have not landed; #42637 and #50199 remain the relevant open follow-ups in the provided context.

## What This Changes

This issue asks OpenClaw to stop silently dropping eligible user-created skills when the skills prompt budget truncates the catalog, by adding visible warnings, prioritization or pinning controls, required-skill behavior, and documentation.

## Best Possible Solution

Keep this open until OpenClaw ships #42637 or equivalent omitted-skill recovery, documents the prompt budget and truncation outcome on public docs, and makes an explicit product/API decision on whether priority, pinning, or required skills belong in the skills config contract or should remain tracked separately in #50199.
