## Summary

Keep this issue open. Current main still drops skills from the prompt-facing catalog when the compact catalog exceeds the configured budget, still exposes only a generic count-based truncation note to the model, and still lacks omitted-skill recovery, user-visible warning, priority, pinning, or required-skill config semantics. The linked omitted-name PR and priority follow-up are both still unresolved in the provided GitHub context.

## What This Changes

The issue asks OpenClaw to stop silently hiding eligible user-created skills when the skills prompt exceeds its budget, by adding visibility, documentation, and/or priority/pinning guarantees.

## Best Possible Solution

Keep this issue open until OpenClaw lands #42637 or an equivalent omitted-skill recovery path, documents the skills prompt budget and truncation outcome on the public docs, and makes a maintainer product decision on whether priority/pinning belongs here or should stay tracked in #50199.
