## Summary

Keep this issue open. Current main has partial mitigations: skills can fall back to compact name/location entries, the budget is configurable, and Telegram topics can narrow the skill set. But the reported trust gap remains: when the compact skills prompt still exceeds the budget, current main still drops a prefix of skills, the truncation note does not name omitted skills, and user-facing context reporting only sees skills that survived prompt construction. The related open PR #42637 is a plausible partial fix for omitted-name visibility, but it is still open and does not cover the full user-facing warning/pinning/docs request.

## Best Possible Solution

Keep this issue open. The best path is to review and either land #42637 or an equivalent omitted-skill-name implementation, then add a user-visible audit surface such as `/context`, `openclaw skills check`, logs, or docs that clearly reports when prompt-budget truncation happened and which skills were excluded. If maintainers want hard guarantees for critical skills, add a small, documented priority/pinning contract or clearly document the supported workaround using per-agent/channel/topic skill filters and larger `maxSkillsPromptChars` budgets.
