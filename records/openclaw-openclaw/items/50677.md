## Summary

Keep open. Current main still truncates the prompt-facing skills list to a fitting sorted prefix, emits only a generic model-facing count warning, and has no omitted-skill list, pinning/priority field, or required-skill guarantee. The related omitted-name fix PR #42637 is still open, and priority configuration remains tracked separately in #50199.

## What This Changes

The issue asks OpenClaw to make skills prompt truncation observable and configurable so user-created or critical skills are not invisibly dropped when the skills list exceeds the prompt budget.

## Best Possible Solution

Keep this issue open until OpenClaw either lands #42637 or an equivalent omitted-skill recovery path, documents the prompt budget and truncation outcome, and makes a maintainer product decision on whether priority/pinning or required-skill config should be implemented here or split into the canonical priority follow-up #50199.
