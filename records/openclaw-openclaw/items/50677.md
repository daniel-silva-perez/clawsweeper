## Summary

Keep this issue open. Current main still truncates the prompt-facing skills catalog to a fitting prefix, the truncation notice remains prompt-facing and count-only, and the config/schema surface still has no priority, pinned, or context-required skill contract. ## What This Changes

The issue requests visible omitted-skill handling and a way to prioritize or force-load critical user-created skills when the skills prompt exceeds its budget.

## Best Possible Solution

Keep the issue open until OpenClaw ships an omitted-skill discovery or recovery path, documents skills prompt budget and truncation outcomes on the public skills docs, and explicitly resolves whether priority, pinning, or context-required skills belong in the skills config contract or are tracked only by narrower follow-ups.
