## Summary

Keep this issue open. Current main still truncates the prompt-facing skills catalog by count and then by fitting compact-prefix budget after alphabetical sorting, and the emitted note remains model-facing and count-only. The public docs now mention the skills prompt budget in a few places, but omitted-skill recovery, user-visible warning behavior, and priority/pinning or required-skill controls are still unresolved; the related narrow PR #42637 and broader priority issue #50199 remain open in the provided GitHub context.

## What This Changes

The issue asks OpenClaw to stop silently omitting eligible user-created skills when the skills prompt budget is exceeded by surfacing truncation and adding prioritization, pinning, required-skill controls, and documentation.

## Best Possible Solution

Keep this open until OpenClaw ships an omitted-skill recovery path, documents the skills prompt budget and omission behavior on the public docs, and makes an explicit maintainer decision on whether priority, pinning, or required skills belong in the skills config contract or should stay tracked by #50199.
