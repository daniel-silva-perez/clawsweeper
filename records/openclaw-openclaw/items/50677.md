## Summary

Keep this issue open. Current main still truncates the prompt-facing skills catalog to a fitting prefix when the compact list exceeds the configured budget, and the warning remains model-facing and count-only. The narrow omitted-name PR and the broader priority-configuration issue are both still open in the provided GitHub context, so the requested visibility and prioritization behavior has not landed.

## What This Changes

The issue asks OpenClaw to prevent eligible user-created skills from disappearing under the skills prompt budget without user-visible warning, documentation, or priority/pinning controls.

## Best Possible Solution

Keep the issue open until OpenClaw ships an omitted-skill recovery path, documents the skills prompt budget and truncation outcome on the public docs, and resolves whether priority/pinning belongs in the skills config contract or remains tracked separately in #50199.
