## Summary

Keep open. Current main still does not implement a custom TUI client display name: the CLI has no `--client-name` or `--display-name` option, `TuiOptions` cannot carry one, `runTui` passes only url/token/password into the Gateway connection, and `GatewayChatClient` still sends the hard-coded `clientDisplayName: "openclaw-tui"`. The Gateway protocol and generic client already support display names, so the remaining work is focused TUI wiring, relaunch preservation, docs, and tests rather than a core protocol gap.

## Best Possible Solution

Add a focused TUI feature: parse `--client-name` or `--display-name` in `src/cli/tui-cli.ts`, extend `TuiOptions` and `GatewayConnectionOptions`, pass a sanitized display name through `runTui` into `GatewayChatClient`, keep the default as `openclaw-tui`, preserve the stable TUI client id, carry the value through `src/tui/tui-launch.ts`, and update TUI docs plus focused tests.
