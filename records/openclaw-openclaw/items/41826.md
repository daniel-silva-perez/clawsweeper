## Summary

Keep this issue open. Current main still has no TUI CLI option or TUI option field for a custom client display name, and the Gateway-backed TUI still connects with `clientDisplayName: "openclaw-tui"`. The Gateway protocol and presence path already support display names, so the remaining work is a focused TUI wiring/docs/test change rather than a protocol gap or a duplicate of the already-implemented WebChat/Control UI display-name items.

## Best Possible Solution

Add a focused TUI implementation: parse `--client-name` or `--display-name` in `src/cli/tui-cli.ts`, extend `TuiOptions` and `src/tui/tui-launch.ts`, pass a sanitized value through `runTui` into `GatewayChatClient.connect`, default to `openclaw-tui`, keep the stable protocol client id unchanged, and update `docs/web/tui.md`, `docs/cli/tui.md`, and focused CLI/GatewayChatClient tests.
