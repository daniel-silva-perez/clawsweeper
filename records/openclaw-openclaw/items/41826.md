## Summary

Keep open. Current main still does not implement a custom TUI client display name: the TUI CLI has no display-name/client-name option, TUI options cannot carry one, `runTui` does not pass one, and `GatewayChatClient` still hard-codes `clientDisplayName: "openclaw-tui"`. The Gateway client/protocol/presence path already supports display names, so this is focused TUI wiring rather than a core protocol gap. The related WebChat/Control UI display-name items do not cover this TUI-specific request.

## Best Possible Solution

Add a focused TUI implementation: parse `--client-name` or `--display-name` in `src/cli/tui-cli.ts`, extend `TuiOptions` and `GatewayConnectionOptions`, pass a sanitized value through `runTui` into `GatewayChatClient`, default to `openclaw-tui`, preserve `GATEWAY_CLIENT_NAMES.TUI` as the protocol id, preserve the option in `src/tui/tui-launch.ts`, and document it in `docs/cli/tui.md` plus `docs/web/tui.md` with focused tests.
