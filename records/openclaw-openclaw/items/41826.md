## Summary

Keep open. Current main still does not implement a custom TUI client display name: `openclaw tui` has no `--client-name`/`--display-name` option, the TUI option and relaunch types cannot carry one, `runTui` does not pass one to the Gateway connection, and `GatewayChatClient` still hard-codes `clientDisplayName: "openclaw-tui"`. The Gateway protocol already supports client display names, so this is a focused TUI wiring/docs/tests task rather than a core protocol gap.

## Best Possible Solution

Add a focused TUI implementation: parse `--client-name` or `--display-name` in `src/cli/tui-cli.ts`, extend TUI/Gateway connection options, pass a sanitized display name through `runTui` into `GatewayChatClient`, keep the default as `openclaw-tui`, preserve the stable TUI protocol id, carry the option through `src/tui/tui-launch.ts`, and document/test the behavior.
