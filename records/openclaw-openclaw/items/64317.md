Keep this issue open. Current main still routes ordinary managed browser viewport screenshots with a per-tab CDP WebSocket through `Page.captureScreenshot`, and the provided GitHub context shows PR #71940 is open, unmerged, and explicitly linked as the implementation candidate for this issue.

The likely implementation belongs in the bundled browser plugin as an opt-in Hyprland compositor-capture fallback for headed viewport screenshots only, with CDP fallback, route-selection tests, cleanup tests, Linux troubleshooting docs, and a focused security review of external binary execution.
