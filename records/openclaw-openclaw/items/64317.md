## Summary

Keep this issue open. Current main still routes ordinary managed browser viewport screenshots with a per-tab WebSocket through CDP `Page.captureScreenshot`, and the provided issue context shows PR #71940 is the open, unmerged implementation candidate. The discussion also usefully points to the Hyprland fake-output plus `grim` workaround that the PR is trying to productize.

## Best Possible Solution

Keep this issue open and review PR #71940, or replace it with an equivalent focused browser-plugin fix. The likely product path is an opt-in Hyprland compositor-capture fallback for headed viewport screenshots only, with CDP fallback, route-selection tests, cleanup tests, Linux troubleshooting docs, and focused security review for external binary execution.
