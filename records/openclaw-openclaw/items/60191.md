## Summary

Keep open. Current main has a real nearby fix for OpenRouter `Key limit exceeded` on 401/403, but I did not find source, docs, or test evidence that it covers the reporter’s exact `403 API key budget limit exceeded (monthly limit)` string. The current 403 path still falls back to generic auth unless the message is already classified as billing.

## Best Possible Solution

Keep the issue open and add targeted coverage for OpenRouter `401`/`403 API key budget limit exceeded (monthly limit)`. The likely fix belongs in the failover classification layer: extend the OpenRouter-scoped matcher to accept `key budget limit` or add a high-confidence budget-exhaustion billing pattern, then add unit coverage and a fallback-chain test proving the next configured model is attempted.
