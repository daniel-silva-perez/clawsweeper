## Summary

Keep open. Current main still has the core maintainability problem described here: the main CI workflow is very large, preflight still exports a wide set of routing outputs, downstream jobs remain tightly coupled to those outputs, and the proposed matrix JSON/reusable workflow/README acceptance criteria are not present.

## What This Changes

The issue requests a behavior-preserving CI refactor that extracts complex matrices, moves repeated setup/check patterns into reusable workflows, shrinks `.github/workflows/ci.yml`, and documents the workflow layout.

## Best Possible Solution

Keep this as an open maintainer-owned CI design/refactor item. The best end state is an incremental, behavior-preserving split that reduces duplicated checkout/setup logic, moves stable routing data out of the main workflow where appropriate, keeps dynamic shard planners in typed/tested scripts when JSON would be brittle, documents the workflow layout, and preserves required check names or provides a deliberate migration plan.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. Current-main structural checks provide high-confidence verification of the cleanup need: the workflow is 2144 lines, the proposed extracted files are absent, and preflight-output coupling remains widespread.

## Solution Assessment

Unclear whether the exact proposed JSON/reusable-workflow split is the best implementation. The direction is plausible, but the safest solution needs maintainer design review because the current workflow mixes static matrices with dynamically generated shard plans and CI security-sensitive execution paths.
