## Summary

Current main still has the maintainability problem described here: the CI workflow remains large, the preflight manifest still exports many booleans and matrices, and the requested JSON/reusable-workflow/documentation extraction does not exist. This should stay open for maintainer design because it is a broad CI control-plane refactor with validation and supply-chain implications.

## What This Changes

Refactor the CI workflow by extracting matrix definitions into JSON files, moving common job patterns into reusable workflows, shrinking `ci.yml`, and documenting the workflow structure.

## Best Possible Solution

Land a maintainer-approved CI decomposition in small audited steps that preserves the existing job graph, aggregate check names, checkout trust model, artifact flow, and smart-scope behavior, with docs updated after the new structure is proven in PR CI.

## Reproduction Assessment

Not applicable as a bug reproduction. The maintenance request is directly verifiable from current main: `ci.yml` is still 2144 lines and the requested `.github/matrices`, private reusable workflows, and workflow README are absent.

## Solution Assessment

Unclear whether the exact proposed JSON/static workflow split is the best final design because current matrices are computed from tested JavaScript helpers. A safer direction is a staged extraction of repeated checkout/setup/build patterns while preserving dynamic shard generation until it has an equivalent tested config contract.
