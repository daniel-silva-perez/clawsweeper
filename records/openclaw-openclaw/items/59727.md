## Summary

Keep open. Current main has partial CI helper reuse, but the central request is not implemented: `.github/workflows/ci.yml` is still a 2,112-line workflow with inline preflight manifest/matrix construction, downstream jobs still consume many `needs.preflight.outputs.*` matrices, there is no `.github/matrices/` directory, and there is no `.github/workflows/README.md` documenting the requested split.

## Best Possible Solution

Keep this issue open as a valid CI maintainability request. The best path is an owner-reviewed refactor series that preserves the current smart-scoped behavior while moving matrix/manifest definitions out of the monolithic `ci.yml`, documenting the workflow layout, and proving parity with existing CI routing tests plus workflow-sanity/actionlint or a controlled validation PR.
