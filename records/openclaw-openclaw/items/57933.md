## Summary

Keep this issue open. Current main still has chat-scoped streaming/tool feedback and session list refreshes, but it does not expose or render the requested per-session heartbeat/liveness model for active turns. The prior ClawSweeper comment is still directionally correct against current main: no heartbeat endpoints/status field, no periodic per-session turn-health emitter, and no Control UI health badge/stall/unresponsive state for the sessions list.

## Best Possible Solution

Implement this as an additive core Gateway plus Control UI feature: define a typed per-session active-turn health state owned by the run/session lifecycle, expose it through sessions.list or a new authenticated sessions heartbeat/subscription method, reuse existing chat/tool streaming events where practical, and render compact live badges/tooltips in the Sessions view. Cover state transitions, no-heartbeat/stall detection, permission scoping, and UI rendering with focused tests.
