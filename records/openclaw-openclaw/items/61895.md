## Summary

Keep open. Current main has adjacent ACP recovery work, including dead-handle re-ensure and one fresh persistent retry after missing-session failures, but it does not implement the central requested cleanup: killing or reaping the previous ACP process tree before repair/replacement, or disposing persistent ACP runtimes on gateway shutdown. The reporter's ppid=1 heartbeat workaround is useful supporting context, but it remains an external workaround rather than shipped behavior.

## Best Possible Solution

Keep this issue open. The best fix is to add ACP/acpx process ownership and cleanup: track child PID/process group for persistent runtime sessions, close or kill the old tree before fresh ensure/repair, dispose cached persistent ACP runtimes during gateway shutdown, and add scoped tests for persistent dead-session repair plus gateway restart/shutdown cleanup. Coordinate shared cleanup primitives with #68916, but keep this persistent-binding leak tracked here.
