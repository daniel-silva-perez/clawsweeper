## Summary

Keep this PR open. Current main already implements and shipped the deferred isolated-cron timeout half, but it still does not implement the PR's per-attempt AbortController/fallback-poisoning fix on the active cron executor path. The remaining work should be rebased against the current run-executor split and the newer shared abort/failover policy.

## Best Possible Solution

Keep this PR, or a rebased replacement, open. The best path is to drop the already-shipped deferred-timeout changes, retarget the remaining per-attempt abort/fallback work to src/cron/isolated-agent/run-executor.ts or the shared embedded-runner failover contract, and add focused tests proving retryable per-attempt provider timeouts can move to later fallback candidates while true cron-budget/user aborts remain terminal. Coordinate with #60388/#62682 before landing a cron-only workaround.
