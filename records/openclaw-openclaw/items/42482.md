## Summary

Keep this PR open. Current main has shipped the deferred isolated-cron execution-timeout behavior, but it still forwards the shared cron abort signal through each model fallback candidate, so the PR's per-attempt AbortController concern is not implemented. The remaining work should be rebased and narrowed against the current cron executor.

## Best Possible Solution

Keep this PR, or a rebased replacement, open. The best maintainer path is to drop the already-shipped deferred-timeout portion, rebase the remaining per-attempt abort isolation onto src/cron/isolated-agent/run-executor.ts and runWithModelFallback, and add focused regression coverage proving retryable cron fallback failures do not poison later candidates while terminal cron-budget aborts still stop the whole run.
