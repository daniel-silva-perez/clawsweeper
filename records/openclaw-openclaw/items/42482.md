## Summary

Keep this PR open. Current main already implements and ships the deferred isolated-cron execution-timeout half, but the per-attempt AbortController/fallback-signal isolation described in the PR is still missing: cron fallback attempts still forward the same outer abort signal into CLI and embedded runs.

## Best Possible Solution

Keep this PR, or a rebased canonical replacement, open. The best maintainer path is to split out the already-shipped #41783 deferred-timeout changes, rebase the remaining work onto src/cron/isolated-agent/run-executor.ts and the shared model-fallback contract, and add regression coverage proving later cron fallback candidates are not killed by a stale attempt signal while terminal cron budget aborts still stop the whole run.
