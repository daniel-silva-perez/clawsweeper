## Summary

Keep this PR open. Current main implements the deferred isolated-cron execution-timeout half, but it does not implement the PR's remaining per-attempt AbortController/fallback-poisoning change on the current cron executor path. The PR is also stale against the run-executor split and the shared failover policy now explicitly treats external aborts as terminal.

## Best Possible Solution

Keep this PR, or a rebased replacement, open. The best maintainer path is to drop the already-shipped deferred-timeout changes, retarget the remaining abort/fallback work to src/cron/isolated-agent/run-executor.ts or the shared embedded-runner failover contract, and add focused coverage proving retryable per-attempt cron provider timeouts can move to later fallback candidates while true parent cron-budget/user aborts remain terminal. Coordinate with the broader #60388/#62682 abort-contract work before landing a cron-only workaround.
