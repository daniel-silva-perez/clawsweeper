## Summary

Keep this PR open. Current main has shipped the deferred isolated-cron execution-timeout behavior, but the PR's per-attempt AbortController/fallback-poisoning concern is still not implemented on the current cron executor path; the executor still forwards the shared cron abort signal into every fallback candidate, and external aborts are intentionally terminal in the failover policy.

## Best Possible Solution

Keep this PR, or a rebased replacement, open. The best path is to drop the already-shipped deferred-timeout changes, rebase the remaining abort/fallback work onto src/cron/isolated-agent/run-executor.ts or the shared agent failover contract, and add focused regression coverage proving retryable per-attempt cron fallback failures do not poison later candidates while true parent cron-budget/user aborts still stop the whole run.
