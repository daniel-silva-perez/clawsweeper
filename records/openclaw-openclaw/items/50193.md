## Summary

Keep this PR open. Current main still renders the Control UI chat model picker as a flat list of model options, so the PR's provider-grouped selector is not implemented. The contribution is plausible, but it is stale against the current model-select state and session-controls split and should be reworked there before maintainer action.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up. Rebase the idea onto ui/src/ui/chat-model-select-state.ts and ui/src/ui/chat/session-controls.ts, represent provider groups in model-select state, render model <optgroup> sections, sort by displayed provider label, preserve current duplicate-name disambiguation, keep sessions.patch behavior unchanged, and add focused tests for grouping, unknown-provider fallback, duplicate labels, selected-label clarity, and provider-qualified values.
