## Summary

Keep open. Current main still renders the Control UI chat model picker as a flat option list, so this PR's intended provider-grouped selector is not implemented. The work is plausible, but the PR is stale against the current model-picker split and needs rework to preserve existing provider/model disambiguation.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up. Rebase it onto the current chat model picker modules, represent grouped provider sections in `ui/src/ui/chat-model-select-state.ts`, render model `<optgroup>` sections in `ui/src/ui/chat/session-controls.ts`, sort by displayed provider label, preserve the existing catalog display helpers for duplicate-name clarity, and add focused tests for grouping, unknown provider fallback, duplicate labels, selected-label clarity, and unchanged `sessions.patch` behavior.
