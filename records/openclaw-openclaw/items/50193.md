## Summary

Current main still renders the Control UI chat model picker as a flat list, so this PR’s provider-grouped selector is not implemented. The PR remains plausible, but it is stale against the current split between chat model state and session control rendering, and it needs rework to preserve existing duplicate-model label clarity.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up. Rebase it onto `ui/src/ui/chat-model-select-state.ts` and `ui/src/ui/chat/session-controls.ts`, model provider groups in state, render model `<optgroup>` sections, sort by displayed provider label, preserve the existing catalog display helpers for duplicate-name clarity, and add focused tests for grouping, unknown-provider fallback, duplicate labels, selected-label clarity, and unchanged `sessions.patch` behavior.
