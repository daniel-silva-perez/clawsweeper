## Summary

Keep open. Current main still renders the Control UI chat model picker as one flat option list, so this PR's central provider-grouped selector behavior is not implemented on main. The PR remains plausible UI work, but it needs maintainer/contributor follow-up to preserve the existing disambiguated model labels and avoid drift from a hard-coded provider-label table.

## Best Possible Solution

Keep this PR open for maintainer or contributor follow-up. The best path is to add a grouped chat-model select state in `ui/src/ui/chat-model-select-state.ts`, render provider `<optgroup>` sections in `ui/src/ui/chat/session-controls.ts`, sort groups by displayed provider label, and reuse the existing catalog display helpers so duplicate model names remain distinguishable. Add focused tests for provider grouping, duplicate names across providers, duplicate names within one provider, unknown provider fallback, and unchanged `sessions.patch` behavior.
