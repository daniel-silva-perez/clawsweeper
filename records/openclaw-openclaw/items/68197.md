## Summary

Keep this PR open. Current main still has the LINE group behavior the PR describes: non-text LINE messages bypass the mention gate because the handler only marks text messages as mention-detectable, and the proposed opt-in config key is not present in the LINE config schema or type. The PR diff is narrowly scoped to the LINE plugin and tests, and the security pass did not find workflow, dependency, lockfile, script, package publishing, generated/vendor, or secret-handling changes.

## What This Changes

This PR adds an optional `requireMentionForNonText` LINE group config field, threads it into LINE group mention gating for non-text messages, updates the strict LINE config schema/type, and adds regression tests for the opt-in behavior.

## Best Possible Solution
