## Summary

Keep this PR open as a valid LINE plugin implementation candidate, but do not merge it as-is. Current main still has the reported non-text LINE group mention-gate bypass, and the proposed opt-in matches the existing mention-gating helper and LINE's text-only mention metadata, but the PR leaves generated config metadata and release-note follow-through stale.

## What This Changes

This PR adds an opt-in `requireMentionForNonText` LINE group setting, uses it to include non-text LINE group messages in mention gating, and adds LINE handler tests while preserving the default fail-open behavior.

## Best Possible Solution

Review this focused LINE plugin PR as the active implementation candidate. If maintainers accept the compatibility-preserving opt-in, merge it after regenerating bundled channel config metadata/config baselines, adding the required changelog entry, and adding any desired LINE/group config documentation for `channels.line.groups.<group>.requireMentionForNonText`.

## Reproduction Assessment

Yes. A current-main reproduction is a LINE group message with `requireMention` enabled and a non-text payload such as an image: the handler sets `canDetectMention` false, and the shared helper therefore does not skip the message. The existing LINE handler test at `extensions/line/src/bot-handlers.test.ts:1002` documents that behavior.

## Solution Assessment

Mostly yes for the runtime behavior: an opt-in flag is the narrow compatibility-preserving way to make LINE non-text group media follow mention gating without changing existing deployments. The proposed patch is not merge-ready until generated config metadata and release-note/docs follow-through are aligned.
