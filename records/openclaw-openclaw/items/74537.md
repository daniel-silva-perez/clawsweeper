## Summary

Keep open. Current main has the earlier bare-command acknowledgement fix, but the reset/session lifecycle paths still emit empty-history hook payloads and session_end events, so the specific lossless-claw/provider 400 path reported here is not proven fixed.

## What This Changes

Skip the model-backed reset/session lifecycle work when `/reset` or `/new` is run against an already fresh session with no prior messages, avoiding provider 400 errors and cooldowns.

## Best Possible Solution

Keep the shipped bare-command acknowledgement behavior, then define and test the empty-transcript lifecycle contract so empty-history resets do not trigger model-backed wrap-up work while preserving useful plugin lifecycle notifications where possible.

## Reproduction Assessment

Yes for triage: the issue provides concrete Telegram steps and provider/cooldown logs, and current source shows empty reset/session lifecycle payloads can still be emitted. I did not run a live Telegram plus lossless-claw reproduction because this review was read-only.

## Solution Assessment

Unclear. Skipping the failing empty-history model work is the right direction, but the safest implementation needs maintainer review of the plugin hook contract so OpenClaw does not silently drop lifecycle events that non-LLM plugins rely on.
