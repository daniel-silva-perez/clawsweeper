## Summary

Keep this PR open. Current main still does not implement the requested per-invocation `--skip-bootstrap` flag for `openclaw agents add`, so the PR is not obsolete. The PR still needs maintainer/reviewer follow-up because the supplied diff adds `skipBootstrap` to the explicit-option watchlist, which would make `openclaw agents add <name> --skip-bootstrap` force non-interactive mode and require `--workspace` instead of allowing the wizard path.

## Best Possible Solution

Keep this PR open and request a focused revision: add the `--skip-bootstrap` option and `AgentsAddOptions.skipBootstrap`, OR it with `agents.defaults.skipBootstrap` at both workspace setup call sites, exclude it from non-interactive flag detection, add CLI-level coverage proving `openclaw agents add <name> --skip-bootstrap` can still use the wizard path, keep non-interactive forwarding coverage, update `docs/cli/agents.md`, and remove or split unrelated test churn.
