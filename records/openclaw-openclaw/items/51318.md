## Summary

Keep this PR open. Current main still does not implement the requested per-invocation `--skip-bootstrap` flag for `openclaw agents add`: the CLI does not register or forward the option, `AgentsAddOptions` has no `skipBootstrap` field, and both workspace setup paths still rely only on the global `agents.defaults.skipBootstrap` config. The PR remains a relevant implementation candidate, but it needs revision because the supplied review discussion correctly identifies that adding `skipBootstrap` to the explicit-option watchlist would force non-interactive mode and block the interactive wizard path when the flag is passed alone.

## Best Possible Solution

Keep this PR open and request a focused revision: add `--skip-bootstrap` to `agents add`, add `AgentsAddOptions.skipBootstrap`, OR it with `agents.defaults.skipBootstrap` at both workspace setup call sites, exclude it from explicit non-interactive flag detection, add CLI-level coverage proving `openclaw agents add <name> --skip-bootstrap` can still use the wizard path, keep non-interactive forwarding coverage, update `docs/cli/agents.md` / the public CLI docs, and remove or split the unrelated image-generation test churn.
