## Summary

Keep this PR open. Current main still lacks a per-invocation `--skip-bootstrap` flag for `openclaw agents add`, so the work is not obsolete, but the branch is not merge-ready because it treats `--skip-bootstrap` as an explicit non-interactive trigger and blocks the interactive wizard path the flag is meant to support.

## What This Changes

The PR adds a `--skip-bootstrap` option to `openclaw agents add`, plumbs `skipBootstrap` through CLI and command setup, ORs it with `agents.defaults.skipBootstrap`, adds agents command tests, and also changes image-generation runtime tests to use `setActivePluginRegistry`.

## Best Possible Solution

Revise this PR so `--skip-bootstrap` is a behavior modifier usable in both interactive and non-interactive `agents add` flows: keep the command plumbing and OR-with-config behavior, remove `skipBootstrap` from explicit non-interactive flag detection, add CLI-level coverage for `openclaw agents add <name> --skip-bootstrap` staying on the wizard path, document the flag at `https://docs.openclaw.ai/cli/agents`, and keep unrelated test cleanup separate from the feature branch.

## Reproduction Assessment

Yes. The proposed patch has a high-confidence source-level reproduction: on the PR branch, `--skip-bootstrap` is included in `hasExplicitOptions`, and `agentsAddCommand` converts `hasFlags` into non-interactive mode that exits when `--workspace` is absent.

## Solution Assessment

No, not as currently written. The intended plumbing is narrow and maintainable, but `skipBootstrap` should be excluded from non-interactive flag detection and covered through the CLI registration layer before this is the best fix.
