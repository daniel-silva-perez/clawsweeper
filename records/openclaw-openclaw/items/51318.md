## Summary

Keep this PR open. Current main still does not implement the requested per-invocation `--skip-bootstrap` flag for `openclaw agents add`; it only supports the global `agents.defaults.skipBootstrap` setting and `openclaw onboard --skip-bootstrap`. The PR remains a relevant implementation candidate, but the provided review context identifies a real behavior issue in the PR diff: counting `--skip-bootstrap` as an explicit non-interactive trigger would make the interactive wizard path unreachable when that flag is used alone.

## Best Possible Solution

Keep the PR open and ask for a focused revision: add the `--skip-bootstrap` option and `AgentsAddOptions.skipBootstrap`, OR it with `agents.defaults.skipBootstrap` at both `ensureWorkspaceAndSessions` call sites, do not count it as a non-interactive trigger, add a CLI-level test proving `openclaw agents add <name> --skip-bootstrap` still uses the wizard path, keep the non-interactive forwarding coverage, update the `agents add` docs, and remove unrelated test churn before maintainer review.
