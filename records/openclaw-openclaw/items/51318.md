## Summary

Keep this PR open. Current main still does not implement the requested per-invocation `--skip-bootstrap` flag for `openclaw agents add`, so the PR remains a relevant implementation candidate. It needs revision before merge because the PR diff still includes `skipBootstrap` in the explicit-option watchlist, which would force non-interactive mode and block the interactive wizard path when the flag is used alone.

## Best Possible Solution

Keep this PR open and request a focused revision: add the `--skip-bootstrap` option and `AgentsAddOptions.skipBootstrap`, OR it with `agents.defaults.skipBootstrap` at both `ensureWorkspaceAndSessions` call sites, do not count it as a non-interactive trigger, add CLI-level coverage proving `openclaw agents add <name> --skip-bootstrap` can still use the wizard path, keep non-interactive forwarding coverage, update the `agents add` docs, and remove unrelated test churn.
