## Summary

Keep this PR open. Current main still returns memory corpus supplements in registration/re-registration order, while the adjacent prompt supplement path sorts by pluginId for cache stability. The PR remains a narrow implementation candidate with focused tests and no observed security or supply-chain expansion in the provided diff.

## Best Possible Solution

Keep this PR open for maintainer review or landing after any needed rebase. The best fix is still the focused one proposed here: sort listMemoryCorpusSupplements() by pluginId in src/plugins/memory-state.ts and keep regression tests for reverse registration order plus same-plugin re-registration stability, unless maintainers choose a different explicit deterministic conflict-resolution contract for overlapping corpus supplements.
