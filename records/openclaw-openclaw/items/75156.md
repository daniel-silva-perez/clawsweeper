## Summary

Keep open. Current main still records mixed text-plus-media block payload media as sent before final reply dedupe runs, and open PR #75180 is already the active implementation candidate for this exact issue.

## What This Changes

Telegram direct-chat replies containing visible text plus a `MEDIA:` attachment should deliver both the text and attachment on the first send.

## Best Possible Solution

Review and land PR #75180, or an equivalent narrow replacement, with regression coverage that proves mixed text-plus-media block replies do not mark media as delivered unless the channel actually delivered that media while media-only duplicate suppression still works.

## Reproduction Assessment

Yes. The reporter supplied live Telegram DM steps and local validation, and current source gives a high-confidence unit reproduction path through `createBlockReplyPipeline`, `getSentMediaUrls()`, and final payload media dedupe.

## Solution Assessment

Yes, at the product level. The narrow maintainable fix belongs in generic block reply sent-media accounting with tests, not in Telegram delivery, because Telegram already supports captioned media sends.
