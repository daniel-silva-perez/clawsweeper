## Summary

Keep this PR open. Current main includes the broader #72397 Feishu parser repair, but it still does not handle the central fallback shape described here: top-level `title` plus nested `elements` arrays containing `tag:"text"` or `tag:"a"` nodes. A newer open PR, #73203, tracks part of the same remaining gap, but it is not merged and does not cover every detail from this PR.

## Best Possible Solution

Keep this PR open until the remaining Feishu parser gap is actually merged. The best path is to land a focused owner-plugin fix in `extensions/feishu/src/send.ts` that flattens nested interactive `elements`, extracts `text` and link nodes, preserves useful title/header text, and adds `getMessageFeishu()` regression tests matching the linked #60380 payload. If #73203 is chosen as the canonical implementation, fold the missing title/link coverage from this PR into that review before closing either item.
