## Summary

Keep open. Current main includes the later #72397 Feishu parser work and a v2026.4.27 changelog entry crediting this PR, but the exact top-level fallback shape described here, `{ title, elements: [[...]] }`, still falls through to `[Interactive Card]`. The contribution remains useful, though the best path is to port the missing case onto the current parser rather than merge the stale branch verbatim.

## What This Changes

The PR extends Feishu interactive-card parsing and tests to extract titles, text, plain-text nodes, and links from nested post-style fallback `elements` arrays.

## Best Possible Solution

Port the missing top-level post-format fallback into the current Feishu parser, ideally by treating `{ title, elements: [[...]] }` as post-like rich-text content while preserving the existing schema 1.0, schema 2.0, i18n, template-variable, and wrapped `post` handling from #72397.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction: send the PR's `msg_type: "interactive"` payload with top-level `title` and nested `elements`; current main rejects the inner array as an interactive element and the post fallback does not map `elements` to rich-text `content`.

## Solution Assessment

No, not as a direct merge into current main. The parsing direction is right, but current main has since been refactored, so the safer solution is a small current-main patch that ports this exact fallback shape into the existing parser instead of replacing the newer #72397 paths.
