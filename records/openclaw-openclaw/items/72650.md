## Summary

Keep this PR open. Current main still rejects the reported symlinked `~/.openclaw` setup, so the branch addresses a real remaining bug, but the proposed trusted-hop code still has an actionable correctness issue around dangling symlinks that should be fixed before merge.

## What This Changes

The PR changes exec approvals path validation to allow one trusted immediate `.openclaw` symlink under the home directory when its target is user-owned and not group/other writable, and adds focused store-helper coverage for that layout.

## Best Possible Solution

Keep the PR open, address the dangling-symlink error path, then rebase and merge the narrow trusted-hop implementation with focused exec approvals tests and the normal changed-surface gate. The linked bug should close only after that implementation lands on main.

## Reproduction Assessment

Yes. The issue has a high-confidence reproduction path: with `OPENCLAW_HOME` resolving to a real home and `.openclaw` as a symlink, current main reaches `assertNoSymlinkPathComponents()` and throws on the first symlinked component before `exec-policy preset yolo` can write the local approvals file.

## Solution Assessment

Mostly yes, but not merge-ready as written. The one-hop, owner/mode-gated allowance is the narrow maintainable direction for the reported dotfile-manager layout, but the new `realpathSync` call should not share the ENOENT-swallowing catch intended for missing `lstatSync` path components.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Rethrow errors from the trusted symlink branch:** `src/infra/exec-approvals.ts:292-293`
  - body: `realpathSync(current)` now runs inside the same `try` block that swallows `ENOENT` from `lstatSync`. If `.openclaw` is a dangling symlink, the target ownership/mode validation is skipped and the command fails later with a less precise mkdir/stat error instead of the hardening error from this branch. Keep the missing-component swallow scoped to `lstatSync`, and rethrow errors once the code has identified a symlink.
  - confidence: 0.86
