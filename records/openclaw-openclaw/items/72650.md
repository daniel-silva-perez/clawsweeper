## Summary

Keep open. Current main still rejects the reported dotfile-managed `~/.openclaw` symlink layout, and this PR is aimed at the right bug, but the proposed symlink branch still has a concrete dangling-symlink error-path bug and overlaps an open broader draft replacement.

## What This Changes

The PR changes exec approvals path validation to allow one trusted immediate `.openclaw` symlink under the home directory when its target is user-owned and not group/other writable, and adds focused store-helper coverage for that layout.

## Best Possible Solution

Keep the PR open, fix the dangling-symlink error path, then either merge this narrow trusted-hop implementation with focused exec approvals tests or explicitly supersede it with the broader #72713 path if maintainers want the macOS/docs hardening bundle.

## Reproduction Assessment

Yes. The high-confidence reproduction is a normal home directory with `.openclaw` as a symlink, then `openclaw exec-policy preset yolo`; current main reaches `assertNoSymlinkPathComponents()` and throws on that symlinked component before writing the local approvals file.

## Solution Assessment

Mostly yes, but not merge-ready as written. A one-hop, owner/mode-gated allowance is the narrow maintainable fix for the reported dotfile-manager layout, but `realpathSync(current)` should not share the `ENOENT` swallow intended only for missing `lstatSync` path components.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Rethrow errors from the trusted symlink branch:** `src/infra/exec-approvals.ts:282-283`
  - body: `realpathSync(current)` now runs inside the same `try` block that swallows `ENOENT` from `lstatSync`. If `.openclaw` is a dangling symlink, target ownership/mode validation is skipped and the command fails later with a less precise mkdir/stat error instead of this hardening branch's failure. Keep the missing-component swallow scoped to `lstatSync`, and rethrow once the code has identified a symlink.
  - confidence: 0.86
