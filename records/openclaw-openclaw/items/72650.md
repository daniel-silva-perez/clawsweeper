## Summary

Keep this PR open. Current main still rejects the reported dotfile-managed `.openclaw` symlink layout, and the PR takes a useful narrow direction, but it is not merge-ready because one new symlink-resolution error path is swallowed by the existing missing-component catch and the user-facing fix is missing the required changelog entry.

## What This Changes

The PR changes exec approvals path validation to allow one trusted immediate `.openclaw` symlink under the home directory and adds focused regression coverage for that dotfile-manager layout.

## Best Possible Solution

Keep the PR open, ask for the dangling-symlink catch scope and changelog entry to be fixed, then choose whether to land this narrow core implementation or replace it with the broader #72713 approach if maintainers want the macOS/node-host policy handled in the same change.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: `openclaw exec-policy preset yolo` writes through `saveExecApprovals()`, whose directory validation rejects `.openclaw` as soon as `lstatSync()` reports a symlink under the resolved home root. The PR finding also has a focused runtime reproduction: a dangling `.openclaw` symlink makes `realpathSync()` throw `ENOENT`, which the PR currently swallows in the missing-component catch.

## Solution Assessment

No, not as written. The one-hop, owner/mode-gated allowance is the narrow maintainable direction for the reported GNU Stow/chezmoi layout, but symlink resolution and target validation should not share the `ENOENT` catch intended only for missing `lstatSync()` path components, and the user-facing fix needs a changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Rethrow trusted symlink resolution errors:** `src/infra/exec-approvals.ts:282`
  - body: `fs.realpathSync(current)` now runs inside the same `try` block that intentionally swallows `ENOENT` from `lstatSync` for missing path components. Scope the `ENOENT` swallow to `lstatSync`, or rethrow once the code has identified a symlink.
