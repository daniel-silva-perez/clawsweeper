Keep this issue open. Current main still restricts `bootstrap-extra-files` to recognized bootstrap basenames, so arbitrary workspace files such as `hooks/REGISTRY.yaml` are still rejected. The open implementation PR #73917 is the right active path to review rather than closing or duplicating this tracker.

## Best Possible Solution

Keep this issue open as the canonical tracker.
