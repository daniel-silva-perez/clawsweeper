## Summary

Keep this PR open. Current main still does not implement the documented `callback_data: <value>` fallback for generic Telegram inline button callbacks, so the intended bug fix remains relevant. The patch is plausible but needs revision against current main to preserve native/slash-command callback routing and add a regression test for the exact prefixed text.

## Best Possible Solution

Keep the PR open and revise it against current main: prefix generic non-command callback fallback text as `callback_data: ${data}`, preserve native `tgcmd:` callbacks and intentional slash-command callbacks, and tighten/add Telegram callback tests so one assertion proves the prefixed agent text while separate assertions prove command callbacks still route as commands.
