## Summary

Keep open. The PR targets a real Telegram mismatch: docs and the linked bug expect inline callback clicks to reach the agent as `callback_data: <value>`, while current main still forwards raw generic callback data. It should not merge as-is until it is rebased onto current main and preserves the newer native-command callback decoding path while prefixing only the generic fallback.

## What This Changes

The PR changes the Telegram callback fallback so inline button callback data is forwarded to the agent with a `callback_data:` prefix.

## Best Possible Solution

Rebase or replace the PR on current main so `parseTelegramNativeCommandCallbackData(data)` still wins for native command buttons, while the non-native fallback sends `callback_data: ${data}` to the agent and has a focused regression test for that exact text.
