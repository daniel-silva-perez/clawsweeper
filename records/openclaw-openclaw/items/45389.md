## Summary

Current main still lacks the requested configurable session-level circuit breaker for consecutive LLM timeouts. It has adjacent protections such as timeout-triggered compaction, in-run retry/failover, provider/profile cooldown behavior, and WebSocket transport cooldowns, but no persisted per-session consecutive-timeout state that pauses or marks a session degraded and emits a notification. Related PR #42933 remains an open implementation candidate, not proof that this issue is fixed.

## Best Possible Solution

Keep this issue open and consolidate it with the active session-breaker work. The best path is to review or rework #42933, or name a canonical tracker, then add a documented per-agent/default timeout breaker threshold that persists per-session consecutive LLM timeout state across heartbeat, cron, and interactive runs; resets on success or explicit operator action; marks or pauses degraded sessions; and emits a sanitized system event/webhook notification.
