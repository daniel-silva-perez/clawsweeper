## Summary

Keep open. Current main has important mitigations for high-context LLM timeouts: timeout-triggered compaction, bounded in-run retries, auth-profile rotation, and model fallback. It still does not implement the requested configurable session-level circuit breaker that tracks consecutive LLM timeouts across runs, pauses/marks the session degraded, and emits a system event or webhook notification. The related open PR #42933 remains the closest implementation candidate rather than proof that this issue can close.

## Best Possible Solution

Keep this issue open and either review/land a reworked #42933 or implement an equivalent generic core/session feature: a configurable per-agent/default consecutive LLM timeout threshold, durable per-session counters across heartbeat/cron/interactive runs, clear reset semantics after success/manual reset, degraded or paused session state, and system-event/webhook or existing failure-notification delivery with focused tests around heartbeat and normal reply retry paths.
