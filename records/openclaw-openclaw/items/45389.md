## Summary

Keep open. Current main has adjacent mitigations for timeout recovery, cron backoff, and Active Memory recall timeouts, but it still does not implement the requested durable session-level LLM timeout circuit breaker with configurable per-agent/default thresholds, persisted per-session timeout state, degraded/paused session handling, and operator notification. The related PR #42933 remains an open implementation candidate, so this should not be closed as implemented or replaced with a duplicate automated repair lane.

## What This Changes

Add a configurable per-agent/default session-level circuit breaker that stops automatic retries after N consecutive LLM timeouts, marks the session degraded, and emits an operator notification.

## Best Possible Solution

Keep this issue open and consolidate it with the active session-breaker work. The desired end state is a documented per-agent/default breaker threshold with persisted per-session timeout state across heartbeat, cron, and interactive runs; success or operator reset clears the streak; tripping the breaker pauses or marks the session degraded; and a sanitized system event or webhook notifies operators.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature request. Source inspection gives a high-confidence missing-capability check: current config schemas, session state, heartbeat/cron paths, and timeout tests lack the requested durable session-level breaker surface.

## Solution Assessment

Unclear for the exact proposed shape. A persisted configurable session-level breaker is the right direction, but maintainers should decide pause/reset/alert semantics and review or rework #42933 rather than treating existing timeout compaction, cron backoff, or Active Memory recall guards as sufficient.
