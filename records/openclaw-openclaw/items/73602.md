## Summary

Keep open. Current main has useful post-release mitigations for WhatsApp socket timing and Telegram transport recovery, but it does not prove the WSL2 regression is fixed. The WhatsApp healthy-with-recent-disconnect status mismatch remains plausible in current code, and Telegram still has the 120s polling-stall watchdog path described by the reporter.

## Best Possible Solution

Keep this issue open for a focused WSL2 channel reliability investigation. Validate the WhatsApp Baileys timing mitigation on WSL2, update WhatsApp status classification so recent flapping is visible even after a brief reconnect, and verify why Telegram can still reach the 120s stall path on the reporter's setup. If the session-lock/model-provider failures reproduce independently, split them into a narrower agent reliability issue.
