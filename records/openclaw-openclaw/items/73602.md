## Summary

Keep open. Current main and v2026.4.29 contain relevant mitigations, but the issue discussion and code evidence still leave the combined WSL2 WhatsApp/Telegram transport regression unvalidated on the affected setup.

## What This Changes

The issue reports a WSL2 regression where WhatsApp Web flaps while status can look healthy, Telegram long polling stalls, and downstream agent/session failures make replies delayed or inconsistent.

## Best Possible Solution

Validate on the affected WSL2 host using current main or a build at least as new as v2026.4.29. If the combined channel symptoms are gone, close later with the release evidence; if not, split the remaining work into focused WhatsApp transport/status, Telegram polling/network, and agent/session follow-ups with fresh logs.
