## Summary

Keep open. This is a recent, concrete WSL2 reliability report against the current release line, and current main shows only partial mitigations: Telegram has watchdog/timeouts/fallbacks but still waits on the 120s stall path described, while WhatsApp has a post-release Baileys timing fix but still can report healthy after a reconnect with a retained lastDisconnect/reconnectAttempts snapshot.

## Best Possible Solution

Keep this issue open for a focused WSL2 reliability investigation. Maintainers should validate the post-release WhatsApp Baileys timing changes on WSL2, make WhatsApp status expose recent flapping or unhealthy reconnect loops even after a brief reconnect, and add targeted diagnostics/regression coverage for WhatsApp 408/EAI_AGAIN/405 plus Telegram getUpdates stalls. If the agent/session lock and provider errors persist independently of transport stalls, split those into a narrower agent-run reliability issue.
