## Summary

Keep open. This is a fresh, concrete WSL2 regression report against v2026.4.26, and current main only partially addresses the symptoms. WhatsApp has an unreleased socket-timing mitigation, but the reported healthy-with-recent-disconnect status mismatch remains plausible. Telegram still carries the same 120s polling-stall watchdog path that matches the reporter's 124s logs, and the relevant stall/timeout files are not materially changed from the release for the core symptom.

## Best Possible Solution

Keep this issue open for a focused WSL2 channel reliability investigation. Validate the unreleased WhatsApp Baileys timing mitigation on WSL2, add status classification for recent WhatsApp flapping even after brief reconnects, and verify why Telegram can still reach the 120s stall path on the reporter's setup. If the session-lock/model-provider failures reproduce independently of channel transport stalls, split them into a narrower agent reliability issue.
