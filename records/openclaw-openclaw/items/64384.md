## Summary

Keep this PR open. Current main still returns early for fresh persisted token totals before the token-threshold check when the transcript-byte trigger is inactive, so the PR's intended fix is not implemented. The linked same-author bug #63892 is still open, and overlapping implementation work such as #66716 is open rather than shipped.

## Best Possible Solution

Keep this PR open until maintainers either rebase and land a minimal fix here or explicitly choose another open implementation such as #66716. The fix should live in `src/auto-reply/reply/agent-runner-memory.ts`, allow fresh persisted totals above the token threshold to reach `shouldRunPreflightCompaction`, preserve the active-transcript byte trigger, and add regression tests for fresh totals above and below threshold while transcript-byte compaction is inactive.

## Evidence

That happens before `shouldRunPreflightCompaction`, so a fresh persisted total above the threshold is skipped on the normal token path.
