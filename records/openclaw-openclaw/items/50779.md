## Summary

Keep this open. Current main still does not render the direct read-tool image result shape in Control UI chat, and the related open PR #52025 is the direct implementation candidate rather than proof that the bug is fixed.

## What This Changes

Render images returned by read/tool results inline in Control UI chat, or provide an obvious preview/thumbnail affordance in the tool output UI.

## Best Possible Solution

Control UI should safely preserve structured image information from tool-result history and live tool-result updates, render direct `{ type: "image", data, mimeType }` read-tool blocks as visible previews or obvious thumbnails, keep existing local-media/CSP protections intact, and add focused regression tests for both history and live stream paths.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: render or stream a tool-result message whose content includes `{ type: "image", data: "...", mimeType: "image/png" }`; current extraction does not produce an image, and the live tool stream turns structured results into text-only tool output.

## Solution Assessment

Unclear until #52025 is reviewed. The right direction is a narrow Control UI/tool-stream fix with regression coverage, not a broad media feature program or a remote-image policy change.
