## Summary

Keep this issue open. Current main still does not fully render read-tool returned images inline in Control UI chat: read/tool image results use structured `{ type: "image", data, mimeType }` blocks, while the Control UI renderer only recognizes nested source/url-style image blocks, tool-message images are shown only inside the expanded tool disclosure, and the live tool stream flattens structured tool output to text. PR #52025 is still open and directly targets this behavior, so the best outcome is to review/land that PR or a focused replacement rather than close this issue before merge.

## Best Possible Solution

Keep this issue open and review/land PR #52025 or a focused replacement. The fix should render read-tool image blocks shaped as `{ type: "image", data, mimeType }`, handle nested tool-result content where applicable, expose an obvious preview outside collapsed tool details, and decide whether live tool events should preserve structured image content. Add focused Control UI regression coverage for read-tool image results without broadening remote image fetching.
