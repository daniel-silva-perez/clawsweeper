## Summary

Keep this issue open. Current main still has the reported Control UI gap: read-tool image results use top-level `{ type: "image", data, mimeType }` blocks, while the chat renderer only turns source/url-style image blocks into renderable images, tool-message images stay inside the collapsed tool disclosure, and the live tool stream flattens structured tool output to text. PR #52025 remains the direct implementation candidate, so this should not be closed before that PR or a replacement lands.

## Best Possible Solution

Keep this issue open and review/land PR #52025 or a focused replacement. The fix should render read-tool image blocks shaped as `{ type: "image", data, mimeType }`, handle nested tool-result content where applicable, expose image previews outside collapsed tool details, and decide whether live tool events should preserve structured image content. Add focused Control UI regression coverage for read-tool image results without broadening remote image fetching.
