## Summary

Keep this issue open. Current main supports several Control UI media paths, but it still does not render the direct tool-result image shape `{ type: "image", data, mimeType }` inline by default, and the focused implementation candidate is still open/unmerged.

## Best Possible Solution

Keep this issue open and review/land #52025 or a refreshed equivalent. The fix should render direct and nested `{ type: "image", data, mimeType }` tool-result blocks visibly inline, preserve accompanying text, cover image-only results and live stream/history reload paths, and keep existing Control UI media safety constraints. The broader #64607 can continue tracking non-image and rich media preview expansion separately.
