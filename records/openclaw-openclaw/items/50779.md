## Summary

Keep this open. Current main still drops the direct read-tool image result shape before Control UI can render it, and the related open PR #52025 is an implementation candidate rather than proof that the bug is fixed.

## What This Changes

The issue asks Control UI chat to show a visible inline preview or clickable thumbnail for images returned by tools such as the read tool.

## Best Possible Solution

Control UI should preserve structured image data from both stored history and live tool-result updates, render direct `{ type: "image", data, mimeType }` blocks as safe inline previews or obvious thumbnails, and keep existing managed-media and local-media security checks intact with focused regression tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists by rendering or streaming a tool-result message whose content includes `{ type: "image", data: "...", mimeType: "image/png" }`; current extraction and live streaming paths do not surface that image as a visible preview.

## Solution Assessment

Unclear whether the open PR is the best implementation until it is reviewed. The best direction is a narrow Control UI/tool-stream fix with regression coverage, not a broad remote-media or all-media feature change.
