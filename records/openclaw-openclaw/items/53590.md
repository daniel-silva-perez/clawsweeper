## Summary

Keep open. Current main improves runtime unknown-model errors and has narrow hooks model validation, but it still does not provide the requested generic startup or doctor warning for invalid configured model IDs after updates.

## What This Changes

The issue requests gateway startup warnings, doctor validation, and changelog guidance when configured model IDs become invalid after OpenClaw/npm updates.

## Best Possible Solution

Add one shared configured-model diagnostics layer used by both Gateway startup and openclaw doctor: collect configured model refs from defaults, fallbacks, agents, hooks, channels, tools, and media settings; resolve aliases consistently; warn only when catalog evidence is authoritative; and document/changelog retired or renamed shipped model slugs when OpenClaw changes them.
