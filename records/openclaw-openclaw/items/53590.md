## Summary

Keep open. Current main has better runtime unknown-model errors and narrow Gmail hook/onboarding diagnostics, but it still does not generically warn at gateway startup or in doctor when configured primary, fallback, per-agent, channel, or tool model refs are absent from an authoritative provider catalog. The open related PR #40355 overlaps the doctor/default-primary subset only, so this should stay with maintainers rather than being closed or duplicated by automation.

## What This Changes

Request startup, doctor, and changelog diagnostics for configured model IDs that become invalid after provider slug renames or retirements.

## Best Possible Solution

Add one shared configured-model diagnostics layer used by both gateway startup and openclaw doctor: collect configured refs across defaults, fallbacks, agents, channels, hooks, and tool/media settings; resolve aliases with runtime-equivalent semantics; compare only against authoritative catalog evidence; emit migration-oriented warnings; and preserve custom/dynamic provider pass-through when catalog authority is unavailable or intentionally incomplete.
