## Summary

Keep open. Current main has clearer runtime unknown-model errors and a narrow Gmail hook model check, but it still lacks the requested generic startup and doctor diagnostics for invalid configured model IDs after updates. The open related PR #40355 overlaps only part of the doctor/default-primary area, so this should stay open for maintainer review rather than being closed or queued as a duplicate automation fix.

## What This Changes

The issue requests proactive warnings in gateway startup, doctor validation, and changelog guidance when configured model IDs become invalid after an OpenClaw update.

## Best Possible Solution

Add one shared configured-model diagnostics layer that both gateway startup and openclaw doctor use: collect refs from defaults, fallbacks, agents, subagents, hooks, channels, tools, and media settings; resolve aliases the same way runtime does; check only authoritative catalog evidence; emit migration-oriented warnings; and preserve custom or dynamic provider pass-through where catalog authority is incomplete.
