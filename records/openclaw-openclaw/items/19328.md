## Summary

Keep this PR open. Current main still constructs `nextEntryPatch` from a cached session entry, including `modelOverride` and `providerOverride`, then merges that stale patch over the fresh store row. The PR remains a focused candidate for the reported model override race, and the related #70575 is a narrower follow-up rather than a superseding fix.

## Best Possible Solution

Keep this PR open and review or land it, or land an equivalent focused fix in `src/gateway/server-methods/agent.ts` that prevents stale cached `entry` fields from being merged over the fresh store row. Preserve targeted regression coverage for stale `loadSessionEntry` data racing with a fresh store row containing `modelOverride`/`providerOverride`, and keep #70575 as the separate delivery-field follow-up.
