## Summary

Keep this PR open. Current main still contains the stale cached session-entry writeback pattern the PR targets: `agent` reads a cached session entry, copies `modelOverride` and `providerOverride` into `nextEntryPatch`, then merges that patch over the freshly loaded store row inside `updateSessionStore`. The PR is a focused two-file gateway change with regression tests, and the linked discussion already separates the residual delivery-field race into #70575.

## Best Possible Solution

Keep this PR open and review or land it, or land an equivalent focused fix in `src/gateway/server-methods/agent.ts` that prevents cached `entry` fields such as `modelOverride` and `providerOverride` from being merged over the fresh store row. Keep targeted regression coverage for a stale `loadSessionEntry` result racing with a fresh store row containing model override fields.
