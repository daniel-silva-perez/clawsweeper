## Summary

Keep this PR open. Current main still contains the stale cached session-entry writeback pattern the PR targets: the gateway agent handler reads a cached session entry, builds a full patch containing `modelOverride` and `providerOverride` from that cached entry, then merges that stale patch over the fresh store row inside `updateSessionStore`. The PR remains a focused implementation candidate, with no obvious supply-chain-sensitive file changes in the provided PR file list.

## Best Possible Solution

Keep this PR open and review or land it, or land an equivalent focused fix in `src/gateway/server-methods/agent.ts` that prevents cached `entry` fields such as `modelOverride` and `providerOverride` from being merged over the fresh store row. Keep the regression tests for the stale-cache/fresh-store race, and leave the narrower delivery-field follow-up to #70575 unless maintainers intentionally broaden scope.
