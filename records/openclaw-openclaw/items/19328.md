## Summary

Keep this PR open. Current main still has the stale cached session-entry writeback pattern this PR targets: the agent handler builds a full session patch from cached `entry` fields, including `modelOverride` and `providerOverride`, then merges that patch over the fresh store row loaded inside `updateSessionStore`. The PR is still a focused implementation candidate, and the existing maintainer/Codex review comment on April 26 also explicitly kept it open for this reason.

## Best Possible Solution

Keep this PR open and review or land it, or land an equivalent focused fix in `src/gateway/server-methods/agent.ts` that stops cached `entry` fields such as `modelOverride` and `providerOverride` from being merged over the fresh store row. Keep the stale-cache regression tests from the PR, and leave the narrower residual delivery-field follow-up to #70575.
