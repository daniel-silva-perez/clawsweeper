## Summary

Keep this PR open. Current main still builds a session patch from the cached `entry` and merges it over the fresh store row inside `updateSessionStore`, including `modelOverride` and `providerOverride`. Main has newer subagent runtime-model persistence, but it does not implement this PR's intended stale-writeback fix or add the proposed regression coverage.

## Best Possible Solution

Keep this PR open and review or land it, or land an equivalent focused change in `src/gateway/server-methods/agent.ts` that stops cached `entry` fields such as `modelOverride` and `providerOverride` from being merged over the fresh store row. Keep the stale-cache/fresh-store regression tests in `src/gateway/server-methods/agent.test.ts`, and leave the narrower delivery-field follow-up to #70575 unless maintainers intentionally broaden scope.
