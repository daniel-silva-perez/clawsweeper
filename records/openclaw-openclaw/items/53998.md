## Summary

Keep open. Current main still does not solve the reported secrets-audit false positives or the fatal mixed auth-profile apply path. The provided issue timeline also shows #54141 as an open, unmerged PR with closing intent for this issue, so the issue should remain open until that PR or a successor lands. Current main does have one-way scrub machinery for targeted apply plans, but that does not close the central audit and configure failure reported here.

## Best Possible Solution

Keep this issue open while #54141 and/or #64344 are reviewed. The best maintainer path is to land a focused fix that recognizes bare `$VAR` and `${VAR}` consistently where OpenClaw treats env-template strings as SecretRefs, prevents auth-profile ref findings from falling through to `PLAINTEXT_FOUND`, and adds regression tests for config and auth-profile audit paths. Then handle the remaining `secrets configure` mixed OAuth/static profile behavior, scrub semantics, and rotation guidance as explicit follow-up work if they are outside the chosen audit fix scope.
