## Summary

Keep open. Current main still matches the reported secrets-audit false-positive path for bare `$VAR` environment references, and the auth-profile audit path can still record a SecretRef and then fall through to `PLAINTEXT_FOUND`. The configure/apply mixed-profile type mismatch is also still fatal. The provided context lists #54141 as an open, unmerged PR with closing intent for this issue, so this issue should remain open until that PR or a replacement lands.

## What This Changes

The issue requests that secrets audit distinguish `$VAR`/`${VAR}` environment references from literal plaintext secrets, avoid aborting configuration migration on out-of-scope OAuth profiles, and provide clearer scrub/rotation guidance after migration.

## Best Possible Solution

Keep this issue open while #54141 is reviewed and either landed or closed. The desired end state is a focused shipped fix for bare `$VAR` and auth-profile ref audit classification, followed by separate maintainer-scoped follow-ups for the mixed OAuth/static configure failure, scrub/rotation messaging, and the related resolved-config `${ENV_VAR}` audit path if those remain after the narrow fix.
