## Summary

Keep open. Current main still matches the reported noisy secrets-audit behavior for bare `$VAR` strings and auth-profile refs, and `secrets apply/configure` still throws on mixed auth-profile type mismatches. The issue also has an open PR with closing intent, so it should remain open until that PR or a replacement lands.

## What This Changes

The issue asks for secrets audit/configure to treat environment-variable references as non-plaintext, scrub migrated plaintext credentials, and handle OAuth/static auth-profile mismatches without aborting valid migrations.

## Best Possible Solution

Review and land #54141 or a focused replacement for the bare `$VAR` and auth-profile audit false positives, then keep or open narrower follow-ups for the mixed OAuth/static configure failure, scrub/rotation messaging, and resolved-config `${ENV_VAR}` audit path where they remain unresolved.
