## Summary

Keep this PR open. Current main still lacks the PR’s no-op sentinel suppression for heartbeat/system-event replies, and the linked #73149 leak remains until this PR or an equivalent fix lands.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #73149. Maintainers should review and land the focused sentinel-suppression approach, with targeted validation that exact no-op tokens are suppressed before outbound delivery for heartbeat/system-event handoffs while meaningful exec summaries still deliver.
