## Summary

Keep this PR open. Current main still does not implement the PR’s intended no-op sentinel suppression for heartbeat/system-event replies, and the linked user-visible leak remains until this PR or an equivalent fix lands.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #73149. Maintainers should review and land the focused sentinel-suppression approach, ensuring exact no-op tokens are suppressed before outbound delivery for heartbeat/system-event handoffs while meaningful exec summaries remain deliverable, with the proposed regression coverage and targeted gates.
