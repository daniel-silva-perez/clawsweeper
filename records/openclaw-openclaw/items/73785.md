## Summary

Keep this PR open. Current main still has the heartbeat exec-completion exception that can re-expose exact `HEARTBEAT_OK`, lacks the proposed centralized no-op sentinel helper, and does not have the new regression coverage for exact `NO_REPLY`/`NO_NEW_AUDIO`/`SESSION_WATCHDOG_OK` or exec system-event suppression. The PR remains the active implementation candidate for #73149.

## Best Possible Solution

Keep this PR open as the focused implementation candidate for #73149. Maintainers should review and land this or an equivalent runtime-level suppression path that drops exact no-op sentinel replies before outbound delivery for heartbeat/system-event handoffs while preserving meaningful exec summaries, with the proposed regression tests or equivalent coverage.
