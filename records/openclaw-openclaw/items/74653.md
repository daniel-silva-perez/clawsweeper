## Summary

Keep open. The reported doctor warning path is still valid, and this PR is not correct as-is because it adds the skipped-probe guard after the gateway adapter has already discarded `embedding.checked: false`. The intended product behavior is right, but the fix needs to propagate the gateway payload's checked state through `probeGatewayMemoryStatus` and cover that adapter path in tests.

## What This Changes

The PR changes `noteMemorySearchHealth` to suppress key-optional memory-provider warnings when a skipped gateway probe is passed in, and updates/adds lmstudio and ollama skipped-probe unit tests.

## Best Possible Solution

Preserve the gateway contract by propagating `payload.embedding.checked === false` through `probeGatewayMemoryStatus`, then keep the renderer suppression for skipped probes while still warning for checked-not-ready failures. Add an adapter-level regression test so the skipped payload from `doctor.memory.status` reaches `noteMemorySearchHealth` as inconclusive rather than failed.
