## Summary

Keep this PR open. Current main still has the BlueBubbles monitor typing restart, start, and stop paths calling `sendBlueBubblesTyping` without the PR's `privateApiEnabled` guards, and current monitor tests still lack the disabled-Private-API regression case. The merged #65447 fixed send/attachment Private API cache behavior, not these monitor typing callbacks, and the open #70799 overlaps but does not implement this PR on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either land the monitor-processing guards and disabled-path regression test if maintainers accept the unknown-cache trade-off, or request a small revision that preserves an explicit unknown-state fallback and reconciles this monitor-level work with the helper-level guard proposed in #70799.
