## Summary

Keep open. Current main still does not expose the requested public `transform_tool_result` hook for third-party plugins. OpenClaw now has async pre-model tool-result middleware, and bundled `tokenjuice` proves the timing is supported, but that seam is explicitly bundled-only; public hooks remain `before_tool_call`, void `after_tool_call`, and synchronous transcript-oriented `tool_result_persist`.

## Best Possible Solution

Keep this issue open until maintainers either add a documented, trust-gated public hook for post-execution/pre-model tool-result mutation, or explicitly decide and document that pre-model result rewriting must remain bundled-only. The clean implementation path is to build on the existing `AgentToolResultMiddleware` ordering and validation model, define external-plugin trust/prompt-injection policy, specify ordering relative to `after_tool_call` and `tool_result_persist`, and add Pi/Codex runtime tests proving the model sees the transformed result in the same turn.
