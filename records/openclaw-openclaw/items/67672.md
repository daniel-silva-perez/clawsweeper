## Summary

Keep open. Current main still stops Slack Socket Mode by setting the private shutdown flag and awaiting Bolt app.stop(), but it does not explicitly await SocketModeClient.disconnect(), so the PR's core leak fix is not already implemented. The report is plausible against the locked Slack dependency behavior and the current stale-socket restart path, though the branch likely needs a small rebase/adaptation because current main moved gracefulStopSlackApp into provider-support.ts.

## What This Changes

The PR adds a Slack SocketModeClient accessor/disconnect helper, calls it before Bolt app stop during Slack cleanup, and removes a redundant SlackProbe cast after review feedback.

## Best Possible Solution

Land a narrow Slack-owned cleanup on current main that awaits the underlying SocketModeClient.disconnect() before Bolt app.stop() in the shared Slack cleanup helper, keeps the stopAccount hook out, preserves the redundant-cast cleanup, and adds a regression test covering disconnect call order/awaiting during abort or stale-socket restart cleanup.
