## Summary

Keep this PR open. Current main still does not contain the optional macOS LaunchAgent watchdog runbook or the updating-guide cross-reference, so the PR is not implemented on main. The diff remains docs-only and does not introduce code execution paths in the repository, but the proposed runbook still needs maintainer review because the documented `openclaw health --json` gate does not fail on degraded channel state after the Gateway RPC succeeds, and the LaunchAgent example may need profile/config environment propagation for non-default macOS services.

## Best Possible Solution

Keep the PR open for maintainer docs review. The best path is to revise `docs/platforms/mac/bundled-gateway.md` so the watchdog uses a predicate that actually fails for the intended readiness/degraded condition, carries the right profile/config environment for named LaunchAgents, and remains clearly optional/operator-managed; then merge it or explicitly defer it to the productized watchdog discussion if maintainers prefer not to document this runbook in core docs.

## Risks / Open Questions

- The PR's documentation example could still mislead operators if it advertises recovery for "process is alive but unhealthy" while gating only on `openclaw health --json` exit status, which current source treats as successful once the Gateway RPC is reachable.
- For named-profile macOS services, the LaunchAgent watchdog example may probe or restart the wrong service unless it carries `OPENCLAW_PROFILE` and any relevant config/state-dir environment through the watchdog job.
- Security review found no repository supply-chain or CI regression because the PR is docs-only, but the documented LaunchAgent runs local repair/restart commands periodically, so correctness of scope, PATH, profile, and probe semantics matters before merge.
