## Summary

Keep this PR open. Current main still lacks the optional macOS LaunchAgent watchdog runbook and updating-guide cross-reference, and the PR is a focused docs-only change. The original launchd PATH footgun was addressed in the PR, but the remaining review concerns around the health predicate and profile/config environment are still source-backed and need maintainer review before merge.

## Best Possible Solution

Keep the PR open for maintainer docs review. The best path is to revise the macOS runbook so the watchdog predicate either matches the intended degraded-but-responsive condition or is clearly framed as reachability-only, propagate OPENCLAW_PROFILE/OPENCLAW_STATE_DIR/OPENCLAW_CONFIG_PATH where relevant, and keep the section explicitly optional/operator-managed. If maintainers prefer a productized watchdog instead, defer this docs runbook to the broader open watchdog work such as #46502 rather than closing it as already implemented.

## Risks / Open Questions

- The proposed runbook could advertise recovery for process-alive-but-unhealthy cases while gating on a command that exits successfully for degraded-but-reachable channel states.
- The separate LaunchAgent example may probe or restart the default service instead of a named-profile or custom config/state service unless it carries the same environment context as the managed Gateway service.
- Although the PR is docs-only, the documented watchdog periodically runs `doctor`, `doctor --fix`, and `gateway restart`, so the predicate, target profile, and restart scope need maintainer review before shipping.
