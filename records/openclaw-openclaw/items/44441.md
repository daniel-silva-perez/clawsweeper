## Summary

Keep this PR open. Current main still lacks the optional macOS LaunchAgent watchdog runbook and the updating-guide cross-reference, while the PR remains focused docs-only work. The author already addressed the original launchd PATH footgun, but the later health-predicate and profile/config environment concerns are still source-backed and need maintainer review before this documentation should ship.

## Best Possible Solution

Keep the PR open for maintainer docs review. The best path is to revise docs/platforms/mac/bundled-gateway.md so the watchdog predicate matches the intended degraded-but-responsive condition, carries profile/config/state context for named LaunchAgents, and remains clearly optional/operator-managed; then merge it or explicitly defer it to the broader productized watchdog discussion if maintainers do not want this runbook in core docs.

## Risks / Open Questions

- The proposed docs could mislead operators if they advertise recovery for process-alive-but-unhealthy cases while gating on a command that exits successfully after Gateway RPC reachability.
- For named-profile or custom state/config macOS services, the separate watchdog LaunchAgent may probe or restart the wrong service unless the docs carry OPENCLAW_PROFILE and relevant config/state environment through the watchdog job.
- Although the PR is docs-only, the documented LaunchAgent periodically runs local doctor, doctor --fix, and gateway restart commands, so probe semantics, PATH, profile, and scope need maintainer review before merge.
