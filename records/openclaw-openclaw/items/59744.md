## Summary

Keep open. Current main still drops the Discord attachment filename when saving inbound media, and an open implementation PR already references this issue with closing syntax but has not merged. The right cleanup path is to review and land or replace that PR, not close the issue early.

## Best Possible Solution

Review and land a cleaned, up-to-date version of PR #59783, or apply the equivalent Discord plugin change directly: pass attachment.filename and sticker candidate.fileName into saveMediaBuffer as originalFilename and keep regression coverage asserting the fifth argument. After that merges, GitHub can close this issue as implemented.
