## Summary

Keep this PR open. Current main does not implement `gateway-probe`, and the PR is a substantive optional observability/Kafka plugin rather than a spam or low-signal drive-by. It is plugin-shaped, but a conservative ClawHub close is not strong enough because current public plugin surfaces do not fully cover the PR's stated inputs: public diagnostics filter out `log.record` and trusted/internal events, internal diagnostic service access is granted only to bundled diagnostics exporters, and there is no production `registerLogTransport` SDK export on current main.

## Best Possible Solution

Do not auto-close this PR in maintainer cleanup. Ask for maintainer product/security judgment: either decline bundled promotion and request an external ClawHub/npm `gateway-probe` plugin plus a narrow public diagnostic/log SDK proposal, or explicitly accept a bundled observability plugin after updating it to current main’s SDK contracts, resolving the log/diagnostic access model, and reviewing the Kafka dependency/egress path.
