## Summary

Keep this PR open. Current main still does not implement an equivalent `gateway-probe` plugin, and the proposal is not safely closeable as ClawHub-only because ordinary plugins can use the session/tool/model/gateway hooks but cannot currently consume the selected structured app-log or trusted internal diagnostic stream that the PR is built around. The author also replied that bot feedback was addressed, so the PR appears to be an active implementation candidate rather than abandoned spam or a duplicate cleanup target.

## Best Possible Solution

Keep this PR open for maintainer product and security judgment. The best path is either to accept a bundled observe-only gateway telemetry plugin after rebase and focused security review, or redirect it to ClawHub after adding a narrow public SDK seam for structured app logs and trusted/internal diagnostic subscriptions so an external plugin can faithfully implement the proposed event catalog.

## Risks / Open Questions

- Do not close as implemented: current main has no equivalent bundled gateway-probe plugin or event catalog.
- Do not close as ClawHub-only yet: an external plugin cannot currently observe the same structured app-log and trusted/internal diagnostic stream without a new, reviewed SDK seam.
- The PR adds optional Kafka egress and a new runtime dependency, so the right next step includes security/supply-chain review for dependency provenance, redaction guarantees, config handling, and failure behavior.
