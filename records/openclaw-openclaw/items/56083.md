## Summary

Keep this PR open. Current main still does not implement the intended cron announce delivery behavior: isolated cron direct delivery builds text/media payloads, strips silent reply tokens, and sends them to outbound delivery without resolving or applying `messages.responsePrefix`. Normal reply routing does apply the effective response prefix, so this PR is still addressing a real consistency gap rather than obsolete work.

## Best Possible Solution

Review and land this PR after rebasing/adapting it to current main, or apply an equivalent main-branch fix in `src/cron/isolated-agent/delivery-dispatch.ts` that resolves the effective `messages.responsePrefix` for the cron delivery target, prefixes text cron announce payloads idempotently, leaves media-only payloads unchanged, keeps awareness events unprefixed, and adds focused regression coverage for global, channel/account, empty, already-prefixed, short-prefix, trailing-whitespace, and synthesizedText fallback cases.
