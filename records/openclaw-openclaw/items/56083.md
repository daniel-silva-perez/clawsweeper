## Summary

Keep this PR open. Current main still sends isolated cron announce/direct delivery payloads through silent-token normalization and `deliverOutboundPayloads` without resolving or applying `messages.responsePrefix`, while normal reply routing does apply the effective prefix. The PR is still addressing a real consistency gap rather than obsolete work.

## Best Possible Solution

Review and land this PR after rebasing/adapting it to current main, or apply an equivalent fix in `src/cron/isolated-agent/delivery-dispatch.ts`: resolve the effective `messages.responsePrefix` for the cron delivery target, prefix text announce payloads idempotently, leave media-only payloads and awareness events unchanged, preserve silent-token filtering, and keep regression coverage for global/channel/account prefixes, empty prefix, already-prefixed text, short prefixes, trailing whitespace, and `synthesizedText` fallback.
