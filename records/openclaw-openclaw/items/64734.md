## Summary

Keep open. Current main still carries the old Discord generated thread-title token cap and timeout, and the affected code path still uses those values directly for the background title-generation request, so this PR is not obsolete. The remaining decision is maintainer review of the proposed defaults and validation, not cleanup closure.

## What This Changes

This PR raises Discord generated auto-thread title generation from 512 to 4096 max tokens and from a 10-second to a 60-second timeout, with the matching unit-test expectation updated.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Maintainers should review the 4096-token and 60-second defaults for the Discord plugin, validate with targeted Discord tests and any feasible live reasoning-model check, and treat the broader observability concern in the PR body as a separate follow-up if desired.
