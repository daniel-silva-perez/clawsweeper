## Summary

Keep open. Current main has more runtime-deps and startup mitigations than the reported 2026.4.24-2026.4.26 builds, but it still preserves a live-PID runtime-deps lock path that can block Docker PID-reuse recovery, and open PR #74950 is the focused fix for that remaining legacy lock case.

## What This Changes

The issue reports that Docker/Unraid deployments on 2026.4.24 and 2026.4.25 never serve Gateway health, and that 2026.4.26 doctor --fix installs bundled runtime deps but then hangs instead of recovering.

## Best Possible Solution

Keep this issue open until #74950 or an equivalent true-legacy stale Docker runtime-deps lock fix lands, then validate a Docker/Unraid or focused Docker stale-lock scenario where the Gateway binds /healthz and openclaw doctor --fix exits cleanly. Track any remaining plugin-loader hot loop or doctor orphan behavior under narrower follow-up issues if it persists.
