## Summary

Keep this issue open. Current main has partial, unreleased plugin-runtime-deps and mirror-cache mitigations related to #73532, but the Docker/Unraid gateway health failure and the post-install `openclaw-doctor` hang are not proven fixed. The related hot-loop report is still open in the provided context, and current startup still performs plugin bootstrap before the HTTP server starts listening, so a plugin/runtime-deps stall can still prevent `/healthz` from ever becoming available.

## Best Possible Solution

Keep the issue open and route it through the plugin runtime-deps/startup path. The next maintainer step should reproduce on current main or a 2026.4.27 candidate in Docker/Unraid, correlate with #73532 CPU/mirror traces, inspect what keeps `openclaw-doctor` alive after `repairBundledRuntimeDepsInstallRootAsync()` completes, and decide whether Gateway should expose shallow `/healthz` before plugin bootstrap or bound/defer runtime-deps mirror work in container startup.
