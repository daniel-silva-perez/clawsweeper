Keep this issue open. Current main has related bundled runtime-deps mitigations, but the Docker/Unraid gateway health failure and post-repair doctor hang are not proven fixed: startup still waits for plugin bootstrap/runtime-deps work before the server calls `startListening()`, so a stall there can still keep `/healthz` unavailable. ## What This Changes

The issue reports Docker/Unraid gateway startup health failures in OpenClaw 2026.4.24 and 2026.4.25, plus a 2026.4.26 `doctor --fix` hang after bundled plugin runtime dependency repair.
