## Summary

Keep open. Current main contains several bundled runtime-deps mitigations, but the inspected lock code still treats an alive PID as a held lock before considering age fallback, and open PR #74950 is specifically targeting the remaining Docker legacy-lock case. Because Gateway HTTP binding still happens after plugin bootstrap, a runtime-deps stall can still keep `/healthz` unavailable rather than merely making Docker's healthcheck noisy.

## What This Changes

The issue reports Docker/Unraid gateway startup failures and `doctor --fix` hangs across OpenClaw 2026.4.24 through 2026.4.26, with `/healthz` never becoming healthy.

## Best Possible Solution

Keep this issue open until #74950 or an equivalent stale Docker runtime-deps lock fix is merged, then verify a Docker/Unraid or focused Docker stale-lock scenario where the Gateway binds `/healthz` and `openclaw doctor --fix` exits cleanly. If the lock fix does not cover the plugin-loader hot loop or doctor orphan behavior, keep those remaining symptoms tracked under their narrower related items.

## Reproduction Assessment

Yes for a focused reproduction path: the issue gives concrete Docker/Unraid steps, and current source still has a PID-alive lock path that can bypass stale-age fallback. A live Docker/Unraid run was not performed in this read-only review.

## Solution Assessment

No, current main is not yet the complete solution. #74950 is a narrow plausible fix for the remaining legacy Docker lock case, but it is still open and should be reviewed, landed, and validated against Gateway health and doctor process-exit behavior.
