## Summary

Keep open. Current main still rejects empty Slack slash-command invocations and has no `contextAware` config surface, so this PR is not obsolete; however, the branch should not merge as-is because it targets the older Slack file layout and routes fetched channel history through prompt fields that directive parsing can treat as command text. Existing review comments also flagged incomplete context wiring and prompt-injection risk.

## What This Changes

The PR adds a Slack `slashCommand.contextAware` option and changes empty slash-command invocations to fetch recent channel history and use it as the agent prompt.

## Best Possible Solution

Implement this as an opt-in setting in the current Slack plugin: fetch recent history only after access checks, filter eligible messages like existing supplemental context, pass the history as untrusted or supplemental context, keep directive parsing tied to the invoker's actual slash text, and update config schema, generated metadata, docs, and targeted tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Keep recent Slack history out of directive parsing:** `src/slack/monitor/slash.ts:597-598`
  - body: When the empty context-aware path builds `effectivePrompt` from channel history and sends it through the request body fields, current directive resolution can fall back from empty command text to prompt text. A prior channel message containing directive-like text could then alter the invoker's turn. Carry history as filtered untrusted or supplemental context and keep directive parsing tied to the slash text.
  - confidence: 0.89
- **[P3] Remove or wire the unused context history:** `src/slack/monitor/slash.ts:563`
  - body: `contextHistory` is assigned from the formatted channel history but is never read, leaving either dead code or the intended structured-context path incomplete. Remove it if prompt-only was intended, or pass it through the chosen context channel.
  - confidence: 0.82

## Security Review

Status: needs_attention
