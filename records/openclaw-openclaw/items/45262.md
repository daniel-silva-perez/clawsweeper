## Summary

Keep open. Current main still rejects empty Slack slash-command prompts and does not expose `slashCommand.contextAware`; the proposed patch is not merge-ready because it targets stale Slack source paths and sends fetched channel history through prompt/command fields without the current Slack context visibility filtering.

## What This Changes

The PR adds a Slack `slashCommand.contextAware` config/schema/resolver option and changes empty Slack slash-command invocations to fetch recent channel history for the agent prompt.

## Best Possible Solution

If maintainers want this feature, implement it in the current Slack plugin as an opt-in setting that authorizes the slash invoker before reading history, applies the existing Slack supplemental-context filtering rules, keeps command fields tied to the actual slash-command text, and passes fetched messages as untrusted or supplemental context with focused tests and docs.

## Reproduction Assessment

Yes. Current-main behavior is reproducible by invoking the configured Slack slash command with no text and observing the `Message required.` response; the PR-specific regressions are reproducible from the diff because empty context-aware mode writes recent channel history into `Body`/`BodyForAgent`.

## Solution Assessment

No. The feature direction may be useful, but this patch is not the narrowest safe implementation because it targets stale paths, treats recent Slack history as prompt text that can enter directive fallback, skips context filtering, and leaves `contextHistory` unused.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep fetched history out of directive fallback:** `src/slack/monitor/slash.ts:597-598`
  - body: When `contextAware` is enabled with an empty slash command, the patch stores recent Slack history in `Body`/`BodyForAgent` while the actual command text is empty. Directive resolution can fall back to prompt text, so prior channel messages can alter the invoker's turn. Keep history in untrusted or supplemental context and keep command fields tied to the slash text.
  - confidence: 0.89
- **[P2] Apply Slack context visibility filtering:** `src/slack/monitor/slash.ts:565-582`
  - body: This branch maps every returned `conversations.history()` message into the prompt. Current Slack thread history removes current-bot messages and applies the supplemental context visibility policy before exposing fetched history, so this path can expose messages normal Slack context would omit.
  - confidence: 0.78
- **[P3] Remove or wire contextHistory:** `src/slack/monitor/slash.ts:563`
  - body: `contextHistory` is assigned from formatted Slack message history but is never read afterward. Remove it if prompt-only behavior is intended, or pass it through the selected structured context path once the context model is fixed.
  - confidence: 0.82
