## Summary

Keep open. The requested Slack behavior is not implemented on current main, and this PR is still useful as a proposal, but the patch should not merge as-is because it targets the older Slack path layout and routes fetched channel history through prompt fields that downstream directive parsing can treat as command text.

## What This Changes

The PR adds a Slack `slashCommand.contextAware` option and changes empty slash-command invocations to fetch recent channel history and use it as the agent prompt.

## Best Possible Solution

Implement the feature in the current Slack plugin as an opt-in setting that carries recent history as filtered untrusted or supplemental context, keeps command and directive parsing tied to the invoker's actual slash text, and updates config schema, generated metadata, docs, and targeted Slack/directive tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep recent Slack history out of directive parsing:** `src/slack/monitor/slash.ts:597-598`
  - body: When the empty context-aware slash path builds `effectivePrompt` from channel history and uses it for the request body, current directive resolution can fall back from empty command text to prompt text. A prior channel message containing directive-like text could then affect the invoker's turn. Carry history as filtered untrusted/supplemental context and keep directive parsing tied to the slash text.
  - confidence: 0.89

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete prompt/directive injection risk by treating recent Slack channel history as the slash-command prompt body.

Concerns:

- **[medium] Recent channel history can drive directives:** `src/slack/monitor/slash.ts:597`
  - body: Any channel participant could place directive-like text in recent history; an authorized user invoking the empty context-aware slash command would then send that history through prompt fields that current directive parsing can consult when command text is empty.
  - confidence: 0.88
