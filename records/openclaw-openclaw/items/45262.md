## Summary

Keep open. Current main has not implemented this feature, and the proposed patch is not merge-ready because it targets stale Slack source paths and introduces unsafe recent-history handling around current directive and supplemental-context contracts.

## What This Changes

The PR adds a Slack slash-command `contextAware` config option and changes empty slash-command invocations to fetch recent channel history for the agent prompt.

## Best Possible Solution

If maintainers want this capability, implement it in the current Slack plugin as an opt-in setting that authorizes the slash invoker before reading history, applies the existing Slack supplemental-context visibility rules, keeps command/directive fields tied to the actual slash-command text, and passes fetched messages as untrusted or supplemental context with focused tests and docs.

## Reproduction Assessment

Yes. Current-main behavior is reproducible by invoking the configured Slack slash command with no text and observing the `Message required.` response; the PR-specific regressions are reproducible from the diff because empty context-aware mode places recent channel history into prompt fields.

## Solution Assessment

No. The feature direction may be useful, but this patch is not the best solution because it targets stale paths, skips current context filtering, and lets fetched history become directive-consulted prompt text when the real command text is empty.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep fetched history out of directive parsing:** `src/slack/monitor/slash.ts:597-598`
  - body: When `contextAware` is enabled with an empty slash command, recent Slack history is assigned to `Body` and `BodyForAgent` while the real command text is empty. Directive resolution can fall back to prompt text in that case, so prior channel messages can alter the invoker's turn. Keep fetched history in supplemental or untrusted context and keep command fields tied to the slash text.
  - confidence: 0.9
- **[P2] Apply Slack context visibility filters:** `src/slack/monitor/slash.ts:565-582`
  - body: This branch maps every returned `conversations.history()` message into the prompt. Current Slack thread history removes current-bot messages and applies the supplemental context visibility policy before exposing fetched history, so this path can reveal messages normal Slack context would omit.
  - confidence: 0.8
- **[P3] Remove or wire contextHistory:** `src/slack/monitor/slash.ts:563`
  - body: `contextHistory` is assigned from formatted Slack message history but is never read afterward. Remove it if prompt-only behavior is intended, or pass it through the selected structured context path once the context model is fixed.
  - confidence: 0.82
