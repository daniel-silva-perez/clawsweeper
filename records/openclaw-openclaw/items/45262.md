## Summary

Keep open. Current main still lacks the requested Slack `contextAware` slash-command behavior, and the proposed patch is not merge-ready because it targets the old Slack path and routes fetched channel history through directive-bearing prompt fields. The prior Greptile and ClawSweeper comments correctly identify concrete issues that should be addressed before this feature lands.

## What This Changes

The PR adds a Slack `slashCommand.contextAware` option and lets empty Slack slash-command invocations fetch up to 10 recent channel messages for the agent response.

## Best Possible Solution

If maintainers want this feature, implement it in the current Slack plugin as an opt-in setting: perform access checks before history reads, filter eligible messages through the existing supplemental-context visibility policy, pass history as untrusted or supplemental context rather than command text, and update config schema, generated metadata, docs, and focused tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep recent history out of directive parsing:** `src/slack/monitor/slash.ts:597-598`
  - body: When the empty context-aware path writes recent Slack history into `Body`/`BodyForAgent`, directive resolution can fall back to prompt text when command fields are empty. A prior channel message containing directive-like text could alter the invoker's turn, so history should stay in untrusted or supplemental context and directive parsing should use only the actual slash text.
  - confidence: 0.89
- **[P3] Remove or wire unused context history:** `src/slack/monitor/slash.ts:563`
  - body: `contextHistory` is assigned from the formatted Slack message history but is never read afterward. Remove it if prompt-only behavior is intended, or pass it through the chosen structured context path.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The patch introduces a concrete prompt/directive-injection risk by treating recent Slack channel history as slash-command prompt text.

Concerns:

- **[medium] Recent Slack history can drive directives:** `src/slack/monitor/slash.ts:597`
  - body: Any channel participant could place directive-like text in recent history; an authorized user invoking an empty context-aware slash command would then send that history through prompt fields that current directive parsing can consult when command text is empty.
  - confidence: 0.88
