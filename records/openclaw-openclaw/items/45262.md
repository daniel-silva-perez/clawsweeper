## Summary

Keep open. Current main still lacks the requested Slack `slashCommand.contextAware` behavior, so this PR is not obsolete, but the proposed patch is not merge-ready because it moves recent Slack channel history into the slash-command prompt/command fields and leaves an unused `contextHistory` variable. The active Slack plugin already has a stricter filtered supplemental-context path that a safe implementation should reuse or mirror.

## What This Changes

The PR branch adds Slack `slashCommand.contextAware` config/schema wiring and changes empty Slack slash-command invocations to fetch recent channel history and use it for the agent response.

## Best Possible Solution

If maintainers want this feature, implement it in the current Slack plugin as an opt-in setting that checks channel access before reading history, filters eligible messages through the existing supplemental-context visibility policy, and passes history as untrusted or supplemental context rather than command text. The final change should include config schema/type wiring, generated metadata/docs if required, and focused tests for empty prompts, directive isolation, and history filtering.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep recent history out of command directive fields:** `src/slack/monitor/slash.ts:597-598`
  - body: When the empty context-aware path puts recent Slack history into the slash turn's prompt/command fields, directive resolution can treat prior channel messages as command text. A channel participant could place directive-like text in history and alter the invoker's empty slash turn, so history should stay in untrusted or supplemental context and directive parsing should use only the actual slash text.
  - confidence: 0.89
- **[P3] Remove or wire contextHistory:** `src/slack/monitor/slash.ts:563`
  - body: `contextHistory` is assigned from the formatted Slack message history but is never read afterward. Remove it if prompt-only behavior is intended, or pass it through the chosen structured context path once the context model is fixed.
  - confidence: 0.82

## Security Review

Status: needs_attention
