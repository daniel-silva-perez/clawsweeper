## Summary

Keep open. Current main still does not implement Slack slash-command context-aware empty prompts, but this PR is not merge-ready because it routes recent Slack channel history into command/prompt fields that downstream directive handling can parse, and the branch targets the older Slack source layout rather than the current plugin path. The prior Greptile and ClawSweeper comments correctly identified the main safety and dead-code issues.

## What This Changes

The PR adds a Slack `slashCommand.contextAware` config/schema/resolver option and changes empty slash-command invocations to fetch recent Slack channel history and use it as the agent prompt.

## Best Possible Solution

If maintainers want this feature, implement it in the current Slack plugin as an opt-in setting that checks channel access before reading history, filters eligible messages through the same visibility policy used for Slack supplemental/thread context, and passes history as untrusted or supplemental context while keeping command/directive fields limited to the actual slash-command text. The change should include config schema/type/generated metadata/docs updates and focused tests for empty prompts, directive isolation, and history filtering.

## Reproduction Assessment

Yes. Current-main behavior is reproducible by invoking the configured Slack slash command with no text: the handler returns `Message required.`. The PR-specific regression is reproducible from the provided diff because the empty context-aware path writes history-derived text into command/prompt fields that directive parsing reads.

## Solution Assessment

No. The feature direction may be useful, but this patch is not the safest maintainable solution because it treats recent channel history as command text and leaves dead `contextHistory` wiring; a safer version should isolate history as untrusted/supplemental context and port to the current plugin layout.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep recent history out of command fields:** `src/slack/monitor/slash.ts:597-598`
  - body: When the empty context-aware path writes recent Slack history into the slash turn's prompt/command fields, downstream directive parsing can treat prior channel messages as command text. A channel participant could place directive-like text in history and alter the invoker's empty slash turn, so history should stay in untrusted or supplemental context and directive parsing should use only the actual slash text.
  - confidence: 0.89
- **[P3] Remove or wire contextHistory:** `src/slack/monitor/slash.ts:563`
  - body: `contextHistory` is assigned from the formatted Slack message history but is never read afterward. Remove it if prompt-only behavior is intended, or pass it through the chosen structured context path once the context model is fixed.
  - confidence: 0.82
