## Summary

Keep open. Current main still lacks the requested Slack context-aware slash-command behavior, and the proposed patch is not merge-ready because it routes recent channel history through prompt fields that current directive parsing can consult when command text is empty.

## What This Changes

The PR adds a Slack `slashCommand.contextAware` option and changes empty Slack slash-command invocations to fetch up to 10 recent channel messages for the agent response.

## Best Possible Solution

Add the feature, if desired, in the current Slack plugin as an opt-in setting: perform access checks before fetching history, filter eligible messages using the existing supplemental-context policy, pass history as untrusted or supplemental context rather than directive-bearing command text, and update config schema, generated metadata, docs, and targeted tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Keep recent Slack history out of directive parsing:** `src/slack/monitor/slash.ts:597-598`
  - body: When the empty context-aware path builds `effectivePrompt` from channel history and writes it into `Body`/`BodyForAgent`, current directive resolution can fall back to prompt text when command fields are empty. A prior channel message containing directive-like text could alter the invoker's turn; keep history in untrusted or supplemental context and keep directive parsing tied to the actual slash text.
  - confidence: 0.89
- **[P3] Remove or wire the unused context history:** `src/slack/monitor/slash.ts:563`
  - body: `contextHistory` is assigned from the formatted Slack message history but is never read afterward. Remove it if prompt-only was intended, or pass it through the chosen structured context path.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The patch introduces a concrete prompt/directive injection risk by treating recent Slack channel history as slash-command prompt text.

Concerns:

- **[medium] Recent channel history can drive directives:** `src/slack/monitor/slash.ts:597`
  - body: Any channel participant could place directive-like text in recent history; an authorized user invoking an empty context-aware slash command would then send that history through prompt fields that current directive parsing can consult when command text is empty.
  - confidence: 0.88
