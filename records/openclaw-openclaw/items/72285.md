## Summary

Keep this PR open. Current main still treats stale session totals as unknown for the Context line and does not fall back to compaction checkpoint `tokensAfter`, while the PR is a focused implementation for open issue #72140 with regression coverage. The only concrete review issue I found is a small unreachable branch in the new guard, not a functional blocker.

## What This Changes

The PR updates status context reporting to use the latest compaction checkpoint `tokensAfter` when `totalTokensFresh` is false and adds a regression test for the post-compaction Context line.

## Best Possible Solution

Merge a focused status formatter change that preserves the stale-total contract, uses the latest valid compaction checkpoint `tokensAfter` only when no fresh total is available, and keeps regression coverage for the post-compaction Context line.
