## Summary

Keep this PR open. Current main has a partial compaction-accounting fix that writes `tokensAfter` as a fresh total when the compaction result supplies it, but the status formatter still renders `entry.totalTokens` directly and does not honor `totalTokensFresh: false` or fall back to `compactionCheckpoints[].tokensAfter`. This PR remains a focused implementation candidate for #72140; it needs the minor unreachable-branch cleanup noted in review plus normal validation before merge.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #72140. The best path is to update `buildStatusMessage()` to use `resolveFreshSessionTotalTokens(entry)`, fall back to the newest valid `compactionCheckpoints[].tokensAfter` only when persisted totals are stale or unavailable, simplify the duplicate guard, and keep the regression test.
