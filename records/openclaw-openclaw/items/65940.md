## Summary

Keep this PR open. Current main still requires `onboardingScopes` for setup-visible web-search providers, DuckDuckGo still omits that metadata, and the PR is paired with the same author’s still-open bug #65862. The patch needs rebasing for the current shared DuckDuckGo metadata shape, but the intended fix is still valid.

## Best Possible Solution

Keep this PR open and review it together with #65862. The current-main fix should add `onboardingScopes: ["text-inference"]` to DuckDuckGo’s shared web-search provider metadata, so both runtime and contract paths inherit it, and add a focused DuckDuckGo assertion in provider/contract/setup coverage to prevent the wizard visibility regression from returning.
