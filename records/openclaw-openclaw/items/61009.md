## Summary

Keep open. The broad stale-docs part discussed in the comments was partially resolved, and current main does allow `host=node` from `auto` when no sandbox runtime is active. But the current source, tests, changelog, repo docs, and live docs still disagree for the sandbox-active case: runtime intentionally rejects per-call `host=node` from `tools.exec.host=auto`, while several docs still say `host=node` is allowed from `auto` without that sandbox caveat. The follow-up comment that this was resolved appears to have missed the node-with-sandbox rejection test.

## Best Possible Solution

Keep this issue open as a docs/runtime-contract cleanup. Either update `docs/tools/exec.md`, `docs/tools/exec-approvals.md`, and `docs/nodes/index.md` to say per-call `host=node` from `auto` is allowed only when no sandbox runtime is active, or make an explicit maintainer product decision to change runtime behavior and then update the tests. Given the changelog entry for the sandboxed-auto guard, the conservative path is a docs clarification plus public docs redeploy.
