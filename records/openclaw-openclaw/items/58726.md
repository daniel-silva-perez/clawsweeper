## Summary

Keep open. Current main still leaves the reported same-tool/same-args `generic_repeat` path warning-only, with hard blocks limited to known poll loops, ping-pong no-progress loops, unavailable-tool repeats, or the separate global no-progress breaker at 30. There are now open PR candidates for overlapping fixes, so this should stay open until maintainers review or land one canonical implementation.

## What This Changes

The issue requests a configurable hard abort after repeated identical tool calls instead of advisory loop warnings.

## Best Possible Solution

Review and land one canonical loop-detector fix, preferably the narrow `generic_repeat` critical-threshold behavior unless maintainers explicitly choose a broader default-on policy. Keep the enforcement in the shared before-tool-call detector path, update docs/tests/changelog, and leave OpenShell-specific validation to the broader #72373 thread.

## Reproduction Assessment

Yes. The issue provides live `session_status` warning logs, and current source/tests give a focused reproduction: enable loop detection, repeat a non-polling same-tool same-args call, and observe warning-level behavior at the critical threshold until the separate global no-progress breaker applies.

## Solution Assessment

Yes, if scoped narrowly. The maintainable fix is to make enabled generic repeated calls escalate to a critical before-tool-call block at the configured threshold; killing the session or globally enabling stricter defaults should be a separate maintainer policy decision.
