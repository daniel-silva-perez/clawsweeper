## Summary

Keep this issue open. Current main still computes the Discord slash-command no-allowlists audit from legacy `dm.allowFrom` plus the pairing-store allowlist only, while top-level `channels.discord.allowFrom` is documented and honored by the runtime path. There is already an open exact-scope fix PR, #72467, linked with closing syntax.

## Best Possible Solution

Keep this issue open and review, update, or land #72467. The fix should stay narrowly in `extensions/discord/src/security-audit.ts`, normalize top-level `discordCfg.allowFrom` together with legacy `dm.allowFrom` and pairing-store allowlists, and keep the regression in `extensions/discord/src/security-audit.test.ts`. No core API change appears necessary.
