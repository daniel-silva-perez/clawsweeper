## Summary

Current main still omits top-level `channels.discord.allowFrom` from the Discord native slash-command allowlist audit, while runtime auth, schema, and docs treat that path as valid. Keep this issue open because open PR #72467 is an exact-scope implementation candidate with closing syntax.

## Best Possible Solution

Keep this issue open and review, update, or land #72467. The implementation should stay narrowly in `extensions/discord/src/security-audit.ts`, include normalized top-level `discordCfg.allowFrom` in `ownerAllowFromConfigured` alongside legacy `dm.allowFrom` and pairing-store allowlists, and add a regression in `extensions/discord/src/security-audit.test.ts`. No core API change appears necessary.
