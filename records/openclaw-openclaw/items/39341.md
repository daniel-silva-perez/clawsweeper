## Summary

Keep this issue open. Current main still omits top-level `channels.discord.allowFrom` from the Discord native slash-command owner allowlist audit, while runtime auth, config schema, docs, and adjacent audit code all treat that path as valid. An exact-scope open PR, #72467, already references this issue with closing syntax and should be reviewed or updated instead of launching duplicate work.

## Best Possible Solution

Keep this issue open and review, update, or close #72467. The implementation should stay narrowly in `extensions/discord/src/security-audit.ts`: include normalized top-level `discordCfg.allowFrom` in `ownerAllowFromConfigured` alongside legacy `dm.allowFrom` and store/pairing allowlists, then add a regression in `extensions/discord/src/security-audit.test.ts` proving top-level `channels.discord.allowFrom` suppresses `channels.discord.commands.native.no_allowlists`. No core API change is needed.
