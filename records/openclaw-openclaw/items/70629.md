## Summary

Keep this PR open. Current main still hardcodes `minimal: 128` for Gemini 2.5 Pro and the `2.5-flash` branch that also catches Flash-Lite, so the PR is not obsolete. The scope still needs maintainer review because current official Google docs document a 512 minimum for Flash-Lite, while Pro and Flash have lower documented minima. ([ai.google.dev](https://ai.google.dev/gemini-api/docs/thinking))

## Best Possible Solution

Keep this PR open for maintainer review. Verify live Google API behavior for `thinkingBudget: 128` on Gemini 2.5 Flash-Lite, Flash, and Pro; then land a scoped Google plugin fix. If public docs remain authoritative, raise only Flash-Lite minimal to 512 and keep Pro/Flash at their documented lower minima; if live calls prove the docs are stale, include that proof and keep the broader constant bump. Extend the transport tests to cover minimal/low/medium/high expectations for the affected ids.
