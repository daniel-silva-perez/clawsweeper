## Summary

Keep open. Current main has the backend pending-approval RPCs and a transient Control UI approval modal, but it still does not provide a discoverable Web dashboard or CLI command to list missed pending exec approvals and act on them; the issue discussion correctly distinguishes this from the existing policy-editing `openclaw approvals` surface.

## What This Changes

The issue requests a Web GUI and/or CLI surface to list pending exec approvals and approve or deny them without relying on channel popups.

## Best Possible Solution

Add a first-class Pending approvals surface that reuses the existing `exec.approval.list` / `exec.approval.resolve` and plugin approval equivalents, refreshes on connect and approval events, removes resolved or expired entries, and documents the workflow in Control UI and CLI docs if a CLI command is added.

## Reproduction Assessment

Yes for the central missing surface: current main can be inspected to show the CLI is policy-only and the Control UI only reacts to live approval events with a modal. The intermittent popup non-delivery itself remains unverified without a live channel reproduction.

## Solution Assessment

Yes in direction, but the implementation should reuse the existing Gateway pending approval RPCs instead of adding another approval store. Maintainers should choose the exact UI/CLI scope because approving exec requests is a security-sensitive operator action.
