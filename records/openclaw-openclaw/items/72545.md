## Summary

Keep open. Current main has backend pending-approval list/resolve RPCs and an event-driven Control UI modal, but it still lacks the requested discoverable Web dashboard or `openclaw approvals` CLI queue command for finding and resolving missed live exec approvals; the issue discussion correctly distinguishes this live queue from the existing policy/allowlist editor.

## What This Changes

Request a Web GUI and/or CLI surface to list and approve or deny live pending exec approvals when pop-up delivery is missed.

## Best Possible Solution

Add a first-class Pending approvals surface that reuses `exec.approval.list` / `exec.approval.resolve` and `plugin.approval.list` / `plugin.approval.resolve`, refreshes on connect and on approval events, presents enough command/session/agent metadata for safe decisions, preserves the `operator.approvals` scope, and updates Control UI plus CLI/docs if maintainers choose to include CLI queue commands.

## Reproduction Assessment

Yes for the central missing recovery surface: current source and docs show the CLI is policy-only and the Control UI only reacts to live approval events with a modal. The intermittent lost-popup delivery failure itself remains unverified without a live channel reproduction.

## Solution Assessment

Yes in direction, but the maintainable implementation should reuse the existing pending approval RPCs instead of adding another approval store. The Web/CLI UX, authorization/audit semantics, and timeout behavior need maintainer product and security review before an automated fix PR.
