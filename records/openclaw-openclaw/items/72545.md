## Summary

Keep open. Current main has backend pending-approval list/resolve RPCs and a transient Control UI modal, but it still lacks a discoverable Web dashboard or `openclaw approvals` CLI queue command for recovering missed exec approvals; the issue discussion correctly separates that live queue need from the existing policy/allowlist editor.

## What This Changes

The issue requests a discoverable Web GUI and/or CLI surface for listing pending exec approvals and approving or denying them when the normal popup is missed.

## Best Possible Solution

Add a first-class Pending approvals surface that reuses `exec.approval.list` / `exec.approval.resolve` and the plugin approval equivalents, refreshes on connect and on approval events, shows enough command/session/agent metadata to make a safe decision, preserves `operator.approvals` authorization, and updates Control UI plus CLI docs if a CLI queue command is added.

## Reproduction Assessment

Yes for the central missing surface: current main can be inspected to show the CLI is policy-only and the Control UI only reacts to live approval events with a modal. The intermittent popup-loss path itself remains unverified without a live channel reproduction.

## Solution Assessment

Yes in direction, but it should reuse the existing pending approval RPCs rather than adding another approval store. The exact Web/CLI UX and audit/security semantics need maintainer product review before an automated fix PR.
