## Summary

Keep open. Current main already has pending approval list/resolve RPCs and a transient Control UI modal, but it still does not provide the requested discoverable Web dashboard or CLI queue for finding and acting on missed live exec approvals.

## What This Changes

Request to add a discoverable Web GUI and/or CLI surface for listing pending live exec approvals and approving or denying them without relying only on popups.

## Best Possible Solution

Add a first-class Pending approvals surface in the Control UI that fetches `exec.approval.list` and `plugin.approval.list` on connect, updates from approval events, shows command/plugin/session/agent/risk metadata, resolves through the existing `*.approval.resolve` RPCs under `operator.approvals`, and clears stale prompts. If maintainers want CLI recovery too, add narrowly scoped `openclaw approvals pending|approve|deny` commands and document both surfaces.

## Reproduction Assessment

Yes for the central missing surface: source inspection shows the CLI is policy-only and the Control UI only renders event-fed modal prompts. The intermittent channel popup failure remains unclear without logs or a live channel reproduction.

## Solution Assessment

Yes in direction, but the implementation should reuse the existing `exec.approval.*` and `plugin.approval.*` pending RPCs rather than adding another approval store. The exact Web/CLI scope and audit/timeout semantics need maintainer product and security review first.
