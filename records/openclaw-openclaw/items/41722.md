## Summary

Keep this issue open. Current main still routes the macOS Skills settings install action through gateway-backed `skills.install`, still defaults `skills.install.preferBrew` to true, still chooses a Homebrew installer whenever `brew` is present, and still formats a failed selected command as generic `Install failed` mapped to `UNAVAILABLE`. I found no current classifier, remediation hint, or safe alternate-installer retry for the reported macOS 26 CLT/Homebrew failure.

## Best Possible Solution

Fix the gateway-backed skill dependency installer path. Add a conservative classifier for Homebrew/Xcode CLT unsupported-macOS failures, return a targeted remediation message, avoid advertising or selecting brew when the host toolchain is known unusable, and only retry a viable alternate installer spec when metadata and policy make that safe. Add focused tests for the classifier, Homebrew selection, and fallback behavior.
