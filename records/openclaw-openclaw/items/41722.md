## Summary

Keep this issue open. Current main still routes the Skills settings install action through gateway-backed skills.install, still selects Homebrew based on brew being present with preferBrew defaulting to true, and still returns generic Install failed output from the selected command. I found no macOS 26 CLT/Homebrew failure classifier, remediation hint, or retry to an alternate installer spec after a selected brew install fails.

## Best Possible Solution

Fix the gateway-backed skill dependency installer path: classify Homebrew/Xcode CLT unsupported-macOS failures, return a targeted remediation message, avoid choosing brew when the host toolchain is known unusable, retry a viable alternate installer spec only when metadata and policy allow it, and add focused tests for the classifier plus installer-selection/fallback behavior.
