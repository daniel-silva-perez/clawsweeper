## Summary

Keep this issue open. Current main still routes macOS Skills settings installs through the gateway-backed `skills.install` dependency-installer path, still prefers Homebrew when `skills.install.preferBrew` is enabled and `brew` is present, and still returns raw installer failure summaries as `UNAVAILABLE` without a classifier or remediation for the reported macOS 26 CLT/Homebrew failure.

## Best Possible Solution

Fix the gateway-backed skill dependency installer path: classify Homebrew/Xcode CLT unsupported-macOS failures, return targeted remediation, avoid selecting or advertising brew when the host toolchain is known unusable, and only retry a viable alternate installer spec when metadata and policy allow it. Add focused regression tests for the classifier, brew selection, and fallback behavior.
