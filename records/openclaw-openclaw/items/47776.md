## Summary

Keep this PR open. Current main still does not implement the intended Unix service-mode cleanup for npm-global `node .../openclaw/... gateway` listeners: the argv verifier exists, but Unix candidates still come from the command-name-filtered `lsof -Fpc` path, so `cnode` listeners are dropped before argv inspection. The branch also remains a poor merge candidate as-is because it bundles unrelated Slack/json changes and a runtime log artifact.

## Best Possible Solution

Keep this PR open as an implementation candidate, but do not merge the current branch as-is. The best maintainer path is a fresh focused patch on current main: collect raw listener PIDs for the gateway port on Unix, verify each by argv via `/proc/<pid>/cmdline` on Linux and `ps` on macOS before signaling, wire that into service-mode stale cleanup, add a regression where `lsof` reports `cnode` while argv identifies `node .../openclaw/dist/entry.js gateway --port 18789`, and drop the unrelated Slack/json/log changes or handle them separately.
