## Summary

Keep this PR open. Current main still does not implement the intended Unix service-mode cleanup for npm-global `node .../openclaw/... gateway` listeners, because service startup still calls only the existing command-name-filtered stale PID cleanup. The PR also remains unsuitable to merge as-is because it bundles unrelated Slack/json changes and a runtime log artifact with security and maintainability concerns from review.

## Best Possible Solution

Keep this PR open as an implementation candidate, but do not merge the current branch as-is. The best maintainer path is a fresh focused patch on current main: collect raw listener PIDs for the gateway port on Unix, verify each by argv via `/proc/<pid>/cmdline` on Linux and `ps` on macOS before signaling, wire that into service-mode stale cleanup, add a regression where `lsof` reports `cnode` while argv identifies `node .../openclaw/dist/entry.js gateway --port 18789`, and split or drop the unrelated Slack/json/log changes.
