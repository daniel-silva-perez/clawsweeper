## Summary

Current main still does not implement this PR's intended Unix service-mode cleanup for npm-global `node ... openclaw ... gateway` listeners. The helper exists, but non-Windows discovery still starts from the command-name-filtered `lsof -Fpc` path, so `node` listeners are filtered out before argv verification. The submitted PR also remains unsafe to merge as-is because its diff bundles unrelated Slack/json/log changes.

## Best Possible Solution

Keep this PR open only as an implementation candidate. The best maintainer path is a fresh, focused change on current main that makes Unix listener discovery collect raw PIDs for the gateway port, verifies each candidate by argv before signaling, wires that into the shared service-mode stale cleanup path, and adds the missing `cnode` regression test. Drop the unrelated Slack/json/log changes from the branch.
