## Summary

Keep this PR open. Current main still does not implement the intended Unix service-mode cleanup for npm-global `node .../openclaw/... gateway` listeners, and the submitted branch is not safe to merge as-is because it bundles unrelated/stale Slack, JSON, and runtime-log changes.

## Best Possible Solution

Keep this PR open as an implementation candidate, but do not merge the current branch as-is. The best maintainer path is a fresh focused patch on current main: collect raw Unix listener PIDs for the gateway port, verify each by argv via `/proc/<pid>/cmdline` on Linux and `ps` on macOS before signaling, wire that into service-mode stale startup cleanup, add a regression for `lsof` reporting `cnode` while argv identifies `node .../openclaw/dist/entry.js gateway --port 18789`, and drop or split the unrelated Slack/json/log changes.
