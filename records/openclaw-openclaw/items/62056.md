## Summary

Keep open. Current main still handles the Feishu bot-added and bot-removed subscription events as log-only cases, and neither the internal hook contract nor the public hook docs expose the requested Feishu membership events. The timeline-related PR is unrelated and explicitly scoped out Feishu hook work.

## What This Changes

The issue requests Feishu bot-added and bot-removed group membership events be exposed as hook events with chat/account/timestamp context.

## Best Possible Solution

Implement the feature inside the Feishu plugin boundary, with only a generic additive hook-contract change if needed: fire documented channel membership hook events from the Feishu monitor handlers, include `chatId`, `accountId`, timestamp, and optional `chatName`, and cover both events with focused monitor and hook contract tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to send or invoke the Feishu dispatcher for `im.chat.member.bot.added_v1` or `im.chat.member.bot.deleted_v1`; current main reaches handlers that only log and never call the hook system.

## Solution Assessment

Yes, the requested direction is the right narrow fix. The maintainable path is Feishu-local event emission plus a small documented hook contract addition, not a core poller or an external plugin that cannot observe these raw Feishu subscription events today.
