## Summary

Keep this issue open. Current main still receives the Feishu bot added/removed subscription events but handles both as log-only cases, and the documented/internal hook surface does not expose the requested `channel:feishu:bot-added` or `channel:feishu:bot-removed` events. The related PR mentioned in the timeline was explicitly scoped to Weixin TTS and did not implement Feishu hook work.

## Best Possible Solution

Implement this in the Feishu plugin boundary: emit hook events from `extensions/feishu/src/monitor.account.ts` for the added and deleted handlers, extend the internal hook contract/docs if `channel:feishu:bot-added` and `channel:feishu:bot-removed` become public event keys, include `chatId`, optional `chatName` when available, `accountId`, and timestamp, and add focused Feishu monitor plus hook contract tests.
