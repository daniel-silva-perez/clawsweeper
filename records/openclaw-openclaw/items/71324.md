## Summary

Keep open: current main still lacks the legacy WebChat history projection this PR targets, and #71312 is closed unmerged while #71229 was narrower. The PR remains useful, but the latest diff has two concrete sanitizer correctness issues that should be addressed before merge.

## What This Changes

The PR extends Gateway WebChat `chat.history` display projection to hide legacy quote-wrapped runtime context, background-task and gateway-restart status rows, startup/subagent runtime prompts, adds session-history regression tests, and adds a changelog entry.

## Best Possible Solution

Land a constrained Gateway display-projection fix that handles already-persisted legacy runtime-only rows while preserving real user text and limiting legacy status stripping to the transcript roles/shapes that actually carried those runtime rows. Keep the regression tests focused on `buildSessionHistorySnapshot`, raw sequence preservation, quote-wrapped internal context, background-task/gateway-restart rows, startup/subagent prompts, and assistant quote non-regression.

## Reproduction Assessment

Yes. The current-main gap is reproducible at the `buildSessionHistorySnapshot` seam with the PR's supplied legacy transcript rows; source inspection shows current main lacks the matchers needed for quote-wrapped delimiters and legacy `System:` status rows. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No, not as written. The PR is aimed at the right Gateway history projection layer, but the sanitizer should preserve trailing user text after legacy prompt headers and should not rewrite assistant-authored text that merely quotes legacy status lines.

  - confidence: 0.83

Scope these legacy repairs to the persisted user/runtime transcript shapes.
  - confidence: 0.78
