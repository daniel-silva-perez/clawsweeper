## Summary

Keep open. The PR is a valid follow-up to the merged visible-transcript work because current main does not cover the legacy background-task and gateway-restart history shapes, but the latest branch still introduces blocking over-stripping behavior in the Gateway history sanitizer.

## What This Changes

The PR adds Gateway history-display sanitizers, regression tests, and a changelog entry to hide legacy quote-wrapped runtime/context, background-task, gateway-restart, subagent-completion, and startup-prompt rows from WebChat/history.

## Best Possible Solution

Keep the Gateway display-projection approach, but make legacy runtime/status cleanup role-aware and tail-preserving, then add regression coverage for reset-tail preservation, assistant quote preservation, and leading-whitespace status prefixes before normal maintainer review.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to project a PR-head user row beginning with the legacy startup/reset prompt followed by real text, or an assistant row quoting a legacy System background-task line; source inspection shows the first is emptied and the second is stripped before any user-role guard.

## Solution Assessment

No, not as written. The projection layer is the right narrow surface, but the sanitizer should strip only legacy runtime/user rows and remove only known runtime prefixes while preserving any following user-authored text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any message that starts with a legacy startup/reset prompt. Legacy rows can combine that runtime header with a real reset tail, and current main explicitly keeps that tail transcript-visible, so this drops user-authored history instead of only removing the runtime header.
  - confidence: 0.87
- **[P2] Limit legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:124-127`
  - body: `stripVisibleTranscriptText` strips legacy background-task and gateway-restart prefixes before the role-specific branch runs. Assistant messages that quote those `System: [...]` lines can therefore be rewritten or emptied even though this repair targets legacy user/runtime transcript rows.
  - confidence: 0.88
- **[P3] Trim before matching legacy status prefixes:** `src/gateway/chat-sanitize.ts:97-100`
  - body: The runtime detector trims before checking the `System:` status regexes, but the stripping helpers test the raw string. A legacy row with leading whitespace before the status prefix can therefore remain visible even though it is the same runtime-only status shape this PR is trying to hide.
  - confidence: 0.74
