## Summary

Keep open: current main does not yet handle the legacy quote-wrapped runtime/status rows this PR targets, and the PR is an active contributor fix for a real Gateway WebChat history bug. The latest PR head still introduces sanitizer data-loss paths around mixed legacy startup rows and assistant messages that quote legacy status text, so it needs author changes and maintainer review before merge.

## What This Changes

The PR adds Gateway chat.history display sanitization for legacy quote-wrapped runtime context, background-task and gateway-restart status rows, filters now-empty visible history rows, adds session-history regression tests, and updates the changelog.

## Best Possible Solution

Update the active PR rather than close or replace it: keep the Gateway display-projection approach, but preserve real user text after legacy startup/reset prompt headers, restrict legacy status/prompt cleanup to proven user/runtime transcript rows, keep raw sequence accounting unchanged, and add focused regressions for mixed reset-tail rows and assistant status-quote preservation.

## Reproduction Assessment

Yes. ## Solution Assessment

No, not as written. Gateway history projection is the right layer for this legacy cleanup, but the current sanitizer needs role-aware stripping and mixed-row preservation to be the narrowest maintainable fix.
