## Summary

Keep open: current main still has the legacy WebChat history gap, but this PR should not merge as-is because the PR-head sanitizer can drop legitimate mixed user text and rewrite assistant-authored quotes of legacy status lines. https://github.com/openclaw/openclaw/pull/71324

## What This Changes

The PR extends Gateway `chat.history` display projection to unwrap or strip legacy runtime-only transcript wrappers, hide now-empty visible rows, preserve raw transcript sequencing, add session-history regressions, and add a changelog entry.

## Best Possible Solution

Land a constrained Gateway display-projection fix that handles already-persisted legacy runtime-only rows while preserving following real user text and limiting legacy runtime/status repairs to the persisted user/runtime transcript shapes that carried those artifacts. Keep raw sequence preservation and focused `buildSessionHistorySnapshot` regression coverage.

## Reproduction Assessment

Yes. The PR-head source gives a high-confidence reproduction path: pass a user row starting with `A new session was started via /new or /reset.` followed by real text, or an assistant row quoting `System: [timestamp] Background task done: ...`, through the new sanitizer paths. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No, not as written. The Gateway history projection layer is the right place for legacy display cleanup, but the patch needs mixed-row preservation and role-aware legacy status stripping before it is the narrowest maintainable fix.

  - confidence: 0.83

  - body: The new legacy Background task and Gateway restart strippers run before any role-specific guard, so assistant messages that quote `System: [...] Background task ...` or restart status text are rewritten or emptied in history. Apply these legacy repairs only to the persisted user/runtime transcript shapes they target.
  - confidence: 0.78
