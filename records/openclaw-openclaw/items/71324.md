## Summary

Keep open. The reported Gateway/WebChat bug is valid and current main still lacks the PR's legacy quote-wrapper, background-task status, gateway-restart status, and legacy startup prompt handling. The PR targets the right chat.history display projection path, but latest PR head still introduces two blocking sanitizer regressions around mixed startup/reset rows and assistant messages that quote legacy status text.

## What This Changes

The PR adds Gateway chat.history display sanitization for legacy quote-wrapped runtime prompts, background-task and gateway-restart status rows, filters now-empty visible history rows while preserving raw transcript sequence accounting, adds regression tests, and updates the changelog.

## Best Possible Solution

Keep the Gateway history projection approach, but make the legacy cleanup role-aware for user/runtime transcript rows, strip only the runtime header/status portion, preserve following real user text, keep raw transcript sequence accounting unchanged, and add focused regressions for reset-tail preservation plus assistant quote preservation.

## Reproduction Assessment

Yes. A high-confidence code-only reproduction is to pass a PR-head user row beginning with the reset startup prompt plus real tail text, or an assistant row quoting a legacy System background-task line, through stripEnvelopeFromMessage/projectChatDisplayMessages and observe the first being emptied and the second being stripped before any user-role guard.

## Solution Assessment

No, not as written. The display projection layer is the right place for this legacy cleanup, but the implementation needs role-aware stripping and mixed-row tail preservation to be the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns an empty string for any text that starts with a legacy runtime prompt. Legacy reset rows can include a real user tail, and current reset tests preserve that tail as transcript text, so the projection should remove only the runtime header instead of dropping the whole row.
  - confidence: 0.86
- **[P2] Gate legacy status stripping by role:** `src/gateway/chat-sanitize.ts:123-127`
  - body: `stripVisibleTranscriptText` strips legacy background-task and gateway-restart prefixes before checking whether the message is a user/runtime transcript row. Assistant messages that quote those `System: [...]` lines can be rewritten or emptied in chat history, even though this repair targets legacy user rows.
  - confidence: 0.88
