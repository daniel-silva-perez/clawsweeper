## Summary

Keep open. Current main still lacks this legacy WebChat/history projection coverage, but the current PR head has two blocking sanitizer regressions that can hide user-authored reset-tail text and rewrite legitimate assistant content before merge.

## What This Changes

The PR adds Gateway chat-history projection sanitizers, regression tests, and a changelog entry to hide legacy quote-wrapped runtime/context, background-task, gateway-restart, subagent-completion, and startup-prompt rows from WebChat/history.

## Best Possible Solution

Keep the Gateway display-projection approach, but make legacy runtime/status cleanup role-aware and tail-preserving, then add regressions for reset-tail preservation and assistant-quote preservation before normal maintainer review.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to project a PR-head user row beginning with the legacy startup/reset prompt followed by real text, or an assistant row quoting a legacy System background-task line; source inspection shows the first is emptied and the second is stripped before any user-role guard.

## Solution Assessment

No, not as written. The projection layer is the narrow maintainable surface, but the implementation should preserve user tail text after known legacy headers and restrict legacy status stripping to user/runtime rows.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve user text after legacy prompt headers:** `src/gateway/chat-sanitize.ts:90`
  - body: `stripLegacyInternalOnlyPrompt` returns `""` for any message that starts with the legacy startup/reset prompt. Legacy rows can combine that runtime header with a real reset tail, and current behavior explicitly keeps that tail transcript-visible, so this drops user-authored history instead of only removing the runtime header.
  - confidence: 0.87
- **[P2] Limit legacy status stripping to user rows:** `src/gateway/chat-sanitize.ts:124-127`
  - body: `stripVisibleTranscriptText` strips legacy background-task and gateway-restart prefixes before the user-role-specific branch is applied. Assistant messages that quote those `System: [...]` lines can therefore be rewritten or emptied even though this repair targets legacy user/runtime transcript rows.
  - confidence: 0.88
