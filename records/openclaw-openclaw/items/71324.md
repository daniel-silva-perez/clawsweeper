## Summary

Keep open. The PR targets a real Gateway history-projection gap that current main does not fully handle, but the current patch can drop user-authored reset-tail text and can rewrite legitimate assistant content that quotes legacy status lines.

## What This Changes

The PR adds Gateway chat-history projection sanitizers and regression coverage to hide legacy quote-wrapped runtime/context, background-task, gateway-restart, subagent-completion, and startup-prompt rows from WebChat/history, plus a changelog entry.

## Best Possible Solution

Keep the Gateway display-projection approach, but make legacy runtime/status cleanup role-aware and prefix-specific, preserve real user text that follows known runtime headers, and add regressions for reset-tail preservation and assistant quote preservation before normal maintainer review.

## Reproduction Assessment

Yes. A high-confidence code-only reproduction is to project a PR-head user row beginning with the legacy startup/reset prompt followed by real text, or an assistant row quoting a legacy System background-task line; source inspection shows the first is emptied and the second is stripped before any user-role guard.

## Solution Assessment

No, not as written. The projection layer is the right narrow surface, but this implementation needs tail preservation and role-aware status stripping before it is the maintainable fix.
