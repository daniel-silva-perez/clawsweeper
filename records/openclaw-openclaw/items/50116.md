## Summary

Keep this PR open. Current main still treats Windows drive-letter `sessionFile` values as relative on POSIX, so the user-visible bug is not implemented on main. The PR is a focused fix attempt, but the submitted basename-only rewrite still needs revision because it can bypass the resolver's existing realpath/cross-agent handling and can persist the rewritten path.

## Best Possible Solution

Keep the PR open and request a revision: detect Windows absolute paths on POSIX with `path.win32` or a shared helper, preserve `agents/<agentId>/sessions/<file>` context where present, route through the existing realpath/containment and cross-agent fallback rules, reject degenerate Windows paths safely, and add focused regression tests for both slash styles plus cross-agent and symlink cases.
