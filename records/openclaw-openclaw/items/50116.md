## Summary

Keep this PR open. Current main still does not implement the Windows-drive session path handling this PR is trying to add: POSIX hosts still treat `C:\...\abc.jsonl` and `C:/.../abc.jsonl` as relative session paths, so the broken concatenation class remains current. The PR is relevant, but the submitted basename-only approach still has unresolved review issues around cross-agent aliasing, symlink/realpath consistency, degenerate Windows paths, and missing regression coverage.

## Best Possible Solution

Keep this PR open and request a revision. The likely fix belongs in `src/config/sessions/paths.ts`: detect Windows absolute paths on POSIX with `path.win32` or a shared helper, preserve enough agent/sessions context instead of collapsing to basename, route through the existing realpath/containment and cross-agent fallback behavior, reject degenerate Windows paths safely, and add targeted tests for both slash styles plus cross-agent and symlink cases.
