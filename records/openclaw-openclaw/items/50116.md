## Summary

Keep this PR open. Current main still treats Windows drive-letter `sessionFile` values as relative on POSIX, so the underlying bug is not implemented on main. The submitted basename-only rewrite is also not ready to close as obsolete or land unchanged because it can lose agent/session-store context and persist that alias.

## Best Possible Solution

Keep the PR open and request a revision: detect Windows absolute paths on POSIX with `path.win32` or a shared helper, preserve `agents/<agentId>/sessions/<file>` context where present, reuse the existing realpath/containment rules, reject degenerate Windows paths safely, and add focused regression tests for both slash styles plus cross-agent and symlink cases.

## Risks / Open Questions

- Closing this PR now would leave the Windows-on-POSIX session path bug unresolved on main.
