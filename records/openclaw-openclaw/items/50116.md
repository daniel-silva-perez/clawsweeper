## Summary

Keep this PR open. Current main still does not handle Windows drive-letter `sessionFile` values correctly on POSIX, and the PR's basename-only rewrite is not safe to close as implemented because it can lose agent/session-store context.

## Best Possible Solution

Keep this PR open, but request a revision: detect Windows absolute paths on POSIX with `path.win32` or an existing shared helper, preserve `agents/<agentId>/sessions/<file>` context, reuse the current realpath/containment rules, reject degenerate paths safely, and add focused regression tests for both slash styles plus cross-agent and fallback cases.

## Risks / Open Questions

- Closing this PR now would leave a current Windows-on-POSIX session path bug untracked in the open PR queue.
- Landing the submitted diff unchanged could rewrite another agent's Windows-authored transcript path to the current agent's basename and persist that alias.
- Regression coverage is still missing for backslash drive paths, forward-slash drive paths, trailing separators or drive roots, symlink/realpath consistency, and cross-agent Windows-authored paths.
