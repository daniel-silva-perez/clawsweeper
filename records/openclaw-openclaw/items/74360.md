## Summary

Keep open. Current main still has the broad private-network SSRF policy, so the PR targets a real security-sensitive gap, but the latest branch is unmerged, currently unmergeable, and still has a blocking public Plugin SDK baseline issue before it can be considered ready.

## What This Changes

The PR makes trusted web-tool endpoints use a strict SSRF policy, adds explicit self-hosted web fetch/search helpers, switches SearXNG to the self-hosted helper, exports the helpers through the Plugin SDK, adds SSRF policy tests, and adjusts one gateway hot-reload fixture.

## Best Possible Solution

Land a maintainer-approved SSRF fix that keeps fixed public web-search providers strict by default, preserves an explicit validated private-network path for SearXNG-style self-hosted endpoints, updates SDK baseline/docs for any public helper exports, resolves the current mergeability issue, and lets #74357 close only after merge.
