## Summary

Keep open. Current main still has the broad trusted web-tools SSRF policy, while this PR is an unmerged security-sensitive implementation that changes SSRF policy, public plugin SDK exports, tests, and Docker build provenance; it needs maintainer/security review rather than cleanup closure.

## What This Changes

The PR splits web-tools SSRF handling into trusted and self-hosted helpers, switches SearXNG to the self-hosted path, exports the helpers through the plugin SDK, adds SSRF policy tests, and changes Docker Bun installation to copy from an oven/bun image.

## Best Possible Solution

Land a maintainer-approved fix that makes fixed public web-search providers strict by default, preserves an explicit tested private-network path for validated SearXNG-style self-hosted endpoints, and either splits or hardens the Docker Bun change with digest/checksum verification and SDK contract follow-through.
