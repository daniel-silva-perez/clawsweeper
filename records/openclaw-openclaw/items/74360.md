## Summary

Keep open. Current main still has the broad web-tools private-network SSRF policy, and this PR is the active implementation for the linked security issue, but the branch is not merge-ready because it expands public Plugin SDK exports without updating the generated SDK API baseline.

This PR makes trusted web-search/web-fetch endpoints use a strict SSRF policy by default, adds explicit self-hosted endpoint helpers exported through the Plugin SDK, switches SearXNG to the self-hosted helper, and adds SSRF policy tests.

Land a maintainer-approved SSRF fix that keeps fixed public web-search providers strict by default, preserves a validated self-hosted private-network path for SearXNG-style endpoints, updates generated SDK baseline/docs if the public helper exports remain, and lets #74357 close only after merge.
