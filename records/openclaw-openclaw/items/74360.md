## Summary

Keep open. The PR addresses a real current-main SSRF boundary problem and the patch direction is coherent, but it adds public Plugin SDK exports without updating the generated API baseline, so it needs a PR-branch fix plus maintainer/security review before merge.

## What This Changes

This PR splits web-tools SSRF handling into strict trusted endpoints and explicit self-hosted helpers, switches SearXNG to the self-hosted helper, exports the helpers through the Plugin SDK, and adds SSRF policy tests.

## Best Possible Solution

Land a maintainer-approved SSRF fix that keeps public web-search providers strict by default, preserves a validated self-hosted private-network path for SearXNG-style endpoints, updates generated SDK baseline/docs for any public helper exports, and lets #74357 close only after merge.
