## Summary

Keep open. Current main still applies the private-network-allowing trusted web-tools SSRF policy to shared web-search helpers, and this PR is the active implementation paired with open issue #74357. The latest PR diff appears focused on SSRF helper, SearXNG, SDK exports, and tests after the earlier Dockerfile change was removed, but the change still needs maintainer/security review because it alters a network security boundary and adds public SDK surface.

## What This Changes

This PR splits web-tools SSRF handling into strict trusted endpoints and explicit self-hosted helpers, switches SearXNG to the self-hosted helper, exports the new helpers through the plugin SDK, and adds SSRF policy tests.

## Best Possible Solution

Review and land a maintainer-approved SSRF fix that makes fixed public web-search providers strict by default, preserves an explicit validated private-network path for SearXNG-style self-hosted endpoints, updates SDK/docs/tests for any public helper exports, and lets #74357 close after merge.
