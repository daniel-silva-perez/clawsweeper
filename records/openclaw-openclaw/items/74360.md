## Summary

Keep this PR open for maintainer/security review. Current main still has both underlying behaviors: the shared trusted web-tools SSRF policy allows private-network targets, and the Dockerfile still installs Bun via `curl | bash`. The branch is useful source work, but it mixes the SSRF fix for #74357 with Docker supply-chain hardening for #74356, and the Docker change introduces a mutable `oven/bun:1.3.9` image tag that already drew review feedback.

## What This Changes

The PR branch splits web-tool SSRF handling into trusted and self-hosted endpoint helpers, switches SearXNG to the self-hosted helper, exports the new helpers through the plugin SDK, and changes the Docker build to copy Bun from an `oven/bun` image instead of running the Bun installer script.

## Best Possible Solution

Review or split the branch into two maintainer-approved fixes: one SSRF change that makes fixed public web-search providers strict by default while preserving an explicit, tested self-hosted/private path for validated SearXNG-style endpoints, and one Docker hardening change that uses a digest- or checksum-verified Bun source and updates any copied Docker guidance.
