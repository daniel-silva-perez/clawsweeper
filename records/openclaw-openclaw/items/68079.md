- **[P2] Compose with the current ZAI stream wrapper:** `extensions/zai/index.ts:279`
  - body: Current main routes ZAI through `wrapZaiStreamFn`, which preserves `tool_stream` and patches ZAI `thinking` payloads. Replacing that hook with a wrapper that only handles tool-stream plus the session header would regress the tested thinking behavior after rebase.
  - confidence: 0.88
- **[P2] Reject invalid header values before injecting:** `extensions/zai/stream.ts:18-21`
  - body: `isInjectableSessionId` accepts any non-empty string up to 256 chars, so values containing CR/LF/NUL can reach `X-Session-Id`. Fetch rejects those header values, turning optional affinity into a failed ZAI request instead of omitting the header.
  - confidence: 0.9
- **[P2] Replace session headers case-insensitively:** `extensions/zai/stream.ts:40-42`
  - body: The merge keeps an existing `x-session-id` and then adds `X-Session-Id`. Header normalization combines those into a comma-joined value, so remove any existing session header case-insensitively before setting the canonical one.
  - confidence: 0.9
- **[P2] Use the repo-current pi-ai version:** `extensions/zai/package.json:7-9`
  - body: The new ZAI runtime dependency pins `@mariozechner/pi-ai` to `0.69.0`, while current main and existing provider plugins use `0.70.6`. That can install a second older provider runtime copy and diverge stream behavior.
  - confidence: 0.87
