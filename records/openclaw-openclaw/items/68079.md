## Summary

## What This Changes

The PR adds a ZAI provider stream wrapper that injects `X-Session-Id` from `options.sessionId`, composes tool-stream defaults, updates docs/changelog, adds ZAI stream tests, and declares `@mariozechner/pi-ai` as a ZAI plugin dependency.

## Best Possible Solution

Rebase the branch onto current main and keep this as ZAI-owned provider behavior: compose session-header injection with the existing ZAI stream wrapper, harden header validation/replacement, align dependency and changelog metadata with current repo policy, and merge only after targeted ZAI/plugin dependency tests pass.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Compose with the current ZAI thinking wrapper:** `extensions/zai/index.ts:279`
  - body: Current main now uses a ZAI `wrapStreamFn` that preserves `tool_stream` and also patches `thinking` for disabled and preserved-thinking modes. This PR's replacement wrapper only composes tool-stream plus the new header, so after rebase it must wrap or incorporate the current `wrapZaiStreamFn` logic or it will regress the tested ZAI thinking behavior.
  - confidence: 0.82
- **[P2] Reject invalid header values before injection:** `extensions/zai/stream.ts:16-20`
  - body: `isInjectableSessionId` accepts any non-empty string up to 256 chars, so values containing CR/LF/NUL can be copied into `X-Session-Id`. Fetch/Undici rejects at least those bytes during header construction, turning a malformed caller `sessionId` into a failed ZAI request instead of omitting the optional affinity header.
  - confidence: 0.88
- **[P2] Replace existing session header case-insensitively:** `extensions/zai/stream.ts:39-41`
  - body: The header merge keeps an existing `x-session-id` and then adds `X-Session-Id`; downstream `Headers` normalization combines those into a single comma-joined value. If provider/request headers already include the session header, ZAI can receive `old, new` and lose sticky routing, so remove any existing key case-insensitively before setting the canonical one.
  - confidence: 0.9
- **[P2] Use the repo-current pi-ai version:** `extensions/zai/package.json:7-9`
  - body: The new ZAI runtime dependency pins `@mariozechner/pi-ai` to `0.69.0`, while current main and existing provider plugins use `0.70.6`. That can install a second older provider runtime copy and diverge `streamSimple` behavior; update the dependency and lockfile to the current repo version.
  - confidence: 0.86
- **[P3] Add the required changelog credit:** `CHANGELOG.md:7`
  - body: The added changelog bullet has no `Thanks @...` attribution, but repo policy requires every added changelog entry to include at least one credited GitHub username. Add the contributor credit before merge.
  - confidence: 0.81
