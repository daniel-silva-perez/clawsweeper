## Summary

Keep this PR open. Current main still has the Web UI tool-card extraction gap this PR targets: embedded `tool_result` blocks only read string `text` or string `content`, so array-shaped `content[0].text` payloads can still render as empty tool output. The PR remains the active contributor path, with maintainer repair and merge-ready/human-review labels in the provided GitHub context, but it should resolve or explicitly accept the bounded-output security concern before landing.

## Best Possible Solution

Keep this PR open as the canonical implementation path. Resolve the bounded-output review concern, then land this branch or an equivalent maintainer patch that extracts and joins array-shaped `text` and `content` fragments in `ui/src/ui/chat/tool-cards.ts`, adds regression coverage in `ui/src/ui/chat/tool-cards.node.test.ts`, and preserves contributor credit in the changelog.
