## Summary

Keep open. Current main still does not implement the PR’s intended Web UI fix: embedded `toolresult` / `tool_result` blocks only extract string `text` or string `content`, so array-shaped `content[0].text` still falls through to the empty-output path. The PR remains a focused implementation candidate with useful regression coverage added after review feedback.

## Best Possible Solution

Keep this PR open as the active fix path. The best maintainer outcome is to review and land this branch, or an equivalent patch, that joins valid string fragments from array-shaped `text` and `content` for embedded tool-result blocks, preserves the added regression tests, and makes an explicit call on whether a bounded output cap is needed for large tool results.
