## Summary

Keep open. Current main still does not implement this PR's intended behavior: embedded `toolresult` / `tool_result` blocks only extract string `text` or string `content`, so array-shaped `content[0].text` output still falls through to missing output. The PR remains a focused implementation candidate, but the current branch should address bounded array-fragment collection before landing.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate. Amend or review the branch to bound collected tool-output fragments by count and total characters, keep the array-shaped `text` and `content` regression tests plus changelog credit, then land it after maintainer/security review and normal validation. If another change lands first, close this PR only after current main contains bounded array extraction with regression coverage.
