## Summary

Keep open. Current main still does not implement the PR’s intended fix: embedded Web UI `tool_result` blocks only extract string `text` or string `content`, so array-shaped payloads such as `content[0].text` can still produce empty output. The PR remains the active contributor path, but the branch should address or explicitly accept the bounded-output security concern before landing.

## Best Possible Solution

Keep this PR open as the canonical implementation path. Resolve the bounded-output review concern, then land this branch or an equivalent maintainer patch that extracts and joins array-shaped `text` and `content` fragments in `ui/src/ui/chat/tool-cards.ts`, adds regression coverage in `ui/src/ui/chat/tool-cards.node.test.ts`, and preserves contributor credit in the changelog.
