## Summary

Keep open. Current main still has the bug this PR targets: embedded `toolresult` / `tool_result` blocks only extract string `text` or string `content`, so array-shaped tool results such as `content[0].text` still render as missing output. The PR remains a focused implementation candidate, but its latest diff should still get maintainer/security review because it joins array fragments without explicit size bounds.

## Best Possible Solution

Keep this PR open as the canonical implementation path. Amend or review the branch to bound collected tool-output fragments, preserve the array-shaped `text` and `content` regression tests and changelog credit, then land it once maintainer/security review and normal validation are satisfied. If another change lands first, close this PR only after current main contains bounded extraction plus regression coverage.
