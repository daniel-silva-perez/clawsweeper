## Summary

Keep open. Current main still does not implement this PR’s intended behavior: embedded Control UI `toolresult` / `tool_result` blocks only extract output when `text` or `content` is already a string, so array-shaped payloads such as `content[0].text` can still render as empty output. The PR remains the focused contributor path, but its array-joining change should address the bounded-output security concern before landing.

## Best Possible Solution

Keep this PR open as the canonical implementation path. Review or amend it to extract and join array-shaped `text` and `content` fragments with explicit fragment/character limits, keep the regression coverage in `ui/src/ui/chat/tool-cards.node.test.ts`, and preserve contributor credit in the changelog before landing.
