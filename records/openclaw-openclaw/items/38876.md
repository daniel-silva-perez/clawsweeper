## Summary

Keep this PR open. Current main at 16906780fd2c481308ec4b80f96eb3b93d0add46 still has the Web UI tool-card bug this PR targets: embedded tool result blocks only extract output when `text` or `content` is already a string, so array-shaped `content[0].text` payloads can still fall through to the No output path. The PR is still the active contributor path, with a recent maintainer repair on the branch and merge-ready/human-review labels, so it is not obsolete or implemented on main.

## Best Possible Solution

Keep this PR open as the canonical implementation path. Review the bounded-output concern, then land this branch or an equivalent maintainer patch that extracts and joins array-shaped `text` and `content` fragments in `ui/src/ui/chat/tool-cards.ts`, keeps regression coverage in `ui/src/ui/chat/tool-cards.node.test.ts`, and preserves contributor credit in the changelog.
