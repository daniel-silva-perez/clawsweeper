## Summary

Keep open. Current main has partial overlapping observability: structured diagnostic tool execution events, OTel/Prometheus export, Control UI agent-event redaction, trajectory export, and apply_patch patch summaries. It still does not implement the full requested surface: INFO/file-log sanitized argument summaries, structured persisted toolResult fields such as errorType/errorRecoverable, or generic context/file/memory writeback audit tracking. The related PR #55846 was closed unmerged, and no canonical open replacement in the provided context fully tracks the remaining work.

## Best Possible Solution

Keep this issue open and implement it by extending the existing diagnostic event, agent-event redaction, OTel/Prometheus, trajectory, and transcript/toolResult surfaces rather than adding a parallel logging path. Maintainers should decide whether structured error metadata belongs in persisted toolResult records, diagnostics only, or both, then add a generic writeback/audit event contract for mutating tools beyond the current apply_patch-only summary.
