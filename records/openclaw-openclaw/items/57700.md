## Summary

Keep open. Current main still relays hook non-delivery and hook error summaries into an agent main session as untrusted system events. The shipped/current fix only moved those events to the target agent session instead of the default agent session; it did not suppress hook delivery artifacts or add a hook-level relay/audience policy.

## Best Possible Solution

Keep this issue open and fix it in core hook/system-event handling. A conservative path is to add an explicit hook delivery relay policy or event audience, default hook delivery artifacts/errors to log/internal-only where appropriate, and update `/hooks/agent`, mapped-hook, and non-delivery/error tests so artifacts no longer enter agent prompts unless intentionally configured.
