## Summary

Keep open. Current main no longer leaks hook completion/error summaries to the default agent session for explicit agent hooks, but it still relays failed or undelivered hook agent runs as untrusted `Hook ...` system events into an agent main session. There is still no hook-level suppression/audience config, and drained system events are still prepended into agent prompt bodies where the model can react and send noisy channel messages.

## Best Possible Solution

Keep this issue open and fix it in core hook/system-event handling. A conservative path is to add an explicit hook delivery relay policy or event audience, default hook delivery artifacts/errors to log/internal-only where appropriate, and update `/hooks/agent` plus mapped-hook non-delivery/error tests. Coordinate with the broader system-event audience discussion so the hook-specific fix does not create another one-off classification path.
