## Summary

Keep open. Current main has partial mitigations for the cron restart flood and Discord gateway lifecycle, and the discussion usefully points to KeepMyClaw as an external snapshot workaround, but the core cleanup gap is not fully resolved: default subagent runs still keep their session rows, while cron.sessionRetention still only sweeps cron run session keys. The related #51767 is still open and should remain the narrower canonical thread for that part of the report.

## Best Possible Solution

Keep this open until maintainers either split the remaining work into narrower canonical issues or implement a core cleanup contract for default subagent/session retention. The most concrete next step is to resolve #51767 by deciding whether default subagent sessions should be archived/deleted by a subagent-specific retention setting, generic session.maintenance, or an expanded retention sweep, then align docs and tests with that behavior. The already-mitigated cron catch-up and Discord gateway lifecycle pieces can be referenced as partial progress, not as a full close reason.
