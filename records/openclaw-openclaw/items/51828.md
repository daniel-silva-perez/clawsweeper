## Summary

Keep open. Current main has partial mitigations for cron restart catch-up and Discord gateway lifecycle, and the discussion usefully identifies KeepMyClaw as an external snapshot workaround, but the concrete default sub-agent session retention gap remains: `sessions_spawn` still defaults to `cleanup: keep`, keep-mode run subagents do not get `archiveAtMs`, and the only sweep path that calls `sessions.delete` is for entries with `archiveAtMs`. The related #51767 remains the narrower canonical thread for that remaining cleanup work.

## Best Possible Solution

Keep this open until maintainers either split the umbrella into narrower canonical issues or implement and document a default subagent/session retention contract. The most concrete next step is to resolve #51767 by deciding whether default keep-mode subagent sessions should be archived/deleted by `agents.defaults.subagents.archiveAfterMinutes`, a new session-maintenance policy, or explicit per-run cleanup semantics, then align tests and the public subagent docs with that behavior. The already-mitigated cron catch-up and Discord lifecycle pieces can be treated as partial progress.
