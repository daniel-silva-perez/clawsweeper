## Summary

Current main still has the exact fallback this PR changes: `stopChannel` aborts an existing controller, but passes `new AbortController().signal` to `stopAccount` when no controller exists. The PR remains a focused implementation candidate rather than something already shipped or superseded.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to add a focused `server-channels.test.ts` case for the no-controller `stopAccount` fallback, then land the one-line `AbortSignal.abort()` change if the targeted gateway tests pass.
