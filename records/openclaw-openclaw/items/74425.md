## Summary

Keep this PR open. Current main does not contain the proposed launcher-level exit, and the linked Windows hang remains plausibly actionable, but this is an open implementation PR that needs maintainer review plus Windows and output-flush validation rather than closure or an automated replacement.

## What This Changes

This PR adds a final `process.exit(process.exitCode ?? 0)` to `openclaw.mjs` after the launcher finishes its help or built-entry import path.

## Best Possible Solution

Land this PR or an equivalent launcher-boundary termination fix only after confirming it preserves intended exit codes, completed stdout/stderr output, cleanup behavior, and foreground command behavior on Windows.
