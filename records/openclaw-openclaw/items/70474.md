## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Clear stale channel refreshes when disconnected:** `ui/src/ui/controllers/channels.ts:171-179`
  - body: When a stronger request is queued during an in-flight channel load, this finally block only clears `pending` if the state is still connected with a client. If the UI disconnects first, the stale stronger request stays in the WeakMap and can replay after reconnect during an unrelated load, causing avoidable heavy status work and potentially replacing the caller's expected state.
  - confidence: 0.87
- **[P2] Guard gateway guidance on failed probes:** `src/wizard/setup.finalize.ts:295-305`
  - body: This `else` is paired with `if (!gatewayProbe.ok && installDaemon)`, so it also runs when the probe succeeds. Successful setup runs can therefore show `Gateway not detected yet` guidance even though the Gateway is reachable; gate this note on a failed probe without daemon install.
  - confidence: 0.9
- **[P2] Reuse the configured result in summary snapshots:** `src/gateway/server-methods/channels.ts:347-354`
  - body: The summary-only path computes `configured` before probe/audit, but then calls `buildChannelAccountSnapshot`, whose default implementation can call `plugin.config.isConfigured` again for accounts without a `configured` field. That keeps the new lightweight status path from guaranteeing a single configuration check; the added test misses this because it mocks the snapshot helper.
  - confidence: 0.78
