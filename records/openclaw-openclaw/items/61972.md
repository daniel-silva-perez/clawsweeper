## Summary

Keep this PR open. Current main still lacks the central behavior: Gemini web_search and xAI web_search/x_search still build requests from fixed provider endpoints, and the Google/xAI plugin manifests and user docs do not expose the requested baseUrl config. The PR appears useful, but it needs maintainer review/refresh for current config ownership before it can land.

## Best Possible Solution

Keep the PR open as a plausible implementation candidate. Refresh it onto current main, align x_search baseUrl with the plugin-owned xAI config surface while preserving any needed legacy fallback, add/update manifest/schema/docs entries, and keep focused tests for Gemini web_search, xAI web_search, xAI x_search, normalization, cache-key separation, and endpoint safety.
