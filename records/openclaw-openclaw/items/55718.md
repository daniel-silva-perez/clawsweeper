## Summary

Keep open. Current main still supports manual session labels and account `name` fields for account/status UI, but it does not implement account-config-driven default labels for newly created inbound channel sessions. The existing ClawSweeper comment matches the current code: `displayLabel` is not a channel account config field, inbound session metadata is derived from message context rather than account config, and the Control UI session list only renders stored session label/display data.

## What This Changes

The issue requests a channel account display-label configuration that automatically seeds friendly labels for new channel sessions shown in the Control UI session selector.

## Best Possible Solution

Implement a narrow shared channel/session enhancement that resolves the configured channel account display name for the inbound `channel + accountId`, seeds the stored session label/display name for new or unlabeled sessions, and documents the supported config spelling with regression coverage for session metadata, gateway rows, and Control UI rendering.

## Reproduction Assessment

Yes. A code-level reproduction is clear: configure a channel account with a friendly `name` or proposed `displayLabel`, create an inbound session with that `AccountId`, and current `recordSessionMetaFromInbound` cannot use that config value because it only receives message context and group resolution.

## Solution Assessment

The exact proposed `displayLabel` field is not clearly the best standalone fix because account `name` already exists as a documented display field. The maintainable path is to reuse `name` as the default session label source, adding `displayLabel` only as an intentional documented alias if maintainers want that public spelling.
