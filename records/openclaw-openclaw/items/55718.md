## Summary

Keep open. Current main supports manually edited per-session labels in Control UI, and channel accounts already have an optional `name` display field, but the requested account-config-driven default label for newly created inbound channel sessions is still missing. I did not find a `displayLabel` channel-account config field or any code path that maps account `name`/`displayLabel` into `session.label` during inbound session metadata creation.

## Best Possible Solution

Keep this open as a valid core/channel-session enhancement. The likely path is to reuse or explicitly alias existing channel account `name` as the default session label, or add a documented generic `displayLabel` if maintainers prefer that public name. Then update inbound session metadata creation to resolve the account from provider/accountId and set `session.label` only for new or unlabeled sessions, with regression coverage around `recordSessionMetaFromInbound`, gateway session rows, and channel/config docs.

## Risks / Open Questions

- There is already an account-level `name` field. Adding a separate `displayLabel` without an alias/migration decision could create overlapping display-name concepts.
- The implementation should only seed labels for new or currently unlabeled sessions, so user-edited `session.label` values are not overwritten.
- This touches a shared channel/session boundary; docs, config schema, gateway session-row behavior, and plugin account surfaces should be kept aligned.
