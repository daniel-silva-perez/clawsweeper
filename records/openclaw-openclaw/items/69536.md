## Summary

Current main still has the reported mismatch. The bundled session-memory hook still writes `memory/YYYY-MM-DD-slug.md`, while memory-core CLI historical discovery, live dreaming daily ingestion, historical seeding, and Gateway Dream Diary backfill still accept only exact `YYYY-MM-DD.md` names. Startup context now reads slugged artifacts, which helps `/new` and `/reset` prompt context, but it does not make those files participate in dreaming or grounded backfill.

## Best Possible Solution

Keep this open and fix memory-core daily-file discovery with a shared dated-memory-file matcher that accepts both canonical date-only files and slugged session-memory artifacts. Use it in CLI `rem-harness`/`rem-backfill`, live dreaming daily ingestion, historical grounded seeding, and Gateway Dream Diary backfill; update docs/tests to make the intended cross-system contract explicit while preserving deterministic ordering and source paths.
