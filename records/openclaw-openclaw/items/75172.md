Current main still has the gateway crash path, and the PR head appears to address the correct boundary by combining defensive async logger guards with logger/runtime map backfill after deferred channel plugin reload; the remaining work is maintainer validation, CI triage, and merge review.

## What This Changes

The PR updates gateway channel startup to tolerate missing channel logger entries, backfills logger/runtime maps after deferred channel plugin reload, and adds a regression test for the deferred-channel startup race.

## Best Possible Solution

Land this PR or an equivalent patch that preserves both pieces of the fix: synchronize `channelLogs`/`channelRuntimeEnvs` after deferred channel plugin reload and defensively guard async logger continuations.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a configured channel plugin that is missing when logger/runtime maps are built, becomes active after deferred plugin reload, and exits during startup; current main can then dereference an undefined logger in the async continuation.

## Solution Assessment

Yes. The current PR head is the narrow maintainable fix because optional logger guards prevent the crash while the deferred-reload backfill restores the required runtime/logger entries for newly active channel plugins.
