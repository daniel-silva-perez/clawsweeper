## Summary

Keep this PR open. Current main still has only the process-local BlueBubbles reply-context cache; it does not implement the opt-in API fallback, the config surface, the cache-miss fetch helper, or this PR's attachment-error/log-redaction follow-ups. The overlapping #73241 is relevant, but it is still open and has not been made canonical by a maintainer; the latest discussion says this PR folded in that typed-client approach while preserving dedupe/cache write-back.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose between this PR and #73241, then land one BlueBubbles-owned implementation that uses the typed `BlueBubblesClient`, preserves SSRF protection and config inheritance, validates reply IDs, writes successful lookups back to the reply cache, covers dedupe/failure cases with tests, and updates docs/changelog as needed.
