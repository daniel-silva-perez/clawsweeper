## Summary

Keep open. Current main still lacks the BlueBubbles reply-context API fallback, and this PR remains an active, scoped implementation candidate. The overlapping #73241 is relevant but has not been chosen as canonical by a maintainer.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose between this implementation and #73241, land one BlueBubbles-owned fallback with SSRF-guarded client behavior, cache-miss/failure/dedupe coverage, config inheritance preserved, and docs/changelog updated, then close or supersede the other PR explicitly.
