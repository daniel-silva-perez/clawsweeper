## Summary

Current main still has the reported elevated-exec contract: allowed elevated sessions fall back to level `on`, the exec tool treats `on`/`ask` as elevated for calls that omit `elevated: false`, and elevated target resolution routes those calls to the gateway unless the target is already `node`. The docs and tests now describe that current behavior, so the requested sandbox-default, per-call opt-in behavior remains unresolved and needs maintainer product/security review rather than cleanup closure.

## What This Changes

The issue asks for `tools.elevated.enabled` to preserve sandbox exec by default, make elevation opt-in per exec call/session, and default `agents.defaults.elevatedDefault` to `off` instead of silently routing plain exec calls to the gateway.

## Best Possible Solution

Keep this open for maintainer decision on the elevated-mode contract. If maintainers choose the safer opt-in behavior, update reply default resolution, directive persistence/status text, exec elevated-mode handling, system prompt copy, public docs, and focused regression tests for sandboxed sessions with `tools.exec.host=auto|sandbox` and explicit per-call `elevated` values. If default-on elevated sessions are intentional, document that product/security decision clearly and close only after maintainer confirmation.
