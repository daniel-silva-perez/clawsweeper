## Summary

Keep open. Current main still matches the reported behavior rather than resolving it: allowed elevated sessions fall back to elevated `on`, omitted per-call `elevated` inherits that active default, and elevated target resolution routes out of the sandbox to `gateway` unless the selected target is already `node`. The docs and tests now codify that contract, and the linked default-off PR was closed unmerged, so this needs maintainer product/security judgment rather than cleanup closure.

## What This Changes

The issue asks for `tools.elevated.enabled` to preserve sandbox exec as the default, with gateway or node host exec only when a session or per-call request explicitly opts into elevation.

## Best Possible Solution

Maintainers should explicitly choose the elevated-mode contract. If sandbox-default opt-in elevation is accepted, update directive resolution, exec elevated-mode handling, prompt/status wording, public docs, and regression tests so enabling `tools.elevated` alone does not route omitted-elevated exec calls to the host. If default-on elevated sessions are intentional, document that security decision clearly before closing.
