## Summary

Keep open. Current main still matches the reported routing path: allowed elevated sessions fall back to `on`, omitted `exec` calls inherit that active elevated mode, and elevated target resolution sends the command to `gateway` unless the target is already `node`. The related default-off PR is closed unmerged, while current docs and tests now encode the default-on/ask-as-on contract, so this needs maintainer product and security judgment rather than cleanup closure.

## What This Changes

The issue asks for `tools.elevated.enabled: true` to preserve sandbox exec by default and route only explicit elevated exec calls to gateway or node host execution.

## Best Possible Solution

Maintainers should settle the elevated-mode contract. If sandbox-default opt-in elevation is accepted, update directive resolution, exec elevated-mode inheritance, prompt/status wording, public docs, and regression tests together so enabling `tools.elevated` alone does not move omitted-elevated exec calls to host execution. If default-on elevated sessions are intentional, keep the current implementation but document the security tradeoff consistently across exec and elevated docs before closing.
