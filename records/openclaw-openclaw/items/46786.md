## Summary

Keep open. Current main still matches the reported routing chain: allowed elevated sessions without an explicit directive, session override, or configured `agents.defaults.elevatedDefault` fall back to `on`; omitted `exec` calls inherit that active elevated default; and elevated target resolution routes to `gateway` unless the selected target is already `node`. The latest release tag also contains the same contract, and the related default-off PR was closed unmerged, so this needs maintainer product/security judgment rather than stale cleanup.

## What This Changes

The issue asks for `tools.elevated.enabled` to leave non-elevated `exec` calls sandboxed by default and make host execution opt-in via an explicit elevated request or approved session mode.

## Best Possible Solution

Maintainers should explicitly choose the elevated-mode security contract. If sandbox-default opt-in elevation is desired, update default resolution, directive persistence/status, exec inheritance, prompt wording, docs, and regression tests together; if default-on host execution is intentional, keep the implementation but make the exec docs and security guidance state the tradeoff clearly.
