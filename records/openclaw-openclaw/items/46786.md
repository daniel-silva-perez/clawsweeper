## Summary

Keep open. Current main still matches the reported routing path: allowed elevated sessions fall back to `on`, omitted `exec` calls inherit that active elevated mode, and elevated target resolution routes to `gateway` unless the selected target is already `node`. The linked default-off PR was closed unmerged, while current docs and tests encode default-on and `/elevated ask` as an alias for `on`, so this needs maintainer product/security judgment rather than stale cleanup.

## What This Changes

The issue requests that enabling `tools.elevated.enabled` not make ordinary sandboxed `exec` calls route to the gateway host unless elevation is explicitly requested or selected.

## Best Possible Solution

Maintainers should decide the elevated-mode contract explicitly. If sandbox-default opt-in elevation is the desired model, update directive resolution, session persistence/status, exec inheritance, prompt wording, public docs, and regression tests together. If default-on elevated sessions are intentional, keep the implementation but reconcile the exec docs and security guidance so the operator tradeoff is clear.
