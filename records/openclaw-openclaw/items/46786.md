## Summary

Keep open. Current main still matches the reported routing path: allowed elevated sessions fall back to `on`, omitted `exec` calls inherit that active elevated mode, and elevated target resolution routes to `gateway` unless the selected target is already `node`. The linked default-off PR is closed unmerged, while current docs and tests encode the default-on and ask-as-on behavior, so this needs maintainer product/security judgment rather than cleanup closure.

## What This Changes

The issue requests sandbox-default exec routing when `tools.elevated.enabled` is true, with gateway elevation only when explicitly requested per call or session.

## Best Possible Solution

Maintainers should decide the elevated-mode contract explicitly. If sandbox-default opt-in elevation is the desired model, update directive resolution, session persistence/status, exec inheritance, prompt wording, public docs, and regression tests together so enabling `tools.elevated` alone does not move omitted-elevated exec calls to host execution. If default-on elevated sessions are intentional, keep the implementation but reconcile the exec docs and security guidance so the operator tradeoff is clear.
