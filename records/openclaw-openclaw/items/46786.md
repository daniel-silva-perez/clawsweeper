## Summary

Keep open. Current main still implements the elevated-exec behavior reported here: allowed sessions fall back to elevated `on`, omitted per-call `elevated` inherits that elevated default, and elevated target resolution leaves the sandbox for `gateway` unless the selected target is already `node`. The public docs and focused tests now describe that behavior, while the related PR that proposed defaulting elevated to `off` was closed unmerged, so this needs maintainer product/security judgment rather than cleanup closure.

## What This Changes

The issue asks for enabled elevated mode to preserve sandbox exec as the default, with host execution only when a session or per-call request explicitly opts into elevation.

## Best Possible Solution

Maintainers should explicitly choose the elevated-mode contract. If sandbox-default opt-in elevation is accepted, update directive resolution, exec elevated-mode handling, prompt/status wording, public docs, and regression tests so `tools.elevated.enabled` alone does not route omitted-elevated exec calls to the host. If default-on elevated sessions are intentional, document that security decision clearly and close this only after maintainer confirmation.
