## Summary

Keep this PR open. Current main still resolves Discord thread bindings through the current bot account only and does not implement the PR’s intended cross-account suppression for active bound threads. The PR remains a focused implementation candidate, with review centered on avoiding false drops from stale or disabled foreign bindings.

## Best Possible Solution

Keep this PR open and review or replace it as the implementation candidate. The best path is a narrow Discord preflight/thread-binding fix that checks for an active foreign thread owner before configured or generic routing, uses the current effective runtime config to ignore stale, removed, disabled, or threadBindings-disabled owners, preserves same-account binding behavior, and lands focused regressions for foreign-owner suppression plus stale-owner false-drop cases.
