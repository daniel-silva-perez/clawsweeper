## Summary

Keep this PR open. Current main still resolves Discord thread bindings through the current bot account only, so it does not implement the PR’s intended cross-account bound-thread suppression. The PR is a focused implementation candidate, but the remaining review issue is on a routing/security boundary: avoid false drops from stale, removed, disabled, or runtime-disabled foreign bindings.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The best path is a narrow Discord fix that checks for an active foreign thread owner before configured/generic routing, uses the current effective runtime config to ignore stale/removed/disabled/threadBindings-disabled owners, preserves same-account binding behavior, and lands focused regressions for foreign-owner suppression plus stale-owner cases. The security review should stay centered on false message drops; the diff does not introduce dependency, workflow, install, release, or package-resolution changes.
