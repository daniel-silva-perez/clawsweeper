## Summary

Keep this PR open. Current main still does not implement the central doctor behavior: when `cron.webhook` is unset, `migrateLegacyNotifyFallback` only emits a warning and leaves `notify: true` in the cron store. The PR remains useful for the linked cron bug, but it needs a focused rebase because it also bundles unrelated provider API-key normalization and unrelated test churn.

## Best Possible Solution

Keep the PR open, but ask for a focused rebase or replacement PR against current main: update `migrateLegacyNotifyFallback` to remove legacy `notify` when `cron.webhook` is unset and delivery behavior can be preserved, keep explicit warnings for announce and unknown modes where needed, add targeted `src/commands/doctor-cron.test.ts` coverage, and split provider API-key normalization into a separate security-reviewed PR.
