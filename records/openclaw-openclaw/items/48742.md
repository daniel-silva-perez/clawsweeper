Keep the PR open, but ask for a focused rebase or replacement PR against current main: update `migrateLegacyNotifyFallback` to remove legacy `notify` when `cron.webhook` is unset and delivery can be preserved, keep warnings for announce and unknown modes as appropriate, add targeted `src/commands/doctor-cron.test.ts` coverage, and update doctor/cron docs if the behavior changes. Split or drop the provider API-key flip-flop work into a separate security-reviewed PR.
