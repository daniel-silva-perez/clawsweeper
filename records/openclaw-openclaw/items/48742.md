## Summary

Keep this PR open. Current main still does not implement the central doctor behavior: when `cron.webhook` is unset, `migrateLegacyNotifyFallback` only warns and leaves `notify: true` in place, so a separate normalization can still save the store while the legacy notify field remains. The PR contains the relevant doctor fix for #44460, and the author addressed the unknown-mode review feedback in `c954846cc1b64f8f57d5c7d843ed66e4d6faf403`, but it still needs a focused rebase because it bundles unrelated provider API-key normalization and test churn.

## Best Possible Solution

Keep the PR open, but ask for a focused rebase or replacement PR against current main: carry only the `migrateLegacyNotifyFallback` behavior for unset `cron.webhook`, preserve explicit warnings for announce and unknown modes where behavior could change, add targeted `src/commands/doctor-cron.test.ts` coverage, and split the provider API-key normalization work into a separate security-reviewed PR.
