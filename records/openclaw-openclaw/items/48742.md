## Summary

Keep this PR open. Current main still does not implement the central doctor behavior: when `cron.webhook` is unset, `migrateLegacyNotifyFallback` warns and leaves `notify: true` in the cron store. The PR remains useful for #44460, but it needs a focused rebase because it also bundles unrelated provider API-key normalization changes and unrelated test churn.

## Best Possible Solution

Keep the PR open, but ask for a focused rebase or replacement PR against current main: update `migrateLegacyNotifyFallback` to remove legacy `notify` when `cron.webhook` is unset and delivery behavior can be preserved, keep warnings for announce and unknown modes where appropriate, add targeted `src/commands/doctor-cron.test.ts` coverage, and split the provider API-key flip-flop work into a separate security-reviewed PR.
