## Summary

Keep this PR open. Current main still does not include the proposed VirtualBox/Vagrant install surface, and the PR head is no longer docs-only: it adds installer/provisioning scripts with unresolved functional and security-sensitive review blockers.

## Best Possible Solution

Keep the PR open for maintainer review or contributor rework. If OpenClaw wants official VirtualBox support, land it as maintained scripts plus docs navigation after fixing the installer, SSH argument forwarding, first-start status handling, raw .env sourcing, gateway bind/auth/origin instructions, and adding manual VM proof or focused tests. If maintainers do not want to own this install path, redirect it to an explicitly unofficial community-maintained guide rather than merging executable setup scripts into core.
