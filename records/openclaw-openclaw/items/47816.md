## Summary

Keep this PR open. Current main still does not implement the proposed VirtualBox/Vagrant install surface, and the PR head is no longer docs-only: it adds host installers, Vagrant provisioning, package-manager changes, firewall/SSH setup, and several shell execution paths. Some earlier review feedback was addressed by moving the install URL under openclaw/openclaw and adding an explicit Vagrant SSH forward, but functional and security-sensitive blockers remain.

## Best Possible Solution

Keep the PR open for maintainer review or contributor rework. If OpenClaw wants official VirtualBox support, it should land as maintained scripts plus docs navigation, with the installer made source-owned and stdin-safe, SSH argument forwarding fixed, raw .env sourcing removed, gateway bind/auth/origin instructions aligned with current source behavior, and manual VM proof or focused tests. If maintainers do not want to own VirtualBox/Vagrant as an official install path, redirect it to an explicitly unofficial community-maintained guide instead of merging executable setup scripts into core.

## Risks / Open Questions

- The PR is not implemented on current main, so closing as implemented would lose a still-open contribution rather than clean up obsolete work.
- The PR head introduces host and guest code execution paths, package-manager configuration, firewall edits, and SSH exposure; those are security-sensitive install surfaces, not docs-only changes.
- The advertised install and headless workflows currently do not match the shell implementation, so merging as-is would give users broken setup instructions.
- The gateway/WebChat instructions could overpromise host access unless bind mode, auth, allowedOrigins, and Vagrant networking are verified together.
