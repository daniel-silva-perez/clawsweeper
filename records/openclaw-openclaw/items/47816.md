## Summary

Keep this PR open. Current main still does not include the proposed VirtualBox/Vagrant install path, so the PR is not obsolete or implemented-on-main. The contribution is also not ready to treat as a clean docs-only change: the live PR head adds installer and provisioning scripts with unresolved functional and security-sensitive blockers.

## Best Possible Solution

Keep the PR open for maintainer review or contributor rework. If OpenClaw wants official VirtualBox support, it should land as maintained scripts plus docs navigation after fixing the installer path, SSH argument forwarding, raw .env sourcing, LAN SSH posture, gateway bind/auth/origin instructions, and adding manual VM proof or focused validation. If maintainers do not want to own this install path, redirect it to an explicitly unofficial community-maintained guide rather than merging executable setup scripts into core.
