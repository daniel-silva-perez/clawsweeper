## Summary

Keep this issue open. Current main still has a plausible channel-specific skill availability divergence: Telegram can pass per-DM/group/topic skill filters, including an explicit empty list, into the shared reply path, while WebChat chat.send has no equivalent channel skill filter. The vLLM/himalaya path also remains plausible because current docs still call out OpenAI-compatible vLLM cases where skills/tools never run or tool calls arrive empty/text-only, and no current source, docs, tests, or related context prove this exact mismatch has been fixed.

## Best Possible Solution

Keep this issue open. Reproduce on current main with the same agent/session, vLLM gpt-oss-20b, himalaya, and Telegram route; capture Telegram DM/group/topic config, effective replyOptions.skillFilter, built skillsSnapshot, runtime tool list, provider/model metadata, and structured-versus-text tool-call events for both WebChat chat.send and Telegram. If Telegram unintentionally filters out himalaya, fix the Telegram filter inheritance or diagnostics and add a Telegram dispatch regression test. If the effective tools match but vLLM emits empty or text-only tool calls, fix or document that provider/model path.
