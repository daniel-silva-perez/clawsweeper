## Summary

Keep this issue open. Current main still has a credible Telegram/WebChat divergence for skill-backed tool availability: Telegram can attach a per-DM/group/topic skill filter, including an explicit empty filter, to the shared reply path, while WebChat `chat.send` does not pass an equivalent channel skill filter. The vLLM path also remains plausible because the current docs describe OpenAI-compatible vLLM cases where skills/tools never run or tool calls arrive as text/empty arrays. There is no current-main source, test, docs, or related-thread evidence showing this exact vLLM + himalaya + Telegram mismatch has been fixed or superseded.

## Best Possible Solution

Keep this issue open. Reproduce on current main with the same agent/session, vLLM `gpt-oss-20b`, himalaya, and Telegram route; capture Telegram DM/group/topic config, effective `replyOptions.skillFilter`, built `skillsSnapshot`, runtime tool list, provider/model metadata, and structured-versus-text tool-call events for both WebChat `chat.send` and Telegram. If Telegram unintentionally filters out himalaya, fix the Telegram filter inheritance/diagnostics and add a Telegram dispatch regression test. If effective tools match but vLLM emits empty or text-only tool calls, fix or document that provider/model path.
