## Summary

Keep this issue open. Current main still has a plausible Telegram/WebChat divergence in skill-backed tool availability: Telegram can pass per-DM/group/topic skill filters, including an explicit empty list, into reply execution, while WebChat `chat.send` does not pass an equivalent channel skill filter. There is no source, docs, test, or related-thread evidence in the provided context showing a merged fix or canonical replacement for the reported vLLM + himalaya mismatch.

## Best Possible Solution

Keep the issue open and reproduce against current main with the same agent/session, vLLM `gpt-oss-20b` model, himalaya skill, and Telegram route. Capture Telegram DM/group/topic config, effective `replyOptions.skillFilter`, built `skillsSnapshot`, available runtime tools, provider/model metadata, and structured-versus-text tool-call events for both WebChat `chat.send` and Telegram. If Telegram unintentionally filters out himalaya, fix Telegram filter inheritance or diagnostics and add a Telegram dispatch regression test; if effective tools match but vLLM emits empty/text-only tool calls, fix or document that provider/model path.
