## Summary

Keep this issue open. Current main still has a plausible Telegram/WebChat divergence for skill-backed tool availability: Telegram can pass per-DM/group/topic skill filters into reply execution, including an explicit empty list, while WebChat `chat.send` does not pass a channel skill filter. The provided discussion has no maintainer resolution, merged fix, accepted workaround, or canonical duplicate for the reported vLLM + himalaya mismatch.

## Best Possible Solution

Keep the issue open and reproduce on current main with the same agent/session, vLLM `gpt-oss-20b` model, himalaya skill, and Telegram route. Capture Telegram DM/group/topic config, effective `replyOptions.skillFilter`, built `skillsSnapshot`, available runtime tools, provider/model metadata, and structured-versus-text tool-call events for both WebChat `chat.send` and Telegram. If Telegram unintentionally loses himalaya through route filtering, fix Telegram filter inheritance or diagnostics and add a Telegram dispatch regression test; if vLLM emits empty or text-only tool calls with the same effective tools, document or fix that provider/model path.
