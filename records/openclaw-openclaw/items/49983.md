## Summary

Keep open. Current main still sends structured Feishu mention targets as pseudo-XML text inside one post `md` element, while this PR preserves the structured mention metadata and emits native Feishu post `at` elements; the remaining blocker is that the live PR is currently dirty/unmergeable against main and needs maintainer follow-up before landing.

## What This Changes

This PR changes Feishu outbound post-message rendering to pass structured mention targets into the post payload builder, emit native `at` elements with optional `user_name` fallbacks, keep body `<at ...>` text literal, add focused send/payload tests, export mention helpers from the Feishu entrypoint, and add a changelog entry.

## Best Possible Solution

Land this PR after rebasing or otherwise resolving the dirty branch, keeping the structured outbound mention path and its regression tests; if the source branch cannot be updated cleanly, replace it with an equivalent narrow Feishu plugin fix that preserves contributor credit and the safer behavior of not reparsing arbitrary body text into native mentions.
