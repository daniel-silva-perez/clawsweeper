## Summary

Keep this PR open. Current main still sends Feishu bot reply mentions by flattening structured mention targets into pseudo-XML text inside a single `md` post element, so the native Feishu `at` element behavior proposed by this PR is not implemented on main. The provided discussion also shows recent maintainer repair work on the PR branch, making it an active implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or replace it with a merged canonical Feishu fix. The fix belongs in the Feishu plugin: pass structured mention targets into `buildFeishuPostMessagePayload`, emit native post `at` elements with `user_name` fallbacks, preserve literal body `<at ...>` text, keep focused regression tests, and preserve contributor changelog credit.
