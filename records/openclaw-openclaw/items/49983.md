## Summary

Keep this PR open. Current main still builds Feishu post replies by converting structured mention targets into pseudo-XML text and sending one `md` post element, so the PR's native Feishu `at` element behavior is not implemented on main. The PR is an active implementation candidate for the closed Feishu mention-color issue and is not superseded by a merged current-main fix.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or replace it with a merged canonical Feishu fix. The fix should stay in the Feishu plugin: pass structured mention targets into `buildFeishuPostMessagePayload`, emit native post `at` elements with `user_name` fallbacks, preserve literal body `<at ...>` text, keep focused regression tests, and include changelog credit.
