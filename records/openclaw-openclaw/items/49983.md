## Summary

Keep this PR open. Current main still flattens Feishu structured mention targets into pseudo-XML text inside a single post `md` element, so the native post `at` element behavior proposed here is not implemented. The PR remains an active implementation candidate after a maintainer branch repair, and the only related native-mention PR in the provided context is closed unmerged rather than a main-branch replacement.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or replace it with an equivalent merged Feishu plugin fix. The best code path is to preserve structured mention targets into `buildFeishuPostMessagePayload`, emit native Feishu post `at` elements with `user_name` fallbacks for explicit mention targets, leave literal body `<at ...>` text as text, keep focused regression tests, and confirm the entrypoint export expansion before merge.
