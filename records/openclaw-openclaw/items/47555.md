## Summary

Keep open. Current main still applies explicit root `--profile` before command dispatch, but `applyCliProfileEnv` leaves inherited `OPENCLAW_LAUNCHD_LABEL` intact; the macOS launchd lifecycle and handoff resolvers prefer that inherited label over `OPENCLAW_PROFILE`, so a restart launched from a managed gateway session can still target the current session's LaunchAgent. The prior fix PR #47594 is closed and unmerged, and #65643 remains open for the same stale-label root cause on `gateway status`.

## What This Changes

The issue asks for `openclaw --profile <name> gateway restart` invoked from an agent or gateway session to restart the specified profile's LaunchAgent instead of the inherited current session profile.

## Best Possible Solution

Explicit root `--profile` selection should make profile-derived service identity win for gateway lifecycle, status, plist resolution, and detached restart handoff paths, while preserving custom launchd label overrides for non-profile managed contexts.
