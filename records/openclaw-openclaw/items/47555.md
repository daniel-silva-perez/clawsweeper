## Summary

Keep open. Current main still lets an explicit root `--profile` coexist with inherited `OPENCLAW_LAUNCHD_LABEL`; the macOS launchd lifecycle code resolves that inherited label before the profile-derived label, so `gateway restart` can still target the current session's LaunchAgent instead of the requested profile. The related fix PR #47594 is closed and unmerged, while #65643 remains open for the same stale-label root cause on `gateway status`.

## What This Changes

Fix `openclaw --profile <profile> gateway restart` from an agent session so it targets the requested profile's gateway instead of the inherited session gateway.

## Best Possible Solution

Explicit root `--profile` selection should win over inherited gateway service identity for lifecycle and status paths. Custom launchd labels should still work in managed contexts that did not pass a root profile, but a profile-targeted command should resolve the profile-derived LaunchAgent label and associated plist/port/config state.
