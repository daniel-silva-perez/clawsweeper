## Summary

Current main still appears to have the reported macOS gateway lifecycle profile override bug. Root `--profile` sets `OPENCLAW_PROFILE`, but it does not clear an inherited `OPENCLAW_LAUNCHD_LABEL`; launchd label resolution still prefers that label, and `gateway restart` passes `process.env` into the service restart path. The prior PR #47594 is closed and unmerged, while #65643 tracks the same stale-label root cause for `gateway status`; this restart issue should remain open because it is the more destructive surface.

## What This Changes

Make `openclaw --profile <name> gateway restart` from an agent/gateway session target the requested profile instead of the inherited current session LaunchAgent.

## Best Possible Solution

Keep this issue open and fix explicit root `--profile` handling so it wins over inherited gateway service context for lifecycle commands. The narrow likely fix is to clear stale `OPENCLAW_LAUNCHD_LABEL` when `applyCliProfileEnv` is invoked for a parsed root profile, then add focused tests proving `openclaw --profile <name> gateway restart/status/stop` resolves the profile-derived LaunchAgent label while preserving intentional custom label behavior when no explicit root profile is supplied. Coordinate the same PR with #65643 and use #47594 as prior context, not as a merged fix.
