## Summary

Keep open. Current main still allows an explicit root `--profile` to coexist with an inherited `OPENCLAW_LAUNCHD_LABEL`, and the macOS restart path resolves that inherited label before the profile-derived label while passing `process.env` into the service restart. The related fix PR #47594 is closed and unmerged, and #65643 remains open for the same stale-label root cause, so this should stay tracked as a narrow lifecycle bug.

## What This Changes

The issue asks for `openclaw --profile <name> gateway restart` to honor the explicit profile inside an agent session instead of restarting the inherited session gateway.

## Best Possible Solution

Make explicit root `--profile` selection win over inherited gateway service identity for lifecycle commands. The desired end state is that `gateway restart`, `status`, and `stop` resolve the profile-derived LaunchAgent label when root parsing accepted `--profile`, while custom launchd labels still work for managed contexts that did not pass an explicit root profile.
