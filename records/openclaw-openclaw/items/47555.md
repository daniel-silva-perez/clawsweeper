## Summary

Current main still appears to have the reported macOS gateway lifecycle profile override bug. The root `--profile` path sets `OPENCLAW_PROFILE`, but it does not clear an inherited `OPENCLAW_LAUNCHD_LABEL`; launchd label resolution still prefers that label over the profile, and gateway restart passes `process.env` through to the service restart implementation. The related PR #47594 is closed and unmerged, while #65643 tracks the same stale-label root cause for `gateway status`; this restart issue should stay open because it is the more destructive surface.

## Best Possible Solution

Keep this issue open and fix explicit root `--profile` handling so it wins over inherited gateway service context for lifecycle commands. The narrow likely fix is to clear stale `OPENCLAW_LAUNCHD_LABEL` when `applyCliProfileEnv` is invoked for a parsed root profile, then add focused tests proving `openclaw --profile <name> gateway restart/status/stop` resolves the profile-derived LaunchAgent label while preserving intentional custom label behavior when no explicit profile is supplied. Coordinate the same PR with #65643.
