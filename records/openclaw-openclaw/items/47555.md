## Summary

Current main still has the reported profile/launchd-label precedence bug: the root `--profile` path sets `OPENCLAW_PROFILE`, but inherited `OPENCLAW_LAUNCHD_LABEL` remains present and is still preferred by the macOS launchd lifecycle resolver. The prior fix PR #47594 is closed and unmerged, and #65643 remains open for the same stale-label root cause on `gateway status`.

## What This Changes

The issue requests that `openclaw --profile <name> gateway restart` honor the explicit profile inside an agent session instead of restarting the inherited session gateway.

## Best Possible Solution

Explicit root `--profile` selection should make profile-derived service identity win for lifecycle and status commands, while still allowing custom launchd labels in managed contexts that do not pass a root profile. The fix should cover restart target resolution, plist/status resolution, and detached restart handoff behavior.
