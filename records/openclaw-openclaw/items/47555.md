## Summary

Keep open. Current main still applies the explicit root `--profile` before dispatch, but it does not clear or re-derive an inherited `OPENCLAW_LAUNCHD_LABEL`; the macOS launchd paths still resolve that inherited label before the profile-derived LaunchAgent label, so `gateway restart` can still target the current session service instead of the requested profile. The linked fix PR #47594 is closed and unmerged, and #65643 remains open for the same stale-label root cause on `gateway status`.

## What This Changes

The issue asks for root `--profile <name>` to take precedence over inherited agent-session gateway identity when running `gateway restart`.

## Best Possible Solution

Explicit root `--profile` selection should make the profile-derived service identity win for launchd lifecycle, status, plist resolution, and detached restart handoff paths, while custom launchd labels remain honored in contexts that did not pass a root profile.
