## Summary

Current main still appears to have the reported macOS gateway lifecycle profile override bug. The root `--profile` path sets `OPENCLAW_PROFILE`, but it does not clear an inherited `OPENCLAW_LAUNCHD_LABEL`; launchd service resolution still prefers that label, and `gateway restart` passes `process.env` into the restart path. The prior fix PR #47594 is closed and unmerged, while #65643 tracks the same stale-label root cause for `gateway status`, so this restart issue should stay open and can be fixed with the same narrow profile-env sanitation work.

## What This Changes

The issue reports that `openclaw --profile <name> gateway restart` from an agent session restarts the inherited session gateway instead of the requested profile gateway.

## Best Possible Solution

Make explicit root `--profile` selection win over inherited gateway service identity for lifecycle commands. The narrow end state is that `gateway restart`, `status`, and `stop` resolve the profile-derived LaunchAgent label when the root parser accepted `--profile`, while custom launchd labels still work for service-managed contexts that did not pass an explicit root profile.
