## Summary

Keep open. Current main still allows an explicit root `--profile` to coexist with an inherited `OPENCLAW_LAUNCHD_LABEL`; the launchd resolver uses that label before deriving the profile label, so a restart from a managed gateway/agent session can still schedule the current session’s LaunchAgent instead of the requested profile. The earlier fix PR is closed and unmerged, and the related status bug remains open under the same stale-label root cause.

## What This Changes

The issue requests that `openclaw --profile <name> gateway restart` target the selected profile’s gateway instead of the inherited agent-session gateway.

## Best Possible Solution

Explicit root `--profile` selection should make profile-derived service identity win for gateway lifecycle, status, plist resolution, and detached restart handoff paths, while preserving custom launchd labels for contexts that did not request a different profile.
