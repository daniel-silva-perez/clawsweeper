## Summary

Keep open. Current main still has the raw Gateway chat projection leak in per-run SDK streams, and this PR addresses a real dogfood finding from #74704, but the proposed fallback drops later chat-only delta frames and needs a narrow fix before merge.

## What This Changes

The PR updates `@openclaw/sdk` run event streaming to suppress duplicate raw Gateway `chat` projection frames, normalize chat-only projections into SDK events, and adds SDK unit coverage plus a changelog entry.

## Best Possible Solution

Keep the SDK contract centered on stable per-run events: suppress raw chat projections only when canonical agent lifecycle/assistant events are present, preserve all chat-only delta projections as `assistant.delta` until a terminal event, keep raw Gateway access in `oc.rawEvents()`, and cover duplicate suppression plus multi-delta fallback in focused SDK tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve later chat-only deltas:** `packages/sdk/src/client.ts:344-345`
  - body: When the fallback path sees its first raw `chat` delta, it sets `hasAssistantRunEvent`, so the next `chat` delta for a chat-only run is treated as a duplicate and dropped. The Gateway can emit multiple `state: "delta"` chat projections as buffered assistant text grows, so chat-only `Run.events()` consumers would receive only the first delta and stale UI until final. Keep canonical-agent detection separate from fallback chat deltas.
  - confidence: 0.88
