## Summary

Keep open. Current main still has the reported SDK leak path, and this PR targets the right surface, but the latest patch still drops later chat-only `delta` projections after the first fallback event, which would leave chat-only `Run.events()` consumers stale until the final event.

## What This Changes

The PR updates `@openclaw/sdk` run event streaming to suppress duplicate raw Gateway `chat` projection frames, normalize chat-only projections, add focused SDK unit coverage, and add a changelog entry.

## Best Possible Solution

Keep the SDK contract centered on stable per-run events: suppress raw chat projections only when canonical agent lifecycle or assistant events are present, preserve every chat-only `delta` projection as `assistant.delta` until a terminal event, keep `oc.rawEvents()` as the raw escape hatch, and lock the behavior with duplicate-suppression plus multi-delta fallback tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve later chat-only deltas:** `packages/sdk/src/client.ts:344-345`
  - body: When the fallback path sees its first raw `chat` delta, it sets `hasAssistantRunEvent`, so the next `chat` delta for a chat-only run is treated as a duplicate and dropped. Gateway chat projections can emit multiple `state: "delta"` frames as buffered assistant text grows, so chat-only `Run.events()` consumers would receive only the first update and stale UI until final. Keep canonical-agent detection separate from fallback chat deltas.
  - confidence: 0.88
