## Summary

Keep this PR open. Current main still has the asymmetry described: bare session reset prompt assembly omits inboundUserContext even though the empty-body guard accepts non-empty InboundHistory, so the PR is still a live implementation candidate rather than obsolete. The production patch is narrow, but the added test still needs to force the bare reset branch before maintainers rely on it as regression coverage.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is to fix the regression test so it actually exercises the bare reset branch, then review and land the one-line baseBodyForPrompt change if focused tests pass.
