## Summary

Keep this PR open. Current main at 6d323ee73640afdf02ed9fa3aded13da23359b98 still lacks the proposed memory-core weighted-recall feedback loop and the three SkillFoundry skills, so this cannot close as implemented. It also should not close as ClawHub-only because the memory-core ranking/feedback portion is a bundled memory-plugin behavior change, even though the new skill documents should likely be split out and handled through ClawHub or a separate maintainer-approved path. PR: https://github.com/openclaw/openclaw/pull/56023.

## Best Possible Solution

Keep this PR open, but ask for it to be split or narrowed. The memory-core work should continue as a focused PR with a stable result/feedback target contract, consistent weighted ranking across all search modes, safe persistence semantics, docs, and targeted tests. The SkillFoundry skills should move through ClawHub or a bundle-style plugin unless maintainers approve bundling them in the OpenClaw repo.

## Risks / Open Questions

- The PR mixes two product surfaces: memory-core search feedback behavior and repo-root SkillFoundry skills. That makes review and ownership harder than a focused PR.
- The feedback CLI as described needs a stable way for users and agents to discover/select feedback targets; current main's public memory result type does not expose chunk IDs.
- The proposed file-backed weight persistence still needs maintainer review for concurrent updates and for applying/re-sorting weights consistently across FTS-only, vector-only, and hybrid paths.
- The three skill documents likely belong in ClawHub or a bundle-style plugin unless maintainers explicitly want these as bundled repo skills.
