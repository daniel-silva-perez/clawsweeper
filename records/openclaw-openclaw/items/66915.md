## Summary

Keep open. Current main still suppresses ordinary verbose-off `edit` mismatch details at the payload layer, adjacent tests preserve generic mutating-tool warnings, and PR #67276 remains the open unmerged implementation candidate for this exact report. The requested config-side workaround is already possible through `tools.deny`, but the central user-facing mismatch guidance is still unresolved on main.

## What This Changes

Improve `edit` exact-match failure surfacing so whitespace and newline mismatches produce actionable user-facing guidance, with a deployment workaround to disable `edit` in favor of `write`.

## Best Possible Solution

Review PR #67276 as the focused fix. If accepted, land a sanitized payload-layer summary with regression coverage for exact-text whitespace/newline mismatches while keeping `edit` semantics unchanged; use existing `tools.deny` policy for deployments that want to disable `edit`, and leave broader edit normalization/read-positioning ideas to #62863.
