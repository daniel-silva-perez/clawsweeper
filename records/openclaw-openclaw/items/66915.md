## Summary

Keep open. Current main still suppresses ordinary verbose-off edit exact-match mismatch details into a generic mutating-tool warning, while the lower edit layer has richer raw mismatch context that should not be surfaced directly. PR #67276 remains the explicit open implementation candidate for the remaining user-facing warning fix.

## What This Changes

The issue asks for clearer, actionable user-facing feedback when the edit tool fails because oldText does not exactly match file contents, plus a way to disable edit in favor of write.

## Best Possible Solution

Review, refresh, and land or replace PR #67276 as the focused fix: emit sanitized actionable guidance for the known edit exact-match mismatch case in verbose-off output, keep edit semantics unchanged, and rely on existing tools.deny policy for deployments that want to disable edit.
