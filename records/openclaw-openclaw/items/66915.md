## Summary

Keep open. Current main still suppresses ordinary verbose-off `edit` mismatch details into a generic mutating-tool failure warning, while the lower edit layer has richer mismatch context that is too raw to surface directly. The config-side disable path already exists through `tools.deny`, and open PR #67276 remains the explicit unmerged implementation candidate for the remaining user-facing warning fix.

## What This Changes

The issue requests clearer user-facing guidance when `edit` `oldText` fails due to exact whitespace/newline mismatch, plus a configuration path to disable `edit` in favor of `write`.

## Best Possible Solution

Review PR #67276 as the focused fix path, or land an equivalent payload-layer summarizer with regression coverage. The desired end state is sanitized exact-text mismatch guidance in verbose-off output, unchanged `edit` semantics, existing `tools.deny` policy as the disable-edit path, and broader edit normalization/read-positioning work left to #62863.
