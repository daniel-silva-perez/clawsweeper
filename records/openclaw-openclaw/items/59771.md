## Summary

Keep this PR open. Current main still has the Feishu `onError` path close the active streaming card, and `FeishuStreamingSession` still lacks the proposed `clearText()` recovery hook, so the PR’s central behavior is not implemented on main. The linked duplicate-reply issue remains open, and the PR is a plausible implementation candidate, but it still needs maintainer review because it also bundles sister sanitizer/reasoning changes and has unresolved functional review concerns.

## Best Possible Solution

Keep this PR open for focused maintainer review. The best path is to either narrow it to the Feishu streaming recovery fix or explicitly coordinate it with #54899/#54898, then prove the recovery path with tests for divergent retry text, streaming close failures, and `onError` sequencing; #49381 should remain the canonical user bug until a verified fix lands.
