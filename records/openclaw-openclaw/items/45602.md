## Summary

Current main still does not implement this PR's intended Web UI behavior. `/status` remains absent from the Control UI local command set, the slash-menu Enter handler only sends immediately for `executeLocal` no-arg commands, and the current regression test explicitly asserts `/status` stays off the local path. The PR remains a plausible implementation candidate for the linked open bug, but the PR discussion and current status implementation show it needs maintainer review/revision before merge so the one-Enter UX does not lose the richer runtime status semantics.

## Best Possible Solution

Keep this PR open, but revise it before merge. The best path is to make selecting `/status` from the Control UI slash menu submit in one Enter while preserving the shared runtime status semantics, either by routing the no-arg agent-side status command directly or by adding a gateway-backed local status executor that reuses the same resolved status inputs. Add focused Control UI coverage for the single-Enter slash-menu flow and status correctness with per-model thinking and fast-mode defaults.
