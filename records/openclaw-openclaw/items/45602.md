## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: `/status` is not a Control UI local command, the slash menu only sends on first Enter for no-arg commands with `executeLocal`, and the current regression test explicitly preserves `/status` on the agent path. The PR remains a plausible implementation candidate for the linked Web UI bug, but the current PR shape still needs maintainer review because it moves `/status` off the richer agent-side status builder and the discussion identifies unresolved per-model default accuracy concerns.

## Best Possible Solution

Keep this PR open, but revise it before merge. The best path is either to make selecting a no-argument agent-routed command like `/status` submit in one Enter while preserving the existing agent-side status builder, or to add a gateway-backed local status path that reuses the same status inputs and resolves per-session thinking and fast-mode defaults accurately. Add focused Control UI regression coverage for the single-Enter slash-menu flow and status correctness with configured per-model defaults.
