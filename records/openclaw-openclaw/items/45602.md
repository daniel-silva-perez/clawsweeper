## Summary

Keep open. Current main still does not implement the PR's intended Control UI behavior: `/status` is not marked `executeLocal`, selecting it from the slash menu still fills the draft instead of sending, current tests assert it stays on the agent path, and there is no local `status` executor on main. The PR remains a plausible implementation candidate for the linked Web UI bug, but it needs maintainer review/revision because the existing agent-side status path is richer and the PR discussion identifies unresolved correctness concerns around per-model thinking and fast-mode defaults.

## Best Possible Solution

Keep this PR open, but revise it before merge. The best path is either to make selecting a no-argument agent-routed command like `/status` submit in one Enter while preserving the shared status builder, or to add a gateway-backed local status path that reuses the same runtime status inputs and resolves per-session thinking and fast-mode defaults accurately. Add focused Control UI coverage for the single-Enter slash-menu flow and status correctness with configured per-model defaults.
