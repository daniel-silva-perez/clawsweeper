## Summary

Keep open. Current main still does not implement this PR's intended one-Enter Web UI behavior for `/status`: the command is still kept off the Control UI local execution path, the slash-menu Enter handler only sends immediately for no-arg `executeLocal` commands, and current tests explicitly assert `/status` is not local. The PR is a plausible implementation candidate for #45569, but its local status path needs maintainer revision so it preserves the shared runtime status semantics rather than replacing them with a simplified UI formatter.

## Best Possible Solution

Keep this PR open, but revise it before merge. The best product path is to make selecting `/status` from the Control UI slash menu submit with one Enter while preserving the shared runtime status semantics, preferably by routing to a gateway-backed/shared status builder or by adding a local executor that uses the same resolved status inputs. Add focused Control UI coverage for the single-Enter menu flow plus status correctness for active-session thinking and fast-mode defaults.
