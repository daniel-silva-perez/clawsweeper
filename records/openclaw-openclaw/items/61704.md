## Summary

Keep this PR open. Current main still logs the original `gmail watcher not started: <reason>` message and the test suite still asserts that exact payload, so the PR's intended Gmail watcher clarification is not implemented on main. The PR is also not merge-ready as-is: its proposed suffix applies “this is expected” to all actionable non-start reasons, while current watcher code can return real failures such as missing `gog` or Tailscale setup errors, and the provided PR diff includes unrelated exec-approval docs/navigation/glossary changes.

## Best Possible Solution

Keep the PR open, but ask for a focused revision: limit changes to the Gmail watcher lifecycle and its test, only add external-webhook guidance for the specific missing-topic/external-watcher scenario, avoid hardcoding a non-configured endpoint, and remove or split the unrelated exec-approval docs changes.
