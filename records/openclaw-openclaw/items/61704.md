## Summary

Keep this PR open. Current main still has the original Gmail watcher warning, so the requested clarification is not implemented there, but the proposed branch is not ready because the new wording treats real watcher startup failures as expected external-webhook operation and the added docs page is malformed/incomplete.

## What This Changes

The PR updates the Gmail watcher warning text and matching test, adds a new exec approval limitations docs page, links it in docs navigation, and adds zh-CN glossary terms.

## Best Possible Solution

Keep the useful diagnostic idea, but implement it narrowly: only add external-webhook guidance for the specific internal-watcher skip reason such as `gmail topic required`, avoid saying real failures are expected, reference the configured hook endpoint or the skip env var accurately, and either remove or complete the unrelated exec-approval docs change in a separate docs PR.

## Reproduction Assessment

Yes for the PR defects: mock `startGmailWatcher` to return `gog binary not found` or a tailscale failure and the proposed warning would say the non-start is expected; inspect the added docs page and the unclosed `bash` fence is visible at EOF. Current main verification also confirms the requested log clarification is not already implemented.

## Solution Assessment

No. The proposed direction is useful, but this patch is broader than needed and the blanket wording is not the narrowest maintainable fix; a conditional message tied to expected internal-watcher skip reasons would be safer.
