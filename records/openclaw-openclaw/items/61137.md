## Summary

Keep open. This issue is recent, has multiple independent confirmations across Telegram, Discord, direct Anthropic, and Codex/OpenAI-style routing, and current main still has a plausible strict JSON.parse path that can surface the reported short SyntaxError text to chat channels. Related #70537 is useful context for adjacent Gateway/OpenResponses serialization hardening, but it does not prove this direct Anthropic/cross-channel parse failure is fixed.

## Best Possible Solution

Keep this issue open. The best maintainer path is to add a focused regression around direct Anthropic anthropic-messages streaming with tool calls and external-channel delivery, then either harden the Anthropic SSE frame parse path or normalize JSON.parse SyntaxError failures so raw parser text cannot be posted to Telegram or Discord. Coordinate with #70537 only for the narrower Gateway/OpenResponses serialization surface; do not treat it as a full canonical replacement for this issue.
