## Summary

Current main still lacks the requested exact `NO_REPLY` guard in `sendMessageTelegram`, while Slack has the parity guard and Telegram only has higher-level delivery normalization. An open PR (#68304) uses closing syntax for this issue, so the issue should stay open until that PR is reviewed/merged or replaced.

## Best Possible Solution

Keep this issue open and review/land #68304 if maintainers accept the narrow adapter guard, or replace it with an equivalent focused Telegram plugin patch that imports `isSilentReplyText`, returns a synthetic suppressed result before Telegram API work for exact bare `NO_REPLY` with no media/reply markup, and adds `extensions/telegram/src/send.test.ts` regression coverage. Track broader pre-sentinel streaming-buffer behavior separately.
