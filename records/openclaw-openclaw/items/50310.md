## Summary

Keep open. Current main has partial stale marking for archived Discord threads and generic manual session maintenance, but it still does not remove orphaned Discord channel/thread session entries, expose an agent-facing sessions_delete/sessions_gc tool, or run a Discord liveness sweep over sessions.json.

## Best Possible Solution

Implement a Discord-owned cleanup path that reconciles agent:<agentId>:discord:channel:* session entries against Discord channel/thread state, removes or hides confirmed orphaned entries from sessions.json/sessions_list, and covers archived, deleted/404, inaccessible/403, multi-account, and missing-transcript cases with focused tests. Reuse the generic session maintenance primitives where appropriate, but add the missing Discord liveness check and an intentionally scoped admin or agent-safe control surface if agents should self-maintain sessions.
