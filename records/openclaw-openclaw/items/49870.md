## Summary

Keep this PR open. Current main has partially addressed inbound conversation normalization through channel-owned resolvers, but it still does not implement the PR's central public contract change: agent hook contexts still lack `conversationId` and `accountId`, and the embedded/native hook call sites still omit those values. The PR is also not safe to merge as-is because its proposed `after_agent_complete` design conflicts with current hook surfaces and has unresolved compile/runtime and streaming-behavior review concerns.

## Best Possible Solution

Keep this PR open and ask for a refresh against current main. Preserve the channel-owned inbound resolver approach already present, add `conversationId` and `accountId` to `src/plugins/hook-types.ts` and `src/agents/harness/hook-context.ts`, wire `params.messageTo` and `params.agentAccountId` through embedded and native agent hook contexts, and add focused regression tests. The `after_agent_complete` portion should either be dropped in favor of current `before_agent_finalize`/`reply_dispatch` surfaces or redesigned explicitly with maintainer approval and streaming-safe semantics.
