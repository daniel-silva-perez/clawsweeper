## Summary

Keep open. Current main has the channel-owned inbound conversation resolver direction, but it still does not implement this PR's core agent-hook contract: `PluginHookAgentContext` has no `conversationId` or `accountId`, and the embedded agent hook call sites still omit those fields. The PR also has unresolved functional review blockers around its added `after_agent_complete` design, so it needs a refreshed patch rather than cleanup closure.

## Best Possible Solution

Keep this PR open and request a refresh on current main: preserve the channel-owned inbound resolver approach already present, add `conversationId` and `accountId` to the current `src/plugins/hook-types.ts` `PluginHookAgentContext` and agent harness context, wire those values through every relevant agent hook phase, and replace or drop the PR's `after_agent_complete` addition in favor of the current `before_agent_finalize`/`reply_dispatch` surfaces with focused regression tests.
