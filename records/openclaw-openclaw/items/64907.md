## Summary

Keep this PR open. Current main does not implement the proposed public `tool_result_before_model` SDK hook or an equivalent external-plugin seam: the typed hook registry, prompt-injection hook policy, hook runner, embedded Pi extension bridge, persistence wrapper, and public docs still expose only `before_tool_call`, `after_tool_call`, and `tool_result_persist` for normal plugins. Main does have bundled-only tool-result middleware, but docs explicitly say external plugins cannot register that middleware, so it does not solve this PR's central user problem.

## Best Possible Solution

Keep this PR open for maintainer review as the focused public SDK seam. If accepted, land a narrow text-only `tool_result_before_model` implementation with hook types, runner support, embedded-runner bridge, prompt-injection policy wiring, docs/API baseline updates, and targeted tests. If maintainers decide pre-model tool-result rewriting should remain bundled-only, close it with that explicit product decision and leave broader async security scanning to #62123.
