## Summary

Keep open. Current main still routes Telegram regular messages through ack reaction and typing context, while Telegram native slash-command handlers for built-in and plugin commands bypass those feedback hooks. The provided context also shows an open, unmerged fork PR that explicitly attempts to close this issue, so the right maintainer action is to review or upstream that implementation candidate rather than close the report now.

## Best Possible Solution

Implement this inside the bundled Telegram plugin by threading ackReactionScope and the existing sendChatActionHandler/reaction capability into native command registration, adding a shared post-skip/post-auth helper for slash-command ack and typing, and covering built-in and plugin native commands with focused tests for direct DMs, group scopes, thread IDs, disabled scopes, and API failure handling. The open fork PR should be reviewed or upstreamed as an implementation candidate; close this issue only after an upstream merge or equivalent main-branch fix.

- Telegram setMessageReaction/sendChatAction failures should stay best-effort and logged so feedback failures do not break command execution.
