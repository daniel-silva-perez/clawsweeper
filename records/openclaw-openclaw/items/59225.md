## Summary

Keep open. Current main partially mitigates the reported harmful `sessions_spawn` case by hiding ACP-only fields when ACP is unavailable and ignoring `streamTo`/`resumeSessionId` for native subagent execution, but it does not implement the requested per-agent tool-schema parameter allowlist/denylist or general schema slimming surface.

## Best Possible Solution

Keep this issue open as the product/API tracker for per-agent tool-schema parameter filtering. A conservative implementation would add a validated config shape under agent tool policy, prune JSON Schema `properties` after whole-tool policy resolution but before provider serialization, preserve required fields, and add focused tests for `sessions_spawn` plus at least one non-session tool to prove the mechanism is generic.

## Risks / Open Questions

- Closing this now would conflate the shipped ACP-only field tolerance with the requested config/API surface for per-agent schema slimming.
- Any eventual implementation needs careful compatibility handling: hiding parameters from a model should not remove runtime acceptance of those parameters for callers that intentionally pass them, and ACP/native subagent behavior must remain distinct.
