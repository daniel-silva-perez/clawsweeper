## Summary

Keep open. Current main addresses the narrower ACP-only `streamTo`/`resumeSessionId` subagent failure, but it does not implement the requested per-agent tool schema parameter filtering; the config schema still models whole-tool policy, and the provider handoff still forwards each tool's full parameter schema.

## What This Changes

The issue requests a per-agent config surface that can hide or allowlist individual tool schema parameters, especially for `sessions_spawn` in dispatcher/subagent workflows.

## Best Possible Solution

Add a backward-compatible, documented per-agent tool schema visibility contract that prunes `parameters.properties` before provider handoff, preserves required fields safely, updates config schema/help/docs, and includes focused tests showing `sessions_spawn` can be exposed to a selected agent with only the intended parameters.

## Reproduction Assessment

Not applicable; this is a feature/config request rather than a runtime bug report. Source and docs inspection provide high-confidence proof that the requested config surface and schema-pruning behavior are absent on current main.

## Solution Assessment

No for current main; the existing runtime-side tolerance for ACP-only fields is useful but does not solve the requested model-visible schema slimming. The maintainable path is a deliberate config/schema contract at the tool-definition or policy boundary, with clear security wording.
