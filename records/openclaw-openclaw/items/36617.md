## Summary

Keep this issue open. Current main has Japanese localization for the browser Control UI and a docs publish-repo i18n pipeline, but the requested core UX scope still remains: CLI/help, onboarding/wizard prompts, workspace templates, and common non-browser labels are still English-first and there is no shared core i18n layer in src/.

## Best Possible Solution

Keep this issue open until maintainers define and implement a core i18n path outside the browser UI: locale resolution for CLI/wizard contexts, English fallback, ja-JP resource files for agreed onboarding/help/template/common-label surfaces, and focused tests for fallback behavior, setup wizard prompts, CLI help rendering, and workspace template generation.
