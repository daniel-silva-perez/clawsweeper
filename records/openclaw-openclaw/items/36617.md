## Summary

Keep this issue open. Current main has Japanese localization for the browser Control UI and docs-publish i18n plumbing, but the requested core UX scope is still not implemented: CLI/help, setup wizard prompts, workspace templates, and common non-browser labels remain English-first, and there is no shared core i18n layer under src/.

## Best Possible Solution

Keep this issue open until maintainers define and implement a core i18n contract outside the browser UI: locale resolution for CLI/wizard contexts, English fallback behavior, ja-JP resource files for agreed onboarding/help/template/common-label surfaces, and focused tests for fallback, setup wizard prompts, CLI help rendering, and workspace template generation.
