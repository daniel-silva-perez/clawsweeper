## Summary

Keep open. Current main has ja-JP localization for the browser Control UI and a separate docs translation pipeline, but the issue asks for core UX localization across onboarding, help, templates, and common labels. The current CLI/wizard/template surfaces still use English string literals, and `src/i18n` only contains a test that imports the UI registry rather than a shared core i18n contract. Related #11015 is closed around the Japanese domain/docs/community context and does not supersede this runtime/core UX request.

## Best Possible Solution

Keep this issue open until maintainers define and implement a core i18n contract outside the browser UI: locale resolution for CLI/wizard contexts, English fallback semantics, ja-JP resource files for agreed onboarding/help/template/common-label surfaces, and focused tests for wizard prompts, CLI help rendering, fallback, and workspace template generation.
