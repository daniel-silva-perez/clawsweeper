## Summary

Keep open. Current main has ja-JP localization for the browser Control UI and a separate ja-JP docs translation pipeline, but this issue asks for first-class core UX localization across onboarding, help, templates, and common labels. The CLI/wizard/template surfaces still use English literals, and there is no shared core i18n implementation under src/ beyond a test that imports the UI registry.

## Best Possible Solution

Keep this issue open until maintainers define the core i18n contract outside the browser UI: locale resolution for CLI/wizard contexts, English fallback semantics, ja-JP resource files for agreed onboarding/help/template/common-label surfaces, and focused tests for wizard prompts, CLI help rendering, fallback, and workspace template generation. The docs and Control UI ja-JP work can remain as partial shipped coverage, but it does not close the broader runtime/core UX request.
