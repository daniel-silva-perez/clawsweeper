## Summary

Keep open. Current main still routes `/model status` through `loadModelCatalog()` and renders provider groups directly from catalog entry providers, while the catalog loader still accepts `ModelRegistry` entries without filtering auto-inferred provider prefixes. There are also open PRs with closing syntax for this issue, so the issue should remain open until one canonical fix is reviewed and merged.

## Best Possible Solution

Keep this issue open while reviewing the linked implementation candidates. The best fix is a focused catalog-level filter, or an equivalent status-rendering guard, that suppresses only auto-inferred native-provider entries already represented by an aggregator provider while preserving explicitly configured native providers, with regression coverage for the OpenRouter `google/...` case. After a PR merges, GitHub can close this issue as implemented.
