## Summary

Keep open. Current main still has the provider-id guard this PR removes, so explicit `params.context1m: true` is still blocked for `github-copilot` and other proxied Claude Opus/Sonnet model refs before the resolver can return the 1M window. The PR remains a narrow implementation candidate for the closed linked bug, but it needs maintainer review around proxy header behavior and changelog attribution before merge.

## Best Possible Solution

Keep this PR open and review or revise it as the canonical implementation for #69353. The best path is a scoped resolver change that makes explicit `params.context1m: true` depend on the Claude Opus/Sonnet model-id capability prefix rather than the provider id, preserves the non-eligible model guard, adds the proxied-provider regression tests, fixes changelog attribution, and verifies whether Copilot/LiteLLM/Bedrock-style proxy transports need matching long-context beta/header handling before landing.
