## Summary

Keep this PR open. Current main still has the provider guard this PR removes, so `context1m: true` remains ineffective for `github-copilot` and other proxied Claude Opus/Sonnet models unless a separate configured/discovered context window happens to override it. The PR is still a focused implementation candidate for the linked #69353 behavior.

## Best Possible Solution

Keep this PR open and review or revise it as the canonical implementation surface for #69353. The best code path is a scoped resolver change that makes explicit `params.context1m: true` depend on the Claude Opus/Sonnet model-id capability prefix rather than the provider id, preserves the non-eligible model guard, adds the PR's proxied-provider regression tests, fixes changelog attribution, and verifies any separate proxy request-header behavior before landing.
