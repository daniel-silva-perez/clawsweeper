## Summary

Keep open. The Chat i18n request is still valid, but this PR is stale and incomplete against current main: Chat copy now spans split helper modules, the supported locale set is larger than the six bundles changed here, generated i18n metadata is part of the locale contract, and the proposed fallback parameters are not supported by the translation helper.

## What This Changes

The PR imports `t()` into `ui/src/ui/views/chat.ts`, replaces selected hard-coded Chat copy with `chat.*` lookups, and adds matching keys to six locale bundles.

## Best Possible Solution

Repair or replace the branch so all current Chat view and helper copy goes through the existing i18n path, English source keys preserve dynamic values such as assistant name and counts, all generated locale bundles plus `.i18n` metadata are refreshed through the Control UI pipeline, and focused UI/i18n tests cover the localized behavior.

  - confidence: 0.92
