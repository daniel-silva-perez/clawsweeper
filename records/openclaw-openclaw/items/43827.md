## Summary

Keep open: the branch is aimed at a valid Control UI i18n gap, but it is not mergeable against current main. Chat rendering has since been split into helper modules the patch does not cover, the locale set expanded beyond the six files edited here, and the `{ fallback: ... }` params called out in prior review still do not match the current `t()` contract.

## What This Changes

The PR replaces selected hard-coded Chat view strings in `ui/src/ui/views/chat.ts` with `t("chat.*")` calls and adds matching `chat.*` keys to six locale files.

## Best Possible Solution

Rework the branch against current main so every visible Chat copy path in the split Chat modules uses the existing i18n helper, English source keys are added once, all supported generated locale bundles and metadata are refreshed through the Control UI i18n pipeline, unsupported fallback params are removed unless a real typed fallback API is added with tests, and focused UI/i18n tests cover the localized behavior.

## Review Findings

Overall correctness: patch is incorrect

  - body: Current `renderChat` delegates controls, queue UI, and status indicators into `ui/src/ui/chat/*`.   - confidence: 0.9

  - confidence: 0.88

  - body: `t()` treats its second argument as interpolation params only, so `{ fallback: "..." }` is ignored unless a translation string contains a `{fallback}` placeholder. Remove these params or add a real typed fallback API with tests.
  - confidence: 0.92
