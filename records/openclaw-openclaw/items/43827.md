## Summary

Keep this PR open, but it is not mergeable as-is. The underlying Chat localization gap is real, and the existing Greptile and ClawSweeper comments correctly identify useful source work, but current main has since split visible Chat controls/status/queue copy into helper modules and expanded the Control UI locale pipeline, so this branch would leave supported UI text unlocalized and locale outputs incomplete.

## What This Changes

The PR replaces selected hard-coded Chat view strings in `ui/src/ui/views/chat.ts` with `t("chat.*")` calls and adds matching `chat.*` keys to six locale files.

## Best Possible Solution

Repair or replace the branch against current main so all visible Chat page copy routes through the existing i18n source of truth, dynamic values such as assistant name and counts remain interpolated, locale outputs and `.i18n` metadata are regenerated by the Control UI pipeline, and focused tests cover the localized Chat surfaces.
