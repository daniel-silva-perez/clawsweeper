## Summary

Keep open: the Chat localization gap is real, but this PR is not merge-ready against current main. It localizes only the older view slice, hand-edits a subset of locales, misses current split Chat helper modules, and would regress the current assistant-specific composer placeholder after rebase.

## What This Changes

This PR replaces selected hard-coded Chat view strings in `ui/src/ui/views/chat.ts` with `t("chat.*")` calls and adds matching `chat.*` keys to six locale files.

## Best Possible Solution

Repair or replace the branch against current main so all current Chat UI copy routes through the existing i18n source of truth, dynamic values such as assistant names and counts use interpolation, every supported locale and generated metadata is regenerated, and focused tests cover the localized Chat surfaces.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main still contains literal Chat copy in `ui/src/ui/views/chat.ts`, `ui/src/ui/chat/run-controls.ts`, `ui/src/ui/chat/status-indicators.ts`, and `ui/src/ui/chat/chat-queue.ts`, so switching locale cannot affect those labels today.

## Solution Assessment

No. The localization direction is right, but the proposed patch is not the best mergeable fix because it targets an older shape of the Chat page, updates only six locale bundles, uses unsupported fallback params, and would lose current assistant-name placeholder context.
