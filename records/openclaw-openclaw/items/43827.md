## Summary

Keep open: the requested Chat localization is still relevant on current main, but this PR is stale and incomplete. ## What This Changes

The PR updates selected Chat view strings to call `t("chat.*")` and adds matching `chat.*` keys to six locale files.

## Best Possible Solution

Rework the branch against current main so every visible Chat copy path in the split modules uses the existing i18n helper, English source keys are added once, all generated locale bundles and `.i18n` metadata are refreshed through the Control UI i18n pipeline, unsupported fallback params are removed unless a real typed fallback API is added, and focused UI/i18n tests cover the localized behavior.
