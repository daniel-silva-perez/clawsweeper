## Summary

Keep open: this PR is useful but incomplete against current main. It localizes selected older Chat view strings, but current Chat copy is split across helper modules that still render English literals, and the Control UI i18n pipeline now supports more generated locale bundles than this branch updates.

## What This Changes

The branch replaces selected hard-coded Chat view strings with `t("chat.*")` calls and adds matching `chat.*` keys to six locale files.

## Best Possible Solution

Rework the branch against current main so every visible Chat copy path in the split Chat modules uses the existing i18n helper, English source keys are added once, generated locale bundles and metadata are refreshed through the Control UI i18n pipeline, unsupported fallback params are removed unless a real typed fallback API is added with tests, and focused UI/i18n tests cover the localized behavior.
