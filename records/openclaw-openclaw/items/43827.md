## Summary

Current main still has hard-coded Chat UI strings, so this PR's central i18n change is not implemented. The PR remains directionally useful, but it needs revision for the current split Chat modules, generated locale pipeline, and existing `t()` fallback/interpolation contract. Security review found no concrete supply-chain, workflow, dependency, publishing, or secrets-handling concern in the provided PR file list.

## Best Possible Solution

Keep this PR open for revision. The best path is to rebase onto current main, localize the current Chat surfaces in `ui/src/ui/views/chat.ts` and `ui/src/ui/chat/*`, add English source keys, regenerate all supported locale bundles through `pnpm ui:i18n:sync`, remove the no-op fallback params or implement a real typed fallback API with tests, and update Chat tests to assert the localized behavior intentionally.
