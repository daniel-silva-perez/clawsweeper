## Summary

Keep this issue open. Current main still uses the numbered `.bak` / `.bak.1` through `.bak.4` backup ring, cleanup still treats non-numbered `.bak.*` files as orphans, and open PRs #39982 and #40392 explicitly target this issue with closing syntax. PR #40392 is also by the same author as this issue, so the issue should remain open until that paired work is resolved or maintainers decide to keep the numbered scheme.

## Best Possible Solution

Keep this issue open until one datetime-suffix backup implementation is reviewed and landed, or maintainers explicitly decide to retain numeric rotation. The best implementation should update `src/config/backup-rotation.ts` and `src/config/config.backup-rotation.test.ts` together, covering bounded timestamp pruning, collision handling, permission discovery, and legacy `.bak.N` migration. The broader separate-directory request can remain tracked by #20460.
