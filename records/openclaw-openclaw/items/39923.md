## Summary

Keep this issue open. Current main still implements config backups as a numbered `.bak` ring, the tests assert that numbered behavior, and open PRs #39982 and #40392 explicitly target this issue with closing syntax. The same-author PR #40392 is still open, so this should remain open until maintainers review and land one implementation or decide to retain numeric rotation.

## Best Possible Solution

Keep this issue open until one datetime-suffix backup implementation is reviewed and landed, or maintainers explicitly decide to keep numeric rotation. The implementation should update `src/config/backup-rotation.ts` and `src/config/config.backup-rotation.test.ts` together, with coverage for bounded timestamp pruning, collision handling, permission discovery, and legacy `.bak.N` migration. The separate directory-placement request can remain tracked by #20460.
