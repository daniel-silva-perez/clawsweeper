## Summary

Keep this open. Current main still uses numbered config backup rotation, the tests still assert that behavior, and two open PRs explicitly target this issue with closing syntax, so the issue should stay open until maintainers land one implementation or decide to keep numeric rotation.

## What This Changes

The issue requests replacing config backup numeric rotation files with timestamp-suffixed backup files while preserving bounded cleanup, permission hardening, collision handling, and migration safety.

## Best Possible Solution

Keep this issue open until one timestamp-suffix backup implementation is reviewed and landed, or maintainers explicitly decide to retain numeric rotation. The landed change should update `src/config/backup-rotation.ts` and `src/config/config.backup-rotation.test.ts` together with coverage for bounded timestamp pruning, collision handling, permission discovery, and legacy numeric backup migration.

## Reproduction Assessment

Yes for the current behavior: reading current main shows the active config write paths call `maintainConfigBackups`, and that helper still creates `.bak` plus numbered `.bak.N` files. This is a feature request rather than a failing runtime reproduction, so static source and test verification is sufficient for triage.

## Solution Assessment

Yes, a bounded timestamp suffix scheme in the config backup module is the narrow maintainable direction if maintainers accept the product change. The best next step is to review and land one of the open paired PRs rather than start a separate ClawSweeper repair branch.
