## Summary

Keep this PR open. Current main still does not enable the assistant-first ordering bootstrap for non-Claude Amazon Bedrock models: the Bedrock plugin delegates replay policy to the shared `anthropic-by-model` hook, and that hook returns a strict Anthropic policy without `applyAssistantFirstOrderingFix`. The proposed provider-local override and regression tests still address a real gap in the current code.

## Best Possible Solution

Review and land this provider-local fix, or an equivalent Bedrock-owned `buildReplayPolicy` override, with the proposed Nova/Claude regression coverage. The behavior belongs in the Amazon Bedrock plugin because the core fallback is intentionally bypassed once a provider owns replay policy.
