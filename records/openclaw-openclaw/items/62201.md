## Summary

Keep open. Current main does not implement this PR's intended replay-time stale diagnostic toolResult omission: there is no replay metadata module, no persistence-time diagnostic tagging, and sanitizeSessionHistory still runs the existing replay cleanup pipeline without age/supersession filtering. The PR remains plausible core agent work, but the provided review context also shows a latest unresolved correctness concern around delimiter collisions in replay metadata drain keys that should be fixed before merge.

## Best Possible Solution

Keep this PR open. Revise the replay metadata pending-state storage so session drains do not depend on colon-delimited compound-key parsing, preserve the replay-time omission behavior for stale/superseded transient diagnostics, and rerun the focused agent replay/persistence tests plus the appropriate changed gate before maintainer review. Do not close until equivalent metadata tagging and replay omission land on main.
