## Summary

Keep this issue open. Current main still appears to allow the reported gateway service markers through the host exec environment, and an open, unmerged PR (#62204) is already linked as an implementation candidate with GitHub closing syntax for this issue. Per policy, the issue should remain open until that PR or a successor lands.

## Best Possible Solution

Keep the issue open while maintainers review and either land #62204 or a successor that strips internal gateway service markers from exec-spawned host environments, preserves OPENCLAW_SHELL=exec, and adds regression coverage for both the auth-precedence failure and clean process exit.
