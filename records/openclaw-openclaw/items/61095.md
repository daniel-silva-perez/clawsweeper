## Summary

Keep open. Current main still matches the reported failure path, and #62204 is an open, unmerged implementation candidate that uses closing syntax for this issue, so the issue should stay open until that PR or a successor lands.

## What This Changes

The issue reports that agent exec-spawned OpenClaw CLI commands inherit gateway service environment markers, causing gateway WebSocket/client authentication failures and sometimes non-exiting child processes.

## Best Possible Solution

Keep the issue open while maintainers review and land #62204 or a narrow successor that strips gateway service runtime markers at the host exec environment boundary, preserves OPENCLAW_SHELL=exec, and adds regression proof for both auth precedence and clean child-process exit.

## Reproduction Assessment

Yes. The source-level reproduction path is high confidence: gateway service startup injects the reported markers, exec inherits process.env, the current sanitizer does not remove those markers, and gateway credentials branch on OPENCLAW_SERVICE_KIND=gateway.

## Solution Assessment

Yes. The narrowest maintainable direction is to sanitize internal gateway service markers before exec-spawned host commands run; changing all CLI credential precedence would be broader and could affect legitimate gateway-service behavior.
