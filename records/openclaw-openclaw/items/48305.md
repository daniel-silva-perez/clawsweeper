## Summary

Keep this PR open. Current main has generic restart-reason plumbing, but the PR’s central config-reload propagation is still missing: the reload handler computes restart reasons and then calls the restart deferrer/emitter without passing them through. The PR remains useful, but should be refreshed with a regression test and the log-stack diagnostic concern resolved before merge.

## Best Possible Solution

Refresh and keep the PR focused: pass `config-reload: ${reasons}` through both `deferGatewayRestartUntilIdle({ reason })` and `emitGatewayRestart(reason)`, add a regression test that observes the propagated config-reload reason, and remove or constrain the missing-reason stack trace warning before maintainer review.
