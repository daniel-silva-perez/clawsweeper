## Summary

Keep this PR open. Current main has generic restart-reason plumbing, but the config reload path still computes meaningful restart reasons and drops them before emitting or deferring the restart. The PR still targets a real missing behavior, though it needs a current-main refresh, a regression test, and review of the proposed missing-reason stack trace diagnostic.

## Best Possible Solution

Refresh this PR onto current main, keep the behavioral change focused on propagating `config-reload: ${reasons}` through both `deferGatewayRestartUntilIdle({ reason })` and `emitGatewayRestart(reason)`, add a regression test that observes the propagated reason, and remove or tightly constrain the missing-reason stack trace diagnostic.
