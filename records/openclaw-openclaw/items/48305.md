## Summary

Current main still has the central gap this PR targets. Restart-reason plumbing exists, but the config reload path computes meaningful reasons and then drops them before deferring or emitting the gateway restart. The PR is still useful, though it needs a current-main refresh, a focused regression test, and cleanup around the proposed stack-trace diagnostic.

## Best Possible Solution

Keep this PR open and refresh it onto current main. The best path is a focused change in `src/gateway/server-reload-handlers.ts` that passes `config-reload: ${reasons}` into both `deferGatewayRestartUntilIdle({ reason })` and `emitGatewayRestart(reason)`, plus a regression test proving the propagated reason. Avoid or tightly constrain the broader missing-reason stack trace warning before merge.
