## Summary

Keep open. Current main still rejects Gateway `agent` requests under `session.sendPolicy=deny` before it computes whether the request wants external delivery, while the embedded agent path only enforces that policy for delivery runs. The paired #73500 PR directly targets this issue but is still open and unmerged, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests allowing non-delivery Gateway agent invocations when `session.sendPolicy=deny`, while still blocking explicit external delivery.

## Best Possible Solution

Land #73500 after review, or apply an equivalent focused change in `src/gateway/server-methods/agent.ts` that only enforces `sendPolicy=deny` for explicit delivery requests, with regression coverage for both non-delivery allowed and delivery blocked cases.
