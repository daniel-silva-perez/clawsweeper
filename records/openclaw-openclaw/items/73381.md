## Summary

Keep open. Current main still matches the reported bug: the Gateway `agent` RPC can reject on `sendPolicy=deny` before delivery intent is computed, while the embedded agent path only enforces that policy for explicit delivery. The related #73500 PR specifically closes this issue but is still open, so the issue should remain open until that fix or a replacement merges.

## What This Changes

The issue asks for non-delivery Gateway `agent` invocations to bypass outbound `session.sendPolicy=deny` while preserving denial for explicit external delivery.

## Best Possible Solution

Land or replace #73500 with a focused change in `src/gateway/server-methods/agent.ts` that enforces `sendPolicy=deny` only when the Gateway agent request will actually deliver externally, with regression tests for `deliver:false` allowed and `deliver:true` blocked.
