## Summary

Keep open. Current main still matches the reported bug: the Gateway `agent` RPC resolves and enforces `sendPolicy=deny` before it computes whether the request wants external delivery, while the embedded agent path only applies the same policy when `deliver` is true. The linked #73500 PR remains open and directly targets this issue, so this should wait for that PR or an equivalent fix to merge.

## What This Changes

The issue asks for non-delivery Gateway `agent` invocations to run even when `session.sendPolicy=deny`, while keeping explicit external delivery blocked.

## Best Possible Solution

Land #73500 or an equivalent focused change in `src/gateway/server-methods/agent.ts` that enforces `sendPolicy=deny` only when `request.deliver === true`, with regression coverage for non-delivery deny-policy allowed and delivery deny-policy blocked.

## Reproduction Assessment

Yes. A source-level reproduction is clear: with a session/config resolving to `sendPolicy=deny`, a Gateway `agent` request with `deliver` omitted or false reaches the rejection at `src/gateway/server-methods/agent.ts:1049` before `wantsDelivery` is computed.

## Solution Assessment

Yes, the linked PR’s direction appears to be the narrowest maintainable fix because it mirrors the embedded `agent-command.ts` policy gate and preserves the block for explicit delivery requests.
