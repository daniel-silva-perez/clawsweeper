## Summary

Keep open. Current main still matches the reported bug: the Gateway `agent` handler enforces `sendPolicy=deny` before it determines whether the request actually wants external delivery, while the embedded agent command only applies that policy when delivery is explicitly requested. The paired open PR #73500 directly targets this issue and should be reviewed, landed, replaced, or closed before this issue is resolved.

## What This Changes

The issue requests that Gateway `agent` RPC calls without external delivery be allowed even when the session send policy denies outbound sends, while preserving the deny behavior for explicit delivery.

## Best Possible Solution

Land #73500 or an equivalent focused change in `src/gateway/server-methods/agent.ts` so `sendPolicy=deny` is enforced only when `request.deliver === true`, with regression coverage for both non-delivery allowed and explicit delivery blocked.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: with a session resolving to `sendPolicy=deny`, a Gateway `agent` request that omits `deliver` reaches the rejection in `src/gateway/server-methods/agent.ts` before `wantsDelivery` is computed.

## Solution Assessment

Yes. The linked PR direction is the narrowest maintainable fix because it mirrors the embedded `agent-command.ts` gate and preserves the deny check for explicit outbound delivery.
