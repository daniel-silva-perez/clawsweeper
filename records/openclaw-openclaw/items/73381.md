## Summary

Keep open. Current main still matches the reported bug: the Gateway `agent` handler enforces `sendPolicy=deny` before it computes whether the request is actually delivering externally, while the embedded agent command only enforces that policy for `deliver === true`. The paired #73500 PR remains the right active follow-up, so this issue should wait for that PR or an equivalent fix to merge.

## What This Changes

A Gateway `agent` request should allow non-delivery agent invocations when session send policy denies external delivery, while still blocking explicit delivery requests.

## Best Possible Solution

Land #73500 or an equivalent narrow change in `src/gateway/server-methods/agent.ts` that checks `sendPolicy=deny` only when `request.deliver === true`, with regression coverage proving non-delivery deny-policy calls run and explicit delivery calls remain blocked.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: configure or load a session entry resolving to `sendPolicy=deny`, then send a Gateway `agent` request with `deliver` omitted or false; the handler rejects at `src/gateway/server-methods/agent.ts:1049` before delivery intent is evaluated.

## Solution Assessment

Yes. The #73500 direction is the narrowest maintainable fix because it mirrors the embedded `agent-command.ts` gate and preserves the deny behavior for explicit outbound delivery.
