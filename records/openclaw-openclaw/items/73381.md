## Summary

Keep open. The checked-out main branch still has the reported Gateway behavior: `handleAgent` rejects on `sendPolicy=deny` before computing delivery intent, while the embedded agent path only enforces the same policy for explicit delivery. The issue is also paired with open, unmerged PR #73500, so it should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that Gateway `agent` RPC calls without external delivery are rejected by `session.sendPolicy=deny`.

## Best Possible Solution

Land PR #73500 or an equivalent focused change in `src/gateway/server-methods/agent.ts` that enforces `sendPolicy=deny` only when `request.deliver === true`, with regression tests for both non-delivery allow and explicit-delivery denial.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a Gateway `agent` request for a session resolving to `sendPolicy=deny` with `deliver` omitted or false reaches the rejection before `wantsDelivery` is computed.

## Solution Assessment

Yes. Gating the Gateway policy check on `request.deliver === true` mirrors the embedded agent path and preserves the outbound-delivery safety check.
