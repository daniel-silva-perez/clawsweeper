## Summary

Keep open. Current main still rejects the gateway `agent` RPC on `sendPolicy=deny` before it determines whether the request asked for delivery or will actually deliver externally. The CLI/docs make `--deliver` optional, and the embedded agent path already applies send policy only when `opts.deliver === true`, so the reported gateway behavior is a real inconsistency rather than implemented-on-main cleanup.

## Best Possible Solution

Fix `src/gateway/server-methods/agent.ts` so session send policy is enforced only for actual external delivery, ideally after delivery planning when the final `deliver` decision and resolved channel are known. Add a regression test for a gateway `agent` request with `deliver:false` and `sendPolicy:deny` that still reaches `agentCommandFromIngress`, plus a companion test proving `deliver:true` remains blocked by the policy.
