## Summary

Keep this issue open. Current main has broad local embedding support, but the narrower request is still missing for the bundled local GGUF provider: `outputDimensionality` is resolved and passed through the generic memory provider request, but `createLocalEmbeddingProvider` returns the full normalized `node-llama-cpp` vector without truncation, and the local provider cache key does not include dimensions for reindex identity. The bonus Qwen3 query-instruction idea is partly present only for Ollama as a hard-coded template, not as configurable `memorySearch.queryInstruction` or local GGUF behavior.

## Best Possible Solution

Keep this issue open and implement it in the bundled memory/local embedding path. The best fix is to add a tested truncation helper for local GGUF embeddings, apply it consistently to query and batch vectors before final normalization, include the requested dimension in local provider runtime/cache key data so dimension changes force a full reindex, and update `docs/reference/memory-config.md` plus config help/tests. Treat configurable `queryInstruction` as either a second field in the same memory-search surface or a follow-up issue, making sure local GGUF and Ollama behavior stay consistent.
