## Summary

Keep this open. Current main still lacks opt-in, domain-scoped OpenClaw agent attestation for arbitrary external HTTP requests, and the adjacent OpenAI/OpenRouter attribution work is provider-transport attribution rather than a general agent-origin attestation surface. The existing maintainer comment also keeps this as a product/security design tracker.

## What This Changes

Request opt-in static or signed agent attestation metadata on OpenClaw-originated external HTTP requests, with domain scoping and optional skill metadata.

## Best Possible Solution

Define the trust model and exact outbound paths first, then add an opt-in, domain-scoped config/API surface with privacy-preserving defaults, clear static-versus-cryptographic semantics, docs, and tests proving allowlisted injection plus non-leakage.
