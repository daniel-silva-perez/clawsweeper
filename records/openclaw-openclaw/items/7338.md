## Summary

Current main still lacks opt-in, domain-scoped agent attestation for arbitrary outbound HTTP requests. Existing OpenAI/OpenRouter attribution is provider-transport attribution only, and the issue already has maintainer review context keeping it open for product/security design rather than cleanup closure.

## What This Changes

The issue requests optional OpenClaw agent attestation headers or signed attestations on external HTTP requests, controlled by user/skill configuration and domain allowlisting.

## Best Possible Solution

Define the trust model and exact outbound-request scope first, then add an opt-in, domain-scoped config/API surface with privacy-preserving defaults, clear static-versus-cryptographic semantics, docs, and tests proving allowlisted injection plus non-leakage.
