## Summary

Keep this open. Current main still lacks opt-in, domain-scoped OpenClaw agent attestation for arbitrary external HTTP requests, and the recent OpenAI/OpenRouter attribution work is provider-transport scoped rather than a general agent-origin attestation surface. The existing maintainer review comment also explicitly kept this as a product/security design tracker.

## What This Changes

Request opt-in OpenClaw agent attestation headers or signed tokens for outgoing external HTTP requests, optionally controlled by config and skill metadata.

## Best Possible Solution

Define the trust model and outbound paths first, then add an opt-in, domain-scoped config/API surface with privacy-preserving defaults, clear static versus cryptographic semantics, docs, and tests proving allowlisted injection plus non-leakage.

## Risks / Open Questions

- Agent attestation is security and privacy sensitive: the trust model, signing authority, key distribution, replay window, and stable identifier leakage need explicit design before implementation.
