## Summary

Keep this PR open. Current main still lacks the PR’s central backend behavior: `sessions.list` has no `lastHash` request field, no cached result/hash response shape, and no `{ unchanged: true }` short-circuit. The PR is also active and review discussion shows branch-side follow-up on the config.patch concern, so it remains a viable implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to review the backend cache contract, invalidation inputs, config.patch/SecretRef write behavior, and targeted regression tests, then either land this split backend PR or replace it with a narrower canonical OpenClaw PR that implements the same `sessions.list` hash/lastHash contract.
