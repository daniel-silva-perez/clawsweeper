## Summary

Keep this PR open. Current main still forces an approval prompt for any heredoc in allowlist mode, while this PR is a plausible but not merge-ready fix because its regex misses unquoted strip-tab heredocs and its added tests appear malformed.

## Best Possible Solution

Keep the PR open, but require a narrow rework before merge: derive heredoc approval from the existing parser semantics or a robust helper that treats quoted and backslash-quoted delimiters as literal while still detecting unquoted `<<EOF` and `<<-EOF`; fix warning capture in tests; add direct `processGatewayAllowlist` coverage for quoted, backslash-quoted, unquoted, and strip-tab heredocs; then run the targeted agents/exec approval tests.
