## Summary

Keep open. Current main has some Feishu duplicate-final protections, but they do not fully cover the reported failover coordination bug. The linked candidate fix #59771 is still open, and main still closes the active Feishu streaming card on reply delivery error while only deduping exact duplicate final text; distinct errored/recovered final payloads can still be delivered as separate visible replies.

## Best Possible Solution

Keep this issue open until #59771 or an equivalent fix lands. The fix should prevent a failed primary attempt from surfacing a user-visible final when a fallback attempt later succeeds, and it should include regression coverage for Feishu streaming recovery plus the reported non-streaming final-reply coordination case if that remains reproducible.
