## Summary

Keep open. Current main has Feishu duplicate-final mitigations, but it still does not fully cover the reported failover coordination bug: the Feishu dispatcher still closes the active streaming card on reply delivery error, the generic final-delivery loop still sends every reply payload, and the tests still preserve distinct non-streaming final payloads. The linked implementation candidate #59771 remains open and unmerged.

## Best Possible Solution

Keep this issue open until #59771 or an equivalent fix lands. The fix should prevent a failed primary attempt from surfacing a user-visible final when a fallback attempt later succeeds, and it should add or require regression coverage for the reported non-streaming Feishu failover path or explicitly split that remaining case into a narrower follow-up.
