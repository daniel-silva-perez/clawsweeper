## Summary

Keep this issue open. Current main has implemented several mitigations and related UX pieces, but the central high-priority requests are still not fully solved: workspace bootstrap context is still a fixed file set with coarse caps/filtering rather than declarative per-file loading conditions, and compaction summaries still become model-visible summary context rather than lazy/collapsible titles. The recent production comment reinforces the conditional workspace-loading need with a concrete workaround, so this is not stale.

## Best Possible Solution

Keep this issue open for maintainer triage, then split it into focused follow-ups: declarative per-bootstrap-file loading/filtering for default context assembly, lazy or bounded compaction-summary reintroduction, and any remaining provider-usage or marketplace-discovery gaps. Mark the already-solved sub-items with links to current docs/code so implementation work can focus on the remaining token-efficiency core.
