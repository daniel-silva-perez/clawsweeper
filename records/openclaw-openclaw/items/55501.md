## Summary

Current main has addressed several sub-items from this umbrella request, including bootstrap caps, heartbeat light/isolated context, per-task HEARTBEAT.md intervals, subagent completion/tool-policy documentation, cron failure notification/retry, config schema output, and ClawHub/plugin discovery surfaces. The central high-priority token-efficiency requests are still not fully implemented: normal workspace bootstrap remains a fixed recognized file set with coarse caps/filtering rather than declarative per-file conditions, and compaction summaries are still persisted and reintroduced as summary context rather than lazy/collapsible titles. The latest issue comment reinforces conditional workspace loading with a production workaround, so this should stay open for maintainer triage or focused follow-up issues.

## Best Possible Solution

Keep this issue open for maintainer triage, then split it into focused follow-ups: declarative per-bootstrap-file loading/filtering for default context assembly, lazy or bounded compaction-summary injection, and any remaining provider-usage or marketplace-discovery items. Mark the solved sub-items with links to current docs/code so the remaining work is clear.
