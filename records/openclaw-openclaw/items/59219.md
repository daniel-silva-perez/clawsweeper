## Summary

Keep open. Current main still applies the root realpath containment check to every skill source, including `openclaw-managed`, so the PR’s central managed-symlink fix is not implemented on main. The PR also appears to remain a plausible implementation candidate for the WSL/ClawHub warning path, though it needs normal maintainer review and rebase/security scrutiny because it intentionally changes skill containment behavior.

## Best Possible Solution

Keep this PR open as the current implementation candidate. The best path is to review or update the PR so current main exempts managed/personal shared roots from root-level realpath containment, keeps per-skill `SKILL.md` containment for all sources, preserves bundled/workspace/project/extra fail-closed behavior, and lands focused regression tests for managed symlink loading and sync.
