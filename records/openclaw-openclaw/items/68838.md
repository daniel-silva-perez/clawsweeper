## Summary

Current main still has the reported key-only follow-up queue cleanup, and the provided GitHub context shows an open same-author PR with closing syntax as the implementation candidate. This issue should stay open until that PR or a successor lands.

## Best Possible Solution

Review PR #68839 as the canonical implementation candidate. If it is correct, land the identity guard around late drain cleanup plus the regression test, then let the merged closing PR resolve this issue. If it is not correct, keep this issue open and request a revised fix that prevents a stale drain from deleting or clearing state for a replacement follow-up queue.
