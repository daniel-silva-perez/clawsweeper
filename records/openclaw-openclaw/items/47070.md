## Summary

Current main still does not attach `inputProvenance` when `spawnSubagentDirect()` starts the child `agent` call. The requested behavior is narrow, valid, and already paired with open draft PR #47087, so this issue should stay open until that PR or an equivalent patch merges.

## Best Possible Solution

Keep this issue open and review/land #47087 or an equivalent narrow patch. The fix should add `inputProvenance` to the child `agent` params in `spawnSubagentDirect()`, preserve requester session/channel lineage, set `sourceTool: "sessions_spawn"`, and add focused regression coverage in the subagent spawn tests.
