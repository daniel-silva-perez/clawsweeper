## Summary

Keep open. Current main still starts native `sessions_spawn` child runs without passing `inputProvenance`, while the existing provenance contract and `sessions_send` path already support the requested metadata. The issue is also paired with open draft PR #47087 using closing linkage, so the issue should remain open until that PR or an equivalent patch lands.

## What This Changes

Attach explicit inter-session `inputProvenance` to child tasks created through `sessions_spawn`, preserving requester lineage and setting `sourceTool: "sessions_spawn"`.

## Best Possible Solution

Land #47087 or an equivalent narrow patch that constructs inter-session provenance at the native spawn boundary, passes it to the child `agent` call, preserves requester session/channel lineage when present, and adds focused regression coverage in the existing subagent spawn test seam.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to call `spawnSubagentDirect()` with the existing mocked gateway seam and inspect the child `agent` params; current main's params do not contain `inputProvenance`.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing `InputProvenance` contract in `spawnSubagentDirect()` rather than redesign provenance or broader multi-agent policy.
