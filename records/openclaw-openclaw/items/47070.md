## Summary

Keep open. Current main still does not attach `inputProvenance` to the native `sessions_spawn` child `agent` call, while an open draft implementation PR by the same author explicitly closes this issue and adds the requested `sourceTool: "sessions_spawn"` payload and test. Per cleanup policy, this should stay open until that PR is reviewed/landed or closed.

## Best Possible Solution

Keep this issue open and review PR #47087 as the canonical implementation candidate. If accepted, rebase/update it against current main so `spawnSubagentDirect()` passes `inputProvenance` on the child `agent` call with `kind: "inter_session"`, requester session/channel lineage, and `sourceTool: "sessions_spawn"`, plus regression coverage in the subagent spawn tests; then close after merge.
