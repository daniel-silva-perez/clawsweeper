## Summary

Keep open. Current `main` still starts native `sessions_spawn` child runs without `inputProvenance`, while existing inter-session provenance plumbing and the `sessions_send` path already support the requested metadata. The issue is also paired with open draft PR #47087 using closing linkage, so it should remain open until that PR or an equivalent patch lands.

## What This Changes

Attach explicit inter-session input provenance to child tasks created through `sessions_spawn`, including `sourceTool: "sessions_spawn"` and requester lineage when available.

## Best Possible Solution

The desired end state is a narrow native-spawn change that constructs an `inter_session` `InputProvenance` payload in `spawnSubagentDirect()`, preserves requester session/channel where present, sets `sourceTool: "sessions_spawn"`, and adds focused coverage through the existing mocked gateway spawn test seam.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to call `spawnSubagentDirect()` through the existing mocked gateway seam and inspect the child `agent` params; current `main` does not include `inputProvenance`.

## Solution Assessment

Yes. Reusing the existing `InputProvenance` contract at the native spawn boundary is the narrowest maintainable fix; a broader provenance schema or trust-label redesign is unnecessary for this report.
