## Summary

Keep this PR open. Current main still does not pass `memorySearch.outputDimensionality` through the OpenAI memory embedding request, while the PR is a plausible implementation candidate for #55126. The code has moved since the PR was opened, so the best path is to rebase/adapt the fix to the current OpenAI plugin and memory-host SDK layout rather than close it as implemented.

## Best Possible Solution

Keep this PR open and ask for a rebase/update against current main. The implementation should add `outputDimensionality` support in the current OpenAI embedding adapter, send OpenAI `dimensions` only when configured for supported models, preserve omission when unset, and add focused tests for request shape and invalid dimensions. Once that lands, it can close #55126.
