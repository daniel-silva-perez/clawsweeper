## Summary

Keep open. Current main has adjacent usage surfaces, including `/status`, `/usage`, the Usage page, and collapsible assistant-message metadata, but it still lacks the requested persistent Control UI chat header widget and does not support the requested OpenRouter, LLMAPI, or Ollama quota display as provider usage windows.

## Best Possible Solution

Keep this open and either coordinate it with #13670 or split it into focused work: add a Control UI chat/header usage widget using fresh session usage and cost data, keep or refine the existing assistant-message metadata, and add provider usage hooks or explicit unlimited/local labels for the requested providers with docs and UI tests.
