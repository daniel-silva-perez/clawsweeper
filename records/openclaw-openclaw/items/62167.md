## Summary

Keep open. Current main has adjacent usage surfaces, including a high-threshold context notice and collapsible per-message assistant metadata, but it does not implement the requested persistent chat header widget or provider quota support for OpenRouter, LLMAPI, and Ollama. The related open PR #73744 explicitly treats this as broader follow-up scope rather than fully closing it.

## What This Changes

Request a persistent Control UI chat token/quota widget, per-assistant-message usage, provider quota/cost support for OpenRouter, LLMAPI, and Ollama, plus optional low-credit alerts.

## Best Possible Solution

Keep this open as the follow-up for chat-visible quota and cost awareness. The maintainable path is to coordinate with #73744, then add provider-owned usage hooks where needed and a Control UI widget that consumes the existing session/provider usage contracts without hard-coding provider policy into generic chat rendering.
