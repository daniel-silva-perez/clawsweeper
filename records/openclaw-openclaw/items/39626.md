## Summary

Keep open. Current main still does not implement sanitizer or repair coverage for the reported bare tool-name-plus-JSON visible text leak (`exec{"command": ...}`), and the relevant shared visible-text pipeline continues to cover XML-style tool blocks, bracketed plain-text tool blocks, provider special tokens, memory tags, reasoning tags, and downgraded `[Tool Call: ...]` scaffolding instead.

## Best Possible Solution

Keep this open as a behavior bug. Add conservative shared handling for bare tool-name-plus-JSON pseudo calls such as `exec{"command": ...}`, preferably in the assistant visible-text sanitizer or a provider repair immediately before visible delivery. Add focused regression coverage for streaming chunks, final message delivery, and Telegram rendering while preserving the existing XML/bracketed/downgraded-tool sanitizers.
