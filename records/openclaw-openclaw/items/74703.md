## Summary

Keep open. Current main still leaves the Windows docs-i18n Codex timeout path on `exec.CommandContext`'s process-only default cancellation while Unix kills the process group; this PR is a narrow, plausible fix with no blocking review finding found. It still needs normal maintainer review, CI, and preferably Windows runtime proof before merge.

## What This Changes

This PR adds a Windows-only custom `Cancel` hook for docs-i18n Codex commands that invokes bounded `taskkill.exe /T /F /PID` cleanup with fallback `Process.Kill`, plus Windows build-tagged cancellation regression tests.

## Best Possible Solution

Land a narrow Windows process-tree cancellation fix for the docs-i18n Codex timeout path after maintainer review and validation, preserving the existing Unix behavior and Windows build-tagged regression coverage.
