## Summary

Keep this PR open. Current main still has the existing fixed TUI footer and lacks the compact helpers and compact branded renderer, while the proposed branch changes a documented footer contract and still has concrete UX issues around elapsed-time refresh and unknown token pressure coloring.

## What This Changes

The PR replaces the fixed verbose TUI footer with a compact branded status line and adds helper functions for compact token labels, percentages, and a progress bar.

## Best Possible Solution

Use maintainer review to decide whether compact footer behavior should be a default, a mode, or part of the broader configurable footer/provider-status design; if pursued, update the renderer, timer refresh path, docs, and focused TUI tests together.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable for the feature request itself, but yes for the main PR defect: source inspection shows the proposed elapsed footer is recomputed only inside `updateFooter()`, while current busy timers update only the status loader.

## Solution Assessment

Is this the best way to solve the issue? Unclear. A compact footer may be useful, but the current patch is a hard replacement of documented fields; a maintainer-approved mode/config path coordinated with #53774 and #64086 is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Refresh the footer while showing elapsed time:** `src/tui/tui.ts:844-852`
  - body: The new footer renders `elapsedLabel` from `statusStartedAt`, but `updateFooter()` is not called by the existing busy-status timers. During a normal waiting/streaming run the footer elapsed segment will stay at its initial value, often `0s`, instead of reflecting run duration. Trigger `updateFooter()` from the same timer/activity transitions or remove the elapsed footer segment.
  - confidence: 0.87
- **[P3] Use neutral color for unknown token pressure:** `src/tui/tui.ts:833-850`
  - body: When token data is unavailable, `percentUsed` is `null`, but `renderCompactProgressBar(null, 12)` returns percent `0`, which selects the success color for the `?%` label. That makes unknown pressure look like an all-clear state; use a neutral/dim color whenever the percent is unknown.
  - confidence: 0.78
