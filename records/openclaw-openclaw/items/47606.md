## Summary

Current main does not implement the requested execution anti-drift guard system. It has durable task records, state-change notifications, task audits, ACP no-output notices, and disabled-by-default tool-loop detection, but no artifact-gated status policy, repeated status circuit breaker, no-artifact/no-phase-change escalation timers, no-idle-after-green rule, `ENFORCEMENT_GUARDS_V1` flag, or structured guard telemetry. This is broad runtime/product policy work and should remain open for maintainer design.

## Best Possible Solution

Keep this issue open for maintainer design. The best path is to define the execution-guard contract first: what counts as an artifact, which status/update surfaces are intercepted, where durable guard state lives, how escalation timers and cooldowns run across restarts, what telemetry schema is acceptable, and how the feature flag rolls out. After that, a focused implementation can update the task registry, task delivery policy, relevant agent stream/status surfaces, config schema/docs, and targeted tests.
