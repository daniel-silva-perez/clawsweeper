## Summary

Keep open. Current main has adjacent reliability pieces, but not the requested enforceable anti-drift guardrail system. Background tasks persist detached task records, tool-loop detection can warn/block repeated tool calls when explicitly enabled, and chat streaming exposes lifecycle/tool progress, but there is no artifact-gated status policy, persisted blocker/escalation state, escalation timers, no-idle-after-green enforcement, or `ENFORCEMENT_GUARDS_V1` feature flag. The related ClawSweeper reports in the provided context cover tool governance and WebChat status visibility, not this runtime reliability control.

## Best Possible Solution

Keep this issue open for explicit maintainer product/runtime design. A plausible implementation would add a feature-flagged guard layer in the agent runtime/status emission path, backed by durable per-run guard state with phase, artifact timestamps, blocker/escalation state, and timers; it should integrate with task records where appropriate but also cover normal chat and heartbeat runs that do not create tasks.
