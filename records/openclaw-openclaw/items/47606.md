## Summary

Keep open. Current main still has adjacent task records, task audits, ACP no-output notices, and disabled-by-default tool-loop detection, but it does not implement the requested artifact-gated execution guard system or its rollout flag/telemetry.

## What This Changes

The issue requests enforceable runtime guardrails for agent execution drift, including durable run state, artifact-gated status updates, loop circuit breaking, escalation timers, no-idle-after-green enforcement, a feature flag, and structured telemetry.

## Best Possible Solution

Define the execution-guard contract first, then implement it across the task/run state layer and the status-delivery surfaces with durable guard state, explicit artifact classification, timer/cooldown semantics, structured guard events, config/docs alignment, and targeted tests before a broad changed-lane gate.

## Reproduction Assessment

No, not as a runtime bug: the issue does not include a concrete session transcript or command sequence that reproduces a status-loop. The missing guard system is nevertheless verifiable by current-main code/docs inspection and repository search.

## Solution Assessment

No, the existing adjacent mechanisms are not a complete solution: task audits, ACP no-output notices, and tool-loop detection do not enforce artifact-gated status updates or post-green/no-phase-change escalation. A maintainable fix needs a designed cross-surface guard contract before code changes.
