## Summary

Current main has useful ingress provenance, external-content marker wrapping, and transcript-scoped persistence hooks, but it still does not carry provenance through tool results into durable state writes or expose read-time provenance for stored state. The related design tracks are still open, so this remains a real core security/plugin-API design item.

## Best Possible Solution

Keep this open as a core security/plugin-API design item unless maintainers explicitly choose #54508 as the canonical tracker. The concrete path is to design a generic provenance/taint contract for tool results and durable writes, likely alongside #48509's before_persistence_write hook, then add read-time metadata, policy hooks, docs, and regression tests across persistent state surfaces.
