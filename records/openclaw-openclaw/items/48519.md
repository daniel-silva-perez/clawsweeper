## Summary

Keep this open. Current main has message-level inter-session provenance and external-content boundary wrapping, but it does not carry provenance through tool results into durable state writes or expose a before_persistence_write/read-time provenance contract. The related #54508 and #48509 threads are useful context, but they are still open and do not make this narrower persistent-state provenance proposal obsolete enough to close.

## Best Possible Solution

Keep this issue open as a design/security API item, or explicitly make it a subtask of #54508 only after maintainers choose that tracker as canonical. The concrete path is to design a generic provenance/taint contract for tool results and durable writes, likely alongside #48509's before_persistence_write event, then add read-time metadata, policy hooks, docs, and focused tests across memory/files/scheduled tasks/config surfaces.
