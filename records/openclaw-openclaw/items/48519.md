## Summary

Keep this issue open. Current main has useful ingress provenance and external-content wrapping, plus transcript-level tool-result/message persistence hooks, but it still does not carry provenance through tool results into durable state writes or expose a read-time persistent-state provenance contract. The related #48503, #48509, and #54508 items are still open design tracks, not shipped replacements.

## Best Possible Solution

Keep this open as a core security/plugin-API design item unless maintainers explicitly choose #54508 as the canonical tracker. The concrete path is to design a generic provenance/taint contract for tool results and durable writes, likely alongside #48509's before_persistence_write hook, then add read-time metadata, policy hooks, docs, and regression tests across persistent state surfaces.
