## Summary

Keep open. Current main has exact Telegram topic ACP bindings, per-topic `agentId` routing, and manual `/acp spawn --thread`, but it still has no per-DM/per-chat `defaultAcp` config or inbound routing path that auto-spawns and binds ACP on the first message in a new unbound topic.

## What This Changes

The issue requests a per-DM/per-chat default ACP binding that automatically spawns and binds an ACP session for new Telegram DM topics or group forum threads.

## Best Possible Solution

Design a first-class, opt-in Telegram default ACP binding surface with clear group/direct/account precedence, per-topic override behavior, per-thread session isolation, explicit security gates, schema/help/docs updates, and focused Telegram plus ACP binding tests.
