## Summary

Keep open. Current main supports exact Telegram topic ACP bindings and manual thread-bound ACP spawns, but it does not provide a per-DM/per-chat default ACP config that automatically spawns and binds a new ACP session on first message in an unbound topic. The related configured-binding isolation report #64199 is also still open, so this should get maintainer design review before implementation.

## What This Changes

Add Telegram per-DM/per-chat default ACP configuration that auto-spawns an ACP session for new DM or group topics, while explicit per-topic configuration overrides the default.

## Best Possible Solution

Design an explicit opt-in Telegram default ACP binding surface with clear precedence, policy gates, and per-topic session-key isolation, then implement it through the existing configured/runtime binding seams with docs and focused Telegram plus ACP spawn tests.

## Risks / Open Questions

- Auto-spawning ACP from inbound chat messages is security-sensitive because ACP harnesses run on the host runtime and can read/write according to their own CLI permissions and selected `cwd`.
- The related configured ACP session-key isolation problem in #64199 should be resolved or explicitly accounted for before adding another automatic ACP binding path.
- The requested config shape spans account, direct, group, and topic precedence; maintainers need to settle override order and whether this belongs only to Telegram or a generic channel binding contract.
