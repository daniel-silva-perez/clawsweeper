## Summary

Keep open. The report is valid and current main contains the intended Telegram config-only accessor behavior, but this issue is paired with open PR #74833 using closing syntax, so conservative cleanup should leave the issue open until that PR is reviewed, landed, or explicitly superseded.

## What This Changes

The issue asks for Telegram status/read-only config accessors to read `allowFrom` and `defaultTo` without strict-resolving file-backed `botToken` SecretRefs, while keeping runtime token resolution strict.

## Best Possible Solution

Resolve the paired implementation path first: review and land #74833 if it is still the desired patch, or close it as superseded by the equivalent current-main implementation, then close this issue as implemented with the exact main commit.

## Reproduction Assessment

Yes. The issue gives a deterministic config shape, and current `extensions/telegram/src/shared.test.ts` covers a file-backed Telegram `botToken` SecretRef while reading `allowFrom` and `defaultTo` through the config adapter.

## Solution Assessment

Yes. A config-only `resolveAccessorAccount` is the narrow maintainable fix because it keeps metadata/status reads read-only while preserving strict `resolveTelegramAccount` behavior for runtime token use.
