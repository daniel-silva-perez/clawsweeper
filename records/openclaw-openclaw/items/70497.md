## Summary

Current main and v2026.4.26 still do not strip JSON Schema conditional keywords `if`, `then`, or `else` in the Gemini schema cleaner, so this PR is not obsolete. The branch still contains the narrow intended Gemini cleaner/test fix, but it also includes an unrelated pairing-store test stabilization, so it should stay open for maintainer review rather than be closed by cleanup.

## What This Changes

The PR branch adds `if`, `then`, and `else` to Gemini's unsupported schema keyword set, adds a regression test for stripping those conditionals, and also changes pairing-store test read-count assertions.
