## Summary

Keep this PR open. Current main still resolves a mixed MCP server config with both non-empty `command` and `url` as stdio before it ever considers HTTP/streamable-http, while docs already say that shape should be rejected. The PR remains a focused implementation candidate for the linked bug and does not appear obsolete on main.

## Best Possible Solution

Review and land this focused PR, or apply the same early guard and tests on main. The fix belongs in `src/agents/mcp-transport-config.ts` before stdio resolution, so mixed `command` plus non-empty `url` configs fail clearly while pure stdio, SSE, and streamable-http configs retain current behavior.
