## Summary

Keep this PR open for maintainer review and merge judgment. Current main still has the silent stdio-first behavior, while the PR is a focused fix with targeted tests and no blocking review findings from this read-only pass.

## What This Changes

This PR adds an early resolver guard that skips MCP server configs with both non-empty `command` and `url`, plus tests for mixed configs, whitespace-only URLs, and sanitized diagnostics.

## Best Possible Solution

Land this narrow resolver guard and tests, or apply the same change on main, so OpenClaw rejects ambiguous MCP stdio/HTTP server configs at config-resolution time while preserving pure stdio, pure SSE, and pure streamable-http behavior.

## Reproduction Assessment

Yes. The current-main code path is deterministic: a config with non-empty `command` and `url` resolves as stdio before HTTP is considered, and the runtime timeout string matches the linked #72111 symptom.

## Solution Assessment

Yes. The PR solves the issue at the smallest maintainable boundary, where both fields and the requested transport hint are visible, without changing lower-level stdio or HTTP transport helpers.
