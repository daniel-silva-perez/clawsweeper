## Summary

Keep open. Current main still lacks Discord CHANNEL_DELETE-driven session cleanup, and this PR remains a plausible implementation candidate, but its patch is stale against the current plugin layout and needs a security-conscious refresh before it can land.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a current-main refresh under `extensions/discord/src/monitor`: add and register a channel-delete listener, delete exact deleted-channel session entries from the correct agent stores inside `updateSessionStore`, collect deleted entries inside the lock, archive and clean up transcripts through a plugin-safe or core-owned helper with `restrictToStoreDir`, add Discord plugin tests, and run the changed gate before merge.
