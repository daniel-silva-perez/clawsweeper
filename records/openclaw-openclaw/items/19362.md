## Summary

Current main still does not implement Discord channel-deletion-driven session cleanup, so this PR remains a useful implementation candidate. It should not be closed as implemented; it needs a refresh from the old `src/discord` layout to the current bundled plugin under `extensions/discord`.

## Best Possible Solution

Keep this PR open as the implementation candidate, but refresh it against current main under `extensions/discord/src/monitor`: add and register a Carbon channel-delete listener, delete only matching `agent:<agentId>:discord:channel:<channelId>` entries through plugin-safe session-store APIs, preserve transcript/archive cleanup semantics, add focused Discord plugin tests, and address the existing review comments before the changed gate.
