## Summary

Keep open. Current main still does not implement Discord channel-deletion-driven session cleanup, and this PR remains a focused implementation candidate for the still-real orphaned Discord channel session problem. The patch needs a refresh because Discord is now under the bundled plugin layout, while the PR was authored against the old `src/discord` tree.

## Best Possible Solution

Keep this PR open as an implementation candidate, but refresh it against current main under `extensions/discord/src/monitor`: add and register a Carbon channel-delete listener, delete only exact `agent:<agentId>:discord:channel:<channelId>` entries through plugin-safe session-store runtime APIs, archive and clean up transcripts with store-dir restrictions, add focused Discord plugin tests, and then run the changed gate before merge.
