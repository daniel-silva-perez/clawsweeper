## Summary

Keep this PR open. Current main still does not implement TUI image attachments: the TUI send contract and Gateway-backed client are text-only, `sendMessage` forwards the raw input text, the slash-command list and public TUI docs have no `/image` command or `@path` syntax, while the Gateway protocol/server already has a generic attachment path that a TUI implementation could use. The PR remains relevant, but it needs maintainer review/rebase and fixes for the known review comments before it is merge-ready.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase or supersede it with a focused TUI implementation that adds image extraction/loading, threads attachments through the shared TUI backend contract, forwards them through `GatewayChatClient`, handles or explicitly rejects local embedded mode, preserves prompt text when no images are present, prevents raw local path leakage, documents `/image` and `@path`, and adds focused parser/send-path tests.
