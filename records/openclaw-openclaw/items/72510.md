## Summary

Keep this PR open for maintainer review. Current main still lacks the requested group-chat targeting metadata, so the work is not obsolete, but the proposed patch is not yet a complete fix for the linked Telegram multi-bot repro because skipped messages only scan for the current bot and the PR test expectation is stale against current main's group-context output.

## What This Changes

The PR adds optional bot-targeting fields to auto-reply history/context types, renders those fields in inbound prompt metadata, adds Telegram detection for mentions and reply-to-bot authorship, and covers the new behavior with auto-reply and Telegram helper tests.

## Best Possible Solution

Rebase the PR onto current main, keep the additive core metadata contract, and either wire a reliable Telegram sibling-bot username source into skipped-history detection or narrow the PR so it does not claim to close the other-bot mention case. The final version should preserve existing group prompt guidance, keep the null/undefined semantics covered, and validate both current-bot and other-bot history annotations.
