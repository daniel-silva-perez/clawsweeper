## Summary

Keep open. Current main still lacks the bot-targeting history/reply metadata this PR is trying to add, so it is not obsolete; however the latest PR head still has an incomplete Telegram sibling-bot path and a stale byte-stability test against current main.

## What This Changes

The PR adds optional bot-targeting metadata to auto-reply history/template context, renders it in inbound prompt metadata, and updates Telegram parsing, reply targets, skipped-history plumbing, and tests to populate mention/reply bot fields.

## Best Possible Solution

Keep the additive metadata contract, but require the PR to either wire a trustworthy Telegram sibling-bot username source into skipped-history detection or clearly narrow the scope to current-bot/reply-only metadata. The final branch should rebase cleanly onto current main, preserve the existing group prompt guidance, and cover current-bot, sibling-bot, reply-to-bot, and ambient history behavior with focused tests.
