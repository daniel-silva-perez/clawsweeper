## Summary

Keep open. Current main has a Discord command serializer capable of emitting `description_localizations`, but the native command spec and plugin command metadata surfaces still only carry English `description` strings, and `createDiscordNativeCommand` only assigns that default description. The report has concrete reproduction steps and is not old enough, vague enough, duplicated, or implemented enough to close.

## What This Changes

The issue requests extending Discord native slash command specs to carry `descriptionLocalizations` so reconcile publishes and preserves localized descriptions instead of English-only descriptors.

## Best Possible Solution

Add a typed localization metadata path for native command descriptions, carry it from built-in and plugin command definitions into Discord `BaseCommand.descriptionLocalizations`, and cover reconcile with serialization/regression tests plus docs/changelog. Keep English `description` as the required fallback and avoid bundling actual locale translations unless maintainers choose a source-of-truth for those strings.

## Reproduction Assessment

Yes. The source path is enough to reproduce with high confidence: create a desired native Discord command from a spec, serialize it, and observe that only English `description` is populated; reconcile then compares that desired payload against live Discord command metadata and edits drift away.

## Solution Assessment

Yes. The suggested direction is the narrow maintainable fix: extend the command spec/API and pass the metadata through the existing Discord serializer, which already knows how to emit `description_localizations`. A broader translation catalog can remain a separate product decision.
