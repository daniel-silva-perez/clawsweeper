## Summary

Keep open. Current main still has the reported Discord native-command localization gap: the Discord serializer can emit `description_localizations`, but the native command spec, command registry, plugin command surface, and Discord command creation path only carry the English fallback description into the desired payload.

## What This Changes

Request support for Discord slash command `description_localizations` so native command reconcile preserves localized command descriptions instead of redeploying only packaged English text.

## Best Possible Solution

Add a typed optional description-localization path for native command definitions, propagate it from built-in, skill, and plugin command specs into Discord `Command.descriptionLocalizations`, and cover serialization plus reconcile behavior with focused regression tests. Keep English `description` as the required fallback and document any plugin-facing metadata that becomes public.

## Reproduction Assessment

Yes. A focused source-level reproduction is enough: current main can build and serialize Discord native commands only from the fallback `description`, while reconcile compares that desired payload against live command metadata and edits on drift.

## Solution Assessment

Yes. The narrow maintainable fix is to extend the existing native command metadata path and pass it through the already-capable Discord serializer; a product-wide translation catalog is not required for this issue.
