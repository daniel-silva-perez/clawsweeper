## Summary

Keep open. Current main still has the reported Discord native-command localization gap: the low-level Discord serializer can emit `description_localizations`, but native command specs, plugin command metadata, and Discord command creation only carry the English `description`, so reconcile can still replace localized live descriptions with the packaged fallback.

## What This Changes

Support Discord slash command `description_localizations` through native command specs, command generation, and reconcile comparisons while keeping English descriptions as fallbacks.

## Best Possible Solution

Add a typed native command localization metadata path for descriptions, propagate it from built-in, skill, and plugin command definitions into Discord `Command.descriptionLocalizations`, and cover both serialization and reconcile behavior with focused regression tests. Keep the required English `description` as the fallback and document the plugin-facing metadata if exposed publicly.

## Reproduction Assessment

Yes. A focused source-level reproduction is enough: build a Discord native command from a spec and serialize it; current main emits the English `description` only, while reconcile compares that desired payload against live Discord command metadata and edits on drift.

## Solution Assessment

Yes. The narrow maintainable solution is to extend the existing command spec/API and pass localization metadata through the already-capable Discord serializer, with tests around generated desired payloads and plugin command specs.
