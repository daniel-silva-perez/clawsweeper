## Summary

Keep this issue open. Current main still does not expose a localhost/private-network opt-in for `web_fetch`; it only exposes the narrower RFC2544 fake-IP proxy exception. The provided GitHub context also shows open same-author PR #39630 with closing syntax, so the right maintainer path is to review, land, replace, or close that PR before closing this issue.

## Best Possible Solution

Keep this issue open and have maintainers review, update, land, replace, or close PR #39630. The best core fix is a default-off web_fetch private-network policy surface aligned with current `tools.web.fetch.ssrfPolicy` patterns, plus strict schema/types/help/docs updates, runtime policy forwarding, cache-key isolation, and SSRF tests for direct hosts, DNS resolution, and redirects.
