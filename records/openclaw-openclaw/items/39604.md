## Summary

Keep this issue open. Current main still does not expose a private-network opt-in for web_fetch; it only exposes the narrower RFC2544 fake-IP proxy exception, and PR #39630 is still open as the same-author implementation candidate.

## Best Possible Solution

Keep this issue open and have maintainers review, update, land, replace, or close PR #39630. The best core fix is a default-off web_fetch private-network policy surface aligned with the current tools.web.fetch.ssrfPolicy pattern, plus strict schema/types/help/docs updates, runtime policy forwarding, policy-aware cache keys, and SSRF tests for direct hosts, DNS resolution, redirects, and default-block behavior.
