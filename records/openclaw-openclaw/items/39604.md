## Summary

Keep this issue open. Current main still lacks a web_fetch private-network opt-in; it only exposes the narrower `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange` fake-IP proxy exception. There is also an open same-author implementation candidate, PR #39630, that uses closing syntax for this issue, so this issue should stay open until that PR is reviewed, landed, replaced, or closed.

## Best Possible Solution

Keep this issue open and have maintainers review, update, land, replace, or close PR #39630. The best core fix is a default-off web_fetch private-network policy surface aligned with the current `tools.web.fetch.ssrfPolicy` shape, with strict schema/types/help/docs updates, runtime policy forwarding, policy-aware cache keys, and SSRF tests for direct hosts, DNS resolution, redirects, and default-block behavior.
