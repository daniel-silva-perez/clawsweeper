## Summary

Keep this issue open. Current main still does not expose a private-network opt-in for `web_fetch`; it only supports the narrower `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange` fake-IP proxy allowance. The same-author implementation candidate #39630 remains open and uses closing syntax for this issue, so maintainers should review, land, replace, or close that PR before resolving this issue.

## Best Possible Solution

Keep this issue open while maintainers review #39630 or choose a replacement design. The strongest core path is likely a default-off `tools.web.fetch.ssrfPolicy` surface with explicit dangerous private-network naming and/or narrower hostname controls, wired through strict schema, exported types, generated help/labels, runtime policy forwarding, cache isolation, docs, and focused SSRF regression tests.
