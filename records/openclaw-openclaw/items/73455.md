## Risks / Open Questions

- Functional risk: review comments identify unresolved mismatches where holdback can suppress `revise`/`auto` freshness modes and where participation can retain a deterministic fallback despite the AI-first no-fallback contract.
- Architecture risk: the PR patch shown in the provided context imports core internals from Matrix production code via relative `src/**` paths, which conflicts with the bundled plugin boundary in `extensions/AGENTS.md`.
- Security/supply-chain risk: unrelated workflow changes add alternate package sources for a live QA workflow, including ref, URL, and artifact paths, and should be separated from the Matrix feature for focused review.
- Privacy/product risk: AI participation/freshness classification sends bounded Matrix room context to a model; that may be acceptable, but it needs explicit docs/defaults and maintainer approval.
- Branch hygiene risk: the PR bundles Matrix feature work with release workflow docs, generated baselines, changelog churn, updater changes, and live-test stabilizations, making review and landing unsafe as-is.
