## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Default the AI gates to explicit opt-in:** `extensions/matrix/src/matrix/monitor/index.ts:243-259`
  - body: `participationEnabled` and `freshnessEnabled` are resolved with `!== false`, with zero member thresholds and AI-backed defaults. That means existing Matrix rooms can start running model-based gating and sending room context to the model without a user adding the new blocks, despite the feature being described as optional. Default these controls off unless explicitly configured, or add a documented migration/approval path.
  - confidence: 0.87
- **[P2] Let revise and auto pass the holdback gate:** `extensions/matrix/src/matrix/monitor/handler.ts:2360-2371`
  - body: When the pre-draft holdback observes newer room activity, the branch suppresses every mode except `send-as-is`. That makes `revise` and `auto` skip drafting entirely instead of allowing the freshness resolver/final gate to revise or choose an allowed action. Restrict the early suppression to `freshnessMode === "suppress"`.
  - confidence: 0.86
- **[P2] Remove the deterministic ai-first fallback:** `extensions/matrix/src/matrix/monitor/participation-policy.ts:378-382`
  - body: `resolveParticipationDecision` can parse raw `text` deterministically when `currentDirective` is absent. That leaves a silent fallback path inside the decision helper even though the PR contract says `ai-first` must not silently fall back to deterministic behavior; remove the raw text fallback and require callers to pass an already resolved directive.
  - confidence: 0.82
- **[P2] Use plugin SDK runtime imports:** `extensions/matrix/src/matrix/monitor/handler.ts:11-15`
  - body: The Matrix plugin production handler imports simple-completion and reply helpers through relative `../../../../../src/**` paths. Bundled plugins are supposed to use `openclaw/plugin-sdk/*` seams, and this repo already exposes simple-completion and reply runtime subpaths, so these imports should be moved to the SDK surface before merging.
  - confidence: 0.9
- **[P2] Fix the AI participation schema prompt:** `extensions/matrix/src/matrix/monitor/participation-policy.ts:314`
  - body: The participation system prompt’s schema string has malformed optional array field names for `includeAgentIds?` and `excludeAgentIds?`. Since the model is being asked to emit structured JSON, keep the schema string syntactically clear so those fields are not under-produced or misread.
  - confidence: 0.78
