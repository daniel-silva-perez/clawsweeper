## What This Changes

The issue requests detection and recovery for a Gateway process that remains alive under systemd while the Node.js event loop stops servicing outbound HTTP and channel traffic.

Keep this open until maintainers decide the recovery contract: either add a supported supervisor watchdog path for Linux/WSL services, document and expose a reliable external probe command, or identify and fix the underlying Feishu/HTTP/event-loop blocker with regression coverage.
