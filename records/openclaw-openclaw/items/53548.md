## Summary

Keep open. Current main still rejects unthreaded persistent `sessions_spawn` in both native subagent and ACP paths, the tests and docs still encode the `thread=true` requirement, and open PR #39372 remains only an implementation candidate rather than shipped behavior.

## What This Changes

Request to allow `sessions_spawn` with `mode:"session"` without `thread:true` so native subagent and ACP sessions can persist headlessly on channels without thread binding.

## Best Possible Solution

Review and harden #39372 or a replacement so native and ACP persistent sessions can exist without thread binding, while preserving existing `thread:true` behavior, returning stable continuation metadata for `sessions_send`, preventing unintended ACP inline delivery for headless sessions, and updating tests plus public docs.
