## Summary

Keep open. Current main still rejects unthreaded persistent `sessions_spawn` in both native subagent and ACP paths, the regression tests and docs still encode the `thread=true` requirement, and the related open PR #39372 is only an implementation candidate rather than shipped behavior.

## What This Changes

Request to allow persistent `sessions_spawn` with `mode: "session"` without requiring `thread: true`, so headless native subagent and ACP sessions can continue through `sessions_send` on channels without thread binding.

## Best Possible Solution

Review and harden #39372 or a replacement: remove both native and ACP unthreaded-session guards, keep `thread=true` behavior unchanged, return stable session metadata for `sessions_send`, prevent ACP headless sessions from inline-delivering to the requester chat unless a bound conversation exists, and update tests plus `docs/tools/subagents.md` and `docs/tools/acp-agents.md`.
