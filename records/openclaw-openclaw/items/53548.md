## Summary

Current main still explicitly rejects `sessions_spawn` with `mode: "session"` unless `thread: true` is set in both native subagent and ACP spawn paths. Tests and public docs still encode that policy, and the related open PR #39372 remains only an implementation candidate rather than shipped behavior.

## Best Possible Solution

Keep this issue open. The best path is to review and harden #39372 or a replacement that decouples persistent session lifecycle from channel thread binding in `src/agents/subagent-spawn.ts` and `src/agents/acp-spawn.ts`, returns stable `childSessionKey`/label metadata for `sessions_send`, delivers unthreaded output through the requester or parent conversation, preserves existing `thread=true` behavior, and updates tests plus `docs/tools/subagents.md` and `docs/tools/acp-agents.md`.
