## Summary

Current main still implements the reported coupling: both native subagent and ACP `sessions_spawn` paths reject `mode: "session"` unless `thread: true` is set. Tests and public docs still encode that behavior, and the related open PR #39372 is only an implementation candidate, not shipped main behavior.

## Best Possible Solution

Keep this issue open. The best path is to review and harden #39372 or a replacement that decouples persistent session lifecycle from channel thread binding in `src/agents/subagent-spawn.ts` and `src/agents/acp-spawn.ts`, returns stable `childSessionKey`/label metadata for `sessions_send`, delivers unthreaded output through the requester or parent conversation, preserves current `thread=true` behavior, and updates tests plus `docs/tools/subagents.md` and `docs/tools/acp-agents.md`.
