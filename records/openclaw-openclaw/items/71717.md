## Summary

Keep open. Current main still has the plausible Windows host-exec failure path: Windows exec resolves PowerShell, sends the command through the supervisor child spawn path, and the shared spawn fallback still retries only EBADF by default, while the related Microsoft tracker is open inventory rather than a merged fix.

## What This Changes

The issue reports that every agent exec command on Windows 11 fails immediately with spawn EPERM when using OpenClaw 2026.4.23 from the global npm package.

## Best Possible Solution

Add a focused native Windows investigation path that records the selected shell path, cwd, command text, service/user context, and original spawn error fields, then land either a constrained validated fallback or a clearer diagnostic with mocked Windows coverage and a native Windows smoke check.

## Reproduction Assessment

No. The issue gives plausible Windows 11 steps and current main has a path where spawn-time EPERM would fail before command execution, but the report lacks native logs or selected-shell evidence needed for a high-confidence reproduction.

## Solution Assessment

Unclear; there is no proposed patch. A blanket EPERM retry is not the best first move for host exec, so the safer path is targeted Windows reproduction followed by a constrained fallback or diagnostic once the failing permission case is known.
