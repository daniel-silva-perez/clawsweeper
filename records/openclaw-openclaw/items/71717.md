## Summary

Keep open. Current main still has no implemented Windows `spawn EPERM` recovery path for normal gateway exec commands: Windows exec selects one PowerShell executable, the runtime passes that single shell argv to the process supervisor, and the shared spawn fallback retries only EBADF/no-detach style failures rather than EPERM. The related Windows exec reports cover different failures and do not supersede this one.

## Best Possible Solution

Keep this issue open for a focused Windows exec investigation. Capture selected shell path, cwd, spawn error code/syscall, and PowerShell availability; then add either a narrow diagnostic or a guarded Windows PowerShell fallback if validated, with mocked supervisor/spawn coverage and a native Windows sanity check.
