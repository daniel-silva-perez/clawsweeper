## Summary

Keep this PR open. Current main still has both behaviors this PR is trying to repair: a deferred same-session `session.message` reload is resolved but not replayed after a locally appendable final assistant event, and a role-less final text/content payload is still treated as a renderable assistant message for final-history-reload suppression. The PR branch is still available at head `4bd0589d823319693363a6c374e2d00aa9d132e7`, and its diff directly changes the implicated UI guard plus regression tests. No conservative close reason applies.

## Best Possible Solution

Keep this PR in review and land it, or land an equivalent patch, so Control UI replays deferred same-session `session.message` reloads after locally appendable final events and only suppresses final-event history reloads for explicit renderable assistant messages. Preserve the regression coverage and changelog credit, then close related chat reload race reports after the fix reaches main.
