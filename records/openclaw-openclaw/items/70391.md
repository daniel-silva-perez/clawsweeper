## Summary

Keep this PR open. Current main still has the exact Control UI gap this PR targets: deferred same-session `session.message` reloads are cleared but not replayed after a renderable final assistant event, and role-less final payloads still suppress the final history reload. The PR also has an active maintainer repair on its branch and merge-ready context, so no conservative close reason applies.

## Best Possible Solution

Keep this PR in review and land it, or land an equivalent patch, so Control UI replays deferred same-session `session.message` reloads after locally appendable final events and only suppresses final-event history reloads for explicit renderable assistant messages. Preserve the included regression coverage and changelog entry, then close the relevant chat reload race reports after merge.
