## Summary

Keep this PR open. Current main still contains the two behaviors this PR is meant to change: a deferred same-session `session.message` reload is cleared but not replayed after a renderable final assistant payload, and role-less visible final payloads still suppress the safer history reload path. The PR diff is focused on Control UI logic/tests plus changelog text; the security pass did not find CI, dependency, lockfile, permission, secret, or package-publishing changes.

## Best Possible Solution

Keep this PR in review and land it, or land an equivalent focused patch, so Control UI replays deferred same-session `session.message` reloads after locally appendable final events and only suppresses final-event history reloads for explicit renderable assistant messages. Preserve the focused regression coverage and then close the related chat reload race reports after the fix reaches main.
