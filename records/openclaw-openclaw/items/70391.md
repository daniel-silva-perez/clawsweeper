## Summary

Keep this PR open. Current main still contains the two behaviors this PR is meant to repair: deferred same-session transcript reloads can be cleared after a locally appended final assistant payload without replaying `chat.history`, and role-less visible final payloads are still treated as renderable assistant messages. The PR diff is focused on UI logic, tests, and changelog text; the security pass did not find supply-chain or permissions-sensitive changes.

## Best Possible Solution

Keep this PR in review and land it, or land an equivalent focused patch, so Control UI replays deferred same-session `session.message` reloads after locally appendable final events and only suppresses final-event history reloads for explicit renderable assistant messages. Preserve the regression coverage and contributor changelog credit, then close related chat reload race reports after the fix reaches main.
