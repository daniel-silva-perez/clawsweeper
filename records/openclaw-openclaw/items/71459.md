## Summary

Keep open. Current main still uses the generic missing-register diagnostics in both plugin-loader paths and still logs malformed bundled channel entries only as a generic contract miss, so this PR's core diagnostics are not implemented elsewhere. The latest branch appears to have restored the normal-plugin debug formatter path, but it still misses setup-entry wrong-loader failures and includes a root TODO.md artifact that should not land.

## What This Changes

The PR adds field-level bundled channel entry warnings, special missing-register diagnostics for bundled channel entries in the full and CLI plugin loaders, regression tests, and a root TODO.md implementation-plan file.

## Best Possible Solution

Land a narrow diagnostics patch that preserves formatMissingPluginRegisterError for normal plugins, handles both bundled-channel-entry and bundled-channel-setup-entry wrong-loader cases in full and CLI loader paths, keeps the field-level bundled contract warnings, removes PR-only artifacts, and covers the behavior with focused loader and bundled-entry tests.

## Reproduction Assessment

Yes. A source-level reproduction is clear: load a bundled setup entry through the legacy plugin loader; resolvePluginModuleExport can return a definition with kind bundled-channel-setup-entry and no register, while the PR's new branch only recognizes bundled-channel-entry.

## Solution Assessment

No, not yet. The direction is appropriate, but the implementation should use a shared bundled wrong-loader predicate/message that includes setup entries, preserve the existing debug formatter for ordinary plugins, and remove the root TODO.md before merge.
