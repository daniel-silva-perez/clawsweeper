Full review comments:

- **[P2] Compare visible memory before rounding:** `src/commands/doctor-wsl.ts:355-356`
  - body: The missing-`.wslconfig` branch rounds `wslVisibleMemoryBytes` to whole GB before checking `< 4`, so 3.5-3.99GB rounds to `4` and suppresses the low-memory warning. Compare bytes or MB against the 4GB threshold and only round for display.
  - confidence: 0.91
- **[P2] Hide WSL2 limits from WSL1 summaries:** `src/commands/doctor-wsl.ts:379-383`
  - body: `buildWSLDiagnosticNotes()` correctly skips resource warnings for WSL1, but `buildWSLInfoSummary()` still appends `.wslconfig` memory/processors whenever present. Since `.wslconfig` resource caps do not apply to WSL1, gate these summary fields on `diag.isWSL2`.
  - confidence: 0.87
- **[P3] Distinguish unreadable config from missing config:** `src/commands/doctor-wsl.ts:211-212`
  - body: `readWSLConfigResources()` returns `null` for missing, unreadable, and unresolved `.wslconfig`, and the diagnostic then says `No .wslconfig found`. Preserve an explicit read state so users do not get a misleading missing-file remediation when the file exists but cannot be read.
  - confidence: 0.75
