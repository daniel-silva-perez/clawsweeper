## Summary

Keep this PR open. Current main only partially honors `authHeader: true`: runtime model setup can add `Authorization: Bearer ...`, but the current Anthropic transport still builds `x-api-key` from the API key, and onboarding verification still has no `authHeader` path for Anthropic-compatible custom providers.

## Best Possible Solution

Rebase and narrow the PR around the current auth contract: make Anthropic-compatible runtime transport use Bearer auth without also generating `x-api-key` when `authHeader: true`, teach custom Anthropic endpoint verification to honor the same setting, preserve default native Anthropic behavior, and add focused tests for both paths.
