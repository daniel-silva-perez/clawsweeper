## Summary

Current main still has the reported Synology Chat webhook behavior: HEAD/non-POST requests are rejected with 405, and accepted POSTs still ACK with 204/no body. The issue is also paired with open closing PRs, especially the reporter's focused PR #53441, so it should stay open until a fix lands or the PRs are explicitly superseded.

## Best Possible Solution

Keep this issue open. Review and either land the focused same-author PR #53441 or explicitly supersede it with #53522. The fix should stay in extensions/synology-chat/src/webhook-handler.ts: accept HEAD probes with 200, return 200 with {"success":true} for successful POST ACKs, and update webhook plus integration tests that currently assert 204/405 behavior.
