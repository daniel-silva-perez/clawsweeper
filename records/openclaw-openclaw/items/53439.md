## Summary

Keep open. Current main still rejects Synology Chat HEAD probes through the non-POST 405 path and still ACKs accepted POST deliveries with 204/no body, while open implementation PR #53441 specifically references and fixes this issue.

## What This Changes

The issue requests that the Synology Chat webhook accept HEAD probes and return 200 with {"success":true} for successful POST acknowledgements instead of 204/no body.

## Best Possible Solution

Land one focused Synology Chat plugin fix that accepts HEAD probes with 200, returns 200 plus {"success":true} for accepted POST ACKs, preserves existing POST auth/rate-limit/allowlist behavior, and updates the webhook plus integration tests that currently expect 204.
