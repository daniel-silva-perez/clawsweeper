## Summary

Keep open. Current main still matches the reported Synology Chat incompatibility: HEAD probes are rejected through the non-POST 405 path, successful POST ACKs still return 204/no body, and the issue is paired with open implementation PRs that should be reviewed or superseded before this issue closes.

## What This Changes

The issue requests changing Synology Chat webhook handling so HEAD probes return 200 and successful POST ACKs return 200 with a JSON success body instead of 204/no body.

## Best Possible Solution

Review and land #53441, or explicitly supersede it with a focused Synology Chat plugin change that accepts HEAD probes, returns 200 with {"success":true} for successful POST ACKs, preserves existing token/allowlist/rate-limit behavior, and updates the webhook plus integration tests that currently expect 204.
