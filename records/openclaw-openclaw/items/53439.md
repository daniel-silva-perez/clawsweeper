## Summary

Keep open. Current main still rejects Synology Chat HEAD probes and still acknowledges successful POST deliveries with 204/no body, while the issue is paired with open implementation candidates, especially the focused same-author PR #53441 that uses closing syntax.

## What This Changes

The issue asks the Synology Chat webhook route to accept HEAD probes with 200 OK and to return 200 with {"success":true} for successful POST acknowledgements instead of 204/no body.

## Best Possible Solution

Review and land #53441, or explicitly supersede it with a narrow replacement that keeps the change inside the Synology Chat plugin: accept HEAD probes, return 200 with {"success":true} for successful POST ACKs, and update the webhook and integration tests that currently expect 204.
