## Summary

Keep open. Current main still has the blanket same-target suppression in both reply paths, and this PR remains the active implementation candidate for the open linked bug. The functional direction is sound, but the branch still has one concrete test-isolation issue before it should merge.

## What This Changes

The PR changes auto-reply/followup delivery to use text/media-level deduplication for same-target messaging-tool sends, adds regression tests, preserves followup account fallback routing, and adds a changelog entry.

## Best Possible Solution

Land this PR or an equivalent focused fix after addressing the test-isolation finding: remove blanket same-target suppression, keep target-gated text/media dedupe, preserve account-scoped followup routing, and then close #59743 as implemented.

## Reproduction Assessment

Yes. Current main has a deterministic unit-level reproduction: same-target messaging-tool metadata plus different final text still returns no reply payloads in `buildReplyPayloads` and `resolveFollowupDeliveryPayloads`.

## Solution Assessment

No, not quite yet. The content-level dedupe approach is the right maintainable direction, but the branch should fix the plugin registry teardown before merge and get maintainer signoff on the partial-metadata dedupe tradeoff.
