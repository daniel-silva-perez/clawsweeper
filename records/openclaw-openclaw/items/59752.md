## Summary

Keep this PR open. Current main still contains the blanket same-target suppression behavior this PR is meant to replace, and current tests still assert the old behavior where non-duplicate final text is dropped after a same-target messaging-tool send. The PR is also paired with an open linked bug from the same contributor and is an active implementation candidate, not obsolete cleanup.

## Best Possible Solution

Keep this PR open for human maintainer review. The likely best path is to land a narrowed version that removes blanket same-target suppression, keeps content-level text/media dedupe, preserves account-scoped routing, and retains regression coverage across agent-runner, followup-delivery, and followup-runner paths. After that merges, the linked bug can close as implemented.
