## Summary

Keep this issue open. Current main still skips MEDIA directives inside fenced Markdown code blocks before validation/extraction, and in-tree tests explicitly assert that fenced MEDIA text remains visible text. The related implementation PR #41980 was closed unmerged, so it is not shipped evidence.

## Best Possible Solution

Keep this issue open. The likely fix belongs in src/media/parse.ts and src/media/parse.test.ts: either extract valid own-line MEDIA directives from fenced blocks while preserving existing validation, or intentionally keep skipping them but add visible warning/logging plus assistant/docs guidance that MEDIA directives must not be wrapped in code fences or backticks. Review or replace the closed #41980 implementation before closing this issue.
