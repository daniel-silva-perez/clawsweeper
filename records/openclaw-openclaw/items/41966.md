## Summary

Keep this issue open. Current main still skips `MEDIA:` directives inside Markdown fenced code blocks, tests explicitly assert that fenced `MEDIA:` remains visible text, and the related implementation PR #41980 was closed unmerged. The docs/prompt guidance also does not currently warn agents or users that fenced directives are not extracted.

## Best Possible Solution

Keep this issue open. Maintainers should choose the intended media-directive contract, then update `src/media/parse.ts` and `src/media/parse.test.ts` either to extract valid own-line fenced `MEDIA:` directives through the existing validation path, or to preserve the skip behavior while adding a visible warning plus clearer system-prompt and docs guidance. Closed PR #41980 is useful prior art but needs review or replacement.

## Risks / Open Questions

- Extracting `MEDIA:` from fenced code could accidentally convert illustrative examples into attachments unless limited to valid own-line directives and routed through the existing validation/trust-boundary path.
- If maintainers intentionally keep fenced `MEDIA:` unsupported, the silent failure remains unless the parser emits an observable warning and prompt/docs guidance explicitly tells agents not to wrap directives in Markdown fences.
