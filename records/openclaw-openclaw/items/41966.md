## Summary

Keep this issue open. Current main still skips MEDIA directives inside Markdown fenced code blocks before the normal MEDIA validation path, regression coverage explicitly asserts that fenced MEDIA remains visible text, and the related fix PR #41980 was closed unmerged. The docs/prompt guidance still does not resolve the mismatch: one public setup page shows a fenced MEDIA example and says OpenClaw extracts it.

## Best Possible Solution

Keep this issue open. Maintainers should choose the intended MEDIA directive contract, then update `src/media/parse.ts` and `src/media/parse.test.ts` either to extract valid own-line fenced MEDIA directives through the existing validation path, or to preserve the skip behavior while adding a visible warning plus explicit prompt and docs guidance. Closed PR #41980 is useful prior art but does not supersede the remaining work.
