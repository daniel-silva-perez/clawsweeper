## Summary

Keep this issue open. Current main still skips `MEDIA:` directives inside Markdown fenced code blocks before normal media matching/validation, and current regression coverage explicitly asserts that fenced `MEDIA:` remains visible text. The related fix PR #41980 was closed unmerged in the provided GitHub context, so it does not supersede the issue. Public/prompt guidance also still leaves a mismatch: the assistant prompt says own-line `MEDIA:` requests attachment delivery without warning about fences, and the public setup docs show a fenced `MEDIA:` example followed by text saying OpenClaw extracts it.

## Best Possible Solution

Keep this issue open. Maintainers should choose the intended `MEDIA:` directive contract, then update `src/media/parse.ts` and `src/media/parse.test.ts` either to extract valid own-line fenced `MEDIA:` directives through the existing validation path, or to preserve the skip behavior while adding a visible warning plus explicit prompt and docs guidance. Closed PR #41980 is useful prior art but does not supersede the remaining work.
