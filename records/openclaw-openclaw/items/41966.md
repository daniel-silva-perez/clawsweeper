## Summary

Keep this issue open. Current main still skips `MEDIA:` directives inside Markdown fenced code blocks, the regression test explicitly preserves that behavior, and user-facing guidance still says own-line `MEDIA:` triggers attachment delivery without warning that fences suppress extraction. The related fix PR in the provided context was closed unmerged, so it is useful prior art but not a superseding implementation.

## Best Possible Solution

Keep this issue open. Maintainers should choose the intended `MEDIA:` contract, then either update `src/media/parse.ts` and `src/media/parse.test.ts` to extract valid own-line fenced directives through the existing validation path, or preserve the skip behavior while adding a warning and updating the assistant prompt plus public docs. Closed PR #41980 is useful prior art but should not close this issue before a maintained fix lands.
