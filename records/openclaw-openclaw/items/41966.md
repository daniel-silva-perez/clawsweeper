## Summary

Keep this issue open. Current main still intentionally skips `MEDIA:` lines inside fenced Markdown blocks, regression coverage asserts that behavior, and user-facing prompt/docs guidance still describes own-line `MEDIA:` as attachment delivery without warning that code fences suppress extraction. The related fix PR #41980 is closed unmerged, so it remains useful prior art but not a superseding implementation.

## Best Possible Solution

Keep this issue open. Maintainers should decide the intended `MEDIA:` contract, then either update `src/media/parse.ts` and `src/media/parse.test.ts` to extract valid own-line fenced directives through the existing validation path, or preserve the skip behavior while adding an observable warning and updating the assistant prompt plus public docs. Closed PR #41980 is useful prior art but should not close this issue before a maintained fix lands.
