## Summary

Keep open. Current main still lacks `--daemon-env` support for onboard daemon installation, so the PR is not obsolete; however, the proposed change touches the persistent daemon service environment boundary and needs maintainer/security review before it is a safe merge candidate.

## What This Changes

The PR adds a repeatable `--daemon-env KEY=VALUE` onboard option, threads it through onboarding options, parses entries, and merges them into gateway daemon service environments for non-interactive and interactive install paths.

## Best Possible Solution

Add a reviewed, narrow daemon-env mechanism that normalizes portable keys, rejects dangerous host-env and override names through the existing policy helpers, gives clear diagnostics for malformed input, and documents the supported bootstrap use case.
