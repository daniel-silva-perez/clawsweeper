## Summary

Close as duplicate/superseded by the open session-reset canonical tracker. Current main still has the reported implicit daily 4 AM behavior, but open #61238 already covers the same silent daily reset/no warning/no discoverable opt-out problem, with #57369 tracking the narrower manual-only reset mode.

## What This Changes

The issue asks OpenClaw to stop treating an unset `session.reset` as an automatic daily 4 AM reset, or to document and surface that implicit default clearly.

## Best Possible Solution

Keep #61238 as the canonical product/UX tracker for silent implicit daily reset behavior and use #57369 for the narrow manual-only reset mode; close this duplicate while preserving its precise code evidence through the close comment.
