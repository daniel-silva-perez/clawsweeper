## Summary

Keep open. The branch is still useful because current main does not provide the proposed first-render locale readiness path, but the submitted patch has two blocking correctness issues and needs to be updated against current main's sanitizer split before it can merge.

## What This Changes

The PR adds awaitable Control UI i18n startup readiness, gates initial rendering on locale loading, and adds tool_use/tool_result mismatch recovery messaging plus focused tests.

## Best Possible Solution

Ship a focused update or replacement that makes startup locale loading awaitable, applies persisted or navigator locale first, then applies `settings.locale` as the authoritative override before the first render. Keep the tool mismatch handling narrow: preserve current transcript repair, add a fallback only for real tool_result/tool_use pairing rejections, and cover false positives in tests.

## Reproduction Assessment

Yes for the PR-introduced defects: source inspection shows the regex matches any string containing `tool_use_id`, and the `Promise.all([i18n.ready, settingsLocale])` approach allows locale setters to race when sources disagree. The underlying i18n flash is also plausible on current main because render is not gated while non-English locale loading is fire-and-forget.

## Solution Assessment

No. The direction is maintainable, but the submitted implementation should serialize locale precedence, narrow the mismatch classifier, and be rebased onto current main's separated sanitizer module before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Tighten the tool_use mismatch regex:** `src/agents/pi-embedded-helpers/errors.ts:839`
  - body: The standalone `tool_use_id` branch matches unrelated provider/schema errors that merely mention that field, so users can be told their session is corrupted and to run `/new` when the real error is different. Require tool-result pairing context such as missing, not-found, or corresponding `tool_use` before rewriting the message.
  - confidence: 0.88
- **[P2] Apply settings locale after i18n readiness:** `ui/src/ui/app.ts:119-133`
  - body: The patch starts initial locale loading and the settings locale load concurrently. If browser storage or navigator and app settings disagree, the slower promise wins and can render the app in the wrong language; wait for `i18n.ready`, then apply `settings.locale` before setting `localeReady`.
  - confidence: 0.84
