## Summary

Keep open. The cosine-similarity bug is still present on current main and in the latest release, but this PR edits the old `src/memory/internal.ts` path while the active scorer now lives in the memory-host SDK package; the existing ClawSweeper comment correctly identified this as useful source work that needs a port/replacement before merge.

## What This Changes

The PR adds trailing-dimension norm accumulation to `src/memory/internal.ts` and extends `src/memory/internal.test.ts` with cosine similarity regression coverage.

## Best Possible Solution

Port the zero-padding cosine norm semantics to `packages/memory-host-sdk/src/host/internal.ts`, add focused regression coverage in `packages/memory-host-sdk/src/host/internal.test.ts`, and add a fallback-search regression in `extensions/memory-core/src/memory/manager-search.test.ts` if practical so the product path cannot drift.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: current main returns an inflated score for `cosineSimilarity([1, 1], [1])` because only the shared prefix contributes to `normA`, while zero-padding semantics should return `1 / Math.sqrt(2)`.

## Solution Assessment

No. The PR's algorithm is the right mathematical fix, but the patch is aimed at a removed path; the narrow maintainable solution is to apply the same semantics to the active memory-host SDK helper and tests.
