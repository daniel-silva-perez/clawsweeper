## Summary

Keep open. The proposed cosine math is correct and useful, but this PR edits the removed `src/memory/internal.ts` path while current main still uses the active helper in `packages/memory-host-sdk/src/host/internal.ts`; the active fallback memory scoring path remains affected.

## What This Changes

The PR adds trailing-dimension norm accumulation and regression tests for `cosineSimilarity` in the old `src/memory/internal.ts` implementation.

## Best Possible Solution

Port the zero-padding norm semantics to `packages/memory-host-sdk/src/host/internal.ts`, add direct helper regression coverage in `packages/memory-host-sdk/src/host/internal.test.ts`, and add a focused fallback-search regression if practical without restoring the removed `src/memory/internal.ts` path.

## Reproduction Assessment

Yes. By inspection, `cosineSimilarity([1, 1], [1])` on the active helper accumulates `normA` only over the shared prefix and returns `1` instead of `1 / Math.sqrt(2)`.

## Solution Assessment

No. The proposed math is the right repair, but applying it to the removed `src/memory/internal.ts` path does not fix current main; the narrow maintainable solution is to port the same semantics to the active memory-host SDK helper and tests.
