## Summary

Keep open. The underlying cosine similarity bug is still present on current main, but this PR is not merge-ready because it updates the old `src/memory/internal.ts` path while the active fallback search now uses the memory-host SDK helper, which still computes both vector norms only over the shared prefix.

## What This Changes

The branch adds trailing-dimension norm accumulation to `src/memory/internal.ts` and expands `src/memory/internal.test.ts` with cosine similarity regression coverage.

## Best Possible Solution

Port the zero-padding cosine norm semantics to `packages/memory-host-sdk/src/host/internal.ts`, add focused regression coverage in `packages/memory-host-sdk/src/host/internal.test.ts`, and add a fallback-search regression in `extensions/memory-core/src/memory/manager-search.test.ts` if practical so the product path cannot drift.

## Reproduction Assessment

Yes. Code inspection gives a high-confidence reproduction: current main returns an inflated score for inputs like `cosineSimilarity([1, 1], [1])` because only the shared prefix contributes to `normA`, while zero-padding semantics should include the trailing dimension and return `1 / Math.sqrt(2)`.

## Solution Assessment

No. The PR's math is the right fix, but applying it to `src/memory/internal.ts` is no longer the narrow maintainable solution because that path is absent on current main; the fix needs to land in the active memory-host SDK helper and tests.
