## Summary

Current main resolves the observable Mattermost problem by removing the regular inbound-message system-event preview that used to flatten Markdown with `/\s+/g`. Mattermost user posts now go directly through the inbound reply context with `BodyForAgent` and `RawBody` preserving the normalized message body, and a focused test asserts regular user posts do not enqueue system events. The fix is on current main only; no tag contains the proof commit yet.

## Best Possible Solution

Close this as implemented on current main. Keep the current behavior that regular Mattermost user posts do not enqueue a compact system-event preview; if a separate debug/log preview mode is still desired, it should be opened as a new request against the current Mattermost logging or diagnostics surface.
