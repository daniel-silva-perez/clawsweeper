## Summary

Keep this PR open. Current main still lacks the delayed Feishu DM acknowledgment, but the proposed patch needs changes before merge because it can send the acknowledgment after visible output has already started, and one added fast-path test can hang under fake timers.

## What This Changes

The PR adds an 8-second delayed Feishu p2p/private-message acknowledgment timer in the Feishu bot dispatch path and bot tests for slow, fast, and group-chat behavior.

## Best Possible Solution

Land a Feishu-local revision that sends the delayed acknowledgment only for p2p/private chats after a real no-visible-output interval, suppresses it as soon as any visible Feishu reply lifecycle starts, preserves the normal final reply flow, and covers slow DM, fast DM, group chat, and early-visible-reply cancellation deterministically.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Cancel delayed ack when visible replies start:** `extensions/feishu/src/bot.ts:1833-1848`
  - body: The timer is only canceled after `withReplyDispatcher` settles, but Feishu can start streaming/card/plain-text/media output before settlement. A long run that already produced visible output can still send the delayed acknowledgment after 8 seconds, violating the PR's no-user-visible-reply condition and creating duplicate messages.
  - confidence: 0.9
- **[P2] Use a fast dispatcher in the fast-path test:** `extensions/feishu/src/bot.test.ts:537`
  - body: The fast direct-message case advances fake timers by only `7_999` ms and then awaits `pending`, but the shared dispatch mock resolves after `9_000` ms. Under fake timers, that promise cannot settle there, so the test can hang instead of validating cancellation before the ack threshold.
  - confidence: 0.88
