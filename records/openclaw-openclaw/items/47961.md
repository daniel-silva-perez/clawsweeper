## Summary

Keep this PR open. Current main still lacks the Feishu delayed-ack behavior, and the PR is a useful localized implementation candidate, but the branch still needs changes before merge: the timer only cancels when dispatch settles, so it can send a duplicate acknowledgment after an early visible reply, and the fast-dispatch test still awaits a 9-second mock after advancing fake timers by only 7.999 seconds.

## What This Changes

The PR adds a Feishu direct-message delayed-acknowledgment timer in `extensions/feishu/src/bot.ts` and adds slow, fast, and group-chat guard tests in `extensions/feishu/src/bot.test.ts`.

## Best Possible Solution

Land a revised Feishu-local implementation that sends the delayed acknowledgment only for p2p/private chats after a true slow no-output interval, cancels as soon as any user-visible reply starts or is delivered, and includes deterministic coverage for slow DM, fast DM, group chat, and early-visible-reply paths before maintainer review.
