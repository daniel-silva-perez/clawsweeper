## Summary

Keep this PR open: current main does not implement the requested Feishu delayed acknowledgment, and the PR branch is a useful narrow implementation candidate but still has merge-blocking correctness defects. The timer is canceled only after dispatcher settlement even though Feishu can start visible streaming/card/text/media replies earlier, and one added fast-path test awaits a 9s fake-timer dispatch after advancing only 7.999s.

## What This Changes

The PR adds an 8-second delayed acknowledgment timer for Feishu p2p/private message dispatches and tests slow, fast, and group-chat behavior.

## Best Possible Solution

Land a Feishu-local revision that sends the delayed acknowledgment only for p2p/private chats after a real no-visible-output interval, cancels or suppresses it as soon as any user-visible Feishu reply starts or is delivered, preserves the final reply flow, and covers slow DM, fast DM, group chat, and early-visible-reply cancellation deterministically.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

- **[P2] Cancel delayed ack when visible replies start:** `extensions/feishu/src/bot.ts:1833-1848`
  - body: The timer is only canceled after `withReplyDispatcher` settles, but Feishu can start streaming/card/plain-text/media output before settlement. A long run that already produced visible output can still send the delayed acknowledgment after 8s, which violates the PR's no-user-visible-reply condition and creates duplicate/confusing messages.
  - confidence: 0.9

  - confidence: 0.88
