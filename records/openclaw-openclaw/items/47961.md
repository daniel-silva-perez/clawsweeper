## Summary

Keep this PR open. Current main still lacks the delayed Feishu direct-message acknowledgment behavior, but the proposed patch has two concrete P2 correctness defects that should be fixed before merge.

## What This Changes

The PR adds an 8-second delayed Feishu p2p/private-message acknowledgment timer in the Feishu bot dispatch path and adds bot tests for slow, fast, and group-chat behavior.

## Best Possible Solution

Land a Feishu-local revision that sends the delayed acknowledgment only for p2p/private chats after a real no-visible-output interval, suppresses or cancels it as soon as any user-visible Feishu reply starts or is delivered, preserves the normal final reply flow, and covers slow DM, fast DM, group chat, and early-visible-reply cancellation deterministically.

## Reproduction Assessment

Yes. The review defects reproduce by inspection with a long Feishu DM run that starts visible output before 8 seconds but settles after 8 seconds, and by the fake-timer fast-path test awaiting a 9,000 ms mock after only 7,999 ms.

## Solution Assessment

No. The PR is appropriately Feishu-local, but the timer needs a visible-output cancellation/suppression hook and the fast-path test needs a dispatcher that actually settles before the acknowledgment threshold.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Cancel delayed ack once visible output starts:** `extensions/feishu/src/bot.ts:1833-1848`
  - body: The timer is only canceled after `withReplyDispatcher` settles, but Feishu can start streaming/card/plain-text/media output before settlement. A long run that already produced visible output can still send the extra acknowledgment after 8 seconds, violating the PR body's no-user-visible-reply condition and creating duplicate/confusing messages.
  - confidence: 0.9
- **[P2] Make the fast-path delayed-ack test settle:** `extensions/feishu/src/bot.test.ts:537`
  - body: The fast direct-message case advances fake timers by only 7,999 ms and then awaits `pending`, while the shared dispatch mock resolves after 9,000 ms. Under fake timers that promise cannot settle there, so the test can hang instead of proving the acknowledgment was canceled before the threshold.
  - confidence: 0.88
