## Summary

Keep this PR open. Current main still lacks the delayed Feishu direct-message acknowledgment behavior, and the proposed branch is useful but not ready to merge because it cancels only when dispatch settles rather than when user-visible output begins, plus one fake-timer test can hang instead of proving the fast path.

## What This Changes

The PR adds an 8-second delayed Feishu p2p/private-message acknowledgment timer in the Feishu bot dispatch path and adds Feishu bot tests for slow, fast, and group-chat delayed-ack behavior.

## Best Possible Solution

Land a Feishu-local revision that sends the delayed acknowledgment only for p2p/private chats after a real no-visible-output interval, cancels as soon as any user-visible reply starts or is delivered, preserves normal final reply flow, and includes deterministic coverage for slow DM, fast DM, group chat, and early-visible-reply cancellation.
