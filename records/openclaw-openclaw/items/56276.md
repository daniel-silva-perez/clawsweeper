## Summary

Keep open. Current main does not implement the requested hostname in the TUI footer/status bar; the adjacent header already shows the connection URL, but the footer renderer and docs still omit host context. This looks like a narrow, valid TUI enhancement rather than stale cleanup.

## What This Changes

The issue requests adding the current host name to the TUI status bar/footer so users can distinguish multiple OpenClaw hosts, especially over Tailscale.

## Best Possible Solution

Add a compact TUI-only host label to the footer after the agent label, deriving it from the resolved gateway URL hostname where possible and from a safe local hostname helper for embedded mode, without exposing URL credentials or changing gateway protocol.

## Reproduction Assessment

Yes. Current-main inspection of `updateFooter` is a high-confidence reproduction path because the rendered footer parts contain no host field, and the public TUI docs list no hostname in the footer.

## Solution Assessment

Yes. A small renderer/helper change in the TUI is the narrowest maintainable fix; adding a gateway protocol field would be unnecessary unless maintainers specifically want server-reported host identity instead of the resolved connection/local hostname.
