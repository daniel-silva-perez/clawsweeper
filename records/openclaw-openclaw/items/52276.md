## Summary

Keep open. Current main still suppresses embedded block replies by text only, so the PR is not redundant, but the PR head is not mergeable as-is: it calls an undefined routed-provider resolver and still loses Telegram topic parity on the origin side.

## What This Changes

The PR makes embedded messaging-tool duplicate suppression route-aware by threading provider, target, account, and thread context through runner/subscriber paths and widening message-send target extraction.

## Best Possible Solution

Land a focused repair or replacement that keeps the route-scoped dedupe goal, defines the routed-provider resolver, carries origin provider/target/account/thread context through the embedded subscription boundary, and aligns message-send target tracking with outbound delivery normalization before suppressing final block replies.
