## Summary

Keep open. Current main still has the text-only duplicate-suppression behavior this PR is trying to replace, but the PR head is not mergeable as-is: it calls an undefined routed-provider helper and still omits the origin thread from Telegram topic suppression matching.

## What This Changes

The PR makes embedded messaging-tool duplicate suppression route-aware by threading provider, target, account, and current-route context through runner/subscriber paths and expanding message-send target extraction tests.

## Best Possible Solution

Land a corrected version that keeps the route-scoped dedupe behavior, defines or imports the routed-provider resolver, carries origin thread metadata through the embedded subscription boundary, and proves same-route plus different-route behavior for normal and Telegram topic sessions.
