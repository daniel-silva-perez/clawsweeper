## Summary

Keep open. Current main still has the text-only embedded reply suppression behavior that this PR is trying to replace, and the route-aware helper is not wired into the subscriber or message-end paths. The PR remains useful source work, but it should not merge as-is because the review context identifies unresolved functional breakage in the branch, including an undefined route-provider resolver and remaining route/thread mismatches.

## What This Changes

The PR changes embedded messaging types, runner/subscriber params, block-reply dedupe checks, message-tool target extraction, and tests so duplicate suppression can be scoped by provider, target, account, and thread instead of normalized text alone.

## Best Possible Solution

Land a focused repair or replacement that carries route provider, target, account, and thread context through the embedded subscription boundary, applies `shouldSuppressMessagingToolReplies` in both streamed and message-end block-reply suppression with a backward-compatible fallback when route context is absent, and keeps message-tool target tracking aligned with outbound send normalization.
