## Summary

Keep this PR open, but it still needs changes before merge. Current main still has text-only embedded messaging-tool duplicate suppression, so the intended route-scoped fix remains useful; however the PR head has a blocking undefined resolver and still loses Telegram topic context in suppression matching.

## What This Changes

The PR threads route/provider/target context through embedded runner and subscriber duplicate suppression, widens messaging-tool send target extraction, and updates route-scoped suppression tests.

## Best Possible Solution

Land a corrected route-scoped dedupe implementation that wires the existing suppression helper into streamed and message_end block-reply paths, derives the real route provider for synthetic automation providers, preserves providerless same-route sends, and carries origin target/account/thread metadata consistently through the embedded subscription boundary.
