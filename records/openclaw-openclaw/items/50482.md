## Summary

Keep open. Current main still preserves Telegram voice notes only as generic audio/transcript context, so agents cannot distinguish a voice note from typed text or ordinary audio. The broader open Telegram metadata request in #40838 is useful naming context, but it does not supersede this narrower request.

## What This Changes

Expose an agent-visible inbound metadata marker that identifies when a message originated as a Telegram voice note rather than typed text or generic audio.

## Best Possible Solution

Add a backwards-compatible generic inbound message-origin field to the shared context, populate it from Telegram before voice is normalized to audio, render it in the agent-visible inbound metadata, and document the public field with focused regression coverage for voice notes versus ordinary audio and text.
