## Summary

Keep open. Current main still turns Telegram voice notes into generic audio/transcript context before agent-visible metadata is rendered, so agents cannot distinguish a dictated voice note from ordinary audio or typed text. The broader open Telegram metadata request in #40838 is related but does not supersede this narrower voice-origin request.

## What This Changes

Expose a voice-origin or generic message-type marker in inbound agent context so agents can distinguish Telegram voice notes after STT from typed text and ordinary audio files.

## Best Possible Solution

Add a backwards-compatible generic inbound message-origin or modality field to the shared context, populate it from Telegram before msg.voice is normalized to generic audio, render it in agent-visible conversation metadata, and document the field with tests proving voice notes differ from ordinary audio and text.
