## Summary

Keep open. Current main has provider catalog and model-list plumbing, but it does not implement configured OpenAI-compatible `/v1/models` refresh for the OpenAI Control UI picker. Existing Bedrock Mantle, vLLM, and Hugging Face discovery paths are useful precedents, while related #74427 and merged #55597 only cover base-URL routing precedents, not dynamic picker/catalog refresh.

## What This Changes

The issue requests an opt-in provider catalog refresh path that fetches configured provider `/v1/models` data, merges or replaces static catalog rows, and updates the Control UI model picker.

## Best Possible Solution

Decide the provider-catalog contract first, preferably by reusing or deliberately extending the existing `catalog` and model-cache architecture, then implement configured OpenAI-compatible discovery with endpoint/auth scoped caching, clear merge/replace semantics, Gateway/Control UI refresh behavior, tests, and docs.
