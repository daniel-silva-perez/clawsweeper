## Summary

Keep this PR open. Current main still does not strip echoed inbound metadata in the shared outbound payload plan, the linked bug #39847 remains open, and the PR is not safely obsolete because its proposed use of `stripInboundMetadata()` can alter legitimate timestamp-prefixed assistant replies.

## Best Possible Solution

Revise and review this PR, or consolidate it with the canonical related fix, instead of closing it. The best implementation is a narrow outbound-safe sanitizer in the shared `createOutboundPayloadPlanEntry()` path that strips copied inbound metadata sentinel blocks before all delivery projections while preserving legitimate assistant output such as timestamp-prefixed logs or transcripts. Add focused tests for both the metadata leak and timestamp preservation, then land it against #39847 if that matches maintainer intent.
