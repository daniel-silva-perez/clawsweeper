## Summary

Keep this PR open. Current main still does not strip copied inbound metadata on the shared outbound payload plan, and the PR remains an implementation candidate for the open bug, but its proposed use of the broader inbound sanitizer can alter legitimate timestamp-prefixed assistant output.

## Best Possible Solution

Keep the PR open for maintainer review or consolidation with #50520/#39847. The best fix is a narrow outbound-safe sanitizer in the shared createOutboundPayloadPlanEntry() path that removes copied inbound metadata sentinel blocks before all delivery projections while preserving legitimate timestamp-prefixed assistant output, with regression tests for both stripping and timestamp preservation.
