## Summary

Keep this PR open. Current main still has the outbound metadata leak described by the PR and linked bug, while the PR’s current implementation remains unsafe to close as implemented because it applies the broader inbound sanitizer to every outbound reply and can strip legitimate timestamp-prefixed assistant output.

## Best Possible Solution

Revise and review this PR, or consolidate it with the canonical related fix, instead of closing it. The best implementation is a narrow outbound-safe sanitizer in the shared `createOutboundPayloadPlanEntry()` path that removes copied inbound metadata sentinel blocks before all delivery projections while preserving legitimate timestamp-prefixed assistant output. Add regression coverage for both metadata stripping and timestamp preservation, then land it against #39847 if that matches maintainer intent.
