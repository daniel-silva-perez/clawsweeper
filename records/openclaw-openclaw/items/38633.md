## Summary

Current main still has the reported mismatch: the public sandbox config example shows `pidsLimit: 256`, while runtime resolution, schema, and Docker argument emission leave `pidsLimit` unset unless the user configures it. Related runtime-default and docs-clarification PRs are closed unmerged, and the broader security issue remains open, so this narrower docs/config alignment item should stay open.

## Best Possible Solution

Keep this issue open until maintainers either clarify the docs that `pidsLimit: 256` is an explicit example and remains unset unless configured, or intentionally add a runtime default in `resolveSandboxDockerConfig` with matching schema/type wording, Docker argument tests, and docs. Coordinate any runtime-default decision with #38604; a docs-only fix can stay narrow, similar to the closed unmerged #38914 direction.
