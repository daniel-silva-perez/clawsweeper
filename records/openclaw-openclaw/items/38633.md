## Summary

Keep this issue open. Current main still shows `pidsLimit: 256` in the sandbox Docker config example while runtime resolution leaves `pidsLimit` unset unless the user configures it, and Docker only emits `--pids-limit` for a positive configured value. The related runtime-default PR #38621 and docs-clarification PR #38914 are closed unmerged, while the broader security issue #38604 remains open and does not fully replace this narrower docs mismatch.

## Best Possible Solution

Keep this issue open until maintainers either clarify the docs that `pidsLimit: 256` is an explicit example and remains unset unless configured, or intentionally add a runtime default in `resolveSandboxDockerConfig` with matching schema/type wording, tests, and docs. Coordinate any runtime-default decision with #38604; a docs-only fix can stay narrow.
