## Summary

Keep this PR open. Current main still does not implement the PR’s intended schema change: `agents.list[]` entries remain strict and still omit `verboseDefault` and `elevatedDefault` from the runtime and generated schemas. The linked issue #73680 is the right bug context, and this PR is an implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or revise it as the implementation for #73680, ensuring the strict schema, generated schema, TypeScript config type, docs, and runtime resolver behavior all agree for per-agent verbose and elevated defaults.
