## Summary

Keep open. Current main still has adjacent protections, but no durable per-session pending-deliverables queue for arbitrary promised chat outputs after task switching without a gateway restart. The shipped queues cover inbound follow-up messages, task completion notifications, restart recovery, or already-materialized outbound payloads, not unfinished commitments the agent promised to deliver later.

## Best Possible Solution

Keep the issue open for a core session-continuity fix. The best path is a typed durable per-session pending-deliverable record, likely integrated with Task Flow or the task ledger, written before preemptive task switches and cleared only when the promised output is sent, explicitly deferred, cancelled, or superseded. Add regression coverage for a direct chat long-running task, an unrelated high-priority interruption without gateway restart, and automatic resumption or visible deferral of the original promised output.
