## Summary

Keep this PR open. Current main still defaults empty TUI/WebChat session selection to the agent main session and does not expose `gateway.webchat.defaultPeerId` in config types, validation, generated config metadata, or WebChat docs. The PR is a plausible focused implementation path, but it needs normal review/rebase rather than cleanup closure.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to rebase it onto current main, wire `gateway.webchat.defaultPeerId` through the typed config/schema/generated metadata/docs, resolve empty TUI/WebChat sessions via `buildAgentPeerSessionKey` with `session.dmScope` and `session.identityLinks`, preserve explicit session and global-scope behavior, and keep focused unit coverage for peer-default and legacy fallback cases.
