## Summary

Keep open. Current main still lacks stable cron add ids, and the PR addresses most of the requested surfaces, but the latest diff still makes the generated Swift `CronAddParams` initializer source-breaking by adding an optional wire field as a required initializer argument. That should be fixed before merge.

## What This Changes

The PR adds optional stable cron job IDs to cron creation, duplicate-id validation, gateway schema, CLI and agent-tool inputs, docs, tests, and generated Swift protocol models.

## Best Possible Solution

Land the stable cron-id feature after the generated Swift protocol surface remains source-compatible, preferably by updating the Swift generator to default optional initializer parameters to `nil` and regenerating both Swift model copies.
