## Summary

Keep open. Current main still lacks stable cron job IDs, so this PR is not obsolete; however, the latest PR head still makes optional `CronAddParams.id` a required Swift initializer parameter in generated client models, which is a source-compatibility regression for existing Swift callers.

## What This Changes

This PR adds optional stable cron job IDs across cron creation, duplicate-id validation, gateway/CLI/agent tool inputs, docs, tests, and generated Swift gateway models.

## Best Possible Solution

Land the stable cron-id feature after the Swift generator defaults optional initializer parameters to `nil` and both generated Swift model copies are regenerated, preserving the additive wire contract without breaking existing Swift callers.
