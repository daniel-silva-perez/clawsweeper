## Summary

Keep open. Current main does not show a fix for the reported Docker `node dist/index.js` startup latency: official Docker compose/docs/setup paths still invoke `dist/index.js`, the direct `src/index.ts` main path still goes through the legacy CLI loader rather than the lean `openclaw.mjs`/`src/entry.ts` startup path, and the checked-in startup benchmark/budget coverage defaults to `openclaw.mjs`. Earlier identical reports were closed as duplicates rather than as implemented, so this issue is the active tracker.

## What This Changes

The issue requests reducing the 24s-49s startup latency for `node dist/index.js` subcommands inside the official Docker image.

## Best Possible Solution

Keep this issue open and land a focused fix that either makes direct packaged `dist/index.js` CLI execution use the same lean startup path as `openclaw.mjs`, or removes `dist/index.js` from official Docker CLI workflows while preserving package-root import behavior. Add startup benchmark coverage for `dist/index.js` so the reported path stays guarded.

## Reproduction Assessment

Yes. The issue provides a high-confidence Docker reproduction with exact `docker compose exec -T openclaw-gateway node dist/index.js ...` commands and timings, and current main still exposes that path in compose/docs/setup code.

## Solution Assessment

Unclear because there is no proposed patch yet. The safest direction is a narrow CLI/Docker startup fix that preserves `dist/index.js` as the package export for imports while making official or direct CLI execution avoid the legacy slow path.
