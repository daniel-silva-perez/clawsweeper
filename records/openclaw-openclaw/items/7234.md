## Summary

Keep open. Current main still models Discord `actions.messages` as one coarse boolean and still uses that same gate for fetch/read/send/edit/delete Discord message actions. The requested backward-compatible granular send/read/edit/delete gates are missing from config types, validation/schema metadata, discovery, runtime checks, docs, and tests.

## Best Possible Solution

Keep this open. Implement a backward-compatible Discord message gate shape such as `channels.discord.actions.messages: boolean | { send?: boolean; read?: boolean; edit?: boolean; delete?: boolean }`, or equivalent separate top-level gates, then update `DiscordActionConfig`, the strict Zod schema, generated config metadata, Discord discovery, runtime checks, docs, and focused Discord tests while preserving existing boolean behavior.
