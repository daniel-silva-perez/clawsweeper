## Summary

Keep open. Current main still advertises Amazon Bedrock thinking levels as `off`, `minimal`, `low`, `medium`, `high`, plus `adaptive` only for Claude 4.6 matches; it does not expose `xhigh` for Bedrock Claude Opus 4.7. Because the runtime accepts a non-empty plugin profile before consulting legacy xhigh hooks, the reported `/think` autocomplete and validation gap remains. The already-open #70821 targets the same Bedrock profile/test gap, so this should stay open pending review or landing of that PR rather than being closed or duplicated.

## What This Changes

The issue asks for the Amazon Bedrock provider thinking profile to advertise `xhigh` for Claude Opus 4.7 and stay aligned with Claude 4.6 adaptive thinking behavior so `/think` autocomplete and validation match the transport support.

## Best Possible Solution

Land or revise the focused Bedrock provider fix so `amazon-bedrock/*claude-opus-4-7*` profiles include `xhigh`, with provider tests for regional/dotted Opus 4.7 ids and negative coverage for non-Opus-4.7 models. The implementation should account for Bedrock-prefixed model ids rather than blindly calling a helper that only matches direct Claude id prefixes.
