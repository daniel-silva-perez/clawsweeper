## Summary

Keep this PR open. Current main still has the DeepSeek V4 payload wrapper that maps `xhigh`/`max` after request construction, but the provider does not expose a `resolveThinkingProfile`, so explicit `/think xhigh` and `/think max` remain rejected before the wrapper can run. The PR is a focused implementation candidate for the linked DeepSeek issue, with matching docs and regression coverage, not obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land the provider-owned `resolveThinkingProfile` for DeepSeek V4, keep the regression test that proves V4 exposes `xhigh|max` while non-V4 DeepSeek models do not, update the DeepSeek/thinking docs, and optionally deduplicate the V4 model-id helper before merge. After merge, the linked #72893 can close as implemented.
