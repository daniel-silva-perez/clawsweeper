## Summary

Keep this PR open. Current main still has DeepSeek V4 request shaping for `xhigh`/`max`, but the DeepSeek provider plugin does not expose a `resolveThinkingProfile`, so `/think xhigh` and `/think max` remain rejected before request construction. The PR is a focused implementation candidate for the linked DeepSeek issue, not obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land the provider-owned DeepSeek V4 `resolveThinkingProfile` with `off|minimal|low|medium|high|xhigh|max`, keep the regression test proving V4-only exposure, update the DeepSeek/thinking docs, optionally deduplicate the V4 model-id helper, and then close #72893 after merge.
