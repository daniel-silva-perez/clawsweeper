## Summary

Current main still lacks the DeepSeek provider `resolveThinkingProfile` hook that this PR adds. DeepSeek V4 request wrapping already maps `xhigh` and `max` to DeepSeek `reasoning_effort: "max"`, but command validation resolves supported thinking levels from the provider profile first, and the current fallback profile excludes those levels. The PR remains a focused implementation candidate for #72893; the security pass found no CI, dependency, lockfile, release, or secret-handling changes in the provided diff.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land the provider-owned DeepSeek V4 `resolveThinkingProfile`, keep the regression coverage proving V4 exposes `xhigh|max` while non-V4 DeepSeek models do not, update the DeepSeek/thinking docs, and optionally deduplicate the V4 model-id helper before merge. After merge, #72893 can close as implemented.
