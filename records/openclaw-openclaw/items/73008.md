## Summary

Current main still does not implement this PR's intended DeepSeek V4 thinking-profile change. The provider can already patch DeepSeek V4 payloads so `xhigh`/`max` map to `reasoning_effort: "max"`, but the DeepSeek provider does not expose `resolveThinkingProfile`, so validation falls back to base levels and rejects those explicit `/think` choices before request construction. The PR remains a focused implementation candidate for #72893/#73718, with no security-sensitive files in the provided diff.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a DeepSeek-owned `resolveThinkingProfile` for `deepseek-v4-flash` and `deepseek-v4-pro`, keep regression coverage proving V4 exposes `xhigh|max` while older DeepSeek models do not, and ship the DeepSeek/thinking docs updates. After merge, #72893 and #73718 can close as implemented.
