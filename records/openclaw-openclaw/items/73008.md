## Summary

Keep open: current main still lacks the DeepSeek provider thinking profile needed for `/think xhigh` and `/think max`, while this PR is a focused implementation for that valid bug and is paired to the open linked issue. The latest PR head is open, clean, and CI is green, but it has no human review yet, so the next step is normal maintainer review and merge rather than cleanup closure.

## What This Changes

The PR adds a DeepSeek V4 `resolveThinkingProfile` and shared V4 model matcher, updates DeepSeek/thinking docs, and adds regression coverage that V4 models expose `xhigh` and `max` while non-V4 DeepSeek models do not.

## Best Possible Solution

Land a provider-owned DeepSeek V4 `resolveThinkingProfile` that advertises `off|minimal|low|medium|high|xhigh|max`, keep the shared local V4 matcher, preserve positive V4 and negative non-V4 regression coverage, update the public DeepSeek/thinking docs, and then close the related DeepSeek reports as implemented.
