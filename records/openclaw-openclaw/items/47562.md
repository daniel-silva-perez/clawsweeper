## Summary

Keep this PR open. Current main still only supports standard Telegram emoji reactions: outbound reaction sending has no customEmojiId option, the Telegram action runtime and shared message-tool schema expose only emoji/remove, and inbound message_reaction handling filters only ReactionTypeEmoji. The PR is directionally useful, but it is stale against the current Telegram runtime split and still needs customEmojiId-only schema/validation coverage before it can land.

## Best Possible Solution

Keep this PR open as a real implementation candidate.
