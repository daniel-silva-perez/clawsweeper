Keep this PR open. Current main still lacks Telegram custom emoji reaction support, and the PR is useful source work, but it is not merge-ready because it targets stale Telegram files and still rejects the advertised custom-only `customEmojiId` action flow before the send helper can use it.

## What This Changes

The PR adds inbound Telegram custom emoji reaction events, outbound `customEmojiId` reaction payload support, Telegram action plumbing, and bot/send tests.

## Best Possible Solution
