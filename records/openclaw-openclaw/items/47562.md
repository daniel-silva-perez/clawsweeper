## Summary

Keep open. Current main still lacks Telegram custom emoji reaction support, and this PR is useful source work, but the branch is not merge-ready because it targets obsolete files on current main and still has an action-layer validation bug for custom-only reaction calls.

## What This Changes

The PR adds Telegram custom emoji reaction support for inbound `message_reaction` updates and outbound reaction sends/actions, plus bot and send tests.

## Best Possible Solution

Port the narrow reaction feature into the current Telegram plugin surfaces: add `customEmojiId` to the shared message-tool schema and Telegram action runtime, send Telegram `custom_emoji` reaction payloads from `reactMessageTelegram`, emit inbound system events for added `custom_emoji` reactions after existing authorization and notification gates, document the behavior, and cover send/action/inbound/schema paths with focused tests.
