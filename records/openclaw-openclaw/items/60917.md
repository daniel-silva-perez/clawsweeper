## Summary

Keep open. The linked fix #60971 was closed unmerged, and current main still serializes the full form-mode config payload without a raw-aware helper to drop stale `__OPENCLAW_REDACTED__` values that are absent from the original config. The gateway still rejects a sensitive-path sentinel when no original value exists, which matches the reported failure shape.

## Best Possible Solution

Keep this issue open and either rework/land the #60971 approach or add an equivalent UI-side fix in the config form save path: drop redacted placeholders that were not present in the original raw/source config, preserve existing real redacted secret paths, and add regression tests around form-mode saves with absent `gateway.remote.token`.
