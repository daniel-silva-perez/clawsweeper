## Summary

Keep this PR open. Current main still lacks an Agent Computer hosting guide, but the PR is not merge-ready because the recommended shortcut currently returns 404, the CLI install command points at the current npm SDK package rather than the Agent Computer CLI, and the port-publish examples use flags that are not present in the current public CLI source.

## What This Changes

The PR adds a new Agent Computer hosting guide and wires it into OpenClaw docs navigation, redirects, FAQ links, hosting provider lists, and the zh-CN glossary.

## Best Possible Solution

Keep the docs-only contribution, but update it to match the live Agent Computer contract: use a verified working OpenClaw shortcut or remove that path, install the CLI through Agent Computer's supported installer or another verified distribution, and document only supported `computer ports publish` flags before merging the navigation entries.

## Reproduction Assessment

Yes. The main review issues reproduce with read-only checks: `curl -I -L https://agentcomputer.ai/openclaw` returns 404 after redirect, the latest npm `aicomputer` tarball has no CLI bin, and the public CLI source lacks the documented port flags.

## Solution Assessment

No, not as submitted. A docs-only hosting guide is the right shape, but the current commands and shortcut need to be corrected against Agent Computer's live docs/source before this is the narrow maintainable fix.
