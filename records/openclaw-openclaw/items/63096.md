## Summary

Keep this PR open. Current main still does not implement the intended completion-wake prompt-context reconstruction: the gateway agent path forwards `request.extraSystemPrompt` unchanged, while subagent completion wakes call that path with `inputProvenance.kind: "inter_session"`, `task_completion` events, and no explicit prompt context. Normal chat still builds inbound/group context separately, so the PR remains a relevant implementation candidate, but it needs refresh against current inbound metadata and group-context behavior before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate for the inter-session `task_completion` completion-wake prompt-context gap. Refresh it onto current main, align synthesized context and tests with `openclaw.inbound_meta.v2` and current group-context behavior, preserve explicit `extraSystemPrompt` pass-through, synthesize persisted inbound/group context only when existing session delivery metadata supports it, and run targeted gateway tests plus `pnpm check:changed` before merge. Keep #63030 open for the broader heartbeat/cache-digest drift unless the PR grows to cover that path too.

## Risks / Open Questions

- The PR is not merge-ready as-is against current main: its tests still expect legacy `chat_id` system metadata, but current `openclaw.inbound_meta.v2` deliberately keeps conversation IDs out of the system prompt.

- The PR addresses the ACP/subagent completion-wake subset of #63030. The broader heartbeat/chat prompt-ordering drift remains separate unless this PR is expanded.
