## Summary

Keep this issue open. Current main still matches the Mattermost draft-preview streaming regression, and the broader channel-level agent-to-agent/bot-to-bot routing behavior is not implemented by the current `tools.agentToAgent` session-tool policy. The discussion is useful: the reporter identified the exact Mattermost config-propagation bug, and PR #73452 is an open implementation candidate for that narrow streaming part only.

## Best Possible Solution

Keep this issue open. Review PR #73452 for the narrow Mattermost draft-preview `blockStreaming` regression, preferably with a focused regression test around reply option propagation. Then either keep this issue open or split a follow-up for the broader channel-level agent-to-agent behavior: define whether OpenClaw should add Mattermost `allowBots`, generic configured-bot awareness, broadcast/fanout routing, or documentation clarifying that `tools.agentToAgent` only controls session tools.
