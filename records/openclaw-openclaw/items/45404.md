## Summary

Keep this issue open. Current main clearly uses `openclaw` as the canonical managed browser profile and preserves `clawd` only as a legacy driver/config compatibility spelling, but I did not find the requested doctor/repair detection or safe archive path for leftover `~/.openclaw/browser/clawd/user-data` residue.

## Best Possible Solution

Add a bundled browser doctor/repair check that detects `~/.openclaw/browser/clawd` or `~/.openclaw/browser/clawd/user-data`, confirms current managed browser routing uses `openclaw`, reports a clear warning, and offers an interactive safe archive/Trash move. Cover it with browser doctor/maintenance tests and document the canonical managed profile path in the browser docs.
