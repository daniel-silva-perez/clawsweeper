## Summary

Keep this PR open. Current main includes a doctor-based repair for malformed legacy cron IDs, but it still does not implement the PR's central health-snapshot fix: `getHealthSnapshot` has no snapshot-wide budget and still awaits account probes sequentially. The runtime cron load path also remains narrower than the PR's claimed load-time cron ID normalization, so the PR is stale but not obsolete.

## Best Possible Solution

Keep this PR open or replace it with a rebased, focused patch: implement bounded and parallel health snapshots in current `src/commands/health.ts`, preserve the existing doctor-based cron ID repair unless maintainers explicitly choose runtime load-time persistence, add regression tests for budget exhaustion and parallel account probes, and fix the bot review/changelog attribution before merge.
