## Summary

Keep this PR open. Current main still lacks the PR's central bounded, parallel live health-snapshot behavior; gateway maintenance and explicit live probes still call the same sequential account-probe path with only a per-probe timeout. The legacy cron-id repair is now covered through doctor, but current runtime load still does not synthesize or coerce malformed ids the way this PR proposed, so the PR is stale and needs maintainer direction rather than cleanup closure.

## Best Possible Solution

Keep this PR open or replace it with a rebased focused patch. The best path is to implement bounded parallel live health snapshots in current `src/commands/health.ts` and the gateway refresh path, add regression coverage for concurrent account probes and deadline exhaustion, and explicitly decide whether malformed cron id repair should stay doctor-only or also persist from runtime load.
