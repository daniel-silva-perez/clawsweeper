## Summary

Keep this PR open. Current main has a doctor-based repair for malformed persisted cron IDs, but it still does not implement the PR's central health-snapshot change: account probes remain sequential, there is no snapshot-wide deadline/budget in `getHealthSnapshot`, and the gateway health refresh path does not pass one. The cron half is also only partially superseded because current runtime load still performs narrow string-only identity normalization while durable repair lives under `openclaw doctor --fix`.

## Best Possible Solution

Keep this PR open or replace it with a rebased, focused patch. The best path is to implement bounded and parallel health snapshots in current `src/commands/health.ts` and the gateway refresh path, add regression tests for deadline exhaustion and concurrent account probes, and either preserve the current doctor-based cron ID repair or make an explicit maintainer-approved runtime-load repair decision. Before merge, fix the stale cron files, bot review cleanup, and changelog attribution.
