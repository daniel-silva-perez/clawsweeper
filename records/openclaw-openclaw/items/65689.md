## Summary

Keep this PR open. Current main still drops persisted OpenClawKit chat-history `errorMessage` values, so the PR addresses a real missing client path. The submitted diff needs revision before merge because its fallback renders any non-empty `errorMessage` without gating to true assistant error turns, which can revive the stale-background-error regression tracked by #13935/#40616.

## Best Possible Solution

Keep this PR open and request a small revision: preserve `errorMessage` on `OpenClawChatMessage`, carry it through copy/reconcile sites, render or synthesize fallback text only for assistant error turns with no visible text, and add regression coverage for both the desired history-refresh error case and stale-success/tool-only `errorMessage` cases. Android/native parity can be considered separately if maintainers want consistent behavior across every app chat client.
