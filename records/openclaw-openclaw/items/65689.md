## Summary

Keep this PR open. Current main still does not preserve `errorMessage` in `OpenClawChatMessage`, so the reported history-refresh gap remains real. The submitted change is directionally correct, but the fallback rendering must be gated to true assistant error turns because main explicitly allows successful assistant turns to carry stale/background `errorMessage` values.

## Best Possible Solution

Keep the PR open and request a narrow revision: preserve `errorMessage` on `OpenClawChatMessage`, pass it through all copy/reconcile/merge sites, and synthesize or render fallback text only when the message is an assistant error turn with no visible text. Add regression coverage for the intended history-refresh error case plus stale-success and tool-only `errorMessage` cases.

## Clownfish Work Prompt

Revise or replace the implementation for openclaw/openclaw#65689 on current main. The observable bug is that OpenClawKit chat history can lose detailed provider failures after a run error because `OpenClawChatMessage` does not preserve `errorMessage`, while the backend stores detailed provider error text in peer `errorMessage` rather than replayed `content`. Implement the fix narrowly in OpenClawKit: add `errorMessage: String?` to `OpenClawChatMessage` coding/init/encode/decode, pass it through all `OpenClawChatMessage` copy/reconcile/merge sites, and synthesize or render fallback text only when the message is an assistant error turn (`role == "assistant"`, `stopReason == "error"`) and has no visible text content. Do not display `errorMessage` for successful assistant turns, tool-only assistant turns, user messages, or tool-result messages. Add focused regression tests in `ChatViewModelTests` for history refresh after an own-run error and for stale-success/tool-only `errorMessage` cases. Do not change backend replay semantics, provider contracts, CI workflows, dependency sources, lockfiles, or package scripts. Because this is user-facing, add a single-line active changelog fix entry if the repo convention requires it, crediting the external contributor/reporter such as `Thanks @javierdici` and not Codex.
