## Summary

Keep open. Current main still does not implement Telegram inline-query UX: inline-query strings appear in allowed-update metadata and BotFather capability probing, but there is no `inline_query` or `chosen_inline_result` runtime handler, no `answerInlineQuery`/`InlineQueryResult` response path, and no config/docs/test surface for enabling or constraining the feature. The related implementation PR #51963 is closed and unmerged, so it is reference material rather than shipped behavior.

## What This Changes

Add Telegram inline query support so users can invoke the Telegram bot from any chat with `@botname query` and share inline results back into that chat.

## Best Possible Solution

Keep this as the canonical Telegram inline-query tracker. The desired end state is a reviewed implementation inside the bundled Telegram plugin with explicit config/schema/docs, conservative sender authorization and session semantics, bounded request handling, `InlineQueryResultArticle` responses, chosen-result handling if needed, and focused Telegram plugin tests; #51963 should be mined only as reference material.
