## Summary

Keep open. Current main has a trusted bundled-plugin tool-result middleware path that can run before the model sees tool output, but the public plugin hook surface still does not expose the requested synchronous `tool_result_before_model` hook or an equivalent external post-execution/pre-model mutation hook. The Agent14 comment is useful corroborating production need and a workaround, but it does not remove the SDK/API gap.

## Best Possible Solution

Keep this issue open until maintainers either land a documented, trust-gated public post-execution/pre-model tool-result mutation hook, or explicitly decide that this capability must remain bundled-only. A complete design should define ordering relative to `before_tool_call`, bundled `registerAgentToolResultMiddleware`, `after_tool_call`, `tool_result_persist`, and `before_message_write`; specify sync/async and failure semantics; coordinate with #62123 and #72733; update SDK docs/types; and add runtime tests proving the model receives only the sanitized result.

## Risks / Open Questions
