## Summary

Keep open. Current main still lacks a public post-execution/pre-model tool-result mutation hook such as `tool_result_before_model`, `transform_tool_result`, or `before_context_send`. The closest implementation is runtime-neutral tool-result middleware, but it is explicitly bundled-only, so the external plugin API gap described here remains real. The Agent14 comment is useful production context, but it describes a workaround rather than an implemented OpenClaw contract.

## Best Possible Solution

Keep this issue open until maintainers design and land a documented, trust-gated public post-execution/pre-model tool-result mutation hook, or explicitly decide that same-turn model-visible tool-result rewriting must remain bundled-only. The design should coordinate with #62123, #64907, and #72733 and define sync/async behavior, ordering relative to `before_tool_call`, `after_tool_call`, bundled middleware, `tool_result_persist`, and `before_message_write`, plus failure/timeout behavior, SDK types/docs, and Pi/Codex tests.
