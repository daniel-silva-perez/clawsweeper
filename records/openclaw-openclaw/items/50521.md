## Summary

Keep this PR open. Current main still does not explicitly reject null bytes in archive entry paths, and the PR remains a narrow security hardening change against the shared archive validation path.

## Best Possible Solution

Review and land this PR, or an equivalent minimal patch, by adding explicit null-byte rejection in `src/infra/archive-path.ts` before normalization and keeping the focused regression coverage in `src/infra/archive-path.test.ts`.
