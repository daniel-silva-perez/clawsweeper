## Summary

Keep this PR open. Current main still does not explicitly reject NUL/null bytes in archive entry paths, and the PR remains a narrow security hardening change to the shared archive validation path rather than obsolete or implemented work.

## Best Possible Solution

Review and land this PR, or an equivalent minimal patch, by adding an explicit entryPath.includes("\0") rejection in src/infra/archive-path.ts immediately after the empty-like path return and before drive-path/normalization checks, with focused regression coverage in src/infra/archive-path.test.ts.
