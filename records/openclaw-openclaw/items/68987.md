## Summary

Current main does not implement this PR's intended change. The entry path still uses a hard-coded argv token scan for `secrets audit`, the proposed command-owned manifest and drift tests are absent, and the auth-store consumers still rely on the env guard. This PR remains a plausible implementation candidate for a security-sensitive cleanup rather than a conservative close candidate.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a narrowed implementation that replaces the hard-coded entry argv sniff with a command-owned read-only manifest or equivalent command metadata hook, adds regression coverage for command drift, and addresses the existing review comments about matcher anchoring and alias scope. The `.planning/` artifacts should be split or removed unless maintainers want that repository convention.
