## Summary

Keep this issue open. Current main still builds the Control UI chat session dropdown from stored `sessions.list` rows plus the active `sessionKey`; configured agents from `agents.list` are available but are not used to add dropdown options for agents with no stored session.

## Best Possible Solution

Keep the issue open and review/land #48671 or an equivalent focused patch. The fix should merge `state.agentsList.agents` into `resolveSessionOptionGroups` as normalized per-agent main-session options, dedupe case-insensitively against stored sessions and the active key, preserve global/unknown/cron filtering, respect the configured `mainKey`, and add regression coverage for configured agents that have no stored session row.
