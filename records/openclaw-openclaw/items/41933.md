## Summary

Keep open. Current main still builds the Webchat session dropdown from stored session rows plus the active session key, so configured agents with no stored session row are still omitted; related implementation work remains open and unmerged.

## What This Changes

The issue asks for the Webchat session dropdown to list every configured agent, including agents that do not yet have a stored chat session.

## Best Possible Solution

Review and land #48671 or an equivalent focused patch that merges `state.agentsList.agents` into `resolveSessionOptionGroups`, creates default per-agent main-session options using the configured main key, dedupes case-insensitively against stored sessions and the active key, preserves global/unknown/cron filtering, and adds regression coverage for configured agents with no stored session row.

## Reproduction Assessment

Yes. A source-level reproduction is to pass an `agentsList` containing `alpha` and `beta` while `sessions.sessions` contains only `agent:alpha:main`; current `resolveSessionOptionGroups` has no path that adds `beta`, so the dropdown omits it.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Control UI resolver, because `agents.list` is already loaded and the gateway already exposes configured agents; changing `sessions.list` would blur the store-backed session API contract.
