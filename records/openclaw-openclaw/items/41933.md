## Summary

Current main still does not implement the requested Control UI behavior. The chat session dropdown is built from stored `sessions.list` rows plus the active `sessionKey`; `agents.list` is loaded and available, but configured agents without stored session rows are not synthesized into dropdown options. Related implementation work remains open in #48671, while prior direct attempts #41955 and #42041 were closed unmerged.

## Best Possible Solution

Keep this issue open and review/land #48671 or an equivalent focused patch. The fix should merge `state.agentsList.agents` into `resolveSessionOptionGroups` as normalized per-agent main-session options, dedupe against stored sessions and the active key case-insensitively, preserve current global/unknown/cron filtering, respect the configured `mainKey`, and add regression coverage for configured agents with no stored session row.
