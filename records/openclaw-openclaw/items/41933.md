## Summary

Keep this issue open. Current main still builds the Control UI chat session dropdown from `sessions.list` rows plus the active session key; `agents.list` is loaded and used for labels/agent panels, but `resolveSessionOptionGroups` does not synthesize entries for configured agents that have no session-store row. The provided GitHub context also shows open PR #41955 with closing syntax for this issue, so the issue should not be closed before that implementation candidate lands, closes, or is superseded.

## Best Possible Solution

Keep this issue open and review, land, or supersede the active implementation path. The likely fix belongs in `ui/src/ui/chat/session-controls.ts`: after adding stored sessions and the active key, iterate `state.agentsList.agents`, add canonical main-session options for configured agents without session rows, dedupe keys case-insensitively, preserve existing global/unknown/cron filtering, and add a regression test for configured agents with no stored session row. PR #41955 and related PR #48671 should be evaluated against that narrow behavior before closing the issue.
