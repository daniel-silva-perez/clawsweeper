## Summary

Keep this issue open. Current main still builds the Control UI chat session dropdown from stored `sessions.list` rows plus the active session key; `agents.list` is available and loaded, but the dropdown resolver only uses `agentsList` for labels on session keys that already exist. The supplied GitHub context also shows PR #41955 is still open and uses closing syntax for this issue, so the issue should remain open until that implementation path lands, closes, or is superseded.

## Best Possible Solution

Keep this issue open and review, land, or supersede the active PR path. The likely fix belongs in `ui/src/ui/chat/session-controls.ts`: after stored sessions and the active key are added, iterate `state.agentsList?.agents`, synthesize canonical `agent:<id>:<mainKey>` options for configured agents without stored session rows, dedupe keys case-insensitively, preserve existing global/unknown/cron filtering, and add regression coverage for configured agents with no stored session row.
