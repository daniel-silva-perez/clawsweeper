## Summary

Keep this issue open. Current main still builds the Control UI chat session dropdown from stored `sessions.list` rows plus the active `sessionKey`, while `agents.list` is only used for labels after an option already exists. Configured agents without a stored session row can still be absent from the dropdown, and the related implementation path is still open rather than landed.

## Best Possible Solution

Keep this issue open and finish the focused Control UI fix, likely by reviewing/landing PR #48671 or an equivalent patch. The resolver should merge `state.agentsList.agents` into the dropdown as normalized per-agent main-session options, dedupe case-insensitively, preserve global/unknown/cron filtering, and add regression coverage for configured agents that have no existing session row.
