## Summary

Keep this issue open. Current main still builds the Control UI chat dropdown from `sessions.list` rows plus the active session key; `agents.list` is loaded for labels elsewhere, but configured agents without stored session rows are not synthesized into dropdown options. The provided GitHub context also shows open PR #41955 uses closing syntax for this issue, so the issue should remain open until that implementation path lands, is closed, or is superseded.

## Best Possible Solution

Keep this issue open and review, land, or supersede the active PR path. The likely fix belongs in `ui/src/ui/chat/session-controls.ts`: after stored sessions and the active key are added, iterate `state.agentsList?.agents`, synthesize canonical `agent:<id>:<mainKey>` options for configured agents without stored session rows, dedupe keys case-insensitively, preserve current global/unknown/cron filtering, and add regression coverage for configured agents with no stored session row.
