## Summary

Keep this issue open. Current main still builds the Control UI chat session dropdown from `sessions.list` rows plus the active session key; `agents.list` is loaded, but the dropdown resolver only uses it to label agent groups that already have session keys. The provided GitHub context also shows open implementation candidates, including PR #41955 with closing syntax for this issue, so the issue should stay open until that PR lands, closes, or is superseded.

## Best Possible Solution

Keep the issue open and review, land, or supersede the active implementation path. The likely fix belongs in `ui/src/ui/chat/session-controls.ts`: after adding stored sessions and the active key, iterate `state.agentsList.agents`, synthesize canonical `agent:<id>:<mainKey>` options for configured agents without session rows, dedupe keys case-insensitively, preserve existing global/unknown/cron filtering, and add a regression test for configured agents with no stored session row.
