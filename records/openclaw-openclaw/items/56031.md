## Summary

Keep this PR open. Current main still builds pi embedded runner final payloads from raw `assistantTexts` and `attempt.lastAssistant`, while `before_message_write` hooks can rewrite or block what is actually persisted. The PR targets a real remaining gap, but the helper needs the blocked-current-user-with-history edge case addressed before merge.

## Best Possible Solution

Keep the PR open and request a focused revision: when `before_message_write` hooks are registered, derive `--expect-final` payloads from the post-hook current-attempt assistant instead of raw `assistantTexts`, and add regression coverage for rewritten assistant, blocked assistant, and blocked-current-user-with-history cases. The fix belongs in the pi embedded runner payload path, not in shared hook execution.
