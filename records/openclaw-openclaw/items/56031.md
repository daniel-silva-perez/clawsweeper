## Summary

Keep this PR open. Current main still builds pi embedded runner payloads from the raw attempt assistant text and last assistant message, while `before_message_write` can rewrite or block persisted messages before the post-hook snapshot. The PR targets a real remaining gap, but the proposed helper still needs maintainer/author handling for the blocked-current-user history edge case raised in review.

## Best Possible Solution

Keep the PR open and request a focused revision: derive `--expect-final` payloads from the post-hook current-turn assistant when `before_message_write` hooks are registered, avoid reusing stale raw `assistantTexts`, and add tests for rewritten assistant, blocked assistant, and blocked-current-user-with-history cases. After that, this looks like the right narrow owner-module fix for the pi embedded runner payload path.
