## Summary

Keep open. The selector-level change is directionally useful, but the PR still allows the announce flow to emit a terminal completed event with `(no output)` when no post-compaction assistant reply is visible within the retry window, preserving the result-loss half of the reported production bug.

## What This Changes

The PR adds compaction-marker handling to subagent output selection and new unit tests so pre-compaction raw tool output is not selected as the child result after auto-compaction.

## Best Possible Solution

Propagate the compaction-continuation state beyond text selection so the announce flow defers and retries later instead of delivering a completed `(no output)` event, while preserving timeout partial-progress behavior and coordinating with the broader raw-tool fallback fix in #74340.

## Reproduction Assessment

Yes. A focused code-path reproduction is available: use a child history with assistant tool use, a tool result, a synthetic compaction system marker, and no later assistant turn; current main returns raw tool output, while the PR returns undefined but the caller still announces a completed no-output result.

## Solution Assessment

No. Detecting the compaction marker is the right lower-level signal, but the PR does not carry that signal to `runSubagentAnnounceFlow`, so it suppresses the raw payload without preventing the premature terminal announcement.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Defer instead of announcing no output while compacting:** `src/agents/subagent-announce-output.ts:324`
  - body: When the last visible child history is a compaction marker, this path makes `readSubagentOutput` return `undefined`, but the caller treats that the same as an empty final result: after the retry window, it builds a `completed successfully` announcement with `(no output)`. In the production timeline from the PR body, the real assistant answer arrives minutes later, so the parent would still receive a terminal completion before the actual result and no later announce would fire. Please propagate a distinct continuation/defer state to the announce flow, or otherwise return `false`/keep the run pending until a post-compaction assistant turn or a true timeout is available.
  - confidence: 0.87
