## Summary

Keep open: current main does not implement the intended cache behavior, and the PR head still has a size-only oversize sentinel that can preserve cache hits across same-length over-cap edits. Because that is a concrete cache-integrity issue in a security-adjacent change, this PR should not be merged as-is.

## What This Changes

The PR replaces mtime-based `ensureOpenClawModelsJson` ready-cache inputs with content hashes, adds a stored `models.json` hash check, hardens file reads and chmod, and adds fingerprint-cache tests.

## Best Possible Solution

Keep the content-hash direction, but change the oversize path so it cannot produce stable cache hits based only on file size: either fail closed as a cache miss or return a bounded content digest that includes truncation and total-size metadata. Add focused regression coverage for same-size over-cap edits before rerunning the touched models-config checks.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: create an over-cap `models.json` or `auth-profiles.json`, warm the cache, rewrite different content with the same byte length, and the size-only sentinel remains equal.

## Solution Assessment

No. The proposed content-hash design is the right narrow approach, but the current oversize sentinel undermines the drift-detection property this PR adds; failing closed or hashing a bounded prefix plus size is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Force a miss for oversized file hashes:** `src/agents/models-config.ts:115`
  - body: When a file is already over the cap, `safeHashRegularFile()` returns `oversize:${lst.size}` without reading any content. Cache hits compare this value to the captured hash, so a same-size edit to an oversized `models.json` or `auth-profiles.json` can keep the readyCache hot and skip re-planning. Return `null` for oversize or include bounded content in the digest so content drift cannot collide by size alone.
  - confidence: 0.88

## Security Review
