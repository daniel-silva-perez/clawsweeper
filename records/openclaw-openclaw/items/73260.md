## Summary

Keep this PR open. Current main still has the mtime-based `auth-profiles.json` and `models.json` fingerprint behavior, so the PR is not obsolete, but the proposed cache-integrity path still has a size-only oversize sentinel that can miss same-size content drift above the cap.

## What This Changes

The PR replaces mtime-based `ensureOpenClawModelsJson` ready-cache inputs with content hashes, adds a stored `models.json` hash check, hardens file reads/chmod around `models.json`, and adds fingerprint-cache tests.

## Best Possible Solution

Keep the PR open and update the file-hash helper so oversize files fail closed as cache misses or use a bounded content digest with size/truncation metadata, then add focused regression coverage for same-size over-cap edits and rerun the touched models-config checks.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: create an over-cap `models.json`, warm the readyCache, change the file contents without changing byte length, and the size-only sentinel still matches the captured `modelsJsonHash`, causing a cache hit instead of re-plan.

## Solution Assessment

No. The content-hash direction is the right narrow approach, but the current oversize handling should be changed before merge because it preserves a deterministic non-content hash for the exact drift-detection path this PR adds.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Force a miss for oversized file hashes:** `src/agents/models-config.ts:115`
  - body: When a file is already over the cap, `safeHashRegularFile()` returns `oversize:${lst.size}` without reading any content. Since cache hits compare this value to the captured `modelsJsonHash`, a same-size edit to an oversized `models.json` or `auth-profiles.json` can keep the readyCache hot and skip re-planning. Return `null` for oversize or include bounded content in the digest so content drift cannot collide by size alone.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The PR improves several file-handling paths, but the new size-only oversize hash is a concrete cache-integrity weakness in the proposed drift-detection mechanism.

Concerns:

- **[medium] Size-only oversize hash can bypass drift detection:** `src/agents/models-config.ts:115`
  - body: For over-cap `models.json` or `auth-profiles.json`, the helper returns a deterministic `oversize:<size>` value without content. An attacker or external process that can rewrite an oversized file to different content with the same length can preserve the cache hash and bypass the new re-plan trigger.
  - confidence: 0.87
