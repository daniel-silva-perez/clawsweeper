## Summary

Keep this PR open. Current main still resolves one static memory-wiki vault path at plugin registration, while this PR remains the active scoped implementation for ctx-aware per-org/per-agent vault isolation and has addressed the earlier placeholder, supplement, and catalog-visibility review concerns.

This PR updates the memory-wiki plugin, tests, docs, and local API barrel to resolve templated `vault.path` values from tool invocation context, skip contextless memory supplements for templated paths, and surface unresolved template errors through visible wiki tool stubs.
