## Risks / Open Questions

- Security review pass: the supplied PR file list touches memory-wiki source, tests, docs, and the local API barrel, with no workflow, dependency, lockfile, install script, publishing metadata, or external code execution changes shown. The security-sensitive surface is the filesystem vault path/data-boundary behavior itself.
- The PR changes where memory-wiki reads and writes data when templating is enabled; maintainer review should focus on missing/unknown placeholders, `..` normalization, CWD-relative paths, and write paths that call recursive directory creation.
- The PR intentionally skips memory prompt/corpus supplements when `vault.path` is templated, which is a plausible minimum-scope safety choice but changes `memory_search`/`memory_get` availability for templated deployments.
- Open #63829 may still become the broader product direction for per-agent plugin configuration, but it has not shipped a replacement for this PR's concrete memory-wiki implementation.
