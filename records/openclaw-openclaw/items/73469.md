## Summary

Keep open. Current main still creates a Feishu streaming card before the final agent text is known, then applies the mention-bearing response through Card Kit content updates and close settings. That path can render mentions but does not route agent-authored `<at user_id="...">...</at>` tags through a notification-capable initial send. Related #60103 overlaps on mention syntax normalization, but this issue is specifically about first-notification timing for streaming cards.

## What This Changes

The issue requests Feishu streaming card replies to send push notifications when the agent writes `<at user_id="...">name</at>` mentions in response text.

## Best Possible Solution

Keep this as a Feishu plugin-owned bug and choose a tested outbound strategy that preserves notification semantics for mention-bearing agent replies while making the streaming tradeoff explicit. The end state should cover agent-authored `<at user_id="...">...</at>` tags, existing inbound `mentionTargets`, card/text syntax conversion, topic replies, duplicate-final suppression, and a clear fallback when Feishu cannot notify from Card Kit updates.
