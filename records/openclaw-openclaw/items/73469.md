## Summary

Keep open. Current main still has the Feishu streaming-card notification gap: streaming starts by sending a placeholder Card Kit reference, then final mention-bearing text is applied later through Card Kit element/settings updates. There is no current path that detects agent-authored `<at user_id="...">...</at>` tags and switches them to a notification-capable initial Feishu send. The related #60103 issue overlaps on mention syntax normalization, but does not cover the first-notification timing problem here.

## What This Changes

The issue asks OpenClaw to preserve Feishu @mention push notifications when agent-authored `<at user_id="...">...</at>` tags are delivered through streaming cards.

## Best Possible Solution

Keep this as a Feishu plugin-owned bug. The desired end state is a tested outbound path that preserves notification semantics for mention-bearing agent replies: either a Feishu-supported way to notify from the initial streaming-card send, or a deliberate fallback/companion-send strategy that handles `renderMode: "card"`, auto-mode cards, `mentionTargets`, agent-authored `<at user_id="...">...</at>` tags, thread replies, and duplicate-final suppression.
