## Summary

Keep open. Current main still has the reported Feishu streaming-card notification gap: the initial Card Kit message is sent as a placeholder card reference, while final mention text is only applied later through Card Kit element updates/close. There is no current fallback that switches mention-bearing streaming replies to a notification-capable non-streaming send path, and the older #60103 overlap is only about raw mention syntax normalization, not the first-notification timing problem here.

## Best Possible Solution

Keep this issue open for a Feishu plugin-owned fix. Determine whether Feishu can notify on Card Kit updates; if not, detect outbound mention-bearing replies and switch to a notification-capable path, such as an initial mention-carrying send or a non-streaming final reply with safe preview cleanup. Cover `renderMode: "card"`, auto-mode card replies, agent-authored `<at user_id="...">` tags, existing `mentionTargets`, thread replies, and duplicate-final suppression in focused tests.
