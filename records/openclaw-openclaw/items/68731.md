## Summary

Keep open. Current main still limits BlueBubbles local Contacts enrichment to the group participant path, while 1:1 DM sender labels continue to rely on BlueBubbles-provided senderName or the raw sender id. The issue has concrete field evidence, a narrow BlueBubbles-owned implementation path, and maintainer signal requesting a PR.

## What This Changes

Extend BlueBubbles Contacts enrichment so 1:1 inbound DMs with missing or raw phone/email sender labels can resolve local Contacts names while preserving group behavior and legacy config compatibility.

## Best Possible Solution

Implement this inside the BlueBubbles plugin by extending the existing Contacts resolver/cache to handle DM sender labels and email handles after access/mention gating, keeping the legacy group flag compatible and updating tests and public docs for the broader behavior.

## Solution Assessment
