## Summary

Keep open. Current main does not implement the requested hold-for-continuous dictation UX; the old browser dictation button and STT state were removed, and the remaining Talk control is a separate realtime voice surface rather than short-click auto-send plus 3-second hold mode.

## What This Changes

Request a smarter Control UI microphone button where a short click records one utterance and auto-sends it, while a 3-second hold starts continuous auto-send voice input with interim transcript and mode-specific tooltip copy.

## Best Possible Solution

Keep this open for maintainer product/design review. If accepted, implement it in Control UI source rather than generated dist output, with explicit dictation modes, pointer/touch/keyboard accessibility, no window globals for recognition restart state, updated docs/copy, and focused tests around auto-send and Talk coexistence.

## Reproduction Assessment

Not applicable as a feature request. Source and test inspection gives a high-confidence current-main check: the requested mic-button path cannot be exercised because the Voice input button and STT state are absent.

## Solution Assessment

Unclear. The issue’s proposed dist-file patch and window-global restart approach is not the best maintainable path; a source-level design that either reintroduces dictation cleanly beside Talk or intentionally documents Talk as the replacement would be safer.
