## Summary

Keep this PR open. Current main still does not implement the central change: there is no bundled ExecuTorch plugin, no ExecuTorch/Parakeet macOS Talk Mode STT backend selector, no transcript-only Talk prompt display helper, and no PermissionManager .app guard. The provided review discussion shows useful fixes and author follow-up on earlier bot blockers, but the branch remains broad and security-sensitive enough that it needs maintainer review or splitting rather than cleanup closure.

## Best Possible Solution

Keep this PR open, but do not merge it as-is. The best path is to split or rework it around current main: land any generic Talk STT/plugin seam separately if maintainers want macOS Talk Mode to consume plugin STT backends, keep the ExecuTorch runtime/plugin isolated and security-reviewed, and handle transcript-only chat display plus the PermissionManager .app guard as smaller focused changes. If the ExecuTorch provider itself is not accepted as bundled core, publish that piece through ClawHub while preserving any generic API work OpenClaw needs.
