## Summary

Keep this PR open. Current main does not implement the central change: there is no bundled ExecuTorch plugin, no ExecuTorch/Parakeet macOS Talk Mode STT backend, no Talk STT backend selector, and no transcript-only Talk prompt display helper. Main has related generic plugin/media surfaces and a parakeet-mlx CLI transcript helper, but that is not the same feature. The PR also remains broad and security-sensitive enough that it needs maintainer review or splitting, not cleanup closure.

## Best Possible Solution

Keep this PR open, but do not merge it as-is. The best path is to split or rework it around current main: use existing `registerMediaUnderstandingProvider` / `registerRealtimeTranscriptionProvider` surfaces where possible, propose a small generic native Talk Mode STT backend seam if macOS needs one, keep the ExecuTorch runtime/plugin work isolated, and separately review the native addon, dependency/download, docs, and PermissionManager fixes.
