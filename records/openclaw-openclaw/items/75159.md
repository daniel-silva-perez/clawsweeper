## Summary

Keep this PR open. It targets a real Feishu topic-session prompt gap from #74903, and the extension-owned boundary is appropriate, but the current patch adds a message-tool directive for every topic-scoped group turn and can contradict the existing automatic-reply prompt mode.

## What This Changes

The PR adds Feishu topic-scope prompt helpers and tests, then appends a thread-reply message-tool hint to `GroupSystemPrompt` for `group_topic` and `group_topic_sender` sessions.

## Best Possible Solution

Keep the Feishu-owned prompt fix, but make it align with source reply delivery: only override `send` to `thread-reply` when visible replies are message-tool-only, or phrase the hint conditionally for cases where the model chooses the message tool. Add coverage for both message-tool-only and automatic topic-group modes.

## Reproduction Assessment

Yes. The source path is clear: current main builds a message-tool-only group prompt with `action=send`, while Feishu topic text/card message actions require `action=thread-reply` plus `messageId` to set `replyInThread: true`.

## Solution Assessment

No, not as written. The extension-local approach is the right boundary, but the prompt should not unconditionally tell automatic-delivery group sessions to use the message tool.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Gate the topic hint to message-tool delivery:** `extensions/feishu/src/bot-content.ts:146-147`
  - body: This hint is appended for every topic-scoped Feishu group session, but group turns can still run in automatic delivery mode. In that mode `buildGroupChatContext()` tells the model not to call the message tool because final replies are already delivered by the Feishu dispatcher; the new unconditional instruction tells it to call `message(action="thread-reply")`, which can cause duplicate or conflicting visible replies for users with `messages.groupChat.visibleReplies: "automatic"` or when message-tool delivery falls back to automatic. Gate this hint to message-tool-only delivery or make it conditional on using the message tool.
  - confidence: 0.86
