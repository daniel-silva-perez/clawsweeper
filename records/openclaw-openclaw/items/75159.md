## Summary

Keep this PR open and do not merge it as-is. The linked Feishu topic-thread bug is valid and the Feishu-owned direction is appropriate, but the patch appends a message-tool instruction for every topic-scoped group turn, which conflicts with automatic visible-reply mode, and it is missing the required changelog entry.

## What This Changes

The PR adds Feishu topic-scope prompt helpers and tests, then appends a `thread-reply` message-tool hint to `GroupSystemPrompt` for `group_topic` and `group_topic_sender` sessions.

## Best Possible Solution

Keep the fix Feishu-owned, but make the topic guidance compatible with source reply delivery mode. A corrected patch should gate the explicit message-tool override to message-tool-only turns or phrase it conditionally for cases where the model chooses the message tool, while preserving automatic topic-thread delivery and adding the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: current main prompts message-tool-only Feishu topic group turns toward `action=send`, while Feishu only sets native thread delivery for `action=thread-reply`; the PR regression is reproduced by configuring automatic group visible replies and observing the contradictory prompt instructions.

## Solution Assessment

No, not as written. The extension-local approach is the right ownership boundary, but the proposed prompt hint must respect source reply delivery mode or be conditional so automatic-delivery sessions are not told to call the message tool.
