## Summary

Keep open. Current main has strong Ollama/local-model tuning and recovery controls, but those are mitigations, not proof that this recent qwen3:14b long-context fallback report is fixed. The maintainer discussion already reproduced native Ollama prompt-eval slowness and asked for exact current-main config, fallback logs, and Ollama `prompt_eval_*` metrics to distinguish an OpenClaw context-budgeting bug from expected Ollama prompt ingestion latency.

## Best Possible Solution

Keep this issue open for the requested current-main retest. The next maintainer step should compare OpenClaw's submitted prompt/context against configured `contextWindow` or `contextTokens` and Ollama `prompt_eval_count`; if OpenClaw exceeds the cap, fix prompt budgeting or compaction, and if the counts match but prompt eval dominates, improve local-model docs/diagnostics around `params.num_ctx`, `timeoutSeconds`, `localModelLean`, model sizing, and fallback behavior.
