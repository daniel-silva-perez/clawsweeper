## Summary

Keep open. Current main has Ollama tuning and mitigation knobs for context caps, native `params.num_ctx`, provider request timeouts, timeout-triggered compaction, and lean local-model tool surfaces, but those are not proof that this recent qwen3:14b long-context fallback report is fixed. A maintainer already reproduced the underlying native Ollama prompt-eval slowness and explicitly asked for exact current-main config/log/`prompt_eval_*` data before deciding whether OpenClaw is exceeding its configured cap or Ollama is simply spending the configured prompt budget in prompt evaluation.

## Best Possible Solution

Keep this issue open for the requested current-main retest. The next maintainer action should compare OpenClaw's submitted prompt/context against configured `contextWindow` or `contextTokens` and Ollama `prompt_eval_count`; if OpenClaw exceeds the cap, fix prompt budgeting or compaction, and if the counts match but prompt eval dominates, improve local-model docs/diagnostics around `params.num_ctx`, `timeoutSeconds`, `localModelLean`, model sizing, and fallback behavior.
