## What This Changes

The issue reports impractically slow prompt ingestion and fallback behavior for long-context local Ollama qwen3:14b OpenClaw runs despite healthy short direct Ollama tests.

## Best Possible Solution

Keep this issue open for a current-main retest that compares OpenClaw's submitted prompt budget and last-call usage with configured contextWindow/contextTokens and Ollama prompt_eval_count. If OpenClaw exceeds the cap, fix prompt budgeting, packaging, or compaction; if the counts match and prompt evaluation dominates, improve local Ollama diagnostics and docs around params.num_ctx, timeoutSeconds, localModelLean, thinking mode, model sizing, and fallback behavior.
