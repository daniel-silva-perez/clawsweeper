## Summary

Keep open. The PR takes the right UI-only direction and preserves full-snapshot gateway deltas, but the new slice fallback can still replay already committed pre-tool text when the next delta is exactly the committed prefix length.

## What This Changes

The branch adds Control UI state for committed chat-stream length, slices subsequent full-snapshot chat deltas, resets that offset across run/session lifecycle paths, adds focused UI unit tests, and updates the changelog.

## Best Possible Solution

Keep gateway chat deltas as monotonic full snapshots for ACP, TUI, and generic subscribers, while making the Control UI projection suppress already committed prefix-only snapshots after tool boundaries and covering that edge case in the UI regression tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Handle exact committed-prefix deltas:** `ui/src/ui/controllers/chat.ts:764`
  - body: When `chatStreamCommittedLen` equals the incoming snapshot length, every character in `next` has already been committed to `chatStreamSegments`, but this fallback assigns the whole prefix back to `chatStream`. The gateway can re-broadcast an unchanged full snapshot after the throttle window, so this can recreate the duplicate pre-tool text below the tool card until a longer delta arrives. Treat `next.length === committedLen` as an empty post-tool slice instead of replaying `next`.
  - confidence: 0.82
