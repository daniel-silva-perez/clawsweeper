## Summary

Keep open. Current main still hardcodes reasoning display as `Reasoning:\n_..._` and does not expose a per-channel or per-agent display-format setting. The closed implementation PR #25336 was not merged and its own review context identified incomplete parameter threading, so it does not implement or supersede the requested feature.

## Best Possible Solution

Keep this issue open. A complete fix should define a small reasoning display-format contract, preserve italic as the compatibility default, choose and document precedence for per-channel versus per-agent/global defaults, add config schema/type/docs coverage, thread the selected format through embedded payloads, subscribe streaming, message-end block replies, and Telegram reasoning lanes, and add regression tests for default italic plus configured code/spoiler/blockquote behavior.
