## Summary

Keep this PR open. Current main still does not implement the PR's central Telegram/runtime performance changes: Telegram sticker/audio preflight remains sequential, command detection lacks the proposed prefix fast path, envelope weekday formatting still constructs new Intl.DateTimeFormat instances, typing keepalive still uses the old setInterval loop, workspace bootstrap reads remain sequential, and Telegram still awaits inbound session recording. The PR is broad and stale, but it remains useful prior art rather than obsolete or implemented.

## Best Possible Solution

Keep this PR open as useful prior art, but ask for a focused rebase or split. Telegram-specific pipeline work should be separated from shared auto-reply/channel/agent optimizations, current fresh-config routing behavior must be preserved, typing lifecycle and markdown chunking concerns should be resolved with tests, and voice-specific latency work can be coordinated with #48179 and #58887.
