## Summary

Keep this PR open. Current main still does not implement native iOS/macOS Adaptive Card marker parsing or inline SwiftUI rendering in the shared chat UI. The linked openclaw-adaptive-cards plugin is useful ecosystem work, but the current plugin and message-presentation APIs do not expose a native Swift chat bubble renderer extension point that would make this PR obsolete as ClawHub-only work. The same contributor's Android companion PR #42304 is also still open, so the platform decision should be reviewed together.

## Best Possible Solution

Keep the PR open for maintainer product and security review. If native iOS/macOS Adaptive Card rendering is wanted in OpenClaw clients, rebase on current main, resolve the parser/layout/fallback/action-data review items, add focused Swift parser/view or app smoke coverage, explicitly gate or document remote image and URL behavior, and coordinate the platform decision with Android PR #42304. If maintainers decide this should stay external, close only after a clear product decision and a documented plugin/API path for native client-side rendering.
