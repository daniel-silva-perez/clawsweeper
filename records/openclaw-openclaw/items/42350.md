Keep this PR open. Current main still does not implement the proposed iOS/macOS Swift Adaptive Card parser or inline SwiftUI chat renderer, the PR is still open and unmerged, and the same contributor has the Android companion PR #42304 open for the same feature set. This is not conservative cleanup material; it needs maintainer product and security review.

## Best Possible Solution

Keep this PR open for maintainer product and security review. If native iOS/macOS Adaptive Card rendering is wanted in core, rebase it on current main, fix the parser/layout/fallback/action-data review items, add focused Swift tests or app smoke coverage, explicitly review remote image and URL-opening behavior, and coordinate the platform decision with open Android companion PR #42304. If maintainers decide native app card rendering should live outside core, first identify or add a native app renderer extension point rather than closing this as already handled by the existing plugin surface.
