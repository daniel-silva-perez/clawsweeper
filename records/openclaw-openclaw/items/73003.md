## Summary

Keep open. Current main still only disables Pi auto-compaction when a context engine owns compaction, while safeguard mode is wired as an OpenClaw extension path rather than a context engine; the open closing PR #73839 is the right place to review the narrow fix before this issue closes.

## What This Changes

Safeguard compaction should coordinate with or disable Pi SDK auto-compaction so OpenClaw preflight compaction does not repeatedly fail with `already_compacted_recently` in long-running sessions.

## Best Possible Solution

Review and land #73839 or an equivalent narrow patch that threads compaction mode into the Pi auto-compaction guard, disables Pi auto-compaction for safeguard-owned lifecycle runs, and adds focused tests for safeguard, default, off, context-engine-owned, and unsupported-settings cases.

## Reproduction Assessment

Yes. The report gives concrete config, model, channel, and repeated `already_compacted_recently` outcomes, and current-main source shows the guard cannot fire for safeguard mode unless a context engine owns compaction.

## Solution Assessment

Yes, the linked direction is the narrowest maintainable fix: keep the behavior in `pi-settings` and its run/CLI callers rather than adding startup migrations or owner-specific policy elsewhere.
