## Summary

Keep open: current main still leaves Pi SDK auto-compaction enabled for `compaction.mode = "safeguard"` unless a context engine declares `ownsCompaction`, and open PR #73839 is the active unmerged implementation candidate that explicitly closes this issue.

## What This Changes

The issue requests that OpenClaw disable or coordinate Pi SDK auto-compaction when `agents.defaults.compaction.mode` is `safeguard`, so safeguard preflight compaction is not blocked by Piâ€™s recent compaction entries.

## Best Possible Solution

Review and land #73839 or an equivalent narrow patch that makes the Pi auto-compaction guard treat safeguard mode as OpenClaw-owned compaction, threads that state through embedded and CLI compaction call sites, and covers safeguard, default/off, context-engine-owned, and unsupported-settings-manager cases.

## Reproduction Assessment

Yes. With `agents.defaults.compaction.mode = "safeguard"` and no owning context engine, current main enables the safeguard extension but `applyPiAutoCompactionGuard` still returns `disabled: false`, leaving Piâ€™s post-turn threshold compaction active.

Yes. The narrow maintainable fix is to put the policy in `pi-settings` and pass the configured compaction mode through the embedded and CLI call sites, rather than adding channel-specific recovery or startup migration logic.
