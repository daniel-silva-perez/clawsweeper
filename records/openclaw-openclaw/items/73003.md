## Summary

Keep open: current main still leaves Pi auto-compaction enabled for safeguard mode unless a context engine owns compaction, and the open unmerged PR #73839 is the active implementation candidate for this issue.

## What This Changes

Safeguard compaction should coordinate with or disable Pi SDK auto-compaction so OpenClaw preflight compaction can run instead of repeatedly failing with `already_compacted_recently`.

## Best Possible Solution

Review and land #73839 or an equivalent narrow patch that makes OpenClaw disable Pi auto-compaction whenever safeguard mode owns the lifecycle, threads that decision through the embedded and CLI compaction call sites, and covers safeguard, default, off, context-engine-owned, and unsupported-settings cases.

## Reproduction Assessment

Yes. The report gives concrete configuration and repeated `already_compacted_recently` outcomes, and a focused source/unit reproduction is clear: with `compaction.mode = "safeguard"` and no owning context engine, current `applyPiAutoCompactionGuard` returns `disabled: false` while the safeguard extension is still enabled from config.

## Solution Assessment

Yes. The linked direction is the narrowest maintainable fix because the policy belongs in `pi-settings` and its run/CLI callers, not in startup migration, channel code, or a separate repair path.
