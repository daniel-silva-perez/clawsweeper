## Summary

Keep open. Current main still does not disable Pi SDK auto-compaction for safeguard mode unless a context engine also declares ownership, and the issue is already paired with open closing PR #73839 for the narrow fix.

## What This Changes

The issue asks OpenClaw to prevent Pi SDK auto-compaction from blocking safeguard-mode compaction and producing repeated `already_compacted_recently` failures.

## Best Possible Solution

Review and land #73839, or an equivalent narrow patch, so safeguard mode is treated as OpenClaw-owned compaction for Pi auto-compaction guard purposes while preserving default/off and context-engine-owned behavior with focused regression tests.

## Reproduction Assessment

Yes. The report provides concrete config, model, channel, and repeated `already_compacted_recently` outcomes, and current-main source shows the guard cannot fire for safeguard mode unless a context engine owns compaction.

## Solution Assessment

Yes. Threading compaction mode into the existing Pi auto-compaction guard is the narrow maintainable solution; the active paired PR is the right review surface before closing this issue.
