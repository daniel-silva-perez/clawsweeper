## Summary

Keep open. Current main still leaves Pi auto-compaction enabled for safeguard mode unless a context engine declares `ownsCompaction`, and open PR #73839 remains the active unmerged implementation candidate that explicitly closes this issue.

## What This Changes

The issue requests that OpenClaw disable or guard Pi SDK auto-compaction when `compaction.mode` is `safeguard` so OpenClaw safeguard compaction is not repeatedly blocked by Pi's internal compaction marker.

## Best Possible Solution

Land #73839 or an equivalent narrow patch that makes the Pi auto-compaction guard use OpenClaw's resolved safeguard-owned compaction state, including provider-forced safeguard mode, while preserving `ownsCompaction` behavior and covering embedded and CLI call sites with focused tests.

## Reproduction Assessment

Yes. A source-level reproduction exists: configure safeguard compaction without an owning context engine, and current main's guard still returns `disabled: false` while the pinned Pi SDK keeps threshold auto-compaction enabled by default.

## Solution Assessment

Yes, the narrow maintainable solution belongs in the Pi settings guard and its embedded/CLI call sites. The implementation should use the resolved safeguard predicate rather than only raw `agents.defaults.compaction.mode`.
