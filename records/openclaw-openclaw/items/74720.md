## Summary

Keep this PR open for maintainer review. Current main still mutates cron jobs before cron-expression validation on the disabled-job path, and this PR is a focused service-level fix with regression coverage and no blocking static review finding; the remaining decision is whether maintainers prefer this service atomicity fix or the overlapping gateway-only #74068 path.

## What This Changes

The PR makes `cron.update` stage patches on a cloned job, validate cron schedules before committing stored state, and adds service regression coverage for disabled invalid cron edits and failed enables.

## Best Possible Solution

Land one canonical fix that makes cron service updates atomic: build the full post-patch job, validate any throwing schedule before replacing stored state, preserve tolerance for already-invalid disabled persisted jobs, and keep gateway error responses as `INVALID_REQUEST`. If this PR is accepted, use it to resolve #74459 and then supersede the gateway-only alternative.
