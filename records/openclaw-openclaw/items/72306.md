## Summary

Keep this PR open. Current main still has the gateway pricing refresh lifecycle gap the PR describes: pricing fetches are timeout-only, the 24-hour refresh timer is a normal ref'd timeout, and the stop callback only clears the scheduled timer. The current draft head has also been force-pushed down to a one-line comment change, so it is not mergeable as the stated fix, but the linked regression remains unresolved and this is not implemented or superseded on main.

## Best Possible Solution

Keep this PR open as draft for maintainer review. The useful path is to restore or replace the original functional change in src/gateway/model-pricing-cache.ts: unref the 24-hour pricing refresh timer, connect LiteLLM/OpenRouter fetches to a shutdown AbortController, abort in-flight refreshes from the returned stop callback, and add the focused regression tests in src/gateway/model-pricing-cache.test.ts. Coordinate with #73486 only for startup deferral; it does not replace the stop/unref lifecycle fix shown here.
