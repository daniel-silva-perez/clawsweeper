## Summary

Keep this PR open. Current main still has the reported `session_status` gap for sparse channel-plugin requester sessions, and this PR is a focused, CI-green fix with regression coverage; the overlapping #74142 PR is narrower and does not replace the later model-override persistence guard added here.

## What This Changes

The PR adds a final `session_status` implicit-current fallback for sparse channel-plugin requester sessions, generates a valid persisted session id when a model override materializes the synthetic entry, and adds regression tests.

## Best Possible Solution

Land this PR or an equivalent canonical fix after maintainer review, preserving literal `current` precedence, strict unknown-key behavior, self-scoped visibility, sparse channel-plugin status cards, and generated session ids for materialized synthetic entries; then close or retire the overlapping #74142 path.

## Reproduction Assessment

Yes: current main can be traced through the `session_status` resolver with a channel-style requester, empty store, and `sessionKey="current"`, where it falls through to `Unknown sessionKey: current`. The PR adds focused unit coverage for that explicit lookup and the omitted-session default lookup.

## Solution Assessment

Yes: this is a narrow maintainable solution because it runs after existing literal/session-id/default-direct resolution paths, scopes the fallback to the requester key, and avoids broadening `resolveCurrentSessionClientAlias` for unrelated tools. The added `mergeSessionEntry` path is the safer choice for model overrides because it prevents empty `sessionId` persistence.
