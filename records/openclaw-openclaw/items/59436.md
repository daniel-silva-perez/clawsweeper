## Summary

Keep this PR open. Current main has already absorbed the cheap search/activity/limit-before-row-enrichment part, but it has not implemented the PR’s request-scoped child-session index; `buildGatewaySessionRow()` still calls `resolveChildSessionKeys()` and that helper still scans `Object.entries(store)` for each built row.

## Best Possible Solution

Keep the PR open, but ask the contributor to rebase onto current main and narrow the remaining change to a request-scoped child-session relationship index. The implementation should preserve the current `resolveChildSessionKeys()` semantics, especially live display-subagent ownership and stale terminal child-link filtering, and add focused coverage or instrumentation for the child lookup complexity.
