## Summary

Keep this PR open for maintainer review: current main has not implemented the PR's intended OpenAI-compatible HTTP session-prefix change, but the proposed patch still derives the prefix from the agent default before request-level model overrides and changes a backward-incompatible generated-session namespace.

## What This Changes

The branch adds provider-prefix resolution for OpenAI-compatible HTTP generated session keys, threads a pre-resolved agent id through gateway request context, and adds gateway tests for provider-prefixed keys.

## Best Possible Solution

Maintainers should first decide the intended namespace for generated OpenAI-compatible HTTP sessions. If provider-prefixed generated keys are desired, derive the prefix from the accepted backend model selection, including `x-openclaw-model`, keep explicit `x-openclaw-session-key` behavior unchanged, and add focused tests for default-provider and override-provider requests.

## Reproduction Assessment

Yes for the patch defect: a default Anthropic agent request with `x-openclaw-model: openai/gpt-5.4` follows the documented override path but the PR computes the generated session prefix before that override is resolved. I did not run tests because this review was explicitly read-only.

## Solution Assessment

No. The current PR is a plausible narrow direction, but it is not the best mergeable fix while it ignores accepted model overrides and leaves the backward-incompatible namespace decision unresolved.
