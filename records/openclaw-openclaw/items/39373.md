## Summary

Keep open. Current main still presents the Control UI as a Vite + Lit app with repo-local CSS and no Tailwind/shadcn/Radix-style component-library migration or technical-selection comparison document. The newer tweakcn theme import work is relevant UI modernization, but it is explicitly scoped to one browser-local theme slot and does not complete the requested component-library assessment or roadmap.

## Best Possible Solution

Keep this open until maintainers either publish a focused Control UI component-library assessment and technical-selection comparison, or explicitly document that OpenClaw’s intended direction is to stay with Lit plus repo-local CSS/CSS variables. The artifact should inventory current primitives, compare the current stack with Tailwind/shadcn/Radix-style options, call out migration cost and security/supply-chain implications, and define a small roadmap or no-migration decision.
