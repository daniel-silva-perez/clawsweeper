## Summary

Keep this issue open. Current main still implements only cache-TTL-triggered context pruning: active sessions inside the TTL window skip pruning, public config/schema only accepts `off` and `cache-ttl`, and docs/tests describe TTL-window suppression rather than rolling per-turn pruning.

## Best Possible Solution

Keep the issue open as a real core context-pruning feature request. The likely fix belongs in the built-in context-pruning path: add a rolling or turn-age option that can run every turn independent of TTL, update config types, Zod validation, generated schema, docs, and add focused tests proving active sessions with continuous cache touches still prune old tool results while preserving the protected recent-turn tail. Review or refresh #43238 if maintainers want to use that PR as the implementation path.
