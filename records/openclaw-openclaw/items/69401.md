## Summary

Keep open. Current main has useful partial hardening: inbound metadata is separated into user-role untrusted JSON blocks, external-content wrappers use random boundary IDs, and Discord guild messages add a wrapped copy of the message body. The requested cross-channel body wrapping and memory provenance are not implemented generally: Slack/Telegram/Teams still pass raw message text as BodyForAgent/RawBody, and the memory/dreaming export strips metadata then indexes plain user text without channel trust attribution.

## Best Possible Solution

Keep this issue open for a maintainer-led security design. The likely direction is a generic inbound user-body wrapper in core prompt assembly using the existing random-ID external-content markers, with channel plugins supplying source/trust metadata through a typed context field. Memory/dreaming should preserve enough source/trust attribution to filter or downgrade channel-originated memories without relying on stripped prompt text.
