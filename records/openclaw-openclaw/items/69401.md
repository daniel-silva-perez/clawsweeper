## Summary

Keep open. Current main has useful partial hardening for inbound metadata, external-content boundaries, and memory corpus cleanup, but the requested channel-message body wrapping is not implemented generically: model-visible BodyForAgent/baseBodyFinal still carries raw current inbound text for major channels, and memory/session export still strips to plain user text without durable channel trust provenance. The KinThai comment's boundary-escape and memory-provenance concerns remain relevant.

## What This Changes

Request a generic security hardening path that wraps inbound channel message bodies as untrusted content and carries source/trust provenance through prompt assembly and memory ingestion.

## Best Possible Solution

Keep this open for maintainer-led security design. The best path is a generic channel ingress contract that wraps the current model-visible inbound body with the existing random external-content boundary mechanics, preserves clean RawBody/CommandBody/BodyForCommands for parsing and authorization, and carries typed channel source/trust provenance through session export, dreaming/indexing, and retrieval so channel-originated memories can be filtered or downgraded.

## Risks / Open Questions

- A narrow wrapper-only change could break command parsing, reset/new-session handling, mention detection, or channel-specific routing if RawBody/CommandBody and BodyForAgent responsibilities are not kept separate.
- Duplicating wrapped content in both BodyForAgent and UntrustedContext could increase prompt size or cause repeated instructions unless the new contract guarantees exactly-once model-visible wrapping.
- Memory changes need a deliberate trust/provenance schema; preserving too much metadata can reintroduce noisy corpus ingestion, while preserving too little leaves the reported long-term memory poisoning path open.
- This is security-sensitive and cross-cutting across channel ingress, prompt assembly, memory export, indexing, and retrieval, so it needs maintainer design review before an automated repair PR.
