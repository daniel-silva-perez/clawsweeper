## Summary

Keep open. Current main still has partial hardening only: inbound metadata/history are rendered as untrusted blocks and the external-content helper has random boundary markers, but the current channel message body still reaches the prompt through raw BodyForAgent/baseBodyFinal paths, and session-corpus export strips metadata into plain user text without durable channel trust provenance. The KinThai comment's boundary-escape and memory-provenance concerns remain relevant.

## Best Possible Solution

Keep this open for maintainer-led security design. The best implementation is a generic channel ingress contract that wraps the current model-visible body with the existing random external-content boundary mechanics, preserves CommandBody/RawBody for parsing, and carries typed source/trust provenance through session export, dreaming, indexing, and retrieval so channel-originated memories can be filtered or downgraded.
