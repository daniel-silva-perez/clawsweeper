Keep this PR open. Current main still has the unsafe fixed-step memory chunk coarse split and still sends raw embedding text/structured inputs to providers without an unpaired-surrogate sanitizer. The PR is also paired with same-author #65782 and is the active implementation candidate for the #27753 failure class, so it should be reviewed/rebased/landed or closed by normal maintainer review rather than swept as obsolete.

After targeted tests and maintainer review, landing this PR should resolve #65782 and the #27753 sanitizer failure class.
