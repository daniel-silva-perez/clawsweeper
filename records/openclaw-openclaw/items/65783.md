## Summary

Keep this PR open. Current main still has the unsafe fixed-step coarse split in `chunkMarkdown` and still sends raw embedding texts/structured inputs to providers without an unpaired-surrogate sanitizer. The PR is tied to the still-open same-author bug report #65782 and the earlier #27753 report, and its package-level chunker patch is now relevant because current main routes the memory-core plugin SDK storage facade to `packages/memory-host-sdk`. It should be rebased/reviewed against the current memory-host-sdk package/source mirror layout rather than closed as obsolete.

## Best Possible Solution

Keep this PR open and ask for a focused current-main rebase/revision. The intended fix should patch the current package implementation used by the plugin SDK storage facade, add the coarse-boundary surrogate regression in the active memory-host-sdk tests, add embed-boundary sanitizer coverage in `extensions/memory-core`, and either align or explicitly retire the remaining `src/memory-host-sdk/host/internal.ts` mirror so future maintainers do not see two divergent chunkers. After review and targeted tests, the corrected PR can resolve #65782 and the #27753 class of failures.
