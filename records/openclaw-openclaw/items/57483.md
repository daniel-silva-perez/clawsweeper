## Summary

Keep this PR open. Current main still blocks cross-agent `sessions_send` under `tools.sessions.visibility` values other than `all` before the a2a policy can allow it, so the PR's intended behavior is not implemented on main. The patch is still plausible, but it needs to be rebased from the old `src/agents/tools/sessions-access.ts` implementation to the current SDK helper and reviewed as a cross-agent messaging authorization change.

## Best Possible Solution

Keep this PR open and rebase the behavior onto `src/plugin-sdk/session-visibility.ts`: let `send` skip the cross-agent visibility gate only when the a2a policy permits the requester and target, keep list/history/status visibility-gated, and add regression coverage for `tree` visibility plus a2a-permitted `sessions_send` alongside denied/disabled a2a cases. Treat sandbox label/session resolution as related but separate work unless the final fix intentionally covers it too.
