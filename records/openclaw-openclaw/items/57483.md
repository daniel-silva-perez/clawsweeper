## Summary

Keep this PR open. Current main still blocks cross-agent `sessions_send` before the a2a policy when `tools.sessions.visibility` is not `all`; the implementation moved from `src/agents/tools/sessions-access.ts` into `src/plugin-sdk/session-visibility.ts`, but the requested behavior is not present there. The PR remains a plausible focused fix for the linked open sessions/a2a visibility bug, though it needs a rebase onto the new source location and maintainer security review for the cross-agent messaging boundary.

## Best Possible Solution

Review or rebase this PR onto `src/plugin-sdk/session-visibility.ts`, add/keep focused regression tests for `visibility=tree` plus a2a-permitted `sessions_send`, and separately decide the sandboxed label/session resolution blocker tracked by related work. Do not close this PR as implemented until current main actually lets a2a policy gate cross-agent sends independently of read visibility.
