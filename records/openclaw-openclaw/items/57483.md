## Summary

Keep this PR open. Current main still blocks cross-agent `sessions_send` under non-`all` session visibility before the a2a policy can allow it, so the request is not implemented; however, the active implementation moved into the plugin SDK helper, so the PR needs a rebase/update before it can be merged.

## What This Changes

The PR changes the session visibility guard so cross-agent `send` skips the visibility check and is authorized by a2a policy, removes the now-dead send visibility message branch, and adds regression tests for send allowed while history remains blocked.

## Best Possible Solution

Apply the send-only visibility exception in `src/plugin-sdk/session-visibility.ts`, keep `list`, `history`, and `status` visibility-gated, add focused regression coverage for `visibility=tree` with a2a-allowed and a2a-denied sends, and track sandboxed label-resolution behavior separately if it remains blocked.

## Reproduction Assessment

Yes. A source-level reproduction is `createSessionVisibilityGuard({ action: "send", requesterSessionKey: "agent:main:main", visibility: "tree", a2aPolicy: allow main+ops }).check("agent:ops:main")`: current main returns the visibility-forbidden branch at `src/plugin-sdk/session-visibility.ts:201` before the a2a policy is checked.

## Solution Assessment

No, not as currently targeted against main. The conceptual fix is narrow and plausible, but the PR must move the condition into the SDK helper and preserve regression tests around the active call path before it is the best mergeable solution.
