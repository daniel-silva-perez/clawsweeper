## Summary

Keep open. Current main has partial mitigations, but it still documents and tests non-off implicit API-thinking defaults for reasoning-capable models and Claude 4.6, cron jobs without explicit thinking can still fall through to the default resolver, and the Microsoft Teams renderer still lacks the reasoning-payload suppression guard used by other channel delivery paths.

## What This Changes

The issue asks OpenClaw to make Claude/Anthropic API thinking opt-in by default, add safer global and cron defaults/audits, and prevent reasoning content from being rendered into messaging channels unless explicitly requested.

## Best Possible Solution

Define one explicit runtime policy: API thinking is off unless explicitly configured or required by the provider/model, cron jobs inherit the same resolved agent/global policy without silent medium defaults, messaging channels suppress reasoning payloads unless the channel deliberately supports that reasoning lane, and docs/release notes flag billing-affecting default changes before upgrade.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: resolve a reasoning-capable catalog model or Claude 4.6 with no explicit thinking setting, run an agentTurn cron payload without thinking, and pass an isReasoning reply payload through the Teams render path.

## Solution Assessment

No. Current main is only a partial mitigation: it exposes global defaults and keeps Opus 4.7 off, but still retains non-off implicit defaults for other reasoning-capable models/Claude 4.6 and lacks the Teams suppression guard.
