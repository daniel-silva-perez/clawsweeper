## Summary

Keep open. Current main has partial mitigation for Anthropic Opus 4.7 and now exposes global reasoning visibility defaults, but it still has non-off API thinking defaults for Claude 4.6/provider-declared profiles and generic reasoning-capable catalog models, cron agent-turn payloads without `thinking` still inherit that resolver, and Microsoft Teams still lacks a channel-side reasoning-payload suppression guard.

## What This Changes

The issue requests cost-safe opt-in defaults for Claude/API thinking, a global reasoning visibility default, startup or doctor warnings for enabled reasoning/thinking, and separation of API-level thinking from chat-visible reasoning output.

## Best Possible Solution

The desired end state is an explicit policy that API thinking is off unless the user configured it or the provider/model truly requires it, cron jobs inherit that policy consistently, messaging channels suppress reasoning payloads unless a surface explicitly supports and requests them, and docs/release notes make any billing-affecting default visible before upgrade.

## Reproduction Assessment

Yes. Current-main source and tests provide a high-confidence reproduction path for the remaining behavior: leave `thinking` unset on a Claude 4.6 or reasoning-capable catalog model, including an isolated cron `agentTurn`, and the resolver reaches a non-off default; Opus 4.7 is now a current-main exception that defaults off.

## Solution Assessment

No. Current main is only a partial solution: it adds the global reasoning visibility knob and special-cases Opus 4.7, but it does not fully implement opt-in API thinking, cron cost safety, or Teams reasoning-payload suppression.
