## Summary

Keep open. Current main partially addresses the report by exposing global reasoning/thinking config and inheriting agent defaults into isolated cron runs, but the core cost behavior remains intentional in source, tests, docs, and changelog: reasoning-capable catalog models still implicitly resolve to a non-off thinking level. Teams also still lacks a local reasoning-payload suppression guard in its plugin delivery path, so the visible-output/security part is not fully closed.

## What This Changes

The issue asks OpenClaw to make Claude/API thinking opt-in by default, add a global reasoning default, warn on enabled reasoning, and prevent reasoning blocks from rendering into messaging channels.

## Best Possible Solution

The desired end state is a cost-safe default policy where API thinking is off unless explicitly configured or a provider/model requires it, global and per-agent defaults apply consistently to chat and cron runs, and all messaging plugins suppress reasoning payloads unless the user explicitly enables a supported reasoning-visibility lane. The release notes and docs should call out any default that can change provider billing or visible channel output.

## Reproduction Assessment

Yes. Source and tests provide a high-confidence reproduction path: configure a reasoning-capable catalog model with no `agents.defaults.thinkingDefault` or per-agent `thinkingDefault`, then the resolver reaches `medium`; Teams can also render a supplied reasoning payload because its plugin renderer has no local suppression guard.

## Solution Assessment

No. Current main is only a partial solution: global defaults and cron agent-default merging exist, but the implicit non-off thinking fallback remains intentional and Teams rendering still lacks the same reasoning guard used by safer channel paths.
