## Summary

Keep open. Current main has partial mitigation through global/per-agent config knobs and an Anthropic Opus 4.7 provider profile default of off, but it still intentionally defaults some Claude 4.6 models to adaptive thinking, keeps the generic reasoning-capable fallback at medium, lets cron jobs without payload thinking inherit that default path, and leaves Microsoft Teams without the reasoning-payload suppression guard used elsewhere.

## What This Changes

The issue asks OpenClaw to make Claude/Anthropic reasoning and thinking opt-in by default, add global default control, warn about enabled reasoning, and prevent reasoning blocks from leaking into messaging channels.

## Best Possible Solution

Set a cost-safe default policy where API thinking is off unless explicitly configured or a provider/model declares thinking mandatory; make global and per-agent thinking and reasoning defaults apply consistently to chat, status, and cron agentTurn runs; add doctor/startup diagnostics for non-off defaults and cron payloads; and suppress or deliberately route reasoning payloads on messaging plugins with clear docs and release notes.

## Reproduction Assessment

Yes. Current-main source and existing tests give a high-confidence reproduction path: use an unset thinking configuration with a reasoning-capable catalog model or a cron agentTurn payload without thinking, and the resolver reaches a non-off default before the Anthropic transport turns that into a thinking request.

## Solution Assessment

No. Current main is only a partial solution: it adds reasoningDefault configuration and sets Anthropic Opus 4.7's profile default to off, but it does not make all Claude/API thinking opt-in, does not add the requested startup/doctor audit, and does not fully close the Teams reasoning-rendering gap.
