## Summary

Keep open. Current main has partially improved the configuration surface by adding `agents.defaults.reasoningDefault`, but it still keeps capability-based thinking/reasoning defaults that can produce non-off API thinking for reasoning-capable models, cron `agentTurn` jobs still inherit that path when `payload.thinking` is unset, and the Microsoft Teams renderer still lacks a reasoning-payload suppression guard.

## What This Changes

The issue asks OpenClaw to stop silently enabling Claude reasoning/thinking by default, add global/default controls and warnings, and prevent reasoning blocks from leaking to messaging channels and cron jobs.

## Best Possible Solution

The desired end state is a cost-safe policy where API thinking is off unless explicitly configured or a provider/model declares it mandatory, global and per-agent defaults apply consistently to chat and cron runs, reasoning payloads are flagged and suppressed on messaging surfaces unless explicitly requested, and docs/release notes call out any default that can change billing or visible output.

## Reproduction Assessment

Yes. Current-main source and tests give a high-confidence reproduction path: use an unset reasoning/thinking configuration with a reasoning-capable catalog model, or an isolated cron `agentTurn` without `payload.thinking`, and the resolver reaches the non-off model default path.

## Solution Assessment

No. Current main is only a partial solution because `agents.defaults.reasoningDefault` exists, but API-level thinking defaults, cron unset-payload behavior, and Teams reasoning rendering are still not fully resolved.
