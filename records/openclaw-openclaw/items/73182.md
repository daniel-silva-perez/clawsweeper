## Summary

Keep open. Current main has partial mitigation for Claude Opus 4.7 and exposes global/per-agent defaults, but it still intentionally defaults some unconfigured reasoning-capable paths to non-off API thinking, still documents Claude 4.6 adaptive-by-default behavior, and still has a Teams render path that can turn reasoning-marked payloads into user-visible messages.

## What This Changes

The issue requests making API thinking and reasoning visibility opt-in by default for Claude/reasoning-capable models, adding global/default and cron-safe controls, warning about billing-impacting enables, and preventing reasoning blocks from reaching messaging channels.

## Best Possible Solution

Define one explicit runtime policy: API thinking stays off unless explicitly configured or truly required/provider-owned, cron jobs inherit the same resolved agent/global policy including the default agent, channel delivery suppresses reasoning-only payloads unless a channel has an intentional reasoning lane, and docs/release notes/doctor warn when billing-impacting thinking is enabled.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: resolve a reasoning-capable catalog model or Claude 4.6 with no explicit thinking setting, run an `agentTurn` cron payload without `thinking`, and pass an `isReasoning` reply payload through the Teams renderer.

## Solution Assessment

No. Current main is not the complete solution because it keeps Opus 4.7 off and exposes defaults, but still retains non-off implicit defaults for other reasoning-capable models and leaves the Teams suppression gap open.
