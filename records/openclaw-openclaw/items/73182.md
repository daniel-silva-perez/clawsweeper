## Summary

Keep open. Current main has partial mitigation for reasoning visibility and Opus 4.7, but the underlying cost-sensitive API thinking default still resolves non-off for other reasoning-capable/default-profile paths, cron jobs without explicit `payload.thinking` still inherit that default resolver, and Microsoft Teams still lacks the reasoning-payload suppression guard used by other channels.

## What This Changes

The issue asks OpenClaw to make Claude/API thinking opt-in by default, add global/per-agent default controls and warnings, apply them consistently to cron runs, and prevent reasoning blocks from leaking into messaging surfaces.

## Best Possible Solution

The desired end state is a cost-safe and disclosure-safe contract: API thinking is off unless explicitly configured or required by a provider/model profile, chat and cron runs honor the same global/per-agent `thinkingDefault` policy, `openclaw doctor` or startup diagnostics flag unset reasoning-capable cron jobs, channel renderers suppress reasoning payloads unless visibility is explicitly enabled, and docs/changelog call out any billing-affecting default changes prominently.

## Reproduction Assessment

Yes. A source-backed reproduction is available on current main: leave `payload.thinking` and `agents.*.thinkingDefault` unset for a reasoning-capable catalog model and the cron/chat resolver reaches the non-off model thinking default; a reasoning payload handed to the Teams renderer is also sent because no suppression guard runs there.

## Solution Assessment

No. Current main is only a partial solution: Opus 4.7 and some reasoning-visibility paths are safer, but capability-based API thinking defaults, cron unset-payload behavior, and Teams reasoning rendering are not fully resolved.
