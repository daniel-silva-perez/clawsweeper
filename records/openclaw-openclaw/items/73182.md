## Summary

Keep open. Current main has partial mitigations, including an exposed global `agents.defaults.reasoningDefault` and an Opus 4.7 default-off special case, but it still intentionally resolves several reasoning-capable or Claude 4.6 paths to non-off API thinking, cron `agentTurn` jobs without explicit `thinking` still fall through to the default resolver, and the Microsoft Teams renderer still lacks the reasoning-payload suppression guard used by other channel paths.

## What This Changes

Request to make Claude/API thinking opt-in by default, add safer global/default controls and startup warnings, make cron jobs inherit the intended thinking default, and prevent reasoning blocks from rendering in chat channels.

## Best Possible Solution

Define one explicit policy across runtime, cron, docs, and channel delivery: API thinking should stay off unless explicitly configured or truly required by the selected provider/model; cron `agentTurn` jobs should inherit the same resolved agent/global thinking default as normal turns; user-visible channel renderers should drop reasoning-marked payloads unless they implement an intentional reasoning lane; and release notes should call out billing-impacting default changes prominently.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by resolving an unconfigured Claude 4.6 or reasoning-capable catalog model, running a cron `agentTurn` without `payload.thinking`, and rendering a Teams payload with `isReasoning` or a `Reasoning:` prefix.

## Solution Assessment

No. Current main is only a partial mitigation: Opus 4.7 defaults off and global visibility config exists, but the broader non-off API-thinking defaults, default-agent cron inheritance gap, and Teams reasoning suppression gap remain.
