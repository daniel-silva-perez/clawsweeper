## Summary

Keep open. Current main still promotes unset reasoning visibility from `off` to the selected model's reasoning capability, and Anthropic Claude models are treated as reasoning-capable. The requested global `agents.defaults.reasoningDefault` is still not schema-backed, `/status` can surface the promoted `Reasoning: on`, and the Teams-specific reply dispatcher still lacks a direct `isReasoning` suppression guard. The exact Anthropic billing impact still needs a transport-level payload regression test, but the reported default/visibility problem remains current.

## Best Possible Solution

Keep this issue open. The best fix is in the core reply/model-selection path: unset reasoning visibility should remain `off` unless the user explicitly configures it or provider metadata declares a true default-on/mandatory reasoning policy. If maintainers accept the global knob, add schema, generated help, docs, and resolution support for `agents.defaults.reasoningDefault` and coordinate that narrower config work with #13607. Add Anthropic Claude unset-config regression coverage for both request payload thinking behavior and reply delivery, and add a Teams suppression test/guard for `isReasoning` payloads.

## Risks / Open Questions

- The exact Anthropic spend increase should be proven with a provider transport/request-payload regression test because API thinking is also controlled through `thinkLevel`, not only `reasoningLevel`.
- OpenRouter currently has changelog-backed behavior that defaults reasoning on for `reasoning: true`; a fix should separate support/default/mandatory metadata instead of blanket-removing provider hooks.
- Channel-specific dispatchers that bypass the generic suppression path, especially Microsoft Teams, need explicit `isReasoning` handling if reasoning payloads must stay hidden by default.
