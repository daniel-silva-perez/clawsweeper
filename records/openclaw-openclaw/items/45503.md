## Summary

Keep open. Current main still has automatic cache-TTL pruning, overflow-triggered tool-result truncation, context-engine compaction hooks, and whole-session manual compaction, but no user- or agent-addressable clear/archive/recover operation for a specific completed tool result. The comment discussion raises real design choices around soft-clearing, recovery, and checkpoint semantics, so this should stay with maintainer product/API review rather than be swept as stale or queued as a narrow repair.

## What This Changes

The issue requests a way to manually purge or archive completed tool results from active model context before automatic TTL pruning runs.

## Best Possible Solution

Define a core prompt-view clear/archive mechanism for completed tool results that preserves raw transcript history and strict provider pairing, exposes intentional user and/or agent controls, supports recoverable placeholders or durable lookup, and documents how it interacts with /context, /compact, pruning, context engines, and session replay.

## Reproduction Assessment

Not applicable as a runtime bug report; this is a feature gap. The gap is reproducible by source inspection on current main: pruning is TTL/policy-driven, /context is inspect-only, /compact is whole-session scoped, and no targeted tool-result clear/archive/recover selector exists.

## Solution Assessment

Unclear. The proposed CLI command, agent function, and clearAfterUse flag are plausible entry points, but the safest maintainable solution is a prompt-view archive/clear design that preserves transcript and provider replay invariants instead of physically deleting history.
