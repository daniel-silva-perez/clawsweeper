## Summary

Keep open. Current main still only provides automatic cache-TTL pruning, live tool-result truncation, overflow recovery, and whole-session compaction; it does not expose a user- or agent-addressable way to clear, archive, or recover a specific completed tool result. The discussion adds real design concerns around archive/recover behavior and compaction checkpoints, so this should stay with maintainer product/API review rather than stale cleanup or a narrow autonomous patch.

## What This Changes

Request to add a user- or agent-accessible way to manually purge or archive completed tool results from model context before automatic TTL pruning runs.

## Best Possible Solution

Define a core prompt-view clear/archive mechanism for completed tool results that preserves raw transcript history and strict provider pairing, exposes intentional user and/or agent controls, supports recoverable placeholders or durable lookup, and documents how it interacts with /context, /compact, pruning, context engines, and session replay.

## Reproduction Assessment

Not applicable as a runtime bug report; this is a feature gap. The gap is reproducible by source inspection on current main: pruning is TTL/policy-driven, /context is inspect-only, /compact is whole-session scoped, and no targeted tool-result clear/archive/recover selector exists.

## Solution Assessment

Unclear. The proposed CLI command, agent function, and clearAfterUse flag are plausible entry points, but the safer maintainable solution is a core prompt-view archive/clear design that preserves transcript and replay invariants instead of deleting history directly.
