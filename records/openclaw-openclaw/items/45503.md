## Summary

Current main still has automatic cache-TTL context pruning, tool-result truncation, context-engine maintenance, and whole-session /compact, but I did not find a manual user- or agent-addressable way to purge a specific completed tool result or mark a result clear-after-use. The issue should stay open for a maintainer product/API decision.

## Best Possible Solution

Keep this open for maintainer product/API design. The conservative direction is a recoverable prompt-view clear/archive mechanism for completed tool results that preserves raw transcript/provider pairing, exposes a deliberate user or agent control surface, and adds docs plus focused pruning, /context, /compact, provider replay, and recovery tests.
