## Summary

Keep this issue open. Current main has the underlying model infrastructure: explicit primary/fallback config, chat/UI model switching, and a configure-time multi-select for the model allowlist. The specific onboarding request is still not implemented: `openclaw onboard` prompts for one default model and applies only that selected primary model, while the multi-select allowlist/fallback flow is wired into configure rather than the onboarding wizard.

## Best Possible Solution

Keep this open for maintainer product direction. The narrow implementation path is to reuse `promptModelAllowlist`, `applyModelFallbacksFromSelection`, and `applyModelAllowlist` in `src/wizard/setup.ts` after provider auth/default-model selection, then add onboarding tests and docs. If maintainers want true multi-provider setup during first run, this needs a broader onboarding UX decision before implementation.
