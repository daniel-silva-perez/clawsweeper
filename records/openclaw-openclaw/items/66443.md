## Summary

Keep open. Current main still routes overflow tool-result recovery through a transcript rewrite helper that branches at the first replacement and re-appends the remaining active-branch suffix without an identity or hash dedupe. Maintainer discussion also keeps this as the concrete Track A issue under #69208, and #72511 remains the explicit open fix candidate.

## What This Changes

The issue asks OpenClaw to make overflow recovery and transcript rewrite persistence idempotent so prior user messages, compactions, and bootstrap context records are not duplicated in session JSONL.

## Best Possible Solution

Keep this issue open as the Track A repro until #72511 or an equivalent narrow fix lands in the transcript rewrite path with regression coverage for duplicate user messages, cloned compactions, and repeated bootstrap custom records.

## Reproduction Assessment

Yes. The report includes concrete JSONL duplicate hashes and timestamps, and current-main code tracing shows overflow recovery reaches a suffix replay helper with no duplicate suppression; a focused reproduction is to trigger overflow tool-result truncation and inspect the raw session JSONL for duplicate message/custom/compaction entries.

## Solution Assessment

Yes. A rewrite-boundary idempotency invariant in src/agents/pi-embedded-runner/transcript-rewrite.ts is the narrow maintainable first fix, with broader duplicate transcript work staying under #69208.
