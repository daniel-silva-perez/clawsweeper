## Summary

Keep open. Current main still replays every active-branch suffix entry in `rewriteTranscriptEntriesInSessionManager` without an idempotency check for duplicate user messages, compaction entries, or bootstrap custom entries. The overflow-recovery tool-result truncation path still calls that rewrite helper, the current transcript-rewrite tests still assert suffix replay rather than duplicate suppression, and PR #72511 remains open and unmerged as the explicit candidate fix for this issue.

## Best Possible Solution

Keep this issue open as the concrete Track A repro under #69208. The best path is to review and land PR #72511, or an equivalent fix, with regression coverage that forces the overflow recovery/transcript rewrite path and asserts raw session JSONL does not gain duplicate `role=user` rows, cloned compaction entries, or repeated `openclaw:bootstrap-context:full` custom records.
