## Summary

Keep this PR open. Current main still does not implement the PR's central loop-detection changes: loop detection remains disabled by default, `generic_repeat` remains warning-only, and exec approval resolution still lets the host approvals ask mode win via `maxAsk`. The overlapping #70877 and #54559 are still open and cover only part of this PR, so they are not a shipped replacement.

## Best Possible Solution

Keep the PR open for maintainer review, or ask the author/maintainer to split it into focused pieces: loop detection default and `generic_repeat` critical handling, exec approval semantics, sandbox-policy file allowlisting, and trailing error-turn recovery. The loop-detection subset can also be coordinated with #70877 and #54559, but this PR should not be closed until a merged implementation covers its intended behavior or a maintainer chooses a narrower canonical path.
