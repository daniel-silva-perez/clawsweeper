## Summary

Current main still has the narrow channel-awareness gap identified in the discussion: ordinary agent runs include channel metadata, but slash-command prompt reconstruction for `/context` and `/export-session` still omits channel/capabilities/shell/channelActions and uses unknown host/os/arch placeholders. The linked fix PR #21271 is closed unmerged, so this issue should stay open as a focused fix candidate.

## Best Possible Solution

Keep this issue open and implement the missing runtime metadata only in `resolveCommandsSystemPromptBundle()`. Reuse or extract the embedded-runner pattern so `/context` and `/export-session` prompts include normalized channel, account-aware capabilities, shell, machine/os details, and channel actions. Treat selective cross-channel reply routing as a separate product/API decision.
