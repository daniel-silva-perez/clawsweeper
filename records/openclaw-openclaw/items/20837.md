## Summary

Keep open. Current main already carries channel metadata through ordinary agent runs and deterministic origin reply routing, but the slash-command prompt reconstruction used by `/context` and `/export-session` still calls `resolveCommandsSystemPromptBundle()` with hardcoded `host/os/arch` values and without `channel`, `capabilities`, `shell`, or `channelActions`. The linked fix PR #21271 is closed unmerged, and no open PR with closing syntax for this issue was found.

## Best Possible Solution

Keep this issue open and implement the missing runtime metadata only in the slash-command prompt reconstruction path. Reuse or extract the embedded-runner runtime metadata pattern so `/context` and `/export-session` prompts include normalized channel, account-aware capabilities, shell, machine/os details, reaction guidance where applicable, and channel actions. Treat prompt-driven selective cross-channel reply routing as a separate product/API decision.
