## Summary

Keep open: the reported Telegram ACP-bound DM topic bug still appears valid on current main, and the proposed branch cannot fix the active runtime because it modifies the retired `src/telegram` tree instead of the bundled Telegram plugin under `extensions/telegram/src`. The PR is useful source work, but it needs to be ported before merge.

## What This Changes

The PR adds a route-level DM-topic suffix guard and regression tests in `src/telegram` so Telegram ACP-bound DM topic follow-ups and native commands keep the bound ACP session key.

## Best Possible Solution

Port the bound-route authority signal into the active Telegram plugin resolver and consume it anywhere DM-topic thread suffixes are derived, with tests proving bound DM topics keep the ACP target key while unbound DM topics keep isolated thread sessions.
