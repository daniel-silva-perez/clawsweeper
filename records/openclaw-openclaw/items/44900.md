## Summary

Keep open. Current main still has the Telegram DM-topic suffixing behavior after bound ACP route resolution, and native command plus session-state lookups can still derive a suffixed key instead of using the bound ACP target key. The PR is useful source work, but it is stale against the current plugin layout and needs to be ported before merge.

## What This Changes

The PR adds a bound-route DM suffix guard to Telegram conversation routing, applies it to inbound messages, native commands, and session-state resolution, and adds regression coverage for bound ACP DM-topic follow-ups.

## Best Possible Solution

Port the bound-route suffix guard into the current Telegram plugin implementation, preserve unbound DM-topic isolation, and add regression coverage proving bound DM-topic inbound messages, native commands, and session-state lookups use the exact ACP target session key.
