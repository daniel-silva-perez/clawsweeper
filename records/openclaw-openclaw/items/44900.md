## Summary

Keep this PR open, but it should not merge as written. The reported Telegram ACP DM-topic session-key bug is still plausible in current main, while the PR patch targets the retired `src/telegram` tree instead of the active bundled Telegram plugin under `extensions/telegram/src`.

## What This Changes

The PR adds a Telegram route-level `skipDmThreadSuffix` flag and regression tests under `src/telegram` so bound DM-topic messages and native commands avoid appending an extra thread suffix.

## Best Possible Solution

Port the bound-route authority signal into `extensions/telegram/src/conversation-route.ts` and consume it anywhere the active Telegram plugin derives DM-topic thread session keys, with regressions proving bound DM topics keep the ACP target key while unbound DM topics retain isolated thread sessions.

## Reproduction Assessment

Yes. The provided PR steps describe the runtime failure, and current main shows a source-level reproduction: bound route resolution can return an ACP target session key, then active Telegram DM-topic message/native-command paths append a second thread suffix.

## Solution Assessment

No, this PR is not the best implementation as written because it changes inactive `src/telegram` files. The narrow route-level guard is a good solution direction once applied to the active `extensions/telegram/src` plugin tree and covered there.
