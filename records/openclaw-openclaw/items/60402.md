## Summary

Keep this PR open. Current main still blocks targeted parent-session wake delivery when heartbeat scheduling is globally disabled or has no scheduled agents, while this PR is a focused implementation candidate for that narrowed ACP gap; the prior P1/P2 review concerns were addressed in follow-up commits, but it still needs maintainer review and rebase/validation against current main.

## What This Changes

The PR updates heartbeat runner scheduling so targeted session-scoped event wakes, including ACP spawn stream events, can run while periodic heartbeat scheduling is disabled, keeps disabled agents blocked, resumes interval scheduling on re-enable, and adds scheduler regression coverage.

## Best Possible Solution

Adapt and land the narrow heartbeat-disabled targeted wake fix if maintainers agree with the scheduler approach, then keep the broader yielded-parent completion work tracked separately as a shared ACP/native completion delivery path that does not depend on optional periodic heartbeat scheduling.

## Reproduction Assessment

Yes. The #52249 discussion gives a heartbeat-disabled ACP streamTo:"parent" reproduction, and current main statically shows the parent relay enqueues a wake that the runner can skip before targeted handling when global heartbeat scheduling or intervals are disabled.

## Solution Assessment

Unclear for the whole paired issue, but yes for the narrow ACP wake gap this PR targets. The patch keeps the bypass limited to session-scoped event reasons and keeps disabled agents blocked; the safer broader endpoint is a shared yielded-parent completion primitive for ACP and native child runs.
