## Summary

Keep open. The IRC reliability fixes are still useful because current main retains the password-only NickServ IDENTIFY, silent sanitized-empty PRIVMSG drop, and immediate transient QUIT behavior, but this PR should not merge as-is because its closed-send guard breaks graceful QUIT delivery and its delayed NickServ reclaim timers outlive client shutdown.

## What This Changes

The PR changes the bundled IRC plugin client, transient send path, and NickServ tests to throw on empty or closed sends, delay transient shutdown, use account-qualified NickServ IDENTIFY, and reclaim fallback nicks with delayed GHOST/NICK commands.

## Best Possible Solution

Merge a repaired IRC plugin change that keeps the useful empty-send visibility, closed-socket detection, transient drain delay, account-qualified IDENTIFY, and fallback-nick reclaim behavior while preserving graceful QUIT delivery, binding delayed reclaim timers to client lifetime, and covering those paths with focused tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve QUIT when guarding closed sends:** `extensions/irc/src/client.ts:175-177`
  - body: The new sendRaw guard rejects whenever `closed` is true, but `quit()` sets `closed = true` before calling `sendRaw("QUIT ...")`. Every graceful shutdown now throws inside `quit()`, swallows the error, and closes the socket without sending a QUIT frame. Move the closed transition until after QUIT is written, or let the shutdown frame bypass this guard.
  - confidence: 0.94
- **[P2] Tie reclaim timers to client shutdown:** `extensions/irc/src/client.ts:343-360`
  - body: The delayed NickServ GHOST/NICK `setTimeout` calls are not stored, cleared, or unref'd. Once scheduled, they can keep one-shot IRC sends alive after `quit()` and can fire against a client that has already been closed. Track these timers and clear or unref them as part of shutdown.
  - confidence: 0.86
- **[P3] Avoid duplicate NickServ reclaim attempts:** `extensions/irc/src/client.ts:343-360`
  - body: This delayed reclaim block can run after `tryRecoverNickCollision()` already sent GHOST/NICK during collision recovery. Gate the post-identify reclaim path on that attempt state so slow or rejected first attempts do not send duplicate NickServ GHOST commands unnecessarily.
  - confidence: 0.74
