## Summary

Keep open. Current main still has the IRC reliability and NickServ fallback gaps this PR is trying to address, but the proposed patch is not mergeable yet because it breaks graceful QUIT delivery and leaves delayed reclaim timers outside the client lifecycle; existing review comments independently flagged the same blockers.

## What This Changes

The PR changes the bundled IRC plugin client, transient send path, and NickServ tests to surface empty or closed sends, add a transient drain delay, use account-qualified NickServ IDENTIFY, and schedule fallback-nick GHOST/NICK reclaim.

## Best Possible Solution

Merge a repaired IRC plugin change that keeps the useful send visibility, transient drain, account-qualified IDENTIFY, and fallback-nick reclaim behavior while preserving graceful QUIT delivery, tying delayed reclaim work to client shutdown, and adding focused coverage for the changed client/send paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve QUIT when guarding closed sends:** `extensions/irc/src/client.ts:175-177`
  - body: The new sendRaw guard rejects whenever closed is true, but quit() sets closed = true before it calls sendRaw("QUIT ..."). Every graceful shutdown now throws inside quit(), swallows the error, and closes the socket without sending a QUIT frame. Move the closed transition until after QUIT is written, or let the shutdown frame bypass this guard.
  - confidence: 0.94
- **[P2] Tie reclaim timers to client shutdown:** `extensions/irc/src/client.ts:354-369`
  - body: The delayed NickServ GHOST/NICK setTimeout calls are not stored, cleared, or unref'd. Once scheduled, they can keep a one-shot IRC send alive after quit() and can fire against a client that has already been closed. Track these timers and clear or unref them as part of shutdown.
  - confidence: 0.86
