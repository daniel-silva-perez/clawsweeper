## Summary

Keep open. Current main still has the IRC reliability gaps this PR targets, but the PR as written introduces a graceful-quit regression and leaves delayed NickServ reclaim timers outside the client lifetime; the patch needs repair before merge. The security pass did not find unrelated workflow, dependency, lockfile, package, or script changes.

## What This Changes

The PR changes the bundled IRC plugin client, transient send path, and NickServ tests to throw on empty/closed sends, delay transient shutdown, use account-qualified NickServ IDENTIFY for fallback nicks, and schedule post-identify GHOST/NICK reclaim.

## Best Possible Solution

Merge a repaired IRC plugin change that preserves the useful empty-send visibility, closed-socket detection, transient drain delay, account-qualified NickServ IDENTIFY, and fallback-nick reclaim behavior, while fixing QUIT ordering, tying delayed reclaim timers to client shutdown, and adding focused client/send coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve QUIT when guarding closed sends:** `extensions/irc/src/client.ts:166-168`
  - body: The new sendRaw guard rejects whenever closed is true, but quit() sets closed = true before it calls sendRaw("QUIT ..."). Every graceful shutdown now throws inside quit(), swallows the error, and closes the socket without sending a QUIT frame. Move the closed transition until after QUIT is written, or bypass this guard for the shutdown frame.
  - confidence: 0.94
- **[P2] Tie reclaim timers to client shutdown:** `extensions/irc/src/client.ts:343-360`
  - body: The delayed NickServ GHOST/NICK setTimeout calls are not stored, cleared, or unref'd. Once scheduled, they can keep one-shot IRC sends alive after quit() and can fire against a client that has already been closed. Track these timers and clear or unref them as part of quit()/close().
  - confidence: 0.86
