## Summary

Keep open. Current main still has the IRC reliability gaps this PR is trying to fix, but the proposed patch introduces concrete shutdown/timer regressions that should block merge until repaired.

## What This Changes

The PR updates bundled IRC client, send, and NickServ test code to surface dropped outbound sends, delay transient disconnects, use account-qualified NickServ IDENTIFY, and reclaim fallback nicks with delayed GHOST/NICK commands.

## Best Possible Solution

Merge a repaired IRC plugin change that keeps the useful send visibility, dead-socket detection, transient drain delay, account-qualified NickServ identify, and fallback-nick reclaim behavior while preserving graceful QUIT delivery and binding delayed reclaim work to client lifetime.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve QUIT when guarding closed sends:** `extensions/irc/src/client.ts:169-171`
  - body: The new `sendRaw` guard rejects whenever `closed` is true, but `quit()` sets `closed = true` before it sends `QUIT` through `sendRaw`. Every graceful shutdown now swallows that error and closes the socket without sending the QUIT frame. Move the closed transition until after QUIT is written, or let the shutdown frame bypass this guard.
  - confidence: 0.94
- **[P2] Tie reclaim timers to client shutdown:** `extensions/irc/src/client.ts:348-367`
  - body: The delayed NickServ GHOST/NICK timers are not stored, cleared, or `unref()`'d. Once scheduled, they can keep one-shot IRC sends alive after `quit()` and can fire against a client that has already been closed. Track them and clear or unref them during `quit()`/`close()`.
  - confidence: 0.86
- **[P3] Avoid duplicate NickServ reclaim attempts:** `extensions/irc/src/client.ts:348-367`
  - body: The delayed reclaim path can run after `tryRecoverNickCollision()` already attempted GHOST/NICK during collision recovery. Gate the post-identify path on that attempt state, or otherwise prevent a second noisy NickServ GHOST for the same desired nick.
  - confidence: 0.73
