## Summary

Keep open. Current main has partial Discord proxy fixes, including custom REST fetch injection and the merged multipart attachment fix from #71383, but it does not yet contain the broader proxy-fetch normalization needed for the reported final JSON reply and slash-command Content-Type failures. The remaining work is actively represented by open PR #66243, which changes `src/infra/net/proxy-fetch.ts` to flatten Request inputs so method, headers, and body are preserved through the undici proxy dispatcher.

## Best Possible Solution

Keep this issue open until #66243 or an equivalent scoped fix lands. The best fix belongs in `src/infra/net/proxy-fetch.ts`: normalize Request/Request-like inputs before adding the undici dispatcher, preserve method/headers/body/signal semantics, keep hop-by-hop header stripping, and add regression coverage for Discord REST JSON Content-Type delivery as well as multipart FormData through the proxy path. After that lands and ships, close this as implemented with the fixing commit and release.
