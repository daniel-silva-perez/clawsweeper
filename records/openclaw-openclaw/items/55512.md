## Summary

Keep open. Current main still sizes code-mode Markdown table columns with JavaScript string length, so CJK cells remain under-padded, and an open draft PR already references this issue with closing syntax; the issue should stay open until that PR or a replacement lands.

## What This Changes

The request is to make Markdown table delivery on Telegram and Discord use display-width-aware alignment for CJK text, with Discord avoiding unsupported raw GFM table rendering.

## Best Possible Solution

Review and land PR #55596 or an equivalent replacement that fixes display-width calculation in the shared Markdown code-table renderer, adds CJK and emoji/grapheme regression tests, and confirms Telegram plus Discord delivery paths consistently receive the corrected table output.
