## Summary

Keep open. Current main has a shared Markdown-table-to-code path that partially addresses Discord raw pipe output, but the active renderer still computes column widths and padding with UTF-16 string length, so CJK cells can still misalign. PR #55596 is an open draft implementation candidate tied to this issue with GitHub closing syntax, so the issue should remain open until that PR or a replacement lands.

## Best Possible Solution

Keep this issue open and review PR #55596 or a replacement that adds display-width-aware table rendering in the shared Markdown table path, with CJK regression coverage for Telegram and Discord. After that implementation merges and ships, close as implemented on main.
