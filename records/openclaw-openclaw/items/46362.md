## Summary

Keep open: current main does not implement spoken Discord voice model switching, and this PR is not merge-ready because the voice changes target the old core Discord voice path, use broad/unconsumed transcript triggers, and hardcode persistent cloud model targets. The unrelated gateway HTTP registry part has already landed on current main via #47902.

## What This Changes

This PR adds spoken Discord voice model-switch triggers in `src/discord/voice/manager.ts` and also adds a global gateway plugin HTTP route registry bridge with regression tests.

## Best Possible Solution

If maintainers want this capability, rebuild it in `extensions/discord/src/voice` as a focused Discord plugin change with explicit low-false-positive voice commands, command consumption before agent dispatch, correct session creation/update behavior, configured or allowlisted model targets, focused voice tests, and Discord docs. Leave the gateway route registry on the merged #47902 implementation path.

## Reproduction Assessment

Yes for the review findings: the PR diff and review comments show the broad trigger, missing command consumption, and missing-session early return directly. Current main lacks the spoken-trigger feature, so live reproduction of the intended behavior would require a rebased PR branch.

## Solution Assessment

No. This PR is not the best way to solve the request because it edits a retired path, hardcodes Anthropic model targets, mixes unrelated registry work, and lets spoken commands continue as normal prompts; an extension-scoped implementation with explicit configured targets is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Require explicit voice mode commands:** `src/discord/voice/manager.ts:84-85`
  - body: The medium-tier pattern matches ordinary speech such as "think about this", so a normal utterance can silently downgrade the persistent session model. Use explicit command phrases with low false-positive risk before changing model state from voice transcripts.
  - confidence: 0.91
- **[P2] Consume matched voice commands before dispatch:** `src/discord/voice/manager.ts:692-693`
  - body: After applying an escalation trigger, the code continues into `agentCommandFromIngress`, so a pure command like "go deep" both changes the model and is sent to the agent as a normal prompt. Return a boolean from the trigger handler and skip dispatch when it matches.
  - confidence: 0.9
- **[P2] Persist overrides for missing session entries:** `src/discord/voice/manager.ts:783-785`
  - body: The session-store update returns when the session entry is absent, so the first spoken escalation in a fresh voice session can be dropped before the later agent command path creates the session entry. Create or update the target session entry when applying the override.
  - confidence: 0.84
- **[P3] Allow clearing the gateway registry without casts:** `src/plugins/runtime.ts:58-59`
  - body: The new getter returns `PluginRegistry | null`, but the setter only accepts `PluginRegistry`, forcing test cleanup to use `null as never`. Widen the setter parameter if this registry API remains in the branch.
  - confidence: 0.86

## Security Review
