## Summary

Keep open. Current main still lacks the spoken Discord voice model-escalation behavior, but this PR is not merge-ready because it targets the old core Discord voice path, mixes in gateway registry work that is already solved on main, and introduces concrete voice-command UX and persistence defects plus a cloud-routing policy risk.

## What This Changes

This PR adds hardcoded spoken Discord voice model-switch triggers in the old core voice manager and also adds a global plugin HTTP route registry bridge with regression tests.

## Best Possible Solution

If maintainers want this capability, rebuild it as a focused Discord plugin change under extensions/discord/src/voice with explicit low-false-positive commands, command consumption before agent dispatch, correct session creation/update behavior, configured or allowlisted model targets, focused voice tests, and Discord docs; leave the gateway registry work on the already-merged #47902 path.

## Reproduction Assessment

Yes for the review findings: the PR diff shows the broad trigger, missing command consumption, and missing-session early return directly. Live Discord testing would still be useful after the command semantics and model-target policy are fixed.

## Solution Assessment

No. The current PR is not the best way to solve the issue because it edits the retired core voice path, hardcodes Anthropic model targets, mixes unrelated registry work, and leaves spoken commands flowing through as normal prompts; an extension-scoped rewrite is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Require explicit voice mode commands:** `src/discord/voice/manager.ts:84-85`
  - body: The medium-tier pattern matches ordinary speech such as "think about this", so a normal utterance can silently downgrade the persistent session model. Use explicit command phrases with low false-positive risk before changing model state from voice transcripts.
  - confidence: 0.91
- **[P2] Consume matched voice commands before dispatch:** `src/discord/voice/manager.ts:692-693`
  - body: After applying an escalation trigger, the code continues into agentCommandFromIngress, so a pure command like "go deep" both changes the model and is sent to the agent as a normal prompt. Return a boolean from the trigger handler and skip dispatch when it matches.
  - confidence: 0.9
- **[P2] Persist overrides for missing session entries:** `src/discord/voice/manager.ts:783-785`
  - body: The session-store update returns when the session entry is absent, so the first spoken escalation in a fresh voice session can be dropped before the later agent command path creates the session entry. Create or otherwise update the target session entry when applying the override.
  - confidence: 0.84
- **[P3] Allow clearing the gateway registry without casts:** `src/plugins/runtime.ts:58-59`
  - body: The new getter returns PluginRegistry | null, but the setter only accepts PluginRegistry, forcing test cleanup to use null as never. Widen the setter parameter if this registry API remains in the branch.
  - confidence: 0.86
