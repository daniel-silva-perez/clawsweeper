## Summary

Keep open for maintainer review. Current main supports a configured static Discord voice model override, but it does not implement spoken escalation triggers; this PR is also stale against the current bundled Discord plugin layout, mixes in gateway registry work already merged by #47902, and has blocking functional and security concerns in the remaining voice feature.

## What This Changes

The PR adds spoken Discord voice model-switch triggers in `src/discord/voice/manager.ts` and also adds a gateway plugin HTTP registry bridge with regression tests.

## Best Possible Solution

If maintainers want this capability, rebuild it in `extensions/discord/src/voice` as a focused Discord plugin change with explicit low-false-positive voice commands, command consumption before agent dispatch, correct session creation/update behavior, configured or allowlisted model targets, focused voice tests, and Discord docs. Leave gateway HTTP-route registry behavior on the current #47902 implementation path.

## Reproduction Assessment

Yes for the PR review issues: the diff directly shows the broad trigger, missing early return after trigger handling, and missing-session no-op in the store update. Current main cannot reproduce the intended spoken-trigger behavior because that feature is absent and the PR branch is stale.

## Solution Assessment

No. This PR is not the best way to solve the request because it edits a retired path, hardcodes Anthropic model targets, mixes unrelated already-merged gateway work, and lets spoken commands continue as normal prompts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Require explicit voice mode commands:** `src/discord/voice/manager.ts:84-85`
  - body: The medium-tier pattern matches ordinary speech such as "think about this", so normal conversation can silently downgrade the persistent session model. Use explicit command phrases with low false-positive risk before changing model state from voice transcripts.
  - confidence: 0.91
- **[P2] Consume matched voice commands before dispatch:** `src/discord/voice/manager.ts:692-693`
  - body: After applying an escalation trigger, execution continues into `agentCommandFromIngress`, so a phrase like "go deep" both changes the model and is sent to the agent as a normal prompt. Return a boolean from the trigger handler and skip dispatch when it matches.
  - confidence: 0.9
- **[P2] Persist overrides for missing session entries:** `src/discord/voice/manager.ts:783-785`
  - body: The session-store update returns when the session entry is absent, so the first spoken escalation in a fresh voice session can be dropped before the later agent command path creates the entry. Create or update the target session entry when applying the override.
  - confidence: 0.84
- **[P3] Allow clearing the gateway registry without casts:** `src/plugins/runtime.ts:58-59`
  - body: The new getter returns `PluginRegistry | null`, but the setter only accepts `PluginRegistry`, forcing test cleanup to use `null as never`. Widen the setter parameter if this registry API remains in the branch.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff introduces persistent cloud model routing from transcribed speech without configured targets or sufficiently explicit commands.

Concerns:

- **[medium] Avoid hardcoded cloud routing from incidental speech:** `src/discord/voice/manager.ts:78`
  - body: A matched transcript can persistently switch future Discord voice turns to hardcoded Anthropic models without an explicit configured target or allowlist decision. That can change privacy, cost, and provider-policy behavior from incidental speech.
