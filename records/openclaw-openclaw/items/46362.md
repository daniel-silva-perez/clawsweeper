## Summary

Keep open. Current main does not implement the PR's central spoken Discord voice model escalation behavior, but the patch is not merge-ready because it targets the old core Discord voice path, bundles a gateway registry fix already superseded by #47902, and introduces blocking UX/security issues in the spoken trigger handling.

## What This Changes

This PR adds hardcoded spoken Discord voice model-switch commands and a global gateway plugin HTTP registry bridge with regression tests.

## Best Possible Solution

If maintainers want this capability, rebuild it as a fresh narrow Discord plugin change under extensions/discord/src/voice with explicit low-false-positive commands, command consumption before agent dispatch, correct session creation/update behavior, configured or allowlisted model targets, focused tests, and Discord docs; leave the gateway registry work to the already-merged #47902 path.

## Reproduction Assessment

Yes for the review findings: the blocking issues are visible in the provided PR diff and current-main code inspection without needing live Discord. A live voice test would still be useful after the command semantics and model-target policy are fixed.

## Solution Assessment

No. The current PR is not the best way to solve the issue because it edits the old core voice path, mixes unrelated registry work, hardcodes cloud model targets, and does not consume matched voice commands; a fresh extension-scoped implementation would be safer and easier to review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Require explicit voice mode commands:** `src/discord/voice/manager.ts:84-85`
  - body: The medium-tier pattern matches ordinary speech such as "think about this", so a normal utterance can silently downgrade the persistent session model. Use explicit command phrases with low false-positive risk before changing model state from voice transcripts.
  - confidence: 0.91
- **[P2] Consume matched voice commands before dispatch:** `src/discord/voice/manager.ts:692-693`
  - body: After applying an escalation trigger, the code continues into agentCommandFromIngress, so a pure command like "go deep" both changes the model and is sent to the agent as a normal user prompt. Return a boolean from the trigger handler and skip dispatch when it matches.
  - confidence: 0.9
- **[P2] Persist overrides for missing session entries:** `src/discord/voice/manager.ts:783-785`
  - body: The session-store update returns when the session entry is absent, so the first spoken escalation in a fresh voice session can be dropped before the later agent command path creates the session entry. Create or otherwise update the target session entry when applying the override.
  - confidence: 0.84
- **[P3] Allow clearing the gateway registry without casts:** `src/plugins/runtime.ts:58-59`
  - body: The new getter returns PluginRegistry | null, but the setter only accepts PluginRegistry, forcing test cleanup to use null as never. Widen the setter parameter if this registry API remains in the branch.
  - confidence: 0.86
