## Summary

Keep open, but do not merge as-is. The central voice-triggered model-switching behavior is not present on current main, while this branch is stale against the moved Discord voice plugin code, contains concrete UX/runtime bugs, and mixes in gateway registry work that current main already handles separately.

## What This Changes

The PR adds spoken Discord voice model-switch triggers in `src/discord/voice/manager.ts` and also adds a plugin HTTP route registry bridge with tests.

## Best Possible Solution

If maintainers want this capability, rebuild it as a focused Discord plugin change under `extensions/discord/src/voice` with explicit low-false-positive commands, command consumption before dispatch, correct missing-session persistence, configured or allowlisted model targets, focused tests, and Discord docs. Leave gateway HTTP-route registry behavior on the already-merged pinned-registry implementation path.

Yes for the PR review issues: the provided diff and comments show the broad trigger, missing command consumption, and missing-session no-op. Current main cannot reproduce the intended spoken-trigger behavior because that feature is absent from the current Discord voice plugin.

## Solution Assessment

No. This PR is not the best way to solve the request because it edits a retired path, hardcodes Anthropic model targets, mixes unrelated already-merged gateway work, and forwards matched voice commands as normal prompts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Require explicit voice mode commands:** `src/discord/voice/manager.ts:84-85`
  - body: The medium-tier pattern matches ordinary speech such as `think about this`, so normal conversation can silently downgrade the persistent session model. Use explicit command phrases with low false-positive risk before changing model state from voice transcripts.
  - confidence: 0.91
- **[P2] Consume matched voice commands before dispatch:** `src/discord/voice/manager.ts:692-693`
  - body: After applying an escalation trigger, execution continues into `agentCommandFromIngress`, so a phrase like `go deep` both changes the model and is sent to the agent as a normal prompt. Return a boolean from the trigger handler and skip dispatch when it matches.
  - confidence: 0.9
- **[P2] Persist overrides for missing session entries:** `src/discord/voice/manager.ts:783-785`
  - body: The session-store update returns when the session entry is absent, so the first spoken escalation in a fresh voice session can be dropped before the later agent command path creates the entry. Create or update the target session entry when applying the override.
  - confidence: 0.84
- **[P3] Allow clearing the gateway registry without casts:** `src/plugins/runtime.ts:58-59`
  - body: The new getter returns `PluginRegistry | null`, but the setter only accepts `PluginRegistry`, forcing test cleanup to use `null as never`. Widen the setter parameter if this registry API remains in the branch.
  - confidence: 0.86
