## Summary

Keep open. The PR addresses a real gap in current main, but the proposed fallback still changes dispatch normalization in a way that can coerce an unavailable raw tool name into a different allowlisted tool. That needs maintainer review and regression tests before merge.

## What This Changes

The PR adds allowlist-backed reverse lookup in the embedded runner tool-call normalizer for underscore-stripped, dot-prefixed, dropped-leading-character, and digit-bearing hex-suffixed malformed tool names.

## Best Possible Solution

Land a narrow embedded-runner normalization fix that recovers only allowlisted malformed tool names when there is strong structural evidence, preserves existing delimiter and alias behavior, fails closed on exact, prefix, underscore-stripped, dropped-leading-character, and cross-plugin collisions, and includes focused regression tests for both recovery and non-recovery cases.
