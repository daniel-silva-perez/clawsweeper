## Summary

Keep this PR open. Current main still lacks the allowlist-backed reverse lookup for malformed hex-suffixed, underscore-stripped, dot-prefixed, and dropped-leading-character tool names, while this PR is a focused implementation candidate for that dispatch path.

## Best Possible Solution

Keep this PR open as an implementation candidate. Maintainers should review or rework the fallback in src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts, keep every recovery path allowlist-backed and fail-closed on ambiguity, and add targeted tests for read5f3787893b0d4c49a, functionsread5f3787893b0d4c49a, blacklist1a2b3c, readfeed, underscore-stripped and dot-prefixed variants, dropped-leading-character collisions, and foo_bar/foobar collisions before landing.
