## Summary

Keep this PR open. Current main still lacks the allowlist-backed reverse lookup needed for malformed hex-suffixed, underscore-stripped, dot-prefixed, and dropped-leading-character tool names, and the PR remains a focused implementation candidate, but it should not land without targeted collision and false-positive regression coverage.

## What This Changes

The PR adds a fallback in the embedded runner's tool-call normalization path that reverse-matches malformed streamed tool names against allowed tool names using exact variants and digit-bearing hex-suffix prefix variants.

## Best Possible Solution

Land a narrow embedded-runner normalization fix that recovers only allowlisted malformed tool names, uses strict hash-suffix detection, preserves existing delimiter and alias behavior, and fails closed on every ambiguous exact, prefix, underscore-stripped, and dropped-leading-character collision, with colocated regression tests for both recovery and non-recovery cases.
