## Summary

Keep open. Current main still lacks the reverse lookup this branch targets, so the PR is not obsolete, but the latest head still has a dispatch-safety bug: dropped-leading-character exact matches can turn an unknown raw tool name into a different allowlisted tool. This needs maintainer review and focused regression coverage before merge.

## What This Changes

The PR adds allowlist-backed reverse lookup to the embedded runner tool-call normalizer for underscore-stripped, dot-prefixed, dropped-leading-character, and digit-bearing hex-suffixed malformed streamed tool names.

## Best Possible Solution

Land a narrow embedded-runner normalization fix that recovers malformed tool names only when there is strong structural evidence, preserves the existing delimiter and alias behavior, fails closed on ambiguous exact, prefix, underscore-stripped, and dropped-leading-character cases, and adds colocated regression tests for both recovery and non-recovery examples.
