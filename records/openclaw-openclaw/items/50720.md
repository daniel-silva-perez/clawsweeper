## Summary

Keep this PR open. Current main still does not implement private `sessions_send` delivery: the tool schema has no `deliveryMode` or `announce` parameter, the result metadata remains announce-only, and the A2A flow still runs the final announce/send path without a private-delivery guard. The PR remains a plausible implementation candidate, but it needs maintainer review, rebase cleanup, and separation or justification for the unrelated secrets-test change.

## Best Possible Solution

Keep this PR open. The best path is to rebase it onto current main, keep the change focused on `sessions_send` schema/execution, A2A announce suppression, private-mode prompt guidance, docs, and targeted sessions/gateway tests, and split or explicitly justify the unrelated secrets-test stabilization change. Maintainers should review the privacy semantics carefully before landing.
