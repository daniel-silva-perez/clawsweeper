## Summary

Keep this PR open. Current main still does not implement private `sessions_send` delivery: the tool schema has no `deliveryMode` or `announce` parameter, result delivery metadata is still announce-only, the A2A flow still performs the final announce/send path, and current docs/tests do not cover private suppression. The PR remains a plausible implementation candidate, but needs rebase, focused maintainer review, and split or justification for the unrelated secrets-test change.

## Best Possible Solution

Keep this PR open. The best path is to rebase it onto current main, keep the diff focused on `sessions_send` schema/execution, A2A announce suppression, private-mode prompt guidance, docs, and targeted sessions/gateway tests, and split or explicitly justify the unrelated secrets-test stabilization. Maintainers should require focused regression coverage for default announce behavior, private suppression, precedence, and prompt guidance before landing.
