## Summary

Keep open. Current main still downloads Feishu message resources with one `messageResource.get` call using the caller-provided `type`, so file-typed iOS video resources that fail as `type=file` still have no 502-to-`media` fallback. This PR is a focused implementation candidate for the still-open Feishu/Lark reproduction in #49855, with narrow code/test/changelog scope and no security-sensitive diff surface.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is to land this narrow Feishu plugin fix, or an equivalent maintainer-approved patch, that retries `downloadMessageResourceFeishu` as `media` only for file downloads that fail with HTTP 502, preserves the original 502 when the retry also fails, keeps focused tests in `extensions/feishu/src/media.test.ts`, and includes the user-facing changelog credit for @alex-xuweilong. After merge, #49855 can close as fixed and the duplicate replacement PRs can be closed against the canonical implementation.
