## Summary

Keep this PR open for maintainer review. Current main has some related exec policy precedence behavior, but it does not implement the PR’s central Exec V2 follow-through: Linux/macOS bare allowlist names still match PATH-resolved executable basenames, explicit safeBinTrustedDirs still accept package-manager/user dirs when configured, cat/ls are not formally rejected as safeBins, and the interpreter/script allowlist targeting described by the PR is absent. The PR also touches security-sensitive exec approval paths and contains unrelated Zalo/formatting churn, so it should be reviewed or replaced by a focused exec hardening PR rather than closed as obsolete.

## Best Possible Solution

Keep the PR open or replace it with a focused exec hardening PR. The best path is to land the missing core exec contract work together: align allowlist matching and docs, filter mutable/raw relative safeBinTrustedDirs at runtime plus audit/doctor, formally reject cat/ls safeBins, add interpreter-script allowlist targeting, and keep the Linux contract tests. Remove unrelated Zalo/formatting churn before review, then run the targeted exec/security tests and the changed gate through the normal OpenClaw validation path.
