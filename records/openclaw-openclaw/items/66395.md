## Summary

Keep open. The Control UI refresh-button fix is still relevant on current main, but this PR also includes an unrelated Slack Socket Mode lifecycle change that would make hard startup failures resolve as clean exits under the gateway channel manager.

## What This Changes

The PR disables the Control UI chat refresh button while a chat turn is sending, running, or streaming, and also changes Slack Socket Mode startup/auth failure paths to log and return with a new provider test.

## Best Possible Solution

Land the focused Control UI disabled-state guard for the active-turn refresh bug, then remove the Slack provider changes from this PR or split them into a Slack-specific change that preserves explicit gateway failure status and has status/retry coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep Slack startup failures on the failure path:** `extensions/slack/src/monitor/provider.ts:502-512`
  - body: The new `return` makes `monitorSlackProvider` resolve on permanent startup failures. `server-channels` treats resolved provider tasks as clean exits, records `channel exited without an error`, and schedules restart behavior, so invalid credentials or exhausted startup retries lose the real failure reason. Keep rejecting here or explicitly publish failed runtime status before resolving.
  - confidence: 0.92
