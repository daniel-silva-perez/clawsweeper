## Summary

Keep open. The advertised Control UI fix is still relevant and current main still leaves the refresh button enabled during `chatSending`, `chatRunId`, or active `chatStream`, but this PR also carries an unrelated Slack Socket Mode lifecycle change. That Slack change makes startup/auth failures resolve instead of reject, while the current gateway channel manager treats resolved provider tasks as clean exits and auto-restarts them with a generic error, so the patch should not merge as submitted.

## What This Changes

The PR disables the Control UI chat refresh button while a chat turn is sending, running, or streaming, and also changes Slack Socket Mode startup auth/max-attempt failures to log and return with a new provider test.

## Best Possible Solution

Land a narrow Control UI guard for #65522, then either remove the Slack provider changes from this PR or split them into a separate Slack-focused change that preserves explicit channel failure status and covers auth/max-attempt startup behavior through gateway status tests.

## Review Findings

- **[P2] Keep Slack startup failures on the failure path:** `extensions/slack/src/monitor/provider.ts:502-512`

Keep rejecting here or explicitly set failed runtime status before resolving.
