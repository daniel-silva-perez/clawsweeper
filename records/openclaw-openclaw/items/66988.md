## Summary

Keep this open. Current main still returns `raw: null` when raw redaction cannot safely round-trip, and the Control UI still treats that as Raw mode unavailable. The related direct fix PRs were closed unmerged, while the open #59330/#59336 path overlaps with the observed Raw-mode-disabled regression but does not by itself settle the display-versus-edit safety contract requested here.

## What This Changes

The issue asks for Config UI raw snapshots to remain viewable as redacted read-only raw text when round-trip safety fails, instead of returning `raw: null` and forcing Form mode.

## Best Possible Solution

Define an explicit config snapshot/UI contract that separates raw display availability from raw editability, preserves safely redacted raw text only when disclosure is acceptable, blocks raw save/apply while round-trip safety is false, exposes a clear reason to the UI, and updates gateway, Control UI, docs, and regression tests in coordination with #59330/#59336.
