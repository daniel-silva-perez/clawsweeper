## Summary

Keep this PR open. Current main still silently swallows memory vector probe failures in the status scan path and the memory status rendering has no probe-error field or display. The intended diagnostics improvement remains valid, but this PR is not a conservative close candidate as-is because it bundles the narrow status fix with unrelated Fly.io, Docker, workflow, script, docs, and plugin SDK changes, plus unresolved review feedback and no failing-probe regression test.

## Best Possible Solution

Keep the PR open only as a candidate, or replace it with a focused status-only PR. The best implementation belongs in the current status scan path: add a structured `probeError` field to `MemoryStatusSnapshot`, capture errors from `probeVectorAvailability()` without dropping empty or non-Error throws, render the warning from `buildStatusMemoryValue`, and add a unit test that mocks a failing probe. Split the Fly/Docker/workflow/script/plugin-SDK work into separate owner-reviewed PRs if still wanted.
