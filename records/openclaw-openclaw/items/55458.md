## Summary

Keep open. The memory-probe diagnostic improvement is valid and current main still hides probe failures, but this PR is not mergeable as-is because the useful status change is bundled with unrelated Fly/Docker/workflow/runtime-config/SDK/docs changes and introduces blocking correctness and security concerns.

## What This Changes

The PR captures `/status` memory probe failures for display, while also adding Fly.io deployment workflow/config, Docker runtime changes, startup config sync, SDK outbound helper re-exports, bundled plugin import updates, docs/template edits, and tests for the new runtime-config script.

## Best Possible Solution

Land a focused current-main replacement that captures every memory probe failure in the shared status snapshot, renders it in the memory status line, and adds a regression test for the failing-probe path; keep Fly deployment, Docker image policy, workflow, SDK boundary, and template changes in separate reviewed PRs.

## Reproduction Assessment

Yes. A focused unit reproduction can mock `probeVectorAvailability()` to throw in `resolveSharedMemoryStatusSnapshot`; current main catches and discards that failure, and the renderer has no `probeError` branch to display it.

## Solution Assessment

No. The intended status behavior is the right direction, but this PR is not the best solution because it mixes unrelated security-sensitive infrastructure changes and applies the status edit to stale files; a narrow replacement against current main is safer.

- **[P1] Preserve explicit OpenAI provider selections:** `scripts/sync-runtime-config.mjs:164`
  - body: `availableProviders` skips `openai` once `openai-codex` consumes `OPENAI_API_KEY`, so an explicit `openai/...` primary model is treated as unavailable and rewritten to Codex on startup. That breaks the script's stated preserve-user-overrides behavior.
  - confidence: 0.91
- **[P1] Restore the non-root runtime user:** `Dockerfile:262`
  - body: The Dockerfile removes the existing `USER node` hardening and adds an entrypoint without switching back, so the gateway and entrypoint run as root. That broadens the impact of any gateway, plugin, or dependency compromise.
  - confidence: 0.9
- **[P2] Fix Docker GPG awk quoting:** `Dockerfile:238`
  - body: The awk program is single-quoted, so the embedded escaped quotes are passed literally and cause an awk parse error when `OPENCLAW_INSTALL_DOCKER_CLI` is enabled. That aborts the Docker build before key fingerprint verification completes.
  - confidence: 0.87
- **[P3] Keep empty-message probe failures visible:** `src/commands/status.scan.ts:183`
  - body: The conditional spread drops `probeError` when the probe throws `new Error("")`, so that failure remains hidden. Preserve the field based on presence, and render based on presence, so every thrown probe failure is visible.
  - confidence: 0.78

## Security Review
