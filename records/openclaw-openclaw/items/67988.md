## Summary

Keep open. Current main still builds the Agents/Control UI primary-model dropdown from configured aliases or raw ids before applying catalog display names, so the reported `opus` and `sonnet` labels can still appear for the Anthropic models. The friendlier catalog display-name path exists for the chat picker, but it is not used by this dropdown.

## What This Changes

Show full display names, including version numbers, for configured Claude Opus 4.7 and Claude Sonnet 4.6 entries in the Control UI Agents model dropdown.

## Best Possible Solution

Update the Agents/Control UI model option builder to resolve configured model values through the qualified model catalog display metadata, with a provider-owned catalog metadata path for Opus 4.7 if the upstream catalog is still missing it. Add focused UI tests for the two reported Anthropic refs and alias-precedence behavior.
