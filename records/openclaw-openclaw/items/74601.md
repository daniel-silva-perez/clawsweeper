## Summary

Keep open. Current main still intentionally scopes Skill Workshop proposals, writes, reviewer context, and proposal storage to workspace skills only, and the requested shared-skill/cross-workspace behavior is not implemented. This is a bundled experimental plugin and shared-skill write-boundary change, so it needs maintainer product/security direction rather than conservative cleanup as implemented, duplicate, stale, or ClawHub-only work.

## What This Changes

Request opt-in Skill Workshop support for shared-skill targeting, cross-workspace pattern promotion, and a `promote_to_shared` action.

## Best Possible Solution

Stage the design inside Skill Workshop: first add explicit shared-target metadata, root resolution, store isolation, and approval-only shared apply; then consider a separate cross-workspace reviewer once maintainers define matching evidence, safety limits, and docs/test expectations.
