## Summary

Keep open. Current main still limits Skill Workshop proposals, review context, proposal state, and writes to workspace skills only; this request is a real bundled-plugin enhancement that changes shared-skill write authority and cross-workspace review behavior, so it needs maintainer product/security direction before implementation.

## What This Changes

Request opt-in Skill Workshop support for proposing shared-skill updates, promoting workspace skills into shared candidates, and detecting similar proposals across workspaces.

## Best Possible Solution

Stage this inside `extensions/skill-workshop`: first add an explicit opt-in shared target model with target-specific root validation, separate proposal metadata/storage, docs, tests, and manual-approval-only shared applies; then handle cross-workspace semantic promotion as a follow-up after maintainers define privacy, evidence, and operator-control limits.
