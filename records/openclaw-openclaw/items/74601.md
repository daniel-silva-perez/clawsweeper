## Summary

Keep open. Current main still scopes Skill Workshop proposals, review context, storage, and writes to workspace skills only, while the requested shared-skill targeting and cross-workspace promotion would expand a bundled experimental plugin across a broader write and review boundary that needs maintainer product/security direction.

## What This Changes

Request to let the Skill Workshop plugin opt into proposing and applying shared-skill refinements, detect repeated cross-workspace patterns, and add a first-class promote-to-shared action.

## Best Possible Solution

Stage this inside `extensions/skill-workshop`: first design an explicit shared-target model with target metadata, separated store keys, root validation, manual-approval-only shared applies, docs, and focused tests; then handle cross-workspace semantic promotion as a separate follow-up after privacy and evidence limits are agreed.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current limitation is high-confidence from static inspection: writes, reviewer context, tool schema, proposal storage, docs, and tests are all workspace-scoped on current main.

## Solution Assessment

Unclear. Shared targeting is a plausible direction, but the proposed three-part implementation combines a narrow write-target expansion with broader cross-workspace review, so the maintainable path should split the security-sensitive pieces and get maintainer design approval first.
