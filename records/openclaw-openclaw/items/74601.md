## Summary

Keep open. Current main still implements Skill Workshop as a workspace-only bundled plugin; the requested shared-skill targeting, cross-workspace promotion, and promote-to-shared action are not present. The request is a real feature gap, but it expands write authority and cross-workspace review boundaries, so it needs maintainer product/security direction rather than conservative cleanup or an automated repair PR.

## What This Changes

Request opt-in Skill Workshop support for targeting shared skills under `~/.openclaw/skills`, promoting recurring cross-workspace patterns, and adding a `promote_to_shared` action.

## Best Possible Solution

Stage this inside `extensions/skill-workshop`: first design an explicit shared-target model with per-proposal target metadata, separate store keys, strict shared-root validation, manual-approval-only shared applies, docs, and focused tests. Treat cross-workspace semantic promotion as a separate follow-up after privacy and evidence limits are agreed.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current limitation is high-confidence from static inspection: writes, reviewer context, tool schema, proposal storage, docs, and tests are workspace-scoped on current main.

## Solution Assessment

Unclear. Shared targeting is plausible, but the proposed three-part implementation combines a narrow write-target expansion with broader cross-workspace review, so the maintainable path should split the security-sensitive pieces and get maintainer design approval first.

Summary: No patch was reviewed, but the requested feature would expand write authority to shared skills and add cross-workspace review, so security design review is needed before implementation.

- **[medium] Define shared-skill write authority:** `extensions/skill-workshop/src/skills.ts:38`
  - body: Current writes are constrained to the active workspace. Letting an agent target `~/.openclaw/skills` would change procedures visible to other agents, so the design needs explicit target metadata, strict root validation, auditability, and manual approval for shared applies.
  - confidence: 0.91
- **[medium] Prevent shared-target auto apply:** `extensions/skill-workshop/src/tool.ts:153`
  - body: The current tool and post-turn paths can auto-apply safe workspace proposals when `approvalPolicy` is `auto`. A shared target must override that behavior so `apply: true` and auto policy cannot silently modify shared skills.
  - confidence: 0.88
- **[medium] Limit cross-workspace reviewer inputs:** `extensions/skill-workshop/src/reviewer.ts:183`
  - body: The current reviewer reads only active workspace skills and the current transcript slice. A cross-workspace reviewer could expose workspace-specific procedures or transcript-derived context unless opt-in, evidence minimization, and retention limits are specified.
  - confidence: 0.86
