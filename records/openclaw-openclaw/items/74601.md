## Summary

Keep open. Current main still keeps Skill Workshop proposal state, review context, tool actions, and writes scoped to workspace skills, while this request would expand a bundled experimental plugin into shared-skill writes and cross-workspace review. That needs maintainer product and security direction before a narrow repair lane is safe.

## What This Changes

Allow the bundled Skill Workshop plugin to optionally propose updates to shared skills, detect recurring cross-workspace patterns, and promote workspace skills into shared-skill candidates.

## Best Possible Solution

Stage the work inside `extensions/skill-workshop`: first design an explicit shared-target model with separate target metadata/storage, root validation, manual-approval-only shared applies, docs, and focused tests; then consider cross-workspace semantic promotion as a follow-up after privacy, similarity, and evidence limits are defined.
