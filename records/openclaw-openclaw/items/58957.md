## Summary

Keep open. Current main has partial recovery for explicit context-overflow errors, but it still lacks a proactive `/model` or Control UI context-size check and still has preflight compaction dead zones for heartbeat turns and fresh persisted token totals, matching the later discussion on this issue.

## What This Changes

The issue asks OpenClaw to make large carried-over session context visible and recoverable when switching models or continuing long sessions, ideally with clearer errors, compaction, warnings, and context-budget observability.

## Best Possible Solution

Keep the issue open and converge the remaining work into a maintainer-approved plan: add a target-model context-budget check for `/model`, `sessions.patch`, and Control UI switches; make proactive compaction evaluate fresh token snapshots and heartbeat sessions where safe; preserve context-overflow/failover cause information in user-facing recovery; and coordinate narrower follow-ups with #50619, #66830, and #67750.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction for key parts of the report: a heartbeat or actively used session with fresh `totalTokens` can bypass proactive preflight compaction before threshold evaluation, and model switching via `sessions.patch` does not compare current session size to the target model budget.

## Solution Assessment

No. Current main has useful partial mitigations for overflow recovery and fallback ordering, but it is not the best complete solution because it still allows model switches without preflight budget checks and still skips proactive compaction in the reported long-session cases.
