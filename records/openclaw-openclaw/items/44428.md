## Summary

Current main still matches the reported bug. `deliveryBestEffort` is resolved and passed into cron delivery, but text delivery still waits on active descendant subagents with the full cron timeout and still suppresses delivery while descendants are active without checking that flag. The latest release tag in this checkout contains the same behavior, and the related fix PR #46160 is closed unmerged.

## Best Possible Solution

Keep this open and land a narrow cron delivery fix: when `delivery.bestEffort` is true, isolated cron text delivery should not wait for active fire-and-forget descendants or suppress the parent delivery because descendants are still active. Preserve existing non-best-effort descendant synthesis/suppression behavior, preserve completed-descendant fallback when descendants already finished, add regression coverage for best-effort plus active descendants, and update docs/changelog if the user-visible contract changes.

## Clownfish Work Prompt

Fix openclaw/openclaw#44428. In isolated cron text delivery, `delivery.bestEffort: true` should mean the parent cron run delivers its own final text immediately and records success even if fire-and-forget descendant subagents are still active. Scope the implementation primarily to `src/cron/isolated-agent/delivery-dispatch.ts` and tests. Do not change subagent execution semantics, non-best-effort behavior, failure-alert policy, or the run-scoped session key behavior.

Expected behavior: when `params.deliveryBestEffort` is true and `countActiveDescendantRuns(params.runSessionKey) > 0`, `finalizeTextDelivery` must not call `waitForDescendantSubagentSummary` with the full cron timeout, must not return from the active-descendant suppression branch, and must not suppress stale/interim parent text solely because descendants exist. It should continue through the existing outbound delivery path for the parent synthesized text. Preserve the existing non-best-effort path that waits for descendant summaries and suppresses partial parent updates. Preserve completed-descendant fallback behavior when descendants already finished before delivery dispatch.

Use PR #46160 only as reference; do not copy it blindly. Its review comments flagged best-effort interim suppression and completed-descendant fallback edge cases, so add regression coverage for those. Suggested tests in `src/cron/isolated-agent/delivery-dispatch.double-announce.test.ts`: set `deliveryBestEffort: true`, mock active descendants, assert `waitForDescendantSubagentSummary` is not called, assert `deliverOutboundPayloads` is called once with the parent text, assert `deliveryAttempted=true` and `delivered=true`, and add an interim-text case so best-effort is not suppressed by the stale interim guard. Keep the existing non-best-effort active-descendant suppression test passing.

Update `docs/automation/cron-jobs.md` and `docs/automation/tasks.md` if the behavior contract is documented, and add a user-facing changelog entry if required by repo policy, crediting the reporter and any reused prior PR context. Do not touch unrelated cron delivery, provider, UI, or subagent files.
