## Summary

Keep open. Current main still treats heartbeat as a periodic agent turn that is serialized through the shared main/global command lane and the target session lane. The existing code and tests intentionally defer or retry heartbeat when lanes are busy, but they do not implement the requested non-blocking heartbeat worker or separate heartbeat execution lane. The KeepMyClaw snapshot comment is useful operational mitigation, and the later reviewer comment correctly frames this as a real architecture feature request rather than an already-fixed scheduler bug.

## Best Possible Solution

Keep this open as an architecture/product feature request. The likely path is a dedicated heartbeat execution lane or worker-style isolated heartbeat turn model that does not occupy normal inbound message processing, with tests for concurrent user messages, in-flight heartbeat state, scheduler retry cadence, delivery ordering, and tool side effects. Existing isolatedSession/lightContext options should be treated as cost/context mitigations, not as completion of this request.
