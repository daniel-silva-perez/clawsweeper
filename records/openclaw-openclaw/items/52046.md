## Summary

Keep this issue open. Current main still treats heartbeat as a scheduled agent turn that is backpressured by the shared main/global lane and target session lane; it does not implement a separate heartbeat worker or non-blocking heartbeat execution lane. The issue discussion correctly frames this as an architecture feature request, and the KeepMyClaw snapshot comment is a workaround rather than a core implementation.

## Best Possible Solution

Keep this open as an architecture/product feature request. The strongest path is a dedicated heartbeat execution lane or worker-style isolated heartbeat turn model, with explicit contracts for session state, delivery ordering, tool side effects, wake retry cadence, and concurrent inbound user messages. Existing `isolatedSession`, `lightContext`, and heartbeat model overrides should stay framed as cost/context mitigations, not completion of the non-blocking execution request.
