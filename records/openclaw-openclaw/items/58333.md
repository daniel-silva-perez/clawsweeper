## Summary

Keep this open: current main and the latest release still have the broad descendant chevron selectors, so the PR is not obsolete and addresses a real Control UI bug. The production CSS direction is narrow, but the proposed regression test still has a concrete guard bug that should be fixed before merge.

## What This Changes

The PR scopes Control UI config chevron rotation to direct details headers and adds browser regression coverage for nested config objects and pending-change diff panels.

## Best Possible Solution

Keep this PR open, or supersede it with an equivalent target-repo fix. The desired end state is the direct-header selector fix plus focused browser coverage for nested config chevrons and pending-change diff chevrons, with the test guard corrected before merge.

## Reproduction Assessment

Yes. Current main has source-level reproduction evidence: nested `.cfg-object` details render under open ancestors, and the existing descendant selector matches child chevrons regardless of the child details state.

## Solution Assessment

No, not as-is. The CSS change is the narrowest maintainable production fix, but the regression test should fail on a missing chevron instead of returning early after a passing `not.toBeUndefined()` assertion.
