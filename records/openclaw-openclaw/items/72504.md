Keep this issue open. Current main still strips every inbound Feishu mention whose `open_id` matches the receiving bot before building the agent-facing prompt, and the provided context shows an open PR with closing syntax for this issue. The right maintainer path is to review and land or reject that PR rather than close the issue early.
