## Summary

Keep open. Current main still strips the receiving Feishu bot's own mention from normalized inbound content, and open PR #72661 already tracks a narrower closing fix, so this should not be closed before that PR is reviewed and merged or rejected.

## What This Changes

The issue requests preserving a Feishu bot's own `<at>` mention in multi-bot group messages so each explicitly mentioned bot can see that it was addressed and reply instead of returning NO_REPLY.

## Best Possible Solution

Land or reject the focused PR #72661, resolve the competing #72532, and end with a narrow Feishu fix that strips only leading addressing prefixes for slash-command compatibility while preserving inline self-mentions in model-facing content with regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to parse a Feishu group text message whose `mentions[]` contains the receiving bot and another bot; current `normalizeMentions()` removes the receiving bot's key before `buildFeishuAgentBody()` builds the prompt.

## Solution Assessment

Yes for the linked leading-prefix direction, but not for removing self-mention stripping wholesale. The narrower approach preserves the #35994 slash-command behavior while fixing the inline multi-bot context loss.
