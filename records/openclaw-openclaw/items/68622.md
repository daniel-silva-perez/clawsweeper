## Summary

Keep this PR open, but do not merge its current head. Current main still lacks the requested hook, so the work is not obsolete; however, the branch has a concrete delegated-announce trust-boundary bug, invokes the hook after core reply-history access in waited sends, and leaves public SDK surface cleanup unfinished. The open replacement/patch PR #75369 appears to address the P1/P2 path and should be reviewed or folded back before this work lands.

## What This Changes

This PR adds a typed `sessions_send` plugin hook to the hook registry and Plugin SDK exports, invokes it from the `sessions_send` tool with delegated/direct fallback handling, updates the generated SDK API baseline hash, and adds sessions tool coverage for declined and delegated hook paths.

## Best Possible Solution

Land a rebased or superseding implementation that keeps the generic `sessions_send` hook seam, derives delegated announce seeds only from trusted hook output or verified reply history, invokes hooks before core-only history reads, preserves current A2A safeguards, and ships the SDK exports, docs, changelog, API baseline, and focused tests together.

## Reproduction Assessment

Yes. The patch inspection path is high-confidence: raw tool args can carry `task.runtime.roundOneReply`, the PR copies it into the hook task, delegated handling passes it to `startA2AFlow`, and current A2A code treats `roundOneReply` as the primary/latest reply seed.

## Solution Assessment

No. The hook direction is plausible, but this branch is not the best fix until it is rebased or replaced with the trust-boundary and ordering fixes, plus the remaining public SDK export/docs/changelog cleanup.

- **[P1] Stop trusting caller-supplied announce metadata:** `src/agents/tools/sessions-send-tool.ts:445`
  - body: In delegated mode this passes `hookTask?.runtime?.roundOneReply`, parsed from raw tool args, into `startA2AFlow`.   - confidence: 0.78

  - confidence: 0.74
