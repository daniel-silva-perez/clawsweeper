## Summary

Keep this PR open. Current main still lacks a `sessions_send` plugin hook, so the proposed SDK seam addresses a real gap, but the current head should not merge because delegated handling trusts caller-supplied task metadata and invokes the hook only after a core history read that can fail before plugin-owned delivery runs.

## What This Changes

The PR adds typed `sessions_send` hook contracts, hook-runner and SDK exports, hook invocation in the `sessions_send` tool, generated API baseline hash updates, and sessions tool tests for declined-hook fallback and delegated wait metadata.

## Best Possible Solution

Land a rebased hook seam that runs after validation/visibility checks but before core-only history reads, derives delegated announce state only from trusted hook output or actual waited run output, exports the full public task/cancel-target contract, and keeps the current A2A provenance and skip safeguards intact.

## Reproduction Assessment

Yes. At PR head, a delegated call with `task.runtime.roundOneReply` can feed caller-controlled text into `runSessionsSendA2AFlow`, and a failing `chat.history` call before line 414 prevents even a direct hook handler from running.

## Solution Assessment

No. The hook direction is maintainable, but this patch is not the best form until it removes caller-controlled announce seeding and moves hook invocation ahead of core baseline-history reads while preserving current-main A2A behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Stop trusting caller-supplied announce metadata:** `src/agents/tools/sessions-send-tool.ts:445`
  - body: In delegated mode this passes `hookTask?.runtime?.roundOneReply`, parsed from raw tool args, into `startA2AFlow`. Since tool args can include extra fields, a caller can seed the A2A ping-pong/announce content without it coming from the delegated run or trusted plugin result; use hook output or the waited reply instead.
  - confidence: 0.86
- **[P2] Invoke the hook before core history reads:** `src/agents/tools/sessions-send-tool.ts:414`
  - body: For waited sends, `baselineReply` is read before `runSessionsSend`; if `chat.history` fails, a plugin that would handle direct or delegated delivery never runs. Move the core baseline read into the fallback/core-delegated wait path so hook-owned delivery is not blocked by core history access.
  - confidence: 0.78
- **[P3] Export the hook task and cancel-target types:** `src/plugin-sdk/core.ts:102-106`
  - body: The new public event/result types refer to `PluginHookSessionsSendTask` and `PluginHookSessionsSendCancelTarget`, but `plugin-sdk/core` only re-exports the context/event/result types. Plugin authors cannot name the task or cancel-target contract from the same SDK surface without type workarounds.
  - confidence: 0.74
