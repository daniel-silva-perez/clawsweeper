## Summary

Keep this PR open. Current main still does not provide a `sessions_send` plugin hook, so the proposed seam is not obsolete, but the branch is dirty against main and the current patch has blocking correctness/security concerns in the delegated path.

## What This Changes

The branch adds a typed `sessions_send` plugin hook, wires `sessions_send` to let hooks return direct or delegated handling, exports SDK hook types, updates the plugin SDK API baseline, and adds sessions tool tests for declined and delegated hook paths.

## Best Possible Solution

Land a rebased version only after the hook runs before default history reads, delegated announce metadata comes from trusted hook output or the actual waited reply, complete public SDK task/cancel-target types are exported, and current main's sessions-send provenance, lane, and ACP-skip behavior is preserved with focused tests and regenerated SDK baseline.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Use only trusted text for delegated announce flow:** `src/agents/tools/sessions-send-tool.ts:445`
  - body: The delegated branch seeds `runSessionsSendA2AFlow` with `hookTask?.runtime?.roundOneReply`, but `hookTask` is parsed from raw tool arguments. A caller can therefore inject announce/ping-pong content that was never produced by the delegated run. The seed should come from trusted hook output or the waited reply.
  - confidence: 0.9
- **[P2] Invoke the hook before reading core history:** `src/agents/tools/sessions-send-tool.ts:414`
  - body: The new hook call happens after `readLatestAssistantReplySnapshot`, so a failing `chat.history` baseline read aborts the tool before a `sessions_send` hook can handle direct or delegated delivery. Move hook interception ahead of core-only baseline work.
  - confidence: 0.88
- **[P3] Export the sessions_send task subtypes:** `src/plugin-sdk/core.ts:102-106`
  - body: `PluginHookSessionsSendEvent` and `PluginHookSessionsSendResult` expose task and cancel-target shapes, but `core.ts` only exports the event/context/result aliases. Plugin authors cannot name the public task or cancel-target types without indexed-access workarounds.
  - confidence: 0.78

## Security Review

Status: needs_attention
