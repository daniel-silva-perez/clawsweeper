## Summary

Keep this PR open. Current main still does not implement the PR's intended Windows spawn behavior: the plugin SDK helper omits `windowsHide: true` for both Win32 direct executable resolution and explicit shell fallback, and downstream spawn call sites pass that missing value through. The PR is focused and low-risk, but it needs a rebase because one test file it edits no longer exists on main.

## Best Possible Solution

Keep the PR open for maintainer follow-up. Rebase it onto current main, move the regression coverage into `src/plugin-sdk/windows-spawn.test.ts`, add `windowsHide: true` to the plugin SDK `direct` and `shell-fallback` Windows outcomes while preserving non-Windows `undefined` behavior and fail-closed unresolved wrappers, then decide whether the memory-host SDK copy belongs in the same fix.
