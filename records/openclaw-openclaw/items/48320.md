## Summary

Keep this PR open. Current main still does not implement the intended Windows spawn fix: `src/plugin-sdk/windows-spawn.ts` omits `windowsHide: true` for the Win32 direct executable path and for explicit shell fallback, and downstream ACP/stdio call sites forward that missing value into `child_process.spawn()`. The branch needs a rebase because one ACPX test file it edits no longer exists, but the core production change remains focused and useful.

## Best Possible Solution

Keep the PR open for maintainer follow-up. Rebase it onto current main, move the regression coverage into `src/plugin-sdk/windows-spawn.test.ts`, update `src/plugin-sdk/windows-spawn.ts` so Win32 `direct` and explicit `shell-fallback` outcomes return `windowsHide: true`, preserve non-Windows `undefined` behavior and fail-closed unresolved wrappers, and decide whether the memory-host SDK copy should be updated in the same change.
