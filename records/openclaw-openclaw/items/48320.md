## Summary

Keep this PR open. Current main still does not implement the intended Windows spawn change: the shared plugin SDK helper omits `windowsHide: true` for Windows `direct` and explicit `shell-fallback` outcomes, current tests still expect the shell fallback value to be `undefined`, and the ACP client forwards that helper value into `child_process.spawn()`. The branch is stale against current ACPX test layout, but the core fix is still actionable.

## Best Possible Solution

Keep the PR open for maintainer follow-up and ask for a rebase. The focused fix should update the current shared Windows spawn helper so Win32 `direct` and explicit `shell-fallback` outcomes return `windowsHide: true`, add direct `.exe` regression coverage, update shell-fallback coverage, preserve non-Windows `undefined` behavior and the fail-closed default for unresolved wrappers, and refresh or drop stale ACPX test-only edits. Consider a small follow-up or same-PR mirror for the memory-host SDK copy if maintainers want consistent Windows process behavior there.
