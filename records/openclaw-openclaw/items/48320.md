## Summary

Keep this PR open. Current main still does not implement the PR’s intended behavior: the shared Windows spawn helper still omits `windowsHide: true` for the Win32 direct executable path and explicit shell fallback, and ACP client spawning forwards that missing value through to `child_process.spawn()`. The PR is focused and has a low security/supply-chain surface, but it needs a rebase/test relocation because one touched ACPX test file no longer exists on main.

## Best Possible Solution

Keep the PR open for maintainer follow-up. Rebase it onto current main, move the regression coverage into `src/plugin-sdk/windows-spawn.test.ts`, add `windowsHide: true` to the plugin SDK `direct` and `shell-fallback` Windows outcomes while preserving non-Windows `undefined` behavior and fail-closed unresolved wrappers, then decide whether the memory-host SDK helper copy should be fixed in the same PR or a dedicated follow-up.
