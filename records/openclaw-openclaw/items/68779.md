## Summary

Keep open. Current main contains partial mitigations for existing-session browser automation, but maintainer discussion explicitly left reconnect/recovery, loaded-profile attach budgets, and regression proof as remaining work; source review confirms those areas are not fully resolved.

## What This Changes

The issue requests more robust existing-session browser automation by widening timeout budgets, adding Chrome MCP session recovery, diagnostics, configuration overrides, and tests for multi-step flows.

## Best Possible Solution

Keep this open and implement a scoped browser-plugin hardening plan: use trace-backed Chrome MCP attach/readiness budgets for existing-session profiles, add bounded session refresh/recovery only where retries are safe, preserve fail-closed navigation checks, and add a regression test plus live/manual proof for a multi-step `profile="user"` workflow.
