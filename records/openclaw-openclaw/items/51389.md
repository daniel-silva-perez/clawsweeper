## Summary

Keep this PR open. Current main has broader model fallback behavior than the original problem statement describes, including covered HTTP 400/422 cases and unrecognized-error continuation while candidates remain, but it still does not implement the PR's intended public `fallbackOnErrors` configuration surface. The branch also changes public config/runtime semantics and still needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer/product review. The best path is to rebase or replace it with a current-main implementation only if OpenClaw wants this public config surface: extend the strict config schema and generated config help, wire every relevant fallback call path, define whether non-chat image/PDF models support the option, preserve current default behavior, fail fast on disallowed custom statuses, preserve the field during config rewrites, and add focused tests for `default`, `all`, custom arrays, queued follow-ups, and non-chat scope decisions.
