## Summary

Keep open. Current main still lacks dynamic GitHub Copilot text-model discovery, and this PR is a relevant implementation candidate, but the PR head is not merge-ready because agent-scoped auth lookup, cache scoping, and fallback thinking behavior need fixes.

## What This Changes

The PR adds runtime GitHub Copilot `/models` discovery with mapping, caching, catalog augmentation, dynamic-model preparation, thinking-profile tests, and small plugin-SDK/test cleanups.

## Best Possible Solution

Land a refined GitHub Copilot plugin implementation that fetches authenticated text-model capabilities through provider-owned hooks, preserves current conservative fallback behavior when discovery fails, scopes cached capabilities to the relevant credential/baseUrl/agent context, and includes focused tests for auth-profile-only, multi-agent/profile switching, cache invalidation, and fallback thinking levels.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main returns an empty Copilot provider catalog and static/fallback limits, while PR head drops `ctx.agentDir` in the shared fetch helper and removes current-main `xhigh` fallback behavior when API data is absent.

## Solution Assessment

No, not as written. Dynamic `/models` discovery is the right direction, but the implementation should pass agent-scoped auth context into discovery, key or invalidate the process cache by credential/baseUrl, and preserve existing known-model thinking fallbacks when live discovery cannot run.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Pass agentDir into model discovery:** `extensions/github-copilot/index.ts:54-57`
  - body: `fetchAndCacheModels` resolves the GitHub token without `ctx.agentDir`, even though the catalog and augmentation hooks receive it. A user authenticated only through an agent-local Copilot profile can pass the catalog preflight and then get an empty dynamic catalog because the actual fetch looks in the default auth store instead of the current agent store.
  - confidence: 0.89
- **[P2] Keep xhigh fallback for known Copilot models:** `extensions/github-copilot/index.ts:463-464`
  - body: When the `/models` request fails, times out, or has not populated the cache, this fallback now returns the generic profile without `xhigh`. That regresses current support for known Copilot GPT/Codex models such as `gpt-5.4` and `gpt-5.2`, where main already exposes `xhigh` from the hardcoded allowlist.
  - confidence: 0.86
- **[P2] Scope the model cache to the credential context:** `extensions/github-copilot/index.ts:44-45`
  - body: Once any request fills `cachedModelCapabilities`, every later request returns that same plan catalog without checking agent dir, auth profile, env token, config, or Copilot baseUrl. In a gateway serving multiple agents or after auth/profile changes, `/models list` and thinking caps can show stale capabilities from another Copilot account.
  - confidence: 0.82
