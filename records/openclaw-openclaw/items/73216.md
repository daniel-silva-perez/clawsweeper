## Summary

Keep this PR open. Current main still does not implement Copilot text-model discovery from the authenticated `/models` API: the provider catalog returns an empty model list, bundled defaults still use a static model table with 128K/8192 limits, unknown Copilot model IDs still use the conservative synthetic fallback, and `xhigh` thinking support is still a hardcoded model-id allowlist. The PR remains a focused implementation candidate, with a normal maintainer review needed for the new authenticated catalog fetch, token-derived base URL handling, SSRF guard policy, mapping semantics, and the previously discussed thinking-level behavior.

## Best Possible Solution

Keep this PR open for normal maintainer review. If accepted, land a refined GitHub Copilot plugin implementation that fetches the authenticated text-model catalog through existing provider hooks, maps limits/capabilities/transport/thinking profiles into OpenClaw model definitions, preserves conservative fallback behavior when discovery fails, validates token/baseUrl/SSRF behavior, and adds the focused tests plus any user-facing changelog or docs updates maintainers require.
