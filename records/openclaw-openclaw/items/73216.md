## Summary

Keep this PR open. Current main still does not implement the PR’s intended dynamic GitHub Copilot text-model catalog: the Copilot provider catalog returns no model rows, default Copilot text models remain a fixed static list with flat 128K/8192 limits, unknown models still use a conservative synthetic fallback, and thinking levels still come from a hardcoded xhigh allowlist. The PR remains a focused implementation candidate, with the discussion showing the author already addressed two bot-review findings and explained the remaining `minimal` thinking-level concern.

## Best Possible Solution

Keep this PR open for normal maintainer review. If accepted, land a refined GitHub Copilot plugin implementation that fetches the authenticated `/models` catalog, maps context/output/vision/thinking capabilities into OpenClaw model definitions, preserves conservative fallback behavior when discovery fails, and includes focused unit plus live Copilot validation for the new network path.
