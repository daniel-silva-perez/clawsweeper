## Summary

Keep this PR open. Current main still does not implement dynamic GitHub Copilot text-model catalog discovery: the provider catalog returns no text models, default Copilot model definitions still use static 128K/8192 limits, unknown models still fall back to the synthetic conservative resolver, and xhigh thinking remains a hardcoded model-id allowlist. The PR is still a focused implementation candidate, but it needs maintainer review for the new authenticated /models fetch, capability mapping, token/baseUrl handling, and auth-profile behavior.

## Best Possible Solution

Keep this PR open for normal maintainer review. If accepted, land a refined GitHub Copilot plugin implementation that fetches the authenticated /models catalog through provider-owned hooks, maps context/output limits, input modalities, transport APIs, compat flags, and thinking profiles into OpenClaw model definitions, keeps the conservative fallback when discovery fails, validates token/baseUrl/SSRF and auth-profile behavior, and includes focused extension tests plus any required changelog/docs updates.
