## Summary

Keep open. Current main has adjacent Discord presence plumbing via static config, auto-presence, and a gated `setPresence` message-action path, but it still does not expose the requested typed `api.runtime.channel.discord.updatePresence()` helper for code plugins.

## Best Possible Solution

Keep this issue open until the Discord plugin either adds a narrow Discord-owned `api.runtime.channel.discord.updatePresence()` helper or explicitly documents a supported code-plugin equivalent. The best implementation should reuse the existing gateway registry/updatePresence path and preserve gating, account selection, validation, connection errors, and auto-presence interaction rules.
