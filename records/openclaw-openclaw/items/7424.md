## Summary

Keep open. Current main still provides a global `agents.defaults.models` catalog/allowlist and per-agent primary/fallback defaults, but no typed per-agent model allowlist and no agent-scoped `models.list` or `sessions.patch` enforcement contract. The existing discussion also treats this as the narrower canonical request: #9010 is closed as a duplicate, while #65557 is broader account/provider policy work and does not supersede this issue.

## What This Changes

Requests a per-agent model allowlist so a specific agent cannot be switched to disallowed or expensive models.

## Best Possible Solution

Add a documented, typed per-agent model catalog/allowlist with explicit composition rules against `agents.defaults.models`, then enforce the effective allowed set consistently in model listing, `/model` selection, session patching, UI/TUI selectors, and regression tests while leaving broader account/provider administration to #65557.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is source and docs inspection showing only the global allowlist plus per-agent primary/fallback defaults, with no agent-scoped catalog field or model-list parameter.

## Solution Assessment

Unclear because there is no proposed patch yet. The maintainable direction is an additive config/protocol surface with explicit inheritance and enforcement semantics rather than overloading the existing strict `agents.list[].model` primary/fallback shape without review.
