## What This Changes

Request a per-agent model allowlist so one agent can restrict visible and selectable models independently of the global model catalog for cost and policy control.

## Best Possible Solution

Add a documented, typed per-agent model catalog/allowlist with explicit composition rules against `agents.defaults.models`, then enforce the effective allowed set consistently in model listing, `/model` selection, Gateway session patching, UI/TUI pickers, and regression tests while leaving broader account/provider administration to #65557.

## Solution Assessment
