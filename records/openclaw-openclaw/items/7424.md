## Summary

Keep open. Current main still has only a global `agents.defaults.models` allowlist/catalog plus per-agent primary/fallback defaults; it does not provide a typed per-agent model allowlist or an agent-scoped Gateway model-list/session-patch contract. The existing discussion also keeps this as the narrower canonical request, with #9010 closed as duplicate and #65557 covering broader account/provider policy.

## What This Changes

The issue requests a per-agent model allowlist so a specific agent can restrict which models users can see or select.

## Best Possible Solution

Add a documented, typed per-agent model catalog/allowlist with explicit composition rules against `agents.defaults.models`, then enforce the effective allowed set consistently in model listing, chat model selection, Gateway session patching, UI/TUI selectors, and regression tests while leaving broader account/provider administration to #65557.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is source and docs inspection showing only the global allowlist plus per-agent primary/fallback defaults, with no agent-scoped catalog field or model-list parameter.

## Solution Assessment

Unclear because there is no proposed patch yet. The maintainable direction is an additive config/protocol surface with explicit inheritance and enforcement semantics rather than overloading the current strict primary/fallback model shape without maintainer API review.
