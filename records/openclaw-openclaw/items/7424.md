## Summary

Keep open. Current main still has only a global `agents.defaults.models` allowlist/catalog plus per-agent primary/fallback defaults; there is no typed per-agent model allowlist and the Gateway listing and session patch validation paths are not agent-scoped. The provided discussion also keeps this as the narrower canonical request: #9010 is closed as a duplicate, while #65557 is broader account/provider policy work and does not supersede it.

## What This Changes

Add a per-agent model allowlist so specific agents can be restricted from selecting costly or inappropriate models.

## Best Possible Solution

Add a documented, typed per-agent model catalog/allowlist with explicit composition rules against `agents.defaults.models`, then enforce the effective allowed set consistently in `models.list`, `/model` selection, session patching, UI/TUI selectors, and regression tests while leaving broader account/provider administration to #65557.
