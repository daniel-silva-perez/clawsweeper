## Summary

Current main still has a global `agents.defaults.models` allowlist plus per-agent primary/fallback model defaults, but it does not provide a typed per-agent model allowlist or agent-scoped catalog/rejection path. Related #9010 is already closed as a duplicate of this narrower request, while #65557 is broader per-account/provider policy and does not supersede the agent-scoped cost-control issue.

## Best Possible Solution

Keep this open as the canonical per-agent model allowlist request. Maintainers should decide the agent-scoped config/API shape, document composition with `agents.defaults.models`, then enforce it consistently in `models.list`, slash/channel pickers, `sessions.patch`, model override paths, Control UI/TUI selectors, generated schema/docs, and regression tests.
