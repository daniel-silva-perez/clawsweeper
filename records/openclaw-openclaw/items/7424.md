## Summary

Keep this open. Current main still has only a global `agents.defaults.models` model catalog/allowlist plus per-agent primary/fallback defaults and params; there is no typed per-agent model allowlist, and the Gateway/session validation paths still resolve allowed models through the global catalog. The existing maintainer/Codex discussion also explicitly keeps this as the narrower request, with #9010 closed as a duplicate and #65557 left as broader account/provider policy work.

## What This Changes

Add a per-agent model allowlist so a configured agent can expose and accept only permitted models, preventing users from switching that agent to expensive or inappropriate models.

## Best Possible Solution

Add a documented, typed per-agent model catalog/allowlist with explicit composition rules against `agents.defaults.models`, then enforce the effective allowed set consistently in model listing, `/models`, `/model` selection, session patching, UI/TUI selectors, and regression tests while leaving broader per-account/provider administration to #65557.
