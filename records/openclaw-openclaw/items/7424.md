## Summary

Keep open. Current main still has a global `agents.defaults.models` allowlist/catalog plus per-agent primary/fallback defaults, but no typed per-agent model allowlist, and the latest discussion explicitly keeps this as the narrower canonical request rather than folding it into the broader account/provider policy issue.

## What This Changes

Request per-agent model allowlists so individual agents can restrict which models are visible and selectable for cost or capability control.

## Best Possible Solution

Add a documented, typed per-agent model catalog/allowlist with explicit composition rules against `agents.defaults.models`, then enforce the effective allowed set in model listing, `/model` selection, Gateway session patching, UI/TUI pickers, and regression tests while leaving broader account/provider administration to #65557.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection showing only global model allowlisting plus per-agent primary/fallback defaults, with no typed agent-scoped catalog or agent-scoped `models.list` parameter.

## Solution Assessment

Unclear because there is no proposed patch yet. The maintainable direction is an additive config/protocol surface with explicit inheritance and enforcement semantics, rather than overloading the current strict primary/fallback model shape without maintainer API review.
