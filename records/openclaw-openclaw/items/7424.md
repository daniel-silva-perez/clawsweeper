## Summary

Current main still lacks a typed per-agent model allowlist. OpenClaw has a global `agents.defaults.models` allowlist/catalog and per-agent `agents.list[].model` primary/fallback defaults, but no agent-scoped model catalog or rejection contract. Related #9010 is closed as a duplicate of this request, while open #65557 is broader per-account/provider policy and does not supersede the agent-scoped cost-control feature.

## Best Possible Solution

Keep this open as the canonical per-agent model allowlist request. Maintainers should decide the additive config/API shape and how it composes with `agents.defaults.models`, then enforce it consistently in `models.list`, `/model` and channel pickers, `sessions.patch`, Control UI/TUI selectors, generated schema/help/docs, and regression tests. #9010 should remain treated as the duplicate, and #65557 can continue tracking the broader account/provider policy separately.
