## Summary

Keep open. Current main still has a global `agents.defaults.models` allowlist/catalog and per-agent primary/fallback defaults, but no typed per-agent model allowlist and no agent-scoped `models.list` or `sessions.patch` enforcement contract. The provided discussion already treats this as the narrower canonical request: #9010 is closed as a duplicate, while #65557 is broader provider/account policy work and does not supersede it.

## What This Changes

Add a per-agent model allowlist so individual agents can hide and reject disallowed models, such as expensive OpenRouter Opus options, without running separate gateways.

## Best Possible Solution

Add a documented, typed per-agent model catalog/allowlist with explicit composition rules against `agents.defaults.models`, then enforce the effective allowed set consistently in model listing, `/model` selection, session patching, UI/TUI selectors, and regression tests while leaving broader account/provider administration to #65557.
