## Summary

Keep open. Current main still supports only the global `agents.defaults.models` catalog/allowlist plus per-agent primary/fallback defaults; there is no typed per-agent model allowlist, and the main Gateway listing and session patch enforcement paths are not agent-scoped. The discussion also keeps this as the narrower canonical request: #9010 is closed as a duplicate, while #65557 is broader account/provider policy work.

## What This Changes

Request to add a per-agent model allowlist so routed agents can restrict visible and selectable models for cost or capability control.

## Best Possible Solution

Add a documented, typed per-agent model catalog/allowlist with explicit composition rules against `agents.defaults.models`, then enforce the effective allowed set consistently in `models.list`, `/model` selection, session patching, UI/TUI selectors, and regression tests while leaving broader account/provider administration to #65557.
