## Summary

Keep this issue open. Current main still provides only the global `agents.defaults.models` allowlist/catalog plus per-agent primary/fallback model overrides; it does not expose a typed per-agent model allowlist, an agent-scoped `models.list` parameter, or complete agent-scoped server-side rejection for all model selection paths. Related #9010 is already closed as a duplicate, while #65557 is broader account/provider policy and does not replace this narrower tracker.

## Best Possible Solution

Keep this open as the canonical per-agent model allowlist request. Define the agent-scoped config/API shape, document how it composes with `agents.defaults.models`, then enforce it consistently in `models.list`, slash/channel pickers, `sessions.patch`, OpenAI-compatible override headers, Control UI/TUI model pickers, generated schema/docs, and regression tests.
