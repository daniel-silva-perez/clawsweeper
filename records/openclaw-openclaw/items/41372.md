## Summary

Current main has meaningful partial fixes, but it does not close the umbrella report. The cron docs still say plain `openclaw cron list` lists all jobs while the CLI/service default is enabled-only unless `--all`/`includeDisabled` is set, the cron JSON wrapper shapes are still not documented in the cron CLI guide, and `cron add` remains create-only with a fresh UUID rather than an idempotent setup/upsert path.

## Best Possible Solution

Keep this issue open for maintainer triage. Fix the cron docs so `openclaw cron list` is documented as enabled-only by default with `openclaw cron list --all` for disabled jobs, document the `jobs` and `entries` JSON wrapper shapes, and decide whether cron should gain an idempotent create/upsert command for setup scripts. Then split any still-current Discord receive, isolated web-search, onboarding token, update, and small-VPS resource observations into focused follow-ups with fresh reproduction data.
