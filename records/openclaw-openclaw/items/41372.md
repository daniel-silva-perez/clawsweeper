## Summary

Keep this issue open. Current main still has actionable cron work from the field report: docs still present plain `openclaw cron list` as listing all jobs, while the CLI/service default is enabled-only unless `--all` is passed; `cron list --json` and `cron runs --json` wrapper shapes remain underdocumented; and `cron add` is still create-only with no idempotent setup/upsert path.

## Best Possible Solution

Keep this issue open for maintainer triage. First fix the cron docs so plain `openclaw cron list` is documented as enabled-only with `openclaw cron list --all` for disabled jobs, document the `jobs` and `entries` JSON wrapper shapes, and decide whether cron needs an idempotent setup/upsert command. Then split or link focused follow-ups for any still-current Discord receive, isolated web-search, onboarding token, update-process, and small-VPS resource findings.
