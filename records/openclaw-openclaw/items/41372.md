## Summary

Keep this issue open. Current main has addressed or improved some parts of the field report, but the central cron cleanup remains actionable: docs still say plain `openclaw cron list` lists all jobs even though the CLI and service default to enabled-only unless `--all`/`includeDisabled` is used; the cron JSON wrapper shapes remain underdocumented in the cron docs; and `cron add` remains create-only with no idempotent setup/upsert path.

## Best Possible Solution

Keep this issue open for maintainer triage. First fix the cron docs so plain `openclaw cron list` is documented as enabled-only with `openclaw cron list --all` for disabled jobs, document the `jobs` and `entries` JSON wrapper shapes for cron scripting, and decide whether cron needs an idempotent setup/upsert command or a documented safe setup pattern. Then split or link narrower follow-ups for any still-current Discord receive, isolated web-search, onboarding-token, update-process, and small-VPS resource findings.
