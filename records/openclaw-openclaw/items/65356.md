## Summary

Keep open. Current main still uses a module-local sandbox backend registry, so the PR’s intended singleton registry fix is not implemented or obsolete. The change targets a public plugin SDK seam used by sandbox backend plugins, so it should be reviewed as a focused core/plugin-API bugfix rather than closed as external plugin work.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a core fix that makes the sandbox backend registry process-global, ideally using the existing `resolveGlobalSingleton`/`Symbol.for` pattern plus a regression test that simulates plugin registration through the plugin SDK loader and runtime lookup through `resolveSandboxContext`.
