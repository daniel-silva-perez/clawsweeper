## Summary

Keep this PR open. Current main still mirrors ACP timeoutSeconds into backend config, still routes /acp timeout through setSessionConfigOption, still applies runtime controls as a single hard-failing block, still omits JSON-RPC data.details from ACP error text, and still drops the handle argument in the acpx wrapper getCapabilities path. The PR is also still open, draft, and unmerged, so it is a live ACP bug-fix candidate rather than conservative cleanup.

## Best Possible Solution

Keep the PR open as the current focused fix candidate. Review the fail-soft classifier carefully, keep the dependency story aligned with upstream openclaw/acpx#282 or a later published acpx release, preserve the local timeout runtime-options path, and land this PR or a cleaner superseding PR only after targeted ACP manager/runtime/commands/acpx tests plus build or changed-gate proof and an ACP smoke verify the timeout frame is no longer sent.
