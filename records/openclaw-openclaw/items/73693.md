## Summary

Keep this PR open: current main still has the ACP timeout/config-control failure paths this branch targets, and the runtime direction is focused, but the PR currently has one concrete changelog correctness blocker because it claims an `acpx@0.6.2` bump that is not in the diff.

The PR drops ACP timeout mirroring into backend config, reroutes timeout commands through local runtime options, adds JSON-RPC error-detail extraction and fail-soft unsupported-control handling, forwards acpx capability input, and updates ACP tests/docs/changelog.

Mostly yes for the runtime behavior: removing redundant timeout round-trips, keeping local timeout enforcement, forwarding capabilities input, and fail-softing only classified unsupported controls is narrower than an acpx-only workaround.
