## Summary

Keep this PR open. Current main still has the ACP timeout mirroring, slash-command backend write path, opaque ACP error formatting, and acpx capability-forwarding gap that the PR targets. It is not implemented on main, and the related items do not supersede it: #23580 is the earlier ACP thread-bound-agent work, while #72080 is a separate ACP completion/liveness report. The PR is draft and still has a dependency/build blocker, so the best maintainer path is review and hardening, not cleanup closure.

## Best Possible Solution

Keep the PR open as a live ACP bug-fix candidate. The best path is to replace the temporary acpx file tarball with the published handle-aware acpx version, keep the OpenClaw-side timeout/local-runtime and fail-soft changes scoped, regenerate the lockfile cleanly, then verify with the targeted ACP manager/runtime/commands/acpx tests plus build or changed-gate proof. Close only after this PR or a cleaner superseding PR lands on main.
