## Summary

Keep open. Current main still has the reported ACP/acpx failure path: it mirrors timeoutSeconds into backend config, sends timeout from slash commands, applies runtime controls in one hard-failing block, omits JSON-RPC detail extraction, and drops handle-aware acpx capability input. This PR is a relevant active fix candidate, not obsolete cleanup.

## What This Changes

This PR removes ACP timeout config mirroring, reroutes timeout slash commands through local runtime options, adds ACP JSON-RPC detail extraction and richer error text, makes unsupported runtime controls fail-soft, forwards acpx handle-aware capabilities, and updates tests/docs/changelog.

## Best Possible Solution

OpenClaw should keep ACP timeouts as client-enforced runtime options, preserve backendExtras as the explicit escape hatch, surface backend JSON-RPC details to users, and apply unsupported ACP controls defensively without masking real user or adapter errors. The final branch should also align docs/changelog with the actual acpx dependency state and the Claude effort workaround.
