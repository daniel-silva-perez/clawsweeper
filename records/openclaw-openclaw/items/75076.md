## What This Changes

The branch adds Control UI build provenance and checksum artifacts, moves assistant media previews to authenticated blob fetches, extends Slack/channel readback health status, updates docs and CI workflows, and adds targeted browser/unit coverage.

## Best Possible Solution

Revise the PR so authenticated local media previews remain compatible with the served CSP, then run the targeted UI/Gateway/Slack/script tests plus the appropriate changed gate before maintainer review.

## Reproduction Assessment

Yes for the review finding: the patch sets audio/video src attributes to blob: URLs, while current main's CSP has no media-src directive and therefore blocks those URLs through default-src. No live browser test was run because this review had to keep the checkout read-only.

## Solution Assessment

No. The hardening direction is reasonable, but converting non-image media to blob URLs must be paired with a media-src 'self' blob: CSP update, or audio/video attachments should remain on authenticated same-origin URLs.
