## Summary

Keep this issue open. Current main still has marker-based external-content wrapping, local web_fetch metadata, narrow user-message provenance, hook blocking, exec approvals, and outbound hooks, but not the requested persistent taint propagation or provenance/risk-aware enforcement across execution, secrets, persistence, and outbound delivery.

## What This Changes

The issue requests an end-to-end external-content security model with persistent taint/provenance, default denial for content-derived execution, exec risk classes, and provenance-aware secret and outbound-action guardrails.

## Best Possible Solution

Keep this as the core security architecture umbrella unless maintainers explicitly split it into canonical implementation issues. The desired end state is structured provenance/taint metadata for external inputs and tool outputs, propagation through summaries and persistence, stable action/risk/provenance fields in hook and approval payloads, and core enforcement that blocks untrusted content from authorizing execution, secret access, or outbound actions without guarded review.
