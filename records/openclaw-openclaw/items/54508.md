## Summary

Keep this issue open. Current main has external-content wrappers, immediate web_fetch metadata, narrow message-level provenance, plugin hook blocking, exec approvals, and outbound hooks, but it still does not implement the requested persistent taint model, content-derived execution denial, exec risk classes, or provenance-aware secret/outbound enforcement. The related issue list supports keeping this as a security architecture umbrella rather than closing it as implemented or superseded.

## What This Changes

The issue requests an end-to-end security model for external content provenance, content-derived execution denial, exec risk classes, and provenance-aware secret and outbound-action guardrails.

## Best Possible Solution

Keep this as the core security architecture umbrella unless maintainers explicitly split it into canonical implementation issues. The desired end state is structured provenance/taint metadata for external inputs and tool outputs, propagation through summaries and persistence, stable action/risk/provenance fields in approval and hook payloads, and core enforcement for content-derived exec, secret access, and outbound actions.
