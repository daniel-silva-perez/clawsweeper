## Summary

Keep this issue open. Current main has external-content markers, immediate web_fetch metadata, narrow message-level provenance, before_tool_call blocking, exec approvals, and outbound hooks, but it still does not implement persistent taint propagation, default content-derived execution denial, exec risk classes, or provenance-aware secret and outbound enforcement. The linked related issues are useful decomposition context, not a canonical shipped fix or superseding tracker.

## What This Changes

The issue requests an end-to-end external-content security model that carries taint/provenance through derived content and gates execution, secret access, persistence, and outbound actions by risk class and direct user intent.

## Best Possible Solution

Keep this as the core security architecture umbrella unless maintainers deliberately split it into canonical implementation issues. The desired end state is structured provenance/taint metadata for external inputs and tool outputs, propagation through summaries and persisted state, stable action/risk/provenance fields in hook and approval payloads, and core enforcement that blocks untrusted content from authorizing execution, secret access, or outbound actions without explicit guarded review.
