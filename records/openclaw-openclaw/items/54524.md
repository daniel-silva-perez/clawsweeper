## Summary

Current main still has the reported overlap. The bundled session-memory hook still writes session-derived Markdown under workspace memory, memory search can still be configured to index both workspace memory and session transcripts, and dedupe remains chunk-id based rather than provenance-aware across the two sources. The prior Codex review comment remains accurate against the current main SHA; related PR #40618 is still an unmerged partial mitigation in the provided context, not a shipped fix for this issue.

## Best Possible Solution

Keep this issue open. The conservative fix is public docs guidance in the memory search/config docs and session-memory hook docs: use `sources: ["memory"]` when relying on the bundled session-memory hook, or disable/redirect the hook when indexing transcripts directly. A stronger code fix would add session provenance or cross-source dedupe keys so memory search can avoid duplicate embeddings/search rows without suppressing legitimate hand-written memory notes. Related PR #40618 can be reviewed as a partial mitigation, but it does not by itself close the duplicate-indexing/docs gap.
