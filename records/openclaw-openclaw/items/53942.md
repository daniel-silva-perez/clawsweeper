## Summary

Keep open. Current main still does not provide a user-level, schema-backed way to selectively override the requested core prompt sections; the available mechanisms are full system prompt replacement, GPT-5 overlay tuning, provider/runtime section replacements, and plugin prompt hooks, none of which cover Safety and OpenClaw Self-Update as user-configurable sections.

## What This Changes

Request to add user-level configuration for selectively replacing or relaxing core system prompt sections such as Safety, Tool Call Style, and OpenClaw Self-Update, or to add a high-trust policy that does so.

## Best Possible Solution

Design a core agent prompt policy surface that is explicit, schema-backed, documented, and available at defaults and per-agent scope. It should make the intended sections addressable without requiring whole-prompt replacement, keep provider/plugin hooks separate from user policy, update config help/generated schema/docs, and add regression tests for Safety, Tool Call Style, and OpenClaw Self-Update behavior.
