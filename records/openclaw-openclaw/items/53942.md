## Summary

Keep this issue open. Current main still does not provide schema-backed user-level overrides or a trust-level policy for the requested Safety, Tool Call Style, and OpenClaw Self-Update prompt sections. The existing support is whole-prompt replacement, narrow provider/runtime section replacement, GPT-5 overlay controls, and plugin prompt hooks; those do not satisfy the requested selective user-level policy.

## Best Possible Solution

Keep this open for core agent configuration design. A conservative implementation would add documented, schema-backed default-level and per-agent prompt policy for selected sections, or an explicit trust policy, then wire it through config types/schema/generated help, gateway config mutation guards, src/agents/system-prompt.ts, docs, and regression tests proving Safety, Tool Call Style, and OpenClaw Self-Update guidance can be selectively replaced or relaxed without requiring full systemPromptOverride. Coordinate with the broader #25493 prompt-template request, but this issue's narrower safety/update-permission requirement remains useful and unsatisfied.
