## Summary

Keep this issue open. Current main has useful pieces for multi-agent routing, session switching, agent configuration, status metadata, and sub-agent/background collaboration, but it still does not implement the central requested product shape: an agent-as-contact chat UI with visible agent availability plus first-class group chats where a user invites multiple agents into one shared conversation.

## Best Possible Solution

Keep this open as a product/UI umbrella unless maintainers intentionally split it. The best path is to define acceptance criteria for a Control UI agent contact/sidebar experience, per-agent session switching/history, visible per-agent status, and a first-class multi-agent conversation or invitation model, while reusing existing agents.list, sessions.list, session routing, and sub-agent primitives where they fit.
