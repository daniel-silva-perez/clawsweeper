## Summary

Keep this issue open. Current main has useful multi-agent primitives and partial UI pieces, including an agent-grouped session selector, Agents configuration view, isolated multi-agent routing, sub-agents, agent/session tools, and experimental WhatsApp Broadcast Groups. It still does not implement the central requested product shape: a chat-app UI where agents appear as contacts with visible availability and users can create first-class group chats that invite multiple agents into one shared conversation.

## Best Possible Solution

Keep this open as a product/UI umbrella unless maintainers intentionally split it. The best path is to define acceptance criteria for a Control UI agent contact/sidebar experience, per-agent conversation switching/history, visible per-agent availability, and a first-class multi-agent conversation or invitation model, while reusing existing `agents.list`, `sessions.list`, session routing, Broadcast Groups, and sub-agent primitives where they fit.
