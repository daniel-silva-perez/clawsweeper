## Summary

Keep open. Current main still has the reported Overview behavior, so this PR is not obsolete or implemented. The UI hunk is also not safe to land as-is because it preserves a gateway token while the endpoint is edited, conflicting with the current per-gateway token/storage boundary. The bundled memory-lancedb retry hunk is unrelated and current main already covers that retry path more completely.

## Best Possible Solution

Keep this PR open for rework against #41545. The best path is to split or drop the memory-lancedb hunk, then implement a Control UI fix that improves in-progress URL editing ergonomics without carrying credentials across a real normalized gateway change: clear the token, reload the token for the selected gateway scope, or require explicit confirmation when the endpoint changes. Add focused regression coverage for both typing/editing behavior and per-gateway token isolation, and coordinate with #41546 and #42001.
