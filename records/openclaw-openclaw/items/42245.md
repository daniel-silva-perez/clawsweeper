## Summary

Keep open. Current main still does not implement the requested `cron add` warning when an agent-turn job omits `--agent`, and the linked cron UX issue remains open. This PR is still a focused implementation candidate rather than obsolete cleanup; the supplied diff only touches the cron CLI command and its tests, so I did not find an unexplained security or supply-chain concern.

## Best Possible Solution

Refresh this PR onto current main and keep it as the focused implementation candidate for the linked cron UX issue. The implementation should add the warning in `src/cli/cron-cli/register.cron-add.ts` after payload classification, emit via `defaultRuntime.error(theme.warn(...))` only for `agentTurn` jobs with no explicit nonblank `--agent`, skip `systemEvent` jobs, preserve JSON stdout, and keep targeted coverage in `src/cli/cron-cli.test.ts` for omitted, blank, explicit-agent, session-key, and system-event cases.
