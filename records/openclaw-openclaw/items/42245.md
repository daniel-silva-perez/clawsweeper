## Summary

Keep open. Current main still does not implement the requested `cron add` warning when an agent-turn job omits `--agent`; the linked cron UX issue remains open, and this PR is still a focused implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Refresh this PR onto current main and keep it as the focused implementation candidate. The implementation should add the warning in `src/cli/cron-cli/register.cron-add.ts` after payload classification, emit through `defaultRuntime.error(theme.warn(...))` only for `agentTurn` jobs with no explicit nonblank `--agent`, skip `systemEvent` jobs, preserve JSON stdout, and keep targeted tests in `src/cli/cron-cli.test.ts` for omitted, blank, explicit-agent, session-key, and system-event cases.
