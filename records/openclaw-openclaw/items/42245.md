The implementation should add the warning in `src/cli/cron-cli/register.cron-add.ts` after payload classification, emit via `defaultRuntime.error(theme.warn(...))` only for `agentTurn` jobs with no explicit nonblank `--agent`, skip `systemEvent` jobs, preserve JSON stdout, and keep targeted coverage in `src/cli/cron-cli.test.ts` for omitted, blank, explicit-agent, session-key, and system-event cases.
