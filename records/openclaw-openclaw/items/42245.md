## Summary

Keep open. Current main still does not implement the requested `cron add` warning when an agent-turn job omits `--agent`; the linked cron UX issue remains open, and the merged cron-tool session-agent inference work is adjacent rather than a replacement for this CLI warning. The PR remains a focused implementation candidate, but it should be refreshed against current main before maintainer review.

## Best Possible Solution

Refresh this PR onto current main and keep it as the focused implementation candidate. The best implementation is to add the warning in `src/cli/cron-cli/register.cron-add.ts` after payload classification, emit it through `defaultRuntime.error(theme.warn(...))` only for `agentTurn` jobs with no explicit nonblank `--agent`, leave `systemEvent` jobs alone, and keep focused tests in `src/cli/cron-cli.test.ts` for omitted, blank, explicit-agent, session-key, and system-event cases.
