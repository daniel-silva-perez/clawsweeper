## Summary

Keep this PR open for maintainer follow-up. Current main still throws when JPEG optimization cannot produce any output, so the reported media failure remains real, but this patch should not merge as written because its catch-all raw fallback can bypass the existing pixel-input guard and can return non-JPEG bytes through a path that labels them as JPEG. The newer open consolidation PR #74243 appears to track the safer policy direction, but it has not merged yet.

## What This Changes

The PR replaces `optimizeImageToJpeg()`'s terminal failure throw with a return of the original source buffer using `resizeSide: 0` and `quality: 0`.

## Best Possible Solution

Maintain the current guard boundaries while adding a scoped fallback for valid, already-under-cap images when the optimizer backend cannot produce output. The fallback should preserve the original MIME type, keep HEIC conversion and pixel/dimension guard failures fail-closed, retain optimizer diagnostics, and include focused regression tests; #74243 looks like the closest open candidate for that end state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep pixel-limit failures fail-closed:** `src/media/web-media.ts:657-662`
  - body: `resizeToJpeg()` enforces the image pixel-input limit before optimizer work. This fallback catches every resize failure and returns the original bytes, so a byte-small but pixel-oversized image that current main rejects can pass through `optimizeAndClampImage()` as long as it is under `maxBytes`. Limit the fallback to safe optimizer/backend failures after preserving the dimension guard.
  - confidence: 0.9
- **[P2] Preserve the original MIME type on fallback:** `src/media/web-media.ts:657-662`
  - body: For non-HEIC sources, `source` is the original buffer. `optimizeImageWithFallback()` wraps every `optimizeImageToJpeg()` return as `format: "jpeg"`, so falling back with PNG/WebP bytes makes `optimizeAndClampImage()` emit `image/jpeg` for non-JPEG data. Move the raw fallback to a layer that can keep the original content type or only fallback for actual JPEG input.
  - confidence: 0.92
