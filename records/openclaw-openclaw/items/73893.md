## Summary

Current main already implements the intended gateway image-capability behavior. Configured `models.providers.*.models[]` entries are merged into the model catalog with `input` metadata preserved, image support resolution uses normalized provider/model lookup, explicit-provider mismatches fail closed, and providerless lookups only use a unique model match. The latest PR head only adds a thin wrapper around the same shared lookup plus one overlapping regression test and a duplicate changelog entry.

## Best Possible Solution

Close this PR as redundant. Keep the current main implementation that merges configured provider model rows into the gateway catalog, preserves configured `input` metadata, and uses the shared normalized lookup semantics. If maintainers still want the exact extra mixed-case fixture from this PR, it can be added as a small test-only follow-up without merging the redundant helper or duplicate changelog entry.
