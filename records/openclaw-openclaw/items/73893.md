## Summary

Current main already implements the intended gateway image-capability behavior. Configured `models.providers.*.models[]` entries are merged into the model catalog with `input` metadata preserved, `resolveGatewayModelSupportsImages` uses the shared normalized catalog lookup, explicit-provider mismatches fail closed, and providerless lookups only use a unique model match. Existing regression tests and docs cover the same behavior this PR is trying to add. The PR diff touches only `CHANGELOG.md`, `src/gateway/session-utils.ts`, and `src/gateway/session-utils.test.ts`, so the security review found no workflow, dependency, lockfile, publishing, or script-surface changes.

## Best Possible Solution

Close this PR as redundant. Keep the current main implementation that centralizes normalized catalog lookup in `findModelCatalogEntry`, preserves configured model `input` metadata in the catalog, and retains the existing regression tests. A future follow-up should only add a small extra test if maintainers want the exact inverse mixed-case fixture from this PR, but no behavior change needs to merge here.
