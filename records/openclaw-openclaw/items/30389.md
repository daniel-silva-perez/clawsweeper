## Summary

Keep open. Current main already covers the attachment-picker and chat model-picker portions, and skills are discoverable through slash-command and Skills surfaces, but the requested composer-level skill/capability menu with user-facing categories and routing semantics is still not implemented or clearly superseded.

## What This Changes

The issue requests a WebChat file attachment button, a model selection dropdown, and a categorized skill/capability menu in the chat UI.

## Best Possible Solution

Keep the shipped upload and model-picker behavior, then either document slash commands and the Skills surfaces as the intended capability UX or design a narrow composer capability menu backed by commands.list, skill metadata, and session tool state after agreeing on categories and routing semantics.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-state check: attachment upload and model picker are present, while no composer-level categorized capability menu exists beyond slash-command and Skills-page discovery.

## Solution Assessment

Unclear. The existing upload and model-picker implementation is a maintainable fit, but the remaining menu should not be implemented until maintainers decide whether a new composer control is preferable to the current slash-command and Skills surfaces.
