## Summary

Keep open. Current main and the latest shipped docs cover the attachment picker and chat model picker portions, and the existing issue discussion correctly identified that partial implementation. The requested composer-level categorized skill/capability menu is still not present or clearly superseded by the slash-command, command-palette, and Skills surfaces.

## What This Changes

Request to add a WebChat attachment button, chat model selection dropdown, and categorized skill/capability menu to the Control UI chat experience.

## Best Possible Solution

Keep the shipped upload and model-picker behavior. For the remaining capability UX, either document slash commands plus the Skills/command-palette surfaces as the intended interface, or design a narrow composer capability menu backed by commands.list, skill metadata, and session-effective tool state after maintainers choose the category and routing model.

## Reproduction Assessment

Not applicable as a feature request. Source, docs, changelog, and test inspection provide a high-confidence current-state check: uploads and model selection are present, while the categorized composer capability menu remains absent.

## Solution Assessment

Unclear. The current upload/model implementation is a maintainable fit, but adding the remaining menu before maintainers decide the capability taxonomy and routing behavior could create a second, confusing command surface.
