## Summary

Keep open. Current main covers the file-upload button and chat header model picker, and it exposes skills through slash commands, command palette, and Skills pages, but the requested composer-level skill/capability menu with user-facing categories and routing semantics is still not implemented or clearly superseded.

## What This Changes

The issue requests WebChat UI additions for discoverable file uploads, model switching, and a categorized skill/capability menu.

## Best Possible Solution

Keep the shipped upload and model-picker work, then either document slash commands and the Skills surfaces as the intended capability UX or add a narrow composer capability menu backed by `commands.list`, skill metadata, and session tool state after agreeing on taxonomy and routing semantics.
