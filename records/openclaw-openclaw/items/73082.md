## Summary

Keep open. Current main still has no skill-declared readonly exec scope or per-skill readonly command allowlist, and the closest existing control, `autoAllowSkills`, is a broader implicit skill-bin trust path rather than the bounded canary/healthcheck contract requested here.

## What This Changes

Request to add a skill-declared readonly exec scope and per-skill readonly command allowlist so healthcheck/canary skills can auto-run bounded introspection commands without repeated exec approval prompts.

## Best Possible Solution

Define an additive exec approval contract that binds readonly auto-approval to trusted skill identity plus explicit argv/pattern rules, keeps known side effects fail-closed, and updates protocol schema, config/CLI/UI surfaces, docs, and focused approval tests together.

## Reproduction Assessment

Not applicable as a runtime bug report. The missing capability is high-confidence statically reproducible on current main by inspecting skill metadata parsing, exec approval schema, docs, and node-host allowlist behavior.

## Solution Assessment

Unclear. The requested direction is plausible, but the exact `execScope: readonly` design should be settled by maintainers/security rather than implemented as a narrow cleanup patch.
