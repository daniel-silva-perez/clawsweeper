## Summary

Keep this PR open. Current main does not implement the PR’s central changes: routed replies still do not carry an outbound provider idempotency key into channel delivery or Telegram send paths, and Discord message actions still pass raw send/poll targets through. The PR also needs maintainer review before merge because the existing bot review identified a plausible Telegram multi-chunk/media dedupe regression in the proposed implementation.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to revise the Telegram idempotency design so chunked text/media sends do not collide, add focused regression coverage for routed idempotency and Discord target normalization, and explicitly decide whether Discord send/poll actions should default bare IDs to channels despite the current docs saying bare numeric IDs are ambiguous.
