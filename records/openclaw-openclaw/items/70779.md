## Summary

Keep open. Current main still unconditionally sends `stream_options.include_usage` for OpenAI-compatible chat-completions streaming, even though the compat model keeps a `supportsUsageInStreaming` flag and preserves explicit `false` overrides. That matches the reporter's suspected regression surface, and I found no current source, docs, changelog, or related merged fix proving the custom Gemini-compatible incremental streaming failure is resolved.

## Best Possible Solution

Keep this issue open and implement a narrow OpenAI-completions request-shaping fix: honor the resolved `supportsUsageInStreaming` capability, including explicit `false`, when deciding whether to send `stream_options.include_usage`, while preserving the field for OpenAI/native endpoints and manifest-declared local providers that support streamed usage. Add regression coverage for a custom Gemini-compatible provider where the field is omitted and incremental text deltas still flow to Control UI/Feishu-style partial delivery.

## Clownfish Work Prompt

Fix the OpenAI-compatible streaming regression reported in openclaw/openclaw#70779. Current `buildOpenAICompletionsParams` in `src/agents/openai-transport-stream.ts` always sends `stream_options: { include_usage: true }`, ignoring `compat.supportsUsageInStreaming`, including explicit `false` preserved by `normalizeModelCompat`. Make request shaping compat-aware so custom Gemini-compatible OpenAI-completions endpoints can omit `stream_options.include_usage` when unsupported, while preserving it for OpenAI/native endpoints and manifest-declared local providers such as vLLM, SGLang, LM Studio, and known local backends that advertise streamed usage. Add/adjust tests in `src/agents/openai-transport-stream.test.ts`, `src/agents/openai-completions-compat.test.ts`, and/or `src/agents/model-compat.test.ts` to cover: explicit `compat.supportsUsageInStreaming: false` omits `stream_options`; explicit true and known local/native providers keep it; generic custom providers do not regain it accidentally. Do not change Feishu card streaming policy or broad channel behavior. Changelog entry expected under Unreleased Fixes, crediting the reporter, because this is user-facing streaming behavior.
