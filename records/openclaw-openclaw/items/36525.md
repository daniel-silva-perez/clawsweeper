## Summary

Keep this issue open. Current main still implements the existing sandbox workspace contract: `rw` selects the agent workspace for `/workspace`, non-`rw` modes are read-only for managed workspace writes, and non-`none` split workspaces can still expose the agent workspace at `/agent`; the linked #46300 is narrower and does not cover this broader semantics refactor.

## Best Possible Solution

Keep this issue open for an explicit sandbox contract decision. If maintainers accept the refactor, update `ensureSandboxWorkspaceLayout`, Docker/browser/SSH/OpenShell mount builders, filesystem bridge write gates, tests, config/status docs, and public sandbox/security docs so `/workspace` consistently represents the sandbox workspace in sandbox mode and the main workspace is exposed only through documented explicit binds. Treat #46300 and #37634 as narrower writable-`none` follow-ups, not as full replacements for this broader contract change.
