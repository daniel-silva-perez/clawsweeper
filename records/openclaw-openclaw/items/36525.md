## Summary

Keep this issue open. Current main still has the existing `workspaceAccess` contract: `rw` maps `/workspace` to the agent workspace, non-`rw` uses the sandbox workspace but keeps managed writes read-only, and `ro` split layouts expose the agent workspace at `/agent`; the requested semantics change is broader than the open writable-`none` PR.

## Best Possible Solution

Keep this open for explicit sandbox/security maintainer review. If maintainers accept the proposed semantics, implement it as a coordinated contract change across sandbox layout resolution, Docker/browser/SSH/OpenShell mount builders, FS bridge write gates, embedded sandbox metadata, memory/bootstrap behavior, tests, config/status surfaces, and public sandbox docs; treat #37634 and #46300 as narrower writable-`none` work, not replacements for this broader decision.
