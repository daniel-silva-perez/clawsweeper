## Summary

Keep open. Current main still implements the existing sandbox workspaceAccess contract: `rw` maps `/workspace` to the agent workspace, non-`rw` modes use the sandbox workspace but keep managed writes read-only, and split non-`none` layouts can still expose the agent workspace at `/agent`. The open #46300 PR is narrower writable-`none` work and does not supersede this broader sandbox semantics refactor.

## Best Possible Solution

Keep this open for explicit sandbox/security maintainer review. If maintainers accept the proposed semantics, implement it as a coordinated contract change across sandbox layout resolution, Docker/browser/SSH/OpenShell mount builders, FS bridge write gates, embedded sandbox metadata, memory/bootstrap behavior, tests, config/status surfaces, and public sandbox docs. Treat #37634 and #46300 as narrower writable-`none` work, not replacements for this broader semantics decision.
