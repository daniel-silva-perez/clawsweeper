## Summary

Keep this PR open. The proposed fix is directionally valid, but current main still swallows `mkdirSync` failures in the active memory host SDK helper, while this PR edits the obsolete `src/memory/internal.ts` path that no longer exists on main.

## Best Possible Solution

Keep this PR open and request a focused rebase: remove the swallowed `mkdirSync` catch in `packages/memory-host-sdk/src/host/internal.ts`, keep `src/memory-host-sdk/host/internal.ts` coherent if that mirror remains maintained, and add a narrow Vitest regression that forces `mkdirSync` to throw and verifies the original filesystem error propagates through `ensureDir` or the memory database open path.
