## Summary

Keep this PR open. Current main still swallows real `mkdirSync` failures in the active memory host SDK implementation, while this PR changes the now-untracked `src/memory/internal.ts` path. The intended bug fix remains valid, but the branch needs a focused rebase onto the current files plus a regression test.

## Best Possible Solution

Keep this PR open and ask for a focused rebase onto current main: remove the swallowed `mkdirSync` failure from `packages/memory-host-sdk/src/host/internal.ts`, keep `src/memory-host-sdk/host/internal.ts` coherent if that source-side copy remains maintained, and add a narrow Vitest regression proving real mkdir failures such as EACCES, ENOSPC, or EROFS propagate from `ensureDir()`.
