## Summary

Keep this PR open. Current main still swallows real `mkdirSync` failures in the active memory-host-sdk `ensureDir()` implementation, so the intended bug fix is not implemented on main. The PR is still useful, but its branch edits the old `src/memory/internal.ts` path, which is no longer tracked, so it needs a focused rebase onto the current memory-host-sdk files rather than cleanup closure.

## Best Possible Solution

Keep this PR open and ask for a focused rebase onto current main: remove the swallowed `mkdirSync` failure in `packages/memory-host-sdk/src/host/internal.ts`, keep the source-side memory-host-sdk path consistent where needed, and add a targeted Vitest regression proving `ensureDir()` propagates real mkdir failures such as EACCES, ENOSPC, or EROFS. Avoid unrelated memory indexing, SDK surface, dependency, or packaging changes.
