## Summary

Keep this PR open. The intended bug fix is still valid, and the discussion already identified the right failure mode, but current main still swallows real `mkdirSync` failures in the active memory host SDK implementation while this PR edits the now-untracked `src/memory/internal.ts` path. The branch needs a focused rebase onto the current files plus a regression test.

## Best Possible Solution

Keep this PR open and ask for a focused rebase onto current main: remove the swallowed `mkdirSync` failure from `packages/memory-host-sdk/src/host/internal.ts`, keep `src/memory-host-sdk/host/internal.ts` coherent if that copy remains maintained, and add a small Vitest regression proving real mkdir failures such as EACCES, ENOSPC, or EROFS propagate from `ensureDir()`.
