## Summary

Keep this PR open. Current main still has the swallowed mkdir failure in the active memory host SDK helper, while the PR patch targets the obsolete `src/memory/internal.ts` path. The change needs a focused rebase onto the current memory-host-sdk files plus a regression test.

## Best Possible Solution

Keep the PR open and request a focused rebase: apply the catch removal to `packages/memory-host-sdk/src/host/internal.ts`, keep `src/memory-host-sdk/host/internal.ts` coherent if that copy remains maintained, and add a narrow Vitest regression that forces `mkdirSync` to throw and verifies the original filesystem error propagates from `ensureDir` or the memory database open path.
