## Summary

Current main still does not make macOS Voice Wake use the selected session target. The macOS voice paths forward with default `sessionKey: "main"` and `channel: .webchat`, and the overlay auto-send path constructs `ForwardOptions` with only `voiceWakeTrigger`. Gateway-level `voicewake.routing` can route by configured wake trigger, but the macOS app does not wire its selected session/WebChat session state into Voice Wake forwarding.

## Best Possible Solution

Keep this issue open and fix the macOS Voice Wake forwarding path to resolve the active selected session target at send time, then pass the resolved session key and delivery channel/to fields into `VoiceWakeForwarder.ForwardOptions`. Add a macOS unit test covering a non-main selected session, and decide whether the source of truth should be `WebChatManager.activeSessionKey`, the menu/session picker state, or a Gateway `voicewake.routing` default target set by the app.
