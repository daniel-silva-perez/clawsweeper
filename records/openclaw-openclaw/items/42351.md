## Summary

Keep open. Current main still does not register `im.chat.access_event.bot_p2p_chat_entered_v1` in the Feishu dispatcher, while the default Feishu connection mode remains WebSocket and the WebSocket monitor passes that fixed dispatcher to the Lark SDK. The only related implementation PR is closed unmerged, so the reported missing-handler bug is still actionable in this repo.

## Best Possible Solution

Keep this issue open. Add a Feishu-owned no-op or low-noise handler for `im.chat.access_event.bot_p2p_chat_entered_v1` in `extensions/feishu/src/monitor.account.ts`, with focused coverage that the handler is registered on the WebSocket monitor path. Separately reproduce the restart path to determine whether only the missing handler needs fixing or whether Feishu WebSocket supervision also needs a generic resilience change.
