## Summary

Current main still does not implement this PR's intended restart-shutdown reply drain. Main has pre-restart deferral for active work and pending replies, but `server.close()` still accepts only reason/restart metadata, does not receive a remaining drain budget, does not poll `getTotalPendingReplies()`, and does not abort active chat runs before closing clients/channels. The existing open contributor PR remains the implementation candidate rather than an obsolete or implemented change.

## Best Possible Solution

Keep this PR open for maintainer review as the active implementation candidate. The best path is a narrow gateway fix that passes the remaining restart drain budget into `server.close()`, drains pending reply dispatchers and active chat runs before socket/channel teardown, aborts remaining active chat runs when the bounded budget is exhausted, preserves normal stop semantics, and keeps unrelated qa-lab changes out unless they are required for regression coverage.
