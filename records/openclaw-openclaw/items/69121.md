## Summary

Current main still does not implement the PR's intended close-time restart drain. Main has pre-restart deferral that counts pending replies, but `runGatewayLoop` does not pass any remaining drain budget to `server.close()`, `GatewayServer.close` has no `drainTimeoutMs` option, and `createGatewayCloseHandler` does not poll pending replies or active chat runs before tearing down channels and sockets. This PR remains an active implementation candidate, with maintainer review needed for the zero-budget abort path and the unrelated qa-lab mock-provider changes.

## Best Possible Solution

Keep this PR open for maintainer review as the active implementation candidate. The best fix is a narrow gateway change that passes the remaining restart drain budget into `GatewayServer.close`, drains pending reply dispatchers and active chat runs before channel/socket teardown, aborts remaining active chat runs when the bounded budget is exhausted including zero-budget restarts, preserves normal stop semantics, and removes or justifies the qa-lab mock-provider changes.
