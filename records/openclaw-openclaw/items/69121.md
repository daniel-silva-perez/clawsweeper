## Summary

Current main still does not implement the PR's intended close-time restart drain. Pre-restart deferral counts pending replies, but `runGatewayLoop` still calls `server.close()` without a remaining drain budget, `GatewayServer.close` has no `drainTimeoutMs` option, and `createGatewayCloseHandler` does not poll pending replies or active chat runs before channel/socket teardown. This PR remains an active implementation candidate that needs maintainer review for the zero-budget abort behavior and the qa-lab scope question.

## What This Changes

The PR adds a bounded restart-shutdown drain for pending reply dispatchers and active chat runs, threads the remaining restart budget from the gateway run loop into server close, expands gateway tests, and also changes qa-lab mock OpenAI memory-trigger behavior.

## Best Possible Solution

Keep this PR open for maintainer review as the active implementation candidate. The best fix is a narrow gateway change that passes the remaining restart drain budget into `GatewayServer.close`, drains pending reply dispatchers and active chat runs before channel/socket teardown, aborts remaining active chat runs when the bounded budget is exhausted including zero-budget restarts, preserves normal stop semantics, and removes or justifies the qa-lab mock-provider changes.
