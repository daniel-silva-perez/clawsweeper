## Summary

Keep open. Current main still does not implement `x-openclaw-workspace` or an equivalent per-request workspace override for `/v1/chat/completions`; the only workspace override path I found is internal spawned-subagent metadata, not an external HTTP contract. The active KinthAI and KeepMyClaw discussion supports a real multi-tenant deployment need, with security design still unresolved.

## Best Possible Solution

Keep this issue open for a deliberate core Gateway design: an opt-in OpenAI-compatible HTTP workspace override with trusted-proxy/spoofing guidance, absolute path and tenant-root validation, tests for session/workspace isolation, and docs on the public OpenAI chat completions API page. The existing spawned-subagent workspace inheritance should stay internal unless maintainers explicitly extend it into a safe HTTP contract.
