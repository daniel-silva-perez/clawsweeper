## Summary

Keep this issue open. Current main still reloads skill directories synchronously through the normal workspace loader; the only reuse I found is session-level `skillsSnapshot` reuse and a narrow bundled-name cache, not the proposed validated SKILL.md cache with metadata invalidation, containment rechecks, pruning, and bounded eviction. The author-linked PR #59547 is closed and unmerged in the provided GitHub context, so it is not a shipped implementation.

## Best Possible Solution

Review a fresh, scoped implementation for the skill loader itself: add a bounded in-memory cache for direct single-skill loads keyed by source and resolved directory, validate hits with `SKILL.md` metadata, re-run containment and anti-symlink checks before serving cached entries, prune entries under scanned roots, and cover unchanged, modified, deleted, symlink-escape, oversized, nested-root, and eviction cases. The session snapshot/watch code can remain a higher-level reuse layer, but it does not replace the loader cache requested here.
