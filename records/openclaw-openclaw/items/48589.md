## Summary

Keep this PR open. Current main still does not implement the PR’s central TUI main-session continuity behavior: the default TUI session still resolves to the normal agent main key, the gateway TUI client still uses a fresh random instance ID per process, there is no affinity-key/origin-target module, and there is no orphan short-reply reattachment path. The PR remains potentially useful, but it needs maintainer review and a fix for the review-raised misrouting risk before it can land.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto current main, align affinity detection with the current `openclaw-tui` client identity, persist the TUI instance ID only for the TUI surface, restrict orphan short-reply reattachment to default/auto main-session aliases, and keep focused tests for default affinity, wrong-client guards, explicit-session non-reattachment, and restart continuity.
