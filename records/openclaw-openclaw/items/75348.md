The issue reports that GitHub Copilot Anthropic-compatible models still receive `eager_input_streaming` through pi-ai on v2026.4.29 and asks OpenClaw to disable eager tool input streaming for Copilot Claude model definitions.

Add a provider-local Copilot compat normalization so every Copilot model using `anthropic-messages` carries `compat.supportsEagerToolInputStreaming: false`, covering manifest/static catalog rows, default model construction, and forward-compatible synthetic model IDs without changing core transport behavior or the pi-ai dependency.
