## Summary

Keep open. Current main still suppresses ordinary non-timeout exec/bash failures at default verbosity before the mutating-tool warning path can run, so a command-not-found failure can still leave success-looking assistant text as the only visible reply. Recent main adds better handling for non-exec mutating failures, but it does not close the exec failure path reported here.

## What This Changes

Ensure failed exec/bash tool calls, especially command-not-found while creating files or scripts, produce a user-visible failure instead of allowing fabricated success text to stand alone.

## Best Possible Solution

Update the embedded-run payload policy so failed mutating exec/bash actions, including command-not-found during file/script creation, emit a concise default-visible failure warning when the assistant does not explicitly acknowledge the failed action. Preserve explicit warning suppression, verbose raw details, and existing cron timeout behavior.
