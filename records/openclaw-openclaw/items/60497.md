## Summary

Keep open. Current main still records and classifies the exec failure, but the embedded-run payload policy suppresses ordinary non-timeout exec/bash errors at default verbosity and the tests still assert that suppression, so success-looking assistant text can remain the only user-visible payload after a command-not-found failure. The NemoClaw sandbox symlink fix and the open destructive-command catalog PR are adjacent context, not the remaining OpenClaw fail-closed payload fix.

## What This Changes

Report failed exec command-not-found tool errors in the agent response instead of allowing fabricated success confirmations after the tool call failed.

## Best Possible Solution

Update the embedded-run payload policy so failed mutating exec/bash actions, including command-not-found, produce a concise default-visible failure warning and prevent success-looking assistant text from standing alone after the matching action failed, while preserving explicit warning suppression, verbose raw-detail behavior, and existing cron timeout handling.
