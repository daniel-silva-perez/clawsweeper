## Summary

Keep open. Current main still has the reported false-negative shape: zero-payload terminal turns are treated as incomplete, while successful `sessions_spawn` results are tracked as side effects but not as terminal completion.

## What This Changes

The issue asks OpenClaw to treat a pure-relay agent turn that ends after a successful `sessions_spawn` call as complete even when it emits no visible text.

## Best Possible Solution

Add a structured current-attempt completion signal for successful `sessions_spawn` results and let the incomplete-turn classifier treat `status: "accepted"` plus a non-empty `childSessionKey` as terminal completion, while preserving the existing warning for failed spawns and empty turns without successful tool completion.

## Reproduction Assessment

Yes. A focused helper test can build a zero-payload stop attempt with `toolMetas: [{ toolName: "sessions_spawn" }]` and replay metadata from that tool; current `resolveIncompleteTurnPayloadText` will still return the side-effect incomplete-turn warning instead of null.

## Solution Assessment

Yes. The narrow maintainable fix is in the embedded-runner completion classifier and attempt metadata, not a global disable of empty-turn detection or an agent-specific workaround.
