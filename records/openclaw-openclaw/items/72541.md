## Summary

Keep open. Current main still has the reported false-negative shape: successful `sessions_spawn` results are available in the tool path, and `sessions_spawn` is classified as a mutating side-effect tool, but the incomplete-turn classifier does not treat an accepted spawn with a non-empty `childSessionKey` as terminal completion for a zero-payload stop.

## What This Changes

The issue requests that gateway incomplete-turn detection treat a successful terminal `sessions_spawn` relay as complete even when the parent agent emits no text payload.

## Best Possible Solution

Add structured current-attempt metadata for successful `sessions_spawn` results and make the incomplete-turn classifier accept `status: "accepted"` with a non-empty `childSessionKey` as terminal completion, while preserving warnings for failed spawns and empty turns without a successful terminal tool action.

## Reproduction Assessment

Yes. A focused unit test can construct a zero-payload stop attempt with empty assistant text, `toolMetas: [{ toolName: "sessions_spawn" }]`, replay metadata built from that tool, and no tool error; current main will return the side-effect incomplete-turn warning instead of accepting the turn.

## Solution Assessment

Yes. The narrow maintainable fix is to derive a terminal-completion signal from the actual `sessions_spawn` result, not to disable empty-turn detection globally or require each relay agent to emit acknowledgement text.
