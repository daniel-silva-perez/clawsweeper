## Summary

Keep open. Current main still has the reported false-negative shape: the embedded-runner incomplete-turn check treats zero-payload terminal turns as errors, while `sessions_spawn` accepted results are only classified as side effects and are not recognized as terminal success.

## What This Changes

The issue requests that gateway incomplete-turn detection treat a successful terminal `sessions_spawn` result as a complete pure-relay agent turn instead of surfacing a zero-payload failure warning.

## Best Possible Solution

Teach the embedded-runner completion classifier to recognize a current-attempt successful `sessions_spawn` result with `status: "accepted"` and a valid `childSessionKey` as terminal completion, while preserving the existing incomplete-turn warning for empty turns without successful tool completion and for failed spawn results.

## Reproduction Assessment

Yes. A focused unit reproduction can construct an attempt with zero payloads, `stopReason: "stop"`, a current `sessions_spawn` tool result of `{status:"accepted", childSessionKey:"..."}`, and verify current `resolveIncompleteTurnPayloadText` still returns the warning path.

## Solution Assessment

Yes, the narrowest maintainable fix is to add a structured success exemption in `src/agents/pi-embedded-runner/run/incomplete-turn.ts` rather than changing `sessions_spawn` itself or globally disabling empty-turn detection.
