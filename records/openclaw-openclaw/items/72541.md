## Summary

Keep open. Current main still has the reported false-negative shape: accepted `sessions_spawn` turns are recorded as mutating side effects, but the incomplete-turn classifier does not treat an accepted spawn with a child session as terminal completion, so a zero-payload stop can still surface the warning.

## What This Changes

The issue asks OpenClaw to stop flagging pure relay agent turns as incomplete when the turn successfully ends with an accepted `sessions_spawn` call and no user-visible text.

## Best Possible Solution

Add structured current-attempt metadata for successful `sessions_spawn` results and make the incomplete-turn classifier treat `status: "accepted"` with a non-empty `childSessionKey` as terminal completion, while preserving the warning for failed spawns and empty turns without successful terminal tool completion.

## Reproduction Assessment

Yes. A focused unit test can construct a zero-payload stop attempt with `assistantTexts: []`, `toolMetas: [{ toolName: "sessions_spawn" }]`, and side-effect replay metadata; current main still returns the side-effect incomplete-turn warning instead of accepting the turn.

## Solution Assessment

Yes. The best fix is a narrow embedded-runner completion signal derived from the actual tool result, not a global disable of empty-turn detection or an agent-specific acknowledgement workaround.
