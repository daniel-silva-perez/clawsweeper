## Summary

Keep open. Current main still does not implement the requested memory indexing exclusion mechanism or user-facing guidance for benchmark/queryset contamination. The existing code reports missing `extraPaths`, which addresses part of the path-drift diagnosis, but `memorySearch.extraPaths` remains include-only and the builtin memory-core path recursively indexes eligible Markdown under those paths.

## Best Possible Solution

Keep this issue open. Add a documented `memorySearch` exclusion mechanism for the bundled memory-core path and shared config: schema/types/help/docs, `resolveMemorySearchConfig`, file discovery/read access, watcher setup, reindex scope hashing, and regression tests proving benchmark/queryset Markdown under `extraPaths` can be excluded. Also update public memory docs with absolute vs workspace-relative `extraPaths`, `memory status --deep`, reindexing after path changes, broad-path noise, and benchmark/evaluation contamination guidance.
