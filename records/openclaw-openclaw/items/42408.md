## Summary

Keep this issue open. Current main still has include-side `memorySearch.extraPaths` for builtin and QMD memory indexing, but no documented or typed builtin `excludePaths`/glob exclusion contract, and the memory docs still do not cover benchmark/queryset contamination or multi-workspace path hygiene enough to resolve the reported local+hybrid retrieval-quality failure mode.

## Best Possible Solution

Keep this issue open. Add a documented `memorySearch` exclusion mechanism for builtin memory indexing, wire it through config schema/types/help, resolved settings, file discovery/read/watch filtering, and reindex scope hashing, then expand the public memory docs with path hygiene, `memory status --deep`, reindexing after path changes, broad-path noise, and benchmark/evaluation contamination guidance.
