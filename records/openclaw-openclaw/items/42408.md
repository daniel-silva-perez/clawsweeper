## Summary

Keep open. Current main still has include-side `memorySearch.extraPaths` but no builtin `excludePaths`/glob exclusion contract, and the memory docs still do not cover benchmark/queryset contamination or multi-workspace path hygiene deeply enough to resolve the reported local+hybrid retrieval-quality failure mode.

## Best Possible Solution

Keep this issue open. Add a documented `memorySearch` exclusion mechanism for builtin memory indexing, wire it through config schema/types/help, resolved settings, file discovery/read/watch filtering, and reindex scope hashing, then expand public memory docs with path hygiene, `memory status --deep`, reindexing after path changes, broad-path noise, and benchmark/evaluation contamination guidance.
