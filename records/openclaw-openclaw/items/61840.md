## Summary

Keep open. The report is coherent and still matches current main: Feishu audio/media uploads can accept a duration, but sendMediaFeishu still uploads routed audio without computing or passing one, so the reported 0-second voice bubble behavior is not implemented on main.

## Best Possible Solution

Implement this in the Feishu plugin media path: compute or preserve audio duration in milliseconds for native Ogg/Opus voice sends, pass it through sendMediaFeishu to uploadFileFeishu when routing.msgType is audio, and add focused tests that assert the Feishu upload payload includes duration for direct .ogg/.opus and audioAsVoice-transcoded replies while preserving file fallback behavior.
