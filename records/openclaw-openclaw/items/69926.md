## Summary

Keep open. Current main still lacks WhatsApp per-group `allowFrom`: the public config type, strict zod schema, generated channel metadata, docs, inbound access gate, and command-authorization path all resolve only account/channel-level `groupAllowFrom` with `allowFrom` fallback. The related WhatsApp prompt, multi-account inheritance, and denylist/blocklist items do not supersede this narrower per-group sender authorization request.

## What This Changes

Add optional per-group `allowFrom` to WhatsApp root and account-scoped `groups` entries so each group can override the channel/account `groupAllowFrom` sender authorization list.

## Best Possible Solution

Keep this issue open and implement an additive WhatsApp plugin change: add optional `allowFrom` to root and account-scoped WhatsApp group entries, update strict schema/generated metadata/docs, resolve the matched group entry's `allowFrom` as an override before falling back to `groupAllowFrom`/`allowFrom`, and cover inbound access plus command authorization without solving the separate #69874 inheritance change in the same PR.
