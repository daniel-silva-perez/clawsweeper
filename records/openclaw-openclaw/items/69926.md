## Summary

Keep open. Current main still does not implement WhatsApp per-group `allowFrom`: the WhatsApp group config type/schema only accepts `requireMention`, `tools`, `toolsBySender`, and `systemPrompt`, and inbound group authorization still resolves a single account/channel-level `groupAllowFrom` with fallback to `allowFrom`. The related deny/blocklist and multi-account inheritance items do not supersede this per-group sender-authorization request.

## Best Possible Solution

Keep this issue open. The best implementation is an additive WhatsApp plugin change: add optional `allowFrom` to `WhatsAppGroupConfig`, the Zod schema, generated config metadata, and docs; resolve the effective group sender allowlist from the matched WhatsApp group entry before falling back to `groupAllowFrom`/`allowFrom`; cover both root `channels.whatsapp.groups` and `channels.whatsapp.accounts.<id>.groups` with regression tests. Coordinate the multi-account inheritance edge with #69874, but do not close this as a duplicate because per-group sender authorization remains separate work.
