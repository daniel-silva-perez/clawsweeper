## Summary

Keep open. Current main still does not implement WhatsApp per-group `allowFrom`: the WhatsApp group config type, strict schema, generated metadata, docs, inbound access gate, and command-authorization path all use only channel/account-level `groupAllowFrom` with `allowFrom` fallback. The related deny/blocklist PRs and WhatsApp multi-account inheritance issue do not supersede this narrower per-group sender-authorization request.

## Best Possible Solution

Keep this issue open for an additive WhatsApp plugin change: add optional `allowFrom` to root and account-scoped WhatsApp group entries, update the strict schema/generated metadata/docs, resolve the matched group entry's `allowFrom` as an override before falling back to `groupAllowFrom`/`allowFrom`, and add regression coverage for inbound access plus command authorization. Coordinate the multi-account inheritance detail with #69874 rather than closing this as a duplicate.
