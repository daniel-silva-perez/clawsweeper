## Summary

Keep this issue open. Current main still carries session and turn-source metadata through the exec approval request path, but the macOS native prompt neither renders the existing sessionKey nor decodes/displays higher-level requester context such as channel or sender.

## Best Possible Solution

Keep this issue open. The narrow fix belongs in apps/macos/Sources/OpenClaw/ExecApprovalsSocket.swift: trim request.sessionKey and render a non-empty Session detail row in ExecApprovalsPromptPresenter.buildAccessoryView. The fuller fix should extend the native prompt request model and gateway decoding to carry safe requester metadata already present in the TypeScript approval request, then display readable Channel/Sender/Thread context where available with focused macOS prompt coverage.
