Keep this PR open. Current main still lacks the intended Android Screen-tab first-visit behavior and still loads the initial chat session list through the default 50-item bootstrap path. The PR is not obsolete, but it still has actionable review work before it should merge.

Keep this PR open and route it through normal review: keep the single `sessions.list` bootstrap path with the larger initial limit, revise the Screen-tab first-visit implementation so it composes in one stable keep-alive slot without flicker or duplicate WebView creation, and add targeted regression coverage or explicit Android manual verification for both behaviors.
