## Summary

Keep this PR open but do not merge it as-is. The reported Discord allowlist bug is still present on current `main`, but the PR edits the old `src/discord` resolver path, while the active runtime resolver now lives under `extensions/discord` and still routes bare numeric entries through channel lookup first.

## What This Changes

The PR changes the old Discord resolver and tests under `src/discord` so bare numeric allowlist entries try a matching guild before falling back to channel lookup.

## Best Possible Solution

Port the guild-first bare-numeric resolution into `extensions/discord/src/resolve-channels.ts`, update the active plugin tests for guild match and channel fallback, keep explicit prefix semantics unchanged, and leave the obsolete `src/discord` path untouched.
