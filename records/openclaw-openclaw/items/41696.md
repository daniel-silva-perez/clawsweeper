## Summary

Keep this PR open. The proposed behavior is still useful, but current main still routes bare numeric Discord allowlist entries through channel lookup, and the open PR modifies the old `src/discord` path instead of the active `extensions/discord` plugin implementation.

## What This Changes

The PR changes Discord allowlist parsing so bare numeric entries resolve as a matching guild before falling back to channel lookup, with regression coverage in the old `src/discord` resolver tests.

## Best Possible Solution

Port the guild-first bare-numeric resolution into `extensions/discord/src/resolve-channels.ts`, update the active resolver tests to expect guild resolution plus channel fallback, and leave explicit `channel:`/`discord:` and `guild:`/`server:` semantics intact.
