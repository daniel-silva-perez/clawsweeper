## Summary

Keep this PR open. Current main still parses a bare numeric Discord group allowlist target as a channel ID and resolves it through `/channels/<id>` before any guild match. The PR is a focused bug fix with supportive review context, but it targets the old `src/discord` path and needs to be ported to the current `extensions/discord` implementation.

## Best Possible Solution

Port the PR’s behavior into `extensions/discord/src/resolve-channels.ts`: parse bare numeric inputs separately, check joined guilds before `/channels/<id>`, preserve explicit `channel:`/`discord:` and `guild:`/`server:` semantics, and keep channel fallback when no guild matches. Update `extensions/discord/src/resolve-channels.test.ts` so the bare guild ID test expects a resolved guild and add/keep fallback coverage for non-guild bare numeric channel IDs. Address the existing review suggestion with a brief comment if maintainers want that priority documented.
