## Summary

Keep this PR open. The intended Discord allowlist fix is still useful, but current main still routes bare numeric Discord allowlist entries through channel lookup first, and the PR branch edits the old `src/discord` files instead of the active `extensions/discord` plugin implementation.

## What This Changes

The PR changes Discord allowlist parsing and tests in the old `src/discord` resolver so bare numeric entries match known guilds before falling back to channel lookup.

## Best Possible Solution

Port the guild-first bare-numeric resolution into `extensions/discord/src/resolve-channels.ts`, update the active resolver tests to expect guild resolution plus channel fallback, and keep explicit `channel:`/`discord:` and `guild:`/`server:` semantics unchanged.
