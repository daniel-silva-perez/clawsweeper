## Summary

Keep this PR open. Current main still implements the old Discord allowlist resolver behavior: a bare numeric entry is parsed and resolved as a channel ID before any guild lookup, and the current regression test still records that behavior. The PR appears to address a real, still-present bug, but it needs to be rebased or ported from the old `src/discord` paths into the current `extensions/discord` plugin paths before it can merge.

## Best Possible Solution

Keep this PR or equivalent work open, port the guild-first bare-numeric behavior into `extensions/discord/src/resolve-channels.ts`, preserve explicit `channel:` and `guild:` semantics plus channel fallback, and update `extensions/discord/src/resolve-channels.test.ts` before merge.
