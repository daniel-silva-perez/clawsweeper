## Summary

Keep open. Current main still lacks session-based exec trust windows: there are no trust/untrust CLI commands, Gateway RPC methods, protocol schemas, in-memory trust cache, audit module, or slash-command handlers. Related open PRs #46956 and #47696 overlap but are not merged or clearly canonical replacements, and this change is security-sensitive because it intentionally bypasses exec approval prompts for a bounded period.

## Best Possible Solution

Keep this PR open until maintainers choose a canonical trust-window design.
