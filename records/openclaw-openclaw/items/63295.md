## Summary

Keep open: current main still omits `inboundUserContext` from the reset prompt branch that reaches the model. Bare `/new` and `/reset` now acknowledge locally, so the original bare-greeting path changed, but reset-tail, soft-reset, and other model-executed reset paths still need the sender/history envelope. Open PR #72866 targets the omission but is not merged and should be reviewed before this issue closes.

## What This Changes

The issue asks for session-reset model prompts to retain the sender envelope so `/new` or `/reset` can use channel-populated sender metadata on the first model turn.

## Best Possible Solution

Review and land #72866 or an equivalent focused fix after making sure the regression test exercises the actual `isBareSessionReset` branch. Keep the current local acknowledgement behavior for truly bare reset commands unless maintainers intentionally change that product behavior, and include `inboundUserContext` for reset tails, soft reset, and other reset paths that still reach the model.
