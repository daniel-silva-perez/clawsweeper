## Summary

Current main still does not implement the requested sender-envelope behavior for reset prompts that reach the model. Bare `/new` and `/reset` now short-circuit to local acknowledgements, so the original Telegram bare-greeting path has changed, but the reset prompt branch in `get-reply-run.ts` still computes `inboundUserContext` and omits it from `baseBodyForPrompt`. An open related PR, #72866, targets that omission and should be reviewed rather than closing this issue now.

## Best Possible Solution

Keep this open until maintainers decide the intended reset UX and either land #72866 or an equivalent fix for reset prompts that reach the model. If bare `/new` and `/reset` should remain local acknowledgements, document that personalized bare reset greetings are not supported and scope the prompt fix to reset tails, soft reset, and other model-executed reset paths.
