## Summary

Keep open: current main still drops `inboundUserContext` from the model prompt whenever `isBareSessionReset` is true, so reset-tail and soft-reset turns can still lose sender/history metadata. The exact bare `/new` greeting path from the original report has changed because no-tail reset commands now return local acknowledgements, but the remaining prompt assembly bug is still real and open PR #72866 is the focused implementation candidate.

## What This Changes

The issue asks for session reset prompts, especially `/new` or `/reset` turns from channels like Telegram, to preserve the sender envelope built from inbound metadata.

## Best Possible Solution

Review and land PR #72866 or an equivalent focused change that includes `inboundUserContext` in the `isBareSessionReset` prompt array, with regression coverage for reset-tail or soft-reset model execution. Keep the current no-tail local acknowledgement behavior unless maintainers intentionally decide to restore model-generated reset greetings.

## Reproduction Assessment

Yes. Current main has a high-confidence static reproduction path: provide a reset-tail or soft-reset turn with sender metadata or inbound history, and `runPreparedReply` computes the envelope but omits it from `baseBodyForPrompt` before calling the agent. The original no-tail `/new` path no longer reproduces as a model greeting because it now returns a local acknowledgement.

## Solution Assessment

Yes, the proposed direction is the narrow maintainable fix for the remaining bug: put `inboundUserContext` into the reset prompt assembly where the non-reset branch already uses it. A broader change to make no-tail reset greetings model-generated again would be a separate product decision.
