## Summary

Current main still has the prompt-assembly gap: `runPreparedReply` computes `inboundUserContext`, but the `isBareSessionReset` branch does not include it in the prompt sent to the model. The original no-tail bare `/new` greeting path has changed because current main locally acknowledges bare resets, but model-executed reset tails, soft resets, and inbound-history-only reset turns can still lose the sender/history envelope. Keep this open until #72866 or an equivalent fix lands.

## What This Changes

The issue asks for `/new` and `/reset` session-reset turns to preserve the sender and inbound metadata envelope so agents can personalize or use context on the first model-executed reset turn.

## Best Possible Solution

Review and land #72866 or an equivalent narrow change that inserts `inboundUserContext` into the `isBareSessionReset` prompt array, with regression coverage for an actual model-executed reset branch. Preserve the local acknowledgement behavior for no-tail bare `/new` and `/reset` unless maintainers intentionally want to restore model-generated greetings there.
