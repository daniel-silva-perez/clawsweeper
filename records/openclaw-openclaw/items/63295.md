## Summary

Current main still omits the inbound sender envelope from the reset prompt branch that reaches the model. Bare `/new` and `/reset` now return local acknowledgements, so the exact original greeting path has changed, but reset-tail/soft-reset/model-executed reset paths still need a fix. Open PR #72866 targets the production omission and should be reviewed rather than closing this issue now.

## What This Changes

Preserve the `Sender (untrusted metadata)` envelope when `/new` or `/reset` reset prompts are sent to the model so agents can personalize first-turn greetings.

## Best Possible Solution

Keep this issue open until maintainers review and land #72866 or an equivalent fix that includes `inboundUserContext` in model-executed reset prompts. If bare `/new` and `/reset` are intended to stay local acknowledgements, document that personalized bare reset greetings are unsupported and scope the code fix to reset tails, soft reset, and other reset paths that still reach the model.
