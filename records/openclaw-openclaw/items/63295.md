## Summary

Keep open: current main still omits `inboundUserContext` from the `isBareSessionReset` prompt branch, so model-executed reset paths can still lose the sender/history envelope. Bare `/new` and `/reset` now return local acknowledgements without invoking the model, which changes the exact greeting path from the original report, but reset tails and soft-reset turns still fall through to the agent and should preserve the same untrusted metadata. Open PR #72866 already targets this gap and should be reviewed before this issue closes.

## What This Changes

The issue asks for session reset prompt assembly to preserve the sender metadata envelope so channel-triggered reset turns can personalize responses using already-populated sender fields.

## Best Possible Solution

Review and land #72866 or an equivalent focused change that includes `inboundUserContext` in the `isBareSessionReset` prompt array, with regression coverage for the actual reset branch. Keep the current local acknowledgement behavior for no-tail `/new` and `/reset` unless maintainers intentionally want to restore model-generated greetings.
