## Summary

Current main implements the requested gateway resilience: plugin-entry-only config invalidity is handled as degraded startup/reload state instead of a hard gateway crash, and the Brave legacy web-search path is now normalized into plugin-owned config before validation.

## Best Possible Solution

Close as implemented on main. Keep the current degraded startup/reload behavior, legacy Brave web-search normalization, and doctor quarantine path; any remaining plugin-specific runtime crash after startup should be filed as a narrower plugin/runtime bug.
