## Summary

Keep this PR open. Current main still rejects empty `aws-sdk` auth in the media image and audio/video paths, while the PR is a focused implementation candidate for the Bedrock SDK-chain regression tracked by #72031; the earlier empty-key rotation review concern is addressed in the current head and the inspected check-runs are green.

## What This Changes

The PR allows `aws-sdk` media auth paths to pass an empty API key through image, MiniMax fallback, and audio/video execution, adds an `allowEmptyKey` rotation escape hatch, and covers the behavior with media-understanding regression tests.

## Best Possible Solution

Land this focused fix after maintainer review, preserving the narrow `aws-sdk`-only empty-key allowance and the audio/video rotation guard so Bedrock media tools match the already-supported chat auth contract.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: configure Bedrock media auth with `mode: "aws-sdk"` and no static key, then current main reaches `requireApiKey` in the image or audio/video resolver and throws before provider execution.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix I see: it mirrors the existing chat-path allowance, keeps non-`aws-sdk` modes on the existing `requireApiKey` failure path, and addresses the empty-key rotation gap raised in review.
