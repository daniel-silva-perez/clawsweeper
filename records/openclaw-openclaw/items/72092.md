## Summary

Keep open for maintainer review. Current main still has the Bedrock `aws-sdk` media-auth gap, while the PR head appears to address both the original image/audio/video `requireApiKey` failure and the later empty-key rotation gap with green check runs.

## What This Changes

The PR lets media-understanding image, MiniMax fallback, and audio/video provider execution pass an empty API key for `aws-sdk` auth mode, adds an empty-key rotation option, and covers the Bedrock media paths with regression tests.

## Best Possible Solution

Land this focused fix after maintainer review so Bedrock media-understanding behavior matches the shipped chat-path auth contract while keeping non-`aws-sdk` modes on the existing explicit-key failure path.

## Reproduction Assessment

Yes. Configure `amazon-bedrock` media auth with `mode: "aws-sdk"` and no static key; current main reaches `requireApiKey` in the image or audio/video resolver and throws before SDK credential resolution can happen.

## Solution Assessment

Yes. The PR is the narrowest maintainable direction I see: it gates empty-key pass-through on resolved `aws-sdk` mode, avoids storing an empty runtime key, and adds the rotation escape hatch only for the confirmed empty-SDK execution case instead of weakening `requireApiKey` globally.
