## Summary

Keep open. Current main has topic-aware Telegram native-command reply delivery, including a test for `/status` inside a forum topic, but native command-menu sync still registers and clears only the default Telegram command scope, and native `/status` still goes through a separate authorization/session path from normal mention-based text routing. Open PR #6457 remains the closest focused implementation candidate for the group-scope/topic-command gap, so this is not implemented on main yet.

## What This Changes

Make Telegram native `/status` respond consistently in group forum topics when normal mention/text routing works in the same thread.

## Best Possible Solution

Keep this issue as the user-facing tracker until maintainers reproduce the Telegram forum-topic `/status` behavior on current main, then land a Telegram-plugin fix that registers and clears the relevant group command scopes while preserving topic thread parameters through native command auth and delivery. If reproduction shows an authorization rejection rather than delivery loss, clarify the docs/diagnostic reply so users can distinguish config denial from missing topic routing.

Unclear. The issue gives concrete Telegram topic steps, and current code provides a strong static explanation for a native-command-only divergence, but there is no live Telegram run, log, or failing current-main test proving the reported inconsistency end to end.
