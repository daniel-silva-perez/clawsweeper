## Summary

Keep open. Current main still has a distinct Telegram native slash-command path for `/status`; it is topic-aware in several places, but it still applies command authorization separately from ordinary mention/text routing and the native command menu sync still registers only the default Telegram command scope. The open related PR #6457 remains the closest implementation candidate for group-scope command registration and forum-topic command reply parameters.

## Best Possible Solution

Keep this issue open. Reproduce `/status` in a Telegram forum topic on current main with `requireMention:false`, then compare native slash-command sender authorization against `commands.allowFrom`, channel `allowFrom`, `groupAllowFrom`, per-group/per-topic `allowFrom`, and `groupPolicy`. If the failure is Telegram native command availability or topic fallback, review, rebase, or replace #6457 in the Telegram plugin with group-scope command registration, matching cleanup, and regression coverage. If the behavior is intentional command authorization, improve diagnostics/docs so operators can see why normal mention text works while `/status` is denied.
