## Summary

Keep open. Current main has recovery mitigations for stale streaming, but the visible TUI state still has no content-complete or finalizing transition and can remain tied to later lifecycle cleanup or the 30s watchdog. The related duplicate tracker #67052 is also still open, and the merged adjacent TUI recovery PR explicitly left this UX timing work unresolved.

## What This Changes

The issue asks for the TUI streaming status to transition promptly after visible assistant content stops instead of waiting 30-120+ seconds for the full gateway chat lifecycle to finish.

## Best Possible Solution

Add a narrow completion UX path that separates visible assistant-content completion from full run lifecycle completion, preferably with an additive content-complete or finalizing signal before channel/media flush work while keeping the existing watchdog as a recovery fallback.
