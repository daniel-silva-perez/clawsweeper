## Summary

Keep open. Current main has several stale-streaming mitigations, including orphan-final cleanup and a 30s watchdog, but it still lacks the requested content-complete or finalizing transition after visible assistant text stops. Normal TUI idle still depends on terminal lifecycle/chat final paths that run after block reply flush and other completion work, and the same-author duplicate tracker #67052 is still open.

## What This Changes

The issue asks for the TUI streaming indicator to stop or switch to a finishing state promptly after the last visible assistant token, instead of staying busy until later gateway lifecycle finalization.

## Best Possible Solution

Add a narrow, additive TUI/gateway completion UX path that separates visible assistant-content completion from full run lifecycle completion, preferably by emitting a content-complete/finalizing signal from the embedded stream path before post-run delivery/flush work while keeping the existing watchdog as recovery.
