## Summary

Keep open. Current main has recovered some stale TUI streaming edge cases and added a 30s watchdog, but it still does not provide a prompt content-complete or finalizing transition after visible assistant text stops; normal completion remains tied to the later `final` chat event from the gateway lifecycle path. The linked merged PR explicitly says this UX timing issue remains open.

## What This Changes

The issue asks for the TUI streaming indicator to stop or transition promptly after visible response content finishes instead of staying active until full chat lifecycle completion.

## Best Possible Solution

Add a narrow TUI/gateway completion UX path that separates visible assistant-content completion from full run lifecycle completion, preferably through an additive content-complete or finalizing signal, while keeping the watchdog as recovery and preserving concurrent-run/tool-event behavior.
