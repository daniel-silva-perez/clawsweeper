## Summary

Keep open. Current main still serializes same-session agent runs, but the lane wait data is only diagnostic logging; the Gateway accepts before dispatch, dispatch failures still become generic UNAVAILABLE errors, and the CLI has no documented or registered busy/force-queue option.

## Best Possible Solution

Keep this issue open and fix the Gateway/CLI contract. Add a typed session-lane busy or lane-wait-timeout result from the queue or agent dispatch path, propagate lane, queueAhead, and waitedMs through the Gateway final/error response, make `openclaw agent` print an actionable message, and add focused regression coverage for blocked same-session CLI dispatch. If waiting remains the default, add and document an explicit queue policy such as `--force-queue` or `--wait`.
