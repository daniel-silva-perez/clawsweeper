## Summary

Keep open. Current main still serializes same-session agent runs, but the queue wait signal remains diagnostic-only and the Gateway/CLI contract still does not surface a typed session-lane busy or lane-wait timeout with lane, queueAhead, waitedMs, or retry guidance.

## What This Changes

The issue requests that `openclaw agent --agent <id> --message ...` surface an actionable busy or timeout error instead of silently exiting when a same-session lane is blocked.

## Best Possible Solution

Add an additive Gateway/CLI contract for blocked same-session agent dispatch: preserve existing safe serialization, expose a typed lane-busy or lane-wait-timeout terminal result with lane, queueAhead, waitedMs, retry guidance, make `openclaw agent` print it clearly in text and JSON modes, and document any explicit wait/queue policy.
