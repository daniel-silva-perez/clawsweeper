## Summary

Keep open. Current main has partially tightened the macOS LaunchAgent scanner, but it still emits canonical gateway cleanup commands after any remaining extra gateway-like service is detected, and an open same-author PR with closing syntax remains an implementation candidate rather than closure evidence.

## Best Possible Solution

Keep this issue open and review PR #52515, or an updated smaller equivalent, to pass detected service context into cleanup hints, avoid suggesting removal of the active canonical gateway, and align exec SecretRef handling with the current doctor/runtime SecretRef contract. Add focused regressions for custom macOS LaunchAgents and the reported exec SecretRef doctor path.
