## Summary

Keep this PR open. Current main still builds WhatsApp automatic ack reaction options from `sender.jid` only, while the inbound/identity path can produce LID-only group senders; the PR is a narrow WhatsApp plugin fix with matching LID-only and JID-precedence regression tests, and its diff does not introduce an obvious security or supply-chain concern.

## What This Changes

This PR changes WhatsApp automatic ack reaction participant construction to prefer `sender.jid` with `sender.lid` as a fallback and adds regression tests for LID-only and JID-precedence group sender cases.

## Best Possible Solution

Keep the PR open for normal maintainer review and CI/security checks, then land the narrow WhatsApp plugin fix if checks pass. The best implementation is the PR’s local `sender.jid ?? sender.lid` fallback in the ack reaction monitor plus the LID-only and JID-precedence tests; no core API or Baileys send-layer change is needed.
