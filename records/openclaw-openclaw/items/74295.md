## Summary

Keep this PR open. Current main still uses only `sender.jid` when building WhatsApp automatic ack reaction options, while the inbound identity path can normalize a group participant as `sender.lid` with no `sender.jid`; the PR is a narrow plugin-local fix with regression coverage and no visible security-sensitive changes.

## What This Changes

This PR changes WhatsApp automatic ack reaction option construction to fall back from `sender.jid` to `sender.lid` and adds a regression test for LID-only group senders.

## Best Possible Solution

Keep the PR open for normal maintainer review, CI, and ideally the extra JID-precedence regression test; the implementation belongs in the WhatsApp plugin ack reaction monitor and should not require a core change.
