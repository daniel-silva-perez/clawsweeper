## Summary

Keep open. Current main and the latest release still ship the reported default status reaction mapping, including `stallSoft: "🥱"` and `stallHard: "😨"`. The override path exists, but the shared defaults used by Discord, Slack, and Telegram have not changed, and current tests still preserve the old defaults.

## Best Possible Solution

Keep this issue open. Make a small shared channel UX change in `src/channels/status-reactions.ts` to use clearer defaults, update Telegram-specific variants/fallbacks so unsupported emoji do not silently become the old confusing defaults, keep `messages.statusReactions.emojis` as the override path, and update the Discord/Slack/Telegram plus core status-reaction tests and config docs/schema comments to match the shipped defaults.
