## Summary

Keep open. Current main still ships the unclear default status reaction emoji reported here: `stallSoft` is `🥱`, `stallHard` is `😨`, and the broader default set still uses the emotional/status-ambiguous mapping. The config override path exists, but the requested default UX change has not landed.

## Best Possible Solution

Keep this issue open and make a small core UX change in `src/channels/status-reactions.ts`: update `DEFAULT_EMOJIS` to a clearer cross-platform mapping, add explicit expected-value tests for the new defaults and stall timers, update Discord/Slack/Telegram status-reaction tests as needed, and align config docs/help comments with the shipped defaults. Keep `messages.statusReactions.emojis` as the per-user override path.
