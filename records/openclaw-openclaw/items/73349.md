## Summary

Keep this PR open, but do not merge it as-is. The requested exec session/agent env feature is still absent from current main and matches the linked feature request, but the PR's node-host implementation only forwards the values as request env overrides; current node-host shell-wrapper sanitization drops those keys before the final child process runs.

## What This Changes

The PR adds docs, tests, and exec-tool changes intended to inject `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` into gateway, sandbox, PTY, and node-host child-process environments.

## Best Possible Solution

Revise the PR so local exec paths inject trusted values after user env, and node-host paths carry session/agent metadata separately from user env overrides before merging it into the final child env after shell-wrapper sanitization in both TypeScript and macOS runtimes. Add final-child coverage for node-host behavior and the required changelog entry.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: `host=node` exec is always shell-wrapped, and the receiving node-host sanitizers drop non-allowlisted env overrides before spawning the child; the PR currently puts the new vars only in that dropped override payload.

## Solution Assessment

No. Adding the vars to `requestedEnv` is not the best fix for node-host execution because that payload is intentionally treated as untrusted env override data; the safer fix is a trusted runtime metadata path that preserves user override blocking while still setting the final child env.
