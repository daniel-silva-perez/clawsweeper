## Summary

Keep this PR open, but this revision is not ready to merge. Current main still lacks the requested exec metadata injection, and the PR addresses a real linked request, but its node-host path puts the trusted values into the normal requested environment where existing shell-wrapper sanitizers drop them before the child process runs.

## What This Changes

The PR injects `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` into exec child-process environments across local exec paths and node-host forwarding, with docs and regression tests.

## Best Possible Solution

Revise the PR so local exec injects the values after user env merging and node-host execution carries the same trusted metadata after shell-wrapper env filtering in both the TypeScript node host and macOS runtime, with tests proving the final child process sees the values and user-provided env cannot spoof them.
