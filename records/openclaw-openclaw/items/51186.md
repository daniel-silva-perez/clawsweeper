## Summary

Keep this issue open. Current main has partial fixes for the `/hooks/agent` session-context and webhook 404 sides, but the BlueBubbles phone-format routing mismatch remains: DM routing still feeds the inbound sender handle into exact peer matching, and BlueBubbles handle normalization still does not canonicalize `+1...` and `1...` variants before generic `bindings[]` selection.

## Best Possible Solution

Keep this issue open. Implement BlueBubbles-specific canonical handle matching for route bindings and configured conversation bindings so inbound `1XXXXXXXXXX` and configured `+1XXXXXXXXXX` resolve to the same peer before wildcard/catch-all routing, with regression coverage for that exact route order. Preserve the current `/hooks/agent` session rebinding and plugin webhook registry behavior.
