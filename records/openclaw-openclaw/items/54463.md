## Summary

Keep this open. The maintainer discussion already narrowed away the stale docs-only claim but kept the runtime contract active, and current main still has no OpenClaw-side QMD traversal knob, minimum safe-QMD version check, or regression proving symlink-loop temp repos cannot break indexing. Upstream QMD 2.1.0 appears to have safer traversal now, which makes the remaining OpenClaw work a version/docs/runtime-contract decision rather than an implemented-on-main close.

## What This Changes

QMD-backed memory indexing should safely handle workspace-visible temporary monorepo checkouts with symlink-loop package structures without failing indexing with `ENAMETOOLONG`.

## Best Possible Solution

Keep this as an active QMD traversal robustness item. Reconcile OpenClaw with current upstream QMD behavior by either requiring/detecting a cycle-safe QMD version and updating docs/diagnostics accordingly, or adding an OpenClaw-side pre-update exclusion/version guard for older QMD installs; add the controlled symlink-loop repro as regression coverage and coordinate the decision with #40438.
