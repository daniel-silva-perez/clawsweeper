## Summary

Keep this open. Current main still documents the QMD symlink-loop case as a live runtime limitation, and the code still passes plain path/pattern collections to QMD before running `qmd update`; I found no OpenClaw-side traversal pruning, realpath-cycle guard, QMD no-follow contract, or regression coverage that would make the reported `ENAMETOOLONG` failure impossible.

## What This Changes

The issue asks OpenClaw to prevent QMD memory indexing from recursing through symlink loops in workspace-visible temporary monorepos, or to otherwise align runtime behavior and docs around that traversal contract.

## Best Possible Solution

Keep this as an active QMD traversal robustness item. Add the controlled symlink-loop repro as regression coverage, then either require a QMD version with cycle-safe/no-follow traversal or add an OpenClaw-side pre-update pruning/exclusion/version guard before `qmd update` can recurse through workspace-visible temporary monorepos; keep the workaround docs until the runtime fix ships.
