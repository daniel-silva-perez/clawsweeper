## Summary

Keep open. Current main has superseded some peripheral pieces, but it still does not implement the central Feishu `/allowlist channel=feishu` config-edit/listing support or mark Feishu as markdown-capable. The PR also has unresolved allowlist account-resolution review concerns and an unrelated CI runtime version bump, so it should be narrowed and reworked rather than closed or merged as-is.

## Best Possible Solution

Keep this PR open for a narrower rework: add a Feishu-owned `allowlist` adapter using the existing plugin SDK surface while preserving effective merged account allowlists, set Feishu `markdownCapable` metadata in both runtime/package discovery surfaces as needed, add focused Feishu `/allowlist channel=feishu` and markdown-capability regression tests, drop the already-superseded security-fix and hook-schema chunks, and remove or split the unrelated Bun/CI runtime bump before maintainer review.
