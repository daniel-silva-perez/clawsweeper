## Summary

Keep this PR open. Current main has superseded some supporting pieces, but the central Feishu work is still not implemented: Feishu still lacks a plugin allowlist adapter, current `/allowlist channel=feishu` targeting is not handled through plugin channel normalization, and Feishu is still not marked markdown-capable in plugin metadata. The branch also has unresolved review concerns and an unrelated CI runtime version bump, so it should be narrowed and reworked rather than closed or merged as-is.

## Best Possible Solution

Keep this PR open for a narrower rework: add a Feishu-owned allowlist adapter using the existing plugin SDK surface while preserving effective merged account allowlists, set Feishu `markdownCapable` metadata in both runtime/package discovery surfaces as needed, add focused Feishu `/allowlist channel=feishu` and markdown-capability regression tests, drop the already-superseded security-fix and hook-schema chunks, and split or remove the unrelated Bun/CI version bump before maintainer review.
