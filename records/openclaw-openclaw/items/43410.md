## Summary

Keep this issue open. Current main still has representative bundled skills with capability-only descriptions, `canvas` still lacks skill frontmatter metadata, and open unmerged PRs still cover only subsets of the requested cleanup. The descriptions are runtime routing metadata, so this is not just cosmetic wording.

## Best Possible Solution

Keep this issue open as the umbrella for bundled skill description quality. Maintainers should review and either land or close #44329 and #40453, then update the remaining affected `skills/*/SKILL.md` descriptions or split leftovers into narrower follow-up issues. The complete path should also decide whether `skills/canvas/SKILL.md` should become a loadable skill with YAML frontmatter and whether public skill-authoring docs should be strengthened to match the bundled skill-creator guidance.
