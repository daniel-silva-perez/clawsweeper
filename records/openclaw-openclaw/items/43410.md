## Summary

Keep open. Current main still has representative bundled skills with capability-only descriptions, `canvas` still has no frontmatter description, and the open related PRs cover only subsets of the requested cleanup. The loader and prompt formatter make skill descriptions part of the runtime routing contract, so this remains functional metadata work rather than cosmetic wording.

## What This Changes

The issue requests trigger-oriented `description` metadata for affected built-in skills, plus cleanup for `canvas` missing a description field and `gh-issues` being near or above the recommended skill body size.

## Best Possible Solution

Use this issue as the umbrella until maintainers either merge or close the related subset PRs, then finish the remaining bundled skill metadata pass in one consolidated change or in clearly scoped follow-ups. The end state should align representative built-in descriptions with `skills/skill-creator` guidance, decide whether `canvas` should be a loadable skill with YAML frontmatter, and keep oversized skill bodies from growing further.
