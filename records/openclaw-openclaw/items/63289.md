## Summary

Keep this issue open. The current code still provides tools, advisory prompt guidance, tool-call normalization, and missing-result repair, but it does not enforce a fresh calendar or raw shell-output invocation when the assistant emits a plain final answer with no matching tool call.

## What This Changes

The issue requests that factual, time-sensitive calendar and raw-output requests require a fresh authoritative tool invocation and never be answered from prior context without a corresponding tool trace.

## Best Possible Solution

Define a narrow product/runtime contract for explicit freshness or raw-output turns, then enforce it in the embedded runner or prompt/tool policy boundary with regression coverage proving the assistant cannot finalize those specific turns without a matching tool event.
