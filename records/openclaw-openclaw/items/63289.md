## Summary

Keep this open. Current main still has advisory prompt/tool guidance, the bundled gog skill, tool-call normalization, diagnostics, and missing-result repair for emitted tool calls, but it does not enforce a fresh calendar/gog/exec invocation before accepting a plain final answer to an explicit factual or raw shell-output request.

## What This Changes

The issue reports that factual calendar and explicit raw shell-output requests can receive fabricated final answers without any corresponding gog/calendar or shell tool invocation.

## Best Possible Solution

Define a narrow runtime contract for explicit freshness or raw-output turns, then enforce it in the embedded runner, tool policy, or a documented hook boundary so the assistant retries, blocks, or asks a clarifying question instead of finalizing when the required tool event is missing.
