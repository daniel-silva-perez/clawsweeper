## Summary

Keep this issue open. Current main has prompt guidance and tool-call plumbing, plus a gog skill with calendar commands, but I did not find a deterministic core guard that prevents an assistant from answering a factual calendar/exec request without a fresh tool invocation. The related reports do not fully supersede this no-tool-call calendar path.

## Best Possible Solution

Keep open and have a maintainer define the desired contract for explicit factual or shell-output requests. A good fix would likely add a narrow, testable guard or stronger run policy for explicit “run this command / paste raw output / check today’s calendar” prompts, plus a live or mocked regression around Anthropic Haiku and the gog calendar skill, without pretending every arbitrary factual question can be forced into a tool call by prompt text alone.
