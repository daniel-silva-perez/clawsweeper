## Summary

Keep this issue open. Current main still has advisory prompt guidance, tool exposure, tool-call diagnostics, and missing-result repair for emitted tool calls, but I found no runtime contract that requires a fresh calendar/gog/exec invocation before a plain final answer to an explicit factual or raw shell-output request.

## What This Changes

The issue reports that factual calendar/raw shell-output requests can receive fabricated final answers without the relevant gog/calendar tool being invoked.

## Best Possible Solution

Define a narrow runtime contract for explicit freshness or raw-output turns, then enforce it in the embedded runner, tool policy, or a documented hook boundary so the assistant retries, blocks, or asks a clarifying question instead of finalizing when the required tool event is missing.
