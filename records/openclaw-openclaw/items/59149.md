## Summary

Keep open. Current main still only resolves `tools.sessions.visibility` and `tools.agentToAgent` from global config, while the issue has an open implementation candidate, #59170, that explicitly resolves this issue. Per policy, this issue should stay open until that PR is reviewed and merged or closed.

## Best Possible Solution

Keep this issue open and use it as the canonical tracker while maintainers review #59170 or a successor implementation. The likely product path is to add typed per-agent config keys, thread the requester agent’s tools config into session list/history/send/status access checks, document precedence and defaults, and add focused tests for scoped allowlists, disabled per-agent A2A, global fallback, and sandbox interactions.
