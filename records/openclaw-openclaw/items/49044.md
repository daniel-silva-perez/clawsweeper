## Summary

Keep this PR open. Current main still does not implement per-account SOUL files: there is no `--soul` CLI flag, no `soulFile` channel/account config surface, no onboarding SOUL-file step, and runtime bootstrap still loads the fixed workspace `SOUL.md`. The PR remains feature work, but it needs a rebase and an end-to-end design before it is mergeable.

## Best Possible Solution

Keep this PR open only as a candidate for revised feature work. The best path is a rebase plus an end-to-end implementation: add a documented config/schema contract for account-level SOUL overrides, wire CLI and onboarding to write it, pass channel/account context into bootstrap resolution and prompt assembly, make bootstrap caches account-aware, keep file reads workspace-boundary safe, and add tests/docs for default fallback plus named-account behavior.
