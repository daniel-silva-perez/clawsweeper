## Summary

Keep this PR open. Current main still does not implement Feishu recalled-message cleanup, so the contribution is not obsolete, but the patch is not ready to merge because recalled-before-response suppression remains unwired and reply outputs are only partially covered.

## What This Changes

The PR adds a Feishu recall handler, registers `im.message.recalled_v1`, tracks reply-path text/card sends, and deletes mapped bot replies when the original user message is recalled.

## Best Possible Solution

Rebase the PR onto current main, wire recalled-before-response suppression before any fallback direct send, cover all reply-path outputs that should be deleted on recall, and add Feishu tests for event dispatch, mapping, duplicate recalls, partial delete retry, fallback-direct behavior, and media or explicitly documented non-media scope.
