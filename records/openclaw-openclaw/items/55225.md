## Summary

Keep this PR open. Current main still lacks Feishu recalled-message handling, so the contribution is not obsolete, but the PR remains functionally incomplete because recalled-before-response suppression is never applied and several reply-producing paths remain outside the new cleanup mapping.

## What This Changes

The PR adds a Feishu recalled-message event handler, an in-memory user-message to bot-reply map with TTL and retry retention, send-result reply-path tracking, and text/card reply registration for recall cleanup.

## Best Possible Solution

Revise the PR so Feishu recall state is enforced at the reply boundary: suppress fallback/direct sends for recalled reply targets, register every reply-path output that should be deleted or explicitly document any excluded paths, and add focused Feishu tests for event dispatch, idempotent duplicate recalls, partial-delete retry retention, pre-response recall suppression, fallback-direct suppression, media replies, and streaming-card behavior.
