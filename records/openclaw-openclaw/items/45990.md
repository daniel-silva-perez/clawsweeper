## Summary

Keep this PR open for maintainer or author revision. Current main still emits raw `llm_input` before prompt submission and raw `llm_output` once after the run, so the requested per-round raw observability is not implemented; the proposed branch is useful but still has blocking contract regressions around failed provider-call observability and usage normalization.

## What This Changes

The PR moves embedded-runner `llm_input` and `llm_output` hook emission from attempt-level start/end into subscription `message_start`/`message_end`, threads hook context into subscriptions, and updates hook tests and comments for per-round semantics.

## Best Possible Solution

Define a stable raw per-call observability contract alongside #64268, then revise this branch so input hooks still cover failed provider submissions, per-call output usage is normalized, aggregate/final-run semantics are preserved or separately labeled, and plugin docs/changelog explain the migration.

## Reproduction Assessment

Yes. The blocking findings reproduce by inspection and focused tests: make `activeSession.prompt(...)` fail before any assistant `message_start`, and feed assistant usage shaped like `{ totalTokens: 42 }` through the proposed per-round output path.

## Solution Assessment

No. The direction is useful, but moving raw input emission to `message_start` is later than the provider-call boundary and the output helper bypasses the normalized usage contract; a safer fix keeps failed-submission observability and labels or versions per-call versus aggregate output semantics.

  - confidence: 0.92

  - confidence: 0.88
