## Summary

Keep open. Current main has partial mitigations since the reported 2026.4.9 behavior, but it still does not fully address the central small/mid local-model prompt-pressure problem: main sessions can still inject the broad default bootstrap set, the default AGENTS.md template remains large, there is no merged agent bootstrap tier knob, and doctor only documents budget-near/truncation warnings rather than a small-model prompt-size warning with remediation.

## What This Changes

The issue requests reducing, tiering, or warning on default workspace bootstrap prompt size so small/mid local models reliably emit structured tool calls instead of hallucinated tool-use text.

## Best Possible Solution

Define one canonical prompt-pressure strategy for local-model users: either land or rework tiered bootstrap loading with a maintainer-approved default for new installs, or keep the current defaults while adding a targeted doctor/runtime warning and documented remediation for small/mid local models. The result should keep rich-context users on an opt-in path while giving constrained local models a lean, reliable default or clear escape hatch.

## Reproduction Assessment

Yes. The issue gives a concrete local OpenAI-compatible setup, a command, and observable session JSONL checks for real toolCall/toolResult events; current main still has a large default bootstrap path, so the reproduction remains specific enough for maintainer validation.

## Solution Assessment

Unclear. Lowering bootstrapMaxChars, trimming templates, adding bootstrapTier, and adding doctor warnings are all plausible, but the best maintainable fix depends on a product/runtime default decision and should be coordinated with the open tiered-bootstrap work rather than swept in as a narrow one-line repair.
