## Summary

Keep open. Current main has partial mitigations since the reported 2026.4.9 defaults, but it still does not fully solve the reported local small/mid-model prompt-pressure problem: main sessions still inject the broad default bootstrap set, the generated AGENTS.md template is still large, there is no merged bootstrapTier surface on main, and the current warnings are truncation/budget-near warnings rather than model-size-aware remediation.

## What This Changes

The issue requests reducing, tiering, or warning on default workspace bootstrap prompt content so small and mid local models keep reliable structured tool dispatch.

## Best Possible Solution

Pick one canonical prompt-pressure strategy for local-model users: land or rework the minimal/tiered bootstrap work with a maintainer-approved default, or keep richer defaults while adding model-aware doctor/runtime warnings with a clear one-command remediation. The final state should preserve full context as opt-in while giving constrained local models a reliable lean path.

## Reproduction Assessment

Yes. The issue gives a concrete vLLM/OpenAI-compatible setup, exact CLI prompt, systemPromptReport size checks, and session JSONL proof criteria for real toolCall/toolResult events before and after lowering bootstrap budgets.

## Solution Assessment

Unclear. Current main is not the final solution: lowering defaults and warning on truncation helps adjacent cases, but the remaining choice between lean defaults, bootstrap tiers, template trimming, and doctor warnings is a product/runtime decision that should be coordinated with the open tiered-bootstrap PRs.
