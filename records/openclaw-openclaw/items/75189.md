## Summary

Keep open. Current main has reduced bootstrap caps and local-model tool-surface mitigations, but the reported local-model prompt-pressure bug is not fully resolved: normal main sessions still inject the broad bootstrap set, the generated templates remain large, no `bootstrapTier` config surface has landed, and model-aware doctor remediation is not present.

## What This Changes

The issue requests reducing or tiering default workspace bootstrap context so small and mid local models do not lose reliable structured tool dispatch under verbose default prompts.

## Best Possible Solution

Define one canonical local-model prompt-pressure strategy: a supported bootstrap profile or tier with maintainer-approved defaults, schema/docs/tests, hook-aware filtering, and a clear doctor or local-model remediation path, while keeping richer bootstrap context available as an explicit opt-in.

## Reproduction Assessment

Yes. The issue provides concrete setup steps, a single `openclaw agent` command, systemPromptReport measurements, and session JSONL checks for real `toolCall`/`toolResult` events; current main still has the broad bootstrap path needed to validate the same failure shape.

## Solution Assessment

Unclear. Lowering `bootstrapMaxChars` is a narrow mitigation, but the best maintainable fix likely needs coordinated tier/default/doctor/template behavior rather than a standalone cap change.
