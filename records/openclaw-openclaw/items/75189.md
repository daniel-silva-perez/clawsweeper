## Summary

Keep open. Current main has partial prompt-pressure mitigations since the reported release, but it still does not fully solve the user-visible small and mid local-model failure mode: main sessions can still inject the broad workspace bootstrap set, the generated template content remains large, and no merged `bootstrapTier` or model-aware doctor remediation exists on main.

## What This Changes

The issue requests reducing or controlling default workspace bootstrap prompt size so small and mid-size local models keep reliable structured tool dispatch.

## Best Possible Solution

Define one canonical local-model prompt-pressure strategy: a supported bootstrap profile or tier with maintainer-approved defaults for new installs, aligned docs/tests/config schema, and clear remediation through doctor or local-model docs while keeping rich bootstrap context available as an opt-in path.

## Reproduction Assessment

Yes. The issue gives concrete setup steps, a single `openclaw agent` command, systemPromptReport measurements, and session JSONL checks for real `toolCall`/`toolResult` events; current main still has the broad bootstrap path that makes the reproduction actionable for maintainer validation.

## Solution Assessment

Unclear. A one-line lower default would be narrow, but the maintainable fix likely needs coordination with the open tiered-bootstrap and template-ordering work so OpenClaw does not trade one class of local-model failures for lost context in richer deployments.
