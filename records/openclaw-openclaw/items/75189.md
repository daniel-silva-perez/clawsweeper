## Summary

Keep open. Current main has partial mitigations for this prompt-pressure class, but it does not fully resolve the reported small/mid local-model tool-dispatch failure: normal main sessions still inject the broad bootstrap file set, the generated templates remain large, there is no `bootstrapTier` config surface on main, and `doctor` still only warns on configured bootstrap budget pressure rather than model-aware prompt pressure.

## What This Changes

The issue requests reducing, tiering, or warning on default workspace bootstrap prompt content so small/mid local models keep reliable structured tool dispatch.

## Best Possible Solution

Define one canonical local-model prompt-pressure strategy: a supported bootstrap profile or tier with maintainer-approved defaults, schema/docs/tests, hook-aware filtering, and a clear doctor/local-model remediation path, while keeping richer bootstrap context available as an explicit opt-in.

## Reproduction Assessment

Yes. The issue provides exact setup steps, an `openclaw agent` command, systemPromptReport measurements, and session JSONL checks for real `toolCall`/`toolResult` events; current main still has the broad bootstrap injection path needed to validate the same failure shape.

## Solution Assessment

Unclear. Lowering `bootstrapMaxChars` alone is a narrow mitigation, but the maintainable fix likely needs coordinated tier/default/doctor/template behavior rather than another standalone cap tweak.
