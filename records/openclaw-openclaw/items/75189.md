## Summary

Keep open. Current main has partial mitigations since the reported 2026.4.9 behavior, but it does not yet solve the central small/mid local-model tool-dispatch problem: the default generated bootstrap content is still large, there is no merged `bootstrapTier` control, and doctor only reports bootstrap budget pressure rather than warning on small-model prompt-size risk.

## What This Changes

The issue requests a smaller, tiered, or warned default bootstrap prompt path so small and mid-size local models reliably dispatch tools instead of hallucinating tool-use results.

## Best Possible Solution

Define one canonical prompt-pressure strategy for local-model users: either land/rework the tiered bootstrap path with a maintainer-approved default for new installs, or keep existing defaults while adding a targeted doctor/runtime warning and documented one-command remediation. The final fix should preserve high-context users' opt-in path while giving small/mid local models a lean default or clearly surfaced escape hatch.

## Reproduction Assessment

Yes. The issue gives concrete setup, command, expected session JSONL/tool-call checks, and before/after config evidence; current main code inspection confirms the default generated bootstrap surface is still large enough that the reported path remains plausible.

## Solution Assessment

Unclear. The requested fixes are directionally reasonable, but the best maintainable solution depends on whether maintainers want to change new-install defaults, trim the generated workspace templates, or keep defaults stable and add detection/remediation around small local models.
