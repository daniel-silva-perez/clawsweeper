## Summary

Keep open. Current main has reduced bootstrap defaults and added partial mitigations, but it still does not solve the central small/mid local-model tool-dispatch risk: main sessions still inject a broad bootstrap set, the default generated templates remain large, warnings are tied to configured truncation budgets rather than model-size prompt pressure, and the tiered bootstrap work remains open rather than merged.

## What This Changes

The issue requests reducing or warning about verbose default workspace bootstrap prompt content because it degrades structured tool dispatch on small and mid local models.

## Best Possible Solution

Define one canonical local-model prompt-pressure strategy: either land/rework tiered bootstrap with a maintainer-approved default for new installs, or keep compatibility defaults while adding targeted doctor/runtime detection plus a documented remediation path. The end state should preserve high-context users' opt-in path while making small/mid local-model tool dispatch reliable or clearly warned before use.
