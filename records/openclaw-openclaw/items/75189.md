## Summary

Keep open. Current main has partial prompt-pressure mitigations, but it still does not resolve the reported small/mid local-model bootstrap bloat path: ordinary sessions can inject the broad workspace bootstrap set, the default generated AGENTS.md remains large, no bootstrapTier surface is merged on main, and doctor still focuses on budget-near/truncation warnings rather than local-model prompt-size remediation.

## What This Changes

The issue asks OpenClaw to reduce, tier, trim, or warn on default workspace bootstrap prompt weight so small/mid local models keep reliable structured tool dispatch.

## Best Possible Solution

Consolidate on one maintainer-approved prompt-pressure strategy for constrained local models: review the open bootstrapTier/doctor-warning work against the older tiered-bootstrap PR, decide whether the default should remain standard or become lean for new local installs, and keep template reordering as the narrow companion fix for truncation-priority behavior.

## Reproduction Assessment

Yes. The issue gives a concrete OpenAI-compatible vLLM setup, exact agent command, systemPromptReport numbers, and objective session JSONL checks showing zero toolCall events before budget reduction and real toolCall/toolResult events afterward; current main still has the large ordinary-session bootstrap path needed to validate it.

## Solution Assessment

Unclear. The proposed fixes are plausible, but the best maintainable path depends on a product/runtime default decision; an opt-in bootstrapTier with a doctor warning is safer than an unreviewed default cap reduction, while template trimming or reordering addresses a related but narrower symptom.
