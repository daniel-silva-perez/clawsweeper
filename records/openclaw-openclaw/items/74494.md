## Summary

Keep open: current main has not fully replaced this UI work, and the branch is still relevant after the related cron and Lit-state dropdown changes. The patch has blocking UX/accessibility regressions and a state-management regression, so it should not be closed as obsolete or merged as-is.

## What This Changes

The PR changes the Control UI mobile chat settings dropdown and related CSS, adds mobile refresh/model/labeled controls and safe-area handling, constrains tool-card overflow, exports the chat model select, and updates a browser render test.

## Best Possible Solution

Fold the intended mobile parity into Control UI while preserving the existing state contract: dismiss the overlay through Lit state, scope overflow containment to the mobile surface, keep page zoom available, and route all visible settings labels through the Control UI i18n source.

  - body: The close button removes the `open` class directly, but `chatMobileControlsOpen` and `aria-expanded` stay true. The next render can reopen the dropdown and assistive tech still sees it as expanded; call the existing state setter instead.

  - confidence: 0.9
- **[P2] Do not cap mobile page zoom:** `ui/index.html:5`
  - body: Adding `maximum-scale=1.0` to the global viewport meta can prevent pinch zoom across the Control UI on mobile browsers and webviews. Keep the safe-area/keyboard viewport behavior without disabling zoom.
  - confidence: 0.84
- **[P3] Localize the new settings copy:** `ui/src/ui/app-render.helpers.ts:498-510`
  - body: The new visible settings title, close label, and select labels are hard-coded English strings. Route them through the existing Control UI locale source so non-English users do not get mixed-language chat controls.
  - confidence: 0.78
