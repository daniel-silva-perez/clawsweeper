## Summary

Keep open. Current main does not expose MiniMax ASR: the MiniMax media-understanding provider only advertises image capability, and the user-facing media matrix leaves MiniMax STT blank. The request is valid as a missing capability, but I would not auto-queue a repair until maintainers confirm the upstream MiniMax ASR endpoint/model contract because the official MiniMax docs index I checked lists speech/T2A and voice APIs, not an ASR/transcription API. ([platform.minimax.io](https://platform.minimax.io/docs/api-reference/api-overview))

## What This Changes

Request to add MiniMax ASR/speech-to-text support to the bundled MiniMax plugin so OpenClaw can transcribe inbound audio through MiniMax.

## Best Possible Solution

If upstream MiniMax ASR is confirmed, add it as a vendor-owned media-understanding audio capability in `extensions/minimax`, preferably by reusing the shared OpenAI-compatible transcription helper when the endpoint contract matches. Update MiniMax manifest metadata, provider docs, the media matrix/audio docs, and focused provider tests without adding MiniMax-specific logic to core.

## Reproduction Assessment

Yes. The missing capability is reproducible by inspecting current main: MiniMax registers image media understanding only, and docs list MiniMax TTS but no MiniMax STT path.

## Solution Assessment

Unclear. The OpenClaw side has the right shared media-understanding seam, but the exact MiniMax endpoint/model/auth contract needs upstream verification before this is safe to implement automatically.
