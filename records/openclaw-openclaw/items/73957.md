## Summary

Keep open. Current main does not expose MiniMax speech-to-text: the MiniMax media-understanding providers and manifest metadata are image-only, and the docs matrix leaves MiniMax STT blank. OpenClaw already has a public audio transcription provider seam, so this is a provider capability request rather than a missing core API, but the MiniMax public docs index I checked does not confirm the reported `/v1/audio/transcriptions` endpoint, so an automated fix would be speculative without upstream confirmation. ([platform.minimax.io](https://platform.minimax.io/docs/llms.txt))

## What This Changes

Request to add MiniMax speech-to-text/ASR support to the bundled minimax plugin so OpenClaw can transcribe inbound audio with MiniMax.

## Best Possible Solution

If maintainers confirm a supported public MiniMax ASR endpoint and model, add the capability inside `extensions/minimax` as an audio `MediaUnderstandingProvider`, update manifest metadata and MiniMax/media docs, and cover it with provider unit and registration/metadata tests. If the endpoint is private or unavailable, document that OpenClaw's MiniMax integration supports TTS but not STT.

## Reproduction Assessment

Not applicable as a feature request. The current-main check is high-confidence: source, manifest metadata, and docs all show MiniMax media understanding is image-only and MiniMax STT is not advertised.

## Solution Assessment

Unclear until the MiniMax ASR contract is verified. The narrowest maintainable implementation would reuse OpenClaw's existing audio media-understanding contract, but only after confirming the upstream endpoint/model instead of assuming OpenAI compatibility.
