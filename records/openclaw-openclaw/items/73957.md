## Summary

Keep open. Current `main` and the latest shipped tag still do not expose MiniMax speech-to-text: the MiniMax media-understanding provider and manifest are image-only, while the docs advertise MiniMax TTS but leave MiniMax STT blank. The existing plugin SDK already has an audio transcription seam, so this is a provider capability request, but MiniMax's current public docs index/API overview list speech synthesis and voice-cloning surfaces rather than a confirmed ASR/audio-transcriptions endpoint, so a fix needs maintainer or upstream API confirmation first. ([github.com](https://github.com/openclaw/openclaw/commit/52e2d4e16a290cc8027d9b418398c65d05d49717))

## What This Changes

Expose MiniMax ASR speech-to-text in the MiniMax plugin so OpenClaw can transcribe audio through MiniMax.

## Best Possible Solution

Confirm the supported MiniMax ASR contract first. If it is public and stable, implement it inside `extensions/minimax` by adding audio `transcribeAudio` support to the MiniMax media-understanding provider, updating manifest metadata and provider/media docs, and adding focused provider registration/unit tests; if the API is unavailable, document MiniMax as TTS-only for speech in OpenClaw.

## Reproduction Assessment

Not applicable as a feature request. Current-main verification is high confidence: source, manifest metadata, release-tag content, and docs all show MiniMax image/TTS support but no MiniMax STT provider.

## Solution Assessment

Unclear until the upstream MiniMax ASR contract is verified. The narrowest maintainable solution would reuse OpenClaw's existing media-understanding audio contract inside the MiniMax plugin, but guessing an OpenAI-compatible `/audio/transcriptions` endpoint would be unsafe without API proof.
