## Summary

Keep open. Current main still does not implement or advertise MiniMax speech-to-text: the bundled MiniMax media-understanding providers are image-only, while OpenClaw already has a generic audio transcription provider seam that could host this if MiniMax confirms a supported ASR endpoint/model.

## What This Changes

Request to add MiniMax speech-to-text/ASR support to the bundled MiniMax plugin so OpenClaw can transcribe audio with MiniMax.

## Best Possible Solution

Confirm the supported MiniMax ASR endpoint, auth path, request shape, and model id from upstream docs, vendor confirmation, or live proof. If confirmed, add audio transcription inside `extensions/minimax` using the existing media-understanding audio contract, update manifest metadata and MiniMax/media docs, and cover registration plus provider request behavior with focused tests; if not confirmed, document MiniMax as TTS-only for speech.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification is high-confidence: source, manifest metadata, and docs all show MiniMax media understanding is image-only and MiniMax STT is not advertised.

## Solution Assessment

Unclear until the upstream MiniMax ASR contract is verified. The narrowest maintainable implementation would be provider-local in `extensions/minimax` through the existing `MediaUnderstandingProvider` audio seam, but coding against an unconfirmed endpoint would be speculative.
