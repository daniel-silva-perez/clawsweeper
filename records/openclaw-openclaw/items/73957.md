## Summary

Keep open. Current main still does not expose MiniMax batch speech-to-text: the bundled MiniMax media-understanding providers are image-only, MiniMax docs in this repo advertise TTS/image/music/video/search but not STT, and the shared plugin SDK already has an audio transcription seam. The remaining blocker is upstream/API confirmation for MiniMax ASR, not a core API gap or an already-implemented feature.

## What This Changes

Request to add MiniMax ASR speech-to-text support to the bundled MiniMax plugin so OpenClaw can transcribe audio through MiniMax.

## Best Possible Solution

Confirm MiniMax's supported ASR endpoint, auth mode, default model, response shape, and region behavior with upstream docs or live/vendor proof. If confirmed, add it inside `extensions/minimax` as an audio `MediaUnderstandingProvider`, update manifest metadata and MiniMax/media/audio docs, and cover provider registration plus transcription request tests; if not confirmed, document that MiniMax currently supports TTS but not STT in OpenClaw.

## Reproduction Assessment

Not applicable as a feature request, but the absence is reproducible by inspecting current main: MiniMax media providers and metadata are image-only, and the docs matrix leaves MiniMax STT blank.

## Solution Assessment

Unclear until the upstream ASR contract is verified. The narrowest maintainable implementation would reuse the existing audio media-understanding contract in the MiniMax plugin, but guessing an OpenAI-compatible `/v1/audio/transcriptions` endpoint would be speculative.
