## Summary

Keep this PR open. The underlying operator-filter part of the request is still missing on current main, but current main already broadly skips deleted/reset archive artifacts before ingestion, and this PR is not merge-ready because its SDK helper implementation is still on the old `src/memory-host-sdk` source layout while current main publishes that surface from `packages/memory-host-sdk`.

## What This Changes

This PR adds memory-core Dreaming transcript classification helpers, `dreaming.sessionFilter` exclusions, ingestion tests, a changelog entry, and internal-system provenance filtering for heartbeat/cron transcript content.

## Best Possible Solution

Rebase the useful work onto current main by moving SDK helper implementation and exports into `packages/memory-host-sdk`, preserving the thin `src/memory-host-sdk` wrappers, documenting `dreaming.sessionFilter` in the public memory/Dreaming docs, and explicitly deciding whether to keep the session-id fallback now that archive artifacts are skipped broadly.

## Reproduction Assessment

Partly yes. The missing operator-facing exclusion path is reproducible by source inspection on current main; the older rotated `.deleted` corpus leak is less clear on current main because deleted/reset archive artifacts are now skipped before content is read.

## Solution Assessment

No, not as submitted. The direction is plausible, but the PR must be rebased onto the current package SDK layout and docs surface; a safer maintainer path may split the operator filters from the overlapping rotated-archive mitigation.
