## Summary

Keep this PR open. Current main still does not implement the central behavior: embedded runs do not expose a final winning auth profile id, and post-run session persistence has no field or write path to update `authProfileOverride` after successful auth-profile rotation. The PR remains useful, but it should be narrowed because the current diff is much broader than the auth persistence fix and the review discussion flags a security-sensitive `allowModelOverride` regression risk.

## Best Possible Solution

Keep the PR open, but rework it into a narrow auth-profile persistence patch: propagate the resolved final winning profile id from the embedded run/attempt result, pass it into post-run session persistence, write it to `authProfileOverride`, preserve the correct source semantics, and add focused regression tests. Keep the command-module refactor and any `allowModelOverride` authorization changes out of this fix unless reviewed separately.
