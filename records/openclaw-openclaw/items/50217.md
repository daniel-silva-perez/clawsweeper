## Summary

Keep this PR open. Current main still does not implement the PR’s central contract: the embedded runner does not return a final winning auth profile id, and the post-run session persistence path has no field or update that can write that final profile back to `authProfileOverride`. Main has adjacent auth-profile selection/cooldown logic and live model-switch auth persistence, but not successful embedded fallback winner persistence. The PR also still needs rework because its diff broadens into an XL command-path refactor with a security-sensitive model override guard regression called out in review.

## Best Possible Solution

Keep this PR open, but do not merge it as-is. The best path is a narrow rebase onto current main that carries the final winning auth profile from the embedded runner into the run result, adds that value to the post-run session persistence contract, writes it to `authProfileOverride` after successful fallback rotation, preserves `authProfileOverrideSource: "user"` where that is the existing user intent, and adds focused regression coverage. The unrelated `src/commands/agent.ts` wholesale refactor should be dropped or separately justified, and the existing ingress `allowModelOverride` guard must remain intact.
