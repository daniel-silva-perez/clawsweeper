## Summary

Keep this PR open. Current main still does not implement the PR's central contract: the embedded runner does not return a final winning auth profile id, and the post-run session persistence path has no field or write that can persist the rotated winner back to `authProfileOverride`. The PR also still needs a narrow rework before merge because its provided diff broadens into a large command implementation rewrite with security-sensitive model-override guard concerns already flagged in review.

## Best Possible Solution

Keep this PR open, but rework it into a narrow patch. The best implementation is to propagate the resolved final winning auth profile id from the embedded attempt/run result, add that value to the post-run session persistence contract, persist it to `authProfileOverride` after successful fallback rotation, preserve `authProfileOverrideSource: "user"` when the session started user-pinned, and add focused regression tests. The unrelated `src/commands/agent.ts` wholesale refactor should be dropped or separately justified, and the existing `allowModelOverride` guards must remain intact.
