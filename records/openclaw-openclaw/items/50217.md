## Summary

Keep this PR open. Current main still does not implement the PR's central auth-profile persistence contract: embedded runs do not return a final winning auth profile id, and the post-run session persistence path has no input or write that would update `authProfileOverride` to the rotated winner. The PR intent is still useful, but the current diff needs rework because it mixes the narrow auth fix with a large command implementation rewrite and review-raised ingress model-override authorization risk.

## Best Possible Solution

Rework this into a narrow patch: propagate the resolved final winning auth profile id from the embedded run/attempt result, pass it into session-store persistence after a successful run, persist it to `authProfileOverride`, preserve `authProfileOverrideSource: "user"` for sessions that started user-pinned, and add focused regression tests. Keep the current `allowModelOverride` authorization guards intact; any command-module refactor should be separate and explicitly reviewed.
