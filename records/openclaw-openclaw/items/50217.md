## Summary

Keep this PR open. Current main still lacks the PR’s central contract: the embedded runner does not expose a final winning auth profile id, and the post-run session persistence path has no input or update that can persist that winner back to `authProfileOverride`. The PR also should not merge as-is because its diff broadens into a large command implementation rewrite with security-sensitive ingress/model override concerns already called out in review.

## Best Possible Solution

Keep this PR open, but rework it into a narrow patch. The best implementation is to carry the resolved final winning auth profile id from the embedded attempt/run result, add that value to the post-run session persistence contract, persist it to `authProfileOverride` after successful fallback rotation, preserve `authProfileOverrideSource: "user"` when the session started from a user-pinned profile, and add focused regression tests. The unrelated `src/commands/agent.ts` wholesale refactor should be dropped or separately justified, and the existing `allowModelOverride` ingress/per-run guard must remain intact.
