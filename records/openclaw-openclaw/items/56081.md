## Summary

Keep this PR open. Current main still does not render the answering provider/model in the per-response usage footer, even though `runReplyAgent` has `providerUsed` and `modelUsed` available before it calls the formatter. The provided PR remains a focused implementation candidate for the still-open model-in-footer request tracked by #65477.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto current main, extend `formatResponseUsageLine` with optional provider/model inputs formatted through `formatProviderModelRef`, pass `providerUsed` and `modelUsed` from `runReplyAgent`, add focused footer tests, and update the public token-use docs if the displayed footer changes.
