## Summary

Keep open. The PR is useful source work for a real Discord debounce mention bug, but current main has moved the runtime to the bundled Discord plugin path and the PR still changes the old `src/discord` layout. Main now preserves the last message prototype, which helps one case, but it still does not accumulate mention metadata from earlier debounced entries, so the observable bug can remain.

## What This Changes

The PR accumulates Discord mentioned users, roles, and `@everyone` flags from all debounced entries when constructing a synthetic message, but it applies that change to the old `src/discord/monitor/message-handler.ts` path.

## Best Possible Solution

Port the all-entry mention aggregation into `extensions/discord/src/monitor/message-handler.ts`, preserving the current prototype/descriptors approach while merging `rawData.mentions`, `rawData.mention_roles`, and `rawData.mention_everyone` across debounced entries, then add a focused regression where the mention metadata appears before the last debounced entry.
