## Summary

Keep open. The PR addresses a real Discord debounce mention-routing problem, but it edits the removed legacy `src/discord` handler while current `main` ships the Discord plugin from `extensions/discord`; the current plugin still builds the debounced synthetic message from the last entry's message data instead of merging mention metadata across all entries.

## What This Changes

The PR branch changes the legacy Discord message handler to materialize mentioned users, mentioned roles, and `@everyone` from all debounced entries when building a synthetic message.

## Best Possible Solution

Port the all-entry mention metadata merge into `extensions/discord/src/monitor/message-handler.ts` so debounced synthetic messages expose merged `mentions`, `mention_roles`, and `mention_everyone` through the current internal `Message` getters, then add focused regression coverage for an earlier debounced entry containing mention metadata when the final entry does not.
