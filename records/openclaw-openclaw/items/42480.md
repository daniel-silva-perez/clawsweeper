## Summary

Keep open. The PR addresses a real Discord debounce mention-metadata problem, and the author handled the prior role/everyone feedback, but the branch does not apply to current main because it modifies the removed old Discord handler; the current plugin handler still builds multi-entry synthetic messages from the last message's metadata.

## What This Changes

The PR accumulates mentioned users, mentioned roles, and @everyone flags across debounced Discord entries when building the synthetic message in the old `src/discord` handler.

## Best Possible Solution

Port the all-entry mention metadata merge into `extensions/discord/src/monitor/message-handler.ts` so synthetic debounced messages expose merged `mentions`, `mention_roles`, and `mention_everyone` through the current internal `Message` getters, then add focused regression coverage where an earlier debounced entry has mention metadata and the last entry does not.
