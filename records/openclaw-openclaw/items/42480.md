## Summary

Keep open. The reported Discord debounce mention bug still appears valid on current main, but this PR is not merge-ready because it changes the deleted legacy Discord path while the active bundled Discord plugin still builds debounced synthetic messages from the final entry's raw message data.

## What This Changes

The PR branch updates the legacy Discord message handler to accumulate mentioned users, mentioned roles, and @everyone metadata from all debounced entries into the synthetic message.

## Best Possible Solution

Implement the mention metadata merge in extensions/discord/src/monitor/message-handler.ts by updating the synthetic message's getter-visible raw data for mentions, mention_roles, and mention_everyone across all debounced entries, then add focused queue/preflight regression coverage for earlier entries that contain mention metadata when the final entry does not.

Yes. A high-confidence source-level reproduction is a debounced Discord batch where an earlier message entry mentions the bot, a role, or everyone and the final entry does not; current main combines the text but leaves mention getters backed by the final entry's raw data.
