## Summary

Keep open. The underlying Discord debounce mention bug still appears valid on current main, but this PR edits the removed legacy `src/discord` handler, so it should not be merged as-is; the useful source work needs to be ported into the active Discord plugin path.

## What This Changes

The PR branch accumulates `mentionedUsers`, `mentionedRoles`, and `mentionedEveryone` across debounced Discord entries when constructing a synthetic message in the legacy Discord message handler.

## Best Possible Solution

Port the mention metadata merge into `extensions/discord/src/monitor/message-handler.ts` on current main, building the synthetic message so its getter-backed raw data includes deduped `mentions`, deduped `mention_roles`, and `mention_everyone` from every debounced entry, then add focused Discord handler/preflight coverage for earlier-entry and final-entry-empty cases.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a debounced Discord batch where an earlier entry contains bot, role, or everyone mention metadata and the final entry does not; current main builds the synthetic message from the final entry's data, so preflight can see no mention metadata.

## Solution Assessment

No. The PR's direction is useful after the follow-up commit, but it is not the best mergeable fix because it targets `src/discord/monitor/message-handler.ts` while current main ships the Discord handler from `extensions/discord/src/monitor/message-handler.ts` with internal getters backed by raw data.
