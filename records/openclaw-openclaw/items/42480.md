## Summary

Keep open. The proposed fix addresses a real Discord debounce mention problem and the author already handled the review feedback about roles and `@everyone`, but the branch edits the legacy `src/discord` handler while current `main` ships Discord from `extensions/discord`. The active plugin path still does not merge mention metadata from earlier debounced entries, so this PR is useful source work but not mergeable as-is.

## What This Changes

The PR branch accumulates `mentionedUsers`, `mentionedRoles`, and `mentionedEveryone` across debounced Discord entries when creating a synthetic message in `src/discord/monitor/message-handler.ts`.

## Best Possible Solution

Port the mention metadata merge into `extensions/discord/src/monitor/message-handler.ts` so the synthetic message's getter-visible raw data contains merged `mentions`, `mention_roles`, and `mention_everyone` from every debounced entry, with focused queue/preflight regression coverage for earlier-entry mentions when the final entry has none.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a debounced Discord batch where an earlier message entry contains the bot mention, role mention, or `@everyone` metadata and the final entry does not; current `main` combines the text but leaves mention getters backed by the final entry's raw data.

## Solution Assessment

No. The PR's intended fix is directionally right after the follow-up commit, but applying it to `src/discord/monitor/message-handler.ts` is not the best solution because that path is no longer the shipped Discord implementation.
