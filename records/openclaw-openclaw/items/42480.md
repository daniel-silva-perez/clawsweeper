## Summary

Keep open. The PR addresses a real Discord debounced-message mention bug, and the follow-up commit handled the earlier review gap for roles and `@everyone`, but the patch targets the removed `src/discord` handler path while current `main` runs the Discord plugin handler under `extensions/discord`. Current `main` still builds multi-entry synthetic messages from the last message's raw mention metadata, so the useful fix needs to be ported with regression coverage before merge.

## What This Changes

The PR accumulates Discord mentioned users, mentioned roles, and `@everyone` flags from all debounced entries when constructing a synthetic debounced message.

## Best Possible Solution

Port the all-entry mention metadata merge into `extensions/discord/src/monitor/message-handler.ts`, preserving the current prototype/descriptors construction while merging `rawData.mentions`, `rawData.mention_roles`, and `rawData.mention_everyone` across debounced entries. Add a focused regression where the first debounced entry carries the mention metadata and the last entry does not.
