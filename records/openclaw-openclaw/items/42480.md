## Summary

Keep open. The mention-loss problem still maps to current main, but this PR branch modifies the deleted legacy Discord handler, so merging it would not repair shipped Discord behavior in the current plugin.

## What This Changes

The PR branch modifies the legacy Discord message handler to collect mentioned users, mentioned roles, and `@everyone` metadata from all debounced messages into the synthetic message.

## Best Possible Solution

Port the all-entry mention metadata merge into `extensions/discord/src/monitor/message-handler.ts` so debounced synthetic messages expose merged `mentions`, `mention_roles`, and `mention_everyone` through the current internal `Message` getters, then add focused regression coverage for earlier debounced entries that contain mention metadata while the final entry does not.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a debounced Discord batch where an earlier entry mentions the bot, a role, or everyone and the final entry does not; current main builds combined text but leaves mention getters backed by the final message's raw data.

## Solution Assessment

No. The direction is correct, and the follow-up commit addressed the role/everyone review gap, but the proposed patch edits the legacy path rather than the current Discord plugin and is not the narrowest maintainable fix for current main.
