## Summary

Keep open. The PR addresses a real Discord debounce mention-routing bug, but it edits the removed legacy `src/discord` handler while current `main` ships Discord from `extensions/discord`; the current plugin still builds debounced synthetic messages from the last entry without merging earlier mention metadata.

## What This Changes

The PR branch modifies the legacy Discord message handler to collect mentioned users, mentioned roles, and `@everyone` from all debounced entries when constructing a synthetic message.

## Best Possible Solution

Port the all-entry mention metadata merge into `extensions/discord/src/monitor/message-handler.ts` so debounced synthetic messages expose merged `mentions`, `mention_roles`, and `mention_everyone` through the current internal `Message` getters, then add focused regression coverage for an earlier debounced entry containing mention metadata when the final entry does not.
