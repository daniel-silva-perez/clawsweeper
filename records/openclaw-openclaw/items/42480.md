## Summary

Keep this PR open. Current main still builds Discord debounced synthetic messages by spreading the last Carbon message without materializing `mentionedUsers`, `mentionedRoles`, or `mentionedEveryone`; the PR contains useful bug-fix logic, but it targets the old `src/discord` layout and needs porting to `extensions/discord` plus regression coverage.

## Best Possible Solution

Keep this PR open as useful bug-fix work. Port the all-entry mention accumulation into `extensions/discord/src/monitor/message-handler.ts`, preserving `mentionedUsers`, `mentionedRoles`, and `mentionedEveryone` on the synthetic message, then add a focused debounce/preflight regression test where mention metadata appears before the last debounced entry.
