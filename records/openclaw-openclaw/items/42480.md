## Summary

Keep open. The PR describes a real Discord debounce mention bug and its follow-up commit addressed the earlier review gap for roles and @everyone, but the branch changes the deleted legacy handler while current main still has the active Discord plugin path unfixed.

## What This Changes

The PR branch accumulates `mentionedUsers`, `mentionedRoles`, and `mentionedEveryone` across debounced Discord entries when constructing a synthetic message in the legacy Discord message handler.

## Best Possible Solution

Port the mention metadata merge into the active Discord plugin so the synthetic message's getter-backed raw data contains merged `mentions`, `mention_roles`, and `mention_everyone` from every debounced entry, then cover the earlier-entry/final-entry-empty cases in focused handler or preflight tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a debounced Discord batch where an earlier entry contains bot, role, or @everyone mention metadata and the final entry does not; current main builds the synthetic message from the final entry raw data, so preflight reads no mention metadata.

## Solution Assessment

No. The intended fix is directionally right after the follow-up commit, but applying it to `src/discord/monitor/message-handler.ts` is not the best solution because current main ships the Discord implementation from `extensions/discord`.
