## Summary

Keep this PR open. Current main still scopes control-plane write buckets by `deviceId|clientIp` for identified or known-IP clients, while this PR proposes changing that behavior to include `connId` and isolate concurrent sessions. The change is not implemented or superseded on main, and it remains a real gateway policy/security decision rather than cleanup-close material.

## Best Possible Solution

Keep this PR open for maintainer policy review or an author update. If per-connection control-plane write budgets are desired, rebase onto current main, preserve the pruning and hard cap, add a same-device/IP multi-connection regression test, update the gateway docs and changelog, and make the security impact explicit. If maintainers prefer the current aggregate `deviceId+clientIp` policy, close the PR manually with that product/security rationale.
