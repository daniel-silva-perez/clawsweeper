## Summary

Keep this PR open because current main still does not let `/verbose on` or `/verbose full` override ordinary group-chat tool-summary suppression, so the branch targets a real remaining bug. Do not merge it unchanged: the current PR condition suppresses native command-source text tool summaries, which conflicts with current main behavior and tests.

## What This Changes

The PR changes `src/auto-reply/reply/dispatch-from-config.ts` to read the stored verbose level and deliver group-chat tool summaries when verbose is `on` or `full` while keeping default group suppression.

## Best Possible Solution

Update the PR against current main so effective verbose `on` or `full` opts ordinary non-Slack group chats into text tool summaries while preserving current native delivery and the existing special gates. Add narrow dispatch tests for group verbose off/on/full and keep the broader configurable group policy tracked in #54663.
