## Summary

Keep open. The reported group-chat verbose gap still exists on current main, but the PR branch is stale against the dispatcher gate that now handles Slack non-DM suppression, Telegram forum-topic delivery, and native direct tool summaries; applying the branch condition as-is would regress those paths.

## What This Changes

The PR changes `dispatch-from-config.ts` so stored `/verbose on` or `/verbose full` can allow group-chat tool-summary delivery while preserving default group suppression and native-source suppression.

## Best Possible Solution

Layer explicit stored verbose `on` or `full` onto current main's `shouldSendVerboseProgressMessages` gate so ordinary non-Slack group chats can receive text tool summaries after user opt-in, while preserving Slack non-DM suppression, Telegram forum-topic delivery, native direct delivery, exec approvals, and media-only fallback.
