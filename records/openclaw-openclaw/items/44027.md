## Summary

Keep open. The reported DingTalk AI Card/tool-summary gap is valid, but this PR is stale against current main's shared verbose delivery gate and would regress Slack non-DM suppression, Telegram forum-topic delivery, and native direct tool-summary delivery.

## What This Changes

The PR reads the stored verbose level in `dispatch-from-config.ts` so `/verbose on` or `/verbose full` can allow group-chat tool summaries while keeping native command-source sessions suppressed.

## Best Possible Solution

Extend current main's shared verbose/tool-summary gate so ordinary non-Slack group chats can deliver text-only tool results when the current session verbose state is `on` or `full`, while preserving Slack non-DM suppression, Telegram forum-topic delivery, native direct delivery, exec-approval/media fallbacks, preview-progress suppression, and focused regression coverage.

Yes. A focused unit test can seed a normal group session with `verboseLevel: "on"`, invoke `opts.onToolResult({ text: "..." })`, and assert `dispatcher.sendToolResult` receives the text payload; current tests cover the negative group case but not this positive verbose override.

## Solution Assessment

No. This PR patches an old group/native condition rather than the current shared delivery gate; the safer fix is to reuse the existing dynamic verbose state and only relax ordinary group suppression where the current surface policy allows it.
