## Summary

Keep this PR open. Current main still suppresses ordinary group text-only tool summaries without consulting the session `/verbose on` or `/verbose full` setting, so the PR targets a real remaining behavior gap. The branch should be rebased and adjusted, because its current native-source suppression conflicts with current main tests that expect native sessions to deliver tool summaries and media.

## Best Possible Solution

Keep this PR open as a plausible focused bug fix, but require a rebase and implementation update against current `main`. The best implementation is to let effective session verbose `on` or `full` opt ordinary non-Slack group chats into text tool summaries while preserving current native-command delivery, Slack non-DM suppression, forum behavior, sendPolicy deny, preview suppression, media-only delivery, and exec approvals. Add focused regression tests for group verbose off/on/full and coordinate broader configurable group tool-result policy in #54663.

- Merging the PR unchanged would likely regress current native command-source behavior because the branch suppresses native text tool summaries while current main tests expect native text summaries and media to be delivered.

- The PR currently lacks focused regression tests for ordinary group chats with verbose off/on/full and for adjacent gates affected by `shouldSendToolSummaries`.
