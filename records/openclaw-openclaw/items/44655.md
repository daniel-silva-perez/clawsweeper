## Summary

Keep open. Current main has improved Z.AI support, including glm-4.7-flash in the seeded catalog and a narrow GLM-5 forward-compat resolver, but it still does not implement authenticated Z.AI /models discovery or dynamic picker/list population for the requested Coding Plan endpoints.

## Best Possible Solution

Keep this issue open as a Z.AI provider enhancement. Implement endpoint-aware /models discovery inside extensions/zai for api.z.ai and open.bigmodel.cn, cache or augment discovered rows through the provider catalog path, preserve static defaults as fallback, allow explicit custom model refs to keep working, and add focused parser/fallback tests plus provider docs updates.
