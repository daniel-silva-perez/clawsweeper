## Summary

Keep this PR open. Current main still does not ship the dedicated Control UI flow for creating a parallel chat thread without resetting the active session. Main has a useful `sessions.create` gateway seam now, but the chat UI is still wired to the reset-style `/new` path and the PR still has unresolved functional review findings.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, wire the dialog to the gateway `sessions.create` API with the current session as `parentSessionKey` and an optional initial task, preserve the active transcript and actionable error on failure, fix modal focus/Escape handling, and add focused UI tests for successful creation, rollback, and in-flight dismissal guards.
