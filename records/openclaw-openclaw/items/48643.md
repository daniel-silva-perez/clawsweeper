## Summary

Current main still does not implement the PR's central UI feature: a chat-level flow to create a parallel thread without resetting the active session. Main now has the backend `sessions.create` seam needed for a better implementation, so this PR is still useful as an implementation candidate but should be rebased and reworked rather than closed as implemented.

## Best Possible Solution

Keep this PR open as a useful implementation candidate. Rebase it onto current main, implement chat thread creation through `sessions.create` with the active session as `parentSessionKey` plus optional `label` and initial `task`/`message`, preserve the active transcript and actionable errors on rollback, fix modal focus/Escape handling, and add focused UI tests for success, failure rollback, and in-flight dismissal guards.
