## Summary

Keep this PR open. Current main has the backend `sessions.create` seam for creating child/dashboard sessions, but it still does not ship the dedicated Control UI chat flow for creating a parallel thread without resetting the active session. The PR remains a plausible implementation candidate, though it needs rework for the newer gateway contract and the unresolved review findings.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, route creation through `sessions.create` with the active session as `parentSessionKey` plus optional initial message/task, preserve the active transcript and actionable error on failure, fix modal focus/Escape handling, and add focused UI tests for successful creation, rollback, and in-flight dismissal guards.
