## Summary

Keep open. The delivery observability direction is useful and current main still lacks the requested deliveryStatus surface, but this draft PR is dirty against main and still has merge-blocking behavior issues around the JSON contract, hook-cancellation semantics, and recently-added result metadata handling.

## What This Changes

This PR modifies command delivery code and tests to add deliveryStatus tracking, diagnostic delivery-failure logging, and deliveryStatus output for `openclaw agent --json --deliver`.

## Best Possible Solution

Keep the deliveryStatus/logging direction, but rebase onto current main, preserve the existing JSON writer and result-meta override contract, consume DeliveryOutcome/allCancelledByHook or equivalent cancellation metadata before classifying empty results, and ensure `--json --deliver` emits exactly one parseable JSON envelope even when strict delivery errors are rethrown.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Emit JSON before rethrowing delivery failures:** `src/agents/command/delivery.ts:330`
  - body: The PR suppresses the existing early JSON envelope whenever `--deliver` is set. In strict delivery mode, `deliverOutboundPayloads` can throw before the later JSON block runs, so `openclaw agent --json --deliver` can produce no parseable stdout even though the CLI JSON contract reserves stdout for JSON.
  - confidence: 0.9
- **[P2] Do not mark hook cancellations as failed delivery:** `src/agents/command/delivery.ts:426`
  - body: `deliverySucceeded = results.length > 0` treats an all-cancelled `message_sending` hook as a failed send. Current outbound delivery intentionally returns no result for cancelled payloads, so expected cancellations would produce false failure status/logging and could drive incorrect retry behavior.
  - confidence: 0.86
- **[P2] Preserve result metadata overrides:** `src/agents/command/delivery.ts:474-486`
  - body: The new JSON and return paths use `result.meta` directly. Current main merges `opts.resultMetaOverrides` first, so merging this branch as-is would drop metadata such as embedded gateway fallback markers from `--json` output and the returned result.
  - confidence: 0.82
