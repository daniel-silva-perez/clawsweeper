## Summary

Keep open. Current main does not implement this PR’s central contract: `deliverAgentCommandResult` still emits JSON before delivery, returns only `payloads`/`meta`, and discards `deliverOutboundPayloads` results instead of returning or logging structured `deliveryStatus`. The PR is also an active draft explicitly waiting on related delivery-outcome work, so the best maintainer path is sequencing and review rather than cleanup closure.

## Best Possible Solution

Keep this PR open as active draft work. Land or fold in the #57843-style `DeliveryOutcome` metadata first, then rebase this PR, resolve the strict JSON error path and the overlap with #57755, and verify the targeted delivery tests plus the changed gate before merge.
