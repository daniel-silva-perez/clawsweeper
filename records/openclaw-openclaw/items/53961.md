## Summary

Keep open. Current main still lacks structured command-delivery status, while this PR is an open draft with dirty merge state and unresolved delivery JSON/cancellation semantics that need maintainer sequencing before merge.

## What This Changes

This PR adds deliveryStatus tracking and diagnostic delivery-failure logging to deliverAgentCommandResult, includes that status in --json --deliver output, and expands command delivery tests.

## Best Possible Solution

Keep this as a useful implementation candidate, but sequence it with #57843, preserve the single parseable --json contract on strict delivery errors, and consolidate the JSON status surface with #57755 before maintainer review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Emit JSON before rethrowing delivery failures:** `src/agents/command/delivery.ts:330`
  - body: The PR suppresses the existing early JSON envelope whenever --deliver is set. In strict delivery mode, deliverOutboundPayloads rethrows before the later JSON block runs, so openclaw agent --json --deliver can produce no parseable JSON on transport failure even though the current command/docs reserve stdout for JSON output.
  - confidence: 0.9
- **[P2] Do not mark hook cancellations as failed delivery:** `src/agents/command/delivery.ts:426`
  - body: deliverySucceeded = results.length > 0 treats every empty result set as a failed send, but deliverOutboundPayloads intentionally returns [] when message_sending hooks cancel all payloads. That creates false failure telemetry/status for expected cancellations, so this should consume cancellation metadata or wait for the DeliveryOutcome work.
  - confidence: 0.86
