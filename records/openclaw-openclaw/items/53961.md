## Summary

Keep open. Current main still lacks the proposed delivery outcome contract: command delivery writes JSON with only payloads/meta, awaits delivery without using the returned result, and outbound delivery still cannot distinguish hook-cancelled zero-result sends from delivery failures. The PR is an open draft with explicit sequencing against #57843 and #57755, so it needs maintainer review rather than cleanup closure.

## What This Changes

This PR adds deliveryStatus tracking and diagnostic delivery-failure logging to deliverAgentCommandResult, surfaces that status in --json --deliver output, and adds command delivery tests for success, preflight, thrown-error, partial-failure, zero-result, and no-payload paths.

## Best Possible Solution

Keep this PR as a useful implementation candidate, but sequence it behind the outbound DeliveryOutcome metadata work, preserve the single parseable --json contract on strict delivery errors, and consolidate the JSON status surface with the related JSON PR before standard maintainer review.
