## Summary

Current main still has the post-save cleanup gap this PR targets. `ensureMediaHosted()` saves media before hostname lookup, port probing, and optional media server startup, but only removes the saved file in the server-required/start-not-allowed branch. The PR is a focused, recently updated bug fix with regression coverage and no apparent security-sensitive or supply-chain surface changes, so it should remain open for maintainer review.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The likely right fix is a focused `src/media/host.ts` change that wraps all post-save hosting setup failures in best-effort cleanup, with `src/media/host.test.ts` regression coverage for hostname lookup failure, unexpected port-probe failure, and media server startup failure.
