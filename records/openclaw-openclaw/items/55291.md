## Summary

Current main still has the post-save cleanup gap this PR targets. `ensureMediaHosted()` saves media before hostname lookup, port probing, and optional media-server startup, but only removes the saved path in the server-required/start-not-allowed branch. The PR is a focused two-file bug fix with regression tests and no apparent security-sensitive or supply-chain changes, so it should remain open for maintainer review rather than be closed as obsolete.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The likely right fix is the focused `src/media/host.ts` shared post-save cleanup path plus `src/media/host.test.ts` regression coverage for hostname lookup failure, unexpected port-probe failure, and media-server startup failure; after maintainer review, land it if CI/test proof remains scoped and green.
