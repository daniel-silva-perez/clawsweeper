## Summary

Keep open. Current main still sends local `infer model run --model` through the agent-aware simple-completion selection path, while open PR #74028 explicitly targets this same bug and remains unmerged; policy says this issue should stay open until that implementation PR lands or is closed.

## What This Changes

`openclaw infer model run --model <provider/model>` should honor the requested provider/model for local probes instead of silently falling back to the default agent model.

## Best Possible Solution

Review PR #74028 and either land it after normal validation or close it with a replacement plan. The desired end state is a focused local `runModelRun` change that prepares fully qualified provider/model overrides directly while leaving gateway forwarding and unqualified agent-aware model selection intact.

## Reproduction Assessment

Yes. The reported command path is traceable on current main: local `infer model run --model github-copilot/...` reaches `runModelRun`, which still delegates to `prepareSimpleCompletionModelForAgent` instead of directly preparing the qualified provider/model; I did not run a live Copilot request in this read-only sweep.

## Solution Assessment

Yes. The linked PR's direction is the narrow maintainable fix: split fully qualified local overrides and call direct simple-completion preparation only for that case, while preserving the existing agent-aware path for unqualified refs.
