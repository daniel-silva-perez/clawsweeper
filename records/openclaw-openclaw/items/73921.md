## Summary

Keep open. Current main still routes local `infer model run --model` through the agent-aware simple-completion selection path, and the provided GitHub context shows open PR #74028 explicitly closes this issue with the narrow repair, so this issue should stay open until that PR lands or is closed.

## What This Changes

The issue requests that `openclaw infer model run --model <provider/model>` honor the requested Copilot provider/model in local model probes instead of silently routing to the configured/default Anthropic model.

## Best Possible Solution

Review and either land PR #74028 or close it with a replacement plan. The desired end state is a focused `runModelRun` change that prepares fully qualified local provider/model overrides directly, keeps unqualified refs on the existing agent-aware selection path, and preserves the already-explicit gateway forwarding behavior.

## Reproduction Assessment

Yes. The current-main source path is enough to reproduce the routing concern at the CLI seam: local `infer model run --model github-copilot/...` reaches `runModelRun`, which still delegates the override to `prepareSimpleCompletionModelForAgent`; I did not run a live Copilot request in this read-only review.

## Solution Assessment

Yes. Based on the linked PR context and current source, the best fix is the narrow local-path split for fully qualified overrides, with focused CLI tests; broader model canonicalization or OpenRouter follow-ups should stay in their separate related items.
