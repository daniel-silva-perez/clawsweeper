## Summary

Keep open. Current main still has the local `infer model run` path going through the agent-aware simple-completion preparation seam, and the issue is already paired with open PR #74028, which explicitly targets this local fully qualified provider/model override behavior. Per policy, the issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

`openclaw infer model run --model <copilot-provider/model>` should honor the requested Copilot model instead of routing the local probe through the agent default model.

## Best Possible Solution

Land a focused fix that direct-prepares fully qualified local `model run` overrides, adds regression coverage for Copilot provider/model probes, and preserves existing behavior for unqualified refs, aliases, profile suffixes, and gateway forwarding.

## Reproduction Assessment

Yes. The issue gives exact CLI commands and an observable response-metadata check; current main also exposes a concrete static seam in the local `runModelRun` path, though this read-only pass did not execute a live provider call.

## Solution Assessment

Yes. The linked PR’s intended direction is the narrowest maintainable fix: only fully qualified local overrides should bypass agent-default selection, while unqualified refs continue using the existing agent-aware resolver.
