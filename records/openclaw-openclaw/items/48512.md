## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Use remaining budget for each idle re-wait:** `src/agents/pi-embedded-runner/wait-for-idle-before-flush.ts:59`
  - body: The loop starts a global timeout window, but each retry calls `waitForAgentIdleBestEffort` with the original `timeoutMs`. If an idle wait resolves late while tools are still pending, cleanup can block for another full timeout window instead of honoring the configured budget.
  - confidence: 0.95
- **[P2] Keep pending-tool probing best-effort:** `src/agents/pi-embedded-runner/wait-for-idle-before-flush.ts:74`
  - body: `hasPendingToolCalls.call(opts.agent)` runs in teardown paths but is not guarded. If this introspection throws during shutdown, the cleanup helper can reject before the caller finishes disposal and lock release work, so the probe should be wrapped like `waitForIdle`.
  - confidence: 0.9
- **[P2] Apply the pending hint to compaction cleanup:** `src/agents/pi-embedded-runner/compact.ts:1005`
  - body: The attempt cleanup paths are wrapped, but compaction still passes `session?.agent` directly into `flushPendingToolResultsAfterIdle`. Tool-capable compaction teardown can therefore still flush during the same retry gap this PR is trying to close.
  - confidence: 0.93
- **[P1] Remove generated bundle artifacts from the branch:** `.gitignore:150`
  - body: The branch adds a `dist.bak/` tree with generated JavaScript bundles unrelated to the idle-flush fix. This makes the diff effectively unreviewable and introduces executable artifacts with unclear provenance into the source tree.
  - confidence: 0.95
- **[P1] Preserve the originating route in hook context:** `src/auto-reply/reply/dispatch-from-config.ts:176-181`
  - body: The new internal `message:received` hook context is keyed from `Surface` and `To`/`From` instead of the canonical inbound hook context that preserves originating route fields. Cross-provider routed sessions can reset or look up hook state under the wrong conversation.
  - confidence: 0.87
- **[P1] Run extraction without shell interpolation:** `src/hooks/bundled/session-memory/handler.ts:186`
  - body: The session-memory hook builds a shell command with interpolated `extractScript` and `sessionId`. Use `execFile` or equivalent argv passing so a crafted workspace path or session file basename cannot alter the command line.
  - confidence: 0.82
- **[P2] Clear only the current conversation's cache timer:** `src/hooks/bundled/cache-ttl-warning/handler.ts:219-223`
  - body: On `/new` or `/reset`, this hook clears every timer in the process-wide store. In multi-conversation gateways, one reset would remove TTL notices for unrelated chats, so it should derive and clear only the triggering conversation key.
  - confidence: 0.84
