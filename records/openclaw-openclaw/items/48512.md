## Summary

Keep this PR open for maintainer handling. Current main still does not implement the proposed pending-tool-call idle-flush hint, but the PR branch is much too contaminated with unrelated generated artifacts, hooks, gateway, command, UI, and code-execution-sensitive changes to merge as-is.

## Best Possible Solution

Keep this PR open for maintainer handling, but do not merge the branch as-is. Request or salvage a clean replacement scoped to the embedded runner: add a best-effort non-throwing pending-tool-call hint, bridge it into attempt error cleanup, normal attempt cleanup, and compaction cleanup, preserve the total timeout budget across re-waits, and add focused retry-gap plus adapter regression tests while dropping unrelated generated/runtime/hook/UI/gateway changes.
