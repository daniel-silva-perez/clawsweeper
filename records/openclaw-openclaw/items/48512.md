## Summary

Keep open. The underlying embedded-runner race is plausible and current main still lacks the proposed pending-tool hint, but this branch is not mergeable as-is because it mixes the narrow fix with generated bundles, unrelated hook/gateway/UI changes, and concrete security-sensitive regressions.

## What This Changes

The branch adds a pending-tool-call idle-flush hint for the embedded runner, while also adding generated `dist.bak` bundles and unrelated hook, gateway, session, delivery, command, and Control UI changes.

## Best Possible Solution

Do not merge this branch. Replace it with a clean embedded-runner-only patch that adds a best-effort pending-tool hint, honors one total timeout budget, covers attempt and compaction cleanup, and adds focused regression tests without generated artifacts or unrelated product-surface changes.

  - body: The new `message:received` hook context derives conversation fields from `Surface` and `To`/`From` instead of the canonical inbound hook context that preserves originating route fields. Cross-provider routed sessions can therefore key hook state to the wrong conversation.
  - confidence: 0.87

  - body: The pending-tool loop calls `waitForAgentIdleBestEffort` with the original timeout on each pass. If an idle wait resolves late while tools remain pending, cleanup can block for another full timeout window instead of honoring the configured budget.
  - confidence: 0.95
- **[P2] Keep pending-tool probing best-effort:** `src/agents/pi-embedded-runner/wait-for-idle-before-flush.ts:74`
  - body: `hasPendingToolCalls` runs in teardown paths but is not guarded. If introspection throws during shutdown, cleanup can reject before disposal and lock release finish, so the probe should be wrapped like `waitForIdle`.
  - confidence: 0.9
- **[P2] Apply the pending hint to compaction cleanup:** `src/agents/pi-embedded-runner/compact.ts:1005`
  - body: The attempt cleanup paths are wrapped, but compaction still passes `session?.agent` directly into `flushPendingToolResultsAfterIdle`. Tool-capable compaction teardown can still flush during the retry gap this PR is trying to close.
  - confidence: 0.93
