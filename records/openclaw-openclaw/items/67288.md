## Summary

Keep this issue open. Current main still lacks the requested Bedrock Mantle discovery opt-out: the manifest does not allow `discovery.enabled`, registration does not pass plugin config into discovery, and the resolver still attempts IAM token generation when no explicit bearer token is present. The issue is also paired with open PR #67341 using closing syntax, so it should remain open until that PR or an equivalent fix is merged or explicitly abandoned.

## What This Changes

The issue requests a `plugins.entries.amazon-bedrock-mantle.config.discovery.enabled` gate so Bedrock Mantle implicit discovery can be disabled without turning off the plugin.

## Best Possible Solution

Land PR #67341 if it still applies, or replace it with an equivalent focused fix in `extensions/amazon-bedrock-mantle`: add `discovery.enabled` to the manifest schema and UI hints, pass validated plugin config into Mantle discovery, short-circuit before IAM token generation and `/v1/models` when disabled, document the option on the Bedrock Mantle provider page, and add regression coverage for disabled/default/enabled discovery behavior.
