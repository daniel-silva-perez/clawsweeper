## Summary

Keep this issue open. Current main still has the reported Bedrock Mantle discovery opt-out gap, and the supplied GitHub context shows an open PR with closing syntax for this exact bug, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests a `plugins.entries.amazon-bedrock-mantle.config.discovery.enabled` opt-out so Bedrock Mantle discovery can be disabled without disabling the entire plugin.

## Best Possible Solution

Land PR #67341 if it still applies, or replace it with an equivalent focused Mantle provider fix: add `discovery.enabled` to the manifest schema and UI hints, pass validated plugin config into Mantle discovery, short-circuit before IAM token generation and `/v1/models` when disabled, document the option on the Bedrock Mantle provider page, and add regression coverage for disabled/default/enabled discovery behavior.

## Reproduction Assessment

Yes. The issue provides concrete steps and logs, and current-main source inspection gives a high-confidence static reproduction path: the requested config key is rejected by the manifest schema, registration does not pass plugin config, and the resolver still attempts IAM bearer-token generation when no explicit token is present.

## Solution Assessment

Yes. A plugin-owned `discovery.enabled === false` short-circuit that mirrors the existing `amazon-bedrock` implementation is the narrowest maintainable fix, as long as it includes manifest schema, runtime wiring, docs, and tests.
