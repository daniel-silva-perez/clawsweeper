## Summary

Keep this PR open. Current main has turn-source delivery-plan hardening that helps immediate outbound routing, but it still does not implement the PR's handler-level session delivery-context precedence/persistence behavior, and the PR discussion includes an unresolved route-field atomicity security concern that should be resolved before landing or closing.

## Best Possible Solution

Keep this PR open. Maintainers should resolve the Aisle accountId/to atomicity concern, adapt the branch to the current turn-source delivery helpers, and then either land this branch or apply an equivalent narrow fix in src/gateway/server-methods/agent.ts with the focused gateway regression tests and changelog credit.
