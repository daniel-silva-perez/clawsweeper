## Summary

Keep open. Current main still has the reported schema/docs mismatch: `agents.list[]` is validated by a strict `AgentEntrySchema` that includes `thinkingDefault`, `reasoningDefault`, and `fastModeDefault`, but not `verboseDefault` or `elevatedDefault`. The linked PR #73696 is an open, unmerged implementation candidate, so this issue should remain open until that PR lands or is superseded.

## Best Possible Solution

Review and land PR #73696 or an equivalent fix. The fix should add `verboseDefault` and `elevatedDefault` to `AgentEntrySchema`, align per-agent TypeScript/runtime config handling for `elevatedDefault`, update generated schema/help labels and public docs, and add focused validation tests for accepted and rejected per-agent values.
