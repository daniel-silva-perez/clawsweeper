## Summary

Keep this issue open. Current main still has the reported config-contract mismatch, and open PR #74643 is the active unmerged closing implementation, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to either accept `verboseDefault` and `elevatedDefault` on `agents.list[]` entries like adjacent per-agent defaults, or remove the per-agent resolver/type surfaces for those fields.

## Best Possible Solution

Review and land PR #74643 or an equivalent narrow fix that aligns `AgentEntrySchema`, public config types, generated schema/help metadata, docs, and focused validation tests before closing this issue.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: `agents.list` is validated through strict `AgentEntrySchema`, and that schema omits the two reported keys, so config validation rejects them as unrecognized; I did not run the CLI because this review is read-only.

## Solution Assessment

Yes. Accepting the two fields in the per-agent schema and documenting their precedence is the narrowest maintainable solution because the resolver, status, and directive paths already treat them as per-agent-capable values.
