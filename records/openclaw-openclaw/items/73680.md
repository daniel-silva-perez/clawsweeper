## Summary

Keep open. Current main still has the reported config contract mismatch: `agents.list[]` is validated by a strict `AgentEntrySchema` that lacks `verboseDefault` and `elevatedDefault`, while adjacent defaults, generated metadata, docs, and runtime paths remain inconsistent.

## What This Changes

The issue requests aligning per-agent `verboseDefault` and `elevatedDefault` handling so config validation, generated schema/help, docs, types, and runtime resolution either accept them consistently or remove the per-agent surface.

## Best Possible Solution

Review and dedupe PR #73696 and PR #74643, then land one focused fix that aligns `AgentEntrySchema`, generated schema/help labels, public config types, docs, and regression tests, with explicit verification that per-agent elevated defaults do not bypass existing elevated-mode gates.

## Reproduction Assessment

Yes. ## Solution Assessment

Yes, the preferred direction is the narrow maintainable fix. Adding the two per-agent schema fields plus generated docs/types/tests is appropriate, but the final implementation should also verify the actual per-agent `elevatedDefault` runtime path rather than only making validation pass.
