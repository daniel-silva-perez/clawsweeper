## Summary

Keep open. Current main still has the reported silent-loss path: Feishu media download/save failures are caught and logged without structured failure state or chat-visible notification, and normal dispatch remains reachable for media messages whose parsed placeholder text is nonempty.

## What This Changes

The issue requests a Feishu plugin fix so oversized media that exceeds `mediaMaxMb` notifies the chat and prevents the agent from acting as if the file was received.

## Best Possible Solution

Fix this in the bundled Feishu plugin by returning structured media-resolution failure state, sending a Feishu-visible warning that names the configured `mediaMaxMb` limit, and blocking ordinary agent dispatch when the triggering input depended on dropped media.

## Reproduction Assessment

Yes. The report has concrete steps, and source inspection gives a deterministic reproduction path: configure `mediaMaxMb`, send oversized Feishu media, hit `saveMediaBuffer` rejection, observe `resolveFeishuMediaList` logging and returning without a failure marker.

## Solution Assessment

Yes. A plugin-local structured failure plus chat notification is the narrowest maintainable solution because Feishu owns reply targeting, thread behavior, and embedded post media parsing.
