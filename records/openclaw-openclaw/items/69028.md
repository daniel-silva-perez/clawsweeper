## Summary

Keep open. Current main still matches the reported Feishu silent-failure path: oversized media makes `saveMediaBuffer` throw, `resolveFeishuMediaList` catches and logs the failure, then `handleFeishuMessage` continues into normal agent dispatch with an empty media payload and no chat-visible Feishu warning.

## Best Possible Solution

Keep this issue open and fix it in the bundled Feishu plugin. The best path is to make Feishu media resolution return structured failure state or throw a typed recoverable media error, send a Feishu-visible warning that includes the configured `mediaMaxMb` limit, and avoid a normal agent run when the triggering input was only dropped media. Add regression tests for oversized single-file media and embedded post media failures that assert both user notification and no unqualified agent dispatch.
