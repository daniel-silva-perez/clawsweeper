## Summary

Keep open. Current main still catches and logs Feishu media download/save failures, including oversized `mediaMaxMb` failures from `saveMediaBuffer`, then continues into normal agent dispatch with an empty or partial media payload and no Feishu-visible warning. The issue is recent, has no protected labels, no maintainer author association, and the related media reports in the provided context are adjacent rather than canonical replacements.

## Best Possible Solution

Keep this issue open and fix it in the bundled Feishu plugin. The strongest path is to make Feishu media resolution return structured failure state or throw a typed recoverable media error, send a Feishu-visible warning that includes the configured `mediaMaxMb` limit, and avoid an unqualified normal agent run when the triggering input was only dropped media. Add regression tests for oversized single-file media and embedded post media failures.
