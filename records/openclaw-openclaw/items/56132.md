## Summary

Keep open. The attachment-backed issue is concrete: it asks to restore `sessions_spawn({ thread: true, mode: "session" })` binding from Feishu plain group chats. Current main still rejects non-topic Feishu group conversations for this path; only DMs and topic/thread conversations are supported. Static Feishu group `bindings` may solve the reporter's per-group persona use case, but they do not restore the requested runtime current-conversation subagent binding.

## Best Possible Solution

Keep this issue open but narrow it to the missing behavior: Feishu plain group `sessions_spawn({ thread: true, mode: "session" })` current-conversation binding. A good fix would live in the Feishu plugin's subagent/thread-binding path, add an opt-in policy if needed, bind `chat:<oc_...>` conversations safely, preserve delivery back to the originating group, and add regression coverage next to `extensions/feishu/src/subagent-hooks.test.ts`. If maintainers decide static group `bindings` are the intended solution, close this with a product/docs clarification rather than treating current main as already implementing the exact request.
