## Summary

Keep this issue open. The attachment-backed request is concrete and current main still rejects Feishu plain group `sessions_spawn({ runtime: "subagent", thread: true, mode: "session" })` binding; related Feishu ACP/topic work does not fully cover the requested plain group subagent behavior.

## Best Possible Solution

Keep this issue open and narrow the work to Feishu plain group native subagent current-conversation binding. A good fix would live in the Feishu plugin subagent/thread-binding path, safely bind `chat:<oc_...>` conversations when policy allows it, preserve completion delivery to the originating group, and add coverage next to `extensions/feishu/src/subagent-hooks.test.ts`. If maintainers prefer static group `bindings` as the supported model, document that product decision and close with that clarification.
