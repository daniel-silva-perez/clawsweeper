## Summary

Keep open. Current main still rejects the requested Feishu plain group `sessions_spawn({ thread: true, mode: "session" })` current-conversation binding. Feishu supports DMs and topic/thread conversations, and static `bindings` can route whole groups to agents, but that does not restore the runtime group-chat subagent binding requested here.

## Best Possible Solution

Keep this issue open and narrow the requested work to Feishu plain group `sessions_spawn({ thread: true, mode: "session" })` current-conversation binding. A good implementation would live in the Feishu plugin subagent/thread-binding path, safely bind `chat:<oc_...>` conversations when policy allows it, preserve completion delivery to the originating group, and add coverage next to `extensions/feishu/src/subagent-hooks.test.ts`. If maintainers decide static group `bindings` are the intended replacement, close with a product/docs clarification instead of treating current main as implemented.
