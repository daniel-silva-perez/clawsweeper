## Summary

Keep open. The reported Telegram voice-note path is a real behavior gap, but this PR only wires `audioAsVoice` through the exported legacy `telegramOutbound` adapter. Current plugin loading uses `telegramPlugin` from `extensions/telegram/src/channel.ts`, and that registered outbound path still builds Telegram send options without `asVoice`, so the claimed TTS voice-note fix would not hold on the normal channel plugin path.

## What This Changes

The PR forwards `audioAsVoice` as `asVoice` in `extensions/telegram/src/outbound-adapter.ts` and adds three unit tests around the legacy Telegram outbound adapter.

## Best Possible Solution

Centralize Telegram voice-note option propagation so both the registered channel plugin path and any legacy exported adapter path derive `asVoice` from `ctx.audioAsVoice` or `payload.audioAsVoice`, then cover the registered `telegramPlugin.outbound.base.sendPayload` or shared helper with a focused regression test.
