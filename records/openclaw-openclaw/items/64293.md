## Summary

Keep this issue open. Current main still does not treat `agents.defaults.heartbeat: {}` as a disable signal: the schema accepts an empty heartbeat object, the default agent remains heartbeat-enabled, interval resolution falls through to the 30m default, the runner schedules any resolved interval, and Anthropic provider defaults can backfill `every` onto an empty object. The linked PR #64540 remains open and unmerged, so it is an implementation candidate rather than a close reason.

## Best Possible Solution

Keep this issue open and review PR #64540 or land an equivalent maintainer patch. The fix should define the empty-object heartbeat contract, apply it consistently in default and per-agent resolution, preserve the documented `every: "0m"` disable behavior, prevent provider default backfill from re-enabling explicit empty configs, and add focused regression tests plus docs/validation updates for the chosen contract.
