## Summary

Keep open. Current main still does not treat `agents.defaults.heartbeat: {}` as a disable signal, and the matching implementation candidate PR remains open and unmerged.

## Best Possible Solution

Keep this issue open and review or land PR #64540, or an equivalent maintainer patch. The fix should define the empty-object heartbeat contract, apply it consistently in enabled-agent checks, interval resolution, runner scheduling, and Anthropic default injection, preserve the documented `every: "0m"` disable behavior, and add focused regression tests plus docs/config guidance for the chosen contract.
