Keep open. Current main still accepts `agents.defaults.heartbeat: {}` and resolves it as an enabled heartbeat with a default cadence, and the open closing PR #64540 has not merged, so this issue should remain the tracker until that fix or an equivalent maintainer patch lands.

## What This Changes

The issue requests that an empty heartbeat config object disable automatic heartbeat polling instead of falling back to the default 30-minute schedule.

## Best Possible Solution
