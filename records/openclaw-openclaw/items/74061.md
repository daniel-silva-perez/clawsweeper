## Summary

Keep open. Current main still routes sandboxed final-reply media through sandbox path resolution before the agent-scoped outbound media access path, so a host-absolute workspace file can still be dropped and converted to the visible media-failure fallback.

## What This Changes

Telegram final replies should deliver local media referenced by `<final>MEDIA:` from an agent workspace path instead of sending `Media failed`.

## Best Possible Solution

Fix the sandboxed normal-reply media path narrowly: after sandbox mapping fails, allow only managed media or paths that pass the existing agent-scoped outbound media access policy for the active workspace/session, then stage through the existing outbound attachment path. Add positive coverage for a sandboxed Telegram-style final reply with a host-absolute agent-workspace path and negative coverage for unrelated host paths; decide explicitly whether cron-owned announce delivery shares the same fix.

## Reproduction Assessment

Yes. The current source and tests provide a high-confidence reproduction path: with `sandboxRoot` present, an absolute host-local workspace `MEDIA:` path is resolved as a sandbox path, rejected as outside the sandbox root, dropped, and converted to `⚠️ Media failed.`

## Solution Assessment

No. Current main is not the best solution because it preserves a blanket sandbox-first rejection for host-absolute paths that may already be valid under the agent-scoped outbound media policy; the safer path is a policy-checked fallback, not an unrestricted host-path allowlist.
