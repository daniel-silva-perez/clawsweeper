## Summary

Keep this PR open. Current main still does not implement the Google-owned prefix stripping this PR targets in the Google model helper, manifest policy, native Google transport URL builder, or Gemini PDF path. The overlapping #41517 work is still open and only covers part of the resolver problem, while this PR still contains useful Google transport/PDF work and unrelated security-sensitive diffs that need maintainer review or splitting.

## Best Possible Solution

Keep the PR open for maintainer follow-up, or fold the useful pieces into a focused Google-owned fix: add Google manifest/helper prefix stripping for google, google-vertex, and google-gemini-cli; normalize and validate native Google transport and geminiAnalyzePdf before provider calls; add regression coverage for OpenAI-compatible Google chat plus native Gemini/PDF paths; coordinate with #41517; and split unrelated Signal/catalog/bundled-dir/browser changes into separate PRs.
