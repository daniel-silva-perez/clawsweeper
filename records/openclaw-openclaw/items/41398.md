## Summary

Keep open. The PR targets a real Google model-prefix regression with useful linked context from the PR body, Greptile review, and prior ClawSweeper review, but current main still does not fully implement the intended fix and the branch needs a current-main rework around the plugin-owned Google seams.

## Best Possible Solution

Keep this PR open and request a current-main rework: implement Google-scoped prefix cleanup through `extensions/google` model-id/manifest/runtime seams, add regression tests for duplicated Google provider prefixes across resolveModel, OpenAI-compatible chat payloads, native Gemini stream URLs, and `geminiAnalyzePdf`, preserve slash-containing model IDs for non-Google providers, and drop the unrelated `.secrets.baseline` change. If a separate canonical PR lands that exact coverage first, close this PR as superseded or implemented then.
