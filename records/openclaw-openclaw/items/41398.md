## Summary

Keep open. Current main still does not implement this PR's intended Google-owned prefix normalization: the Google helper and manifest only alias bare IDs, the generic manifest normalizer can only strip prefixes declared by the provider policy, and both native Google transport and Gemini PDF URL construction can still use prefixed model IDs. #41517 is an open narrower same-provider resolver candidate, not a merged superseding fix for the Google transport and PDF paths.

## Best Possible Solution

Keep this PR open for maintainer follow-up or combine it with #41517: implement Google-owned prefix cleanup in the Google manifest/helper/runtime transport and Gemini PDF path, preserve slash-containing IDs for non-Google providers, keep the URL path validation from the latest PR revision, and add regression tests for OpenAI-compatible Google chat plus native Gemini/PDF URL construction.

## Risks / Open Questions

- A generic slash-splitting fix can regress valid slash-containing model IDs for providers such as OpenRouter, NVIDIA, MLX, or Hugging Face; the durable fix should stay Google-owned or same-provider scoped.
- The PR touches request URL construction for Google provider calls. The latest guard appears aimed at the reported path-injection concern, but maintainers should still review `models/` and `tunedModels/` handling before merge.
- Current main likely still fails native Google or Gemini PDF paths when a prefixed Google model id reaches those surfaces, even if some session-boundary model-ref cleanup has improved other call paths.
