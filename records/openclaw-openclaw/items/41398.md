## Summary

Keep open. Current main has only partial session-boundary cleanup for duplicated provider prefixes; it does not implement this PR's intended Google-owned model normalization. Google helper/manifest paths still do not strip `google/` or `models/`, native Gemini/PDF URL construction can still use prefixed IDs, and #41517 is an open narrower implementation candidate rather than a merged superseding fix.

## Best Possible Solution

Keep this PR open for a current-main rework: implement Google-owned prefix cleanup through `extensions/google` manifest/helper/runtime seams, add regression coverage for Google OpenAI-compatible chat plus native Gemini/PDF URL construction, preserve slash-containing IDs for non-Google providers, and coordinate with #41517 if it becomes the canonical same-provider `resolveModel` fix.

## Risks / Open Questions

- A generic slash-splitting fix can regress valid slash-containing model IDs for providers such as OpenRouter, NVIDIA, MLX, or Hugging Face; the implementation should stay scoped to Google-owned normalization or same-provider duplication rules.
- The PR should address or explicitly dismiss the Aisle URL-construction finding before merge, because it touches provider request construction under operator credentials.
- #41517 may supply the better current-main `resolveModel` piece, but maintainers likely need to combine or split that work with this PR's Google transport and PDF coverage.
