## Summary

Keep this PR open. Current main has partial provider/model-prefix handling in selection and session override paths, but it still does not fully implement the PR’s intended Google-owned prefix stripping before Google API calls. The Google helper and manifest only alias bare IDs, resolved models are not transport-normalized for Google, OpenAI-compatible payloads still use `model.id` verbatim, and the native PDF URL still interpolates `params.modelId` directly.

## Best Possible Solution

Keep the PR open but request a current-main rework in the Google-owned seams: strip Google-specific duplicated prefixes through `extensions/google` model-id or manifest normalization, ensure resolved Google transport and native PDF paths cannot send `google/gemini-...` to Google APIs, add regression tests for OpenAI-compatible chat and `geminiAnalyzePdf`, preserve nested model IDs for non-Google providers, and drop the unrelated `.secrets.baseline` change.
