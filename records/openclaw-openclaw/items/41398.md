## Summary

Keep open. Current main still does not implement this PR's intended Google prefix stripping across the Google helper, manifest policy, native Google transport URL construction, or Gemini PDF path. The related #41517 PR is open and narrower; it does not supersede the Google transport/PDF work.

## Best Possible Solution

Keep this PR open for maintainer follow-up or fold the useful pieces into a focused Google-owned fix: add Google manifest/helper prefix stripping, normalize native Google transport and Gemini PDF URL construction before provider calls, keep slash-containing IDs intact for non-Google providers, preserve URL path validation, and add regression tests for OpenAI-compatible Google chat plus native Gemini/PDF paths. Split unrelated browser and Signal security hardening into their own PR if those changes are still desired.
