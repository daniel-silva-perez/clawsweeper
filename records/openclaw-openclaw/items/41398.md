## Summary

Keep open. Current main still does not implement this PR's intended Google-owned model-id prefix cleanup across manifest normalization, Google helper/runtime paths, PDF native calls, or OpenAI-compatible payload construction. The related #41517 is an open narrower implementation candidate for one same-provider resolveModel path, but it does not supersede the broader Google transport and PDF hardening in this PR.

## Best Possible Solution

Keep this PR open and rework it against current main: add Google-owned `stripPrefixes` policy and helper/runtime normalization, cover Google OpenAI-compatible chat plus native Gemini and PDF URL construction with focused tests, address model-id URL safety, and coordinate with #41517 if maintainers want the generic same-provider resolveModel piece split out.
