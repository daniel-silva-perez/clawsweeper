## Summary

Keep this PR open. Current main still lacks the Google-owned prefix stripping that this PR targets in the Google model helper, manifest policy, native Google transport URL builder, and Gemini PDF path. The overlapping #41517 is still open and only covers part of the resolver path, while this PR also contains unrelated security-sensitive changes that need maintainer review or splitting.

## Best Possible Solution

Keep this PR open for maintainer follow-up, or fold the useful parts into a focused Google-owned fix: add stripPrefixes for google, google-vertex, and google-gemini-cli; make normalizeGoogleModelId strip Google-owned provider/API prefixes before aliasing; normalize and validate model IDs before native Google transport and geminiAnalyzePdf URL construction; add regression tests for OpenAI-compatible Google chat plus native Gemini/PDF paths; coordinate with #41517; and split unrelated browser, Signal, catalog, and bundled-dir changes into separate PRs.
