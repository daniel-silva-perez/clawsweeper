## Summary

Keep open. Current main still does not implement this PR's intended Google prefix stripping across the Google helper, manifest policy, native Google transport URL construction, or Gemini PDF path. The related #41517 PR overlaps only the resolver same-provider path and remains open, while this PR now includes unrelated security-sensitive changes that need split or explicit review.

## Best Possible Solution

Keep this PR open for maintainer follow-up or fold the useful pieces into a focused Google-owned fix: add Google manifest/helper prefix stripping, normalize and validate native Google transport and `geminiAnalyzePdf()` before provider calls, preserve slash-containing model ids for non-Google providers, add regression tests for OpenAI-compatible Google chat plus native Gemini/PDF paths, coordinate with #41517, and split unrelated browser, Signal, catalog, and bundled-dir changes into separate PRs.
