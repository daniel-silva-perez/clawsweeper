## Summary

Keep this PR open. Current main still lacks the Google-owned prefix stripping this PR is trying to add in the Google helper, manifest policy, native Google transport URL construction, and Gemini PDF path. The overlapping #41517 work only covers part of the same-provider resolver problem, while this PR still contains useful transport/PDF work and security-sensitive unrelated diffs that need maintainer review or splitting.

## Best Possible Solution

Keep the PR open for maintainer follow-up or fold the useful parts into a focused Google-owned fix: add Google manifest/helper prefix stripping for `google`, `google-vertex`, and `google-gemini-cli`; normalize and validate native Google transport and `geminiAnalyzePdf()` before provider calls; preserve slash-containing model IDs for non-Google pass-through providers; add regression tests for OpenAI-compatible Google chat plus native Gemini/PDF paths; coordinate with #41517; and split unrelated Signal/catalog/bundled-dir/browser changes into separate PRs.
