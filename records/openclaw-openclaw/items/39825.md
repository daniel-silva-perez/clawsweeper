## Summary

Keep this issue open. Current main still has the reported Discord voice join abort shape: `/vc join` waits up to a hard-coded 15 seconds for `VoiceConnectionStatus.Ready`, destroys the connection on failure, and only records a voice session after that wait succeeds, so `/vc status` can still report no active sessions after an abort.

## Best Possible Solution

Keep this issue open as a Discord native voice readiness-abort tracker unless maintainers intentionally consolidate it into #65039. A concrete fix likely belongs in `extensions/discord/src/voice/manager.ts`: add regression coverage for `entersState` aborts/timeouts before session activation, improve join diagnostics around the voice WebSocket/UDP readiness phase, and decide whether to add bounded retry/backoff, a longer default, or a documented `channels.discord.voice.connectTimeoutMs` setting.
