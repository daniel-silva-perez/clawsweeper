## Summary

Keep this issue open. Current main still waits for Discord voice `Ready` with a fixed 15s timeout, destroys the connection before recording a session when that wait aborts, and `/vc status` still reports no active sessions in that path. Related #65039 remains open with a more precise trace for the same pre-Ready abort family, so this is not implemented or safely consolidated yet.

## Best Possible Solution

Keep this open as a Discord plugin voice join readiness-abort tracker unless maintainers explicitly consolidate it into #65039. The fix likely belongs in `extensions/discord/src/voice/manager.ts`: add regression coverage for `entersState` aborts before session activation, log the pre-Ready voice networking stage clearly, then decide between bounded retry/backoff, a longer or configurable `channels.discord.voice.connectTimeoutMs`, DAVE-enabled fallback, or documented rejection/deprecation of `daveEncryption=false` if Discord now requires DAVE.
