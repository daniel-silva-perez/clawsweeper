## Summary

Keep this issue open. Current main still uses the same Discord voice join path: `/vc join` waits for `VoiceConnectionStatus.Ready` with a fixed 15s timeout, destroys the connection before recording a session when that wait aborts, and `/vc status` still reports no active session. Related #65039 remains open with a narrower pre-Ready trace, and there is no merged fix or maintainer consolidation decision yet.

## Best Possible Solution

Keep this open as a Discord plugin voice join readiness-abort tracker unless maintainers explicitly consolidate it into #65039. The likely fix belongs in `extensions/discord/src/voice/manager.ts`: add regression coverage for `entersState` aborts before session activation, log the pre-Ready voice networking stage clearly, then decide between bounded retry/backoff, a longer or configurable `channels.discord.voice.connectTimeoutMs`, DAVE-enabled fallback, or documented rejection/deprecation of `daveEncryption=false` if Discord now requires DAVE.
