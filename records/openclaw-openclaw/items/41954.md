## Summary

Keep this PR open. Current main still has the cron Control UI implemented in a single `ui/src/ui/views/cron.ts` module, so the proposed split into focused `cron/filters.ts`, `cron/helpers.ts`, `cron/jobs.ts`, and `cron/runs.ts` modules is not implemented. The work is coherent but stale against newer cron UI behavior and needs a rebase before maintainer review.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase the module split onto current main, preserve the newer cron UI behavior, clean up the stray shell text in the PR body, coordinate with #41892 if that PR proceeds, and run targeted cron view/UI validation. If maintainers do not want refactor-only churn here, close it explicitly as a maintenance-priority decision rather than as implemented.
