## Summary

Keep this PR open. Current main still has the cron view as one 1,786-line `ui/src/ui/views/cron.ts`, with the proposed `ui/src/ui/views/cron/filters.ts`, `helpers.ts`, `jobs.ts`, and `runs.ts` modules absent. The PR is coherent and narrowly scoped, but it is stale against newer cron UI behavior, especially sanitized markdown rendering and in-app run-chat navigation.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up. The useful path is to rebase the module split onto current main, extract the current `cron.ts` behavior into focused sibling modules while preserving sanitized markdown rendering, interactive markdown click handling, and run-chat navigation, then run targeted cron view UI tests. If maintainers do not want refactor-only churn, close it explicitly as a maintenance-priority call rather than as already implemented.
