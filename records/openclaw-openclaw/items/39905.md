## Summary

Keep this PR open. Current main still lacks the central Telegram per-conversation reply and preview-streaming override surface: scoped direct/group/topic config types and strict schemas do not accept `replyToMode`, `streaming`, or legacy `streamMode`, and normal/native Telegram dispatch still receives account-level values captured at startup. The related quoted-DM streaming work is adjacent, but it does not supersede this broader scoped configuration feature.

## Best Possible Solution

Keep this PR, or a maintainer-owned follow-up, open until the feature is ported or explicitly rejected. The best implementation is a focused current-main patch that adds typed and schema-validated Telegram scoped overrides under `direct`, `groups`, and `topics`; resolves effective `replyToMode` and preview streaming with topic > direct/group > account precedence for normal dispatch and native commands; uses the current nested streaming config shape; updates Telegram docs/config metadata; adds focused tests; and removes unrelated baseline/test churn.
