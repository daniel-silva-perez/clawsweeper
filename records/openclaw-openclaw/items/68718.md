## Summary

Keep open for maintainer-directed rework. Current main has the narrow detached-runtime seam that this work should build on, but it does not implement the durable SQLite queue, retry/stall engine, or minions executor from the branch; the PR also still carries blocking correctness and security/supply-chain review concerns.

## What This Changes

The PR adds a new SQLite-backed `src/minions` job queue with gateway startup, CLI, protocol, docs, TaskRegistry shadow-write, subagent spawn tracking, and plugin SDK surfaces for durable detached execution.

## Best Possible Solution

Keep Tasks and Task Flow as the canonical core ledger/orchestration model, retain the landed detached-runtime registration seam, and rework the durable queue as a smaller executor implementation behind that contract. Split unrelated install-security scanner behavior into its own narrowly tested PR.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the patch-level defects: the PR diff and review thread identify deterministic state-machine paths for duplicate respawn and stuck parent jobs; the broader durability feature itself is a design change rather than a single current-main bug reproduction.

## Solution Assessment

Is this the best way to solve the issue? No. The maintainable direction is the landed detached-runtime contract plus a narrower durable executor behind it, not a second core `minions` substrate with separate CLI/protocol/docs/state beside Tasks and Task Flow.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Keep tracked spawns non-expiring until lifecycle updates them:** `src/agents/tools/sessions-spawn-tool.ts:357`
  - body: The tracking row is inserted as an `active` job with a 30s lock, but the original live subagent run does not renew that lock or complete the minion row. Once `lock_until` expires, normal long-running or already-finished spawns can be requeued as stalled and launched again without a gateway crash.
  - confidence: 0.87
- **[P1] Resolve parents for ignored child failures:** `src/minions/queue.ts:827-848`
  - body: `applyChildFailPolicy` only resolves parents for the `remove_dep` policy. For `ignore` or `continue`, a parent whose remaining children all fail or die can stay in `waiting-children` forever because no later completion calls `resolveParentIfDone`.
  - confidence: 0.9
- **[P2] Tighten workspace symlink trust in the scanner:** `src/plugins/install-security-scan.runtime.ts:160-193`
  - body: The install-security scan change allows outside-install-root symlink targets when they sit under sibling `extensions`, `packages`, or `ui` workspace paths. That weakens the blocked-dependency scan for plugin installs and should be split or narrowed to exact trusted OpenClaw peer links with focused tests.
  - confidence: 0.78
