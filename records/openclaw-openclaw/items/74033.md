## Summary

Keep open. Current main still performs the expensive external CLI credential reads before checking whether a stored profile can use them, so this PR is not implemented on main. The proposed one-file fix is relevant, but it likely needs revision because current tests intentionally assert missing-profile bootstrap behavior for Codex/Claude CLI and the PR adds no regression test updates. It also overlaps with the broader open external-CLI scoping work in #73952 and the linked regression #74006, so maintainer review should reconcile the intended product behavior before landing.

## Best Possible Solution

Keep this PR open for maintainer review, but request changes before landing. The best fix is a narrowly tested gate that avoids external CLI credential reads when no configured or stored auth path can use them, while either preserving the intentional first-time bootstrap contract or explicitly changing it with tests/docs. Reconcile the approach with #73952 so there is one coherent provider-scope/opt-out story for #74006 and related external CLI sync reports.
