## Summary

Keep open. Current main still keeps `deriveSessionTitle` in `src/gateway/session-utils.ts`, still derives titles as `displayName -> subject -> firstUserMessage -> sessionId`, and still lacks the dedicated `session-title.ts` module plus explicit `entry.label` precedence from this PR. The PR remains useful, but its `origin.label` precedence is broader than the heartbeat/user-label fix and overlaps with the still-open TUI session-title work in #39722 and #39790.

## Best Possible Solution

Keep this PR open for revision or maintainer review. A conservative implementation would keep the dedicated `session-title.ts` extraction and consolidated tests, add explicit `entry.label` precedence ahead of transcript-derived titles for the heartbeat/user-label case, and either leave `origin.label` behind `firstUserMessage` or coordinate that broader behavior change with #39722 and #39790 across the Gateway `sessions.list` and TUI `/sessions` surfaces.
