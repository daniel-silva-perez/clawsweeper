## Summary

Keep open. Current main still does not implement the PR's central session-title extraction or explicit label precedence, and the submitted PR still needs maintainer judgment because its `origin.label` precedence can change TUI titles for unlabeled direct-message sessions.

## Best Possible Solution

Keep this PR open for revision or maintainer review. A conservative path is to preserve the dedicated `session-title.ts` extraction and tests, add explicit `entry.label` precedence ahead of transcript-derived titles for the heartbeat/user-label case, and keep automatic `origin.label` behind `firstUserMessage` unless maintainers intentionally want that broader TUI behavior change. Coordinate with #39722 and #39790 so metadata stripping, external display names, and title precedence stay consistent.
