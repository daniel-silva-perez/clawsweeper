## Summary

Keep open. Current main still lacks the dedicated `session-title.ts` module/test file and still derives titles from `displayName`, `subject`, then `firstUserMessage` without checking explicit session labels. The PR remains relevant, but its submitted `origin.label` precedence is a functional review risk for TUI session titles and overlaps open TUI title work in #39722/#39790.

## Best Possible Solution

Keep this PR open and request revision before merge: preserve explicit `entry.label` ahead of transcript-derived titles if maintainers want the heartbeat label fix, keep automatically populated `origin.label` behind `firstUserMessage` unless product behavior intentionally changes, retain the useful test/module consolidation if desired, and coordinate with #39722/#39790.
