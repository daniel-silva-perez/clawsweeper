## Summary

Keep open. The PR addresses a real current-main gap and the author’s latest update appears to keep auto-populated origin labels behind transcript-derived titles, but the extracted helper still drops current main’s defensive normalization for persisted session title fields, so the patch can throw while listing malformed or manually edited session-store entries.

## What This Changes

The PR extracts `deriveSessionTitle` into `src/gateway/session-title.ts`, re-exports it from `session-utils`, moves the title tests into a dedicated file, and changes explicit session labels to outrank transcript-derived fallback titles.

## Best Possible Solution
