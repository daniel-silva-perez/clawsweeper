## Summary

Keep open. Current main still does not implement the PR's central title-derivation change: `deriveSessionTitle` remains in `src/gateway/session-utils.ts`, has no `entry.label` or `entry.origin?.label` precedence branch, and the dedicated `session-title.ts` / `session-title.test.ts` files are absent. The PR remains a plausible implementation candidate, but its `origin.label` precedence still needs maintainer judgment because current code automatically populates origin labels for direct chats.

## Best Possible Solution

Keep this PR open and request revision before merge: add explicit `entry.label` precedence ahead of transcript-derived titles if maintainers want the heartbeat/user-label fix, keep automatically populated `origin.label` behind `firstUserMessage` unless that product behavior is intentional, and retain the dedicated module/test consolidation if desired. Coordinate with the related TUI session-title work in #39722 so metadata stripping and title precedence stay consistent.
