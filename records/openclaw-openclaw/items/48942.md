## Summary

Keep this PR open. Current main still does not include the extracted session-title module or label-aware deriveSessionTitle behavior, and the submitted origin.label precedence has a real functional review risk for TUI session titles.

## Best Possible Solution

Keep the PR open but require revision before merge: preserve explicit entry.label ahead of transcript-derived titles if maintainers still want that heartbeat-label fix, keep auto-populated origin.label behind firstUserMessage unless product behavior intentionally changes, and coordinate tests/implementation with the open #39722/#39790 TUI title work. The extraction can still be considered if it meaningfully clarifies the helper ownership.
