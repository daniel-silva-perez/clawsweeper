## Summary

Keep open. The branch addresses the narrow explicit-label precedence problem and the earlier origin-label concern, but it still introduces a regression by replacing current defensive string normalization with direct `.trim()` calls on persisted session-store fields.

## What This Changes

The PR extracts `deriveSessionTitle` into `src/gateway/session-title.ts`, re-exports it from `session-utils`, moves title tests into `session-title.test.ts`, and changes explicit session labels to outrank transcript-derived titles while origin labels remain a fallback.

## Best Possible Solution

Keep the extraction and explicit `entry.label` precedence, keep `origin.label` behind transcript-derived titles unless the broader #39722/#39790 policy changes, and restore `normalizeOptionalString` handling for every optional title candidate with a focused malformed-store regression test.

## Reproduction Assessment

Yes. Current main can be checked by calling `deriveSessionTitle({ sessionId, updatedAt, label: "My App" }, "heartbeat")`, which still returns the transcript title; the PR regression can be checked with a persisted-style entry such as `{ displayName: 42 }`, which now throws at `.trim()` during derived-title listing.

## Solution Assessment

No as submitted. The direction is narrow and maintainable, but the extracted helper should preserve current defensive normalization instead of trusting persisted JSON field types.

## Review Findings
