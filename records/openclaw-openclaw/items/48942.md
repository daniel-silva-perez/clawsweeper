## Summary

Keep this PR open. Current main still has neither the extracted `session-title.ts` module nor the label-aware `deriveSessionTitle` behavior, but the submitted `origin.label` precedence remains a functional review risk because origin labels are auto-populated and the TUI currently renders `derivedTitle` ahead of `displayName`.

## Best Possible Solution

Keep the PR open but require revision before merge: preserve explicit `entry.label` ahead of transcript-derived titles if maintainers want the heartbeat label fix, keep auto-populated `origin.label` behind `firstUserMessage` unless product behavior intentionally changes, and coordinate the tests and implementation with the open TUI session-title work. The extraction itself is acceptable if it clarifies ownership without changing unrelated title precedence.
