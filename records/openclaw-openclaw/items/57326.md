## Summary

Keep this issue open. Current main still has the reported helper-path gaps, and the provided GitHub context shows same-author PR #72499 is open, unmerged, and explicitly targets this issue with closing syntax. The code path that works in the main runner resolves the configured CLI runtime before checking CLI-ness, but queued followups, auth probes, slug generation, and cron still do not consistently do that.

## Best Possible Solution

Keep this issue open and review, rebase, or close PR #72499. The fix should apply resolveCliRuntimeExecutionProvider before isCliProvider in queued followups, auth probes, slug generation, and cron, route CLI-backed helper runs through runCliAgent, preserve runtime/auth/session/routing context, and add focused regression coverage for those helper paths.
