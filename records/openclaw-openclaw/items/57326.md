## Summary

Keep this issue open. Current main still has direct embedded/API helper paths for queued followups, `models status --probe`, LLM slug generation, and the cron executor still checks CLI-ness before resolving a canonical provider to its configured CLI runtime. Same-author PR #72499 is also open and explicitly uses closing syntax for this issue, so it should be reviewed, rebased, merged, or rejected before this issue is closed.

## Best Possible Solution

Keep this issue open and review/rebase PR #72499, or land an equivalent scoped patch. The fix should resolve configured CLI runtimes before isCliProvider(...) in queued followups, auth probes, slug generation, and cron; route CLI-backed runs through runCliAgent(...); preserve auth/session/routing/cancellation context; and add focused regression coverage. The stale persisted agentHarnessId == CLI runtime alias behavior from the newest comment should be split into a separate hardening issue or added deliberately if maintainers want it in the same patch.
