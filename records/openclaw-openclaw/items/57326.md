## Summary

Keep this issue open. Current main still has the reported helper-path gaps, and same-author PR #72499 remains open as an unmerged implementation candidate that targets this issue.

## Best Possible Solution

Keep this issue open and review or rebase PR #72499, or land an equivalent scoped patch. The fix should apply resolveCliRuntimeExecutionProvider(...) before isCliProvider(...) in queued followups, auth probes, slug generation, and cron; route CLI-backed helper runs through runCliAgent(...); preserve runtime config, auth, session, routing, cancellation context; and add focused regression coverage for each helper path.
