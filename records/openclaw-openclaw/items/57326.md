## Summary

Keep this issue open. Current main still has the reported remaining helper-path gaps: queued followups, auth probes, and LLM slug generation call the embedded runner directly, and the cron executor still checks CLI-ness on the provider override before resolving a canonical provider to a configured CLI runtime. Same-author PR #72499 is also open and linked as an implementation candidate for this issue, so the best maintainer path is to review/rebase/land or reject that PR before closing.

## Best Possible Solution

Keep this issue open and review/rebase PR #72499, or land an equivalent scoped patch. The fix should resolve configured CLI runtimes before `isCliProvider(...)` in queued followups, auth probes, LLM slug generation, and cron; route CLI-backed runs through `runCliAgent(...)`; preserve auth/session/routing/cancellation context; and add focused regression coverage for canonical-provider plus CLI-runtime configurations. Split the stale persisted `agentHarnessId` alias hardening into a separate issue unless maintainers intentionally want it bundled.
