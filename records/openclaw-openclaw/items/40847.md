## Summary

Keep open. The PR targets a real media-understanding diagnostics gap, and current main still only warns for aggregate failed decisions, but the proposed patch is not mergeable as-is because the no-provider warning is unreachable and all-skipped zero-output image attempts remain silent.

## What This Changes

The PR adds warning logs in `src/media-understanding/runner.ts` when selected image attachments produce no media-understanding outputs, including failed attempt details and a no-provider message.

## Best Possible Solution

Implement the diagnostics at the actual runner decision points: warn before the no-provider early return, warn when selected image attempts all skip or return empty output, preserve failed-attempt context, use `logWarn`, and add focused tests without changing image routing or model selection.

## Reproduction Assessment

Yes. The control-flow reproduction is high confidence from source: no resolved entries return before the PR’s new block, and all empty or skipped attempts produce a skipped aggregate decision with no warning-level log.

## Solution Assessment

No. The requested direction is narrow and maintainable, but this patch places one warning after an early return, misses the all-skipped path, and bypasses the existing logger contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Move the no-provider warning before the early return:** `src/media-understanding/runner.ts:817-821`
  - body: When no providers resolve, `runCapability` returns at the existing `resolvedEntries.length === 0` guard, so this branch is unreachable and operators still get no warning for selected images with no image-understanding provider.
  - confidence: 0.94
- **[P2] Warn when every image attempt is skipped:** `src/media-understanding/runner.ts:801-821`
  - body: If resolved image providers all return `skipped`, `failedAttempts` is empty and `allAttempts` is non-empty, so neither warning branch runs. That leaves a zero-output path this PR is meant to diagnose silent.
  - confidence: 0.92
- **[P3] Use logWarn for the new diagnostics:** `src/media-understanding/runner.ts:811`
  - body: The new warnings call `console.warn` directly while existing media-understanding warnings use `logWarn`, which also routes messages through OpenClaw logging. Operators relying on normal gateway logs may miss these diagnostics.
  - confidence: 0.82
