## Summary

Keep open. Current main still lacks the requested selected-image zero-output diagnostics, and this PR is a useful narrow direction, but the proposed warning block misses key paths: one branch is unreachable after the no-provider early return, and all-skipped attempts still produce no warning. The author also acknowledged the unreachable no-provider branch in the review discussion.

## What This Changes

The PR adds warning logs in `src/media-understanding/runner.ts` when selected image attachments produce no media-understanding outputs, including failed-attempt details and a no-provider message.

## Best Possible Solution

Implement the diagnostics at the actual runner decision points: warn before the no-provider early return, warn when selected image attempts all skip or return empty output, preserve failed-attempt context, route warnings through `logWarn`, and add focused tests without changing successful image routing or model selection.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: no resolved image providers return before the PR's new block, and all-empty or skipped provider attempts produce a skipped aggregate decision with no warning-level log. No tests were run because this was a read-only review.

## Solution Assessment

No. The requested direction is narrow and maintainable, but this patch places one warning after an early return, misses the all-skipped path, and uses `console.warn` instead of the existing OpenClaw logger path.
