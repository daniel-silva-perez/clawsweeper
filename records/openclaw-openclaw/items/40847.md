## Summary

Keep this PR open. Current main still does not implement the central diagnostic behavior for selected image attachments that produce zero outputs, and the PR diff as provided still has the unresolved no-provider and all-skipped gaps called out in review.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up. The best fix is to revise src/media-understanding/runner.ts so selected-image zero-output diagnostics use logWarn, emit the no-provider warning before the early return, cover all-skipped attempts, include sanitized attempted provider/model/reason context, and add targeted Vitest coverage for failed, no-provider, and all-skipped selected-image paths.
