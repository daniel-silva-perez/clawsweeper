## Summary

Keep open. Current main still lacks warning diagnostics for selected image attachments that produce zero media-understanding outputs, so the PR is not obsolete, but the proposed patch is not ready because it misses key zero-output paths and bypasses the existing logging contract.

## What This Changes

The PR adds warning logs in `src/media-understanding/runner.ts` when selected image attachments produce no media-understanding outputs, including failed-attempt details and a no-provider message.

## Best Possible Solution

Implement the diagnostics at the actual runner decision points: warn before the no-provider early return, warn when selected image attempts all skip or return empty output, preserve failed-attempt context, use `logWarn`, and add focused tests without changing successful image routing or model selection.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: no resolved image providers return before the PR’s new block, and all-empty or skipped provider attempts produce a skipped aggregate decision with no warning-level log.

## Solution Assessment

No. The requested direction is narrow and maintainable, but this patch places one warning after an early return, misses the all-skipped path, and uses `console.warn` instead of the existing OpenClaw logger path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Move no-provider warning before returning:** `src/media-understanding/runner.ts:817-821`
  - body: When no providers resolve, `runCapability` returns at the existing `resolvedEntries.length === 0` guard before this branch can run, so selected images with no image-understanding provider still get no warning.
  - confidence: 0.94
- **[P2] Warn when every image attempt is skipped:** `src/media-understanding/runner.ts:801-821`
  - body: If resolved providers all return empty output or skip errors, `failedAttempts` is empty and `allAttempts` is non-empty, so neither warning branch fires. That leaves a zero-output path this PR is meant to diagnose silent.
  - confidence: 0.92
- **[P3] Route diagnostics through logWarn:** `src/media-understanding/runner.ts:811`
  - body: The new diagnostics call `console.warn` directly while existing media-understanding warnings use `logWarn`, which also reaches OpenClaw logging. Operators relying on normal gateway logs may miss these messages.
  - confidence: 0.82
