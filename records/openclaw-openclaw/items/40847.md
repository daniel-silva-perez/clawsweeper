## Summary

Keep open. The diagnostic goal is still valid, but the branch is not ready: the no-provider warning is unreachable, and selected-image attempts that all end as skipped still produce no warning. Current main only warns failed media-understanding decisions, so the remaining behavior is not already implemented.

## What This Changes

The branch adds warning logs in `src/media-understanding/runner.ts` when image media-understanding selects attachments but produces zero outputs.

## Best Possible Solution

Implement the diagnostic in the runner control flow itself: warn before the no-provider early return, warn when all selected-image attempts are skipped, preserve the existing failed-attempt warning path with provider/model/reason context, route warnings through `logWarn`, and add focused Vitest coverage for failed, no-provider, and all-skipped selected-image zero-output cases.
