## Summary

Keep this PR open. Current main has a generic warning only for failed media-understanding decisions, while selected-image zero-output paths that resolve no providers or only skipped attempts still do not warn. The PR is narrowly scoped, but its current diff has the same unresolved logic gaps identified in review, so it is not obsolete and also is not ready to close as implemented.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up. The best path is to revise src/media-understanding/runner.ts to use the existing logWarn logging path, move any no-provider diagnostic before the early return, decide and implement warning behavior for all-skipped selected-image attempts, include attempted provider/model/reason context without leaking secrets, and add targeted Vitest coverage for failed, no-provider, and all-skipped selected-image zero-output cases.
