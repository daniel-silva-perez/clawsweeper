## Summary

Keep open. Current main still has Auto-TTS off by default and compaction does not directly flip `ttsAuto`, but the normal agent tool list still exposes the `tts` tool and its model-facing description does not restrict use to explicit user requests. That leaves a narrow, plausible current-main path for unsolicited dashboard audio replies in long or compacted conversations.

## Best Possible Solution

Keep this issue open and land a narrow hardening PR: update the agent-facing `tts` tool guidance, and matching test/docs or prompt surface if needed, so the assistant only calls `tts` when the user explicitly requests speech/audio/voice or an intentional TTS workflow requires conversion. Do not change compaction, disable existing TTS commands, or alter configured Auto-TTS behavior.
