## Summary

Keep open. Current main still appends Codex ACP `-c model=...` and `-c model_reasoning_effort=...` startup args without checking whether the configured adapter command already contains those keys, and the latest release tag has the same helper. Open PR #74339 explicitly references this issue as its fix, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Fix the ACPX Codex path so configured model/thinking overrides either work with Codex ACP or fail with a specific validation error instead of `AcpRuntimeError: Internal error`.

## Best Possible Solution

Land a narrow ACPX runtime fix that preserves configured Codex ACP adapter args while skipping runtime model/thinking `-c` keys already present, with coverage for full overlap, partial overlap, and wrapper-command variants; then close this issue after the fix reaches main or a shipped release.

## Reproduction Assessment

Yes. The issue gives concrete operator steps, and current main statically reproduces the risky construction path: configured Codex ACP args are preserved, then runtime model/thinking can append the same config keys before session startup.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the ACPX Codex command-normalization/runtime override path rather than core `sessions_spawn`, because `sessions_spawn` is correctly forwarding generic ACP runtime options.
