## What This Changes

The issue reports that ACPX Codex worker sessions fail with `AcpRuntimeError: Internal error` when the configured Codex ACP command includes model and reasoning-effort `-c` overrides.

## Best Possible Solution

Review and either land or supersede PR #74339 with focused coverage for configured Codex ACP `-c model` and `-c model_reasoning_effort` args, including the runtime override path; close this issue only after that fix is merged or a live current-main smoke proves the configured-command path is already healthy.

## Reproduction Assessment

Yes for the reported release: the issue gives concrete v2026.4.26 steps, config args, runtime call shape, logs, and the reporter isolated the failure to the configured `-c` overrides. Current-main live reproduction remains unverified in this read-only review.

## Solution Assessment

Unclear until PR #74339 is reviewed, but its direction is narrow and plausible: avoid appending duplicate Codex ACP config keys while preserving configured adapter args. A live ACPX Codex smoke or equivalent targeted tests should be the acceptance bar.
