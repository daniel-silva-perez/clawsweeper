## Summary

Keep open. Current main still has a plausible duplicate Codex ACP `-c model` / `-c model_reasoning_effort` append path, the latest tagged release lacks a duplicate-key guard, and open PR #74339 is explicitly paired to this issue with closing syntax, so this should remain open until that PR or an equivalent fix is reviewed and merged.

## What This Changes

The report asks OpenClaw to make ACPX Codex worker runs succeed, or fail with a specific validation error, when configured Codex ACP commands include model and reasoning-effort overrides.

## Best Possible Solution

The desired end state is a focused ACPX runtime fix that preserves configured Codex ACP adapter args, avoids duplicating model/thinking config keys when runtime overrides overlap, covers full and partial overlap in tests, and closes this issue only after the fix lands or a live current-main smoke proves the configured-command path is fixed.

## Reproduction Assessment

Yes. The issue provides concrete operator steps and current main statically reproduces the risky path: configured Codex ACP `-c` args are preserved, then runtime model/thinking overrides can append another pair before session startup; the only missing proof is a live Codex-auth smoke on current main.

## Solution Assessment

Yes, the narrowest maintainable solution belongs in the ACPX Codex command-normalization/runtime override path rather than core `sessions_spawn`. PR #74339’s direction of checking existing Codex ACP `-c` config keys before appending runtime overrides matches that boundary, subject to normal review and validation.
