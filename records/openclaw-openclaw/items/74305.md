## What This Changes

The issue reports that ACPX Codex worker runs fail with an internal ACP runtime error when Codex model and reasoning-effort overrides are configured.

## Best Possible Solution

Land a focused ACPX runtime fix that preserves configured Codex ACP adapter args while skipping runtime model/thinking -c keys that are already present, with tests for full and partial overlap; then close this issue as implemented after the fix reaches main or a shipped release.

## Reproduction Assessment

Yes. The issue gives concrete operator steps, and current main statically reproduces the risky path: configured Codex ACP -c args are preserved, then sessions_spawn model/thinking runtime options can append another pair before session startup.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the ACPX Codex command-normalization/runtime override path, not in core sessions_spawn, because sessions_spawn correctly forwards generic ACP runtime options while the Codex ACP command builder owns startup config args.
