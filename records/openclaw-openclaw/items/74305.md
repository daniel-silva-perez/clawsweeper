## Summary

Keep open. The report is recent, actionable, and now has an open implementation PR that targets the same ACPX Codex override path. Current main has newer Codex ACP plumbing than v2026.4.26, but it still appends Codex ACP `-c` model/thinking overrides without detecting keys already present in the configured command, so the reported surface is not ready for cleanup closure before PR review or live proof.

## What This Changes

ACPX Codex worker runs fail with `AcpRuntimeError: Internal error` when the configured Codex ACP command includes `model` and `model_reasoning_effort` overrides.

## Best Possible Solution

Land or replace a narrow ACPX runtime fix that preserves configured Codex ACP adapter args while avoiding duplicate `model` and `model_reasoning_effort` `-c` keys when runtime overrides are also supplied, with regression coverage for configured-only, partial-overlap, and runtime-override cases plus a live smoke before shipping.
