## Summary

Keep this PR open. Current main has absorbed part of the CJK estimator work into the active pi-hooks context-pruning path, but it has not fully implemented this PR's stated behavior: soft-trim eligibility and assistant tool-call argument estimation still use raw UTF-16 length, and this PR still targets the removed pi-extensions path. The PR needs a rebase or replacement patch, not conservative closure.

## Best Possible Solution

Keep this PR open as useful prior work, or replace it with a maintainer patch rebased onto src/agents/pi-hooks/context-pruning/pruner.ts. The best fix should keep using the shared src/utils/cjk-chars.ts estimator, apply weighted estimation to assistant tool-call argument JSON, make soft-trim eligibility and head/tail extraction use consistent units, and add focused regressions for CJK-heavy tool results and tool-call arguments. After a replacement lands, close this stale old-path PR as superseded or implemented by the replacement.
