## Summary

Keep open. Current main contains part of the CJK estimation work, but this PR targets the removed legacy pruner path while the active pruner still has raw-length gaps for CJK soft-trim eligibility and serialized tool-call arguments.

## What This Changes

The PR branch adds a CJK/non-Latin weighted string estimator to the legacy `pi-extensions` context pruner, applies it across message and tool-result estimates, and adds estimator unit coverage.

## Best Possible Solution

Port the useful CJK-aware behavior to `src/agents/pi-hooks/context-pruning`, reuse `src/utils/cjk-chars.ts` for every text-bearing estimate including serialized tool-call arguments, align soft-trim budgeting and extraction semantics, and add focused regressions for CJK tool results, CJK tool-call arguments, CJK Extension B, and ASCII behavior.

## Reproduction Assessment

Yes. A source-level reproduction is a CJK-heavy prunable `toolResult` whose raw `p.length` is below `softTrim.maxChars` but whose weighted estimate is above it, or an assistant `toolCall` with CJK arguments; current main still uses raw length in both active-pruner paths.

## Solution Assessment

No. The PR direction is useful, but this patch is not the best way to solve the issue because it edits an obsolete runtime path and its weighted soft-trim eligibility change is not paired with weighted extraction semantics.
