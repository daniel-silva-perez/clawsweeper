## Summary

Keep this PR open. It is useful source work, but current main has only partially absorbed the CJK estimator work and the proposed patch is now stale against the active pruner path; closing or merging it as-is would leave CJK-heavy tool-result soft trimming and assistant tool-call argument estimation undercounted.

## What This Changes

This PR adds a CJK-aware string-length estimator to the old pi-extensions context pruner, replaces several token-estimation `.length` calls with it, and adds estimator unit tests.

## Best Possible Solution

Replace or rebase this work onto the active pi-hooks context-pruning implementation, using the shared CJK estimator consistently for message-size estimation and keeping soft-trim thresholding, slicing, and user-facing trim notes in one unit system with focused regressions for CJK tool results and tool-call arguments.

Full review comments:

- **[P1] Use one unit for soft-trim measuring and slicing:** `src/agents/pi-extensions/context-pruning/pruner.ts:55`
