## Summary

Keep open: this PR is useful source work, but it is not mergeable as-is. Current main has moved context pruning to the pi-hooks implementation and only partially covers the requested CJK-aware estimation, while the PR patch also introduces a soft-trim unit mismatch in the old path.

## What This Changes

This PR adds a CJK-aware string-length estimator to the old pi-extensions context pruner, applies it to message and tool-call token estimation paths, and adds estimator unit tests.

## Best Possible Solution

Port the useful parts to the active pi-hooks context-pruning implementation, using the shared CJK estimator for serialized tool-call arguments and designing CJK-aware soft trimming so eligibility checks, slice budgets, and trim notes all use a consistent unit system with focused regressions.

- **[P1] Use one unit for soft-trim budgets:** `src/agents/pi-extensions/context-pruning/pruner.ts:55`
