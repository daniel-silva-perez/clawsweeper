## Summary

Keep open. Current main has partial CJK-aware estimation in the active context pruner, but this PR still cannot be merged as-is because it edits the obsolete `pi-extensions` path and the active `pi-hooks` pruner still has raw-length gaps for soft-trim eligibility and serialized assistant tool-call arguments.

## What This Changes

The PR branch adds a local CJK/non-Latin weighted string-length helper to the old `pi-extensions` context-pruning pruner, applies it to several estimation paths, and adds estimator tests for ASCII, CJK, mixed text, empty strings, and supplementary-plane CJK.

## Best Possible Solution

Port the behavior into the active `src/agents/pi-hooks/context-pruning` pruner, reuse the shared CJK estimator for all text-bearing size estimates, make soft-trim eligibility and extraction budgets coherent, and add focused regressions for CJK soft trimming and serialized assistant tool-call arguments.
