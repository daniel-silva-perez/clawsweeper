## Summary

Keep open. Current main partially incorporates CJK-aware estimation in the active context pruner, but this PR cannot be merged as-is because it targets the obsolete `pi-extensions` path, while the active `pi-hooks` pruner still has raw-length gaps in soft-trim eligibility and serialized tool-call argument sizing.

## What This Changes

The branch adds weighted CJK/non-Latin string-length estimation to the old `pi-extensions` context-pruning pruner and expands tests for ASCII, CJK, mixed text, empty strings, and surrogate-pair CJK characters.

## Best Possible Solution

Port the behavior into the active `src/agents/pi-hooks/context-pruning` pruner, use the shared CJK estimator consistently for all text-bearing size estimates, keep soft-trim eligibility and extraction budgets coherent, and add focused regressions for CJK soft trimming and serialized tool-call arguments.
