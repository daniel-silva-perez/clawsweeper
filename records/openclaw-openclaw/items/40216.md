## Summary

Keep this PR open. Current main has absorbed the shared CJK estimator and applies it to part of the active context-pruning path, but it has not fully implemented this PR's stated coverage: soft-trim eligibility still uses raw UTF-16 length, assistant tool-call arguments still use raw JSON string length, and the PR modifies the removed pi-extensions path instead of the active pi-hooks implementation.

## Best Possible Solution

Keep this PR open as useful prior work, or replace it with a rebased maintainer patch against src/agents/pi-hooks/context-pruning/pruner.ts. The best fix should use the shared src/utils/cjk-chars.ts estimator for assistant tool-call argument JSON, make soft-trim eligibility and head/tail extraction use consistent units, and add focused regressions for CJK-heavy tool results and assistant tool-call arguments. After that lands, this stale old-path PR can be closed as superseded or implemented.
