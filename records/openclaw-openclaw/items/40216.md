## Summary

Keep open. The PR contains useful CJK token-estimation work, but it should not merge as-is: current main uses the active `pi-hooks` context-pruning path, the PR targets the old `pi-extensions` path, and the latest patch still mixes weighted CJK units with UTF-16 soft-trim slicing.

## What This Changes

This PR adds a CJK-aware string-length estimator to the old pi-extensions context pruner, applies it to message/tool-call/token-estimation paths including soft-trim length checks, and adds estimator tests for ASCII, CJK, mixed text, empty strings, and surrogate pairs.

## Best Possible Solution

Port the useful behavior to `src/agents/pi-hooks/context-pruning/pruner.ts`, use the shared CJK estimator for serialized tool-call arguments, and make CJK-aware soft trimming internally consistent with focused regressions in the active pi-hooks tests.
