## Summary

Keep open. Current main has the shared CJK estimator and partially uses it in the active context pruner, but the PR is not obsolete or merge-ready: it targets the removed `pi-extensions` path while current runtime loads `pi-hooks`, and the remaining active path still has CJK estimation gaps for soft-trim eligibility and assistant tool-call arguments.

## What This Changes

The PR adds weighted CJK/non-Latin character estimation to context-pruning token estimates and tests it against ASCII, CJK, mixed, empty, and surrogate-pair inputs.

## Best Possible Solution

Port the useful behavior into `src/agents/pi-hooks/context-pruning/pruner.ts`, use the shared estimator consistently for CJK-heavy text and serialized tool-call arguments, make soft trimming compare and slice in one non-overlapping budget unit, and add active-path regressions before closing this source PR as superseded or implemented.
