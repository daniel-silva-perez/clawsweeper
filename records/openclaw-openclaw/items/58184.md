## Summary

Keep open. Current `main` still emits the broad Control UI CSP `connect-src 'self' ws: wss:` directive, so the PR is not obsolete or implemented. The PR also needs revision before merge: the provided current head keeps a blanket `wss:` source despite the title/body saying both blanket WebSocket schemes are dropped, and it risks blocking documented `ws://` remote-gateway workflows.

## Best Possible Solution

Keep the PR open for maintainer review and ask for a revised CSP design that blocks unintended WebSocket exfiltration without breaking supported `gatewayUrl` workflows. A good path would align docs, runtime behavior, and tests around explicit allowed gateway origins or a same-origin proxy path, with regression coverage for same-origin, documented `wss://`, and documented `ws://` remote gateway cases.
