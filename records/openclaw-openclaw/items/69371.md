## Summary

Keep open. Current main has adjacent ARM/low-power improvements, notably the 15s configurable gateway handshake and doctor startup-optimization hints, plus some cron retry/backoff behavior. The central request is still not implemented as a whole: sessions/status do not expose a lightweight ARM mode, compaction lacks the requested context high-water/interval config, `openclaw message send` has no background retry flags, `sessions_send` returns run status rather than a delivery receipt, and there is no `doctor --edge-device` mode.

## Best Possible Solution

Keep this issue open, but split it before implementation. The best maintainer path is to treat #46097 as the already-addressed handshake slice, then create focused follow-ups or PRs for: a lightweight bounded `sessions`/`status` mode for large stores, explicit context-pressure compaction policy if desired, cron retry semantics for failed agent/tool runs, durable/background message delivery with idempotency rules, `sessions_send` receipt semantics, and an ARM edge-device doctor profile.
