The PR is small, on-topic, and includes a targeted regression test for a real transcript-debug shadowing case; the related PR #68837 is a separate Active Memory config-normalization fix and does not supersede it.

## Best Possible Solution

Keep this PR open and review or land it, or land an equivalent patch that removes the `?? {}` fallback in `readActiveMemorySearchDebug` and keeps the regression test proving a newer payload-less `memory_search` result does not shadow older real debug data.
