Current `main` still matches the reported literal `session.store` failure mode, and a linked closing PR is open rather than merged. The issue should remain open until #54716, or an equivalent fix, lands.

## What This Changes

The issue reports that `sessions_list` and the Control UI sessions dashboard return only the main session when multiple sessions are active.

## Best Possible Solution

Review and land #54716, or an equivalent fix, so literal per-agent paths under `agents/<id>/sessions/sessions.json` use all-agent discovery while true shared literal stores keep their current single-store behavior. The fix should include regression coverage for the literal `agents/main/sessions/sessions.json` configuration.
