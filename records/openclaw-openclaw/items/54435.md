## Summary

Keep this issue open. Current main still has the literal `session.store` control-flow path that loads only one store and returns before all-agent discovery, and the linked implementation candidate #54716 is still open/unmerged in the provided GitHub context.

## Best Possible Solution

Keep this issue open while #54716 is reviewed and either landed or replaced. The fix should update the combined gateway session-store loader so literal paths that are structurally under `agents/<id>/sessions/sessions.json` discover sibling agent stores, while preserving true shared literal-store behavior, and add regression coverage for that literal per-agent path.
