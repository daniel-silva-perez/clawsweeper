## Summary

Keep open. Current main still reads a literal non-template `session.store` as one default-agent store and returns before all-agent discovery, while `sessions.list` and the Control UI depend on that combined store. #54716 is an open closing PR with reported before/after API proof, so this issue should remain open until that or an equivalent fix lands.

## What This Changes

The issue requests that `sessions.list` and the Control UI Sessions dashboard return all active agent/channel sessions instead of only the main session when multiple sessions exist.

## Best Possible Solution

Review and land #54716, or an equivalent current-main fix in `src/config/sessions/combined-store-gateway.ts`, so literal paths matching `agents/<id>/sessions/sessions.json` use all-agent discovery while true shared literal stores keep single-store semantics. Add regression coverage for a literal `agents/main/sessions/sessions.json` configuration.
