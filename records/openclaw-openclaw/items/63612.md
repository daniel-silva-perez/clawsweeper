## Summary

Keep this issue open. Current main still routes pre-prompt compaction estimation through the upstream token estimator without a local malformed-message guard, and the provided GitHub context shows the direct implementation PR #63636 is still open and unmerged with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and have maintainers review, land, or supersede PR #63636. The durable fix should add a shared safe token-estimation path or normalize malformed replay history before every pre-prompt/compaction estimation call, with regression tests that exercise the real failure shapes: missing assistant content arrays, missing text/thinking/name fields, missing or non-array toolResult/custom content, and missing bash/summary fields.
