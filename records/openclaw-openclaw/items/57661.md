## Summary

Keep open. This is a recent sessions_send timeout report with an independent confirming comment, and current main still does not clearly fix the reported one-to-many A2A timeout or non-existent target behavior. The related PR #64622 is still open/unmerged, so it is an implementation candidate rather than shipped code.

## Best Possible Solution

Keep this issue open as the user-facing bug report. Maintainers should review, revise, or replace #64622, with regression coverage for sessions_send to missing key-shaped targets and one-to-many A2A sends, then implement scoped target-resolution and timeout hardening in src/agents/tools/sessions-send-tool.ts, src/agents/tools/sessions-resolution.ts, and the gateway agent/session persistence path as needed.
