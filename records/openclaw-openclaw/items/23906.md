## Summary

Keep this issue open. Current main still does not expose a per-call `sessions_spawn` tool selector or tool-less mode, even though lower-level embedded-run code already has `disableTools` and `toolsAllow`. The issue discussion also adds plugin/MCP inheritance and local-model no-tools requirements that are not fully covered by the existing static subagent config policy or by the related open issues.

## Best Possible Solution

Keep this issue open and let maintainers pick the canonical API across this issue, #15032, and #62695. The eventual fix should add a backward-compatible `sessions_spawn` tool policy field, validate and document it, carry it through `SpawnSubagentParams` and gateway agent RPC, map tool-less spawns to embedded `disableTools:true`, map subsets to `toolsAllow`, and cover core tools, plugin/MCP tools, config precedence, sandbox/security behavior, and local-model no-tools runs.
