## Summary

Keep open. Current main still does not expose a per-spawn tool selector or tool-less mode on `sessions_spawn`; the public schema, native `SpawnSubagentParams`, and closed gateway `agent` RPC schema have no `tools`, `toolsAllow`, `disableTools`, or deny-list field. Lower-level embedded-run support exists, but it is not plumbed through the spawn API, and the issue discussion adds unresolved plugin/MCP tool-resolution semantics that need maintainer API judgment across #15032 and #62695.

## Best Possible Solution

Keep this open and have maintainers choose the canonical per-spawn tool-policy API across this issue, #15032, and #62695. A complete fix should add a backward-compatible `sessions_spawn` field, validate and document it, carry it through `SpawnSubagentParams` and the gateway `agent` RPC, map tool-less runs to embedded `disableTools: true`, map subsets to `toolsAllow`, and define plugin/MCP tool resolution plus precedence against existing policy layers.
