## Summary

Keep open. Current main still has no per-spawn `sessions_spawn` tool selector or tool-less mode: the public tool schema, native `SpawnSubagentParams`, and gateway `agent` RPC schema do not carry `tools`, `toolsAllow`, or `disableTools`. Lower-level embedded-run primitives exist, but they are not exposed through the spawn API, and the issue discussion adds unresolved plugin/MCP resolution semantics.

## Best Possible Solution

Keep this open and let maintainers choose the canonical per-spawn tool-policy API across this issue, #15032, and #62695. The implementation should add a backward-compatible `sessions_spawn` field, validate and document it, carry it through `SpawnSubagentParams` and the gateway `agent` RPC, map tool-less runs to embedded `disableTools:true`, map subsets to `toolsAllow`, and define plugin/MCP resolution plus precedence against existing tool policies.
