## Summary

Keep this PR open. Current main partially reduces the user-visible problem by defaulting group/channel turns to message-tool-only visible replies, but it has not implemented this PR's intended text-vs-attachment guidance for the still-supported legacy automatic reply mode. In that path, current main still tells the agent not to use the message tool for the same group, so the media/attachment contradiction can still occur.

## Best Possible Solution

Keep this PR open for a narrow maintainer rebase. The best path is to preserve current `message_tool_only` behavior, update only the `automatic` group/channel prompt so text replies remain direct while files/images/attachments may use `message(action=send)` targeting the same group, and refresh the prompt tests around the current untrusted-metadata structure.

## Risks / Open Questions

- Applying the PR as-is would be stale against current main: it needs to preserve the new `message_tool_only` branch and update only the remaining `automatic` branch.
- The rebase should preserve the trusted/untrusted prompt split; group subjects, members, and other human-controlled metadata should stay in untrusted inbound context, not in the trusted group prompt.
- The functional/security-sensitive surface is prompt policy, not supply chain: the final wording should allow same-group message sends for files/images/attachments without encouraging unnecessary visible text sends in automatic mode.
