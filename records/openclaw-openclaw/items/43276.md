## Summary

Keep this PR open. Current main partially addresses the user-visible default path by making group/channel replies message-tool-only by default, but it has not implemented this PR's narrower text-vs-attachment prompt fix for the still-supported legacy `automatic` group reply mode. In that configured path, the prompt still tells the agent not to use the message tool for the same group, so the attachment/media conflict this PR targets can still occur.

## Best Possible Solution

Keep this PR open for a narrow maintainer rebase. The best path is to preserve the current `message_tool_only` prompt, update only the `automatic` branch so text replies remain direct while files/images/attachments may use `message(action=send)` targeting the same group, and adjust the existing prompt tests without weakening the current untrusted-metadata structure.
