## Summary

Keep this PR open. Current main partially reduced the user-visible failure by defaulting group/channel turns to message-tool-only visible replies, but it has not fully implemented this PR’s intended fix: the supported legacy/native `automatic` path still injects the blanket same-group message-tool prohibition that blocks attachment/file/image sends in the prompt.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is a narrow rebase that preserves the current `message_tool_only` default, updates the remaining `automatic` group/channel prompt to direct text replies normally while permitting `message(action=send)` for files/images/attachments to the same group, and updates the current prompt tests around that branch.

## Risks / Open Questions

- Applying the PR as-is would be stale: current main now has a `message_tool_only` branch and tests/docs around `visibleReplies`, so the fix should target only the remaining automatic/native prompt path.
