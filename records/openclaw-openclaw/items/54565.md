## Summary

Keep open. Current main still does not accept or apply group-scoped `debounceMs` for WhatsApp: the config type and strict schema only allow debounce at channel/account scope, and the runtime passes one scalar debounce window into the listener. The reported image/image/text case also remains unsolved because WhatsApp media messages currently bypass the debounce path.

## What This Changes

Request per-group `debounceMs` configuration for WhatsApp group entries, with possible parity for other group-capable channels.

## Best Possible Solution

Implement the WhatsApp plugin path first: accept `groups.<jid>.debounceMs`, resolve exact group before wildcard before account/channel/global fallback, and ensure the batch preserves the earlier media context when a later text message triggers the turn. Treat other channels as deliberate plugin-specific follow-up work or a separate shared-contract design.
