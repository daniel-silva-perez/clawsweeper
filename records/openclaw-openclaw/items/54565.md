## Summary

Keep open. Current main still only supports WhatsApp inbound debounce at global/channel/account scope, not per WhatsApp group. The strict group schema, generated config metadata, docs, runtime debounce resolution, and listener wiring all show that group-specific `debounceMs` is not implemented, and the reported image/image/text use case remains unresolved because media messages are explicitly excluded from debounce batching.

## Best Possible Solution

Implement the WhatsApp case in the WhatsApp plugin first: add `debounceMs?: number` to `WhatsAppGroupConfig`, the strict Zod schema, generated metadata/help/UI/docs, and focused regression tests. At runtime, resolve exact group then wildcard group debounce per inbound group message and pass it through `createInboundDebouncer.resolveDebounceMs`, falling back to account/channel/global debounce. Explicitly test the reported image/image/text sequence and then track Telegram, Discord, and other channels as plugin-specific follow-ups unless maintainers choose a shared public contract.
