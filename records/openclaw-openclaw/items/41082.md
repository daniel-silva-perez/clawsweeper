## Summary

Keep this issue open. Current main has useful compaction safeguards and a CLI-specific latest-prompt wrapper, but the default embedded/legacy session path still assembles the compaction summary plus post-compaction history without a generic guarantee that a fresh unrelated user prompt is treated as the active task over stale summarized context.

## Best Possible Solution

Add a focused embedded/default-session regression that builds a transcript with an old-topic compaction summary, appends a fresh unrelated user prompt after the compaction boundary, and asserts the assembled prompt or trajectory treats the latest user request as the active task while keeping the summary as historical background. If it fails, fix the generic context/prompt assembly path rather than adding channel-specific or keyword-only heuristics; update the compaction/session docs once the intended semantics are explicit.
