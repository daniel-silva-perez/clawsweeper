## Summary

Keep this issue open. Current main still has useful compaction mitigations, including CLI reseed tagging and structured compaction summaries, but I did not find a generic default embedded-session guarantee that a fresh unrelated user prompt after compaction is elevated over the old compaction summary. The existing discussion and prior ClawSweeper comment are still aligned with current main behavior.

## Best Possible Solution

Add a focused embedded/default-session regression that builds a transcript with an old-topic compaction summary, appends a fresh unrelated user prompt after the compaction boundary, and asserts the assembled prompt or trajectory treats the latest user request as the active task while keeping the summary as historical background. If it fails, fix the generic prompt/context assembly path rather than adding channel-specific or keyword-only heuristics.
