## Summary

Keep this PR open. Current main still does not implement the PR’s intended container-specific onboarding behavior: brew-only skill installers are still shown as installable, and selecting one still leads to the Homebrew recommendation path when brew is missing. The linked Docker/Homebrew bug (#14593) and alternative Linuxbrew PR (#42303) are both still open, so this PR remains a plausible implementation candidate rather than something already implemented or cleanly superseded.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide the Docker skill-install policy for #14593, then either rebase and land this container-hiding UX change with its focused tests, or explicitly supersede it with a merged Linuxbrew/Docker solution such as #42303 if maintainers choose that direction.
