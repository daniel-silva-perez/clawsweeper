## Summary

Keep this PR open. Current main still does not implement the PR's intended Docker onboarding behavior: brew-only missing skill dependencies are still offered during skills onboarding, and selecting one still leads to the Homebrew recommendation path when brew is absent. The linked Docker/Homebrew bug #14593 and alternative Linuxbrew Docker PR #42303 are still open in the provided context, so this remains a plausible implementation candidate rather than something already shipped or cleanly superseded.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide the Docker skill-install policy for #14593, then either rebase and land a focused onboarding fix that uses or consolidates the existing container detector, filters brew-only missing skills only when Linux container plus brew missing, shows the container-specific note, and keeps the normal Homebrew prompt elsewhere; or explicitly supersede it after a merged Docker/Linuxbrew solution such as #42303 provides an equivalent safe user path.
