## Summary

Keep open. Current main does not implement the requested opt-in webchat-to-last-external-channel mirroring flag; the current Control UI/WebChat routing code and tests explicitly keep webchat replies on the internal webchat surface in the relevant cases.

## Best Possible Solution

Keep the issue open and implement a focused opt-in session setting, likely `session.mirrorWebchatToLastExternalChannel`, in the session schema/types/docs and WebChat dispatch path. The implementation should default to false, mirror only after normal WebChat delivery succeeds, require an established non-internal route with `lastChannel`/`lastTo` or delivery context, ignore/log mirror failures without blocking WebChat, and add regression coverage for no-route, external-origin, webchat-loop, and failure cases. PR #52068 is useful prior art but is closed unmerged, so the best path is to revive or reimplement that work against current routing tests.
