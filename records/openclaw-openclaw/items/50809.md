## Summary

Keep this issue open. Current main has Android node SMS send/search support, but it does not implement the requested `sms.read` command, does not expose id/threadId reads, and does not register SMS as an OpenClaw messaging channel. The linked implementation PR #50808 is closed unmerged, and the remaining Android BroadcastReceiver/node-event path is not something a normal external channel plugin can provide without core/mobile support.

## Best Possible Solution

Keep this issue open and either rework the closed implementation PR into a smaller core/mobile change or split it into two reviewed pieces: an Android node `sms.read` command with id/threadId semantics, and a secure SMS channel ingress path that adds Android RECEIVE_SMS/BroadcastReceiver support, gateway `sms.received` handling, allowlist configuration, tests, and public docs. If maintainers want the channel logic outside core, first add the missing generic Android node event/read APIs needed by an external plugin.
