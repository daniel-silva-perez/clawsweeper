## Summary

Keep open. Current main still lacks both requested pieces: there is no Android `sms.read` invoke command and no Android-node SMS channel ingress path. The linked implementation PR is closed unmerged, and the remaining work crosses Android SMS permissions, private-message ingestion, agent-trigger authority, outbound SMS replies, allowlist semantics, and channel/plugin boundaries, so it needs maintainer product/security review before implementation.

## What This Changes

Request adds an Android `sms.read` command for message/thread reads and a node-mediated SMS messaging channel with allowlist-gated inbound texts and replies sent through an Android node.

## Best Possible Solution

Define a narrow, security-reviewed Android `sms.read` command contract first, then separately decide the SMS ingress product boundary and implement the approved path with Android permission UX, node command policy, gateway event handling, allowlist normalization, channel/plugin docs, and targeted tests.
