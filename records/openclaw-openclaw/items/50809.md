## Summary

Current main still does not implement the requested `sms.read` Android node command or SMS as a messaging channel. It only exposes `sms.send` and `sms.search`; the linked implementation PR #50808 is closed unmerged, so it remains useful context rather than a shipped fix.

## Best Possible Solution

Keep this issue open. The best path is a maintainer-reviewed split: first define and implement a narrow Android `sms.read` node command for id/threadId reads with policy/tests/docs, then separately design SMS channel ingress with Android receiver forwarding, gateway event handling, allowlist configuration, outbound reply behavior, docs, and labeler updates. If maintainers want SMS channel ownership outside core, first add the missing generic Android node/event/plugin seam needed to implement it without core bypasses.
