## Summary

Keep open. Current main still lacks the requested Android `sms.read` command and the Android-node-mediated SMS channel ingress path, and the linked implementation PR #50808 is closed unmerged. The request is still actionable, but broad enough and security-sensitive enough to need maintainer product/security review before implementation.

## What This Changes

Add an Android `sms.read` invoke command for SMS lookup by message ID or thread ID, plus an allowlist-gated SMS messaging channel routed through connected Android nodes.

## Best Possible Solution

Keep the issue open and split the work: define a narrow, security-reviewed `sms.read` Android node command first, then separately decide whether Android SMS ingress should be a bundled core channel, a plugin-backed channel using a new generic node-event seam, or out of scope for OpenClaw core.
