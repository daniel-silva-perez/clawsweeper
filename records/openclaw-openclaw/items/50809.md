## Summary

Keep this issue open. Current main still lacks both requested pieces: there is no Android `sms.read` invoke command and no SMS messaging channel/`sms.received` ingress path; the linked implementation PR #50808 is closed unmerged, so it remains useful context rather than a shipped fix.

## What This Changes

Add an Android `sms.read` invoke command and an allowlist-gated SMS messaging channel routed through connected Android nodes, with inbound SMS events and outbound replies via `sms.send`.

## Best Possible Solution

Keep this issue open and split the work under maintainer review: first define a narrow Android `sms.read` command with id/threadId semantics, gateway node-command policy, tests, and docs; then separately decide whether SMS channel ingress belongs in core or behind a new generic Android-node/channel plugin seam before implementing allowlist-gated inbound routing and `sms.send` replies.
