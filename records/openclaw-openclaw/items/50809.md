## Summary

Keep open. Current main still lacks the requested Android `sms.read` invoke command and the Android-node-mediated SMS channel ingress path, while the linked implementation PR is closed unmerged. The remaining work is actionable, but it crosses Android SMS permissions, gateway node-event handling, channel registration, allowlist policy, and privacy/security boundaries that need maintainer judgment before implementation.

## What This Changes

Add an Android `sms.read` invoke command for SMS lookup by message ID or thread ID, plus an allowlist-gated SMS messaging channel routed through connected Android nodes.

## Best Possible Solution

Keep the issue open and split the work: first define a narrow, security-reviewed Android `sms.read` command, then separately decide whether SMS ingress through Android nodes belongs as a bundled channel, a plugin-backed channel with a new generic node-event seam, or outside OpenClaw core.
