## Summary

Keep open. Current main still lacks both requested pieces: there is no Android `sms.read` invoke command, and there is no Android-node-mediated SMS channel ingress path. The linked implementation PR #50808 is closed unmerged, and the remaining work crosses Android SMS permissions, node command policy, gateway event ingress, channel registration, and sender allowlist/privacy decisions, so it needs maintainer product/security review before a repair lane is queued.

## What This Changes

Add an Android `sms.read` invoke command for message/thread reads and make SMS a text-only Android-node-backed messaging channel with allowlist-gated inbound routing and SMS replies.

## Best Possible Solution

Keep the issue open and split the work into maintainer-approved pieces: first define a narrow, security-reviewed Android `sms.read` command contract, then separately decide whether Android-node SMS ingress should be a bundled channel, plugin-backed channel with any needed generic node-event seam, or outside OpenClaw core.
