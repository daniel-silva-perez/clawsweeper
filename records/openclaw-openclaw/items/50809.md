## Summary

Keep open. Current main still lacks both requested surfaces: Android exposes `sms.send` and `sms.search` but no `sms.read`, and the gateway has no `sms.received` ingress path for routing SMS into agents. The linked implementation PR #50808 is closed unmerged, and this should stay with maintainers because reading private SMS plus letting inbound texts trigger agent runs and outbound replies needs product and security decisions first.

## What This Changes

Request adding an Android `sms.read` invoke command for message/thread reads plus an Android-node-mediated SMS channel with allowlist-gated inbound routing and SMS replies.

## Best Possible Solution

Keep this open and turn it into maintainer-approved slices: first define a narrow, security-reviewed Android `sms.read` command contract, then separately decide whether SMS ingress belongs as a bundled channel, an external plugin with any needed generic node-event seam, or outside OpenClaw core.
