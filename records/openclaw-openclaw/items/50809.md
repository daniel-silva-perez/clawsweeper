## Summary

Keep open. Current main still lacks the requested Android `sms.read` invoke command and the Android-node-mediated SMS channel ingress path; the linked implementation PR is closed unmerged, and the remaining work needs maintainer product/security decisions before implementation.

## What This Changes

The issue requests an Android `sms.read` node command plus an allowlist-gated SMS messaging channel that routes inbound Android SMS messages to agents and sends replies through `sms.send`.

## Best Possible Solution

Keep the issue open and turn it into a maintainer-approved design in narrow pieces: define a security-reviewed Android `sms.read` contract first, then decide whether SMS ingress belongs as a bundled channel, an external plugin using existing channel runtime APIs, or a new generic node-event seam before implementing allowlist-gated routing and replies.
