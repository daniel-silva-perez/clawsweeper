## Summary

Keep open. Current main still lacks the requested Android `sms.read` invoke command and the Android-node-mediated SMS channel ingress path; the linked implementation PR is closed unmerged, and the remaining work needs maintainer product/security decisions around SMS permissions, privacy, allowlists, and the channel/plugin boundary.

## What This Changes

Request to add an Android `sms.read` node invoke command and treat SMS as a text-only messaging channel routed through a connected Android node with sender allowlist gating.

## Best Possible Solution

Land a maintainer-approved design in narrow pieces: first a security-reviewed Android `sms.read` command contract with tests and docs, then a separately reviewed SMS ingress path that defines the node event, channel/plugin ownership, sender allowlist behavior, and outbound reply policy.
