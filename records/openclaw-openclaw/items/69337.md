## Summary

Keep open. Current main still does not implement this PR's intended TUI commentary fallback, but the proposed one-line global fallback changes a shared assistant visibility helper whose current contract intentionally hides commentary-only assistant text in gateway previews, chat history/live projection, and Control UI surfaces.

## What This Changes

This PR adds a commentary-phase fallback to `extractAssistantVisibleText` in `src/shared/chat-message-content.ts` after the `final_answer` lookup and before the legacy unphased fallback.

## Best Possible Solution

Keep this PR open for maintainer review, but do not merge the shared-helper fallback as-is. The best fix is a scoped TUI/live-stream projection change, or provider/phase normalization that turns completed visible assistant output into `final_answer`, while preserving commentary suppression in session previews, gateway live/history paths, delivery surfaces, and Control UI.
