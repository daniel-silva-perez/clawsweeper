## Summary

Keep open. Current main does not implement this PR’s intended commentary fallback, and the proposed one-file change is too broad because `extractAssistantVisibleText` is shared by gateway previews and Control UI paths that currently hide commentary-only assistant text by design.

## Best Possible Solution

Keep the PR open for maintainer review, but do not merge the current global fallback. The best direction is either a scoped TUI/live-stream projection change with regression coverage, or a provider/phase-normalization fix that turns actual completed assistant output into `final_answer` while preserving commentary suppression in history, previews, and Control UI.
