## Summary

Keep this PR open. Current main still does not implement the intended Discord media fixes: regular sends, component uploads, and native interaction replies still drop detected MIME type, and the shared media loader still normalizes non-GIF images through PNG/JPEG optimization. The same-author bug #41256 and overlapping implementation candidates #41419 and #41714 remain open, so this needs maintainer reconciliation rather than cleanup closure.

## Best Possible Solution

Keep this PR open while maintainers reconcile it with #41256, #41419, and #41714. The best fix should preserve verified media MIME type across regular Discord sends, component uploads, and native replies; add a clear shared media preservation contract for WebP/AVIF and genuinely animated formats; keep Discord media caps and remote fetch limits aligned; and land focused regression tests for MIME forwarding, WebP preservation, animated/APNG handling, and cap errors.
