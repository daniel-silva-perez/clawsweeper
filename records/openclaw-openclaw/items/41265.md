## Summary

Keep this PR open. Current main at 891c7d9f1c1f5e0f52935fadfeacf695b0ef1eb8 still does not implement the intended Discord media fix: regular sends, component uploads, and native replies still drop detected MIME type, and the shared media loader still optimizes non-GIF images through PNG/JPEG without WebP/AVIF/animated preservation. The linked bug #41256 and overlapping PR candidates #41419 and #41714 are still open, so this needs maintainer reconciliation rather than cleanup closure.

## Best Possible Solution

Keep this PR open while maintainers choose the canonical fix for #41256 and reconcile overlap with #41419 and #41714. The final implementation should preserve verified `media.contentType` in regular Discord sends, component uploads, and native interaction replies; add an explicit shared media preservation contract for WebP/AVIF and genuinely animated formats; keep Discord account media caps and remote fetch caps aligned; and add focused regression tests for MIME type forwarding, WebP preservation, animated/APNG behavior, and cap handling.
