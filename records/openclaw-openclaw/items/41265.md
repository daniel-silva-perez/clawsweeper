## Summary

Keep this PR open. Current main still does not implement the PR's intended Discord media fix: uploads are still wrapped in typeless Blobs, Discord component and interaction-reply media still do not pass content type through, and the shared web-media loader still optimizes every non-GIF image without a WebP/AVIF or animated-image preservation option. The PR is also paired with the still-open linked bug #41256 and overlaps two other open implementation attempts, so the right maintainer path is review/consolidation rather than cleanup closure.

## Best Possible Solution

Keep this PR open as an implementation candidate, but consolidate it with #41256 and the overlapping open PRs before merge. A current-main fix should pass detected content type into Discord Blob construction across regular sends, component sends, and interaction replies; add a deliberate shared media-loader/API option for preserving WebP or animated formats; keep Discord-specific caps and remote fetch limits correct; and add regression tests for content type preservation, WebP preservation, oversized preserved media, and any animated-image scope the final patch claims to support.
