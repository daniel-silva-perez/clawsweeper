## Summary

Keep open. Current main still has the exact core protocol gap reported: `AgentParamsSchema` is a closed object with no `adapterMeta`/`adapterContext` namespace and no `paperclip` escape hatch, and the gateway rejects unknown top-level `agent` params before the request reaches runtime handling. The linked Paperclip-side issue may resolve that adapter's symptom, but it does not supersede the OpenClaw protocol request.

## Best Possible Solution

Keep this issue open and implement an additive Gateway protocol change: add a namespaced `adapterMeta` or `adapterContext` record to `AgentParamsSchema`, update protocol docs and generated artifacts, add validation/server regression tests for third-party metadata, and coordinate Paperclip to use the namespace or keep stripping top-level metadata until the OpenClaw release ships.
