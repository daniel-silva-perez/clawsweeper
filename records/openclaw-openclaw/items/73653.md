## Summary

Keep this issue open. Current main still has the reported Gateway protocol gap: `AgentParamsSchema` is a closed object with no `adapterMeta`, `adapterContext`, or `paperclip` field, and the `agent` handler rejects unknown top-level params before runtime handling. The linked Paperclip-side issue is useful mitigation, but it does not supersede the OpenClaw protocol request.

## Best Possible Solution

Keep this issue open and implement an additive Gateway protocol change: add a namespaced `adapterMeta` or `adapterContext` record to `AgentParamsSchema`, update protocol docs/generated surfaces, add regression coverage for third-party metadata on `agent` requests, and coordinate Paperclip to use the namespace or keep stripping top-level metadata until the OpenClaw release ships.
