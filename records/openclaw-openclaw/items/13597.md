## Summary

Keep this issue open. Current main still does not include a dedicated AWS deployment guide or AWS hosting navigation entry. Existing VPS, Docker, Docker VM, Kubernetes, and Bedrock docs cover useful fragments, but they do not satisfy the requested EC2/ECS/Fargate/Lambda production deployment guide. The issue discussion remains concrete and actionable, including production notes from Pinchy, AWS VM experience, and an external AWS sample repository to evaluate.

## Best Possible Solution

Keep this issue open until OpenClaw ships a dedicated AWS install page, likely docs/install/aws.md linked from docs/docs.json and docs/install/index.md. The guide should make Docker Compose the primary EC2/Lightsail path, cover durable state and backups, gateway bind/reverse proxy/ALB/TLS/auth/origin settings, security groups, IAM roles/task roles, AWS Secrets Manager, logging/monitoring, troubleshooting, performance and hardening, ECS/Fargate task/service/networking, Lambda applicability or non-support, multi-agent topology, and evaluate or link the AWS sample repository from the discussion after review.
