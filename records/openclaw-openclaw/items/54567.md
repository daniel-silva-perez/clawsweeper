## Summary

Keep open. Current main still runs `service.install()` before `ensureSystemdUserLingerNonInteractive()` in the non-interactive daemon install path, and Linux service install still activates the unit with `systemctl --user enable` inside `installSystemdService`. The PR’s intended one-call reorder is not implemented on main, and the linked #54517 reproduction remains relevant even though that issue is closed.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land the focused reorder, preferably with a narrow regression test asserting `ensureSystemdUserLingerNonInteractive` runs before `service.install()` in `installGatewayDaemonNonInteractive`, or with explicit live headless Ubuntu verification if maintainers decide the source-level ordering is sufficient.
