## Summary

Keep this PR open. Current main does not implement the intended linger-before-install behavior: the non-interactive daemon install still calls `service.install()` before `ensureSystemdUserLingerNonInteractive()`, and the Linux service install path still reaches `systemctl --user enable` during install. The PR is not obsolete, though it should be rebased/reviewed against the current preflight that checks systemd user availability before any linger attempt.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto current main, make sure linger is attempted early enough for headless Linux installs before any `systemctl --user enable` path or premature unavailable-systemd skip, and add a focused test that locks the ordering or the headless preflight behavior.
