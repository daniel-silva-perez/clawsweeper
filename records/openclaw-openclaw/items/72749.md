## Summary

Keep open. Current main still has the noisy full inbound metadata behavior that the linked reports describe, and this PR is a useful implementation candidate, but the current PR head has a blocking correctness regression: compact direct prompts remove the metadata that the display sanitizer uses to recover sender labels for chat history.

## What This Changes

The branch adds optional `messages.inboundMetadataMode: "compact-direct"` config/schema/docs/tests to compact direct inbound prompt metadata, adds stripping support for the compact line, and adjusts plugin CLI loading for `wiki`/`memory` commands to include the configured memory-slot plugin.

## Best Possible Solution

Land a revised compact metadata implementation that reduces Telegram DM prompt noise while preserving display/history sender attribution, keeps default and group/rich-context behavior unchanged, and separates the plugin CLI loader behavior if maintainers do not want it coupled to this fix.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence reproduction path: build a compact-direct prompt for a direct Telegram-style context, pass it through `stripEnvelopeFromMessage` without an explicit `senderLabel`, and the current PR code cannot recover a sender label because only the removed JSON blocks are parsed.

## Solution Assessment

No, not as-is. The compact prompt direction is narrow and appropriate for the linked reports, but the patch must preserve the existing sender-label display contract through a non-model-visible field or a compact extraction path with a gateway sanitizer regression test.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve sender labels in compact history:** `src/auto-reply/reply/inbound-meta.ts:291`
  - body: Compact direct mode skips the `Sender` block and only writes IDs in the compact line. `stripEnvelopeFromMessage` backfills `senderLabel` by parsing the existing sender/conversation metadata before stripping it, so compact direct messages without an explicit `senderLabel` lose display attribution in chat history. Preserve that label out-of-band or teach the sanitizer a compact label path with a regression test.
  - confidence: 0.86
