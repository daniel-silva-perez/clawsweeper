## Summary

Keep this issue open. Current main still treats inbound audio as media-understanding/STT input and forwards text plus image prompt attachments to the agent, not raw audio. The directly related native-forwarding implementation candidate, #48900, is still open and unmerged.

## Best Possible Solution

Keep this issue open as the product tracker for native inbound audio forwarding. The best path is to review and harden #48900 or a successor that adds a typed prompt-media contract for audio/video, config for transcript-only versus raw/audio+transcript behavior, capability detection, size/token limits, provider adapter support, docs, and regression coverage for Telegram/Signal-style voice notes.
