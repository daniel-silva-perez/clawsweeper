## Summary

Keep open. Current main still writes transcript header `cwd` from process-global `process.cwd()`, and there is an open same-author PR with closing syntax that directly targets this bug, so the issue should remain open until that implementation is reviewed, landed, or closed.

## Best Possible Solution

Keep the issue open and review PR #50454 as the canonical implementation candidate. The best fix is to thread an explicit target agent workspace/cwd into transcript header creation and add a focused regression test proving transcript headers do not follow `process.cwd()` under per-agent workspace isolation.
