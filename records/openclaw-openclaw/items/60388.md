## Summary

Keep this issue open. Current main still does not distinguish the run's own timeout budget from retryable timeout-shaped failures in the model fallback layer, and open PR #62682 is an unmerged implementation candidate that explicitly targets this issue.

## Best Possible Solution

Keep the issue open and review #62682 as the focused implementation path. The preferred fix is to pass terminal abort context into `runWithModelFallback` or otherwise classify the run controller's timeout reason so run-budget timeouts fail immediately, while provider timeouts and other retryable failover errors continue to use the existing fallback chain. Keep #42482 separate because it addresses related cron AbortController/timer behavior rather than this fallback-layer distinction.
