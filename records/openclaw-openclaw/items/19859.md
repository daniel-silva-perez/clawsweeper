## Summary

Keep this issue open. Current checked-out main still hard-codes plain Enter/Return-to-send in browser Control UI/WebChat and native macOS WebChat, and there is no persisted UI setting, gateway config/schema field, or public WebChat docs entry for a send shortcut preference. The overlapping #8147 remains open, and local ClawSweeper context shows open PR #73390 already targets the canonical implementation path, so the right next action is PR review rather than closing this issue.

## Best Possible Solution

Keep this issue open while #8147 and PR #73390 are unresolved. The best product/code path is to review, validate, and either land or request changes on #73390, ensuring the final fix supports Enter-as-newline with Ctrl/Cmd+Enter send, preserves Shift+Enter/newline and IME behavior, wires the chosen setting through the browser and native WebChat composers as appropriate, and updates settings UI, config/schema/docs, and focused composer tests. After that lands, close this issue as implemented or explicitly consolidate it into #8147.
