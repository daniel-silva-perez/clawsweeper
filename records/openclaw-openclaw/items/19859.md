## Summary

Current main still has the reported WebChat behavior and does not expose a send-shortcut preference. The related canonical issue remains open and the maintainer-labeled implementation PR is still open/unmerged, so this should stay open rather than be closed as implemented or split into another automated fix lane.

## What This Changes

The issue requests a WebChat setting or UI toggle that lets users choose whether plain Enter sends immediately or inserts a newline, with a modifier shortcut used for sending in newline-first mode.

## Best Possible Solution

Keep this issue open until the canonical WebChat send-shortcut path lands or is explicitly consolidated. The final implementation should support Enter-as-newline with Ctrl/Cmd+Enter send, preserve Shift+Enter newline and IME behavior, and update the relevant UI persistence/config/docs plus focused composer tests.
