## Summary

Current main still has the reported behavior: browser Control UI/WebChat and native macOS WebChat send on plain Enter/Return, with no persisted shortcut preference or WebChat config/docs option. The overlapping #8147 remains open and #73390 is the active open implementation candidate, so this issue should stay open until that path is resolved.

## What This Changes

Add a WebChat option or UI toggle that lets users choose between plain Enter-to-send and Enter-as-newline with a modifier shortcut for sending.

## Best Possible Solution

Keep this issue open while #8147 and #73390 are unresolved. Review, validate, and either land or request changes on #73390, ensuring the final behavior supports Enter-as-newline with Ctrl/Cmd+Enter send, preserves Shift+Enter/newline and IME behavior, and updates the appropriate UI, config/schema/docs, and focused composer tests. After that lands, close this as implemented or explicitly consolidate it into #8147.
