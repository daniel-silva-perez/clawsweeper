Current main still has the reported WebChat behavior, and a maintainer-routed review comment already kept this open while the canonical #8147 path and open maintainer-labeled PR #73390 are unresolved. This should not be closed as implemented or spun into a separate automated repair lane yet.

## What This Changes

The issue requests a WebChat setting or UI toggle that lets users choose whether plain Enter sends immediately or inserts a newline, with a modifier shortcut used for sending in newline-first mode.

## Best Possible Solution

Keep this issue open until the canonical WebChat send-shortcut work lands or maintainers explicitly consolidate it. The final implementation should support Enter-as-newline with Ctrl/Cmd+Enter send, preserve Shift+Enter newline and IME behavior, and update the relevant persistence/config/docs plus focused composer tests.
