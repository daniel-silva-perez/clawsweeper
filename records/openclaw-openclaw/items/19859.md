Current main still hard-codes plain Enter/Return-to-send in browser Control UI/WebChat and native macOS WebChat, with no persisted shortcut preference, gateway config/schema field, or public WebChat docs entry. The overlapping #8147 is still open and open PR #73390 is the active implementation candidate, so this issue should remain open for PR review or later consolidation rather than be closed now.

## Best Possible Solution

Keep this issue open while #8147 and PR #73390 are unresolved. After that lands, close this issue as implemented or explicitly consolidate it into #8147.
