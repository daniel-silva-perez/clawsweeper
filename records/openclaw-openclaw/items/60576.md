## Summary

Keep this PR open. Current main still treats an undefined or null unhandled-rejection reason as an unclassified rejection and exits, and the PR’s two-file diff is a focused candidate for the reported Slack Socket Mode crash path rather than an obsolete or ClawHub-only change.

## Best Possible Solution

Review and land this focused PR or implement an equivalent core unhandled-rejection policy with targeted fatal-detection coverage for undefined and null. If maintainers want narrower behavior, the Slack plugin could register a handler through the existing runtime API, but current main does not yet solve the reported crash path.

## Evidence

Because the classifiers return false for null/undefined, the PR’s central guard is not present on main.
