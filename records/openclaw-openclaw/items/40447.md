## Best Possible Solution

Land a docs-only update that keeps the current README `--target` example, then resolve the provider guide by aligning install instructions, source repository, and support channel to a maintained package path, or by removing the broken Issues bullet and documenting forks as community alternatives with clear provenance.

Yes. This is reproducible with static docs inspection plus external metadata checks: the PR head changes the links to `wende`, current main still has the old provider links, GitHub reports `wende` Issues disabled, and npm metadata still advertises the original repo.

## Solution Assessment

No. The README edit is correct and already on main, but the provider-link edit is not the narrowest maintainable fix because it presents a fork as canonical while leaving npm installation and support metadata pointed elsewhere.
