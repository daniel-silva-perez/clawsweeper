Keep open: current main already covers the README flag half, but the provider-link update is still unresolved and the proposed replacement has concrete docs correctness and supply-chain concerns because the installed npm package metadata still points to the original repository while the proposed new Issues link is disabled.

## What This Changes

This PR changes the README quick-start `openclaw message send` example to `--target` and rewrites the Claude Max API Proxy provider guide GitHub and Issues links from `atalovesyou` to `wende`.

## Best Possible Solution

Land or replace this with a docs-only update that keeps the already-correct README state and makes the provider guide internally consistent: the install command, source repository, package metadata, and support channel should align, or maintained forks should be documented explicitly as community alternatives with provenance and an enabled support path.

## Reproduction Assessment

Yes. Static inspection plus metadata checks reproduce the state: current main has the README `--target` fix, current main still has old provider links, npm metadata points to `atalovesyou`, and GitHub reports the proposed `wende` Issues/Discussions surfaces disabled.

## Solution Assessment

No. The README edit is correct but superseded on main, while the provider-link edit is not the narrowest maintainable solution because it presents a fork as canonical without aligning the install guidance, package metadata, or a working support channel.
