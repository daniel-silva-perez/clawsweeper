## Summary

Keep open. The README flag portion is already solved on current main, but the provider-guide portion remains unresolved and this PR is not merge-ready because the proposed source/support links do not match the published npm package metadata and the proposed Issues tracker is disabled.

## What This Changes

This PR changes the README quick-start `openclaw message send` example to `--target` and rewrites the Claude Max API Proxy provider guide GitHub and Issues links from `atalovesyou` to `wende`.

## Best Possible Solution

Land or replace this with a docs-only update that keeps the already-correct README state and makes the provider guide internally consistent: the install command, source repository, and support channel should either align with the published package metadata or clearly document maintained forks as community alternatives with provenance and an enabled support path.

## Solution Assessment
