## Summary

Keep open. Current main still authorizes config writes only at channel/account scope, so the requested per-DM/group/topic/conversation policy remains absent from the authorization model, command gates, bundled channel schemas, docs, and tests; the two existing ClawSweeper comments remain accurate for the checked-out main state.

## What This Changes

Request per-context `configWrites` overrides for DMs, groups, conversation types, and future multi-context channels instead of only channel/account-wide grants.

## Best Possible Solution

Add a backward-compatible generic config-write policy model: normalize conversation identity at the channel/plugin boundary, extend origin and target scopes to carry that identity, define inheritance from channel/account defaults into DM/group/topic/thread overrides, expose narrow SDK helpers, update bundled multi-context schemas and docs, and add contract tests for allow, deny, and inheritance cases.
