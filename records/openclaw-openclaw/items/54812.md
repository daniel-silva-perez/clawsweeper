## Summary

Keep open. Current main still gates config writes only at channel/account scope, not per DM, group, topic, or conversation context as requested.

## Best Possible Solution

Keep this open and implement it as a generic core/plugin contract change: extend config-write scope and target types with normalized conversation identity, add backward-compatible schema/types for context-scoped configWrites, pass that identity through /config and /allowlist authorization, expose narrow SDK helpers for channel plugins, add contract tests for inheritance and denial behavior, then update bundled multi-context channel schemas and docs as channels adopt it.
