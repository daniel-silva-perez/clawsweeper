## Summary

Keep open. Current main still has channel/account-only config-write authorization, so the requested per-DM/group/topic/conversation policy remains missing from the shared authorization model, command gates, bundled channel schemas, docs, and contract tests. The existing ClawSweeper comments on this issue remain accurate for the checked-out main state.

## What This Changes

Add per-context `configWrites` permissions so channel DMs, groups, topics, or conversations can grant or deny config writes independently of the channel/account default.

## Best Possible Solution

Add a backward-compatible generic config-write policy model that carries normalized conversation identity through channel/plugin boundaries, extends origin and target scopes for DM/group/topic/thread overrides, defines inheritance from channel/account defaults, updates bundled channel schemas/docs, and adds contract tests for allow, deny, and inheritance behavior.
