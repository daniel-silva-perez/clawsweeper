## Summary

Keep open. Current main still represents config-write permission at channel/account scope only; conversation-specific DM/group/topic identity is absent from the authorization model, the /config and /allowlist write gates, the bundled channel schemas, and the documented config surface. The prior ClawSweeper comment was accurate and remains true on the current main SHA.

## What This Changes

Request per-context `configWrites` controls so DMs, admin groups, client groups, and other channel contexts can inherit or override config-write permission independently of channel/account-level settings.

## Best Possible Solution

Add a backward-compatible generic config-write policy model: normalize conversation identity at the channel/plugin boundary, extend origin and target scopes to carry that identity, define inheritance from channel/account defaults into DM/group/topic/thread overrides, expose narrow SDK helpers, update bundled multi-context schemas/docs, and add contract tests for allow, deny, and inheritance cases.
