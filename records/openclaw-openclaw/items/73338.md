## Summary

Keep this PR open. The checked-out current main still builds default TUI Gateway URLs from config/env/default port data, while the gateway lock payload does not persist the resolved runtime port. The PR remains a focused implementation candidate for #42461 rather than obsolete work.

## Best Possible Solution

Keep this PR open for maintainer review as the current canonical implementation candidate for #42461. The best path is to land one focused fix that records the resolved Gateway runtime port in the active lock, reads that lock for default local `openclaw tui` connections, preserves explicit `--url` and remote config precedence, and keeps the new lock/TUI regression coverage plus the user-facing docs note.
