## Summary

Keep this PR open. Current main still does not implement the intended behavior: default `openclaw tui` connections do not read an active Gateway lock port, the Gateway lock payload does not persist the resolved runtime port, and the linked bug #42461 remains open. This PR is still an active implementation candidate, not obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review as the active implementation candidate for #42461. The best path is to land or revise a focused fix that records the resolved Gateway runtime port in the lock, reads a verified active lock for default local `openclaw tui` connections, preserves explicit `--url` and remote-config precedence, and keeps the regression tests plus the user-facing docs note.
