## Summary

Keep open. Current main still does not implement the requested shared-server multi-tenancy model with tenant identity propagation, tenant RBAC, tenant-scoped resources/plugins, per-tenant quotas, or tenant-attributed audit boundaries. The discussion is active and includes useful external implementation context from KinthAI, including concrete missing primitives rather than just a vague feature request.

## Best Possible Solution

Keep this open for maintainer product and security triage. If shared-server multi-tenancy is in scope, design tenant identity propagation through gateway auth/request context and protocol, tenant-scoped config/state/plugins/skills/model auth, per-tenant quotas, tenant-attributed audit logs, memory partitioning guarantees, and cross-tenant denial tests/docs. If it is out of scope, document that hostile shared-Gateway tenancy is unsupported and split narrower follow-ups for any plugin/API seams external managed-service implementations need.
