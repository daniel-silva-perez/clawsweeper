## Summary

Keep this PR open. Current main has part of the bundled MCP cleanup path for normal isolated cron embedded runs, but it does not fully implement this PR's intended fix: isolated cron rollover cleanup is still skipped, isolated heartbeat and queued follow-up runs still do not forward one-shot cleanup, `GetReplyOptions` still lacks the cleanup flag, and the session MCP runtime manager still has stale in-flight deletion hazards that the PR addresses. The closed #68450 source PR is useful context, and this PR is the narrow replacement that preserves the cleanup direction without the unrelated Matrix edits.
