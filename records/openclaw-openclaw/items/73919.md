## Summary

Keep this PR open. Current main has some bundled MCP cleanup support, but it does not fully implement this PR's intended fix: isolated cron rollover cleanup is still skipped, isolated heartbeat and queued follow-up runs still do not pass one-shot cleanup through, `GetReplyOptions` lacks the cleanup flag, and the session MCP runtime manager still has the stale in-flight map-deletion race the PR addresses. The prior #68450 discussion is useful context and this PR appears to be the narrow replacement that preserves that cleanup direction without the unrelated Matrix edits.

## Best Possible Solution

Keep this PR open and review it as the focused implementation candidate. The best path is to land or revise the narrow cleanup patch on current main: preserve the #68450 credit, avoid the unrelated Matrix changes from the source PR, keep cleanup failures non-fatal to reply/typing teardown, cover isolated cron rollover, isolated heartbeat, queued follow-up, agent-runner forwarding, and stale in-flight MCP runtime replacement, then run the changed gate requested by the PR.
