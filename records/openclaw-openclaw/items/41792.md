## Summary

Keep open. Current main still lacks the requested `openclaw/plugin-sdk/backup` subpath, so the PR is not obsolete; however, it promotes backup path-planning data into the public SDK and needs maintainer API/security-boundary review, docs, and a rebase before merge.

## What This Changes

The PR adds a public `openclaw/plugin-sdk/backup` subpath that re-exports backup plan helpers and types, plus matching package export and SDK entrypoint metadata.

## Best Possible Solution

If maintainers want backup planning to become a supported plugin API, land a rebased narrow SDK subpath with docs that define the contract and data boundary, updated package/entrypoint metadata, and SDK export validation; otherwise keep `createBackupArchive` as the only public backup helper and leave raw planning internals private.
