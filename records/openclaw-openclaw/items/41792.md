## Summary

Keep open. Current main still does not expose the requested `openclaw/plugin-sdk/backup` subpath, and the backup plan helpers remain internal even though the underlying backup planning code exists. The PR is stale and has a blank template, but the diff is a focused additive SDK proposal rather than something already implemented on main or clearly outside the repository.

## Best Possible Solution

Keep the PR open and ask for a current-main revision. If maintainers want backup planning to be a supported plugin API, land a narrow `src/plugin-sdk/backup.ts` facade, keep `scripts/lib/plugin-sdk-entrypoints.json`, `package.json`, docs such as `docs/plugins/sdk-subpaths.md`, and the API/export guardrails aligned, and explicitly decide whether raw `BackupPlan` is stable enough or should be wrapped in a narrower plugin-facing adapter.
