## Summary

Keep open. Current main still does not expose `openclaw/plugin-sdk/backup`, so this PR is not obsolete; however, it should not merge as-is because it promotes backup planning and local path inventory into a public SDK contract without matching documentation, API-baseline alignment, and explicit trust-boundary review.

## What This Changes

The PR adds a new `openclaw/plugin-sdk/backup` subpath that re-exports backup archive helpers, backup plan resolution/types, and backup create option/result types, then registers the subpath in package exports and SDK entrypoint metadata.

## Best Possible Solution

Land a rebased version only if maintainers want backup planning as supported plugin API: define the intended consumers, document the absolute-path exposure and trusted native-plugin boundary, regenerate the SDK API baseline/export metadata, and keep the lower-level backup planning surface private if `createBackupArchive` is the only supported operation.
