## Summary

Keep this PR open. Current main still lacks the requested `openclaw/plugin-sdk/backup` subpath, but the branch is not merge-ready because the new public SDK surface is undocumented, omitted from SDK docs/API metadata, and exposes local backup path-planning data without an explicit trust-boundary note. The older build-entry review concern appears superseded by current main's JSON-driven SDK build entries, so the remaining blockers are API documentation and boundary review.

## What This Changes

The PR adds an `openclaw/plugin-sdk/backup` subpath that re-exports backup archive helpers, backup plan resolution/types, and backup create option/result types, then registers the subpath in package exports and SDK entrypoint metadata.

## Best Possible Solution

Land a rebased version only if maintainers want backup planning as a supported plugin API: keep the subpath narrow, document the trusted-plugin/local-data boundary, add SDK docs metadata and subpath docs, regenerate the API baseline, and verify package export/build alignment.

## Reproduction Assessment

Not applicable as a runtime reproduction case; this is an additive SDK PR. The static review path is high-confidence: current main lacks the subpath, the PR adds it, and the missing docs/metadata plus path-exposure boundary are visible in the current SDK rules and backup plan types.

## Solution Assessment

No. The proposed subpath is a plausible narrow shape, but the best maintainable solution needs explicit API/security-boundary documentation and generated SDK metadata/baseline updates before third-party plugins rely on it.
