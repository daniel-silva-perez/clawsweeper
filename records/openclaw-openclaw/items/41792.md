## Summary

Keep this PR open because current main still lacks the requested backup SDK subpath, so it is not obsolete. It should not merge as-is: it promotes a new public SDK/local-data contract without documenting the trust boundary and does not update the tracked SDK API baseline.

## What This Changes

The PR adds an `openclaw/plugin-sdk/backup` subpath that re-exports backup archive helpers, `resolveBackupPlanFromDisk`, backup plan types, and backup create option/result types, then registers the subpath in package exports and SDK entrypoint metadata.

## Best Possible Solution

Land a rebased version only if maintainers want backup planning as supported plugin API: document the absolute-path/local-data boundary in plugin SDK docs, regenerate the SDK API baseline, keep package/export metadata aligned, and leave the resolver private if backup plugins only need archive creation.

## Reproduction Assessment

Not applicable as a runtime reproduction case; this is an additive SDK PR. The static review path is high-confidence: current main lacks the subpath, and the patch adds it without the docs and baseline updates required for a public SDK expansion.

## Solution Assessment

No. The patch is directionally narrow, but the best maintainable solution needs an explicit API/security-boundary decision plus docs and generated API baseline updates before third-party plugins depend on this contract.
