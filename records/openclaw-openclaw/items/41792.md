## Summary

Keep open. Current main still does not expose the requested `openclaw/plugin-sdk/backup` subpath, even though the underlying backup planning helpers exist internally. The PR remains a plausible core SDK-boundary change rather than obsolete cleanup, but it needs a current-main revision, docs/API contract follow-through, and explicit security review for exposing backup path planning to plugins.

## Best Possible Solution

Keep this PR open, but require a rebase and a complete SDK-contract revision. If maintainers want backup planning available to backup-related plugins, add a narrow `src/plugin-sdk/backup.ts` subpath, wire it through `scripts/lib/plugin-sdk-entrypoints.json` and `package.json`, update plugin SDK docs and API/export checks, and decide whether the public API should expose raw `BackupPlan` or a narrower plugin-facing backup-plan adapter.
