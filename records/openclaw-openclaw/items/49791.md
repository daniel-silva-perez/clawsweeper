## Summary

Keep open. Current main still routes Feishu group slash/directive commands through command authorization based on per-group `allowFrom` or top-level `channels.feishu.allowFrom`, and unauthorized command-only/directive-only messages still produce no visible reply. The reporter’s workaround confirms configuration can make `/think` work, but the requested group UX fix is not implemented and the only linked fixing PR is closed unmerged.

## Best Possible Solution

Keep this open and implement a focused Feishu/group command UX fix: when an explicitly mentioned group command or directive-only message such as `/think medium` is blocked by command authorization, return a non-sensitive visible rejection pointing to `commands.allowFrom`, `channels.feishu.allowFrom`, or `channels.feishu.groups.<chat_id>.allowFrom`. Add regression coverage for unauthorized and authorized Feishu group `/think` paths, and verify legacy `allowall` plus binding configurations without weakening group sender authorization.
