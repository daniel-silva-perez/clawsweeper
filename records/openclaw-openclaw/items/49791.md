## Summary

Keep open. Current main still makes Feishu group slash/directive commands depend on explicit sender command authorization, and unauthorized command-only/directive-only messages still produce no visible rejection. The reporter confirmed that configuring Feishu group command authorization makes /think work, but the remaining requested UX fix is still not implemented, and the linked fixing PR is closed unmerged.

## Best Possible Solution

Keep this open and implement a focused Feishu/group command UX fix: when an explicitly mentioned group command or directive-only message such as /think medium is blocked by command authorization, return a non-sensitive visible rejection that points to commands.allowFrom, channels.feishu.allowFrom, or channels.feishu.groups.<chat_id>.allowFrom. Add regression coverage for unauthorized and authorized Feishu group /think paths, and verify legacy allowall/groupPolicy configurations without broadening who can run system commands.
