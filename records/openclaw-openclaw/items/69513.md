## Summary

Keep open. Current main still validates cron announce channel selection but does not validate explicit `delivery.accountId` or `delivery.failureDestination.accountId` against configured channel accounts before persistence. The item is also paired with open candidate PR context, so the maintainer path is to review/land or close that work rather than close the issue now.

## Best Possible Solution

Keep this issue open and review, port, land, or close the existing candidate PR. The code fix should add generic gateway validation for explicit `delivery.accountId` and `delivery.failureDestination.accountId` during `cron.add` and `cron.update`, using configured channel account-list semantics and focused regression coverage in the gateway cron tests. A user-facing changelog entry is appropriate when the fix lands.
