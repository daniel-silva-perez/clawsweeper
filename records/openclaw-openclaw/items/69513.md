## Summary

Keep open. Current main still validates cron announce delivery channel selection but does not validate explicit `delivery.accountId` or `delivery.failureDestination.accountId` against configured channel accounts before persistence. There is also an open candidate PR with closing syntax for this issue, so the maintainer path is to review/land or close that PR rather than closing this issue now.

## Best Possible Solution

Keep this issue open and review, port, land, or close the existing candidate PR. The code fix should add generic gateway validation for explicit `delivery.accountId` and `delivery.failureDestination.accountId` during `cron.add` and `cron.update`, using configured channel account-list semantics and focused regression coverage in the gateway cron tests.
