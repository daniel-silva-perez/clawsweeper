## Summary

Keep open. The linked Control UI flicker bug is real on current main, and this PR is a useful narrow direction, but the latest patch is not merge-ready because the new streaming text path preserves template indentation visibly while `white-space: pre-wrap` is active.

## What This Changes

The PR changes Control UI chat rendering so streaming assistant markdown and reasoning use Lit text interpolation during streaming, adds a `.chat-text--streaming` CSS rule, and keeps completed messages on sanitized markdown HTML.

## Best Possible Solution

Amend this PR, or replace it with an equivalent focused Control UI patch, so streaming markdown and reasoning updates render as stable text without visible template whitespace, completed messages continue using sanitized markdown HTML, and grouped-render regression coverage locks both render branches before #68655 is closed.

## Reproduction Assessment

Yes. Current main statically reproduces the risky path because streaming messages still render changing markdown via `unsafeHTML(toSanitizedMarkdownHtml(markdown))`, and the PR regression is reproducible from the diff because `pre-wrap` preserves the indented template text around `${markdown}`.

## Solution Assessment

No. The streaming plain-text fallback is a reasonable minimal direction, but this implementation should bind streaming text without surrounding template whitespace, preserve intended newlines, and add targeted tests before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Remove preserved indentation from streaming text:** `ui/src/ui/chat/grouped-render.ts:1432`
  - body: The new streaming markdown branch puts `${markdown}` on an indented standalone line while `.chat-text--streaming` uses `white-space: pre-wrap`. Those static newlines and spaces become visible in the bubble, so streaming replies can start with a blank/indented line and carry trailing whitespace until the final markdown render; bind the text directly between the tags or through an equivalent text-content helper, and fix the duplicate regular-message branch too.
  - confidence: 0.91
- **[P3] Preserve reasoning line breaks while streaming:** `ui/src/ui/chat/grouped-render.ts:1409`
  - body: The PR changes streaming `reasoningMarkdown` to raw text inside `.chat-thinking`, but that class does not preserve whitespace. `formatReasoningMarkdown()` joins the reasoning label and lines with `\n`, so visible streaming reasoning collapses into one run until final markdown rendering; add a streaming thinking style/helper that preserves newlines without preserving template indentation.
  - confidence: 0.78
