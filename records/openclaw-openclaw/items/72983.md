## Summary

Keep open. Current main still renders active Control UI streaming markdown through full sanitized HTML, and this PR is the active implementation candidate for the linked flicker bug. The branch is directionally useful, but it introduces a visible whitespace regression because the new `pre-wrap` streaming text path preserves the template indentation around the markdown interpolation.

## What This Changes

The PR adds a streaming-only plain-text render path and CSS class for Control UI chat markdown/reasoning blocks while leaving completed messages on sanitized markdown HTML rendering.

## Best Possible Solution

Keep this PR open and amend it, or replace it with an equivalent focused Control UI patch, so active streaming markdown updates text without full HTML subtree replacement, does not preserve template indentation as visible whitespace, covers both tool and regular message branches, and adds a focused grouped-render regression test before closing #68655.
