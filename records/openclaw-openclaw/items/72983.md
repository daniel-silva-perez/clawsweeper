## Summary

Keep open. Current main still renders streaming chat markdown through the full sanitized-HTML path, and this PR targets the right Control UI renderer, but the proposed streaming branch is not merge-ready because it preserves template indentation as visible text during streaming.

## What This Changes

The PR adds streaming-only plain-text rendering for Control UI chat message and reasoning markdown plus a `.chat-text--streaming` CSS rule, while keeping completed messages on sanitized markdown HTML.

## Best Possible Solution

Amend this PR, or replace it with an equivalent focused Control UI patch, so streaming markdown and reasoning updates render as stable text without visible template whitespace, completed messages continue using sanitized markdown HTML, and grouped-render regression coverage locks both rendering branches before #68655 is closed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Remove preserved indentation from streaming text:** `ui/src/ui/chat/grouped-render.ts:1432`
  - body: The streaming markdown branch puts `${markdown}` on an indented standalone line while `.chat-text--streaming` uses `white-space: pre-wrap`. Those static newlines and spaces are preserved in the bubble, so streaming replies can start with a blank/indented line and carry trailing whitespace until the final markdown render. Put the text binding directly between the tags, or use an equivalent text-content binding, and fix the duplicate regular-message branch at line 1503 too.
  - confidence: 0.91
