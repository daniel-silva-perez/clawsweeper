## Summary

Keep open. Current main and the v2026.4.27 release tree already exclude `*.trajectory.jsonl` runtime sidecars from the primary transcript helper used by doctor, but the issue has an open, unmerged closing PR (#70812), so cleanup automation should not close it before that PR is resolved.

## What This Changes

Doctor should stop treating live `<session>.trajectory.jsonl` sidecar files as orphan primary transcripts and avoid offering to archive them.

## Best Possible Solution
