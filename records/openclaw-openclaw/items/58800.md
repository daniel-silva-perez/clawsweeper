## Summary

Keep this PR open. Current main still does not implement the intended Slack edit identity preservation: normal Slack sends already receive the resolved agent identity, but the edit paths used for draft streaming, preview finalization, and status-final updates still omit username/avatar fields. The PR remains a plausible fix candidate for #58737, though the review comments identify functional issues that should be addressed before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate for #58737. The best path is to rebase/adapt it on current main, reuse or share the existing `SlackSendIdentity`/scope-fallback logic, pass the resolved `slackIdentity` through draft stream, preview finalization, and status-final edit paths, and keep the proposed regression tests for send/edit identity propagation before maintainer review and merge.
