## Summary

Keep open. Current main still drops Slack agent identity on edit paths, and this PR targets the right regression, but the proposed patch is not merge-ready because one preview-finalization path still does not forward the resolved identity and edit calls need the same missing-scope fallback behavior that sends already have.

## What This Changes

The PR threads optional Slack sender identity through Slack message edit, draft stream, preview-finalization, and status-final update paths, with regression tests for preserving agent display name and avatar on edited messages.

## Best Possible Solution

Port the Slack edit identity fix onto current main, reuse the existing `SlackSendIdentity`/customize-scope fallback contract for edits, pass the resolved `slackIdentity` through draft-stream edits, preview finalization, and status-final completion updates, and keep the linked bug open until a corrected PR lands.

## Reproduction Assessment

Yes. The linked bug has concrete Slack steps, and a focused source check reproduces the underlying defect on current main: sends can include `identity`, but `editSlackMessage` and the preview/status edit call sites do not pass username or avatar fields to `chat.update`.

## Solution Assessment

No, not as currently proposed. The direction is the narrow maintainable fix, but the patch must pass the actual resolved Slack identity to final preview edits and preserve send-path fallback parity for missing `chat:write.customize` before merge.
