## Summary

Keep this PR open. Current main still has the exact limit mismatch described by #66573: `sessions_history` accepts and forwards oversized limits, while `chat.history` validates `limit <= 1000` before its handler clamp can run. The related alternative #66615 is closed unmerged, and this PR is a narrow implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open as the focused implementation candidate for #66573. The maintainer path is to request or add the missing execute-level runtime-clamp test, then review and land this narrow schema-plus-runtime clamp if CI is otherwise healthy. If maintainers instead want to raise the `chat.history` protocol cap, that should be handled as a separate gateway contract change rather than as a reason to close this PR.
