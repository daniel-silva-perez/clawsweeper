## Summary

Keep this PR open. Current main still accepts and forwards oversized `sessions_history` limits while `chat.history` rejects `limit > 1000` during protocol validation, and the linked bug remains open while the alternative gateway-cap PR is closed unmerged.

## What This Changes

This PR adds a 1000 maximum to the `sessions_history` limit schema, clamps outgoing `chat.history` requests to 1000, and adds a schema assertion in the sessions tool test.

## Best Possible Solution

Land the narrow schema-plus-runtime clamp after adding or accepting the execute-level regression test for oversized limits. If maintainers prefer changing the `chat.history` protocol cap instead, that should be handled as a separate gateway contract decision rather than closing this focused tool-side fix.
