## Summary

Keep open. Current main and the latest release still have the repo-root cache parent cleanup gap, so this PR is not obsolete, but the proposed `rmdirSync(tempParent)` cleanup can introduce a cross-process race and should be revised before merge.

## What This Changes

The PR adds best-effort removal of `.openclaw-config-doc-cache` after isolated config-surface retry cleanup and asserts the cache parent is absent after the fallback test.

## Best Possible Solution

Revise the cleanup so config/doc tooling removes repo-root residue without deleting a shared parent that another retry may be about to use, while preserving repo-root dependency resolution. Then keep the regression assertion, skip a changelog entry as internal tooling hygiene, and validate the targeted config-surface test plus the normal changed gate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Avoid deleting the shared temp parent during another retry:** `scripts/load-channel-config-surface.ts:191`
  - body: The new `rmdirSync(tempParent)` can remove the shared parent after another process has created it with `mkdirSync` but before that process calls `mkdtempSync`. In that interleaving the other isolated retry throws `ENOENT`, so concurrent config/doc checks can fail instead of recovering. Use a per-run repo-root temp directory or retry `mkdtempSync` after recreating the parent.
  - confidence: 0.86
