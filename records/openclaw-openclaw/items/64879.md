## Summary

Keep this issue open. Current main has provider/model tagging in the qa-lab mock, but the scenario dispatcher still does not branch on that signal, and the PR/release parity workflows still compare OpenAI and Anthropic through the same `mock-openai` dispatcher rather than a real-provider parity-report lane.

## What This Changes

The issue requests provider-aware qa-lab parity proof, either by adding provider-specific mock dispatch plans or by adding a secret-gated live OpenAI versus Anthropic parity lane.

## Best Possible Solution

Land the deterministic provider-aware mock-dispatch path first: derive the provider variant inside the dispatcher, make at least three agentic parity scenarios return intentionally different but valid plans for OpenAI and Anthropic, and pin those differences in qa-lab mock server tests. Treat a real-provider parity workflow as a separate maintainer-approved follow-up.
