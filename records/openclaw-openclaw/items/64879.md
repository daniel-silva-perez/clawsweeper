## Summary

Keep this issue open. Current main still records provider/model metadata in the qa-lab mock, but the scenario dispatcher itself does not branch on that provider/model signal, and the CI parity workflows still compare OpenAI and Anthropic lanes through the same mock dispatcher rather than through a real-provider parity-report lane.

## What This Changes

The issue asks for qa-lab parity proof to stop producing identical mock scenario plans for OpenAI and Anthropic, either by adding provider-aware mock dispatch or by adding a secret-gated live-frontier parity lane.

## Best Possible Solution

Keep this open and land the provider-aware mock-dispatch path first: derive a provider variant before scenario dispatch, make at least three agentic parity scenarios return intentionally different but valid OpenAI vs Anthropic plans, and pin those differences in qa-lab tests. If maintainers want real-model proof beyond that, add a separate secret-gated live-frontier workflow that produces two real summaries and feeds them into `qa parity-report`.
