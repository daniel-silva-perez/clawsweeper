## Summary

Keep open. Current main still lacks the requested JSON provider-usage/billing object, but this PR is not merge-ready: the patch has a type error, performs OpenRouter billing fetches on non-JSON status output, and can report partial or non-2xx OpenRouter key failures as healthy. It also needs maintainer review for the billing-data privacy contract and plugin-boundary placement.

## What This Changes

The PR adds an `auth.providerUsage` JSON object to `openclaw models status --json`, adds a core OpenRouter metered billing fetcher with tests, and rewires provider usage fetching in the models status command.

## Best Possible Solution

Define a small additive provider-usage JSON contract, keep OpenRouter-specific credential resolution and billing endpoint parsing in the OpenRouter provider plugin through the existing usage hooks or a backwards-compatible SDK extension, fetch metered billing only for the JSON surface that renders it, report partial/non-2xx failures accurately, and document the fields at https://docs.openclaw.ai/cli/models.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use the awaited OpenRouter usage type:** `src/commands/models/list.status-command.ts:343`
  - body: `ReturnType<typeof loadOpenRouterMeteredUsage>` is the Promise type, but this variable stores the awaited snapshot and later reads `.kind`, `.account`, and `.keys`. Typechecking will fail; use `Awaited<ReturnType<...>>` or an explicit snapshot type.
  - confidence: 0.95
- **[P2] Gate OpenRouter billing fetches to JSON output:** `src/commands/models/list.status-command.ts:374-377`
  - body: This call runs for every non-plain status invocation, but OpenRouter billing is only rendered in the JSON payload. With an OpenRouter key configured, the default rich status path gains an unnecessary authenticated billing request and timeout risk.
  - confidence: 0.91
- **[P2] Report partial OpenRouter key failures as errors:** `src/infra/provider-usage.openrouter.ts:272-281`
  - body: Key request failures are not reflected in provider status when any key succeeds, and non-2xx key responses are converted to `null`. A revoked or timed-out key can be omitted while OpenRouter still reports `status: "ok"`.
  - confidence: 0.88
