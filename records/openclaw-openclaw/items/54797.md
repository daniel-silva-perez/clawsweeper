## Summary

Keep this issue open. Current main still renders the Control UI chat session selector as a native grouped select with no clear-completed action, per-row dismiss control, or completed-subagent separator. The backend and Sessions tab already expose deletion machinery, and subagent lifecycle fields are available, but that separate management workflow does not satisfy the requested on-demand cleanup in the dropdown.

## Best Possible Solution

Keep this issue open and implement a focused Control UI selector cleanup path: derive eligible rows from subagent session keys plus terminal lifecycle fields such as status, endedAt, subagentRunState, and hasActiveSubagentRun; expose either an adjacent Clear completed control or an accessible custom selector with dismiss actions/separators; call the existing sessions.delete RPC for eligible rows; refresh sessions.list; and add UI tests that prove completed subagents can be cleared without affecting active or persistent sessions.

## Risks / Open Questions

- The cleanup predicate must be conservative: only terminal subagent rows should be deleted, not active subagents, the current session, main/channel/group sessions, or session-mode subagents intended to remain bound to a thread.
