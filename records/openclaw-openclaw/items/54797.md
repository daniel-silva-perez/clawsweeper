## Summary

Keep this issue open. Current main still renders the Control UI chat session dropdown as a native grouped select with no clear-completed action, per-entry dismiss control, or completed-subagent separator. The Sessions tab can bulk-delete selected sessions through `sessions.delete`, but that is a separate management workflow and does not satisfy the requested dropdown cleanup behavior.

## Best Possible Solution

Keep the issue open. Add a status-aware Control UI chat session selector cleanup path: identify terminal subagent rows using the session key plus `status`/`endedAt`/`subagentRunState`, expose either an adjacent `Clear completed` action or a custom menu with per-row dismiss and separators, call the existing `sessions.delete` RPC for eligible rows, refresh `sessions.list`, and add focused UI tests for completed subagent rows in the selector.
