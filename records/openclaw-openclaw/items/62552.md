## Summary

Keep this PR open. Current main still uses `acp:<uuid>` for ACP bridge fallback sessions and still requires exact pending-prompt session-key equality, so the central ACP bridge fix is not implemented. The task-maintenance part also remains unimplemented, but the proposed cron change needs review because an existing PR review comment flagged a plausible regression for active cron jobs whose `childSessionKey` is not the executing run session.

## Best Possible Solution

Keep the PR open as an implementation candidate. The safest path is to land the ACP fallback-key and raw/canonical pending-prompt matching with focused regression tests, then either split or revise the task-maintenance change so cron liveness remains anchored on cron runtime authority and durable cron run history. Update the ACP docs if `acp-bridge:<uuid>` becomes the intended default.
