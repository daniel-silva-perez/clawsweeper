## Summary

Keep open. Current main does not implement the PR's intended opt-in Gateway HTTP CORS support: there is no `gateway.http.cors` config surface, no CORS handler/preflight path, no CORS docs/tests, and existing origin handling is Control UI/auth protection rather than `Access-Control-Allow-*` response behavior. The provided PR context shows a focused active implementation with review feedback addressed by follow-up commits, so conservative cleanup should not close it.

## Best Possible Solution

Keep the PR open for normal review. The best path is a core Gateway implementation, not ClawHub: add a default-disabled `gateway.http.cors` config, schema/help/generated docs/baseline updates, CORS classification and preflight handling for the Gateway HTTP API routes, security validation for origins and headers, focused gateway tests, and public docs. Review or request revisions on the PR rather than closing it as obsolete.
