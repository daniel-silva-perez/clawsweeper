## Summary

Keep open. Current main still selects isolated cron embedded-agent primaries from `subagents.model`, but the fallback resolver and embedded failover observation still read only payload/default/per-agent `model` fallback policy, so `subagents.model.fallbacks` can be skipped exactly as reported.

## What This Changes

Make isolated cron embedded-agent runs fail over to fallbacks configured on `agents.defaults.subagents.model` or per-agent `subagents.model` when the selected primary model times out.

## Best Possible Solution

Add generic sub-agent-aware fallback-policy plumbing so the same `subagents.model` object that supplies the isolated cron primary also supplies its fallback chain when payload fallbacks are absent, while preserving explicit empty fallback arrays, user strict selections, timeout classification, and existing default-model fallback behavior.

## Reproduction Assessment

Yes. Configure `agents.defaults.subagents.model` or per-agent `subagents.model` as `{ primary: "kimi/kimi-code", fallbacks: [...] }`, run an isolated cron `agentTurn` without payload fallbacks, and the current source path selects the sub-agent primary while fallback resolution and `fallbackConfigured` do not read those sub-agent fallbacks.

## Solution Assessment

Yes. The narrow maintainable fix is a generic sub-agent-aware fallback resolver shared by isolated cron and embedded failover observation, not a Kimi-specific timeout workaround or provider-policy change.
