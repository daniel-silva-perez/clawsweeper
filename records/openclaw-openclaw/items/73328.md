## Summary

Keep this issue open. Current main still resolves Bedrock discovery context windows from an explicit known-model table or the 32K default, the issue-named Claude 3/3.5 IDs are not covered in discovery, and two open PRs already target the same issue with closing syntax, so closure should wait for maintainer review and merge of one canonical fix.

## What This Changes

The issue requests Amazon Bedrock discovery to assign correct per-model context windows for Claude 3.x/4.x models, or expose per-model override config, instead of applying the 32K discovery default to every unmatched model.

## Best Possible Solution

Resolve this in the Amazon Bedrock plugin-owned discovery path with one canonical merged fix, regression coverage for missing Claude 3/3.5 IDs and region-prefixed inference profiles, and matching schema/docs updates if a config override is accepted.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: an unmatched ID such as anthropic.claude-3-5-sonnet-20241022-v2:0 is absent from the known table, resolveKnownContextWindow returns undefined, and discovery falls back to the 32K default.

## Solution Assessment

Unclear. The issue's two proposed directions are both plausible, but the best final path depends on maintainer product choice between a narrow built-in Claude-family fallback and a broader user-configurable override contract.
