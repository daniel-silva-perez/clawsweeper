## Summary

Keep this issue open. Current main still resolves Bedrock context windows from explicit known entries or the 32K discovery default, and the requested Claude 3/3.5 coverage or per-model override surface is not merged. Two open PRs already target the issue, so the next step is maintainer selection between those implementations rather than closure or a duplicate repair job.

## What This Changes

Requests Amazon Bedrock model discovery to respect per-model Claude 3.x/4.x context windows instead of assigning unmatched discovered models the 32K default.

## Best Possible Solution

Resolve this in the Amazon Bedrock plugin-owned discovery path by landing one canonical fix with regression coverage for missing Claude 3/3.5 IDs and inference-profile variants, plus aligned plugin schema/docs if a config knob is accepted.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: an unlisted ID such as `anthropic.claude-3-5-sonnet-20241022-v2:0` is absent from `KNOWN_CONTEXT_WINDOWS`, `resolveKnownContextWindow` returns undefined, and discovery falls through to the 32K default.

## Solution Assessment

Unclear. The report's two suggested directions are both plausible, but maintainers should choose whether OpenClaw wants a built-in Claude-family fallback, an admin-configurable override map, or both before closing this issue.
