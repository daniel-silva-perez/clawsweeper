## Summary

Keep this issue open. Current main still falls back to the global Amazon Bedrock discovery default for unlisted Claude 3/3.5 model IDs, and two open PRs already target this exact issue, so cleanup should not close it or spawn a duplicate fix lane.

## What This Changes

The issue asks Amazon Bedrock discovery to avoid assigning the 32000 default context window to Claude 3.x/4.x models by adding known per-model values or a per-model override mechanism.

## Best Possible Solution

Land one canonical Amazon Bedrock plugin-owned fix with regression coverage for the missing Claude 3/3.5 IDs and inference-profile variants. If maintainers choose the override-config path, keep the manifest schema, zod schema, exported config type, cache key behavior, UI hints, and Bedrock docs aligned; otherwise keep the fix to a known-model/family resolver.
