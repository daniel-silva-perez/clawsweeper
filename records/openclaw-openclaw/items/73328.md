## Summary

Keep this issue open. Current main still resolves Bedrock discovery context windows from an explicit known-model table or the 32K discovery default, and it has neither the requested Claude 3/3.5 family coverage nor a per-model override surface; #73382 and #73959 are active unmerged PRs that explicitly target this issue.

## What This Changes

The issue requests Amazon Bedrock discovery to stop assigning the 32K default context window to Claude 3.x/4.x models by adding per-model/family context windows or a configurable override map.

## Best Possible Solution

Land one canonical Amazon Bedrock plugin-owned fix that covers the missing Claude 3/3.5 IDs and inference-profile variants, with focused regression tests and aligned plugin schema/docs if the configurable override surface is accepted.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: a discovered ID such as `anthropic.claude-3-5-sonnet-20241022-v2:0` has no known-window or family match, so `toModelDefinition` falls through to the configured discovery default of 32K.

## Solution Assessment

Unclear. The two proposed directions are both plausible, but the best maintainable fix needs maintainer selection of the public contract before this issue can close.
