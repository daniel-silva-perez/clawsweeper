## Summary

Keep this issue open. Current main still has a table-driven Bedrock context-window resolver with a 32000 default fallback, and it does not include either the requested configurable per-model override path or a broad Claude 3.x/4.x family fallback. Two open, unmerged PRs also explicitly target this issue, so the right maintainer path is to review and choose between those implementations rather than close this issue early.

## What This Changes

The issue requests that Amazon Bedrock model discovery stop defaulting Claude 3.x/4.x models to a 32000 context window and instead honor per-model or configurable context-window overrides.

## Best Possible Solution

Resolve this through the Bedrock plugin-owned path: review #73382 and #73959, choose the desired product contract, add discovery and inference-profile regression coverage for missing Claude 3.x/4.x IDs, update the manifest/docs if a config knob is accepted, and close this issue only after the selected PR lands or maintainers explicitly reject the request.
