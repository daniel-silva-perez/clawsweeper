## Summary

Keep this issue open. Current main still uses an explicit Bedrock context-window table plus the global discovery default, while the reported Claude 3 Opus/Sonnet and Claude 3.5 Sonnet IDs, a Claude-family fallback, and a per-model override config surface are still absent; two open implementation PRs already target the same issue, so cleanup should not close or duplicate the work.

## What This Changes

Amazon Bedrock discovery should stop assigning the 32000 default context window to Claude 3.x/4.x models that support larger context windows, either through built-in per-model/family handling or a supported override config.

## Best Possible Solution

Land one canonical Amazon Bedrock plugin-owned fix with regression coverage for the missing Claude 3/3.5 IDs and inference-profile variants. If the override-config approach is accepted, keep the manifest schema, zod schema, exported config type, cache key behavior, UI hints, and Bedrock docs aligned; otherwise keep the fix to the known-model/family resolver without expanding the public config contract.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: a model ID such as anthropic.claude-3-5-sonnet-20241022-v2:0 is absent from KNOWN_CONTEXT_WINDOWS, there is no family or override resolver, and discovery falls through to the global default context window.

## Solution Assessment

Unclear. The requested outcome is valid, but the issue intentionally offers two possible solutions and #73382 and #73959 implement different product-contract paths; maintainer review should choose the narrower durable fix before this issue closes.
