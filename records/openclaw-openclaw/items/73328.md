## Summary

Keep this issue open. Current main still resolves Bedrock discovery context windows from an explicit known-model table or the 32K discovery default, with no Claude-family fallback and no per-model override config; two open PRs already target this issue, so the next step is maintainer selection rather than closure or duplicate automated repair.

## What This Changes

Amazon Bedrock model discovery should avoid assigning the 32K default context window to Claude 3.x/4.x models by adding per-model or family-aware context-window handling.

## Best Possible Solution

Land one canonical Amazon Bedrock plugin fix that gives missing Claude 3/3.5/4 IDs the correct context window, preserves explicit higher-capacity known entries, covers region-prefixed inference profiles, and updates schema/docs only if maintainers choose a user-configurable override surface.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: an ID such as `anthropic.claude-3-5-sonnet-20241022-v2:0` is absent from the known map, `resolveKnownContextWindow` returns undefined, and discovery falls through to the 32K default path.

## Solution Assessment

Unclear. Both proposed directions are plausible, but maintainers should choose between a built-in Claude-family fallback, a public `modelContextWindowOverrides` config contract, or both before resolving this issue.
