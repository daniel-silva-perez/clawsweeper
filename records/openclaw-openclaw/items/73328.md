## Summary

Keep this issue open. Current main still resolves Bedrock discovery context windows from an explicit known-model table or the global discovery default, and it does not include the reported Claude 3 Opus/Sonnet and Claude 3.5 Sonnet coverage, a Claude-family fallback, or a per-model override config surface. Two open, unmerged PRs already target this issue, so maintainer review should choose the canonical implementation rather than closing or spawning duplicate repair work.

## What This Changes

The issue requests Amazon Bedrock model discovery to report appropriate per-model context windows for Claude 3.x/4.x models instead of falling back to the global 32K discovery default.

## Best Possible Solution

Land one canonical Amazon Bedrock plugin-owned fix with regression coverage for missing Claude 3/3.5 IDs and inference-profile variants. If the config override path is accepted, align the plugin manifest, zod schema, exported config type, and Bedrock docs; otherwise keep the fix to the known-model/family resolver without changing the public config contract.
