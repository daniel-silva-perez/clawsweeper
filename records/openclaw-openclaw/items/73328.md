## Summary

Keep this issue open. Current main still falls back to the Amazon Bedrock discovery default for unlisted Claude 3/3.5 IDs, and two open PRs already explicitly target the same issue, so cleanup should not close it or spawn another repair lane.

## What This Changes

The issue requests per-model or family-specific Amazon Bedrock context-window handling so Claude 3.x/4.x discoveries do not receive the global 32K default when the model supports a larger window.

## Best Possible Solution

Land one canonical Amazon Bedrock plugin-owned fix with regression coverage for the reported Claude 3/3.5 IDs and region-prefixed inference-profile variants. If maintainers choose the override-config path, keep the manifest schema, zod schema, exported config type, cache key, UI hints, and Bedrock docs aligned; otherwise keep the change to a known-model/family resolver.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: an active Bedrock summary for anthopic/Claude 3.5 Sonnet v2 is not resolved by KNOWN_CONTEXT_WINDOWS, and toModelDefinition falls through to defaults.contextWindow, documented as 32000.

Unclear. The requested outcome is valid, but the issue intentionally allows two implementation directions and the two open PRs pick different contracts; the narrowest durable fix depends on maintainer selection.
