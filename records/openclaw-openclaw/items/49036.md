## Summary

Keep open. Current main has runtime stale gateway-lock recovery, but `openclaw doctor` still only wires session-lock health and has no gateway-lock doctor contribution/helper/test. There is also an open unmerged PR, #49112, whose body uses closing syntax for this issue, so the issue should remain open until that PR is reviewed and landed or closed.

## Best Possible Solution

Keep this issue open while #49112 remains open. The best path is to review/rebase that PR or implement the same focused change on current main: add a doctor gateway-lock health contribution that locates `resolveGatewayLockDir()` / `gateway.*.lock`, reports dead, stale, missing, or malformed owner PID state, supports `openclaw doctor --fix` cleanup, and includes targeted doctor and gateway-lock tests.
