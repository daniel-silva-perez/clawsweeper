## Summary

Keep this issue open. Current main has adjacent updater, backup, health/status, Control UI update, and manual rollback primitives, but it still lacks the requested opt-in safe-update workflow tying pre-update snapshots, retention, gateway/channel/real-agent smoke checks, and automatic downgrade plus state restore into one operator-facing path. The latest maintainer-facing discussion also explicitly kept the broader request open, and the related items are narrower partials rather than canonical superseding coverage.

## What This Changes

The issue requests an optional safe-update workflow that creates a pre-update backup, performs the update, runs post-update health and smoke checks, and rolls back or restores state on failure.

## Best Possible Solution

Design this as an explicit core updater mode, or reject that scope after maintainer review. If accepted, the updater should orchestrate existing update, backup, doctor, status/health, channel-status, Control UI update, and package rollback primitives; add pre-update backup creation with retention controls; run gateway, channel, and real-agent smoke checks; and provide controlled downgrade plus state restore behavior with clear secret-preservation guarantees.
