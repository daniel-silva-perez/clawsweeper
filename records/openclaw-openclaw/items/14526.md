## Summary

Keep this issue open. Current main has adjacent updater, backup, health/status, Control UI update, and manual rollback primitives, but it still does not implement the requested opt-in safe-update workflow with update-tied snapshots, retention, channel/real-agent smoke checks, and automatic downgrade plus state restore. The latest maintainer-facing issue comment also explicitly kept this open, and the related open items are narrower partials rather than superseding coverage.

## What This Changes

The issue requests an optional safe self-update workflow that snapshots OpenClaw state before updating, verifies gateway/status/channel/agent health afterward, and rolls back or restores state when checks fail.

## Best Possible Solution

Design this as an explicit core updater mode, or reject that scope after maintainer review. If accepted, the updater should orchestrate existing update, backup, doctor, status/health, channel-status, Control UI update, and package rollback primitives; add pre-update backup creation with retention controls; run gateway, channel, and real-agent smoke checks; and provide controlled downgrade plus state restore behavior with clear secret-preservation guarantees.
