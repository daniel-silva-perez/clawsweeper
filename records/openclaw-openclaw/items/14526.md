## Summary

Keep open. Current main has adjacent update, auto-update, doctor, gateway health, status, backup, and narrow package-install rollback primitives, but it still does not provide the requested opt-in safe-update workflow tying pre-update state snapshots, retention policy, gateway/channel/real-agent smoke checks, and automatic rollback or restore into one operator path.

## What This Changes

Add an optional safe-update workflow that snapshots OpenClaw state before updating, runs post-update health and smoke checks, and can rollback or restore state on failure with clear retention and secrets handling.

## Best Possible Solution

Make an explicit maintainer decision on whether safe update belongs in core. If accepted, implement it as a first-class updater mode that orchestrates the existing update, backup, doctor, health/status, channel-status, Control UI, and package rollback primitives with update-tied backups, retention controls, gateway/channel/real-agent smoke checks, and controlled downgrade plus state restore behavior.

## Security Review

Status: needs_attention

Summary: No PR diff was reviewed, but the requested feature is security-sensitive because it would archive and restore local state and may automate downgrade after migrations.

Concerns:

- **[medium] Define secret-safe backup retention and restore policy:** `docs/cli/backup.md:80`
  - body: Safe-update backups can include credentials, auth profiles, sessions, and workspace data, while current backup docs leave size and retention unmanaged. The design needs explicit permissions, storage, retention, deletion, and leakage controls before implementation.
  - confidence: 0.87
- **[medium] Constrain automatic downgrade and state restore:** `docs/install/updating.md:197`
  - body: Rollback docs are currently manual. Automating downgrade plus state restore can reintroduce older code against state or config changed by a newer version, so compatibility checks and confirmation rules should be designed before automation.
  - confidence: 0.84
