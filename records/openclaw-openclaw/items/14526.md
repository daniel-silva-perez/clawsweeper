## Summary

Keep open. Current main has adjacent updater, backup, doctor, health/status, auto-update, and package-staging safety primitives, but it still does not implement the requested opt-in safe-update workflow with update-coupled backups, retention, channel or real-agent smoke checks, and automatic rollback plus state restore. Recent issue discussion also explicitly kept the broader request alive.

## What This Changes

The issue requests an optional safe update workflow that snapshots OpenClaw state before updating, runs post-update health and smoke checks, and rolls back or restores state when checks fail.

## Best Possible Solution

Make a maintainer decision on whether safe update belongs as a first-class core updater mode. If accepted, implement it as an explicit orchestrator around the existing update, backup, doctor, health/status, channel-status, Control UI, and package rollback primitives, with retention controls, secret-preserving backup behavior, gateway/channel/real-agent smoke checks, and controlled downgrade plus state restore semantics.
