## Summary

Keep this issue open. Current main has useful adjacent update, backup, health, auto-update, and rollback documentation primitives, but it still does not implement the requested opt-in safe-update workflow: no update-tied pre-update state backup, no backup retention policy, no channel or real-agent smoke verification, and no automatic downgrade plus state restore path. The latest discussion also points to real operator evidence, related open follow-ups, and an open partial rollback PR that does not supersede the broader request.

## What This Changes

Request an optional safe-update workflow that snapshots OpenClaw state before updates, runs post-update health and smoke checks, and rolls back/restores automatically or with confirmation when checks fail.

## Best Possible Solution

Keep this open until maintainers either design and implement an opt-in core safe-update workflow or explicitly decide the workflow belongs outside core. A core path should orchestrate existing update, backup, doctor, status/health, channel status, Control UI update, and rollback primitives; add automatic pre-update backup creation, retention controls, post-update gateway/status/channel/real-agent smoke checks, and controlled downgrade plus state restore; and coordinate scope with #65011, #17215, #39999, #65035, and the external openclaw-auto-update-kit prior art.

## Risks / Open Questions

- This is a broad operational workflow spanning update execution, backup archives, config/state migration, secrets handling, service restart, and rollback semantics; an automated one-shot fix could easily create unsafe restore or downgrade behavior.
- Backups include sensitive state and credentials, so retention, permissions, storage location, and leakage risks need explicit maintainer design before implementation.
- Automatic rollback across schema changes can make downgrade recovery worse unless config migrations and doctor/restore semantics are coordinated.
- Open PR #39999 is useful partial work but does not cover pre-update backup, retention, channel or real-agent smoke tests, or state restoration, so closing this issue against that PR would lose the broader requirement.
