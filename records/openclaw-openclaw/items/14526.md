## Summary

Keep this issue open. Current main has adjacent updater, backup, health, auto-update, and rollback documentation primitives, but it still does not implement the requested opt-in safe-update workflow: no update-tied state snapshot, no backup retention policy, no channel or real-agent smoke verification, and no automatic downgrade plus state restore path. The existing related items are narrower or partial, including open PR #39999, so they do not supersede the broader request.

## What This Changes

The issue requests an optional safer self-update workflow that creates a pre-update backup, runs post-update health and smoke checks, and rolls back or restores state when verification fails.

## Best Possible Solution

Design this as an opt-in core update mode or explicitly reject it from core after maintainer review. If accepted, the core updater should orchestrate existing update, backup, doctor, status/health, channel status, Control UI update, and rollback primitives; add automatic pre-update backup creation with retention controls; run gateway/status/channel/real-agent smoke checks; and provide controlled downgrade plus state restore behavior with clear secret-handling guarantees.
