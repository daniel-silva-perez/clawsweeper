## Summary

Keep this issue open. Current main has useful adjacent primitives: manual backup create/verify, staged package updates, post-update plugin sync, gateway restart/version/plugin-health checks, auto-update timing policy, Control UI update triggering, and manual rollback docs. It still does not implement the requested opt-in safe-update workflow: no automatic pre-update state backup tied to update, no backup retention policy, no channel or real-agent smoke verification, and no automatic downgrade plus state restore path. Related #65011 and #17215 are open follow-ups, and open PR #39999 is only a narrow partial rollback-on-doctor-failure proposal rather than a complete replacement.

## Best Possible Solution

Keep this open until maintainers either design and implement an opt-in core safe-update mode or explicitly decide the workflow belongs outside core. A core path should orchestrate existing update, backup, doctor, status, health, channel status, and Control UI primitives; add automatic pre-update backup creation, retention controls, post-update gateway/status/channel/real-agent smoke checks, and controlled downgrade plus state restore; and coordinate scope with #65011, #17215, open PR #39999, #65035, and the external openclaw-auto-update-kit prior art.
