Keep open. Current main still has the Windows ACL helper-resolution gap, while this PR is a focused security hardening that appears useful but still needs maintainer/security review because it touches command execution and overlaps related Windows env-policy hardening PRs.

## What This Changes

This PR adds SYSTEMROOT and WINDIR to host environment policy artifacts, exports the shared Windows system-root default, resolves Windows ACL audit/reset helpers through validated Windows install roots, and adds dotenv, host-env, and Windows ACL regression tests.

## Best Possible Solution

Land one maintainer-approved Windows hardening that blocks untrusted workspace dotenv Windows trust-root overrides through the shared host env policy, validates Windows system roots before resolving audit/fix helper binaries, uses validated System32 paths for execution, keeps generated policy artifacts in sync, and coordinates the related reg.exe, taskkill.exe, shell, and ACL helper surfaces into a single coherent policy.
