## Summary

Keep open. Current main has several adjacent cron, delivery, timeout, and Telegram approval reliability fixes, but it still does not close this multi-symptom regression: direct exec cron remains unimplemented, catch-up is restart-scoped rather than a configurable missed-delivery policy, and the Telegram delivery/approval/OOM/fallback parts still need current-version logs and a split reproduction.

## What This Changes

The issue reports unreliable Telegram-based cron automation, unstable chat-triggered exec workflows, approval timing problems, Node OOM/partial runs, and missing catch-up or fallback delivery for missed jobs.

## Best Possible Solution

Keep the issue open, collect a latest-release or current-main retest with sanitized cron configuration, run history, Telegram delivery target, approval transcript, and OOM logs, then split confirmed defects by owner: direct exec cron to the open direct-exec trackers, Telegram delivery to the Telegram/outbound path, approval timing to shared exec approvals plus the Telegram adapter, and scheduler missed-run/failure-reporting only if reproduction shows a core cron bug.
