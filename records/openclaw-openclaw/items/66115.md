## Summary

Keep open. Current main still exposes `/v1/models` as an OpenAI-compatible agent-target list, while provider/model discovery is available through Gateway RPC `models.list` and Control UI paths. The request is valid, but changing `/v1/models` would conflict with the documented contract, so this needs maintainer API design rather than conservative cleanup.

## Best Possible Solution

Keep this open for maintainer API design. The likely best path is a compatibility-safe authenticated HTTP discovery endpoint, or explicit extension fields on a safe endpoint, backed by existing `models.list` / `loadGatewayModelCatalog()` behavior while preserving `/v1/models` as the documented agent-target list.
