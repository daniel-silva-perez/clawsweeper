## Summary

Keep open. Current main still exposes `/v1/models` as the documented OpenAI-compatible agent-target list, while actual provider model discovery exists through Gateway RPC `models.list` and Control UI callers. The feature request remains valid, but the safe path needs maintainer API design because changing `/v1/models` would alter a documented compatibility contract.

## What This Changes

The issue requests HTTP API discovery of actual provider model IDs available through the gateway instead of only OpenClaw agent-target IDs from `/v1/models`.

## Best Possible Solution

Keep this open for maintainer API design. The likely best path is an authenticated, compatibility-safe HTTP provider-model discovery endpoint or explicit extension fields backed by existing `models.list` / `loadGatewayModelCatalog()` behavior, while preserving `/v1/models` as the documented agent-target list unless maintainers intentionally choose a breaking contract change.
