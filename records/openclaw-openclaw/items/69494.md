## Summary

Current main still has the macOS browser-proxy auth gap: `GatewayEndpointStore.localConfig()` resolves local `gateway.auth.token` and `gateway.auth.password` only as strings/env/launchd values, while `MacNodeBrowserProxy.defaultEndpoint()` forwards those resolved values into browser proxy requests. This PR remains relevant, but it adds a security-sensitive `/usr/bin/security` keychain subprocess path and should stay open for maintainer review before merge.

## Best Possible Solution

Keep this PR open for maintainer review and land it, or land an equivalent patch. The fix should stay in the macOS app `GatewayEndpointStore` local auth path, resolve supported local `gateway.auth.token` and `gateway.auth.password` SecretRefs through a small injected keychain resolver seam, preserve env and launchd precedence, keep remote auth behavior unchanged unless separately designed, cover helper and actual `localConfig()` browser-proxy paths, and bound/review the keychain subprocess behavior.
