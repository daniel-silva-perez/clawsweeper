## Summary

Keep this issue open. Current main still has the reported status visibility gap: plugin context engines can do proactive/background compaction in afterTurn/maintenance, but the public ContextEngine contract and lifecycle code still provide no generic way to report those compactions into the session-store counter rendered by /status and session_status.

## Best Possible Solution

Keep this open and implement an additive plugin-facing compaction reporting contract, such as notifyCompaction(count?) in afterTurn/runtimeContext or a typed afterTurn/stats result. Core should persist that signal into the session store used by /status and session_status, with focused tests for an ownsCompaction context engine that compacts during afterTurn and increments the displayed count.
