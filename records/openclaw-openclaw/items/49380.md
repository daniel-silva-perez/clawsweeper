## Summary

Keep this issue open. Current main still has the reported visibility gap: /status and session_status render the persisted session-store compactionCount, while plugin context-engine afterTurn/turn-maintenance paths still have no typed callback, return value, stats method, or session-store bridge for plugin-owned leaf/proactive compactions.

## Best Possible Solution

Keep this open and implement an additive plugin-facing compaction reporting contract, such as notifyCompaction(count?) in afterTurn/runtimeContext or a typed afterTurn/maintain/stats result. Core should persist that signal into the session-store counter used by /status and session_status, with focused tests for an ownsCompaction context engine that compacts during afterTurn or background maintenance.
