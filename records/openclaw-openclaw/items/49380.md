## Summary

Keep this issue open. Current main still has the reported visibility gap for plugin-owned proactive or leaf compactions: /status renders the persisted session compactionCount, but the generic ContextEngine afterTurn path has no compaction notification, return value, stats method, or session-store update path for background compactions. Existing increments cover runtime auto-compaction metadata and explicit manual/CLI compaction paths, not plugin afterTurn compactions.

## Best Possible Solution

Keep the issue open and implement an additive plugin-facing compaction reporting seam, such as notifyCompaction(count?) in afterTurn/runtimeContext or a typed ContextEngine stats/result contract. Wire that signal into the session store used by /status and session_status, then add focused tests for an ownsCompaction context engine that compacts during afterTurn and increments the displayed count.
