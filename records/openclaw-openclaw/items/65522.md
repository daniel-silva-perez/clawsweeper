## Summary

Keep open. Current main and the latest release still leave the Control UI refresh button enabled during `chatSending`, active `chatRunId`, and `chatStream` states, and an open maintainer PR already targets the narrow fix. Per policy, this issue should stay open until that PR lands or is closed.

## What This Changes

The issue requests disabling or safely ignoring Control UI chat refresh while an agent turn is active so refresh cannot race the live run and wedge the page or gateway.

## Best Possible Solution

Ship a focused Control UI button-state guard with browser render coverage, then let the issue close through the merged PR; keep the broader active-run responsiveness and media-mirroring work separate under #73836.
