## Summary

Keep this issue open. Current main still lacks a doctor preflight warning or guard for the reported config-path divergence, and #63268 is an open, unmerged implementation PR explicitly paired with this issue.

## What This Changes

The issue requests that `openclaw doctor --fix` warn or guard when it is about to create or repair a different config path than the running gateway/service uses, especially root `/root/.openclaw` versus a service HOME such as `/home/node`.

## Best Possible Solution

Keep this issue open and review #63268 or request a narrow replacement PR. The best fix is a focused doctor preflight warning or guard when the config path doctor will write differs from the running gateway/service config path or a clearly detected richer service config, with tests for `OPENCLAW_CONFIG_PATH`, `OPENCLAW_STATE_DIR`, `OPENCLAW_HOME`/`HOME` precedence, and no broad rewrite of config resolution.
