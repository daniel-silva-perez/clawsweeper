## Summary

Keep open: current main still lacks a doctor preflight guard for the reported config-path divergence, and #63268 is an open, unmerged implementation PR explicitly paired with this issue.

## What This Changes

The issue requests that `openclaw doctor --fix` warn or guard when it would write a root-owned minimal config that differs from the running gateway's real config path.

## Best Possible Solution

Add a focused doctor preflight warning or guard that compares the planned doctor write path with the running gateway/service config path or another generic active-config signal, reports the `OPENCLAW_CONFIG_PATH` remediation, and covers env/home precedence without rewriting general config resolution.
