Keep open. Current main still does not add the requested `doctor --fix` config-path divergence warning/guard, and the discussion already has an open paired implementation PR with closing syntax, so this should stay open until that PR or a replacement lands.

## What This Changes

The issue asks for `openclaw doctor --fix` to warn or guard before writing a CLI-resolved root config path when the running gateway or service uses a different config path.

## Best Possible Solution

Review and land #63268 or a replacement that adds a focused doctor preflight/write-path warning or guard using a generic active service/gateway config signal, while preserving existing config resolution semantics and documenting the `OPENCLAW_CONFIG_PATH` remediation.
