## Summary

Keep this PR open. Current main still does not implement the PR's stated missing Discord message-id behavior: bot/structured sends, webhook sends, and component sends can still return `"unknown"`, and the shared cron delivery path can still treat a non-empty result as delivered. The PR also needs a fresh, scoped update because it targets the old `src/discord` path and bundles unrelated Matrix, agent model-selection, Ollama logging, and Windows schtasks changes.

## Best Possible Solution

Revise or replace this PR with a fresh, scoped Discord plugin patch under `extensions/discord`. The fix should define the missing-message-id contract, cover bot/structured, webhook, and component send paths, account for webhook `wait` behavior, and add regression coverage proving failed Discord delivery is not recorded as delivered by shared outbound and direct cron delivery. Keep unrelated Matrix, agent, Ollama, and schtasks changes out of this PR.
