## Summary

Keep open. Current main still does not implement the PR's stated behavior: Discord bot sends, webhook sends, and component sends can still return messageId "unknown" instead of rejecting a missing Discord message id. The PR is not safe to merge as-is because it is stale against the current Discord plugin path and bundles unrelated changes with review-identified compile and reliability risks.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but replace or heavily revise it before merge. The best path is a fresh, scoped Discord plugin patch under extensions/discord that defines the missing-message-id contract, covers bot, webhook, and relevant component send paths, accounts for webhook wait=false semantics, and adds regression coverage proving cron/direct delivery reports failure instead of delivered when Discord returns no usable message id.
