## Summary

Keep this PR open. Current main still does not implement the stated Discord missing-message-id behavior: bot, webhook, and component send paths can still return `"unknown"`, and shared/cron delivery code still treats a returned result as delivered. The PR should be revised rather than closed as obsolete, but it should not merge as-is because it targets an old `src/discord` path and bundles unrelated Matrix, agent model-selection, Ollama logging, and Windows schtasks changes.

## Best Possible Solution

Revise or replace this PR with a fresh, scoped Discord plugin patch under `extensions/discord`. The fix should define the missing-message-id contract, cover bot, webhook, and component send paths, handle the existing webhook `wait` option deliberately, and add regression coverage proving failed Discord delivery is not recorded as delivered by shared outbound and direct cron delivery.
