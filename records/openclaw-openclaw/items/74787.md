## Summary

Keep open. Current main still lacks OpenClaw-side cleanup for recorded acpx/Codex ACP process trees, and the PR is an active implementation candidate, but the latest diff is not merge-ready because startup reaping still uses the configurable acpx session state directory instead of the generated wrapper directory and still warns on hosts where Unix `ps` is unavailable.

## What This Changes

The PR adds an acpx process reaper, wires pid-backed process-tree cleanup into cancel/close and startup, hardens generated Codex/Claude ACP wrappers, and updates acpx tests, ACP docs, and the changelog.

## Best Possible Solution

Keep the fix in the acpx plugin boundary, use the generated wrapper root for startup ownership checks, keep cancel/close cleanup scoped to pid records whose command is OpenClaw-owned, make unsupported process enumeration quiet, then rerun the focused acpx tests plus the changed gate before maintainer review.

## Reproduction Assessment

Yes. Current main reproduces the cleanup gap by source inspection: cancel/close do not read pid metadata, startup has no stale orphan reaper, and docs still describe manual stale process cleanup; the PR-specific startup bug reproduces with a custom acpx `stateDir` because wrappers live under `ctx.stateDir/acpx` but the PR filters with `pluginConfig.stateDir`.

## Solution Assessment

No, not as-is. The acpx plugin is the right owner boundary and the acpx 0.6.1 session record contract supports pid-based cleanup, but the PR should match the generated wrapper root for startup reaping and quietly skip unsupported Unix process scans.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Use the generated wrapper dir for startup reaping:** `extensions/acpx/src/service.ts:222`
  - body: The wrapper is generated under `ctx.stateDir/acpx`, but startup reaping filters wrapper ownership with `pluginConfig.stateDir`. With a custom acpx session store `stateDir`, stale OpenClaw-generated wrappers no longer match the ownership filter and remain orphaned. Pass the generated wrapper root, or return it from the auth-bridge preparation step, instead of using the session store directory here.
  - confidence: 0.89
- **[P3] Skip unsupported startup process scans quietly:** `extensions/acpx/src/service.ts:221-230`
  - body: Startup calls the stale-process reaper unconditionally, and the default lister shells out to `ps -axo`. On Windows or minimal hosts without `ps`, every acpx service start logs a warning even though PPID=1 orphan scanning is Unix-specific. Guard this path or make unsupported process listing return an empty result without warning.
  - confidence: 0.82
