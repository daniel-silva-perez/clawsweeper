## Summary

Keep open. The PR targets a real acpx cleanup gap that is still present on current main, but the latest head still misses generated-wrapper startup cleanup for custom acpx state directories and can log noisy startup warnings on unsupported hosts.

## What This Changes

The PR adds acpx-owned process reaping for generated Codex/Claude ACP wrappers and plugin-runtime-deps process trees on startup, cancel, and close, hardens generated wrappers, and updates tests, docs, and the changelog.

## Best Possible Solution

Keep the cleanup inside the acpx plugin boundary, pass the generated wrapper root derived from ctx.stateDir into startup reaping, make unsupported process enumeration a quiet no-op, and keep the focused tests, docs, and changelog aligned with that behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence repro: configure a custom acpx plugin stateDir, then stale generated wrappers under ctx.stateDir/acpx will not match the PR head's startup reaper filter using pluginConfig.stateDir.

## Solution Assessment

No, not as-is. The direction and owner boundary are right, but the startup ownership root and unsupported-host process listing behavior should be corrected before this is the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use the generated wrapper root for startup reaping:** `extensions/acpx/src/service.ts:222`
  - body: Wrappers are generated under `ctx.stateDir/acpx`, but startup reaping filters with `pluginConfig.stateDir`. With a custom acpx session-store stateDir, stale generated wrappers no longer match the ownership path and remain orphaned; pass `ctx.stateDir` or the derived wrapper root to the reaper instead.
  - confidence: 0.9
- **[P3] Skip unsupported startup process scans quietly:** `extensions/acpx/src/service.ts:220-232`
  - body: Startup calls the stale-process reaper unconditionally, and the default lister shells out to `ps -axo`. On Windows or minimal hosts without `ps`, every acpx service start can log a warning even though PPID=1 orphan scanning is Unix-specific; guard this path or return an empty process list quietly.
  - confidence: 0.82
