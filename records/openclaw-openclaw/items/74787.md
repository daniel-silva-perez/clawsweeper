## Summary

Keep open. Current main still has the acpx process-cleanup gap this branch targets, and this is an active implementation PR, but the latest branch has a blocking startup-reaper bug for custom acpx state directories plus a lower-priority portability issue.

## What This Changes

The PR adds an acpx process reaper, wires recorded process-tree cleanup into cancel/close and startup, hardens generated Codex/Claude ACP wrappers, and updates tests, docs, and the changelog.

## Best Possible Solution

Fix the startup ownership filter so it matches the generated wrapper directory even when the acpx session store stateDir is customized, keep the cancel/close cleanup scoped to recorded OpenClaw-owned process trees, and make unsupported startup process listing quiet or explicitly skipped before maintainer review and CI completion.

## Reproduction Assessment

Yes. Current main reproduces the cleanup gap by inspection: cancel/close do not use stored pid metadata, startup has no stale orphan reaper, and docs still prescribe manual stale process cleanup; the PR-specific bug is reproducible with a custom acpx config stateDir because generated wrappers live under ctx.stateDir while startup filters with pluginConfig.stateDir.

## Solution Assessment

No, not as-is. The acpx plugin is the right owner boundary and the acpx 0.6.1 session record contract supports pid-based cleanup, but the branch should use the generated wrapper root for startup matching rather than the configurable session stateDir and should quiet unsupported Unix process scans.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Use the generated wrapper state dir for startup reaping:** `extensions/acpx/src/service.ts:222`
  - body: The wrapper is generated under `ctx.stateDir/acpx`, but the startup reaper filters wrapper ownership with `pluginConfig.stateDir`. If a user configures the acpx plugin `stateDir`, startup no longer matches OpenClaw's own generated wrapper path, so stale wrapper orphans remain. Pass the wrapper base, or return the generated wrapper root from the auth-bridge preparation step, instead of using the session store stateDir for this filter.
  - confidence: 0.89
- **[P3] Quietly skip Unix process scans where ps is unavailable:** `extensions/acpx/src/service.ts:221-222`
  - body: Startup now calls the stale-process reaper unconditionally, and the default lister shells out to `ps -axo`. On Windows or minimal hosts without `ps`, this logs a warning on every acpx service start even though PPID=1 orphan reaping is Unix-specific. Guard this path or make unsupported process listing a quiet no-op.
  - confidence: 0.82
