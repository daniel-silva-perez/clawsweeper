## Summary

Keep this PR open. Current main still has only the global `tools.agentToAgent` policy, while the PR's central feature is a per-requester outbound override at `agents.list[].tools.agentToAgent.allow`. The PR is still a plausible implementation candidate, but it needs a rebase onto the current `src/plugin-sdk/session-visibility.ts` policy surface and maintainer review because it changes cross-agent authorization behavior.

## Best Possible Solution

Keep this PR open for maintainer review or an author refresh. The best path is to rebase onto current main, implement the per-agent override in `src/plugin-sdk/session-visibility.ts`, add typed strict-schema/help/docs/generated-schema support for `agents.list[].tools.agentToAgent.allow`, preserve global participation plus outbound-only requester semantics, prune deleted agent IDs immutably, and validate with focused A2A/session tests plus the changed gate.
