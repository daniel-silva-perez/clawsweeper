## Summary

Keep this PR open. Current main still only has a global `tools.agentToAgent` policy; it does not type, validate, document, enforce, or prune the requested `agents.list[].tools.agentToAgent.allow` per-agent outbound override. The PR remains a plausible implementation candidate, especially after the author followed up and earlier review feedback was partly addressed.

## Best Possible Solution

Keep this PR open for maintainer review or an author refresh. The best path is to implement the override in the shared A2A policy, add typed and strict-schema support for `agents.list[].tools.agentToAgent.allow`, document the config surface, preserve global participation plus outbound-only requester semantics, prune deleted agent IDs immutably, and validate with focused session/A2A tests plus changed-gate proof.
