## Summary

Keep this PR open. Current main still has only the global `tools.agentToAgent` gate for cross-agent session tools; it does not expose or enforce the proposed per-requester `agents.list[].tools.agentToAgent.allow` override. The PR remains a plausible implementation candidate, but it needs maintainer review, security-focused A2A policy review, and a rebase onto the current `src/plugin-sdk/session-visibility.ts` policy location.

## Best Possible Solution

Keep this PR open for maintainer review or author refresh. The best path is to rebase onto current main, implement the per-requester outbound override in `src/plugin-sdk/session-visibility.ts`, add strict schema/type/generated-help/docs support for `agents.list[].tools.agentToAgent.allow`, preserve the global participation gate plus outbound-only requester semantics, prune deleted agent IDs immutably, and validate with focused A2A/session tests plus the changed gate.
