## Summary

Keep this PR open. Current main still only implements the global `tools.agentToAgent` participation gate for cross-agent session tools; the PR’s central per-requester outbound override at `agents.list[].tools.agentToAgent.allow` is not present in source, strict config schema, generated config help, docs, or tests. The PR remains a plausible implementation candidate, but it changes cross-agent authorization and needs maintainer review/rebase against the current `src/plugin-sdk/session-visibility.ts` policy location.

## Best Possible Solution

Keep this PR open for maintainer review or author refresh. The best path is to rebase onto current main, implement the per-agent requester override in `src/plugin-sdk/session-visibility.ts`, add strict schema/type/generated-help/docs support for `agents.list[].tools.agentToAgent.allow`, preserve global participation plus outbound-only requester semantics, prune deleted agent IDs immutably, and validate with focused A2A/session tests plus the changed gate.
