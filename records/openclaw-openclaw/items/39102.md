## Summary

Keep this PR open. Current main still implements agent-to-agent session access as a single global `tools.agentToAgent` policy, while the PR's central feature is a typed per-requester outbound override at `agents.list[].tools.agentToAgent.allow`. The PR remains a plausible implementation candidate, but it needs maintainer review because it changes cross-agent authorization behavior.

## Best Possible Solution

Keep this PR open for maintainer review or an author refresh. The best path is to rebase onto current main, implement the per-agent override in the shared session visibility/A2A policy, add typed strict-schema support and config help/docs for `agents.list[].tools.agentToAgent.allow`, keep global participation plus outbound-only requester semantics, prune deleted agent IDs immutably, preserve the current sandbox/session-status safeguards, and validate with focused session/A2A tests plus the changed gate.
