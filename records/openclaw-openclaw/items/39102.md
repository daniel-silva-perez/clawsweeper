## Summary

Current main still does not implement the PR's intended per-agent outbound A2A allowlist override. The shipped A2A policy remains global-only, per-agent `tools.agentToAgent` is not typed or accepted by the strict agent tools schema, session tool denials still refer only to global `tools.agentToAgent`, and agent deletion pruning still only scrubs the global allowlist. The PR remains a plausible implementation candidate, not an obsolete or implemented change.

## Best Possible Solution

Keep this PR open for maintainer review or an author refresh on current main. The best path is to implement the override in the current shared A2A policy location, add typed and strict-schema support for `agents.list[].tools.agentToAgent.allow`, document the config surface, preserve global participation plus outbound-only requester semantics, prune deleted agent IDs immutably, and validate with targeted session/A2A tests plus the changed gate.
