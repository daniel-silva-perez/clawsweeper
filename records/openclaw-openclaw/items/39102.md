## Summary

Current main still does not implement this PR’s intended per-agent outbound A2A allowlist override. The shipped policy remains global-only, the strict per-agent tools type/schema/help surface lacks `agents.list[].tools.agentToAgent.allow`, and the session tools still emit only global `tools.agentToAgent` denial semantics. This PR remains a plausible implementation candidate, not obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review or ask the author to refresh it on current main. The best path is to add a strict `agents.list[].tools.agentToAgent.allow` type/schema/help/docs surface, evaluate global A2A participation before requester-specific outbound overrides, preserve outbound-only semantics, keep sandbox/session-status behavior intact, prune deleted agent IDs immutably, and validate with targeted A2A/session-status tests plus the changed gate before merge.
