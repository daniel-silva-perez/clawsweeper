## Summary

Keep open. Current main still filters Mattermost `post_edited` WebSocket events before they can reach the existing post, mention, and wake path, and the linked implementation PR #72513 remains open and unmerged.

## What This Changes

The issue requests Mattermost handling for `post_edited` WebSocket events so @mentions added by editing an existing post wake the addressed agent.

## Best Possible Solution

Review and land PR #72513, or an equivalent Mattermost-only patch, so `post_edited` is treated as a post-bearing event, the updated body flows through the existing mention gate, and replay handling remains safe for edited revisions.
