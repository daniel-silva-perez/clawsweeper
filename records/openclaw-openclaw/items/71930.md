## Summary

Keep open. Current main still drops Mattermost `post_edited` WebSocket payloads before the post parser, mention gate, and wake path, and open PR #72513 is already the active implementation candidate linked with closing syntax.

## What This Changes

The issue requests Mattermost `post_edited` WebSocket events to be re-evaluated so @mentions added by editing a message can wake the tagged agent.

## Best Possible Solution

Review and land PR #72513, or an equivalent Mattermost-only patch, so `posted` and `post_edited` events that carry `data.post` feed the same post parsing and mention-gate flow while deliberately preserving self-message and duplicate-delivery protections.

## Reproduction Assessment

Yes. A `post_edited` WebSocket payload on current main follows deterministic early returns in `monitor-websocket.ts` before `onPosted`, and the issue provides concrete Mattermost steps confirmed on April 25, 2026.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Mattermost plugin WebSocket parser/dispatcher path; the source and extension boundary do not show a need for a core channel API change.
