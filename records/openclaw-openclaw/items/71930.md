## Summary

Keep this issue open. Current main still filters Mattermost WebSocket post handling to `posted` only, so `post_edited` events do not reach mention handling. PR #72513 is open, unmerged, and explicitly targets this bug; it should be reviewed/landed or revised before this issue is closed.

## Best Possible Solution

Keep this issue open while maintainers review PR #72513. The best fix path is a focused Mattermost plugin change that routes `post_edited` events through the existing post/mention flow and makes replay dedupe edit-revision-aware, with targeted WebSocket and replay-guard tests; after that PR or an equivalent replacement merges, close this issue as implemented.
