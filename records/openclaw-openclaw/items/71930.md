## Summary

Current checked-out main still drops Mattermost `post_edited` WebSocket events before mention handling: both `parsePostedPayload` and the WebSocket dispatcher only accept `posted`. The replay guard is also still keyed only by post id, so the issue remains valid. PR #72513 is open, unmerged, and explicitly targets this bug with Mattermost-only source/test changes, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open while maintainers review PR #72513. The best fix path is a focused Mattermost plugin change that routes `post_edited` events through the existing post/mention flow and makes replay dedupe edit-revision-aware via `edit_at`, with targeted WebSocket and replay-guard tests; after that PR or an equivalent replacement merges, close this issue as implemented.
