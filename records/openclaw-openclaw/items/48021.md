## Summary

Keep this PR open. Current main still propagates `file.name` directly into bootstrap context stats and `/context` renders that raw label, so the linked `undefined` display bug is not implemented on main. The PR remains a narrow implementation candidate for open issue #47941, with no security-sensitive surfaces in the provided diff.

## Best Possible Solution

Keep this PR open and review or land a small fix in `src/agents/bootstrap-budget.ts` that derives a safe display name from nonempty `file.name`, normalized basename of `file.path`, then a placeholder, with regression coverage for missing and empty names. After the fix lands, #47941 can close as implemented.
