## Summary

Keep this PR open. Current main still propagates `file.name` directly into context-report bootstrap stats, and the `/context` renderer prints that raw name, so the linked `undefined` label bug is still present. The PR remains a focused implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open and review or land a small fix in `src/agents/bootstrap-budget.ts` that derives a safe display name from nonempty `file.name`, normalized `basename(file.path)`, then a placeholder, with regression coverage for missing and empty names. After that lands, #47941 can close as implemented.
