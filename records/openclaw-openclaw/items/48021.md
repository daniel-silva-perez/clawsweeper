## Summary

Keep this PR open. Current main still returns `file.name` directly for bootstrap context stats, and `/context` renders that raw value, so the linked `undefined` display bug is not implemented on main. The linked issue #47941 is still open, and this PR remains a focused implementation candidate.

## Best Possible Solution

Keep this PR open for maintainer review, or land an equivalent small fix in `src/agents/bootstrap-budget.ts` that derives a safe display name from nonempty `file.name`, normalized basename of `file.path`, then a placeholder, with regression coverage for missing and empty names. After that lands, #47941 can close as implemented.
