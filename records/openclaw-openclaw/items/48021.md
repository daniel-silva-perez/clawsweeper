## Summary

Keep this PR open. Current main still returns the raw bootstrap file name in context stats, so hook-injected files with a valid path but missing or empty name can still render as `undefined` in `/context` output. The supplied PR is a narrow implementation candidate for the still-open #47941, with no security-sensitive files in its diff.

## Best Possible Solution

Keep this PR open for maintainer review, or land an equivalent small fix in `src/agents/bootstrap-budget.ts` that derives a safe display name from nonempty `file.name`, then normalized basename of `file.path`, then a placeholder. Add regression coverage for missing and empty names, preferably also asserting `injectedChars` and `truncated`, then close #47941 as implemented.
