## What This Changes

The PR removes the `schedule.kind === "at"` condition from cron `applyJobResult` deletion eligibility and adds regression tests for successful `every`/`cron` deletion plus error retention.

## Best Possible Solution

Review or rebase this focused PR, decide the recurring `deleteAfterRun` product contract, and land this guard change or an equivalent with regression coverage for successful `every`/`cron` deletion and failed/skipped retention.
