## Summary

Keep open. Current main still does not implement the central configured-agent dropdown behavior, and the PR branch is not safe to merge as-is because it bundles unrelated UI/infra/Slack changes, including a request-body size-limit regression and a tracked runtime log artifact.

## What This Changes

This PR seeds the webchat session dropdown with configured agent main sessions and adds regression tests, while also changing JSON file-write retry behavior, Slack HTTP URL-verification handling, chat controls/model-picker UI, and adding a runtime log file.

## Best Possible Solution

Extract or request a narrow current-main patch that makes the chat session selector include each configured agent's main session when sessions.list lacks a row, with regression coverage for the no-store-row case and duplicate avoidance. Drop the unrelated Slack, JSON-file retry, mobile controls, model picker, and log-file changes from this PR path.

## Reproduction Assessment

Yes. A source-level reproduction is clear: with agentsList containing main and test-agent, sessionsResult containing only agent:main:main, and sessionKey set to agent:main:main, current resolveSessionOptionGroups can only return the stored row plus the active key, so agent:test-agent:main is absent.

## Solution Assessment

No. The PR's core idea of seeding configured agent main sessions is the right direction, but this branch is not the best implementation because it mixes unrelated surfaces and introduces a Slack HTTP request-body guard regression; a focused UI/session-control patch is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Limit Slack request bodies before buffering:** `src/slack/monitor/provider.ts:234`
  - body: The new URL-verification path calls `readRequestBody(req)` before any byte or timeout guard is attached, so an unauthenticated Slack HTTP request can be fully buffered in memory before `SLACK_WEBHOOK_MAX_BODY_BYTES` has a chance to reject it. Apply the existing bounded body reader or enforce the cap during the first read.
  - confidence: 0.96
- **[P1] Keep the chat session selector mounted:** `ui/src/ui/app-render.helpers.ts:233`
  - body: This branch removes the session `<select>` from the rendered `renderChatControls` output and only adds a separate `renderChatSessionSelect` helper in the same file. Because the PR does not update the main chat layout to render that helper, the primary session switcher can disappear from the chat header.
  - confidence: 0.82

  - body: The added log file is local runtime output with install paths and connection identifiers, not source or test data. Keeping it in the branch adds noise and can expose operational telemetry, so it should be removed from the PR.
  - confidence: 0.98
