## Summary

Keep this PR open. Current main still does not implement the PR’s central behavior: the chat session dropdown is still built from sessions.list rows plus the active session key, not from all configured agents. The branch also remains too broad and risky to merge as-is because the provided PR context shows unrelated infra/Slack changes and a committed runtime log artifact.

## Best Possible Solution

Do not merge this branch as-is. Extract or request a narrow patch on current main that makes the chat session dropdown include each configured agent’s main session from state.agentsList, or makes sessions.list expose configured-agent main sessions without inventing misleading history. Keep focused regression coverage for the configured-agent/no-store-row case and drop the unrelated Slack, infra, mobile-control, model-picker, and log-file changes.
