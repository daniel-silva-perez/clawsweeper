## Summary

Keep this PR open. Current main still does not implement the PR’s central dropdown behavior: the chat session selector is still derived from session-store rows plus the active session key, not all configured agents. The PR also remains too broad to merge as-is because it bundles unrelated infra/Slack changes and a runtime log artifact with the focused UI fix.

## Best Possible Solution

Do not merge this branch as-is. Ask for or extract a narrow patch on current main that makes the chat session dropdown include each configured agent’s main session from state.agentsList, or otherwise makes sessions.list expose configured agent main sessions without inventing misleading history. Keep focused regression coverage for the configured-agent/no-store-row scenario and drop the unrelated Slack, json-file retry, mobile controls, model picker, and log-file changes.
