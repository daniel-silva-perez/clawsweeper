## Summary

Keep this issue open. Current main still documents and enforces flat Mattermost DMs, has no configurable Mattermost WebSocket ping/pong keepalive, stores thread history only in the in-memory channelHistories map, and drops mention-only messages after normalization; related #51104, #53186, and #59981 cover parts of the area but do not supersede this broader reliability request.

## What This Changes

Mattermost should gain robust threading support: configurable WebSocket keepalive, optional DM thread parity, thread-history backfill after restart/session clear, and bare @mention handling.

## Best Possible Solution

Keep this open as Mattermost-plugin-owned reliability work. Preserve flat DMs by default, add an explicit DM/per-chat-type threading option for users who want threaded DMs, add configurable WebSocket ping/pong keepalive alongside the existing update_at poll, define safe access-control filtering before thread-history backfill, handle mention-only invocations, and update Mattermost tests, config metadata, and public docs.
