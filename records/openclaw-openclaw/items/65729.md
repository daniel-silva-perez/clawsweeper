## Summary

Keep this open. Current main now covers the WebSocket ping/pong keepalive portion, but the broader Mattermost threading request remains unresolved: DMs are still intentionally forced flat, DM thread history is still disabled, thread backfill is absent, and mention-only messages still return before dispatch.

## What This Changes

The issue requests stronger Mattermost threading reliability: proxy-safe WebSocket keepalive, configurable DM threading parity, restart-safe thread history backfill, and bare @mention handling.

## Best Possible Solution

Keep this as Mattermost-plugin-owned reliability work: retain flat DMs by default, add an explicit opt-in DM threading or per-chat-type reply mode if maintainers want that behavior, define authorized thread-history backfill semantics before fetching server-side thread contents, accept mention-only invocations, and update schema, tests, docs, and changelog together.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: direct chats still force replyToMode/effectiveReplyToId/history off, and a mention-only post normalizes to empty text and returns before dispatch.

## Solution Assessment

No. Current main only solves the WebSocket keepalive portion; the maintainable fix is a maintainer-approved Mattermost plugin design with opt-in DM threading/backfill semantics rather than silently flipping the documented flat-DM default.
