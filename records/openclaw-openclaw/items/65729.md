## Summary

Keep this issue open. Current main still preserves the existing Mattermost flat-DM contract, lacks a configurable WebSocket ping/pong keepalive, keeps DM thread history disabled, has no Mattermost thread-history backfill path, and still drops mention-only messages after normalization. Related #51104, #53186, and #59981 cover parts of the area but do not supersede this broader reliability request.

## Best Possible Solution

Keep this open as Mattermost-plugin-owned reliability work. Preserve flat DMs by default, add an explicit DM/per-chat-type threading option for users who want threaded DMs, add configurable WebSocket ping/pong keepalive alongside the existing update_at poll, backfill empty thread-scoped histories from Mattermost only after access-control filtering is defined, handle mention-only invocations, and update Mattermost tests, config metadata, and public docs.
