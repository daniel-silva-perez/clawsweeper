## Summary

Keep this open. Current main has added fixed-interval Mattermost WebSocket ping/pong keepalive, but the broader request remains unresolved: direct messages are still forced non-threaded, DM history is still disabled, thread-history backfill is absent, and mention-only messages are still dropped after normalization.

## What This Changes

Request Mattermost threading reliability improvements covering configurable WebSocket keepalive, configurable DM threading/session history, thread-history backfill after restarts, and bare mention handling.

## Best Possible Solution

Keep this as Mattermost-plugin-owned reliability work: retain flat DMs by default, add an explicit per-chat-type or DM-threading option for users who want threaded DM sessions, keep the new ping/pong keepalive and expose tuning only if operators need it, backfill empty thread histories after authorization rules are defined, accept mention-only invocations, and update schema, tests, and public docs together.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: a Mattermost post containing only @botname normalizes to an empty body and returns early, and direct chats still force replyToMode/effectiveReplyToId/historyKey off regardless of config.

## Solution Assessment

No current implementation fully solves this issue. The new fixed ping/pong keepalive addresses part of the report, but the maintainable path is an explicit opt-in DM threading/backfill design rather than changing the existing flat-DM default implicitly.
