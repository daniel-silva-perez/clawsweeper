## Summary

Keep this issue open. Current main still keeps Mattermost DMs explicitly non-threaded, disables direct-message history, lacks Mattermost WebSocket ping/pong keepalive, has no thread-history backfill from Mattermost, and still drops mention-only messages after normalization. Related #51104, #53186, and #59981 cover important parts, but they do not supersede the broader reliability request here.

## Best Possible Solution

Keep this open and implement it as Mattermost-plugin-owned reliability work under extensions/mattermost. Coordinate #51104 and its keepalive PR candidate, preserve the current documented flat-DM default, add an explicit DM/per-chat-type threading option, add configurable WebSocket ping/pong keepalive, safely backfill empty thread-scoped history from Mattermost, handle mention-only invocations, and update Mattermost plugin tests plus config/docs metadata.
