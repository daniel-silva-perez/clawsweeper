## Summary

Keep this PR open. Current main and v2026.4.26 still use strict minute-field equality for top-of-hour cron stagger detection, so the linked bug remains real. The PR is relevant and narrow, but it needs a correctness pass for mixed comma-list entries, Croner-supported step syntax, and end-to-end default-stagger coverage before merge.

## What This Changes

This PR adds a minute-field helper and tests so top-of-hour cron stagger detection treats step, list, range, range-step, and wildcard minute fields as firing at minute 0.
