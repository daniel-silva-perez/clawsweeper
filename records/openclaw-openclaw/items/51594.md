## Summary

Keep this PR open. Current main still has the reported strict minute-field check, and the PR is useful source work, but the branch is not merge-ready because it still misses Croner-valid composite minute lists and adds coverage for unsupported numeric-prefix step syntax.

## What This Changes

The PR adds a minute-field helper and tests so top-of-hour cron stagger detection treats step, list, range, range-step, and wildcard minute fields as firing at minute zero.

## Best Possible Solution

Land the cron stagger fix after the helper mirrors Croner-compatible minute-field semantics for both 5-field and 6-field expressions, including recursive comma-list entries, and avoids recognizing syntax that OpenClaw's Croner configuration rejects.

## Reproduction Assessment

Yes. Current main has a high-confidence reproduction path by inspecting src/cron/stagger.ts and checking that expressions like */15 * * * *, 0,30 * * * *, and 0-5 * * * * are not treated as top-of-hour stagger candidates.

## Solution Assessment

No. The PR is pointed at the right narrow helper, but the implementation should recurse through comma-separated entries and remove numeric-prefix step support so the helper matches OpenClaw's actual Croner contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Recurse through comma-separated minute entries:** `src/cron/stagger.ts:28`
  - body: `minuteFieldIncludesZero` returns from the comma branch before checking range or step syntax inside each list item. Croner parses list entries recursively, so expressions like `0-5,30 * * * *` and `*/15,45 * * * *` fire at minute zero but this helper returns false, leaving those jobs without the default stagger this PR is meant to add.
  - confidence: 0.94
- **[P3] Remove unsupported numeric-prefix step coverage:** `src/cron/stagger.ts:23`
  - body: OpenClaw uses Croner with default options, and Croner 10 rejects numeric-prefix stepping such as `0/15` unless sloppy range parsing is enabled. This helper and its new test should not lock in `0/15` as supported OpenClaw cron syntax; use Croner-valid forms such as `*/15` or `0-59/15` instead.
  - confidence: 0.86
