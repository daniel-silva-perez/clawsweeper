## Summary

Keep open. The underlying cron stagger bug is still present on current main, but this PR should not merge as-is because the proposed parser still misses Croner-valid composite comma-list entries and adds coverage for numeric-prefix stepping that Croner rejects with OpenClaw's default options.

## What This Changes

The PR adds a minute-field zero detector and tests so top-of-hour cron stagger detection recognizes step, list, range, range-step, and wildcard minute fields in 5-field and 6-field cron expressions.

## Best Possible Solution

Tighten the helper to match Croner-compatible minute-field semantics for both 5-field and 6-field expressions, including recursive comma-list entries, avoid unsupported numeric-prefix syntax, and land regression coverage across the helper, normalization, service defaulting, and doctor migration paths before closing #51587.
