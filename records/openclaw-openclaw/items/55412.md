## Summary

Keep open. Current main no longer falls all the way through to the raw generic `Agent failed before reply` copy for real `GatewayDrainingError`, but it still surfaces a user-visible restart message and does not auto-retry the run. The linked fix PR was closed unmerged, and the broader open incident report does not supersede this narrower retry request.

## Best Possible Solution

Implement a one-shot gateway-drain retry in the agent run loop, ideally reusing the existing transient retry structure with a bounded delay or gateway-health wait, then add regression coverage proving a `GatewayDrainingError` retries and succeeds without emitting user-visible restart/error text. Keep the existing friendly restart message only for true restart-abort or retry-exhausted cases.
