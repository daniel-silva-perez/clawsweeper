## Summary

Keep open. Current main handles `GatewayDrainingError` more deliberately than the original generic failure path, but it still fails the reply operation and returns a user-visible “Gateway is restarting” message instead of waiting and retrying the run. The linked fix PR #55470 was closed unmerged, and the newer restart/session-loss issues are adjacent rather than canonical replacements for this narrow retry request.

## What This Changes

Treat `GatewayDrainingError` during gateway restart drain windows as a transient condition that waits briefly and retries instead of surfacing an agent failure to the user.

## Best Possible Solution

Add a bounded retry path for gateway-drain lifecycle errors at the agent-runner boundary, using the existing transient-error loop shape where appropriate, with tests proving that a first `GatewayDrainingError` waits and retries without sending a user-visible failure while repeated drain failure still terminates predictably.
