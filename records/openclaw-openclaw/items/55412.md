## Summary

Keep open. Current main no longer falls through to the old generic `Agent failed before reply` copy, but it still treats `GatewayDrainingError` as a terminal reply failure, marks the reply operation `gateway_draining`, and returns user-visible restart text instead of retrying. The prior retry PR #55470 is closed and unmerged, while #63279 and #73918 are broader restart/session-loss reports rather than canonical replacements for this narrow retry gap.

## What This Changes

The issue requests that `GatewayDrainingError` during a gateway restart drain be handled as a bounded transient retry instead of surfacing a user-visible failure or restart prompt.

## Best Possible Solution

Add a bounded gateway-drain retry at the agent-runner boundary, reusing the existing transient-error loop shape where practical, with tests proving that the first drain error waits and retries without sending a user-visible failure while repeated drain failure exits predictably.
