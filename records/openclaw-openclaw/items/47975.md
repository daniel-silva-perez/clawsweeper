## Summary

Keep open. Current main has partial stale-subagent hardening, but it still does not satisfy the reported cleanup/visibility behavior: one-shot sessions_spawn defaults to keep-mode, keep-mode run subagents are not scheduled for archive, sessions.list still surfaces stored completed subagent rows, and task maintenance still treats a subagent task as backed when the zombie child session row exists.

## Best Possible Solution

Keep this open as the broader core tracker. The best fix is to make run-mode subagent completion and retention explicit: either archive or hide terminal keep-mode subagent sessions from operator session lists after a short retention window, add a configurable subagent retention knob, and make task reconciliation check subagent run/process liveness rather than treating any child session row as authoritative backing. Verify with focused coverage for sessions.list, Control UI session rows, /tasks or status output, and post-completion message responsiveness.
