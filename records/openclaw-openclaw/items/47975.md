## Summary

Keep open. Current main still keeps one-shot subagent sessions by default, does not schedule keep-mode run sessions for `sessions.delete`, and the default session list path can still return completed child rows. Recent code improves stale active/historical status and task/status honesty, and #49000 covers a narrower delete-mode archive retry bug, but the broader session-retention and responsiveness report is not fully implemented or superseded.

## Best Possible Solution

Keep this issue open as the broader subagent lifecycle and responsiveness tracker. Maintainers should first decide the product contract for one-shot subagent retention, then implement a focused default behavior for hiding or pruning terminal run-mode child sessions, reconcile stale running subagent tasks when retained child rows are no longer live execution, and leave scheduler/announce-path starvation to narrower related issues such as #64253 and #54488.
