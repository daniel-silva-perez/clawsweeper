## Summary

Keep open. Current main has useful mitigations for stale child-link display and task lost-state reconciliation, but it still preserves default one-shot subagent sessions and still lets default session-list/UI paths enumerate stored child rows. The remaining behavior is broad enough that maintainers should decide the product contract before changing retention, visibility, cleanup, and status semantics.

## What This Changes

The issue asks OpenClaw to clean up completed one-shot subagent sessions, keep completed child sessions out of normal session/status surfaces, and prevent stale subagent state from making the main session or Control UI appear unresponsive.

## Best Possible Solution

Maintainers should define the intended contract for one-shot subagent history: whether terminal run-mode child sessions are hidden, archived, or pruned by default, how cleanup=keep remains inspectable, and how Control UI/status surfaces distinguish active work from retained history. Then implement that contract consistently across subagent registry cleanup, sessions.list defaults, task reconciliation, attachment retention, docs, and focused UI/session tests, while leaving lane-starvation work to narrower trackers such as #64253 and #54488.
