## Summary

Keep open. Current main has useful partial mitigations for stale child-link display, orphaned subagent recovery, and task lost-state reconciliation, but the central requested behavior is still unresolved: default one-shot subagent sessions are retained, completed keep-mode rows remain in the stored session list, and Control UI still renders the unfiltered `sessions.list` result. This needs maintainer product judgment before changing retention, visibility, cleanup, and status semantics.

## What This Changes

The issue requests automatic cleanup or hiding of completed one-shot subagent sessions, including attachment-backed sessions, so the main session and Control UI remain responsive after subagent completion.

## Best Possible Solution

Define the intended contract for one-shot subagent history: whether terminal run-mode child sessions are hidden, archived, or pruned by default; how `cleanup: "keep"` remains inspectable; and how Control UI/status surfaces distinguish active work from retained history. Then implement that contract consistently across subagent spawn defaults, registry cleanup, `sessions.list`, Control UI filtering/status display, task reconciliation, attachment retention, docs, and focused regression tests, while leaving broader lane-starvation work to narrower trackers such as #64253 and #54488.
