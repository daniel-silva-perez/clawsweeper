## Summary

Keep open. Current main has partial mitigations for stale child-link display and task lost-state reconciliation, but the core requested behavior is still not implemented: default one-shot subagent sessions are retained, keep-mode runs do not get archive timers, and normal session-list/Control UI paths still enumerate stored completed subagent rows. This needs maintainer product judgment before changing retention, visibility, cleanup, and status semantics.

## What This Changes

The issue asks OpenClaw to clean up or hide completed one-shot subagent sessions so stale child state does not make the main session, status surfaces, or Control UI appear unresponsive.

## Best Possible Solution

Define the intended contract for one-shot subagent history: whether terminal run-mode child sessions are hidden, archived, or pruned by default, how cleanup=keep remains inspectable, and how Control UI/status surfaces distinguish active work from retained history. Then implement that contract consistently across subagent registry cleanup, sessions.list defaults, task reconciliation, attachment retention, docs, and focused UI/session tests, leaving lane-starvation work to narrower trackers such as #64253 and #54488.
