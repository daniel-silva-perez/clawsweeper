## Summary

Keep open. Current main still defaults one-shot subagent cleanup to retained history, leaves keep-mode runs without archive timers, and returns retained subagent rows through unfiltered session-list and Control UI paths. Recent liveness and task-maintenance work improves some stale status display, but it does not settle the cleanup, retention, visibility, and responsiveness contract reported here.

## What This Changes

The issue requests that completed one-shot subagent sessions be cleaned up or hidden so stale child sessions do not leave the main session, session list, or Control UI appearing unresponsive.

## Best Possible Solution

Define the intended contract for terminal one-shot subagent history: whether completed run-mode child sessions are hidden, archived, or pruned by default; how cleanup=keep remains inspectable; and how status surfaces distinguish active work from retained history. Then implement that contract consistently across subagent registry cleanup, sessions.list defaults, Control UI filters/status, task reconciliation, attachment retention, docs, and focused regression tests.
