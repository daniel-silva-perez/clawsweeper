## Summary

Keep open. Current main still defaults one-shot subagent cleanup to keep, does not assign archive timers to keep-mode runs, and the ordinary session-list path used by Control UI still returns retained completed child sessions. Recent liveness and task-status work reduces stale active display in some surfaces, but it does not settle the cleanup, visibility, retention, attachment, and responsiveness contract reported here.

## What This Changes

The issue requests cleanup or hiding of completed one-shot subagent sessions and immediate main-session responsiveness after subagent completion.

## Best Possible Solution

Define the intended contract for terminal one-shot subagent history, including whether completed run-mode child sessions are hidden, archived, or pruned by default, how cleanup=keep remains inspectable, how attachments are retained, and how status surfaces distinguish active work from retained history. Then implement that consistently across subagent registry cleanup, sessions.list defaults, Control UI filters/status, task reconciliation, docs, and focused regression tests.

## Reproduction Assessment

Yes for the retained completed-session symptom: current spawn defaults, registry archive logic, session-list code, UI calls, and tests show completed keep-mode child rows remain listable. The delayed main-session responsiveness symptom is only partially reproducible from static evidence and still needs a live gateway/UI scenario to separate it from related lane-starvation issues.

## Solution Assessment

Unclear. A narrow flip of the cleanup default or blanket hiding of historical subagents is not obviously the best fix because it changes user-visible history, debugging, attachment retention, and task status semantics; the safer path is a maintainer-approved retention and visibility contract followed by a targeted implementation.
