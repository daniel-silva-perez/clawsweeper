## Summary

Keep open. Current main has partial mitigations for subagent status noise and task snapshots, but the core behavior described here still remains: one-shot subagents default to retained sessions, keep-mode runs are not scheduled for `sessions.delete`, and default session-list/UI paths can still surface completed child sessions from the store.

## What This Changes

The issue requests cleanup, hiding, or reconciliation of completed subagent sessions so stale child rows do not make the main session, status surfaces, or Control UI appear stuck.

## Best Possible Solution

Maintainers should define the product contract for run-mode subagent retention, then implement it consistently: hide or prune terminal one-shot child sessions by default, keep explicit history access available where intended, reconcile stale-running tasks from run lifecycle rather than retained store rows alone, and leave lane/announce starvation to narrower trackers such as #64253 and #54488.
