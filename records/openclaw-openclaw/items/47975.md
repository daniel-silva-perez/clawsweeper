## Summary

Keep open. Current main still retains ordinary one-shot subagent sessions by default, does not assign archive timers to keep-mode runs, and the default session-list path still returns retained completed child rows to Control UI consumers. Recent task and liveness reconciliation work improves stale status display, but it does not define or implement the cleanup, visibility, retention, and responsiveness contract reported here.

## What This Changes

The issue requests automatic cleanup or hiding of completed one-shot subagent sessions and immediate main-session responsiveness after subagent completion.

## Best Possible Solution

Define the intended contract for terminal one-shot subagent history: whether completed run-mode child sessions are hidden, archived, or pruned by default; how cleanup='keep' remains inspectable; how attachment retention works; and how status surfaces distinguish active work from retained history. Then implement that consistently across subagent registry cleanup, sessions.list defaults, Control UI filters/status, task reconciliation, docs, and focused regression tests.

## Reproduction Assessment

Yes for the retained completed-session symptom: current docs, spawn defaults, registry archive logic, session-list code, and tests show completed keep-mode child rows remain listable. The delayed main-session responsiveness symptom is only partially reproducible from static evidence and would still need a live gateway/UI scenario to isolate from related lane-starvation issues.

## Solution Assessment

Unclear; a narrow code change to flip cleanup defaults or hide all historical subagents is not obviously the best fix because it changes user-visible history and debugging retention. The safer path is maintainer-approved retention and visibility semantics followed by a targeted implementation.
