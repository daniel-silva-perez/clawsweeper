## Summary

Keep open. Current main still defaults one-shot subagent runs to retained sessions, does not schedule keep-mode runs for `sessions.delete`, and the default `sessions.list` / Control UI path can still return completed child rows. Recent code improves stale/historical status display and task maintenance, but the broader retention, status-noise, and responsiveness report is not fully implemented or superseded by the narrower related issues.

## What This Changes

The issue requests cleanup or hiding of completed subagent sessions and restoration of immediate main-session responsiveness after failed or completed subagent runs.

## Best Possible Solution

Keep this issue open as the broader subagent lifecycle and responsiveness tracker. Maintainers should decide the product contract for one-shot subagent retention, then implement a focused default for hiding or pruning terminal run-mode child sessions, reconcile stale-running subagent tasks when retained rows are no longer live execution, and leave announce/lane starvation details to narrower related issues such as #64253 and #54488.
