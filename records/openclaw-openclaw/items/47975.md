## Summary

Keep open. Current main still preserves completed default run-mode subagent sessions and exposes retained child rows through normal session-list and Control UI paths. Recent liveness and task-reconciliation code mitigates stale child-link display and lost task records, but it does not settle the requested cleanup/visibility contract.

## What This Changes

The issue reports that one-shot subagent sessions persist after completion and can leave the main session or Dashboard unresponsive until cleanup, reload, or gateway restart.

## Best Possible Solution

Define the intended contract for terminal one-shot subagent history: whether completed run-mode child sessions are hidden, archived, or pruned by default; how `cleanup: "keep"` remains inspectable; and how status surfaces distinguish active work from retained history. Then implement that contract consistently across subagent registry cleanup, `sessions.list`, Control UI filtering/status display, task reconciliation, attachment retention, docs, and focused regression tests.
