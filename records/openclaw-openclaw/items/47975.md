## Summary

Keep open. Current main still has only partial mitigations: one-shot subagent spawns default to retained `keep` cleanup, keep-mode runs do not get archive timers, and unfiltered `sessions.list` plus Control UI still enumerate retained completed child session rows. The remaining fix needs a clear maintainer decision on retention and visibility semantics rather than cleanup closure.

## What This Changes

The issue requests automatic cleanup or hiding of completed and failed one-shot subagent sessions so the main session and Control UI remain responsive after subagent completion.

## Best Possible Solution

Define the product contract for one-shot subagent history: whether terminal run-mode child sessions are hidden, archived, or pruned by default; how `cleanup: "keep"` remains inspectable; and how status surfaces distinguish active work from retained history. Then implement that contract consistently across subagent registry cleanup, `sessions.list`, Control UI filtering/status display, task reconciliation, attachment retention, docs, and focused regression tests.
