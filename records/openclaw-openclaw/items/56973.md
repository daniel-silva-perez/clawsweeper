## Summary

Keep open. Current main still has the two exported `normalizeChannelId` wrappers with different meanings, and the reported aliased imports in the agent session helper are still present. The cleanup is valid, but full removal of the plugin-registry wrapper needs a backwards-compatible SDK path because that name is exported through public plugin SDK subpaths.

## What This Changes

Request to remove or deprecate ambiguous `normalizeChannelId` wrapper exports and update call sites to use `normalizeChatChannelId` and `normalizeAnyChannelId` directly.

## Best Possible Solution

Land a non-breaking cleanup: migrate internal imports to the canonical names, remove the non-public chat-channel wrapper once callers are updated, expose/use `normalizeAnyChannelId` where appropriate, and keep any plugin SDK `normalizeChannelId` export as a deprecated compatibility alias until a versioned breaking release.

## Reproduction Assessment

Yes. This is a static refactor issue, and current-main inspection reproduces it: both wrapper exports still exist and `src/agents/tools/sessions-send-helpers.ts` still aliases both names in the way the report describes.

## Solution Assessment

The requested direction is mostly right, but the safest maintainable solution is staged. Internal callers can move to canonical names now; public SDK removal should be handled as deprecation or major-version work rather than a drive-by delete.

## Repair Work Prompt

Fix the normalization-name collision tracked by https://github.com/openclaw/openclaw/issues/56973 without breaking the plugin SDK. Migrate repo-internal callers away from ambiguous `normalizeChannelId` imports and toward `normalizeChatChannelId` for bundled chat channels and `normalizeAnyChannelId` for registered plugin channels. Start with `src/agents/tools/sessions-send-helpers.ts`, `src/channels/plugins/session-conversation.ts`, and other imports from `src/channels/registry.ts` or `src/channels/plugins/index.ts` that alias or mean the canonical names. Remove the non-public `src/channels/registry.ts` `normalizeChannelId` wrapper only after its internal callers are gone. For `src/channels/plugins/registry.ts`, add or expose the canonical `normalizeAnyChannelId` path for internal use, but keep `normalizeChannelId` as a deprecated compatibility alias for `openclaw/plugin-sdk/channel-targets` and `openclaw/plugin-sdk/channel-runtime`; do not break package exports, SDK baselines, or third-party plugins. Update the SDK contract/baseline tests if a public canonical export is added. Preserve lazy-loading and runtime import-side-effect behavior. Changelog is only needed if a public SDK export is added; pure internal import cleanup needs no changelog.
