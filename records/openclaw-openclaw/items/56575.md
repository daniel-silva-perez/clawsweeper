## Summary

Keep this PR open. Current main does not implement the proposed model-level session maintenance modes, so the branch is not obsolete, but the PR head still has blocking test and behavior defects and needs maintainer direction before merge because it changes gateway, agent, session, and config behavior.

## What This Changes

The PR adds per-model `sessionCacheTtl` and `timeBasedContextCompact` settings, session activity/cache timestamps, gateway maintenance sweep logic for idle compact/reset modes, Anthropic defaults, and related tests.

## Best Possible Solution

Land a rebased, maintainer-approved session lifecycle feature that cleanly separates prompt-cache TTL behavior from provider-agnostic reset policy, uses robust activity timestamps, documents the new config knobs, and coordinates any keep-warm follow-up with #62475.

  - confidence: 0.91

  - body: `resolveCacheTouchAt` always prefers `lastCacheTouchAt` over `lastAssistantMessageAt`. If maintenance was disabled while assistant turns continued and is later re-enabled, a stale cache-touch timestamp can make a recently active session reset or compact immediately.
  - confidence: 0.84
