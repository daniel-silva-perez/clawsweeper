## Summary

Keep open. Current main does not implement the PR’s explicit `sessionCacheTtl` plus `timeBasedContextCompact` compact/reset maintenance surface; it only has the existing cache-TTL tool-result pruning and heartbeat guidance. The PR remains relevant, especially to the related cache keep-warm/session timestamp discussion, but the branch is not ready to merge because it has deterministic test and behavior regressions.

## What This Changes

The PR branch adds model-level time-based session maintenance params, session activity/cache timestamps, and a gateway sweep that can compact idle sessions or reset expired sessions.

## Best Possible Solution

Implement the compact/reset lifecycle on top of the current cache-pruning, provider-eligibility, session-reset, and config-schema code, or intentionally fold the behavior into the existing `contextPruning`/heartbeat surfaces. Before merge, the branch should fix the failing cache-TTL tests, make reset provider-agnostic where configured, use robust activity timestamp baselines, and add matching docs/schema coverage.
