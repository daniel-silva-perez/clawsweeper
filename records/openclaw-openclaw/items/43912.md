## Summary

Keep open. Current main still skips bundled pre-install resolution for scoped specs, and the PR is a narrow valid fix using the existing exact npmSpec lookup; the remaining work is maintainer review, a small warning-text adjustment, rebase, and refreshed validation.

## What This Changes

The PR changes the CLI bundled install planner so scoped/non-bare install specs can resolve by exact bundled npmSpec before registry/npm installation, with helper tests for matching and non-matching scoped specs.

## Best Possible Solution

Land or adapt the narrow resolver change so exact bundled npmSpec matches win before later registry/npm installs, unmatched scoped or versioned specs still fall through, runtime duplicate precedence remains unchanged, and bypass guidance points users to npm:<package>.

## Reproduction Assessment

Yes. Current main provides a high-confidence source-level reproduction: @openclaw/voice-call is asserted to skip bundled pre-planning, and the install command uses that null result before proceeding to later registry/npm paths.

## Solution Assessment

Yes, with a small wording fix. The PR uses the existing exact npmSpec bundled lookup and adds focused match/no-match tests; the safer final patch should also update the bare-spec warning to the current npm:<package> bypass.
