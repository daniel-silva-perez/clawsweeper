## Summary

Keep open. Current main still skips bundled pre-NPM lookup for scoped/non-bare npm specs, and the current helper test explicitly asserts that old behavior, so this PR is not implemented or obsolete. The patch remains a focused candidate, but it changes documented install resolution semantics and should get maintainer review plus docs alignment before merge.

## What This Changes

The PR changes the CLI install pre-NPM planner so exact scoped/non-bare install specs can resolve to a matching bundled plugin `npmSpec`, with helper tests for matching and non-matching scoped specs.

## Best Possible Solution

Keep this PR open for maintainer review. If maintainers want exact bundled `npmSpec` matches to win before ClawHub/npm, land or adapt the narrow helper change, preserve bare plugin-id lookup and the explicit `npm:` bypass, keep unmatched/versioned specs falling through, and update the CLI/plugin install docs in the same change.
