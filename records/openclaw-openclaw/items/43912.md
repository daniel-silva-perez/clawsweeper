## Summary

Keep open. Current main still skips the bundled pre-NPM plan for scoped npm specs, and the existing test locks that old behavior; this PR addresses a real install-flow gap by reusing the existing exact `npmSpec` bundled lookup. It needs normal maintainer review and a small UX wording update, not closure as obsolete.

## What This Changes

This PR changes the CLI bundled pre-NPM install planner so scoped/non-bare specs can resolve by exact bundled `npmSpec`, with helper tests for matching and non-matching scoped specs.

## Best Possible Solution

Land or adapt the narrow resolver change if maintainers want exact bundled `npmSpec` matches to win before ClawHub/npm, preserve unmatched scoped/versioned fall-through, keep runtime duplicate precedence unchanged, and point explicit npm-only bypass guidance at `npm:<package>`.
