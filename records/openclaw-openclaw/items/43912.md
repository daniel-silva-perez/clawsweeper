## Summary

Keep open. Current main still bypasses bundled pre-install planning for scoped npm specs, and this PR is a narrow useful fix rather than obsolete work; it needs maintainer review, rebase/refreshed validation, and a minor warning-copy tweak before merge.

## What This Changes

The PR changes the CLI bundled install planner and tests so scoped/non-bare install specs can resolve by exact bundled npmSpec before ClawHub/npm fallback, while bare specs still resolve by plugin id.

## Best Possible Solution

Land or adapt the resolver change so exact bundled npmSpec matches win before registry/npm install, unmatched scoped or versioned specs still fall through, runtime duplicate precedence remains unchanged, and the bypass guidance points users to npm:<package>.

## Reproduction Assessment

Yes. Current main has a high-confidence source-level reproduction: @openclaw/voice-call is asserted to skip bundled pre-planning, and the install command uses that null result before proceeding to ClawHub/npm fallback.

## Solution Assessment

Yes, with a small wording fix. Reusing the existing exact npmSpec bundled lookup is the narrowest maintainable solution; the safer final patch should also update the bare-spec bypass warning to npm:<package>.
