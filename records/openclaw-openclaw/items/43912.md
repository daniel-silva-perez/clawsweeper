## Summary

Keep open. Current main still skips bundled pre-NPM planning for scoped npm specs, and this PR remains a narrow valid fix using the existing exact bundled npmSpec lookup. The only definite issue I found is low-priority warning text that should point users at the explicit npm: bypass before merge.

## What This Changes

The PR changes the CLI bundled install planner so scoped/non-bare specs can resolve by exact bundled npmSpec, and replaces the old scoped-spec skip test with matching and non-matching scoped-spec coverage.

## Best Possible Solution

Land or adapt the narrow resolver change so exact bundled npmSpec matches win before later registry installs, unmatched scoped/versioned specs still fall through, runtime duplicate precedence remains unchanged, and bypass guidance points users to npm:<package>.
