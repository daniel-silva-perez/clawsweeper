## Summary

Keep open. Current main still skips bundled pre-NPM planning for scoped npm specs, and the existing helper test still asserts that old behavior; this PR addresses a real CLI install-flow gap by reusing the existing exact bundled `npmSpec` lookup. The patch looks functionally sound, with one low-priority UX wording issue for the bundled bypass hint.

## What This Changes

This PR changes the CLI bundled pre-NPM install planner so scoped/non-bare specs can resolve by exact bundled `npmSpec`, and updates helper tests for matching and non-matching scoped specs.

## Best Possible Solution

Land or adapt the narrow resolver change if maintainers want exact bundled `npmSpec` matches to win before ClawHub/npm, while preserving unmatched scoped/versioned fall-through, leaving runtime duplicate precedence unchanged, and pointing explicit npm-only bypass guidance at `npm:<package>`.
