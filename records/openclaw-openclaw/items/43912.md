## Summary

Keep open. Current main still skips bundled pre-install planning for scoped/non-bare specs, and the existing test asserts that old behavior, so this PR is not implemented on main and remains a narrow useful fix pending maintainer review.

## What This Changes

The PR changes the CLI bundled pre-NPM install planner so scoped/non-bare specs can resolve by exact bundled `npmSpec`, with helper tests for matching and non-matching scoped specs.

## Best Possible Solution

Land or adapt the narrow resolver change if maintainers want exact bundled `npmSpec` matches to win before ClawHub/npm, keep `npm:` as the explicit npm-only bypass, preserve unmatched/versioned spec fall-through, and update the CLI install guidance in the same change.
