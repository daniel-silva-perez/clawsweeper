## Summary

Keep open. Current main still skips bundled pre-planning for scoped npm specs, while this PR contains the narrow code/test change needed to route exact bundled npmSpec matches before ClawHub/npm fallback. The patch looks functionally sound, with one low-severity warning-text issue to fix before merge.

## What This Changes

The PR updates the CLI bundled install planner and helper tests so scoped or other non-bare install specs are looked up by exact bundled npmSpec, while bare specs still use pluginId lookup.

## Best Possible Solution

Land or adapt the resolver change so exact bundled npmSpec matches win before registry/npm installation, unmatched scoped or versioned specs still fall through, runtime duplicate precedence remains unchanged, and bundled-bypass guidance points users to the explicit npm: prefix.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: @openclaw/voice-call is asserted to skip bundled pre-planning, and the install command uses that null result before later ClawHub/npm fallback paths.

## Solution Assessment

Yes, with a small wording fix. Reusing the existing exact npmSpec bundled lookup is the narrowest maintainable implementation; the safer final patch should also update the bare-spec bypass warning to npm:<package>.
