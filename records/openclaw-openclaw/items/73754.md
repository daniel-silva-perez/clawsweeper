## Summary

Keep open. Current main contains the Microsoft Teams JWT CommonJS/ESM unwrap and a regression-shaped test, but the report is release-facing: the latest published release is still v2026.4.26, that tag still has the broken dynamic-import destructuring, and current main still leaves the Teams plugin package metadata at 2026.4.25 while JWT validation failures remain debug-only.

## What This Changes

The issue reports that openclaw v2026.4.26 ships a bundled msteams plugin whose JWT validator drops inbound Teams messages because jsonwebtoken is not unwrapped from its CommonJS default export, and asks for a patched release plus better JWT diagnostics and smoke coverage.

## Best Possible Solution

Keep this issue open until maintainers cut and verify a release or patch release whose published OpenClaw package and bundled/published @openclaw/msteams artifact contain b3bc60ae259b4f1d01a775532bf1e8e77d129d58. The release check should inspect the built artifact, not only TypeScript source; a follow-up should also decide whether JWT validation failures should log at warn/error and whether the existing mock-based test is enough smoke coverage.
