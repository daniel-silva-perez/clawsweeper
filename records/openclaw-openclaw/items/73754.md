## Summary

Keep open. Current main contains the Teams JWT CommonJS/ESM unwrap fix and regression coverage, but the issue’s user-facing problem is that the latest shipped release still missed that fix. The remote tag check shows `v2026.4.26` and no `v2026.4.27*` tag, and the `v2026.4.26` source still has the broken dynamic-import destructuring.

## Best Possible Solution

Keep this issue open until maintainers cut and verify a release or patch release that includes the fixed Microsoft Teams JWT runtime loader in both the OpenClaw package and the bundled/published `@openclaw/msteams` artifact. Before closing, inspect the shipped artifact for the default-export unwrap and decide whether to add warning-level logging for unexpected JWT validation exceptions.
