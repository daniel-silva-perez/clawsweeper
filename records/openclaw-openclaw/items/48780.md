Keep open. Current main still does not contain a sanitizer or normalization step for the reported `</arg_value>>` suffix, and the active exec/read paths still forward incoming strings unchanged. ## What This Changes

The issue requests a Windows/DashScope fix that prevents leaked XML-like `</arg_value>>` fragments from corrupting `exec` commands and `read` paths.

## Best Possible Solution

OpenClaw should normalize the leaked suffix at the active tool-argument boundary for `exec` and `read`, with regression coverage for the reported Qwen/DashScope shape and negative coverage preserving legitimate XML-like content.
