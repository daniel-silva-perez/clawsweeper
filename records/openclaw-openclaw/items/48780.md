## Summary

Keep this open. Current main still does not strip the reported `</arg_value>>` artifact from valid string-valued exec/read arguments, and the issue is already paired with open implementation PR #48792 using closing syntax, so cleanup closure would be premature.

## What This Changes

The issue reports Windows exec and read tool arguments from Qwen/DashScope being corrupted by a trailing `</arg_value>>` fragment.

## Best Possible Solution

Keep this issue open until PR #48792 or a replacement lands a narrow current-main fix for exec command strings and read path strings, with regression coverage for Qwen/DashScope suffixes and negative coverage for legitimate XML-like content.
