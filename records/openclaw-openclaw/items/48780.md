## Summary

Keep this issue open. Current main has adjacent malformed tool-call argument repair for OpenAI-compatible streams, but it does not strip the reported literal `</arg_value>>` suffix from valid `exec` command strings or `read` paths. PR #48792 remains an open implementation candidate with closing syntax for this issue, so the issue should stay open until that PR is reviewed, landed, or explicitly superseded.

## Best Possible Solution

Review PR #48792 or implement an equivalent provider/tool-argument normalization fix in the current code layout, with focused coverage for `exec.command` and read/write/edit string arguments containing trailing `</arg_value>` fragments from Qwen/DashScope-style OpenAI-compatible tool calls.
