## Summary

Keep open. Current main still routes truthy OpenAI-completions `delta.content` directly into the text stream without a runtime type guard or typed-block unpacker, and an open PR already references this issue with closing syntax, so this should wait for that PR to be reviewed, rebased, landed, or replaced.

## What This Changes

Fix Mistral Small reasoning streams so typed non-string `delta.content` blocks do not render as `[object Object]` before the assistant reply.

## Best Possible Solution

Review and land #68418 or an equivalent narrow parser fix that unpacks typed-block `delta.content` arrays in order, preserves text/thinking stream semantics, and keeps regression coverage in `src/agents/openai-transport-stream.test.ts`; close this issue only after that implementation reaches main.

## Reproduction Assessment

Yes. A high-confidence repro is to stream a chunk where `choice.delta.content` is an array of `{type:"thinking"}` and `{type:"text"}` blocks; current main's content branch still treats that value as text, and #68418 documents the mocked failing output containing `[object Object]`.

## Solution Assessment

Yes, the linked PR's approach appears to be the narrow maintainable direction: fix the OpenAI-completions stream parser shape handling and cover it with focused tests. The closed skip-only attempt in #68855 would avoid some coercion but would not preserve the typed text/thinking deltas as well.
