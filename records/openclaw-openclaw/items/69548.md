## Summary

Keep this PR open. Current main still lacks a memory-core `dreaming.staggerMs` config surface even though the generic cron scheduler already supports `CronSchedule.staggerMs`, so the branch is not obsolete. The latest branch fixes the earlier zero-value and blank-string concerns, but it still leaves a user-visible unset/clear semantics bug that should be resolved before merge.

## What This Changes

The branch adds optional `dreaming.staggerMs` manifest/schema metadata, parses it through the memory host SDK, and passes/logs it on the memory-core managed dreaming cron schedule.

## Best Possible Solution

Expose dreaming stagger as a documented memory-core config field that preserves explicit `0`, uses the cron service default when unset, and has tested behavior for changing or clearing a previously configured stagger.
