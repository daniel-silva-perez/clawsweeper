## Summary

Keep open. Current main still does not expose `dreaming.staggerMs`, and this PR is the active implementation candidate, but the latest diff still leaves stale explicit stagger values in place after the config field is removed because the memory-core reconciliation guard skips unset stagger comparison while cron patch merge preserves the old schedule value.

## What This Changes

The PR adds a `dreaming.staggerMs` memory-core manifest/schema/docs field, parses it in the memory host SDK, and passes/logs it on the managed dreaming cron schedule.

## Best Possible Solution

Expose `dreaming.staggerMs` as a documented memory-core setting whose create, update, explicit `0`, unset, and removal semantics are predictable, with focused tests covering config parsing, schedule creation, schedule updates, and clearing a previously configured stagger.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Clear stale stagger when config is removed:** `extensions/memory-core/src/dreaming.ts:294`

  - confidence: 0.89
