## Summary

Keep this PR open. Current main still does not expose or pass `dreaming.staggerMs` through memory-core, even though the core cron layer already supports `CronSchedule.staggerMs`. The PR remains the focused implementation tracker for the already-closed related feature request, but it still needs maintainer review around unset/clear semantics and should add targeted tests/docs before merge.

## Best Possible Solution

Keep this PR open as the canonical implementation path for `dreaming.staggerMs`. Before merge, the best path is to add focused host-SDK and memory-core cron reconciliation tests, update the public dreaming/memory config docs, and have a maintainer decide whether unsetting the field should clear prior explicit stagger or whether `staggerMs: 0` is the documented escape hatch.
