## Summary

Keep open. Current main still does not provide the requested Discord middle-ground behavior: bot-owned autoThread threads bypass mention gating, while Discord config/schema/docs have no `requireMentionInThreadReplies`, `threadRequireMention`, or equivalent option. The related #40895 discussion is relevant and confirms the same autoThread noise pattern, but it remains broader and open rather than a strong canonical duplicate for this focused config request.

## Best Possible Solution

Keep this focused issue open and implement the Discord plugin behavior explicitly. A conservative path is to add a config-backed Discord thread-reply mention policy under the guild/channel entry or `channels.discord.thread`, wire it through `src/config/types.discord.ts`, `src/config/zod-schema.providers-core.ts`, generated config metadata/help, docs, and `extensions/discord/src/monitor/message-handler.preflight.ts`, then add regression tests for bot-owned `autoThread` follow-up replies with and without explicit mentions. Coordinate with #40895, but use this issue for the narrower product shape unless maintainers choose a canonical tracker.
