## Summary

Keep open. Current main still does not implement the requested compaction modes, and this PR contains useful source work, but the proposed patch has blocking runtime semantics issues around warn-mode prompting and none-mode overflow recovery, plus smaller contract/changelog/scope problems that need author or maintainer rework before merge.

## What This Changes

This PR extends agent compaction config/schema/help with `warn`, `error`, and `none`, adds a Pi pre-prompt token guard and mode-aware Pi auto-compaction disabling, lowers the reserve floor, adds a changelog entry, and adds a bundled `ceospace` skill placeholder.

## Best Possible Solution

A mergeable version should first settle the exact semantics for `warn`, `error`, and `none`, then implement them consistently across exported config types, strict/generated schemas, docs/help, mode resolution, Pi auto-compaction, pre-prompt guard accounting, overflow recovery, lifecycle hooks, and focused tests while preserving existing default behavior and splitting the unrelated bundled skill change out of the compaction PR.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Stop warn mode before submitting the prompt:** `src/agents/pi-embedded-runner/run/attempt.ts:2929-2933`
  - body: The latest `warn` branch emits the warning through `onBlockReply` and `assistantTexts.push`, but never sets `skipPromptForCompactionGuard`. Execution therefore reaches the later `activeSession.prompt(...)` block, so near-limit warn-mode turns still call the provider even though this mode is documented as stopping before overflow.
  - confidence: 0.93
- **[P1] Honor none mode in overflow recovery:** `src/agents/pi-settings.ts:109-110`
  - body: Adding `none` here disables only Pi's built-in auto-compaction. The separate OpenClaw overflow-recovery loop still calls `contextEngine.compact()` after model-limit errors, so `mode: "none"` can still auto-compact and retry instead of failing at the model limit as the new help text promises.
  - confidence: 0.89
- **[P2] Use model-visible token accounting for the guard:** `src/agents/pi-embedded-runner/run/attempt.ts:2902-2917`
  - body: The new guard estimates raw `activeSession.messages` and prompt messages directly, while the existing precheck strips non-model-visible `toolResult.details` and applies the estimator safety margin. This can make `warn`/`error` fire on transcript data the model will not receive, or miss near-limit prompts the existing accounting would catch.
  - confidence: 0.82
- **[P2] Preserve the capped reserve floor:** `src/agents/pi-settings.ts:5`
  - body: Lowering `DEFAULT_PI_COMPACTION_RESERVE_TOKENS_FLOOR` from 20,000 to 1,000 changes existing default and safeguard behavior despite the stated scope boundary. Current main already caps the 20,000 floor against the active context window, so keep that default and reuse the capped effective reserve for the new guard.
  - confidence: 0.84
- **[P3] Add the required changelog attribution:** `CHANGELOG.md:10`
  - body: The user-facing changelog entry omits the required `Thanks @...` attribution for added bullets. Add the credited GitHub username or remove the entry until the feature is ready.
  - confidence: 0.86
