## Summary

Keep open. Current main still has only the existing `default`/`safeguard` compaction contract, while this PR proposes new `warn`, `error`, and `none` modes plus runtime behavior that is not present on main. The PR is useful source work, but review context and the public patch show unresolved product/runtime issues around whether `warn` blocks, whether `none` disables all automatic recovery compaction, hook/finalization semantics, changelog attribution, and an unrelated bundled `ceospace` skill addition.

## What This Changes

This PR extends agent compaction config/schema/help/runtime handling with `warn`, `error`, and `none` modes, adds a Pi pre-prompt token guard and mode-aware auto-compaction disabling, lowers the Pi reserve-token floor, updates the changelog, and adds a bundled `skills/ceospace` placeholder skill.

## Best Possible Solution

A mergeable version should define the product semantics for the new modes first, then implement them consistently across exported config types, strict/generated schemas, public docs/help, Pi mode resolution, auto-compaction disabling, pre-prompt guard behavior, overflow and timeout recovery, hook/finalization paths, and focused tests. It should keep the compaction change isolated by removing or separately justifying the bundled `ceospace` skill and fixing the changelog entry before review.
