## Summary

Keep open. Current main still lacks the requested compaction modes, and this PR remains useful source work, but the current patch is dirty against main and has blocking runtime semantics issues for `warn` and `none` before it can be merged.

## What This Changes

The PR extends agent compaction config/schema/help with `warn`, `error`, and `none`, adds a Pi pre-prompt context guard and auto-compaction changes, adds a changelog entry, and adds a bundled `ceospace` skill placeholder.

## Best Possible Solution

A mergeable version should rebase onto current main, settle exact mode semantics, implement the modes through the existing precheck and overflow-recovery lifecycle, keep hooks/finalization behavior consistent, update config schema/help/docs/tests together, and remove unrelated skill scope from the compaction PR.

## Reproduction Assessment

Yes. The `warn` scenario can be checked by setting `agents.defaults.compaction.mode` to `warn` with a high reserve; source inspection of the PR head shows the warning path leaves `skipPromptForCompactionGuard` false and then reaches `activeSession.prompt(...)`. For `none`, a provider context overflow still reaches the current overflow-recovery path that calls `contextEngine.compact()`.

## Solution Assessment

No. The feature direction may be useful, but this implementation is not the best fix because it adds a second token guard instead of extending the current precheck/overflow machinery and does not consistently honor the new modes across all compaction paths.
