## Summary

Keep open. Current main still only supports the existing `default` and `safeguard` compaction modes in the public config contract, validation schema, generated schema/help, docs, Pi extension resolver, and auto-compaction guard. The PR is useful feature work but is not implemented on main and still needs maintainer review because the branch changes core model-call and compaction recovery semantics, has unresolved correctness concerns, and includes an unrelated bundled skill addition.

## What This Changes

This PR extends agent compaction configuration/schema/help/runtime handling with `warn`, `error`, and `none` modes, adds a proactive Pi runner token guard, changes Pi auto-compaction disabling and reserve-floor behavior, updates the changelog, and adds a `skills/ceospace` placeholder skill.

## Best Possible Solution

Keep this PR open for maintainer and author iteration. A mergeable version should first settle the product semantics for `warn`, `error`, and `none`, then implement them consistently across exported config types, strict/generated schemas, docs/help, Pi mode resolution, auto-compaction disabling, pre-prompt guard behavior, overflow/timeout recovery, hooks/finalization, and focused tests; it should also remove or separately justify the bundled `ceospace` skill and fix changelog attribution.
