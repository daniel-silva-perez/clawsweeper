## Summary

Keep open. Current main still exposes only the existing `default` and `safeguard` compaction modes, while this PR is not merge-ready because the proposed `warn`/`none` semantics are not enforced consistently through prompt submission and overflow recovery, and the branch also includes unrelated bundled skill scope.

## What This Changes

This PR extends agent compaction config, schemas, help text, and Pi runtime wiring with `warn`, `error`, and `none` modes, adds a pre-prompt context guard, updates Pi auto-compaction disabling, adds a changelog entry, and adds a bundled `ceospace` skill placeholder.

## Best Possible Solution

A mergeable version should first settle the exact semantics for each mode, then implement them consistently across exported config types, strict and generated schemas, help/docs, Pi mode resolution, auto-compaction disabling, pre-prompt guard behavior, overflow recovery, lifecycle hooks/finalization, and focused tests. The compaction change should stay isolated from unrelated bundled skill additions.

## Reproduction Assessment

Yes. A high-confidence reproduction path exists from the PR body and diff: configure `agents.defaults.compaction.mode` to `warn` with a high reserve to exercise the guard, and configure `none` then trigger provider context overflow to observe whether the separate recovery path still compacts.

## Solution Assessment

No. The feature direction may be useful, but the current PR is not the best implementation because it publishes new modes before their runtime guarantees, token accounting, overflow behavior, lifecycle hooks, docs, and tests are aligned.

- **[P1] Stop warn mode before submitting the prompt:** `src/agents/pi-embedded-runner/run/attempt.ts:2929-2933`
  - body: The `warn` branch emits the warning through `onBlockReply` and records assistant text, but it does not set `skipPromptForCompactionGuard`. Execution can still reach `activeSession.prompt(...)`, so a near-limit warn-mode turn may still call the provider even though this mode is described as stopping before overflow.
  - confidence: 0.93
- **[P1] Honor none mode in overflow recovery:** `src/agents/pi-settings.ts:109`
  - body: Adding `none` here disables only Pi's built-in auto-compaction. The separate overflow recovery loop still calls `contextEngine.compact()` after model-limit errors, so `mode: "none"` can still auto-compact and retry instead of failing at the model limit as documented.
  - confidence: 0.89
- **[P2] Use the existing model-visible token accounting:** `src/agents/pi-embedded-runner/run/attempt.ts:2902-2917`
  - body: The new guard estimates raw session messages directly instead of reusing the existing precheck path that strips non-model-visible `toolResult.details` and applies the estimator safety margin. That can make `warn` or `error` fire on transcript data the model would not receive, or miss near-limit prompts.
  - confidence: 0.82

  - body: The user-facing changelog entry omits the required `Thanks @...` attribution for added bullets. Add the credited GitHub username or remove the entry until the feature is ready.
