## Summary

Keep open. Current main still exposes only `default` and `safeguard` compaction modes, so this PR is not obsolete, but the PR branch remains incorrect because the new modes are not consistently honored across prompt submission, overflow recovery, token accounting, and changelog hygiene.

## What This Changes

This PR extends agent compaction config/schema/help with `warn`, `error`, and `none`, adds a Pi pre-prompt context guard and mode-aware Pi auto-compaction disabling, lowers the reserve floor, adds a changelog entry, and adds a bundled `ceospace` skill placeholder.

## Best Possible Solution

A mergeable version should define each new mode's behavior first, then implement it consistently across exported config types, strict and generated schemas, help/docs, mode resolution, Pi auto-compaction, pre-prompt guard accounting, overflow recovery, lifecycle hooks, and focused tests, with unrelated skill additions removed from the branch.

## Reproduction Assessment

Yes for the review findings: set `agents.defaults.compaction.mode` to `warn` with an intentionally high reserve to exercise the guard, and set `none` then trigger provider context overflow to observe the separate recovery compaction path. I did not execute these because this review is read-only, but the diff and current-main control flow give a high-confidence reproduction path.

## Solution Assessment

No. The requested feature may be useful, but this PR is not the best implementation yet because it publishes new config modes before their semantics are fully enforced through prompt submission, overflow recovery, token accounting, lifecycle hooks, docs, and tests.

Full review comments:

- **[P1] Stop warn mode before submitting the prompt:** `src/agents/pi-embedded-runner/run/attempt.ts:2929-2933`
  - body: The `warn` branch emits the warning through `onBlockReply` and records assistant text, but it does not stop the later prompt path. A near-limit warn-mode turn can still call the provider even though the mode is described as warning before overflow or auto-compaction.
  - confidence: 0.93
- **[P1] Honor none mode in overflow recovery:** `src/agents/pi-settings.ts:109`
  - body: Disabling Pi auto-compaction for `none` does not disable OpenClaw's separate overflow recovery loop, which still calls `contextEngine.compact()` after a model-limit error. Thread the resolved mode into overflow recovery so `none` actually fails at the model limit as documented.
  - confidence: 0.89
- **[P2] Reuse model-visible token accounting for the guard:** `src/agents/pi-embedded-runner/run/attempt.ts:2902-2917`
  - body: The new guard estimates raw session messages directly instead of reusing the existing precheck accounting that strips non-model-visible `toolResult.details` and applies the estimator safety margin. That can make `warn` or `error` fire on transcript data the model would not receive, or miss near-limit prompts.
  - confidence: 0.82
- **[P3] Add the required changelog attribution:** `CHANGELOG.md:10`
  - body: The user-facing changelog entry omits the required `Thanks @...` attribution. Add the credited GitHub username or remove the entry until the feature is ready.
