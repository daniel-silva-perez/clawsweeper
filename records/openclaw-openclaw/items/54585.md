## Summary

Keep open. Current main still lacks the requested compaction modes, so this PR is not obsolete, but the PR head is not merge-ready because the proposed warn/none semantics do not line up with the runtime paths and the branch is stale against later compaction work.

## What This Changes

The PR extends agent compaction config/schema/help with `warn`, `error`, and `none` modes, adds a Pi pre-prompt context guard, changes Pi auto-compaction and reserve-floor behavior, adds a changelog entry, and adds a bundled `ceospace` skill placeholder.

## Best Possible Solution

A mergeable version should rebase onto current main, settle exact semantics for each mode, then implement them through the existing config, schema, docs/help, Pi mode resolution, precheck token accounting, overflow recovery, hooks/epilogue, and focused tests while keeping unrelated skill additions out of the compaction PR.

## Reproduction Assessment

Yes. The warn-mode scenario from the PR body can be reproduced by setting a high reserve to trigger the guard; source inspection of the PR head shows the warning path leaves `skipPromptForCompactionGuard` false. For `none`, triggering context overflow would still reach the current overflow recovery path that calls `contextEngine.compact()`.

## Solution Assessment

No. The feature direction may be useful, but this implementation is not the best fix because it adds a second guard instead of extending the existing precheck/overflow machinery and does not consistently honor the new modes across all compaction paths.
