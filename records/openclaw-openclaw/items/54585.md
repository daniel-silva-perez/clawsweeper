## Summary

Keep open. Current main does not implement the requested `warn`, `error`, or `none` compaction modes, and the submitted PR remains useful but not merge-ready because its runtime semantics still conflict with the documented behavior for `warn` and `none`.

## What This Changes

This PR extends agent compaction config/schema/help with `warn`, `error`, and `none`, adds a Pi pre-prompt context guard and mode-aware Pi auto-compaction disabling, lowers the reserve floor, adds a changelog entry, and adds a bundled `ceospace` skill placeholder.

## Best Possible Solution

A mergeable version should settle the exact semantics for each new mode, then implement them consistently across exported config types, strict/generated schemas, docs/help, mode resolution, Pi auto-compaction, pre-prompt guard accounting, overflow recovery, lifecycle hooks, and focused tests while keeping unrelated skill additions out of the compaction PR.

## Reproduction Assessment

Yes for the review findings: configure `agents.defaults.compaction.mode` to `warn` with an intentionally high reserve to exercise the guard, and configure `none` then trigger provider context overflow to observe the existing recovery compaction path. I did not execute these because this review is explicitly read-only, but the diff and current-main control flow give a high-confidence reproduction path.

## Solution Assessment

No. The requested direction may be useful, but the current PR is not the best implementation because it adds public modes without fully threading their semantics through prompt submission, overflow recovery, model-visible token accounting, lifecycle hooks, docs, tests, and changelog hygiene.

  - body: The `warn` branch emits the warning through `onBlockReply` and records assistant text, but it does not stop the later prompt path. A near-limit warn-mode turn can still call the provider even though the mode is described as warning instead of overflowing or auto-compacting.
  - confidence: 0.93
- **[P1] Honor none mode in overflow recovery:** `src/agents/pi-settings.ts:109`
  - body: Disabling Pi auto-compaction for `none` does not disable OpenClaw's separate overflow recovery loop, which still calls `contextEngine.compact()` after a model-limit error. Thread the resolved mode into overflow recovery so `none` actually fails at the model limit as documented.
  - confidence: 0.89
- **[P2] Reuse model-visible token accounting for the guard:** `src/agents/pi-embedded-runner/run/attempt.ts:2902-2917`
  - body: The new guard estimates raw session messages directly instead of reusing the existing precheck accounting that strips non-model-visible `toolResult.details` and applies the estimator safety margin. That can make `warn` or `error` fire on transcript data the model would not receive, or miss near-limit prompts.
  - confidence: 0.82
