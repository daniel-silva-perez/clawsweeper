## Summary

Keep open. Current main still exposes only the existing two compaction modes, and the PR remains useful source work but is not merge-ready because the proposed mode semantics are inconsistent with the runtime paths it changes.

## What This Changes

The PR extends agent compaction config/schema/help with `warn`, `error`, and `none`, adds a Pi pre-prompt token guard, changes Pi auto-compaction disabling and reserve-floor behavior, adds a changelog entry, and adds a bundled `ceospace` skill placeholder.

## Best Possible Solution

A mergeable version should first settle the exact public semantics for each mode, then implement them consistently across exported config types, strict/generated schemas, public docs/help, Pi mode resolution, auto-compaction disabling, pre-prompt guard accounting, overflow recovery, lifecycle hooks, and focused tests while keeping unrelated bundled skill work out of the compaction PR.

  - confidence: 0.89

  - body: The new guard estimates raw `activeSession.messages` and prompt messages directly.   - confidence: 0.82

Current main already caps the 20,000 floor against the active context window for small models, so the new modes should reuse that capped effective reserve instead of weakening the default floor globally.
