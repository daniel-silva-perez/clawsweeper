## Summary

Keep open. Current main still only supports `default` and `safeguard` compaction modes, while this PR's central feature is explicit `warn`, `error`, and `none` modes plus mode-specific pre-call behavior. Main has adjacent pre-prompt overflow recovery, but it still auto-compacts/truncates and does not expose the requested user controls. The PR also remains risky as-is because its own final diff has unresolved functional review concerns and an unrelated bundled skill addition.

## Best Possible Solution

Keep this PR open for maintainer review or revision. If the feature is desired, implement it as a core agents/config change: add the three modes consistently across exported types, Zod/generated schema, help/docs, and tests; make `warn`/`error` flow through normal attempt epilogue and hook semantics; make `none` bypass all automatic compaction/retry paths; sanitize token accounting; and remove unrelated skill/changelog churn from the PR.
