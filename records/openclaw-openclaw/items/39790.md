## Summary

Keep this PR open. Current main at 69fb7455c6223682a45dd8b8fb720bc3231bfa43 still lacks both intended behaviors: transcript title extraction does not strip inbound metadata, and derived session titles cannot prefer the computed outer session-row displayName. The linked #39722 report remains open/actionable, and the related open #48942 test/title refactor is not merged into current main and does not make this PR obsolete by itself.

## Best Possible Solution

Keep this PR or a successor open until main implements the narrow gateway fix: strip inbound metadata inside session title text extraction before directive stripping, pass the computed outer session-row displayName into deriveSessionTitle or equivalent title logic, and add focused regression tests for both #39722 behaviors while coordinating with #48942 if that refactor proceeds.
