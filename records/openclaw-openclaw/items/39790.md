## Summary

Keep this PR open. Current main at 955b4df0939f90b4b6de5e7e6f3149dd125e1e50 still lacks both intended behaviors: title extraction does not strip inbound metadata, and derived session titles cannot prefer the computed outer session-row displayName. The linked #39722 remains open/actionable, and open #48942 may overlap title precedence tests/refactoring but does not supersede this fix on current main.

## Best Possible Solution

Keep this PR or a successor open until main implements the narrow gateway fix: strip inbound metadata inside session title text extraction before directive stripping, pass the computed outer session-row displayName into deriveSessionTitle or equivalent title logic, and add focused regression tests for both #39722 behaviors while coordinating with #48942 if that refactor proceeds.
