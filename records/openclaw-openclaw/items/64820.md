## Summary

Keep this PR open because it targets a real Feishu group-mention crash tracked by #64783, and current main still has the implicated eager import shape. The patch is directionally useful, but it is not merge-ready against current main: the extracted parser is stale relative to current parser fields, the lazy imports bypass the repo's `*.runtime.ts` boundary rule, and the branch needs to cover the newer bot-menu handler split.

## What This Changes

The PR extracts Feishu message parsing into `message-parser.ts`, changes selected Feishu monitor dispatches to lazy-load message/card handlers, updates sequential-key parsing imports, and adds parser plus module-isolation tests.

## Best Possible Solution

Land a rebased Feishu fix that preserves the parser/import-graph break, keeps current message-context semantics, routes lazy handler loading through runtime boundary modules for all current handler paths, and proves the bundled build does not inline the lazy boundary back into the problematic chunk.

## Reproduction Assessment

Yes. The linked bug report gives concrete Feishu multi-bot group mention steps and the current source still matches the eager import graph implicated by the report; I did not run a live Feishu gateway test in this read-only pass.

## Solution Assessment

No. The parser split and lazy loading are the right direction, but this PR needs a current-main rebase, runtime-boundary lazy imports, preservation of current parser fields, and less brittle structural tests before it is the narrowest maintainable fix.
