Land a rebased Feishu plugin fix that keeps the parser/import-graph break, preserves current `FeishuMessageContext` fields, routes heavy handler loading through dedicated `*.runtime.ts` boundaries for message, reaction, card-action, and bot-menu paths, replaces fragile source-text tests with durable coverage, and proves the build emits no `[INEFFECTIVE_DYNAMIC_IMPORT]` warning.

  - body: Current main's parser carries `reply_target_message_id` and `suppress_reply_target` into `FeishuMessageContext`, and reply dispatch uses those fields to decide whether to reply to the source message.   - body: The new lazy loaders import `./bot.js` and `./card-action.js` directly. Repo rules require a dedicated `*.runtime.ts` boundary for this pattern; without one, tsdown/esbuild can keep the import ineffective and the TDZ cycle this PR is trying to break may remain in the same bundle chunk.
  - confidence: 0.84

  - body: The new module-isolation tests use `new URL(import.meta.url).pathname` before `readFileSync`. That path remains URL-encoded and is platform-shaped differently on Windows, so the tests can fail in supported workspaces with spaces or Windows paths even when the source files exist.
  - confidence: 0.82
