## Summary

Keep this issue open. Current main still lets the Docker gateway derive the default sandbox workspace under the gateway container namespace and pass that path directly as the Docker bind source. The bundled Docker sandbox setup enables sandboxing without configuring a host-visible sandbox workspace root, while the docs explicitly describe DooD as requiring host-absolute paths plus host/container path parity.

## Best Possible Solution

Keep this issue open and fix the Docker gateway sandbox path contract. The maintainer fix should either translate gateway-container sandbox roots back to host OPENCLAW_CONFIG_DIR or OPENCLAW_WORKSPACE_DIR paths before Docker creation, or have Docker setup configure agents.defaults.sandbox.workspaceRoot to a host-visible path with an identical gateway-visible parity mount. Add regression coverage proving the default Docker gateway sandbox path does not emit /home/node/.openclaw/sandboxes/... as a Docker daemon bind source on the Docker Desktop/macOS DooD setup path.
