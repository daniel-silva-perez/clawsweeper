## Summary

Keep open. Current main has the narrower #71868 mitigation for old replay markers and separates some runtime/untrusted context from the visible prompt, but it still does not implement the central requested behavior: when a current turn includes media, only current-turn images should be active by default. Recent history image blocks and media markers are intentionally preserved, and prompt image hydration still scans the submitted prompt text it receives.

## Best Possible Solution

Implement a focused embedded-runner media policy: when the current turn has attachments, pass only current-turn images by default; ignore or mask media refs inside injected memory/untrusted context before prompt-ref hydration; prune or hide recent history image blocks for image turns unless the user explicitly requests comparison; and add regression coverage for Telegram topic sessions, injected context/media refs, and recent image-heavy history.
