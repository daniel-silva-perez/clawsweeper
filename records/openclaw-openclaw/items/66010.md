## Summary

Keep open. Current main still has sub-agent spawn limits, task lifecycle tracking, run timeouts, delivery retries, and provider/tool-loop circuit-breaker-like protections, but not the requested native sub-agent cascade breaker with persisted consecutive-failure state, cooldown/tripped precheck, force/reset path, breaker notification, config schema, CLI, or Control UI surface.

## What This Changes

The issue requests a native sub-agent cascade circuit breaker with consecutive-failure tracking, configurable thresholds and cooldowns, override/reset, notification, config schema, CLI status, and Control UI visibility.

## Best Possible Solution

Keep this issue as the focused tracker for a core sub-agent breaker. The end state should persist breaker state keyed by a maintainer-approved requester/target/task signature, update it from terminal subagent outcomes, check it before sessions_spawn dispatch, support explicit force/reset with auditability, expose structured breaker errors and notifications, and document/configure it alongside existing subagent and task controls.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature request. There is a high-confidence verification path for the current gap: source and docs inspection show no breaker state, config block, spawn precheck, force/reset contract, or status surface on current main.

## Solution Assessment

Unclear whether the exact proposed shape is the best final design. A native core implementation is the right surface for spawn-time blocking, but maintainer judgment is needed on keying, failure taxonomy, half-open/cooldown semantics, UI/CLI exposure, and whether provider billing breakers stay separate.
