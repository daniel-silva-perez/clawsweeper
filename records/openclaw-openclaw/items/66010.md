## Summary

This issue should remain open. Current main still has subagent spawn guardrails and per-run/task lifecycle tracking, but no native subagent cascade circuit breaker: no consecutive-failure breaker state, no cooldown/trip precheck in `sessions_spawn`, no force/manual-reset contract, and no `agents.defaults.subagents.circuitBreaker` config or CLI/UI status surface. The discussion also includes concrete cascade/cost evidence and distinguishes adjacent provider-level breaker work from the requested subagent breaker.

## Best Possible Solution

Keep this issue open and implement the requested native subagent cascade circuit breaker in the core subagent spawn and registry path. Persist breaker state keyed by requester plus target agent and/or normalized task signature, update it on terminal subagent completion, reset on success, precheck before dispatch, expose explicit force/manual reset, add strict config/schema/docs, and surface state through task/subagent status plus Control UI. Treat provider-level rate-limit/billing breakers as adjacent follow-up work unless maintainers intentionally combine the scopes.

## Risks / Open Questions

- Provider billing/rate-limit circuit breakers are related but not the same as the requested subagent spawn breaker; provider-level cooldowns should not be treated as closing this issue.
- Breaker keying needs maintainer judgment: target-agent-only can over-block unrelated tasks, while task-signature-only can miss broken agent configuration.
- A native breaker should be observable and overrideable so reliability guardrails do not turn repeated failures into silent non-dispatches.
