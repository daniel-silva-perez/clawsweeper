## Summary

Current main still does not implement the requested native sub-agent cascade circuit breaker. The issue remains actionable core reliability work: subagent spawning has depth, active-child, allowlist, explicit-agent, sandbox, timeout, and cleanup guardrails, while registry/tasks record individual run outcomes only. There is no persisted consecutive-failure state, tripped/cooldown precheck, forced override/manual reset, half-open/probe behavior, breaker notification, config schema, CLI/docs, or Control UI surface for subagent breaker state.

## Best Possible Solution

Keep this issue open and implement a native subagent cascade breaker in the core spawn and registry path. Persist breaker state keyed by requester plus target agent and/or normalized task signature; classify failures; update state on terminal subagent completion; reset on success; block dispatch before child session creation when tripped; support explicit force/manual reset and half-open probing; return structured breaker errors to the parent; add strict config/schema/docs; and surface state through tasks/subagent status plus Control UI. Treat provider-level billing/rate-limit breakers as a related but separate follow-up unless maintainers intentionally combine the scopes.
