## Summary

Keep this PR open. Current main still lacks the abort-aware Discord lifecycle suppression that the PR adds, and a live `reconnect-exhausted` event during an abort-driven health-monitor restart still follows the fatal `stop` path. The provided PR context shows a focused, recently repaired branch with only Discord lifecycle code, its colocated regression test, and changelog touched; no security-sensitive or supply-chain changes are present.

## Best Possible Solution

Keep this PR open for maintainer review or landing. The best path is the focused Discord plugin change in `extensions/discord/src/monitor/provider.lifecycle.ts`, plus a regression test in `extensions/discord/src/monitor/provider.lifecycle.test.ts` that synchronously emits a live `reconnect-exhausted` event after the abort signal is set and verifies it is logged as expected instead of rejecting the lifecycle. No core API or broader architecture change appears necessary.
