## Summary

Keep open. Current main still lacks a ZAI-specific endpoint rewrite for vision models configured against Coding Plan base URLs, and #70915 is open and unmerged with closing syntax for this issue, so the issue should not be closed ahead of that implementation path.

## What This Changes

The issue asks OpenClaw's image tool to analyze images with ZAI vision models such as `zai/glm-4.6v` when ZAI is configured with Coding Plan endpoints instead of returning a provider 404.

## Best Possible Solution

Review and either land #70915 or replace it with an equivalent narrow ZAI-owned fix that routes only ZAI vision models from Coding Plan base URLs to the matching standard multimodal endpoint, while keeping text/coding models on the configured Coding Plan endpoint and adding regression coverage for both paths.

## Reproduction Assessment

Yes for maintainers with a ZAI key: configure ZAI with `coding-global` or `coding-cn`, set `agents.defaults.imageModel` to `zai/glm-4.6v`, and call the `image` tool. In this read-only review, reproduction is source-backed rather than live-backed because no ZAI credential was available.

## Solution Assessment

Yes, keeping this issue open is the right triage outcome because an open closing PR is the active implementation candidate. The code fix should stay in the ZAI provider/media-understanding boundary or in a narrowly tested model-routing seam, not in a broad core migration.
