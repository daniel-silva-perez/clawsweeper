## Summary

Keep open. Current main still routes ZAI media-understanding through the generic image completion path while ZAI Coding Plan onboarding gives the whole catalog a coding base URL, and an open closing PR already targets the same endpoint-routing bug.

## What This Changes

Fix the built-in image tool so ZAI vision models such as `zai/glm-4.6v` do not fail when the ZAI provider was configured with a Coding Plan endpoint.

## Best Possible Solution

Keep this issue open while maintainers review #70915 or land an equivalent narrow ZAI-provider fix. The final implementation should route only ZAI vision models from Coding Plan base URLs to the matching standard multimodal endpoint, preserve text/coding models on Coding Plan endpoints, and add regression coverage for both global and CN endpoint choices.

## Reproduction Assessment

Yes. Maintainers can reproduce by configuring ZAI with a Coding Plan endpoint, setting `agents.defaults.imageModel` to `zai/glm-4.6v`, and invoking the `image` tool; this review verifies the route source-backed, not live-backed, because no ZAI key was present.

## Solution Assessment

Yes for the triage outcome: keeping the issue open is the right path because #70915 is an open closing PR. The code fix should stay in the ZAI provider/media-understanding boundary rather than a broad core migration.
