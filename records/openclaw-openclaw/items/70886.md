## Summary

Keep open. Current `main` still has the same ZAI image route: Coding Plan endpoint selection is stored as the provider-level `baseUrl`, ZAI vision catalog entries do not override it, and the ZAI media provider still delegates directly to the generic image helpers. Open PR #70915 is an active closing implementation candidate, so this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue requests fixing the built-in image tool so `agents.defaults.imageModel` values like `zai/glm-4.6v` work with ZAI Coding Plan configurations instead of returning 404.

## Best Possible Solution

Review and land or replace #70915 with a narrow ZAI-owned route fix that sends known ZAI vision models from Coding Plan base URLs to the matching standard multimodal endpoint while leaving text/coding models on the configured Coding Plan route, with regression coverage for global and CN endpoints.

## Reproduction Assessment

Yes. Configure ZAI with `zai-coding-global` or `zai-coding-cn`, set `agents.defaults.imageModel` to `zai/glm-4.6v`, and call the image tool; current source still leaves the coding base URL on the resolved image model.

## Solution Assessment

Yes for triage: keeping this issue open while the closing PR remains open is the right path. The final implementation should stay in the ZAI provider/media-understanding boundary or a narrow provider-routing seam, not a broad core migration.
