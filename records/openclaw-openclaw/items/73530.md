## Summary

Current main already fixes the reported bundled plugin runtime-deps mirror churn. The v2026.4.26 code rebuilt external runtime mirrors by deleting/renaming plugin trees during load, while current main records mirror metadata, skips fresh mirrors, refreshes in place, and has regression coverage for reloads that must not rewrite unchanged mirrors. The supplied related reports are adjacent runtime-deps install/repair cases, but this report's fs link/unlink CPU profile maps to the mirror refresh path that was fixed after v2026.4.26.

## Best Possible Solution

Keep the main-branch mirror metadata and in-place refresh implementation, ship it in the next package/release, and reopen or file a new focused bug only if the CPU/link-unlink loop reproduces on a build containing `92016b82` and `32c98762`.
