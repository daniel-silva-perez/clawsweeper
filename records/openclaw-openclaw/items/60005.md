## Summary

Keep open. Current main still lacks MCP resource support, so the PR is not obsolete, but the proposed branch still has a blocking secret-exposure path in the MCP config resource and needs to be rebased onto the current plugin-tools handler seam before it can be considered mergeable.

## What This Changes

The PR adds MCP resources for channel status/conversations and plugin tool/config inspection, plus resource handlers and tests for the plugin-tools server.

## Best Possible Solution

Rebase the feature onto current main, add resources through the existing MCP server helper/handler structure, serialize config resources only through the shared config redaction path, and add regression coverage for arbitrary secret-like MCP server fields plus sensitive URL query keys before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Use shared redaction for MCP config resources:** `src/mcp/plugin-resources.ts:105-143`
  - body: `openclaw://config/mcp` clones each server config and only masks `headers`, `env`, and selected URL pieces before returning the rest of the object. Since MCP server entries allow arbitrary keys, fields such as `apiKey`, `authToken`, or `clientSecret` would be exposed to any MCP client reading this resource; route this payload through the existing config redaction helper instead.
  - confidence: 0.92
- **[P2] Preserve the current plugin-tool handler seam:** `src/mcp/plugin-tools-serve.ts:86-90`
  - body: The branch's plugin-tools server still calls `tool.execute` directly. Current main now routes this server through `createPluginToolsMcpHandlers`, which filters `ownerOnly` tools and enforces `before_tool_call` approvals, so the resource rebase needs to keep that helper instead of restoring the older direct execution path.
  - confidence: 0.77

## Security Review

Status: needs_attention

Summary: The PR introduces a security-sensitive config inspection resource that can still leak secrets unless it uses the shared config redaction path and preserves current tool approval hooks.

Concerns:

- **[high] MCP config resource can leak custom secret fields:** `src/mcp/plugin-resources.ts:105`
  - body: The proposed `openclaw://config/mcp` resource returns arbitrary MCP server properties after masking only a few known fields. Config entries allow additional properties, so auth-like custom fields can be exposed to MCP clients.
  - confidence: 0.92
- **[medium] Stale tool handler can bypass approval hooks:** `src/mcp/plugin-tools-serve.ts:86`
  - body: The PR branch predates the current handler split and contains direct plugin tool execution. If rebased by keeping that shape, plugin-tool owner filtering and `before_tool_call` approval enforcement would be bypassed.
  - confidence: 0.77
