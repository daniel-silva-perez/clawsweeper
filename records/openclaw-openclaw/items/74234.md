## Summary

Keep this issue open. Current main still validates only the first Docker apt key fingerprint before dearmoring the whole key file, while open PR #74254 is the concrete implementation candidate that adds the requested single-primary-key guard and focused test coverage.

## What This Changes

The issue asks Dockerfile GPG key validation to reject Docker apt key files containing more than one primary public key instead of validating only the first fingerprint.

## Best Possible Solution

Review and land PR #74254 or an equivalent narrow Dockerfile hardening change that counts exactly one primary `pub` key before fingerprint comparison and before `gpg --dearmor`, keeps the focused Dockerfile test, and validates both the official Docker Debian key path and a multi-primary-key rejection case. After that merges, the issue can close as implemented.
