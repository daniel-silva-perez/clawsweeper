## Summary

Keep this issue open. Current main still validates the first `fpr` record from Docker's apt key and then dearmors the entire downloaded key file, so a multi-primary-key file would not be rejected before trust setup. The linked open PR #74254 is the concrete fix candidate and should be reviewed or replaced by an equivalent narrow hardening change.

## What This Changes

Require the Dockerfile Docker apt key validation to reject downloaded GPG key files containing more than one primary public key before dearmoring them.

## Best Possible Solution

Land PR #74254 or an equivalent narrow Dockerfile hardening change that counts exactly one primary `pub` key before fingerprint comparison and `gpg --dearmor`, with focused test coverage for the official Docker key and a multi-primary-key rejection case.
