## Summary

Keep this PR open, but it is not merge-ready. The hardening direction is valid, yet the branch adds several schema maxima that are lower than current-main runtime/protocol contracts, so legitimate tool calls would be rejected or timed out before reaching working code.

## What This Changes

The PR adds TypeBox descriptions plus maxItems and numeric min/max bounds across built-in agent tool schemas for browser, canvas, gateway, memory, message, nodes, sessions, subagents, and web_fetch.

## Best Possible Solution

Land a rebased hardening patch that encodes only contract-backed limits, applies browser bounds to the active browser plugin schema, preserves current long-running/session/media workflows, and adds focused schema contract tests for the affected tools.

## Reproduction Assessment

Yes. Static review reproduces the regressions by comparing the PR's proposed schema maxima with current-main runtime, protocol schema, docs, and provider-backed behavior for node recordings, sessions.list, sessions_spawn, browser snapshots, and Discord media channels.

## Solution Assessment

No. Schema-only hardening is the right direction, but this branch is not the best implementation because several caps are too low for supported behavior and one browser change targets a stale core path instead of the active plugin schema.
