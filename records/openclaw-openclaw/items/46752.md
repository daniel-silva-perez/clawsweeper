## Summary

Keep this PR open. The schema-hardening direction is useful, but current main confirms several proposed bounds would reject supported node media, sessions_list, subagent, browser, and Discord media-thread workflows; the branch also still edits an obsolete core browser schema path while the active browser tool schema now lives in the browser plugin.

## What This Changes

The PR adds TypeBox descriptions plus maxItems and numeric min/max bounds across built-in agent tool schemas for browser, canvas, gateway, memory, message, nodes, sessions, subagents, and web_fetch.

## Best Possible Solution

Land a rebased hardening patch that encodes only contract-backed limits, applies browser bounds to the active browser plugin schema, preserves existing long-running/session/media workflows, and adds focused schema contract tests for the affected tools.

## Reproduction Assessment

Yes. Static review reproduces the regressions by comparing the PR's proposed schema maxima with current-main runtime code, protocol schemas, docs, and regression tests for node recordings, sessions.list, sessions_spawn, browser snapshots, and Discord media threads.

## Solution Assessment

No. Schema-only hardening is a maintainable direction, but this branch is not the best implementation because several maxima are below supported behavior and one browser change targets a stale schema path instead of the active plugin-owned schema.

## Review Findings

Full review comments:

- **[P1] Raise node timeouts to cover recordings:** `src/agents/tools/nodes-tool.ts:113-114`
  - body: The new timeoutMs maximum is below valid media durations: current main keeps durationMs capped at 300000ms and screen_record clamps to that value, so recordings over 120s cannot set a gateway timeout long enough to complete.
  - confidence: 0.93
- **[P2] Keep sessions_list compatible with 200-row lookups:** `src/agents/tools/sessions-list-tool.ts:36-37`
  - body: Capping sessions_list.limit at 50 rejects a pattern OpenClaw still uses internally: announce-target resolution asks sessions.list for 200 rows, and the gateway protocol accepts larger positive limits.
  - confidence: 0.9
- **[P2] Preserve long-running subagent timeouts:** `src/agents/tools/sessions-spawn-tool.ts:44-46`
  - body: The runtime accepts any finite non-negative runTimeoutSeconds, with 0 meaning no timeout. Adding maximum: 3600 rejects legitimate subagent or ACP jobs intended to run past an hour.
  - confidence: 0.91
- **[P2] Harden the active browser schema:** `src/agents/tools/browser-tool.schema.ts:98-102`
  - body: Current main registers the browser tool from extensions/browser, so editing the old core browser schema path leaves the active surface unchanged. The proposed 1000 snapshot cap is also below the backend cap of 2000.
  - confidence: 0.84
- **[P2] Permit Discord media channel types:** `src/agents/tools/message-tool.ts:329`
  - body: The type maximum blocks Discord GuildMedia, which the current Discord runtime treats as forum-like and tests cover for thread creation. Valid media-channel thread behavior would fail schema validation.
  - confidence: 0.88
