## Summary

Keep this PR open, but do not merge it as-is. The hardening goal is valid and current main still has unbounded tool-schema fields, but this branch adds several caps that are lower than current supported runtime/protocol behavior and also edits an obsolete browser schema path.

## What This Changes

The PR adds TypeBox descriptions plus maxItems and numeric minimum/maximum metadata across built-in agent tool schemas for browser, canvas, gateway, memory, message, nodes, sessions, subagents, and web_fetch.

## Best Possible Solution

Land a rebased hardening patch that encodes only limits backed by runtime, protocol, dependency, or provider contracts; moves browser schema changes to the active browser plugin; preserves current long-running workflows; and adds focused schema contract tests for the affected tools.

## Reproduction Assessment

Yes. Static review reproduces the regressions by comparing the PR's new schema maxima with current-main runtime and protocol paths for node recordings, sessions.list, subagent timeouts, active browser registration, and Discord GuildMedia support.

## Solution Assessment

No. Schema-only hardening is the right direction, but this branch is not the narrowest maintainable solution because several caps are below current valid behavior and one browser change targets a moved file path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Raise node timeouts to cover recordings:** `src/agents/tools/nodes-tool.ts:113-114`
  - body: The new 120s timeoutMs maximum is lower than valid media durations: nodes still allow 300000 ms recordings and screen_record can run for that full duration. A caller recording near the max can no longer set a gateway timeout long enough for the operation to complete.
  - confidence: 0.93
- **[P2] Keep sessions_list compatible with 200-row lookups:** `src/agents/tools/sessions-list-tool.ts:36-37`
  - body: Capping sessions_list.limit at 50 rejects a pattern OpenClaw still uses internally: announce-target resolution asks sessions.list for 200 rows, and the gateway protocol has no maximum. Larger installs can lose the ability to find older or less-active sessions.
  - confidence: 0.9
- **[P2] Preserve long-running subagent timeouts:** `src/agents/tools/sessions-spawn-tool.ts:44-46`
  - body: The runtime still allows any non-negative runTimeoutSeconds value, with 0 meaning no timeout. Adding maximum: 3600 makes legitimate subagent jobs intended to run past an hour fail schema validation before reaching the existing runtime behavior.
  - confidence: 0.91
- **[P2] Move browser hardening to the active schema:** `src/agents/tools/browser-tool.schema.ts:98`
  - body: This branch edits the old browser schema path and lowers snapshot limit to 1000, while current main registers the plugin-owned browser schema. A rebased patch should harden the active schema without reducing supported snapshot size.
  - confidence: 0.84
- **[P2] Permit Discord media channel types:** `src/agents/tools/message-tool.ts:329`
  - body: Discord's GuildMedia channel type is supported by the current Discord runtime as forum-like. Capping channel management type at 15 blocks valid channel-create/thread behavior for that supported channel type.
  - confidence: 0.88
