## Summary

Keep this PR open, but it should not merge as-is. Current main still lacks the requested broad schema hardening, so the contribution is not obsolete; however, several proposed maxima are below current valid runtime/protocol behavior, and the browser changes target a stale core path instead of the active bundled browser plugin schema.

## What This Changes

The PR adds TypeBox descriptions plus maxItems and numeric minimum/maximum metadata across built-in agent tool schemas for browser, canvas, gateway, memory, message, nodes, sessions, subagents, and web_fetch.

## Best Possible Solution

Land a rebased hardening patch that encodes only limits backed by runtime, protocol, dependency, or provider contracts; moves browser bounds to the active browser plugin schema; preserves current long-running workflows; and adds focused schema contract tests for the affected tools.

## Reproduction Assessment

Yes. Static review reproduces the regressions by comparing the PR's new schema maxima with current-main runtime, protocol, docs-adjacent behavior, and tests for node recordings, sessions.list, subagent timeouts, browser snapshots, and Discord media threads.

## Solution Assessment

No. Schema-only hardening is the right direction, but this branch is not the best maintainable solution because several caps are too low for current behavior and one browser change targets an inactive path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Raise node timeouts to cover recordings:** `src/agents/tools/nodes-tool.ts:113-114`
  - body: The new timeoutMs maximum is lower than valid media durations: nodes still allow 300000 ms recordings and screen_record can run for that full duration. A caller recording near the max can no longer set a gateway timeout long enough for the operation to complete.
  - confidence: 0.93
- **[P2] Keep sessions_list compatible with 200-row lookups:** `src/agents/tools/sessions-list-tool.ts:36-37`
  - body: Capping sessions_list.limit at 50 rejects a pattern OpenClaw still uses internally: announce-target resolution asks sessions.list for 200 rows, and the gateway protocol has no maximum. Larger installs can lose the ability to find older or less-active sessions.
  - confidence: 0.9
- **[P2] Preserve long-running subagent timeouts:** `src/agents/tools/sessions-spawn-tool.ts:44-46`
  - body: The runtime still allows any non-negative runTimeoutSeconds value, with 0 meaning no timeout. Adding maximum: 3600 makes legitimate subagent jobs intended to run past an hour fail schema validation before reaching the existing runtime behavior.
  - confidence: 0.91
- **[P2] Harden the active browser schema:** `src/agents/tools/browser-tool.schema.ts:98`
  - body: This branch edits the old browser schema path, while current main registers the plugin-owned browser schema. It also lowers snapshot limit to 1000 even though current snapshot backends still accept up to 2000 nodes, so the active surface remains unhardened and supported requests would be rejected if ported directly.
  - confidence: 0.84
- **[P2] Permit Discord media channel types:** `src/agents/tools/message-tool.ts:329`
  - body: The type maximum of 15 blocks Discord GuildMedia, which the current Discord runtime treats as forum-like and tests cover for thread creation. Valid media-channel thread behavior would fail schema validation.
  - confidence: 0.88
