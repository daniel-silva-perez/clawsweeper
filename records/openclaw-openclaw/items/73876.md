## Summary

Current main still does not accept the reported direct `moonshotai/kimi-k2.6` form. Provider parsing splits on the first slash and normalizes only the provider segment; `normalizeProviderId` has no `moonshotai` or `moonshot-ai` alias, while the Moonshot plugin, catalog, auth metadata, onboarding, and docs remain canonicalized as `moonshot`. Related PR #73934 proposed the narrow alias fix but is closed unmerged, so this remains valid and actionable.

## Best Possible Solution

Keep this issue open and implement a backward-compatible provider alias: normalize direct `moonshotai/<model>` and likely `moonshot-ai/<model>` to canonical provider `moonshot`, while preserving the existing `moonshot` plugin id, auth profile provider, config keys, onboarding output, and docs as canonical. Add regression tests and a short docs/changelog note; #73934 is useful prior art but is closed unmerged.

## Clownfish Work Prompt

Fix the direct Moonshot provider slug compatibility reported in this issue. Current main parses `moonshotai/kimi-k2.6` as provider `moonshotai` and model `kimi-k2.6`, but the bundled direct provider is canonical `moonshot`, so users copying OpenRouter's `moonshotai/...` org slug get `Unknown model` and may fall back to another provider.

Implement a backward-compatible alias, preferably in the existing provider normalization path: accept direct leading provider ids `moonshotai` and `moonshot-ai` by normalizing them to canonical `moonshot`. Preserve the existing canonical provider/plugin/auth/config id `moonshot`; do not rename the Moonshot plugin, auth profiles, `models.providers.moonshot`, env vars, onboarding output, or existing docs examples away from `moonshot`. Do not alter nested proxy model ids: refs like `openrouter/moonshotai/kimi-k2.6`, `together/moonshotai/Kimi-K2.5`, and `deepinfra/moonshotai/Kimi-K2.5` must keep provider `openrouter`/`together`/`deepinfra` and model `moonshotai/...`.

Use #73934 as prior art, but make the patch fresh on current main. Add regression tests for `normalizeProviderId`, `parseModelRef('moonshotai/kimi-k2.6', ...)`, case variants, `moonshot-ai`, canonical `moonshot`, and a nested proxy ref such as `openrouter/moonshotai/kimi-k2.6`. If docs are touched, document `moonshot` as canonical and `moonshotai` as a compatibility alias rather than a rename. Add a single-line changelog fix entry under Unreleased/Fixes with proper thanks attribution, likely thanking @jimdawdy-hub and @hclsys if carrying forward the closed PR's prior work.

Do not change unrelated provider aliases, provider pricing metadata, package versions, lockfiles, or release metadata.
