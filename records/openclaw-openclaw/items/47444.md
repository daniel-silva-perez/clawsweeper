## Summary

Keep open. Current main has broad assistant-visible scaffolding sanitizers, but the exact reported XML-style `<function_calls><invoke ...>` plus `<function_response>` shape is not covered. The sanitizer strips JSON-like `<function_calls>` blocks, while a current regression test explicitly preserves XML payload examples with nested `<invoke>`, and no sanitizer path recognizes `<function_response>` as an internal block.

## Best Possible Solution

Implement a narrow assistant-visible sanitizer extension for standalone XML-style `<function_calls>` blocks with nested `<invoke>` payloads and paired `<function_response>` blocks, then add regression coverage through `sanitizeAssistantVisibleText`/`extractAssistantVisibleText` and at least one outbound or Telegram delivery path. Preserve fenced code and clearly literal prose examples, or update the existing preservation test to encode the intended boundary.
