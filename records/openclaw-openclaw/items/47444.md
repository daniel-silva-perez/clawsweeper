## Summary

Keep open. Current main has shared assistant-visible sanitizers for JSON-style `<function_calls>` and singular `<function>` XML payloads, but the exact reported XML-style plural `<function_calls><invoke ...>` plus `<function_response>` shape is still not covered; a current regression test explicitly preserves that nested XML form.

## Best Possible Solution

Keep this open and implement a narrow assistant-visible sanitizer extension for standalone XML-style `<function_calls>` blocks with nested `<invoke>` payloads plus paired `<function_response>` blocks. Add regression coverage through `sanitizeAssistantVisibleText` / `extractAssistantVisibleText` and at least one outbound or Telegram delivery path, while preserving fenced code and clearly literal prose examples.
