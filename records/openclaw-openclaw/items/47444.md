## Summary

Current main still has a sanitizer gap for the exact reported workflow leak shape: plural XML-style `<function_calls><invoke ...>` payloads are preserved, and `<function_response>` is not handled anywhere in the shared assistant-visible sanitizer or Telegram delivery path. Related sanitizer work in #60619 and later XML-tool-call hardening covers adjacent forms, but not this issue's nested plural XML plus response envelope.

## Best Possible Solution

Keep this open. Add a narrow assistant-visible sanitizer extension for standalone XML-style plural `<function_calls>` blocks with nested `<invoke>` payloads and paired `<function_response>` blocks. Cover it in `sanitizeAssistantVisibleText` / `extractAssistantVisibleText` tests and one outbound or Telegram delivery-path test, while preserving fenced code and literal prose examples.
