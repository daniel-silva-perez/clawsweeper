## Summary

Keep this issue open. Current main still skips the runtime primary workspace whenever `resolveMemoryDreamingWorkspaces(cfg)` returns any configured agent workspaces, which matches the reported AGI-subagent-only dreaming run.

## What This Changes

The issue reports that scheduled memory-core dreaming processes configured AGI sub-agent workspaces but omits the primary agent workspace at the default dreaming time.

## Best Possible Solution

Merge the runtime primary/default workspace with configured agent workspaces in the shared dreaming workspace collection path, keep the result deduped by normalized path, and align runtime dreaming, doctor status, artifact listing, and regression tests around that single contract.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a config whose `agents.list` contains only sub-agent workspaces while the runtime/default primary workspace is supplied through `ctx.workspaceDir` or `agents.defaults.workspace`; current code processes only the configured list and never adds the primary unless that list is empty.

## Solution Assessment

Yes. The narrow maintainable fix is to merge the runtime primary workspace into the configured dreaming workspace set with normalized dedupe, rather than adding AGI-specific IDs or changing unrelated dreaming phase behavior.
