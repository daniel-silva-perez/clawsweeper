## Summary

Keep this PR open. Current main still does not accept `/activate` as an alias for `/activation`: the parser, command registry, tests, and docs all remain `/activation`-only. The PR’s corrected final regex is directionally useful and low security risk, but a complete fix should also account for command detection/normalization because WhatsApp group mention bypass currently depends on `hasControlCommand()`.

## Best Possible Solution

Review and land this PR or an equivalent fix, but make it complete: accept `/activate` in `parseActivationCommand()`, add regression tests for `/activate` and `/activate:` variants, and consider adding `/activate` to the activation command `textAliases` plus user-facing docs so WhatsApp group gating and command discovery recognize the alias consistently.
