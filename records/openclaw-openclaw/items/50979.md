## Summary

Keep this PR open. Current main still does not implement the PR's final intended `messageToolHints` change, and the related merged Slack mrkdwn work is not equivalent: it adds inbound response-format metadata, while this PR targets outbound message-tool guidance. The PR also still needs maintainer review because plain `message` text is Markdown-converted for Slack, but Slack presentation/native block text is sent as `mrkdwn`, so an unconditional “use standard Markdown” hint would be too broad.

## Best Possible Solution

Revise this PR or replace it with a maintainer-scoped fix: add outbound guidance that says plain `message` text should use standard Markdown because OpenClaw converts it for Slack, while Slack presentation/native block text is rendered as Slack `mrkdwn`. Reconcile that wording with the existing inbound `response_format` metadata from #59100 so the prompt surfaces do not contradict each other.
