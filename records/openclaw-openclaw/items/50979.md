## Summary

Keep this PR open. Current main still has no outbound Slack formatting guidance in `messageToolHints`, and the related merged inbound mrkdwn work does not implement this PR's message-tool prompt change. The PR's final wording also still needs scoping because plain Slack message text is Markdown-converted, while Slack presentation/native blocks are sent as mrkdwn.

## Best Possible Solution

Keep the PR open, or replace it with a maintainer-scoped fix that distinguishes plain `message` text from presentation/native block text: plain Slack message text should use standard Markdown because OpenClaw converts it, while Slack presentation/native block text should use Slack mrkdwn. Reconcile that wording with the inbound formatting metadata from #59100 and update the Slack prompt tests accordingly.
