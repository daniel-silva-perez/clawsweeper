## Summary

Keep this PR open. Current main does not contain the PR’s final intended change: Slack `messageToolHints` still only cover interactive replies, while the PR now proposes standard-Markdown outbound guidance. The current tree also has later Slack `inboundFormattingHints` that say to use Slack mrkdwn, so this area needs maintainer review rather than cleanup closure.

## Best Possible Solution

Do not close this PR as implemented. A maintainer should review whether Slack outbound prompt guidance should be added, then scope it carefully: standard Markdown for plain-text sends that pass through `markdownToSlackMrkdwnChunks`, and explicit mrkdwn guidance for raw Slack Block Kit `blocks` if agents are expected to author those payloads. The existing inbound `response_format` wording should be checked at the same time so Slack prompts do not give contradictory formatting instructions.
