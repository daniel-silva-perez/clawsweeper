## Summary

Keep this PR open. Current main still does not contain the PR's final intended `messageToolHints` change, and the later merged #59100 is related but not equivalent: it adds inbound Slack mrkdwn guidance while the PR now proposes outbound standard-Markdown guidance. The correct fix needs maintainer review because plain-text Slack sends are Markdown-converted, but Slack Block Kit `blocks` are forwarded directly and still expect mrkdwn inside block text objects.

## Best Possible Solution

Do not close this PR as implemented. A maintainer should either revise and land it with scoped wording, or close it through normal review if they choose #59100's inbound-metadata approach instead. The likely best code direction is to document standard Markdown for plain-text Slack message content that passes through `markdownToSlackMrkdwnChunks`, preserve explicit mrkdwn guidance for raw Slack Block Kit `blocks`, and reconcile `inboundFormattingHints` so Slack prompts do not contradict the outbound renderer.
