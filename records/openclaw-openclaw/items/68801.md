## Summary

Keep this PR open. Current main still does not prune orphaned `agentRunStarts`, so the PR targets a real remaining leak, but the proposed pure TTL sweep can delete the fallback `startedAt` for legitimate long-running active runs before terminal lifecycle events arrive. The best cleanup needs to preserve active-run timestamps or make terminal events reliably carry `startedAt`, with focused regression coverage.

## Best Possible Solution

Revise and keep this PR as the implementation candidate: add a cleanup path that distinguishes orphaned/inactive `agentRunStarts` entries from still-active long-running runs, or first ensure terminal lifecycle emitters reliably include `startedAt`. Add fake-timer regression coverage for both a lost terminal event and a run that ends after the TTL without `startedAt`, then route to gateway maintainers for review.
