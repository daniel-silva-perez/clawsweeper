## Summary

Keep this PR open. Current main still only prunes `agentRunCache`; `agentRunStarts` is populated on lifecycle start and removed only on lifecycle end/error, so the orphaned-start leak described by the PR remains possible. The PR is a focused implementation candidate, but the review comments identify a real correctness concern: the proposed 10-minute age sweep can also delete starts for still-active long-running runs, and current tests show terminal lifecycle events may omit `startedAt` and rely on the `agentRunStarts` fallback.

## Best Possible Solution

Revise and keep this PR as the implementation candidate: add a safe cleanup path for orphaned `agentRunStarts` entries without deleting active long-running start timestamps, or first make terminal lifecycle emitters reliably carry `startedAt`. Add focused regression tests around `waitForAgentJob`/`pruneAgentRunCache()` that cover a lost terminal event and a run whose terminal event arrives after the cache TTL without `startedAt`.

## Risks / Open Questions

- The proposed PR patch would bound `agentRunStarts`, but a pure age-based sweep at `AGENT_RUN_CACHE_TTL_MS` can remove the fallback start timestamp for a legitimate run that lasts longer than 10 minutes and ends with a terminal event that omits `startedAt`.
- There is no direct unit test around `pruneAgentRunCache()` or orphaned `agentRunStarts` cleanup yet, so the final fix should add focused fake-timer coverage for both orphan cleanup and long-running active-run preservation.
