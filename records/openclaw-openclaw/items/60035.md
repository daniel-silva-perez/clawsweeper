## Summary

Keep open. Current main now has substantial HTTP forward-proxy support via `proxy.enabled`, `proxy.proxyUrl`, `OPENCLAW_PROXY_URL`, and standard env proxy bootstrapping, but it does not fully satisfy this issue's requested HTTP/SOCKS plus per-tool/per-agent proxy model. SOCKS is explicitly rejected, web_search/web_fetch config still has no proxy override field, and the managed browser intentionally does not inherit gateway/provider proxy env vars.

## Best Possible Solution

Keep this issue open as the canonical follow-up for the remaining proxy gaps. The shipped HTTP forward-proxy support should stay, but maintainers should either scope this issue down to SOCKS plus per-tool/per-agent overrides, or split narrower issues for top-level SOCKS support, web tool proxy overrides, and browser proxy UX that preserves SSRF safety.
