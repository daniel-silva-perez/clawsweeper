Current main still has the reported budget mismatch: the embedded attempt path resolves and threads `contextTokenBudget` into session/context-engine budgeting, but the default tool-result guard is still installed from `model.contextWindow`. ## What This Changes

The PR changes `runEmbeddedAttempt` so the default tool-result context guard prefers `params.contextTokenBudget` before falling back to the model context window, max tokens, or default token budget.

## Best Possible Solution

Land the targeted fallback change, preferably with a small attempt-level regression asserting that the default tool-result guard receives `contextTokenBudget` when it differs from `model.contextWindow`, then let the linked bug close through the PR merge.
