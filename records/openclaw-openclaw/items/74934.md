## Summary

Keep open for normal maintainer review. Current `main` still installs the default tool-result context guard from `params.model.contextWindow` while the surrounding attempt/runtime paths use the resolved `contextTokenBudget`, and this PR is a narrow one-call-site fix for the linked behavior bug; the later related PR #74978 was closed unmerged, so it does not supersede this PR.

## What This Changes

The PR changes `runEmbeddedAttempt` so the default tool-result context guard prefers `params.contextTokenBudget` before falling back to model `contextWindow`, `maxTokens`, or the default token budget.

## Best Possible Solution

Land the targeted fallback change, preferably with a small wiring regression for the attempt call site, then let the linked bug #74917 close through the PR merge.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by resolving a model where `contextTokenBudget` exceeds `model.contextWindow` and observing that current `main` installs the default guard with the smaller model window.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix if OpenClaw treats `contextTokens` as the effective runtime budget, because it reuses the already-resolved budget instead of adding new policy or normalization logic.
