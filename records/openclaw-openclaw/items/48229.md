## Summary

Current main does not safely qualify for closure. The WebSocket path still clears requested scopes for device-less shared-token/password operator connections, and the provided GitHub context shows an open PR explicitly using closing syntax for this issue. That PR should be reviewed or closed before this issue is resolved.

## Best Possible Solution

Keep this issue open. The best path is to review PR #57889 or another focused gateway fix that explicitly decides whether token-authenticated device-less WebSocket operator clients may retain `operator.write` on non-loopback/LAN deployments, adds targeted coverage for that contract, and avoids weakening the existing device-identity protections unintentionally.
