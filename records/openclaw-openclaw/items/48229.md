## Summary

Current main still preserves the reported failure mode for ordinary device-less WebSocket operator clients authenticated by shared token: the connection can authenticate, but requested operator scopes are cleared before RPC authorization. The HTTP default-scope fix mentioned in discussion is a different path, and an open implementation PR still references this issue with closing syntax.

## Best Possible Solution

Keep this issue open. Review PR #57889 or replace it with a focused gateway change that explicitly decides the contract for token-authenticated, device-less WebSocket operator clients on LAN/Docker; add a regression covering `gateway.bind: "lan"`, token auth, requested `operator.admin`/`operator.write`, and a write-scoped RPC, while preserving the existing device-identity hardening for unrelated shared-secret clients.
