## Summary

Keep this PR open. Current main does not contain the submitted file-lock/updateConfigFile approach, and the current non-interactive agents add path still computes the next config before committing. Main now passes a baseHash into replaceConfigFile, which can reject a stale write, but that is not the same as serializing concurrent adds so every agent entry is preserved. The PR is also paired with the still-open multi-agent orchestration issue by the same author.

## Best Possible Solution

Keep the PR open for maintainer review or a current-main rewrite. The best fix is to apply the agent-add mutation to a freshly read config inside a serialized read-modify-write section, preserve 0700 config directory creation, and add regression coverage showing concurrent non-interactive agents add calls preserve every agent and binding.
