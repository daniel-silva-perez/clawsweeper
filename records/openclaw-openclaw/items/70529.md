## Summary

Keep open. Current main has removed the legacy EasyClaw/extension relay path, but the Desktop-specific `user` profile failure remains: the built-in existing-session profile still falls through to `npx -y chrome-devtools-mcp@latest`, and the upstream MCP stdio transport spawns that executable directly. The maintainer discussion also explicitly scoped this as separate from the broader browser attach and login UX reports.

## What This Changes

The issue reports that Genspark Claw Desktop cannot reuse existing Chrome sessions because the legacy EasyClaw relay OAuth flow fails and the built-in `user` existing-session profile tries to spawn ambient `npx`, which the packaged app lacks.

## Best Possible Solution

Make the packaged Desktop/browser runtime handle the built-in `user` profile without relying on ambient `npx`: either ship or resolve a pinned Chrome MCP runner through the app/package boundary, or fail early with a structured missing-runner diagnostic that points users to the managed `openclaw` profile and an explicit `mcpCommand` override. Keep the removed extension relay path retired.

## Reproduction Assessment

Yes for the remaining Desktop `user` profile failure: current main defaults the built-in profile to existing-session, builds `npx -y chrome-devtools-mcp@latest` args, and the MCP transport spawns that command directly. The EasyClaw relay half is obsolete on current main because the extension relay lifecycle is now a no-op.

## Solution Assessment

Unclear because there is no patch here and the right fix requires a maintainer packaging strategy. The documented `mcpCommand` workaround is useful, but it is not the best complete solution for a built-in Desktop profile that users expect to work without installing `npx` separately.
