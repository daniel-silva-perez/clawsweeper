## Summary

Current main still leaves the Desktop `user` existing-session failure valid. The legacy EasyClaw/extension relay path is now obsolete because the relay helper is an intentional no-op, but the built-in `user` browser profile still resolves to Chrome MCP existing-session attach and still defaults to spawning ambient `npx -y chrome-devtools-mcp@latest`. Docs describe `mcpCommand`/`mcpArgs` as an override, not as a packaged Desktop-safe default, and the MCP SDK stdio transport spawns the configured executable directly, so a packaged app environment without `npx` can still fail with `spawn npx ENOENT`. Maintainer discussion already scoped this as the Desktop-specific variant rather than a duplicate of #62288 or #63926.

## Best Possible Solution

Keep this open and resolve the Desktop/browser runtime strategy for existing-session attach. The best fix is either to make the built-in `user` profile work in packaged Desktop without ambient `npx` by bundling or resolving a pinned Chrome MCP executable, or to add a structured missing-runner diagnostic that clearly recommends the managed `openclaw` profile and an explicit `mcpCommand` override. Do not restore the removed EasyClaw extension relay path; treat #62288 and #63926 as broader related UX/diagnostic work, not as replacements for this packaged-app failure.
