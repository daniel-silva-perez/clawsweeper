## Summary

Keep open. Current main still routes the built-in `user` profile through the Chrome MCP existing-session path, and that path still defaults to spawning ambient `npx -y chrome-devtools-mcp@latest`. The legacy EasyClaw/Chrome extension relay path is now removed, but the Desktop-specific packaged-app failure called out by the maintainer remains valid and is not fully covered by the broader open browser attach and login UX issues.

## Best Possible Solution

Keep this issue open and decide the Desktop/runtime strategy: either make existing-session attach work without ambient `npx` in the packaged app by bundling or resolving a pinned Chrome MCP executable, or add a structured missing-runner diagnostic that recommends the managed `openclaw` profile and an explicit `mcpCommand` override. The fix should stay in the browser plugin/Desktop packaging boundary, not in generic core startup migration.
