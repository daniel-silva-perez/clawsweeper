## Summary

Keep open. Current main does not contain any `iMessageLite` handling, so this PR is not obsolete; however, the patch only normalizes webhook `chatGuid` values and bare raw GUID targets. The documented `chat_guid:` target path still bypasses the new helper and can pass `iMessageLite` through to send/reaction actions unchanged, so the PR needs a small correctness fix before merge.

## What This Changes

The PR adds a BlueBubbles `normalizeChatGuidService` helper, applies it to inbound webhook `chatGuid` normalization and bare raw target parsing, and adds monitor-normalize coverage for satellite `iMessageLite` GUIDs.

## Best Possible Solution

Revise this PR or a narrow replacement so BlueBubbles has one chatGuid service canonicalization path applied anywhere a `chatGuid` is parsed, compared, cached, or sent, then land it with targeted tests for webhook ingestion, bare raw GUID targets, and prefixed `chat_guid:` targets.
