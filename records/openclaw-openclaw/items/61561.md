## Summary

Keep this PR open. The reported BlueBubbles satellite-message bug still appears real on current main, but the proposed patch misses a documented target path: prefixed `chat_guid:`/`guid:` values bypass the new bare-GUID normalization and would still send `iMessageLite` to macOS Messages.

## What This Changes

The PR adds BlueBubbles `iMessageLite` to `iMessage` chatGuid normalization in webhook ingestion and bare raw chat GUID target parsing, with tests in `monitor-normalize.test.ts`.

## Best Possible Solution

Land a narrow BlueBubbles fix that canonicalizes chatGuid service names through one helper anywhere a chatGuid is parsed or compared, including webhook ingestion, bare raw GUID targets, prefixed `chat_guid:`/`guid:` targets, and any allowlist path maintainers decide to support.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

Apply the helper to prefixed chat GUID parsing and add a target/send test for that form.
