## Summary

Keep this PR open. Current main has the schema/status piece from merged PR #71247, but it still does not implement the PR’s stated embedded-runner behavior: `resolveContextWindowInfo` has no agent identity or agent config input, and the main run, compaction, queued/context-engine compaction, and extension factory budget paths still resolve context tokens from model metadata plus `agents.defaults.contextTokens`, not `agents.list[].contextTokens`. The submitted PR is also incomplete as-is because its review discussion identifies missing compaction/extension call sites and a precedence bug when a per-agent override should supersede a stricter global default.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a corrected rebase: make `agents.list[].contextTokens` an authoritative per-agent override over `agents.defaults.contextTokens` while still respecting the resolved model/provider window, thread the active agent id or resolved agent config through embedded run setup, direct compaction, queued/context-engine compaction, and extension factory setup, then add focused tests for main run budgeting, compaction, extension runtime budgeting, and the global-default override case.
