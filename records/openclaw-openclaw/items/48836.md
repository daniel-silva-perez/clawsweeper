## Summary

Keep this PR open. Current main still does not implement the central request: chat UI controls and labels remain hardcoded in English, the proposed chat locale keys are absent from the active chat locale maps, and the current UI i18n workflow requires generated non-English locale output rather than the PR’s older hand-edited zh-CN-only shape. The PR is still a coherent localization candidate, but it needs a refresh against current main.

## Best Possible Solution

Refresh the PR against current main: move the remaining hardcoded chat labels in ui/src/ui/views/chat.ts, ui/src/ui/chat/run-controls.ts, and the chat settings toggle to t(...) calls; add the English source keys; remove or wire closeSessionDetails; run the Control UI i18n sync workflow so zh-CN and metadata are generated consistently; and update run-controls tests to assert localized labels through the current i18n path.
