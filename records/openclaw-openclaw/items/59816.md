## Summary

Keep open. Current main still lacks the history write in the Discord `allowBots="mentions"` drop path, so the PR targets a real remaining bug, but the added regression test does not currently force execution through that branch and should be fixed before merge.

## What This Changes

The PR adds a `recordPendingHistoryEntryIfEnabled()` call before dropping unmentioned bot-authored Discord guild messages under `allowBots="mentions"` and adds a unit test for the recorded guild history entry.

## Best Possible Solution

Land a rebased version that keeps the source fix in the Discord plugin preflight branch and updates the regression test to disable normal guild `requireMention` gating, proving the history entry is recorded specifically by the `allowBots="mentions"` missing-mention path.

## Reproduction Assessment

Yes. A high-confidence reproduction is a guild message from another bot with `allowBots: "mentions"`, `requireMention: false`, `historyLimit > 0`, and no bot mention; current main reaches the allowBots missing-mention return without recording the pending history entry.

## Solution Assessment

No as written. The source change is the narrow maintainable fix, but the added test must configure mention policy so it reaches the changed allowBots branch instead of the older no-mention branch.
