## Summary

Keep this PR open. Current main still has the Discord `allowBots="mentions"` drop path returning before recording pending guild history, while nearby mention-related drop paths do record history. The PR’s code change is a plausible fix, but the regression test needs the already-posted review feedback addressed so it actually reaches the allowBots-only branch.

## Best Possible Solution

Keep this PR open and ask for a small test correction: configure the test guild/channel with `requireMention: false`, pass `historyLimit > 0` with a shared `guildHistories` map, and assert the recorded entry matches the dropped bot message. Then land this PR or an equivalent source change in `extensions/discord/src/monitor/message-handler.preflight.ts`.
