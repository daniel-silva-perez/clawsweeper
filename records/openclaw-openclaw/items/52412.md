## Summary

Keep open. Current main has gateway-level abort infrastructure for agent RPC runs and hook contexts expose run IDs, but the requested plugin runtime method `api.runtime.agent.abort({ runId })` is still missing from the runtime type, runtime builder, test runtime mock, and SDK docs. The linked implementation PRs are closed unmerged, and #66532 is adjacent rather than an exact replacement.

## Best Possible Solution

Keep this open unless maintainers explicitly fold it into #66532. The best implementation path is an additive, documented plugin-runtime cancellation seam in `src/plugins/runtime/types-core.ts` and `src/plugins/runtime/runtime-agent.ts`, backed by the existing abort infrastructure with clear runId/sessionKey authorization semantics, plus SDK docs, runtime mock/API baseline coverage, and targeted tests for success, missing run, cross-session denial, and cleanup behavior.

## Risks / Open Questions

- Closing now would conflate the existing gateway `chat.abort` support with the still-missing in-process plugin runtime API.
- The API still needs maintainer judgment on runId/sessionKey authorization semantics, whether plugins may abort any known runId, and how it should interact with embedded runs versus gateway RPC runs.
- #66532 may become a better umbrella if maintainers intentionally broaden it, but today it tracks a different embedded-Pi abort family rather than this exact runId-based method.
