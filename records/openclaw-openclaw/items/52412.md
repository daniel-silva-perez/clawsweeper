## Summary

Keep open. Current main has gateway-level `chat.abort` support for agent RPC runs and hook contexts expose `ctx.runId`, but the requested plugin runtime method `api.runtime.agent.abort({ runId })` is still not exposed in the PluginRuntime type, runtime builder, test runtime mock, or SDK docs. The prior implementation PRs referenced from this issue are closed unmerged, and the newer adjacent #66532 tracks an abortEmbeddedPiRun-family surface rather than the exact runId-based runtime abort API.

## Best Possible Solution

Keep this open unless maintainers explicitly fold it into #66532. The best implementation path is a supported plugin-runtime cancellation seam in `src/plugins/runtime/types-core.ts` and `src/plugins/runtime/runtime-agent.ts`, with SDK docs, API baseline/test-helper updates, and clear semantics for runId/sessionKey authorization and interaction with the existing `chat.abort` controller registry.
