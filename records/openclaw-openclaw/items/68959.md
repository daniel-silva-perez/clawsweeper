## Summary

Keep this issue open. Current main still has a combined chat session picker grouped by agent, and the Agents tab has its own configuration selector, but there is no Control UI chat agent switcher that lists every configured agent and can switch/start a chat for an agent with no existing session row. Related #64121 remains the clearest implementation direction, but it is still open and not merged behavior.

## Best Possible Solution

Keep this issue open. Implement a Control UI chat agent picker backed by agents.list, show configured agent display names/identity, switch to the latest session for the chosen agent or buildAgentMainSessionKey({ agentId }) when none exists, keep the current per-agent session selector for session choice, and add focused tests for multi-agent configs where a non-default agent has no session row. Coordinate with #64121 rather than closing this before the split lands.
