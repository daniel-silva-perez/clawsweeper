## Summary

Keep this issue open. Current main has a combined chat session picker grouped by agent, so users can switch among existing agent-scoped sessions, but it still does not expose the requested Control UI agent switcher that lists configured agents and can switch/start chat for an agent with no existing session row.

## Best Possible Solution

Keep this issue open. Implement a Control UI chat agent picker backed by `agents.list`, show configured agent display names/identity, switch to the most recent session for that agent or `buildAgentMainSessionKey({ agentId })` when none exists, keep the current per-agent session selector for session choice, and add focused tests for multi-agent configs where a non-default agent has no existing session row. Coordinate with #64121 and the open chat-selector PR rather than closing this early.
