## Summary

Keep this issue open. Current main has partial google-vertex metadata/credential detection, but the requested project/location endpoint construction and onboarding path are still not implemented. There are also open implementation PRs (#60860 and #62103) that explicitly close this issue, including a same-author PR, so this should stay open until a canonical PR is reviewed and merged or rejected.

## Best Possible Solution

Keep this issue open while maintainers review a canonical Google Vertex implementation, likely the broader #60860 or a consolidated successor. The fix should live in the Google plugin: register a real google-vertex provider, resolve project/location with GOOGLE_CLOUD_LOCATION, CLOUD_ML_REGION, and a safe default, route to the Vertex AI project/location endpoint, handle ADC/OAuth bearer auth, add onboarding/docs, and cover the route/auth behavior with focused tests. After that PR merges, this issue can close as implemented.
