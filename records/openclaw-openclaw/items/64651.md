## Summary

Keep open. Current main still leaves the reported Control UI chat and config/form inputs below the 16px iOS Safari focus threshold, and open PR #64673 explicitly references this issue with closing syntax, so the issue should stay open until that PR is merged, replaced, or closed.

## What This Changes

Set focusable Control UI chat, composer, and form/config text inputs to at least a 16px computed font size on iOS/touch-primary browsers to prevent Safari focus auto-zoom.

## Best Possible Solution

Keep this issue open. Review and land PR #64673 if its coverage is adequate, or replace it with a maintainer patch that sets focusable Control UI text inputs to at least 16px on iOS/touch-primary devices. The fix should cover the main chat composer, agent chat textarea, generic .field inputs/textareas/selects, config .cfg-input/.cfg-textarea controls including small variants, and add a focused CSS regression check. PR #63644 is adjacent Safari layout/viewport work but should not be treated as resolving the font-size threshold by itself.
