## Summary

Keep open. Current main still calls PDF.js without `standardFontDataUrl` in the bundled `document-extract` PDF fallback, the fallback page-render path is still reachable, v2026.4.27 has the same omission, and open PRs #70936 and #74355 are active implementation candidates that should be reviewed or consolidated before closing this issue.

## What This Changes

The issue requests wiring PDF.js standard font assets into the Node PDF extraction fallback renderer and adding regression coverage to prevent `standardFontDataUrl` warnings.

## Best Possible Solution

Land one narrow plugin-owned fix that resolves the installed `pdfjs-dist/standard_fonts/` directory as a slash-terminated filesystem path, passes it as `standardFontDataUrl` to the active `document-extract` `getDocument` call, and adds focused regression coverage that fails without the option.
