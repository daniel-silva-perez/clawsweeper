## Summary

Keep open. Current main still calls PDF.js without `standardFontDataUrl` in the active `document-extract` fallback, the page-image render path remains reachable, and open PR #70936 is the explicit unmerged implementation candidate. This issue should stay open until that PR or an equivalent narrow fix lands.

## What This Changes

The issue requests that the PDF extraction fallback renderer pass PDF.js a slash-terminated standard font data path when rendering PDF pages in Node.

## Best Possible Solution

Land PR #70936, or an equivalent narrow fix in the bundled `document-extract` plugin, so the extractor resolves the installed `pdfjs-dist/standard_fonts` directory as a slash-terminated filesystem path, passes it to `getDocument`, updates the local type surface, and locks the behavior with focused regression coverage.
