## Summary

Keep open. Current main still omits `standardFontDataUrl` from the active bundled `document-extract` PDF.js call, the low-text fallback still reaches page rendering, the latest release tag has the same omission, and the provided GitHub context shows open PR #70936 explicitly closes this issue while PR #74355 is another active implementation candidate.

## What This Changes

The issue requests wiring PDF.js `standardFontDataUrl` into the Node PDF extraction fallback renderer so standard-font PDF rendering avoids repeated warnings.

## Best Possible Solution

Land one narrow plugin-owned fix that resolves the installed `pdfjs-dist/standard_fonts/` directory as a slash-terminated filesystem path, passes it through the local `PdfJsModule.getDocument` type and call site, and adds regression coverage against the active `document-extract` path; consolidate the overlapping open PRs rather than creating another branch.
