## Summary

Keep open. Current main still omits `standardFontDataUrl` from the bundled `document-extract` PDF.js call, the low-text fallback still reaches `page.render(...)`, the latest release has the same call shape, and open PR #70936 explicitly closes this issue while #74355 is another active implementation candidate.

## What This Changes

Pass PDF.js `standardFontDataUrl` into the Node PDF extraction fallback renderer and add regression coverage for standard-font PDFs.

## Best Possible Solution

Land one narrow `document-extract` plugin fix that resolves the installed `pdfjs-dist/standard_fonts/` directory as a slash-terminated filesystem path, passes it to `getDocument`, and preserves the behavior with focused regression coverage against the current plugin path.
