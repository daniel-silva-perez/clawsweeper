## Summary

Keep open. Current main still omits `standardFontDataUrl` from the active PDF.js fallback renderer, the page-image render fallback remains reachable, and open PR #70936 is the explicit unmerged implementation candidate for the same bug.

## What This Changes

Provide PDF.js standard font assets to the Node PDF extraction fallback renderer so fallback page rendering no longer emits missing-`standardFontDataUrl` warnings.

## Best Possible Solution

Review and land PR #70936, or an equivalent narrow plugin-owned fix, so `document-extract` resolves the installed `pdfjs-dist/standard_fonts` directory as a slash-terminated filesystem path, passes it into `getDocument`, updates the local legacy type surface, and keeps focused regression coverage in the document-extractor tests.
