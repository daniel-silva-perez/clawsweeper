## Summary

Keep open. Current main still omits `standardFontDataUrl` from the active bundled `document-extract` PDF.js call, the fallback page-render path remains reachable, the latest release tag has the same omission, and open PR #70936 explicitly closes this issue while #74355 is another active implementation candidate.

## What This Changes

The issue requests that the PDF extraction fallback renderer pass PDF.js a Node filesystem path for standard font assets so standard-font PDFs do not emit `standardFontDataUrl` warnings.

## Best Possible Solution

Land one narrow plugin-owned fix that resolves the installed `pdfjs-dist/standard_fonts/` directory as a slash-terminated filesystem path, passes it through the local `PdfJsModule.getDocument` type and call site, and adds focused regression coverage against the active `document-extract` path; consolidate the overlapping open PRs rather than creating another branch.
