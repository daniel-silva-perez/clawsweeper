## Summary

Keep open. Current main still omits `standardFontDataUrl` in the bundled `document-extract` PDF.js call, the page-render fallback remains reachable, the latest release has the same omission, and open PR #70936 explicitly tracks this issue with closing syntax; #74355 is another current implementation candidate.

## What This Changes

The issue requests configuring the PDF.js fallback renderer with `standardFontDataUrl` so Node fallback PDF rendering can find standard font assets without warnings.

## Best Possible Solution

Land one narrow plugin-owned fix that resolves the installed `pdfjs-dist/standard_fonts/` directory as a slash-terminated filesystem path, passes it through the local `PdfJsModule.getDocument` type and call site, and adds focused regression coverage that would fail without the option.
