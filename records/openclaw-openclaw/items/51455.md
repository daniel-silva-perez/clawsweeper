## Summary

Keep open. Current main still omits `standardFontDataUrl` from the bundled `document-extract` PDF.js call, the low-text fallback can still render pages through `page.render(...)`, the latest release has the same call shape, and open PR #70936 explicitly closes this issue while PR #74355 is another active implementation candidate.

## What This Changes

The issue asks for the PDF extraction fallback renderer to pass a Node filesystem-path `standardFontDataUrl` to PDF.js and add regression coverage so standard-font warnings stop.

## Best Possible Solution

Land one narrow plugin-owned fix that resolves the installed `pdfjs-dist/standard_fonts/` directory as a slash-terminated filesystem path, passes it through the local `PdfJsModule.getDocument` type and call site, and adds focused regression coverage that fails without the option.
