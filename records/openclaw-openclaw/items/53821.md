## Summary

Keep this PR open. Current main still accepts the `agents.list[].sandbox.sessionToolsVisibility` config shape, but the runtime session visibility path still resolves only the global default clamp and the affected session tools still do not receive `requesterAgentIdOverride`. The PR remains a focused implementation candidate for an access-control-sensitive feature, not something already implemented on main.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. The best path is to review or rework it so `sessions_list`, `sessions_history`, `sessions_send`, and `session_status` all resolve the effective requester agent consistently, apply `agents.list[].sandbox.sessionToolsVisibility` before falling back to `agents.defaults`, preserve A2A checks, handle custom `session.mainKey`, and add focused tests/docs. Coordinate with the broader open per-agent session/A2A scoping work in #59170 if that lands first.
