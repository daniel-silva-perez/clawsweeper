## Summary

Keep this PR open. Current main has the config schema for `agents.list[].sandbox.sessionToolsVisibility`, but the runtime still resolves sandbox session visibility from the global default and the session tools are not wired to apply the requester agent override. The PR is therefore not obsolete or implemented on main.

## Best Possible Solution

Keep this PR open as the implementation candidate. The best path is to review and harden the runtime wiring so `agents.list[].sandbox.sessionToolsVisibility` overrides `agents.defaults.sandbox.sessionToolsVisibility` consistently across `sessions_list`, `sessions_history`, `sessions_send`, and `session_status`, including `requesterAgentIdOverride`, spawned-session filtering, custom `session.mainKey`, and A2A guard checks, then land it with the added focused agent-tool tests.
