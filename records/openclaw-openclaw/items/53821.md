## Summary

Keep this PR open. Current main accepts `agents.list[].sandbox.sessionToolsVisibility` in the config schema, but the session-tools runtime still resolves the sandbox clamp only from `agents.defaults.sandbox.sessionToolsVisibility` and does not thread the requester agent override through sessions_list, sessions_history, sessions_send, or session_status. The PR remains a focused implementation candidate for that missing runtime behavior, with access-control-sensitive review comments that should be handled through normal PR review rather than cleanup closure.

## Best Possible Solution

Keep the PR open as the implementation candidate. The best path is to review or rework it so every session tool resolves the effective requester agent consistently, applies `agents.list[].sandbox.sessionToolsVisibility` before falling back to defaults, preserves A2A checks, covers custom `session.mainKey`, and adds focused tests for list/history/send/status override flows.
