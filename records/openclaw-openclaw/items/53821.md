## Summary

Keep this PR open. Current main accepts the `sessionToolsVisibility` sandbox field in shared agent config, but the runtime still resolves the sandbox session-tools clamp only from `agents.defaults.sandbox.sessionToolsVisibility`; it does not check `agents.list[]` overrides or thread `requesterAgentIdOverride` through the session list/history/send/status access paths. The PR is also not a cleanup-close candidate because the feature broadens session data visibility and the provided PR context still shows unrelated file changes plus access-control review concerns that need maintainer review or rework.

## Best Possible Solution

Keep the PR open and ask for a focused rebase against current main. The best implementation path is to move the per-agent clamp lookup into the current shared `src/plugin-sdk/session-visibility.ts` path, normalize and match `agents.list[].id`, thread `requesterAgentIdOverride` through `createOpenClawTools` into sessions_list, sessions_history, sessions_send, and session_status, preserve `session.mainKey` when synthesizing requester roots, and add focused tests for default fallback, per-agent override, spawned filtering, cross-agent/A2A enforcement, and custom main keys. The unrelated msteams/tlon/harness changes should be removed or split into separate PRs.
