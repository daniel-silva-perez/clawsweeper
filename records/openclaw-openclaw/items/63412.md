## Summary

Keep this PR open. Current main has a Discord chunker, but it still only preserves fenced-code and reasoning-italic state; it does not track or re-seed blockquote prefixes when a long quoted line is split across Discord message chunks. The PR discussion is useful: Greptile flagged the initial empty-blockquote-line regression, and the author followed up with a narrower continuation-only fix plus tests. Related PR #63460 may supersede this direction, but it is still open and unmerged, so this focused PR should not be closed as implemented.

## Best Possible Solution

Review this PR as a focused Discord chunking fix: preserve blockquote prefixes only for mid-line continuation splits, keep normal line-boundary blockquote chunks free of empty `> ` artifacts, and land the accompanying regression tests. If maintainers prefer the broader markdown-preservation work in #63460, make that the explicit canonical PR and close this PR only after that decision or after #63460 merges.
