## Summary

Keep this PR open. Current main still only loads immediate child skill directories under each configured skills root, so `skills/<group>/<skill>/SKILL.md` remains undiscovered. The PR is an active implementation candidate for the linked open regression, with maintainer follow-up already pushed to the branch and no evidence that main has an equivalent fix.

## Best Possible Solution

Keep this PR open for maintainer review and land it, or an equivalent patch, as the canonical fix for grouped skill discovery. The fix should add bounded one-level discovery under configured skill roots, preserve direct skill behavior, keep containment and scan limits intact, and ship the regression tests plus docs/changelog entry already sketched in the PR.
