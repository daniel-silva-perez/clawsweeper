## Summary

Keep open. Current main still lacks a proactive fallback-chain test command: `/models` only parses provider/list/add flows, `openclaw models status --probe` remains an auth-profile probe, and the runtime fallback resolver still begins with the requested primary before configured fallbacks. The direct implementation PR #43342 is closed and unmerged, while #63145 is a broader per-model health-check request rather than a replacement for this targeted command.

## What This Changes

Add a `/models test-fallback` command that verifies configured model fallback chains proactively instead of waiting for a real provider failure.

## Best Possible Solution

Ship a first-class fallback-chain probe exposed through `/models test-fallback` that reuses OpenClaw's real model/auth fallback resolution, intentionally begins at configured fallback candidates, reports the responding provider/model and failures, supports first-fallback and all-fallback modes, and documents how it differs from auth-profile probes and broader per-model health checks.
