## Summary

Keep open. Current main still lacks a `/models test-fallback` command or equivalent fallback-chain probe: the CLI and slash-command surfaces only expose adjacent model status/list/fallback management, `models status --probe` remains an auth-profile probe, and the runtime fallback path starts with the requested primary before configured fallbacks. The direct implementation PR #43342 is closed and unmerged, while #63145 is a broader per-model health-check request rather than a replacement for this targeted chain test.

## What This Changes

Add a `/models test-fallback` command that deliberately skips the primary model, probes configured fallback models, reports which model responded, and optionally tests the full fallback chain.

## Best Possible Solution

Ship a first-class fallback-chain probe that reuses the real fallback/auth resolution path, intentionally begins at configured fallback candidates, reports which model answered, supports first-fallback and all-fallback modes, and documents how it differs from auth-profile probes and broad per-model health checks.
