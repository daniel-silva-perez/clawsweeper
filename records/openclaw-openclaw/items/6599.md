## Summary

Keep open. Current main still lacks a proactive fallback-chain test command: the slash `/models` parser only supports provider/list/add flows, the CLI status probe remains an auth-profile probe, and the runtime fallback resolver still starts with the requested primary before configured fallbacks. The direct implementation PR #43342 is closed and unmerged, while #63145 is a broader per-model health-check request rather than a replacement for this targeted command.

## What This Changes

Request to add a `/models test-fallback` command that proactively skips the primary model and probes configured fallback models, with an option to test the whole chain.

## Best Possible Solution

Ship a first-class fallback-chain probe exposed through `/models test-fallback` that reuses OpenClaw's real model/auth fallback resolution, intentionally begins at configured fallback candidates, reports the responding provider/model and per-candidate failures, supports first-fallback and all-fallback modes, and documents how it differs from auth-profile probes and broad per-model health checks.
