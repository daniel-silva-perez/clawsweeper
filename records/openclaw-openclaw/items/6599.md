## Summary

Keep open. Current main still lacks a proactive fallback-chain test command: neither the CLI nor slash `/models` command surface registers `test-fallback`, the existing `models status --probe` path is an auth/profile probe, and the runtime fallback resolver still begins with the effective primary before configured fallbacks. The direct implementation PR #43342 is closed and unmerged, while #63145 is a broader per-model health-check request rather than a replacement for this targeted command.

## What This Changes

Add a `/models test-fallback` command that intentionally skips the primary model and probes the configured fallback chain, optionally testing all fallback candidates.

## Best Possible Solution

Ship a first-class fallback-chain probe exposed through `/models test-fallback` that reuses OpenClaw's real model/auth fallback resolution, intentionally begins at configured fallback candidates, reports the responding provider/model and per-candidate failures, supports first-fallback and all-fallback modes, and documents how it differs from auth-profile probes and broader per-model health checks.
