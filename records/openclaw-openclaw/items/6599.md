## Summary

Keep open. Current main still lacks a proactive fallback-chain test surface: the slash command only routes provider/list/add flows, the CLI status probe remains auth-profile oriented, and the runtime fallback runner still starts from the requested primary before configured fallback candidates. The prior direct implementation PR is closed and unmerged, while the broader health-check issue does not replace this narrower command.

## What This Changes

The issue requests a `/models test-fallback` command that intentionally skips the primary model and verifies configured fallback candidates before a real provider failure.

## Best Possible Solution

Ship a first-class fallback-chain probe that reuses OpenClaw's real model/auth fallback resolution, intentionally begins at configured fallback candidates, reports the responding provider/model plus per-candidate failures, supports first-fallback and all-fallback modes, and documents how it differs from auth-profile probes and broader per-model health checks.
