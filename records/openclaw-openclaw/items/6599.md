## Summary

Keep this issue open. Current main still has model fallback runtime and live auth probing, but it does not provide `/models test-fallback` or an `openclaw models test-fallback` CLI command that intentionally skips the primary model and verifies the configured fallback chain. The directly related implementation PR, #43342, is closed and unmerged.

## Best Possible Solution

Keep this open until maintainers add or explicitly decline a first-class fallback-chain probe. The best implementation would add `/models test-fallback` plus a CLI equivalent, reuse the real fallback resolution/run path with an intentional primary skip, support first-fallback and all-fallback modes, respect active agent/session auth context, avoid session/config mutation, add focused slash/CLI/runtime tests, and document how it differs from `models status --probe`.

## Clownfish Work Prompt

Implement the fallback-chain probe requested by #6599. Add `/models test-fallback` and an `openclaw models test-fallback` CLI equivalent that intentionally skips the configured primary model and verifies the configured fallback chain. Default to probing only the first fallback; support an all-chain mode (`all` for slash and `--all` for CLI). Reuse the real model fallback resolution/run path or a shared helper derived from it, but do not mutate session/config state or persist auto fallback overrides during the probe. Respect the active agent/session auth context for slash usage and `--agent` for CLI. Return a clear success/failure summary naming the attempted fallback model(s), which model responded, and redacted errors. Document how this differs from `models status --probe`, which is auth/profile reachability rather than fallback-chain execution. Add a user-facing changelog entry with `Thanks @jonwiley` attribution. Do not change unrelated provider fallback policy, auth ordering semantics, or model selection behavior.
