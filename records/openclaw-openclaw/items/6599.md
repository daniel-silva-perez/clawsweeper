## Summary

Keep open. Current main still does not provide a proactive fallback-chain test surface: `/models` only routes provider/list/add flows, `openclaw models status --probe` is auth-profile oriented, and the runtime fallback runner still begins with the requested primary before configured fallbacks. The direct implementation PR is closed and unmerged, and the broader health-check issue does not replace this narrower request.

## What This Changes

Add a `/models test-fallback` command that intentionally skips the primary model and probes one or all configured fallback candidates to report which model responds.

## Best Possible Solution

Ship a first-class fallback-chain probe that uses OpenClaw's real model/auth fallback resolution, begins at configured fallback candidates rather than the primary, reports the responding provider/model plus per-candidate failures, supports first-fallback and all-fallback modes, and documents how it differs from auth-profile probes and broader per-model health checks.
