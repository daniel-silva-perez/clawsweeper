## Summary

Keep open. Current main still lacks the requested fallback-chain test command: the CLI has model status/list/set/fallback management and auth probes, the slash `/models` parser only supports provider/list/add flows, and the runtime fallback resolver begins with the requested primary rather than intentionally starting at the first fallback. The direct implementation PR #43342 is closed and unmerged, while #63145 tracks a broader per-model health-check idea rather than this targeted command.

## What This Changes

Add a `/models test-fallback` command, optionally with an all-fallback mode, to proactively verify the configured model fallback chain without waiting for a real provider failure.

## Best Possible Solution

Ship a first-class fallback-chain probe that reuses OpenClaw’s real model/auth fallback resolution, intentionally begins at configured fallback candidates, reports which model answered, supports first-fallback and all-fallback modes, and documents how it differs from auth-profile probes and broader per-model health checks.
