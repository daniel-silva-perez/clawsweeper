## Summary

Keep open. Current main still lacks the requested proactive fallback-chain test command: `/models` only parses provider/list/add flows, the CLI model surface exposes status/list/set/fallback management/auth/scan but no fallback-chain test, and `models status --probe` remains an auth-profile/provider probe rather than a primary-skipping fallback walk. The direct implementation PR #43342 is closed unmerged, while #63145 is a broader per-model health-check request rather than an exact replacement.

## What This Changes

The issue requests a `/models test-fallback` command that intentionally skips the primary model and verifies one or all configured fallback candidates before a real outage occurs.

## Best Possible Solution

Ship a first-class fallback-chain probe that reuses OpenClaw's real model/auth fallback resolution, intentionally begins at configured fallback candidates rather than the primary, reports the responding provider/model and per-candidate failures, supports first-fallback and all-fallback modes, and documents how it differs from auth-profile probes and broader per-model health checks.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is static inspection of the current command registration, slash parser, docs, and fallback runtime, which shows the requested command is absent.

## Solution Assessment

Yes, the requested direction is the narrowest maintainable solution if implemented as a shared fallback-chain probe rather than overloading `models status --probe`. Broader all-model health checks should stay with #63145 unless maintainers intentionally merge the surfaces.
