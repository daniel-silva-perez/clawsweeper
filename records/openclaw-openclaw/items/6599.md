## Summary

Keep open. Current main still does not implement the requested proactive fallback-chain test command: the CLI exposes status/list/set/fallback-management/auth probes, the `/models` slash parser only handles provider/list/add flows, and the existing status probe remains auth-profile oriented rather than intentionally starting at configured fallback candidates. The direct implementation PR #43342 is closed and unmerged, while #63145 is related but broader per-model health checking.

## What This Changes

Add a `/models test-fallback` command, with an optional all-fallback mode, to verify configured model fallback candidates without waiting for a real primary-model failure.

## Best Possible Solution

Ship a first-class fallback-chain probe that reuses OpenClaw's real model/auth fallback resolution, intentionally begins at configured fallback candidates, reports provider/model/status for the responding candidate, supports first-fallback and all-fallback modes, and documents how it differs from auth-profile probes and broader per-model health checks.
