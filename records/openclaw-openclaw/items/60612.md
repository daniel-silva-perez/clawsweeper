## Summary

Keep open. Current main still warns for version-managed Gateway Node runtimes and PATH entries, and doctor only repairs that warning by migrating to a supported system Node when one is available. The newly documented `--wrapper` path is useful for custom service launch behavior, but it is not a dedicated persistent Node-path setting or a scoped acknowledgement/suppression for intentional NVM/ASDF use.

## Best Possible Solution

Add an explicit, documented Gateway service runtime policy. The likely implementation is either a persistent `gateway install --node-path <path>` / config setting that doctor repair also honors, or a scoped doctor acknowledgement such as `allowVersionManagedNode` for users who intentionally run NVM/ASDF. Keep the warning by default, include docs explaining when it is safe to ignore versus when to migrate to Homebrew/system Node, and cover NVM plus ASDF in service-audit and doctor tests.

## Risks / Open Questions

- `--wrapper` can be used by advanced users to build custom launch behavior, but treating that as the complete fix would leave the normal NVM/ASDF install path noisy and undocumented for intentional use.
- The existing warning may be valid because version-manager paths can break after upgrades, so any suppression should be explicit and documented rather than silently hiding the audit.
