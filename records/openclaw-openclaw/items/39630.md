## Summary

Keep this PR open. Current main still does not expose the requested broad `tools.web.fetch.allowPrivateNetwork` opt-in for `web_fetch`; it only has the narrower `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange` exception, and the paired feature issue #39604 remains open. The PR also has unresolved security/product policy feedback around metadata endpoints and likely needs rework against the current guarded-fetch and schema shape.

## Best Possible Solution

Keep this PR open for maintainer direction or revision. The best path is to implement a documented private-network opt-in aligned with the current `tools.web.fetch.ssrfPolicy` surface, thread it through the strict runtime schema, public types, generated config metadata, runtime guard policy, cache partitioning, tests, docs, and changelog, and explicitly resolve the metadata-endpoint policy with #39604 as the canonical feature request.
