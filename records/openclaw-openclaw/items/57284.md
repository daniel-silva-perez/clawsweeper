## Summary

Keep open. Current main still routes `openclaw agent --deliver` through the normal agent execution path and only attempts outbound delivery after an agent result produces payloads. Empty/NO_REPLY payloads still return without delivery, and the provided context shows open implementation candidates (#57300 and a fork mirror) with closing syntax, so this issue should remain open until a fix lands or maintainers choose a replacement.

## Best Possible Solution

Keep this issue open and review/land #57300 or implement the same behavior on current main: for `--deliver` without `internalEvents`, bypass the LLM turn, synthesize a delivery result from `opts.message`, and pass it through `deliverAgentCommandResult`; preserve the LLM path for internal-event announcements and add regression coverage for direct delivery, internal events, and Gateway routing. Separately decide whether the session-injection request needs a CLI/API surface distinct from `sessions.send`.
