## Summary

Keep open. Current main still treats `openclaw agent --deliver` as an agent turn plus reply delivery, not as direct outbound notification delivery. The CLI sends `deliver: true` to the Gateway `agent` RPC, the Gateway dispatches a normal agent run, and `deliverAgentCommandResult` only sends to the channel after result payloads exist; empty/NO_REPLY output still returns `No reply from agent.` without delivering the original message. The provided context also shows open PR #57300 as the focused implementation candidate, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land #57300 or implement the same behavior on current main: for `--deliver` without `internalEvents`, bypass the LLM turn, synthesize a delivery result from `opts.message`, and pass it through `deliverAgentCommandResult`; preserve the LLM path for internal-event announcements and add regression coverage for direct delivery, internal events, and Gateway routing. Separately decide whether direct session injection needs a new CLI/RPC surface instead of `sessions.send`.
