## Summary

Keep open. Current main still routes `openclaw agent --deliver` through the normal agent run before delivery, and delivery only happens after agent payloads exist. The provided GitHub context also shows open implementation candidates (#57300 and a fork mirror #1654) with closing syntax for this issue, so the issue should remain open until a PR lands or maintainers choose a replacement fix.

## Best Possible Solution

Keep the issue open and review/rebase #57300 or implement the same fix on current main: for `--deliver` with no `internalEvents`, bypass the LLM turn, synthesize a result from `opts.message`, and pass it through `deliverAgentCommandResult`; preserve the LLM path for internal-event announcements and add regression coverage for direct delivery, internal events, and gateway routing.
