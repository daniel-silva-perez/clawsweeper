## Summary

Keep open. Current main still lacks Feishu Card Kit markdown sanitization for streaming updates, so the PR targets a real unimplemented truncation path, but the branch is not mergeable as-is because sanitized partial state is later merged with raw final text and the sanitizer misses common angle-bracket and code-span cases.

## What This Changes

This PR adds Feishu streaming-card markdown sanitization, sanitizer tests, summary cleanup, reasoning-tag stripping for partial Feishu stream updates, and a post-format fallback for markdown-card replies.

## Best Possible Solution

Keep the PR open and request fixes so Feishu streaming cards use one canonical sanitized representation across update and close paths, escape unsafe tag-shaped angle-bracket text while preserving allowed Feishu tags, and add focused regressions for final close merging, `<thinking>`-style content, and valid code-span forms before running the Feishu targeted checks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Sanitize final text before merging stream state:** `extensions/feishu/src/streaming-card.ts:399`
  - body: `update()` now stores sanitized text in `state.currentText`, but `close(finalText)` still merges raw final text and sends it through `updateCardContent`. For `x < 10`, the partial state becomes `x \\< 10`; the raw final `x < 10` no longer overlaps, so close appends it and emits duplicated text with an unsanitized `<`, making the truncation fix unreliable.
  - confidence: 0.92
- **[P2] Escape tag-shaped unsafe angle brackets:** `extensions/feishu/src/streaming-card.ts:74`
  - body: The angle-bracket regex only catches `<` runs that have no `>` before the end of the line. The linked bug includes `<thinking>`-style text, which has a `>` but is not a whitelisted tag, so it remains unescaped and can still be parsed as an unclosed tag by Card Kit.
  - confidence: 0.86
- **[P2] Preserve multi-backtick code spans before escaping:** `extensions/feishu/src/streaming-card.ts:69`
  - body: The code-span placeholder only handles single-backtick delimiters. Valid Markdown spans such as ``x < y`` are not protected, so the later angle-bracket escape can rewrite code content to include `\\<` and visibly corrupt streamed output.
  - confidence: 0.82
