## Summary

Keep open. Current main still does not implement the Feishu Card Kit markdown sanitizer, and the PR head has concrete correctness regressions in final closeout merging and sanitizer coverage that should block merge. Earlier fenced-code and mention-format concerns were rechecked against the latest PR head; those specific points were addressed or are no longer present, but the current branch remains incorrect.

## What This Changes

The PR adds Feishu streaming Card Kit markdown sanitization and sanitizer tests, strips reasoning tags from partial streaming updates, cleans card summaries, and falls back to post-format replies for markdown cards.

## Best Possible Solution

Revise this PR so Feishu streaming cards keep one canonical sanitized representation across partial updates and final closeout, escape unsafe tag-shaped angle-bracket text while preserving intentional Feishu/HTML tags, and protect all valid Markdown code span forms before angle-bracket escaping. After rebase, add regression coverage for final closeout after sanitized partial text, `<thinking>`-style text outside code, and multi-backtick code spans.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Sanitize final text before merging stream state:** `extensions/feishu/src/streaming-card.ts:399`
  - body: `update()` now stores sanitized content in `state.currentText`, but `close(finalText)` still merges raw final text. For `x < 10`, the partial state becomes `x \\< 10`; the raw final `x < 10` no longer overlaps, so close appends it and sends duplicated, unsanitized content.
  - confidence: 0.92
- **[P2] Escape tag-shaped unsafe angle brackets:** `extensions/feishu/src/streaming-card.ts:74`
  - body: The angle-bracket regex only catches `<` runs with no `>` before line end. The linked truncation case includes tag-shaped text such as `<thinking>`, which is not whitelisted but remains unescaped and can still be parsed as a tag by Card Kit.
  - confidence: 0.86
- **[P2] Preserve multi-backtick code spans before escaping:** `extensions/feishu/src/streaming-card.ts:69`
  - body: The code-span placeholder only handles single-backtick delimiters. Valid Markdown spans delimited by multiple backticks leave their contents exposed to the angle-bracket escape, so examples like double-backtick `x < y` are rewritten with an inserted backslash.
  - confidence: 0.82
