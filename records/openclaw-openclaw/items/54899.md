## Summary

Keep open. The Feishu truncation bug is real and current main still lacks Card Kit markdown sanitization, but this PR branch is not safe to merge because the sanitizer creates a mixed raw/sanitized stream state and still misses tag-shaped angle-bracket text and valid multi-backtick code spans.

## What This Changes

The PR adds Feishu streaming Card Kit markdown sanitization, sanitizer tests, partial-stream reasoning tag cleanup, and a markdown-card reply fallback to post-format replies.

## Best Possible Solution

Revise the Feishu plugin patch so streaming cards use one canonical sanitized representation through partial updates and final closeout, escape unsafe tag-shaped angle-bracket text while preserving intended Feishu/HTML tags, and protect all valid Markdown code span/fence forms before escaping. Add focused regressions for final closeout after sanitized partial text, '<thinking>'-style text outside code, and multi-backtick code spans.

## Reproduction Assessment

Yes for the PR findings: the sanitizer and merge behavior can be reproduced with a pure read-only Node probe using the PR logic, and the linked Feishu report gives the user-facing streaming scenario with backticks and angle brackets. I did not need live Feishu credentials to prove these branch-level regressions.

## Solution Assessment

No. Keeping the fix inside the Feishu plugin is the right boundary, but the current implementation is not the narrowest safe solution because it sanitizes partials without sanitizing the final merge input and leaves important Markdown/tag cases uncovered.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Sanitize final text before merging stream state:** `extensions/feishu/src/streaming-card.ts:399`
  - body: update() now stores sanitized content in state.currentText, but close(finalText) still merges raw final text. For x < 10, the partial state becomes x \\< 10; the raw final x < 10 no longer overlaps, so close appends it and sends duplicated, unsanitized content.
  - confidence: 0.92
- **[P2] Escape tag-shaped unsafe angle brackets:** `extensions/feishu/src/streaming-card.ts:74`
  - body: The angle-bracket regex only catches '<' runs with no '>' before line end. The linked truncation case includes tag-shaped text such as <thinking>, which is not whitelisted but remains unescaped and can still be parsed as a tag by Card Kit.
  - confidence: 0.86
- **[P2] Preserve multi-backtick code spans before escaping:** `extensions/feishu/src/streaming-card.ts:69`
  - body: The code-span placeholder only handles single-backtick delimiters. Valid Markdown spans delimited by multiple backticks leave their contents exposed to the angle-bracket escape, so examples like double-backtick `x < y` are rewritten with an inserted backslash.
  - confidence: 0.82
