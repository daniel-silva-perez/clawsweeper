## Summary

Keep this PR open. Current main has internal tool-card expansion state plumbing, but the Control UI still has no persisted user setting or toolbar toggle for expanding tool calls by default; `renderApp` hardcodes `autoExpandToolCalls: false`. The PR’s intended feature is therefore not implemented on main, though it should be updated to use the newer expansion-state helper and address the existing review comments before merge.

## Best Possible Solution

Keep this PR open as a valid implementation candidate, but update it against current main: add a persisted Control UI setting and desktop/mobile toolbar toggle, wire that setting into the existing `autoExpandToolCalls`/`syncToolCardExpansionState` path, preserve manual expansion state across rerenders, fix the tooltip/onboarding review comments, and add focused storage/rendering tests.
