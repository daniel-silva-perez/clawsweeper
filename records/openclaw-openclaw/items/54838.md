## Summary

Keep open. Current main still lacks a persisted Control UI preference or toolbar control for expanding tool-call output by default, while this PR is useful but stale against the stateful expansion path now on main. The main blocking issue is that the PR drives raw disclosure `open` state instead of the existing expansion-state seam, which would collapse manually opened tool output again on rerender when the preference is off.

## What This Changes

The PR adds a browser-local `chatExpandToolCalls` setting, desktop/mobile Control UI chat toolbar buttons, render plumbing, English copy, and fixture/test updates so tool-call output can expand by default.

## Best Possible Solution

Land a refreshed implementation that adds the browser-local preference and toolbar control, wires it into `autoExpandToolCalls` plus `syncToolCardExpansionState`, preserves manual expand/collapse state across streaming and history rerenders, and uses accurate disabled-state copy for onboarding versus hidden tool calls.
