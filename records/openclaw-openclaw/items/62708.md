## Summary

Keep open. Current main has not implemented the requested cleanup: `claudeCliSessionId` is still part of `SessionEntry`, still dual-written for `claude-cli`, still cleared/preserved through several session lifecycle paths, and still has an explicit legacy read fallback with tests. The issue is recent, author association is `NONE`, has no protected labels, and the provided timeline only shows external RemoteClaw upstream-tracking cross-references rather than a merged OpenClaw fix or canonical duplicate.

## Best Possible Solution

Keep this issue open as an actionable internal cleanup. The best implementation would intentionally sunset the compatibility field: remove `claudeCliSessionId` from `SessionEntry`, stop dual-writing and passthrough in CLI/session/gateway/reset/usage paths, update tests that currently assert legacy persistence, and decide whether old stores should be handled by a one-time migration or doctor/fix path rather than a live runtime fallback.
