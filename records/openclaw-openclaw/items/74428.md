## Summary

Keep open. Current main does not implement the requested MCP serve scope selector, and this PR is a focused implementation with docs/tests/changelog coverage; the remaining decision is maintainer review of auth-scope behavior and supersession of the older overlapping PR, not conservative cleanup closure.

## What This Changes

This PR adds a repeatable `openclaw mcp serve --scope` option limited to `operator.read`, `operator.write`, and `operator.approvals`, threads selected scopes through the MCP server/bridge to `GatewayClient`, documents the flag, adds regression tests, and updates the changelog.

## Best Possible Solution

Land this or an equivalent narrow implementation after maintainer review, preserving the existing default read/write/approvals request when `--scope` is omitted, limiting explicit MCP bridge scopes to the intended allowlist, documenting/testing the contract, and then resolving #74030 as superseded if this branch replaces it.
