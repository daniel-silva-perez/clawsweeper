## Summary

Keep this PR open. Current main still lacks the requested explicit-agent-mention precedence behavior, and the PR remains the active unmerged implementation candidate. It should not be closed as obsolete, but it also should not land as-is because the current patch still mixes unrelated install/test changes and carries functional plus install-time security review risks.

## Best Possible Solution

Keep this PR open for maintainer review, but require a narrowed revision before landing: split the postinstall and PDF-test changes, restore Discord `memberRoleIds`, make suppression identify another OpenClaw agent rather than any platform mention, decide the channel opt-in policy, and update the channel SDK docs/tests for the new public mention-policy contract.

## Risks / Open Questions

- Closing this PR would discard the active implementation while current main still lacks the behavior requested by #67445.
- Landing as-is would drop Discord `memberRoleIds` from the preflight payload, losing role context consumed by downstream Discord processing.
- The PR treats generic platform mentions as proof that another OpenClaw agent was mentioned; for some channels that can suppress normal group chatter mentioning a user, role, or everyone.
- The PR bundles unrelated postinstall cleanup and a PDF test model constant change with mention routing, which expands review scope and regression risk.
- The postinstall diff changes recursive deletion behavior during install and needs boundary checks or a separate security-reviewed PR before landing.
