## Summary

Keep open. Current main still lacks the requested explicit-agent-mention precedence behavior, and the PR remains an active implementation candidate, but the current patch should not land as-is because it is incomplete for Discord, can suppress on generic non-agent mentions, drops Discord role context, and includes unrelated install-time deletion logic.

## What This Changes

The PR adds an opt-in shared inbound mention policy for always-on agent suppression, wires several channel plugins to it, adds mention-gating tests, and also changes bundled-plugin postinstall pruning plus a PDF test model constant.

## Best Possible Solution

Keep the useful mention-gating direction, but land it as a narrowed channel/SDK change: add a typed fact for explicit mentions of other configured OpenClaw agents, wire only channels that can supply that fact correctly, keep Discord role context intact, document the new plugin-facing contract, and move postinstall/PDF-test changes to separate reviews.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restore Discord member roles in preflight context:** `extensions/discord/src/monitor/message-handler.preflight.ts:1083`
  - body: The PR removes `memberRoleIds` from the preflight payload, but downstream Discord context construction forwards that field as `MemberRoleIds`. Without it, guild messages lose sender role context used by role-aware routing, auth, and prompts.
  - confidence: 0.92
- **[P1] Do not treat generic mentions as agent mentions:** `src/channels/mention-gating.ts:136`
  - body: The suppression condition uses `hasAnyMention` as proof that another agent was mentioned. Several callers populate that from generic platform mentions, so always-on agents can drop normal group messages that mention a human, role, or everyone.
  - confidence: 0.9
- **[P1] Guard postinstall pruning before recursive removal:** `scripts/postinstall-bundled-plugins.mjs:670`
  - body: `pruneBundledDistPluginNodeModules` removes nested `node_modules` paths without first proving `dist/extensions` is a real directory inside the package root. If that parent is redirected, postinstall can delete outside the OpenClaw install tree.
  - confidence: 0.87
- **[P2] Keep Discord wired into the suppression policy:** `extensions/discord/src/monitor/message-handler.preflight.ts:966`
  - body: The final diff deletes the Discord opt-in and drop path, so always-on Discord guild messages still will not suppress when another agent is explicitly targeted. That leaves a labeled channel and related Discord reports unfixed.
  - confidence: 0.84
