## Summary

Keep this issue open. Current main still documents the Discord elevated allowFrom fallback, but runtime only gets fallback allowlists from a channel plugin elevated hook, and the bundled Discord plugin does not provide that hook. `sandbox explain` also still reads only `tools.elevated.allowFrom.<channel>` for elevated diagnostics, so the documented fallback path remains unresolved. The related open PR #53259 uses closing syntax and remains the implementation candidate rather than a reason to close early.

## Best Possible Solution

Keep this issue open and review, land, or supersede #53259. The fix should be Discord/channel-owned: expose or wire an `elevated.allowFromFallback` that resolves `channels.discord.allowFrom` only when `tools.elevated.allowFrom.discord` is omitted, reuse the same effective resolution in `sandbox explain`, and add regression coverage for omitted fallback, explicit empty override, and diagnostics consistency.
