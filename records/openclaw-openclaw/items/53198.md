## Summary

Keep open. Current main still documents the Discord elevated allowFrom fallback, but runtime authorization only receives a fallback from a channel plugin hook that Discord does not expose, and sandbox explain still reads only explicit tools.elevated.allowFrom values. The open #53259 PR remains the implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests that Discord elevated authorization and sandbox diagnostics honor the documented fallback from channels.discord.allowFrom when tools.elevated.allowFrom.discord is omitted.

## Best Possible Solution

The end state should be a Discord-owned elevated fallback adapter or shared resolver that feeds both runtime elevated authorization and sandbox explain, with regression coverage for omitted fallback, explicit empty override, and diagnostics consistency.
