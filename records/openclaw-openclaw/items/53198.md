## Summary

Keep this issue open. Current main still documents the Discord elevated allowFrom fallback, but runtime resolution only uses a channel plugin elevated fallback hook and the bundled Discord plugin still does not provide one. `sandbox explain` also still reads only `tools.elevated.allowFrom.<channel>`. The provided GitHub context shows #53259 is open, unmerged, and uses closing syntax for this issue, so it remains an implementation candidate rather than a reason to close early.

## Best Possible Solution

Keep this issue open and review, land, or supersede #53259. The fix should be Discord/channel-owned: expose an `elevated.allowFromFallback` that resolves `channels.discord.allowFrom` only when `tools.elevated.allowFrom.discord` is omitted, reuse the same effective resolution in `sandbox explain`, and add regression coverage for omitted fallback, explicit empty override, and diagnostics consistency.
