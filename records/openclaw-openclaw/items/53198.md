## Summary

Keep open. Current main still documents the Discord elevated allowFrom fallback, but runtime fallback resolution depends on a channel plugin `elevated.allowFromFallback` hook that the bundled Discord plugin still does not expose; `sandbox explain` also continues to read only explicit `tools.elevated.allowFrom.<channel>` values. The linked open PR #53259 remains the implementation candidate rather than a reason to close early.

## What This Changes

Discord elevated exec should accept `channels.discord.allowFrom` as the fallback allowlist when `tools.elevated.allowFrom.discord` is omitted, and `sandbox explain` should report the same effective authorization path.

## Best Possible Solution

Keep the issue open and review, land, or supersede #53259. The end state should be a Discord-owned elevated fallback adapter, reused by runtime elevated auth and `sandbox explain`, with regression coverage for omitted fallback, explicit empty override, and diagnostics consistency.
