## Summary

Keep open. Current main still lacks a blocking plugin interception point for streamed preview creates/edits or in-place final preview edits; normal outbound sends can run `message_sending`, but the streaming preview paths update visible channel messages directly. The related Discord before-deliver PR was useful context but closed unmerged and does not solve preview-edit interception, while the per-chat streaming-mode request is adjacent rather than a substitute.

## What This Changes

Request a blocking plugin hook or streaming override so plugins can inspect, modify, or cancel full streamed outbound content before it becomes visible in chat.

## Best Possible Solution

Define a shared, documented blocking pre-delivery contract for streamed preview creates, preview edits, and final preview edits, then wire it through Telegram and the generic draft-preview finalizer before adding channel-specific tests. If maintainers choose the safer operations-first path, add a scoped streaming-disable override as a separate feature, not as the only content-filtering fix.

## Reproduction Assessment

Yes. A high-confidence reproduction path is a Telegram plugin that registers `message_sending`, enables `channels.telegram.streaming.mode="partial"`, and observes that partial/final preview edits use `draftStream.update`/`editMessageTelegram` while the test-covered path avoids `deliverReplies`, where the hook currently runs.

## Solution Assessment

Unclear. The issue’s options are directionally valid, but a final-only `message_edit_sending` hook is not sufficient for the sensitive-data use case because partial previews can already expose content; the maintainable fix should cover every user-visible preview update with clear metadata and cancellation semantics.

## Risks / Open Questions

- A final-only hook would not prevent sensitive data that appears in earlier partial preview edits from becoming visible first.
- Changing `message_sending` to run on every preview edit could add latency, rate-limit pressure, and repeated plugin side effects unless the contract is explicit and tested.
- The fix likely needs a shared contract across Telegram, Discord, Slack, Mattermost, and Signal delivery paths so channel behavior does not diverge.
