## Best Possible Solution

Keep this PR open for maintainer review and iteration. The best fix is either to ensure `sandbox.mode: "off"` cannot reach Docker at all, or to use a narrow shared Docker-daemon-unavailable classifier only where probing is genuinely optional, without masking required-sandbox failures. Add focused regression coverage for #73586’s cron/heartbeat/sub-agent mode-off path, custom/default sandbox images, browser image setup, and doctor behavior; add a user-facing changelog entry if the behavior fix lands.
