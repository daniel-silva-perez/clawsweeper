## Summary

Keep this PR open. Current main still has the unguarded Docker image-inspect behavior that the PR is trying to address, and the linked bug #73586 remains open. The PR is an active implementation candidate, but it still needs maintainer review because the proposed patch changes Docker image resolution/pull behavior, lacks regression coverage for the reported mode-off startup path, and appears to have an import/export gap in the shared daemon helper.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to first verify which cron, heartbeat, or sub-agent startup path is touching Docker while `sandbox.mode` is off, then either prevent that path from probing Docker before sandbox context resolution or add a narrow shared Docker-daemon-unavailable classifier with tests. The final fix should preserve the documented default-image fail-fast policy unless maintainers explicitly decide to change image pull/substitution behavior.

## Risks / Open Questions

- The PR changes Docker image resolution policy by automatically pulling and tagging `debian:bookworm-slim` as the default sandbox image, conflicting with current docs/tests that require the OpenClaw sandbox image to be built explicitly because it carries required tooling.
- The PR also attempts `docker pull` for arbitrary custom images, which is a supply-chain-sensitive behavior change and may unexpectedly contact registries during sandbox startup.
- The proposed daemon-unavailable classifier uses broad substrings such as `no such file or directory` and `connection refused`; if applied on sandbox-enabled paths, it could hide unrelated Docker failures unless narrowed and tested.
- The patch imports `isDockerDaemonUnavailable` from `../agents/sandbox.js` in doctor-sandbox.ts but does not update the sandbox barrel export, so the proposed branch still appears to need a build/typecheck fix.
- The provided PR context does not include focused regression coverage for the reported cron, heartbeat, or sub-agent mode-off startup path with Docker unavailable.
