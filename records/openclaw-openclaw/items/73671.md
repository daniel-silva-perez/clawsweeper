## Summary

Keep this PR open. Current main still lacks Docker-daemon-unavailable handling on the sandbox image-probe paths, the linked bug remains open, and the PR is an active implementation candidate rather than obsolete. The PR also still needs maintainer review because its diff changes Docker image resolution behavior and appears to have build/API-surface issues.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to verify the exact startup path that reaches Docker while sandbox mode is off, then either prevent Docker probing before Docker is touched or add a narrow shared Docker-daemon-unavailable classifier with regression tests for mode-off cron, heartbeat, and sub-agent sessions. The PR should avoid changing default-image pull/substitution policy unless maintainers explicitly approve that product and supply-chain behavior.
