Keep this PR open. Current main still lacks Docker-daemon-unavailable handling in the sandbox image probe paths, the linked bug #73586 is still open, and this PR is an active implementation candidate rather than work already implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to trace the mode-off cron/heartbeat/sub-agent startup path that still reaches Docker, then either prevent Docker probing when the resolved sandbox context is null or add a narrow shared Docker-daemon-unavailable classifier for the image-probe helpers with focused regression tests for base image, custom image, browser image, and doctor behavior. Avoid masking unrelated Docker errors and keep sandbox CODEOWNERS in the loop.

## Risks / Open Questions

- The current PR changes a security-sensitive sandbox runtime path; even though it does not add dependencies or CI/package execution surfaces, it should keep `@openclaw/secops` review.
- The PR’s stderr classifier includes broad connection-failure strings. If too broad, it could hide unrelated Docker/image-inspect errors and make real sandbox setup failures harder to diagnose.
