## Best Possible Solution

Keep this PR open for maintainer review and iteration. The best fix is to ensure `sandbox.mode: "off"` cannot reach Docker at all, or to add a narrow shared Docker-daemon-unavailable classifier at the image-probe helpers without masking unrelated Docker failures. Add focused regression coverage for default/custom sandbox images, browser image setup, doctor behavior, and the cron/heartbeat/sub-agent mode-off path tied to #73586.
