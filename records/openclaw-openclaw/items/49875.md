## Summary

Keep open. Current main still does not implement the PR's intended broadcast `runInboundClaim` dispatch for regular, non-plugin-bound channel messages, and the existing tests still encode the old no-broadcast behavior. The PR is useful but still needs functional fixes and updated dispatch-path coverage before it can be considered obsolete or safe to merge.

## Best Possible Solution

Keep this PR open as useful but incomplete implementation work. The best path is to refresh it against current main so regular, non-plugin-bound messages can be claimed by broadcast `runInboundClaim` before normal agent reply dispatch, while preserving plugin-owned binding precedence, avoiding broadcast after plugin fallback, preserving fast-abort behavior for `stop`/`abort` messages, calling `markIdle` on a claimed early return, and replacing the old no-broadcast assertion with focused regression tests for claimed and unclaimed regular messages.
