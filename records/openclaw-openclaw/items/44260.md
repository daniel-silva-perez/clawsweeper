## Summary

Keep this PR open. Current main still does not implement 0o644 for user-facing Write/Edit outputs, and the PR remains useful, but the proposed patch is incomplete because it only handles host workspace Write/Edit and does not propagate a mode through sandboxed Write/Edit paths. It should be revised together with the related sandbox permission regression in #44077.

Keep this PR open and request a focused revision: add nullable-safe mode plumbing through writeFileWithinRoot, the legacy path, the pinned helper, SandboxFsBridge, buildPinnedWritePlan, and sandbox mutation helpers; pass 0o644 from both host and sandboxed user-facing Write/Edit operations; keep internal defaults at 0o600; coordinate with #44077; and add targeted permission regression tests.
