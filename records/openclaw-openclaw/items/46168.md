## Summary

Current main still has the skill discovery bug this PR targets: it caps sorted candidate directories with `maxSkillsLoadedPerSource` before checking for `SKILL.md`, so early non-skill directories can hide a later valid skill even when `maxCandidatesPerRoot` allows scanning it. The skills fix and regression test direction are useful, but this PR should not merge as-is because its head also carries unrelated dependency/lockfile, Matrix/Zalo package, Telegram, and TUI changes outside the stated skills fix.

## What This Changes

The PR changes skills discovery to scan candidate directories up to `maxCandidatesPerRoot` before applying the loaded-skill cap, adds a regression test for non-skill directories before a real skill, and also modifies dependency/lockfile, Matrix/Zalo package metadata, Telegram retry/tests, and TUI backspace behavior.

## Best Possible Solution

Keep this PR open as useful source work, but replace or repair it with a narrow branch containing only the skills discovery change, the focused regression test, and any required changelog credit. Exclude all package/lockfile, channel, Telegram, Zalo, Matrix, and TUI changes, then run the targeted skills test and the normal changed gate in the maintainer flow.
