## Summary

Keep open. Current main still has the skill discovery bug this PR targets: child directories are capped with the loaded-skill limit before the loader checks whether they contain `SKILL.md`. The focused skills fix is useful, but the PR branch should not merge as-is because it also carries unrelated dependency/lockfile, Matrix/Zalo package, Telegram, and TUI changes.

## What This Changes

The PR changes skills discovery to scan candidate directories up to `maxCandidatesPerRoot` before applying the loaded-skill cap, adds a regression test for non-skill directories before a real skill, and also modifies dependency/lockfile, Matrix/Zalo package metadata, Telegram retry/tests, and TUI backspace behavior.

## Best Possible Solution

Land a narrow skills-only fix that scans up to `maxCandidatesPerRoot`, keeps `maxSkillsLoadedPerSource` as the loaded-result cap, preserves the existing realpath and file-size protections, and adds the unequal-limit regression test. The unrelated dependency, channel, Telegram, Zalo, Matrix, and TUI changes should be excluded or handled separately.
