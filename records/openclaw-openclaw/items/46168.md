## Summary

Keep open. Current main still has the skills discovery scan-cap bug this PR targets, and the skills hunk plus regression test match the intended fix, but the branch is not mergeable as-is because it mixes that fix with unrelated package, lockfile, channel, Telegram, Zalo, Matrix, and TUI changes, including supply-chain-sensitive dependency downgrades.

## What This Changes

The PR changes workspace skills discovery to scan candidate directories up to `maxCandidatesPerRoot` before applying `maxSkillsLoadedPerSource`, adds an unequal-limit regression test, and also modifies dependency metadata, Telegram retry/tests, TUI backspace handling, and Matrix/Zalo package metadata.

## Best Possible Solution

Land a narrow skills-only implementation that separates candidate scanning from loaded-result limiting, adds the unequal-limit regression, and splits dependency/channel/TUI work into separate reviewed changes only if those changes are still desired.
