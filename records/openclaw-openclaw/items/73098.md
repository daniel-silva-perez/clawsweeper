The PR adds a post-merge safety guard that allows agent-facing gateway `config.patch`/`config.apply` changes to `diagnostics.cacheTrace.enabled` and content-capture flags only when the resulting config keeps all raw content capture disabled, plus regression tests for safe and unsafe patches.

## Best Possible Solution

Land this narrow gateway-tool guard after adding the missing changelog entry and confirming the targeted checks, preserving the fail-closed default for cache-trace content capture and `filePath` changes.

## Solution Assessment

Yes, with the changelog added. Checking the merged `nextConfig` is the narrow maintainable fix because it permits cache-decision observability only when the final config keeps all raw content capture fields disabled.
