## Summary

Keep open. The latest PR head addresses the earlier remote-env security concern by making explicit config-disabled cache-trace content capture win over truthy env variables, while keeping the agent-tool allow path narrow. Current main still blocks the safe cache-trace patch from #72883, so this remains a live implementation PR rather than a cleanup-close candidate.

## What This Changes

The branch special-cases agent-facing gateway config.patch/config.apply so diagnostics.cacheTrace enabled/include paths can change only when the resulting config keeps message, prompt, and system capture disabled, updates cache-trace env precedence, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Land this PR after normal maintainer review and CI completion, preserving the post-merge config guard and the runtime rule that explicit config-disabled content capture cannot be re-enabled by env overrides. A small docs clarification for the new env precedence would make the operator contract clearer.
