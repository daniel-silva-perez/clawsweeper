The PR allows agent-facing gateway `config.patch`/`config.apply` mutations of `diagnostics.cacheTrace.enabled` and content-capture flags only when the resulting cache trace config has all raw content capture disabled, with regression coverage for safe and unsafe patches.

## Solution Assessment

Yes, with the changelog added. Checking the merged `nextConfig` is the narrow maintainable approach because it permits cache-decision observability only when the final config keeps all raw content capture fields disabled.
