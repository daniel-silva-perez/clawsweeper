## Summary

Keep this PR open. Current main still resolves `--agent <id> --to <target>` through the selected agent's implicit main-session key before the `--to` path can derive a target-specific key. The PR remains a focused implementation candidate, but it should be rebased/integrated carefully with the newer default-agent routing work.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. Rebase or port the change onto current `src/agents/command/session.ts`, derive selected-agent `--to` keys before the implicit agent-main fallback, preserve explicit `sessionKey` and `sessionId` precedence, keep qualified `--to` rebinding to the selected agent scope, and retain the focused regression tests. Review it alongside #44448, but do not close this PR until the explicit `--agent + --to` behavior lands on main or a maintainer chooses a different canonical fix.
