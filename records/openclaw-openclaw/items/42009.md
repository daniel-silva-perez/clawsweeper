## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: `openclaw agent --agent <id> --to <target>` resolves through the selected agent's implicit main-session key before `--to` can derive an agent-scoped peer key. The PR remains a focused implementation candidate, and the related merged default-agent fix is adjacent rather than a replacement.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. The best path is to rebase or integrate it onto current main, derive selected-agent `--to` keys before the implicit agent-main fallback, preserve explicit `sessionKey` and `sessionId` precedence, retain qualified `--to` rebinding to the selected agent scope, and keep the focused regression tests. It should also be considered alongside the related open routing report #44448, but not closed until this behavior lands on main.
