## Summary

Keep this PR open. Current main still does not implement the intended `--agent <id> --to <target>` behavior: the resolver derives the selected agent's main-session key before the `--to` path can derive a target-specific key. The merged default-agent work in #72414 is related, but it does not replace this explicit-agent routing fix.

## Best Possible Solution

Keep this PR open and review or port the focused fix. The resolver should derive selected-agent `--to` keys before the implicit agent-main fallback, preserve explicit `sessionKey` and `sessionId` precedence, keep qualified `--to` rebinding scoped to the selected agent, and retain the regression tests. Coordinate with #44448 only for the broader channel delivery-context issue.
