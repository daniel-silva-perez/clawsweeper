## Summary

Keep this PR open. Current main still collapses explicit `--agent <id> --to <target>` requests to the selected agent's main session before the `--to` target can derive an agent-scoped session key. The shipped default-agent routing work is related but narrower, and the PR remains a focused implementation candidate with targeted regression coverage and no evident supply-chain or infrastructure drift in the provided diff.

## Best Possible Solution

Keep this PR open and review or port the focused resolver change. The best fix is to derive selected-agent `--to` keys before the implicit agent-main fallback, preserve explicit `sessionKey` and `sessionId` precedence, keep requested-agent scoping for session-id lookup and qualified `--to` rebinding, and retain the regression tests. Treat #44448 as a broader delivery-context follow-up rather than a replacement for this PR.
