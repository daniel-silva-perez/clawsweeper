## Summary

Keep this PR open. Current main still routes WhatsApp inbound text and successful audio transcripts into raw model-facing fields, while the submitted patch only wraps legacy envelope/body strings and group-history entries. The hardening goal remains relevant, but the PR is not merge-ready because it misses the active `BodyForAgent`/`BodyStripped` prompt path and can destabilize WhatsApp echo suppression.

## Best Possible Solution

Keep this PR open as an incomplete security-hardening candidate. The maintainer path is to revise or replace it so WhatsApp separates deterministic echo/envelope text from prompt-facing content, wraps or otherwise isolates the actual `BodyForAgent`/transcript/reply-context/group-history text that reaches the agent, keeps trusted operator `messagePrefix` outside untrusted markers, and adds focused WhatsApp regression tests before landing.
