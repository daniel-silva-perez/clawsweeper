## Summary

Keep this PR open. Current main still reports the bundled Discord skill against the single-account `channels.discord.token` requirement, while docs and config types support account-scoped Discord tokens. The PR remains a focused implementation candidate for the linked bug, but it should address the existing review concern that `channels.discord.accounts` is too broad as a truthy requirement.

## Best Possible Solution

Keep the PR open and ask for a small revision: add the OR-config requirement surface consistently across parsing, runtime eligibility, status formatting, and tests, but make the Discord gate token-aware rather than satisfied by any non-empty `accounts` object. A regression test should cover `channels.discord.accounts.default.token` as ready and empty/tokenless `accounts` as not ready.
