## Summary

Keep this issue open. Current main still hardcodes the requested Safety, OpenClaw CLI Quick Reference, and self-update guidance in the core prompt builder. The only supported customization paths I found are whole-prompt replacement, plugin/hook prompt mutation, and provider-owned overrides for three named sections; there is still no SAFETY.md-style or template-backed override for OpenClaw-owned prompt sections. The closed #25938 PR remains useful prior art, while open #53942 is related but narrower.

## Best Possible Solution

Keep this open for maintainer product judgment. The best path is to decide whether OpenClaw-owned prompt sections such as Safety, Tool Call Style, CLI Quick Reference, and Self-Update should become documented schema-backed or workspace-file-backed overrides. If yes, implement it in src/agents/system-prompt.ts and src/agents/workspace.ts with explicit trust-boundary docs and regression tests. If maintainers decide those sections should stay fixed, document systemPromptOverride, before_prompt_build hooks, and provider overlays as the supported escape hatches and coordinate that decision with #53942.
