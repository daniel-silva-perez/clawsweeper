## Summary

Keep open. Current main still hardcodes the requested OpenClaw-owned system prompt sections and only supports whole-prompt replacement, plugin/hook mutation, or provider-owned overrides for three named sections. The directly related implementation PR #25938 is closed and unmerged, #53942 is open but narrower, and a recent maintainer review explicitly kept this open for product/security judgment.

## Best Possible Solution

Keep this open for maintainer product/security judgment. The best path is to decide whether OpenClaw-owned sections such as Safety, Tool Call Style, CLI Quick Reference, and Self-Update should become schema-backed or workspace-file-backed overrides. If yes, implement that in src/agents/system-prompt.ts and src/agents/workspace.ts with explicit trust-boundary docs and regression tests; if not, document systemPromptOverride, before_prompt_build, agent:bootstrap, and provider overlays as the supported customization paths and coordinate that answer with #53942.
