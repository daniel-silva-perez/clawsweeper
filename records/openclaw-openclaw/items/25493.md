## Summary

Keep this issue open. Current main still hardcodes the requested Safety and OpenClaw CLI Quick Reference prompt sections, and the supported customization paths remain whole-prompt replacement, provider/runtime overrides for only three named sections, and hooks. The related implementation PR #25938 is closed unmerged, and the related #53942 is narrower rather than a full replacement for this broader template/file override request.

## Best Possible Solution

Keep this issue open. The best maintainer path is to decide whether selected OpenClaw-owned prompt sections should become documented, schema-backed or workspace-file-backed template overrides, then implement that through src/agents/system-prompt.ts, workspace/config plumbing, docs, and regression tests. If maintainers instead want Safety, CLI reference, and self-update guidance to stay fixed, document systemPromptOverride, prompt hooks, and provider overlays as the explicit escape hatches and coordinate that decision with #53942.
