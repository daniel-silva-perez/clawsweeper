## Summary

Keep this PR open. Current main still lacks the proposed global/per-agent sender-scoped tool policy, so the branch is not obsolete, but the patch still changes a security-sensitive tool capability boundary and leaves channel-scoped sender matching dependent on raw provider strings rather than OpenClaw's canonical channel IDs.

## What This Changes

The PR adds global and per-agent `toolsBySender` config, introduces explicit `channel:<provider>:<senderId>` sender keys, and threads the resulting sender policy through core, plugin, and bundled-tool filtering.

## Best Possible Solution

Keep the PR open for maintainer review and finish it on the contributor branch: canonicalize provider identities before channel-scoped sender matching, settle and document wildcard behavior for runs without sender context, add targeted regression tests for core/plugin/bundled filtering, and keep generated config metadata aligned.
