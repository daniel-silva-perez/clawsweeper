## Summary

Keep open. Current main still lacks the requested global/per-agent sender tool policy, so the PR is not obsolete; however, the branch changes a security-sensitive capability boundary and still has a concrete channel-alias matching bug plus missing targeted proof before it should merge.

## What This Changes

This PR adds global and per-agent `toolsBySender` config, explicit channel-scoped sender keys, and sender-policy filtering across main, plugin, and bundled tool resolution paths.

## Best Possible Solution

Land the feature only after channel-scoped sender keys use canonical channel ids consistently, the wildcard/no-sender semantics are explicit, generated config schema and user docs are aligned, and targeted regression tests cover main, plugin, group, and bundled-tool filtering paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Canonicalize providers before sender key matching:** `src/config/group-policy.ts:250-251`
  - body: The channel-scoped lookup lowercases the raw `messageProvider` value, so aliases such as `imsg` or plugin aliases can miss a configured canonical key like `channel:imessage:<senderId>` or `channel:msteams:<senderId>` and fall through to a less specific policy.
