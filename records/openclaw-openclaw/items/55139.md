## Summary

Keep this PR open. Current main has not implemented the PR’s intended startup recursion hardening: `resolveConsoleSettings()` still calls `readLoggingConfig()` before setting `loggingState.resolvingConsoleSettings`, and patched console timestamping still calls `getConsoleSettings()` without a guard. The PR is focused and the author addressed review feedback, but it needs maintainer review and a redaction-safe regression test before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a revised implementation that sets the recursion guard before any config read, keeps a safe non-recursive console path that still redacts output, and adds a targeted regression test for a config warning emitted while console capture and timestamp prefixing are active.
