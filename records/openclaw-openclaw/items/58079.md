## Summary

Keep this PR open. Current main still does not expose the PR’s requested full model-facing context on `before_tool_call`; the public event type, docs, tool wrappers, and embedded-runner wiring only pass tool name, params, run/tool IDs, and context IDs. The PR is a plausible narrow plugin API change split out of the broader closed guardrails PR #6095, and the remaining related issues are adjacent rather than exact superseding implementations.

## Best Possible Solution

Keep the PR open for maintainer review or ask for a cleaned-up revision. The best implementation is an additive plugin contract change that adds optional context fields, plumbs cloned/frozen snapshots from the embedded runner into built-in, unwrapped MCP/LSP-style, and client tool paths, updates public plugin docs/contract tests, and explicitly documents the expanded data access for installed plugins.
