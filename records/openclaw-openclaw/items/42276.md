## Summary

Keep open. Current main still has the same TUI gap: backend/session code can accept and emit reasoning stream data, but the TUI slash-command surface still omits `stream` and the TUI agent-event handler still ignores `stream: "thinking"`, so the requested live reasoning-stream experience is not implemented.

## What This Changes

Request live OpenAI/Grok-style reasoning updates in the OpenClaw TUI for `/reasoning stream`, with clearer command discovery and preferably a rewritable chat/status/footer presentation.

## Best Possible Solution

Land a focused TUI fix that uses the existing session `reasoningLevel` and agent `stream: "thinking"` contract: expose `stream` in `/reasoning` help/completions/usage, render live thinking updates only while the session is in stream mode, clear the transient display on final/error/abort, and align TUI/thinking docs with the shipped behavior.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is available: current `helpText()`/slash command metadata omit `stream`, and a synthetic active-run `AgentEvent` with `stream: "thinking"` would fall through `handleAgentEvent` without a chat or status update.

## Solution Assessment

Yes, a narrow TUI-client repair is the best direction. Core session validation and backend event emission already exist, so changing the TUI command surface, TUI event consumption, tests, and docs is smaller and safer than changing gateway or agent-stream semantics.
