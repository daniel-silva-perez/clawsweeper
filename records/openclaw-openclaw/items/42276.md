## Summary

Keep this issue open. Current main still does not implement the requested TUI reasoning-stream experience: the TUI command metadata/help still advertises only `on|off`, the command handler usage still says `on|off`, and the TUI event handler only renders chat deltas plus `tool`/`lifecycle` agent streams, not `stream: "thinking"` reasoning events. The provided discussion is useful: #42312 targeted part of this gap, but it was closed unmerged, and later related PRs appear to target gateway/web reasoning stream rather than fully resolving this TUI request.

## Best Possible Solution

Keep the issue open and land a focused core/TUI fix: add `stream` to TUI `/reasoning` completion/help/usage, render `stream: "thinking"` agent events as a transient live reasoning row or footer/status update when the session is in reasoning stream mode, clear it on final/error/abort, and add TUI tests for render/ignore/cleanup behavior. Also align docs and clarify existing persistent options (`reasoningLevel` session patch and `agents.list[].reasoningDefault`) or add the missing persistence surface if users still see resets after gateway restart.
