## Summary

Keep this issue open. Current main still does not implement the requested OpenAI/Grok-style TUI reasoning stream: TUI command metadata and usage still omit `stream`, the TUI agent-event handler does not render `stream: "thinking"`, and the backend only emits that event when an `onReasoningStream` callback is supplied. The prior focused PR in the discussion, #42312, was closed unmerged, so it is not a basis for closing this issue.

## Best Possible Solution

Keep the issue open and land a focused TUI/gateway fix: expose `stream` in `/reasoning` completions/help/usage, ensure gateway-backed and embedded TUI runs emit or receive `stream: "thinking"` reasoning events when `reasoningLevel` is `stream`, render those updates as a transient live reasoning row or footer/status update, clear it on final/error/abort, and add TUI tests for render/ignore/cleanup behavior. Then align the public thinking and TUI docs around the shipped behavior and clarify the existing persistence surfaces (`sessions.patch` and `agents.list[].reasoningDefault`).
