## Summary

Current main still has the reported contact-backed Feishu sender-name path enabled by default, and live peer listing still uses the Feishu contact directory. The issue remains valid, but an overlapping maintainer-labeled PR is open and the issue is assigned for maintainer follow-up, so this should stay open rather than being auto-closed or queued as a competing repair.

## What This Changes

Remove the Feishu plugin's contact-directory dependency from ordinary sender-name resolution and gate contact-backed directory listing behind explicit opt-in behavior.

## Best Possible Solution

Make basic inbound Feishu messages contact-free by default: use verified event or chat-member metadata when available, fall back quietly to the stable sender id, and require explicit opt-in for contact-backed profile lookup and live user-directory listing with docs and regression tests.
