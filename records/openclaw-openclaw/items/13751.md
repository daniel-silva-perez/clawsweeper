## Summary

This issue should stay open because current main and the latest release still keep Feishu sender-name lookup contact-backed by default, and the related maintainer PR is open rather than merged.

## What This Changes

The issue asks for Feishu sender-name resolution and live peer listing to stop requiring broad Feishu contact-directory read access by default.

## Best Possible Solution

Make ordinary inbound Feishu messages contact-free by default: use verified event or chat-member metadata where available, fall back quietly to the stable sender id, and require explicit opt-in for contact-backed profile lookup and live user-directory listing, with docs and regression tests covering the default behavior.
