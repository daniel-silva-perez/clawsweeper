## Summary

Keep this issue open. Current main still performs Feishu sender-name resolution through the contact API by default, still documents/tests `resolveSenderNames` as default-on, and still uses live contact-directory lookup for Feishu peer listing.

## Best Possible Solution

Keep this issue open. Make basic Feishu inbound bot operation contact-free by default: use webhook sender fields when available, otherwise use open_id/user_id without calling `contact.user.get`; keep profile lookup behind an explicit opt-in. For directory peers, prefer configured static peers unless live contact-directory lookup is explicitly enabled. Update Feishu config metadata, docs, changelog, and regression tests for the new default.
