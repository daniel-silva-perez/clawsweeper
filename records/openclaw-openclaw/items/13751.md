## Summary

Current main still has the reported Feishu contact-backed sender-name path enabled by default, and live peer listing still uses the Feishu contact directory. The issue remains valid, but the provided GitHub context shows an assigned maintainer and an open maintainer-labeled overlapping PR, so this should stay open for coordinated maintainer follow-up rather than be auto-closed or queued as a competing repair.

## What This Changes

Request to make basic Feishu sender-name handling contact-free by default, remove the default dependency on organization-wide contact read scope, and gate live contact-directory listing behind explicit operator choice.

## Best Possible Solution

Make ordinary inbound Feishu messages contact-free by default: use verified event or chat-member metadata where available, fall back quietly to the stable sender id, and require explicit opt-in for contact-backed profile lookup and live user-directory listing, with docs and regression tests covering the default behavior.
