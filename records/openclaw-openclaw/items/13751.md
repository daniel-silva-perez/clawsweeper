## Summary

This should stay open: current main and the latest release still use contact-backed Feishu sender lookup by default, and the existing mitigation only suppresses one stale-scope permission notice rather than removing the default contact lookup. The overlapping maintainer PR is still open, so there is no merged fix to close against.

## What This Changes

The issue asks for Feishu sender-name resolution and live peer listing to stop requiring contact-directory read permissions by default, using event/chat metadata or quiet ID fallback unless operators explicitly opt in.

## Best Possible Solution

Make ordinary inbound Feishu messages contact-free by default: use verified event or chat-member metadata where available, fall back quietly to the stable sender id, and put contact-backed profile lookup plus live user-directory listing behind explicit operator opt-in with updated setup docs and regression tests.
