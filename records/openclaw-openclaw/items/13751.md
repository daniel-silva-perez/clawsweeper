## Summary

Current main still has the reported least-privilege problem: Feishu sender-name resolution is enabled by default and calls the contact user API, while live peer listing still enumerates contact users. The issue should stay open because it is valid, assigned for maintainer follow-up, and overlaps open maintainer-labeled PR #73399 rather than being safe to close or queue as a competing automated repair.

## What This Changes

The issue requests making Feishu sender-name handling and related directory behavior work without the broad contact:contact.base:readonly permission by default.

## Best Possible Solution

Make ordinary inbound Feishu messages contact-free by default: use verified event or chat-member metadata when available, fall back quietly to the stable sender id, and put contact-backed profile lookup plus live user-directory listing behind explicit operator opt-in with updated docs and regression tests.
