## Summary

Current main still keeps Feishu sender-name resolution contact-backed and default-on. The inbound path calls `resolveFeishuSenderName` unless `resolveSenderNames` is disabled, the resolver still calls `client.contact.user.get`, and live peer listing still calls `client.contact.user.list`; open PR #73399 overlaps display-name behavior but is still open and preserves contact fallback rather than making basic inbound operation contact-free by default.

## What This Changes

Remove or gate Feishu sender-name and live-directory contact API lookups so basic inbound bot operation does not require organization-wide contact-directory permission.

## Best Possible Solution

Keep this issue open and resolve it with a focused Feishu plugin change: make basic inbound bot operation contact-free by default by using webhook sender fields when available and otherwise falling back to open_id/user_id without calling `contact.user.get`; gate contact-backed profile lookup and live contact-directory peer listing behind explicit opt-in config; coordinate with open PR #73399 so DM display-name fallback, permission-scope reduction, docs, config metadata, changelog, and focused regression tests land coherently.
