## Summary

Keep open. Current main still resolves Feishu sender names through the contact API by default and still uses live contact-directory listing for user peers; the report remains valid, but the issue is already assigned and an open maintainer-labeled PR overlaps the Feishu display-name path, so this should stay in maintainer coordination rather than be auto-closed or queued as a competing repair.

## What This Changes

The issue requests making Feishu sender-name resolution and live user-directory listing avoid broad contact-directory permissions by default.

## Best Possible Solution

Make ordinary inbound Feishu messages contact-free by default: use verified event or chat-member metadata where available, fall back quietly to the stable sender id, and require explicit opt-in for contact-backed profile lookup and live user-directory listing, with docs and regression tests covering the default behavior.
