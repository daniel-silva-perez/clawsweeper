## Summary

Current main still has the reported Feishu contact-directory dependency on the default inbound sender-name path and on live peer listing. The overlapping maintainer PR is open and unmerged, and its patch still preserves contact fallback, so this should stay open for coordinated Feishu maintainer work.

## What This Changes

The issue requests making basic Feishu inbound bot operation avoid `contact:contact.base:readonly` by not using contact-directory APIs for sender-name resolution or default live peer listing.

## Best Possible Solution

Land a focused Feishu plugin change that makes basic inbound messages contact-free by default, uses verified low-scope event or chat-member data when available, falls back quietly to the stable sender id, and gates contact-backed profile lookup plus live user-directory listing behind explicit opt-in config with docs and regression coverage.
