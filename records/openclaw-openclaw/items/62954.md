## Summary

Current main still has the stale-display path: heartbeat-aware runners know the turn is a heartbeat, but session usage persistence has no heartbeat preservation signal and still writes the just-used runtime model/provider into persisted session fields that display surfaces resolve. The direct fix PR #62987 is closed unmerged, so this remains a valid narrow bug.

## What This Changes

The issue asks for heartbeat turns using `agents.defaults.heartbeat.model` to stop overwriting the session's persisted `model` and `modelProvider` display fields.

## Best Possible Solution

Add a heartbeat-aware preservation mode at the session usage persistence boundary so heartbeat turns keep the existing persisted session model identity while normal turns continue updating runtime model metadata and all usage/context/cost fields keep their current semantics.
