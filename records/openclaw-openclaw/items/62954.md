## Summary

Keep open. Current main still has the reported stale-display path: heartbeat-aware runners know the turn is a heartbeat, but the session usage persistence API has no heartbeat preservation flag and still writes the just-used runtime model/provider into the session fields that session list/display code resolves. The direct fix PR #62987 is closed unmerged, so there is no merged canonical implementation yet.

## What This Changes

Heartbeat turns should not overwrite a session's persisted model/modelProvider fields that UI and session APIs use to display the active model.

## Best Possible Solution

Add a heartbeat-aware preservation mode at the session usage persistence boundary so heartbeat turns keep the existing persisted session model identity while normal turns continue updating runtime model metadata and all usage/context/cost fields retain their current semantics.
