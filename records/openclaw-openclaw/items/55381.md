## Summary

Current main still promotes non-`sessions_yield` mixed assistant/tool-call text as subagent output. The specific requested guard is not implemented, the current unit test asserts the opposite behavior, and the related fix PR #55510 is closed unmerged.

## Best Possible Solution

Keep this issue open. If maintainers accept the stricter contract, update `src/agents/subagent-announce-output.ts` so assistant messages containing tool-call blocks do not update `latestAssistantText` or frozen completion output, except for an explicitly supported final-answer marker if desired. Then update `src/agents/subagent-announce-output.test.ts` and completion-capture coverage to prove mixed tool-use planning text is ignored, assistant-only completions still work, `sessions_yield` behavior remains preserved, and any `<final>` exception is intentional.
