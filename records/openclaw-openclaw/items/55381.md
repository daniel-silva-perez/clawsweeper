## Summary

Current main still does not implement the requested broad filter. `summarizeSubagentOutputHistory()` only clears assistant text for `sessions_yield`, then promotes other assistant text from mixed assistant/tool-call messages into `latestAssistantText`; the current unit test explicitly expects non-`sessions_yield` mixed tool-use text to be returned. The related fix PR #55510 is closed and unmerged, so this issue remains actionable.

## Best Possible Solution

Keep this issue open for maintainer follow-up. If the stricter contract is accepted, change `src/agents/subagent-announce-output.ts` so assistant messages containing tool-call blocks do not update `latestAssistantText` or frozen completion output, except for explicit final-answer content such as `<final>...</final>` if that syntax is part of the intended contract. Then update `src/agents/subagent-announce-output.test.ts` and completion-capture coverage to prove ignored mixed turns, preserved assistant-only completions, preserved `sessions_yield` behavior, and any allowed explicit-final exception.
