## Summary

Keep open. Current main still accepts `audio` and `video` in configured model inputs through the config schema, public TypeScript type, generated schema, and configured-catalog preservation path, while the docs and runtime checks still describe/use only text and image behavior for model input capability.

## What This Changes

The issue requests stricter validation for `models.providers.*.models[*].input`, invalid/valid modality tests, CI failure before deployment, and contributor/operator docs for the expected model-input contract.

## Best Possible Solution

Define one explicit configured-model input contract and enforce it consistently across schema, exported types, generated schema, catalog normalization, tests, and docs: either reject unsupported values such as `audio`/`video`, or implement/document their runtime meaning so they are no longer silently accepted metadata.
