## Summary

Keep open. Current main still accepts and preserves audio/video values in configured model inputs, while docs and runtime image handling describe only text/image behavior; the partial manifest-catalog tightening does not implement the requested `models.providers.*.models[*].input` guardrail.

## What This Changes

The issue requests schema validation, targeted tests, CI failure, and docs so unsupported configured model input modalities such as `audio` are rejected before deployment.

## Best Possible Solution

Make the configured model input contract explicit across schema, exported types, generated schema, catalog handling, tests, and docs: either narrow `models.providers.*.models[].input` to text/image and reject unsupported values, or define and implement the runtime meaning of audio/video so they are no longer silently accepted without behavior.
