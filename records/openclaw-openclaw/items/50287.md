## Summary

Keep open. Current main still does not implement the requested guardrail: `models.providers.*.models[*].input` accepts `audio` and `video` in the core Zod schema, public config type, and generated JSON schema, while docs and runtime image-capability checks still describe/use text and image behavior. A current provider-catalog test explicitly preserves configured audio/video modalities, so this needs maintainer contract review before an automated tightening PR.

## What This Changes

Add schema, CI, test, and docs guardrails so unsupported `models.providers.*.models[*].input` modalities are rejected before deployment.

## Best Possible Solution

Keep this open for maintainer contract review. If `models.providers.*.models[*].input` is intended only for chat-turn native inputs, narrow the schema/type/generated schema/provider-catalog handling and add focused valid text/image plus invalid audio/video config-validation tests. If audio/video are intended model metadata, document and implement their runtime meaning so they no longer silently pass without a defined behavior.
