## Summary

Keep open. Current main still accepts and preserves `audio` and `video` in configured provider model inputs, while the documented and runtime capability behavior remains centered on text/image handling. Tightening this now would change a public config/schema/type contract and overturn an explicit preservation test, so this is not conservative cleanup.

## What This Changes

The issue requests validation guardrails for `models.providers.*.models[*].input` so unsupported modalities fail before deployment, with targeted tests, CI coverage, and docs.

## Best Possible Solution

Define one explicit configured-model input contract and enforce it consistently across source schema, exported types, generated schema, catalog normalization, runtime capability checks, tests, and docs: either reject unsupported values such as `audio`/`video`, or implement and document their runtime meaning.
