## Summary

Keep open. Current main still does not implement the requested guardrail: configured model inputs accept `audio` and `video`, and there is an explicit provider-catalog test preserving them, while user docs and runtime chat-turn handling still describe only text/image-native behavior. That is a contract decision, not a conservative cleanup close.

## What This Changes

Add schema, CI, test, and docs guardrails so unsupported `models.providers.*.models[*].input` modalities are rejected before deployment.

## Best Possible Solution

Make the configured model input contract explicit. If the field means chat-turn native inputs, narrow schema, types, generated schema, provider-catalog handling, docs, and tests to text/image. If audio/video are intended model metadata, document and implement their runtime meaning so they no longer silently validate without defined behavior.
