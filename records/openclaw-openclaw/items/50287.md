## Summary

Keep open. Current main and the latest release still accept and preserve `audio` and `video` in configured provider model inputs, while docs and runtime capability checks still treat configured model input as text/image behavior. This is not conservative cleanup-closeable because rejecting those values would change a shipped public config/type contract and an explicit preservation test.

## What This Changes

Add validation guardrails so unsupported values in `models.providers.*.models[*].input` fail before deployment.

## Best Possible Solution

Define one configured-model input contract and enforce it consistently across source schema, exported types, generated schema, catalog normalization, runtime capability checks, tests, and docs: either reject unsupported values such as `audio`/`video`, or implement and document their runtime meaning so they are not silently accepted metadata.
