## Summary

Keep open. Current main still has the CLI compaction resolve-before-initialize path, and the PR is a focused, plausible fix with matching regression coverage; the remaining action is maintainer review and CI/Testbox validation, not a cleanup close.

## What This Changes

The PR injects and calls `ensureContextEnginesInitialized()` before CLI compaction resolves the context engine, and adds a Vitest regression test for the cold-registry call order.

## Best Possible Solution

Land this PR or an equivalent initializer-before-resolver change for CLI compaction after validation, while keeping the subagent-spawn cold-start fix tracked separately. That preserves the existing lazy registry contract and avoids broader top-level registry side effects.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a cold CLI-backed run that reaches `runCliTurnCompactionLifecycle` before any context-engine initializer has run; current main shows that call order and the registry throws the reported unregistered `legacy` error when empty.

## Solution Assessment

Yes. For the CLI compaction half, calling the existing idempotent initializer immediately before the resolver is the narrowest maintainable fix and matches adjacent initialized-first runtime paths.
