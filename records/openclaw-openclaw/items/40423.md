## Summary

Keep this PR open. Current main still exposes only the existing OPENCLAW_SHELL runtime markers for exec, ACP client, and TUI local shell; it does not implement OPENCLAW_AGENT_ID or OPENCLAW_SESSION_KEY on those child-process environments. The PR remains useful but needs rebase, security semantics, and reconciliation with overlapping #72980.

## Best Possible Solution

Keep this PR open for maintainer review or rework. Rebase it onto current main, decide the security and override semantics for OPENCLAW_SESSION_KEY and OPENCLAW_AGENT_ID, implement the markers in the current exec runtime, gateway approval follow-up, ACP client helper, and TUI local-shell paths, update the public docs, add focused tests for explicit-agent precedence, session-key fallback, missing context, caller override behavior, PTY/sandbox/gateway approval execution, ACP client env, and TUI local shell env, then reconcile the overlapping exec-only #72980 before landing.
