## Summary

Keep this PR open. Current main still only exposes the existing OPENCLAW_SHELL markers for exec, ACP client, and TUI local-shell child processes; the requested OPENCLAW_AGENT_ID and OPENCLAW_SESSION_KEY runtime markers are absent from current source, tests, and docs. The related open #72980 covers only a narrower exec session-key subset and does not supersede this broader runtime PR.

## Best Possible Solution

Keep this PR open for maintainer review or rework. Rebase it onto current main, decide the security and override semantics for exported runtime markers, implement the current code paths in src/agents/bash-tools.exec-runtime.ts, src/agents/bash-tools.exec.ts, src/agents/bash-tools.exec-host-gateway.ts, src/acp/client-helpers.ts, src/tui/tui-local-shell.ts, and src/tui/tui.ts, update the public docs, and add focused tests for explicit-agent precedence, session-key fallback, missing context, caller override behavior, PTY/sandbox/gateway approval execution, ACP client env, and TUI local shell env. Reconcile #72980 before landing.
