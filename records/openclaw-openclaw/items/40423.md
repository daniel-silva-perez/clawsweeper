## Summary

Keep this PR open. Current main at 891c7d9f1c1f5e0f52935fadfeacf695b0ef1eb8 still only exposes the existing OPENCLAW_SHELL markers for exec, acp-client, and tui-local child process paths; it does not implement or document OPENCLAW_AGENT_ID or OPENCLAW_SESSION_KEY. The related #72980 is open and narrower, so it does not make this broader PR obsolete.

## Best Possible Solution

Keep this PR open for rebase and maintainer review. The best path is a current-main implementation that defines the approved env contract, threads explicit agent/session context through exec direct/sandbox/PTY and gateway approval follow-up execution, updates the current ACP helper in src/acp/client-helpers.ts, passes current TUI agent/session state into createLocalShellRunner, documents the security semantics, and adds tests for explicit-agent precedence, session-key fallback, missing context, env override behavior, and approved gateway execution.
