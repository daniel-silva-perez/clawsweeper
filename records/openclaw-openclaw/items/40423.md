## Summary

Keep this PR open. Current main still only exposes the existing `OPENCLAW_SHELL` markers for `exec`, `acp-client`, and `tui-local`; the requested `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` markers are absent from current source, tests, and docs. The related #72980 is open but narrower, so it does not supersede this broader PR.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is a rebase onto current main that defines the approved env contract, threads explicit agent/session context through exec direct/sandbox/PTY and gateway approval follow-up execution, updates the current ACP helper in `src/acp/client-helpers.ts`, passes TUI agent/session state into `createLocalShellRunner`, documents the security semantics, and adds tests for explicit-agent precedence, session-key fallback, missing context, env override behavior, and approved gateway execution.

## Risks / Open Questions

- The open #72980 overlaps only the exec session-key subset; maintainers should reconcile scope, but it is not a complete replacement for this broader PR.
