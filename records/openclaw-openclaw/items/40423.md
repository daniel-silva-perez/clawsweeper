## Summary

Keep this PR open. Current main still only injects the existing `OPENCLAW_SHELL` markers for exec, ACP client, and TUI local shell child processes; the requested `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` runtime markers are absent from current source, tests, and docs. The related open #72980 covers only the exec session-key subset, so it does not supersede this broader runtime PR.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is to rebase onto current main, define the approved security and override semantics for these runtime markers, thread explicit agent/session context through exec direct/sandbox/PTY and gateway approval follow-up execution, update the current ACP helper in `src/acp/client-helpers.ts`, pass TUI agent/session state into `createLocalShellRunner`, document the contract, and add focused tests for explicit-agent precedence, session-key fallback, missing context, override behavior, and approved gateway execution.
