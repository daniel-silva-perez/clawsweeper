## Summary

Keep this PR open. Current main still only injects the existing `OPENCLAW_SHELL` markers for exec, ACP client, and TUI local shell paths; the requested `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` contract is absent from source, tests, and docs. The open #72980 overlaps only the exec session-key subset and does not supersede this broader runtime PR.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is a rebase onto current main that defines the approved env contract, threads explicit agent/session context through exec direct/sandbox/PTY and gateway approval follow-up execution, updates the current ACP helper in `src/acp/client-helpers.ts`, passes TUI agent/session state into `createLocalShellRunner`, documents the security semantics, and adds focused tests for explicit-agent precedence, session-key fallback, missing context, env override behavior, and approved gateway execution.
