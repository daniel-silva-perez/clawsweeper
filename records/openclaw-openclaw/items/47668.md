## Summary

Keep this issue open. Current main has substantial adjacent context controls, including auto-compaction, manual /compact, memory flush, configurable tool-result caps, and session pruning, but the reported user problem is not fully resolved. The linked open PR #70605 is a partial implementation candidate for the strikethrough-context subproblem, and current main still does not strip strikethrough content from injected Project Context files.

## Best Possible Solution

Keep this issue open. Review or land the linked strikethrough sanitizer PR if it is correct, then either keep this issue as the broader quality-of-life umbrella for long-session context pressure or split follow-ups for Ollama/local-model pruning defaults, duplicate/noisy tool-output handling, and smoother compaction/checkpoint UX.
