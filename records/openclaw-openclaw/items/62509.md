## Summary

Keep this issue open. Current main still has the installer finalization path calling unbounded `npm` prefix probes and an unbounded `openclaw daemon status --json` subprocess. The related #46849 cleanup helps the daemon status command path, but it does not add the installer-level process deadlines requested here.

## Best Possible Solution

Keep this issue open. Add a portable installer deadline helper for best-effort finalization probes, apply it to the npm prefix/config probes and `openclaw daemon status --json`, warn and continue when a probe times out, and add focused install-sh tests with stubbed sleeping `npm`/`openclaw` commands. The related #46849 daemon-status fast path remains useful but does not close this installer-specific hang.
