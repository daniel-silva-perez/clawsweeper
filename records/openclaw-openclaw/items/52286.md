## Summary

Keep this issue open. Current main has partial Telegram topic support for same-chat message-tool sends and normal reply media, but it does not solve the reported case where the agent explicitly targets the Telegram user ID: that path still looks like a direct chat and bypasses topic auto-thread injection.

## Best Possible Solution

Keep this as the canonical Telegram topic file-routing bug. Implement a deterministic current-conversation attachment path or message-tool guard so file/media replies in Telegram forum topics preserve chat_id + thread_id, while still allowing intentional cross-DM sends. Add regression coverage for omitted target, same-group target, and accidental sender-user-id target in a forum topic.
