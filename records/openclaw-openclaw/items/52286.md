## Summary

Current main still has the reported Telegram forum-topic routing gap. It preserves topic context when the message tool has no explicit target or targets the same Telegram chat, but an explicit positive numeric Telegram user ID is parsed as a direct chat and does not receive the originating group chat_id + thread_id.

## Best Possible Solution

Keep this issue open as the canonical Telegram forum-topic file-routing bug. Implement a deterministic current-conversation attachment path or a Telegram-specific message-tool guard so file/media replies in forum topics preserve chat_id + thread_id when the model targets the current sender by mistake, while retaining an explicit, auditable way to intentionally DM a user. Add regression coverage for omitted target, same-group target, same-topic target, accidental sender-user-id target, and intentional cross-DM target.
