## Best Possible Solution

Keep this issue open while maintainers review and land a generic bundled-plugin package-identity cache invalidation fix, with a regression test for same-path packaged upgrades; avoid a browser-specific workaround in core unless the generic loader/runtime-deps seam proves insufficient.
