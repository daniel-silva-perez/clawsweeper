## Summary

Keep this PR open. Current main still uses the schtasks/Startup-folder Windows daemon contract, so the PR is not obsolete, but the proposed patch has blocking regressions in the PowerShell task action, schtasks fallback, and uninstall cleanup paths.

## What This Changes

This PR changes Windows gateway Scheduled Task management in `src/daemon/schtasks.ts` to prefer PowerShell ScheduledTasks cmdlets over `schtasks.exe`, with schtasks kept as a fallback.

## Best Possible Solution

Keep the PowerShell ScheduledTasks direction only if the patch preserves a valid task action command, keeps the schtasks fallback valid, always removes Scheduled Task plus Startup-folder/script artifacts on uninstall, updates the Windows docs/tests/changelog for the chosen admin/fallback policy, and validates the daemon flow on Windows.

## Reproduction Assessment

Yes. The blocking findings can be reproduced with focused unit tests/mocks by asserting the generated PowerShell `New-ScheduledTaskAction` argument, simulating PowerShell uninstall success with existing fallback artifacts, and exercising the schtasks fallback create args.

## Solution Assessment

No. PowerShell ScheduledTasks may be the right direction for the 72-hour and battery settings problem, but this implementation is not the narrowest safe merge path until the command quoting, fallback flags, uninstall cleanup, docs, and tests are corrected.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Quote the task script for cmd.exe, not PowerShell:** `src/daemon/schtasks.ts:977`
  - body: The PowerShell action stores `/d /s /c '${safeScriptPath}'`; PowerShell converts the doubled quotes into literal single quotes, and `cmd.exe` does not use single quotes for command quoting. The primary PowerShell install/update path can register a task that cannot launch the generated script, so build this argument with the existing cmd quoting semantics instead.
  - confidence: 0.88
- **[P2] Remove the idle flag from the ONLOGON fallback:** `src/daemon/schtasks.ts:1027-1028`
  - body: The schtasks fallback appends `/I 0` while also using `/SC ONLOGON`. `/I` is only valid for `ONIDLE` schedules and valid values are 1-999, so machines that need the fallback can reject task creation instead of installing the gateway task.
  - confidence: 0.96
- **[P1] Run uninstall cleanup after PowerShell task removal:** `src/daemon/schtasks.ts:1105`
  - body: Returning immediately after `Unregister-ScheduledTask` skips the cleanup below that deletes the Startup-folder launcher and generated gateway script. If a previous fallback launcher exists, uninstall can leave auto-start behavior behind.
  - confidence: 0.97
