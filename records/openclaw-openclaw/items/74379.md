## Summary

Keep open. Current main has exec host routing, security/ask policy, allowlist, safeBins, strict inline-eval handling, and approval flows, but no config schema or runtime gate for `denyPathPatterns`/`denyPathAction`. The safeBins premise needs calibration because current safeBins are stdin-only and `cat` is not a default safe bin, but the requested defense-in-depth path denylist is still unimplemented and security-sensitive.

## What This Changes

Add a `tools.exec` config-level path-pattern denylist/action that rejects or escalates exec commands whose literal arguments target protected paths before execution.

## Best Possible Solution

Define a security-reviewed exec path policy in `tools.exec`, then enforce it in the shared gateway and node exec paths before any approval-backed or direct run. The implementation should update config types/schema/help/docs and add focused tests for literal argv path matching, tilde/home handling, allowlist/safeBins interaction, and explicitly documented v1 bypasses.
