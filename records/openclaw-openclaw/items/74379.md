## What This Changes

The request is to add an opt-in exec path denylist, with path-pattern matching before host command execution to reduce accidental secret exfiltration through exec arguments.

## Best Possible Solution

Use PR #75101 as the canonical implementation candidate: review the security semantics, land a tested config/schema/docs/runtime denylist if acceptable, or close that PR with a clear design decision and keep this issue as the tracking item for the chosen exec path policy.
