## Summary

Current main still has the reported behavior: the onboarding skills flow only suppresses the Homebrew prompt on `win32`, so FreeBSD remains eligible for the prompt when a selected skill has a brew installer and `brew` is missing. The issue should stay open because multiple open PRs already reference it with closing intent, so the right cleanup is to review or choose one of those fixes rather than close the report early.

## What This Changes

Stop `openclaw onboard` from recommending or showing Homebrew install instructions on FreeBSD and other unsupported platforms during skills setup.

## Best Possible Solution

Land one narrow onboarding fix that changes the Homebrew prompt gate to supported platforms only, adds a FreeBSD regression test that proves `detectBinary("brew")` is not consulted on unsupported platforms, and includes appropriate changelog/report credit; handle the WSL note as a separate product call if needed.
