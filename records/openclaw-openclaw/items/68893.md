## Summary

Keep open. Current main still gates the onboarding Homebrew recommendation only by excluding Windows, so FreeBSD remains eligible for the prompt when a selected skill has a brew installer and `brew` is missing; the report is also already paired with open closing PRs, so it should not be closed before one lands.

## What This Changes

Stop `openclaw onboard` from recommending Homebrew on FreeBSD during the skills dependency setup flow.

## Best Possible Solution

Land one narrow onboarding fix that allows the Homebrew prompt only on platforms Homebrew supports, adds a FreeBSD regression test proving `detectBinary("brew")` is not consulted on unsupported platforms, and credits the reporter and useful prior PR attempts in the changelog.

## Reproduction Assessment

Yes. A source-level reproduction is clear: on `freebsd`, selecting any brew-backed missing skill with no `brew` binary makes `process.platform !== "win32"` true, so the Homebrew note branch runs.

## Solution Assessment

Yes for the FreeBSD bug. A small platform allowlist around the Homebrew prompt plus focused regression coverage is the narrowest maintainable fix; WSL behavior should be handled separately if maintainers want different Linux-in-WSL messaging.
