## Summary

Keep open. Current main still does not provide the requested `openclaw skills uninstall <slug>` CLI surface. The underlying ClawHub skill lifecycle writes and reads `.clawhub/lock.json` for install/update, but there is no uninstall/removal helper or CLI command, and the user-facing docs still list search/install/update/list/info/check only. The original request #14264 was already closed specifically in favor of this issue, while the prior dashboard PR #17503 is closed/unmerged and targeted a different bundled-skill UI/RPC surface. Open PR #69319 is related but only targets a Gateway RPC and does not supersede the CLI request on current main.

## Best Possible Solution

Keep the issue open. The best path is to implement `openclaw skills uninstall <slug>` in `src/cli/skills-cli.ts`, backed by a safe ClawHub skill uninstall helper in `src/agents/skills-clawhub.ts` that removes the active workspace skill directory and the `.clawhub/lock.json` entry, with confirmation/dry-run/JSON behavior, docs in `docs/cli/skills.md` and `docs/tools/clawhub.md`, and focused tests in the existing skills CLI and ClawHub skill test files. If #69319 lands first, reuse its backend removal primitive where appropriate and keep this issue as the CLI-facing follow-through unless maintainers explicitly choose RPC-only.
