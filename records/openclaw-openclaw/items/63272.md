## Summary

Keep open. Current main still has ClawHub-backed skills search/install/update/list/info/check, but no `openclaw skills uninstall <slug>` CLI command and no backend helper that removes a workspace skill plus its `.clawhub/lock.json` entry. The related prior request was closed in favor of this issue, the older dashboard PR is closed/unmerged, and the open Gateway RPC PR does not supersede the requested CLI surface.

## Best Possible Solution

Keep the issue open and implement `openclaw skills uninstall <slug>` as the CLI-facing inverse of ClawHub skill install/update. The implementation should safely resolve the workspace skill path, remove the skill directory, remove the matching `.clawhub/lock.json` entry, support confirmation/dry-run/JSON behavior, update `docs/cli/skills.md` and `docs/tools/clawhub.md`, and add focused tests in the existing skills CLI and ClawHub backend test files. If #69319 lands first, reuse its backend primitive where it matches the CLI semantics.
