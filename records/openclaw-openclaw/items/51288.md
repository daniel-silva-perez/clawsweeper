## Summary

Keep this PR open. Current main still does not strip plugin-injected `<relevant-memories>` blocks from user-role Control UI extraction or Gateway `chat.history` projection, while the bundled `memory-lancedb` plugin still injects that exact `prependContext` shape. The PR is a relevant implementation candidate, but it overlaps with open PR #59697 and still needs consolidation/review for user-authored tag preservation and malformed-prefix handling.

## Best Possible Solution

Keep this PR open and consolidate it with #59697. The final fix should add a shared user-visible sanitizer for plugin-injected leading `<relevant-memories>` blocks in `ui/src/ui/chat/message-extract.ts`, `ui/src/ui/chat/message-normalizer.ts`, and `src/gateway/chat-sanitize.ts`; preserve literal user-authored tags outside the injected-prefix shape; fail open on malformed user text; keep assistant global stripping unchanged; and add regression tests for string and array content paths.
