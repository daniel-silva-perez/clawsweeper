## Summary

Keep this PR open. Current main still does not implement the PR's intended user-message `<relevant-memories>` sanitization in the Control UI or Gateway history projection paths, and the bundled `memory-lancedb` plugin still injects that block through `prependContext`. The PR is relevant, but it should not be closed as implemented; unresolved review feedback and broader overlap with open PR #59697 mean the best path is to consolidate and land a complete sanitizer fix.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, but consolidate it with #59697. The landed fix should add a shared user-visible sanitizer for plugin-injected leading `<relevant-memories>` context in `ui/src/ui/chat/message-extract.ts`, `ui/src/ui/chat/message-normalizer.ts`, and `src/gateway/chat-sanitize.ts`; preserve literal user-authored tags outside the injected-prefix shape; fail open on malformed user text; keep assistant global stripping unchanged; and add regression tests for string and array content in both Control UI and Gateway history paths.
