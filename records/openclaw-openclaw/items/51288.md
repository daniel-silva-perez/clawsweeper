## Summary

Keep this PR open. Current main still does not implement the intended user-message `<relevant-memories>` sanitization in the Control UI or Gateway history projection paths, while the bundled memory plugin still injects those blocks via `prependContext`. The PR is relevant, but it is not merge-ready as-is because it is stale against current test config/rendering paths and has unresolved sanitizer edge-case review feedback; the remaining work should be consolidated with the overlapping open PRs #53588 and #59697.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, but consolidate it with #53588 and #59697. The fix should add a user-visible, leading injected-context sanitizer in the shared Gateway/UI display paths (`chat-sanitize`, `message-normalizer`, and `message-extract` as needed), preserve literal user-authored `<relevant-memories>` prose outside the injected-prefix shape, fail open on malformed user text, and add regression tests for string and array content in both chat.history and Control UI rendering.
