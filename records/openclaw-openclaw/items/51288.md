## Summary

Keep this PR open. Current main still does not implement the intended user-message `<relevant-memories>` sanitization in the Control UI or Gateway history projection paths, while `memory-lancedb` still injects that block via `prependContext`. The PR remains relevant, but it is not merge-ready as-is because review feedback flags sanitizer edge cases and the remaining work overlaps the broader open PR #59697.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, but consolidate it with #59697. The fix should add a shared user-visible sanitizer for plugin-injected leading `<relevant-memories>` context in `ui/src/ui/chat/message-extract.ts`, `ui/src/ui/chat/message-normalizer.ts`, and `src/gateway/chat-sanitize.ts`; preserve literal user-authored tags outside the injected-prefix shape; fail open on malformed user text; keep assistant global stripping unchanged; and add regression tests for string and array content in the Control UI and Gateway history paths.
