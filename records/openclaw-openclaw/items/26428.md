## Summary

Keep open. Current main still installs and detects fish completions through `config.fish` profile sourcing, the setup/doctor/docs surfaces still describe profile-based installation, and the directly related implementation PR #27135 was closed unmerged rather than shipped.

## What This Changes

Install fish shell completions into fish's autoload completions directory instead of sourcing the cached completion script from `config.fish`.

## Best Possible Solution

Add a fish-specific completion install contract that writes or copies the generated cached script to `~/.config/fish/completions/<binName>.fish`, treats that autoload file as installed, updates setup/doctor/docs messaging, and narrowly cleans up OpenClaw-owned legacy `config.fish` entries.
