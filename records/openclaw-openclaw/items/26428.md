## Summary

Keep open. Current main still installs and detects fish completions through `config.fish`, and the docs, doctor, and setup hints still describe profile sourcing. The directly related fix PR #27135 was closed unmerged, so the requested fish autoload behavior remains valid and narrow.

## What This Changes

Request installing fish completions as `~/.config/fish/completions/openclaw.fish` instead of sourcing the cached fish script from `~/.config/fish/config.fish`.

## Best Possible Solution

Add a fish-specific completion install contract: generate or reuse the cached fish script, copy it to `${XDG_CONFIG_HOME:-$HOME/.config}/fish/completions/<binName>.fish`, detect that file as the installed state, and remove only OpenClaw-owned legacy `config.fish` completion entries during upgrade without disturbing user fish config.
