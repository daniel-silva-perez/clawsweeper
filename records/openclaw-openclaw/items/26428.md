## Summary

Current main still installs fish completions by sourcing the cached script from `~/.config/fish/config.fish`, detects installation from that profile, and documents/profile-messages shell-profile installation. The requested fish autoload path under `~/.config/fish/completions/<binName>.fish` is still missing, and the directly related implementation PR #27135 is closed unmerged.

## Best Possible Solution

Keep this issue open and implement a focused fish-specific completion install/detection path in `src/cli/completion-runtime.ts`: copy the cached fish script to `${XDG_CONFIG_HOME:-$HOME/.config}/fish/completions/<binName>.fish`, detect that file as installed, safely remove legacy OpenClaw-owned `config.fish` completion blocks/source lines, and update doctor/setup messaging, docs, changelog, and regression tests. PR #27135 is useful prior art, but it needs to be rebased and reviewed against current main.
