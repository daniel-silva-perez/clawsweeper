## Summary

Keep open. Current main still treats fish like a profile-sourced shell completion: install and detection go through `config.fish`, docs describe `--install` as adding a shell-profile source block, and the directly related PR that proposed fish autoload installation was closed unmerged.

## What This Changes

Request fish shell completion installation to place `openclaw.fish` under the fish autoload completions directory instead of sourcing it from `config.fish`.

## Best Possible Solution

Implement a fish-specific install path that writes the generated cached script to the fish autoload completions directory, updates fish detection to treat that file as installed, and migrates only OpenClaw-owned legacy `config.fish` completion blocks or source lines without disturbing user-authored fish config.
