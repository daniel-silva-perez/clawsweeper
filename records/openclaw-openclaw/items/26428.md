## Summary

Current main still installs and detects fish completions through `~/.config/fish/config.fish` profile sourcing, while fish's documented autoload convention is `~/.config/fish/completions/<command>.fish`. The directly related implementation PR #27135 is closed unmerged, so the requested behavior remains valid and narrow enough for a focused fix PR.

## Best Possible Solution

Keep this issue open and implement a focused fish-specific completion install/detection path in `src/cli/completion-runtime.ts`: copy the cached fish script to `${XDG_CONFIG_HOME:-$HOME/.config}/fish/completions/<binName>.fish`, detect that file as installed, clean up only legacy OpenClaw-owned `config.fish` entries, and update doctor/setup messaging, CLI docs/help text, changelog, and regression tests. PR #27135 is useful prior art but is closed unmerged, so a fresh/rebased fix is the right path.
