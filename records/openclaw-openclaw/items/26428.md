## Summary

Keep open. Current main still installs fish completions through `~/.config/fish/config.fish`, detects installation from that profile, and documents/profile-messages the profile-based path. Fish's own docs point user completions at `~/.config/fish/completions`, and the directly related PR #27135 is closed unmerged, so the requested behavior is still missing.

## Best Possible Solution

Implement a fish-specific completion install and detection path in `src/cli/completion-runtime.ts`: install or copy the cached fish script to `~/.config/fish/completions/<binName>.fish`, detect that file as installed, safely remove legacy OpenClaw-owned `config.fish` blocks, and update doctor/setup messaging, docs, changelog, and focused regression tests. Review PR #27135 as prior art but rebase the behavior onto current main.
