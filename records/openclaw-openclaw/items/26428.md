Keep open. Current main still installs, detects, and documents fish completions through `config.fish`, so the requested fish autoload installation remains unresolved. The directly related implementation PR was closed unmerged, and the issue is narrow enough for a focused repair PR.

## What This Changes

Install fish completions into `~/.config/fish/completions` instead of sourcing the generated completion script from `~/.config/fish/config.fish`.

## Best Possible Solution

Fish completion install and detection should follow fish's autoload contract by writing the generated fish completion file to `~/.config/fish/completions/<binName>.fish`, treating that file as the installed state, narrowly cleaning only OpenClaw-owned legacy `config.fish` completion entries, and aligning setup, doctor, update behavior, docs, tests, and changelog wording.
