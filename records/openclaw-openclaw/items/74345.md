## Summary

## What This Changes

The issue reports a 2026.4.27 regression where task-registry ACP cleanup saturates the gateway event loop, leaks ACP sessions/processes, and stalls Telegram and Discord at idle.

## Best Possible Solution

Make ACP task-maintenance cleanup bounded and idempotent: avoid reattempting the same failing orphan close every sweep without backoff, preserve live persistent bindings, safely clear or quarantine stale metadata when the backend is unavailable, and add regression coverage for repeated close failures plus large session-store scans.

Current-main reproduction is not yet high-confidence because this read-only review did not run a live gateway idle test, but current source still contains the plausible maintenance path.
