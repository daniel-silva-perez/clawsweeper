## Summary

Keep this PR open. Current main still lacks the proposed runtime catch re-arm and independent watchdog, and the PR is paired with the same-author open bug report #73166 via closing syntax. The patch is plausible, but active review discussion still flags a double-arm edge case and missing direct coverage for the `.catch()` re-arm path.

## Best Possible Solution

Keep the PR open and review it together with #73166. The best path is either to revise this PR to avoid the double-arm edge case and add direct catch-path coverage, or to land an equivalent narrowly tested runtime watchdog/recovery implementation in `src/cron/service/timer.ts` with lifecycle wiring in `ops.ts` and `state.ts`.
