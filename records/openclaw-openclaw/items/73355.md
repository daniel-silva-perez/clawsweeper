## Summary

Keep this PR open. Current main does not implement the runtime timer-chain recovery or independent cron watchdog proposed here, and the PR is paired with an open same-author issue that it closes. The linked #68112 covers a different startup catch-up failure path, not this runtime watchdog path.

## Best Possible Solution

Keep the PR open as the active implementation candidate for the linked runtime cron-stall issue. Maintainers should review whether a simple catch-handler re-arm is sufficient or whether the independent watchdog is justified, strengthen the regression tests to exercise the actual failure path, and then land or revise the PR with targeted cron timer validation.
