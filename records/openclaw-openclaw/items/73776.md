## Summary

Keep this PR open. Current main still has the explicit `openclaw crestodian` no-TTY path logging the error and returning normally, while the CLI wrapper only exits non-zero when the action throws. The PR is a focused implementation candidate for open bug #73646, not obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or replace it with an equivalent fix that makes the explicit `openclaw crestodian` no-TTY path exit non-zero and adds regression coverage in `src/crestodian/crestodian.test.ts`.
