## Summary

Keep this PR open. Current main still has the behavior the PR intends to change: expired or unstamped `lost` task audit findings are errors, and `startedAt < createdAt` has no audit-level tolerance. The existing registry timestamp normalization covers persisted registry paths, but it does not make this PR obsolete because the audit helper and current tests still encode the stricter behavior directly.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide whether terminal `lost` audit findings should always be warnings, update the docs if that policy changes, keep or refine the audit-level skew regression coverage if tolerance is desired beyond registry normalization, and remove the redundant assertion before merge if maintainers agree.
