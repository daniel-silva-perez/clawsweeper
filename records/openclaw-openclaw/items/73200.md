## Summary

Keep this PR open for maintainer review. Current main still has the old task-audit behavior, and the PR is not obsolete; however, it changes the documented `lost` severity contract without updating the public task docs, so it needs policy confirmation and a small docs cleanup before merge.

## What This Changes

This PR changes task audit so `lost` task findings always warn, adds a 1-second tolerance for `startedAt < createdAt` audit skew, and updates the colocated audit tests.

## Best Possible Solution

Keep the PR open and decide the intended task-audit severity contract. If all terminal `lost` findings should be warnings, update the automation and CLI docs alongside the code and keep the new regression tests; otherwise preserve the current warn/error policy and close or revise the PR.
