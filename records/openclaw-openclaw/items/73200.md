## Summary

Keep this PR open. Current main does not already implement the PR's two intended audit behavior changes: expired/non-retained `lost` task findings still sort and count as `error`, and `startedAt < createdAt` still has no race tolerance in the audit helper. The existing registry timestamp normalization reduces the real-world skew path for persisted records, but it does not make this PR obsolete because the audit function and its current tests still encode the stricter behavior directly.

## Best Possible Solution

Do not close this PR as obsolete. Review it as a focused behavior change: decide whether terminal `lost` audit findings should always be warnings, update the docs if that policy changes, keep the audit-level skew regression coverage if maintainers want tolerance beyond registry normalization, and clean up the redundant assertion before merge if requested.
