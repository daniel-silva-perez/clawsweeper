## Summary

Keep open. Current main still lacks reserved internal namespace rejection for external hook session keys, so the PR addresses a real gateway security boundary. The branch is currently dirty and rebaseable=false, and the fix must be adapted without regressing current main's newer static-versus-templated mapping session-key policy.

## What This Changes

The PR adds reserved-session namespace validation for hook default keys, direct hook requests, mapped hook session keys, and related gateway regression tests.

## Best Possible Solution

Rebase or adapt the reserved-namespace guard onto current main with shared validation in hook session-key resolution plus config/static-mapping load paths, while keeping the existing template-rendered mapping safeguards and adding focused regressions for nested agent-scoped keys and whitespace after unwrap boundaries.

## Reproduction Assessment

Yes. ## Solution Assessment

Yes, conceptually. A shared hook session-key reserved-prefix helper plus config-load checks for default and static mapping keys is the narrow maintainable fix; the safer final version is the same approach rebased onto current main without losing newer mapping policy behavior.
