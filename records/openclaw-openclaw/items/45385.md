## Summary

Keep open. Current main still has the user-facing formatter gap: Anthropic `tool_use`/`tool_result` mismatch payloads are not recognized before the generic `invalid_request_error` fallback, so the raw provider message can still be surfaced. The repair path is stronger now, but this issue is paired with open PR #45393 using closing syntax, so it should remain open until that PR is reviewed, landed, or replaced.
