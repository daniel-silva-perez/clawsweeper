## Summary

Keep open. Current main still treats transport timeouts as non-persistent auth-health signals, so timed-out auth profiles are not put into cooldown. Main has adjacent mitigations that rotate profiles and can fall back after rotation is exhausted, but it does not implement the requested consecutive-timeout cooldown, primary-provider failover budget, retry cap, or per-agent run timeout override.

## Best Possible Solution

Keep this open for a focused failover policy change. The safest direction is a model-scoped, short-lived timeout circuit breaker or bounded primary-provider failover budget that counts consecutive timeouts without treating isolated transport stalls as permanent auth failure. Add tests for multi-profile timeout sequences reaching configured fallback quickly, single timeout recovery, external aborts, and model-scoped cooldown behavior.
