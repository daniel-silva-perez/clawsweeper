## Summary

Keep open. This PR targets a real Matrix E2EE bug that still exists on current main and is tracked by #74504, but the latest head still has merge blockers: the MAS metadata probe uses raw global fetch in Matrix runtime code, the changelog entry lacks required attribution, and the patch leaves an unused helper behind.

## What This Changes

The PR adds Matrix MAS homeserver capability detection, rewrites cross-signing UIA handling to honor advertised stages and sessions with typed reset errors, expands crypto-bootstrap tests, and adds a Matrix changelog entry.

## Best Possible Solution

Revise the Matrix plugin fix so the MAS auth metadata probe goes through the existing guarded Matrix transport/http-client path while preserving the session-aware UIA parsing and typed MAS reset error behavior; add required changelog attribution, remove or use the dead mismatch helper, then validate the Matrix targeted tests and raw-fetch guard before merge.
