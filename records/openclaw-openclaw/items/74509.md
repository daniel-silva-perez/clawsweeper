## Summary

Keep this PR open. The latest head addresses the earlier guarded-transport and changelog-attribution concerns, but the patch still has a merge-blocking correctness regression in the password UIA path: it trims the configured Matrix password before sending it to the homeserver.

## What This Changes

The PR updates Matrix cross-signing bootstrap to parse UIA 401 stages, detect MAS auth metadata through the guarded Matrix transport, surface typed MAS reset errors with reset URLs, add regression tests, and add a Matrix changelog entry.

## Best Possible Solution

Keep the PR direction, but request changes to preserve the raw configured password while using a trimmed value only for the presence check, add a regression test for a whitespace-containing Matrix password, update the Matrix verification docs for MAS reset recovery, then run the focused Matrix bootstrap tests and changed gate.

## Reproduction Assessment

Yes. The central reported behavior is reproduced from current-main code and the companion report: a MAS UIA 401 advertising org.matrix.cross_signing_reset falls through to the old password fallback. The introduced PR bug is reproducible with a unit test where channels.matrix.password contains leading or trailing whitespace and the server advertises m.login.password.

## Solution Assessment

No, not as currently patched. The stage-aware UIA handling and guarded auth_metadata probe are the right shape, but trimming the password before submitting m.login.password is not a safe implementation detail for opaque credentials.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve the configured password when submitting UIA:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:227`
  - body: The patch uses `params.password?.trim()` as the value sent in the `m.login.password` auth dict. Current main sends the configured password unchanged, and Matrix passwords are opaque credentials; accounts whose password intentionally starts or ends with whitespace will now fail UIA. Keep the trimmed value for the presence check, but submit the original `params.password`.
  - confidence: 0.88
