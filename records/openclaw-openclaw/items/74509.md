## Summary

Keep open. The PR addresses a real Matrix E2EE/MAS bootstrap bug tracked by #74504, but the current patch introduces blocking regressions: the new MAS metadata probe bypasses the existing guarded Matrix transport, and the new typed reset error escapes the non-strict startup bootstrap path that currently keeps Matrix startup non-fatal.

## What This Changes

The PR updates the Matrix channel cross-signing bootstrap to detect MAS, parse UIA challenge stages, surface MAS reset URLs, preserve MSC3967 first-publish uploads, and add Matrix bootstrap tests plus a changelog entry.

## Best Possible Solution

Revise this PR so the MAS capability probe uses the existing Matrix guarded HTTP path with timeout, DNS pinning, SSRF policy, and dispatcher/proxy support; keep explicit or strict verification bootstrap surfacing MatrixCrossSigningResetRequiredError, but preserve non-fatal startup behavior for the initial conservative bootstrap; add regression tests for both paths and add the required changelog attribution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Route MAS probing through the guarded transport:** `extensions/matrix/src/matrix/sdk.ts:891`
  - body: This new bare `fetch()` bypasses the Matrix transport that applies `ssrfPolicy`, `dispatcherPolicy`, DNS pinning, redirects, and timeout handling. Deployments that rely on Matrix proxy/private-network controls would make this auth metadata request outside the validated request path, so the probe should go through `MatrixAuthedHttpClient`, `doRequest`, or the same guarded fetch machinery used by the SDK client.
  - confidence: 0.92
- **[P2] Preserve non-fatal startup bootstrap failures:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:403-404`
  - body: `bootstrapCryptoIfNeeded()` calls the initial bootstrap with `allowAutomaticCrossSigningReset: false` and without strict mode so Matrix startup can continue when crypto repair is incomplete. This unconditional rethrow happens before that branch can return `{ ready: false, published: false }`, so a MAS reset UIA challenge during the conservative startup pass can abort Matrix startup instead of only surfacing from explicit/strict bootstrap flows.
  - confidence: 0.88
