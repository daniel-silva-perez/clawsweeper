## Summary

Keep open. Current main does not contain the linked PR’s `--yolo` or `--approval-mode` fix; the Gemini CLI backend still passes only `--skip-trust` for noninteractive JSON prompt runs, and the current regression test pins that shape. Upstream Gemini CLI documentation says write tools require approval by default and noninteractive `ask_user` approvals are denied, so the reported file-write failure is still plausible on current defaults. The separate OAuth account-risk concern has been addressed in current Google provider docs and setup prompts.

## What This Changes

Configure the Google Gemini CLI backend so noninteractive OAuth-backed agent turns can write or edit workspace files such as `IDENTITY.md` and `SOUL.md`.

## Best Possible Solution

Decide the intended Gemini CLI headless approval posture, then update the Google plugin backend defaults, resume defaults, docs, and regression coverage together so file edits work in noninteractive OpenClaw sessions while the OAuth caution remains visible.
