## Summary

Keep this PR open. Current main has made the direct overflow-clipping part mostly stale because `.agent-chat__input` no longer sets `overflow: hidden`, but the PR's second user-visible behavior, keeping the active slash-menu item visible during ArrowUp/ArrowDown navigation, is still missing.

## Best Possible Solution

Keep the PR open and ask the contributor to rebase on current main, then narrow the patch to the still-missing active-item auto-scroll behavior. Only keep the wrapper/layout changes if current clipping is still reproducible after rebase; otherwise avoid extra layout churn and add focused UI coverage or explicit manual proof.
