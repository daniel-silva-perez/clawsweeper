## Summary

Keep this PR open. Current main has made the overflow-clipping wrapper part mostly obsolete because `.agent-chat__input` no longer has `overflow: hidden`, but current main still lacks the PR’s second user-visible fix: keyboard navigation does not scroll the active slash-menu item into view.

## Best Possible Solution

Keep this PR open and ask the contributor to rebase on current main, narrow the patch to the still-missing slash-menu active-item auto-scroll behavior, and retain wrapper/layout changes only if clipping still reproduces after rebase. The implementation should scope DOM lookup to the current chat instance and include focused UI coverage or explicit manual proof.
