## Summary

Keep open. Current main covers the normal native `sessions_spawn` default-timeout path, but it does not implement this PR’s intended registry/control/reactivation change: several current `main` paths still coerce omitted `runTimeoutSeconds` to `0`, and one test explicitly expects that reactivation behavior. The PR is stale against the refactored files, but the underlying change is not fully implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review or an updated patch. The best path is to either prove the remaining `?? 0` registry/control/reactivation paths are intentionally unreachable with omitted timeouts, or port the fix to the current run-manager/control/reactivation files with regression coverage for omitted timeout propagation and explicit `0` semantics.
