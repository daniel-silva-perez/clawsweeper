## Summary

Keep this PR open. Current main still has the text/media-only `handleMessageEnd` gate, so the PR’s intended tool-call-only assistant update behavior is not implemented or superseded. The work is a plausible focused bug fix, but it still needs maintainer review on empty-delta consumers and focused regression coverage before landing.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to either land a scoped fix in `src/agents/pi-embedded-subscribe.handlers.messages.ts` with regression coverage for tool-call-only assistant messages, or ask the author to broaden the patch if maintainers want HTTP/OpenResponses/ACP consumers to receive explicit non-empty progress or tool-call metadata too.
