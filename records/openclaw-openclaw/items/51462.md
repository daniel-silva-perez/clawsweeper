## Summary

Keep this PR open. Current main still lacks the PR’s intended tool-call-only assistant update path, and the prior review discussion remains relevant: the focused subscription-handler fix is plausible, but maintainers still need regression coverage and a product decision for empty assistant events on relay consumers.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or adapt a scoped `handleMessageEnd` fix with regression coverage for tool-call-only assistant messages, including empty/missing ID cases, then explicitly decide whether relay consumers should receive structured tool-call metadata or a non-empty progress marker for tool-only turns.
