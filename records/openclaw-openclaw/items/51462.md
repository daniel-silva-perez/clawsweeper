## Summary

Keep this PR open. Current main still does not emit a final assistant update for structured tool-call-only assistant messages, so the PR’s intended fix is not obsolete. The focused change remains plausible, but maintainers should land or adapt it with regression coverage and an explicit decision for empty-text relay consumers.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or adapt a scoped `handleMessageEnd` fix that detects structured tool-call blocks by presence, adds regression tests for tool-call-only assistant messages with blank or missing IDs, and separately decides whether relay consumers should receive structured tool-call metadata or a non-empty progress marker.
