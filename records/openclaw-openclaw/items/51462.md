## Summary

Keep this PR open. Current main still does not implement the PR’s intended behavior: `handleMessageEnd` only emits the final assistant update when there is cleaned text or media, so a tool-call-only assistant message with no text/media is still skipped. The PR remains a plausible focused fix, but it needs maintainer review, regression coverage, and a decision on whether empty assistant events should also carry visible progress or tool-call metadata for relay consumers.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a scoped `handleMessageEnd` fix with regression coverage for tool-call-only assistant messages, then explicitly decide whether relay consumers should receive a non-empty progress marker or structured tool-call metadata for tool-only turns.
