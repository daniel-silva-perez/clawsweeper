## Summary

Keep open. Current main has adjacent Discord draft-preview edits, status reactions, channel busy snapshots, and a stronger task ledger with `lost`/CLI cancellation behavior, but it still does not provide the requested single persistent editable status surface for normal long-running channel turns with explicit `done`, `failed`, `unknown`, or `interrupted` semantics. The stale label should not drive closure because the issue is recent, has active follow-up context, and related #59719/#71630 are adjacent rather than superseding.

## What This Changes

Request a Discord-first persistent task-status surface for long-running channel turns that edits one status message in place, uses explicit terminal/unknown states, and treats send/edit failures as best-effort only.

## Best Possible Solution

Define an additive runtime/channel status transport that can create, update, finalize, and best-effort clear one Discord status message for a normal long-running turn, backed by enough ownership state to fail closed to `unknown` or `interrupted` after restart, disconnect, or uncertain continuity.
