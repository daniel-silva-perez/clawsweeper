## Summary

Keep open. Current main has durable background task tracking, channel busy/active-run snapshots, Discord preview/tool-progress edits, and status reactions, but it still does not provide the requested persistent per-run channel status message with explicit terminal/unknown semantics for normal long-running Discord/chat turns. The related open PR #59719 is a useful partial CLI/background liveness slice, not a replacement for this feature request.

## Best Possible Solution

Keep this issue open as the canonical product/design request for a durable, best-effort channel-turn status surface. Review #59719 as a partial CLI/background liveness slice, then design an additive runtime/channel status transport that can create, update, finalize, and best-effort clear one Discord status message for normal long-running turns, backed by enough ownership state to degrade to unknown/lost after restart or uncertain continuity.
