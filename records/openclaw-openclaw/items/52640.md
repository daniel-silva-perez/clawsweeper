## Summary

Keep open. Current main has adjacent task-ledger reconciliation, Discord draft-preview edits, lifecycle reactions, and internal channel busy snapshots, but it still does not provide the requested single persistent editable status surface for normal long-running channel turns with explicit done, failed, unknown, or interrupted semantics.

## What This Changes

The issue requests a durable, editable per-run status surface for long-running channel turns, initially Discord, with best-effort delivery and explicit active, terminal, and unknown/interrupted states.

## Best Possible Solution

Define an additive runtime/channel status transport that can create, update, finalize, and best-effort clear one Discord status message for a normal long-running turn, backed by enough ownership state to fail closed to unknown or interrupted after restart, disconnect, cancellation, or uncertain continuity.
