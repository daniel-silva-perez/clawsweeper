## Summary

Keep open. Current main has adjacent busy-state accounting, Discord draft preview edits, reactions, task-ledger terminal/lost states, and restart recovery, but I did not find the requested single persistent editable channel-turn status object with explicit done, failed, unknown, or interrupted semantics.

## What This Changes

Add a persistent, best-effort, one-message status surface for long-running channel turns, with Discord-first editing and explicit terminal/unknown states.

## Best Possible Solution

Design an additive channel status transport that can create, update, finalize, and best-effort clear one Discord status message per normal long-running turn, backed by enough run ownership state to fail closed to unknown or interrupted after restart, disconnect, cancellation, or uncertain continuity.

## Reproduction Assessment

Not applicable as a feature request. The gap is reproducible by inspection: current main exposes draft preview, reactions, task records, and internal busy patches, but no persisted editable channel-turn status object.

## Solution Assessment

Unclear until maintainers choose the contract. The requested Discord-first editable status message is plausible, but the narrowest maintainable fix should be an additive channel/runtime transport rather than overloading task records or draft reply preview.
