## Summary

Keep open. Current main has adjacent busy snapshots, Discord preview edits/reactions, and durable background-task terminal/lost semantics, but normal long-running Discord/chat turns still do not have one authoritative persistent editable status surface with explicit done/failed/unknown or interrupted states. The related open PR #59719 is useful background exec/CLI liveness work, but it is scoped differently and does not replace this channel-turn product request.

## What This Changes

Add a first-class persistent task-status surface for long-running channel turns, starting with Discord, that updates one editable status message and ends in explicit terminal or unknown/interrupted state without blocking normal reply delivery.

## Best Possible Solution

Design an additive runtime/channel status transport that can create, update, finalize, and best-effort clear one Discord status message for normal long-running turns, backed by enough ownership state to degrade to unknown/interrupted after restart, disconnect, or uncertain continuity. Keep the existing task ledger and preview streaming as adjacent infrastructure rather than treating either as the final user-facing channel-turn status contract.
