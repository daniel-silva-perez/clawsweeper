## Summary

Keep open. Current main has adjacent task-ledger status, Discord draft preview edits, status reactions, typing, and internal channel busy snapshots, but it still lacks the requested single persistent editable status surface for normal long-running channel turns with explicit done, failed, unknown, or interrupted semantics. The related open PR is scoped to CLI/background exec liveness, not this Discord-first channel-turn UX.

## What This Changes

Requested a persistent, editable, session-scoped task-status surface for long-running channel turns, Discord first, with explicit terminal and unknown/interrupted semantics.

## Best Possible Solution

Design an additive runtime/channel status transport that can create, update, finalize, and best-effort clear one Discord status message for a normal long-running turn, backed by enough ownership state to fail closed to unknown or interrupted after restart, disconnect, cancellation, or uncertain continuity.
