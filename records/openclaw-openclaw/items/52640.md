## Summary

Keep open. Current main has adjacent task-ledger terminal/lost states, Discord draft preview edits, status reactions, channel busy snapshots, and restart recovery, but it still lacks the requested single authoritative editable status surface for normal long-running channel turns with explicit done, failed, unknown, or interrupted semantics. The related open PR tracks CLI/background exec liveness rather than this channel-turn UX, so the remaining work needs maintainer product/API design.

## What This Changes

Add a persistent, session/thread-scoped status surface for long-running channel turns, Discord first, with lifecycle states and best-effort status send/edit failures.

## Best Possible Solution

Design an additive runtime/channel status transport that can create, update, finalize, and best-effort clear one Discord status message for a normal long-running turn, backed by enough ownership state to fail closed to unknown or interrupted after restart, disconnect, cancellation, or uncertain continuity.
