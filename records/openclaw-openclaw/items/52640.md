## Summary

Keep open. Current main has adjacent task-ledger terminal/lost states, Discord draft preview edits, reactions, and internal channel busy snapshots, but not the requested single persistent editable status surface for normal long-running channel turns with explicit done, failed, unknown, or interrupted semantics.

## What This Changes

Add an optional persistent per-run status surface for long-running chat/channel turns, Discord first, with explicit lifecycle states and best-effort send/edit handling.

## Best Possible Solution

Design an additive runtime/channel status transport that can create, update, finalize, and best-effort clear one Discord status message for a normal long-running turn, backed by enough ownership state to fail closed to unknown or interrupted after restart, disconnect, or uncertain continuity.
