## Summary

Keep open. Current main has adjacent task tracking, Discord preview edits, status reactions, and channel busy snapshots, but it still does not provide one persistent per-run channel status message with terminal and unknown/interrupted semantics for normal long-running Discord/chat turns. The related open PR #59719 is a partial background exec/CLI liveness slice, not a replacement for this channel-turn feature request.

## What This Changes

Request a persistent, best-effort per-run channel status surface for long-running chat/channel turns, with explicit terminal and unknown/interrupted states.

## Best Possible Solution

Keep this issue open as the canonical product/design request for a durable channel-turn status surface. Review #59719 as a partial background exec/CLI liveness fix, then design an additive runtime/channel status transport that can create, update, finalize, and best-effort clear one Discord status message for normal long-running turns, backed by enough ownership state to degrade to unknown/interrupted after restart, disconnect, or uncertain continuity.
