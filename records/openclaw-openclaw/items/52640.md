## Summary

Keep open. Current main has adjacent task-ledger lost/cancelled states, Discord draft-preview edits, lifecycle status reactions, and internal channel busy snapshots, but it still lacks the requested single persistent editable status surface for normal long-running channel turns with explicit done, failed, unknown, or interrupted semantics. The stale label should not drive closure because the issue is recent and #59719/#71630 are adjacent open work, not superseding fixes.

## What This Changes

Request a Discord-first persistent, editable per-run status message for long-running channel turns with explicit lifecycle and terminal/unknown states, delivered best-effort so status transport failures cannot block normal completion.

## Best Possible Solution

Define an additive runtime/channel status transport that can create, update, finalize, and best-effort clear one Discord status message for a normal long-running turn, backed by enough ownership state to fail closed to unknown or interrupted after restart, disconnect, or uncertain continuity.
