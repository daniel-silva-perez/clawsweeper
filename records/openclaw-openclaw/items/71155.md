## Summary

Keep this PR open. Current main still drops stability-bundle error message, stack, and cause fields, and the discussion identifies this PR as the preferred base for #71071 over closed duplicate #71352, but the live head still needs correctness and diagnostics-privacy fixes before it is mergeable.

## What This Changes

This PR extends the diagnostic stability bundle error metadata to include message, stack, and cause, adds a sanitizer, and updates stability bundle tests to expect persisted error messages.

## Best Possible Solution

Revise this PR so generated failure bundles capture useful message, stack, and single-level cause details with explicit redaction, truncation, and NUL stripping, while imported bundles and support exports continue to enforce the documented diagnostics privacy model.

## Reproduction Assessment

Yes. Current main has a focused test path for startup-failure bundle writing that still expects raw error messages to be absent, and the source shows the write path only serializes name/code.

## Solution Assessment

No. This PR is the right base, but the current implementation should not merge until it separates write-time capture from import/export sanitization, preserves structured cause metadata, and implements the stated NUL/stack/message safety contract with tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Redact imported bundle messages before exporting:** `src/logging/diagnostic-stability-bundle.ts:144-155`
  - body: `readSafeErrorMetadata` is also used by `parseDiagnosticStabilityBundle`, which feeds `openclaw gateway stability --bundle ... --export`. Preserving `message`, `stack`, and `cause` here lets arbitrary imported error text flow into support zips, undoing the existing privacy tests and docs that say shareable diagnostics omit message/error payloads. Split write-time capture from import/export sanitization or redact these fields before export.
  - confidence: 0.88
- **[P2] Preserve cause as bounded metadata:** `src/logging/diagnostic-stability-bundle.ts:152-155`
  - body: `String(cause)` collapses an `Error.cause` to text and drops useful safe fields such as the cause code/name/message. The PR contract calls for a single-level cause object without recursion, so bundles still lose the actionable cause details this patch is meant to preserve.
  - confidence: 0.82
- **[P2] Strip NUL bytes from bundle text:** `src/logging/diagnostic-stability-bundle.ts:133-136`
  - body: The sanitizer removes ANSI escapes and truncates, but it never removes embedded NUL bytes even though the PR safety contract and review thread require NUL stripping for error message, stack, and cause text. Add an escaped `\x00` replacement and coverage so diagnostics consumers do not receive `\u0000` text.
  - confidence: 0.76
