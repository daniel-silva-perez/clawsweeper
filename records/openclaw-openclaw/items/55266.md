## Summary

Keep open. The reported proxied OpenAI Responses `compat.supportsStore: true` bug still exists on current main, but this PR is not correct as-is because main moved the store decision into the centralized payload policy and the branch is dirty against main.

## What This Changes

The PR changes the embedded OpenAI stream wrapper to force `store: true` for proxied OpenAI Responses models with `compat.supportsStore: true`, decouples default server compaction from that opt-in, and adds helper-level unit tests.

## Best Possible Solution

Implement the proxy `compat.supportsStore: true` opt-in in the centralized OpenAI Responses payload capability/policy layer, keep the provider allowlist and `supportsStore: false` stripping semantics, avoid enabling default server compaction for proxied store opt-ins, and update focused tests plus user-facing docs/changelog.
