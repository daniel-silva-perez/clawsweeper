## Summary

Keep this PR open. Current main still does not honor the PR's intended proxy opt-in: the centralized OpenAI Responses payload policy only forces `store: true` for known native OpenAI endpoints, so `compat.supportsStore: true` does not override a proxy-like `baseUrl`. The PR is useful source work, but it is now dirty against main and needs to be moved from the old helper-level implementation into the current central policy.

## What This Changes

The PR adds an explicit `compat.supportsStore: true` store-forcing path for proxied OpenAI Responses requests, decouples server compaction defaults from that opt-in, and adds unit coverage for the store decision helper.

## Best Possible Solution

Implement the opt-in in the centralized OpenAI Responses payload capability/policy layer, preserving the provider allowlist and `supportsStore: false` stripping, keeping proxy store opt-in separate from server compaction defaults, and covering the proxy opt-in with focused policy and embedded-runner payload tests.
