## Summary

Keep this PR open. Current main still does not implement the PR's intended proxy opt-in: the central OpenAI Responses payload policy only forces `store: true` for known native OpenAI endpoints, while proxy-like `baseUrl` routes still fall through without honoring `compat.supportsStore: true`. The PR remains a plausible focused fix, but it would need to be adapted to the current centralized policy shape rather than the older helper-only implementation.

## Best Possible Solution

Keep the PR open and ask for a rebase/update against current main's centralized OpenAI Responses payload policy. The best fix is to make `compat.supportsStore === true` explicitly allow `store: true` for guarded OpenAI Responses proxy routes, keep `supportsStore: false` stripping intact, keep server compaction decoupled from proxy store opt-in, and add regression coverage in the central policy or embedded-runner payload tests.
