## Summary

Keep open. Current main still lacks cron removal artifact cleanup, so this PR is useful for #46369, but the branch is not merge-ready because it can still recreate ghost sessions/run logs when a job is removed during an active run and it regresses the current cron_changed removed-event payload contract.

## What This Changes

The PR adds asynchronous cleanup of cron session-store entries and per-job run-log files after `cron.remove`, adds cleanup tests, and includes small unrelated test-helper removal and gateway formatting changes.

## Best Possible Solution

Update or replace the PR on current main so cron removal cleans the correct per-agent canonical and run session keys plus the run log, preserves the current removed-event job snapshot contract, and performs a second cleanup after any active run and pending log append settle. Drop unrelated model/gateway formatting churn unless it is still required by the rebase.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Re-clean artifacts after active cron runs finish:** `src/cron/service/ops.ts:348`
  - body: `cron.remove` can be called while an isolated job is still running, but this patch only cleans once immediately after deleting the job. The current run path persists the cron session again near completion and the gateway appends the finished event to the job log asynchronously, so a run that finishes after this call can recreate both the visible session entry and the `.jsonl` file the PR just removed. Please coordinate cleanup with active-run completion or schedule a post-finish cleanup/drain for removed jobs.
  - confidence: 0.88
- **[P2] Keep the removed job snapshot in events:** `src/cron/service/ops.ts:344`
  - body: Current main emits removed cron events with the deleted job snapshot, and the plugin hook docs/tests rely on that so external schedulers can reconcile removals. The PR head emits only `{ jobId, action: "removed" }`, which would regress the `cron_changed` payload when this branch is merged onto current main. Please preserve `job: removedJob` while adding cleanup.
  - confidence: 0.9
