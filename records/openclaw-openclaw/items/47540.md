## Summary

Keep open. Current main still lacks the requested pricing-threshold split, and merged #71297 explicitly left this PR's threshold behavior out of scope. The branch is not mergeable as-is because it is stale against the authenticated bootstrap handler, bases pricing hints on aggregate provider usage, and carries obsolete generated config-baseline artifacts.

## What This Changes

The PR adds `gateway.controlUi.chat.showPricingThreshold`, wires it through Gateway bootstrap and Control UI state, and changes chat context notices, styles, and tests to separate higher-rate pricing-threshold hints from model context-limit warnings.

## Best Possible Solution

Refresh or replace the branch on current main, add the chat setting inside the already-authorized bootstrap response, implement the threshold split in the current `ui/src/ui/chat/context-notice.ts` view-model path with a reliable fresh request-context signal, update only the current config-docs hash artifact, and cover the Gateway bootstrap plus UI context-notice behavior with focused regressions.
