## Summary

Keep this PR open because current main still lacks the requested pricing-threshold split and config toggle, but the branch is not mergeable as-is. The PR head is stale against current Control UI bootstrap auth hardening, bases pricing notices on aggregate session input tokens rather than the fresh context signal used on main, and carries obsolete generated config-baseline artifacts.

## What This Changes

The PR adds `gateway.controlUi.chat.showPricingThreshold`, wires it through Gateway bootstrap and Control UI state, and changes chat context notices, styles, and tests to separate higher-rate pricing-threshold hints from model context-limit warnings.

## Best Possible Solution

Refresh or replace the branch on current main, implement the threshold split inside the current `ui/src/ui/chat/context-notice.ts` view-model path, preserve bootstrap auth and fresh-token semantics, define a reliable pricing-threshold signal, and update schema/help/docs hash artifacts plus focused Gateway and UI regressions.
