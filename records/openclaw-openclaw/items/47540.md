## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks the proposed pricing-threshold setting, so the change is not obsolete; however the branch is stale against newer Control UI bootstrap auth, current fresh-context notice behavior, and the generated config-doc artifact workflow.

## What This Changes

The PR adds `gateway.controlUi.chat.showPricingThreshold`, wires it through Gateway bootstrap and Control UI state, and changes chat context notices, styles, and tests to separate pricing-threshold hints from model context-limit warnings.

## Best Possible Solution

Refresh or replace the branch on current main, add the chat setting inside the already-authorized bootstrap response, implement the threshold split in the current `ui/src/ui/chat/context-notice.ts` model, use a fresh context/request usage signal for threshold claims, and update only the current generated-doc hash artifact.

## Reproduction Assessment

Yes. The auth regression is reproducible by comparing PR-head `src/gateway/control-ui.ts:347-376` with current main `src/gateway/control-ui.ts:759-808`; the pricing false positive is reproducible from the branch logic with high `inputTokens` and low or missing fresh `totalTokens`.

## Solution Assessment

No. The current PR is useful source work, but it is not the best current solution because it applies the new behavior to stale gateway/UI paths and bases billing-threshold notices on aggregate session input rather than fresh context/request usage.
