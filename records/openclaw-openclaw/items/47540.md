## Summary

Keep this PR open. Current main still implements only the generic fresh-context Control UI notice and lacks the proposed pricing-threshold toggle, bootstrap payload, and model-threshold behavior, while the merged #71297 explicitly left this PR's threshold scope unresolved. The branch needs maintainer review/refresh before merge because it overlaps the newer context freshness and compaction CTA code and has an unresolved accuracy question about pricing-threshold hints using aggregate input tokens versus fresh context/pricing metadata.

## What This Changes

The PR branch adds `gateway.controlUi.chat.showPricingThreshold`, wires it through config and Gateway Control UI bootstrap state, and changes the Control UI chat notice rendering/styles/tests to separate higher-rate pricing threshold hints from model context-limit warnings.

## Best Possible Solution

Refresh the branch onto current main, keep the generic context notice helper and #71297 compaction behavior intact, then add the threshold hint only after deciding whether it should be driven by fresh context usage, provider/model pricing metadata, or another documented runtime signal. The final change should keep config schema/help/baselines, Gateway bootstrap contract, UI rendering, docs, and focused regression tests aligned.
