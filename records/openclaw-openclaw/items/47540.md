## Summary

Keep this PR open. Current main includes the later generic Control UI fresh-context notice and compaction CTA, but it still lacks this PR's requested split between higher-rate pricing-threshold hints and model context-limit warnings, lacks `gateway.controlUi.chat.showPricingThreshold`, and does not pass threshold display settings through the Gateway bootstrap path. The PR should get maintainer review and a rebase rather than cleanup closure.

## What This Changes

The PR proposes splitting Control UI context notices into separate higher-rate pricing threshold and model context limit signals, adding `gateway.controlUi.chat.showPricingThreshold`, and wiring that setting through config, Gateway bootstrap, UI rendering, styles, and tests.

## Best Possible Solution

Keep this PR open for maintainer review or refresh. The best path is to rebase it onto current main, preserve #71297's fresh-context and compaction CTA behavior, decide whether higher-rate threshold hints should use fresh context usage or provider/model pricing metadata, and then land a narrow config/Gateway/UI/test/docs update if the feature is still desired.
