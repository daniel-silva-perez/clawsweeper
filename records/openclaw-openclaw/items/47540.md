## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks the requested pricing-threshold split and `gateway.controlUi.chat.showPricingThreshold`, so the PR is not obsolete; however the branch is stale against current Control UI auth, context-freshness, and config-doc generation behavior, and the proposed pricing signal is still based on aggregate input tokens rather than the fresh context snapshot.

## What This Changes

The PR adds a Control UI chat pricing-threshold setting, wires it through Gateway bootstrap and UI state, and changes chat context notice rendering, styling, and tests to separate pricing-threshold hints from model context warnings.

## Best Possible Solution

Rebase or replace the branch on current main, implement the threshold split in the current `ui/src/ui/chat/context-notice.ts` view-model path, add the new chat setting inside the already-authorized bootstrap response, use a fresh request/context usage source for threshold notices, and update only the current config docs hash artifact with focused Gateway and Control UI regressions.
