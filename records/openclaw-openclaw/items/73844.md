## Summary

Keep this PR open. Current main still has the exact `isServerErrorMessage()` guard this PR changes, and the existing regression tests currently assert that plain `Status: Internal Server Error` text is not failover-worthy while the `code:500` variant is. The PR is a narrow implementation candidate, not obsolete on main, but it needs maintainer review of the false-positive tradeoff plus a targeted test update.

## Best Possible Solution

Keep the PR open as a focused bug-fix candidate. Maintainers should decide whether a pure `Status: Internal Server Error` message is failover-worthy, then update the targeted failover classification tests to cover the accepted behavior and run the relevant agent helper tests before merging or closing the PR.
