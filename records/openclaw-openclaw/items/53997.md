## Summary

Keep open. Current main does not implement the PR's wrapper-owned ACPX terminal-truth artifacts, and the diagnostic need is still relevant through open #66465 and #48580. The PR should not be closed as implemented, but it also should not be merged as-is because current main now delegates ACPX runtime execution to the pinned `acpx` package and this branch patches the older in-repo CLI-wrapper shape; the diff also persists raw route arguments, which is security-sensitive around MCP server env payloads.

## What This Changes

This PR adds ACPX per-run `route.json`, `terminal.json`, `stdout.log`, and `stderr.log` artifacts, strict terminal-state finalization, abort-before-spawn handling, route-mirror/run-id retry fixes, and focused runtime/artifact tests against the older ACPX runtime wrapper.

## Best Possible Solution

Implement durable ACPX terminal-truth and useful stdout/stderr persistence at the current runtime ownership boundary, preferably upstream in `openclaw/acpx` if the behavior is reusable, then consume it from the plugin with redacted artifact metadata, focused abort/setup-failure/stderr/route-mirror tests, and ACP docs once artifact locations and retention are stable.
