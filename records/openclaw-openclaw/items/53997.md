## Summary

Keep open for maintainer review. The PR still represents useful ACPX terminal-truth work, but it is not implemented on current main, targets an older wrapper-owned runtime shape, and should not merge as-is because the diff persists secret-bearing route arguments.

## What This Changes

The PR adds wrapper-owned ACPX per-run `route.json`, `terminal.json`, `stdout.log`, and `stderr.log` artifacts, strict terminal-state finalization, abort-before-spawn handling, collision-safe run-id allocation, and focused tests for the older CLI-wrapper ACPX runtime.

## Best Possible Solution

Add durable ACPX terminal-truth and post-mortem stdout/stderr persistence at the current ACPX runtime ownership boundary, preferably in the published `openclaw/acpx` package when reusable, then consume it from the bundled plugin with redacted artifact metadata, explicit retention/location semantics, and focused tests for terminal states, abort/setup failures, stderr capture, and route-mirror behavior.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable for the PR's feature request as a whole, but yes for the blocking security finding: configure `plugins.entries.acpx.config.mcpServers.*.env`, run an ACPX turn on the PR branch, and inspect `route.json` to see the encoded `--agent` payload persisted through `route_args`.

## Solution Assessment

Is this the best way to solve the issue? No. The PR solves the older wrapper-owned spawn path and stores raw route metadata; the maintainable path is to add the contract at the current delegated `acpx` runtime boundary and only persist sanitized route information.

Full review comments:

- **[P1] Redact route args before persisting artifacts:** `extensions/acpx/src/runtime.ts:579`
  - body: `args` can include `--agent <mcp-proxy payload>` when MCP servers are configured, and that payload encodes `mcpServers` including `env` values. Writing it verbatim to `route.json` durably copies credentials into the state artifact directory; persist a sanitized route view or omit sensitive flags.
  - confidence: 0.92
- **[P2] Clean up the abort listener when artifact setup fails:** `extensions/acpx/src/runtime.ts:572`
  - body: The abort listener is registered before `createRunArtifacts`, but listener removal only happens in the later `try/finally`. If artifact creation throws, for example on an unwritable state directory, the signal keeps a stale listener that can call `cancel` later and accumulate across retries.
  - confidence: 0.86
