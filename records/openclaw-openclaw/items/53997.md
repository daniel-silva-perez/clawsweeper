## Summary

Keep open. Current main does not implement this PR’s terminal-truth artifact layer: the ACPX plugin now delegates runtime execution to the published `acpx/runtime` package, and the proposed `run-artifacts.ts`/artifact tests are absent. The PR may still contain useful product work, but it would need to be rebased or redesigned against the current ACPX runtime boundary and the unresolved security review concern around persisted route args should be addressed before merge.

## Best Possible Solution

Keep this PR open for maintainer judgment. The best path is to rebase or split the useful terminal-truth/stderr artifact work against the current `acpx/runtime` boundary, preferably upstream in `openclaw/acpx` if it is generic ACPX behavior, and require a redaction design plus focused tests before merging.

## Risks / Open Questions

- The PR targets a pre-refactor runtime implementation and cannot be treated as already implemented on current main.
