## Summary

Keep open. Current main does not implement the terminal-truth artifact behavior, but this PR is not merge-ready because it targets an older ACPX CLI-wrapper shape while current main delegates runtime execution to the published `acpx/runtime` package, and the proposed artifact metadata has a concrete secret-persistence defect.

## What This Changes

The PR adds wrapper-owned ACPX per-run `route.json`, `terminal.json`, `stdout.log`, and `stderr.log` artifacts, strict terminal-state finalization, abort-before-spawn handling, route-mirror/run-id retry fixes, and focused tests for the older CLI-wrapper ACPX runtime.

## Best Possible Solution

Implement durable ACPX terminal-truth and stdout/stderr persistence at the current runtime ownership boundary, preferably upstream in `openclaw/acpx` when reusable, then consume it from the plugin with redacted metadata, retention/diagnostic docs, and focused setup-failure, abort, stderr, route-mirror, and MCP-secret regression coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Redact secret-bearing route args before writing artifacts:** `extensions/acpx/src/runtime-internals/run-artifacts.ts:191`
  - body: `route_args` stores the full prompt command arguments in `route.json`. In the MCP-proxy path those args include a `--payload` generated from `mcpServers`, including `env` values, so this would durably copy API tokens or other secrets into the OpenClaw state directory on every turn. Persist a sanitized route summary or omit sensitive flags before writing the artifact.
  - confidence: 0.92
- **[P2] Clean up the abort listener when artifact setup fails:** `extensions/acpx/src/runtime.ts:572-584`
  - body: The abort listener is attached before `createRunArtifacts`, but listener removal only happens in the later `try/finally`. If artifact setup throws, for example because the state directory is unwritable, `runTurn` exits with the listener still attached to the caller's signal and it can later invoke `cancel` unexpectedly. Move artifact creation inside the cleanup scope or add an outer `finally`.
  - confidence: 0.86
