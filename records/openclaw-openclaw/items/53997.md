## Summary

Keep this PR open. Current main still lacks the wrapper-owned ACPX terminal-truth artifact contract, and the PR now targets an older runtime shape than current main, which delegates turns to the packaged `acpx/runtime`. The branch remains potentially useful for ACPX post-mortem debugging, but it needs maintainer rework and a security pass before merge.

## What This Changes

This PR adds ACPX wrapper-owned per-run artifacts (`route.json`, `terminal.json`, `stdout.log`, `stderr.log`), strict terminal states, abort-before-spawn handling, and focused artifact tests against the older in-repo runtime wrapper.

## Best Possible Solution

Keep this PR open for maintainer review, but require a current-main rebase or replacement design. The best path is to implement terminal-truth artifacts at the current ACPX runtime ownership boundary, preferably in `openclaw/acpx` if the behavior is reusable; persist useful stdout/stderr for #66465; redact or omit sensitive route metadata; add focused artifact, abort, setup-failure, stderr, and route-mirror tests; and update ACP docs once artifact locations and retention are stable.
