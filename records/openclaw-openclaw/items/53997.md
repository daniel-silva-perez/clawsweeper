## Summary

Keep open for maintainer review. Current main does not implement the requested durable ACPX artifact contract, but the PR is not merge-ready because it targets the older wrapper-owned spawn loop while main now delegates turns to the published `acpx/runtime` package, and the PR introduces a credential-persistence regression in `route.json`.

## What This Changes

The PR adds wrapper-owned ACPX per-run `route.json`, `terminal.json`, `stdout.log`, and `stderr.log` artifacts, strict terminal-state finalization, collision-safe run IDs, abort-before-spawn handling, and focused tests to the older ACPX CLI-wrapper runtime.

## Best Possible Solution

Add durable ACPX terminal truth and post-mortem stream capture at the current ACPX ownership boundary, preferably in the published `openclaw/acpx` runtime when reusable, then consume it from the bundled plugin with redacted artifact metadata, explicit retention/location behavior, and tests for terminal states, setup-failure cleanup, stderr capture, route-mirror behavior, and MCP env redaction.

## Reproduction Assessment

Not applicable for the feature request as a whole. Yes for the blocking security finding: on the PR branch, configure `plugins.entries.acpx.config.mcpServers.*.env`, run an ACPX turn, and inspect `route.json` to see `route_args` persist the `--agent` payload path that can include those env values.

## Solution Assessment

No. The PR is not the best current solution because it patches the pre-refactor wrapper-owned subprocess path and persists unsanitized route metadata; the safer path is a current-runtime artifact contract with sensitive route data omitted or redacted.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Redact route args before persisting artifacts:** `extensions/acpx/src/runtime.ts:579`
  - body: `args` can include `--agent <payload>` when MCP servers are configured, and that payload can carry `mcpServers` entries with `env` values. Persisting it verbatim in `route.json` durably copies credentials into the state artifact directory, so store a sanitized route view or omit sensitive flags.
  - confidence: 0.92
- **[P2] Move artifact creation inside abort cleanup:** `extensions/acpx/src/runtime.ts:572`
  - body: The abort listener is registered before `createRunArtifacts`, but listener removal only happens in the later `finally`. If artifact setup throws, for example on an unwritable state directory, the signal keeps a stale listener that can call `cancel` later and accumulate across retries.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete credential-exposure risk by durably persisting raw ACPX route arguments.

Concerns:

- **[high] Secret-bearing route args are persisted:** `extensions/acpx/src/runtime.ts:579`
  - body: `routeArgs: args` is written into durable `route.json`; in MCP mode those args can include an `--agent` payload built from configured MCP servers, including `env` values. This can copy API tokens or other secrets into per-run artifacts.
  - confidence: 0.92
