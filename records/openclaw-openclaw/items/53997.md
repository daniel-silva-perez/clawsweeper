## Summary

Keep open. Current main does not implement the requested ACPX per-run terminal-truth artifacts, and this PR still needs maintainer handling before merge because it targets a pre-refactor wrapper-owned runtime path, persists secret-bearing route arguments, and leaves an abort listener cleanup bug on artifact setup failures.

## What This Changes

The PR adds wrapper-owned ACPX `route.json`, `terminal.json`, `stdout.log`, and `stderr.log` artifacts, strict terminal-state finalization, collision-safe run IDs, abort-before-spawn handling, and focused tests.

## Best Possible Solution

Keep the terminal-truth/debugging goal, but implement it at the current ACPX ownership boundary: preferably as a reusable artifact/result contract in the published `acpx` runtime, then expose only sanitized, documented metadata through the bundled plugin with tests covering terminal states, setup-failure cleanup, route mirror failure, stderr capture, and MCP env redaction.

## Reproduction Assessment

Yes for the blocking findings: by code trace on the PR head, configuring ACPX `mcpServers.*.env`, running a turn, and inspecting the generated `route.json` would show `route_args` containing the `--agent` payload. The broader feature request is not a bug reproduction, and current main can only be checked for absence of the artifact surface.

## Solution Assessment

No. The PR is not the best solution as-is because it patches an older wrapper-owned subprocess path and persists unsanitized route metadata; the safer direction is a current-runtime artifact contract with secret-bearing route data omitted or redacted.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Redact route args before persisting artifacts:** `extensions/acpx/src/runtime.ts:579`
  - body: `args` can include `--agent <payload>` when MCP servers are configured, and that payload encodes `mcpServers` entries including env values. Persisting it verbatim in `route.json` copies credentials into durable artifacts, so store a sanitized route view or omit sensitive flags.
  - confidence: 0.92
- **[P2] Clean up abort listeners when artifact setup fails:** `extensions/acpx/src/runtime.ts:572`
  - body: The abort listener is registered before `createRunArtifacts`, but listener removal only happens in the later `finally`. If artifact setup throws, for example on an unwritable state directory, the signal keeps a stale listener that can call `cancel` later and accumulate across retries.
  - confidence: 0.86
- **[P3] Avoid cancelling twice after pre-spawn aborts:** `extensions/acpx/src/runtime.ts:607`
  - body: If the signal fires after the listener is attached but before the post-artifact abort check, `onAbort` already called `cancelOnAbort`; this branch calls it again. Drop the second call or gate it so abort-before-spawn does not send duplicate cancel requests.
  - confidence: 0.73

## Security Review

Status: needs_attention

Summary: The PR introduces a credential-exposure risk by durably persisting raw ACPX route arguments.

Concerns:

- **[high] Secret-bearing route args are persisted:** `extensions/acpx/src/runtime.ts:579`
  - body: `routeArgs: args` is written into durable `route.json`; in MCP mode those args can include an `--agent` payload built from configured MCP servers, including env values. This can copy API tokens or other secrets into per-run artifacts.
  - confidence: 0.92
