## Summary

Keep open and do not merge as-is. The PR is useful source work for durable ACPX terminal artifacts, but current main has moved the ACPX plugin to a delegated `acpx@0.6.1` runtime boundary, while this branch patches the older CLI-wrapper spawn loop and also persists secret-bearing route arguments into durable artifacts.

## What This Changes

The PR adds ACPX wrapper-owned per-run `route.json`, `terminal.json`, `stdout.log`, and `stderr.log` artifacts, strict terminal-state finalization, abort-before-spawn handling, collision-safe run-id allocation, and focused tests for the older ACPX CLI-wrapper runtime.

## Best Possible Solution

Implement durable ACPX terminal-truth and useful stdout/stderr persistence at the current runtime ownership boundary, preferably in the published `openclaw/acpx` runtime when the behavior is reusable, then consume it from the bundled plugin with redacted artifact metadata, explicit retention/location semantics, focused abort/setup-failure/stderr/route-mirror tests, and ACP docs once the operator contract is stable.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable for the PR as a feature implementation, but the blocking security finding has a concrete path: configure ACPX `mcpServers.*.env`, run a turn on the PR branch, and inspect `route.json` to see the encoded `--agent` payload persisted through `route_args`.

## Solution Assessment

Is this the best way to solve the issue? No. The branch solves the old wrapper shape rather than current main's delegated ACPX runtime, and a safer solution should add the artifact contract at the current `acpx@0.6.1` boundary while redacting or omitting secret-bearing route metadata.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Redact route args before persisting artifacts:** `extensions/acpx/src/runtime.ts:579`
  - body: `args` can include `--agent <mcp-proxy payload>` when MCP servers are configured, and that payload encodes `mcpServers` including `env` values. Writing it verbatim to `route.json` durably copies credentials into the state artifact directory; persist a sanitized route view or omit sensitive flags.
  - confidence: 0.92
- **[P2] Clean up the abort listener when artifact setup fails:** `extensions/acpx/src/runtime.ts:572`
  - body: The abort listener is registered before `createRunArtifacts`, but listener removal only happens in the later `try/finally`. If artifact creation throws, for example on an unwritable state directory, the signal keeps a stale listener that can call `cancel` later and accumulate across retries.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete credential-exposure risk by persisting raw ACPX route arguments that can include encoded MCP server environment values.

Concerns:

- **[high] Secret-bearing route args are persisted:** `extensions/acpx/src/runtime.ts:579`
  - body: `routeArgs: args` is written into durable `route.json`; on the PR branch, MCP mode builds those args with `--agent` pointing to a base64url payload that includes configured MCP servers and their `env` values. This can copy API tokens or other secrets into per-run artifacts.
  - confidence: 0.92
