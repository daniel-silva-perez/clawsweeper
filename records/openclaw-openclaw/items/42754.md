## Summary

Keep open. Current main still drops Feishu card-action sibling fields before dispatch: the event type only exposes `action.value`/`action.tag`, the webhook parser reconstructs `action` from only those two fields, and the decoder/fallback path only consumes `event.action.value`. The provided discussion also shows active implementation candidates, including same-author PR #42908 for the full `option`/`options`/`form_value` request and PR #73514 for a narrower `form_value` subset, so this should remain open until an appropriate Feishu-owned fix lands.

## Best Possible Solution

Review and land a Feishu-owned implementation that preserves raw card-action sibling fields through `parseFeishuCardActionEventPayload`, widens the handler/decoder types, defines the intended command or JSON fallback behavior, and adds focused tests for dropdown, multi-select, and form submissions. PR #42908 is the broad same-author candidate; PR #73514 may be useful for the form-submission subset but is not sufficient for the full issue by itself.
