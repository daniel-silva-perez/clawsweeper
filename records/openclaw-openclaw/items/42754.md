## Summary

Keep open. Current main still only preserves `action.value` and `action.tag` for Feishu card actions, and the decoder/fallback path only reads `event.action.value`, so `action.option`, `action.options`, and `action.form_value` are still dropped before dispatch. The GitHub context also shows an open same-author implementation candidate, #42908, plus a newer narrower PR #73514, so this issue should remain open until an appropriate PR lands.

## Best Possible Solution

Review and land a Feishu-owned implementation that preserves the raw card-action sibling fields through `parseFeishuCardActionEventPayload`, widens the handler/decoder types, defines the intended command or JSON fallback format, and adds focused tests for dropdown, multi-select, and form submissions. #42908 is the broad same-author candidate; #73514 may be useful for the form submission subset but is not sufficient by itself for the full request.
