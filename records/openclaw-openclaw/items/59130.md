## Summary

Current main still matches the reported memory-lancedb behavior: recall is semantic-vector ranked, the auto-recall path injects the top matches without recency-aware ranking, and the injected context does not include createdAt/source timestamp provenance. There is also an open same-author PR, #59141, whose body uses closing syntax for this issue, so policy says this issue should remain open until that PR or an equivalent fix is merged.

## Best Possible Solution

Keep this issue open. Review and update PR #59141 or land an equivalent memory-lancedb fix that detects freshness intent, widens the candidate set, reranks with `createdAt` and future source timestamps, exposes recorded/provenance hints in recalled context or tool details, and adds a regression test where semantic similarity would otherwise pick the older memory.
