## Summary

Keep open: current main still performs memory-lancedb recall by semantic LanceDB order only for freshness-sensitive prompts, and the explicitly linked implementation PR remains open/unmerged. The existing context warning helps with prompt safety but does not provide timestamped provenance or recency-aware ranking.

## What This Changes

The issue requests recency-aware and provenance-aware memory-lancedb recall so latest/current queries do not answer from stale semantic matches.

## Best Possible Solution

Land PR #59141 or a current-main replacement that keeps the fix extension-owned: detect freshness intent in memory-lancedb, widen recall candidates, rank by semantic score plus timestamp, include timestamp/provenance cues in injected context and tool output, and add a regression where semantic similarity alone would choose the older memory.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to stub LanceDB vectorSearch so an older high-similarity memory precedes a newer lower-similarity memory for a latest/current prompt; current before_prompt_build injects the returned semantic order unchanged.

## Solution Assessment

Unclear. The linked PR's direction is the right narrow owner-module fix, but it should be reviewed against current before_prompt_build behavior and the maintainers should decide whether structured source provenance must ship in the same change or as a follow-up.
