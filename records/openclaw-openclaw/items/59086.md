## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: chat-level streamed reasoning is not forwarded through Gateway chat events, the Control UI live stream model is still string-only, and the streaming renderer explicitly disables reasoning. The PR also has useful review context, including bot comments that identify P2 edge cases to resolve before landing.

## Best Possible Solution

Keep this PR open and review it as an implementation candidate. The best path is to refresh it against current main, address the P2 review comments, compare or consolidate with #68982, and land one scoped implementation that adds chat-level thinking deltas, maps real run ids to client run ids, and renders structured reasoning/text/tool blocks in order without regressing existing final reasoning or tool visibility behavior.
