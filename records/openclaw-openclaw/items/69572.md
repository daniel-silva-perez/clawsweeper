## Summary

Keep this issue open. Current main still implements Feishu reply typing as a persistent `Typing` message reaction, and the docs/tests still describe and assert reaction-based typing. The older #8722/#43084 cleanup only confirms that reaction-based typing and keepalive behavior exist; it does not resolve this newer report asking for native Feishu typing/status behavior instead of message reactions.

## Best Possible Solution

Keep the issue open as a Feishu bundled-plugin behavior bug. The best path is to verify the official Feishu/Lark native typing or follow-up status API, then update the Feishu-owned typing lifecycle in `extensions/feishu/src/typing.ts` and `extensions/feishu/src/reply-dispatcher.ts` to use that native behavior when supported, while preserving config gating/backoff and updating tests plus `https://docs.openclaw.ai/channels/feishu`. If Feishu has no native bot typing API, document that limitation and avoid presenting message reactions as native typing.
