## Summary

Keep open. Current main still implements Feishu typing feedback as `im.messageReaction.create/delete` with the `Typing` emoji, and the dispatcher, tests, and Feishu docs still describe/assert reaction-based typing. The related open Feishu PRs found in the supplied context and live search are reaction-target/customization/ack-reaction work, not a native Feishu/Lark typing-status replacement.

## Best Possible Solution

Keep this issue open as a bundled Feishu plugin behavior bug. A maintainer should confirm the official Feishu/Lark API and permissions for native typing/status/follow-up behavior, then update the Feishu-owned typing lifecycle in `extensions/feishu/src/typing.ts` and `extensions/feishu/src/reply-dispatcher.ts`, plus the reaction/comment cleanup paths if they share the same user-facing setting. Tests and `https://docs.openclaw.ai/channels/feishu` should be updated to distinguish native typing from reaction fallback. If Feishu has no native bot typing API, document that limitation and stop presenting message reactions as native typing.
