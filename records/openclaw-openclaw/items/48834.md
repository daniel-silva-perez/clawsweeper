## Summary

Keep open. Current main has the Feishu ACP/session-binding foundation and a generic `/focus`/`/unfocus` path, but it still does not implement this PR's full intended change: non-streaming Feishu ACP block delivery is still suppressed, Feishu `threadBindings` config/policy/lifecycle support is incomplete, user-facing command/docs text still omits Feishu focus support, and the PR's ACP env/SDK changes need a current-main security rebase.

## Best Possible Solution

Keep this PR open for a focused rebase against current main: reuse the generic conversation-binding resolver already present for Feishu, implement Feishu ACP block delivery without final-reply dedupe loss, add Feishu `threadBindings` schema/config metadata/startup gating plus idle and max-age lifecycle hooks, update `/focus` and `/unfocus` user-facing docs/registry text, and add targeted tests for DM, topic, `group_topic_sender`, disabled-policy, lifecycle, and repeated ACP block delivery. The ACP environment and SDK/export changes should get an explicit security pass before merge.

## Risks / Open Questions

- The PR touches ACP child-process environment handling and plugin SDK/runtime exports; those are security- and supply-chain-sensitive surfaces and need a fresh review against current main before merge.
- The PR branch references older paths such as `src/plugin-sdk/feishu.ts` and `extensions/acpx/src/runtime-internals/process.ts` that do not exist on current main, so a direct merge would likely revive stale architecture or conflict with the current generic SDK layout.
- Feishu ACP block delivery should avoid routing block deltas through final-text dedupe; current main still suppresses those blocks, and the earlier review thread identified truncation risk if repeated ACP block chunks are promoted as finals.
- Feishu thread-binding config must be wired through schema, metadata, startup policy, lifecycle setters, and tests together; adding only the config shape or only the manager registration would leave disable flags and idle/max-age behavior misleading.
