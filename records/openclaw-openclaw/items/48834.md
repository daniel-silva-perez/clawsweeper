## Summary

Keep open. Current main has the Feishu ACP/session-binding foundation and a generic conversation-binding command path, but it still does not fully implement this PR's intended change: Feishu ACP block delivery for non-streaming block payloads, Feishu threadBindings config/policy lifecycle support, user-facing docs/command descriptions for Feishu focus controls, and the PR's ACP env handling still need a current-main security/rebase review.

## Best Possible Solution

Keep this PR open and ask for a focused revision against current main: use the generic conversation-binding contract already present for Feishu command context, implement Feishu ACP block delivery without routing block deltas through final-text dedupe, add Feishu threadBindings schema/config metadata/lifecycle gating with idle and max-age behavior, update user-facing docs and command descriptions, and cover DM/topic/group_topic_sender plus block-delivery behavior with targeted tests. The ACP environment and SDK/export changes should get an explicit security-oriented maintainer pass.
