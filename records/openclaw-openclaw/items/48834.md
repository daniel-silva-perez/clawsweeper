Keep open. The PR also still needs maintainer review for sender-scoped account config and security-sensitive runtime/SDK boundary changes.

## Best Possible Solution

Keep this PR open, but require a focused revision or maintainer replacement that lands the remaining Feishu behavior on current main: deliver Feishu ACP block chunks when streaming-card delivery is inactive, add Feishu threadBindings schema and lifecycle hooks through the generic ChannelConversationBindingSupport contract, update /focus and /unfocus command/docs surfaces, cover DM/topic/sender-scoped routing with targeted tests, and resolve the active-account config concern before merge.
