## Summary

Keep open. Current main has part of the Feishu conversation-binding foundation and generic `/focus` plumbing, but it does not implement the full PR intent: Feishu ACP block delivery is still suppressed in the non-streaming block path, Feishu does not expose `threadBindings` config/lifecycle/setter hooks through the channel binding surface, user-facing command/docs copy still names only Discord/Telegram for `/focus` and `/unfocus`, and the branch includes a security-sensitive ACP child-process environment change that needs a current-path review before merge.

## Best Possible Solution

Keep this PR open for a focused rebase or replacement against current main: reuse the generic conversation-binding and `conversation-runtime` surfaces already present, fix non-streaming Feishu ACP block delivery with regression tests, add Feishu `threadBindings` schema/config metadata and lifecycle/update hooks through `conversationBindings`, update `/focus` and `/unfocus` command/docs copy, and redo the ACP environment change as a current-path security-reviewed patch.
