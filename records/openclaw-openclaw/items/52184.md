## Summary

Keep this issue open. Current main still only normalizes Homebrew Cellar Node paths; a supported Volta tools/image Node executable would flow into the generated service arguments unchanged. The issue also has open PRs with closing syntax, so the right maintainer path is to review/land or close those implementation candidates, not close the issue early.

## Best Possible Solution

Keep the issue open and resolve it through the focused daemon/runtime-path change: teach `resolveStableNodePath` to detect Volta tools/image Node paths, honor `VOLTA_HOME` or the user home shim where appropriate, fall back safely when the shim is unavailable, and add targeted tests for the macOS LaunchAgent path selection. Review PR #52252 or its fork #1230 as the implementation candidates.

## Evidence
