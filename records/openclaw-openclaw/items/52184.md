## Summary

Keep open. Current main still only normalizes Homebrew Cellar Node paths; a Volta tools/image Node executable would pass through unchanged and then be written as the LaunchAgent runtime. The issue also has open implementation candidates with closing syntax, so the right maintainer path is to review/land or close those PRs rather than closing this issue early.

## Best Possible Solution

Keep this issue open and resolve it through the focused daemon/runtime-path change: teach `resolveStableNodePath` to detect Volta tools/image Node paths, prefer the stable Volta shim from `VOLTA_HOME` or the user's home when accessible, fall back to the original executable when the shim is unavailable, and add targeted tests for resolver and macOS service argument selection. Review PR #52252 or the fork PR BingqingLyu/openclaw#1230 as the implementation candidates.
