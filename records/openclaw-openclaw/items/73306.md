## Summary

Keep open. Current main still has the Active Memory timeout-boundary mismatch at the center of the report: the outer watchdog and prompt-build hook include setup grace, but the embedded runner still receives the raw configured timeout. An open closing PR already targets that exact gap, so this issue should remain open until that PR or an equivalent fix lands.
