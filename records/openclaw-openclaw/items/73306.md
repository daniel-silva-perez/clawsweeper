## Summary

Keep open. Current main still has the Active Memory timeout-budget mismatch reported here: the hook/watchdog use the setup-grace-aware budget, but the embedded recall runner still receives the raw configured timeout. The issue also has an open closing PR, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports Active Memory timing out with zero recalled summary output during chat turns after upgrading, with provider-independent logs and token-bloat impact when recall fails.

## Best Possible Solution
