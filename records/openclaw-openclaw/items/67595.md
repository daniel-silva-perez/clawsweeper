## Summary

Keep open. Current main appears to have addressed the LaunchAgent token/service-env embedding loop, and a maintainer explicitly confirmed that as a partial fix, but the separate `openclaw channels list` provider-usage failure is still present in current main.

## Best Possible Solution

Keep this issue open, or split it only after creating a focused follow-up for the remaining `openclaw channels list` behavior. The remaining fix should make provider usage enrichment optional and non-fatal for both human and JSON output, avoid raw `TypeError: fetch failed` output for a simple channel/auth listing, and add regression coverage for no configured chat channels plus auth profiles with a failing usage provider.
