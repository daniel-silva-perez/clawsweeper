## Summary

Keep open. The LaunchAgent/service-token half appears fixed on current main and a maintainer explicitly confirmed that partial fix, but the separate `openclaw channels list` provider-usage failure is still present in current source: usage fetching is enabled by default, human output still prints raw fetch errors, and JSON output can still fail outright when usage fetching rejects.

## Best Possible Solution

Keep this issue open, or split it only after creating a focused follow-up for the remaining `openclaw channels list` behavior. The fix should make provider usage enrichment optional and non-fatal for both human and JSON output, avoid raw `TypeError: fetch failed` output for channel/auth listing, and add regression coverage for no configured chat channels plus auth profiles with a failing usage provider.
