## Summary

Keep open. Current main still has only adjacent compaction safeguards: silent memory flush, start/end user notices, post-compaction context reinjection, compaction provider hooks, and observe-only lifecycle hooks. It does not implement configurable pre-compaction warning thresholds, a structured visible pre-compaction handoff turn with timeout/prompt policy, or a bounded/audited deferral mechanism. The prior ClawSweeper comment remains accurate against the checked-out main SHA.

## Best Possible Solution

Keep this open for maintainer product/API design. A complete direction would define validated warning/critical threshold config, channel/UI delivery semantics for pre-compaction warnings, a structured pre-compaction handoff turn with timeout and prompt policy, and a bounded/audited deferral contract if maintainers want agents or plugins to pause imminent compaction. It should reuse existing memoryFlush/notifyUser/hooks where possible and update config schema/types/help/docs plus focused runtime tests.
