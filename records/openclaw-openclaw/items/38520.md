## Summary

Keep open. Current main at ab5c8025c9d0a31873d311ae3e73f3664b0a1185 still implements only adjacent compaction safeguards: silent memory flush, start/end user notices, post-compaction context reinjection, compaction provider hooks, and observe-only lifecycle hooks. It does not implement configurable pre-compaction warning thresholds, a structured visible pre-compaction handoff turn with timeout/prompt policy, or a bounded/audited deferral mechanism.

## Best Possible Solution

Keep this open for maintainer product/API design. A complete direction would define validated warning/critical threshold config, channel/UI delivery semantics for pre-compaction warnings, a structured pre-compaction handoff turn with timeout and prompt policy, and a bounded/audited deferral contract if maintainers want agents or plugins to pause imminent compaction. It should reuse existing memoryFlush, notifyUser, and compaction hooks where possible, then update config schema/types/help/docs plus focused runtime tests.
