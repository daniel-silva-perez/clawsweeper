## Summary

Keep this PR open. Current main still does not implement passthrough provider-model metadata in `agents.defaults.models`: the runtime schema, TypeScript config type, generated schema, docs, and `applyModelDefaults()` behavior all still use the narrower `alias`/`params`/`streaming` contract. The PR remains the active implementation candidate for closed companion feature request #44468, but it still needs the normalized model-ref lookup concern and generated schema/docs follow-through resolved before merge.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is to revise the passthrough lookup to normalize provider/model refs consistently with existing model-selection helpers, keep the mutation-preservation fix and tests, add coverage for `contextWindow`, `maxTokens`, novel passthrough fields, and normalized provider aliases, then regenerate schema/help and update public config docs before maintainer review and merge.
