## Summary

Keep this PR open. Current main still lacks the requested passthrough model metadata behavior, and the PR is a useful focused implementation candidate, but the branch is not ready to merge because its override lookup bypasses the existing normalized model-ref contract and the public generated config schema/docs remain stale.

## What This Changes

The PR changes `agents.defaults.models` entries to accept arbitrary passthrough fields, merges non-`alias`/`params`/`streaming` keys into provider model definitions in `applyModelDefaults()`, and adds tests for `contextWindow`, `maxTokens`, known-key filtering, and novel fields.

## Best Possible Solution

Land a revised config change that treats passthrough model metadata as a public config contract: use the existing normalized model-ref path for lookup, keep provider model defaulting deterministic, and refresh generated schema, docs, and tests before maintainer merge.

## Reproduction Assessment

Yes. A static reproduction is clear: configure `models.providers.amazon-bedrock` and `agents.defaults.models["bedrock/<model>"].contextWindow`; existing selection helpers normalize that key to `amazon-bedrock/<model>`, but this PR’s exact lookup only searches `amazon-bedrock/<model>` and drops the override.

## Solution Assessment

No, not yet. The passthrough direction is maintainable after the mutation-preservation fix, but the implementation should reuse the normalized allowlist/model-ref contract and update the generated schema/docs before merge.
