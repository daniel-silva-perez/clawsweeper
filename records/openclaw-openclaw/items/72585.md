## Summary

Keep this PR open. Current main still has the reported Control UI bug: the Set Default handler writes `agents.defaultId`, while the strict config schema only accepts `agents.defaults` and `agents.list`, and default-agent resolution already uses `agents.list[].default`. The PR is a focused implementation candidate for open issue #65565, though its regression test should be improved to exercise the real handler wiring.

## Best Possible Solution

Keep this PR open as the current focused fix candidate for #65565. Before landing, move the set-default mutation into a small controller/helper that `app-render.ts` calls and test that helper, or add a test that invokes the actual rendered `onSetDefault` path, then land the UI-only fix after the normal changed-surface checks.
