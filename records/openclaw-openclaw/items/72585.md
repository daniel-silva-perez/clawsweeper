## Summary

Keep this PR open. Current main still has the Control UI bug: the Set Default button is wired to write `agents.defaultId`, while the strict agents config schema only accepts `agents.defaults` and `agents.list`, and runtime default-agent selection already uses `agents.list[].default`. The PR is still a relevant implementation candidate for #65565, though its regression test should be tightened to exercise the real handler path.

## Best Possible Solution

Keep this PR open as the focused fix candidate for #65565. Before landing, move the set-default mutation into a small UI controller/helper that `app-render.ts` calls and test that helper, or add coverage that invokes the actual rendered `onSetDefault` path. Then land the UI-only fix after the normal changed-surface checks.
