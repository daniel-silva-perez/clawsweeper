## Summary

Keep this PR open. Current main still does not implement the PR's intended behavior: Feishu inbound `thread_id`/`root_id` still forces `threadReply`, and the reply dispatcher converts that into `reply_in_thread` even when `replyInThread` is configured as `"disabled"`. The PR is not obsolete, but it needs a current-main port and a maintainer decision because existing tests now explicitly lock in the opposite behavior.

## Best Possible Solution

Keep this PR open. Maintainers should decide whether `replyInThread: "disabled"` is a hard off switch. If yes, port the change to `extensions/feishu/src/bot-content.ts`, preserve `group_topic` and `group_topic_sender` anchoring, and update the current regression test. If following existing Feishu thread context is intended, update the config schema/docs/test wording so disabled no longer promises normal inline replies. Coordinate with #40147 before landing either approach.
