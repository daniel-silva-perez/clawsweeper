## Summary

Keep this PR open. Current main still does not implement the PR’s intended behavior: Feishu inbound `thread_id`/`root_id` still forces `threadReply` and then `replyInThread`, even when configured as `"disabled"`. The PR remains a plausible focused bug fix, but it is stale against the current `bot-content.ts` split and overlaps with an unresolved product-semantics decision rather than being safely closeable.

## Best Possible Solution

Keep this PR open. If maintainers want `replyInThread: "disabled"` to be a hard off switch, port the change to `extensions/feishu/src/bot-content.ts`, preserve `group_topic` and `group_topic_sender` anchoring, and update the current regression test that expects forced thread replies under disabled config. If follow-existing-thread is intended, update the config schema/docs/test wording so disabled no longer promises normal inline replies.
