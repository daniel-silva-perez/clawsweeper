## Summary

Keep open. Current `main` still contains the synchronous iOS permission snapshot path that this PR targets: `makeConnectOptions(stableID:)` remains synchronous and calls `currentPermissions()`, and `currentPermissions()` still directly queries `SFSpeechRecognizer.authorizationStatus()`, constructs a throwaway `CLLocationManager()`, and calls `CLLocationManager.locationServicesEnabled()`. The PR’s async permission/refreshed-config work and added regression tests have not been absorbed into `main`, and the related CoreLocation-only PR #51433 was closed unmerged rather than superseding this work.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or a narrowed equivalent that moves the speech/CoreLocation permission snapshot work off the warning-prone main-actor path, avoids throwaway CoreLocation permission snapshots, preserves gateway auth/TLS/reconnect semantics across async gaps, and keeps focused regression tests for permission snapshots plus reconnect/refresh races.
