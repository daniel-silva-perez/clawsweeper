## Summary

Keep open. Current main still has the synchronous iOS permission snapshot path this PR targets: `makeConnectOptions(stableID:)` still calls `currentPermissions()`, and `currentPermissions()` still directly calls `SFSpeechRecognizer.authorizationStatus()`, constructs a throwaway `CLLocationManager()`, and calls `CLLocationManager.locationServicesEnabled()`. The PR implementation and added regression tests have not been absorbed into main, and the related CoreLocation-only PR #51433 was closed unmerged rather than superseding this work.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or a narrowed equivalent that moves the speech permission probe off the main actor, avoids throwaway CoreLocation permission snapshots, preserves gateway auth/TLS/reconnect semantics across async gaps, and keeps focused regression tests for permission snapshots plus reconnect/refresh races.
