## Summary

Keep open for maintainer review: current main still couples the Agents Skills panel and the top-level Skills tab through the same `skillsFilter` state, so the PR’s intended state split is not implemented on main. The diff is small and appears functionally sound, but the linked user-visible bug was closed through a separate autofill mitigation path, making the remaining question a maintainer UX/product call rather than an automatic cleanup close.

## What This Changes

The PR adds an `agentSkillsFilter` Control UI state field and rewires the Agents Skills panel to read and write that field instead of the top-level Skills tab filter.

## Best Possible Solution

If maintainers want independent filters, land a rebased equivalent of this small state split with focused regression coverage for the two Control UI filter surfaces. If the autofill mitigation is considered sufficient, close this as superseded by the closed #41605/#42293 bug path rather than as implemented on main.
