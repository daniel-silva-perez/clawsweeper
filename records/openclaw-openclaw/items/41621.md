## Summary

Keep this PR open. Current main still couples the Agents Skills panel and the top-level Skills tab through the same `skillsFilter` state, while the PR’s small diff isolates only the Agents panel read/write path and does not introduce a concrete functional or security regression.

## What This Changes

The PR adds an `agentSkillsFilter` state field, includes it in `AppViewState`, and rewires the Agents Skills panel filter prop and change handler to use it while leaving the top-level Skills tab on `skillsFilter`.

## Best Possible Solution

Land this PR after rebase and targeted verification, or apply the same isolated-state patch on current main with a small regression test/manual proof that filtering in Agents -> Skills does not affect the top-level Skills tab and vice versa.
