## Summary

Keep open. Current main still couples the Agents Skills panel and top-level Skills tab through the same `skillsFilter` state, so this PR's intended isolation is not implemented on main. The patch itself is narrow and appears correct, but the linked user symptom was closed through a different autofill-focused mitigation, so maintainers should decide whether the separate-filter UX is still wanted.

## What This Changes

The PR adds an `agentSkillsFilter` state field, exposes it in `AppViewState`, and rewires the Agents Skills panel to read and update that field instead of the top-level `skillsFilter`.

## Best Possible Solution

Decide whether independent cross-view filters are still part of the desired Control UI behavior; if yes, land this patch or an equivalent rebased version with focused proof, and if no, close it as superseded by the autofill-specific resolution.
