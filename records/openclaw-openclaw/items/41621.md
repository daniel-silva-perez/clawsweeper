## Summary

Current main still shares the same `skillsFilter` state between the Agents Skills panel and the top-level Skills tab, so this PR’s intended state isolation has not been implemented or superseded. The PR remains a plausible narrow UI bug fix rather than cleanup-close material.

## Best Possible Solution

Keep this PR open or replace it with an equivalent maintainer patch. The fix should add an isolated `agentSkillsFilter` state, include it in `AppViewState`, route only the Agents Skills panel filter prop and callback through it, keep the top-level Skills tab on `skillsFilter`, and add a small regression test or documented manual verification for switching between the two surfaces.
