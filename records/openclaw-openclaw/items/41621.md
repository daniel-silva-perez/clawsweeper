## Summary

Keep this PR open. Current main still routes both the Agents -> Skills panel and the top-level Skills tab through the same `skillsFilter` state, so the PR’s intended filter-state isolation is not implemented or superseded. The earlier linked bug was closed for a separate autofill mitigation, but this narrower cross-view persistence bug remains present.

## Best Possible Solution

Keep this PR open or apply an equivalent current-main patch: add `agentSkillsFilter` to `OpenClawApp` and `AppViewState`, wire only the Agents Skills panel `filter` and `onSkillsFilterChange` to it, keep the top-level Skills tab on `skillsFilter`, and add a focused regression test or manual verification for cross-view filter isolation.
