## Summary

Keep open. Current main still treats `agent_end` as a raw conversation hook, blocks it for non-bundled plugins unless `allowConversationAccess` is enabled, and has no `allowStateAccess` or settled lightweight turn-ended plugin contract. The reporter's linked Interaction Bridge use case is valid plugin API feedback, but changing this boundary needs maintainer product and security judgment rather than cleanup closure.

## What This Changes

Request a state-only hook permission or lightweight turn-ended lifecycle signal so non-bundled plugins can track agent completion without granting raw conversation access.

## Best Possible Solution

Keep raw-content hooks behind `allowConversationAccess` and define a minimal lifecycle-only plugin contract for turn completion. The end state should expose only run/session/state metadata, document when plugin authors should use it, and align config schema/help, SDK docs, hook docs, and contract/loader tests.
