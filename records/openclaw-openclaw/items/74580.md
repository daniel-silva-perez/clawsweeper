## Summary

Keep open. Current main still has only the raw conversation-access gate: `agent_end` is classified with `llm_input`, `llm_output`, and `before_agent_finalize`, and there is no `allowStateAccess` or other state-only lifecycle permission. The linked hardware/display plugin use case is concrete, but the issue should be narrowed because current main does not put `before_tool_call` or `before_agent_reply` behind `allowConversationAccess`.

## What This Changes

Request a granular plugin hook permission or state-only lifecycle hook so hardware/status plugins can observe agent activity without enabling raw conversation access.

## Best Possible Solution

Add a backwards-compatible sanitized lifecycle/state hook, or a separate minimal agent-end event, that emits run/session state such as started, tool-active, replying, and ended without messages, prompts, assistant text, or tool params. Keep existing raw-content hooks gated by `allowConversationAccess`, then update the public config schema, generated help, plugin docs, loader tests, and each agent harness that emits lifecycle events.
