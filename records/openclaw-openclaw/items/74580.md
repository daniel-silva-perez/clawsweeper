## Summary

Keep open. Current main still gates `agent_end` as a raw conversation hook and has no `allowStateAccess` or equivalent state-only turn-completion API for external plugins. The follow-up comment narrows the confirmed ask to `agent_end`/turn-ended state, which is valid plugin API work but needs maintainer product and security direction before changing the permission boundary.

## What This Changes

The issue requests a state-only way for plugins to detect agent turn completion without granting full conversation-content hook access.

## Best Possible Solution

Keep raw-content hooks behind `allowConversationAccess` and define a minimal state-only lifecycle contract for external plugins. The clean end state is either a documented/hardened lifecycle-only event subscription path or a sanitized turn-ended hook/event shape with only run/session/state fields, plus aligned schema/help, plugin docs, SDK docs, and contract/loader tests.
