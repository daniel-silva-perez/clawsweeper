## What This Changes

The PR adds identity-checked cleanup for the cron service's per-store lock Map and adds a white-box unit test for cleanup after the final queued operation and retention while another operation is queued.

## Best Possible Solution

Review and ship the identity-checked cleanup with the regression test, preserving same-state and same-store serialization while pruning only the final current per-store lock entry.

## Reproduction Assessment

Yes. The source-level reproduction is direct: each unique `storePath` call stores a `keepAlive` promise in `storeLocks`, and current main has no path that removes the key after the operation settles.

## Solution Assessment

Yes. The proposed identity check is the narrowest maintainable fix because earlier queued operations cannot delete a newer tail, future calls already fall back to a resolved promise, and the same cleanup pattern exists in the keyed async queue helper.
