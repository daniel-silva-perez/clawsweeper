## What This Changes

This PR adds identity-checked cleanup of completed per-store cron lock promises in `locked()` and adds a unit test for cleanup after the last queued operation while retaining the lock during queued work.

## Best Possible Solution

Review and ship the identity-checked `finally` cleanup in the cron lock helper with the regression test, preserving same-state and same-store serialization semantics while pruning the final resolved per-store lock entry.

## Reproduction Assessment

Yes. The current-main code path is directly reproducible by calling `locked()` with a unique `storePath`: the Map is set and never deleted; the proposed test covers that cleanup expectation, although it was not executed in this read-only review.

## Solution Assessment

Yes. The identity check is the narrowest maintainable fix because it lets earlier completed operations avoid deleting a newer queued lock, while future calls after the last operation already fall back to `Promise.resolve()`.
