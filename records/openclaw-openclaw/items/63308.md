## Summary

Keep open. Current main still exposes `sessionKey`, `label`, and `agentId` as independent optional `sessions_send` fields and still hard-rejects calls that include both `sessionKey` and `label`. The latest release tag shows the same guard. There is also an open PR, #63377, that uses closing syntax for this issue, plus related open runtime follow-up #64699/#64846, so the right cleanup is maintainer review/landing or consolidation of those PRs rather than closing this issue now.

## Best Possible Solution

Keep this issue open while maintainers choose the canonical fix. The strongest product fix is likely executor-side canonicalization where a present `sessionKey` is authoritative, paired with clearer schema/description text and regression coverage for mixed `sessionKey`+`label` calls. Then close this issue through the merged PR and consolidate #64699/#64846 as appropriate.
