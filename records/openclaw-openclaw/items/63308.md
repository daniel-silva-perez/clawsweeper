## Summary

Keep open. Current main still exposes sessionKey, label, and agentId as independent optional sessions_send fields and still hard-rejects calls that include both sessionKey and label. The latest release tag v2026.4.26 has the same behavior, and related open PRs #63339 and #64846 propose overlapping but incomplete/alternative fixes that need maintainer consolidation rather than cleanup closure.

## Best Possible Solution

Keep this issue open until a canonical PR lands. The strongest fix is to make sessionKey authoritative when present or otherwise define the exact mixed-selector semantics, add clear schema descriptions and docs examples for the two valid addressing modes, and cover sessionKey-only, label-only, and mixed sessionKey+label calls in focused sessions_send tests.
