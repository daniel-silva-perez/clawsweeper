## Summary

Keep open. Current main still exposes `sessionKey`, `label`, and `agentId` as independent optional `sessions_send` parameters and still returns the reported `Provide either sessionKey or label (not both).` error when both target selectors are present. The latest release tag `v2026.4.26` has the same behavior. Related implementation work is open but not merged, so this should stay open until a canonical fix lands.

## What This Changes

The issue asks to improve `sessions_send` selector guidance and/or normalize executor behavior so redundant `label`/`agentId` fields do not make a known `sessionKey` send fail.

## Best Possible Solution

Keep this issue open until a canonical PR lands. The strongest fix is to make `sessionKey` authoritative when present, ignore redundant `label`/`agentId` for that path, add clear model-facing schema/description/docs guidance for the two valid addressing modes, and cover `sessionKey`-only, `label`-only, and mixed-selector calls in focused `sessions_send` tests. Open PR #74009 looks like the runtime/test path to review, with #63339/#63556 carrying useful schema/docs guidance.
