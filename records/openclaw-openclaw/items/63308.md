## Summary

Keep open. Current main still exposes `sessionKey`, `label`, and `agentId` as independent optional `sessions_send` parameters and still returns the reported `Provide either sessionKey or label (not both).` error when both selectors are present. The latest release tag has the same guard, and related implementation/guidance PRs are open but unmerged.

## What This Changes

The issue asks OpenClaw to improve `sessions_send` selector guidance and/or normalize executor behavior so redundant `label` or `agentId` fields do not make a known `sessionKey` send fail.

## Best Possible Solution

Keep this issue open until a canonical fix lands: make `sessionKey` authoritative when present, ignore redundant `label` and `agentId` on that path, add clear model-facing schema/description/docs guidance for the two valid addressing modes, and cover `sessionKey`-only, `label`-only, and mixed-selector calls in focused tests.
