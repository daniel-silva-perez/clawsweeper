## Summary

Keep open. Current main at e46dccb still exposes `sessionKey`, `label`, and `agentId` as independent optional `sessions_send` parameters and still returns the reported mixed-selector error before send resolution. The latest release tag has the same guard, while #74009 and #63339 are open, unmerged fix paths for the runtime and schema/docs sides.

## What This Changes

The issue asks OpenClaw to improve `sessions_send` selector guidance and/or normalize execution so a known `sessionKey` is not rejected when redundant `label` or `agentId` fields are also present.

## Best Possible Solution

Land one canonical fix that makes `sessionKey` authoritative when present, ignores redundant `label` and `agentId` on that path, documents the two valid addressing modes in schema/tool/docs text, and adds focused regression coverage for `sessionKey`-only, `label`-only, and mixed-selector calls.
