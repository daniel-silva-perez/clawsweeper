## Summary

Keep this PR open. Current main still has the Daily Log versus grounded replay mapping bug, and the PR fixes the central summary/list mapping, but the branch still leaves the section-local clear action wired to grounded-only cleanup, so daily-only rows newly shown in that section would not be affected by the visible clear button.

## What This Changes

The PR changes the Dreaming Advanced tab to count and render `dailyCount > 0` entries for the From Daily Log summary/section, adjusts origin badge logic, keeps replay metadata visible, and adds UI coverage for daily-only and grounded-only entries.

## Best Possible Solution

Land a revised PR that uses `dailyCount` for the Daily Log summary and staged list, keeps grounded replay metadata visible, and either removes/renames the section-local clear control or wires it to an action whose effect matches the rows displayed there.

## Reproduction Assessment

Yes. A Dreaming Advanced fixture with `dailyCount > 0` and `groundedCount === 0` reproduces current main's empty or misleading Daily Log section, and the PR's own added fixture models that path; the clear-action regression is reproduced by clicking the section clear control on a daily-only row and following it to grounded-only cleanup.

## Solution Assessment

No, not as-is. The `dailyCount` mapping is the narrow maintainable fix for the reported UI bug, but the PR should also align the staged section's clear control with the newly displayed daily-backed entries or split grounded replay cleanup into a separate action.
