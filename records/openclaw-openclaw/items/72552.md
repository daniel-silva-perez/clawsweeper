## Summary

Keep this PR open. The reported current-main bug is real and the PR is aimed at the right surface, but the latest diff leaves a section-local clear action wired to grounded-only cleanup after changing that section to render daily-log entries.

## What This Changes

The PR changes the Dreaming Advanced tab to count and render `dailyCount > 0` entries for the From Daily Log summary/section, keeps replay metadata visible, adjusts origin badge logic, and adds view coverage for daily-only and grounded-only entries.

## Best Possible Solution

Land a revised PR that uses `dailyCount` for the Daily Log summary/list while making grounded replay cleanup a separate visible action or removing/retargeting the section-local clear control so each control affects the entries it appears to manage.

## Reproduction Assessment

Yes. A `DreamingProps` fixture with `dailyCount > 0` and `groundedCount === 0` reproduces current main's empty/misleading Daily Log section because the view filters by `groundedCount`; the PR's added test models that path.

## Solution Assessment

No, not as-is. Switching the summary and staged section to `dailyCount` is the narrow maintainable fix for the reported bug, but the section's existing clear action still targets grounded-only entries and should be separated, removed, or retargeted before merge.
