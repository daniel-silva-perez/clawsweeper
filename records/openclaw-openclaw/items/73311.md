## Summary

Keep this PR open. The feature boundary is useful and matches the linked Control UI issue, but the patch introduces new user-facing filter labels as hardcoded English strings, regressing the existing localized cron-filter surface and conflicting with the Control UI i18n workflow.

## What This Changes

The PR replaces the chat-header cron-only session hide toggle with a UI-local multi-kind session filter, adds a session-kind classifier and desktop/mobile menu UI, updates focused tests, styles the menu, and documents the new Control UI filtering surface.

## Best Possible Solution

Keep the UI-local filtering approach, but route all new filter title, aria label, group label, and kind label copy through the Control UI locale source and generated bundles before merge. After that, maintainers can decide whether to address the lower-priority details-dropdown UX comments in this PR or a follow-up.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: switch Control UI to a non-English locale and the new filter menu would still render inline English strings from the PR branch, while the current cron toggle uses localized chat keys.

## Solution Assessment

No, not as submitted. The UI-local session filtering boundary is a reasonable narrow solution while the gateway schema lacks a stable kind taxonomy, but the implementation should preserve localization by adding i18n keys and regenerated locale outputs.
