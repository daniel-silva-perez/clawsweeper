## Summary

Keep open. Current main still has the Telegram observability gap: text delivery logs successful `sendMessage` calls, but the media delivery branches do not emit success logs for successful media sends. PR #68836 is already open and linked as the fixing implementation, so this issue should remain open until that PR is reviewed and merged or replaced.

## What This Changes

The issue requests Telegram outbound media success log lines for voice, audio, document, video, photo, and animation sends, parallel to the existing text `sendMessage` success logs.

## Best Possible Solution

Review and land PR #68836 if it cleanly adds centralized media success logging after Telegram returns a message id, or replace it with an equivalent narrow patch plus regression coverage in the Telegram delivery tests. The issue should close only after that implementation is merged.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main has explicit success logging for text sends, while the media branches complete successful sends and mark delivery without any media success log line; the targeted `rg` search confirms the missing strings.

## Solution Assessment

Yes. Adding a success log immediately after each successful media send result is resolved to a Telegram message id is the narrow maintainable fix, ideally with one helper or shared point so all media kinds stay consistent.
