## Summary

Keep this issue open. Current main still routes `corpus="sessions"` into the builtin memory manager, but the hybrid keyword leg still converts `searchKeyword` failures to empty results and the FTS helper has no general MATCH-error fallback or diagnostic path for the reported English and common CJK terms. The report is concrete, current, and narrow enough for a focused memory-core fix PR.

## Best Possible Solution

Fix the builtin `memory-core` keyword path for session searches: capture/log FTS MATCH errors in a privacy-preserving way, add a bounded fallback that preserves model and source filters for current-tokenizer terms, and add regression tests for `corpus="sessions"` with English and CJK terms plus a thrown-MATCH path. Add a cheap deep-status FTS query probe only if it can avoid expensive scans and sensitive output.
