## Summary

Current main still lacks the Slack duplicate-reaction behavior that this PR is meant to add. The shared Slack helper still forwards `reactions.add` errors directly, the agent `react` action depends on that helper before returning success, and the only existing `already_reacted` suppression remains localized to the status-reaction adapter. The PR remains an active implementation candidate for linked issue #69005, with maintainer discussion already asking for exactly the helper-level shape now shown in the PR diff.

## Best Possible Solution

Keep this PR open for maintainer review and land it, or land an equivalent Slack plugin patch that treats `already_reacted` from `reactSlackMessage()` as idempotent success, preserves propagation for unrelated Slack errors, keeps helper-level regression coverage, and then closes linked issue #69005 while superseding the older overlapping Slack reaction PRs.
