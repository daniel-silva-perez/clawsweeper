## Summary

Keep this issue open. Current main has partial active-registry reuse for plugin tools, but the reported provider-auth path can still derive manifest-backed candidate/evidence maps during repeated tool/model auth checks, and the issue has concrete production CPU-profile evidence plus an independent macOS data point.

## What This Changes

The issue requests a fix for gateway event-loop stalls caused by repeated synchronous plugin registry and provider-auth environment resolution during tool creation on OpenClaw 2026.4.27.

## Best Possible Solution

Fix the hot path with scoped reuse: tool construction should reuse the active gateway/plugin registry where freshness allows, and provider-auth candidate/evidence maps should be computed once per stable config/workspace/environment snapshot and threaded through repeated model/tool checks without adding an unbounded stale global metadata cache.

Yes. The issue provides a high-confidence production reproduction path and CPU-profile fingerprint against 2026.4.27, and source inspection on current main confirms the same repeated lookup shape remains possible; this review did not run a live gateway load replay.

## Solution Assessment
