## Summary

Keep open. Current main fixes the original nine missing root-package runtime deps via the merged #74213 work, but the later issue discussion adds a fresh runtime-deps mirror failure involving a missing dist chunk after update. Current main still prepares the shared dist mirror in place and memoizes mirror preparation with only shallow existence checks, so the broader partial-mirror integrity concern is not proven fixed.

## What This Changes

The issue reports that the packaged runtime-deps mirror in 2026.4.26 omits root-package bare imports from dist, with later discussion adding a partial dist mirror extraction/update failure.

## Best Possible Solution

Keep the #74213 mirror allowlist and drift guard, then either retitle/scope this issue or open a narrower canonical tracker for runtime-deps dist mirror integrity. The desired end state is a mirror refresh path that validates imported dist chunks before plugin load, repairs missing chunks deterministically, and avoids exposing partially refreshed top-level dist trees.
