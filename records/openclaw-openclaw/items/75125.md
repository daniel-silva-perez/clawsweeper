## Summary

Keep open. This PR is aimed at the right security boundary and improves the inline skill tool-dispatch path, but the proposed call to `createOpenClawCodingTools(...)` does not pass a resolved sandbox context, so sandbox-specific tool policy and sandboxed execution still diverge from normal agent turns. Because this is an authorization hardening PR with a blocking review finding, it needs a maintainer/author update before merge.

## What This Changes

This PR replaces inline skill tool-dispatch's raw `createOpenClawTools(...)` path with `createOpenClawCodingTools(...)`, forwards session/group/sender/model context, updates focused regression tests, and removes the old `openclaw-tools.runtime.ts` shim.

## Best Possible Solution

Revise the inline tool-dispatch path to resolve or receive the same effective runtime tool context used by normal agent turns, including sandbox state and sandbox tool policy, then add focused regression coverage for restricted profiles, leaf subagents, and sandbox allow/deny policy before merging.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a sandboxed session whose restrictions rely on `tools.sandbox.tools`, plus a user-invocable `command-dispatch: tool` skill targeting a tool denied only by sandbox policy; the PR's inline call does not pass `sandbox`, so that policy layer is not applied.

## Solution Assessment

No. Reusing `createOpenClawCodingTools(...)` is the right direction, but this patch is not the complete policy-parity fix until the inline path supplies the resolved sandbox/run context or a shared effective-tool resolver that includes it.
