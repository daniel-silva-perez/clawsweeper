## Summary

Keep this PR open. Current main still lacks the active reply-run guard in the heartbeat runner, while the reply-run registry already exposes exactly the active session state this patch needs; the linked bug remains open and the earlier identical PR was closed unmerged by automation.

## What This Changes

The PR adds a heartbeat-runner guard that returns the existing `requests-in-flight` skip while a reply run is active for the resolved session, plus a regression test for an active reply operation with an idle session lane.

## Best Possible Solution

Refresh this branch or land an equivalent narrow patch on current main: after heartbeat preflight resolves the canonical `sessionKey`, consult `resolveActiveReplyRunSessionId(sessionKey)` before the session-lane queue-size check and return the existing retryable `requests-in-flight` skip, with the active-reply regression test preserved alongside the existing busy-lane coverage.
