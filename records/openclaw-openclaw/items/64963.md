## Summary

Keep this PR open. Current main still lacks the active reply-run guard that this PR adds: `runHeartbeatOnce` only checks the global main lane and then the resolved session lane size, while `resolveActiveReplyRunSessionId` is available in the reply-run registry but is not used by the heartbeat runner. The linked bug remains open, and the prior identical PR was closed unmerged by automation rather than superseded by a shipped fix.

## Best Possible Solution

Review and land this PR or an equivalent narrowly scoped fix: add an active reply-run guard in `src/infra/heartbeat-runner.ts` after heartbeat preflight resolves `sessionKey` and before the existing resolved session-lane queue check, returning the existing `requests-in-flight` skip so `heartbeat-wake` retries. Keep the regression test that creates an active `ReplyOperation` while the queue reports idle, and keep #64810 open until a fix merges.
