## Summary

Keep this PR open. Current main still uses the 30s TUI streaming watchdog and still emits the user-visible `streaming watchdog` system message, so the PR's intended behavior is not implemented or superseded. The related merged orphan-final fix preserved watchdog reset mechanics but did not address this timeout/message product change.

## Best Possible Solution

Keep the PR open for maintainer product review or revision. The current TUI still has the reported 30s warning behavior, so this is a live implementation candidate; the best path is to either merge a maintainer-approved longer default/softer message or fold the work into the configurable watchdog direction tracked by #68596, with the watchdog tests updated accordingly.
