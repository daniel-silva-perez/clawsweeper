## Summary

Keep open. The PR has the protected `maintainer` label, and latest head still has a blocking Gateway runtime-registry bug: the deferred startup plugin swap refreshes HTTP routes but leaves the pinned channel registry on the pre-runtime registry, so post-bind startup channel plugins can remain invisible to registry-backed outbound lookup.

## What This Changes

The PR changes CLI plugin/channel inspection policy, explicit bundled runtime-dependency repair, Gateway post-bind startup plugin runtime staging/readiness, docs/changelog, and regression tests for bundled runtime deps.

## Best Possible Solution

Refresh both the HTTP route registry and channel registry when deferred startup plugin runtime replaces the attached plugin runtime, or defer channel pinning until the loaded startup registry is available, with focused regression coverage for post-bind startup channel lookup.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: under the PR, create Gateway runtime state before startup plugin load, then load a startup channel plugin post-bind and call a registry-backed outbound/channel lookup; the lookup still reads the pinned pre-runtime channel registry.

## Solution Assessment

No. The PR direction is close, but it is not the best mergeable solution until the deferred runtime replacement preserves the channel registry surface as well as HTTP routes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Repin the channel registry after deferred plugin load:** `src/gateway/server.impl.ts:1002`
  - body: The branch creates Gateway runtime state before startup plugins load, which pins the channel registry to the pre-runtime registry. The post-bind replacement only repins HTTP routes, while channel/outbound loaders read `getActivePluginChannelRegistry()`, so a startup channel loaded after bind can remain invisible for outbound lookup. Repin the channel registry alongside the HTTP registry or defer the initial channel pin until startup plugins are loaded.
