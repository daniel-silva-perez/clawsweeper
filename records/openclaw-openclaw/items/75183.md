## Summary

Keep open. The PR has the protected `maintainer` label and the latest head still has a blocking correctness regression: the new no-repair policy applies to every `channels` command path that preloads plugins, including operational channel paths that still need bundled runtime-dependency repair on fresh or pruned packaged installs.

## What This Changes

The branch changes CLI/channel/plugin inspection and Gateway startup loading so selected read-only paths avoid implicit bundled runtime-dependency repair, adds explicit repair/runtime-inspect controls, memoizes prepared bundled runtime roots, and updates docs, tests, and changelog.

## Best Possible Solution

Keep the PR open, fix the channel policy so only static/read-only channel surfaces avoid bundled runtime-dependency repair, preserve automatic repair for operational channel commands and Gateway startup, then rerun the targeted CLI/channel/Gateway tests plus the normal changed gate under maintainer review.

## Reproduction Assessment

Yes. A high-confidence reproduction path is a packaged install with a configured bundled channel plugin whose runtime deps are missing or pruned, then running a channel command that still preloads plugins; source inspection shows the PR head would pass `installBundledRuntimeDeps: false` before the command action and fail during runtime registry load.

## Solution Assessment

No, not yet. The intended boundary is maintainable, but the current implementation uses `commandPath[0] === "channels"` as too broad a switch; a safer fix is an explicit read-only allowlist or exact command policies that avoid runtime loading where possible and keep repair enabled where runtime loading is required.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Limit no-repair mode to read-only channel paths:** `src/cli/plugin-registry-loader.ts:26-30`

Scope this no-repair policy to static/read-only surfaces or opt the affected commands out of runtime loading.
