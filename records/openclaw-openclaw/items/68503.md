## Summary

Keep open. Current main has not superseded the PR’s intended status-noise cleanup, but the proposed patch has concrete blocking issues in its new test and log summarization behavior, so it should be revised before merge.

## What This Changes

The PR adds `status --all` gateway log-tail filtering/grouping, stops `readLastGatewayErrorLine` from falling back to arbitrary last log lines, and adds focused tests for those paths.

## Best Possible Solution

Revise the PR so it preserves actionable timestamped gateway/auth failures, fixes the diagnostics test mock to match the actual restart-log dependency, makes grouped lane-wait text accurate, and includes the expected user-facing changelog entry before normal maintainer review.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: the added diagnostics test mocks a module that `diagnostics.ts` does not import, and `summarizeLogTail` would drop a timestamp-prefixed `Embedded agent failed before reply: OAuth token refresh failed...` line before reaching its parser.

## Solution Assessment

No. The direction is useful, but this patch is not the best mergeable form until it parses or exempts actionable timestamped lines before blanket filtering, fixes the test dependency mock, and either updates grouped bases on repeat or changes the lane-wait label away from `last`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Mock the restart-log helper used by diagnostics:** `src/daemon/diagnostics.test.ts:17-19`
  - body: `readLastGatewayErrorLine` imports `resolveGatewayLogPaths` from `./restart-logs.js`, but this test mocks `./launchd.js`, so the hoisted paths are never used. With the real helper the default stderr file is `gateway.err.log`, the `path.includes("stderr")` branch does not run, and the new curated-error test resolves `null` instead of the expected line.
  - confidence: 0.94
- **[P2] Preserve timestamped embedded auth failures:** `src/commands/status-all/gateway.ts:75-77`
  - body: This blanket skip runs before the embedded OAuth parser, so a timestamped line like `2026-... Embedded agent failed before reply: OAuth token refresh failed for openai-codex: ...` is discarded because the stripped content does not start with `[`. Current main would at least show that raw line in the gateway tail, so this hides an actionable auth-refresh outage.
  - confidence: 0.9
- **[P2] Make the lane-wait label match the grouped value:** `src/commands/status-all/gateway.ts:181-184`
  - body: `addGroup` increments the count but keeps the first `base` string, while this new summary labels the duration as `last`. If waits progress from 5s to 45s, the output still says `last 5s ×2`; update the base when a newer occurrence arrives or label it as the first observed wait.
  - confidence: 0.88
