## Summary

Keep open. Current main still blocks the sandbox env propagation this PR is trying to add, and the PR is not obsolete or already implemented. The branch is directionally tied to real reports, but it changes secret handling across Docker sandbox creation and sandbox exec env, and the supplied review thread still has a security-sensitive mismatch that needs maintainer review before merge.

## What This Changes

The PR adds skill-derived sensitive-env allowlisting through sandbox creation, mirrors injected skill env values into sandbox exec env, and adds sanitizer, skills, sandbox, and embedded-runner regression tests.

## Best Possible Solution

Revise the branch so sandbox-create allowlisting and sandbox exec env syncing come from one intentional eligible, active, and configured skill source, or explicitly keep skills.entries env/apiKey host-only and limit the fix to sandbox.docker.env. Update the sandboxed-skills docs to match the chosen contract, then run targeted sanitizer, skills, sandbox, embedded-runner tests plus the changed-lane gate before merge.
