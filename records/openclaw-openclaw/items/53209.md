## Summary

Keep open. Current main still lacks the PR's sandbox skill-env allowlist and sandbox exec-env sync behavior, while the PR changes sandbox secret propagation and has a remaining security-sensitive review concern that needs maintainer/security judgment before merge.

## What This Changes

The PR adds skill-derived sensitive-env allowlisting to Docker sandbox creation, mirrors injected skill env values into sandbox exec env, and adds sanitizer, skills, sandbox, and embedded-runner regression tests.

## Best Possible Solution

A maintainer/security reviewer should choose the intended sandbox skill-env contract, then revise the branch so container-create allowlisting and exec-env syncing come from one eligible, active, configured skill source; alternatively, keep `skills.entries.*` host-only and limit the fix to `sandbox.docker.env`. The chosen behavior should be documented in the sandboxed-skills docs before merge.
