## Summary

Keep open. Current main still lacks the skill-declared sandbox env allowlist, so this PR is addressing a real remaining gap, but the proposed patch changes sandbox secret propagation and still has blocking security-sensitive contract issues around inactive skills and per-agent skill filters.

## What This Changes

The PR adds skill-derived sensitive-env allowlisting to Docker sandbox creation, mirrors injected skill env keys into sandbox exec env, and adds sanitizer, sandbox, skill-env, and embedded-runner regression tests.

## Best Possible Solution

Rebase the branch on current main, then make sandbox-create allowlisting and sandbox exec-env syncing derive from one active, eligible, configured skill source that preserves per-agent skill filters; alternatively keep skills.entries.* host-only and limit the fix to sandbox.docker.env. Update sandboxed-skills docs to match the chosen contract before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restrict synced env keys to eligible skills:** `src/agents/pi-embedded-runner/run/attempt.ts:1717-1724`
  - body: On the non-snapshot path this applies env overrides to the loaded skillEntries and then syncs every injected key into sandbox exec env. Because that override source is not constrained to the same eligible skill set as the sandbox-create allowlist, a disabled or otherwise ineligible configured skill can still inject a secret and have it passed to sandbox execs. Filter the override/sync source to active eligible skills in both run and compaction.
  - confidence: 0.89
- **[P1] Preserve per-agent skill filters in sandbox plumbing:** `src/agents/pi-embedded-runner/run/attempt.ts:1685-1689`
  - body: Current main passes the resolved agent id into skill entry resolution and prompt building so agents.list[].skills is enforced. This branch's new helper and edited call sites omit that agent id, so a multi-agent setup can collect, prompt, or sync skills outside the active agent's allowlist. Thread the resolved agent id through the allowlist helper, skill entry resolution, and prompt resolution in both run and compaction.
  - confidence: 0.84
