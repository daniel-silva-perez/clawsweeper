## Summary

Keep open. Current main still lacks the skill-derived sandbox env allowlist and sandbox exec env sync that this PR is trying to add, so the PR is not obsolete. The PR also is not close-ready because the latest review context identifies a security-sensitive gap: the non-snapshot path can still apply configured env overrides from raw, not eligibility-filtered, skill entries before syncing those keys into sandbox exec env.

## Best Possible Solution

Keep this PR open for maintainer/reviewer follow-up. The best path is to revise the branch so sandbox create allowlists and sandbox exec env sync are derived from the same eligible, active skill set or from a clearly defined configured-override set; then update the sandboxed skills docs if `skills.entries.*.env/apiKey` should enter sandbox execs, and rerun the targeted sanitizer, sandbox create args, skills runtime, and changed-lane validation.
