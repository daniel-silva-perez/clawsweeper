## Summary

Keep open. Current main still has skills metadata, status, and `openclaw skills` list/info/check surfaces, but it does not define an inspectable skills capability manifest or an `openclaw skills inspect` command. The related open threads, especially #12219 and #10827, are relevant prior art rather than a completed or clearly canonical replacement for this narrower inspect-only RFC, and the public permission vocabulary needs maintainer product/security review before implementation.

## What This Changes

The issue proposes an optional skills capability manifest plus `openclaw skills inspect` to render tools/files/exec declarations without changing runtime behavior.

## Best Possible Solution

Maintain this as an open design/RFC item until maintainers decide the skills capability vocabulary and its relationship to #12219/#10827. If accepted, implement a typed schema, parser/status propagation, CLI rendering, public docs, and focused tests with explicit inspect-only wording.

## Reproduction Assessment

Not applicable as a feature/RFC rather than a runtime bug. Source and docs inspection on current main gives high-confidence verification that the requested schema and command are absent.

## Solution Assessment

Unclear. The inspect-only v0 is a plausible narrow first step, but the best maintainable path depends on maintainer agreement about the public permission vocabulary and whether this should be split from or folded into the broader open security/manifest threads.
