## Summary

Keep open. Current main still has skill load-time metadata, local status rendering, and `openclaw skills` list/info/check flows, but it does not define an inspectable skill capability manifest or an `openclaw skills inspect` command. Related #12219 and #10827 are useful prior art, but this narrower inspect-only slice remains a maintainer product/security decision rather than a cleanup close.

## What This Changes

The issue proposes an optional skills-first capability manifest for tools, files, and exec plus an `openclaw skills inspect <skill-id>` command that renders declarations without changing runtime behavior.

## Best Possible Solution

Make an explicit maintainer product/security decision on the skill capability vocabulary and its relationship to #12219/#10827. If accepted, implement a typed schema, parser/status propagation, CLI rendering, public docs, and targeted tests with clear inspect-only wording.

## Reproduction Assessment

Not applicable as a feature/RFC rather than a runtime bug. Source and docs inspection on current main gives high-confidence verification that the requested schema and command are absent.

## Solution Assessment

Unclear. The proposed v0 is a plausible narrow first step, but the best maintainable path depends on maintainer agreement about the public permission vocabulary and whether it should be split from or folded into the broader open security/manifest threads.
