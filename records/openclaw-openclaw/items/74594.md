## Summary

Keep open. Current main still has skill metadata, local skill status, and `openclaw skills info/list/check`, but no structured skill capability manifest or `skills inspect` command. The RFC is coherent and security-adjacent, so it needs maintainer product/security review rather than automated cleanup or a narrow fix job.

## What This Changes

The issue proposes adding an optional skill capability manifest and an `openclaw skills inspect` command that displays declared tool, file, and exec access without changing runtime behavior.

## Best Possible Solution

Make an explicit maintainer decision on the public skill capability vocabulary. If accepted, add a typed schema through the skill metadata/parser/status path, render it via a CLI command and docs, and include clear non-enforcement wording; if not, consolidate the discussion into #10827 as the canonical security design tracker.
