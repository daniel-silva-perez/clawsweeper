## Summary

Keep open. Current main has skill load-time metadata, local skill status rendering, and `openclaw skills` list/info/check commands, but it does not define an inspectable skill capability manifest or an `openclaw skills inspect` command. The RFC is coherent and security-adjacent, so it needs maintainer product/security review rather than cleanup closure.

## What This Changes

The issue proposes an inspect-only v0 skill capability manifest plus an `openclaw skills inspect <skill-id>` command for declared tools, file access, and exec expectations without changing runtime behavior.

## Best Possible Solution

Make an explicit maintainer decision on the public skill capability vocabulary and its relationship to #10827; if accepted, add a typed schema, parser/status propagation, CLI rendering, docs, and targeted tests with clear inspect-only wording.
