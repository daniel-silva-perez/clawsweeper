## Summary

Keep open. Current main still has skill load-time metadata, status rendering, and `openclaw skills` list/info/check commands, but it does not define a skills capability manifest or an `openclaw skills inspect` command. The request is coherent and security-adjacent, with clear overlap against still-open #12219 and security-labeled #10827, so it needs maintainer product/security review rather than cleanup closure.

## What This Changes

The issue proposes a skills-first optional capability manifest and `openclaw skills inspect <skill-id>` command that displays tools/files/exec declarations without changing runtime behavior.

## Best Possible Solution

Make an explicit maintainer decision on the public skill capability vocabulary and its relationship to #12219 and #10827; if accepted, add a typed schema, parser/status propagation, CLI rendering, docs, and targeted tests with clear inspect-only wording.
