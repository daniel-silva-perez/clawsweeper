## Summary

Keep open. Current main still flattens built-in session transcript hits to `corpus: "memory"`, while this PR adds the narrow conditional mapper and regression coverage needed for the linked bug; the remaining action is maintainer refresh/review rather than cleanup closure.

## What This Changes

The PR changes memory-core's `memory_search` result mapper to emit `corpus: "sessions"` for results whose `source` is `"sessions"` and adds a regression test for that path.

## Best Possible Solution

## Solution Assessment
