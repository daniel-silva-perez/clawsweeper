## Summary

Keep this PR open. Current main at 6d323ee73640afdf02ed9fa3aded13da23359b98 still does not implement the OpenClawBrain package alias: discovery still derives `openclaw` from `@openclawbrain/openclaw`, install still emits the manifest/package mismatch info for `openclawbrain`, and there is no source or regression-test alias for `openclawbrain` on main. The PR is a focused candidate, though the reviewer note about centralizing the alias map is still a real maintainability concern.

## Best Possible Solution

Keep this PR open and review or land it after adjusting the implementation to use one source of truth for package-id aliases. The shipped fix should map `@openclawbrain/openclaw` to `openclawbrain` in discovery fallback, suppress the install mismatch info only for that exact alias pair, and keep targeted discovery/install regression tests.
