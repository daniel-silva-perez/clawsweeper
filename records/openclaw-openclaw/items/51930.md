## Summary

Keep this PR open. Current main still does not implement the OpenClawBrain install identity normalization: there is no `openclawbrain` or `@openclawbrain/openclaw` alias in source/tests/docs, discovery still falls back from a scoped package name to the unscoped package segment, and install still emits the manifest/package mismatch info whenever `packageNameMatchesId` is false.

## Best Possible Solution

Keep this PR open and review/land it, or an equivalent current-main patch, after adjusting the implementation to use a single source of truth for package-id aliases. The shipped behavior should map `@openclawbrain/openclaw` to `openclawbrain` in discovery fallback, suppress the known install mismatch info for that exact alias pair, and keep targeted discovery/install regression tests.
