## Summary

Keep this PR open. Current main still does not implement the OpenClawBrain install identity normalization: discovery has no full-package alias lookup for `@openclawbrain/openclaw`, install still treats manifest/package equivalence as exact-or-unscoped only, and there is no OpenClawBrain alias entry or regression coverage on main.

## Best Possible Solution

Review and land this PR or an equivalent current-main patch that centralizes plugin package-id aliases, normalizes `@openclawbrain/openclaw` to `openclawbrain` during discovery when no valid manifest id is available, suppresses the known install mismatch info for that canonical alias pair, and keeps targeted discovery/install regression coverage.
