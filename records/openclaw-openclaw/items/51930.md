## Summary

Keep this PR open. Current main still does not implement the OpenClawBrain install identity normalization: there is no `openclawbrain` alias on main, discovery still derives `openclaw` from `@openclawbrain/openclaw` when no valid manifest id is available, and install still treats that manifest/package pair as a mismatch. The PR remains a plausible core plugin install/discovery fix, though the final patch should avoid duplicating alias tables.

## Best Possible Solution

Review and land this PR or an equivalent current-main patch that centralizes plugin package-id aliases, maps `@openclawbrain/openclaw` to `openclawbrain` during discovery fallback, suppresses the known install mismatch info for that exact canonical alias pair, and keeps targeted discovery/install regression tests.
