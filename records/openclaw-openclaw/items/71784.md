## Summary

Keep open. Current main still routes live builtin memory search through a single query embedding request with timeout only and no bounded retry for the reported transient fetch/undici/TLS socket failures. The related open PR #44167 is useful but only covers batch indexing retry/splitting for #44166, not the live query embedding path used by memory search.

## What This Changes

The issue asks for live memory search to tolerate transient remote embedding transport failures such as `fetch failed`, `other side closed`, `ECONNRESET`, `EPIPE`, and TLS socket disconnects without failing the user-visible search.

## Best Possible Solution

Keep this issue open and land a focused memory transport resilience fix for the live query path: classify narrow transient fetch/undici/socket/TLS errors, apply bounded retry/backoff around the single-query embedding call used by builtin memory search and active-memory recall, preserve non-retryable auth/config failures, and coordinate with #44167 so batch and query retry semantics share one policy where possible.

## Repair Work Prompt

Fix live builtin memory search retry handling for transient remote embedding transport failures. Current main still has `MemoryManager.search()` calling `embedQueryWithTimeout(cleaned)`, and `embedQueryWithTimeout()` only applies a timeout around `provider.embedQuery(text)`. Add a narrow transient transport-error classifier for `fetch failed`, `other side closed`, `ECONNRESET`, `EPIPE`, `UND_ERR_SOCKET`, socket hangups, TLS socket disconnects, and similar connection-class failures, then apply bounded retry/backoff to the live query embedding path used by `openclaw memory search` and active-memory recall. Preserve fail-fast behavior for auth/config/status/validation errors. Coordinate with #44167/#44166: if the batch retry PR has landed, reuse its classifier/helpers; if it has not, keep the implementation small and shaped so the batch PR can converge without duplicate policies. Do not add provider-specific branching, dispatcher keep-alive tuning, or a new config knob in this PR unless strictly required. Add focused tests covering query retry success after one transient failure, non-retry of auth/config errors, and classifier coverage for the reported error strings. Add a single-line changelog fix entry under the active version if runtime behavior changes, crediting the reporters/commenters with `Thanks @kevinheinrichs, @heithmurray, and @kaufmanchang`.
