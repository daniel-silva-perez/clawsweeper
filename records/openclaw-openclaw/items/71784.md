## Summary

Keep open. Current main still leaves the live memory-search query embedding path as a single remote provider call wrapped only by a timeout, while the existing retry helper is applied to batch embedding paths. The issue discussion includes concrete repro loops and later provider-diverse confirmations, and the related open batch-retry PR does not close this live-query surface.

## What This Changes

The issue asks OpenClaw to make live memory search resilient to intermittent remote embedding transport failures such as `fetch failed`, `other side closed`, `ECONNRESET`, `EPIPE`, and TLS socket disconnects.

## Best Possible Solution

Add a narrow transient transport-error classifier and bounded retry/backoff for live query embeddings, preferably by reusing the existing memory embedding retry helper. Preserve immediate failures for auth, config, schema, and non-retryable HTTP errors; treat dispatcher keep-alive tuning or a new user config knob as follow-up only if focused tests or live proof show the scoped retry is insufficient.

## Repair Work Prompt

Fix live memory search remote embedding transport flakiness for https://github.com/openclaw/openclaw/issues/71784. Current main retries batch embedding but `embedQueryWithTimeout()` calls `provider.embedQuery(text)` only once through the timeout wrapper, so `openclaw memory search` can fail on transient `fetch failed`, `other side closed`, `ECONNRESET`, `EPIPE`, `UND_ERR_SOCKET`, TLS socket disconnect, or similar connection-class failures. Implement a narrow retry/backoff for live query embeddings, preferably by reusing `runMemoryEmbeddingRetryLoop` and extending the retry classifier only for transient transport errors. Preserve immediate failures for auth, config, schema, and non-retryable HTTP errors; do not add provider-specific branches, config knobs, broad Undici pool tuning, QMD lock/hang changes, or unrelated remote HTTP rewrites. Add focused tests for live query retry and classifier coverage, and add a user-facing `CHANGELOG.md` Fixes entry crediting @kevinheinrichs plus follow-up reporters if the maintainer wants those credits.
