## Summary

Keep open. Current main still has no Xiaomi `webSearch` config surface and no Xiaomi provider stream wrapper that appends the native MiMo `web_search` tool; the reporter's follow-up also adds useful live API evidence rather than making the request obsolete.

## What This Changes

The issue requests opt-in Xiaomi MiMo native `web_search` support by appending MiMo's provider-native search tool to Xiaomi chat-completions requests.

## Best Possible Solution

Add a provider-owned Xiaomi implementation in the bundled Xiaomi plugin: declare `plugins.entries.xiaomi.config.webSearch`, normalize bounded search options, append the native `web_search` tool only for eligible Xiaomi OpenAI-compatible chat requests when enabled, preserve existing function tools, avoid duplicate native entries, and document Xiaomi console activation, supported models, key/endpoint expectations, and billing behavior.

## Reproduction Assessment

Not applicable as a bug reproduction, but the absence check is high-confidence: current main lacks a Xiaomi `webSearch` schema, runtime payload wrapper, tests, and docs. The reporter's working API example provides a concrete live target for validation once credentials are available.

## Solution Assessment

Yes. A narrow Xiaomi plugin change using the existing `wrapStreamFn` payload-patch hook is the most maintainable path; core changes should only be added if implementation proves the current provider hook cannot safely express this native-tool behavior.
