## Summary

Keep this issue open. Current main has partial noise reduction for WebChat/Control UI tool output: tool rows and inline tool cards are collapsed by default, and there is a persisted show/hide toggle. It does not implement the requested single per-turn summary like "5 tool calls" or a detailed-vs-summary presentation mode; the item builder still emits each tool message separately and the renderer gives each tool message/card its own disclosure button.

## Best Possible Solution

Add the feature in the Control UI/WebChat chat rendering path: introduce a persisted presentation setting such as detailed vs summary, aggregate contiguous tool messages for one agent turn into a single counted tool-summary chat item, and expand that summary into the existing per-tool cards/details. Keep the current show/hide toggle and add focused tests for multiple tool calls collapsing into one summary row, expansion behavior, and setting persistence.
