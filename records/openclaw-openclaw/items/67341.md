## Summary

Keep this PR open. Current main still lacks the requested `amazon-bedrock-mantle` `config.discovery.enabled` opt-out, the linked bug remains open, and this PR is a focused implementation candidate rather than obsolete work. It should be adapted before merge because the current sibling provider pattern resolves plugin config from the runtime `ctx.config` snapshot and tests the resolver-level gate, while this PR only gates the catalog handler from startup `api.pluginConfig`.

## Best Possible Solution

Keep this PR open as an implementation candidate for #67288. Update it against current main to add Mantle manifest schema/uiHints and public docs, resolve current plugin config from `ctx.config`, pass config into `resolveImplicitMantleProvider`, gate before IAM bearer-token generation, and add focused regression tests for `discovery.enabled: false`, `true`, and unset/default behavior. Then land this PR or an equivalent replacement.
