## Summary

Keep this PR open. Current main still lacks the requested `amazon-bedrock-mantle` `config.discovery.enabled` opt-out, and the PR is a focused implementation candidate for the open linked bug, but it needs adaptation to current provider config patterns plus tests/docs before merge.

## Best Possible Solution

Keep this PR open as an implementation candidate for #67288. Update it against current main to add Mantle manifest schema/uiHints and docs, resolve current plugin config from `ctx.config` with the established provider pattern, pass config into `resolveImplicitMantleProvider`, gate before IAM bearer-token generation, and add focused regression tests for disabled, enabled, and default discovery behavior. Then land this PR or an equivalent replacement.
