## Summary

Keep this issue open. Current main still only recognizes camelCase tool-call block types in the session transcript repair path, so the reported snake_case `tool_use` / `tool_call` case can still bypass repair. The discussion and open linked PR #50802 identify a plausible minimal fix, but that PR is still open and uses closing syntax, so the conservative maintainer cleanup path is to leave this issue open until the implementation PR is reviewed, merged, or explicitly superseded.

## Best Possible Solution

Review PR #50802 against current main, or replace it with a focused maintainer patch. The fix should update raw tool-call recognition consistently in `src/agents/session-transcript-repair.ts`, related helper/type sets such as `src/agents/tool-call-id.ts`, and regression tests for `tool_use`, `tool_call`, and `function_call` blocks missing input. Once the PR is merged or explicitly superseded, close this issue as implemented with release provenance.
