## Summary

Keep open. Current main still recognizes only camelCase tool-call block types in the session transcript repair path, the latest release tag has the same predicate, and open PR #50802 remains an unmerged implementation candidate linked to this issue.

## Best Possible Solution

Review and either land PR #50802 or replace it with a focused maintainer patch. The fix should centralize or consistently update tool-call block recognition for `tool_use`, `tool_call`, and `function_call` across session repair, tool-call ID extraction, Anthropic turn validation, and embedded attempt normalization, then add regression tests for malformed snake_case blocks missing `input` or `arguments`. After merge, close as implemented with the fixing SHA and release provenance.
