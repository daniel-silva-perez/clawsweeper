## What This Changes

Request to make the bundled memory-lancedb plugin handle or document failures when its LanceDB data path is a Windows-host Docker bind mount.

## Best Possible Solution

Keep this open and add a narrow memory-lancedb follow-up: document that Windows Docker Desktop users should store LanceDB data on a Docker named volume, add a targeted diagnostic for LanceDB metadata/read/permission failures on local dbPath storage, and only add bounded initialization retry behavior if tests show it is safe for partially-created tables.
