## Summary

Keep open. Current main still leaves heartbeat final reply payloads on the `isHeartbeat` pass-through path before channel delivery, and the shared bracket-block sanitizer still does not cover the reported same-line/non-JSON `[TOOL_CALL]...[/TOOL_CALL]` and `[TOOL_RESULT]...[/TOOL_RESULT]` shapes. The issue also has open closing-syntax PR #54165, so it should remain open until that PR or an equivalent focused fix lands.

## What This Changes

The issue requests that heartbeat reply delivery strip raw bracket-format `[TOOL_CALL]` and `[TOOL_RESULT]` blocks before Telegram, Discord, or other user-facing channel messages are sent.

## Best Possible Solution

Keep this issue open and land a focused fix in `src/auto-reply/reply/agent-runner-payloads.ts` that strips uppercase bracket tool-call/result blocks from heartbeat reply text before final channel delivery, with regression coverage in `src/auto-reply/reply/agent-runner-payloads.test.ts`. Track the broader interactive/XML leak separately if it still reproduces after the heartbeat fix.
