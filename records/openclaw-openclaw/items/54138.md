Current main still does not provide a targeted heartbeat bracket-block fix, and an open same-author PR with closing syntax is still the implementation candidate. The issue should remain open until PR #54165 or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #54165, or land an equivalent current-main fix in `src/auto-reply/reply/agent-runner-payloads.ts` with focused coverage in `src/auto-reply/reply/agent-runner-payloads.test.ts`. The fix should strip uppercase `[TOOL_CALL]...[/TOOL_CALL]` and `[TOOL_RESULT]...[/TOOL_RESULT]` blocks from heartbeat reply payload text before channel delivery, including same-line and non-JSON payload variants. The broader interactive/XML leak from later comments should be tracked separately if still reproducible.
