## Summary

Keep open. Current main still redacts agent-visible session-history text and exposes only `contentRedacted`; it does not include the requested stable machine-readable marker, `notice`, or redacted-fields metadata. The related PR #68483 is still an open, unmerged implementation candidate with closing syntax, so this issue should remain open until that PR or an equivalent fix is reviewed and landed.

## Best Possible Solution

Keep this issue open and review PR #68483, or an equivalent narrow patch, for landing. The fix should add a stable machine-readable marker or notice to agent-visible redacted tool output, preserve existing redaction semantics, and add regression coverage for both the shared redaction helper and `sessions_history` behavior.
