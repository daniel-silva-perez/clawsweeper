## Summary

Keep this issue open. Current main partially mitigates the reported HEARTBEAT.md/subagent failure by filtering subagent and cron bootstrap files to a fixed allowlist and by offering an opt-in lightContext flag for individual subagent spawns. The broader request is still not implemented: there is no schema-backed bootstrap exclude/session-type filter, no section-level bootstrap filtering, no bootstrap !command preprocessing, and the stable system prompt intentionally exposes timezone only rather than current clock time.

## Best Possible Solution

Keep this issue open and narrow it into a core bootstrap-context design: add a documented, schema-backed per-session bootstrap filter surface; decide whether dynamic bootstrap content should be core !command preprocessing or a documented hook/plugin pattern; and align any time-context addition with the existing prompt-cache policy, likely via deterministic runtime/message metadata or session_status rather than volatile stable prompt text.
