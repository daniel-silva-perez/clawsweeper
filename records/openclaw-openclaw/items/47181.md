## Summary

Keep this PR open. Current main still lacks a first-class Azure Claude onboarding path, but this branch is stale against the current provider-onboarding/plugin-auth architecture and still has concrete validation/routing bugs that should block merge.

## What This Changes

The PR adds a core `anthropic-azure-api-key` onboarding choice with CLI flags, env vars, Azure Claude URL/resource normalization, auth-profile/config wiring, Anthropic docs, and focused onboarding/unit tests.

## Best Possible Solution

Land a rebased, provider-owned implementation through the current plugin auth-choice and provider setup hooks. It should expose manifest CLI/env metadata, implement interactive and non-interactive setup in the owning provider surface, strictly validate `.services.ai.azure.com` resource labels and `/anthropic` paths, store auth-profile metadata safely, update provider docs, and cover URL/resource/env/generic-token/SecretRef cases.

## Reproduction Assessment

Yes. The current diff gives high-confidence static reproductions: malformed full-URL paths are accepted and rewritten, invalid URL resource labels bypass bare-resource validation, and the apiKey token-provider route ignores generic `--token` input.

## Solution Assessment

No. The current PR is useful source work, but a hard-coded core auth branch is not the narrowest maintainable solution now that provider plugins support manifest auth choices and setup hooks; the branch also needs the validation and generic-token fixes before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reject unsupported Azure endpoint paths:** `src/commands/anthropic-azure-utils.ts:59-60`
  - body: `normalizeAnthropicAzureBaseUrl` appends `/anthropic` to any full URL path that does not already end in that suffix. Inputs like `https://fabric-hub.services.ai.azure.com/anthropic/v1` are stored as `.../anthropic/v1/anthropic`, so onboarding succeeds with a malformed runtime base URL instead of failing fast.
  - confidence: 0.92
- **[P2] Validate Azure resource labels in URL input:** `src/commands/anthropic-azure-utils.ts:50-53`
  - body: The full-URL branch only checks that the hostname ends with `.services.ai.azure.com`. A value such as `https://fabric_hub.services.ai.azure.com/anthropic` bypasses the bare-resource validation and persists an invalid Azure resource host, pushing the failure to runtime.
  - confidence: 0.9
- **[P2] Honor generic token input for apiKey routing:** `src/commands/auth-choice.apply.api-providers.ts:177`
  - body: `--auth-choice apiKey --token-provider anthropic-azure --token ...` normalizes into this Azure branch, but the branch passes only `anthropicAzureApiKey` to the API-key helper. Existing scripted API-key flows that use the generic `--token` will be ignored and fall back to env or prompts.
  - confidence: 0.86
