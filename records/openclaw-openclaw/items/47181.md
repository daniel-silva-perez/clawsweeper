## Summary

Keep this PR open. Current main still lacks a first-class Azure Claude onboarding path, but the proposed branch has blocking correctness issues and is stale against the current manifest/plugin-owned provider auth design, so it needs maintainer/provider review rather than cleanup closure.

## What This Changes

The PR adds a core `anthropic-azure-api-key` onboarding choice with CLI flags, env-var handling, Azure Claude base URL normalization, auth profile/config wiring, docs, and focused onboarding tests.

## Best Possible Solution

Land a rebased, provider-owned implementation through the current plugin auth-choice and provider setup hooks. It should expose manifest CLI/env metadata, implement interactive and non-interactive setup in the owning provider surface, strictly validate `.services.ai.azure.com` resource labels and `/anthropic` paths, store auth-profile metadata safely, update provider docs, and cover URL/resource/env/generic-token/SecretRef cases.

## Reproduction Assessment

Yes. The current PR diff gives high-confidence static reproductions: malformed full-URL paths are accepted and rewritten, invalid URL resource labels bypass bare-resource validation, and the `apiKey` token-provider route ignores generic `--token` input.

## Solution Assessment

No. The PR is useful source work, but a core hard-coded auth branch is not the narrowest maintainable solution now that provider plugins support manifest auth choices and setup hooks; the branch also needs the validation and generic-token fixes before merge.

  - body: `normalizeAnthropicAzureBaseUrl` appends `/anthropic` to any full URL path that does not already end in that suffix.   - confidence: 0.92

  - confidence: 0.9
- **[P2] Honor generic token input for apiKey routing:** `src/commands/auth-choice.apply.api-providers.ts:177`
  - body: `--auth-choice apiKey --token-provider anthropic-azure --token ...` normalizes into this Azure branch, but the branch passes only `anthropicAzureApiKey` to the API-key helper. Existing scripted API-key flows that use generic `--token` will be ignored and fall back to env or prompts.
  - confidence: 0.86
