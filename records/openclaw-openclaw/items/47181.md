## Summary

Keep open. Current main does not provide the exact Azure Claude onboarding choice, so this PR is not obsolete, but the branch is not mergeable as-is: main now has manifest/plugin-owned provider auth seams, the Microsoft Foundry provider is OpenAI-focused, and the proposed core hard-coded Azure Claude path still has concrete URL-validation and generic-token routing bugs.

## What This Changes

The PR adds a core `anthropic-azure-api-key` onboarding choice with CLI flags/env-var handling, Azure Claude URL normalization, auth-profile/config wiring, docs, and focused onboarding tests.

## Best Possible Solution

Add Azure Claude through the current provider-plugin setup surface, either by extending `extensions/microsoft-foundry` to support Anthropic deployments or by adding a separate provider plugin. The implementation should publish manifest auth/CLI metadata, support interactive and non-interactive setup, validate `.services.ai.azure.com` host labels and `/anthropic` paths strictly, preserve generic `apiKey`/SecretRef flows, update provider docs, and cover resource, URL, env, and ref-mode cases.

## Reproduction Assessment

Yes. The PR diff gives high-confidence static reproductions: malformed full URLs such as `https://fabric-hub.services.ai.azure.com/anthropic/v1`, invalid URL hosts such as `fabric_hub.services.ai.azure.com`, and `--auth-choice apiKey --token-provider anthropic-azure --token ...` all exercise paths called out in the changed code and review comments.

## Solution Assessment

No. The requested user problem is valid, but the proposed hard-coded core auth choice is not the narrowest maintainable solution now that current main has provider-owned manifest auth choices and plugin setup hooks; a provider-owned implementation with the validation/token fixes is safer.

  - body: `normalizeAnthropicAzureBaseUrl` appends `/anthropic` to any full URL path that does not already end in that suffix.   - confidence: 0.92

  - confidence: 0.9

  - body: `--auth-choice apiKey --token-provider anthropic-azure --token ...` normalizes into this Azure branch, but the branch passes only `anthropicAzureApiKey` to the API-key helper. Existing scripted API-key flows that use generic `--token` will be ignored and fall back to env or prompts.
  - confidence: 0.86
