## Summary

Keep this PR open. Current main still does not implement WhatsApp `errorPolicy` / `errorCooldownMs`; the config schema, resolved account type, docs, and WhatsApp auto-reply runtime have no matching support. The PR is also paired with the still-open same-author feature issue #59782, so it remains the active implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open as the concrete implementation path for #59782. The best next step is maintainer review of the remaining once-policy delivery semantics, then landing the WhatsApp config/schema/runtime/tests/docs together if that behavior is accepted.
