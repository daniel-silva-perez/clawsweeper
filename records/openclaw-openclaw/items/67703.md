## Summary

Current main still does not solve the reported `gateway.auth.mode="none"` internal RPC probe failure. The loopback probe still sends no device identity when no token/password is available, `auth.mode="none"` only clears the shared-secret auth check, and non-Control-UI operator clients without shared auth still fall into the `device identity required` 1008 close path. The related #69066 RFC is relevant, but it remains an open design item rather than a shipped fix.

## Best Possible Solution

Keep this issue open. The best fix is a narrowly scoped internal-service identity path, or a tightly constrained loopback-only internal caller exception, for Gateway-owned probe/followup/backend RPC clients under `gateway.auth.mode="none"`. Add regression coverage for `probeGateway` with no token/password/device identity and the affected exec-approval followup RPC path, while preserving fail-closed behavior for remote, browser-origin, node-role, and non-internal clients. Align the durable design with #69066.
