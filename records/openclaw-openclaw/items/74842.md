## Summary

Keep open. Current main still makes Discord guild/channel turns private-by-default unless visible replies are explicitly restored, and I did not find a Discord doctor warning, migration, or upgrade diagnostic for existing configs that lack the new visible-reply setting.

## What This Changes

The issue asks OpenClaw to preserve visible Discord guild/channel final replies after upgrade, or clearly warn/migrate when `messages.groupChat.visibleReplies` is absent.

## Best Possible Solution

Add a Discord-owned doctor preview warning, and optionally a repair hint if maintainers want it, for configured Discord guild/channel usage with no explicit visible-reply choice. Keep the shipped private-by-default runtime policy unless maintainers explicitly choose a broader product change.

## Reproduction Assessment

Yes. A high-confidence reproduction path is visible in source and tests: a Discord guild message is treated as `ChatType: "channel"`, absent visible-reply config resolves to `message_tool_only`, and setting `messages.groupChat.visibleReplies` to `automatic` restores visible final replies.

## Solution Assessment

Yes for the narrow doctor diagnostic; no for reverting the default globally because #73046 intentionally changed group/channel rooms to private-by-default. A Discord-local warning is the smallest maintainable fix for the upgrade confusion.
