## Summary

Keep open. Current main still intentionally defaults Discord guild/channel turns with no explicit visible-reply setting to message-tool-only delivery, and the docs/changelog explain the `automatic` opt-in, but I did not find the requested upgrade diagnostic, migration, or doctor warning that would prevent the silent-reply confusion reported here.

## What This Changes

The issue requests that upgraded Discord guild/channel configurations continue posting visible final replies, get migrated, or receive a clear `openclaw doctor` warning when `messages.groupChat.visibleReplies` is absent.

## Best Possible Solution

Add a Discord-owned doctor preview warning, and possibly a repair hint if maintainers want it, for configured Discord guild/channel usage with no explicit visible-reply choice. Keep the shipped private-by-default runtime policy unless a maintainer explicitly chooses a broader product change.

## Reproduction Assessment

Yes. A high-confidence reproduction path is visible in source and tests: a Discord guild message is treated as `ChatType: "channel"`, absent visible-reply config resolves to `message_tool_only`, and setting `messages.groupChat.visibleReplies` to `automatic` restores visible final replies.

## Solution Assessment

Yes for the narrow doctor diagnostic; no for reverting the default globally because #73046 intentionally changed group/channel rooms to private-by-default. A Discord-local warning is the smallest maintainable fix for upgrade confusion.
