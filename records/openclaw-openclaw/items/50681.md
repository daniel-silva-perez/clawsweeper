## Summary

Current main still does not implement the requested Telegram unknown-group warn/auto-leave behavior. Telegram group allowlist enforcement fails closed by dropping/logging unknown groups before assistant execution, but there is no `unknownGroupAction`, warning message, cooldown, or `leaveChat` path in config, schema, docs, or runtime handling.

## Best Possible Solution

Keep this open for maintainer-approved Telegram plugin work. The likely implementation belongs in the existing Telegram inbound preflight path for `group-chat-not-allowed`: add opt-in `unknownGroupAction: "ignore" | "warn" | "leave"` plus optional message and cooldown fields, key cooldowns by account/chat, send a configured warning before optional `leaveChat`, and update schema/types/UI hints/docs plus targeted Telegram group-access/handler tests.
