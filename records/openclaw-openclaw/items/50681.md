## Summary

Keep open. Current main still only denies unknown Telegram groups by dropping/logging normal messages or sending a static native-command denial, with no config schema, type, docs, or runtime support for configurable ignore/warn/leave behavior; open PR #50682 remains the paired implementation path.

## What This Changes

The issue requests first-class Telegram unknown-group actions (`ignore`, `warn`, `leave`) with optional warning text and cooldown before assistant execution.

## Best Possible Solution

Review and land PR #50682 or a narrow successor that implements unknown-group actions in Telegram preflight, keeps behavior consistent across normal messages and native commands, updates public config schema/types/generated metadata/docs, and validates cooldown plus forum-topic delivery.
