## Summary

Keep open. Current main at 2d53b49b20e19790afbba4f4cbb2b78042bb3530 still fail-closes unknown Telegram groups by dropping or statically rejecting them, but it does not implement the requested configurable ignore/warn/leave behavior; same-author PR #50682 remains the paired implementation path.

## What This Changes

Request support for Telegram unknown-group preflight handling that can ignore, warn, or warn then auto-leave before assistant execution.

## Best Possible Solution

Land PR #50682 or a narrow successor that implements the unknown-group action in the Telegram preflight path, updates public config types/schema/generated metadata/docs, shares behavior across normal messages and native commands, and validates cooldown plus forum-topic delivery before closing this issue.
