## Summary

Keep open. Current main still lacks the requested Telegram unknown-group action surface: unknown groups are denied by the existing allowlist path, but there is no configurable ignore, warning message, cooldown, or auto-leave behavior. The same-author PR #50682 is still the paired implementation path, so this should stay open pending maintainer review of that PR.

## What This Changes

The issue requests configurable Telegram unknown-group handling with ignore, warn, cooldown, and optional auto-leave behavior before assistant execution.

## Best Possible Solution

Land PR #50682 or a narrow successor that implements the behavior in the bundled Telegram plugin preflight, updates public config types/schema/generated metadata/docs, shares the action path across normal messages and native commands, preserves forum-topic warning semantics, and validates cooldown plus approved-group behavior before closing this issue.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to configure Telegram with an explicit groups allowlist, send from an unlisted group, and observe current main returning group-chat-not-allowed/logging not-allowed without any configurable warning or leave action; the same gap is visible in source and existing tests.

## Solution Assessment

Yes, Telegram inbound preflight is the right boundary for this behavior because it runs before assistant execution. The maintainable version should distinguish unknown chats from sender-authorization failures and reuse one action helper for normal messages and native commands.
