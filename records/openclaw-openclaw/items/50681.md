## Summary

Keep this issue open. Current main still fails closed by dropping/logging unknown Telegram groups, but it does not implement configurable `unknownGroupAction`, `unknownGroupMessage`, cooldowns, or `leaveChat`; the same author also has open PR #50682 as the paired implementation candidate, so this issue should stay open until that PR or a successor lands.

## What This Changes

Request Telegram config and runtime support for fail-closed unknown-group handling with optional warning, cooldown, and auto-leave before assistant execution.

## Best Possible Solution

Keep this issue open and resolve it through PR #50682 or a narrow successor. The accepted implementation should live in the Telegram plugin preflight path, share behavior across normal messages and native commands, precisely distinguish unknown groups from other allowlist failures, use thread-safe Telegram warning sends before optional `leaveChat`, bound cooldown state, validate or diagnose `warn` without a message, and update config types/schema/generated metadata/docs/tests before closing this issue.

## Risks / Open Questions

- Auto-leave is access-control-sensitive: a misclassified allowed group, wildcard group, or General-topic forum thread could make the bot leave a valid chat.
- Warning messages can leak bot presence in arbitrary groups or spam if cooldown state is not account/chat-scoped and bounded.
- The change extends the public Telegram config contract, so schema, generated metadata, docs, UI/config hints, and tests need to stay aligned.
- The paired PR needs maintainer review because existing comments call out native command bypass risk, precise unknown-vs-configured classification, cooldown pruning, and validation/diagnostics for `warn` without a message.
