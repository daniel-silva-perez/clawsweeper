## Summary

Keep open. Current main still fail-closes unknown Telegram groups by dropping/logging normal messages or sending a static native-command denial, but it does not implement the requested configurable ignore/warn/leave behavior; the same-author PR #50682 remains the paired implementation path.

## What This Changes

The issue requests Telegram unknown-group preflight handling so unapproved groups can be ignored, warned, or warned and left before assistant execution.

## Best Possible Solution

Land PR #50682 or a narrow successor that implements the unknown-group action in Telegram preflight, updates public config types/schema/generated metadata/docs, shares behavior across normal messages and native commands, and validates cooldown plus forum-topic delivery before this issue is closed.
