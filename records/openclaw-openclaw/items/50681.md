## Summary

Keep this issue open. Current main already drops non-allowlisted Telegram groups before assistant execution, but it does not implement the requested first-class unknown-group action surface: no `unknownGroupAction`, warning message/cooldown config, or automatic `leaveChat` behavior exists in Telegram config, docs, tests, or runtime. The supplied related items are adjacent Telegram feature reports, not a canonical duplicate for this request.

## Best Possible Solution

Implement this in the bundled Telegram plugin rather than closing it: add typed Telegram config/schema/metadata/docs for the unknown-group action, message, and cooldown semantics; handle `group-chat-not-allowed` in the existing inbound preflight before assistant execution; add focused tests for ignore, warn with cooldown, and leave behavior. Keep core generic unless another channel needs the same API.
