## Summary

Keep this issue open. Current main still fail-closes unknown Telegram groups by rejecting/logging them, but it does not provide configurable `unknownGroupAction`, warning text, cooldown handling, or `leaveChat`; open PR #50682 remains the paired implementation candidate from the same author.

## What This Changes

Request support for Telegram unknown-group preflight handling that can ignore, warn, or warn then auto-leave before assistant execution.

## Best Possible Solution

Land PR #50682 or a narrow successor that implements the unknown-group action in the Telegram preflight path, updates schema/types/generated config/docs, shares behavior across normal messages and native commands, and validates cooldown and forum-topic delivery before closing this issue.
