## Summary

Keep this open. Current main still only blocks unknown Telegram groups by dropping/logging normal messages or sending a static native-command denial; it does not provide the requested configurable ignore, warn, cooldown, or auto-leave behavior, and open PR #50682 remains the paired implementation path.

## What This Changes

The issue requests configurable Telegram unknown-group handling so unapproved groups can be ignored, warned, or auto-left before assistant execution.

## Best Possible Solution

Land PR #50682 or a narrow successor that implements the unknown-group action in Telegram preflight, updates config types/schema/generated metadata/docs, shares behavior across normal messages and native commands, and validates cooldown plus forum-topic delivery before this issue is closed.
