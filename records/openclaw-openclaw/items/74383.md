## Summary

Keep open. Current main still omits the reported broad Windows SIDs/principals from world classification, and this PR is a focused implementation candidate for an open security issue; it should receive secops/code-owner review and targeted validation rather than be closed or replaced automatically.

## What This Changes

The PR expands Windows ACL world-equivalent principal/SID lists and adds classifier plus filesystem-audit regression tests for broad Windows principals such as Anonymous Logon, Guests, Interactive, Local, and Network.

## Best Possible Solution

Review and land this PR or an equivalent focused security patch after secops confirms the SID policy, preserving trusted SYSTEM/Administrators/current-user handling and keeping both classifier and filesystem-audit regression coverage. Add a changelog entry if maintainers treat the severity change as user-visible.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: on current main, an ACL entry such as `*S-1-5-7:(F)` misses `WORLD_SIDS`, returns `group`, and therefore drives `groupWritable` rather than `worldWritable`.

## Solution Assessment

Yes, pending security-owner policy sign-off. The proposed direction is the narrow maintainable fix because it updates the classifier allowlists and tests both the classification seam and the resulting filesystem-audit severity without changing POSIX permissions or ACL parsing.
