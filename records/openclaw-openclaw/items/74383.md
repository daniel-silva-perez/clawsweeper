The PR expands Windows ACL world-equivalent principal/SID classification and adds classifier plus filesystem-audit regression tests for Anonymous Logon, Guests, Interactive, Local, and Network.

## Best Possible Solution

Land a focused src/security fix, after secops review, that keeps the Windows ACL classifier authoritative for broad well-known principals, preserves SYSTEM/current-user trusted handling, updates the stale guard comment, and locks the behavior with the proposed classifier and filesystem-audit regression coverage.

Yes. ## Solution Assessment

Mostly yes. Expanding the classifier-owned SID/principal sets with unit and filesystem-audit regressions is the narrow maintainable fix, but the final policy on Interactive, Local, and Network should be approved by the security owner and the stale guard comment should be refreshed.
