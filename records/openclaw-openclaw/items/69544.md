## Summary

Keep open. Current main still derives fallback mention regexes from `identity.emoji`, Matrix still feeds those shared regexes into room mention gating, and there is no Matrix, shared group-chat, or per-agent emoji-only opt-out. The adjacent scoped mention-policy PR #70864 remains open but does not cover identity-derived emoji fallback control, while the focused fix PR #74211 was closed unmerged.

## What This Changes

The issue requests a configurable opt-out so Matrix/group-chat fallback mention matching does not treat an agent’s identity emoji as an implicit mention trigger.

## Best Possible Solution

Add a backward-compatible shared group-chat setting with global and per-agent precedence that defaults to current behavior but lets users exclude only the `identity.emoji`-derived fallback regex. Native Matrix mentions, name-derived fallback matching, and explicit `groupChat.mentionPatterns` should remain unchanged, with schema/types/help/docs and targeted tests updated together.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure an agent with `identity.emoji`, omit explicit `groupChat.mentionPatterns`, call `buildMentionRegexes` for that agent, and Matrix `resolveMentions` will mark any message containing that emoji as mentioned through the regex fallback path.

## Solution Assessment

No for the Matrix-only switch in the title; the fallback derivation is shared, so a shared global/per-agent control is the narrower maintainable fix. #70864’s scoped pattern policy is adjacent but does not currently solve identity-derived emoji fallback control.
