## Summary

Keep open. Current main still derives fallback mention regexes from `identity.emoji`, Matrix still feeds those shared regexes into room mention gating, and no Matrix-level or shared group-chat emoji-only opt-out exists. The adjacent scoped mention-policy PR #70864 is open but the issue discussion correctly notes it does not currently solve the emoji-derived fallback; the focused fix PR #74211 was closed unmerged, so this remains live work.

## What This Changes

The issue requests a configuration switch to let users disable agent `identity.emoji` as an automatic fallback mention trigger in Matrix/group-chat mention detection.

## Best Possible Solution

Add a backward-compatible shared group-chat setting, with global and per-agent precedence, that defaults to preserving current emoji-derived fallback matching but lets users exclude only `identity.emoji` from derived regexes. Native Matrix mentions, name-derived fallback matching, and explicit `groupChat.mentionPatterns` should keep working, with schema/types/help/docs and targeted tests updated together.

## Reproduction Assessment

Yes. A static current-main reproduction is clear: configure an agent with `identity.emoji`, omit `groupChat.mentionPatterns`, call `buildMentionRegexes` for that agent, and Matrix `resolveMentions` will mark any message containing that emoji as mentioned through the fallback regex path.

## Solution Assessment

Unclear for the exact proposed surface. A shared global/per-agent toggle is the narrowest maintainable direction because emoji fallback derivation is shared, but maintainers should decide whether it belongs in #70864 or in a focused follow-up and choose the final public config name.
