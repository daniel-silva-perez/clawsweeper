## Summary

Keep open. Current main still derives fallback mention regexes from `identity.emoji`, Matrix still feeds those shared regexes into room mention gating, and neither shared group-chat config nor Matrix config has an emoji-only opt-out. The focused related PR #74211 is closed unmerged, while the broader #70864 remains open and unmerged, so the requested behavior is not implemented on main.

## What This Changes

Add a Matrix or per-agent configuration switch so an agent's identity emoji does not automatically act as a mention trigger in Matrix group rooms.

## Best Possible Solution

Keep this open and land a narrow backward-compatible emoji mention opt-out, preferably in the shared group-chat mention derivation path with per-agent override support unless maintainers explicitly choose a Matrix-only channel setting. The implementation should default to current behavior, preserve native Matrix mentions and explicit `mentionPatterns`, update config schema/types/generated help/docs, and add targeted tests for the Matrix gating path and shared mention helper.
