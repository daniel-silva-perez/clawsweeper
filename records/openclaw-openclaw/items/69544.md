## Summary

Keep open. Current main still derives fallback mention regexes from `identity.emoji`, Matrix still feeds those shared regexes into room mention gating, and neither Matrix config nor shared group-chat config has an emoji-only opt-out. The related scoped mention-policy PR #70864 is open but the discussion says it does not currently cover identity-derived emoji fallback control, while the focused PR #74211 was closed unmerged.

## What This Changes

The issue requests a configuration switch to let users disable agent `identity.emoji` as an automatic fallback mention trigger in Matrix/group-chat mention detection.

## Best Possible Solution

Add a backward-compatible shared group-chat setting with global and per-agent precedence that defaults to current behavior but lets users exclude only `identity.emoji` from identity-derived fallback regexes. Native Matrix mentions, name-derived fallback matching, and explicit `groupChat.mentionPatterns` should remain unchanged, with schema/types/help/docs and targeted tests updated together.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure an agent with `identity.emoji`, omit explicit `groupChat.mentionPatterns`, call `buildMentionRegexes` for that agent, and Matrix `resolveMentions` will mark any message containing that emoji as mentioned through the regex fallback path.

## Solution Assessment

The Matrix-only switch proposed in the title is probably not the best final shape because the fallback derivation is shared. A shared global/per-agent control over identity-derived mention patterns is the narrower maintainable solution, but maintainers should choose the final public config name and whether to coordinate it with #70864.
