## Summary

Keep open. Current main still derives default mention regexes from both `identity.name` and `identity.emoji`, Matrix consumes those regexes for room mention gating, and neither Matrix config nor shared per-agent group-chat config has the requested emoji-specific opt-out. There is an open overlapping PR, #70864, for scoped mention-pattern policy, but it is unmerged and broader than an emoji-only Matrix/per-agent switch.

## Best Possible Solution

Keep this issue open. The best path is for maintainers to decide whether #70864 should be adjusted to cover emoji-only identity-derived mention control, or whether a focused follow-up should add a backward-compatible opt-out that preserves native Matrix mentions and name-derived mention behavior while excluding `identity.emoji` from derived regexes when disabled.

## Risks / Open Questions

- The requested behavior is narrower than the open scoped mention-pattern policy PR: users may want to keep name-derived mentions while disabling only emoji-derived mentions.
- The shared mention helper is consumed by many channels, so a generic default change could regress Discord, Slack, Telegram, WhatsApp, Signal, iMessage, and other group/channel mention behavior unless the new control is explicitly scoped and backward-compatible.
- Adding public config fields requires aligned types, strict zod schemas, generated schema/help metadata, docs, tests, and a changelog entry; otherwise config may validate inconsistently across CLI, docs, and plugin runtime surfaces.
- A new fix PR would likely conflict with or duplicate #70864 unless maintainers first decide whether that broader PR should absorb the emoji-specific requirement.
