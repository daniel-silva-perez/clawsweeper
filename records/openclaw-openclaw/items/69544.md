## Summary

Keep open. Current main still derives fallback mention regexes from both `identity.name` and `identity.emoji`, Matrix feeds those shared regexes into room mention gating, and neither Matrix config nor shared per-agent group-chat config has an emoji-only opt-out. The related scoped mention-policy PR #70864 is open and unmerged, so this needs maintainer follow-up rather than cleanup closure.

## Best Possible Solution

Keep this issue open. Maintainers should decide whether #70864 should be adapted to explicitly cover identity-derived emoji mention control, or whether a focused backward-compatible follow-up should add an opt-out that preserves native Matrix mentions and name-derived matching while excluding `identity.emoji` from derived regexes when disabled. The chosen path should update config schema/types, generated config help, Matrix/group-chat docs, and targeted mention-gating tests.
