## Summary

Keep open. Current main removed the direct Unix `xargs` pipeline from `format:docs:check`, but the replacement still sends every tracked docs temp path to a single `spawnSync` call. With 463 tracked docs files, that leaves the native Windows command-line-length concern unresolved, and the latest release still ships the old `xargs` command. An open related PR, #48887, remains the right place for maintainer reconciliation rather than closing this issue now.

## Best Possible Solution

Keep this issue open while maintainers reconcile the docs formatter path. The best outcome is to add chunking/native PowerShell proof to the current `scripts/format-docs.mjs` implementation or explicitly close #48887 as superseded with equivalent proof; after that, close this issue as implemented on main or in the first release that ships the fix. Keep #44291 as the broader native PowerShell smoke-coverage follow-up.
