## Summary

Keep open. Current main still finalizes an isolated cron run from the first yielded embedded-runner result: the embedded runner now exposes `meta.yielded`/paused liveness, but the cron outcome contract does not carry that state, and the timer clears `runningAtMs`, records `lastStatus: ok`, and computes `durationMs` as soon as `runIsolatedAgentJob` resolves.

## What This Changes

The issue requests that isolated cron jobs using `sessions_yield` not be reported as terminal `ok` with a short duration before descendant subagents and the resumed parent have actually completed.

## Best Possible Solution

Keep this issue open and have maintainers choose the cron lifecycle contract. The strongest product fix is to make yielded isolated cron runs nonterminal, preserving a running or explicit yielded/pending state until descendant subagents and the resumed parent reach a true terminal outcome, then emitting `finished` and computing `durationMs` from the full orchestration. If maintainers prefer a smaller step, first add yielded metadata to `CronRunOutcome` and run logs while tracking the remaining duration/finalization gap explicitly.
