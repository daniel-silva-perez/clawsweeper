## Summary

Keep open. Current main still intentionally blocks normal Gateway startup when the config is missing or lacks `gateway.mode=local`, while the no-provider WebUI crash and Raw mode complaint do not have enough concrete reproduction detail to pick a narrow fix.

## What This Changes

The issue asks for Gateway and WebUI startup to work without fully configured settings, providers, or models, and adds that Raw mode is currently not working.

## Best Possible Solution

Split this into concrete decisions: decide whether normal loopback Gateway startup should gain a safer first-run/configless mode or keep the explicit `gateway.mode` guard, then add focused WebUI regressions only for a reproduced no-provider crash or a specific Raw mode snapshot failure.

## Reproduction Assessment

Partial/unclear. There is a high-confidence static path for the Gateway startup guard via `src/cli/gateway-cli/run.option-collisions.test.ts`, but no high-confidence reproduction for the alleged no-provider WebUI crash or Raw mode failure from the issue data.

## Solution Assessment

Unclear as a single fix. The maintainable path is to preserve the existing safety model until maintainers choose a startup policy change, then handle WebUI no-provider and Raw mode as separate focused regressions if they reproduce.
