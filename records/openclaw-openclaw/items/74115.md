Keep open. Current main has several related safeguards, but it still intentionally blocks normal Gateway startup when `gateway.mode=local` is absent, and the no-provider WebUI crash plus Raw mode complaint are not concrete enough to close or hand directly to an automated fix PR.

## Best Possible Solution

Keep this open for maintainer triage and split it into concrete checks: decide whether normal loopback Gateway startup should gain a safer configless mode or keep the current `gateway.mode` guard; reproduce the no-provider WebUI crash with exact config/browser state before adding a regression; and ask for the exact Raw mode snapshot state before changing Control UI behavior.
