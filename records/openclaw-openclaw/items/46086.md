## Summary

Keep open. Current main still rejects `memorySearch.multimodal` for providers whose memory embedding adapter does not declare multimodal support, and the bundled `local` adapter remains text-only. The shipped multimodal path is scoped to Gemini Embedding 2, so restoring local multimodal behavior needs maintainer provider/product judgment rather than conservative cleanup.

## Best Possible Solution

Keep this open for maintainer judgment. The clean path is either to document and confirm that bundled local multimodal memory is unsupported, or to add a real local multimodal embedding adapter only if the local dependency stack can accept structured non-text inputs; any implementation should cover capability validation, indexing behavior, privacy-preserving fallback behavior, and docs.
