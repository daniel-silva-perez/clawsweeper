## Summary

Keep open. Current main still rejects `memorySearch.multimodal` when the configured memory embedding provider adapter does not declare multimodal support, and the bundled `local` adapter remains text-only. The shipped multimodal path is scoped to Gemini Embedding 2, so restoring local multimodal behavior needs maintainer product/provider judgment rather than conservative cleanup.

## Best Possible Solution

Keep this open for maintainer judgment. The clean path is either to document/confirm that bundled local multimodal memory is unsupported, or to add a real local multimodal embedding adapter only if the local dependency can accept structured non-text inputs, with tests covering provider capability validation, indexing behavior, privacy-preserving fallback behavior, and docs.

- The report does not include the exact 2026.2.12 local-provider configuration or local runtime/model that appeared to work, so maintainers may need that detail before deciding whether this was a regression or an intentionally narrowed capability.
