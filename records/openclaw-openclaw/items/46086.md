## Summary

Current main still matches the reported problem: enabling `memorySearch.multimodal` with an explicit local provider is rejected unless the configured embedding adapter advertises multimodal support. The only bundled adapter that currently advertises that support is Gemini for `gemini-embedding-2-preview`; docs and tests also describe/enforce the Gemini-only path. The adjacent related reports about Gemini provider registration/model resolution do not supersede the local-provider regression described here.

## Best Possible Solution

Keep this issue open. The maintainer path is to decide whether 2026.2.12 local multimodal behavior should be restored as a supported compatibility mode, or whether the Gemini-only requirement is intentional. A code fix would likely belong around `src/agents/memory-search.ts`, `extensions/memory-core/src/memory/provider-adapters.ts`, and the embedding/indexing path in `extensions/memory-core/src/memory/manager-embedding-ops.ts`, with tests covering `provider: "local"` plus `memorySearch.multimodal`. If Gemini-only is intentional, update the issue with that product decision and add migration guidance instead of auto-closing it as already fixed.
