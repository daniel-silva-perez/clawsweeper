## Summary

Keep this PR open. Current main still does not implement the Microsoft Teams Graph upload-session fallback: both OneDrive and SharePoint outbound upload helpers still perform single authenticated `/content` PUT requests, and current tests cover only the direct upload path. The PR remains a focused implementation candidate, but it needs a rebase onto current Teams header/preconnect expectations, the known stall-count fix, and maintainer review of the Graph upload-session contract.

## Best Possible Solution

Keep this PR open for Microsoft Teams maintainer review. Rebase it onto current main, add upload-session fallback inside the current OneDrive and SharePoint helpers, preserve current User-Agent/preconnect behavior, align the size threshold with the intended Graph/Teams behavior, fix the stall guard, and land focused large-file tests for both storage paths before merging.

## Risks / Open Questions

- The PR touches a security-sensitive network path: Graph upload-session URLs are preauthenticated, so review should ensure chunk PUTs omit `Authorization`, session URLs are not logged, and retries do not expose tokens or URLs.
- The PR predates current main’s `buildUserAgent()` and `withFetchPreconnect` test expectations, so a rebase must preserve current Microsoft Graph request-header and preconnect behavior.
- The PR has known functional review feedback: the stall-detection guard allows one more stalled response than `UPLOAD_SESSION_MAX_STALLS` implies.
