## Summary

Keep this PR open. Current main still lacks the Microsoft Teams Graph upload-session fallback for large OneDrive and SharePoint uploads; the active upload helpers still use single `/content` PUT requests, and current tests only cover the direct upload path. The PR remains a focused implementation candidate, but it needs maintainer review, a rebase onto current Teams request-header/test expectations, and the known stall-count fix before it should land.

## Best Possible Solution

Keep this PR open for Microsoft Teams maintainer review. Rebase it onto current main, add the upload-session fallback to the current OneDrive and SharePoint helpers, preserve current User-Agent/preconnect behavior, fix the stall guard to match the intended limit, and land focused large-file tests for both storage paths.
