## Summary

Keep this PR open. Current main still lacks the Microsoft Teams Graph upload-session fallback: OneDrive and SharePoint uploads both send one direct `/content` PUT with the full buffer, current tests only cover direct uploads, and the related earlier PRs #26668 and #26712 are closed unmerged. This remains the focused implementation candidate, with maintainer review still needed for the upload-session flow and the existing stall-count review comment.

## Best Possible Solution

Keep this PR open for Microsoft Teams maintainer review. Rebase it onto current main, add a shared upload-session fallback inside the current `uploadToOneDrive` and `uploadToSharePoint` helpers, preserve the small-file direct path plus current User-Agent/preconnect behavior, fix the stall-count guard, and land focused tests for session creation, chunk progression, retry/stall handling, and both OneDrive and SharePoint paths.
