## Summary

Keep this PR open. Current main still lacks the Microsoft Teams Graph upload-session fallback for large OneDrive and SharePoint uploads, while this PR remains the focused implementation candidate split from the broader closed work. It still needs maintainer review and a rebase because the proposed patch predates current User-Agent/preconnect behavior and has an already-flagged stall-count off-by-one.

## Best Possible Solution

Keep this PR open for Microsoft Teams maintainer review. Rebase it onto current main, add a shared Graph upload-session fallback inside the existing `uploadToOneDrive` and `uploadToSharePoint` helpers, preserve the small-file direct path plus current User-Agent/preconnect behavior, fix the stall-count guard, and land focused tests for session creation, chunk progression, retry/stall handling, and both OneDrive and SharePoint paths.
