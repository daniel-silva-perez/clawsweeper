## Summary

Keep this PR open. Current main still lacks the Microsoft Teams Graph upload-session fallback for large OneDrive/SharePoint uploads; the active upload helpers and tests only cover direct `/content` PUTs. The earlier related attempts #26668 and #26712 were closed unmerged, and the Microsoft tracker #49126 is also closed, so there is no merged or open canonical target-repo item that supersedes the remaining work.

## Best Possible Solution

Keep this PR open for Microsoft Teams maintainer review. Rebase it onto current main, preserve `buildUserAgent()` behavior on Graph calls, add the upload-session path to the current `graph-upload.ts`, fix the stall-count guard, and land focused OneDrive plus SharePoint large-file regression tests.

## Risks / Open Questions

- The PR needs a rebase onto current main because `graph-upload.ts` and `graph-upload.test.ts` now include OpenClaw User-Agent expectations and preconnect-aware test helpers that the PR branch does not preserve.
- The upload-session URL returned by Graph is preauthenticated; the implementation should continue avoiding `Authorization` on chunk PUTs and must not log or expose that URL.
- The stalled upload-session guard currently allows one more stalled response than `UPLOAD_SESSION_MAX_STALLS` implies; that should be fixed before landing.
