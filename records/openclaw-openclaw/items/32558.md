## Summary

Keep this PR open. Current main still does not implement the Microsoft Teams Graph upload-session fallback for large OneDrive or SharePoint uploads; the outbound Teams file paths still call helpers that perform single direct `/content` PUT uploads. The PR remains a focused implementation candidate split from broader closed work, but it needs maintainer review, a rebase onto current User-Agent/preconnect behavior, and the already-flagged stall-count fix before it is merge-ready.

## Best Possible Solution

Keep this PR open for Microsoft Teams maintainer review. Rebase it onto current main, add the upload-session fallback inside the existing `uploadToOneDrive` and `uploadToSharePoint` helpers, keep the small-file direct `/content` path and current User-Agent/preconnect behavior, fix the stall guard, and land focused tests for session creation, chunk progression, retry/stall handling, and both OneDrive and SharePoint paths.
