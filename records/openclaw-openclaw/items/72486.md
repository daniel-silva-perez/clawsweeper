## Summary

Keep this issue open. Current main still does not provide Vietnamese Control UI locale support: the Locale union, lazy registry, navigator fallback, generated locale assets, English language picker labels, i18n generator entries, docs, and shipped-locale tests all omit `vi`/`vi-VN`. This is a core Control UI localization task, not a ClawHub/plugin cleanup item.

## Best Possible Solution

Add Vietnamese as a first-class Control UI locale in core: extend the `Locale` type, lazy registry, `vi`/`vi-VN` browser fallback, English language label, `scripts/control-ui-i18n.ts` locale entry/pretty label, generated `vi` locale bundle and `.i18n` metadata/glossary, Control UI docs, and i18n registry/translation tests.
