## Summary

Keep this issue open. Current main still defaults Control UI and backend-created isolated agent-turn cron jobs to announce delivery, and the no-channel path still resolves through the same "Channel is required" failure unless best-effort delivery is explicitly enabled. The related discussion and closed PRs are useful context, but they do not supersede the remaining default no-channel cron regression.

## Best Possible Solution

Keep this issue open and fix the cron product path directly. A conservative maintainer fix would make Control UI-created local/webchat-only isolated cron jobs that do not have a deliverable channel use delivery.mode="none" by default, or downgrade unresolved implicit announce delivery non-fatally while preserving hard failures for explicit delivery targets. Add regression coverage for a Control UI-created isolated cron job in a no-channel config, backend-created isolated agentTurn defaults, and the public cron docs/workaround.
