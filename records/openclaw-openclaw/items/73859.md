## Summary

Keep open. The report is concrete, current main still has the same plugin load surface for the affected bundled plugins, and I found no source, launcher, docs, or package-acceptance evidence that default-stack Windows imports of the built minimax, google, and talk-voice entries now succeed.

## What This Changes

The issue requests a Windows fix for built-in minimax, google, and talk-voice plugin load failures caused by `RangeError: Maximum call stack size exceeded` during Gateway startup.

## Best Possible Solution

Keep the issue open and fix the built plugin import shape or runtime loading path so minimax, google, and talk-voice import successfully under the default Windows Node stack. The preferred end state is a shallow enough package build plus a Windows package smoke that imports those dist entries without requiring users to set `NODE_OPTIONS=--stack-size=65536`.

## Reproduction Assessment

Yes for the reported shipped package: install OpenClaw v2026.4.26 on Windows Server 2019 with Node v24.14.1 and require the affected `dist/extensions/*/index.js` entries under default stack, then compare with `--stack-size=65536`. Current-main reproduction is still unproven in this read-only Linux checkout.

## Solution Assessment

No current solution is proven. The existing loader diagnostics are useful containment, and the Windows native-require fast path is relevant, but the maintainable fix should make the packaged entries importable under default Windows stack settings and prove that in a Windows/package lane.
