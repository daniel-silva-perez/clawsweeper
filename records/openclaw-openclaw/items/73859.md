## Summary

Keep open. The issue is a new, concrete Windows/package report against v2026.4.26, and current main does not contain a clear implementation that fixes the reported default-stack import failure for minimax, google, or talk-voice. The loader already records import failures and continues, but that only limits blast radius; it does not make the affected bundled plugin capabilities load successfully when module evaluation overflows the stack.

## Best Possible Solution

Keep this issue open and implement a focused packaging/runtime fix: either make the affected bundled plugin entrypoints shallow enough to import under default Windows Node stack settings, or add a well-scoped Windows launcher/runtime mitigation. Add a Windows or package-acceptance smoke that imports/loads `dist/extensions/minimax/index.js`, `dist/extensions/google/index.js`, and `dist/extensions/talk-voice/index.js` with default Node stack, and keep the existing non-fatal plugin-load logging behavior as a secondary safety net.
