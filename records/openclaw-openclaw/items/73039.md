## Summary

Keep this PR open. Current main at ad57a6d61658f4e73b926138a3b010da5db4e225 does not implement the PR's intended memoization: `createOpenClawTools` still calls the media, PDF, and web factories directly on each invocation, and `src/agents/tools/memoize-tool-factory.ts` is absent. The PR is still an active implementation candidate, though it needs maintainer review for cache-key correctness before merge.

## Best Possible Solution

Keep this PR open and review it as the active implementation candidate. Before landing, tighten the memoization keys or scope so every factory-time and execute-time captured value that can vary across turns is represented, then add focused regression tests for runtime web metadata swaps, config reload/cache pruning, sandbox/fs policy changes, and video/music requester-origin delivery.
