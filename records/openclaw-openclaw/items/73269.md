## Summary

Keep open. Current main has a partial mitigation for group/channel rooms and a global direct-chat opt-in, but it does not implement the requested per-agent auto-reply disablement or a shared final-only/public-internal delivery boundary; the broader related audience-boundary report is also still open.

## What This Changes

Suppress automatic channel delivery of assistant text when tool calls follow, either with per-agent auto-reply disablement or by sending only final public assistant text.

## Best Possible Solution

Define one shared visible-delivery contract that can mark assistant text as public or internal and suppress automatic preview, block, partial, and final delivery unless the policy allows it, while preserving explicit message-tool sends. That contract should cover direct and group/channel conversations, Feishu streaming cards, generic dispatch, transcript extraction, docs, config schema/help, and regression tests.

## Reproduction Assessment

Partly. Current unit tests and source give a high-confidence reproduction of the residual policy gap: direct/source chats still default to automatic delivery while message-tool-only mode suppresses delivery; the exact Feishu text-then-tool-call transcript still needs a focused fixture or live check.

## Solution Assessment

No. The current messages.visibleReplies mechanism is a useful partial mitigation, but it is not the complete requested fix because it lacks per-agent/directive control and an explicit public/internal or final-only assistant text boundary.
