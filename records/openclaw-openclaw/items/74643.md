## Summary

Keep this PR open. The branch addresses the current-main schema gap and, after the latest update, fixes normal reply and directive-status precedence, but it still leaves the direct `openclaw agent --agent ...` path resolving verbose defaults from `agents.defaults` only, so the newly accepted per-agent `verboseDefault` is not effective across all command surfaces described by the PR.

## What This Changes

The PR adds per-agent `verboseDefault` and `elevatedDefault` config schema/type/generated metadata/docs coverage, adds parser and precedence tests, surfaces `elevatedDefault` in resolved agent config, and updates reply/directive-level resolution to prefer agent entries before global defaults.

## Best Possible Solution

Land one consolidated fix for #73680 that keeps the schema/types/docs/generated metadata aligned and uses one effective per-agent-before-global default path for normal replies, directive status/current-level reporting, status text, verbose progress, and direct `openclaw agent` CLI runs, with parser and runtime regression coverage plus the required changelog entry.

## Reproduction Assessment

Yes. The original bug is source-reproducible on current main from the strict AgentEntrySchema omission, and the remaining PR defect is source-reproducible by following `openclaw agent --agent <id>` preparation to `cfg.agents?.defaults` and `agentCfg?.verboseDefault` without consulting the resolved agent entry.

## Solution Assessment

No. The proposed schema, docs, generated metadata, and reply-directive changes are the right direction, but the complete fix should also thread `resolveAgentConfig` or an equivalent agent entry through the direct agent command path before claiming per-agent verbose defaults work consistently.
