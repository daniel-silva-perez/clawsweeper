## Summary

Keep this PR open. Current main still has the null-unsafe support-capture path, while the PR is narrowly scoped to defensive handling and regression coverage for partial skill snapshot entries.

## What This Changes

The PR widens support path redaction to tolerate nullish inputs, filters/falls back skill capture entries with missing names, and adds trajectory metadata regression tests.

## Best Possible Solution

Land or lightly refine the focused defensive consumer hardening in trajectory metadata and support redaction, with producer-side normalization in the skills loader left as a follow-up if maintainers want to remove the source of partial entries.

## Reproduction Assessment

Yes. A constructed `SkillSnapshot` with a partial `resolvedSkills` entry can reach current main's `buildSkillsCapture`, where undefined paths hit `file.startsWith` and undefined names can hit `localeCompare`.

## Solution Assessment

Yes. The PR's defensive consumer-side change is the narrowest maintainable fix for the support-capture crash and preserves the existing symlink escape rejection behavior; producer cleanup would be a separate hardening improvement.
