## Summary

Keep this PR open for maintainer review. Current main still has the brittle support-capture path, and the proposed patch is a narrow defensive fix with no blocking review findings from this read-only pass.

## What This Changes

The PR widens support path redaction to accept nullish file values, filters and falls back malformed skills snapshot entries in trajectory metadata, and adds regression tests.

## Best Possible Solution

Land or lightly refine the focused consumer hardening in trajectory metadata and support redaction, leaving producer-side skills-loader normalization as a separate follow-up if maintainers want to remove the source of partial entries.

## Reproduction Assessment

Yes. A constructed `SkillSnapshot` with a partial `resolvedSkills` entry can reach current main's `buildSkillsCapture`, where undefined paths hit `file.startsWith` and undefined names can hit `localeCompare`.

## Solution Assessment

Yes. The PR's consumer-side filtering and null handling are the narrowest maintainable crash fix and preserve the existing symlink escape rejection behavior; producer cleanup would be complementary hardening, not a prerequisite.
