## What This Changes

The PR branch adds Grok/xAI web_search parser null guards, timeout AbortError conversion, a 60s Grok default timeout, plus status-command lazy-loading and status model alias changes.

## Best Possible Solution

Yes. The parser crash is reproducible by passing null entries in output or content to extractXaiWebSearchContent on current main; #58733 also provides a current-main timeout-boundary reproduction where the same request fails around 30s and succeeds with 60s.

## Solution Assessment

No. The intended fix direction is sound, but this PR is not the best merge shape because it targets a removed/unloaded provider file and includes unrelated status-command edits; a narrow replacement against the live xAI runtime is safer.
