## Summary

Keep open. Current main still has no ZAI vision-model endpoint rewrite, while this PR is a narrow provider-local fix for the linked ZAI image 404 reports. Static review did not find a blocking correctness or security issue in the latest patch; the remaining work is maintainer review plus targeted CI and live ZAI validation.

## What This Changes

The PR wraps the bundled ZAI media-understanding provider so known ZAI vision models rewrite `/api/coding/paas/vN` base URLs to `/api/paas/vN`, and adds colocated unit tests for model detection and URL conversion.

## Best Possible Solution

Land a narrow ZAI-provider fix that routes only known ZAI vision models away from coding-plan base URLs while preserving text/coding models on the configured coding endpoint, with focused tests and a small changelog note if maintainers require one.

## Reproduction Assessment

Yes. Configure `agents.defaults.imageModel` as `zai/glm-4.6v` with a ZAI coding base URL and call the image tool; the linked reports show 404/abort failures, and current main source shows no provider-specific endpoint rewrite before generic image resolution.

## Solution Assessment

Yes from static review. A provider-local wrapper is the narrowest maintainable place for ZAI-specific endpoint routing, and the current resolver applies the patched `providerConfig.baseUrl` after model discovery; targeted CI/live validation should confirm it before merge.
