## Summary

Keep this PR open. The target-session `/status` change is focused and addresses a real current-main gap, but the added `ResolvedAgentConfig.contextTokens` field drops inherited `agents.defaults.contextTokens`, which makes the proposed resolver behavior incomplete before merge.

## What This Changes

This PR adds `contextTokens` to resolved agent config, routes `/status` with the target session entry's persisted context window, and adds regression tests for both paths.

## Best Possible Solution

Keep the PR focused, but update the resolver addition to return the effective context-token cap with `entry.contextTokens ?? agentDefaults?.contextTokens`, add an inherited-default regression test, and then rerun the targeted tests and type/lint checks listed by the author.

## Reproduction Assessment

Yes. The current-main focused check is visible in `handleStatusCommand`: it selects `targetSessionEntry` but forwards `params.contextTokens`; the PR's added test reproduces that mismatch. The review finding is reproducible with a config that sets only `agents.defaults.contextTokens` and then reads `resolveAgentConfig(cfg, agentId)?.contextTokens`.

## Solution Assessment

No, not yet. The `/status` change is the narrow right direction, but the resolver change should preserve inherited defaults before the new field is safe to rely on.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve the default context token cap:** `src/agents/agent-scope-config.ts:133`
  - body: The new resolved config field only returns `entry.contextTokens`. If an agent inherits `agents.defaults.contextTokens`, `resolveAgentConfig(cfg, id)?.contextTokens` is `undefined`, while this resolver already folds defaults into fields like `verboseDefault` and `contextLimits`. Any caller that uses the new field as the effective context-window cap will drop the configured default; return `entry.contextTokens ?? agentDefaults?.contextTokens` and cover that inherited-default case.
  - confidence: 0.86
