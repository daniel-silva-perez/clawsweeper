## Summary

Keep open. Current main has partial hardening for allow-always, including exact-command durable fallback when no safer path pattern can be persisted, but it does not implement the issue’s general binary+args matching or progressive permission generalization. POSIX allowlist matching remains path/name based, docs still describe argument scope as the operator’s responsibility, and the related discussion still has open security/product follow-ups. The PitStop skill reference in the comments is useful prior art for policy dry-run ideas, but it does not close the core exec-approval gap.

## Best Possible Solution

Keep this open and narrow the product/security design. The strongest path is a cross-platform exec-approval policy model with argument-aware rules, deny-over-allow full-command matching, a dry-run matcher for operators, and a separate opt-in rule-generalization workflow that proposes scoped patterns only after explicit user confirmation. Cross-link or split remaining work with #41140, #6615, #39979, and PR #46956 as maintainers choose.
