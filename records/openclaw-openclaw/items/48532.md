## Summary

Current main has partial exec-approval hardening, but it does not implement the issue’s central cross-platform binary+args permission model or progressive permission generalization. POSIX allowlist matching remains path/name based, allow-always still persists ordinary POSIX executable path patterns without argv constraints, and the docs still describe allowlist argument scope as the operator’s responsibility.

## Best Possible Solution

Keep this open and narrow the maintainer design: a cross-platform exec-approval policy model with argument-aware allow rules, deny-over-allow full-command matching, a dry-run matcher, an approval/audit log, and an opt-in rule-generalization workflow that proposes scoped patterns only after explicit user confirmation. Cross-link or split remaining work with #41140, #6615, #39979, and PR #46956 as maintainers choose.
