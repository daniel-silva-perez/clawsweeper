## Summary

Keep this PR open. Current main still does not refresh QMD session exports from transcript-update events: QMD creates a sessions collection when enabled, but initialization only starts the filesystem watcher, that watcher skips session collections, and there is no QMD transcript listener/timer path. The PR head does implement the memory-core behavior with a debounced, agent-scoped forced `session-delta` update and a focused test, but it now bundles unrelated browser, Signal, channel contract, doctor, plugin-loader, and plugin-id changes that need maintainer review or splitting before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land the memory-core QMD change or an equivalent patch that subscribes QMD session exports to transcript updates when `qmd.sessions.enabled` is active, filters to the current agent's sessions directory, debounces updates, cleans up listener/timer state on close, and uses the forced queued `session-delta` update path. Ask the author to split or separately justify the unrelated browser, Signal, channel contract, doctor, and plugin-loader changes before merge.
