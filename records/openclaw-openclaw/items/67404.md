## Summary

Keep this PR open. Current main still does not implement the QMD transcript-update refresh path, while the latest PR head now appears to address the earlier forced-queue concern. This is still relevant implementation work for memory-core, not conservative cleanup-close material.

## Best Possible Solution

Keep the PR open and continue normal review. The best path is to land this PR or an equivalent memory-core change that subscribes QMD session export refreshes to transcript-update events behind `qmd.sessions.enabled`, filters to the manager's agent session directory, debounces and cleans up the listener/timer, and uses the forced queued update path so an in-flight QMD update always gets a later session export/index pass. Keep or strengthen regression coverage for debounce/filtering and the pending-update follow-up behavior, then require green CI or scoped maintainer-approved validation.
