## Summary

Keep this PR open. Current main still does not implement the central QMD behavior: session transcript exports are created and indexed during QMD updates, but QMD does not subscribe to transcript-update events, and its file watcher skips session collections. The PR remains a plausible implementation candidate, especially after the linked predecessor #65017 was closed as superseded by this PR, but the current PR context shows a broad diff that should be narrowed or split before merge.

## Best Possible Solution

Keep this PR open, but ask for a narrower memory-core patch or a clear split. The best implementation is to subscribe QMD session exports to canonical transcript-update events when qmd.sessions.enabled is active, filter events to the current agent's sessions directory, debounce refreshes, clean up listener/timer state on close, and use the forced queued session-delta update path so transcript writes during an in-flight qmd update are not dropped. Separately review or remove the unrelated security-sensitive and loader/test changes before merge.
