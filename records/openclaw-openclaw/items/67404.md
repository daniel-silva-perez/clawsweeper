## Summary

Keep this PR open. Current main still does not implement the requested QMD transcript-update refresh, and the supplied PR discussion identifies a real remaining correctness issue in the branch: transcript-triggered refreshes use a non-forced update path that can be absorbed by an already-running QMD update. The PR is focused and still addresses a valid memory-core gap, but it needs revision or maintainer boundary review before merge.

## Best Possible Solution

Keep this PR open and request revision or maintainer review. The best path is to add a QMD session transcript listener only when QMD session export is enabled, filter events to the manager’s agent session directory, debounce updates, clean up timers/subscriptions on close, and schedule transcript-triggered refreshes through a forced or queued update path so an in-flight QMD update always gets a follow-up export/index pass. Maintainers should also choose the intended public boundary, likely the existing runtime events or memory-core host seam unless `agent-harness` should intentionally expose this subscription.
