## Summary

Keep this PR open. Current main still lacks QMD session-export refresh on transcript updates, and the PR remains useful but needs revision around update queuing and SDK boundary choice before merge.

## Best Possible Solution

Keep this PR open and request revision or maintainer review. The best fix is to add a QMD session transcript listener only when QMD session export is enabled, filter events to the manager’s agent session directory, debounce updates, clean up timers/subscriptions on close, and schedule transcript-triggered refreshes through a forced or explicitly queued update path so an in-flight QMD update always gets a later export/index pass. Maintainers should also choose the intended public boundary, likely the existing memory-core host foundation or runtime events seam unless `agent-harness` should intentionally expose this subscription.
