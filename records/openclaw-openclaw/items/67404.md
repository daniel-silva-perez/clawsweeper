## Summary

Keep this PR open. Current main still does not refresh QMD session exports from transcript-update events, and the latest PR head still appears to schedule transcript-triggered QMD refreshes through the non-forced update path that can be absorbed by an already-running update.

## Best Possible Solution

Keep the PR open and request a focused revision: keep the QMD transcript listener behind `sessions.enabled`, filter it to the manager’s agent session directory, debounce and clean up the listener/timer, and route transcript-triggered refreshes through a forced or explicitly queued follow-up update so an in-flight QMD update always gets a later export/index pass. Add a regression test that simulates a pending QMD update and proves the transcript event schedules the follow-up work.

## Risks / Open Questions

- As currently patched, a transcript update during an in-flight QMD update can be dropped because `scheduleSessionSync()` uses the non-forced update path; with periodic QMD updates disabled, exported sessions may remain stale indefinitely.
