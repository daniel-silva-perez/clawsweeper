## Summary

Keep this issue open. Current main has a concrete `model.usage` diagnostic payload shape in source and tests, but the public logging/OpenTelemetry docs still do not publish exact JSON keys, a sample event, or which local surface external read-only parsers should rely on. The adjacent open PR #39888 is about adding CLI-agent emission and does not resolve the docs/schema request.

## Best Possible Solution

Keep the issue open and resolve it with a docs/contract update. Add a `model.usage` field table and JSON example to `docs/logging.md` and/or `docs/gateway/opentelemetry.md`, clarify camelCase OpenClaw fields versus provider-native snake_case counters, explain `usage` versus `lastCallUsage` versus `context`, and state which local surface external tools should parse. Also decide whether CLI-agent runs are in scope before documenting coverage, since #39888 tracks that adjacent emission path.
