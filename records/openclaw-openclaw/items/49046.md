## Summary

Keep this issue open. Current main has a concrete `model.usage` diagnostic payload shape in source and tests, but the user-facing logging/OpenTelemetry docs still describe the event only by categories and do not publish the exact JSON keys or a sample entry for external parsers.

## Best Possible Solution

Keep the issue open and resolve it with a docs/contract update: add a `model.usage` payload table and JSON example to the public logging/OpenTelemetry docs, clarify camelCase OpenClaw fields versus provider-native snake_case token counters, explain `usage` versus `lastCallUsage` versus `context`, and explicitly state which local surface external read-only tools should parse.
