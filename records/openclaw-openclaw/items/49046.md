## Summary

Keep this issue open. Current main has a concrete `model.usage` diagnostic payload shape in source and tests, but the public logging/OpenTelemetry docs still do not publish exact JSON keys, a sample entry, or the supported local parsing surface for external read-only usage tools. An adjacent open PR tracks CLI agent emission, but it does not resolve this docs/schema request.

## Best Possible Solution

Keep the issue open and resolve it with a docs/contract update. Add a `model.usage` field table and JSON example to the public logging and/or OpenTelemetry docs, clarify camelCase OpenClaw fields versus provider-native snake_case counters, explain `usage` versus `lastCallUsage` versus `context`, and state which local surface external tools should parse. Also decide whether CLI agent runs are in scope before documenting coverage, since #39888 tracks that adjacent emission path.
