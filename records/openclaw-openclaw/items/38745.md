## Summary

Current main still has the reported Telegram model-list parser bug. `buildProviderKeyboard()` emits provider IDs unchanged in `mdl_list_${provider.id}_1`, but `parseModelCallbackData()` still only accepts `[a-z0-9_-]+`, so `mdl_list_hf.co_1` remains unparsed. The latest release tag has the same parser, and the two related fix PRs were closed unmerged.

## Best Possible Solution

Keep this issue open and land a narrow Telegram plugin fix: allow dots in the `mdl_list_{provider}_{page}` provider capture, add regression tests for `hf.co`, and avoid changing model-selection callback semantics or core model catalog behavior. Because this is a user-facing Telegram bug, include a changelog fix entry with proper reporter/contributor credit.
