## Summary

Current main still has the reported Telegram model-list callback mismatch. `buildProviderKeyboard()` emits raw provider IDs into `mdl_list_${provider.id}_1`, so `hf.co` becomes `mdl_list_hf.co_1`, but `parseModelCallbackData()` still parses `mdl_list_*` with a provider capture that excludes dots. The latest release tag `v2026.4.26` has the same parser, and the two exact fix PRs in the related context were closed unmerged.

## Best Possible Solution

Keep this issue open and land a narrow Telegram plugin fix: allow dots in the `mdl_list_{provider}_{page}` provider capture, add regression coverage for `hf.co` parsing and provider-button round-trip behavior, and add a user-facing changelog fix entry with reporter/contributor credit. Do not change core model catalog behavior or model-selection callback semantics.
