## Summary

Current main still has the reported Telegram model-list callback bug. `buildProviderKeyboard()` emits raw provider IDs into `mdl_list_${provider.id}_1`, so `hf.co` becomes `mdl_list_hf.co_1`, while `parseModelCallbackData()` still uses a list-callback regex whose provider capture excludes dots. The latest release tag `v2026.4.26` has the same source behavior, and the two related fix PRs were closed unmerged, so this should stay open and is a good narrow fix-PR candidate.

## Best Possible Solution

Keep this issue open and land a narrow Telegram plugin fix: allow dots in the `mdl_list_{provider}_{page}` provider capture, add regression coverage for `hf.co` parsing plus provider-button round-trip behavior, and add a user-facing changelog fix entry with proper credit. Do not change core model catalog behavior, model-selection callback semantics, or generated dist files by hand.
