## Summary

Keep open. Current main still has the observable behavior behind the report: the first target run is session-only, but normal `sessions_send` still schedules the A2A reply/announce path, and fresh targets can be stamped with the internal `webchat` channel as delivery metadata. An open fix candidate, #70344, addresses the same boundary, so this should stay open pending review or replacement of that PR rather than being closed as cleanup.

## What This Changes

The issue reports that `sessions_send` from a visible webchat-backed session to an internal session can surface target replies through the A2A announce/reply path and bind the destination session to webchat delivery context.

## Best Possible Solution

Review and either land or replace #70344 with a focused fix that makes `sessions_send` from visible/internal contexts session-local by default, avoids persisting internal `webchat` as target delivery context, and adds regression coverage proving the target reply stays out of visible WebChat history unless an explicit user-visible handoff is requested.
