## Summary

Current main still matches the core of the report: `sessions_send` starts the target run as an internal WebChat-channel run, reports announce-mode delivery, starts the A2A follow-up/announce flow, and the Gateway can persist that WebChat channel hint onto an otherwise internal target session. Related cleanup reports about preserving established external routes do not cover this no-external-route/internal target case.

## Best Possible Solution

Keep this issue open. The likely fix belongs in core session-send/Gateway routing: prevent inter-session `deliver:false` WebChat/internal requests from seeding target `deliveryContext`, and either skip A2A announce for internal/no-deliverable targets or add an explicit internal-only mode with docs and regression tests for visible WebChat source to internal target.
