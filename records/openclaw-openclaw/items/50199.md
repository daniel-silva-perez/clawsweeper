## Summary

Keep open. Current main still has fixed source precedence, eligibility filtering, alphabetical prompt ordering, prompt-size limits, and truncation notices, but it still lacks any user-configurable skill priority/ordering contract or `openclaw skills priority` CLI surface. Related work is useful context, but the closed priority PR did not merge and the open truncation items do not supersede the broader priority semantics requested here.

## What This Changes

The issue requests user-configurable skill priority or ordering so overlapping skills can be preferred or treated as fallbacks consistently.

## Best Possible Solution

Define one first-class skills ordering contract in the existing config schema and docs, apply it only after eligibility and per-agent filters are resolved and before prompt count/character limits, keep non-priority ordering deterministic for prompt-cache stability, and wire the same order through snapshots, runtime materialization, CLI inspection, docs, and the related truncation/pinning work.
