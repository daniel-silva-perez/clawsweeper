## Summary

Keep open. Current main still has deterministic source precedence, eligibility filtering, alphabetical prompt ordering, prompt limits, and truncation warnings, but it does not provide a user-configurable skill priority/ordering contract or an `openclaw skills priority` CLI surface. The related truncation issue is useful context, but it does not supersede this broader ordering request.

## What This Changes

Request adds user-configurable skill priority/ordering and CLI management so overlapping skills can be preferred deterministically.

## Best Possible Solution

Define one first-class skills ordering contract in the existing config schema and docs, apply it after eligibility and per-agent filters but before prompt count/character limits, preserve deterministic fallback ordering, and wire the same order through snapshots, runtime materialization, CLI inspection, and the related truncation/pinning work.
