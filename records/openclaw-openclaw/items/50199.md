## Summary

Keep open. Current main still lacks a user-configurable skill priority or ordering contract, and the related open truncation report plus closed unmerged priority PRs do not supersede this broader feature request.

## What This Changes

The issue requests configurable skill priority/ordering so overlapping skills can be selected or surfaced predictably.

## Best Possible Solution

Define one first-class skills ordering contract in the existing config schema and docs, apply it after eligibility and per-agent filters but before prompt count/character limits, preserve deterministic fallback ordering, and wire the same order through snapshots, runtime materialization, CLI inspection, docs, and the related truncation/pinning work.

## Reproduction Assessment

Not applicable as a feature request. The absence is source-verifiable on current main through the loader, prompt builder, strict config schema, CLI registration, docs, and alphabetical ordering test.

## Solution Assessment

Unclear until maintainer product review. The proposed separate priority file and CLI are directionally useful, but the maintainable solution should likely use the existing config/API contract rather than bypassing `skills.entries`, agent allowlists, and snapshot semantics.
