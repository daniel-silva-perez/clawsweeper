## Summary

Keep open. Current main still has deterministic source precedence, eligibility filtering, alphabetical prompt ordering, prompt limits, and truncation warnings, but it does not provide a user-configurable skill priority/ordering contract or an `openclaw skills priority` CLI surface. Related priority PRs are closed and unmerged, and the open truncation/preference issues are adjacent rather than canonical replacements for this broader feature request.

## What This Changes

The issue requests user-configurable skill priority so overlapping skills can be selected or surfaced deterministically according to user preference.

## Best Possible Solution

Define one first-class skills ordering contract in the existing config schema and docs, apply it after eligibility and per-agent filters but before prompt count/character limits, keep non-priority fallback ordering deterministic, and wire the same order through snapshots, runtime materialization, CLI inspection, docs, and related truncation/pinning work.
