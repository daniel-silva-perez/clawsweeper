## Summary

Keep open. Current main has deterministic source precedence, agent allowlists, per-skill enabled/env/apiKey config, and alphabetical prompt ordering, but it still has no user-configurable skill priority or `openclaw skills priority` CLI surface. The request is concrete and not old enough for stale-insufficient-info cleanup, but priority semantics need maintainer product/API design because they could affect prompt order, slash-command order, same-name source precedence, filtering, or model-facing selection behavior.

## Best Possible Solution

Keep this open for maintainer design. The best path is to decide whether priority belongs under existing `skills.entries.*`, a top-level ordered list in `~/.openclaw/openclaw.json`, skill metadata, or a different mechanism; define whether it affects prompt ordering, slash-command ordering, source precedence, filtering, or runtime selection; then update the loader, config schema/help, CLI/docs, and skill prompt tests together.
