## Summary

Keep open. Current main has deterministic source precedence, eligibility filtering, and alphabetical prompt ordering for skills, but it still has no user-configurable priority, pinning, or `openclaw skills priority` CLI surface. The adjacent open issues are related but do not supersede this request, and the remaining work needs maintainer product/API design before an implementation PR is safe.

## What This Changes

Add a user-configurable skill priority or ordering mechanism, with config and CLI support, so overlapping skills can be preferred consistently.

## Best Possible Solution

Define a first-class skill ordering contract in the existing skills config/schema/docs, keep eligibility and per-agent allowlists authoritative, and explicitly decide which runtime surfaces consume the ordering before adding CLI management and focused tests.
