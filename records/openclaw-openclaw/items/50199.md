## Summary

Keep open. Current main still has fixed source precedence, eligibility filtering, alphabetical prompt ordering, prompt limits, and truncation warnings, but it does not expose a user-configurable skill priority or ordering contract. The adjacent truncation/pinning report is useful context, not a canonical replacement for this broader API/config decision.

## What This Changes

Add a user-configurable skill priority/ordering system, including config storage, priority-aware skill discovery or prompt ordering, and `openclaw skills priority` CLI commands.

## Best Possible Solution

Define one first-class skills ordering contract in the existing config schema and docs, apply it after eligibility and per-agent filters but before prompt count/character limits, preserve deterministic fallback ordering for prompt-cache stability, and wire the same order through snapshots, runtime materialization, CLI inspection, docs, and the adjacent truncation/pinning work.

## Reproduction Assessment

Not applicable as a feature request. The absence is source-verifiable on current main through the loader, prompt builder, strict config schema, CLI registration, docs, and ordering test.

## Solution Assessment

Unclear until maintainer product review. The proposed `priority.json` and CLI are directionally relevant, but the maintainable fix should settle one core config/API contract rather than adding an ad hoc external file that bypasses existing `skills.entries` and agent allowlist semantics.
