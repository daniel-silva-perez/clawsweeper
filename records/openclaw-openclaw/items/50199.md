## Summary

Keep open. Current main has deterministic source precedence, eligibility filtering, and alphabetical prompt ordering for skills, but it still has no user-configurable priority, pinning, or `openclaw skills priority` CLI surface. The related truncation/pinning report complements this request but does not supersede the broader priority semantics requested here.

## What This Changes

Add user-configurable skill priority or ordering so overlapping skills can be preferred predictably without weakening existing skill gating and allowlists.

## Best Possible Solution

Define a first-class skill ordering contract in the existing skills config/schema/docs, apply it only after eligibility and per-agent allowlists are resolved, and explicitly wire the same ordering through prompt construction, truncation behavior, CLI inspection, and any user-facing skill command catalog.
