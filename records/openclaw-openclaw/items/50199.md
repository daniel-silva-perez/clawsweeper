## Summary

Keep open. Current main has deterministic skill source precedence, eligibility filtering, alphabetical prompt ordering, prompt limits, and truncation warnings, but it still has no user-configurable skill priority/ordering contract or `openclaw skills priority` CLI surface. The closed unmerged priority PR and the open truncation report are useful context, but neither supersedes the broader priority semantics requested here.

## What This Changes

Add user-configurable skill priority/ordering so overlapping skills can be preferred or used as fallbacks consistently, with config support and CLI management.

## Best Possible Solution

Define one first-class skills ordering contract in the existing config schema/docs, apply it only after eligibility and per-agent filters are resolved and before prompt count/character limits, keep non-priority ordering deterministic for prompt-cache stability, and wire the same order through snapshots, runtime materialization, CLI inspection, docs, and the related truncation/pinning work.
