## Summary

Keep open. Current main still has deterministic skill source precedence, eligibility filtering, alphabetical prompt ordering, prompt limits, and truncation warnings, but it does not provide a user-configurable skill priority or ordering contract, nor an `openclaw skills priority` CLI surface. The exact priority PR #42669 is closed unmerged, and the open truncation work in #50677/#42637 is related but narrower, so it does not supersede this broader feature request.

## What This Changes

The issue requests a user-configurable priority or ordering system for skills so overlapping skills can be selected predictably.

## Best Possible Solution

Define one first-class skills ordering contract in the existing config schema and docs, apply it after eligibility and per-agent filters but before prompt count/character limits, preserve deterministic fallback ordering for prompt-cache stability, and wire the same order through snapshots, runtime materialization, CLI inspection, and docs.
