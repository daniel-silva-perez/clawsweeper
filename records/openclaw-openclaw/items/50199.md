## Summary

Keep open. Current main has deterministic source precedence, eligibility filtering, and alphabetical prompt ordering for skills, but it still has no user-configurable skill priority, pinning, or `openclaw skills priority` CLI surface. The request is concrete and related open items do not supersede it; the remaining work needs maintainer API/product design because priority could affect source precedence, prompt ordering, slash-command discovery, truncation, and model-facing selection behavior.

## What This Changes

Add user-configurable skill priority or ordering so overlapping eligible skills can be preferred deterministically.

## Best Possible Solution

Keep this open for maintainer design. Decide whether priority belongs under `skills.entries.*`, a top-level ordered list in `~/.openclaw/openclaw.json`, skill metadata, or another mechanism; define whether it affects source precedence, eligibility filtering, prompt order, truncation survival, slash-command ordering, or only model guidance; then update loader/config schema/help, CLI/docs, and focused skill prompt tests together.
