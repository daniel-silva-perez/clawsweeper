## Summary

Keep this PR open. Current main still rejects Feishu external media keys containing embedded `..`, so the PR’s intended fix is not implemented. The PR is still useful, but it should be refreshed to keep the narrow security posture: allow embedded `..` in opaque Feishu keys while still rejecting `/`, `\`, control characters, empty values, and exact dot-segment tokens such as `..`.

## Best Possible Solution

Keep the PR open and refresh it into the narrow Feishu plugin fix: preserve empty, length, control-character, `/`, and `\` rejection; allow embedded `..` inside opaque Feishu media keys; explicitly reject exact dot-segment tokens such as `..` and likely `.`; and add regression tests for accepted embedded-dot keys plus rejected separator and exact-dot inputs.
