## Summary

Keep this PR open. Current main still rejects opaque Feishu media keys containing embedded `..`, so the intended fix for Feishu image analysis is not implemented. The PR is narrowly scoped and its later revision preserves slash/backslash rejection, but it still needs exact `..` hardening before merge because these keys flow into Feishu media API path parameters.

## Best Possible Solution

Keep this PR open and refresh it into a narrow Feishu plugin fix: allow embedded `..` inside opaque Feishu media keys, preserve empty/length/control-character plus `/` and `\` rejection, explicitly reject `normalized === ".."`, and add regression tests for accepted embedded-dot keys plus rejected separator and exact-dot inputs.
