## Summary

Keep this PR open. Current main still rejects opaque Feishu media keys containing embedded `..`, so the intended fix for the linked Feishu image-analysis bug is not implemented. The PR is close, but its latest patch should also reject standalone dot-segment tokens, especially `..`, before the normalized key is passed into Feishu media API path parameters.

## Best Possible Solution

Keep this PR open and refresh it into a narrow Feishu plugin fix: allow embedded `..` inside opaque Feishu media keys, preserve empty/length/control-character `/` and `\` rejection, explicitly reject exact dot-segment tokens such as `..` and likely `.`, and add regression tests for accepted embedded-dot keys plus rejected separator and exact-dot inputs.
