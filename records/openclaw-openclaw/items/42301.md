## Summary

Keep this PR open. Current main still rejects any Feishu external media key containing embedded `..`, so the intended bug fix is not implemented. The PR is useful, but it should be refreshed narrowly: allow embedded `..` in opaque Feishu keys while still rejecting path separators and exact dot-segment tokens such as `..`.

## Best Possible Solution

Keep the PR open and refresh it into a narrow Feishu plugin fix: preserve empty, length, control-character, `/`, and `\` rejection; allow embedded `..` inside opaque Feishu media keys; explicitly reject exact dot-segment tokens such as `..` and likely `.`; and add regression tests for accepted embedded-dot keys plus rejected separator and exact-dot inputs.
