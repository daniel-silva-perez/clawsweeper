## Summary

Keep open. Current main has fixes for tagged `commentary`/`final_answer` leakage and Telegram cron delivery now prefers `finalAssistantVisibleText` when available, but that does not solve this report: untagged DeepSeek preamble is still ordinary unphased assistant-visible text, and cron payloads still do not expose a generic final-output boundary such as `enforceFinalTag` or `outputStartsWith`. A maintainer also explicitly left this issue open for a provider-agnostic cron final-output design rather than a DeepSeek/emoji sanitizer.

## Best Possible Solution

Keep this issue open for a generic cron final-output contract. The maintainable path is to trace the 2026.3.13 to 2026.4.5 delivery-path change, define a provider- and channel-agnostic boundary for isolated cron final output, expose it through the cron schema/tool contract if needed, and add regression coverage for unphased preamble before a formatted Telegram report. Coordinate with broader #25592, but do not close this issue until the cron-specific final-output behavior is implemented or explicitly superseded.
