## Summary

Keep this PR open. Current main has the implied-scope behavior and already expects `operator.write` to normalize to `operator.read` plus `operator.write`, but it has not implemented this PR’s actual one-line test clarification: changing the pre-existing overwritten scope from `operator.read` to an unrelated scope so the overwrite behavior is unambiguous. The prior same-author PR #53153 was closed unmerged, so this PR remains the active candidate for that cleanup.

## Best Possible Solution

Review and land this small test-only PR, or apply the same one-line fixture change directly in `src/shared/device-auth-store.test.ts` if maintainers prefer not to merge the contributor branch. No production or security-sensitive changes are needed.
