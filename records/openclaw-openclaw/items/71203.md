## Summary

Keep this PR open. Current main still does not refresh configured non-default agent `models.json` caches during startup; it only schedules primary-model prewarm, which returns when there is no explicit default primary model and otherwise refreshes one active/default agent dir. The PR remains a focused implementation candidate for the stale-cache problem described in #71192 and is not superseded by #73420, which addressed nonblocking startup prewarm ordering.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The best implementation is a startup refresh helper that refreshes unique configured agent dirs independently of primary-model prewarm, preserves active override/default-dir semantics, avoids unintended synthetic-main writes, handles per-dir refresh failures without aborting gateway startup, and keeps focused regression coverage for configured non-default agents and the no-default-primary case.

## Risks / Open Questions

- Current main can still leave configured non-default agent `models.json` files stale after restart until some later per-agent path rewrites that exact directory.
- The PR broadens startup-time writes from the active/default agent dir to configured concrete agent dirs; maintainers should review override semantics (`OPENCLAW_AGENT_DIR` / `PI_CODING_AGENT_DIR`), deduplication, and whether the refresh should block channel startup or run best-effort in the background.
