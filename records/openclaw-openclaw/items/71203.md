## Summary

Keep this PR open. Current main still does not refresh configured non-default agent models.json caches during startup; it only schedules the primary-model prewarm path, which returns when no explicit default primary model is configured and otherwise refreshes one active/default agent dir. The PR remains a focused implementation candidate for the stale-cache behavior described in #71192, and #73420 addressed startup prewarm ordering rather than configured-agent cache propagation.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The best implementation is a startup refresh helper that refreshes unique configured agent dirs independently of primary-model prewarm, avoids unintended synthetic-main or override-path writes, handles per-dir refresh failures without aborting gateway startup, preserves the recent nonblocking channel-start behavior, and keeps focused regression coverage for configured non-default agents and the no-default-primary case.
