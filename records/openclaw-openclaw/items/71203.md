## Summary

Keep this PR open, but it still needs changes before merge. Current main still lacks the configured-agent startup refresh needed for the stale non-default agent cache report, and the PR is aimed at the right gateway surface; however, the latest branch still drops the explicit empty provider discovery scope for CLI-only configurations, which can broaden startup provider discovery.

## What This Changes

The PR adds a gateway startup helper that refreshes the active/default and configured agent `models.json` caches, wires it before channel startup, exports it for tests, and adds gateway startup regression coverage.

## Best Possible Solution

Land a revised startup refresh that regenerates unique active/default and configured agent `models.json` files independently of primary-model prewarm, while always keeping startup discovery metadata-only and scoped, including explicit empty scopes for CLI-only provider sets.

## Reproduction Assessment

Yes. A code-level reproduction is clear: configure the default/configured agent models so every provider is CLI-backed; the PR filters the provider list to `[]`, converts that to `{}`, and calls `ensureOpenClawModelsJson` without scoped metadata-only discovery options.

## Solution Assessment

No for the current PR head. The feature direction is narrow and maintainable, but the implementation should preserve `providerDiscoveryProviderIds: []` together with `providerDiscoveryEntriesOnly: true` and the startup timeout, or explicitly disable discovery for the CLI-only case.
