## Summary

Keep open. Current main still lacks the requested non-default agent `models.json` startup refresh, and this PR is aimed at the right gateway startup path, but the latest head still has a blocking scoped-discovery bug for CLI-only model configurations.

## What This Changes

The PR adds a gateway startup helper that refreshes `models.json` for the active/default and configured agent directories, wires it before channel startup, exports it for tests, and expands gateway startup coverage for custom agents.

## Best Possible Solution

Land a revised startup refresh that regenerates unique active/default and configured agent `models.json` files independently of primary-model prewarm, while always keeping startup provider discovery bounded to metadata-only scoped providers, including explicit empty scopes.

## Reproduction Assessment

Yes. The PR head has a direct code reproduction: make all configured model providers CLI-backed, so `providerDiscoveryProviderIds` becomes `[]`; the ternary drops the discovery options and current discovery receives an undefined scope.

## Solution Assessment

No for the current PR head. The feature direction is narrow and maintainable, but the implementation should pass `providerDiscoveryProviderIds: []` with `providerDiscoveryEntriesOnly: true` and the startup timeout, or otherwise disable discovery for CLI-only refreshes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve empty provider discovery scope:** `src/gateway/server-startup-post-attach.ts:163-170`
  - body: When every configured model provider is CLI-backed, `providerDiscoveryProviderIds` becomes `[]`, but this ternary turns the scoped startup options into `{}`. `ensureOpenClawModelsJson` then reaches provider discovery without either an explicit empty scope or `providerDiscoveryEntriesOnly`, so a CLI-only startup can load unrelated provider runtimes before channels start. Please preserve the empty scope, or explicitly disable discovery for this case, and cover it with a regression test.
  - confidence: 0.87
