## Summary

Keep this PR open. Current main still only prepares `models.json` through the primary-model prewarm path, returns before any cache work when no default primary model is configured, and refreshes one active/default agent dir rather than enumerating configured agent dirs. The PR remains a focused implementation candidate for #71192; it is not superseded by the #73420 nonblocking-prewarm work, though its startup ordering and broadened write behavior need maintainer review.

## What This Changes

The PR adds a gateway startup helper that refreshes `models.json` for the default and configured agent dirs with scoped provider discovery, exports it for testing, and adds gateway startup tests for non-default agents with and without a default primary model.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The best fix is a startup refresh path that refreshes unique default/configured agent dirs independently of primary-model prewarm, preserves `OPENCLAW_AGENT_DIR`/configured `agentDir` semantics, avoids synthetic duplicate main writes, handles per-dir failures as best-effort, keeps provider discovery scoped and metadata-only, and does not unnecessarily delay channel startup.
