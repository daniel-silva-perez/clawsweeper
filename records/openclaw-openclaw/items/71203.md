## Summary

Keep this PR open. Current main still lacks the startup helper this PR adds: startup schedules the existing primary-model prewarm, and that path returns when no default primary model is configured and otherwise refreshes only the active default agent dir. The PR is still a focused implementation candidate for the stale non-default agent `models.json` cache problem from #71192, and it is not superseded by #73420, which handled startup prewarm blocking rather than configured-agent cache refresh.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The best implementation is a startup refresh helper that refreshes unique concrete configured agent dirs independently of primary-model prewarm, preserves active override/default-dir semantics, avoids unintended synthetic-main writes, handles per-dir refresh failures without aborting gateway startup, and keeps focused regression coverage for configured non-default agents and the no-default-primary case.

## Risks / Open Questions

- Current main can still leave configured non-default agent `models.json` files stale after restart until a later per-agent execution path regenerates that exact directory.
- The PR broadens startup-time writes from one active/default agent dir to multiple configured agent dirs; maintainers should review `OPENCLAW_AGENT_DIR` / `PI_CODING_AGENT_DIR` override semantics, synthetic `main` fallback behavior, and whether refresh should block channel startup or run best-effort in the background.
