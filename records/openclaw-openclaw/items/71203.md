## Summary

Keep this PR open. Current main still has no startup helper that refreshes configured non-default agent models.json caches; startup only schedules the existing primary-model prewarm, which returns without writing when no default primary model is configured and otherwise writes the active default agent dir only. The PR remains a focused implementation candidate for the stale custom-agent cache problem described by #71192, with remaining review needed around override/synthetic-main semantics.

## Best Possible Solution

Keep this PR open for review or revision. The best implementation is a startup refresh helper that refreshes unique concrete configured agent dirs independently of primary-model prewarm, preserves default and override agent-dir semantics, avoids unintended synthetic-main writes, catches per-dir refresh failures without aborting gateway startup, and keeps focused regression coverage for configured non-default agents and the no-default-primary case.
