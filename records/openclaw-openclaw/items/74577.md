## Summary

Current main still has the reported config-contract mismatch: `browser.tabCleanup` is typed, documented, labeled, and read by the browser runtime, but the strict root config schema and generated base schema still do not accept it. The issue should stay open because the bug remains valid and an open linked PR (#74588) uses closing syntax for this issue, so the right maintainer action is to review that PR rather than close the issue early.

## What This Changes

The issue asks OpenClaw to accept documented `browser.tabCleanup` settings in `openclaw.json` instead of rejecting them as an unrecognized browser config key.

## Best Possible Solution

Land a focused schema fix, likely via the linked PR if it passes review, that adds a strict optional `browser.tabCleanup` object to the root config schema, regenerates checked-in schema/docs outputs, and covers both valid documented overrides and invalid nested keys with regression tests.
