## Summary

Keep open. Current main still has the reported config-contract mismatch: `browser.tabCleanup` is exposed in types, help/docs, and runtime resolution, but the strict root Zod schema and generated base schema still omit the key. The issue is also paired with open implementation PRs, including a closing PR, so it should remain open until a maintainer reviews, deduplicates, and lands or rejects that work.

## What This Changes

The issue asks OpenClaw to accept documented `browser.tabCleanup` config overrides during validation because the browser runtime already reads those settings.

## Best Possible Solution

Land one focused schema fix that adds a strict optional `browser.tabCleanup` object to the root config schema, regenerates the checked-in schema/hash outputs, and adds validation regressions for accepted documented keys plus rejected invalid nested keys; then close this issue after the chosen PR merges.
