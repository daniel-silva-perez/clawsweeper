## Summary

Keep open. Current main has friendlier restart wording, but drain-window inbound work is still rejected before it enters the preserved command queue, so the triggering message can still be lost and require a manual resend. The related Kafka plugin PR was closed unmerged and would not replace native in-process replay for this UX path.

## What This Changes

The issue requests friendlier user-facing restart messaging and native replay of inbound messages rejected while the gateway is draining for restart.

## Best Possible Solution

Native in-process restart handling should accept inbound command entries during the SIGUSR1 drain window, keep them queued without starting new work while the gateway is draining, and let resetAllLanes replay preserved lane queues in FIFO order after startup resumes; durable full-stop recovery can stay as separate follow-up work.
