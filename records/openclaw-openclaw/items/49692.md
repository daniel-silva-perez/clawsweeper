## Summary

Keep open. Current main improves the visible restart wording, but drain-window inbound work is still rejected before it enters the preserved command queue, so users can still lose the triggering message and have to resend after restart.

## What This Changes

Request friendlier gateway-drain UX plus automatic FIFO retry/replay of inbound messages rejected during in-process gateway restarts.

## Best Possible Solution

Native in-process restart behavior should accept inbound command entries during the drain window, hold them without starting new work while the gateway is draining, and let the existing restart reset pump the preserved lane queues in FIFO order after startup resumes.
