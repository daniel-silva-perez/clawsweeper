## Summary

Keep this PR open. Current main still has no Feishu `threadFollowUp` or `requireMentionInThread` config and still returns at the group mention gate before dispatching unmentioned thread replies, so the PR’s intended behavior is not implemented. The overlapping open PR #40530 is not a clean supersession because it proposes a different boolean policy surface, and this PR still has a security-sensitive mention-gate design choice plus an unresolved broadcast-scope review concern to settle before merge.

## Best Possible Solution

Keep this PR open until maintainers choose the canonical Feishu thread mention policy. If this PR’s `threadFollowUp` design is selected, fix the `skipMentionForThread` broadcast scope issue, add docs/changelog coverage for the new config, and land regression coverage for `requireMention: true` topic-thread follow-ups including broadcast mode. If #40530’s simpler `requireMentionInThread` design is selected instead, close this PR only after that implementation lands or maintainers explicitly choose it as the canonical replacement.
