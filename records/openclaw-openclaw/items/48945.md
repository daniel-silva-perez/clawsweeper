## Summary

Keep this PR open. Current main still has no Feishu `threadFollowUp` or `requireMentionInThread` config, and unmentioned group thread replies still return at the existing `requireMention && !ctx.mentionedBot` gate before dispatch. The PR is not implemented on main and is not cleanly superseded because open PR #40530 proposes a different, simpler policy surface; maintainers should choose the canonical Feishu thread mention design.

## Best Possible Solution

Keep this PR open until a maintainer chooses the Feishu thread mention policy. The best path is to pick one canonical API between this PR's `threadFollowUp: off|topic|active` active-topic design and #40530's `requireMentionInThread` override, then land a rebased implementation with docs/changelog updates and regression coverage for `requireMention: true` topic-thread follow-ups, including broadcast-mode behavior.
