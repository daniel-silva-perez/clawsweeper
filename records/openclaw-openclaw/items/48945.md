## Summary

Keep this PR open. Current main still drops unmentioned Feishu group thread replies when `requireMention` is true, and the overlapping open implementation in #40530 uses a different boolean API rather than this PR's active-topic `threadFollowUp` model. The PR is not implemented or cleanly superseded yet, and it still needs maintainer product/security review plus the unresolved broadcast-scope fix from review.

## Best Possible Solution

Keep this PR open until a maintainer chooses the canonical Feishu thread mention policy. If this PR's design is selected, fix the broadcast `skipMentionForThread` scope issue, add docs/changelog entries for the new config, and land regression coverage for `requireMention: true` topic-thread follow-ups including broadcast mode. If #40530's simpler `requireMentionInThread` design is selected instead, close this PR only after that implementation lands or maintainers explicitly choose it as the canonical path.
