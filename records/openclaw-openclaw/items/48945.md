## Summary

Keep this PR open. Current main still does not implement the Feishu thread-follow-up mention policy, and the overlapping open implementation in #40530 uses a different API, so this PR is not cleanly obsolete or implemented.

## Best Possible Solution

Keep this PR open until a maintainer chooses the canonical Feishu thread mention policy. If this design is selected, fix the skipMentionForThread broadcast/scope issue, then land with docs, changelog, and regression coverage for requireMention: true topic-thread follow-ups, including broadcast-mode behavior. If #40530's simpler requireMentionInThread API is selected instead, close this PR as superseded after that implementation lands.
