## Summary

Keep open. Current main still lacks the PR's intended ACP final emit, harness diagnostics in `/acp doctor`, and legacy `CLAWDBOT_SYSTEMD_UNIT` fallback, while the PR branch still has two concrete correctness issues already called out by maintainer review.

## What This Changes

The PR adds ACP child final-output buffering and normalized final events, process-local harness selection diagnostics surfaced in `/acp doctor`, and legacy `CLAWDBOT_SYSTEMD_UNIT` fallback in gateway systemd/restart helpers with tests.

## Best Possible Solution

Land a revised version that preserves the tail of long ACP child output, records strict no-fallback harness failures as none/failed or skips selection diagnostics for thrown requests, keeps the legacy unit fallback with precedence and escaping coverage, and leaves broader split-install reliability work to #74047.

## Reproduction Assessment

Yes. The PR-specific defects have source-level reproduction paths: stream more than 12k non-commentary ACP output with the final marker at the end, and force a missing harness with fallback disabled while inspecting the new diagnostics ring.

## Solution Assessment

No. The PR is directionally scoped to the right surfaces, but the current patch is not the best mergeable solution until it tail-preserves final output and records no-fallback failures accurately.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve tail content in final buffer:** `src/agents/acp-spawn-parent-stream.ts:396-398`
  - body: When the accumulated final text reaches the 12k guard, this branch stops appending all later child output. Long ACP runs can therefore emit only early transcript text and omit the actual concluding answer, which regresses the purpose of the new final event.
  - confidence: 0.94
- **[P2] Record strict harness failures as failures:** `src/agents/harness/selection.ts:118`
  - body: This strict no-fallback path records `selectedHarnessId: "pi"` immediately before throwing, even though PI never runs. `/acp doctor` would then report a fallback that did not happen; the same failure state should be recorded as none/failed or skipped.
  - confidence: 0.9
