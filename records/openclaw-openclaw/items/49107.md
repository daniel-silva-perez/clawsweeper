## Summary

Keep this PR open. Current main still does not implement the proposed non-interactive onboarding flags for trusted-proxy gateway auth, trusted proxy IPs, trusted-proxy allowUsers, or Control UI allowed origins. The underlying config and interactive configure flow already support the requested shape, so the PR remains a plausible focused onboarding parity change rather than obsolete or out of scope.

## Best Possible Solution

Keep this PR open or replace it with an updated focused PR that adds non-interactive onboarding support for `gateway.auth.mode="trusted-proxy"`, `gateway.trustedProxies`, `gateway.auth.trustedProxy.userHeader`, `requiredHeaders`, `allowUsers`, and `gateway.controlUi.allowedOrigins`, with a deliberate decision on `allowLoopback`, targeted tests, and docs aligned with the current trusted-proxy security model. Coordinate with #69066 for broader trust-model wording, but do not close this PR as implemented or superseded.
