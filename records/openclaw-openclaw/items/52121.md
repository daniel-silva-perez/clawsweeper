## Summary

Keep this PR open. Current main has related security-audit checks and a hard startup refusal for unauthenticated non-loopback gateway binds, but it still does not implement this PR’s central behavior: startup diagnostics for conflicting exec, sandbox, and elevated configuration. The PR also still needs maintainer review for effective-config handling, per-agent overrides, and non-throwing startup behavior.

## Best Possible Solution

Keep this PR open for a refresh against current main if maintainers still want startup diagnostics. The best path is a non-throwing detector that runs after effective runtime config is known, reuses existing security-audit semantics, evaluates default and per-agent exec/elevated/sandbox combinations, treats already-enforced no-auth bind cases consistently, fixes `exec.security` wording, and adds regression tests for agent overrides, CLI/runtime bind overrides, SSH sandbox config, and the original user flow.
