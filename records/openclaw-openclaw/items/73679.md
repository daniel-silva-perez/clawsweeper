## Summary

Keep this PR open. Current main still has the reported Control UI behavior: session switching clears the active in-memory chat queue, there is no per-session queue store, and queued follow-ups are client-side UI state until flushed. The PR remains the active implementation candidate for open bug #73621 and is not obsolete or superseded by main.

## Best Possible Solution

Review and either land or request changes on this PR as the focused fix for #73621. The likely maintainer path is a small Control UI change that stores queued messages per session, routes all session-switch surfaces through one helper, and adds the proposed regression coverage without changing gateway or disk persistence.
