## Summary

Keep this PR open. Current main still has the reported Control UI behavior: queued chat messages live only in the active in-memory `chatQueue`, both session-switch paths clear that queue, and there is no per-session queue store on main. The PR remains a focused implementation candidate for the linked open bug.

## Best Possible Solution

Review and either land or request changes on this PR as the focused fix for the linked Control UI bug. The likely path is to store queued messages per session in the UI, route all session-switch surfaces through one helper, and keep the new regression test without changing gateway or disk persistence.
