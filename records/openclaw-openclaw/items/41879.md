## Summary

Current main still has the reported defaults-only heartbeat scope behavior: helper enablement ignores `agents.defaults.heartbeat`, scheduler resolution falls back to one default-agent entry, runtime skips non-default agents through the same helper, and tests still assert that a defaults-only non-default agent is disabled. The zero-config part needs maintainer policy alignment because current docs still document default-on heartbeats disabled via `0m`.

## Best Possible Solution

Keep this issue open as the canonical tracker for the defaults-only multi-agent heartbeat scope bug. Maintainers should choose the zero-config/default-on policy explicitly, then update `src/infra/heartbeat-summary.ts`, `src/infra/heartbeat-runner.ts` scheduler/runtime behavior, health/status summaries, docs, and regression tests together. The narrower doctor-warning work can remain tracked separately in #46101.
