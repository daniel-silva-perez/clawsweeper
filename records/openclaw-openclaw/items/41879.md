## Summary

Current main still matches the reported defaults-only heartbeat scope bug. The enablement helper and scheduler only treat `agents.list[].heartbeat` blocks as explicit heartbeat scope, then fall back to the resolved default agent. Docs describe `agents.defaults.heartbeat` as global heartbeat behavior, and the existing test still encodes the default-agent-only result. The issue should remain open as the canonical tracker; related fix PRs in the provided context are closed without merge.

## Best Possible Solution

Keep this issue open as the canonical tracker. Fix the heartbeat owner path by aligning `isHeartbeatEnabledForAgent()`, `resolveHeartbeatAgents()`, scheduler coverage, summary/system-prompt gating, and regression tests so `agents.defaults.heartbeat` applies to the configured agent set when no per-agent heartbeat blocks exist, while preserving explicit per-agent scoping when any `agents.list[].heartbeat` block exists. Treat zero-config heartbeat disablement as a separate product/docs decision.
