## Summary

Keep this PR open. Current main still does not implement the Groq validation-error shape in `formatAssistantErrorText`, and a direct check of the current two regexes returns no match for the reported `attempted to call tool ... which was not in request.tools` message. The PR remains a plausible narrow fix, but its test should assert the sandbox-policy rewrite path rather than only checking that the raw fallback contains the tool name.

## Best Possible Solution

Keep this PR open. Rebase or recreate the small formatter change on current main, add an end-to-end formatter regression that proves the extracted Groq tool name drives `formatSandboxToolPolicyBlockedMessage`, and decide whether `src/agents/tool-loop-detection.ts` should receive the same pattern or share a helper.
