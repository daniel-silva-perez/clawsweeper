## Summary

Keep open. Current main still does not recognize the Groq `attempted to call tool ... which was not in request.tools` validation error, so the PR's intended parser change is still useful. The PR should not be closed as implemented; it needs a stronger regression test before maintainer merge consideration.

## Best Possible Solution

Keep this PR open. The implementation belongs in `formatAssistantErrorText`, where unknown-tool provider errors are already normalized before sandbox policy guidance. Before merging, strengthen the regression test with sandbox config/session data that makes the Groq error return `Tool "gog" blocked by sandbox tool policy`, so the test fails on current main and passes only when the Groq tool-name extraction works.
