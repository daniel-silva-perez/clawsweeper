## Summary

Keep open. Current main still preserves the reported failure shape: the generated workspace template places operational safety/tool sections after lower-priority startup and memory text, while bootstrap truncation keeps fixed head/tail slices without section priority, so a 1500-character per-file budget drops the critical middle sections.

## What This Changes

The issue asks OpenClaw to preserve critical tool-use and safety guidance from the generated `AGENTS.md` when users lower `agents.defaults.bootstrapMaxChars`.

## Best Possible Solution

Move the generated template's safety and tool-use guidance ahead of lower-priority onboarding/memory prose, add a regression that proves those sections survive a 1500-character bootstrap budget, and document how existing workspaces can adopt the safer ordering without overwriting personal edits.

## Reproduction Assessment

Yes. The source-level reproduction is deterministic: with the current generated template and `maxChars=1500`, the current trim math keeps the start and tail of `AGENTS.md` while excluding `Red Lines`, `External vs Internal`, and `Tools`.

## Solution Assessment

Yes for the near-term direction. Reordering the generated template is the narrowest maintainable fix; content-aware priority markers or tiered loading can remain broader follow-up work if maintainers want a generalized mechanism.
