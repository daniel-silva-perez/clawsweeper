## Summary

Keep this PR open. Current main has not implemented the PR's origin-aware doctor auto-enable change, and the work is core doctor/plugin activation logic rather than ClawHub work. The PR still needs maintainer review because current activation policy requires non-bundled channel owners to be explicitly trusted, and the supplied review thread identifies workspace and restrictive-allowlist semantics that are not fully settled.

## Best Possible Solution

Keep this PR open for maintainer review and revision in core doctor/plugin auto-enable logic. The best fix should decide the explicit-trust contract for already discovered global/workspace channel plugins, skip config materialization only when the plugin will actually be trusted and runnable, preserve restrictive `plugins.allow` behavior, and add regression tests for global, workspace, bundled, and restrictive-allowlist cases before landing or replacing this PR with a canonical fix.
