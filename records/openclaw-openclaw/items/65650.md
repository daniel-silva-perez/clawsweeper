## Summary

Current main still does not implement the requested durable pre-dispatch inbound message store. Slack ingress accepts and debounces messages in memory, then dispatches directly into the auto-reply path; the existing SQLite task registry and restart drain logic cover different surfaces. The related open PR #49431 remains the clearest implementation candidate, so this issue should stay open for maintainer review, landing, or superseding of that work.

## Best Possible Solution

Keep this issue open and use it to review, land, or supersede #49431. The best implementation is a durable pending-inbound store at the channel/gateway ingress boundary before active dispatch, with startup replay, shutdown/restart handling, idempotent delivery semantics, and targeted regression coverage for an accepted Slack turn interrupted before final reply delivery.
