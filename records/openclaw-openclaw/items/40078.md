## Summary

Keep this issue open. Current main still lacks a Discord guild/channel `preflightTranscribe` option or equivalent silent history ingestion mode. Discord can preflight-transcribe audio for DMs and spoken mention detection, but unmentioned `requireMention` guild audio still records the original `<media:audio>` history entry and returns before dispatch.

## Best Possible Solution

Keep this open and implement an explicit Discord guild/channel option such as `preflightTranscribe`. Add it to config types, strict zod schema, generated metadata/help, and Discord docs. In Discord preflight, after authorization and before the no-mention return, transcribe eligible unmentioned audio when enabled, store the transcript text in pending guild history instead of `<media:audio>`, suppress agent dispatch, and add focused tests proving a later @mention receives the transcript in history context.

## Risks / Open Questions

- Silent background transcription has cost and privacy implications, so it should stay explicit opt-in and run only after guild/channel/member authorization checks.
- An implementation should avoid doing a second STT call if the same audio is later processed during a mentioned turn.
- Docs currently describe group audio preflight for mention detection, which can be mistaken for this broader silent history-ingestion workflow unless the Discord behavior is documented precisely.
