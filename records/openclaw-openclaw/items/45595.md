Keep open. Current main still has the reported exec-approval identity mismatch, and the provided GitHub context shows open PR #73543 uses closing syntax for this issue, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Canonicalize exec approval identity so allowlist/trust checks and approved execution use the same executable path when PATH entries resolve through symlinks.

## Best Possible Solution

Review and land #73543 or an equivalent focused fix with regression coverage for real-path allowlist matching through a symlink and symlink target drift. If maintainers keep that PR narrow, track any remaining canonical-identity surfaces as explicit follow-up instead of closing this issue early.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is available by constructing a resolution with `resolvedPath` set to a symlink path and `resolvedRealPath` set to the real binary: current `matchAllowlist()` misses a real-path pattern while approved execution later pins to the realpath.

## Solution Assessment
