## Summary

Keep this issue open. Current main at ef58307f843eed1a7832e9c23c519591f1b7cdbd still validates model configuration structurally rather than against the model catalog, preserves configured refs that are absent from the catalog, and the direct implementation candidate PR #40355 is open/unmerged with closing syntax for this issue.

## Best Possible Solution

Keep this issue open until PR #40355 or an equivalent change is reviewed and merged. The best implementation is a catalog-aware warning path for configured primaries and fallbacks across defaults, per-agent config, and related model surfaces, while preserving custom/forward-compatible refs as warnings unless a provider contract can prove the model is invalid.
