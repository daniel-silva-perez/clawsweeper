## Summary

Keep open. Current main still forwards the configured compaction reserve into built-in Pi summarization without clamping it to the selected model output cap; the pinned Pi dependency derives output `maxTokens` as 80% of that reserve, and the Anthropic transport sends a positive requested value as `max_tokens`. PR #54392 remains an open implementation candidate linked to this issue, so this should stay open until that PR or an equivalent patch lands.

## What This Changes

The issue asks OpenClaw to cap built-in compaction summarization output requests to the selected model's output limit so Anthropic 1M-context compaction does not send `max_tokens` above 128000.

## Best Possible Solution

Land PR #54392 or an equivalent maintainer-approved patch that clamps the built-in LLM compaction reserve/output budget against the resolved summarization model's output cap before Pi `generateSummary` or `completeSimple` can request `max_tokens`, while preserving provider/plugin compaction behavior and Anthropic header handling. Add regression coverage for an Anthropic Claude 4.6 1M-context setup with `reserveTokensFloor: 300000`.
