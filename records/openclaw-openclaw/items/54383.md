## Summary

Keep open. Current main still has the reported built-in compaction path: OpenClaw forwards the configured reserve token budget into Pi summarization, Pi derives `maxTokens` from that reserve, and the Anthropic transport preserves a positive requested `maxTokens` as `max_tokens`. The documented `context1m` setup is supported, and open PR #54392 remains the same-author implementation candidate, so this issue should not be closed before that work lands or is replaced.

## What This Changes

The issue requests that compaction summarization for Anthropic 1M-context models clamp its output token budget to the selected model's real output limit instead of requesting values such as 240000 against a 128000-token cap.

## Best Possible Solution

Land PR #54392 or an equivalent maintainer-approved patch that clamps the built-in LLM compaction reserve/output budget against the resolved summarization model's output cap before Pi `generateSummary` or `completeSimple` can request `max_tokens`, while preserving provider/plugin compaction behavior and Anthropic header handling. Add regression coverage for an Anthropic Claude 4.6 1M-context setup with `reserveTokensFloor: 300000` proving the request cannot exceed 128000 output tokens.
