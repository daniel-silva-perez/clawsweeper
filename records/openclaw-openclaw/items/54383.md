## Summary

Keep open. Current main still forwards the compaction reserve token budget into Pi summarization without clamping it to the selected model output cap, while the pinned Pi dependency derives output maxTokens as 80% of that reserve. The linked same-author PR #54392 remains open and unmerged, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that Anthropic 1M-context compaction cap summarization max_tokens to the model output limit instead of sending a reserve-derived value such as 240000.

## Best Possible Solution

Land PR #54392 or an equivalent maintainer-approved patch that clamps the built-in LLM compaction reserve/output budget against the resolved summarization model's output cap before Pi generateSummary can request max_tokens, while preserving plugin/provider compaction behavior and Anthropic beta-header handling. Add regression coverage for an Anthropic Claude 4.6 1M-context setup with reserveTokensFloor: 300000 proving the request cannot exceed 128000 output tokens.
