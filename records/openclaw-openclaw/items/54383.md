## Summary

Keep this issue open. Current main still does not clamp the built-in compaction summarization reserve/output budget to the selected model's maxTokens, and the same-author implementation PR remains open and unmerged with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and route the remaining work through review, rebase, and landing of PR #54392 or an equivalent maintainer-approved patch. The patch should clamp the reserve/output budget passed to Pi generateSummary against the selected model's maxTokens across default and safeguard compaction paths, and add regression coverage for an Anthropic Claude 4.6 1M-context setup with reserveTokensFloor=300000 proving max_tokens cannot exceed 128000.
