## Summary

Keep this PR open. Current main still does not implement the intended iOS chat attachment processing: the composer feeds raw image bytes into the view model, the view model rejects over-5 MB inputs before resize/transcode, accepted bytes are stored unchanged, and send still base64-encodes attachment data from the @MainActor path. The live PR is open, unmerged, and draft; it is an active implementation candidate for the still-open #68524 rather than obsolete work.

## Best Possible Solution

Keep this PR open as the draft implementation candidate for #68524. The best path is to rebase it onto current main, complete real-device iOS validation, review the shared `JPEGTranscoder` changes, keep the new privacy/resize tests, run the targeted Swift checks, and then merge or close based on maintainer review.
