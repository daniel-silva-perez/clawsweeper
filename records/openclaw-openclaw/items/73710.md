## Summary

Keep open. Current main still lacks the iOS composer preprocessing this PR targets, but the latest branch still violates its advertised 1600 px long-edge contract by passing that policy through a width-only transcoder API. The PR should stay in normal maintainer review for branch correction and validation rather than being closed or replaced automatically.

## What This Changes

The PR adds iOS chat attachment preprocessing that re-encodes images as resized JPEGs with metadata stripping, runs processing off the MainActor, extends JPEGTranscoder with alpha flattening, and adds Swift tests.

## Best Possible Solution

Land a corrected version that adds a true long-edge mode or equivalent adapter calculation, keeps metadata stripping and alpha flattening centralized in JPEGTranscoder, adds portrait and already-narrow-tall test coverage, removes or rewrites the misleading 5 MB safety-net comment, and validates with the OpenClawKit Swift tests plus a physical iPhone composer/send round trip.

## Reproduction Assessment

Yes. A source-level reproduction is any portrait/tall image whose oriented width is at or below 1600 while height exceeds 1600, or a portrait image where width clamping yields output like 1600x2133; the latest diff and the contributor's device evidence both match that path.

## Solution Assessment

No, not as currently proposed. Reusing JPEGTranscoder off the MainActor is the right maintainable direction, but the implementation needs a true longest-edge contract before it is the best fix for the stated behavior.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Clamp the actual long edge:** `apps/shared/OpenClawKit/Sources/OpenClawKit/ChatImageProcessor.swift:54`

Because `JPEGTranscoder` only scales from oriented width, portrait and already-narrow tall images can still return above 1600 px on the long edge; please compute the scale from `max(orientedWidth, orientedHeight)` or add a true long-edge transcoder mode with portrait coverage.
  - confidence: 0.96

  - confidence: 0.86
