## Summary

Keep this PR open. Current main still does not implement the intended iOS/shared chat attachment processing: camera-roll and paste/file images enter the chat view model as raw data, hit the 5 MB gate before any resize/transcode, are stored unchanged, and are base64-encoded from the @MainActor view model. The provided PR context shows this is an open draft implementation candidate for the still-open iOS large-image bug #68524, with review feedback partly addressed but real-device validation and rebase still pending.

## Best Possible Solution

Continue reviewing this draft PR as the implementation candidate for #68524: rebase it onto current main, keep the ImageIO work consolidated in JPEGTranscoder, preserve the new chat privacy/resize tests, resolve any shared-transcoder contract concerns, then validate on a physical iPhone before moving it out of draft and landing.
