## Summary

Keep open. Current main still lacks the central Signal quote-reply behavior, so this PR is not obsolete; however the proposed branch still has blocking patch issues in the provided diff/context, including committed conflict markers, extension-boundary violations, and stale quote metadata risk.

## What This Changes

The PR branch wires bidirectional Signal quote replies through Signal send/monitor paths, shared outbound reply metadata, SDK reply payload typing, and new Signal/outbound regression tests.

## Best Possible Solution

Keep the useful Signal-owned quote-reply work, but make the branch mergeable by removing conflict markers, replacing core deep imports with SDK/local plugin seams, ensuring quote metadata is cleared with consumed reply targets, preserving expected thread behavior for other channels, and keeping focused regression coverage for Signal inbound/outbound quote ids, group quote-author lookup, formatted sends, media/chunk delivery, debounce, and queued recovery.
