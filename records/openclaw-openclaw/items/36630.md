## Summary

Keep this PR open. Current main still does not implement the PR's central bidirectional Signal quote-reply behavior: Signal sends have no native quote options or signal-cli quote params, the Signal outbound adapter does not pass reply metadata into formatted/direct sends, monitor replies are sent without quote metadata, and inbound quote context still lacks a structured ReplyToId. The PR remains an implementation candidate, but the live PR diff still has merge-conflict markers and extension-boundary/security-sensitive issues that need maintainer cleanup before merge.

## Best Possible Solution

Keep this PR open and rework it as a smaller, Signal-owned quote-reply implementation: add plugin-safe Signal quote timestamp/author plumbing through an SDK or local plugin facade, route formatted/direct/monitor text and media sends through quote-aware delivery, surface inbound Signal quote ids as ReplyToId with quote body/sender, remove conflict markers and core deep imports, and add focused regression coverage for group quote-author lookup, formatted sends, monitor replies, media/chunk/sendPayload consumption, debounce, and queued recovery.
