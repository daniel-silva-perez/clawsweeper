## Summary

Keep this PR open. Current main still does not implement the central bidirectional Signal quote-reply behavior: Signal sends do not emit signal-cli quote RPC parameters, Signal outbound paths do not pass reply/quote-author metadata, monitor reply delivery sends unquoted messages, and inbound Signal quote context still lacks a structured ReplyToId. The PR remains a plausible implementation candidate, but its provided review context shows branch blockers that need maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open and rework it into a narrower, Signal-owned quote-reply change: add plugin-safe quote timestamp/author plumbing through the SDK or a Signal-local facade, pass reply metadata through formatted/direct/monitor text and media sends, surface inbound Signal quote ids as ReplyToId with quoted body/sender, remove conflict markers and core deep imports, and keep focused regression coverage for group quote-author lookup, formatted sends, monitor replies, media/chunk/sendPayload consumption, debounce, and queued recovery.
