## Summary

Keep this PR open. Current main still does not implement the PR’s central bidirectional Signal quote-reply behavior: Signal sends have no `replyTo`/`quoteAuthor` options and never emit signal-cli quote parameters, Signal outbound adapters do not forward quote metadata, monitor replies are still sent unquoted, and inbound quote context still lacks a structured `ReplyToId`. The PR remains an implementation candidate, but it needs maintainer review and cleanup before merge because the live patch still crosses extension/core boundaries and touches shared outbound delivery behavior.

## Best Possible Solution

Keep this PR open and rework it as a smaller, Signal-owned quote-reply implementation: add plugin-safe Signal quote timestamp/author plumbing, route formatted/direct/monitor sends through quote-aware delivery, surface inbound `ReplyToId` from the Signal quote timestamp with quote body/sender, avoid core deep imports from extension production code, and add focused regression tests for group quote-author lookup, formatted sends, monitor replies, media/chunk/sendPayload consumption, and queued recovery.
