## Summary

Keep this PR open. Current main still does not implement the PR's central bidirectional Signal quote-reply behavior: Signal sends do not emit signal-cli quote parameters, outbound adapters do not forward quote metadata, monitor replies are unquoted, and inbound quote context still lacks the Signal quote timestamp as `ReplyToId`. The discussion and current PR head show useful implementation work, but the PR is still an implementation candidate with build, plugin-boundary, and quote-metadata lifecycle issues to resolve before merge.

## Best Possible Solution

Keep this PR open and rework it into a smaller, hardened Signal quote-reply change: add Signal-owned `quote-timestamp`/`quote-author` support through plugin-safe SDK or local plugin seams, route formatted/direct/monitor sends through quote-aware delivery, surface inbound `ReplyToId` plus quote body/sender safely, remove conflict markers and core deep imports, and add targeted tests for formatted sends, monitor replies, group quote-author lookup, chunk/media/sendPayload consumption, and queue recovery.
