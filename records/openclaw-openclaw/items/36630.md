## Summary

Keep this PR open. Current main still does not implement the central bidirectional Signal quote-reply behavior: outbound Signal sends have no signal-cli quote RPC parameters, Signal outbound adapters do not pass reply metadata or quote author metadata, monitor replies are sent without quote metadata, and inbound Signal quote context still does not surface a structured ReplyToId. The PR remains a plausible implementation candidate, but the current head is not cleanup-closeable because it still carries merge-conflict markers and extension-boundary violations in the diff.

## Best Possible Solution

Keep this PR open and rework it into a narrower Signal-owned quote-reply change: add plugin-safe quote timestamp/author plumbing through the SDK or a Signal-local facade, route formatted/direct/monitor text and media sends through quote-aware delivery, surface inbound Signal quote ids as ReplyToId with quote body/sender, remove conflict markers and core deep imports, and keep focused regression coverage for group quote-author lookup, formatted sends, monitor replies, media/chunk/sendPayload consumption, debounce, and queued recovery.
