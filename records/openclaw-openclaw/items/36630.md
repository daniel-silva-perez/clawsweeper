## Summary

Keep this PR open. Current main at 5828dcdb05aad587bff831467e7fa51f73312ff8 still lacks the central bidirectional Signal quote-reply behavior: outbound Signal sends do not emit native quote parameters, and inbound Signal quote context remains partial without ReplyToId or richer quote metadata. The PR remains a plausible implementation candidate, but it needs cleanup and security/quality review before it can land.

## Best Possible Solution

Keep this PR open and review or rework it into a hardened Signal quote-reply implementation: add quote timestamp/author support through Signal send and formatted delivery paths, resolve group quote authors from the referenced message where possible, surface inbound ReplyToId and richer quote metadata, keep extension production imports behind plugin SDK/local boundaries, remove conflict markers, and add targeted regression coverage for formatted sends, monitor replies, chunk/media delivery, debounced inbound quote context, and queued/recovered delivery.
