## Summary

Keep open. Current main still lacks native Signal quote-send parameters and complete structured inbound quote context, while the PR head is not merge-ready because it contains a parse-blocking conflict, bundled plugin boundary violations, and remaining quote-routing/validation regressions.

## What This Changes

The PR widens reply payload typing, adds Signal quote metadata parsing and reply-delivery helpers, wires quote metadata through Signal send paths, and adds Signal/outbound regression tests.

## Best Possible Solution

Land a smaller Signal-owned implementation after adding any needed public SDK seams, keeping quote-author resolution inside the Signal plugin, routing formatted/direct/queued sends consistently, preserving existing inbound visibility policy, and validating quote ids/authors with focused Signal plus shared outbound coverage.

## Reproduction Assessment

Yes. Static reproduction is high confidence: the PR head test file contains unresolved conflict markers at lines 5-9, and the current-main Signal send/monitor paths demonstrably lack quote parameters in the inspected source.

## Solution Assessment

No. The feature direction is useful, but this branch is not the best merge vehicle until the extension-boundary violations, formatted-send bypass, media fallback regression, and quote-id validation issue are corrected or replaced by a narrower implementation.
