## Summary

Keep this PR open. Current main has partial inbound Signal quote visibility, but it does not implement the PR’s central bidirectional Signal quote-reply behavior: Signal sends still never emit signal-cli quote parameters, the Signal outbound adapter does not forward reply/quote metadata into formatted or attached-result sends, monitor-generated replies are sent without quote metadata, and inbound Signal quote context still does not surface the quote timestamp as ReplyToId. The open PR remains an implementation candidate, not obsolete.

## Best Possible Solution

Keep this PR open and rework it as a smaller, Signal-owned quote-reply implementation: add plugin-safe Signal quote timestamp/author plumbing, route formatted/direct/monitor sends through quote-aware delivery, surface inbound ReplyToId from the Signal quote timestamp along with quote body/sender, remove conflict markers and core deep imports, and add focused tests for group quote-author lookup, formatted sends, monitor replies, media/chunk/sendPayload consumption, and queued recovery.
