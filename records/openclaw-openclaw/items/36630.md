## Summary

Keep open. Current main still lacks native Signal quote-reply send parameters, and this PR contains useful work, but the branch has concrete merge blockers in parsing, plugin boundaries, formatted delivery, inbound debounce handling, media fallback, and quote-id validation.

The PR adds Signal inbound quote context parsing, outbound Signal quote metadata plumbing, reply payload null/quoteAuthor type changes, shared outbound delivery changes, and Signal/outbound regression tests.

## Best Possible Solution

Land a smaller Signal-owned implementation after adding any needed narrow public SDK seams, keeping quote-author lookup inside the Signal plugin, routing formatted/direct/queued sends consistently, preserving inbound quote visibility policy, and validating id/author handling with focused Signal plus shared outbound coverage.

## Solution Assessment
