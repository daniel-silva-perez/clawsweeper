## Summary

Keep open. Current main still lacks the requested bidirectional Signal quote-reply behavior, so the PR is not obsolete, but the current PR head has hard merge blockers: unresolved conflict markers, production Signal plugin imports that cross the documented extension boundary, failing CI, and a concrete stale quote-author attribution risk in shared outbound reply planning.

## What This Changes

The PR wires Signal quote-reply metadata through outbound send/delivery and inbound monitor/context handling, extends shared reply payload typing, adjusts several channel reply paths, and adds Signal/outbound regression tests.

## Best Possible Solution

Keep the useful Signal-owned quote-reply work, but make it mergeable by resolving the conflict markers, replacing production core deep imports with SDK or plugin-local seams, hardening quote id and quote-author handling, preserving existing thread semantics for other channels, and validating with focused Signal/outbound tests plus the changed gate in Testbox.
