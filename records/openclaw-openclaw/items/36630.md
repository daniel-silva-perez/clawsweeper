## Summary

Keep open. Current main still lacks the PR's central Signal quote-reply behavior, so this PR is not obsolete; however the current PR head is not mergeable because the diff still contains conflict markers, production Signal plugin code deep-imports core internals across the plugin boundary, and quote metadata handling has a concrete stale-author attribution risk.

## What This Changes

The PR wires Signal quote-reply metadata through outbound Signal send/delivery and inbound Signal quote context handling, widens shared reply payload typing across several channel adapters, and adds Signal/outbound regression tests.

## Best Possible Solution

Keep the useful Signal-owned quote-reply work, but make it mergeable by resolving the conflict markers, moving production helper imports behind SDK or Signal-local seams, hardening quote id and quote-author handling, preserving existing thread semantics for other channels, and validating with focused Signal/outbound tests plus the changed gate in Testbox.
