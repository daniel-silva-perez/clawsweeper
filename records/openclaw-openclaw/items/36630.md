Keep open. Current main still lacks native Signal quote-reply send parameters, so this PR is not obsolete, but the branch has merge blockers in parsing, plugin boundaries, formatted Signal delivery, debounced quote-only handling, and quote-id validation.

Land a smaller Signal-owned implementation after adding any needed public SDK seams, keeping quote-author resolution inside the Signal plugin, routing formatted/direct/queued sends consistently, preserving inbound visibility policy, and validating quote ids/authors with focused Signal plus shared outbound coverage.
