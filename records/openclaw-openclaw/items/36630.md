## Summary

Keep open. Current main still lacks outbound Signal native quote parameters and only has partial inbound quote context, so the requested user-visible behavior is not fully implemented; the PR remains useful source work but is not mergeable as-is because the latest diff has conflict markers, plugin-boundary violations, and unresolved quote-routing/attribution bugs.

## What This Changes

The PR widens reply payload typing, adds Signal quote metadata parsing and reply-delivery helpers, wires Signal quote metadata through outbound send paths, and adds Signal/outbound regression tests.

## Best Possible Solution

Land a smaller Signal-owned implementation after it adds any needed public SDK seams, wires quote metadata through formatted/direct delivery and queued replay, preserves current inbound visibility policy, validates quote ids/authors strictly, and passes focused Signal plus shared outbound lanes.
