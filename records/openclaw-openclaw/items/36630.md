## Summary

Keep this PR open. Current main at 7120f5b25487f2fc021d7642c663e2cea279e015 still does not implement the PR’s central bidirectional Signal quote-reply behavior: Signal sends do not emit signal-cli quote parameters, Signal outbound adapters do not pass quote metadata, monitor-generated Signal replies are sent without quote metadata, and inbound quote context remains partial because quote timestamps are not surfaced as ReplyToId.

## Best Possible Solution

Keep this PR open as an implementation candidate, but rework it into a smaller Signal-owned change: add plugin-safe Signal quote timestamp/author plumbing, route formatted/direct/monitor sends through quote-aware delivery, surface inbound ReplyToId from the Signal quote timestamp with quote body/sender, remove conflict markers and core deep imports, and add focused tests for group quote-author lookup, formatted sends, monitor replies, media/chunk/sendPayload consumption, and queued recovery.
