## Summary

Keep open. Current main still lacks the PR's central outbound Signal native quote path, so the contribution is not obsolete, but the latest branch should not merge as-is because it still has parse-breaking conflict markers, production plugin-boundary violations, and several concrete quote/threading regressions visible in the PR head diff.

## What This Changes

The branch widens reply payload typing, adds Signal quote metadata parsing and reply-delivery helpers, wires Signal quote metadata through outbound send paths, and adds Signal/outbound regression tests.

## Best Possible Solution

Keep the useful Signal quote-reply direction, but land it only after the branch is made mergeable, the Signal-owned helpers use public SDK or local plugin seams, the default formatted outbound path actually carries quote metadata, queue/recovery and reply-consumption semantics are hardened, and the Signal plus shared outbound lanes are validated.
