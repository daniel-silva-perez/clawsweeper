## Summary

Current main still preserves the reported Mattermost split-state failure mode: route registration can load the slash route through the sync bundled-entry loader while monitor activation writes to the runtime import path, and the shared state is still a module-local Map. The issue should stay open while #68089 remains the active implementation candidate; #69038 was closed unmerged as a duplicate alternate patch, not as a shipped fix.

## What This Changes

Restore Mattermost native slash commands so registered `/oc_*` callbacks execute after activation instead of returning HTTP 503 "Slash commands are not yet initialized."

## Best Possible Solution

Keep this issue as the canonical bug until a Mattermost plugin fix lands. The best end state is one reviewed fix, likely #68089 or an equivalent, that guarantees slash route registration and monitor activation share the same account state across supported loader paths and adds a regression test for the initialized callback no longer returning the 503 placeholder.
