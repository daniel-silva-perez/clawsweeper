## Summary

Keep open. Current main still clears self-declared operator scopes for device-less shared token/password operator connections outside a narrow direct-local backend exception, and current gateway auth tests explicitly assert the resulting empty scopes and missing-scope behavior. There are also open PRs #51413 and #57889 with closing syntax for this issue, so the issue should not be auto-closed independently before those PRs are reviewed, closed, or merged.

## Best Possible Solution

Review #51413 and #57889 against the current gateway auth security model. Either land a scoped fix with regression coverage for authenticated non-local token/password backend clients retaining operator scopes, or explicitly document/reaffirm that remote operator scopes require device identity and close the candidate PRs before closing this issue.
