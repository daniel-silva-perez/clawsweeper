## Summary

Keep open. Current main still clears requested operator scopes for device-less shared token/password WebSocket operator connections outside the narrow direct-local backend exception, and the current regression tests assert the resulting missing-scope behavior. The provided GitHub context also shows open PRs #51413 and #57889 with closing syntax for this issue, so conservative cleanup should not close the issue before those implementation candidates are reviewed, merged, or closed.

## Best Possible Solution

Review #51413 and #57889 against the current gateway auth security model. Either land a scoped WebSocket fix with regression coverage proving authenticated non-local token/password backend clients retain operator scopes safely, or explicitly document/reaffirm that remote operator WebSocket scopes require device identity and then close the candidate PRs before closing this issue.
