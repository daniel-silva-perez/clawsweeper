## Summary

Keep this PR open. Current main still lacks native Azure OpenAI handling for memory-lancedb embeddings, but the proposed patch is not merge-ready because it only adjusts the direct client path and leaves bearer auth, Azure URL normalization, configurable API versioning, and shared Azure host detection unresolved.

## What This Changes

The PR changes the memory-lancedb direct OpenAI-compatible embedding client to detect `.openai.azure.com` base URLs and add Azure `api-key` plus hard-coded `api-version=2024-02-01` defaults.

## Best Possible Solution

Add Azure-aware memory embedding request shaping that parses hostnames, reuses the repo’s Azure suffix coverage, sends `api-key` without bearer auth, normalizes Azure embedding URLs, exposes an optional API-version setting in config/manifest/docs, and covers direct plus provider-backed embedding paths with focused tests.

## Reproduction Assessment

Yes. A code-level reproduction is clear: configure memory-lancedb with `embedding.provider: "openai"`, an `apiKey`, and an Azure resource `baseUrl`; current main selects the direct OpenAI-compatible client and posts to `/embeddings`, while the PR would still pass `apiKey` to openai-node and therefore keep bearer auth unless changed.

## Solution Assessment

No. The proposed patch is a useful start, but it is not the best fix because it uses substring detection, keeps the generic OpenAI auth path, hard-codes the API version, and leaves provider-backed memory embeddings unchanged.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Remove bearer auth on Azure embedding requests:** `extensions/memory-lancedb/index.ts:174-179`
  - body: The Azure branch still passes `apiKey` into the OpenAI client and only adds an `api-key` default header. In openai-node v6.34.0, the client still builds `Authorization: Bearer <apiKey>` before merging default headers, so Azure requests can carry the wrong auth shape unless Authorization is explicitly removed or the request path avoids that client auth.
  - confidence: 0.9
- **[P2] Normalize Azure embedding URLs before posting:** `extensions/memory-lancedb/index.ts:176-179`
  - body: This path leaves `baseURL` unchanged and later posts to `/embeddings`. If a user points `baseUrl` at an Azure resource root, the request goes to the wrong shape instead of the Azure `/openai/...` embedding endpoint pattern used elsewhere in the repo.
  - confidence: 0.83
- **[P2] Expose the Azure API version through config:** `extensions/memory-lancedb/index.ts:179`
  - body: The patch hard-codes `api-version=2024-02-01`, but the memory-lancedb runtime config, manifest schema, and docs have no API-version field. Users whose Azure deployment needs another supported version would still have no supported config path.
  - confidence: 0.91
- **[P2] Detect Azure hosts by parsed hostname and shared suffixes:** `extensions/memory-lancedb/index.ts:173`
  - body: `baseUrl?.includes(".openai.azure.com")` is both narrower and less precise than the current Azure handling elsewhere. It misses valid `.services.ai.azure.com` and `.cognitiveservices.azure.com` hosts and is not constrained to the parsed hostname.
  - confidence: 0.88
