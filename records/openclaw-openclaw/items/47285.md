## Summary

Keep this PR open, but the current patch should not merge: current main still lacks native Azure shaping for memory-lancedb embeddings, while the PR only partially adds it and leaves request auth, URL shape, API-version configurability, and host detection mismatched with the repo's existing Azure patterns.

## What This Changes

The PR changes the memory-lancedb direct OpenAI-compatible embedding client to detect `.openai.azure.com` base URLs and add Azure `api-key` and `api-version=2024-02-01` defaults.

## Best Possible Solution

Add Azure-aware memory embedding request shaping that parses Azure hostnames, sends `api-key` without bearer auth, normalizes to deployment-scoped embedding URLs, exposes an API-version setting through config/manifest/docs, and covers both direct and provider-backed memory embedding paths with focused tests.

## Reproduction Assessment

Yes. A code-level reproduction is clear: configure memory-lancedb with `embedding.provider: "openai"`, a plugin `apiKey`, and an Azure resource `baseUrl`; current main uses the direct OpenAI-compatible client and posts to `/embeddings`, while the PR would still leave bearer auth and root-style URL behavior unless changed.

## Solution Assessment

No. The proposed patch is a useful start, but it is not the best fix because it relies on substring host detection, keeps the SDK bearer-auth path, hard-codes the API version, and does not align the shared provider-backed embedding path with Azure behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Remove bearer auth from Azure embedding requests:** `extensions/memory-lancedb/index.ts:174-178`
  - body: The Azure branch still passes `apiKey` into the OpenAI client and only adds an `api-key` default header. In openai-node v6.34.0, the client still builds `Authorization: Bearer <apiKey>` unless that header is explicitly removed, so Azure requests can retain the wrong auth shape.
  - confidence: 0.91
- **[P2] Normalize Azure embedding URLs before posting:** `extensions/memory-lancedb/index.ts:175-179`
  - body: This leaves `baseURL` unchanged while the existing embed path posts to `/embeddings`. If a user follows the PR goal and points `baseUrl` at an Azure resource root, the request is sent to the wrong endpoint instead of Azure's deployment-scoped `/openai/deployments/{deployment}/embeddings` shape.
  - confidence: 0.87
- **[P2] Expose the Azure API version through config:** `extensions/memory-lancedb/index.ts:179`
  - body: The patch hard-codes `api-version=2024-02-01`, but the memory-lancedb runtime config, manifest schema, and docs have no API-version field. Users whose Azure deployment requires another supported version would still have no supported override path.
  - confidence: 0.9
- **[P2] Parse Azure hosts with the shared suffix set:** `extensions/memory-lancedb/index.ts:173`
  - body: `baseUrl?.includes(".openai.azure.com")` is narrower and less precise than the repo's existing Azure handling. It misses recognized `.services.ai.azure.com` and `.cognitiveservices.azure.com` hosts and can match text outside the parsed hostname.
  - confidence: 0.86
