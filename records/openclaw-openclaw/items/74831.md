## Summary

Keep open. The PR tackles a real contention point in the current lock-backed sandbox registry, but the patch drops same-entry update/remove serialization and can resurrect a registry entry when ensure/update races with recreate, prune, or remove for the same runtime.

## What This Changes

The PR replaces monolithic sandbox container/browser registry JSON files with per-entry JSON files, adds single-entry lookup for Docker ensure, migrates legacy files, and rewrites registry tests for sharded storage.

## Best Possible Solution

Keep the per-entry storage direction, but add per-entry coordination or equivalent tombstone/compare semantics for the same runtime name, preserve focused migration behavior, and restore race coverage proving update-after-remove cannot re-add a removed runtime.

## Reproduction Assessment

Yes for the PR regression. Current main already has focused tests that delay a registry write, race removeRegistryEntry/removeBrowserRegistryEntry, and assert the entry stays deleted; the branch removes that coverage while changing remove to a plain file removal.

## Solution Assessment

No. Sharding is a good direction for disjoint entries, but removing all coordination for same-filename mutations is not the narrowest maintainable fix; per-entry locking or an equivalent same-entry ordering protocol would preserve the intended performance win without weakening remove semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve same-entry remove/update ordering:** `src/agents/sandbox/registry.ts:235`
  - body: Current main serializes update and remove for a single registry entry, and the existing tests prove a remove that starts after an in-flight update does not resurrect the entry. This patch makes remove a plain file deletion while update still writes by tmp-file rename, so an ensure/update for the same container can rename after recreate/prune/remove and re-add stale registry state. Keep per-entry coordination or an equivalent tombstone/compare step for same-entry mutations.
