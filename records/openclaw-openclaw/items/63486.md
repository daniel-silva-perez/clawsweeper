## Summary

Keep open. This is a recent Matrix regression report with useful rollback evidence, and current main still routes the affected behavior through the bundled Matrix plugin. I found adjacent startup, E2EE, event-routing, and verify-status fixes, but not a source/test/release link proving that the Docker plus local Synapse no-response/no-inbound-log symptom is fixed.

## Best Possible Solution

Keep this issue open for a Matrix owner to reproduce or verify current main or the next release in Docker with local Synapse. Capture Matrix startup logs, `openclaw channels status --probe`, `openclaw matrix verify status --verbose`, room encryption state, and whether inbound events reach `room.event`, `room.message`, `room.decrypted_event`, or `room.failed_decryption`. If current main fixes it, close as implemented with the exact Matrix commit and release; otherwise patch `extensions/matrix` around the isolated startup, sync, encryption, routing, or logging gate.
