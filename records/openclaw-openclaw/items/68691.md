## Summary

Keep this issue open. Current main still creates Docker sandbox containers whose long-lived process is `sleep infinity`, with no `--init`, tini/dumb-init entrypoint, or other PID 1 reaper in the standard sandbox path. The reporter’s follow-up specifically narrowed the observed problem to PID 1 being `sleep infinity`, and the latest release tag still contains the same sandbox command shape.

## Best Possible Solution

Implement a real reaper for Docker sandbox containers, preferably by adding Docker `--init` or an explicit tiny init in the sandbox image/create path, then add a focused regression test asserting sandbox container creation includes the reaper behavior and update sandbox docs/changelog if the runtime contract changes.
