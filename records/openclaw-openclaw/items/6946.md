## Summary

Current main still does not implement a Telegram standalone processing-indicator lifecycle. Telegram has adjacent typing, ack/status reactions, stream previews, and native-command progress placeholders, but there is no accepted `processingIndicator`/`waitingTips` config and no inbound placeholder message id carried through dispatch cleanup. The open #40421 waiting-tips PR overlaps, so this should stay open for maintainer product/config direction.

## Best Possible Solution

Keep this issue open as the canonical bundled Telegram processing-indicator request. The best path is a maintainer decision on how it relates to #40421, then an opt-in Telegram plugin change with typed config/schema/help/docs, dispatch wiring that sends a standalone placeholder after an eligible inbound message is accepted, carries the sent message id through dispatch, deletes it after successful final delivery, and edits or replaces it on failure.
