## Summary

Keep open. Current main still has Telegram typing cues, ack reactions, status reactions, draft streaming, and generic error replies, but it does not accept or drive a standalone processing-placeholder message that is sent while work is running and deleted or edited on completion/failure. The overlapping waiting-tips PR is still open, while the direct processing-indicator PRs were closed unmerged, so this remains a maintainer product/config choice rather than a cleanup-close item.

## What This Changes

Add an opt-in Telegram processing indicator that sends a visible placeholder message while an inbound message is being processed, deletes it after a successful reply, and edits or replaces it on failure.

## Best Possible Solution

Choose the intended bundled Telegram UX, then land an opt-in Telegram plugin change with typed config/schema/help/docs, dispatch wiring that sends and tracks a standalone placeholder after an eligible inbound message is accepted, deletion after successful final delivery, and edit-or-replace behavior on failure with focused regression coverage.
