## Summary

Keep open. Current main still lacks the requested standalone Telegram processing-indicator message lifecycle and does not accept the proposed config keys; adjacent typing, reactions, streaming preview progress, native-command placeholders, and generic error replies do not provide the requested ordinary-turn placeholder that is sent immediately and then deleted or edited. The open waiting-tips PR overlaps the same UX decision, while the direct implementation PRs were closed unmerged, so this needs maintainer product/config direction.

## What This Changes

The issue requests an opt-in Telegram processing indicator message that is sent while a request is being processed, deleted after a successful reply, and edited or replaced on failure.

## Best Possible Solution

Choose the intended bundled Telegram feedback UX, then land an opt-in Telegram plugin change with typed config/schema/help/docs, dispatch wiring that sends and tracks a standalone placeholder after an eligible inbound message is accepted, deletion after successful final delivery, and edit-or-replace behavior on failure with focused regression coverage.
