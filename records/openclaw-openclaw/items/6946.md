## Summary

Keep open. Current main still provides Telegram typing cues, ack reactions, and optional status reactions, but it does not accept or drive a standalone processing-placeholder message that is later deleted or edited. The overlapping waiting-tips PR is open and unmerged, while the direct processing-indicator PRs were closed unmerged, so this needs maintainer product/config direction rather than cleanup closure.

## What This Changes

Add an opt-in Telegram processing indicator that sends a visible placeholder while a request is running, deletes it after a successful reply, and edits or replaces it on failure.

## Best Possible Solution

Land an opt-in Telegram plugin feature with typed config/schema/help/docs, dispatch wiring that sends and tracks a standalone placeholder after an eligible inbound message is accepted, deletion after successful final delivery, and edit-or-replace behavior on failure with focused regression coverage.
