## Summary

Keep open. Current main intentionally migrates legacy `claude-cli/*` model refs to canonical `anthropic/*` refs while recording execution through `agents.defaults.agentRuntime.id: "claude-cli"`; that part is source-, test-, docs-, and changelog-backed. However, the Agents UI and `agents.list` protocol surface still expose only the effective model, not the effective runtime, so the reporter's UI confusion remains actionable even though preserving the legacy model prefix would conflict with the current provider/runtime split.

## Best Possible Solution

Keep the canonical `anthropic/*` model plus `agentRuntime.id: "claude-cli"` behavior, and address the actionable part by surfacing the effective runtime in the Agents UI/status path, ideally by adding a runtime field to `agents.list` and rendering it beside the primary model with tests. Do not reintroduce `claude-cli/*` as the canonical stored model ref unless maintainers explicitly reverse the current provider/runtime design.
