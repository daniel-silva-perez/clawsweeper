## Summary

Keep this PR open. Current main still does not implement the intended Signal quote metadata forwarding: Signal inbound context has no `ReplyToId`, the Signal quote type still omits `id` and `authorNumber`, and quote sender resolution still treats `quote.author` as the phone-number source instead of preferring `authorNumber` and routing UUID-looking values as UUIDs.

## Best Possible Solution

Review and land this focused Signal plugin PR, or apply an equivalent main-branch change that forwards visible signal-cli quote id/body/sender metadata into `ReplyTo*` context, uses `authorNumber`/UUID-aware sender resolution, clears stale reply metadata across debounced merges, and keeps the added regression tests plus changelog entry.
