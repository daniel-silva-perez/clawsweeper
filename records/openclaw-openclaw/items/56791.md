## Summary

Keep this PR open. Current main still does not implement the intended Signal quoted-reply metadata forwarding: Signal does not pass ReplyToId into inbound context, its quote type omits signal-cli id/authorNumber, and sender resolution still treats quote.author as a phone source rather than using authorNumber/UUID-aware routing. The PR discussion is useful: the original Greptile concerns were real, and the later branch repair appears aimed at those exact issues, but the equivalent change is not on main.

## Best Possible Solution

Keep this PR open and either review/land it or apply an equivalent Signal plugin change on main: type signal-cli quote id/authorNumber, route authorNumber before UUID fallback, forward ReplyToId/ReplyToBody/ReplyToSender/ReplyToIsQuote through finalizeInboundContext, clear stale reply metadata on debounced merges, and keep the focused regression tests plus contributor changelog credit.
