## Summary

Keep this PR open. Current main still does not implement the PR’s intended Signal quote metadata forwarding: Signal inbound context has no ReplyToId path, the Signal quote type still omits signal-cli `id` and `authorNumber`, sender resolution still cannot prefer `authorNumber`, and existing Signal tests only cover quote body/sender/isQuote, not the quoted message id or modern signal-cli sender fields.

## Best Possible Solution

Keep this PR open and either review/land it or apply an equivalent Signal plugin change on main: type signal-cli quote id/authorNumber, route authorNumber before UUID fallback, forward ReplyToId/ReplyToBody/ReplyToSender/ReplyToIsQuote through finalizeInboundContext, clear stale reply metadata on debounced merges, and retain the focused regression tests plus contributor changelog credit.
