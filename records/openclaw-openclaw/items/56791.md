## Summary

Keep this PR open. Current main still does not implement the PR’s intended Signal quote metadata forwarding: Signal inbound context has ReplyToBody/Sender/IsQuote only, Signal quote typing still omits signal-cli quote id and authorNumber, and quote sender resolution still treats quote.author as a phone-number source. The PR remains a focused implementation candidate with tests and no CI/supply-chain-sensitive file changes in the diff.

## Best Possible Solution

Review and land this focused Signal plugin PR, or apply an equivalent main-branch fix that forwards visible signal-cli quote id/body/sender metadata into Signal inbound ReplyTo* context, keeps quote metadata behind the existing context-visibility gate, clears stale reply metadata on debounced merges, and preserves the added regression coverage and changelog credit.
