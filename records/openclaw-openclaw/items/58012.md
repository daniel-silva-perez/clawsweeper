## Summary

Keep this issue open. Current main still applies OpenRouter replay policy through the passthrough-Gemini hook, which does not select `toolCallIdMode: "strict9"` for Mistral-family model IDs, while direct Mistral strict9 support already exists and the docs still describe Mistral model-id based strict9 behavior.

## What This Changes

Restore strict9 tool-call-id sanitization for Mistral-family models routed through proxy providers such as OpenRouter after provider replay/capability refactors.

## Best Possible Solution

Add a provider-owned replay-policy path that detects Mistral-family proxy model IDs and selects strict9 tool-call-id sanitization while preserving existing OpenRouter Gemini passthrough behavior, with regression tests for positive Mistral/Codestral-style IDs and negative non-Mistral IDs.

## Repair Work Prompt

Fix the Mistral proxy-provider strict9 tool-call-id regression tracked by openclaw/openclaw#58012. Current main applies OpenRouter replay policy through `PASSTHROUGH_GEMINI_REPLAY_HOOKS`, which does not set `sanitizeToolCallIds` or `toolCallIdMode: "strict9"` for Mistral-family model IDs; direct Mistral already sets strict9. Implement the fix in the current provider-owned replay-policy path, not the removed `src/agents/provider-capabilities.ts` path. Likely approach: add a shared Mistral-family model-id detector/replay helper or compose OpenRouter/proxy replay hooks so `mistralai/...`, `mistral...`, `codestral`, `mixtral`, `pixtral`, `devstral`, and `ministral` style proxy routes resolve strict9, while non-Mistral OpenRouter models and existing Gemini thought-signature cleanup remain unchanged. Add focused regression tests for OpenRouter Mistral/Codestral positive cases and non-Mistral negative cases, plus any shared helper tests. Account for stale related PRs #58014 and #44581; do not revive the deleted provider-capabilities files. Changelog entry only if maintainers consider this user-facing enough; if added, use the active changelog section with contributor attribution per repo policy.
