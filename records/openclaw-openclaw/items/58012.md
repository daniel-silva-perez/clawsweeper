## Summary

Current main still does not apply Mistral strict9 tool-call-id replay policy when a Mistral-family model is routed through proxy providers such as OpenRouter or Together. Native Mistral has strict9, but replay policy is now provider-plugin owned, OpenRouter only wires the passthrough-Gemini replay family, and Together has no replay hook. The issue discussion about the old provider-capabilities PR becoming stale after #60126 matches the current code shape.

## Best Possible Solution

Keep the issue open. Implement a current-architecture fix in the provider replay layer: either a shared provider replay family/helper that detects Mistral-family model ids on OpenAI-compatible proxy providers, or explicit OpenRouter/Together plugin buildReplayPolicy handling, returning sanitizeToolCallIds: true and toolCallIdMode: "strict9" for Mistral-family routes while preserving existing Gemini passthrough behavior. Add focused transcript-policy/plugin tests for OpenRouter + mistral-large-latest, OpenRouter + codestral, a non-Mistral OpenRouter model, and any intended Together/custom-proxy behavior.
