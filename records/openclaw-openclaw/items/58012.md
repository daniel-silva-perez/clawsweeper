## Summary

Keep this open. Current main still routes OpenRouter replay policy through the passthrough-Gemini plugin hook, and that hook only handles Gemini thought-signature cleanup; it does not select strict9 tool-call-id sanitization for Mistral-family model IDs even though direct Mistral replay policy already does.

## What This Changes

Restore strict9 tool-call-id sanitization for Mistral-family models routed through proxy providers such as OpenRouter after provider replay-policy refactors.

## Best Possible Solution

Add a provider-owned replay-policy path, preferably as a shared replay helper composed by OpenRouter, that recognizes Mistral-family wrapped model IDs and selects strict9 tool-call-id sanitization while preserving existing OpenRouter Gemini thought-signature cleanup and turn-validation behavior.
