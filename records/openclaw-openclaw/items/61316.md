## Summary

Keep this PR open. Current main still lowercases the input value but builds custom truthy/falsy sets directly from caller-provided values, so mixed-case custom entries still miss and malformed non-iterable option values can still throw. The proposed regression coverage is also absent, and the related earlier attempt #59535 is closed/unmerged rather than a canonical shipped fix.

## Best Possible Solution

Review and land a focused patch that normalizes custom truthy/falsy arrays case-insensitively, ignores malformed non-array option values without throwing, adds the proposed unit coverage, and decides whether to update the browser plugin’s copied helper in the same change. Address the small review note by filtering custom-list entries to strings before lowercasing.
