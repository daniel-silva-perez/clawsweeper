## Summary

Keep open. The PR is a useful implementation path for the CJK compaction underestimation bug, but it is not merge-ready because the preemptive truncation route still mixes script-aware token overflow with raw-character recovery thresholds; the author has acknowledged that remaining defect in review discussion.

## What This Changes

The PR adds a CJK-aware wrapper around the upstream compaction token estimator, threads it through compaction and preemptive prompt estimation, and adds focused regression tests for ASCII, CJK text, and CJK tool-call arguments.

## Best Possible Solution

Keep this PR as the active implementation path, but rebase it onto current main and update preemptive truncation routing so overflow recovery is compared in a single consistent unit, preferably CJK-aware tokens or weighted chars, with a regression that proves CJK-heavy tool results can route to truncate-only when truncation would fit.

## Reproduction Assessment

Yes. The remaining bug is reproducible by code inspection: the PR makes the pre-prompt overflow estimate CJK-aware at the changed line, while the existing route still multiplies that token count by 4 and compares it with raw reducible tool-result characters.

## Solution Assessment

No, not yet. The high-level CJK-aware estimator direction is right, but the proposed patch is incomplete until the preemptive route compares recovery and overflow in the same script-aware units.
