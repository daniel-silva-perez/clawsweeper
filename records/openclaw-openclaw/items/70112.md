## Summary

Keep open. Current main still uses the upstream CJK-blind token estimator in the compaction and preemptive-compaction paths, so this PR is not obsolete; however the proposed patch still has a concrete preemptive-routing bug that should be fixed before merge.

## What This Changes

The PR adds a CJK-aware `estimateTokens` wrapper, threads it through agent compaction and preemptive compaction estimates, and adds focused regression tests for ASCII, CJK text, and CJK inside toolCall arguments.

## Best Possible Solution

Keep the PR open as the active implementation path for the CJK compaction estimator, but require one more update that keeps preemptive truncation routing script-aware end to end, preserves current main's transcript-sanitizing behavior during rebase, and lands with focused regression coverage.
