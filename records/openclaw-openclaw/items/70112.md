## Summary

Keep open. The CJK estimator fix is still needed because current main still sends compaction through the upstream chars/4 estimator, and the contributor addressed the earlier lint, toolCall, and test feedback. The PR is not merge-ready yet because it makes preemptive prompt estimation script-aware while the truncation route still compares recovery through a fixed raw-character conversion, which can cause unnecessary compaction for CJK-heavy tool results.

## What This Changes

The PR adds an `estimateTokensCjkAware` wrapper in agent compaction, threads it through embedded compaction and preemptive prompt estimation, and adds focused compaction tests for ASCII, CJK text, and CJK toolCall arguments.

## Best Possible Solution

Land the CJK-aware compaction estimator after updating preemptive truncation routing so overflow and tool-result recovery are compared in the same token-aware or weighted-character units, with a regression test covering CJK-heavy tool-result recovery and the current main sanitization guards preserved.
