## Summary

Keep open. The current heartbeat path still calls the normal reply path with `isHeartbeat: true` and no `heartbeatModelOverride` when `heartbeat.model` is unset; that reply path applies stored session overrides when no explicit heartbeat model was resolved. Current tests also assert that behavior, and model-failover docs say auto fallback overrides remain selected until reset. That leaves a plausible heartbeat-specific stale auto-fallback path after `agents.defaults.model.primary` changes, separate from the closed reply-path stale runtime-field issue in #51677.

## Best Possible Solution

Keep this open and fix the heartbeat/default-model-change path narrowly: when a heartbeat run has no explicit `heartbeat.model`, an auto-sourced fallback override whose original selected model no longer matches the current effective agent default should be ignored or cleared before model selection, while user overrides and unchanged auto fallback state remain intact. Add a regression test for the `agent:main:main` heartbeat session after `agents.defaults.model.primary` changes, and update docs/changelog only if the visible behavior changes.
