## Summary

Keep open. Current main still has the reported split-state behavior: the shared state directory resolver honors `OPENCLAW_STATE_DIR`, but exec approval file and socket defaults still resolve to hard-coded `~/.openclaw` paths, and the related fix PRs were closed unmerged.

## What This Changes

The issue asks for `exec-approvals.json` and the exec approval socket defaults to live under the configured `OPENCLAW_STATE_DIR` instead of creating `~/.openclaw`.

## Best Possible Solution

Route only the default exec approval file and socket locations through `resolveStateDir()`, preserve explicit `socket.path` values, define a legacy read or migration path for existing home-based approval files, and update the policy docs plus focused regression tests so doctor, CLI, gateway, and node-host behavior agree.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: with `OPENCLAW_STATE_DIR=~/openclaw`, `resolveStateDir()` resolves the override, while `ensureExecApprovals()` and `saveExecApprovals()` still write through `resolveExecApprovalsPath()` to `~/.openclaw/exec-approvals.json`.

## Solution Assessment

No current solution is in place. The narrow maintainable fix is to make the default exec approval file/socket state-dir-backed while explicitly handling existing home-based approval files so security policy is not lost.
