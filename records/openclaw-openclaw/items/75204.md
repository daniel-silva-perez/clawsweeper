## Summary

Keep open. Current main still honors `OPENCLAW_STATE_DIR` for the general state directory but keeps exec approval file and socket defaults hard-coded under `~/.openclaw`, so the reported split-state behavior remains. Two related fix PRs, #75210 and #75222, were closed unmerged, so there is no merged implementation to close against.

## What This Changes

The issue requests that `exec-approvals.json` respect `OPENCLAW_STATE_DIR` and be stored under the active OpenClaw state directory instead of `~/.openclaw`.

## Best Possible Solution

Move only the default exec approval file and socket paths to `path.join(resolveStateDir(), ...)`, preserve explicit configured socket paths, define a legacy read or migration path for existing home-based approval files, and update approval docs plus regression tests so storage and doctor behavior agree.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: with `OPENCLAW_STATE_DIR` set, `resolveStateDir()` resolves the override while `ensureExecApprovals()` and `saveExecApprovals()` still write through home-based exec approval path resolvers.

## Solution Assessment

No current solution is merged. The maintainable fix is narrower than a broad state migration: change the default exec approval store/socket resolution, then handle legacy-file compatibility and tests explicitly.
