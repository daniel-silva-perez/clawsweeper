## Summary

Keep open. Current main still honors `OPENCLAW_STATE_DIR` for the general state directory but the TypeScript exec-approval store resolves its file and socket from hard-coded `~/.openclaw` defaults, so the reported split-state behavior is still present.

## What This Changes

The issue asks for `exec-approvals.json` to be stored under the configured `OPENCLAW_STATE_DIR` instead of always under `~/.openclaw`.

## Best Possible Solution

Align the TypeScript exec approval file and socket defaults with `resolveStateDir()`, add regression coverage for `OPENCLAW_STATE_DIR`, and update approval docs and any policy-source labels after deciding how to migrate or read existing home-based approval files safely.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: with `OPENCLAW_STATE_DIR` set, `resolveStateDir()` points at the override, but `ensureExecApprovals()` and `saveExecApprovals()` still use `resolveExecApprovalsPath()`, which expands a hard-coded `~/.openclaw/exec-approvals.json`.

## Solution Assessment

No current solution is in place. The narrow maintainable fix is state-dir-backed path resolution plus a compatibility plan for existing approval files and socket paths.
