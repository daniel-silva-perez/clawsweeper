## Summary

Keep open. Current main still filters Matrix multi-account bot traffic by sender and resolves mentions only for the current account, so a human room message that explicitly mentions another configured Matrix bot can still dispatch for an account with `requireMention: false`. The related `allowBots` work and open Matrix participation PR do not implement this narrow pre-dispatch skip.

## What This Changes

Request an opt-in Matrix behavior that skips dispatch for non-mentioned accounts when a room message explicitly mentions a different configured bot account, avoiding typing indicators and LLM calls in multi-account rooms.

## Best Possible Solution

Add a documented, backward-compatible Matrix account/room opt-in that runs before typing, media/history work, session recording, and LLM dispatch. It should skip only when a visible mention targets another configured Matrix bot account while preserving self mentions, DMs, commands, `@room`, and existing sender-based `allowBots` behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction path is a Matrix room event from a human sender with `m.mentions.user_ids` containing another configured bot, current account `requireMention: false`, and no self mention; current code does not drop it before dispatch.

## Solution Assessment

Yes, a narrow deterministic Matrix plugin policy is the best fit. Prompt-only suppression or a late AI participation/freshness gate would still allow typing and token usage, which is the user-visible problem here.
