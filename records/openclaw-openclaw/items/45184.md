## Summary

Keep open. Current main still only filters Matrix bot-to-bot traffic by sender and only gates mentions against the current account; a human-authored room message that visibly mentions another configured Matrix bot can still reach LLM dispatch when this account has `requireMention: false`.

## What This Changes

Add an opt-in Matrix policy to skip pre-dispatch handling when a room message explicitly mentions a different configured bot account in the same OpenClaw instance.

## Best Possible Solution

Matrix should add a documented, backward-compatible opt-in room/account policy that runs before typing, session recording, media work, and LLM dispatch; it should detect visible mentions of configured Matrix bot user IDs other than the current account while preserving self mentions, `@room`, DMs, authorized control commands, and the existing `allowBots` sender policy.
