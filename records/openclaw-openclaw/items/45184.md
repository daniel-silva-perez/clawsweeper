## Summary

Keep open. Current main handles Matrix bot-authored messages with `allowBots` and standard self-mention gating, but it does not implement the requested opt-in skip for human messages explicitly addressed to a different configured Matrix account. The closed related Matrix bot-to-bot visibility issue led to adjacent `allowBots` work, but that does not solve this dispatch suppression case.

## What This Changes

Add an opt-in Matrix room/account setting so non-mentioned Matrix accounts skip dispatch when a room message explicitly mentions another configured bot account.

## Best Possible Solution

Matrix should expose a documented opt-in room/account policy that runs before typing, session recording, and LLM dispatch, detects visible mentions of configured Matrix bot user IDs other than the current account, preserves normal self mentions, @room mentions, DMs, and control-command behavior, and leaves existing `allowBots` sender filtering intact.
