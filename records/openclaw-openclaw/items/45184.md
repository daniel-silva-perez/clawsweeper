## Summary

Keep open. Current main still lacks a Matrix pre-dispatch gate for human room messages that explicitly mention a different configured Matrix bot account; the existing Matrix logic only gates messages sent by configured bot accounts and whether the current account was mentioned.

## What This Changes

The issue requests an opt-in Matrix routing policy that skips LLM dispatch for non-mentioned bot accounts when a room message explicitly mentions a different configured Matrix bot account.

## Best Possible Solution

Add a backward-compatible Matrix account and per-room opt-in that runs before typing, media/history work, session recording, and LLM dispatch. It should skip only when a visible explicit mention targets another configured Matrix bot account and the current account is not mentioned, while preserving existing command, DM, @room, self-loop, and allowBots semantics.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a Matrix room event from a human sender whose m.mentions.user_ids or visible Matrix mention targets another configured bot, while the current account has requireMention false and no self mention; current main has no gate that drops that event before dispatch.

## Solution Assessment

Yes. A narrow Matrix plugin policy is the best way to solve the typing/token problem; prompt-only suppression or a late AI participation gate would still allow unnecessary dispatch work.
