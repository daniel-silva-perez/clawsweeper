## Summary

Keep open. Current main still only filters Matrix messages by sender through `allowBots` and only resolves whether the current account was mentioned; a human-authored room message that visibly mentions another configured Matrix bot can still pass when this account has `requireMention: false` and reach reply dispatch. The related closed Matrix bot-to-bot issue #45450 was adjacent, but it addressed visibility of bot-authored messages rather than suppressing dispatch for messages addressed to a different bot.

## What This Changes

Add an opt-in Matrix room/account setting so non-mentioned Matrix accounts skip dispatch when a room message explicitly mentions another configured bot account.

## Best Possible Solution

Matrix should expose a documented opt-in room/account policy that runs before typing, session recording, and LLM dispatch, detects visible mentions of configured Matrix bot user IDs other than the current account, preserves normal self mentions, `@room`, DMs, and authorized control commands, and leaves existing `allowBots` sender filtering intact.
