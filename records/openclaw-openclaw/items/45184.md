## Summary

Keep open. Current main still has only sender-scoped Matrix bot filtering and current-account mention detection, so a human-authored room message that mentions another configured Matrix account can still reach dispatch when this account has `requireMention: false`. The closed `allowBots` work for #45450 helps bot-authored Matrix traffic, but it does not implement this other-bot-mention skip policy.

## What This Changes

Add an opt-in Matrix routing policy that skips dispatch for non-mentioned bot accounts when a room message explicitly mentions a different configured Matrix account.

## Best Possible Solution

Matrix should gain a documented, backward-compatible opt-in account/room policy that runs before typing, media fetches, session recording, history consumption, and LLM dispatch. It should skip only when a visible mention targets another configured Matrix bot account while preserving self mentions, DMs, commands, room mentions, and existing sender-based `allowBots` behavior.
