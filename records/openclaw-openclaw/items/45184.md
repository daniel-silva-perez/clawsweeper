## Summary

Keep open. Current main still has only sender-based Matrix bot filtering through `allowBots` plus current-account mention gating through `requireMention`; a human-authored room message that mentions another configured Matrix bot can still reach reply dispatch when this account has `requireMention: false`. The adjacent closed Matrix issue #45450 addressed visibility of bot-authored messages, not pre-dispatch suppression for messages addressed to a different configured account.

## What This Changes

Add an opt-in Matrix policy that skips LLM dispatch before typing when a room message explicitly mentions another configured Matrix bot account but not the current account.

## Best Possible Solution

Matrix should add a documented, backward-compatible, opt-in per-account/per-room policy that runs before typing, session recording, and LLM dispatch; it should detect visible mentions of configured Matrix bot user IDs other than the current account, preserve self mentions, `@room`, DMs, and authorized control commands, and leave existing `allowBots` sender filtering intact.
