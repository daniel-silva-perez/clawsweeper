## Summary

Keep open. Current main still lacks an editable, durable paired-device naming workflow: it displays client-supplied `displayName` or the raw device id, but has no operator-owned label field, no paired-device rename RPC/schema/handler, and no CLI or Control UI rename action. The related #23916 is broader device metadata display work, while closed #57424 was folded into this device-distinguishability cluster rather than superseding the rename request.

## What This Changes

Add human-friendly, user-editable names for paired devices in the paired-device list, CLI, and Control UI.

## Best Possible Solution

Add an additive paired-device label workflow: persist an operator-owned label separate from client `displayName`, define display precedence as operator label then client `displayName` then raw `deviceId`, expose a typed `device.pair.rename` Gateway method with authorization, add CLI and Control UI rename actions, and document the behavior without merging it into broader automatic metadata display.
