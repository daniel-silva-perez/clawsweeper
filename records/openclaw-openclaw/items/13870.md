## Summary

Keep open. Current main still only stores/displays client-supplied device `displayName` metadata or the raw `deviceId`; it does not provide the requested durable operator-owned label, paired-device rename protocol/handler, CLI command, or Control UI action. The related #23916 remains broader metadata-display work and does not supersede this rename workflow.

## What This Changes

Request to let operators assign and display durable human-friendly labels for paired devices in the CLI and Control UI instead of relying on hash IDs or client metadata.

## Best Possible Solution

Add an additive paired-device label workflow with storage for an operator-owned label, explicit display precedence over client metadata, a `device.pair.rename` Gateway method, matching CLI and Control UI rename actions, docs, and focused tests; keep broader automatic metadata display under #23916.
