## Summary

Current main still lacks the requested operator-owned paired-device rename workflow. It displays client-supplied displayName metadata in the CLI and Control UI, but there is no devices rename CLI command, no gateway rename method/schema/handler, and no persisted user label with precedence over reconnect metadata.

## Best Possible Solution

Implement paired-device label management end to end: add an additive gateway method/schema/handler for paired-device rename, persist an operator-owned label or override with documented precedence over reconnect metadata, expose `openclaw devices rename --device <id> --name <displayName>`, add a Control UI rename action, cover storage/protocol/CLI/UI behavior with focused tests, and update public devices/protocol docs. Keep `node.rename` separate and leave #23916 focused on broader metadata display.
