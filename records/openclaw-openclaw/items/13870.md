## Summary

Current main still lacks the requested operator-owned paired-device rename workflow. It displays client-supplied `displayName` metadata in CLI and Control UI lists, but there is no `openclaw devices rename` command, no `device.pair.rename` gateway method/schema/handler, and no persisted label field with precedence over reconnect metadata.

## Best Possible Solution

Implement paired-device label management end to end: add an additive `device.pair.rename` method/schema/handler, persist an operator-owned label or override with documented precedence over reconnect metadata, expose `openclaw devices rename --device <id> --name <displayName>`, add a Control UI rename action, cover storage/protocol/CLI/UI behavior with focused tests, and update the devices/protocol docs. Keep `node.rename` separate and leave #23916 focused on broader metadata display.
