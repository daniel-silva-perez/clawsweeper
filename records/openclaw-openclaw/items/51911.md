## Summary

Keep open. Current main still writes Anthropic setup-token and paste-token credentials only to the selected agent store, while sibling-agent fan-out exists only on the OAuth helper path. PR #52912 is still open, unmerged, and uses closing syntax for this issue, so policy says this issue should remain open until that implementation candidate lands or is closed.

## Best Possible Solution

Keep this issue open and review PR #52912. If landing it, make sure the implementation covers all current token persistence surfaces that can create Anthropic setup-token/manual profiles, adds regression coverage for sibling agent stores, and follows up with setup-token prompt/docs clarity plus post-write validation so successful onboarding cannot leave missing auth profiles.
