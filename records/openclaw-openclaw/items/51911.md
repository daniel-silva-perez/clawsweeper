## Summary

Keep open. The report is still live on current main: Anthropic setup-token / paste-token persistence writes only the selected agent directory, while sibling-agent sync exists only on the OAuth helper path. Also, PR #52912 remains open and unmerged with GitHub closing syntax for this issue, so policy says the issue should stay open until that implementation candidate lands or is closed.

## Best Possible Solution

Keep the issue open. Review and either land or close PR #52912; if landing, make sure the fix covers the actual token persistence surfaces in current main, including interactive onboarding/provider auth, non-interactive Anthropic setup-token, and models paste-token as appropriate. Add regression coverage for multi-agent sibling stores, then follow up with setup-token prompt/docs clarity and post-write validation so successful onboarding cannot leave auth-profiles.json missing or stale.
