## Summary

Keep this PR open. Current main does not implement the PR’s stated startup hardening: there is no `normalizeStateDirEnv` helper, the gateway CLI path does not normalize `OPENCLAW_STATE_DIR` / `CLAWDBOT_STATE_DIR` before config loading, and direct `startGatewayServer()` startup also lacks that normalization. The related bug report #52250 is closed, but this PR is a separate follow-up hardening change rather than work already shipped on main.

## Best Possible Solution

Keep the PR open for review or ask the author to rebase and narrow it. The best path is a focused startup-path normalization patch on current main, with tests proving relative `OPENCLAW_STATE_DIR` and legacy `CLAWDBOT_STATE_DIR` are pinned before cwd changes, plus a clear decision on the related `OPENCLAW_CONFIG_PATH` review comment and removal or regeneration of any unrelated generated SDK baseline drift.
