## Summary

Keep open. Current main still evaluates the image, video, and music provider lists before the shared resolver can short-circuit on a complete explicit modelConfig, and #74097 is still the paired open implementation candidate for the narrow lazy-provider fix.

## What This Changes

The issue reports that image/video/music generation tool model-config resolution eagerly lists runtime capability providers before the explicit modelConfig short-circuit, adding plugin registry reload latency.

## Best Possible Solution

Review and land #74097, or an equivalent narrow fix, so resolveCapabilityModelConfigForTool receives a provider thunk, invokes it only after complete explicit modelConfig resolution fails, updates the image/video/music callers, and adds regression coverage for complete and incomplete config paths.

## Reproduction Assessment

Yes. Source inspection on current main shows each tool evaluates listRuntime*GenerationProviders before calling the shared resolver, while the complete explicit modelConfig short-circuit happens inside that resolver; the issue also includes instrumented stack traces for the runtime reload path.

## Solution Assessment

Yes. Lazily passing providers is the narrowest maintainable fix for this tool-level waste because it preserves candidate fallback behavior for incomplete configs while avoiding provider resolution when explicit config already resolves.
