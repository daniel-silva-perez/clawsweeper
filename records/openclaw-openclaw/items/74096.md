## Summary

Keep open. Current main still evaluates image, video, and music provider lists before the shared resolver can return a complete explicit modelConfig, and the paired open fix PR #74097 still references this issue with closing syntax, so this should not be closed before that PR merges or is replaced.

## What This Changes

The issue requests avoiding eager image/video/music capability-provider listing during media tool model-config resolution when explicit modelConfig is already complete.

## Best Possible Solution

Review and land #74097, or an equivalent narrow patch, so the shared resolver accepts a provider thunk, invokes it only after explicit modelConfig fails, updates the three media tool callers, and adds regression coverage for both complete explicit config and incomplete candidate resolution.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: with complete agents.defaults image/video/music modelConfig, the three callers still evaluate listRuntime*GenerationProviders before resolveCapabilityModelConfigForTool can return the explicit config.

## Solution Assessment

Yes. The lazy provider thunk proposed by the paired PR is the narrowest maintainable fix because it preserves provider-based candidate resolution when modelConfig is incomplete while avoiding provider discovery when explicit config is already complete.
