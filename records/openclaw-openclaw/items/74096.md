## Summary

Keep open. Current main still evaluates image/video/music provider lists before the shared resolver can short-circuit on a complete explicit modelConfig, and #74097 is the paired open implementation candidate with closing syntax, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests lazy provider listing for image/video/music tool model-config resolution so explicit complete modelConfig paths avoid unnecessary runtime capability-provider discovery.

## Best Possible Solution

Review and land #74097, or an equivalent narrow patch, so the shared resolver accepts a provider thunk, invokes it only after explicit modelConfig fails, updates the three media tool callers, and keeps regression coverage for both explicit complete config and incomplete candidate resolution.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure complete `agents.defaults.imageGenerationModel`, `videoGenerationModel`, and `musicGenerationModel` with `plugins.entries` present, then call the three `resolve*GenerationModelConfigForTool` wrappers; current main still evaluates each `listRuntime*GenerationProviders` call before the explicit-config return path can run.

## Solution Assessment

Yes. The lazy provider thunk proposed by the paired PR is the narrowest maintainable solution because it preserves provider-based candidate resolution when modelConfig is incomplete while avoiding provider discovery on the complete explicit-config path.
