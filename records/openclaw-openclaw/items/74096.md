## Summary

Keep open. Current main still evaluates image/video/music provider lists before the shared resolver can short-circuit on a complete explicit modelConfig, and PR #74097 remains the paired open implementation candidate with closing syntax.

## What This Changes

The issue requests deferring image/video/music capability provider listing during media tool model-config resolution so complete explicit modelConfig paths avoid unnecessary plugin capability provider lookups.

## Best Possible Solution

Land PR #74097, or an equivalent narrow change, so resolveCapabilityModelConfigForTool accepts a lazy provider supplier and only lists providers when explicit modelConfig is incomplete, with regression coverage for both paths.
