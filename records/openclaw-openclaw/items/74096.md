## Summary

Keep open. Current main still requires eager provider arrays in the shared media tool resolver, and the image/video/music callers still compute those arrays before the explicit modelConfig short-circuit can return. The issue is also paired with open implementation PR #74097, so the issue should remain open until that PR, or an equivalent fix, is reviewed and merged.

## What This Changes

The issue requests deferring image/video/music capability-provider listing during media tool model-config resolution when an explicit complete modelConfig already satisfies the tool.

## Best Possible Solution

Land #74097 or an equivalent narrow patch that changes the shared resolver to accept a provider thunk, invokes it only after explicit modelConfig fails, updates the three media tool callers, and keeps regression coverage for both complete and incomplete config paths.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: with complete agents.defaults image/video/music modelConfig, each media tool caller still evaluates listRuntime*GenerationProviders before resolveCapabilityModelConfigForTool can return the explicit config.

## Solution Assessment

Yes. The lazy provider thunk is the narrowest maintainable fix because it preserves provider-based candidate resolution for incomplete modelConfig while avoiding provider discovery when explicit config is already sufficient.
