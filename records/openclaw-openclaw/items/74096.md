## Summary

Keep open. Current main still evaluates the image/video/music provider lists before the shared resolver can short-circuit on a complete explicit modelConfig, and #74097 remains the paired open implementation candidate with closing syntax for this issue.

## What This Changes

The issue requests avoiding eager image, video, and music capability-provider listing when an agent already has a complete explicit media modelConfig.

## Best Possible Solution

Review and land #74097, or an equivalent narrow patch, so the shared resolver accepts a provider thunk, invokes it only after explicit modelConfig fails, updates the three media tool callers, and keeps regression coverage for both complete explicit config and incomplete config resolution.

## Reproduction Assessment

Yes. The source-level path is high confidence: configure complete agents.defaults image/video/music modelConfig, then the three callers still evaluate listRuntime*GenerationProviders before resolveCapabilityModelConfigForTool can return the explicit config.

## Solution Assessment

Yes. The lazy provider thunk proposed in the paired PR is the narrowest maintainable fix because it preserves candidate resolution when modelConfig is incomplete while avoiding provider discovery on the explicit-config path.
