## Summary

Keep open. Current main still eagerly evaluates image/video/music provider lists before the shared resolver can short-circuit on a complete explicit modelConfig, and #74097 remains the paired open implementation candidate with closing syntax.

## What This Changes

The issue requests lazy provider listing for image/video/music tool model-config resolution so complete explicit modelConfig paths avoid unnecessary plugin capability lookups.

## Best Possible Solution

Keep this issue open while #74097 remains open. The best path is to review and either land that PR or an equivalent narrow fix that changes the shared media-tool resolver to accept a provider thunk, invokes it only after complete explicit modelConfig fails, updates the image/video/music callers, and keeps regression coverage for both complete explicit config and incomplete fallback paths.
