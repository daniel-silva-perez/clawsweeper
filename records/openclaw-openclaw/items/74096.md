## Summary

Keep open. Current main still evaluates the image/video/music provider lists before the shared resolver can short-circuit on a complete explicit modelConfig, and PR #74097 is still the paired open implementation candidate with closing syntax for this issue.

## What This Changes

The issue requests a performance fix so image/video/music generation tool model-config resolution defers capability provider listing until provider candidates are actually needed.

## Best Possible Solution

Keep this issue open while PR #74097 remains open. The best path is to review and land that PR, or an equivalent narrow fix that changes resolveCapabilityModelConfigForTool to accept a provider thunk, invokes it only after explicit modelConfig fails, updates the image/video/music callers, and adds regression coverage for both complete explicit config and incomplete config paths.
