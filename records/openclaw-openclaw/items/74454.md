## Summary

Keep open. Current main still has the env-derived `reg.exe` lookup and does not block `SystemRoot`/`WINDIR` from workspace dotenv files, while this PR appears to address that focused security gap with source and test updates. It should continue through maintainer/security review rather than cleanup closure, especially while required CI is still incomplete.

## What This Changes

The PR blocks `SystemRoot` and `WINDIR` in workspace `.env` loading and changes Windows registry probing so `reg.exe` is resolved from a fixed Windows system path, with matching unit tests.

## Best Possible Solution

Land a maintainer-approved version that prevents workspace `.env` from controlling Windows system-root variables and prevents registry probing from executing workspace-controlled binaries, while preserving the intended Windows install-root compatibility contract and keeping the focused regression tests.
