## Summary

Keep open. Current main still only carries iMessage reply context when the incoming payload already has resolved `reply_to_*` fields; it does not accept GUID-only inline reply metadata or enrich from `chat.db`. The issue discussion and open PR #42683 remain the active maintainer path, so this should not be closed or routed to a duplicate repair lane.

## What This Changes

The issue requests that iMessage inbound inline replies include the original replied-to message context before dispatching the message to the agent.

## Best Possible Solution

Land or replace the existing iMessage-focused fix so the bundled iMessage plugin resolves reliable inline-reply origin metadata into `ReplyToId`, `ReplyToBody`, and `ReplyToSender` before dispatch. Keep the behavior in `extensions/imessage/src/monitor/*`, use the existing `dbPath` surface for any read-only fallback, and add regression coverage for GUID-only replies, missing quoted rows, absent `dbPath`, and old-schema or failing `chat.db` lookups.
