## Summary

Keep open. Current main still has the unresolved Gemini signature-bloat and tagged-reasoning delivery paths; the mixed text-before-tool path has partial flush protection, but that does not cover the main replay/signature and final-tag suppression problems.

## What This Changes

The issue reports Gemini 2.5 Pro session failures caused by large stored/replayed text signatures, embedded think-tag handling, and mixed text/tool responses dropping visible text.

## Best Possible Solution

Handle this at the Google provider and replay boundary: preserve only Gemini signatures required for valid continuation, avoid unbounded persistence/replay of visible text signatures, and align Gemini reasoning delivery with native thinking or a safe provider-specific fallback so final-less tagged output cannot silently suppress user replies.

## Reproduction Assessment

Yes for a high-confidence source-level reproduction: a same-provider Google assistant text block with textSignature is serialized back to Gemini as thoughtSignature, and enforced final-tag delivery suppresses text without a <final> start tag. The live Gemini plus Telegram end-to-end path was not run because this was a read-only review.

## Solution Assessment

No current-main solution is complete. The mixed text/tool flush protection is useful, but the remaining signature-retention and Gemini tagged/native reasoning choices need a provider-owner decision before a narrow implementation should proceed.
