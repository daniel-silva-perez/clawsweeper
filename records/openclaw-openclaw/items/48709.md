## Summary

Keep open. Current main still has the core Gemini signature-bloat path: Google visible text blocks can persist `textSignature`, same-provider replay sends it back as `thoughtSignature`, and the generic signature sanitizer does not target that stored field. The mixed text/tool delivery path is partly improved by block flushing and final-answer fallback, but tagged Gemini reasoning and strict no-`<final>` suppression still leave meaningful parts of the report unresolved.

## What This Changes

The issue reports Gemini 2.5 Pro session failures from persisted Google text signatures, embedded reasoning tags in visible text, and dropped visible text around mixed text/tool responses.

## Best Possible Solution

Handle this in the Google provider/session replay boundary: preserve required Gemini function-call signatures, bound or externalize stale text-only signatures so they do not grow replay context, and align Gemini reasoning delivery with the native-thinking direction so final-less or malformed `<think>` output cannot silently suppress user replies.
