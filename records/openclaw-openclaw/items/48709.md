## Summary

Keep this issue open. Current main still persists Gemini text-part signatures as `textSignature` and replays them as Google `thoughtSignature`; the sanitizer does not target that stored field. Mixed text/tool delivery is partly covered now, but the signature-bloat path and strict no-`<final>` suppression remain unresolved.

## What This Changes

The issue requests Gemini 2.5 Pro session handling changes so text signatures do not bloat/replay indefinitely, embedded `<think>` output does not suppress visible replies, and mixed text plus tool-call responses still deliver user-visible text.

## Best Possible Solution

Handle this as a provider-owned Gemini session/replay policy: preserve required Google function-call signatures, bound or externalize stale text-only `textSignature` data so JSONL/context cannot balloon, and align the remaining `<think>`/`<final>` behavior with the native Gemini reasoning direction already tracked in related issues.
