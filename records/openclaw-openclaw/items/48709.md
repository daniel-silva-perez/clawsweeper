## Summary

Keep open. Current main still has the central Gemini failure paths: visible-text `textSignature` values can be persisted and replayed to Gemini, the replay sanitizer does not target that field, and direct Gemini still uses tagged/final-tag reasoning. Mixed text before tool execution now has flush protection, but that only partially addresses the report.

## What This Changes

The issue requests fixing Gemini 2.5 Pro failures caused by persisted/replayed `textSignature` bloat, embedded `<think>` tags, and mixed text/tool turns dropping user-visible output.

## Best Possible Solution

Handle this at the Google provider and replay boundary: retain only Gemini signatures required for valid continuation, avoid replaying stale visible-text signatures into future turns, and align Gemini reasoning delivery with native thinking or a provider-specific fallback so malformed tagged output cannot silently suppress replies.
