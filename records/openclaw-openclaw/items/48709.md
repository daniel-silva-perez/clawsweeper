## Summary

Keep this issue open. Current main still stores Gemini text-part `thoughtSignature` values as assistant `textSignature` and replays them to the same Google model, while the replay sanitizer only strips `thought_signature`/`thoughtSignature` fields and does not bound persisted `textSignature` bloat. The mixed text/tool delivery path now has flush and fallback coverage, but the signature-bloat and strict tagged-reasoning no-`<final>` suppression parts remain unresolved.

## What This Changes

The issue requests Gemini 2.5 Pro session handling changes so text signatures do not bloat/replay indefinitely, embedded `<think>` output does not suppress visible replies, and mixed text plus tool-call responses still deliver user-visible text.

## Best Possible Solution

Keep this issue open and handle it as a provider-owned Gemini replay/session fix: preserve required current-turn Google function-call signatures, bound or externalize stale text-only `textSignature` data so JSONL and context cannot balloon, and adjust tagged-reasoning delivery so visible answer text after `<think>` is not silently dropped when Gemini omits `<final>`. Keep the existing mixed text/tool flush behavior and add targeted regression coverage across Google transport replay, session-history sanitization, final-tag delivery, and payload fallback.
