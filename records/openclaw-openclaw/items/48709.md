## Summary

Keep this issue open. Current main still persists Gemini text-part thought signatures as `textSignature` and replays them for the same Google model, while the replay sanitizer does not bound or externalize those stored text signatures. The mixed text/tool delivery concern appears partly covered now, but the signature-bloat and tagged-reasoning/final-tag delivery risks still need maintainer design.

## Best Possible Solution

Keep this open and handle it as a provider-owned Gemini replay/session design: preserve required current-turn function-call signatures exactly, bound or externalize stale text-only signatures so JSONL/context cannot balloon, and adjust Gemini tagged-reasoning delivery so visible answer text after `<think>` is not silently dropped when no `<final>` tag is present. Add targeted tests for native Google textSignature replay, old-turn signature pruning, tagged reasoning delivery, and mixed text/tool final delivery.
