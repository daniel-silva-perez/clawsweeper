## Summary

Keep this issue open. Current main has partial mitigations for tagged reasoning delivery and mixed text/tool streaming, but the central Gemini `textSignature` bloat path is still present: Google SSE `thoughtSignature` values on normal text are persisted as `textSignature`, same-model replay sends them back as Google `thoughtSignature`, and the Google replay sanitizer strips `thoughtSignature`/`thought_signature` fields but not `textSignature`. The issue discussion's session-bloat and compaction-pressure concern is therefore not fully resolved.

## Best Possible Solution

Keep the issue open and fix the Google provider/replay path. The likely maintainer direction is to decide whether Gemini text signatures should be dropped, capped, externalized, or retained only for replay-critical recent turns, then add focused regression coverage showing large `textSignature` values do not accumulate or get resent unbounded while valid Gemini tool/thinking signature replay still works.
