## Summary

Keep open. Current main and the latest release still preserve and replay Google text-part signatures, and Gemini still uses tagged reasoning with strict final-tag suppression. The mixed text-before-tool path now has block-flush protection, but the remaining signature-bloat and tagged-reasoning failure modes are still valid.

## What This Changes

The issue reports Gemini 2.5 Pro session failures from stored/replayed textSignature data, embedded reasoning tags that suppress replies, and visible text being dropped around mixed text/tool responses.

## Best Possible Solution

Handle this in the Google provider and replay boundary: preserve required Gemini function-call/native-thinking signatures, bound or avoid replaying stale visible-text signatures, and move Gemini reasoning delivery toward native thinking so malformed or final-less tagged output cannot silently suppress user replies.
