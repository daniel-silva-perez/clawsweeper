## Summary

Keep open. Current main still has the central Gemini signature-bloat path: Google visible text blocks can persist `textSignature`, same-provider replay sends it back as `thoughtSignature`, and the replay sanitizer only targets `thought_signature` / `thoughtSignature`. Mixed text before tool execution has partial protection now via block flushing, but Gemini remains on tagged reasoning with strict final-tag suppression, so meaningful parts of the report are still unresolved rather than stale or implemented.

## What This Changes

The issue requests Google/Gemini provider and session handling changes to prevent `textSignature` replay bloat, avoid `<think>`/`<final>` delivery suppression, and preserve visible text from mixed text-plus-tool responses.

## Best Possible Solution

Handle this at the Google provider and replay boundary: preserve required Gemini function-call/native-thinking signatures, bound or avoid replaying stale visible-text signatures, and align Gemini reasoning delivery with native thinking so malformed or final-less tagged output cannot silently suppress user replies.
