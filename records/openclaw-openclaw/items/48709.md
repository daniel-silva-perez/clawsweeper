## Summary

Keep open. Current main still preserves Gemini visible-text signatures in assistant history, replays those stored `textSignature` values back to Google as `thoughtSignature`, and the generic replay sanitizer still does not target the stored `textSignature` field. The mixed text-before-tool path now has partial protection through block flushing, but the remaining signature-bloat path and Gemini tagged-reasoning/final-tag suppression behavior mean the central report is still valid rather than stale or implemented.

## What This Changes

The issue reports that Gemini 2.5 Pro sessions can bloat and fail because visible-text signatures are persisted/replayed, embedded `<think>` tags can suppress delivery, and mixed text plus tool-call turns can drop text.

## Best Possible Solution

Handle this at the Google provider and replay boundary: preserve only provider-required Gemini function-call/native-thinking signatures, bound or avoid replaying stale visible-text signatures, and align Gemini reasoning delivery with native thinking so malformed or final-less tagged output cannot silently suppress user replies.
