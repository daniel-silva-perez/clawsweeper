## Summary

Keep open. Current main still preserves and replays Gemini visible-text signatures, the generic signature sanitizer does not target the stored `textSignature` field, and Google Gemini still combines native thinking request configuration with tagged reasoning/final-tag enforcement. The mixed text-before-tool path has partial protection through block flushing, so the report is narrower now, but the central signature-bloat and Gemini reasoning-suppression problems are not implemented away or fully superseded.

## What This Changes

The issue asks OpenClaw to prevent Gemini 2.5 Pro session failures caused by large replayed text signatures, embedded thinking tags, and mixed text plus tool-call turns dropping visible replies.

## Best Possible Solution

Handle this at the Google provider and replay boundary: preserve only provider-required Gemini function-call/native-thinking signatures, bound or avoid replaying stale visible-text signatures, and align Gemini reasoning delivery with native thinking so malformed or final-less tagged output cannot silently suppress user replies.
