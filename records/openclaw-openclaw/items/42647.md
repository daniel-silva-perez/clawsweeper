## Summary

Keep this open. Current main has adjacent file/line citations, promotion source labels, memory-host events, and a provenance-rich Memory Wiki layer, but it still does not provide the requested canonical durable-memory provenance model with typed source references, multiple supporting sources, merge/edit lineage, or a gate that prevents source-less durable promotion.

## What This Changes

Request to add a durable memory provenance model so every long-term memory carries source attribution across promotion, retrieval, merge, edit, and review flows.

## Best Possible Solution

Implement provenance in the canonical Memory MVP schema and write pipeline: typed source references for session/message/file origins, support for multiple supporting sources, required provenance before durable promotion, provenance-preserving create/merge/edit/review history, and retrieval/review output that can render those references clearly. Memory Wiki can consume or mirror that model, but should not be treated as the core durable-memory contract.

## Reproduction Assessment

Not applicable as a feature/schema request rather than a runtime bug. The current-main check is source and docs inspection, which shows the requested durable provenance contract is still absent.

## Solution Assessment

Unclear until the related record-model and write-pipeline design is settled. The requested direction is appropriate for core Memory MVP work, but the current adjacent citations and Memory Wiki provenance are not sufficient as the final solution.
