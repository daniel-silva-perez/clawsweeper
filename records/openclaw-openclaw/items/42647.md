## Summary

Current main still has adjacent retrieval citations, promotion source labels, event logs, and the separate memory-wiki provenance layer, but it does not implement the requested canonical durable-memory provenance model with typed source refs, multiple supporting sources, preserved merge/edit lineage, or a durable-promotion gate for source-less records.

## Best Possible Solution

Keep this issue open and implement provenance in the canonical Memory MVP schema and write pipeline: add typed sourceRefs/provenance to durable records, require at least one source before durable promotion, preserve provenance through create/merge/edit/conflict revisions, support multiple supporting sources, and expose those refs in retrieval and review flows. Memory Wiki can consume or mirror that model, but should not be treated as the default durable-memory provenance contract.
