## Summary

Keep this issue open. Current main has useful adjacent attribution in memory search citations, short-term promotion source markers, event logs, and the separate Memory Wiki plugin, but it still lacks the requested durable-memory provenance model: typed source refs, multiple supporting sources, provenance-preserving merges/edits, and a promotion gate that rejects source-less durable records.

## Best Possible Solution

Keep this issue open and implement it with the related Memory MVP schema and write-pipeline work: add typed sourceRefs/provenance to canonical durable memory records, require at least one source before durable promotion, preserve provenance through create/merge/edit/conflict outcomes and revision history, and expose those refs consistently in retrieval and review flows. Memory Wiki can mirror or consume that model, but should not be treated as the active durable-memory provenance contract.
