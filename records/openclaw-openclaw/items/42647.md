## Summary

Keep this issue open. Current main has adjacent source labels in memory search, short-term promotion output, event logs, and the separate memory-wiki plugin, but it still does not implement the requested durable-memory provenance model with typed source refs, multiple supporting sources, provenance-preserving merge/edit history, or a durable-promotion gate that rejects source-less records.

## Best Possible Solution

Keep this issue open and implement provenance together with the Memory MVP schema and write-pipeline work: add typed sourceRefs/provenance to canonical durable memory records, require at least one source before durable promotion, preserve provenance through create/merge/edit/conflict revisions, and expose those refs in retrieval and review flows. Memory Wiki can consume or mirror that model, but should not be treated as the shipped canonical durable-memory provenance contract.
