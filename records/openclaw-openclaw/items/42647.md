## Summary

Keep open. Current main has adjacent retrieval citations, single-source promotion markers, memory event logs, public memory artifacts, and Memory Wiki provenance, but it still does not implement the requested durable-memory provenance model with required structured source refs, multiple supporting sources, merge/edit lineage, or a canonical source-backed durable record contract.

## Best Possible Solution

Keep this issue open and implement it with #42646 and #42648: add typed `sourceRefs`/provenance to canonical durable memory records, require at least one source before durable promotion, preserve provenance through create/merge/edit/conflict outcomes, and expose those refs consistently in retrieval and review flows. Memory Wiki can consume or mirror that model, but should not substitute for the active durable-memory provenance contract.
