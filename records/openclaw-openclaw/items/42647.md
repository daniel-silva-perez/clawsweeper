## Summary

Keep this issue open. Current main still has adjacent source labels in retrieval and promotion output, memory host event logs, and the memory-wiki companion plugin, but it does not implement the requested canonical durable-memory provenance model with typed source refs, multiple supporting sources, provenance-preserving merge/edit history, or a durable-promotion gate for source-less records.

## Best Possible Solution

Keep this issue open and implement provenance as part of the canonical Memory MVP schema and write pipeline: add typed sourceRefs/provenance to durable memory records, require at least one source before durable promotion, preserve provenance through create/merge/edit/conflict revisions, and expose those refs in retrieval and review flows. Memory Wiki can consume or mirror that model, but should not be treated as the shipped canonical durable-memory provenance contract.
