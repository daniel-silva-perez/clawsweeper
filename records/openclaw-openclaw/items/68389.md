## Summary

Keep open. Current main still has only the empty-allowlist warning path and still returns before warning when `plugins.allow` is populated, so the linked bug is not implemented on main. The PR is useful and narrow, but it is currently dirty against main and its changelog entry does not satisfy the PR changelog gate.

## What This Changes

This PR changes the plugin loader allowlist warning logic to emit a mismatch warning for populated allowlists with no discovered plugin-id matches, adds loader regression tests for mismatch, partial-match, and bundled-id cases, and adds a changelog entry.

## Best Possible Solution

Land a rebased version that keeps the legacy empty-allowlist warning stable, adds the populated-but-unmatched warning against the full discovered plugin-id set, preserves quiet behavior for partial and bundled-id matches, and fixes the changelog line before running the focused loader tests and changed gate.
