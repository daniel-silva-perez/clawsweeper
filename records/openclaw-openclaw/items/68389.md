## Summary

Keep open. Current main still returns early for any non-empty `plugins.allow` and has no mismatch-warning tests, so this PR is not implemented on main. The linked #68352 issue remains open and this PR is the active implementation path, but it needs rebase/maintainer review before merge.

## What This Changes

This PR changes the plugin loader warning path to emit a specific message when populated `plugins.allow` entries match no discovered plugin ids, adds loader tests for mismatch and valid-match cases, and adds a changelog entry.

## Best Possible Solution

Land a rebased version that keeps the existing empty-allowlist warning stable, adds the populated-but-unmatched allowlist warning against the full discovered plugin-id set, preserves quiet behavior for partial and bundled-id matches, and fixes the changelog attribution. Broader auto-load policy questions from #68352 should remain separate unless maintainers intentionally expand scope.
