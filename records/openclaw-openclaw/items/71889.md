## Summary

Keep this issue open. Current main still uses the runtime workspace resolver as the interactive `agents add` prompt default, and that resolver still nests non-default agents under `agents.defaults.workspace`. The docs still describe peer-level `workspace-<agentId>` paths, and open implementation PRs with closing syntax already exist, so maintainers should review and land or close one canonical PR rather than close this issue early.

## Best Possible Solution

Keep this issue open and review the existing implementation PRs, preferably landing one narrow fix that changes the interactive `agents add` default for brand-new agents to the documented peer-level `workspace-<id>` path while preserving explicit workspaces and existing runtime resolver compatibility. Add focused regression coverage for the wizard prompt default and resolver edge cases, plus a user-facing changelog entry when the fix merges.
