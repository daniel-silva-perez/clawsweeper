## Summary

Keep open. Current main still seeds the interactive `agents add` workspace prompt from the runtime resolver, the resolver still nests non-default agents under `agents.defaults.workspace`, the docs still describe peer-level `workspace-<agentId>` workspaces, and open implementation PRs already reference this issue for maintainer review.

## What This Changes

The issue asks for the `agents add <id>` wizard to pre-fill a peer-level workspace path such as `~/.openclaw/workspace-<id>` instead of nesting under the main workspace.

## Best Possible Solution

Land one narrow fix from the existing PR set that changes the interactive default for brand-new agents to a peer-level workspace path, preserves explicit and legacy workspace paths, adds focused wizard/resolver regression coverage, and includes a user-facing changelog entry.

## Reproduction Assessment

Yes. The source path is deterministic: setup persists `agents.defaults.workspace`, `agents add` uses `resolveAgentWorkspaceDir` as the prompt default, and the resolver joins that fallback with a non-default agent id.

## Solution Assessment

Yes. The requested wizard-default fix is the narrowest maintainable direction; changing existing runtime resolution broadly would risk relocating existing agents unless maintainers explicitly choose that compatibility tradeoff.
