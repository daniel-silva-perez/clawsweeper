## Summary

Keep open. Current main still seeds the interactive add-agent workspace prompt from `resolveAgentWorkspaceDir`, and that resolver still appends a non-default agent id under `agents.defaults.workspace`, while the docs and CLI examples continue to document peer-level `workspace-<agentId>` paths. The supplied GitHub context shows multiple open fix PRs for this exact issue, including PRs with closing syntax, so the issue should remain open until one fix lands.

## What This Changes

The issue asks for the interactive `openclaw agents add <id>` wizard to pre-fill a peer-level workspace path such as `~/.openclaw/workspace-<id>` instead of nesting under the main workspace.

## Best Possible Solution

Land one narrow fix from the existing PR set that changes only the new-agent wizard default to the documented peer-level workspace path, preserves explicit and legacy existing workspace paths, adds focused resolver/wizard regression coverage, and includes the required user-facing changelog entry.

## Reproduction Assessment

Yes. The source path is deterministic: setup persists `agents.defaults.workspace`, the add-agent wizard uses `resolveAgentWorkspaceDir(cfg, agentId)` as the prompt default, and the resolver appends the new non-default agent id under that fallback.

## Solution Assessment

Yes. The requested direction is the narrowest maintainable fix, but it should be applied as a new-agent default/prompt behavior change without silently migrating existing nested or explicit workspace paths.
