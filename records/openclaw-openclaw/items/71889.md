## Summary

Keep open. Current main still routes the interactive add-agent workspace prompt through the resolver path that appends the new agent id to `agents.defaults.workspace`, while the docs continue to document peer-level `workspace-<agentId>` paths; the linked open PRs are the right path to resolve it rather than closing early.

## What This Changes

The issue requests changing the interactive `agents add <id>` wizard default workspace from nested `~/.openclaw/workspace/<id>` to the documented peer-level `~/.openclaw/workspace-<id>` form.

## Best Possible Solution

Land one narrow fix from the existing PR set that changes the interactive default for newly added agents to a documented peer-level workspace path, preserves explicit and legacy workspace paths, adds focused wizard/resolver regression coverage, and includes the required user-facing changelog entry.

## Reproduction Assessment

Yes. The source path is deterministic: setup persists `agents.defaults.workspace`, the add wizard uses `resolveAgentWorkspaceDir(cfg, agentId)` as the prompt default, and the resolver joins that fallback with the non-default agent id.

## Solution Assessment

Yes. The requested direction is the narrowest maintainable fix, but it should be implemented as a new-agent wizard default change without silently migrating existing agents that already have nested or explicit workspace paths.
