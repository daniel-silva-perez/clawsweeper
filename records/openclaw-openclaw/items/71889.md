## Summary

Keep this issue open. Current main still has the reported behavior: the interactive `agents add` wizard seeds `Workspace directory` from `resolveAgentWorkspaceDir(cfg, agentId)`, and that resolver still nests a non-default agent under `agents.defaults.workspace` when that default is set. The docs still describe the peer-level `workspace-<agentId>` convention, and there are open fix PRs with closing syntax, so the best cleanup action is to review and land or close one of those PRs rather than close the issue early.

## Best Possible Solution

Keep this issue open and review the open fix PRs, preferably landing one narrow implementation that changes the `agents add` wizard default for brand-new agents to a peer-level `workspace-<agentId>` path while preserving existing explicit workspaces and runtime resolver compatibility. Add focused regression coverage in `src/commands/agents.add.test.ts` and/or `src/agents/agent-scope.test.ts`, then close this issue after the fix merges and ships.
