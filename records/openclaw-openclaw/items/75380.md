## Summary

Keep open. Current main still has append-only diagnostic JSONL sinks for cache trace and the current provider payload logger path, while documented rotation only applies to the primary Gateway file logger. The requested bounded rotation/defaults are valid, narrow, and not superseded by the related gateway.err.log, journald, or transcript-growth reports.

## What This Changes

The issue requests configurable max-size and archive rotation for provider payload and cache trace diagnostic JSONL files, plus optional environment overrides.

## Best Possible Solution

Add a reusable bounded-rotation policy for queued diagnostic JSONL writers, expose documented max-size/archive controls for cache trace and the provider payload logger, and keep the primary Gateway log, gateway.err.log, journald, and session transcript rotation work in their separate tracked lanes.

## Reproduction Assessment

Yes. Source inspection provides a high-confidence reproduction path: enable cache trace or the payload logger, generate repeated model turns, and the active JSONL file appends indefinitely because no max-size/archive option is passed. I did not live-run traffic because this review is read-only, but the implementation path is direct.

## Solution Assessment

Yes, the narrow maintainable solution is to reuse or generalize the existing file-log rotation semantics for these queued JSONL sinks rather than adding manual truncation or cron cleanup. The only open design detail is exact naming/config placement for the current payload logger versus the reported `provider-payload.jsonl` name.
