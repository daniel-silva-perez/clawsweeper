## Summary

Keep open. Current main still has the reported slug-generator path: the session-memory LLM slug helper calls `runEmbeddedPiAgent` without `authProfileFailurePolicy: "local"`, while the embedded runner already supports helper-local auth failure scoping and still persists classified billing/auth failures when the policy is not local. The fix is narrow and safe to queue as one focused PR.

## Best Possible Solution

Keep this issue open and fix the slug-generator call site by making its embedded run helper-local for auth-profile failure state, preferably by passing `authProfileFailurePolicy: "local"` to `runEmbeddedPiAgent` in `src/hooks/llm-slug-generator.ts`. Add focused regression coverage so slug-helper billing/auth failures fall back without poisoning shared auth state, while preserving the existing billing classifier for user-facing/provider runs.
