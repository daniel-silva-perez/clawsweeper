## Summary

Keep open. Current main still lets the session-memory slug helper call the embedded runner without the existing local auth-profile failure policy, while the runner can persist billing-classified prompt failures into shared auth state; the existing local-policy seam and the Active Memory sibling fix make this a narrow valid runtime bug.

## What This Changes

The issue asks OpenClaw to prevent session-memory LLM slug-generator failures from marking a shared auth profile as billing-disabled and causing a five-hour profile-wide outage.

## Best Possible Solution

Pass `authProfileFailurePolicy: "local"` from the session-memory slug-generator embedded run, add a focused regression test that asserts the option is supplied, and keep the broader dedicated slug-model/congestion work in #33962 separate.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to route an Anthropic extra-usage prompt failure through `generateSlugViaLLM`: current main classifies that text as billing and, without the local policy on the slug helper, allows the embedded runner to write shared `disabledUntil` billing state.

## Solution Assessment

Yes for a narrow repair. The existing local policy is the smallest maintainable fix because it prevents helper-local failures from poisoning shared auth health without changing normal billing/failover classification.
