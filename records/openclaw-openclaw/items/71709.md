## Summary

Current main still matches the reported slug-generator auth-health leak. `generateSlugViaLLM` calls `runEmbeddedPiAgent` for `sessionKey: "temp:slug-generator"` without `authProfileFailurePolicy: "local"`, while the embedded runner already supports local helper scoping and still persists classified billing/auth failures when that policy is omitted. This is a narrow, valid fix candidate rather than a close candidate.

## Best Possible Solution

Keep this issue open and fix the slug-generator call site by passing `authProfileFailurePolicy: "local"` to `runEmbeddedPiAgent` in `src/hooks/llm-slug-generator.ts`. Add a focused regression test in `src/hooks/llm-slug-generator.test.ts` asserting the helper passes local auth failure policy while preserving existing timeout/model behavior. Avoid broad failover-classifier changes unless a maintainer explicitly chooses that larger product direction.
