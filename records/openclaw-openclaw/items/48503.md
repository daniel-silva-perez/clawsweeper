## Summary

Keep open. Current main still has enforceable before_tool_call hooks and input provenance plumbing elsewhere, but the hook event does not expose the requested action classification, capability metadata, or input provenance fields. Related capability/provenance proposals and the open hook-context PR remain active design context, not evidence that this is implemented.

## Best Possible Solution

Keep this open and implement a coordinated additive SDK/runtime change: define stable tool action metadata or capability descriptors, thread normalized input provenance from the active run/session into before_tool_call, update plugin docs and contract tests, and align the final shape with #48510, #48519, and the open #58079 discussion before closing.
