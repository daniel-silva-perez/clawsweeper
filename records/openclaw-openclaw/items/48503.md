## Summary

Keep open. Current main has enforceable before_tool_call hooks, approval routing, trusted tool policies, display metadata, and existing InputProvenance plumbing, but it still does not expose the requested actionCategory/capability classification or inputProvenance on PluginHookBeforeToolCallEvent. The related proposals remain open, and the HELM comment adds useful design context rather than making the core request obsolete.

## Best Possible Solution

Keep this issue open and implement a coordinated additive SDK/runtime change: define stable tool action metadata or capability descriptors, thread normalized InputProvenance from the active run/session into before_tool_call, update public plugin docs and contract tests, and align the shape with #48510/#48519 and the open #58079 discussion before closing.
