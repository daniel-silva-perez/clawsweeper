## Summary

Keep open. Current main has enforceable before_tool_call hooks and separate input-provenance plumbing, but the public hook event still does not expose the requested action classification, capability flags, or inputProvenance. The open hook-context PR and related capability/provenance issues are design context, not an implementation of this request.

## Best Possible Solution

Keep this open and implement a coordinated additive SDK/runtime change: define stable tool action metadata or capability descriptors, thread normalized active input provenance into before_tool_call, document the public contract, add contract/runtime tests, and align the final shape with #48510, #48519, and open PR #58079 before closing.
