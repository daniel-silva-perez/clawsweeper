## Summary

Keep open. Current main does not implement the PR’s intended hidden WebChat bootstrap flow: the Gateway `chat.send` schema and generated Swift models do not include `hideUserMessage`, the Control UI send path always locally echoes the user turn, and URL settings parsing has no `autostart` handling. This PR is therefore not obsolete or implemented on main; it needs maintainer review/landing or an explicit product decision.

## Best Possible Solution

Keep this PR open for maintainer review. If the product direction is accepted, rebase the final branch onto current main and land a core WebChat/Gateway implementation with additive protocol schema, generated Swift models, WebChat URL parsing, hidden-send safeguards, docs, and targeted gateway/UI tests. If maintainers do not want hidden bootstrap prompts in core, close it explicitly on product/security grounds rather than as already implemented.
