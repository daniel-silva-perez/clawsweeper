## Summary

Keep open. Current main still matches the reported recovery deadlock: a CLI paired with only `operator.read` cannot call the pairing-scoped approve/reject RPCs, `devices reject` has no local loopback fallback, and the documented fallback path only covers list/approve.

## What This Changes

The issue requests a supported recovery path when a read-only paired CLI is stuck behind an auto-reissued broader device repair/scope-upgrade request.

## Best Possible Solution

Add or document one supported same-device recovery path for local repair/scope-upgrade requests, with explicit constraints around loopback/local state, caller identity, and requested scopes, plus tests and docs for the chosen flow.
