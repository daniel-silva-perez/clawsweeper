## Summary

Keep open. Current main and the latest shipped release still send WhatsApp read receipts before durable inbound storage or successful handler delivery, so the reported crash-loss window remains real; Ryce's KeepMyClaw comment is useful complementary recovery context, but it does not replace a bundled WhatsApp plugin durability or delayed-read-receipt fix.

## What This Changes

The issue requests crash-safe WhatsApp inbound acknowledgment ordering, such as WAL, persist-before-ack, or delayed acknowledgment, so messages are not silently lost if the gateway crashes mid-processing.

## Best Possible Solution

Fix the bundled WhatsApp plugin so accepted inbound messages are either durably recorded for replay before `readMessages` or are only marked read after successful `onMessage` delivery and inbound dedupe finalization, with docs and focused tests covering the new acknowledgment boundary.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/50563 in the bundled WhatsApp plugin. Current behavior sends `sock.readMessages` from `extensions/whatsapp/src/inbound/monitor.ts` before append filtering, enrichment, inbound dedupe claim, enqueueing, and successful `onMessage` delivery, while `extensions/whatsapp/src/inbound/dedupe.ts` uses memory-only claimable dedupe. Implement a narrow plugin-owned fix so accepted inbound WhatsApp messages are either durably recorded for replay before read receipts or `readMessages` is delayed until after successful `onMessage` delivery and inbound dedupe finalization. Keep the fix inside the WhatsApp plugin unless a genuinely generic SDK seam is required. Update tests that currently assert eager read receipts, add coverage for crash/retry-safe ordering or handler failure behavior, update `docs/channels/whatsapp.md`, and add a user-facing changelog fix entry with proper attribution. Preserve self-chat and disabled-read-receipt behavior, and do not change unrelated channel/core delivery semantics. Ryce's KeepMyClaw comment is complementary recovery context, not a substitute for the bundled plugin fix.
