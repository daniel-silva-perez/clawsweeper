## Summary

Keep open. Current main still sends WhatsApp read receipts before append filtering, enrichment, inbound dedupe claim/commit, and `onMessage` delivery, and the WhatsApp inbound dedupe guard is not configured for persistent replay. Ryce's KeepMyClaw context is useful complementary crash-state recovery, but it does not replace a bundled WhatsApp plugin ordering or durability fix.

## What This Changes

Prevent WhatsApp inbound message loss on gateway crash by durably recording messages before acknowledgment or delaying acknowledgment until processing is safely underway or complete.

## Best Possible Solution

Fix the bundled WhatsApp plugin so accepted inbound messages are either durably recorded for replay before `readMessages` or only marked read after successful `onMessage` delivery and inbound dedupe finalization, with focused tests and docs reflecting the new acknowledgment boundary.
