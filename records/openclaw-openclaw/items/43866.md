## Summary

Keep open. Current main still drops `accountId` before `/hooks/agent` reaches isolated cron delivery, while the cron delivery path itself already supports `delivery.accountId`. The provided GitHub context also shows an open same-author implementation candidate, #43951, with closing syntax, so this issue should stay open until that PR or a replacement lands.

## Best Possible Solution

Review and land or replace #43951 with a focused current-main fix. The fix should add and normalize `accountId` for direct `/hooks/agent` payloads and mapped hook config/actions/transforms, include it in replay scope, propagate it into `CronDelivery.accountId` during dispatch, update public hook docs, and add regression coverage proving hook dispatch reaches isolated cron delivery with the selected account.
