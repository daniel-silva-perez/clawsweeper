## Summary

Keep open. Current main still omits `accountId` from the direct `/hooks/agent` payload, mapped hook action/config surfaces, replay cache scope, and the hand-built cron delivery object, even though the cron delivery pipeline itself now supports `delivery.accountId`. The issue also has an open closing PR (#43951), so it should remain open until that PR or an equivalent fix is reviewed and merged.

## What This Changes

The issue requests preserving `accountId` through `/hooks/agent` and mapped hook dispatch into isolated cron delivery for multi-account channel routing.

## Best Possible Solution

Land or replace the focused hook bridge fix so direct and mapped hooks accept a trimmed `accountId`, include it in idempotency scope, construct `CronDelivery.accountId`, document the public hook field, and cover the path with direct hook, mapped hook, and isolated cron delivery regression tests.
