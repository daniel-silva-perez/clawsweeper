## Summary

Keep open. Current main still exposes `web_fetch` as a generic URL fetch tool and does not implement tool-level domain routing, a Twitter/X blocklist, a configurable tool description override, or an FxTwitter rewrite. The docs do document `x_search` as an alternative for exact X status URLs, and the provided GitHub context shows #66047 attempted a targeted FxTwitter fallback but was closed unmerged, so the central product request remains unresolved.

## Best Possible Solution

Keep this issue open and implement a conservative tool-layer fix: add a typed, documented `web_fetch` domain policy or routing hook with tests and config schema coverage, or explicitly provide a plugin/provider extension point for domain-specific fetch handling. #66047 is a useful prior attempt to review, but a final fix should account for fallback behavior, security/SSRF expectations, third-party dependency risk, and docs for `web_fetch` plus `x_search`.
