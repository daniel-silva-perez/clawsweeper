## Summary

Keep open. Current main has a bundled `bootstrap-extra-files` hook for adding extra recognized bootstrap basenames, but it does not implement the requested `agents.defaults.bootstrapFiles` / `agents.list[].bootstrapFiles` config key and it still rejects arbitrary files like `RULES.md` or `CONVENTIONS.md`. The KeepMyClaw comment is useful backup/restore context, but it is not a startup Project Context injection mechanism. The related model-aware AGENTS/SUBAGENTS stack was narrower and closed without providing this generic custom-file allowlist.

## Best Possible Solution

Keep this issue open and implement a schema-backed, documented way to add explicit workspace-relative bootstrap files, or intentionally extend `bootstrap-extra-files` to support arbitrary configured basenames such as `RULES.md` under the existing boundary and budget guards. The change should update `AgentDefaultsConfig`, per-agent config if desired, strict schemas, generated config docs/help, bootstrap loading tests, hook docs, and truncation/diagnostic coverage.

## Risks / Open Questions

- Allowing arbitrary bootstrap files could widen prompt-injection and accidental secret-exposure risk; any implementation should keep explicit opt-in config, workspace-boundary checks, file and total character budgets, diagnostics, and docs aligned.
- There is already a hook-based partial workaround for extra recognized bootstrap basenames, so maintainers should decide whether to extend that hook, add a core config key, or document the supported hook path before implementing a new parallel surface.
