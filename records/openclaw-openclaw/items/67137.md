## Summary

Keep this PR open for maintainer review. Current main still has the exact prompter-boundary and downstream raw `.trim()` assumptions that the PR changes, and the proposed regression tests are not present. The original linked crash appears to have been closed as covered by the separate docs-link fix in #67086, but that does not implement this PR's narrower hardening of `WizardPrompter.text` and channel setup text handling.

## What This Changes

The PR normalizes undefined text-prompt results in the Clack prompter and channel setup token paths, replaces direct `.trim()` calls at two setup boundaries, and adds regression tests for undefined text responses.

## Best Possible Solution

Review the PR as a small hardening change: either land a narrow version that makes `WizardPrompter.text` always return a string and adds focused undefined-result tests, or explicitly close it after deciding the locked Clack behavior plus the separate #67086 docs-link fix are sufficient.
