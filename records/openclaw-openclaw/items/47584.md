## Summary

Current main still does not implement the requested native iOS Siri/App Intent flow. The app links AppIntents but has metadata generation disabled and no Swift AppIntent/AppShortcutsProvider/ProvidesDialog implementation; existing support is a deep-link/gateway-hook workaround. Related PR #40874 is still open and only proposes opening Talk Mode, not a parameterized Ask OpenClaw intent with spoken reply.

## Best Possible Solution

Keep this issue open. The best path is for iOS maintainers to review #40874 first, then add or split a focused Ask OpenClaw AppIntent that accepts a message, respects the existing gateway/deep-link trust policy, can safely reach the paired gateway, and returns a Siri/Shortcuts dialog with device or simulator proof and docs updates.
