## Summary

Keep open. Current main still does not implement the requested native iOS Siri/App Intent flow: there is no Ask/OpenClaw message AppIntent, no AppShortcutsProvider/ProvidesDialog path on main, and the iOS project still disables App Intents metadata generation. The existing deep-link and Gateway hook paths are useful building blocks, and open PR #40874 overlaps Action Button/AppIntent work, but that PR is still open and only adds an Open Talk Mode shortcut rather than a typed ask-and-spoken-reply intent.

## What This Changes

The issue requests native iOS Siri/App Intent support so users can ask OpenClaw a dictated question from Siri or Shortcuts and receive a spoken/dialog response through the paired gateway.

## Best Possible Solution

Add a focused iOS AppIntent/AppShortcutsProvider surface in apps/ios that accepts a message parameter, uses the existing paired-gateway trust model or a maintainer-approved successor, dispatches through a narrow agent-request or /hooks/agent helper, returns a Siri/Shortcuts dialog result, and includes Swift tests plus simulator/device verification and docs updates.

## Reproduction Assessment

Yes. Source search and project metadata on current main reproduce the feature gap with high confidence: no native Ask OpenClaw AppIntent exists and App Intents metadata generation remains disabled; runtime Siri testing was not needed to prove absence.

## Solution Assessment

No current implementation solves the issue. The best path is a maintainer-reviewed native iOS intent built on the existing gateway/deep-link trust primitives, not closing this as covered by the deep-link workaround or by the open Talk Mode PR.
