## Summary

Keep this issue open. Current main does not implement a native Siri/Shortcuts App Intent that accepts a question parameter and returns a spoken `ProvidesDialog` result. The related open PR #40874 partially overlaps by adding an App Intent to open Talk Mode, but it is unmerged and does not cover the issue's core ask-OpenClaw-with-message workflow.

## Best Possible Solution

Keep this issue open and implement a native iOS App Intent in the iOS app, likely under an app-owned Shortcuts/AppIntents source area, that accepts a message/question parameter, routes through the paired gateway using the existing auth/session model or documented hook endpoint, returns a spoken dialog result, and documents the supported foreground/background limits. If #40874 lands first, use its Talk Mode shortcut as a foundation but add a separate parameterized Ask OpenClaw intent for this issue.
