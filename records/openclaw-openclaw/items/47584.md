## Summary

Keep open. Current main still lacks the requested native iOS Siri/App Intent flow: there is no Swift AppIntent/AppShortcutsProvider/ProvidesDialog implementation, and the iOS target currently disables App Intents metadata generation. The existing openclaw://agent deep-link path and Gateway /hooks/agent endpoint are useful building blocks, but they do not provide a native typed Siri/Shortcuts action with a spoken dialog reply.

## What This Changes

Add native iOS Siri/AppIntent support so users can ask OpenClaw a spoken question, send it to the paired Gateway, and receive a spoken Siri/Shortcuts response.

## Best Possible Solution

Add a focused iOS AppIntent/AppShortcutsProvider surface in apps/ios that accepts a message parameter, uses the existing paired-gateway trust model, routes through an approved agent-request or /hooks/agent helper, returns a Siri/Shortcuts dialog, and includes focused Swift tests plus simulator or device verification and docs updates.

## Reproduction Assessment

Yes for the current-main feature gap: source search and project metadata show no native AppIntent implementation and disabled App Intents metadata generation. Full Siri runtime reproduction was not attempted because this was a read-only review.

## Solution Assessment

No current implementation solves the request. The maintainable path is a focused native iOS intent built on the existing gateway/deep-link trust primitives, rather than closing this as covered by the deep-link workaround or by the overlapping Talk Mode PR.
