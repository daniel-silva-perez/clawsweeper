## Summary

Keep open. Current main still does not implement the requested native iOS Siri/App Intent flow. The prior ClawSweeper review remains consistent with the current checkout: AppIntents is linked but metadata generation is disabled, there is no Swift AppIntent/AppShortcutsProvider/ProvidesDialog implementation, and the existing path is a deep-link or gateway hook workaround rather than a native Ask OpenClaw Siri action.

## Best Possible Solution

Keep this issue open. The best path is for iOS maintainers to review #40874, then add or split a focused AskOpenClaw AppIntent that accepts a message parameter, uses the existing paired-gateway trust model, respects deep-link/gateway dispatch safety boundaries, returns a Siri/Shortcuts dialog or value, and includes simulator or device proof plus docs updates.
