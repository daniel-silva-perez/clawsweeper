## Summary

Keep this PR open. Current main still silently switches an unwritable Linux npm prefix to `~/.npm-global` with only `ui_info "Configuring npm for user-local installs"`; it does not warn before `npm config set prefix`, does not warn users away from future `sudo npm i -g openclaw@latest`, and has no current-main regression test for that warning order. The closed source PR #50479 and the already-closed issue #44365 are useful context, but they do not make this repaired PR obsolete.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is a focused installer fix that warns before `npm config set prefix "$HOME/.npm-global"`, warns users to update with non-sudo `npm i -g openclaw@latest`, includes the regression test proving warning order, and preserves the #50479/@Sayeem3051 attribution in the changelog or merge notes.
