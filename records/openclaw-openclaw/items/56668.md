## Summary

Keep open for maintainer review. Current main still lacks a gateway server WebSocket protocol ping, and this PR is a focused implementation that matches the `ws` dependency contract; the main remaining decision is whether to land this narrow keepalive or let the broader open #74783 connection-health PR become the single ping owner.

## What This Changes

The PR adds a 25-second per-connection server-side `socket.ping()` interval in the gateway WebSocket handler and clears that timer during connection teardown.

## Best Possible Solution

Keep exactly one gateway-owned WebSocket protocol ping path: merge this focused keepalive if maintainers want the smallest fix now, or land #74783 first and treat its maintenance-loop ping/pong implementation as canonical after verifying it covers the same idle-drop behavior.
