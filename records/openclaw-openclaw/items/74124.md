## Summary

Keep open. Current main still only installs and updates ClawHub skills in the active workspace, and #74466 is an open closing PR for the requested global/shared install target, so this issue should remain open until that PR lands or is explicitly superseded.

## What This Changes

Add an explicit global/shared target for `openclaw skills install` so ClawHub skills can be installed outside the active agent workspace.

## Best Possible Solution

Review and land #74466, or replace it with an equivalent narrow PR, adding an explicit shared install/update target while keeping the existing workspace default and covering CLI behavior, installer targeting, docs, and conflict cases.

## Reproduction Assessment

Yes. The observable gap can be verified from current main without running a mutating install: the CLI has no global/target option, passes only `workspaceDir`, and the installer resolves targets under the active workspace `skills` directory.

## Solution Assessment

Yes. An explicit opt-in global/shared flag or target is the narrowest maintainable solution; changing the default workspace behavior would be broader and would risk surprising existing multi-agent setups.
