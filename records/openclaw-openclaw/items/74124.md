## Summary

Keep open. Current main still only installs ClawHub skills into the active agent workspace and has no explicit shared/global install target, but open PR #74466 already uses closing syntax for this issue, so the maintainer path is to review, land, replace, or close that PR rather than close this issue early.

## What This Changes

Request to add an explicit `openclaw skills install` global/shared target, such as `--global` or `--target global`, for installing ClawHub skills into a shared skills directory while preserving the current workspace default.

## Best Possible Solution

Land a reviewed explicit opt-in global install/update target, likely via #74466 if correct, that reuses the existing ClawHub slug validation, origin metadata, lock/update flow, and docs/tests while rejecting ambiguous `--global` plus `--agent` combinations.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the CLI has no `--global` or target option, and the installer always resolves destinations under `<workspace>/skills/<slug>`; no write-producing live install was needed for this read-only review.

## Solution Assessment

Yes. An explicit opt-in global target is the narrow maintainable direction because shared roots are already supported for loading, while the current default workspace behavior should remain unchanged.
