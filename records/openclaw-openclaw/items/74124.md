## Summary

Keep open. Current main still lacks an explicit global/shared target for `openclaw skills install`; the CLI and ClawHub installer continue to target an active agent workspace, while shared skill roots are only load locations today. The linked open PR #74466 appears to be the active implementation candidate and should keep this issue open until it is merged or closed.

## What This Changes

Request an explicit global/shared install target, such as `openclaw skills install <slug> --global`, so ClawHub skills can be installed into a shared skills directory while preserving the current workspace-scoped default.

## Best Possible Solution

Land a narrow reviewed implementation that adds an explicit shared target for native ClawHub skill install/update, most likely `--global` targeting the managed shared root, rejects ambiguous combinations with `--agent`, prints the resolved path, and updates focused CLI tests plus public docs. Close this issue only after that implementation merges.

## Reproduction Assessment

Yes. Source and docs give a high-confidence reproduction path: `openclaw skills install <slug>` resolves an active workspace and the installer writes under `<workspace>/skills/<slug>`, with no current `--global` option to choose a shared root.

## Solution Assessment

Yes. An explicit opt-in shared target is the narrowest maintainable direction because it preserves existing workspace defaults and matches the already-supported shared load roots; the open PR should be reviewed against that boundary.
