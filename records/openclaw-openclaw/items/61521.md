## Summary

Keep this PR open. Current main at c0ec58f4b6906ad06fd4c6f265d43be053b2d193 still lacks the deferred candidate-config validation seam: setup adapters only expose synchronous raw-input validation, the setup wizard applies each input as it is collected, and `openclaw channels add` validates before assembling the final candidate config. The PR is a plausible core plugin API change for the linked VK stack, but it is not obsolete or implemented on main and still needs maintainer review of the unresolved account-id and sparse-input review findings.

## Best Possible Solution

Keep the PR open as a candidate core plugin API change. The best path is to revise it so deferred setup builds a candidate config, validates complete/async hooks against that candidate, propagates the resolved account ID consistently, avoids validating sparse draft input as if it were complete raw input, and adds focused tests plus SDK/setup documentation for the new contract. The VK channel itself can remain plugin-owned, but this shared validation seam belongs in core if third-party channel plugins need it.
