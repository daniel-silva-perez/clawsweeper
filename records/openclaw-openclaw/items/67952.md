## Summary

Keep this PR open. Current main still has `ltm list` implemented as a count-only command, duplicating `ltm stats`, while this PR is a focused memory-lancedb plugin patch that adds the missing record-listing behavior. The related open reindex/drop-index PR is separate and does not supersede this work.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. Request focused memory-lancedb CLI tests and maintainer confirmation on unbounded default output, then land this patch after rebase/merge validation or apply an equivalent current-main patch that adds `MemoryDB.list`, JSON `ltm list`, validated `--limit`, optional newest-first ordering, and keeps `ltm stats` count-only.

- The PR should add focused regression coverage for JSON `ltm list`, `--limit`, invalid limits, newest-first ordering, and preserving `ltm stats` as the count-only command.
- Default `ltm list` would print every stored memory row; that may be acceptable for an inspection command, but it is a local data-disclosure and large-output behavior that should be intentional.
- The PR branch is based on an older memory-lancedb file; the implementation is localized, but it should be rebased or applied carefully so current hook/cursor initialization changes remain intact.
