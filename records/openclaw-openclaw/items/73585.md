## Summary

Keep open. This is a fresh current-release Discord plugin bug report with concrete logs and an actionable mitigation request. Current main still has the same hardcoded 10s `/gateway/bot` metadata timeout, logs every transient fallback, and has no exposed `OPENCLAW_DISCORD_GATEWAY_INFO_TIMEOUT_MS` or similar config knob. The issue is not superseded by the related older Discord gateway reports, which cover crash/stuck-ready/worker-timeout classes rather than this metadata fetch timeout/log-flood behavior.

## Best Possible Solution

Keep this issue open for a Discord-plugin-owned fix. A conservative path is to add a bounded configurable metadata timeout, rate-limit or demote repeated successful-fallback warnings, and add focused tests for the new timeout/log policy. If maintainers confirm connection reuse is the root cause, keep that fix inside `extensions/discord/src/monitor/gateway-plugin.ts` or a plugin SDK HTTP seam without moving Discord-specific policy into core.
