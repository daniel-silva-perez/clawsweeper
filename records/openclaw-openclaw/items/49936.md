## Summary

Keep this PR open. Current main has the runtime WebChat delivery path and some related generic/UI coverage, but it still does not contain this PR's focused gateway regression tests for internal WebChat-visible agent turns created through the `agent` RPC path in both main-session and direct WebChat session-key cases. The PR is narrow, test-only coverage for the already-fixed #46106 behavior, not an obsolete runtime patch.

## Best Possible Solution

Keep the PR open for maintainer review. Rebase and land these tests, or add equivalent coverage in `src/gateway/server.agent.gateway-server-agent-b.test.ts` proving that internal agent turns created through the `agent` RPC path deliver final WebChat `chat` events for both the main-session `channel: "last"` path and a direct WebChat session key. No runtime or security-sensitive change appears necessary.
