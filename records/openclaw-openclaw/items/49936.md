## Summary

Keep this PR open. Current main has the runtime plumbing and nearby generic coverage, but it still does not contain the two focused regression tests this PR adds for implicit `agent` RPC run-context registration delivering internal assistant/lifecycle events to WebChat observers on both the main-session and direct WebChat session-key paths. The linked bug #46106 is closed, but this PR remains a narrow test-only regression coverage candidate rather than obsolete work.

## Best Possible Solution

Keep the PR open for maintainer review. If this regression coverage is still desired after #46106's runtime fix, rebase and land these tests or add equivalent coverage in `src/gateway/server.agent.gateway-server-agent-b.test.ts` proving that internal `agent` runs created through the RPC path deliver final WebChat chat events for both main-session and direct WebChat session keys.
