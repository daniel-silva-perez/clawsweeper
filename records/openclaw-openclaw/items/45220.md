## What This Changes

This PR adds `logs` to the CLI state-migration preflight exemption in `shouldMigrateStateFromPath()` and adds argv/path regression coverage for `openclaw logs --follow`.

## Best Possible Solution

Land the focused CLI startup-policy fix with regression coverage so `openclaw logs --follow` connects without doctor preflight notes, while leaving Gateway `logs.tail`, log formatting, and migration behavior for mutating commands unchanged.

## Solution Assessment

Yes. Adding `logs` to the existing read-only command exemption is the narrowest maintainable fix; no Gateway RPC or log-tail behavior needs to change.
