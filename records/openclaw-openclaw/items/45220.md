## Summary

Keep this PR open for maintainer review. Current main and the latest release still omit `logs` from the state-migration preflight skip list, while the PR is a narrow policy/test change that matches the existing non-mutating command pattern.

## What This Changes

The PR adds `logs` to `shouldMigrateStateFromPath()` read-only exemptions and adds argv/path regression tests for `openclaw logs --follow`.

## Best Possible Solution

Land the focused CLI startup-policy fix with regression coverage so `openclaw logs --follow` connects without doctor preflight notes, while leaving Gateway `logs.tail`, log formatting, and migration behavior for mutating commands unchanged.

Yes. A source-level reproduction is high confidence: on current main, `shouldMigrateStateFromPath(["logs"])` returns true by falling through, and `ensureConfigReady()` uses that result to run doctor preflight before command execution.

## Solution Assessment

Yes. The PR changes the implicated command-path gate directly and adds coverage for both full argv parsing and direct command-path evaluation, which is the narrowest maintainable fix for this startup-policy bug.
