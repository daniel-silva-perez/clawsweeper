## Summary

Keep this PR open for maintainer review. Current `main` and the latest release still run the doctor config preflight for `openclaw logs --follow`, while the proposed diff is focused, matches the existing read-only command exemption pattern, and has no blocking review findings.

## What This Changes

This PR adds `logs` to the CLI state-migration preflight exemption and adds argv/path regression cases for `openclaw logs --follow`.

## Best Possible Solution

Land this focused CLI startup-policy change with regression coverage so `openclaw logs --follow` connects without doctor preflight notes, while leaving Gateway `logs.tail`, log formatting, and migration behavior for mutating commands unchanged.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: `openclaw logs --follow` resolves to the `logs` command path, current `shouldMigrateStateFromPath()` does not exempt it, and `ensureConfigReady()` runs doctor preflight from that result before command execution.

## Solution Assessment

Yes. Adding `logs` to the existing read-only command exemption is the narrowest maintainable fix; no Gateway RPC or log-tail behavior needs to change.
