## Summary

Keep open. Current main has ClawHub search for CLI/Gateway/UI and a hot-reloaded skills snapshot, but it does not implement the requested agent-facing runtime skill discovery tool, structured capability metadata in OpenClaw skill frontmatter, or cross-tier semantic routing. The related items are adjacent but not canonical replacements for this broader design request.

## Best Possible Solution

Keep this as a product/API design item. A good implementation would reuse ClawHub semantic search where appropriate, add a deliberate agent-facing discovery surface or plugin API for local eligible skills, keep installs explicit and approval-backed, define optional capability/tag metadata without breaking existing SKILL.md files, and add docs/tests around snapshot refresh, allowlists, and agent-visible tool policy.

## Risks / Open Questions

- Closing as implemented would conflate user-facing ClawHub search with the requested agent-facing runtime discovery path.
- Closing as clawhub would be premature: a plugin can register agent tools, but the request also touches core-owned skill snapshots, prompt composition, local skill metadata, and approval-sensitive install/discovery behavior.
