## Summary

Keep open. Current main has static per-turn skill snapshots, prompt guidance to scan the injected `<available_skills>` list, ClawHub CLI search, and hot refresh of skill snapshots, but it does not provide an agent-callable runtime `skill_search` tool, semantic matching surface, or structured skill capability metadata requested here.

## What This Changes

Add runtime skill discovery and semantic matching so agents can search eligible skills with richer metadata instead of relying only on the startup `<available_skills>` list.

## Best Possible Solution

Define a maintainer-approved runtime discovery contract that respects existing skill allowlists, load-time gates, prompt budgets, and third-party skill trust boundaries; if approved, expose it through a narrow core or plugin API and document optional metadata without breaking AgentSkills compatibility.

## Reproduction Assessment

Not applicable as a feature request. The source check is high-confidence: `rg -n "skill_search"` returns no implementation, while current prompt code only injects a static `<available_skills>` catalog.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer design around whether search belongs in core, a plugin tool, ClawHub, or a shared API seam before a focused code PR should be attempted.
