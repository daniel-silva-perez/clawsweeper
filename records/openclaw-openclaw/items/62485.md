## Summary

Keep open. Current main does not implement the PR’s intended fix: the CLI banner help path still statically pulls in the config barrel and synchronously calls `createConfigIO().loadConfig()` while Commander is generating help. The PR is still relevant to the linked CPU-hang report, but its current patch also needs functional correction before merge because the dynamic-import result is not actually cached after the async promise settles.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #62484, but request a corrected patch: remove the static config import behind a proper runtime boundary, cache the async result correctly, preserve configured tagline behavior when config is already available, and add focused regression coverage for `openclaw tui -h` help rendering plus banner tagline modes.
