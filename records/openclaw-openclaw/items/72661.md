## Summary

## What This Changes

The PR changes Feishu mention normalization to strip only a leading receiving-bot mention and adds regression tests that preserve inline self-mentions in multi-bot group messages.

## Best Possible Solution

Land this PR or an equivalent rebased patch as the canonical fix for #72504, keeping leading-prefix stripping for command UX while preserving inline self-mentions for multi-bot routing. The broader open #72532 approach should not supersede it because preserving every self-mention would risk the direct-message slash-command path.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: current main passes `botOpenId` to `normalizeMentions`, and that function globally replaces every matching self mention with an empty string, so `Hey @_bot_a @_bot_b please coordinate` loses BotA's own tag before the model sees it.

## Solution Assessment

Yes. The leading-prefix-only strip is the narrowest maintainable fix I found because it keeps the #35994 `@Bot /command` behavior while restoring semantic inline self-mentions needed for the linked multi-bot group bug.
