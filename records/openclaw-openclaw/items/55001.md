## Summary

Keep this PR open. Current main still matches the reported failure mode: `/queue` is included in the inline-directive clearing guard, and prompt-bearing messages then pass through `clearInlineDirectives`, which clears the queue directive before `perMessageQueueMode` is computed. The PR remains plausible focused bug-fix work rather than obsolete cleanup; the provided review context shows an initial metadata-restore gap was identified and the PR added a second commit intended to restore the additional raw queue fields.

## Best Possible Solution

Keep the PR open, rebase or refresh it against current main, and land a focused fix that preserves queue directive state through the inline clearing path while restoring `hasQueueOptions` and all raw queue fields. Add regression tests around `resolveReplyDirectives` or the smallest seam that proves `/queue steer <prompt>` produces a per-message queue mode and that mixed queue options still validate/apply correctly.
