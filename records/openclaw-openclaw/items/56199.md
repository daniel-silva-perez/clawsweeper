## Summary

Keep open. Current main has a bundled Volcengine provider and improved auth-probe plumbing, but the repository evidence does not prove that a Volcano Engine Lite/DeepSeek UUID-like key now succeeds on real inference, and OpenClaw itself does not reject that key format locally.

## What This Changes

The issue reports that a Volcano Engine Lite/DeepSeek key in UUID-like format passes OpenClaw's test path but fails during actual model use with `401 The API key format is incorrect`.

## Best Possible Solution

Verify the failing path with a Volcengine Lite/DeepSeek credential, then either document it as a service/account setup mismatch or adjust the Volcengine plugin so setup/probe checks the same endpoint and model family that real inference will use.

## Solution Assessment
