## Summary

Keep this PR open, but do not merge the current revision. Current main does not already implement the Android-side normalization, and the follow-up commit addressed the earlier `<final>` role/nesting concerns, but the cron-noise heuristic can still hide real user-authored chat history messages.

## What This Changes

The PR changes Android `ChatController` history parsing to drop user messages matching a cron-completion text heuristic and to unwrap top-level `<final>...</final>` wrappers from assistant/system text before rendering.

## Best Possible Solution

Keep the Android rendering fix focused, but replace the cron substring heuristic with an exact synthetic marker contract or metadata-backed signal and add a small Android parser regression test for both the dropped marker and a legitimate user sentence that mentions cron output. The `<final>` unwrap can stay role-scoped if tests cover user-authored tag examples and nested/multiple wrapper preservation.

## Reproduction Assessment

Yes. A PR-branch `chat.history` payload with a user text like `A cron job I scheduled just completed successfully; can you explain the logs?` would pass the new `isCronDeliveryNoise` check and be removed during `parseHistory`.

## Solution Assessment

No, not in the current form. The Android-side `<final>` normalization is narrow enough after the follow-up commit, but the cron filter should not infer a control message from broad user-visible prose.
