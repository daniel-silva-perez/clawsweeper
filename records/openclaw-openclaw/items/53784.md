## Summary

Keep this PR open. Current main still routes ordinary Feishu MP3/WAV/M4A/FLAC/AAC audio as generic files unless the newer explicit voice-intent path transcodes them, so the PR’s central behavior is not implemented. The branch remains relevant, but it needs maintainer review because it overlaps with `audioAsVoice`, adds direct media-duration parsing, and relies on mapping non-Opus codecs to Feishu `file_type: "opus"`.

## What This Changes

The PR changes Feishu outbound media routing so common audio formats are uploaded and sent as playable Feishu audio messages, adds duration parsing through `music-metadata`, and updates Feishu media tests.

## Best Possible Solution

Keep the PR open for maintainer review and rework. Maintainers should decide whether ordinary Feishu audio attachments should become native audio bubbles by direct `file_type: "opus"` mapping, by reusing/transcoding through the existing `audioAsVoice` path, or only when an explicit voice intent is present; then update `extensions/feishu/src/media.ts`, focused Feishu media tests, package/lockfile metadata, and docs consistently.
