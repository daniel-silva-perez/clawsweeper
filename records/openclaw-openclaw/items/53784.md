## Summary

Keep open. Current main deliberately keeps ordinary MP3 and other common audio attachments as Feishu files unless voice delivery is explicitly requested, so this PR is not implemented or obsolete. The branch may be useful, but it changes Feishu audio policy and adds dependency/duration behavior, which needs maintainer review before merge.

## What This Changes

The PR broadens Feishu outbound audio routing to treat common audio extensions and audio MIME types as playable audio messages, maps those uploads to `file_type: "opus"`, adds `music-metadata` duration parsing, and updates Feishu media tests/package metadata.

## Best Possible Solution

Decide the Feishu audio contract explicitly. If direct playable bubbles for common audio formats are desired, merge or request a narrow rework that updates Feishu routing, tests, docs, package metadata, and lockfile together. If not, keep the shipped `audioAsVoice`/`asVoice` transcode path and close or supersede this branch with that policy clearly stated.
