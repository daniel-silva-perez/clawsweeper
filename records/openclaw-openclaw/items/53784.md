## Summary

Keep open. Current main does not implement this PR's default behavior for common Feishu audio attachments: MP3/WAV/M4A-style media still route as files unless voice delivery is explicitly requested. The PR is useful and the discussion includes live Feishu validation plus the community `@m1heng-clawd/feishu` plugin as supporting context, but it is not merge-ready because it changes the shipped Feishu audio policy without matching docs and adds a runtime dependency without updating the Feishu lockfile importer.

## What This Changes

The PR broadens Feishu outbound media routing so common audio extensions and audio MIME types upload as `file_type: "opus"` and send as `msg_type: "audio"`, adds `music-metadata` duration parsing, and updates Feishu media tests/package metadata.

## Best Possible Solution

Decide the Feishu audio contract explicitly. If direct playable bubbles for common audio formats are desired, land a rebased narrow change that updates routing, duration handling, docs, package metadata, lockfile, and tests together; otherwise keep the shipped explicit `audioAsVoice` / `asVoice` transcode flow and close or supersede this branch with that policy stated.
