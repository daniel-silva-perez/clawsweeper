## Summary

Keep open. Current main still does not implement the PR's direct common-audio playable-bubble default; it deliberately keeps ordinary MP3 audio as a file attachment and uses explicit `audioAsVoice`/`asVoice` transcoding for Feishu voice delivery. The PR contains useful work and responsive discussion, but it changes Feishu product behavior and needs maintainer policy, package/lockfile, docs, and rebase review rather than cleanup closure.

## What This Changes

The PR broadens Feishu outbound routing so common audio extensions and audio MIME types upload as `file_type: "opus"` and send as `msg_type: "audio"`, adds `music-metadata` duration parsing, and updates Feishu media tests/package metadata.

## Best Possible Solution

Decide the Feishu audio contract explicitly. If direct playable bubbles for common formats are desired, land a rebased narrow change that updates routing, duration handling, tests, docs, package metadata, and lockfile together; otherwise keep the shipped explicit voice-transcode path and close or supersede this branch with that policy stated.
