## Summary

Keep this PR open. Current main still sends ordinary Feishu MP3/WAV/M4A/FLAC/AAC media as generic files unless the newer voice-intent path explicitly transcodes them, so the PR's central behavior is not implemented on main. The PR remains relevant but needs maintainer rework because current main now has an `audioAsVoice` transcoding path and the branch adds a direct `music-metadata` parser dependency plus non-Opus-to-`file_type=opus` behavior that should be a Feishu product/API decision.
