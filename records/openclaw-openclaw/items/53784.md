## Summary

Keep open. Current main does not implement the PR's intended default behavior: ordinary MP3/WAV/M4A/FLAC/AAC Feishu media still route as regular files unless an explicit voice intent triggers the Ogg/Opus transcode path. The branch is focused, but it needs maintainer policy review because it changes Feishu audio semantics by mapping non-Opus codecs directly to `file_type: "opus"` and adds a runtime metadata dependency.

## What This Changes

This PR changes Feishu outbound media routing so common audio extensions and audio MIME types upload as `file_type: "opus"` and send as `msg_type: "audio"`, adds `music-metadata` duration parsing, and updates Feishu media tests/package metadata.

## Best Possible Solution

The desired end state is a maintainer-approved Feishu audio policy: either keep ordinary audio attachments as files and use the explicit `asVoice` transcode path, or deliberately support direct playable audio bubbles for common formats with matching Feishu tests, docs, package metadata, and dependency/lockfile updates.
