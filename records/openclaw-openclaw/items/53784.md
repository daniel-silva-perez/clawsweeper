## Summary

Keep open. Current main still keeps ordinary MP3/WAV/M4A-style Feishu attachments as files unless voice intent is set, so the PR's central behavior is not implemented; the branch still needs maintainer policy review plus docs and lockfile alignment before it is mergeable.

## What This Changes

The PR changes Feishu outbound media routing so common audio extensions and audio MIME types upload as `file_type: "opus"` and send as `msg_type: "audio"`, adds `music-metadata` duration parsing, and updates Feishu package metadata and media tests.

## Best Possible Solution

Make the Feishu audio contract explicit. If direct playable bubbles for common formats are desired, land a rebased narrow change that updates routing, duration handling, tests, docs, package metadata, and lockfile together; otherwise keep ordinary attachments as files and route common audio through the existing `asVoice`/TTS transcode path.
