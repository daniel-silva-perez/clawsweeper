## Summary

Keep open. Current main has a deliberate Feishu policy where ordinary MP3-style attachments remain files unless the caller requests voice delivery, so this PR is not obsolete; however, the proposed default-audio behavior still needs maintainer policy review plus docs, lockfile, and rebase alignment before it is mergeable.

## What This Changes

The PR broadens Feishu outbound media routing so common audio extensions and audio MIME types upload as `file_type: "opus"` and send as `msg_type: "audio"`, adds `music-metadata` duration parsing, and updates Feishu media tests/package metadata.

## Best Possible Solution

Decide the Feishu audio contract explicitly. If direct playable bubbles for common audio formats are desired, land a rebased narrow change that updates routing, duration handling, tests, docs, package metadata, and lockfile together; otherwise keep the shipped explicit voice-transcode path and close or supersede this branch with that policy stated.
