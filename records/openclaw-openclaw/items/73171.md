## Summary

Keep this PR open. Current main does not implement the bounded gateway auth-resolution fallback: status scans still await auth resolution before the probe timeout is applied. The PR is a focused robustness candidate tied to the open local status/gateway false-negative report #73535, and it needs normal maintainer review rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a bounded auth-resolution fallback in the status scan path with focused tests for the timeout behavior, consider whether rejection should also fall back, and verify it against the related #73535 status false-negative surface.
