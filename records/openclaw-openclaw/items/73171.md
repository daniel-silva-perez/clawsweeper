Keep this PR open. Current main does not implement the PR's bounded gateway auth-resolution fallback; it still awaits `resolveGatewayProbeAuthResolution` before starting the probe, while the related user-facing status false-negative was fixed separately by a local status-RPC fallback. ## What This Changes

The PR adds a timeout-bounded fallback around status gateway auth resolution and a focused test where a never-settling auth lookup falls back to probing with empty credentials plus a warning.

## Best Possible Solution

Land a rebased version only if maintainers still want auth-stall hardening after the shipped local RPC fallback: the timeout should compose with current `resolveGatewayProbeSnapshot`, preserve warning visibility, and include focused timeout coverage plus rejection handling if that is desired. Otherwise, close after an explicit maintainer decision that the current local reachability fix is sufficient.
