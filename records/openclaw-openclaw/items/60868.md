## Summary

Keep open. The requested chain-aware overflow detection is not on current main, and the latest PR revision is useful but still has a blocking security regression in the embedded runner: raw nested prompt-error messages can be selected as overflow diagnostic text and then logged or passed into recovery handling.

## What This Changes

The PR adds a shared error-chain message collector and applies chain-aware context-overflow/failure classification across failover helpers, model fallback, embedded overflow compaction, and auto-reply handling, with regression tests plus small Discord/Gateway system-event trust and MiniMax/changed-lanes test edits.

## Best Possible Solution

Revise the branch so raw chain messages are used only as classifier candidates, while embedded-runner diagnostic, log, reset-helper, metadata, Control UI, and user-facing text use redacted formatter output or an equivalent redacted display candidate. Preserve chain-wide .error/.cause detection, billing/rate-limit precedence, structured status/code hints, and focused regression coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact prompt overflow text before diagnostics:** `src/agents/pi-embedded-runner/run.ts:1296-1300`
  - body: The new promptError path adds raw chain messages to errorTexts and then returns the matching segment as contextOverflowError.text. That text is logged by the existing context-overflow diagnostic path and can flow into recovery handling, so a nested provider error like `context length exceeded Authorization: Bearer ...` bypasses formatErrorMessage()/redactSensitiveText(). Use raw chain strings only for classification and keep the selected diagnostic text redacted.
  - confidence: 0.92

## Security Review

Status: needs_attention

Summary: The diff still lets raw nested prompt-error messages reach embedded-runner diagnostics.

Concerns:

- **[high] Raw nested prompt errors can leak secrets:** `src/agents/pi-embedded-runner/run.ts:1296`
  - body: Provider and transport errors can include Authorization headers, API keys, or serialized request metadata inside nested .error/.cause.message fields. The embedded-runner change opts into raw chain messages and can select one as contextOverflowError.text, bypassing the existing redacted formatErrorMessage() boundary before diagnostics and recovery handling.
  - confidence: 0.9
