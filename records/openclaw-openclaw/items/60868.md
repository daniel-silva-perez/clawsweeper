## Summary

Keep open. Current main still only checks formatted top-level or ordinary cause-chain text in the affected runtime paths, so the PR is not obsolete. The latest PR head addresses part of the earlier redaction concern for auto-reply output, but the embedded runner still opts into raw nested messages and then logs/stores the selected overflow text, which is a blocking security regression before merge.

## What This Changes

The PR adds a shared error-chain message collector, applies chain-aware context-overflow classification in model fallback, embedded Pi overflow handling, and auto-reply failure classification, and adds regression tests plus small Discord/Gateway system-event trust and changed-lanes test updates.

## Best Possible Solution

Revise the PR so raw chain candidates are used only for classification, while embedded-runner diagnostics, result metadata, Control UI surfaces, and external replies use redacted text. Keep the useful chain-aware overflow and billing-precedence coverage, and let a maintainer decide whether the unrelated trust/lane edits belong in this branch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact raw overflow chain text before surfacing it:** `src/agents/pi-embedded-runner/run.ts:1296`
  - body: The embedded runner adds `collectErrorChainMessages(promptError, { redact: false })` to the overflow candidates and then reuses the selected `errorText` in the context-overflow log and `meta.error.message`. A nested provider error that contains an Authorization header, API key, or serialized request metadata can therefore leak even though the collector redacts by default. Keep raw candidates classifier-only and derive the logged/stored text from a redacted candidate.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: Security review found a concrete secret-redaction regression in the embedded-runner overflow path.

Concerns:

- **[high] Raw nested overflow messages can leak:** `src/agents/pi-embedded-runner/run.ts:1296`
  - body: The PR opts the embedded runner into raw chain messages and then logs/stores the selected overflow text. Provider and transport errors can include bearer tokens, API keys, request headers, or serialized request metadata in nested `.error` or `.cause.message` fields, so this needs a redacted display/log value separate from raw classifier candidates.
  - confidence: 0.9
