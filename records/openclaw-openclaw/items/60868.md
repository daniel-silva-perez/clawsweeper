## Summary

Keep open. Current main still lacks the PR's full `.error` / `.cause` chain message scanning, so the work is not obsolete, but the latest PR head still has a blocking security regression in the embedded-runner overflow diagnostic path.

## What This Changes

The PR adds a shared error-chain message collector and applies chain-aware context-overflow/failure classification across failover, model fallback, embedded overflow compaction, and auto-reply handling, with tests plus small Discord/Gateway system-event trust and changed-lanes test updates.

## Best Possible Solution

Revise the branch so raw chain messages are used only as classifier candidates, while any diagnostic text, logs, stored failure text, Control UI output, or user-facing reply comes from `formatErrorMessage()`, `redactSensitiveText()`, or an equivalent redacted display path. Keep the chain-wide overflow detection and billing/rate-limit precedence coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep raw chain messages out of overflow diagnostics:** `src/agents/pi-embedded-runner/run.ts:1296`
  - body: `collectErrorChainMessages(promptError, { redact: false })` can now provide the selected overflow `errorText`, and the existing context-overflow diagnostic logs that text. A nested provider error containing an Authorization header or API key plus an overflow hint would bypass the redacted `formatErrorMessage()` path and leak into logs; use raw candidates only for matching, then surface redacted text.
  - confidence: 0.92

## Security Review

Status: needs_attention

Summary: The PR still introduces a concrete secret-redaction regression in the embedded-runner overflow diagnostic path.

Concerns:

- **[high] Raw nested error messages can leak to logs:** `src/agents/pi-embedded-runner/run.ts:1296`
  - body: The embedded runner opts into raw chain messages for prompt-error overflow detection, then propagates the matching raw segment as diagnostic text. Provider and transport errors can include bearer tokens, API keys, or serialized request metadata in nested `.cause` or `.error.message` fields, so the raw value should stay classifier-only or be redacted before surfacing.
  - confidence: 0.9
