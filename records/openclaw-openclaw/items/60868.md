## Summary

Keep this PR open. Current main has partial nested error support through `formatErrorMessage()` and failover classification, but it still does not implement the PR’s full `.error`/`.cause` chain-message behavior in the model fallback, embedded-runner overflow, or auto-reply classification paths. The PR remains useful, but the supplied review comments identify functional precedence issues that need a focused maintainer revision before landing.

## What This Changes

The PR adds a shared error-chain message collector and applies it to agent model fallback, embedded overflow compaction, and auto-reply error classification, with related regression tests plus minor docs and MiniMax test ordering edits.

## Best Possible Solution

Revise the PR around a shared, cycle-safe chain-candidate helper that supports both `.error` and `.cause`, then use it consistently in model fallback, embedded-runner overflow compaction, and auto-reply classification while preserving billing/rate-limit precedence and structured error hints. Keep focused regression tests for `Error.cause`, custom `Error.error`, plain transport objects, and long billing wrappers.
