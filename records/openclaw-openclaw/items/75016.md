## Summary

Keep open. The PR targets a real edge in reasoning streaming, but the proposed same-line delta still is not appendable against the previously emitted formatted text, so it can leave the formatter's premature closing underscore in the accumulated stream instead of producing the new formatted reasoning snapshot.

## What This Changes

This PR changes `emitReasoningStream` so a prior formatted reasoning message ending in `_` can be treated as a prefix match when computing `data.delta` for same-line reasoning appends.

## Best Possible Solution

Keep the PR open and revise the reasoning stream delta path so same-line appends either compute the delta before applying markdown wrappers or explicitly use replacement/snapshot semantics; add focused coverage for two consecutive thinking updates on the same line.

## Reproduction Assessment

Yes. A deterministic source-level reproduction is `formatReasoningMessage("Checking files")` followed by `formatReasoningMessage("Checking files done")`; the PR delta appends to the prior formatted text as `Reasoning:\n_Checking files_ done_`, which is not the expected formatted output.

## Solution Assessment

No. Ignoring the prior trailing underscore avoids the full-snapshot fallback, but it does not repair the already-emitted closing underscore; the maintainable fix should preserve append semantics on raw reasoning text or signal replacement for formatted snapshots.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

- **[P2] Preserve appendable thinking deltas:** `src/agents/pi-embedded-subscribe.ts:847`
  - body: For a same-line append from `Reasoning:\n_Checking files_` to `Reasoning:\n_Checking files done_`, this branch emits `data.delta === " done_"`. A client that appends deltas to the previous event text gets `Reasoning:\n_Checking files_ done_`, leaving the premature closing underscore in the middle of the line. Please compute the delta before adding the per-line markdown wrapper, or emit an explicit replacement/snapshot for this case.
