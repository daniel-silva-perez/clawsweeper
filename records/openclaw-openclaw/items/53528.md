Keep this PR open. Current main still lacks a cron-owned watcher or idle reload path for direct jobs.json writes after an empty startup, and the current PR diff is not merge-ready because the watcher only invalidates memory without waking the scheduler or handling self-write and rename/missing-file cases. ## Risks / Open Questions

- Current main can still miss direct jobs.json writes after an empty cron startup because no timer or watcher is active to reload the file.
