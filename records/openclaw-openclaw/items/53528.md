## Summary

Keep open. The PR targets a real cron gap that current main still has, and the linked issue was closed specifically because this PR is the canonical tracker. The patch is not ready to merge because the watcher only clears cache state, does not wake an idle scheduler, can react to the service's own writes, and is fragile when the watched file is missing or replaced.

## What This Changes

The PR adds a `fs.watch` helper for `jobs.json`, stores watcher cleanup on cron service state and the test harness, and starts/stops the watcher with the cron service.

## Best Possible Solution

Implement external `jobs.json` change handling inside the cron service: observe the file or parent directory robustly, suppress internal persistence events, serialize reload/recompute under the existing lock, wake or re-arm scheduling after external changes, clean up watchers on stop, and cover empty-startup, missing-file, rename, and self-write cases with focused tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Wake the scheduler when the watch fires:** `src/cron/service/store.ts:88-91`
  - body: The callback only clears the cached store. If cron started with no runnable jobs, `armTimer()` left no timer active, so an external write can fire this callback but no later path calls `ensureLoaded()` or re-arms the scheduler; the new job remains unscheduled until another cron operation or restart.
  - confidence: 0.94
- **[P1] Ignore internal store writes before invalidating state:** `src/cron/service/store.ts:88-91`
  - body: `saveCronStore()` rewrites `jobs.json`, so add/update/remove can trigger this watcher from cron's own persist. Clearing `state.store` between `persist()` and the caller's following `armTimer()` can leave the scheduler unarmed or make later optional store accesses silently no-op. Suppress self-write events and do reload/re-arm work under the cron lock.
  - confidence: 0.9
- **[P2] Retry watching when jobs.json is absent or replaced:** `src/cron/service/store.ts:88-101`
  - body: The PR installs one file-bound `fs.watch(path)` and swallows setup failures. If `jobs.json` is missing at startup, or if rename-based persistence replaces the watched inode, no directory watch, retry, or fallback wake is installed, so later external writes can be missed permanently.
  - confidence: 0.86
