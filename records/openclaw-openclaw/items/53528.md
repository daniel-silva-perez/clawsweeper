## Summary

Keep open. Current main still lacks a cron-owned external observation path for direct `jobs.json` edits, and the docs describe external edit reload behavior, so the PR is not obsolete. The submitted branch is not merge-ready because its watcher only clears the cache, does not wake or re-arm an idle scheduler, and leaves race/missing-file/rename-watch cases unresolved; the security pass did not find unrelated dependency, CI, secret, or package-script changes.

## What This Changes

The PR adds a cron service `fs.watch` helper for `jobs.json`, stores a cleanup callback in cron service state, starts and stops that watcher with the cron service, and updates the cron test-harness state shape.

## Best Possible Solution

Implement this as cron-owned external-change handling rather than passive cache invalidation: observe `jobs.json` robustly, suppress internal writes, run the reload/recompute/persist-if-needed path under the existing lock, wake or re-arm scheduling after external changes, clean up on stop, and cover the empty-startup direct-write case with focused tests.
