## Summary

Keep this issue open. Current main still backs up the entire state directory by default and only skips installed plugin dependency trees; it does not provide built-in large-state exclusions, custom exclude patterns, progress, max-size warnings, or clearer large-archive failure policy. The discussion also points to open PR #44288 as an implementation candidate using closing language, so the issue should remain open until that PR is reviewed/landed, replaced, or closed. The keepmyclaw ClawHub plugin linked in discussion is a useful workaround, but it does not replace fixing the built-in backup command.

## Best Possible Solution

Keep this issue open and review or replace PR #44288 as the current implementation candidate. The built-in backup command should gain conservative large-state exclusions or configurable exclude patterns, restore-safe manifest/verify behavior, progress or size/failure reporting, focused tests, and docs. The keepmyclaw ClawHub plugin is useful to mention as a workaround, but it should not close the core backup bug.

## Risks / Open Questions

- The tiny dangling `.tmp` symptom appears partly mitigated by current atomic temp archive handling and `finally` cleanup, so maintainers should scope remaining work carefully around large-state defaults, progress, failure reporting, and timeout/kill cleanup rather than assuming every old temp-file failure mode is still present.
