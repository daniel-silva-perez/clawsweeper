## Summary

Keep open. The checked-out main at 0f3a9d812b40cbc8e545abcb80a1f12a93946cf6 still backs up the whole state directory by default, only filters installed plugin node_modules, and has no built-in large-state exclusions, custom exclude patterns, progress, max-size warning, or explicit large-archive failure policy. The provided discussion also shows open PR #44288 is still the implementation candidate with closing syntax for this issue, so the issue should remain open until that PR is reviewed, landed, replaced, or closed.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #44288 as the current implementation candidate. The first-party backup command should gain conservative large-state exclusions or configurable exclude patterns, restore-safe protected-path behavior, manifest and verify semantics, progress or size/failure reporting, focused tests, docs, and security review.
