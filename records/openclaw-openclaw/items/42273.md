## Summary

Keep open. Current `main` and the latest shipped release still lack the requested large-state backup controls, while open PR #44288 remains the active implementation candidate with closing syntax for this issue.

## What This Changes

The issue requests that `openclaw backup create` handle large `.openclaw` directories without silent stalls, either by completing reliably or by adding clear exclusions, progress, and failure reporting.

## Best Possible Solution

Keep this issue open until PR #44288 or a narrower replacement lands. The first-party backup command should gain restore-safe large-state exclusions or configurable patterns, clear progress/failure behavior for long archives, focused docs/tests, and separate handling for timeout temp-file cleanup where #50442 remains relevant.

## Reproduction Assessment

Partially yes: current-main source checks reproduce the missing-control path because `--no-include-workspace` still leaves the full state asset in scope and no large-state exclude/progress/size flags exist. The exact multi-GB stall itself was not live-reproduced in this read-only review.
