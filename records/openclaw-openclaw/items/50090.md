## Summary

Keep open. Current main has addressed important discovery, watcher, and gateway-backed installer-scan pieces, but the umbrella still contains unresolved OpenClaw-side security and product work: direct ClawHub skill installs use a separate copy path, the skill contract has no permission/signing/provenance or trigger-quality schema, and the related narrower trackers do not supersede the whole topic.

## What This Changes

Requests a reliable community skill ecosystem around ClawHub, including skill discovery/loading, publish reliability, watcher hot reload, trust/security controls, composition, trigger/linting, and agent-pack style sharing.

## Best Possible Solution

Preserve this as a consolidation tracker or split it into maintained narrow trackers: mark discovery and watcher items as solved, route pure ClawHub publish/service bugs to the ClawHub project, and define OpenClaw core/API work for direct ClawHub install scanning, permission/provenance manifests, lifecycle/audit hooks, and trigger/lint schema.
