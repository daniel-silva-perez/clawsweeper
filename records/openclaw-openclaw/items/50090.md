## Summary

Keep open. Current main has implemented major parts of the skills ecosystem, including documented local/workspace/extra-dir discovery and watcher roots, but the umbrella still contains live core/security work: direct ClawHub skill install/update still bypasses the built-in scanner and before_install policy path, permission-manifest/trust work remains open, post-install lifecycle/audit hooks are not present, and trigger-quality fields/linting are not implemented. Narrower trackers exist, but there is no single maintainer-designated canonical superseder for the whole issue.

## Best Possible Solution

Keep this issue open until maintainers explicitly consolidate it. The practical path is to mark discovery/watcher items as shipped, route ClawHub CLI/service publish bugs to openclaw/clawhub, and keep or link concrete OpenClaw trackers for direct ClawHub install scanning, permission manifests/provenance, skill install lifecycle hooks including post-install audit events, and trigger-quality/linting fields if maintainers want those in the core skill contract.
