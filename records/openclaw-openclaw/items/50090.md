## Summary

Keep open. Current main has solved meaningful skill discovery and watcher pieces, but the issue still contains unresolved OpenClaw-side trust, direct ClawHub install-scan coverage, permission/provenance schema, trigger-quality schema, and external ClawHub CLI/service ownership questions that are not safely closable as implemented or fully superseded.

## What This Changes

The issue asks OpenClaw to make community skill development and ClawHub publishing/installing reliable, discoverable, composable, and safer to trust.

## Best Possible Solution

Keep this as a consolidation tracker or split it into maintained narrow trackers: mark discovery and watcher items as solved, route pure ClawHub publish/service bugs to ClawHub ownership, and define OpenClaw work for direct ClawHub install scanning, permission/provenance manifests, lifecycle/audit hooks, and trigger/lint schema.

## Reproduction Assessment

Not applicable as a single reproduction: this is an umbrella ecosystem/security tracker, not one isolated bug. Focused source checks do confirm that discovery/watcher paths are covered while direct ClawHub install scanning and skill manifest/trigger fields remain incomplete.

## Solution Assessment

No as a one-shot implementation path. The maintainable solution is a maintainer-scoped split between OpenClaw runtime/API work, direct native ClawHub install behavior, external ClawHub CLI/service work, and protected security design.
