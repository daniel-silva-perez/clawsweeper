## Security Review

Status: needs_attention

Summary: The issue remains security-sensitive because third-party skill supply-chain trust, permission/provenance schema, and native install-time enforcement gaps are still unresolved.

Concerns:

- **[medium] Native ClawHub installs bypass local install scanning:** `src/agents/skills-clawhub.ts:254`
  - body: The native ClawHub path downloads, extracts, verifies SKILL.md, and copies the skill directory, while scanner/before_install enforcement is only wired through gateway-backed installer metadata. That leaves direct OpenClaw skill installs without the same local enforcement surface.
  - confidence: 0.9
- **[medium] Skill permission and provenance schema is undefined:** `src/agents/skills/types.ts:19`
  - body: The current SKILL.md contract supports basic metadata, gating, invocation controls, and installer specs, but source searches found no permission manifest, provenance/signing, trigger_examples, or skill_type schema for operator review and enforcement.
  - confidence: 0.86
