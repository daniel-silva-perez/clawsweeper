## Summary

Keep open. Current main has shipped important pieces of the skill ecosystem, especially multi-root discovery, root-based skill watching, native ClawHub search/install/update, and scanner/hooks for gateway-backed skill dependency installs. The umbrella still contains unresolved OpenClaw-side security and product work: direct ClawHub skill install is still separate from the scanned installer-metadata path, permission manifests/signing/trust are not implemented in the skill contract, there is no post-install audit hook, and trigger-quality/linting fields are absent. Related narrower trackers exist, but no single maintainer-designated superseder covers the whole umbrella.

## What This Changes

Request to make community skill development and ClawHub reliable by fixing skill discovery, publish/install reliability, watcher reloads, security/trust boundaries, composition, and trigger-quality gaps.

## Best Possible Solution

Keep this issue open for maintainer consolidation. Mark discovery and watcher concerns as shipped, route pure ClawHub CLI/service publish bugs to openclaw/clawhub, and either keep this umbrella or link narrower OpenClaw trackers for direct ClawHub install scanning, permission/provenance/trust manifests, post-install lifecycle/audit hooks, and trigger-quality/linting contract design.
