## Summary

Keep open. Current main has shipped several concrete skill-ecosystem pieces, including workspace/managed/extra-dir discovery, root-based skill watching, native ClawHub search/install/update, gateway-backed installer scanning, and a generic `before_install` hook. The umbrella still contains unresolved OpenClaw-side security and product work: direct ClawHub skill install/update still bypasses the scanned installer-metadata path, permission manifests/signing/trust are not implemented, post-install lifecycle/audit hooks are absent, and trigger-quality/linting fields are not part of the skill contract. Related narrower trackers exist, but there is no single maintainer-designated superseder for the whole issue.

## Best Possible Solution

Keep this issue open for maintainer consolidation. Mark the discovery and watcher portions as shipped, route standalone ClawHub CLI/service publish bugs to openclaw/clawhub, and keep or link concrete OpenClaw trackers for direct ClawHub install scanning, permission manifests/provenance, lifecycle hooks including post-install audit events, and trigger-quality/linting if maintainers want those in the core skill contract.
