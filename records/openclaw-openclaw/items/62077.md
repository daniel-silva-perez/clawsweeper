## Summary

Keep open. Current main still builds full per-message inbound metadata for direct Telegram chats, has no `messages.inboundMetadataMode` / compact-direct config surface, and the linked compact-metadata PR is open and unmerged rather than shipped.

## Best Possible Solution

Keep this issue open and use it as supporting user evidence while reviewing PR #72749 or an equivalent core change. The likely fix belongs in core prompt assembly/config: add a documented `messages.inboundMetadataMode`-style option, route it into `buildInboundUserContextPrefix`, keep group and rich-context metadata full by default, strip any compact AI-facing prefix from user-visible history, and cover Telegram/direct, group, sanitizer, schema, and docs behavior before closing as implemented.
