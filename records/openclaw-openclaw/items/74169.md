Keep this PR open for maintainer review. ## Best Possible Solution

Keep this PR open but request maintainer review/changes. The best fix should either be a clearly scoped generic `callGateway` default alignment with a regression test for no env/config, or a retargeted fix for #74164 at the CLI/message plugin-action timeout boundary. For the linked issue, preserve the plugin boundary, avoid WeCom-specific core logic, and add focused tests around `message.action` timeout propagation plus docs/changelog if a user-facing CLI timeout changes.
