## Summary

Keep open for maintainer review. Current main still has the generic gateway-call wrapper fallback at 10000ms, and this PR's one-file fallback change does not yet line up with the linked WeCom regression because gateway-executed message actions pass an explicit 10000ms timeout before the generic fallback is reached.

## What This Changes

The PR modifies `src/gateway/call.ts` so calls without an explicit timeout resolve the shared preauth handshake timeout default of 15000ms instead of falling back to 10000ms.

## Best Possible Solution

Decide the scope explicitly. If the goal is generic gateway-call consistency, simplify the fallback, add a no-config/no-env regression test, and avoid claiming it fixes the WeCom media path. If the goal is the linked WeCom bug, align the timeout used by gateway-executed message actions at the CLI/outbound boundary with focused tests, keeping the core change plugin-agnostic.

## Review Findings
