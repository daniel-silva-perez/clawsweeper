## Summary

Keep this PR open: the generic timeout fallback change is directionally useful, but it does not fix the linked CLI media-send regression because the message-action gateway path still supplies an explicit 10000ms timeout before reaching the changed fallback.

## What This Changes

The PR changes `src/gateway/call.ts` so gateway calls without an explicit timeout resolve the shared preauth handshake timeout default instead of falling back to 10000ms.

## Best Possible Solution

Align the timeout at the gateway-executed message-action/send boundary with the shared preauth timeout semantics, keep the change plugin-agnostic, add regression tests for no-config/default and CLI media dispatch, and then land the useful generic fallback cleanup.
