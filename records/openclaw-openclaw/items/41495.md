## Summary

Keep this issue open. Current main still has the same contract mismatch: the agent prompt tells models to use legacy `buttons=[[{text,callback_data,style?}]]`, while the shared message tool exposes semantic buttons through `presentation.blocks`, and Telegram delivery renders buttons only from normalized presentation/interactive/channelData payloads rather than parsing raw assistant text.

## Best Possible Solution

Keep this open as a contract-alignment bug. Update `src/agents/system-prompt.ts` to instruct models to call `message(action="send")` with `presentation.blocks[{type:"buttons", buttons:[{label,value,style?}]}]`, update the prompt/schema and Telegram docs to match the shared presentation contract, and add a Telegram regression proving presentation button blocks render as inline keyboard buttons. Only add legacy raw-text or top-level `buttons` compatibility if maintainers explicitly want it and it is tightly validated.
