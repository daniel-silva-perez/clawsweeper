## Summary

Keep this issue open. Current main still has the contract mismatch: the agent prompt tells models to use legacy `buttons=[[{text,callback_data,style?}]]`, while the shared message tool and Telegram delivery path expect `presentation.blocks`/interactive payloads. The prior ClawSweeper comment correctly identified this, and current main at `bdfb408ce6eeb724a638c106104dbbcae50b612c` still shows the same behavior.

## Best Possible Solution

Keep this open as a contract-alignment bug. Update `src/agents/system-prompt.ts` and Telegram docs to instruct models to call `message(action="send")` with `presentation.blocks[{type:"buttons", buttons:[{label,value,style?}]}]`, update the stale prompt/schema tests, and add a Telegram delivery regression showing presentation button blocks render as inline keyboard buttons. Only add legacy raw-text or top-level `buttons` compatibility if maintainers explicitly want that behavior and it is tightly validated.
