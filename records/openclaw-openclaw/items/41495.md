## Summary

Keep this issue open. Current main still has the reported contract mismatch: the agent prompt and Telegram docs still steer models toward legacy `buttons=[[{text,callback_data,style?}]]` / top-level `buttons`, while the shared message tool and Telegram runtime now use `presentation.blocks[].buttons` or normalized interactive payloads. Raw assistant text containing JSON-like button rows is still delivered as text, not parsed into Telegram inline buttons.

## Best Possible Solution

Keep this open as a contract-alignment bug. Update `src/agents/system-prompt.ts` to instruct models to call `message(action="send")` with `presentation.blocks[{type:"buttons", buttons:[{label,value,style?}]}]`, update the prompt test and Telegram docs to match the shared presentation contract, and add a Telegram regression proving presentation button blocks render as inline keyboard buttons. Only add legacy raw-text or top-level `buttons` compatibility if maintainers intentionally want it and it is narrowly validated.
