## Summary

Keep this issue open. Current main still has the same contract mismatch described in the prior ClawSweeper comment: the agent prompt tells models to use legacy `buttons=[[{text,callback_data,style?}]]`, while the shared message tool schema exposes `presentation.blocks` and the Telegram runtime renders buttons from normalized presentation/interactive payloads, not raw assistant text.

## Best Possible Solution

Keep this open as a contract-alignment bug. Update `src/agents/system-prompt.ts` and Telegram docs to instruct models to call `message(action="send")` with `presentation.blocks[{type:"buttons", buttons:[{label,value,style?}]}]`, update the prompt/schema tests, and add a Telegram regression showing presentation button blocks render as inline keyboard buttons. Only add legacy raw-text or top-level `buttons` compatibility if maintainers explicitly want it and it is tightly validated.
