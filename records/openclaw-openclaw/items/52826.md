## Summary

Keep open. Current main has adjacent generic agent streams, live tool/session broadcasts, session lifecycle status, and an experimental `update_plan` tool, but it does not provide the requested durable session-level process/step event contract with the named `process.*` and `step.*` events or stable replayable step identity for reconnect/reload.

## Best Possible Solution

Keep this issue open and implement an additive Gateway protocol surface for durable session process events, likely a typed `process` or `session.process` event family plus a per-session/per-run replayable process ledger. It should bridge lifecycle, planning, tool/item, message, and artifact relationships into stable step ids, document the contract, and test live streaming plus reconnect/reload recovery.
