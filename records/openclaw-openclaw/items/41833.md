## Summary

Keep open. Current main still provides cron scheduling, cron run history, background task records, Task Flow orchestration, standing-order guidance, compaction preservation, and a narrow reminder guard, but it does not implement the requested native commitment/deferred-task primitive with `openclaw commit add/list/done`, first-class commitment records, generic done/cancel/status operations, cron/task reconciliation into commitment completion, overdue escalation, or startup/session surfacing for unscheduled promises. The related PENDING.md items are workaround/plugin-pattern discussions and do not supersede this core product request.

## Best Possible Solution

Keep this issue open and scope a first-class commitment record: due time, optional linked cron/task run, `add/list/done/cancel/status` operations, reconciliation from cron/task outcomes, overdue or failed-run escalation, and startup/session surfacing for uncompleted commitments. PENDING.md, standing orders, skills, and ClawHub packages can remain optional workarounds, but they should not be treated as a replacement for the requested native primitive.

## Risks / Open Questions

- Natural-language commitment capture needs product design for false positives, user consent, and when a promise should become durable state.
- Overdue escalation needs explicit channel routing, authority, notification, and privacy boundaries.
- The implementation should reuse cron, task records, Task Flow where appropriate, and the existing reminder guard instead of creating a parallel scheduler.
