## Summary

Keep this issue open. Current main has cron scheduling, background task records, Task Flow, standing orders, compaction preservation, and a reminder guard, but it still does not provide the requested native commitment primitive with `openclaw commit add/list/done`, durable commitment records, completion reconciliation, overdue escalation, or startup/session surfacing.

## Best Possible Solution

Design a native commitment layer above cron/tasks/Task Flow: durable commitment records with due time, owner/channel target, optional linked cron/task/flow run, `add/list/done/cancel/status` operations, reconciliation from cron/task outcomes, overdue or failed-run escalation, and startup/session surfacing for uncompleted commitments. Reuse the existing scheduler and task ledger rather than building a parallel runner.
