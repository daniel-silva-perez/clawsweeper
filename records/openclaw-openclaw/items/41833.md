## Summary

Keep this issue open. Current main still does not provide a native commitment/deferred-task primitive with `openclaw commit add/list/done`, durable commitment records, completion reconciliation, overdue escalation, or startup/session surfacing. The existing implementation covers cron scheduling, background task records, Task Flow, compaction preservation, and a narrow reminder guard, but those remain partial mitigations rather than the requested product lifecycle. The provided related PENDING.md reports are workspace-pattern or ClawHub workaround material and do not supersede this core native request.

## Best Possible Solution

Keep the issue open and scope a native commitment record with due time, optional linked cron/task/Task Flow run, `add/list/done/cancel/status` operations, reconciliation from cron/task outcomes, overdue or failed-run escalation, and startup/session surfacing for uncompleted commitments. Reuse cron, background tasks, Task Flow, plugin hooks, and the reminder guard where they fit rather than creating a parallel scheduler.

## Risks / Open Questions

- Natural-language commitment capture needs product design for false positives, explicit consent, and when conversation text becomes durable state.
- Overdue escalation needs clear routing, authority, privacy, notification, and failure semantics across channels and sessions.
- A ClawHub skill or workspace pattern can prototype part of the workflow, but treating it as the full solution would leave the requested native CLI/API and core lifecycle unresolved.
