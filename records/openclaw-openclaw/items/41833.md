## Summary

Current main still does not implement the requested native commitment/deferred-task primitive. The repo has cron scheduling, background task records, Task Flow, standing-order guidance, compaction preservation, plugin hooks, and a narrow reminder guard, but there is still no first-class `openclaw commit add/list/done` CLI/API, durable commitment record, completion reconciliation, overdue escalation, or startup/session surfacing for missed conversational commitments.

## Best Possible Solution

Keep this issue open and scope a native commitment record with due time, optional linked cron/task/flow run, `add/list/done/cancel/status` operations, reconciliation from cron/task outcomes, overdue or failed-run escalation, and startup/session surfacing for uncompleted commitments. The implementation should reuse cron, background tasks, Task Flow, plugin hooks, and the existing reminder guard where they fit, rather than creating a parallel scheduler.

## Risks / Open Questions

- Natural-language commitment capture needs product design for false positives, explicit user consent, and when conversational text should become durable state.
- Overdue escalation needs clear routing, authority, privacy, notification, and failure semantics across channels and sessions.
- A plugin or PENDING.md pattern can prototype parts of this, but treating that as the full solution would leave the requested native CLI/API and core status lifecycle unresolved.
