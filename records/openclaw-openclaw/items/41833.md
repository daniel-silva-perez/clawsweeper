## Summary

Current main still does not implement the requested native commitment/deferred-task primitive. OpenClaw has cron scheduling, background task records, Task Flow, standing-order guidance, compaction preservation, plugin hooks, and a narrow reminder guard, but there is still no first-class `openclaw commit add/list/done` CLI/API, durable commitment record, completion reconciliation, overdue escalation, or startup/session surfacing for missed conversational promises. The issue discussion and related PENDING.md reports describe useful workarounds, not a shipped core solution.

## Best Possible Solution

Keep this issue open and scope a first-class commitment record with due time, optional linked cron/task run, `add/list/done/cancel/status` operations, reconciliation from cron/task outcomes, overdue or failed-run escalation, and startup/session surfacing for uncompleted commitments. PENDING.md, standing orders, skills, and ClawHub packages can remain optional workarounds, but they should not replace the requested native primitive.

## Risks / Open Questions

- Natural-language commitment capture needs product design for false positives, explicit user consent, and when a conversational promise should become durable state.
- Overdue escalation needs clear channel routing, authority, privacy, notification, and failure semantics.
- A robust implementation should reuse cron, background task records, Task Flow, plugin hooks where useful, and the existing reminder guard instead of creating a parallel scheduler.
