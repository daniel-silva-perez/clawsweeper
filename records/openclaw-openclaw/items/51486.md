## Summary

Keep this PR open. Current main still has the broad `schtasks /Query` preflight in `readScheduledTaskRuntime()` before the task-scoped `/Query /TN ... /V /FO LIST` lookup, so the PR’s intended Windows daemon status fix has not been implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review or port the narrow fix onto current main: remove the broad availability preflight from `readScheduledTaskRuntime()`, query the task-scoped Windows scheduled task directly, preserve Startup-folder fallback and unknown-status behavior when that query fails or throws, and land the focused regression tests from the PR. The linked bug issue being closed is not enough to close this implementation candidate while main still lacks the code change.
