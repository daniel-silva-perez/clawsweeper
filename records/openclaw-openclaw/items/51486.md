## Summary

Keep open. Current main still has the broad Windows `schtasks /Query` preflight on the daemon runtime-status path, while this PR narrowly removes that preflight and preserves fallback handling, so it is still materially relevant and should get maintainer review rather than cleanup closure.

## What This Changes

The PR removes the global `schtasks /Query` preflight from Windows daemon runtime status reads, catches task-scoped query throws for fallback handling, and updates startup-fallback tests and fixtures.

## Best Possible Solution

Land or port the narrow daemon fix on current main: Windows runtime status should skip only the broad availability preflight, keep install/stop/restart availability checks intact, preserve Startup fallback and unknown-status behavior for failed or thrown task-scoped queries, and retain focused regression coverage for the single-call and thrown-query cases.

## Solution Assessment

Yes. The PR is the narrowest maintainable direction because it changes only runtime status reads, leaves lifecycle preflights in place, and handles thrown task-scoped queries through the existing non-zero/fallback branch.
