## Summary

Current main still has the reported Feishu streaming-card overlap bug: partial replies use snapshot merging, but block payloads still append as deltas, and the dispatcher test still expects the duplicate `hellolo world` result. Keep this issue open because open PR #74146 already targets the fix and references this issue with closing syntax.

## Best Possible Solution

Keep this issue open and review PR #74146 as the focused implementation candidate. The intended fix belongs in the Feishu plugin dispatcher/tests: reconcile overlapping block payloads with partial snapshot state using existing `mergeStreamingText` snapshot semantics, while preserving block-only fallback, exact duplicate suppression, non-overlap block behavior, final delivery, media delivery, and `disableBlockStreaming`.
