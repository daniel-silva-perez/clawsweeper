Fix Feishu streaming-card duplicate content when a partial snapshot and a block payload overlap. In current main, `extensions/feishu/src/reply-dispatcher.ts` sends block payloads through `queueStreamingUpdate(text, { mode: "delta", dedupeWithLastPartial: true })` while partial replies use snapshot merge semantics; `extensions/feishu/src/reply-dispatcher.test.ts` still expects partial `hello` plus block `lo world` to close as `hellolo world`. Implement a focused Feishu plugin fix so overlapping partial+block content closes/updates as `hello world`, preferably by reusing the existing `mergeStreamingText`/snapshot semantics or an equivalent narrowly scoped reconciliation. Preserve block-only streaming fallback, exact repeated partial/block suppression, non-overlapping block chunks, legitimate repeated text, final delivery, media delivery, and `disableBlockStreaming`. Do not change the core block-streaming pipeline, Feishu CardKit throttling/backoff, auth/config surfaces, or unrelated duplicate-final races. Update/rename the existing delta test and add regression coverage for partial+block overlap, block-only content, exact duplicate block matching the latest partial, and non-overlap block behavior.
