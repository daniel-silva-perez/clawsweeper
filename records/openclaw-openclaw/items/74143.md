## Summary

Keep open. Current main still has the reported Feishu streaming-card overlap bug: block payloads are appended with `mode: "delta"`, the dispatcher test still expects `hellolo world`, and open PR #74146 already references this issue with closing syntax, so the issue should stay open until that PR is reviewed and merged or closed.

## What This Changes

The issue requests that Feishu streaming-card block payloads be merged overlap-aware with partial snapshot text instead of appended as deltas, with corresponding dispatcher and regression-test updates.

## Best Possible Solution

Keep this issue open and review PR #74146 as the focused implementation candidate. The fix belongs in the Feishu plugin dispatcher/tests: route overlapping card-mode block payloads through the existing snapshot/`mergeStreamingText` semantics or an equivalent overlap-aware merge, then update the delta-encoded test and add regression coverage for overlapping partial snapshots plus block payloads. If PR #74146 merges, this issue can close as implemented by that PR; if it is closed unmerged, keep this issue as the canonical bug report or queue a narrow replacement.
