## Summary

Keep open. Current main still has the reported Feishu streaming-card overlap bug, the latest release tag still contains the same block delta path, and open PR #74146 explicitly closes this issue but is not merged.

## What This Changes

The issue asks OpenClaw to prevent Feishu streaming cards from duplicating overlapped text when partial snapshot replies and block payload replies both carry content for the same response.

## Best Possible Solution

Keep this issue open until PR #74146, or an equivalent narrow Feishu plugin dispatcher/test fix, lands. The desired end state is block fallback text using the existing overlap-aware snapshot merge while preserving block-only fallback delivery, with regression coverage for overlapping partial snapshots plus block payloads.

## Reproduction Assessment

Yes. Current main has a deterministic source-level reproduction: partial `hello` followed by block `lo world` is expected by the test suite to close as `hellolo world`, matching the reported duplicate-overlap behavior.

## Solution Assessment

Yes. Routing Feishu block fallback updates through the existing snapshot merge semantics is the narrowest maintainable fix because it reuses `mergeStreamingText` and keeps the change in the Feishu plugin dispatcher/tests.
