## Summary

Keep open. Current main still has the reported Feishu streaming-card overlap bug: block payloads are appended through the delta path, while partial replies use the snapshot merge path, and the current dispatcher test still expects duplicated `hellolo world` output. PR #74146 is already an open, unmerged fix candidate with closing syntax, so the issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

Fix Feishu card-mode streaming so overlapping `deliver({ kind: "block" })` payload text merges with partial snapshots instead of being concatenated into duplicate card content.

## Best Possible Solution

Feishu’s streaming-card dispatcher should treat overlapping block payloads as snapshot-style updates, or otherwise perform the same overlap-aware merge as partial replies, while preserving block-only fallback delivery and exact-duplicate suppression. The regression tests should assert `hello` plus `lo world` closes as `hello world` and cover a longer partial-snapshot plus block-overlap sequence.
