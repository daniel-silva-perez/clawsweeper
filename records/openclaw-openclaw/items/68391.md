## Summary

Keep open. Current main still has the reported silent-scope bug: the `models` parent command defines `--agent`, but the `set` and `set-image` write paths ignore that option and update the global defaults. The linked fix PR #68399 is closed and unmerged, so the issue remains valid and narrow enough for a focused repair.

## What This Changes

The issue asks for `openclaw models --agent <id> set <model>` to either update the named agent’s model override or clearly reject the unsupported scoped flag instead of changing the global default.

## Best Possible Solution

Add an explicit guard for `--agent` on `models set` and `models set-image` until those commands intentionally support per-agent writes, preserving existing `status`, `list`, and `auth` behavior and documenting the supported model-scope boundary.
