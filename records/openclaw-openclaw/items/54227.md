## Summary

Keep this issue open. Current main still consumes Discord interactive component entries by the clicked component id only, while docs and schema describe non-reusable components as single-use by default. The closed related #54233 does not supersede this remaining per-panel consumption bug.

## Best Possible Solution

Keep this issue open as the focused Discord plugin bug. Implement panel-wide consumption for non-reusable component messages, likely by assigning a shared consumption group to entries created from the same component message or panel and invalidating sibling non-reusable entries after the first authorized click. Add regression coverage for two sibling buttons where the second click returns the expired response and does not dispatch, while preserving reusable:true repeated use, unauthorized-click behavior that leaves entries active, and modal trigger/form lifecycle.
