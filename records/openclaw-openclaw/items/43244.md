## Summary

Keep this PR open. Current main still does not serialize gateway config.set/config.patch/config.apply mutations, so the PR is not obsolete or implemented on main. It remains a relevant implementation candidate, but needs a current-main rebase/reimplementation and the existing review gaps resolved before it can land.

## Best Possible Solution

Keep this PR open as the canonical gateway RPC serialization candidate. Rebase or reimplement it on current main so config.set, config.patch, and config.apply serialize the full read/base-hash-check/validate/write sequence, or pass the caller base hash into a fresh lower-level mutation that recomputes from current state before writing. Preserve current SecretRef and shared-auth behavior, add stale concurrent-write coverage for all three methods, and resolve the queue rejection review comments before landing.
