## Summary

Keep this PR open. Current main still does not implement the intended serialization for gateway `config.set`, `config.patch`, and `config.apply`, and the PR remains a relevant implementation candidate for the lost-update behavior linked from #43150. It needs rebase/reimplementation over current auth and SecretRef behavior plus the existing review gaps addressed before it can land.

## Best Possible Solution

Keep this PR open as a still-relevant implementation candidate. Rebase or reimplement it on current main so `config.set`, `config.patch`, and `config.apply` serialize the full read/base-hash-check/validate/write sequence, or pass the caller base hash into a fresh lower-level mutation that recomputes from current state before writing. Preserve current SecretRef and shared-auth behavior, add stale concurrent-write coverage for all three methods, and resolve the queue rejection review comments before landing.
