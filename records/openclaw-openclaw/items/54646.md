## Summary

Keep this PR open. Current main still has the gateway-rpc timeout validation bug this PR targets: the CLI advertises a 30000 ms default, delegates without validation, and the runtime still coerces invalid timeout strings with Number(...), allowing NaN to reach the gateway fallback path. The latest PR context also shows maintainer-side repair to keep this contributor PR canonical, so this should be reviewed or landed rather than closed as obsolete.

## Best Possible Solution

Keep this PR open for maintainer review or land it after resolving any remaining review/CI concerns. The best fix is strict positive-integer validation at the gateway-rpc CLI boundary, preserving the omitted-option default at 30000 ms, rejecting explicit empty, whitespace, and partial-numeric strings, keeping focused regression tests, and auditing shared parser effects across related CLI timeout consumers. If maintainers prefer the broader approach in #40953, explicitly supersede this PR there instead of closing it as already implemented.
