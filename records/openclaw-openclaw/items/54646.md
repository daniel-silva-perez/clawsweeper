## Summary

Keep this PR open. Current main still has the reported gateway-rpc timeout validation bug: the CLI default is 30000 ms, but malformed explicit values are still coerced to NaN and then treated as unset by the gateway call path. The PR remains an active implementation candidate with recent maintainer branch work, and the overlapping timeout-validation PRs are open rather than merged superseders.

## Best Possible Solution

Keep this PR open for maintainer review, landing, or explicit supersession. The best fix is strict positive-integer validation at the gateway-rpc CLI boundary, preserving the omitted-option default at 30000 ms, rejecting explicit empty/whitespace and partial numeric strings, and keeping shared parser consumers covered by focused regression tests. If #40953 becomes the canonical broader fix, close this PR only after that path lands or maintainers deliberately supersede it while preserving contributor credit.
