## Summary

Keep this PR open. Current main still does not recognize the standalone `Transport endpoint is not connected` systemd/DBus error in the shared systemd-unavailable classifier, so the PR’s intended availability, daemon-reload, hint, and machine-scope fallback behavior remains unimplemented. The related maintainer PRs improved adjacent user-systemd failures but explicitly excluded this PR for central security handling rather than superseding it.
