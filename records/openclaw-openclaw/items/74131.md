## Summary

Keep open. Current main still lacks the requested subagent readiness/finalize/sweep retry hardening, and this PR is still a plausible direction, but the proposed patch has blocking correctness issues: the branch imports a non-exported helper, adds a read-scoped readiness probe before higher-scope lifecycle calls, and names a sweep helper as retrying while it still performs only one delete attempt.

## What This Changes

The PR adds a subagent spawn gateway readiness preflight, wraps announce finalize `sessions.patch`/`sessions.delete` calls, wraps registry sweep deletion, and expands focused agents tests for those lifecycle paths.

## Best Possible Solution

Keep this PR open, request changes, and narrow the fix to a type-safe lifecycle retry implementation: export or centralize the transient classifier, use a readiness check that does not pair at a lower gateway scope before write/admin lifecycle calls, implement real bounded retry for registry sweep deletion, and keep focused tests for spawn readiness, announce finalize retry, and sweep cleanup retry.

## Reproduction Assessment

Yes for the PR defects: the non-exported import is reproduced by the failing `check-prod-types` annotation on the PR head, and the sweep helper/readiness-scope issues are reproducible by inspecting the PR diff against current method-scope and wrapper code. I did not run tests locally because the task is read-only.

## Solution Assessment

No. The requested direction is reasonable, but this patch is not the best mergeable solution until it fixes the export/type error, avoids the lower-scope readiness preflight, and actually retries transient sweep delete failures inside the new helper.
