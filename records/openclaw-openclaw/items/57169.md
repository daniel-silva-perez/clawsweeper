## Summary

Keep this issue open. Current main still has the reported policy mismatch: the macOS node advertises and implements `screen.record`, but the gateway default command policy treats `screen.record` as dangerous and blocks it unless `gateway.nodes.allowCommands` opts in. The linked PR #57192 is still an open implementation candidate with closing syntax, so the issue should remain open until that PR is reviewed and merged or explicitly closed.

## Best Possible Solution

Keep the issue open and resolve it through the linked implementation candidate. If maintainers want macOS `screen.record` callable by default, review/land #57192 or an equivalent patch that updates gateway policy, setup denylist behavior, tests, and docs consistently. If the intended product behavior is explicit opt-in, close or retarget #57192 and update user-facing macOS/CLI guidance so the `gateway.nodes.allowCommands: ["screen.record"]` requirement is unmistakable.
