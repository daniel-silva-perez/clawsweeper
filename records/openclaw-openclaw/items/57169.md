## Summary

Keep this issue open. Current main still has the reported default-policy mismatch: the macOS app node declares and implements screen.record, but the gateway default allowlist excludes it and the runtime rejection path still returns the reported error. The issue is also paired with open PR #57192 using closing syntax, so it should remain open until that PR or an equivalent maintainer decision lands.

## Best Possible Solution

Keep this issue open and resolve it through #57192 or an equivalent maintainer decision. If macOS screen recording should be callable by default, update the gateway policy, setup denylist behavior, tests, and docs together. If explicit opt-in is the intended product/security behavior, close or retarget #57192 and make the user-facing macOS node docs clearly say screen.record is supported but gated by gateway.nodes.allowCommands.
