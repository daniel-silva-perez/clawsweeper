## Summary

Keep open. Current main no longer treats unavailable gated core tools as plugin-only in the warning text, but the reported routed-provider execution problem is still unresolved: `apply_patch` is only constructed for exact `openai` or `openai-codex` provider IDs, and the linked policy-pipeline PR was closed unmerged after the discussion shifted back to provider/model capability gating.

## What This Changes

The issue asks OpenClaw to recognize and make `apply_patch` available for eligible agent-routed runs instead of warning about it as an unknown/plugin-only tool and preventing execution.

## Best Possible Solution

Define the `apply_patch` capability contract for routed providers, then either add a guarded provider-alias or capability override with focused tests, or keep the strict OpenAI/OpenAI Codex gate and make diagnostics/docs clearly explain why routed providers cannot receive the tool.
