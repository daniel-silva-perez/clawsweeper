## Summary

Current main still lacks the PR's central behavior: `feishu_chat` has no `list` action, no `accountId` parameter, no pagination/sort chat-directory listing, and still creates the chat client from the first enabled Feishu account. Adjacent Feishu tools already use the per-agent `createFeishuToolClient` routing pattern, so the PR remains directionally useful, but it needs maintainer review, rebase, and the unresolved disabled-account security cleanup before merge.

## Best Possible Solution

Keep this PR open for Feishu maintainer review. The best path is to rebase it onto current main, preserve existing `member_info` behavior, add the `feishu_chat` `list` action with pagination/sort forwarding, switch chat tool registration to the existing Feishu tool-account routing seam, enforce both account-level enabled status and per-account `tools.chat` at execution time, and add focused tests for list behavior, routing priority, explicit override, disabled account rejection, and disabled chat-tool rejection.
