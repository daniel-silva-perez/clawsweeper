## Summary

Keep this PR open. Current main still does not implement the PR’s central Feishu chat-tool behavior: `feishu_chat` has no `list` action, no `accountId` override, no list pagination/sort parameters, and still routes through the first enabled Feishu account rather than the existing per-agent tool-account seam. The PR remains directionally useful because adjacent Feishu tools already use `createFeishuToolClient`, but it needs maintainer review, rebase, and the remaining disabled-account routing security check before merge.

## Best Possible Solution

Keep this PR open for Feishu maintainer follow-up. The best path is to rebase it onto current main, preserve existing `member_info` behavior, add the `feishu_chat` `list` action with pagination and sort forwarding, switch chat tool registration to the existing Feishu tool-account factory pattern, reject disabled accounts and disabled per-account chat tools at execution time, and add focused tests for list behavior, routing priority, explicit override, disabled account rejection, and disabled chat-tool rejection.
