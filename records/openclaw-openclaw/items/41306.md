## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: `feishu_chat` has no `list` action and still constructs its client from the first enabled Feishu account instead of using the calling agent's `agentAccountId` or an explicit `accountId`. The PR is not a duplicate of #68443, which targets message-history listing for a specific chat/container, while this PR lists joined chats and fixes chat-tool account routing. Prior review discussion is useful: the author addressed the redundant type alias and per-account `tools.chat` check, but the later disabled-account routing concern still needs cleanup before merge.

## Best Possible Solution

Keep the PR open for Feishu plugin owner review. The best path is to rebase it onto current main, preserve the shipped `member_info` action and current TypeBox import style, implement `list` through `feishu_chat`, convert chat registration to the same `createFeishuToolClient`/tool-factory routing pattern used by wiki/drive/doc tools, and add focused tests for list pagination/sort, per-agent routing, explicit override, disabled account rejection, and disabled `tools.chat` rejection.
