## Summary

Keep this PR open. Current main still lacks the proposed `feishu_chat` `list` action and still registers the chat tool against the first enabled Feishu account instead of routing through the per-agent tool-account seam. The related open Feishu PR #68443 covers message history by container/channel id, not this chat-directory listing and per-agent routing work.

## Best Possible Solution

Keep this PR open for Feishu owner review and rebase. The best path is to add `list` to `feishu_chat`, route chat execution through the existing Feishu tool-account factory seam with `ctx.agentAccountId` plus explicit `accountId` override, preserve `member_info`, reject disabled accounts and disabled per-account chat tools, and add focused tests for listing, pagination/sort forwarding, contextual routing, explicit override, disabled-account rejection, and disabled-tool rejection. #68443 should remain separate because it tracks message-history listing, not chat directory listing or per-agent chat-tool routing.
