## Summary

Keep this PR open. Current main still lacks the proposed `feishu_chat` chat-directory `list` action and still builds the chat tool around the first enabled Feishu account instead of the existing per-agent tool-account routing seam. The PR is useful, but it needs a current-main rebase and a disabled-account execution check before it is merge-ready; open PR #68443 covers message-history listing, not this chat-directory/routing change.

## Best Possible Solution

Keep this PR open for Feishu owner review and rebase. The best path is to add `list` to `feishu_chat`, route execution through the existing Feishu tool-account seam with `ctx.agentAccountId` plus explicit `accountId` override, preserve current `member_info`, reject disabled accounts and disabled per-account chat tools, and add focused tests for listing, pagination/sort forwarding, contextual routing, explicit override, disabled-account rejection, and disabled-tool rejection. Treat #68443 as separate message-history work.
