## Summary

Keep this PR open. Current main still lacks the proposed `feishu_chat` chat-directory `list` action and still routes the chat tool through the first enabled Feishu account rather than the tool-account routing seam. The related open message-history PR is not a substitute, and this PR still needs a current-main rebase plus account-disablement security cleanup before it is merge-ready.

## Best Possible Solution

Keep this PR open for Feishu owner review and rebase. The best path is to add a `list` action to `feishu_chat`, route the chat tool through the existing Feishu tool-account seam with `ctx.agentAccountId` and explicit `accountId` override, preserve current `member_info` behavior, reject disabled accounts and disabled per-account chat tools at execution time, and add focused tests for listing, pagination/sort forwarding, contextual routing, explicit override, disabled-account rejection, and disabled-tool rejection. Treat #68443 as separate message-history work.
