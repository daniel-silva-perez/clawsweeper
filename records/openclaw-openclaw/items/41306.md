## Summary

Keep this PR open. Current main still lacks the proposed `feishu_chat` `list` action and still routes chat tool execution through the first enabled Feishu account instead of the per-agent tool-account seam. The PR is useful, but it needs a rebase and one security-sensitive account-disablement check before it is merge-ready.

## Best Possible Solution

Keep this PR open for Feishu owner review and rebase. The best path is to add `list` to `feishu_chat`, route execution through the existing Feishu tool-account seam with `ctx.agentAccountId` plus explicit `accountId` override, preserve `member_info`, reject disabled accounts and disabled per-account chat tools, and add focused tests for listing, pagination/sort forwarding, contextual routing, explicit override, disabled-account rejection, and disabled-tool rejection. The related open Feishu message-history PR should remain separate because it tracks message history by container/channel id, not chat-directory listing or per-agent chat-tool routing.
