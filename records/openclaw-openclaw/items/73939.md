## Summary

Keep this issue open. Current main still rejects custom top-level fields under `skills.entries.<skill>`, and an open unmerged PR (#74024) is already linked as the implementation candidate for this exact issue. The reported reload rollback path is partly stale because `config.patch` now validates before write, but the requested capability is still not implemented.

## Best Possible Solution

Keep the issue open while maintainers review, land, or close #74024. If direct custom skill fields are accepted, the fix should update `SkillEntrySchema`, generated schema output, focused schema tests, docs, and any redaction/secret metadata needed for custom fields. If maintainers prefer the stricter contract, the docs should clearly route custom data to `skills.entries.<skill>.config` and route Tavily users to `plugins.entries.tavily.config.webSearch.*`.
