  - body: For `gateway.bind="netbird"`, the PR seeds HTTPS localhost origins but not the actual `https://<netbird-ip>:<port>` origin the browser will send. Because Control UI origin checks exact-match scheme, host, and port, default NetBird setups can still fail with `origin not allowed` unless the detected NetBird origin is seeded or the mode requires explicit `allowedOrigins`.
  - confidence: 0.89

  - body: Current main re-exports `resolveControlUiLinks` from `src/gateway/control-ui-links.ts`, and dashboard/status/setup call that shared helper. Updating the older onboarding helper body leaves current Control UI URLs without NetBird host/TLS behavior after refresh, so the implementation needs to move to the shared helper and its current callers/tests.
  - confidence: 0.86
- **[P2] Load the auto-TLS fingerprint for status probes:** `src/cli/daemon-cli/status.gather.ts:199-200`
  - body: The PR makes default NetBird probes use `wss://`, but the status probe TLS-runtime path still only loads a fingerprint for explicit `gateway.tls.enabled`. A default NetBird setup uses a generated self-signed cert, so `openclaw daemon status --probe` can fail certificate verification unless it loads the same effective TLS config.
  - confidence: 0.87

  - confidence: 0.78

  - body: The PR updates the runtime enum/help text but does not include the generated config schema/help artifact updates. That leaves schema consumers and config docs stale until `src/config/schema.base.generated.ts` and related generated hashes are refreshed.
  - confidence: 0.82
