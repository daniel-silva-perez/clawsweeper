## Summary

Keep this PR open. Current main still does not implement or document `gateway.bind: "netbird"`, and the current plugin surface does not expose a gateway bind-mode registration path that would make this a ClawHub-only cleanup. The PR is stale and security-sensitive, but it remains a plausible implementation candidate for the NetBird gateway request rather than something conservative cleanup should close.

## Best Possible Solution

Keep this PR open for maintainer review and author refresh. If OpenClaw wants first-class NetBird support, implement it in core gateway/config/docs/tests because current plugin surfaces do not expose gateway bind-mode registration: update the bind enum/schema/help, NetBird interface detection, bind resolver, status/probe paths, auto-TLS runtime and fingerprint handling, Control UI links/origins, onboarding, docs, and regression tests. If maintainers prefer not to add service-specific bind modes, open a deliberate gateway bind extension/API design instead of closing this implementation as obsolete.
