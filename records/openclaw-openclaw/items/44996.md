## Summary

Keep this PR open. Current main at 17811480dae20f65b526454d75e971ae8424ec10 still does not implement or document `gateway.bind: "netbird"`: the bind type and strict schema omit it, the gateway resolver only handles loopback/lan/tailnet/custom/auto, TLS remains config-driven, and CLI/docs still list only the existing bind modes. The PR remains a plausible implementation candidate for the NetBird gateway request rather than a conservative cleanup close.

## Best Possible Solution

Keep this PR open for maintainer review and author refresh. If OpenClaw wants first-class NetBird support, implement it in core gateway/config/docs/tests because current plugins cannot register gateway bind modes. If maintainers prefer service-specific VPN support outside core, first design a gateway bind-mode extension API, then move the NetBird implementation to a plugin or ClawHub path.
