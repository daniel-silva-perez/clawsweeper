## Summary

Keep this PR open. Current main still does not implement or document `gateway.bind: "netbird"`, and the current plugin surface does not provide a gateway bind-mode registration path that would make this a ClawHub-only cleanup. The PR remains relevant, but the proposed patch needs maintainer review and refresh because it still has blocking default NetBird Control UI and status-probe issues.

## What This Changes

The PR adds a `gateway.bind: "netbird"` mode across config/schema, gateway bind resolution, auto-TLS, CLI/status, onboarding, Control UI links/origins, tests, and changelog using `wt*` NetBird interface detection.

## Best Possible Solution

Refresh this PR or a narrow replacement against current main, fix the NetBird origin and auto-TLS fingerprint gaps, update the current gateway/control-ui/status/onboarding/docs/generated-schema surfaces, and land it only if maintainers want first-class core NetBird support. If the product direction is to avoid service-specific core bind modes, the better end state is a gateway bind-mode extension API followed by a plugin implementation.

## Security Review

Status: needs_attention

Summary: No dependency, workflow, lockfile, or package-publishing change was found, but the patch changes gateway network exposure and uses a weak NetBird interface attribution heuristic that needs maintainer security review.

Concerns:

- **[medium] Strengthen NetBird interface attribution:** `src/infra/netbird.ts:11`
  - body: The added detector treats any non-internal interface whose name starts with `wt` as NetBird before binding the Gateway and auto-enabling TLS.
