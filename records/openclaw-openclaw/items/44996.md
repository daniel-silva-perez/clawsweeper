## Summary

Keep this PR open. Current main still does not implement the requested NetBird Gateway bind mode, and this PR remains the canonical implementation candidate for the closed NetBird feature request, but the patch still needs maintainer review because it changes Gateway network exposure and has blocking origin/status/link-surface defects before merge.

## What This Changes

The PR adds a `gateway.bind: "netbird"` mode across Gateway config/schema, CLI/status/onboarding surfaces, NetBird `wt*` interface detection, bind resolution, auto-generated self-signed TLS, tests, and changelog.

## Best Possible Solution

Refresh this PR against current main, implement NetBird support in the current shared Gateway bind/link/status/origin paths, seed or explicitly require the actual NetBird HTTPS origin, load the generated TLS fingerprint anywhere auto-TLS is used, and land only after maintainers accept the NetBird interface trust boundary. If core should avoid service-specific VPN bind modes, keep the user problem tracked while designing a generic VPN bind profile or plugin seam.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Seed the detected NetBird origin:** `src/config/gateway-control-ui-origins.ts:35-38`
  - body: For `gateway.bind="netbird"`, this seeds HTTPS localhost origins but the Control UI is meant to open on the detected NetBird IP. `checkBrowserOrigin()` exact-matches origins, so `https://<netbird-ip>:<port>` can still fail with `origin not allowed` unless the detected VPN IP origin is seeded or explicitly required.
  - confidence: 0.88
- **[P2] Load the auto-TLS fingerprint for status probes:** `src/cli/daemon-cli/status.gather.ts:198-200`
  - body: The status summary switches NetBird probes to `wss://`, but the later fingerprint load still only follows explicit `gateway.tls.enabled`. Default NetBird setups auto-generate a self-signed cert, so `openclaw daemon status --probe` can fail certificate verification without a pinned fingerprint.
  - confidence: 0.84
- **[P2] Update the current shared link helper:** `src/commands/onboard-helpers.ts:459-480`
  - body: This branch updates an older onboarding link implementation, but current main routes dashboard, status, setup, and onboarding output through `src/gateway/control-ui-links.ts`. After rebasing, that shared helper still needs NetBird host/TLS handling or those surfaces will keep printing/probing the wrong scheme or host.
  - confidence: 0.78
- **[P3] Add the required changelog attribution:** `CHANGELOG.md:9`
  - body: The added user-facing changelog bullet is missing the repository-required `Thanks @...` attribution. Add the reporter or contributor credit on the same single-line entry.
  - confidence: 0.93

## Security Review

Status: needs_attention

Summary: No dependency, workflow, lockfile, or package-publishing change was found, but the patch changes Gateway network exposure and relies on a weak NetBird interface attribution heuristic.

Concerns:

- **[medium] Strengthen NetBird interface attribution:** `src/infra/netbird.ts:14`
  - body: The new detector treats any non-internal interface whose name starts with `wt` as NetBird before binding the Gateway and enabling TLS.
