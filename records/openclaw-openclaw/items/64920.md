## Summary

Keep open. Current main still serializes Feishu inbound messages by account and chat only, so the PR’s core bug fix is not implemented on main. The PR appears useful, and the discussion shows the author addressed real edge cases, but the branch also contains unrelated runtime/SDK/test changes and should be split or rebased before maintainer review.

## Best Possible Solution

Keep the PR open, but ask for a focused rebase/split: land only the Feishu sequential-key change adapted to current main, preserve chat-wide `/stop` and `/btw` lanes, keep DMs and non-topic groups on chat-wide FIFO, derive topic lanes only for topic-scoped group sessions, and add direct sequential-key regression tests for different topic IDs in the same chat. Move the unrelated CI/runtime/SDK cleanup into separate PRs or drop it from this branch.
