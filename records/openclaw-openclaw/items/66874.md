## Summary

Keep open. Current main still supports the Anthropic Claude CLI backend, but the official Docker image and release workflow do not install Claude Code, and the Docker/provider/backend docs still do not give Docker users a specific persistent `claude` install plus `~/.claude` credential recipe.

## What This Changes

The issue requests that the official Docker image either include the Claude Code CLI or document a persistent Docker install and credential path for the Claude CLI backend.

## Best Possible Solution

Add Docker-specific Claude CLI guidance that covers persistent binary installation, `~/.claude` credential persistence, explicit `agents.defaults.cliBackends.claude-cli.command` plus `agentRuntime.id: "claude-cli"` config, onboarding verification, and Anthropic API-key fallback; treat image preinstallation as a separate maintainer decision.
