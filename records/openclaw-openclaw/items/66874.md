## Summary

Keep open. Current main still supports the Anthropic Claude CLI backend, but the official Docker image/release path does not preinstall Claude Code, and the Docker/provider/backend docs still do not give Docker users a persistent `claude` install plus `~/.claude` credential recipe. The existing ClawSweeper comments remain accurate; related #2440 is credential-detection history, not a canonical tracker for this Docker image/docs gap.

## What This Changes

The issue requests either preinstalling Claude Code in the official Docker image or documenting a persistent Docker install and credential path for the Anthropic Claude CLI backend.

## Best Possible Solution

Add Docker-specific Claude CLI guidance covering persistent binary installation, `~/.claude` credential persistence, explicit `claude-cli.command` plus `agentRuntime.id: "claude-cli"` config, onboarding verification, and Anthropic API-key fallback. Treat image preinstallation as a separate maintainer decision.

## Repair Work Prompt

Fix the Docker Claude CLI documentation gap for openclaw/openclaw#66874 without changing the official Docker image, package metadata, lockfiles, or release workflow. Add Docker-specific guidance for using the bundled Anthropic `claude-cli` backend when `claude` is not preinstalled: choose a persistent binary/npm prefix or mounted path, preserve Claude Code state under `~/.claude` using a named `/home/node` volume or explicit bind mount/symlink, show an `agents.defaults.agentRuntime.id: "claude-cli"` config plus `agents.defaults.cliBackends.claude-cli.command` pointing at the persisted binary, include onboarding/verification commands, and mention Anthropic API key as the operational fallback. Keep the wording in user-facing docs as `plugin/plugins`, not `extension/extensions`. Treat preinstalling `@anthropic-ai/claude-code` in the image as out of scope unless a maintainer separately approves it. No changelog entry is expected for this docs-only clarification unless maintainers request one.
