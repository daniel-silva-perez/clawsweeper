## Summary

Keep this open. Current main still supports the Anthropic Claude CLI backend, but the official Docker image and Docker release workflow do not install Claude Code, and the Docker/provider/backend docs still lack a Docker-specific persistent install plus `~/.claude` credential recipe. The prior ClawSweeper keep-open review remains accurate; related #2440 is adjacent credential-detection history, not a canonical tracker for Docker image provisioning or persistence docs.

## What This Changes

Request that official Docker support Claude Code CLI either by installing a pinned `@anthropic-ai/claude-code` in the image or documenting a persistent Docker install and credential path.

## Best Possible Solution

Add Docker-specific Claude CLI guidance that covers persistent installation, credential persistence, command override config, onboarding verification, and API-key fallback; treat pre-installing Claude Code in the official image as a separate maintainer decision because it changes the release artifact.

## Repair Work Prompt

Create a narrow docs-only PR for openclaw/openclaw#66874. Do not change `Dockerfile`, Docker release workflows, package dependencies, lockfiles, or runtime behavior. Add a Docker-specific Claude CLI section to `docs/install/docker.md`, cross-link it from `docs/providers/anthropic.md` and `docs/gateway/cli-backends.md`, and keep terminology as plugin/plugins where relevant. The docs should clearly say the official image does not currently include Claude Code, show a persistent install approach for `@anthropic-ai/claude-code` using a Docker-persisted location such as `OPENCLAW_HOME_VOLUME` or a mounted npm prefix, explain how to preserve Claude Code OAuth state under `/home/node/.claude`, include an `agents.defaults.agentRuntime.id: "claude-cli"` plus `agents.defaults.cliBackends.claude-cli.command` example for the installed binary path, and include API-key fallback/rollback guidance. Verify npm prefix/bin paths against current npm/Claude Code behavior before documenting exact commands. No changelog entry is expected for docs-only guidance unless maintainers request one.
