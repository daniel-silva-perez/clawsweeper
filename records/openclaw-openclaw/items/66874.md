## Summary

Keep open. Current main still supports the Anthropic Claude CLI backend, but the official Docker image/release workflow does not install Claude Code and the Docker/provider/backend docs still do not give Docker users a persistent `claude` install plus `~/.claude` credential recipe. Related #2440 is adjacent credential-detection history, not a canonical tracker for this Docker image/docs gap.

## What This Changes

The issue requests that the official Docker image either include the Claude Code CLI or document a persistent Docker install and credential path for the Claude CLI backend.

## Best Possible Solution

Add Docker-specific Claude CLI guidance covering persistent binary installation, `~/.claude` credential persistence, explicit `agents.defaults.cliBackends.claude-cli.command` plus `agentRuntime.id: "claude-cli"` config, onboarding verification, and Anthropic API-key fallback. Treat official-image preinstallation as a separate maintainer decision.

## Reproduction Assessment

Not applicable as a failing runtime reproduction because this is a feature/docs gap. The current-main source and docs verify the reported gap by inspection: the Docker image path does not install `claude`, while the docs require a host-local logged-in Claude CLI without Docker persistence steps.

## Solution Assessment

Yes, a docs-only Option B repair is the narrowest maintainable next step for this issue. Pre-installing Claude Code in the official image may still be desirable later, but it should not be bundled into an automated cleanup because it changes supply-chain and release-artifact policy.
