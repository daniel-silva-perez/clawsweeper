## Summary

Keep open. Current main still matches both reported failure modes: OpenClaw-managed Serve re-applies `tailscale serve --bg --yes <port>` during gateway post-attach startup with no Funnel preservation/status check, and OpenClaw-managed Funnel still rejects token auth with the exact password-only error. The relevant gateway, Tailscale, config schema, and docs files are unchanged from the prior ClawSweeper keep-open review to current `main`.

## Best Possible Solution

Keep this issue open for maintainer product/security handling. A fix should either preserve externally managed Funnel routes when OpenClaw runs in `gateway.tailscale.mode: "serve"`, document and validate that manual Funnel management requires `gateway.tailscale.mode: "off"`, or allow token auth for OpenClaw-managed Funnel only after an explicit public-exposure security review. Add focused tests for the chosen startup/config behavior and update `https://docs.openclaw.ai/gateway/tailscale`.
