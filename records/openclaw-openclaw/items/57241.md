## Summary

Keep open. Current main still matches both reported failure modes: Serve mode is started on every gateway startup with `tailscale serve --bg --yes <port>` and no preservation/status check for existing Funnel paths, while Funnel mode still rejects token auth with the reported password-only error. Official Tailscale behavior supports the overwrite concern because the most recent Serve/Funnel command controls whether a port is private or public.

## Best Possible Solution

Keep this issue open for maintainer product/security handling. A fix should either preserve externally managed Funnel routes when OpenClaw is in `serve` mode, document and validate that manual Funnel management requires `gateway.tailscale.mode: "off"`, or deliberately change the Funnel auth policy to allow token auth only after an explicit public-exposure security review. Add focused tests for the selected startup/config behavior and update https://docs.openclaw.ai/gateway/tailscale.
