## Summary

Keep open. Current main still matches both reported failure modes: `gateway.tailscale.mode="serve"` runs `tailscale serve` on each gateway startup, and `gateway.tailscale.mode="funnel"` still rejects token auth with the exact password-only error. Official Tailscale docs also confirm that the latest Serve/Funnel command controls whether the same port is private or public, so the startup Serve command can plausibly overwrite a manually configured Funnel path.

## Best Possible Solution

Keep this issue open for maintainer product/security handling. A fix should either add a safe owner-controlled way to preserve externally managed Funnel routes when OpenClaw is in `serve` mode, document and validate that manual Funnel requires `gateway.tailscale.mode: "off"`, or deliberately change the Funnel auth policy if token-auth public exposure is accepted. The password-only Funnel requirement appears intentional today, so that part needs explicit maintainer judgment rather than cleanup closure.
