## Summary

Keep open. Current main still has the incompatible behavior, while the updated PR head is an active implementation against the right memory-lancedb surface. This should continue through normal maintainer review and validation rather than being closed, replaced, or marked implemented.

## What This Changes

The PR updates memory-lancedb's OpenAI-compatible embedding path, config parser, manifest schema, and tests so configured dimensions are not sent upstream and returned vectors are truncated locally with non-positive dimension checks.

## Best Possible Solution

Land a narrow memory-lancedb fix that keeps the current direct /embeddings transport, proxy setup, timeout/retry behavior, and response normalization, while omitting upstream dimensions, locally truncating normalized vectors, and validating configured dimensions consistently across runtime config and manifest metadata.
