## Summary

Keep open. Current main still forwards configured `embedding.dimensions` in the memory-lancedb OpenAI-compatible embeddings request and still has a regression test expecting that outbound field, while this PR is a narrow live fix for that behavior. The updated branch is extension-only and security-sensitive surfaces are not touched, but exact-head CI currently has failing check runs and maintainers should review the remaining dimensions-validation detail before merge.

## What This Changes

This PR changes memory-lancedb's OpenAI-compatible embedding path to omit `dimensions` from `/embeddings` requests, truncate returned vectors locally, and add regression tests for local truncation, short vectors, and non-positive dimensions.

## Best Possible Solution

Land a narrow memory-lancedb fix that omits `dimensions` on OpenAI-compatible requests, preserves the current direct `client.post("/embeddings")` transport with proxy, timeout/retry, and response normalization behavior, locally truncates after validating configured dimensions, and keeps focused regression coverage for omitted-field, truncation, and failure paths.
