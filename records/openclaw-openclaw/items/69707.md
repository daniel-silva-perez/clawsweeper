## Summary

Keep open. The PR targets a valid memory-lancedb compatibility bug, but current main still forwards configured `embedding.dimensions` in the OpenAI-compatible `/embeddings` request and still has a regression test expecting that outbound field. The PR also patches the older `client.embeddings.create()` shape, while current main now uses direct `client.post("/embeddings")` plus response normalization, timeout, and retry behavior that a rebase must preserve.

## Best Possible Solution

Keep this PR open and update it against current main. The fix should remove `dimensions` from the direct `/embeddings` request body, preserve `ensureGlobalUndiciEnvProxyDispatcher`, `client.post`, timeout/maxRetries, and `normalizeEmbeddingVector`, then validate and locally truncate the normalized vector when `embedding.dimensions` is configured. Focused regression coverage should replace the current outbound-dimensions assertion with omitted-field, local-truncation, short-vector failure, and invalid-dimension behavior.
