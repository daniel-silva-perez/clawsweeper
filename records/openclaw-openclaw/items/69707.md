## Summary

Keep this PR open. Current main still forwards configured `dimensions` in the `memory-lancedb` OpenAI-compatible embedding request and has a test that asserts that behavior, so the PR is not implemented on main. The PR is also now behind main: current main moved the path to direct `/embeddings` transport plus `normalizeEmbeddingVector`, while this PR's patch is based on the older `embeddings.create` shape and GitHub reports it as dirty.

## Best Possible Solution

Keep the PR open and ask for a current-main rebase. The rebase should update `OpenAiCompatibleEmbeddings.embed` so the `/embeddings` request body contains only `model` and `input`, preserve existing `client.post` timeout/maxRetries behavior and `normalizeEmbeddingVector`, then locally validate/truncate the normalized vector to configured `embedding.dimensions`. Keep focused regression coverage in `extensions/memory-lancedb/index.test.ts`; docs only need a small note if maintainers want to clarify that `dimensions` is local for OpenAI-compatible providers.
