## Summary

Keep open for normal maintainer review. Current main still forwards configured memory-lancedb embedding dimensions upstream, while this PR head addresses that compatibility bug narrowly and I did not find a PR-introduced functional or security finding; exact-head CI still needs maintainer follow-up before merge.

## What This Changes

The PR changes memory-lancedb OpenAI-compatible embedding requests to omit configured dimensions, truncates normalized returned vectors locally, validates non-positive dimensions in config and manifest metadata, and updates focused regression tests.

## Best Possible Solution

Land a narrow memory-lancedb fix that preserves the direct /embeddings transport, proxy setup, timeout/retry behavior, and response normalization while omitting upstream dimensions, truncating locally after validation, and keeping config, manifest metadata, and regression tests aligned.

## Reproduction Assessment

Yes. Current main has a source-level reproduction and a regression test that still expects dimensions in the outbound request; the PR head adds focused tests for omitted dimensions, local truncation, short-vector failure, and invalid dimensions.

## Solution Assessment

Yes. The PR direction is the narrow maintainable fix for OpenAI-compatible providers that reject dimensions: keep the existing direct transport and response normalization, remove only the incompatible outbound field, and locally enforce the configured vector length.
