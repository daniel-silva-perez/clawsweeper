## Summary

Keep open. Current main implements a shipped subset of the request: Gemini Embedding 2 can index image/audio files from configured `memorySearch.extraPaths` as structured multimodal inputs, with provider and fallback gating. The issue still asks for meaningful unimplemented scope: raw PDF/video embeddings and a safe workflow for media sent to an agent and marked worth remembering. The prior ClawSweeper comment reached the same conclusion, and current main still matches it.

## What This Changes

Request support for indexing raw non-Markdown media, including images, video, audio, and PDFs, in Gemini Embedding 2-backed memory search.

## Best Possible Solution

Keep this issue open and narrow the remaining scope: decide whether OpenClaw should add PDF/video modalities to the memory indexing contract, and separately design a safe remembered-inbound-media ingestion path. The shipped Gemini image/audio `extraPaths` implementation should remain the foundation; #44734 can continue tracking the narrower OGG/Opus audio-format regression.
