## Summary

Keep open. Current main implements only the narrower Gemini Embedding 2 image/audio indexing path for files discovered through memorySearch.extraPaths; PDF/video modalities and an explicit user-sent-media remember workflow remain outside the current public memory contract.

## What This Changes

Request multimodal memory indexing for images, video, audio, and PDFs with Gemini Embedding 2, including media sent to an agent and marked worth remembering.

## Best Possible Solution

Keep the shipped image/audio extraPaths foundation, then split the remaining work into a PDF/video multimodal memory contract and a separate safe persisted-attachment ingestion design for media users explicitly ask the agent to remember.
