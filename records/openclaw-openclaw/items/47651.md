## Summary

Keep open. Current main ships the narrower Gemini Embedding 2 path for opt-in image and audio files under `memorySearch.extraPaths`, but the requested PDF/video modalities and user-sent-media ingestion workflow are still missing from the public memory contract.

## What This Changes

The issue requests Gemini Embedding 2 multimodal memory indexing for user-sent and file-backed images, video, audio, and PDFs so media can be stored and retrieved through memory search.

## Best Possible Solution

Keep the shipped image/audio `extraPaths` foundation, then split the remaining work into a public PDF/video modality contract and a separate safe persisted-media ingestion path for attachments users explicitly ask the agent to remember.
