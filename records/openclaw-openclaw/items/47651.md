## Summary

Keep open. Current main implements a useful subset of the request: Gemini Embedding 2 can index image/audio files from configured memory extra paths as structured multimodal inputs and return those paths through memory search. The issue still asks for meaningful unimplemented scope: PDF/video raw media embeddings and a remembered-inbound-media workflow for attachments sent to an agent.

## Best Possible Solution

Keep this issue open and narrow the remaining scope: decide whether OpenClaw should support raw PDF/video embeddings and define a safe remembered-inbound-media ingestion path. The shipped Gemini image/audio extraPaths implementation should remain the foundation, while #44734 handles the narrower current audio-format regression.
