## Summary

Keep open. Current main implements a useful Gemini Embedding 2 subset for opt-in image and audio indexing from configured extra paths, but the request still has meaningful missing scope: PDF/video modalities and a safe workflow for user-sent media that should become searchable memory. The related #44734 is narrower audio-format fallout, not a complete tracker for the remaining feature.

## What This Changes

Support Gemini Embedding 2 native multimodal memory indexing for non-Markdown media, including images, video, audio, PDFs, and media users ask the agent to remember.

## Best Possible Solution

Keep the shipped image/audio extraPaths foundation, then explicitly split the remaining scope into a public PDF/video modality contract and a separate safe persisted-media ingestion path for attachments users ask the agent to remember. Leave #44734 focused on the OGG/Opus audio-format regression.
