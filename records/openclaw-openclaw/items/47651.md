## Summary

Keep open. Current main has the narrower Gemini Embedding 2 path for opt-in image/audio files discovered through `memorySearch.extraPaths`, but PDF/video modalities and a user-sent-media remember workflow remain outside the public memory contract.

## What This Changes

Support Gemini Embedding 2 multimodal memory indexing for raw images, video, audio, PDFs, and user-sent media marked worth remembering, with media matches retrievable through `memory_search`.

## Best Possible Solution

Keep the image/audio `extraPaths` foundation, then design PDF/video support as an explicit memory modality contract and handle user-sent attachments through a separate opt-in retained-media ingestion path with clear privacy and provider-upload rules.

## Reproduction Assessment

Not applicable as a bug reproduction. There is a high-confidence current-main verification path by inspecting the modality classifier, schema, docs, tests, and attachment staging code, which show image/audio `extraPaths` support only.

## Solution Assessment

No, the current shipped subset is not a complete solution for this request. The maintainable path is an explicit public contract for new media modalities and retained attachments, rather than silently indexing every inbound file.
